饱和模型提供了一种控制结构内部实现哪些类型的方法，并且它们充当规范的大型模型，其中已经实现了小参数集的所有一致描述，因此不需要进一步扩展来见证这些类型。 

在本节中，让$L$是一阶语言，让$T$成为一个$L$-理论，并让$\mathcal M \models T$成为一个$L$- 具有域的结构$M$。 我们研究子集上的类型如何$M$在里面表现$\mathcal M$。 

### 定义 6.52（集合上类型的实现）

 让$A \subseteq M$，并让$p(x)$是部分类型$A$。 我们这么说$p(x)$实现于$\mathcal M$如果存在$a \in M^n$这样：$$
\mathcal M \models \varphi(a)
$$对于每个公式$\varphi(x) \in p(x)$。 

该定义将早期的实现概念专门用于环境结构固定的情况，并且强调实现取决于类型和结构。 

### 定义 6.53（饱和模型）

 让$\kappa$成为无限基数。 结构$\mathcal M$据说是$\kappa$-饱和如果对于每个子集$A \subseteq M$和：$$
|A| < \kappa,
$$每种类型$p(x)$超过$A$这符合$\operatorname{Th}(\mathcal M)$实现于$\mathcal M$。 

因此$\kappa$- 饱和度意味着$\mathcal M$实现大小严格小于的参数集上的所有类型$\kappa$，前提是这些类型是一致的。 

什么时候$\kappa = |M|$，该模型简称为饱和模型。 

### 解释

 饱和模型包含与其所满足的理论相关的尽可能多的元素，从某种意义上说，对小参数集的任何一致描述都已经在模型内部有了见证，因此不需要扩展来实现该描述。 

### 示例 6.54

 让：$$
\mathcal N = (\mathbb N; <).
$$考虑类型结束$\mathbb N$:$$
p(x) = \{x > n : n \in \mathbb N\}.
$$这种类型与理论是一致的$(\mathbb N; <)$并且是有限可满足的，但它并没有在$\mathbb N$本身，因为没有自然数大于所有自然数。 

所以$(\mathbb N; <)$不是$\aleph_0$-饱和。 

### 示例 6.55

 让$\mathcal M$是一个非标准的算术模型。 然后$\mathcal M$包含大于每个标准自然数的元素，这些元素实现以下类型：$$
\{x > n : n \in \mathbb N\}.
$$因此，非标准模型在类型的实现方面更加丰富，并且足够大的模型可以饱和。 

### 引理 6.56（饱和意味着有限可满足类型的实现）

 让$\mathcal M$是$\kappa$- 饱和，让$A \subseteq M$和$|A| < \kappa$，并让$p(x)$成为一种类型$A$是有限满足的$\mathcal M$。 然后$p(x)$实现于$\mathcal M$。 

证明

 自从$p(x)$是有限满足的$\mathcal M$，每个有限子集$p(x)$实现于$\mathcal M$。 由紧致性可知$p(x)$符合$\operatorname{Th}(\mathcal M)$。 

自从$\mathcal M$是$\kappa$-饱和和$|A| < \kappa$，每种类型结束$A$这符合$\operatorname{Th}(\mathcal M)$实现于$\mathcal M$。 所以$p(x)$实现于$\mathcal M$。 

### 定义 6.57（可数饱和模型）

 一个模型$\mathcal M$是可数饱和的，如果是$\aleph_1$-饱和，即有限或可数参数集上的每种类型都实现为$\mathcal M$。 

这个概念在分析和代数中特别重要，其中许多结构都涉及可数参数集。 

### 引理 6.58（饱和模型的扩展）

 让$\mathcal M$成为理论的模型$T$，并让$\kappa$成为无限基数。 那么存在一个初等扩展$\mathcal N \succ \mathcal M$这样$\mathcal N$是$\kappa$-饱和。 

证明

 我们构建$\mathcal N$通过超限过程，逐步添加类型的实现。 

枚举子集上的所有类型$M$尺寸小于$\kappa$，直到逻辑等价，如：$$
(p_\alpha : \alpha < \lambda),
$$在哪里$\lambda$是一个足够大的序数。 

我们构建了一系列基本扩展：$$
\mathcal M = \mathcal M_0 \prec \mathcal M_1 \prec \mathcal M_2 \prec \cdots
$$在阶段$\alpha$，如果类型$p_\alpha$尚未实现$\mathcal M_\alpha$，然后延伸$\mathcal M_\alpha$到一个模型$\mathcal M_{\alpha+1}$其中$p_\alpha$被实现了。 这是可能的，因为$p_\alpha$与理论是一致的。 

在极限阶段，取并集：$$
\mathcal M_\beta = \bigcup_{\alpha < \beta} \mathcal M_\alpha.
$$基本链的并集又是每个阶段的基本扩展。 

在构建结束时，定义：$$
\mathcal N = \bigcup_{\alpha < \lambda} \mathcal M_\alpha.
$$通过构造，每个类型的大小子集小于$\kappa$实现于$\mathcal N$， 所以$\mathcal N$是$\kappa$-饱和。 

### 引理 6.59（来回法）

 让$\mathcal M$和$\mathcal N$是两个$\kappa$-同一完整理论的饱和模型$T$， 和：$$
|M| = |N| = \kappa.
$$然后$\mathcal M$和$\mathcal N$是同构的。 

证明

 我们构造一个同构$\mathcal M$和$\mathcal N$通过来回论证。 

枚举：$$
M = \{a_\alpha : \alpha < \kappa\},
\quad
N = \{b_\alpha : \alpha < \kappa\}.
$$我们构建一系列部分同构：$$
f_\alpha : M_\alpha \to N_\alpha,
$$在哪里$M_\alpha \subseteq M$和$N_\alpha \subseteq N$是有限集或小集。 

在每一步中，我们都会扩展映射以包含一个结构中的下一个元素，使用饱和度在另一个结构中查找满足已匹配元素相同类型的匹配元素。 

更准确地说，假设$f_\alpha$被定义。 将其扩展为包括$a_\alpha$，考虑类型$a_\alpha$超过$M_\alpha$，并通过以下方式传输此类型$f_\alpha$到一个类型结束$N_\alpha$。 自从$\mathcal N$是$\kappa$-饱和，该类型实现于$\mathcal N$，所以我们可以选择$b$在$N$匹配$a_\alpha$。 

同样，我们向另一个方向延伸以确保满射性。 

在极限阶段，取部分地图的并集。 

最后，我们得到一个双射：$$
f : M \to N
$$保留所有关系和功能，因此是同构。 

### 饱和度和清晰度

 饱和度与可定义性密切相关。 在饱和模型中，小参数集上的每个一致描述都对应于模型的实际元素，因此可以将类型视为点。 

特别地，可定义集合可以从类型的角度来理解：可定义集合对应于全部包含特定公式的类型的集合，并且在饱和模型中，每个这样的类型都由元素实现。 

这种视角允许人们在由公式和类型给出的句法描述与由模型的元素和子集给出的语义对象之间自由移动。 

### 备注 6.60

 饱和模型通常被用作理论的规范代表，因为它们消除了小尺寸引起的意外限制，并且它们确保小参数集上的所有逻辑上可能的配置已经存在于模型内部。 

在实践中，模型论中的许多论点通过在足够饱和的模型中工作而得到简化，其中类型可以在不离开结构的情况下实现。
