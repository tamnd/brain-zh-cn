## 11.4 停机问题

 **停止问题**询问图灵机最终是否停止。 

给定一台机器$M$和一个输入词$w$，我们问：$$
\text{Does } M \text{ halt on input } w?
$$这里的“halt”是指计算最终达到停止状态，例如：$$
q_{\mathrm{accept}}
$$或者：$$
q_{\mathrm{reject}}
$$如果$M$经过有限步后达到这些状态之一，然后$M$停止于$w$。 如果计算永远持续下去，那么$M$不停止$w$。 

### 停止的语言

 我们可以将停机问题定义为一种语言：$$
\mathrm{HALT}_{\mathrm{TM}}
===========================
{\langle M,w\rangle : M \text{ halts on input } w}
$$输入是一个编码：$$
\langle M,w\rangle
$$在哪里$M$是一个图灵机并且$w$是一个输入词。 

问题是这个编码对是否属于$\mathrm{HALT}_{\mathrm{TM}}$。 

### 可识别但不可判定

 停止问题是**可识别的**。 

有一个图灵机可以识别$\mathrm{HALT}_{\mathrm{TM}}$:

 1. 给定$\langle M,w\rangle$, 模拟$M$在$w$。 
2.如果$M$停止，接受。 
3.如果$M$永远运行，永远保持模拟。 

所以可以发现阳性病例。 如果$M$真的停了下来，模拟最终看到了它。 

但负面案例才是问题所在。 如果$M$继续运行，可能没有有限的时刻我们可以安全地说：$$
M \text{ will never halt}
$$主要定理表明，没有算法可以解决所有机器和输入的问题。 

### 定理

 没有图灵机可以决定：$$
\mathrm{HALT}_{\mathrm{TM}}
$$同样，没有算法能够始终确定任意机器是否因任意输入而停止。 

要成为决策者，机器必须在每次输入时停止并正确回答：

 | 案例 | 正确答案 |
 | ------------------------ | -------------- |
 |$M$停止于$w$| 接受|
 |$M$不停止$w$| 拒绝|

 该定理表明这样的决策者不存在。 

### 证明想法

 为了矛盾，假设有一个决策者：$$
H
$$对于停机问题。 

然后$H$接受输入：$$
\langle M,w\rangle
$$并且总是停止。 

它回答：$$
H(\langle M,w\rangle)=
\begin{cases}
\mathrm{accept}, & \text{if } M \text{ halts on } w,\
\mathrm{reject}, & \text{if } M \text{ does not halt on } w.
\end{cases}
$$现在我们建造一台新机器$D$使用$H$作为子程序。 

输入时$\langle M\rangle$, 机器$D$这样做：

 1. 跑步$H$在$\langle M,\langle M\rangle\rangle$。 
2.如果$H$说$M$停止在自己的代码上，然后永远循环。 
3.如果$H$说$M$不会停止自己的代码，然后停止。 

所以$D$行为与什么相反$H$预测自我应用。 

用符号表示：$$
D(\langle M\rangle)=
\begin{cases}
\text{loop forever}, & \text{if } H(\langle M,\langle M\rangle\rangle)=\mathrm{accept},\
\text{halt}, & \text{if } H(\langle M,\langle M\rangle\rangle)=\mathrm{reject}.
\end{cases}
$$现在询问当$D$运行在自己的代码上：$$
D(\langle D\rangle)
$$有两种情况。 

首先，假设$D$停止于$\langle D\rangle$。 

然后$H$必须说的是$D$停止于$\langle D\rangle$。 

但根据定义$D$， 如果$H$这么说，那么$D$永远循环。 

矛盾。 

其次，假设$D$不停止$\langle D\rangle$。 

然后$H$必须说的是$D$不停止$\langle D\rangle$。 

但根据定义$D$， 如果$H$这么说，那么$D$停止。 

又矛盾了。 

这两种可能性都会导致矛盾。 因此假设的决策者$H$不可能存在。 

### 证明的用途

 证明取决于两个早期的想法。 

首先，机器可以编码为字符串：$$
\langle M\rangle
$$其次，一台机器可以在另一台机器的编码上运行，甚至可以在它自己的编码上运行：$$
M(\langle M\rangle)
$$这使得自我参考成为可能。 

矛盾源于设计$D$不同意$H$关于特殊输入：$$
\langle D\rangle
$$这是一个对角论证。 它在本质上与康托尔的对角线证明相似，但应用于计算。 

### 结果的含义

 停止问题并不是说我们永远无法证明某个特定的程序停止了。 

可以分析很多具体的程序。 例如，在有限列表上进行简单循环的程序显然会停止。 

该定理表达了更强大、更精确的内容：$$
\text{There is no single algorithm that works for all programs and all inputs.}
$$所以失败是全球性的，而不是局部性的。 

对于某些单机$M$并输入$w$，我们可以证明$M$停止或证明它没有停止。 但没有一个通用的决策程序总是能够自动做到这一点。 

### 与程序分析的关系

 停机问题解释了为什么完美的程序分析是不可能的。 

一个工具可能会检测到许多无限循环，但它无法在始终给出正确答案的同时检测到所有无限循环。 

编译器可以证明许多程序终止，但它不能证明每个终止程序都终止，也不能拒绝每个非终止程序。 

这不是当前技术的限制。 这是一个数学极限。 

### 接受与停止

 区分两个相关的问题很有用。 

验收问题提出：$$
\text{Does } M \text{ accept } w?
$$停机问题询问：$$
\text{Does } M \text{ halt on } w?
$$机器可能会因接受或拒绝而停止。 因此，停止比接受更广泛。 

这些语言是：$$
A_{\mathrm{TM}}
===============
{\langle M,w\rangle : M \text{ accepts } w}
$$和$$
\mathrm{HALT}_{\mathrm{TM}}
===========================
{\langle M,w\rangle : M \text{ halts on } w}
$$两者都是不可判定的核心问题。 

停止问题标志着计算的边界。 有些问题可以问得很清楚，编码得很精确，但仍然没有算法解决方案。
