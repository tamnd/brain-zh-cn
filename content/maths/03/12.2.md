## 12.2 图灵度

 图灵可约性通过计算能力来比较集合。 如果$$
A \leq_T B
$$然后是一个预言机$B$足够强大来决定$A$。 

这种比较自然地将集合分组为等价类。 

两套$A$和$B$如果每个都可以计算另一个，则具有相同的**图灵度**：$$
A \equiv_T B
$$在哪里：$$
A \leq_T B
\quad \text{and} \quad
B \leq_T A
$$图灵度是一个等价类$\equiv_T$。 简单来说，就是计算能力的水平。 

### 可计算度数

 所有可计算集都具有相同的最低度。 

这个学位写的是：$$
\mathbf{0}
$$如果$A$是可计算的，则：$$
A \equiv_T \varnothing
$$空集$\varnothing$被用作该程度的简单代表。 可计算集的预言机不会提供额外的能力，因为普通计算已经可以决定成员资格。 

所以：$$
A \in \mathbf{0}
$$意味着$A$是可计算的。 

### 不可计算的度数

 不可计算的集合的度严格高于$\mathbf{0}$。 

对于这样的一套$A$:$$
\mathbf{0} <_T \deg(A)
$$这意味着$A$包含普通算法无法再现的信息。 

停止问题给出了最重要的早期例子。 让：$$
K = {e : \text{program } e \text{ halts on input } e}
$$程度$K$写道：$$
\mathbf{0'}
$$这称为**零跳跃**，或**图灵跳跃**$\mathbf{0}$。 

学位$\mathbf{0'}$严格高于可计算度：$$
\mathbf{0} <_T \mathbf{0'}
$$一个神谕$K$可以回答停止问题，因此它比普通计算具有更多的能力。 

### 度数作为偏序

 图灵度按可归约性排序。 

我们写：$$
\deg(A) \leq_T \deg(B)
$$如果：$$
A \leq_T B
$$这是明确定义的，因为同等代表具有相同的预言机权力。 

该顺序是自反的：$$
\deg(A) \leq_T \deg(A)
$$它是传递性的：$$
\deg(A) \leq_T \deg(B)
\quad \text{and} \quad
\deg(B) \leq_T \deg(C)
\quad \Longrightarrow \quad
\deg(A) \leq_T \deg(C)
$$通过等价商后，它也是反对称的。 因此图灵度形成偏序。 

### 不可比性

 偏序并不要求每一对都具有可比性。 

可能有学位$\mathbf{a}$和$\mathbf{b}$这样双方都无法计算对方：$$
\mathbf{a} \nleq_T \mathbf{b}
$$和：$$
\mathbf{b} \nleq_T \mathbf{a}
$$这样的程度被称为**无与伦比**。 

这是图灵度的结构比简单的尺度更丰富的原因之一。 不可计算的问题不会从易到难排列在一条链中。 有些有不同类型的信息。 

### 代表及学位

 学位是一个等价类，而不是单个集合。 

如果：$$
A \equiv_T B
$$然后$A$和$B$可能看起来很不同，但它们具有相同的计算强度。 

例如，停机问题的许多不同版本都具有程度$\mathbf{0'}$。 他们可能使用不同的编码或提出略有不同的停止问题，但他们可以互相计算。 

因此，当我们说“停止问题的程度”时，我们指的是相当于停止问题的整个集合类。 

### 图灵跳跃

 图灵跳跃是一种需要一定程度并产生严格更强的程度的操作。 

对于一套$A$, 定义其跳转$A'$作为与 Oracle 相关的停机问题$A$:$$
A' = {e : \text{oracle program } e^A \text{ halts on input } e}
$$然后：$$
A <_T A'
$$跳跃衡量的是更高水平的计算难度。 从可计算度开始，我们得到：$$
\mathbf{0}
<
\mathbf{0'}
<
\mathbf{0''}
<
\mathbf{0'''}
<
\cdots
$$这里$\mathbf{0''}$是跳跃$\mathbf{0'}$， 等等。 

每一步都可以访问更强大的暂停预言。 

### 直觉

 图灵度衡量的是当一个集合可用作预言机时可以计算的内容。 

如果$A$学历低，那么$A$提供有限的额外信息。 如果$A$拥有高学历，那么$A$可以解决更困难的决策问题。 

但度数并不形成一条线。 有链、跳跃、不可比对、极小度以及许多其他结构现象。 

图灵度提供了一种精确的语言来表示一个不可判定的问题比另一个问题更弱、更强或无法比较。
