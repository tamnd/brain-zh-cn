## 3.4 精度与可读性

 数学写作必须足够精确以支持证明，并且必须足够可读以支持理解。 这些目标是相关的，但它们往往朝着不同的方向发展。 

精确意味着每个陈述都有明确的含义。 声明对象，陈述假设，并根据这些假设得出结论。 

可读性意味着读者可以理解论点，而无需重建太多隐藏的上下文。 

陈述可以是精确的，但难以理解。$$
\forall \varepsilon > 0,\ \exists \delta > 0,\ \forall x \in X,\ d_X(x,a) < \delta \Rightarrow d_Y(f(x), f(a)) < \varepsilon
$$这是准确的，但第一次引入时可能很难解析。 可读版本可以放在它之前：

 功能$f$连续于$a$如果点足够接近$a$被发送到接近的点$f(a)$。 

然后正式条件就可以遵循。 散文给出了这个想法。 公式确定了含义。 

| 风格| 实力| 风险|
 | -------------- | ----------------- | ------------------- |
 | 完全具有象征意义| 精确紧凑| 很难读|
 | 完全口头 | Easy to read      | 可能有歧义|
 | 混合| 清晰准确| 需要纪律 |

 好的数学写作通常采用混合风格。 它用散文介绍一个想法，用符号精确地表述它，然后解释如何使用它。 

差：

 让$f$乖一点。 然后结果如下。 

更好：

 让$f : X \to Y$是连续的。 自从$X$是紧凑的并且$Y$是豪斯多夫，图像$f(X)$是紧凑的$Y$。 

第二个版本较长，但它陈述了使定理有效的假设。 

当一个陈述可以在不同的背景下改变真值时，精确度最为重要。 

| 模棱两可的短语| 精准更换|
 | ------------------------ | | -------------------------------------- |
 | 对于所有人$x$| 对于所有人$x \in X$|
 | 函数有界 |$f$的边界为$A$|
 | 序列收敛 |$(a_n)$收敛于$X$|
 | 地图保留结构 |$f(a \cdot b) = f(a) \cdot f(b)$|
 | 该物体是独一无二的 | 直到指定的等效性为止唯一 |

 当正式声明隐藏了主要思想时，可读性最为重要。 如果论证结构简单，那么证明就不应该是一堵符号墙。 

例如，不要只写：$$
\forall x,y \in V,\ \forall \lambda \in k,\ T(x+y)=T(x)+T(y),\ T(\lambda x)=\lambda T(x)
$$可以写：

 地图$T : V \to W$是线性的：它保留向量加法和标量乘法。 

然后，如果需要，可以提供方程。 

精确并不意味着每一行都极其正式。 这意味着足够的正式性以消除歧义。 

可读性不应该意味着模糊的语言。 这意味着论证的结构是可见的。 

一个有用的模式是：

 | 步骤| 目的|
 | ---------------------- | ---------------------------------------------------- |
 | 说明设置 | 声明对象和假设 |
 | 给个直觉| 解释为什么该陈述应该是正确的 |
 | 陈述正式主张 | 修正确切含义 |
 | 以清晰的步骤证明 | 将假设与结论联系起来 |
 | 名称依赖项 | 显示使用的内容 |

 例如：

 让$G$成为一个团体并让$H \leq G$。 我们想知道什么时候陪集$H$可以相乘。 这恰好在以下情况下起作用：$H$是正常的。 在这种情况下，商$G/H$继承组结构$G$。 

这一段是可读的，因为它提供了动力。 之所以准确是因为$G$,$H$、正态性和$G/H$在上下文中具有标准含义。 

精度的主要失败模式是过早地让读者过多地了解低级细节。 可读性的主要失败模式是忽略影响正确性的假设。 

平衡的文本将层次分开。 

| 层 | 示例|
 | ------------- | -------------------------------------------------------------------------- |
 | 概念 | 连续性意味着小的输入变化会带来小的输出变化|
 | 定义 | 对于每一个$\varepsilon > 0$, 存在$\delta > 0$|
 | 定理| 连续函数保持紧凑性 |
 | 证明| 使用打开的图像封面并将其拉回 |
 | 形式化| 对拓扑空间、映射、覆盖和紧致性进行编码 |

 并非每次都需要每一层。 教科书部分可能需要所有这些。 一篇研究论文可能只需要定理和证明。 参考页可能会强调定义。 

符号密度应与读者的任务相匹配。 在计算过程中，密集符号很有用。 在解释过程中，密集的符号可能会模糊概念。 

比较：$$
(a+b)^2 = a^2 + 2ab + b^2
$$该公式简短易读，因为符号很熟悉。 

但是，包含许多未命名组件的长公式应该分解为命名部分。 命名可以减少认知负担。 

贫穷的：$$
\phi(x)=\int_A K(x,y)g(y),dy+\lambda\int_B L(x,z)h(z),dz
$$更好：

 定义$$
F(x)=\int_A K(x,y)g(y),dy
$$和$$
G(x)=\int_B L(x,z)h(z),dz.
$$然后$$
\phi(x)=F(x)+\lambda G(x).
$$第二个版本更长，但更容易推理。 它暴露了结构。 

类似的原则也适用于证明。 当相同的想法出现多次或从属权利要求具有独立价值时，应将长论证分解为引理。 

| 情况| 更好的结构|
 | -------------------- | ------------------------ | |
 | 重复计算| 定义符号 |
 | 反复论证| 提取引理 |
 | 许多假设| 将它们分组到一个设置中 |
 | 案例众多| 说出每个案例 |
 | 长依赖链 | 陈述中间主张 |

 精度还需要明确的量词顺序。 量词的顺序会改变含义。$$
\forall x \in X,\ \exists y \in Y,\ P(x,y)
$$意味着每个$x$可能有自己的$y$。$$
\exists y \in Y,\ \forall x \in X,\ P(x,y)
$$意思是一$y$适用于每个$x$。 

这些是不同的说法。 散文应该保留这种区别。 

“每个输入都有一个输出”对应于第一句话。 “有一个输出适用于每个输入”对应于第二个。 

当量词被翻译成清晰的散文时，可读性会提高，但当顺序很重要时，符号版本应该保持可用。 

精度还取决于平等和对等。 说两个对象“相同”往往过于模糊。 文本应该说明它是否意味着相等、同构、同构、等效、全等或达到规范识别的相等。 

| 短语 | 更好的形式|
 | -------------------------------------- | ---------------------------------------------------- |
 |$A$和$B$是一样的 |$A = B$|
 | 组是相同的 |$G \cong H$|
 | 空格相同 |$X$和$Y$是同构的 |
 | 功能在本地都是一样的| 他们就社区达成一致|
 | 这些对象仅在坐标上有所不同 | 在基础改变的情况下它们是等效的 |

 在陈述了精确的关系之后，可读的文本可以非正式地使用“相同”。 在此之前，应该避免走捷径。 

最好的规则是局部显性。 不要在每句话中重述整个基础，而是在可能出现歧义的地方说明读者需要什么。 

数学写作应该边界准确，中间可读。 定义、定理陈述和最终结论需要高精度。 动机和证明草图可以使用更多的散文，只要它们不改变主张。 

精度保护正确性。 可读性保护传输。 如果缺少其中任何一个，数学文本就会失败。
