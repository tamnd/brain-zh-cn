## 4.1 结构与实例

 结构就是一种模式。 实例是该模式的具体实现。 

这种区别是数学推理的核心。 它让我们能够在结构层面上证明一次结果，然后将它们应用到许多例子中。 

组是一个结构。 加法下的整数是该结构的一个实例。 乘法下的非零有理数是另一个例子。 正方形的对称性是另一回事。 

| 结构| 实例|
 | ----------------- | -------------------------------------------------------- |
 | 集团|$(\mathbb{Z}, +)$|
 | 集团| 乘法下的非零有理数 |
 | 集团| 正方形的对称性|
 | 向量空间|$\mathbb{R}^n$超过$\mathbb{R}$|
 | 图| 公路网|
 | 拓扑空间| 实数线及其通常的开集 |

 该结构指定需要哪些数据和法律。 该实例提供满足这些要求的实际对象。 

对于一个组来说，结构需要：

 | 部分| 要求 |
 | ------------- | ---------------------------------------------------------------------------------- |
 | 承运人 | 一套$G$|
 | 运营| 二元运算$\ast : G \times G \to G$|
 | 关联性|$(a \ast b) \ast c = a \ast (b \ast c)$|
 | 身份 | 存在$e \in G$这样$e \ast a = a \ast e = a$|
 | 逆 | 对于每一个$a \in G$, 存在$a^{-1}$和$a \ast a^{-1} = e$|

 实例只有满足规律才有效。 

例如，$(\mathbb{Z}, +)$是一个团体。 承运人是$\mathbb{Z}$。 运算是加法。 身份是$0$。 的逆$n$是$-n$。 

相比之下，$(\mathbb{N}, +)$不是一个群，如果$\mathbb{N} = {0,1,2,\dots}$，因为大多数元素内部没有加性逆元$\mathbb{N}$。 

这说明了一个一般规则：具有相似的符号并不能保证相同的结构。 

表达式$a + b$可能同时出现在$\mathbb{Z}$和$\mathbb{N}$，但结构规律不同。 

### 为什么区别很重要

 结构推理在模式层面起作用。 

如果一个定理对于所有群都被证明，那么它适用于每个有效的群实例。 

示例：

 在任何群体中，单位元都是唯一的。 

一旦这个定理在结构上得到证明，我们就不需要针对整数、矩阵、排列或对称性重新证明它。 每个实例都会继承结果。 

这是抽象的主要好处。 

| 水平| 工作完成 |
 | ----------- | ---------------------- |
 | 结构| 证明一般定理 |
 | 实例| 验证公理 |
 | 应用 | 使用继承的结果 |

 使用结构的代价是必须检查实例。 在应用群定理之前，我们必须知道对象是一个群。 在应用向量空间定理之前，我们必须知道对象满足向量空间公理。 

### 实例可以携带许多结构

 同一个对象可以实例化多个结构。 

真实数字$\mathbb{R}$可以看作：

 | 查看 | 结构|
 | -------------------------------- | ----------------- |
 |$(\mathbb{R}, +)$| 添加剂组|
 |$(\mathbb{R}, +, \cdot)$| 领域 |
 |$(\mathbb{R}, <)$| 订购套装 |
 |$(\mathbb{R}, d)$| 度量空间|
 |$\mathbb{R}$具有开区间 | 拓扑空间|
 |$\mathbb{R}$超过$\mathbb{Q}$| 向量空间|

 每种结构都支持不同的定理。 

如果我们治疗$\mathbb{R}$作为一个域，我们可以讨论非零元素的乘法逆元。 如果我们把它看作一个拓扑空间，我们就可以讨论开集和连续性。 如果我们将其视为有序集，我们可以讨论上限和单调性。 

实例本身并不能确定活动结构。 文本必须陈述它或从上下文中清楚地表达出来。 

### 忘记结构

 有时我们故意忽略结构的一部分。 这就是所谓的遗忘结构。 

场可以被视为一个环。 环可以被视为加性基团。 度量空间可以被视为拓扑空间。 

| 结构更丰富 | 被遗忘的结构|
 | ---------------- | ---------------------------- |
 | 领域 | 戒指|
 | 戒指| 加法下的阿贝尔群 |
 | 度量空间 | 拓扑空间|
 | 向量空间| 设置|
 | 集团| 设置|

 忘记结构会导致可用的操作减少。 它还可能使更多地图变得有效。 

线性映射保留向量空间结构。 底层集合之间的函数不需要保留任何内容。$$
T : V \to W
$$因为线性映射比线性映射更具限制性$$
f : |V| \to |W|
$$作为底层集合之间的普通函数。 

符号$|V|$有时用来表示底层的集合$V$在忘记了它的额外结构之后。 

### 添加结构

 相反的过程是添加结构。 可以给一个集合一个拓扑、一个群运算、一个顺序或一个度量。 

例如，集合$\mathbb{R}$可以配备普通拓扑、分立拓扑或其他拓扑。 它们在同一底层集合上产生不同的拓扑空间。 

| 底层集合 | 添加结构| 结果 |
 | ---------------- | ----------------- | ------------------ |
 |$\mathbb{R}$| 常见拓扑| 标准实线|
 |$\mathbb{R}$| 离散拓扑| 离散空间|
 |$\mathbb{Z}$| 加法 | 添加剂组|
 | 有限集 | 边缘 | 图|
 | 一组点 | 距离函数 | 度量空间 |

 添加结构可以增加可说的内容。 它还增加了地图必须保留的内容。 

### 模型作为实例

 在逻辑中，理论的实例通常称为模型。 

理论给出了公理。 模型给出了满足这些公理的对象。$$
M \models T
$$意味着结构$M$满足理论$T$。 

例如，群公理定义了一个理论。 每个群体都是该理论的模型。 

这将结构思维与形式逻辑联系起来。 结构不仅仅是普通数学中的模式。 它也可以是对正式语言的解释。 

### 规范和非规范实例

 有些事例是自然而然地出现的。 其他人需要选择。 

向量空间$k^n$是一个标准模型$n$维向量空间$k$。 每一个$n$维向量空间$k$同构于$k^n$，但同构通常取决于基础的选择。$$
V \cong k^n
$$这在结构上是正确的，但除非选择了基础，否则不是规范的。 

区别很重要。 

| 声明| 意义|
 | --------------------------------- | ---------------------------------------------------- |
 |$V \cong k^n$| 存在同构 |
 |$V = k^n$| 这些对象实际上是相等的 |
 | 一个基础确定$V$和$k^n$| 一个选择给出了具体的表示 |

 结构的相同性并不会自动给出字面的同一性。 

### 表示与结构

 一个实例通常有一个表示。 表示是我们存储或写入它的方式。 结构就是表示的含义。 

选择碱基后，矩阵可以表示线性映射。$$
A \in M_{m \times n}(k)
$$代表一张地图$$
T : k^n \to k^m
$$改变基数会改变矩阵，但不会改变底层的线性映射。 

因此，矩阵表示是依赖于坐标的。 线性映射是结构性的。 

| 对象| 代表| 结构意义|
 | ----------------- | ----------------- | -------------------------------- |
 | 矢量| 坐标栏 | 向量空间的元素 |
 | 矩阵| 标量数组 | 线性变换 |
 | 图| 邻接表| 具有邻接关系的顶点 |
 | 多项式| 系数列表 | 形式表达式或函数 |
 | 拓扑空间| 公开组列表 | 社区结构 |

 将表征与结构混淆会导致推理脆弱。 

### 结构运输

 同构允许结构从一个对象转移到另一个对象。 

如果$f : A \to B$是一个双射并且$A$有手术$\ast$，我们可以定义一个操作$B$通过运输通过$f$:$$
b_1 \star b_2 = f(f^{-1}(b_1) \ast f^{-1}(b_2)).
$$这使得$B$结构上等价于$A$。 

结构传递解释了为什么同构对象具有相同的结构特性。 它还解释了为什么等式对于数学来说往往过于严格。 

### 实用规则

 进入新主题时，分四层。 

| 层 | 问题 |
 | ------------------ | ------------------------------------------------ |
 | 底层对象| 有哪些要素？ |
 | 结构| 存在哪些操作、关系或法则？ |
 | 代表| 对象是如何写入或存储的？ |
 | 等价| 什么时候两个实例被认为是相同的？ |

 这可以防止类别错误。 

例如，在研究图时，询问顶点是否已标记或未标记。 研究向量空间时，询问基是否是固定的。 研究拓扑空间时，询问正在使用哪种拓扑。 

相同的数据可以支持不同的结构。 相同的结构可以有不同的表示。 同一结构对象可以有许多同构实例。 

当这些层次保持分离时，数学就会变得更加清晰。
