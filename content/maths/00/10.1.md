## 10.1 论文的结构

 数学论文是一个想法的书面路径。 它不仅列出结果。 它解释了正在研究什么问题、为什么这个问题很重要、使用了哪些工具以及已经证明了什么。 

对于初学者来说，将论文视为精心组织的解释是很有用的。 每个部分都有一个工作。 标题告诉读者这篇论文的内容。 引言给出了主要问题。 定义准备语言。 定理陈述了主要主张。 这些证据解释了为什么这些说法是正确的。 这些例子帮助读者理解结果的含义。 

### 标题和摘要

 标题应该告诉读者主要主题。 模糊的标题会使论文更难找到、更难理解。 

例如，标题“代数的一些结果”就太宽泛了。 更好的标题是“二项式定理的简短证明”或“计算有限图中的路径”。 这些标题让读者有了更明确的期待。 

摘要是论文的简短摘要。 应该说研究什么问题，取得什么结果。 在高级论文中，摘要可能是技术性的。 初学者写作时，应该直接、简单。 

### 简介

 引言解释了本文的目的。 它回答了基本问题：

 我们正在研究什么问题？ 
为什么这个问题很有趣？ 
主要结果是什么？ 
将使用什么想法？ 

假设一篇论文证明了一个求和公式$$
1 + 2 + 3 + \cdots + n.
$$引言可能会说论文研究如何添加第一个$n$正整数并证明总和是$$
\frac{n(n+1)}{2}.
$$这告诉读者在细节开始之前会发生什么。 

### 背景和定义

 在证明任何事情之前，论文必须使其语言清晰。 这是定义出现的地方。 

例如，如果一篇论文谈论偶数，它应该定义“偶数”的含义：

 一个整数$n$即使存在一个整数$k$这样$$
n = 2k.
$$定义可以防止混淆。 它们使论文的其余部分更加精确。 读者不必猜测重要单词是如何使用的。 

### 主要结果

 主要结果通常以定理、命题或引理的形式表述。 

定理是一个重要的结果。 引理是用于证明较大结果的较小结果。 一个命题通常是一个中等重要性的有用结果。 

例如：

 **定理。** 对于每个正整数$n$,$$
1 + 2 + 3 + \cdots + n = \frac{n(n+1)}{2}.
$$一个好的定理陈述应该在证明开始之前就清楚。 它应该准确地说明假设的内容和声称的内容。 

### 证明

 证据是证明主张合理的部分。 它应该从已知的事实逐步走向期望的结论。 

例如，要证明公式$$
1 + 2 + 3 + \cdots + n,
$$一种可能的方法是归纳法。 首先证明公式为$n = 1$。 然后假设这对某些人来说是正确的$n$并证明它$n + 1$。 

证明不应该是无法解释的符号的集合。 需要用文字来告诉读者为什么要执行每个步骤。 

### 示例

 示例可以帮助读者了解结果的含义。 

对于公式$$
1 + 2 + 3 + \cdots + n = \frac{n(n+1)}{2},
$$我们可以测试$n = 5$:$$
1 + 2 + 3 + 4 + 5 = 15.
$$公式给出$$
\frac{5(5+1)}{2} = \frac{30}{2} = 15.
$$该例子并不证明定理，但它使定理更容易理解。 

### 结论

 结论提醒读者所展示的内容。 它还可能提到接下来可能出现的问题。 

例如，在证明第一个总和的公式之后$n$正整数，结论可能会询问平方和：$$
1^2 + 2^2 + 3^2 + \cdots + n^2.
$$一个好的结论不会重复整篇论文。 它结束了讨论并指出了接下来会发生什么。 

### 实用的结构

 对于初学者数学写作，简单的结构通常就足够了：

 1. 陈述问题
 2. 给出所需的定义
 3. 陈述主要结果
 4. 证明结果
 5. 举例说明
 6. 解释所学到的内容

 这种结构使论文具有可读性。 它可以帮助作者避免过快地跳入符号，也可以帮助读者从头到尾理解数学论证。
