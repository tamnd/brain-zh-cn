## 7.3 归纳与递归

 归纳法是分阶段构建对象的标准证明技术。 递归是匹配的构造技术。 它们一起描述了如何定义、计算和证明有关自然数、列表、树、语法、算法和其他生成结构的陈述。 

最简单的形式是自然数的数学归纳法。 

为了证明一个说法$P(n)$对于每个自然数$n$，证明两件事。 

首先，证明基本情况：$$
P(0).
$$其次，证明归纳步骤：$$
P(n) \Rightarrow P(n+1).
$$两者成立后，结论如下：$$
P(n) \text{ holds for all } n \in \mathbb{N}.
$$这个想法是，基本案例启动链条，归纳步骤将真理从一个阶段带到下一阶段。 

例如，证明$$
1 + 2 + \cdots + n = \frac{n(n+1)}{2}
$$对于每个整数$n \geq 1$。 

为了$n=1$，左边是$1$，右边是$$
\frac{1(1+1)}{2} = 1.
$$所以基本情况成立。 

现在假设该公式对于某些情况成立$n$:$$
1 + 2 + \cdots + n = \frac{n(n+1)}{2}.
$$然后$$
1 + 2 + \cdots + n + (n+1)
==========================
\frac{n(n+1)}{2} + (n+1).
$$因式分解右侧：$$
\frac{n(n+1)}{2} + (n+1)
========================
# (n+1)\left(\frac{n}{2}+1\right)
\frac{(n+1)(n+2)}{2}.
$$这是所需的公式$n+1$代替$n$。 因此，该公式对于所有$n \geq 1$。 

暂时假设$P(n)$成立称为归纳假设。 它必须仅用于证明下一种情况。 这并不是假设所有值的结果都已知。 

归纳法之所以有效，是因为自然数是通过重复应用后续运算从起点生成的。 证明反映了物体的构造。 

递归是同一想法的构造方面。 递归定义的对象由基础数据和生成更大案例的规则指定。 

例如，阶乘函数定义为$$
0! = 1
$$和$$
(n+1)! = (n+1)n!.
$$该定义告诉我们如何根据早期值计算每个阶段的值。 归纳法告诉我们如何证明这些值的属性。 

强归纳是一种变体，其中归纳步骤可以使用所有早期案例。 证明$P(n)$为所有人$n$, 证明如果$P(k)$对于每一个都成立$k < n$， 然后$P(n)$成立。 

当下一个案例比前一个案例更依赖时，这非常有用。 

例如，许多整除性和因式分解论证都使用强归纳法，因为数字可能会减少到一个不完全减一的较小因数。 

结构归纳法将同样的想法扩展到自然数之外。 如果对象是由构造函数构建的，那么这些构造函数后面就会有一个证明。 

对于列表，先证明空列表的一个属性，然后证明它是否适用于列表$L$，它也适用于通过添加一个元素形成的列表$L$。 

对于树，我们需要证明叶子的性质，然后证明如果它适用于子树，那么它也适用于由子树构建的树。 

对于形式语法，首先证明原子表达式的属性，然后证明每个形成规则都保留该属性。 

这使得归纳法成为逻辑和计算机科学中的主要证明技术之一。 许多关于程序、语法、公式和数据结构的定理都是结构归纳论证。 

归纳法的主要原则是选择正确的陈述。 有时，所需的主张太弱而无法通过归纳法证明，因为该步骤需要更强的假设。 在这种情况下，解决方案是加强归纳语句。 

例如，关于算法的定理可能需要一个不变量：在执行的每个阶段都保留的属性。 不变量可能比最终结果更强，但它使得归纳步骤成为可能。 

常见的失败是证明基本情况，然后假设结论而不是证明步骤。 归纳步骤必须表明该属性从一个阶段到下一阶段都得到保留。 

另一个常见的失败是对错误的参数使用归纳法。 该参数应该测量对象的构造：大小、长度、深度、步数或自然数索引。 

当对象具有递归结构时，归纳法是有效的。 它将无限的声明族转变为有限的证明模式：正确地开始，然后在每一步中保持正确性。 

递归构建对象。 归纳证明了如此构建的对象的属性。
