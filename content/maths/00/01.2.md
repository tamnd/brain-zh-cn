## 1.2 集合、类型和宇宙

 数学需要一种方法来说明物体是什么样的东西。 在我们证明一个陈述、定义一个函数或比较两个对象之前，我们需要一个论域。 集合、类型和域提供了组织该域的不同方式。 

集合是对象的集合，称为元素。 我们写：```text
x ∈ A
```意思是`x`是集合的一个元素`A`。 

例如：```text
3 ∈ N
-2 ∈ Z
1/2 ∈ Q
π ∈ R
```该集合告诉我们对象所在的位置。 它还告诉我们哪些操作和语句应该是有意义的。 加法是在自然数、整数、有理数、实数、复数、向量、矩阵和许多其他结构上定义的，但每种设置都赋予加法特定的含义。 

类型扮演着类似的角色，但更强调形成规则。 这么说`x`有类型`T`意味着`x`是一个有效的类型对象`T`。```text
x : T
```这种表示法在类型论、证明助手和编程语言语义中很常见。 

集合和类型在非正式数学中通常表现相似。 声明“让`x ∈ R`”和声明“让`x : R`” 两者都声明`x`是一个实数。 但它们来自不同的基础。 

| 想法| 集合论观点| 类型论观点 |
 | -------------- | -------------------------------------------------- | ---------------------------------------------------- |
 | 会员资格 |`x ∈ A`| 通常用输入 | 代替
 | 分类| 对象属于集合 | 对象有类型 |
 | 平等| 经常是全球性的 | 通常对类型敏感 |
 | 功能| 产品的子集或集合之间的映射 | 将一种类型转换为另一种类型的术语 |
 | 错误处理 | 格式错误的表达方式仍可能在非正式场合出现 | 错误类型的表达式会被拒绝 |

 在集合论中，许多对象存在于一个大的集合宇宙中。 数字、函数、关系、有序对、图形和空间都可以编码为集合。 这提供了统一的基础。 图可以表示为一对集合。 函数可以表示为一组有序对。 拓扑可以表示为一组子集。 

在类型论中，每个对象都属于一个类型。 根据键入规则检查表达式。 从自然数到实数的函数具有如下类型：```text
f : N → R
```这说的是`f`接受一个自然数并返回一个实数。 表达式`f(3)`有意义如果`3 : N`。 表达式`f(circle)`没有意义，除非`circle`有类型`N`，通常不会。 

这种类型的纪律有助于防止无意义的表达。 它在形式化数学和验证软件中尤其重要。 

宇宙解决了尺寸问题。 如果每个集合都可以被视为同类的集合，就会出现悖论。 经典的例子是“所有不包含自身的集合的集合”。 为了避免此类矛盾，基金会区分了级别。 

宇宙是一个足够大以包含许多数学对象的域，但又不能大到可以不受限制地包含自身。 

在集合论中，这通过累积层次结构出现。 场景是分阶段构建的。 在每个阶段，我们都会收集之前的子集。 越来越大的关卡会产生越来越复杂的集合。```text
V0, V1, V2, V3, ...
```确切的形式结构可能有所不同，但想法很简单：对象按大小或等级进行分层。 

在类型论中，宇宙通常写成：```text
Type 0
Type 1
Type 2
...
```小型类型可能居住在`Type 0`。 小型类型可能居住的类型`Type 1`。 这避免了使一种类型以不受限制的方式包含自身。 

| 水平| 非正式角色 |
 | ------------- | --------------------------------------- |
 |`Type 0`| 普通数学对象 |
 |`Type 1`| 元素为普通类型的类型 |
 |`Type 2`| 较大类型集合的类型 |
 | 更高水平| 进一步分层|

 宇宙在普通数学中通常是不可见的。 教科书很少提及群、环或拓扑空间属于哪个宇宙。 但在构建基础、编写证明辅助代码或讨论大型对象的类别时，这个问题就变得很重要。 

例如，所有组的类别都很大。 它的对象是组，每个组本身包含一组元素。 如果我们不小心将“所有群体”视为普通集合，我们可能会遇到规模问题。 范畴论通过区分小范畴、大范畴和宇宙来处理这个问题。 

集合对于扩展推理来说很方便。 当两个集合具有相同的元素时，它们相等。```text
A = B  exactly when  every element of A is an element of B,
and every element of B is an element of A.
```类型对于内涵推理来说很方便。 对象的构造方式可能很重要，而相等性可能需要更多的结构。 在许多正式系统中，类型之间的转换必须是明确的。 

例如，自然数`3`, 整数`3`, 有理数`3`，以及实数`3`可以被视为通过强制连接的不同类型的对象。```text
3 : N
3 : Z
3 : Q
3 : R
```非正式数学常常压制这些区别。 证明助理通常会强制指定它们。 

这种压制是有用的，但也是危险的。 它使数学变得可读，但它可以隐藏假设。 当定理表明函数连续时，我们必须知道域和余域上的拓扑。 当定理表明序列收敛时，我们必须知道周围空间。 当定理指出矩阵可逆时，我们必须知道系数域或系数环。 

这句话“让`x`在我们知道背景之前，“成为一个物体”是不完整的。 

一个好的数学陈述声明了足够的上下文，使其符号有意义。 例如：```text
Let V be a finite-dimensional vector space over a field k.
Let T : V → V be a linear map.
```这给出了载体、标量场、维度假设和结构保持图。 后面关于特征值、不变子空间或行列式的陈述现在有了定义的设置。 

不明确的上下文会导致错误或模棱两可的主张。 等式：```text
x^2 + 1 = 0
```在实数中无解，但在复数中有两个解。 一个陈述的真实性取决于它被解释的宇宙。 

| 声明| 背景 | 结果 |
 | --------------------------------------------------- | ---------------- | ---------------- |
 |`x^2 + 1 = 0`|`x ∈ R`| 没有解决办法|
 |`x^2 + 1 = 0`|`x ∈ C`| 存在解决方案|
 | 每个非零元素都有一个逆 | 整数 | 假 |
 | 每个非零元素都有一个逆 | 有理数 | 真实|
 | 每个有界序列都有一个收敛子序列 | 实数 | 真实|
 | 每个有界序列都有一个收敛子序列 | 有理数 | 假 |

 这个教训是直接的：对象、陈述和证明都存在于选定的上下文中。 

集合、类型和宇宙不仅仅是相互竞争的符号。 它们代表了对数学组织的不同态度。 

集合问：哪些元素属于这个集合？ 

类型询问：这个表达式允许是哪种类型的对象？ 

宇宙问：我们量化的领域有多大？ 

一位在职数学家经常非正式地使用这三个想法。 正式的系统迫使这些区别变得精确。 最佳实践是编写足够的上下文，让读者知道哪些对象存在、哪些操作有效以及正在使用哪种等式关系。 

当每个符号都有一个归属时，数学就会变得更安全。
