## 2.2 形式系统和语义

 数学通过形式系统进行运作。 正式系统指定如何编写陈述以及如何构建证明。 语义解释了这些语句在结构内的含义。 

正式系统有四个组成部分。 

| 组件| 角色 |
 | ---------------- | ----------------------------------- |
 | 字母表| 用于编写表达式的符号|
 | 语法 | 形成有效表达式的规则 |
 | 公理| 初始陈述假设为真 |
 | 推理规则 | 派生新陈述的方法 |

 这些组件定义了句法层。 它们确定哪些符号序列算作有效公式以及哪些推导算作证明。 

命题逻辑中有一个简单的例子。 字母表包括以下符号：`P`,`Q`,`¬`,`∧`,`∨`,`→`。 语法指定如何构建公式：```text id="a1z9rk"
If P and Q are formulas, then (P ∧ Q), (P ∨ Q), (P → Q) are formulas.
If P is a formula, then ¬P is a formula.
```公理提供了基本真理。 推理规则，例如肯定前件，允许推导：```text id="c8m2ht"
From P and (P → Q), infer Q
```证明是有限的公式序列，其中每个步骤要么是一个公理，要么是根据推理规则从先前的步骤得出的。 

语义赋予这些符号意义。 结构通过将符号映射到对象和操作来解释语言。 

对于命题逻辑，结构将真值分配给变量：```text id="v5p3ld"
P = true
Q = false
```然后根据真值表评估复合式。 

对于一阶逻辑，语义更丰富。 一个结构包括：

 | 组件| 意义|
 | ---------| -------------------------------------------------- |
 | 域名 | 对象集 |
 | 常数 | 域的具体元素 |
 | 功能| 元素之间的映射 |
 | 关系 | 域上的属性或关系 |

 声明如：```text id="t2n7gw"
∀x (P(x) → Q(x))
```通过检查域中的所有元素来评估。 如果每个元素满足`P`也满足`Q`。 

满足关系连接语法和语义：```text id="f0r8ks"
M ⊨ φ
```这读作“结构`M`满足声明`φ`”。 它定义了模型内部的真相。 

正式系统操纵符号而不参考意义。 语义解释模型内的这些符号。 两层之间的相互作用是核心。 

| 层 | 焦点 |
 | ---------| ------------------- |
 | 语法 | 符号操作|
 | 语义 | 意义与真理|

 系统的设计使得句法推导与语义真理相对应。 这会带来前面讨论过的健全性和完整性。 

不同的系统使用不同的语言。 算术使用数字、加法、乘法和顺序的符号。 群论使用二元运算和恒等式。 集合论使用隶属度`∈`。 

每种语言都限制了可以表达的内容。 一阶逻辑无法在不编码的情况下直接量化子集或函数。 高阶系统允许更丰富的表达，但也更复杂。 

模型提供了灵活性。 一种形式语言可以有许多模型。 

| 语言 | 可能的型号 |
 | ------------ | -------------------------------------------------------- |
 | 算术| 标准自然数、非标准模型 |
 | 群论| 所有团体 |
 | 图论 | 所有图表 |
 | 秩序理论| 所有部分有序集 |

 某个陈述在某些模型中可能是正确的，而在其他模型中可能是错误的。 这导致了依赖于模型的真理。 

理论是给定语言中的一组公理。 它的模型是所有公理都成立的结构。```text id="u9d4eq"
M ⊨ T   means   M satisfies all axioms in T
```理论描述了一类结构。 例如，群论的公理描述了所有群。 实闭域公理描述了类似于实数的结构。 

语义允许理论和结构之间的比较。 它回答了以下问题：

 | 问题 | 意义|
 | ------------------------------------------------------ | ------------------- |
 | 有模型存在吗？ | 一致性|
 | 所有模型都满足某个陈述吗？ | 有效期 |
 | 某个陈述在特定模型中成立吗？ | 该模型中的真相|

 形式系统也支持理论之间的解释。 通过映射符号和保留结构可以将一种理论转化为另一种理论。 这在代数和逻辑中很常见。 

形式化推理和非正式推理之间存在实际区别。 数学家很少写出完全形式化的证明。 他们依赖于对语义的共同理解。 正式系统提供了在需要时确保严格性的基础。 

证明助手使这一点变得明确。 每一步都必须遵循语法规则。 每个对象都必须有一个类型。 语义通过定义和结构进行编码。 

工作模式：

 | 步骤| 行动|
 | ---- | --------------------------------- |
 | 1 | 选择语言 |
 | 2 | 定义公理 |
 | 3 | 从语法上推导出结果 |
 | 4 | 解释模型中的结果 |

 这种模式出现在数学的所有领域。 代数定义运算和公理，然后研究它们的结果。 分析定义限制和连续性，然后研究功能。 几何定义空间和关系，然后研究它们的属性。 

正式的系统提供精确性。 语义提供意义。 数学两者都需要。
