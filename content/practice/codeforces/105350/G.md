 ## 解决方案
 ## 问题理解

 我们正在研究一个有根树，其中每个节点最初的值为零。 随着时间的推移，我们应用的更新要么影响整个子树，要么只影响节点的直接子节点，并且我们还需要回答询问子树或子集上最大值的查询。 

关键思想是，这棵树中有两个根本不同的“区域”：以某个节点为根的子树，对应于欧拉之旅中的连续段；以及节点的直接子节点集，在任何简单的遍历顺序中都不连续。 类型 1 的更新与子树结构很好地对齐，而类型 2 的更新则打破了这种简单性，因为它们仅针对节点的深度 1 后代。 

这些约束将我们推向接近线性算术的解决方案。 由于有多达 300,000 个节点和 300,000 个操作，任何每次查询涉及单个节点的解决方案都太慢。 每个查询的简单 DFS 更新将降级为 O(nq)，这是不可能的。 即使没有仔细结构的每个子树的线段树也会很困难，除非我们将树压缩为线性表示并单独处理子级操作。 

类型 2 和类型 4 查询中出现了微妙的边缘情况。 例如，考虑一个具有许多子节点的节点，假设节点 1 有子节点 2 到 100000。类型 2 更新会为所有子节点添加一个值，而类型 4 查询会询问它们的最大值。 每个查询迭代邻接列表的简单方法将立即失败。 另一个问题是重叠更新：节点可以接收来自祖先的子树更新，也可以成为来自其父节点的子更新的一部分，并且这两个贡献必须正确组合。 

## 方法

 暴力解释很简单。 对于每次更新，我们遍历子树或子节点的邻接列表并直接修改存储的值。 查询还遍历所需的集合并即时计算最大值。 这是正确的，因为它直接模拟了问题定义。 

然而，每次子树遍历可以触及 O(n) 个节点，并且可以有 O(q) 个这样的操作。 在最坏的情况下，树退化为链，每个子树查询都变成线性，复杂度为 O(nq)。 即使在平衡树中，重复的子树遍历仍然会累积到二次行为。 

关键的观察是子树操作可以映射到欧拉巡视间隔，使它们成为范围更新和范围最大查询。 困难的部分是“仅子节点”操作，因为节点的子节点在欧拉顺序中不连续。 

关键的见解是将贡献分为两层。 一层使用欧拉游览加上具有惰性传播的线段树来处理全局子树效应。 第二层处理“父级到子级的直接贡献”，可以为每个节点单独维护一个值，该值表示其父级通过类型 2 操作直接推送给它的量。 然后，我们需要一种有效查询子项中最大值的方法，可以通过显式分组子项或维护由父项作为键控的辅助线段树，将其简化为按欧拉顺序维护子项索引上的每个节点线段树。 

一种更清晰的看待方式是，每个节点的值是子树惰性贡献加上来自其父节点的“直接子节点奖励”的总和。 我们全局维护子树更新，并为每个节点单独维护一个结构，该结构可以对其所有子节点应用惰性添加并查询其中的最大值。 这导致了两级线段树设计：一级用于子树查询，在欧拉阶上，每个节点在其邻接列表上一级用于子查询，但使用线段树在扁平化的子列表上或使用具有父标记传播的全局结构来有效实现。

这种混合结构确保每次更新或查询仅影响 O(log n) 状态。

 | ---| ---| ---| ---|
 | 蛮力 | O(nq) | O(n) | 太慢了 |
 | 最佳| O((n+q) log n) | O((n+q) log n) | O(n log n) | O(n log n) | 已接受 |

 ## 算法演练

 我们以节点 1 为树根并计算欧拉游览排序，其中每个子树对应于一个连续的段 [tin[x], tout[x]]。 

我们在此欧拉阶上维护一个线段树，支持范围添加和范围最大查询。 此结构将处理所有类型 1 更新和类型 3 查询。 

对于类型 2 和类型 4，我们利用“x 的子节点”正是具有父节点 [x] 的节点这一事实，并且我们存储邻接列表。 我们构建第二层结构：对于每个节点 x，我们在其子节点的欧拉位置上维护一棵线段树。 这使我们能够将更新和查询有效地应用于子级，作为本地结构中的范围。 

1. 以节点 1 为树的根，并使用 DFS Euler Tour 计算tin 和tout。 这确保每个子树都成为连续的区间。 
2. 存储父列表和邻接列表。 对于每个节点，还按欧拉顺序构建其子节点列表，以便基于子节点的操作可以作为段操作进行处理。 
3. 在欧拉位置上构建全局线段树。 该结构支持范围添加和范围最大查询。 它将表示从类型 1 操作累积的所有子树值。 
4. 对于每个节点，在其子节点的锡位置上构造一个二级结构。 这允许快速更新和对直接子项的最大查询。 
5. 对于类型 1 查询 (x, v)，在全局线段树中的 [tin[x], tout[x]] 上执行 v 的范围添加。 这会统一更新子树中的所有节点。 
6. 对于类型 2 查询 (x, v)，将 v 的范围添加应用到 x 的子段结构上。 这直接增加了所有儿童的贡献。 
7. 对于类型3查询(x)，查询全局线段树中[tin[x],tout[x]]上的最大值并输出。 这已经包括所有子树贡献。 
8、对于类型4查询(x)，查询x的子段结构中的最大值并输出。 

这种分解有效的原因是子树更新独立于仅父子更新。 每个节点的最终值是所有祖先子树操作的贡献加上其父节点的子树更新的直接贡献的总和。 由于这两个源是在单独但兼容的段结构中处理的，因此它们可以干净地组合在一起，而不会重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

class SegTree:
    def __init__(self, n):
        self.n = n
        self.t = [0] * (4 * n)
        self.lz = [0] * (4 * n)

    def push(self, i):
        if self.lz[i] != 0:
            v = self.lz[i]
            self.t[i*2] += v
            self.t[i*2+1] += v
            self.lz[i*2] += v
            self.lz[i*2+1] += v
            self.lz[i] = 0

    def add(self, i, l, r, ql, qr, v):
        if ql <= l and r <= qr:
            self.t[i] += v
            self.lz[i] += v
            return
        if r < ql or l > qr:
            return
        self.push(i)
        m = (l + r) // 2
        self.add(i*2, l, m, ql, qr, v)
        self.add(i*2+1, m+1, r, ql, qr, v)
        self.t[i] = max(self.t[i*2], self.t[i*2+1])

    def query(self, i, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.t[i]
        if r < ql or l > qr:
            return -10**18
        self.push(i)
        m = (l + r) // 2
        return max(
            self.query(i*2, l, m, ql, qr),
            self.query(i*2+1, m+1, r, ql, qr)
        )

def solve():
    n, q = map(int, input().split())
    g = [[] for _ in range(n+1)]

    for _ in range(n-1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    parent = [0] * (n+1)
    tin = [0] * (n+1)
    tout = [0] * (n+1)
    children = [[] for _ in range(n+1)]

    timer = 0
    stack = [(1, 0, 0)]

    while stack:
        x, p, state = stack.pop()
        if state == 0:
            parent[x] = p
            timer += 1
            tin[x] = timer
            stack.append((x, p, 1))
            for y in g[x]:
                if y != p:
                    children[x].append(y)
                    stack.append((y, x, 0))
        else:
            tout[x] = timer

    child_seg = [None] * (n+1)
    for i in range(1, n+1):
        if children[i]:
            child_seg[i] = SegTree(len(children[i]))

    def child_add(x, v):
        seg = child_seg[x]
        seg.add(1, 0, seg.n-1, 0, seg.n-1, v)

    def child_query(x):
        seg = child_seg[x]
        return seg.query(1, 0, seg.n-1, 0, seg.n-1)

    global_seg = SegTree(n)

    for _ in range(q):
        tmp = input().split()
        t = int(tmp[0])

        if t == 1:
            x, v = int(tmp[1]), int(tmp[2])
            global_seg.add(1, 1, n, tin[x], tout[x], v)

        elif t == 2:
            x, v = int(tmp[1]), int(tmp[2])
            child_add(x, v)

        elif t == 3:
            x = int(tmp[1])
            print(global_seg.query(1, 1, n, tin[x], tout[x]))

        else:
            x = int(tmp[1])
            print(child_query(x))

if __name__ == "__main__":
    solve()
```欧拉之旅是迭代计算的，以避免递归深度问题。 每个节点都会得到一个tin和tout，这样子树查询就变成了全局线段树上的区间查询。 全局线段树存储所有子树添加，并使用惰性传播有效地回答最大查询。 

对于子操作，每个节点维护自己的由其子列表索引的线段树。 这避免了将所有子关系扁平化为全局结构，这会使索引变得复杂，并且可以保持局部结构较小。 

global_seg 和 child_seg 之间的分离是中心实现思想。 一个处理子树结构，另一个处理基于邻接的结构。 

## 工作示例

 考虑一棵小树：1 是根，有子树 2 和 3，2 有子树 4。 

初始状态全为零。 

处理类型 1 查询后`1 2 5`，2的子树中的所有节点都变为5。全局线段树更新节点2和4的欧拉区间。 

处理类型2查询后`2 1 3`，子节点 2 和 3 在节点 1 的子段中都获得 +3。 

| 步骤| 运营| 全局子树值 | 儿童捐款|
 | ---| ---| ---| ---|
 | 1 | 初始化| 全部 0 | 全部 0 |
 | 2 | 添加子树(2,+5) | 节点2=5，节点4=5 | 0 |
 | 3 | 添加孩子(1,+3) | 不变| 节点2=3，节点3=3 |

 现在，节点 1 上的类型 3 查询询问 1 的子树中的最大值，这是在 global_seg 中隐式组合贡献的所有节点的最大值。 如果节点 2 或 4 在完全合并的解释中接收到两个贡献，则最大值变为 8。 

节点 1 上的类型 4 查询返回子节点中的最大值，即 max(3,3)=3。 

此跟踪显示子树和子贡献保持分离，并且查询访问正确的层。

 ## 边缘情况

 每个节点只有一个子节点的倾斜树测试子树间隔是否正确折叠。 在这种情况下，每个子树都是欧拉阶的后缀，并且重复的类型 1 操作会在重叠的间隔上累积。 该算法可以处理这个问题，因为即使在简并链中，欧拉之旅也能保证连续性。 

以根为 1 且所有其他节点为子节点的星形树强调类型 2 和类型 4。类型 2 更新适用于除根之外的所有节点，类型 4 查询需要大型邻接列表中的最大值。 每个节点的子线段树确保这些操作保持对数，并且根的子结构直接捕获所有更新，而无需扫描邻接列表。 

节点接收来自祖先的子树更新和来自其父节点的子树更新的混合序列可确保值正确组合。 由于全局线段树和子线段树是独立的，因此任意节点的最终值都是两者贡献之和，并且两个结构之间不会发生干扰。
