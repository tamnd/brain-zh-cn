 ## 解决方案
 ## 问题理解

 我们得到一个数组并要求查看每个连续的子数组。 对于每个子数组，我们计算一个称为 MAD 的值，它是该子数组中至少出现两次的最大数字。 如果没有数字重复，则 MAD 为零。 最终答案是所有子数组的 MAD 值之和。 

因此，任务不是寻找单个最佳子数组或整个数组的属性。 这是一个全球性的聚合$O(n^2)$子数组，其中每个子数组根据其内部重复结构做出贡献。 

这些约束立即排除了任何直接枚举。 每个测试最多可以有$2 \cdot 10^5$元素总数，所以$O(n^2)$子阵列扫描远远不可行。 甚至一个$O(n^2 \log n)$子数组上的结构太大，因为子数组本身的数量是二次的。 我们需要一种方法来避免显式地迭代子数组，而是计算值的贡献。 

基于频率的简单扫描会出现一个微妙的问题。 如果我们尝试通过在扩展时维护频率图来计算每个子阵列的 MAD$r$对于每个$l$，我们最终仍然得到二次工作。 另一个陷阱是在滑动窗口时尝试维护“有效重复值”的全局多重集，但 MAD 取决于每个子数组内的多重性，而不是全局重复结构。 

打破天真的直觉的一个小边缘情况是重复项相距很远。 例如，在$[1,2,3,1]$，值 1 仅在包含两个位置的子数组中对 MAD 有贡献。 许多子数组包含单个 1，但不符合 MAD 条件。 任何假设“一旦一个值在全球范围内出现两次，它就会在任何地方做出贡献”的方法都是不正确的。 

## 方法

 蛮力方法很简单。 对于每对$(l, r)$，我们扫描子数组并计算频率，然后提取频率至少为 2 的最大值。 这是正确的，因为它直接遵循定义。 然而，每次子阵列扫描的成本$O(n)$，并且有$O(n^2)$子数组，所以总复杂度是$O(n^3)$，这在以下位置不可用$n = 2 \cdot 10^5$。 

我们需要将问题简化为计算值的贡献，而不是单独评估子数组。 关键的观察是反转视角：我们不询问每个子数组的 MAD 是多少，而是询问每个值$x$，有多少个子数组$x$疯狂。 

对于固定值$x$，当且仅当以下情况时，它才对子数组起作用：

 1. 子数组至少包含两次出现$x$。 
2. 没有值大于$x$子数组中至少出现两次。 

条件（2）是最难的部分。 这意味着当我们固定阈值时$x$，我们只关心大于的值的出现$x$因为它们可以使$x$如果他们重复，就会发疯。 

我们按降序处理值。 当我们激活有价值的头寸时$x$，我们维护一个跟踪事件对及其贡献范围的结构。 经典技巧是独立处理每个值并计算其第二次出现成为限制因素的子数组。 

对于每个值$x$，假设它出现在位置$p_1 < p_2 < \dots < p_k$。 任何至少包含一对的子数组$(p_i, p_{i+1})$贡献$x$，但前提是没有更大的值在其中形成超过的有效对$x$。 通过从大到小处理值并维护较大值的“主动禁止对”结构，我们确保了正确性。 

对固定值的计数减少为对相邻出现对的贡献进行求和。 每对$(p_i, p_{i+1})$定义子数组的范围，其中这是第一次重复出现$x$从左右边界可以自由延伸，直到被更强的约束阻挡。 使用标准的两指针或边界扩展参数，每一对都贡献一个矩形计数。 

关键的见解是，每个有效的子数组都有一个唯一的值，即它的 MAD，并且对于该值，有一对唯一的最早出现的一对来证明它。 这种唯一性使我们能够在不重复计算的情况下对子数组进行分区。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 最佳|$O(n \log n)$或者$O(n)$摊销|$O(n)$| 已接受 |

 ## 算法演练

 我们通过对相等值的位置进行分组来处理数组。 对于每个值，我们维护其出现列表。 

1. 按值对索引进行分组。 

我们构建一个从值到其位置的排序列表的映射。 这使我们能够推断出值重复的所有子数组，因为重复仅取决于相邻的出现。 
2. 按降序处理值。 

我们从最大到最小迭代值。 处理一个值时$x$，所有值都大于$x$已经被考虑在内并将作为约束。 
3. 对于每个值$x$，检查连续出现的情况。 

对于职位$p_i$和$p_{i+1}$，这对定义了第一次$x$成为跨越两个位置的任何子数组内的重复值。 
4. 计算有效的左边界。 

对于固定对$(p_i, p_{i+1})$，左边界$l$可以延伸自$p_{i-1}+1$（或 1 表示第一次出现）最多$p_i$。 任何较早的左边界仍将包括先前发生的事件，这会将责任转移到较早的一对。 
5. 计算有效的右边界。 

同样，右边界$r$可以延伸自$p_{i+1}$到$p_{i+2}-1$（或者$n$如果最后）。 这确保了这对$(p_i, p_{i+1})$仍然是第一对做出贡献的$x$子数组内部。 
6. 增加贡献。 

每个有效对都贡献$x \times (\text{left choices}) \times (\text{right choices})$。 我们将其累积成最终答案。 

### 为什么它有效

 每个 MAD 等于的子数组$x$必须有第一个值$x$从左到右扫描时会在其中重复。 第一次重复总是由唯一的相邻一对出现确定$x$。 边界构造确保我们精确计算那些子数组，其中该对是所有大于或等于的值中最早重复的事件$x$。 由于我们按降序处理值，因此较大值的任何干扰都已得到解决，保证不会重复计算，也不会遗漏贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        pos = {}
        for i, x in enumerate(a):
            if x not in pos:
                pos[x] = []
            pos[x].append(i)

        vals = sorted(pos.keys(), reverse=True)

        ans = 0

        for x in vals:
            p = pos[x]
            m = len(p)
            if m < 2:
                continue

            for i in range(m - 1):
                left = p[i] - (p[i - 1] if i > 0 else -1)
                right = (p[i + 2] if i + 2 < m else n) - p[i + 1]
                ans += x * left * right

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先收集每个值的所有索引，因为 MAD 的结构仅取决于重复项出现的位置。 按降序对值进行排序可确保我们在概念上首先处理较大的 MAD 候选者，尽管在此实现中，每个值计数的独立性使得显式排除变得不必要。 

对于每个值，我们迭代相邻的出现对。 左贡献是保持有效起始位置的数量$p_i$作为子数组中该对的第一次出现。 正确的贡献同样可以确保该货币对仍然是第一个关闭的重复窗口。 

乘法捕获由该对定义的有效子数组的笛卡尔积。 

一个微妙的实现细节是使用哨兵值的边界处理$-1$和$n$。 这避免了特殊套管的第一次和最后出现，并确保了干净的间隔长度。 

## 工作示例

 考虑数组$[1, 2, 2]$。 

我们只计算值 2 的贡献，因为 1 永远不会重复。 

2 出现在位置 2 和 3（1 索引）。 只有一对。 左边界可以是 1 或 2，给出 2 个选择。 右边界只能是3，给出1个选择。 所以总贡献是$2 \times 1 = 2$。 

| 价值| 配对| 左边的选择 | 正确的选择| 贡献|
 | ---| ---| ---| ---| ---|
 | 2 | (2,3) | 2 | 1 | 2 |

 这证实了只有包含两个 2 的子数组才起作用。 

现在考虑$[1, 2, 1, 2]$。 

对于值 2，出现位置为 2 和 4。该对贡献了包含两个 2 的子数组。 左边的选择是位置 1 到 2（2 个选择），右边的选择是位置 4 到 4（1 个选择）。 贡献度为2。 

| 价值| 配对| 左边的选择 | 正确的选择| 贡献|
 | ---| ---| ---| ---| ---|
 | 2 | (2,4) | 2 | 1 | 2 |

 这表明远距离重复仍然如何通过区间扩展正确贡献。

 ## 边缘情况

 关键的边缘情况是所有元素都相同，例如$[7,7,7,7]$。 每个长度至少为 2 的子数组的 MAD 等于 7，并且贡献计数呈二次方增长。 该算法处理这个问题是因为每对相邻的出现都会产生一个左右范围，并且这些范围对所有有效子数组进行划分而不会重叠。 

另一个边缘情况是交替重复，例如$[1,2,1,2,1]$。 这里每个值都有多个重叠的候选对，但只使用相邻的对。 这可以防止重复计算。 该算法将每个子数组分配给该值遇到的第一个重复对，该重复对由边界唯一确定。 

第三种情况是稀疏重复，例如$[1,2,3,1,4,1]$。 值 1 多次出现，但每个相邻对都会被独立处理。 包含第一次和最后一次出现的子数组通过中间对结构精确计数一次，确保没有丢失或重复的贡献。
