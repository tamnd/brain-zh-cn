 ## 解决方案
 ## 问题理解

 我们得到一个整数$n$每个测试用例，我们想要选择两个不同的数字$a$和$b$在范围内$[1, n]$。 约束是二进制表示$a$和$b$不得共享两者均为 1 的任何位置，这意味着它们的按位 AND 为零。 在所有这些有效对中，我们想要最大可能的值$\gcd(a, b)$。 

因此，任务不是输出该对，而是推断两个不相交位数的哪个结构仍然可以共享大公约数。 

约束条件$n \le 10^9$迫使我们远离任何对的枚举。 甚至迭代所有可能的候选者$a$或者$b$是不可能的。 我们需要一个仅依赖于二进制结构的构造$n$，理想情况下与位数呈线性关系。 

当较小的值时会出现微妙的边缘情况$n$行为与大的不同。 例如，当$n = 3$，唯一有效的对是$(1,2)$，答案是$1$。 天真的直觉可能表明，随着数字的增长，更高的 gcd 成为可能，但二元重叠的限制阻止了这种情况的发生。 

另一个重要的失败案例是假设采用简单的结构对，例如$(g, 2g)$始终致力于最大化 gcd。 这很诱人，因为$\gcd(g, 2g) = g$，但按位条件经常由于进位或重叠二进制移位而失败。 

## 方法

 如果我们尝试暴力破解，我们将迭代所有对$(a,b)$，检查是否$a \& b = 0$，计算他们的 gcd，并跟踪最大值。 这立即变得不可行，因为有$O(n^2)$每个测试用例对，以及$n$可以达到$10^9$。 甚至将自己限制在单一的范围内$n$, 枚举至$10^9$价值观是不可能的。 

关键的结构简化来自于结构的重构。 假设我们修复了一个候选 gcd$g$。 那么两个数字必须是$g$，所以我们可以写$a = g x$,$b = g y$。 条件变为$(g x) \& (g y) = 0$。 我们不是直接推理所有这样的对，而是寻找一个特别简单的对，它在有效时总是能实现最佳的 gcd 结构。 

最好的可用结构是$a = g$,$b = 2g$。 这很有吸引力，因为它自动保证$\gcd(a,b)=g$，并减少按位条件来检查是否$g$和$2g$二进制重叠。 仅当以下情况时，移位一位才不会引入重叠：$g$没有两个相邻的设置位。 如果$g$包含连续的，移动会产生碰撞。 

这将问题简化为选择最大整数$g \le \lfloor n/2 \rfloor$其二进制表示不包含相邻的 1。 一旦我们认识到这一点，问题就变成了标准的贪婪二元构造问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解对 |$O(n^2)$|$O(1)$| 太慢了 |
 | 构造有效$g \le n/2$贪婪地|$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将每个测试用例简化为构造一个数字$m = \lfloor n/2 \rfloor$。 任务变成寻找不超过的最大整数$m$其二进制表示不包含连续的 1。 

1. 计算$m = n // 2$。 这是最大可能的值$g$， 自从$b = 2g \le n$是必须的。 
2. 从最高位到最低位一点一点地构建答案。 在每个位位置，我们尝试决定是否放置 1。 
3. 当考虑将某个位设置为 1 时，我们必须确保两个条件成立。 首先，我们不超过$m$。 其次，我们不会创建一个 1 与另一个 1 相邻的情况，这会违反有效性所需的“无连续的”结构。$g, 2g$建造。 
4. 如果设置钻头是安全的，我们将其放置； 否则我们将其保留为 0。该决策是贪婪的，因为较高的位主导该值，并且任何早期的有效布局都无法在不破坏可行性的情况下在以后进行改进。 
5. 返回构造的值作为测试用例的答案。 

### 为什么它有效

 任意最优解都可以转化为一对形式$(g, 2g)$不降低gcd。 条件$g \& (2g) = 0$等价于要求中没有两个相邻的位$g$两者都为 1，因为左移一位会在相邻位位置之间产生潜在的重叠。 因此，每个有效的候选 gcd 对应于一个没有连续的二进制字符串，最大化 gcd 相当于选择界限下最大的数字$n/2$。 贪婪构造之所以有效，是因为二进制权重是位置性的，并且首先固定较高位可以保持最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build(m: int) -> int:
    bits = []
    for i in range(31, -1, -1):
        bits.append(i)

    res = 0
    prev = 0  # whether previous bit was set

    for i in range(31, -1, -1):
        if m & (1 << i):
            # try setting bit i
            if prev == 0:
                # check feasibility: setting this bit must not exceed m in prefix sense
                # we tentatively set it and verify by greedy completion assumption
                res_candidate = res | (1 << i)
                if res_candidate <= m:
                    res = res_candidate
                    prev = 1
                else:
                    prev = 0
            else:
                prev = 0
        else:
            prev = 0

    return res

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        m = n // 2
        print(build(m))

if __name__ == "__main__":
    solve()
```代码独立处理每个测试用例。 功能`build(m)`构造最大数量不超过$m$避免连续设置位。 变量`prev`跟踪前一位是否被选为 1，从而防止邻接。 每一步都会贪婪地尝试从高到低设置位，确保结果值保持在以下范围内$m$。 

一个微妙的点是约束检查`res_candidate <= m`就足够了，因为我们是从最高位向下构建的，因此任何违反上限的行为都会立即发生在最高不同位上。

 ## 边缘情况

 当$n = 2$，我们只有一对可能的$(1,2)$，gcd 为 1。该算法计算$m = 1$，不超过 1 的最大有效数就是 1 本身，得出正确答案。 

什么时候$n$略低于 2 的幂，例如$n = 7$，中间界$m = 3$限制了建设。 虽然 3 的数值很大，但它的二进制形式$11$由于相邻的值而无效，迫使算法下降到 2。这演示了二进制约束如何直接影响最终值而不是数值大小。 

什么时候$n$是 2 的幂，例如$n = 8$，该算法适用于$m = 4$，并且 4 已经是一个干净的二进制数，没有相邻的数。 该构造保持不变地传递它，确认贪婪过程在不存在冲突时保留最优值。
