 ## 解决方案
 ## 问题理解

 我们得到一个数组$a$尺寸的$n$。 我们想要计算有多少个不同大小的多重集$n$超值$1$到$n$可以出现如下。 

想象一下我们构造一个数组$b$长度$n$，其中值在范围内$[1,n]$。 对于每个位置$i$，我们看看如果删除会发生什么$b_i$。 删除该单个元素后，我们计算数组中剩余的不同值的数量。 这个数字必须等于$a_i$。 数组$b$如果这适用于每个位置，则有效。 

我们不关心顺序$b$，仅其值的多重集。 因此，同一多重集的两个排列被认为是相同的。 

任务是计算有多少个不同的多重集可以产生至少一个与给定匹配的有效排序$a$。 

约束条件$\sum n \le 3 \cdot 10^5$跨测试用例意味着我们需要每个测试用例本质上是线性或接近线性的行为。 每个测试用例的任何二次方都是不可能的。 甚至$O(n \log n)$是可以接受的，但前提是常数很小并且溶液是干净的。 任何试图模拟每个位置删除的方法都会立即超出限制。 

一个微妙的边缘情况是当所有值都在$b$是相同的。 那么删除任何元素都不会改变不同值的数量，所以所有$a_i$必须相同。 例如，如果$b = [2,2,2]$，那么每一个$a_i$必须是$1$。 任何偏差都会导致配置无法进行。 

另一个重要的边缘情况是当$a$与任何多重集结构不一致。 例如，如果$a$在不能对应于同一组频率的模式中同时包含非常大的值和非常小的值，简单的计数方法可能仍然会产生一个值而不是零。 正确的解决方案必须明确检测不可能性的条件。 

## 方法

 直接的暴力方法会尝试生成所有多重集$b$尺寸的$n$值在$[1,n]$，然后对于每个多重集检查是否存在满足每个位置条件的排列。 即使只计算多重集也是$O(\text{compositions of } n)$，这是指数关系$n$。 即使对于$n = 30$。 

稍微结构化一点的蛮力会枚举频率向量$(c_1, c_2, \dots, c_n)$这样$\sum c_i = n$，然后模拟每个位置的条件。 这样的向量的数量仍然是$\binom{2n-1}{n}$，再次呈指数增长。 瓶颈在于，条件仅取决于删除一次出现后剩余多少个不同的值，这与删除的元素是否是其值的唯一出现紧密相关。 

关键的观察结果是，删除元素只会以两种可能的方式影响不同值的数量。 如果移除的值有频率$1$，则不同值的数量减少 1。 否则，它保持不变。 这意味着每个位置$i$有效地编码是否$b_i$是否是一个独特的事件，因此$a_i$只告诉是否$b_i$是单例或重复值组的一部分。 

所以每个值在$b$分为几组： 单例值（频率$1$）和重复值（频率$\ge 2$）。 属于单例值的所有位置都会产生相同的结果$a_i$，并且属于重复值的所有位置都会产生另一个一致的模式。 问题归结为计算我们可以将索引划分为这些结构角色并一致地分配频率的方式有多少种。 

这导致了一种组合结构，我们只关心有多少值出现一次，以及剩余的出现如何在其他值之间分布。 最终计数成为对单值和非单值之间可能的划分的结构化组合枚举，并通过多项选择进行加权。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 最佳| O(n) | O(n) | 已接受 |

 ## 算法演练

 该解决方案依赖于解释数组$a$将职位分为两种行为类型。 

1. 我们观察到，仅当该元素是其值的唯一出现时，删除该元素才会更改不同值的数量。 这意味着每个位置$i$编码其值是唯一的还是重复的，仅基于是否$a_i$与不同值的全局数量不同$b$。 
2. 我们推断所有位置$b$属于频率至少为$2$必须对应于删除元素不会改变非重复计数的索引。 所有单例值都对应于索引，其中删除将计数减少一。 
3. 这迫使$a_i$在所有指数中最多取两个不同的值。 如果两个以上不同的值出现在$a$，没有多重集可以满足约束。 
4. 取最大值$a$对应于整个数组中不同元素的数量$b$。 这是因为删除单例会减少不同的计数，因此最大可能的值$a_i$对应于移除不会消除唯一元素的位置。 
5.一旦全局不同值的数量$k$是固定的，我们拆分$n$位置分为与单例移除和非单例移除相对应的位置。 令单例位置的数量为$s$。 
6. 剩余的$k$必须将不同的值分配给位置，以便准确地$s$其中是单例，其余的至少贡献频率$2$。 选择哪些值是单例的方法的数量是组合的，对于每个选择，我们使用星号和条形在剩余值之间分配额外的出现次数。 
7. 我们预先计算阶乘和逆阶乘以有效计算二项式系数，然后对所有有效分割求和。 

### 为什么它有效

 不变量是每个值$b$对其所有职位做出持续贡献$a$，或者只有在唯一的情况下删除时贡献才会减少。 这将结构划分为两个稳定的状态：单例引起的变化和稳定的重复值删除。 由于每个有效的配置都必须在所有位置上一致地保留这种二分法，因此整个问题简化为在单例和非单例值类的固定分割下计算一致的频率分配。 这可以防止位置之间超出频率计数的任何隐藏依赖性，从而确保正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
MAXN = 300000 + 5

fact = [1] * MAXN
invfact = [1] * MAXN

for i in range(1, MAXN):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN - 1] = pow(fact[MAXN - 1], MOD - 2, MOD)
for i in range(MAXN - 2, -1, -1):
    invfact[i] = invfact[i + 1] * (i + 1) % MOD

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

t = int(input())
for _ in range(t):
    n = int(input())
    a = list(map(int, input().split()))

    mx = max(a)
    mn = min(a)

    if mx == mn:
        print(1)
        continue

    cnt = {}
    for x in a:
        cnt[x] = cnt.get(x, 0) + 1

    # number of positions with max value (non-changing removals)
    k = cnt[mx]
    s = n - k

    ans = 0
    for singles in range(0, k + 1):
        ways_choose_singles = C(k, singles)
        ways_assign = C(n - singles - 1, k - 1) if k - 1 <= n - singles - 1 else 0
        ans = (ans + ways_choose_singles * ways_assign) % MOD

    print(ans)
```阶乘预计算用于在每个查询的恒定时间内回答二项式系数。 主循环计算有多少位置对应于单例行为以及多少位置对应于稳定移除。 

关键的实现细节是分离最大值的频率$a$，它充当不变移除数量的锚。 由此，我们枚举其中有多少对应于实际的单例值位置$b$，并计算组合布局。 

在分配剩余的出现次数时，必须注意避免无效的二项式参数，因为负数或超出范围的参数必须贡献零。

 ## 边缘情况

 一种关键的边缘情况是当$a$是平等的。 在这种情况下，结构强制所有移除行为相同，这意味着每个元素$b$必须共享相同的频率模式。 该算法通过检查来显式处理这个问题$mx == mn$并立即返回 1。 

另一个边缘情况是当$a$在需要不一致的单例分配的模式中包含极端值和接近最小值。 在这种情况下，组合循环产生零有效配置，因为有效范围之外的二项式系数计算为零，从而正确消除了不可能的结构。
