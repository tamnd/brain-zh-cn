 ## 解决方案
 ## 问题理解

 我们给出了一个大小为的矩形网格$n \times m$，其中每个单元格包含 0 或 1。在一次移动中，我们可以精确地选择一个当前包含 1 的单元格。一旦选择，与其共享边或角的每个单元格都变为 1。所选单元格本身保持不变，并且不允许我们多次执行此操作。 

问题是是否存在起始 1 单元格的有效选择，以便在应用此单一扩展后，整个网格都充满 1。 

重要的观点是将操作视为选择一个中心单元，然后“覆盖”其周围的 3×3 块。 操作完成后，只有所选单元格的 3×3 邻域内的单元格才可能从 0 变为 1。该块之外的所有内容都将永远保持不变。 

约束允许最多 500×500 的网格，即大约 250,000 个单元格。 任何尝试模拟每个可能中心的操作并重新计算整个网格的解决方案都需要大约$O(n^2 \cdot m^2)$在最坏的情况下，这个数字太大了。 这促使我们对全局结构进行推理，而不是进行简单的模拟。 

一些边缘情况很重要：

 如果网格已经只包含 1，则答案很简单，因为不需要任何操作，所以答案就是“WIN”。 

如果距离任何潜在中心正好有一个零，例如：```
1 1 1
1 1 1
1 1 0
```那么除非选择的中心是相邻的，否则没有一个 3×3 邻域可以覆盖它，所以答案一定是“LOSE”。 

天真的思维的一个微妙的失败案例是假设如果零“靠近”，那么它总是可能的。 例如：```
1 0 1
0 0 0
1 0 1
```即使零是聚集的，问题也归结为以 1 单元为中心的单个 3×3 块是否可以同时覆盖所有零。 

## 方法

 蛮力的想法很简单：尝试每个包含 1 作为所选中心的单元格，模拟将其所有 8 个邻居变为 1，然后检查网格是否全部变为 1。 每次模拟费用$O(nm)$，并且最多可以有$nm$候选中心，导致$O((nm)^2)$，这大约是$6.25 \times 10^{10}$最坏情况下的操作。 这已经远远超出了极限。 

关键的观察结果是该操作仅影响固定的局部区域，特别是所选单元格周围的 3×3 正方形。 这意味着唯一重要的零点是它们相对于潜在中心的位置。 我们可以从零开始推理，而不是测试中心。 

如果操作必须在一次移动中将所有零变成 1，则每个零必须位于所选中心的 3×3 邻域内。 这立即意味着一个几何约束：所有零单元必须适合某个 3×3 窗口。 一旦找到全零的边界框，我们只需要检查它是否适合这个区域，以及是否存在至少一个可以选择的有效中心单元（1）。 

这减少了从测试许多中心到检查从零分布导出的恒定数量条件的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O((nm)^2)$|$O(1)$额外 | 太慢了|
 | 零边界框 |$O(nm)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 ### 1. 找到所有零单元格

 扫描网格并记录所有包含 0 的单元格中的最小和最大行索引和列索引。如果根本没有零，则网格已经完全填满，我们立即返回“WIN”。 

此步骤有用的原因是零是唯一的障碍； 那些是无关紧要的，因为它们永远不需要改变。 

### 2. 计算零的边界矩形

 设最小和最大的零行索引为$r_{min}$和$r_{max}$，并且类似地$c_{min}$和$c_{max}$。 

该矩形表示包含所有有问题的单元格的最小轴对齐框。 

### 3. 检查单个操作的几何可行性

 由于一次操作仅影响 3×3 邻域，因此所有零都必须位于这样的邻域内。 这相当于：$$r_{max} - r_{min} \le 2 \quad \text{and} \quad c_{max} - c_{min} \le 2$$如果任一维度超过 2，则没有一个中心可以覆盖所有零。 

### 4. 验证是否存在有效的中心单元

 即使所有零都适合 3×3 区域，该区域的中心也必须是值为 1 的单元格，因为只能选择 1 个单元格。 

因此，我们检查是否存在至少一个值为 1 的单元格位于可以覆盖零边界框的所有有效中心的交集中。 该区域是：$$[r_{min}+1, r_{max}-1] \times [c_{min}+1, c_{max}-1]$$如果该区域中至少有一个1-cell，则操作可以集中在那里。 

### 为什么它有效

 该操作仅在每个方向上扩展影响一步，因此每个受影响的单元格必须位于所选中心的切比雪夫距离 1 内。 该约束既是必要的也是充分的：必要性来自于操作定义，充分性来自于任何这样的 3×3 邻域在一次移动中被完全覆盖的事实。 因此，整个问题简化为检查所有零是否可以包含在锚定于 1 个单元的单个此类邻域中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = []
    
    rmin, rmax = n, -1
    cmin, cmax = m, -1
    has_zero = False
    
    for i in range(n):
        row = list(map(int, input().split()))
        grid.append(row)
        for j, v in enumerate(row):
            if v == 0:
                has_zero = True
                rmin = min(rmin, i)
                rmax = max(rmax, i)
                cmin = min(cmin, j)
                cmax = max(cmax, j)
    
    if not has_zero:
        print("WIN")
        return
    
    if rmax - rmin > 2 or cmax - cmin > 2:
        print("LOSE")
        return
    
    # search for valid center
    for i in range(n):
        for j in range(m):
            if grid[i][j] != 1:
                continue
            if rmin <= i <= rmax and cmin <= j <= cmax:
                # must ensure center can cover bbox with radius 1
                if rmin >= i - 1 and rmax <= i + 1 and cmin >= j - 1 and cmax <= j + 1:
                    print("WIN")
                    return
    
    print("LOSE")

if __name__ == "__main__":
    solve()
```该实现首先将问题压缩到零位置的几何结构中。 早期的退出处理琐碎的全一网格和不可能的拉伸零模式。 最终的嵌套扫描仅检查候选 1-cell，并且条件确保所选中心的 3×3 邻域完全包含全零。 

一个常见的错误是仅检查边界框大小，而不验证可行中心区域内是否存在有效的 1-cell。 这个额外的约束是必不可少的，因为空的或无效的中心集否则会错误地产生“WIN”。

 ## 边缘情况

 当网格不包含零时，算法立即返回“WIN”，因为没有什么可修复的。 边界框变量保持未初始化状态，但过早退出会阻止它们的使用。 

当零紧密聚集但移向边缘时，边界框条件可能仍会通过，但中心可行性检查会拒绝所需区域中不存在有效 1-cell 的情况。 这可以防止在几何形状允许覆盖但不存在合法起始单元的情况下错误地接受图案。 

当零组成一个完美的 3×3 块时，该块内的任何 1 个单元都可以作为中心。 该算法通过边界框大小和至少一个有效中心单元的存在来检测这一点，即使存在多个候选单元，也能确保正确性。
