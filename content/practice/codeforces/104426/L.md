 ## 解决方案
 ## 问题理解

 我们将具有整数坐标的点放置在无限网格上，并且我们希望至少适合$K$以原点为中心的圆内部或边界上的不同网格点。 任务是找到最小的整数半径$R$使得满足的格点数$x^2 + y^2 \le R^2$至少是$K$。 

每个人对应一个格点，并且没有两个人可以占据相同的坐标，因此问题简化为计算一个圆内有多少个整数对。 半径必须是整数，并且要求我们提供最小的半径，该半径至少允许$K$有效职位。 

关键约束是$K \le 10^9$。 这立即排除了网格点的任何直接模拟或强力枚举，因为圆中网格点的数量大致与面积成正比，即$O(R^2)$。 对半径和每个半径的点数进行简单扫描将需要重复重新计算格点计数，一旦计算起来就会变得太慢$R$增长为大值。 

一个微妙的边缘情况是$K$非常小。 例如，如果$K = 2$，答案是$1$，因为半径$0$只包含原点。 另一个边缘情况是当$K$就在完美对称的晶格点层之上，其中缺少或包含边界点会突然改变计数。 任何近似圆形面积而不是计算精确整数点的解决方案在这里都会失败。 

## 方法

 蛮力方法会尝试增加半径$0$向上，并且对于每个半径计数所有满足的整数点$x^2 + y^2 \le R^2$。 对于每个$R$，这需要迭代所有$x$在$[-R, R]$并计算有多少有效$y$价值观是存在的。 这是$O(R)$每个半径，并且因为$R$本身可以上升到大约$10^5$或更多对于大$K$，总工作量大致变为$O(R^2)$，这对于$K = 10^9$。 

关键的观察是单调性。 随着半径的增加，圆内的格点数量不会减少。 这意味着我们可以对答案进行二分搜索$R$。 缺少的部分是如何计算固定的格点数量$R$高效。 我们不是枚举所有点，而是迭代整数$x$从$0$到$R$，并且对于每个$x$,确定最大整数$y$这样$x^2 + y^2 \le R^2$。 这给出了$y = \lfloor \sqrt{R^2 - x^2} \rfloor$。 每一个这样的$x$贡献$2y + 1$点（考虑所有四个象限和轴的对称性）。 这将计数减少到$O(R)$，并结合二分查找$R$，我们得到了一个有效的解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(R^2)$|$O(1)$| 太慢了|
 | 二分查找+计数|$O(R \log R)$|$O(1)$| 已接受 |

 ## 算法演练

 我们利用圆内格点的数量随着半径稳定增长，因此我们可以搜索至少达到的最小半径$K$点。 

## 算法演练

 1. 定义一个函数`count(R)`计算有多少个整数格点位于半径圆内部或半径圆上$R$。 我们通过扫描所有整数来做到这一点$x$从$0$到$R$, 计算$y_{\max} = \lfloor \sqrt{R^2 - x^2} \rfloor$，并添加$2y_{\max} + 1$到计数。 我们只迭代非负数$x$并使用对称性，避免冗余计算。 
2. 对于每个半径$R$，值$y_{\max}$代表我们在该水平切片上可以垂直移动多远。 这确保了我们精确地计算所有满足圆不等式的整数点，而不会丢失边界点。 
3. 设置二分查找范围。 下界是$0$，并且上限可以安全地设置为类似的值$2 \cdot 10^5$，因为面积增长确保我们达到$10^9$远早于该半径的点。 
4. 执行二分查找。 对于中半径，计算`count(mid)`。 如果大于或等于$K$，我们尝试较小的半径。 否则，我们增加半径。 
5、继续二分查找，直到收敛，返回满足条件的最小半径。 

### 为什么它有效

 对于任意半径$R$，满足的格点集合$x^2 + y^2 \le R^2$完全由整数几何决定并且单调增长为$R$增加。 计数函数是精确的而不是近似的，因为每个整数$x$与有效的精确整数范围配对$y$。 这保证了二分搜索在没有歧义的单调谓词上运行，因此第一个半径达到$K$是最小的有效值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def count_points(R):
    res = 0
    r2 = R * R
    for x in range(R + 1):
        y2 = r2 - x * x
        if y2 < 0:
            break
        y = math.isqrt(y2)
        res += 2 * y + 1
    return res

def solve():
    K = int(input().strip())

    lo, hi = 0, 200000

    while lo < hi:
        mid = (lo + hi) // 2
        if count_points(mid) >= K:
            hi = mid
        else:
            lo = mid + 1

    print(lo)

if __name__ == "__main__":
    solve()
```功能`count_points`是解决方案的核心。 它计算第一象限中的精确格点，并通过隐式乘以对称性$2y + 1$结构每$x$。 循环提前停止一次$x^2 > R^2$，防止不必要的迭代。 

二分搜索维护一个候选半径范围，并根据当前半径是否已经支持至少缩小它$K$点。 

一个常见的实现陷阱是使用浮点平方根，这可能会在大值时引入精度误差。 使用`math.isqrt`确保整数正确性。

 半径 1 不够，但半径 2 已经超过 6，所以答案是 2。

 ## 边缘情况

 一个关键的边缘情况是当$K = 1$。 唯一有效的点是原点，因此半径$0$必须归还。 计数函数正确返回 1$R = 0$，因此二分查找立即确定为 0。 

另一种情况是当$K$正好低于晶格密度的大幅跃升。 例如，附近$R = 2$，计数从 5 跳到 13。任何使用面积的近似方法$\pi R^2$可能会根据舍入错误地选择半径 1 或 3。 精确的整数计数完全避免了这种情况，因为每个格点都是通过整数几何而不是近似来枚举的。 

最后一个微妙的情况是边界包含的正确性。 积分满意$x^2 + y^2 = R^2$必须包括在内。 使用`isqrt`确保正确处理相等情况，因为它以整数形式返回平方根的下限，保留边界点而不会出现浮点漂移。
