 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个问题中，我们都从一系列困难问题开始。 我们可以删除这些值的任何子集，但必须至少留下一个元素。 删除后，剩余组的“分数”定义为其算术平均值。 

任务是选择一个非空子集，其平均值尽可能小。 

约束立即塑造解决方案空间。 所有测试用例的元素总数最多为 100000，因此任何对每个测试用例进行二次甚至重复重新扫描子集的方法都会失败。 我们需要每个测试用例都是线性的或者更好的东西。 

一个微妙的问题是，最佳子集不一定是所有元素或只是最小元素。 例如，如果我们有`[100, 1, 2]`，仅取`1`给出平均值`1`，这是最优的。 但在`[1, 100, 100]`，仅取`1`仍然是最好的，即使大多数元素都很大。 这暗示大元素永远不会有助于降低平均值，但我们需要准确说明原因。 

边缘情况出现在选择大小周围：

 如果所有数字都相等，则任何子集都具有相同的平均值，因此删除元素不会执行任何操作。 简单的方法可能仍会尝试复杂的子集评估，但答案仍然是该值。 

另一个角落是当存在唯一的最小元素时。 即使如此，添加任何更大的元素都会严格增加平均值，因此最佳子集仍然是包含最小值的单例。 

错误推理的主要风险是假设我们可能需要多个元素来“平衡”平均值。 在没有约束的情况下，这种直觉是错误的。 

## 方法

 强力解释将考虑每个非空子集，计算其总和和大小，并跟踪最小平均值。 这是正确的，但立即不可行。 子集的数量是指数级的，具体来说$2^N - 1$，这对于$N = 10^5$甚至无法表示，更不用说迭代了。 

我们需要了解添加元素时平均值的表现。 假设我们选择了一组平均数$x$。 如果我们添加一个新元素$a$，新的平均值变为$$\frac{S + a}{k + 1}$$在哪里$S/k = x$。 这个新平均值严格小于$x$仅当$a < x$。 这给出了一个结构性的见解：提高平均值需要添加小于当前平均值的元素。 

现在考虑从任何有效的子集开始。 如果它包含多个元素，则其平均值至少是该子集中的最小元素，并且包含任何大于当前平均值的元素也无济于事。 这表明了一个贪婪的收缩论点：删除元素永远不会增加对最小平均值的控制，并且添加元素不能击败已经存在的最小元素，除非它们更小。 

因此，最优策略分解为一个非常简单的观察：通过采用由单个元素组成的子集来实现最佳可能的平均值，并且为了最小化它，我们选择数组中的最小元素。 

我们比较以下方法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举所有子集 | O(2^N) | O(2^N) | O(N) | 太慢了|
 | 通过排序或 DP | 检查所有子集平均值 O(N^2) 或更糟 | O(N) | 太慢了|
 | 取最小元素 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 对于每个测试用例，阅读一系列困难。 我们独立对待每个测试，因为删除不会跨测试用例交互。 
2. 扫描一次数组，同时跟踪遇到的最小值。 这个想法是为最终子集平均值保留最佳可能的候选者。 
3.处理完所有元素后，输出最小值。 这对应于选择仅由该单个最小元素组成的子集。 

### 为什么它有效

 任何子集的平均值至少是其最小元素，因为数字的平均值不能小于集合中的最小值。 因此，每个有效子集的平均值的下限是数组的全局最小值。 通过选择仅包含该最小元素的子集可以实现此界限，因此没有其他构造可以产生更小的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n = int(input())
    arr = list(map(int, input().split()))
    print(min(arr))
```该解决方案完全依赖于计算每个测试用例的单个最小值。 测试用例的循环确保我们尊重输入分离。 使用`min(arr)`是安全的，因为每个测试用例的数组大小至少为 1，因此我们永远不会违反子集必须保持非空的要求。 

关键的实现细节是我们根本不尝试构建或跟踪子集。 考虑到数学上减少到单个值，任何此类尝试都是不必要的开销。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        out.append(str(min(arr)))

 ## 边缘情况

 单元素数组是最严格的约束：唯一有效的子集是元素本身，因此算法直接输出它。 用于输入`[1234]`，扫描将最小值设置为 1234 并毫无歧义地返回它。 

对于最小值出现多次的数组，例如`[2, 100, 2, 50]`，算法在多个位置遇到 2，但运行最小值始终保持为 2，并且输出为 2。任何包含其他元素的子集都不能将平均值降低到 2 以下，因此重复项不会改变行为。 

对于严格增加的数组，例如`[1, 2, 3, 4]`，最小值立即在第一个元素处找到。 即使后面的元素更大，该算法也永远不会替换当前的最佳值，从而正确地忽略所有添加。 

这些案例证实，解决方案仅取决于确定全局最小值，并且排序或分组的结构属性不会影响最终结果。
