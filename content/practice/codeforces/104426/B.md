 ## 解决方案
 ## 问题理解

 我们有一棵具有指定根的树。 任务是分配顶点的排列，这意味着每个顶点接收一个从 1 到 n 的唯一整数，以便标签沿着每个根到叶方向严格增加。 形式上，每当节点 u 位于从根到另一个节点 v 的路径上时，分配给 u 的值必须小于分配给 v 的值。在所有有效分配中，当我们按顶点顺序 1 到 n 列出值时，我们希望得到字典顺序最小的序列。 

这不是变相的一般偏序问题，它实际上是一个有根树约束，其中每个根到节点的路径必须严格递增。 该树产生了优先级约束的层次结构，但与任意 DAG 不同，每个节点都有一个祖先链。 

约束 n 高达 2·10^5 会立即消除任何尝试枚举排列或对状态执行任何全局排序的解决方案。 即使 O(n log n) 也是可以接受的，但前提是它本质上是每个节点的线性工作。 任何二次或阶乘都是不可能的。 树结构建议采用仔细排序的 DFS 或 BFS 遍历。 

词典编纂的极简性产生了一个微妙的问题。 如果我们只被要求产生任何有效的排列，我们可以任意按深度递增的顺序分配标签。 但是字典极小性迫使早期顶点接收尽可能小的值，并且由于输出是按顶点 id 索引的，因此我们必须首先最小化编号较小的顶点的值，而不是遍历顺序。 

另一种边缘情况是当根不是顶点 1 时。许多简单的 DFS 方法假设根 1 并分配递增的发现顺序。 这是失败的，因为字典顺序取决于顶点索引，而不是访问顺序。 另一个陷阱是使用 DFS 顺序而不强制按顶点标签的递增顺序处理节点的子节点，这是最小化早期分配所必需的。 

## 方法

 强力解释将尝试构建排列并检查有效性。 人们可以想象生成 1 到 n 的所有排列，验证每个排列是否每条边都遵循祖先排序。 检查有效性需要计算祖先关系或对每个候选者执行 DFS，每次检查的时间复杂度为 O(n)。 这已经变成了 O(n!·n)，即使对于 n = 12 也是完全不可行的。 

一种稍微结构化的强力方法是将约束视为偏序并尝试拓扑排序，同时始终选择最小的可用顶点。 然而，关键的困难在于约束不是任意的边，而是有根树中的祖先关系。 该结构意味着一旦一个节点被分配了一个值，它的所有后代都必须稍后出现，并且不同子树之间的相对顺序是自由的。 

关键的观察结果是，约束仅强制沿着根到叶路径进行排序。 这意味着我们构建的排列必须是有根树偏序的线性扩展。 字典顺序上最小的这种线性扩展是通过始终以递增的顶点 ID 顺序将最小的可用标签分配给最早可能的顶点来获得的，同时尊重父级必须在子级之前分配。 

这将问题简化为构建有根树的拓扑排序，其中每个节点都位于其所有后代之前，并且在所有有效顺序中，我们希望使标签数组 p[1..n] 按字典顺序最小化。 这是通过模拟一个过程来实现的，在该过程中，我们按升序分配从 1 到 n 的标签，始终选择其父节点已被分配的编号最小的顶点。

我们维护哪些节点“可”接收下一个标签：一旦其父节点已经接收到标签，节点就变得可用。 在每一步中，我们都会选择最小的索引可用节点。 这种贪婪的选择确保了字典最小性，因为 p 中较早的位置对应于较小的标签，并且我们总是按升序将标签分配给与约束一致的最小可能顶点索引。

 ## 算法演练

 我们将问题重新解释为构建排列 p[1..n]，其中 p[v] 是分配给顶点 v 的标签，但反向思考更容易：我们按顺序分配从 1 到 n 的标签。 

1. 以 x 为树根并计算邻接表。 我们还使用从 x 开始的 DFS 或 BFS 来跟踪父子关系。 这确保我们知道哪些节点依赖于其他节点。 
2. 维护可用顶点的优先级结构（最小堆）。 如果顶点的父顶点已分配了标签，则该顶点可用。 最初，只有根可用，因为它没有父约束。 
3. 重复从堆中提取最小索引顶点，并按升序为其分配下一个标签。 一旦我们为顶点 u 分配了标签，我们就将 u 标记为已处理。 
4. 对于 u 的每个子节点 v，减少其依赖关系，如果满足其所有先决条件（在树中这意味着现在已处理其父节点），则将 v 推入堆中。 
5. 继续，直到所有顶点都被分配了标签。 

关键的设计选择是我们总是选择最小的可用顶点，而不是按 DFS 或 BFS 顺序最早出现的顶点。 这就是强制词典编排最优性的原因。 

它的工作原理与两个耦合的不变量有关。 首先，当顶点的父顶点已经被分配了更小的标签时，顶点就会被插入到可用集合中，因此祖先约束总是得到满足。 其次，在任何步骤中，在可以合法接收下一个最小标签的所有顶点中，选择最小索引可以最小化结果排列中两个有效解决方案可能不同的最早位置。 任何偏差都会增加第一个不同索引处的 p，违反字典最小性。 因为树约束沿着祖先是单调的，所以一旦节点变得可用，就可以安全地延迟它而不会破坏有效性，但是延迟较小索引的节点会立即恶化字典顺序，因此必须立即选择它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    parent = [0] * (n + 1)
    order = []
    stack = [x]
    parent[x] = -1

    while stack:
        u = stack.pop()
        order.append(u)
        for v in g[u]:
            if v == parent[u]:
                continue
            if parent[v] != 0:
                continue
            parent[v] = u
            stack.append(v)

    import heapq
    heap = []
    assigned = [False] * (n + 1)

    heapq.heappush(heap, x)
    label = 1
    p = [0] * (n + 1)

    while heap:
        u = heapq.heappop(heap)
        assigned[u] = True
        p[u] = label
        label += 1

        for v in g[u]:
            if parent[v] == u and not assigned[v]:
                heapq.heappush(heap, v)

    print(*p[1:])

if __name__ == "__main__":
    solve()
```该解决方案使用显式父数组构建有根树，然后使用最小堆始终选择当前可用的最小顶点。 每个顶点都按升序分配标签，确保满足祖先约束。 

父数组至关重要，因为它定义节点何时变得合格。 我们仅在处理完父级后才推送子级。 堆通过在有效候选者之间选择顶点索引来确保词典编排的最小性。 

一个微妙的实现细节是初始化：最初只插入根。 如果我们错误地推送所有节点或在没有堆的情况下按 DFS 顺序进行遍历，即使结果仍然是有效的拓扑排序，字典顺序条件也可能会失败。

 ## 边缘情况

 一种边缘情况是树是单链。 堆在每一步只包含一个节点，因此算法退化为简单的线性遍历。 例如，对于根 1 和边 1-2、2-3、3-4，堆永远不会分支，并且标签按严格的路径顺序分配，根据需要生成 1 2 3 4。 

另一个例子是一颗星星扎根于一片叶子上。 如果 root 为 1 并且所有其他节点都连接到它，则在分配 root 后，所有子节点同时变得可用。 堆确保它们以递增的顶点顺序进行处理，这对于字典最小性是必要的。 如果没有堆，任何 DFS 顺序都会产生有效的排列，但不一定是最小的排列。 

更微妙的情况是根不是顶点 1 并且最小的顶点位于树的深处。 该算法仍然仅在其父链被解析时才发现它，并且一旦可用，就会由于堆排序而立即选择它。 这可以防止过早分配较大索引的节点，否则会破坏字典顺序。
