 ## 解决方案
 ## 问题理解

 我们正在与一家$n \times n$网格，其中每个单元格可以为空，也可以包含整数大小的珍珠。 网格通过一系列操作进行更新，其中每个操作要么在单元格中放置一颗珍珠，要么替换现有珍珠，要么将其删除。 

每次更新后，我们必须确定当前配置是否满足全局一致性规则：每对水平或垂直相邻的占用单元必须具有相同的珍珠大小之和。 换句话说，如果我们查看连接两个都包含珍珠的相邻单元格的任何边，则此时网格中所有此类边的值之和必须相同。 

关键的困难在于这个条件是全局的并且同时依赖于所有相邻对，并且必须在动态点更新下维持它。 

限制条件$n, q \le 10^5$立即排除每次更新后检查网格的任何方法。 在密集配置中，即使迭代已更改单元的所有邻居也可能太慢，因为单个单元可能最多有四个邻居，但网格大小太大而无法显式维护。 真正的问题是我们甚至无法存储或遍历整个网格。 

当根本没有相邻的占用单元对时，就会出现一种微妙的极端情况。 在这种情况下，条件完全为真，所以答案应该是“是”。 错误地假设必须存在至少一条边的幼稚实现可能会错误地输出“否”。 

当值以边缘消失的方式更新时，会出现另一种边缘情况。 例如，如果两个相邻单元格都包含值并且其中一个被删除，则约束不再适用于该对。 未正确删除与已删除单元格相关的约束的解决方案可能会错误地继续强制执行过时的条件。 

## 方法

 强力解释将维护整个网格，并在每次更新后扫描所有单元并检查每对相邻的占用邻居。 这是有效的，因为条件纯粹是本地的：我们只比较相邻的单元格。 然而，在最坏的情况下，电网$n^2$单元格和每次更新都需要扫描$O(n^2)$职位，导致$O(n^2 q)$的操作，这是完全不可行的。 

关键的观察结果是，条件不是关于绝对值，而是关于边缘之间的差异。 如果每个有效边都有相同的和$S$，那么对于任意两个相邻的占用单元格$u, v$，我们有$a_u + a_v = S$。 这意味着强大的结构限制：一旦像元值固定，其所有邻居都受到约束，并且只有在形成或修改新边缘时才会在局部出现不一致。 

我们不检查整个网格，而是维护一组活动边缘（相邻占用单元对）。 我们跟踪这些边的总和并确保它们全部匹配单个全局候选值。 困难在于边缘是动态的，因此当更新单元格时，只有与该单元格相关的边缘会发生变化。 这使我们能够逐步保持一致性。 

我们维护边和的多重集（或频率图），并确保所有活动边中最多存在一个不同的和。 如果边缘为零，那么答案显然是“是”。 每次更新最多影响四个边，因此我们可以在每次操作的恒定时间内更新结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2 q)$|$O(n^2)$| 太慢了|
 | 最佳 |$O(q)$|$O(n + q)$| 已接受 |

 ## 算法演练

 我们将每个占用单元视为动态图中的一个节点，其中正交相邻占用单元之间存在边。 每条边都有一个约束：其端点值必须加起来为一个公共全局值。 

1. 我们维护一个从单元格坐标到值的哈希映射。 如果单元格为空，则不会存储它。 这可以避免实现整个网格。 
2. 我们维护第二个结构，用于跟踪相邻占用单元之间的所有当前边缘和。 每条边都被考虑一次，例如仅计算右邻和下邻。 
3. 对于单元格的每次更新$(x, y)$，我们首先删除该单元格的贡献（如果它以前存在）。 这需要删除所有连接的边$(x, y)$到其现有的被占领的邻居。 这些边的总和将从我们的频率结构中删除。 
4.然后我们插入新值$v$如果它非零。 插入后，我们在之间添加后边$(x, y)$及其当前占领的邻居，相应地更新其总和。 
5. 处理更新后，我们检查所有活动边是否共享相同的总和。 这相当于验证不同边和的集合的大小至多为一。 
6. 如果没有边，我们输出YES。 否则，只有当恰好存在一个不同的边和时，我们才输出 YES。 

为什么每个步骤都很重要是局部一致性：每次更新仅更改涉及修改单元的邻接关系，因此可以通过仅修复其邻域来更新全局有效性。 

### 为什么它有效

 该条件定义了所有活动边上的统一约束。 由于每个约束恰好涉及两个端点，因此任何违规都必须出现在至少一个边缘上。 更新仅修改与更新的单元相关的边，因此所有其他边保留其先前的有效状态。 因此，维护边缘和的多集足以检测是否出现冲突的和。 如果存在多个不同的总和，则没有全局值$S$可以同时满足所有约束条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, q = map(int, input().split())

grid = {}
from collections import defaultdict

cnt = defaultdict(int)
distinct = 0

def add(val):
    global distinct
    if cnt[val] == 0:
        distinct += 1
    cnt[val] += 1

def remove(val):
    global distinct
    cnt[val] -= 1
    if cnt[val] == 0:
        distinct -= 1

def get(x, y):
    return grid.get((x, y), 0)

for _ in range(q):
    x, y, v = map(int, input().split())

    if (x, y) in grid:
        old = grid[(x, y)]

        for dx, dy in [(1, 0), (-1, 0), (0, 1), (0, -1)]:
            nx, ny = x + dx, y + dy
            if (nx, ny) in grid:
                neighbor = grid[(nx, ny)]
                remove(old + neighbor)

        del grid[(x, y)]

    if v != 0:
        for dx, dy in [(1, 0), (-1, 0), (0, 1), (0, -1)]:
            nx, ny = x + dx, y + dy
            if (nx, ny) in grid:
                neighbor = grid[(nx, ny)]
                add(v + neighbor)

        grid[(x, y)] = v

    if distinct <= 1:
        print("YES")
    else:
        print("NO")
```该实现仅将占用的单元存储在字典中，这避免了不可能的情况$n^2$内存占用。 每次更新都会在插入新边缘之前仔细删除旧边缘，以确保过时的贡献永远不会保留在结构中。 

这`cnt`映射跟踪边和的频率，而`distinct`跟踪当前存在多少个不同的金额。 这允许在每次更新后进行恒定时间检查。 唯一微妙的一点是，我们必须在删除单元格之前删除旧边缘，并在插入后重新添加新边缘，否则我们会错过边缘或重复计算边缘。

 ## 边缘情况

 关键的边缘情况是在先前形成边缘后删除所有单元格。 例如，如果存在两个相邻单元格并形成有效的和，则两者都被删除，系统必须返回“是”，因为不存在任何约束。 该算法可以处理此问题，因为删除两个端点会删除所有相应的边和条目，从而使频率图为空。 

另一个边缘情况是同一单元上的重复更新。 如果一个单元被多次覆盖，则在添加新边缘之前必须删除旧边缘。 该实现通过始终在插入之前处理删除来确保这一点，因此不会保留过时的边缘和。 

最后一种边缘情况是单元格在空状态和非空状态之间快速切换。 由于每次转换都会完全重建其邻接贡献，因此系统保持一致，并且不会累积重复的边计数。
