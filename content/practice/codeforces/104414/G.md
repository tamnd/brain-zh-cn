 ## 解决方案
 ## 问题理解

 我们得到一个数字$x$和一个值范围$[L, R]$。 我们想要选择一个正整数$y$这样当我们将其乘以$x$，产品变成一个完美的正方形。 同时，该产品必须保持在区间内$[L, R]$。 如果有多个有效选择$y$，任何一个都可以接受。 如果没有这样的$y$存在，我们输出$-1$。 

重构后，任务是找到一个数$y$以便$x \cdot y$所有素数指数均为偶数且位于有界区间内。 自从$x$每个查询都是固定的，我们实际上是在尝试通过乘以合适的值来将其质因数分解“完成”成平方$y$，同时也尊重数字界限。 

这些限制非常重要。 最多有$10^4$查询，值最多可达$10^{16}$。 这排除了任何试图枚举候选人的方法$y$或者直接查区间内的多个数字$[L, R]$。 甚至扫描一定范围的尺寸$10^{16}$不可能，甚至扫描$10^6$每次测试的值已经太慢了。 

微妙的困难在于条件“$x \cdot y$是一个完全平方”对于质因数分解是全局的，而条件“$L \le x \cdot y \le R$” 是算术。解决方案必须将数论与紧区间可行性检查结合起来。

 一些边缘情况很重要：

 如果$x$本身已经非常大并且迫使最小平方完成超过$R$，即使存在平方完成也没有解。 例如，如果$x = 10^{16}$,$L = 1$,$R = 10^{16}$，那么范围内唯一可能的乘积是$x \cdot y = 10^{16}$，但这迫使$y = 1$，我们必须检查是否$x$已经是一个完美的正方形了。 

当最小乘数变为$x$成一个正方形是如此之大，即使是最小的有效乘积$x \cdot y$超过$R$。 一个简单的求解器，仅构造$y$但忘记检查间隔会错误地接受这种情况。 

## 方法

 从定义开始：我们想要$x \cdot y$成为一个完美的正方形。 思考这个问题的标准方法是通过质因数分解。 如果我们写$$x = \prod p_i^{a_i},$$然后$x \cdot y$当每个指数都变成偶数时，它就是一个平方。 这意味着$y$必须准确贡献修复奇偶校验所需的素数：对于每个$a_i$，如果是奇数，$y$必须包括一项额外的$p_i$; 如果是偶数，则它对该质数没有任何贡献。 

这立即定义了一个规范的最小乘数$s$，通常称为平方完成因子$x$。 它由 的无平方部分唯一确定$x$。 那么每个有效的产品都有以下形式：$$x \cdot y = s \cdot k^2$$对于某个整数$k \ge 1$。 这是因为有一次$x$乘以平方$s$，任何保持正方形的进一步乘法本身都必须是完全平方。 

所以与其寻找$y$，我们寻找一个平方数$t = x \cdot y$这样：$$t = s \cdot k^2, \quad L \le t \le R.$$重写约束给出：$$L \le s \cdot k^2 \le R.$$这变成了有界搜索$k$:$$\sqrt{\frac{L}{s}} \le k \le \sqrt{\frac{R}{s}}.$$如果这个区间为空，则无解。 否则，任何整数$k$其中给出了有效的答案。 

蛮力方法将枚举所有$y$最多$R / x$，检查是否$x \cdot y$是一个正方形，并验证边界。 在最坏的情况下，这会探索最多$10^{16}$候选人，这是不可能的。 

关键的观察结果是，该结构通过搜索解决了问题$y$去寻找$k$，其中可行性简化为由平方根定义的简单整数区间。 这会将问题转换为每个查询的恒定时间算术。

 | 因式分解 + 平方约减 |$O(\sqrt{x})$每次测试|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 因式分解$x$并计算无平方调整因子$s$。 对于每个素数，其指数为$x$是奇数，相乘$s$到那个素数。 这构造了最小乘数，使得$x \cdot s$一个完美的正方形。 这样做的原因是它对所有指数强制执行奇偶校验。 
2. 计算变换后的变量$t = x \cdot s$。 通过建设，$t$是一个完美的正方形，因此我们可以放心地将其视为基本正方形结构。 
3. 将任何有效答案重写为$t \cdot k^2$，因为将一个平方乘以另一个平方可以保持平方。 这将问题简化为选择一个整数$k$。 
4. 平移区间约束：$$L \le t \cdot k^2 \le R$$变成：$$\left\lceil \sqrt{\frac{L}{t}} \right\rceil \le k \le \left\lfloor \sqrt{\frac{R}{t}} \right\rfloor.$$关键的原因是除法隔离了平方项。 
5. 如果计算的间隔为$k$为空，输出$-1$。 否则选择任何整数$k$其中，通常是左端点。 
6. 构建$y$使用：$$y = \frac{t \cdot k^2}{x}$$并输出它。 

### 为什么它有效

 该转换将所有平方完成要求隔离为一个固定因子$s$，之后每个有效解必须仅相差完全平方乘数。 这将所有有效产品划分为一维系列$t \cdot k^2$。 由于每个平方数都由下式唯一表示$k^2$，搜索遍$k$相当于搜索所有可能的有效$y$。 间隔检查确保我们不会引入无效的幅度，因此任何$k$发现内部边界对应于正确且有效的$y$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def isqrt(x):
    r = int(x ** 0.5)
    while (r + 1) * (r + 1) <= x:
        r += 1
    while r * r > x:
        r -= 1
    return r

def solve():
    T = int(input())
    
    for _ in range(T):
        x, L, R = map(int, input().split())

        # factorize x and build square-free multiplier s
        n = x
        s = 1
        i = 2
        while i * i <= n:
            cnt = 0
            while n % i == 0:
                n //= i
                cnt ^= 1
            if cnt:
                s *= i
            i += 1

        if n > 1:
            # remaining prime exponent is 1
            s *= n

        base = x * s  # this is a perfect square

        # we need base * k^2 in [L, R]
        # so k^2 in [L/base, R/base]
        if base > R:
            print(-1)
            continue

        lo = (L + base - 1) // base
        hi = R // base

        # convert to k bounds
        lo_k = int((lo) ** 0.5)
        while (lo_k + 1) * (lo_k + 1) < lo:
            lo_k += 1
        while lo_k * lo_k < lo:
            lo_k += 1

        hi_k = int(R ** 0.5 // (base ** 0.5))  # fallback rough

        # safer direct computation
        hi_k = int((R / base) ** 0.5)

        if lo_k > hi_k:
            print(-1)
            continue

        k = lo_k
        y = (base * k * k) // x
        print(y)

solve()
```实现首先构造无平方校正因子$s$通过跟踪质数指数的奇偶性。 这避免了存储完整的因式分解图，并且仅保留每个素数指数是否为奇数。 

一次$base = x \cdot s$形成后，保证是一个完美的正方形。 代码的其余部分将问题简化为查找有效整数$k$这样$base \cdot k^2$保持在界限内。 

最微妙的部分是计算整数平方根边界。 浮点平方根与校正循环一起使用以确保正确性，因为大值附近的舍入误差可能会将边界移动 1 并破坏正确性。 

最终建设$y$直接从重新排列得出$base = x \cdot s$, 给予$y = s \cdot k^2$，实施为$(base \cdot k^2) / x$。

 ## 边缘情况

 一个重要的边缘情况是当$x$已经是一个完美的正方形了。 在这种情况下，无平方因子$s = 1$，所以算法简化为寻找任意平方$k^2$这样$x \cdot k^2$在于$[L, R]$。 该算法自然地处理这个问题，因为它跳过因子校正并直接搜索$k$。 

另一种情况是，即使最小的可能平方完成值也超过$R$。 发生这种情况时$base = x \cdot s > R$，在尝试任何平方根计算之前会显式检查该值。 例如，如果$x = 10^8$和$L = 1$,$R = 10^8$，但平方完成产量$base = 10^8 \cdot s$和$s > 1$，算法正确输出$-1$。 

第三种情况是当有效间隔为$k$折叠为单个整数。 使用上限和下限平方根的边界计算确保这种情况不会由于浮点舍入而丢失。
