 ## 解决方案
 ## 问题理解

 有$n$学生帐户标记$1$到$n$，加上一个特殊的教师帐户，标记为$-1$。 我们还得到了一个序列$t$事件。 每个事件描述一名学生$x$谁打开了另一个帐户的关注列表$y$，然后关注当前所在的所有人$y$的关注列表。 如果$x$已经关注某人，没有任何变化。 如果$x$试图跟随自己，该行动被忽略。 

最初，学生$1$跟随老师$-1$。 所有其他学生都不追随，老师也不追随。 

这个过程纯粹是为了扩大关注关系。 每天都会将一组跟随边从一个节点复制到另一个节点，并限制自循环被忽略并且重复的跟随是无害的。 

目标是在处理所有事件后确定哪些学生跟随老师$-1$。 

限制非常小：$n \le 100$,$t \le 1000$。 这立即允许任何$O(t \cdot n^2)$甚至$O(t \cdot n)$模拟。 除了简单的集合或位集传播之外，不需要高级数据结构或渐近优化。 

一个微妙的边缘情况来自这样一个事实：教师帐户$-1$行为类似于图中的普通节点，但除了作为目标之外，被排除在最终输出域之外。 另一个极端情况是重复复制相同的邻居列表：由于集合是幂等的，因此重复操作不得重复计算或破坏正确性。 

一个最小的令人困惑的情况是复制包含时$-1$本身。 例如，如果将跟随老师的学生作为来源，其他人可能会立即间接继承老师的跟随。 

## 方法

 该过程可以解释为随时间演变的有向图。 每个帐户都维护一组代表其关注者的传出边缘。 每个操作都需要完整的传出邻居集$y$并将其合并到$x$已定。 

暴力模拟自然表明了自己：为每个节点维护一个集合，并为每个操作迭代所有元素$y$的设置并将它们插入$x$已定。 Each insertion is$O(1)$使用哈希集进行平均，因此每个事件的成本$O(n)$在最坏的情况下。 超过$t$这是事件$O(tn)$，这很容易被接受$n \le 100$,$t \le 1000$。 

除此之外，不需要进行优化，因为系统从不需要最短路径、可达性闭包或全局结构的重复重新计算。 每次更新都是一小部分更新的本地传播。 

一个更结构化的视图是认为每个节点都维护一个如下的位掩码。 然后每个操作都是掩码的按位或，并清除自位。 这减少了常数因素并使正确性变得明显。 

关键的观察结果是系统是单调的：一旦添加跟随边缘，它就永远不会被删除。 这可以防止振荡并确保简单传播在每次操作后立即收敛。

 ## 算法演练

 我们维护一个结构`follow[i]`，代表用户关注的账户集合$i$。 老师被索引为$-1$，我们将其映射到索引$0$为了方便同学们$1..n$映射到$1..n$。 

我们也根据语句初始化系统：student$1$跟随老师，所以`follow[1] = {teacher}`所有其他集合都是空的。 

For each event$(x, y)$，我们合并所有元素`follow[y]`进入`follow[x]`，同时小心避免自我跟随。 

处理完所有事件后，我们扫描哪些学生的关注集中有老师。 

### Steps

 1. 初始化集合数组`follow`尺寸的$n+1$，其中索引$i$代表学生$i$，以及作为特殊标识符的教师的单独表示。 这种分离避免了与负索引混淆。 
2. 设置初始条件`follow[1]`遏制老师，既然学生$1$从以下开始$-1$。 
3. 对于每个事件$(x, y)$，迭代每个帐户$v$在`follow[y]`。 
4. 对于每一个这样的$v$， 如果$v \neq x$， 插入$v$进入`follow[x]`。 根据系统规则的要求，自检可确保没有帐户跟随自身。 
5.处理完所有事件后，迭代students$1..n$，并收集这些索引$i$这样老师就在`follow[i]`。 
6. 在输出之前对结果列表进行排序，尽管迭代来自$1$到$n$已经保证排序顺序。 

### 为什么它有效

 在任何时刻，`follow[i]`准确地表示通过结束于以下位置的一系列“复制跟随列表”操作可访问的帐户集$i$，从初始配置开始。 每个操作仅添加另一个集合中已存在的元素，因此不会引入人工元素。 由于使用了集合，因此重复项不会影响状态，并且由于仅添加边，因此不会丢失早期的关系。 这保证了在处理完所有操作之后，成员资格`follow[i]`准确地反映了是否$i$曾经通过任何一连串的抄袭事件继承了老师。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, t = map(int, input().split())
    
    TEACHER = -1
    
    follow = [set() for _ in range(n + 1)]
    
    follow[1].add(TEACHER)
    
    for _ in range(t):
        x, y = map(int, input().split())
        
        # copy all follows from y to x
        for v in follow[y]:
            if v != x:
                follow[x].add(v)
    
    res = []
    for i in range(1, n + 1):
        if TEACHER in follow[i]:
            res.append(i)
    
    print(len(res))
    print(*res)

if __name__ == "__main__":
    main()
```该解决方案直接实现了集合传播模型。 老师用哨兵值来表示$-1$，这是安全的，因为所有学生证都是正面的。 在每次操作期间，我们都会迭代当前的快照`follow[y]`并延伸`follow[x]`。 自检可防止非法自跟随边缘。 

最终扫描只是检查教师在每个学生组中的成员身份。

 ## 边缘情况

 一个重要的边缘情况是教师的唯一来源是间接的。 例如，如果学生$2$从不直接跟随$-1$但后来复制了学生$1$，谁已经关注了$-1$，然后是学生$2$终究还是要跟着老师走。 该算法处理这个问题是因为`follow[1]`已经包含$-1$，复制它会将教师标记向前转移。 

另一种情况是重复的自我复制尝试。 如果一个操作是`(1, -1)`什么时候`follow[-1]`是空的，没有任何变化。 如果重复多次，集合将保持稳定，因为集合结构会忽略重复项，并且自我跟踪会被显式阻止。
