 ## 解决方案
 ## 问题理解

 我们正在建模一个重复的随机过程，其中每个“周期”都由绘图组成，直到我们获得罕见事件，然后重置。 每次抽奖都有概率产生 6 星结果，但这个概率不是固定的。 它以某个基本速率开始，并在当前周期中尝试失败的阈值次数后线性增加，直到达到确定性。 

更具体地说，在您上次获得 6 星后，您再次开始拉动。 新周期中的前几次拉动都具有相同的成功概率（百分之一）。 如果连续 c 次拉动未能获得 6 星，则从下一次拉动开始，每次超出 c 次的失败拉动成功概率都会增加 b%，上限为 100%。 一旦成功，该过程就会重置，您将以基本概率重新开始。 

任务是计算两次连续成功之间的预期拉动次数，即一个此类周期的预期长度。 

约束很小：a、b、c 最多都为 100，测试用例最多为 1000 个。 这立即表明任何涉及恒定大小状态机或最多数百个状态的直接期望 DP 的解决方案都是可行的。 每个测试用例的任何二次或更差的情况原则上仍然是可以接受的，但是任何涉及无限时间的模拟或每个测试用例的许多步骤的迭代收敛的事情都是有风险的。 

主要的边缘情况结构来自极端概率。 当 a 为 100 时，答案就是 1，因为每次拉取都会立即成功。 当 a 非常小而 c 大时，在怜悯斜坡开始之前，该过程可能会在平坦概率阶段花费很长时间。 另一个微妙的情况是，当 b 足够大时，概率在 c 之前或之后不久达到 100，从而有效地截断尾部。 

一个幼稚的错误是直接模拟该过程。 尽管每个周期的预期都很短，但罕见的长尾使得蒙特卡罗不稳定并且在严格的精度保证下收敛缓慢。 

## 方法

 考虑期望的一种强力方法是重复模拟一个周期的过程。 我们将从零失败开始，重复采样伯努利试验，其概率取决于当前的失败次数，并在成功发生时停止。 多次重复此操作并取平均值即可得出期望的估计值。 

这在概念上是正确的，但它并没有给出具有保证精度的确定性答案。 即使我们尝试通过在某个大深度处截断模拟来计算期望，当 a 很小时，尾部概率也是不可忽略的，并且速率递增机制会显着延迟收敛。 核心问题是该过程原则上具有无限长度。 

关键的观察结果是，如果我们跟踪“自上次成功以来有多少次失败”直到概率变为 100%，则系统只有有限数量的不同状态。 一旦成功，一切都会重置，因此我们只需要计算马尔可夫链中吸收的预期命中时间。 

每个状态对应一个失败计数i，其中i的范围从0向上。 对于每个 i，我们知道成功概率 p(i)。 从状态 i 开始，我们要么成功并终止，要么失败并移动到 i + 1。这是标准的期望递归：

 E[i] = 1 + (1 - p(i)) * E[i + 1]

 当 p(last) = 1 时，最终条件是 E[last] = 1，因为成功已成定局。 

状态的数量是有限的，因为一旦概率达到 100%，我们就停止延伸链条。 由于 a、b、c ≤ 100，因此在超过 c 最多大约 100 步后概率变得确定。 因此我们可以安全地将州数限制在几百个。 

该解决方案简化为从最后一个状态向后计算有限线性递归。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(未知，高方差) | O(1) | O(1) | 不可靠|
 | DP 故障状态 | 每次测试 O(c + 100) | O(c + 100) | 已接受 |

 ## 算法演练

 我们根据自上次 6 星以来连续失败的次数对状态进行索引。 

1. 计算每个失败次数 i 的成功概率。 对于 i ≤ c，只有在超过 c 后，概率才是 a + 0·b、a + 1·b，依此类推。 实际上，我们定义 p(i) = min(100, a + max(0, i - c) * b)。 这将怜悯规则转换为状态上的确定性函数。 
2. 确定截止索引 L，其中 p(L) 变为 100。从该状态开始，下一次拉动始终会成功，因此 E[L] = 1。 
3. 初始化数组 E，其中 E[i] 表示从故障状态 i 开始所需的预期拉取次数。 
4. 使用递推式 E[i] = 1 + (1 - p(i)) * E[i + 1] / 100 将数组从 L - 1 向后填充到 0。除以 100 来自将百分比概率转换为真实概率。 
5. 输出E[0]，这是成功后立即开始的预期拉取次数。 

后向DP起作用的原因是每个状态仅依赖于下一个状态。 没有分支到更早的状态，因此一旦知道 E[i + 1]，E[i] 就变成直接计算。 

### 为什么它有效

 该过程是一维吸收马尔可夫链，其中状态 i 仅转换到 i + 1（失败）或吸收（成功）。 每条路径最终要么成功，要么达到概率为 1 的状态，保证终止。 递归精确地编码了以第一步为条件的总期望定律。 由于所有依赖项都在 i 中向前移动，向后计算会产生系统的唯一不动点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        a, b, c = map(int, input().split())

        # build probabilities until they reach 100
        p = []
        i = 0
        while True:
            if i <= c:
                val = a
            else:
                val = a + (i - c) * b

            if val >= 100:
                p.append(1.0)
                break

            p.append(val / 100.0)
            i += 1

        n = len(p)
        E = [0.0] * n

        # terminal state: guaranteed success
        E[n - 1] = 1.0

        for i in range(n - 2, -1, -1):
            E[i] = 1.0 + (1.0 - p[i]) * E[i + 1]

        print(E[0])

if __name__ == "__main__":
    solve()
```该实现构造概率序列，直到达到确定性为止，这充当吸收边界。 这避免了对状态计数的显式上限的推理。 

后向DP是递归的直接翻译。 关键的微妙之处是确保概率标准化为 [0, 1] 浮点数，并且最终状态设置为恰好 1.0 预期步骤，因为它总是立即成功。 

## 工作示例

 ### 示例 1

 输入：

 a = 1、b = 1、c = 1

 我们计算状态：

 | 我| p(i) | p(i) | E[i] |
 | --- | --- | --- |
 | 3 | 1.0 | 1.0 |
 | 2 | 0.03 | 0.03 1 + 0.97 * 1 = 1.97 |
 | 1 | 0.02 | 0.02 1 + 0.98 * 1.97 = 2.9306 |
 | 0 | 0.01 | 0.01 1 + 0.99 * 2.9306 ≈ 3.9013 |

 最终值是从头开始的预期拉动计数。 这表明，在怜悯被激活之前，即使是很小的概率也会因重复的失败而累积很大的期望值。 

### 示例 2

 输入：

 | 我| p(i) | p(i) | E[i] |

 在这里，怜悯系统永远不会激活，因为概率已经很快达到 100%。 预期主要由早期成功机会而非长尾行为主导。

 ## 边缘情况

 当 a 为 100 时，在 i = 0 时立即达到吸收状态。DP 数组包含 p(0) = 1 的单个状态，因此 E[0] 直接设置为 1。这可以防止除以零或不必要的转换。 

当a很小而c很大时，在怜悯激活之前会积累许多状态。 DP 仍然可以正确处理这个问题，因为每个 E[i] 仅取决于 E[i + 1]，并且没有引入近似值。 

当 b 足够大以至于 p(i) 在到达 c 之前跳到 100 时，链会提前终止。 一旦 val ≥ 100，构造循环就会停止，确保我们不会分配不必要的状态并且终止条件是准确的。
