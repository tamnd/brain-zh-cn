 ## 解决方案
 ## 问题理解

 给定两个字符串，我们可以通过第一个字符串的前缀并将其与第二个字符串的后缀连接来构建新字符串。 这里的前缀表示第一个字符串的任何起始段，包括空字符串，后缀表示第二个字符串的任何结束段，也包括空字符串。 唯一的限制是我们不计算完全为空的结果。 

对于每一对选择，第一个字符串中剪切一个前缀，第二个字符串中剪切一个后缀，我们通过将前缀粘贴在后缀前面来形成一个字符串。 任务是计算在所有选择中以这种方式可以生成多少个不同的字符串。 

这些约束意味着每个测试用例字符串可能非常大，并且所有测试的总长度达到一百万。 这立即排除了对所有前缀后缀对的任何二次枚举。 即使每对索引的线性传递也是可以接受的，但任何依赖于长度乘积的东西都是不可接受的。 

一个微妙的问题是不同的切割对可以生成相同的字符串。 例如，如果第一个字符串的所选前缀的末尾与第二个字符串的所选后缀的开头完全重叠，则移动剪切边界不会更改生成的连接字符串。 这是创建重复项的唯一机制：允许“滑动”边界而不更改字符串的重叠。 

一种简单的方法会生成所有对并将结果字符串插入到哈希集中。 这会立即失败，因为最多有 (n + 1)(m + 1) 对，即使在考虑字符串构建成本之前，这也太大了。 

## 方法

 蛮力策略很简单：迭代 s 的每个前缀长度 i 和 t 的每个后缀起始位置 j，构建字符串 s[:i] + t[j:]，并将其插入集合中。 这是正确的，因为它明确地构建了所有可能的结果。 失败点是生成的字符串的数量，在最坏的情况下是二次的，并且每个字符串比较或散列也取决于其长度，使得实际中的总工作量为立方。 

关键的观察结果是，两个不同的对 (i, j) 和 (i', j') 可以生成相同字符串的唯一原因是前缀和后缀之间的边界可以在不更改字符的情况下移动。 当 s 前缀的结尾与 t 后缀的开头匹配时，就会发生这种情况。 如果存在这样的匹配，我们可以将一个字符从前缀侧移动到后缀侧，反之亦然，而不改变最终的串联。 

这意味着每个有效的结构都可以通过扩展 s 所选前缀末尾与 t 所选后缀开头之间的重叠来重复“压缩”。 最终，这个过程达到了一个独特的规范表示，其中不可能进一步扩展。 每个不同的字符串恰好对应于一个这样的规范状态。 

因此，我们不是计算所有对，而是计算存在多少个规范对。 如果我们无法扩展重叠，或者一侧已经耗尽，则一对 (i, j) 是规范的。 通过模拟重叠链可以延伸多远，这给出了一个可以在线性时间内遍历的结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(nm·(n + m)) | O(纳米) | 太慢了 |
 | 规范压缩 | O(n + m) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。

1. 我们考虑通过选择以位置 i 结束的 s 前缀和以位置 j 开始的 t 后缀形成的一对。 我们没有将这对视为最终的，而是尝试通过消除重叠来对其进行标准化。 
2. 虽然 s 中所选前缀的最后一个字符与 t 中所选后缀的第一个字符匹配，但我们可以将边界移动一个位置。 我们从前缀侧删除该字符并将其包含在后缀侧。 这不会改变生成的连接字符串，只会改变它的表示方式。 
3. 我们重复这个收缩和扩展过程，直到前缀变空或后缀变空，或者边界字符不再匹配。 此时，表示无法进一步压缩。 
4. 对于每个不同的构造字符串，结果对都是唯一的，因此我们不是计算所有原始对，而是计算此压缩过程之后存在多少个规范对。 
5. 为了有效地计算答案，我们使用两个指针隐式模拟这个过程。 我们迭代 s 中所有可能的前缀端点 i，并且对于每个 i，我们在 t 中维护最远的有效后缀起始 j，使得该对已经处于规范形式。 当我们移动 i 时，我们通过检查边界匹配以摊余常数时间更新 j。 

关键的想法是，每次存在边界匹配时，它都会消耗一个自由度，并且一旦消耗，它就不能独立地重新出现在其他状态中。 这保证了所有状态之间的转换总数是线性的。 

### 为什么它有效

 压缩过程定义了所有对 (i, j) 的等价关系：如果可以通过重复跨越边界移动匹配字符将一对转换为另一对，则两对是等价的。 每个等价类都有一个唯一的代表，不可能进一步移动。 该算法精确计算每个类别的一名代表。 由于每个变换都严格减少重叠或达到边界，因此循环是不可能的，并且规范形式是明确定义的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        s = input().strip()
        t = input().strip()

        n = len(s)
        m = len(t)

        i = 0
        j = 0

        # We maintain a sliding canonical boundary.
        # (i, j) represents current prefix of s up to i and suffix of t from j.
        # We try to maximize overlap compression.
        ans = 0

        for i in range(n + 1):
            if i == 0:
                j = 0
            else:
                # try to maintain canonical form after increasing prefix
                while j < m and i > 0 and s[i - 1] == t[j]:
                    i -= 1
                    j += 1

            # count this canonical pair if not empty string
            if not (i == 0 and j == m):
                ans += 1

        print(ans)

if __name__ == "__main__":
    solve()
```实现的核心是我们总是尝试立即消除边界匹配的不变量。 当扩展 s 的前缀时，我们检查新的边界字符是否与 t 的当前后缀开头匹配。 如果是这样，我们将边界向内移动，有效地合并重叠部分。 

计数器仅在不完全为空的状态下递增，因为问题排除了空字符串。 循环结构确保每个字符在整个过程中最多跨过边界一次，这保证了线性复杂性。

 ## 边缘情况

 当两个字符串相同并且由重复字符组成时，每个边界位置都是可合并的。 该算法反复移动边界，直到一侧变空，确保所有等效表示崩溃为单个规范状态。 例如，对于 s = "aaa" 和 t = "aaa"，每个中间对都简化为相同的完全合并形式，并且算法仅计算不同的结果而不重复。 

当 s 和 t 的边界之间根本没有匹配字符时，不会触发压缩。 每个前缀和后缀组合都保持不同，并且算法独立地对每个规范对进行计数，而无需激活任何合并步骤。
