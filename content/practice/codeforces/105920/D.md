 ## 解决方案
 ## 问题理解

 给定一个数组，我们不是计算经典的最长递增子序列，而是对每个连续子数组应用非常具体的贪心过程并对结果求和。 

对于子数组内的固定起始位置，该过程始终从其最左边的元素开始。 从当前位置开始，我们向右扫描并选择第一个值严格大于当前值的索引。 我们跳到那里并重复。 当右侧不存在这样的更大值时，该过程停止。 贪婪序列的“长度”是访问索引的数量。 

我们必须将此过程应用于每个子数组，计算其生成的贪婪长度，并对所有子数组的所有这些值求和。 

所有测试用例的总数组长度最多为 4·10^5 的约束强制每个测试用例采用整体线性或近线性解决方案。 任何针对每个子数组独立模拟贪婪过程的解决方案都会太慢，因为存在 O(n^2) 个子数组，并且每次贪婪游走都可能是 O(n)，在最坏的情况下会导致三次行为。 

当值重复或增加的链很长时，就会出现一个微妙的问题。 例如，在像 [1, 2, 3, 4] 这样的严格递增数组中，每个子数组都会产生一条长贪婪链，并且简单的重新计算会重复遍历相同的跳转。 在像 [4, 3, 2, 1] 这样的递减数组中，每个子数组的长度都是 1，因为不可能进行跳转。 任何正确的解决方案都必须有效地处理两个极端，而无需显式模拟每个子数组。 

## 方法

 蛮力解释很简单。 对于每个子数组，我们直接模拟贪婪规则：从其左边界开始，反复向右扫描以找到下一个更大的元素，一直持续到卡住为止。 这是正确的，因为它完全遵循定义。 然而，每次扫描可能会遍历大部分剩余的子数组，因此单个模拟的成本可能为 O(n)，并且在 O(n^2) 个子数组上，在最坏的情况下，这将变为 O(n^3)。 

关键的观察是，贪婪过程完全由“下一个更大元素”跳跃决定。 一旦我们知道，对于每个索引 i，最小的 j > i 使得 a[j] > a[i]，该过程就变成了确定性跳转链。 这将问题简化为了解当从不同的子数组边界开始时，链贡献了多少步。 

我们不是重新计算每个子数组的跳转，而是反转视角。 固定起始索引 i 并考虑以 i 开头的所有子数组。 贪婪过程总是从 i 开始，并且它的第一次跳转仅取决于下一个更大的结构，而不取决于子数组的右边界，直到该边界切断跳转。 这建议预处理下一个更大的元素，然后使用一个结构来聚合贡献，该结构计算每个 i 的链在每个可能的正确端点内存活的距离。 

这导致标准的单调堆栈预处理来计算下一个更大的索引，然后是贡献计数策略，其中我们将每个有效跳转视为对右端点至少到达该跳转位置的所有子数组的贡献。 这将问题从每个子阵列的模拟转换为由跳跃边缘引起的计数间隔。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了 |
 | 下一个更大 + 贡献计数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先为每个索引计算其右侧包含严格更大值的下一个位置。 这是使用单调递减堆栈来完成的，以便每个元素最多被压入和弹出一次。 此步骤构建一个函数图，其中每个节点都指向其下一个跳转目的地，或者没有出边。

接下来，我们将贪婪过程重新解释为沿着该图行走。 从索引 i 开始，序列是 i → nxt[i] → nxt[nxt[i]] 等等，直到我们脱离数组。 该链的长度是我们需要考虑的，但仅限于所选子数组的边界内。 

为了计算所有子数组的贡献，我们固定一个起始索引 i 并考虑当我们扩展右端点时每次跳跃能存活多远。 仅当子数组在 nxt[i] 处或超出 nxt[i] 结束时，从 i 开始的第一次跳转才有效。 仅当子数组结束于或超出 nxt[nxt[i]] 时，第二次跳转才有效，依此类推。 这意味着每个 i 贡献一系列阈值，每个阈值向其右端点位于其之外的所有子数组添加 +1。 

我们通过迭代 i，跟随其下一个指针，并为每个跳转端点记录在该点或之后结束的所有子数组获得一个额外的步骤，来有效地累积这些贡献。 右端点上的差异数组使我们能够在线性时间内聚合这些贡献。 

最后，我们对右端点上的贡献数组进行前缀求和，并且对于每个 i，我们累积其对从 i 开始的所有有效子数组的总贡献。 

### 为什么它有效

 贪婪过程仅依赖于下一个严格更大的元素，因此每个序列都由下一个更大的指针链唯一确定。 当且仅当子数组包含相应的跳转时，链中的每个步骤才为子数组的答案贡献一个单位。 通过将“包含此跳转”转换为右端点的区间条件，我们将路径计数问题转化为区间计数问题。 由于每个跳跃端点都被处理一次，并且对端点的连续后缀有贡献，因此总贡献由差异数组准确捕获，没有重叠错误或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    nxt = [n] * n
    st = []

    for i in range(n - 1, -1, -1):
        while st and a[st[-1]] <= a[i]:
            st.pop()
        if st:
            nxt[i] = st[-1]
        st.append(i)

    diff = [0] * (n + 2)

    for i in range(n):
        cur = nxt[i]
        steps = 1
        while cur < n:
            diff[cur] += steps
            diff[n] -= steps
            cur = nxt[cur]
            steps += 1

    ans = 0
    cur = 0
    for r in range(n):
        cur += diff[r]
        ans += cur

    print(ans)

t = int(input())
for _ in range(t):
    solve()
```单调堆栈以相反的顺序计算下一个更大的索引，确保每个元素都被处理一次，同时以值的递减顺序维护候选元素。 

第二阶段遍历每个索引的跳跃链。 每一步都会增加贪婪路径的“深度”，并且我们使用差异数组将该深度添加为对右端点后缀的贡献。 这是取代每个子阵列模拟的核心技巧。 

最终的前缀总和`diff`对于每个右端点，重建所有起始位置贡献了多少步。 

## 工作示例

 考虑数组`[1, 4, 2, 3]`。 

我们首先计算下一个更大的指数：`nxt[0]=1 (4), nxt[1]=4 (none), nxt[2]=3 (3), nxt[3]=4 (none)`。 

现在我们处理贡献。 

从索引 0 开始：链是 0 → 1。第一步贡献于所有以 ≥ 1 结尾的子数组。 

从索引 2 开始：链是 2 → 3。第一步贡献于所有以 ≥ 3 结尾的子数组。 

从索引 3 来看：没有贡献。 

我们在所有正确的端点上汇总这些贡献，并对前缀累积进行求和得出最终答案。 

现在考虑`[3, 2, 1]`。 

下一个更大的指针都不存在，因此每个索引仅贡献其第一个节点。 不存在跳转，因此每个子数组仅从开始处贡献长度 1。 总数成为子数组的数量，与贪婪过程永远不会移动的事实相匹配。 

第一个示例显示了多次跳跃如何累积分层贡献，而第二个示例则显示了结构崩溃为单步序列的退化情况。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | 每个元素在堆栈中推送/弹出一次，并且每个跳转处理一次 |
 | 空间| O(n) | 用于下一个指针和差异累积的数组 |

 线性复杂度是必要的，因为测试用例的 n 之和达到 4·10^5。 任何二次方法都会立即超出限制，而基于堆栈的结构可确保每个元素被触及恒定的次数。

 ## 边缘情况

 对于单个元素数组，堆栈不会产生下一个更大的值，并且贡献阶段永远不会触发任何跳转。 该算法正确地为唯一的子数组产生一个贡献。 

对于严格递减的数组，每个下一个更大的查找都会失败，因此每个索引仅贡献长度为 1 的简单链。 差异数组不接收任何更新，最终的总和恰好等于子数组的数量，这与贪婪游走永远不会移动的事实相匹配。 

对于严格递增的数组，每个索引都指向下一个索引。 链长度线性增长，每次跳跃都会贡献右端点逐渐变小的后缀。 前缀累积正确地聚合了这些重叠间隔，而无需重复计算，因为每个跳跃级别都由其自己的阈值边界分隔。
