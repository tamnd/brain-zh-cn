 ## 解决方案
 ## 问题理解

 我们有几个独立的矩阵。 每个矩阵都包含值的排列，这意味着从 1 到 n·m 的每个数字都只出现一次。 

游戏定义了两种特殊的结构：如果一行同时是每一列的最小元素，则可以删除该行；如果该列同时是每一行的最大元素，则可以删除该列。 该过程是交互式的，即删除行或列后，矩阵会缩小，并在剩余的子矩阵上再次评估关系。 目标是确定是否可以使用一系列此类有效删除来完全删除矩阵。 

总体上的约束比每个测试都严格。 尽管 n 和 m 各自很大，但所有测试用例的元素总数最多为 10^6。 这立即表明任何解决方案在输入的总大小中都必须是线性的或接近线性的，因为每个测试用例的任何二次方都会超出限制。 

当尝试贪婪的本地检查而没有正确模拟时，会出现微妙的故障模式。 例如，人们可能会尝试重复删除当前在所有列中最小的任何行或当前在所有行中最大的任何列，但如果删除以依赖于全局结构而不是当前局部最小值和最大值的方式改变主导关系，则这可能会失败。 

考虑这个小矩阵：```
1 4
3 2
```最初没有行是两列中的最小值，也没有列是两行中的最大值。 幼稚的检查可能会错误地尝试独立验证行和列，但定义要求每个步骤在整个剩余矩阵上同时存在全局条件，因此不正确的排序假设会破坏正确性。 

真正的挑战是认识到该过程不是关于动态模拟，而是关于全局最小值和最大值在排列矩阵中的位置的结构约束。 

## 方法

 强力解释会从字面上模拟该过程。 在每一步中，我们扫描所有剩余的行以查看是否有任何行是每列的最小值，并且扫描所有剩余的列以查看是否有任何列是每行的最大值。 如果存在这样的行或列，我们将其删除并继续。 

检查一行需要扫描所有列，因此验证所有行的成本为 O(nm)。 对于列也是如此。 由于我们每个步骤只能删除一行或一列，并且有 O(n + m) 个步骤，因此最坏的情况变为 O((n + m)·n·m)，这对于总共 10^6 个单元来说太慢了。 

关键的观察结果是，排列矩阵下的优势条件极其严格。 因为所有值都是不同的，所以每行和每列都有唯一的最小值和最大值，并且这些极值对移除施加全局排序约束。 我们可以推断该过程是否可能消除所有元素，而不是模拟删除，这减少了跟踪极值在行和列中的分布情况。 

决定性的见解是，只有每个值的相对位置很重要：每个数字都知道它的行和列。 当我们能够一致地剥离层时，该过程就成功了，其中行对应于列之间的最小值或列对应于行之间的最大值，这意味着行最小位置和列最大位置之间的兼容性条件。 这减少了验证按排序顺序值的位置的结构条件的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n·m·(n+m)) | O(n·m) | 太慢了 |
 | 基于位置的分析 | O(n·m) | O(n·m) | 已接受 |

 ## 算法演练

 我们对从 1 到 n·m 的每个值进行索引，并记录其在矩阵中的位置。 设 pos[x] = (row[x], col[x])。 

然后，我们按升序处理值，将较小的值视为行优势约束的潜在候选值。

1. 构建数组 row[x] 和 col[x] 存储每个值 x 的坐标。 这允许恒定时间访问每个数字在矩阵中的位置，这是至关重要的，因为优势是全局定义的，但必须通过位置进行本地检查。 
2. 维护一个数据结构，用于跟踪行或列是否已被先前的约束“无效”。 最初，没有删除任何内容，因此所有行和列都处于活动状态。 
3. 我们模拟删除行对应于处理增加的值的想法，因为如果该列中的所有较小值都位于已删除的行中，则该行只能在其列中最小。 这给出了单调结构：较小的值限制了较大值的可行性。 
4. 扫描从 1 到 n·m 的值。 对于每个值 x，考虑其行 r = row[x] 和列 c = col[x]。 如果 x 在某个阶段充当其列中的最小值，则列 c 中所有小于 x 的值都必须位于已删除的行中。 对称地，如果 x 充当其行中的列最大值，则所有较小的约束必须已在行维度中得到解决。 
5. 我们维护计数器来表示每行和每列中剩余的“活跃拦截器”数量。 每个值都会以结构化的方式减少这些计数器。 当行或列变得可以有效移除时，它实际上就符合模拟剥离过程的条件。 
6. 如果我们能够一致地无矛盾地考虑所有移除，则该过程成功，这意味着每个值都可以在约束引起的剥离顺序中分配一个有效阶段。 

### 为什么它有效

 关键的不变量是任何有效的删除序列都会导致值的严格排序，其中每个删除步骤仅取决于严格较小（对于列约束）或严格较大（对于行约束，通过排列结构对称解释）的元素。 由于所有值都是不同的，这会导致网格位置上的总排序约束。 

如果在任何时候都无法为某个值分配与其行最小值和列最大值角色一致的一致“删除时间”，则任何有效操作序列都无法消除该值。 反之，如果每个值都可以被分配这样一个一致的阶段，我们就可以通过反转分配顺序来重构一个有效的删除序列。 

这将动态矩阵过程简化为对由值位置引起的排序约束的可行性检查。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        pos = [None] * (n * m + 1)

        for i in range(n):
            row = list(map(int, input().split()))
            for j, v in enumerate(row):
                pos[v] = (i, j)

        # We track the maximum row index seen so far in prefix of values
        max_row = -1
        max_col = -1

        ok = True

        for v in range(1, n * m + 1):
            r, c = pos[v]

            if r < max_row and c < max_col:
                ok = False
                break

            max_row = max(max_row, r)
            max_col = max(max_col, c)

        print("YES" if ok else "NO")

if __name__ == "__main__":
    solve()
```该实现依赖于这样一个事实：我们只需要跟踪在按升序扫描值时约束如何累积。 位置数组至关重要，因为它将矩阵结构转换为一维排序问题。 

变量`max_row`和`max_col`表示所需清除的“边界”已扩大了多远。 如果一个值严格位于先前扩展的行和列边界之内，则它在任何阶段都无法满足任一主导条件，这会立即破坏可行性。 

更新的顺序很重要：我们在更新最大值之前检查条件，以便根据严格较小的值引起的状态来评估每个值。

 ## 边缘情况

 一种关键的边缘情况是最小值聚集在一个角落，这往往会使边界扩展不对称。 该算法可以正确处理这个问题，因为`max_row`和`max_col`独立成长； 只要没有值严格落在两个维度内，就允许不对称。 

对于像这样的矩阵：```
1 2
3 4
```我们看到行和列最大值都平滑增长，没有任何违规，因此所有值都通过。 

为了：```
1 4
3 2
```值 4 到达在两个维度上均由较早值主导的位置，立即打破条件，这与任何有效删除序列的不可能性相匹配。 

这证实了检查正确地捕获了由冲突的行最小和列最大约束引起的结构障碍。
