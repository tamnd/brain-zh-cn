 ## 解决方案
 ## 问题理解

 我们得到一个数组`f`尺寸的`n`。 该数组据称是不相交集并集结构的最终状态，该结构从`f[i] = i`对于每个元素，然后最多有`n`呼叫`merge(u, v)`手术。 

DSU 实现稍微不对称。 每次合并都会找到以下代表`u`和`v`，然后设置根`u`被设置为指向根`v`。 没有大小或等级的联合，也没有平衡逻辑。 唯一的压缩发生在`find`，这不会影响存储的父数组`f`。 因此，我们给出的数组实际上是由根的重复定向附件创建的父指针森林。 

任务是决定是否存在某个最多的序列`n`可以准确生成此父数组的合并。 如果是，我们还必须输出一个有效的合并操作序列； 否则，我们输出`-1`。 

限制条件很大：总计`n`跨测试最多可达`4 · 10^5`。 这迫使我们采用近线性或线性时间的结构。 任何试图模拟任意序列或搜索可能性的方法都会太慢。 

一个微妙的问题是`f`不保证是处于有效 DSU 状态的树。 例如，像这样的循环`0 → 1 → 0`这是不可能的，因为 DSU 家长总是形成一个有根的树木森林。 另一个问题是多个节点可能指向同一个父节点，但根必须定义良好且非循环。 

无效案例的最小示例是`n = 3, f = [1, 2, 0]`。 这形成了一个循环，并且不能来自任何合并序列，因为 DSU 父指针总是严格减少朝向根的深度。 

更微妙的无效情况是当两个节点具有父指针时，这意味着无法从单个根结构中导出冲突的合并方向。 例如，如果一个节点指向一个本身与任何合并顺序不一致的非根节点，它就会破坏构造顺序约束。 

关键的挑战是重建是否存在可以创建这个确切森林的定向联合操作的顺序。 

## 方法

 蛮力想法会尝试向后或向前模拟 DSU 构造。 正向模拟意味着尝试所有可能的合并序列并检查我们是否可以达到`f`。 即使我们最多限制自己`n`操作中，可能的合并序列的数量是天文数字，相当于重复选择对的顺序，这是指数级的`n`。 即使验证单个序列也是线性的，因此这立即是不可行的。 

当我们重新解释合并时，问题的结构变得更加清晰。 每个合并操作都将一个根附加到另一个根，这意味着它在两个组件之间创建了一条有向边，并逐渐构建一个森林，最终每个连接的组件只有一个根。 

所以最终的数组`f`编码一个有向图，其中每个节点都指向其父节点，根是其中的节点`f[i] = i`。 有效的 DSU 构造必须生成一个森林，其中每个节点最终都通向一个根，并且每个非根在其集合合并到另一个集合时都恰好选择一个父节点。 

关键的观察是我们可以自由选择合并的顺序。 这意味着如果我们按照从叶子向上的仔细顺序处理节点，如果结构有效，我们总是可以模拟构造。 除根之外的每个节点都必须恰好“附加”一次，因此我们希望确保在两个端点都是其当前组件的根时，可以实现每个有向边。 

这将问题转变为验证必须是森林的功能图，然后生成从孩子到父母的类似拓扑的附件排序。 

一个关键的简化是将最终结构视为有根森林，定义为`f`。 我们首先验证它是否有效：每个节点最终必须到达一个固定点，其中`f[i] = i`，并且除了自循环之外不能有循环。 然后我们可以通过按从叶到根的逆拓扑顺序处理节点来重建合并。 

一旦我们将每个节点的父节点视为它最终必须附加到的组件，我们就可以处理其子节点已经解析的节点，确保我们始终将较小的组件合并到已经构造的组件中。

 ## 算法演练

 我们通过将最终的父数组视为有向森林并自下而上构建它来重建 DSU 操作。 

1. 首先，我们通过检查所有索引来确定哪些节点是根`i`这样`f[i] = i`。 这些节点代表组件的最终代表。 如果一个节点错误地指向自己或者参与了一个循环，我们稍后会检测到它。 
2. 我们构建一个反向邻接表，其中`children[x]`包含所有节点`i`这样`f[i] = x`。 这会将父指针表示形式转换为显式树结构。 
3.我们计算孩子的数量（逆图中的入度）。 所有具有零个子节点的节点都是森林中的叶子，这意味着它们可以首先被安全地处理。 
4. 我们初始化一个包含所有叶子的队列。 这些节点不需要等待在它们下面构建任何子树。 这个想法是叶子可以提前附加，因为没有什么取决于它们稍后合并。 
5. 当队列不为空时，我们删除一个节点`v`并考虑其父级`p = f[v]`。 如果`p == v`，它是一个根，除了将其保留为最终组件锚点之外，我们什么也不做。 
6. 如果`p != v`，我们记录之间的合并操作`v`和`p`。 此时此刻，`v`表示一个完全构建的子树，并且`p`代表它所属的组件，因此合并它们与 DSU 行为一致。 
7、后处理`v`，我们减少剩余的子项数量`p`。 如果`p`现在没有剩余的未解决的子级，它有资格被处理，因此我们将其推入队列。 
8. 过程完成后，我们检查是否能够一致地处理所有节点。 如果存在循环，某些节点将永远不会成为叶子，从而使它们未被处理，这表明这是不可能的。 

### 为什么它有效

 该构造依赖于以下不变量：只有在处理完节点子树中的所有节点后，我们才会处理该节点。 这确保了当我们应用合并操作时`(v, f[v])`，两个端点都对应于已经形成的 DSU 组件，因此合并可以解释为将一个完整的组件附加到另一个组件，而不违反任何早期的结构。 如果存在循环，则该循环中的任何节点都不能成为叶子，因此算法正确地失败。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n = int(input())
    f = list(map(int, input().split()))
    
    children = [[] for _ in range(n)]
    deg = [0] * n
    
    for i in range(n):
        if f[i] == i:
            continue
        children[f[i]].append(i)
        deg[f[i]] += 1
    
    q = deque(i for i in range(n) if deg[i] == 0)
    ans = []
    
    while q:
        v = q.popleft()
        p = f[v]
        
        if v != p:
            ans.append((v, p))
            deg[p] -= 1
            if deg[p] == 0:
                q.append(p)
    
    # check if all nodes were processed structurally
    # nodes in cycles or invalid structure won't be fully resolved
    if len(ans) > n:
        print(-1)
        return
    
    # additional validation: ensure forest consistency
    # (if cycles exist, some nodes never got processed)
    visited = set()
    for v, p in ans:
        visited.add(v)
        visited.add(p)
    
    if len(visited) != n:
        print(-1)
        return
    
    print(len(ans))
    for u, v in ans:
        print(u, v)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该实现首先构建父数组的反向结构，计算每个节点有多少个直接子节点。 具有零个子节点的节点最初被排队，因为它们代表可以在反向构造过程中首先安全“分离”的叶子。 

每次弹出一个节点时，如果它还不是根，我们会将其连接到其父节点。 这对应于颠倒概念性 DSU 构建顺序，其中叶首先附着，根保留到最后。 

基于队列的缩减确保我们仅在处理完节点的所有后代后才处理节点，这对于避免过早合并（可能会违反最终 DSU 的隐含结构）至关重要。 

最终验证检查是否考虑了所有节点，因为任何循环都会阻止某些节点达到零个未解析的子节点。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import deque
    
    t = int(sys.stdin.readline())
    out = []
    
    def solve():
        n = int(sys.stdin.readline())
        f = list(map(int, sys.stdin.readline().split()))
        
        children = [[] for _ in range(n)]
        deg = [0] * n
        
        for i in range(n):
            if f[i] != i:
                children[f[i]].append(i)
                deg[f[i]] += 1
        
        q = deque(i for i in range(n) if deg[i] == 0)
        ans = []
        
        while q:
            v = q.popleft()
            p = f[v]
            if v != p:
                ans.append((v, p))
                deg[p] -= 1
                if deg[p] == 0:
                    q.append(p)
        
        visited = set()
        for u, v in ans:
            visited.add(u); visited.add(v)
        
        if len(visited) != n:
            out.append("-1")
        else:
            out.append(str(len(ans)))
            out.extend(f"{u} {v}" for u, v in ans)
    
    for _ in range(t):
        solve()
    
    return "\n".join(out)

assert run("1\n3\n0 1 0\n") == "-1"

 ## 边缘情况

 一个关键的边缘情况是与循环混合的自循环重配置。 例如，`f = [1, 2, 0]`创建一个长度为三的循环。 在该算法中，每个节点至少有一个入边，因此没有节点达到零度。 队列保持空状态，不产生任何操作，最终访问集没有覆盖所有节点，导致`-1`。 

另一种情况是具有多个独立根的有效森林，例如`f = [0, 0, 2, 2]`。 这里，节点1和3是叶子，因此首先处理它们，生成合并`(1,0)`和`(3,2)`。 然后所有节点都一致解析，产生有效的序列。 

微妙的情况是长链状`f = [1,2,3,4,4]`。 叶子从底部向上处理，确保每次合并将完全形成的子树附加到其父树中，最终到达根。 这表明逆向处理顺序与 DSU 构造约束完全一致。
