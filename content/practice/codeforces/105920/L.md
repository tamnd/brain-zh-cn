 ## 解决方案
 ## 问题理解

 我们在从 1 到 100000 的数轴上给出了几个隐藏位置。这些位置代表萤火虫群的位置。 我们不知道位置，并且多个组可能存在于不同的点。 

唯一允许的操作是选择坐标 x 并接收等于从 x 到所有隐藏位置的总距离的响应。 每个响应都是 x 与每个隐藏点之间的绝对差之和。 目标是每个测试用例使用最多 40 个此类查询来识别至少一个隐藏位置。 

重要的结构细节是响应函数的行为类似于整数上的凸函数。 当我们沿着线移动 x 时，总距离会减小，直到到达点分布的“中心”，然后增加。 该中心对应于隐藏位置多重集的中值。 任何中间位置都保证是该问题中的实际坐标之一，因为所有点都是离散线上的整数。 

这些约束意味着每个测试用例 n 可以大到 10000，并且最多可以有 1000 个测试用例，但 n 的总和是有界的。 坐标范围固定为 100000。这意味着我们无法承受任何尝试显式重建所有点或查询每个位置的方法。 相反，每个测试都必须使用绝对距离函数的结构，通过坐标空间中的对数查询来解决。 

一个天真的误解是假设我们可以直接“反转”单个查询。 一个查询仅给出全局聚合，并不揭示局部结构。 如果假设最小响应点在所有情况下都是唯一的，则会出现另一种微妙的失败情况。 当所有点都相同时，每个查询都会返回与答案相同的坐标，并且二分搜索仍然必须正确收敛，而不依赖于斜率噪声。 

## 方法

 暴力思维会尝试探测从 1 到 100000 的每个坐标 x 并计算响应。 这将清楚地识别函数的最小值，并且该最小值对应于中值位置。 然而，这需要每个测试用例 100000 个查询，这远远超出了 40 个查询的限制，因此即使在考虑多个测试用例之前它也会立即失败。 

关键的观察结果是，定义为绝对距离之和的函数 f(x) 具有可预测的离散导数。 如果我们观察 f 从 x 移动到 x+1 时如何变化，如果每个隐藏点位于 x 或其左侧，则每个隐藏点贡献 +1，如果位于 x 右侧，则贡献 -1。 这将差值 f(x+1) − f(x) 转换为直接编码 x 两侧有多少个点的量。 

这意味着我们可以仅使用函数的两次评估来恢复最多 x 的点的前缀计数。 一旦我们可以评估有多少个隐藏点 ≤ x，我们就可以二分搜索最小的 x，使得该计数至少达到 (n+1)/2，即中位位置。 该坐标必须与隐藏的萤火虫位置之一一致。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力扫描所有 x | O(100000) 次查询 | O(1) | O(1) | 太慢了 |
 | 使用斜率恢复进行二分搜索 | O(log 100000) 查询 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们依赖这样一个事实：我们可以使用距离函数的差异来计算前缀计数。

1. 我们定义一个函数 f(x) 来查询交互器并返回到所有隐藏点的距离总和。 我们将其视为黑盒评估。 
2. 对于任何 x，我们计算 f(x) 和 f(x+1)。 根据这两个值，我们使用恒等式 count_le(x) = (f(x+1) − f(x) + n) / 2 计算小于或等于 x 的隐藏点的数量。这是可行的，因为每个点根据其位于 x 的哪一侧对差异贡献 +1 或 -1。 
3. 我们在坐标范围 [1, 100000] 上进行二分搜索以找到最小的 x，使得 count_le(x) 至少为 (n+1)//2。 这确保我们达到中值阈值。 
4. 在二分查找过程中，每个中点需要两次查询，一次在 mid ，一次在 mid+1 ，因此我们仔细确保不超过 40 次查询的限制。 
5. 一旦二分查找收敛，我们就输出该坐标作为答案。 

这样做的原因是函数 f(x) 是凸函数，并且当 x 通过隐藏点的排序列表时，其离散导数从负单调变为正。 从差异中提取的前缀计数是单调的，这使得二分查找有效。 中值阈值保证我们落在一个实际上被至少一个萤火虫群占据的点上，而不仅仅是它们之间的几何中心。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())

        def query(x):
            print("?", x)
            sys.stdout.flush()
            return int(input())

        def pref_le(x):
            fx = query(x)
            fx1 = query(x + 1)
            return (fx1 - fx + n) // 2

        lo, hi = 1, 100000
        target = (n + 1) // 2

        while lo < hi:
            mid = (lo + hi) // 2
            if pref_le(mid) >= target:
                hi = mid
            else:
                lo = mid + 1

        print("!", lo)
        sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该解决方案是围绕将全局距离查询转换为局部结构信号而构建的。 函数 pref_le 仅使用两次交互调用即可重建某个位置左侧有多少个隐藏点。 这是将问题转变为针对单调谓词的标准二分搜索的关键转换。 

二分搜索是在坐标域而不是索引上进行的，因为我们无法访问已排序的位置列表。 每个决策步骤都是有效的，因为前缀计数在 x 中是单调的，因此一旦我们超过中值阈值，我们就永远不会回到它以下。 

查询功能被隔离，以确保每次交互后立即进行刷新，这是避免交互问题中的空闲错误所必需的。 

## 工作示例

 由于这是交互式的，我们使用固定的隐藏数组来模拟这个想法。 

假设隐藏位置为 [1, 5, 10, 8, 9, 6, 13]，因此 n = 7，中位数为位置 8。 

我们展示二分搜索在概念上是如何进行的。 

| 步骤| 瞧| 你好| 中| pref_le(中) | 决定|

 | 1 | 1 | 100000 | 50000 | 小 (7) | 向左走|

该轨迹表明我们永远不需要显式地定位所有点。 我们只需要足够的分辨率来识别中值阈值交叉。 

第二种情况是所有点都相等，例如 [42, 42, 42]。 每个查询都会返回一个以 42 为中心的对称线性函数。前缀计数在 x = 42 处从 0 跳到 7，因此二分搜索准确地收敛到 42，不会产生歧义。

def make_solver(hidden):
    def run(inp: str) -> str:
        data = inp.strip().split()
        t = int(data[0])

        out_lines = []

 ## 边缘情况

 当所有隐藏点相同时，函数 f(x) 变成以该坐标为中心的简单 V 形函数。 前缀重建仍然有效，因为差值 f(x+1) − f(x) 恰好在该点从负跳到正。 因此二分搜索收敛而没有振荡。 

当 n 为偶数时，有两个经典意义上的中位数，但两者都对应于该问题设置中隐藏点之间的有效坐标。 该算法通过 (n+1)//2 瞄准较低的中位数，确保确定性收敛到有效答案。 

当所有点都位于边界 1 或 100000 附近时，二分搜索仍然可以正确运行，因为查询保持在边界内，并且即使在边缘处也保留了前缀函数的单调性。
