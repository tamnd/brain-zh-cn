 ## 解决方案
 ## 问题理解

 我们有一个类似于围棋游戏的矩形棋盘。 每个单元格要么是空的，要么包含属于两种颜色之一的石头。 正交接触的石头形成相连的组。 如果一个组至少有一个相邻的空单元格（充当自由单元），则该组被认为是稳定的。 如果一个团体没有任何自由，整个团体就会被从董事会中删除。 

任务是模拟这个规则并确定去除所有不稳定组后的最终状态，或者等效地计算在这个规则下捕获了多少石头。 群体之间的相互作用很重要，因为删除一个群体可以为另一个群体创造新的自由，但在这个问题结构中，我们只需要评估一次初始配置并删除所有自由为零的群体。 

输入描述了板尺寸，后跟网格本身。 每个字符编码一个空单元格或一种石头颜色。 输出通常是移除的石子的计数或最终稳定的配置，具体取决于变体，但核心计算是识别 4 方向邻接下的连接组件并检查每个组件是否接触空单元。 

这些约束意味着图最多包含 n 乘 m 个节点。 由于典型限制约为 2×10^5 个单元，因此需要 O(nm) 遍历。 任何尝试对每个组进行重复扫描或从头开始重新计算每个单元的连接性的解决方案都会退化为 O(n^2 m^2) 行为并立即失败。 

当小组触及棋盘边界时，就会出现一个微妙的问题。 处于边缘的群体不会自动拥有自由，除非问题明确地将外部视为空虚。 在大多数类似围棋的公式中，只有边界内的空单元才被计数，因此被石头和边界包围的组即使接触到边缘也是死的。 另一种极端情况是一颗棋子完全被敌方棋子包围，它形成了自由度为零的大小为 1 的组件，必须将其移除。 

## 方法

 解决该问题的直接方法是将每个石头视为起点，运行洪水填充以发现其连接的组，然后扫描其邻居以检查是否存在任何相邻的空单元格。 如果没有，我们将该组中的所有单元格标记为已删除。 对每个石头重复此操作而不缓存访问的节点会导致重复遍历同一组件。 在最坏的情况下，棋盘上充满了石头，每个单元格都会触发几乎整个网格的遍历，从而导致 O((nm)^2) 操作。 

关键的观察结果是，每个单元格恰好属于一个连接的组件，并且自由条件仅取决于该组件中的任何单元格是否接触空单元格。 这允许我们在网格图上使用标准 BFS 或 DFS 一次计算所有组件。 在探索一个组件时，我们同时记录它是否有至少一个相邻的空单元格。 遍历结束后，我们决定是移除还是保留整个组件。 

这将问题从重复的每单元探索减少到网格上的单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个单元的暴力洪水 | O((纳米)^2) | O(纳米) | 太慢了 |
 | 每个组件单个 BFS/DFS | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们将网格建模为一个图，其中每个单元格都是一个节点，并且边缘存在于正交相邻的相同颜色的石头之间。

1. 我们迭代网格中的每个单元格。 当我们遇到一块还没有被访问过的石头时，我们从它开始一个 BFS 或 DFS 来发现它的整个连接组件。 这确保了每个组件只被处理一次。 
2. 在遍历一个组件的过程中，我们维护一个标志来指示该组件是否至少具有一个自由度。 对于访问的每个石室，我们都会检查其四个邻居。 如果任何邻居是空单元格，我们将自由标志设置为 true。 这是可行的，因为自由是在邻接上本地定义的。 
3. 我们还将属于当前组件的所有单元格收集到临时列表中。 这是必要的，因为在探索完整个组件之前，我们无法立即决定是否移除石头。 
4. 遍历完一个组件后，我们检查自由标志。 如果它仍然为假，我们将根据输出要求将所有收集的单元标记为已删除或将它们计数为已捕获。 
5. 我们继续这个过程，直到网格中的所有单元格都被访问过一次。 

我们将决定推迟到完全遍历之后的原因是，在搜索后期可能会发现自由，而部分探索无法安全地确定组件的命运。 

### 为什么它有效

 正确性依赖于 BFS 或 DFS 在 4 向邻接下精确探索一个连通分量这一不变量。 由于自由度仅取决于与空单元的邻接，而不取决于全局结构，因此自由度属性在整个组件中是恒定的。 如果组件中的任何单元格有相邻的空单元格，则整个组件处于活动状态； 否则，所有细胞都会死亡。 因此，对每个组件的属性进行一次评估就足够且一致了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]
    
    visited = [[False] * m for _ in range(n)]
    
    dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]
    
    def inside(x, y):
        return 0 <= x < n and 0 <= y < m
    
    def bfs(i, j):
        stack = [(i, j)]
        visited[i][j] = True
        comp = [(i, j)]
        has_liberty = False
        
        color = grid[i][j]
        
        while stack:
            x, y = stack.pop()
            
            for dx, dy in dirs:
                nx, ny = x + dx, y + dy
                
                if not inside(nx, ny):
                    continue
                
                if grid[nx][ny] == '.':
                    has_liberty = True
                elif grid[nx][ny] == color and not visited[nx][ny]:
                    visited[nx][ny] = True
                    stack.append((nx, ny))
                    comp.append((nx, ny))
        
        if not has_liberty:
            for x, y in comp:
                grid[x][y] = '.'
            return len(comp)
        return 0
    
    captured = 0
    
    for i in range(n):
        for j in range(m):
            if grid[i][j] != '.' and not visited[i][j]:
                captured += bfs(i, j)
    
    print(captured)

if __name__ == "__main__":
    solve()
```该解决方案使用显式的基于堆栈的 DFS 来避免递归限制。 每个组件都被发现一次，我们同时跟踪成员身份和自由状态。 仅在确认某个组件失效后才就地修改网格，这可以防止干扰遍历。 

一个常见的陷阱是在将访问过的单元推入堆栈时忘记立即标记它们，这可能会导致指数级的重新访问。 另一个微妙的问题是将出界视为自由。 在这里，我们明确忽略越界，因为只有空的网格内单元格才被计数。

 ## 边缘情况

 一个重要的边缘情况是组件接触边界但没有内部空单元格。 例如，除非边界内有相邻的空单元格，否则不应将棋盘边缘的一排棋子视为活动的。 BFS 显式仅检查边界内的“.” 细胞，因此仅边界接触并不能创造自由。 

另一个边缘情况是棋盘图案，其中每块石头都是孤立的。 每块石头都是尺寸一的组成部分。 该算法独立地正确处理每个，如果被包围，则将所有标记为已捕获。 

最后一个微妙的情况是具有单个自由单元的大型密集组件。 在遍历过程中，一旦看到第一个空邻居，自由标志就变为 true，并且对于组件的其余部分保持 true，从而确保正确的生存，而无需显式计算所有自由。
