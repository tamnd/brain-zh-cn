 ## 解决方案
 ## 问题理解

 我们得到了一组垂直的棍子，每根棍子都装有一堆盘子。 每个板都有一种颜色和尺寸，每种颜色恰好有七个板，尺寸从 0 到 6。初始配置将这些板任意分布在棒上，但关键限制是每种颜色总共出现七次，每个尺寸出现一次。 

允许的移动是从一根棍子上取出顶部连续的盘子块，如果它们按顺序堆叠的话，可能是多个盘子，然后将整个块移动到另一根棍子上。 仅当目标棒为空或其当前顶板与移动块颜色相同且尺寸严格大于移动块底板时，移动才是合法的。 这会在每种颜色中引入“通过减小尺寸进行堆叠”规则，并且不同颜色永远不会以破坏排序限制的方式混合。 

我们的目标是重新排列盘子，以便对于每种颜色，其七个盘子按照从上到下递增的尺寸顺序形成一个干净的塔，并且我们希望最大限度地减少所需的移动次数。 

输入尺寸小且结构化：正好有 n 根棍子，正好有 n − 2 种颜色，每种颜色恰好有 7 个板。 这立即表明，每种颜色的核心结构都是独立的，因为除了通过共享棒之外，颜色不会相互作用。 任何试图天真地模拟所有颜色的全局移动的解决方案都已经在一个太大而无法明确探索的状态空间中运行。 

主要约束含义是每种颜色都贡献一个具有恒定大小七个元素的固定大小子问题。 即使我们必须模拟所有可能的移动，每种颜色的状态空间也受到一个常数的限制，因此该常数的指数解在理论上是可以接受的。 然而，仍然没有必要对所有颜色组合进行全面模拟。 

一个微妙的边缘情况是由于移动在堆栈上而不是单个盘子上进行的事实而产生的。 例如，如果一种颜色的印版与其他颜色以交错的方式分割在多个棒上，则粗心的模拟可能会错误地假设印版可以独立移动。 另一种失效模式是忽略“严格更大的尺寸”约束，这迫使刚性单调结构处于有效的中间配置中。 

一个最容易混淆的例子是颜色板显示为：

 棒1：A2 A0

 棒2：A1 A3 A4 ...

 一种简单的方法可能会尝试首先独立移动 A0，但除非 A1 已经处于有效位置，否则它会被阻止。 正确的推理必须将七块板视为一个受约束的有序序列，最终必须组装在一座塔中。 

## 方法

 蛮力的想法是模拟所有有效的移动序列。 在每一步中，我们从一根棍子中选择一个堆栈段，并尝试将其移动到满足颜色和大小约束的任何其他棍子上。 这自然会导致一个巨大的状态图，其中每个节点都是所有板在棍上的分布，并且每条边都是合法的移动。 

即使对于单一颜色，配置的数量也增长得非常快，因为七个板中的每一个都可以安装在任何具有订购限制的棒上。 在所有颜色中，这成为独立状态空间的产物，即使每种颜色的尺寸很小，暴力很快就变得不可行。

关键的观察结果是，除了共享棍子之外，颜色不会干扰最佳结构，并且在每种颜色中，问题相当于将固定大小的受限堆栈分类为单个有序塔。 移动规则强制要求板只能以严格更大的尺寸放置，这与构造单调堆栈的结构完全相同，其中元素必须以受控方式从最小到最大组装。 

一旦我们分离出单一颜色，我们就会认识到该过程相当于在经典堆栈传输约束下移动一座 7 号塔。 每种颜色都是独立的，因此总答案是每种颜色的最佳移动之和。 

对于具有七个有序板的单一颜色，最小移动次数遵循与经典的“移动具有堆叠约束的所有磁盘”结构相同的确定性模式。 重复的结论是，为了正确放置尺寸 k，我们必须首先清除并放置较小的尺寸，从而导致跨级别的加倍行为。 这将产生闭合形式 2^7 − 每种颜色移动 1 次。 

由于有 n − 2 种颜色，最终答案变为 (n − 2) × (2^7 − 1)，简化为 127 × (n − 2)。

 ## 算法演练

 ## 算法演练

 1. 读取棒数n。 由此，确定颜色的数量，即 n − 2。这直接由构造规则暗示，即每种颜色恰好贡献七个板。 
2. 认识到每种颜色形成一个独立的子问题，因为移动保留颜色结构并且从不需要跨颜色依赖性来满足有效性。 
3. 对于单一颜色，请注意我们必须以从顶部尺寸 0 到底部尺寸 6 的单个有序堆栈结束。 任何有效的移动序列最终都必须组装出这个精确的配置。 
4. 将装配过程建模为从最小到最大逐渐固定尺寸。 当放置尺寸 k 时，所有较小的尺寸必须已经正确排列，并且堆叠约束迫使它们表现得像递归子结构。 
5. 这种递归依赖性产生了双重结构：要移动大小为 0..k 的正确块，我们必须首先将 0..k−1 移到一边，放置 k，然后恢复较小的块。 这与受限的塔楼重复循环相同。 
6. 求解 k = 6 的递归式（总共七块板），得出每种颜色 2^7 − 1 次移动。 
7. 将该成本乘以颜色数 n − 2 以获得最终答案。 

### 为什么它有效

 关键的不变量是，在任何单一颜色中，尺寸为 0 到 k 的板形成一个连续的逻辑结构，一旦部分组装，该结构必须始终作为一个单元移动。 移动限制强制要求较小的板不能以违反顺序的方式放置在较大的板之上，因此任何部分配置都会分解为颜色的“完整前缀堆栈”。 

因为每个操作要么推进此前缀的构造，要么以对称方式临时分解它，因此该过程的行为与大小级别上的二进制递归完全相同。 这迫使一个独特的最小移动序列，其长度仅取决于级别的数量，而不取决于初始分布或棒布局。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    # Each color contributes exactly 7 plates, and there are n-2 colors.
    # Each color requires 2^7 - 1 = 127 moves.
    print((n - 2) * 127)

if __name__ == "__main__":
    solve()
```该代码完全依赖于问题的结构简化。 除了读取 n 之外，无需解析各个堆栈，因为板的配置不会影响最终的最佳移动计数。 唯一微妙的一点是确保 n − 2 计算正确，因为这里的差一会错误地缩放整个答案。

 ## 边缘情况

 对于 n = 3，只有一种颜色。 该算法直接返回127，这相当于完​​全建造了一座7板塔。 

当 n = 4 时，有两种独立的颜色。 每个都独立地影响总成本，并且不会发生干扰，因为移动永远不会以影响最优性的方式混合颜色约束。 

对于非常大的 n，乘法仍然是安全的，因为计算是单个整数运算，并且 Python 中不存在溢出风险。 

在所有情况下，关键属性是每种颜色的成本保持固定，并且对初始堆栈配置的隐藏依赖性不会改变该成本。
