 ## 解决方案
 ## 问题理解

 我们得到了一个部分填充的 9 x 9 数独网格。 每个单元格要么已包含 1 到 9 的数字，要么为空。 任务是完成网格，使每行包含每个数字恰好一次，每列包含每个数字恰好一次，并且每个 3 x 3 子网格也包含每个数字恰好一次。 

输入可以被视为具有约束的固定棋盘状态，输出是在不违反数独规则的情况下扩展此状态的任何有效的已完成棋盘。 如果存在多个完成，则任何一个有效的完成都是可接受的。 

问题的结构立即表明最坏情况下的指数搜索空间，因为每个空单元格最多可以容纳九个值。 然而，网格大小固定为 81 个单元，因此真正的困难不是输入大小的增长而是约束交互。 这将问题从“大规模计算”转变为“仔细修剪小型组合空间”。 

即使在具有许多空白的单个拼图上，尝试每个空单元格的所有数字而不进行修剪的朴素递归填充也会爆炸，因为分支因子高达 9，深度高达 81。 

典型的隐藏故障模式来自于没有一致地执行所有三个约束。 例如，如果实现仅验证一个维度，则将在行中有效但在列或框中无效的数字仍然可以通过部分检查。 

如果在回溯期间未正确回滚更新，则会出现另一个微妙的问题。 一种常见的错误实现将数字标记为行或列中使用的数字，但在回溯时忘记取消标记，从而导致错误的修剪和错误的“无解”结论。 

## 方法

 暴力方法将数独视为空单元格的纯粹搜索问题。 我们扫描网格，选择第一个空单元格，尝试数字 1 到 9，然后递归地继续。 这是正确的，因为它探索了与之前的选择一致的每一个可能的分配。 

然而，其最坏情况的复杂度约为$9^k$， 在哪里$k$是空单元格的数量。 即使从有效性检查中进行适度的修剪，对于具有许多空白的情况来说，分支因子仍然太大。 

使数独在实践中可解的关键观察是约束检查非常便宜且高度结构化。 每个放置恰好影响三组：一行、一列和一个 3 x 3 块。 这使我们能够维护增量状态，以便有效性检查是 O(1)，并且我们可以立即积极地修剪无效分支。 

进一步的改进来自于更智能地选择下一个单元。 我们总是选择有效候选者最少的空单元格，而不是按固定顺序填充单元格。 这极大地减少了典型谜题中的分支，因为受约束的单元会提前折叠搜索树。 

我们可以为行、列和框维护三个布尔约束表，并在递归期间更新它们。 这将解决方案转变为引导回溯搜索，每次移动都进行恒定时间有效性检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(9^k) | O(9^k) | O(81) | 太慢了 |
 | 带约束的优化回溯 | O(小指数，重剪枝) | O(81) | 已接受 |

 ## 算法演练

 1.解析9乘9的网格并记录所有空位置。 在此过程中，构建三个约束跟踪器：每行、每列和每个 3 x 3 子网格中已使用哪些数字。 这使我们能够在常数时间内回答“我可以在这里放置数字 d”。 
2. 定义一个函数，使用单元格坐标的整数除法来计算单元格的 3 x 3 框索引。 这是必要的，因为数独约束被划分为固定块。 
3. 实现一个尝试填充网格的递归求解器。 每次调用时，选择下一个空单元格。 不要随意选择，而是根据当前限制选择有效数字最少的那个。 这减少了早期的分支，这是大多数指数节省的来源。 
4. 对于所选单元格，迭代数字 1 到 9。对于每个数字，检查其行、列和框中是否不存在该数字。 如果有效，则放置它并将其标记为在所有三个约束结构中使用。 
5.递归求解剩余网格。 如果递归成功，则立即向上传播成功。 
6. 如果没有数字得出解决方案，则撤消放置并回溯。 这将恢复所有约束结构，以便早期决策保持一致。 
7. 如果所有单元格都已填满，则返回成功，棋盘完成。 

### 为什么它有效

 在每一步中，算法都会保持所有填充单元格都满足数独约束的不变性。 约束表确保不会放置无效数字，回溯则保证部分分配是可逆的。 由于到目前为止，每个递归状态仅探索与约束一致的配置，因此搜索空间不包含无效分支，仅包含潜在的完整解决方案。 由于网格是有限的，并且每个步骤都会严格减少空单元的数量，因此递归必须最终通过找到有效的完整分配或穷尽所有可能性来终止。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_sudoku(board):
    rows = [[False]*10 for _ in range(9)]
    cols = [[False]*10 for _ in range(9)]
    boxes = [[False]*10 for _ in range(9)]
    empties = []

    def box_id(r, c):
        return (r // 3) * 3 + (c // 3)

    for r in range(9):
        for c in range(9):
            if board[r][c] == 0:
                empties.append((r, c))
            else:
                d = board[r][c]
                rows[r][d] = True
                cols[c][d] = True
                boxes[box_id(r, c)][d] = True

    def get_candidates(r, c):
        b = box_id(r, c)
        return [d for d in range(1, 10)
                if not rows[r][d] and not cols[c][d] and not boxes[b][d]]

    def dfs():
        if not empties:
            return True

        best_idx = -1
        best_cands = None

        for i, (r, c) in enumerate(empties):
            cands = get_candidates(r, c)
            if not cands:
                return False
            if best_cands is None or len(cands) < len(best_cands):
                best_cands = cands
                best_idx = i

        r, c = empties.pop(best_idx)
        b = box_id(r, c)

        for d in best_cands:
            if not rows[r][d] and not cols[c][d] and not boxes[b][d]:
                board[r][c] = d
                rows[r][d] = cols[c][d] = boxes[b][d] = True

                if dfs():
                    return True

                rows[r][d] = cols[c][d] = boxes[b][d] = False
                board[r][c] = 0

        empties.append((r, c))
        return False

    dfs()
    return board

def main():
    board = []
    for _ in range(9):
        line = input().strip()
        row = []
        for ch in line:
            if ch in "0.":
                row.append(0)
            else:
                row.append(int(ch))
        board.append(row)

    solve_sudoku(board)

    for r in range(9):
        print("".join(str(x) for x in board[r]))

if __name__ == "__main__":
    main()
```求解器为行、列和框构建恒定时间约束表。 DFS 例程始终首先选择最受约束的空单元，这是防止病理分支的关键启发式方法。 撤消逻辑与放置逻辑对称，确保回溯期间的正确性。 一个微妙的实现细节是，所选单元格会暂时从空列表中删除，并且仅在分支失败时才恢复，这避免了重新访问相同的决策状态。

 ## 边缘情况

 ### 完全空的网格

 输入是全零的网格。 该算法从最大分支开始，但一旦发生前几个放置，就立即受益于约束传播。 启发式选择可以防止均匀分支爆炸。 

### 已经解决网格

 空列表从一开始就是空的。 由于满足基本条件，DFS 立即终止，返回原始网格不变。 

### 单强制单元

 具有一个缺失值的几乎完整的网格测试约束表是否正确识别唯一有效的数字。 该算法计算该单元格的候选者并直接填充它而无需回溯。 

### 高度约束的中心单元

 当最受约束的单元格位于网格中间时，启发式方法首先正确地选择它。 这避免了探索不相关的外围选择并保持浅层递归。
