 ## 解决方案
 ## 问题理解

 我们得到一个用二进制数组表示的一维房子。 某些位置包含加热器，标有`1`，而空位置是`0`。 每个加热器一旦启动，就会在其位置周围连续加热一段时间。 加热半径由固定参数决定`r`，表示加热器位于`pos`涵盖所有索引`pos - r + 1`到`pos + r - 1`。 

任务不仅仅是覆盖加热器位置，而是确保整个数组中的每个索引都来自`1`到`n`位于至少一个已启动加热器的覆盖范围内。 在所有现有的加热器中，我们必须选择最小数量的加热器来启动，以便它们的覆盖范围的联合完全覆盖整个房子。 如果这是不可能的，意味着无论我们选择什么，某些位置都不能被任何加热器覆盖，我们返回`-1`。 

限制条件`n, r ≤ 1000`意味着二次甚至三次解仍然是可以接受的，但问题的结构表明采用贪婪区间覆盖方法。 每个加热器变成固定间隔，目标缩减为覆盖一条线段`[1, n]`以最小间隔。 

关键的边缘情况是某个位置在任何范围内都没有可以覆盖它的加热器。 例如，如果所有`a[i] = 0`，那么不存在区间，答案是立即`-1`。 另一个微妙的情况是当加热器存在但在其覆盖范围之间留下间隙时。 例如，与`r = 2`, 加热器位置`1`和`4`离开位置`3`如果它们的范围不重叠或弥合间隙，则尽管有多个加热器，但仍无法实现完全覆盖。 

## 方法

 暴力方法会将其视为间隔内的集合覆盖问题。 我们可以生成所有加热器间隔，然后递归地或通过位掩码 DP 尝试所有子集并检查它们的并集是否覆盖`[1, n]`。 这是正确的，但很快就变得不可行，因为可能有多达`n`加热器，因此`2^n`子集，其增长远远超出任何合理的限制，即使对于`n = 1000`。 

问题的结构比一般的集合覆盖更具体。 每个加热器对应一条线上的固定间隔，并且所有间隔都是独立的。 一旦我们按位置对它们进行排序或扫描，激活哪个加热器的决定就变得局部贪婪：当站在最左边的未覆盖点时，我们应该选择在该点之前或当时开始覆盖的加热器，并尽可能向右扩展覆盖范围。 

这将问题转化为经典的线段最小区间覆盖问题。 在每一步中，我们都会维护当前未覆盖的点，并查找左边界在范围内的所有加热器。 其中我们选择覆盖范围最远的。 这种贪婪的选择之所以有效，是因为选择任何较短的间隔只会减少未来的影响范围，而无助于以后的决策。

 | --- | --- | --- | --- |
 | 蛮力 | O(2^k·n) | O(2^k·n) | O(k) | 太慢了|
 | 最优贪心 | O(n) | O(n) | 已接受 |

 ## 算法演练

 每个加热器在位置`i`（在哪里`a[i] = 1`) 定义一个区间`[i - r + 1, i + r - 1]`。 这些间隔可能会延伸到外部`[1, n]`，所以我们夹紧它们。 

然后我们从左到右扫描房子，总是试图覆盖最早未被覆盖的位置。 

1. 建立所有有效的加热器间隔。 如果加热器位于`i`，计算其左右覆盖范围并存储它们。 
2. 按间隔的左端点对间隔进行排序，以便我们可以按顺序扫描它们。 
3. 初始化指针`i = 1`代表第一个未覆盖的位置，一个索引`idx = 0`间隔时间和激活加热器的计数器。 
4. 虽然`i ≤ n`, 搜索所有左端点为的区间`≤ i`。 其中，找到右端点最大的那个。 
5. 如果没有区间覆盖位置`i`，那么没有加热器可以达到这一点，任务是不可能的。 
6. 否则，激活找到的最佳间隔，增加答案，然后移动`i`到`best_right + 1`。 
7. 继续，直到覆盖整个范围。 

关键的不变量是在每次迭代之前，所有位置`< i`已经覆盖，我们正在选择可以覆盖位置的最佳间隔`i`。 由于区间是固定的，任何解必须至少包含一个区间覆盖`i`，并且选择延伸最远的一个永远不会降低未来职位的可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, r = map(int, input().split())
    a = list(map(int, input().split()))

    intervals = []
    for i in range(n):
        if a[i] == 1:
            l = max(1, i + 1 - r + 1)
            rr = min(n, i + 1 + r - 1)
            intervals.append((l, rr))

    intervals.sort()

    i = 1
    idx = 0
    ans = 0
    m = len(intervals)

    while i <= n:
        best = -1
        while idx < m and intervals[idx][0] <= i:
            best = max(best, intervals[idx][1])
            idx += 1

        if best < i:
            print(-1)
            return

        ans += 1
        i = best + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将每个加热器转换为覆盖区间，小心地从基于 0 的索引转换为基于 1 的索引。 夹紧确保边缘附近的加热器不会产生无效范围。 

主循环维持当前未覆盖的位置`i`。 内循环前进所有可以在之前或正好开始的时间间隔`i`。 其中，它跟踪最远的可到达端点。 这是关键的贪婪步骤：我们一次性消耗当前位置的所有可行选项，并致力于最佳扩展。 

指针`idx`只向前移动，因此每个区间都处理一次，排序后保持解的线性。

 ## 边缘情况

 临界边缘情况是当加热器存在但由于半径限制而未覆盖当前未覆盖的位置时。 在这种情况下，内循环找不到左端点的区间`≤ i`延伸过去`i`，算法正确返回`-1`。 例如，与`n = 5, r = 1`和加热器仅在位置`1`和`5`，中间位置保持未被覆盖，因为每个加热器只覆盖自身，因此贪婪扫描恰好在该位置失败`2`。 

另一个微妙的情况是间隔严重重叠时。 即使存在许多加热器，也只需要几个。 贪婪的选择确保我们永远不会浪费在同一决策点提供比其他可用选项更少的扩展的加热器。
