 ## 解决方案
 ## 问题理解

 我们得到两个非常大的二进制数，不是整数而是字符串。 第一个数字在整个过程中是固定的，而第二个数字则不断缩小。 

这个过程是机械的：从全部价值开始`b`。 在每一步中，计算之间的按位与`a`和当前的`b`，将该 AND 结果解释为普通整数值（不是二进制字符串），并将其添加到累积答案中。 然后丢弃最低有效位`b`将其除以二，然后重复直到`b`变为零。 

思考这个问题的一个有用的方法是，我们不断地滑动一个缩小版本`b`穿过`a`，每次按位重叠，并对它们的数值求和。 

限制很大，两个字符串都可能高达 200,000 位。 任何为每次移位重新计算完整按位 AND 的解决方案都将有效地完成 O(nm) 的工作，这太慢了，在最坏的情况下约为 400 亿次操作。 这立即排除了任何通过完整扫描显式模拟每个步骤的方法。 

在简单的实现中出现的一个微妙问题是将每个 AND 结果视为二进制字符串并重复将其转换为整数。 即使位运算被优化，每次移位的完整数字的重复重建也会导致二次行为。 

边缘情况主要与结构有关，而不是与大小有关。 如果一个字符串全是 1，则每次移位都会产生很大的影响，并且简单的重新计算将变得极其昂贵。 如果`b`有很长一段时间的零，一个幼稚的实现仍然会不必要地迭代它们，即使它们对最终答案没有任何贡献。 

## 方法

 蛮力方法从字面上解决问题。 对于每个状态`b`，我们将其与`a`，计算所有重叠位置上的按位 AND，将该二进制结果解释为数字，将其添加到答案中，然后移位`b`正好一。 每个 AND 计算最多扫描 O(n) 位，并且有 O(m) 次移位，总工作量为 O(nm)。 当n和m达到2e5时，这是完全不可行的。 

关键的观察是，我们实际上不需要为每个班次独立地重新计算完整的 AND 字符串。 相反，我们可以直接根据每个匹配位对的最终数字权重重新解释它们的贡献。 

有点到位`j`的`a`有助于轮班时的最终 AND 结果`i`仅当有匹配时`1`在`b`在位置`j + i`。 当这种情况发生时，它会准确地贡献最终整数中该位的值，这仅取决于它在`a`。 这允许我们交换求和的顺序：我们不是迭代轮班，而是对每个位置的贡献求和`a`基于它与中的重叠次数`b`。 

这将整个问题简化为前缀或后缀计数`b`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(纳米) | O(n) | 太慢了|
 | 最佳 | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 1. 将两个二进制字符串转换为字符数组，保留索引，使索引 0 代表最高有效位。 这允许稍后对位权重进行一致的解释。 
2. 预先计算 2 模 998244353 中每个位置的幂`a`。 索引处的位`j`贡献价值`2^(n-1-j)`当它出现在 AND 结果中时。 这避免了重复重新计算幂运算。 
3. 构建后缀数组`b`在哪里`suf[j]`存储 1 的数量`b[j..m-1]`。 此步骤捕获有多少未来的转变将对齐给定的位置`a`与一个`1`在`b`。 
4. 迭代每个位置`j`在`a`。 如果`a[j]`为零，它没有任何贡献，可以完全跳过。 
5.如果`a[j]`是一，那么每一个`1`在`b`在位置`k >= j`最终将在一个班次内与它对齐。 此类对齐的数量为`suf[j]`。 
6. 添加`pow2[j] * suf[j]`对答案进行累加模 998244353。 
7. 输出最终累加和。 

正确性取决于每对匹配的事实`(a[j], b[k])`在轮班时只贡献一次`i = k - j`，并准确贡献`2^(n-1-j)`到答案。 

### 为什么它有效

 每对索引`(j, k)`这样`a[j] = 1`和`b[k] = 1`恰好有助于一次 AND 计算：其中`b`被移动以便位置`k`与位置对齐`j`。 该贡献独立于所有其他位。 因为从这个意义上说，按位与对于贡献是线性的，所以我们可以成对地求和贡献，而不是每次移位重新计算完整的二进制数。 这将类似移位卷积的过程转变为对齐过程上的简单计数问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, m = map(int, input().split())
    a = input().strip()
    b = input().strip()

    # suffix count of ones in b
    suf = [0] * (m + 1)
    for i in range(m - 1, -1, -1):
        suf[i] = suf[i + 1] + (1 if b[i] == '1' else 0)

    # precompute powers of 2
    pow2 = [1] * (n)
    for i in range(n - 2, -1, -1):
        pow2[i] = (pow2[i + 1] * 2) % MOD

    ans = 0

    for j in range(n):
        if a[j] == '1':
            ans = (ans + pow2[j] * suf[j]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```后缀数组`suf`是取代重复重新计算移位重叠的关键优化。 功率阵列对每个位位置的固定贡献进行编码`a`。 最后的循环直接组合两个部分，避免了对换档过程的任何显式模拟。 

一个常见的错误是索引`pow2`错误地。 自从`a[0]`是最高有效位，它的权重是`2^(n-1)`，因此幂数组必须相应对齐，而不是使用递增指数。

 ## 边缘情况

 当`a`由单个`1`后面跟着零，只有第一个位置有贡献，答案简化为计算`b`。 该算法直接处理这个问题，因为所有其他位置都被跳过。 

什么时候`b`由于后缀数组高度稀疏，因此我们不会在班次之间重复扫描零，因为每个位置在累积结构中都会计算一次。 

当两个字符串都完全密集时，朴素移位将重复重新计算相同的重叠，但优化的解决方案在单次传递中聚合所有贡献而没有冗余，即使在最大输入大小下也能确保稳定性。
