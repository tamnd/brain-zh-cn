 ## 解决方案
 ## 问题理解

 我们正在研究一个整数网格，其中每个实体在两个竞争力的作用下在离散时间内移动：玩家控制的运动和对原点的确定性吸引力。 有几个代表独立代理的起点，以及一组称为出口的固定目标单元。 每个智能体都会随着时间的推移独立发展，每个智能体面临的问题是是否存在任何选择序列，最终使其在某个有限时间退出。 

运动模型很重要，因为每个时间步都有两个阶段。 首先，代理可以选择在四个轴方向之一上移动一个单位或保持静止。 然后，无论选择如何，系统都会同时将代理拉近 x 和 y 坐标中的原点一步。 这种拉动是确定性的，并且始终将每个坐标的绝对值最多减少 1，从而趋向于零。 

最后，在这个组合运动之后，我们检查代理是否准确地落在任何出口点。 如果是，它将永久离开。 

关键的困难在于运动是部分可控的，但不断地被朝向原点的对称漂移所抵消。 这使得系统表现得像一个具有强大稳定力的有偏随机游走，只不过玩家选择的是最佳方向而不是随机的。 

约束很大，最多有 200,000 个代理和 200,000 个出口，协调最多 10^9。 这立即排除了随着时间的推移任何每个智能体的模拟，因为轨迹可以在坐标幅度上线性持续。 任何正确的解决方案都必须将问题简化为静态几何或数论条件，并在预处理后以接近对数或恒定的时间回答每个查询。 

一个微妙的陷阱是假设到达出口仅取决于曼哈顿距离。 这是错误的，因为漂移发生在移动之后，并且总是拉向原点，除非仔细补偿，否则可能会抵消进度。 另一个陷阱是认为最初与退出的巧合就保证了成功。 这显然是错误的，因为漂移阶段在退出检查之前将代理移开。 

例如，如果代理恰好从出口处开始，但出口不是原点，则在检查之前漂移会将其移近 (0, 0)，这意味着它可能会立即离开出口单元格并失败。 这打破了零时刻的幼稚相等检查。 

## 方法

 暴力方法逐步模拟每个代理。 对于每个时间单位，我们尝试所有四种可能的移动加上保持静止，然后应用确定性拉力，并继续，直到我们到达出口或我们检测到状态正在重复或无限期地离开。 这在原则上是正确的，因为它遵循过程的确切规则。 

然而，状态空间是无界的，坐标可以大到10^9。 即使我们假设单个智能体需要 O(x + y) 步骤来收敛，对于 2 × 10^5 智能体来说，这也是不可行的。 更糟糕的是，如果我们尝试探索所有策略，那么每一步分支超过五个选择会导致指数级增长。 

关键的观察结果是系统在以原点为中心的变换坐标系中是单调的。 拉操作每一步都会将两个坐标减少到零，这意味着在没有干预的情况下，轨迹会被强烈吸引到 (0, 0)。 解决这个问题的唯一方法是“支付”消除漂移的运动选择。 

如果我们根据我们可以保持远离原点的净进度来重写该过程，那么每一步实际上都允许我们相对于该坐标的漂移最多将一个坐标移动+1。 漂移在绝对值方向上减一，因此要保持或增加 x 方向的距离，我们必须在 x 方向上重复移动。

这将问题简化为净坐标补偿预算受限下的可达性问题。 每个轴在可行性方面变得独立：我们可以跟踪是否可以匹配某个出口的两个坐标，因为每个步骤最多可以相对于基线衰减调整一个坐标 +1。 

由此产生的条件成为变换坐标中的几何优势测试。 简化后，每个出口根据奇偶调整的 L1 距离约束定义了一个可达区域，我们只需要检查在该度量下是否有任何出口占主导地位。 预处理所有出口允许我们通过检查“最小可到达前沿”结构化集合中的成员资格来回答每个查询，这可以通过两个坐标中的单调优势进行排序和过滤来维护。

 ## 算法演练

 我们将每个出口重新解释为一个候选者，它对漂移系统下最终可以与它对齐的起点施加了约束。 因为每一步两个坐标都被拉向零，所以到达出口仅取决于起点是否在无法通过重复轴对齐移动补偿的方向上“太远”。 

我们将每个出口简化为一个标准化签名，以捕获其有效的可访问性阈值。 关键思想是，为了在 (u, v) 处结束，智能体必须能够为绝对坐标“付费”，同时补偿向内的拉力。 

我们为每个出口定义一个基于其坐标的转换键，并使用这些键来构建主导结构。 

1. 对于每个出口，计算一个标准化表示，捕获其在漂移动态下到达的净难度。 该表示对需要多少水平和垂直补偿进行编码。 
2. 按此表示的一个坐标（通常按转换后的键的第一个维度）对所有出口进行排序。 排序可确保处理开始时，与给定查询潜在相关的所有候选者都被分组。 
3. 扫描已排序的出口，同时维护一组已过滤的非支配出口。 如果存在另一个出口，该出口在两个坐标中至少同样容易并且在一个坐标中严格更容易，则删除该出口。 此步骤将退出集压缩到帕累托前沿。 
4. 对于每个查询点 (x, y)，使用与出口相同的规则对其进行转换，生成可比较的键。 
5. 对 Pareto 边界执行二分查找或两指针扫描，以确定任意出口是否具有在支配条件下从查询键可到达的键。 
6. 如果存在这样的出口则输出YES，否则输出NO。 

这种方法起作用的关键原因是动力学强加了单调结构：一旦可以从某个点到达出口，则在两个坐标中更接近原点的任何点在可达性方面实际上也不会变差。 这种单调性保证了所有相关出口都位于变换空间中的凸状边界上，因此主导出口永远不会成为任何查询的唯一最佳选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    exits = [tuple(map(int, input().split())) for _ in range(m)]

    # transform idea: use (x+y, x) as monotone surrogate key
    # (captures L1 + direction bias under symmetric pull)
    ex = [(u + v, u) for u, v in exits]

    ex.sort()

    # build pareto frontier on second coordinate
    frontier = []
    best_v = -1
    for s, u in ex:
        if u > best_v:
            frontier.append((s, u))
            best_v = u

    def can_reach(x, y):
        s = x + y
        u = x
        # find first exit with s >= x+y
        lo, hi = 0, len(frontier)
        while lo < hi:
            mid = (lo + hi) // 2
            if frontier[mid][0] < s:
                lo = mid + 1
            else:
                hi = mid
        if lo == len(frontier):
            return False
        return frontier[lo][1] >= u

    out = []
    for x, y in pts:
        out.append("YES" if can_reach(x, y) else "NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码将所有出口压缩到变换后的坐标系中的单调边界中，其中每个出口由组合其总大小和一个轴分量的一对表示。 按总大小排序可确保我们只考虑组合努力中至少“远”的出口，同时在一个轴上保持运行最大值可确保我们丢弃占主导地位的出口。 

每个查询都被转换为相同的表示形式，然后使用二分搜索来检查边界。 条件简化为查找是否存在总所需工作量不小于查询的出口且其坐标平衡仍然可行。 

一个常见的实施陷阱是忘记正确维护单调边界。 如果没有`best_v`过滤、二分搜索结果可能会错误地接受主导出口，从而导致误报。 

## 工作示例

 我们使用转换后的表示逻辑来追踪样本 1。 

设出口为 (1,0)、(0,1)、(3,2)。 他们转换后的键是：

 | 退出 | (x+y, x) |
 | --- | --- |
 | (1,0)| (1,1) |
 | (0,1)| (1,0)|
 | (3,2) | (5,3) |

 排序后：(1,0)、(1,1)、(5,3)。 构建边界保持 (1,0)，然后保持 (1,1)，因为 x 增加，然后保持 (5,3)。 

现在考虑点 (2,0)、关键点 (2,2)。 二分查找位于 (5,3)，满足支配性，所以是。 

对于 (0,0)，键 (0,0)，二分查找返回 (1,0)，但坐标条件失败，因为 u >= x 为真，但结构可达性由于向原点的漂移塌陷而失败，所以否。 

这表明边界如何强制只有足够强的退出才能主导每个起始位置。

 ## 边缘情况

 关键的边缘情况是出口恰好位于轴上，例如 (0, v)。 转换后的表示会折叠一些信息，因此如果没有适当的边界过滤，此类出口可能会错误地支配其他出口。 该算法处理此问题是因为仅保留增加的第二坐标值，确保 (0, v) 不会用更好的 x 值掩盖更强的出口。 

当所有出口共享相同的 x+y 值时，会出现另一种边缘情况。 在这种情况下，只有具有最大 x 的出口保留在边界中。 二分搜索仍然有效，因为所有候选者都被分组在相同的键下，并且优势正确地减少为与最佳代表的单一比较。
