 ## 解决方案
 ## 问题理解

 我们有两个非常大的整数，每个整数代表一个写在单独记录卡上的数字。 任务不是计算全部总和，而只是确定总和的最后一位数字。 然后将该值用作机械书架锁的一位数字访问代码。 

更具体地说，输入由两个整数组成$A$和$B$，每个都可以非常大，最多可达$10^{100}$。 输出是一位数字，对应于$(A + B) \bmod 10$。 

尺寸限制是这里的关键信号。 数字高达$10^{100}$不能以标准 64 位整数类型存储，如果我们只关心最后一位数字，那么即使任意长度的整数加法也将是不必要的矫枉过正。 任何尝试解析完整整数并直接使用本机整数算术计算其总和的解决方案要么会失败，要么会变得不必要的繁重。 

在考虑输入格式时会出现一种微妙的边缘情况。 由于数字可能非常大，因此它们可能包含远远超出任何标准数字类型容量的前导数字。 例如：

 输入：```
99999999999999999999999999999999999999999999999999 1
```正确输出：```
0
```尝试转换为整数类型的简单实现在这里会失败。 即使使用大整数，当只有最后一位数字很重要时，计算总和也是不必要的工作。 

另一种边缘情况是当一个或两个数字都是单位值时，逻辑仍应表现一致：

 输入：```
7 8
```输出：```
5
```输出的简单性掩盖了这样一个事实：相同的逻辑必须在所有量级上一致地工作。 

## 方法

 暴力方法会将两个数字解释为完整整数，将它们转换为大整数类型，计算总和，然后提取最后一位数字。 这在概念上是正确的，因为问题简化为模运算。 然而，在没有任意精度支持的语言中，这会由于溢出而立即失败。 即使在支持大整数的语言中，这种方法也是浪费的，因为它会处理所有数字，即使只有最后一位数字很重要。 

关键的观察是总和的最后一位数字仅取决于加数的最后一位数字。 这是从模算术得出的：$$(A + B) \bmod 10 = ((A \bmod 10) + (B \bmod 10)) \bmod 10$$因此，我们不需要解析整个数字，只需读取它们的最终字符，将它们转换为整数，然后计算它们的和以 10 为模。 

这将问题从任意长度算术减少到恒定时间计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（全大整数加法）| O(n) | O(n) | 已接受但没有必要 |
 | 最佳（仅使用最后一位数字）| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 将两个数字读取为字符串而不是整数。 这可以避免任何溢出风险，并准确地保留给定的所有数字。 问题的结构保证了除了字符检查之外我们不需要数字解析。 
2. 提取第一个字符串的最后一个字符并将其转换为整数。 这代表$A \bmod 10$，因为十进制数的最后一位完全决定了其模 10 的余数。 
3. 提取第二个字符串的最后一个字符并将其转换为整数。 这代表$B \bmod 10$出于同样的原因。 
4. 计算这两个数字的总和。 
5. 对结果取模 10 以处理任何超出一位数的进位，因为输出也必须是一位数。 
6. 输出结果数字。 

工作原理：十进制表示形式保留了相对于 10 的模块化结构，这意味着除了最后一个数字之外的每个数字都贡献了 10 的倍数，因此在模 10 下消失。该算法依赖于以下不变量：对于以 10 为基数编写的任何整数，在最后一个数字之前结尾的所有前缀都可以被 10 整除，因此它们不会影响最终的余数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b = input().split()

da = ord(a[-1]) - ord('0')
db = ord(b[-1]) - ord('0')

print((da + db) % 10)
```该解决方案将两个值读取为字符串，并避免任何超出单个字符的数字转换。 表达式`a[-1]`和`b[-1]`无论长度如何，都可以安全地访问最后一位数字。 使用`ord(...)-ord('0')`是一种将字符数字转换为整数的快速且显式的方法，而无需调用较慢的解析例程。 

即使两位数之和超过 9，最终的模运算也能确保正确性，否则会产生两位数的中间结果。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    a, b = sys.stdin.readline().split()
    return str((int(a[-1]) + int(b[-1])) % 10)

assert run("1 1") == "2"
assert run("114514 1919810") == "4"

 ## 边缘情况

 对于两个数字都以 9 结尾的输入，进位可能表明需要完全加法，但算法会安全地忽略所有更高的数字。 

输入：```
9 9
```执行：```
da = 9, db = 9
sum = 18
output = 8
```尽管真实的和是 18，但只需要最后一位数字，并且该方法直接产生它，而不需要计算完全加法。 

对于零重的情况，例如：```
0 99999999999999999999
```只有第二个数字的最后一位数字很重要：```
da = 0, db = 9
output = 9
```该算法正确地忽略了长前缀零和大数字，因为它们在模 10 算术中是不相关的。
