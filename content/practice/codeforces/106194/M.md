 ## 解决方案
 ## 问题理解

 我们给定一个放置在圆上的点，每个点都有一个角度 θ 和一个独立的“激活”概率 p。 激活后，每三个激活点形成一个三角形，所有这些三角形一起充当防御区域。 核心问题是原点是否至少被这些三角形之一覆盖。 如果没有被覆盖，则攻击成功。 

因此，任务简化为计算圆心不包含在由激活点形成的任何三角形中的概率。 

从几何角度来看，这相当于询问激活点集何时无法“包围”原点。 平面几何中众所周知的事实是，圆上的一组点包含其凸包中的原点当且仅当这些点不包含在任何半圆中时。 由于由这些点形成的三角形生成完整的凸包，因此当且仅当激活的集合不包含在任何半圆中时，原点才会被覆盖。 

因此，问题变成：计算所有激活点位于某个半圆内的概率。 

输入大小 n 高达 10^5 排除了子集的任何指数枚举，因为 2^n 远远超出了可行的计算范围。 即使 O(n^2) 方法也太慢了。 排序后我们需要一个 O(n log n) 或 O(n) 的解决方案。 

一个微妙的问题是角度是连续浮动的。 如果两个点具有几乎相同的角度，或者如果发生 2π 的环绕，则天真的窗口逻辑可能会被破坏，除非我们仔细规范化循环排序。 

边缘情况出现在以下情况：

 首先，所有点都确定地位于半圆内，但概率不同。 例如，如果所有 p = 1 并且点位于 180 度范围内，则答案必须为 1。 

其次，当没有两个点靠近且所有点均匀分布时，只有在宽跨度上至少有 3 个点激活时，它们包含在半圆中的概率才变为 0。 幼稚的独立性假设会失败，因为该条件耦合了许多变量。 

第三，如果处理不当，重复或接近重复的角度可能会打破严格的“< π”与“≤ π”边界。 

## 方法

 暴力方法是枚举激活节点的每个子集。 对于每个子集，检查其所有点是否都位于某个半圆内，并对恰好该子集出现的概率求和。 这是正确的，因为激活是独立的，但它需要迭代所有 2^n 个子集，甚至检查每个子集的半圆包含成本为 O(n)，导致 O(n 2^n)，这对于 n 高达 10^5 来说是不可能的。 

观察的关键是倒转观点。 我们不是对子集进行推理，而是按角度固定点的排序，并要求“好”子集，这意味着所有选定的点都位于圆上某个长度为 π 的区间内。 如果我们固定该间隔中的最左边的点，则有效子集正是包含在从该点开始的角宽度 π 的滑动窗口中的子集，并选择该点作为代表锚点。 

在对数组进行排序和复制后，这会将全局循环约束转换为线性窗口约束。 然后，我们使用“未选择”事件的前缀积来累积概率。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 最佳 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们首先按角度对点进行排序。 为了处理循环环绕，我们通过在末尾添加每个角度加上 2π 来复制排序列表。 

对于原始数组中的每个点 i，我们找到最远的点 j，使得角度差 θ[j] − θ[i] 严格小于 π。 这定义了从 i 开始的最大半圆。

接下来，我们将 i 解释为该半圆中最左边的激活点。 所有激活点都位于半圆内的每个有效配置都有一个唯一的表示，其中 i 是该配置中的最小角度激活点。 

我们计算每个 i 的贡献如下。 

1. 将 i 固定为角度顺序中的第一个激活点。 这会产生一个因子 p[i]，因为 i 必须处于活动状态。 
2. 对于严格从 i 到 j 的所有点，每个点 k 必须是不活动的或活动的，但如果是活动的，则它必须保留在窗口内。 然而，由于我们只要求所有活动点都位于窗口内，因此禁止在窗口外进行任何激活。 因此，窗口外的所有点都必须处于非活动状态。 
3.对于i之后的窗口内的点，它们可以自由地处于活动或不活动状态，但我们必须通过确保i是第一个活动点来避免重复计算。 这是通过要求循环顺序中 i 之前的所有点都处于非活动状态来处理的，这自然是通过使用滑动窗口按排序顺序扫描来保证的。 
4. 因此，贡献变为：

 p[i] 乘以 (1 − p[k]) 的 (i, j] 中 k 的乘积，表示窗口内没有较早的点破坏“第一点”结构。

 我们预先计算 (1 − p[i]) 的前缀乘积，以在 O(1) 内评估这些乘积。 

我们对所有 i 求和，并以 998244353 为模。 

### 为什么它有效

 某个半圆中包含的每个有效激活子集都通过选择其最小角度点作为 i 来具有唯一的表示。 该子集对总和仅贡献一次。 滑动窗口确保我们只考虑角跨度小于 π 的子集，并且乘法结构与激活事件的独立性相匹配。 这在有效子集和贡献之间建立了双射，确保了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def main():
    n = int(input())
    pts = []
    angles = []

    for _ in range(n):
        theta, x, y = input().split()
        theta = float(theta)
        x = int(x)
        y = int(y)
        p = x * modinv(y) % MOD
        pts.append((theta, p))
        angles.append(theta)

    pts.sort(key=lambda x: x[0])

    # duplicate for circular handling
    arr = pts + [(theta + 6.283185307179586, p) for theta, p in pts]

    m = len(pts)
    p = [arr[i][1] for i in range(len(arr))]
    ang = [arr[i][0] for i in range(len(arr))]

    j = 0
    res = 0

    pref = [1] * (len(arr) + 1)
    for i in range(len(arr)):
        pref[i + 1] = pref[i] * (1 - p[i]) % MOD

    def range_prod(l, r):
        return pref[r] * modinv(pref[l]) % MOD

    for i in range(m):
        while j < i + m and ang[j] - ang[i] < 3.141592653589793:
            j += 1
        # contribution: p[i] * product of (1-p[k]) for k in (i+1..j-1)
        res = (res + p[i] * range_prod(i + 1, j)) % MOD

    print(res % MOD)

if __name__ == "__main__":
    main()
```该解决方案首先使用逆模算术将所有激活概率转换为模形式。 按角度排序在圆上建立线性顺序，而重复则消除了环绕问题。 

指针 j 从 i 开始扩展最大半圆。 前缀乘积数组允许在任何时间间隔内对乘积进行恒定时间计算，这对于效率至关重要。 

其中一个微妙之处是保持浮点比较和 π 阈值之间的一致性。 由于 θ 的给出毫米精度高达千分之一，因此在这种情况下使用双重比较是安全的。 

## 工作示例

 ### 示例 1

 考虑圆上均匀分布的三个点，每个点的概率为 1/2。 

我们计算贡献：

 | 我| 窗口 [i, j) | 贡献 |
 | --- | --- | --- |
 | 0 | 0..2 | 0..2 p0 · (1-p1)(1-p2) | p0 · (1-p1)(1-p2) |
 | 1 | 1..0（换行处理）| p1 · (1-p2)(1-p0) | p1 · (1-p2)(1-p0) |
 | 2 | 2..1 | 2..1 p2 · (1-p0)(1-p1) | p2 · (1-p0)(1-p1) |

 每一项对应于为半圆中包含的子集选择唯一的起点。 将这些相加得出最终概率。 

这表明每个有效子集只计算一次。 

### 示例 2

 如果所有点确定性地位于半圆内并且所有概率均为 1，则 j 始终覆盖任意 i 的所有点。 

| 我| 窗户尺寸| 贡献 |
 | --- | --- | --- |
 | 0 | 所有点| 1 |
 | 1 | 所有点| 排除逻辑后为 0 |
 | 2 | 所有点| 0 |

 总和变为 1，与中心始终位于覆盖范围之外的事实相匹配。 

这证实了极端确定性情况下的正确性。

    def solve():

                j += 1

 ## 边缘情况

 对于紧密聚集在半圆内的点，该算法始终生成覆盖整个簇的 j，并且该簇内的每个 i 贡献与所有有效子集一致的项。 滑动窗口确保不会计算超出 π 的子集。 

对于围绕整个圆均匀分布的点，对于每个 i，j 保持较小，从而防止包含无效的宽子集。 复制可确保正确处理环绕子集，而不会丢失跨越 0 角的配置。 

对于重复的角度，角度差的严格不平等可以防止在冲突窗口中重复包含相同的边界点，从而保留锚表示的唯一性。
