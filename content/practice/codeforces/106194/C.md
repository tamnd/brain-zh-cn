 ## 解决方案
 ## 问题理解

 我们得到了一系列表示系统中存储的测量值的值。 这些值已被一系列操作破坏。 每个操作都会选择数组的一个连续段，并翻转该段中每个值的符号。 

这些操作被依次应用，我们给出的数组反映了所有翻转发生后的最终损坏状态。 任务是重建在任何这些符号翻转发生之前数组的原始总和。 

数组中的每个位置都会受到一定数量的范围翻转的影响。 如果一个位置被翻转偶数次，它最终相对于原始位置没有变化。 如果翻转奇数次，则最终数组中的符号会反转。 这种奇偶效应就是问题的整个结构。 

约束达到五十万个元素和五十万个操作。 任何涉及每个操作的每个元素的解决方案都会尝试 10^11 次更新，这远远超出了一秒限制所允许的范围。 这立即排除了直接模拟的可能性。 

一个天真的但重要的错误是直接在阵列上模拟每次翻转。 例如，如果我们重复反转段，则每次操作都会花费线性时间。 即使输入适中，也会失败，因为数组大小和操作数量都很大。 

另一个微妙的错误是尝试通过以相反顺序反转操作并重新应用翻转来重建原始数组。 这在逻辑上是正确的，但在计算上与模拟相同，并且不会提高复杂性。 

缺少的关键见解是我们根本不需要知道数组的中间状态。 只有每个索引翻转次数的奇偶性才重要。 

## 方法

 直接模拟方法通过迭代区间中的所有索引并乘以负数来应用每个范围翻转。 这是正确的，因为它准确地反映了操作的定义。 然而，每个操作最多可以触及 n 个元素，因此在最坏的情况下总工作量变为 m 乘以 n。 两者都达到五十万，这太慢了几个数量级。 

改进来自于观察到每个操作仅对每个索引产生计数。 我们不需要重复应用符号更改； 我们只需要知道覆盖物的总数是偶数还是奇数。 这将问题转化为维护一系列增量，然后计算每个索引的最终奇偶校验。 

这是差分数组起作用的典型情况。 每个操作都会将一个段上的覆盖计数器加一。 处理完所有操作后，我们重建每个索引被覆盖的次数。 只有奇偶性才重要，所以我们将计数取模 2。 然后我们相应地调整每个元素的符号。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(纳米) | O(1) 额外 | 太慢了 |
 | 差异阵列（奇偶校验跟踪）| O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们处理所有范围操作的方式使我们能够计算每个索引翻转的次数，而无需显式更新每个查询的每个位置。

1. 我们创建一个初始化为零的差异数组。 该结构将对每个位置开始和结束的翻转操作进行编码。 范围增量通过在左端点加一并在右端点后减一来表示。 
2. 对于每个操作区间 [l, r]，我们在 l 处递增差值数组，并在 r + 1 处递减它。这确保了当我们稍后计算前缀和时，区间中的每个索引恰好收到一个附加计数。 
3. 我们计算差异数组的前缀和。 对于每个位置，这会重建有多少操作包含该索引。 结果是应用于每个元素的符号翻转次数。 
4. 对于每个位置 i，我们检查翻转计数是偶数还是奇数。 如果为偶数，则该值保持最终数组中给定的值。 如果它是奇数，则原始值一定是给定值的负数，因此我们翻转它的符号。 
5. 我们累加所有校正值以计算原始总和。 

这种方法有效的关键原因是每个操作对每个索引都有独立的贡献，并且符号翻转形成一个仅奇偶校验重要的组。 多次翻转成对取消，因此只有总计数是否为奇数才会改变该值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    m = int(input())

    diff = [0] * (n + 2)

    for _ in range(m):
        l, r = map(int, input().split())
        diff[l - 1] += 1
        diff[r] -= 1

    cur = 0
    ans = 0

    for i in range(n):
        cur += diff[i]
        if cur % 2 == 0:
            ans += a[i]
        else:
            ans -= a[i]

    print(ans)

if __name__ == "__main__":
    solve()
```该实现使用偏移到从零开始的索引的差异数组。 每个操作在左边界处添加，并在右边界后减去，以便前缀累积正确地重建覆盖计数。 运行变量 cur 跟踪有多少活动翻转影响当前索引。 

决策步骤完全基于奇偶校验。 如果 cur 是奇数，则该元素必须相对于给定的最终状态反转。 

## 工作示例

 考虑第一个样本：

 我们从一个数组和两个翻转范围开始。 将每个范围转换为差异数组后，我们构建一个前缀和，告诉我们每个位置受到影响的次数。 

| 索引 | 最终值| 翻转计数奇偶校验 | 贡献 |
 | ---| ---| ---| ---|
 | 1 | 1 | 奇数| -1 |
 | 2 | 2 | 奇数| -2 |
 | 3 | -1 | 甚至| -1 |
 | 4 | 5 | 奇数| -5 |
 | 5 | 3 | 奇数| -3 |

 重构后的总和变为-12。 这符合这样的想法：翻转奇数次的索引必须相对于提供的状态进行反转。 

对于第二个样本，多次重叠和重复翻转在多个位置抵消，留下混合奇偶校验图案。 计算前缀覆盖率后，我们得到：

 | 索引 | 最终值| 翻转计数奇偶校验 | 贡献 |
 | ---| ---| ---| ---|
 | 1 | 1 | 甚至| 1 |
 | 2 | -1 | 奇数| 1 |
 | 3 | -4 | 奇数| 4 |
 | 4 | -5 | 奇数| 5 |
 | 5 | 1 | 甚至| 1 |
 | 6 | 4 | 甚至| 4 |

 所得总和为 -4，确认同一索引上的重复翻转仅通过奇偶校验取消才起作用。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n + m) | 每个操作更新两个位置，并且单个前缀传递计算所有翻转计数 |
 | 空间| O(n) | 数组和输入数组存储的区别|

 这些约束允许最多 500000 个元素和操作，因此每个操作的线性时间方法至关重要。 该解决方案每次操作仅执行恒定的工作，并对阵列进行一次扫描，在限制范围内舒适地拟合。

 ## 边缘情况

 应用一次全范围翻转即可演示核心奇偶校验逻辑。 用于输入`n=4, a=[2,2,2,2], m=1, [1,4]`，每个索引的翻转计数为 1。该算法计算所有位置的 cur=1，因此每个值都被取反，最终总和变为 -8。 这符合单一全局反转反转所有贡献的预期。 

无操作情况证实该算法不会引入意外的修改。 和`m=0`，差值数组在任何地方都保持为零，前缀计数都是偶数，并且输出等于数组的原始和。 

单元素边缘情况可确保索引的正确性。 和`n=1`，差异更新变成单个增量和减量，仍然产生正确的前缀奇偶校验。 该算法根据单个范围是否包含唯一值来正确翻转或保留该值。
