 ## 解决方案
 ## 问题理解

 我们得到一个仅由数字组成的字符串，长度最多为 100。任务是确定该字符串是否满足非常特定的对称条件，并输出一个简单的是或否的结论。 

我们正在寻找的结构纯粹是关于从左到右读取的字符与从右到左读取的字符的比较。 没有数字的算术解释，不需要解析数字或执行转换。 整个问题归结为将字符序列作为静态对象进行推理。 

由于最大长度仅为 100，因此任何直接检查字符的解决方案都已经足够高效。 即使重复扫描字符串的方法仍然会在相对于典型的竞争性编程限制的恒定时间内运行。 这立即排除了对性能优化的担忧，并将注意力完全转移到正确性和处理边界情况上。 

主要的微妙之处在于识别非常小的输入的边缘行为。 单个字符串的行为与多字符字符串不同，因为对称性变得微不足道，而双字符字符串可能满足也可能不满足所需条件，具体取决于两端是否匹配。 

这里的一个天真的错误是假设基于部分观察的属性，例如仅检查第一个和最后一个字符而不验证完整的结构。 例如，输入`343`会错误地通过首尾检查，但仍然需要完整的验证以确保完全对称。 

## 方法

 最直接的方法是检查字符串向前和向后读取是否相同。 这意味着比较对称位置：第一个字符与最后一个字符，第二个字符与倒数第二个字符，依此类推，直到到达中间。 

考虑这个问题的一种强力方法是反转字符串并将其与原始字符串进行比较。 这是有效的，因为反转将所有对称约束编码为单个对象比较。 成本与字符串长度成线性关系，因为构造反转字符串并比较它都需要 O(n) 时间。 

考虑到长度最多为 100，即使重复比较或简单扫描也可以忽略不计。 然而，如果我们概括这个想法，关键的结构见解是回文检查不需要存储整个反向字符串。 相反，我们可以直接比较镜像索引，这样可以避免额外的分配，并使逻辑更清晰。 

暴力破解之所以有效，是因为它明确地构造了反向表示并检查相等性。 当我们意识到比较是成对独立的时，它就变成了不必要的开销，一旦发现不匹配就允许提前终止。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 反转并比较 | O(n) | O(n) | 已接受 |
 | 两指针检查 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们重点关注两指针方法，因为它最直接地表达了问题的结构。 

1. 从两个索引开始，一个位于字符串的开头，一个位于字符串的末尾。 这些表示字符串必须匹配才能保持对称的字符。 如果它们在任何一点不同，对称性就会立即被打破。 
2. 比较两个索引处的字符。 如果它们不相等，我们可以停下来并得出字符串不对称的结论。 这种提前退出是有效的，因为单个不匹配违反了全局条件。 
3. 如果它们匹配，则向内移动两个指针，使左索引向前移动，右索引向后移动。 这确保我们逐步验证所有镜像位置而不重复。 
4. 重复此过程，直到指针交叉或相遇。 如果我们完成所有比较而没有发现不匹配，则该字符串满足对称性要求。 

此过程背后的原因是每个有效的对称字符串必须同时满足所有镜像位置的相等性。 独立检查它们可以保证完整性，因为每次比较都会强制执行全局结构的一个必要条件。 

### 为什么它有效

 该算法保持了当前指针范围之外的所有字符对都已被验证匹配的不变性。 每个步骤都会减少字符串中未经验证的部分，同时保持正确性。 如果任何地方存在不匹配，则它一定发生在某个镜像对中，并且最终将直接检查该镜像对。 因此，该算法不能错过违规，并且如果它完成时没有找到违规，则该字符串必须是对称的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

i, j = 0, len(s) - 1
ok = True

while i < j:
    if s[i] != s[j]:
        ok = False
        break
    i += 1
    j -= 1

print("YES" if ok else "NO")
```该代码读取输入字符串并在其两端初始化两个指针。 该循环强制执行前面描述的镜像比较逻辑。 一旦发现不匹配，我们就会提前退出，因为进一步的检查无法修复对称性。 

一个微妙的实现细节是使用`strip()`以确保尾随换行符不会干扰索引。 另一个重要的点是处理字符串长度为 1 的情况，在这种情况下循环永远不会运行并且结果正确地默认为`YES`。

 ## 边缘情况

 单个数字输入，例如`7`是最简单的场景。 算法集`i = 0`和`j = 0`，因此循环不执行。 结果依然`YES`，这是正确的，因为根据定义任何单个字符串都是对称的。 

对于两个字符不匹配，例如`90`，第一个比较`s[0]`与`s[1]`立即失败。 算法正确输出`NO`无需进行不必要的检查。 

对于更长的对称输入，例如`1221`，比较进行如下`(1,1)`和`(2,2)`处于镜像位置。 所有检查均成功，确认算法正确地将局部相等性聚合为全局对称性。
