 ## 解决方案
 ## 问题理解

 我们有一个大小为的正方形网格$n \times n$。 第一行是固定的：它包含从 1 到$n$为了。 下面的每个单元格都是确定性生成的：每个条目都是同一列中其正上方数字的平方。 

所以如果我们看一列$j$，价值观就这样演变。 顶部是$j$，下一个单元格是$j^2$， 然后$j^4$， 然后$j^8$， 等等。 每一步都是前一个值的平方，这意味着指数每行都加倍。 

任务不是构建网格，而是计算所有行和列中任意位置出现的不同整数的数量。 

约束条件达到$n = 10^5$和$t = 10^5$。 任何显式模拟网格的解决方案都是不可能的，因为即使存储网格也是如此$O(n^2)$，甚至单个测试也太大了。 这立即推动我们按照值结构而不是每个单元进行推理。 

一个微妙的问题是价值增长得非常快。 即使对于小$j$，数字在第三行或第四行时爆炸超出正常整数范围。 然而，问题在于独特性，而不是大小，因此溢出在 Python 中无关紧要，但对于直觉来说仍然很重要。 

一些边缘情况值得隔离。 

什么时候$n = 1$，网格完全是 1，所以答案显然是 1。 

当$n = 2$，网格包含$1,2$在第一行，然后$1,4$。 不同的值是$\{1,2,4\}$。 天真的解释可能会忘记这一点$4$不会与第一行中的任何物体发生碰撞，但在较大的情况下，来自完美幂的碰撞成为中心问题。 

关键的困难在于网格中的不同路径可以以不同的方式生成相同的数字，例如$16 = 4^2 = 2^4$，所以我们必须避免重复计算相同的整数。 

## 方法

 网格结构意味着每个值都具有以下形式$$j^{2^k}$$在哪里$j$是列索引并且$k$是从零开始的行索引。 

暴力方法将显式地为所有人生成所有值$j \le n$和所有行，直到数字变得太大。 这会产生$n^2$值，然后我们将它们插入到一个集合中。 这在原则上是正确的，但立即就被打破了，因为$n^2$达到$10^{10}$，这远远超出了可行的计算范围。 

关键的观察是网格不会生成任意数字。 它只产生完美的幂，更具体地说，指数是 2 的幂。 因此，我们可以根据集合来思考，而不是根据网格中的位置来思考$$\{ j^{1}, j^{2}, j^{4}, j^{8}, \dots \}$$对于每个$j$。 

现在问题变成：计算所有完美数字中的不同整数$k$-th 次幂，其中$k$是二的任意幂。 

这种重新制定完全删除了网格。 我们只需要生成表格的所有数字$a^k$在哪里$1 \le a \le n$和$k \in \{1,2,4,8,\dots\}$，然后计算唯一值。 

仅当相同的数字可表示为不同的指数模式时才会出现重复项，例如$16 = 2^4 = 4^2$。 使用全局集自然可以解决这个问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力网格模拟 |$O(n^2)$|$O(n^2)$| 太慢了 |
 | 枚举全部$a^k$具有二次幂 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们间接构造所有有效的网格值。 

1. 从包含从 1 到 1 的所有整数的集合开始$n$。 这些对应于网格的第一行，它贡献每一列值而不进行任何平方。 
2. 预先计算网格中出现的所有指数。 这些都是$2^0, 2^1, 2^2, \dots$。 我们跳过$2^0 = 1$因为它已经被计入第一行，并且只考虑从2开始的指数。 
3. 对于每个指数$k$在此序列中，迭代所有基值$a$从 1 到$n$, 计算$a^k$，并将结果插入到集合中。 这代表了网格的每个更深行级别的贡献。 
4. 继续下去，直到指数变得太大而无法使用。 实际上，这个序列非常短，因为它每次都会加倍，因此在任何合理的范围内只有大约 16 到 17 个相关指数。 
5. 答案很简单，就是集合的大小。 

这种构造起作用的原因是网格中的每个单元格都是这些指数值之一，并且每个这样的值在每个（基数，指数级）对中恰好生成一次。 该集合消除了由产生相同整数的不同指数路径引起的所有冲突。 

### 为什么它有效

 每个网格值唯一对应于一对$(j, 2^k)$，表示一个基柱和多个方形台阶。 该值始终是$j^{2^k}$。 相反，在此过程中生成的任何数字都会出现在网格中的某个位置，因为列$j$产生的正是那个权力序列。 因此，所有列和所有有效指数的并集与网格值集完全匹配。 唯一剩下的问题是同一整数的不同表示形式之间的重复，这可以通过将所有内容存储在一个集合中来解决。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())

        seen = set()

        for a in range(1, n + 1):
            seen.add(a)

        k = 2
        # exponents are powers of two: 2, 4, 8, ...
        # k grows fast, so only a small number of iterations exist in practice
        while k <= 60:
            for a in range(1, n + 1):
                val = pow(a, k)
                seen.add(val)
            k *= 2

        print(len(seen))

if __name__ == "__main__":
    solve()
```该解决方案首先插入整个第一行，其中包含指数为 1 的所有值。然后，它迭代与重复平方相对应的指数级别：2、4、8 等。 对于每个级别，它都会生成所有值$a^k$为了$a \le n$。 这`set`确保像这样的碰撞$16 = 2^4 = 4^2$只计算一次。 

指数循环的上限为一个小常数，因为指数每次都会加倍，并且很快就与在实践中产生新的不同值变得无关。

 ### 示例 1

 让$n = 3$。 

第一行给出：$1, 2, 3$。 

指数 2 加上：$1, 4, 9$。 

指数 4 增加：$1, 16, 81$。 

| 步骤| 指数 k | 产生的价值| 设置尺寸|
 | ---| ---| ---| ---|
 | 初始化| - | {1,2,3} | 3 |
 | 1 | 2 | {1,4,9} | 5 |
 | 2 | 4 | {1,16,81} | 7 |

 最终设定是$\{1,2,3,4,9,16,81\}$，所以答案是 7。 

该迹线显示了新值如何仅来自更高的幂，以及重复 1 之类的冲突如何不会导致计数膨胀。

 ## 边缘情况

 对于$n = 1$，无论指数级别如何，每个单元格都是 1。 该算法在初始行中插入 1 一次，所有后续求幂也产生 1，但集合将其保留为单个元素，从而产生正确的输出 1。 

对于小$n$，例如 2 或 3，许多更高的幂会折叠成先前行中已经存在的值或在不同的指数级别上重复。 例如，$16$可以同时显示为$2^4$和$4^2$，但该集合确保它被计数一次。 

对于较大的$n$，大多数高指数值都非常大并且与之前的指数值不同。 即使存在重复项，它们也只能通过完美幂的代数恒等式出现，这正是基于集合的构造旨在处理的情况。
