 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例都描述一棵树，即一个连接的非循环图。 任务是确定这棵树上是否存在至少访问每个节点一次但最多访问两次的游走。 允许步行从任何节点开始，并且起始节点被认为在时间零时已经被访问过。 步行的每一步都必须沿着边缘移动。 

因此，我们不会被要求构建步行道，而只是确定这样的步行道是否存在。 

重新解释条件的一个关键方法是考虑遍历过程中每个节点进入的次数。 由于图是一棵树，任何重复的访问都来自于沿边的回溯。 “每个节点最多两次”的约束非常严格：它禁止在多次离开子树后重新访问顶点，这立即表明只有非常有限的树形状才能工作。 

约束条件很大：所有测试用例的节点总数最多可达 200,000 个。 这排除了任何模拟行走、尝试所有起点或执行指数回溯的解决方案。 即使每个测试用例采用二次方法也会太慢。 我们需要每个测试用例都是线性的，或者最坏的情况是所有测试都是线性的。 

一个天真的错误是假设任何树都有效，因为我们总是可以进行 DFS 欧拉之旅。 这是错误的，因为在标准 DFS 遍历中，某些节点可以被访问两次以上。 例如，在一棵星形树中，在一次完整的遍历中，中心会被访问多次。 

另一个常见的误解是认为我们总是可以选择哈密顿路径并稍微扩展它。 但树通常有分支，迫使通过中心节点重复返回。 

一个打破天真的直觉的小例子是一颗星星：

 输入：

 n = 4

 1 连接到 2、3、4

 从叶子开始的简单 DFS 遍历将是：

 2 → 1 → 3 → 1 → 4

 节点 1 被访问了 3 次，违反了条件。 这里的正确答案是否定的。 

因此，问题在于树的结构限制，允许以非常有限的重访进行遍历。 

## 方法

 暴力的想法是模拟从每个节点开始的所有可能的行走，探索所有路径并跟踪访问计数。 由于每个步骤都可以重新访问节点，并且除了隐式约束之外，我们对行走长度没有限制，因此这成为一个巨大的状态空间问题。 即使我们将行走长度限制为 2n 之类的值，分支选择也会呈指数级增长，因为在每个节点，我们可以前往除前一个节点之外的任何邻居。 

这会失败，因为状态必须包含当前节点和所有节点上的访问计数向量，而该向量太大了。 

关键的见解是将视角从“构造一条路径”转变为“如果每个节点最多被访问两次，树会施加什么结构约束”。 

在树上行走本质上是一个边遍历序列。 每次我们进入一个节点时，我们要么继续深入，要么最终通过同一条边返回。 保持低访问量的唯一方法是确保树不会强制重复“中心交叉”。 

关键的观察是，如果一个节点的度数为 3 或更高，那么覆盖所有分支的任何遍历都必须以超出允许限制的方式多次经过该节点。 更准确地说，度数≥3的节点至少创建三个不相交的子树，并且覆盖所有子树的任何遍历都必须多次返回到该节点。 人们可以形式化地认为，这样的节点在任何覆盖步行中至少强制进行三次访问。

这立即表明这棵树一定非常像路径。 事实上，唯一有效的树要么是简单路径，要么是“控制一个额外分支点的路径，以便重新访问不超过两个”。 仔细推理表明，该条件简化为检查树是路径还是近路径结构，其中最多两个节点的度数大于 2 并且形成单链排列。 然而，更稳健和标准的推导显示了更清晰的特征：当且仅当树是简单路径时，答案是“是”。 

为什么？ 因为任何分支点至少引入三个方向，并且为了覆盖所有叶子，步行必须至少重新访问分支节点的次数等于分支数量减一的次数，这立即超出了允许的限制。 

所以我们只需要检查每个节点的度数是否最多为 2，并且树是否是连通的，它总是连通的。 这意味着树最多只能有两个 1 度节点，其他所有节点都是 2 度，这正是一条路径。 

因此，解决方案减少为检查树是否是路径图。

 | ---| ---| ---| ---|
 | 蛮力步行模拟| 指数| O(n) | 太慢了 |

 ## 算法演练

 1. 读取每个测试用例的树并计算每个节点的度。 这捕获了本地分支结构，该结构决定是否可以发生强制重访。 
2. 计算有多少个节点的度数恰好为 1。这些是树的叶子。 在一条路径中，必须恰好有两片叶子。 
3. 检查所有节点以确保没有一个节点的度数大于 2。任何此类节点都会引入分支，从而在任何覆盖行走期间强制多次返回。 
4. 如果叶子数正好为 2，且没有节点的度数大于 2，则输出 YES，否则输出 NO。 

这样做的原因是所有度数最多为 2 的树必然是一条路径。 路径的端点正是度数为 1 的节点。任何与此结构的偏差都会引入一个分支点，迫使重新访问超出允许的限制。 

### 为什么它有效

 在每个节点的度数最多为 2 的树中，只有一条简单的链连接所有节点。 从一个端点开始的任何遍历都可以遍历链到另一个端点，只访问每个节点一次。 这立即满足约束。 

如果任何节点的度数至少为 3，则该节点将树拆分为多个分支。 为了覆盖单个连接遍历中的所有分支，遍历必须重复返回该节点，从而将其访问计数增加到超过两次。 因此，这样的配置在约束下是不可能的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        deg = [0] * (n + 1)
        
        for _ in range(n - 1):
            u, v = map(int, input().split())
            deg[u] += 1
            deg[v] += 1
        
        leaves = 0
        ok = True
        
        for i in range(1, n + 1):
            if deg[i] > 2:
                ok = False
            if deg[i] == 1:
                leaves += 1
        
        if n == 2:
            print("YES")
        elif ok and leaves == 2:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```该实现是结构条件的直接转换。 度数组捕获邻接信息。 边缘上的循环构建了树。 验证步骤检查两个约束：没有超出 2 级的分支并且恰好有两个端点。 

特殊情况 n = 2 隐式是一条路径，逻辑已经处理了它，但为了清晰起见，它保持显式，因为两个节点的度数都是 1 并且满足条件。

 ## 边缘情况

 关键的边缘情况是最小的有效树，n = 2。该算法将两个节点都计为叶子，并且看不到度数大于 2，因此它输出 YES，与约束内单条边可轻松遍历的事实相匹配。 

另一个重要的情况是一棵树几乎是一条路径，但有一个额外的分支，例如 1-2-3-4 和一个额外的边 2-5。 这里节点 2 的度数为 3。算法在度数检查阶段立即拒绝它。 任何有效的遍历都需要经过 1-2-3-2-4-2-5 或类似的路径，迫使节点 2 至少被访问 3 次。 

最后一个微妙的情况是一条长链，在端点附近附有一个额外的叶子。 尽管在视觉上它类似于一条路径，但额外的叶子创建了一个 3 度节点，并且适用相同的推理。 该算法在本地检测，无需模拟任何行走结构。
