 ## 解决方案
 ## 问题理解

 我们得到一个数组并允许最多执行一次单个操作。 该操作选择一个连续的段，计算该段的 mex，然后用该 mex 值覆盖该段内的每个元素。 在不执行任何操作或仅执行一次此类覆盖之后，我们查看整个数组的 mex 并希望将其最大化。 

因此，任务不是直接优化段，而是了解仔细选择的子数组“压缩”如何增加全局 mex。 

数组的 mex 是其中缺失的最小非负整数，因此增加 mex 意味着我们试图让尽可能多的小整数出现，同时确保第一个缺失的整数被尽可能推到最右边。 

这些约束允许数组大小最大为 10^6，这会立即排除尝试所有段的任何解决方案。 即使具有大量常量的 O(n log n) 也是可以接受的，但任何分段上的二次方都是不可能的。 

当考虑手术是否只会有所帮助或有时会造成伤害时，就会出现一个微妙的边缘情况。 例如，如果我们选择一个已经有 mex 0 的段，我们会用 0 覆盖它，这可能会破坏现有结构并降低全局 mex。 另一个问题是假设最好的片段总是与原始的 mex 直接相关，如果没有关于 mex 如何在全球范围内变化的结构性论证，这是不正确的。 

## 方法

 强力策略会尝试每对索引 l 和 r，计算该子数组的 mex，应用覆盖，重新计算完整数组 mex，并跟踪最佳结果。 天真地计算每个段的 mex 成本已经是 O(n)，并且有 O(n^2) 个段，所以这变成了 O(n^3)，即使 n 约为 5000，这也是完全不可行的。 

即使通过频率重置将每个段的 mex 计算改进为摊销 O(1) 也无法挽救这种方法，因为迭代所有段仍然占主导地位。 

关键的结构观察是整个数组的 mex 完全取决于从 0 开始的所有值是否出现在某个地方。 该操作仅将一个连续区域更改为恒定值，因此改进 mex 的唯一方法是帮助确保小值仍然存在，同时引入新的缺失值。 

令 m0 为原始数组的 mex。 从 0 到 m0 − 1 的所有值至少出现一次，而 m0 根本不出现。 

如果我们想在一次操作后改进 mex，唯一的候选改进是使 m0 出现在数组中（因为它当前缺失），同时保留所有值 0 到 m0 − 1 都存在。 如果我们成功了，新的 mex 就会变成 m0 + 1。我们不能跳得更远，因为我们最多引入一个新的统一值。 

因此，问题简化为检查是否存在这样的段：

 替换时，该段生成等于 m0 的 mex，并且替换不会破坏 0 到 m0 − 1 中任何值的所有出现。 

这变成了每个值出现位置的纯粹位置条件。 

我们为每个值 v 定义其在数组中的第一次和最后一次出现。 为了确保替换后从 0 到 m0 − 1 的每个值仍然存在，所选段不得完全覆盖任何此类 v 的所有出现。同时，为了确保段的 mex 恰好变为 m0，该段必须包含从 0 到 m0 − 1 的每个值至少出现一次。 

仅当存在与值 0 到 m0 - 1 的每个出现间隔相交的单个线段时，这才是可能的。这成为间隔相交问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对段进行暴力破解 | O(n^3) | O(n^3) | O(n) | 太慢了 |
 | 区间交叉检查| O(n) | O(n) | 已接受 |

 ## 算法演练

### 1. 计算原始 mex

 我们扫描数组并标记出现的值。 未见过的最小值是m0。 

这给出了基线：值 0 到 m0 − 1 都保证存在。 

### 2.记录出现边界

 对于 [0, m0 − 1] 中的每个值 v，我们计算它在数组中的第一次和最后一次出现。 

这些边界描述了每个值“存在”的完整区域。 

### 3.建立交集条件

 我们想要一个线段 [l, r] 至少包含 0 到 m0 − 1 中每个值的出现一次。这意味着对于每个这样的值 v，其区间 [Lv, Rv] 必须与 [l, r] 相交。 

这相当于要求至少存在一个索引位置同时位于所有这些区间内。 

所以我们计算全局交集：

 对于所有 v < m0，我们取 L = max(Lv) 和 R = min(Rv)。 

### 4. 决定可行性

 如果 L ≤ R，则存在位于每个出现区间内的位置。 我们可以选择一个恰好包含该位置的线段，确保它与所有所需的值相交，同时不会以破坏性的方式完全覆盖任何区间。 

如果 L > R，则没有任何单个线段可以同时与所有这些间隔相交，因此我们不能强制进行 mex 改进。 

### 为什么它有效

 关键的不变量是每个值 v < m0 必须在操作中幸存下来，这意味着 v 的至少一个出现必须保留在所选段之外。 同时，为了使 m0 在操作后出现，该段必须将其生成为它的 mex，这要求该段“看到”从 0 到 m0 − 1 的每个值至少一次。 

这会产生一种紧张气氛，并最终陷入单一条件：所有这些值区间必须在至少一个共同指数中重叠。 如果存在这样的点，我们可以将线段锚定在它周围； 否则，任何段都必然会错过某些所需的值或消除另一个段的所有出现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    seen = [False] * (n + 2)
    for x in a:
        if x <= n + 1:
            seen[x] = True

    mex0 = 0
    while seen[mex0]:
        mex0 += 1

    if mex0 == 0:
        print(1)
        return

    INF = 10**18
    L = INF
    R = -INF

    first = [-1] * (mex0)
    last = [-1] * (mex0)

    for i, x in enumerate(a):
        if x < mex0:
            if first[x] == -1:
                first[x] = i
            last[x] = i

    for v in range(mex0):
        L = min(L, first[v])
        R = max(R, last[v])

    print(mex0 + 1 if L <= R else mex0)

if __name__ == "__main__":
    solve()
```该解决方案首先通过标记所有当前值来计算原始 mex。 然后，它记录所有小于该 mex 的值的第一次和最后一次出现。 这些边界定义间隔，其重叠决定单个段是否可以同时与所有所需值交互。 

最终对比`L <= R`是整个决策点：它检查所有这些区间是否共享至少一个公共索引。 

## 工作示例

 ### 示例 1

 输入：```
5
0 1 2 4 3
```这里 mex 是 5，因为 0-4 都出现了。 

我们计算间隔：

 0：[0,0]、1：[1,1]、2：[2,2]、3：[4,4]、4：[3,3]

 现在我们将它们相交：

 L = 最大值(0,1,2,4,3) = 4

 R = min(0,1,2,4,3) = 0

 | 步骤| 左 | 右 |
 | ---| ---| ---|
 | 开始| 0 | 0 |
 | 处理后间隔| 4 | 0 |

 由于 L > R，因此不存在公共点。 

所以答案是 mex0 = 5。 

这显示了值过于分散而无法同时保留在一个段内的情况。 

### 示例 2

 输入：```
4
1 0 2 1
```墨西哥是 3。 

间隔：

 0：[1,1]

 1：[0,3]

 2：[2,2]

 交叉口：

 L = 最大值(1,0,2) = 2

 R = min(1,3,2) = 1

 | 步骤| 左 | 右 |
 | ---| ---| ---|
 | 开始| 0 | 3 |
 | 处理后间隔| 2 | 1 |

 由于 L > R，答案是 mex0 = 3。 

这表明，即使值存在，它们也不会在涉及所有所需出现的单个位置上重叠。

 ## 边缘情况

 一个关键的边缘情况是原始 mex 为 0 时。在这种情况下，0 不会出现在任何地方，因此任何段都有 mex 0 并且替换它无助于引入丢失的结构。 该算法通过返回 1 直接处理此问题，因为我们始终可以通过选择仅包含非零值的任何段来引入 0。 

另一种边缘情况是当所有值已经从 0 到 n − 1 连续出现时。在这种情况下，mex 是 n，并且 0 到 mex − 1 之间不存在违反条件的值。 交集条件一般成立，算法正确返回 n + 1。 

第三种微妙的情况是所需值的出现极其分散。 间隔交集检查精确地捕获了这一点：即使每个值都存在，如果它们的出现间隔不重叠，则没有单个段可以同时包含它们全部，因此不可能进行改进。
