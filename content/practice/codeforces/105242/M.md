 ## 解决方案
 ## 问题理解

 我们得到了一些糖果，称为 Zingers，最初由 Kaito 持有。 在进行任何分发之前，一个名叫 Taim 的角色被允许秘密接受`k`辛格斯为自己。 这次盗窃之后，剩余的 Zinger 将根据仅取决于剩余总数的固定规则分配给三个人。 

分布规则是确定性的。 如果剩下的数字能被3整除，则由三人平分。 如果不能被 3 整除但能被 2 整除，Taim 将获得剩余 Zingers 的一半，而另一半则分配给其他两名成员。 否则，泰姆将获得一切。 

我们想要选择 Zingers Taim 抢断的数量，介于零和之间`k`，以最大化他的最终总数，即他窃取的东西加上他从剩余的分配中获得的东西的总和。 

约束条件允许`n`最多 10^9 和最多 1000 个测试用例。 这立即排除了任何直接尝试所有可能的盗窃金额的方法，因为每个测试用例对最多 10^9 值进行线性扫描是不可能的。 即使每个测试用例的 O(n) 模拟也太慢了。 

窃取和可分性之间的相互作用产生了一种微妙的边缘情况。 例如，如果`n = 6`，偷 1 得到剩余 5，这属于“否则”情况，即 Taim 得到一切，但偷 2 得到剩余 4，触发可被 2 整除的规则。 这些不连续性意味着答案不是以明显的方式单调，所以贪婪的直觉`k`不直接起作用。 

## 方法

 蛮力策略很简单：尝试每一个可能的值`x`从`0`到`k`，计算剩余的`n - x`，应用分布规则，并添加`x`回到Taim的分享。 这正确地探索了所有可能性，但它的性能高达`k + 1`每个测试用例的评估次数，在最坏的情况下约为 10^9 次操作，远远超出了可接受的限制。 

关键的观察是，最终的表达式可以重新组织，以将窃取的影响与分布的结构分开。 如果我们让`m = n - x`，那么 Taim 的总数就变成`x + f(m)`，即`n - m + f(m)`。 这可以重写为`n + (f(m) - m)`。 期限`n`是常数，所以问题归结为选择`m`在范围内`[n - k, n]`最大化`f(m) - m`。 

现在结构变得简单多了。 功能`f(m)`仅取决于 2 和 3 的整除性，因此它完全由下式决定`m mod 6`。 这意味着`f(m) - m`只有六种可能的模式。 不必扫描区间内的所有值，只需检查最佳候选值就足够了`m`在每个残基类模 6 的有效范围内。这将问题减少到每个测试用例的评估数量恒定。

 | ---| ---| ---| ---|

 | 模块化候选优化 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们根据窃取后的剩余金额重写问题，因为这隔离了过程中唯一的非线性部分。 

1. 将决策变量从“Taim 偷了多少”转换为“偷完后还剩下多少”。 如果泰姆偷窃`x`，那么剩下的就是`m = n - x`，Taim 的总数变为`x + f(m)`。 这种转变很有用，因为`x`和`m`是直接链接的，所以我们可以优化`m`而不是枚举偷窃。 
2. 将目标重写为`n + (f(m) - m)`。 此步骤至关重要，因为它完全消除了非线性部分对窃取变量的依赖。 自从`n`是固定的，最大化总数相当于最大化`f(m) - m`。 
3. 限制`m`到间隔`[n - k, n]`。 这直接来自于Taim最多可以偷窃的限制`k`，所以剩余金额不能低于`n - k`。 
4.观察`f(m)`仅取决于是否`m`能被 3 整除，能被 2 整除，或者都不能被整除。 这意味着`f(m)`完全由`m mod 6`，所以表达式`f(m) - m`在大小为 6 的块上具有重复结构。 
5. 对于每个残留类别`r`在`{0, 1, 2, 3, 4, 5}`，找到最大的`m`在`[n - k, n]`这样`m % 6 = r`。 评价`f(m) - m`仅适用于那些候选人。 
6. 取所有有效候选值中的最大值并加回`n`以获得最终答案。 

正确性来自于以下事实：在每个残基类别中，`f(m) - m`直到线性平移为止都是恒定的`m`，因此类别中的最佳值总是出现在最大有效值处`m`那个班级的。 

## 为什么它有效

 该转换将优化分离为以下函数`m`具有固定的域间隔。 关键的不变量是每一个有效的策略都对应于一个值`m`在`[n - k, n]`，以及每一个这样的`m`正好对应一个有效的盗窃金额。 因为目标分解为一个常数加上一个函数`m`，最大化抢断次数相当于最大化此间隔。 

的周期结构`f(m)`确保在每个模类中，目标的行为可预测，并且在六个剩余情况之外不存在隐藏的局部不规则性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def f(m):
    if m % 3 == 0:
        return m // 3
    if m % 2 == 0:
        return m // 2
    return m

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n, k = map(int, input().split())
        
        L = n - k
        R = n
        
        best = -10**30
        
        for r in range(6):
            m = R - (R - r) % 6
            if m < L:
                continue
            val = f(m) - m
            if val > best:
                best = val
        
        ans = n + best
        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```其实现直接遵循模块化缩减思想。 辅助函数`f(m)`完全按照规定对分配规则进行编码。 对于每个测试用例，代码仅搜索六个候选值，每个残基类一个，选择最大的有效值`m`在允许的区间内。 最终答案加回常数`n`。 

一个常见的错误是迭代所有残基，但未能将候选者限制在区间内`[n - k, n]`。 另一个微妙的问题是忘记了目标取决于`f(m) - m`，不仅仅是`f(m)`本身。

 考虑`n = 10, k = 2`。

 ## 边缘情况

 当`k = 0`，该算法简化为仅评估`m = n`。 候选生成仍然产生六个残基，但只有一个位于有效范围内，因此无需任何特殊处理即可正确计算最大值。 

什么时候`n`能被 3 整除，有多个值`m`区间内可以满足均分规则。 模块化扫描确保仍然检查所有残留类别，并正确选择其中最好的类别。 

什么时候`k`非常大，有潜力`k = n`，区间变为`[0, n]`。 候选构造仍然只考虑六个边界对齐点，并且由于每个最优`m`必须属于一个残留类别，才能保证将其包含在扫描中。
