 ## 解决方案
 ## 问题理解

 我们给出一串小写英文字母和许多查询，每个查询集中于由索引定义的子字符串$l$和$r$。 对于每个这样的范围，我们首先查看其中出现的不同字符。 假设子串$s[l..r]$包含$k$不同的字母。 在所有可能的子串中完全包含在里面$[l,r]$，我们想要仍然包含所有这些的最短的$k$至少一次不同的字符。 每个查询的输出是最短有效子字符串的长度。 

所以任务不是计算频率或重新排列字符。 它是在一个固定段内找到仍然“覆盖”该段中出现的所有字符类型的最小窗口。 

限制条件$n, q \le 10^5$立即排除任何根据查询从头开始重新计算信息的解决方案。 即使每个查询进行线性扫描也会导致$10^{10}$在最坏的情况下进行操作，这远远超出了限制。 唯一可行的方向是预处理字符串，然后使用紧凑的每字符结构回答每个查询，理想情况下每个查询的成本接近不同字符的数量（最多 26 个）。 

当字符分布不均匀时，会出现微妙的边缘情况。 例如，如果一个字符仅在查询的左边缘附近出现一次，而另一个字符仅在右边缘附近出现一次，则最佳窗口可能被迫包含这两个极端。 始终独立地获取第一次出现或最后一次出现的幼稚策略可能会失败，因为最佳解决方案可能会混合不同字符的不同出现。 

另一种失败模式是假设最佳子字符串必须从所有必需字符中最左边的出现位置开始。 这是不正确的，因为向右移动起点有时会显着缩小结束边界。 

## 方法

 直接的暴力解决方案将枚举每个子字符串$[i,j]$里面$[l,r]$，计算其不同字符集，并将其与中的字符集进行比较$s[l..r]$。 对于每个查询，有$O((r-l+1)^2)$子串，并且计算不同计数至少需要$O(1)$或者$O(26)$。 这会导致整个输入的立方行为，即使对于小实例也是不可行的，并且完全破坏了$10^5$。 

关键的观察结果是，感兴趣的身份只是查询范围中不同字符的集合。 一旦知道了该集合，问题就变成了：在范围内选择每个字符的一次出现，以使最早和最新选择的出现之间的跨度最小化。 

这将问题转化为经典的“覆盖 k 个排序列表的最小范围”结构。 每个角色都会贡献一个其位置的排序列表。 对于给定的查询，我们将每个列表限制为以下位置：$[l,r]$，并且我们希望从每个列表中精确挑选一个元素，以最小化最大和最小选择位置之间的差异。 

由于只有 26 个字母，k 很小。 这使得维护一个小的活动指针集（每个字符一个）变得可行，并使用堆迭代地调整它们以始终移动当前贡献最差边界的指针。 每次调整都是局部引导的，但共同收敛到最佳覆盖范围。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$每个查询|$O(1)$| 太慢了 |
 | 最优（26 个列表 + 堆指针）|$O(26 \log 26)$按查询摊销 |$O(n)$| 已接受 |

 ## 算法演练

 我们首先通过存储每个字符来预处理字符串$c$，它出现的位置的排序列表。 

对于每个查询$[l,r]$，我们只考虑实际出现在该片段中的角色，因为其他角色无关紧要。 

1.对于每个角色$c$，找到它在里面第一次出现的位置$[l,r]$在其位置列表上使用二分搜索。 如果不存在，则跳过它，因为它不是所需的不同集的一部分。 
2. 在第一个有效位置处为每个活动字符列表初始化一个指针。 这些指针代表每个字符所选择的一次出现。 
3. 维护一个可以检索所选位置中当前最小值和最大值的数据结构，通常是与当前最大值跟踪配对的最小堆。 
4. 使用这些选定的位置计算初始窗口长度。 
5. 反复识别当前选择位置是所有选择位置中最小的字符，并将其指针前进到内部的下一个出现位置$[l,r]$。 移动后，更新当前的最小值和最大值，并重新计算候选窗口长度。 
6. 当任何指针移出范围或到达其列表段的末尾时停止，因为不存在进一步的有效覆盖。 

答案是在此过程中观察到的最小窗口长度。 

### 为什么它有效

 在任何时刻，我们都会对每个所需字符进行一次选择，形成有效的覆盖集。 每个有效的解决方案都对应于范围内每个字符出现一次的某种选择。 该算法通过在每个列表内单调前进指针来探索可到达的所有相关配置。 对当前范围的任何改进都必须来自于向内移动至少一个端点，这对应于前进当前定义边界的字符的指针。 由于每个候选配置都可以通过这样的指针移动来到达，并且我们总是跟踪最佳可见范围，所以找到的最小值是最佳覆盖窗口。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from bisect import bisect_left, bisect_right
import heapq

def solve():
    n = int(input())
    s = input().strip()

    pos = [[] for _ in range(26)]
    for i, ch in enumerate(s):
        pos[ord(ch) - 97].append(i)

    q = int(input())
    out = []

    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1

        lists = []
        for c in range(26):
            arr = pos[c]
            if not arr:
                continue
            i = bisect_left(arr, l)
            if i < len(arr) and arr[i] <= r:
                lists.append((c, arr, i))

        k = len(lists)
        if k == 0:
            out.append(0)
            continue

        ptr = [i for _, _, i in lists]

        heap = []
        current_max = -10**18

        for idx, (c, arr, i) in enumerate(lists):
            val = arr[i]
            heapq.heappush(heap, (val, idx))
            if val > current_max:
                current_max = val

        best = current_max - heap[0][0] + 1

        while True:
            mn_val, mn_idx = heapq.heappop(heap)
            c, arr, i = lists[mn_idx]
            ni = ptr[mn_idx] + 1
            if ni >= len(arr) or arr[ni] > r:
                break

            ptr[mn_idx] = ni
            new_val = arr[ni]
            heapq.heappush(heap, (new_val, mn_idx))

            if new_val > current_max:
                current_max = new_val
            else:
                current_max = max(arr[ptr[j]] for j in range(k))

            best = min(best, current_max - heap[0][0] + 1)

        out.append(str(best))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```预处理步骤构建 26 个位置列表，以便每个查询可以快速隔离相关字符。 对于每个查询，我们对每个列表进行二分搜索以查找间隔内的第一个有效出现。 这确保我们永远不会考虑范围之外的字符。 

堆维护每个字符当前选择的出现次数。 堆的最小元素给出当前窗口的左边界，而跟踪的最大值给出右边界。 每一步都会推进当前贡献左边界的角色，因为缩小左侧是在不丢失覆盖范围的情况下可能缩小跨度的唯一方法。 

一个微妙的实现细节是正确处理终止条件。 一旦指针到达其有效段的末尾，就不可能进一步完全覆盖，我们必须立即停止。 

## 工作示例

 考虑字符串`abaaba`以及覆盖全范围的查询$[1,6]$。 

我们跟踪职位`a`和`b`。 

| 步骤| 选择了一个 | 选择b | 最小 | 最大| 窗口|
 | ---| ---| ---| ---| ---| ---|
 | 1 | 1 | 2 | 1 | 2 | 2 |
 | 2 | 3 | 2 | 2 | 3 | 2 |
 | 3 | 3 | 5 | 3 | 5 | 3 |

 最佳窗口长度为 2，当两个字符相邻时尽早实现。 

现在考虑一个倾斜的情况：`aabbbaba`， 询问$[1,8]$。 

| 步骤| 位置 | b-位置 | 最小 | 最大| 窗口|
 | ---| ---| ---| ---| ---| ---|
 | 1 | 1 | 3 | 1 | 3 | 3 |
 | 2 | 4 | 3 | 3 | 4 | 2 |
 | 3 | 4 | 6 | 4 | 6 | 3 |

 最优窗口变为$[3,4]$，说明为什么有必要为每个字符选择不同的出现次数。 

这些痕迹表明，最佳窗口可以远离极端情况，并取决于字符之间的协调选择。

 ## 边缘情况

 单字符字符串演示了最简单的行为：答案始终为 1，因为唯一有效的子字符串是任何单个位置。 该算法正确初始化一个列表并立即返回一个大小为 1 的窗口。 

高度倾斜的分布，例如`a`集中在两端和`b`中间测试算法是否可以选择非极端情况。 指针机制确保一旦左边界由于选择不当而太大，推进负责的字符会将窗口向内移动并纠正跨度。 

所有字符都相同的情况可确保不同集大小为 1。该算法简化为跟踪单个列表，并且从不执​​行堆竞争，从而产生等于 1 的稳定答案。
