 ## 解决方案
 ## 问题理解

 我们有一个最初是空的`n × m`网格。 操作可以在任何单元执行。 

类型 1 运算填充所选行中的每个单元格和所选列中的每个单元格。 

类型 2 运算填充穿过所选单元格的两条对角线上的每个单元格。 

该任务不仅是确定填充整个网格所需的最少操作数，而且还要输出实现该最小值的操作序列。 网格尺寸满足`1 ≤ n, m ≤ 1000`，所有测试用例的单元总数最多为`10^6`。 

输出是有建设性的。 我们可以自由选择任何最佳序列。 

关键的观察结果是，答案仅取决于网格的较小尺寸。 自从`n`和`m`至多是`1000`，任何使用`O(min(n,m))`操作很容易在限制范围内。 

一个常见的错误是花精力利用对角线运算。 它们看起来很强大，但为了证明最优性，它们实际上是不必要的。 一旦确定了最小数量，仅使用类型 1 操作的非常简单的构造就足够了。 

考虑一个`1 × 5`网格。 对唯一行的一次类型 1 操作会立即填充每个单元格。 正确答案是`1`， 不是`5`。 

考虑一个`2 × 2`网格。 执行类型 1 操作`(1,1)`填充第 1 行和第 1 列，留下`(2,2)`裸露。 仍需进行第二次手术。 正确答案是`2`。 

这些例子表明最小值是由较小的尺寸而不是细胞总数控制的。 

## 方法

 蛮力的观点是将网格视为一个集合覆盖问题。 每个可能的操作都覆盖单元格的某些子集，并且我们希望其并集是整个网格的最小操作集合。 即使对于一个`1000 × 1000`网格中有一百万个可能的中心，因此任何详尽的搜索都是没有希望的。 

关键的观察结果是，类型 1 操作已经给出了完整的解决方案。 

认为`n ≤ m`。 如果我们在每一行中执行一次类型 1 操作，那么每一行都会被完全填充。 毕竟`n`行已被选择，整个网格已被填充。 对称地，如果`m < n`，在每一列中执行一次类型 1 操作会填充整个网格。 

这立即给出了上限`min(n,m)`运营。 

为了证明这是最优的，假设`n ≤ m`。 每个操作最多可以直接指定一行作为其选择行。 少于`n`操作时，至少有一行从未被选择。 从未被选择的行不能完全由少于`n`跨所有列和对角线的操作，因此无法完全覆盖网格。 因此至少`n`操作是必要的。 根据对称性，下界为`min(n,m)`。 

由于上限和下限匹配，因此最小操作数恰好是`min(n,m)`。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| 指数| 太慢了 |
 | 最佳施工| O(min(n,m)) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 阅读`n`和`m`。 
2.如果`n ≤ m`， 输出`n`运营。 
3. 对于每一行`i`从`1`到`n`，输出类型 1 操作`(i, 1)`。 

选择行`i`保证整行都被填满。 处理完所有行后，网格的每个单元格都属于某个选定的行。 
4.否则，输出`m`运营。 
5. 对于每一列`j`从`1`到`m`，输出类型 1 操作`(1, j)`。 

选择栏目`j`保证整个列都被填充。 处理完所有列后，网格的每个单元格都属于某个选定的列。 

### 为什么它有效

 如果`n ≤ m`，该结构每行只执行一次类型 1 操作。 每行都被完全填满，因此网格的每个单元格都被填满。 运算次数为`n`。 

如果`m < n`，对称参数表明每列的一个操作恰好使用填充整个网格`m`运营。 

下界也是`min(n,m)`，所以构造是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())

out = []

for _ in range(t):
    n, m = map(int, input().split())

    if n <= m:
        out.append(str(n))
        for i in range(1, n + 1):
            out.append(f"1 {i} 1")
    else:
        out.append(str(m))
        for j in range(1, m + 1):
            out.append(f"1 1 {j}")

sys.stdout.write("\n".join(out))
```实施直接跟随构造。 

什么时候`n ≤ m`，每个操作都是类型 1 并放置在不同的行中。 所选列始终是`1`因为它的确切值并不重要。 无论以哪一列为中心，该操作都会填充整行。 

什么时候`m < n`，对称结构每列使用一个操作，并且总是选择行`1`。 

不需要网格模拟。 输出本身就是解决方案。

## 测试用例```python
# helper: run solution on input string, return output string
import sys
import io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

    input = sys.stdin.readline
    t = int(input())

    out = []

    for _ in range(t):
        n, m = map(int, input().split())

        if n <= m:
            out.append(str(n))
            for i in range(1, n + 1):
                out.append(f"1 {i} 1")
        else:
            out.append(str(m))
            for j in range(1, m + 1):
                out.append(f"1 1 {j}")

    return "\n".join(out)

# minimum size
assert run("1\n1 1\n") == "1\n1 1 1"

# single row
assert run("1\n1 5\n") == "1\n1 1 1"

# single column
assert run("1\n5 1\n") == "1\n1 1 1"

# square grid
assert run("1\n2 2\n") == "2\n1 1 1\n1 2 1"

# rectangular grid
assert run("1\n3 4\n") == "3\n1 1 1\n1 2 1\n1 3 1"

 ## 边缘情况

 对于一个`1 × 5`网格：```
1
1 5
```该算法输出：```
1
1 1 1
```对唯一行的类型 1 操作会立即填充每个单元格。 答案是`1`。 

对于一个`5 × 1`网格：```
1
5 1
```该算法输出：```
1
1 1 1
```对唯一列的类型 1 操作会立即填充每个单元格。 

对于一个`2 × 2`网格：```
1
2 2
```该算法输出：```
2
1 1 1
1 2 1
```第一个操作填充第 1 行。第二个操作填充第 2 行。每个单元格都被覆盖，操作次数等于`min(2,2)=2`。 

同样的推理适用于所有更大的网格。 构造总是准确地使用`min(n,m)`操作并填充整个板。
