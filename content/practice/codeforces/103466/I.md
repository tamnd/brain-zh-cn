 ## 问题理解

 我们得到了索引从 0 到 n 的空间站集合。 每个站点都有一个整数能量值，站点 0 是我们的起点。 一开始，我们已经拥有等于 0 号站的能量，并且每当我们访问一个站时，我们都会永久获得它的能量。 

移动受到限制：我们只能从当前位置移动到能量不大于我们当前持有能量的位置。 由于能量只会在我们访问站点时增加，这意味着任何移动都必须遵守一个永远不会随着时间的推移而降低的动态阈值。 

任务是计算在此规则下从站 0 开始，有多少种不同的顺序可以访问所有站一次。答案需要对 1e9 + 7 求模。 

关键的结构约束是 n 可以大到 100000，而能量是 0 到 50 范围内的非常小的整数。这种不匹配立即表明解决方案不能直接以二次或三次方式依赖于站的数量，而必须使用较小的能量值范围来压缩状态。 

尝试模拟所有可能的排列的简单方法显然会失败，因为即使 n = 20 也已经产生了 20！ 可能性，这是不可行的。 即使子集上的 DP 也会爆炸超过 2^n。 

当许多站共享相同的能量值时，就会出现微妙的边缘情况。 例如，如果所有站点具有相同的能量，则每个排列都是有效的，因为约束永远不会阻止移动。 这种情况下的答案应该是（n+1）！ 任何错误地将等能量跃迁视为受限的方法都会严重低估。 

另一个极端情况是站 0 具有最大能量时。 那么所有其他站的能量必须≤ a0，因此任何排序都是有效的。 这测试算法是否正确处理初始阈值已经包含所有符合条件的 n
