 ## 解决方案
 ## 问题理解

 我们被赋予了一个封闭的世界，由分布在各个房间的人组成，没有空房间。 每个人都属于三种行为类型之一：婚姻的支持者、婚姻的反对者和中立的参与者。 

王子的目标是确定公主的确切房间。 他可以通过查询与人们互动，每个查询都会产生一个可能真实也可能不真实的答案，具体取决于受访者的隐藏类型。 支持者总是如实回答，反对者总是撒谎，中立的参与者可以任意回应。 王子事先并不知道任何人的类型。 

交互模型是对抗性的，因为答案并不是以对王子有帮助的方式固定的。 对手可以为中立参与者选择一致的行为模式，并利用说谎者的存在来最大化模糊性。 任务是确定是否可以保证在所有可能的世界中识别公主的房间与每种类型的给定计数一致，如果可以，则计算最坏情况下所需的最少查询数。 

主要困难在于查询不直接揭示结构。 每个问题仅产生通过潜在的对抗性真实性过滤的信息。 因此，问题的根本在于，当一小部分响应被保证是真实的、一小部分被保证是对抗性的、而其余的则不受限制时，可以提取多少可靠的信息。 

从复杂性的角度来看，每个类别最多允许 200,000 名参与者。 这排除了对个人或配置的任何状态空间模拟或交互式搜索。 任何解决方案都必须将问题简化为少量派生量，并对信息限制进行代数推理，而不是模拟交互。 

当真实参与者的数量相对于对抗性不确定性而言太少时，就会出现微妙的边缘情况。 在这种情况下，即使任意多的询问也无法孤立公主，因为总是可以安排矛盾的答案。 例如，如果恰好有一名支持者和一名反对者，那么针对他们的任何查询都可以被镜像或否定，从而无法以一致的方式消除公主位置的歧义。 

另一种极端情况是根本没有支持者。 如果每个人要么保持中立，要么撒谎，那么任何查询都无法产生有保证的真实信号。 即使重复提问也无济于事，因为可以选择中立的答案，而说谎者总是会颠倒事实。 在这种情况下，系统没有真理的锚点，问题就变得无法解决。 

## 方法

 蛮力视角将尝试模拟所有可能的真值分配配置和所有可能的查询序列。 每个查询的分支取决于受访者是否诚实、撒谎或任意。 参与者数量和查询深度很快就会呈指数级增长。 即使将注意力限制在少数人身上，分支因子也会随着每次交互而成倍增长，使得探索即使是中等深度也是不可能的。 

核心观察是，唯一有用的结构不是个人身份，而是真实信息源和敌对信息源之间的不平衡。 支持者提供一致的全局约束，而反对者则注入系统反转。 中立者可以被视为最坏情况的对手，因为他们的行为方式可能会保持模糊性。

这个问题简化为询问是否存在一系列问题可以消除最坏情况下公主的所有可能位置，除了一个可能的位置。 这相当于确定可靠信号的数量是否可以压倒对抗性失真的数量。 一旦认识到这一点，问题就会分解为 a、b 和 c 上的确定性算术条件。 

每个有效查询可以被视为最多提取一个单位的可靠区分能力，但敌对参与者可以根据奇偶性和一致性约束消耗或取消该能力。 由此产生的结构是线性的：要么真实的基础足以打破对称性，要么根本不可能。

 ## 算法演练

 1. 读出代表支持者、反对者和中立者的三个整数a、b和c。 
2. 检查系统是否至少包含1个支撑体。 如果没有支持者，那么就无法以有保证的方式信任任何查询，因为每个答案都可以直接（反对者）或任意（中立者）进行对抗性操纵。 这种情况下，立即输出任务不可能完成。 
3. 如果恰好有一名支持者和至少一名反对者，则确定歧义无法解决。 支持者发出的任何真实信号都可以被单个敌对受访者反映或抵消，而中立者可以保留这种模糊性。 因此，输出不可能。 
4. 如果存在支持者并且对抗性结构不能完全中和他们，则计算所需的最小查询数量作为总不确定性质量的函数。 每个查询通过隔离或确认一个约束来有效地减少歧义，最坏的情况要求对应于覆盖所有敌对参与者并解决公主的最终位置。 
5. 输出 YES，后跟计算出的最小查询数。 

### 为什么它有效

 该算法依赖于只有支持者贡献单调、不可逆信息的不变量。 反对者总是否定信息，而中立者可以根据保持模糊性的方式来模拟任何一种行为。 只要相对于对抗压力存在足够数量的支持者，每个查询就可以被解释为消耗一个单位的不确定性。 如果无法建立这种平衡，对手就可以无限期地维持多个一致的世界，从而使唯一识别变得不可能。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, c = map(int, input().split())

    # No truthful anchor => impossible
    if a == 0:
        print("NO")
        return

    # If there is exactly one supporter and at least one opponent,
    # adversary can always maintain ambiguity
    if a == 1 and b > 0:
        print("NO")
        return

    # Otherwise, solution exists; minimal queries reduce to resolving all uncertainty
    # Each adversarial participant effectively contributes one unit of ambiguity
    ans = b + c

    print("YES")
    print(ans)

if __name__ == "__main__":
    solve()
```代码首先读取计数并立即处理结构上不可能的情况。 第一个检查删除没有真实锚点的配置。 第二项检查抓住了脆弱的边界，其中单一真实来源不足以对抗任何敌对存在。 

如果这两个条件都不成立，则算法将每个非支持者参与者视为贡献一个必须通过查询解决的不确定性单位。 这导致直接线性计算。 

一个微妙的实现细节是第二个条件是严格的`a == 1 and b > 0`，不包括中立者。 中立者本身不会引入矛盾倒置，但反对者会引入矛盾倒置，并且在最坏情况的对抗性选择反应下，单个支持者甚至无法可靠地击败一个系统性说谎者。

 ## 边缘情况

 当 a = 0 时，每个参与者都可以表现出对抗性，因此任何查询序列都允许多种一致的解释。 该算法立即正确返回“否”，而无需尝试任何计算。 

当 a = 1 且 b > 0 时，单个诚实的参与者甚至无法支配一个系统性的说谎者，因为每一个信息性陈述都可以被否定。 该算法明确拒绝这种情况，匹配不可能性条件。 

当 b = 0 且 c > 0 时，不存在对抗性约束的说谎者，只有任意响应者。 该算法将中性因素视为造成不确定性的因素，但仍然允许解决问题，因为至少有一个支持者锚定了事实。 例如，输入 2 0 5 会通过 5 个查询得出“是”，并且每个中性值实际上代表一个必须解决的单独歧义。 

当a很大且b+c为零时，世界是完全真实的，无需查询即可有效确定公主位置。 该算法在零查询的情况下输出“是”，这与不需要提取信息的事实相匹配。
