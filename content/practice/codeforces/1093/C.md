 ## 解决方案
 ## 问题理解

 我们得到一个隐藏的非递减数组`a`长度均匀的`n`。 我们从未见过`a`直接地。 相反，我们被告知了其结构的一半：对于每对对称位置，末端元素的总和是已知的。 具体来说，对于每个`i`从左边开始，`a[i] + a[n - i + 1]`给出，形成一个数组`b`。 

任务是重建任何有效的数组`a`它是非递减的，由非负整数组成，并且匹配所有这些成对的和。 许多不同的数组都可以满足条件，并且任何一种有效的构造都是可以接受的。 

这些约束推动线性或近线性重建。 和`n`最多`2 · 10^5`， 一个`O(n^2)`甚至`O(n log n)`具有大量常量是可以接受的，但是任何涉及重复全局搜索或回溯每个可能的分割的事情`b[i]`太慢了，因为每个`b[i]`可以对应许多可能的分解成一对`(x, y)`。 

主要的困难是每一项总和`b[i]`隐藏两个未知值，并且这些值受到对完整重构数组进行排序的要求的全局约束。 一种独立分割每个的简单方法`b[i]`将会失败，因为它忽略了不同对之间的相互作用。 

当贪婪地分割每一个时，就会出现一个常见的陷阱`b[i]`进入`(0, b[i])`或者`(b[i]/2, b[i]/2)`。 例如，使用输入`n = 4`,`b = [1, 100]`，选择`(0,1)`和`(0,100)`产生`a = [0,0,100,1]`，即使每对单独有效，它也没有排序。 

另一种失败模式是从左到右独立地分配对称对，而不强制左端点不减少且右端点不增加。 局部自由正是破坏全球一致性的原因。 

## 方法

 一个蛮力的想法是尝试每个分割的所有分割`b[i]`。 对于每一对，我们选择`x`从`0`到`b[i]`并设置`y = b[i] - x`，然后检查我们是否可以将所有对排列成全局排序序列。 这立即爆炸了：每个`n/2`对最多有`10^{18}`的可能性，使其完全不可行。 

关键的结构见解是我们不需要独立分配对。 相反，我们从外向内构造数组，同时保留排序约束。 最终的数组可以看作两个单调序列：左半部分是非递减的，右半部分是非递增的，并且每个`b[i]`从每一侧绑定一个元素。 

我们按降序处理总和，以便首先确定大的约束。 在每一步中，我们都会决定一对`(x, y)`这样`x + y = b[i]`，同时保持到目前为止构建的序列保持单调性一致。 贪心的选择是推`y`尽可能大，因为由于右侧的要求不增加，较大的右侧值以后更难容纳。 一次`y`是固定的，`x`是被迫的。 

这将问题简化为具有几个运行边界的单个线性扫描。

 ## 算法演练

 我们解读每一个`b[i]`定义一对`(x_i, y_i)`在哪里`x_i + y_i = b[i]`，并且在最终数组中所有`x_i`形成左半部分，而所有`y_i`形成右半部分。 

我们强制执行两个单调结构：`x_i`必须是非递减的，并且序列`y_i`必须是非增加的。 此外，对于每一对，我们必须有`x_i ≤ y_i`以便合并后的数组保持排序。 

为了一致地执行这些约束，我们处理`b`按降序排列。 

## 算法步骤

 1. 排序`b`按非递增顺序。 

首先处理大和可以防止后来的小和在受限位置强制不可能的大值。 
2.初始化两个运行变量：`prev_x = 0`和`prev_y = +∞`。`prev_x`跟踪最后选择的左侧值，确保左侧的顺序不降序。`prev_y`跟踪最后选择的右侧值，确保右侧的顺序不递增。 
3. 对于每个值`b_i`按排序顺序，计算右端点的可行范围`y`。 

自从`x = b_i - y`必须满足`x ≥ prev_x`，我们得到`y ≤ b_i - prev_x`。 

另外，为了保持右侧的单调性，`y ≤ prev_y`。 

所以`y`必须满足`y ≤ min(prev_y, b_i - prev_x)`。 
4. 选择`y`在有效范围内尽可能大：`y = min(prev_y, b_i - prev_x)`。 

这种贪婪的选择为未来的步骤保留了最大的灵活性。 
5. 设置`x = b_i - y`。 

这自动满足总和约束和`x ≥ prev_x`。 
6. 更新`prev_x = x`和`prev_y = y`，并存储`x`和`y`到单独的列表中。 
7.处理完所有对后，左边就是收集到的`x`按顺序排列值，右侧为收集到的值`y`值按相反顺序排列。 将它们连接起来形成最终的数组。 

## 为什么它有效

 该结构保持严格的不变量：处理每个`b_i`，所选择的`x`值形成与已经固定的决策一致的最小可能的非递减序列，而`y`值形成与相同决策一致的最大可能的非递增序列。 因为每一步都贪婪地最大化`y`在可行性约束下，任何后续分配都不能迫使右侧出现矛盾，并且推导出来`x`自动是与总和和左单调性兼容的最小值。 这可以防止未来的总和需要小于已固定值的左值，这是构造失败的唯一方法。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    b = list(map(int, input().split()))
    
    b.sort(reverse=True)
    
    prev_x = 0
    prev_y = 10**30
    
    left = []
    right = []
    
    for val in b:
        y = min(prev_y, val - prev_x)
        x = val - y
        
        left.append(x)
        right.append(y)
        
        prev_x = x
        prev_y = y
    
    a = left + right[::-1]
    print(*a)

if __name__ == "__main__":
    solve()
```实现直接遵循贪心构造。 排序`b`确保我们首先锁定最大的约束。 这两个运行变量编码了唯一重要的全局限制：重建的一半的单调性。 的选择`y`故意最大化以避免打破未来的约束，而`x`由总和方程确定性地确定。 

最后的串联`left + reversed(right)`之所以有效，是因为右侧是以非递增顺序构造的，因此反转它会产生最终数组所需的非递减后缀。

 ## 边缘情况

 当所有的情况出现时，就会出现微妙的边缘情况`b[i]`是平等的。 在这种情况下，每一对都具有相同的灵活性，并且幼稚的结构通常会以破坏全局顺序的方式对称分裂。 贪心方法通过总是推动来避免这种情况`y`达到其最大允许值，无论对称性如何，都会产生严格一致的单调结构。 

另一种情况是当一个小`b[i]`出现在大的之后。 如果不进行排序，一小笔钱可能会过早地限制`prev_x`，使得无法拟合早期的大值。 排序确保首先放置大的约束，因此结构永远不会变得不可行。 

最后，情况`b[i]`与之前的选择勉强一致测试严格的边界`y ≤ b_i - prev_x`。 该算法依赖于这种不等式的精确执行； 任何放松都会导致无效的负值或减少的左侧值，这会立即破坏排序。
