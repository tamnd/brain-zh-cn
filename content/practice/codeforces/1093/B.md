 ## 解决方案
 ## 问题理解

 我们得到了几个仅由小写英文字母组成的独立字符串。 对于每个字符串，我们可以按照我们想要的任何顺序重新排列其字符。 重新排列后，我们必须决定是否可以获得一个不是回文的字符串，或者报告无论我们如何排列字母，每个得到的排列仍然是回文。 

回文是向前和向后读相同的字符串。 任务不是构造所有排列，而是确定是否存在至少一种非回文排列。 如果存在，我们输出任何此类重新排列。 

约束很小：每个字符串的长度最多为 1000，并且最多有 100 个查询。 这立即排除了任何枚举排列的尝试，因为即使对于长度 10，排列数量也已经是 360 万个，而对于 1000 个排列则是不可行的。 每个字符串的线性或线性算术解就足够了。 

当字符串具有所有相同的字符或其结构高度对称时，就会出现关键的微妙情况。 例如，如果字符串是`"aaa"`，每个排列仍然是`"aaa"`，这是一个回文。 相似地，`"aa"`不能改变成任何非回文的东西。 

一个不那么简单的情况是，字符串具有多个不同的字符，但在所有排列下仍然被迫成为回文。 只有当所有字符都相同时才会发生这种情况，因为排列的任何变化都会引入不对称性。 

一个天真的错误是假设字符串像`"ab"`或者`"aba"`行为类似。 实际上，`"aba"`可以重新排列成`"aab"`或者`"baa"`，都是非回文，所以除非字符串完全一致，否则总是可能的。 

## 方法

 暴力策略将生成字符串的所有不同排列，并检查每个排列是否为回文。 这是正确的，因为它详尽地探索了所有可能的重新排列的搜索空间。 然而，排列的数量随字符串长度呈阶乘增长，即使删除重复字符也无法使长度达到 1000 时可行。 

关键的观察是，我们没有被要求在排列中找到特殊的结构，只是为了完全避免回文。 仅当每次重新排列都相同时（当所有字符都相同时才会发生这种情况），字符串才不可能成为非回文。 在这种情况下，交换不会改变任何事情。 

如果至少有两个不同的字符，我们总是可以产生一个非回文。 保证这一点的一个简单方法是对字符串进行排序。 如果排序后的字符串已经不是回文，我们就完成了。 如果它是回文，则将最后一个字符交换到任何不同的位置都会破坏对称性，因为至少存在两个不同的字母，因此总是存在不匹配的情况。 

因此，问题简化为检查所有字符是否相同，否则打印除全相同字符情况之外的任何排列。

 | 排序+简单构造| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们独立地为每个字符串构造一个有效的答案。 

1. 读取字符串并通过将每个字符与第一个字符进行比较来计算所有字符是否相同。 此步骤确定了唯一不可能的情况。 
2. 如果所有字符都相同，则输出`-1`因为每个排列都是相同的字符串，并且该字符串是回文。 
3. 否则，按字典顺序递增对字符串进行排序。 排序可确保字符的确定性排列。 
4. 通过比较镜像位置来检查排序后的字符串是否为回文。 
5. 如果不是回文，则直接输出，因为已经满足要求。 
6. 如果是回文，则将第一个字符与任何与其不同的字符交换。 由于并非所有字符都相同，因此必须存在这样的位置。 这种交换保证了不对称性，产生非回文字符串。 

### 为什么它有效

 每个排列都是回文的唯一情况是所有字符都相等，因为任何重新排列都会保持一致性。 如果至少存在两个不同的字符，则必须至少有一个位置的字符与另一位置不同。 排序的排列将相同的字符集中在一起，并且涉及不同字符的任何交换都必然会破坏某个位置的镜像对称性。 因此，在这种情况下我们总是可以构造一个非回文排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(s: str) -> str:
    n = len(s)
    if all(c == s[0] for c in s):
        return "-1"

    s = sorted(s)
    if s != s[::-1]:
        return "".join(s)

    s[0], s[-1] = s[-1], s[0]
    return "".join(s)

def main():
    t = int(input())
    out = []
    for _ in range(t):
        s = input().strip()
        out.append(solve_one(s))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先检测所有字符都匹配的微不足道的不可能情况。 排序创建基线排列。 回文检查确保我们不会在边缘情况下意外输出对称字符串，例如`"aabbaa"`。 如果出现对称性，则交换端点可保证镜像位置之间至少存在一处不匹配。 

交换策略是安全的，因为至少两个不同字符的存在保证了第一个和最后一个位置在所有情况下都不能相同，因此在末端打破对称性总是足够的。

 ## 边缘情况

 统一字符大小写是唯一真正的阻塞配置。 用于输入`"zzzz"`，支票立即返回`-1`因为没有重新排列会改变字符串。 

为了`"aabbaa"`，排序产生回文。 然后该算法交换端点，但由于两端可能相同，因此简单的交换可能会失败。 正确的执行扫描具有不同字符的位置并与其交换，产生一个类似的字符串`"baaaba"`，这不是对称的。 

这确保了即使排序意外产生回文，至少两个不同字符的存在也能保证有效的破坏操作始终存在。
