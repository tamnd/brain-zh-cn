 ## 解决方案
 ## 问题理解

 给定一个无向图，并要求为每个顶点分配集合 {1, 2, 3} 中的一个值。 仅当每条边都连接两个值之和为奇数的顶点时，该分配才有效。 任务是计算存在多少这样的分配，以 998244353 为模。 

条件“和为奇数”立即限制哪些对可以出现在边缘上。 当一个端点为奇数而另一个端点为偶数时，总和恰好为奇数。 在允许的标签中，1和3是奇数，而2是偶数。 这将问题从使用三个任意标签转换为使用二分约束：每条边必须将标记为奇数的顶点连接到标记为偶数的顶点。 

因此，图的每个连接组件都必须是二分的，否则无法沿边一致地分配奇偶校验。 

约束条件很大，所有测试用例的顶点和边总数最多为 3×10^5。 这排除了任何尝试通过回溯或强力枚举分配为每个顶点独立分配值的方法，因为即使是大小为 n 的单个组件也会导致 3^n 种可能性。 使用图遍历，每个测试用例的预期解决方案必须是线性或接近线性的。 

一些边缘情况很重要：

 诸如长度为 3 的环之类的三角形图使答案为零。 例如，n=3，边为 (1-2, 2-3, 3-1)。 任何改变平价的尝试都会失败，因为它会在周期中引发矛盾。 

孤立的顶点始终有效并且独立地对计数做出贡献。 例如，n=1 且 m=0 的单节点图有 3 个有效分配：{1,2,3}。 

断开连接的图需要以乘法方式组合跨组件的答案，因为不同组件上的分配不会相互作用。 

## 方法

 暴力方法会尝试将值 1、2 或 3 分配给每个顶点，并检查每条边是否满足奇偶校验约束。 这需要检查 3^n 个分配并在 O(m) 中验证每个分配，给出 O(3^n · m)，即使 n 约为 20，这也是不可行的。 

关键的简化来自于将值解释为奇偶校验类。 每个顶点要么是奇数（1 或 3），要么是偶数（2）。 边缘约束仅取决于奇偶校验，而不取决于所选的确切奇数值。 如果我们首先决定将图二分为两个奇偶校验类，然后我们可以细化每个类内的分配。 

对于连接的二分组件，有两种有效的方法来分配奇偶校验：选择起始节点为偶数或奇数并传播。 如果一个组件不是二分的，传播就会导致矛盾，这意味着答案为零。 

一旦奇偶性固定，每个分配奇数的顶点可以独立地为 1 或 3，从而为奇数侧的每个顶点提供 2 个选择。 每个分配的顶点都必须是 2，没有自由度。 然而，我们可以自由地交换每个组件的全局奇偶校验方向，这会翻转哪一侧被认为是奇数。 这意味着对于大小为 a 和 b 的二分组件，有效分配的数量为 2^a + 2^b。 

由于组件是独立的，我们将这些贡献乘以所有组件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(3^n·m) | O(3^n·m) | O(n + m) | 太慢了 |
 | DFS + 二分计数 | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。

1. 构建图的邻接表。 这种表示是必要的，因为我们将重复遍历组件，并且邻接表允许线性时间遍历。 
2. 维护一个颜色数组，其中每个节点都未被访问或分配有两种奇偶校验颜色之一。 我们使用 DFS 或 BFS 沿边缘分配交替颜色。 
3. 对于每个未访问的节点，开始遍历并尝试对其连接的组件进行 2 色着色。 在遍历过程中，我们计算每个颜色类别中有多少个节点。 这给出了组件的二分区大小。 
4. 如果在遍历过程中发现有一条边连接两个颜色相同的节点，则该图不是二分图。 在这种情况下，不存在有效的赋值，因为奇偶校验约束会强制矛盾，因此我们立即返回 0。 
5. 对于有效组件，将其贡献计算为 2^a + 2^b，其中 a 和 b 是两个颜色类的大小。 我们将此值乘以全局答案。 
6. 对所有组件重复此操作，并输出对 998244353 取模的最终结果。 

其原理与奇偶校验的传播方式有关。 一旦为节点分配了奇偶校验，所有邻居都被迫采用相反的奇偶校验，并且此约束会传播到整个组件。 这意味着每个连接的组件的行为就像一个二部图问题，具有两个全局方向选择。 固定方向后，奇数侧的每个节点独立地在两个奇数标签之间进行选择，而偶数节点则固定为 2。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    color = [-1] * n
    ans = 1

    for i in range(n):
        if color[i] != -1:
            continue

        stack = [i]
        color[i] = 0
        cnt = [0, 0]

        ok = True

        while stack:
            u = stack.pop()
            cnt[color[u]] += 1

            for v in g[u]:
                if color[v] == -1:
                    color[v] = color[u] ^ 1
                    stack.append(v)
                elif color[v] == color[u]:
                    ok = False

        if not ok:
            return 0

        a, b = cnt[0], cnt[1]

        pow2_a = pow(2, a, MOD)
        pow2_b = pow(2, b, MOD)

        ans = ans * (pow2_a + pow2_b) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```邻接表构造确保我们可以在 DFS 期间遍历每条边一次。 颜色数组对二分尝试进行编码； -1表示未访问过，0/1表示两侧。 

基于堆栈的 DFS 避免了递归深度问题，因为 n 可能很大。 每个节点处理一次，每条边检查两次，保持复杂性线性。 

关键的实现细节是在遍历期间计算每种颜色的节点数。 这直接给出了哪一边是“奇数”的两种可能的解释。 我们分别计算两边的 2 幂，因为每个标记为 odd 的顶点都有两个选择：1 或 3。 

如果发现冲突，立即返回 0 以避免不必要的计算。 

## 工作示例

 ### 示例 1：单边

 输入：

 n=2，边（1-2）

贡献为 2^1 + 2^1 = 2 + 2 = 4。

 ## 边缘情况

 孤立的顶点被视为单节点组件。 DFS 为其分配颜色 0，生成计数 (1,0)。 贡献变为 2^1 + 2^0 = 2 + 1 = 3，这与三个可能的标签匹配。 

断开连接的图是逐个组件处理的，乘法步骤确保组件之间分配的独立性。 

当已着色的邻居违反奇偶校验一致性时，在 DFS 期间会检测到非二分循环。 此时，算法将中止并返回零，从而防止任何不正确的部分计数。
