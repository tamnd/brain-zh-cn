 ## 解决方案
 ## 问题理解

 我们得到一个部分指定长度的数组$n$。 每个位置要么已经包含一个固定值$1$和$k$，或者未知并标记为$-1$。 我们必须用以下值替换每个未知位置$1$到$k$，产生一个完全填充的数组。 

如果填充数组不包含精确的连续块，则该数组被认为是有效的$len$相等的值。 换句话说，没有索引$i$使得该段$a[i], a[i+1], \dots, a[i+len-1]$由相同的数字组成。 

任务是计算有多少种方法可以替换所有$-1$条目，以便最终数组满足此约束，模$998244353$。 

这些限制立即迫使我们进入动态编程的思维方式。 数组长度最大为$10^5$，因此任何尝试显式分支分配的解决方案在数量上都是指数级的$-1$值并且完全不可行。 字母大小$k$很小，最多 100，这强烈表明取决于先前值或较短历史记录的转换是可行的。 

一个微妙的点是，禁止条件完全取决于长度相等的游程$len$，不仅仅取决于相邻元素之间的不平等。 这排除了简单的贪婪填充或本地检查，因为运行早期的决定可能会使许多步骤之后的配置无效。 

打破朴素推理的边缘情况很容易构建。 如果$len = 1$，那么每个元素都自己形成一个禁止段，因此答案始终为零，除非问题定义隐式保证否则； 这迫使算法立即检测并处理这个问题。 另一个重要的情况是当数组完全固定并且已经包含长度的游程时$len$， 例如$len = 3$， 大批$[5, 5, 5]$，即使不涉及任何选择，答案显然为零。 如果假设所有违规都是在分配期间创建的，则仅在填充期间检查约束的简单方法可能会错过这一点。

 ## 方法

 蛮力策略会尝试为每个值分配值$-1$递归并在每次完成后检查有效性。 每个位置最多有$k$选择，因此在最坏的情况下，所有条目等于$-1$，作业数量为$k^n$，对于$n = 10^5$。 Even pruning invalid partial runs only helps locally, but does not prevent exponential branching.

 关键的观察结果是，就运行而言，约束纯粹是局部的：唯一重要的是当前相等值的后缀有多长。 我们不需要记住数组的完整历史记录，只需要记住最后一个值以及到目前为止它已经连续重复了多少次。 一旦跑步达到长度$len$，禁止对该相同值进行任何进一步扩展。 

这将问题简化为动态编程状态，跟踪最后一个值及其游程长度，或者更有效地跟踪每个位置的当前游程对于每个可能值的长度。 因为$k \le 100$，我们可以在值上维护 DP，并使用游程长度上的前缀和来压缩转换。 

更有效的公式是定义 DP，我们从左到右构建数组，并为每个位置维护在该位置结束的方式数量$i$与最后一个值$v$，同时确保当前游程长度严格小于$len$。 当我们延长运行时，我们要么继续相同的值（如果允许），要么切换到不同的值，这会将运行长度重置为 1。

 关键的技术技巧是避免显式地跟踪每个状态的完整游程长度，而是为每个值维护有多少种方法可以准确地结束游程$t$，然后使用前缀和压缩转换，以便扩展运行变成$O(1)$每个州。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(k^n)$|$O(n)$| 太慢了 |
 | 带有跑步追踪功能的 DP |$O(n \cdot k)$|$O(k \cdot len)$或优化$O(k)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组并维护描述在每个位置结束的有效前缀的 DP 状态。 

1.我们定义一个DP表，其中$dp[v][c]$表示填充到当前位置的方式数，使得最后一个值是$v$当前的连续游程长度是$c$， 在哪里$1 \le c < len$。 这直接通过构造对禁止条件进行编码，因为我们从不存储达到长度的状态$len$。 
2.我们在位置0处初始化DP。如果第一个元素固定为一个值$x$，我们设置$dp[x][1] = 1$。 如果是的话$-1$，我们设置$dp[v][1] = 1$为所有人$v \in [1, k]$。 这反映出每个有效数组必须以长度为 1 的游程开始。 
3. 对于每个下一个位置，我们构建一个新的 DP 表$ndp$初始化为零。 我们迭代所有值$v$和游程长度$c$当前计数非零。
 4. 如果当前数组位置固定为一个值$x$，我们只允许转换到新的最后一个值变为的状态$x$。 否则，我们考虑所有值$1 \dots k$。 
5.来自一个州$(v, c)$，我们考虑两种类型的转换。 如果我们放置相同的值$v$，我们可以将运行延伸到$c+1$仅当$c+1 < len$。 这有助于$dp[v][c]$到$ndp[v][c+1]$。 
6. 如果我们设置不同的值$u \ne v$，运行重置，所以我们贡献$dp[v][c]$到$ndp[u][1]$。 既然有很多可能$u$，我们使用所有状态的总和来汇总贡献，并减去保持相同值的贡献。 
7.处理完所有状态后，我们替换$dp$和$ndp$并继续。

 处理完所有位置后，答案是所有值和游程长度上所有有效 DP 状态的总和。 

### 为什么它有效

 DP 状态准确地编码了确定未来扩展的有效性所需的信息：只有最后一个值以及它重复了多长时间。 以相同最后值和相同游程长度结尾的任何两个前缀在未来的决策中可以互换。 由于转换仅取决于与最后一个值的相等或不相等，因此没有额外的历史记录会影响正确性。 保持的不变性是每个 DP 状态对应于一个没有禁止运行的有效前缀，并且每个有效前缀在一个状态中只被计数一次。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, k, L = map(int, input().split())
    a = list(map(int, input().split()))
    
    if L == 1:
        print(0)
        return
    
    dp = [[0] * (L - 1) for _ in range(k + 1)]
    
    if a[0] == -1:
        for v in range(1, k + 1):
            dp[v][0] = 1
    else:
        dp[a[0]][0] = 1
    
    for i in range(1, n):
        ndp = [[0] * (L - 1) for _ in range(k + 1)]
        
        total = 0
        for v in range(1, k + 1):
            for c in range(L - 1):
                total = (total + dp[v][c]) % MOD
        
        if a[i] == -1:
            for v in range(1, k + 1):
                for c in range(L - 1):
                    val = dp[v][c]
                    if not val:
                        continue
                    # switch to same value
                    if c + 1 < L - 1:
                        ndp[v][c + 1] = (ndp[v][c + 1] + val) % MOD
                    # switch to different values
                    ndp[v][0] = (ndp[v][0] + (total - val)) % MOD
        else:
            x = a[i]
            for v in range(1, k + 1):
                for c in range(L - 1):
                    val = dp[v][c]
                    if not val:
                        continue
                    if v == x:
                        if c + 1 < L - 1:
                            ndp[v][c + 1] = (ndp[v][c + 1] + val) % MOD
                    else:
                        ndp[x][0] = (ndp[x][0] + val) % MOD
        
        dp = ndp
    
    ans = 0
    for v in range(1, k + 1):
        for c in range(L - 1):
            ans = (ans + dp[v][c]) % MOD
    
    print(ans)

if __name__ == "__main__":
    solve()
```DP表通过值和当前游程长度减一来索引，这样就避免了存储长度$len$本身，因为它是无效的。 每次迭代都会重新计算一个新的表，确保位置之间没有污染。 

一个关键的实现细节是处理不同值的转换。 我们没有明确地迭代所有可能的目标值，而是使用所有 DP 状态的总和并减去当前状态，这避免了额外的因子$k$在转换逻辑的那部分。 

另一个微妙之处是游程长度的逐一处理。 该代码存储从 0 而不是 1 开始的游程长度，因此状态为$c = 0$对应于长度为 1 的游程。这简化了索引，但在扩展游程时需要仔细的边界检查。

 ## 边缘情况

 当$len = 2$，约束禁止任何相等的相邻对。 DP 立即退化为没有相等邻居的标准着色，并且过渡绝不能延长运行。 该算法自然会处理这个问题，因为任何增加长度超过长度 1 的尝试都会被拒绝。 

当数组完全固定时，DP 永远不会分支。 状态只是通过固定值传播，并且任何违规都会自动消除，因为没有达到非法游程长度的有效 DP 状态存活下来。 

什么时候$k = 1$，唯一可能的数组是全一。 答案是 1 如果$n < len$，否则为 0。DP 折叠成单条游程扩展链，当游程长度达到时，转换规则直接阻止禁止情况$len$。
