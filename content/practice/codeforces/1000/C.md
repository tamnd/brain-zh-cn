 ## 解决方案
 ## 问题理解

 我们得到了一条非常大的数轴上的线段集合。 每个线段覆盖其端点之间的每个整数点，包括两端。 不同的线段可以以任意方式重叠：它们可以是不相交的、嵌套的、相同的或部分相交的。 

任务不是找到单个点的覆盖范围，而是了解覆盖范围的全局分布。 对于每个可能的 k 值，我们需要计算恰好 k 个段覆盖了多少个整数位置。 

换句话说，如果我们想象沿着数轴扫描，并在每个整数坐标处计算当前覆盖它的线段数，我们想知道有多少个坐标产生计数 1，有多少个坐标产生计数 2，依此类推，直到 n。 

约束是迫使设计选择的因素。 由于有多达 200,000 个线段和大至 10^18 的端点，我们无法显式检查每个整数点。 任何迭代坐标的方法都是立即不可行的，因为坐标空间在这个尺度上实际上是无界的。 我们需要根据覆盖范围的变化而不是单个点来进行推理。 

一种常见的失败模式是尝试在不压缩坐标的情况下模拟每个点或每个单位间隔的覆盖范围。 例如，对于段 [0, 3]、[1, 3]、[3, 8]，对每个整数的简单扫描将适用于这个微小的输入，但如果间隔延伸到 10^18，则这是不可能的。 另一个微妙的错误是将间隔不一致地视为半开。 由于端点是包容性的，覆盖范围在 l 和 r + 1 处发生变化，而不是在 r 处发生变化。 

## 方法

 蛮力的想法很简单：对于位于至少一个段内的每个整数坐标，计算有多少段覆盖它，然后递增由该计数索引的频率数组。 从概念上讲，对于每个分段，我们标记其覆盖的所有点，并为每个点维护一个全局计数器。 

这在逻辑上是可行的，但会立即破坏性能。 单个段最多可以跨越 10^18 个整数，因此即使是一个区间也可能需要不可能的工作。 即使我们试图将自己限制为仅观察到的端点，我们仍然面临这样的问题：覆盖范围仅在事件点之间恒定，而不一定在事件点处恒定。 

关键的观察结果是覆盖计数仅在段边界发生变化。 在数轴上两个连续的“事件位置”之间，活动线段的数量是恒定的。 因此，我们不是考虑每个整数点，而是将问题压缩到覆盖范围不变的区间内。 

这引出了扫描线的想法：我们将每个片段转换为两个事件，一个事件增加 l 处的覆盖范围，另一个事件减少 r 之后的覆盖范围。 对这些事件进行排序使我们能够按升序处理该行并维护当前的活动段计数。 每当我们从一个事件位置移动到下一个事件位置时，段计数在整个间隙中保持固定，因此我们可以将该间隙的长度添加到该计数的答案中。 

这减少了从每点计数到每间隔聚合的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(总覆盖点) | O(1) 到 O(n) | 太慢了 |
 | 扫线| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个片段转换为两个边界事件，一个在片段结束后增加覆盖范围，一个减少覆盖范围。 然后我们按位置对这些事件进行排序。 

我们维护一个运行变量`active`表示当前有多少段覆盖扫描位置。 我们还按顺序迭代排序的事件位置。 

1. 将每个段 [l, r] 转换为两个事件：在 l 处添加 +1，在 r + 1 处添加 -1。 此编码可确保包含间隔的正确性，因为该段通过 r 进行贡献，并从 r + 1 开始停止贡献。 
2. 按坐标对所有事件进行排序。 排序是必要的，因为我们需要按从左到右的顺序处理覆盖范围的变化。 
3. 初始化`active = 0`和一个空的结果数组`ans`大小为 n + 1。 
4. 遍历已排序的事件，同时跟踪先前的坐标`prev`。 对于每个不同的坐标`x`，我们首先考虑来自`prev`到`x - 1`。 在整个区域内，活动段的数量是恒定的，因此该范围内的每个整数点都有助于`ans[active]`。 
5. 考虑间隙后，应用坐标处的所有更新`x`, 调整`active`因此。 
6. 移动`prev`到`x`并继续。 
7. 处理完所有事件后，不再存在任何间隙。 

关键的正确性点是，我们只在没有事件发生的稳定区域上分配计数，确保每个整数点只被计数一次，并且与其真实的覆盖值完全一致。 

### 为什么它有效

 扫描线保持两个连续事件坐标之间的不变性，没有段开始或结束。 因此，活动段的集合在整个时间间隔内是恒定的。 由于每个整数点恰好属于一个这样的间隔，并且我们使用该常量值来计算其完整长度，因此每个点对正确的频率桶仅贡献一次。 端点移位 +1 可确保处理包容性，而不会在边界处进行重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    events = []
    
    for _ in range(n):
        l, r = map(int, input().split())
        events.append((l, 1))
        events.append((r + 1, -1))
    
    events.sort()
    
    ans = [0] * (n + 1)
    
    active = 0
    prev = None
    
    i = 0
    m = len(events)
    
    while i < m:
        x = events[i][0]
        
        if prev is not None and prev < x:
            length = x - prev
            ans[active] += length
        
        delta = 0
        while i < m and events[i][0] == x:
            delta += events[i][1]
            i += 1
        
        active += delta
        prev = x
    
    print(*ans[1:])

if __name__ == "__main__":
    solve()
```该解决方案将段端点编码为差异数组样式事件。 关键细节是使用 r + 1 进行递减事件，它将包含覆盖范围与半开区间处理对齐。 对同一坐标处的所有更改进行排序，并且我们在继续之前聚合它们，以便同一位置的中间状态不会错误地影响活动计数。 

间隙处理`length = x - prev`是实际累积覆盖范围的地方。 这是唯一“计算”整数点的地方，并且它正确地计算了从 prev 到 x - 1 的所有整数。

 输入：```

```我们举办活动：

 (0, +1), (4, -1), (1, +1), (4, -1), (3, +1), (9, -1)

 排序：

 (0, +1), (1, +1), (3, +1), (4, -2), (9, -1)

 | 步骤| x| 上一页 | 之前活跃 | 段长度| 贡献 | 活动后 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 1 | 0 | - | 0 | - | - | 1 |
 | 2 | 1 | 0 | 1 | 1 | ans[1] += 1 | 2 |
 | 3 | 3 | 1 | 2 | 2 | 答案[2] += 2 | 3 |
 | 4 | 4 | 3 | 3 | 1 | 答案[3] += 1 | 1 |
 | 5 | 9 | 4 | 1 | 5 | 答案[1] += 5 | 0 |

 最后答：

 k=1→6，k=2→2，k=3→1

 此跟踪表明，从未明确计算每个整数的覆盖率。 相反，事件坐标之间的每个稳定区域都被压缩为按其长度加权的单个贡献。

 ## 边缘情况

 一种重要的边缘情况是段退化为单个点。 对于输入：```
1
5 5
```事件变为 (5, +1) 和 (6, -1)。 该算法将区间 [5, 5] 视为 5 和 6 之间的单位长度贡献。活动计数恰好在该区间上变为 1，因此 ans[1] 增加 1，与正好覆盖一个整数点这一事实相匹配。 

当多个事件共享同一坐标时，会出现另一种边缘情况。 例如：```
2
1 3
3 5
```在坐标 3 处，一个线段结束，而另一线段开始。 因为我们在前进之前聚合了每个坐标处的所有增量，所以转换是原子处理的。 用于间隔 [1, 3] 的活动计数是正确的，下一个间隔以更新的活动计数开始，没有任何中间损坏。
