 ## 问题理解

 我们得到一个非负整数数组和一个非常灵活的操作，允许我们将任意数量的值从一个位置移动到另一个位置，只要没有元素变为负数。 每个操作都会选择两个索引并将任意非负数从一个索引传输到另一个索引。 我们最多可以执行这个操作$n$次，但由于每个操作都可以移动一个很大的值，因此该过程的真正威力在于它允许我们几乎任意地重新分配数组的总和。 

经过这些重新分配后，我们最终得到一个总和与原始数组相同的新数组。 目标是选择一种最终的排列方式，最大限度地减少数组中所有元素的按位或运算。 

关键数量是整个数组的或，它仅取决于哪些位出现在至少一个元素中。 任何元素中设置的任何位都会影响最终答案。 

约束条件$n \le 2 \cdot 10^5$意味着任何解决方案本质上都必须是线性或近线性的。 我们无法模拟重新分配操作或搜索配置。 该操作的结构强烈表明，问题在于对总和进行全局推理，而不是模拟走势。 

当除了一个大值之外所有数字都为零时，就会出现一种微妙的边缘情况。 在这种情况下，不需要重新分配，答案就是那个值。 另一个边缘情况是当总和与$n$，其中许多元素可以为零，这表明限制因素是我们如何均匀地分布总和。 

一种幼稚的方法可能会尝试贪婪地重新分配值，同时跟踪按位贡献，但这种模拟会失败，因为可能的数组空间巨大，即使它们共享相同的总和。 

## 方法

 暴力破解的观点是考虑允许的传输的每个可能的序列并计算所得的 OR。 即使将我们限制在最终状态，这也成为将总和的所有非负整数分区枚举为的问题$n$即使金额不大，也是天文数字。 这立即变得不可行。 

关键的观察结果是该操作允许对总和进行完全重新分配，因此唯一的不变性是所有元素的总和。 因此，我们可以自由选择任何数组$n$非负整数之和为$S$， 在哪里$S$是初始总和。 

一旦理解了这一点，问题就简化为选择多组大小$n$与总和$S$最小化所有元素的按位或。 OR 由任何选定数字中出现的最高位模式确定。 如果我们设法确保所有数字都在某个范围内$M$，那么 OR 不能超过$M$，事实上恰好变成了构造值的按位或，我们的目标是使其尽可能小。 

最佳结构是将总和压缩为尽可能少的大的相等块，因为将总和分散得太细会迫使数字变小，但会增加贡献不同位的元素的数量。 最严格的约束就是我们可以使最大元素有多小，同时仍将总和分配给$n$职位。 

由此得出的结论是，答案受最小可能上限的控制$M$这样$S$可以分布在$n$每项数量不超过$M$，这正是$S/n$。

 ## 算法演练

 让$S$是数组中所有元素的总和。 

1. 计算总和$S = \sum a_i$。 这将捕获在允许的操作下保留的所有信息。 
2. 观察任意次数的操作后，我们可以实现任意的配置$n$总和仍为的非负整数$S$。 这使我们能够将焦点从操作转移到纯粹的分区。 
3.我们要构建$n$OR 尽可能小的数字。 数组的 OR 是由任意元素中出现的位的并集确定的，因此控制各个元素的大小可以直接控制 OR。 
4. 考虑强制每个最终元素最多为某个值$M$。 如果我们能够实现这样的配置，那么数组的 OR 不能超过$M$，因为没有元素引入超出的位$M$。 
5.给定的可行性条件$M$是那个$S$最多可以分为$n$部分，最多每个$M$。 在此约束下打包 sum 的最佳方法是使用尽可能多的$M$尽可能多，再加一余数。 这正是可能发生的时候$S \le n \cdot M$。 
6.最小的这样$M$因此是$M = \left\lceil \frac{S}{n} \right\rceil$。 
7. 我们可以建设性地采取$\lfloor S/M \rfloor$元素等于$M$，一个元素等于余数，并用零填充其余元素。 这尊重约束并实现 OR 等于$M$。 

### 为什么它有效

 该算法依赖于这样一个事实：唯一的全局不变量是总和。 一旦我们确定了候选最大值$M$，任何可行的总和分布$n$可以安排零件，以便没有元素超过$M$。 由于按位 OR 在每个元素中都是单调的，因此减少最大可能值会直接使 OR 最小化。 最小的可行上限恰好是最紧的堆积约束$S/n$，因此任何较小的值都将导致无法分配总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

S = sum(a)

# minimal possible maximum value per element
ans = (S + n - 1) // n

print(ans)
```该代码首先读取数组并计算其总和。 最终答案是使用上限除法计算的，它直接对应于重新分配后每个元素的最小可行上限。 不需要模拟操作，因为操作模型允许任意重新分配总和。 

一个常见的陷阱是试图推理原始数字的各个位。 这是不必要的，因为该操作会破坏所有位置结构，同时仅保留总和。

这与最佳答案相匹配，因为任何较小的界限都不允许将 29 个单位分布在 7 个位置上。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 一个关键的边缘情况是当$n = 1$。 算法返回$S$，因为唯一的元素不能以减少 OR 的方式更改。 这是正确的，因为不可能进行重新分配。 

另一种边缘情况是除了一个大值之外所有元素都为零。 总和等于该值，并且上限除法返回相同的值，这与没有重新分配可以减少该值的事实相匹配。 

当总和小于$n$，上限变为 1，意味着某个元素中必须至少出现一个单位。 这正确地反映了我们可以将总和分成许多零和几个一，使得 OR 等于 1$S > 0$。
