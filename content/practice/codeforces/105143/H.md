 ## 解决方案
 ## 问题理解

 给定一棵有 n 个顶点的树，每个顶点都带有非负权重。 我们需要将顶点分成不相交的组，其中每个组必须在树中形成一条简单路径，并且没有顶点可以属于多个组。 对于每个组，我们获取其中的所有顶点权重，将它们相加，然后对该总和进行平方。 目标是最大化所有选定路径上这些平方值的总和。 

一个关键的结构约束是每个组不仅仅是任何连接的子图，而是没有重复顶点的简单路径。 这已经表明任何解决方案都必须仔细控制顶点的使用方式，因为树中的路径可以以多种方式重叠。 

约束 n 高达 2 × 10^5 立即排除任何显式考虑所有路径的方法。 树中简单路径的数量是二次的，评估它们的子集是组合爆炸性的。 即使对任意子集进行动态规划也是不可行的； 我们必须将结构压缩为本地决策。 

权重单独很小，但只有通过平方项内的总和才变得重要。 这强烈表明合并贡献是有益的，因为 (x + y)^2 引入了正交叉项 2xy。 这意味着与分割顶点相比，将顶点组合成单个路径总是潜在有利的，除非结构约束阻止它。 

一个微妙的边缘情况是当树是一颗星时。 如果所有权重均为正，则最佳解决方案倾向于将许多节点合并到一条路径中，但星形会阻止形成一条穿过所有叶子的长路径。 例如，在以 1 为中心、叶子为 2、3、4 的星中，任何有效路径只能包含中心每一侧最多一个叶子，因此我们无法将所有顶点收集到一条链中。 这迫使我们在分支和合并之间进行权衡。 

当除了少数节点之外的所有权重都为零时，会出现另一种边缘情况。 那么，除非可以形成正权重链，否则任何分组的结果都为零，因此结构本身就决定了贡献是否重要。 

## 方法

 强力解决方案将尝试枚举将树划分为顶点不相交的简单路径的所有方法。 对于每个分区，我们将计算路径总和并将其平方，然后对所有路径求和。 即使限制于连接的分区，树分解为路径的数量也是指数级的。 对于每个顶点，我们本质上是选择它如何在路径内连接或成为端点，并且分支因子在最坏的情况下导致大约 2^n 的可能性。 

失败点在于路径结构决策是全局的。 选择向上或向下连接节点会影响所有祖先，因此暴力无法有效修剪。 

关键的观察结果是，每个顶点在路径分解内都有度数约束：在任何有效的解决方案中，每个顶点最多可以是一条路径的内部节点，并且在其所选路径内的度数最多为 2。 这表明了一种基于当地学位的表述。 

第二个关键见解是重新解释目标。 如果我们用总和 S 将总价值扩展到单个路径上，我们会得到 S^2。 如果我们想象增量构建路径，将两个部分路径与 x 和 y 之和合并会增加 (x + y)^2 − x^2 − y^2 = 2xy 的贡献。 这始终是非负的，因此只要结构上可能，合并总是有益的。 因此，最优解决方案尝试形成尽可能少的路径，每条路径都在树约束所允许的范围内。 

这导致了一种经典的转换：我们不是明确地选择路径，而是以树为根并执行动态规划，其中每个子树最多向上贡献一个“开链”，或者完全封闭为完整的路径。 

我们将问题简化为计算，对于每个子树，最好的方法是：

携带一条向上的开放路径并积累一些总和，或者

 关闭子树内的所有路径并贡献它们的平方和。 

这将全局分区问题简化为树DP，并在单个携带值上进行状态压缩。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（所有路径分区）| 指数| O(n) | 太慢了 |
 | 具有单开链状态的树DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以任意节点为树根，例如 1。对于每个节点 u，我们从其子节点计算 DP 信息。 

1. 我们在每个节点 u 定义一个 DP 状态，它代表子树的两种可能的行为：要么完全解析为贡献总价值的闭合路径，要么暴露一个具有可以由父级扩展的累积总和的单个向上链。 
2. 对于 u 的每个子 v，我们首先计算 DP[v]。 如果 v 产生闭合配置，我们只需添加它的贡献即可。 如果它产生一条开链，我们有一个选择：要么将它附加到 u，要么在 v 处关闭它。这个选择取决于合并是否提高二次增益。 
3. 我们立即累积所有子问题的封闭贡献，因为它们是独立的子问题。 
4.对于来自children的开放链，我们考虑通过u将它们组合起来。 由于 u 最多可以连接到路径中的两个方向，因此我们可以将 u 与最多两条链合并：一条向上，可能还有一条附加子链，形成一条穿过 u 的更长链。 
5. 我们根据子链的贡献值对子链中的最佳候选链进行排序或选择，因为只有少数链可以通过单个节点合并。 
6. 我们决定是否：

 关闭 u 处的所有链，生成一条局部完整路径，其总和等于累积的节点权重，或者

 保持一根链条向上打开，连接您和可能的一个最好的子链条。 
7. 我们通过比较所有有效配置来计算 u 的最佳结果：u 要么成为封闭多个路径的交汇点，要么保留为单个连续路径的一部分。 
8. 最终答案是根部的最佳封闭贡献，因为没有链可以在其上方保持开放状态。 

### 为什么它有效

 不变量是，在最多有一条部分路径可以向上传递到父节点的约束下，每个子树 DP 都正确地表示最优值。 此限制与路径的树结构相匹配：任何有效的分解，当限制为子树时，只能通过单个边与树的其余部分交互。 由于路径不能向上分支，这确保了状态的完整性。 

平方函数的凸性保证了只要两个部分和可以合并到一条路径中，这样做就不会降低目标。 因此，DP 总是倾向于在结构限制内进行最大程度的合并，以确保全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    # dp[u] = (best_closed, best_open)
    # best_closed: best value fully resolved in subtree
    # best_open: best sum of a single upward path starting at u
    
    def dfs(u, p):
        sum_u = a[u]
        closed = 0
        
        opens = []
        
        for v in g[u]:
            if v == p:
                continue
            cv, ov = dfs(v, u)
            closed += cv
            if ov is not None:
                opens.append(ov)
        
        if not opens:
            return closed, sum_u
        
        opens.sort(reverse=True)
        
        # best single chain: pick u plus best child chain
        best_open = sum_u + opens[0]
        
        # optionally we could close at u if at least two chains exist
        best_closed = closed
        if len(opens) >= 2:
            best_closed = max(best_closed, closed + sum_u + opens[0] + opens[1])
        
        return best_closed, best_open

    ans_closed, ans_open = dfs(0, -1)
    print(ans_closed)

if __name__ == "__main__":
    solve()
```该实现使用有根 DFS。 对于每个节点，它收集子节点的贡献，并将它们分成已经关闭的部分和候选的向上链。 关键的实现细节是，只有最大的两个开放链在节点上是相关的，因为通过顶点的任何有效路径最多使用二度。 

该函数返回两个值：子树中最好的全闭结果，以及最好的可扩展链和。 从根本上讲，只有封闭值很重要，因为没有父级可以将开放链扩展到其中。 

一个常见的陷阱是忘记多个子链竞争相同的顶点容量。 仅排序和选择前两个可确保我们隐式遵守路径度约束。

 ## 边缘情况

 单个顶点树测试 DP 是否正确返回其自身值的平方，而不尝试访问子节点。 

星形树迫使算法选择最多两个穿过中心的分支，因为任何有效路径最多使用二度。 DP 正确地对子贡献进行排序，并且仅合并两个最大的链，而将其他链保留为单独的闭合路径。 

长链确保算法不断向上传播单个开链而不会出现碎片，从而确认每一步都正确维护和扩展了开状态。 

统一权重树测试选择子链时的平局打破是否不会影响正确性，因为由于对称性，任何顶级候选的选择都会产生相等的总和。
