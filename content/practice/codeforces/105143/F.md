 ## 解决方案
 ## 问题理解

 我们被赋予了一个隐藏的$n \times n$网格填充范围内的正整数$[1, n^2]$。 网格不是任意的：值在两个方向上都是单调的，这意味着当我们向右或向下移动时，它们永远不会减少。 所以每一行都是非递减的，每一列都是非递减的。 

我们无法直接看到网格。 相反，我们可以查询单个单元格$(i, j)$有门槛$x$，法官告诉我们该单元格的隐藏值是否最多为$x$。 每个查询仅显示与所选阈值的布尔比较。 

任务是确定$k$- 所有值中的最大值$n^2$最多使用 50000 个此类查询的条目。 

这些约束意味着很大的搜索空间：$n$最多可达 1000，因此矩阵最多包含一百万个值。 线性扫描是不可能的，因为每个单元都不能直接读取，并且由于交互作用，每次检查都已经很昂贵。 任何解决方案都必须显着减少探针的数量并尽可能地重用每个查询。 

一个幼稚的错误是将其视为对值的简单二分搜索，同时独立查询每个单元格。 例如，如果我们尝试通过检查每个单元格的每个值阈值来确定完整排序的列表，我们将需要$O(n^2 \log n)$查询，远远超出了限制。 

另一个微妙的陷阱来自于忽视单调性。 例如，如果只查询任意单元而不利用结构，则相同的值或平坦区域可能会导致重复的不必要的搜索，因为多个单元可以共享相同的阈值行为。 

## 方法

 思考这个问题的一个直接方法是认识到每个查询都回答这个问题“这个位置是否位于值集合中”$\le x$”。如果我们修复$x$，我们可以从概念上计算有多少个细胞$\le x$，然后将此计数与$n^2 - k$来决定是否$x$足够大来作为答案。 

暴力方法将尝试独立评估每个候选人的计数$x$。 这需要查询每个单元格以确定它是否是$\le x$，导致$n^2$每个值检查的查询。 如果我们对中的值进行二分搜索$[1, n^2]$，我们得到$O(n^2 \log n)$查询，这是不可行的$n = 1000$。 

关键的结构观察是矩阵在行和列方向上都排序。 这意味着如果我们固定一个阈值$x$，满足的单元格集合$a_{i,j} \le x$形成左上角的连续区域：如果单元格有效，则上方和左侧的所有内容也有效。 这将计数变成了几何遍历问题，而不是独立的单元格检查。 

我们可以通过将某个位置处的查询视为对该单调区域边界的探测来利用这一点。 我们不是独立测试每个单元，而是重用空间关系：一旦我们知道某个区域低于或高于阈值行为，我们就会立即消除大块。 

标准归约是将问题转化为寻找最小值，使得至少$n^2 - k + 1$元素是$\le x$。 这变成了对值的二分搜索，其中每个检查都是单调谓词。 挑战是在查询限制下有效地实现谓词。 

为了评估谓词，我们从左下角执行引导遍历。 由于行和列已排序，向右移动会增加值，向上移动会减少值。 每个查询都会根据响应消除整个行段或列段，因此每个步骤都会丢弃至少一个行或列方向，使每次检查的总查询数限制为$O(n)$。 

这产生了总复杂度$O(n \log n)$查询量，正好适合 50000 次以下$n \le 1000$。

 ## 算法演练

 1. 将任务解释为寻找最小值$x$这样至少$n^2 - k + 1$细胞满足$a_{i,j} \le x$。 这会将订单统计量转换为计数谓词。 
2. 定义函数`count_leq(x)`计算矩阵中有多少个元素$\le x$使用交互式查询。 
3. 计算`count_leq(x)`, 从位置开始$(n, 1)$，左下角。 选择此值是因为它允许确定性移动：值向右增加并向下增加。 
4、每一步查询当前位置$(i, j)$有门槛$x$。 如果答案为 1，则值为$\le x$，所以同一列中的所有内容也是$\le x$, 贡献$i$该列段中的有效元素。 然后向右移动到$j + 1$因为当前列已完全解析为该行前缀。 
5. 如果答案为0，则该值超过$x$，因此该行右侧的所有内容也无效。 我们向上移动到$i - 1$，丢弃当前单元格并继续以较小的值搜索。 
6. 每当确认列段时就累积贡献，确保计数区域之间没有重叠。 
7. 对取值范围进行二分查找$[1, n^2]$, 呼叫`count_leq(mid)`每次。 谓词是单调的，因为增加$x$只能增加有效单元格的数量。 
8.返回最小的$x$这样`count_leq(x) >= n^2 - k + 1`。 

### 为什么它有效

 正确性依赖于矩阵的单调几何形状。 在任何固定阈值$x$，有效单元格集合形成右下封闭区域。 遍历模拟该区域上的边界行走：每个查询消除完整的列贡献或无法再次与区域边界相交的行段。 因为每个步骤严格向左或向上/向右移动，所以没有单元被处理两次，并且每次检查的查询总数与$n$。 二分查找是有效的，因为谓词“元素的数量$\le x$” 不减$x$，确保不会错误地跳过答案范围。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())

def count_leq(x):
    i, j = n, 1
    cnt = 0
    while i >= 1 and j <= n:
        print("?", i, j, x)
        sys.stdout.flush()
        res = int(input())
        if res == 1:
            cnt += i
            j += 1
        else:
            i -= 1
    return cnt

lo, hi = 1, n * n
ans = hi

while lo <= hi:
    mid = (lo + hi) // 2
    if count_leq(mid) >= n * n - k + 1:
        ans = mid
        hi = mid - 1
    else:
        lo = mid + 1

print("!", ans)
sys.stdout.flush()
```核心实现围绕`count_leq`例行公事，这是与法官互动的唯一部分。 遍历使用左下起点，以便每个查询确定性地删除行前缀或列。 积累`cnt += i`之所以有效，是因为当一个细胞$(i, j)$确认有效，则同一列中其上方的所有单元格在单调性下也有效。 

二分搜索在值空间而不是索引空间上进行操作，这避免了重建整个矩阵的需要。 差一边界是通过将第 k 大的等级转换为第 k 小的等级来处理的，具体来说$n^2 - k + 1$。

 ## 边缘情况

 一个最小的案例$n=1$有一个单细胞。 立即遍历查询$(1,1)$一次，二分查找收敛到该值。 该算法不会进入无限循环，因为指针移动总是会减少搜索空间。 

统一矩阵（其中每个条目都相同）会导致每个查询返回 1。遍历向右移动直至耗尽，正确累积所有列贡献。 由于每个阈值的行为都相同，二分搜索很快就会崩溃为单个重复值。 

在严格递增矩阵中，值在两个维度上都快速增加，许多早期查询返回 0，强制向上移动，直到找到有效边界。 每次失败的比较仍然会消除整个行前缀或列后缀，确保遍历逐步保持线性，而不是退化为完全扫描。
