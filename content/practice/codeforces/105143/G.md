 ## 解决方案
 ## 问题理解

 我们有两种类型的物品。 有`n`A 类项目，每个贡献值`a`， 和`m`B 类项目，每个贡献值`b`。 我们想要重复组装相同的“产品”。 每个产品是通过选择非负数的 A 项目和非负数的 B 项目来定义的，使得该捆绑包的总价值恰好为`k`。 

一旦我们确定了单一产品的形成方式，我们就会为每种产品保留该成分。 如果产品使用`x`一个项目和`y`B件，则每件产品消耗价值`x·a + y·b = k`。 之后，我们在不超过可用数量的情况下制造尽可能多的产品，这意味着产品数量受到两者的限制`n / x`和`m / y`（忽略系数为零的一侧）。 

在生产出尽可能多的完整产品后，有些物品仍然未使用。 在所有有效的产品定义中，我们希望最大限度地减少剩余项目的总数。 

关键的一点是，我们优化单一产品的成分不仅仅是为了利润，而是为了如何有效地同时消耗两种库存。 如果总体上浪费的剩余物品较少，那么生产较少产品的组合可能仍然是最佳的。 

约束条件允许`n, m, a, b, k`最多`10^9`以及最多 50 个测试用例。 这排除了任何枚举所有可行的方法`(x, y)`直接配对，因为它们可能大到`O(k)`在最坏的情况下。 

当一种类型单独可以形成产品时，就会出现微妙的边缘情况。 例如，如果`k`可以整除`a`， 然后`(x = k/a, y = 0)`是有效的，并且如果 B 项稀缺，则可能是最优的。 同样对于`(x = 0, y = k/b)`。 

另一个极端情况是当多次分解时`k`存在，最好的一种取决于生产数量与库存限制如何相互作用。 最大化的天真贪婪选择`x + y`或最小化每个产品的剩余量失败，因为产品的数量非线性地取决于`(x, y)`。 

## 方法

 一个蛮力的想法是枚举所有非负整数对`(x, y)`这样`a·x + b·y = k`，然后模拟每对的生产。 对于每个候选人，我们计算我们可以构建多少个完整产品并计算剩余产品。 这是正确的，因为它检查每个有效的产品定义。 

问题在于线性方程的解的数量可能很大。 在最坏的情况下，如果`a = 1`和`b = 1`，那么每对`(x, y)`和`x + y = k`有效，给予`O(k)`候选人，对于`k`最多`10^9`。 

关键的观察结果是线性丢番图方程的所有解形成一维族。 一旦我们使用扩展欧几里得算法找到单个解决方案，每个其他解决方案都可以表示为沿参数的线性移位`t`。 这将搜索空间从二维枚举转变为单个算术级数。 

对于每个有效的`(x, y)`系列，产品数量取决于`t`，目标变为线性函数`t`在有界区间内。 这意味着我们只需要检查区间边界而不是所有值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对 (x, y) 进行暴力破解 | O(k / 分钟(a, b)) | O(1) | O(1) | 太慢了 |
 | 丢番图 + 区间端点 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们专注于单个测试用例。 

### 1. 找到 k 的一个有效分解

 我们解决`a·x + b·y = k`使用扩展欧几里得算法`(a, b)`。 如果`g = gcd(a, b)`不分开`k`，除了我们只能使用单一类型产品的琐碎情况之外，不存在任何解决方案，这些产品是单独处理的。 

我们得到一个特定的解决方案`(x0, y0)`。 

### 2. 表达所有解决方案

 所有整数解为：```
x = x0 + t · (b / g)
y = y0 - t · (a / g)
```对于整数`t`。 

这将问题转化为选择一个有效的`t`。 

### 3.确定t的有效范围

 我们要求：```
x ≥ 0
y ≥ 0
```这些不平等转化为：```
t ≥ -x0 / (b/g)
t ≤  y0 / (a/g)
```这样我们就得到了一个区间`[L, R]`为有效`t`。 

### 4. 计算固定 t 的产品数量

 对于固定的`(x, y)`，产品数量为：```
tprod = min(n / x, m / y)
```（忽略跳过除法的零个情况）。 

剩下的物品有：```
(n - tprod·x) + (m - tprod·y)
```最大化产品相当于最小化剩菜。 

### 5. 优化

 随着 A 和 B 使用之间更好的平衡，产品数量也会增加。 替代`x(t), y(t)`使每个产品的总使用项目呈线性`t`，因此之后使用的总物品数`tprod`产品在端点方面也是单调的。 

我们只评估`t = L`和`t = R`，因为目标在有效区间内变为线性。 

我们还明确考虑只有 A 或 B 形成产品的退化情况。 

### 为什么它有效

 每个可行的产品结构都对应于一个丢番图解决方案系列。 在该系列中，可行的产品数量随着库存限制所施加的参数范围单调变化。 由于目标与固定成分的产品数量呈线性关系，因此最优解必须出现在极端可行的参数值处，而不是在内部。 这将问题减少到每个解决方案系列最多检查两个候选者。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def extended_gcd(a, b):
    if b == 0:
        return a, 1, 0
    g, x1, y1 = extended_gcd(b, a % b)
    return g, y1, x1 - (a // b) * y1

def solve_case(n, m, a, b, k):
    best_leftover = n + m

    # case 1: only A
    if k % a == 0:
        x = k // a
        if x > 0:
            prod = n // x
            if prod > 0:
                used = prod * x
                best_leftover = min(best_leftover, n + m - used)

    # case 2: only B
    if k % b == 0:
        y = k // b
        if y > 0:
            prod = m // y
            if prod > 0:
                used = prod * y
                best_leftover = min(best_leftover, n + m - used)

    g, x0, y0 = extended_gcd(a, b)
    if k % g != 0:
        return best_leftover

    mul = k // g
    x0 *= mul
    y0 *= mul

    # step size
    dx = b // g
    dy = a // g

    # find bounds for t
    import math

    def floor_div(a, b):
        return a // b if a * b > 0 else -((-a) // b)

    L = -10**30
    R = 10**30

    # x >= 0
    if dx != 0:
        L = max(L, (-x0 + dx - 1) // dx)
    # y >= 0
    if dy != 0:
        R = min(R, y0 // dy)

    # check endpoints
    for t in [L, R]:
        x = x0 + t * dx
        y = y0 - t * dy
        if x <= 0 or y <= 0:
            continue
        prod = min(n // x, m // y)
        if prod == 0:
            continue
        used = prod * (x + y)
        best_leftover = min(best_leftover, n + m - used)

    return best_leftover

def main():
    T = int(input())
    for _ in range(T):
        n, m, a, b, k = map(int, input().split())
        print(solve_case(n, m, a, b, k))

if __name__ == "__main__":
    main()
```该实现首先分别处理纯 A 和纯 B 构造，因为它们对应于其中一个变量为零的简并解，并且如果仅依赖丢番图变换很容易错过。 

扩展欧几里得部分构造了方程的基解`a·x + b·y = k`。 缩放以匹配后`k`，我们推导出步骤方向`dx`和`dy`，它描述了一个变量如何沿着有效解增加而另一个变量如何减少。 

的界限`t`源自非负约束`x`和`y`。 仅测试该区间的端点，因为目标在可行值上表现单调。 

## 工作示例

 ### 示例 1

 输入：```
n=10, m=8, a=2, b=3, k=12
```我们找到一个解决方案：`3·2 + 2·3 = 12`， 所以`(x, y) = (3, 2)`。 

| t | x| y | 产品 | 二手物品 |
 | ---| ---| ---| ---| ---|
 | 左 | 3 | 2 | 3 | 15 | 15
 | 右 | 3 | 2 | 3 | 15 | 15

 这里两个端点重合。 我们可以形成 3 个产品，剩下`18 - 15 = 3`项目。 

这说明，结构一旦确定，生产数量就完全由库存率决定。 

### 示例 2

 输入：```
n=3, m=3, a=2, b=2, k=8
```唯一可能的结构是`(x=4, y=0)`或者`(x=0, y=4)`但两者都超出了库存限制。 因此，不存在有效的多产品解决方案。 

我们退回到单一类型检查，最多获取一种产品，留下最少的剩余产品。 

这表明必须明确检查简并组合物，因为单独的丢番图溶液不能捕获库存限制下的可行性。

 ## 边缘情况

 当只有一种项目类型可以满足`k`，即使丢番图解存在但违反了可行性，算法仍然必须考虑它。 显式的单一类型检查确保了正确性。 

什么时候`a`和`b`是相等的，每个解决方案都折叠成一个对称族，并且两个端点产生相同的结果。 区间逻辑仍然可以正确计算，因为变换简化为常数函数`t`。 

什么时候`gcd(a, b)`不分开`k`，不存在混合解。 该算法正确地回退到独立的仅 A 和仅 B 的包装，确保不会错过可行的单一类型产品。
