 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，并允许重复执行非常灵活的操作：选择任何连续的段并循环旋转它。 这里的旋转意味着将所选片段分成两部分并交换它们的顺序，同时保留每个部分内的内部顺序。 

目标是将字符串转换为单调形式，其中所有零在前，所有在后，使用尽可能少的此类操作。 

约束允许字符串长度最大为 100,000，因此任何解决方案都必须本质上是线性或接近线性的。 模拟操作或尝试所有子串的二次策略是立即不可行的，因为即使是对所有子串的单个操作也已经是$O(n^2)$，并且探索操作序列将会以组合方式爆炸。 

一个微妙的困难是一个操作非常强大。 它可以将整个字符块移动到所选段内的另一个块上，因此该过程不是简单的相邻交换模型。 这使得忽略全局结构的贪婪局部修复变得不可靠。 

一个常见的失败案例来自于假设每个反转都可以独立修复。 例如，在像这样的字符串中`1010`，一个天真的想法可能是两次反转需要两次操作，但对整个字符串的一次旋转已经可以同时重新排列多个放错位置的字符。 实际成本取决于连续运行的结构而不是单个对的结构。 

另一个棘手的情况是所有零已经形成后缀或所有1已经形成后缀的字符串。 例如，`000111`需要零操作，但是`001110`看起来仍然几乎排序，同时需要至少一次移动将尾随零重新定位到前缀区域。 

核心挑战是确定一种结构性措施，准确捕获必须纠正多少独立的“错位块”。 

## 方法

 如果我们尝试直接模拟该过程，我们将考虑所有子串和所有可能的旋转，对段和旋转偏移的选择进行分支。 每个操作都可以通过多种方式转换一个段，因此即使一个步骤也会产生很大的分支因子。 即使我们将自己限制在贪婪的选择上，我们仍然需要评估每个操作如何改变全局无序，这导致每一步至少进行二次扫描。 

关键的观察是我们实际上不需要跟踪段内的任意排列。 最终的目标形状非常严格：所有零必须位于左侧，所有1都位于右侧。 这意味着每个操作只有在将一组 1 移动到零区域时才有用。 在最大的 1 块内，内部顺序是无关紧要的。 

这将视角从单个角色转移到连续的运行。 一旦我们将字符串压缩为相同字符的串，我们就会发现只有出现在最后的零区域之前的串才需要移动。 每个这样的运行就像一个必须重新定位到右侧的单元。 

仔细选择的段上的单个循环移位可以在一组零上重新定位一个这样的“错位的单游”，但它不能同时修复与零交错的多个单独的单游。 这使得每次这样的运行都独立地对答案做出贡献。 

因此，问题简化为计算连续运行了多少次`1`s 出现在其右侧仍存在零的区域。

 ## 算法演练

 我们通过将字符串压缩为相同字符的连续块并仅推理这些块的结构来解决该问题。 

1. 扫描字符串并识别相同字符的所有最大连续段。 每个段要么是由 0 组成的块，要么是由 1 组成的块。 
2. 找到字符串中最后一个零的位置。 该位置右侧的所有内容都必须完全由最终排序形式的内容组成，因此完全在这一点之后的任何一个块都已经正确放置。 
3. 迭代所有连续的 1 块。 对于每个单块，检查它是否从最后一个零位置或之前开始。 
4. 将每个这样的一个块算作贡献一个操作。 
5. 返回此计数作为答案。 

此过程起作用的原因是，出现在其右侧某处零之前的一块与其正确的最终区域至少相隔一个零。 如果没有专门的操作将整个块移过这些零，就无法解决这种分离问题。 由于循环移位每次操作仅重新定位一个连续的块，因此每个这样的块都需要自己的移动。 

### 为什么它有效

 不变的是，在每次操作之后，所选段外的字符的相对顺序保持不变，而段内仅执行一次循环剪切。 这意味着一个块只能作为一个整体移动到相邻的零区域，而不能在一个步骤中合并或拆分为多个独立的重定位。 因此，每个被右侧至少一个零“阻塞”的1块必须被独立解析，并且任何操作都不能将此类阻塞块的计数减少超过1。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    last_zero = -1
    for i, ch in enumerate(s):
        if ch == '0':
            last_zero = i

    if last_zero == -1:
        print(0)
        return

    ans = 0
    i = 0
    while i < n:
        if s[i] == '1':
            j = i
            while j < n and s[j] == '1':
                j += 1
            if i <= last_zero:
                ans += 1
            i = j
        else:
            i += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先计算最后一次出现的零，这定义了必须全部为零的区域和必须全部为1的区域之间的边界。 然后它以线性时间扫描字符串，将连续的字符串分组为运行。 

一个微妙的点是条件`i <= last_zero`。 这确保我们只计算未完全包含在最终后缀区域中的 1 块。 任何严格在最后一个零之后开始的单块已经位于最终排列的正确一侧，并且不需要任何操作。 

其余逻辑是带有指针跳转的标准线性扫描，以避免在运行中重新处理字符。

 | 步骤| 运行开始| 运行结束| 类型 | 开始 ≤ 最后一个零 | 计数 |

 ## 边缘情况

 对于仅由 1 组成的字符串，最后一个零位置是`-1`，因此算法立即返回零。 这与字符串已经处于排序形式的事实相匹配。 

对于仅由零组成的字符串，不存在单块，因此扫描永远不会增加答案。 正如预期的那样，输出为零。 

对于所有已经在后缀中的情况，例如`000111`，最后一个零位于两个区域之间的边界，所有的一块都在它之后开始，所以没有一个被计算在内。 

对于交替字符串，例如`010101`，每个一块都位于最后一个零之前，因此每个块都独立贡献，匹配每次运行都必须单独修复的最坏情况行为。
