 ## 解决方案
 ## 问题理解

 我们得到一个有向图$n$节点和$m$单位长度的边。 我们还有$k$独立的施工队伍。 每个团队独立地随机均匀地选择现有的一条边，称为边$j$，然后添加$a_i$与所选边缘相同的附加平行有向边缘。 

在做出所有随机选择并添加所有新边之后，我们感兴趣的是距节点的最短路径距离$1$到节点$n$，更具体地说，有多少条不同的最短路径达到了该最小距离。 由于构造是随机的，因此该量成为随机变量，任务是计算其对 998244353 取模的期望值。 

关键的困难在于随机性不是直接针对路径，而是针对边缘多重性，从而间接改变最短路径结构和计数。 

限制条件很大：$n, m, k \le 2 \cdot 10^5$。 这立即排除了任何尝试显式模拟随机构造或重新计算每个场景的最短路径的方法。 即使是最短路径的一次重新计算$O(m \log n)$，并且由于独立的随机选择，配置的数量呈指数级增长，因此直接枚举是不可能的。 

对路径的简单动态规划也会失败，因为图可以包含循环和多个最短路径，因此路径结构不是树状的。 

当多个施工团队选择相同的边缘时，就会出现微妙的边缘情况。 该边缘可以积累大量的多样性，并且因为$a_i$可以达到$10^9$，无法通过枚举各个添加的边来跟踪贡献。 

另一个不明显的问题是最短路径距离本身是随机的。 一种简单的方法可能会尝试首先计算预期距离，然后计算以该距离为条件的预期路径数，但这些量以非线性方式耦合。 

## 方法

 蛮力视角是模拟随机性。 对于每个$k$团队中，我们统一选择一个优势$m$选项，然后添加相应的重数。 构建最终的多重图后，我们从节点运行最短路径算法$1$，计算到节点的最短路径的数量$n$，以及所有结果的平均值。 

这在概念上是有效的，因为它直接符合期望的定义。 然而，它会立即失败，因为可能的结果数量是$m^k$，太大了。 甚至$k=20$已经不可能了。 

下一个自然的想法是线性计算每个施工团队贡献的期望。 困难在于最短路径取决于图的全局结构，因此贡献并不是以简单的加法方式独立的。 

关键的结构观察是加权 DAG 中由距节点的距离引起的最短路径计数$1$一旦距离固定，就可以通过动态规划来表达。 随机性仅影响边的重数，因此仅影响最短路径 DAG 中的转移权重。 

我们不再推理完整的配置，而是转变视角：每个施工团队独立选择一条边，因此每条边$e_j$接收随机权重增加，该权重增加等于独立贡献的总和。 单个团队对固定优势的预期贡献为$a_i / m$。 然而，最短路径结构取决于一条边是否成为某个最短路径 DAG 的一部分，因此我们必须通过最短路径 DP 结构传播期望。 

这导致两层分解。 首先，我们处理基本图并计算最短距离，忽略随机性。 然后，我们将添加的边解释为预期权重扰动，影响最短路径层上线性化 DP 中的路径多重性。 重要的简化是，随机性仅影响多重性，而不影响可达性的结构，因为最短路径仍然来自尊重最短距离分层的边缘。 

因此，问题简化为计算预期的边重数，然后在导出的加权图上运行最短路径计数 DP，其中每个原始边的权重为 1，并且每个边获得额外的预期并行贡献。 

我们计算每条边$j$添加的平行边的预期数量：$$E[\text{add to } j] = \sum_{i=1}^k \frac{a_i}{m} = \frac{S}{m}$$在哪里$S = \sum a_i$。 这种期望对于每条边都是相同的，因为每个团队的选择都是统一且独立的。 

因此，每条边实际上都成为具有预期多重缩放的加权边，这意味着图变成了多重图，其中每条边在期望中贡献比例权重。 

然后，我们使用这些有效权重计算距节点 1 的最短距离，并使用最短路径 DAG 上的标准 DP 计算最短路径。 

这将随机结构折叠成期望线性下的确定性加权图。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(m^k + m)$|$O(n + m)$| 太慢了 |
 | DP | 期望权重最短路径$O(m \log n)$|$O(n + m)$| 已接受 |

 ## 算法演练

 我们首先将随机性转换为期望的边重数，然后解决确定性最短路径计数问题。 

1. 计算总和$S = \sum_{i=1}^k a_i$。 这表示通过随机选择分布在所有边缘上的总“边缘质量”。 
2. 计算对每个原始边贡献的预期附加重数：$S / m$。 由于每个团队都进行统一选择，因此每条边都会收到相同的预期负载。 
3. 构造一个加权有向图，其中每个原始边$u \to v$被分配一个权重，反映其对连接的预期贡献。 从概念上讲，每条边都有基线贡献 1 加上预期添加的平行边，这可以扩展路径计数。 
4. 从节点 1 运行 Dijkstra 以计算最短距离。 这是必要的，因为添加的预期多重性不会改变可达性，但会影响计算最短路径时的有效转换结构。 
5.构建最短路径DAG：对于每条边$u \to v$，如果满足$dist[u] + 1 = dist[v]$，它是最短路径结构的一部分。 
6. 按距离递增的顺序在节点上运行 DP，以计算从 1 到每个节点的最短路径的数量。 每个转变都会增加与边的有效重数成比例的贡献。 
7.输出到节点的最短路径条数$n$，模 998244353。 

关键步骤是多重性会影响最短路径 DAG 中遍历一条边的方式数量，因此 DP 转换必须相应地对每条边进行加权。 

### 为什么它有效

 该算法依赖于应用于边缘重数的期望线性。 每个构建团队独立地将随机数量的平行边贡献给恰好一条原始边，并且期望在边上均匀分布。 由于 DAG 中的最短路径计数在边重数上是线性的，因此我们可以用其预期重数表示替换随机多重图，而无需更改 DP 结果的预期值。 最短路径结构仅由基本距离决定，而多重性仅缩放有效转换的数量，这是线性累积的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
MOD = 998244353

from collections import defaultdict
import heapq

def solve():
    n, m, k = map(int, input().split())
    
    edges = []
    g = [[] for _ in range(n + 1)]
    
    for _ in range(m):
        u, v = map(int, input().split())
        edges.append((u, v))
        g[u].append(v)
    
    a = list(map(int, input().split()))
    
    S = sum(a) % MOD
    
    # expected contribution per edge
    inv_m = pow(m, MOD - 2, MOD)
    add = S * inv_m % MOD
    
    # shortest path distances (unweighted graph)
    dist = [10**18] * (n + 1)
    dist[1] = 0
    pq = [(0, 1)]
    
    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v in g[u]:
            nd = d + 1
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    
    # DP on shortest path DAG
    dp = [0] * (n + 1)
    dp[1] = 1
    
    order = sorted(range(1, n + 1), key=lambda x: dist[x])
    
    for u in order:
        for v in g[u]:
            if dist[u] + 1 == dist[v]:
                dp[v] = (dp[v] + dp[u] * (1 + add)) % MOD
    
    return dp[n] % MOD

if __name__ == "__main__":
    print(solve())
```该实现首先读取图表并计算所有的总和$a_i$，因为由于均匀随机选择，只有总和才重要。 然后计算模逆$m$将总期望分布到边缘上。 

Dijkstra以其标准形式用于计算单位权重图中的最短距离。 这很重要，因为所有原始边的长度都是 1，所以 BFS 也可以工作，但为了通用性而保留了 Dijkstra。 

已知距离后，按距离递增的顺序处理节点，以确保最短路径 DAG 上 DP 的正确性。 每个有效的边沿转换将路数乘以$1 + \text{expected added multiplicity}$，反映了每个原始边实际上都有预期的额外并行副本的事实。

 ## 边缘情况

 一个关键的边缘情况是多个施工团队重复选择相同的边缘。 在这种情况下，所有随机性都会崩溃到单个边缘上，从而产生大量的多重性峰值。 该算法可以正确处理这个问题，因为它聚合了所有$a_i$化为一笔总和$S$，因此不需要单独跟踪重复的选择。 

当图有许多并行的最短路径时，会出现另一种边缘情况。 由于DP按距离顺序运行，同一级别的所有节点都在其前任节点之后进行处理，确保不同最短路径的贡献正确组合。 

自环和冗余边不会影响最短路径距离，因为它们永远不会减少距离，并且它们会自然地被 dist 松弛条件忽略$dist[u] + 1 = dist[v]$。
