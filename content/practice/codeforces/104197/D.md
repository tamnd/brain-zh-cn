 ## 解决方案
 ## 问题理解

 我们得到了假设图中节点之间成对距离的完整描述，但只有这些距离的奇偶性才重要。 对于每一对顶点，我们知道它们的最短路径距离是奇数还是偶数，任务是确定是否存在至少一个无向图，其实际最短路径距离产生完全相同的奇偶校验模式。 

因此，输出是一个二元决策：要么我们可以构造一些图，其最短路径距离奇偶性与每对的给定规范相匹配，要么我们不能。 

关键的困难在于，我们不是构建具有加权自由度的特定图，而是验证所有对最短路径上的全局奇偶约束是否与某些底层未加权图结构一致。 

原始问题陈述隐含的约束表明三次 Floyd-Warshall 式推理是可能的，因此节点数量足够小，可以接受对所有对进行推理或运行所有对最短路径松弛。 这立即排除了需要比 O(n^3) 更糟糕的方法，并强烈表明成对一致性检查或传递闭包式推理是核心。 

当局部奇偶校验约束一致但全局不可能嵌入到图形度量中时，就会出现微妙的失败情况。 

必须检测的这种不一致的一个简单例子是三个节点形成一个矛盾奇偶校验要求的循环。 例如，假设我们有节点 1、2、3，所需的奇偶性意味着 dist(1,2) 是奇数，dist(2,3) 是奇数，但 dist(1,3) 也是奇数。 在任何未加权图中，链接在一起的两个奇数长度最短路径通常通过串联在端点之间强制执行偶奇偶校验，因此这种配置可能可行也可能不可行，具体取决于中间结构。 该算法必须检测是否存在像这样的全局矛盾。 

## 方法

 强力解释是尝试构建一个图，然后验证其最短路径距离是否与所需的奇偶校验矩阵匹配。 人们可以尝试所有可能的边，计算最短路径，并检查奇偶校验一致性。 即使我们将自己限制在未加权的图上，图的搜索空间也是 2^{n(n-1)/2}，甚至验证一个候选者也需要进行全对最短路径计算。 这是完全不可行的。 

一种更结构化的暴力方法是假设一个候选图，使用每个节点的 BFS 计算所有对最短路径，然后比较奇偶校验矩阵。 这将每个候选图的检查减少到 O(n(n + m))，但仍然留下了构建或搜索图的问题。 

关键的观察是我们实际上不需要显式地构建图。 问题陈述本身暗示了一种建设性的变换：对于所需距离奇偶性为奇数的每一对顶点，我们可以想象在它们之间添加一条边。 这不一定是最终的图，但它创建了一个规范的超图，以有用的方式保留奇偶校验约束。 

关键思想是奇偶校验约束的行为类似于最短路径结构上的二分条件。 如果我们解释距离 mod 2，那么每条边都会翻转奇偶校验，并且最短路径奇偶校验就相当于两个节点相对于某个 BFS 树是否位于相同或相反的奇偶校验层。 这表明该结构从根本上检查隐含的奇偶关系是否与最短路径层引起的有效 2 着色一致。

一旦我们将“奇数距离对”视为需要奇偶闭合意义上的邻接，我们就可以将问题简化为检查生成的导出图是否一致，这可以归结为 BFS 分层下的连通性和奇偶一致性。 这可以通过运行 Floyd-Warshall 或多源 BFS 奇偶校验传播并检查矛盾来验证。 

最后的简化是，我们可以将奇偶校验约束视为标有 0/1 的完整图，并尝试将节点嵌入到奇偶校验等于最短路径奇偶校验的图形度量中。 这相当于检查最短路径层的一致分配是否存在，这可以通过闭包一致性在 O(n^3) 中进行验证。

 | 弗洛伊德-沃歇尔平价一致性 | O(n^3) | O(n^3) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们将给定的奇偶校验矩阵视为最短路径距离上的约束系统。 

1. 初始化一个矩阵`d`在哪里`d[i][j]`如果所需的奇偶校验为偶数，则为 0；如果所需的奇偶校验为奇数，则为 1。 该矩阵表示我们希望在某些图中使用最短路径奇偶性满足的约束。 
2. 运行类似于 Floyd-Warshall 的收敛过程，但基于奇偶可行性而不是数字距离进行操作。 对于每一个三元组`(i, j, k)`，我们解释如果有一条最短路径`i`到`k`和来自`k`到`j`，则奇偶性来自`i`到`j`必须与沿路径奇偶校验之和一致。 
3. 通过强制传递性更新约束：如果我们知道奇偶关系`i -> k`和`k -> j`， 然后`i -> j`必须等于`(i -> k + k -> j) mod 2`。 如果先前设置的值与该派生值相矛盾，则无法进行配置。 
4. 通过将奇奇偶校验隐含的边缘视为连接强制关系，同时确保连接一致性。 如果任何节点在隐式结构下变得不可达，则该配置无效。 
5. 完成闭包后，检查所有三元组中的所有约束是否一致。 如果没有发现矛盾，则输出YES，否则输出NO。 

为什么它有效

 任何未加权图中最短路径的奇偶校验的行为类似于模 2 缩减的度量。这会产生一个约束系统，其中每个中间节点都强制沿路径奇偶校验的可加性。 Floyd-Warshall 式闭包枚举了可以定义最短路径的所有可能的中间节点。 如果奇偶校验约束是可实现的，那么所有中间体下的闭包必须保持一致。 在关闭过程中发现的任何矛盾都对应于奇偶校验约束的不可能循环，该循环无法嵌入到任何图形度量中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    g = [list(map(int, input().split())) for _ in range(n)]

    # interpret as parity constraints (0 = even, 1 = odd)
    d = [[g[i][j] % 2 for j in range(n)] for i in range(n)]

    # consistency matrix
    ok = True

    for k in range(n):
        for i in range(n):
            for j in range(n):
                via = (d[i][k] + d[k][j]) & 1
                if d[i][j] != via:
                    # if already assigned and conflict arises, impossible
                    ok = False
                d[i][j] = via

    print("YES" if ok else "NO")

if __name__ == "__main__":
    solve()
```该代码首先将所有给定的成对值减少为奇偶校验信息，因为只有奇数或偶数性质才重要。 然后，它应用三重循环闭合，将每个节点视为最短路径分解中的潜在中间点。 每次迭代都强制奇偶校验必须通过中间节点一致地传播。 如果检测到任何矛盾，系统将被标记为无效。 

作业`d[i][j] = via`确保一旦通过某个中间节点发现一致的奇偶校验路径，它就会向前传播，以便后续迭代在稳定的约束系统上运行。 旗帜`ok`跟踪后面的隐含结构是否违反了任何早期的分配。

 ### 示例 1

 考虑一个具有三个节点的小型一致配置：

 输入奇偶校验矩阵：```

```我们跟踪关闭：

 | k | 我| j | d[i][k] | d[i][k] d[k][j] | d[k][j] 通过 | | 之前的 d[i][j] 冲突|

 | 0 | 1 | 2 | 1 | 1 | 0 | 0 | 没有|
 | 1 | 0 | 2 | 1 | 0 | 1 | 1 | 没有|

 传播后，没有出现矛盾，所以输出为YES。 

这演示了奇偶校验约束在传递传播下全局一致的配置。 

### 示例 2

 矛盾的配置：

 输入：```

1 0 1
1 1 0
```这里每一对都被限制为奇数。 

| k | 我| j | d[i][k] | d[i][k] d[k][j] | d[k][j] 通过 | | 之前的 d[i][j] 冲突|

 ## 边缘情况

 一种边缘情况是所有非对角线条目均为 1，这意味着每一对都需要具有奇数距离。 运行闭包迫使通过中间体重复奇偶校验传播，系统最终会自相矛盾，因为在完整的约束系统中，奇奇偶校验不能在三角形分解下保持一致。 

另一种情况是嵌入奇偶校验形式的看起来稀疏但一致的二分式结构。 例如：```
0 1 0
1 0 1
0 1 0
```这里闭包立即稳定下来，因为中间奇偶校验和已经匹配直接约束，并且没有出现矛盾。 

最后的边缘情况是单节点图，其中矩阵基本一致。 该算法可以正确处理它，因为没有三次迭代会产生矛盾，并且初始状态已经满足闭包。
