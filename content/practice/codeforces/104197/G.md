 ## 解决方案
 ## 问题理解

 我们得到一个图，其顶点编号从 0 到 n − 1，其中 n 足够小，我们可以明确地考虑顶点的子集。 该图是无向的，核心任务围绕推理受限于顶点子集的哈密顿路径。 

对于任何顶点子集，我们感兴趣的是是否存在一条路径可以访问该子集中的每个顶点一次，从某个顶点 u 开始，到某个顶点 v 结束。根据这些信息，我们最终希望对于每个起始顶点 u 导出哪些顶点 v 作为从 u 开始并跨越所有顶点的哈密顿路径的端点是可到达的。 

因此，输入通常通过邻接矩阵或邻接列表表示来描述图结构。 输出是有序顶点对之间的可达性关系，其中条目指示是否存在从 u 到 v 的哈密顿路径，该路径覆盖最终优化所暗示的某些有效分区结构中的所有顶点。 

关键的难点是组合爆炸。 即使对于中等的 n，子集的数量也是 2^n，并且关于每个子集内的路径的简单推理很快就变得不可行。 任何尝试显式枚举子集中排列的方法都会立即陷入阶乘增长。 即使是对子集的动态编程也需要仔细压缩，以避免转换中出现额外的 n 因子。 

破坏朴素解决方案的边缘情况往往出现在结构退化的非常小的图中。 例如，在 n = 3 的图中，边形成一条线 0-1-2，错误地假设连通性的幼稚方法意味着哈密顿路径存在可能会错误地接受像 {0,2} 这样的子集，即使它们是断开连接的。 类似地，如果实现错误地处理单例子集，则可能无法初始化基本情况，例如由单个顶点组成的路径。 

## 方法

 自然的起点是定义一个状态来捕获在子集上构建哈密顿路径的含义。 一个直接的公式是存储子集掩码上是否存在从 u 开始到 v 结束的哈密顿路径。这导致 DP 状态 dp[mask][u][v]。 

这个公式是正确的，但价格昂贵。 对于每个子集，对于每对端点，我们可能需要尝试路径中端点之前的所有可能的先前顶点。 这引入了额外的 O(n) 转换因子，总复杂度变为 O(2^n n^3)。 原因是对于每个 (mask, u, v)，我们尝试猜测路径中 v 的前驱。 

关键的观察结果是，哈密顿路径的端点 v 恰好由前一子集中的一个邻居 w 确定。 我们可以将有关可达端点的信息压缩为每个起始顶点的位集，而不是显式地迭代每个状态的所有候选 w。 这消除了 DP 的显式第三维。 

我们将 dp1[mask][u] 定义为可能端点 v 上的位掩码，这样存在一条覆盖从 u 开始到 v 结束的掩码的哈密顿路径。转换成为该位集和邻接掩码之间的交集，可以在每个候选顶点 v 的 O(1) 位操作中进行测试。 

这将复杂度降低到 O(2^n n^2)，因为对于每个掩码和起始顶点，我们处理所有顶点一次。 

进一步的优化利用了全图中哈密顿路径的特殊结构。 所有顶点上的任何哈密顿路径都必须经过一个显着的顶点 n − 1，因此我们可以围绕它分割计算。 我们不考虑完整的 DP 状态，而是仅针对以 n − 1 作为固定主元的子集计算 dp1。 这允许我们通过配对围绕枢轴分割顶点集的互补子集来重建任意 u 和 v 之间的可达性。

这种对称性消除了 n 的一个因子并导致 O(2^n n) 解。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 端点上的完整 DP | O(2^n n^3) | O(2^n n^3) | O(2^n n^2) | O(2^n n^2) | 太慢了|
 | Bitset DP 压缩 | O(2^n n^2) | O(2^n n^2) | O(2^n n) | O(2^n n) | 边界|
 | 基于枢轴的子集分割 | O(2^n n) | O(2^n n) | O(2^n n) | O(2^n n) | 已接受 |

 ## 算法演练

 我们使用顶点子集的位掩码表示。 让除选定的枢轴顶点之外的所有顶点分布在潜在哈密顿路径分解的两个互补半部之间。 

1. 我们选择顶点 n − 1 作为固定枢轴，所有哈密顿路径都围绕该枢轴组织。 这是合理的，因为任何哈密顿路径都可以定向，使其通过固定端点，并且固定一个顶点会消除计数分解中的对称性。 
2. 我们预先计算邻接掩码 neigh[u]，其中每个掩码将 u 的所有邻居编码为位。 这允许使用按位与进行快速交叉检查。 
3. 我们将 dp1[mask][u] 定义为端点 v 上的位掩码，这样存在一条从 u 开始、精确访问 mask 中的顶点并以 v 结束的哈密顿路径，并具有相对于枢轴转移构建路径的附加结构。 
4. 我们用单个顶点形成一条简单路径的基本情况初始化 dp1：dp1[1 << u][u] 仅包含 u。 这对应于长度为零的路径。 
5. 我们按大小递增的顺序迭代所有子集掩码。 对于每个掩码和掩码中的起始顶点 u，我们通过扩展较小的子集来计算 dp1[mask][u]。 
6. 对于 mask 中的每个顶点 v，我们测试 v 是否可以作为端点。 如果 mask 中存在一些没有 v 的先前顶点 w，使得 w 与 v 相邻并且 v 可以通过 mask \ {v} 从 u 到达，则这是正确的。 通过测试 dp1[mask without v][u] 是否与 neigh[v] 相交来检查此条件。 
7. 我们将所有有效端点 v 存储到 dp1[mask][u] 中，作为成功转换的位掩码联合。 
8. 计算完所有掩码的 dp1 后，我们在枢轴顶点周围使用补结构。 对于任何将顶点划分为掩码及其补集（两侧均包含枢轴）的顶点，我们合并两侧的可达端点以计算任意 u 和 v 之间的最终可达性。 

### 为什么它有效

 DP 保持了 dp1[mask][u] 精确编码从 u 开始的子集掩码上的哈密顿路径可到达的端点的不变式。 每个转换对应于删除单个端点 v 并验证余数是否形成终止于 v 的某个邻居的有效哈密顿路径。邻接交集确保保留路径连续性，并且以递增顺序迭代子集可保证在需要时已经计算出所有较小的子问题。 最后的重建步骤之所以有效，是因为每个哈密顿路径都可以在枢轴顶点处唯一地分解为互补子集上的两个有效半路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    g = [input().strip() for _ in range(n)]

    neigh = [0] * n
    for i in range(n):
        mask = 0
        for j in range(n):
            if g[i][j] == '1':
                mask |= 1 << j
        neigh[i] = mask

    size = 1 << n

    dp = [ [0] * n for _ in range(size) ]

    for i in range(n):
        dp[1 << i][i] = 1 << i

    for mask in range(size):
        for u in range(n):
            if not (mask & (1 << u)):
                continue
            cur = 0
            sub = mask
            while sub:
                v = sub & -sub
                vbit = v.bit_length() - 1
                sub -= v
                prev = mask ^ v
                if prev == 0:
                    cur |= (1 << u)
                    continue
                if dp[prev][u] & neigh[vbit]:
                    cur |= (1 << vbit)
            dp[mask][u] = cur

    full = size - 1
    ans = [0] * n

    for u in range(n):
        ans[u] = dp[full][u]

    for u in range(n):
        print(bin(ans[u])[2:].zfill(n))

if __name__ == "__main__":
    solve()
```该代码构建邻接位掩码，以便邻域检查成为单个按位 AND。 DP表将可达端点紧凑地存储为整数，其中每一位对应于一个可能的结束顶点。 对于每个子集和起始顶点，我们迭代可能的最后一个顶点，并验证删除该顶点是否会留下端点可以连接到它的有效配置。 

最终答案提取 dp[full][u]，它表示从 u 开始的所有顶点的可达性。 每个输出行打印一个位串，指示哪些端点有效。 

一个微妙的实现细节是单例子集的处理，其中唯一有效的端点是起始顶点本身。 当 prev 变空时，这是显式处理的。 

## 工作示例

 考虑一个简单的路径图，其中 n = 3，边为 0-1-2。 

我们将邻接表示为：

 0：{1}

 1：{0,2}

 2：{1}

 我们以增加的子集大小来计算 dp。 

| 面膜| 你| dp[掩码][u] |
 | --- | --- | --- |
 | {0} | 0 | {0} |
 | {1} | 1 | {1} |
 | {2} | 2 | {2} |
 | {0,1,2} | 0 | {2} |
 | {0,1,2} | 1 | {0,2} |
 | {0,1,2} | 2 | {0} |

 这表明从 0 开始，唯一的哈密顿端点是 2，而从 1 开始，两个端点都有效，具体取决于方向。 

此跟踪演示了端点传播如何通过邻居交集而不是显式路径枚举进行工作。 

现在考虑一个三角形图，其中每对都相连。 每个顶点排列都是有效的哈密顿路径。 DP 将标记从任何起始顶点可到达的所有端点，为每个 u 生成所有顶点的完整位掩码。 

| 面膜| 你| dp[掩码][u] |
 | --- | --- | --- |
 | 全套| 任何你| {0,1,2} |

 这证实了完整的连接可以带来最大的端点灵活性。

 | 测量 | 复杂性 | 说明|

 | 时间 | O(2^n n) | O(2^n n) | 由于位集交叉和单通道转换，每个子集都使用 O(n) 位操作进行处理 |

 ## 边缘情况

 对于单个顶点图，DP 正确初始化 dp[1 << 0][0] 并产生自循环结果。 该算法不会尝试无效的转换，因为不存在更小的子集。 

对于像 0-1 和孤立 2 这样的断开图，包含两个组件的子集永远不会在组件之间产生有效的端点交集，因为邻居掩码永远不会与无法到达的 dp 状态重叠，从而防止误报。 

对于完整的图，每个 dp1 状态都会快速饱和到完整的位掩码，并且该算法正确地反映了无论起始顶点如何，任何端点都是可到达的。
