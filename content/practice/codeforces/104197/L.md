 ## 解决方案
 ## 问题理解

 我们得到奇数个顶点或经过小调整的偶数个顶点，并且我们必须显式构造标记节点之间的边的结构化列表。 节点在概念上排列在一个圆上，边在重复的“块”中生成，其中每个块由围绕起始索引的局部连接模式定义，然后循环缠绕。 

任务不是计算答案，而是输出遵循严格几何图案的特定边缘结构。 每个块恰好贡献 k 个边，并且通过迭代所有有效起始位置 i 并发出该块定义的边来形成完整的构造。 

尽管该语句是几何结构的，但基本要求是纯粹的组合：生成遵循循环移位模式的确定性边序列，并且在偶数情况下，合并以一致方式连接的附加中心节点。 

约束没有明确给出，但这种类型的问题通常允许 n 高达 200000 或更高。 这立即排除了任何试图维持动态连接或重新计算每条边结构的模拟。 唯一可行的方法是在 O(1) 时间内构造每个边并以线性总时间输出。 

朴素推理的一个微妙的失败案例是假设边缘是唯一的或者重复并不重要。 在此结构中，边缘可能会在块之间出现多次，但这是预期的。 另一个问题是循环索引处理不当，尤其是当 i − j 变为非正数时。 如果没有模运算，索引就会悄无声息地崩溃。 最后，对于偶数 n，忘记中心节点会破坏对称性并使预期结构完全失效。 

## 方法

 一种简单的方法会尝试显式模拟连接性：维护图形，逐块添加边，并可能验证结构属性或删除重复边。 这已经是不必要的了，因为问题从不要求验证，只要求构造。 即使我们忽略这一点，在最坏的情况下，维护动态图结构和检查组件将至少花费 O(n^2)，因为有 O(nk) 条边，并且 k 与 n 成正比。 

关键的观察是该结构是完全明确的。 每条边都由一个包含 i 和偏移量 j 的简单算术公式描述。 除了循环索引之外，块之间不存在任何依赖性。 一旦我们接受每个块是独立的，整个任务就简化为在模运算下生成对 (i − t, i + t + 1)。 

对于奇数 n，单独的圆就足够了。 对于偶数 n，我们引入一个中心节点，该节点连接到每个块的一个附加端点，保留相同的对称性，同时修复奇偶校验问题，否则会阻止相同的连接参数起作用。 

蛮力方法“失败”不是因为它不正确，而是因为它忽略了构造已经是最优且公式化的。 每个块只是一个圆上的平移图案的观察结果将一切都简化为直接公式生成。

 | 暴力模拟 | O(n²) | O(n²) | 太慢了 |
 | 直接施工| O(n²) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们描述了两种情况的构造，但实现逻辑是相同的：使用算术移位生成边。 

### 奇数 n = 2k + 1

 1. 计算 k = (n − 1) // 2. 这确定每个块从其中心延伸多远。 
2. 将 i 从 1 迭代到 n。 每个 i 定义一个以节点 i 为中心的块。 
3. 对于每个块，将 t 从 0 迭代到 k − 1。 
4. 对于每个 t，在 (i − t) 和 (i + t + 1) 之间创建一条边，以 n 为模处理索引。 
5. 立即输出每个边。

每个块代表从基边（i，i+1）向外扩展的k条对称边。 配对确保每个偏移量 t 沿圆的相反方向连接等距的两个点。 

### 偶数 n = 2k + 2

 1. 计算 k = (n − 2) // 2。 
2. 将节点1到2k+1视为一个环，将节点2k+2视为特殊中心节点。 
3. 将 i 从 1 迭代到 2k + 1。 
4. 对于每个 i，生成与奇数情况相同的 k 个循环边。 
5. 另外，将“额外端点”i + k + 1 连接到中心节点。 
6. 按块顺序输出所有边。 

中心边缘补偿了奇数周期中存在的奇偶校验对称性缺失，确保每个块都贡献正确的结构平衡。 

### 为什么它有效

 每个块在 i 周围的对称偏移处强制执行一对节点。 在所有块中，每条边都参与受控的重叠模式：连续的块以保证语句中描述的连接属性的方式共享端点。 循环对称性确保均匀覆盖，而偶数情况的中心节点通过吸收不匹配的端点来恢复平衡，否则会破坏结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    if n % 2 == 1:
        k = (n - 1) // 2
        for i in range(1, n + 1):
            for t in range(k):
                u = i - t
                v = i + t + 1
                u = (u - 1) % n + 1
                v = (v - 1) % n + 1
                print(u, v)
    else:
        k = (n - 2) // 2
        m = n - 1
        center = n
        for i in range(1, m + 1):
            for t in range(k):
                u = i - t
                v = i + t + 1
                u = (u - 1) % m + 1
                v = (v - 1) % m + 1
                print(u, v)
            print(i + k + 1, center)

if __name__ == "__main__":
    solve()
```奇数情况直接编码循环公式。 模转换确保了概念圈上的环绕行为。 

在偶数情况下，我们将前 n−1 个节点隔离为一个环，并将最后一个节点视为固定集线器。 生成 k 个循环边后，我们将最终端点附加到中心节点。 索引移位 i + k + 1 是安全的，因为 i 的范围仅达到 n−1，确保端点保持在边界内。 

一个常见的实现错误是忘记了必须仔细规范化带有负数的 Python 模。 使用 (x−1) % n + 1 完全避免了零索引歧义。 

## 工作示例

 我们为小的奇数和偶数情况构建了两条说明性轨迹。 

### 示例 1（奇数 n = 5，k = 2）

 对于每个 i，我们生成两条边。 

| 我| t | 你（原始）| v（原始）| 你（模组）| v（模）| 边缘|
 | --- | --- | --- | --- | --- | --- | --- |
 | 1 | 0 | 1 | 2 | 1 | 2 | (1,2) |
 | 1 | 1 | 0 | 3 | 5 | 3 | (5,3) |
 | 2 | 0 | 2 | 3 | 2 | 3 | (2,3) |
 | 2 | 1 | 1 | 4 | 1 | 4 | (1,4) |

 对 i = 3, 4, 5 继续类似操作会产生完整的循环重叠。 

该迹线显示了环绕如何自然地产生“穿过”边界的边缘，这对于圆形对称性至关重要。 

### 示例 2（即使 n = 6，k = 2）

 循环节点为 1..5，中心为 6。 

| 我| t | 你| v | 边缘|
 | --- | --- | --- | --- | --- |
 | 1 | 0 | 1 | 2 | (1,2) |
 | 1 | 1 | 5 | 3 | (5,3) |
 | 1 | - | - | - | (3,6) |
 | 2 | 0 | 2 | 3 | (2,3) |
 | 2 | 1 | 1 | 4 | (1,4) |
 | 2 | - | - | - | (4,6) |

 这显示了额外的中心连接，确保每个块正确关闭。 

该轨迹突出显示偶数情况仅在每个块的最终边缘上有所不同，同时保留相同的循环模式。

 ## 边缘情况

 对于 n = 1，该算法不会产生边，因为 k = 0。循环永远不会执行，因此输出为空，这符合单个节点没有边的想法。 

对于 n = 3、k = 1，每个 i 恰好产生一条边 (i, i+1)。 模数确保我们仍然产生有效的循环边（如 (3,1)），从而确认环绕的正确性。 

对于 n = 4、k = 1（偶数情况），每个块贡献一个循环边以及到节点 4 的连接。对于 i = 1，我们得到 (1,2) 和 (3,4)，这已经显示了中心如何吸收最终端点。 该算法保持稳定，因为对于 [1,3] 中的 i，i + k + 1 永远不会超过 4。
