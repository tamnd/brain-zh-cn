 ## 解决方案
 ## 问题理解

 我们正在使用长度的二进制数组$n$，其中每个元素为 0 或 1。来自任何此类数组$a$，派生数组$b$通过固定窗口大小上的滑动总和来定义$k$。 各位置$b_i$代表总和$k$的连续元素$a$，通过分析中的索引关系隐式地环绕循环结构。 

问题不是重建单个数组，而是计算有多少个二进制数组产生相同的派生数组$b$。 换句话说，我们想要在“取所有长度-”变换下二进制数组的等价类的大小$k$滑动金额”。

 尽管输入描述看起来像直接数组转换，但关键的困难在于多个不同的二进制数组可以折叠成相同的滑动和表示，而任务是准确量化有多少个这样做。 

这些限制（隐含在编辑结构中）表明$n$可能很大，因此任何以简单组合方式显式枚举数组甚至循环的解决方案都是不可行的。 任何指数级的东西$n$立即被排除，甚至每个测试用例的二次推理也会太慢。 这推动我们对指数进行结构性分解。 

当滑动总和数组恒定时，会出现微妙的边缘情况。 在这种情况下，元素之间的约束局部消失，整组变量变成自由选择。 幼稚的重建会错误地假设唯一性，从而错过了组合爆炸。 

当出现另一种边缘情况时$k$和$n$不是互质的。 指数分为不相交的循环，误解这种结构会导致对独立成分的计数过高或过低。 

## 方法

 暴力破解的视角很简单：尝试每个二进制数组$a$长度$n$，计算其滑动和数组$b$，并与目标进行比较。 这在概念上是正确的，因为转换是确定性的。 然而，搜索空间包含$2^n$数组和计算$b$对于每项费用$O(n)$，导致$O(n2^n)$，即使对于中等程度的人来说，这也是不可能的$n$。 

突破来自于注意到滑动总和约束仅连接模同余的索引$k$。 更准确地说，如果我们比较相邻值之间的差异$b$，我们得到形式的线性约束$a_{i+k} - a_i = b_{i+1} - b_i$。 这意味着数组分解为通过步进形成的独立循环$+k$模数$n$。 

每个这样的周期都有长度$\frac{n}{\gcd(n,k)}$，并且有$\gcd(n,k)$独立循环。 在每个循环内，约束要么完全确定二进制模式，要么留下全局二进制翻转自由度，具体取决于差异是否全为零。 

因此，我们不是直接对数组进行计数，而是将循环分为两种类型：约束循环（其中值唯一固定）和均匀循环（其中所有值必须相等，但可以自由选择全零或全一）。 计数问题简化为选择分配这些统一周期中的哪些周期，并对跨配置的贡献进行求和。 

这将问题简化为循环分区上的组合问题，并且使用二项式恒等式将最终表达式折叠成可以通过快速求幂求值的形式。

 ## 算法演练

 ### 1.计算结构参数

 我们计算$g = \gcd(n, k)$。 该值决定了在重复添加的情况下索引如何拆分$k$。 每个位置恰好属于一个由重复相加形成的循环$k$模数$n$。 

这很重要的原因是逐步$k$将索引划分为独立的组件，因此约束永远不会跨越循环边界。 

### 2. 确定周期长度和减小的尺寸

 每个周期都有长度$l = \frac{n}{g}$，并且有$g$总共循环。 我们单独对待每个周期，因为约束不会混合不同周期中的元素。 

这种减少使得全局依赖性问题变成了重复的相同局部问题。 

### 3.按约束强度对循环进行分类

 在每个周期内，如果差异引起$b$不全为零，则该周期中的所有值都是唯一确定的。 否则，循环是“一致的”，这意味着它的所有元素必须相等，但可以全为 0 或全 1。 

这种区别至关重要，因为它决定了一个循环是乘法（固定）还是组合（选择）做出贡献。 

### 4. 均匀循环的计数配置

 让$t$是均匀循环的数量。 每个这样的循环可以是全零或全一，但全局结构与滑动总和相互作用，这意味着只有全一循环的数量总体上很重要。 

所以对于一个固定的$t$， 有$t+1$有意义的选择与他们中有多少人做出贡献相对应。 

### 5. 总结所有可能的情况$t$现在我们总结了选择哪些周期是统一的以及如何分配它们的所有方法。 这导致了以下形式的二项式和：$$\sum_{t=0}^{g} \binom{g}{t} (t+1) A^{g-t}$$在哪里$A$编码受限循环的内部可变性。 

使用标准二项式恒等式将该表达式简化为包含$(A+1)^g$和$g(A+1)^{g-1}$，实现快速评估。 

### 为什么它有效

 关键的不变量是每个循环在滑动和引起的变换下独立运行。 变换仅连接远处的索引$k$，永远不会离开循环。 因此，每个有效数组都由每个周期分类做出的独立选择唯一地表示。 计数减少为在不受干扰的情况下组合独立贡献，确保跨周期不会出现过度计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    n, k = map(int, input().split())
    g = 1
    import math
    g = math.gcd(n, k)

    l = n // g

    a = (pow(2, l, MOD) - 2) % MOD
    if a < 0:
        a += MOD

    # final expression: (a+1)^g + g*(a+1)^(g-1)
    base = (a + 1) % MOD

    ans = modpow(base, g)
    ans = (ans + g * modpow(base, g - 1)) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了代数简化。 我们使用 gcd 计算循环数，然后将问题简化为评估封闭式表达式。 模幂可以有效地处理大幂。 

计算时必须小心$2^l - 2$，因为在应用模数之前减法可能会产生负值。 第二项使用$g \cdot base^{g-1}$，也必须以 MOD 为模进行约简。

 ## 边缘情况

 当$g = \gcd(n,k) = 1$，整个数组形成一个循环。 在这种情况下，算法将所有内容简化为单个组合组件。 公式变为$(a+1) + 1$，反映多个周期之间不存在交互作用。 假设多个独立组件的简单实现在这里会被过度计算。 

什么时候$k = n$，每个滑动窗口都是整个数组。 这迫使所有导出的和都相同，并且循环结构完全退化。 该算法正确给出$g = n$，这意味着每个索引都是它自己的平凡循环，因此组合爆炸是通过求幂来捕获的$n$独立的二元选择。
