 ## 解决方案
 ## 问题理解

 给定一个整数数组，任务是决定是否可以在这些部分的 XOR 值定义的约束下将其拆分为多个连续的部分。 整个问题的关键对象是段的异或，分区的可行性完全取决于这些异或值如何相互关联。 

根据整个数组的异或，该问题在两种根本不同的情况下表现。 如果所有元素的异或都不为零，那么结构就非常灵活：我们可以以任何我们喜欢的方式将数组分成两个段，并且保证这两个段的异或值不同。 有趣的困难仅在总异或为零时出现，因为那时某些分区变得不可能，我们必须更仔细地推理结构。 

从复杂性的角度来看，数组大小达到了典型的竞争性编程限制，因此对所有分割点或子数组进行二次或三次探索会立即变得太慢。 任何显式检查所有分区的解决方案都需要检查 O(n²) 或更差的段 XOR，这对于 n 大约 10⁵ 来说是不可行的。 这促使我们寻求一种依赖于前缀异或结构和线性扫描的解决方案。 

当所有元素都为零时，会出现微妙的边缘情况。 在这种情况下，每个子数组 XOR 都为零，因此没有有意义的分区可以区分段。 另一种边缘情况是当数组恰好包含一个非零值或当所有非零值崩溃为单个重复 XOR 标识时，这会使分区逻辑退化。 

## 方法

 暴力破解的想法是尝试一切可能的方法将数组分割成段并计算每个段的异或。 对于每个候选分区，我们将验证是否满足所需的 XOR 约束。 即使我们将自己限制为两个或三个段，由于重新计算或扫描子数组，计算段异或会重复导致 O(n²) 工作。 

效率低下的原因是在重叠的子数组上重复重新计算异或。 虽然 XOR 本身很便宜，但候选分区的数量太大。 关键的观察结果是，可以使用前缀 XOR 以 O(1) 的时间复杂度计算任何段上的 XOR，这会降低每次检查的成本，但不会减少检查次数。 

真正的简化来自于分析有效分区的结构。 当总异或非零时，没有结构限制阻止我们选择切割。 当它为零时，问题就简化为检查我们是否可以找到一个前缀结构，该结构允许将剩余后缀分成具有不同 XOR 属性的两个段。 关键的观察是，如果这样的分割是不可能的，则后缀必须受到极大的约束：某个点之后的每个前缀异或只能取两个值，迫使所有元素折叠成非常受限的模式。 

这将问题从搜索分区转变为扫描特定的结构违规。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) 或更糟 | O(1) | O(1) | 太慢了|
 | 前缀异或+结构检查| O(n) | O(n) 或 O(1) | 已接受 |

 ## 算法演练

1. 计算整个数组的异或。 如果它不为零，我们可以立即接受并构造任何有效的分裂，因为该机制中不存在结构性障碍。 
2. 如果总异或为零，则从左侧开始扫描，查找元素非零的第一个索引。 该元素定义了约束所有有效段异或的参考值。 
3. 如果不存在非零元素，则数组完全为零，无法形成有效的分区，因此我们拒绝。 
4. 固定第一个非零位置作为前缀端点。 这将数组分为左部分和右部分。 左侧部分的 XOR 等于由第一个非零元素引起的所选参考结构。 
5. 现在计算后缀上的前缀 XOR，并尝试找到一种方法将其分成两个连续的部分，以便这两个部分避免崩溃为禁止的 XOR 值。 实际上，这意味着搜索第一后缀段的异或既不为零也不等于参考异或值的有效剪切位置。 
6. 如果存在这样的分裂点，我们接受。 否则，前缀 XOR 的结构意味着每个后缀前缀 XOR 仅限于两个值，这迫使后缀中的所有元素为零或参考值，从而无法进行有效划分。 

### 为什么它有效

 正确性取决于零总异或约束下前缀异或的行为。 一旦总 XOR 为零，任何有效分区到多个段都必须在段之间完美平衡 XOR 贡献。 如果存在有效的分割，则后缀内必须有一个前缀，其 XOR 避免崩溃为由第一个非零元素的结构引起的两个禁止值。 

如果不存在这样的前缀，则后缀 XOR 空间是退化的：每个前缀 XOR 要么为零，要么固定为相同的非零值。 这意味着该数组实际上仅由两个值组成：零和参考 XOR 值，这防止形成三个不同的 XOR 一致段。 这种结构刚性保证了算法不会错误地接受无效配置或拒绝有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    total = 0
    for x in a:
        total ^= x

    if total != 0:
        print("YES")
        return

    first_nz = -1
    for i, x in enumerate(a):
        if x != 0:
            first_nz = i
            break

    if first_nz == -1:
        print("NO")
        return

    target = a[first_nz]

    # compute prefix xor on suffix
    seen = set()
    cur = 0

    for i in range(first_nz + 1, n):
        cur ^= a[i]
        # we try to find a prefix XOR that is valid
        if cur != 0 and cur != target:
            print("YES")
            return

    print("NO")

if __name__ == "__main__":
    solve()
```该解决方案首先检查全局异或，这立即解决了简单的情况。 然后它识别第一个非零元素，该元素充当零异或机制中结构参数的锚。 后缀扫描维护运行的 XOR 并检查后缀的任何前缀是否转义两个禁止的 XOR 值。 该条件直接对应于有效的第二次切割的存在。 

一个常见的微妙之处是确保扫描严格在第一个非零元素之后开始。 包含它会混合锚的定义并破坏后缀的结构解释。 

## 工作示例

 ### 示例 1

 输入：```
5
1 2 3 0 0
```这里总的异或是`1 ^ 2 ^ 3 = 0`。 第一个非零元素位于索引 0 处，值为 1。我们设置 target = 1 并扫描后缀`[2,3,0,0]`。 

| 我| 一个[我] | 当前异或 | 当前有效吗？ |
 | --- | --- | --- | --- |
 | 1 | 2 | 2 | 是（2！= 0 和 2！= 1）|

 在第二个元素本身，我们已经找到了一个既不是 0 也不是 1 的前缀 XOR，因此我们可以成功分割。 

这表明，即使总异或为零，“足够丰富”的后缀也会立即产生有效的分区。 

### 示例 2

 输入：```
4
1 0 1 0
```总异或为零。 第一个非零是索引 0 处的 1。后缀是`[0,1,0]`。 

| 我| 一个[我] | 当前异或 | 当前有效吗？ |
 | --- | --- | --- | --- |
 | 1 | 0 | 0 | 没有|
 | 2 | 1 | 1 | 没有|
 | 3 | 0 | 1 | 没有|

 避免后缀中无前缀异或`{0,1}`，因此不存在有效的分割。 

这显示了退化结构，其中所有值都限制为`{0, target}`，防止任何成功的分区。

 ## 边缘情况

 当所有元素都为零时，算法正确地将总异或计算为零，并且找不到非零主元。 它立即返回 NO，与任何分区都不能创建不同的 XOR 段的事实相匹配。 

当数组恰好有一个非零元素时，总异或非零，因此算法立即返回 YES，无需进行后缀分析。 这符合任何拆分在非零总 XOR 下都有效的直觉。 

当数组在零和固定值之间交替时，例如`[1,0,1,0]`，后缀扫描永远不会在外部产生前缀异或`{0,1}`，因此算法正确拒绝，反映了阻止有效分区的受限 XOR 空间。
