 ## 解决方案
 ## 问题理解

 我们得到了大小的排列$n$，并通过一系列交换进行修改。 每次修改后，我们需要计算一个称为当前排列的“美度”的值。 

其美丽取决于排列的结构特性，但它最终会分解为一些基于奇偶校验的条件和简单的全局统计数据。 该定义从基于反转的推理开始，但最终的决定不需要显式地计算反转或在每次交换后重新计算它们。 

每个查询通过交换两个位置来改变排列。 每次交换后，我们必须输出结果排列的美感。 

重要的约束含义是$n$并且查询的数量足够大，每次交换后从头开始重新计算与反转相关的属性会太慢。 任何在线性时间内重新评估每个查询的反转或周期的解决方案都会导致粗略的结果$O(nq)$，这远远超出了这种形式的典型 Codeforces 约束的可接受限制。 预期的解决方案必须保持每次交换的更新次数恒定或对数。 

当排列已经排序时，会出现微妙的边缘情况。 在这种情况下，每个子序列都保持排序，并且美感退化为特殊值$-1$。 这种情况必须明确分开，否则它会被错误地分类为奇偶校验情况之一。 

当交换创建或破坏固定点或影响奇偶条件而不改变全局结构太多时，就会出现另一种边缘情况。 例如，交换两个已经正确的位置可能会意外地翻转排列奇偶校验，即使局部结构似乎没有变化。 

## 方法

 直接方法会在每次交换后重新计算所需的属性。 人们可以尝试每次重新计算反转计数或循环分解。 这在概念上是可行的，因为美的定义是从反转结构和排列奇偶性导出的，但在计算上却失败了。 

重新计算每个查询的反转成本$O(n)$或者$O(n \log n)$，并这样做是为了$q$操作导致$O(nq)$或者$O(nq \log n)$，当两者都达到时太大$2 \cdot 10^5$。 

关键的观察结果是，最终公式仅取决于三个全局属性：

 排列的奇偶性，索引的数量，其中$p_i \equiv i \pmod 2$，以及固定点的数量$p_i = i$。 

其中每一个都可以增量维护。 在交换下，排列的奇偶性会发生可预测的变化：任何交换都对应于一个转置，这会翻转排列奇偶性。 奇偶校验也可以使用排列奇偶校验等于的经典事实来初始化$n - \text{cycles}$，或更实际地通过反转奇偶校验计算一次。 

第二个量，匹配奇偶校验位置，仅取决于每个元素的索引奇偶校验是否与其值奇偶校验匹配。 交换仅影响两个位置，因此此计数更新$O(1)$。 这同样适用于固定点。 

一旦维护了这些，每个查询都会减少为恒定时间分类。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个查询的强力重新计算结构 |$O(nq)$|$O(n)$| 太慢了|
 | 增量维护奇偶校验+计数器|$O(n + q)$|$O(n)$| 已接受 |

 ## 算法演练

 在此过程中，我们维护三条信息：排列的奇偶校验、索引奇偶校验与值奇偶校验匹配的位置的计数器以及固定点的计数器。 

### 步骤

 1.初始化排列并计算每个位置是否满足$p_i = i$以及是否$p_i \bmod 2 = i \bmod 2$。 这给出了两个计数器的初始值。 
2. 计算排列的初始奇偶校验。 这可以通过计算反转或通过计算循环分解并使用来完成$n - \text{cycles}$。 
3. 对于每个仓位之间的掉期查询$a$和$b$，首先更新所有受这两个位置影响的计数器。 这意味着在交换之前删除他们的旧贡献并在交换之后添加他们的新贡献。 
4. 在排列数组中应用交换。 
5. 翻转排列的奇偶校验，因为单个交换是转置并改变奇偶校验。 
6. 应用交换后，依次检查三个条件：排列是否为奇数、所有位置是否满足奇偶对齐、所有位置是否都是不动点。 
7、根据这些条件输出对应的美感值。 

决策逻辑是分层的，因为每个条件都代表一个逐渐受到更多约束的排列结构。 

### 为什么它有效

 正确性来自于美值完全由排列状态的三个不变量决定：排列奇偶性、索引和值的奇偶性对齐以及非固定结构的存在。 每次交换仅修改固定数量的这些不变量，并且所有这些不变量都得到准确维护。 由于这些不变量之外的隐藏结构属性不会影响最终分类，因此每次更新后的决策都是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    p = list(map(int, input().split()))

    cnt_eq = 0
    cnt_same_par = 0

    for i in range(n):
        if p[i] == i + 1:
            cnt_eq += 1
        if (p[i] % 2) == ((i + 1) % 2):
            cnt_same_par += 1

    inv_parity = 0
    seen = [0] * (n + 1)
    for x in p:
        seen[x] = 1
    # permutation parity via cycles
    cycles = 0
    vis = [0] * (n + 1)
    for i in range(1, n + 1):
        if not vis[i]:
            cycles += 1
            cur = i
            while not vis[cur]:
                vis[cur] = 1
                cur = p[cur - 1]

    inv_parity = (n - cycles) % 2

    def get_answer():
        if inv_parity == 1:
            return n
        if cnt_same_par != n:
            return n - 1
        if cnt_eq != n:
            return n - 2
        return -1

    for _ in range(q):
        a, b = map(int, input().split())
        a -= 1
        b -= 1

        for i in (a, b):
            if p[i] == i + 1:
                cnt_eq -= 1
            if (p[i] % 2) == ((i + 1) % 2):
                cnt_same_par -= 1

        p[a], p[b] = p[b], p[a]

        for i in (a, b):
            if p[i] == i + 1:
                cnt_eq += 1
            if (p[i] % 2) == ((i + 1) % 2):
                cnt_same_par += 1

        inv_parity ^= 1

        print(get_answer())

if __name__ == "__main__":
    solve()
```该实现维护数组中的排列并仅更新两个交换的位置。 在交换之前，我们删除他们对两个计数器的贡献，然后应用交换，然后重新添加贡献。 这避免了任何完全重新计算。 

排列奇偶校验在每次交换后直接切换，因为每次交换都是一次转置。 

答案函数完全按照推理中的描述对层次逻辑进行编码：奇数排列首先占主导地位，然后是奇偶校验对齐，然后是定点完整性。 

一个常见的陷阱是在交换之前忘记减去贡献； 仅在交换后进行更新会导致重复计算错误。 

## 工作示例

 考虑大小的小排列$4$:$[1, 3, 2, 4]$。 

我们跟踪每一步的计数器。 

### 示例 1

 初始状态：

 | 我| p[i] | p[i] 定点| 奇偶匹配 |
 | ---| ---| ---| ---|
 | 1 | 1 | 是的 | 是的 |
 | 2 | 3 | 没有| 没有|
 | 3 | 2 | 没有| 没有|
 | 4 | 4 | 是的 | 是的 |

 所以`cnt_eq = 2`,`cnt_same_par = 2`，假设初始奇偶校验为偶数。 

交换 (2, 3) 后，排列变为$[1, 2, 3, 4]$。 

| 我| p[i] | p[i] 定点| 奇偶匹配 |
 | ---| ---| ---| ---|
 | 1 | 1 | 是的 | 是的 |
 | 2 | 2 | 是的 | 是的 |
 | 3 | 3 | 是的 | 是的 |
 | 4 | 4 | 是的 | 是的 |

 现在`cnt_eq = 4`,`cnt_same_par = 4`，排列已排序且奇偶校验。 

输出变为$-1$，因为排列是完全排序的。 

此跟踪显示两个计数器如何仅在排序情况下收敛到完全对齐。 

### 示例 2

 排列：$[2, 1, 4, 3]$| 我| p[i] | p[i] 定点| 奇偶匹配 |
 | ---| ---| ---| ---|
 | 1 | 2 | 没有| 是的 |
 | 2 | 1 | 没有| 是的 |
 | 3 | 4 | 没有| 是的 |
 | 4 | 3 | 没有| 是的 |

 这里`cnt_eq = 0`,`cnt_same_par = 4`。 

如果排列奇偶性是偶数，则条件`cnt_same_par == n`成立但是`cnt_eq != n`，所以输出变成$n - 2 = 2$。 

这演示了第二级后备：满足奇偶校验对齐，但结构不完全一致。

 ## 边缘情况

 最微妙的情况是完全排序的排列。 在那种情况下，双方`cnt_same_par`和`cnt_eq`保持最大，唯一有效的输出是$-1$。 计数器初始化中的任何错误或未能正确处理固定点都将错误地归咎于$n$或者$n-1$。 

另一个边缘情况是交换两个都是固定点的元素。 定点的计数器下降 2，然后重新正确添加，但排列奇偶校验仍然翻转。 根据元素值而不是交换操作更新奇偶校验的简单实现会错过此翻转。 

第三种情况是奇偶校验全局成立但不存在固定点。 这产生了$n - 2$如果条件顺序错误，很容易错误地跳过该分支。
