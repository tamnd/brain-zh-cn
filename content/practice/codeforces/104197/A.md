 ## 解决方案
 ## 问题理解

 我们得到了一个数字列表，我们想将它们放在一个圆圈周围。 一旦放置，每个元素都会通过与圆上两个相邻元素的乘积来贡献总分。 目标是选择使这些相邻乘积之和最大化的元素排序。 

圆的结构很重要，因为每个元素都有两个邻居，因此每个元素恰好参与两个乘积项。 一个好的安排会尝试尽可能频繁地将大数与大数配对，但循环约束阻止简单地将所有内容分组在一起。 

关键的约束是，最优排列总是存在，并且具有通过以非递增顺序对数组进行排序而得出的非常具体的结构。 这意味着问题不是关于排列的搜索，而是关于证明和使用确定性构造。 

从复杂性的角度来看，输入大小足够大，以至于任何涉及子集的排列或动态规划的解决方案都是不可能的。 对所有循环排列的阶乘搜索将增长为 (n−1)!，即使 n 约为 12，这也是不可行的。这迫使我们采用基于排序的构造，即 O(n log n)，完全在最多 200000 个元素的典型限制内。 

当许多值相等时，会出现微妙的边缘情况。 在这种情况下，多种排列可能看起来是等效的，但在不尊重已证明结构的情况下对连续的大元素进行分组的天真贪婪仍然可能在精心设计的输入上失败，其中即使值重复，邻接交互也会不同。 

另一种边缘情况是 n = 2 或 n = 3。对于非常小的圆，“奇偶交替”结构会退化，假设至少一个完整交替周期的粗心实现可能会错误索引或放错元素。 

## 方法

 强力解决方案将枚举数组的每个可能的循环排列，计算每个循环排列的乘积之和，并取最大值。 这是正确的，因为它直接检查所有有效配置。 然而，循环排列的数量为 (n−1)!，每次评估成本为 O(n)，导致大约 O(n·(n−1)!) 次操作。 即使对于 n = 12，这也变得太大了。 

目标函数的结构是关键的观察结果。 每个相邻对贡献一个乘积，重新排列四个连续元素 a、b、c、d 会改变贡献，当 a ≥ b 且 c ≥ d 时，可以使用不等式 ac + bd ≥ ad + bc 进行比较。 这与重排不平等变体的交换论点相同。 

这个局部交换论点意味着，如果我们想要最大化贡献，大元素应该以受控交替模式与相对较大的邻居配对，而不是聚集在一起。 声明中的证明表明，我们可以逐步强制最大的 k 个元素形成一个连续的段，其末端的行为方式可预测。 扩展这个结构导致最终的安排，其中排序的序列被分成两个交错的组：一组放置在圆的奇数位置，另一组填充反向结构中的偶数位置，这相当于语句中描述的模式。 

一旦知道正确的排序，剩下的任务就是简单地计算相邻乘积的循环和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·(n−1)!) | O(n·(n−1)!) | O(n) | 太慢了|
 | 最优（排序+构建）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们首先按非递增顺序对数组进行排序，使得 a1 ≥ a2 ≥ ... ≥ an。 

## 施工阶段

1. 我们通过首先将所有将占据最终循环中奇数位置的元素来构建循环顺序。 这些按顺序a1、a3、a5 等进行。 这确保了最大的元素间隔开而不是相邻，从而防止过度集中的配对。 
2. 放置所有奇数索引元素后，我们按照与自然配对顺序相反的顺序附加剩余元素 a2、a4、a6 等。 此步骤与交换论证相一致，该论证表明，当较小的元素填充较大锚点之间的间隙时，配对结构会最大化。 
3. 所得序列代表最佳循环排列。 

## 评估阶段

 1. 我们计算这个循环序列中连续元素的乘积之和。 
2. 我们还包括最后一个和第一个元素的乘积来闭合循环，因为排列是循环的。 

每个产品都与循环中的一条边完全对应，因此每个元素都通过其邻接关系在构建的排序中做出贡献。 

## 为什么它有效

 正确性来自于任意四个连续元素的交换参数。 如果两个较大的元素以次优方式分离，则由于当 a ≥ b 且 c ≥ d 时不等式 ac + bd ≥ ad + bc，交换段可以改善或保留目标。 反复应用这个想法会迫使结构进入一种不存在改进交换的状态。 该稳定状态恰好对应于排序元素的奇偶交替放置，这意味着任何偏差都可以得到改善，直到达到这种形式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort(reverse=True)
    
    order = []
    
    i = 0
    while i < n:
        order.append(a[i])
        i += 2
    
    i = 1
    while i < n:
        order.append(a[i])
        i += 2
    
    ans = 0
    for i in range(n):
        ans += order[i] * order[(i + 1) % n]
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先按降序对数组进行排序，这是应用结构参数所必需的。 该结构将索引分为奇数和偶数，有效地分离大元素以避免聚类。 

邻接和是在循环数组上单次计算的，模索引确保环绕。 这避免了第一个或最后一个元素的特殊外壳。 

一个常见的实现错误是忘记最后一个元素和第一个元素之间的圆形边缘。 另一个是从错误的奇偶校验开始错误地交错索引，这破坏了预期的结构。 

## 工作示例

 考虑 n = 5 且数组为 [9, 7, 5, 3, 1] 的输入。 排序后仍然是[9,7,5,3,1]。 

| 步骤| 订单已建 | 说明|
 | --- | --- | --- |
 | 赔率| [9,5,1]| 索引 0,2,4 |
 | 采取偶数| [7, 3] | 索引 1,3 |
 | 决赛| [9,5,1,7,3]| 连接|

 现在我们计算循环积：

 (9·5) + (5·1) + (1·7) + (7·3) + (3·9) = 45 + 5 + 7 + 21 + 27 = 105。 

该迹线显示了如何分离大元素，同时仍然通过交替邻接做出强烈贡献。 

对于第二个示例，采用 n = 6 和数组 [8, 6, 4, 3, 2, 1]。 

| 步骤| 订单已建 | 说明|
 | --- | --- | --- |
 | 赔率| [8,4,2]| 索引 0,2,4 |
 | 采取偶数| [6,3,1]| 索引 1,3,5 |
 | 决赛| [8, 4, 2, 6, 3, 1] | 连接|

 循环总和变为：

 (8·4) + (4·2) + (2·6) + (6·3) + (3·1) + (1·8)

 = 32 + 8 + 12 + 18 + 3 + 8 = 81。 

这些例子证实了该结构一致地交替幅度层并产生稳定的配对结构。

 ## 边缘情况

 当n = 2时，算法简化为一对，并且循环积只是在循环计算中对a1·a2计数两次。 该构造仍然有效，因为奇偶分割在每组中产生一个元素，并且循环求和正确地环绕。 

对于 n = 3，奇数组接收 a1 和 a3，而偶数组包含 a2。 最终顺序变为 [a1, a3, a2]，这保留了大元素的预期分离。 循环和仅包含所有三个边一次，并且不会遗漏任何邻接。 

当所有元素都相等时，任何排列都会给出相同的结果，但构造仍然会产生有效的循环。 该算法不依赖于值的唯一性，仅依赖于排序顺序，因此关系不会影响正确性。
