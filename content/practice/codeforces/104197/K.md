 ## 解决方案
 ## 问题理解

 我们得到了一个有向图$n$顶点。 每个顶点代表一个包含数字的位置，并且该图对特定元素（“国王”）允许的移动进行编码，或者等效地，允许位置之间的交换。 

只能沿着有向边移动。 微妙之处在于，使用边缘的能力取决于存储在其端点的当前值：来自的操作$u$到$v$当值为$u$大于该值$v$。 目标是了解从任何配置开始，我们最终是否可以重新排列元素，以便通过一系列有效操作可以实现任何排列。 

核心问题简化为有关有向图本身的结构性问题：有向运动施加的约束是否允许完全的重新排列自由。 

从约束的角度来看，这是一个典型的大规模图问题，其中$n$和$m$可以足够大，任何$O(n^2)$推理立即不可行。 这促使我们采用线性或近线性图遍历技术，例如深度优先搜索或广度优先搜索，并远离任何显式模拟交换或排列的尝试。 

一个常见的失败案例来自于假设本地交换灵活性意味着全局重新安排。 例如，如果图包含长度为 3 的有向环，但不是全局强连接的，则可能会错误地认为仅环就允许所有顶点的完全排列。 事实上，该循环之外的元素仍然是孤立的，因此只能进行部分重新排列。 

当将可达性视为对称时，会出现另一个微妙的失败。 图表可以允许从$u$到$v$但不会返回，任何忽略方向性的解决方案都会错误地得出交换在全球范围内都是可能的结论。 

## 方法

 天真的想法是直接模拟交换过程。 人们可以尝试通过重复应用有效的移动和跟踪排列来探索所有可达的配置。 这很快就变得不可能了，因为排列的数量是$n!$，甚至限制为可达状态，每个操作都会分支出多种可能性。 即使对于中等程度的状态空间也会立即爆炸$n$，使得这种方法完全不可行。 

关键的结构观察是，详细的交换规则仅在有向循环上局部起作用。 在循环内，值可以任意排列，因为重复旋转允许我们在不离开循环的情况下移动元素。 这意味着每个强连接组件的行为就像一个完全灵活的容器，其中的元素可以自由地重新排列。 

一旦我们认识到这一点，全局问题就简化为理解所有顶点是否都属于单个这样的灵活组件。 这正是强连通有向图的定义：每个顶点都可以到达其他每个顶点。 

因此，我们不需要模拟交换，只需要检查图是否是强连通的。 如果是，则基于循环的参数保证完全的排列能力。 如果不是，则某些顶点在一个方向上无法到达，因此某些交换或重新排列基本上是不可能的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | ---| --- |
 | 蛮力状态模拟 |$O(n!)$|$O(n!)$| 太慢了 |
 | 强连接检查（DFS）|$O(n + m)$|$O(n + m)$| 已接受 |

 ## 算法演练

 ## 算法演练

 1. 从顶点 1 开始，沿着所有有向边运行深度优先搜索。 这会检查从 1 可以到达哪些顶点。如果无法到达任何顶点，则 1 无法影响整个图，因此全局重新排列是不可能的。 
2. 反转图的所有边，并从顶点 1 开始再次运行深度优先搜索。这将检查原始图中的每个顶点是否都能达到 1。 如果没有此属性，则至少存在一个在传入可达性方面与 1 永久隔离的顶点。 
3. 如果两次 DFS 遍历都访问了所有顶点，则声明该图是强连通的，并得出完全重排是可能的结论。 否则，断定这是不可能的。 

我们运行两次遍历而不是尝试直接测试所有对的可达性的原因是强连接性具有经典特征：一个节点到达所有其他节点并且所有其他节点到达它是充分且必要的。 

### 为什么它有效

 有向循环内的交换参数意味着每个强连接组件的行为就像一组完全可重新排序的元素。 如果整个图是一个强连通分量，则任何两个顶点都位于某个有向环上，并且在该环内我们可以有效地任意交换元素。 这使得模拟任何元素对的转置成为可能，这足以生成任何排列。 如果图不是强连接的，则至少有两个组件被单向屏障分隔开，从而防止某些元素在两个方向上相互作用，从而阻止完全重新排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def bfs(start, adj, n):
    vis = [False] * (n + 1)
    q = deque([start])
    vis[start] = True
    cnt = 1

    while q:
        u = q.popleft()
        for v in adj[u]:
            if not vis[v]:
                vis[v] = True
                cnt += 1
                q.append(v)

    return cnt == n

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n + 1)]
    radj = [[] for _ in range(n + 1)]

    for _ in range(m):
        u, v = map(int, input().split())
        adj[u].append(v)
        radj[v].append(u)

    if bfs(1, adj, n) and bfs(1, radj, n):
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```前向 BFS 检查顶点 1 是否可以到达所有其他顶点。 反向 BFS 检查所有顶点是否可以到达顶点 1。它们一起实现标准强连接测试，以避免显式计算完整的强连接组件。 

唯一微妙的实现细节是在输入时构造反向邻接表。 这避免了在遍历期间重新计算反向边并保持解的线性。 

## 工作示例

 考虑一个有 4 个顶点和边的图$1 \to 2$,$2 \to 3$,$3 \to 1$,$3 \to 4$。 组件$\{1,2,3\}$是强连接的，但顶点 4 只能从 3 到达。

这表明循环可以实现全面交互，而当循环连接所有内容时，单向边缘不会限制全局可达性。

 ## 边缘情况

 单个顶点图始终满足强连通性，因为在两个方向上的可达性都是微不足道的。 该算法处理这个问题是因为两个 DFS 调用都从唯一的节点开始并立即访问所有顶点。 

没有边和多个顶点的图无法通过可达性检查。 从顶点1开始，没有到达其他顶点，因此前向DFS立即失败。 

完全循环图的行为符合预期：由于循环之间的相互可达性，两次 DFS 遍历都覆盖了所有顶点，从而确认了交换参数全局适用。
