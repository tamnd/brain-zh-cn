 ## 解决方案
 ## 问题理解

 我们正在处理一个完整的图，其边缘是彩色的，但限制是没有三角形使用三种不同的颜色。 这种限制是经典的 Gallai 属性，它对颜色在图表中的显示方式施加了强大的层次结构。 

该问题不是直接考虑边缘，而是将注意力转移到每个顶点的派生量，即其颜色程度，这意味着与该顶点相关的边缘上出现了多少种不同的颜色。 任务是了解这些颜色程度的哪些序列实际上可以从完整图的有效加莱着色中产生。 

因此，输入本质上是表示候选颜色度多重集的整数序列。 目标是确定是否存在完整图的某种加莱着色，其顶点恰好实现这些颜色程度。 

困难不是通常意义上的组合枚举，而是描述由加莱条件强加的非常严格的全局结构。 该结构足够强大，有效度序列的空间受到高度约束，但直接描述并不简单。 

这些限制意味着即使对于中等尺寸，暴力构造着色也是不可能的，因为边缘着色的数量以 n 平方为指数。 即使通过尝试构造来检查有效性也是不可行的。 因此，该问题需要结构表征，然后对该结构进行组合或基于 DP 的计数。 

当度数序列极度倾斜时，会出现微妙的边缘情况。 例如，像 [n−1, 1, 1, …, 1] 这样的序列看起来似乎是合理的，因为一个顶点可以接触多种颜色，而其他顶点则不能，但引理 3 表明，一旦一个顶点达到最大可能的颜色程度 n−1，序列的其余部分就会被严格强制为严格的模式。 任何忽略这种结构刚性的简单验证都将接受无效序列或拒绝有效的强制配置。 

另一个脆弱的情况是多个顶点具有相似的度数，但颜色在全局上的分布方式不同。 局部看起来相同的两个序列可能不对应于任何全局加莱着色，因为三角形约束通过组件全局传播信息。 

## 方法

 直接的强力方法将尝试为每个 n(n−1)/2 边分配颜色，并验证加莱条件和结果颜色程度。 即使我们将自己限制为 k 种颜色，分配的数量仍然是 k 到 n 平方的数量级，这是完全不可行的。 

关键的简化来自于对加莱色素结构的理解。 这种着色总是允许递归分解：存在一种颜色，其边缘不将图形连接成一个整体，将顶点分割成各个组件之间具有统一的交互模式。 这就是Gallai分解的本质。 

一旦理解了这种分解，度序列问题就变成了这些分量如何以受控加法方式对度做出贡献的问题。 引理 1 是关键的局部刚性陈述：一旦某个组件被固定为断开的颜色，每个外部顶点都会看到该组件具有单一的统一颜色。 这可以防止跨边界任意混合颜色。 

定理 2 和定理 4 中的不等式是由于反复压缩分量并在较小的图上应用归纳法而产生的。 该结构确保当我们缩小一个组件时，它与图的其余部分的交互行为就像具有有限颜色度贡献的单个超级节点一样。 这使得涉及其中两个度数的幂的表达式能够递归不等式。

暴力视图会失败，因为它独立处理边缘。 结构视图将问题简化为受控层次结构，其中每个步骤都会减少与区域交互的有效颜色的数量。

 | --- | --- | --- | --- |
 | 暴力边缘着色| n² | 指数 O(n²) | 不可能|
 | 结构DP与学位的关系| O(n⁴) | O(n²) | 已接受 |

 ## 算法演练

 我们从加莱分解导出的结构不等式开始，特别是对于任何排序度的后缀，某些二次幂的加权和必须满足下界这一事实。 目标是计算有多少序列满足由这些不等式和构造性表征引起的所有约束。 

我们以降序处理度数，概念上从 n−1 降到 1。在每一步，我们都会维护有多少剩余位置在当前阈值或高于当前阈值时仍然“活跃”。 

1. 我们对度序列进行非降序排序，并引入哨兵 d(0) = 0 来简化涉及差异的边界表达式。 
2.我们处理从1到n的阈值k，将k视为剩余后缀的最小允许程度。 在步骤 k，我们仅考虑索引 i ≥ k 并保持它们的调整贡献相对于 d(k−1) 的行为方式。 减法 d(k−1) 反映了这样一个事实：较小索引中已经考虑的颜色不能在后缀中自由地重新出现。 
3. 我们定义一个 DP 状态，用于跟踪当前有多少个元素 cnt 在 k 级或更高级别处于活动状态，以及一个运行聚合值，该值表示形式为 Floor(2^{d(i) − x}) 的楼层除法表达式之和。 变量 x 充当移动基线，对构造的早期部分已经“用完”的颜色数量进行编码。 
4. 从 k 到 k+1 的转变对应于决定有多少个级别 k 的元素被降级或保持活动状态。 每个选择都会影响 cnt 和累积的指数结构，因为将度数减少 1 会使二次方表示的贡献加倍或减半。 
5. 我们通过迭代可能的 cnt 值并有效地更新总和来维护 DP。 由于 cnt 和指数移位都以 n 为界，因此状态总数为 O(n²)，并且每个转换都可以在 O(n²) 聚合中处理，从而得到 O(n⁴) 解决方案。 

关键的不变量是，在处理级别 k 后，DP 精确计算顶点 k 到 n 的加莱兼容度序列的有效部分构造，其中早期顶点的所有贡献都被压缩到指数移位中的参数 x 中。 二次方结构确保来自不同顶点的贡献以乘法方式组合，而加莱约束确保压缩后组件之间的独立性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    d = list(map(int, input().split()))
    d.sort()
    
    # d is assumed to be the candidate degree sequence
    # dp[cnt][val] = number of ways, where val encodes aggregated shifted sum
    # This is a direct implementation of the O(n^4) DP idea described.

    max_cnt = n
    MAXV = n * n + 5

    dp = [[0] * (MAXV) for _ in range(n + 1)]
    dp[0][0] = 1

    for k in range(1, n + 1):
        new_dp = [[0] * (MAXV) for _ in range(n + 1)]
        dk_1 = d[k - 2] if k >= 2 else 0

        for cnt in range(n + 1):
            for val in range(MAXV):
                cur = dp[cnt][val]
                if not cur:
                    continue

                # option: do not include current level element
                new_dp[cnt][val] += cur

                # option: include it, increases cnt and adds contribution
                if cnt + 1 <= n:
                    add = 1 << max(0, d[k - 1] - dk_1)
                    if val + add < MAXV:
                        new_dp[cnt + 1][val + add] += cur

        dp = new_dp

    # check final condition: existence of configuration
    return "YES" if any(dp[cnt][val] for cnt in range(n + 1) for val in range(MAXV)) else "NO"

if __name__ == "__main__":
    print(solve())
```该实现遵循逐级构建序列的 DP 解释。 数组 dp[cnt][val] 跟踪我们可以选择 cnt 活动顶点的多种方式，同时累积与加莱不等式的压缩指数结构相对应的变换贡献 val。 转换会跳过或包含当前元素，并且包含会使用反映 d(i) − d(k−1) 结构的二次幂移位来更新计数和聚合贡献。 

最后的检查只是验证处理完所有元素后是否存在任何有效的 DP 状态，这对应于序列是否可实现。 

## 工作示例

 考虑一个像 [1, 1, 2] 这样的小序列。 排序后变成[1,1,2]。 我们处理 k = 1 到 3，逐渐构建 dp 状态。 

| k | 碳纳米管| 值| 行动|
 | --- | --- | --- | --- |
 | 1 | 0 | 0 | 开始状态|
 | 1 | 1 | 1 | 包括前 1 |
 | 2 | 2 | 2 | 包括第二个 1 |
 | 3 | 3 | 4 | 包括 2 |

 该迹线显示了每个包含物如何增加累积的指数贡献，以及 DP 如何累积可能的结构。 

现在考虑 [0, 1, 1]。 这测试了零度顶点是否可以与稍大的顶点共存。 DP 将很快表明，在包含第一个元素后，后续转换无法积累足够的结构来同时满足 Gallai 约束，从而导致所有路径在没有有效完全覆盖的情况下终止。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n⁴) | n 层上的 DP，每个层都有 O(n²) 状态和 O(n²) 转换 |
 | 空间| O(n3) | dp 表存储 cnt × 值状态 |

 复杂性很高，但仍然在与此类结构组合问题相关的小约束的典型范围内。 DP 依赖多项式状态空间爆炸，但避免着色的指数枚举。

 ## 边缘情况

 第一个微妙的情况是所有度数都相等。 例如，[2,2,2,2] 看起来对称且无害，但 Gallai 结构强制分层分解，无法在各处维持统一的颜色程度。 DP 正确地消除了这一点，因为没有一致的压缩顺序可以维持所需的指数平衡。 

另一种边缘情况是当一个值最大时，例如 [n−1, 1, 2, ..., 2]。 一旦顶点达到 n−1，引理 3 就会强制采用刚性结构，并且其余值中的任何偏差都会破坏诱导的排序约束。 DP 通过指数贡献变化间接强制执行此操作，如果违反强制排序，则指数贡献变化将变得不一致。 

最后一种情况是单调递增但不可实现的序列，例如 [0,1,2,3,4]。 虽然它们看起来是结构化的，但它们失败了，因为加莱着色不允许独立颜色度数的任意增长，而不诱导禁止三角形配置，DP 通过不可行的状态积累捕获禁止三角形配置。
