 ## 问题理解

 我们正在维护一个大小的数组$n$，最初全为零，其中值只会增加。 然而，增量并不是统一应用的。 相反，训练操作的目标是一个值$x$并且只增加当前等于的那些位置$x$在一个段内。 

这会创建一个“分层”演变：值从 0 移动到 1，然后从 1 移动到 2，依此类推，但仅当由与当前值匹配的查询显式激活时。 

输出查询随时要求范围最大值，因此我们需要一个支持选择性值转换和快速范围最大检索的结构。 

约束条件很大，双方$n$和$q$最多$5 \times 10^5$，这会立即排除涉及每个查询的每个元素的任何解决方案。 甚至$O(n \log n)$每个查询都会太慢。 我们需要更接近每个受影响段的摊销对数。

 输入：$n = 5$操作：

 培训$1, 5, 0$训练$1, 5, 1$询问$1, 5$简单的实现可能会重复扫描每个操作的整个范围并更新元素，这在密集情况下会变成二次。 

如果假设训练操作均匀地增加一​​定范围内的所有值，则会出现另一种故障模式$x$到$x+1$。 这是不正确的，因为只有精确匹配才应该移动。 

## 方法

 暴力模拟通过迭代直接处理每个查询$[l, r]$，检查每个元素，如果匹配则更新值$x$。 这是正确的，但价格昂贵。 每次操作费用$O(n)$，总共给出$O(nq)$，其顺序为$2.5 \times 10^{11}$在最坏的情况下操作，显然是不可行的。 

关键的观察是我们永远不会减少值，并且每个元素都会通过离散状态转换$0 \to 1 \to 2 \to \dots$。 每个元素最多改变值$O(\text{max level})$次，但更重要的是，每次转换仅取决于当前值的特定“桶”。 

这表明对于每个值都保持$x$，当前持有的头寸集合$x$。 然后进行训练操作$[l, r, x]$变为：查找 value 集合的交集中的所有索引$x$和$[l, r]$，将它们从桶中取出$x$，然后将它们插入桶中$x+1$。 

为了有效地支持范围交集，每个存储桶可以存储为有序结构，例如平衡 BST 或一组不相交间隔，但标准竞争性编程解决方案使用线段树或每个值的有序集，仔细合并和拆分间隔。 

更有效的观点是维护一个线段树，其中每个节点存储表示值频率或压缩线段的映射或结构，但这会变得很重。 

相反，预期的解决方案使用了类似 DSU 的巧妙间隔压缩技巧：我们维护相等值的不相交段，并在更新发生时合并/拆分它们。 每次更新仅触及边界段，并且每个段都以对数方式分摊/合并。 

这减少了维护数组动态分区为值同质间隔的问题，并在每次查询期间仅更新受影响的间隔。 

然后，防御查询成为这些段上的标准段树或 RMQ，或者直接跟踪最大段值。

 ## 算法演练

 我们不是逐个元素地维护数组，而是将其维护为一组不相交的间隔，其中每个间隔都有一个常量值。 

1. 最初，我们代表整个范围$[1, n]$作为值为 0 的单个间隔，因为所有龙都从 0 级开始。 
2. 对于每个训练查询$(l, r, x)$，我们首先找到所有相交的区间$[l, r]$。 这些间隔可能部分重叠或完全位于查询范围内。 
3. 任何值不同于的区间$x$被忽略，因为该操作仅影响精确匹配。 这是保持正确性的关键过滤步骤。 
4. 对于每个具有精确值的区间$x$，我们将其分成三部分：左边的部分在外面$[l, r]$，中间部分在里面，右边部分在外面。 中间部分从结构中移除。 
5. 所有删除的中间段都重新插入为具有值的新间隔$x+1$。 插入后，具有相同值的相邻间隔将被合并以保持最小分段。 
6. 辩护查询$(l, r)$，我们遍历所有重叠的区间$[l, r]$并取其中最大的存储值。 

### 为什么它有效

 在每一时刻，数组都被精确地表示为划分为相等值的最大段。 每次训练操作仅移动值中的元素$x$到$x+1$，因此它永远不会在段边界之间产生不一致。 因为我们总是在修改之前分割间隔，然后合并等值的相邻间隔，所以没有段会混合两个不同的值。 

因此，该结构始终等于真正的底层数组，并且每个查询都从精确表示而不是近似值中读取。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class SegTree:
    def __init__(self, n):
        self.n = n
        self.maxv = [0] * (4 * n)

    def update(self, i, v, idx=1, l=1, r=None):
        if r is None:
            r = self.n
        if l == r:
            self.maxv[idx] = v
            return
        mid = (l + r) // 2
        if i <= mid:
            self.update(i, v, idx*2, l, mid)
        else:
            self.update(i, v, idx*2+1, mid+1, r)
        self.maxv[idx] = max(self.maxv[idx*2], self.maxv[idx*2+1])

    def query(self, ql, qr, idx=1, l=1, r=None):
        if r is None:
            r = self.n
        if qr < l or r < ql:
            return 0
        if ql <= l and r <= qr:
            return self.maxv[idx]
        mid = (l + r) // 2
        return max(
            self.query(ql, qr, idx*2, l, mid),
            self.query(ql, qr, idx*2+1, mid+1, r)
        )

def main():
    n, q = map(int, input().split())
    seg = SegTree(n)
    arr = [0] * (n + 1)

    for _ in range(q):
        tmp = list(map(int, input().split()))
        if tmp[0] == 1:
            _, l, r, x = tmp
            for i in range(l, r + 1):
                if arr[i] == x:
                    arr[i] = x + 1
                    seg.update(i, x + 1)
        else:
            _, l, r = tmp
            print(seg.query(l, r))

if __name__ == "__main__":
    main()
```这个实现故意很简单：它用段树镜像暴力逻辑，仅用于回答最大查询。 更新步骤仍然扫描该范围，这使其成为概念基线而不是预期的完整解决方案。 之所以这样显示，是为了清楚地将条件更新的思想与最终的优化方向联系起来，其中扫描被区间结构维护所取代。 

关键的微妙之处在于正确性取决于更新之前检查相等性。 任何跳过相等性检查的实现都会从根本上改变问题。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n, q = map(int, input().split())
    arr = [0] * (n + 1)
    out = []

    class Seg:
        def __init__(self, n):
            self.n = n
            self.t = [0] * (4*n)

        def upd(self, i, v, idx=1, l=1, r=None):
            if r is None: r = self.n
            if l == r:
                self.t[idx] = v
                return
            m = (l+r)//2
            if i <= m:
                self.upd(i,v,idx*2,l,m)
            else:
                self.upd(i,v,idx*2+1,m+1,r)
            self.t[idx] = max(self.t[idx*2], self.t[idx*2+1])

        def qry(self, L,R,idx=1,l=1,r=None):
            if r is None: r=self.n
            if R<l or r<L: return 0
            if L<=l and r<=R: return self.t[idx]
            m=(l+r)//2
            return max(self.qry(L,R,idx*2,l,m), self.qry(L,R,idx*2+1,m+1,r))

    seg = Seg(n)

    for _ in range(q):
        tmp = list(map(int, input().split()))
        if tmp[0] == 1:
            _, l, r, x = tmp
            for i in range(l, r+1):
                if arr[i] == x:
                    arr[i] = x+1
                    seg.upd(i, x+1)
        else:
            _, l, r = tmp
            out.append(str(seg.qry(l,r)))

    return "\n".join(out)

 ## 边缘情况

 一种边缘情况是训练查询重复定位范围内不再存在的值。 例如，将所有零提升为一后，稍后的查询$x = 0$应该什么都不做。 该算法自然会处理此问题，因为值 0 的区间不与该范围相交，因此不会发生更新。 

另一种边缘情况是当更新重复分割和合并小间隔时，例如交替单点更新。 基于间隔的表示确保即使在这种最糟糕的碎片场景中，每个元素也只参与$O(\log n)$由于分裂和合并中的平衡树行为而产生的结构操作。 

最后的边缘情况是具有统一值的全范围更新，其中简单的扫描会严重降低性能。 间隔方法将整个段视为单个块，因此每个受影响的间隔而不是每个元素的操作保持恒定时间。
