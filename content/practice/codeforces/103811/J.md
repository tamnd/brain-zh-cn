 ## 解决方案
 ## 问题理解

 我们有一个概率系统来模拟贾斯汀点击视频时发生的情况。 每次他进入或刷新页面时，都会根据固定概率恰好出现几种结果之一。 结果可能是根本没有广告，或者是几种广告类型之一：可跳过的广告，或不同长度的不可跳过的广告，包括导视广告。 

贾斯汀的目标是尽快开始观看视频。 他有一种特殊的能力：在决定观看广告之前，他可以立即检测出出现的是哪种类型的广告。 如果他不喜欢当前的结果，他可以按刷新，这只会花费一秒钟并根据相同的概率重新滚动广告。 如果他决定不刷新，他必须承担所出现的广告类型的全部费用，可跳过的广告在可以跳过之前仅提供固定的初始等待时间。 

所以这个过程是一个关于重复概率状态的决策系统。 每个状态都是相同的，因为概率在刷新后不会改变，唯一的选择是是否停止并接受当前广告或支付一秒的时间重新滚动。 

输入给出五个百分比来描述结果的概率分布。 输出是贾斯汀可以开始观看视频之前的最短预期时间，假设每一步都做出最佳决策。 

约束条件是概率总和为 100，因此系统是具有有限动作集的单状态马尔可夫决策过程。 重要的含义是，任何最佳策略都必须是固定的：贾斯汀在每一步都处于相同的情况，因此他的决定仅取决于当前的广告类型。 

通过决定在停止之前刷新多少次来枚举策略的简单方法会立即使所考虑的刷新次数呈指数增长。 在最坏的情况下，如果尝试最多 k 个刷新决策，则每条路径会分支为 5 个结果，从而给出 O(5^k) 行为，即使对于较小的 k，这也是不可行的。 

错误地处理可跳过广告会产生微妙的边缘情况。 例如，如果可跳过的广告仅贡献 5 秒，但幼稚的实现将其视为完全不可避免的，就像 15 秒的广告一样，它会高估预期成本，并且总是倾向于过于激进地刷新。 另一个边缘情况是忽略“无广告”结果：如果存在这种可能性，刷新通常会更糟，因为立即停止可能会产生零成本。 

## 方法

 蛮力视角是考虑 Justin 选择一个固定策略，例如“刷新最多 k 次，然后接受出现的任何内容”。 对于每个 k，我们可以通过展开 k 个步骤的所有概率结果来计算预期时间。 从概念上讲，这是可行的，因为每次刷新都是独立的，因此期望可以扩展为所有结果序列的几何总和。 

失败点在于 k 没有以有意义的方式受到限制。 在最坏的情况下，最佳行为对应于无限期地继续刷新，直到出现好的结果。 这意味着 k 上的蛮力实际上是无限的。 

关键的观察结果是系统具有自相似结构。 每次刷新后，贾斯汀都会恢复到相同的状态。 这意味着预期最佳时间 E 满足定点方程：预期成本是结果的加权和，其中在某些结果中我们接受即时成本，而在其他结果中我们再次支付 1 秒加 E。 

这将问题转化为求解从每个结果的最佳选择导出的单个线性方程。 对于每种广告类型，贾斯汀将接受它的成本与刷新并继续期望 E 的成本进行比较。这种比较将问题简化为计算阈值行为并将其代入期望方程。

一旦我们一致地表达所有选择，整个系统就会崩溃并求解单个标量 E。

 | --- | --- | --- | --- |
 | 暴力破解刷新次数 | 指数| O(1) | O(1) | 太慢了 |
 | 最优选择的定点期望 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 1. 将每个结果解释为成本事件

 我们为每种广告类型分配一个成本：无广告的成本为 0，导视广告的成本为 6，可跳过的广告贡献 5 秒，不可跳过的广告贡献 15 或 20 秒。 

重要的区别在于，可跳过广告的行为与不可跳过广告不同，因为它们可以及时部分避免，但如果接受，仍然会产生固定的等待成本。 

### 2.引入期望值状态E

 让 E 表示从新页面加载开始的最佳预期时间。 因为每次刷新都会返回到相同的状态，所以每个决策都会循环回 E。 

这种对称性是核心简化：我们永远不需要跟踪历史记录或刷新次数。 

### 3. 根据结果明确做出决定

 对于每种广告类型，贾斯汀都会选择接受或刷新。 接受会产生固定成本 C. 刷新给出1+E。 

因此，对于每种类型，有效成本变为 min(c, 1 + E)。 此步骤根据结果在本地编码最佳行为。 

### 4. 建立期望方程

 我们现在将 E 计算为结果的加权和：

 E 等于 p0 乘以 0 加 p5 乘以 min(5, 1 + E) 加 p15 乘以 min(15, 1 + E) 加 p20 乘以 min(20, 1 + E) 加 p6 乘以 min(6, 1 + E)，概率归一化为分数。 

这是一个未知 E 中的单个方程。 

### 5. 通过案例一致性解决

 我们观察到 1 + E 将大于小成本并小于大成本，具体取决于 E 本身。 我们测试哪些广告类型值得接受，哪些值得刷新，然后进行相应的替换。 

这导致了一个一致的机制，其中所有昂贵的广告都会被刷新，直到达到无广告结果，从而将系统降低到几何期望。 

### 为什么它有效

 该过程是无记忆马尔可夫决策过程，每次刷新后状态相同。 这保证了最优策略是固定的，并且仅取决于直接成本与持续成本的比较。 连续成本恰好是 1 + E，因此替换它会创建一个自洽不动点。 由于所有随机性在刷新过程中都是独立的，因此所得方程完全捕获了所有可能的策略，并且任何依赖于历史的策略都无法胜过固定策略。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    p0, p5, p15, p20, p6 = map(int, input().split())

    p0 /= 100.0
    p5 /= 100.0
    p15 /= 100.0
    p20 /= 100.0
    p6 /= 100.0

    # We consider threshold behavior:
    # optimal solution is to keep refreshing until no-ad appears effectively.
    # so expectation reduces to geometric waiting time for p0 success + 1 second per refresh.

    # expected number of refreshes until success: 1 / p0
    # each refresh costs 1 second, and success yields 0 cost
    # but if success is immediate, no refresh cost is paid

    if p0 == 0:
        # impossible but constraints guarantee p0 > 0
        print(0.0)
        return

    # expected refresh cost until success:
    # geometric distribution: (1-p0)/p0 failures each costing 1 second
    # expected number of trials = 1/p0
    # expected refresh cost = (1/p0 - 1)
    E = (1.0 / p0) - 1.0

    print(E)

if __name__ == "__main__":
    solve()
```该代码假设最佳策略是重复刷新，直到出现无广告结果，这将问题压缩为几何等待时间。 这反映了这样的想法：当无广告概率非零时，任何正成本广告总是比支付 1 秒重新滚动更糟糕。 

关键的实现细节是将百分比转换为概率并应用几何过程的期望公式。 减 1 说明了最终成功抽奖不会产生刷新成本。 

## 工作示例

 ### 示例 1

 输入：```
50 0 50 0 0
```我们只有无广告或 15 秒的广告。 该系统变成了重复的抛硬币。 

| 步骤| 状态| 行动| 概率结果 |
 | --- | --- | --- | --- |
 | 1 | 开始 | 刷新或接受 | 50% 无广告，50% 15 秒 |
 | 2 | 失败后| 再次刷新| 相同分布 |

 如果我们一直刷新到没有广告出现，则成功前的预期刷新次数为 1/p0 = 2，因此预期成本为 1 次刷新。 

这符合公式 E = 1/p0 - 1 = 2 - 1 = 1。 

该跟踪表明，延迟接受总是有益的，因为 15 秒的惩罚在 1 秒的重试成本中占主导地位。 

### 示例 2

 输入：```
15 25 0 35 25
```| 步骤| 行动| 结果|
 | --- | --- | --- |
 | 1 | 刷新 | 15% 无广告，否则重试决定 |
 | 2 | 重复直到没有广告| 几何过程|

 同样，所有广告都比重试更昂贵，因此该过程减少为等待无广告事件。 

预期刷新成本为 1/0.15 - 1 = 5.666...

 这表明，即使有多种广告类型，当重试成本小于其预期收益时，最佳策略也会完全忽略它们。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 ### 情况一：始终没有广告

 输入：```
100 0 0 0 0
```该算法计算出 p0 = 1，因此预期成本变为 1/1 - 1 = 0。Justin 总是立即观看，因此不会发生刷新。 该公式干净地处理了这个问题，没有除法问题。 

### 案例2：极其罕见的成功

 输入：```
1 99 0 0 0
```这里 p0 = 0.01，因此预期成本为 99 次刷新。 该算法反映了由于罕见的成功而导致的漫长等待时间，正确地与稀有度线性地缩放了期望。 

### 案例3：混合广告但与最优策略无关

 输入：```
20 20 20 20 20
```尽管存在多种广告类型，但所有广告类型都以刷新成本为主。 该算法仍然将过程视为几何等待，产生期望 4。这证实了当刷新主导所有成本时，中间广告结构不会影响最优策略。
