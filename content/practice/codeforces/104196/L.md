 ## 解决方案
 ## 问题理解

 我们有一个代表公园的小网格。 每个单元要么是空水，标记为 -1，要么包含一个具有唯一正高度的雕像。 所有雕像都是不同的，因此我们可以认为它们具有从小到大的严格全局顺序。 

公园管理员想要根据非常具体的规则“重新排列”雕像。 我们首先选择网格的四个角之一。 该选择决定了一系列对角线：单元格按照距该角的曼哈顿式对角线距离进行分组。 一旦这个角被固定，所有单元格都位于对角线上，从角开始按递增顺序编号。 

然后，我们通过增加高度对所有雕像进行排序，并将它们按顺序分配到这些对角线组中。 最小的雕像进入第一个对角线组，下一批进入第二个对角线组，依此类推。 每个批次的大小取决于每个对角线上存在多少个非水细胞。 水细胞会被忽略并且不会收到雕像。 

在对角组内，放置是灵活的。 分配给该对角线的任何雕像都可以放入该对角线的任何有效单元格中。 这种自由意味着唯一重要的是雕像最终分配到哪条对角线，而不是它在对角线内的确切位置。 

配置的成本是尚未位于与其指定对角线匹配的单元中的雕像数量。 同样，我们希望最大化最终分配到与其原始位置的对角线相匹配的对角线的雕像数量。 

任务是尝试所有四个角的选择并选择最佳的排列方式，最大限度地减少必须移动的雕像数量。 

网格最多为 50 x 50，因此最多有 2500 个单元格。 这立即排除了任何尝试显式排列分配或以阶乘或指数方式模拟重新排列的解决方案。 甚至$O(n^3)$方法是安全的，但考虑到较小的限制，任何涉及按配置排序或重复匹配的方法也可以。 

一个微妙的边缘情况是水细胞不均匀地打破对角线。 例如，具有相同几何结构的两条对角线可能具有不同的容量，具体取决于存在多少个水单元。 这很重要，因为雕像的分配仅取决于对角线容量，而不仅仅取决于几何形状。 

另一个陷阱是假设我们可以独立地将雕像放置在对角线内并贪婪地对待每个单元格。 这是错误的：限制来自于按高度排序，这会全局固定哪些雕像进入哪个对角线。 

## 方法

 暴力解释将尝试模拟整个过程。 固定一个角，然后重复将剩余的最小雕像分配到当前对角线上，直到填满，移动到下一个对角线，然后继续。 构建完整的最终布局后，我们将其与原始网格进行比较并计算不匹配的情况。 这已经接近预期的过程了，而且是正确的，但是对每个角落仔细地做这件事仍然需要排序和记账。 

关键的观察是，我们实际上不需要模拟对角线内单个雕像的运动。 一旦角点被固定，分配结构就变得确定了：

 每条对角线都有固定的容量，等于其中非水单元的数量。 当我们按高度对雕像进行排序时，它们会被分成大小与这些容量相匹配的连续块。 因此，每个雕像都纯粹根据其在排序列表中的排名来分配对角线。 

这将整个问题简化为比较问题。 对于固定角点，我们计算两件事：每个单元格的对角线索引，以及分配给该对角线的排名间隔。 然后我们简单地检查有多少雕像最初位于对角线索引与分配给它们的对角线相匹配的单元格中。 

最佳答案是四个角上最好的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 模拟每个角的分配 |$O(4 \cdot n m \log(nm))$|$O(nm)$| 已接受 |
 | 直接对角线映射+计数|$O(4 \cdot n m \log(nm))$|$O(nm)$| 已接受 |

 两者最终的复杂性相似，但后者避免了不必要的模拟并使正确性透明。 

## 算法演练

 我们独立处理四个可能的角点。 

### 1. 计算对角线恒等式

 对于固定角点，根据每个单元格与该角点的距离为每个单元格分配一个对角线索引。 例如，从左上角开始，对角线的索引为$i + j$。 从其他角度来看，公式也会相应变化。 

水电池在结构上仍然是对角线的一部分，但它们不贡献容量。 

此步骤是必要的，因为所有后续推理都取决于按所选方向一致地对单元格进行分组。 

### 2. 计算对角线容量

 对于每个对角线索引，计算有多少个单元格不是水。 这给出了将分配给该对角线的雕像数量。 

这个容量是唯一决定每个对角线有多少个等级的因素。 

### 3.按高度对雕像进行排序

 从网格中提取所有雕像值并对它们进行递增排序。 由于所有高度都是不同的，这会产生从小到大的严格排序。 

此排序是用于分配的全局序列。 

### 4. 构建每个对角线的排名间隔

 按索引递增顺序遍历对角线。 在已排序的雕像列表上保持一个运行指针。 对于每个具有容量的对角线$c$，将其分配给下一个$c$雕像按顺序排列。 这为每个对角线创建了一个连续的等级区间。 

此步骤将整个分配规则编码为简单的索引范围。 

### 5. 与原来的放置比较

 对于网格中的每个雕像，确定两个值：其原始对角线索引及其在排序列表中的排名。 使用排名，找到它被分配到的对角线间隔。 如果这些匹配，则雕像不需要移动。 

计算有多少雕像满足这个条件。 

### 6. 尝试所有角落

 对所有四个角重复步骤 1 至 5，并获取最大数量的固定雕像。 答案是雕像总数减去这个最大值。 

### 为什么它有效

 一旦角点被固定，该过程就会定义从排序等级到对角线的确定性映射。 在每条对角线内部，充分的灵活性消除了位置限制。 唯一重要的不变量是雕像的原始对角线是否与其指定的对角线间隔相匹配。 由于所有分配仅取决于连续的排名段，因此问题简化为匹配同一组单元格的两个固定标签。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(n)]

    statues = []
    for i in range(n):
        for j in range(m):
            if grid[i][j] != -1:
                statues.append((grid[i][j], i, j))

    if not statues:
        print(0)
        return

    statues.sort()  # by height
    total = len(statues)

    def get_diag_id(i, j, corner):
        if corner == 0:  # top-left
            return i + j
        if corner == 1:  # top-right
            return i + (m - 1 - j)
        if corner == 2:  # bottom-left
            return (n - 1 - i) + j
        return (n - 1 - i) + (m - 1 - j)

    best = 0

    for corner in range(4):
        diag_cells = {}
        diag_id = [[0] * m for _ in range(n)]

        for i in range(n):
            for j in range(m):
                d = get_diag_id(i, j, corner)
                diag_id[i][j] = d
                if grid[i][j] != -1:
                    diag_cells[d] = diag_cells.get(d, 0) + 1

        # build assignment ranges
        order = sorted(diag_cells.keys())
        start = {}
        cur = 0
        for d in order:
            start[d] = cur
            cur += diag_cells[d]

        # for each statue, check match
        fixed = 0
        for rank, (val, i, j) in enumerate(statues):
            d = diag_id[i][j]
            # find which diagonal this rank goes to
            # locate d's interval by checking next boundary
            # since small, linear scan over keys is fine
            # but we precompute list of boundaries
            # build once
            pass

        # rebuild intervals properly
        boundaries = []
        for d in order:
            boundaries.append((start[d], start[d] + diag_cells[d], d))

        fixed = 0
        for rank, (val, i, j) in enumerate(statues):
            d_orig = diag_id[i][j]
            for L, R, dd in boundaries:
                if L <= rank < R:
                    if dd == d_orig:
                        fixed += 1
                    break

        best = max(best, fixed)

    print(total - best)

if __name__ == "__main__":
    solve()
```代码首先提取所有雕像单元并按高度对它们进行排序，这为我们提供了全局分配顺序。 对于每个角，它一致地计算每个单元格的对角线标识符。 然后，它计算每个对角线必须接收多少个雕像，这完全确定了排序后的排名到对角线间隔的划分。 

稍微间接的部分是将排名映射回对角线间隔。 这是通过扫描区间列表来实现的，这是可以接受的，因为对角线的数量最多为$n + m$，以 100 为界。 

最后，我们将每个雕像的原始对角线与分配给其等级的对角线进行比较。 每场比赛都对应一个留在原地的雕像。

 ## 边缘情况

 当对角线仅包含水细胞时，就会出现极端情况。 在这种情况下，其容量为零，因此不会为其分配任何等级。 该算法自然会处理这个问题，因为这样的对角线永远不会出现在分配区间中。 

另一种情况是所有雕像都位于一个角下的同一条对角线上。 那么所有雕像都属于一个区间，这意味着每个雕像都被分配了相同的对角线。 该算法仍然有效，因为每个等级都映射到该一个区间，并且比较减少为检查原始对角线一致性。 

第三种情况是多个角产生相同的对角分区。 该算法独立评估每个结构，因此重复的结构不会影响正确性或性能。 

这些行为直接源于以下事实：对角线纯粹被视为容量桶，而空桶在排名分配中根本不贡献任何间隔。
