 ## 解决方案
 ## 问题理解

 我们得到从 1 到 m 的数字的随机排序。 可以将其视为对不同的标记进行洗牌并一一揭示它们。 

除此之外，我们还得到了一张由 n 个不相交的数字对组成的“卡片”。 从 1 到 m 的每个数字最多出现在一对中，因此该结构是不同顶点上的一组 n 个边，其余 m − 2n 个数字未使用。 

一旦一对数字中至少有一个出现在显示的序列中，就认为该对已完成。 如果所有对都完成的那一刻正好发生在第 p 个显示的数字上，则玩家恰好获胜。 这意味着两个同时发生的条件。 在揭示前 p 个数字后，每对都至少被命中一次。 在揭示前 p − 1 个数字后，至少有一对仍然完全未被触及，并且在第 p 步时，最后一个缺失的对变得完整。 

任务是计算该事件在 1 到 m 的均匀随机排列上的概率，并将其输出为约简分数。 

约束足够小，使得对子集的指数推理是可行的。 关键参数是 m ≤ 33，这意味着 n ≤ 16。这立即表明任何迭代对子集的方法（例如 2^n 包含-排除或对掩码上的 DP）都是可行的。 然而，直接迭代数字的排列或子集是不可能的，因为 m! 即使 m = 33，增长也太快。 

当 p = 0 时，会出现一种微妙的边缘情况。在这种情况下，我们询问在任何抽奖之前是否已经完成所有对，这只有在 n = 0 时才有可能。另一种边缘情况是当 p 太小而无法覆盖 n 对时，因为每个步骤最多只能完成一对新对，从而使某些配置不可能并强制概率为零。 

## 方法

 强力解释将枚举 m 个数字的所有排列并模拟该过程，检查最后一个未覆盖的对是否在步骤 p 处被精确覆盖。 这在概念上是正确的，但立即不可行。 即使 m = 20 也已经成为 m！ 天文数字般大，这里 m 可以是 33。 

使问题可解的结构是事件仅取决于哪些数字出现在前 p-1 个位置以及哪个元素位于位置 p。 剩余后缀的内部顺序与有效性无关，仅与计算重数有关。 

这使我们能够从排列切换到组合分解：选择一个大小为 p − 1 的集合 T 表示前 p − 1 个元素，并选择一个与 T 不相交的位置 p 的元素 x。每个这样的对 (T, x) 恰好对应于 (p − 1)！ · (m − p)! 排列，因为 T 的内部顺序和后缀可以自由排列。 

我们现在用集合重新表述条件。 必须恰好存在一对 k，其两个端点都不在 T 中。其他每一对必须在 T 中至少有一个端点。此外，x 必须属于该特殊对 k，以便它恰好在时间 p 完成。 

核心困难在于计算大小为 p − 1 的子集 T，“击中”除一对之外的所有对，同时完全避免该一对。 因为 n ≤ 16，我们可以固定特殊对 k 并对剩余的 n − 1 对使用包含-排除来计算有效 T。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举排列 | O(米！) | O(1) | O(1) | 太慢了|
 | 子集 + 包含-排除对 | O(n·2^n) | O(n·2^n) | O(2^n) | O(2^n) | 已接受 |

 ## 算法演练

 我们通过确定哪一对是最后激活的来构建解决方案。

1. 将 k 对固定为候选“步骤 p 之前的最终未覆盖对”。 我们假设 k 的两个元素都不存在于前 p − 1 个位置中，并且其中一个恰好出现在位置 p 处。 
2. 定义前 p − 1 个位置的可用元素池。 该池包含除 k 的两个元素之外的所有数字，因此它的大小为 m − 2。 
3. 现在，我们对从该池中抽取的大小为 p − 1 的子集 T 进行计数，使得每隔一对 j ≠ k 都将其两个元素中的至少一个贡献给 T。这是对对的标准覆盖约束。 
4. 对剩余的 n − 1 对使用包含-排除。 对于这些对的子集 S，我们对 T 避开 S 中每对的两个端点的配置进行计数。如果选择 S，我们将删除 2|S| 从池中取出元素，留下 m − 2 − 2|S| 可用的选择。 在这个受限宇宙中选择 T 的方法数量为 C(m − 2 − 2|S|, p − 1)。 包含-排除符号是(−1)|S|。 
5. 对所有 S 求和得出该固定 k 的有效 T 的数量。 
6. 对于每个有效的 T，第 p 个元素 x 可以通过两种方式选择，对应于 k 对的两个端点。 
7. 每个配置 (T, x) 对应于 (p − 1)！ T 的内部排列和 (m − p)! 后缀的排列。 相应地相乘。 
8. 将 k 的所有选项相加，然后除以 m！ 以获得最低限度的最终概率。 

正确性取决于以下事实：事件“除 k 之外的所有对都在 T 中命中”恰好是 n − 1 个独立 2 元素集上的命中条件，包含-排除可以干净地解决。 

### 为什么它有效

 该过程将排列分成三个独立的组成部分：大小为 p − 1 的子集、显着的第 p 个元素和后缀排列。 该条件仅取决于哪些元素出现在前缀中以及每对是否至少命中一次。 因为每对恰好贡献两个元素并且 n 很小，所以对之间的依赖关系可以通过对避免事件的子集包含-排除来完全捕获。 前缀内的排序信息不会影响成员资格之外的有效性，这使得减少听起来很合理。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def nCk(n, k):
    if k < 0 or k > n:
        return 0
    k = min(k, n - k)
    num = 1
    den = 1
    for i in range(k):
        num *= (n - i)
        den *= (i + 1)
    return num // den

def solve():
    m, n, p = map(int, input().split())

    if n == 0:
        return "1/1" if p == 0 else "0/1"
    if p == 0:
        return "0/1"

    if p < n:
        return "0/1"

    pairs = n
    others = pairs - 1

    # factorials for counting permutations
    fact = [1] * (m + 1)
    for i in range(1, m + 1):
        fact[i] = fact[i - 1] * i

    prefix_fact = fact[p - 1]
    suffix_fact = fact[m - p]

    favorable = 0

    for k in range(pairs):
        # inclusion-exclusion over remaining pairs
        res = 0
        for mask in range(1 << others):
            bits = 0
            avail = m - 2
            for j in range(others):
                if mask & (1 << j):
                    bits += 1
                    avail -= 2

            ways = nCk(avail, p - 1)
            if bits % 2 == 1:
                res -= ways
            else:
                res += ways

        # two choices for the p-th element
        res *= 2

        favorable += res

    favorable *= prefix_fact * suffix_fact

    total = fact[m]

    g = gcd(favorable, total)
    favorable //= g
    total //= g

    return f"{favorable}/{total}"

if __name__ == "__main__":
    print(solve())
```该实现直接反映了组合分解。 阶乘用于计算与每个有效的前缀-后缀分割相对应的排列数量。 包含-排除循环枚举剩余对的子集并相应地调整可用元素池。 

一个微妙的细节是池大小从 m − 2 开始，因为所选的最终对 k 完全排除在前缀之外。 包含-排除子集中的每个排除对恰好删除两个元素，这就是为什么术语 m − 2 − 2|S| 出现。 

## 工作示例

 ### 示例 1

 输入：```
10 4 5
```我们确定最后一对 k 的候选者。 对于该对，我们排除它的两个元素，为前 4 个位置留下 8 个元素。 我们使用包含-排除来计算命中剩余 3 对的大小为 4 的子集。 每个有效子集对应最终位置的 2 个选择。 

| 步骤| 意义| 价值|
 | --- | --- | --- |
 | 泳池尺寸| 米- 2 | 8 |
 | 前缀大小 | p − 1 | 4 |
 | 有效 T 计数 | IE 过对 | 计算值|
 | x 的选择 | k | 的端点 2 |

 对所有 k 求和并按 10! 归一化后，结果简化为 8/45，与预期输出匹配。 

这个例子表明，多对竞争成为最终未被覆盖的对，并且 k 上的对称性贡献了 n 的乘法因子。 

### 示例 2

 输入：```
10 4 3
```这里 p = 3，但至少存在 4 对必须被击中。 由于每个前缀元素最多可以覆盖一对，因此在两次抽奖之后，不可能除了一对之外的所有都已经击中。 由于大小为 2 的子集无法完全覆盖 4 对减 1，因此包含-排除之和会崩溃为零。 

| 步骤| 意义| 价值|
 | --- | --- | --- |
 | 所需配对 | n | 4 |
 | 前缀大小 | p − 1 | 2 |
 | 可行的覆盖范围| 不可能| 0 |

 该算法正确返回 0/1。

 ## 边缘情况

 n = 0 的情况是单独处理的，因为没有需要满足的约束。 唯一有效的结果是在 p = 0 时立即成功，否则概率为零，因为不存在“最终完成时刻”。 

p < n 的情况在结构上是不可能的，因为必须接触至少 n 个不同的对，并且每个前缀元素最多可以促成一个新对的完成。 该算法在进行任何组合工作之前正确地消除了这些情况。 

排除最后一对 k 至关重要。 如果我们错误地允许其元素位于前缀内，我们就会对最后一对过早完成的配置进行计数，从而违反了“恰好位于 p”条件。
