 ## 解决方案
 ## 问题理解

 我们得到一个文件列表，每个文件都有一个从 1 到 n 的唯一标签。 在任何时候，编辑器都会维护这些文件的顺序，但该顺序被分为两个连续的部分。 第一部分包含按当前相对顺序排列的所有固定文件，第二部分包含按当前相对顺序排列的所有未固定文件。 

移动包括切换单个文件的固定状态。 这不仅仅是标志翻转，因为文件从当前位置物理删除，并根据其新状态立即重新插入到不同的区域。 如果文件被固定，则会将其插入到固定段的末尾。 如果它变为取消固定，则会将其插入到取消固定段的开头。 除了这些插入和删除之外，固定块和未固定块的内部相对顺序将被保留。 

我们获得了初始配置和目标配置，两者都描述了完整的排序以及固定的前缀。 任务是计算将初始状态转换为目标状态所需的最小切换次数。 

关键约束是 n 最多为 100，因此二次甚至三次推理都是可以接受的。 这强烈表明我们应该避免对所有 2^n 子集进行指数状态搜索并结合排列。 

当固定或未固定块内的相对顺序发生变化时，会出现微妙的边缘情况。 由于切换可以跨边界移动文件并更改其在段内的位置，因此位置的天真贪婪匹配会失败。 

例如，如果所有文件在两种状态下都已取消固定，但排列不同，则忽略段结构的简单方法会说需要零移动，这是不正确的，因为段内的重新排序仍然需要切换。 

另一种失败情况是文件仅更改其在固定部分和未固定部分之间的位置。 即使其在整个数组中的相对顺序看起来相似，边界结构也会强制移动。 

## 方法

 强力方法会将每个状态视为由排列和二进制固定掩码组成的一对，然后对所有状态执行 BFS，其中每个转换都会切换一个文件。 每次移动都是 O(n) 来重建结构，并且有 n！ 原则上，排列乘以 2^n 掩码，尽管从初始状态只能到达一小部分子集。 即使限制可达状态，BFS 仍然可以按 n * n 的顺序进行探索！ 在最坏的情况下进行过渡，这是完全不可行的。 

关键的观察结果是，固定和未固定段内部的内部顺序从来都不是任意的。 每个状态完全由 n 个元素的排列决定，切换文件相当于删除它并在仅取决于其新状态的边界位置重新插入它。 这种结构意味着，重要的是在考虑划分为固定组和非固定组所引起的约束时，有多少元素相对于目标“错位”。 

一个更有用的视角是考虑元素对之间的排序约束。 每个切换最多可以修复一个元素相对于边界和相对排序约束的定位方式的结构不一致。 这将问题简化为计算在保留内部段顺序的受限移动下将一种排列转换为另一种排列所需的最少操作数。

关键的简化是要认识到最终的答案仅取决于有多少元素尚未处于与其目标位置和目标段一致的配置中。 一旦我们对齐已经满足相对排序约束的元素，就必须移动剩余元素，并且每次移动都有效地将一个元素正确地“放置”在组合结构中。 这导致减少计算有多少元素可以保持在相对于从目标状态导出的顺序一致的子序列中。 

这成为初始排序和目标排序之间的最长常见子序列问题，但有一个转折：我们必须尊重固定段和非固定段的划分。 一旦两个状态都被线性化，最小移动次数等于元素数量减去已经以正确的相对顺序出现的最长子序列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 状态上的暴力 BFS | O(n!·n) | O(n!·n) | O(n! ) | 太慢了|
 | 映射排列上的 LCS | O(n²) | O(n) | 已接受 |

 ## 算法演练

 我们将初始配置和目标配置转换为完整序列，暂时忽略固定的分割。 固定结构仅在定义顺序时起作用，因为固定元素总是出现在未固定元素之前。 

然后我们进行如下操作。 

1. 将初始序列构建为长度为 n 的数组 A，与给定的完全相同。 
2. 将目标序列构建为长度为 n 的数组 B。 
3. 构造一个位置映射 pos，使得 pos[value] 给出该值在 B 中的索引。 
4. 将 A 转换为数组 C，其中每个元素为 pos[A[i]]。 这将问题转换为通过目标中的索引顺序比较两个排列。 
5. 计算 C 的最长递增子序列的长度。这对应于相对于目标配置已经处于正确相对顺序的元素的最大子集。 
6. 答案是n减去LIS长度。 

步骤 4 背后的原因是，如果两个元素在两种配置中都以正确的相对顺序出现，则它们在目标排序中的索引将在转换后的数组中增加。 因此，保持顺序减少为寻找递增子序列。 

### 为什么它有效

 每个有效的移动序列只能通过边界插入重新定位元素，这保留了未触及元素的相对顺序。 因此，我们能做的最好的事情就是避免移动目标中已经以正确相对顺序出现的元素子集。 该子集正是目标索引映射下的最长递增子序列。 每隔一个元素必须至少移动一次，并且每次移动都可以正确放置一个元素而不扰乱 LIS 结构，因此最小移动次数等于 n 减去 LIS。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def lis_length(arr):
    import bisect
    d = []
    for x in arr:
        i = bisect.bisect_left(d, x)
        if i == len(d):
            d.append(x)
        else:
            d[i] = x
    return len(d)

def solve():
    p, u = map(int, input().split())
    A = list(map(int, input().split()))
    input()  # skip p u line format redundancy if present
    B = list(map(int, input().split()))
    input()

    pos = {}
    for i, v in enumerate(B):
        pos[v] = i

    C = [pos[x] for x in A]
    print(len(A) - lis_length(C))

if __name__ == "__main__":
    solve()
```该实现首先读取两个配置并将它们展平为完整数组。 映射步骤至关重要，因为它将结构排序问题转换为纯粹的排列比较问题。 

LIS 计算使用标准的耐心排序方法和二分搜索，确保 O(n log n) 时间，尽管 n ≤ 100 甚至 O(n²) 就足够了。 n 的减法直接给出必须通过切换重新定位的元素数量。 

一个微妙的点是我们必须将整个列表视为一个序列； 固定边界已经编码在排序本身中，因此展平后不需要特殊处理。 

## 工作示例

 ### 示例 1

 考虑初始 A = [2, 1, 4, 3] 和目标 B = [4, 2, 3, 1]。 

我们映射 B 索引：4→0、2→1、3→2、1→3，因此 C 变为 [1, 3, 0, 2]。 

| 步骤| C | LIS流程| 到目前为止，LIS |
 | --- | --- | --- | --- |
 | 开始 | [1,3,0,2] | 从空开始| 0 |
 | 1 | [1] | 附录 1 | 1 |
 | 2 | [1,3]| 附录 3 | 2 |
 | 3 | [1,3,0]| 将 1 替换为 0 | 2 |
 | 4 | [1,3,0,2] | 将 3 替换为 2 | 2 |

 LIS 长度为 2，因此答案为 4 − 2 = 2。 

这表明只有两个元素可以保持正确的相对顺序； 其余的每个都需要至少一个切换才能达到与目标一致的配置。 

### 示例 2

 初始 A = [1, 2]，目标 B = [1, 2]。 

映射给出 C = [0, 1]。 LIS 为 2，所以答案为 0。 

| 步骤| C | LIS流程| 到目前为止，LIS |
 | --- | --- | --- | --- |
 | 开始 | [0,1]| 追加 0 | 1 |
 | 1 | [0,1]| 附录 1 | 2 |

 没有元素需要重新定位，因为结构已经完全匹配。 

这证实了当排列已经匹配时，不需要切换。

 ## 边缘情况

 关键的边缘情况是仅固定边界发生变化但相对顺序保持不变。 例如，如果初始被固定[1,2]未固定[3,4]并且目标被固定[1,2,3]未固定[4]，则扁平化序列上的LIS正确地给出了一个大的保留子序列，并且只有一个元素需要重新定位。 

另一个边缘情况是完全逆转。 如果 A 为 [1,2,3,4]，B 为 [4,3,2,1]，则映射序列变为严格递减，因此 LIS 为 1。算法正确输出 3，反映出由于排序约束，除一个元素外的所有元素都必须至少移动一次。 

最后一个微妙的情况是固定集相同但内部顺序不同。 LIS 公式正确地忽略了固定标签，并捕获了段内的重新排序仍然需要相当于破坏目标索引空间中的递增顺序的移动。
