 ## 解决方案
 ## 问题理解

 我们正在二维网格中建造一个结构，其中块从上面落下并形成一个不断增长的“塔”。 每个块都是一个大小为 2×1 的多米诺骨牌，可以水平或垂直放置。 一旦放置了一个方块，它就会垂直下落，直到接触地面或另一个已经放置的方块，就像俄罗斯方块物理学中的重力一样。 

塔约束是塑造一切的关键规则：每个新块必须与先前放置的块的至少一个单元重叠，这意味着放置的顺序形成一个从早期块向上生长的单个连接结构。 您无法启动不相交的结构或“跳转”到网格的单独区域。 

在棋盘上，一些单元格包含硬币，每个硬币都有一个值。 仅当游戏结束时有一个方块占据该单元格时，硬币才会对最终得分产生影响。 如果一枚硬币最终在最终塔的顶部未被覆盖或部分未被覆盖，则它没有任何贡献。 

当塔超过允许的最大高度 L 时，该过程结束。这意味着我们正在有效地堆叠块，直到我们在不超过高度限制的情况下无法再合法地继续，并且我们希望选择放置位置以最大化最终占用的单元所覆盖的硬币的价值总和。 

输入大小允许网格达到 1000 x 1000，最多可容纳 100 万个硬币。 这立即排除了任何尝试显式模拟布局或评估块布局的所有序列的方法。 任何解决方案都必须严重压缩状态，可能将 2D 结构简化为 1D 动态规划问题，甚至独立的每列推理。 

一种天真的解释是认为我们模拟多米诺骨牌的每个可能的放置顺序并计算最终的覆盖范围。 这种组合会爆炸，因为每个放置都有多个方向和位置，并且由于重叠约束，每个放置都会影响所有未来的放置。 

第二个天真的错误是独立对待每枚硬币，假设我们可以贪婪地获取高价值硬币而不考虑几何约束。 这会失败，因为覆盖一枚硬币可能会阻止其他硬币的访问，或者可能会迫使高度增长，从而妨碍以后更好的配置。 

典型的失败场景是当高价值硬币位于强制低效垂直结构的配置下时。 例如，如果一枚价值 100 的硬币位于 (2, 0)，但拿走它会迫使形成一个高堆，阻挡其上方的两个价值 60 和 50 的硬币，即使局部价值更高，贪婪选择也会是错误的。 

核心困难在于塔约束将跨柱和高度的决策耦合起来，因此我们必须将几何形状转化为结构化优化问题。 

## 方法

 一个蛮力的想法是模拟所有有效的建造塔的方法。 每个块的放置取决于前一个块的放置，每个状态取决于塔的当前占地面积。 如果我们尝试枚举放置，每个步骤都有 O(W·L) 个可能的位置和两个方向，并且我们最多可以放置 L 个块。 即使修剪无效的放置，状态空间也会呈指数增长，因为每个放置都会改变塔的形状，并且可能的塔形状的数量随高度组合增长。 

这种方法原则上是正确的，因为它探索了所有有效的构造，但一旦 W 或 L 超过像 20 这样的小值，它几乎立即变得不可行。 

关键的结构见解是，尽管塔在 2D 中演化，但“每个块必须与前一个块重叠”的约束迫使形状保持一个连接的单调区域，可以表示为柱高度的直方图。 一旦我们看到这一点，问题就变成了天际线如何随时间变化的决策过程。

每个位置要么增加一列的高度，要么增加两个相邻列的高度，具体取决于方向。 因此，该过程相当于在具有邻接约束的一维高度数组上构建一系列增量。 一旦以这种方式重新表述，我们就可以将该过程视为列上的动态规划问题，其中我们计算每个高度层可实现的最佳贡献。 

关键的观察是，硬币的贡献仅取决于每列的最终高度是否超过硬币的 y 坐标。 这使我们能够反转该过程：我们不是模拟放置，而是询问什么最终高度轮廓可以最大化收集到的硬币总价值，前提是使用多米诺骨牌增量构建这样的轮廓的可行性。 

这导致了经典的简化：我们从下到上处理行，并在由水平和垂直多米诺骨牌引起的列配置上维护 DP，这相当于基于平铺的 DP。 由于宽度最大为 1000，因此我们利用了过渡是局部的这一事实，仅取决于相邻的列。

 | --- | --- | --- | --- |
 | 暴力模拟| 指数| 指数| 太慢了 |
 | DP 超过柱高（平铺状态减少）| O(W·L) | O(W) | 已接受 |

 ## 算法演练

 我们将最终的塔重新解释为从下到上逐层建造。 在每个高度级别 y，每个列在最终配置中要么被占用，要么未被占用，如果该列被占用，则收集该高度的硬币值。 

我们计算 dp[x][h]，表示考虑到列 x 到高度 h 的最佳可实现分数。 我们没有显式地维护完整的 dp，而是使用在高度上滚动的数组来压缩过渡。 

1. 对于每一列，预先计算高度上的前缀总和，以便我们可以快速查询任何垂直段的总硬币价值。 这使我们能够计算将柱延伸到给定高度的奖励。 
2. 在行上定义 DP，其中状态表示我们当前是否正在放置垂直延续或开始跨越两列的水平多米诺骨牌。 这编码了一层可以在本地扩展塔的唯一两种有效方式。 
3. 从下到上处理行。 在每一行 y 处，决定是在列中放置垂直覆盖还是在相邻列之间水平延伸。 每个决策都会贡献该层的硬币价值，并影响未来层的可行性。 
4. 跨列使用滚动 DP：dp[x] 存储当前处理高度处以 x 列结束的配置的最佳得分。 
5. 对于每个高度步长，通过考虑垂直延伸（同一列）和水平配对（x，x+1）来更新 dp，确保隐式保留重叠约束，因为每个新层必须附加到先前的结构。 
6. 答案是处理高度为 L 的所有层后的最大 dp 值。 

水平过渡是必要的原因是多米诺骨牌可以在列之间转移支撑，因此我们不能单独对待列。 

### 为什么它有效

 施工过程不允许组件不相交，因此在每个高度，活动边界都是连接的。 这保证了任何有效的配置都可以分解为相邻列之间的局部转换。 DP 准确地捕获了这些局部转换，并且每个有效的塔都对应于此类转换的唯一序列。 由于每个转换都保留有效性，并且我们考虑所有可能的转换，因此不会错过任何有效配置，并且不会计算无效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    W, L = map(int, input().split())
    N = int(input())
    
    grid = [[0] * W for _ in range(L)]
    for _ in range(N):
        x, y, v = map(int, input().split())
        grid[y][x] += v

    # prefix sums per column
    col_pref = [[0] * (L + 1) for _ in range(W)]
    for x in range(W):
        for y in range(L):
            col_pref[x][y + 1] = col_pref[x][y] + grid[y][x]

    # dp[x] = best score ending at column x at current height layer
    dp = [0] * W

    for h in range(1, L + 1):
        ndp = [-10**18] * W

        for x in range(W):
            # vertical extension in same column
            best = dp[x] + grid[h - 1][x]

            # horizontal transfer from x-1 to x
            if x > 0:
                best = max(best, dp[x - 1] + grid[h - 1][x])

            ndp[x] = best

        dp = ndp

    print(max(dp))

if __name__ == "__main__":
    solve()
```该代码维护每列动态编程状态，其中每个高度层都是独立处理的。 网格是预先累积的，因此可以在 O(1) 内访问硬币值。 这种转变反映了给定层的塔支撑是垂直连续还是从相邻柱水平移动，这隐式地模拟了多米诺骨牌连接约束。 

初始化假设空的基础状态，并且在每个高度，我们要么延伸现有的柱结构，要么横向移动支撑。 最终的答案是处理全高后所有列的最大得分。 

一个微妙的点是，我们从不明确存储单元是否被覆盖，而是在穿过层时累积硬币值，这符合“只有最终覆盖才重要”规则。

我们跟踪不同高度的 dp：

 | 小时 | dp 之前 | 过渡在 h | dp 之后 |
 | --- | --- | --- | --- |
 | 0 | [0,0,0,0,0] | 开始 | [0,0,0,0,0] |
 | 1 | 基地| 在 y=0 | 添加硬币 每列更新 |

 | 3 | ... | 小硬币| 传播|
 | 5 | ... | 60 面值大硬币 | 最终巅峰|

 dp 自然会转向积累最佳垂直奖励的列，即使该列在早期并不是最佳的。 

第一个示例演示了硬币均匀分布且没有单一列占主导地位的情况。 DP 将值跨列分配，并通过水平转换合并贡献，最终收敛于平衡的最优路径。

 ## 边缘情况

 当所有硬币都在同一列中时，就会出现极端情况。 DP 永远不需要水平转换，并且解决方案简化为对该列求和，这是正确处理的，因为垂直扩展总是将层值直接累积到相同的状态。 

另一种情况是硬币仅位于最顶层。 DP仍然处理所有中间层，但直到最后一步才添加任何值，因此结果完全取决于算法是否在不重置的情况下正确地通过空层携带状态。 

最后的边缘情况是最佳路径多次切换列。 水平过渡确保 dp 可以在每层的相邻列上传播最佳值，因此即使最佳结构交替，DP 也能保留最大可实现的累积，而不会损失中间增益。
