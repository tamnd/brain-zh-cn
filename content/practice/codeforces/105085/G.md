 ## 解决方案
 ## 问题理解

 我们正在使用一个只有两行和大量列的网格。 每个单元格都从零开始，我们可以执行非常具体的本地操作。 操作选择一行中的一个单元格并将其减一，同时增加另一行中的两个对角相邻的单元格。 由于几何形状的原因，选择第一行中的一个单元格会影响第二行中的两个相邻单元格，反之亦然，总是左右移动一列。 

我们的目标不是达到某种任意的配置，而是达到一种完全统一的配置。 网格中的每个单元格最终都必须等于相同的整数值 V。对于每个测试用例，我们会被问到两个问题：这是否可能，如果可能，实现它所需的最少操作数。 

限制条件非常大。 列数 M 最多可以达到 10^9，并且最多可以有 2×10^5 个测试用例。 这立即消除了任何模拟网格甚至存储每列状态的方法。 任何解决方案都必须将整个网格压缩为少量的派生量。 

第一个微妙之处是每个操作都会将所有单元格的总和更改为 +1。 一个单元格损失 1，两个单元格各增加 1，因此净变化为 +1。 这意味着最终的总和由操作数决定，与结构无关。 由于目标配置是所有单元都等于 V，因此最终总和必须为 2MV，因此操作数必须恰好为 2MV。 

这已经给出了必要条件，但还不够，因为操作还必须可以跨列分布，而不破坏边界约束。 

第二个结构性约束来自端点。 第 1 列和第 M 列的行为不同，因为它们各自缺少一个对角邻居，因此无法在此处应用某些操作。 这导致边界处的不平衡约束取决于 M 模小值。 

当 M 很小时，就会出现边缘情况。 对于 M = 1，根本没有有效的操作，因为每个操作都需要相对行中的两个对角邻居。 所以唯一可行的配置是全零网格。 这意味着V必须为零，否则不可能。 忽略这种退化的幼稚实现会错误地声称可行性。 

对于M = 2，由于同样的原因仍然无法执行操作：每个位置都缺少至少一对所需的对角线。 同样，只有 V = 0 是可能的。 

第一个有趣的行为从 M ≥ 3 开始，其中操作可以在内部传播质量。 

## 方法

 蛮力视角将尝试显式地对网格状态进行建模并模拟操作，试图达到目标配置。 每个操作影响三个单元，并且交互跨列传播，因此状态空间是2M整数。 即使对于小 M，这在结构上也是指数的，因为操作可以应用于导致相同中间状态的许多序列。 这使得直接搜索变得不可行。 

一种更受控制的暴力思想是将每个操作视为在 2×M 维空间中贡献一个固定向量。 问题变成询问目标向量（全V）是否可以表示为这些操作向量的非负整数组合。 这是一个大整数线性可行性问题。 像高斯消元法这样的标准方法并不直接适用，因为系数必须是非负整数，而不是实数。

关键的简化来自于认识到网格是一个具有半径为 1 的局部交互的一维链。每个操作仅涉及交替行中长度为 3 的段。 这意味着系统在内部是平移不变的，因此只有边界效应很重要。 一旦 M 足够大，内部行为就会一致，并且约束会分解为由守恒定律和边界不平衡导出的少量方程。 

通过写入每个列对的守恒并对交替差求和，系统简化为仅依赖于 M 模 3 以及 V 与操作次数之间的线性关系来检查可行性条件。 这导致了封闭式解决方案：要么由于奇偶校验/边界不匹配而不可能，要么通过固定操作计数唯一确定。

 ## 算法演练

 该解决方案依赖于两个独立的约束：总质量守恒和边界可行性。 

1. 计算cell总数，即2M，并推导出所需的最终总和，即2MV。 由于每次操作都会使总和恰好增加 1，因此操作次数必须恰好为 2MV。 如果配置完全可以实现，这将修复答案。 
2. 检查对于给定的 M 是否可以进行任何操作。当 M 小于 3 时，不存在有效的操作，因为每个操作都要求两个对角邻居同时存在。 这使得系统冻结在初始状态，因此唯一可达到的目标是 V = 0。 
3. 对于M ≥ 3，验证边界一致性。 每个操作都会跨列移动质量，但会在边缘附近引入方向偏差。 最左边和最右边的列接收不对称的贡献，整个网格的净不平衡取决于 M 的奇偶性减去 2 个内部自由度。 这可以分解为一个简单的条件：该配置对于所有 V 都是可行的，因为一旦 M ≥ 3，内部传播就可以生成均匀分布。 
4. 如果可行，输出强制操作次数2MV。 否则输出-1。 

它为何起作用与守恒定律有关。 每个操作都保留相邻列之间差异的线性结构（边界除外）。 当 M ≥ 3 时，这些边界效应会被抵消，因为至少存在一个允许重新分配的内部缓冲列。 从任何均匀目标向量都位于由运算向量生成的半群的意义上来说，系统变得完全可控。 对于 M ≤ 2，不存在这样的缓冲区，因此半群仅折叠到零向量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        M, V = map(int, input().split())

        if M < 3:
            if V == 0:
                print(0)
            else:
                print(-1)
            continue

        print(2 * M * V)

if __name__ == "__main__":
    solve()
```该实现遵循小型网格和大型网格之间的结构二分法。 乘法 2 * M * V 是使用 Python 整数计算的，因此溢出不是问题。 

关键的决定是 M < 3 处的截止点。这是操作不再存在的地方，很容易错过，因为问题陈述仍然正式描述操作，但边界要求使其对于小宽度无效。

def run(inp: str) -> str:

 ## 边缘情况

 对于 M = 1 和 M = 2，算法正确地拒绝所有正 V。例如，对于输入 (M=2，V=4)，算法直接返回 -1，因为条件 M < 3 触发。 这与无法应用有效操作的事实相匹配，因此网格不能从全零开始变化。 

对于 M = 3，请考虑 (M=3，V=1)。 算法输出 6。这对应于 2 × 3 × 1，符合守恒规则。 由于操作存在于这个宽度内，所以系统是完全可控的，并且没有边界障碍阻止统一分配。 

对于较大的 M，例如 M = 10^9 和 V = 10^9，该算法仍然执行单个乘法并输出 2×10^18，这非常适合 Python 整数，并确认该解决方案不依赖于最终公式中超出线性缩放的网格大小。
