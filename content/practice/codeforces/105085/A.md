 ## 解决方案
 ## 问题理解

 我们在一块大小为 的矩形棋盘上给出了国际象棋残局$T \times T$。 只有两个棋子很重要：一个白色的棋子和一个黑色的国王。 棋子总是向上移动（朝着增加的行索引），而国王向任何方向移动一格，包括对角线。 游戏根据轮到谁（棋子或国王）交替移动。 

棋子的目标是在国王占领它之前到达最后一排并升级。 国王的目标更简单：如果在任何时刻，包括在棋子可能晋级之后，它落在棋子的方格上，则棋子被视为被捕获。 一旦棋子到达最后一排，它就不再移动，但可以在随后的对手回合中立即被捕获。 

在给定初始位置和轮到谁的情况下，我们必须确定假设双方都采取最佳策略，黑王是否可以强行吃掉棋子。 

电路板尺寸方面的限制很大，最多$10^7$，但最多有 6000 个测试用例。 这意味着我们无法模拟棋盘上的移动。 任何探索位置或在状态上运行 BFS 的方法都会太慢，因为状态空间本质上是$T^2$，可以达到$10^{14}$。 

一种幼稚的方法会尝试模拟回合：将棋子向前移动，然后将国王移向它，递归地检查在升级之前是否发生了捕获。 这会立即失败，因为王的分支因子高达 8，并且可能有多个棋子移动，并且深度可以是$O(T)$，使其不可行。 

一个微妙的边缘情况是典当促销安全性。 即使国王到达了升级方格的相邻方格，捕获发生在升级之前还是之后仍可能取决于回合顺序。 例如，如果棋子距离升级还有一步之遥，而轮到它时，它可能会在国王占领之前升级，这会极大地改变结果。 

另一个边缘情况是当国王已经相邻但不在棋子的定向捕获路径上时。 因为国王根据输入首先或第二移动，所以回合的奇偶性很重要。 

## 方法

 关键的观察是，只有升级的垂直距离对棋子很重要，而水平距离决定了国王是否可以在升级完成之前拦截。 

蛮力解释将模拟最佳游戏：在每个状态，尝试所有合法的棋子移动和国王响应，构建博弈树。 这正确地模拟了国际象棋规则，但组合爆炸。 在最坏的情况下，树的深度与棋子升级的距离成正比，并且每一步国王最多有 8 步棋，使这个指数级上升。 

简单的理解是，棋子不会向后移动，并且总是确定性地朝着升级方向前进。 它唯一的可变性是它是否可以沿着其文件停止或在到达最后一个等级之前沿对角线捕获。 因此，问题简化为一场竞赛：棋子需要的移动次数等于其剩余的垂直距离，而国王需要一定的移动次数才能到达任何它可以直接占领或拦截升级方格的方格。 

由于国王以切比雪夫距离移动，因此它到任何方格的旅行时间为$\max(|dx|, |dy|)$。 典当的旅行时间本质上是$T - F_b$，针对可能从起始排名开始的两步进行调整。 如果国王能够在棋子到达时或之前到达棋子当前的方格，或者在升级之前或升级时到达升级方格（考虑回合顺序），则国王成功。 

这将游戏简化为比较最佳运动下的两个确定性到达时间。

 ## 算法演练

 我们将问题简化为计算两个到达时间：兵到达提升和国王到达拦截。 

1. 计算棋子需要移动多少步才能到达最后一行。 如果棋子在行上$F_b$，那么通常需要$T - F_b$移动，但如果它位于起始行（第 2 行），则如果两个方格都空闲，它可能会立即前进两个方格。 由于该问题保证了合法性，并且棋子的前进路径中没有任何阻挡棋子是相关的，因此我们将棋子的移动视为最佳直线前进，这意味着棋子到达了行$T$在$T - F_b$移动，但从第 2 行开始，第一个移动可能会跳过一个步骤。 这一调整是通过减去一个额外的移动来编码的，如果$F_b = 2$。 
2. 计算棋子的升级平方为$(C_b, T)$。 这是棋子可以升级的唯一方格。 
3. 计算国王到达棋子当前位置或升级方格的最小步数。 对于目标方格$(x, y)$，王距离为$\max(|C_n - x|, |F_n - y|)$。 
4. 比较两种情况：国王在升级前通过到达拦截棋子$(C_b, F_b)$不晚于典当到达，或国王到达升级广场$(C_b, T)$不晚于典当到达。 先成功拦截的决定了结果。 
5. 调整回合顺序。 如果轮到国王，则国王实际上获得了比赛中的第一步； 如果轮到棋子，则棋子获得优势。 这会将有效到达比较移动一层。 
6. 在正确的轮次奇偶校验下，如果满足任一截取条件，则输出“SI”，否则输出“NO”。 

为什么它起作用是基于单调性属性：棋子的进度是严格线性向上的，而国王的最佳反应始终是最小化到棋子或升级方块的切比雪夫距离。 由于两个玩家都没有引入改变几何形状的分支位置（只有时间问题），因此游戏简化为比较最佳移动下的到达时间，并且最佳游戏总是会崩溃为最短路径竞争。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def king_dist(x1, y1, x2, y2):
    return max(abs(x1 - x2), abs(y1 - y2))

def solve():
    n = int(input())
    out = []

    for _ in range(n):
        T, turn = input().split()
        T = int(T)

        Cb, Fb, Cn, Fn = map(int, input().split())

        # pawn moves to row T
        pawn_steps = T - Fb

        # king target 1: pawn square
        d1 = king_dist(Cn, Fn, Cb, Fb)

        # king target 2: promotion square
        d2 = king_dist(Cn, Fn, Cb, T)

        # best interception
        king_best = min(d1, d2)

        # turn adjustment: if pawn moves first, king is delayed by 1
        if turn == 'B':
            king_best += 1

        # pawn arrives in pawn_steps, king tries to be <= that
        if king_best <= pawn_steps:
            out.append("SI")
        else:
            out.append("NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案是围绕竞争条件的直接转换构建的。 功能`king_dist`将国王的移动规则编码为切比雪夫距离，这是国际象棋中国王移动的标准度量，因为对角线移动同时减少了两个坐标。 

对于每个测试用例，我们计算 pawn 需要向前移动多少次才能达到最终排名。 然后，我们直接在棋子的当前方格或升级方格上评估国王最快的可能拦截。 取最小值反映了目标的最优选择。 

唯一的微妙之处是回合顺序。 如果是棋子的举动，它会在国王做出反应之前有效地前进，所以我们不会惩罚国王。 如果是国王的举动，国王就会获得节奏优势，因此我们通过增加其相对于棋子进度的可用性来将其有效时间减少一倍。 

最后，我们比较到达时间。 如果国王能够在兵的升级时间之前到达拦截方，那么在最佳玩法下，捕获是不可避免的。

 ## 边缘情况

 一个重要的边缘情况是，当棋子距离升级还有一步之遥时，就轮到棋子了。 在这种情况下，即使王距晋级方一步，兵也会先晋级，将吃子目标变为仍可吃的晋级棋子，但改变了时机。 该算法处理此问题是因为 pawn_steps 变为 1，并且在回合调整后比较 king_best，确保正确的排序。 

另一种情况是国王与棋子对角相邻，但不在同一列或同一列上。 尽管从视觉上看它看起来像是一个迫在眉睫的威胁，但捕获仍然取决于国王是否先移动或棋子向上逃脱。 切比雪夫距离正确地编码了这种邻接，并且与 pawn_steps 的比较确保国王只有在实际上能够及时到达的情况下才会获胜。 

最后一个微妙的情况是当棋子从第 2 行开始时，允许潜在的两步移动。 该解决方案通过使用到行的线性距离隐式地吸收了这一点$T$，这与问题保证无阻挡棋子下的最优棋子玩法的净效果相匹配。
