 ## 解决方案
 ## 问题理解

 我们给出了几个独立的场景。 每一个都有一个客户服务时间列表，任务是将这些客户分成两个结账队列。 每个顾客必须恰好进入一个队列，而队列的处理时间只是分配给它的顾客时间的总和。 

目标是尽可能平衡两个队列。 更准确地说，在所有可能的分配中，我们希望两个队列总数尽可能接近。 找到这样的分割后，我们输出第一个队列和第二个队列的总时间，约定如果它们不同，则第二个打印较大的总时间。 

问题的结构是一个经典的划分任务：我们将正整数的多重集划分为两组，以使它们和的绝对差最小化。 

约束是这里的关键信号。 每个单独的时间最多为 8000，但更重要的是所有测试用例的所有时间总和最多为 8000。这意味着虽然客户数量可能很大，但我们需要推理的总权重很小。 任何依赖于总和作为状态空间的解决方案都是立即可行的。 

一种简单的方法是尝试将 N 个客户的所有分配分配到两个队列中，这对应于 2^N 种可能性。 即使对于中等的 N，这也是不可能的。 对于 N = 3000，这是完全不可能的。 

第二个天真的想法是贪婪分配，总是将下一个客户放入当前较轻的队列中。 这会失败，因为早期的本地平衡决策可能会阻碍更好的全局分区。 例如，对于时间 [8, 7, 6]，贪婪给出 (8+6, 7) = (14, 7)，差值为 7，而最优值为 (8+7, 6) = (15, 6)，差值 9 在这种情况下更差，但其他小设计的输入（如 [6, 5, 5]）会在相反方向打破贪婪，这表明局部平衡不可靠。 

真正的结构是，这是一个总和较小的子集和划分问题。 

## 方法

 暴力破解的观点是将每个客户独立分配到队列 1 或队列 2，并计算两个总和。 这将探索所有 2^N 分区。 即使每次评估都是 O(N)，总工作量也会呈指数级增长，并且几乎立即变得不可行。 

关键的观察是只有一个队列的总和才重要。 如果我们确定第一个队列的总和，则第二个队列的总和将固定为总和减去该总和。 因此，我们不需要选择分配，只需要知道哪些子集和是可以实现的。 在所有可实现的金额中，我们想要最接近总金额一半的金额。 

这将问题简化为经典的背包可达性任务：计算直到 S 的所有可能子集总和，其中 S 是所有时间的总和。 然后选取不超过S/2的最大可达值。 该值定义了最佳分割。 

由于所有测试用例中的 S 最多为 8000，因此动态编程或位集方法足够快。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了 |
 | 子集 DP（背包）| O(N·S) | O(N·S) | O(S)| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。

1. 计算测试用例中所有客户时间的总和S。 这决定了分区问题的目标规模。 
2. 构建一个布尔数组 dp，其中 dp[x] 表示是否可以使用某些数字前缀获得恰好为 x 的子集和。 最初，dp[0] 为真，因为不选择任何内容都会产生和为零。 
3.对于每个客户时间t，从右到左更新dp数组。 对于从 S 到 t 的每个可能的和 x，如果 dp[x - t] 为真，则 dp[x] 为真。 这可确保每个项目最多使用一次。 
4.处理完所有客户后，从S/2向下扫描到0，找到使dp[s]为真的最大值s。 这是最接近总数一半的可实现总和。 
5.输出s和S-s。 通过构造，S - s 至少为 s，因此自然满足所需的排序规则。 

关键思想是 DP 维护每个前缀处的所有可实现的总和，因此最终它完全表征了所有可能的分区。 

### 为什么它有效

 在每一步中，dp 都会对可以使用已处理元素的子集形成的子集和集进行精确编码。 向后转换确保每个元素要么被包含一次，要么被完全排除，从而保持分区 0/1 性质的正确性。 由于表示了所有可能的子集，因此最佳 s 的最终选择直接对应于所有有效分配中的最佳划分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    c = int(input())
    for _ in range(c):
        arr = list(map(int, input().split()))
        n = arr[0]
        vals = arr[1:]
        
        S = sum(vals)
        dp = [False] * (S + 1)
        dp[0] = True
        
        for v in vals:
            for s in range(S, v - 1, -1):
                if dp[s - v]:
                    dp[s] = True
        
        target = S // 2
        best = 0
        for s in range(target, -1, -1):
            if dp[s]:
                best = s
                break
        
        a = best
        b = S - best
        if a > b:
            a, b = b, a
        print(a, b)

if __name__ == "__main__":
    solve()
```实施直接遵循DP制定。 转换中的反向循环是防止在单次迭代中多次重用同一元素的关键细节。 最终从 S/2 向下扫描保证所选择的子集尽可能接近总和的一半，这正是最小化两个队列之间的差异的原因。 

最终交换确保输出排序符合较大队列时间出现在第二位的要求。

 ## 边缘情况

 对于单个客户，DP 从仅和 0 和 0 加上该元素开始，因此最佳分割始终是 (0, T)。 该算法自然地将所有内容分配给一个队列，并且输出排序规则确保第二个打印较大的值。 

对于像 [2, 2, 2, 2] 这样的完美平衡输入，可达集恰好包含总和的一半，因此该算法毫无歧义地找到相等性并产生相同的队列时间。 

对于具有许多小的相同值的情况，DP 会密集扩展，但由于总和很小，因此仍然可以有效地跟踪每个可达值。
