 ## 解决方案
 ## 问题理解

 该任务描述了一个机器人在由不同类型的单元组成的二维网格上移动，其中一些单元可以通过，而另一些单元则被阻挡。 机器人始终有一个面向方向，其运动规则区分转弯和前进。 转动仅改变其方向，而向前移动则使其朝当前面向的方向前进一个单元格。 

关键的限制是向前移动受到限制，而旋转则不受限制。 这立即改变了问题的本质：机器人可以在没有成本的情况下任意多次重新定向，但进入相邻单元的每一步实际都会消耗有限的向前推进预算。 目标是确定从起始位置到达目标单元格所需的最小向前移动次数，或者报告这是不可能的。 

该语句隐含的网格结构使用字符来表示细胞类型。 一个符号标记机器人的起始位置，另一个标记目的地，其余符号要么是空的，要么是被阻挡的。 移动仅限于有效的空单元格，并且无论方向如何，机器人都无法穿过被阻挡的单元格。 

从复杂性的角度来看，这种类型的网格通常总共有大约 10^5 到几百万个单元。 该范围立即排除了任何重复重新计算每个状态的路径或对方向历史使用指数探索的解决方案。 显式跟踪方向并尝试所有转弯和移动序列的简单模拟将有效地每一步分支四种方式，从而导致路径呈指数增长。 

当解决方案错误地将方向视为状态的一部分并为转弯序列分配不同的成本时，就会出现微妙的失败情况。 例如，如果机器人开始面朝上，但最佳路径需要首先向右移动，则简单的方法可能会尝试首先“对齐”方向，并错误地过度计算步数或修剪有效路径。 由于转弯是免费的，因此任何收取旋转成本的方法都会高估真正的最小值。 

如果人们认为机器人必须继续向前移动，直到撞到墙壁才能转动，就会出现另一个常见的陷阱。 这种解释会错误地限制运动顺序。 实际上，机器人可以在每个单元格之间交替转动和步进，这意味着无论之前的方向如何，一次向前移动始终可以到达每个相邻的单元格。 

## 方法

 暴力解释将该问题视为由位置和方向组成的扩展状态空间上的最短路径。 在每个状态下，机器人都可以无成本地向左或向右旋转，并且如果下一个单元格有效，则可以向前移动。 在此状态图上运行标准 BFS 或 Dijkstra 是正确的。 然而，方向维度将状态数量乘以四，并且每个单元转换可以考虑多个冗余方向变化。 在最坏的情况下，这会成为不必要的开销，因为轮换不会增加成本。 

关键的观察结果是旋转是自由的，因此方向永远不会限制可达性。 在任何单元中，机器人在移动之前都可以选择四个方向中的任何一个。 这会将所有方向状态折叠到每个网格单元的单个节点中。 如果目标单元没有被阻挡，每次移动就相当于以单位成本步进到四个相邻单元中的任何一个。 该问题简化为网格图上的未加权最短路径。 

这将解决方案转换为仅对单元格进行标准广度优先搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 状态 BFS（单元，方向）| O(4nm) | O(4nm) | 正确但比需要的重 |
 | 网格 BFS（折叠状态）| O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

### 1.解析网格并定位关键位置

 扫描整个网格一次以识别起始单元格和目标单元格。 存储它们的坐标以进行 BFS 初始化。 这一步确保我们从正确的起点开始探索，没有歧义。 

### 2. 将网格建模为单元图

 将每个开放单元视为一个节点。 如果两个节点在垂直或水平方向上相邻并且都没有被阻挡，则它们是连接的。 机器人的转向能力使我们能够完全忽略方向，因此不需要额外的状态。 

### 3. 从起始单元初始化 BFS

 将起始位置推入队列并将其标记为已访问。 该单元格的 BFS 距离为零，因为尚未使用前向移动。 

### 4.逐级扩展邻居

 在每一步中，使一个单元出队并尝试在四个基本方向上移动。 对于每个尚未访问过的有效邻居，为其分配一个等于当前距离加一的距离并将其入队。 这确保了我们第一次到达任何单元格时，都使用了最少的向前移动次数。 

### 5.到达目标后停止

 一旦目标单元出队或首次发现，就返回其记录的距离。 BFS 保证这是所需的最小前移次数。 

### 为什么它有效

 不变的是 BFS 按所使用的前向移动的非递减顺序处理单元。 由于每条边具有相同的成本，并且旋转不会影响成本，因此到单元格的任何路径都精确对应于一系列前向步骤。 一旦访问了某个单元，就没有其他路径可以通过更少的前向移动到达它，因为 BFS 首先探索所有较短的路径。 删除方向不会失去正确性，因为任何方向变化都可以在每次移动之前完成，而无需任何成本，这意味着每个邻接始终可用，独立于先前的状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    grid = [line.rstrip("\n") for line in sys.stdin if line.strip() != ""]
    if not grid:
        return

    n = len(grid)
    m = len(grid[0])

    start = None
    target = None

    for i in range(n):
        for j in range(m):
            if grid[i][j] == 'S':
                start = (i, j)
            elif grid[i][j] == 'T':
                target = (i, j)

    dist = [[-1] * m for _ in range(n)]
    q = deque()

    si, sj = start
    ti, tj = target

    dist[si][sj] = 0
    q.append((si, sj))

    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while q:
        x, y = q.popleft()

        if (x, y) == (ti, tj):
            print(dist[x][y])
            return

        for dx, dy in dirs:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < m:
                if grid[nx][ny] != '#' and dist[nx][ny] == -1:
                    dist[nx][ny] = dist[x][y] + 1
                    q.append((nx, ny))

    print(-1)

if __name__ == "__main__":
    solve()
```该实现首先提取网格并找到起始标记和目标标记。 然后，它使用队列运行标准 BFS，将距离存储在矩阵中。 方向列表对自由旋转后四种可能的向前移动进行编码。 访问数组可以防止重新访问单元格并保证线性复杂度。 

一个微妙的点是我们从不存储方向。 这是主要的简化：由于转动是自由的，机器人总是可以在任何移动之前自行对齐，因此在步骤之间不需要保持方向。

 ## 边缘情况

 当起点和目标相邻但除一个方向外均被墙壁隔开时，就会出现关键边缘情况。 BFS 仍然可以正确识别单个有效路线，因为它独立探索所有四个方向而不依赖方向。 

另一种情况是当网格有多条同样短的路线时。 由于 BFS 标记了第一次发现时访问过的单元，因此它永远不会用较长的路径覆盖较短的路径。 例如，如果两条路径在相同深度到达同一单元，则只有一条路径被排队，但正确性得以保留，因为所有路径具有相同的成本。 

最后一种情况是根本不存在路径。 BFS 耗尽了可到达的组件，并且目标仍然未被访问，正确地生成 -1 作为输出。
