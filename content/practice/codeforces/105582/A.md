 ## 解决方案
 ## 问题理解

 给定一个无向简单图，最多有 100 个顶点和 100 条边。 任务不是在计算意义上“绘制”任何几何图形，而是确定该图是否可以解释为称为苹果的非常具体的结构。 

一个苹果由两个顶点不相交的简单结构组成，它们恰好共享一个共同的起始顶点。 一种结构是简单循环，另一种结构是简单路径。 循环必须至少包含三个顶点，路径必须至少包含两个顶点。 除了共享起始顶点之外，循环和路径不得相互接触或重用顶点。 问题归结为决定我们是否可以在这些严格的结构约束下将图的边划分为一个简单的循环和一个具有公共根的简单路径。 

约束足够小，指数推理在理论上是可能的，但没有必要。 当 n、m ≤ 100 时，甚至 O(n3) 或 O(n·m) 图形分析也非常快。 这立即表明我们应该以结构分解为目标，而不是任何强力的几何排列或嵌入枚举。 

当图看起来“几乎”像一个带尾部的循环，但尾部重用循环中的顶点，或者循环太小时，就会出现微妙的边缘情况。 例如，如果我们有一个三角形 1-2-3-1 和一条路径 1-4，则这是有效的。 但如果路径也触及顶点 2，形成类似 1-2-4 的结构，则分解不再有效，因为路径和循环在根之外相交。 另一种故障模式是存在大小为 2 的循环，这在简单图中是不可能的，但在推理基于度的方法时作为概念陷阱仍然值得注意。 

## 方法

 一种强力解释会尝试猜测根顶点，然后决定哪些关联边属于循环，哪些属于路径，检查我们是否可以恰好形成满足所有约束的一个简单循环和一个简单路径。 这自然会导致枚举边的子集或执行与根相关的边的回溯分配。 即使 n ≤ 100，这也会成为组合：如果一个顶点的度数为 d，则将其入射边分成两组已经创建了 2^d 种可能性，并且在所有顶点上，这种情况会爆炸远远超出可行的限制。 

关键的观察结果是该结构极其刚性。 该图恰好由两个仅在一个顶点相交的连通分量组成，并且每个分量都有一种非常特定的形式。 其中一个组件是一个循环，这意味着其中的每个顶点在该子图中的度数都是 2。 另一个是路径，意味着该子图中恰好有两个顶点的度数为 1，而所有其他顶点的度数为 2，除了共享根（在路径组件中度数为 1 但全局共享）。 

这种刚性使我们能够将问题从“找到一张图”转变为“检查我们是否可以将图分解为两个非常具体的子图，并且共享一个铰接点”。 自然的方向是尝试将每个顶点作为潜在的共享根，并验证其事件结构是否可以分为一个循环分量和一个路径分量。 

环路条件成为检查是否存在完全包含在图中的环路，该环路可以“锚定”在根处，而根部的剩余边形成简单路径。 由于图很小，我们可以明确尝试基于 DFS 的循环检测和基于每个候选的路径验证。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力边缘分配 | 指数| O(n + m) | 太慢了 |
 | 使用 DFS 进行基于根的结构检查 | O(n·(n + m)) | O(n·(n + m)) | O(n + m) | 已接受 |

 ## 算法演练

1. 为图构建邻接表。 这允许在尝试验证结构约束时有效地遍历邻居。 
2. 迭代每个顶点，将其视为循环和路径的潜在共享根。 正确的配置必须恰好使用一个这样的顶点，因此尝试所有可能性是安全的。 
3. 对于一个固定的根，尝试识别一个不以任意方式经过该根的循环，但可以将其视为“循环分量”。 我们在图中执行 DFS，同时跟踪父边并检测形成循环的后边。 
4. 一旦找到循环，标记属于该循环的所有顶点。 这一点至关重要，因为循环必须内部一致：循环结构中的每个顶点必须恰好有两个邻居。 
5. 验证根是否恰好连接到循环中的两个顶点或以与共享顶点一致的方式参与。 如果不能干净地分离循环，则该根无效。 
6. 移除或忽略循环边缘并检查剩余结构。 与根相关的其余边必须形成简单路径。 这意味着从根开始，剩余边上的 DFS 应该在线性链结构中恰好访问所有剩余顶点一次，没有分支。 
7. 验证剩余的组件确实是一条路径：剩余子图中除端点之外的每个顶点都必须具有度 2，并且恰好有两个端点必须具有度 1。 
8. 如果任意根同时满足循环条件和路径条件，则返回“Yes”。 

如果没有根产生有效的分解，则返回“否”。 

### 为什么它有效

 正确性来自于目标物体的结构刚性。 任何有效的配置都必须有一个顶点同时属于这两个组件。 该顶点是分隔循环和路径的接合点。 删除它会将图分成两个具有固定度数约束的连接结构：一个必须是循环（内部度数均为 2），另一个必须是路径（两个端点，其余度数为 2）。 由于每个有效的苹果都恰好有一个这样的分解点，因此迭代所有顶点可以保证我们最终会测试正确的分解点（如果存在）。 DFS 检查强制执行局部约束，这些约束唯一地表征无向图中的循环和路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def find_cycle(n, adj, start):
    parent = [-1] * (n + 1)
    visited = [False] * (n + 1)
    stack = [(start, -1)]
    visited[start] = True

    while stack:
        u, p = stack.pop()
        for v in adj[u]:
            if v == p:
                continue
            if not visited[v]:
                visited[v] = True
                parent[v] = u
                stack.append((v, u))
            else:
                # found a cycle, reconstruct it
                cycle = set()
                x = u
                cycle.add(v)
                while x != v and x != -1:
                    cycle.add(x)
                    x = parent[x]
                return cycle
    return set()

def is_path_component(n, adj, root, banned):
    visited = [False] * (n + 1)

    def dfs(u):
        visited[u] = True
        for v in adj[u]:
            if v in banned:
                continue
            if not visited[v]:
                dfs(v)

    dfs(root)

    nodes = [i for i in range(1, n + 1) if visited[i] and i not in banned]
    if not nodes:
        return False

    deg = {}
    for u in nodes:
        deg[u] = 0
        for v in adj[u]:
            if v not in banned:
                deg[u] += 1

    endpoints = [u for u in nodes if deg[u] == 1]
    internal = [u for u in nodes if deg[u] == 2]

    return len(endpoints) == 2 and len(endpoints) + len(internal) == len(nodes)

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n + 1)]
    for _ in range(m):
        a, b = map(int, input().split())
        adj[a].append(b)
        adj[b].append(a)

    for root in range(1, n + 1):
        cycle = find_cycle(n, adj, root)
        if not cycle:
            continue
        if root not in cycle:
            continue
        if is_path_component(n, adj, root, cycle):
            return "Yes"

    return "No"

print(solve())
```该实现尝试将每个顶点作为共享连接点。 对于每个候选根，它运行基于 DFS 的循环检测，在发现后沿时重建一个循环。 该循环被视为循环组件。 剩下的检查验证在排除循环顶点后图的其余部分是否形成干净的路径。 关键的微妙之处在于，循环检测并不是试图枚举所有循环，而只是枚举一个可以作为所需组件的有效循环，这就足够了，因为任何有效的解决方案都必须包含至少一个通过正确根的这样的循环。 

路径验证显式计算受限图中的度数。 这避免了单独依赖遍历顺序，这在具有多个分支的图中是脆弱的。 

## 工作示例

 ### 示例 1

 输入：```
5 5
1 2
2 3
1 3
1 4
1 5
```我们尝试将每个节点作为根。 对于根 1，存在循环 1-2-3-1，因此循环 = {1,2,3}。 移除它会留下边缘 1-4 和 1-5。 

| 步骤| 循环 | 剩余图| 路径有效 |
 | --- | --- | --- | --- |
 | 根=1 | {1,2,3} | 1-4, 1-5 | 没有 |

 其余结构在节点 1 处分支，在残差图中的根处产生度数 2，这违反了路径结构。 因此根 1 失败，并且没有其他根产生有效的分解。 

输出：没有。 

### 示例 2

 输入：```
4 4
1 2
2 3
1 3
1 4
```对于根 1，周期为 {1,2,3}。 剩余优势为 1-4。 

| 步骤| 循环 | 剩余图| 路径有效 |
 | --- | --- | --- | --- |
 | 根=1 | {1,2,3} | 1-4 | 1-4 是的 |

 剩下的部分是一条单边，它是长度为 2 个顶点的有效路径。 循环的大小为 3，满足约束条件。 

输出：是的。

 ## 边缘情况

 关键的边缘情况是当存在循环但其余结构在根部分支时。 例如，一个三角形有两条来自同一顶点的额外边。 即使存在循环，残差图也不符合路径度数条件，因为根最终在剩余组件中的度数大于 1。 

另一个微妙的情况是存在多个循环时。 该算法依赖于检测每个根的一个周期，但并非所有周期都是有效的。 如果检测到的循环不包括正确的分解结构，则根将在稍后的路径验证步骤中失败，这确保不正确的循环选择不会产生误报。 

最后一种情况是图是连通的但不包含简单循环。 在这种情况下，循环检测对于所有根都返回空，并且算法正确地输出“否”，而无需尝试不必要的路径检查。
