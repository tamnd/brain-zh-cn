 ## 解决方案
 ## 问题理解

 我们得到了行星的有向完整图，其中每对有序的行星可能有也可能没有可用的超级跳跃。 从行星 i 到行星 j 的跳跃的存在由邻接矩阵中的字符编码，每个字符对应于少数跳跃类型之一。 每种跳跃类型均由四个参数定义，并且行程时间取决于您开始跳跃的时间。 

关键的复杂性是，当你到达一个行星时，你不会被迫立即跳跃。 在开始下一次跳转之前，您可以等待任意整数时间。 如果在时间 s 开始类型 i 的跳跃，则持续时间不是固定的，而是计算为 ((a s + b) mod c) + d，因此开始时间和模交互都会影响行程时间。 

任务是在初始时间 s 从行星 1 出发，并在尽可能短的时间内到达行星 n，允许在中间行星任意等待。 

n 高达 2000 和 m 高达 50 的约束立即表明 O(n^2) 图遍历是可以接受的，但是任何重复地重新计算每个边的昂贵转换而不小心的风险都会变得太慢。 由 ci 限制到 2000 的模运算的存在是主要的结构暗示，表明某些每边预计算或有界搜索是可能的。 

将边视为具有固定权重的朴素最短路径方法将会失败，因为等待会以非线性方式改变每个边的有效权重。 一个更微妙的问题是，最佳决策不是“立即采取边缘”或“等到某个启发式时间”，而是取决于对齐起始时间模 ci。 

一些失败的案例说明了其中的困难。 

如果您总是在到达后立即跳跃，您可能会错过一个严格更好的时间表，等待将开始时间转移到更有利的剩余类别。 相反，如果您只考虑等到 ci 的时间倍数，您可能会错过最佳解决方案，因为最佳比对可能发生在任意残基处。 

另一个微妙的情况是，当等待时间长于 ci 时，由于线性项 a s 似乎很有用，但实际上，超过一个完整周期的更长等待时间不会改善模数部分，只会增加总时间，因此固定边沿的任何最佳开始时间始终位于相对于到达时间的有界窗口内。 

## 方法

 直接建模会将每个状态视为一对（行星，当前时间），然后转换将包括等待和跳跃。 这会产生一个无限状态图。 即使我们将时间离散化到一定范围，可能的时间范围也太大了，因为到达时间并没有以有用的方式受到输入大小的限制。 

正确的观点是仅将图保留在行星上，但重新定义边缘松弛：当我们在时间 t 位于行星时，我们通过在获取边缘之前选择最佳等待时间来计算到达邻居的最佳可能到达时间。 

对于具有参数 a、b、c、d 和当前时间 t 的固定边缘类型，我们希望选择起始时间 s ≥ t 最小化 s + ((a s + b) mod c)。 最小化过程中可以忽略常数 d 和 t 的减法。 

关键的结构观察是模项仅取决于 s 对 c 的模，而线性项随 s 增长。 对于任何固定残基 r，最佳选择是最小的 s ≥ t，使得 s == r mod c。 任何具有相同余数的更大选择只会将成本增加 c 倍，而不改变模部分，因此严格来说更糟糕。 

这减少了每条边的优化，最多检查 c 个候选开始时间：每个残基类别的第一个有效时间。 

这将每次松弛变成对残基的有界扫描，这是可行的，因为 ci ≤ 2000 且 m ≤ 50。

然后，我们在行星上运行 Dijkstra，其中每次松弛都会计算通过每个传出边缘类型的最佳可实现到达时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 随时间变化的状态图 | O(无穷大) | O(无穷大) | 不可能|
 | 时间范围内的强力每边模拟 | O(n·m·T) | O(n·m·T) | O(nT) | 太慢了 |
 | 每边缘残留扫描 + Dijkstra | O(n^2 + n·m·c) | O(n^2 + n·m·c) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们维护一个距离数组，其中 dist[v] 是行星 v 的最早已知到达时间。我们使用优先级队列按照当前最佳到达时间的升序处理行星，与 Dijkstra 完全相同，但具有非标准边缘松弛。 

1. 将 dist[1] 初始化为起始时间 s，并将所有其他 dist 值初始化为无穷大。 将 (s, 1) 推入最小堆。 这表示在时间 s 处于行星 1，没有任何延迟。 
2. 从堆中弹出时间 t 最小的状态 (t, u)。 如果该值与 dist[u] 相比已经过时，请忽略它。 这确保我们只扩展最佳的已知到达人数。 
3.对于每个存在从u到v的跳跃的行星v，识别相应的跳跃类型及其参数a、b、c、d。 这给出了该边缘的时间相关成本函数。 
4. 计算从当前时间 t 开始从 u 到 v 的最佳可能到达时间。 对于从 0 到 c − 1 的每个残数 r，计算满足 s == r mod c 的最早时间 s ≥ t。 然后计算成本 = s + ((a s + b) mod c) + d。 保持所有残留物中的最小值。 此步骤是核心优化，因为它探索所有有意义的开始时间，而无需扫描无限时间。 
5. 如果计算的到达时间改善了 dist[v]，则更新它并将 (dist[v], v) 推入堆中。 
6. 继续，直到堆为空。 答案是 dist[n]，如果从未达到则为 -1。 

### 为什么它有效

 在时间 t 的任何行星 u 上，任何获取特定优势的有效策略都必须选择某个开始时间 s ≥ t。 在所有此类 s 中，该边的最佳值始终是具有给定余数模 c 的最小 s，因为将 s 增加 c 会使总时间恰好增加 c，同时保持模表达式不变。 因此，通过检查每个残留类别的一个代表来捕获每个最佳选择。 由于 Dijkstra 总是首先确定最小的已知到达时间，一旦处理了一个节点，后面的替代路径就无法通过它产生更好的到达时间，从而在这些修改的边权重下保留标准最短路径不变式。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

INF = 10**30

def solve():
    n, m, start = map(int, input().split())
    
    types = {}
    params = {}
    
    for _ in range(m):
        line = input().split()
        z = line[0]
        a, b, c, d = map(int, line[1:])
        types[z] = (a, b, c, d)
    
    adj = [[] for _ in range(n)]
    
    for i in range(n):
        row = input().strip()
        for j, ch in enumerate(row):
            if ch != '.':
                adj[i].append((j, types[ch]))
    
    dist = [INF] * n
    dist[0] = start
    
    pq = [(start, 0)]
    
    while pq:
        t, u = heapq.heappop(pq)
        if t != dist[u]:
            continue
        
        for v, (a, b, c, d) in adj[u]:
            best = INF
            
            base = t % c
            
            for r in range(c):
                if r >= base:
                    s = t + (r - base)
                else:
                    s = t + (c - (base - r))
                
                val = s + ((a * s + b) % c) + d
                if val < best:
                    best = val
            
            if best < dist[v]:
                dist[v] = best
                heapq.heappush(pq, (best, v))
    
    print(-1 if dist[n - 1] == INF else dist[n - 1])

if __name__ == "__main__":
    solve()
```邻接列表存储每个行星的传出边缘及其参数集。 Dijkstra 循环是标准的，但松弛步骤用计算出的残基对齐开始时间的最小值代替固定权重。 

一个常见的陷阱是尝试独立于当前时间来预先计算转换成本。 但这会失败，因为最佳开始时间取决于到达时间 t。 另一个微妙的问题是忘记每个残基的最早有效 s 就足够了； 扫描多个周期是不必要的，并且会增加运行时间而没有任何好处。 

## 工作示例

 考虑一个具有单一边缘类型的简化场景，其中 c 很小，因此我们可以明确地看到残差行为。 假设我们在时间 t = 3 且 c = 5 时到达一颗行星。每个残基的候选开始时间是残基 0 到 4 模 5 的下一个时间 ≥ 3，即 5、6、7、8 和 9。每个残基都会评估一次，并选择最佳的到达时间。 

| 残基 r | 开始时间 s | 计算成本组成|
 | --- | --- | --- |
 | 0 | 5 | 5 + (a·5 + b 模 c) + d |
 | 1 | 6 | 6 + (a·6 + b 模 c) + d |
 | 2 | 7 | 7 + (a·7 + b 模 c) + d |
 | 3 | 8 | 8 + (a·8 + b 模 c) + d |
 | 4 | 9| 9 + (a·9 + b 模 c) + d |

 该跟踪表明，残基对齐完全捕获了等待决策，并且不需要考虑更大的等待窗口。 

作为第二种情况，想象一个可以通过次优路径更早到达的节点，该路径到达速度很快，但有不利的剩余时间。 Dijkstra 确保即使先处理提前到达，任何在更好时间到达的后续改进都将正确更新状态并重新处理转换，因为每次松弛都会从新时间重新计算最佳等待决策。 

## 复杂度分析

 | 时间 | O(n·(n + m·c)) | O(n·(n + m·c)) | Dijkstra 在 n 个节点上，最多具有 n^2 条边（以矩阵形式），每个松弛扫描最多 m 个类型的 c 个残基 |

 ## 边缘情况

 当某个残差的开始时间已经是最佳时，算法会立即为该残差选择 s = t，而无需等待，从而正确捕获立即出发的情况。 如果更好的残基需要等待，则计算出的 s 在同一周期内向前移动，确保不会丢失对齐。 

当 c = 1 时，每个 s 具有相同的残差，因此残差上的循环会折叠为单个候选 s = t，并且边缘的行为类似于成本为 t + (a t + b mod 1) + d = t + d 的固定权重边缘，因为模项始终为零。 该算法自然地处理这个问题，无需特殊的外壳。 

当多个边连接同一对行星时，每个边都会被独立评估，并且 Dijkstra 会正确选择最佳的到达时间。
