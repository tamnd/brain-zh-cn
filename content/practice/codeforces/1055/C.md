 ## 解决方案
 ## 问题理解

 两个人在天数线上都有“良好间隔”的周期性模式。 每个模式都由重复周期内连续几天的固定部分组成。 对于爱丽丝来说，在每个长度块内`t_a`，天数从`l_a`到`r_a`（包括在内）都是好的，同一块中该段之外的所有内容都是坏的。 相同的结构适用于带参数的 Bob`l_b`,`r_b`,`t_b`。 

两种模式都通过将其良好部分移动其周期的倍数来无限重复。 如果某一天处于一个人的轮班阶段，那么这一天对一个人来说就是好事。 

任务是确定连续天数块的最大长度，以便该块中的每一天同时对 Alice 和 Bob 都有好处。 

关键的困难在于，两种模式都是周期性的，但周期不同，因此交集结构不是周期性的，周期明显较小，而且在大范围的天数内进行强力对齐是不可能的，因为周期值可能大到 10^9。 

试图在整个最小公倍数周期上显式构造或模拟模式的简单方法是立即不可行的，因为两个大整数的 LCM 可能超过 10^18 并且该结构仍然太大而无法枚举。 

当一个间隔完全包含在另一个间隔中以进行残基的某些对齐时，就会出现一种微妙的边缘情况。 例如，如果爱丽丝总是擅长`[0, 4] mod 10`鲍勃很擅长`[2, 3] mod 10`，答案是 2，但错误地转变直觉可能会建议跨越周期边界进行环绕或合并，这是无效的，因为每个周期都会独立重置。 

当最佳重叠跨越一个时期的边界但不跨越另一个时期的边界时，就会出现另一种棘手的情况。 例如，鲍勃可能在一个周期结束和下一个周期开始时表现良好，而爱丽丝则持续表现良好。 尽管交叉点跨越了两个模块块，但在绝对时间上它仍然是一个连续的片段。 

## 方法

 蛮力的观点是检查每个开始日并向前延伸，同时爱丽丝和鲍勃都处于良好的间隔。 对于每一天`x`，我们在 O(1) 中测试 Alice 的周期间隔和 Bob 的周期间隔的隶属度，然后延长直到条件满足。 在最坏的情况下，答案本身可能很大，但更重要的是，我们将为每个起始位置重复工作，从而导致在巨大的隐式天数范围内出现 O(N^2) 行为。 

这失败了，因为结构周期性地重复，并且从头开始重新计算重叠忽略了只有两个周期之间的相对相位重要的事实。 关键的观察结果是，两种模式都是移位间隔的并集，它们的相互作用仅取决于它们的位置以各自周期为模。 

我们不是随着时间的推移进行扫描，而是查看两个周期性模式的相对排列。 确定爱丽丝周期中的某一天，并确定鲍勃的良好间隔相对于该日期的位置。 两个周期段系统之间的重叠模式随周期重复`t_a`，因为爱丽丝的结构已经具有该周期的周期性。 因此，我们可以将注意力限制在 Alice 的单个循环上，并计算其中每个点的交集向前持续的时间。 

在爱丽丝的周期中，鲍勃的好日子表现为多个转移的片段。 我们将 Bob 的间隔映射到 Alice 的坐标系中，然后检查线性化段中间隔之间的重叠`[0, t_a - 1]`，注意处理 Bob 片段的环绕。 一旦这两个集合都在可比较的模空间中表达，问题就简化为找到圆上间隔之间的最长重叠，这可以通过复制循环来线性化。 

然后，我们扫描重叠线段的端点并计算最大连续交叉长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 隐式时间内的 O(N²) | O(1) | O(1) | 太慢了 |
 | 周期对齐+区间交集| O(t_a + t_b) | O(t_a + t_b) | 已接受 |

 ## 算法演练

 我们将与爱丽丝周期相关的所有内容标准化，因为爱丽丝的结构已经是一个干净的长度周期划分`t_a`。 

1.将Alice的好段转换为一个周期内的单个区间，即`[l_a, r_a]`。 这代表了一个代表性时期内爱丽丝表现良好的所有点。 
2. 将 Bob 的好片段扩展为与 Alice 周期一致的形式。 每个鲍勃间隔`[l_b + k t_b, r_b + k t_b]`投影为模余数`t_a`。 我们不需要枚举无限多个移位，我们只需要 Bob 模的模式`t_a`，可以通过采用 Bob 的基本间隔并考虑其与长度为滑动窗口的重叠来生成`t_a`。 
3. 内部构建一组候选区间`[0, t_a)`爱丽丝和鲍勃都很好。 这是通过将 Alice 的单个间隔与可能在一个周期内与其相交的所有移位的 Bob 间隔相交来完成的。 
4. 由于 Bob 的区间可能会环绕循环边界，因此分割任何跨越的区间`t_a - 1`分成两个线性段。 这将问题转换为标准线间隔相交问题。 
5. 将所有交叉区间按起点排序并合并。 在合并期间，跟踪任何单个合并段的最大长度。 

答案是任何合并区间的最大长度。 

### 为什么它有效

 爱丽丝的美好时光由重复的相同线段形成整数线的完美平铺`t_a`。 因此，当通过爱丽丝的结构观察时，任何全局交叉模式都必须以相同的周期重复。 通过将分析限制在 Alice 的一个完整周期内，我们不会丢失任何相对重叠的配置。 

在该周期内，鲍勃的贡献完全通过他的周期段投射到剩余空间中来体现。 每个可能的同时良好拉伸完全对应于该缩减域内的交叉段。 因为我们仅在完全投影后合并间隔，所以我们保留连续性，并且不会意外连接跨循环边界的不相交重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_intervals(l, r, t, limit):
    res = []
    # generate occurrences that can intersect [0, limit)
    k = 0
    while l + k * t < limit:
        start = l + k * t
        end = r + k * t
        if start < limit and end >= 0:
            res.append((start, min(end, limit - 1)))
        k += 1
        if k > limit // max(1, t) + 5:
            break
    return res

def intersect(a1, a2, b1, b2):
    l = max(a1, b1)
    r = min(a2, b2)
    if l <= r:
        return (l, r)
    return None

def solve():
    l_a, r_a, t_a = map(int, input().split())
    l_b, r_b, t_b = map(int, input().split())

    # we work inside one cycle of Alice
    limit = t_a

    alice = [(l_a, r_a)]

    bob_intervals = build_intervals(l_b, r_b, t_b, limit + t_b)

    candidates = []
    for a1, a2 in alice:
        for b1, b2 in bob_intervals:
            inter = intersect(a1, a2, b1, b2)
            if inter:
                candidates.append(inter)

    if not candidates:
        print(0)
        return

    candidates.sort()
    merged = []
    cur_l, cur_r = candidates[0]

    best = 0
    for l, r in candidates[1:]:
        if l <= cur_r + 1:
            cur_r = max(cur_r, r)
        else:
            best = max(best, cur_r - cur_l + 1)
            cur_l, cur_r = l, r

    best = max(best, cur_r - cur_l + 1)
    print(best)

if __name__ == "__main__":
    solve()
```该代码首先将问题简化为有界大小窗口内的交集`t_a`，这已经足够了，因为爱丽丝的模式完全重复每个`t_a`。 然后，它生成可能与此窗口重叠的相关 Bob 段，而不是尝试抽象地推理无限算术级数结构。 

每个候选重叠是通过直接区间交集来计算的。 然后将它们合并以形成最大连续段，其中两者同时良好。 最终的答案是最长的此类合并片段。 

一个微妙的点是Bob间隔到窗口的截断`[0, t_a + t_b)`。 这确保我们捕获可以包含在 Alice 的第一个周期中的所有转变，同时避免无限枚举。 

## 工作示例

 ### 示例 1

 输入：```
0 2 5
1 3 5
```我们按照爱丽丝周期工作`[0, 4]`，爱丽丝擅长的地方`[0, 2]`。 鲍勃产生间隔`[1, 3]`,`[6, 8]`等，但仅在第一个周期内`[1, 3]`很重要。 

| 步骤| 爱丽丝间隔| 鲍勃间隔 | 交叉口|
 | ---| ---| ---| ---|
 | 1 | [0,2]| [1,3]| [1,2]|

 唯一的重叠是`[1,2]`，长度为 2。这就是答案。 

这证实了即使鲍勃的模式在第一个周期之后继续存在，但只有与爱丽丝的第一个周期相交的部分对于最大连续重叠很重要。 

### 示例 2

 输入：```
0 1 4
2 3 6
```爱丽丝很擅长`[0,1]`每 4 次重复一次。鲍勃很擅长`[2,3]`每 6 次重复。 

在爱丽丝的周期内`[0,3]`, Alice 的良好间隔为`[0,1]`。 鲍勃在该区域没有贡献重叠。 

| 步骤| 爱丽丝间隔| 鲍勃间隔 | 交叉口|
 | ---| ---| ---| ---|
 | 1 | [0,1]| 没有重叠| []|

 答案是0。 

这显示了周期性结构在循环内永远不会对齐的情况，因此全局交集为空。

 ## 边缘情况

 一种边缘情况是鲍勃的间隔在其周期接近结束时开始并进入下一个周期。 例如，鲍勃`[4, 1] mod 6`有效地分为`[4,5]`和`[0,1]`。 该算法通过生成移位间隔来处理此问题，当投影到爱丽丝循环中时，该间隔自然覆盖两个部分，确保不会错过任何交集。 

另一个边缘情况是当 Alice 的区间覆盖整个周期时，即`l_a = 0, r_a = t_a - 1`。 在这种情况下，答案就简化为 Bob 模式中最长的连续块。 交集逻辑仍然有效，因为 Alice 不再限制域，并且所有 Bob 重叠都被正确保留和合并。 

当两个周期都很大但几乎相等时，就会出现最后的边缘情况，导致重叠在周期之间缓慢漂移。 将计算限制为单个 Alice 周期可确保即使是缓慢的漂移模式也能完全捕获，而无需模拟整条线路。
