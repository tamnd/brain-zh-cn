 ## 解决方案
 ## 问题理解

 地铁系统可以看作是从 1 到 n 的一系列车站，有两种定向运动方式。 一条轨道允许从较小的索引移动到较大的索引，而另一条轨道允许沿相反方向移动。 Bob 从车站 1 出发，想要到达车站 s。 

复杂的是每个车站可能或可能不允许独立地在每个方向停车。 即使火车经过某个车站，如果该车站对该轨道方向开放，鲍勃也只能将其用作换乘点或停靠点。 如果它关闭，火车仍然通过，但鲍勃无法在该方向上车或下车，这实际上删除了该车站作为该有向轨道上的可用节点的地位。 

任务是确定在可到达的车站是否存在任何有效登车和可能的转乘序列，允许从车站 1 移动到车站 s。 

输入大小 n 最多为 1000，这意味着 O(n^2) 遍历甚至 O(n) 个节点和 O(n) 个边上的简单图搜索就足够了。 任何指数都是不必要的，但即使是简单的 BFS 或邻接模拟也很容易足够快。 

当 Bob 最初无法访问站点 1 的前向时，就会出现一种微妙的边缘情况。在这种情况下，即使通过先向后然后切换方向存在有效路径，如果我们假设单调移动或忽略方向切换，也可能会错过该路径。 

另一种情况发生在车站 s 只能通过绕道到较远的车站然后沿相反方向返回才能到达的情况下。 例如，如果向前方向提前被阻止但向后方向​​开放，则鲍勃可能需要首先前往最右边的可达站点。 

## 方法

 该结构自然形成一个有向图，其中每个站都是一个节点。 如果第一条轨道在 i 开放，Bob 可以从站 i 移动到 i+1，如果第二条轨道在 i 开放，则 Bob 可以移动到 i-1。 该问题简化为从节点 1 到节点 s 的可达性。 

暴力方法将尝试枚举所有可能的路线，可能会通过反复访问来探索路径。 这是正确的，但复杂性可能会爆炸，因为带有循环的图中的行走次数呈指数增长。 即使 n 达到 1000，简单的路径枚举也是不可行的。 

关键的观察结果是，这是具有 n 个节点和最多 2n 个边的图中的标准可达性问题。 一旦正确建模，简单的 BFS 或 DFS 就足够了，因为每个站最多有两个传出转换，一个向前，一个向后。 这些约束确保访问每个节点一次就足够了。 

蛮力之所以有效，是因为所有转换都是局部的和确定性的，但当重新访问造成重复状态的爆炸时，就会失败。 我们只关心节点是否可达，而不关心如何到达，这一观察将问题简化为图遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解路径 | O(2^n) | O(2^n) | O(n) 递归堆栈 | 太慢了 |
 | BFS/DFS 可达性 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个站建模为图中的一个节点。

1. 根据允许的移动建立邻接。 从车站 i 开始，如果 i + 1 ≤ n 且 a[i] = 1，则添加一条从 i 到 i + 1 的边。如果 i − 1 ≥ 1 且 b[i] = 1，则添加一条从 i 到 i − 1 的边。这些边表示 Bob 是否可以实际登上沿该方向继续行驶的火车并在下一站停靠。 
2. 初始化一个大小为 n + 1 的已访问数组，以跟踪哪些站点已被处理。 这可以防止多次重新访问同一站点，否则会导致冗余探索。 
3. 从站点 1 启动 BFS（或 DFS），将其标记为已访问。 站 1 始终是 Bob 的起点，因此很容易到达。 
4. 当 BFS 队列中有未处理的站点时，删除一个站点 u 并探索通过允许的边可到达的所有邻居 v。 如果v还没有被访问过，则标记它并将其添加到队列中。 
5、遍历结束后，检查s站是否被访问过。 如果是，则输出“YES”； 否则输出“NO”。 

### 为什么它有效

 在任何时刻，BFS 都保持不变，即使用有效转换可以从站 1 到达每个访问站。 每条边都对应于车站和轨道约束下的合法运动，因此任何新发现的车站也是可达的。 由于 BFS 不重复地探索所有可达状态，因此它最终会发现从 1 开始可达的所有站点。如果站点 s 不在其中，则没有有效的运动序列可以到达它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, s = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    adj = [[] for _ in range(n + 1)]

    for i in range(n):
        if i + 1 < n and a[i] == 1:
            adj[i + 1].append(i + 2)
        if i - 1 >= 0 and b[i] == 1:
            adj[i + 1].append(i)

    visited = [False] * (n + 1)
    q = deque([1])
    visited[1] = True

    while q:
        u = q.popleft()
        for v in adj[u]:
            if not visited[v]:
                visited[v] = True
                q.append(v)

    print("YES" if visited[s] else "NO")

if __name__ == "__main__":
    solve()
```邻接构造将方向约束直接编码到边中，因此 BFS 在标准图上运行。 前向边缘使用 a[i]，因为它们对应于从 i 到 i + 1 的移动。后向边缘使用 b[i]，因为它们对应于从 i 到 i − 1 的移动。 

一个常见的实现陷阱是差一索引，因为输入是从 1 开始的，但数组通常是从 0 开始的。 这里，使用 i + 1 作为节点索引构建邻接关系，因此所有图节点始终从 1 开始，避免混淆。 

另一个微妙的问题是忘记移动取决于当前车站，而不是目的地车站。 添加边之前必须在 i 处检查条件。
