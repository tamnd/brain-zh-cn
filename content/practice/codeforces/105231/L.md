 ## 解决方案
 ## 问题理解

 我们得到一个代表校园的加权无向图。 每个节点包含一定数量的游客，并且某些节点被指定为门。 每个门仅在特定时间间隔内激活。 随着从 1 到 T 的时间，活动门的集合发生变化。 

在任何固定的时刻，每位游客都必须通过当前开放的大门离开校园。 游客沿着边权重以单位速度沿着图中的最短路径行进，因此从节点 u 开始的游客的成本是从 u 到所选活动门的最短路径距离。 每个游客总是选择最好的登机口，即最近的当前开放的登机口。 

对于每个时刻，我们必须计算最优选择下所有游客的总旅行距离。 如果某个时刻没有门打开，那么任何人都不可能离开，此时的答案定义为-1。 

该图有多达 100,000 个节点和边，但门的数量最多为 15 个。这是关键的结构约束：虽然图很大，但特殊决策点（门）的集合很小，这强烈建议预先计算距门的距离。 

每个时刻重新计算最短路径的简单方法是立即不可能的，因为 T 可以是 100,000。 即使每分钟一次 Dijkstra 也需要 10^10 次操作。 

当考虑每个门配置的重新计算时，会出现一个更微妙的问题。 活动的门集随着时间的推移而变化，但它不是每个查询任意的，它是由间隔定义的。 这意味着活动集是分段常数，并且仅在间隔端点处变化。 

一个典型的陷阱是尝试错误地重新计算每个节点或每个门的距离，而没有意识到正确的结构是一个小的动态源子集上的多源最短路径问题。 

## 方法

 直接的暴力策略是独立处理每个时刻。 对于固定时间 t，我们确定哪些门处于活动状态，然后对于每个节点，我们计算到任何活动门的最短距离。 这需要从每个活动门运行 Dijkstra 或使用所有活动门初始化的多源 Dijkstra。 

即使我们每次优化并使用多源Dijkstra，最坏的情况下T仍然高达100,000。 每次运行的成本为 O(m log n)，因此总复杂度变为 O(T m log n)，这远远超出了可行的限制。 

关键的观察结果是重新计算最短路径是不必要的。 每个节点到每个门的距离可以预先计算一次。 由于 k 至多为 15，因此我们可以从每个门独立运行 Dijkstra。 这给了我们一个距离表，其中对于每个节点 u 和门 g，我们都知道 dist[g][u]。 

经过此预处理后，每个时刻唯一剩下的任务是评估每个节点当前活动门之间的最小距离。 由于 k 很小，我们可以简单地扫描每个节点的活动门。 

第二个结构见解是活动门组不会每时每刻都改变。 每个门最多贡献两个事件，打开和关闭，因此整个时间线最多分为 O(k) 个常数段。 在每个段内，活动门的集合是固定的，因此我们可以为每个段计算一次答案。 

这将问题从 T 次重新计算减少到 O(k) 次重新计算，每次成本为 O(nk)，这是可以接受的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 使用 Dijkstra 每次重新计算 | O(T·m log n) | O(T·m log n) | O(n + m) | 太慢了|
 | 预计算距离+分段评估| O(k·m log n + k·n·k + T) | O(k·n) | O(k·n) | 已接受 |

 ## 算法演练

 ### 1. 计算每个门的最短路径

 我们从每个门节点开始独立运行 Dijkstra。 这会生成每个门到每个节点的距离数组。

这样做的原因是门是唯一可能的目的地，因此我们需要可重用的最短路径信息，而不是重复重新计算它。 

### 2. 将门间隔转换为事件

 每个门在区间 [l, r] 上处于活动状态。 我们将其转换为两个事件：在时间 l 时，门变为活动状态，在时间 r+1 时，门变为不活动状态。 

我们按时间对所有事件进行排序。 在连续的事件时间之间，活动门设置不会改变。 

### 3.构建时间段结构

 我们扫描从 1 到 T 的时间，维持当前活动的门集。 每当我们传递一个事件时，我们都会更新该集合。 集合不变的每个最大间隔成为一个段。 

这减少了多次评估固定配置的问题。 

### 4. 预计算节点贡献

 我们存储每个节点的游客数量。 对于固定的活动集S，节点u的贡献是a[u]乘以dist[g][u]的S中所有g的最小值。 

### 5. 评估每个部分

 对于每个段，我们通过迭代所有节点来计算总答案。 对于每个节点，我们扫描所有活动门并取最小距离。 

如果活动集为空，则该段中每个时间的答案都是 -1。 

### 为什么它有效

 对于每个时刻，每个游客独立地选择一条到达最近的活动门的最短路径。 由于最短路径独立于其他游客，因此总成本只是节点上的总和。 预先计算距每个门的距离可确保对于任何活动子集，我们都可以重建正确的最近门距离。 分段参数确保我们永远不会重新计算相同的门配置两次，因此每个时刻都用正确的活动信息精确地计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

INF = 10**18

def dijkstra(start, n, adj):
    dist = [INF] * (n + 1)
    dist[start] = 0
    pq = [(0, start)]
    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in adj[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

def solve():
    n, m, k, T = map(int, input().split())
    a = [0] + list(map(int, input().split()))

    adj = [[] for _ in range(n + 1)]
    for _ in range(m):
        u, v, w = map(int, input().split())
        adj[u].append((v, w))
        adj[v].append((u, w))

    gates = []
    events = []
    for i in range(k):
        p, l, r = map(int, input().split())
        gates.append(p)
        events.append((l, i, 1))
        events.append((r + 1, i, -1))

    events.sort()

    # preprocess distances
    dist = []
    for g in gates:
        dist.append(dijkstra(g, n, adj))

    active = set()
    ans = [-1] * (T + 2)

    ei = 0
    for t in range(1, T + 1):
        while ei < len(events) and events[ei][0] == t:
            _, idx, typ = events[ei]
            if typ == 1:
                active.add(idx)
            else:
                active.discard(idx)
            ei += 1

        if not active:
            ans[t] = -1
            continue

        total = 0
        for i in range(1, n + 1):
            best = INF
            for g in active:
                best = min(best, dist[g][i])
            total += best * a[i]
        ans[t] = total

    for t in range(1, T + 1):
        print(ans[t])

if __name__ == "__main__":
    solve()
```该代码首先构建图的邻接列表，然后读取门位置并将其可用间隔转换为事件点。 之后，它为每个门运行一次 Dijkstra，存储从每个门到所有节点的完整距离图。 

在时间扫描期间，活动的门集逐渐更新。 对于每个时刻，如果没有门处于活动状态，则结果立即为 -1。 否则，该算法通过扫描活动门并采用最小预先计算距离来计算每个节点的贡献。 这是安全的，因为所有最短路径信息已经是固定的并且与时间无关。 

一个微妙的点是我们使用 r + 1 作为移除事件，确保包容性区间处理而无需特殊的大小写。 

## 工作示例

 考虑一个在一条线上有三个节点的小图，其中节点 2 是仅在时间 1 活跃的门，并且所有节点都有一名游客。 

在时间 1，活动集包含节点 2。到节点 2 的距离为 1、0 和 1，因此总成本为 2。在时间 2，没有门处于活动状态，因此结果变为 -1。 

| 时间 | 主动门| 节点 1 分钟距离 | 节点 2 | 节点 3 | 总计 |
 | ---| ---| ---| ---| ---| ---|
 | 1 | {2} | 1 | 0 | 1 | 2 |
 | 2 | {} | - | - | - | -1 |

 这显示了空集条件如何直接传播到输出而不进行任何计算。 

现在考虑两个在时间上重叠的门，一个靠近图的左侧，一个靠近图的右侧。 中间的节点将根据哪个提供更小的预计算距离来切换最近的门。 下表说明了两者都处于活动状态的单个时间。 

| 节点| 距离 G1 | 距离 G2 | 分钟 | 贡献 |
 | ---| ---| ---| ---| ---|
 | 1 | 0 | 5 | 0 | 0 |
 | 2 | 2 | 2 | 2 | 2a2 | 2a2
 | 3 | 5 | 0 | 0 | 0 |

 这证实了该算法正确解决了跨多个源的每节点最小化问题。

 ## 边缘情况

 一个重要的边缘情况是所有门同时处于非活动状态。 在这种情况下，算法立即输出 -1，因为活动集为空。 这避免了在空集上计算最小值的任何尝试。 

另一个边缘情况是重叠间隔导致频繁切换。 基于事件的扫描确保每个切换只处理一次，因此不会错过或重复任何时刻。 

最后一个边缘情况是游客数量较多的节点。 由于贡献值乘以 a[i]，该算法在 Python 中隐式累加到 64 位整数变量中，因此溢出不是问题，但在其他语言中必须小心处理。
