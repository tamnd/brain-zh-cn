 ## 解决方案
 ## 问题理解

 我们有一个大的数字网格，将其视为值的矩阵。 我们还修复了尺寸较小的矩形“过滤器”的尺寸$k \times l$。 该过滤器将在大网格的每个有效位置上滑动，并在每个位置通过对所覆盖元素进行加权求和来产生一个输出值。 整个过程的最终输出是另一个包含所有这些滑动窗口结果的矩阵。 

关键的自由是我们没有获得过滤器本身。 相反，我们可以独立选择过滤器的每个单元格，但每个单元格必须是三个值之一：$-1$,$0$， 或者$1$。 选择滤波器后，我们计算卷积输出矩阵，然后对该输出中的所有值求和。 任务是最大化这个最终总和。 

约束条件$n, m \le 1000$意味着输入矩阵最多可以包含一百万个单元。 任何试图枚举所有可能的内核的解决方案都是不可能的，因为内核的数量是$3^{k \cdot l}$，呈指数增长。 即使对单个内核简单地计算卷积，如果重复的话也会太慢。 

真正的约束压力是我们需要一些大致线性或接近线性的东西$nm$，因为像这样的事情$O(nm \cdot k \cdot l)$已经处于最大尺寸的边界。 

一个可以打破天真的推理的微妙问题是假设内核选择跨位置相互作用。 例如，人们可能错误地认为内核的某个单元以需要全局优化的耦合方式影响重叠区域。 另一个陷阱是假设我们必须为每个候选核值显式模拟卷积。 两者都是不必要的，并且会导致过于复杂化。 

## 方法

 思考这个问题的一种直接方法是修复内核并计算卷积输出，然后对所有内容求和。 这很简单：对于每个输出位置，我们乘以$k \times l$具有相应子矩阵和总和的内核。 对所有输出位置重复此操作得出的总成本为$O((n-k+1)(m-l+1)kl)$，在最坏的情况下大约是$10^{12}$的行动，远远超出了可行的限度。 

主要的简化来自于求和顺序的颠倒。 我们不认为“每个输出单元依赖于应用于补丁的内核”，而是认为“每个内核单元对许多输出单元做出贡献”。 一旦我们交换总和，每个内核位置$(x,y)$乘以完全从输入矩阵导出的固定量。 该数量不依赖于其他内核单元。 

经过此转换后，问题不再是滑动窗口问题，而是成为简单的每单元优化：对于每个内核位置，选择$-1$,$0$， 或者$1$最大化线性表达式。 最优选择通过符号立即出现。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力卷积 |$O(nmkl)$|$O(1)$| 太慢了|
 | 重新排序求和 |$O(nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 我们以将内核选择与输入聚合分开的方式重写总分。 

1. 计算辅助矩阵$S$尺寸的$k \times l$，其中每个条目$S[x][y]$表示输入网格对核位置的总贡献$(x,y)$跨所有输出布局。 这是通过计算当内核单元被覆盖时每个输入单元被覆盖的次数来获得的。$(x,y)$用于卷积。 
2. 观察固定的内核单元$(x,y)$贡献$K[x][y] \cdot S[x][y]$到最终的答案。 这意味着总分是内核单元的独立贡献的总和。 
3. 由于每个内核条目都可以从 {-1, 0, 1} 中独立选择，因此分别最大化每个项。 如果$S[x][y]$为正，选择$K[x][y] = 1$。 如果为负数，则选择$K[x][y] = -1$。 如果为零，则选择$0$。 
4. 最终答案就是所有的绝对值之和$S[x][y]$。 

因此，关键的计算任务是计算所有$S[x][y]$高效。 每个$S[x][y]$对应于输入网格的固定子矩阵的总和，特别是从$(x,y)$与高度$n-k+1$和宽度$m-l+1$。 这可以使用 2D 前缀和在每个单元的恒定时间内计算出来。 

### 为什么它有效

 关键的属性是卷积和的线性。 交换求和后，目标变成核矩阵和从输入导出的固定矩阵之间的点积。 一旦采用这种形式，每个决策变量$K[x][y]$仅影响总和中的一项，与其他项没有耦合。 这消除了所有组合结构，将问题减少到每个单元的独立符号选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m, k, l = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(n)]

    # prefix sum
    ps = [[0] * (m + 1) for _ in range(n + 1)]
    for i in range(n):
        row_sum = 0
        for j in range(m):
            row_sum += a[i][j]
            ps[i + 1][j + 1] = ps[i][j + 1] + row_sum

    def get(x1, y1, x2, y2):
        return ps[x2][y2] - ps[x1][y2] - ps[x2][y1] + ps[x1][y1]

    h = n - k + 1
    w = m - l + 1

    ans = 0
    for i in range(k):
        for j in range(l):
            x1, y1 = i, j
            x2, y2 = i + h, j + w
            s = get(x1, y1, x2, y2)
            if s > 0:
                ans += s
            else:
                ans -= s

    print(ans)

if __name__ == "__main__":
    main()
```前缀和构造将任何子矩阵和查询压缩为常数时间，这一点至关重要，因为每个内核位置都需要一个这样的查询。 矩形边界直接来自对内核单元参与有效卷积放置的频率的观察。 

一个常见的实现错误是前缀和中的逐一处理。 这里，前缀数组的大小$(n+1) \times (m+1)$以便子矩阵查询变得干净并且不需要特殊的套管边缘。 

## 工作示例

 考虑一个结构可见的小矩阵：

 输入：```
3 3 2 2
1 2 3
4 5 6
7 8 9
```这里$h = 2$,$w = 2$。 我们计算$S$超过内核位置：

 | (i,j) | 子矩阵和 | S值| 选择K | 贡献 |
 | ---| ---| ---| ---| ---|
 | (0,0) | (0,0) | 1+2+4+5 = 12 | 12 | 12 1 | 12 | 12
 | (0,1)| 2+3+5+6 = 16 | 16 | 16 1 | 16 | 16
 | (1,0)| 4+5+7+8 = 24 | 24 | 1 | 24 |
 | (1,1) | 5+6+8+9 = 28 | 28 | 28 1 | 28 | 28

 最终答案是$12 + 16 + 24 + 28 = 80$。 

该轨迹表明，每个内核位置最终都有利于$+1$因为所有子矩阵和都是正数，因此确认解正确地简化为绝对累加。

 ## 边缘情况

 极端情况是给定内核位置的所有子矩阵和为零。 在这种情况下，该单元格的最佳选择是将内核值设置为零，不做出任何贡献。 该算法自然会处理这个问题，因为$|0| = 0$，所以它既不增加也不减少任何东西。 

另一种情况是当$k = n$和$l = m$。 这里卷积只有一个位置，缩减中的矩形大小变为$1 \times 1$。 该算法简化为根据相应输入单元的符号选择每个内核单元，这与完全重叠的直接解释相匹配。 

最后一个微妙的情况是输入值的幅度很大但符号混合。 由于该解决方案隐式使用 64 位 Python 整数，因此不存在溢出问题，并且前缀和保持稳定，因为它们只累加有界整数的加法。
