 ## 解决方案
 ## 问题理解

 我们得到一个由数组表示的一维韭菜字段。 工作人员从固定位置开始，沿着这条线移动固定数量的时间步长。 每一步，所有韭菜都均匀地生长相同的数量。 生长完成后，工人会在当前位置收割韭菜，从而将该位置重置为零，然后可以选择向左、向右移动或留在原地。 

关键是，在一个位置收集的价值不仅取决于它的初始值，还取决于自上次收获以来已经过去了多长时间，因为增长是在每一步中全局应用的。 随着时间的推移，每个位置都在不断增加，但每当我们收获它时，它就会被重置并从零开始再次增长。 

任务是在固定数量的步骤中选择移动策略，以最大化总收获价值。 

限制条件非常大。 测试用例的数量最多可达 100000 个，所有测试用例的总数组大小最多可达 200000 个，而每个测试的步骤数最多可达 10^6。 这立即排除了任何显式处理每个步骤的模拟。 任何解决方案都必须将问题简化为主要取决于每次测试的 n 的结构，理想情况下是线性或近线性的。 

一个微妙的困难来自于运动和重复收获之间的相互作用。 天真的直觉是，重新审视某个位置可能是有益的，因为增长会积累，但重置操作会破坏这种积累。 这会产生不明显的边缘行为。 

一些小例子可以澄清陷阱。 

如果我们从不移动并停留在单个单元格上，我们会因全球增长而不断收集不断增加的价值，但我们也会不断重置同一单元格，这意味着我们在第一次访问后失去了基础值的所有累积收益。 保持静止的天真贪婪显然是次优的。 

如果我们行动过于激进，我们就会花费脚步行走而不是收获，从而失去潜在的收获。 最佳解决方案必须在行驶距离与收集高价值路段之间取得平衡。 

另一个棘手的边缘情况是 k 为零时。 那么价值根本不会增长，问题就简化为选择一条最大化最初访问价值总和的步行，而无需重新审视收益。 这迫使解决方案统一处理静态和动态情况。 

## 方法

 第一种方法是直接逐步模拟该过程。 在每个时间步，我们增加所有值，将当前位置添加到答案中，重置它，然后选择下一步。 这是正确的，但完全不可行。 每个步骤都需要更新整个数组或维护惰性结构，并且每个测试用例最多 10^6 个步骤，这在最坏的情况下会导致大约 10^11 次操作。 

关键的观察结果是，全球增长成分与路径无关。 无论位置如何，每个步骤都贡献相同的加法项 k 乘以当前时间步骤。 这意味着增长的全部贡献可以与运动优化分开。 一旦分离，剩下的问题就纯粹是关于初始值和重新访问带来的惩罚。 

消除均匀增长贡献后，问题就变成选择一条线上的行走，其中每个位置只贡献其初始值一次，并且重新访问某个位置不会增加其贡献，但会浪费移动预算。 这将问题简化为选择一系列不同的访问位置形成连续的线段，因为任何在没有重复的情况下访问节点的线上的最佳行走都必须位于线段上。

访问一个段的成本取决于我们如何从初始位置开始遍历它。 从固定的起点 p 开始，覆盖区间 [l, r] 需要首先步行到一个端点，然后扫过该区间。 根据方向，这会产生两种可能的遍历成本。 

这将问题转化为具有几何可行性约束的最大和子数组：我们想要包含 p 的最佳区间，以便可以在 t0 步内完全遍历该区间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 步进模拟 | O(t0·n) | O(t0·n) | O(n) | 太慢了 |
 | 具有遍历约束的区间优化 | 每次测试 O(n)（摊销）| O(n) | 已接受 |

 ## 算法演练

 我们将问题简化为选择包含起始位置 p 的段 [l, r]，以便可以在允许的移动次数内访问段中的所有位置。 片段的分数是该片段上 v[i] 的总和。 

1. 修复包含p的段[l,r]。 工作人员必须按某种顺序访问该段中所有选定的位置，而不能重复，因为重新访问不会增加收集的基础值。 
2. 计算从 p 开始并访问 [l, r] 中所有点所需的最少步数。 有两种自然遍历顺序。 如果我们先向左走，我们会从 p 走到 l，然后扫到 r，成本为 (p - l) + (r - l)。 如果我们先向右走，我们会从 p 走到 r，然后回到 l，成本为 (r - p) + (r - l)。 
3. 该段的真实成本是这两个值中的最小值。 我们要求该成本最多为 t0 才能使该段可行。 
4. 现在我们需要在包含 p 的所有可行段上最大化 v[i] 的总和。 这是一个具有非平凡可行性约束的经典区间优化问题。 
5. 我们对 l 和 r 使用两指针技术。 对于每个固定的l，我们在保持可行性的同时尽可能地扩展r。 我们维护前缀和，以便可以在 O(1) 内计算段和。 
6. 我们通过使用导出的成本公式检查可行性来隐式评估两种遍历情况，确保对于每个 (l, r) 我们只考虑有效的配置。 

### 为什么它有效

 任何最佳策略都对应于访问一组不同位置的步行。 在一条线上，如果我们想避免浪费穿过空白空间的步骤，任何这样的集合都必须形成连续的区间。 对于固定间隔，任何最佳遍历都会减少为从 p 开始的两个单调扫描之一。 由于增益只取决于访问了哪些位置，而不取决于访问它们的顺序，因此我们只需确保区间的可行性，然后最大化其总和即可。 这将动态运动问题转换为静态区间选择问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, p = map(int, input().split())
    a = list(map(int, input().split()))
    k, t0 = map(int, input().split())

    p -= 1

    # prefix sums for interval queries
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]

    def seg_sum(l, r):
        return pref[r + 1] - pref[l]

    ans = 0
    r = p

    # two pointers over left endpoint
    for l in range(p, -1, -1):
        if r < l:
            r = l

        while r < n:
            # cost if go left first then right
            cost_left = (p - l) + (r - l)
            # cost if go right first then left
            cost_right = (r - p) + (r - l)
            if min(cost_left, cost_right) <= t0:
                r += 1
            else:
                break

        ans = max(ans, seg_sum(l, r - 1))

    print(ans)

if __name__ == "__main__":
    solve()
```该实现预先计算前缀和以计算恒定时间内的任何间隔和。 主循环固定左端点并贪婪地扩展右端点，同时满足遍历约束。 可行性检查直接对从起始位置开始的两个可能的扫描顺序进行编码。 

一个常见的实现陷阱是两指针扩展中的差一处理。 指针 r 保持为独占，因此每次可行性检查都使用 r - 1 作为实际区间端点。 另一个微妙之处是确保每一步都考虑两个遍历方向，因为当起始位置不在中心位置时，忽略一个方向会导致丢失最佳线段。 

## 工作示例

 考虑 n = 5、p = 3、值 [1, 3, 2, 5, 4] 的小型配置，并假设 t0 较小，以便只有短段可行。 

我们检查区间如何扩展。 

| 我| r | 段 | 成本左优先 | 成本右优先| 可行| 总和|
 | --- | --- | --- | --- | --- | --- | --- |
 | 3 | 3 | [4] | 0 | 0 | 是的 | 5 |
 | 2 | 3 | [3,4]| 1 | 2 | 是的 | 7 |
 | 1 | 3 | [2,3,4]| 2 | 4 | 也许取决于 t0 | 10 | 10

 这表明扩大区间会增加总和，但也会增加遍历成本，并且最优解是由最大可行区间而不是纯粹由值密度决定的。 

p 靠近端点的第二个示例演示了不对称性。 

令 n = 4，p = 1，值为 [10, 1, 1, 10]。 从左端开始，先向右走始终是最佳选择，因此只有一个遍历方向很重要。 该算法自然地捕获了这一点，因为成本公式自动选择更便宜的方向。

 ## 边缘情况

 当 p 位于位置 1 时，算法必须正确避免使用左优先遍历，因为不存在左移动。 成本公式仍然表现正确，因为只要 l 等于 p，(p - l) 就为零，并且自然排除不可行的区间。 

当t0非常大时，整个数组变得可行，并且算法正确返回整个数组的和，因为两指针窗口扩展到全范围。 

当所有值都为零时，无论移动如何，结果都为零，并且算法永远不会错误地选择较长的段，因为所有候选者的前缀总和保持为零。
