 ## 解决方案
 ## 问题理解

 我们得到一个正整数序列。 我们可以重复选择序列中的任意两个位置，并使用确定性变换替换该对：一个位置成为该对的 gcd，另一个位置成为该对的 lcm。 该操作可以按任意顺序应用任意次数。 

任务不是模拟这些操作，而是在执行任何此类操作序列后确定所有元素之和的最大可能值。 最终答案必须以 998244353 为模。 

每个测试用例都是独立的，并且在所有测试用例中，元素总数足够大，以至于任何解决方案在总输入大小上都必须接近线性或线性对数。 

关键的结构约束是每个操作都保留两个所选数字的乘积，因为 gcd(a, b) · lcm(a, b) = a · b。 这立即意味着，虽然值可以单独移动和改变，但在整个数组的素因式分解层面上存在严格的守恒定律。 

一种天真的解释表明，gcd 和 lcm 的任意重复混合可能会导致许多可能的状态，但素数指数如何重新分布的限制严重限制了最终配置的实际外观。 

当所有数字都相同时，就会出现微妙的边缘情况。 在这种情况下，每个操作都会使数组保持不变，并且任何假设“我们总是可以改进总和”的解决方案都会错误地尝试无济于事的转换。 另一个边缘情况是当数字成对互质时； 然后 gcd 变为 1，lcm 变为乘积，粗心的局部推理可能会高估其在多个操作中的推进程度。 

## 方法

 强力解释将尝试模拟所有可能的操作序列。 每个操作都会选择一对并用 gcd 和 lcm 替换它们，并且由于我们可以任意重复此操作，因此我们正在有效地探索整数向量的巨大状态空间。 即使对于很小的 n，每一步的分支因子也是 n² 的数量级，并且步数是无限的，因此这种方法立即不可行。 

关键的观察结果是，该运算在素数指数级别上完全是局部的。 如果我们将每个数字写为素数的乘积，那么对于任何固定素数 p，对指数为 x 和 y 的两个数字的运算只需将它们替换为 min(x, y) 和 max(x, y)。 这意味着对于每个素数，独立的指数多重集被保留，仅在位置之间重新分配。 

因此，整个过程分解为独立的排序问题，每个素数一个。 对于每个素数 p，我们获取数组中的所有指数，并且可以在索引之间自由排列它们。 全局结构是每个索引为每个素数接收一个指数，但是这些分配在素数之间必须一致，因为它们都描述了该索引处相同的最终数字。 

下一步是确定如何组合这些独立的重新排序以最大化结果数字的总和。 由于每个元素的值与素数相乘，并且所有素数对指数都有积极且独立的贡献，因此当不同素数的大指数在相同索引上对齐时，总和最大化。 这正是指数空间中应用的重排不等式：我们按降序对每个素数的指数进行排序，并将每个素数的第 k 大指数分配给相同的位置 k。 

这给出了最终数组的单一一致构造，之后我们计算所有值并对它们求和。

## 算法演练

 让输入中的最大值达到 10^6。 我们预先计算最小素因子 (SPF) 筛，以便我们可以有效地分解数字。 

### 1. 构建素数指数桶

 我们处理每个数字并使用 SPF 将其分解。 对于每个素数 p，我们提取该数字中的指数并将其附加到与 p 关联的列表中。 对于每个素数，这给出了一个列表，描述该素数如何在数组中分布。 

这样做的原因是 gcd/lcm 操作永远不会混合不同的素数，因此我们可以安全地通过素数分离整个问题。 

### 2. 按质数对指数列表进行排序

 对于每个素数 p，我们按降序对其指数列表进行排序。 

此步骤反映了这样一个事实：该操作允许对每个素数的指数进行任意排列，因此排序描述了完整的可达空间。 

### 3. 通过对齐构造最终值

 我们迭代从 0 到 n − 1 的索引。对于每个索引 i，我们将 p^(exponent[p][i]) 与所有素数 p 相乘。 

这个对齐步骤是关键的结构选择：索引 i 从每个素数中收集第 i 个最大的指数，确保大的贡献被组合成相同的数字。 

### 4. 求和结果

 我们将所有重建值以 998244353 为模求和。 

### 为什么它有效

 该运算将指数对 (x, y) 转换为 (min(x, y), max(x, y))，这正是对两个值的排序运算。 跨任意对的重复应用允许独立地对每个素数的指数多重集进行完整排序。 

由于每个有效配置对应于为每个素数选择其指数多重集的排列，因此唯一剩下的自由度是这些排列如何在素数之间对齐。 当较大的指数在所有素数的相同索引处重合时，乘积之和最大化，这是从成对交换参数得出的：将较大的指数交换为已经很大的乘积会增加总和，并且此过程收敛到完全对齐。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
MAXV = 10**6

spf = list(range(MAXV + 1))
for i in range(2, int(MAXV**0.5) + 1):
    if spf[i] == i:
        for j in range(i * i, MAXV + 1, i):
            if spf[j] == j:
                spf[j] = i

def factor(x):
    res = {}
    while x > 1:
        p = spf[x]
        cnt = 0
        while x % p == 0:
            x //= p
            cnt += 1
        res[p] = cnt
    return res

t = int(input())
for _ in range(t):
    n = int(input())
    a = list(map(int, input().split()))

    prime_map = {}

    for v in a:
        f = factor(v)
        for p, c in f.items():
            if p not in prime_map:
                prime_map[p] = []
            prime_map[p].append(c)

    # ensure missing exponents are zeros implicitly handled later

    for p in prime_map:
        prime_map[p].sort(reverse=True)

    ans = 0

    # compute each index value
    for i in range(n):
        val = 1
        for p, arr in prime_map.items():
            if i < len(arr):
                val = val * pow(p, arr[i], MOD) % MOD
        ans = (ans + val) % MOD

    print(ans)
```SPF 筛选器构建一次并在测试用例中重复使用，确保因式分解对于总输入大小保持足够快。 

因式分解步骤使用重复除以最小质因数，这保证了每个数字的对数行为。 

字典`prime_map`存储每个素数的指数列表。 由于列表短于​​ n，某些索引处素数缺失的贡献被隐式视为零。 

最终的重建乘以每个索引的素数的模幂。 使用 Python 的内置模幂运算可以保持此步骤的高效。

 ## 边缘情况

 一种边缘情况是所有数字都相同。 每个质数指数列表都有相同的条目，因此排序不会改变任何内容。 重建的数组与输入相同，并且不会发生错误的膨胀。 

另一个边缘情况是当数字成对互质时。 每个数字都贡献不同的质数，因此指数列表的大小大多为 1。排序仍然很简单，并且重建会产生相同的多组值，从而防止任何人为放大。 

最后的边缘情况是素数分布不均匀，其中某些素数出现的数量非常少。 在这种情况下，缺失的索引被视为零指数，这确保了这些素数仅在它们最初存在的地方起作用，并且对齐步骤不会制造额外的值。
