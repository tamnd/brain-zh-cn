 ## 解决方案
 ## 问题理解

 给定平面上的一组点，我们应该将每对点视为由线段连接。 每个这样的线段对应一个“敌人”，可以出现在该线段的任何位置，但其确切位置并不是预先固定的。 

单个动作包括在平面上的任意位置选择一个圆。 任何位于圆圈上或圆圈内的敌人都会被消灭。 由于敌人可能出现在其线段上的任何一点，因此保证消除边缘的唯一方法是确保整个线段安全地位于圆圈内。 即使有一个端点在外面，敌人也可能出现在那里并存活下来。 

因此，每对点都贡献一条边，如果线段的两个端点都在圆内，则该边被视为被圆“覆盖”。 该问题询问，对于每个可能的边数 m，圆的最小半径是多少，以便存在某个中心，该圆覆盖至少 m 个边。 

由于最多有 n ≤ 100 个点，因此边的数量最多为 4950。这立即告诉我们，我们可以进行三次甚至稍差的几何预处理，但任何尝试动态迭代所有圆或进行指数子集搜索的方法都会太慢。 

一个微妙的点是，圆不需要以给定点之一为中心，也不需要穿过任何特定的结构。 这使得几何形状连续，但重要的圆的极值结构仍然由少数点决定。 

一个常见的陷阱是，当只有部分线段位于圆内时，将边缘解释为被覆盖。 这是不正确的，因为敌人可能出现在该路段的任何地方，因此部分覆盖并不能保证。 另一个错误是假设我们只需要考虑以输入点为中心的圆，这是失败的，因为最佳圆通常由两个或三个边界点定义。 

## 方法

 蛮力的想法是考虑每一个可能的圆，并为每个圆计算有多少个线段的两个端点都在其中。 然而，圆构成了一个连续的空间，因此直接枚举它们是不可能的。 

关键的结构观察是，对于固定圆，“两个端点都在内部”的条件仅取决于端点，而不取决于线段内部。 因此，我们只关心圆盘中有多少个点，并且覆盖边缘的数量完全由该计数决定。 如果一个圆盘包含 k 个点，那么它正好覆盖其中的 k(k−1)/2 个边。 

这将问题转化为几何堆积问题：对于每一个 k，我们想要一个可以包含至少 k 个点的圆盘的最小半径。 

现在几何图形变得易于管理。 任何选定的点子集的最小外接圆始终由最多三个边界点定义：直径对或外接圆紧密的三角形。 这意味着某个子集的每个候选“最佳磁盘”都可以从 O(n²) 对和 O(n³) 三元组生成。 

我们枚举所有此类候选圆，计算它们的半径，并计算它们包含多少个点。 然后，我们按半径对圆进行排序，并跟踪迄今为止看到的最大包含点数量。 随着半径的增加，可实现的最佳计数是单调的，因此我们第一次到达 k 个点时给出了该 k 的最小半径。 

最后，我们使用恒等式 m = k(k−1)/2 将“磁盘中的 k 个点”转换为“覆盖的 m 个边”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解圆圈| 无限/棘手| O(1) | O(1) | 太慢了|
 | 枚举候选圈+扫 | O(n3 log n + n3·n) | O(n3 log n + n3·n) | O(n3) | 已接受 |

 ## 算法演练

 ### 步骤1：生成所有候选圈子

我们用两种方式构建圆圈。 首先，对于每对点，我们以该线段为直径形成圆。 其次，对于每个三重非共线点，我们计算穿过它们的唯一外接圆。 这些圆就足够了，因为子集的任何最小外接圆都由最多三个边界点定义。 

### 步骤 2：计算圆半径和覆盖范围

 对于每个圆，我们计算它的圆心和半径。 然后我们计算有多少个输入点位于该圆的内部或边界上。 这给出了该圆的 k 值。 

这样做的原因是，如果一个圆包含 k 个点，它会自动覆盖这些点之间的 k(k−1)/2 条边。 

### 步骤 3：按半径对圆排序

 我们按半径递增的顺序对所有候选圆进行排序。 这使我们能够模拟逐渐增加允许的圆大小。 

### 步骤 4：保持最佳可达到的分数

 我们按排序顺序迭代圆，并维护迄今为止看到的任何圆所覆盖的最大点数。 如果新圆覆盖的点比之前的最佳圆多，我们就会更新它。 

在任何时刻，该值都代表当前半径阈值的最佳可能 k。 

### 步骤 5：将点数转换为边缘要求

 对于每个所需的 m，我们确定最小的 k，使得 k(k−1)/2 ≥ m。 m 的答案是我们的扫描至少达到 k 个点的第一个半径。 

### 为什么它有效

 每个有效的解决方案都对应一个包含某些点子集的磁盘。 对于该子集，存在由至多三个边界点定义的最小外接圆，该外接圆包含在我们的候选集中。 因此，每个可行的 k 点配置都在枚举中表示。 按半径排序可确保我们在任何更大的半径配置之前遇到实现每个 k 的最小可能半径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

EPS = 1e-10

def dist(a, b):
    return math.hypot(a[0] - b[0], a[1] - b[1])

def circle_two(a, b):
    cx = (a[0] + b[0]) / 2
    cy = (a[1] + b[1]) / 2
    r = dist(a, b) / 2
    return cx, cy, r

def circle_three(a, b, c):
    x1, y1 = a
    x2, y2 = b
    x3, y3 = c

    d = 2 * (x1*(y2 - y3) + x2*(y3 - y1) + x3*(y1 - y2))
    if abs(d) < EPS:
        return None

    ux = ((x1*x1 + y1*y1)*(y2 - y3) +
          (x2*x2 + y2*y2)*(y3 - y1) +
          (x3*x3 + y3*y3)*(y1 - y2)) / d

    uy = ((x1*x1 + y1*y1)*(x3 - x2) +
          (x2*x2 + y2*y2)*(x1 - x3) +
          (x3*x3 + y3*y3)*(x2 - x1)) / d

    r = dist((ux, uy), a)
    return ux, uy, r

def count_points(center, r, pts):
    cx, cy = center
    cnt = 0
    rr = r + 1e-9
    for x, y in pts:
        if (x - cx) ** 2 + (y - cy) ** 2 <= rr * rr:
            cnt += 1
    return cnt

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    circles = []

    for i in range(n):
        for j in range(i + 1, n):
            cx, cy, r = circle_two(pts[i], pts[j])
            cnt = count_points((cx, cy), r, pts)
            circles.append((r, cnt))

    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                c = circle_three(pts[i], pts[j], pts[k])
                if c is None:
                    continue
                cx, cy, r = c
                cnt = count_points((cx, cy), r, pts)
                circles.append((r, cnt))

    circles.sort()

    best = 0
    max_k = [0] * (n + 1)

    idx = 0
    for r, cnt in circles:
        best = max(best, cnt)
        if best <= n:
            max_k[best] = r

    for i in range(1, n + 1):
        if max_k[i] == 0:
            max_k[i] = max_k[i - 1]

    total_edges = n * (n - 1) // 2
    k_needed = [0] * (total_edges + 1)

    k = 0
    for m in range(total_edges + 1):
        while k * (k - 1) // 2 < m:
            k += 1
        k_needed[m] = k

    out = []
    for m in range(1, total_edges + 1):
        out.append(str(max_k[k_needed[m]]))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现显式地构造所有直径圆和外接圆，然后评估每个圆捕获了多少个点。 重要的细节是我们在几何过程中不尝试直接跟踪边缘； 我们总是将问题简化为计算磁盘内的点数。 

检查共线性和包含性时，浮点容差由小 epsilon 处理。 这是必要的，因为当点几乎共线时，外接圆计算在数值上很敏感。 

从边缘到所需 k 的映射是单独完成的，以避免在几何扫描期间重复重新计算组合数学。

 ## 边缘情况

 一个关键的边缘情况是三个点几乎共线。 在这种情况下，由于行列式接近于零，外接圆公式在数值上变得不稳定或不确定。 该算法明确地跳过这些三元组并依赖于直径圆，它们仍然正确地代表此类配置的最佳磁盘。 

另一种情况是最佳圆恰好包含两个点。 当三点圈没有改善覆盖范围时就会发生这种情况。 直径结构保证了这些情况仍然被考虑。 

最后，多个圆达到相同半径的情况可以通过首先获取最大覆盖来自然地处理； 关系不会影响正确性，因为只有达到给定 k 的第一个出现才重要。
