 ## 解决方案
 ## 问题理解

 我们得到了几块已经被切成楔形块的圆形煎饼。 每一块都有一定的角度尺寸，我们可以通过进行径向切割来进一步分割任何一块。 一次切割会占用一个现有的角度切片$X$并将其分成两片，其角度之和为$X$，允许任意实分割点。 

目标是服务$D$食客。 每个用餐者必须收到完全相同的一片，并且所有提供的切片必须具有完全相同的角度大小。 我们可以自由选择目标尺寸，但一旦选择，每件服务的作品都必须与其完全匹配。 我们可能会丢弃剩余的碎片。 

操作成本是执行的切割次数。 每次切割都会使碎片数量增加一个。 我们希望最小化获得至少所需的切割总数$D$单一共同尺寸的切片。 

输入大小表明有两种机制。 片数$N$大多数情况下最多为 300，在某些隐藏情况下最多为 10000。 就餐人数$D$最多为 50。这强烈表明解决方案应该迭代候选目标切片结构并有效评估可行性，而不是显式模拟切割。 对所有分割切片的方法进行强力搜索在这两个方面都是指数级的$N$以及切割的次数，远远超出了限制。 

一个微妙的边缘情况是当前切片已包含多个相同大小的片段但数量不够。 例如，如果我们已经有 2 个相等的切片，但需要 3 个用餐者，我们必须决定是进一步分割现有切片之一还是完全重新定位不同的尺寸更好。 另一种边缘情况是，一个大切片可以最佳地分割成许多相等的部分，但这样做所需的切割次数比组合多个中等切片要少。 

核心困难在于，我们没有被要求最大化服务的总面积，而是强制所有服务的块具有相同的尺寸，这通过全局目标值将所有切片耦合起来。 

## 方法

 一种简单的方法是选择目标切片大小并尝试模拟如何从每个煎饼切片中获取它。 对于固定目标尺寸$x$，每个原始切片的大小$A_i$可以分为$\lfloor A_i / x \rfloor$可用的部件，并且需要$\lfloor A_i / x \rfloor - 1$假设我们进行最佳分割，则可以通过切割来产生这些片段。 如果所有切片中可用片段的总和至少为$D$， 然后$x$是可行的。 

困难在于正确的目标$x$尚不清楚。 但是，任何有效的解决方案都必须对应于选择以下形式的目标大小$A_i / k$对于某个整数$k$，因为每个最终片段都来自于均匀细分一些原始切片。 这极大地减少了搜索空间：我们只需要考虑从每个切片派生的候选分割，而不是连续值。 

对于每对$(A_i, k)$， 在哪里$k \in [1, D]$，我们可以治疗$x = A_i / k$作为候选目标尺寸。 对于该候选者，我们计算每个切片可以贡献多少块以及需要多少次切割。 然后我们对所有候选者取最小值。 

关键的改进是认识到最佳解决方案始终在每个原始片段内均匀地对齐切片划分，这意味着除了均匀分区之外，我们永远不需要不规则地部分重用切片。 这将问题从连续优化减少到最多离散枚举$N \cdot D$候选尺寸。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有切割配置进行暴力破解 | 指数| O(1) | O(1) | 太慢了 |
 | 枚举候选尺寸$A_i / k$并评估| O(N^2D) | O(N^2D) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们评估从现有煎饼中得出的所有可能的目标切片尺寸。 对于每个候选者，我们计算可以形成多少个相等的块以及切割成本。 

1. 对于每一片$A_i$，我们考虑每一个可能的件数$k$从 1 到$D$。 我们将此视为建议最终切片大小为$x = A_i / k$。 这已经足够了，因为任何最佳解决方案都必须将至少一个原始切片划分为与最终大小匹配的相等片段。 
2. 对于每个候选值$x$，我们计算可以从每个原始切片中获得多少个切片。 一片大小$A_j$贡献$\lfloor A_j / x \rfloor$可用的件。 每次我们将切片分成$m$零件，我们需要$m - 1$削减。 
3. 我们累加所有切片中可用块的总数。 如果这个总数小于$D$，该候选者无效并被丢弃。 
4. 如果有效，我们通过对所有切片值求和来计算所需的切割总数$\lfloor A_j / x \rfloor - 1$对于至少贡献一块的切片。 
5. 我们对所有有效候选者进行最小削减计数。 

仅评估这些候选者背后的原因是，在任何最佳解决方案中，至少一个原始切片被划分为与最终选择的大小完全匹配的相等片段，并且该切片确定整个配置的粒度。 

### 为什么它有效

 确定最终工件尺寸的最佳解决方案$x$。 考虑至少贡献一个最终作品的任何原始切片。 该切片必须分为整数个大小相等的段$x$， 所以$x = A_i / k$对于某个整数$k$。 因此，每个最佳解决方案都代表在我们枚举的候选集中。 由于我们明确评估每种此类配置的成本，因此我们不会错过最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        N, D = map(int, input().split())
        A = list(map(int, input().split()))

        INF = 10**30
        ans = INF

        for i in range(N):
            for k in range(1, D + 1):
                x = A[i] / k
                total_pieces = 0
                cuts = 0

                for a in A:
                    m = int(a // x)
                    if m > 0:
                        total_pieces += m
                        cuts += m - 1

                if total_pieces >= D:
                    ans = min(ans, cuts)

        print(f"Case #{tc}: {ans}")

if __name__ == "__main__":
    solve()
```实现直接遵循候选枚举思想。 我们测试每个可能的目标切片大小，该大小是通过将每个输入切片除以一个整数而得出的$D$。 对于每个这样的目标，我们模拟每个切片可以使用整数除法产生多少块。 

内循环既考虑可行性又考虑成本。 可行性至少需要积累$D$件。 切割计数来自将切片分割成的观察$m$同等的部分总是要付出代价$m - 1$无论剪切顺序如何，都会进行剪切。 

一个微妙的点是浮点除法$x$。 在更仔细的实现中，这将被有理算术取代以避免精度问题，但这里的约束和整数结构通常保持比较稳定。 生产级解决方案将使用整数算术或存储进行缩放和比较$k / A_i$比率。 

## 工作示例

 我们追踪两个案例来了解候选人选择的行为。 

### 示例 1

 输入：```
N = 3, D = 2
A = [10, 5, 3]
```我们尝试每个切片的候选大小。 

| 我| k | x = A[i]/k | 总件数 | 削减 | 有效 |
 | ---| ---| ---| ---| ---| ---|
 | 0 | 1 | 10 | 10 1 | 0 | 没有|
 | 0 | 2 | 5 | 2 | 1 | 是的 |
 | 1 | 1 | 5 | 2 | 1 | 是的 |
 | 2 | 1 | 3 | 1 | 0 | 没有|

 最佳有效配置使用尺寸 5，总共产生 2 件并进行 1 次切割。 这符合我们只需要分割一个切片一次的想法。 

### 示例 2

 输入：```
N = 2, D = 3
A = [8, 4]
```我们评估候选人。 

| 我| k | x| 8 件起 | 4 件 | 总计 | 削减 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 0 | 2 | 4 | 2 | 1 | 3 | 2 |

 该候选是有效的，并且通过 2 次切割恰好产生 3 个尺寸为 4 的块，这是最佳的，因为将 8 分成四个 2 或混合尺寸将需要更多切割。 

这些痕迹表明，最优值是通过枚举切片大小的除数自然产生的。

 ## 边缘情况

 一种常见的失败模式是假设贪婪地获取尽可能大的碎片总是有效的。 例如，用切片`[8, 4]`和`D = 3`，贪心方法可能会将 8 视为 4+4 并忽略 4，但最佳解决方案需要恰好三个相等的 4 大小的块，与两个切片完美对齐。 该算法正确地捕获了这一点，因为它评估候选大小 4 并计算总贡献。 

另一个边缘情况是多种候选尺寸产生相同数量的零件但切割成本不同。 例如，根据所选目标，12 的切片可以分为 3+3+3+3 或 4+4+4。 两者都能生产足够的碎片，但切割数量不同。 枚举确保两者都被评估，因为两者对应于不同的$k$值在$A_i / k$，并选择最小值。
