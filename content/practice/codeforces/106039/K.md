 ## 解决方案
 ## 问题理解

 我们得到了一套原料和安东尼奥拒绝使用的原料清单。 该配方分为几个阶段，每个阶段指定该步骤所需的成分子集。 

当安东尼奥准备舞台时，他只是省略了所有他不喜欢的成分。 这可能会减少该阶段实际使用的成分数量。 如果该阶段中缺失成分的比例超过该阶段原始成分数量的三分之一，则该阶段将变得不可接受。 

任务是找到发生这种情况的最早阶段，即缺失成分数量严格大于阶段大小三分之一的第一个阶段。 如果没有阶段违反此条件，我们输出减一。 

这些限制意味着所有阶段提及的成分总数最多为 300,000 次。 这立即排除了任何以二次方式重复扫描每个阶段的大型结构的解决方案。 每阶段对其自身成分进行线性扫描是可以接受的，因为每种成分在每个阶段仅出现一次，并且跨阶段的总大小是有限的。 

当舞台尺寸较小时，会出现微妙的边缘情况。 如果某个阶段的大小为 1，那么缺少一种成分就已经使该阶段无效，因为 1 大于 1/3。 同样，对于尺寸 2，缺少至少一种成分就足够了，因为 1 > 2/3 是错误的，但 2 > 2/3 成立。 因此在实践中，阈值对整数除法很敏感，必须以严格的不等式处理而不是四舍五入。 

另一个边缘情况是在任何阶段都没有出现不喜欢的成分。 正确的输出是负一，并且错误地初始化答案或忘记检查所有阶段的幼稚实现可能会意外返回零或未初始化的值。 

## 方法

 解决问题的直接方法是独立模拟每个阶段。 对于给定的阶段，我们扫描其所有成分并计算有多少属于安东尼奥不喜欢的集合。 之后，我们将此计数与舞台大小的三分之一进行比较。 如果严格大于该值，则该阶段无效。 

这种蛮力方法已经非常接近最优，因为每种成分每个阶段都会处理一次，并且所有阶段大小的总和最多为 300,000。 唯一真正关心的是我们如何检查某种成分是否不受欢迎。 如果我们将不喜欢的成分存储在哈希集中，则成员资格检查平均为 O(1)，从而使整个解决方案与总输入大小呈线性关系。 

关键的观察结果是阶段之间不存在相互依赖。 每个阶段都可以仅使用与不喜欢的集合的重叠计数来独立评估。 除了构建集合之外，这消除了对前缀结构、排序或预处理的任何需要。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每阶段扫描的暴力破解 | O(总成分) | O(B) | 已接受 |
 | 哈希集成员计数| O(总成分) | O(B) | 已接受 |

 在实践中，两种描述都会分解为相同的实现，因为输入的结构已经限制了总工作量。 

## 算法演练

 ### 最优算法

 1. 读取不喜欢的成分集并将它们存储在哈希集中。 这允许持续时间检查安东尼奥是否忽略了任何成分。 
2. 按顺序迭代每个阶段。 对于每个阶段，请阅读其成分列表。 
3. 对于当前阶段，计算有多少个成分出现在不喜欢的集合中。 该计数代表实际中缺少的成分数量。 
4. 检查该计数是否严格大于阶段大小的三分之一。 不使用浮点除法，而是比较`3 * missing > M`，这避免了精度问题并将所有内容保持为整数。 
5. 如果条件成立，立即返回当前阶段索引作为答案。 
6. 如果处理完所有阶段后没有阶段满足条件，则返回减一。 

### 为什么它有效

 每个阶段仅根据阶段的成分集和不喜欢的集之间的交集大小进行独立评估。 该情况仅取决于缺失成分的数量，而不取决于它们的身份或位置。 由于我们精确地计算每个阶段的计数，并且在没有近似的情况下检查不等式，因此以正确的顺序找到第一个违规阶段。 任何阶段都不会被跳过或受到其他阶段的影响，因此始终能够正确检测到最早的故障。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, A, B = map(int, input().split())
    disliked = set(map(int, input().split()))

    for i in range(1, N + 1):
        data = list(map(int, input().split()))
        M = data[0]
        arr = data[1:]

        missing = 0
        for x in arr:
            if x in disliked:
                missing += 1

        if 3 * missing > M:
            print(i)
            return

    print(-1)

if __name__ == "__main__":
    solve()
```该解决方案首先将所有不喜欢的成分加载到哈希集中，这是唯一需要的预处理。 然后按顺序处理每个阶段。 线路`3 * missing > M`是在没有浮点运算的情况下编码“超过三分之一”的关键比较。 

成分循环是安全的，因为所有阶段的成分条目总数以 300,000 为界，因此即使在最坏的情况下，内部循环也总共运行 300,000 次迭代。

 ## 边缘情况

 小的阶段尺寸可以使条件快速触发。 例如，如果一个阶段有一种成分并且不受欢迎，那么`missing = 1`和`3 * 1 > 1`立即成立，在第一阶段产生拒绝。 该算法自然地处理这个问题，因为无论大小如何，都会使用相同的整数不等式。 

当缺少的成分正好等于舞台的三分之一时，舞台仍然必须被接受。 例如，尺寸为 3 且缺少 1 个成分的阶段给出`3 * 1 = 3`，它并不严格大于 3。该实现通过使用严格的不等式正确地保留了此边界条件。 

如果没有阶段包含任何不喜欢的成分，则每个阶段`missing`为零，因此不等式永远不成立，并且算法在处理所有阶段后正确输出减一。
