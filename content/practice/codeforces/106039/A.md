 ## 解决方案
 ## 问题理解

 我们给定一个长度序列$2N$，其中每个值来自$1$到$N$正好出现两次。 您可以将其视为沿直线放置的成对相同符号。 

任务是在严格的移动规则下选择一对相等的符号。 如果您决定将某个位置的出现配对$i$，您必须立即将其与其在某个位置的匹配出现配对$j > i$。 完成此操作后，您将继续从该点向前扫描，并且永远无法返回。 每个选定的对贡献的成本等于其端点之间的距离，$j - i$。 

目标有两个层次。 首先，您希望在尊重仅前向过程的同时最大化可以形成的此类对的数量。 其次，在实现最大对数的所有策略中，您希望最小化总成本。 

约束的关键含义是$N \le 10^5$，所以数组长度可达$2 \cdot 10^5$。 任何尝试模拟选择或组合搜索配对的解决方案都会失败。 甚至$O(N^2)$推理已经太大了，甚至$O(N \log N)$解决方案必须精心构建。 

一个微妙的问题是贪婪配对并不是那么安全。 将符号与其第一个可能的匹配配对可能会阻止更好的全局结构。 例如，在类似的序列中$1, 2, 1, 2$，配对第一个$1$与第二个$1$这是不可能的，所以你被迫进行交叉交互，并且排序约束很重要。 另一个微妙的情况是，最佳配对需要跳过早期的机会，因为它们会导致未来更长的联系。 

更深层次的困难是我们不是简单地独立匹配区间。 每对消耗数组的一段，并且一对的选择会影响哪些符号在前向扫描中仍然可以匹配。 

## 方法

 强力解释是将过程模拟为对形成非重叠前向对的所有有效方法的搜索。 在每个位置，我们要么跳过它，要么尝试将其与右侧某处的匹配出现配对，然后从那里递归。 这可以正确地探索所有有效序列，并可以在累积成本的同时计算可能有多少对。 

问题是分支因子很大。 对于每个位置，可能存在多个有效的匹配伙伴，并且所有对组合的递归迅速爆炸。 在最坏的情况下，这会呈指数增长$N$，因为每对决策都以非局部方式限制未来的可能性。 

结构洞察力是我们实际上不需要在任意配对中进行选择。 每个值恰好出现两次，因此每个元素定义其两次出现之间的固定间隔。 唯一真正的决定是给定的间隔是否可以被视为有效前向链的一部分，而不违反先前选择所暗示的排序约束。 

一旦我们将每个数字视为一个区间$(l_i, r_i)$，问题就变成了在单调扫描过程中选择最大数量的间隔，其中我们必须按照左端点的升序处理间隔，并确保所选间隔遵循非递减边界。 

第二个关键思想是最大化线对数量与成本无关。 由于在有效的前向构造中每个数字都必须配对或不配对，因此最大配对数量取决于在贪婪前向扫描中可以链接多少个间隔而不会重叠。 一旦确定了最大集合，最小化成本就会减少为始终以最小化总距离的方式使用最近的可能有效匹配。 

这自然会导致基于堆栈的解释：当我们扫描数组时，我们会尽可能匹配，但我们确保始终与最近的可用空出现配对，因为延迟配对只会增加距离。

 ## 算法演练

 我们将序列视为流，并使用堆栈维护当前不匹配的事件集。 每次我们看到一个交易品种时，我们都会决定它是否关闭之前打开的交易品种。 

1. 初始化一个空堆栈和计数器，用于记录对数和总成本。 
2. 从左到右迭代数组。 

当我们遇到一个值时$x$，我们检查是否$x$已经在堆栈顶部。 如果不是，我们将这次事件视为一个空缺并将其推入堆栈。 

这是有效的，因为此时我们还不知道这种情况的最佳配对在哪里，任何过早的配对决策都会违反前向约束。 
3. 如果当前值与堆栈顶部匹配，则我们已找到同一交易品种最近不匹配的开盘的正确收盘价。 

我们把它弹出并形成一对。 距离贡献是当前索引减去所存储的开头出现的索引。 
4. 每当形成一对时，增加对计数并将距离添加到总成本中。 

这种贪婪闭包是正确的，因为将一个元素与后来出现的不是最接近的可用匹配的元素配对只会延迟闭包并严格增加成本，而不会提高可行性。 
5.处理完全数组后，输出形成的对的数量和累计成本。 

### 为什么它有效

 在任何时候，堆栈都代表一组活动的、不匹配的符号，按照它们打开的顺序排列。 该算法强制我们始终首先关闭最近打开的兼容符号。 这可以防止交错，而交错会在以后强制建立更长的连接。 

不变的是，对于当前堆栈中的每个符号，其匹配伙伴尚未以允许更紧密配对而不违反顺序的方式传递。 因为每个符号恰好出现两次，所以我们第一次看到它时打开一个区间，第二次看到它时必须关闭它。 堆栈确保以后进先出的方式解析嵌套结构，这正是避免在仅向前遍历中交叉依赖的唯一方法。 

任何偏离此策略的行为都会跳过有效的立即匹配或强制在更远的距离进行稍后的匹配，从而在不增加配对数量的情况下增加成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    stack = []
    pos = {}
    pairs = 0
    cost = 0

    for i, x in enumerate(a):
        if stack and stack[-1] == x:
            j = stack.pop()
            cost += i - j
            pairs += 1
        else:
            stack.append(i)

    print(pairs, cost)

if __name__ == "__main__":
    solve()
```该实现扫描数组一次并在堆栈中维护不匹配空位的索引。 关键的设计选择是存储索引而不是值，因为成本取决于位置。 每次匹配时，我们都会立即计算准确的距离，确保不需要以后进行修改。 

条件`stack[-1] == x`确保我们仅在符号与最近的不匹配出现匹配时关闭符号，从而保留最佳配对所需的嵌套结构。

 ## 边缘情况

 像这样的最小情况$1, 1$确认算法正确处理立即关闭。 堆栈接收第一个元素，第二个元素立即匹配它，生成成本为 1 的一对。 

完全嵌套的序列，例如$1, 2, 3, 3, 2, 1$证明了 LIFO 行为的重要性。 当我们打开新符号并以相反顺序解析时，堆栈会增长，只产生一对有效的对，因为较早的打开会阻止独立配对。 

交叉图案，例如$1, 2, 1, 2$表明并非所有出现都形成独立的区间。 该堆栈确保仅形成一个配对，并且成本反映了强制长连接，从而确认过早的配对尝试将违反排序约束。
