 ## 解决方案
 ## 问题理解

 我们得到了固定的夜晚数 D，以及一个人可以睡觉的 N 个可能的地方的集合。 每个地方 i 都有一个约束 di ，限制可以在该地方连续度过多少个夜晚。 如果一个人在同一个地方连续停留超过两天，他们就会被立即驱逐，所以任何有效的计划都必须在超过限制之前打破长期连续。 

任务是构造长度为 D 的任意序列，其中每个元素都是从 1 到 N 的索引，描述该人每晚睡觉的地方。 该序列必须遵守以下规则：索引 i 不会出现在比 di 长的连续块中。 如果 di 为零，则该地方根本无法使用，因为即使是一个晚上也已经违反了约束。 

约束 N、D ≤ 100000 意味着任何解决方案都必须至多为 O(D log D) 或 O(D)。 D 中的任何二次方或涉及每个位置重复全扫描的操作都会失败。 在 D 很大且有效配置很紧的最坏情况下，尝试在违规发生后修复违规的幼稚回溯或贪婪模拟会太慢。 

当所有可用的 di 值都很小但 D 很大时，就会出现微妙的故障情况。 例如，如果所有di都是1，那么没有一个地方可以连续使用两次，所以我们必须完美交替。 如果 N = 1 且 D = 2 且 d1 = 1，则这是不可能的，但如果 N = 2，则存在解。 这表明可行性不仅取决于总容量，还取决于我们是否总能在达到极限之前“切换出去”。 

当所有 di 为零时，会出现另一种边缘情况。 那么每个地方都是禁止的，连D=1都不可能。 忽略 di = 0 的贪婪方法会错误地尝试使用这些索引。 

## 方法

 一个蛮力的想法是每天构建序列，并每天尝试所有可能的位置，检查附加该位置是否违反了连续约束。 如果确实如此，我们会尝试另一个地方。 我们维护当前位置的当前条纹长度，并在扩展序列时更新它。 

这在逻辑上是有效的，因为我们只接受尊重约束的序列，但问题是在最坏的情况下我们可能每天尝试许多替代方案。 如果我们遇到当前位置已耗尽并且必须切换的情况，我们可能会重复扫描所有 N 个候选，从而导致 O(DN) 行为，在最坏的情况下是 10^10 次操作。 

关键的观察结果是该约束仅取决于连续使用。 一旦我们选择了一个地方，我们最多可以连续呆几天，但没有什么强迫我们提前离开。 这表明我们应该始终使用一个地方来获得最大的安全连续性，然后再切换。 

我们可以将每个地方视为一个“资源”，可以发出长度最大为 di 的块。 我们想要将这些块拼接成一个总长度为 D 的序列。唯一的困难是确保当前块结束时我们始终拥有有效的下一个块。 

一个自然的贪婪策略是当我们需要开始或重新启动一个段时，总是选择具有最大剩余允许条纹的地方。 如果我们总是选择当前最强的选项，我们就可以避免陷入无法填补剩余时间的弱选项中。 

为了有效地支持这一点，我们维护一个由 di 键控的最大堆。 我们重复使用最好的可用位置，使用它长达 min(di,remaining_days)，然后如果它仍然有剩余容量，则将其推回（或者简单地重用，因为 di 不会全局减少，只有连续使用才重要；相反，我们通过重用同一个池来显式管理条纹）。

关键的结构是我们只关心防止连续运行时间过长，因此每当我们离开一个地方时，我们都会立即重置其资格。 这将问题变成重复选择与上次使用的位置不同的位置，同时仍然最大化允许的条纹长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(DN) | O(N) | 太慢了|
 | 最优（贪婪+堆）| O(D log N) | O(D log N) | O(N) | 已接受 |

 ## 算法演练

 我们维护一个可用位置的优先级队列，按 di 降序排列。 我们还会跟踪最后选择的地点以及我们连续使用该地点的次数。 

1. 将所有索引 i 插入到由 di 键控的最大堆中，使得 di > 0。 这些代表可用的地方； di = 0 条目将被完全忽略，因为它们永远不会出现在有效的序列中。 
2. 初始化一个空结果列表，变量last_place = -1，streak = 0。 
3. 每天从 1 点到 D 点，我们选择下一个睡觉的地方。 我们首先尝试从堆中选择最好的可用位置。 如果该地点与last_place相同并且使用它会超出其允许的连续限制，我们会暂时跳过它并采用下一个最佳候选者。 

跳过的原因是堆不编码“当前连续状态”，因此我们必须手动强制执行该约束。 

1. 一旦我们选择了候选地点 x，我们将其附加到答案中并更新 last_place。 如果 x 等于last_place，我们就增加 streak，否则我们将 streak 重置为 1。 
2. 使用x后，只有当它对将来的选择仍然有用时，我们才将其重新插入堆中。 由于 di 是最大连续配额而不是可消耗资源，因此它在堆中保持不变，但我们通过仅通过条纹跟踪阻止过度使用来确保正确性。 
3. 如果在任何时候我们都找不到与last_place不同的有效候选，而streak已经到达d[last_place]，我们就得出结论，构造是不可能的。 

为什么这种选择策略是正确的，因为每当我们被迫切换时，任何有效的解决方案也必须在此时切换。 选择最大的可用 di 可确保我们最大限度地提高未来细分市场的灵活性。 

### 为什么它有效

 在每天的边界上，唯一可能破坏可行性的约束是被迫将运行时间延长到超出其 di 限制。 该算法保持当前游程长度永远不会超过其所在位置的 di 的不变性，并且每当需要切换时，任何有效的解决方案也必须在该点或之前执行切换。 由于我们总是选择最宽松的可用下一个区块，因此我们永远不会过度降低未来的可行性，并且我们永远不会以某种方式消耗一个位置，从而使其比所需的时间更早无法使用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

def solve():
    n, d = map(int, input().split())
    arr = list(map(int, input().split()))
    
    heap = []
    for i, v in enumerate(arr):
        if v > 0:
            heapq.heappush(heap, (-v, i + 1))
    
    if not heap:
        print(-1)
        return
    
    res = []
    last = -1
    streak = 0
    last_limit = 0
    
    for _ in range(d):
        if not heap:
            print(-1)
            return
        
        v1, i1 = heapq.heappop(heap)
        v1 = -v1
        
        if i1 == last and streak == v1:
            if not heap:
                print(-1)
                return
            v2, i2 = heapq.heappop(heap)
            v2 = -v2
            heapq.heappush(heap, (-v1, i1))
            
            chosen = i2
            if i2 == last:
                streak += 1
            else:
                streak = 1
            last = i2
            res.append(chosen)
        else:
            chosen = i1
            heapq.heappush(heap, (-v1, i1))
            
            if i1 == last:
                streak += 1
            else:
                streak = 1
            last = i1
            res.append(chosen)
    
    print(*res)

if __name__ == "__main__":
    solve()
```该实现通过 di 保留最大堆位置。 每一天，它都会尝试使用最强大的可用位置。 如果该位置由于当前的连胜而违反了连续限制，它会暂时跳过它并使用第二好的候选者。 

关键的微妙之处在于我们不会从堆中永久删除元素。 每个位置仍然可用，但正确性由连胜逻辑强制执行。 交换机制确保当最佳候选位置与上次使用的位置相同但已经饱和时，我们永远不会过早陷入困境。 

必须小心准确地追踪连胜； 差一错误很常见，因为 streak 会计算当前运行（包括当天）。

 ## 边缘情况

 所有 di 都为零的情况会立即通过空堆检查进行处理。 由于没有插入任何位置，因此算法在尝试任何构造之前会打印 -1。 

当只有一个有效位置 di = 1 但 D > 1 时，堆总是返回相同的元素，并且条纹条件立即阻止进一步使用。 第二堆回退不存在，因此算法正确地以 -1 终止。 

当多个位置的 di = 1 相等时，堆始终提供交替候选对象。 条纹逻辑确保即使由于堆排序相同的索引再次出现在顶部，它也会被跳过以支持另一个可用索引，从而产生有效的交替序列。
