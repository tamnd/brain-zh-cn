 ## 解决方案
 ## 问题理解

 我们有两年的起始年，一年是魏大师，一年是凯。 从那些年开始，每个人积累的“经验”等于他们开始编程以来已经过去的年数。 所以在任何一年$Y$，魏先生的经验是$Y - W$，而 Kai 的经验是$Y - K$。 

我们被要求找到一年$Y$这样，Wei 的经验正好是 Kai 经验的两倍。 用代数术语来说，我们正在寻找满足以下条件的唯一整数年：$$Y - W = 2 \cdot (Y - K)$$限制很大，长达数年$10^9$，但结构是纯线性的。 这立即表明我们应该避免随着时间的推移进行模拟，因为扫描从$W$在这个规模上向上是完全不可行的。 

一个微妙的问题是，方程可能会产生非整数解或早于 Kai 开始的一年。 由于两个经验值必须对应于实际经过的年份，因此解决方案还必须尊重$Y \ge K$，否则 Kai 的经验在预期的解释中将是负面的或未定义的。 

不存在隐藏的分支情况：一旦我们正确设置了方程，要么只有一个有效年份，要么在约束范围内没有有效年份。 天真的推理的主要风险是试图“模拟未来几年”并逐年检查情况，但在时限内会失败。 

## 方法

 暴力方法将从年开始$K$逐年递增，计算每一步Wei和Kai的经验并检查这种关系是否成立。 这在逻辑上是正确的，因为条件是直接根据经验的定义来评估的，如果存在的话，最终会达到正确的年份。 

然而，最坏的情况迫使搜索空间跨越$10^9$迭代。 每次迭代都会做恒定的工作，因此总复杂度变为$O(10^9)$，这远远超出了一秒的限制所能处理的范围。 

关键的观察结果是条件是单变量的线性方程。 我们不迭代可能的年份，而是直接用代数方法求解年份。 展开方程：$$Y - W = 2Y - 2K$$重新排列术语：$$- W + 2K = Y$$所以答案很简单：$$Y = 2K - W$$这将问题从大范围的搜索减少到单个算术计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(K - W)$|$O(1)$| 太慢了|
 | 最佳 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1.读取两个整数$W$和$K$，代表魏、凯的起始年月。 这些根据未知的目标年份定义了两个线性经验函数。 
2. 使用经验的定义将条件转化为方程。 魏先生的经历是$Y - W$，凯的是$Y - K$，我们强制执行平等$Y - W = 2(Y - K)$。 这一步至关重要，因为它将叙述条件转换为代数。 
3. 展开方程并分离$Y$。 这会产生$Y - W = 2Y - 2K$，并重新排列产量$Y = 2K - W$。 这是唯一可能满足条件的候选年。 
4. 直接输出计算值作为答案。 

### 为什么它有效

 两个经验函数在同一变量中都是线性的$Y$。 该条件使两个线性表达式相等，这保证最多有一个解。 一旦我们导出了方程，我们就不会做出任何近似或启发式选择，而只是进行精确的代数变换。 由于转换是可逆的，任何有效的解决方案都必须满足派生表达式，并且表达式本身总是产生唯一的候选年份。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    W, K = map(int, input().split())
    print(2 * K - W)

if __name__ == "__main__":
    solve()
```程序读取两个输入值并直接应用导出公式。 在 Python 的任意精度整数下，乘法是安全的，因此即使中间值达到约 100 左右，也不存在溢出问题$2 \cdot 10^9$。 

关键的实现细节是确保表达式中的操作顺序正确$2K - W$。 写作$2 * (K - W)$是不正确的，因为它编码了不同的代数关系。

 ## 边缘情况

 一个重要的边缘情况是两个程序员在同一年开始工作。 例如，$W = K = 5$。 公式给出$Y = 2K - W = 5$。 换回来，魏的经验是$5 - 5 = 0$，凯的也是$0$，所以等式成立。 

另一种情况是当$W = 0$和$K = 1$。 计算出的年份是$Y = 2 \cdot 1 - 0 = 2$。 在$Y = 2$，Wei有2年经验，Kai有1年经验，满足翻倍条件。 

边界场景是两个值都最大时，例如$W = K = 10^9$。 结果是$Y = 10^9$，它保持在界限内并保持正确性，因为此时两个经验值都保持为零。
