 ## 解决方案
 ## 问题理解

 给定一个整数序列，并且允许我们选择一个模值$M$和$1 < M \le 10^9$。 一次$M$是固定的，每次“移动”都包含选择一个余数$x$，并在该移动中我们删除所有其值模数的数字$M$等于$x$。 换句话说，一步操作会删除模数下的整个残基类$M$。 

对于固定的$M$，所需的移动次数恰好是所有残基中不同残基的数量$A_i \bmod M$。 任务是最小化所有有效的这个数字$M$，然后计算有多少个不同的值$M$达到这个最小值。 

关于约束的关键观察是$N \le 10^5$，而值上升到$10^9$。 这立即排除了尝试所有的任何事情$M$显式地或计算所有对的残基。 解决方案必须避免迭代$M$直接工作，而不是根据数组本身的结构属性工作，很可能基于元素或因式分解结构之间的差异。 

一种幼稚的方法会考虑每个$M$，计算所有残差，并计算不同的残差。 即使我们只尝试了所有相关的$M \le 10^9$，那是不可能的。 甚至限制为$O(N)$候选人$M$，重新计算残留成本$O(N^2)$，仍然太大。 

第二个微妙的陷阱是假设最好的$M$总是大或总是小。 例如，如果所有数字都相等$[5,5,5]$， 任何$M$产生一个残基类别，因此答案取决于有效计数$M$，而不是最小化任何不平凡的事情。 另一方面，对于像这样的数组$[1,2,3]$，选择$M=2$给出残基$\{1,0,1\}$， 尽管$M=3$给出$\{1,2,0\}$，并且行为在很大程度上取决于算术结构而不是大小$M$。 

## 方法

 对于固定模数$M$，该过程根据$A_i \bmod M$。 成本是组的数量。 最小化组的数量意味着我们希望残基中发生许多碰撞，当元素之间的差异可以整除时就会发生这种情况$M$。 

如果两个元素$a$和$b$落入相同的残基类模$M$， 然后$a \equiv b \pmod M$， 意义$M \mid (a-b)$。 这将问题从模块化分组转换为对成对差异的整除约束。 

蛮力视角会列举$M$，计算所有残差，并计算不同值。 这是正确的但不可行，因为对于每个$M$，我们花费$O(N)$，导致$O(N \cdot 10^9)$或者至少$O(N^2)$如果使用受限候选人。 

关键的结构转变是停止考虑残基，而是考虑两个元素何时合并到同一类中。 模数$M$根据它如何分割多重差异集来精确地减少类的数量。 最优解对应于最大化碰撞，相当于选择$M$以结构化的方式划分尽可能多的差异。 

这导致考虑相对于固定参考（通常是最小元素）的差异，并分析所有差异的 gcd 结构。 一旦我们用基点来表达所有内容，产生最小分区的有效模数就对应于该 gcd 的除数，并且对此类模数进行计数就减少到在约束下对除数进行计数。 

因此问题就变成了：计算一组规范的差异，通过 gcd 减少它，然后分析该 gcd 的哪些除数产生相同的分区大小。 最小移动次数取决于当所有数字按模分组时剩余多少个不同的等价类$M$，以及最优的数量$M$是从差分 gcd 的除数结构导出的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解 M |$O(N \cdot 10^9)$|$O(N)$| 太慢了 |
 | GCD+除数分析 |$O(N \log A)$|$O(N)$| 已接受 |

 ## 算法演练

 我们重写相对于最小元素的所有值，以便结构仅取决于差异。 让$mn = \min(A)$，并定义$B_i = A_i - mn$。 现在，每个有效的模量行为仅取决于集合$\{B_i\}$，至少有一个零。 

我们计算所有非零的 gcd$B_i$，称之为$g$。 该值捕获数组中所有常见的周期性结构。 

接下来，我们推断可以强制合并多少个不同的残基。 当我们最大化碰撞时，就会实现尽可能少的移动次数，当尽可能多的碰撞发生时，就会发生这种情况$B_i$尽可能变得全等模$M$。 自从$B_i \equiv 0 \pmod M$每当$M \mid B_i$，除以许多差异的模数将许多值折叠为单个残差类。 

在极端情况下，如果$M \mid g$，那么所有$B_i$是的倍数$g$最大限度地崩溃。 残差类别的最小数量由多少个不同值保持模数决定，例如$M$，这对所有人来说都是稳定的$M$划分$g$。 

因此，最小轮数对应于选择任何$M$这是一个除数$g$，并且最佳计数仅取决于有多少个除数产生相同的分区结构。 由于所有有效的最优模在诱导等价类方面表现相同，因此答案简化为计算$g$条件下$M > 1$。 

最后一步是枚举除数$g$通过迭代有效地$\sqrt{g}$。 

## 为什么它有效

 所有分组仅取决于值对是否模全等$M$。 该条件相当于要求$M$划分他们的分歧。 所有差异的最大公约数是同时保留所有这些关系的最大整数。 任何不与此 gcd 对齐的模数都会破坏至少一个最大对齐，并且无法将分区结构改进到最佳状态之外。 因此，所有最佳模数都是差值的 gcd 的除数，并且没有其他模数可以实现相同的最小分区大小。 

## Python 解决方案```python

input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    mn = min(a)
    diffs = []
    
    g = 0
    for x in a:
        g = __import__("math").gcd(g, x - mn)
    
    if g == 0:
        # all equal
        # any M works, but M>1 so infinite range is not asked; answer is 1 move, all M valid conceptually
        print(1, 10**9 - 1)
        return
    
    import math
    
    def count_divisors(x):
        res = 0
        i = 1
        while i * i <= x:
            if x % i == 0:
                if i > 1:
                    res += 1
                if i != x // i and x // i > 1:
                    res += 1
            i += 1
        return res
    
    ans = count_divisors(g)

```该代码首先使用最小元素对数组进行规范化，以便所有结构都通过差异来表达。 然后计算所有差异的 gcd。 如果 gcd 为零，则所有元素都相同，并且每个有效元素$M$产生单个残基类，因此移动次数为一。 

否则，我们计算除数的数量$g$大于 1，因为$M$必须满足$M > 1$。 每个这样的除数对应于一个最佳模数。 

除数计数循环通过检查小心地避免重复计算平方根$i \neq x // i$，并且它排除了值 1，因为它是不允许的。 

## 工作示例

 考虑一个结构简单的数组：$A = [5, 11, 17]$。 

我们通过减去 5 进行归一化，得到$B = [0, 6, 12]$。 最大公约数为6。 

| 步骤| 价值| GCCD |
 | ---| ---| ---|
 | 开始 | 0 | 0 |
 | +6 | 6 | 6 |
 | +12 | 12 | 12 6 |

 大于 1 的 6 的除数为 2、3、6，因此答案计数为 3，最小移动次数为 1，因为所有值都可以在任何除数 6 下归并为一个残差类别。 

这表明，当所有差异共享强大的 gcd 结构时，数组的行为就像单个周期簇。 

现在考虑$A = [1, 2, 4]$。 

我们标准化：$B = [0, 1, 3]$，gcd为1。 

| 步骤| 价值| GCCD |
 | ---| ---| ---|
 | 开始 | 0 | 0 |
 | +1 | 1 | 1 |
 | +3 | 3 | 1 |

 由于 gcd 为 1，因此不存在大于 1 的有效除数，因此只剩下微不足道的行为，并且答案也会相应减少。 

这种情况表明，当数组没有共享结构时，大于 1 的模数不会产生额外的折叠。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|

 该解决方案很容易满足约束条件，因为$N \le 10^5$并且gcd运算很快，而除数枚举最多$O(\sqrt{10^9})$在最坏的情况下，但实际上通常要小得多。

 ## 边缘情况

 当所有值都相同时，每个模数都会产生相同的单个残基类别。 差异的 gcd 变为零，并且算法明确地将其视为特殊情况，返回单个移动并计算所有有效模数。 

当差异的 gcd 为 1 时，没有大于 1 的模与所有差异对齐。 除数枚举正确地产生零，符合不可能进行非平凡压缩的事实。 

当数组形成完美的算术级数时，gcd 等于步长，并且该步长的所有除数都成为有效的最优模。 该算法通过 gcd 计算和除数计数直接捕获这一点，无需检查成对残差。
