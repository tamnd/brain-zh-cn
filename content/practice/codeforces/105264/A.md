 ## 解决方案
 ## 问题理解

 我们有一个团队，每个玩家都报告一些“贡献”。 贡献可以是进球，也可以是帮助其他球员进球的助攻。 每个进球只有一名得分手，并且可以选择配备一名助理。 球员的相同贡献不能同时用于同一目标的得分和助攻，并且每个贡献在最终重建中仅使用一次。 

输入数据为我们提供了每个球员总共贡献了多少贡献，但它并没有告诉我们其中有多少是进球或助攻。 根据这些部分信息，我们需要确定总共可以进球多少个，特别是与所有球员的贡献计数一致的最小和最大可能进球数。 

关键的结构性约束是每个进球消耗一项得分贡献，并且可以选择性地消耗一项助攻贡献。 这立即限制了分解的灵活性：贡献是全局固定的，但助攻与目标的配对不是。 

测试用例中所有贡献的总和可以高达 3 · 10^5，这排除了对玩家的任何二次推理或任何试图显式匹配贡献成对的构造。 每个测试用例的线性通过是必要的。 

当所有贡献都集中在一个玩家身上时，就会出现微妙的边缘情况。 例如，如果一名球员有5个贡献，而其他所有球员有0个，我们仍然必须遵守助攻不能超过进球数的规则。 简单地假设“每个贡献都是一个目标或独立的协助”而不强制它们之间的一致性的天真的方法将会过多计算可能的辅助结构。 

当尝试贪婪地分配助攻而不考虑每个助攻都必须附加到有效目标时，会出现另一种失败情况。 例如，将所有贡献视为可自由配对会导致不可能的配置，即助攻多于进球。 

## 方法

 强力解释会尝试将每个 A 总贡献分配为进球或助攻，然后验证所得到的分配是否可以分组为有效目标。 这本质上变成了一个约束划分问题：每个进球必须有一个得分贡献，最多有一个助攻贡献。 

在最坏的情况下，有 2^A 种方法可以为贡献分配角色，即使我们提前修剪无效状态，配置数量仍然会随着 A 呈指数增长。当 A 达到 3·10^5 时，这是完全不可行的。 

关键的观察结果是，真正重要的唯一全局约束是总贡献之间的平衡以及其中有多少可以配对成辅助边缘。 每个进球恰好贡献一个“强制单位”（得分手），也可以选择贡献一个额外单位（助攻）。 因此，如果我们将 G 表示为进球数，将 S 表示为所用助攻数，则每个进球都会消耗 1 或 2 个贡献，这给出了一个连接所有内容的方程式：

 A = G + S

 结构限制是每个辅助必须属于某个目标，因此 S 不能超过 G。这将问题转化为整数上的简单有界优化，而不是组合分配问题。 

由此可见，最大化目标对应于最小化S，而最小化目标对应于在S≤G下最大化S。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分配角色 | O(2^A) | O(2^A) | O(A) | 太慢了 |
 | 代数约束约简 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 通过对所有 ai 求和来计算贡献总数 A。 这是我们必须分配给进球和助攻的“单位”总数。 
2.观察每个进球至少消耗一个单位，如果有助攻则可能消耗两个单位。 引入变量 G 表示进球，S 表示辅助进球。 
3. 将总贡献约束表示为 A = G + S，因为每个进球贡献 1 个得分单位，每个助攻进球贡献 1 个额外助攻单位。 
4. 使用结构约束，即每次助攻都必须属于一个目标，这强制 S ≤ G。 
5. 将 G = A − S 代入不等式 S ≤ G，得到 S ≤ A − S，简化为 2S ≤ A。 
6. 由此，推导出最大可能的 S 为下限（A / 2），因为 S 必须是整数。 
7. 计算最小进球数为 Gmin = A − Floor(A / 2)，相当于 ceil(A / 2)。 
8. 通过最小化 S 来计算最大进球数，这使得 S = 0，因此 Gmax = A。 

### 为什么它有效

 整个结构崩溃了，因为贡献之间的唯一相互作用是通过将助攻与进球配对。 一旦我们抽象出个别球员，每个有效的配置都可以通过有多少个进球与助攻“配对”来充分描述。 任何违反 S ≤ G 的任务都需要比进球更多的助攻，这是不可能的，因为每次助攻都必须附加到一个不同的目标。 相反，有效范围内的任何S都可以通过将S贡献配对为助攻而将其余贡献作为非助攻目标来实现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        total = sum(arr)

        max_goals = total
        min_goals = (total + 1) // 2

        print(min_goals, max_goals)

if __name__ == "__main__":
    solve()
```该实现将一切简化为计算每个测试用例的贡献总和。 一旦知道总数，剩下的逻辑就是对导出公式的直接评估。 唯一的微妙之处是对总和的一半使用整数上限，实现为 (total + 1) // 2，这可以避免浮点错误。 

## 工作示例

 考虑一个输入，其中贡献分布为 [3, 2, 1]。 总数为6。 

对于最大目标，我们将每个贡献视为一个单独的目标，给出 6 个目标。 

对于最低进球数，我们尝试最大化助攻数，但我们受到助攻数不能超过进球数的限制。 6 的一半等于 3，因此我们可以有 S = 3 个助攻进球，G = 3 个总进球。 

| 步骤| 总A | S 选择 | G = A − S |
 | ---| ---| ---| ---|
 | 开始| 6 | 0 | 6 |
 | Max S 外壳 | 6 | 3 | 3 |
 | 结果 | 6 | 3 | 3 |

 这表明将一半的贡献与助攻-目标关系配对可以最大限度地减少进球数量。 

现在考虑[5]。 总数是5。 

| 步骤| 总A | S 选择 | G = A − S |
 | ---| ---| ---| ---|
 | 开始| 5 | 0 | 5 |
 | Max S 外壳 | 5 | 2 | 3 |
 | 结果 | 5 | 2 | 3 |

 这证明了天花板效应：一项贡献与助攻不成对，迫使额外的独立进球。

 ## 边缘情况

 贡献为零的单个玩家表明最小和最大都崩溃为零。 该算法计算总 A = 0，给出 Gmin = (0 + 1) // 2 = 0 和 Gmax = 0，这与唯一可能的配置匹配。 

当单个球员的贡献数为奇数时，例如 A = 5，算法计算出 Gmin = 3。这反映了这样一个事实，即最多可以配对两次助攻，留下一个必须形成独立目标的未配对贡献，从而正确执行上限行为。 

当所有玩家均等贡献时，例如四名玩家各贡献 1 个，则总数为 4，最小值变为 2。这对应于将贡献配对为两个辅助目标，这使得约束 S ≤ G 完全相等。
