 ## 解决方案
 ## 问题理解

 我们得到了一组整数。 还有第二个多重集，从空开始。 两名玩家交替从第一个多重集中移除一个元素。 当他们删除一个数字x时，他们不会直接将x放在任何地方； 相反，他们计算派生值 F(x)，定义为 x 的最大真因数，并将该值插入到第二个多重集中。 

当第一个多重集为空时游戏结束。 Ammar 首先发挥作用，并尝试最大化第二个多重集中出现的素数数量，而 Antwan 则尝试最小化它。 一个关键的转折是“素数”的定义：在此问题中数字 1 被明确视为素数。 

任务是确定在最佳玩法下放入第二个多重集合中有多少元素将是素数。 

这些约束意味着每个测试用例最多 10^5 个数字，跨测试总计最多 10^5 个数字。 这排除了游戏树的任何每次移动模拟或对多重集子集的动态规划。 任何解决方案都必须在大约 O(1) 或 O(log n) 摊余时间内处理每个数字。 

当 x 为素数时，会出现微妙的边缘情况。 如果 x 是素数，则其唯一真因数是 1，因此 F(x) = 1，在问题的定义下始终是素数。 如果 x 是合数，则 F(x) 是大于 1 的某个约数，并且可能是也可能不是素数，具体取决于结构。 

另一个重要的边缘情况是当 x 是素数的完全平方时，例如 4 或 9。那么 F(x) 等于 x / p，即 p，素数。 因此，在第二个多重集中生成素数方面，这些值的行为与素数类似。 

一个天真的误解是假设只有素数重要或只检查 x 本身的素数，但实际贡献完全取决于 F(x)，而不是 x。 

## 方法

 直接的暴力解释将游戏视为多重集合上的组合游戏。 每次移动都会删除一个元素并产生一个确定性值 F(x)。 人们可能会尝试模拟博弈树，让每个玩家选择要选择的元素，以便最大化或最小化生成的素数的最终数量。 

这立即变得棘手。 即使我们忽略策略并仅通过递归或极小极大模拟最佳游戏，分支因子也是剩余元素的数量，从而导致游戏状态的阶乘增长。 当n达到10^5时，这是完全不可能的。 

关键的观察结果是，除了回合顺序之外，游戏实际上并不依赖于不同元素之间的交互。 每个元素都独立地转换为固定值 F(x)，唯一的“选择”是这些转换发生的顺序。 后面的操作不会影响前面的 F(x) 值。 

因此，游戏简化为：有 n 个独立的项目，每个项目根据 F(x) 是否为素数，对最终答案贡献 0 或 1。 玩家只控制哪些项目被提前或推迟处理，但由于每一步都会立即产生确定性的贡献，因此素数的总数在排序下是不变的。 

策略可能重要的唯一方式是未来的选择是否影响当前值，但每个元素的 F(x) 都是固定的。 因此，最优游戏不会改变产生值的多重集； 它仅在它们出现时才会排列。 

这将游戏分解为一个简单的计数问题：计算每个元素的 F(x) 并计算有多少个结果是素数（1 被视为素数）。

 ## 算法演练

1. 使用筛子预先计算 10^6 以内的所有整数的最小素因数。 这允许对每个 x 进行快速因式分解和除数推理。 
2. 对于每个数字x，确定其最小素因数p。 如果 x 是素数，则 F(x) = 1 立即，因为它唯一的真因数是 1。 
3. 如果 x 不是素数，则通过确定最大真因数来计算 F(x)。 最大真因数是 x / p，其中 p 是 x 的最小质因数。 
4. 确定 F(x) 是否为素数，记住 1 被视为素数。 这意味着如果 F(x) = 1 或 F(x) 恰好有一个素因数，则 F(x) 有效。 
5. 计算有多少个 x 值产生素数 F(x) 并输出该计数。 

关键的推理步骤是步骤 3。合数的最大真因数必须排除补码形式的最小素因数，因为除以最小素因数会产生最大可能的商。 

### 为什么它有效

 每个元素都向第二个多重集贡献一个值，独立于所有其他元素且独立于移动顺序。 游戏只决定时间，而不决定结果。 由于 F(x) 是确定性的并且不依赖于未来或过去的选择，因此两个参与者都有效地选择了固定贡献列表的排列。 最终计数仅取决于这些贡献中有多少是质数，因此最佳游戏无法改变结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXV = 10**6

spf = list(range(MAXV + 1))
for i in range(2, int(MAXV**0.5) + 1):
    if spf[i] == i:
        for j in range(i * i, MAXV + 1, i):
            if spf[j] == j:
                spf[j] = i

def is_prime_like(x):
    return x == 1 or (x > 1 and spf[x] == x)

def f_of(x):
    if x == 1:
        return 1
    if spf[x] == x:
        return 1
    return x // spf[x]

t = int(input())
for _ in range(t):
    n = int(input())
    arr = list(map(int, input().split()))
    ans = 0
    for x in arr:
        fx = f_of(x)
        if is_prime_like(fx):
            ans += 1
    print(ans)
```筛子计算最大 10^6 的最小质因数，以便可以在恒定时间内对每个数字进行分类。 函数 f_of 实现了素数映射到 1 和复合映射到 x 除以其最小素因数的结构规则。 

素性检查有意最小化：1 被视为素数，任何以其自身为最小素因数的数字都是素数。 这与数学结构和问题的修改定义相匹配。 

## 工作示例

 考虑多重集为的输入`[5, 6, 8]`。 

对于 5，它是素数，因此 F(5) = 1。 

对于 6，最小质因数为 2，因此 F(6) = 3。 

对于 8，最小质因数为 2，因此 F(8) = 4。 

现在我们根据问题的规则检查素数：1 是素数，3 是素数，但 4 不是素数。 所以答案是2。 

| x| SPF(x) | F(x) | F(x) | 像总理一样？ |
 | --- | --- | --- | --- |
 | 5 | 5 | 1 | 是的 |
 | 6 | 2 | 3 | 是的 |
 | 8 | 2 | 4 | 没有|

 这表明合数仍然可以通过 F(x) 贡献素数，特别是当 x 是素数的两倍时。 

现在考虑`[4, 9, 10]`。 

对于 4，F(4) = 2，素数。 

对于 9，F(9) = 3，素数。 

对于 10，spf 为 2，因此 F(10) = 5，素数。 

| x| SPF(x) | F(x) | F(x) | 像总理一样？ |
 | --- | --- | --- | --- |
 | 4 | 2 | 2 | 是的 |
 | 9 | 3 | 3 | 是的 |
 | 10 | 10 2 | 5 | 是的 |

 这表明，即使是完全平方和混合复合也总是通过其最小因子结构塌陷为素数。

 ## 边缘情况

 对于素数输入（例如 x = 2），算法直接映射到 F(x) = 1。由于 1 被视为素数，因此每个素数输入都会为答案贡献 1。 筛子正确地将 2 识别为其自身的最小质因数，从而触发质数分支。 

对于像 x = p^2 这样的数字，例如 9，最小的素因数是 p，因此 F(x) 变为 p。 即使输入不是素数，该算法也能正确生成素数输出，这表明 x 的素数不是相关条件。 

对于像 x = 2p 这样的数字，例如 10，最小质因数是 2，产生 F(x) = p。 这表明偶数也可以产生大素数，并且逻辑正确地捕获了这一点，无需 SPF 计算之外的特殊外壳。
