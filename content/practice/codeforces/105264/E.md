 ## 解决方案
 ## 问题理解

 我们有一棵有 n 个顶点的树。 我们可以精确保留 k 个顶点并考虑它们导出的子图。 由于所选顶点仍必须形成连通树，因此选择实际上被限制为原始树的连通 k 节点子树。 

在所有k个顶点的选择中，我们希望在最坏的情况下内部距离尽可能小。 换句话说，我们在选定的 k 个节点上获取归纳树，计算以最长简单路径上的顶点数测量的树的直径，并且我们希望最小化该直径。 

输出是一个整数：所有大小为 k 的相连子树的最小可能直径。 

约束 n ≤ 1000 足够小，二次或三次方法仍然可以通过，但节点子集上的任何指数都立即不可行，因为大约有 2^1000 个子集。 O(n^3) 或 O(n^2 log n) 的解决方案是可以接受的。 

一个微妙的点是我们不允许选择任意 k 个节点。 所选择的集合必须产生一个连通的子树，因此选择在原始树中相距较远但在产生的结构中断开的节点是无效的。 这排除了贪婪的“选择 k 个最佳中心”的想法，除非仔细保留连接性。 

一个天真的误解是认为我们选择 k 个节点来最小化原始树中的最大成对距离，忽略诱导结构。 这将允许断开的集合并错误地产生较小的值。 

## 方法

 思考该问题的直接方法是考虑大小为 k 的每个可能的连通子集并计算其直径。 对于每个候选集，我们可以计算仅限于该子集的所有对最短路径，或者从其中的每个节点运行 BFS。 由于每个子集都会生成一棵树，因此一旦邻接已知，计算其直径就可以在 O(k) 内完成。 

问题是枚举连接的 k 子树。 即使我们尝试通过 DFS 或 BFS 来增长子集，此类子集的数量在星形或路径等一般树形状中也是指数级的。 在路径中，长度为 k 的每个连续段都是有效的，即 O(n)，但在星形中，如果我们不注意结构，选择子集的组合会变得很大。 暴力枚举很快就会退化为尝试所有节点组合，这是不可能的。 

关键的观察是任何树的直径都由两个端点控制。 如果我们将两个节点 u 和 v 固定为所选 k 节点子树的潜在端点，那么包含它们的最佳子树将尝试以保持连通性的方式“填充”沿 u 和 v 之间的路径及其周围的节点。 最终导出子树中 u 和 v 之间的距离本质上决定了直径。 

这建议重新构建问题：我们不直接选择 k 个节点，而是考虑两个节点之间的候选路径，并询问可以包含多少个节点，同时将该路径保留为子树的“主干”。 如果我们对树进行根运算并预先计算距离，我们就可以为每对节点评估包含它们的最小子树的结构。 该子树中的节点数是它们之间的所有简单路径上的所有顶点的并集的大小，这相当于计算最小连接子树中的节点数。 

对于任何对 (u, v)，连接它们的最小子树只是它们之间路径的并集，但这只是两个端点。 为了到达 k 个节点，我们必须从该路径向外扩展，但只有当扩展迫使沿着更长的路径增长时，它才会增加直径。 因此，最佳结构往往是路径周围的“中心”子树，我们尝试最小化距中心线段的最大距离。

形式化的标准方法是对答案 D 进行二分搜索，并检查是否存在大小为 k 且直径最大为 D 的连通子树。对于固定的 D，子树必须适合某个中心节点或边缘周围的“半径 D/2”。 这是经典的树半径-直径关系：树的直径 ≤ D 当且仅当它有一个中心，并且所有节点都在距它 ⌊D/2⌋ 的距离内（对于奇数 D，中心是一条边）。 

因此，对于固定的 D，我们询问是否存在一个节点或边，其半径为 r 的球至少包含 k 个节点，其中 r = D // 2。 

这将问题简化为计算每个节点有多少个节点在距离 ≤ r 内。 此外，由于当 D 为奇数时，中心可能位于边缘上，因此我们还通过将每条边缘视为潜在中心来考虑中点。 

我们可以通过 BFS 计算每个节点的距离，时间复杂度为 O(n^2)，这已经足够了。 然后，对于每个候选中心，我们计算半径 r 内的可达节点，并检查是否有至少达到 k 的节点。 

因此，最优解变成了对 D 的二分搜索结合 O(n^2) 预计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有连接的 k 子树进行暴力破解 | 指数| O(n) | 太慢了 |
 | 二分搜索直径 + BFS 距离 | O(n^2 log n) | O(n^2 log n) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们使用每个节点的 BFS 预先计算树中所有对的最短路径距离。 由于图是一棵树，因此每个 BFS 的复杂度为 O(n)，因此总共为 O(n^2)。 

然后我们从 1 到 n 对答案 D 进行二分查找。 

对于固定的 D，我们设置 r = D // 2。我们检查是否存在有效的中心。 

1. 对于每个节点 u，我们统计有多少个节点 v 满足 dist(u, v) ≤ r。 如果任何这样的计数至少为k，我们可以将u作为中心并构建直径≤D的子树。 
2.对于奇数D，我们还必须考虑位于边缘的中心。 对于每条边（u，v），我们想象一个中点中心。 如果 min(dist(u, w), dist(v, w)) ≤ r，则节点 w 位于该中点的半径 r 内，但更准确地说，在经过 u 或 v 的路径上到边缘中心的距离为 min。在树中，这简化了通过一步调整检查 w 是否位于来自任一端点的 r 内，这可以通过使用预先计算的距离仔细测试两个方向来处理。 
3. 如果任意节点或边中心满足条件，则 D 可行，否则不可行。 
4.二分查找最小可行D。 

这背后的原因是，任何直径为 D 的树都有一个中心（节点或边缘），使得所有节点都在距它 ⌊D/2⌋ 的距离内。 相反，如果存在这样一个中心，其半径为 r 的球内至少有 k 个节点，我们可以选择这 k 个节点，它们形成直径≤D 的连通子树。 

工作原理：每个直径为 D 的连通子树都有一个中心，可以最小化到所有节点的最大距离。 该中心产生一个半径为⌊D/2⌋的覆盖球，其中包含子树的所有节点。 因此 D 的可行性相当于存在一个半径为 r 的球，该球在某个有效的中心度量中至少包含 k 个节点，并且检查所有可能的中心隐式覆盖所有子树。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import deque

n, k = map(int, input().split())
adj = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    adj[u].append(v)
    adj[v].append(u)

dist = [[0] * n for _ in range(n)]

def bfs(src):
    q = deque([src])
    d = [-1] * n
    d[src] = 0
    while q:
        u = q.popleft()
        for v in adj[u]:
            if d[v] == -1:
                d[v] = d[u] + 1
                q.append(v)
    dist[src] = d

for i in range(n):
    bfs(i)

def can(D):
    r = D // 2

    for u in range(n):
        cnt = 0
        du = dist[u]
        for v in range(n):
            if du[v] <= r:
                cnt += 1
        if cnt >= k:
            return True

    if D % 2 == 1:
        for u in range(n):
            for v in adj[u]:
                if u < v:
                    cnt = 0
                    for w in range(n):
                        duw = dist[u][w]
                        dvw = dist[v][w]
                        if min(duw, dvw) <= r:
                            cnt += 1
                    if cnt >= k:
                        return True

    return False

lo, hi = 1, n
ans = n
while lo <= hi:
    mid = (lo + hi) // 2
    if can(mid):
        ans = mid
        hi = mid - 1
    else:
        lo = mid + 1

print(ans)
```该代码首先使用 BFS 计算每个节点的所有成对距离，并将它们存储在矩阵中。 然后，可行性函数使用半径表征检查给定直径边界是否可以支持 k 节点子树。 

节点中心检查计算每个节点的半径 r 内有多少个节点。 仅当 D 为奇数时才需要边缘中心检查，因为奇数直径树的最佳中心位于边缘上。 

二分查找缩小了最小可行直径的范围。

 | 2 | {1,2,3} | 3 |

def run(inp: str) -> str:

    from collections import deque

    n, k = map(int, input().split())
    adj = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        adj[u].append(v)
        adj[v].append(u)

    dist = [[0] * n for _ in range(n)]

    def bfs(src):
        q = deque([src])
        d = [-1] * n
        d[src] = 0
        while q:
            u = q.popleft()
            for v in adj[u]:
                if d[v] == -1:
                    d[v] = d[u] + 1
                    q.append(v)
        dist[src] = d

    for i in range(n):
        bfs(i)

    def can(D):
        r = D // 2
        for u in range(n):
            if sum(1 for v in range(n) if dist[u][v] <= r) >= k:
                return True
        if D % 2 == 1:
            for u in range(n):
                for v in adj[u]:
                    if u < v:
                        cnt = 0
                        for w in range(n):

                                cnt += 1
                        if cnt >= k:
                            return True
        return False

    lo, hi = 1, n
    ans = n
    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    return str(ans)

 ## 边缘情况

 一个重要的边缘情况是当 k = 1 时。按照惯例，任何单个节点都已经是直径为 1 的有效子树。 该算法可以处理此问题，因为当 r = 0 时，每个节点仅计算其自身，并且对于 D = 1 时可行性立即成立。 

另一种边缘情况是星形树，对于任何 k ≥ 2，最优解始终为 2。在这种情况下，中心节点覆盖半径 1 内的所有节点，因此二分搜索很快收敛。 甚至不需要边缘中心逻辑，因为节点中心已经主导了可行性。 

最后一个微妙的情况是，最佳结构直观地看起来像路径段而不是围绕中心的球。 即使在这些情况下，基于半径的表征仍然可以正确捕获可行性，因为任何最小化直径的树都可以被视为以中间节点或边缘为中心，并且所有节点都位于相应的半径范围内。
