 ## 解决方案
 ## 问题理解

 每个测试用例给出一个编号$n$，我们从概念上构建行编号$1$到$n$。 排$i$是的二进制表示$i$，不带前导零，每一位对应一个打开 (1) 或关闭 (0) 的灯泡。 

完成此构造后，从左到右扫描时，将执行一项操作，删除每行中偶数位置的每个灯泡。 位置的索引为 1，因此我们保留第 1、第 3、第 5 个位置并丢弃第 2、第 4、第 6 等位置。 剩下的是每行一个较短的二进制字符串，我们被要求计算有多少个 1 在所有行中幸存下来$1$到$n$。 

约束条件$n \le 10^9$意味着我们无法单独模拟行。 即使为每个数字显式计算二进制表示也太慢，因为最多有$10^4$测试用例，因此任何按数字或按位的线性扫描立即不可行。 该解决方案必须在每个测试用例的大致对数时间内工作，理想情况下仅取决于测试用例的位数$n$，最多为 30。 

一个微妙的边缘情况是偶数位置的删除取决于每个二进制表示的长度。 例如，数字意义上的相同位位置并不总是对应于相同的保留/丢弃状态，因为无论位位于左侧的奇数索引还是偶数索引，移位长度都会改变。 例如，数字$4 = 100_2$只保留第一和第三位置，而$5 = 101_2$尽管两者都有三位，但行为不同。 这排除了独立于数字长度来处理位位置的情况。 

另一个棘手的情况是当$n$略低于 2 的幂。 然后大多数数字共享相同的位长度，但最后一个块不完整，因此如果不仔细处理，任何关于长度完全一致性的假设都可能失败。 

## 方法

 直接方法将迭代每个整数$i$从$1$到$n$，将其转换为二进制，删除偶数索引字符，并计算个数。 这是正确的，但太慢了。 每次转换成本$O(\log n)$，并为所有人做这件事$n$值导致大约$n \log n$每个测试用例的操作次数，远远超出了限制$n$达到$10^9$。 

关键的观察结果是，如果我们停止考虑每个数字，而是考虑每个位位置和每个二进制长度，那么该结构就会变得易于管理。 位的生存条件取决于两个独立的事实：该位是否在数字中设置，以及它在二进制表示中的位置是否为奇数。 第二个条件仅取决于数字二进制形式的总长度。 

这允许我们按位长度对数字进行分组。 对于固定长度$L$, 范围内的所有数字$[2^{L-1}, \min(n, 2^L - 1)]$对于左侧的位置具有相同的奇偶行为。 在这样的块内，我们只需要计算有多少个数字在固定位置设置了给定位，这是一个标准的二进制计数问题，可以用以下方法解决$O(1)$per bit using periodic patterns.

 该转换将问题从迭代所有数字减少为迭代位长度和位位置，其范围约为 30 个级别。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \log n)$|$O(1)$| 太慢了 |
 | 位长分组+位计数 |$O(\log^2 n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 划分范围$1$到$n$分成数字共享相同二进制长度的块$L$。 每个块对应$[2^{L-1}, \min(n, 2^L - 1)]$。 这很重要，因为在一个块内，“左侧奇数位置”的定义是一致的。 
2.对于固定长度$L$，确定哪些位位置幸存。 如果是奇数，则保留左侧的位置。 将其转换为从右侧位位置开始的从零开始的索引$k$被包括在内，如果$k \bmod 2 = (L-1) \bmod 2$。 这将左位置规则转换为标准位索引上的稳定条件。 
3. 对于每个有效位位置$k$，计算当前块中有多少个整数有位$k$放。 这是通过对大小间隔进行定期计数来完成的$2^{k+1}$，其中每个周期在该位位置包含固定数量的 1。 
4. 将所有有效贡献的总和$k$对于当前长度$L$，然后累积所有$L$。 

步骤 3 中的核心计算使用以下事实：$k$-th 位在大小块中交替$2^{k}$0 和 1，因此我们可以分别计算完整周期和余数，而无需迭代每个数字。 

### 为什么它有效

 每个数字的每一位贡献都是独立的，“生存”规则仅取决于数字的长度和位索引。 通过首先固定长度，我们消除了耦合不同位位置的唯一依赖关系。 分离之后，每一位的行为就像整数上的独立周期函数，并且对不相交的长度间隔求和可以保持正确性，因为每个整数恰好出现在一个长度块中。 分解是完整且不重叠的，因此不会重复计算或遗漏任何贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_ones_upto(x, k):
    if x <= 0:
        return 0
    block = 1 << (k + 1)
    full = (x + 1) // block
    rem = (x + 1) % block
    ones = full * (1 << k)
    extra = max(0, rem - (1 << k))
    return ones + extra

def count_ones_range(l, r, k):
    return count_ones_upto(r, k) - count_ones_upto(l - 1, k)

t = int(input())
for _ in range(t):
    n = int(input())
    ans = 0

    max_l = n.bit_length()

    for L in range(1, max_l + 1):
        left = 1 << (L - 1)
        right = min(n, (1 << L) - 1)
        if left > right:
            continue

        parity = (L - 1) & 1

        for k in range(L):
            if (k & 1) == parity:
                ans += count_ones_range(left, right, k)

    print(ans)
```该代码首先实现一个标准函数来计算有多少个达到给定限制的数字具有特定的位集。 它依赖于将数轴分割成完整的长度周期$2^{k+1}$，其中$k$-th 位是均匀分布的。 

对于每个测试用例，循环$L$隔离该位长度的有效数值范围。 奇偶校验计算确定哪些位位置能够通过从左到右的过滤。 累积贡献时仅考虑那些位索引。 

这里的一个常见错误是直接混合左索引和右索引。 该实现通过在任何计数开始之前将规则转换为标准位索引上的奇偶校验条件来完全避免这种情况。 

## 工作示例

 考虑一个小输入，其中$n = 5$。 二进制表示为：

 1 → 1

 2 → 10

 3 → 11

 4 → 100

 5 → 101

 从左侧删除偶数位置后，每行贡献其原始 1 位的子集。 

我们按长度块进行追踪。 

为了$L = 1$，仅包含数字 1。 只有位 0 存在，并且被保留。 贡献度为1。 

对于$L = 2$，包括数字 2 和 3。 仅具有奇偶匹配的位置$(L-1)=1$生存，对应于位 1。只有数字 3 设置了位 1，贡献 1。 

| 左 | 范围 | 允许的位 | 贡献 |
 | --- | --- | --- | --- |
 | 1 | [1,1]| k = 0 | 1 |
 | 2 | [2,3]| k=1 | k=1 1 |

 这给出了从完整块到 3 的总计 2。然后$L = 3$类似地处理 4 和 5。 

为了$n = 5$，包含所有块后最终结果变为4。 

该跟踪显示了如何永远不会重新访问相同的数字，以及如何通过长度分组干净地隔离比特贡献。

 ## 边缘情况

 当$n = 1$，只存在一位，并且总是处于奇数位置，因此必须对其进行计数。 该算法正确地处理了这个问题，因为长度块$L=1$仅包含数字 1 并选择位 0。 

当$n$正好是 2 的幂，例如$8$，最后一个块仅包含一个元素。 范围计算确保不完整的块仍然被正确处理，因为$\min(n, 2^L - 1)$剪辑间隔。 

什么时候$n$略低于 2 的幂，例如$7$，所有数字都落入单个长度块中，并且该算法仍然有效，因为位计数与块是完整的还是部分的无关。
