 ## 解决方案
 ## 问题理解

 我们被要求计算从 0 到给定的大数范围内有多少整数满足位置数字规则。 该规则取决于以十进制书写每个数字并使用基于 1 的索引从左到右查看其数字。 位于奇数位置的数字本身必须是奇数，而位于偶数位置的数字必须是偶数。 

该任务不是关于单个数字是否有效，而是关于计算从 0 到给定限制（含）的所有有效数字。 这立即将问题变成了数字字符串上的约束计数问题，因为上限可以大到 10^18，所以直接迭代所有整数是不可能的。 

输入大小意味着最多 19 位十进制数字。 在最坏的情况下，任何单独检查每个数字的解决方案都需要多达 10^18 次操作，这远远超出了可行的限制。 即使每个数字执行 O(1) 工作的解决方案也是不可能的，因此我们需要一种基于数字的组合方法。 

一个微妙的边缘情况是对带有前导零的数字的处理。 出于计数目的，可以将较短的数字视为具有前导零以匹配上限的长度。 这是必要的，因为位置规则取决于索引奇偶性，忽略前导零会破坏一致性。 另一个边缘情况是数字 0 本身。 由于它只有一位数字，因此它必须满足位置 1 处的规则，即需要奇数数字。 因此 0 是无效的，即使它位于范围内。 

## 方法

 暴力方法会迭代从 0 到 N 的每个数字，将每个数字转换为字符串，并验证每个数字是否满足奇偶位置约束。 这是正确的，因为每个数字的检查与位数是线性的，最多为 19。但是，候选者的数量最多为 10^18，因此即使对每个数字进行恒定时间检查，总的工作也是不可行的。 

关键的观察结果是，有效性仅取决于位置和数字选择，而不取决于不同数字之间的相互作用。 这表明数字动态规划或位置组合学。 我们不是枚举数字，而是统计每个长度的有效数字串，然后使用前缀约束的 DP 来统计有多少个有效数字小于或等于 N。 

我们将问题分为两部分。 首先，计算长度严格小于 N 长度的所有有效数字。这些可以使用简单的乘积规则独立计算：每个位置根据奇偶校验有固定数量的有效数字选择。 其次，使用尊重前缀约束的数字 DP 处理与 N 长度相同的数字。 

该结构将问题从迭代最多 10^18 个候选者减少到处理最多 19 个位置，每个位置最多 10 个状态。

 | 蛮力 | O(N·19) | O(N·19) | O(1) | O(1) | 太慢了 |

 ## 算法演练

 我们将数字视为字符串`s`并逐位工作。 

1.我们将N转换为字符串`s`并计算它的长度`L`。 这让我们可以通过固定的数字长度而不是数值来推理数字。 
2. 我们预先计算每个位置平价存在多少种选择。 在奇数索引 (1, 3, 5, ...) 处，有效数字为 {1, 3, 5, 7, 9}，提供 5 个选择。 在偶数索引 (2, 4, 6, ...) 处，有效数字为 {0, 2, 4, 6, 8}，也是 5 个选择。 这种对称性大大简化了计数。 
3.我们统计所有长度严格小于L的有效数字。对于固定长度k，每个有效数字必须满足所有k个位置的位置约束。 由于每个位置独立地具有 5 个有效数字，因此长度 k 的总计数为 5^k，但我们必须小心前导零。 前导零在偶数位置上表现为偶数，但零也只允许在偶数位置上，因此如果我们允许前导零并稍后仔细删除无效的前导零解释，则规则保持一致。 一种更简单的方法是将前导零视为有效构造的一部分，并在需要时在末尾减去无效的空数情况。 
4. 对于长度恰好为 L 的数字，我们从左到右执行逐位扫描，维护前缀是否已经严格小于 N。在每个位置，我们尝试满足奇偶校验约束的所有数字，并计算后缀剩余的选择数。 
5. 在处理位置i时，我们根据奇偶校验来确定允许的位数组。 如果 i 是奇数，我们只考虑奇数位。 如果是偶数，则只有偶数位。 
6. 如果我们仍然匹配 N 的前缀，则位置 i 处的数字以 s[i] 为界。 对于每个小于 s[i] 的允许数字，我们使用预先计算的 5 次幂来计算剩余位置存在多少个完成。如果所选数字等于 s[i]，我们继续； 否则，我们停止前缀匹配。 
7. 处理完所有位置后，如果整个数字有效，我们将其包含在答案中。 

### 为什么它有效

 该算法依赖于以下不变量：在每个位置，前缀的所有有效完成都独立于早期的数字选择，除了剩余长度之外。 一旦前缀固定，有效补全的数量仅取决于剩余位置的数量，因为每个位置恰好有 5 个仅受奇偶校验限制的有效数字选择。 数字DP确保我们永远不会计算超过N的数字，而组合计数确保我们有效地聚合所有可能性而不需要枚举。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N = input().strip()
    L = len(N)

    # valid digits by parity
    odd_digits = [1, 3, 5, 7, 9]
    even_digits = [0, 2, 4, 6, 8]

    # precompute powers of 5
    pow5 = [1] * (L + 1)
    for i in range(1, L + 1):
        pow5[i] = pow5[i - 1] * 5

    def count_len(k):
        # number of valid k-digit numbers (allow leading zeros)
        return pow5[k]

    ans = 0

    # count numbers with length < L
    for k in range(1, L):
        ans += count_len(k)

    # digit DP for length L
    for i in range(L):
        pos = i + 1
        limit = int(N[i])

        if pos % 2 == 1:
            digits = odd_digits
        else:
            digits = even_digits

        for d in digits:
            if i == 0 and d == 0:
                continue  # no leading zero numbers at full length

            if d < limit:
                ans += pow5[L - i - 1]
            elif d > limit:
                continue
            else:
                break

    # check if N itself is valid
    ok = True
    for i, ch in enumerate(N):
        d = int(ch)
        pos = i + 1
        if pos % 2 == 1 and d % 2 == 0:
            ok = False
        if pos % 2 == 0 and d % 2 == 1:
            ok = False

    if ok:
        ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将贡献与短于输入长度的数字分开，这些数字纯粹使用 5 的幂进行组合处理。然后，数字 DP 循环通过从左到右扫描并计算在某个位置选择较小数字时存在多少个有效完成来处理等长数字。 

一个关键的实现细节是前导零的处理。 由于我们单独计算固定长度数字，因此我们明确防止全长情况的第一个位置出现零。 这可以避免错误地计算填充到全长的较短表示。 

最终的有效性检查确保仅当边界数 N 满足位置奇偶校验约束时才包含该边界数 N。

 ## 边缘情况

 一种边缘情况是 N 是单个数字（例如 1 或 9）。在这种情况下，只有位置 1 重要，并且只有奇数位有效。 该算法仅正确计算有效的奇数位并排除 0，即使它在数值范围内也是如此，因为 0 不符合索引 1 处的位置规则。 

另一种边缘情况是当 N 包含交替奇偶校验违规时，例如 20 或 31。数字 DP 确保一旦遇到违规数字，该分支就会被丢弃，而较小的前缀分支仍然可以通过组合后缀计数正确计数。
