 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例都是代表一行卡片的整数序列。 从这一行中，我们可以选择任何连续的段，对于该段，我们计算一个分数，定义为奇数值的数量减去其中素数的数量。 任务是找到所有非空连续段的最大可能得分。 

该结构是纯线性的，因此每个决定都与子数组的开始和结束位置有关。 除了连续间隔之外没有任何组合选择，这立即表明基于前缀或 Kadane 式推理，而不是任何指数推理。 

约束足够小，以至于$O(n^2)$每个测试解决方案都会通过。 总共有$n \le 2000$，即使是计算所有子数组的双循环也需要大约四百万次操作，这在 Python 中很好。 然而，每个测试用例的预期解决方案都是线性的。 

一个微妙的点在于“奇”与“素”的相互作用。 除 2 之外的每个素数都是奇数，因此素数与奇数有很大的重叠。 这意味着贡献不是独立的类别； 数字可以增加奇数计数，同时减少素数计数。 数字 2 是唯一一个非奇数素数，这使得它的行为与其他素数不同。 

一个幼稚的错误是将分数视为两个独立的数组，或者分别尝试赔率和素数的前缀计数，而没有意识到它们重叠。 另一个错误是假设素数仅作为赔率的子集起作用。 事实上，即使是非奇数素数（只有 2 个）仍然会降低分数，而如果处理不当，奇数素数的贡献净为零。 

例如，考虑一个仅包含 2 的段。分数为$0 - 1 = -1$。 如果粗心地只将奇素数视为“坏几率”，就会错误地将其视为中性。 

## 方法

 暴力解决方案评估每个子数组并直接计算其分数。 对于每对索引$l, r$，我们扫描该段并计算赔率和素数。 这是直接且正确的，因为它直接遵循定义。 然而，它反复重复相同的工作：每个段从头开始重新计算计数。 

该方法的时间复杂度为$O(n^3)$每个测试用例如果按字面意思实现，因为有$O(n^2)$子数组和每个子数组$O(n)$来评估。 即使前缀和将评估减少到$O(1)$，我们仍然需要仔细编码奇数和质数指标，但这只会将其简化为$O(n^2)$，在约束条件下可以接受，但不是最优的。 

关键的观察结果是每个元素对分数的贡献都是独立的。 对于每个值$a_i$，我们可以给它分配一个权重：

 如果是奇数，则贡献+1； 如果是素数，则贡献-1； 如果两者都是（奇素数），则净贡献为 0； 如果两者都不是，则贡献 0。由于 2 是唯一的偶素数，因此贡献 -1。 每个奇素数贡献 +1 和 -1，抵消为 0。 

这将问题简化为在转换后的数组上查找最大子数组和，其中每个元素都有固定的权重。 这正是 Kadane 的算法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$或者$O(n^2)$|$O(1)$| 太慢/在限制下接受|
 | 最佳（Kadane）|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 预先计算每个数是否为素数。 由于值上升到$10^6$，我们全局使用一次筛子。 这避免了每个测试用例内重复的素性检查。 
2. 对于每个元素$a_i$，计算其贡献：

 如果是奇数，则在其基础分数上加+1； 如果它是素数，则减 1。这会生成一个代表问题的整数数组。 
3. 在此变换后的数组上运行最大子数组和算法。 保持这里的最佳成绩和全球最佳成绩。 
4. 在每个位置，决定是延长前一段还是重新开始。 这种选择反映了负前缀是否会损害未来的收益。 
5. 跟踪所有头寸的最大值。 

步骤 4 背后的原因是任何最佳段必须要么包含前一个元素，要么重新开始； 跳过所选子数组内的元素没有任何好处。 

### 为什么它有效

 转换后，每个元素为任何子数组贡献一个固定的独立值。 子数组的分数成为此变换数组中其元素的总和。 任何连续段最大化总和正是经典的最大子数组问题。 Kadane 算法中的贪婪转换可确保丢弃任何具有负总数的前缀，因为它会减少该段中所有未来的总和，因此重新启动会产生更好或相同的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXV = 10**6

# Sieve for primality
is_prime = [True] * (MAXV + 1)
is_prime[0] = is_prime[1] = False
for i in range(2, int(MAXV ** 0.5) + 1):
    if is_prime[i]:
        step = i
        start = i * i
        for j in range(start, MAXV + 1, step):
            is_prime[j] = False

t = int(input())
for _ in range(t):
    n = int(input())
    arr = list(map(int, input().split()))

    best = -10**18
    cur = -10**18

    for x in arr:
        val = 0
        if x % 2 == 1:
            val += 1
        if is_prime[x]:
            val -= 1

        if cur == -10**18:
            cur = val
        else:
            cur = max(val, cur + val)

        best = max(best, cur)

    print(best)
```该筛子在全局范围内构建一次，以确保在每个测试用例期间素性检查的时间恒定。 每个数字都独立地转换为其贡献。 Kadane 步骤使用标准递归：在当前元素处启动一个新子数组或扩展前一个子数组。 

的初始化`cur`使用哨兵可以避免第一个元素的歧义，确保第一个值始终干净地开始一个段。 

## 工作示例

 考虑一个数组：`[1, 2, 3, 4]`。 

我们计算贡献：

 - 1：奇数，非质数 → +1
 - 2：偶素数 → -1
 - 3: 奇素数 → +1 - 1 = 0
 - 4：偶数非素数 → 0

 所以变换后的数组是`[1, -1, 0, 0]`。 

| 索引 | 价值| 贡献| 最好的结局在这里| 全球最佳|
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 1 | 1 | 1 |
 | 2 | 2 | -1 | 0 | 1 |
 | 3 | 3 | 0 | 0 | 1 |
 | 4 | 4 | 0 | 0 | 1 |

 最大段为`[1]`值为 1。这证实了负贡献（如 2）会重置潜在收益。 

现在考虑`[2, 3, 5, 4]`。 

贡献：

 - 2 → -1
 - 3 → 0
 - 5 → 0
 - 4 → 0

 | 索引 | 价值| 贡献| 当前 | 最佳|
 | ---| ---| ---| ---| ---|
 | 1 | 2 | -1 | -1 | -1 |
 | 2 | 3 | 0 | 0 | 0 |
 | 3 | 5 | 0 | 0 | 0 |
 | 4 | 4 | 0 | 0 | 0 |

 最优段是`[3, 5, 4]`给出 0，表明在负元素之后开始可以改善结果。

 ## 边缘情况

 单个元素等于 2 是最重要的边界情况。 该算法为其分配值 -1，并且 Kadane 正确地将其视为独立子数组，因为从负前缀扩展只会减少未来的总和。 

仅包含奇数素数的数组产生所有零贡献。 最大子数组和变为 0，Kadane 自然选择任何非空段，确认取消处理正确，无需特殊大小写。 

具有交替大正值和单个负值的数组演示了 Kadane 的重新启动行为。 每当出现 -1 时，运行总和就会下降，如果有益的话，算法可能会在其后重新启动，以确保最佳分割。
