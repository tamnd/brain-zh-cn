 ## 解决方案
 ## 问题理解

 我们从按级别排列的人工智能机器人群体开始。 最初只有 n 级机器人，并且有 k 个。 时间以固定的 5 分钟步长前进，每一步每个机器人只执行一个动作。 

1 级机器人会产生一个“单词”，这是我们最终关心的。 任何级别高于 1 的机器人都会生成一个低一级的新机器人。 因此，随着时间的推移，i 级机器人 (i > 1) 会将自身转换为 (i-1) 级机器人，而 1 级机器人会积累单词而不是产生更多机器人。 

系统以 5 分钟的离散批次确定性地演化。 t 分钟后，我们想知道总共产生了多少个单词。 

关键的困难在于机器人以级联方式繁殖：较高级别的机器人不断生成较低级别的机器人，而这些机器人后来又开始产生更多的机器人或单词。 这创建了一个分层的增长过程，类似于时间步长和级别的卷积。 

这些约束在时间上很小 (t ≤ 10^4)，但在种群规模上可能很大（k 高达 10^9）且深度适中 (n ≤ 100)。 这立即表明我们无法模拟单个机器人。 任何方法都必须汇总每个级别和每个时间步长的计数。 

一个幼稚的错误是独立模拟每个机器人。 这会将 k 到 10^9 视为显式对象，这是不可行的。 另一个微妙的错误是将转换视为同一时间步长内的瞬时连锁反应。 实际上，该过程每 5 分钟同步一次，因此新创建的机器人只会从下一步开始执行操作。 

当 t 太小而无法达到级别 1 时，就会出现另一种边缘情况。如果 n 很大而 t 很小，则根本不会生成任何单词，如果假设最终会生成单词，则很容易处理不当。 

此边缘情况的示例：

 输入：```
40 100 50
```在这里，每 5 分钟的步骤最多只能降低 1 级。在 50 分钟内，我们只能执行 10 步，因此 40 级机器人永远不会达到 1 级。 正确的输出为 0，但错误的连续或指数模型可能会错误地生成非零值。 

## 方法

 直接模拟会在每 5 分钟的步骤中跟踪每个级别存在多少个机器人。 在第 0 步，我们初始化 cnt[n] = k。 每一步，我们都会向下分配机器人：每个 i 级机器人都会产生一个 (i-1) 级机器人。 如果 i = 2，则产生 1 级机器人； 如果 i = 1，它会产生单词。 

这种方法是正确的，但必须谨慎实施，因为更新是同时发生的。 如果我们就地更新关卡，我们会意外地让新创建的机器人在同一步骤中执行操作，从而导致计数错误。 

瓶颈不是时间复杂度，而是清晰度：简单的模拟是 O(n * t)，这很好，因为 n ≤ 100 且 t ≤ 10^4，大约需要 10^6 次操作。 

然而，我们可以观察到更结构化的解释。 每个初始 n 级机器人都会生成一个组合级联：经过 t/5 个步骤后，它会在各个级别之间形成类似于二项式系数的分布。 具体来说，每个机器人都会有效地经历一个过程，在每个步骤中，它要么停留在其级别，要么在确定性链中向下移动一个级别。 在恰好 d 个步骤后达到级别 1 的方式数量对应于选择哪些步骤降低级别。 

因此，每个机器人都会贡献一个组合计数，计算出成为单词所需的路径数。 这将问题简化为将二项式系数求和到一定深度。 

我们可以逐级预先计算组合和聚合贡献，但最简单的稳定实现仍然是随时间和级别变化的 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 步模拟 DP | O(n * t) | O(n) | 已接受 |
 | 组合/二项式视图 | O(n * t + 预计算) | O(n * t) 或 O(n) 优化 | 已接受 |

 ## 算法演练

 我们将时间转换为离散的步骤，每个步骤为 5 分钟。 令 s = t // 5 为完整转换的数量。 

我们维护一个数组 dp，其中 dp[i] 表示当前步骤存在多少个 i 级机器人。 我们还为累积的 1 级输出维护一个变量词。 

1. 初始化 dp[n] = k 和所有其他 dp[i] = 0，并且 Words = 0。这反映了仅存在顶级机器人的初始状态。 
2. 对于从 1 到 s 的每个步骤，创建一个初始化为零的新数组 ndp。 我们将从当前状态计算下一个状态，而不混合更新。 
3. 对于从 n 到 2 的每个级别 i，将所有 dp[i] 移至 ndp[i-1]。 该模型对每个较高级别的机器人进行建模，每一步恰好生成一个较低级别的机器人。 
4. 对于第 1 级，所有 dp[1] 都直接对单词做出贡献，因此我们将 dp[1] 添加到单词中，但它们不会产生新的机器人。 
5.处理完所有级别后，将dp替换为ndp。 这样就完成了一个同步时间步长。 
6. 继续，直到处理完所有 s 个步骤，然后输出模 10^9 + 7 的单词。 

### 为什么它有效

 关键的不变性是，在每个步骤之后，dp[i] 正确地表示在该时间边界处可用的 i 级机器人的数量，而不受同一步骤更新的干扰。 因为每个机器人每一步只执行一次，并且只产生一个输出实体（较低级别的机器人或一个单词），所以这种转换是质量向下一级的完美线性转移。 这可以保持所有步骤的正确性，因为系统在当前计数之外是无记忆的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, k, t = map(int, input().split())
    steps = t // 5
    
    dp = [0] * (n + 1)
    dp[n] = k
    words = 0
    
    for _ in range(steps):
        ndp = [0] * (n + 1)
        
        for i in range(n, 1, -1):
            ndp[i - 1] = (ndp[i - 1] + dp[i]) % MOD
        
        words = (words + dp[1]) % MOD
        dp = ndp
    
    print(words % MOD)

if __name__ == "__main__":
    solve()
```实现完全遵循逐步的状态转换。 数组 dp 始终从头开始重建，以避免在同一迭代中意外重用更新的值。 反向循环确保了清晰度，尽管在此公式中顺序并不重要，因为每个 dp[i] 独立地对 ndp[i-1] 做出贡献。 

考虑到 k 可以大到 10^9 并且增长会在许多步骤中累积，因此始终应用模数以防止溢出。

 ## 边缘情况

 一种重要的边缘情况是 t 小于 5。在这种情况下，s = 0 并且不会发生任何转换。 该算法正确地保持 dp 不变并且单词保持为 0。 

另一个边缘情况是当 n = 1 时。所有初始机器人每一步都会立即生成单词。 DP 在每次迭代中正确地将 dp[1] 累积到单词中，从而产生时间上的线性增长。 

第三种情况是 n 比 t 大。 DP 永远不会达到较低的水平，因此所有中间 dp 值都保持在较高的指数直到耗尽，并且单词始终保持为零。
