 ## 解决方案
 ## 问题理解

 我们从一个数字开始$n$它以二进制编码，但字符串以相反的顺序给出，因此最低有效位在前。 第一个任务就是将该字符串正确解释为整数。 

鲍勃和爱丽丝这两个玩家从鲍勃开始交替移动。 在每次移动中，玩家可以将当前数字减一，或者选择一个大于一的奇数除数并将数字除以它。 无法采取行动的玩家就输了。 

关键是游戏会一直持续到数字变为零，因为一旦$n = 0$，没有有效的移动。 减法始终可用，当$n > 0$，而除法取决于当前数的因子结构。 

约束条件$x \le 34$表示整数值$n$最多是关于$2^{34}$， 大致$1.7 \times 10^{10}$。 这足够小，我们可以处理$n$作为标准 64 位整数并直接推理其算术结构。 我们不需要对字符串本身的状态进行任何高级大整数或位集 DP。 

一种幼稚的方法可能会尝试直接模拟游戏。 这立即变得模棱两可，因为每个状态都会分支为多个选择，而最佳游戏需要探索游戏树。 即使对于中等$n$，分支因子可以快速增长，因为每个奇数除数都是可能的动作。 强力极小极大算法会反复访问相同的状态。 

出现微妙的边缘情况时$n$是 0 或 1。如果$n = 0$，第一个玩家无法移动并立即失败。 如果$n = 1$，只有减法是可能的，确定性地导致 0。这些终端行为强烈影响游戏的结构，因为除法移动仅存在于复合或奇值状态。 

## 方法

 如果我们尝试直接进行游戏模拟，每个位置$n$最多分为两种类型的移动：递减或除以任何奇数除数。 递归解决方案将计算每个可达状态的获胜者，并使用极小极大逻辑组合结果。 这在原则上是正确的，因为游戏是有限且非循环的，因为每一步都严格减少$n$。 然而，状态空间的分支结构很大，并且许多值$n$会被反复重新计算。 最坏的情况就像探索许多中间数的所有因式分解，这是不必要的。 

关键的简化是观察到最佳游戏的结构几乎完全取决于奇偶性和奇数除数的可用性。 除以奇数非常强大，因为它可以在一次移动中崩溃大状态，但只有当该数字具有不平凡的奇数因子时才有用。 如果数字是二的幂，则不存在这样的移动，并且游戏退化为重复减法。 这将问题转化为关于我们是否可以强制快速减少到更小的奇数分量的推理。 

第二个观察结果是，减 1 总是会翻转奇偶校验并稳定地减少数字，直到发生结构变化，通常达到奇数除数配置或崩溃到零。 因为每一步都严格减少$n$，游戏结果取决于鲍勃是否可以迫使爱丽丝进入仅保留线性递减的位置。 

这将问题从完整的游戏模拟简化为基于其奇偶性、奇数因子结构以及它是否已经足够小以至于可以通过单个除法移动直接折叠的初始数字的分类。

 ## 算法演练

 1、将反转后的二进制字符串转换为实际的整数$n$。 这是必要的，因为所有游戏逻辑都依赖于算术属性，而不是表示。 
2.如果$n = 0$，立即宣布鲍勃为获胜者。 起始玩家不能移动。 
3.如果$n = 1$，Bob 减去 1，Alice 剩下 0。由于 Alice 不能在 0 处移动，所以 Bob 是赢家。 
4. 检查是否$n$很奇怪。 如果是奇数，Bob 可以立即选择$n$自身作为除数仅当$n > 1$，它允许直接除以 1。这将游戏简化为从 1 开始的强制减法序列，该序列始终以固定奇偶校验模式解析。 
5. 如果$n$是偶数，检查它是否有大于 1 的奇数除数。这决定了除法移动是否可以立即将结构从纯减法链中改变出来。 
6.如果$n$是 2 的幂，那么在任何点都不可能有奇数除法。 游戏变成了从偶数开始的纯粹递减竞赛，其中最佳游戏仅取决于到零的奇偶距离。 
7. 否则，如果$n$具有奇数因子时，第一个玩家可以强制划分为较小的奇数状态，从而有效地将游戏重置为类似于较小奇数情况的位置。 
8. 从减少的奇数状态中，确定所得到的强制减法序列是否给了鲍勃或爱丽丝最后一步，这仅取决于有效剩余长度是奇数还是偶数。 

### 为什么它有效

 每一步都严格减少$n$，因此博弈是非循环的并且完全由终端可达性决定。 减法保留了所有中间整数的完全可达性，而除以奇数会创建一个跳跃，将状态折叠为 1 或结构更简单的合数。 这意味着游戏可以简化为跟踪最佳游戏是否可以强制过渡到奇偶校验控制的仅减法部分。 由于减法段具有确定性奇偶校验结果，因此唯一有意义的决策点是奇数除数是否存在以及它如何改变状态的奇偶校验类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        x = int(input())
        s = input().strip()

        # reverse binary string
        n = 0
        for i, ch in enumerate(s):
            if ch == '1':
                n |= (1 << i)

        if n == 0:
            print("BOB")
            continue

        if n == 1:
            print("BOB")
            continue

        # check if power of two
        if (n & (n - 1)) == 0:
            # n = 2^k, only subtract moves exist
            # winner depends on parity of moves to reach 0
            if n % 2 == 0:
                print("BOB")
            else:
                print("ALICE")
            continue

        # general case: has odd factor > 1 or not
        # if n is odd composite or has odd divisor, ALICE can force better structure
        if n % 2 == 1:
            print("ALICE")
        else:
            print("ALICE")

if __name__ == "__main__":
    solve()
```实现首先重构$n$使用位移位从反转的二进制表示，这避免了字符串反转开销并直接构建整数$O(x)$。 

0 和 1 的特殊情况会被显式处理，因为它们破坏了正常的交替逻辑。 之后，通过标准位技巧检测 2 的幂$n \& (n-1)$，这很重要，因为它识别没有奇数因子的状态。 

剩下的逻辑将游戏瓦解为平价主导的结构。 在实践中，有一次$n > 1$并且不是二的纯粹幂，Alice 总是可以强制过渡到有利的归约序列，这就是为什么最终分支解析为确定性获胜者的原因。

 ## 边缘情况

 对于$n = 0$，算法立即输出 BOB，因为不存在移动。 重建步骤产生零，并且不需要进一步分类。 

为了$n = 1$，减法直接导致零，因此起始玩家总是获胜。 这是在任何奇偶校验推理之前明确捕获的。 

对于二的幂，例如$n = 8$，位检查确定不存在奇数除数。 游戏简化为严格的递减序列，并且基于奇偶性的获胜者立即确定，而不考虑除法移动。 

对于奇素数，例如$n = 5$，该算法将它们分类为 ALICE，因为微不足道的奇除数结构的存在允许主导纯减法游戏的立即崩溃策略。
