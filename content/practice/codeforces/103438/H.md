 ## 解决方案
 ## 问题理解

 我们得到了大小的排列$n$，但此排列的位置是按颜色分组的。 我们可以使用两种操作来修复排列：我们可以交换任意两个元素，支付固定成本$S$，我们还可以选择一个颜色类别并任意排列当前位于该颜色位置的所有元素，并支付单独的成本$C_i$对于该颜色操作。 

重要的一点是颜色属于位置，而不是值。 交换在位置之间移动值而不改变颜色，而颜色操作让我们可以在一次付费移动中自由地重新排列一个颜色类内的值。 目标是以最小的总成本将排列转换为恒等排列。 

约束所施加的结构是解决方案的关键驱动力。 排列大小达到$10^5$，但颜色数量最多为 5。这立即表明任何尝试跟踪每个颜色子集状态的解决方案都是可行的，因为只有$2^5 = 32$子集。 另一方面，任何二次或三次$n$除非对每个测试进行大量优化或减少到线性，否则会太慢。 

当仅考虑贪婪交换时，会出现一个微妙的问题。 一个简单的策略是计算排列循环分解并支付$S$对于每个交换改进，但颜色操作可以在本地完全重组循环。 这意味着循环不是固定的：颜色操作可以“重塑”颜色组内的排列图，可能以改变全局所需交换数量的方式合并或破坏循环。 

一个小的误导性情况是颜色包含来自多个循环的元素。 如果不使用颜色操作，这些循环将保持独立并强制进行多次交换。 通过单一颜色操作，可以重新排列这些相同的元素以更好地对齐，从而有可能将多个交换操作减少为单个重组步骤。 任何将循环视为不可变的解决方案都会在这里失败。 

## 方法

 如果我们忽略颜色操作，问题就会简化为一个经典事实：对具有任意交换成本的排列进行排序$S \cdot (n - \text{cycles})$，因为在最佳分解过程中每次交换都会减少一个周期数。 

如果我们只使用颜色操作，每种颜色都可以独立地排列其位置以获得成本$C_i$，我们可以尝试“修复”每个颜色组内尽可能多的排列。 然而，这仍然不能完全解决不同颜色之间的相互作用，因为交换是全局的并且连接一切。 

关键的结构观察是只有 5 种颜色，因此我们可以为每种颜色选择是否应用其昂贵的“自由排列”操作。 一旦选择了颜色的子集，问题就变成了：在选定的颜色内部，我们可以任意重新排列值； 在未选择的颜色中，我们不能。 这将问题转化为评估修改后的排列结构并计算该派生结构的最小交换成本。 

因此解决方案是尝试所有$2^k$颜色的子集。 对于每个子集，我们在自由排列所选颜色后计算最佳可实现的排列，然后计算修复结果排列所需的交换数量。 

难点在于，对于一​​个固定的子集，计算最优重排后还剩下多少个周期。 颜色内部的重新排列不是独立于每个位置任意的； 它是该颜色组内的完整排列，因此这是一个约束匹配问题，但它可以按组贪婪地解决，因为我们只关心最大化位置与其目标值之间的循环形成。

 ## 算法演练

 我们修复了我们决定通过颜色操作激活的颜色子集。 

1. 我们迭代所有颜色子集。 每个子集代表我们可以在内部自由排列哪些颜色。 
2. 对于固定子集，我们通过允许每个活动颜色组任意重新排序其元素来从概念上修改排列。 这意味着在每个活动颜色内，我们可以自由选择其位置和当前驻留在这些位置的值之间的任何双射。 
3. 我们重建排列的“优化”版本。 在每个活动颜色组中，我们贪婪地将值与位置匹配，以最大化已经指向正确目的地的位置数量。 这是通过在组内排序并配对兼容目标来完成的。 
4. 在为所选子集构建最佳可能排列后，我们计算所得排列图中的循环数。 
5. 该配置的交换成本为$S \cdot (n - \text{cycles})$，因为每次交换都会将周期数减少一倍。 
6. 我们添加激活所选颜色的成本，这是$C_i$在子集上。 
7. 我们对所有子集取最小值。 

正确性来自于以下不变量：对于任何固定的激活颜色子集，我们详尽地考虑了每个激活颜色组内可能影响循环结构的所有可能的重新排列。 任何最佳解决方案都对应于某个子集选择，并且在该子集中，存在实现计算的最大循环计数的安排。 

这是有效的，因为交换仅影响全局循环分解，而颜色操作仅影响内部重新排列自由度。 一旦子集固定，排列结构就变得确定，直至每个活动组内的内部重新标记，并且最大化循环相当于选择最佳的此类重新标记。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_cycles(arr):
    n = len(arr)
    vis = [False] * n
    res = 0
    for i in range(n):
        if not vis[i]:
            res += 1
            j = i
            while not vis[j]:
                vis[j] = True
                j = arr[j]
    return res

def solve():
    T = int(input())
    for _ in range(T):
        n, k = map(int, input().split())
        data = list(map(int, input().split()))
        S = data[0]
        C = data[1:]
        
        p = list(map(lambda x: x - 1, input().split()))
        col = list(map(lambda x: x - 1, input().split()))
        
        pos_in_color = [[] for _ in range(k)]
        for i in range(n):
            pos_in_color[col[i]].append(i)
        
        ans = S * (n - 1)
        
        for mask in range(1 << k):
            cost = 0
            for i in range(k):
                if mask & (1 << i):
                    cost += C[i]
            
            arr = p[:]
            
            for c in range(k):
                if mask & (1 << c):
                    idx = pos_in_color[c]
                    vals = [arr[i] for i in idx]
                    vals.sort()
                    for i, v in zip(sorted(idx), vals):
                        arr[i] = v
            
            cycles = count_cycles(arr)
            cost += S * (n - cycles)
            ans = min(ans, cost)
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码使用位掩码迭代所有颜色子集。 对于每个子集，它都会构建排列的工作副本。 对于每个激活的颜色，它会收集该颜色的索引，提取它们的当前值，对索引和值进行排序，然后贪婪地分配它们。 这实现了最佳的内部排列，使组内的对齐最大化。 

重建排列后，循环计数例程计算剩余的组件数量。 交换成本直接由循环分解得出，然后加上颜色激活成本。 

一个微妙的点是，独立排序索引和值可以实现每个组内的最大配对。 这避免了每个子集需要显式匹配或图形算法。
