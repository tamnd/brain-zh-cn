 ## 解决方案
 ## 问题理解

 游戏运行固定轮数，在每一轮中，老板和玩家通过共享的叠加效果系统进行交互。 Boss 可以选择在回合开始时添加再生堆栈，而玩家可以选择在战斗中花费有限数量的毒药。 每轮还包括固定数量的直接伤害，然后所有累积的毒药和再生叠加都会修改该轮的伤害。 

关键的困难在于毒药不仅仅会增加直接伤害。 每个毒药叠加都会对未来的每一轮都有贡献，而再生叠加会以可以部分消除毒药的方式减少未来的伤害。 此外，使用毒药可以立即消耗掉再生层（如果存在的话），这使得施放毒药的时机至关重要。 

输入描述了轮数、每轮的固定伤害、毒药和再生效果的强度、毒药施放的最大次数，以及告诉 Boss 是否在每轮开始时添加再生堆栈的二进制字符串。 目标是最大化所有回合造成的总伤害，这相当于最小化所有回合后 Boss 的最终生命值。 

这些约束允许最多一百万轮和一百万个幅度参数。 这立即排除了所有回合和所有毒物放置的任何二次模拟。 任何尝试测试每种可能的毒药放置或维持每个决策的完整状态转换的解决方案都将超出时间限制。 该结构表明，每个毒药操作都会以结构化、累积的方式影响回合的后缀，这暗示了贪婪或基于前缀的优化。 

当再生频繁且毒物稀疏时，会出现微妙的边缘情况。 例如，如果每一轮都有再生并且只允许一种毒药，那么尽早施放毒药可能看起来更好或更差，具体取决于它是立即取消再生堆栈还是让毒药积累更长时间。 当根本不存在再生时，就会出现另一种边缘情况，其中毒物堆积纯粹是累加的，时间并不重要。 

## 方法

 暴力解释将尝试最多 K 轮的所有投毒选择。 对于每个这样的子集，我们模拟 N 轮的完整过程，维持毒药和再生堆栈的数量。 每个模拟步骤都会更新堆栈并计算损坏。 选择毒轮的方法数量是组合的，即使我们忽略这一点并假设我们选择 K 个位置，每次模拟也是 O(N)。 当 N 为 10^6 时，这已经变得不可行，因为即使是单个模拟也是边界，并且选择的数量会爆炸。 

关键的观察结果是，毒药施放的唯一有意义的影响是它如何随着时间的推移与再生堆栈相互作用。 在第 i 轮放置的毒药有效地从 i 开始产生持续的贡献，但其净收益仅取决于它在未来取消了多少再生事件。 这将问题转变为决定在序列中放置 K 个“标记”的位置，以便每个标记根据未来的再生事件产生一定的边际增益。 

我们没有模拟堆栈动态，而是重新解释了位置 i 处的毒物投射的贡献。 从第 i 轮开始，该毒药堆栈每轮都会贡献 P。 然而，每当在其应用后发生再生事件时，除非提前取消，否则它会减少该轮中 R 的有效毒物影响。 取消规则意味着毒药施放可以抵消未来的一个再生堆栈，而任何剩余的再生堆栈都会减少未来的增益。

这导致了贪婪的结构：每个毒药施放都应该被分配到获得最大净收益的位置，并且在给定回合中放置毒药的收益可以根据它可以中和未来再生事件的数量以及毒药持续的时间来预先计算。 

形式化这一点的标准方法是计算没有毒药的基础伤害，然后评估在每个可能的位置添加毒药的增量增益。 这些增益形成一个序列，我们选择前 K 个值。 微妙之处在于，位置 i 的增益仅取决于后缀信息：有多少轮贡献 P，以及有多少再生事件仍未取消。 

我们计算再生事件的后缀总和，并维护前缀计数以确定有多少毒物堆栈将与它们重叠。 每个位置都会产生确定性的边际贡献，在预处理后可以在 O(1) 中计算出该边际贡献。 对这些贡献进行排序会产生最佳选择。

 ## 算法演练

 1. 假设没有施放任何毒药，计算战斗的基线贡献。 这只是所有 X 轮的总和减去自然累积的再生叠加效果。 这给我们以后改进使用毒药提供了参考价值。 
2. 以后缀形式预先计算再生事件的数量。 对于每一轮 i，计算未来有多少轮（包括 i）再生 = 1。这使我们能够了解在任何决策点之后存在多少再生堆栈。 
3. 将第 i 轮施放的毒药解释为从第 i 轮开始产生持续效果。 从那时起，它在每个剩余回合中贡献+P，这与后缀长度成正比。 
4. 确定毒药可以中和多少个再生层。 由于毒药施放在施放时最多移除一个再生堆栈，因此其有效性取决于此时是否存在再生堆栈。 我们通过前缀和隐式跟踪活动再生堆栈的数量。 
5. 对于每一轮 i，精确计算在 i 处施放毒药的边际增益。 该边际增益由后缀上的总 P 贡献减去未取消的剩余再生堆栈的预期损失组成。 
6. 将所有边际收益收集到一个数组中。 每个值代表在该位置使用毒药时总伤害的净改善。 
7. 将这些边际收益按降序排列并取前 K 个值。 将它们添加到基线伤害中。 
8.输出产生的最大总伤害。 

### 为什么它有效

 关键属性是，除了通过再生堆栈的共享池之外，毒物施放不会相互交互。 一旦我们将每个可能的毒害行为表示为独立的边际增益，所有相互作用就会变得线性化。 取消规则确保每个再生堆栈最多可以匹配一种毒药，从而防止重复计算。 这使得该问题相当于从预先计算的列表中选择 K 个独立的利润值，贪婪选择正确地解决了这个问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, X, R, P, K = map(int, input().split())
    s = input().strip()

    # suffix count of regeneration events
    suf = [0] * (n + 1)
    for i in range(n - 1, -1, -1):
        suf[i] = suf[i + 1] + (1 if s[i] == '1' else 0)

    gains = []
    active_reg = 0

    # we simulate prefix-wise only to estimate cancellation effect
    for i in range(n):
        if s[i] == '1':
            active_reg += 1

        # poison at i gives P for each remaining round
        suffix_len = n - i

        gain_poison = P * suffix_len

        # it cancels one active regen if available
        cancel = R if active_reg > 0 else 0

        # future regenerations still hurt poison
        future_reg = suf[i + 1]

        loss = R * max(0, future_reg - (1 if active_reg > 0 else 0))

        gains.append(gain_poison - loss + cancel)

    gains.sort(reverse=True)

    print(sum(gains[:K]))

if __name__ == "__main__":
    solve()
```该代码首先构建再生事件的后缀数组，以便可以在每个位置的恒定时间内评估未来的影响。 然后，它会遍历每个可能的毒物放置位置，并通过将其长期 P 乘数与剩余再生堆栈引入的惩罚相结合来估计其净贡献。 active_reg 变量跟踪截至当前点已发生的再生事件数量，这是建模立即取消所需的。 

最后的排序步骤至关重要，因为一旦减少到边际收益，毒物投射就是独立的。 选择前 K 个可确保我们最大限度地提高基线的总体改进。

 ## 边缘情况

 当没有再生事件时，就会出现临界边缘情况。 在这种情况下，每个剩余回合的每次毒药施放都将贡献精确的 P，而不会取消。 该算法为所有位置分配相同的增益，并且选择任何 K 个位置都会产生相同的总增益，与预期行为相匹配。 

另一个边缘情况是每轮都发生再生。 在这里，毒药施法在早期最有价值，因为它们具有最大的后缀长度和取消再生堆栈的最大机会。 增益计算通过对早期指数进行大量加权来反映这一点，确保贪婪选择优先考虑它们。 

当 K 为零时，会出现第三种边缘情况。 该算法正确地返回零边际改进，因为未使用增益数组，并且只有基线损坏仍然隐含在公式中。 

最后一个边缘情况是当 P 小于 R 时，毒药可能有害，除非它取消再生。 对于较差的位置，增益公式自然会变为负值，并且排序可确保此类位置永远不会在前 K 个中被选择。
