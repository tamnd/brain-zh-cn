 ## 解决方案
 ## 问题理解

 我们得到了纸张尺寸的层次结构$A_0$下降到$A_N$，其中每个级别的大小恰好是前一个级别的一半。 您从每种尺寸的板材的一些初始库存开始，并且您还有一个想要实现的目标库存。 唯一允许的操作是取一张尺寸为$A_i$，将其切成两片大小$A_{i+1}$，并递归地重复此过程。 

任务是确定将初始库存至少转变为每个尺寸级别所需的库存所需的最小切割次数，或者确定这是不可能的。 

每次切割都会严格地将一个单位的纸张向下移动一级，同时将其计数加倍。 这就产生了从较大尺寸到较小尺寸的强烈方向依赖性，这意味着较高水平的盈余总是可以“细化”到较低水平，但反之则不然。 

约束条件$N \le 2 \cdot 10^5$意味着任何解必须接近线性或$O(N \log N)$。 跨级别重新分配的二次模拟可能涉及在每个指数的所有级别上传播过剩，当$N$很大。 

当剩余水平较高而需求水平较低时，就会出现一个关键的微妙之处。 如果不能以结构化方式正确向下传播容量，则仅检查局部差异的天真贪婪就会失败。 

一种重要的失败情况是本地转换似乎足够，但全局传播却不够：

 输入：```
N = 2
a = [0, 0, 2]
b = [2, 0, 0]
```在这里，我们有两个$A_2$床单但需要两张$A_0$床单。 正确答案是不可能的，因为每个$A_2$只生产$A_1$，并且需要进一步削减，但天真的推理可能会错误地匹配计数，而无法正确跟踪转换深度。 

另一个微妙的情况是中间级别必须充当缓冲区。 忽视它们会导致低估所需的削减。 

## 方法

 蛮力视角独立处理每张纸。 我们模拟重复选择任何有盈余的水平，并一次减少一个水平，直到所有赤字都得到解决。 每次操作都会更新全局计数，当满足所有需求或不存在更多有益的削减时，我们就会停止。 

这种方法是正确的，因为它准确地反映了实际过程。 然而，最坏的情况是灾难性的。 单张纸位于$A_0$可能会被反复砍断$N$水平，并且可以有$O(N)$这样的床单。 这导致$O(N^2)$最坏情况下的行为，这是不可能的$2 \cdot 10^5$。 

关键的观察是我们不需要模拟单独的切割。 每张纸水平$i$如果完全分解，则对所有较低级别有确定性贡献。 我们没有考虑离散操作，而是将剩余视为向下流动的流量。 切割次数正是我们在一个单元通过关卡时将其分割的次数。 

我们从顶层向下处理，保持有多少多余的纸张可以向下传送。 在每个级别，我们首先使用可用库存加上传入盈余来满足需求。 任何剩余的盈余都会向下传递，每传递一个单位，就会在该级别贡献一份削减。

 ## 算法演练

 我们将这个过程视为来自$A_0$下降到$A_N$，向下承载过剩产能。 

1. 从零结转盈余开始。 这表示已从较高级别转换为当前级别的工作表。 
2. 每个级别$i$从 0 到$N$，计算该级别可用的总数为$a_i + \text{carry}$。 这是所有可用的纸张，可以满足需求或进一步拆分。 
3. 如果总数小于$b_i$，这个级别是不可能满足需求的，所以我们立即返回$-1$。 这是因为未来的操作无法使纸张向上移动。 
4. 否则，我们使用$b_i$以满足需求并保留剩余部分。 
5. 每张多余的纸都处于水平状态$i$要么保持在这个水平，要么进一步削减。 为了最大限度地减少削减，我们只削减将盈余向下传递所必需的部分。 每个单元从水平向下传递$i$代表一刀切。 
6. 我们将所有剩余盈余添加到结转水平中$i+1$。 这种进位隐含地代表了已经切割的碎片，因此除了计算我们通过了多少个单位之外，不需要额外的簿记。 
7. 累积的切割次数恰好是各级向下流动的材料总量。 

### 为什么它有效

 关键的不变量是在每个级别$i$，我们从不延迟需求满足：我们总是在向下传递任何东西之前消耗可用的材料。 任何从水平面移动的单位$i$到$i+1$必须在级别上恰好被分割一次$i$，这是该阶段唯一对该单元的削减计数有贡献的时刻。 

由于流动总是向下且不可逆转，因此每个盈余单位在其通过的每个级别上都会被精确计算一次。 这保证了向下传输的总数等于最小切割次数，因为任何替代策略都需要至少相同数量的分割才能实现相同的分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    carry = 0
    cuts = 0
    
    for i in range(n + 1):
        available = a[i] + carry
        
        if available < b[i]:
            print(-1)
            return
        
        remaining = available - b[i]
        
        cuts += remaining
        carry = remaining
    
    print(cuts)

if __name__ == "__main__":
    solve()
```该实现维护两个变量：`carry`，它存储了从之前的水平向下推了多少张纸，以及`cuts`，累积向下分割的总数。 

在每个级别，我们都会结合原始库存和进货进货。 如果这不足以满足需求，该功能会提前停止。 否则，我们计算盈余并将其向下推。 每个单位的盈余都对应于这一水平的削减，因为它必须被分割才能进一步进行。 

一个微妙的细节是，我们在计算切割次数时不区分原始纸张和携带纸张。 这是正确的，因为一旦达到一个级别，两者就代表相同的单位。

这表明早期盈余可以部分向下传播，但在全球范围内仍然不足。

 ## 边缘情况

 一种极端情况是所有需求都集中在顶层。 在这种情况下，下面的任何盈余都是无关紧要的，也无济于事。 该算法会立即检测到需求超过可用供应的第一级故障，因为尚不存在利差。 

当高水平存在大量盈余但较低水平的需求温和时，就会出现另一种边缘情况。 该算法通过逐步向下传递盈余来正确处理此问题，确保每个单元在每次级别转换时准确地贡献一次削减。 

最后一个微妙的情况是，单独携带可以满足零直接供应水平的需求。 该算法统一对待进位和原始供给，因此不需要区分。 正确性来自于以下事实：两者都代表已经转换到该级别的相同单位。
