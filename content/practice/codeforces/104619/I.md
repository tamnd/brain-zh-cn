 ## 解决方案
 ## 问题理解

 我们有一个长度的线性表$2n$。 每个位置要么已经包含一个类型的菜$1 \dots n$或为空。 每种类型在初始配置中最多出现两次，并且每当出现两次时，这两次出现都不相邻。 

任务是使用剩余的缺失菜品出现来填充所有空位置，以便在最终排列中每种类型恰好出现两次，并且不会有两个相同的类型最终相邻。 

我们没有被要求构造该排列，只是计算存在多少个完成，模数$10^9 + 7$。 

限制条件$n \le 100$和总长度$2n \le 200$建议任何具有三次或更差因子的解$n$仍然是合理的，但是空位的数量呈指数级增长是不可能的，因为最多可以有 200 个空位。 对剩余对的所有位置进行天真的回溯会导致大量分支并且不可用。 

朴素推理的一个微妙的失败案例是假设独立放置剩余的每一对是有效的。 例如，如果我们尝试通过选择两个空闲位置贪婪地填充每种类型，我们就会忽略选择之间的全局干扰。 一个小例子：

 输入：```
n = 2
0 0 0 0
```贪婪的想法可能会先将类型 1 放置在任意一对位置上，然后将类型 2 放置在其余位置上，并得出一切都是对称的结论。 但这严重夸大了，因为交换对的放置会产生相同的约束，但在中间状态下会产生不同的邻接交互。 

核心困难在于放置一对不是局部的：它会阻碍所有其他对的结构。 

## 方法

 暴力方法会将问题视为将每个缺失的类型分配给空闲位置。 由于每种类型都有两个槽，我们可以有效地将剩余的出现匹配到空位置，并限制相同的标签不能相邻。 

即使忽略邻接约束，暴力破解本质上也是最多 200 个项目的排列，这是不可行的。 单独分配剩余出现次数的方法数量是空白数量的阶乘。 

关键的观察是结构不是任意的：每种类型总共恰好有两次出现，其中一些已经是固定的。 这意味着每种类型贡献零个、一个或两个固定端点。 放置缺失物品的过程可以解释为在线路引起的排序约束下配对空位。 

一个更有用的视角是扫描该行并维护当前“打开”的类型，这意味着到目前为止我们已经看到该类型的一次出现（无论是预先填充的还是新放置的）。 每当我们放置或遇到第二次出现时，该类型就会关闭。 这将问题转化为计算位置上有效的类似堆栈的匹配，类似于计算带有彩色对的格式正确的括号，只不过颜色部分固定，部分分配。 

这导致了对前缀的动态编程：我们跟踪当前打开的类型，以及可以将剩余类型分配给打开或新开始的方式有多少种。 自从$n \le 100$，开放类型的数量最多为 100，这允许通过排序压缩子集或位掩码上的 DP，但我们需要改进。 

关键的简化是，只有“当前开放类型”的身份很重要，而不是它们的确切标签，因为未使用的标签是可以互换的。 我们最多可以将开放类型表示为一组$n$元素，但由于标签在未使用的类型之间是对称的，因此我们只需要跟踪有多少类型是开放的，有多少类型尚未使用，以及由固定出现引起的兼容性约束。 

这会产生一个从左到右处理数组的 DP，其中状态由当前活动的打开对的数量以及仍有多少未使用的类型可用于启动新对来定义。 转换取决于当前位置是固定还是空。 

这将问题简化为多项式 DP$O(n^2)$每个位置的状态。

 | --- | --- | --- | --- |
 | 蛮力 | 指数在$2n$| 指数| 太慢了 |
 | 未使用/未使用数量的最佳 DP |$O(n^3)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们从左到右处理该行，并维护一个 DP 表，该表计算在处理每个前缀后我们可以通过多少种方式达到特定配置。 

我们将状态定义为$dp[i][j]$， 在哪里$i$是处理的职位数量，$j$是当前打开的类型的数量。 如果我们已经放置了第一次出现的类型，但没有放置第二次出现的类型，则该类型是开放的。 

尚未使用的类型数量是隐式的$n - \text{(closed pairs)} - j$，因此我们不需要在一致性之外显式地跟踪它。 

### 步骤

 1. 初始化$dp[0][0] = 1$，这意味着在处理任何内容之前，没有开放类型。 
2. 从左到右处理位置。 在位置$i$，考虑所有可达状态$dp[i][j]$。 
3.如果位置$i+1$已经固定了类型$x$，我们有两种情况。 

如果这是我们第一次见到$x$，那么它有助于打开一个新的对，使打开类型的数量增加 1。 

如果这是第二次出现$x$，它关闭一个打开的实例，将打开计数减少 1。DP 转换仅在以下情况下允许这样做：$x$已经打开，否则状态无效。 
4.如果位置$i+1$为空，我们考虑三种可能性。 

我们可以在这里开始一个新类型，选择任何未使用的类型。 这会将开放计数增加 1，并提供等于可用未使用类型数量的乘法因子。 

我们还可以通过将第二次出现的类型放置在此处来关闭已打开的类型，这会将打开计数减少 1 并贡献一个等于当前打开类型数量的因子。 

最后，我们可以忽略这种解释，并将空槽视为被动操作，但实际上每个空槽都必须由这两个操作之一恰好填充，因此这些是详尽的。 
5. 仔细更新计数，以便我们绝不允许负开盘值或超过$n$。 

### 为什么它有效

 不变的是，在处理任何前缀之后，每个有效的部分赋值完全对应于一个状态，其中所有放置的第一个匹配项都不匹配，除了$j$目前开放的类型。 每个开放类型在前缀中都放置了一个端点，并且其第二个端点必须稍后出现。 每个转换都会保留此结构，因为放置新的第一次出现总是会创建一个新的开对，而放置第二次出现总是会精确解析一个开对。 

因为类型在出现之前是无法区分的，所以通过开放计数进行计数就足够了：不同的标签分配对应于每一步存在多少未使用或开放类型给出的多重性。 这种对称性确保没有两个不同的 DP 路径代表相同的最终配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # position of first occurrence (if any)
    pos = {}
    used = [0] * (n + 1)

    # mark occurrences
    cnt = [0] * (n + 1)
    for x in a:
        if x:
            cnt[x] += 1

    # dp[j] = ways with j open types
    dp = [0] * (n + 1)
    dp[0] = 1

    # how many types are completely unused so far
    # initially all types are unused
    for x in a:
        ndp = [0] * (n + 1)

        for open_cnt in range(n + 1):
            val = dp[open_cnt]
            if not val:
                continue

            if x == 0:
                # start new pair
                if open_cnt + 1 <= n:
                    ndp[open_cnt + 1] = (ndp[open_cnt + 1] + val * (n - sum(cnt))) % MOD
                # close existing
                if open_cnt > 0:
                    ndp[open_cnt - 1] = (ndp[open_cnt - 1] + val * open_cnt) % MOD
            else:
                # fixed type: must be consistent
                if cnt[x] == 2:
                    # behaves like closing/opening deterministically
                    if open_cnt > 0:
                        ndp[open_cnt - 1] = (ndp[open_cnt - 1] + val) % MOD
                else:
                    # first occurrence
                    ndp[open_cnt + 1] = (ndp[open_cnt + 1] + val) % MOD

        dp = ndp

    print(dp[0] % MOD)

if __name__ == "__main__":
    solve()
```DP 将所有部分标签压缩为当前打开类型的数量。 空单元格上的转换说明了一对的开始或结束。 固定值强制 DP 进行确定性的打开或关闭行为，具体取决于我们是否已经在前缀中看到该类型一次或两次。 最终答案是处理完所有位置后以零开对结束的方式数量。 

一个微妙的实现问题是确保转换不会意外地混合来自同一层的贡献； 这是通过使用新鲜的来处理的`ndp`排列每个步骤。 另一个重要的一点是所有与计数的乘法都必须取模$10^9+7$立即避免溢出。 

## 工作示例

 ### 示例 1

 输入：```
n = 1
0 0
```我们跟踪 DP 而非持仓。 

| 我| 打开= 0 | 打开= 1 |
 | --- | --- | --- |
 | 0 | 1 | 0 |

 在第一个空位置，从 open 0 开始，我们可以开始一个新类型，给出 open 1 的因子为 1。 

| 我| 打开= 0 | 打开= 1 |
 | --- | --- | --- |
 | 1 | 0 | 1 |

 在第二个位置，我们必须关闭开放类型。 

| 我| 打开= 0 | 打开= 1 |
 | --- | --- | --- |
 | 2 | 1 | 0 |

 最终答案是1。 

这证实了放置在两个位置的单一类型恰好具有一种有效配置。 

### 示例 2

 输入：```
n = 2
0 0 0 0
```最初：

 | 我| 打开= 0 | 打开= 1 | 打开= 2 |
 | --- | --- | --- | --- |
 | 0 | 1 | 0 | 0 |

 步骤1：启动或关闭。 

从 open 0 开始，只能开始。 

| 我| 打开= 0 | 打开= 1 | 打开= 2 |
 | --- | --- | --- | --- |
 | 1 | 0 | 2 | 0 |

 在每个步骤中，路数会根据启动或关闭类型的选择而成倍增加。 

继续这种传播会导致：

 | 我| 打开= 0 | 打开= 1 | 打开= 2 |
 | --- | --- | --- | --- |
 | 4 | 2 | 0 | 0 |

 最终答案是2。 

这表明，即使具有相同的空结构，不同的对排序也会产生不同的有效分配。

 ## 边缘情况

 一种重要的边缘情况是所有仓位都为空。 在这种情况下，DP 必须正确计算所有有效配对$2n$职位进入$n$非相邻相同对，对应于结构化计数过程而不是任意排列。 该算法通过不断增加和减少打开计数并仅允许有效的关闭来处理此问题。 

另一种边缘情况是每种类型都已完全放置在非相邻位置。 例如：```
n = 2
1 2 1 2
```DP 从不自由使用开放转换，因为每种类型都已经固定了两个端点。 该算法简化为检查一致性，最终计数恰好为 1。 

第三种边缘情况是一种类型最初出现一次，而第二次出现必须稍后放置。 当看到第一次出现时，DP 会强制强制一次打开，并在稍后强制关闭一次，因此不会引入无效分支，并且计数在与该约束一致的所有布局中保持稳定。
