 ## 解决方案
 ## 问题理解

 我们得到一个整数$n$代表 Jerry 派出多少个 ICPC 团队参加比赛。 每个团队需要缴纳固定报名费4000美元。 任务是计算注册所有团队所需的总金额。 

输入不涉及超出此单一计数的任何结构，输出只是按每个团队费用缩放后的总成本。 

约束条件$n \le 20$意味着输入尺寸非常小。 这立即排除了对效率的任何担忧，典型整数范围的溢出在 Python 中也不是问题，甚至在固定宽度整数语言中，最大值$20 \cdot 4000 = 80000$是微不足道的。 

要注意的主要边缘情况是尽可能小的输入。 自从$n$是一个正整数，最小的情况是$n = 1$，应该产生 4000。任何不相上下的想法，例如治疗$n$因为从零开始或错误地添加额外的常量会立即出现在这种情况下。 

## 方法

 暴力解释实际上是逐队模拟注册过程，为每个队添加 4000。 这看起来像是将总计初始化为零并循环$n$次，每次增加4000。 这是正确的，因为每个团队都贡献独立的固定成本，因此积累与乘法相匹配。 

这种方法运行在$O(n)$，这已经可以忽略不计了$n \le 20$。 然而，问题的结构使得重复添加变得不必要。 由于总和中的每一项都是相同的，因此整个计算会分解为直接乘法$4000 \cdot n$。 观察结果是，我们正在对一个常数序列求和，而不是处理变化的输入，因此我们可以用单个算术运算代替迭代。

 | --- | --- | --- | --- |
 | 蛮力（循环累积）|$O(n)$|$O(1)$| 已接受 |
 | 最优（直接乘法）|$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 我们以最直接的方式计算总成本。 

1.读取整数$n$来自输入。 这代表正在注册的团队数量。 
2. 计算总成本为$n \times 4000$。 此步骤直接应用每个团队成本的定义，无需模拟。 
3. 输出计算值作为最终答案。 

### 为什么它有效

 每个团队独立贡献 4000。 因此，总成本是$n$相同的项，在数学上等价于乘法。 由于团队之间没有条件规则、折扣或交互，因此除了扩展计数之外不需要其他逻辑。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    print(n * 4000)

if __name__ == "__main__":
    solve()
```该解决方案读取单个整数并立即将其乘以 4000。使用`strip()`确保任何尾随换行符不会干扰整数解析。 没有循环，没有条件，也没有边界问题的风险，因为计算是单个算术表达式。

 ## 边缘情况

 唯一有意义的边缘情况是最小的输入，因为没有结构变化或特殊规则。 

对于输入：```
1
```算法读取$n = 1$，计算$1 \times 4000 = 4000$，并输出 4000。没有迭代，因此不存在逐一循环错误或初始化错误的风险。 正确性直接来自单个乘法步骤，该乘法步骤已经与成本模型的定义相匹配。
