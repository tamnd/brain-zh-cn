 ## 解决方案
 ## 问题理解

 我们有两个独立的地区比赛，桃园和雅加达。 对于每场比赛，我们知道两个数据：重新计算的我们团队在该比赛中的排名以及总结该比赛整体实力和规模的“站点得分”。 我们的目标是决定在这两场比赛中哪一场我们晋级下一阶段的机会更大。 

关键思想是每次比赛都会为我们的团队产生一个单一的派生“力量值”。 该值取决于我们的排名相对于比赛规模的好坏（由网站得分捕获），因此两个输入都很重要。 较低的值代表更好的晋升机会。 

限制很小：排名始终低于 66，分数是有限范围内的浮点数，缩放后精度最多为两位小数。 这立即意味着任何正确的解决方案都不需要复杂的数据结构或搜索。 每个测试用例的一切都减少到恒定的算术量。 

唯一微妙的失败案例来自浮点精度。 幼稚的实现可能会直接使用浮点算术来比较计算出的比率，这可能会导致在边界情况下做出不正确的相等决策。 例如，两个几乎相等的值可能仅在最后一个二进制浮动错误上有所不同，从而产生错误的区域选择。 正确的做法是完全避免划分，进行叉积比较。 

## 方法

 强力解释将按字面意思计算由选择公式定义的每个区域的最终分数，该公式在概念上减少为形式排名除以站点分数的比率（直到恒定的变化）。 人们可以计算这两个值并直接比较它们。 

这在数学上是正确的，但它依赖于浮点除法两次然后进行比较。 由于两个站点得分均以十进制数字给出，因此重复除法会导致精度不稳定。 虽然问题的约束很小，但正确性取决于精确的排序。 

结构观察是我们只关心哪个比率更小。 如果我们有两个分数

 RT/ST 和 RJ/SJ，

 我们实际上从来不需要它们的十进制形式。 比较它们相当于比较叉积：

 RT * SJ 与 RJ * ST。 

这完全消除了浮点运算，并将问题转化为小产品之间的整数比较。 

这是关键的简化：整个 ICPC 规则描述分解为两个比率之间的单调分数比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（浮除）| O(1) | O(1) | O(1) | O(1) | 有风险（精度问题）|
 | 交叉乘法 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将每个区域的分数降低到根据排名和网站分数得出的可比分数。 比较两个区域时，精确的缩放常数会相互抵消，因此只有相对顺序才重要。 

1. 从输入读取 RT、RJ、ST、SJ。 这些代表了我们在两次比赛中的排名和网站得分。 
2. 将桃园的“机会值”解释为与 RT / ST 成正比，将雅加达的“机会值”解释为 RJ / SJ。 ICPC 规则的确切推导与比较无关，因为所有共享常量都被取消。 
3. 为了避免浮点除法，请比较 RT * SJ 和 RJ * ST，而不是直接比较分数。 这保留了排序，因为所有值都是正数。 
4. 如果 RT * SJ 较小，Taoyuan 产量较好（较小）的比率，因此输出 TAOYUAN。 
5. 如果RJ * ST 较小，则输出JAKARTA。 
6. 如果两个乘积相等，则机会相同，因此输出 SAME。 

关键的推理步骤是，两边都乘以正分母可以保留顺序，因此变换是无损的。 

### 为什么它有效

每次竞赛都会产生与排名除以站点分数成正比的分数。 由于两个站点得分都严格为正，因此我们可以安全地相乘而不改变不平等方向。 这将有理比较转换为整数比较。 因为两个表达式共享相同的结构，所以 ICPC 规则中的所有隐藏常量都被取消，只留下简单的比率比较作为不变量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    RT, RJ, ST, SJ = input().split()
    RT = int(RT)
    RJ = int(RJ)
    ST = float(ST)
    SJ = float(SJ)

    left = RT * SJ
    right = RJ * ST

    if abs(left - right) < 1e-12:
        print("SAME")
    elif left < right:
        print("TAOYUAN")
    else:
        print("JAKARTA")

if __name__ == "__main__":
    main()
```该实现遵循直接归约到交叉乘法。 尽管我们仍然读取浮点站点分数，但我们立即避免除法。 相等性检查使用小的 epsilon 保护，因为输入可能涉及十进制解析工件。 比较本身是整数浮点运算，因此它保持稳定。 

一个微妙的点是我们没有直接规范或简化 ICPC 规则； 尝试模拟它们既不必要又容易出错。 该结构保证只有相对比率才重要。 

## 工作示例

 考虑第一个样本：

 输入：

 RT = 1，RJ = 2，ST = 34.56，SJ = 56.78

 我们计算：

 | 步骤| 桃园 | 雅加达 |
 | --- | --- | --- |
 | 价值| 1 × 56.78 = 56.78 | 1 × 56.78 = 56.78 2 × 34.56 = 69.12 |

 由于 56.78 < 69.12，桃园较好。 

输出是桃园。 

现在是第二个示例：

 输入：

 RT = 2，RJ = 3，ST = 45.67，SJ = 98.01

 | 步骤| 桃园 | 雅加达 |
 | --- | --- | --- |
 | 价值| 2 × 98.01 = 196.02 | 3 × 45.67 = 137.01 | 3 × 45.67 = 137.01

 由于 137.01 < 196.02，雅加达更好。 

输出是雅加达。 

这些痕迹表明，决策仅取决于排名和站点得分之间的相对比例，而不取决于绝对量级。

 ## 边缘情况

 一种边缘情况是两个区域的比率实际上相同。 例如，RT = 2、ST = 10 且 RJ = 4、SJ = 20。两个比率均减少至 0.2。 该算法计算 2 × 20 和 4 × 10，均等于 40，正确生成 SAME。 这证实了在没有浮点敏感性的情况下保留了相等性。 

另一个边缘情况是排名相同但站点得分显着不同。 例如，RT = RJ = 1，ST = 1.00，SJ = 100.00。 交叉乘法的结果是 1 × 100 与 1 × 1，因此桃园获胜，这符合直觉，即较小的站点得分可以提高比率。 

最后一种情况是排名不同但网站得分几乎可以弥补。 因为我们从不除法，所以即使是极端的小数不平衡也无法因为精度而翻转顺序，从而确保所有有效输入的行为一致。
