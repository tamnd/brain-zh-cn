 ## 解决方案
 ## 问题理解

 我们给出一个正整数 α，它被定义为数字 x 及其倒数 1/x 之和。 换句话说，x 是满足 x + 1/x = α 的某个（可能是复数）数字。 根据这个隐式定义，我们需要计算 x^β + (1/x)^β 模 m 的值。 

关键的困难是 x 没有明确给出。 它仅通过二次关系来定义。 扩展定义，x 必须满足 x^2 - αx + 1 = 0，因此 x 是具有整数系数的二次多项式的根。 尽管 x 可能是无理数或复数，但最终表达式 x^β + x^{-β} 保证为整数。 

输出仅取决于 α、β 和 m，而不取决于选择哪个根。 如果 x 是一个根，则 1/x 是另一个根，交换它们不会改变 x^β + x^{-β} 的值。 

这些约束意味着 α、β 和 m 可以大到 2^64。 这立即排除了任何显式扩展幂或在 β 线性时间内工作的方法。 我们需要一种能够减少对数时间求幂并完全避免浮点运算的东西。 

当 α 很小时，会出现微妙的边缘情况。 例如，如果α=1，则根是复数1/2±i√3/2，并且直接数值计算变得不稳定。 另一个边缘情况是当 x = 1 或 x = -1 时，这种情况发生在扩展形式的 α = 2 或 α = -2 时，但这里 α 是正数，因此只有 α = 2 才会导致 x = 1 的重复根情况。 

核心挑战是在不明确求解二次方程的情况下计算二次方程根的对称幂和。 

## 方法

 直接解释将尝试根据二次公式 x = (α ± √(α^2 - 4)) / 2 计算 x，然后将其提高到 β 并与其倒数相加。 这种方法很脆弱，因为它涉及无理数或复数，并且需要高精度算术。 即使使用大浮点或符号计算来实现，对大 β 求幂也会太慢。 

关键的结构观察是 x^k + x^{-k} 的行为类似于线性递归。 如果我们定义 S_k = x^k + x^{-k}，则乘以 (x + x^{-1}) = α 得到递推：

 S_{k+1} = (x + x^{-1})(x^k + x^{-k}) - (x^{k-1} + x^{-(k-1)})

 这简化为：

 S_{k+1} = α S_k - S_{k-1}

 这种递推完全消除了 x。 整个问题简化为计算初始值 S_0 = 2 且 S_1 = α 的二阶线性递推的 β 项。 

一旦我们认识到这种结构，问题就变成了标准的快速加倍或矩阵求幂任务。 我们可以使用 2x2 矩阵求幂或用于类似斐波那契递归的快速加倍方法在对数时间内计算 S_β。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 使用根直接计算 | O(β) 或更差 | O(1) | O(1) | 太慢/数值不稳定 |
 | 通过快速求幂实现线性递推 | O(log β) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们用递归 S_k = x^k + x^{-k} 来重写该问题。 目标是计算 S_β。 

1.我们初始化与递归相对应的两个基值。 我们设置 S_0 = 2，因为 x^0 + x^0 = 1 + 1 = 2，并且 S_1 = α，因为 x + 1/x = α 是直接给出的。 这两个值完全决定了顺序。 
2. 我们观察到后面的每一项都可以使用 S_{k+1} = α S_k - S_{k-1} 从前两项生成。 这个关系来自于将 S_k 乘以 α 并展开 α = x + 1/x，然后重新组合 x 的幂。 
3. 我们不是迭代 β 次，而是使用快速求幂来计算 S_β。 我们将递归视为向量 (S_k, S_{k-1}) 上的线性变换。 
4.我们定义一个变换矩阵：$$\begin{pmatrix}
S_{k+1} \\
S_k
\end{pmatrix}
=
\begin{pmatrix}
\alpha & -1 \\
1 & 0
\end{pmatrix}
\begin{pmatrix}
S_k \\
S_{k-1}
\end{pmatrix}$$5. 我们对该矩阵求幂 β-1 并将其应用于初始向量 (S_1, S_0)。 得出 (S_β, S_{β-1})。 
6. 所有运算均以 m 为模，在减去 S_{k-1} 时小心处理负值。 

正确性来自于在步骤 k 处的不变量，向量 (S_k, S_{k-1}) 准确地表示 (x^k + x^{-k}, x^{k-1} + x^{-(k-1)})。 该转换保留了这种恒等式，因为它反映了乘以 x + x^{-1} 的代数展开式。 由于递推唯一确定所有 S_k，并且初始条件与真实值匹配，因此计算序列必须与所有 k 的所需表达式一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def mat_mul(a, b, mod):
    return [
        [(a[0][0]*b[0][0] + a[0][1]*b[1][0]) % mod,
         (a[0][0]*b[0][1] + a[0][1]*b[1][1]) % mod],
        [(a[1][0]*b[0][0] + a[1][1]*b[1][0]) % mod,
         (a[1][0]*b[0][1] + a[1][1]*b[1][1]) % mod]
    ]

def mat_pow(mat, exp, mod):
    res = [[1, 0], [0, 1]]
    while exp > 0:
        if exp & 1:
            res = mat_mul(res, mat, mod)
        mat = mat_mul(mat, mat, mod)
        exp >>= 1
    return res

def solve():
    alpha, beta, m = map(int, input().split())
    
    if beta == 0:
        print(2 % m)
        return
    if beta == 1:
        print(alpha % m)
        return
    
    base = [[alpha % m, (m - 1) % m],
            [1, 0]]
    
    p = mat_pow(base, beta - 1, m)
    
    s1 = alpha % m
    s0 = 2 % m
    
    ans = (p[0][0] * s1 + p[0][1] * s0) % m
    print(ans)

if __name__ == "__main__":
    solve()
```该实现将递归编码为 2x2 转换矩阵，其中每个应用程序将序列向前推进一步。 递归中的减法是通过加 m-1 模 m 来实现的，以避免出现负值。 

快速求幂例程在对数时间内将矩阵求到 (β-1) 次幂，仅在指数的二进制表示需要时重复平方和乘法。 

最后，通过将矩阵应用于初始向量（S_1，S_0）来提取结果。 边缘情况 β = 0 和 β = 1 被显式处理，因为它们直接对应于基本定义，而不需要求幂。 

## 工作示例

 ### 示例 1

 输入：α = 5，β = 4，m = 321

 我们建立 S_0 = 2 和 S_1 = 5，然后应用递归 S_{k+1} = 5S_k - S_{k-1}。 

| k | S_k | S_{k-1} | αS_k - S_{k-1} |
 | --- | --- | --- | --- |
 | 1 | 5 | 2 | 5*5 - 2 = 23 | 5*5 - 2 = 23 |
 | 2 | 23 | 23 5 | 5*23 - 5 = 110 | 5*23 - 5 = 110
 | 3 | 110 | 110 23 | 23 5*110 - 23 = 527 | 5*110 - 23 = 527

 所以 S_4 = 527，对 321 求模得到 527 mod 321 = 206。 

这证实了递归构造可以正确地产生更高的幂，而无需显式使用 x。 

### 示例 2

 输入：α = 3，β = 3，m = 333

 我们再次一步步计算。 

| k | S_k | S_{k-1} | αS_k - S_{k-1} |
 | --- | --- | --- | --- |
 | 1 | 3 | 2 | 3*3 - 2 = 7 | 3*3 - 2 = 7
 | 2 | 7 | 3 | 3*7 - 3 = 18 | 3*7 - 3 = 18

 因此 S_3 = 18，模 333 仍为 18。 

这表明，即使 x 是复数（因为 α^2 - 4 < 0），递归也完全保持整数。

 ## 边缘情况

 当 α = 2 时，二次 x^2 - 2x + 1 = 0 折叠为 (x - 1)^2 = 0，因此 x = 1。在这种情况下，每一项 S_k = 1^k + 1^{-k} = 2。递归仍然产生 S_{k+1} = 2S_k - S_{k-1}，并且当 S_0 = 2 且 S_1 = 2 时，它保持不变 常数。 该算法自然地保留了这一点，因为转换矩阵变为 [[2, -1], [1, 0]]，其特征值为 1，重数为 2。 

对于 α^2 < 4 的情况，根是复共轭。 递归避免了对这些数字的任何直接使用，并完全保留在整数算术中。 S_k 作为根中的对称多项式的不变性确保无论 x 的性质如何，计算都保持有效。 

接近 2^64 的大 β 不会影响正确性，因为求幂是通过二进制提升完成的，这仅取决于 β 的位长度而不是其大小。
