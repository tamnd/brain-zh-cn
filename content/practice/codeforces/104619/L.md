 ## 解决方案
 ## 问题理解

 我们在二维网格上得到一组点，每个点代表一个位置，例如房屋或公寓。 我们必须选择一个新点$(x, y)$那里将建一个充电站。 候选位置的质量是通过从该位置到每个给定点的曼哈顿距离之和来衡量的。 两点之间的曼哈顿距离是水平距离加上垂直距离。 

任务是找到一个使总距离最小化的位置。 如果多个位置达到相同的最小总和，我们必须选择最小的一个$x$。 如果仍然平局，我们选择最小的$y$。 

输入大小可达$n = 100000$，坐标范围为$[-100000, 100000]$。 这立即排除了任何评估网格中所有候选位置或计算距每个可能点的距离的方法。 对候选位置的暴力破解太大了，因为即使坐标范围本身也给出了$200001^2$可能性，这是不可行的。 

第二个天真的想法是测试所有输入点作为潜在的设施位置。 这减少了候选人$n$，但每次评估仍然需要花费$O(n)$，导致$O(n^2)$总工作量。 和$n = 10^5$，这变成了$10^{10}$距离计算，速度太慢。 

当多个点共享相同的最佳中值结构时，就会出现微妙的边缘情况。 例如，如果所有点都位于垂直线上$x = 0$，那么任意一点$x = 0$和中位数$y$值是最优的。 独立计算中位数但不一致强制打破平局的粗心实现可能会返回不同的有效坐标，但问题需要确定性选择。 

## 方法

 关键的观察是曼哈顿距离分为独立的贡献$x$和$y$:$$|x - x_i| + |y - y_i| = |x - x_i| + |y - y_i|$$因此总成本变为：$$\sum |x - x_i| + \sum |y - y_i|$$这种分解至关重要，因为它意味着我们可以优化$x$和$y$独立。 

我们首先考虑一维版本：选择一个值$x$最小化$\sum |x - x_i|$。 绝对偏差的一个众所周知的属性是多重集的任何中值都会最小化这个总和。 直观上，向左或向右移动所选点会增加距离，至少与减少的点一样多。 

因此，最优的$x$是的中位数$x_i$值，并且类似地最优$y$是的中位数$y_i$价值观。 

蛮力方法将评估每个候选人$x$和$y$，每次重新计算总和，成本$O(n)$每次评估。 这导致$O(n^2)$整体复杂性。 中值洞察力使用排序将整个搜索空间折叠为单个确定性计算。 

打破平局需要小心。 什么时候$n$是偶数，有多个中位数。 规则“最小化$x$，然后最小化$y$” 翻译为选择较低的中值索引（按排序顺序的左侧中间元素），因为它产生最佳解决方案中的最小有效坐标。

 | 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最优（中值）|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们对待$x$和$y$独立，因为成本函数完全分裂。 

1. 读取所有点并存储它们$x$和$y$分别坐标。 这种分离是必要的，因为我们将对每个维度进行独立排序。 
2. 对所有列表进行排序$x$- 坐标。 需要排序是因为中位数是根据顺序统计数据而不是原始值定义的。 
3. 选择中位数$x$-协调。 如果$n$是奇数，这是中间的元素。 如果$n$是偶数，我们选择中间较低的索引来满足最小化的要求$x$在所有最优解中。 
4. 重复相同的过程$y$- 坐标，提取中位数$y$。 
5、输出$(x, y)$作为最终答案。 

### 为什么它有效

 正确性依赖于以下性质：对于任何实数多重集，函数$f(t) = \sum |t - a_i|$在集合的任何中位数处最小化。 如果我们搬家$t$在中位数的左侧，右侧的点多于左侧的点，从而增加了总距离； 如果我们向右移动，则会发生相反的情况。 由于目标分解为独立的总和$x$和$y$，为每个坐标选择最佳中位数同时最小化完整的二维曼哈顿目标。 通过选择较低的中值索引来满足平局打破规则，确保所有最佳解决方案中字典顺序最小的坐标。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    xs = []
    ys = []
    
    for _ in range(n):
        x, y = map(int, input().split())
        xs.append(x)
        ys.append(y)
    
    xs.sort()
    ys.sort()
    
    # lower median for tie-breaking
    x_ans = xs[(n - 1) // 2]
    y_ans = ys[(n - 1) // 2]
    
    print(x_ans, y_ans)

if __name__ == "__main__":
    solve()
```该解决方案将输入拆分为两个数组，以便可以独立应用排序。 对每个数组进行排序可确保我们可以在预处理后在恒定时间内直接访问中位数。 指数$(n - 1) // 2$专门选择来执行打破平局规则，确保当存在多个最佳中位数时，我们选择最小的可行坐标。 

一个常见的实施错误是使用`n // 2`盲目地不考虑均匀大小的情况和词典编排的限制。 使用`(n - 1) // 2`通过始终选择左侧中位数来避免该问题。 

## 工作示例

 ### 示例 1

 输入：```
3
0 2
2 3
1 0
```已排序$x$：[0,1,2]

 已排序$y$：[0,2,3]

 | 步骤| xs | ys | 选择x | 选择 y |
 | ---| ---| ---| ---| ---|
 | 已排序 | [0,1,2]| [0,2,3]| - | - |
 | 中位数| - | - | 1 | 2 |

 中位数元素是$x = 1$,$y = 2$。 该点最小化曼哈顿总距离。 

这证实了中间元素平衡两侧距离的性质。 

### 示例 2

 输入：```
2
0 0
2 2
```已排序$x$: [0, 2]

 已排序$y$: [0, 2]

 | 步骤| xs | ys | 选择x | 选择 y |
 | ---| ---| ---| ---| ---|
 | 已排序 | [0,2]| [0,2]| - | - |
 | 中位规则 | - | - | 0 | 0 |

 这里两个中位数都不是唯一的。 该规则选择较低的中位数，给出$(0, 0)$，这是最优的并且满足平局决胜。 

这表明均匀大小的输入如何需要一致的中值选择。

 ## 边缘情况

 关键的边缘情况是偶数$n$，其中存在多个中位数并且打破平局很重要。 

考虑：```
2
0 0
2 2
```已排序$x = [0, 2]$, 已排序$y = [0, 2]$。 该算法选择索引$(2 - 1) // 2 = 0$, 给予$x = 0$,$y = 0$。 如果我们选择上中位数，我们会得到$(2, 2)$，具有相同的总成本，但违反了最小化的要求$x$， 然后$y$。 较低的中值选择强制执行正确的字典顺序最小最佳点。 

另一种情况是所有点都相同时：```
3
7 7
7 7
7 7
```两个排序数组都是常量，因此任何中值索引都会产生 7。算法返回$(7, 7)$一致地匹配唯一可行的最优解决方案。
