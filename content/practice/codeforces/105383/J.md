 ## 问题理解

 我们得到一个正实数，写成带小数点的字符串。 任务是计算其下限，即不超过该值的最大整数，并输出该整数，不带任何小数部分。 

输入格式有意最小化，只有一行，例如`1999.99`,`2.00000`，或潜在的值，例如`0.123`。 该结构保证恰好有一位小数点并且其两侧至少有一位数字。 该值的上限为 10^8，但由于输入是字符串，因此真正重要的限制是它能够轻松地放入内存中并且可以逐字符进行处理。 

从复杂性的角度来看，输入大小上限为 15 个字节，这立即排除了对解析库或高性能数值算法的任何需要。 任何扫描一次字符串的解决方案就足够了。 即使每个角色持续工作的 O(n) 方法也非常快。 

主要的微妙之处在于，编程语言中的浮点解析不是我们应该依赖的。 转换为浮点数然后应用下取整函数可能会导致较大或笨拙的小数表示形式出现精度错误。 基于字符串的方法完全避免了这种情况。 

边缘情况以两种形式出现。 首先，已经是整数但在小数点后添加尾随零的数字。 例如，`2.00000`应该输出`2`。 简单的浮点转换可能会产生`1`如果发生舍入错误，尽管在大多数语言中这种特定输入是安全的，但这仍然是不必要的风险。 

其次，小于 1 的数字，例如`0.99999`或者`0.12345`。 在这些情况下，地板是`0`。 如果尝试不正确地解释字符串，则错误地将整数部分提取为数字并假设其非零的简单方法将会失败。 

第三种微妙的情况是非常大的整数部分，例如`100000000.000`，其中正确性取决于准确保留整数子字符串而没有溢出或格式更改。 

## 方法

 暴力破解的想法是将输入视为实数：将字符串转换为浮点值，应用向下取整运算，然后打印结果。 这是有效的，因为它符合数学定义，并且对于许多典型情况它会通过。 

然而，这种方法依赖于二进制浮点表示。 像这样的数字`1999.99`不能完全用二进制表示，因此中间舍入可能会引入微小的误差。 虽然这些错误通常是不可见的，但楼层操作对它们很敏感，因为像这样的值`1.99999999997`可能被正确地解释为`1`，但稍微歪曲的边界情况可能会意外发生变化。 

更重要的是，这个问题根本不需要任何算术。 输入的结构已经直接对答案进行了编码：小数点之前的所有内容都是整数部分，floor 忽略它之后的所有内容。 

关键的观察结果是，对于任何以标准形式编写的正小数，下限恰好是解释为整数的小数点之前的子字符串。 无需进一步计算。 

这减少了从数值计算到字符串解析的任务，完全消除了精度风险，并将解决方案减少为单个线性扫描。

 ## 算法演练

 我们直接从输入字符串构造结果。 

1. 读取输入字符串。 
2. 扫描字符串直到遇到小数点。 
3. 提取小数点前的所有字符。 
4. 将此子字符串作为整数输出。 

每个步骤都通过十进制表示的结构来证明。 小数点是整数和小数部分之间的精确分隔符，因此它后面的所有内容与地板无关。 

### 为什么它有效

 正十进制数的下限仅取决于该值完全包含多少个整数单位。 小数点前的数字精确编码该数量。 小数部分无论大小，都无法增加下限值，因为它始终严格小于 1。因此，丢弃小数部分会在所有有效输入中保留正确的整数结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

i = 0
while i < len(s) and s[i] != '.':
    i += 1

print(int(s[:i]))
```该解决方案将输入读取为字符串并进行扫描，直到找到小数点。 该点之前的所有内容都是整数部分。 将该子字符串转换为整数会自动处理诸如前导零之类的情况（如果存在），尽管该问题保证大于或等于 1 的数字不会有前导零。 

重要的实现细节是避免任何浮点转换。 尽管 Python 可以安全地处理许多输入，但字符串方法是确定性的，并且与下限的定义完全一致。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    s = input().strip()
    i = 0
    while i < len(s) and s[i] != '.':
        i += 1
    return str(int(s[:i]))

# provided samples
assert run("1999.99\n") == "1999"

 ## 边缘情况

 一种边缘情况是数字小于 1，例如`0.123`。 该算法扫描直到小数点，如果数字以以下开头，则生成一个空前缀`0.`。 转换`0`正确地产生地板，即`0`。 

另一种情况是像这样的输入`2.00000`，其中小数部分非空但完全为零。 扫描在小数点处停止并忽略所有尾随零。 整数前缀`2`正确代表地板。 

更大的整数边界情况，例如`99999999.999`表明不需要溢出或浮点转换。 前缀被安全地提取为字符串并转换为整数，而不会损失精度。 

在所有情况下，关键的不变量是小数点之前的子字符串始终等于标准十进制表示中的所有有效输入的数字的数学下限。
