 ## 解决方案
 ## 问题理解

 我们给定了一个矩形排列的乘客，建模为$n \times m$网格。 每个单元代表一个座位，包含一名明确感染的乘客、一名绝对健康的乘客或一名独立感染概率的不确定乘客$1/2$。 

成本模型取决于感染在当地的传播方式。 如果乘客被感染，他们将承担固定的隔离费用$d_0$。 与至少一名受感染乘客直接相邻的任何健康乘客都会做出贡献$d_1$。 任何不与任何受感染乘客边缘相邻但与至少一名受感染乘客角落相邻的健康乘客都有贡献$d_2$。 如果一名健康的乘客根本没有受感染的邻居（即使是对角线），那么他们的成本为零。 

任务不是计算一个确定性值，而是计算未知单元的所有随机分配的预期总隔离成本。 每个“？”独立地被感染或未被感染的概率$1/2$。 必须按照该乘积分布下的期望精确计算答案，然后输出对一个大素数取模。 

限制因素$n, m \le 100$至多暗示$10^4$细胞。 任何枚举所有的方法$2^{nm}$配置是不可能的。 即使对邻居的所有子集进行每个单元的枚举，只有在它仍然存在的情况下才是可接受的$O(nm)$或者$O(nm \cdot 8)$。 这强烈表明该解决方案必须分解为局部贡献并避免远程单元之间的耦合。 

一种微妙的边缘情况是，一个单元的成本类别取决于至少一个受感染邻居的存在，而不是数量。 这种“至少一个”条件引入了补码事件，这比直接计数更容易处理。 

另一个重要的情况是完全未知的网格。 例如，如果所有单元格都是“？”，则每个单元格都是对称的，但依赖关系仍然存在，因为邻居重叠。 将每个细胞的独立概率相乘的简单方法会错误地忽略重叠事件，例如两个邻居都被感染。 

## 方法

 暴力方法将为每个“？”分配一个值$\{0,1\}$, 枚举所有$2^k$配置，通过扫描所有单元并检查邻居来计算每个配置的总成本，然后求平均值。 这是正确的，但即使对于$k=100$， 自从$2^{100}$是一个天文数字。 

关键的观察结果是，预期总成本可以分解为每个单元的预期贡献之和。 期望的线性消除了全局考虑联合配置的需要。 每个细胞在预期中都是独立贡献的，但其贡献取决于其自身及其邻居的局部感染概率。 

对于每个细胞，我们只需要知道它是否被感染以及它是否在4个方向和对角线意义上感染了邻居。 关键的简化是我们不需要邻居的完全联合分布； 我们只需要诸如“一组中至少有一个被感染的邻居”之类的事件的概率。 

这些可以使用补集来计算：对于一组独立的伯努利邻居，没有人被感染的概率是它们“未感染”概率的乘积。 一旦我们预先计算每个细胞的感染概率，就可以在恒定时间内计算每个细胞的预期贡献。 

因此，问题简化为迭代所有单元格，计算：

 细胞被感染的概率，以及它没有被感染的 4-邻居或没有被感染的对角邻居的概率，并根据确定性成本规则将它们组合起来。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^{nm} \cdot nm)$|$O(nm)$| 太慢了 |
 | 期望值分解|$O(nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 ### 步骤1：将每个单元格转换为感染概率

 我们定义$p_{i,j}$作为细胞被感染的概率。 如果是“V”，那么$p_{i,j}=1$。 如果是‘.’，那么$p_{i,j}=0$。 如果是“？”，那么$p_{i,j}=\frac{1}{2}$。 

我们还定义$q_{i,j} = 1 - p_{i,j}$，细胞未被感染的概率。 这种双重表示很有用，因为所有邻域条件都简化为“这些细胞都没有被感染”。 

### 步骤 2：预先计算模算术值

 由于答案需要模数$10^9+7$，我们代表$1/2$作为 2 的模逆。这使得所有概率都可以精确地以模形式维护。 

### 步骤 3：计算受感染细胞的贡献

 每个细胞都贡献$d_0$乘以被感染的概率。 我们将其直接累积到答案中：$p_{i,j} \cdot d_0$。 

这是有效的原因是期望的线性：每个受感染的细胞贡献独立于其他细胞的固定量。 

### 步骤 4：计算健康细胞的边缘相邻影响

 对于未被感染的细胞来说，它有助于$d_1$如果它至少有一个被感染的 4-邻居。 

我们不是直接计算这个事件，而是计算它的补数：所有 4 个邻居都没有被感染。 

所以：$$P(\text{at least one infected 4-neighbor}) = 1 - \prod (1 - p_{\text{neighbor}})$$我们将此概率乘以$q_{i,j} \cdot d_1$，因为该规则仅适用于细胞本身未被感染的情况。 

### 步骤 5：计算对角相邻影响力

 类似地，如果细胞没有被感染并且没有被感染的 4 个邻居，则它有助于$d_2$如果至少一个对角邻居被感染。 

我们计算：$$P(\text{diagonal trigger}) = P(\text{no 4-neighbor infected}) \cdot (1 - \prod (1 - p_{\text{diag}}))$$然后乘以$q_{i,j} \cdot d_2$。 

当 4 个邻居已经触发更高的成本时，这种条件结构确保我们不会重复计算对角线影响。 

### 步骤 6：对所有单元格求和

 每个细胞都独立地对期望做出贡献，因此我们将所有贡献相加。 

### 为什么它有效

 该算法依赖于两个属性。 首先，期望的线性允许将全局成本分解为单元的总和，而无需担心不同单元状态之间的相关性。 其次，每个本地事件（“至少一个受感染的邻居”）可以表示为独立事件的补充，从而可以使用简单的乘积进行精确的概率计算。 这两个事实消除了相关邻居配置可能产生的指数依赖性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
INV2 = (MOD + 1) // 2

def mul(a, b):
    return (a * b) % MOD

def add(a, b):
    s = a + b
    return s % MOD

def solve():
    n, m, d0, d1, d2 = map(int, input().split())
    
    grid = [input().strip() for _ in range(n)]
    
    p = [[0] * m for _ in range(n)]
    q = [[0] * m for _ in range(n)]
    
    for i in range(n):
        for j in range(m):
            if grid[i][j] == 'V':
                p[i][j] = 1
            elif grid[i][j] == '.':
                p[i][j] = 0
            else:
                p[i][j] = INV2
            q[i][j] = (1 - p[i][j]) % MOD
    
    ans = 0
    
    dirs4 = [(1,0),(-1,0),(0,1),(0,-1)]
    dirs8 = [(dx,dy) for dx in (-1,0,1) for dy in (-1,0,1) if not (dx == 0 and dy == 0)]
    
    for i in range(n):
        for j in range(m):
            # infected contribution
            ans = (ans + p[i][j] * d0) % MOD
            
            # compute 4-neighbor infection probability
            prod4 = 1
            for dx, dy in dirs4:
                ni, nj = i + dx, j + dy
                if 0 <= ni < n and 0 <= nj < m:
                    prod4 = prod4 * (1 - p[ni][nj]) % MOD
            
            has4 = (1 - prod4) % MOD
            
            # compute diagonal (8-neighbors excluding 4-neighbors)
            prod8 = 1
            for dx, dy in dirs8:
                ni, nj = i + dx, j + dy
                if 0 <= ni < n and 0 <= nj < m:
                    prod8 = prod8 * (1 - p[ni][nj]) % MOD
            
            has_any = (1 - prod8) % MOD
            
            # ensure diagonal only applies if no 4-neighbor
            diag_only = (has_any - has4) % MOD
            
            ans = (ans + q[i][j] * d1 % MOD * has4) % MOD
            ans = (ans + q[i][j] * d2 % MOD * diag_only) % MOD
    
    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现反映了算法中的分解。 数组`p`和`q`存储感染和不感染的概率。 对于每个细胞，首先添加受感染的贡献。 

4-邻居计算使用补集的乘积，这避免了重复计算重叠的邻居依赖性。 同样的想法扩展到所有八个邻居，然后差异隔离了纯粹对角线触发的情况。 

减法`diag_only = has_any - has4`至关重要，因为它强制执行规则层次结构：对角线成本仅在不存在边缘相邻感染时适用。 如果没有这种分离，边缘和对角线均受感染邻居的细胞将被过度计数。

 ## 边缘情况

 关键的边缘情况是当一个单元同时具有边缘相邻和对角相邻的受感染邻居时。 例如：```
V ?
? .
```中心细胞看到这两种类型的影响。 该算法通过减去来确保当存在边缘相邻感染时抑制对角线贡献`has4`从`has_any`。 这保证了只有正确的层适用。 

另一个边缘情况是边界细胞，其中缺失的邻居不能被视为零概率感染细胞。 该实现在乘以概率之前显式检查边界，确保不存在的邻居不会影响产品。
