 ## 解决方案
 ## 问题理解

 我们给定一个隐藏的长度为整数的数组$n$，其中每个元素都在 1 到 100 之间。我们不能直接看到该数组。 相反，我们得到的是由该数组中每对相邻元素形成的乘积的多重集。 

具体来说，如果未知数组是$A = [a_1, a_2, \dots, a_n]$，然后我们得到无序集合$$\{a_1 a_2, a_2 a_3, \dots, a_{n-1} a_n\}.$$任务是重建任何有效的数组$A$它可以准确地生成相邻产品的多重集，或者确定不存在这样的数组。 

困难在于产品的顺序丢失。 我们不知道哪个产品属于哪个位置，因此邻接结构必须纯粹从因式分解约束推断出来。 

约束条件$n \le 18$是中心提示。 它排除了对整个范围内所有数组的任何指数搜索$[1, 100]^n$，这将是一个天文数字，但它可以轻松地允许通过对边的排列或分配进行修剪来进行回溯。 每个乘积最多为 10000 个，因此因子对是有限的，这进一步限制了分支。 

一个天真的尝试可能会尝试排列所有$n-1$产品并按顺序将它们分配为边，然后重建数组并检查一致性。 这已经导致$(n-1)!$可能性，即使在$n = 18$。 另一个天真的想法是尝试所有可能的起始值$a_1$，然后贪婪地分解每个乘积，但是贪婪会失败，因为早期错误的因子选择仍然会在以后留下有效的完成，并且没有什么会强制唯一的延续。 

当产品有许多因素对时，就会出现微妙的失败案例。 例如，如果产品为 36，则它可能来自 (1, 36)、(2, 18)、(3, 12)、(4, 9)、(6, 6)。 错误地选择一条边仍然可以完成剩余边，但会导致最终多重集使用不一致。 

## 方法

 关键的困难在于邻接结构是隐藏的，但每个有效数组都会产生一个类似路径的结构，其中每个位置恰好贡献两条边（端点除外）。 这建议考虑在未知顶点之间放置边，其中每条边都由产品标记，并且每个顶点值必须与所有关联边一致。 

暴力解释是尝试所有排列$n-1$产品以及沿链一致分配要素对的所有方法。 对于每个排列，我们选择一个起始值$a_1$，然后推导出$a_2 = b_{\pi(1)} / a_1$，然后确定性地继续。 问题在于排列空间和因子选择都会爆炸。 即使忽略因子分支，也有$(n-1)!$订单，这已经是不可行的。 

关键的观察是，一旦我们修复了一条边并确定了它的两个端点值，数组的其余部分几乎就被强制了。 如果我们知道$a_i$，然后对于任何剩余未使用的产品$b$，如果可以写成$a_i \cdot x$， 然后$x$成为下一个值的候选者。 这将问题转化为构建一条长度的路径$n$其中每个步骤都会消耗一个未使用的产品。 

因为$n \le 18$，我们可以对部分结构进行深度优先搜索。 状态很小：数组中的当前位置、当前值和剩余产品的多重集。 在每一步中，我们都会尝试所有可被当前值整除的未使用的乘积，并向前或向后扩展序列。 分支因子是有限的，因为每个乘积最多有几个最多 100 的除数。 

该结构本质上是一个关于值的受限路径构建问题，其中边只能重用一次，并且每个步骤都必须与可用产品匹配。 方向的对称性（我们可以解释为$a_i a_{i+1}$通过将每个产品视为双向来自然地处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 边的强力排列 |$O((n-1)! \cdot n)$|$O(n)$| 太慢了 |
 | 具有多集回溯的 DFS |$O(n! \cdot d)$最坏的情况，在实践中被大量修剪|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题视为构造一个序列$n$准确消费时的数字$n-1$产品。 

1. 我们将所有产品存储在多重集结构（频率图）中，因为重复很重要，我们必须恰好消耗每个出现一次。 该模型模拟了相同产品在输入中无法区分但边缘仍然不同的事实。 
2.我们尝试每一种产品作为潜在的起点。 对于所选产品$b = x \cdot y$，我们尝试两个方向：$a_1 = x, a_2 = y$和$a_1 = y, a_2 = x$。 这是必要的，因为邻接方向未知。 
3. 对于每次尝试，我们都会运行深度优先搜索，从左到右构建数组。 在任何时候，我们都有一个当前值$a_i$和部分序列。 
4. 来自$a_i$，我们尝试扩展到$a_{i+1}$通过迭代所有剩余的产品。 如果一个产品$p$满足$p \mod a_i = 0$， 然后$a_{i+1} = p / a_i$是下一个候选值。 
5. 我们使用该产品（降低其频率），附加新值，然后递归。 如果递归失败，我们将通过恢复乘积并删除附加值来回溯。 
6. 当我们放置时，搜索成功停止$n$数字和所有产品都只使用一次。 
7. 如果任何起始产品和方向成功，我们将输出构造的数组。 

### 为什么它有效

 在构造的数组的任何前缀处，剩余的未使用的产品正是那些尚未分配给相邻对的产品。 DFS 保持了每个选定步骤对应于最终邻接多重集中的真实边缘的不变性。 由于每个扩展都严格消耗一个产品并将序列长度增加一，因此无效的完成不会错误地满足完整约束。 任何有效的原始数组都对应于至少一个从其第一条边开始的有效 DFS 路径，因此搜索空间是完整的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    b = list(map(int, input().split()))
    
    from collections import Counter
    
    def backtrack(path, cnt):
        if len(path) == n:
            return True
        
        last = path[-1]
        
        for p in list(cnt.keys()):
            if cnt[p] == 0:
                continue
            if p % last != 0:
                continue
            nxt = p // last
            if nxt < 1 or nxt > 100:
                continue
            
            cnt[p] -= 1
            path.append(nxt)
            
            if backtrack(path, cnt):
                return True
            
            path.pop()
            cnt[p] += 1
        
        return False
    
    cnt0 = Counter(b)
    
    for i in range(n - 1):
        for a, c in [(b[i], True)]:
            # try factor pairs
            for x in range(1, 101):
                if a % x == 0:
                    y = a // x
                    if 1 <= y <= 100:
                        cnt = cnt0.copy()
                        cnt[a] -= 1
                        path = [x, y]
                        if backtrack(path, cnt):
                            print("Yes")
                            print(*path)
                            return
                        cnt[a] += 1
    
    print("No")

if __name__ == "__main__":
    solve()
```该解决方案首先将输入视为边缘的多重集。 每个候选起始边缘被扩展为允许值范围内的所有有效因子对。 然后，DFS 通过仅允许当前值干净地划分未使用的乘积的转换来强制一致性，从而生成有效的下一个值。 

微妙之处在于我们从不假设剩余产品的订购。 相反，我们总是在每一步搜索所有未使用的候选者，这是可以接受的，因为$n$很小，一旦约束传播，分支就会很快崩溃。 

## 工作示例

 ### 示例 1

 考虑一个存在有效重建的小情况：

 输入产品：$[8, 12, 6]$,$n = 4$我们尝试从产品 12 开始。 

| 步骤| 路径| 剩余产品 | 行动|
 | --- | --- | --- | --- |
 | 1 | [3, 4] | [8, 6] | 选择 12 = 3×4 |
 | 2 | [3,4,2]| [8] | 使用 8 = 4×2 |
 | 3 | [3,4,2,4]| []| 使用 6 = 2×4 |

 在每一步中，我们都会选择一个与当前端点匹配的产品。 构建之所以成功，是因为每个剩余的产品都与不断发展的链条保持一致。 

该迹线表明，一旦初始边缘固定，阵列的其余部分就会受到强制，从而证明了约束的强大传播效应。 

### 示例 2

 输入产品：$[36, 6, 6]$,$n = 4$我们尝试从 36 = 6×6 开始。 

| 步骤| 路径| 剩余产品 | 行动|
 | --- | --- | --- | --- |
 | 1 | [6, 6] | [6, 6] | 选 36 |
 | 2 | [6,6,1]| [6] | 使用 6 = 6×1 |
 | 3 | 失败| 回溯| 没有有效的延续 |

 这表明因素选择的模糊性。 即使初始选择有效，选择 1 作为下一个值也会阻止进一步扩展。 DFS 确保我们探索替代分解，防止过早承诺。

 ## 边缘情况

 一种边缘情况是所有乘积都相等，例如 36 的多个副本。DFS 必须正确尝试 36 的多次分解，而不是锁定到单个拆分。 多集跟踪确保每个副本仅被消耗一次，并且如果特定链失败，回溯会探索替代分解。 

当存在有效数组但需要尽早选择非明显因素时，会出现另一种边缘情况。 例如，为 36 选择 6×6 可能会阻碍进度，而 4×9 则可以。 该算法通过重试每个候选产品的所有因子对并在失败时完全恢复状态来处理此问题，从而确保探索的完整性。
