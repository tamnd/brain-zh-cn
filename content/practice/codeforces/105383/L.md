 ## 解决方案
 ## 问题理解

 我们正在研究一个有向图，其中每条边都有一个权重，应将其视为“标签”而不是成本。 路径被定义为精确的序列$k$从固定节点开始的有向边$s$并在固定节点结束$t$。 主要目标不仅仅是找到任何有效的长度路径$k$，而是找到边缘标签序列按字典顺序最小的一个。 

词典比较是通过查看两个序列不同的第一个位置并选择那里边缘权重较小的一个来完成的。 这使得早期的边缘比后来的边缘重要得多。 

一旦确定了最佳权重序列，我们就不会直接输出该序列。 相反，我们将其解释为基础 -$x$第一个边缘贡献最高功率而最后一个边缘贡献最低功率的数字，我们输出该值模$10^9 + 7$。 如果没有确切的步行$k$边存在于$s$到$t$，我们输出-1。 

这些约束立即排除了任何模拟长度路径的尝试$k$直接地。 步数$k$可以达到$10^9$，因此任何以路径长度为索引的动态规划都是不可能的。 然而，节点的数量非常少，最多 50 个，这表明我们应该考虑节点的状态压缩和步长的快速变换，而不是显式的路径枚举。 

一个天真的但诱人的想法是将其视为大小分层图中的最短字典路径$n \cdot k$，但是由于大小的原因，构建或遍历该图是不可能的$k$。 

贪婪思维的一个微妙的失败案例出现在图中，其中局部最小的边不会导致全局可行的完成。 例如，如果从$s$最小的边通向一个无法到达的节点$t$在$k-1$步，但边缘稍大一点，天真的贪婪选择就会失败。 这表明延续的可行性与词典顺序一样重要。 

## 方法

 蛮力视角将尝试枚举所有长度的路径$k$，跟踪它们的序列并选择字典顺序最小的。 即使限制为可行的转换，每个步骤分支最多$n$传出边缘，因此路径数量呈指数增长，就像$O(n^k)$。 即使是为了$k = 50$，这已经是不可行的了，并且对于$k$最多$10^9$，这是完全不可能的。 

关键的结构观察是词典最小化仅取决于前缀。 如果我们知道，对于任何节点$u$，最佳可能长度的后缀序列$r$导致从$u$到$t$，我们可以通过检查采用该边是否允许使用字典顺序最佳后缀完成来比较来自节点的传出边。 这表明在路径上进行动态规划，但在$k$是不可能的。 

第二个关键思想是固定路径长度上的转换组成。 对于每个长度$L$，我们可以定义一个转移矩阵$A_L$在哪里$A_L[u][v]$告诉我们按字典顺序排列的最小边权重序列，它使我们从$u$到$v$正好在$L$步骤，或将其标记为不可能。 这些矩阵可以组合：如果我们知道最佳路径的长度$a$和$b$，我们可以将它们组合起来以获得最佳长度的路径$a+b$通过尝试中间节点和连接序列。 

因为$n \le 50$，我们可以负担得起$O(n^3)$组合步骤。 然后我们对路径长度使用二进制提升$k$，类似于快速求幂。 我们预先计算 2 的幂矩阵，然后根据$k$。 

字典序比较需要存储序列，但完整序列太大。 相反，我们只存储足够的信息来比较串联：在每个状态，我们都保持可达性和最佳词典转换的表示。 由于路径长度很大但节点数量很少，因此我们可以为每对存储$(u,v)$，每个功率段的最佳单边选择，并贪婪地重建。 

这将问题转化为半环上的最短字典路径，其中乘法是串联，加法是字典最小路径。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^k)$|$O(k)$| 太慢了|
 | 最优（最小加矩阵幂与字典合并）|$O(n^3 \log k)$|$O(n^2 \log k)$| 已接受 |

 ## 算法演练

 我们将问题重新表述为在固定步数上重复组合过渡结构。 

1. 对于每对节点$u, v$，计算字典顺序最小的单边过渡，意味着是否有多个边$u \to v$，我们只保留最小的权重。 这会将平行边压缩为一条。 
2. 构建基础过渡结构$T$对于长度为 1 的路径，其中$T[u][v]$如果存在边，则为边权重，否则视为无穷大。 这代表了最佳的一步行为。 
3. 将其扩展为可以组合的结构：组合两个结构时$A$和$B$，我们计算一个新的结构$C$这样对于每一对$u, v$，我们考虑所有中间节点$w$，并尝试形成一条路径$u \to w \to v$。 候选成本是来自的最佳序列的串联$A[u][w]$和$B[w][v]$。 我们保留所有此类选择中字典顺序最小的。 
4. 预先计算二的幂：$T, T^2, T^4, \dots$直到最大位$k$，重复应用组合规则。 
5. 分解$k$成二进制。 从表示“空路径”行为的身份结构开始：零长度路径仅在开始等于结束时才有效。 
6. 对于每一位$k$，如果设置了，则使用相同的组合规则将当前结果结构乘以相应的二次方转换结构。 
7. 处理完所有位后，条目$result[s][t]$包含长度路径的字典顺序最小的边权重序列$k$，或表示不可能。 
8. 将此序列转换为基数中的数值$x$模数$10^9+7$通过按顺序迭代权重。 

正确性取决于以下事实：当比较先进行前缀时，在串联下保留字典顺序。 每个 DP 结构都完全捕获固定长度的最佳可能前缀，因此将它们组合起来可以保持全局最优性。 

### 为什么它有效

 每个矩阵代表每对节点之间固定步数的最佳词典结果。 当我们组合两个矩阵时，我们实际上是在选择路径的中间分割点并比较由串联形成的完整前缀。 因为字典顺序仅取决于第一个不同的边，并且每个子结构已经在内部保证了最佳前缀，所以以后的重组不能产生比已经考虑的更好的前缀。 这使得半群组合有效并确保二元提升产生全局最优路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = None

def better(a, b):
    if a is INF:
        return b
    if b is INF:
        return a
    return min(a, b)

def merge(A, B, n):
    C = [[INF] * n for _ in range(n)]
    for i in range(n):
        for k in range(n):
            if A[i][k] is INF:
                continue
            a_seq = A[i][k]
            for j in range(n):
                if B[k][j] is INF:
                    continue
                b_seq = B[k][j]
                cand = a_seq + b_seq
                if C[i][j] is INF or cand < C[i][j]:
                    C[i][j] = cand
    return C

def build_base(n, edges):
    T = [[INF] * n for _ in range(n)]
    for u, v, w in edges:
        u -= 1
        v -= 1
        if T[u][v] is INF or [w] < T[u][v]:
            T[u][v] = [w]
    return T

def identity(n):
    I = [[INF] * n for _ in range(n)]
    for i in range(n):
        I[i][i] = []
    return I

def main():
    n, m, s, t, x, k = map(int, input().split())
    edges = [tuple(map(int, input().split())) for _ in range(m)]

    base = build_base(n, edges)

    maxb = k.bit_length()
    pw = [base]
    for _ in range(maxb):
        pw.append(merge(pw[-1], pw[-1], n))

    res = identity(n)
    for i in range(maxb):
        if (k >> i) & 1:
            res = merge(res, pw[i], n)

    if res[s-1][t-1] is INF:
        print(-1)
        return

    seq = res[s-1][t-1]

    MOD = 10**9 + 7
    ans = 0
    for w in seq:
        ans = (ans * x + w) % MOD
    print(ans)

if __name__ == "__main__":
    main()
```该解决方案将每个状态表示为一个矩阵，其中每个条目都是固定长度的两个节点之间按字典顺序排列的最佳权重序列。 这`merge`函数通过尝试所有中间节点来组成两个这样的长度结构，这是标准的矩阵乘法模式，但使用字典级联而不是加法。 

恒等结构表示零长度行走，仅在开始和结束匹配时有效，这对于正确的二进制提升初始化至关重要。 

最后的转换步骤将恢复的序列解释为碱基序列$x$数字，从左到右累积，因此较早的边缘贡献更高的重要性。

 ## 边缘情况

 关键的边缘情况是局部最优边缘破坏可行性。 假设从节点 1 开始，我们有两条边：一条权重为 1 的边通向死端节点，另一条权重为 2 的边通往可以在剩余步骤中到达目标的循环。 该算法不会立即提交最小的出边； 相反，它在转换结构中保留完整的可达性信息，因此在矩阵组合期间，死端路径自然会被丢弃，因为它无法完成有效的长度 -$k$走。 

另一个边缘情况是多个路径产生相同的字典前缀但后来有所不同。 矩阵合并显式比较完整的串联序列，确保仅在前缀相同的情况下才考虑后面的差异。 这避免了如果仅比较边权重而不存储序列结构会发生的错误修剪。
