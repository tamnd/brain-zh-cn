 ## 解决方案
 ## 问题理解

 我们有一系列商店，每个商店都有一个当前利润值，可以是正值，也可以是负值。 目标是在应用最多一个称为蓝色咒语的全局操作和任意数量的称为绿色咒语的本地操作之后最大化总利润，但限制条件是不能在为蓝色咒语选择的间隔内应用绿色咒语。 

蓝色咒语选择一个连续的段并将其中的每个值加倍。 绿色咒语选择单个位置，但它只能应用于蓝色部分之外，并翻转所选元素的符号。 

因此，选择蓝色部分后，每个元素都属于三个类别之一。 在该段内，值会加倍且无法翻转。 在外部，值可以保持不变或单独否定。 

输出是最佳选择蓝色段并决定翻转哪些外部元素后可实现的最大总和。 

约束最多可达 3×10^5 个元素，其量级值最多可达 10^9。 这立即排除了尝试所有 O(n^2) 段或重新计算每个选择的总和的任何解决方案。 任何可行的解决方案都必须接近线性或线性算数。 

一个微妙的边缘情况来自这样一个事实：翻转对于段外的每个元素都是独立的。 对于蓝色线段之外的任何值 x，我们可以独立选择 max(x, −x)。 这造成了一个常见的陷阱：将绿色法术视为全局选择而不是每个元素的优化。 

另一个重要的边缘情况是蓝色段为空或实际上无用时。 例如，如果所有值都是负数，则将它们加倍会使情况变得更糟，但向外翻转仍可能挽救某些部分。 始终将蓝色应用于正区域的幼稚策略在完全负数组上失败。 

## 方法

 暴力方法会尝试所有可能的蓝色段 [L，R]。 对于每个选择，我们都会计算剩余元素的最佳可实现结果。 在段内，每个元素变为 2·a[i]。 在段之外，每个元素贡献 max(a[i], −a[i]) = |a[i]|。 因此对于固定段，总和为：

 加倍后段内求和 + 取绝对值后段外求和。 

我们可以预先计算数组的总绝对和。 然后对于选定的段，我们只需要替换段内元素的贡献：最初它们贡献|a[i]|，但选择蓝色段后它们贡献2a[i]。 因此，选择细分市场的收益是：

 2a[i] - |a[i]|。 

这将问题简化为在变换后的数组 b[i] = 2a[i] − |a[i]| 上查找最大子数组和。 然而，这仍然很微妙，因为基线假设外部的所有内容都是绝对值，这始终是最佳的。 

对段的暴力破解是 O(n^2)，这对于 n 高达 3×10^5 来说太慢了。 关键的观察结果是蓝色部分的效果完全是局部的和相加的，因此选择减少到派生权重数组的最大子数组问题。 一旦认识到这一点，问题就变成了标准的 Kadane 算法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对段进行暴力破解 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 变形+卡丹| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 关键重新制定

 1. 首先计算未使用蓝色咒语的基线贡献。 如果有利的话，每个元素都可以独立翻转，因此基线是 |a[i]| 的总和。 这对应于对待每个元素就好像我们可以在任何地方自由地应用绿色魔法一样。 
2. 现在考虑引入蓝色段[L,R]。 在该段内，我们失去了翻转的自由度，而是强制每个值变为 2·a[i]。 因此，与段中索引 i 的基线相比，净变化为：

2a[i] - |a[i]|。 

这是数组中唯一受选择段影响的部分。 
3. 定义辅助数组 b，其中 b[i] = 2a[i] − |a[i]|。 问题归结为选择一个连续的段来最大化 b[i] 的总和。 
4. 使用线性扫描计算 b 上的最大子数组和。 该值代表通过引入蓝色段可实现的最佳改进。 
5. 将此最佳改进添加到基线绝对值总和中。 结果就是答案。 

### 为什么它有效

 关键的不变量是，在所选的蓝色部分之外，绿色咒语的最佳策略始终是每个元素独立的，并且等于取绝对值。 这意味着任何全局决策只会影响段本身，而数组的其余部分会分解为固定的常量贡献。 一旦进行了这种分解，唯一剩下的优化就是选择一个连续区域，使相对于该基线的增量增益最大化，这正是最大子阵列问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    base = 0
    b = []

    for x in a:
        base += abs(x)
        b.append(2 * x - abs(x))

    best = b[0]
    cur = b[0]

    for i in range(1, n):
        cur = max(b[i], cur + b[i])
        best = max(best, cur)

    print(base + best)

if __name__ == "__main__":
    solve()
```该代码首先计算绝对值的基线总和，这对应于在任何蓝色段之外的任何地方以最佳方式应用绿色咒语。 然后，它构建转换后的数组，其中每个元素代表将该索引包含在蓝色段内的增益。 

Kadane 扫描跟踪此变换数组中的最佳连续段总和。 变量 cur 表示以当前索引结束的最佳片段，并且 best 跟踪最佳的整体片段。 最后，我们将此改进添加到基线中。 

一个常见的实现陷阱是忘记蓝色段必须是非空的。 Kadane 初始化通过从第一个元素而不是零开始来确保这一点。 

## 工作示例

 ### 示例 1

 输入：```
4
-2 5 -3 4
```基线绝对和为 2 + 5 + 3 + 4 = 14。 

我们计算 b[i] = 2a[i] − |a[i]|：

 | 我| 一个[我] | b[i] |
 | --- | --- | --- |
 | 1 | -2 | -2 |
 | 2 | 5 | 5 |
 | 3 | -3 | -3 |
 | 4 | 4 | 4 |

 现在我们运行 Kadane：

 | 我| b[i] | 当前| 最好的|
 | --- | --- | --- | --- |
 | 1 | -2 | -2 | -2 |
 | 2 | 5 | 5 | 5 |
 | 3 | -3 | 2 | 5 |
 | 4 | 4 | 6 | 6 |

 最好的改进是 6，所以最终答案是 14 + 6 = 20。 

这表明最佳蓝色片段不一定与单个符号模式对齐，而 Kadane 自然会捕获最佳混合物。 

### 示例 2

 输入：```
7
-1 -1 -1 -1 -1 -1 -1
```基线绝对和为 7。 

计算 b[i]：

 每个 a[i] = -1 给出 b[i] = 2(-1) − 1 = -3。 

卡丹：

 | 我| b[i] | 当前| 最好的|
 | --- | --- | --- | --- |
 | 1 | -3 | -3 | -3 |
 | 2 | -3 | -3 | -3 |
 | 3 | -3 | -3 | -3 |
 | ... | ... | ... | ... |

 最好的改进是 -3，这意味着我们实际上更愿意选择除了单个元素之外没有任何有益的蓝色片段，总体结果变为 7 − 3 = 4。 

这凸显了强制蓝色片段可能是有害的，而 Kadane 正确地解释了破坏性最小的选择。

 ## 边缘情况

 一个关键的边缘情况是所有值都为负时。 在这种情况下，天真的解释可能会建议选择长蓝色段来减少损失，但加倍负数只会使段内的损失恶化。 相反，正确的解决方案完全依赖于外部的绝对值基线，并通过 Kadane 选择影响最小的段，这自然避免了过度使用。 

另一个边缘情况是单元素数组。 该算法正确地简化为比较 2a[i] 与 |a[i]|，并且 Kadane 确保该段被选择或有效地最小化到该单个索引。 

最后一个微妙的情况是值的大小很大，接近 10^9。 由于所有转换都保持线性，并且只涉及乘以 2 和绝对值，因此 64 位整数算术就足够了，并且在 Python 中不会出现溢出问题，尽管这在固定宽度语言中可能很重要。
