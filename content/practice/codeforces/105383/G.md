 ## 解决方案
 ## 问题理解

 我们得到了一系列非负值，代表在游戏的每个级别获得的奖励。 对于从位置开始的任何连续段`l`并结束于`r`，玩家的表现得分计算为该段的舍入平均值：取该段上的值的总和，除以其长度，然后四舍五入到最接近的整数（标准四舍五入，即添加 0.5，然后取整）。 

对于每个起始索引`l`，我们被要求选择一个结束索引`r ≥ l`使得该舍入平均值最大化。 在所有选择中`r`为了获得最高分，我们必须选择最小的线段长度。 

因此，对于每个前缀起点，我们正在有效地解决“从这里开始的最佳后缀段”问题，其中目标是舍入平均值而不是原始总和或平均值。 

这些限制迫使我们仔细思考。 测试用例的总大小高达 2×10^5，因此任何解决方案总体上都必须接近线性或至多线性对数。 每个起始索引的简单二次扫描将立即超出时间限制，因为在最坏的情况下它将检查最多 O(n^2) 段。 

一个微妙的困难是目标函数不是明显单调的。 扩展一段可以增加或减少平均值，并且舍入会引入平台，其中不同的真实平均值会崩溃到相同的整数分数。 

暴露幼稚错误的边缘情况包括由于分布效应而添加较小元素会增加舍入平均值的序列，或者多个段长度产生相同舍入值但只应选择最短的长度的序列。 例如，考虑`[1, 100, 1]`从索引 1 开始。最好的舍入平均值是通过仅取`[1]`给出分数 1，或者可能是平均分略低于 1.5 且仍四舍五入为 1 的较大分段； 仅基于平均值局部增加的天真贪婪扩展可能会错误地持续太远。 

常量数组会出现另一个极端情况，例如`[5, 5, 5, 5]`，其中所有段长度产生相同的平均值，因此产生相同的舍入分数。 在这里，我们必须为每个起始位置显式返回长度 1，因为它给出了最佳解决方案中的最小长度。 

## 方法

 暴力解决方案修复起始索引`l`，然后尝试所有可能的终点`r`并维护运行总和来计算每个段的平均值。 对于每个段，我们计算`(sum / length + 0.5)`并跟踪最大值。 这是正确的，因为它显式检查所有有效段。 

但是，这需要每个起始索引执行 O(n) 工作，从而导致每个测试用例执行 O(n^2) 工作。 由于元素总数高达 2×10^5，在最坏的情况下这将涉及多达 4×10^10 次操作，这远远超出了可行的限制。 

关键的观察结果是，分数仅取决于分段的平均值，并且舍入函数将平均值划分为离散整数桶。 我们可以将所有内容相乘并以整数进行推理，而不是考虑浮点平均值：一个段`[l, r]`有分数`x`当且仅当其平均值位于`[x - 0.5, x + 0.5)`。 这变成了涉及前缀和的线性不等式。 

对于固定的起点，我们想要最佳可实现的整数`x`。 而不是搜索所有`r`，我们可以将问题重新解释为找到达到最佳“密度水平”的最小前缀长度。 这导致了单调结构：一旦分段平均值低于阈值，进一步扩展它就无法恢复更高的整数分数。 

这种单调性支持从右到左的两指针式扫描，维护候选段端点并确保每个`l`，我们找到最短的`r`从而达到最佳的四舍五入平均值。 平均值的结构确保了增加的最佳分段`l`可以通过重复使用先前位置的计算来导出，从而避免重新计算。

 ## 算法演练

 我们从右到左处理数组，维护一个移动窗口，该窗口代表从每个位置开始的最佳片段。 中心思想是，当我们将起始索引左移一位时，我们可以重用先前计算的段的结构。 

1. 开始于`l = n`，其中唯一可能的部分是`[n, n]`。 这个位置的答案长度一般为 1。 
2. 移动`l`到`n-1`。 我们现在比较是否从`l`包括`l+1`提高可实现的最佳舍入平均值，或者立即停止是否已经是最佳选择。 
3. 维持当前最佳分段`[l, r]`及其总和。 当我们通过增加来扩展该段时`r`，我们增量更新总和并重新计算舍入平均值。 我们继续延伸`r`而四舍五入后的分数不会减少。 
4. 对于每个`l`，我们模拟最小的`r`达到在此扩展过程中找到的最大舍入分数。 如果有多个`r`给出相同的最佳分数，我们保留最小的`r`，这相当于在平均值下降之前尽早停止。 
5. 存储长度`r - l + 1`作为位置的答案`l`，并移动`l`向左，重用当前`r`指针而无需重置它。 

关键效率来自于以下事实：`r`在整个算法中只向前移动，从不向后移动，因此每个元素作为扩展窗口的一部分最多处理一次。 

### 为什么它有效

 对于每个起始索引`l`，算法隐式搜索后缀的最小前缀`[l, n]`达到最大可能的四舍五入平均值。 单调运动`r`确保我们永远不会跳过可以提高分数的候选片段，因为任何改进都必须来自添加元素，并且一旦添加元素停止改进舍入值，进一步的扩展就无法恢复它。 这创建了一个不变量：对于每个`l`，维护的窗口`[l, r]`始终是从 开始的所有分段中实现最佳四舍五入分数的最短分段`l`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        # We maintain a right pointer and sliding sum.
        r = n
        total = 0

        # We will build answers from right to left.
        ans = [0] * n

        # Start with empty window and extend greedily.
        # We simulate adding elements from right to left,
        # but maintain r as the farthest endpoint we may need.
        r = n - 1
        total = 0

        # We process l from n-1 to 0
        # For each l, we expand r if needed.
        for l in range(n - 1, -1, -1):
            total += a[l]

            # We try to ensure r is at least l
            if r < l:
                r = l

            # Current best is at least taking single element
            best_score = (a[l] * 2 + 1) // 2
            best_len = 1

            curr_sum = 0

            # Expand r forward greedily while it improves or matches score
            for rr in range(l, n):
                curr_sum += a[rr]
                length = rr - l + 1

                score = (curr_sum * 2 + length) // (2 * length)

                if score > best_score:
                    best_score = score
                    best_len = length
                elif score == best_score:
                    best_len = min(best_len, length)

            ans[l] = best_len

        out.append(" ".join(map(str, ans)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```上面的实现通过扫描每个起点的所有后缀来直接解释问题，这使得逻辑透明但没有优化。 关键的计算量是变换后的舍入表达式`(sum * 2 + length) // (2 * length)`，它避免了浮点运算并直接模拟舍入行为。 

决策逻辑跟踪最佳分数和实现该分数的最小长度，确保多个段产生相同舍入值时的正确性。

 ### 示例 1

 输入数组：`[1, 2, 3]`我们计算每个起始索引的答案。 

| 我| 段已尝试| 总和级数| 最好成绩| 最佳长度|
 | --- | --- | --- | --- | --- |
 | 3 | [3] | 3 | 3 | 1 |
 | 2 | [2], [2,3] | 2 → 5 | 2 → 3 | 2 |
 | 1 | [1], [1,2], [1,2,3] | 1 → 3 → 6 | 1 → 2 → 2 | 3 |

 为了`l = 1`，最佳舍入平均值是在全长 3 处实现的，因为平均值稳步提高并且仅在全段处舍入到 2。 

该轨迹表明，一旦平均值在舍入下稳定下来，如果较长的段保持相同的舍入值，则它们仍然是最佳的。

 输入数组：`[5, 1, 1]`| 我| 段已尝试| 总和级数| 最好成绩| 最佳长度|

 ## 边缘情况

 对于常量数组，例如`[3, 3, 3, 3]`，每个段具有相同的平均值，因此每个舍入分数都是 3。该算法始终保持最小长度，因为每当遇到相等分数时，只有在改进时才会更新最佳长度。 从每一个开始`l`，第一个元素已经达到最佳分数，所以答案仍然是 1。 

对于严格增加的数组，例如`[1, 2, 3, 4]`，扩展段总是会增加真实平均值，而舍入保留了这种单调改进。 因此，该算法不断扩展，直到达到早期位置的完整后缀，从而产生从左到右递减的答案。 

对于严格递减的数组，例如`[4, 3, 2, 1]`，任何扩展都会立即将平均值降低到足以导致舍入不会改善的程度。 对于每个起始位置，算法在长度为 1 时停止，因为没有扩展可以匹配初始舍入分数。
