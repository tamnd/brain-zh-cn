 ## 解决方案
 ## 问题理解

 我们得到了一组学生和他们之间的相遇日志。 一些学生最初被感染，每当被感染的学生被记录为与另一名学生见面时，感染就会通过这种接触关系传播。 处理完所有记录的遭遇后，我们需要确定哪些学生最终被感染。 

重新表述输入的一个有用方法是将学生视为图中的节点，并将每个记录的遭遇视为无向边。 除了接触将两个学生联系起来这一事实之外，感染并不取决于时间或方向。 一旦学生被感染，通过接触链接触到的每个学生也会被感染。 

输出通常是传播稳定后受感染学生的最终集合或计数，这意味着不可能出现新的感染。 

从约束的角度来看，该结构强烈建议大约$10^5$学生和联系人。 任何尝试对每个受感染节点进行重复图遍历的方法在密集情况下都会降级为二次行为，因为感染可以通过大型连接组件级联。 这推动我们走向接近线性的解决方案，理想情况下$O(n + m)$或者$O(m \alpha(n))$。 

一些边缘情况在这里很重要。 如果最初没有学生被感染，则无论接触者如何，答案都必须保留为空，因为没有传播源。 如果所有学生最初都被感染，那么即使没有接触者，每个学生仍然被感染。 当联系人列表为空但存在初始感染时，会出现另一种微妙的情况； 在这种情况下，由于没有传播路径，因此仅报告初始集。 

## 方法

 直接模拟将重复扫描所有接触者并分轮传播感染。 在每一轮中，我们都会检查每个边缘，并在一个端点被感染时标记新的感染。 这是正确的，因为感染仅沿着边缘传播并且只能向外扩展。 然而，在最坏的情况下，感染可能会跨长链传播$m$，每轮扫描所有$m$边缘，生产$O(nm)$行为。 和$m$靠近$10^5$，这很快就变得不可行。

 关键的观察结果是，感染仅取决于连接性，而不取决于遭遇顺序。 一旦两个学生通过任意顺序的接触联系起来，无论时间顺序如何，感染都可能在他们之间传播。 这将问题简化为在无向图中查找连接的组件，然后将包含至少一个最初感染节点的任何组件标记为完全感染。 

此结构正是不相交集并集 (DSU) 或并查集结构有效捕获的结构。 每当记录到接触时，我们就会合并学生，然后检查哪些组件包含受感染源。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 模拟（边缘上的多遍 BFS）|$O(nm)$|$O(n + m)$| 太慢了|
 | DSU / 连接组件 |$O(m \alpha(n))$|$O(n)$| 已接受 |

 ## 算法演练

 1. 初始化 DSU 结构，每个学生从自己的组件开始。 这表示最初没有已知的接触，因此每个节点都是孤立的。 
2. 读取所有接触对并合并 DSU 中的端点。 每个联合合并两个组件，因为直接相遇保证了它们之间的连接。 
3. 维护一个布尔数组，指示哪些学生最初被感染。 
4. 完成所有并集后，为每个学生确定其 DSU 组件的代表。 如果该组件中的任何学生受到感染，则将该组件标记为已感染。 
5. 最后，遍历所有学生并输出那些组件被标记为感染的学生。 

我们将感染传播推迟到所有联合之后的原因是 DSU 编码独立于处理顺序的连接。 如果我们尝试在联合操作期间传播感染，我们最终仍然需要维护组件级状态，这在最终传递中计算起来更简单。 

### 为什么它有效

 DSU 将学生划分为等价类，其中每个类对应于接触图中的一个连接组件。 沿边缘的感染是单调的：一旦组件中的一个节点被感染，该组件中的每个节点都是可访问的，因此也被感染。 这在“受感染的学生”和“包含至少一个最初受感染节点的组件”之间创建了直接等价物，确保了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]

def solve():
    n, m = map(int, input().split())
    infected_count = int(input())
    infected = list(map(int, input().split())) if infected_count else []

    infected = [x - 1 for x in infected]

    dsu = DSU(n)

    for _ in range(m):
        u, v = map(int, input().split())
        dsu.union(u - 1, v - 1)

    infected_root = set()
    for x in infected:
        infected_root.add(dsu.find(x))

    res = []
    for i in range(n):
        if dsu.find(i) in infected_root:
            res.append(str(i + 1))

    print(" ".join(res))

if __name__ == "__main__":
    solve()
```DSU 实现使用路径压缩和按大小联合来确保每个操作的摊余复杂性接近恒定。 联合步骤从联系人列表中逐步建立连接。 之后，感染集被映射到组件代表中，以便感染成为整个组件而不是单个节点的属性。 

最终扫描检查受感染组件的成员资格。 这避免了任何重复的图遍历，并确保每个节点被处理恒定的次数。 

## 工作示例

 ### 示例 1

 假设有 5 名学生，接触者：(1,2)、(2,3)、(4,5)，最初感染的学生为 1。 

处理并集后，我们得到组件{1,2,3}和{4,5}。 感染从组件 {1,2,3} 开始，因此该组件中的所有学生都会受到感染。 

| 步骤| 运营| DSU 组件 | 受感染的根|
 | ---| ---| ---| ---|
 | 1 | 初始化| {1}{2}{3}{4}{5} | {1} |
 | 2 | 联盟(1,2) | {1,2}{3}{4}{5} | {1} |
 | 3 | 联盟(2,3) | {1,2,3}{4}{5} | {1} |
 | 4 | 联盟(4,5) | {1,2,3}{4,5} | {1} |
 | 5 | 标记感染| 相同| {1,2,3} 组件 |

 输出是`1 2 3`。 

这证实了感染遵循连接性，而不是仅仅直接相邻性。 

### 示例 2

 考虑 4 名学生，接触者 (1,2)、(3,4)，以及最初感染的学生 1 和 3。 

我们得到两个断开连接的组件，每个组件都有一个受感染的源。 

| 步骤| 运营| DSU 组件 | 受感染的根|
 | ---| ---| ---| ---|
 | 1 | 初始化| {1}{2}{3}{4} | {1,3} |
 | 2 | 联盟(1,2) | {1,2}{3}{4} | {1,3} |
 | 3 | 联盟(3,4) | {1,2}{3,4} | {1,3} |

 两个组件都被感染，因此所有学生都被感染。 

输出是`1 2 3 4`。 

这表明多个互不相连的疫情是独立但统一处理的。

 ## 边缘情况

 一个关键的边缘情况是感染开始但没有接触者。 例如，如果学生 2 在零边缘的 3 学生系统中被感染，则 DSU 仍保留隔离节点。 受感染的根集仅包含学生 2 的代表，并且没有联合体对此进行修改。 输出正确地仅列出了学生 2。 

另一种情况是所有学生最初都被感染，但图表已断开连接。 DSU 仍然产生多个组件，但每个组件都已经包含感染，因此无论结构如何，每个节点都被标记为感染。 这证实了该算法并没有错误地依赖连接来创建感染，而只是传播它们。
