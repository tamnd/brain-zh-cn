 ## 解决方案
 ## 问题理解

 我们有一组放置在数轴上的多米诺骨牌。 每块都位于整数坐标处并具有高度。 从选定的坐标 K 开始，Alexey 向左和向右观察，并识别出严格位于 K 左侧的最近的多米诺骨牌和严格位于 K 右侧的最近的多米诺骨牌。如果存在这样的邻居，他会推动它，使其分别向外、向左或向右掉落。 

一旦多米诺骨牌开始倒下，它不会在撞到下一块后就停止。 相反，它的触及范围由其高度决定。 位于 X 位置、高度为 H 的多米诺骨牌倒下，如果向右倒下，将影响从 X+1 到 X+H 区间内的每张多米诺骨牌，如果向左倒下，则影响从 X−H 到 X−1 区间内的每张多米诺骨牌。 该范围内的任何多米诺骨牌也会倒下，并依次以其自身的高度和方向递归地触发相同的过程。 

最后的任务是确定在两次初始推动通过系统传播后最终会倒下多少多米诺骨牌。 

约束条件很大，多米诺骨牌多达 10 万张，坐标多达 1000 万个。 一个简单地模拟每次跌倒并重复扫描新受影响的多米诺骨牌的解决方案在最坏的情况下会退化为二次行为，这远远超出了极限。 这立即排除了任何重复线性搜索新击中棋子或重新计算到达间隔而不进行预处理的方法。 

连锁效应带来了一个微妙的困难。 多张倒下的多米诺骨牌可能会到达多米诺骨牌，但只应计算一次。 当 K 的一侧不存在多米诺骨牌时，就会出现另一种边缘情况，这意味着只有一个级联启动或根本没有启动。 同样重要的是，坐标是稀疏且无序的，因此在不排序的情况下直接对位置进行索引是不可能的。 

一个小的说明性边缘情况是多米诺骨牌的间距刚好超出了可及范围：

 输入：```
3 10
0 3
5 3
9 1
```如果最右边的多米诺骨牌被推动但无法到达中间，链条会立即停止。 假设连续位置的简单方法会错误地通过空白空间传播。 

## 方法

 暴力模拟将反复扩大当前倒下的多米诺骨牌组。 每次多米诺骨牌倒下时，我们都会扫描所有其他多米诺骨牌，看看它们是否位于其到达区间且尚未倒下。 这会导致直接的 O(N²) 最坏情况行为，因为 N 个多米诺骨牌中的每一个都可能触发对所有剩余骨牌的另一次扫描。 

效率低下的原因是在任意间隔内反复搜索新受影响的多米诺骨牌。 关键的观察是我们实际上并不需要每次都搜索所有位置。 如果多米诺骨牌按坐标排序，则每个传播步骤都会成为排序数组的范围扩展。 我们可以维护一个结构，使我们能够快速找到可达间隔内的所有多米诺骨牌并对其进行一次处理，而不是扫描所有内容。 

这将问题转化为排序列表上的可达性过程，其中每个多米诺骨牌最多被访问一次。 使用队列或类似 DFS 的传播，我们可以扩展间隔并在排序数组上推进指针，而无需重新访问元素。 每个多米诺骨牌都会排队一次并贡献一次扩展，从而在排序后提供线性复杂度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N²) | O(N) | 太慢了 |
 | 排序传播| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 首先，按坐标对所有多米诺骨牌进行排序。 这是至关重要的，因为它允许我们将空间范围视为线性阵列中的连续段。 

其次，确定初始激活点。 我们使用对排序坐标进行二分搜索来定位 K 的严格左侧最近的多米诺骨牌和 K 的严格右侧最近的多米诺骨牌。 这些是唯一的起点，因为最初只推送这两个点。 

第三，我们使用队列分别模拟左右方向的传播。 每个队列条目代表一个刚刚倒下的多米诺骨牌，现在将其影响向外扩散。 

第四，在处理多米诺骨牌时，我们根据其方向计算其可达区间。 对于位置为 X、高度为 H 的右倾多米诺骨牌，我们要激活坐标在 (X, X+H] 中的所有多米诺骨牌。由于列表是已排序的，因此我们可以从当前位置前进指针以找到此范围内的所有索引。类似地，对于左倾多米诺骨牌，我们扩展到 (X−H, X)。

 第五，每一个新到达的尚未被访问的多米诺骨牌都被标记为倒下，并以与触发它的多米诺骨牌相同的方向推入队列。 这可确保传播正确继续。 

最后，我们计算所有访问过的多米诺骨牌。 

### 为什么它有效

 关键的不变量是，每当多米诺骨牌出队时，位于其到达间隔内的所有多米诺骨牌要么已经被处理，要么将只被排队一次。 因为我们在排序数组中单调向前扫描，所以不会重新访问或错误地跳过任何索引。 每个多米诺骨牌从不可见到可见的转变仅一次，并且其影响在其传播方向上仅应用一次。 这保证了总访问集与通过两次初始推送的链式间隔扩展可到达的所有多米诺骨牌完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from bisect import bisect_left, bisect_right
from collections import deque

def solve():
    n, k = map(int, input().split())
    dom = []
    for _ in range(n):
        x, h = map(int, input().split())
        dom.append((x, h))
    
    dom.sort()
    xs = [x for x, _ in dom]
    
    vis = [False] * n
    q = deque()

    # left initial
    i = bisect_left(xs, k) - 1
    if i >= 0:
        vis[i] = True
        q.append((i, -1))  # direction: left

    # right initial
    j = bisect_right(xs, k)
    if j < n:
        vis[j] = True
        q.append((j, 1))   # direction: right

    while q:
        idx, d = q.popleft()
        x, h = dom[idx]

        if d == 1:
            reach = x + h
            nxt = idx + 1
            while nxt < n and dom[nxt][0] <= reach:
                if not vis[nxt]:
                    vis[nxt] = True
                    q.append((nxt, 1))
                nxt += 1
        else:
            reach = x - h
            nxt = idx - 1
            while nxt >= 0 and dom[nxt][0] >= reach:
                if not vis[nxt]:
                    vis[nxt] = True
                    q.append((nxt, -1))
                nxt -= 1

    print(sum(vis))

if __name__ == "__main__":
    solve()
```该解决方案首先对多米诺骨牌进行排序，使空间关系成为基于索引的间隔。 两个初始推动是使用 K 周围的二分搜索找到的。一个微妙之处是我们必须严格选择最近的邻居，而不是每一侧的所有多米诺骨牌，这就是我们使用的原因`bisect_left`和`bisect_right`。 

BFS队列同时存储索引和方向，因为方向决定了传播间隔。 一旦多米诺骨牌倒下，它总是会继续沿着它被推动的方向传播，因为问题将方向定义为每次初始推动时固定的方向，并且传播会保留它。 

内部循环单调地推进指针，确保每个方向链最多访问每个索引一次。 这样可以避免从头开始重复扫描，并保证线性总遍历。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(N log N) | O(N log N) | 排序占主导地位； 每个多米诺骨牌在 BFS 中处理一次 |
 | 空间| O(N) | 多米诺骨牌列表、访问数组和队列的存储 |

 该算法非常适合约束条件，因为每个多米诺骨牌最多被插入队列一次，并且每个多米诺骨牌在传播过程中仅在一次单调传递中被扫描。 唯一的超线性步骤是排序。

 ## 边缘情况

 一种边缘情况是 K 位于两个多米诺骨牌之间，但任何一方都无法进一步传播。 在这种情况下，仅访问初始邻居。 该算法可以处理此问题，因为队列最多使用两个条目进行初始化，并且两者都会快速耗尽其可达间隔而不添加新节点。 

另一种情况是一条长链，其中每个多米诺骨牌几乎无法到达下一个。 该算法仍然可以正确执行，因为每个多米诺骨牌进入队列一次，并且传播通过排序索引线性继续，而无需重新检查较早的元素。 

最后一种情况是所有多米诺骨牌都位于 K 的一侧。二等分逻辑确保只创建一个初始种子，并且 BFS 自然地成为对连续线段的单向遍历，仍以 O(N) 为界。
