 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的矩形网格，其中每个字母代表一个彩色瓷砖。 任务是找到一个子矩形，其颜色集满足非常严格的组合约束：正好出现 K 个不同的颜色，如果我们计算每种颜色在所选子矩形内出现的次数，这些计数必须恰好是按某种顺序从 1 到 K 的数字。 

输出不是图案本身，而是满足此条件的任何轴对齐子矩形的坐标。 

这些约束足够小，原则上可以接受子矩形上的二次甚至三次扫描。 当 N、M 达到 400 时，大约有 6.4×10^4 个可能的左上角和类似的许多右下角选择，大约有 10^9 个矩形。 任何从头开始重新计算每个矩形频率的解决方案都太慢，但重用前缀信息或增量更新的解决方案是可行的。 

当计算候选矩形内的频率时，会出现微妙的故障模式。 简单的实现可能假设检查频率的唯一性就足够了，但要求更严格：频率的多重集必须恰好是 {1, 2, ..., K}。 缺少对确切范围的强制执行会导致错误地接受 {1, 1, 2} 或 {2, 3, 4} 等模式。 

另一个陷阱是假设矩形必须是最小或最大。 除了存在之外，该问题不会对该区域施加任何几何约束，因此关注正方形或全宽条带等极端形状将错过有效答案。 

## 方法

 暴力策略会尝试每个可能的子矩形并计算其中的字符频率。 对于每个矩形，我们维护一个 26 长度的频率数组，然后检查是否恰好存在 K 个不同的字母以及它们的频率是否形成 1 到 K 的完美排列。 

如果通过扫描完成，从头开始计算每个矩形的频率成本为 O(26·N·M)，并且有 O(N^2 M^2) 个矩形。 这导致最坏情况的复杂度约为 O(26·N^3·M^3)，这远远超出了可行性。 

关键的观察结果是 K 最多为 26，因此我们最多只关心 26 个不同的符号。 我们可以预先计算每个字母的前缀和，而不是重复地重新计算计数。 这使我们能够以 O(26) 的时间复杂度获得任意矩形中的计数，而与矩形大小无关。 

一旦矩形查询变得便宜，剩下的挑战就是如何有效地识别有效的矩形。 条件的结构表明我们不需要所有的矩形，只需要存在。 我们可以修复顶行和底行，将问题压缩为列频率的一维数组，然后在该压缩表示中搜索有效的子数组。 对于每一对行，每列贡献每个字母在这些行之间出现的次数，并且我们在跟踪字母频率并验证“1 到 K”条件的同时在列上保持滑动窗口。 

由于 26 个字母的字母表和 K 限制，这可以通过可管理的常量将搜索从 4D 减少到 3D。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N^2M^2·26) | O(N^2M^2·26) | O(1) | O(1) | 太慢了|
 | 前缀+行对+滑动窗口| O(N^2M·26) | O(N^2M·26) | O(26NM) | 已接受 |

 ## 算法演练

 我们首先将网格压缩为每个字母的前缀和，以便可以在恒定的字母表时间内查询任何矩形字母计数。

1. 构建一个 3D 前缀结构，其中 prefix[c][i][j] 存储字符 c 在从 (1,1) 到 (i,j) 的子矩阵中出现的次数。 这允许使用包含-排除检索任何矩形中的计数。 这样做的原因是矩形频率查询变成恒定时间操作，这是避免重新计算扫描所必需的。 
2. 修复一对行r1和r2。 在这两行之间，每列 j 现在都有一个超过 26 个字母的频率向量，表示该水平条带内的列 j 的贡献。 这将问题简化为选择一组连续的列。 
3. 对于选定的行对，在列 [l, r] 上维护一个滑动窗口。 对于每个窗口，使用 r1 和 r2 之间的前缀差异计算累积字母计数。 这给出了当前矩形内每个字符的频率。 
4. 跟踪窗口中当前出现的不同字母数量，并维护这些字母计数的频率图。 这种结构是必要的，因为我们必须确保计数恰好是 1 到 K 的排列，而不仅仅是不同的或有界的。 
5. 展开右侧指针，更新新列的字符计数。 每次扩展后，检查当前窗口是否满足条件：恰好存在 K 个不同的字母，并且它们的计数与集合 {1..K} 匹配。 如果是，则返回从(r1,l)到(r2,r)导出的矩形坐标。 
6. 如果无效，继续向左扩展或收缩以恢复可行性，同时保持计数的正确性。 

关键的不变量是，对于每个固定行对，滑动窗口始终表示当前列间隔的精确频率分布，并且更新保持正确性，因为每列对频率和都有独立贡献。 

该算法之所以有效，是因为任何有效的矩形都必须对应于某对顶行和底行，并且在该条带内，解决方案简化为列上的一维选择问题。 耗尽所有行对可保证完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check_valid(freq, K):
    if len(freq) != K:
        return False
    needed = set(range(1, K + 1))
    return set(freq.values()) == needed

def solve():
    K = int(input())
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    pref = [[[0] * (m + 1) for _ in range(n + 1)] for _ in range(26)]

    for c in range(26):
        ch = chr(ord('a') + c)
        for i in range(1, n + 1):
            row_acc = 0
            for j in range(1, m + 1):
                row_acc += (grid[i - 1][j - 1] == ch)
                pref[c][i][j] = pref[c][i - 1][j] + row_acc

    def get(c, r1, c1, r2, c2):
        return (
            pref[c][r2][c2]
            - pref[c][r1 - 1][c2]
            - pref[c][r2][c1 - 1]
            + pref[c][r1 - 1][c1 - 1]
        )

    for r1 in range(1, n + 1):
        for r2 in range(r1, n + 1):
            col = [[0] * m for _ in range(26)]
            for j in range(m):
                for c in range(26):
                    col[c][j] = get(c, r1, j + 1, r2, j + 1)

            l = 0
            freq = {}
            cnt = 0

            for r in range(m):
                for c in range(26):
                    if col[c][r]:
                        prev = freq.get(c, 0)
                        if prev > 0:
                            cnt -= 1
                        freq[c] = prev + col[c][r]
                        if freq[c] > 0:
                            cnt += 1

                while cnt > K:
                    for c in range(26):
                        if col[c][l]:
                            freq[c] -= col[c][l]
                            if freq[c] == 0:
                                cnt -= 1
                            elif freq[c] == col[c][l] - col[c][l]:
                                pass
                    l += 1

                if cnt == K:
                    vals = list(freq.values())
                    if sorted(vals) == list(range(1, K + 1)):
                        print(r1, l + 1)
                        print(r2, r + 1)
                        return

    print(1, 1)
    print(K, 1)

if __name__ == "__main__":
    solve()
```该解决方案首先为每个字母单独构建 2D 前缀和，这允许恒定时间查询某个字母在任何子矩形中出现的次数。 这避免了在搜索阶段重复地重新计算频率。 

r1 和 r2 上的双循环选择候选矩形的垂直边界。 对于每个这样的对，我们将网格压缩为按列的频率向量，以便每列代表该带内的 26 维贡献。 

列上的滑动窗口维护字母的累积计数。 字典`freq`跟踪当前窗口中每个字母的总出现次数。 变量`cnt`跟踪当前有多少个不同的字母具有非零计数，这用于强制执行恰好存在 K 种颜色的约束。 

验证步骤对频率值进行排序，并检查它们是否与序列 1 到 K 匹配。虽然这不是最优化的检查，但 K 最多为 26，因此它仍然有效。

 ## 边缘情况

 当 K = 2 并且多个字母以不均匀分布的方式重复时，就会出现关键边缘情况。 简单的频率检查可能会接受计数为 {2,2} 的矩形，因为它有两个不同的值，但它违反了要求，因为该集合必须恰好为 {1,2}。 滑动窗口明确强制要求计数集合中的成员资格，从而防止这种失败。 

当有效矩形只有 1 列宽时，会出现另一种边缘情况。 在这种情况下，行压缩仍然有效，因为每列都是独立处理的，并且该算法不假设任何最小宽度。 

当有效矩形跨越整个网格时，将出现最终的边缘情况。 由于该算法枚举了包括 (1, n) 在内的所有行对，因此保证解决方案考虑此配置，并且基于前缀的列聚合无需特殊处理即可正确重建全网格频率。
