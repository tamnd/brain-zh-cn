 ## 解决方案
 ## 问题理解

 我们有一个系统可以产生总“体积”，最初是某个值$M$。 我们想要调整一组$N$独立调节器使最终体积精确$V$。 每个调节器通过比例因子成倍地贡献于总体积：将调节器设置为值$x$总体积乘以$x$相对于其最大配置。 

每个调节器都有一个最大允许设置$m_i$。 我们从所有调节器的最大值开始，并且我们可以将它们减少到以下任意值：$1$或不超过的素数$m_i$。 最终体积是通过将所有选定的调节器值相乘并将其与最大配置的乘积进行比较来确定的。 

一个有用的重新表述是将一切都视为一个比率。 令最大可达到的体积与$\prod m_i$。 设定调节器$i$重视$a_i$将体积缩放一个因子$a_i / m_i$。 因此最终条件变成了一个纯乘法方程：$$\prod a_i = V$$的约束条件是每个$a_i$是$1$或素数不超过$m_i$。 

所以任务是给每个调节器分配一个值$1$或素数，使得它们的乘积等于$V$，尊重每个索引的上限。 

约束条件$N \le 50$足够小，我们可以对子集或因子分配进行指数或伪多项式推理。 然而，$V < 10^9$意味着素因数分解是有界的但不平凡； 我们必须依靠因素结构而不是粗暴地列举价值。 

一个关键的边缘情况是当$V = 1$。 那么所有调节器必须设置为$1$，但我们仍然必须确保$1$是允许的（明确允许的）。 另一个边缘情况是当$V$有一个比所有素因子都大的素因子$m_i$; 在这种情况下，即使因式分解在代数上存在，也不可能进行赋值。 

另一种微妙的失败模式是假设我们可以自由地使用素数而不考虑每个调节器的限制。 例如，如果$V = 49$我们需要两个 7，但只有一个调节器允许 7，我们不能任意分割该因子。 

## 方法

 蛮力的想法是尝试监管机构的所有任务，选择其中之一$1$或一些允许的素数，并检查乘积是否等于$V$。 每个调节器大约需要$\pi(m_i)$主要选择加上$1$，所以即使我们积极简化，分支因子也很大。 在最坏的情况下，这大约变成$O((\text{number of primes up to }10^9)^N)$，即使对于非常小的情况也是完全不可行的$N$。 

使问题可解决的结构是目标是单个整数并且所有允许的值都是素数或$1$。 这迫使最终产品因式分解$V$最多进入$N$部分，每个部分是$1$或一个素因数$V$。 因此，我们可以考虑分配质因数分解，而不是搜索分配$V$跨职位。 

这将问题转化为质数权力的约束分配。 对于每个素数$p$划分$V$，我们必须分配所有出现的$p$一些监管机构的最大支持至少$p$。 由于每个调节器最多贡献一个素因数（因为它是$1$或单个素数），任务就变成跨索引分配素数指数。 

我们一一处理素数，并贪婪地将它们分配给具有足够容量的可用监管者。 如果在任何时候都无法将主要因素分配给足够多的不同监管机构，那么答案就是不可能的。

 | 蛮力 | 指数为$N$和值范围| O(N) | 太慢了|
 | 素因子分布 |$O(N \sqrt{V})$| O(N) | 已接受 |

 ## 算法演练

 我们将问题重新解释为分配质因数$V$给监管机构。 

1. 因式分解$V$转化为具有重数的素数。 我们存储对$(p, e)$。 这是唯一重要的分解，因为任何有效的解决方案都必须准确地重现此分解。 
2. 对于每个调节器$i$，确定它可以支持哪些素数。 监管者只能采取素数$p$如果$p \le m_i$。 否则必须设置为$1$。 
3.我们对每个素数进行维护$p$可以托管它的候选索引列表。 每个监管器最多可以托管一个素数，因此我们必须确保没有索引被多次使用。 
4. 我们按降序处理素数$p$。 这种顺序很有帮助，因为大素数的合格监管者较少，因此我们首先分配它们以避免阻塞。 
5. 对于每次出现的素数$p$，我们将其分配给候选列表中未使用的调节器。 如果我们用完了可用的监管机构，我们会立即断定这是不可能的。 
6. 分配所有素数出现后，任何未使用的调节器均设置为$1$。 使用的调节器被设置为其指定的初始值。 

### 为什么它有效

 正确性取决于每个调节器恰好贡献一个乘法因子，或者$1$或素数。 这意味着每个调节器最多可以在因式分解中使用一次$V$。 问题变成素数出现和调节器之间的二分匹配，其中仅当调节器允许该素数时才存在边缘。 

素数递减顺序中的贪婪分配是有效的，因为较大的素数具有更严格的放置约束。 如果无法放置大素数，则使用较小素数的重新排列无法在不违反容量限制的情况下解决该问题，因为交换较小素数不会增加大素数的资格。 因此，首先处理最受约束的元素可以保留可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def factorize(x):
    primes = []
    d = 2
    while d * d <= x:
        if x % d == 0:
            cnt = 0
            while x % d == 0:
                x //= d
                cnt += 1
            primes.append((d, cnt))
        d += 1
    if x > 1:
        primes.append((x, 1))
    return primes

def solve():
    M, V, N = map(int, input().split())
    m = list(map(int, input().split()))

    factors = factorize(V)

    # expand prime factors into list of occurrences
    items = []
    for p, c in factors:
        for _ in range(c):
            items.append(p)

    # sort primes descending
    items.sort(reverse=True)

    used = [False] * N
    ans = [1] * N

    # for each prime occurrence, assign to best available regulator
    for p in items:
        idx = -1
        for i in range(N):
            if not used[i] and m[i] >= p:
                idx = i
                break
        if idx == -1:
            print(-1)
            return
        used[i] = True
        ans[i] = p

    print(*ans)

if __name__ == "__main__":
    solve()
```代码首先进行因式分解$V$，因为任何有效的配置都必须准确地再现其主要结构。 我们将因式分解扩展为素数出现的平面列表，以便自然地处理重数。 

然后，我们按降序对素数进行排序，以确保我们首先分配较大的素数。 分配循环尝试将每个素数放入可以支持它的未使用的调节器中。 如果不存在这样的监管机构，我们会提前终止。 

每个调节器一旦收到素数就被标记为已使用，从而强制每个调节器最多贡献一个素数因子的约束。 仍保留未使用的调节器$1$，匹配允许的中性元素。 

一个微妙的点是，对有效索引的贪婪扫描是线性的，这很好，因为$N \le 50$。 由于规模较小，不需要更复杂的匹配。

 ## 边缘情况

 一个重要的边缘情况是当$V = 1$。 因子列表变为空，因此每个调节器都保持未使用状态，应设置为$1$。 该算法自然地处理这个问题，因为没有进行素数分配并且初始数组全是 1。 

另一种情况是当所需素数大于所有素数时$m_i$。 例如，如果$V = 17$和所有$m_i = 10$，因式分解产生单个 17，但没有监管机构可以接受它。 赋值循环立即失败并返回$-1$。 

第三种情况是重复素数超出可用槽位。 例如，$V = 8 = 2^3$但只有两个监管机构允许$2$。 该算法将成功分配两次出现的情况，然后在第三次出现时失败，正确检测不可能性，而不是尝试重新使用调节器。
