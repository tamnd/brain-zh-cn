 ## 解决方案
 ## 问题理解

 我们得到三个平行列表，每个列表的长度$N$，代表三种不同角色的音乐家：吉他手、贝斯手和鼓手。 每个音乐家都有一个技能值，我们必须准确地形成$N$组，其中每个组包含每个角色的一个人。 

一个团体的价值被定义为其三名成员中技能最高的人。 总利润是这些组值的总和。 任务是将音乐家分配到组中，以便每个音乐家只被使用一次，并且组最大值的总和尽可能大。 

约束条件$N \le 10^5$迫使我们粗略地思考$O(N \log N)$或更好的条款。 尝试三个数组之间的所有匹配的任何解决方案都是立即不可能的，因为这将涉及分配中的阶乘增长。 即使是分支或回溯的成对匹配策略也会发生组合爆炸。 

当贪婪配对忽略全局结构时，就会出现微妙的失败情况。 例如，假设一个角色有许多小值，而另一个角色有一些非常大的峰值。 如果我们在没有计划的情况下进行本地配对，我们可能会在不增加最大值的组中“浪费”大量值。 

考虑这个小例子：

 输入：```
G: 1 100
B: 1 2
D: 1 3
```如果我们在没有结构的情况下贪婪地配对，我们最终可能会在 1 和 1 的组中使用 100，产生 100，然后剩下的组是 (1,2,3) 给出 3，总共 103。但是这里不可能有不同的配对，所以这很好。 当一个数组中存在许多大值并且必须小心地跨组分布时，真正的问题就变得显而易见。 

主要风险在于，错误方向的天真排序或任意不匹配的极端可能会减少大值的总贡献。 

## 方法

 暴力方法将尝试枚举在三个数组中分配三元组的所有方法。 即使我们修复一种排列并匹配其他排列，我们仍然面临$N!$的可能性。 每次评估费用$O(N)$，所以这远远超出了可行的范围。 

更结构化的暴力可能会尝试对数组进行排序并以固定模式将最大的与最大的配对。 这很快，但需要理由。 问题是我们是否总是可以假设一种顺序，使最佳分配易于表达。 

关键的洞察力来自于观察对答案的实际贡献。 每个组都贡献最大的值，因此只有所选的三个组中最大的元素才重要。 如果我们反过来思考，每个大值都可以“声明”一个组，只要它不被已分配给该组的更大值阻止即可。 

这表明应尽可能将大值放入不同的组中，因为将它们堆叠在一起会浪费贡献：只有最大计数，因此将大值合并到同一组中会丢弃潜在的收益。 

这导致了一种贪婪策略：按降序对所有角色的所有值进行排序，并将它们逐一分配到组中，始终以跨角色的结构化循环方式将下一个最大的未使用值放入下一个可用组中。 更准确地说，我们对齐位置，以便每个索引$i$组成一个团体，由$i$每个排序角色的第 -th 元素。 那么每个组的最大值就是其三个对齐值中的最大值。 

这种对齐方式之所以有效，是因为排序确保了每个位置$i$，所有剩余元素都不比之前的元素大，因此我们永远不会因为延迟放置而“失去”潜在的最大值。

 | 蛮力 | 指数| O(N) | 太慢了|
 | 排序+贪心对齐| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们通过将问题简化为三个同步排序列表来构建解决方案。 

1. 阅读代表吉他手、贝斯手和鼓手的三个数组。 这些代表独立的池，必须在各个职位之间一对一匹配。 
2. 按非升序对三个数组中的每一个进行排序。 这确保了当我们从左到右迭代时，我们总是首先查看每个角色中最强的剩余候选人。 这种顺序至关重要，因为我们希望较大的值能够影响早期的群体形成。 
3. 按索引对元素进行配对：形成组$i$使用$i$第一位吉他手，$i$-th 贝斯手，以及$i$-th 鼓手。 这创造了$N$没有歧义的固定组。 
4. 对于每个组，将其贡献计算为三个选定值中的最大值。 将其添加到运行总计中。 
5. 输出总和。 

唯一不明显的步骤是为什么排序后的索引配对是有效的。 直觉是排序可以跨角色对齐“等级”，因此$i$每个数组中的第 - 个元素代表$i$- 剩下的最强候选人。 匹配相等的等级可以避免将非常强的元素与非常弱的元素混合在一起，从而浪费已经很强的群体中的强元素。 

### 为什么它有效

 该结构依赖于这样一个事实：只有每个三元组中的最大值才重要。 排序后，位置上的任意元素$i$保证至少与位置上的所有元素一样大$j > i$。 这意味着当我们组成小组时$i$，我们可以从剩余元素中获得的最佳可能最大值已经由当前对齐的三元组表示。 任何尝试将较大元素移动到较后组的替代重新排列都会将较小的最大值强制到较早的组中，而不会提高总和​​，因为最大值不会从同一组中的多个大元素中累积。 这保留了交换下的最优性：一旦数组被排序和对齐，组之间交换分配就不能增加总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    g = list(map(int, input().split()))
    b = list(map(int, input().split()))
    d = list(map(int, input().split()))

    g.sort(reverse=True)
    b.sort(reverse=True)
    d.sort(reverse=True)

    ans = 0
    for i in range(n):
        ans += max(g[i], b[i], d[i])

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案读取三个角色数组并按降序对它们进行排序。 排序可确保最强的剩余候选者在相同索引处对齐。 然后，循环通过索引隐式构造每个组，并且我们将每个组的贡献计算为其三个对齐值中的最大值。 该实现依赖于 Python 的高效排序，并避免任何额外的数据结构。 

一个常见的错误是按升序排序或错误地混合索引。 另一个微妙的问题是假设我们需要明确的分组结构，这是不必要的，因为每个组只有最大的问题。

 ## 边缘情况

 一种边缘情况是，一个数组具有一个极大的值，而其他数组都非常小。 例如：```
n = 3
G = [100, 1, 1]
B = [2, 2, 2]
D = [3, 3, 3]
```排序后：```
G = [100, 1, 1]
B = [2, 2, 2]
D = [3, 3, 3]
```按索引分组给出：

 | 我| G | 乙| d | 最大|
 | ---| ---| ---| ---| ---|
 | 0 | 100 | 100 2 | 3 | 100 | 100
 | 1 | 1 | 2 | 3 | 3 |
 | 2 | 1 | 2 | 3 | 3 |

 该算法正确地将 100 人隔离在自己的组中，并防止其被稀释。 

另一种情况是所有数组都相同但未排序：```
G = [3, 1, 2]
B = [2, 3, 1]
D = [1, 2, 3]
```排序后，全部变为[3,2,1]，分组产生一致对齐的最大值。 如果不进行排序，任意配对可能会将大值分散到次优位置，从而减少总和。
