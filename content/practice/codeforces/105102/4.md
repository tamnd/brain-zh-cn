 ## 解决方案
 ## 问题理解

 每个测试用例都给出一个整数矩形网格，其中每个单元格代表一个“味道”值，可以是负值或正值。 对于每个这样的网格，任务是选择任意大小的方形子网格，并找到所有包含的单元格之和尽可能大的子网格。 答案只是最大和的值，而不是平方本身。 

关键的自由度在于，正方形可以具有从 1 到网格较小尺寸的任意边长，并且可以将其放置在网格内的任何位置，只要它完全位于边界内即可。 

这些约束的构建方式改变了我们对复杂性的看法。 每个网格最多可以为 500 x 500，但测试用例中所有高度的总和最多为 500。这意味着我们永远不会同时面对许多大网格。 相反，我们可能有一个大网格或几个较小的网格，但总体处理的行总数是有上限的。 

这使得在单个网格维度边界上立方的解决方案是可以接受的，但任何比粗略更糟糕的解决方案$O(500^3)$每个最坏情况的网格都是危险的。 独立地重新计算每个方格总和的幼稚方法会过于频繁地重复工作。 

一些边缘情况对于正确性很重要：

 除一个单元格外所有值均为负值的网格。 答案必须是单个单元格，因为 1 x 1 的正方形始终是允许的。 例如：

 输入：```
1
2 2
-5 -6
-7 10
```输出：```
10
```只考虑更大的正方形或假设更大的面积更好的天真的解决方案在这里会失败。 

另一种边缘情况是当最佳正方形的尺寸不是最大但也不是 1 x 1 时。例如：```
2
3 3
1 -2 1
-2 10 -2
1 -2 1
```最佳正方形是完整的 3 x 3 网格，但在许多情况下，最佳正方形可能是较大矩阵内的 2 x 2。 任何解决方案都必须正确比较所有尺寸。 

## 方法

 一种直接的方法是枚举网格中每个可能的方格。 对于每个左上角，对于每个可能的边长，从头开始计算该平方和。 简单地计算每个方块的成本$O(k^2)$，所以在所有位置和尺寸上，这大致变成$O(n^4)$最坏情况下每个网格。 和$n = 500$，这是完全不可行的。 

第一个改进是避免从头开始重新计算总和。 二维前缀和允许我们在预处理后在恒定时间内计算任何子矩阵和。 这降低了评估一平方的成本$O(1)$。 

即使在优化之后，我们仍然需要考虑每一个可能的方格。 对于固定边长$k$，我们滑动一个$k \times k$网格上的窗口并计算每个总和$O(1)$。 有$O(nm)$每个位置$k$，直到$O(\min(n, m))$的选择$k$。 这导致$O(nm \cdot \min(n, m))$每个网格的解决方案，考虑到总输入大小限制，这是可以接受的。 

关键的观察是，我们不需要任何更高级的结构，如线段树或 Kadane 式 2D 压缩，因为前缀和已经将每个候选方的评估减少到恒定时间，并且由于总行数的全局约束，候选总数是可控的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（重新计算每个方块）|$O(n^4)$|$O(1)$| 太慢了 |
 | 前缀和+枚举平方|$O(nm \cdot \min(n,m))$|$O(nm)$| 已接受 |

 ## 算法演练

 我们独立处理每个网格。 

1. 构建一个 2D 前缀和数组，其中每个条目存储矩形中从左上角到该单元格的所有值的总和。 这允许对任何子矩形进行恒定时间查询。 
2. 迭代所有可能的正方形边长，从 1 到网格尺寸的最小值。 每个尺寸代表不同的候选方块族。 
3. 对于固定边长，将正方形滑过网格中每个有效的左上角位置。 每个位置定义一个独特的正方形。 
4. 对于每个方格，使用前缀和表在常数时间内计算其总和。 这避免了重复重新计算内部总和。 
5. 跟踪所有大小的所有方格遇到的最大总和。 
6.检查所有配置后输出最终最大值。 

### 为什么它有效

 前缀和表保证每个平方和查询都是精确的并且在恒定时间内计算。 由于每个可能的方块都会通过其左上角和边长检查一次，因此不会错过任何候选方块。 该算法不依赖于任何关于大小或位置的贪婪假设，因此最大值是通过对所有有效方块进行详尽而有效的枚举来找到的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        a = [list(map(int, input().split())) for _ in range(n)]

        ps = [[0] * (m + 1) for _ in range(n + 1)]

        for i in range(n):
            row_sum = 0
            for j in range(m):
                row_sum += a[i][j]
                ps[i + 1][j + 1] = ps[i][j + 1] + row_sum

        def get_sum(x1, y1, x2, y2):
            return ps[x2][y2] - ps[x1][y2] - ps[x2][y1] + ps[x1][y1]

        ans = -10**18
        limit = min(n, m)

        for k in range(1, limit + 1):
            for i in range(n - k + 1):
                for j in range(m - k + 1):
                    s = get_sum(i, j, i + k, j + k)
                    if s > ans:
                        ans = s

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案构建了一个标准的 2D 前缀和，以便在恒定时间内通过包含-排除来计算任何矩形和。 功能`get_sum`是防止循环内重复求和的核心抽象。 

三环结构是有意的：边长、行位置和列位置。 交换这些循环不会影响正确性，但会稍微影响缓存行为。 所选择的顺序使逻辑简单明了，并最大限度地减少索引中的错误。 

一个常见的陷阱是不正确的前缀索引。 该实现使用大小为的填充前缀数组$(n+1) \times (m+1)$，因此每个矩形查询都避免负索引和边界检查。

 ## 边缘情况

 除一个单元格外全部为负值的网格测试算法是否正确考虑 1 x 1 方格。 前缀和方法仍然评估 k = 1，因此正确选择了最大单个单元格。 

最佳答案是整个矩阵的网格可确保较大的正方形不会被忽略。 由于该算法迭代所有 k 直到 min(n, m)，因此在有效时会显式评估整个网格。 

具有混合符号但局部峰值较强的网格检查滑动窗口是否正确捕获局部最大值。 因为每个位置都是针对每个正方形大小进行评估的，所以不会跳过任何区域，并且前缀和保证了每个候选评估的正确性。
