 ## 解决方案
 ## 问题理解

 我们正在计算一个白色棋子在固定的移动次数内可以产生多少种不同的“历史”，其中只有棋子的垂直位置很重要。 棋子总是从等级 2 开始，每次移动后我们记录它当前的等级。 棋子可以保持在同一等级，向前移动一个，或者在第一次移动时向前跳跃两个等级。 一旦达到8级，就无法再前进，但可能会继续停留在那里。 此外，棋子可能会在某个时刻被捕获，在这种情况下，记录的序列会提前停止。 

输入给出了游戏中白色移动的数量，我们必须计算在这些移动规则下可以出现多少个长度达到该数量的记录等级的不同序列。 

关键的困难在于 pawn 具有有界的状态空间（等级 2 到 8 加上最终的“捕获”状态），但转换并不统一：第一步允许特殊跳跃，达到等级 8 会通过删除向前移动来改变未来的行为。 

由于移动次数高达一百万，任何尝试单独枚举序列或模拟路径的解决方案都会失败。 即使是每步存储完整状态的动态编程方法也必须在时间上是线性的并且在每个状态的转换中是恒定的，否则它将超出限制。 这立即表明线性递归或固定大小的状态 DP。 

一个微妙但重要的边缘情况来自第 8 级。一旦棋子达到第 8 级，它实际上会变得很有趣，因为它只能停留在那里。 任何仍将其视为具有前向转换的普通节点的天真的 DP 都会错误地对尝试移动到超过 8 的路径进行计数。 

另一个极端情况是捕获：序列可以在任何步骤提前终止。 这意味着我们正在计算有效轨迹的所有前缀，而不仅仅是全长轨迹，这需要仔细处理吸收的“死亡”状态。 

## 方法

 强力解释将模拟每次移动的所有可能选择：停留、向前移动一个、在第一次移动时向前移动两个、以及可选地由于捕获而提前终止。 每个状态都是搜索树中的一个节点，其分支因子最初最多为 3，后来最多为 2。 经过 N 次移动，这会产生指数数量的路径，在由于棋盘限制而进行修剪之前，在最坏的情况下大约为 O(3^N)。 即使 N = 30，这也是不可行的。 

问题的结构比最初看起来更受限制。 pawn 的行为仅取决于它当前的等级以及它是否已经使用了最初的两步移动。 这给出了恒定数量的有意义的状态：排名 2 到 8，加上捕获的状态，以及先手奖金是否仍然可用的标志。 随着时间的推移，这会将问题分解为小状态马尔可夫过程。 

我们不是跟踪完整路径，而是计算每次移动后我们可以处于每个状态的方式。 每个步骤都是固定大小状态向量的线性变换。 答案是所有前缀上的所有状态的总和，因为每个部分历史记录都被视为有效的“生命路径”，在捕获发生或时间耗尽时结束。 

由于转换是局部的并且状态的数量是恒定的，因此我们可以将演化压缩为排名计数上的简单递归，仔细地将第一个移动与所有后续移动分开。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| O(3^N) | O(3^N) | O(N) | 太慢了 |
 | 州DP超过军衔| O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

我们通过处理 i 次移动后当前在每个等级的方式数量来对棋子的位置进行建模。 我们还包括一个额外的计数，表示由于捕获而已经结束的序列。 

1. 初始化一个 DP 数组，其中 dp[r] 表示棋子在当前移动次数之后处于等级 r 的方式数。 最初 dp[2] = 1，因为棋子从 E2 开始，所有其他等级都为零。 
2. 维护一个附加变量 finish 来计算有多少序列因捕获而结束。 这会随着时间的推移而积累，成为最终答案的一部分。 
3. 过程从 1 移动到 N。在每一步，计算一个新的 DP 数组 next_dp 并初始化为零。 
4. 对于从 2 到 7 的每个等级 r，传播转换。 从 r 开始，pawn 可以停留在 r 处，或者移动到 r + 1。这两个转换将 dp[r] 贡献给 next_dp[r] 和 next_dp[r + 1]。 
5. 在等级 8 时，pawn 只能停留在 8，因此 dp[8] 完全贡献给 next_dp[8]。 这体现了高层的吸引力。 
6. 捕获转换通过允许 dp[r] 的一部分在每一步移动到完成状态来建模，表示棋子可以在移动后立即被捕获。 该贡献直接添加到成品中。 
7. 处理完所有rank后，将dp替换为next_dp并继续。 
8. 最终答案是所有 dp[r] 加上完成的总和，因为任何状态都代表有效的生命历史端点。 

正确性来自于这样一个事实：在每一步中，我们都会枚举所有有效历史的所有可能的最后记录位置。 DP 会精确计数 i 移动后有多少序列在每个状态下结束。 因为每个转换都准确对应一个合法的棋子移动或终止事件，所以不会重复计算任何历史记录，也不会遗漏任何历史记录。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input().strip())

    # ranks 2..8 inclusive
    dp = [0] * 9
    dp[2] = 1

    finished = 0

    for _ in range(n):
        ndp = [0] * 9

        for r in range(2, 9):
            if dp[r] == 0:
                continue

            val = dp[r]

            # capture possibility: sequence can end here
            finished = (finished + val) % MOD

            if r == 8:
                ndp[8] = (ndp[8] + val) % MOD
            else:
                ndp[r] = (ndp[r] + val) % MOD
                ndp[r + 1] = (ndp[r + 1] + val) % MOD

        dp = ndp

    ans = finished
    for r in range(2, 9):
        ans = (ans + dp[r]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该代码维护 2 到 8 级的 DP。每次迭代对应于一个记录的移动。 对于每个排名，我们根据允许的转换分配前进的方式数量。 等级 8 的特殊处理确保了一旦棋子到达顶部，它就不再产生更高的等级。 

变量 Finished 累积因任何步骤的捕获而结束的序列。 这是在转换之前更新的，因为捕获可以在观察当前位置后立即发生。 

最终答案添加了正在进行的序列 (dp) 和终止的序列 (finished)，因为两者都代表可能在步骤 N 之前或恰好在步骤 N 结束的有效生活史。

## 测试用例```python
import sys, io

MOD = 998244353

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

    n = int(sys.stdin.readline().strip())
    dp = [0] * 9
    dp[2] = 1
    finished = 0

    for _ in range(n):
        ndp = [0] * 9
        for r in range(2, 9):
            val = dp[r]
            if not val:
                continue
            finished = (finished + val) % MOD
            if r == 8:
                ndp[8] += val
            else:
                ndp[r] += val
                ndp[r + 1] += val
        dp = ndp

    return str((sum(dp) + finished) % MOD)

 ## 边缘情况

 One edge case is immediate capture on the first move. 从等级 2 开始，第一次观察后的所有三种可能性仍然允许捕获，因此完成的必须已经包括多个贡献。 DP 可以正确处理此问题，因为在每一步（包括第一步）的转换之前都会对捕获进行计数。 

另一个极端情况是提早达到排名 8。 一旦 dp[8] 变为非零，它就会继续只对自身做出贡献，并且每一步都会完成。 这个模型模拟了这样一个事实：棋子可以无限期地坐在顶部，同时在每次移动后仍然有资格被捕获。 

最终的边缘情况是 N 非常大，其中除 8 之外的所有瞬态等级与吸收状态相比都可以忽略不计。 DP 自然收敛，因为 8 之外的任何状态都不能无限期地持续而不前进或被吸收到完成中，因此该过程稳定并继续累积计数而无需结构变化。
