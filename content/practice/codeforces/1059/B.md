 ## 解决方案
 ## 问题理解

 我们得到一个二进制网格，表示由墨水单元组成的目标绘图。 每个单元格要么已填充，要么为空，我们想要确定最终的图案是否可以使用特定的冲压工具从最初为空的网格中生成。 

该工具不会绘制任意形状。 相反，它总是绘制 3 x 3 正方形的边框，而中心单元格保持不变。 每个应用程序都以某个单元格为中心，并在该 3 x 3 邻域中绘制八个周围的单元格。 该中心本身永远不会受到影响。 仅当完整的 3 x 3 块位于网格内时才能使用该操作。 

问题是是否存在一系列这样的标记操作，可以从所有空单元格开始精确地生成给定的网格。 

网格大小可达 1000 x 1000，这意味着最多 100 万个单元。 如果每次检查都很昂贵，任何尝试在每个可能的位置模拟冲压并以简单的方式重复更新网格的解决方案都有可能变得太慢。 因此，我们需要一种方法，其中每个单元仅被处理恒定次数，或者至少在恒定时间内评估每个潜在标记。 

当存在不可能属于任何有效印章边界的墨迹单元时，就会出现微妙的困难。 例如，单个孤立的`#`无法解释在一片点的海洋中间，因为每张邮票都画成一个由八个相邻邮票组成的环。 另一个有问题的模式是`#`任何可能的 3 x 3 配置都不支持的单元格，这意味着它永远不是任何有效边界的一部分。 

天真的推理的关键失败模式是假设每个`#`必须是某个邮票的中心或者每个`#`本身必须直接涂漆。 事实上，只有边界很重要，而且覆盖范围可能会以复杂的方式重叠。 

## 方法

 一个蛮力的想法是模拟冲压。 对于 3 x 3 块的每个可能的中心，如果它与目标网格不矛盾，我们可以尝试应用图章，并标记它将绘制的单元格。 在尝试了所有可能的放置位置后，我们将生成的绘制网格与目标进行比较。 

这种方法在概念上是正确的，但其缺点是它没有强制执行必要性。 我们可能会贴上不需要的印章，更糟糕的是，我们可能会错过一些事实`#`除非我们仔细检查覆盖限制，否则小区永远不会被覆盖。 在最坏的情况下，我们会检查每个单元作为潜在的中心，对于每个中心，我们检查最多 9 个单元，导致大约 900 万次操作，虽然处于临界点，但仍然是可控的。 然而，正确性变得很棘手，因为我们需要确保每个`#`是可以解释的，而不仅仅是可以通过任意标记来复制。 

关键的洞察力是扭转思维。 我们不是向前构建网格，而是检查是否每个`#`单元可以被“证明”为属于至少一个有效标记的边界。 仅当单元周围存在某个中心且位于 3 x 3 边框图案中时，才可以绘制单元。 这意味着对于每个`#`，我们检查它是否可以是某个有效中心的顶部、底部、左侧或右侧邻居。 如果不存在这样的中心，则无法进行配置。 

这减少了验证每个单元周围的局部可行性的问题。 我们扫描所有可能的中心，标记它们可以产生哪些边界细胞，然后验证每个`#`至少有一个有效的印章中心覆盖。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n·m) 至 O(n·m · 9) | O(n·m) | 太慢/难以推理 |
 | 基于中心的验证 | O(n·m) | O(n·m) | 已接受 |

 ## 算法演练

 1.我们创建一个布尔网格`good`标记以给定位置为中心的 3 x 3 印记是否有效。 如果所有八个边界单元都满足，则中心有效`#`。 这确保了在那里应用印记不会引入任何不正确的点。 
2.我们迭代所有可能的中心`(i, j)`一个完整的 3 x 3 块适合的地方。 对于每个中心，我们检查周围的八个位置。 如果他们都是`#`，我们将此中心标记为可用。 
3. 我们创建另一个布尔网格`covered`与输入网格的大小相同，最初都是 false。 这将跟踪哪些单元格可以由至少一个有效的标记来解释。 
4. 对于每个有效的中心`(i, j)`，我们将其八个边界单元标记为`covered`是真的。 这模拟了这样一个事实：该印章可能为这些细胞提供了墨水。 
5. 处理完所有中心后，我们迭代整个网格并检查每个中心`#`细胞。 如果有的话`#`没有标记在`covered`，那么就不能用任何有效的标记配置来解释，所以答案是否定的。 
6.如果全部`#`单元格被覆盖，我们返回 YES。 

### 为什么它有效

 关键的不变量是每个有效的最终`#`单元格必须可解释为至少一个 3 x 3 边框的一部分，且其中心位于网格内。 任何有效的结构仅由此类印记组成，并且每个印记仅向其边界单元提供墨水。 因此，如果一个单元格不在任何有效边界内，则任何标记序列都无法生成它。 反之，如果每一个`#`至少在一个有效边界内，我们可以以任何顺序贪婪地分配图章，因为允许重叠并且永远不会去除墨水。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]
    
    covered = [[False] * m for _ in range(n)]
    
    def ok(i, j):
        return (g[i-1][j-1] == '#' and g[i-1][j] == '#' and g[i-1][j+1] == '#' and
                g[i][j-1] == '#'                     and g[i][j+1] == '#' and
                g[i+1][j-1] == '#' and g[i+1][j] == '#' and g[i+1][j+1] == '#')
    
    for i in range(1, n - 1):
        for j in range(1, m - 1):
            if ok(i, j):
                covered[i-1][j-1] = True
                covered[i-1][j] = True
                covered[i-1][j+1] = True
                covered[i][j-1] = True
                covered[i][j+1] = True
                covered[i+1][j-1] = True
                covered[i+1][j] = True
                covered[i+1][j+1] = True
    
    for i in range(n):
        for j in range(m):
            if g[i][j] == '#' and not covered[i][j]:
                print("NO")
                return
    
    print("YES")

if __name__ == "__main__":
    solve()
```实现的核心是`ok`函数，它检查邮票是否以`(i, j)`是有效的。 我们只考虑完全位于网格内的中心，因为边界中心无法形成 3 x 3 的图案。 

这`covered`矩阵跟踪所有有效邮票中心的可达性。 每个有效中心恰好对周围八个单元做出贡献，精确匹配冲压规则。 最终扫描可确保所需的墨水单元均得到支持。 

边界处理在这里至关重要。 循环结束`i in range(1, n-1)`和`j in range(1, m-1)`确保我们在检查 3 x 3 块时永远不会访问越界邻居。

 输入：```

 ## 边缘情况

 一个极端的情况是当网格包含`#`边界附近的细胞。 例如，一个`#`在位置`(0, 0)`永远不会被任何图章覆盖，因为没有 3 x 3 块将其包含为边界单元。 该算法自然会处理这个问题，因为没有有效的中心可以将其标记为`covered`，因此被检测为未覆盖。 

另一种情况是当所有细胞都`#`。 每个中心`(i, j)`完整的 3 x 3 邻域变得有效，并且每个单元都被多个重叠的图章覆盖。 该算法仍然只需要至少一个覆盖范围，因此它可以正确输出 YES，而无需推理重叠结构。
