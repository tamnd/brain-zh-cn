 ## 解决方案
 ## 问题理解

 我们有一个固定长度的工作日和一系列服务间隔，收银员必须持续在场。 在这些服务间隔之间，存在空闲时间的间隙。 在这些空闲时间段内，收银员可以重复休息，每次休息时间恰好是固定的分钟数，但不允许错过任何客户服务时段。 

任务是计算在客户之间的所有可用空闲时间中可以放置多少个相同长度的完整休息时间，假设休​​息时间不能与服务间隔重叠并且必须完全适合空闲时段。 

从结构上看，一天是从时间0到时间L的一个线段。有些子线段被客服间隔挡住了，其他的都是免费的。 该问题简化为计算有多少个长度为 a 的不相交线段可以放入所有空闲线段的并集中。 

这些约束足够大，任何解决方案都必须相对于客户数量以线性时间运行。 由于间隔多达 100,000 个且 L 高达 1e9，任何逐分钟模拟的尝试都立即不可能。 即使在精细分辨率下迭代每个间隙也是可以接受的，前提是每个间隔都处理一次。 

当根本没有顾客时，就会出现微妙的边缘情况。 然后一整天都是空闲的，我们只需计算 L 除以 a。 当服务间隔连续且没有间隙时，会出现另一种边缘情况，在这种情况下，在它们之间不能插入中断。 第三个重要的情况是当间隙存在但小于 a 时，因此它们会导致零中断。 

一个幼稚的错误是独立处理每个间隙，但忘记包括第一个客户之前的初始间隙或最后一个客户之后的最终间隙。 另一个常见错误是尝试贪婪放置而不仔细更新当前时间边界，这会导致重复计算或跳过可用空间。 

## 方法

 强力解释将明确构建时间线，标记所有繁忙的间隔，然后逐分钟扫描，计算空闲时间并贪婪地放置休息时间。 这在概念上很简单：模拟从 0 到 L 的时间，跳过服务间隔，每当我们有空闲时间时，就以大小 a 的块消耗它。 

正确性是显而易见的，因为我们实际上是在对过程进行建模。 失败点在于复杂性。 由于 L 可以达到 10^9，因此不可能迭代每分钟，每个测试用例需要大约 10^9 次操作，这远远超出了限制。 

关键的观察是我们永远不需要查看空闲段的内部。 每个空闲段独立地贡献地板（长度/a）休息。 服务间隔已经排序且不重叠，因此空闲时间自然地划分为不相交的间隙。 一旦我们认识到该结构，问题就变成了间隔内的单个线性扫描，同时维护最后一个服务间隔结束的指针。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O（左）| O(1) | O(1) | 太慢了 |
 | 间隔间隙计数| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 初始化变量`current`到0，代表我们可以自由考虑的最早时间。 
2. 初始化`answer`到 0，这将累积完全中断的次数。 
3.对于每个客户间隔`(t, l)`，计算该客户开始作为细分之前的空闲时间`[current, t)`。 这是下一次强制占领之前唯一可用的窗口。 
4. 计算该空闲段的长度为`t - current`。 添加`floor((t - current) / a)`答案，因为每个中断都必须完全适合这个区间。 
5. 更新`current`到`t + l`，因为在为该顾客提供服务后，收银员只有在服务结束后才能再次空闲。 
6. 处理完所有客户后，处理最后的空闲段`current`到`L`以同样的方式，添加`(L - current) // a`到答案。 

关键思想是我们永远不需要显式地模拟中断位置。 我们只计算有多少个完整的尺寸段`a`适合每个最大自由区间。 

### 为什么它有效

 该算法隐式地将时间线划分为交替的阻塞段和空闲段。 由于服务间隔保证不重叠，因此每个空闲段都是最大且独立的。 任何有效中断必须完全位于这些段之一内，并且任何中断都不能跨越服务间隔。 因此，最佳解决方案就是所有空闲段的总和，其中包含多少个长度a块。 段上的贪婪累积准确地保留了该分区，因此不会重复计算或错过任何中断。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, L, a = map(int, input().split())
    
    current = 0
    ans = 0
    
    for _ in range(n):
        t, l = map(int, input().split())
        
        if t > current:
            ans += (t - current) // a
        
        current = max(current, t + l)
    
    if current < L:
        ans += (L - current) // a
    
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案保留单个指针`current`跟踪最后占用的间隔的末尾。 每次遇到新客户时，我们都会立即评估当前空闲时间与下一次服务开始之间的差距。 这可以避免存储任何间隔或构建时间线。 

使用`max(current, t + l)`是一种安全措施，即使严格不重叠的推理放松，也能确保正确性，但在这个问题中，它主要保持干净的前进过程。 最后一段单独处理，以避免错过当天的尾声。

 ## 边缘情况

 没有顾客的时候，`current`保持为 0，直到处理完最后一个段。 该算法直接计算`(L - 0) // a`，它可以正确生成全天的完全休息次数。 

当保养间隔相互接触时，例如`(0,2)`和`(2,4)`，更新`current = max(current, t + l)`确保它们之间不会人为地产生间隙。 第二个客户之前计算的空闲航段为零，因此不会错误地添加中断。 

当空闲段小于`a`，整数除法自然会贡献零，从而防止部分中断被计算在内。
