 ## 解决方案
 ## 问题理解

 我们得到一个整数$N$，这是一些隐藏的骰子滚动序列的乘积。 每个骰子都是一个标准骰子，因此隐藏序列中的每个因子都是从 1 到 6 的整数。任务是重建此类骰子值的任何有效多重集，其乘积等于$N$，并在所有有效重建中，选择滚动值总和最大的重建。 

如果 1 到 6 范围内的数字的多组不能产生乘积$N$，答案是不可能的，我们输出$-1$。 还有一个输出上限：如果最佳可能总和变得非常大，我们必须输出$10^9 + 7$反而。 

关键的困难在于，我们不被要求重建一个有效的因式分解，而是在乘法约束下优化所有有效的因式分解。 这立即使其成为因式分解和组合优化问题，而不是直接分解。 

约束条件$N \le 10^5$意味着我们可以自由分解$N$在$O(\sqrt{N})$或者$O(\log N)$型行为并且仍然安全。 任何尝试枚举所有多重集或搜索骰子值组合的方法都会爆炸，因为即使是中等乘积也可能有许多分解。 

骰子上的值 1 是一个微妙的问题。 由于乘以 1 不会改变乘积，但会增加总和，因此允许任意 1 将使答案无界。 这意味着 1 在任何最优构造中都必须被视为无关紧要，并且在推理产品结构时我们实际上忽略了它。 

通常破坏朴素解决方案的边缘情况是包含外部素数因子的输入$\{2,3,5\}$，特别是7，例如$N=7$，其中不存在分解。 另一个棘手的情况是$N=1$，我们必须决定是否允许空产品。 正确的解释是不需要掷骰子，总和为 0，因为引入任意数量的 1 都不是有效的有界最优构造。 

## 方法

 强力解释会尝试生成从 1 到 6 的所有多重值集，其乘积为$N$，然后计算它们的总和并取最大值。 即使将我们限制在数字 2 到 6，这仍然意味着探索一个分支过程，其中每一步都会划分剩余的乘积，并且状态数量随着因子的数量呈指数增长。 例如，即使是中等数量，例如$N = 2^{20}$已经对应了大量的 2、4 和组合分区，使得在时间限制内无法进行枚举。 

关键的观察是该结构完全是乘法的，并且每个有效的解决方案都对应于因式分解$N$转化为数字从$\{2,3,4,5,6\}$。 这些数字本身由素数 2、3 和 5 组成，其中 4 贡献$2^2$和 6 贡献者$2 \cdot 3$。 任意因式分解$N$在素数 2、3、5 之外，立即无法得出答案。 

一旦我们用素数指数来表达一切，问题就变成了一个重新分配任务：我们决定如何在有利的时候将 2 和 3 捆绑成 6，否则将它们保留为 2 和 3。 优化取决于比较不同分组如何影响总和，同时保留相同的主要产品。 唯一重要的贸易是在$6$和那对$3 + 2$，因为两者代表相同的素数乘积但总和不同。 

这将问题从组合搜索简化为对素数计数的贪婪构造。

 ## 算法演练

 ### 算法演练

 1. 因式分解$N$分解为素数，计算 2、3 和 5 出现的次数。 如果出现 2、3、5 以外的素数，则无法构造，所以答案为$-1$。 此步骤确保我们仅在可表示的骰子因子范围内工作。 
2.让$c_2, c_3, c_5$分别是素数 2、3、5 的个数。 这些完整地描述了乘法结构$N$。 
3. 构造尽可能多的 6。 每个 6 消耗一个 2 和一个 3。 6 的数量是$k = \min(c_2, c_3)$。 这是最优的，因为 6 与 (2,3) 具有相同的素数成本，但产生的总和大于 2+3。 
4. 形成6s后减少剩余计数：$c_2 \leftarrow c_2 - k$,$c_3 \leftarrow c_3 - k$。 
5. 将剩余素数直接转换为骰子面。 剩下的每个 3 都会变成 3，剩下的每个 2 都会变成 2，每个 5 都会贡献一个 5。 
6. 将总和计算为$6k + 3c_3 + 2c_2 + 5c_5$。 
7. 如果$N = 1$，直接返回0，因为不需要卷。 
8. 如果计算的总和超过$10^9 + 7$， 输出$10^9 + 7$。 

### 为什么它有效

 任何有效的构造都完全对应于分布素因数$N$分成形成骰子值的组。 在不改变乘积的情况下改变总和的唯一自由是我们如何将 2 和 3 分组为单独的面或组合的 6。由于 6 在总和中严格支配 2+3，同时保持相同的因式分解，因此每个最佳解决方案必须最大化此类配对的数量。 所有剩余因素都没有可在不改变可行性的情况下提高总和的替代表示，因此 (2,3) 的贪婪配对充分表征了最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input().strip())
    
    if n == 1:
        print(0)
        return
    
    c2 = c3 = c5 = 0
    x = n
    
    while x % 2 == 0:
        c2 += 1
        x //= 2
    while x % 3 == 0:
        c3 += 1
        x //= 3
    while x % 5 == 0:
        c5 += 1
        x //= 5
    
    if x != 1:
        print(-1)
        return
    
    k = min(c2, c3)
    c2 -= k
    c3 -= k
    
    total = 6 * k + 3 * c3 + 2 * c2 + 5 * c5
    
    if total > MOD:
        print(MOD)
    else:
        print(total)

if __name__ == "__main__":
    solve()
```代码首先处理特殊情况$N=1$，直接返回 0。然后它执行完整的素因数分解，但只跟踪 2、3 和 5 的计数。任何剩余的因子立即意味着无效的构造。 

之后，它执行唯一有意义的优化步骤，将 2 和 3 贪婪地配对为 6。 这是使用最少的计数来完成的，确保不会浪费配对机会。 

最后，它使用导出公式计算总和。 上限检查应用在最后，因为总和随着计数线性增长并且不能溢出Python整数，但问题需要截断。

 ## 边缘情况

 案例$N = 7$触发无效的素数检查。 在分解过程中，去除 2、3 和 5 个贡献后，剩余值变为 7。 由于它不是 1，因此算法正确地拒绝它并输出$-1$。 尝试将 7 贪婪地分割成允许的面的简单方法将会失败，因为不存在有效的分解。 

案例$N = 1$在因式分解逻辑之前处理。 如果没有这个保护，算法会将其视为具有零素数并隐式生成和 0，这仍然是正确的，但显式处理它可以避免空乘积语义的歧义并确保与预期解释的一致性。
