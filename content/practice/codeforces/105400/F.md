 ## 解决方案
 ## 问题理解

 我们得到一排书，每本书都有一个价值，以及一个随着时间的推移从末端移除书籍的过程。 每一秒，有两件事按顺序发生：首先，Keys可能会从当前段中的任何地方恰好拿走一本剩余的书，然后火会烧掉当前最左边和最右边的书。 之后，剩余的段向两侧向内收缩一个位置，重复该过程，直到没有剩余为止。 

关键的限制是，无论 Keys 之前是否已经拿走了这些书，燃烧总是会删除当前的末端。 凯斯试图在他设法挑选的书籍被毁坏或因缩小而变得无法访问之前最大化其价值总和。 

因此，问题是一个缩小间隔内的调度问题：在时间 t 时，仅保留一个中心子数组，并且在每个时间步 Key 都可以在删除末尾之前从该子数组中选取一个元素。 

输入大小 N 最大可达 100000，这会立即排除任何模拟所有选取位置选择或在子集上使用指数 DP 的解决方案。 任何正确的方法都必须至多为 O(N log N) 或 O(N)。 

一个微妙的点是移除的时间很重要。 位于位置 i 的书只有在收缩过程从任一侧到达它之前才可用。 这意味着每个位置都有一个仍然可以被拾取的最后可能时间，这完全取决于它到较近一端的距离。 这创建了一个隐藏的结构：每个项目都有一个截止日期，我们每秒最多可以选择一个项目。 

一个常见的错误是假设您应该在每一步中贪婪地从当前间隔中选择最大的剩余值。 这会失败，因为如果您延迟选择一个即将被销毁的大值，而选择另一个具有更多冗余的大值，则可能会错过它。 

例如，在 [100, 1, 1, 100] 这样的数组中，两个 100 都靠近末尾。 如果你总是选择每次收缩后剩余的全局最大值，你可能会在错误的一侧浪费一步并完全失去 100 秒中的一个，即使如果安排正确的话，两者都是安全的。 

另一种故障模式是将其视为所有子阵列上的间隔 DP。 这将需要 O(N^2) 个状态，这太大了。 

核心困难是认识到该过程为每个索引定义了一个时间窗口，而决策只是将哪些元素安排到哪些时隙中。 

## 方法

 强力解释将尝试模拟每秒选择哪本书的所有可能选择，同时跟踪不断缩小的边界。 在时间 t 时，剩余的间隔是固定的，我们选择其中的一个元素。 选择的数量与当前间隔大小成正比，因此分支因子从 N 开始并线性减小。 即使进行剪枝，可能的序列数量也会以 N 为指数，因为在每一步中，我们都会从缩小的集合中选择一个元素，而选择之间没有明确的主导关系。 

这会失败，因为早期步骤的决策会影响高价值元素是否能够存活下来以供以后选择，并且仅对值不存在简单的贪婪排序。 

关键的观察是翻转视角。 我们不考虑间隔随着时间的推移而缩小，而是为每个位置 i 分配一个“生存时间”，即它被烧毁之前的步数。 由于两端被对称地移除，位置 i 准确地存在，直到 min(i, N - i + 1) 层燃烧到达它。 这意味着每本书的截止日期都等于该值。 每一秒，我们最多可以拿一本书，因此我们可以有效地安排单位时间的工作，并确定截止日期和利润。

这将问题简化为经典的贪婪调度问题：通过每个时间段最多选择一个作业来最大化总利润，其中每个作业都有一个截止日期。 最佳策略是按价值降序对书籍进行排序，并将每本书分配到不超过其截止日期的最新可用位置。 

我们维护一个结构来跟踪哪些时间段是空闲的，并且对于按价值降序排列的每本书，我们贪婪地将其放入它仍然可以占用的最新可用时间段中。 这确保了只有较早的书籍已经被占用时，高价值的书籍才会消耗较晚的书籍。

 ## 算法演练

 ### 关键思想

 我们将收缩过程重新解释为给每个位置一个截止日期，该截止日期等于它在被烧毁之前存活的轮数。 

### 步骤

 1. 对于每个索引 i，将其生存极限计算为火灾到达它之前的层数。 这是 min(i, N - i + 1)。 

这代表着这本书还可以拍摄的最后一秒。 
2. 将每本书视为一项具有利润 a[i] 和截止日期 d[i] 的工作。 

每一秒对应一个可用槽。 
3. 按价值降序对所有书籍进行排序。 

我们优先考虑高价值的书籍，因为一旦一个位置被占用，它就永远无法重复使用。 
4. 维护表示从 1 到 N 的空闲时隙的结构，通常是具有“下一个可用时隙”跳转的 DSU 或布尔数组。 
5. 按排序顺序遍历书籍。 对于每本书，尝试将其放置在最新的可用位置≤其截止日期。 
6. 如果存在这样的槽，则将书分配到那里并将其值添加到答案中。 否则跳过它。 

### 为什么它有效

 在任何时候，如果两者都可行，将一本书分配到较晚的位置总是比将其分配到较早的位置更好，因为它为截止日期更紧的书籍保留了较早的位置。 通过始终首先处理较高的值，我们确保如果可以安排一本书，那么它会以尽可能最少的限制方式进行安排。 贪婪交换论证表明，任何最优调度都可以转换为这种贪婪调度，而不会减少总价值，因为在可行的槽中将较低价值的分配与较高价值的分配进行交换永远不会减少结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def find(parent, x):
    if parent[x] != x:
        parent[x] = find(parent, parent[x])
    return parent[x]

def union(parent, x, y):
    parent[x] = y

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    jobs = []
    for i, v in enumerate(a):
        d = min(i + 1, n - i)
        jobs.append((v, d))

    jobs.sort(reverse=True)

    parent = list(range(n + 1))

    def get_slot(x):
        x = find(parent, x)
        return x

    ans = 0

    for v, d in jobs:
        if d <= 0:
            continue
        slot = get_slot(d)
        if slot == 0:
            continue
        ans += v
        parent[slot] = slot - 1

    print(ans)

if __name__ == "__main__":
    solve()
```实现首先使用每个索引到较近端的距离将其转换为截止日期。 这可以准确捕捉火势到达该位置的时间。 

排序步骤确保我们始终尝试将价值较高的书籍放在第一位。 这就是核心的贪婪优先级。 

联合查找结构用作“下一个可用槽”跟踪器。 每次我们占用一个槽时，我们都会向后合并它，以便将来的查询有效地跳转到下一个空闲位置。 这避免了对每个位置进行线性扫描，使解决方案保持接近 O(N)。 

一个微妙的细节是使用 min(i + 1, n - i) 而不是 min(i, n - i + 1)，它来自基于 0 的索引和基于 1 的索引之间的切换。 逻辑保持不变：到最近边界的距离。

 ## 边缘情况

 对于单元素数组，截止时间为 1，如果我们选择该元素，则始终会取该元素。 算法直接将其分配给槽 1，因此答案正是该值。 

对于交替的高值和低值，例如 [100, 1, 100, 1, 100, 1]，该算法确保在低值消耗空间之前将所有高值首先放入可用槽中。 每个高值元素都有足够的期限来分配给不同的槽，因此最终的总和正确地包括所有大元素。 

对于所有值都相等的数组，该算法有效地选择大小等于总可用时隙的任何有效子集，这与最佳行为匹配，因为所有选择都是等效的。
