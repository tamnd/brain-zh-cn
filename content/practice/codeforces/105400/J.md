 ## 解决方案
 ## 问题理解

 我们有一个 3 行 N 列的网格。 每个单元格包含 0 或 1，代表对两名候选人的投票。 我们可以将每个单元格视为带有二元标签的单位城市。 

我们必须将所有 3N 个单元格精确地划分为 N 个组，其中每个组恰好包含 3 个单元格，并在网格图中形成一个连通分量。 连接性意味着单元格通过边缘而不是对角线相邻。 每个这样的组称为一个区。 如果一个区的三个单元格中至少有两个为 0，则椰子获胜，否则橙色获胜。 

任务是将网格划分重新排列成连接的三元组，以最大化零占多数的区域数量。 

关键的自由在于我们不受原始垂直切片成列的限制。 任何连接的三联骨形状的组件都是允许的，只要每个单元都只使用一次。 

约束很小，N 最大为 100，T 最大为 10，这立即表明每个测试用例甚至 O(N^3) 也是可行的。 然而，该结构不是任意组合，它是一个只有 3 行的网格，这通常表明按列动态编程或跨列的状态压缩是可能的。 

当零稀疏但以允许将它们分组为许多不同的连接三元组的方式排列时，就会出现微妙的边缘情况。 将每个列独立分组的天真的贪婪方法可能会失败，因为最佳区域通常会跨越列边界。 另一种失败情况是局部最优分组过早使用了太多零，留下了无法在以后配对成获胜三元组的孤立零。 

例如，考虑具有模式 (0, 0, 1) 的单个列。 天真的分组会立即组成一个区，并将其视为胜利。 但在多栏设置中，将该栏与相邻栏配对可能会允许在各区之间更有效地重新分配零，从而增加总胜利。 

关键的困难在于每个三元组必须在网格中连接，这强烈限制了可能的形状：3 行网格中的每个有效的 3 单元连接组件要么是 3 的垂直直线，要么是覆盖 2 个相邻列的弯曲形状，要么是水平链状形状。 这种结构使得问题变得容易处理。 

## 方法

 暴力方法会尝试枚举所有可能的方法，将 3×N 网格划分为连接的三元组。 每个状态都代表对连接组件的选择，然后递归地删除它并继续。 3×N 网格中连接的三联骨牌数量已经很大，并且分区数量呈超指数增长。 即使 N 约为 10，这也很快变得不可行，因为在每一步我们都有多个形状放置和重叠约束。 

瓶颈在于，在一列中做出的决策会影响剩余单元格的分组方式，但强力方法不会重用重叠的子问题。 相同的部分列配置重复出现，但会重新计算。 

关键的见解是逐列处理网格并编码部分形成的区域如何跨列边界延伸。 因为只有 3 行，所以可能的“活动边界态”数量很少。 每个状态描述当前列和前一列中的哪些单元格已被使用或正在等待下一列中完成。 这将问题转化为具有小状态空间的列上的动态规划。 

在每一步中，我们决定如何形成涉及当前列的连接三元组，可能与前一列中的待处理单元格配对。 由于每个区的大小恰好为 3，因此我们可以跟踪部分形成的区的单元数量已被占用，从而确保保留连接约束。

我们还维护每个已完成的地区迄今为止包含多少个零，并决定关闭时它是否成为获胜地区。 由于每个选区都很小，因此状态不需要存储完整的历史记录，仅对当前打开的组件进行部分计数。 

这将指数分区问题简化为列转换上的可管理 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | N 的指数 | 指数| 太慢了 |
 | DP 超过列状态 | O(N × S)，其中 S 是小常数 | O(S)| 已接受 |

 ## 算法演练

 我们从左到右处理列并维护一个 DP 状态，描述当前边界中的单元如何连接到未完成的组件。 

每个状态将 3 行中的哪一行当前有待连接到下一列进行编码，以及在部分形成的区域中已经使用了多少个单元。 由于每个区的大小为 3，因此唯一可能的部分状态是那些已将 0、1 或 2 个像元分配给开放组件的状态。 

我们还跟踪每个部分形成的组件中包含多少个零，以便当它完成时，我们可以决定它是否有助于答案。 

转换包括通过放置有效的连接三元组尝试所有方法将当前部分组件扩展到下一列。 由于网格的高度为 3，因此可以通过枚举其 3 个单元的子集并将它们与前一列的剩余状态配对来处理每一列。 

## 算法演练

 1. 我们在列上定义一个 DP，其中每个状态表示当前列中的单元如何连接到来自前一列的未完成组件。 这是必要的，因为连接可以跨列，因此我们无法在本地完成决策。 
2. 对于每个状态，我们尝试使用有效的连接配置将当前列的 3 个单元格分配到大小为 3 的组中。 每个配置要么完全在列中完成一个区域，要么连接到前一列中的待处理单元。 这确保了所有组件在网格中保持连接。 
3. 每当一个分量达到大小 3 时，我们就计算它包含多少个零。 如果至少存在 2 个零，我们将增加分数。 该评估仅在该地区完全关闭时进行，这保证了正确性。 
4. 我们根据处理当前列后有多少部分组件保​​持打开状态来更新下一列的 DP 状态。 这些开放组件携带所需的连接信息。 
5. 处理完所有列后，我们只考虑没有部分组件保​​持开放的状态，因为所有选区都必须完全形成。 这些州中得分最高的就是答案。 

### 为什么它有效

 DP 不变量是，在处理第 i 列之后，每个有效状态都表示将列 [1..i] 中的所有单元格正确划分为已完成的区域以及一组一致的未完成组件，这些组件仍可以扩展为有效的连接三元组。 不允许无效的部分分组，因为每个转换仅构建与网格邻接一致的大小最多为 3 的连接集。 由于每个最终状态都对应于一个完整分区，并且每个可能的分区都有相应的转换序列，因此保留了最优解。

 ## 边缘情况

 一种重要的边缘情况是零在不同列中被隔离，但只能通过跨列形状组合。 单纯的按列分组在这里会失败，但 DP 正确地跨列携带部分组件，从而允许稍后将这些零分组到单个区域中。 

另一种边缘情况是，在一列中形成垂直三元组似乎在本地是最佳的，但会妨碍下一列中更好的配对。 DP 通过保留所有部分配置来避免贪婪提交，确保延迟的分组决策稍后仍能恢复最佳结构。
