 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 N 排列成一行的数字的排列。 目标是使用交换将这种排列转换为 1 到 N 的排序顺序，但对允许的交换有严格的限制。 

仅允许在只有一个人的两个职位之间进行交换。 换句话说，您可以交换位置 i 和 i+2，但不能交换相邻位置，也不能交换相距较远的位置。 每次移动都会保留数组的其余部分，并且只交换这两个元素。 

任务是确定是否可以在此约束下达到排序排列，如果可以，则计算所需的此类交换的最小数量。 

尽管 N 最多为 1000，这表明 O(N^2) 或 O(N^3) 解决方案可能是可以接受的，但约束足够严格，任何尝试任意交换序列或排列的 BFS 的模拟都是不可行的。 排列的状态空间是阶乘的，因此即使进行修剪，探索配置也是不可能的。 

这个问题的一个微妙的方面是交换操作以移位的方式保留索引奇偶校验：交换 i 和 i+2 意味着元素始终在相同奇偶校验类别的位置内移动。 这造成了直接的可行性限制。 

一些边缘情况值得强调。 如果 N = 1，则数组已排序，答案为零。 如果应该位于偶数索引的元素从奇数索引开始，则它永远无法到达其正确位置，因为每次交换都会保留索引的奇偶性。 例如，如果 N = 3 并且数组为 [3, 1, 2]，则元素 1 位于索引 2（偶数）处，但在排序数组中，它必须位于索引 1（奇数）处。 这种不匹配使得排序变得不可能，并且天真的贪婪交换模拟可能会错误地继续交换而不认识到这种不变的违规。 

## 方法

 暴力尝试会尝试直接模拟排序过程。 人们可以重复定位一个不合适的元素，并尝试使用有效的交换将其移向目标位置，可能使用排列的 BFS 或贪婪的局部校正。 这在原则上是正确的，因为每次交换都在状态空间中保留合法的转换，并且最终可以找到任何可达的配置。 

问题是，即使是简单的排列 BFS 也会立即爆炸。 有N个！ 状态，并且每个状态最多有 O(N) 种可能的交换，因此即使对于 N = 10^3，搜索空间也远远超出了任何可行的限制。 即使是反复修复位置的贪婪模拟也可能会循环或无法找到最佳交换次数，因为局部改进不是独立的。 

关键的观察结果是，交换仅连接相差 2 的索引，因此数组分为两个独立的子序列：奇数位置上的元素和偶数位置上的元素从不交互。 在每个子序列内，我们实际上可以交换相邻元素，因为原始数组中的位置 i 和 i+2 对应于压缩奇偶校验序列中的连续位置。 

这将问题简化为两个独立的子序列排序问题，每个问题都使用相邻的交换。 对序列进行排序所需的最小相邻交换次数正是其反转计数。 因此，答案是奇数索引子序列和偶数索引子序列的反转计数之和，前提是这两个子序列恰好包含其奇偶校验类所需的正确值多重集。 

如果违反奇偶校验约束，则无法进行排序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力模拟/BFS | O(N!) | O(N!) | 太慢了 |
 | 奇偶校验分割+反转计数| O(N^2) | O(N^2) | O(N) | 已接受 |

 ## 算法演练

1. 根据位置奇偶性将数组分成两个序列，一个包含索引 1、3、5 处的元素，另一个包含索引 2、4、6 处的元素。这反映了交换永远不会在这些组之间移动元素的事实。 
2. 通过验证两个组是否包含与排序数组中奇偶校验位置完全正确的元素来检查可行性。 奇数索引组必须准确包含应按排序顺序出现在奇数位置的数字，对于偶数位置也类似。 如果此条件失败，则返回 -1。 
3. 独立计算每个奇偶校验组内所需的最小交换次数。 在组内，允许的操作实际上变成交换该压缩序列中的相邻元素。 
4. 对于每个组，计算反转计数。 对于每个元素，计算有多少个先前见过的元素大于它。 该计数表示在冒泡排序意义上将该元素移动到正确位置所需的交换次数。 
5. 将两组的反转计数相加并输出结果。 

应用反转计数的关键原因是，原始数组中的每个有效交换对应于交换奇偶校验压缩数组之一中的相邻元素，并且已知相邻交换精确地按照其反转数对序列进行排序。 

### 为什么它有效

 关键的不变量是元素永远不会改变其索引的奇偶性。 这将排列划分为两个独立的序列，它们在没有相互作用的情况下演化。 在每个序列中，允许的交换形成完整的邻接图，这意味着该子序列的任何排列都可以使用相邻交换来实现。 由于相邻交换生成所有排列，并且每次交换恰好纠正一个反转，因此交换的最小数量恰好是反转计数。 由于两个奇偶校验序列独立演化，因此它们的成本不会相互干扰地增加。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def inversion_count(arr):
    n = len(arr)
    inv = 0
    for i in range(n):
        for j in range(i):
            if arr[j] > arr[i]:
                inv += 1
    return inv

n = int(input())
a = list(map(int, input().split()))

odd_pos = []
even_pos = []

for i in range(n):
    if i % 2 == 0:
        odd_pos.append(a[i])
    else:
        even_pos.append(a[i])

target_odd = sorted(range(1, n + 1))[::2]
target_even = sorted(range(1, n + 1))[1::2]

if sorted(odd_pos) != target_odd or sorted(even_pos) != target_even:
    print(-1)
else:
    print(inversion_count(odd_pos) + inversion_count(even_pos))
```该代码首先根据索引奇偶校验将数组划分为两个序列。 这符合交换永远不会混合这两个组的结构约束。 然后，它从排序数组中为每个奇偶校验类构建预期的多组值，并将它们与实际内容进行比较。 如果它们不同，则任何有效交换序列都无法修复排列。 

inversion_count 函数使用简单的嵌套循环计算每个子序列中的反转次数。 尽管 O(N^2)，但只要 N ≤ 1000 就足够了。 

最后，反转计数的总和被打印为交换的最小总数。

 ## 边缘情况

 对于 N = 1，该算法分为单元素奇数子序列和空偶数子序列。 两个反转计数都为零，并且由于单个值已经就位，因此可行性微不足道。 

对于奇偶校验不匹配的情况，例如输入 [1, 3, 2]，奇数位置包含 [1, 2]，而 N = 3 的预期奇数集为 [1, 3]。 在任何反转计数之前，不匹配会立即触发拒绝。 这可以防止在无法访问的配置上浪费计算。 

对于完全反转的数组，每个奇偶校验子序列变成递减序列，并且反转计数正确捕获每个独立链内所需的最大交换次数。
