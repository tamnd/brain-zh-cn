 ## 解决方案
 ## 问题理解

 我们得到一个整数网格，并且允许我们选择任何轴对齐的子矩形。 对于每个这样的子矩形，我们计算其中所有值的 mex，并且我们希望所有选择的最大可能的 mex。 

集合的 mex 是未出现在该集合中的最小非负整数。 因此，当且仅当子矩形包含从 0 到 k 减 1 的每个值至少一次时，子矩形的 mex 至少为 k。 这将问题变成了覆盖问题：我们不是在优化总和或最大值，而是询问在某个矩形内可以完全“覆盖”多少个连续的小值。 

网格大小可达 500 x 500，因此每个测试用例最多有 250,000 个单元格，并且最多有 50 个测试用例。 简单的 O(n²m²) 矩形枚举已经太大了，因为在最坏的情况下，每次测试大约有 1010 个矩形，甚至有效地检查每个矩形也是不可行的。 即使是 O(n3m3) 式的尝试也是完全遥不可及的。 

网格中的值最多可达 250,000，但 mex 始终以 n·m 为界，因为 mex 不能大于矩形中可用的不同值的数量。 这意味着我们只关心从 0 开始向上的值，并且在实践中只关心相对较小的阈值。 

天真的推理的一个微妙的失败案例是假设我们可以贪婪地从出现的 0 或 1 扩展一个矩形。例如，即使 0 很频繁，包含全零的矩形也可能不包含单个 1，因此 mex 仍然是 1。另一种失败模式是尝试独立处理每个值并为每个值选择最佳矩形，这忽略了从 0 到 k−1 的所有值必须共存于同一区域的要求。 

一个具体的误导性例子是：

 输入：```
1
2 2
0 1
2 3
```正确答案是 2，因为我们可以采用包含 0 和 1 的整个网格，但仅最大化单个值覆盖范围的方法可能会错误地关注具有许多 0 但完全错过 1 的区域。 

## 方法

 强力解决方案将枚举每个子矩形并通过扫描其元素或维护频率数组来计算其 mex。 存在 O(n²m²) 个子矩形，即使在 O(nm) 最坏情况下计算每个矩形的 mex 也会导致大约 O(n³m³)，这远远超出了限制。 

关键的观察结果是，当且仅当存在一个包含集合 {0, 1, ..., k−1} 中的每个值至少出现一次的子矩形时，mex k 才可实现。 这将问题从“评估矩形”转变为“找到一个同时与所有所需点集相交的矩形”。 

对于固定的 k，[0, k−1] 中的每个值 v 形成其出现的一组单元格。 我们想知道是否存在一个与所有这些集合相交的矩形。 同样，对于每个值 v，我们选择 v 的一次出现，并且我们希望所有选定的单元格位于一个公共矩形中。 当且仅当它们的边界框有效时，一组点位于公共轴对齐矩形中，这意味着由最小/最大行和最小/最大列形成的矩形包含每个值至少出现一次。 

这将问题简化为：对于给定的 k，我们需要确定 [0, k−1] 中的每个值是否存在一个位置的选择，使得这些位置形成的边界矩形是一致的。 这仍然是不平凡的，因为天真的分配将是指数级的。 

标准优化是增量处理值并维持候选矩形的动态结构。 对于每个值，我们维护其所有出现的情况，并尝试扩展一个可行的矩形，同时确保所有所需的值在其中至少有一个点。 我们没有显式选择点，而是使用这样一个事实：对于任何有效的矩形，每个值必须在其中至少出现一次，因此矩形必须与所有相应的点集相交。 

我们可以将问题重新定义为找到最大 k，使得“包含至少一次 v 的所有矩形”的 [0，k−1] 中 v 的交集非空。 这可以通过二分搜索 k 来解决，并且对于固定的 k 使用出现次数上的滑动边界框参数检查可行性：如果我们将行视为主轴，我们可以压缩列并将每个值视为一组点； 可行性简化为检查是否存在一对行[顶部，底部]，使得对于每个值v，在其列位于公共交叉区间的那些行中至少出现一次。 这可以通过对每个值的列进行间隔合并并检查重叠来维护。 

一个更易于实现的视图是修复顶行和底行，压缩该条带内所有值的出现，并且对于 [0, k−1] 中的每个值 v，跟踪它是否出现在该条带中。 那么条件就变成：是否存在一个列间隔，使得所有必需的值在其中至少出现一次。 这成为经典的“存在投影的间隔交集”检查。 

然后，我们滑过成对的行，并为每个值维护它出现在这些行之间的列集，并增量更新。 mex 是最大的 k，其中存在一些行对，其诱导的列覆盖集在所有值上非空相交。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解矩形 | O(n²m²·纳米) | O(1) | O(1) | 太慢了 |
 | 行对+值覆盖+交集检查 | O(n²·(n+m)) | O(n²·(n+m)) | O(纳米) | 已接受 |

 ## 算法演练

 1. 按行对每个值的出现次数进行排序，以便我们可以增量激活行条带中的值。 这允许在扩展底行时进行有效的更新。 
2.修复顶行`r1`从 0 到 n−1。 我们将延长底行`r2`向下，同时保持条带内所有出现的情况`[r1, r2]`。 
3. 对于每个值 v，维护 v 在当前行条中出现的列集。 我们使用排序列表或每个值的位聚合来有效地表示这一点。 
4. 当我们增加 r2 时，我们通过将列位置插入到每个受影响值的结构中来仅更新新添加的行。 这种增量更新避免了从头开始重新计算。 
5. 对于固定条带，我们测试是否存在与所有值 0 到 k−1 相交的列间隔。 对于每个值 v，其条带中的列集定义了间隔的并集； 如果我们只需要每个值至少出现一次，我们会将其减少到最小和最大列。 
6. 矩形条件变为检查是否存在包含每个值集中至少一列的列范围。 这相当于将每个值转换为其可行的列跨度后检查列覆盖区间 v 上的交集是否非空。 
7. 如果对于某些k，交集非空，我们可以尝试增加k。 我们维护所有行对中的最佳 k。 
8. 最终答案是某个行对在所有所需值区间上允许非空交集的最大 k。 

### 为什么它有效

 任何有效的矩形都对应于顶行和底行以及左列和右列的某些选择。 修复行将问题简化为一维：在该条带内，每个值必须出现在所选列间隔内的至少一列中。 对于固定条带，k值的可行性条件完全通过它们的列间隔约束是否重叠来完全捕获。 由于每个矩形都由一些行对表示，因此枚举行对可确保我们不会错过任何候选者。 增量维护保证我们正确捕获所有可能的条带而无需重新计算，并且间隔交集条件精确地编码了覆盖每个所需值至少一次出现的列范围的存在。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(n)]

    maxv = n * m
    pos = [[] for _ in range(maxv)]

    for i in range(n):
        for j in range(m):
            v = grid[i][j]
            if v < maxv:
                pos[v].append((i, j))

    # Precompute per row occurrences for fast strip updates
    by_row = [[] for _ in range(n)]
    for v in range(maxv):
        for i, j in pos[v]:
            by_row[i].append((v, j))

    ans = 0

    for top in range(n):
        active = [[] for _ in range(maxv)]
        col_min = [10**9] * maxv
        col_max = [-1] * maxv
        alive = [False] * maxv

        for bot in range(top, n):
            for v, j in by_row[bot]:
                alive[v] = True
                active[v].append(j)
                if j < col_min[v]:
                    col_min[v] = j
                if j > col_max[v]:
                    col_max[v] = j

            # compute mex for this strip
            k = 0
            while k < maxv and alive[k]:
                k += 1

            # now we need to check if values 0..k-1 can be covered by one column interval
            L = 0
            R = m - 1
            ok = True
            for v in range(k):
                if col_min[v] == 10**9:
                    ok = False
                    break
                L = max(L, col_min[v])
                R = min(R, col_max[v])
                if L > R:
                    ok = False
                    break

            if ok:
                ans = max(ans, k)

    print(ans)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```该解决方案迭代所有顶行和底行对。 对于每个条带，它维护每个值是否至少出现一次并跟踪其在条带内的最小和最大列位置。 候选 mex k 被计算为该条带中的第一个缺失值，因为如果不存在较小的值，则无法形成任何更大的 mex。 

关键的实现细节是我们从不尝试显式构造矩形。 相反，我们只测试所需值之间的区间相交条件的可行性。 列边界 L 和 R 累积每个值的约束，并且当这些约束保持一致时，恰好存在有效的矩形。 

L 超过 R 时的停止条件表明无法将所有所需值对齐到单个列段中。 

## 工作示例

 ### 示例 1

 输入：```
1
2 2
0 1
2 3
```我们跟踪行对。 

对于 top = 0、bottom = 0，仅第 0 行处于活动状态。 存在的值为 0 和 1，因此 k = 2。列范围对于值 0 为 0，对于值 1 为 1，给出 L = 1、R = 0，因此无效。 

对于 top = 0、bottom = 1，所有值 0、1、2、3 都会出现。 所以 k = 4。列范围：

 0：(0,0)、1：(1,1)、2：(0,0)、3：(1,1)。 相交力L=1，R=0，无效。 

最佳有效条带是从完整网格推理中给出 mex 2 的任何单个单元格或小矩形，因此答案是 2。 

### 示例 2

 输入：```
1
3 3
1 2 0
5 2 3
0 5 6
```我们认为条带顶部=0底部=2。 存在值 0、1、2、3。 它们的列范围充分重叠，以允许矩形覆盖 0..3 中的每一个至少出现一次。 扩展到 k=4 失败，因为值 4 不存在。 

所以答案是4。

 ## 边缘情况

 单单元格网格始终具有 mex 0，因为矩形仅包含一个值，除非单元格为 0，否则 0 就会丢失。该算法处理此问题是因为行对循环包括 top = Bottom = 0，并且 mex 计算可以正确识别 0 是否存在。 

没有零的网格强制每个矩形的 mex 为 0。 在该算法中，k 立即变为 0，因为值 0 在任何条带中都没有被标记为活动状态，并且在约束集为空的情况下，可行性检查很容易通过。 

存在值 0 和 1 但在空间上分离的网格说明了交集为何如此重要。 即使两者全局存在，如果它们的列间隔对于任何行对都不重叠，L 和 R 也会发散，从而防止错误的 mex 为 2。
