 ## 解决方案
 ## 问题理解

 我们得到一个仅由四个字符组成的长字符串：`b`,`d`,`p`， 和`q`。 该字符串打印在刚性玻璃物体上。 在坠落过程中，对象会通过一系列操作重复进行变换，并且要求我们在应用所有变换后确定字符串的外观。 

每个变换都是对象的刚性运动，由三个操作之一描述`h`,`v`， 或者`r`。 这些操作不会以任意方式“编辑”字符串。 相反，它们对应于物体的物理翻转或 180 度旋转，并且字母不断变化，就好像它们是在表面上绘制的刚性符号一样。 

关键点是每个操作都必须应用于整个配置，并且由于对象是刚性的，因此重复的操作会组成一个整体变换。 字符串长度可达 500,000，因此通过重建整个字符串来简单地应用每个操作会太慢。 

该约束立即排除了在线性时间内每个操作处理一次字符串的任何解决方案，因为最坏的情况是$5 \cdot 10^5 \times 5 \cdot 10^5$，这远远超出了可行的限度。 我们需要一种表示形式，其中每个操作都在恒定时间内处理，并且最终结果构造一次。 

一个微妙的困难是，相同的字母可以在输入中出现多次，并且在转换下，每次出现的行为必须一致。 任何试图在没有全局状态的情况下在本地重新解释每个位置的字母的方法很快就会变得容易出错。 因此，转换必须表示为统一应用于所有角色的全局状态更改。 

## 方法

 一种简单的方法是直接在字符串上模拟每个转换。 每个操作都会扫描整个字符串并根据该转换下每个字符的变化来替换每个字符。 这在概念上是正确的，因为变换是全局且统一的。 

但每次操作的成本$O(n)$，并且最多可以有$5 \cdot 10^5$运营。 这会导致最坏的情况$O(n \cdot m)$，这大约是$2.5 \cdot 10^{11}$人物更新，远远超出任何时间限制。 

关键的观察结果是，每个操作都是由对象的刚性运动引起的四个符号的固定排列。 由于刚性运动形成一个小的封闭群，因此重复应用它们只能组成排列。 我们可以维护迄今为止由所有操作引起的字母表的单个当前排列，而不是重复更新字符串。 

处理完所有操作后，我们将最终的排列应用于整个字符串$O(n)$时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每次操作重建字符串 |$O(nm)$|$O(n)$| 太慢了|
 | 组合转换然后应用一次 |$O(n + m)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将每个操作建模为集合上的排列`{b, d, p, q}`。 这个想法是，每个变换仅重命名符号，而变换的组合只是这些重命名的函数组合。 

我们维护当前的映射`f`，最初是恒等映射，意味着每个字符映射到其自身。 

我们还预定义了每个操作的映射`h`,`v`， 和`r`。 其中每一个都是四个字母的排列。 应用操作意味着更新`f`通过将其与相应的排列组合起来。 

## 步骤

 1. 初始化映射`f`这样`f(x) = x`对于所有角色`x in {b, d, p, q}`。 这表示尚未应用任何转换。 
2. 定义三种固定排列`H`,`V`， 和`R`对应于变换`h`,`v`， 和`r`。 这些取决于每个刚性运动如何重新标记四个符号。 
3. 迭代转换字符串`t`从左到右。 对于每个字符：

 - 如果是的话`h`， 更新`f(x) = H(f(x))`。 
- 如果是的话`v`， 更新`f(x) = V(f(x))`。 
- 如果是的话`r`， 更新`f(x) = R(f(x))`。 

此顺序可确保转换按正确的顺序组成，因为每个新操作都会作用于已转换的配置。 
4. 处理完所有操作后，通过替换每个字符来构造最终的字符串`c`在原始字符串中`s`和`f(c)`。 

### 为什么它有效

 每个操作都是有限字母表上的双射`{b, d, p, q}`。 因此，每个变换序列对应于大小最多为 24 的有限组中的单个排列。操作的组合是结合的，因此分组的顺序并不重要，重要的是序列。 

通过维持累积排列`f`，我们保留了处理操作前缀后的不变量，`f`准确地代表了这些操作的净效果。 正在申请`f`最后一次产生的结果与将每个操作顺序应用于整个字符串相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def compose(a, b):
    # return a ∘ b (apply b first, then a)
    return {c: a[b[c]] for c in "bdpq"}

def solve():
    s = input().strip()
    t = input().strip()

    # These mappings are consistent permutations of the alphabet
    H = {'b': 'p', 'd': 'q', 'p': 'b', 'q': 'd'}
    V = {'b': 'd', 'd': 'b', 'p': 'q', 'q': 'p'}
    R = {'b': 'q', 'q': 'b', 'd': 'p', 'p': 'd'}

    f = {c: c for c in "bdpq"}

    for op in t:
        if op == 'h':
            f = compose(H, f)
        elif op == 'v':
            f = compose(V, f)
        else:
            f = compose(R, f)

    res = ''.join(f[c] for c in s)
    print(res)

if __name__ == "__main__":
    solve()
```该实现依赖于将每个转换表示为字典排列。 帮手`compose`确保我们正确地按顺序累积变换。 

最后循环结束`s`应用一次累积的映射，这是我们唯一一次接触完整的字符串。

 ## 边缘情况

 一个微妙的情况是同一个字母多次出现，如`bbq`。 正确性依赖于映射是均匀的这一事实。 例如，在任何变换下`f`，两次出现`b`必须映射到相同的符号，因为`f`是字母表上的函数而不是位置上的函数。 

另一个重要的例子是重复变换，例如`vvvvvv`。 由于每次运算都是一种排列，重复应用最终会循环。 该算法自然地处理这个问题，因为排列的组合仍然是排列，并且没有引入每个字符的状态。 

最后，长混合序列，例如`hvrhv`证明顺序很重要。 该算法通过按顺序组合映射来保留顺序，确保早期的转换不会丢失或被错误地覆盖。
