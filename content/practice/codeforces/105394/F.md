 ## 解决方案
 ## 问题理解

 我们给出了代表蛋糕的简单多边形的边界。 多边形由其顶点按逆时针顺序描述，并且具有很强的结构特性：在 180 度旋转下保持不变。 换句话说，存在一个点$O$使整个形状旋转$O$半圈将形状映射到自身上。 

任务不是分析多边形本身，而是找到一条无限直线，将蛋糕切成面积完全相同的两块，并且在切口后切割恰好产生两个相连的块。 如果不存在这样的线，我们必须报告不可能。 

坐标可以大到$10^6$，并且顶点数量达到$10^5$，这立即表明对输入的任何超出线性或近线性的工作都是不可接受的。 任何尝试尝试许多候选线或在每个方向执行几何模拟的解决方案都会太慢。 

问题最微妙的部分是对称性、区域分割和所得碎片的连通性之间的关系。 天真的几何直觉可能表明任何穿过对称中心的线都是有效的，但这忽略了切割是否产生单个干净的分裂或多个碎片组件。 

一些边缘情况值得强调。 

如果多边形是中心对称的，但所选线与边界相交超过两次，则切割会产生两个以上的部分。 简单的方法可能仍然认为这是有效的，因为两个半平面具有相等的面积，但问题明确需要恰好两个结果部分。 

另一种失败模式是尝试错误地计算对称中心。 如果我们假设质心是对称中心，我们将在倾斜形状上得到错误的答案，因为质心通常与 180 度旋转对称的中心不同。 

最后，数字或表示问题对输出很重要。 该问题允许分数坐标，因此即使顶点是整数，中心也可能是非整数。 

## 方法

 强力解释是考虑由点对或边确定的所有可能的线，并测试每条线是否将多边形分割成相等的面积，并在切割后恰好产生两个连接的部分。 对于每条候选线，我们需要计算与所有边缘的交点并评估结果区域。 甚至限制候选人$O(n^2)$线已经使这种方法不可行，并且每个测试都需要$O(n)$或者$O(n \log n)$几何导致的整体复杂性远远超出了$n = 10^5$允许。 

关键的观察来自对称条件。 绕点旋转 180 度时的多边形不变量$O$是中心对称的。 对于简单的多边形，这种对称性意味着一个更强的结构事实：多边形是凸的。 一旦知道凸性，几何形状就会变得更加简单。 穿过凸多边形内部的任何直线与其边界恰好相交两次，这保证了切割恰好产生两个连接的部分。 

这将整个任务简化为一个几何事实：找到对称中心$O$，然后输出任何经过它的线。 

对称中心可以直接从顶点排序中恢复。 由于顶点按逆时针顺序给出，并且多边形具有 180 度对称性，因此顶点$i$必须对应于顶点$i + n/2$。 任何此类对的中点都是同一点，它唯一地标识$O$。 

一次$O$众所周知，任何方向向量都定义了有效的切割线。 选择简单的轴对齐线可以避免分数坡度复杂化。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力削减候选人|$O(n^2 \cdot n)$|$O(n)$| 太慢了|
 | 对称中心 + 任何穿过它的线 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们利用中心对称多边形的结构来重建对称中心，然后输出有效的切割线。 

1. 按顺序读取多边形顶点。 顺序是一致的并且已经是 CCW，因此不需要预处理。 
2.利用对称性：顶点$i$映射到顶点$i + n/2$。 计算这两点的中点。 这个中点就是对称中心$O$。 
3.重复或隐式信任一致性：由于完美180度对称的保证，所有这些中点都重合。 
4. 构造任意穿过的线$O$。 一个简单的选择是水平线或具有小合理斜率的线。 例如，使用积分$(x_O, y_O)$和$(x_O + 1, y_O)$。 
5. 以所需的分数格式输出两个选定的点。 由于直接允许整数，因此不需要标准化。 

### 为什么它有效

 多边形的 180 度旋转对称意味着通过中心的任何直线一侧的每个点都会镜像到另一侧的点。 这确保了穿过中心的任何线的面积划分相等。 

唯一剩下的问题是切割是否产生恰好两个连接的组件。 在简单的中心对称多边形中，形状必然是凸的，这保证了任何直线最多与边界相交两次。 因此，切割始终是单个和弦分割，恰好产生两个片段。 

该算法不会失败，因为两个必需条件（面积相等和恰好有两个部分）同时由凸性和中心对称性强制执行。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    
    # symmetry center from i and i + n/2
    x0 = y0 = 0
    
    half = n // 2
    for i in range(half):
        x0 += pts[i][0] + pts[i + half][0]
        y0 += pts[i][1] + pts[i + half][1]
    
    # each pair contributes twice the center, so divide by n
    # center = (sum of pairs) / n
    # but we accumulated both sides, so divide by n
    if x0 % n == 0:
        cx = x0 // n
        cx_d = 1
    else:
        cx = x0
        cx_d = n
    
    if y0 % n == 0:
        cy = y0 // n
        cy_d = 1
    else:
        cy = y0
        cy_d = n
    
    # output a horizontal line through center
    # second point uses fraction form if needed
    print(f"{cx} {cx_d} {cy} {cy_d} {cx+1} {cx_d} {cy} {cy_d}")

if __name__ == "__main__":
    solve()
```该实现直接对顶点进行配对$i$和$i + n/2$计算对称中心。 对两个坐标求和可确保每个镜像对的所有贡献都包含一次。 

选择输出线通过计算的中心水平。 由于该问题允许任意分数，因此两个坐标表示相同的分母就足够了，并且避免了不必要的简化。 

计算中点时必须保持一致。 一个常见的错误是单独划分每一对并累加浮点值，这可能会引入精度问题。 使用整数累加可以保持准确性。

 ### 示例 1

 | 总和 x | 0 |
 | 总和 | 0 |

 该算法输出一条通过原点的水平线。 这清楚地将正方形分成面积相等的两半，并恰好与两个边界边相交。 

### 示例 2

 对于更不规则的中心对称多边形，假设顶点对总和为$(14, 26)$超过$n=8$。 

| 步骤| 价值|
 | ---| ---|
 | 总和 x | 14 | 14
 | 总和 | 26 | 26
 | 中心| (14/8, 26/8) = (7/4, 13/4) | (14/8, 26/8) = (7/4, 13/4) |
 | 已选线路| y = 13/4 |

 该线穿过精确的对称中心。 根据对称性，线上方的每个线段在其下方都有一个镜像对应部分，从而保持相等的面积。 

该迹线表明分数中心是自然处理的，不需要几何重建。

 ## 边缘情况

 关键的边缘情况是对称中心为非整数时。 在这种情况下，强制整数除法的简单实现将移动线并破坏对称性。 正确的处理使用坐标的理性表示，确保线仍然准确地通过中点。 

另一种情况是由于差一索引导致顶点配对不正确。 如果顶点$i$与$i + n/2 - 1$，计算出的中心发生漂移，并且生成的线无效。 该算法严格依赖于精确的半旋转分度。 

最后，如果在不检查对称性的情况下假设任何任意直线都是有效的，则可以在中间推理中构造一条与多边形相交两次以上的直线。 凸性保证可以防止这种情况发生，但前提是正确识别和使用中心对称性。
