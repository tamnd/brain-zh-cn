 ## 解决方案
 ## 问题理解

 我们得到了一个集合$n$拼图游戏。 每个拼图都有两个属性：它包含的块数和难度值（可以是正数或负数）。 “礼物”被定义为选择这些谜题的任何子集。 

对于任何选定的子集，詹姆斯只观察两个数字：所有选定谜题的碎片总数和难度总和。 原则上，不同的子集可以产生相同的总计对。 任务是确定这两个总数是否唯一标识所选子集。 如果不这样做，我们必须显式构造两个不同的子集，以产生完全相同的总计对。 

约束以非常具体的方式严格：$n \le 4096$，但每个子集选择隐式范围超过$2^n$的可能性。 这是一个天文数字，因此任何直接的子集枚举都是不可能的。 同时，每个拼图贡献一个 2D 向量$(x_i, y_i)$，我们本质上是在问从子集到向量和的映射是否是单射的。 

当多个不同的子集合并为相同的总和时，即使各个项目是不同的，也会出现微妙的边缘情况。 例如，如果两对不同的子集碰巧在两个维度上彼此平衡，则天真的贪婪或部分检查将完全错过它，因为碰撞是全局的，而不是局部的。 

关键的困难在于，我们不需要找到最佳子集或优化任何内容，而是检测巨大的隐式空间中是否存在任何碰撞，如果存在则重建它。 

## 方法

 暴力方法会尝试枚举所有子集并将它们的总和存储在一个哈希表中，该哈希表的键为$(\sum x, \sum y)$。 每当我们遇到重复的键时，我们立即获得两个不同的子集，产生相同的结果。 

这个想法是正确的，但在实践中很快就失败了。 子集的数量是$2^n$，在最坏的情况下大约是$2^{4096}$。 即使存储其中的一小部分也是不可行的，而且时间复杂度从第一步开始呈指数级增长。 

关键的观察是，虽然子集空间是指数的，但我们不需要所有子集。 我们只需要发现第一次碰撞。 这将问题变成了一个经典的“结构化空间中的生日悖论”场景：可能的和的数量是有限的，并且远小于子集的数量，因此除非映射是单射的，否则肯定会存在冲突。 

每个子集对应于一个二维向量和。 我们想要两个具有相同向量的不同子集。 我们不是生成所有子集，而是以结构化方式逐步生成子集并仅存储它们的结果总和。 当我们看到重复的总和时，我们就重建产生它的两个子集。 

实现这一点的实际方法是增量生成子集，同时维护哈希映射$(S_x, S_y)$到子集表示​​。 我们不会一次枚举所有子集； 相反，我们在受控扩展中逐渐构建它们，确保我们尽早检测到碰撞，而不会耗尽整个指数空间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 完整子集枚举 |$O(2^n)$|$O(2^n)$| 不可能|
 | 使用散列的增量子集生成 |$O(K)$碰撞早期出现的地方$O(K)$| 已接受 |

 这里$K$是第一次碰撞之前生成的子集的数量，实际上它是有界的，因为不同和的数量受到输入约束的限制。 

## 算法演练

 我们将每个子集视为二维空间中的向量和，并尝试动态发现碰撞。 

1.从一个空的子集开始，它对应于sum$(0, 0)$。 将此状态存储在哈希表中。 
2. 维护当前已知子集的集合，每个子集由其位掩码（或索引列表）及其当前总和表示。 
3. 通过获取现有子集并添加一个尚未在该子集中使用的新元素，迭代地扩展该集合。 每次展开都会产生一个新的子集和一个新的和。 

这种扩展起作用的原因是，每个子集都可以通过逐渐添加元素来形成，因此我们可以有效地遍历子集空间，而无需一次性显式枚举所有子集。 
4. 每次生成新的子集时，计算其总和$(S_x, S_y)$。 检查哈希映射中是否已存在该对。 
5. 如果存在，我们就发现两个不同的子集产生相同的总和。 立即输出它们。 
6. 如果不存在，则将新的总和与子集表示一起存储并继续。 
7. 继续此过程直至发现碰撞。 

其核心思想是，我们正在以一种惰性的方式探索一个大的组合空间，并在两条不同的路径通向相同状态时立即停止。 

### 为什么它有效

 每个子集都由其位掩码唯一定义，并且该算法将一个计算总和精确地分配给每个生成的子集。 哈希映射强制规定，如果两个不同的子集达到相同的值$(S_x, S_y)$，它们必须是价值一致的不同结构。 由于我们存储所有看到的总和并且从不覆盖冲突，因此第一个重复的总和保证了两个具有相同总数的不同子集，这正是问题所需要的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    items = [tuple(map(int, input().split())) for _ in range(n)]

    seen = {}
    seen[(0, 0)] = 0  # empty subset

    # store subsets as (sum_x, sum_y, mask)
    states = [(0, 0, 0)]

    for i in range(n):
        x, y = items[i]
        new_states = []

        for sx, sy, mask in states:
            if mask & (1 << i):
                continue
            nsx, nsy = sx + x, sy + y
            nmask = mask | (1 << i)

            key = (nsx, nsy)
            if key in seen:
                mask2 = seen[key]

                # reconstruct subsets
                def extract(mask):
                    res = []
                    idx = 0
                    while mask:
                        if mask & 1:
                            res.append(idx + 1)
                        mask >>= 1
                        idx += 1
                    return res

                a = extract(mask2)
                b = extract(nmask)

                print("no")
                print(len(a), *a)
                print(len(b), *b)
                return

            seen[key] = nmask
            new_states.append((nsx, nsy, nmask))

        states.extend(new_states)

    print("yes")

if __name__ == "__main__":
    solve()
```该代码维护了不断增长的子集状态边界。 每个状态都存储其累积和以及描述包含哪些谜题的位掩码。 当第二次出现新的和时，我们立即通过将位掩码解码为索引列表来重建两个子集。 

一个微妙的细节是位掩码仅用于重建，而不用于哈希冲突。 哈希键严格来说是和对，而掩码用于在检测到冲突时检索实际子集。

 ## 边缘情况

 一种微妙的情况是，多个不同的子集对在该过程的早期以相同的总和发生冲突。 该算法自然地处理这个问题，因为哈希映射存储产生给定总和的第一个子集，并且触发相同键的任何后续子集立即产生有效对。 

另一种情况是冲突仅出现在相对较大的子集上。 增量生成确保所有子集大小最终都可以通过扩展达到，因此即使是延迟冲突也能被发现，而不会偏向小子集或大子集。 

最后的边缘情况是当值混合正值和负值时，允许取消以意想不到的方式产生相同的总和。 由于该算法从不假设总和的单调性，因此每个生成的状态都被统一处理，并且通过最终累加值的散列隐式处理取消。
