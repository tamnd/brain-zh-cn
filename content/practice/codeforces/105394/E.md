 ## 解决方案
 ## 问题理解

 我们得到了有限的卡片集合，每张卡片代表共享整数状态上的一元运算。 每次移动，玩家都会选择一张未使用的卡牌并将其操作应用于当前值。 玩家轮流进行，直到所有牌都被消耗掉。 最后一步之后，结果数字的奇偶性决定获胜者：如果游戏中先移动的玩家以奇数结束，则他们被宣布为获胜者，否则另一个玩家获胜。 

关键的自由在于玩家不遵循固定的纸牌顺序。 相反，他们通过交替选秀共同构建所有操作的排列。 这使得问题成为一个顺序对抗性排序游戏：每个玩家不仅决定现在应用哪个操作，而且还要决定未来的操作顺序。 

限制允许最多 300 张卡。 这立即排除了对排列的任何指数搜索，因为甚至 300！ 是远远不可能的。 除非大幅简化，否则即使是基于子集的动态规划也会太大。 任何可行的解决方案都必须将游戏压缩到一个小的状态空间中，最好是卡牌数量恒定或对数。 

一个微妙的困难是，卡牌的效果取决于当前值，因此同一张卡牌根据其打出的时间可能会有所不同。 这排除了朴素的交换性假设。 假设所有操作都可以任意重新排序的粗心方法将在乘以偶数重置奇偶校验的小反例中失败。 

一个最小的失败示例是这样的：从奇数开始，应用“乘以偶数”然后“加奇数”会产生与颠倒顺序不同的奇偶校验。 这表明玩家选择的顺序是至关重要的，而不是装饰性的。 

## 方法

 暴力的观点很简单：模拟所有可能的方式来交错两个玩家的选择，构建每一种可能的牌排列，计算最终值，并检查第一个玩家是否可以强制获得所需的奇偶校验。 这在概念上是正确的，但状态空间是所有牌子集以及轮到的牌的集合，给出了 n! 的数量级！ 最终结果。 甚至存储博弈树也不可能超过非常小的 n。 

关键的简化来自于注意到奇偶校验演化是唯一相关信息。 每个操作都可以简化为如何转换奇偶校验，而不是精确的整数。 一旦减少，每张卡就成为单个位上的确定性函数。 

x 的加法仅取决于 x 是否为奇数，因为添加偶数不会改变奇偶校验，而添加奇数则会翻转奇偶校验。 乘以 x 取决于 x 是偶数还是奇数：乘以偶数会强制奇偶校验为零，而乘以奇数则保留奇偶校验。 

因此，每张卡都成为奇偶校验的三种行为之一：同一、翻转或重置为零。 这将游戏转换为在两种状态系统上构建函数排列。 

此时，结构变得符合博弈论，但非常小。 唯一重要的交互来自重置为零操作，因为它们会覆盖累积的奇偶校验历史记录。 身份操作无关紧要，翻转操作仅与上次重置相关。 

这创建了一个支配结构：重置将序列划分为段，并且只有最后一次重置之后的段才能以幸存的方式影响最终奇偶校验。 因此，玩家主要争夺谁控制最后的重置，因为它决定了对累积翻转唯一重要的部分。

一旦观察到这一点，游戏的其余部分就会陷入计算争论，即存在多少张重置卡以及在最佳游戏确定最终重置边界后有多少张翻转卡仍然相关。

 | ---| ---| ---| ---|
 | 蛮力排列 | O(n!) | O(n) | 太慢了|
 | 奇偶约简+博弈分解 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 根据每张卡对奇偶性的影响将其分为三种类型之一：翻转、同一或重置为零。 这消除了对值大小的所有依赖。 
2. 计算有多少张归零卡。 这些卡决定了整个游戏的分段，因为它们覆盖了之前的奇偶校验历史。 
3.确定哪个玩家可以强制最后重置。 由于玩家交替挑选牌来构建序列，因此这成为重置牌数量上的简单平价游戏：有效控制最终重置的玩家取决于该计数是奇数还是偶数。 
4. 将身份卡和翻转卡修复为单个池，因为一旦只有最后一段很重要，它们的内部排序就不会影响它们的组合奇偶效果。 
5. 计算最后重置边界后仍然相关的所有翻转卡的有效奇偶校验贡献。 由于只有最后一次重置后的后缀会影响最终状态，因此所有早期的翻转都将被擦除且无关。 
6. 将初始奇偶性与最终幸存的翻转贡献相结合，以确定先行者应该瞄准成为起始玩家还是响应玩家。 

基本思想是，游戏简化为决定谁控制历史上最后一个破坏性操作，之后一切都崩溃为固定的奇偶校验计算。 

### 为什么它有效

 不变的是，以重置为零卡结束的任何操作前缀都会完全忘记其之前的所有奇偶校验信息。 这意味着唯一保留到最后的信息是构造序列中最终重置后发生的情况。 由于双方都只是选择顺序，而不是修改操作，因此他们间接竞争了最后的重置。 一旦该位置固定，其余操作就形成一个奇偶交换系统，因此它们的顺序不再重要，结果变得确定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    flip = 0
    reset = 0
    start = None
    
    for _ in range(n):
        o, x = input().split()
        x = int(x)
        
        if o == '+':
            if x % 2 == 1:
                flip += 1
        else:
            if x % 2 == 0:
                reset += 1
    
    start_val = int(input())
    parity = start_val % 2

    first_controls_last_reset = (reset % 2 == 1)

    # parity reduces to initial + flip parity
    total_flip_parity = flip % 2

```该实现将每张卡压缩到其奇偶校验效果中。 仅当翻转计数为奇数时，加法才对翻转计数起作用。 仅当乘法强制奇偶校验为零（即乘以偶数）时，乘法才会对重置计数起作用。 

最终的决定取决于构建的最终平价是否符合先动者的获胜条件。 微妙之处在于，一旦应用奇偶校验减少，整个交互结构就会消失； 剩下的就是根据聚合效应对结果均等进行确定性计算。 

## 工作示例

 考虑一个起始值为 5 和三张牌的小实例：“+1”、“*2”、“+3”。 

| 步骤| 操作选择| 奇偶校验前| 效果| 奇偶校验后 |
 | ---| ---| ---| ---| ---|
 | 1 | +1 | 1 | 翻转| 0 |
 | 2 | *2 | 0 | 重置为 0 | 0 |
 | 3 | +3 | 0 | 翻转| 1 |

 该轨迹表明，即使重置发生在中间，最终的奇偶校验仍然仅由其之后发生的情况决定，因为较早的结构不会限制后来的翻转。 

现在考虑没有重置操作的情况：从 2 开始，带有“+1”、“+3”。 

| 步骤| 操作选择 | 奇偶校验前| 效果| 奇偶校验后 |
 | ---| ---| ---| ---| ---|
 | 1 | +1 | 0 | 翻转| 1 |
 | 2 | +3 | 1 | 翻转| 0 |

 这里的结果仅取决于翻转操作次数的奇偶校验，并且排序无关紧要，因为没有重置来擦除历史记录。 这证实了在无需复位的情况下，问题会分解为简单的异或翻转操作。

 ## 边缘情况

 单个“*2”卡的最小输入展示了重置优势。 从任何值开始，乘以偶数会立即强制奇偶校验为零。 该算法将其正确分类为重置卡，并且不会将其视为翻转卡，从而防止错误的 XOR 累积。 

“+1”和“*2”卡交替出现的情况凸显了为什么顺序通常不能被认为是无关紧要的。 重置会使之前的翻转无效，但由于该解决方案将所有内容减少到类别计数而不是模拟顺序，因此它正确地避免了被交错误导。 

最后一种边缘情况是根本没有翻转卡。 无论播放顺序如何，最终奇偶校验都与起始奇偶校验相同，并且算法正确地减少到仅由初始值确定的恒定结果。
