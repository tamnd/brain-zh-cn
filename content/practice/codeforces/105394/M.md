 ## 解决方案
 ## 问题理解

 我们得到了一系列钢琴键相对于第一个键的音高偏移。 这些值描述当前的相对调音，而不是绝对频率，但它们在将每个键移动相同常数表示有效的绝对调音的意义上是一致的。 

目标是将钢琴转变为完美的“音乐楼梯”，其中每个琴键都比前一个琴键高一个半音。 这意味着最终的配置必须形成一个等差级数，连续键之间的差值为 1。 我们可以自由选择绝对起始音高，但一旦起始值固定，所有其他键都将被强制。 

我们可以独立更改每个键，成本是应用于每个键的绝对调整的总和。 任务是选择最好的最终楼梯，以使总成本最小化。 

约束最多为 100000 个键，这会立即排除任何通过强力对齐尝试每种可能的目标配置的二次策略。 在最坏的情况下，任何在线性时间内重新计算每个候选起始值成本的操作都会导致大约 10^10 次操作，这远远超出了限制。 这促使我们采用一种公式，将问题简化为对单个参数的众所周知的优化。 

输入值是相对的，而不是绝对的，这一事实引起了一个微妙的问题。 一种幼稚的方法可能会尝试直接将序列 t 与干净的递增序列对齐，但会忽略只有差异才重要，而不是绝对基线。 另一个常见的错误是假设目标序列从零开始是固定的，但起始音高实际上是自由的，必须进行优化。 

## 方法

 如果我们确定候选最终调整，则成本很容易计算：我们将每个期望值与当前值进行比较，并对绝对差值求和。 蛮力的想法是尝试最终楼梯的所有可能的起始间距。 如果第一个键是 x，则序列变为 x、x+1、x+2 等等，我们计算每个 x 的总成本。 

这是正确的，因为每个有效的解决方案完全由单个参数确定。 然而，可能的 x 值的范围很大，对于每个 x，我们将扫描所有 n 个键。 当 n 达到 100000 并且 x 跨越 200000 量级的范围时，这会导致大约 2×10^10 次操作，这太慢了。 

关键的观察是我们不应该直接与原始序列进行比较。 相反，我们将目标序列的强制结构与数据分开。 如果最终目标是 x + (i−1)，我们以将 x 与输入的索引相关部分隔离的方式重写成本。 这将问题转化为选择一个数字 x 来最小化到派生数组的 L1 距离。 

一旦问题以这种形式出现，它就成为选择一个点来最小化绝对偏差之和的经典问题，其解是中位数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解起始值 | O(nR) | O(1) | O(1) | 太慢了|
 | 中值变换 | O(n log n) | O(n log n) | O(n) | 已接受 |

 这里R代表可能的起始偏移量的范围，它足够大以至于暴力破解不可行。 

## 算法演练

1. 将 i 键所需的最终音高重写为 x + (i−1)，其中 x 是未知的起始音高。 这体现了这样一个事实：一旦选择了第一个密钥，最终配置就完全确定了。 
2. 将每个输入值 t[i] 转换为归一化值 b[i] = t[i] − (i−1)。 这消除了目标结构中的强制线性趋势，并将所有可变性隔离到单个共享参数中。 
3. 观察到总成本变为 |(x + (i−1)) − t[i]| 的 i 之和，这正好简化为总和 |x − b[i]|。 此时，问题仅取决于选择x。 
4. 对数组 b 进行排序。 需要排序是因为最小化绝对偏差之和的最佳值取决于排序。 
5. 选择 x 作为 b 的中值元素。 如果 n 是奇数，则这是中间元素。 如果 n 为偶数，则两个中间元素之间的任何值都会给出相同的最佳成本，因此选择任一中间元素就足够了。 
6. 将最终答案计算为 |x − b[i]| 之和 奥沃岛

 ### 为什么它有效

 该变换将所有约束简化为单变量优化问题，其中成本函数是凸且分段线性的。 对于此类函数，斜率在数据中值处改变符号。 x 远离中位数的任何移动都会增加一侧的不平衡，多于另一侧减少的不平衡，从而保证中位数最小化总绝对偏差。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    t = list(map(int, input().split()))

    b = [t[i] - i for i in range(n)]

    b.sort()
    x = b[n // 2]

    ans = 0
    for v in b:
        ans += abs(v - x)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先构建转换后的数组 b，该数组消除了由于相邻键相差正好 1 的要求而引起的强制斜率。 这是关键的标准化步骤； 如果没有它，这个问题看起来就像一个约束向量拟合问题，而不是一个单参数优化问题。 

对 b 进行排序是必要的，因为中位数仅在有序结构中才有意义。 直接选择中间元素，因为任何中值都会最小化 L1 目标。 

最后一个循环计算与中值的总绝对偏差，这直接对应于总调整工作。 

一个常见的实现错误是忘记 b[i] = t[i] − i 中的索引移位。 另一种方法是尝试在不排序的情况下计算答案，并错误地选择平均值而不是中位数，这对于平方误差是正确的，但对于绝对误差则不是。 

## 工作示例

 考虑一个示例序列 t = [0, 1, 4, 3, 6]。 转换后，我们计算 b[i] = t[i] − i：

 | 我| t[i] | t[i] | b[i] |
 | ---| ---| ---|
 | 0 | 0 | 0 |
 | 1 | 1 | 0 |
 | 2 | 4 | 2 |
 | 3 | 3 | 0 |
 | 4 | 6 | 2 |

 对 b 进行排序得到 [0, 0, 0, 2, 2]，因此中位数为 x = 0。成本为 0+0+0+2+2 = 4。 

该轨迹表明，归一化后，多个关键点已经与最佳楼梯对齐，并且只有偏差起作用。 

现在考虑 t = [0, -2, 10, 6, 7, -1]。 我们计算 b：

 | 我| t[i] | t[i] | b[i] |
 | ---| ---| ---|
 | 0 | 0 | 0 |
 | 1 | -2 | -3 |
 | 2 | 10 | 10 8 |
 | 3 | 6 | 3 |
 | 4 | 7 | 3 |
 | 5 | -1 | -6 |

 排序得到 [-6, -3, 0, 3, 3, 8]。 中位数范围在 0 到 3 之间，因此我们选择 x = 0。最终成本为 6 + 3 + 0 + 3 + 3 + 8 = 23。 

此示例表明，即使值广泛传播，最佳锚点也位于变换后的分布的中心。

 | 测量 | 复杂性 | 说明|

 | 时间 | O(n log n) | O(n log n) | 排序占主导，线性传递成本|

 该解决方案完全符合约束条件，因为在 Python 中对 100000 个元素进行排序是高效的，并且其余计算是线性的。

 ## 边缘情况

 对于 n = 2，该算法简化为选择单个 x 来最小化两个绝对偏差。 变换后，b 有两个值，任一中值选择都给出相同的最优成本。 例如，t = [0, 5] 变为 b = [0, 4]，选择 x = 0 会产生成本 4，而 x = 4 也会产生成本 4，这与它们之间的任何点在 L1 中都是最优这一事实相匹配。 

对于已经是完美楼梯的输入，例如 t = [0, 1, 2, 3]，变换后的数组将变为常数，b = [0, 0, 0, 0]。 中位数为 0，每个偏差都为零，因此算法无需任何调整即可正确返回 0。
