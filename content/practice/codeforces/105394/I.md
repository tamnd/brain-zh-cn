 ## 解决方案
 ## 问题理解

 我们在一条非常大的一维线上进行了一系列操作，其中位置最多可达 1e9。 在线执行两种类型的操作。 

第一种类型插入波浪。 每个波从某个位置开始`p`，跨越一段连续的长度`ℓ`，并且具有振幅`a`。 波形的形状是固定的：每 4 个位置重复一次，从正峰值开始`p`。 具体来说，当我们从`p`，贡献遵循重复模式`+a, 0, -a, 0, +a, 0, -a, 0, ...`截断为段长度。 因此，只有每四个位置有贡献，在正振幅和负振幅之间交替。 

第二种类型是查询，询问应用所有先前添加的波浪后单个位置的总高度。 由于波会累积，每个查询都会询问该坐标处所有活动波的贡献总和。 

关键的困难在于位置很大，最多 1e9，但操作数量很少，最多 4000。这立即排除了维护显式大小的数组`w`或进行每个位置的模拟。 每次更新迭代所有受影响单元的任何解决方案都会花费高达 O(n·w) 的成本，这是不可能的。 

一个更微妙的困难是每次更新都不是统一的范围添加。 它是一种结构化的周期性模式。 一个幼稚的错误是将每个波视为间隔内 +a 的简单范围更新，但这会忽略交替符号并产生不正确的结果。 

当更新重叠时，会出现另一种常见的失败情况：两个波可能会相互干扰，并且它们的相位取决于绝对位置，而不是每个波内的局部对齐。 例如，从位置 1 开始的波和从位置 2 开始的波即使在同一段上也会产生完全不同的残留模式，因此仅按间隔分组是不够的。 

主要的边缘情况来自于间隔的逐一处理`[p, p + ℓ - 1]`以及在整个坐标系上正确对齐 4 周期相位。 

## 方法

 直接模拟将维护一个阵列，并且对于每个波，迭代其间隔并仅在与周期性模式匹配的位置应用贡献。 每个长度为 ℓ 的波都会花费 O(ℓ)，导致最坏的情况约为 4e3 × 1e9 次操作，这是完全不可行的。 

关键的观察结果是波形仅取决于`position mod 4`。 每个位置恰好属于四个独立算术级数之一：余数 0、1、2 和 3 模 4。在每个余数类中，波贡献`+a`,`-a`， 或者`0`，最重要的是，这种行为对于该残基的所有位置都是一致的。 

这使我们能够将问题分成四个独立的范围添加结构。 然而，我们仍然面临更新是在任意大坐标上的问题。 这是通过坐标压缩解决的，但不是以简单的方式全局解决。 相反，我们在每个残基类中单独压缩，因为只有具有相同残基的位置才会在相同的更新规则下相互作用。 

一旦每个残差被压缩，每个波更新就变成坐标子集上的范围加法：我们将所有位置定位在`[p, p + ℓ - 1]`属于残留物`p mod 4`（他们收到`+a`），类似地那些属于残基的`(p + 2) mod 4`（他们收到`-a`）。 另外两个残基不受影响。 

每个残基类别均由支持范围更新和点查询的芬威克树维护，使我们能够以对数时间针对每个受影响的类别应用每个波。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n·ℓ) | O(n·ℓ) | O(w) | 太慢了|
 | 最优（残渣分解+BIT）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 收集所有重要的坐标：每个查询位置和每个波间隔的两个端点。 这确保我们只在相关点上构建结构，而不是整个坐标范围。 
2. 根据它们模 4 的值将这些坐标分为四组。每组将被独立处理，因为波贡献永远不会混合残差。 
3. 对于每个残基组，对坐标进行排序并构建从坐标值到压缩索引的映射。 这使我们能够使用紧凑的芬威克树而不是稀疏的地图。 
4. 为每个残基组初始化一个 Fenwick 树。 每棵树存储仅影响该残留类别的累积贡献。 
5. 按顺序处理操作。 为了一波`! p ℓ a`，确定其活跃区间`[p, p + ℓ - 1]`。 
6.在残渣类中`r = p mod 4`，所有位置都符合`r`收到`+a`每当他们位于间歇期时。 我们找到与这些有效位置相对应的压缩索引的子范围，并应用范围添加`+a`。 
7.在残渣类中`r2 = (p + 2) mod 4`，贡献是`-a`，因此我们应用具有负幅度的相同范围更新。 
8. 查询`? x`， 决定`r = x mod 4`, 定位`x`在其残基组中，并查询该索引处的 Fenwick 树以获得累加和。 

### 为什么它有效

 每个位置恰好属于一个残基类别，并且波型完全由相对于波起点的残基偏移确定。 在固定的残差类内，每个波在压缩坐标的连续段上贡献一个统一的值，因此芬威克树范围更新可以正确累积所有贡献。 由于我们从不混合残差，因此波之间的干扰可以通过四个结构的独立求和来完全捕获。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class BIT:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_add(self, l, r, v):
        if l > r:
            return
        self.add(l, v)
        if r + 1 <= self.n:
            self.add(r + 1, -v)

def solve():
    n, w = map(int, input().split())
    ops = []

    coords = [[] for _ in range(4)]

    for _ in range(n):
        tmp = input().split()
        if tmp[0] == '?':
            p = int(tmp[1])
            ops.append(('q', p))
            coords[p % 4].append(p)
        else:
            p, l, a = map(int, tmp[1:])
            ops.append(('u', p, l, a))
            r1 = p
            r2 = p + l - 1
            coords[p % 4].append(r1)
            coords[p % 4].append(r2)
            coords[(p + 2) % 4].append(r1)
            coords[(p + 2) % 4].append(r2)

    mp = [{} for _ in range(4)]
    bit = []

    for r in range(4):
        vals = sorted(set(coords[r]))
        mp[r] = {v: i + 1 for i, v in enumerate(vals)}
        bit.append(BIT(len(vals)))

    def range_add(r, lval, rval, val):
        if lval > rval:
            return
        l = mp[r].get(lval)
        rgt = mp[r].get(rval)
        if l is None or rgt is None:
            return
        bit[r].range_add(l, rgt, val)

    def point_get(r, x):
        i = mp[r][x]
        return bit[r].sum(i)

    for op in ops:
        if op[0] == 'u':
            _, p, l, a = op
            r1 = p % 4
            r2 = (p + 2) % 4
            L = p
            R = p + l - 1

            range_add(r1, L, R, a)
            range_add(r2, L, R, -a)

        else:
            _, x = op
            r = x % 4
            print(point_get(r, x))

if __name__ == "__main__":
    solve()
```该实现依赖于按残差类别分离贡献，这将周期性结构转变为统一范围更新。 每个BIT都支持前缀累加，范围更新通过标准差分技巧实现。 关键细节是所有坐标处理都是按残基完成的； 混合它们会破坏波形的相位对齐。 

映射步骤确保我们永远不会分配大小的数组`w`，并且所有操作仍然受到更新或查询实际触及的不同坐标数量的限制。

 ## 边缘情况

 当波间隔恰好在属于负相位的位置处结束时，会发生一种微妙的情况。 由于该模式是周期性的且具有固定的全局对齐方式，因此最后一个元素不一定对应于完整的周期。 该实现自然地处理了这个问题，因为更新是通过过滤精确的坐标成员资格而不是迭代循环来应用的。 

当两个波从不同的残基开始但间隔重叠时，就会出现另一种边缘情况。 例如，从 2 开始的波和从 3 开始的波都覆盖位置 6，但它们的贡献存储在不同的残差 BIT 中，直到查询时对它们进行求和。 这种分离可以防止在简单的合并结构中发生的相位损坏。 

最后一个边缘情况是`ℓ = 1`，其中只有起始残基收到贡献。 范围更新退化为单点更新，并且 Fenwick 逻辑仍然适用，因为压缩区间正确地折叠为一个索引。
