 ## 解决方案
 ## 问题理解

 我们正在研究从 1 到 n 的位置的一维巷道。 在某些位置，我们可以放置或移除灯，每个灯都有一个正的亮度值。 环境有雾，光线不能自由传播：每传播一米，就会乘以相同的衰减系数，即$1 - p$， 在哪里$p$被给出为严格介于 0 和 1 之间的实数。 

当灯放置在位置时$x_i$有亮度$b_i$，它对任意一点都有贡献$x$沿着小巷。 该贡献仅取决于灯和点之间的距离，并随该距离呈指数衰减。 如果距离是$d = |x - x_i|$，那么灯贡献$b_i \cdot (1 - p)^d$到亮度$x$。 某一点的总亮度是所有当前活动灯的亮度之和。 

该系统是动态的。 灯被插入和移除，我们必须回答询问给定位置的当前亮度的查询。 

约束条件很大：最多$2 \cdot 10^5$运营。 任何重新计算每个查询的所有灯的贡献的方法都会立即变得太慢，因为在最坏的情况下，这将是$O(nq)$，这远远超出了可接受的范围。 该结构表明我们需要更接近对数或摊销常数的每次操作时间。 

一个微妙的困难来自于距离的绝对值。 灯的贡献取决于它位于查询点的左侧还是右侧，这会阻止单个简单的前缀结构直接工作。 

第二个问题是数值结构：衰减是乘性的并且取决于距离，这表明将位置的变化转换为乘性因子的变换，这些乘性因子可以被预先吸收到存储的值中。 

当在具有相同亮度的相同坐标处插入和移除灯时，会出现边缘情况，因为结构必须将它们视为独立事件，而不是单独按位置合并。 另一个微妙的情况是在没有灯的情况下进行查询，其中答案必须为零而不是未初始化的值。 

## 方法

 直接方法通过迭代所有活动灯并对它们的贡献求和来评估每个查询。 这是正确的，因为它遵循字面定义：每个灯根据距离独立做出贡献。 然而，每次查询都会花费$O(n)$，并且最多$2 \cdot 10^5$查询，总工作量达到$O(nq)$，这大约是$4 \cdot 10^{10}$最坏情况下的操作，远远超出可行的执行范围。 

关键的观察是，基于距离的指数衰减可以分为两种情况，具体取决于灯位于查询位置的左侧还是右侧。 对于固定查询位置$x$，我们将灯分成具有位置的灯$i \le x$和那些有$i > x$。 这消除了绝对值并使每条边在代数上一致。 

为了$i \le x$，贡献是$b \cdot (1 - p)^{x - i}$，可以重写为$(1 - p)^x \cdot b \cdot (1 - p)^{-i}$。 为了$i > x$，贡献是$b \cdot (1 - p)^{i - x} = (1 - p)^{-x} \cdot b \cdot (1 - p)^i$。 

此转换隔离了对查询位置的所有依赖$x$分解为简单的乘法因子，而所有对灯位置的依赖都可以预先聚合。 问题简化为在位置上维护两个动态前缀样式聚合，这建议使用芬威克树或线段树。 

我们维持两种独立的结构：一种是商店$b \cdot (1 - p)^{-i}$，以及其他商店$b \cdot (1 - p)^i$。 每次更新只影响一个索引，每个查询都成为前缀和、后缀和以及两个预先计算的幂的组合$(1 - p)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(nq)$|$O(n)$| 太慢了|
 | 芬威克 + 变换 |$O(q \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们定义$r = 1 - p$，每米衰减系数。 

### 步骤

 1. 预先计算值$r^i$和$r^{-i}$对于所有相关指数，直至$n$。 

这些能力使我们能够将距离偏移转换为独立于查询的乘法调整。 
2. 在位置 1 到 n 上维护两棵 Fenwick 树。 

第一棵树存储值$b \cdot r^{-i}$、以及第二家店$b \cdot r^i$。 
3. 当将灯插入位置时$x$有亮度$b$，更新索引处的两棵树$x$： 添加$b \cdot r^{-x}$到第一棵树和$b \cdot r^x$到第二棵树。 

删除对负值执行相同的更新。 
4. 回答该位置的查询$x$，计算前缀和$S_1 = \sum_{i \le x} b_i r^{-i}$从第一棵树开始。 
5. 计算后缀和$S_2 = \sum_{i > x} b_i r^i$通过从总和中减去前缀来从第二棵树中得到。 
6. 使用以下方法合并两个贡献：$$\text{answer}(x) = r^x \cdot S_1 + r^{-x} \cdot S_2$$### 为什么它有效

 每个灯的贡献都被重写为一种形式，其中对查询位置的依赖被完全分解。 芬威克树仅存储位置相关的组件，这些组件在更新时保持稳定。 每个灯在任何查询的左分区或右分区中都被计算一次，并且代数变换确保不会发生重叠或遗漏。 由于更新和查询仅操作聚合的转换值，因此正确性来自求和的线性度和距离项的精确分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0.0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0.0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

def solve():
    n, q, p = input().split()
    n = int(n)
    q = int(q)
    p = float(p)

    r = 1.0 - p

    bit1 = Fenwick(n)
    bit2 = Fenwick(n)

    # precompute powers
    rp = [1.0] * (n + 1)
    rm = [1.0] * (n + 1)
    for i in range(1, n + 1):
        rp[i] = rp[i - 1] * r
        rm[i] = rm[i - 1] / r

    total2 = 0.0

    for _ in range(q):
        tmp = input().split()
        if tmp[0] == '+':
            b = int(tmp[1])
            x = int(tmp[2])
            bit1.add(x, b * rm[x])
            bit2.add(x, b * rp[x])
            total2 += b * rp[x]

        elif tmp[0] == '-':
            b = int(tmp[1])
            x = int(tmp[2])
            bit1.add(x, -b * rm[x])
            bit2.add(x, -b * rp[x])
            total2 -= b * rp[x]

        else:
            x = int(tmp[1])
            s1 = bit1.sum(x)
            s2 = total2 - bit2.sum(x)
            ans = (rp[x] * s1) + (rm[x] * s2)
            print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现直接反映了代数分解。 第一棵芬威克树在按位置标准化后累积了左侧灯的贡献。 第二棵树对所有灯执行相同的操作，但用于通过总减前缀技巧重建右侧贡献。 

一个微妙的点是维护第二个转换数组的运行总计，因为后缀查询不直接受 Fenwick 树支持，无需从全局总和中反转或减去。 

所有算术都以浮点形式执行，因为指数衰减在问题陈述中本质上是实值。 最后的模归约对应了问题对表达最终有理结果取模的要求$10^9 + 7$。

 ## 边缘情况

 一个关键的边缘情况是在同一位置添加和移除灯。 由于每个灯在 Fenwick 结构中都是独立跟踪的，因此移除时必须减去与添加的完全相同的转换值。 代数保证取消。 

另一种情况是查询根本没有灯的位置。 在这种情况下，两棵 Fenwick 树都包含零，因此前缀和后缀贡献的计算结果均为零，无需特殊处理即可产生正确的结果。 

最后，放置在巷子最末端的灯测试前缀和后缀分解的正确性。 位置 1 处的灯完全通过右侧公式为远处的查询做出贡献，并且转换确保不需要边界调整。
