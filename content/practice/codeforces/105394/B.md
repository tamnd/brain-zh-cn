 ## 解决方案
 ## 问题理解

 我们得到了一组长方体形状的书籍，每本书在放置之前都可以自由地进行 3D 旋转。 所有书籍必须以单水平行直立放置在书架上，这意味着每本书在书架表面上恰好贡献一个矩形足迹，并且这些足迹并排放置，不重叠。 目标是最小化这些足迹占用的总宽度。 

有一个垂直约束：书架有一个固定的净空高度H，所以在选择书籍的方向后，垂直的尺寸不得超过H。如果没有书籍的方向满足这个约束，则整个排列就变得不可能。 

因此，每本书的关键决定是选择哪个面作为底座，因为这决定了占地面积以及剩余高度是否适合 H。一旦选择了有效的方向，占地面积宽度实际上就是所选底座的面积，因为书籍排列在单行中，并且我们正在最小化所需的总书架宽度。 

输入大小高达 100,000 本书，因此任何尝试所有方向排列或任何全局组合排列的解决方案都是不可行的。 我们需要对每本书进行常量决策。 

当一本书在大多数方向上的尺寸恰好大于 H 一个尺寸但仍然具有至少一个有效旋转时，就会出现微妙的失败情况。 未正确检查所有三个方向的简单方法可能会错误地声明其不可能或选择非最佳方向。 当多个方向有效但只有一个方向最小化占地面积时，会出现另一种边缘情况。 

## 方法

 蛮力方法会考虑每本书的所有可能方向，并考虑书籍在书架上的排序过程中如何相互作用。 对于每本书，有 6 种维度排列，对于 n 本书，即使独立选择方向也可以，但任何联合优化排列顺序或组合的尝试很快就会爆炸。 如果我们尝试将其视为全局打包优化，我们最终将探索指数配置，远远超出 n 高达 10^5 的可行性。 

重要的观察是，除了通过宽度之和之外，书籍不会以任何几何方式相互作用。 一旦选择了方向，贡献就独立于所有其他方向。 这消除了项目之间的任何耦合。 局部重要的唯一约束是是否至少可以选择一侧的高度 ≤ H。一旦确定了有效方向，每本书的最佳选择就是有效方向中具有最小可能底面积的一侧。 

因此，问题就简化为，对于每本书，枚举其维度为垂直的三种选择，丢弃无效的，并在有效的中取最小底面积。 如果没有一个是有效的，那么答案是不可能的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(6^n) 或更糟，具体取决于耦合尝试 | O(n) | 太慢了|
 | 最佳 | O(n) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们独立处理每本书。 

1. 对于每本尺寸为（l、w、h）的书，考虑选择垂直尺寸的三种方法。 每个选择都定义了一个潜在的方向。

我们依次将每个尺寸视为高度，并根据 H 检查可行性。 
2. 如果我们选择 l 作为高度，则 w 和 h 形成底。 仅当 l ≤ H 时，该方向才有效，其成本为 w × h。 
3. 如果我们选择 w 作为高度，则 l 和 h 形成底。 仅当 w ≤ H 且成本为 l × h 时，这才有效。 
4. 如果我们选择 h 作为高度，则 l 和 w 构成底。 仅当 h ≤ H 且成本为 l × w 时，这才有效。 
5. 对于每本书，计算这三个候选书的最小有效底面积。 如果所有三个方向都无效，我们立即得出不可能的结论。 
6. 将所有书籍所选择的最小底面积相加，以获得所需的书架总宽度。 

关键的设计选择是每本书都被贪婪地、独立地对待。 协调书籍之间的方向没有任何好处，因为约束仅限制高度，而不限制水平排序或超出线性串联的打包。 

### 为什么它有效

 每本书都贡献了一个连续的书架宽度，等于其所选方向的占地面积。 由于书籍不会重叠并且必须放置在单行中，因此总宽度是独立选择的总和。 高度约束仅过滤可行的方向，但不会引入书籍之间的任何耦合。 因此，单独最小化每一项会产生全局最优解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, H = map(int, input().split())
total = 0

for _ in range(n):
    l, w, h = map(int, input().split())

    best = float('inf')

    if l <= H:
        best = min(best, w * h)
    if w <= H:
        best = min(best, l * h)
    if h <= H:
        best = min(best, l * w)

    if best == float('inf'):
        print("impossible")
        sys.exit(0)

    total += best

print(total)
```其实现完全遵循三向枚举。 每个分支对应于选择一个维度作为高度。 乘法计算货架表面上的足迹面积。 哨兵值`float('inf')`用于检测是否存在任何有效方向。 

一个关键细节是提前终止：一旦无法放置一本书，整个配置就会失败。 这避免了不必要的处理并保持运行时严格线性。 

## 工作示例

 我们跟踪两个小案例来了解每本书最小化是如何工作的。 

### 示例 1

 输入：```
1 3
10 2 5
```有一本书。 

| 预订 | （长、宽、高）| 我作为H？ | 洗？ | h 作为 H? | 最佳|
 | ---| ---| ---| ---| ---| ---|
 | 1 | (10,2,5) | (10,2,5) | 无效| 有效：10×5=50 | 有效：10×2=20 | 20 |

 唯一有效的方向是高度为 2 或 5 的方向，因为 10 超过 H=3。 在有效选项中，高度为 2 时底面积为 50，高度为 5 时底面积为 20。最佳值为 20。 

输出：```
20
```这表明我们必须评估所有有效的方向，而不仅仅是第一个可行的方向。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from math import inf

    n, H = map(int, input().split())
    total = 0

    for _ in range(n):
        l, w, h = map(int, input().split())
        best = float('inf')

        if l <= H:
            best = min(best, w * h)
        if w <= H:
            best = min(best, l * h)
        if h <= H:
            best = min(best, l * w)

        if best == float('inf'):
            return "impossible"

        total += best

    return str(total)

# provided samples
assert run("1 3\n10 2 5\n") == "20"
assert run("1 3\n10 4 5\n") == "impossible"

# custom cases

assert run("3 100\n1 2 3\n2 3 4\n3 4 5\n") == str(6 + 8 + 12), "all valid"
assert run("1 1000000000\n1 1 1\n") == "1", "minimum case"
assert run("2 1\n1 2 3\n1 1 1\n") == "impossible", "boundary constraint"

 ## 边缘情况

 一种关键的边缘情况是只有一个方向有效。 该算法仍然评估所有三种可能性并正确地忽略无效的可能性。 

输入：```
1 5
6 5 100
```这里，只有高度为 5 的方向才有效。 

执行：

 检查仅将 w ≤ H 的情况标记为有效，生产成本 l × h = 6 × 100 = 600。其他方向将被拒绝。 该算法返回 600。 

另一种极端情况是，多本书单独可行，但其中一本不可行。 

输入：```
2 10
3 4 5
6 7 8
```第一本书给出了有效的最小面积。 第二本书的所有维度都≤ 10，因此它也是有效的。 该算法将两者相加。 如果任何一本书的所有维度都 > 10，则该过程将立即终止，以防止部分错误的总和。
