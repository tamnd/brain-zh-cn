 ## 解决方案
 ## 问题理解

 我们得到一个整数列表，我们想要确定哪些位置在删除操作下表现“特殊”。 对于每个索引，我们暂时删除该元素并查看剩余的数组。 如果一个数组中存在一个元素与同一数组中所有其他元素的总和完全匹配，我们称该数组为好数组。 任务是找到所有索引，通过删除这些索引可以使结果数组变得更好。 

关键的微妙之处在于每次删除都是独立的。 我们实际上并没有永久删除元素，因此每次检查都是针对原始数组作为参考来执行的。 

约束很大，元素多达 200,000 个。 任何重新计算总和或扫描数组以进行每次删除的方法在最坏的情况下都会导致二次复杂度，这会太慢。 这促使我们寻求一种解决方案，可以重用预先计算的信息并在恒定或接近恒定的时间内回答每个候选索引。 

一些边缘情况很重要。 

例如，如果所有元素都相等`[1, 1, 1, 1]`，删除一个元素可能会也可能不会产生一个好的数组，具体取决于剩余结构是否允许一个元素等于其他元素的总和。 幼稚的方法可能会错误地假设对称性保证所有指数的有效性。 

如果只有一个大元素，其余的都很小，比如`[10, 1, 1, 1]`，删除不同的索引会改变总和关系是否成立，并且在删除的元素周围进行本地检查是不够的，因为条件取决于全局总和。 

最后，带有零或重复值的数组可能会欺骗假设“特殊”元素唯一性的天真的逻辑。 

## 方法

 暴力方法会尝试每个索引，重建剩余数组，计算其总和，然后扫描它以查看是否有任何元素等于剩余数组总和的一半。 这需要每个索引进行 O(n) 工作，并且由于有 n 个索引，总复杂度变为 O(n²)，当 n 达到 200,000 时，这太慢了。 

关键的观察是“存在一个元素等于其余元素之和”的条件可以用数组的总和重写。 假设剩余数组的总和为 S。如果候选元素 x 满足条件，则 x = S − x，这意味着 2x = S。因此，条件简化为检查 S 是否为偶数以及是否存在等于 S/2 的元素。 

这将问题变得简单得多。 对于每次删除，我们可以在 O(1) 中计算新的和，然后只需要检查剩余的多重集中是否存在该半和。 为了支持快速成员资格检查，我们维护所有元素的频率图。 当我们删除一个元素时，我们临时调整它的频率，计算新的和，并测试 S/2 是否存在非零频率。 

这将每次检查减少到恒定时间，从而给出整体线性解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) 或 O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 计算数组中所有元素的总和。 这为我们提供了一个基线，可以从中快速导出每次删除。 
2. 构建所有值的频率图。 这允许我们在暂时删除元素后检查候选值是否存在。 
3. 迭代每个索引 j，将 a[j] 视为要删除的元素。 
4. 计算新的总和 S' 作为total_sum − a[j]。 一个好的数组的目标条件是检查是否存在一个元素 x 使得 2x = S'。 
5. 暂时降低 a[j] 的频率，因为该元素在缩减后的数组中不可用。 
6. 如果S'为偶数，则检查S'/2是否存在于计数为正的频率图中。 如果是，则将索引 j 标记为有效。 
7. 在移动到下一个索引之前恢复 a[j] 的频率。 

这样做的原因是定义一个好的数组的条件仅取决于是否存在等于总和一半的值。 一旦删除后总和发生变化，问题就会简化为多重集中的单个成员资格查询。 

### 为什么它有效

 对于任何数组，良好等价于存在一个元素 x，并且 x 等于所有其他元素的总和。 设总和为S，那么x = S − x，所以2x = S。这意味着一个好的数组完全由S是否为偶数以及数组中是否存在S/2来表征。 由于删除后的每个候选数组只是删除了一个元素的原始数组，因此检查条件减少为重新计算 S 并测试频率结构中 S/​​2 的成员资格。 其他结构属性都不重要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

total = sum(a)

from collections import Counter
freq = Counter(a)

res = []

for i, x in enumerate(a):
    freq[x] -= 1
    new_sum = total - x

    if new_sum % 2 == 0:
        half = new_sum // 2
        if freq[half] > 0:
            res.append(i + 1)

    freq[x] += 1

print(len(res))
if res:
    print(*res)
```该解决方案首先预先计算总和，以便每次删除只需要进行减法，而不是从头开始重新计算总和。 频率图跟踪假设删除后每个值剩余的副本数量。 

在循环内部，关键步骤是暂时降低当前元素的频率。 这确保了当我们检查 S'/2 是否存在时，我们不会意外地使用被删除的元素本身。 检查后，我们恢复频率，以便后续迭代在原始状态上运行。 

一个常见的错误是忘记恢复频率，这会破坏状态并为以后的索引产生错误的答案。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))

    total = sum(a)
    from collections import Counter
    freq = Counter(a)

    res = []

    for i, x in enumerate(a):
        freq[x] -= 1
        new_sum = total - x

        if new_sum % 2 == 0:
            half = new_sum // 2
            if freq[half] > 0:
                res.append(i + 1)

        freq[x] += 1

    out = [str(len(res))]
    if res:
        out.append(" ".join(map(str, res)))
    return "\n".join(out)

# sample
assert run("5\n2 5 1 2 2\n") == "3\n1 4 5"

# all equal

# no valid
assert run("3\n1 2 4\n") == "0"

# single dominant element
assert run("4\n10 1 1 1\n") == "1\n1"

 ## 边缘情况

 一个重要的边缘情况是被删除的元素本身就是潜在的“和元素”。 在这种情况下，未能暂时将其从频率图中删除会导致错误地接受索引。 该算法在检查之前显式减少频率，以防止这种自引用错误。 

当 S' 为奇数时，会出现另一种微妙的情况。 即使数组包含很大的值，也没有整数 x 可以满足 2x = S'，因此检查必须立即拒绝而不进行任何查找。
