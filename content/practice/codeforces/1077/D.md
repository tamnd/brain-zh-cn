 ## 解决方案
 ## 问题理解

 给定一个整数多重集，我们想要构造一个固定长度的模式数组$k$。 一旦我们选择了这种模式，我们就会反复尝试从原始多重集中“提取”它：每次提取都会消耗模式中每个值的一次出现，并且在每次完全提取后，消耗的元素会从池中永久消失。 

目标是选择模式以使完整提取的数量尽可能大。 我们可以自由地排列模式并重复其中的值，但每个副本都必须完全受到原始数组中可用频率的支持。 

关键的观点是，答案仅取决于每个值在最终模式中出现的次数，因为顺序与可行性无关。 如果出现一个值$x$次，然后每次完全提取消耗$x$数组中该值的副本。 

约束允许最多$2 \cdot 10^5$元素，因此尝试所有候选模式或模拟许多选择的提取的任何解决方案都会太慢。 任何平方项$n$或依赖于枚举值的所有子集被立即排除。 

当频率极度倾斜时，就会出现微妙的边缘情况。 如果一个数字经常出现，而其他数字很少出现，则天真的贪婪选择可能会过度使用模式中的频繁数字，从而阻碍平衡模式的可能性$k$职位。 另一个棘手的情况是当$k$相对于不同元素的数量来说很大，迫使必须通过频率约束而不是直觉来仔细控制重复。 

## 方法

 暴力方法会尝试构建所有可能的大小多重集$k$从不同的价值观$s$，然后模拟可以提取多少次。 即使我们将自己限制为仅存在于$s$，可能的模式数量呈指数增长$k$，对于每个候选者，我们需要计算极限频率比，其成本$O(n)$。 这很快就变得不可行。 

中心观察是，如果我们固定一个候选模式，它可以被提取的次数纯粹由频率决定：如果一个值$v$出现$c_v$次出现在模式中，并出现$f_v$次，那么我们最多可以提取出模式$\min_v \lfloor f_v / c_v \rfloor$次。 

这提出了一个相反的观点：我们不是选择一个模式并计算其分数，而是决定每个值在所有提取的副本中应占据多少个总“槽”。 如果我们的目标是$x$副本，那么我们需要一个模式，其每个值的总需求不超过可用频率。 也就是说，每个值在所有副本中的总使用量必须最多为$f_v$，所以每个副本最多可以使用$\lfloor f_v / x \rfloor$值的出现$v$。 

这导致了一个关键的转变：对于固定数量的副本$x$，我们可以通过采取贪婪的方式构建最佳可能的模式$\lfloor f_v / x \rfloor$每个值的出现次数，直至总大小$k$。 如果我们能达到尺寸$k$， 然后$x$副本是可行的。 

现在问题变成了寻找最大值$x$。 由于可行性是单调的$x$（如果我们可以提取$x$副本，我们可以提取更少），我们可以二分搜索$x$。 对于每位候选人$x$，我们测试可行性并贪婪地构造一个模式。 

最后，一旦达到最大$x$已知，我们输出在该约束下构造的任何有效模式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 二分查找+贪心构造 |$O(n \log n)$| O(n) | 已接受 |

 ## 算法演练

 1. 统计数组中每个值出现的频率。 这给出了每个可能的模式元素的最大供应约束。 
2. 定义一个函数来检查我们是否可以构建至少支持的模式$x$完全提取。 对于每个值$v$，计算它可以在单个副本中出现多少次：$\lfloor f_v / x \rfloor$。 这是每份的最大限额，且不超过该值$x$重复。 
3. 将所有这些每个值的容量相加。 如果总数小于$k$，那么即使是最慷慨的模式也无法达到长度$k$， 所以$x$副本是不可能的。 
4. 如果可行，通过迭代值并将每个值相加来构建模式$\lfloor f_v / x \rfloor$次直到图案达到长度$k$。 
5. 使用二分查找$x$介于 1 和$n$，反复测试可行性并调整界限。 
6.找到最大可行后$x$，使用相同的贪心构造重建最终模式。 

这样做的原因是对于固定数量的副本$x$，最佳可能的模式始终是通过将每个值完全饱和到其每个副本的限制来获得的。 任何偏差都会减少总尺寸或浪费可用频率容量。 二分查找确保我们找到最大的$x$仍然允许构建尺寸的全长模式$k$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(x, freq, k):
    total = 0
    for v in freq:
        total += freq[v] // x
        if total >= k:
            return True
    return total >= k

def build(x, freq, k):
    res = []
    for v in freq:
        cnt = freq[v] // x
        take = min(cnt, k - len(res))
        res.extend([v] * take)
        if len(res) == k:
            break
    return res

def solve():
    n, k = map(int, input().split())
    arr = list(map(int, input().split()))

    freq = {}
    for v in arr:
        freq[v] = freq.get(v, 0) + 1

    lo, hi = 1, n
    best = 1

    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid, freq, k):
            best = mid
            lo = mid + 1
        else:
            hi = mid - 1

    ans = build(best, freq, k)
    print(*ans)

if __name__ == "__main__":
    solve()
```实施将可行性检查与重建分开。 这`can`函数计算是否可以在不构建实际模式的情况下实现给定数量的副本，从而保持二分搜索的效率。 这`build`然后函数使用相同的逻辑贪婪地重建模式，确保与可行性测试的一致性。 

一个微妙的细节是，重建必须遵守与中使用的完全相同的每值限制。`can`。 即使可行性测试是正确的，两者之间的任何不匹配都会导致错误的模式。 

## 工作示例

 考虑输入：```
7 3
1 2 3 2 4 3 1
```我们首先计算频率：

 | 价值| 频率 |
 | ---| ---|
 | 1 | 2 |
 | 2 | 2 |
 | 3 | 2 |
 | 4 | 1 |

 我们测试可行性。 为了$x = 2$，每个值贡献$\lfloor f_v / 2 \rfloor$:

 | 价值| 贡献 |
 | ---| ---|
 | 1 | 1 |
 | 2 | 1 |
 | 3 | 1 |
 | 4 | 0 |

 总共是3个，足够了$k = 3$，所以 2 个副本是可能的。 为了$x = 3$，除了不可能达到 3 个元素之外，贡献值全部为零，因此失败。 因此最好$x = 2$。 

重建选择 1、2、3 中的每一个，给出`[1, 2, 3]`。 

现在考虑一个倾斜的情况：```
6 4
1 1 1 1 2 2
```频率：

 | 价值| 频率 |
 | ---| ---|
 | 1 | 4 |
 | 2 | 2 |

 尝试$x = 2$: 贡献值 1 为 2，价值 2 为 1，总共 3，不足$k = 4$。 所以只有$x = 1$作品。 我们最多可以采用 4 个一和 2 个二，但只需要 4 个元素，因此有效的模式是`[1, 1, 1, 2]`或类似的。 

该跟踪表明该算法自然地平衡频繁元素和稀有元素，而无需明确推理排序。

def run(inp: str) -> str:

    arr = list(map(int, input().split()))

    freq = {}
    for v in arr:
        freq[v] = freq.get(v, 0) + 1

        mid = (lo + hi) // 2
        if can(mid):
            best = mid
            lo = mid + 1
        else:
            hi = mid - 1

    res = []
    for v in freq:
        cnt = freq[v] // best
        res.extend([v] * min(cnt, k - len(res)))
        if len(res) == k:
            break

    return " ".join(map(str, res))

# provided sample
assert run("7 3\n1 2 3 2 4 3 1\n") in {"1 2 3", "1 3 2", "2 1 3"}

# all equal
assert len(run("5 2\n1 1 1 1 1\n").split()) == 2

# k = n
assert len(run("4 4\n1 2 3 4\n").split()) == 4

 ## 边缘情况

 对于所有元素都相同的输入，例如`5 3`和`[7 7 7 7 7]`，算法计算出每个副本只能包含一个值，二分查找最多发现$\lfloor 5/3 \rfloor = 1$完全提取是可能的。 重建简单地输出任意三个 7，这符合可行性约束。 

对于以下情况$k = n$，该算法有效地减少了选择没有重复约束的完整多重集，并且二分搜索仍然返回有效的最大副本数，通常为 1。重建将整个数组填充到$n$，这基本上满足正确性。 

对于高度不平衡的输入，例如`[1,1,1,1,2]`与大$k$，可行性检查可确保稀有元素不会被过度分配，从而防止过度乐观的模式在提取过程中失败。
