 ## 解决方案
 ## 问题理解

 我们有一排公寓，每间公寓的灯要么亮着，要么关着。 配置是一个二进制数组，其中 1 表示亮，0 表示暗。 如果我所在位置的居民目前处于两个邻居都开着灯而自己的灯都关着的模式中间，他们会感到不安。 

更准确地说，当三元组 (a[i−1], a[i], a[i+1]) 等于 (1, 0, 1) 时，索引 i 处会发生扰动。 目标不是直接计算这些干扰，而是确定居民必须关灯的不同公寓的最小数量，以便阵列中的任何位置都不存在这种模式。 

操作受到限制：我们可以选择恰好 k 个不同的位置，并强制这些位置变为 0。所有其他位置保持不变。 任务是找到最小的 k 来消除模式 1,0,1 的每次出现。 

约束很小，n 最大为 100，这意味着即使是三次或二次解也足够快。 然而，问题的结构表明，最佳解决方案应该比指数子集的强力枚举简单得多。 

当多个干扰重叠时，就会出现微妙的边缘情况。 例如，在数组 1 0 1 0 1 中，关闭位置 3 处的中间零可修复以位置 2 和 4 为中心的两个重叠的潜在模式。独立处理每个干扰的简单方法可能会超出所需操作的计数。 

另一种边缘情况发生在完全没有干扰的情况下。 在这种情况下，答案为零，任何贪婪或模拟方法都必须正确保留这一点，而不会意外地引入不必要的修改。 

## 方法

 直接暴力策略将尝试关闭所有位置子集，模拟结果数组，并检查是否有任何索引 i 仍然满足模式 1,0,1。 对于每个子集，我们修改数组并扫描一次，其成本为 O(n)。 由于有 2^n 个子集，因此变为 O(n·2^n)，对于 n = 100 来说已经很大了。 

关键的观察结果是，干扰是局部的，并且以一种非常结构化的方式独立。 位置 i 处的扰动仅取决于三个连续元素。 为了消除它，我们必须确保在最终配置中 i−1、i、i+1 三个位置中至少有一个变为 0。 

然而，重要的变化是关闭中间位置 i 不仅仅可以修复一个干扰。 它可以同时消除以相邻索引为中心的重叠模式。 这表明了一种贪婪的从左到右的构造：每当我们检测到模式 1,0,1 时，我们应该立即以最有效的方式修复它。 

如果我们从左到右扫描，每当我们发现以 i 为中心的干扰时，最佳移动就是关闭位置 i+1。 这种选择至关重要，因为它解决了当前的干扰，并防止了未来因保持右侧不变而出现的重叠。 如果我们关闭 i 或 i−1，我们可能会留下一个残余结构，迫使稍后进行额外的操作。 

这种贪婪的决策是安全的，因为每个扰动都锚定在其中心，一旦我们通过索引 i，任何后续决策都不会影响之前的索引。 因此，我们只需要确保在单次线性扫描期间每个事件都被恰好处理一次。

 | 蛮力 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |

 ## 算法演练

1. 使用从 1 到 n−2 的索引 i 从左到右扫描数组。 我们避免端点，因为干扰需要三个连续的位置。 
2. 在每个位置 i，检查是否 a[i−1] = 1、a[i] = 0 和 a[i+1] = 1。这识别出以 i 为中心的受干扰配置。 
3. 如果找到这样的模式，请将答案加 1，因为我们必须使用一次操作来修复此干扰。 
4. 修复 i 处的干扰后，我们通过设置 a[i+1] = 0 来模拟关闭位置 i+1。这很重要，因为它确保以后不会再次计算涉及 i+1 的重叠模式。 
5. 继续向前扫描，不要向后退。 即使我们修改了数组，我们也不会重新访问早期的索引，因为涉及它们的任何干扰都已经解决，或者不会受到未来更改的影响。 
6. 扫描完成后，返回执行的操作总数。 

### 为什么它有效

 正确性依赖于这样一个事实：每个扰动都唯一锚定在其中心索引 i 上，通过强制 a[i+1] = 0 来解决它会破坏唯一可以维持它的配置，同时最大限度地减少对早期决策的干扰。 一旦我们移过索引 i，未来的操作就无法重新引入以 i 为中心的有效 1,0,1，因为我们只会将元素从 1 变为 0，并且永远不会恢复更改。 这保证了每次干扰最多被计算一次，并且贪婪的选择永远不会阻止更便宜的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    k = 0
    i = 1
    
    while i < n - 1:
        if a[i - 1] == 1 and a[i] == 0 and a[i + 1] == 1:
            k += 1
            a[i + 1] = 0
            i += 2
        else:
            i += 1
    
    print(k)

if __name__ == "__main__":
    solve()
```该解决方案读取数组并维护一个用于修复次数的计数器。 扫描从索引 1 开始，因为我们需要左邻居，并出于同样的原因在最后一个索引之前停止。 

当发现干扰时，我们立即通过关闭右邻居来“修复”它。 该选择通过设置进行编码`a[i+1] = 0`。 然后指针向前跳 2，以避免重新考虑该操作已经中和的重叠结构。 

其余的逻辑是简单的线性遍历，确保每个位置最多处理一次。

 ## 边缘情况

 对于像这样的输入`1 0 1 0 1`，算法在索引 1 处遇到干扰，立即翻转右邻居，并向前移动。 这可以防止同一区域被重复计算。 当它后来到达索引 3 时，更新后的数组会反映早期的修改，确保仅处理真正剩余的干扰。 

在完全均匀的数组中，例如`0 0 0 0`，扫描永远不会触发条件，因此计数器保持为零并且不会引入不必要的操作。
