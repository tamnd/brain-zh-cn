 ## 解决方案
 ## 问题理解

 青蛙从位置零开始沿数轴移动。 它的运动是严格交替的：它首先向右跳固定距离`a`，然后向左`b`，然后再次通过`a`， 等等。 每个查询独立地描述一只这样的青蛙，并询问它在准确执行后最终会到达哪里`k`跳跃。 

输入由多个独立的场景组成。 每个场景都提供两个步长和跳跃次数。 每个场景的输出是模拟这种交替运动后的最终坐标。 

关于约束的关键观察是`k`可以大到10^9，所以直接模拟每次跳跃是不可能的。 简单的模拟每个查询将花费 O(k) 时间，在最坏的情况下，每个查询需要 10^9 次操作，并且立即超出时间限制。 对于多达 1000 个查询，这是完全不可行的。 该解决方案必须在每个查询的恒定时间内计算最终位置。 

当出现微妙的边缘情况时`k = 1`。 只发生一次右跳，所以答案正是`a`。 另一个重要的情况是当`k`是偶数并且`a`等于`b`，因为所有运动都抵消了，青蛙返回到零。 如果奇偶校验处理不正确，这些取消可能会误导逐步实施。 

## 方法

 一种直接的方法是依次模拟每次跳跃。 从零开始，我们交替添加`a`并减去`b`。 这是正确的，因为它完全遵循定义。 然而，对于大`k`，这需要每个查询重复相同的两个操作最多 10^9 次。 即使采用高效的算术，这种方法也太慢了。 

运动的结构是周期性的，周期为第二。 每对跳跃都包含净运动`a - b`。 这一观察结果使我们能够将整个序列折叠成两次跳跃的块。 一旦我们将跳跃分成对，我们只需要处理是否有额外的最后跳跃`k`很奇怪。 

如果`k`是偶数，正好有`k/2`完整的对，每个人都有贡献`a - b`，所以结果是`(k/2) * (a - b)`。 如果`k`是奇数，有`(k//2)`完整的对加上一个额外的向右跳跃`a`。 

这将问题从线性模拟减少到每个查询的恒定时间算术。

 | ---| ---| ---| ---|
 | 暴力模拟| 每次查询 O(k) | O(1) | O(1) | 太慢了 |
 | 对压缩 | 每个查询 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个查询。 

1. 阅读`a`,`b`， 和`k`。 这些定义了交替步长以及要应用的移动次数。 
2. 计算有多少个满`(right, left)`使用整数除法存在对`k // 2`。 每对贡献的净位移为`a - b`因为青蛙首先向右移动`a`然后离开`b`。 
3. 将完整对的数量乘以`(a - b)`累积所有完整周期的贡献。 这会捕获除最后一次之外的所有跳转`k`很奇怪。 
4.如果`k`为奇数，向右添加一个额外的跳跃尺寸`a`，因为序列总是以右跳开始，奇数长度序列以右移动结束。 
5. 输出计算出的位置。 

### 为什么它有效

 运动序列分解为独立的两步块。 每个块总是从相同的结构状态开始：右跳，然后左跳。 因为一行上的加法是关联的，所以最终位置仅取决于这些块的总和以及当`k`很奇怪。 中间位置不会影响未来的步长或方向，因此折叠成对可以准确保留总位移。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    a, b, k = map(int, input().split())

    pairs = k // 2
    res = pairs * (a - b)

    if k % 2 == 1:
        res += a

    print(res)
```该实现直接反映了算法中导出的分解。 整数除法隔离完整的两步循环，奇偶校验处理最终不匹配的右跳转。 该算术使用 Python 整数，因此即使值达到 10^18 规模，也不会出现溢出问题。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        a, b, k = map(int, input().split())
        pairs = k // 2
        res = pairs * (a - b)
        if k % 2 == 1:
            res += a
        out.append(str(res))
    return "\n".join(out)

 ## 边缘情况

 当`k = 1`，算法正确计算`k//2 = 0`所以没有添加对贡献，最后的奇数步添加`a`。 这与定义相符，因为只发生一次右跳转。 

什么时候`k = 2`，该算法准确地产生`a - b`，因为有一对完整的且没有剩余的跳跃。 这直接匹配运动`0 → a → a - b`。 

什么时候`a = b`，每对贡献为零，因此只有奇数剩余跳跃才能产生非零结果。 对于甚至`k`，结果始终为零，并且对于奇数`k`，正是`a`。
