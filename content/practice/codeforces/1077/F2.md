 ## 解决方案
 ## 问题理解

 我们得到一系列排列成行的图片，每张图片都有一个美感值。 我们需要准确选择`x`这些图片的重新发布，但选择受到覆盖规则的限制：至少任何连续的长度块`k`必须包含至少一张选定的图片。 

此约束强制所选索引在数组中足够频繁。 如果你想象沿着阵列行走，你永远不会被允许走`k`连续位置而不选择至少一个元素。 同时，我们想要准确地选择`x`元素并最大化它们的美丽值的总和。 

关键的张力在于间距和价值之间。 选择高价值元素固然很好，但跳过太多会产生违反覆盖规则的间隙。 因此，解决方案从根本上讲是平衡“到目前为止我们挑选了多少”和“最后一次挑选发生在哪里”。 

限制条件`n ≤ 5000`和`x ≤ n`排除指数子集枚举。 立方或更差的解决方案`(position, picks, last pick)`可能仍然勉强以优化形式通过，但是尝试所有子集的任何事情都是不可能的。 该结构强烈建议对前缀进行动态编程，并附加一个跟踪已进行了多少次选择的状态。 

出现微妙的边缘情况时`k = 1`。 在这种情况下，长度为 1 的每个段都必须包含一个选定的元素，从而强制选择所有元素。 任何假设可以自由跳过的解决方案都会立即失败。 另一种边缘情况发生在`x`相对于`n / k`，使得无法以所需的密度覆盖阵列。 例如，如果`n = 10, k = 3`，我们至少需要 4 个选择来覆盖数组； 如果`x < 4`，即使我们选择最优，答案也是不可能的。 

## 方法

 暴力方法会尝试选择大小的所有子集`x`并检查间距约束是否成立。 可以通过扫描所选指数并验证连续所选位置之间的差距没有超过来测试有效性`k - 1`。 这已经花费了`O(n choose x)`子集，甚至检查每个子集也需要`O(n)`，使得除了微小的输入之外完全不可行。 

约束的结构提出了不同的观点：我们不是全局地考虑子集，而是局部地考虑最后选择的位置。 如果我们知道最后选择的索引，我们就可以确定在不违反规则的情况下允许我们移动多远。 这自然会导致我们从左到右处理数组并维护到目前为止已选择的元素数量的 DP。 

关键的观察是在每个位置`i`，如果我们决定选择它，我们只需要保证之前的选择不超过`k`位置远离。 如果更远，则状态无效。 这将问题简化为受限选择动态规划，其中转换取决于有界间隙，并且我们可以通过仅向后扫描直到`k`职位。 

我们定义`dp[i][j]`作为使用第一个的最大总和`i`元素并精确选择`j`元素，最后选择的元素位于位置`i`。 覆盖约束确保从先前状态的转换仅来自以下位置`[i-k, i-1]`，否则我们会产生至少`k`没有选择。 

这将问题简化为具有滑动窗口优化的结构化 DP`k`职位。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力子集 | O(nC x · n) | O(nC x · n) | O(n) | 太慢了 |
 | 带有窗口优化的 DP | O(n·x·k) | O(n·x·k) | O(n·x) | O(n·x) | 已接受 |

 ## 算法演练

 我们构建了一个动态编程表，其中每个状态跟踪给定数量的选定图片的最佳可实现总和并在特定位置结束。 

1.初始化一个DP数组`dp[i][j]`表示第j张所选图片在位置i时的最大美感和。 除了单选之外，所有状态一开始都是无效的。 
2. 对于每个位置`i`，我们允许通过仅选择该元素作为第一个选择来开始一个新序列，因此`dp[i][1] = a[i]`。 这为 DP 提供了有效的单元素子序列的种子。 
3. 对于每个位置`i`并数数`j > 1`，我们尝试将第 j 个选定元素放置在`i`。 为了维持约束，先前选择的元素必须位于`[i-k, i-1]`。 
4.我们计算`dp[i][j]`通过在所有有效的先前位置中取最佳值`p`在`[i-k, i-1]`使用`dp[p][j-1] + a[i]`。 这确保了计数的正确性和间距的有效性。 
5. 避免重新计算上一次的最大值`k`对于每个状态的位置，我们为每个状态维护一个滑动最大结构`j-1`层，随着我们的移动而更新它`i`。 
6.填完DP后，答案就是所有的最大值`dp[i][x]`对于所有有效的结束位置`i`。 
7. 如果没有说明`x`选择有效，我们输出`-1`。 

滑动窗口优化使这变得可行：没有它，每个转换都会扫描`k`位置，导致全面`O(n^2 x)`方法。 

### 为什么它有效

 在任何时候，问题施加的唯一限制是连续选择的元素之间允许的最大间隙。 此属性仅取决于最后选择的索引，而不取决于完整的历史记录。 因此，每个有效选择都可以唯一地分解为连续选择位置之间的转换，每个位置都被限制在距离内`k`。 DP 准确地捕获这些转换，并且每个有效序列都对应于通过该状态图的一条路径。 由于所有转换都保留可行性并考虑所有有效的前驱，因此 DP 状态中的最大值必定是最优解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, x = map(int, input().split())
    a = list(map(int, input().split()))

    INF = -10**18

    # dp[j][i]: best sum using j picks, last pick at i
    dp_prev = [INF] * n
    dp_cur = [INF] * n

    # initialize: 1 pick
    for i in range(n):
        dp_prev[i] = a[i]

    if x == 1:
        print(max(dp_prev))
        return

    for j in range(2, x + 1):
        dp_cur = [INF] * n
        best = INF

        for i in range(n):
            # maintain window [i-k, i-1]
            if i - 1 >= 0:
                best = max(best, dp_prev[i - 1])

            if i - k - 1 >= 0:
                best = max(best, dp_prev[i - k - 1])

            if best != INF:
                dp_cur[i] = best + a[i]

        dp_prev = dp_cur

    ans = max(dp_prev)
    print(ans if ans > INF else -1)

if __name__ == "__main__":
    solve()
```该实现将 DP 压缩为两层：先前的选择数和当前的选择数。 每次迭代都会构建在每个位置放置下一个选定元素的最佳值。 变量`best`充当过去的滑动最大值`k`合格的职位，确保我们只考虑有效的前任。 

一个常见的陷阱是忘记窗户是`[i-k, i-1]`， 不是`[i-k+1, i]`。 另一个微妙的问题是处理不可能的状态：我们传播`-inf`因此无效的转换永远不会通过加法意外地变为有效。

 ### 示例 1

```我们通过挑选次数来跟踪 DP 层。 

| j（选秀）| 我=0 | 我=1 | 我=2 | 我=3 | 我=4 |
 | ---| ---| ---| ---| ---| ---|
 | 1 | 5 | 1 | 3 | 10 | 10 1 |

 | 3 | - | - | 9 | 18 | 18 19 | 19

 最终答案是 18，通过根据转换选择索引 (0, 2, 3) 或 (0, 2, 3) 来实现，但要注意间隙永远不会超过 1。 

该轨迹显示了 DP 如何在保持局部约束的同时逐渐构建更长的有效序列。

 | ---| ---| ---| ---| ---|
 | 1 | 1 | 100 | 100 1 | 100 | 100
 | 2 | - | 101 | 101 200 | 200 201 | 201

 最佳答案是 201，选择位置 1 和 3。 

这演示了约束如何强制展开选择，同时仍然允许对高值的贪婪偏好。

 ## 边缘情况

 当`k = 1`，每个位置都必须被选择，因为每个单元素段都必须包含一个选定的元素。 该算法简化为对所有值求和并精确选择`n`如果没有明确处理，任何 DP 跳过转换都会违反正确性。 

什么时候`x`太小，无法满足覆盖范围，例如`n = 10, k = 4, x = 2`，任何排列都无法避免出现由 4 个连续未选择元素组成的块。 DP 自然地反映了这一点，因为所有状态都变得无效，不留任何可达状态`dp[n][x]`。 

当值高度倾斜时，例如中间有一个非常大的元素，DP 必须正确平衡采用它与失去未来的可行性。 窗口约束确保即使我们采用最佳局部值，我们仍然可以通过状态转换强制间隔来保持全局有效性。
