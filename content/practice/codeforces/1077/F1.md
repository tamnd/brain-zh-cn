 ## 解决方案
 ## 问题理解

 我们得到了一系列图片，每张图片都有一个代表其美丽的值。 任务是精确选择固定数量的图片，并且在所有这些选择中我们希望获得最大的整体美感。 然而，这些选定的图片如何沿线分布存在结构性限制。 

该约束表示，如果您查看任何长度至少为连续的块`k`，该块必须包含至少一张选定的图片。 换句话说，你永远不被允许有一段时间`k`连续未选择的位置。 这迫使所选位置相当分散，但仍然可以灵活地分配剩余的选择。 

输出要么是所选美女的最佳可实现总和，要么是不存在有效选择的声明。 

限制条件`n ≤ 200`立即建议三次甚至高二次动态规划解决方案是可以接受的。 任何子集上的指数方法都是不可能的，因为选择`x`元素已经给出$\binom{200}{100}$-规模组合。 这是一个经典信号，表明问题是基于前缀或位置的结构化 DP。 

“覆盖范围”限制产生了一个微妙的问题。 总是选择最好的的天真贪婪的想法`x`元素会失败，因为高值元素可能会聚集，从而在没有选定位置的情况下留下很长的间隙。 例如，如果较大的值集中在开头和结尾，但长度的中间段`k`为空，即使总和很大，解也无效。 

另一个失败案例来自于不尊重准确的计数`x`。 即使配置满足间距约束，它也是无效的，除非完全符合`x`元素被选择。 这将问题变成了对计数和结构的约束优化。 

## 方法

 蛮力方法会尝试所有方法来准确选择`x`索引并验证间距约束。 每个子集检查都需要扫描数组以确保没有运行`k`存在连续未选择的位置，如果有效则求和值。 子集的数量是$\binom{n}{x}$，即使对于中等程度的人来说，这也会变得巨大`n`，以及每次验证成本`O(n)`，使得这完全不可行。 

关键的结构观察是约束是局部的，并且仅取决于所选位置的距离有多远。 如果我们从左到右处理数组，那么在任何时候唯一重要的是我们已经选择了多少元素以及在不强制选择的情况下允许我们走多远。 

这自然会导致动态规划公式，我们跟踪位置、我们进行了多少次选择以及自上次选择元素以来已经过去了多长时间。 尺寸无间隙的约束`k`可以通过确保每当我们前进超过`k-1`步骤而不选择，我们被迫进入已经包含选择的状态。 

我们通过前缀定义 DP 状态：对于每个位置、每个选定元素的数量以及每个允许的间隙长度，我们计算最佳可能的总和。 转换要么选择当前元素，要么跳过它，同时尊重跳过不能超过`k-1`连续位置而不强制选择。 

这将全局间距约束转换为有界局部状态机，这就是解决方案变得易于处理的原因。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(\binom{n}{x} \cdot n)$|$O(n)$| 太慢了 |
 | DP 相对于位置、计数和差距 |$O(n \cdot x \cdot k)$|$O(x \cdot k)$| 已接受 |

 ## 算法演练

 我们构建一个 DP，在其中跟踪我们已经选择了多少个元素以及距离最后一个选择的元素有多远。 目标是确保我们永远不会违反长度段的规则`k`不能完全取消选择。 

1. 初始化一个DP表，其中`dp[i][j]`表示处理某个前缀后的最大美感总和，选择`j`元素，并具有由状态转换隐式强制执行的当前后缀约束。 

关键思想是，我们不是明确跟踪最后选择的索引，而是跟踪仍然允许多少次连续跳过。 
2. 各位置`i`，我们考虑两个选择：要么我们选择`a[i]`或者我们跳过它。 

选择会重置跳过约束，因为最后选择的位置已更新。 
3.如果我们选择`a[i]`，我们从一个状态移动到`j-1`选择的元素到一个状态`j`选择的元素和增益`a[i]`在价值上。 
4. 如果我们跳过`a[i]`，只有在我们还没有积累的情况下我们才允许这样做`k-1`连续跳过。 这确保我们永远不会创建长度为禁止的块`k`没有选择。 
5. 我们在保持最佳值的同时向前传播转换，仔细确保永远不会创建违反约束的状态。 
6. 最终答案是各州之间的最佳值`x`处理所有位置后已选择元素。 

### 为什么它有效

 每个状态都编码一个有效的部分结构，其中没有创建禁止间隙。 跳跃约束保证任意两个选择的位置之间的距离最多为`k-1`，因此每个长度段`k`必须包含至少一个选定的索引。 由于DP仅扩展有效状态并且从不引入非法间隙，因此不会产生无效配置。 同时，每个有效配置都对应于通过这些状态的一条路径，从而确保完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, x = map(int, input().split())
    a = list(map(int, input().split()))

    INF = -10**18

    # dp[i][j][t]:
    # i - position
    # j - number chosen
    # t - how many consecutive skips since last pick (0..k-1)
    dp = [[[INF] * k for _ in range(x + 1)] for _ in range(n + 1)]

    dp[0][0][0] = 0

    for i in range(n):
        for j in range(x + 1):
            for t in range(k):
                if dp[i][j][t] == INF:
                    continue

                cur = dp[i][j][t]

                # skip
                if t + 1 < k:
                    dp[i + 1][j][t + 1] = max(dp[i + 1][j][t + 1], cur)

                # take
                if j < x:
                    dp[i + 1][j + 1][0] = max(dp[i + 1][j + 1][0], cur + a[i])

    ans = max(dp[n][x])

    print(-1 if ans < 0 else ans)

if __name__ == "__main__":
    solve()
```DP 使用三个维度，因为约束本质上与计数和间距有关。 第三个维度至关重要，因为如果不跟踪连续的跳跃，我们将允许违反“每个 k 长度段包含一个选择”规则的无效序列。 

初始化仅将空前缀设置为有效。 每个转换要么消耗一个值，要么扩展一个间隙，无效的间隙扩展会被显式阻止`t + 1 < k`健康）状况。 

最终答案检查所有可能的终端间隙状态，因为约束仅在确保不形成非法间隙时起作用，而不是在最后一次选择之后发生的情况。

 ## 边缘情况

 关键的边缘情况发生在以下情况：`k = 1`。 在这种情况下，必须选择每个位置，并且任何尝试跳过的 DP 状态都必须立即无效。 转换规则自然会阻止所有跳过，因为`t + 1 < 1`从来都不是真的，迫使不断的选择。 

另一个微妙的情况出现时`x`相对于`n`。 DP 仍然必须正确处理由于间距限制而无法选择太少元素的情况。 该算法自然会以以下状态结束：`j < x`没有有效的完成，产生`-1`。 

第三种情况是当高值元素聚集得比`k`，迫使 DP 在每个窗口中最多选择一个。 跳跃状态维度确保一旦选择了一个元素，下一个元素就会被选中。`k-1`位置仍然可以安全地导航，但任何违反间距的尝试都会在破坏状态空间之前被拒绝。
