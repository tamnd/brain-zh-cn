 ## 解决方案
 ## 问题理解

 我们有一个有两个数量的系统：机器人和能量棒。 最初有`n`机器人和`m`能量棒。 当能量棒的数量可以除以机器人的数量时，配置被认为是公平的，这意味着能量棒可以均匀地分配给所有剩余的机器人而没有剩余。 

我们被允许执行操作，每个操作花费一个单位：

 我们可以创建一个额外的能量条，或者摧毁一个机器人。 然而，消灭机器人有一个严格的限制：我们不允许消灭所有机器人，因此至少必须保留一个机器人。 

任务是确定达到条形数量可被机器人数量整除的状态所需的最小操作数量。 

这些约束允许最多多个测试用例，其中`n`和`m`大至 10^8。 这立即排除了任何逐步模拟操作的方法。 在最坏的情况下，即使对可能的机器人计数进行 O(n) 扫描也是不可能的，因为两个参数都很大并且每个测试用例都是独立的，因此解决方案必须依赖于关于可分性的直接数学观察而不是增量搜索。 

当不执行机器人删除时，会出现微妙的边缘情况。 例如，如果`m`已经可以被整除`n`，答案为零。 当我们考虑减少机器人时，会出现另一种边缘情况：由于我们不允许删除所有机器人，因此最小可能的机器人数量是一个，并且该端点通常决定最坏情况的后备成本。 最后，一个天真的贪婪想法，例如独立尝试通过调整来修复整除性`m`单独使用会失败，因为约简机器人会改变除数本身，这会极大地改变所需的余数结构。 

## 方法

 蛮力的想法很简单：尝试所有可能数量的机器人`k`从`n`下降到`1`，计算制作所需的最小条数`m`可除以`k`，并将其与删除成本相结合`n-k`机器人。 对于每个固定`k`，所需的条数为`(k - m % k) % k`。 对这些进行总结就得出了候选答案。 这是正确的，因为它探索了机器人删除下的所有可到达状态。 

然而，这种方法太慢了，因为它迭代了`n`每个测试用例都有可能的值，在最坏的情况下，每个测试大约需要 10^8 次操作，而对于多达 1000 个测试用例，这变得不可行。 

关键的见解是我们不需要考虑所有的价值`k`。 相反，我们观察到最优解决方案是由以下关系决定的：`m`和`k`通过部门结构。 成本函数仅取决于如何`m`表现为模数`k`，并且作为`k`减少，剩余模式以结构化方式重复。 这使我们能够处理范围`k`具有相同商数的`m / k`总的来说，将搜索空间减少到大约 O(sqrt(m)) 候选转换。 

这个想法是翻转观点：我们不是迭代所有机器人计数，而是迭代商的可能值`q = m / k`。 对于固定商，所有`k`位于一个连续的区间，其中`m % k`变化可预见。 这将问题转化为可以有效评估的少量区间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有k | 每次测试 O(n) | O(1) | O(1) | 太慢了 |
 | 商/区间优化| 每次测试 O(sqrt(m)) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从保持所有机器人不变的基线答案开始。 计算制作所需的成本`m`可除以`n`，即`(n - m % n) % n`。 这代表只增加杆数而不改变机器人。 
2. 考虑逐个减少机器人，但不是线性迭代，而是根据值进行推理`k`我们保留的。 固定费用`k`是`(n - k)`用于机器人删除加上`(k - m % k) % k`用于添加酒吧。 
3.观察评估每一个`k`是不必要的，因为`m // k`仅在特定阈值下发生变化。 这些阈值划分了范围`[1, n]`分成段，其中`m // k`保持不变。 
4. 对于每个可能的商值`q = m // k`，确定区间`k`该商保持的值。 该间隔可以使用整数除法界限计算：`k ∈ [m // (q + 1) + 1, m // q]`。 
5. 对于每个这样的区间，仅计算边界点，因为在固定商区间内，表达式`(k - m % k)`表现单调，在端点处具有可预测的极值。 
6. 结合机器人删除和钢筋插入的成本，并跟踪所有评估的候选者的最小值。 
7. 最后，将所有候选者与不删除任何机器人的基线情况进行比较，并输出最小成本。 

### 为什么它有效

 正确性取决于成本函数分解为线性项`k`（机器人删除）和仅取决于的模块化校正项`m % k`。 整数除法的结构确保`m % k`仅在以下位置发生变化`m / k`变化，最多发生 O(sqrt(m)) 次。 在每个常商区间内，函数除了端点外没有隐藏的局部最小值，因此检查边界值就足够了。 这保证了不会跳过最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())

        # case 1: don't delete any robot
        ans = (n - m % n) % n

        # try reducing robots
        k = 1
        while k <= n:
            if m // k == 0:
                # for all k > m, m % k == m
                # cost is (n - k) + (k - m)
                ans = min(ans, n - k + max(0, k - m))

                k += 1
                continue

            q = m // k
            r = m // q

            # evaluate interval [k, r]
            # endpoints are sufficient
            for cand in (k, min(n, r)):
                if cand >= 1:
                    cost = (n - cand) + (cand - m % cand) % cand
                    ans = min(ans, cost)

            k = r + 1

        print(ans)

if __name__ == "__main__":
    solve()
```该实现从无删除基线开始，然后以常数值定义的分组间隔扫描机器人计数`m // k`。 而不是迭代每个`k`，它使用计算出的在段边界之间跳转`r`。 这确保循环以大约 O(sqrt(m)) 步运行。 

模块化校正`(cand - m % cand) % cand`干净地处理环绕时`m`已经可以被整除`cand`。 最终答案是所有配置中的最小值。

 ## 边缘情况

 一种边缘情况是当`n = 1`。 由于我们无法删除最后一个机器人，唯一的选择是调整栏，答案就变成了`0`因为任何`m`可以整除`1`。 该算法正确地处理了这个问题，因为`(1 - m % 1) % 1`始终为零。 

另一个边缘情况是当`m = 0`。 然后成本就会降低到可能删除机器人或什么也不做。 该公式正确地评估了所有候选者，并且自然地更喜欢零运算，因为整除性适用于任何`k`除零。 

第三种边缘情况是当`m < n`，其中减少机器人可能会突然变得比增加酒吧更有利。 基于间隔的搜索确保这些案例通过小型测试`k`值明确，因此不会错过最佳值。 

最后一个边缘情况是`n`很大并且`m`很小，其中大多数商区间塌陷为`m // k = 0`政权。 该区域的特殊处理确保了正确性，而不会错过以下位置的转换点：`k = m`。
