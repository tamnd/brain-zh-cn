 ## 解决方案
 ## 问题理解

 两支由五名玩家组成的队伍进行回合制淘汰游戏。 每个玩家一开始都有一定的生命值。 在每次移动中，一支球队都会选择对方球队中的一名球员，并将该球员的生命值减少恰好一个。 一旦玩家的生命值达到零，该玩家就会被视为退出游戏，而一旦所有五名玩家都被淘汰，一支球队就会失败。 

蓝队总是先行一步，两队都在充分了解情况的情况下发挥最佳水平。 任务是确定哪一方将是最后能够让至少一名玩家存活的一方。 

输入由两个大小为 5 的固定长度数组组成，代表蓝色和红色玩家的初始生命值。 输出是一个表示获胜团队的单词。 

关键的结构观察是，游戏不依赖于玩家的个人身份，除了消除他们所需的剩余“工作量”之外。 每个玩家只是一个必须通过攻击耗尽的生命值池。 

约束允许每个健康值最多约为 4×10^8。 这就排除了任何逐一处理每次攻击的模拟，因为简单模拟中的移动总数可能达到 10^9 或更多。 即使每次攻击的对数或每次点击模拟也太慢。 任何正确的解决方案都必须将问题简化为对聚合量的常数或对数推理。 

当一支球队的生命值分布非常不均匀时，就会出现微妙的边缘情况。 例如，蓝色可能有值`[1, 1, 1, 1, 100000000]`而红色是`[5, 5, 5, 5, 5]`。 如果忽略回合顺序效果，像“总是击中剩余最大敌人生命值”这样天真的贪婪直觉可能会产生误导。 另一种失败的情况是假设仅以总生命值来决定胜负，而不考虑步数的平价。 

## 方法

 暴力策略会逐步模拟游戏。 在每个回合中，它都会选择一个活着的对手玩家并减少他们的生命值。 当玩家达到零时，他们就会被移除。 这是正确的，因为它完全遵循规则，但其成本与两个团队的所有健康值的总和成正比。 在最坏的情况下，这大约是 2×10^9 次操作，这远远超出了时间限制。 

关键的见解是，每个团队都在有效地尝试“在总伤害方面生存更长时间”，但他们轮流进行，每次攻击都会减少对方总生命值的一个单位。 因此，游戏简化为一场简单的竞赛，即每队在被彻底消灭之前可以吸收多少总命中。 

每支球队的总耐久度等于其球员生命值的总和。 由于蓝色先行动，因此在两队同时耗尽的任何情况下，蓝色都会获得一次额外的攻击。 这就产生了奇偶性转变：结果不仅取决于哪一个总和更大，还取决于差异是否足以补偿先手优势。 

这将整个问题简化为在基于回合的交替模型下比较两个整数，而不是跟踪五个单独的实体。

 ## 算法演练

 1. 计算蓝队和红队的总生命值。 这些代表了每支球队可以生存的总单次打击减少次数。 
2. 观察到蓝色总是先行动，因此蓝色在整个游戏过程中有效地获得了一个额外的行动。 这意味着如果两队的总生命值相等，蓝队将首先完成红队。 
3. 比较两个总数。 如果蓝色的总生命值严格大于红色的总生命值减去回合顺序的优势，则蓝色的生存时间更长。 否则，红就会这么做。 
4. 将其转化为直接比较条件并输出相应的获胜者。 

将问题减少到总数背后的原因来自于这样一个事实：每一步都会减少一个单位的对手耐久度，并且任何一步都不能被重定向或浪费。 唯一重要的结构是双方在交替移动下同时达到零之前可以维持多少总减少量。 

### 为什么它有效

 在游戏中的任何时刻，状态都可以用两个数字来描述：蓝色的剩余总生命值和红色的剩余总生命值。 每回合严格将这两个总数中的一个严格减一，在玩家之间轮流进行。 因为没有任何策略选择可以改变每个动作恰好消耗一个单位的相对耐久度的事实，所以唯一相关的问题是在交替减法下哪个总数首先达到零。 先手优势将有效比较转移到对蓝方有利的一方，这完全决定了结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    B = list(map(int, input().split()))
    R = list(map(int, input().split()))
    
    sumB = sum(B)
    sumR = sum(R)
    
    # Blue starts first, so Blue effectively gets the last move
    if sumB > sumR:
        print("Blue")
    else:
        print("Red")

if __name__ == "__main__":
    solve()
```该实施将每个团队分解为一个聚合值。 唯一重要的部分是正确认识到每个玩家的目标决策并不重要，因为每次命中对获胜条件的影响都是相同的。 

一个常见的错误是试图模拟单个玩家的最佳目标。 这在不改变结果的情况下增加了复杂性，因为将伤害分散到五个目标不会影响消灭一个团队所需的总命中数。

 ## 边缘情况

 一种极端情况是两队的总生命值相同。 在这种情况下，蓝色获胜，因为它总是先行动，并且可以在红色最后可能的响应之前完成最终的减少。 例如，对于两支球队`[2,2,2,2,2]`，交替序列确保 Blue 执行最终成功的还原。 

另一种边缘情况是分配极度不平衡，例如一名玩家几乎持有所有生命值。 该算法仍然有效，因为它只取决于总和； 该分布不会影响总体需要减少多少单位。
