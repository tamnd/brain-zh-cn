 ## 解决方案
 ## 问题理解

 我们得到一排从 1 到 n 标记的树，我们总是从 1 棵树开始，到 n 棵树结束。 有效的“树路径”是通过选择一系列已访问的树（包括两个端点）来定义的，其中每个下一个移动向前跳跃至少 k 个位置。 序列的长度固定为 m，因此我们有效地选择 m 个索引 a1 到 am，使得 a1 等于 1，am 等于 n，序列严格递增，并且每一步都满足 ai+1 − ai ≥ k。 

如果两条路径选择的访问索引集不同，则认为两条路径不同。 任务不是计算此类路径的确切数量，而只是确定该数量是奇数还是偶数。 

The constraints are extremely large: n and m can be up to 10^9 and the number of test cases can reach 10^5. 这立即排除了任何迭代位置或使用 n 或 m 动态规划的解决方案。 任何即使每个测试用例以 m 为线性的方法也会太慢。 该解决方案必须将每个测试用例减少到恒定或对数时间，最好使用封闭形式的组合表征。 

一个微妙的边缘条件是当 m 接近 2 或 k 很大时。 如果 m 等于 2，则在距离约束允许的情况下，路径被迫仅为 {1, n}，因此至多有一个解。 另一个棘手的情况是当 k 等于 1 时，它消除了跳转限制，并将问题转变为选择包含端点的从 1 到 n 的任何长度为 m 的递增序列。 在这两种情况下，如果直接计算中间值，未经仔细转换的简单计数会导致不正确的组合解释或溢出问题。 

## 方法

 强力解释将尝试构造长度为 m 从 1 到 n 且间隔至少为 k 的所有有效序列并对它们进行计数。 这在概念上很简单：我们递归地从当前位置到任何后续有效位置选择下一个位置。 然而，分支因子可以很大并且深度为m，因此状态数与n组合增长。 即使对于中等值，这也会以 m 为指数，并且在约束下是不可能的。 

关键的观察结果是间距约束可以被标准化。 每次跳跃至少需要 k 距离，因此我们可以通过减去强制间距来“压缩”序列。 一旦应用此变换，每个有效序列都精确对应于从缩小的范围中选择 m−2 个中间点，除了排序之外没有任何限制。 这将问题转换为二项式系数计数问题。 

一旦简化为二项式系数，我们就不再对其进行数值计算。 我们只需要它的平价。 二项式系数的奇偶性具有众所周知的结构：它仅取决于二进制表示，并且可以使用从卢卡斯定理模 2 导出的按位条件进行检查。这完全避免了阶乘、模逆或大型算术。 

| 方法| 时间复杂度| 空间复杂度| 判决|
 |---|---|---|---|
 | 暴力枚举| 指数| O(m) 递归 | 太慢了 |
 | 压缩+二项式奇偶校验| 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从约束序列 a1 = 1 和 am = n 开始，其中每个相邻差值至少为 k。 由于端点是固定的，因此我们关注序列的内部结构。 

2. 通过删除强制间距来变换每个位置。 定义一个新序列 xi = ai − (i−1)(k−1)。 这消除了每个步骤中 k−1 的“内置松弛”，同时保留了顺序。 

3. 变换后，约束变为 xi+1 − xi ≥ 1，这意味着 xi 构成严格递增的整数序列。

4. 计算有效终点。 由于 am = n，我们得到 xm = n − (m−1)(k−1)。 这将问题简化为选择从 1 开始并在此压缩端点结束的 m 个不同整数。 

5. 修正 x1 = 1 和 xm 的计算结果。 剩余的 m−2 值是区间 (1, xm) 内任意严格递增的整数。 有效选择的数量等于从 xm−2 个可用位置中选择 m−2 个元素的方法数。 

6. 这给出了二项式系数 C(N, K)，其中 N = xm − 2 且 K = m − 2。 

7. 我们只需要C(N,K)的奇偶性。 使用二进制条件：当且仅当 K 和 N−K 加法时不发生二进制进位时，C(N, K) 才是奇数，相当于按位条件 K & (N − K) = 0。 

### 为什么它有效

 该变换通过将强制距离吸收为线性平移，将约束间距问题转换为纯组合问题。 这保留了原始路径和变换后的整数序列之间的一一对应关系。 一旦减少，计数就成为标准子集选择，并且奇偶校验仅取决于二进制结构，因为模 2 二项式系数的行为类似于按位包含关系。 正确性取决于原始路径和压缩后无约束整数选择之间的双射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_odd_binom(n, k):
    if k < 0 or k > n:
        return False
    return (k & (n - k)) == 0

t = int(input())
out = []

for _ in range(t):
    n, m, k = map(int, input().split())

    if m == 1:
        out.append("yes")
        continue

    # transformed endpoint
    end = n - (m - 1) * (k - 1)
    N = end - 2
    K = m - 2

    if N < K or K < 0:
        out.append("no")
        continue

    out.append("yes" if is_odd_binom(N, K) else "no")

sys.stdout.write("\n".join(out))
```该解决方案首先将几何跳跃约束简化为线性平移，然后将计数问题转换为二项式系数。 最后一步避免直接计算系数，而是使用按位奇偶校验测试。 一个常见的实现陷阱是在转换为 C(N, K) 时忘记最终移位 2，这来自固定端点 1 和 n。 另一个微妙的问题是处理转换后的端点变得太小的情况，这必须立即产生零有效序列。 

## 工作示例

 考虑一个输入，其中 n = 10, m = 3, k = 2。转换后，端点变为 end = 10 − 2·1 = 8。然后 N = 6，K = 1。我们正在计算 C(6, 1)，它是 6，一个偶数，所以答案是否定的。 

| 步骤| 结束 | 尼 | 克 | 奇偶校验|
 |---|---|---|---|---|
 | 变换| 8 | 6 | 1 | 计算 C(6,1) |

 这演示了间距约束如何减少可用范围以及最终奇偶校验如何仅取决于二项式结构。 

现在考虑 n = 15，m = 5，k = 3。我们得到 end = 15 − 4·2 = 7，因此 N = 5，K = 3。C(5,3) 等于 10，这是偶数，所以答案仍然是“否”。 

| 步骤| 结束 | 尼 | 克 | 奇偶校验|
 |---|---|---|---|---|
 | 变换| 7 | 5 | 3 | 计算 C(5,3) |

 这种情况显示了一种情况，变换后的空间变得非常小，并且仅存在几种组合，但奇偶校验仍然纯粹由二进制结构而不是大小决定。

 ## 边缘情况

 一种重要的边缘情况是 m 等于 2 时。在这种情况下，没有内部节点，因此唯一的候选路径是从 1 到 n 的直接连接。 该算法设置 K = 0 和 N = end − 2，并且二项式奇偶校验在可行时正确地将 C(N, 0) 评估为始终为 1，这与只有一个结构这一事实相匹配。 

当 k 为 1 时，会出现另一种边缘情况。变换消除了零间距，因此 end 变为 n，问题简化为选择包含端点的任何长度为 m 的递增序列。 该公式仍然产生正确的二项式形式，并且奇偶校验纯粹以组合方式运行，无需特殊的大小写。 

当压缩端点端变得小于 m 时，就会出现最后的边缘情况。 在这种情况下，不可能放置足够多的不同点，并且算法通过在应用奇偶校验规则之前检查 N < K 来正确拒绝它。
