 ## 解决方案
 ## 问题理解

 我们得到一排从左到右索引的奶牛。 每头牛都有一个重量。 然后我们会得到许多独立的“突袭”。 突袭由两个数字定义：起始位置和步长。 从给定的索引开始，我们以固定的步长向前跳跃来重复获取奶牛，因此我们收集索引 a、a + b、a + 2b 等，当超出最后一头奶牛时停止。 

对于每个查询，我们必须计算该算术级数访问过的所有奶牛的总重量。 

重要的细节是每个查询都是独立的。 我们不会删除奶牛，并且查询之间没有任何变化，因此我们在静态数组上的算术级数上重复求和值。 

约束很大：最多 3×10^5 头牛和 3×10^5 查询。 在最坏的情况下，每个查询的直接模拟最多可能需要 O(n) 个步骤，这会导致 O(n·p) ≈ 10^10 次操作。 这远远超出了可行的限度。 

关键的困难在于每个查询的步长 b 可能会有所不同，这会阻止单个前缀和技巧统一处理所有查询。 

一些边缘情况很重要。 如果b = 1，则查询只是从a到n的范围后缀和。 如果 a 接近末尾，则级数可能仅包含一个元素。 如果 b 很大，则每个查询只涉及很少的元素，但我们不能依赖这一点，因为最坏情况 b = 1 使其呈线性。 

## 方法

 暴力破解的想法很简单：对于每个查询，从索引 a 开始，不断添加 a、a + b、a + 2b 直到超过 n。 每一步都会为答案增加一个值。 这是正确的，因为它完全遵循袭击的定义。 然而，在最坏的情况下，b = 1，每个查询都会触及 O(n) 个元素，因此总复杂度变为 O(n·p)，这太慢了。 

关键的观察结果是，步长的行为根据幅度的不同而有很大不同。 如果 b 很大，则访问的元素数量很小，因为 a + kb 很快就会超过 n。 如果 b 很小，那么尽管每个查询都会访问许多元素，但只有几个不同的步长，我们可以使用残差类上的动态规划来预先计算所有小 b 的答案。 

标准优化是将查询分为两种模式。 对于大 b，我们直接模拟级数，因为它很短。 对于小 b，我们使用 DP 表预先计算答案，其中我们从数组末尾沿着大小 b 的步长向后累加总和。 这避免了重复地重新计算重叠的算术级数。 

这是可行的，因为所有具有固定 b 的跳转通过余数模 b 形成独立的链，并且我们可以在每个 b 的线性时间内以相反的顺序处理每个链。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n 每个查询) | O(1) | O(1) | 太慢了 |
 | 步长的 sqrt 分解 | O(n √n + p √n) | O(n √n) | O(n √n) | 已接受 |

 ## 算法演练

 我们选择 √n 附近的阈值 B。 步长分为小步长（b ≤ B）和大步长（b > B）。 

1. 通过在数组上构建 DP 表来预先计算所有小步长 b 的答案。 对于固定的 b，我们处理从 n 到 1 的索引，如果 i + b ≤ n，则维持 dp[b][i] = w[i] + dp[b][i + b]，否则仅维持 w[i]。 这让我们可以在 O(1) 时间内回答任何小 b 查询。 
2.对于大步长b，我们不预先计算任何东西。 相反，对于每个查询，我们直接模拟 i、i + b、i + 2b 直到超过 n，并一路求和权重。 由于b很大，所以访问的位置数量最多为n/B，这个数量很小。 
3. 对于每个查询，我们检查是否 b ≤ B。如果是，我们返回 dp[b][a]。 否则，我们通过手动逐步遍历数组来计算总和。 
4. 独立输出每个查询的结果。

这种分离起作用的原因是每个查询要么具有很少的不同步骤链（小 b），要么每个链只有很少的元素（大 b），因此我们避免了两个维度上最坏情况的爆炸。 

### 为什么它有效

 固定步长 B. 索引根据残数模 b 分成独立的序列。 在每个序列中，每个位置的贡献仅取决于同一序列中的下一个位置。 通过从右向左处理，我们确保在计算 dp[b][i] 时，dp[b][i + b] 的值是已知的。 这将创建与 raid 总和的定义完全匹配的正确重复。 不同残基之间的重叠不会影响正确性，因为跳跃永远不会跨越残基类别。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    w = list(map(int, input().split()))
    p = int(input())

    B = int(n ** 0.5) + 1

    dp = [None] * (B + 1)
    for b in range(1, B + 1):
        dp[b] = [0] * n
        for i in range(n - 1, -1, -1):
            j = i + b
            if j < n:
                dp[b][i] = w[i] + dp[b][j]
            else:
                dp[b][i] = w[i]

    out = []
    for _ in range(p):
        a, b = map(int, input().split())
        a -= 1

        if b <= B:
            out.append(str(dp[b][a]))
        else:
            s = 0
            i = a
            while i < n:
                s += w[i]
                i += b
            out.append(str(s))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```DP表构建是核心优化。 每个 dp[b][i] 表示从 i 开始到步骤 b 的算术级数之和。 我们向后计算它，以便转换已经解决。 

对于大的步骤，循环故意简单：我们只是沿着进度走。 关键的正确性点是，由于所选的阈值，迭代次数足够小，可以通过。 

一个微妙的实现细节是索引：输入是从 1 开始的，但 DP 是建立在从 0 开始的索引之上的，因此 a 必须递减一次并以该形式一致使用。

 ## 边缘情况

 对于 b = 1，该算法完全依赖于 dp[1]，它实际上成为整个数组的后缀和。 这种情况确保了最大长度级数的正确性。 

对于 b ≥ n，遍历恰好访问一个元素，因为 a + b 立即超出界限。 该算法在 DP 和仿真路径中自然地处理这个问题。 

对于交替的小值，例如 w = [1, 0, 1, 0, ...] 且 b = 2，DP 确保我们仅正确累积相同奇偶校验索引，从而确认残差类分解是合理的。
