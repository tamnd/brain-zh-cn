 ## 解决方案
 ## 问题理解

 我们得到一棵树，意味着一个连通的无环图$n$节点与$n-1$边缘。 任务是计算该树中出现了多少个特定形状的不同子图。 如果两个子图由完全相同的边集组成，则它们被认为是相同的，因此我们可以有效地计算形成所需模式的不同边集。 

这种图案被非正式地描述为“鸡脚”，对应于一个以节点为中心并分支成三个独立边缘的小型连接结构，就像 Y 形或三叉星形。 每个有效实例完全通过选择一个中心节点并选择其三个关联边来确定，因为在树中任何此类选择都唯一地确定三个相邻节点以及连接它们的边。 

约束条件$n \le 5 \cdot 10^5$立即排除任何显式枚举子图或尝试节点或边组合的方法。 即使迭代所有的边三元组也会太慢，因为边的数量是线性的$n$在最坏的情况下，组合会爆炸成立方行为。 因此，解决方案必须将问题简化为可以在线性或近线性时间内计算的问题，最好是通过聚合每个节点的局部结构信息来计算。 

当树较小或具有低度节点时，会出现微妙的边缘情况。 如果$n < 4$，任何节点都不可能具有三个关联边，因此答案必须为零。 另一个常见的陷阱是误解了不同子图的构成：即使两个星形中心位于不同的节点，但碰巧看起来结构相似，如果它们的边集不同，它们也是不同的。 

## 方法

 思考这个问题的一种直接方法是尝试枚举每个可能的连通子图并检查它是否形成所需的三管齐下的结构。 在一棵树上与$n$节点，连接的子图的数量在最坏的情况下已经是指数级的，因为保持连接的每个边子集都是候选。 即使我们将自己限制为大小为 4 个节点的小子图，我们仍然需要探索边的组合并验证结构，这至少会导致$O(n^2)$或者在密集程度区域中表现更差。 这很快就变得不可行了$5 \cdot 10^5$节点。 

关键的观察结果是“鸡脚”结构完全由单个中心节点决定。 一旦中心确定，唯一的自由就是选择哪三个邻居参与。 在树中，与节点相关的任何三个不同的边都会自动形成所需形状的有效子图，因为不存在循环并且每条边都通向唯一的子树。 这将全局计数问题分解为纯粹的局部组合问题。 

对于每个节点$v$有学位$\deg(v)$，选择三个入射边的方法数量为$\binom{\deg(v)}{3}$。 将所有节点的总和得出有效“鸡脚”子图的总数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 子图的暴力枚举 |$O(n^2)$到$O(2^n)$|$O(n)$| 太慢了 |
 | 度数组合计数 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 计算变成对树结构的单次传递。 

1. 读取树并通过计算与每个节点相关的边数来计算每个节点的度。 这已经足够了，因为我们计算的结构仅取决于局部邻接，而不取决于全局拓扑。 
2. 将累加器初始化为零。 该变量将存储每个节点贡献的总和。 
3.对于每个节点$v$,考察其程度$d$。 如果$d \ge 3$，计算从其邻接列表中选择三个邻居的方法数，即$d(d-1)(d-2)/6$，并将该值添加到答案中。 度小于 3 的节点没有任何贡献，因为它们无法形成所需的分支结构。 
4. 输出累加模的和$998244353$。 

其工作原理清晰的原因是每个有效的配置都只有一个中心。 星形子图不能以树中多个节点为中心，因为中心是该子图中唯一具有三度的顶点，而其他顶点具有一度。 这种唯一性确保每个有效结构在处理其中心节点时都被精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def nC3(x):
    if x < 3:
        return 0
    return x * (x - 1) * (x - 2) // 6

def main():
    n = int(input())
    deg = [0] * (n + 1)

    for _ in range(n - 1):
        u, v = map(int, input().split())
        deg[u] += 1
        deg[v] += 1

    ans = 0
    for i in range(1, n + 1):
        ans += nC3(deg[i])

    print(ans % MOD)

if __name__ == "__main__":
    main()
```该实现的核心是度数组，它在读取边缘时跟踪局部结构。 每条边增加两度，每个端点增加一度，确保最终数组完全捕获邻接信息。 

组合函数直接使用整数运算。 由于Python安全地处理大整数，因此不存在溢出问题，但最终结果会模数减少$998244353$根据需要。 

一个微妙的细节是除以 6 是在整数空间中相乘之后完成的。 这是安全的，因为三个连续整数的乘积总是能被 6 整除，因此不会出现精度问题。 

## 工作示例

 考虑一棵形状像连接到四片叶子的中心节点的小树。 让边缘成为$(1-2), (1-3), (1-4), (1-5)$。 

| 节点| 学位| 贡献$\binom{d}{3}$|
 | --- | --- | --- |
 | 1 | 4 | 4 |
 | 2 | 1 | 0 |
 | 3 | 1 | 0 |
 | 4 | 1 | 0 |
 | 5 | 1 | 0 |

 答案是 4，对应于选择与节点 1 关联的 4 条边中的任意 3 条。这证实了三个叶子的每次选择都会生成一个不同的子图。 

现在考虑五个节点的路径：$1-2-3-4-5$。 

| 节点| 学位| 贡献 |
 | --- | --- | --- |
 | 1 | 1 | 0 |
 | 2 | 2 | 0 |
 | 3 | 2 | 0 |
 | 4 | 2 | 0 |
 | 5 | 1 | 0 |

 每个节点的度都小于 3，因此不存在有效的“鸡脚”。 答案是零，这符合路径没有分支点的直觉。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(n)$| 每条边都被处理一次以计算度数，并且每个节点被访问一次以计算组合 |
 | 空间|$O(n)$| 度数组每个节点存储一个整数 |

 该解决方案的线性结构非常适合以下约束：$5 \cdot 10^5$节点。 内存和时间使用量都与输入图的大小成正比。

 ## 边缘情况

 在星形输入中，算法将所有结构分配给中心节点。 例如，中心节点 1 连接到其他四个节点，则节点 1 的度数变为 4，而其他节点的度数仍为 1。$\binom{4}{3} = 4$正确计算三个边的所有不同选择，并且每个边对应一个唯一的边集，因此不会发生过度计数。 

在简单路径中，每个节点的度数最多为 2，因此组合公式永远不会触发。 该算法自然地产生零，无需任何特殊的大小写，这与没有节点可以充当分支点的事实相匹配。 

在具有多个分支节点的树中，例如两个相邻节点在不同子树中都具有高度，每个节点独立贡献。 由于有效的“鸡脚”必须具有唯一的中心，因此贡献之间不存在重叠，并且局部求和仍然正确。
