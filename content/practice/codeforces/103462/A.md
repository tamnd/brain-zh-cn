 ## 解决方案
 ## 问题理解

 我们正在构建一个长度的数组$n$使用将其分成块的过程。 最初数组是空的。 任意时刻，如果当前长度为$m$，我们选择一个数字$x$之间$1$和$n-m$。 选择后$x$，我们附加序列$1, 2, 3, \dots, x$到数组。 我们重复此操作，直到数组长度恰好变为$n$。 

关键点是，每个决策不只是添加一个数字，而是添加一个从 1 开始一直到所选值的整个递增块$x$。 不同的选择序列可以产生不同的最终数组，任务是计算给定的可能有多少个不同的最终数组$n$, 取模$10^9+7$。 

约束条件$n \le 10^6$直至$10^3$测试用例立即排除任何尝试模拟所有构造或对每个测试用例进行指数递归的方法。 甚至$O(n^2)$在最坏的情况下，每个测试用例都会太慢。 解决方案必须将问题简化为封闭形式，或者最坏的情况是简单的线性预计算。 

出现微妙的边缘情况时$n = 1$。 这个过程迫使一个单一的选择$x = 1$，仅生成数组$[1]$。 任何正确的公式在这里必然会产生 1。 一个天真的误解是认为即使所选块的总和相同，不同的序列也可能产生不同的数组，但这里每个块一次都是完全确定的$x$被选择，所以只有序列$x$-价值观很重要。 

## 方法

 该构造可以被视为重复选择要附加的元素数量，直到我们完全填充长度$n$。 假设我们将所选值记录为$x_1, x_2, \dots, x_k$。 每个$x_i$准确贡献$x_i$元素到最终数组，所以我们必须有$$x_1 + x_2 + \cdots + x_k = n.$$一旦这个顺序固定下来，最终的数组就完全确定了，因为每个块总是固定的模式$1,2,\dots,x_i$。 块内没有额外的自由。 

所以问题就变成了计算有多少个有序的正整数序列之和为$n$。 这正是组成的数量$n$。 

解决这个问题的一种强力方法是动态规划，其中$dp[i]$计算求和的方法$i$。 来自一个州$i$，我们尝试所有可能的下一个块大小$x \ge 1$，过渡到$i+x$。 这是正确的，但有一个明确的成本：对于每个$i$，我们可以迭代到$n-i$, 给予$O(n^2)$在最坏的情况下每个测试用例，这对于$n = 10^6$。 

结构洞察力是$n$相当于在之间放置分隔符$n$原子单位。 长度序列中每对相邻元素之间$n$，我们要么砍，要么不砍。 有$n-1$间隙，每个间隙独立地决定是否开始新的块。 这正好产生$2^{n-1}$的可能性。 

这将整个问题简化为模幂运算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 DP 求和 |$O(n^2)$每次测试 |$O(n)$| 太慢了 |
 | 二次幂公式|$O(\log n)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 1. 观察每个构造都对应于一系列选定的块大小$x_1, x_2, \dots, x_k$。 这是因为每个操作都精确地附加$x_i$元素，因此该过程完全由这些值描述。 
2. 将构造约束转化为方程$x_1 + x_2 + \cdots + x_k = n$，其中每个$x_i \ge 1$。 这完全删除了数组结构，并将问题简化为计算正整数的有效序列。 
3. 将每个解解释为一行的一个分区$n$位置到连续的段中。 每个段对应一个选定的块，并且内部固定为$1,2,\dots,x_i$。 内部值不影响计数。 
4. 将分区视图转换为二元决策问题：在每对相邻位置之间$i$和$i+1$，决定是否剪切（开始新块）或不剪切（继续当前块）。 有$n-1$这样的决定。 
5. 计算所有可能的切割图案。 每个$n-1$差距有两个独立的选择，所以结构总数是$2^{n-1}$。 
6. 预先计算 2 的幂$10^6$模数$10^9+7$，然后通过返回预先计算的值来在恒定时间内回答每个查询。 

### 为什么它有效

 正确性取决于有效构造序列和子集之间的双射$n-1$立场之间的界限。 每个有效的块大小序列都精确定义了一种将数组切割成连续递增段的方法，并且每个切割选择都唯一地确定了有效的块大小序列。 由于两个方向都是确定性且无损的，因此计数减少为对一组大小的子集进行计数$n-1$，即$2^{n-1}$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
MAXN = 10**6

# precompute powers of 2
pow2 = [1] * (MAXN + 1)
for i in range(1, MAXN + 1):
    pow2[i] = (pow2[i - 1] * 2) % MOD

t = int(input())
out = []

for _ in range(t):
    n = int(input())
    out.append(str(pow2[n - 1]))

print("\n".join(out))
```该解决方案预先计算 2 的所有幂一次，从而避免重复求幂$10^3$测试用例。 复发$pow2[i] = 2 \cdot pow2[i-1]$直接编码增加一个潜在切割位置的加倍性质。 

每个查询都简单地映射$n$到$2^{n-1}$，注意处理基本情况$n=1$，这正确地产生$2^0 = 1$。

import sys, io

MOD = 10**9 + 7
MAXN = 10**6

pow2 = [1] * (MAXN + 1)
for i in range(1, MAXN + 1):
    pow2[i] = (pow2[i - 1] * 2) % MOD

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append(str(pow2[n - 1]))
    return "\n".join(out)

# minimum size

# small case

# multiple tests
assert solve("3\n1\n2\n5\n") == "1\n2\n16", "mixed cases"

# large boundary check (just structure)
assert solve("1\n10\n") == str(pow2[9]), "n=10 check"

 ## 边缘情况

 对于$n = 1$，算法返回$2^{0} = 1$。 这匹配唯一可能的结构：选择$x=1$，生成数组$[1]$。 边界公式仍然成立，因为没有内部间隙需要决定，因此切割图案的数量为一。 

对于非常大的$n$， 例如$n = 10^6$，该算法仍然表现一致，因为它仅依赖于预先计算的模幂。 价值$2^{n-1} \bmod (10^9+7)$是明确定义和提前计算的，没有溢出或递归深度问题。
