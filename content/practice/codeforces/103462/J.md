 ## 解决方案
 ## 问题理解

 给定一个数组，其长度恰好是 2 的幂，具体来说$2^k$， 在哪里$k$最大可以为 20。每个操作都会获取当前数组并丢弃其中的一半（左半部分或右半部分），并保持另一半不变。 重复此操作直到我们停止。 

目标不是对数组进行完全排序，而是达到剩余数组不减的状态。 由于每次操作都会将大小减半，因此之后$t$操作剩余数组有长度$2^{k-t}$。 因此，这个过程实际上是选择一系列左/右​​半部分，该序列以一个连续的块结束，其长度仍然是 2 的幂。 

关键的困难在于并非每个连续的长度段$2^d$是可达的。 只有那些与重复减半创建的隐式二进制分区结构一致的片段才是有效的结果。 

从约束条件来看，$k \le 20$， 所以$n \le 2^{20}$，大约一百万个元素。 这立即表明$O(n \log n)$甚至$O(n k)$解决方案是可行的，而任何二次$n$会太慢。 

一种尝试左/右删除的所有可能序列的简单方法将探索$2^k$选择，但每个选择都会影响后续数组的结构，因此从最坏的意义上来说，这很快就会变成指数，并且无法扩展到非常小的范围$k$。 

一个微妙的边缘情况是当数组已经非递减时。 在这种情况下，我们希望零操作。 另一种情况是当只有非常小的段是单调的时，例如当数组上下交替时，因此只有长度为 1 的段才有效。 解决方案必须正确返回$k$，因为我们必须缩小到单个元素。 

## 方法

 直接暴力策略模拟每一个可能的删除序列。 在每一步中，我们选择是保留左半部分还是右半部分，对结果数组进行递归，并检查最终剩余的数组是否是非递减的。 这探索了深度二元决策树$k$，所以有$2^k$路径，每条路径都涉及验证最多可达的条件$2^k$处于中间状态的元素。 即使检查被优化，状态的数量本身也使得这对于$k = 20$。 

关键的观察结果是，该过程永远不会在块内创建新的相对顺序。 每个操作仅丢弃当前段的一半，剩余段始终是长度为 2 的幂并与二元分区对齐的连续间隔。 因此，我们可以直接考虑最终状态，而不是考虑操作序列：它总是某个长度的片段$2^d$在通过重复减半确定的边界上对齐。 

这将问题简化为找到长度最大的对齐段$2^d$这已经是非减少的了。 一旦达到最大值$d$众所周知，答案很简单，就是我们需要减半多少次$2^k$下降到$2^d$，即$k - d$。 

为了测试一个段是否有效地不递减，我们预先计算反转发生的位置，即位置$i$这样$a[i] > a[i+1]$。 当一个段内部不包含反转时，它就是有效的。

 | --- | --- | --- | --- |
 | 暴力破解删除序列|$O(2^k \cdot 2^k)$|$O(2^k)$| 太慢了 |
 | 使用预先计算的反转检查二元线段 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1.建立一个辅助数组，标记数组减少的每一个位置，意思是在哪里$a[i] > a[i+1]$。 这将“未排序”的概念压缩为单个点。 
2. 计算该标记数组的前缀和，以便我们可以在恒定时间内查询任意间隔内存在多少个反转。 
3. 对于每个可能的二次方长度$2^d$，扫描与该长度对齐的所有有效起始位置，这意味着索引可以整除$2^d$。 对于每个这样的段，检查其内部的反转次数是否为零。 
4. 跟踪最大值$d$其中至少有一个对齐的长度段$2^d$是完全不减的。 
5. 将这个最佳可实现的段大小转换为答案。 如果最佳线段有长度$2^d$，然后我们从$2^k$并需要$k - d$减半才能达到它。 

对齐很重要的原因是每个操作总是将当前数组精确地切成两半，因此可到达段的起始索引必须遵循这种二进制分区结构。 

### 为什么它有效

 每个操作序列都对应于原始数组的完整二进制分解，其中每个级别同时将所有候选段减半。 这迫使最后一段成为二元区间，而不是任意子数组。 由于数组永远不会重新排序，因此有效的唯一障碍是段内存在递减的相邻对。 因此，有效性完全简化为检查候选二元段是否包含任何反转，并且最大化其大小直接确定所需的最小减半数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    k = int(input().strip())
    n = 1 << k
    a = list(map(int, input().split()))

    bad = [0] * (n - 1)
    for i in range(n - 1):
        if a[i] > a[i + 1]:
            bad[i] = 1

    pref = [0] * (n)
    for i in range(n - 1):
        pref[i + 1] = pref[i] + bad[i]
    pref[n - 1] = pref[n - 2] + 0 if n > 1 else 0

    def has_bad(l, r):
        if l == r:
            return False
        return (pref[r] - pref[l]) > 0

    best_d = 0

    for d in range(k + 1):
        length = 1 << d
        ok = False
        for start in range(0, n, length):
            if start + length <= n:
                if not has_bad(start, start + length - 1):
                    ok = True
                    break
        if ok:
            best_d = d

    print(k - best_d)

if __name__ == "__main__":
    main()
```该实现首先将所有无序信息压缩到相邻比较的二进制数组中。 然后，前缀和结构允许恒定时间检查任何段是否包含违规。 外部循环尝试以 2 的递增幂的形式尝试段大小，而内部循环仅检查有效的二元对齐起始位置。 一旦找到给定大小的有效段，我们就会更新可实现的最佳深度。 

一个常见的微妙之处是前缀和中的边界处理，因为我们正在处理大小为$n-1$用于比较，但查询原始数组的索引范围。 该实现小心地移动索引，以便段查询与相邻比较的范围精确对应。 

## 工作示例

 考虑数组$[1, 3, 2, 4]$和$k = 2$。 

我们计算反转标记：仅$3 > 2$是一种违规行为。 

| d | 长度| 开始 | 段 | 里面不好| 有效 |
 | --- | --- | --- | --- | --- | --- |
 | 0 | 1 | 0,1,2,3 | 单个元素 | 没有| 是的 |
 | 1 | 2 | 0,2 | [1,3], [2,4] | [2,4] 好的 | 是的 |
 | 2 | 4 | 0 | 完整阵列 | 包含 3>2 | 没有|

 最佳有效二元线段长度是$2^1 = 2$，所以答案是$2 - 1 = 1$。 

现在考虑一个完全排序的数组$[1,2,3,4]$。 

| d | 长度| 开始 | 段 | 里面不好| 有效 |
 | --- | --- | --- | --- | --- | --- |
 | 2 | 4 | 0 | 完整阵列 | 无 | 是的 |

 这里最好的是$d = 2$，所以答案是$0$。 这符合不需要任何操作的事实。 

这些痕迹表明，我们不是在搜索任意子数组，而是在搜索二元对齐的子数组，并且在其中我们只是检查内部单调性是否成立。

 ## 边缘情况

 当数组已经是非递减的时，每个级别的每个二元段都是有效的。 该算法以最大可能的方式检测到这一点$d = k$，因为完整段不包含反转，并且立即返回零操作。 

当数组严格递减时，长度大于 1 的每个段都包含反转。 检查全部失败$d > 0$，仅留下$d = 0$有效的。 这迫使答案是$k$，相当于通过重复减半缩小为单个元素。 

当反转稀疏时，例如在大的其他排序区域内的单个局部下降，则仅对避开该确切位置的二元段进行计数。 由于对齐限制了哪些段甚至是可测试的，因此该算法正确地忽略了不可到达的间隔，并仅关注结构上有效的间隔。
