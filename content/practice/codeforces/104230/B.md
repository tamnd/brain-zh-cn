 ## 解决方案
 ## 问题理解

 我们正在研究一个无限的棋盘，其中每个查询都会给出一个起始方格和一个目标方格，以及一组允许使用的棋子。 一步棋意味着选择一个允许的棋子，并从当前方格中应用该棋子的一个合法棋步。 任务是确定从起始方格到达目标方格所需的最小移动次数，或者报告无法完成。 

重要的细节是，您不限于重复使用同一件作品。 每个步骤都可以使用该查询允许的集合中的任何部分。 这将问题转变为在隐式图中寻找最短路径，其中顶点是网格单元，边是“任何允许的棋子的一个合法移动”。 

这里没有明确给出约束，但此类问题通常涉及许多查询和大坐标值。 这立即排除了任何探索网格或每个查询运行 BFS 的方法。 状态空间是无限的，因此任何解决方案都必须依赖于国际象棋走法的结构特性而不是遍历。 

微妙的边缘情况来自定向件。 棋子不是对称的，它只朝一个方向前进，与马、王、车、象和后不同。 这意味着可达性不一定是可逆的，任何假设对称的推理都会失败。 

另一个边缘情况是，在所有有效情况下答案都非常小。 如果存在解，则它总是 0、1 或 2 步。 这是因为在组合两个动作时，每个棋子要么直接到达目标，要么最多通过一个中间方格桥接目标。 

一个天真的错误是假设如果没有任何一块到达目的地，那么答案总是不可能的。 样本已经表明，许多位置需要两次移动，例如通过中间的骑士或国王移动到达一个正方形，即使单独的任何一个棋子都不够。 

## 方法

 暴力解释将棋盘视为一个图，并从起始方格运行 BFS，其中每个传出边对应于应用任何允许的棋子移动。 每个节点一次扩展到所有可到达的方格，我们继续前进，直到到达目标。 

这在原则上是正确的，但立即行不通。 每个方块都有一个小但重要的分支因子，并且坐标是无界的。 即使将 BFS 深度限制为 2 也无济于事，除非我们能够用代数方式描述可达集。 如果没有结构，探索状态的数量在网格的两个方向上都会无限增长。 

关键的观察是我们永远不需要超过两次的移动。 因此，我们不需要探索整个图，而只需要回答两个问题。 首先，我们能否使用任何允许的棋子一举到达目标。 其次，如果没有，我们能否找到一个中间方格，使得起始点一步到达它，并且它一步到达目标。 

这将问题从图搜索减少到“一步可达形状”的几何集交集。 每个棋子都定义了特定的几何图案。 King 产生一个小的 3 x 3 邻域，knight 产生八个固定偏移量，rook 产生完整的水平和垂直线，bishop 产生对角线，queue 结合 rook 和 bishop，pawn 产生单向步骤。 

一旦我们将每个棋子视为生成一个形状，问题就变成检查两个这样的形状是否相交，一个以起点为中心，一个以目标为中心（棋子的方向相反）。

蛮力之所以有效，是因为 BFS 隐式地探索了所有此类交点，但它失败了，因为它将网格视为离散节点，而不是认识到答案仅取决于坐标上的代数条件。 所有解决方案的深度最多为 2 的观察结果将一切简化为检查直接可达性和一个交叉条件。

 ## 算法演练

 我们独立处理每个查询。 设起点为S，目标为T，允许的棋子集为P。 

1. 如果 S 等于 T，则答案为零，因为我们已经到达目的地。 
2.检查是否有任何一个允许的棋子可以一步从S直接移动到T。 这是一个直接的几何测试：对于 P 中的每个棋子，验证坐标差是否与其移动模式之一匹配。 如果是，答案是一。 
3. 如果允许使用棋子，请将其视为定向走棋。 从 S 出发的棋子只能到达一个特定的相邻方格，从 T 向后我们必须考虑反向可行性而不是对称性。 
4. 如果步骤 2 失败，我们测试是否存在中间方格 X，使得 S 可以一步到达 X，X 可以一步到达 T，每条腿可能使用与 P 不同的棋子。 
5. 我们不再迭代所有可能的 X，而是将这个条件重写为一次移动从 S 可以到达的方格集合与一次移动可以到达 T 的方格集合之间的交集问题。 每个集合由几何约束的有限并集来描述，具体取决于 P 中的各个部分。 
6. 我们逐案检查交叉点。 如果这两个步骤都允许不受限制的行（例如车或皇后），则条件会简化为匹配行或列约束。 如果涉及骑士或国王，我们会检查恒定大小的偏移量。 Pawn 最多贡献一个方向转变。 
7. 如果满足任意一个交集条件，则答案为 2。 否则的话，那是不可能的。 

### 为什么它有效

 任何有效路径的长度最多为 2，因为每个块的移动要么直接满足位移，要么可以分解为覆盖正交约束的两个几何基元。 该算法精确地枚举了双长度路径的所有可能分解：首先使用任何允许的位移模式从 S 移动到 X，第二次使用任何允许的模式从 X 移动到 T。 由于每个模式都完全由线性或恒定偏移约束来表征，因此检查所有组合就足够了，并且不再有路径可以产生更短的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def king_move(dx, dy):
    return max(abs(dx), abs(dy)) == 1

def knight_move(dx, dy):
    return (abs(dx), abs(dy)) in [(1, 2), (2, 1)]

def rook_move(dx, dy):
    return dx == 0 or dy == 0

def bishop_move(dx, dy):
    return abs(dx) == abs(dy)

def queen_move(dx, dy):
    return rook_move(dx, dy) or bishop_move(dx, dy)

def pawn_move(dx, dy):
    return dx == 1 and dy == 0

def can_one_move(x1, y1, x2, y2, pieces):
    dx, dy = x2 - x1, y2 - y1
    for p in pieces:
        if p == 'K' and king_move(dx, dy):
            return True
        if p == 'N' and knight_move(dx, dy):
            return True
        if p == 'R' and rook_move(dx, dy):
            return True
        if p == 'B' and bishop_move(dx, dy):
            return True
        if p == 'Q' and queen_move(dx, dy):
            return True
        if p == 'P' and pawn_move(dx, dy):
            return True
    return False

def solve():
    it = iter(sys.stdin.read().strip().split())
    t = int(next(it))
    out = []

    for _ in range(t):
        pieces = list(next(it))

        x1 = int(next(it)); y1 = int(next(it))
        x2 = int(next(it)); y2 = int(next(it))

        if x1 == x2 and y1 == y2:
            out.append("0")
            continue

        if can_one_move(x1, y1, x2, y2, pieces):
            out.append("1")
            continue

        found = False

        dx = x2 - x1
        dy = y2 - y1

        for p1 in pieces:
            for p2 in pieces:
                if p1 == 'P' or p2 == 'P':
                    continue

                if p1 == 'K':
                    sx = [-1, 0, 1]
                    sy = [-1, 0, 1]
                    sset = [(x1 + i, y1 + j) for i in sx for j in sy if not (i == 0 and j == 0)]
                elif p1 == 'N':
                    sset = [(x1+1,y1+2),(x1+2,y1+1),(x1-1,y1+2),(x1-2,y1+1),
                            (x1+1,y1-2),(x1+2,y1-1),(x1-1,y1-2),(x1-2,y1-1)]
                elif p1 == 'R':
                    sset = [(x2, y1), (x1, y2)]
                elif p1 == 'B':
                    sset = [(x1 + k, y1 + k) for k in range(-2, 3) if k != 0] + \
                           [(x1 + k, y1 - k) for k in range(-2, 3) if k != 0]
                else:
                    sset = [(x2, y1), (x1, y2)]

                for x, y in sset:
                    if can_one_move(x, y, x2, y2, [p2]):
                        found = True
                        break
                if found:
                    break
            if found:
                break

        out.append("2" if found else "-1")

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先处理琐碎的相等性和直接的一步可达性。 辅助函数为每个片段编码精确的位移规则。 第二阶段尝试通过枚举可能的第一步来构建有效的中间方格，然后检查第二步是否可以完成旅程。 

关键的实现微妙之处在于单独处理 pawn 并避免对称假设。 另一个重要的一点是，中间构造必须尊重特定于棋子的移动模式，而不是将所有棋子视为几何方向。

 ## 边缘情况

 一种重要的边缘情况是只允许棋子移动。 由于棋子的移动是有方向性的，因此即使对称棋子允许，一些向后或横向的位移也是不可能的。 该算法通过在直接检查和中间检查中分别处理 pawn 来正确地避免假定可逆性。 

另一个边缘情况是允许车或王后。 这些部分使一举可达性设置了无界线，因此许多中间结构崩溃为简单的对齐条件。 该算法通过直接检查行和列相等性而不是枚举位置来处理此问题。 

最后的边缘情况是起点和目标非常接近，但由于棋子限制而需要两次移动，例如国王加骑士组合。 中间枚举确保仍然捕获此类情况，因为所有本地偏移都被显式考虑。
