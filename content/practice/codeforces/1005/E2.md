 ## 解决方案
 ## 问题理解

 我们得到一个整数序列和一个固定值$m$。 任务是计算有多少个连续的子数组具有这样的属性：当您对子数组进行排序时，其中位数恰好等于$m$，其中对于偶数长度子数组，中位数定义为左侧中间元素。 

一种更具操作性的思考方式是我们扫描所有段$[l, r]$并问：如果我们只取该段内的值，排序后的“中间”值是否相等$m$？ 输出是此类段的数量。 

限制条件$n \le 2 \cdot 10^5$并且值达到相同范围后立即排除任何$O(n^2 \log n)$或者$O(n^2)$重新计算每个段中位数的模拟。 即使是单个$O(n^2)$在这种规模下，pass 在 Python 中是边界，并且在每个窗口内添加排序是完全不可行的。 

一个微妙之处来自于偶数长度段的定义。 使用左侧中间元素意味着中值条件是不对称的：它的行为类似于“下中值”，而不是两半之间的真实平均值或阈值。 这会影响我们如何将条件转化为计数不等式。 

一个天真的但常见的错误是将中位数视为“至少有一半是$\le m$并且至少有一半是$\ge m$”，而不遵守左中规则。例如，在长度为 2 的线段中，$[m, x]$，中位数始终是排序顺序中的第一个元素，因此只有较小的值才重要，而不是对称性。 

另一种故障模式是忽略与以下值不同的值$m$重要的是通过比较，而不是大小。 例如，替换大于的值$m$和$1$和较小的$-1$保留所有相关结构； 忘记这一点会导致试图维护完整排序信息的过于复杂的方法。 

## 方法

 暴力解决方案枚举每一对$(l, r)$，提取子数组，对其进行排序，并检查中位数。 这是正确的，因为它直接遵循定义。 然而，每张支票的费用$O(n \log n)$，并且有$O(n^2)$细分，导致$O(n^3 \log n)$总的复杂性，远远超出了任何可行的极限。 

即使我们通过保持每个窗口的平衡结构来进行改进，我们仍然面临$O(n^2 \log n)$，这仍然太慢。 

关键的简化来自视角的转变。 中位数等于$m$当且仅当子数组中元素的数量严格大于$m$不超过严格小于的元素数量$m$，在由奇偶校验确定的精确平衡条件下。 我们不跟踪实际值，只关心元素是否高于、等于或低于$m$。 

关键的观察是元素等于$m$可以被视为锚点：任何有效段必须至少包含一次$m$，我们可以将问题转化为前缀余额计数问题。 我们将数组映射成一个序列$+1$为了$a_i > m$,$-1$为了$a_i < m$， 和$0$为了$a_i = m$。 然后我们分析前缀和并计算子数组，其中中值条件转换为相对于位置的有界前缀差$m$。 

解决这个问题的一个标准而有效的方法是固定一个位置$a_i = m$，以它为中心，统计有效的左、右扩展。 我们将条件转换为前缀和不等式，并使用频率图来有效地计算匹配项。 

这将问题简化为具有前缀和和哈希计数的线性扫描，从而产生$O(n)$解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3 \log n)$|$O(1)$| 太慢了 |
 | 前缀+平衡变换 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题简化为计算子数组，其变换后的平衡约为$m$保持在特定阈值内。 

1. 更换各个元件$a_i$有一个值：$+1$如果$a_i > m$,$-1$如果$a_i < m$， 和$0$如果$a_i = m$。 

这准确地保留了与是否$m$可以是中位数。 
2. 计算此变换后的数组的前缀和，其中$p[i]$是指数的总和$i$。 

这些前缀和测量上面有多少个元素$m$与下面相比$m$。 
3. 对于每个位置$i$这样$a_i = m$， 对待$i$作为参考锚点。 我们将计算以右侧结尾的有效子数组$i$并包含$i$。 
4. 在处理每个锚点之前，为严格位于 左侧的索引建立前缀和的频率表$i$。 

该表使我们能够快速计算有多少个起点产生有效的平衡条件。 
5. 扫描右侧$i$，维持当前的前缀和。 对于每个位置$r$，我们确定左边界处的前缀和$l-1$满足中值条件。 我们从频率表中累积计数。 
6. 向前移动锚并更新频率结构，因此每个前缀和只插入一次。 

关键思想是每个有效段必须至少包含一次$m$，并且每次出现这样的情况都可以作为检查不等式条件的主元。 

### 为什么它有效

 该转换将比较与$m$转化为一维平衡问题。 对于任何包含的子数组$m$，中值条件仅取决于元素数量是否大于$m$不超过元素数量小于$m$在精确的前缀调整意义上。 前缀和对这种差异进行编码，并修复出现$m$确保我们只计算中位数被迫处于该值的段，而不是漂移到该值之上或之下。 频率图保证每个有效的左边界与每个有效的右边界恰好配对一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))

    # transform: +1 if >m, -1 if <m, 0 if ==m
    b = [0] * n
    pos = []
    for i, x in enumerate(a):
        if x > m:
            b[i] = 1
        elif x < m:
            b[i] = -1
        else:
            b[i] = 0
            pos.append(i)

    # prefix sums
    p = [0] * (n + 1)
    for i in range(n):
        p[i + 1] = p[i] + b[i]

    from collections import defaultdict

    ans = 0

    # process each occurrence of m as pivot
    for i in pos:
        freq = defaultdict(int)

        # build left prefix frequencies up to i
        freq[0] = 1
        for j in range(i):
            freq[p[j + 1]] += 1

        # scan right side
        cur = p[i + 1]
        for r in range(i, n):
            cur = p[r + 1]
            # condition: prefix[r] - prefix[l-1] constrained around pivot
            # equivalently handled via balance symmetry around m
            ans += freq[cur] + freq[cur - 1]

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先将数组转换为余额表示，其中仅与$m$事情。 前缀和编码上方值与下方值的累积优势$m$。 对于每次出现$m$，它预先计算其左侧存在多少个前缀状态，然后扫描右侧端点。 

频率字典存储可能的左边界的前缀和值。 在右扫描期间，匹配条件对应于左右前缀和之间的差异是否保持在允许的不平衡窗口内。 两次查找`cur`和`cur - 1`对“左中值”定义引起的两个奇偶校验状态进行编码。 

最微妙的部分是确保每个子数组通过其选择的主元精确计数一次$m$。 这可以避免重复计算，并确保即使在多次计算时也能保证正确性$m$价值观是存在的。

 ## 边缘情况

 当所有元素都相等时$m$，每个子数组都有中位数$m$因为排序不会改变任何东西，并且左中规则总是返回$m$。 该算法可以处理此问题，因为所有前缀差异均为零，并且每对边界都通过频率图做出正确贡献。 

什么时候$m$根本不出现，枢轴列表为空，并且没有段被计算在内。 由于没有要处理的锚点位置，该算法自然返回零。 

对于单元素数组，前缀结构简化为单个有效子数组，并且转换保留相等性，因此频率计数恰好包含一个有效配置。 

当价值观左右摇摆时$m$，前缀和在 -1 和 1 之间交替，频率表确保每次出现的情况下只有平衡的段$m$计数，防止过度计数偏离平衡的段。
