 ## 解决方案
 ## 问题理解

 给定两个字符串，并且允许我们重复删除任一字符串中最左边的字符。 每次删除都会将字符串之一缩短一个字符。 目标是使用尽可能少的删除来使两个结果字符串相同，并且我们希望删除的次数最少。 

The key observation is that deletions only affect prefixes. 一旦我们从每个字符串的左侧删除一定数量的字符，剩下的就是原始字符串的后缀。 因此，这个问题相当于找到两个相等的后缀（每个字符串一个），并最小化到达它们所需的删除字符总数。 

约束允许每个字符串的长度最多为 200,000。 任何比较所有后缀对或模拟每种可能性的删除的二次方法在最坏的情况下都需要大约 10^10 次操作，这远远超出了可行的限制。 这立即排除了后缀的强力成对匹配或嵌套循环内的重复字符串切片。 

当字符串根本没有公共后缀时，就会出现微妙的边缘情况。 例如，如果`s = "abc"`和`t = "xyz"`，没有非空后缀匹配，因此使它们相等的唯一方法是将两者完全删除。 答案是 6。如果尝试从末尾对齐而不正确处理空匹配情况，假设至少有一个匹配字符的简单方法可能会失败。 

当一个字符串已经是另一个字符串的后缀时，会出现另一种边缘情况。 例如，`s = "abcde"`和`t = "cde"`。 最佳策略是仅从较长的字符串中删除，直到两者匹配`"cde"`，给出 2 步。 任何试图在不检查后缀包含的情况下同步双方删除的方法都可能会过度计数操作。 

## 方法

 暴力观点首先考虑从两个字符串左侧删除字符的所有可能方法。 如果我们删除`i`字符来自`s`和`j`字符来自`t`，我们最终比较`s[i:]`和`t[j:]`。 我们想要一对`(i, j)`这使得这些后缀相等，同时最小化`i + j`。 

这种方法是正确的，因为每个有效的删除序列完全对应于选择每个字符串中保留多少个字符。 然而，尝试一切`(i, j)`对给出 O(nm) 比较，并且如果简单地完成，在最坏的情况下每次比较可能花费 O(n) 时间。 对于 200,000 长度的字符串来说，这变得太慢了。 

关键的见解是我们不需要搜索所有后缀对。 相反，我们颠倒了观点：我们比较两个字符串末尾的字符并计算它们的后缀匹配延伸了多长时间。 一旦字符发散，我们就无法进一步扩展匹配，因此最长公共后缀完全决定了最优解。 

如果最长公共后缀有长度`k`，那么我们可以保留那些`k`两个字符串中的字符。 之前的所有内容都必须删除。 所以答案是`(len(s) - k) + (len(t) - k)`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从两个字符串的最后一个字符开始，因为删除仅影响前缀，后缀相等取决于后缀结构。 我们试图最大化两个字符串向后匹配的距离。 
2.最后初始化两个指针`s`和`t`。 同时初始化一个计数器`k = 0`跟踪匹配后缀的长度。 
3. 当两个指针都有效并且这些位置的字符相等时，增加`k`并将两个指针向左移动一步。 这直接从右到左构建了最长的公共后缀。 
4. 当任一指针越界或发生不匹配时停止。 此时，不可能进一步扩展公共后缀，因为任何不匹配都会破坏后缀相等性。 
5. 将答案计算为`(len(s) - k) + (len(t) - k)`，因为必须删除匹配后缀之外的所有字符。 

### 为什么它有效

 每个有效的最终配置都对应于选择后缀`s`和后缀`t`是相同的。 该算法通过从末尾贪婪地扩展匹配来找到最大可能的后缀，这是最优的，因为任何较短的匹配只会增加删除的数量。 由于字符必须在后缀中进行位置匹配，因此任何分歧都会立即使较长的扩展无效，从而使贪婪的向后扫描既充分又最佳。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    t = input().strip()

    i = len(s) - 1
    j = len(t) - 1

    while i >= 0 and j >= 0 and s[i] == t[j]:
        i -= 1
        j -= 1

    common_suffix_len = len(s) - (i + 1)
    # equivalently len(t) - (j + 1), but both are same at this point

    ans = (len(s) - common_suffix_len) + (len(t) - common_suffix_len)
    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了算法中描述的反向扫描。 该循环维护从每个字符串末尾移动的两个索引，并计算后缀匹配持续的时间。 最终公式从两个字符串中减去匹配的后缀长度以计算删除。 减法`(i + 1)`将最终的不匹配位置转换为匹配区域的长度。 

必须注意循环条件在索引之前检查边界，因为 Python 允许负索引，如果没有适当保护，这会默默地产生不正确的匹配。 明确的`i >= 0 and j >= 0`可以防止这个问题。

 ## 边缘情况

 对于`s = "abc"`和`t = "xyz"`，算法从末尾开始比较：`c`与`z`，立即不匹配，所以`k = 0`。 答案就变成了`3 + 3 = 6`，正确反映必须完全删除两个字符串。 

为了`s = "abcde"`和`t = "cde"`，比较进行如下`e`与`e`,`d`与`d`,`c`与`c`，然后停在`b`与字符串结尾。 这里`k = 3`，所以答案是`2`，匹配只需要删除`"ab"`从第一个字符串开始。 

这些案例证实该算法正确处理了两个极端：完全不匹配和完全包含。
