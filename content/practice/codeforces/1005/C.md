 ## 解决方案
 ## 问题理解

 我们得到了一个整数的多重集，并且我们可以删除它们的任何子集。 删除后，我们希望剩余元素满足配对条件：每个剩余值必须能够找到至少一个其他剩余值，使得它们的总和等于 2 的幂。 

此条件是每个元素的本地条件，但合作伙伴的选择必须来自相同的最终集合。 因此，问题实际上是要求最大可能的子集，其中每个元素至少参与一个有效的“二次幂和对”，然后我们从原始数组长度中减去该大小。 

n 高达 120000 的约束迫使我们远离任何二次方程，甚至是简单形式的 n√n。 任何尝试所有对或独立地重新计算每个元素的匹配而不重用的解决方案都会超时，因为在最坏的情况下检查所有候选对将需要大约 n2 次操作，大约为 10^10。 

当根本无法配对时，就会出现微妙的边缘情况。 例如，如果数组包含像[16]这样的单个元素，则无法满足条件，因此最佳答案是删除所有内容。 另一个有趣的情况是，当值太大而无法配对到两个和范围的任何幂时，例如 [4, 16]，其中即使是最佳可能的补码 (4 + 16 = 20) 也不是 2 的幂。 

关键的困难在于，该条件不是构造性对称的：每个元素都有多个可能的目标，但我们必须全局确保所有元素都被某种匹配结构覆盖。 

## 方法

 一种直接的方法是尝试数组的每个子集并测试它是否良好。 对于固定子集，我们将检查每个元素并尝试在其余元素中找到其总和为 2 的幂的伙伴。 即使我们使用哈希集优化成员资格检查，在最坏的情况下验证一个子集的成本也为 O(n²)。 由于有 2ⁿ 子集，这是完全不可行的。 

即使我们将自己限制在每个元素贪婪配对的想法上，我们很快就会遇到冲突。 一个元素可能有多个可能的伙伴，并且在本地选择一个元素可能会阻止另一个元素找到其所需的匹配项。 因此，从头开始对每个元素进行纯粹的贪婪匹配会失败，因为它忽略了全局一致性。 

关键的观察是，只有 2ᵈ 形式的和才重要，并且每个值 aᵢ 只需要找到一个伙伴。 这建议将问题重新定义为图上的匹配问题：每个数字都是一个节点，如果 aᵢ + aⱼ 是 2 的幂，我们就连接 i 和 j。 我们希望删除尽可能少的节点，以便每个剩余节点在导出子图中的度数至少为 1。 

我们没有明确地构建这个巨大的图表，而是颠倒了视角。 对于每个值 x，其有效伙伴是 2ᵈ − x 形式的数字。 由于值高达 10⁹，因此 d 只需要达到约 31 或 32。这极大地限制了每个元素的搜索空间。 

然后我们使用频率图并贪婪地尝试形成有效的对。 关键的结构是配对是对称的，因此我们可以迭代值并将每个出现的值与可用的补集贪婪地匹配，确保每个元素最多使用一次。 

最终目标相当于最大化参与至少一次成功配对的元素数量，这减少了尽可能重复形成不相交的对。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²) 或更糟 | O(n) | 太慢了 |
 | 频率+功率枚举| O(n log A) | O(n log A) | O(n) | 已接受 |

 ## 算法演练

1. 构建数组中所有值的频率图。 这允许恒定时间访问一个数字剩余的未使用副本。 
2. 迭代映射中的每个不同值 x。 对于每个 x，尝试将其出现的情况一一配对。 
3. 对于每次出现的 x，尝试两个和 2ᵈ 的所有幂，使得 2ᵈ − x 成为候选值。 如果补码 y = 2ᵈ − x 存在于图中且剩余频率为正，则形成一对 (x, y) 并减少两个计数。 
4. 继续此过程，直到无法为 x 形成更多对，然后移至下一个值。 
5. 计算成功配对的元素数量。 由于每对贡献两个保留元素，因此答案是 n 减去成功配对中使用的元素数量。 

我们尝试每个值的所有 2 的幂的原因是每个元素只有 O(log maxA) 潜在伙伴，这可以保持搜索较小并确保我们不会错过任何有效的配对。 

### 为什么它有效

 每个有效配置都分解为不相交的对，因为每个元素必须至少有一个伙伴，并且一个伙伴就足够了。 一旦我们修复了只关心选择兼容性图的边的问题，最大化保留的元素就减少为选择尽可能多的有效边而不重用顶点。 由于每个顶点最多参与构造的解决方案中的一条边，因此对所有可能的功率目标进行贪婪配对就足够了，因为只要每条边独立地满足条件，任何最佳解决方案都可以重新排列为这种不相交的匹配而不会造成损失。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    from collections import Counter
    cnt = Counter(a)
    
    powers = [1 << i for i in range(32)]
    
    used = Counter()
    ans_pairs = 0
    
    for x in list(cnt.keys()):
        while cnt[x] > 0:
            matched = False
            for p in powers:
                y = p - x
                if y < 0:
                    continue
                if cnt[y] > 0:
                    cnt[x] -= 1
                    cnt[y] -= 1
                    ans_pairs += 1
                    matched = True
                    break
            if not matched:
                break
    
    print(n - 2 * ans_pairs)

if __name__ == "__main__":
    solve()
```该实现依赖于频率计数器，因此我们不会显式构建所有对。 2 的幂列表预先计算到 2^31，在给定约束的情况下，它安全地涵盖了所有可能的总和。 

对于每个值，我们反复尝试找到形成有效的二次幂和的补码。 一旦形成一对，两个频率都会立即降低，确保没有元件被重复使用。 

一个微妙的点是，我们根据频率图的当前状态而不是固定快照重新计算补充，因为早期的配对会影响以后的可用性。

 ## 边缘情况

 像[16]这样的单元素数组证明了任何假设每个值都可以找到合作伙伴的策略都是失败的。 该算法处理该值，在任何 2ᵈ − 16 中找不到补集，并使其不匹配，从而正确地实现完全删除。 

在像 [1, 1] 这样的情况下，算法立即发现 1 + 1 = 2（2 的幂）消耗了两次出现，并产生零删除。 基于频率的结构确保两个副本得到对称处理，而不会重复计数。 

在存在多个配对选项的数组中，例如 [1, 3, 2, 8]，算法对频率的顺序消耗确保一旦在一对中使用某个值，它就不会错误地参与另一对，从而在不相交对要求下保持正确性。
