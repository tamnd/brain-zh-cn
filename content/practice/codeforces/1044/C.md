 ## 解决方案
 ## 问题理解

 我们得到一个凸多边形，其顶点已按顺时针顺序列出。 几何形状是固定的：我们不能移动点，只能选择顶点的子集。 

对于任意整数$k$，我们考虑可以通过精确选择来形成的所有简单（非自相交）多边形$k$这些顶点并以某种保持简单性的循环顺序连接它们。 在所有此类多边形中，我们需要最大可能的周长，其中边长是使用曼哈顿距离来测量的。 

任务是计算每个区域的最大周长$k = 3, 4, \ldots, n$。 

一个关键的困难是，尽管输入多边形是凸多边形，但除非我们保持循环顺序，否则所选子集不会自动形成凸多边形。 真正的约束是组合的：我们必须按循环顺序选择顶点的子序列，然后周长是沿该循环的曼哈顿距离之和。 

约束条件$n \le 3 \cdot 10^5$排除任何尝试枚举子集甚至对所有对进行二次动态规划的解决方案。 任何超出粗略范围的事情$O(n \log n)$或者$O(n \cdot \text{polylog} n)$已经是临界点了。 

一个天真的尝试是修复一个子集$k$顶点并计算其最佳循环排列。 即使我们假设凸性简化了排序，迭代所有$\binom{n}{k}$子集是不可能的。 如果没有结构，即使是跨区间的 DP 仍然会太大。 

一个微妙的边缘情况是曼哈顿距离的表现与欧几里德距离不同。 欧几里得周长问题的最佳结构通常是完整的多边形或其凸包，但这里我们必须根据轴对齐的贡献进行推理。 

另一个陷阱是假设最佳多边形总是使用连续的顶点。 对于曼哈顿度量来说，这通常是错误的，因为跳过顶点可以以有用的方式增加总水平或垂直跨度。 

## 方法

 蛮力视图从定义开始：选择任意$k$顶点并按循环顺序排列它们，然后沿边求和曼哈顿距离。 即使我们将顺序固定为按输入顺序循环，我们仍然需要决定要删除哪些顶点。 对于每个子集，我们在线性时间内计算周长，从而得到$O(n \cdot 2^n)$行为，这是立即不可行的。 

关键的结构观察来自于重写曼哈顿距离。 为了两点，$$|x_1 - x_2| + |y_1 - y_2|$$分成独立的 x 和 y 贡献。 在顺时针排序的凸多边形上，边总是以角度单调移动，因此沿边界的 x 和 y 的差异以结构化方式表现。 

更深入的见解是，每个选定的多边形都对应于删除$n-k$循环中的顶点。 每次移除都会用一个较长的边缘替换两个边界边缘，并且增益仅取决于局部几何形状。 这将问题转化为选择能够最大化增量收益的移除。 

我们可以考虑从完整的多边形开始，反复删除顶点。 删除顶点$i$替换边缘$(i-1, i)$和$(i, i+1)$和$(i-1, i+1)$。 去除的增益$i$是：$$d(i-1, i+1) - d(i-1, i) - d(i, i+1)$$由于多边形是凸的，因此除了移除后的邻接变化之外，这些增益是明确定义且独立的。 该结构使我们能够维护一组动态的候选者，并始终以效果递增的顺序选择最佳的可用删除，这导致排序或基于堆的处理优于增益。 

然而，这个特定问题的关键简化是，在曼哈顿度量下，所有去除增益在凸循环中都是独立的，因此我们可以预先计算所有增益一次，然后对它们进行排序。 每一个选择$k$对应于采取最好的$n-k$搬迁。 

我们从完整的周长开始，然后反向减去最小的去除量。

 ## 算法演练

 1. 使用给定顺序的连续顶点上的曼哈顿距离（包括从最后一个到第一个的边）计算完整多边形的周长。 这是我们对应的起始值$k = n$，因为没有顶点被删除。 
2. 对于每个顶点$i$，计算移除它的成本：$$gain[i] = d(i-1, i+1) - d(i-1, i) - d(i, i+1)$$其中指数是循环获取的。 这测量了顶点的周长变化量$i$在循环中被跳过。 该表达式为负数或零，因为删除顶点通常会缩短边界。 
3. 将所有收益收集到一个列表中。 按升序对它们进行排序。 此顺序对应于首先以最有利的方式删除顶点，即那些减少周长最少的顶点。 
4. 现在按排序顺序模拟移除。 保持运行值`cur`从整个周边开始。 当我们“移除”一个带有增益的顶点时$g$，我们更新：$$cur += g$$表演后$t$移除，我们有一个多边形$n - t$顶点，和`cur`正是它的周长。 
5.记录答案：对收益排序后，收益的前缀和直接给出$f(k)$为所有人$k$， 在哪里$k = n, n-1, \ldots, 3$。 反向索引产生的答案越来越多$k$。 

### 为什么它有效

 关键的特性是，在曼哈顿距离下的凸多边形中，移除顶点引起的周长变化仅取决于移除时的直接邻居，但凸结构确保“最佳移除”的相对顺序永远不会以影响最优性的方式发生变化。 这使得移除序列相当于对所有局部增益进行一次排序并贪婪地应用它们。 每个最优的顶点删除序列都对应于选择相同的多组增益，只是顺序不同，因此排序足以重建所有最优的中间状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def manhattan(a, b):
    return abs(a[0] - b[0]) + abs(a[1] - b[1])

n = int(input())
p = [tuple(map(int, input().split())) for _ in range(n)]

# full perimeter
cur = 0
for i in range(n):
    cur += manhattan(p[i], p[(i + 1) % n])

gains = []
for i in range(n):
    a = p[(i - 1) % n]
    b = p[i]
    c = p[(i + 1) % n]

    gain = manhattan(a, c) - manhattan(a, b) - manhattan(b, c)
    gains.append(gain)

gains.sort()

# suffix sums of gains
suffix = [0] * (n + 1)
for i in range(n - 1, -1, -1):
    suffix[i] = suffix[i + 1] + gains[i]

# answers: remove t vertices => k = n - t
# we need k >= 3 => t <= n - 3
out = []
for t in range(n - 3):
    k = n - t
    out.append(str(cur + suffix[0] - suffix[t]))

print(" ".join(out))
```该代码首先计算原始多边形的周长。 这是保留所有顶点对应的基线状态。 

这`gain`计算捕获因移除顶点而引起的精确周长变化。 由于多边形是循环的，因此索引会环绕。 对这些值进行排序可确保我们按照周长损失最小化的顺序应用移除。 

后缀和技巧将“应用第一个最佳删除”转换为 O(1) 查询，这是必要的，因为我们必须输出所有的答案$k$。 

一个微妙的实现点是循环索引：忘记增益计算中的环绕会产生不正确的局部贡献。 另一个是我们在删除后从不显式更新邻接关系，因为排序参数取代了动态结构维护的需要。

 ## 边缘情况

 最小凸三角形测试没有超出的删除$k=3$需要。 该算法处理这个问题是因为循环$t$停在$n-3$，恰好产生一个值。 

点形成近矩形的情况突出表明多个顶点可能具有相同的去除增益。 排序自然地处理了这个问题，因为可以以任何顺序应用相等的增益，而不会影响前缀和。 

一个看似退化但仍然凸出的形状，具有极端的坐标，考验着曼哈顿的溢出风险。 使用 Python 整数可以避免溢出问题，并且由于不涉及浮点运算，因此所有计算都保持精确。
