 ## 解决方案
 ## 问题理解

 我们被赋予了一个巨大的网格，但运动并不是逐个单元地移动。 相反，车从左下角开始，只要没有任何东西阻挡其直线路径，就可以沿整行或整列传送。 

有两种障碍。 一种阻止特定垂直边界处的两个相邻列之间的移动，有效地切断所有行跨该列边界的水平移动。 另一种类型会阻止沿给定行上的特定水平线段的移动，从而阻止垂直通过该线段。 

目标是确定必须移除的障碍物的最小数量，以便从左下角的起始单元格到顶行的任何单元格之间至少存在一条路径。 

重新解释这一点的一个有用方法是忘记巨大的网格，并考虑被障碍分隔的区域之间的连通性。 垂直法术的作用就像放置在某些 x 坐标处的垂直墙壁，而水平法术的作用就像放置在特定 y 坐标处的部分水平墙壁。 车的移动将网格变成一个图表，其中每个自由矩形区域都与其相邻区域相连，除非咒语阻挡了它们之间的边界。 

限制很大，纵向最多十万个，横向十万个。 这立即排除了任何模拟网格或构建显式单元图的解决方案。 即使明确表示所有区域也是不可能的，因为坐标高达 10^9。 任何解决方案都必须将结构压缩为咒语数量的线性形式，可能使用排序和扫描。 

一个微妙的边缘情况来自这样一个事实：水平段彼此不重叠，但它们可以跨行任意放置。 这保证了一种垂直的有序结构，这对于投影后将问题转化为一维问题至关重要。 

一个天真的错误是假设独立删除所有水平或所有垂直块就足够了。 例如，如果所有垂直块都被移除，但单个水平段完全跨越某行的宽度，则车可能仍被困在其下方。 相反，单独删除水平段可能仍会留下隔离起始列的垂直分区。 

另一种误导情况发生在水平线段稀疏但策略性对齐以便每条可能的垂直路线至少与其中一条相交时。 删除“最大”段或最频繁类型的贪婪方法可能会失败，因为阻塞效应与可达性有关，而不是与频率有关。 

## 方法

 如果我们尝试直接模拟该过程，我们将尝试将每个单元视为一个节点并连接相邻的单元，除非被阻塞。 这立即变得不可行，因为网格是 10^9 x 10^9。 即使压缩坐标仍然给我们留下一个大的平面图，其中删除下的最短路径或连接性将是昂贵的。 

一个更有用的视角是将问题视为寻找一条穿过障碍物之间的一系列“间隙”的路径。 车从左下角开始，只有在每个可能的 x 范围内至少有一条垂直走廊没有被水平段完全切断时才能到达顶部。 

关键的洞察力是翻转视角。 我们不考虑路径，而是考虑“仍然可用的列”。 如果我们可以在柱内从下到上垂直移动，而不会被完全覆盖该柱的水平段阻挡，那么该柱就是可用的。 然而，垂直法术会迫使我们切换列，因此我们必须考虑由垂直墙分隔的列的间隔。

这自然会导致问题被压缩到垂直墙之间的各个部分。 每个垂直咒语将 x 轴分成独立的部分。 在每个分段内，车可以自由水平移动。 因此，唯一真正的障碍是每个这样的段是否在某个 y 水平上垂直完全被阻挡。 

现在问题变成：对于每个垂直线段，确定需要多少个水平线段才能将其从下到上完全挡住。 我们想要选择一组最少的法术来移除，以便至少一个垂直部分保持畅通。 

这转化为一个覆盖问题，其中水平线段“覆盖”垂直线段，并且垂直拼写定义独立区域。 该解决方案简化为对 x 坐标进行排序和扫描，并有效地评估覆盖区间，通常使用两指针或基于事件的扫描。

 ## 算法演练

 1. 按 x 坐标对所有垂直拼写进行排序。 这将网格划分为连续的垂直条带，条带内不存在垂直障碍。 
2、将每个条带自下而上视为一条独立的走廊。 车可以在带内水平自由移动，因此仅当在带之间过渡或垂直被阻挡时，运动限制才有意义。 
3. 对于每个条带，确定哪些水平线段与其相交。 如果水平段的 x 范围与条带的 x 间隔重叠，则水平段会影响条带。 
4. 对于每个条带，计算是否存在从 y = 1 到 y = 10^9 的“垂直路径”，以避免所有水平段与该条带重叠。 由于水平线段在同一行内不重叠，因此它们在不同高度充当不相交的阻挡物。 
5. 如果存在一系列水平段，这些水平段共同形成从左到右横跨条带可访问的 x 范围的障碍，则观察到条带被阻挡。 由于条带是独立的，因此我们单独评估每个条带。 
6. 对于每个条带，计算需要移除的水平段的最小数量，以便在垂直方向上保留至少一个间隙。 这成为一种经典的间隔刺击最小化：我们希望确保至少有一条未被覆盖的垂直链。 
7. 答案是使该条带通过所需的最小移除次数。 同样，我们计算有多少水平段阻挡了每个可能的垂直通道，并选择最便宜的条带进行修复。 

### 为什么它有效

 垂直墙将平面分割成独立的 x 分量。 在每个组件中，车都可以水平自由移动，因此只有垂直进展才重要。 水平段是唯一可以阻止向上前进的结构，并且它们在每个带上都是独立的。 由于条带一旦被垂直墙分开就不会相互作用，因此当且仅当至少一个条带允许完全垂直遍历时，全局路径才存在。 因此，最小化删除减少到最小化最有利条带中的删除，这保证了全局可行路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    vertical = []
    for _ in range(n):
        vertical.append(int(input()))
    vertical.sort()

    horizontal = []
    for _ in range(m):
        x1, x2, y = map(int, input().split())
        horizontal.append((x1, x2, y))

    if n == 0:
        # only one strip [1, INF]
        # we just need to ensure at least one vertical path exists
        return print(m)

    # build strips between vertical walls
    # include boundaries 1 and 1e9+1
    xs = [0] + vertical + [10**9 + 1]

    best = m

    # For each strip, evaluate cost
    for i in range(len(xs) - 1):
        L, R = xs[i], xs[i + 1]

        # count horizontal segments that fully block this strip in some way
        blockers = 0
        for x1, x2, y in horizontal:
            if not (x2 < L or x1 > R - 1):
                blockers += 1

        best = min(best, blockers)

    print(best)

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现遵循垂直咒语引起的条带分解。 垂直位置列表经过排序并用于定义连续的 x 范围。 每个范围都是独立处理的，我们计算有多少水平线段与该范围相交。 此类计数的最小值表示车在垂直方向上受约束最少的条带，并且删除那些相交的线段足以打开完整的路径。 

一个微妙的点是正确处理边界：垂直拼写定义 x 和 x+1 之间的墙，因此每个条带对应于连续垂直切割之间的包含整数范围。 相差一处理`R - 1`确保我们不会错误地将边界本身包含为可达区域的一部分。

 ## 边缘情况

 一个关键的边缘情况是没有垂直咒语。 在这种情况下，整个网格形成一个条带，因此答案仅取决于水平部分是否共同阻止向上运动。 该算法自然会分解为评估单个区间。 

当垂直法术密集排列时会出现另一种边缘情况，从而形成许多狭窄的条带。 该解决方案仍然可以处理这个问题，因为每个条带都是独立的； 即使大多数条带被阻挡，单个可行的条带也可以确定答案。 

最后一个微妙的情况是不同行的水平段重叠。 即使它们不因约束相交，它们仍然可以共同阻挡一条带。 该算法正确地对每个条带进行计数，而不假设行之间有任何顺序。
