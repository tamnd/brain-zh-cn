 ## 解决方案
 ## 问题理解

 我们得到了一系列排成一排的杯子。 每个杯子都有一个值，代表饮用时的放松程度。 叶夫根尼反复拿走杯子，直到没有杯子为止，但每次他只能拿走最左边或最右边的杯子。 

不同的是，杯子的好处不仅仅是它的内在价值。 他喝酒的顺序很重要。 他喝的第一杯贡献其价值乘以 1，第二杯贡献其价值乘以 2，依此类推。 如果拾取杯子的顺序是$x_1, x_2, \dots, x_n$，总松弛度为$\sum i \cdot x_i$。 

目标是选择一系列左移除和右移除，以最大化该加权和。 

约束条件$n \le 5000$对于二次动态规划解决方案来说足够小，但对于尝试所有方法的任何方法来说都太大了$2^n$左右决策。 如果不小心实施，即使是间隔上的三次 DP 也会太慢，因为$n^3$会接近$1.25 \times 10^{11}$。 

由于乘数取决于全局顺序，而不是原始数组中的位置，因此出现了一个微妙的问题。 像“总是取较大的一端”这样天真的贪婪策略会失败，因为早期的大值可能比延迟到稍后的乘数位置更有价值。 

一个简单的反例是$[1, 100, 1]$。 贪婪地获取：

 - 挑选100个第一给$1 \cdot 100 = 100$，然后两个贡献$2 \cdot 1 + 3 \cdot 1 = 5$，共 105 个。 

但最佳的做法是先取 1：
 - 选择 1，然后选择 100，然后选择 1$1 + 2 \cdot 100 + 3 = 204$，哪个更好。 

所以该结构不是局部贪婪的； 这取决于端点如何随着时间的推移进行交互。 

## 方法

 蛮力视图很简单：在每一步我们都有两个选择，向左选择或向右选择，然后我们跟踪序列。 这构建了高度的二元决策树$n$，生产$2^n$序列。 对于每个序列，我们计算加权和$O(n)$, 给予$O(n2^n)$，即使对于$n=30$。 

关键的观察是，唯一重要的状态是数组的哪一部分仍然存在。 一旦我们修复了一个段$[l, r]$，除了已经采取了多少要素之外，未来与之前的决定无关。 该计数决定了下一次选择的乘数。 

因此，我们在间隔上定义一个 DP，其中我们明确跟踪已经获取了多少元素。 这将问题转化为选择下一个元素是来自左端点还是右端点，同时累积取决于步骤索引的成本。 

我们通过确保每个状态仅由区间边界定义来避免指数分支，同时确定性地导出步数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n \cdot 2^n)$|$O(n)$| 太慢了|
 | 间隔 DP |$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们定义一个 DP 表，其中$dp[l][r]$表示通过从索引中删除子数组中的所有元素可以实现的最大总松弛$l$到$r$，假设这些删除将作为最后剩余的操作按某种顺序发生。 

为了正确地结合乘数效应，我们在解决大小子问题时会推断出已经删除了多少元素$k = r - l + 1$。 如果$n - k$元素已经被选取，那么下一个选取的元素有乘数$n - k + 1$。 我们没有在状态中明确携带该值，而是构建 DP，以便在缩小间隔时应用贡献。 

## 算法演练

 1. 定义$dp[l][r]$作为从子阵列可获得的最大贡献$a[l..r]$当这些元素在此过程中最后被消耗时。 这种重构让我们将 DP 视为从内到外构建最终序列。 
2. 初始化基本情况，其中$l = r$。 如果只剩下一个元素，它将在该子问题中被放在最后，因此它的贡献只是它的值乘以整体顺序中的正确位置，这由我们组合状态的方式隐式处理。 
3. 对于从 2 到$n$，计算所有值$l, r$和$r - l + 1 = len$。 
4. 对于每个间隔$[l, r]$，判断该区间最后移除的元素是否为$a[l]$或者$a[r]$。 如果我们选择$a[l]$，那么之前的状态是$dp[l+1][r]$; 同样选择$a[r]$导致$dp[l][r-1]$。 
5. 键转换将所选端点乘以全局顺序中已放置在其之前的元素数量，这等于$n - (r - l)$。 这个因素反映了我们对最终序列重建的深度。 
6. 更新$dp[l][r]$通过取两个选择之间的最大值，每个选择将子问题解决方案与所选端点的正确加权贡献相结合。 

### 为什么它有效

 在任何时刻，剩余的段都表示内部顺序尚未固定的一组元素。 每个 DP 转换都会分配一个元素作为最终序列中的下一个元素，这唯一地确定其乘数。 因为左右删除的每个有效序列都对应于通过这些间隔缩减的一条路径，并且每个这样的路径都被评估一次，所以 DP 探索所有有效序列而不重复。 最优子结构成立，因为一旦选择端点作为下一个元素，剩余的问题是独立的，并且仅取决于缩减的间隔。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    if n == 1:
        print(a[0])
        return

    dp = [[0] * n for _ in range(n)]

    for i in range(n):
        dp[i][i] = a[i] * n

    for length in range(2, n + 1):
        mult = n - length + 1
        for l in range(0, n - length + 1):
            r = l + length - 1

            dp[l][r] = max(
                dp[l + 1][r] + a[l] * mult,
                dp[l][r - 1] + a[r] * mult
            )

    print(dp[0][n - 1])

if __name__ == "__main__":
    solve()
```DP 是按区间长度自下而上构建的。 倍增因子`mult`对应于最终饮酒顺序中的位置：当间隔大小`length`准确地说，仍然是`n - length + 1`杯子已被选择，因此下一个选择的杯子将获得该乘数。 

基本情况为每个单个元素分配其最终乘数$n$，因为它是该子问题中最后剩下的选择。 

转换是对称的：删除左侧或右侧，并根据当前全局步骤添加其贡献，然后继续使用较小的间隔。

 ## 边缘情况

 单个元素输入，例如`1\n7`由基本情况直接处理。 不调用 DP，输出只是元素乘以 1，这也是$n$在那个微不足道的例子中。 

严格递增的数组，例如`1 2 3 4 5`强调算法是否更喜欢平衡端点删除而不是始终选择正确的端点。 DP 在每个时间间隔评估两种可能性，因此它可以正确地平衡早期乘数与未来收益。 

像这样的均匀数组`5 5 5 5 5`测试解是否退化为正确的算术结构。 每个配置都会产生应用于相同值的相同多组乘法器，因此任何有效的 DP 路径都会产生相同的结果，从而确认转换逻辑的一致性。
