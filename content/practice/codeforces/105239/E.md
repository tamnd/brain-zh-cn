 ## 解决方案
 ## 问题理解

 我们得到一个代表淹没地形的矩形网格，其中每个单元格都有一个水深值。 旅行者从左上角的单元格开始，想要到达右下角的单元格。 只允许向左、向右、向下移动，因此旅行者可以在一行内自由水平移动，但不能向上移动。 

路径是这些规则下的任何相邻移动序列。 路径的成本定义为沿该路径访问的所有单元（包括两个端点）之间的最大水深。 任务是选择一条最小化最大深度的有效路径。 

网格大小可以大到 500 x 500，因此最多有 250,000 个单元。 每个单元格值可以大到 10^9，这排除了任何依赖于以简单方式迭代所有可能阈值或显式探索所有路径的方法。 任何试图枚举路径的算法都是立即不可行的，因为即使是中等网格，由于行内的自由左右移动，也有指数级的许多可能的路线。 

一个微妙的方面是水平移动会在一行内产生循环。 尽管垂直运动严格向下，但在向下之前，您可以在同一行中向左和向右任意行走多次，因此该图不是无环的。 

天真的思维的一个失败案例是假设你可以贪婪地移动到局部最小的邻居。 例如，在每个步骤中选择最小的相邻深度可能会将您困在一行中，从而迫使您稍后出现较大的垂直下降。 

另一个失败案例是假设仅向右和向下移动就足够了。 因为允许向左移动，所以有时您需要向左移动才能到达更好的垂直进入点。 

## 方法

 直接的暴力破解思想是枚举从开始到结束的所有有效路径并计算每条路径上的最大值，然后取这些最大值中的最小值。 虽然概念上是正确的，但路径的数量增长得非常快，因为每一行的行为都像一条相连的走廊：在下降之前你可以用多种不同的方式穿过它。 在 500 x 500 的网格中，即使在考虑每条路径评估成本高达 O(nm) 之前，这也远远超出了任何可行的计算。 

关键的观察是我们不关心确切的路径，只关心阈值 T 是否可行。 如果我们固定一个值T并且只允许踩到深度至多为T的单元格，问题就变成了导出图中(1,1)是否可以到达(n,m)。 这将问题转化为单调可行性检查：如果 T 存在一条路径，那么对于任何更大的 T 也存在该路径。 

这种单调性允许有两种标准解决方案。 一种是对 T 进行二分搜索并结合使用 BFS 或 DFS 的可达性检查。 另一种是更直接的表述，即极小极大最短路径问题，其中每个节点的权重等于其单元值，路径成本是沿路径的最大权重。 该版本可以通过 Dijkstra 来解决，将到节点的距离视为迄今为止遇到的最小可能最大值。 

两种观点是等价的，但 Dijkstra 避免了二分查找中额外的对数因子，并且更直接地表达结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有路径进行暴力破解 | 指数| O(nm) 递归堆栈 | 太慢了|
 | 二分查找+BFS | O(nm log 10^9) | O(nm log 10^9) | O(纳米) | 已接受 |
 | 极小极大迪杰斯特拉 | O(nm log(nm)) | O(nm log(nm)) | O(纳米) | 已接受 |

 ## 算法演练

 我们在网格上使用 Dijkstra，其中状态是一个单元，距离表示从一开始在任何路径上遇到的最小可能的最大深度。

1. 初始化一个距离矩阵，其中每个值都是无穷大，除了设置为自己的深度的起始单元格之外。 这表示任何有效路径都必须包含起始成本。 
2. 将起始单元推入按当前路径成本排序的优先级队列。 队列总是以迄今为止最小的已知最大深度扩展状态。 
3. 从队列中提取当前成本最小的单元。 该成本代表了到达该单元的最佳已知方法，同时最小化遇到的最大深度。 
4. 对于从当前单元（左、右或下）可到达的每个邻居，计算等于当前路径成本和邻居单元深度的最大值的候选成本。 该模型模拟了路径成本由迄今为止看到的最差单元决定的事实。 
5. 如果该候选成本改善了该邻居先前记录的距离，则更新它并将该邻居推入优先级队列。 
6. 继续，直到处理完所有可达状态。 答案是右下角单元格的最终距离值。 

使用优先级队列的关键原因是，一旦单元以最小的可能最大值弹出，后面的路径就无法改进它，因为任何替代路径的成本至少都一样大。 

### 为什么它有效

 在每一步中，该算法都保持不变，即优先级队列包含到达前沿单元的最佳已知方法，并按其瓶颈值排序。 当一个单元被最终确定（弹出为最小值）时，它的值是到达它的所有有效路径中最小的可能最大深度。 这是成立的，因为到该单元的任何替代路径要么包含更大的中间值，要么由于队列的排序而已经以更好或相等的成本更早地探索过。 转移成本的单调结构（取最大值）确保了延长路径永远不会降低其成本，这正是 Dijkstra 式正确性所需的条件。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(n)]

    INF = 10**18
    dist = [[INF] * m for _ in range(n)]

    dist[0][0] = grid[0][0]
    pq = [(dist[0][0], 0, 0)]

    dirs = [(0, 1), (0, -1), (1, 0)]

    while pq:
        cost, x, y = heapq.heappop(pq)

        if cost != dist[x][y]:
            continue

        if x == n - 1 and y == m - 1:
            print(cost)
            return

        for dx, dy in dirs:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < m:
                ncost = max(cost, grid[nx][ny])
                if ncost < dist[nx][ny]:
                    dist[nx][ny] = ncost
                    heapq.heappush(pq, (ncost, nx, ny))

solve()
```网格直接存储为整数，距离表跟踪每个单元格的已知瓶颈值。 优先级队列确保我们始终在最小化最大深度方面扩展当前最有希望的路径。 

方向数组精确编码允许的移动：右、左和下。 向上移动被排除在外，这一点至关重要，因为它保留了到达所有状态而无需沿无效方向重新访问行的可行性。 

到达右下单元格时提前退出是安全的，因为 Dijkstra 保证我们第一次弹出它时，我们已经找到了它的最佳值。

 ## 边缘情况

 一种重要的情况是需要水平移动以绕过垂直障碍物。 例如，如果低成本列被阻挡在中间行中，则唯一的方法是绕行到前面的行中的左侧或右侧。 该算法自然地处理这个问题，因为 Dijkstra 对称地探索所有水平转换，因此它会在尝试向下移动之前将最佳瓶颈值传播到整个可到达的行段。 

另一种情况是最佳路径需要从另一侧重新访问行位置。 即使由于左右移动而存在循环，优先级队列也确保忽略任何具有更差成本的重复访问。 这可以防止无限循环，同时仍然允许必要的水平探索。 

最后一种情况是，最佳答案位于只有通过单个高值门单元后才能到达的区域。 该算法正确地将该门值分配为所有下游路径的瓶颈，因为每个候选路径都必须在某个点合并它，并且最大操作在所有扩展中保留它。
