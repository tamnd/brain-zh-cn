 ## 解决方案
 ## 问题理解

 每次商店之旅都会为巴特鲍尔提供两种独立的物资：一些米饭和一些鸡胸肉。 在所有旅行中，他都希望准备固定份量的饭菜$k$克，但规则限制了成分的组合方式。 

单餐必须准确消耗$k$克，并且所使用的每一克必须来自所购买的库存之一。 关键的限制是混合只是“部分全局的”：在一次旅行中，大米和鸡肉作为一个包装捆绑在一起，但在不同的旅行中，相同的成分类型是可以分开的。 

正式来说，对于每次旅行$i$，我们有一对$(a_i, b_i)$。 每次旅行中，米饭可以独立于同一旅行中的鸡肉使用，但任何膳食都以三种方式之一形成。 要么我们从同一次旅行中同时获取米饭和鸡肉，将该旅行的贡献分成$x + y = k$。 或者我们完全用米饭做一顿饭，可能会在不同的旅程中混合米饭部分。 或者我们完全用鸡肉制成一顿饭，将不同旅程中的鸡肉结合起来。 

限制是，一旦用米饭和鸡肉“混合”一顿饭，它就必须来自一次旅行。 当我们决定在旅行中混合来源时，我们就仅限于单一成分类型。 

所以每顿饭要么是“一趟混饭”，要么是“纯米饭餐”，要么是“纯鸡肉餐”。 

输出要求我们可以形成不相交膳食的最大数量，消耗资源而不重复使用任何克。 

界限$n \le 500$和$k \le 500$表明涉及对总使用重量或每次行程贡献进行伪多项式动态规划的解决方案是可行的。 的大值$a_i, b_i \le 10^9$立即暗示我们不能明确地迭代克； 一切都必须被压缩成由残差或容量索引的状态，最多$k$，不是实际数量。 

一个微妙的边缘情况是，贪婪策略试图在每次旅行中总是吃“最好的”正餐。 例如，如果一次旅行有$(k, 0)$另一个有$(0, k)$，贪婪的思维可能会建议立即吃两顿饭。 这在这里可行，但一旦需要跨行程部分重用，它就会失败。 当行程总重量足够时，会出现另一种失败情况$a_i + b_i \ge k$但最佳分配需要与全球大米或鸡肉分配进行协调，而不是当地决策。 

真正的困难在于，每次旅行都会提供一个带有两种资源的有界背包式结构，但我们可以将它们绑定在一起（在一次旅行中混合）或将它们完全分成两个独立的背包。 

## 方法

 蛮力方法将尝试决定所有行程中的每克大米和鸡肉如何分配到膳食中。 这将有效地将问题视为多维划分问题，最多可达$10^9$规模能力。 即使我们对每次旅行进行离散化，我们仍然需要考虑每次旅行的所有可能的分割方式$(a_i, b_i)$混合餐以及剩余的米饭和鸡肉分配。 这导致每次行程的状态数量呈指数级增长，因为每次行程都可以部分贡献多种组合中的许多餐食。 

关键的简化是分离膳食的结构：每顿饭要么完全在一次旅行中（混合情况），要么完全在跨旅行的单一成分类型中。 这意味着我们不需要跟踪各个行程中精确的克级分配； 在决定每次旅行带多少混合餐后，我们只需要知道我们“留下可用的”米饭和鸡肉的数量。 

一旦我们确定了这种解释，每次旅行都可以被视为提供有限数量的“混合餐”，而剩下的米饭和鸡肉成为独立的资源，有助于两个独立的背包般的大小积累$k$。 

关键的观察是，由于每顿饭都消耗了精确的$k$，每个资源的状态仅与模相关$k$。 这将每个维度最多减少为$k$有意义的状态。 

然后，我们对旅行进行动态规划，跟踪我们可以用米饭、鸡肉形成多少份正餐，以及我们在旅行中已经承诺了多少份混合餐。

 ## 算法演练

 我们维护一个 DP 表，其中状态代表我们以余数模数结转多少未使用的大米和鸡肉$k$。 每次过渡都会处理一次旅行，并决定从中获取多少混合餐，同时将剩菜分配到全球大米和鸡肉池中。 

1. 初始化一个DP数组，其中$dp[r][c]$表示迄今为止获得的最大正餐数量，其中$r$克米渣和$c$克鸡肉残渣仍可用于制作未来的膳食。 初始状态是$dp[0][0] = 0$。 
2. 处理每一次行程$(a_i, b_i)$依次。 对于每个现有状态，考虑从这次旅行中提取混合膳食的所有可行方法。 混合餐消耗$x$大米和$y = k - x$来自同一次旅行的鸡。 既然两者$x$和$y$必须是整数并受可用资源限制，$x$范围从$0$到$k$，但只有有效的分割尊重$x \le a_i$和$y \le b_i$被考虑。 
3. 每份混合餐选择$t$，我们相应地减少旅行的可用资源，留下$a_i - x \cdot t$大米和$b_i - y \cdot t$鸡。 这些剩菜并没有丢失； 相反，它们被添加到全局池中，并有助于稍后形成纯米饭或纯鸡肉餐。 
4、混合分配后，我们将剩下的米饭和鸡肉转化为形成充分的贡献$k$单一类型的大小餐。 我们跟踪这些贡献的模数$k$，因为只有剩余部分对未来的完成很重要。 
5. 我们通过将之前的残差状态与行程中的新残差相结合来更新 DP 转换，并且计算当残差总和跨越$k$。 
6. 处理完所有行程后，答案是所有 DP 状态的最大值。 

### 为什么它有效

 该算法保持了所有已形成的膳食都已完成并从系统中删除的不变性，而所有剩余资源仅通过剩余量表示$k-1$米饭和鸡肉。 因为每顿新餐都需要准确$k$单一类型或单程混合物的单元，具有相同残基的任何配置都会导致相同的未来可能性。 这确保了与最优未来决策相关的信息不会丢失，并且所有有效的资源分解都是通过有界状态空间上的 DP 转换来探索的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    trips = [tuple(map(int, input().split())) for _ in range(n)]

    dp = [[-10**18] * k for _ in range(k)]
    dp[0][0] = 0

    for a, b in trips:
        ndp = [[-10**18] * k for _ in range(k)]

        for r in range(k):
            for c in range(k):
                if dp[r][c] < 0:
                    continue

                base = dp[r][c]

                # try number of mixed meals from this trip
                for x in range(k + 1):
                    y = k - x
                    if x > a or y > b:
                        continue

                    rem_a = a - x
                    rem_b = b - y

                    nr = (r + rem_a) % k
                    nc = (c + rem_b) % k

                    add_full = (r + rem_a) // k + (c + rem_b) // k

                    ndp[nr][nc] = max(ndp[nr][nc], base + add_full)

        dp = ndp

    ans = 0
    for r in range(k):
        for c in range(k):
            ans = max(ans, dp[r][c])

    print(ans)

if __name__ == "__main__":
    solve()
```该实现在剩余的大米和鸡肉模上保持二维 DP$k$。 嵌套转换将旅行的所有可能拆分迭代为混合餐，这是有效的，因为每个混合餐对应一个唯一的整数分区$x + y = k$。 对于每个配置，剩余资源被折叠到剩余更新中，并且任何完整的大小组$k$立即转化为成品餐。 

一个常见的实施陷阱是忘记在计算正餐之前，必须将剩菜与之前的残留物合并； 单独进行转换会破坏正确性，因为跨状态的进位会丢失。

 ## 边缘情况

 当单次行程有足够的总重量但没有有效的拆分时，就会出现一种边缘情况。 对于像这样的输入$k=5$,$(3,1)$，该算法正确地避免了形成混合餐，因为没有整数$x$满足两者$x + y = 5$和资源限制。 DP 转换只是跳过所有无效的分割，使残基保持不变。 

另一种情况是，最佳解决方案需要在一种类型中留下残留物，以便将来完成。 例如，如果早期消费产生 2 米的剩余量，而稍后的行程提供 3 米，则 DP 会将剩余量向前推进，并且仅在总和达到 5 时才转换为全餐。模数跟踪可确保保留此积累，而不会失去未来的可组合性。 

最后的边缘情况是所有行程都相同并且相对于$k$。 DP 永远不会积累足够的残渣来形成一顿饭，并且所有状态都保持为零，这与正确的输出相匹配，因为没有有效的$k$-分区以任意组合存在。
