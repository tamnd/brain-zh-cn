 ## 解决方案
 ## 问题理解

 我们被要求计算有多少种方法可以完全覆盖高度的矩形网格`m`和极大的宽度`n`使用 1×2 多米诺骨牌。 每个多米诺骨牌在水平或垂直方向上恰好覆盖两个相邻的单元格，并且网格的每个单元格必须恰好覆盖一次。 如果存在至少一个单元，其中覆盖该单元的多米诺骨牌在两个平铺之间不同，则认为两个平铺是不同的。 

关键的结构细节是`m`非常小，最多 6 个，而`n`可以大到 10^18 天文数字。 这立即排除了任何在线性时间内逐一处理列的方法`n`，因为即使是 O(n) 也是不可能的。 唯一可行的方向是压缩问题，以便减少对问题的依赖`n`变为对数或通过周期性或矩阵求幂完全消除。 

一个微妙的限制是平铺对相邻列之间的边界相互作用敏感。 单个垂直多米诺骨牌跨越同一列中的两行，而水平多米诺骨牌跨越两列，这意味着不能独立于下一列确定一列的状态。 这种依赖性迫使我们对部分列填充状态进行建模。 

边缘情况出现时`m = 1`,`m = 2`，和小`n`，其中结构退化并且朴素的状态推理可能会意外地多计数或少计数：

 当`m = 1`，唯一可能的平铺是水平多米诺骨牌，所以答案是 1 如果`n`为偶数，否则为 0。 假设存在垂直放置的天真 DP 将错误地引入无效状态。 

什么时候`m = 2`，问题简化为类似斐波那契的转换，但粗心的实现经常忘记垂直多米诺骨牌占据单列中的两行，而水平多米诺骨牌将约束传播到下一列。 

对于较大的`m ≤ 6`，状态空间呈指数增长`m`，但仍然可以管理，因为 2^6 = 64。 

## 方法

 直接的暴力策略将尝试逐列平铺网格，在每一步尝试放置多米诺骨牌。 在任何时候，我们都会跟踪当前边界中的哪些单元格已经被从前一列延伸出来的多米诺骨牌占据。 对于每种状态，我们递归地尝试使用垂直和水平放置的所有方法来填充剩余的空单元格。 

这种方法是正确的，因为它显式枚举了每个有效的平铺。 然而，它的成本会爆炸式增长，因为每列最多`2^m`占用模式和状态之间的转换涉及探索列配置的所有平铺，其本身呈指数分支`m`。 超过`n`列，这会导致大约 O(n × m 中的指数) 运算，当`n`达到10^18。 

关键的观察结果是，该过程仅取决于当前列的占用模式，而不是完整的历史记录。 一旦我们将列编码为大小的位掩码`m`，从一个列状态到下一列状态的转换是固定的。 这将问题转化为对最多 64 个节点的有限有向图中的游走进行计数。 从一个列掩码转换到另一个列掩码的方式数量可以使用单个列上的 DFS 来预先计算。 

一旦我们有了这个转换图，我们就可以有效地计算长度-`n`在图中行走，这是一个标准的矩阵求幂问题。 邻接矩阵最多为 64×64，因此 O(64^3 log n) 中的求幂很容易实现。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力 DFS 平铺 | O(n × m 指数) | O(2^m) | O(2^m) | 太慢了|
 | Profile DP + 矩阵求幂| O(2^{3m} log n) | O(2^{3m} log n) | O(2^{2m}) | O(2^{2m}) | 已接受 |

 ## 算法演练

 我们逐列处理网格，并使用大小的位掩码对每个列边界处的“待定多米诺骨牌的形状”进行编码`m`。 位设置为 1 意味着该单元已被来自前一列的水平多米诺骨牌占据。 

我们首先计算掩码之间的所有有效转换。 从面具的转变`a`掩盖`b`意思是：从一些已经填充的单元格开始`a`，我们可以使用垂直多米诺骨牌和水平放置来完全填充当前列，从而准确产生下一列的占用`b`。 

为了计算转换，我们对单列中的行运行 DFS。 

1.我们从口罩开始`a`表示当前列中预填充的单元格和空的下一个掩码`b`。 我们从上到下扫描行，始终选择第一个未填充的行。 
2.如果当前行已经被填写`a`，我们移动到下一行。 这反映出该单元已经被来自前一列的多米诺骨牌占据。 
3. 如果当前行未填充，我们尝试在当前列内放置一个垂直多米诺骨牌，填充该单元格和下一行单元格。 仅当下一行存在且未填充时才有效`a`。 此选择不会影响下一列掩码。 
4. 我们还尝试放置一个水平多米诺骨牌，它占据当前单元格并延伸到下一列。 这标记了相应的位`b`，因为下一列中的单元格已预先填充。 
5. 当我们到达处理完所有行的列末尾时，我们记录从`a`到`b`。 

这给了我们一个转移矩阵`T`尺寸的`S × S`在哪里`S = 2^m`。 

一旦构建了转换，我们将每一列解释为将此转换应用于状态向量。 最初，第一列没有传入水平多米诺骨牌，因此开始状态为掩码0。处理后`n`列，我们希望返回掩码 0，因为多米诺骨牌不应超出网格边界。 

然后我们计算`T^n`使用二进制求幂并读取从状态 0 到状态 0 的条目。 

### 为什么它有效

 每个平铺都可以唯一地分解为一系列列状态，描述每个边界上哪些单元已被占用。 DFS 精确地枚举了与给定边界条件一致的所有局部填充，因此转换矩阵捕获了所有有效的列到列的演化。 由于状态完全捕获了扩展平铺所需的所有信息，因此不会有两个不同的历史以影响未来选择的方式合并到同一状态，从而在位掩码上形成 DP 马尔可夫。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def mat_mul(A, B):
    n = len(A)
    res = [[0] * n for _ in range(n)]
    for i in range(n):
        Ai = A[i]
        Ri = res[i]
        for k in range(n):
            if Ai[k]:
                aik = Ai[k]
                Bk = B[k]
                for j in range(n):
                    Ri[j] = (Ri[j] + aik * Bk[j]) % MOD
    return res

def mat_pow(M, e):
    n = len(M)
    res = [[0] * n for _ in range(n)]
    for i in range(n):
        res[i][i] = 1
    base = M
    while e > 0:
        if e & 1:
            res = mat_mul(res, base)
        base = mat_mul(base, base)
        e >>= 1
    return res

def build_transitions(m):
    S = 1 << m
    trans = [[0] * S for _ in range(S)]

    def dfs(row, cur_mask, next_mask, m):
        if row == m:
            trans[cur_mask][next_mask] += 1
            return

        if cur_mask & (1 << row):
            dfs(row + 1, cur_mask, next_mask, m)
            return

        if row + 1 < m and not (cur_mask & (1 << (row + 1))):
            dfs(row + 2, cur_mask, next_mask, m)

        dfs(row + 1, cur_mask, next_mask | (1 << row), m)

    for mask in range(S):
        dfs(0, mask, 0, m)

    return trans

def solve():
    m, n = map(int, input().split())
    trans = build_transitions(m)
    Tn = mat_pow(trans, n)
    print(Tn[0][0] % MOD)

if __name__ == "__main__":
    solve()
```该解决方案的核心是过渡构建器。 它修复了起始掩码并枚举了合法填充一列的所有方法，同时生成了下一个掩码。 递归通过跳过预填充的单元格、放置垂直多米诺骨牌或放置有助于下一个状态的水平多米诺骨牌，仔细确保每个单元格仅使用一次。 

然后矩阵求幂组成这些列转换`n`次。 恒等初始化确保在处理任何列之前我们处于干净的空边界状态，并且在恰好之后`n`步骤我们必须返回到空以获得有效的完整平铺。 

一个微妙的点是乘法顺序很重要`mat_mul`，因为我们正在处理通过转换隐式演化的行向量。 该实现被编写为应用于状态转移矩阵的标准矩阵幂。 

## 工作示例

 ### 示例 1：m = 2，n = 3

 我们跟踪尺寸为 2 的掩模：`00, 01, 10, 11`。 

我们从状态向量开始：

 | 步骤| 00 | 00 01 | 10 | 10 11 | 11
 | ---| ---| ---| ---| ---|
 | 初始化| 1 | 0 | 0 | 0 |

 计算转换并应用一列后，我们得到中间分布，3 个步骤后，我们返回到状态 00，值为 3，对应于 2×3 的标准斐波那契平铺计数。 

该跟踪证实水平多米诺骨牌传播正确地承载了跨列的约束。 

### 示例 2：m = 1，n = 4

 各州是`0`和`1`，但仅`0`作为边界状态是有效的，因为没有垂直的多米诺骨牌适合。 

转换强制每两列仅进行一次有效移动。 

| 步骤| 0 |
 | ---| ---|
 | 初始化| 1 |
 | 1 | 0 |
 | 2 | 1 |
 | 3 | 0 |
 | 4 | 1 |

 最终答案是 1，因为 1×4 正好有一个平铺。 

这表明该算法通过转换结构自然地消除了无效的奇偶校验状态。

 ## 边缘情况

 对于`m = 1`，该算法简化为两种状态系统，其中垂直移动永远不可能。 DFS 转换构建器将仅生成与水平配对相邻单元相对应的转换，因此仅偶数`n`贡献非零路径。 根据输入运行`1 4`，初始掩码为`0`，并且重复的转换正确地循环回到`0`，恰好生成一个有效的平铺。 

为了`m = 2`，DFS正确生成经典的多米诺骨牌递归结构。 天真的实现经常忘记放置水平多米诺骨牌会在下一个列掩码中创建依赖关系，但这里它被显式编码为`next_mask`。 这确保了诸如交错水平放置之类的配置不会被重复计算。 

对于较大的`m = 6`，状态空间是最大的，但仍然是完全枚举的。 DFS 保证不会将无效的部分填充带入转换，因为每一行要么立即填充，要么一致地传递到下一个状态。
