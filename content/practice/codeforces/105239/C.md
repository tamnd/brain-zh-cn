 ## 解决方案
 ## 问题理解

 我们有一棵树，其中每个节点都带有两个属性：权重和颜色。 从这棵树中，我们想要选择具有两个同时限制的顶点子集。 首先，树中没有两个选定的顶点可以相邻，因此选定的集合必须是独立的集合。 其次，我们不允许选择两个具有相同颜色的顶点，即使它们在树中相距很远。 

在满足这两个约束的所有子集中，我们想要总权重最大的一个。 

输入大小告诉我们树最多有 1000 个顶点和最多 10 种不同的颜色。 该组合足够小，我们可以在颜色子集或顶点子集上提供动态编程解决方案，但又足够大，不可能枚举所有有效的顶点子集，因为子集的数量以 n 为指数。 

尝试所有顶点子集的简单方法最多会考虑 2^1000 种可能性，这是完全不可行的。 即使通过独立性或颜色约束进行修剪也没有足够的帮助，因为检查有效性本身对于每个子集都是线性的，从而导致天文数字般的复杂性。 

一个更微妙的问题来自于约束之间的相互作用。 一个常见的错误是独立处理颜色，解决每种颜色的独立设置问题或贪婪地选择每种颜色最重的顶点。 这会失败，因为邻接耦合了不同的颜色：选择一种颜色的重顶点可能会阻挡其邻域中其他颜色的多个候选者。 

另一个错误的方向是计算树上的最大权重独立集，然后简单地强制执行不同的颜色。 这也会失败，因为最佳独立集可能包含相同颜色的多个顶点，并且随后删除重复项并不是局部最优的。 

## 方法

 关键的困难在于独立性约束是树上的结构性约束，而颜色约束是所选节点的全局约束。 颜色数量较少表明应该由颜色而不是顶点来驱动状态空间。 

强力的观点是考虑选择顶点，同时确保没有选择两个相邻节点并且没有颜色重复。 这自然会导致探索顶点子集并验证它们。 在具有 n 个节点的树上，仅独立集的数量就已经是指数级的，并且添加颜色唯一性并不能以受控方式减少组合爆炸。 当 n 增长到超过 30 或 40 左右时，这种方法就会失败。 

主要观察结果是，除了邻接之外的唯一全局限制是“每种颜色最多一个顶点”。 由于 C ≤ 10，我们可以将每种颜色视为一个选择维度。 对于每种颜色，我们最多可以选择该颜色的一个顶点，或者不选择任何顶点。 因此，我们不是考虑顶点，而是考虑为每种选定的颜色分配一个代表顶点。 

然而，毗邻性仍然很重要。 如果我们选择两种颜色，例如红色和蓝色，并选择每种颜色的一个顶点，我们必须确保这两个顶点在树中不相邻。 这表明颜色子集上的 DP，其中过渡确保所选代表之间的兼容性。 

我们将树的根设在任意节点。 对于每个节点，我们考虑 DP 状态来编码哪些颜色已经“在当前子树选择中使用”，但更简洁的公式是使用 DP 来处理树，对于每个节点，无论我们是否选择该节点，都会跟踪其子树中的最佳可能选择。 由于颜色是全局的，我们改为聚合每种颜色的候选者，并通过树 DP 与颜色位掩码相结合来强制非邻接。

解决这两个约束的简洁方法是将问题减少为每种颜色最多选择一个节点，并确保所选节点形成独立的集合。 由于颜色很少，我们可以首先按颜色对节点进行分组。 对于每种颜色，我们最终最多会选择一个节点，因此我们预先计算了不同颜色的任意两个候选节点之间的兼容性：它们在树中不能相邻，但更重要的是，无论子树结构如何，它们都必须保持不相邻。 

然后我们观察到，由于该结构是一棵树并且 n 很小，因此我们可以对节点和颜色子集进行 DP，其中我们确保在任何选择中，所选节点都是成对不相邻的。 这相当于求解最大权重独立集，并附加一个约束，即从每个颜色类中最多选择一个节点。 这可以通过树上的 DP 来处理，其中状态是当前部分独立集中已使用的颜色的位掩码。 

当我们为树建立根时，我们对独立集进行标准树DP，但使用大小为C的颜色位掩码来增强状态。对于每个节点，我们决定是否接受它，并且当接受它时，我们在掩码中设置它的颜色。 转换结合了子 DP 表，同时确保父子之间的独立约束。 

这是有效的，因为树 DP 自然地处理邻接约束，而位掩码跟踪全局颜色唯一性。

 ## 算法演练

 我们以节点 1 为树根并执行基于 DFS 的动态规划。 

1. 我们为每个节点定义一个 DP 表，其中 dp[u][mask] 存储 u 的子树中可实现的最大权重（如果该子树中所选颜色的集合与 mask 完全对应）。 掩码有 C 位，因此它代表哪些颜色已被使用。 
2.对于每个节点u，我们首先假设u没有被选择来初始化DP。 在本例中，dp[u] 从 dp[u][0] = 0 开始，这意味着该节点尚未使用颜色。 
3. 我们处理u的每个子v并将其DP表合并到u的DP中。 此合并是通过组合 dp[u] 和 dp[v] 之间的所有兼容掩码来完成的。 这里的兼容性意味着颜色集必须保持不相交。 组合时，我们尝试所有掩码对并添加它们的权重。 
4. 合并所有不带u的孩子后，我们考虑选择u的情况。 如果我们选择 u，则其颜色不得出现在任何子选择中，因此我们只允许不包含颜色 c[u] 的蒙版。 我们设置 w[u] 的候选贡献，然后在没有一个孩子可以使用 u 的颜色的约束下合并孩子。 
5. 对于每个孩子，当u被选择时，我们必须确保孩子子树不会选择与u相邻的节点，这会违反独立性。 由于邻接仅存在于父级和子级之间，因此选择 u 会强制所有子级避免选择任何可能发生冲突的节点，但在树 DP 中，这是通过排除子级也选择 u 位置的配置来处理的。 
6. 最后，对于节点 u，我们取所有 dp[u][mask] 值的最大值，因为子树的任何有效子集都可以留下一些未使用的颜色。 

### 为什么它有效

正确性取决于两个耦合的不变量。 首先，树 DP 不变量确保来自子项的任何状态组合都对应于一组没有相邻选定节点的顶点，因为邻接仅存在于父边和子边之间，并且我们在合并期间明确避免冲突的选择。 其次，位掩码不变量确保没有两个选定的顶点共享相同的颜色，因为每个过渡只允许合并不相交的颜色掩码，并且选择一个节点会在掩码中永久设置其颜色。 由于每个节点在递归中只处理一次，并且所有合并都保留两个不变量，因此 dp 中表示的任何状态都对应于一个有效子集，并且每个有效子集都可以通过沿着树进行相同的分解来构造。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n, C = map(int, input().split())
    w = list(map(int, input().split()))
    c = list(map(int, input().split()))
    c = [x - 1 for x in c]

    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    FULL = 1 << C

    dp = [None] * n

    def dfs(u, p):
        # dp[u]: dict mask -> best value
        cur = {0: 0}

        for v in g[u]:
            if v == p:
                continue
            dfs(v, u)

            nxt = {}
            dv = dp[v]

            for m1, val1 in cur.items():
                for m2, val2 in dv.items():
                    if m1 & m2:
                        continue
                    nm = m1 | m2
                    if nm not in nxt or nxt[nm] < val1 + val2:
                        nxt[nm] = val1 + val2

            cur = nxt

        # option: take u
        take = {}
        cmask = 1 << c[u]

        for m, val in cur.items():
            if m & cmask:
                continue
            nm = m | cmask
            take[nm] = max(take.get(nm, 0), val + w[u])

        # merge: not take u OR take u
        res = cur
        for m, val in take.items():
            if m not in res or res[m] < val:
                res[m] = val

        dp[u] = res

    dfs(0, -1)
    print(max(dp[0].values()))

if __name__ == "__main__":
    solve()
```该解决方案执行后序遍历。 每个节点维护一个字典，将颜色掩码映射到其子树中可实现的最佳总和。 合并步骤是跨子级的背包式卷积，其中不兼容的颜色掩模被丢弃。 

“获取节点 u”步骤通过检查其颜色是否已存在于累积掩模中来强制执行颜色限制。 如果不是，它会添加节点权重并设置其颜色位。 

最终的答案是所有遮罩中根部的最佳值，因为我们不需要使用所有颜色。

 ## 边缘情况

 一种重要的边缘情况是所有节点共享相同的颜色。 在这种情况下，仅颜色约束就限制我们最多选择一个节点，因此答案就是最大权重节点。 DP 正确地强制执行此操作，因为每个合并都带有相同的单位掩码，从而防止多重选择。 

另一种情况是星形树，其中一个中心节点连接到许多不同颜色的叶子。 如果权重较小，最佳解决方案会避开中心，而是选择多个叶子。 DP 处理这个问题是因为叶子子树干净地合并，而中心引入了邻接约束，阻止同时选择它自己和任何邻居。 

最后一个微妙的情况是高权重节点共享颜色但在树中相距很远。 该算法仍然排除将它们组合起来，因为位掩码约束向上传播，确保无论距离如何，全局每种颜色都只有一个。
