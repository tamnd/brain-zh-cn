 ## 解决方案
 ## 问题理解

 我们在三堆独立的石头上玩了一个小型组合游戏。 每一步都包括选择一堆石子并从中取出一些石子。 移除的石子数量必须属于输入中给出的固定允许集合。 玩家交替走棋，不能做出合法棋步的玩家失败。 

任务是确定在最佳游戏下，由三个堆大小定义的起始位置对于第一个玩家来说是赢还是输。 

虽然有三个桩，但关键的结构特性是桩之间的移动永远不会相互作用。 每堆独立进化，一次移动仅影响状态的一个坐标。 这是公正博弈析取和的经典特征。 

约束非常小：每个堆大小最多为 100，并且移动集大小也受该最大值限制。 这立即排除了对完整状态的任何指数博弈树探索，因为朴素递归将在没有记忆的情况下反复重新访问状态，并且由于边界较小，原则上仍然可行，但鉴于结构是不必要的。 

当堆变得太小而无法执行任何允许的移动时，就会出现微妙的边缘情况。 例如，如果允许的移动为 {3, 5} 并且一堆的大小为 2，则该堆不贡献合法的移动。 如果所有牌堆都处于这种情况，则当前玩家立即失败。 这很重要，因为假设至少一堆始终可玩的简单模拟会错误地继续递归。 

另一种极端情况是当移动集包含 1 时。在这种情况下，每个正堆总是可逐步还原的，这通常会导致 Grundy 值出现周期性或单调行为。 任何假设稀疏转换的不正确 DP 可能仍然有效，但有忽视完整转换结构的风险。 

## 方法

 暴力方法会将每个状态视为由三元组 (n1, n2, n3) 定义的游戏图中的一个节点。 从任何状态，我们枚举所有合法的移动：对于每个桩，减去每个不超过桩大小的允许的ai，并递归地评估结果位置。 如果一个州至少有一项举措导致该州失败，则该州获胜。 

这立即导致状态空间的大小最多为 101 立方，即大约一百万个状态。 每个状态最多有 3k 个转换，其中 k 最多可达 100，因此每个状态大约有 300 个转换。 在最坏的情况下，这会产生大约 3 亿条边，并且简单的递归会在没有记忆的情况下重复重新计算状态，使其速度太慢。 即使有记忆，该结构也表明我们正在重新计算一个标准的公正游戏 DP，这可以更好地系统化处理。 

关键的见解是，这是相同子游戏的不相交总和，每堆一个。 每堆都是一个独立的减法游戏：从一堆大小为 x 的堆中，您可以移动到 x - ai，对于任何允许的 ai ≤ x。 此类游戏是通过计算所有堆大小直至最大值的 Grundy 数来解决的。 

一旦我们计算单堆的 Grundy(x)，完整的游戏状态 (n1, n2, n3) 的 Grundy 值等于三个堆 Grundy 值的异或。 这是根据公平博弈的斯普拉格-格伦迪定理得出的。 当该 XOR 为零时，第一个玩家的起始位置将会失败。 

因此，问题简化为计算单个维度上的标准 DP，然后组合三个值。

 ## 算法演练

 我们首先计算从 0 到 n1、n2 和 n3 中最大可能值的每个堆大小的 Grundy 数。

对于每个值 x，我们查看可以应用的所有移动 ai。 每次移动都会导致一个可达位置 x - ai，从而产生一组可达状态的 Grundy 值。 

x 的 Grundy 值定义为那些可达值中不存在的最小非负整数。 

计算该数组后，我们通过对 Grundy(n1)、Grundy(n2) 和 Grundy(n3) 进行异或来评估最终状态。 如果结果为零，则第一名玩家的位置为负； 否则它就赢了。 

### 为什么它有效

 每堆都是一个独立公正的博弈，因此整个博弈就是它们的析取和。 Sprague-Grundy 定理保证每个这样的游戏都可以分配一个 Grundy 值，并且组件 Grundy 值的 XOR 完全表征了获胜和失败的状态。 DP 构造确保每个 Grundy(x) 正确编码来自该堆大小的所有可能的移动，因此不会省略任何可达选项，并且在 mex 计算中不会包含不可到达的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n1, n2, n3 = map(int, input().split())
    k = int(input())
    moves = list(map(int, input().split()))

    max_n = max(n1, n2, n3)

    grundy = [0] * (max_n + 1)

    for x in range(1, max_n + 1):
        reachable = set()
        for a in moves:
            if a <= x:
                reachable.add(grundy[x - a])
        g = 0
        while g in reachable:
            g += 1
        grundy[x] = g

    result = grundy[n1] ^ grundy[n2] ^ grundy[n3]

    if result == 0:
        print("Vasya")
    else:
        print("Peter")

if __name__ == "__main__":
    main()
```DP 阵列`grundy`存储单个桩大小的 Grundy 值。 对于每个尺寸，我们收集通过减去每个允许的移动而产生的所有可达状态。 mex 计算是通过递增来完成的`g`直到在可达集合中找不到它为止。 这是小规模的，因为值最多以 100 为界。 

最终的异或结合了三个独立的堆。 输出直接取决于 XOR 是否为零。 

## 工作示例

 ### 示例 1

 输入：```
1 1 1
1
1
```我们计算的 Grundy 值最大为 1。 

| x| 可达 | 格伦迪(x) |
 | ---| ---| ---|
 | 0 | {} | 0 |
 | 1 | {0} | 1 |

 现在评价：

 Grundy(1) 异或 Grundy(1) 异或 Grundy(1) = 1 异或 1 异或 1 = 1。 

所以结果是第一个玩家获胜，输出 Peter。 

这证实，仅移除 1 个石子，游戏的行为就像大小为 1 的标准 Nim 堆，并且三个相同的堆会产生非零 XOR。 

### 示例 2

 输入：```
10 10 10
2
3 4
```我们只关心 10 以内的 Grundy。DP 基于减去 3 或 4 来构建值。 

| x| 可从 x | 到达 格伦迪(x) |
 | ---| ---| ---|
 | 0 | {} | 0 |
 | 1 | {} | 0 |
 | 2 | {} | 0 |
 | 3 | {0} | 1 |
 | 4 | {0} | 1 |
 | 5 | {1} | 0 |
 | ... | ... | ... |

 我们根据该表计算 Grundy(10)，并对三个相同的值进行异或。 由于所有堆都是相同的，XOR 会抵消，得到零。 

所以输出是Vasya。 

这证明了对称性：当 Grundy 值匹配时，相同的堆会导致 XOR 下的取消。

 ## 边缘情况

 一个关键的边缘情况是所有桩都为零。 输入是：```
0 0 0
...
```无法移动，因此当前玩家立即失败。 DP 为所有桩分配 Grundy(0) = 0，因此 XOR 为 0，输出为 Vasya，符合失败条件。 

另一种情况是移动集仅包含较大值，例如：```
3 3 3
1
5
```没有堆可以移动，因此对于 x < 5，所有超过零的 Grundy 值仍然为零。计算得出每个堆的 Grundy(3) = 0，并且 XOR 为零。 该算法正确输出 Vasya。 

最后一个微妙的情况是当 k = 1 且 a1 = 1 时。然后每个正堆都有一系列强制向下移动到零。 DP 产生交替的 Grundy 值，并且 XOR 规则仍然根据 Grundy 序列中编码的奇偶校验正确识别获胜状态。
