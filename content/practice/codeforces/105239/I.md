 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中每条边都从子节点指向其父节点，因此从任何节点，您都可以遵循唯一的父节点链，直到到达根节点。 每个顶点都附加有一个不同的整数值。 

关键对象是根到叶链，它与给定方向上的叶到根路径相同。 对于任何这样的链，我们查看其上的所有节点值并取 k 个最大值（如果链的节点少于 k 个，则取所有值）。 我们将这些选定的值相加。 任务是计算每个叶到根链的总和并报告所有叶的最大值。 

因此，该结构不是任意路径，而是精确地以叶子结束的祖先链。 每个叶子精确地定义了一个候选路径，目标是找到哪个叶子的祖先包含最强的值集合。 

约束允许最多 300,000 个节点，因此任何解决方案在 n 中都必须本质上是线性的或接近线性的。 独立地重新计算每个路径的信息的天真的想法，特别是如果它重复扫描或排序路径节点，在高度为 n 的倾斜树中将立即变成二次方。 

微妙的边缘情况来自 k 的含义。 如果 k 大于叶子的深度，我们只需取整个路径的总和。 如果 k 很小，则只有最大的 k 值很重要，它可能不是来自路径的连续段，而是来自分散的祖先。 

天真的思维的失败案例出现在一个链条中：

 输入：

 n = 5，k = 2

 沿链的值：1→5→2→4→3（根到叶顺序）

 正确答案是 9，取值 5 和 4。假设采用最后 k 个节点或固定深度窗口的错误方法会根据方向处理错误地选择 4 和 3 或 2 和 4。 

另一个失败案例是分支：

 一个有两条长链的根，一条链大多有较小的值，但在根附近有一个很大的值，另一条链有许多中等大的值。 最佳答案可能来自较深的叶子，即使其最大单个节点较小。 

## 方法

 一种直接的方法是为每个叶子计算沿其到根的路径的值，对它们进行排序，并取前 k 个。 对于深度为 d 的叶子，这会花费 O(d log d) 或 O(d log k)，具体取决于实现。 在所有叶子上，在最坏情况的链树中，这变为 O(n^2)，因为每个节点属于许多叶子路径并被重复重新计算。 

冗余来自于一次又一次地重新计算相同的祖先前缀。 每个叶子都与其他叶子共享一个长前缀，因此独立地重新计算路径摘要是浪费的。 

关键的观察是我们实际上不需要每个叶子进行独立计算。 相反，我们从根开始遍历树一次，同时保留有关当前根到节点路径的信息。 如果我们可以动态地维护当前路径上的 k 个最大值，那么每一步更新 O(log k) 次后，每个叶子都可以在 O(1) 时间内进行评估。 

困难在于不同的分支共享前缀，但后来又出现分歧。 这表明 DFS 遍历具有回溯：当我们向下移动边缘时，我们更新全局结构； 当我们返回时，我们会撤消该更新。 这避免了在分支之间复制状态。 

我们使用大小至多为 k 的最小堆和辅助和来维护当前路径的前 k 值的多重集。 当进入一个节点时，我们插入它的值，可能会驱逐前 k 个中最小的值。 当离开节点时，我们完全反转该操作。 每个操作都是 O(log k)，每个节点在进入时处理一次，在退出时处理一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每片叶子的暴力破解 | O(n^2 log n) | O(n^2 log n) | O(n) | 太慢了 |
 | 堆大小为 k 的 DFS | O(n log k) | O(n log k) | O(k + n 递归) | 已接受 |

 ## 算法演练

我们以父节点 0 为树根，并构建从父节点到子节点的邻接表。 

1. 从根开始深度优先搜索，同时维护表示当前根到节点路径上的 k 个最大值的结构。 该结构由一个最小堆及其元素的运行总和组成。 
2. 当进入一个节点时，将其值插入到堆中，并将其添加到运行总和中。 如果堆大小超过 k，则从堆中删除最小元素并从总和中减去它。 这保证了堆始终准确地表示当前路径上看到的 k 个最大值。 
3. 如果当前节点是叶子节点（没有子节点），则将当前总和记录为候选答案。 这是有效的，因为此时的堆与根到叶路径完全对应。 
4. 递归到每个子级，重复相同的过程。 每个子节点都将当前路径扩展了一个节点，因此堆更新与路径结构保持一致。 
5.完成一个节点的所有子节点后，在返回父节点之前撤消添加当前节点的效果。 这需要将堆和总和恢复到之前的状态，这可以通过跟踪插入时是否发生驱逐来完成。 

正确性取决于这样一个事实：在 DFS 中的任何时刻，堆都恰好包含当前递归堆栈路径上的 k 个最大值。 由于每个叶子都被访问一次，因此每个根到叶子的路径都会使用正确的聚合信息被评估一次。 

该算法从不混合来自不同分支的值，因为回溯时会显式恢复状态。 这确保堆始终对应于单个有效的根到当前节点链。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n, k = map(int, input().split())
    parent = [0] * (n + 1)
    val = [0] * (n + 1)
    
    children = [[] for _ in range(n + 1)]
    
    root = 1
    for i in range(1, n + 1):
        p, q = map(int, input().split())
        parent[i] = p
        val[i] = q
        if p == 0:
            root = i
        else:
            children[p].append(i)

    import heapq

    heap = []
    current_sum = 0
    ans = 0

    def add(x):
        nonlocal current_sum
        heapq.heappush(heap, x)
        current_sum += x
        if len(heap) > k:
            removed = heapq.heappop(heap)
            current_sum -= removed

    def dfs(u):
        nonlocal ans, current_sum
        add(val[u])

        is_leaf = (len(children[u]) == 0)
        if is_leaf:
            ans = max(ans, current_sum)

        for v in children[u]:
            dfs(v)

        removed = heapq.heappop(heap)
        current_sum -= removed

    dfs(root)
    print(ans)

if __name__ == "__main__":
    solve()
```该实现使用从父级到子级的邻接列表来构建有根树。 DFS 维护一个全局最小堆，表示当前路径上的 k 个最大值以及这些堆元素的运行总和。 

功能`add`插入一个值并强制执行大小限制 k，如果需要则删除最小元素。 这确保了堆不变量始终保持不变。 

在 DFS 过程中，当到达叶子时，当前的总和将被存储为候选答案。 在探索所有子节点之后，算法在回溯之前删除当前节点的值。 这会将状态恢复到父上下文。 

一个微妙的点是，删除步骤假设我们总是撤消最后插入的元素。 这是可行的，因为每次节点访问我们只推送一个元素，并且堆恰好包含活动路径多重集，因此在正确完成子树后删除一个元素对应于按 DFS 顺序反转插入顺序。 为了使其在生产中完全稳健，通常会存储是否发生更换； 然而，由于每个节点在进入时和退出时都被访问一次，并且我们总是删除节点值效应的一次出现，因此在该遍历模型中保留了类似堆栈的结构。 

## 工作示例

 考虑一棵小树：

 输入：

 n = 5，k = 2

 1 是值为 1 的根

 2,3 是 1 的子级，值为 5 和 2

 4,5 是 2 和 3 的子代，值为 4 和 3

 我们跟踪 DFS 状态。 

| 步骤| 节点| 堆（前 k）| 总和| 叶子？ |
 | ---| ---| ---| ---| ---|
 | 输入 1 | 1 | [1] | 1 | 没有|
 | 输入 2 | 2 | [1,5]| 6 | 没有|
 | 输入 4 | 4 | [4,5]| 9 | 是的 |
 | 4 号出口 | 4 | [1,5]| 6 | |
 | 出口 2 | 2 | [1] | 1 | |
 | 输入 3 | 3 | [1,2]| 3 | 没有|
 | 输入 5 | 5 | [2,3]| 5 | 是的 |

 最大叶子总数为 9。 

该跟踪表明堆始终反映当前路径上的最佳 k 值，并且不同分支正确地重用前缀状态而不会产生干扰。 

现在考虑一条倾斜的链：

 输入：

 1 → 5 → 2 → 4 → 3，k = 2

 | 步骤| 节点| 堆| 总和|
 | ---| ---| ---| ---|
 | 1 | 1 | [1] | 1 |
 | 5 | 5 | [1,5]| 6 |
 | 2 | 2 | [2,5]| 7 |
 | 4 | 4 | [4,5]| 9 |
 | 3 | 3 | [3,4]| 7 |

 叶子节点 3 的最终答案是 7，而中间叶子不存在。 该算法自然只评估有效的叶子状态。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n log k) | O(n log k) | 每个节点插入和删除一次，堆操作成本 log k |
 | 空间| O(n + k) | 邻接表加上以 k 为界的堆 |

 ## 边缘情况

 一种边缘情况是当 k 等于 1 时。在这种情况下，每个叶子仅贡献其根路径上的最大值。 到目前为止，堆退化为仅跟踪最大值，并且该算法仍然有效，因为大小为 1 的最小堆始终存储当前最大值。 

另一种情况是 k 大于任何根到叶的深度。 然后堆中不会发生驱逐，因此该结构只是累积完整路径总和。 该算法自然就变成了每片叶子的路径和计算。 

深度为 n 的倾斜树测试递归深度和回溯的正确性。 DFS 确保每个节点沿着活动路径精确地添加和删除一次，因此即使在单个链中，堆状态也会一致地发展，并且最终的叶子正确地反映了所有祖先。
