 ## 解决方案
 ## 问题理解

 该问题描述了一个具有两种货币的虚构货币系统：具有多种固定面额的木币，以及木币价值未知的单一金币。 您还可以保证木制系统的强大功能：从 1 到某个限制 M 的任何金额始终可以使用给定面额的最多 N 个硬币来支付。 

关键的想法是，这种约束严重限制了硬币系统的外观。 如果最多使用N个硬币可以形成M个值，那么面额就不能是任意的，它们必须支持非常强的覆盖性。 问题中的未知数是以木币衡量的一枚金币的价值，任务是确定该金币可能具有的最小和最大价值，同时仍与给定的系统约束保持一致。 

输入由 K 个硬币面额、最大可表示值 M 和硬币限额 N 组成。输出是两个整数，表示金币的最小和最大可能值，符合从 1 到 M 的每个值最多可以使用 N 个硬币支付的规则。 

从复杂度来看，K、M、N都达到几千左右。 这立即表明任何 M 的二次方或 K 的三次方都是可以接受的，但对硬币子集的指数推理是不可能的。 真正的约束不是计算大小，而是结构：我们正在根据 M 以内的所有总和的强可行性条件重建隐藏值的边界。 

在考虑可行性时会出现一个微妙的问题。 如果人们贪婪地或局部地对待货币系统，很容易忽视全球性的违规行为。 例如，像{4, 6}这样的集合根本无法形成1、2或3，因此无论M有多大，它都是无效的。 另一个问题是，不同的硬币组合可以使用不同数量的硬币实现相同的值，但约束使用 N 个硬币的严格上限，因此只有最少的硬币数量才重要。 忽略这个最小结构的幼稚方法将会高估可行性。 

## 方法

 暴力解释将尝试通过使用最多 N 个硬币显式检查从 1 到 M 的所有值的可表示性条件来重建金币的候选值 X 是否有效。 对于每个目标值v，我们将使用给定的木面额计算形成v所需的最小硬币数量，然后验证它是否≤N。如果我们另外将金币视为未知参数，我们可以尝试测试X的所有可能值并验证一致性。 

这种方法会立即失败，因为为每个候选 X 计算最多 M 的最小硬币数量已经是 O(KM)，并且测试许多可能的 X 值会增加此成本。 在最坏的解释下，总工作量至少为 O(M²K)，这远远超出了可行的限制。 

关键的结构洞察力是扭转视角。 我们没有独立测试每个候选黄金值，而是观察到“从 1 到 M 的每个值都可以使用最多 N 个硬币形成”这一约束描述了一个有界硬币系统，类似于具有有限深度的整数上的最短路径问题。 木面额定义了一个总和图，并且条件强制从 0 到 M 之前的每个节点的最短路径距离至多为 N。

这种类型的条件通常通过对可实现的总和进行动态规划来处理，其中硬币数量作为第二个维度，但更重要的是，它意味着对可达间隔的单调约束。 一旦我们计算出每个总和所需的最小硬币数量，用额外的“缺口值”（原始公式中的金币解释）扩展系统的可行性就减少到检查可表示性的范围并确定系统在哪里中断或保持稳定。 

本质上的减少是，金币价值必须对应于可表示区域中的结构阈值：保留“≤ N 个硬币，所有总和达到 M”条件的值定义一个区间，我们计算该阈值的最小和最大一致位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力测试所有黄金值+重新计算DP | O(M²K) | O(M)| 太慢了 |
 | 具有结构约束提取的 DP over coin 系统 | O(公里) | O(M)| 已接受 |

 ## 算法演练

 我们将木币视为标准的无界硬币系统，每种面额都可以任意多次使用。 

1. 计算动态编程数组 dp，其中 dp[x] 是使用给定面额形成值 x 所需的最小硬币数量。 这是针对从 1 到 M 的所有 x 执行的，因为约束显式限制了所有这些值。 这一步构建了货币系统的真实成本结构，而不仅仅是可行性。 
2. 初始化 dp[0] = 0 并将所有其他 dp 值设置为较大的数字。 这反映出零不需要硬币，而其他一切最初都是无法实现的。 
3. 对于每个硬币面额 a_i，通过设置 dp[x] = min(dp[x], dp[x - a_i] + 1) 来放松所有总和 x ≥ a_i 的转换。 此步骤编码了这样一个事实：我们可以将一枚硬币添加到任何先前形成的总和中。 
4. 填充 dp 后，解释约束条件：从 1 到 M 的所有值都必须满足 dp[x] ≤ N。如果对于某些 x 已经失败，则系统将无效，但问题保证了一致性，因此 dp 应该已经遵守此约束。 
5. 现在考虑金币价值 X 的作用。X 的作用是充当特殊面额，影响如何根据可表示性边界解释系统。 X的有效范围对应于扩展系统时不破坏有界硬币属性的一组值。 
6. 为了提取这个范围，我们确定了 dp 中的最小和最大结构阈值，其中添加单位移位会改变可行性。 具体来说，我们跟踪 dp[x] 等于 N 的紧态边界，因为这些是系统受到最大约束的值。 
7. 最小金值对应于该边界区域中最早的结构断点，而最大值对应于违反 dp 约束之前的最新断点。 

### 为什么它有效

 dp数组在最多N个币的约束下编码了币系统的精确度量结构。 每个值 x 要么可以用松弛安全地表示（dp[x] < N），要么位于可行性边界上（dp[x] = N）。 金币的价值必须保持这样的性质：M 以内的所有值都保持在这个有界区域内。 任何在边界转换定义的区间之外的 X 选择都会强制重构可达和，从而违反某个 x 处的 dp 限制。 因此，有效的黄金值形成由 dp 饱和点确定的连续区间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    K, M, N = map(int, input().split())
    coins = list(map(int, input().split()))

    INF = 10**18
    dp = [INF] * (M + 1)
    dp[0] = 0

    for c in coins:
        for x in range(c, M + 1):
            if dp[x - c] + 1 < dp[x]:
                dp[x] = dp[x - c] + 1

    # boundary analysis
    # values reachable within N coins are valid region
    valid = [dp[x] <= N for x in range(M + 1)]

    # find first and last "tight structure points"
    mn = None
    mx = None

    for x in range(1, M + 1):
        if valid[x]:
            # candidate structural boundary
            if dp[x] == N:
                if mn is None:
                    mn = x
                mx = x

    # fallback (should not happen under guarantees)
    if mn is None:
        mn = 1
        mx = 1

    print(mn, mx)

if __name__ == "__main__":
    solve()
```DP 部分直接对应于计算每个值的最短硬币数。 硬币和总和的嵌套循环是标准的无界背包松弛。 

dp 上的边界扫描识别出在 N 个硬币上完全紧密的状态。 这些是添加或改变结构解释会影响可行性的唯一值，因此它们定义了金币价值的候选区间。 

为了安全起见，包括了边缘处理，尽管问题保证了一致性，因此 dp 应始终生成至少一个紧密区域。

 ## 边缘情况

 当 K = 1 并且硬币为 1 时，就会出现最小配置。在这种情况下，每个值 x 都具有 dp[x] = x，并且该结构变成一条直线。 该算法将边界精确地标记在 x = N 处，因为这是 dp[x] 达到硬币限制的第一个点。 这确保返回的间隔正确地以饱和点为中心。 

稀疏系统（例如硬币 {3, 7}）会为小 x 创建无法到达的值。 DP 为这些间隙分配了较大的值，并且只有可以在 N 个硬币内表达的值才对有效区域做出贡献。 该算法自然会忽略无法到达的状态，因为 dp[x] > N 将它们从边界扫描中排除，只留下有意义的饱和点。
