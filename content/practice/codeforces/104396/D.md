 ## 解决方案
 ## 问题理解

 我们得到了平面上的一组点。 每个点代表一颗星星。 为每一位明星$i$，我们想象站在那颗星星前，画一条穿过它的直线。 要求这条线不穿过任何其他恒星。 一旦这条线被画出来，它就会分割所有其他线$n-1$根据星星位于线的哪一侧，将星星分为两组。 

对于固定起始星$i$，我们感兴趣的是有多少种不同的方式可以选择这样一条线，以便两侧之一恰好包含$k$星星。 每条有效行为相应的行提供一个选择$k$。 我们需要的答案是一个矩阵，其中条目$A_{i,k}$计算通过点的线数$i$产生一个侧面$k$点。 

关键的难点是线路不固定。 可以绕点旋转$i$，每个方向都会对剩余点产生不同的分割。 我们正在有效地计算每个点周围有多少个方向分割产生每个可能的边尺寸。 

约束条件意味着$n \le 2000$。 点对的二次方数量已经达到大约$4 \cdot 10^6$，如果小心处理的话，这在 Python 中是没问题的。 任何立方体都太慢了。 这立即表明我们应该独立处理每个中心点，并为每个中心花费线性或接近线性的时间。 

一个微妙的边缘情况是当许多点几乎与参考点共线时。 不仔细处理“半平面边界”的简单角度排序方法可能会重复计数或错过配置。 另一个常见的错误是忘记我们计算开放半平面时严格避开边界线上的点，这迫使我们确保没有其他点恰好位于所选方向上。 

## 方法

 蛮力的想法是固定一个点$i$，然后枚举所有可能的行$i$由配对决定$i$与其他所有点$j$。 每个这样的线方向定义将所有剩余点分成两组，具体取决于它们位于有向线的哪一侧。 对于每个方向，我们计算一侧有多少个点并递增相应的点$A_{i,k}$。 

这是正确的，因为任何有效的分隔线通过$i$必须平行于由$i$另一个点，因为线是由它的方向决定的，我们只关心它如何划分有限的点集。 然而，对于每个固定$i$，这需要$O(n)$方向以及每个方向$O(n)$点的分类，给出$O(n^3)$总操作，这对于$n = 2000$。 

关键的观察是对于固定中心$i$，重要的只是周围所有其他点的角度顺序$i$。 如果我们对所有向量进行排序$i \to j$通过极角，则任意半平面对应于长度小于的连续弧$180^\circ$在此循环订购中。 我们可以使用两指针扫描来维护从每个方向开始的半圆窗口内有多少个点，而不是重新计算每个方向的点数。 这将每个中心的成本降低到$O(n \log n)$用于排序加$O(n)$用于扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了|
 | 角度扫掠|$O(n^2 \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 对于每个点$i$，我们将其视为原点并分析与它相关的所有其他点。 

1.对于固定的$i$，计算向量$i$到其他所有点$j$。 每个向量都有一个角度$[0, 2\pi)$。 这将几何分裂问题转换为角度排序问题。 
2. 将所有这些向量按极角排序。 排序顺序代表围绕点走动$i$完整的一圈。 
3. 通过再次附加每个向量和角度来复制排序列表$+2\pi$。 这使我们能够使用线性结构来模拟圆形环绕。 
4. 使用两指针技术。 对于每个起始索引$l$, 移动指针$r$向前只要角度差小于$180^\circ$。 窗户$[l, r)$表示位于从 方向 开始的有效半平面中的所有点$l$。 
5. 令该窗口内的点数为$w = r - l - 1$。 这意味着选择与方向对齐的边界线$l$准确地产生$w$点在一侧。 
6. 增量$A[i][w]$对于从以下位置开始的每个有效窗口加 1$l$。 这将计算每条不同的分隔线一次。 
7. 对所有人重复$i$，产生完整的矩阵。 

重要的细节是每条有效的有向线$i$恰好对应于一个角度起始方向，并且扫描确保我们对每个最大半平面恰好计数一次。 

### 为什么它有效

 角度扫描通过编码每条可能的分隔线$i$作为一个点和下一个点之间的边界（按角度顺序）。 因为半平面约束相当于严格小于的角跨度$180^\circ$，每个可行的配置对应于循环排序中的连续弧。 两指针窗口找到从每个方向开始的所有最大弧，并且每个这样的弧唯一地确定有效线一侧上的点集。 由于没有其他点恰好位于问题构造的边界上，因此不会遗漏或重复计算任何配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    ans = [[0] * (n - 1) for _ in range(n)]

    for i in range(n):
        x0, y0 = pts[i]
        arr = []

        for j in range(n):
            if i == j:
                continue
            x, y = pts[j]
            dx = x - x0
            dy = y - y0
            arr.append((math.atan2(dy, dx), dx, dy))

        arr.sort(key=lambda t: t[0])
        m = n - 1

        ext = arr + [(a + 2 * math.pi, dx, dy) for a, dx, dy in arr]

        r = 0
        for l in range(m):
            if r < l:
                r = l
            while r < l + m and ext[r][0] - ext[l][0] < math.pi:
                r += 1
            w = r - l - 1
            ans[i][w] += 1

    out = []
    for i in range(n):
        out.append(" ".join(map(str, ans[i])))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现使用以下方法在每个点周围构建角度向量`atan2`，这确保了所有象限中的正确排序。 复制步骤为`+2π`对于在没有模运算的情况下处理环绕至关重要。 

仔细维护两指针窗口，以便对于每个左端点，我们仅向前扩展右端点，确保每个中心的摊销线性时间。 减法`r - l - 1`从计数中删除枢轴方向本身。 

一个常见的陷阱是忘记重置`r`当它落后的时候`l`，这会破坏单调性并导致不正确的窗口大小。

 ## 边缘情况

 看起来退化的配置是指许多点几乎与枢轴共线。 在这种情况下，小浮点精度误差`atan2`排序可以改变角度列表中的邻接关系。 该算法依赖于严格的排序，因此需要稳定的排序和一致的角度表示以避免半平面边界的错误分类。 

另一种边缘情况是所有点都位于围绕枢轴的半圆中。 在这种情况下，滑动窗口将始终扩展以包含所有点，从而在行中仅产生一个非零条目。 该算法可以正确处理这个问题，因为条件`angle < π`包括完整的可达集，没有溢出或环绕问题。 

最后一个微妙的情况是点几乎均匀地分布在圆周围。 这里每个起始方向都会产生不同的最大弧，并且算法必须确保每个弧被精确地计数一次。 右指针的严格单调前进保证了起始索引和计数配置之间的一对一映射。
