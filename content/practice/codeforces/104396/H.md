 ## 解决方案
 ## 问题理解

 我们有两个长度相等的字符串。 将它们视为字符序列的两个版本，其中第一个字符串是起始配置，第二个字符串是目标配置。 我们的目标是使用非常具体的操作将第一个字符串转换为第二个字符串。 

唯一允许的操作是选择字符串的后缀，选择 1 到 25 之间的移位值 k，然后对该后缀中的每个字符应用循环移位。 移位意味着将字母表中的字母向前移动，然后环绕，以便在“z”之后出现“a”。 关键限制是每个操作仅影响一个后缀，并且在该后缀内每个字符都会移动相同的量。 

我们希望最大限度地减少需要的此类后缀操作的数量。 

约束 n 高达 2 × 10^5 立即排除任何尝试模拟所有操作或搜索后缀和移位选择的解决方案。 任何独立考虑配对位置的方法都是可疑的，因为后缀操作将所有字符耦合到右端。 我们需要一些线性或近线性的东西。 

当不同的位置需要不同的调整量时，会出现微妙的边缘行为。 例如，如果 S 是“aaa”而 T 是“abc”，一个天真的想法可能会尝试从左到右独立地修复每个字符。 但这会失败，因为任何后缀操作都会同时影响多个位置，因此纠正一个位置可能会意外地干扰之前固定的位置，除非仔细控制结构。 

另一个棘手的情况是当调整“抵消”模 26 时。总是独立地固定每个位置而不考虑之前累积的移位的贪婪策略将会多算或少算操作，因为多个移位可能会累积并回绕。 

## 方法

 蛮力的观点是从状态的角度思考：在每一步，选择一个后缀和一个移位，并模拟直到S变成T。这在原则上是正确的，但分支因子是巨大的。 后缀有 n 个选择，最多有 25 个移位值，并且可能有很多步骤，因此状态空间呈指数增长。 即使修剪也无济于事，因为中间状态在各个位置之间并不独立。 

关键的观察结果是操作始终适用于后缀，这表明从右到左进行处理。 如果我们首先修复最后一个字符，那么影响较早位置的任何未来操作都不会干扰它，因为在该索引之前结束的后缀无法到达它。 这给出了不可逆的自然方向：一旦相对于包含该位置的所有操作正确调整了位置，如果我们只继续向左工作，它就会保持正确。 

我们不是显式跟踪字符，而是跟踪每个后缀位置应用了多少累积移位。 重要的简化是，只有 S[i] 和 T[i] 之间的差异（通过已应用的移位进行调整）才在位置 i 上起作用。 一旦我们决定在 i 处需要多少额外的移位，我们就可以应用从 i 开始的后缀操作来修复它，并且相同的操作会以受控的方式自动贡献给所有较早的位置。 

这将问题简化为从右到左贪婪地处理，保持运行累积移位。 在每个索引处，我们在考虑了先前后缀操作的累积移位后，计算需要多少额外移位来匹配目标字符。

 ## 算法演练

我们处理从最后一个字符到第一个字符的字符串，并维护一个值，该值表示已应用于从当前位置或右侧开始的后缀的总移位。 

1. 初始化变量`shift`为 0，表示当前位置由于先前选择的后缀操作而移动了多少。 这种移位总是以 26 为模进行解释。 
2. 从索引 n − 1 向下移动到 0。在每个位置 i，在应用累积移位后计算 S[i] 的有效字符。 这相当于添加`shift`到它的字母表索引。 
3. 将该有效字符与T[i]进行比较。 差异告诉我们在这个位置需要多少额外的移位才能使其匹配 T[i]。 
4. 将这个差值转换为[0, 25]中的值d。 这表示应用一个后缀运算，从 i 开始，移位 d。 
5. 将答案加 1，因为我们只执行一次操作来固定位置 i。 
6. 更新`shift`通过添加 d 模 26，因为这个新的后缀运算也会影响左侧的所有位置。 
7. 继续到下一个位置，结转更新的班次。 

这个贪心步骤有效的原因是，一旦我们到达位置 i，其右侧的所有位置都已在正确性方面完成确定，并且我们应用的任何新操作都必须包含位置 i 来修复它。 在 i 处使用多个操作没有任何好处，因为单个操作可以实现任何所需的模 26 调整。 

### 为什么它有效

 该算法保持一个不变量，即在应用先前选择的操作的累积偏移后，所有大于 i 的索引都已经等于其目标值。 在步骤 i，任何有效的解决方案都必须确保 S[i] 变为 T[i]，并且影响 S[i] 的唯一方法是通过在位置 ≤ i 处应用的操作。 通过选择修复 i 处不匹配的最小单个后缀操作，我们永远不会失去最优性，因为影响 i 的多个操作的任何序列都可以合并到应用于最右侧相关步骤的单个等效净移位模 26 中。 这种操作的可压缩性保证了贪婪计数与最小可能的操作数相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def idx(c):
    return ord(c) - ord('a')

n = int(input().strip())
s = input().strip()
t = input().strip()

shift = 0
ans = 0

for i in range(n - 1, -1, -1):
    cur = (idx(s[i]) + shift) % 26
    target = idx(t[i])
    
    need = (target - cur) % 26
    
    if need != 0:
        ans += 1
        shift = (shift + need) % 26

print(ans)
```该实现完全遵循从右到左的扫描。 辅助函数将字符转换为数字索引，以便模运算变得简单。 

变量`shift`存储迄今为止选择的所有后缀操作的累积效果。 在每个索引处，我们计算应用此移位后的当前有效字符。 表达式`(target - cur) % 26`确保我们始终选择有效范围 0 到 25 内的前移，与操作定义匹配。 

关键的微妙之处在于我们只在需要时为每个索引添加一个操作。 我们不会尝试“部分”修复头寸，因为任何部分修复仍需要稍后进行完整的模块化调整，并且只会增加操作数量。

 考虑一个简单的情况：

 S =“aaa”，T =“abc”

 | 我| S[i] | 班次 | 有效 S[i] | T[i] | 需要| 回答 |

 ## 边缘情况

 一种边缘情况是完全换行，其中每个字符都必须循环穿过字母表边界。 对于输入 S =“z”、T =“a”，该算法在索引 0 处计算 need = 1，应用一个操作，并正确返回 1。 

另一种情况是，由于模运算，较早的转变会产生大量的向后看要求。 对于 S = "aaa"、T = "bbb"，从右到左处理可确保一旦索引 2 固定，索引 1 和 0 自然会继承正确的累积移位，从而恰好产生 3 个操作。 

最后一种边缘情况是 S 和 T 相同。 移位始终保持为零，不应用任何操作，并且算法立即以零成本退出，从而确认没有引入不必要的后缀操作。
