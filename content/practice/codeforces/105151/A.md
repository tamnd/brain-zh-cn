 ## 问题理解

 我们给出了五个整数来描述楼梯结构的不同部分中存在多少级台阶。 该图片（我们不需要明确地需要）从下到上编码了一组可能的路线，其中每条路线对应于选择一系列连接的段。 每个路段贡献自己的步数，任务是在从起点到终点的所有有效路线中找到尽可能最小的总步数。 

关键的解释是我们没有对所有五个值求和。 相反，每个值都属于小型图状结构中的固定节点，并且有效路径恰好选择一系列连接的节点。 每条路径都恰好有三个段，因此答案始终是三个选定值的总和，但允许的三元组根据图中的连接性而有所不同。 

由于只有五个值和固定的小结构，因此计算时间是常数。 每个值最多 10^8 的约束只是告诉我们算术溢出或特殊处理在 Python 中是无关紧要的，我们应该只专注于选择正确的组合。 

这里的一个天真的错误是假设可以选择任意三个值。 例如，使用输入`1 3 2 4 5`，人们可能会错误地取三个最小值`1 + 2 + 3 = 6`，但该组合不一定是楼梯布局中的有效路径。 该结构限制了允许使用哪些三元组，因此正确性取决于遵守这些限制。 

另一个微妙的问题是假设对称或尝试动态计算最短路径。 这有点矫枉过正了，因为图表是固定的而且很小； 答案由一小组候选总和确定。 

## 方法

 如果我们忽略结构并尝试显式探索所有可能的路线，我们将枚举图中从底部节点到顶部节点的所有路径。 每条路径对应于通过连接段的一系列移动，对于每条路径，我们计算其三个段值的总和。 对于五个节点，这种强力枚举的大小是恒定的，但概念上的行为类似于图遍历。 

只有当我们将其推广时，这种公式的低效率才会出现：路径枚举在一般图中是指数级的。 在这里这是不必要的，因为图是固定的并且只有少数有效路线。 

关键的观察结果是，该图仅定义了一些有效组合，并且所有有效路径都简化为五个变量上的一小组候选表达式。 一旦我们确定了这些候选和，问题就变成了恒定数量的表达式的简单最小值。 

楼梯的结构恰好暗示了三种有意义的路线模式。 每个模式对应于在每个分支点选择两个替代方案之一。 我们不探索路径，而是直接评估这些模式。

 ## 算法演练

 楼梯可以建模为一个小型定向结构，沿途有三个决策。 每个决策都会在两个相邻的路段之间进行选择，并且通过组合每个决策点的一个选择来形成一条完整的路线。 

1. 识别由图表编码的三种有效路径模式。 每种模式对应于从底部到顶部的不同遍历方式，同时尊重楼梯中的相邻性。 
2. 将每个模式表示为五个值中三个值的总和。 关键步骤是将图中的每个运动映射到其关联的输入值。 
3. 直接使用算术计算所有三个候选和。 
4. 返回这些候选总和中的最小值作为最终答案。 

我们明确枚举这些总和而不是尝试动态推理的原因是该图没有环，也没有大的分支结构，因此每个有效路径都是预先确定的。 

### 为什么它有效

 从下到上的每条有效路线都必须恰好经过楼梯结构中两个中间交汇处的每一个选择。 这迫使路径空间折叠成三种组合。 由于每个组合完全由固定的段选择决定，因此评估所有组合并选择最小值保证了最短的可能路径。 此枚举之外不存在其他路由，因为结构中没有其他分支点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b, c, d, e = map(int, input().split())

# three valid paths in the staircase graph
path1 = a + b + d
path2 = a + c + e
path3 = a + c + d

print(min(path1, path2, path3))
```该实现直接遵循从路径到固定表达式的简化。 每个表达式对应于一条结构上有效的楼梯路线。 

一个常见的实施陷阱是错误识别哪些段属于哪条路线。 正确的映射来自于跟踪图的邻接关系，而不是来自对值的排序或贪婪选择。 

另一个微妙之处是所有计算都是独立的，因此不需要中间存储或迭代更新。

 ## 边缘情况

 例如，当所有值都相等时`1 1 1 1 1`，每个候选总和的计算结果为 3。该算法计算所有三个表达式并返回 3，因此不需要特殊处理。 

当一种价值观支配其他价值观时，例如`10 1 100 1 1`，正确的路径尽可能避免大 100。 候选评估明确包括使用的两条路线`c`以及避开它的路线，因此正确的最小值变为 12。 

当大值出现在端点处时，例如`100 1 1 100 100`，算法仍然独立评估每个固定组合。 即使存在多个大值，也只有一种组合可以最小化重叠，并且最小操作可以正确选择它。 

每种情况都纯粹通过预定义结构的算术评估来处理，因此除了计算和比较三个总和之外，不需要动态推理或条件分支。
