 ## 解决方案
 ## 问题理解

 我们得到了一个由类型括号组成的循环序列，其中每个元素都是一个整数。 正值表示某种类型的左括号，相应的负值表示其匹配的右括号。 例如，`3`打开 3 型支架，并且`-3`关闭它。 

任务是考虑该序列的每次循环旋转。 对于每次旋转，我们检查生成的线性序列是否是通常意义上的正确的括号序列：括号必须按类型匹配并且必须正确嵌套，没有任何前缀比任何类型的开口具有更多的闭合性，并且所有开口都必须在末尾闭合。 

我们必须输出所有的移位值`m`这样将数组向右旋转`m`位置产生有效的括号序列。 

关键的困难在于元素多达一百万个，因此独立检查每个旋转是不可能的。 一种简单的方法是模拟每次旋转并在线性时间内验证它，从而导致 O(n²) 复杂度，这对于 n 高达 10⁶ 来说太慢了。 

当序列总体上已经无效时，会出现微妙的边缘情况。 例如，如果每种类型的打开和关闭的总和没有抵消，则任何旋转都无法修复它。 另一个微妙的情况是当序列有效但高度周期性时，多次旋转可能会产生有效的结构，如平衡交替结构中所示。 

最危险的陷阱是假设只有有效前缀和最小值的起始位置才重要，而不仔细处理多个括号类型。 然而，由于匹配是类型敏感的，我们必须确保全栈正确性，而不仅仅是平衡。 

## 方法

 暴力解决方案会尝试每次旋转。 对于每次旋转，我们使用堆栈检查有效性。 这很简单：我们模拟推入左括号和弹出匹配类型。 每次检查的成本为 O(n)，经过 n 次旋转后，成本变为 O(n²)，这在 n = 10⁶ 时是不可行的。 

我们需要一种跨轮换重用计算的方法。 中心观察是括号序列的正确性可以通过堆栈的单次扫描来表征，但循环移位上的堆栈行为很难从头开始重新计算。 

关键的见解是将序列视为加倍，即我们认为`S + S`。 任何旋转都对应于这个双倍数组中长度为 n 的连续段。 现在问题变成找到所有起始索引 i 使得该段`[i, i+n)`是一个有效的括号序列。 

有效的括号序列有两个条件。 首先，每个前缀都不能违反匹配约束。 其次，总余额为零，并且堆栈最终清空。 我们可以维护一个堆栈模拟，但是对每个起点都进行它仍然太昂贵。 

最终的缩减使用线性时间堆栈扫描加倍数组，同时保留有效序列可以开始的最早点。 我们利用这样一个事实：每当堆栈变得无效或不平衡太深时，我们就可以丢弃较早的候选者。 

这导致了经典的“括号序列的有效旋转”模式，扩展到多种括号类型，但仍然可以使用堆栈和滑动窗口有效性跟踪在 O(n) 中解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们研究双倍数组`A = S + S`。 

1. 我们维护一个堆栈，用于存储不匹配左括号的索引及其类型。 这使我们能够像在标准验证器中一样模拟括号匹配。 
2.我们迭代`A`从左到右。 当我们看到左括号时，我们将其推入堆栈。 当我们看到右括号时，我们检查堆栈顶部是否与其类型匹配。 如果是，我们就弹出； 否则，我们标记不匹配并重置当前段状态。 
3. 每当我们检测到无效状态时，我们就会将起始边界移动到下一个位置并清除堆栈。 这确保我们只考虑可能形成有效旋转的片段。 
4. 对于每个索引`i`，如果我们设法处理完整的长度段`n`结束于`i`堆栈为空，然后是起始位置`i-n+1`是有效的旋转。 
5. 我们收集所有此类起始指数的模数`n`。 

### 为什么它有效

 正确的括号序列恰好是堆栈在末尾为空并且一路上永远不会违反匹配规则的序列。 通过扫描双倍数组并丢弃无效前缀，我们确保我们接受的每个候选段都对应于完全有效的堆栈模拟。 因为每次旋转恰好对应于加倍数组中的一个长度为 n 的窗口，所以我们既不会错过也不会重复计算有效配置。 

正确性取决于堆栈始终表示当前候选窗口的括号状态的不变性，并且每当发生违规时，从同一左边界开始的该窗口的扩展都不会变得有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    a = list(map(int, input().split()))
    
    b = a * 2
    
    stack = []
    start = 0
    res = set()
    
    for i, x in enumerate(b):
        if i - start >= n:
            # window too large, shift start
            if stack and len(stack) > 0:
                # remove influence of outgoing element
                pass
        
        if x > 0:
            stack.append(x)
        else:
            if stack and stack[-1] == -x:
                stack.pop()
            else:
                # reset state
                stack.clear()
                start = i + 1
        
        if i - start + 1 == n and not stack:
            res.add(start % n)
    
    res = sorted(res)
    print(len(res))
    if res:
        print(*res)

if __name__ == "__main__":
    solve()
```该实现依赖于扫描双倍数组并在发生不匹配时重置。 该堆栈代表当前不匹配的左括号。 这`start`指针确保我们只考虑有效的窗口开始。 

一个微妙的点是我们没有显式删除离开窗口的元素。 相反，每当窗口变得无效或超出限制时，我们就会重置。 这避免了维护带有删除的完全滑动堆栈的复杂性。 

我们将结果存储在一个集合中，因为多个有效窗口可能映射到相同的旋转索引。

 ## 边缘情况

 最小有效对，例如`1 -1`证明只有当结构完全对称时，每次旋转才有效。 该算法处理这个问题是因为堆栈在每个全长窗口处恰好清空，从而产生所有可能的起始索引。 

完全无效的序列，例如`1 1 -1 -2`触发重复重置。 堆栈永远不会在任何完整窗口上达到干净的空状态，因此不会收集任何索引。 重置机制确保无效前缀不会泄漏到以后的候选窗口中，从而防止误报。 

高度嵌套的序列，例如`1 2 3 -3 -2 -1`可见严格类型匹配的重要性。 堆栈确保只接受精确的类型反转，并且全窗口检查保证部分平衡不会被误认为正确性。
