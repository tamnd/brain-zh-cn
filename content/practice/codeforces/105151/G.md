 ## 解决方案
 ## 问题理解

 该板非常高，但只有两列宽，因此每一行只是一个左侧或右侧的单元格。 白色棋子从左下角的单元格开始，逐行向上移动，直到它卡住或到达顶行的高度$10^{18}$。 

每行的每个单元格最多可以包含一个黑色块。 棋子一次向上移动一行，但列的选择以及是否可以交叉完全取决于目标单元格是否包含黑色棋子。 仅当目标单元格为空时，才允许在同一列内正常向前移动。 仅当目标单元格包含黑色棋子时，才允许像对角线一样移动到另一列，在这种情况下，棋子会移动到那里并移除该棋子。 

因此，每一行的行为就像一个小的本地决策点：来自行的给定列$r$，棋子决定如何进入行$r+1$，并且该决定仅取决于行中的两个单元格中的每一个是否$r+1$被阻止。 

关键的困难是行数达到$10^{18}$，但仅$10^5$其中包含任何黑色碎片。 所有其他行都是完全空的，并且空行的行为是确定性的：一旦您在某列中输入空行，您将被迫保留在该列中所有连续的空行，直到包含一个片段的下一行强制进行更改。 

输出只是是否存在到达 row 的任何合法移动序列$10^{18}$。 

约束条件$n \le 10^5$立即排除任何逐行迭代的模拟$10^{18}$。 即使每个事件的处理也必须是线性的或$n \log n$。 任何尝试显式模拟每个行转换的方法都是不可能的。 

当黑色棋子稀疏但有策略地放置时，会出现微妙的边缘情况。 例如，如果所有棋子都在一行中，但阻挡了两个“强制交叉方向”，则即使大多数行都是空的，棋子也可能会永久卡住。 另一个边缘情况是当棋子必须在列之间来回多次时； 如果不将其作为全局状态传播问题处理，每行的贪婪决策可能会失败。 

## 方法

 直接模拟将尝试逐行保持棋子的位置。 在每一行，我们检查下一行的两个单元格并应用移动规则。 这在概念上是有效的，因为棋子的状态只是它的列，但问题是行数：在连续的黑块行之间，可以有间隙$10^{18}$。 简单的模拟会花费时间迭代这些间隙，这是不可能的。 

关键的观察结果是空行是同质的。 如果一行$r$没有黑棋子，则从纵列进入$c$强制您将其从同一列中保留$c$，并且此行为对于每个连续的空行都重复相同。 如此长时间的空延伸不会改变状态； 他们只是“拖延时间”。 

这意味着我们只需要推理包含至少一个黑色棋子的行。 我们将问题压缩成一系列按高度排序的“有趣的行”。 在两个连续的有趣行之间，pawn 的列不会改变，因此我们只需要向前传递一个状态。 

现在系统变成了排序行上的二态动态过程。 在每个感兴趣的行，根据哪些单元格被阻止，某些转换可能会强制切换列或可能允许停留。 我们向前传播可达状态； 如果在任何时候这两种状态都变得不可能，则该过程结束。 

这将问题减少到扫描已排序的行并维护在每个事件行处可访问两列中的哪一列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力行模拟|$O(10^{18})$|$O(1)$| 太慢了 |
 | 只处理占用的行 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们只处理至少包含一个黑色块的行，按行索引排序。 

1. 按行索引对所有黑色棋子进行分组，存储第 1 列和/或第 2 列是否被阻挡在该行中。 这将同一行中的多个部分减少为简单的 2 位掩码。 这很重要，因为只有每列的占用率会影响转换，而不是片段的数量。 
2. 对至少包含一件的行进行排序。 我们将仅按升序模拟这些行之间的移动。 
3. 维护一个状态数组`dp`大小为 2，其中`dp[c]`意味着可以站在纵队中`c`就在处理当前感兴趣的行之前。 
4. 初始化`dp = [True, False]`因为我们从第 1 行的第 1 列开始，并且最初只能到达该状态。 
5. 按升序遍历已排序的行。 对于每一行`r`，计算下一个状态`ndp`基于当前所有可访问的列：

 - 如果我们位于行前的第 1 列`r+1`，那么：

 - 如果该单元格未被阻止，我们可以直接移动到第 1 列。 
- 如果第 2 列被阻挡在该行中，我们可以对角移动到第 2 列（因为这允许捕获和转换）。 
- 如果我们位于行前的第 2 列`r+1`，适用对称逻辑。 

此步骤独立应用于每个可达状态，并将结果合并为`ndp`。 
6. 更换`dp`和`ndp`。 如果两个状态都变为假，我们就会提前终止，因为棋子被卡住了。 
7. 处理完所有感兴趣的行后，我们还有最后的延伸$10^{18}$。 由于它完全是空的，因此不会发生进一步的转换； 这两种状态都是有效的最终状态。 如果在最后处理的行处可以访问任一列，我们可以无限地继续到顶部。 
8、如果最后至少有一个列状态可达，则输出YES； 否则输出NO。 

### 为什么它有效

 不变量是在处理所有行达到一定高度后$r$,`dp[c]`正确表示棋子是否可以在列中$c$在行$r$。 空行保留此不变式，因为它们不会修改列。 状态的每次变化仅发生在包含碎片的行上，并且这些行完全由 DP 中编码的转换规则捕获。 由于转换仅取决于紧邻的下一行而不取决于更深的历史记录，因此不需要超出当前列状态的额外内存。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
rows = {}

for _ in range(n):
    c, r = map(int, input().split())
    if r not in rows:
        rows[r] = [False, False]
    rows[r][c - 1] = True  # black piece exists

events = sorted(rows.items())

dp = [True, False]  # at row 1, column 1

prev_r = 1

for r, mask in events:
    ndp = [False, False]

    # we process jump from prev_r to r, but empty rows don't change state
    # so dp remains valid until r

    for c in (0, 1):
        if not dp[c]:
            continue

        # move to same column if empty
        if not mask[c]:
            ndp[c] = True

        # move to other column if there is a piece there
        oc = 1 - c
        if mask[oc]:
            ndp[oc] = True

    dp = ndp

    if not dp[0] and not dp[1]:
        print("NO")
        exit()

# after last event, infinite empty rows to the top
print("YES" if dp[0] or dp[1] else "NO")
```该实现将每行的所有部分压缩为布尔掩码。 这避免了对同一行中的多个片段进行重复处理。 

DP 转换直接遵循移动规则：保持在同一列需要目标单元中不存在棋子，而切换列需要存在要捕获的棋子。 最终答案检查最后一个相关行之后的可达性，因为没有任何东西可以改变超出该点的状态。 

## 工作示例

 ### 示例 1

 输入：```
1
1 2
```| 步骤| 行| dp 之前 | 行掩码| dp 之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 2 | [T，F] | [T，F] | [F，T] |

 棋子从第 1 行的第 1 列开始。在第 2 行，第 1 列被阻挡，因此不可能直线移动。 第 2 列被阻止，允许捕获移动到第 2 列。此后，不再存在任何事件，因此可以达到最终状态。 

这显示了单个障碍物如何迫使列开关。 

### 示例 2

 输入：```
1
1 1000000000000000000
```| 步骤| 行| dp 之前 | 行掩码| dp 之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 10^18 | 10^18 [T，F] | [T，F] | [F，T] |

 逻辑是相同的：单个后期障碍仅在其所在行中起作用。 然而，由于目标行本身被阻挡在第 1 列中并且需要交互才能到达第 2 列，因此 pawn 无法正确前进到最终目标配置。 

这表明即使是单个遥远的约束也可以完全确定可达性。

 ## 边缘情况

 一个关键的边缘情况是当唯一的黑色棋子出现在远高于起点的行中时。 例如，如果第一个占用的行是$10^{18}$，棋子不受约束地穿过所有较低的行。 该算法可以处理此问题，因为 DP 在第一个事件行之前保持不变，从而有效地压缩了空前缀。 

另一种情况是当一行中的两个单元格都被阻塞时。 在这种情况下，每个传入状态都会转换为该行的死状态，并且 DP 变空。 算法检测后立即正确返回NO`dp`一切都变得虚假。 

第三种情况涉及跨多行交替强制切换。 由于每行仅使用两个状态进行独立处理，因此重复振荡不需要存储超出当前 DP 向量的历史记录。
