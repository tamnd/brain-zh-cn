 ## 解决方案
 ## 问题理解

 我们得到一个整数$n$，我们想把它分成一对有序的正整数$(a, b)$这样$a + b = n$。 并非每个分割都是有效的：两个数字都必须至少有两位数字，并且它们必须满足结尾之间的数字兼容性条件$a$和开始$b$。 具体来说，最后一位数字$a$必须等于第一个数字$b$。 

任务是计算有多少个有序对$(a, b)$满足所有这些限制。 

约束条件$n \le 10^{18}$立即排除迭代所有可能的分裂$a \in [1, n-1]$。 线性扫描已经需要最多$10^{18}$的操作，这是完全不可行的。 甚至$O(\sqrt{n})$如果我们正确地构建问题，那么所有分割上的数字DP是不必要的，因为每个候选分割仅取决于局部数字属性$a$和$b$。 

微妙的部分是数字条件：它结合了后缀$a$前缀为$b$，这意味着我们不能独立地处理这两个数字，即使它们的总和约束很简单。 

一些边缘情况值得注意。 如果$n < 20$，不存在有效的对，因为两个数字必须至少为 10。另一个微妙的情况是$a$以 0 结尾，强制$b$以 0 开头，这是不允许的，因为数字不能有前导零。 仅检查数字相等性但忽略前导数字约束的粗心方法将过多计算这些无效转换。 

## 方法

 暴力方法尝试每个分割点$a$从 10 到$n-10$，计算$b = n-a$，并直接检查数字约束。 每次检查都需要提取最后一位数字$a$和第一个数字$b$，这需要花费$O(\log n)$。 这导致$O(n \log n)$复杂性，这对于$n = 10^{18}$。 即使是为了$n \le 10^6$，这在优化的 C++ 中几乎不会通过，而在 Python 中会失败。 

关键的观察是总和约束固定$b$一次$a$被选择，但我们实际上不需要枚举所有$a$。 相反，我们反转视角并枚举有效转换的可能边界数字。 

条件“最后一位数字$a$等于第一个数字$b$” 在十进制表示中创建一个数字边界$n$。 如果我们想到$n$当一个数字被分成两个加数时，我们可以用进位来解释逐列的加法。 后缀和前缀之间的交互变得局部：只有边界数字位置重要，其他一切都由进位决定。 

这表明了对分解的数字动态规划方法$n$进入$a + b$，我们跟踪进位并强制两个数字保持有效（至少两位数）并遵守恰好一个位置的边界数字条件。 

我们从最低有效位到最高有效位处理数字，并保持我们是否处于分割点$a$转变为$b$。 在该边界处，我们强制要求最后一位数字$a$等于第一个数字$b$。 每个其他数字位置仅受带有进位的标准加法约束的控制。 

这将问题从枚举数字减少到对进位传播下的数字分配进行计数，这是数字数量的多项式，即$O(18 \cdot 2 \cdot 10 \cdot 10)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n \log n)$|$O(1)$| 太慢了 |
 | 数字DP |$O(d \cdot C)$在哪里$d \le 18$|$O(d \cdot C)$| 已接受 |

 ## 算法演练

 我们使用动态规划对进位和分割位置从最低有效侧开始逐位处理数字。 

1. 将数字取反$n$，因此我们可以从最低有效位向上处理加法。 这使得进位处理变得自然，因为加法沿该方向传播。 
2. 定义一个 DP 状态，跟踪当前位置、前一位加法的进位以及我们是否已经通过了分割点，其中$a$转变为$b$。 分割位置至关重要，因为只有一位数字边界决定了分割位置$a$结束和$b$开始。 
3. 在每个数字位置，我们决定当前数字如何$n$由数字组成$a$,$b$，并携带。 这意味着我们尝试所有可能的数字对$(a_i, b_i)$满足：$$a_i + b_i + carry \equiv n_i \pmod{10}$$具有有效的下一个进位。 
4. 我们强制执行结构约束：数字属于$a$必须组成一个至少由两位数组成的数字，并且对于$b$。 这是通过确保分割位置不太靠近任一端来解决的。 
5. 在分割边界，我们强制执行数字条件：最后一位数字$a$（按相反顺序分割之前的数字）必须等于第一个数字$b$（按正序拆分后紧接的数字）。 在反向处理中，这成为链接 DP 转换中两个相邻位置的约束。 
6. 处理完所有数字后，我们接受进位为零且两个数字都满足最小长度要求的状态。 

### 为什么它有效

 每个有效对$(a, b)$恰好对应于加法的一位数字级分解$a + b = n$具有独特的进位序列。 DP 枚举了所有可能的数字分解，没有重复，因为每个状态都将部分和精确编码到一个位置。 拆分约束仅在一次转换时强制执行，确保我们只计算数字边界条件成立的对。 由于没有跳过有效分配并且没有接受无效分配，因此计数是准确的。

 ## 边缘情况

 一种边缘情况是当$n < 20$。 任何拆分都会产生至少一个低于 10 的数字，因此答案必须为零。 DP 正确地拒绝了这些，因为它决不允许两个数字同时达到两位数长度。 

另一个边缘情况是当候选人分裂时$b$从零开始。 例如，如果拆分意味着$b = 05$，自第一个数字激活以来，DP 不允许此状态$b$需要一个非零数字，以防止计算无效的前导零。 

第三种情况是当进位传播到整个数字时，这可能会在有效数字对齐发生的位置发生移位。 DP 自然地处理这个问题，因为在每个位置都明确跟踪进位，确保整个加法链的一致性。
