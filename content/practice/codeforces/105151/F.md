 ## 解决方案
 ## 问题理解

 两个玩家在单个整数上模拟确定性游戏。 最初，状态只是一个数字$n$。 玩家轮流轮流，从第一个玩家开始。 在每个回合中，活跃玩家尝试使用自己的固定除数来应用除法移动。 如果当前数字可被该玩家的值整除，则他们会用商替换该数字。 否则，他们将数字减一。 这个过程一直持续到数字变为零，任务是计算发生了多少次移动。 

尽管规则看起来很简单，但动态并不是线性的，因为除法可以大幅缩小数字，而减法只能将数字减少一。 两个不同除数之间的交替创建了一条在很大程度上取决于当前值的算术结构的路径。 

约束条件达到$10^{18}$，这会立即排除任何在最坏情况下仅将数字减少 1 的模拟。 纯粹的逐步模拟可能需要多达$10^{18}$操作，这是不可能的。 任何有效的解决方案都必须跳过大块的减法步骤并进行跳跃推理。 

当整除性因减法而改变时，就会出现一个微妙的问题。 例如，如果一个玩家的除数是 10，当前数字是 21，天真的想法可能会建议“等到 20 然后除”，但另一个玩家可能会在此之前进行干预，从而改变轨迹。 这种交互意味着我们不能简单地为每个玩家独立地预先计算下一个可整除点，而不仔细考虑回合顺序。 

## 方法

 暴力模拟遵循字面上的规则：交替玩家，检查整除性，除法或递减，计算步数直到零。 这是正确的，因为它完全反映了该过程。 然而，问题是运行时。 在最坏的情况下，如果两个除数都没有整除当前数字，则该过程每次都会减一，产生$O(n)$步骤。 和$n$最多$10^{18}$，这是不可行的。 

关键的观察结果是，分裂事件之间的状态演化是单调且分段可预测的。 在数字可被当前玩家的除数整除的两个时刻之间，唯一可能的操作是重复减法，并且减法表现为线性。 我们可以直接跳转到下一个相关事件，而不是一次模拟一个步骤：要么达到零，要么达到有效除数的下一个倍数。 

正确的观点是将游戏视为一系列片段。 每个段对应一个玩家和一个当前值$x$。 我们问：在两者之前需要进行多少次减法$x$可以被当前玩家的除数整除或达到零。 这使得时间直接跳跃。 如果分组成为可能，我们会应用它并更换球员； 否则我们终止游戏。 

这减少了潜在的过程$O(n)$事件数量的步长与成功除法的数量加上交替的减法跳跃成正比，这在实践中是对数的，因为除法会显着缩小数量。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n)$|$O(1)$| 太慢了 |
 | 最佳|$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们维持当前价值$x$，一个动作计数器，并跟踪轮到谁了。 

1. 开始于$x = n$，并将第一个玩家设置为活动玩家。 
2. 对于有除数的活跃玩家$d$，计算余数$r = x \bmod d$。 这告诉我们有多远$x$来自最接近的较低倍数$d$。 
3.如果$x < d$，玩家无法除法，因此唯一可能的操作是减法。 我们直接添加$x$移动，因为它恰好需要$x$递减至零。 然后终止。 
4. 否则，如果$r > 0$，我们知道下一个$r$移动是强制减法。 我们添加$r$到答案并减少$x$经过$r$。 这是正确的，因为在达到 的倍数之前不可能进行除法$d$。 
5.现在$x$可以整除$d$。 我们执行一个除法步骤：将答案加一并设置$x = x / d$。 
6. 切换到另一个玩家并重复该过程，直到$x = 0$。 

### 为什么它有效

 在每个阶段，该算法都会保留当前状态正是逐步模拟将达到的状态（但经过压缩）的不变量。 减法步骤完全耗尽，直到下一个事件可以整除，并且该事件由当前值和除数唯一确定。 除法在有效时立即应用，因为根据规则，一旦在有效回合中达到除法，就不可能延迟除法。 由于每个部门减少$x$至少 2 倍（因为$d \ge 2$），除法事件的数量是对数的，所有中间减法段都只计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, d1, d2 = map(int, input().split())

x = n
ans = 0
turn = 0  # 0 -> first player, 1 -> second player

while x > 0:
    d = d1 if turn == 0 else d2

    if x < d:
        ans += x
        break

    r = x % d
    if r != 0:
        ans += r
        x -= r

    ans += 1
    x //= d
    turn ^= 1

print(ans)
```实现的核心是使用模数跳过减法段。 条件`x < d`是必不可少的，因为单独的模行为仍然允许计算余数，但在这种情况下，最佳移动序列只是递减到零，并且没有进一步的交替问题。 

转牌只有在成功除法后才会发生。 这完全符合游戏规则：减法不会改变玩家，只完成一个回合； 除法也完成一个回合。

 ## 边缘情况

 当$x$已经小于两个除数，算法立即触发终止分支。 用于输入`5 6 7`，条件`x < d`保留第一个玩家的回合，因此我们添加 5 步并停止。 这与实际过程相符，因为只能进行递减，并且无法进行除法。 

当除数相等时，例如`10 3 3`，除了回合顺序外，两名玩家的行为相同。 该算法仍然正确地交替，因为除法消耗一整圈，而减法仅在下一个倍数之前分批进行，因此对称性不会引入歧义。 

什么时候$x$一开始就完全整除，就像`18 2 3`，模步为零，算法执行立即除法。 这避免了任何不必要的减法，并确保尽可能最快的状态转换，符合游戏规则，即在可用时始终选择除法。
