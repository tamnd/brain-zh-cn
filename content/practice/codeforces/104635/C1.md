 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，这些整数序列是由底层隐藏的素数序列产生的。 每个整数表示该隐藏序列中两个相邻素数的乘积。 从这些产品中，我们期望重建原始的素数序列，然后通过将不同的素数按字母顺序映射到字母来将其解码为字符串。 

关键的困难在于我们不能直接看到任何素数，只能看到它们的成对乘积。 一旦素数序列被恢复，每个不同的素数必须被分配一个字母`A`按照素数递增的顺序向前，将每个素数替换为对应的字母，形成解码后的字符串。 

输入大小足够大，任何独立分解每个数字的尝试都会太慢。 如果对许多测试用例天真地进行，即使每个元素进行单个 64 位整数分解也可能变得临界。 这促使我们使用相邻产品之间的结构信息，而不是孤立地处理每个值。 

当相同的乘积重复或小素数主导早期值时，就会出现微妙的边缘情况。 例如，如果序列以非常小的素数（如 2 和 3）开始，则相邻的乘积（如 6、6、10）可能会误导简单的因式分解方法，因为合数通常存在多重因式分解。 然而，这里每个值都保证是两个素数的乘积，这消除了歧义，但仍然需要仔细传播。 

另一种失败模式来自于错误地假设我们总是可以直接分解第一个元素。 例如，如果第一个产品是 15，它可能是 3 × 5 或 5 × 3，如果没有上下文，我们无法确定方向。 该解决方案来自相邻产品之间的重叠约束。 

## 方法

 暴力方法会尝试将每个乘积独立地分解为两个素数，然后尝试通过检查相邻位置之间的一致性将它们缝合在一起。 对于每个值，我们可以尝试所有素因数直至其平方根。 在最坏的情况下，如果值很大并且有很多测试用例，这会变得昂贵。 单独的因式分解步骤可以接近 O(n√m)，其中 m 是值的大小，这在典型约束下是不可行的。 

关键的见解是相邻产品共享素数。 如果我们有两个连续的值`q[i] = p[i] * p[i+1]`和`q[i+1] = p[i+1] * p[i+2]`，那么它们的最大公约数正是`p[i+1]`。 这消除了显式因式分解的需要。 一旦我们恢复了序列中的一个素数，剩下的就可以通过简单的除法得到。 

然后重建就变得确定了。 我们找到一个位置，其中两个连续乘积的不同之处在于产生一个不平凡的 gcd，提取共享素数，并在两个方向上向外扩展。 

最后，一旦知道完整的素数列表，对其不同值进行排序就可以得到字母表映射。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力分解| O(n√m) | O(n√m) | O(n) | 太慢了|
 | GCD 传播 | O(n log m) | O(n log m) | O(n) | 已接受 |

 ## 算法演练

 我们假设产品数组是`q`长度`n`，隐藏素数序列为`p`长度`n+1`。 

1. 扫描相邻的产品对，直到找到索引`i`在哪里`q[i] != q[i+1]`。 这保证了共享素数隐藏在相同值后面并不是微不足道的，并且它为我们提供了一个可用的 gcd 信号。 
2. 计算`g = gcd(q[i], q[i+1])`。 这个值正好是`p[i+1]`，两种产品共享的素数。 这样做的原因是两个乘积都包含相同的中间素数，因此它必须将两个数字相除。 
3.立即恢复周围素数：`p[i] = q[i] / g`和`p[i+2] = q[i+1] / g`。 此步骤将重建锚定在已知中心周围。 
4. 从索引向左传播`i`下降到`0`使用`p[j] = q[j] / p[j+1]`。 每个乘积唯一地确定了前一个素数，因为我们已经知道下一个素数。 
5. 从索引向右传播`i+2`最多`n`使用`p[j] = q[j-1] / p[j-1]`。 这同样会向前重建链。 
6. 收集所有不同的素数`p`并对它们进行排序。 指定从以下位置开始的字母`'A'`按质值递增的顺序。 
7. 通过替换中的每个素数来构建最终的解码字符串`p`及其指定的字符。 

### 为什么它有效

 重构依赖于每个产品的不变量`q[i]`正是`p[i] * p[i+1]`，并且所有值都是素数。 一旦锚定了任何单个正确的相邻对，后续的每个步骤都会使用除以已知素数的方法，该素数唯一地确定下一个值。 由于素数没有非平凡因数，因此任何传播步骤都没有歧义，并且 gcd 步骤保证我们从正确的共享素数开始。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve():
    n = int(input().strip())
    q = list(map(int, input().split()))
    
    p = [0] * (n + 1)
    
    # find first non-equal adjacent pair
    idx = 0
    while idx < n - 1 and q[idx] == q[idx + 1]:
        idx += 1
    
    g = gcd(q[idx], q[idx + 1])
    p[idx + 1] = g
    p[idx] = q[idx] // g
    p[idx + 2] = q[idx + 1] // g
    
    for i in range(idx - 1, -1, -1):
        p[i] = q[i] // p[i + 1]
    
    for i in range(idx + 3, n + 1):
        p[i] = q[i - 1] // p[i - 1]
    
    distinct = sorted(set(p))
    mp = {v: chr(ord('A') + i) for i, v in enumerate(distinct)}
    
    res = ''.join(mp[x] for x in p)
    print(res)

if __name__ == "__main__":
    solve()
```实现首先在相邻产品不同的地方定位一个稳定的断点。 这确保了 gcd 干净地揭示共享素数。 一旦知道中心素数，就仅使用除法在两个方向上向外填充数组，这是安全的，因为每个值都保证是素数。 

一个常见的错误是尝试在完全重建序列之前分配字母，这可能导致不一致的映射。 另一个微妙的问题是忘记素数序列长度比乘积数组长度大一，因此输出总是长一个字符。 

## 工作示例

 ### 示例 1

 输入：```
n = 5
q = [6, 15, 35, 14, 10]
```我们计算相邻的 gcd 直到找到不同的对。 在`15`和`35`, 最大公约数是`5`。 

| 步骤| q[i] | q[i+1] | q[i+1] GCCD | p 重建 |
 | --- | --- | --- | --- | --- |
 | 寻找支点 | 15 | 15 35 | 35 5 | p[i+1]=5 | p[i+1]=5
 | 展开 | 6 | 15 | 15 - | p[0]=2 | p[0]=2
 | 展开 | 35 | 35 14 | 14 - | p[4]=7 | p[4]=7 | p[4]=7
 | 决赛| - | - | - | [2,3,5,7,2,5] |

 不同的素数是`[2,3,5,7]`，映射到`A,B,C,D`。 

结果字符串变为：`A B C D A C`→`ABCDAC`该轨迹表明，一旦中心 5 被固定，整个序列就会通过除法确定性地崩溃。 

### 示例 2

 输入：```
n = 3
q = [21, 14, 35]
```枢轴位于`21`和`14`, 最大公约数是`7`。 

| 步骤| q[i] | q[i+1] | q[i+1] GCCD | p 重建 |
 | --- | --- | --- | --- | --- |
 | 枢轴| 21 | 21 14 | 14 7 | p = [3,7,2,5] |

 不同素数`[2,3,5,7]`映射到字母`A,B,C,D`。 

输出变为`B D A C`→`BDAC`。 

这个例子表明，即使序列很短，相同的基于 gcd 的锚定也可以毫无歧义地工作。

 ## 边缘情况

 只有一个产品的最小长度情况测试实现是否正确生成两个字符的输出。 由于单个乘积对应于两个素数，因此该算法仍必须分配大小为 2 的完整素数数组并一致地映射两个值。 

连续乘积相等的情况可以隐藏 gcd 信号。 例如，如果`q = [6, 6, 6]`，循环必须跳过相同的相邻值，直到找到不同的对。 一旦遇到不相等的对，就应用相同的重建。 如果没有这个跳过，幼稚的实现可能会计算相同值的 gcd，并错误地假设不存在的中心结构。 

枢轴位于数组开头或结尾的边界情况需要仔细索引。 当不同的对位于索引处时`0`，最初只能进行前向传播，反向循环变得微不足道。 基于除法的重建仍然有效，因为每一步仅取决于先前已知的素数，而不取决于对称性。
