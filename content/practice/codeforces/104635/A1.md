 ## 解决方案
 ## 问题理解

 给定一个以十进制形式写成的大整数，我们需要将其拆分为两个非负整数，其总和等于原始数字。 附加约束是两个结果数字都不允许在其十进制表示中包含数字 4。 

考虑输入的一个有用方法是，它不是我们应该进行算术处理的数字，而是允许我们逐位分解的数字字符串。 输出是两个表示数字的字符串，它们的数字总和必须准确地重构原始输入。 

约束通常足够大，数字可以达到数万位。 这立即排除了任何涉及整数算术或重复减法的方法。 任何对每个数字进行恒定工作的解决方案都是可以接受的，但是任何数字数量的二次方仍然可以。 我们必须避免的是任何涉及前缀重新计算或以简单的方式模拟进位加法的事情。 

一个微妙的边缘情况来自等于 4 的数字。如果我们尝试简单地任意分割数字而不考虑约束，我们可能会意外地在其中一个输出中留下 4。 例如，如果输入的数字是4，并且我们将其完全分配给一个数字，则该数字无效。 另一个边缘情况是前导零：其中一个输出可能自然具有前导零，这是可以接受的，因为我们正在使用数字字符串，而不是规范的整数格式。 

## 方法

 一种强力的想法是将问题视为找到两个数字 A 和 B，使得 A + B 等于 N 并且都不包含数字 4。可以想象迭代所有可能的方式来分配进位和数字分割，甚至生成候选对并检查有效性。 这很快就变得不可行，因为即使对于中等大小的输入，可能的分解数量也会随着位数呈指数增长。 如果我们尝试任意分割，每个数字实际上都会使决策空间加倍，这会导致 d 个数字有 2^d 种可能性。 

关键的观察是我们不需要优化任何东西或找到独特的分解。 我们只需要任何有效的分解。 这消除了数字之间的所有耦合。 输入的每个数字都可以独立处理，因为不需要最小化或最大化任一输出数字，只需保留总和并避免输出中的数字 4。 

这允许一种直接构造策略：独立处理每个数字，并将其分配给两个输出数字，以保证没有数字变成 4。最简单的安全分割是给一个数字所有原始数字减去所选部分，另一个数字剩下余数。 特别是，每当我们看到 4 时，我们都会将其分为 3 和 1。对于所有其他数字，我们可以将整个数字分配给第一个数字，将 0 分配给第二个数字。 

这是可行的，因为这里不带进位的数字加法就足够了：我们通过确保在每个位置，构造的数字之和完全等于原始数字来故意避免进位交互。

 | 暴力枚举| 指数| O(n) | 太慢了|

 ## 算法演练

 我们构造两个与输入数字长度相同的字符串 A 和 B。

1. 将输入的数字作为字符串读取，以便每个数字都可以独立处理。 这可以避免任何整数溢出或不必要的算术。 
2. 初始化两个空字符串 A 和 B。它们将存储两个结果数字的数字。 
3. 迭代输入字符串中的每个字符 d。 在每个位置，决定如何在 A 和 B 之间分配该数字。 
4. 如果当前数字为“4”，则在 A 后附加“3”，在 B 后附加“1”。这样可以保证 A 和 B 都不包含禁止数字，同时仍保留该位置的和。 
5. 如果数字不是“4”，则将数字本身附加到 A 并将“0”附加到 B。这样可以将贡献完全保留在一个数字中，同时确保 B 保持干净和简单。 
6. 处理完所有数字后，将 A 和 B 作为两个单独的整数输出。 

为什么它有效可以归结为每个数字的不变量。 在每个索引处，A 中的数字加上 B 中的数字等于原始数字，并且构造的数字都不会变成 4。由于我们从未引入进位，因此每个位置都是独立的，并且数字相等直接意味着结果和的完全数值相等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    a = []
    b = []

    for ch in s:
        if ch == '4':
            a.append('3')
            b.append('1')
        else:
            a.append(ch)
            b.append('0')

    print("".join(a), "".join(b))

if __name__ == "__main__":
    solve()
```该实现准确地反映了数字结构。 为了提高效率，我们构建两个字符数组，而不是重复连接字符串，由于不可变性，这在 Python 中会是二次的。 对数字 4 的特殊处理是唯一需要的分支逻辑。 其他的都直接通过。 

一个微妙的点是我们永远不需要从第二个数字中去掉前导零。 即使 B 以零开头，在这种情况下它仍然是非负整数的有效表示。 

## 工作示例

 考虑输入`940`。 

| 数字| 到目前为止| 到目前为止 B | 评论 |
 | --- | --- | --- | --- |
 | 9 | 9 | 0 | 复制的非 4 位数字 |
 | 4 | 93 | 93 01 | 将 4 分成 3 和 1 |
 | 0 | 930 | 930 010| 零到 A |

 输出是 A = 930，B = 010。它们的和是 940，并且都不包含数字 4。 

现在考虑`444`。 

| 数字| 到目前为止| 到目前为止 B | 评论 |
 | --- | --- | --- | --- |
 | 4 | 3 | 1 | 分裂|
 | 4 | 33 | 33 11 | 11 分裂|
 | 4 | 333 | 333 111 | 111 分裂|

 输出为 A = 333，B = 111，总和恰好为 444。这种情况强调受约束数字的重复出现，并表明位置之间的独立性就足够了。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |

 该解决方案在位数上是线性的，这是最佳的，因为我们必须至少读取输入一次。 内存使用也是线性的且不可避免，因为输出本身的大小是线性的。

 ## 边缘情况

 最小输入，例如`4`是最直接的压力测试。 该算法读取单个数字，将其检测为特殊情况，并生成 A = 3 和 B = 1。总和是正确的，并且两个输出都避免了数字 4。 

对于诸如这样的输入`4000`，算法独立处理每个位置。 第一个数字变成 3 和 1，其余的零变成 0 和 0。这会产生 A = 3000 和 B = 1000，它们的和精确地重建原始数字，位置之间没有任何相互作用。 

对于像这样的数字`9994`，只有最后一位数字触发分割。 前三位数字直接复制到 A 中，而 B 则在那里得到零。 在最后一个数字处，拆分确保了本地正确性，并且由于没有进位，因此较早的数字不受影响。
