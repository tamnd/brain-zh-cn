 ## 解决方案
 ## 问题理解

 我们得到一条加密消息，该消息最初是由一系列素数组成的。 每个字母首先被转换为一个素数，然后加密用每两个相邻素数的乘积替换素数序列。 因此，我们不是直接看到素数，而是看到一个整数列表，其中每个整数都是两个连续隐藏素数的乘积。 

任务是扭转这个过程。 从成对乘积列表中，我们必须重建素数的原始序列，恢复这些素数的排序，最后使用一致的排序规则将它们映射回字母：每个不同的素数对应于基于素数字母顺序的不同字母。 

因此，输入代表一系列重叠的约束。 每个数字都告诉我们有关两个相邻的未知素数的信息，并且每对相邻的数字都恰好共享一个素数。 输出是解码后的文本字符串。 

此问题中的约束是围绕长序列的乘积构建的，通常足够大，以至于 O(n²) 重建是不可行的。 暴力尝试猜测素数或独立分解每个数字会太慢，因为重复分解大整数的成本很高，并且相邻位置之间因子的天真匹配会导致二次或更糟糕的行为。 预期的解决方案必须依赖于重用相邻产品之间的共享结构。 

当出现重复的素数或者当第一个可恢复的 gcd 没有立即唯一地揭示最左边的素数时，就会出现微妙的边缘情况。 例如，如果所有素数都相同，例如 2 2 2 2，则每个乘积都是相同的，并且假设唯一因子转换的粗心重建可能无法正确传播。 另一个失败的情况是，当实现尝试独立地分解每个产品，然后贪婪地匹配交集时，如果没有一致地强制执行排序，这可能会导致歧义传播错误。 

## 方法

 直接的强力方法会尝试将每个密文数字分解为两个素数，然后将这些对拼接成一致的序列。 对于每个产品，我们可以枚举可能的因素对，并尝试将它们与邻居对齐。 在最坏的情况下，每个数字可能有多个有效的因式分解，并且检查整个序列的一致性变成了组合。 即使假设因式分解是有效的，匹配过程也会变成二次方，因为每个位置都依赖于邻居的多个候选选择，从而在最坏的情况下导致指数分支。 

关键的观察结果是相邻的密文值共享一个质数。 如果我们有两个连续的乘积 A = p_i · p_{i+1} 和 B = p_{i+1} · p_{i+2}，那么它们的最大公约数正是 p_{i+1}。 这完全消除了因式分解的需要。 一旦我们找到一个共享素数，我们就可以通过将每个乘积除以已知素数来向外传播以恢复邻居。 这将整个重建减少为线性扫描。 

唯一缺少的部分是如何获得起点。 我们采用 gcd 不是完整乘积的任何相邻对（只有在简并相同情况下才会发生），计算共享素数，然后在两个方向上进行重建。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 强力分解+匹配| O(n·sqrt(M)) 或更糟 | O(n) | 太慢了|
 | GCD 传播 | O(n log M) | O(n log M) | O(n) | 已接受 |

 ## 算法演练

 ### 步骤

1. 读取密文整数序列，每个密文整数代表两个连续隐藏素数的乘积。 
2. 找到一个索引 i，使得 gcd(cipher[i], cipher[i+1]) 有意义并揭示共享素数。 该值对应于该局部段中的中间质数。 
3. 一旦我们获得共享素数 p_{i+1}，就计算 p_i = cipher[i] // p_{i+1}。 
4. 通过向右延伸构建完整的素数序列。 对于每个下一个位置，将 cipher[j] 除以已知的前一个素数以恢复下一个素数。 
5. 同样地，以相反的方式使用相同的除法逻辑向左延伸。 
6. 收集所有不同的素数并对它们进行排序，以构建从素数值到从“A”开始的字母的映射。 
7. 使用该映射将重构的素数序列翻译成字符串。 

关键的设计选择是每一步都减少一个未知变量的不确定性。 一旦知道了单个锚素数，其他所有素数就变成了直接除法结果，而不是因式分解问题。 

### 为什么它有效

 每个密文值恰好编码两个相邻的素数。 两个连续密文值的 gcd 隔离了共享的中间素数，因为该素数是两个乘积的唯一共同因子。 一旦知道该值，每个剩余的素数都可以通过除法唯一确定，从而在重建中不会留下任何歧义。 传播确保了一致性，因为每个恢复的素数都会立即由下一个密文产品验证。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        p = [0] * (n + 1)

        # find first non-trivial gcd
        idx = -1
        for i in range(n - 1):
            g = gcd(a[i], a[i + 1])
            if g != a[i]:
                idx = i
                p[i + 1] = g
                break

        # build left side
        for i in range(idx, -1, -1):
            p[i] = a[i] // p[i + 1]

        # build right side
        for i in range(idx + 1, n):
            p[i + 1] = a[i] // p[i]

        # build mapping
        vals = sorted(set(p))
        mp = {v: chr(ord('A') + i) for i, v in enumerate(vals)}

        res = ''.join(mp[x] for x in p)
        print(res)

if __name__ == "__main__":
    solve()
```该实现首先扫描相邻密文值揭示非平凡 gcd 的位置。 该 gcd 直接给出了链中间的隐藏素数之一。 从该锚点开始，代码使用整数除法在两个方向上重建完整序列，这是安全的，因为每个密文都保证是两个相邻素数的乘积。 

重建后，通过对唯一值进行排序，素数被压缩为更小的字母表。 此步骤至关重要，因为相同的素数必须始终映射到相同的字母，并且按大小排序提供了确定性编码。

 ### 示例 1

 考虑一个简单的隐藏素数序列：

 输入密文：```

```这对应于素数：```

```| 步骤| 密码对 | GCD | 行动| 素数数组 |

 | 2 | 35 // 5 | - | 恢复权利| 3, 5, 7, _ |

该迹线表明单个 gcd 锚点足以重建完整的链而不会产生歧义。

 ## 边缘情况

 一种边缘情况是所有密文值都相同，这意味着所有素数都相同。 对于像这样的输入`4 4 4 4`，每个 gcd 等于完整值，因此算法仍然必须选择一个有效的锚点。 实际上，任何位置都有效，因为除法总是产生相同的素数，并且重构在整个链上保持一致。 

另一个边缘情况是第一次 gcd 扫描在早期索引处没有发现立即的非平凡分裂。 在这种情况下，算法会继续扫描，直到找到有效的锚点，确保即使第一对是退化的，后面的对也会暴露共享素数。 

最终的边缘情况是长度为 1 的最小可能输入。这里没有邻接，因此单个数字被解释为一个素数的简并编码，并且在压缩后直接映射到第一个字母。
