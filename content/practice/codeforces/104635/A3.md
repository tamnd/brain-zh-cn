 ## 解决方案
 ## 问题理解

 我们得到一个非常大的非负整数，写成字符串。 任务是将这个数字拆分为两个非负整数，分别称为 A 和 B，这样当我们将它们按数字相加时，我们可以恢复原始数字，并且 A 和 B 的十进制表示形式都不包含数字 4。 

关键的限制不在于算术难度，而在于表示。 我们可以自由地逐位选择 A 和 B，只要它们的和与原始数字匹配，并且两个构造的数字都完全避免数字 4。 由于输入可能非常大，可能远远超出标准整数限制，因此所有操作都必须在字符串表示上完成。 

从复杂性的角度来看，输入长度是关键因素。 如果数字最多为 10^5 位，则任何二次解或涉及循环内重复字符串重建的解决方案都将失败。 唯一可行的方法是数字数量呈线性。 

当输入包含强制仔细拆分的数字时，会出现微妙的边缘情况。 例如，当数字为 4 时，我们不能将其直接分配给 A 或 B，因为这会立即违反约束。 如果一种简单的方法试图通过后处理或进行调整来“修复”这个问题，就有可能破坏总和不变性。 

例如，考虑输入：```
44
```粗心的尝试可能会直接分配两个数字，产生 A = 44 和 B = 0，这是无效的，因为 A 包含 4。即使尝试“在构造后用其他东西替换 4”也可能会破坏求和关系。 

核心困难在于我们必须同时满足按数字求和约束和禁止数字约束。 

## 方法

 暴力破解的想法是尝试所有可能的方法将原始数字的每个数字拆分为两个数字 (a, b)，使得 a + b 等于原始数字，并且 a 和 b 都不是 4。对于每个数字，我们可以枚举所有有效的对并将它们组合到整个字符串中。 

这是可行的，因为如果我们强制不执行进位传播，则每个数字在无进位的加法方面都是独立的。 然而，一旦我们尝试处理一般的数字分割，组合的数量就会随着数字的长度呈指数增长。 即使对于 30 位数字，这也是不可行的，因为每个位置可能有多个有效的分割，导致 2^n 或更糟糕的可能性。 

关键的观察是我们没有被要求以任何方式优化 A 或 B。 我们只需要存在。 这允许贪婪的每位数构造。 对于输入中的每个数字 d，我们直接分配一对 (a, b)，使得 a + b = d 并且两个数字都避免 4。最简单的稳定结构是：

 如果 d 不为 4，则指定 a = d 且 b = 0。 

如果 d 为 4，则指定 a = 1 且 b = 3。 

这是可行的，因为 1 + 3 = 4，并且 1 和 3 都不包含禁止数字。 所有其他数字在放入 A 中和将 0 放入 B 中时已经满足约束。 

数字的独立性是关键的结构属性。 不存在进位交互，因为我们从不允许在单个位置内进行多位分解。

 ## 算法演练

 1.将输入数字作为字符串读取，这样我们就可以安全地处理任意大的值而不会溢出。 
2. 初始化两个空列表 A 和 B，它们将存储两个结果数字的数字。 
3. 迭代输入字符串的每个字符数字 d。 
4. 如果 d 不是“4”，则将 d 附加到 A 并将“0”附加到 B。 
5. 如果 d 为“4”，则将“1”附加到 A，将“3”附加到 B。 
6. 处理完所有数字后，将 A 和 B 连接成字符串。 
7. 输出A和B，用空格分隔。 

第 5 步中的选择是唯一重要的部分。 我们特意将 4 分成 1 和 3，因为两者都是有效数字，并且它们的总和恰好为 4，从而确保每个位置的局部正确性。 

### 为什么它有效

 该算法依赖于每个数字的不变量：在每个位置 i，我们确保 A[i] + B[i] 等于位置 i 处的原始数字，不引入或不需要进位。 由于每个数字都是独立处理的，并且没有任何步骤会产生大于 9 的数字，因此永远不会出现跨位置交互。 这保证了按数字求和准确地连接到原始数字，同时也确保构造的数字都不包含数字 4。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    a = []
    b = []

    for ch in s:
        if ch == '4':
            a.append('1')
            b.append('3')
        else:
            a.append(ch)
            b.append('0')

    print("".join(a), "".join(b))

if __name__ == "__main__":
    solve()
```该解决方案并行构建两个数字。 使用列表而不是重复的字符串连接至关重要，因为字符串在 Python 中是不可变的，并且重复的连接会将性能降低到二次级。 

循环内部的逻辑直接编码数字分割规则。 不需要额外的状态，因为每个数字都是独立的。

 ## 边缘情况

 最重要的边缘情况是输入完全由数字 4 组成。在这种情况下，每个位置都被迫分割 (1, 3)，并且算法会生成全 1 和全 3 的两个字符串。 对于输入：```
444
```该过程产生 A = 111 和 B = 333。每个位置独立地满足总和约束，因此全局结果仍然有效。 

另一种情况是根本没有4s。 该算法永远不会触发特殊拆分，因此 A 变为原始数字，B 变为全零。 对于输入：```
709
```我们得到 A = 709 和 B = 000，这很容易满足约束，因为这两个数字都不包含 4。 

边界情况是一位数输入。 相同的逻辑无需修改即可应用，因为该算法不依赖于位置上下文或进位。
