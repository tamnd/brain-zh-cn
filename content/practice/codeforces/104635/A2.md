 ## 解决方案
 ## 问题理解

 我们得到一个写成十进制数字字符串的数字。 任务是将这个数字拆分成另外两个数字，这样将它们相加就可以用正常的以 10 为底的加法逐位准确地重建原始数字，并且两个结果数字都不允许包含数字 4。 

可以将其视为获取原始数字的每个数字，并决定如何在两个新数字 A 和 B 之间分配其值，以便 A + B 等于原始数字，而无需在任一构造数字中放置 4。 

输入是写为字符串的单个整数。 输出是两个相同长度的整数，分别代表A和B。 

这些约束隐含地很大，因为这是一个 Codeforces 字符串操作问题。 位数可以合理地达到 10^5 或更多，这排除了任何通过多次进位来模拟算术或尝试搜索组合的方法。 任何位数的二次方都会立即失败。 该解决方案必须通过独立处理每个数字来以线性时间运行。 

当数字包含 0 或 4 时，会出现一种微妙的边缘情况。一种天真的想法可能是将所有内容分配给一个数字，而将另一个数字保留为零，但当数字为 4 时，这种方法会失败，因为 4 是被禁止的。 另一个潜在的错误是试图在保留全局进位结构的同时贪婪地调整数字，这是不必要的并且容易出错。 关键特性是每个数字都可以独立处理而不会产生干扰。 

例如，如果输入是 940，简单的分割可能会尝试 A = 940 和 B = 0，但这是无效的，因为 A 包含 4。正确的答案是 A = 930 和 B = 010，它相加正确并且避免了两个数字中的数字 4。 

## 方法

 暴力破解的想法是将其视为数字分解问题，其中输入的每个数字都可以拆分为两个数字，其总和等于它，然后验证结果数字是否满足不包含数字 4 的约束。简单的实现可能会尝试每个数字的所有可能拆分，传播进位，并全局检查有效性。 即使每个数字只有几个选项，管理所有位置的进位的需要也会创建一个组合状态空间。 在最坏的情况下，这会随着位数呈指数增长，因为每个位置都取决于先前的进位状态和所有先前的决策。 

关键的观察是，我们实际上不需要优化任何东西或考虑每个数字的多个分解。 我们只需要一个有效的构造。 如果我们确保不需要进位，则可以单独处理每个数字。 这建议选择按数字进行分割，其总和与原始数字精确相加，而不产生进位。 保证这一点的最简单方法是确保每个位置独立贡献，这意味着 A[i] + B[i] = N[i] 没有进位传播。 

一旦我们确定了这个想法，构建就变得确定了。 对于除 4 之外的每一位数字，我们可以将其完全分配给一个数字，并将零分配给另一个数字。 对于数字 4，我们将其拆分为 2 和 2。这样可以保证不会出现数字 4，并保持逐位的正确性。

 ## 算法演练

1. 将输入的数字作为字符串读取，以便我们可以逐位处理它，而无需转换为整数。 这可以避免溢出并保持逻辑纯粹的位置性。 
2. 初始化两个空列表或字符串 A 和 B，用于存储构造的数字。 
3. 迭代输入字符串中的每个字符数字 d。 每一步的目标是选择两个数字 a 和 b，使得 a + b = d 并且 a 和 b 都不等于 4。 
4. 如果当前数字为“4”，则分配 a = 2 且 b = 2。这是唯一安全的分割，可以避免引入禁止数字，同时也防止进位。 
5. 否则，分配 a = int(d) 且 b = 0。这将保持 A 中的值不变，同时保持 B 干净简单。 
6. 将所选数字分别附加到 A 和 B，将它们构建为字符串。 
7. 处理完所有数字后，输出A和B。 

### 为什么它有效

 该结构保证在每个位置，A 和 B 中的数字之和等于原始数字，并且由于我们从未产生任何总和超过 9 的数字对，因此位置之间不存在进位。 这种跨数字的独立性确保了局部正确性意味着全局正确性。 唯一的特殊情况是数字 4，通过将其均匀分割来解决，完全避免禁止的数字，同时仍保留数字和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    a = []
    b = []
    
    for ch in s:
        if ch == '4':
            a.append('2')
            b.append('2')
        else:
            a.append(ch)
            b.append('0')
    
    print("".join(a), "".join(b))

if __name__ == "__main__":
    solve()
```该解决方案将数字读取为字符串，并在一次传递中构造两个输出。 关键的实现细节是我们永远不会在整数和字符串之间来回转换整个数字，这避免了性能问题和意外进位逻辑。 

唯一重要的决定是数字 4 的处理。分配 (2, 2) 确保有效性和正确性。 所有其他数字都可以安全地完全分配给一侧，因为它们不违反约束。

 ## 边缘情况

 关键的边缘情况是输入仅包含零。 用于输入`0`，算法产生`0`和`0`，这是有效的，因为没有出现禁止数字并且总和保持正确。 

另一个边缘情况是单个数字 4。算法直接将其映射到`2`和`2`，产生有效的分解，而不需要任何周围的上下文或进位处理。 

更微妙的情况是数字以 4 结尾，例如`1234`。 最后一位数字变为`2`和`2`，并且由于前面的数字未受影响，因此任何进位都不能向后或向前传播，从而保持了正确性。
