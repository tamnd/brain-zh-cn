 ## 解决方案
 ## 问题理解

 我们有一个大小为 n × n 的正方形网格。 一个名叫 Lydia 的人已经选择了一条从左上角到右下角的路径，仅向右或向下移动。 该路径被编码为长度为 2n − 2 的字符串，其中每个字符代表一次移动：向右一步或向下一步。 

我们的任务是构建从相同起点到相同目的地的任何其他有效路径，具有相同的移动约束，这样我们的路径与莉迪亚的路径不相同。 

输出也是一串长度为 2n − 2 的字符串，由相同的移动符号组成，但它必须至少在一个位置与给定符号不同。 

关键约束是两条路径必须是有效的网格路径，这意味着它们在每个方向上恰好包含 n − 1 次移动。 这确保我们始终保持在网格内并到达右下角。 

尽管输入大小足够小，线性扫描微不足道，但路径有效性的约束意味着我们不能任意排列移动，除非我们保留计数。 

一种简单的方法是尝试生成所有有效路径并将它们与莉迪亚的进行比较。 这会立即失败，因为有效路径的数量是组合的，大约为$\binom{2n-2}{n-1}$，随着 n 呈指数增长。 

当试图在保持可行性的同时贪婪地“稍微修改”莉迪亚的路径时，会出现一个更微妙的失败案例。 例如，在不检查可行性的情况下提前更改移动可能会强制使用不可能的后缀。 

失败场景示例：

 输入：```
3
E S E S
```如果我们尝试翻转第一个字符以匹配相反的方向而不考虑计数，我们可能会因稍后产生太多的移动而破坏有效性，从而无法到达目的地。 

真正的要求更简单：我们只需要一条在某处不同的有效路径，而不是字典顺序约束或局部最优的路径。 

## 方法

 暴力方法会枚举从 (0, 0) 到 (n − 1, n − 1) 的所有路径，对照 Lydia 的路径检查每条路径，并在第一个不匹配处停止。 这是正确的，因为考虑了每条有效路径，但此类路径的数量随 n 呈指数增长，即使对于中等值也是不可行的。 在最坏的情况下，大约有$O(2^{2n})$候选序列。 

关键的观察是，除了有效性和差异性之外，我们不需要任何结构。 任何不相同的有效路径都是可接受的。 这消除了除了保持 E 和 S 移动的正确数量之外的所有排序限制。 

由于 Lydia 的路径恰好包含 n − 1 个 E 移动和 n − 1 个 S 移动，因此我们可以通过独立地转换每个移动同时保留全局计数来构造另一条有效路径。 最简单的一致变换是交换每个动作：将每个 E 替换为 S，将每个 S 替换为 E。这保证了我们仍然准确地生成每个动作的 n − 1 个，因为在交换下保留了多个动作集。 

这种构造还保证了与 Lydia 路径的差异，因为每个位置都被翻转，所以相等是不可能的，除非字符串在交换 E 和 S 下是对称的，这对于非空有效路径来说是不可能发生的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| O(2^(2n)) | O(2^(2n)) | O(2n) | O(2n) | 太慢了|
 | 交换每一步| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 读取整数 n 和长度为 2n − 2 的路径字符串 s。该字符串对 Lydia 在网格上的移动进行编码。 
2. 创建一个空结果字符串r。 我们将一个一个地构建我们的替代道路。 
3. 迭代 s 中的每个字符。 对于每个位置，应用确定性转换：如果字符是“E”，则将“S”附加到r，如果是“S”，则将“E”附加到r。 这确保了每个动作都是独立翻转的。 
4.处理完所有字符后，输出r作为最终路径。 

### 为什么它有效

 该构造保留了每次移动的总数，因为每个 E 都会转换为 S，每个 S 都会转换为 E。由于 Lydia 的路径已经包含每个方向的 n − 1 个，因此转换后的路径也包含每个方向的 n − 1 个，确保它从开始到结束都是有效的网格路径。 由于每个字符的双射翻转，两条路径在每个位置都不同，因此不可能与原始路径相等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()
    
    res = []
    for c in s:
        if c == 'E':
            res.append('S')
        else:
            res.append('E')
    
    print("".join(res))

if __name__ == "__main__":
    solve()
```该实现一次读取输入路径并构造输出。 转换逻辑是每个字符本地的，这避免了跟踪前缀约束或网格坐标的需要。 唯一微妙的一点是确保输入字符串被完全消耗，并且除了读取 n 和路径之外不需要额外的解析。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 一种边缘情况是最小的网格，其中 n = 2 并且路径长度为 2。输入可能类似于 ES。 该算法翻转每个字符，产生 SE，它仍然是一条有效路径，因为它仍然包含一个 E 和一个 S。 

另一种情况是Lydia的路径严格交替时，例如ESESESE。 该转换仍然有效，因为每个字符独立于相邻字符，因此交替模式在结构上保留但反转，产生 SESESES，这仍然有效。 

第三种情况是当路径被分组时，例如所有E移动后面跟着所有S移动。 即使在这里，翻转也会产生所有 S，然后是所有 E，保持精确的计数和有效性，同时保证完全不同的路线。
