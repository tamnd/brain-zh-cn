 ## 解决方案
 ## 问题理解

 我们有一个大小为的正方形网格$N \times N$。 其他人 Lydia 已经选择了从左上角到右下角的有效路径，每一步仅向右或向下移动。 她的路径以由两个字符组成的字符串形式提供，其中每个字符代表网格中的一步。 

我们的任务是构建一条从相同起点到相同目的地的不同有效路径，也仅使用向右和向下移动，但约束条件是我们的路径不得与 Lydia 的路径相同。 换句话说，我们的路线中至少有一步的位置与她的不同，所以整体的动作顺序并不完全相同。 

输入对单个测试用例进行编码：网格的大小和 Lydia 的路径字符串。 输出是仍然到达目的地的相同长度的任何有效替代路径。 

约束足够小，我们只需要对字符串进行线性处理。 由于路径长度为$2N - 2$， 任何$O(N)$建设就足够了。 任何涉及回溯或路径枚举的事情都是不必要的，并且会引入可避免的复杂性。 

此类问题中的一个天真的错误是试图构建一条早期偏离的路径，而不保证它仍然在右下角结束。 例如，如果我们尝试“交换”移动而不跟踪向右和向下步骤的剩余平衡，我们可能会产生无效的前缀。 

如果试图任意更改一个字符，就会出现另一个微妙的问题。 如果莉迪亚的路径早期在一个方向上大部分是单调的，如果没有在上下文中仔细处理，天真的翻转仍然可能意外地再现相同的完整序列。 

## 方法

 蛮力的想法是仅使用向右和向下的移动来尝试从左上角到右下角的所有可能的有效路径，并将每条路径与莉迪亚的路径进行比较。 有$\binom{2N-2}{N-1}$此类路径呈指数增长$N$。 即使对于中等$N$，这变得完全不可行，因为路径的数量已经是天文数字了。 

关键的观察是，我们不需要“特殊”的不同路径，只需要任何不相同的路径。 这几乎消除了问题的所有复杂性。 由于有效路径完全由多组移动决定，因此保留向右和向下移动计数的任何重新排列仍然会产生有效的解决方案。 

莉迪亚的路径恰好包括$N-1$向右移动并且$N-1$向下移动。 如果我们构造一条交换每种移动类型的路径，将每个右转为下，每个下转为右，我们会自动保留有效性：计数保持相同，因此端点仍然正确，并且在每个前缀处，我们永远不会超出网格边界，因为我们只是以相反方向模式遍历对称路径。 

重要的结构见解是，只要我们保持在前缀可行性范围内，有效性仅取决于每次移动的总数，而不是它们的顺序，而前缀可行性在这种完全反转下得以保留。

 ## 算法演练

 1.读取整数$N$和字符串$S$描述莉迪亚的道路。 字符串长度为$2N - 2$，所以它完全决定了移动的顺序。 
2. 初始化一个空结果列表，该列表将逐个字符存储我们的替代路径。 
3.遍历中的每个字符$S$从左到右。 
4. 对于每个字符，应用直接转换：如果字符是“E”，则在结果中附加“S”； 如果是“S”，则附加“E”。 这保证了每个步骤都与 Lydia 的相应步骤不同，同时保留了每种移动类型的总数。 
5. 处理完整字符串后，将结果列表连接成单个字符串并输出。 

### 为什么它有效

 构建的路径包含与莉迪亚路径完全相同数量的“E”和“S”移动，因为每次出现都是一对一交换的。 这确保了端点保持不变，因为到达右下角仅取决于水平和垂直移动的正确计数。 同时，序列中的每个位置都与莉迪亚对应的位置不同，因此路径不可能相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()
    
    res = []
    for c in s:
        if c == 'E':
            res.append('S')
        else:
            res.append('E')
    
    print("".join(res))

if __name__ == "__main__":
    solve()
```实施直接反映了构造。 该循环对输入字符串执行单次传递，确保线性时间行为。 使用列表来构建输出的决定避免了重复的字符串连接，否则会在 Python 中引入二次开销。 

一个微妙的点是网格边界不需要额外的验证。 由于变换保留了每次移动的确切数量，因此我们隐式地维护从开始到结束的有效格子路径。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    
    n = int(sys.stdin.readline().strip())

 ## 边缘情况

 当莉迪亚的路径一开始就完全由一个方向组成时，就会出现一种极端情况，例如所有“E”首先移动。 尝试仅更改单个位置的简单方法可能仍会生成一条在结构上与长前缀一致的路径，但完全反转可保证每个位置立即不同。 

另一种情况是最小的网格，其中$N = 2$。 每种类型只有一个动作，因此任何正确的解决方案都必须交换这一对。 该算法自然会处理这个问题，因为它独立地处理每个字符。 

对于高度交替的路径，转换仍然保持有效性，因为它不依赖于模式或结构，仅依赖于每个字符的反转。
