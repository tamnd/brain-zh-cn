 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，代表 Tanya 在爬楼梯时所说的话。 每次进入一个新的楼梯时，她都会从 1 开始计数，并继续逐步向上，直到该楼梯结束。 然后她立即继续另一个楼梯，再次从 1 开始。 

因此，该序列是通过连接多个连续的运行而形成的，其中对于某个正整数 k，每个运行的形式恰好为 1, 2, 3, ..., k。 我们的任务是将序列分成这些最大有效运行，并输出存在多少个这样的运行及其长度。 

该结构极其严格：每个楼梯都必须从 1 开始，接下来的每个元素恰好增加 1，一旦这种模式被打破，就必须开始一个新的楼梯。 

约束 n ≤ 1000 足够小，即使 O(n²) 方法也是可以接受的，但结构表明直接线性扫描就足够了。 

一些边缘情况很重要。 该序列可能由单个楼梯组成，例如 1 2 3 4 5，在这种情况下，答案是一个长度为 5 的块。它也可能由许多单步楼梯组成，例如 1 1 1 1，其中每个元素都是自己的楼梯，因为每个 1 都不能向前延伸。 另一个微妙的情况是当序列在中断后干净地重新开始时，例如 1 2 3 1 2 3 4，我们不能错误地跨越重置点进行合并。 

一个天真的错误是假设只要序列减少或保持不变，就会开始一个新的楼梯。 虽然这很接近，但正确的规则更严格：当违反预期的下一个值时，新的楼梯恰好开始，并且该预期值重置为 1。 

## 方法

 强力解释是尝试将序列所有可能的划分为片段，并检查每个片段是否形成有效的 1 到 k 级数。 对于每个候选分段，验证有效性需要分段长度的线性时间，并且分区的数量随着 n 呈指数增长，导致计算量爆炸。 即使是尝试在每个位置进行削减的更克制的方法，仍然可能面临二次或更糟糕的行为，具体取决于检查的实施方式。 

关键的观察结果是有效性条件是局部的和确定性的。 一旦我们进入楼梯，下一个期望的数字总是 current_index_in_stairway + 1。这意味着我们不需要猜测楼梯的终点在哪里，一旦模式中断，我们就可以贪婪地检测到它。 

我们只需从左到右扫描，保持当前楼梯中的预期下一个值。 如果当前数字符合预期，我们将继续。 如果没有，则前一个楼梯结束，并从该位置开始新的楼梯。 

这将整个问题简化为对数组的单次传递。

 ## 算法演练

1. 开始从左到右扫描序列，将每个连续的有效递增运行视为一个楼梯。 当我们发现这些楼梯时，我们将存储它们的长度。 
2. 将第一个楼梯长度初始化为 1，因为在任何有效序列中第一个元素必须始终为 1。 我们不需要在全球范围内验证这一点，因为该问题保证了有效性，但我们仍然用它来锚定结构。 
3. 对于序列中的每个下一个元素，将其与前一个元素加一进行比较。 这是当前楼梯的预期延续。 如果匹配，我们将当前楼梯长度延长 1。 
4. 如果数值与预期不符，则说明上一个楼梯已经结束。 我们存储它的长度，开始一个新的楼梯，并重置新段的期望。 新的段必须从1开始，这与序列结构一致。 
5. 处理完所有元素后，确保记录最后的楼梯长度，因为它不会因不匹配而关闭。 

### 为什么它有效

 在每个索引处，该算法都强制执行有效楼梯的唯一可能的延续规则：下一个数字必须恰好是前一个数字 + 1。因为每个楼梯都是独立的并且始终从 1 开始，所以一旦此条件失败，就没有其他分段仍然可以生成跨该边界的有效楼梯。 这使得每次切割都是强制的而不是选择的，因此贪婪分区是唯一确定的并且与序列的原始构造相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    res = []
    current_len = 0
    prev = None
    
    for x in a:
        if current_len == 0:
            current_len = 1
        else:
            if x == prev + 1:
                current_len += 1
            else:
                res.append(current_len)
                current_len = 1
        prev = x
    
    res.append(current_len)
    
    print(len(res))
    print(*res)

if __name__ == "__main__":
    solve()
```该实现保留当前楼梯的运行长度。 当连续的+1模式中断时，当前段被推入结果列表并开始新的段。 变量`prev`需要在 O(1) 时间内有效地检查每个元素的连续性条件。 

唯一的微妙之处是确保最后的段附加在循环之后，因为终止不会自然地触发中断条件。

 ## 边缘情况

 一个关键的边缘情况是序列频繁交替，例如`1 1 1 1`。 该算法将每次与预期增量的不匹配视为强制削减。 在第一个元素上，一个新的段开始。 在第二个元素上，预期值为 2，但我们看到的是 1，因此我们立即关闭第一个段并开始一个新段。 对每个元素重复此操作，生成四个长度为 1 的段，这与唯一有效的解释匹配。 

另一种情况是完美的连续爬升，比如`1 2 3 4 5`。 此处，预期值始终与实际值匹配，因此扫描期间不会触发剪切。 结果是单个段，算法仅将其附加在末尾，确认不存在不匹配对应于单个楼梯。 

混合结构如`1 2 3 1 2`证明边界的正确性。 第一个不匹配发生在第四个元素处，其中预期值为 4，但我们看到的是 1，强制在长度 3 之后进行分割。然后第二段从该重置点干净地继续进行，产生与原始结构唯一匹配的最终分段。
