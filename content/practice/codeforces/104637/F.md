 ## 解决方案
 ## 问题理解

 我们给出了几个独立的正整数对。 对于每一对，我们重复应用确定性变换：识别两个数字中较大的一个并从中减去较小的一个。 如果两个数字相等，我们将另一个数字减去一个，这会立即产生零。 

每个操作严格地减少至少一个数字，并且我们继续直到至少一个分量变为零。 任务是计算每个输入对在终止之前发生了多少次此类减法运算。 

约束最多允许 1000 对，每个值最大为 10^9。 逐一模拟每次减法的解决方案可能会退化为每步的线性功。 在最坏的情况下，例如成对的连续斐波那契数，运算次数与数字本身的大小成正比，使得直接模拟完全不可行。 

当数字相等时，会出现微妙的边缘情况。 例如，从(5, 5)开始，该过程执行一次操作并立即到达(0, 5)。 任何假设严格递减比率或仅考虑不等减少的推理都必须明确考虑这种终止行为，否则可能会在最终步数计数中出现相差一的错误。 

## 方法

 直接模拟准确地模拟了该过程。 在每一步中，我们都会从较大的值中减去较小的值并递增计数器。 这是正确的，因为它忠实地遵循规则，但其成本取决于需要多少减法步骤。 

关键问题是该过程可能需要执行非常多的步骤。 在最坏的情况下，当一个数字仅比另一个数字稍大时，每个操作都会将较大的值减少一点点。 这会导致更新链与较大的数字成比例，这远远超出了高达 10^9 的值可接受的范围。 

关键的观察是，在重复减法过程中，我们没有改变较小的数字，而较大的数字仍然明显更大。 相反，我们不断地减去相同的值，直到较大的数字低于它。 这正是整数除法的行为。 如果a远大于b，那么我们在一个概念块中执行floor(a / b)减法，将a减少到a % b。 这与欧几里得算法中使用的变换相同，只是我们还累积了压缩的减法次数。 

我们反复用较小值的余数替换较大值，并将商添加到答案中。 这一过程持续进行，直到一个值变为零。

 ## 算法演练

 ## 算法演练

1. 处理该对，使第一个值始终是较大的值。 交换确保我们始终在一致的方向上执行归约，这简化了我们减去多少次的推理。 
2. 如果较小的值为零，则立即停止。 无法进行进一步的操作，因此累计计数已经是最终的。 
3. 使用整数除法计算较大值能容纳较小值多少次。 该商准确地表示在较大值降至较小值以下之前，朴素过程将执行多少次减法运算。 
4. 将此商添加到操作的运行总计中。 这将许多重复的单个减法压缩为一个算术步骤，而不改变最终状态。 
5. 将较大值替换为对较小值取模后的余数。 这反映了一次执行所有这些减法之后的状态。 
6. 如有必要，再次交换角色，使较大的值始终位于最前面，然后重复该过程。 

这种迭代归约反映了欧几里得算法，不同之处在于我们不仅跟踪 gcd 进度，而且还累积每个归约阶段所采取的商步总数。 

### 为什么它有效

 在 a 大于 b 的任何时刻，减法过程都会重复将 (a, b) 变换为 (a - b, b)，然后 (a - 2b, b)，依此类推，直到 a 小于 b。 这样的步数正好是floor(a / b)，结果值为a mod b。 由于每次减法都保留了较小数字在此阶段保持不变的不变性，因此将整个序列压缩为除法不会错误地跳过或合并不同的结构状态。 因此，该过程相当于对欧几里德归约过程中遇到的所有商求和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_pair(a, b):
    ops = 0
    while b:
        if a < b:
            a, b = b, a
        ops += a // b
        a %= b
    return ops

def main():
    n = int(input())
    out = []
    for _ in range(n):
        a, b = map(int, input().split())
        out.append(str(solve_pair(a, b)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案的结构是反复强制我们总是从较大的值中减去较小的值。 循环不变式是`b`是每个除法步骤之前的较小元素。 表达式`a // b`准确计算之前发生了多少次朴素减法运算`a`低于`b`， 和`a %= b`将状态更新为所有这些减法之后的余数。 

终止条件`while b:`确保当一个数字变为零时进程停止，符合原始停止规则。 交换步骤至关重要，因为取模后，数字的角色可能会颠倒。

 ## 边缘情况

 当两个数字相等时，算法在终止前只执行一项操作。 对于输入 (5, 5)，交换是无关紧要的，第一次除法给出 5 // 5 = 1，之后余数变为 0。循环立即停止，符合相等导致单个减法步骤的规则。 

当一个数字远大于另一个数字时，例如 (100, 1)，算法会执行商 100 余数 0 的单个除法步骤。这与简单过程会重复减去 1 直到较大数字消失为止的事实相符。 

当数字是连续的斐波那契数时，例如 (21, 13)，每个模步骤都会将该对减少到前一个斐波那契数对，确保该过程采用多个欧几里德层。 该算法正确地累加每个商，反映了长减法链，否则在强力模拟中会很明显。
