 ## 解决方案
 ## 问题理解

 我们被要求计算有多少个 1 到 n 范围内的整数可以写成两种特殊形式中的至少一种：完全平方或完全立方。 如果一个数字可以两种写法，它仍然应该只计算一次。 

因此，对于每个查询 n，我们本质上是计算两个集合的并集的大小：i² 形式的所有数字和 j³ 形式的所有数字，限制为最多 n。 

就测试用例数量而言，输入规模较小，最多 20 个，但 n 值较大，最多 10⁹。 这立即排除了任何迭代直到 n 的所有整数的方法。 每个测试用例的线性扫描最多需要 10⁹ 次操作，这远远超出了一秒钟的范围。 

一个更微妙的约束含义是答案本身是由整数幂驱动的。 正方形和立方体都生长得很快，因此它们到n为止的计数约为n^(1/2)和n^(1/3)，当n为10⁹时，它们最多分别约为31623和1000。 这表明迭代根是可行的。 

一个幼稚的错误是重复计算既是平方又是立方的数字。 例如，64 既是 8² 又是 4³，因此它会出现在两个序列中。 任何简单地将正方形和立方体的数量相加的方法都会过多计算这些数字。 

第二个微妙的边缘情况是最小值。 当 n = 1 时，正方形和立方体序列都包含 1，但它仍然应该只贡献一次。 

## 方法

 强力解释是迭代从 1 到 n 的每个整数 x 并检查它是否是完美的平方或完美的立方。 检查 x 是平方还是立方需要计算整数根并验证它们，每次检查的时间复杂度为 O(1)。 这导致每个测试用例的时间复杂度为 O(n)，在 n = 10⁹ 时，每个查询大约有十亿次检查，即使在考虑多个测试用例之前也太慢了。 

关键的结构观察是我们不需要检查每个整数，只需要检查正方形和立方体的生成元。 每个正方形由某个整数 i 生成，其中 i² ≤ n，每个立方体由某个整数 j 生成，其中 j3 ≤ n。 这将问题简化为计算有效索引 i 和 j，即 sqrt(n) 和 cbrt(n)。 

然而，简单地计算 sqrt(n) + cbrt(n) 就会过多计算平方和立方数。 当一个数字是六次方时，它既是平方又是立方，因为 lcm(2,3) = 6。因此 k⁶ 形式的数字出现在两个列表中，并且必须减去一次。 

这将问题转化为幂集上的经典包含排除计数：计算平方、计算立方、减去六次方。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每次测试 O(n) | O(1) | O(1) | 太慢了|
 | 最佳 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 通过幂进行最佳计数

 1. 对于给定的 n，计算有多少个整数 i 满足 i² ≤ n。 这相当于floor(sqrt(n))。 这将计算直到 n 的所有方格。 
2. 计算有多少个整数j满足j³ ≤ n，即floor(cbrt(n))。 这将计算直到 n 的所有立方体。 
3. 计算有多少个整数k满足k⁶ ≤ n，即floor(n^(1/6))。 这计算的是平方和立方的数字，因为它们恰好是六次方。 
4. 将平方数和立方数相加，然后减去六次方数。 这会删除正方形和立方体集中计数的重复项。 
5. 输出每个测试用例的结果值。 

每个步骤有效的原因是我们不是直接迭代数字，而是迭代它们唯一的生成指数，并且每个变换在生成器变量中都是单调的。 

### 为什么它有效

对于某个整数 i，每个整数 x ≤ n 都有一个唯一的表示 x = i²，类似地，每个立方体都有一个唯一的表示 x = j³。 这创建了从整数 i 和 j 到有效数字集的两个单射映射。 唯一的重叠发生在一个数字可以同时写为 i2 和 j3 时，这意味着 x 是完美的六次方。 由于六次方在每组中计算一次，因此减去它们的计数可以纠正每个重叠元素的过度计数一次。 不存在其他重叠，因为指数表示在素数分解项中是唯一的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def isqrt(n: int) -> int:
    lo, hi = 0, 10**9
    while lo <= hi:
        mid = (lo + hi) // 2
        if mid * mid <= n:
            lo = mid + 1
        else:
            hi = mid - 1
    return hi

def icbrt(n: int) -> int:
    lo, hi = 0, 10**6
    while lo <= hi:
        mid = (lo + hi) // 2
        if mid * mid * mid <= n:
            lo = mid + 1
        else:
            hi = mid - 1
    return hi

def i6th(n: int) -> int:
    lo, hi = 0, 10**3
    while lo <= hi:
        mid = (lo + hi) // 2
        p = mid * mid
        p = p * p * p
        if p <= n:
            lo = mid + 1
        else:
            hi = mid - 1
    return hi

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = isqrt(n)
        b = icbrt(n)
        c = i6th(n)
        print(a + b - c)

if __name__ == "__main__":
    solve()
```该实现将三个计数任务分为整数根计算。 每个辅助函数都使用二分搜索而不是浮点运算，避免了大边界（如 10⁹）的精度问题。 

平方根函数最多搜索 10⁹，因为 sqrt(10⁹) 约为 31623，安全地在范围内。 立方根搜索空间的上限为 10⁶，这已经远远高于 cbrt(10⁹) ≈ 1000。第六根的上限为 103，因为 (103)⁶ = 101⁸，它轻松超过了所有可能的输入。 

最终公式直接应用包含-排除，而不存储任何中间集。 

## 工作示例

 ### 示例 1：n = 10

 我们计算平方、立方和六次方。 

| 步骤| 价值|

 这包括 100 以内的所有平方、100 以内的所有立方，并且从重复计算中删除了六次方。 

这些迹线表明，重叠校正始终准确地删除了两个序列中出现的那些数字。

 ## 边缘情况

 当 n = 1 时，平方和立方计数都等于 1，六次方计数也为 1。该公式生成 1 + 1 − 1 = 1，正确避免了重复计算。 

对于 n = 10⁹，平方根为 31622，立方根为 1000，六次方根为 10。减法可确保 10 的六次方不会计算两次。 该算法纯粹通过整数根计算来处理这个问题，没有溢出或精度问题。
