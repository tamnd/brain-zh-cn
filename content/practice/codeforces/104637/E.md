 ## 解决方案
 ## 问题理解

 我们得到了一个描述门打开顺序的序列。 每扇门都属于房屋的两个出口之一，左出口或右出口。 该序列没有描述几何或结构，它只是一步步告诉我们下一个打开的门是属于左侧还是右侧。 

在打开该序列的某个前缀后的任何时刻，每侧的一些门子集都已打开。 如果整个出口完全打开，布莱克先生就可以离开房子，这意味着属于该出口的每一扇门都已经出现在打开的前缀中。 

任务是找到最早的前缀长度，其中该条件对于两个出口中的至少一个为真。 

最多 200,000 个门的限制意味着对前缀的任何二次模拟都会太慢。 通过重新计算哪些门完全打开来检查每个前缀的方法在最坏的情况下将导致大约 n² 次操作，这远远超出了一秒限制可以处理的范围。 该解决方案必须依赖于单个线性扫描或每个元素的恒定量的额外工作。 

当一个出口的最后一扇门在序列中非常靠前，而另一个出口则晚得多时，就会出现微妙的边缘情况。 例如，如果所有右门都出现在开头附近，最后一个右门位于位置 3，但左门一直持续到位置 n，则答案为 3，即使左侧的大部分序列仍然不完整。 幼稚的前缀模拟可能会错误地等待两侧完全打开，而这不是必需的。 

## 方法

 暴力方式是检查每个前缀长度k，并检查左出口的所有门或右出口的所有门是否出现在该前缀内。 为了直接做到这一点，人们需要知道每一侧的门总数并计算到目前为止已经看到的门的数量。 一个简单的实现是，对于每个 k，扫描前缀并验证完整性，或维护集合并与总数进行比较。 

这是正确的，因为条件纯粹是关于类型的所有出现的覆盖范围。 然而，重新计算每个前缀的覆盖范围会导致重复工作。 对于 n 个元素，扫描前缀会导致大约 1 + 2 + ... + n 次操作，其数量级为 n²。 

关键的观察是，我们实际上并不关心中间计数，只关心每一边何时完全包含在前缀中。 当我们遇到序列中最后一次出现时，一侧完全打开。 在那一刻之前，它至少有一扇门还没有打开过。 从那一刻起，它就永远保持完全开放。 

这将问题简化为查找序列中每个值的最后一个位置。 答案就是这些完成点中最早的一个，因为一旦任何一方完成，退出就成为可能。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳（最后一次出现扫描）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 扫描序列一次，记录左门最后出现的位置和右门最后出现的位置。 这已经足够了，因为一侧的完成仅取决于其最终出现，而不取决于中间结构。 
2. 一旦知道最后两个位置，就将它们进行比较。 先完成的一方决定最早退出的机会。 
3. 输出最后出现的两个索引中的最小值。 

关键的想法是，我们不是在模拟这个过程，而是确定双方完全满意的确切时刻。 

### 为什么它有效

对于每个出口，其可用的时刻由其在序列中的最终出现决定。 在该位置之前，至少一扇所需的门仍未打开。 在该位置之后，该方所需的所有门都保证出现在前缀中。 因此，每一方都贡献一个阈值索引，答案是这些阈值中最小的一个。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_str(data: str) -> str:
    it = iter(data.strip().split())
    n = int(next(it))
    last0 = last1 = 0

    for i in range(1, n + 1):
        x = int(next(it))
        if x == 0:
            last0 = i
        else:
            last1 = i

    return str(min(last0, last1))

def solve():
    data = sys.stdin.read()
    sys.stdout.write(solve_str(data))

if __name__ == "__main__":
    solve()
```该实现仅依赖于跟踪两个整数，即每种类型的最后看到的索引。 该循环是 1 索引的，以直接匹配概念前缀长度，避免最后进行差一调整。 最终答案是在扫描后的恒定时间内计算出来的。 

一个常见的错误是试图维护运行计数并检查计数是否与总数匹配。 这种方法有效，但需要明确地了解总计数，并且如果实施效率低下，仍然存在额外开销的风险。 最后出现的方法完全避免了这种情况。

 ## 边缘情况

 一种类型的所有门都很早就出现的情况说明了为什么仅跟踪完成时刻有效。 用于输入`1 1 1 0 0 0`，最后一个右门位于位置 3，因此即使左门随后继续，也可以在前缀 3 处退出。 该算法正确地捕获了这一点，因为它记录了每种类型的最终出现而不是模拟前缀。 

在相反的情况下，比如`0 1 0 1 0 1`，两边都是交错的，但每一边的最后一次出现决定了它们何时独立完成。 这些最终位置中的最小值可以正确识别最早的可用前缀，而无需模拟每个步骤。
