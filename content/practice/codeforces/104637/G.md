 ## 解决方案
 ## 问题理解

 青蛙从数轴上的位置 0 开始，以固定的模式反复左右跳跃。 第一次跳跃将其向右移动`a`，第二次跳跃将其向左移动`b`，并且这种交替持续进行`k`总跳跃次数。 每个查询给出不同的值`a`,`b`， 和`k`，并且我们必须在准确执行这些跳跃后确定青蛙的最终位置。 

这里重要的是，运动是完全确定性的，并且是周期性的，以两次跳跃为周期。 一个周期由尺寸的右移组成`a`然后左移尺寸`b`。 青蛙的位置以简单的交替和的形式演化，但跳跃次数可以高达 10^9，因此显式模拟每一步是不可能的。 

这些限制意味着任何每次跳跃模拟都是立即不可行的。 由于最多有 1000 个查询，每个查询最多 10^9 个步骤，O(k) 方法在最坏的情况下最多需要 10^12 次操作，这远远超出了限制。 我们必须将这个过程压缩为每个查询的恒定时间。 

当出现微妙的边缘情况时`k`是奇数还是偶数，因为最后一个部分周期的行为不同。 例如，如果`a = 5`,`b = 2`， 和`k = 3`，序列是`+5, -2, +5`，结果为 8。如果`k = 4`，最后一步是`-2`，改变平衡。 任何假设成对而不处理剩余部分的解决方案都会失败。 

另一个常见的陷阱是假设对称性或超出完整对的抵消。 什么时候`a`等于`b`，完整的周期取消，但不完整的周期仍然产生非零值。 例如，与`a = b = 1`和`k = 999999999`，结果是 1，而不是 0。 

## 方法

 直接模拟保持当前位置并交替添加`a`并减去`b`每次跳跃。 这是简单且正确的，因为它完全遵循定义。 然而，它每次跳转执行一次算术运算，因此对于大的`k`它变得太慢了。 

关键的观察是跳跃是成对重复出现的。 每两次跳跃贡献的净位移为`a - b`。 如果我们将过程分成全对，我们只需要计算有多少个完整的两次跳转循环，以及最后是否有额外的单次跳转。 

如果`k`是偶数，青蛙完全完成`k/2`完整的周期，每个周期都有贡献`a - b`。 如果`k`是奇数，有`(k-1)/2`完整循环加上最后一次右跳`+a`。 

这将问题从线性模拟简化为基于奇偶校验的简单算术。

 | --- | --- | --- | --- |
 | 暴力模拟| O(k) | O(1) | O(1) | 太慢了|
 | 算术对压缩| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 对于每个查询：

 1. 阅读`a`,`b`， 和`k`。 这些定义了固定的交替运动模式以及我们执行的步数。 
2. 使用以下命令计算存在多少个完整的跳跃对`k // 2`。 每对由一个`+a`移动后跟随一个`-b`移动。 
3. 将完整对的数量乘以`(a - b)`以获得所有完整周期的总贡献。 这是有效的，因为每个完整循环返回的净位移等于向前和向后跳跃之间的差值。 
4.如果`k`奇数，多加一个`+a`最后一个不完整的步骤。 这是必要的，因为序列总是以向前跳跃开始，奇数长度序列以向前移动结束。 
5. 输出结果位置。 

### 为什么它有效

 该序列是严格周期性的，周期为 2，每个周期贡献固定的净变化`a - b`。 与完美配对的唯一偏差发生在以下情况：`k`很奇怪，最后一步总是向前跳跃。 由于该过程是线性且可加的，因此分成不相交的对加上可能的余数可以保留精确的总位移。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
out = []

for _ in range(t):
    a, b, k = map(int, input().split())
    
    pairs = k // 2
    res = pairs * (a - b)
    
    if k % 2 == 1:
        res += a
    
    out.append(str(res))

print("\n".join(out))
```该解决方案独立处理每个查询。 关键计算是整数除法`k // 2`，计算完整的 (a, -b) 周期。 乘以`(a - b)`将所有这些循环折叠成一个算术表达式。 

有条件的`if k % 2 == 1`处理剩余的单个向前跳跃时`k`很奇怪。 由于第一次跳跃总是向前，所以剩下的一步永远不会是减法。 

所有算术都是用 Python 整数完成的，它自然支持高达 10^9 * 10^9 的大值，而不会出现溢出问题。

 ## 边缘情况

 当`k = 1`，算法集`pairs = 0`并添加`+a`。 用于输入`a = 7, b = 3, k = 1`，结果变为 7，与定义匹配，因为仅发生第一次向前跳转。 

什么时候`a = b`，每对完整的贡献为零。 为了`a = b = 10, k = 5`，我们得到`pairs = 2`，所以贡献为 0，剩下的奇数步加上`+10`，产生 10。天真的取消假设将错误地返回 0。 

当`k`是均匀的，没有剩余，结果完全取决于`(a - b)`。 为了`a = 5, b = 2, k = 4`，我们得到`2 * 3 = 6`，匹配显式序列`5 - 2 + 5 - 2`。
