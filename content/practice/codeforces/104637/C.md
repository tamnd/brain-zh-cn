 ## 解决方案
 ## 问题理解

 我们正在通过重复闹钟来模拟一个非常特殊的睡眠周期。 波利卡普入睡并最初等待固定的分钟数，然后第一个警报响起。 此后，每次醒来，他都会检查自己是否积累了足够的总睡眠时间。 如果他仍然没有达到所需的量，他会经历另一个周期，闹钟被重置，然后他花一些时间再次入睡，并且只有下一个闹钟间隔的一部分有助于实际睡眠。 

关键思想是，时间被分成交替的阶段：睡眠阶段有助于积累睡眠，而“入睡”阶段会消耗时间，但不会增加睡眠。 警报以固定的时间间隔重复中断此过程，只是第一次的时间间隔不同。 

我们被要求计算累积睡眠首次达到至少给定阈值的确切时间。 如果进程的结构使得不可能积累足够的睡眠，则进程将永远运行，我们必须输出 -1。 

约束最多允许 1000 个测试用例，每个参数可以大到 10^9。 这立即排除了模拟逐分钟或逐事件进展的可能性。 在值很大且过程需要数十亿个周期的最坏情况下，即使每个警报进行线性模拟也会太慢。 

当睡眠积累相对于“入睡开销”增长太慢时，就会出现微妙的失败情况 d。 在这种情况下，即使闹钟无限期地响起，每个周期获得的有效睡眠也可能为零或不足，从而导致无限循环。 

例如，如果 c ≤ d，则每当闹钟在入睡期间响起时，Polycarp 都会重置它，并且除了第一次中断之外永​​远不会达到任何有意义的睡眠阶段。 这可能会导致累积睡眠没有任何进展。 

另一个边缘情况是 b 已经超过或等于 a，这意味着 Polycarp 在第一个警报后立即获得了足够的睡眠。 在这种情况下，不会发生进一步的循环，答案就是 b。 

最后，存在一种情况，睡眠以重复模式累积，我们必须避免显式模拟所有周期，而是推理每个周期的净增益。 

## 方法

 暴力方法将模拟每个警报周期。 我们跟踪当前时间、累计睡眠时间，并重复处理：等到闹钟响起，加上该时间间隔内获得的睡眠时间，然后减去入睡时间并继续。 每个周期的时间复杂度为 O(1)，但周期数可以大到 a/b，甚至与 a/c 成正比，最坏情况下达到 10^9。 这使得直接模拟变得不可行。 

关键的观察结果是，在第一次警报之后，该过程变得周期性。 每个周期都会贡献固定量的睡眠增益并消耗固定量的实时时间。 我们不需要模拟每次醒来，而只需要了解每个完整周期获得了多少睡眠。 

在时间 b 第一次醒来后，Polycarp 检查到目前为止的睡眠是否充足。 如果没有，他会进入一个重复模式：每个周期包括花费 d 分钟入睡，然后等待重置后 c 分钟后的下一个闹钟，在此期间，根据重叠结构，只有最后一部分有助于睡眠。 重要的减少是，每个完整周期将总时间增加 c，并根据重叠贡献 c 或 max(0, c - d) 有效睡眠。 

这将问题转化为线性累积问题，并且可能在第一步之后提前终止。 

该过程要么立即完成，要么在几个相同的增量后完成，要么永远不会进行，因为每个周期的净睡眠增益为零。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(答案/分钟(c, b)) | O(1) | O(1) | 太慢了|
 | 基于循环的算术 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 算法演练

 1.首先检查Polycarp在第一个闹钟发出后是否已经满足睡眠要求。 如果在第一段 b 中获得的睡眠至少为 a，则答案就是 b，因为不需要进一步等待。 
2. 如果不满意，计算第一次醒来后有效获得的额外睡眠时间。 第一个循环给出了已知的初始贡献，之后每次重复的行为都相同。 
3. 观察醒来后，波利卡普花了 d 分钟入睡。 在此期间，如果下一个警报间隔 c 不够大，无法“与入睡重叠”，则在未来的周期中不会获得额外的睡眠。 
4. 如果 c ≤ d，则在闹钟发挥任何作用之前，每个周期都完全被入睡所消耗。 这意味着在第一次唤醒后不会再累积睡眠，因此如果 b < a，则答案是不可能的。 
5. 否则，当 c > d 时，每个周期在第一次醒来后精确地贡献 (c - d) 分钟的有效睡眠增益。 我们可以将其视为线性进展。 
6. 计算需要多少个这样的周期才能达到至少 a 的总睡眠。 这是第一次警报后剩余所需睡眠的简单上限划分。 
7. 通过添加初始 b，然后为每个循环添加 c，将循环数转换为总时间。 

### 为什么它有效

 在第一个警报之后，该过程变成确定性线性循环。 每个周期具有相同的结构，并且对时间和累积睡眠贡献恒定增量，或者如果 c ≤ d，则根本不贡献睡眠。 这意味着每次唤醒后的系统状态完全由迄今为止的总睡眠时间来表征，并且该值呈线性变化。 由于后面的循环中没有分支或随机性，一旦我们对增量是正还是零进行分类，结果就固定了：要么有限算术级数达到阈值，要么序列永远不会进展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        a, b, c, d = map(int, input().split())

        # first alarm
        if b >= a:
            print(b)
            continue

        # if no progress possible after first cycle
        if c <= d:
            print(-1)
            continue

        # remaining sleep needed
        need = a - b

        gain = c - d

        # number of full useful cycles needed
        cycles = (need + gain - 1) // gain

        # total time: initial b + cycles * c
        ans = b + cycles * c
        print(ans)

if __name__ == "__main__":
    solve()
```该代码直接将模拟问题的简化编码为算术级数。 提前退出处理第一个警报已经满足要求的情况。 条件`c <= d`捕获系统无法累积进一步睡眠的退化情况。 最终公式计算有多少个相同的大小增量`gain`需要弥补剩余的赤字并将其转化为时间。 

一个常见的陷阱是忘记第一部分的贡献与其他部分不同。 该代码通过处理来隔离它`b`在进入统一循环逻辑之前分别进行。

 ## 边缘情况

 当`b >= a`，算法立即返回`b`无需输入任何循环逻辑。 这对应于第一个警报已经提供了足够的休息并且没有关于后续结构问题的假设的情况。 

什么时候`c <= d`，系统在第一次唤醒后无法累积额外的睡眠。 在这种情况下，算法以 -1 终止。 推理与真实过程相符，因为每次尝试达到下一个有用的睡眠间隔都完全被入睡时间消耗掉。 

什么时候`c > d`但所需的睡眠非常小，在除顶后计算的周期数可能为零。 公式`(need + gain - 1) // gain`通过确保仅在需要时至少有一个周期来正确处理这一问题，避免提前终止时出现差一错误。
