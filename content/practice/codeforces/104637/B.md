 ## 解决方案
 ## 问题理解

 我们正在努力生产目标数量的火炬。 每个火把只消耗一根棍子和一根煤，所以如果我们想要$k$火把，我们最终需要$k$棍棒和$k$煤炭。 

我们一开始没有任何有用的库存，除了我们可以通过两次贸易操作来操纵棍子。 第一个操作让我们花一根棍子来接收$x$棍子，这有效地增加了我们的棍子数量。 第二次操作让我们花$y$坚持获得一颗煤炭。 我们可以以任意顺序重复应用这些操作，并且每次应用都算作一笔交易。 目标是尽量减少所需的交易总数，以便我们至少可以组装$k$火把。 

关键的困难在于棍子既是资源又是货币。 我们可以增加它们，但也可以消耗它们来生产煤炭。 运营顺序很重要，因为早期交易会改变未来煤炭的价格。 

约束条件很大，最多可达$2 \cdot 10^4$测试用例和值高达$10^9$。 任何一项一项模拟交易的解决方案都是立即不可行的，因为即使$O(k)$每个测试用例将远远超出限制。 该解决方案必须是对数的，或者在最坏的情况下是与有意义的状态变化的数量成线性的，而不是$k$。 

当出现幼稚但危险的边缘情况时，$y = 1$。 在这种情况下，煤炭的成本恰好是一根，因此，一根棍同时是生产和消费单位，而粗心的贪婪推理“总是先乘数”很容易多算或少算交易。 

另一个微妙的情况是当$x = 2$和$y$很大。 在这里，繁殖木棍虽然缓慢但必要，而且过早生产煤炭会永久限制木棍的生长，如果我们不明确地对权衡进行建模，就会导致错误的贪婪选择。 

## 方法

 暴力解释将问题视为状态搜索：每个状态都是一对$(s, c)$代表当前的棒材和煤炭，过渡适用于这两种交易中的任何一种。 我们从最小的非零配置开始，并尝试达到可以生产的状态$k$煤炭，并且至少还有$k$可用于制作的棍子。 

这自然表明 BFS 或 Dijkstra 优于状态，其中每条边都会花费一笔交易。 虽然原则上是正确的，但状态空间立即爆炸。 棍子可以倍增增长到$10^9$，而煤炭生产取决于它们的支出。 即使我们积极地限制状态，分支结构实际上也是无界的，因为在任何煤炭生产之前，棒乘法可以任意重复多次。 

关键的观察结果是该过程实际上并不需要任意交错。 这两种操作具有不同的作用：一种是木棒的几何生长机制，另一种是将木棒转化为煤炭的线性提取机制。 一旦我们确定了执行每个操作的次数，最终的资源平衡就变得确定了。 

我们可以反向推理：得到$k$火把，我们需要$k$煤炭采购。 每份煤炭成本$y$棒，所以煤炭总成本是$k \cdot y$棍子。 生成木棍的唯一方法是重复应用乘法交易。 因此，问题简化为：从 1 根棍子开始，应用“乘以$x$”和“转换$y$以某种顺序坚持煤炭业务，最大限度地减少总运营量，同时确保我们有能力承担$k \cdot y$煤炭交易的价值。 

一个关键的结构简化是，一旦我们决定了进行多少煤炭交易，就可以贪婪地应用棍子乘数以最大化效率。 最佳策略是选择一个从增长转向消费的点，因为超过该点的交错永远不会提高每笔交易的棒数比率。 

这导致了对达到阈值需要多少棒乘法步骤的封闭式评估，并结合煤炭采购的直接计数。

 ## 算法演练

 我们将这个过程分为两个阶段：建造煤炭和支出煤炭。 

1. 计算所需采购的煤炭总数，即$k$，因为每个火炬需要一颗煤。 这确定了独立于订购的煤炭总需求。 
2、每份煤炭成本$y$煤棒，因此对煤炭的总棒材需求为$k \cdot y$。 我们将其视为在开始支付煤炭费用之前必须提供的最低预算。 原因是，一旦我们开始消耗煤棒，过早减少煤棒池只会使未来的增殖效率降低。 
3. 只考虑棒乘运算。 每次使用可替换 1 根棒$x$坚持，所以净收益是$x-1$。 然而，这仅在第一根棒存在后才重要，因此每个操作都会在有用的情况下有效地倍增增加棒计数。 
4. 确定需要多少次乘法交易才能达到至少$k \cdot y$从 1 开始，每一步都会改变当前的棒数$s$进入$s \cdot x$，所以之后$t$我们有的步骤$x^t$。 我们找到最小的$t$这样$x^t \ge k \cdot y$。 
5.一旦积累了足够的木棒，我们就进行煤炭交易。 每笔煤炭贸易成本$y$粘住并产出 1 块煤，所以我们的表现准确$k$此类交易。 
6. 经营总数为乘数贸易加上煤炭贸易之和。 

微妙的一点是，我们永远不会以混合的方式交织这些阶段。 任何早期的煤炭购买都会降低指数增长的有效基础，而在过度生长后的任何延迟煤炭购买都不会减少所需的倍增步骤的数量。 这种单调分离保证了最优性。 

### 为什么它有效

 该算法依赖于单调权衡：棒乘法严格增加了未来煤炭采购的能力，而煤炭采购严格减少了可用于进一步乘法的资源。 由于这两种操作都具有固定的线性成本和确定性效应，因此可以重新安排将它们交错的任何调度，以便所有乘法首先发生，而无需增加操作数量。 这表明最佳解决方案始终由乘法交易的前缀和后跟煤炭交易的后缀组成，从而将问题简化为简单的阈值计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        x, y, k = map(int, input().split())

        need = k * y

        # compute minimum t such that x^t >= need
        if need == 1:
            mult = 0
        else:
            cur = 1
            mult = 0
            while cur < need:
                cur *= x
                mult += 1

        # coal trades
        coal = k

        print(mult + coal)

if __name__ == "__main__":
    solve()
```该解决方案明确地将指数增长阶段与线性消耗阶段分开。 循环计算最小指数$t$这样重复乘以$x$达到所需的粘性阈值。 这是安全的，因为$x \ge 2$，因此增长是严格递增的并且循环很快终止。 

煤炭阶段很简单：一旦存在足够多的木棒，每个火炬就需要一次煤炭交易，因此我们添加$k$。 无需模拟个人消费，因为每个单位成本是固定且独立的。 

一个常见的实现陷阱是尝试在单个模拟循环中混合乘法和消耗。 这种方法往往会出现错误计数，因为它不能保留棒积累的指数增长结构。

 ## 边缘情况

 当$y = 1$，每根煤棒实际上都是免费的，因此限制因素只是煤棒的倍增速度。 该算法通过仍然正常计算乘法阈值然后添加来处理这个问题$k$，这正确地反映了即使棒成本极低，每种煤炭仍然需要一次交易。 

什么时候$x$非常大，一次乘法可能已经超出了所需的预算。 循环在第一步后立即终止，算法正确返回$1 + k$，匹配一次增长操作后直接消费的最优策略。 

什么时候$k$处于最大值时，乘法阶段仅以对数方式占主导地位。 该算法仍最多执行约 30 次迭代，即使在最坏情况的输入大小下也能保持稳定。
