 ## 解决方案

 ## 问题理解

 我们得到一系列音符，每个音符位于钢琴键盘上的某个水平位置。 目标是将每个音符分配给五个手指之一，编号从 1 到 5，产生另一个相同长度的序列。 

分配受到连续音符在键盘上移动方式的限制。 如果下一个音符在右边，则手指数量必须严格增加。 如果向左移动，则手指数量必须严格减少。 如果下一个音符位于同一位置，我们就被迫交换手指，因此在这种情况下禁止相等。 

这将旋律变成了手指序列中连续位置之间的一系列不等式约束。 我们并不是试图优化任何东西，只是为了确定是否存在有效的分配并输出其中的任何一个。 

高达 100000 的约束 n 强制采用线性或近线性解。 任何试图回溯所有可能的手指分配的方法都会爆炸，因为每个位置最多有 5 个选择，而简单的探索会导致 5^n 种状态。 即使是依赖于先前选择的修剪也是危险的，除非它是非常局部的。 

当做出贪婪选择而不考虑相同值可能出现多次时，就会出现微妙的失败情况。 例如，在像 3 3 3 3 这样的平坦部分中，我们必须交替手指，但如果我们在早期决策中限制太多，手指的可用性有限可能会导致长时间的恒定运行无法满足。 

真正的困难在于约束仅取决于相邻元素之间的比较。 该局部性建议对“当前位置和当前手指”状态进行动态编程或图形遍历。 

## 方法

 一个蛮力的想法是将其视为状态图。 每个状态由索引 i 和手指 f 定义。 从每个状态，我们尝试从 1 到 5 满足关于 a[i] 和 a[i+1] 的不等式约束的所有可能的下一个指 f2。 这将构建一个分层图，其中每层有 5 个节点。 

这是正确的，因为它明确地探索了所有有效的指法转换。 然而，它每个索引处理 5 个状态，每个状态最多 5 个转换，导致大约 25n 次操作，这很好。 真正的问题是，幼稚的实现经常尝试完整路径枚举或回溯，这会呈指数级增长。 

关键的观察是我们只需要每个位置跨越 5 个状态的可达性，并且我们还希望重建任何有效路径。 这是每层都有一个小型固定状态空间的经典 DP。 由于手指的数量是恒定的，我们可以安全地存储转换并向前传播可行性。 

该结构本质上是一条穿过大小为 n × 5 的分层图的路径。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力回溯 | O(5^n) | O(5^n) | O(n) | 太慢了 |
 | DP 超过（食指、手指）| O(25n) | O(25n) | O(5n) | O(5n) | 已接受 |

 ## 算法演练

 我们将每个位置 i 和每个手指 f 视为一个状态，意思是“可以演奏到以手指 f 结尾的 i”。

1. 对从 1 到 5 的所有 f 初始化 dp[1][f] = true，因为第一个音符没有先前的约束并且可以从任何手指开始。 我们还存储一个前趋指针以便稍后重建解决方案。 
2. 从 1 到 n - 1 迭代 i，并尝试从位置 i 处的每个可到达的手指 f 转换到位置 i + 1 处的手指 f2。我们只允许尊重 a[i] 和 a[i+1] 之间关系的转换。 如果 a[i] < a[i+1]，我们要求 f < f2。 如果 a[i] > a[i+1]，我们要求 f > f2。 如果 a[i] == a[i+1]，我们要求 f != f2。 
3. 每当转换有效且 dp[i][f] 为 true 时，我们将 dp[i+1][f2] 设置为 true 并存储parent[i+1][f2] = f 以记住我们是如何到达那里的。 我们只需要存储一个父级，因为任何有效路径都是可以接受的。 
4. 填写表格后，我们检查是否有任何 dp[n][f] 为 true。 如果没有一个为真，则不存在有效的指法，我们输出 -1。 
5. 否则，我们选择任何有效的结束状态，并通过从 (n, f) 向后跟随父指针到 (1, *) 来重建序列。 

关键思想是，除了最后选择的手指之外，每个决策都是本地的并且独立于早期的选择，这正是使其成为有效的 DP 状态定义的原因。 

### 为什么它有效

 不变量是 dp[i][f] 为真当且仅当存在至少一个有效的指法用于直到 i 且以指 f 结尾的前缀。 过渡步骤保留了有效性，因为它只允许满足连续音符之间所需的不等式约束的移动。 由于每个有效的完整解决方案都必须对应于该分层状态图中的一条路径，并且我们标记的每个状态都可以通过这样的路径到达，因此我们既不会错过有效的解决方案，也不会引入无效的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    dp = [[False] * 6 for _ in range(n)]
    parent = [[-1] * 6 for _ in range(n)]

    for f in range(1, 6):
        dp[0][f] = True

    for i in range(n - 1):
        for f in range(1, 6):
            if not dp[i][f]:
                continue
            for nf in range(1, 6):
                if a[i] < a[i + 1] and f >= nf:
                    continue
                if a[i] > a[i + 1] and f <= nf:
                    continue
                if a[i] == a[i + 1] and f == nf:
                    continue
                if not dp[i + 1][nf]:
                    dp[i + 1][nf] = True
                    parent[i + 1][nf] = f

    end_f = -1
    for f in range(1, 6):
        if dp[n - 1][f]:
            end_f = f
            break

    if end_f == -1:
        print(-1)
        return

    res = [0] * n
    cur_f = end_f
    for i in range(n - 1, -1, -1):
        res[i] = cur_f
        if i > 0:
            cur_f = parent[i][cur_f]

    print(*res)

if __name__ == "__main__":
    solve()
```DP 表按位置和手指进行索引。 为了清晰起见，我们明确保留基于 1 的手指索引，这可以避免在比较转换时出现相差一的错误。 父表存储用于重建的前一个手指，每个状态仅填充一次以确保单个一致的重建路径。 

转换逻辑直接对来自相邻音符的三个约束进行编码，确保没有无效的指法被标记为可达。

 ## 边缘情况

 单个音符输入很简单，因为每个手指都是有效的。 DP 将所有五个起始状态初始化为有效，因此重建立即返回单个手指。 

像 2 2 2 2 这样的恒定序列迫使手指严格交替。 该算法处理这个问题是因为过渡不允许相同的手指，但仍然允许在其余四个选项之间移动，这足以保持任何长度的可行性。 

像 1 2 3 4 5 这样的严格增加序列需要严格增加指分配。 DP 自然地将状态集中到更高的手指并消除无效的向后移动，留下至少一条递增路径，例如 1 2 3 4 5。 

严格递减序列的行为是对称的，并且相同的状态传播确保在可能的情况下存在有效的严格递减指路径。
