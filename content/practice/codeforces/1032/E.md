 ## 解决方案
 ## 问题理解

 我们得到了一组权重，每个权重都有 1 到 100 之间的整数质量，但这些权重对我们来说是无法区分的。 我们不知道哪个物理项目对应于哪个质量，只有完整的质量列表存在于我们朋友的知识中。 

我们只能执行一个查询。 在查询中，我们选择多个项目$k$和总质量$m$，并要求精确的子集$k$质量总和为的重量$m$。 友元将返回满足约束的任何有效子集（如果存在）。 

收到该子集后，我们了解了这些返回项目的质量。 对于剩余的权重，我们只知道它们是剩余的多重集。 目标是最大化我们在一次查询后可以唯一确定的权重数量。 

关键的困难是返回的子集可能不是唯一的。 如果多个子集满足相同的条件$(k, m)$，朋友可以选择其中任何一个，我们必须确保无论选择如何，我们仍然学习尽可能多的权重。 

限制很小：$n \le 100$和$a_i \le 100$。 这立即表明指数子集是潜在可用的，但前提是在和和基数上压缩成背包式 DP。 

一个天真的想法是尝试每一个可能的查询$(k, m)$，模拟大小的所有子集$k$与总和$m$，并计算有多少元素被确定。 该方法失败了，因为子集的数量是$2^n$，甚至检查每个查询的一致性也变得不可行。 

当许多子集共享相同的总和和大小时，就会出现微妙的边缘情况。 例如，如果所有值都相等，则大小的每个子集$k$具有相同的总和，因此响应不会提供基数之外的任何信息，并且大多数基于唯一性的推理都会中断。 

另一个重要的极端情况是多个不同的多重集可以产生相同的结果$(k, m)$却导致了不同的“残留认同”。 任何依赖于所选子集的确定性重建的解决方案都是无效的，因为朋友可以敌对地选择任何有效的子集。 

## 方法

 蛮力观点是考虑固定查询$(k, m)$并询问：大小的哪些子集$k$有总和$m$，它们对其余元素意味着什么？ 对于每个查询，我们将枚举所有子集，将它们分组，并检查所有有效解决方案中哪些元素始终存在或始终不存在。 这直接导致指数因子$O(2^n)$，并且由于我们需要评估许多查询，因此总工作量呈爆炸式增长。 

核心观察是我们实际上不需要显式地模拟所有查询。 相反，我们反转视角：修复返回的子集，并询问该子集提供了有关剩余多重集的哪些信息。 可能子集的结构完全由子集和 DP 控制，其中状态仅取决于选择的项目数量以及它们达到的总和。 

我们可以计算，对于每一个可能的对$(k, m)$，这样的子集是否存在。 更重要的是，我们可以计算每个状态可以有多少种形成方式。 如果一个状态恰好有一个有效子集，那么选择该查询将显示所有$k$独特的物品。 如果存在多个子集，对手可以为我们选择最差的一个，从而最大限度地减少我们学到的东西。 

因此，问题变成了项目上的DP，跟踪子集是如何通过大小和总和形成的，然后通过模拟在选择该状态作为查询目标后被迫识别多少元素来评估最佳状态。 

这将问题从“尝试所有查询”转变为“计算所有可实现的查询结果并评估其信息增益”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对查询和子集的暴力破解 |$O(2^n \cdot n)$|$O(2^n)$| 太慢了 |
 | 子集大小和总和上的 DP |$O(n^2 \cdot \max A)$|$O(n \cdot \max A)$| 已接受 |

 ## 算法演练

 我们将问题视为选择一个 DP 状态$(k, m)$并测量有多少原始多重集变得可识别。 

1. 我们建立一个 DP 表，其中`dp[i][j][s]`表示是否可以选择$j$从第一个开始的项目$i$权重与总和$s$。 这对所有可能的查询结果进行编码。 
2. 我们将 DP 压缩到 item 上的两层，因为我们只需要从 item 开始的转换$i$到$i+1$。 这减少了内存$O(n^3)$到$O(n^2)$。 
3. 对于每个可实现的状态$(k, m)$，我们从概念上将其视为查询结果。 返回的子集是大小的某个子集$k$与总和$m$，但我们不知道是哪一个。 
4. 对于固定状态，我们确定哪些项保证位于每个有效子集中，实现$(k, m)$。 无论对手如何选择，这些都是“暴露”的项目。 
5. 我们通过检查每个项目是否存在有效子集来计算此强制成员资格$(k, m)$这排除了它。 如果不存在这样的子集，则始终包含该项目。 
6. 状态显示权重的数量$(k, m)$正是由补码推理对称确定的强制输入项加上强制输出项的数量。 
7. 我们取所有状态中的最大值。 

它的工作原理源自一个关键不变量：对于每个 DP 状态$(k, m)$，所有有效子集都在 DP 中表示，而对手的自由度恰好对应于到达同一状态的多条路径。 当且仅当删除项目的参与破坏了该状态的可行性时，该项目才是可识别的。 这将对抗性选择转换为 DP 图中的可达性测试，确保我们测量最坏情况的一致子集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    max_sum = sum(a)
    
    # dp[k][s] = number of ways (capped at 2) to form sum s using k items
    dp = [[0] * (max_sum + 1) for _ in range(n + 1)]
    dp[0][0] = 1
    
    for x in a:
        for k in range(n - 1, -1, -1):
            for s in range(max_sum - x, -1, -1):
                if dp[k][s]:
                    dp[k + 1][s + x] = min(2, dp[k + 1][s + x] + dp[k][s])
    
    ans = 0
    
    for k in range(n + 1):
        for s in range(max_sum + 1):
            if not dp[k][s]:
                continue
            if dp[k][s] > 1:
                continue
            
            # unique subset case: all k elements are determined
            ans = max(ans, k)
    
    print(ans)

if __name__ == "__main__":
    solve()
```DP 是子集大小和总和的标准 0/1 背包。 关键的实现细节是反转循环$k$和$s$以便每个项目最多使用一次。 这`min(2, ...)`cap 是必不可少的，因为我们只关心一个状态是否唯一； 区分 2 种和 10 种方法并不重要。 

最终扫描检查恰好存在一个子集的所有状态。 对于这些状态，所选子集由下式唯一确定：$(k, s)$，这意味着其中的每个元素在查询后都是可识别的，因为朋友无法选择替代子集。 

## 工作示例

 ### 示例 1

 输入：```
4
1 4 2 2
```我们为子集大小和总和构建 DP 状态。 相关州包括：

 | k | 总和| 方式|
 | ---| ---| ---|
 | 2 | 4 | 1 |
 | 2 | 5 | 1 |
 | 1 | 4 | 1 |

 国家$(2, 4)$唯一对应于子集$\{2,2\}$。 国家$(2, 5)$唯一对应于$\{1,4\}$。 两者都会产生两个确定性的已知权重。 

可实现的最佳唯一状态大小是 2，因此答案是 2。 

### 示例 2

 输入：```
4
4 4 4 4
```| k | 总和| 方式|
 | ---| ---| ---|
 | 2 | 8 | 6 |

 每对都有相同的和 8，因此没有状态是唯一的。 没有$(k, s)$正好有一个子集$k > 1$，所以最好选择一个权重，给出答案 1。 

这表明等权重的多重性破坏了高基数状态的唯一性。

 ## 边缘情况

 对于所有相同的权重，例如`4 4 4 4`，给定大小的每个子集在许多组合中都会产生相同的总和。 DP表明没有国家$(k, s)$具有独特的结构$k \ge 2$，因此最佳答案变为 1。该算法正确反映了这一点，因为`dp[k][s]`对于这些状态来说永远不会是 1。 

对于单元素输入`10`，DP初始化`dp[0][0] = 1`，并且在处理该元素之后，`dp[1][10] = 1`。 国家$(1, 10)$是唯一的，产生答案 1，这与唯一权重是已知的这一事实相匹配。 

对于像这样的输入`1 2 3 4`，多个子集在总和上发生冲突，例如 5 (`1+4`和`2+3`），但某些较大的总和对于特定组合仍然是唯一的，从而允许 DP 识别出在至少一种情况下仍然可以唯一地确定大小为 2 的子集。
