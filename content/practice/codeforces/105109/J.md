 ## 解决方案
 ## 问题理解

 Bob 有一组从 1 到 n 编号的记录。 在接下来的 n 天里，他每天都会听这些唱片的一部分。 如果鲍勃在任何早些时候从未听过某张唱片，则该唱片在某一天被视为新唱片。 

每一天，我们都会关心当天播放的唱片中有多少是第一次出现在鲍勃的整个收听历史中。 任务是确定所有日期中此类“新记录数”的最大值。 

输入是 n 个每日收听日志的序列。 每个日志都会提供当天播放的记录 ID 列表。 记录可以在一天内或几天内重复，但只有每条记录的首次出现才会计入“新”计数，并且仅在其首次出现的当天。 

输出是一个数字：任意一天中出现的以前未见过的记录的最大数量。 

限制很小：n 最多为 1000，最坏情况下录制的播放总数最多为 10^6。 这立即表明 O(总条目数) 或 O(n^2) 模拟就足够了，因为在 2 秒限制下，Python 中大约 10^6 次操作是安全的。 

一个微妙的问题是，一天内的重复次数不应增加计数。 如果一条记录在同一天出现多次，如果它是第一次整体出现，则它仍然只作为“新”贡献一次。 

第二个微妙之处是，我们必须仅在处理当天的记录后将记录标记为全局可见，否则同一天内的重复项可能会过早被错误地抑制。 

## 方法

 一种直接的方法是每天模拟该过程，同时维护鲍勃已经听到的所有记录的全局集合。 每天，我们扫描所有 m 个条目并计算有多少个条目不在全局集中。 每当我们遇到尚未看到的记录时，我们都会增加当天的计数并将其标记为已看到。 

这是有效的，因为“新”的定义仅取决于记录是否出现在前一天，而不取决于它在当天的位置。 唯一需要注意的是，如果同一记录在当天的列表中重复，请避免在一天内对同一记录进行多次计数。 这可以通过每天临时设置来处理，也可以通过确保我们仅在全局范围内标记一次并忽略同一天的后续重复来处理。 

暴力行为在结构上已经是最佳的：每个记录在第一次作为新记录遇到时都会被处理一次，因此全局集中的有效插入总数以 n 为界。 开销是扫描所有条目，最多 10^6 次操作，完全在限制范围内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 使用全局集扫描 | O(总米) | O(n) | 已接受 |
 | 每天嵌套检查 | 最坏 O(n * m) | O(n) | 已接受但不必要的开销 |

 ## 算法演练

 1. 初始化一个空集合seen，用于存储前几天已经出现过的所有记录。 该集合代表了鲍勃收听的全球历史。 
2. 将变量答案初始化为 0。这将跟踪任意一天观察到的新记录的最大数量。 
3. 阅读每天播放的唱片列表。 
4. 对于这一天，将计数器 day_new 初始化为 0。 
5. 迭代当天列表中的每条记录。 如果没有看到该记录，则意味着这是它的第一次出现。 将 day_new 加 1 并将记录添加到可见。 
6. 如果该记录已经被看到，则完全忽略它，因为它对新的发现没有贡献。 
7.处理完当天的所有记录后，将answer更新为max(answer, day_new)。 
8. 处理完所有日期后，输出答案。

关键的实施决策是在全局第一次遇到记录时立即看到更新。 这可以确保以后正确地将其视为旧的，并且还可以防止以后扫描中的重复计算。 

### 为什么它有效

 在处理过程中的任何时刻，看到的集合都与早期出现的所有不同记录的集合完全匹配。 当我们处理新的一天时，每条记录都会根据是否可见进行正确分类。 每条记录都只占一天的计数，特别是它出现的第一天。 由于我们仅在遇到以前未见过的记录时才增加 day_new，因此不会对任何记录进行两次计数，并且不可能错过贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    seen = set()
    ans = 0

    for _ in range(n):
        arr = list(map(int, input().split()))
        m = arr[0]
        records = arr[1:]

        day_new = 0
        for r in records:
            if r not in seen:
                seen.add(r)
                day_new += 1

        ans = max(ans, day_new)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案使用单个全局集来跟踪记录以前是否已被看到。 每天的输入被解析成一个列表，第一个元素是计数 m。 其余的是记录 ID。 

重要的实现细节是，当我们检测到新记录时，我们立即更新可见集。 这确保了几天内的正确性。 我们不需要每天的集合，因为一天内的重复项并不重要：一旦记录被标记为“已见”，同一天的后续出现的记录将自然被忽略。

 ## 边缘情况

 一天内有重复记录的案例证实，日内重复记录不会夸大计数。 例如，输入`1: [1, 1, 1]`应该只计算一次。 该算法会处理此问题，因为一旦将 1 添加到 saw 中，后续出现的情况就会被忽略。 

先前多次重复后记录出现较晚的情况证实了全局跟踪。 例如，如果 5 仅在最后一天出现，则它仍应仅在当天计数一次。 该集合确保它不会被提前计数。 

每天仅重复以前见过的元素的情况证实了第一天之后答案可能为零。 一旦看到包含所有记录，所有未来的日子贡献为零，并且从早期计算中正确保留最大值。
