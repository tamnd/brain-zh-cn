 ## 解决方案
 ## 问题理解

 我们得到一个数字数组和一个转换，该转换用其前缀产品版本重复替换该数组。 转换的一个应用程序采用一个数组，并将每个位置转换为该索引之前所有内容的乘积。 因此，第一个元素保持不变，第二个元素成为前两个元素的乘积，第三个元素成为前三个元素的乘积，依此类推，所有元素均以固定素数为模。 

我们没有被要求直接模拟这个过程。 相反，我们收到了几个查询。 在应用此前缀乘积转换一定次数后，每个查询都会请求数组中的特定位置。 

困难来自于这样的事实：转换被重复应用，并且每个应用程序都复合了所有先前位置的依赖关系。 对每个查询的直接模拟将重复重新计算前缀乘积，导致成本与操作数乘以数组大小成正比，即使对于中等输入，数组大小也会很快变得太大。 

约束表明，数组大小和每个查询的转换次数都可以达到一千个，并且总共有一千个查询。 在最坏的情况下，为每个查询和每个转换步骤重新计算数组的简单方法将涉及 10^9 次操作，这远远超出了两秒内可行的范围。 

反复的转型增长会产生一个微妙的问题。 即使正确地实现了一次前缀积，每次查询迭代 k 次也会太慢。 另一个常见的陷阱是假设多次转换后元素之间是独立的，但每个位置越来越依赖于必须通过分析捕获的结构化方式的各种早期元素。 

## 方法

 最直接的方法是从字面上模拟该过程。 对于单个转换，我们在线性时间内计算前缀乘积。 每个查询重复此 k 次，每个查询的复杂度为 O(nk)。 虽然正确，但当 n 和 k 都很大时，这会爆炸，特别是因为查询是独立的并且可能在相同的数组结构上重复工作。 

关键的观察结果是每次转换都不会引入新的交互作用； 它只是重新组织每个原始元素对每个位置的贡献次数。 一步之后，每个位置都是原始数组的前缀的乘积。 经过两个步骤后，每个位置都成为这些前缀乘积的前缀的乘积，这意味着每个原始元素以结构化的多重性出现多次。 

如果我们跟踪指数而不是值，那么指数空间中的乘法就会变成加法。 前缀运算的每次应用对应于获取指数贡献的前缀和。 重复类似前缀和的运算是一种经典结构：在 k 次重复之后，元素的贡献根据二项式系数扩散。 具体来说，k 次操作后元素 a[j] 对位置 i 的贡献等于简单格解释中的单调路径数，其计算结果为组合系数。 

这将问题从重复模拟减少到计算，对于每个查询，计算所有前缀元素的乘积，其中每个 a[j] 被提升为仅取决于 k、i 和 j 的可预先计算的二项式指数。 

预先计算阶乘和模逆可以快速评估这些二项式系数，然后可以在数组前缀的线性时间内回答每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(q·k·n) | O(q·k·n) | O(n) | 太慢了 |
 | 组合指数公式| O(q·n) | O(q·n) | O(n) | 已接受 |

 ## 算法演练

我们对素数进行模运算，因此指数算术和二项式系数是明确定义的。 

### 1. 预计算阶乘和逆阶乘

 我们计算 2000 以内的阶乘及其模逆。 这使我们能够在恒定时间内计算任何二项式系数。 

这是必要的，因为每个查询都需要许多二项式计算，并且重新计算它们将主导运行时。 

### 2. 独立处理每个查询

 对于在 k 次操作后要求位置 x 的查询，我们通过累积从 1 到 x 的所有索引 j 的贡献来计算该位置的最终值。 

### 3.计算每个元素的贡献

 对于每个 j ≤ x，我们计算指数：

 C(k + x − j − 1, k − 1)

 该值表示原始元素 a[j] 经过 k 次重复前缀变换后对位置 x 乘法贡献的次数。 

### 4. 累加结果

 我们将答案初始化为 1 并将其乘以：

 a[j] 求算出的指数

 对于从 1 到 x 的所有 j。 

所有运算均以 1e9+7 为模进行。 

### 为什么它有效

 每个前缀运算将数组转换为累积乘积，这对应于将指数转换为前缀和。 重复此过程 k 次对应于在指数空间中应用前缀和运算符 k 次。 位置 j 处的 delta 的 k 倍前缀和传播到位置 i，其重数等于在距离 i − j 上分布 k 个相同前缀操作的方式数量，这恰好是一个二项式系数。 由于乘法在指数空间中变成加法，因此最终值是通过对这些贡献进行乘法求和来确定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

MAXN = 2005

fact = [1] * MAXN
invfact = [1] * MAXN

for i in range(1, MAXN):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN - 1] = pow(fact[MAXN - 1], MOD - 2, MOD)

for i in range(MAXN - 2, -1, -1):
    invfact[i] = invfact[i + 1] * (i + 1) % MOD

def nCr(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

t = int(input())
for _ in range(t):
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    for _ in range(q):
        k, x = map(int, input().split())

        res = 1
        for j in range(x):
            exp = nCr(k + x - j - 1, k - 1)
            res = res * pow(a[j], exp, MOD) % MOD

        print(res)
```由于限制是全局的，阶乘预处理在所有测试用例之间共享。 二项式函数保持最小以避免开销，因为它被调用很多次。 

在每个查询中，我们迭代直到 x 的前缀。 指数公式直接编码了k次重复前缀乘积运算的效果，因此没有构造中间数组。 

一个常见的实现错误是二项式项中的差一索引。 正确的移位是 k + x − j − 1 选择 k − 1，它来自于将第一个前缀操作对齐为从每个索引开始的单层累积。 

## 工作示例

 考虑一个小数组 a = [2, 3, 5]。 

对于 k = 1 且 x = 3，变换是单个前缀乘积，因此结果应为 [2, 6, 30]。 公式给出：

 | j | 指数 C(1 + 3 − j − 1, 0) | 贡献 |
 | ---| ---| ---|
 | 1 | 1 | 2 |
 | 2 | 1 | 3 |
 | 3 | 1 | 5 |

 产品为 2 · 3 · 5 = 30，与前缀产品数组的第三个元素匹配。 

现在考虑 k = 2，x = 3。 

我们计算指数：

 | j | 指数 C(2 + 3 − j − 1, 1) |
 | ---| ---|
 | 1 | C(3,1) = 3 | C(3,1) = 3
 | 2 | C(2,1) = 2 |
 | 3 | C(1,1) = 1 |

 因此结果变为 2^3 · 3^2 · 5^1 = 8 · 9 · 5 = 360。 

这与我们显式应用 prefix-product 两次所发生的情况相匹配：首先是 [2, 6, 30]，然后是 [2, 12, 360]。 

该跟踪显示了重复的前缀操作如何根据可预测的组合模式扩展早期索引的贡献。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(q·n) | O(q·n) | 每个查询计算 x 乘法，每个乘法具有 O(1) 二项式计算 |
 | 空间| O(n) | 高达 ~2000 的阶乘和逆阶乘 |

 这些限制使查询总数足够小，以便迭代每个查询的前缀仍然有效。 预计算可确保二项式系数不会引入隐藏的对数因子。

import sys, io

MOD = 10**9 + 7

def solve():
    input = sys.stdin.readline
    MAXN = 2005

    fact = [1] * MAXN
    invfact = [1] * MAXN
    for i in range(1, MAXN):
        fact[i] = fact[i-1] * i % MOD
    invfact[MAXN-1] = pow(fact[MAXN-1], MOD-2, MOD)
    for i in range(MAXN-2, -1, -1):
        invfact[i] = invfact[i+1] * (i+1) % MOD

    def nCr(n, r):
        if r < 0 or r > n:
            return 0
        return fact[n] * invfact[r] % MOD * invfact[n-r] % MOD

    t = int(input())
    out = []

    for _ in range(t):
        n, q = map(int, input().split())
        a = list(map(int, input().split()))

        for _ in range(q):
            k, x = map(int, input().split())
            res = 1
            for j in range(x):
                exp = nCr(k + x - j - 1, k - 1)
                res = res * pow(a[j], exp, MOD) % MOD
            out.append(str(res))

    return "\n".join(out)

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    return solve()

 ## 边缘情况

 单元素数组表明，无论 k 是多少，所有二项式系数都会崩溃为 1，因为不存在跨索引的扩散。 该算法简化为单个值与指数 1 的重复乘法，并且代码正确返回原始元素。 

当 k = 1 时，二项式系数变为 C(x − j, 0)，对于所有有效 j 均为 1，这意味着每个元素只贡献一次。 这与一个前缀运算产生一个简单的前缀乘积的事实相匹配。 该实现自然地处理了这个问题，因为 nCr(n, 0) 返回 1。 

对于较大的 k 和较小的 x，当指数超出有效范围时，许多二项式系数会变为零。 nCr 防护确保这些贡献消失，防止无效组合项的错误乘法。
