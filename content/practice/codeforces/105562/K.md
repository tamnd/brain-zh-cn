 ## 解决方案
 ## 问题理解

 我们正在研究一个加权图，其中交叉路口是节点，自行车道是具有正长度的无向边。 Karlijn 从节点 1 开始，想要到达节点 n。 

有些节点包含商店。 每个商店独立地要么仍然具有给定概率的 kruidnoten，要么是空的。 在开始旅行之前，她会检查哪些商店目前有货。 如果至少有一家商店有货，则她从节点 1 行进到节点 n，但可以选择访问至少一家有货商店的路径。 在所有这些有效路径中，她采用总边长最短的一条。 如果商店根本没有货，她就不会去，答案是不可能的。 

任务是计算这条最佳路线的预期长度，对所有随机库存配置进行平均。 

重要的结构困难在于，所选择的路径取决于哪个节点子集可用作“有效商店节点”，并且该子集是随机的，每个节点具有独立的概率。 

该图最多可以有 200,000 个节点和边，这会立即排除任何从头开始重新计算每个商店子集的最短路径或枚举库存节点子集的解决方案。 即使处理所有子集也是不可能的，因为 k 可能很大，并且状态空间是指数的。 

一种天真的想法是考虑库存商店的每个子集，运行一条从 1 到 n 的最短路径，限制访问至少一个这些节点，并按概率求平均值。 这会立即失败，因为即使计算这样一条受约束的最短路径也是昂贵的，并且子集的数量呈指数级增长。 

一个更微妙的错误想法是尝试独立对待每个商店，例如将按概率加权从 1 到每个商店以及从每个商店到 n 的最短路径相加。 这种重复计算并忽略了这样一个事实：在每种情况下，只有最便宜的可到达商店才重要。 

一个关键的边缘情况是所有概率都很小时。 例如，如果每个商店的概率都小于 1，则总有非零的可能性没有商店可用。 在这种情况下，无论距离有多远，答案都必须是“不可能”。 

## 方法

 核心困难在于，对于可用商店的每个随机子集，最优路径使用该子集中可能最好的商店。 这立即暗示了一个观点，其中每个节点的行为就像一个候选“枢轴”，我们可以通过它路由从 1 到 n 的路径。 

对于任何固定节点 v，考虑从 1 到 v，然后从 v 到 n 的最短路径。 该路径的长度为 dist1[v] + dist2[v]，其中 dist1 是到 1 的最短距离，dist2 是到 n 的最短距离。 如果 v 不是商店，则无关紧要，因为要求是参观商店。 所以只有商店节点才重要。 

现在考虑活跃商店的特定子集。 最佳路径就是 dist1[v] + dist2[v] 的所有活动商店节点 v 中的最小值。 这将问题简化为计算加权元素的随机子集的预期最小值。 

这将图问题转换为关于值 w[v] = dist1[v] + dist2[v] 的概率顺序统计问题，其中每个元素以概率 p[v] 独立存在。 

剩下的挑战是计算随机子集最小值的期望。 一个标准技巧是按权重对候选节点进行排序，并计算给定节点是最小活跃节点的概率。 要使节点 v 成为最小值，必须满足两个条件：v 必须处于活动状态，并且所有 w[u] < w[v] 的节点 u 必须处于非活动状态。 这导致所有严格更好的节点的缺席概率的乘积。

为了提高效率，我们按 w[v] 对节点进行排序，并按升序扫描，同时保持已处理节点均不处于活动状态的概率。 每个节点贡献其权重乘以它是该排序中的第一个活动元素的概率。 最后，我们还处理根本没有商店处于活动状态的概率。 

唯一剩下的部分是计算 dist1 和 dist2，这是通过两次 Dijkstra 运行完成的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | 指数| 指数| 太慢了|
 | Dijkstra + 概率扫描 | O((n + m) log n + k log k) | O(n + m) | 已接受 |

 ## 算法演练

 1. 从节点 1 运行 Dijkstra 来计算到所有节点 v 的最短距离 dist1[v]。这捕获了从起点到每个潜在商店或中间节点的最佳可能旅行成本。 
2. 从反转图上的节点 n 运行 Dijkstra 来计算 dist2[v]，即每个节点 v 到目的地的最短距离。 这避免了重复重新计算单源最短路径。 
3. 对于每个商店节点 v，计算组合成本 w[v] = dist1[v] + dist2[v]。 如果选择 v 作为所访问的商店，这表示最佳可能路线。 
4. 仅收集商店节点的所有对 (w[v], p[v])。 如果没有商店，成功的概率为零，答案是不可能的。 
5. 按 w[v] 按升序对这些对进行排序。 这种排序反映了在已实现的场景中成为成本最低的商店的优先级。 
6. 保持运行值 cur_fail 初始化为 1。这表示所有先前处理的（较便宜的）商店不可用的概率。 
7. 浏览已排序的列表。 对于每个商店 v，将术语 w[v] * cur_fail * p[v] 添加到答案中。 然后更新 cur_fail *= (1 - p[v])。 这强制 v 仅当它是按排序顺序的第一个可用商店时才做出贡献。 
8. 处理完所有商店后，检查 cur_fail。 这是根本没有商店可用的概率。 如果大于零，则输出“不可能”。 否则输出累积期望。 

### 为什么它有效

 按 w[v] 排序可确保在处理节点时，所有更便宜的候选节点已被视为潜在最小值。 节点 v 是最小活跃店铺的概率就是该节点处于活跃状态且每个 w 值较小的节点处于不活跃状态的概率。 因为激活是独立的，所以该概率干净地分解为 p[v] 乘以所有早期节点 u 上的 (1 - p[u]) 的乘积。 这使得扫描保持正确的累积故障概率，并且每个贡献都根据其成为唯一最小值的事件精确加权。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def dijkstra(n, graph, start):
    INF = 10**30
    dist = [INF] * (n + 1)
    dist[start] = 0
    pq = [(0, start)]

    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

def solve():
    n, m, k = map(int, input().split())

    graph = [[] for _ in range(n + 1)]
    for _ in range(m):
        a, b, w = map(int, input().split())
        graph[a].append((b, w))
        graph[b].append((a, w))

    dist1 = dijkstra(n, graph, 1)
    dist2 = dijkstra(n, graph, n)

    shops = []
    for _ in range(k):
        i, p = input().split()
        i = int(i)
        p = float(p)
        if dist1[i] < 10**29 and dist2[i] < 10**29:
            shops.append((dist1[i] + dist2[i], p))

    if not shops:
        print("impossible")
        return

    shops.sort()

    cur_fail = 1.0
    ans = 0.0

    for w, p in shops:
        ans += w * cur_fail * p
        cur_fail *= (1.0 - p)

    if cur_fail > 1e-15:
        print("impossible")
    else:
        print(f"{ans:.10f}")

if __name__ == "__main__":
    solve()
```代码的图形部分是标准的 Dijkstra，运行两次以获得从起点到目标的距离。 这两个数组是唯一需要的特定于图的预处理。 

每个商店都被转换为一个标量权重，代表穿过该商店的最佳可能绕行路线。 这种减少是消除所有路径结构随机性的关键步骤。 

扫描逻辑使用浮点运行乘积。 变量`cur_fail`跟踪所有以前见过的（较便宜的）商店都不可用的概率。 每个新商店都准确地贡献了它是按排序顺序第一个可用商店的概率。 

最后检查`cur_fail`处理没有可用商店的事件。 如果该概率非零，则在问题定义下期望未定义，因此我们输出“不可能”。

 ## 边缘情况

 一种微妙的情况是多个商店共享相同的 w[v]。 在这种情况下，它们排序后的顺序并不重要，因为每个贡献的概率都以所有较早的不活跃的概率为条件。 如果交换两个等权节点，则表达式仍然划分相同的概率空间，因为两者对应于相同的成本级别，并且只有其中一个可以是相等值中的第一个活跃节点。 

另一种情况是从任何一方都无法到达商店。 必须尽早排除此类节点。 如果 dist1[v] 或 dist2[v] 是无限的，则 w[v] 毫无意义，因为没有有效的路径可以使用该商店。 该算法自然会通过在商店收集期间忽略这些节点来处理此问题。 

最后一个极端情况是所有商店都不存在或概率为零。 在这种情况下，cur_fail 始终保持为 1，算法正确输出“不可能”，而无需尝试计算期望。
