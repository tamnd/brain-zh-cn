 ## 解决方案
 ## 问题理解

 我们在无限的城市街区网格上有五个特殊的十字路口。 每个街区都由严格水平或垂直延伸的道路连接，并且沿行或列的每一对相邻的交叉口对应于一个相同长度的路段。 我们可以“修复”个别路段，修复后的路段就可以使用了。 

目标是确保对于五个给定重要地点中的每一对，它们之间都存在一条最短的曼哈顿路径，且仅使用已修复的路段。 这里的最短路径是指在网格意义上永远不会走不必要的弯路的路径，因此它仅沿着网格线在x和y上单调移动。 

我们必须选择最少数量的单位路段进行修复，以便该条件适用于所有对。 

关键的困难在于我们没有建立任意的连接。 我们强制所有对之间的最短路径连接，这意味着非常强大的几何结构：两点之间的任何路径必须保持在其轴对齐的边界框内，并且仍然是最短的。 

坐标最多为1000，因此原则上对所有网格边进行直接几何或组合探索是可行的，但网格在概念上是无限的，因此我们必须避免全局枚举所有边。 就特殊点而言，问题规模很小，恰好是五个，这强烈建议采用结构驱动的解决方案，而不是一般的图搜索。 

一个微妙的边缘情况是所有五个点都位于一条水平或垂直线上。 在这种情况下，最佳结构会塌陷为单个片段，而天真的成对思维往往会过度计算，因为它独立地对待每一对，而不是共享修复后的片段。 

当点形成类似矩形的配置时，会出现另一种失败情况。 通过独立的曼哈顿路径连接每一对的简单方法会重复计算许多共享边，从而产生很大的高估。 

因此，正确的解决方案必须确定一个同时支持所有对的最短路径的共享结构。 

## 方法

 思考该问题的一种强力方法是对包含所有五个点的边界框中的每个网格交叉点进行建模，并考虑每个边子集，检查是否存在所有所需的最短路径。 这是立即不可行的，因为即使是 1000 x 1000 的网格也包含大约一百万个顶点和两百万条边，并且枚举边子集在该大小中是指数级的。 

即使我们将自己限制为仅包含五个点的边界框内的边，候选边的数量仍然约为 10^6，并且任何子集枚举都是不可能的。 

第二个天真的想法是考虑每一对点并选择它们之间的一条最短的曼哈顿路径，然后取所有这些路径的并集。 这保证了固定路径选择的正确性，但是失败了，因为路径的选择不是独立的。 同一对之间的两条不同的最短路径可能与其他对的路径共享结构，并且贪婪地选择每对不会最小化重叠。 

关键的观察结果是，最佳解决方案不是路径的任意并集，而是形成轴对齐线段的并集，其端点位于给定点之间或仔细选择的类似斯坦纳的交叉点处。 由于只有五个点，最优解的结构受到足够的约束，所有最短路径的并集可以表示为一个小的组合结构。

一个关键的重新表述是从水平和垂直覆盖的角度来思考。 如果使用 x 处的垂直线段，则它可以为多个点提供服务，这些点的 x 间隔需要穿过该线。 对于水平线段也是如此。 问题变成选择一组网格线（沿选定的 x 或 y 坐标的线段），这些网格线一起允许所有对之间的最短曼哈顿连接。 

因为两点之间的每条最短路径都完全位于它们定义的矩形内，所以唯一相关的边是所有这些矩形的并集内的边。 这将几何形状简化为由五个 x 坐标和五个 y 坐标引起的小排列。 

对于五个点，只有五个可能的垂直“事件线”和五个水平“事件线”。 可以假设任何最佳解决方案都位于这些坐标上，因为将线段移离输入点的坐标不能改善覆盖范围，而只会增加或保留成本。 

这将问题简化为在最多 25 个候选交叉节点的小网格上的有限组合优化。 

然后，我们将连续 x 和 y 坐标之间的每个单元格解释为潜在的边缘位置。 每个候选解决方案对应于选择水平和垂直单元段的子集，这些单元段以确保每对点可以通过单调路径路由的方式连接选定的矩形。 

此时，问题就相当于选择一个非常小的隐式网格图的最小成本子图，使得所有五个终端都在最短路径约束下连接。 由于最短路径约束强制单调性，因此任何可行的结构都必须允许通过共享的“交叉”结构独立地在 x 坐标和 y 坐标之间移动。 

最终的简化是，最佳解决方案始终对应于选择一组候选网格点，这些网格点在五个点上形成连接的直线斯坦纳树，但具有路径必须最短的额外约束。 在这种特殊情况下，最短路径约束迫使解本质上像 5 点集上的曼哈顿施泰纳树一样，其中边仅沿着点的投影。 

因此，我们将问题简化为评估少量候选的基于中值的结构，这可以通过枚举中心“枢纽”交叉点的所有选择来解决，该中心“枢纽”交叉点是通过从给定的 x 坐标中选取 x 并从给定的 y 坐标中选取 y 形成的，并计算通过该枢纽结构连接所有点所需的最小总线段。 

根据公式，这会产生 O(5^2) 或 O(5^3) 风格的评估，这是微不足道的。

 ## 算法演练

 我们将问题压缩为对五个点的一小组 x 和 y 坐标的推理。 设点为 P0 到 P4。

1. 收集五个点的所有 x 坐标和 y 坐标。 这些定义了所有相关的垂直和水平“事件线”，其中最佳解决方案可以改变结构。 任何最佳解决方案都可以假设仅使用这些坐标。 
2. 尝试通过将任意 x 坐标与任意 y 坐标配对形成的所有候选中心位置。 这最多给出 25 个候选网格交叉点。 直觉是，在最佳共享结构中，连接性是通过终端定义的主要垂直线和水平线的交叉点来实现的。 
3. 对于每个候选中心，计算使用曼哈顿路径将所有点连接到其的成本，但仅考虑所需水平和垂直线段的并集。 从点到集线器的每个连接都沿着其 x 对齐和 y 对齐投影贡献线段。 
4. 计算并集时，确保共享段不被重复计算。 这是通过跟踪沿每条水平线和垂直线的哪些单元段至少使用一次来处理的。 
5. 集线器的成本是该联合中使用的唯一单元边的数量。 我们通过在由排序的 x 和 y 坐标引起的离散网格上标记分段来计算这一点。 
6. 答案是所有候选中心的最低成本。 

其原理与曼哈顿几何中最短路径的结构有关。 两点之间的任何最短路径完全由 x 和 y 方向的独立运动决定。 为了同时满足所有线对的需求，解决方案必须提供沿 x 和 y 坐标的共享走廊，以允许在多个线对之间重复使用。 任何最佳配置都可以转换为所有分支都发生在输入对齐线的交叉点处的配置，因为将段移离这些线无法在保留最短路径可行性的同时提高重用性。 这会创建一个小的有限搜索空间，其中单个交叉路口充当所有路线的组合合并点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def segments_between(sorted_coords):
    # returns number of unit segments covered when connecting all consecutive coords fully
    return sorted_coords[-1] - sorted_coords[0]

def solve():
    pts = [tuple(map(int, input().split())) for _ in range(5)]
    
    xs = sorted(set(p[0] for p in pts))
    ys = sorted(set(p[1] for p in pts))

    ans = float('inf')

    for cx in xs:
        for cy in ys:
            used_h = set()
            used_v = set()

            for x, y in pts:
                # horizontal segment from x to cx at row y
                if x != cx:
                    a, b = sorted((x, cx))
                    for xx in range(a, b):
                        used_h.add((y, xx))
                # vertical segment from y to cy at column cx
                if y != cy:
                    a, b = sorted((y, cy))
                    for yy in range(a, b):
                        used_v.add((xx := cx, yy))

            ans = min(ans, len(used_h) + len(used_v))

    print(ans)

if __name__ == "__main__":
    solve()
```该代码尝试通过从输入点中选择 x 和 y 形成的每个候选交集。 对于每个这样的中心，它构建从每个点到该中心的曼哈顿路径的联合。 每个路径被分解为水平和垂直单元段，我们将这些段存储在集合中，以确保共享使用被计算一次。 

关键的实现细节是将边缘表示为整数坐标之间的单位步长。 水平线段存储为`(y, x)`对，代表之间的段`(x, y)`和`(x+1, y)`。 垂直线段存储为`(cx, y)`代表之间的边的对`(cx, y)`和`(cx, y+1)`。 

使用集合就足够了，因为我们只需要联合中唯一段的基数。

 ## 边缘情况

 关键的边缘情况是所有点共享相同的 x 坐标。 在这种情况下，任何正确的解决方案都会简化为连接最小和最大 y 的单个垂直线段。 该算法自然地处理这个问题，因为每个候选集线器都位于相同的 x 坐标上，因此所有路径都会折叠成垂直联合而不会重复。 

当点形成两个相距很远的密集簇时，就会出现另一种边缘情况。 简单的成对构造会尝试连接集群中的每个点，但基于中心的构造可确保在集群之间仅创建一个桥接走廊，并且所有集群内路径都重用本地段。 

最后一个微妙的情况是，最佳中心不在视觉上“中心”，而是与极端坐标之一对齐。 对所有输入 x 和 y 值的枚举保证了这种情况得到显式测试，因此该算法不依赖于有关对称性的几何直觉。
