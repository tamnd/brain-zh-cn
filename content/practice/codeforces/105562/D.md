 ## 解决方案
 ## 问题理解

 我们有一组政党，每个政党都有一定数量的席位。 “联盟”只是这些政党的一个子集。 我们想要计算有多少子集满足成为有效执政联盟的非常具体的概念。 

如果一个子集拥有所有席位的绝对多数，则该子集是可接受的，这意味着该子集内的席位总和大于所有政党总和的一半。 然而，这还不够。 联盟也必须是最小的，因为如果你从联盟中删除任何一个政党，它就会立即不再是绝对多数。 因此，联盟内的每个政党对于维持多数席位都至关重要。 

这些限制允许最多 60 个政党，每个政党最多有 10,000 个席位。 这立即表明直接枚举所有子集原则上是可能的，因为大约有 2^60 个子集，但在实践中显然是不可能的。 任何有效的解决方案都必须利用“最小多数”条件下的结构，而不是暴力破解子集。 

天真的尝试会尝试所有子集并检查这两个条件。 这失败了，因为 2^60 是一个天文数字，远远超出了任何可行的计算。 

更微妙的失败案例来自于仅检查多数条件。 例如，如果我们取一个总和刚刚超过一半的子集，那么它不一定是最小的。 考虑像 60, 50, 49 这样的情况，其中总数为 159。子集 {60, 50} 的总和为 110，这是多数，但删除 50 留下 60，这仍然是多数。 所以即使通过了第一个条件也是无效的。 

相反的错误也很常见：确保最少而不确保多数。 每个元素“相对于其余元素较大”的子集可能仍然无法超过总数的一半。 

## 方法

 暴力方法很简单：迭代所有子集，计算它们的总和，然后对于子集中的每个元素检查删除它是否会破坏多数条件。 这是正确的，但子集枚举和每个子集验证都具有指数复杂性。 对于 60 个元素，这变得完全不可行。 

关键的观察是以一种方式重新表述极简性，消除对单独检查子集中每个元素的依赖。 假设一个子集有总和`S_sub`。 如果删除任何元素会使其失去多数，则对于每个元素`x`在子集中我们必须有：`S_sub - x <= S_total / 2`重新排列给出：`x >= S_sub - S_total / 2`这意味着子集中的每个元素必须至少达到某个阈值，该阈值取决于子集总和本身。 这种依赖关系一开始看起来是循环的，但如果我们修复子集中的最小元素，它就变得易于管理。 

如果我们假设一个子集具有最小元素`m`，那么所有元素至少`m`，则条件变为：`S_sub <= S_total / 2 + m`同时，我们还需要：`S_sub > S_total / 2`因此，对于固定的最小元素，有效子集正是那些总和位于由该最小值控制的狭窄区间内的子集。 

这建议按哪个元素是最小值进行排序或迭代，并计算所有元素都来自至少为该值的元素后缀的子集。 这将问题转化为在总和范围的约束下计算子集总和。 

我们可以使用动态规划预先计算每个后缀的子集和计数，然后对于每个可能的最小元素，查询有多少个子集和落入所需的区间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^n·n) | O(2^n·n) | O(1) | O(1) | 太慢了 |
 | 后缀 DP + 最小主元 | O(n·S) | O(n·S) | O(n·S) | O(n·S) | 已接受 |

 这里`S`是座位总数，最多大约6e5个。 

## 算法演练

 我们重新制定计数，以便通过首先选择其最小元素，然后用更大或相等的元素扩展它来对每个联盟进行计数。 

1. 计算所有席位的总和，并将多数阈值定义为该总和的一半。 
2. 按席位值对各方进行排序，或者同等地按席位大小的递增顺序处理索引，以便后缀代表“最小元素锚定子集”的有效选择。 
3. 构建动态规划表，其中`dp[i][s]`表示仅使用索引中的各方来选择子集的方法数`i`到`n`总和为`s`。 
4. 从右到左计算这个 DP。 对于每一方`i`，我们要么排除它，要么包含它，并相应地转移总和。 这确保每个后缀都有其完整的子集和分布。 
5. 对于每个索引`i`, 招待方`i`作为联盟的最小元素。 任何包含它的有效联盟必须从索引中选择所有其他成员`i+1`向前。 
6. 如果最小元素是`p[i]`，然后对于选定的后缀子集，其总和`t`，完整的联盟总和变为`t + p[i]`。 我们将有效性条件转化为约束`t`:

 联盟必须满足`S_total / 2 < t + p[i] <= S_total / 2 + p[i]`，这简化为`S_total / 2 - p[i] < t <= S_total / 2`。 
7. 将所有 dp 计数相加`t`在这个区间内对于每个`i`，并对结果进行累加。 

正确性来自于每个有效联盟都有一个唯一的最小元素这一事实。 一旦该元素被固定，联盟的其余部分将不受限制，除非至少是该最小值（由后缀限制保证），并且多数/最小值约束变成简单的总和界限。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    total = sum(a)
    half = total // 2
    
    max_sum = total
    
    dp_next = [0] * (max_sum + 1)
    dp_next[0] = 1
    
    ans = 0
    
    for i in range(n - 1, -1, -1):
        p = a[i]
        
        dp_cur = dp_next[:]  # not take p[i]
        for s in range(max_sum - p, -1, -1):
            if dp_next[s]:
                dp_cur[s + p] += dp_next[s]
        
        dp_next = dp_cur
        
        low = half - p + 1
        high = half
        
        if low < 0:
            low = 0
        
        for t in range(low, high + 1):
            ans += dp_next[t]
    
    print(ans)

if __name__ == "__main__":
    solve()
```DP 是从末尾开始构建的，因此`dp_next`始终表示严格位于当前索引之后的后缀子集。 当我们处理一个新元素时，我们克隆 DP 并添加包含当前座位数的转换。 

区间查询使用变换后的不等式。 下界是`half - p[i] + 1`因为我们需要在左侧严格不等式，而上限包含在`half`。 

一个微妙的点是我们有意重用`dp_next`在为每个索引更新它之后，因此每一方都被正确地视为其后缀中潜在的最小子集。 

## 工作示例

 考虑带座位的小型配置`[3, 2, 2]`。 总数是 7，所以一半是 3。 

我们构建后缀DP表：

 对于索引 2（值 2），子集为`{}`,`{2}`所以总和是 0 和 2。 

对于索引 1（值 2），子集超过`[2,2]`给出总和`{0,2,2,4}`。 

对于索引 0（值 3），子集超过`[3,2,2]`给出所有组合。 

现在评估每个最小值：

 对于至少 3 个，后缀为`[2,2]`。 我们需要`t`这样`3.5 - 3 < t <= 3.5`， 意义`0.5 < t <= 3.5`，所以有效`t`是`{2}`。 这给出了子集`{3,2}`和`{3,2}`（取决于选择哪 2 个），贡献两个联盟。 

对于索引 1 处的最小值 2，后缀为`[2]`。 我们需要`3.5 - 2 < t <= 3.5`， 所以`1.5 < t <= 3.5`, 给予`t=2`。 这产生了联盟`{2,2}`。 

| 最低 | 后缀 | 有效 t 范围 | 有效的联盟 |
 | --- | --- | --- | --- |
 | 3 | [2,2]| (0.5, 3.5] | {3,2} 变体 |
 | 2 | [2] | (1.5, 3.5] | {2,2} |

 这证实了每个联盟通过其最小元素被精确计数一次。

 ## 边缘情况

 对于单身派对`[x]`，总计为`x`一半是`x/2`。 唯一的子集是`{x}`，总是满足多数和极少数。 DP 正确地对此进行计数，因为后缀 DP 以空子集开头并且范围查询包括`t = 0`。 

对于不平衡较大的情况，例如`[100, 1, 1, 1]`，只有锚定在 100 的联盟才能以最小的方式达到多数约束。 DP 正确地限制了后缀和，使得较小的元素不能形成独立的多数集。 

对于对称情况，例如`[2,2,2]`，每一对都是有效的最低多数联盟。 每个联盟只计算一次，因为它与其唯一的最小索引相关联，从而防止过度计算。
