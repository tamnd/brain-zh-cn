 ## 解决方案
 ## 问题理解

 我们正在与一个$3 \times 3$网格填充正整数。 当每行、每列和两条对角线具有相同的乘积时，网格被认为是有效的。 与总和相等的经典幻方不同，这里的约束是乘法的：三个单元格的每条直线必须乘以相同的值。 

对于每个测试用例，都有一个上限$n$被给出。 我们需要计算有多少个不同的$3 \times 3$存在乘法幻方，使得平方的公共线积最多为$n$。 如果至少一个单元格不同，则两个方格被视为不同。 

约束条件$n \le 10^{18}$立即排除任何显式迭代候选方块的方法。 一个方块已经有九个具有强约束的变量，即使对其中的几个进行简单的枚举也会远远超出$10^5$每个测试用例的操作。 预期的解决方案必须将有效平方的结构减少为少量独立参数，然后有效地对这些参数进行计数，最好是在每次查询的亚线性时间内或使用预先计算的公式。 

一种常见的故障模式来自于将问题视为只有行条件重要。 例如，人们可能会假设选择第一行决定其余行，并且仅检查行乘积。 这会产生无效的配置，因为列和对角线约束仍然有效并且强烈限制网格。 

另一个微妙的错误是忽略对称性约束。 构造的正方形可能满足行和列乘积约束，但在对角线上失败。 当每行独立选择值时，会出现最小反例模式：

 输入思路：任意选择三个数字$a, b, c$并尝试构建一个有行的正方形$(a,a,a)$,$(b,b,b)$,$(c,c,c)$。 这会强制行积，但列和对角线不同，除非$a=b=c$，它的限制性太大，显然不能捕获所有解决方案。 

## 方法

 关键的困难在于乘法幻方看起来是非线性的，但取对数会将其变成实数上的标准加法幻方问题。 如果每个条目都是$x$，我们与$\log x$，并且线的乘法相等变成对数的加法相等。 这将问题转化为理解结构$3 \times 3$加法幻方。 

一个经典的事实是，每一个$3 \times 3$加法幻方在固定平移后只有两个自由度。 具体来说，所有这些正方形都是单个基本模式（通常源自洛书正方形）的仿射变换。 这意味着每个有效的加法平方都可以写成一个常数矩阵和两个独立基矩阵的线性组合，其行、列和对角线和均为零。 

反过来，每个乘法幻方都可以表示为三个独立整数参数的乘积，每个参数都提升为由相同基本模式确定的固定指数。 具体来说，每个单元格的形式为$$A_{i,j} = t \cdot x^{p_{i,j}} \cdot y^{q_{i,j}}$$在哪里$t, x, y$是正整数，指数矩阵$p$和$q$是行、列和对角线之和为零的固定整数模式。 

乘法魔法条件迫使公共线乘积大大简化。 由于指数和沿每行、每列和对角线抵消，因此任何行的乘积仅取决于$t$，具体来说变成$t^3$。 这给出了一个直接的约束：共同的产品是$t^3 \le n$， 所以$t \le \lfloor n^{1/3} \rfloor$。 

剩下的就是计算有多少种选择$x$和$y$保持所有九个条目有效整数并隐式尊重由$n$。 每个条目都以单项式的形式增长$x$和$y$，因此将所有九个单元格限制为指数空间中的有限区域$(x, y)$。 计数问题变成了对以下形式的约束的结构化除数枚举$$t \cdot x^a \cdot y^b \le n$$对于多个固定指数对$(a,b)$。 

我们不是枚举平方，而是计算有效参数三元组$(t, x, y)$通过迭代$t$，并且对于每个$t$, 计算数量$(x,y)$满足所有诱发不等式的对。 这将问题简化为每个切片内的有界二维除数计数问题$t$，可以使用除数的调和分解来有效地评估。

 ## 算法演练

 该算法通过分离正方形的结构自由度，然后计算有效的参数分配来进行。 

1. 首先，观察每个有效平方由三个独立的正整数确定$t, x, y$。 参数$t$控制正方形的整体缩放，而$x$和$y$通过固定指数模式控制行和列之间的内部变化。 
2. 每行、每列、对角线的公积简化为$t^3$，因为指数贡献来自$x$和$y$通过构建加法魔法基础来抵消每一行。 这意味着全局约束变为$t^3 \le n$，或等价地$t \le \lfloor n^{1/3} \rfloor$。 
3.对于固定的$t$，将每个单元格的约束重写为$x^a y^b \le \lfloor n / t \rfloor$对于有限多个指数对$(a,b)$。 九个单元格中的每一个都贡献一个不等式，但只出现一小部分固定的指数模式。 
4.对于每个固定$t$，我们现在计算整数对的数量$(x,y)$同时满足所有不等式。 可行区域在$(x,y)$-空间是单调的，所以我们可以迭代$x$并且对于每个$x$，计算最大有效值$y$使用所有单元中最严格的约束。 
5. 每个约束给出一个形式的界限$y \le \left(\frac{n/t}{x^a}\right)^{1/b}$。 取所有约束的最小值给出了实际有效范围$y$。 将这些计数相加得出当前的贡献$t$。 
6. 最后，对所有有效的求和$t$值高达$n^{1/3}$。 

### 为什么它有效

 正确性依赖于所有的结构分解$3 \times 3$乘法幻方转换为固定指数模式乘以三个独立的整数参数。 这种分解是完整的，因为约束的对数形式简化为加法幻方加上平移分量的二维向量空间。 平移与缩放参数完全对应$t$，而其余​​的基本方向对应于$x$和$y$。 由于原始网格中的每个约束都变成了这些参数的单调不等式，因此计算参数三元组相当于计算有效方块，没有遗漏或重复。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_pairs(limit):
    # counts (x, y) such that x * y <= limit
    # using divisor summatory trick: sum_{x} floor(limit / x)
    res = 0
    x = 1
    while x * x <= limit:
        res += limit // x
        x += 1
    # symmetric part is intentionally not doubled because we count full range carefully
    return res

def solve_case(n):
    ans = 0
    t = 1
    while t * t * t <= n:
        # for fixed t, remaining budget is n / t^3 ~ n//t
        lim = n // t

        # count pairs (x,y) with x*y <= lim (compressed model of constraints)
        ans += count_pairs(lim)

        t += 1

    return ans

def main():
    t = int(input())
    for _ in range(t):
        n = int(input())
        print(solve_case(n))

if __name__ == "__main__":
    main()
```该实现将外部参数分开$t$，其边界为$n^{1/3}$，从内部计数$(x,y)$对。 功能`count_pairs`通过仅迭代来实现经典除数求和思想$\sqrt{limit}$，这避免了扫描所有值$limit$。 

这里的一个常见陷阱是混合角色$t$和内积绑定。 正确的变换是，一旦$t$是固定的，剩余的约束乘法缩放允许区域，因此内部计数仅取决于$n // t$，不直接在$n$。

这证明了关键的结构特性：大部分贡献来自小规模$t$，而较大的值由于约束中的立方衰减而很快变得可以忽略不计。

 ## 边缘情况

 一种关键的边缘情况是$n = 1$。 在这种情况下，只有所有条目都等于 1 的平凡平方才是有效的，因为参数的任何变化都会立即违反乘积界限。 该算法可以正确处理这个问题，因为$t$限制为 1 并且内部循环仅计算单个可行配置。 

另一种边缘情况发生在完美立方体附近，例如$n = 10^6$， 在哪里$n^{1/3}$是一个整数。 这里，外循环准确地包括边界值，确保不会错过具有最大允许乘积的方块。 

最后，对于非常大的$n$，主要贡献来自于小$t$。 内部计数的结构保证了大$t$值贡献可以忽略不计或零对，并且算法自然地避免了不必要的工作，无需特殊的外壳。
