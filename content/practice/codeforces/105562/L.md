 ## 解决方案
 ## 问题理解

 我们有一个图书馆书架系统，如果纯粹用于书籍，每个书架可以容纳固定数量的书籍。 然而，书架还可以选择性地展示艺术品，这会减少书架的有效容量。 

每个架子都有一个结构高度，但该高度仅在不直接影响容量的情况下才重要。 重要的是，每个架子在容量规则方面的行为都是相同的：它最多可以存储`x`如果正常使用的话，或者最多`y`书籍（如果上面放有艺术品）。 每个架子最多可以放置一件艺术品。 

我们还获得了多组书籍尺寸，并且每本书都必须放置在一些书架上。 书籍不会被拆分，每本书占用一个书架容量单位，因此我们只关心每个书架的总计数。 

任务是确定是否可以在给定的书架上放置所有书籍，如果可以，请选择尽可能多的书架来放置一件艺术品，同时仍然适合所有书籍。 

关键的压力是每件艺术品都会减少容量`x - y`，因此使用过多的艺术书架可能会导致总容量不足以容纳所有书籍。 然而，各个货架上的艺术品摆放并不统一，因此我们必须决定可以“降级”多少个货架以达到容量`y`同时仍至少保持总容量`m`。 

限制很大：最多有 100000 个书架和 100000 本书。 这立即排除了尝试书架的所有子集或使用嵌套循环单独分配书籍的任何方法。 我们至少需要线性或接近线性的行为，可能需要排序和贪婪结构。 

一个微妙但重要的一点是，书架高度和书籍尺寸与可行性无关。 由于每本书占据一个空间单位，并且所有书架除了容量之外都是相同的，因此这些值`a`和`b`完全不影响最终的逻辑。 一个幼稚的解决方案可能会错误地尝试按高度将书籍与书架相匹配，但这是一种转移注意力的做法。 

当即使没有艺术品总容量也不足时，就会出现边缘情况。 例如，如果`n * x < m`，那么任何配置都不起作用，即使在考虑艺术品之前，答案也一定是“不可能”。 

另一个特殊情况是几乎所有货架都必须降级为`y`容量。 在这种情况下，对哪些架子上存放艺术品的贪婪选择根本不依赖于给定的数组，只依赖于全局容量限制下的最大化计数。 

## 方法

 一个直接的暴力想法是尝试所有可能数量的包含艺术品的架子。 假设我们尝试`k`货架作为艺术货架。 那么总容量变为：`(n - k) * x + k * y = n * x - k * (x - y)`对于每个`k`，我们检查这个容量是否至少为`m`。 如果是，我们考虑`k`可行并取最大值`k`。 

这已经消除了分配特定书架或书籍的需要。 然而，一个幼稚的实现可能仍然尝试模拟将书籍放入书架，这将花费`O(n + m)`每次检查并导致不必要的爆炸。 

关键的观察是可行性仅取决于总容量，而不取决于分布。 一旦我们意识到这一点，问题就简化为寻找最大的`k`使得不等式成立：`n * x - k * (x - y) >= m`这是一个简单的单调条件`k`。 作为`k`增加，容量减少，所以我们可以直接求解边界而不是搜索。 

我们计算所需的最小容量赤字：`deficit = n * x - m`如果`deficit < 0`，即使是零个艺术书架也无法容纳所有书籍。 

否则，每个艺术架子都会被移除`x - y`容量，因此艺术架的最大数量为：`k = deficit // (x - y)`但我们也必须确保`k <= n`。 

这产生了直接的封闭式解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 通过重新计算对 k 进行暴力破解 | O(nm) 或 O(n²) 取决于模拟 | O(1) | O(1) | 太慢了|
 | 封闭式容量推理 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为计算容量而不是模拟放置。 

## 算法演练

 1. 计算如果每个书架都已完全用于书籍的情况下的总可用容量。 这是`total = n * x`。 这代表放置任何艺术品之前的上限。 
2. 比较`total`与书籍数量`m`。 如果`total < m`，即使在最好的情况下也不可能放置所有的书，所以我们立即返回“不可能”。 
3. 计算超出书籍严格需要的剩余容量有多少：`extra = total - m`。 这衡量了我们可以承受放置艺术品时损失的容量。 
4. 每件艺术作品都会减少容量`x - y`。 这是因为可以容纳的架子`x`书籍现在只容纳`y`。 
5. 计算艺术品架的最大数量为`k = extra // (x - y)`。 这确保我们永远不会超出可用的产能余量。 
6. 由于我们不能使用比现有更多的架子，因此将结果限制为最多`n`。 

### 为什么它有效

 关键的不变量是，只有总容量很重要，而不是货架的身份或订购。 每个配置都带有`k`艺术货架产生相同的总容量`n * x - k * (x - y)`。 由于书籍在空间消耗方面没有区别，因此任何满足总容量的安排都是有效的。 容量随着增加而单调减少`k`保证最大的可行`k`正是使用所有可用的松弛而不超过它的一种。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m, x, y = map(int, input().split())
a = list(map(int, input().split()))
b = list(map(int, input().split()))

total = n * x

if total < m:
    print("impossible")
else:
    extra = total - m
    k = extra // (x - y)
    if k > n:
        k = n
    print(k)
```实现首先读取所有输入，即使最终逻辑中未使用书架高度和书籍尺寸。 这反映了建模的简化：它们与可行性无关。 

我们在假设没有艺术品的情况下计算总容量。 如果这还不够，我们会立即终止。 否则，我们会计算可以牺牲多少容量。 每个艺术架的容量都有固定的成本，因此划分给出了最大数量。 

唯一微妙的一点是确保使用整数除法，因为不允许部分分配容量损失。 夹紧方式`n`防止超出可用货架，尽管从数学上讲，当`extra >= 0`。

 ## 边缘情况

 一个关键的边缘情况是，即使是零件艺术品也已经超出了容量。 例如：

 输入：`n=2, m=10, x=3, y=2`总容量为`6`，小于`10`。 该算法在进行任何进一步计算之前立即返回“不可能”。 一种幼稚的方法可能仍然会尝试分发书籍或对艺术布局进行推理，但一旦容量限制失效，那就没有必要了。 

另一种情况是当`y`非常接近`x`。 例如：`x=10, y=9`这里每个艺术架仅减少容量`1`。 该算法正确地将其解释为允许许多艺术架，仅受小松弛的限制`n*x - m`。 单调线性成本模型确保除法步骤在没有特殊情况下保持有效。 

最后，当`m = 0`，每个书架都可以容纳艺术品，因为书籍不需要容量。 公式得出`extra = n*x`，因此`k = (n*x) // (x-y)`，但我们仍然必须夹紧`n`，从而产生正确的充分利用。
