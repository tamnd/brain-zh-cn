 ## 解决方案
 ## 问题理解

 每台起重机位于平面上的固定点，并具有垂直塔高度。 我们在每座塔的顶部安装了一根旋转水平梁。 梁长必须为正整数且不能超过塔高。 

当梁设置为某个方向和长度时，它在塔的高度处的水平面上成为直线段。 如果该段穿过另一个塔的地面位置，而梁处于或低于该塔的高度，则会发生碰撞，因为梁会与另一个起重机的垂直结构物理相交。 如果光束恰好穿过塔的边界，则仍然允许接触并且不被视为碰撞。 

任务是为每台起重机分配一个长度，以便在所有可能的旋转中，至少一根梁可以到达的总区域最大化。 由于每个起重机覆盖的圆盘的半径等于其选定的长度（但仅达到其高度限制），因此问题简化为为每个点选择最大的安全半径，该半径不允许与禁止的塔有任何交叉。 

关键的几何限制是，如果一台起重机的梁可以到达另一台起重机的位置，并且另一台起重机足够高，梁可以与它相交，则该起重机只会与另一台起重机发生冲突。 因为所有的高度都不同，所以每对起重机都有明显的优势关系：对于起重机i，只有高度大于或等于h_i的起重机才能阻挡它。 任何较短的塔都可以安全地忽略，因为光束会从其上方经过。 

独立尝试所有可能半径的简单方法会失败，因为特定方向上的单个远距离高起重机可以急剧限制允许的半径，并且这种限制对于每个起重机来说都是不同的，具体取决于哪些其他起重机更高。 

当多台高起重机稀疏地包围一台低起重机时，就会出现微妙的故障情况。 例如，即使所有附近的点都很小，单个远但高的起重机仍然可能是限制因素。 任何仅考虑网格中的本地邻居或按距离排序而不考虑高度顺序的策略都会高估允许的半径。 

约束足够小，以至于$O(n^2)$每个起重机的几何比较是可行的，但足够大，以至于涉及每对角度扫描或每个半径重新计算的任何事情都将是不必要的复杂性。 

## 方法

 暴力解释从每台起重机独立开始，并询问：在撞上禁塔之前，其梁可以延伸多远。 对于固定式起重机，其他所有起重机要么因为较短而无关紧要，要么如果梁正好朝它旋转，则它定义了潜在的碰撞。 因此，最大安全长度受到欧几里德距离中最近的此类阻塞起重机的限制。 

这导致了一个简单的计算：对于每台起重机 i，检查至少同样高的其他所有起重机 j。 如果光束到达 j 的位置，就会发生碰撞，因此可用长度必须严格小于 i 和 j 之间的距离。 取所有此类约束的最小值即可得出 i 的答案。 

这已经表明$O(n^2)$解决方案，但重要的结构改进是认识到我们根本不需要模拟光束旋转或角度几何。 每个起重机的可行区域只是一个圆盘，其半径被欧几里得距离中最近的更高或相等的塔所裁剪。 

剩下的问题是计算问题：如何有效地评估这些约束。 自从$n \le 500$，直接两两比较就足够了，但我们也可以通过按高度降序处理起重机来组织计算。 在加工起重机时，所有已加工的起重机都严格更高，因此它们恰好形成了一组潜在的阻碍因素。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 成对暴力破解 |$O(n^2)$|$O(1)$额外 | 已接受 |
 | 按高度+增量最小距离排序 |$O(n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将起重机从最高到最矮进行处理，以便在评估起重机时，保证所有先前插入的起重机都严格更高。 

### 1. 按高度递减对起重机进行排序

 我们对起重机进行重新排序，以便当我们到达起重机时，之前见过的每台起重机都是有效的潜在障碍物。 

这避免了重复检查高度条件，并将过滤变成迭代顺序的结构。 

### 2.维护一套已经加工好的起重机

 我们保留了一份严格高于当前起重机的清单。 这些是唯一可以限制当前起重机梁长度的起重机。 

### 3. 对于每台起重机，计算最近的阻挡距离

 对于位置 (xi, yi) 处的当前起重机 i，我们计算到集合中已有的每个起重机 j 的欧氏距离平方：$$d_{ij}^2 = (x_i - x_j)^2 + (y_i - y_j)^2$$如果起重机 j 位于距离 d 处，则选择梁长度 L ≥ d 将使梁到达 j，但如果 j 足够高，则这是无效的。 因此 j 的安全界限是严格小于 d 的最大整数。 

我们不直接计算平方根，而是将其转换为整数运算：满足的最大有效整数 L$L^2 < d_{ij}^2$是：$$L_{ij} = \lfloor \sqrt{d_{ij}^2 - 1} \rfloor$$我们对所有较高的起重机取该值的最小值。 

### 4.应用塔高度约束

 即使没有起重机在几何上阻挡，梁长也不能超过塔高。 所以最终的答案是：$$L_i = \min(h_i, \text{minimum blocking limit})$$如果不存在更高的起重机，答案很简单$h_i$。 

### 为什么它有效

 对于任何起重机 i，每个有效的梁方向都对应于平面上的某个点。 当该点与另一台起重机的位置重合时，碰撞就会发生，而另一台起重机的塔架高度足以与梁的高度相交。 因此，每个此类起重机都独立地对允许半径施加严格的上限。 这些边界中最严格的限制是由欧几里德距离中最接近的有效阻挡者确定的，并且没有中间几何体改变这一事实，因为碰撞仅取决于端点是否到达禁止坐标，而不取决于沿线段发生的情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def isqrt(x):
    # floor(sqrt(x)) for x >= 0
    if x <= 0:
        return 0
    r = int(x ** 0.5)
    while (r + 1) * (r + 1) <= x:
        r += 1
    while r * r > x:
        r -= 1
    return r

n = int(input())
cranes = []
for _ in range(n):
    x, y, h = map(int, input().split())
    cranes.append((h, x, y))

cranes.sort(reverse=True)  # sort by height descending

processed = []
ans = [0] * n

for i in range(n):
    hi, xi, yi = cranes[i]

    best = hi  # cannot exceed height

    for hj, xj, yj in processed:
        dx = xi - xj
        dy = yi - yj
        d2 = dx * dx + dy * dy

        # largest L such that L^2 < d2
        limit = isqrt(d2 - 1)
        if limit < best:
            best = limit

    processed.append((hi, xi, yi))
    ans[i] = best

print(*ans)
```该实施依赖于按高度递减的顺序处理起重机，以便列表`processed`始终包含可以限制当前起重机的确切位置。 每个成对距离被评估一次，并且通过将答案初始化为立即强制执行高度约束`hi`。 

一个常见的实施陷阱是处理距离约束中的严格不平等。 使用`floor(sqrt(d2))`会错误地让梁准确地到达另一台起重机，这是不允许的。 减法`d2 - 1`正确地强制执行严格性而无需浮点比较。

 ## 边缘情况

 单个起重机案例表明，当不存在阻挡物时，算法正确地默认高度约束。 处理后的集合仍然为空，因此初始值保持不变。 

两台起重机的紧密配置和一台高度接近的起重机对严格的不平等处理进行了测试。 如果距离为 1，则较低起重机的有效半径必须为 0，因为任何正长度都会立即与较高塔相交。 

高度递减的单调线确保了排序逻辑的正确性。 每台起重机都会看到自己面前的所有更高的起重机，并且每一步都会累积更严格的界限，从而确认仅处理顺序就足以强制执行所有约束。
