 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$具有从单元格开始的单个标记（“片段”）的网格$(r, c)$。 游戏以离散的回合进行。 在每一轮中，计算机首先宣布一个数字$k$，那么我们就可以准确地执行$k$移动，每次移动都是该块到相邻未删除单元格的一步（或者如果不存在相邻未删除单元格，则实际上不执行任何操作）。 在我们移动之后，计算机恰好删除了一个仍然存在的单元格。 如果删除的单元格可能是我们移动后棋子的位置，我们可以立即宣布该轮胜利。 

关键的问题是我们不跟踪单个位置。 相反，我们会跟踪每轮之后棋子可能所在的一组所有单元格，因为我们不观察或控制确切的运动。 我们只关心在删除某个单元格时，是否仍然可以通过与所有先前删除一致的某些有效移动序列来到达该单元格。 

输出是最早的轮次索引，使得该轮次删除的单元格仍然位于该片段的可达集合内。 

约束条件允许$n, m \le 1000$，所以网格最多有$10^6$细胞，轮数也可达$10^6$。 这立即表明，任何尝试从头开始重新计算每轮可达性的方法都太慢，因为网格上的 BFS 或洪水填充已经$O(nm)$，并且重复这样做会超出限制几个数量级。 

一个更微妙的问题是，该片段不是标准的 BFS 代理，因为每轮允许的移动次数各不相同，并且可能非常大，因此在删除之间，该片段可以有效地遍历仍然可用单元的任何连接区域。 这将问题转化为删除与移动源的动态连接，我们只关心在某些时间限制下当前删除是否位于与起始单元相同的“可用区域”。 

一个幼稚的错误是将该块视为始终位于单个 BFS 边界单元中。 例如，如果我们假设它保持在距起点的最短路径距离处，我们就会错过它可以在删除之间的连接组件内任意徘徊。 另一种故障模式是忽略删除可能会断开网格的连接，从而创建随着时间的推移而演变的多个组件。 

一个特别微妙的边缘情况是，起始单元格被很晚删除的单元格包围，但较远的单元格很早就被隔离。 在标准的最短路径意义上，正确的答案可能会在该部分能够物理“到达”该区域之前出现，因为大$k$值允许在剩余组件内任意重新定位。 

## 方法

 强力解释将在每轮之后明确维护完整的可到达单元集。 每次删除后，我们都会在剩余网格上使用 BFS/DFS 重新计算从一开始就可以到达的单元格。 这正确地回答了删除的单元是否可到达，因为该块可以在许多步骤中在剩余单元的连接组件内自由移动。 

然而，这种简单的方法重新计算了大小的图遍历$O(nm)$对于每个最多$nm$轮，导致$O((nm)^2)$，这大约是$10^{12}$最坏情况下的操作。 这远远超出了任何可行的限度。 

关键的观察结果是，仅当删除断开部分网格时，可达性才会发生变化。 我们有效地逐个删除节点，并询问特定节点何时从剩余图中的一开始就断开连接。 这是一个经典的离线动态连接问题，但反过来：我们可以将删除向后处理为插入，而不是及时向前删除节点，从而逐渐重建网格。 

如果我们反转该过程，我们从一个空网格开始，并以与删除相反的顺序重新添加单元格。 在每一步中，我们都会维护哪些单元格连接到起始位置$(r, c)$。 使用不相交的集合联合结构，每次添加一个单元格时，我们都会将其与其已经活跃的邻居联合起来。 当起始单元连接到与前向过程中的删除步骤相对应的新添加的单元时，我们可以确定仍然可以到达删除的最早时间。 

这将问题转化为动态增长网格上的并集查找连接查询，其中每个单元格被激活一次，并与最多四个邻居执行并集。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每次删除后重新计算 BFS |$O((nm)^2)$|$O(nm)$| 太慢了|
 | 逆向流程+DSU |$O(nm \alpha(nm))$|$O(nm)$| 已接受 |

 ## 算法演练

 我们以相反的顺序处理删除序列，将删除转变为激活。 

1. 按顺序读取网格尺寸、起始位置和删除列表。 每次删除对应于前向过程中一个单元消失的时间。 
2. 创建一个结构，为每个单元格存储其在删除时间线中的位置。 我们将每个单元格映射到它被删除的步骤。 
3. 在所有网格单元上初始化 DSU，但我们从所有单元均处于非活动状态开始。 我们还维护一个布尔网格，标记单元是否在相反的过程中被激活。 
4. 从最后一个删除项向后迭代到第一个删除项。 在步骤$t$，我们激活当时删除的单元格$t$在前进的过程中。 
5. 当一个单元被激活时，我们将它与其四个邻居连接（如果它们已经处于活动状态）。 这将逐步构建剩余（未来活跃）网格的连接组件。 
6. 每次激活后，检查起始单元是否处于活动状态。 如果是，并且它（在 DSU 中）连接到刚刚激活的包含当前已删除单元的结构，我们将更新答案。 最早的此类步骤对应于可以到达已删除单元的最早回合。 
7. 继续，直到所有单元都被反向激活。 最终的答案是最小的前向索引，其删除仍然位于与激活时刻的开始相同的连接组件中。 

### 为什么它有效

 在前进过程中的任何时刻，该片段都可以在未删除单元的连接组件内任意移动。 因此，当删除的单元位于与删除的单元的补图中的起始位置相同的连接组件中时，该单元是“可到达的”。 

反向处理按照可用性递增顺序精确构建这些组件。 DSU 保持了它代表当前可用小区之间的连接性的不变性。 当我们激活一个单元格时，我们会恢复前向“活动网格”中存在的所有边缘。 因此，DSU 中的连接性与相应时间的前向游戏中的可达性相匹配。 这种等价性确保起始组件包含单元的最早激活时刻精确对应于该单元仍可到达的最早前向删除步骤。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, m, r, c = map(int, input().split())
_ = input()

t = n * m

cells = []
for _ in range(t):
    k, i, j = map(int, input().split())
    cells.append((i - 1, j - 1))

# DSU
parent = list(range(n * m))
size = [1] * (n * m)

def find(x):
    while parent[x] != x:
        parent[x] = parent[parent[x]]
        x = parent[x]
    return x

def union(a, b):
    ra, rb = find(a), find(b)
    if ra == rb:
        return
    if size[ra] < size[rb]:
        ra, rb = rb, ra
    parent[rb] = ra
    size[ra] += size[rb]

active = [[False] * m for _ in range(n)]

def id(i, j):
    return i * m + j

start = id(r - 1, c - 1)

active[start // m][start % m] = True

ans = 0

dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

for idx in range(t - 1, -1, -1):
    x, y = cells[idx]
    active[x][y] = True
    cur = id(x, y)

    for dx, dy in dirs:
        nx, ny = x + dx, y + dy
        if 0 <= nx < n and 0 <= ny < m and active[nx][ny]:
            union(cur, id(nx, ny))

    if active[r - 1][c - 1]:
        if find(cur) == find(start):
            ans = idx + 1

print(ans)
```DSU 结构以相反的过程对当前活动单元之间的连接进行编码。 每次激活都会添加一个节点并将其连接到已经活动的邻居，从而保持组件结构的正确性。 

2D 坐标和 DSU 索引之间的映射确保恒定时间的邻居检查和联合操作。 答案被跟踪为最早的索引，其中激活的单元与起始单元属于同一组件。

 ## 边缘情况

 一个关键的边缘情况是起始单元很晚被删除，但在反向过程的早期被隔离。 DSU 仍然可以正确处理此问题，因为连接仅取决于当前活动的邻居，而不取决于未来的激活。 

另一种情况是电网提前断开，但随后又恢复连接。 该算法可以处理这个问题，因为反向激活只会添加边缘，而不会删除它们，从而保持单调正确性。 

最后的边缘情况是起始单元本身是最后一个激活的单元。 该算法仍然有效，因为只有在启动处于活动状态后才会检查连接性，确保不会报告过早的连接。
