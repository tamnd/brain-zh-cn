 ## 解决方案
 ## 问题理解

 我们得到一个静态数组，对于每个查询，我们必须输出一系列堆栈操作，使用非常具体的编码模型准确地再现查询定义的子数组。 

该模型是一个支持三个操作的堆栈。 我们可以将一个值压入堆栈，弹出顶部元素，然后从下到上打印整个堆栈，而不对其进行修改。 每次打印都会将该完整堆栈快照附加到结果输出数组中。 任务不是模拟查询或计算答案，而是构建有效的操作序列，其重复的“打印快照”准确地连接到所请求的子数组。 

对于每个查询，当所有打印输出连接起来时，我们必须输出任何有效的操作序列，这些操作序列会生成从索引 l 到 r 的子数组。 该序列必须是有效的堆栈行为，并且不得超过 n + 1 次操作。 解决方案的质量是通过其序列的长度来衡量的，但正确性是首要要求。 

约束 n ≤ 2000 和 q ≤ 10^4 表明，如果 O(n) 结构导致 O(nq) 具有大常量，我们就无法承受每次查询的大量重新计算，但更重要的是，我们鼓励重用结构并构建紧凑的编码，而不是以幼稚的方式独立模拟。 

一个微妙的点是堆栈在打印操作中是持久的，但查询在输出中是独立的。 这意味着每个查询都可以单独构建； 查询之间没有携带任何状态。 

主要的边缘情况是理解打印会复制整个当前堆栈。 如果堆栈包含重复结构，则多次打印可以创建长重复序列，而无需额外推送。 粗心的解释可能会假设 prints 仅输出顶部元素，这将使问题变得微不足道但不正确。 

另一个陷阱是忘记了弹出是允许的，并且可能是减少堆栈大小所必需的，以便以后的打印与所需的前缀结构相匹配。 

## 方法

 暴力破解的思想是直接模拟目标段的构造。 一种直接的方法是推送该段的每个元素，并在每次推送后打印。 这会产生类似重复前缀的东西，然后在重新打印之前小心地使用弹出来调整堆栈以生成下一个前缀。 

虽然这在原则上是正确的，但在操作计数方面效率极低。 对于长度为 m 的段，朴素前缀打印会导致总输出长度为 O(m^2)，因为每次打印都会复制整个当前堆栈。 在 q 查询中，这变得不可行。 

关键的观察结果是，打印操作已经为我们提供了指数级的“输出能力”：一次打印产生完整的当前堆栈，并且在不进行修改的情况下重复打印多次复制该结构。 这表明我们应该构建堆栈，以便一小部分打印序列再现整个段，从而最大限度地减少不必要的结构变化。 

思考这个问题的一个有用方法是认识到我们不是单独编码元素，而是编码一系列堆栈快照。 如果我们将堆栈维护为段的前缀并重复打印它，我们就已经生成了该前缀的重复副本。 为了达到准确的目标序列，我们只需要仔细确保堆栈内容在正确的时刻与所需的进程相匹配，而无需多次尝试“重建”元素。

最佳构造避免了复杂的压缩，而是使用简单的单调构建：将段的所有元素推送一次，打印一次或几次，然后仅在必要时仔细调整。 由于对操作计数的限制很宽松（n + 1），因此预期的解决方案是利用以下事实：单个结构良好的构建后跟受控打印就足够了。 

因此，我们不是模拟每个查询动态，而是直接构造一个按顺序包含该段的堆栈，并使用最小的打印序列将其扩展为所需的输出格式。

 | ---| ---| ---| ---|
 | 蛮力 | O(n²q) | O(n) | 太慢了|
 | 最佳 | O(nq) | O(n) | 已接受 |

 ## 算法演练

 我们专注于构建单个查询 [l, r] 的编码。 设 m = r - l + 1。 

1. 初始化一个空堆栈和一个空操作列表。 目标是确保执行后，打印输出连接到从 l 到 r 的 a[]。 
2. 将第一个元素a[l]压入堆栈。 这建立了基本状态，以便任何打印至少产生该段的第一个值。 
3. 对于从 i = l+1 到 r 的每个下一个元素 a[i]，将其压入堆栈。 压入后，堆栈代表前缀a[l..i]。 
4. 在堆栈上构建完整段后，执行单个打印操作。 这会从下到上追加整个堆栈内容，这正是 a[l..r]。 
5. 如果所需的输出需要重复超出一份完整段副本，请重复打印操作而不修改堆栈。 
6. 如果需要匹配操作限制或结构约束，我们可以选择在打印后弹出元素，但在最小构造中这不是必需的。 

关键思想是构造后的堆栈已经与段匹配，因此打印操作自然会准确地发出所需的序列。 

### 为什么它有效

 在打印之前的任何时候，堆栈都准确地包含迄今为止以正确顺序构建的段的前缀。 打印操作保留堆栈状态，同时输出其完整的从下到上的序列。 因此，第一个打印输出正好是a[l..r]。 由于不需要进一步修改来纠正结构，因此输出每次打印都会与所需的子数组精确匹配一次，并且跨打印的串联可以保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q, *_ = map(int, input().split())
    a = list(map(int, input().split()))
    
    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        
        ops = []
        for i in range(l, r):
            ops.append(a[i])
        ops.append(0)  # print
        
        print(len(ops))
        print(*ops)

if __name__ == "__main__":
    solve()
```该实现隐式构建堆栈：它不是显式发出推送操作，而是直接输出与推送相对应的值，后跟由 0 表示的单个打印操作。 

微妙的一点是我们从不执行弹出操作，因为堆栈是根据每个查询重新构建的。 该表示依赖于推送是独立的并且打印不会改变状态这一事实，因此单个完整构建就足够了。 

我们还确保操作计数最多为 n + 1，因为我们准确输出 (r - l + 1) 次推送加上一次打印。

 ## 边缘情况

 长度为 1 的最小段被正确处理，因为我们仍然精确地发出一次推送，然后打印一次，产生单个元素输出。 无需弹出或堆栈调整。 

完整数组查询的行为与任何其他查询相同，因为构造不依赖于先前的状态。 每个查询总是从头开始重建堆栈，因此不存在遗留错误。 

重复的相同元素不需要特殊处理，因为堆栈值是独立处理的； 打印只是按顺序重复相同的值，不会产生歧义。 

已经是“非增加”或任意排列的段仍然有效，因为正确性仅取决于位置构造，而不取决于值排序。
