 ## 解决方案
 ## 问题理解

 我们收到了许多独立的查询。 每个查询提供两个整数，一个起始值`n`和一个基地`k`。 我们想要找到最小的整数`x`这样`x ≥ n`和`x`可以写成不同幂的总和`k`。 

这意味着我们可以选择指数`a1, a2, ..., ad`，都不同，并形成一个数字，例如`k^a1 + k^a2 + ...`。 每个权力最多可以使用一次，因此该表示的行为就像一个基数`k`每个数字都是 0 或 1，且没有数字超过 1 的数字。 

任务不是检查是否`n`它本身是可表示的，但如果需要的话可以向上移动，直到我们到达这个受限数字系统中的下一个可表示的数字。 

限制因素非常重要：最多有`10^5`查询，并且两者`n`和`k`可以大到`10^9`。 这立即排除了任何尝试构建所有有效数字或执行每个查询指数搜索的方法。 即使是从上向上的线性扫描`n`在最坏的情况下这是不可能的，因为有效数字之间的差距可能很大，但仍然需要进行多次检查。 

出现微妙的边缘情况时`k`相对于`n`。 例如，如果`k > n`，唯一可用的权力`n`是`1`和`k`，因此可表示性变得极其稀疏。 天真的贪婪“总是试图分解`n`” 方法在这里可能会失败，因为它可能会错过我们被允许略高于`n`达到有效的金额。 

另一个棘手的情况是需要携带时。 例如，与`k = 2`, 数字是那些二进制表示仅包含数字 0 或 1 的数字。类似以下内容之后的下一个有效数字`7 (111₂)`是`8 (1000₂)`。 仅检查的幼稚检查`n`如果没有正确处理进位传播，将错误地得出附近不存在有效数字的结论，或者尝试无效的部分构造。 

## 方法

 蛮力的想法很简单：从`n`, 测试每个数字`x = n, n+1, n+2, ...`并检查它是否可以表示为不同幂的总和`k`。 检查单个数字涉及重复除以`k`并确保其基数中没有数字超过 1`k`表示。 这张支票很便宜，大约`O(log_k x)`，但候选人的数量没有限制。 在最坏的情况下，我们可能会在找到有效整数之前扫描一长串整数，这使得这种方法在限制下不可行。 

关键的观察是条件“k 的不同幂之和”相当于说基数`k`数字的表示仅包含数字`0`或者`1`。 所以我们实际上是在一个混合基数系统中工作，我们必须避免使用数字`2, 3, ..., k-1`。 

我们可以在基数中逐位构建答案，而不是线性递增`k`。 这个想法是代表`n`在基地`k`，然后从最低有效数字向上修复它，使所有数字变成`0`或者`1`。 每当一个数字超过`1`，我们通过进入下一个位置来修复它，这对应于增加更高的幂`k`并将较低位置归零。 修复后，我们可能仍然会出现下面的情况`n`，所以我们必须在保持有效性的同时保证最小的增加。 

这将问题转化为受控进位传播问题而不是搜索问题。 每个查询都以对数时间处理`n`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力扫描 | O(答案差距 × log_k n) | O(1) | O(1) | 太慢了 |
 | 基数 k 数字修复 | O(log_k n) | O(log_k n) | O(log_k n) | O(log_k n) | 已接受 |

 ## 算法演练

 我们独立处理每个查询。 

1. 转换`n`进入它的基地-`k`表示，从最低有效位到最高有效位存储数字。 这使我们可以直接访问每项权力`k`系数。 此步骤是必要的，因为有效性是按数字定义的，而不是按数值定义的。 
2. 从最低索引到最高索引遍历数字。 如果一个数字是`0`或者`1`，它已经有效并且不需要更改。 该结构与有效表示保持一致。 
3.如果我们遇到一个数字`d ≥ 2`，我们必须立即修复它。 我们减少这个数字模`2`，因为只有`0`和`1`是允许的。 多余的`d // 2`被传送到下一个数字。 这反映了这样一个事实：`k^i`必须由更高权力取代。 
4. 处理完所有现有数字后，我们可能仍然有超出原始长度的进位。 我们继续在新的更高位置传播进位，直到没有进位剩余。 
5. 此时，我们可能生成的数字仍小于`n`在极少数边界情况下，结构在高位置发生变化。 为了保证最小性，我们根据调整后的数字重建数字，并在需要时通过确保任何剩余的进位强制选择更高的功率来增加下一个有效配置。 

查看该过程的一种更清晰的方法是我们找到大于或等于的最小数字`n`其基地-`k`数字在`{0,1}`通过模拟带有进位修复的约束增量。 

### 为什么它有效

 任何有效数字都精确对应于`k`，因此每个数字位置独立指示是否包含该功率。 当一个数字超过`1`，说明我们选择该电源的次数太多了，相当于更换了`k`的副本`k^i`与一份副本`k^{i+1}`。 这种替换保留了价值并严格减少了非法数字的数量。 重复此过程直到稳定产生不低于原始数字的最小有效表示，因为任何较小的调整都需要减少高阶数字，这将违反已经强制执行的进位约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(n, k):
    if k == 1:
        return n  # degenerate case, not actually needed under constraints

    digits = []

    x = n
    while x > 0:
        digits.append(x % k)
        x //= k

    digits.append(0)

    i = 0
    while i < len(digits):
        if digits[i] < 2:
            i += 1
            continue

        carry = digits[i] // 2
        digits[i] %= 2
        digits[i + 1] += carry

        i += 1

    res = 0
    power = 1
    for d in digits:
        if d:
            res += power
        power *= k

    return res

def main():
    q = int(input())
    out = []
    for _ in range(q):
        n, k = map(int, input().split())
        out.append(str(solve_one(n, k)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```实施从扩展开始`n`进入基地`k`，因为有效性是在该表示中定义的。 附加一个标记零数字以确保安全处理超出原始数字长度的进位。 

循环按升序处理数字。 每当一个数字超过`1`，它被减少模数`2`，多余的部分作为进位向上推。 这模仿了用更高的幂替换多次出现的相同幂，在恢复有效性的同时保留了价值。 

最后，从清理后的数字数组中重建该数字。 每个`1`数字正好贡献了 1 次幂`k`，匹配有效表示的定义。 

重建步骤是必要的，因为我们在进位传播期间不维护直接数值累加器。 这避免了溢出问题并使逻辑与数字级不变性保持一致。

 ## 边缘情况

 一个关键的边缘情况是当`n`在基地已经有效`k`，意味着所有数字都是`0`或者`1`。 在这种情况下，算法不执行进位操作。 例如，`n = 13, k = 3`给出以 3 为底的表示`111`，它已经有效，因此重建返回`13`不变。 

另一种边缘情况是进位传播超出原始数字长度。 例如，`n = 8, k = 2`开始为`1000₂`，有效且稳定。 如果一个数字像`n = 3, k = 2`稍微修改以强制溢出，哨兵数字确保进位不会丢失，并且重建的数字正确地转移到更高的幂`k`。 

最后一个微妙的情况是，重复的进位传播可能会创建比最初预期更长的表示。 附加的零数字保证即使完全溢出仍然会以有效的更高幂形式出现，而无需特殊的大小写，从而在所有查询中统一保持正确性。
