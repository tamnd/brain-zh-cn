 ## 解决方案
 ## 问题理解

 我们正在从原点开始在无限网格上构建一条行走。 每次移动都是四个基本方向之一的一个单位：右、左、上或下，每个方向都有给定的可用计数。 最后的步行必须完全使用所有动作。 

有一个被称为“邪恶坐标”的单一禁格点。 这个要求比仅仅在其他地方结束更强烈：在行走过程中的任何时候，包括中间前缀，我们都不允许准确地落在该坐标上。 

任务是确定所有移动的顺序，以避免踏入该禁区，或者确定不存在这样的顺序。 

因此，输入结构是四个整数，描述我们在每个方向上可以采取的步数，加上两个描述禁止坐标的整数。 输出是有效的移动序列，通常作为字符串，或者声明这是不可能的。 

约束足够小，使得输出与移动次数成线性关系，但又足够大，任何模拟所有排列或强力排序的尝试都会立即失败。 总移动次数最多为 10^5，任何解决方案都必须在 O(n) 时间内构建序列。 

关键的困难在于，像“做所有权利，然后所有向上，然后所有向左，然后所有向下”这样的简单结构可能会意外地穿过禁止坐标。 发生这种情况是因为单调线段的中间前缀可以完全匹配邪恶点（如果它位于这些线段扫过的矩形内）。 

当禁止点恰好位于两个连续方向块形成的区域时，就会出现微妙的故障情况。 例如，如果我们先移动所有权利，然后移动所有向上，则任何 x ≤ R 且 y ≤ U 的点 (x, y) 都将在某个前缀处命中。 如果其中包含邪恶坐标，则即使最终端点良好，构造也是无效的。 

## 方法

 一个蛮力的想法是尝试四个方向块的所有排列，并检查是否有任何排序避免了禁止点。 每个排列都定义了一个单调路径形状，我们将模拟每个排列的行走。 

这在逻辑上是有效的，因为任何有效的解决方案都必须是多组移动的某种排序。 然而，只有4个！ = 24 种排列，因此即使检查它们也很便宜。 真正的问题是，每个排序仍然需要仔细验证路径是否与禁止坐标相交，更重要的是，问题的结构表明，只有一种非常特定的故障才会发生：在单个单调象限转换内触及禁止点。 所以蛮力是矫枉过正的并且隐藏了结构。 

关键的观察是，只有当我们在两个朝向禁止坐标移动的垂直方向之间切换时，路径才会变得“危险”。 对于相对于原点的每个象限，只有一对方向重要：右/上、右/下、左/上或左/下。 所有其他移动都是正交的，一旦我们离开其区域，就无法重新创建禁止坐标。 

因此，我们不需要全局搜索排列，而只需要确定相关象限中两个块的相对顺序。 之后可以安全地附加所有剩余的动作。 

这将问题简化为在一对方向块的两种排序之间进行选择，以便前缀路径永远不会恰好落在 (x, y) 上。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(n) | O(n) | 已接受但没有必要 |
 | 象限贪婪排序| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将移动次数表示为 R、L、U、D，将禁止坐标表示为 (x, y)。 

### 1. 识别禁点位于哪个象限

我们将目标方向分类为 (x, y)。 如果x非负，则需要向右移动才能到达它； 如果x为负数，则需要向左移动。 y 轴和垂直运动也类似。 这告诉我们哪两种移动类型可以潜在地“建立”禁止坐标。 

另外两种移动类型方向相反，不能参与构成禁止前缀。 

### 2. 只关注两个相关方向

 如果 x ≥ 0 且 y ≥ 0，则只有 R 和 U 对于潜在命中 (x, y) 很重要。 如果 x ≥ 0 且 y < 0，则危险对为 R 和 D。如果 x < 0 且 y ≥ 0，则为 L 和 U。如果 x < 0 且 y < 0，则为 L 和 D。 

所有剩余的移动都可以在不创建禁止坐标的情况下自由放置在之前或之后，因为它们至少在一个轴上远离它。 

### 3. 决定两个关键块的顺序

 我们现在考虑将一个块放在另一个块之前是否会导致前缀命中 (x, y)。 

以x≥0且y≥0的情况为例。如果我们先放置所有R移动，然后所有U移动，则R块的每个前缀都位于x轴上，并且切换到U后的每个前缀位于固定x=R处的垂直线上。 

如果 x ≤ R 且 y ≤ U，那么在过渡到 U 块的过程中，我们最终将精确地达到 (x, y)，这是禁止的。 

为了避免这种情况，我们交换顺序并将 U 放在 R 之前。在这种安排中，我们首先独立于 x 增加 y，因此永远不会以相同的方式形成匹配 (x, y) 的前缀。 

相同的逻辑对称地应用于其他象限：我们选择的顺序可以避免以在同一前缀步骤中触及禁止点的方式对齐两个坐标。 

### 4. 附加剩余的两个方向

 修复关键对的安全顺序后，我们将所有移动添加到相反的方向（在右/上象限情况下工作时为 L 或 D）。 这些移动只会减少坐标，并且不能重新引入禁止点，因为我们已经在构造性前缀阶段避免了匹配它。 

### 5.输出构造好的序列

 最终序列只是所选块的串联。 

### 为什么它有效

 不变量是，在任何前缀处，路径永远不会同时匹配在单调运动的同一阶段到达禁止点所需的两个坐标。 在此构造中，精确落在 (x, y) 上的唯一方法是两个轴在同一块排序中以同步方式累积。 通过选择解耦两个坐标累积的顺序，我们确保没有任何前缀与禁止坐标重合。 

由于所有其他移动都远离相关象限目标，因此它们不能在序列的后面重新引入丢失的对齐。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    R, L, U, D, x, y = map(int, input().split())
    
    res = []
    
    # Decide quadrant of dangerous point
    if x >= 0 and y >= 0:
        # dangerous pair: R and U
        if not (x <= R and y <= U):
            res.append("R" * R)
            res.append("U" * U)
        else:
            res.append("U" * U)
            res.append("R" * R)
        res.append("L" * L)
        res.append("D" * D)
    
    elif x >= 0 and y < 0:
        # dangerous pair: R and D
        y = -y
        if not (x <= R and y <= D):
            res.append("R" * R)
            res.append("D" * D)
        else:
            res.append("D" * D)
            res.append("R" * R)
        res.append("L" * L)
        res.append("U" * U)
    
    elif x < 0 and y >= 0:
        # dangerous pair: L and U
        x = -x
        if not (x <= L and y <= U):
            res.append("L" * L)
            res.append("U" * U)
        else:
            res.append("U" * U)
            res.append("L" * L)
        res.append("R" * R)
        res.append("D" * D)
    
    else:
        # dangerous pair: L and D
        x = -x
        y = -y
        if not (x <= L and y <= D):
            res.append("L" * L)
            res.append("D" * D)
        else:
            res.append("D" * D)
            res.append("L" * L)
        res.append("R" * R)
        res.append("U" * U)
    
    print("".join(res))

if __name__ == "__main__":
    solve()
```实施直接遵循象限分解。 每个分支隔离可以共同构造禁止坐标的两个方向，并确保它们的排序避免同时前缀对齐。 

一个微妙的点是，在检查每个象限情况内的可行性时，我们将负坐标转换为正比较。 这使得条件“如果我们按这个顺序组合这两个块，我们会击中（x，y）吗”在所有情况下保持一致。 

其余方向无条件附加，因为一旦避免了关键对齐，它们就无法帮助到达禁止点。 

## 工作示例

 ### 示例 1

 输入：

 R = 2、L = 0、U = 2、D = 0、禁止 = (1, 1)

 我们处于第一象限情况，因此 R 和 U 至关重要。 

| 步骤| 行动| 职位洞察|
 | ---| ---| ---|
 | 1 | 尝试 R 然后 U | R 前缀达到 (1,0)，然后 U 达到 (1,1) → 禁止 |
 | 2 | 切换到 U，然后切换到 R | U 构建 (0,1)，R 构建 (2,1)，从不命中 (1,1) |

 输出通过破坏 x 和 y 累加之间的同步来避免禁止点。 

### 示例 2

 输入：

 R = 3，L = 1，U = 2，D = 2，禁止 = (2, -1)

 这是右下象限的情况。 

| 步骤| 行动| 职位洞察|
 | ---| ---| ---|
 | 1 | 检查 R 然后 D | 将在对齐的前缀 | 处到达 (2,-1)
 | 2 | 切换至 D，然后切换至 R | D 首先垂直移动，因此 x 和 y 对齐是解耦的 |

 确定顺序后，L 和 U 会被安全地附加。 

这些例子表明，唯一的危险事件是两个关键方向的同时前缀对齐。

 | 测量 | 复杂性 | 说明|

 | 时间 | O(R + L + U + D) | 每个动作都打印一次 |

 该解决方案与移动次数呈线性关系，这是最佳的，因为每个移动都必须出现在输出中。 这完全符合最多 10^5 甚至 10^6 总操作的典型限制。

 ## 边缘情况

 当禁止坐标正好位于单调块将到达的边界上时，就会出现一种微妙的边缘情况。 例如，如果R = 5且U = 5，并且禁止点为(3, 3)，则对R然后U排序将不可避免地通过(3, 3)作为前缀。 在这种情况下，算法会检测对齐条件并将顺序切换为 U 然后 R，确保没有前缀同时匹配两个坐标。 在执行过程中，U 首先产生点 (0, k)，因此在该阶段 x 永远不会是 3，而当随后应用 R 时，y 固定为 5，因此永远不会达到 (3, 3)。 

另一种边缘情况是一个方向计数为零时。 如果U = 0或R = 0，则路径被迫位于一条直线上，并且只有当禁止点正好位于该线段上时才能命中。 该算法会处理此问题，因为可行性检查 x ≤ R 和 y ≤ U 自动失败或触发无害交换，但仍会产生有效的线性路径。 

最后的边缘情况是当禁止坐标与所有运动处于“不同象限”时，例如第一象限情况中的 x > R 或 y > U。 然后，该条件会阻止任何交换，并且可以安全地使用直接排序，因为该路径根本无法物理到达禁止点。
