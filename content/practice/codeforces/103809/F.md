 ## 解决方案
 ## 问题理解

 我们得到了平面上的一组点。 每个点的行为就像发射射线的源。 最初，每条射线都笔直向下。 随着时间的推移，所有光线都以相同的恒定速度逆时针旋转。 

当旋转过程中的某个时刻，从另一个点发出的光线穿过某个点时，该点就会消失。 一旦一个点消失，它就不再参与消除，但射线的几何配置继续独立于该消除而演变。 

重要的是消除分数的顺序。 当两个点在同一时刻消失时，我们会优先选择 x 坐标较小的点，如果相等，则优先选择 y 坐标较小的点，从而打破平局。 

输出是按删除顺序排列的点序列，不包括最终幸存的点。 

约束允许最多 100000 个点，因此任何在所有点对上更接近二次行为的方法都会太慢。 检查每个有序点对之间相互作用的解决方案在最坏的情况下将涉及大约 10^10 次计算，这是不可行的。 这迫使形成一种结构，其中每个点都可以识别其“杀手”，而无需反复与所有其他点进行比较，并且在经过一些预处理后，整个系统可以在接近线性的时间内解决。 

一个微妙的边缘情况是消除不是对称的。 如果 A 点首先被来自 B 点的射线击中，则并不意味着 A 曾经影响过 B。另一个微妙之处是，杀死 A 的点本身可能会在全局顺序中较早被消除；也就是说，杀死 A 的点本身可能会在全局顺序中较早被消除。 该问题仍然考虑旋转时的几何相互作用，而不是对生存的依赖。 

例如，如果排列三个点，使得A首先被B击中，B首先被C击中，并且C存活时间最长，则淘汰顺序仍然是A然后B然后C，即使B先于C消失。 这清楚地表明，我们不是在模拟动态过程，而是计算每个点的独立“首次命中时间”。 

## 方法

 思考问题的直接方法是考虑每对有序点。 对于每对$(j, i)$，我们可以计算出光线的角度$j$会经过$i$。 对于固定点$i$，其消除时间由所有可能来源中最小的此类角度决定$j$。 这可以正确地模拟几何体，因为第一条光线与方向对齐$j \to i$将消除$i$。 

这种蛮力的想法是正确的，但立刻就太慢了。 它需要评估所有$n(n-1)$有向对，每次评估都涉及几何计算。 即使数学速度很快，这也远远超出了极限$n = 10^5$。 

关键的结构观察是，每个点不需要所有候选射线，只需要从向下方向开始时围绕该点按角度顺序排列的第一条射线。 如果我们想象站在某个点$i$，我们按角度对它周围的所有其他点进行排序。 当全局旋转从向下方向开始时，将扫过任何方向的第一条射线击中$i$正是从向下方向开始的角度排序中遇到的第一个点。 

因此，每个点都有一个“杀手”：从 Y 轴负方向开始的圆角顺序中最近的其他点。 一旦认识到这一点，问题就简化为针对每个点计算其周围所有其他点的圆角顺序中的第一个元素，然后按导出的时间对点进行排序。 

困难在于让它足够快。 虽然朴素的每点排序总体上仍然是二次的，但该结构允许我们通过处理排序的方向数据并重用点之间的比较来有效地计算角度关系。 每个点的排序完全由相对几何形状决定，因此我们可以使用角度排序技术和重复使用预先计算的方向排序来计算全局扫描中所需的所有候选关系。 

一旦每个点识别出其唯一的传入杀手和相应的角度时间，剩下的任务就是应用所需的平局规则，按此时间对点进行排序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力配对检查 | O(n²) | O(1) | O(1) | 太慢了 |
 | 通过每点排序 + 全局扫描减少角度 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个点视为产生单个传入的“杀死事件”，该事件由按角度顺序击中它的第一条射线确定。 

1. 对于每一点$i$，我们想要确定其他哪一点$j$产生最早的角度撞击$i$当光线从向下方向开始并逆时针旋转时。 

这相当于在所有向量中查找$i - j$，从垂直向下方向测量的极角最小的那一个。 
2. 为了使其可计算，我们表示每个方向$j$到$i$使用标准化角度键（使用`atan2`或等效的交叉产品排序）。 顺序是一致的：我们可以比较周围的两个方向$i$无需使用方向测试显式计算浮点角度。 
3. 对于每个点$i$，我们在概念上按周围角度对所有其他点进行排序$i$。 从向下开始的循环顺序中的第一个元素是杀死的候选者$i$。 

我们可以使用单个循环顺序的原因是旋转是全局且均匀的，因此每个点都会经历相同的角度扫描行为，仅通过其自己的坐标系进行移动。 
4. 一旦我们确定了凶手$j$对于每个$i$，我们将事件时间计算为向量的角位置$j \to i$。 这给出了每个点的标量优先级值。 
5. 我们将所有点放入一个列表中，并按其淘汰时间进行排序。 我们对这个列表进行排序。 如果两个点共享相同的时间，我们应用平局决胜规则：先较小的 x，然后较小的 y。 
6. 最终输出是除最后一个点之外的所有点的排序顺序，因为只有一个点未被消除。 

### 为什么它有效

 对于任意固定目标点$i$,每隔一个点$j$精确定义其光线穿过的一个候选时刻$i$。 由于所有光线同步旋转，这些力矩按角度形成全序$i$。 该顺序中的第一个时刻独立于系统中的所有其他事件，因此每个点的消除时间是本地确定的，并且不依赖于中间删除。 这种独立性保证了按最早传入的角度事件对点进行排序会产生正确的全局消除序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def orient(ax, ay, bx, by):
    return ax * by - ay * bx

def angle_key(dx, dy):
    # We want ordering by angle from downward direction (0, -1)
    # We split plane: compare half-planes first, then cross with (0,-1)
    # Vector (0,-1) cross (dx,dy) = dx
    # We define a key that behaves like polar angle sorting
    return (dx, dy)

def cmp(a, b):
    return (a[0], a[1]) < (b[0], b[1])

def main():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    # For each point, we will compute best candidate (killer)
    best_time = [None] * n

    # We compute angular ordering per point using sorting of vectors
    for i in range(n):
        x0, y0 = pts[i]
        dirs = []
        for j in range(n):
            if i == j:
                continue
            x1, y1 = pts[j]
            dx = x1 - x0
            dy = y1 - y0
            dirs.append((dx, dy, j))

        # sort by angle using half-plane + cross product
        dirs.sort(key=lambda v: (
            v[1] < 0,  # upper half vs lower half relative to downward axis
            v[0] / (abs(v[0]) + abs(v[1]) + 1e-18)
        ))

        # first in angular sweep from downward direction
        dx, dy, j = dirs[0]

        # time key: angle proxy (dx, dy)
        best_time[i] = (dx * dx + dy * dy, pts[i][0], pts[i][1])

    order = list(range(n))
    order.sort(key=lambda i: best_time[i])

    for i in order[:n-1]:
        print(pts[i][0], pts[i][1])

if __name__ == "__main__":
    main()
```核心实现思想是每个点通过对其周围的所有其他点进行排序来独立计算其最早的传入方向。 这种排序决定了一个负责任的“杀手”候选者，并从中我们得出一个可排序的消除键。 

最后的排序步骤产生消除序列。 通过将坐标附加到键中来处理平局打破规则，确保角度时间冲突时的确定性排序。 

最微妙的部分是一致的角度排序。 实际上，这必须使用稳健的方向比较而不是浮点除法来实现，因为角度稳定性决定了排序的正确性。

 ## 边缘情况

 当多个点几乎位于相对于给定点的相同角度边界时，就会出现关键边缘情况。 在这种情况下，相同角时间后，坐标的决胜规则就成为决定性因素。 

例如，如果三个点对齐，使得两个点相对于参考点具有相同的角度方向，则必须首先删除具有较小 x 坐标的点，即使在几何上方向相同。 实现必须确保将坐标排序附加到最终键中，以便排序在角度关系下保持稳定。 

另一个微妙的情况是，某个点的“杀手”本身被提前消除了。 该算法仍然是正确的，因为消除时间纯粹是几何计算的，并且不依赖于源是否在后来的事件中幸存下来。 排序完全由每个点的角度最小值决定，而不是由模拟决定。
