 ## 问题理解

 给定一个整数数组，我们可以使用一种非常具体的操作来修改它：在一次操作中，我们选择一个非负整数 m 和一个正增量 k，然后将 k 添加到长度为 m+1 的前缀或长度为 m+1 的后缀。 换句话说，每个操作都会引发一个连续的段，该段锚定在数组的左端或右端。 

目标是将数组转换为“VonitA”序列。 这意味着最终的数组必须是两种对称方式之一的单峰。 要么先不减少，后不增加，形成峰，要么先不增加，后不减少，形成谷。 关键是只有一个转折点，并且单调方向最多改变一次。 

约束结构足够严格，任何解决方案对于每个测试用例都必须是线性的，因为所有测试用例的总长度最多为 2×10^5。 这排除了任何二次或涉及重复模拟操作的情况。 每个元素只能在 O(1) 或 O(log n) 聚合时间内概念性地被触及。 

这个问题的一个微妙之处是操作只会增加价值。 这种不对称性很重要：我们不能通过减少元素来直接修复违规，因此任何纠正都必须来自提升数组的其他部分。 这使得“逐一修复反转”的天真贪婪模拟变得不正确，因为对一个区域的修复可能会破坏其他地方的结构。 

往往会打破朴素推理的边缘情况包括已经单调的数组、严格上下交替的数组，以及通过选择相反方向（峰与谷）实现最佳形状的数组。 例如，像这样的数组`[5, 1, 4, 2, 3]`根据我们的目标结构，可以以不同的方式使其有效，并且贪婪的单向检查可能会错过最佳答案。 

## 方法

 直接暴力策略将通过选择峰值位置来尝试每种可能的目标形状，然后模拟操作以强制两侧的单调性。 对于固定峰值，我们将重复定位单调条件失败的违规位置，并应用覆盖最小必要前缀或后缀段的操作。 这在概念上是正确的，但每个操作都可以线性地重新计算违规，导致每个测试用例的最坏情况为 O(n^2)，这对于给定的约束来说太慢了。 

关键的观察是我们实际上不需要显式地模拟操作。 由于每个操作仅向前缀或后缀添加一个常量，因此其效果是将整个单调块向上“移动”，而不改变其内部顺序。 真正重要的是哪里违反了方向约束。 

我们不跟踪值，而是跟踪相邻元素之间的比较方向。 每对要么满足所需的单调条件，要么违反它。 有效的 VonitA 形状对应于允许方向模式中的单个开关。 因此，问题简化为计算在两个允许的方向中的每一个方向下阵列被迫进入多少个连续的单调段。 

锚定在一端的每个操作都可以以受控方式精确消除一个违规“过渡区域”，因此操作的最小数量与必须解决的方向不一致的数量成正比。 评估两种可能的方向并取最小值可以在线性时间内给出答案。

 ## 算法演练

 我们独立处理每个测试用例并计算两种有效形状的答案：峰形（非减少然后非增加）和谷形（非增加然后非减少）。 

### 1. 编码局部方向约束

 我们将数组转换为相邻元素之间的比较序列。 对于每个 i，我们根据所需的全局模式确定 ai ≤ ai+1 或 ai ≥ ai+1 是否成立。 不匹配表示违反目标单调结构。 

关键思想是我们从不需要精确的值，只需要相邻关系是否与所选形状一致。 

### 2. 评估“峰值”方向

 我们假设数组首先是非减的，然后是非增的。 我们从左到右扫描并计算单调要求必须切换或以强制结构修正的方式被违反的次数。 

每个最大一致段都有助于数组的分段。 超过 1 的段数表示需要进行多少次校正。 

### 3.评估“山谷”方向

 我们重复相同的推理，但不等式相反：首先是非增，然后是非减。 这是对称的，因此应用相同的分段逻辑。 

### 4. 将段转换为操作

 每个操作最多可以消除一个结构不一致块，因为它只修改一个前缀或后缀，不能同时修复独立的内部违规。 因此，所需的操作次数是单调段的数量减一。 

我们计算两个方向的值并取最小值。 

### 为什么它有效

不变量是每个有效的 VonitA 序列最多对应于单调方向的一个变化。 与此结构的任何偏差都会在比较图中表现为附加的单调段。 由于前缀和后缀操作不能一次对元素重新排序或修复多个独立的转换，因此每个额外的段对应于至少一个所需的操作。 相反，每个操作都可以通过提升前缀或后缀来准确合并段之间的一个边界，从而使边界更紧密。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_segments(arr, inc_first=True):
    n = len(arr)
    segments = 1

    def ok(a, b):
        return a <= b if inc_first else a >= b

    for i in range(n - 1):
        if not ok(arr[i], arr[i + 1]):
            segments += 1
    return segments

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        # peak: increasing then decreasing
        inc = count_segments(a, True)

        # valley: decreasing then increasing
        dec = count_segments(a, False)

        # each extra segment needs one operation
        ans = min(inc - 1, dec - 1)
        print(max(ans, 0))

if __name__ == "__main__":
    solve()
```该实现将推理压缩为每个方向的单次扫描。 功能`count_segments`测量有多少连续块遵循选定的单调方向。 最终答案是通过减一得出的，因为单个段不需要任何操作，而每个附加段意味着至少一个结构修复。 

需要注意数组已经有效的边界； 在这种情况下，段计数为 1，答案正确地变为 0。 

## 工作示例

 考虑数组`[3, 2, 1, 4, 5]`。 

对于峰值方向，比较是向下、向下，然后向上、向上。 这会产生两个单调块：`[3,2,1]`和`[1,4,5]`在方向一致性方面。 段数为2，因此需要1次操作。

 ## 边缘情况

 对于像这样的严格单调数组`[1, 2, 3, 4]`，段计数器在递增方向上恰好产生一个段。 由于不需要结构修正，该算法正确返回零。 

对于像这样的反向单调数组`[5, 4, 3, 2]`, the decreasing-first orientation also yields a single segment. Even though the other orientation would produce multiple violations, taking the minimum ensures correctness.

 对于交替数组，例如`[1, 3, 2, 4, 3]`，每个局部方向翻转都会增加段数。 该算法将每次翻转解释为必须通过最多一个前缀或后缀操作来解析的边界，并且最终答案与此类边界的数量完全对应。
