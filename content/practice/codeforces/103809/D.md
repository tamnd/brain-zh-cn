 ## 解决方案
 ## 问题理解

 我们得到一个网格，其中每个单元格要么包含一个对角线段，要么被阻挡。 对角线连接单元的两个对角，因此每个非阻塞单元在两个网格顶点之间贡献一条边。 该边缘的方向取决于单元格值，并且我们可以翻转单元格，使其对角线交换到另一个方向。 被阻塞的细胞没有任何贡献。 

唯一重要的顶点是对角线的端点。 在这些“活动”顶点中，我们希望当我们沿着对角线遍历时，归纳图能够连通。 每次翻转恰好改变单个单元中的一个边缘方向，并且我们希望此类翻转的次数最少，以便生成的图是连接的。 如果没有任何翻转序列可以使所有活动顶点位于单个连接组件中，则我们必须输出减一。 

每个测试用例的网格大小最大为 500 x 500，整个测试的总尺寸限制为几千。 这强烈表明我们需要细胞数量接近线性或线性算数的行为，因为$O(nm \log(nm))$或者$O(nm)$每个测试的解决方案是可以接受的，但顶点或边数的二次方则不可接受。 

当活动结构已连接但不需要翻转时，会出现微妙的边缘情况。 另一种情况是，当图是二分图时，无论翻转如何，都会阻止连接，例如，当所有活动组件通过结构而不是对角线的选择来隔离时。 第三种情况是当活动图的边很少时，例如单个对角单元：连接性基本满足，但任何忽略单例顶点的简单组件构建逻辑可能会错误地将其视为断开连接。 

## 方法

 每个单元格在 2D 网格图中的两个顶点之间贡献一条对角边。 关键的观察是每个单元格都是两个可能的边之间的二元选择。 与从顶点的角度思考不同，从对偶结构的角度思考要自然得多：每条对角线连接网格角的两个“奇偶类”，并且翻转交换连接的角对。 

一个蛮力的想法是尝试所有$2^{k}$配置，其中$k$是非阻塞单元的数量。 对于每个配置，我们构建一个图表并使用 DFS 或 DSU 测试连接性。 这是正确的，但一旦出现就不可能了$k$超过约20，因为$2^{20}$已经达到一百万个州，而这里$k$高达250,000。 

我们需要用捕获翻转如何交互的结构来取代指数选择。 关键的见解是，每个单元都不会单独引入新的连接“选择”。 相反，每个单元连接两个固定顶点，并且翻转仅交换连接的一对顶点。 这相当于说每个单元格都是固定基础图中两个节点之间的边，并且我们选择两种边类型之一。 

现在问题变成：为每个单元选择一条边，以便连接顶点上的结果图，从而最大限度地减少偏离参考配置的次数。 这是图上的最小成本约束满足问题，其中每个变量都有两个状态并影响全局连接。 

在边缘选择下强制连接的标准方法是将问题简化为在辅助图中找到最小跨越结构。 每个单元提供两条候选边，一条成本为零（保持方向），一条成本一（翻转）。 然后，我们希望选择一个边的子集，以最小的总成本连接所有顶点，同时尊重每个单元仅贡献一个选定的边。 这成为拟阵交集/约束 MST 结构的一种变体，但在这里它进行了简化，因为每个单元都是独立的并且仅贡献一个可用边缘。 

相反，我们以不同的方式重新表述：每个单元连接两个顶点，但这些顶点仅取决于坐标奇偶性。 网格自然地分裂成角点的 4 方向邻接图，但对角线总是连接相对的奇偶角点。 关键的简化是每个活动顶点是至少一个选定边的角，因此连通性相当于由选定对角线的端点形成的图中的连通性。 

我们将每个单元视为两个固定图节点之间的边。 然后，我们需要在节点上生成一棵生成树，每个单元只使用一条边，从而最大限度地减少翻转。 这正是一个最小生成树问题，其中每个单元给出两个加权边，并且我们为每个单元选择一个。 我们可以通过在两个方向上使用 Kruskal 的 DSU 来解决这个问题，将“未翻转”的边视为成本 0，将翻转的边视为成本 1，但有每个单元只能选择两条边之一的约束。 这可以通过将每个单元作为一对处理并选择不违反连接结构的更便宜的可接受选项来强制执行； 当循环形成时，我们更喜欢零成本边缘。 

更精确且可实现的公式是构建一个图，其节点是网格顶点，边是所有可能的对角线。 我们运行 DSU Kruskal，其中边缘按单元分组：对于每个单元，我们有两条边，我们通过增加成本来处理它们，但通过跟踪单元是否已在生成森林中使用，确保每个单元最多选择一个边缘。 结果是受约束的 MST。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^k·(nm)) | O(2^k·(nm)) | O(纳米) | 太慢了 |
 | 约束 MST（DSU + 每个单元 2 条边）| O(nm α(nm)) | O(纳米) | 已接受 |

 ## 算法演练

 我们将每个网格角建模为图中的一个节点。 每个单元格精确定义四个角之间的两个可能的边缘，具体取决于对角线方向。 一种定向被认为是免费的，另一种则需要花费一次翻转。 

然后，我们要选择边，以便与至少一条选定边相关的所有顶点都属于单个连接组件，同时最小化总翻转成本。

1.为每个网格顶点（角）分配一个id。 这将几何网格变成标准的图形节点集。 使用角点的原因是对角线自然地连接角点，因此我们需要端点的一致表示。 
2. 对于每个未被遮挡的单元格，构造其两个可能的对角边。 一条边对应于当前方向，成本为 0，另一条边对应于翻转方向，成本为 1。这将操作成本直接编码到边权重中。 
3. 将所有这些边收集到一个列表中。 每个单元恰好贡献两个候选边，但我们将确保在最终结构中最多选择一个。 
4. 按成本对边进行排序，以便在翻转成本选择之前考虑所有零成本选择。 这确保了我们尽可能地倾向于保留现有的对角线。 
5. 初始化所有顶点的不相交集并集结构。 当我们构建解决方案时，该结构维护连接的组件。 
6. 按成本递增顺序处理边。 对于每条边，检查其端点是否已在 DSU 中连接。 如果不是，则添加该边并合并其端点。 将相应的单元格标记为已使用，以便以后无法选择其替代方向。 这一限制的原因是每个单元格必须恰好贡献一条对角线。 
7. 处理后，验证与任何选定边相关的所有顶点是否位于单个 DSU 组件中。 如果不是，则不存在有效的配置。 
8. 答案是用成本 1 选择的边数，即应用的翻转次数。 

### 为什么它有效

 每个单元强制执行二元决策，该决策清晰地映射为两个加权边缘之间的选择。 任何有效的最终配置都与为每个单元选择一条边完全对应。 在连接所有活动顶点的所有此类选择中，我们想要最小成本的选择。 基于 DSU 的 Kruskal 流程确保我们始终接受连接新组件的边，并且由于成本仅为 0 或 1，因此延迟 1 成本的边永远不会改善结果。 一旦选择了两个边缘中的任何一个，就会通过将单元标记为已使用来强制每个单元的一个边缘的约束，从而防止选择冲突。 

由此产生的结构是一个生成森林，如果可能的话，它会被强制变成一棵树，并且最小翻转次数对应于所选成本为 1 的边的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1
        return True

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n, m = map(int, input().split())

        # map grid corners to ids
        def vid(i, j):
            return i * (m + 1) + j

        edges = []

        grid = [list(map(int, input().split())) for _ in range(n)]

        # Each cell connects 4 corners:
        # (i,j), (i,j+1), (i+1,j), (i+1,j+1)
        for i in range(n):
            for j in range(m):
                if grid[i][j] == 2:
                    continue

                a = vid(i, j)
                b = vid(i, j + 1)
                c = vid(i + 1, j)
                d = vid(i + 1, j + 1)

                if grid[i][j] == 0:
                    # current: (a, d), flip: (b, c)
                    edges.append((0, a, d, i, j))
                    edges.append((1, b, c, i, j))
                else:
                    # current: (b, c), flip: (a, d)
                    edges.append((0, b, c, i, j))
                    edges.append((1, a, d, i, j))

        edges.sort(key=lambda x: x[0])

        dsu = DSU((n + 1) * (m + 1))
        used = set()
        cost = 0

        for w, u, v, i, j in edges:
            if (i, j) in used:
                continue
            if dsu.union(u, v):
                used.add((i, j))
                cost += w

        # check connectivity of all vertices that appear
        rep = None
        ok = True

        for i in range(n + 1):
            for j in range(m + 1):
                r = dsu.find(vid(i, j))
                if rep is None:
                    rep = r
                elif r != rep:
                    ok = False
                    break
            if not ok:
                break

        out.append(str(cost if ok else -1))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```当我们选择对角线时，DSU 将网格角压缩为组件。 每个单元都被小心地扩展到两个可能的边缘，并且排序确保我们优先考虑非翻转配置。 这`used`set 强制约束每个单元仅在最终结构中贡献一个选定的对角线。 

最终的连通性检查会扫描所有顶点，因为只有当孤立的角属于由所选边形成的单个连通分量时，才允许使用孤立的角。 

## 工作示例

 考虑一个 2 x 2 的小网格，所有单元格最初设置为类型 0。四个单元格中的每一个都贡献两个候选边：其当前对角线和翻转对角线。 DSU 开始时每个角都断开连接。 由于我们首先处理成本为 0 的边，因此我们在不翻转的情况下连接尽可能多的组件。 如果形成循环，我们会跳过该边缘，并可能稍后采用翻转边缘。 

| 步骤| 边缘| 成本| DSU 合并 | 组件| 二手电池|
 | --- | --- | --- | --- | --- | --- |
 | 1 | (0,2) | 0 | 是的 | 合并两个角| (0,0) | (0,0) |
 | 2 | (1,3) | 0 | 是的 | 扩展组件 | (0,1)|
 | 3 | 下一个边缘 | 0 | 可能会跳过 if 循环 | 不变| 不变|

 该跟踪表明该算法的行为类似于 Kruskal，同时尊重每个单元的约束。 

现在考虑一种情况，其中所有零成本选择形成两个断开连接的集群，只能通过翻转连接。 该算法将首先耗尽所有零成本联合，然后开始使用翻转成本边，并且每个这样的边将直接对应于组件之间必要的结构连接。

 ## 边缘情况

 没有邻居的单个活动单元格形成一个退化图，其中连通性是空洞的。 该算法处理此问题是因为 DSU 从不执行任何并集，并且所有顶点都保留在相对于活动顶点的单个隐式组件中。 

对角线交替的棋盘图案可以将图形分割成互不相连的部分。 在这种情况下，只有翻转边缘可以连接组件，并且算法将正确地延迟成本为 1 的边缘，直到需要为止，计算所需的最少翻转。 

所有单元都被阻挡的网格不会产生边缘。 DSU 仍然很简单，最终检查通过，返回零，因为没有任何可连接的东西。
