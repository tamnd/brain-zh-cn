 ## 解决方案
 ## 问题理解

 我们给出了从今天开始的时间表，其中今天固定为星期一。 然后日历一天一天地进行，我们想要计算接下来有多少个`m`天是工作日。 

有两种类型的限制定义非工作日。 首先，周末遵循固定的周期模式：每周六和周日都是假期。 其次，有一个额外假期的列表，以绝对日指数的形式给出，从今天作为第一天开始。这些额外假期可能会重复，并且可能与周末重叠。 如果某一天是周末或出现在额外假期列表中，则为非工作日。 否则就是工作日。 

任务是计算从第 1 天到第 1 天的间隔中有多少个工作日`m`， 在哪里`m`可以非常大，最多 10^18。 额外假期数`n`很小，最多 300，但是那些天的值也很大，最多 10^18。 

约束的关键含义是我们无法在几天内进行迭代。 即使扫描最多 10^18 也是不可能的。 任何解决方案都必须依赖于周末的周期结构和范围内的直接算术推理。 额外假期列表的规模较小，表明我们可以在处理常规周末模式后单独处理它们。 

一个天真的错误是将其视为直接模拟问题并迭代所有`m`天，同时检查每一天是周末还是一组假期。 这会立即失败，当`m`很大。 

另一个微妙的陷阱是忘记周末仅取决于位置模 7，但映射取决于第一天是星期一这一事实。 粗心的模对齐错误会导致所有周末都错误地移动。 

最后，假期列表中的重复项不会改变任何内容，但如果忘记删除重复项或错误地计算了两次重叠，则答案将是错误的。 

## 方法

 蛮力方法很简单：每天从 1 到`m`，使用以下方法计算是星期六还是星期日`(day % 7)`并检查是否出现在额外假期集中。 如果存储在哈希集中，每次检查的复杂度为 O(1)，因此总复杂度为 O(m)。 这是正确的但不可行，因为`m`可以大到10^18，这远远超出了任何计算极限。 

关键的观察结果是，周末是周期性的，周期为 7，因此它们在较长时间间隔内的贡献可以按块计算，而不是按天计算。 我们不进行迭代，而是计算其中包含多少个完整周`m`，乘以 2，然后明确处理剩余天数。 

一旦考虑到周末，唯一剩下的复杂因素就是额外的假期清单。 自从`n ≤ 300`，我们可以独立处理每个假期。 每个假期`a_i`那在于`[1, m]`，我们检查是否已经是周末。 如果没有，则会从总数中减去一个工作日。 

因此策略变为：从总天数开始`m`，减去通过算术计算得出的周末天数，然后减去工作日中独特的额外假期的数量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(米) | O(n) | 太慢了|
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算间隔内包含多少个完整周`[1, m]`通过划分`m`到 7。每个整周正好有 2 个周末天。 这给出了周末天数的基本计数：`weeks * 2`。 
2. 确定整周后还剩下多少天。 这些是第一个`m % 7`上一周后的天数。 由于第一天是星期一，这些剩余的日子总是对应于每周周期的相同前缀，因此我们可以明确检查这些天是星期六还是星期日，并添加到周末计数中。 
3. 计算周末的总天数`[1, m]`通过添加全周周末和剩余一周周末。 
4. 初始化工作日数为`m - weekend_count`，因为最初假定每个非周末都正常工作。 
5. 将所有额外的假期插入一个集合中，但只保留以内的假期`[1, m]`。 这避免了处理不相关的大值。 
6. 对于每个独特的额外假期，检查它是否是周末。 如果不是，则该天被错误地计为工作日，必须从答案中减去。 如果已经是周末，则没有任何影响，因为周末已经被排除在外。 
7. 输出最终调整后的工作日数。 

### 为什么它有效

 正确性取决于将日历分为两个独立的结构：周末的固定周期结构和一组稀疏的不规则排除。 周末计算是准确的，因为从周一开始，每个 7 天的段都具有相同的结构。 额外假期的调整步骤是有效的，因为每个这样的日子都会独立地将工作日切换为非工作日，除非它已被周期性结构排除。 由于重叠只发生在两个独立的排除系统之间，并且不会造成超出简单集合成员资格之外的重复计算，因此仅减去非周末假期即可得出确切的工作日数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_weekend(day: int) -> bool:
    # Day 1 is Monday:
    # 6 -> Saturday, 0 -> Sunday when (day-1) % 7 mapping
    r = (day - 1) % 7
    return r == 5 or r == 6

def solve():
    out = []
    while True:
        line = input().strip()
        if not line:
            continue
        n, m = map(int, line.split())
        if n == 0 and m == 0:
            break

        arr = list(map(int, input().split()))
        
        # weekends
        full_weeks = m // 7
        rem = m % 7

        weekend = full_weeks * 2
        for i in range(rem):
            if is_weekend(full_weeks * 7 + i + 1):
                weekend += 1

        working = m - weekend

        # extra holidays
        seen = set()
        for a in arr:
            if 1 <= a <= m:
                seen.add(a)

        for d in seen:
            if not is_weekend(d):
                working -= 1

        out.append(str(working))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先通过分隔整周和剩余块来将日历转换为模块化表示。 这`is_weekend`函数对工作日对齐进行编码，其中第一天是星期一，确保星期六和星期日的正确分类。 

周末计数步骤仔细处理剩余部分，而不是假设它总是孤立地从星期一开始。 表达式`full_weeks * 7 + i + 1`重建剩余部分的绝对日索引，以便工作日分类保持一致。 

套装`seen`当输入列表包含重复的假期时，可以防止重复减法。 仅考虑每个假日是否位于查询间隔内。 

减法步骤仅适用于非周末假期，因为周末已经被批量去除，否则会出现双重减法。

 ### 示例 1

 输入：```
n=4, m=8
holidays = [1,2,3,4]
```我们计算整周：`8 // 7 = 1`, 余数`1`。 

| 步骤| 价值|
 | --- | --- |
 | 整整几周 | 1 |
 | 整周的周末| 2 |
 | 剩余天数 | 1 |
 | 周末总计 | 2 |
 | 基本工作日 | 8 - 2 = 6 |
 | 假期范围 | {1,2,3,4} |

 现在我们检查每个假期。 只有第 1 天是星期一而不是周末，因此减去 1。最终答案是 5。 

这表明重叠的工作日假期仅影响非周末。

 第 3 天不是周末，因此我们减去 1。 

这证明了对超出范围的假期的正确过滤。

 ## 边缘情况

 一个重要的边缘情况是当`m`小于 7。在这种情况下，没有完整的周，因此所有周末计算必须来自余数逻辑。 该算法自然地处理这个问题，因为`m // 7`变为零并且仅循环结束`rem`被使用。 

另一个极端情况是周末放假。 例如，如果第 6 天（星期六）被列为假期，则不得从工作日中减去它，因为它从未被算作工作。 该算法通过检查来处理这个问题`is_weekend(d)`减法之前，确保不发生重复计算错误。 

最后一个边缘情况涉及假期列表中的重复项。 如果同一天出现多次，简单的减法会错误地多次减少答案。 使用集合可确保每天只处理一次，即使重复输入也能保持正确性。
