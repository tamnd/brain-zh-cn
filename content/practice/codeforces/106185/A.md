 ## 解决方案
 ## 问题理解

 我们得到了一系列测试用例。 每个测试用例包含一个整数$n$，对于每一个，我们在概念上构建一个$n \times n$乘法表所在行的单元格$a$和列$b$包含产品$a \cdot b$。 任务是计算该表中所有条目的总和。 

所以对于一个固定的$n$，所需值为$$\sum_{a=1}^{n} \sum_{b=1}^{n} a \cdot b.$$输入最多可以包含 100 个值$n$，每个都在 1 到 100 之间（含），并且当出现单个零时输入结束。 

界限非常小。 即使是最天真的双循环$a$和$b$至多是$100 \times 100 = 10^4$每个测试用例的操作，在 100 个测试用例中是微不足道的。 这已经告诉我们，任何合理的$O(n^2)$甚至$O(n^3)$解决方案会通过，但表达式的结构允许更清晰的东西。 

边缘情况大多较小$n$。 什么时候$n = 1$，答案是$1$。 什么时候$n = 2$，表为$$\begin{matrix}
1 & 2 \\
2 & 4
\end{matrix}$$总和是$9$。 当人们试图将问题解释为仅对角线求和或仅对不同的乘积求和时，通常会出现错误的方法，但问题明确地计算每个单元格，包括重复项，例如$a \cdot b$和$b \cdot a$作为单独的条目。 

另一种微妙的失败模式是将总和视为涉及唯一对$a \le b$，这会错误地将非对角线贡献减半。 

## 方法

 最直接的方法就是模拟表格。 对于每个测试用例，迭代所有对$(a, b)$，计算乘积并累加。 这是简单且正确的，因为它准确地反映了定义。 每个测试用例的成本是$O(n^2)$，并与$n \le 100$，这最多意味着$10^4$每个案例的乘法，可以忽略不计。 

但该表达式具有很强的可分离性。 总和$$\sum_{a=1}^{n} \sum_{b=1}^{n} a \cdot b$$可以通过因式分解重新排列：$$\left(\sum_{a=1}^{n} a\right) \cdot \left(\sum_{b=1}^{n} b\right).$$两个总和相同，因此结果变为：$$\left(\sum_{i=1}^{n} i\right)^2.$$内部和是标准算术级数：$$\sum_{i=1}^{n} i = \frac{n(n+1)}{2}.$$所以最终的答案是：$$\left(\frac{n(n+1)}{2}\right)^2.$$关键的观察是认识到乘积的双和会分解为和的乘积，因为索引是独立的。 这将每个测试用例减少为常数时间算术。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 已接受 |
 | 最佳|$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 我们一一处理测试用例，直到遇到零。 

1. 读取一个整数$n$。 如果$n = 0$，停止处理。 这是必要的，因为零是一个标记，而不是输入域的一部分。 
2. 计算 1 到 的整数之和$n$使用公式$s = n(n+1)/2$。 这取代了对所有值的迭代，并且是有效的，因为该序列是常数差为 1 的算术序列。 
3. 计算值的平方$s$以获得最终答案。 这对应于在乘法表的行和列上组合两个独立的相同和。 
4. 立即输出结果。 

为何有效：每个细胞$(a, b)$贡献$a \cdot b$。 自从$a$独立出现$b$, 每一个选择$a$与每一个选择配对$b$，因此每个维度的总贡献会成倍增加。 这将双和完全分解为两个相同线性和的乘积，确保不会遗漏或多算交互项。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    out = []
    while True:
        line = input().strip()
        if not line:
            break
        n = int(line)
        if n == 0:
            break
        s = n * (n + 1) // 2
        out.append(str(s * s))
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现读取每个测试用例，直到终止零。 对于每个$n$，它使用整数算术计算三角数，确保没有浮点精度问题，然后对其进行平方。 使用整数除法可以保证正确性，因为$n(n+1)$总是均匀的。 

一个微妙的点是正确处理输入终止。 循环必须在零处停止而不是尝试处理它，因为包含它会错误地产生额外的输出行。

 ## 边缘情况

 对于$n = 1$，网格有一个单元格，其中包含$1 \cdot 1$。 该算法计算$s = 1$，和输出$1$，与表完全匹配。 

为了$n = 2$，网格包含四种产品：1、2、2 和 4。该算法计算$s = 3$，然后平方得到 9。这与所有四个条目的总和相匹配，确认对称的非对角线贡献不会被重复计算或遗漏。 

对于较大的$n$，例如 100，计算保持稳定，因为所有中间值都适合 64 位整数。 该结构避免了迭代累积，因此不存在累积浮点错误或中间循环溢出的风险。
