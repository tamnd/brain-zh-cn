 ## 解决方案
 ## 问题理解

 我们得到一排位置，每个位置都带有一个标记有左括号或右括号的标记。 我们不采用通常意义上的子串。 相反，我们通过沿着这些位置行走来构建序列。 

我们首先选择任何位置并打印其字符。 之后，每个下一步都必须移动到该行的相邻位置（左侧或右侧），然后我们在新位置打印字符。 我们可以多次重访位置，但不允许我们留在原地，所以每一步都必须移动到邻居。 

此遍历会产生一串括号。 我们被要求计算有多少有序的起始位置和结束位置对允许至少一次这样的遍历来产生正确的括号序列，这意味着前缀余额永远不会变为负值并且最终余额为零。 

所有测试用例的约束 n 高达 2×10^5 意味着每个测试用例具有二次行为的任何解决方案都是不可能的。 每个测试用例甚至需要 O(n log n) 或 O(n)，因为总输入大小很大但已聚合。 

微妙的一点是，步行不需要简单。 我们可以来回移动，因此序列不限于子字符串或简单路径。 这使得诸如“仅检查子串平衡”之类的幼稚解释不正确。 

当子字符串具有相同数量的“(”和“)”但不是有效的前缀平衡序列时，就会出现典型的失败情况。 

例如，考虑 s = ")()(" ，如果我们错误地假设子串有效性，我们选择 i = 2, j = 3。子串 "()"" 是有效的，但对于像 ")()(" 这样更复杂的情况，如果顺序固定，一些具有相等计数的子串无法形成有效的序列。但是，因为我们可以通过步行重新排序，所以这种直觉会改变并成为解决方案的关键。

 ## 方法

 直接蛮力方法会尝试每一对开始位置和结束位置，并针对每一对尝试模拟它们之间所有可能的游走，检查是否可以生成有效的括号序列。 即使我们将自己限制在合理的行走范围内，可能的路径数量也会呈指数级增长，因为在每一步我们都可以选择向左或向右，并且可以任意重新访问节点。 即使对于较小的 n，这也很快变得不可行。 

关键的观察是，线条结构允许我们绕道“重新排列”字符。 通过在边缘上来回移动，我们可以有效地交换打印序列中的相邻元素。 重复这个想法表明，在选定的开始和结束之间，只要我们尊重我们走在一条连通的路径上，我们就可以实现访问字符的任何排列。 

一旦理解了这种灵活性，问题就不再是路径构造，而是关于计算其多组字符可以形成有效括号序列的段。 当且仅当“(”的数量等于“)”的数量时，多重集才能形成有效的序列，并且我们可以排列它们，使所有左括号都在前面。 这意味着唯一真正的约束是计数相等，而不是段内的顺序。 

因此，任务简化为对端点对进行计数，以使它们之间的总余额为零。 这成为一个前缀和计数问题，具有额外的奇偶校验约束，该约束来自步行交替步骤的事实，并且序列的长度必须与底层前缀表示中端点的奇偶校验相匹配。

 | 步行中的蛮力 | 指数| O(n) | 太慢了|
 | 前缀平衡分组| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个 '(' 转换为 +1，将每个 ')' 转换为 -1，并计算数组上的前缀和，其中 prefix[0] = 0。

然后，我们将一对有效的端点重新解释为两个前缀索引 i 和 j，使得它们之间的净和为零。 该条件变为 prefix[j] = prefix[i - 1]。 

我们从左到右迭代所有可能的 j，对于每个 j，我们想要计算有多少个较早的位置 i - 1 具有相同的前缀值。 

然而，步行的结构还有一个额外的限制：端点之间打印的字符数必须与交替移动所需的奇偶校验相匹配。 这意味着要求 (j - (i - 1)) 具有偶数长度，这相当于 i - 1 和 j 具有相同的奇偶校验。 

我们维护一个由前缀总和值索引的频率表，并且在每个前缀值中，我们通过索引奇偶校验来分隔计数。 

对于每个位置 j，我们添加与 j 具有相同奇偶校验的前缀 [j] 先前出现的次数。 

最后，我们将其累积到所有 j 上。 

它之所以有效，是因为将行走自由度压缩为前缀和。 来回移动的能力确保我们不局限于固定的子串； 相反，只有净余额才重要。 奇偶校验条件对每个步骤更改位置的事实进行编码，因此打印的字符数与前缀索引模型中的端点奇偶校验对齐。 该算法是正确的，因为每个有效的构造都恰好对应于一个这样的前缀对，并且每个这样的配对都可以通过适当的遍历来实现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    while True:
        line = input().strip()
        if not line:
            continue
        n = int(line)
        if n == 0:
            return
        s = input().strip()

        pref = 0
        cnt = {}
        cnt.setdefault(0, [0, 0])
        cnt[0][0] = 1

        ans = 0

        for i in range(1, n + 1):
            c = s[i - 1]
            if c == '(':
                pref += 1
            else:
                pref -= 1

            p = i & 1
            if pref in cnt:
                ans += cnt[pref][p]

            if pref not in cnt:
                cnt[pref] = [0, 0]
            cnt[pref][p] += 1

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案动态构建前缀和，并维护一个字典，将每个前缀值映射到它在偶数和奇数索引中出现的次数。 每次我们扩展到位置 i 时，我们都会立即计算与前缀值和奇偶校验要求匹配的有效先前端点，然后更新结构。 

主要的微妙之处在于一致地使用基于 0 和基于 1 的索引。 这里，循环中索引 i 从 1 开始，奇偶校验直接从 i 中获取，这符合比较 i - 1 和 j 得出的要求。 

## 工作示例

 考虑一个小例子 s = "()()"。 前缀和为 0, 1, 0, 1, 0。 

我们通过奇偶校验来跟踪每个前缀值的出现：

 | 我| 字符 | 首选项 | 奇偶校验(i) | 匹配上一个 | 运行 ans |
 | --- | --- | --- | --- | --- | --- |
 | 1 | ( | 1 | 奇数 | 无 | 0 |
 | 2 | ) | 0 | 甚至| pref 0 偶数：i=0 | 1 |
 | 3 | ( | 1 | 奇数 | pref 1 奇数： i=1 | 2 |
 | 4 | ) | 0 | 甚至| pref 0 偶数：i=0,2 | 4 |

 这显示了重复的前缀值如何直接对应于有效的端点对。 

现在考虑 s = ")))(((" ，这是局部严重不平衡的。前缀值变化很大，并且仅在前缀总和返回到较早值的情况下才会发生匹配，这表明只有全局平衡的段才有贡献。

 ## 边缘情况

 一种重要的边缘情况是前缀和立即重复但奇偶校验不匹配。 例如，像“()()”这样的模式在偶数和奇数位置都创建前缀值 0。 该算法将奇偶校验桶分开，因此仅计算有效的转换。 

另一种边缘情况是完全不平衡的前缀，例如 "))))((((("。这里前缀值几乎从不匹配，因此答案为零，并且字典只存储不同的值而不会发生冲突。

 最后，n = 2 的情况确保了奇偶校验处理的正确性。 对于“()”，前缀在 i = 2 处返回到零，并且由于奇偶校验与 i = 0 处的较早出现匹配，因此该对仅计数一次，与预期的单个有效结构匹配。
