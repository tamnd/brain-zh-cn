 ## 解决方案
 ## 问题理解

 我们给定一个圆$2n$座位。 每个座位最初都有一个带标签的物品，并且标签重复，以保证每个标签出现偶数次。 座位的排列使得每个位置$i$有着独特的对立位置$i+n$在圆圈上。 

目标是执行一系列交换，其中每次交换交换圆上两个相邻座位的物品，以便在最终排列中，每对相对的座位都具有相同的标签。 

任务不是明确构建最终排列，而是计算实现这种配置所需的最小相邻交换数量。 

关键的限制是$2n$所有测试用例的总和最多为$2 \cdot 10^5$，这强烈建议围绕$O(n \log n)$或者$O(n)$每个测试用例。 任何试图直接模拟交换或搜索排列的策略都是立即不可行的，因为状态数量随着$2n$，甚至贪婪的交换模拟也会降级为$O(n^2)$每次测试。 

这个问题的一个微妙问题是，保证了“配对可行性”，但不保证唯一性。 幼稚的方法可能会错误地假设配对相同的值是简单的，而不考虑交换如何在整个循环中进行全局干扰。 

如果我们一次贪婪地修复一对相反的对，就会出现一种常见的失败情况。 例如，如果我们尝试将每对独立的匹配元素放在一起，我们很容易就会过多计算交换，因为移动一个元素会影响其他元素的未来距离。 

另一个陷阱是将圆视为线性并忽略位置之间的环绕相互作用$1$和$2n$。 由于交换在循环上是相邻的，因此结构不是纯线性的，但相反对约束提供了固定的分解，使我们能够将其简化为线性分配问题。 

## 方法

 一种蛮力的观点是反复挑选相对位置之间的任何不匹配，并尝试通过使用相邻交换沿圆移动一个元素来修复它。 这是正确的，因为相邻交换会生成所有排列，但它太慢了，因为每次校正都可能需要$O(n)$，并且有$O(n)$不匹配，导致$O(n^2)$每个测试用例。 

关键的观察是，最终条件并不取决于每个相对对内的确切排列，只取决于每对中的两个位置包含相同的标签。 这使我们能够考虑将每个标签出现分配给某个相反对的一侧。 

有$n$相反的对$(i, i+n)$，并且每对最终必须持有两个相同的标签。 因此，我们有效地将每个标签出现分配给其中的一个“槽”$2n$位置，在每对接收相同标签两次出现的约束下。 

一旦我们确定了这样的分配，将初始排列转换为目标排列所需的最小相邻交换数量正是两个序列之间的反转距离。 这将问题简化为构建最小化该反演距离的最佳目标分配。 

当我们独立查看每个标签时，结构进一步简化。 由于标签的出现必须分成对，如果标签出现$k$次，它将正好占据$k/2$相反的对。 在最终分配中避免交叉的最佳方法是按排序顺序处理出现的事件并按顺序配对，然后按顺序将它们分配给相反的对。 这种单调结构避免了会增加反转成本的交错。 

因此，解决方案简化为排序的出现位置和排序的目标槽之间的贪婪对齐（$n$左右两侧的索引对重复）。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 强力相邻固定 |$O(n^2)$|$O(1)$-$O(n)$| 太慢了|
 | 排序 + 贪婪地将出现次数分配给对槽 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过将圆变成一组固定的相对对来构建解决方案，然后将每个出现的位置分配给其中一对内的位置。 

1. 拆分$2n$职位进入$n$相反的对$(i, i+n)$。 这些对是唯一允许我们用相同标签“填充”的结构单元。 
2. 对于每个标签，收集其出现的所有索引。 由于每个标签出现偶数次，因此我们可以在按位置排序后将其出现分成连续的对。 
3. 对于出现在位置的标签$p_1 < p_2 < \dots < p_k$，我们配对$p_1$和$p_2$,$p_3$和$p_4$， 等等。 每对这样的座位最终将占据一对相对座位的两侧。 这可以防止标签内的交叉分配，这只会增加交换成本。 
4. 现在我们将每个配对组视为需要分配给一个相反的配对索引$i$（意味着一次出现进入位置$i$，另一个去$i+n$）。 我们维护所有可用货币对索引的列表$1 \dots n$。 
5. 我们对标签对列表（按照其第一次出现的顺序）和可用的对索引进行排序，然后按顺序匹配它们。 这种对齐最大限度地减少了总位移成本，因为两个序列都是独立块的单调表示。 
6. 分配一对所贡献的成本$(x, y)$到相反的一对$i$是$|x - i| + |y - (i+n)|$。 将所有分配的总和得出总反转距离，它等于相邻交换的最小数量。 

### 为什么它有效

 关键的不变量是我们决不允许以强制交叉的方式在不同标签之间交错分配。 两个标签块之间的任何交叉分配都可以在不违反可行性的情况下取消交叉，并且将严格减少或保留总位移。 这与使具有绝对成本的最优分配问题可分离为基于排序的匹配的结构属性相同。 一旦所有标签块都是非交叉的，问题就分解为排序位置和排序对索引之间的独立一维分配，这可以通过单调匹配得到最佳解决。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    pos = {}
    for i, x in enumerate(a):
        pos.setdefault(x, []).append(i)
    
    pairs = []
    
    for x in pos:
        arr = pos[x]
        for i in range(0, len(arr), 2):
            pairs.append((arr[i], arr[i+1]))
    
    pairs.sort()
    
    # assign pair indices 1..n
    # matching in order minimizes displacement
    ans = 0
    for i, (u, v) in enumerate(pairs, start=1):
        # i is 1-based pair index
        # compute cost to map u->i, v->i+n
        ans += abs(u - (i - 1)) + abs(v - (i - 1 + n))
    
    print(ans)

```该代码首先按标签对索引进行分组，因为只有相同标签的相对位置对于决定内部结构很重要。 每个标签提供成对的出现，这些成对代表必须占据相对位置的“单元”。 

形成这些对后，对它们进行排序可确保较早出现的位置与较早相对的位置相匹配，从而防止交叉，从而增加交换距离。 

最后的循环直接计算位移成本：每个元素从其原始索引移动到最终排列中的目标槽，并且相邻的交换完全对应于沿圆的线表示移动的单位距离。 

一个微妙的实现点是将循环索引转换为线性索引$0 \dots 2n-1$，这使我们能够将相反的位置干净地视为$i$和$i+n$。

 ## 边缘情况

 关键的边缘情况是所有相同的标签已经聚集但未与相对的对对齐。 在这种情况下，天真的贪婪的“修复不匹配”方法可能会尝试在本地进行交换并超出最小值。 基于配对的公式通过全局分配而不是局部校正来完全避免这种情况。 

另一个微妙的情况是标签严重交错时，例如交替模式，如$1,2,1,2,\dots$。 在这里，贪婪的相邻修复倾向于振荡元素，但排序的配对策略通过一致地配对出现并将它们单调地分配给相反的槽，立即揭示了最佳结构。
