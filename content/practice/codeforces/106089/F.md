 ## 解决方案
 ## 问题理解

 我们得到一组不同的整数，每个整数至少有两位数字。 对于每个有序的不同数字对，我们要检查一个简单的数字条件：第一个数字的最后一位数字必须与第二个数字的第一位数字匹配。 满足此规则的每一对都会被计数，并且方向很重要，因此交换顺序可以改变该对是否有效。 

任务是计算最多 200,000 个数字中有多少有序对满足此条件。 每个数字最多为 10^9，因此提取第一个和最后一个数字非常简单，并且每个数字的时间恒定。 

对 n 的约束立即排除了对之间的任何二次比较。 在最坏的情况下，简单的 O(n^2) 扫描将执行大约 4×10^10 检查，这远远超出了一秒的限制。 这表明我们需要按数字属性聚合信息，而不是直接比较元素。 

一个微妙的边缘细节是，以 0 结尾的数字仍然可以作为对的第一个元素出现，但如果没有以 0 开头的数字，它们永远不可能成为有效对的第二个元素。由于输入没有前导零并且至少为 10，因此第一个数字始终在 1 到 9 中。这使得数字分组干净且有界。 

一个典型的错误是从排序或邻接的角度来思考。 对数字进行排序没有帮助，因为条件仅取决于数字，而不取决于数字的接近程度。 另一个错误是在不考虑方向的情况下重复计算无序对，但这里方向至关重要，因此我们必须单独计算所有有效的 (i, j)。 

## 方法

 暴力破解的想法很简单：迭代索引 i 和 j 的每个有序对，计算 ai 的最后一位数字和 aj 的第一位数字，并计算匹配项。 这是正确的，因为它直接实现了定义。 然而，它检查 n(n−1) 对，每次检查的时间复杂度为 O(1)，导致在最大输入大小下大约需要 4×10^10 次操作，这是不可行的。 

关键的观察结果是，数字的值仅通过两个属性起作用：其第一位数字和最后一位数字。 一旦知道了这些，完整的数字就变得无关紧要了。 这使我们能够将整个数组压缩为 9×10 结构的计数：有多少数字以数字 d 开头，有多少数字以数字 d 结尾。 

我们不是显式地配对元素，而是计算每个兼容的数字转换存在多少种选择。 对于固定数字ai，其贡献仅取决于有多少个数字以最后一位数字开头。 将所有人工智能的这些贡献相加就能在线性时间内得到答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 数字计数| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 对于每个数字，提取其第一位数字和最后一位数字。 这将每个元素减少为小固定范围内的一对整数。 
2. 维护一个频率数组 cntStart[d]，表示有多少个以数字 d 开头的数字。 这会压缩对的所有候选第二元素。 
3. 再次迭代所有数字，对于每个数字 ai，将 cntStart[lastDigit(ai)] 添加到答案中。 这会计算所有有效对，其中 ai 是第一个元素。 
4. 当一个数字可以与自身配对时，如果需要，减 1。 在这个问题中，所有数字都是不同的，因此永远不会发生自配对，并且不需要校正。 
5. 输出累计和。 

核心思想是每个数字根据其最后一位数字独立地贡献所有有效的传出对，并且我们永远不需要显式枚举目的地。 

### 为什么它有效

每个有效的有序对 (i, j) 在处理 i 时只计算一次。 该条件仅取决于 ai 的最后一位和 aj 的第一位，并且 cntStart 直接编码 j 的多少个选择满足该约束。 由于每个 j 都恰好计数在一位数字桶中，因此不会遗漏任何有效对，也不包含无效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def first_digit(x: int) -> int:
    while x >= 10:
        x //= 10
    return x

def last_digit(x: int) -> int:
    return x % 10

n = int(input())
a = list(map(int, input().split()))

cnt_start = [0] * 10

for x in a:
    cnt_start[first_digit(x)] += 1

ans = 0
for x in a:
    ans += cnt_start[last_digit(x)]

print(ans)
```该解决方案首先将所有数字压缩为前导数字的计数。 这是唯一需要的全局聚合。 第二遍使用尾随数字计算每个数字的贡献。 

一个微妙的实现细节是，提取第一位数字需要重复除法，但由于数字最多有 10 位数字，因此这是常数时间。 使用字符串转换也是可以接受的，但整数除法可以避免开销。 

我们不会为自对减去任何东西，因为问题表明所有数字都是不同的，并且一个数字不能有等于其最后一位数字的第一位数字以有问题的方式贡献给自己，除非它被算作与自身配对，而由于索引不同，这种情况永远不会发生。

 ## 边缘情况

 一个边缘情况是所有数字共享相同的第一位数字。 例如：```
3
10 11 12
```这里 cntStart[1] = 3，但所有最后一位数字都是 0、1、2，除了可能的 1 之外，它们与任何起始数字都不匹配。如果最后一位数字是 1，则只有 11 起作用。该算法干净地处理这个问题，因为每个数字独立查询 cntStart[lastDigit]，并且不会发生人工配对。 

另一种边缘情况是数字以不作为任何第一位数字出现的数字结尾。 例如：```
3
10 21 32
```这里最后的数字是 0, 1, 2，但是如果没有数字以 0 开头，那么涉及 10 的贡献自动为零，因为 cntStart[0] = 0。该算法自然会过滤掉无效的转换，而无需特殊的大小写。 

最后的结构边缘情况是数字形成一个完整的循环。 即便如此，每一对都是通过频率查找独立计数的，并且该算法不会假设任何超出数字兼容性的排序或结构。
