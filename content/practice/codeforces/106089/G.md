 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 我们可以执行一个操作：在数组中选择两个不同的位置，将给定值 x 添加到一个选定的元素，并将另一个值 y 添加到另一个选定的元素。 在这一次修改之后，我们计算所有数组元素的 XOR，并希望使该值尽可能大。 

关键结构是操作只影响两个位置，其他一切保持不变。 所以最终的异或可以看作是数组的原始异或，只不过两个元素被它们的递增版本替换。 任务是选择最好的索引对并以最佳方式在它们之间分配 x 和 y。 

约束最多允许 2·10^5 个元素，因此任何对的二次枚举都会太慢。 解决方案需要将配对搜索减少为更接近线性或 n log n，通常使用支持快速“最佳合作伙伴”查询的结构。 

当贪婪推理应用不正确时，就会出现微妙的失败情况。 例如，如果我们总是选择两个最大的数字，或者总是尝试最大化一个元素上的本地增量，我们可能会错过依赖于两个修改值之间的位交互的全局异或改进。 当假设分配 x 和 y 的顺序无关紧要时，会出现另一种失败情况，但实际上交换哪个索引获取哪个值会改变 XOR 结果。 

例如，如果数组是 [1, 2, 3, 4, 5]，x = 2，y = 3，一个天真的想法可能是同时应用于最大元素 4 和 5。但是，将变换应用于不同的对可以产生更大的 XOR，因为 XOR 就数值而言不是单调的。 

## 方法

 直接的强力方法将尝试所有有序的索引对 (i, j)，i ≠ j，并考虑分配 x 和 y 的两种方法。 对于每个候选者，我们将修改数组并重新计算所有元素的异或。 重新计算 XOR 需要 O(n)，因此这会导致总体 O(n^3)，这远远超出了限制。 即使我们预先计算数组的基本异或，更新两个位置仍然需要仔细重新计算贡献，并且检查所有对仍然是 O(n^2)，这对于 2·10^5 来说太慢了。 

主要的简化来自于隔离修改单个元素时 XOR 的变化方式。 整个数组的异或可以分解为所有未触及元素的异或与两个修改位置的异或贡献相结合。 这意味着我们可以独立地考虑每个位置的影响。 

令 S 为原始数组的 XOR。 如果我们用 a + v 替换元素 a，则该索引对总 XOR 的贡献从 a 变为 (a + v)。 这种变化可以表示为 a 的异或和 (a + v) 的异或。 因此，每个索引都会贡献一个仅取决于其自身值的转换项。 

这使我们能够将问题重写为选择两个索引 i 和 j 并为它们分配两个角色之一，其中每个角色都会产生从原始元素派生的确定性“转换值”。 在此重新表述之后，任务变为从两个派生数组中选择两个值以最大化全局 XOR 表达式，该表达式可以简化为最大 XOR 对查询。 这就是二进制 trie 技术的应用场合。 

最终的想法是构建允许快速查询每个元素的最佳合作伙伴的结构。 由于我们还需要保证i≠j，所以我们在查询时暂时排除当前元素。

 ## 算法演练

1. 计算原始数组的基 XOR S。 该值本身并不是最终答案，但它有助于分离表达式的固定部分和变化部分。 
2. 对于每个元素 a[i]，计算两个变换后的值。 一个假设该元素接收 x，另一个假设它接收 y。 称它们为 A[i] = a[i] XOR (a[i] + x) 和 B[i] = a[i] XOR (a[i] + y)。 这些表示位置 i 的 XOR 贡献如何根据其接收的操作而变化。 
3. 根据分配以两种可能的方式选择索引 i 和 j 后重写最终的 XOR：

 一种是 i 得到 x，j 得到 y，另一种是角色交换。 每种情况都成为 A[i] XOR B[j] 与常数 S 相结合的函数。 
4. 将常数S吸收到一侧，使得目标成为两组值之间的最大异或对查询。 这将问题转化为寻找两个元素，一个来自数组的转换版本，另一个来自另一个，以最大化异或。 
5. 在两个集合之一上构建一个二元特里树。 trie 存储该集合的所有值，从而能够对其他集合中的任何候选值进行快速最大异或查询。 
6. 对于每个索引i，暂时从trie中删除其对应的元素，查询最佳伙伴值，然后恢复。 这确保我们永远不会将元素与其自身配对。 
7. 对交换的角色案例重复相同的过程，并在两个方向上取最大值。 

### 为什么它有效

 关键属性是修改一个索引的效果仅取决于该索引，而不取决于任何其他元素。 这允许操作后的总 XOR 分解为固定常数 S 与两个独立贡献（每个选定索引一个）相结合。 一旦建立了这种分离，剩下的问题就纯粹是关于最大化两个多组数字之间的异或。 trie 保证对于每个候选元素，我们都能在对数时间内找到全局最优的伙伴，并且临时删除该元素可确保精确保留唯一性约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Trie:
    def __init__(self):
        self.nxt = [[-1, -1]]
        self.cnt = [0]

    def add(self, x, delta):
        v = 0
        self.cnt[v] += delta
        for b in range(30, -1, -1):
            bit = (x >> b) & 1
            if self.nxt[v][bit] == -1:
                self.nxt[v][bit] = len(self.nxt)
                self.nxt.append([-1, -1])
                self.cnt.append(0)
            v = self.nxt[v][bit]
            self.cnt[v] += delta

    def query(self, x):
        v = 0
        res = 0
        for b in range(30, -1, -1):
            bit = (x >> b) & 1
            want = bit ^ 1
            if self.nxt[v][want] != -1 and self.cnt[self.nxt[v][want]] > 0:
                v = self.nxt[v][want]
                res |= (1 << b)
            else:
                v = self.nxt[v][bit]
        return res

n, x, y = map(int, input().split())
a = list(map(int, input().split()))

def solve_order(A, B, base_xor):
    t = Trie()
    for v in B:
        t.add(v, 1)

    best = 0
    for i in range(n):
        t.add(B[i], -1)
        best = max(best, t.query(A[i]))
        t.add(B[i], 1)

    return best ^ base_xor

S = 0
for v in a:
    S ^= v

A = [v ^ (v + x) for v in a]
B = [v ^ (v + y) for v in a]

ans = solve_order(A, B, S)
A2 = B
B2 = [v ^ (v + x) for v in a]
ans = max(ans, solve_order(A2, B2, S))

print(ans)
```trie 存储一侧的候选值并支持 XOR 的贪婪逐位最大化。 功能`solve_order`评估一个固定的分配方向，通过在查询之前临时删除它来强制同一索引不会使用两次。 

变换数组`A`和`B`编码添加 x 或 y 后每个元素的贡献如何变化。 选择最佳对后，与基和S进行异或并入最终结果。 

## 工作示例

 考虑输入：```
n = 5, x = 2, y = 3
a = [1, 2, 3, 4, 5]
```我们首先计算 S = 1 XOR 2 XOR 3 XOR 4 XOR 5 = 1。 

现在我们构建转换后的值 A[i] = a[i] XOR (a[i] + 2)、B[i] = a[i] XOR (a[i] + 3)。 

| 我| 一个[我] | a[i]+2 | A[i] | a[i]+3 | B[i]|
 | --- | --- | --- | --- | --- | --- |
 | 0 | 1 | 3 | 1 异或 3 = 2 | 4 | 1 异或 4 = 5 |
 | 1 | 2 | 4 | 6 | 5 | 7 |
 | 2 | 3 | 5 | 6 | 6 | 5 |
 | 3 | 4 | 6 | 2 | 7 | 3 |
 | 4 | 5 | 7 | 2 | 8 | 13 |

 对于每个 i，我们删除 B[i] 并从 trie 中查询最佳 A[i] XOR B[j]。 找到的最佳对对应于最大化组合位结构而不是数字接近度的索引。 

该轨迹表明，像 13 这样的大值并不总是最佳伙伴； 重要的是它们的二进制表示如何在 XOR 下交互。

 ## 边缘情况

 当 x = 0 或 y = 0 时，其中一个变换变为恒等式。 在这种情况下，A 或 B 之一到处都为零，并且特里树仍然正常工作，因为最佳伙伴选择减少到仅最大化非零变换的效果。 

当 x = y 时，两个赋值方向都会生成相同的转换数组。 该算法仍然有效，因为它评估两个方向，但它们崩溃为相同的计算。 

当 n = 2 时，只有一对有效的索引，并且两个方向都在同一对上进行评估。 trie 中的临时删除确保我们不会错误地重复使用同一索引两次，并且结果与评估单个可能的操作完全对应。
