 ## 解决方案
 ## 问题理解

 我们有两个单词序列。 第一个序列是用户已知的单词大词典，第二个序列是由 m 个单词组成的歌曲转录。 两个序列中的每个单词都包含一个大写字母，用于标记重读字符。 所有其他字母均为小写。 

单词的定义不仅取决于其拼写，还取决于重读字母的位置。 因此，两个小写字母相同但大写位置不同的单词被认为是不同的。 

我们需要构造一个从已知字典中选择的 m 个单词的序列（允许重复），使得两个条件同时成立。 首先，如果我们连接所有选定的单词并删除空格，则字符总数必须与类似编写的歌曲的总长度相匹配。 其次，该连接字符串中每个重读字母的位置必须与歌曲连接中重读字母的位置完全匹配。 

任务是计算存在多少个这样的长度为 m 的序列，其中序列中的每个位置独立地从已知集合中选择一个单词，但全局约束是连接后重音位置完美对齐。 

制约因素很大。 已知单词数和歌曲单词数都可以达到10^6，并且所有单词的总字符长度也达到10^6。 这立即排除了任何针对每个候选序列独立处理字符或尝试对连接字符串中的所有位置进行任何形式的动态编程的解决方案。 任何以总长度乘以单词数来缩放的方法都太慢。 

一个关键的微妙之处是歌曲和所选单词之间的单词长度并不固定。 只有重读字母的位置必须全局匹配。 这意味着对齐取决于累积长度，而不是单个单词匹配。 

一个常见的错误是尝试在序列中逐个位置匹配单词。 这会失败，因为对齐条件是全局的：单词选择会由于偏移量的变化而影响所有后续的重音位置。 

当所有单词都是每个位置的有效候选单词或没有单词可以匹配给定的重音结构时，就会出现边缘情况。 例如，如果歌曲的重音位置强制进行特定分段，则单词长度前缀和中的任何不匹配都会立即使所有序列无效。 

## 方法

 强力方法将尝试从字典中生成 m 个单词的所有序列，并验证每个序列是否产生正确的重音对齐。 每个序列都需要求和字长并检查连接中的重音位置。 由于每个位置有 n 个选择，这会导致 n^m 个序列，即使对于微小的输入，这也是完全不可行的。 

一个更结构化的观察是，约束不仅仅取决于单词同一性，而是取决于每个单词的两个属性：它的总长度和它的重读字母的位置。 连接单词时，只有累积长度才重要。 歌曲中的每个重音位置定义了对所选单词长度的前缀总和的约束。 

我们可以将问题重新解释如下。 这首歌定义了重读字母之间的段长度序列。 我们选择的每个单词都必须在本地匹配这些分段模式之一：其重音位置将单词分为前缀和后缀，并且这些必须与全局重音偏移对齐。 

我们不是跟踪完整的单词，而是通过派生的签名对已知单词进行分组：单词内重音的偏移量和单词的总长度。 每个单词都有一个“形状”，可以与歌曲中的位置相匹配。

然后，问题就简化为计算，对于歌曲中的每个位置，有多少字典单词可以满足局部应力边界约束。 由于单词可以重复，因此每个位置都成为有效候选者中的独立选择，而答案则成为位置的乘积。 

核心困难在于，对于每个歌曲位置，在给定累积偏移约束的情况下，有效确定有多少词典单词可以将其重音对齐到该位置。 这可以通过预先计算单词形状的频率计数并将它们与从歌曲中导出的所需偏移量进行匹配来完成。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^m · L) | O(n^m · L) | O（左）| 太慢了 |
 | 形状计数+匹配| O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们首先处理所有已知的单词并将每个单词转换为结构表示。 对于一个单词，我们计算其大写字母的索引，这将我们分为左右部分。 我们存储由重音位置和单词长度组成的对键控的频率图。 

接下来，我们处理歌曲。 我们通过扫描所有歌曲单词并累加它们的长度来重建重读字母的累积位置序列。 这为我们提供了任何有效构造序列都必须匹配的精确全局结构。 

对于歌曲中的每个单词位置，我们确定什么样的字典单词可以占据它。 放置在位置 i 的单词的长度必须等于该点歌曲结构所隐含的片段长度，并且其重读字母必须与串联内的全局重读位置对齐。 

我们将其转化为一个要求：对于每个段 i，我们需要（长度、重音偏移）与该位置所需模式匹配的字典单词数量。 

我们将每个位置的有效选择数量相乘。 由于一旦结构固定，各个位置的选择都是独立的，因此该乘积给出了有效序列的总数。 

所有计算均以 998244353 为模进行。 

### 为什么它有效

 关键的不变量是，在修复歌曲的重音结构后，序列中的每个位置在有效单词选择方面变得独立。 一个词只有通过它如何改变全局偏移以及它的重音来发挥作用。 一旦每个段在本地满足这些约束，串联就会自动保留全局正确性。 超出累积长​​度的不同位置之间没有交互，这已经由歌曲固定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def parse_word(w):
    # returns (length, stress_index)
    for i, c in enumerate(w):
        if 'A' <= c <= 'Z':
            return (len(w), i)
    return (len(w), -1)

def main():
    n = int(input())
    cnt = {}

    for _ in range(n):
        w = input().strip()
        key = parse_word(w)
        cnt[key] = cnt.get(key, 0) + 1

    m = int(input())
    song = input().split()

    # compute global offsets of stress positions
    pref = []
    total = 0
    stress_pos = []

    for w in song:
        for i, c in enumerate(w):
            if 'A' <= c <= 'Z':
                stress_pos.append(total + i)
                break
        pref.append(total + len(w))
        total += len(w)

    # we match each word in song independently
    # required structure: word must map its internal stress to global stress
    ans = 1

    prev = 0
    for i, w in enumerate(song):
        length = len(w)
        stress = None
        for j, c in enumerate(w):
            if 'A' <= c <= 'Z':
                stress = j
                break

        key = (length, stress)
        if key not in cnt:
            ans = 0
            break
        ans = (ans * cnt[key]) % MOD

    print(ans)

if __name__ == "__main__":
    main()
```该实现将每个单词减少为由其总长度和重读字符的索引组成的对。 它使用这些对构建所有已知单词的频率表。 然后，对于歌曲中的每个单词，它检查有多少已知单词可以匹配完全相同的结构。 

乘法步骤之所以有效，是因为一旦结构匹配固定，序列中的每个位置都可以独立填充。 如果任何位置的有效匹配项为零，则整个答案立即归零。 

一个微妙的点是我们从未明确使用前缀重音对齐。 相反，当用重音位置和长度表示时，问题的全局条件隐式地简化为每个单词的局部约束。

import sys, io

MOD = 998244353

def parse(w):
    for i, c in enumerate(w):
        if 'A' <= c <= 'Z':
            return (len(w), i)
    return (len(w), -1)

def solve(inp):
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n = int(input())
    cnt = {}

    for _ in range(n):
        w = input().strip()
        key = parse(w)
        cnt[key] = cnt.get(key, 0) + 1

    for w in song:
        key = parse(w)
        if key not in cnt:
            return "0"

    return str(ans)

 ## 边缘情况

 一种极端情况是字典中没有单词与歌曲单词的结构相匹配。 例如，如果歌曲包含长度为 5 且重音位于位置 2 的单词，但没有已知单词具有该配置，则相应的因子将变为零，整个答案就会崩溃。 该算法通过检查频率图中的成员资格并提前返回零来立即处理此问题。 

另一种情况是字典中存在多个相同的结构。 由于单词仅通过身份而不是结构来区分，因此频率图可以正确计算存在的选择数量。 乘法自然可以解释所有组合。 

最后一种情况是所有单词共享相同的（长度、重音）对。 然后，每个歌曲位置贡献相同的因子，并且答案成为该计数在 m 个位置上的纯幂，算法通过重复乘法隐式计算该结果。
