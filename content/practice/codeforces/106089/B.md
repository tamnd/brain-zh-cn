 ## 解决方案
 ## 问题理解

 我们得到了一系列每日速度。 我们必须选择一个连续的日子段，其长度至少为`k`。 

对于一个段`[l, r]`，其质量是该段内所有速度的算术平均值：$$\frac{v_l + v_{l+1} + \dots + v_r}{r-l+1}$$任务是找到任何具有最大可能平均值的线段并输出其端点。 

约束中最有趣的部分是条件$$n \cdot k \le 10^6.$$虽然`n`可以达到`100000`，只有当`k`非常小。 任何算法，其复杂度与`nk`安全地在限制范围内，同时`O(n^2)`是不可能的，当`k = 1`。 

一个常见的错误是假设最佳线段的长度必须精确`k`。 

考虑：```
n = 3, k = 2
1 100 100
```长度段`2`最大平均值是`[2,3]`， 平均的`100`。 整个数组有平均数`67`，所以长度`2`确实更好。 

现在考虑：```
n = 3, k = 2
50 100 50
```最好的长度 2 平均值是`75`，但整个数组也有平均值`66.67`。 这里的长度`2`保持最佳状态。 

这两个例子都不能证明任何事情。 关键点是，如果没有证明，就不能简单地忽略较长的段。 

另一个微妙的问题是使用浮点数比较平均值。 考虑：```
k = 2
1000000000 999999999
999999999 1000000000
```不同细分市场的平均值可能存在微小差异。 使用浮点比较可能会意外选择错误的段。 分数的交叉乘法完全避免了这种情况。 

## 方法

 蛮力的想法很简单。 计算前缀和，枚举每个有效段，计算其总和`O(1)`，计算其平均值，并保留最好的一个。 

问题在于有效段的数量。 有`Θ(n²)`其中。 和`n = 100000`，这远远超出了一秒钟可以处理的范围。 

关键的观察来自于观察很长的片段。 

假设一个线段的长度至少为`2k`。 将其分成两个连续的部分，每个部分的长度至少`k`。 

如果左边部分有平均值`A`右边的部分有平均值`B`，那么整个段的平均值就是加权平均值`A`和`B`。 加权平均值永远不能超过其两个组成部分。 因此，至少两个部分之一的平均值大于或等于整个段的平均值。 

这意味着至少一段长度`2k`永远不可能严格优于其所有有效子部分。 我们可以反复缩短这样的一段，直到它的长度小于`2k`，而不降低平均值。 

因此，一些最佳答案的长度总是在范围内$$k \le L \le 2k-1.$$现在搜索空间变小了。 只有`k`可能的长度。 对于每个长度，我们扫描所有起始位置。 总工作量为$$O(n \cdot k),$$这最多是`10^6`因为约束。 

前缀总和给出了恒定时间内每个段的总和，并且使用交叉乘法来精确比较平均值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有细分市场 | O(n²) | O(n) | 太慢了 |
 | 枚举从 k 到 2k-1 的长度 | O(nk) | O(nk) | O(n) | 已接受 |

 ## 算法演练

 1.构建前缀和数组`pref`， 在哪里`pref[i]`存储第一个的总和`i`元素。 
2. 维持当前最佳段。 存储其总和、长度和端点。 
3. 枚举每一个可能的长度`L`从`k`到`min(n, 2k - 1)`。 

关键定理保证至少有一个最优片段位于该范围内。 
4. 对于每个长度`L`，滑过该长度的所有段。 

使用前缀和，段和为$$pref[r] - pref[l-1].$$5. 将当前段的平均值与迄今为止看到的最佳平均值进行比较。 

与其划分，不如比较$$\frac{sum_1}{len_1}
\quad\text{and}\quad
\frac{sum_2}{len_2}$$通过检查$$sum_1 \cdot len_2
\quad\text{vs}\quad
sum_2 \cdot len_1.$$这是精确的并且避免了浮点错误。 
6. 如果当前平均值较大，则替换存储的答案。 
7. 处理完所有长度后，输出保存的端点。 

### 为什么它有效

 正确性取决于一个属性。 

任何长度至少为`2k`可以分成两个连续的段，长度至少`k`。 整个分段的平均值是这两个部分的平均值的加权平均值。 因此，至少一个部分的平均值至少与整个段一样大。 

从任意最优段开始，每当其长度至少为`2k`。 平均值永远不会减少，最终长度会落入`[k, 2k-1]`。 

所以最佳答案总是存在于该长度范围内。 该算法枚举该范围内每个长度的每个段，并选择具有最大平均值的段。 由于检查了每个候选者并准确比较了平均值，因此返回的段是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    v = list(map(int, input().split()))

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + v[i]

    best_sum = -1
    best_len = 1
    best_l = best_r = 1

    max_len = min(n, 2 * k - 1)

    for length in range(k, max_len + 1):
        for l in range(1, n - length + 2):
            r = l + length - 1
            cur_sum = pref[r] - pref[l - 1]

            if best_sum == -1 or cur_sum * best_len > best_sum * length:
                best_sum = cur_sum
                best_len = length
                best_l = l
                best_r = r

    print(best_l, best_r)

solve()
```前缀和结构使得每个段和在恒定时间内可用。 如果没有它，计算总和将需要扫描该段，这将增加长度的另一个因素的复杂性。 

变量对`(best_sum, best_len)`将当前最佳平均值表示为分数`best_sum / best_len`。 当考虑一个新的段时，比较```
cur_sum * best_len > best_sum * length
```检查是否$$\frac{cur\_sum}{length}
>
\frac{best\_sum}{best\_len}.$$不进行除法，因此不存在精度问题。 

长度上的循环停止于`2k - 1`。 该界限是解决方案符合约束条件的全部原因。 

所有索引都在从 1 开始的坐标中处理，因为所需的输出是从 1 开始的。 前缀和数组是用额外的前导零构建的，使得公式```
pref[r] - pref[l - 1]
```每个部分统一工作。

 ## 边缘情况

 考虑：```
1 1
42
```唯一有效的段是`[1,1]`。 该算法检查长度`1`，根据前缀数组计算其总和，并立即将其存储为答案。 输出是：```
1 1
```现在考虑最佳段长于的情况`k`:```
3 2
3 10 3
```长度 2 段的平均值`6.5`。 长度 3 段的平均值$$\frac{16}{3} \approx 5.33$$实际上这个并不大，所以让我们使用：```
3 2
5 10 5
```长度 2 平均值为`7.5`，而长度 3 平均值是`20/3 ≈ 6.67`。 该算法会检查长度并正确比较分数，而无需浮点运算。 

最后，考虑大值：```
2 1
1000000000 1000000000
```总和达到`2 * 10^9`，在比较过程中我们乘以长度直到`10^5`。 在某些语言中，32 位整数会溢出。 该解决方案使用整数算术比较平均值，Python 的任意精度整数可以安全地处理这些值。
