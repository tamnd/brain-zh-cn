 ## 问题理解

 我们得到了一个元素序列和一个控制子数组“有趣”的参数。 任务是考虑数组的所有连续子段，并根据其内容的多样性确定其中有多少满足约束。 在此问题中，子段只是数组的连续切片，条件取决于该切片内不同值的数量。 

尽管源代码中的语句文本有部分乱码，但核心组合对象是标准的：我们正在处理数组的所有区间，并且需要根据“k-interesting”规则对它们进行过滤，这在典型的公式中意味着区间内不同元素的数量以 k 为界。 

因此，输入表示一个长度为 n 后跟整数 k 的数组。 输出是一个数字：连续子数组的总数，其不同元素集的大小最多为 k。 

此类问题中的约束模式通常意味着 n 最多约为 200000。这立即排除了子数组的任何二次枚举，因为 n 的平方约为 10^10 运算，这远远超出了典型限制。 任何可行的解决方案都必须接近线性或线性对数，理想情况下为 O(n)。 

当值以不规则模式重复时，就会出现微妙的情况。 幼稚的方法通常会在以下情况下失败：

 一种故障模式是假设独立检查每个子数组没问题，这会导致超时。 例如，如果数组交替许多不同的值，例如 1 2 3 4 5 ...，则必须检查每个子数组，并且每个检查本身可能会扫描间隔，从而导致立方行为。 

另一种失败模式是扩展窗口时错误处理不同计数更新。 例如，在像 1 2 1 2 3 这样的数组中，删除或添加元素需要仔细维护频率计数，否则重复项可能会被重复计算或丢失。 

## 方法

 暴力破解的想法很简单：通过选择左端点和右端点来枚举每个子数组，然后计算内部有多少个不同的值。 这可以通过扫描子数组并使用集合来计算不同元素来完成。 虽然概念上正确，但此方法对每个子数组执行 O(n) 工作，并且有 O(n^2) 个子数组，导致最坏情况下的复杂性为 O(n^3)。 即使使用哈希集优化内部扫描也不会改变渐近成本。 

关键的结构观察是，如果我们固定左端点，则有效右端点的集合形成一个连续的范围。 当我们向右移动右指针时，不同元素的数量只会增加或保持不变。 这种单调性使我们能够维持滑动窗口：满足条件时扩大右边界，违反条件时缩小左边界。 

这减少了维护当前窗口内元素的频率图的问题。 每次扩展右指针时，都会增加该元素的计数。 如果它成为一个新的非重复元素，我们就会增加非重复计数器。 如果不同元素的数量超过 k，我们向前移动左指针，直到恢复约束，减少频率并可能减少不同计数。 任一指针的每次移动最多发生 n 次，因此总复杂度变为线性。

 | --- | --- | --- | --- |
 | 蛮力 | O(n3) | O(n) | 太慢了|
 | 滑动窗 | O(n) | O(n) | 已接受 |

 ## 算法演练

我们维护两个定义当前窗口 [l, r) 的指针、窗口内值的频率字典以及当前存在多少个不同值的计数器。 

1. 初始化 l = 0，结果 = 0，以及一个空的频率图。 右指针将在数组上扩展。 
2. 对于从 0 到 n - 1 的每个位置 r，通过增加其频率将 a[r] 包含到窗口中。 如果此元素以前不存在，则增加非重复计数器。 
3.如果不同计数器超过k，则重复向前移动l，同时降低a[l]的频率。 当频率变为零时，减少相异计数器。 继续，直到窗口再次有效。 此步骤确保窗口始终满足约束。 
4. 一旦窗口有效，所有以 r 结尾并从 l 到 r 任意位置开始的子数组都有效。 此类子数组的数量为 (r - l + 1)，因此我们将此值添加到结果中。 
5. 继续扩展 r 直到数组末尾。 

核心不变量是，在每一步，窗口 [l, r] 最多包含 k 个不同元素，并且 l 是为当前 r 维护此属性的最小索引。 这保证了我们永远不会错过有效的子数组，也永远不会计算无效的子数组。 每个有效子数组在其右端点固定时都会被唯一计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    freq = {}
    distinct = 0
    l = 0
    ans = 0

    for r in range(n):
        x = a[r]
        if x not in freq or freq[x] == 0:
            freq[x] = 0
            distinct += 1
        freq[x] += 1

        while distinct > k:
            y = a[l]
            freq[y] -= 1
            if freq[y] == 0:
                distinct -= 1
            l += 1

        ans += (r - l + 1)

    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了上面描述的滑动窗口。 频率字典跟踪每个值在当前窗口中出现的次数。 独特的计数器避免了重复扫描字典。 

关键的实现细节是缩小窗口时的更新顺序。 我们必须首先减少频率，并且只有当频率达到零时才减少不同的计数器。 任何偏离此逻辑的行为都会导致对唯一元素的错误计数。

 ## 边缘情况

 常见的边缘情况是 k 等于 1，这会强制每个有效子数组仅包含相同的元素。 像这样的序列`5 5 5 5`，窗口永远不会超出相同的值，并且算法正确地计算由重复元素形成的所有三角形子数组。 

另一种边缘情况是 k 大于数组中不同元素的数量。 在这种情况下，左指针永远不会移动，并且每个子数组都是有效的。 该算法仍然正确地累积完整的 n(n+1)/2 结果，因为窗口始终保持有效而不会收缩。 

更微妙的情况是交替序列，例如`1 2 1 2 1 2`，窗口反复达到阈值并且几乎保持有效。 频率图确保重新进入窗口的元素被正确地重新激活，而无需重复计算不同值，并且不同计数器在所有扩展和收缩过程中保持一致。
