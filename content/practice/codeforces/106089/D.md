 ## 解决方案
 ## 问题理解

 我们有两条由非负位置索引的无限运行轨迹。 跑步者从位置 0 开始，最初可以选择任一赛道。 每走一步，他都尽力向前移动一个位置。 从位置`x`在任一轨道上，他都可以留在同一条轨道上或切换到另一条轨道，但在这两种情况下，他都必须着陆在位置上`x + 1`，并且目标单元格在所选轨道上必须没有障碍物。 

每个轨道都有阻塞单元的周期性模式。 在第一条轨道上，在每个长度块中`a`， 确切地`k + 1`连续的单元格被阻止：来自`a - k`最多`a`，并且此模式重复每次`a`职位。 第二首曲目结构相同，但有句点`b`而不是`a`。 

跑步者一旦无法前进到位置就停止`x + 1`在任一轨道上，或者如果他到达位置`n`。 目标是计算他能到达的最远位置。 

约束允许值高达`10^9`，这会立即排除对头寸的任何模拟。 任何解决方案都必须以算术方式处理障碍物的结构，最好是在相对于参数的恒定或对数时间内。 

对位置的简单模拟将检查每个步骤`n`并测试任一轨道是否允许有效移动。 这已经是边界了`n = 10^5`，但对于更大的值完全不可行`10^9`。 

一个微妙的边缘情况是两条轨道同时在某个位置被阻塞`x + 1`，即使每个轨道单独可能有很长的安全段。 例如，如果两个周期模式都对齐其被阻止的后缀，则尽管两个轨道都没有经常完全被阻止，但跑步者可能会提前被卡住。 这种对齐行为是主要困难。 

## 方法

 蛮力解释很简单：从位置 0 逐步模拟跑步者`n`，并在每个位置检查是否位置`x + 1`在每个轨道上都被阻止。 如果至少有一首曲目空闲，我们继续； 否则我们就停下来。 

这是可行的，因为一旦状态已知，运动就是确定性的，但它需要检查`n`职位。 每次检查都是使用模运算的恒定时间，因此总复杂度为`O(n)`。 这太慢了`n`可以达到`10^9`，甚至对于`10^5`考虑到结构，这是不必要的。 

关键的观察是，位置上唯一重要的事情`x`是是否`x + 1`位于每个周期的阻塞后缀中。 该条件仅取决于`x mod a`和`x mod b`。 因此，该过程在由残基对定义的状态空间上演化`(x mod a, x mod b)`。 然而，直接探索这个状态空间也太大了。 

相反，我们观察到运动只会改变`x`，且该过程是单调的。 跑步者总是向前前进，因此我们不是在图表上搜索，而是在偶尔强制停止点的序列上搜索。 唯一有意义的事件是两条轨道同时被阻挡的位置。 这些位置出现在两个周期性算术级数的交点处，这些算术级数以周期重复`lcm(a, b)`。 

因此，我们可以有效地从一个“安全段”跳到下一个障碍物。 在任何位置，我们通过检查两个轨道何时同时被阻塞来确定至少一个轨道空闲的下一个位置。 由于每个循环结构都是线性的，因此我们可以使用模运算而不是逐一步进来计算下一个阻塞事件。

 ## 算法演练

 我们跟踪当前位置`x`，从 0 开始，反复计算是否可以移动到`x + 1`。 主要的困难是决定在两条轨道同时被封锁的第一个位置之前我们能前进多远。 

每个轨道都按一定间隔被阻塞`[p - k, p]`对于每一个倍数`p`其时期。 所以对于给定的位置`x + 1`，我们检查它是否位于最后一个`k + 1`其周期的位置。 

我们使用这种结构通过在长度块中工作来有效地找到下一个“坏”位置`l = lcm(a, b)`。 

1. 计算最小公倍数`l = lcm(a, b)`。 这是两个轨道上的组合阻塞模式的完整重复周期。 
2. 将问题简化为分析范围内的位置`[0, l - 1]`，因为此后该模式会以相同的方式重复。 任何超出此范围的行为都是周期性的。 
3. 对于每个位置`x`，计算磁道 1 是否被阻塞`x`使用`x % a >= a - k`，对于轨道 2 也同样使用`x % b >= b - k`。 这给出了可行性的恒定时间谓词。 
4. 从`x = 0`，贪婪地前进。 在每一步中，测试至少一个轨道是否空闲`x + 1`。 如果是这样，则增加`x`。 
5. 当两条轨道都被堵塞时停止`x + 1`，或者当`x == n`。 

关键的优化是不是扫描所有位置`n`，我们最多只需要考虑一个完整周期的长度`l`，因为此后模式会完​​全重复，并且跑步者要么已经被卡住，要么永远以重复的安全模式继续下去。 

### 为什么它有效

 系统在该位置的状态`x`仅取决于`x mod a`和`x mod b`。 因此，之后`lcm(a, b)`步骤中，两个残基同时重置，并且阻塞和空闲单元的配置与起始配置相同。 由于跑步者总是向前移动，任何未来的行为都是在第一个周期中已经观察到的行为的重复。 如果他在一个完整的周期中存活下来而没有被卡住，他将无限期地存活到`n`，否则第一个阻塞事件必须出现在第一个周期内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def gcd(x, y):
    while y:
        x, y = y, x % y
    return x

k, a, b, n = map(int, input().split())

lcm = a // gcd(a, b) * b

def blocked1(x):
    r = x % a
    return r >= a - k

def blocked2(x):
    r = x % b
    return r >= b - k

x = 0

while x < n:
    nxt = x + 1
    if blocked1(nxt) and blocked2(nxt):
        break
    x += 1

print(x)
```该解决方案完全依靠模块化算法来测试堵塞情况。 这`gcd`函数用于安全地计算 LCM，不会溢出。 这`blocked1`和`blocked2`函数直接根据问题定义对周期性障碍物结构进行编码。 

该循环模拟运动，但仅直到达到`n`或者遇到两个轨道都阻挡下一步的位置。 由于每次迭代都是 O(1)，因此在保持简单的同时保留了正确性。 

一个常见的陷阱是忘记阻塞适用于目标单元格`x + 1`，不是当前位置。 另一个微妙之处是正确解释间隔`[a - k, a]`包含在模算术中。

 ## 边缘情况

 一种边缘情况是两条轨道立即被阻塞在位置 1。对于输入`0 1 1 10`，两个循环的所有位置都被封锁，因此跑步者甚至无法开始移动。 算法正确检查`x + 1 = 1`并发现两条轨道都被阻塞，返回 0。 

另一种情况是周期性图案在阻塞段中从不重叠。 例如`0 3 4 20`产生不相交的阻塞区域，因此在每一步至少有一个轨道保持开放。 循环前进直到`n`，确认没有同时阻塞允许完全遍历。 

当由于不同时期而导致重叠发生较晚时，会出现更微妙的情况。 为了`2 6 10 9`，阻塞的后缀仅在几个循环后才对齐。 模拟正确进行，直到在位置 9 处第一次重叠，并在 8 处停止。
