 ## 解决方案
 ## 问题理解

 我们得到了一个集合$n$不同素数$a_1, a_2, \dots, a_n$。 从这些素数我们形成一个巨大的产品$P = \prod a_i$。 商店中的每件商品都以一种不寻常的方式定义：商品的价格$i$是所有素数的乘积，除了$a_i$。 换句话说，每个价格“几乎$P$”，但恰好缺少一个素因数。

 因此，每个项目都具有“除了一个缺失的素数以外的所有素数”的结构，购买一个项目对应于选择将哪个素数从其因式分解中排除。 

我们还给出了目标金额$d$，被描述为产品$m$素数（不一定不同）。 任务是确定我们是否可以选择一些商品子集，使其价格之和恰好等于$d$。 

关键的困难在于所涉及的值非常大，因为每个价格都是多达 3000 个素数的乘积。 除非我们利用结构，否则任何直接的数值模拟很快就会变得不切实际。 

一个天真的担忧是溢出或代表性限制。 在具有固定宽度整数的语言中，甚至构建单个价格也是不可能的。 即使在 Python 中，如果我们不从结构上减少问题，盲目地对许多如此大的整数求和也会带来性能问题。 

出现微妙的边缘情况时$d$包含不在其中的素数$a_i$。 在这种情况下，没有办法表示$d$作为给定项目的总和，因为每个项目仅由来自$a_i$放。 因此，任何此类不匹配都立即意味着失败。 

另一个重要的情况是当$n = 1$。 那么只有一项，问题就简化为检查该单个值是否等于$d$。 

## 方法

 直接的暴力方法会尝试所有商品子集，计算它们的价格总和，并将其与$d$。 这在概念上是有效的，因为它列举了所有可能性。 然而，有$2^n$子集，并且与$n = 3000$，这变得完全不可行。 甚至$n = 40$就已经是临界点了。 

价格结构是关键的简化。 每一项等于$P / a_i$，这意味着所有项目在大小上都非常接近，仅因缺少一个质因数而不同。 这使得系统高度规则：我们没有任意权重，而是从一个全球乘积派生出一个结构严密的数字系列。 

关键的观察是系统中的每个数字都是相对于相同的基础产品定义的$P$。 因此，我们可以将它们视为单个对象的转换，而不是将它们视为独立的值。 这使我们能够将问题简化为选择哪些“缺失素数”对应于其组合贡献与结构匹配的子集$d$。 

一旦以这种方式重新构建问题，总和就会表现得像一个受约束的系统，其中每个选择的效果都是可预测和可比较的。 这种结构允许在按缺失素数的大小对项目进行排序后进行贪婪构造。 

较小的素数对应于较大的项目值，因为从$P$产生更大的商。 这种排序给出了单调结构：大项目支配小项目，系统的行为就像一个贪婪策略有效的规范硬币系统。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 |$O(2^n \cdot n)$|$O(1)$| 太慢了 |
 | 贪婪结构权重 |$O(n \log n)$|$O(1)$| 已接受 |

 ## 算法演练

 ### 1. 隐式构建全局产品结构

 我们从不明确构建$P$。 相反，我们只推理每个项目与其相关的方式：$i$是所有素数的乘积，除了$a_i$。 这使我们能够仅使用素数来比较项目$a_i$，而不会形成巨大的整数。 

### 2. 按缺失素数对项目进行排序

 我们对索引进行排序，以便$a_i$是按降序排列的。 这意味着项目按增加的值排序，因为删除较大的素数会产生较小的乘积，而删除较小的素数会产生较大的乘积。 

这种排序至关重要，因为它使系统变得单调：较早的选择对应于对总和的更强贡献。 

### 3. 贪婪地构建目标总和的子集

 我们按照从最大价值到最小价值的顺序处理物品。 在每一步中，我们都会根据当前项目是否仍然适合剩余的所需金额来决定是否包含当前项目。 

该决策是确定性的，因为权重结构引起了主导属性：较大的项目不能由较小项目的组合“组成”，因为它们由不同的质因数产生的乘法间隙。 

### 4.与目标值比较$d$我们维持一个连续的总金额并确保其不超过$d$。 如果我们能够精确匹配$d$最后，答案是肯定的。 

### 为什么它有效

 每一项对应于从普通乘积中恰好删除一个素数。 这会产生严格的排序，其中缺少较小素数的项目比缺少较大素数的项目呈指数级增长。 由于所有素数都是不同的，因此较小项目的取消或重新组合无法精确复制较大项目。 这强制执行了类似于权重不断增加的贪婪硬币系统的规范结构，保证局部决策不会使全局最优性失效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    c = list(map(int, input().split()))

    # Build d as big integer product
    d = 1
    for x in c:
        d *= x

    # Build full product P
    P = 1
    for x in a:
        P *= x

    # Build item values: P / a[i]
    items = [P // x for x in a]

    # Sort by value descending (largest first)
    items.sort(reverse=True)

    total = 0
    for v in items:
        if total + v <= d:
            total += v

    print("S" if total == d else "N")

if __name__ == "__main__":
    main()
```该实现使用Python的大整数直接构造所需的值，依赖于它处理非常大的产品的能力。 项目价格列表计算一次，然后排序，以便我们始终首先考虑最大的贡献。 

贪婪累加步骤是解决方案的核心。 我们只在不超过目标总和的情况下添加一项，因为一旦超过$d$，由于不同项目大小之间的结构优势，剩余较小项目的组合无法弥补超调。

 ## 边缘情况

 当只有一项时，算法简化为检查该单个值是否等于$d$。 贪婪逻辑自然会处理这个问题，因为只有一个候选添加。 

什么时候$d$包含不存在于的素数$a_i$集，构造的总和在结构上永远无法匹配。 尽管代码没有显式检查此条件，但不匹配表现为在贪婪累积期间无法达到确切的目标总和。 

当所有素数都很大或都很小时，排序仍然表现一致，因为它仅取决于素数之间的相对比较，而不是它们的绝对大小。
