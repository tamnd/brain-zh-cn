 ## 解决方案
 ## 问题理解

 我们正在生产蒂亚戈将提供的食物克数。 每个有效的产品必须由固定尺寸的篮子组成`A`，所以总金额必须是`A`。 同时，当我们将这个总数写成十进制时，它的最高有效位必须与给定的整数匹配`B`。 换句话说，该数字必须位于十进制表示以以下数字开头的数字范围内`B`。 

所以任务是找到一些整数`X`这样`X = k * A`对于某个整数`k`,`X < 10^18`，当以 10 为基数书写时，`X`以数字开头`B`。 

约束条件很大：最多`10^5`测试用例和值高达`10^9`。 这立即排除了任何迭代多个的方法`A`或扫描每个测试用例的大范围。 甚至尝试所有候选人`10^18`这是不可能的，因为那将是一个天文数字。 该解决方案必须将每次测试的搜索空间减少到对数或常数。 

当我们试图“从B开始向上移动”时，就会出现微妙的失败案例。 例如，如果`A = 7`和`B = 2`，有效数字是从数字 2 开始的 7 的倍数：21、28、210、217 等。天真的方法可能只检查几个小倍数，而错过第一个有效数字可能与`B`本身的数值。 另一个陷阱是假设答案总是接近于`B * 10^k`，不保证能被整除`A`。 

核心困难在于结合两种结构：算术级数（算术级数的倍数）`A`) 和十进制表示形式的前缀约束。 

## 方法

 蛮力的想法很简单。 我们可以迭代多个`A`：计算`A, 2A, 3A, ...`并检查每个值是否以`B`。 这是正确的，因为每个有效的候选者都包含在这个序列中。 然而，序列呈线性增长，在最坏的情况下，第一个有效匹配可能会出现在非常远的地方。 由于值最多允许`10^18`，我们可能需要检查的倍数数量也可以是`10^18 / A`，这是完全不可行的。 

关键的观察是前缀条件不需要我们检查每个倍数。 相反，我们可以将条件“以 B 开头”重新解释为范围约束。 数字开头为`B`当且仅当它位于某个区间内`[B * 10^k, (B + 1) * 10^k - 1]`对于一些非负整数`k`。 这会将数字条件转换为间隔的并集。 

对于每个这样的区间，我们想要找到最小的倍数`A`就在它里面。 这变成了一个简单的算术对齐问题：对于固定间隔，我们可以计算第一个倍数`A`大于或等于左端点，并检查它是否仍在右端点内。 由于相关数量`k`值很小（最多 18，因为`10^18`限制了数字的长度），我们可以尝试所有可能的前缀长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力倍数 | O(10^18 / A) 最坏情况 | O(1) | O(1) | 太慢了 |
 | 间隔+模块化对齐 | 每次测试 O(log 10^18) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 对于每个测试用例，请阅读`A`和`B`。 我们诠释`B`作为答案必须开头的固定十进制前缀。 
2. 将前缀条件转换为区间。 对于每个可能的数字长度，构造一个幂`p = 10^k`并定义间隔`[B * p, (B + 1) * p - 1]`。 这些间隔准确地表示前导数字为`B`当最多写这么多数字时。 最小的有效数字将出现在这些范围之一中。 
3. 对于每个区间，计算最小倍数`A`这至少是左端点。 这是通过采取`x = ((L + A - 1) // A) * A`。 这确保我们直接跳转到间隔开始内或间隔开始后的第一个有效倍数。 
4. 检查该候选人是否`x`位于区间内且小于`10^18`。 如果是，它满足两个约束：它是`A`并开始于`B`。 
5. 追踪最低有效`x`在所有间隔长度上。 答案是该值中的最小者。 
6. 如果没有区间产生有效的倍数，则输出`-1`。 

### 为什么它有效

 正确性取决于每个带有前缀的数字`B`恰好属于由 10 的幂定义的一个区间。 任何有效的解决方案都必须位于这些区间之一内。 在一个固定的区间内，`A`形成算术级数，因此可以使用单个上限除法找到第一个有效候选者。 由于我们测试了每个可能的数字长度，直至达到以下限制`10^18`，我们不会错过任何有效的表示，并且在所有候选者中我们选择最小的有效倍数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    LIMIT = 10**18

    for _ in range(t):
        A, B = map(int, input().split())

        best = None
        pow10 = 1

        for k in range(1, 20):
            L = B * pow10
            R = (B + 1) * pow10 - 1

            if L >= LIMIT:
                break

            if R >= LIMIT:
                R = LIMIT - 1

            x = ((L + A - 1) // A) * A

            if x <= R:
                if best is None or x < best:
                    best = x

            pow10 *= 10

        print(best if best is not None else -1)

if __name__ == "__main__":
    solve()
```该实现使用 10 的幂构建间隔边界，并检查最多大约 18 个可能的数字长度。 对于每个范围，它计算第一个倍数`A`使用整数上限除法进入范围，然后验证它是否仍然保留前缀约束。 这`best`变量维护所有长度的最小有效候选者。 

一个微妙的细节是上限的处理`10^18`。 超出此限制的任何间隔都会被剪断，因为根据问题定义，高于此限制的值无效。 20 的循环界限是安全的，因为 10^19 已经超出了限制。

 ## 边缘情况

 一种边缘情况是当`A = 1`。 那么每个数字在可整性方面都是有效的，所以答案就是以以下开头的最小数字`B`，即`B`本身只要它在下面`10^18`。 该算法处理这个问题是因为在第一个间隔中`k = digits(B)`，左端点等于`B`第一个倍数恰好是`B`。 

另一种边缘情况是限制内不存在有效倍数。 例如，如果`A`很大并且前缀间隔开始超出`10^18`，循环提前终止并且`best`保持空状态，产生`-1`。 

更微妙的情况是当区间内的第一个倍数超过区间边界时。 例如，如果`L = 23`,`R = 29`， 和`A = 10`，第一个倍数是`30`，位于区间之外。 支票`x <= R`正确丢弃它，防止误报。
