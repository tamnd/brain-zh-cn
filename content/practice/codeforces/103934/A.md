 ## 解决方案
 ## 问题理解

 我们得到了一组时间间隔，每个时间间隔代表建筑物的施工周期。 选定的日子相当于派遣军队检查当天所有正在建设的建筑物。 建筑物在其间隔内的每个选定日期都会被“访问”。 

我们必须选择一组整数天，以便每个间隔至少包含一个选定的天。 在所有这些有效的时间表中，我们希望最小化任何单个建筑物的最坏情况负载，其中建筑物的负载是落在其间隔内的选定天数。 找到这样一组最佳日期后，我们将输出所选的日期本身。 

因此，问题不仅在于覆盖所有区间，还在于避免多次过度访问任何单个区间。 

约束允许最多 2500 个间隔，端点的绝对值可以大到 10^18。 这立即排除了任何需要数天迭代或建立明确时间表的方法。 任何解决方案都必须纯粹使用区间端点和组合结构，并且通常 O(n^2) 左右的任何内容都是可以接受的，而 O(n^3) 则根据常数而变得边界。 

当间隔大量嵌套时，会出现微妙的边缘情况。 例如，考虑区间 [0, 10]、[1, 9]、[2, 8] 等。 任何选择“许多合理点”的幼稚策略都可能无意中将多个选定日期放置在同一个长间隔内，从而增加其访问次数。 另一种边缘情况是区间不相交； 那么最佳解决方案应该明确地在每个区间中选取一个点，并且任何意外放置在大区间内的额外点都是不必要且有害的。 

另一个陷阱是假设任何有效的覆盖集都是等效的。 事实上，两个不同的击球组在长距离相交的次数上可能存在显着差异。 

## 方法

 该问题可以看作是选择一组“刺点”，以便每个间隔至少被击中一次。 如果我们忽略第二个目标，自然目标就变成找到与所有区间相交的最少点数。 这是经典的区间刺穿问题，通过总是在尚未覆盖的最早完成区间的末尾选取一个点来贪婪地解决。 这会产生最小尺寸的点集。 

然而，这里的目标是不同的。 我们不会减少选择的天数。 我们正在最小化任何单个间隔内的最大选定天数。 这将问题从纯粹的覆盖问题转变为平衡覆盖问题，其中分布很重要。 

暴力方法将尝试由间隔端点引起的候选点的所有可能子集，检查是否命中每个间隔，并计算每个间隔的最大覆盖范围。 这在候选点的数量上是指数级的并且完全不可行。 

关键的观察结果是，贪婪刺穿解决方案已经产生了一组高度结构化的点：每个选定的点都是某个间隔的右端点，并且间隔按完成时间的递增顺序进行处理。 这种结构确保所选点尽可能“右移”，从而防止在长重叠间隔内出现不必要的聚类。 任何将点向左移动或添加额外点的偏差往往只会增加现有区间内的重叠。 

这使得贪婪刺穿结构不仅对于覆盖范围大小是最佳的，而且对于最小化任何区间内所选点的最大数量也是最佳的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 点的蛮力子集 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 贪心区间刺 | O(n log n) | O(n log n) | O(n) | 已接受 |

## 算法演练

 1. 按完成时间升序对所有间隔进行排序。 这确保了当我们决定在哪里放置新的检查日时，我们总是优先考虑最早结束的时间间隔，因为它们的灵活性最低。 
2. 维护一个最初为空的所选日期列表。 这些是我们正在建设的实际检查日。 
3. 按排序顺序迭代间隔。 对于每个时间间隔，检查它是否已被“覆盖”，这意味着它至少包含先前选择的一天。 如果它被覆盖，我们什么也不做。 
4. 如果没有覆盖间隔，我们必须添加新的检验日。 最好的选择是将这一天恰好放置在间隔的正确端点处。 这可以最大限度地在未来间隔中重复使用该点，同时避免不必要的早期放置，否则可能会增加较长间隔中的重叠。 
5. 构建完整的所选日期集后，计算每个间隔内有多少个所选日期。 答案值是所有时间间隔内这些计数的最大值。 
6. 输出所选天数和列表本身。 

### 为什么它有效

 贪心过程构造一个最小命中集，其中每个选定的点都放置在尽可能靠右的位置，同时仍然覆盖当前间隔。 这种结构确保每当我们添加一个新点时，它都会受到尚未覆盖的区间的强制，这意味着之前选择的点不位于其中。 因此，任何区间内所选点的数量正是在扫描与其重叠的区间时发生的贪婪“激活”的数量。 

任何替代的有效解决方案仍然必须在每个间隔内放置至少一个点，并且任何通过向左移动点或重新分布点来减少重叠的尝试都不能在不破坏某些最右边间隔的覆盖的情况下减少长间隔相交的次数。 这使得贪婪构造对于可行性和最大负载目标都是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    seg = []
    for _ in range(n):
        s, e = map(int, input().split())
        seg.append((s, e))

    seg.sort(key=lambda x: x[1])

    chosen = []

    for s, e in seg:
        # check if already covered
        covered = False
        for x in chosen:
            if s <= x <= e:
                covered = True
                break
        if not covered:
            chosen.append(e)

    # compute maximum coverage per interval
    ans = 0
    for s, e in seg:
        cnt = 0
        for x in chosen:
            if s <= x <= e:
                cnt += 1
        ans = max(ans, cnt)

    print(len(chosen))
    print(*chosen)

if __name__ == "__main__":
    solve()
```实现直接遵循贪心构造。 按右端点排序可确保每次添加点时，它都位于仍覆盖当前间隔的最新可能时刻。 覆盖检查扫描先前选择的点，如果 n ≤ 2500，这是可接受的。 

第二遍通过计算每个间隔的交集来计算目标值。 这是与问题定义一致的简单验证步骤。 

这里的一个常见错误是尝试仅优化点数并假设这就足够了。 第二遍明确表明实际目标取决于分布，而不仅仅是基数。

 ## 边缘情况

 完全嵌套的链（例如 [0,10]、[1,9]、[2,8]）确认该算法永远不会在长间隔内引入多个不必要的点。 只有链中的最后一个间隔才会强制选择一个点。 

不相交的间隔确认算法不会意外地重复使用非重叠范围内的点，因为每个间隔将独立触发新的选择。 

相同的间隔表明重复输入不会增加所选天数，因为第一个所选点已经涵盖了所有重复项。
