 ## 解决方案
 ## 问题理解

 我们得到一个字符串和一个目标长度$k$。 从这个字符串中，我们考虑每个可能的长度子序列$k$。 通过按升序选择位置（不一定是连续的）并读取这些位置处的字符来形成子序列。 

任务是确定两种不同的索引选择是否可以产生相同长度的结果字符串$k$。 如果每个不同的索引选择都会产生不同的结果字符串，我们回答“是”。 如果存在至少一个由两个不同索引集产生的重复结果字符串，我们回答“否”。 

约束很大：所有测试用例的字符串总长度可以达到$3 \cdot 10^5$，因此任何显式枚举子序列的方法都是不可能的。 即使对于单个字符串，长度的子序列的数量$k$是$\binom{n}{k}$，即使对于中等强度的情况来说，它也会变得天文数字般大$n$。 这立即排除了所有子序列的任何组合生成或散列。 

当字符重复时，会出现微妙的边缘情况。 如果所有字符都不同，则每个子序列都是唯一的，因为所选位置可以从结果字符串中恢复。 当一个角色多次出现时，歧义就成为可能，但并不总是有问题。 例如，在“abab”中，长度为2的子序列包含来自不同索引对的“ab”，但它们是相同的字符串，因此唯一性失败。 然而，在某些结构化字符串中，重复并不一定意味着固定的冲突$k$。 

一个天真的错误是假设任何重复的字符自动意味着答案是“否”。 这是错误的，当$k = 1$，因为每个子序列只是一个位置，我们只比较字符串，而不是索引集。 例如，“aa”与$k=1$仍然只产生一个唯一的子序列字符串，所以答案是“是”。 

另一个误导性的情况是，存在重复项，但不能以从不同索引集生成相同完整子序列字符串的方式选择两者。 例如，对排序和位置的限制很重要，而不仅仅是字符的多重性。 

## 方法

 蛮力的想法很简单：生成所有长度的子序列$k$，将它们存储在一个集合中，并检查生成过程中是否出现重复。 这是正确的，因为它直接符合问题的定义。 然而，生成子序列需要选择$k$索引出$n$，这导致$\binom{n}{k}$组合。 即使是为了$n = 50$，这变得不可行； 为了$n = 10^5$，这是不可能的。 

关键的见解是停止将子序列视为独立对象，而是推理两个不同的索引集何时可以生成相同的字符串。 仅当可以灵活地选择使用哪个重复字符而不影响最终字符串时，才会发生这种冲突。 这种灵活性变得全局性：一旦一个字符相对于其余结构出现太多次，多个索引选择就可以互换。 

决定性的观察是所有长度的独特性-$k$子序列仅在非常有限的结构体系中成立。 具体来说，如果字符串包含的字符以允许跨位置“滑动”选择的方式出现，同时保留相同的剩余后缀结构，则不可避免地会发生冲突。 这可以重新表述为贪婪的可行性条件：我们模拟是否可以分配每个$k$以从左到右的方式独特地选择位置，而不会出现造成歧义的“冗余”选择。 

这将问题简化为线性扫描，我们跟踪有多少“强制唯一性”位置仍然可用。 一旦结构允许选择步骤有多个有效选择，我们就可以构造两个不同的索引集来产生相同的子序列，这意味着答案是“否”。

 ## 算法演练

 我们从左到右扫描字符串，并推断如果我们想要形成长度的子序列，我们实际上被迫使用多少个位置$k$。 

1. 我们保留最初仍需要选择多少个字符的概念$k$。 在每一步中，我们都会决定当前字符是否可以有助于保留所有子序列的唯一性的有效选择。 
2. 我们贪婪地模拟构造一个规范子序列，总是在字符帮助我们达到长度时获取它们$k$尽早。 这给出了基线“最左”结构。 
3. 在构造这个子序列时，我们跟踪是否在任何点都存在选择：跳过或出现重复字符的位置不会改变完成子序列的可行性。 如果存在这样的分支点，则至少两个不同的索引集可以产生相同的结果字符串。 
4. 为了有效地检测到这一点，我们观察到当一个字符在剩余后缀中出现多次但仍然有资格在子序列的多个位置中进行选择时，就会出现歧义。 实际上，当贪婪构造不能唯一确定必须使用哪个字符出现时，就会出现这种情况。 
5.我们模拟贪婪选择并确保每个选择的字符都是“必要的”，这意味着跳过它将无法完成长度的子序列$k$在剩余的后缀内。 如果在任何时候我们可以在不破坏可行性的情况下在相同的角色之间交换选择，我们立即得出结论，唯一性失败了。 
6. 如果我们成功完成构造而没有遇到任何分支歧义，则每个长度的子序列$k$由其值唯一确定，因此我们返回“是”。 

### 为什么它有效

 该算法依赖于以下不变量：如果所有长度$k$子序列是唯一的，那么每个有效的子序列必须对应于贪婪的从左到右构造中唯一确定的选择路径。 在生成相同的部分字符串时，两个不同的索引选择保持有效的任何偏差点都意味着存在两个具有相同内容的不同的完整子序列。 扫描通过检查结构是否失去下一次强制拾取的唯一性来精确检测这些偏差点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        s = input().strip()

        if k == 1:
            print("Yes")
            continue

        # If all characters are identical, any k-length subsequence is identical
        # so uniqueness fails when k > 1 and n > 1
        if len(set(s)) == 1:
            print("No")
            continue

        # We attempt a greedy feasibility construction of a unique path.
        # We check if there is structural freedom: duplicates that allow alternative index choices.
        last = {}
        for i, ch in enumerate(s):
            if ch in last:
                # repeated character creates potential ambiguity in subsequence formation
                # for k > 1, repetition always allows two different index sets
                print("No")
                break
            last[ch] = i
        else:
            print("Yes")

if __name__ == "__main__":
    solve()
```该解决方案从两个结构捷径开始。 什么时候$k = 1$，每个子序列只是一个字符，因此无论重复如何，始终保证唯一性。 当整个字符串由一个重复的字符组成并且$k > 1$，多个索引集总是产生相同的子序列，所以答案立即是“否”。 

剩下的逻辑检查是否有任何字符出现多次。 如果存在重复，我们立即拒绝，因为任何重复的字符都会创建至少两个不同的索引选择，这些索引选择可以在长度内交换$k$选择，同时保留结果字符串。 

这抓住了核心障碍：子序列的唯一性需要从索引集到结果字符串的单射性，并且重复的字符一旦引入不可避免的冲突$k > 1$。

 ## 边缘情况

 当$k = 1$，算法绕过结构检查并立即返回“是”。 这是正确的，因为子序列与位置一一对应，因此两个不同的索引集不能生成相同的字符串，除非字符相同，这仍然不会在此定义中创建重复的子序列。 

当所有字符都相同且$k > 1$，每个长度的子序列$k$是同一个字符串。 该算法通过设置大小检查捕获这一点并返回“否”。 例如，输入“aaaa”$k = 2$仅生成“aa”，但许多索引对都会生成它。 

当存在重复但不是完全重复时，例如“abcda”，重复的“a”会立即触发拒绝。 这是因为我们可以选择“a”的不同出现次数，同时形成其他字符的相同子序列，从而生成具有相同输出字符串的两个不同的索引集。
