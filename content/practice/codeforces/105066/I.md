 ## 解决方案
 ## 问题理解

 给定一个整数数组，然后重复应用由非负整数参数控制的变换$x$。 对于固定选择$x$, 每个数组元素$a_i$与$x$，并将所有这些结果相加。 这会产生一个值$$S(x) = \sum_{i=1}^{n} (a_i \oplus x)$$任务不是计算这个$x$，但要了解所有可能的值$S(x)$作为$x$范围覆盖所有非负整数，然后计算有多少不同结果落在该区间内$[l, r]$。 

关键的难点在于$x$不受输入限制，因此该函数$S(x)$是在无限域上定义的，但其结构仅取决于$x$，并且仅达到出现在任何$a_i$。 既然所有$a_i \le 10^5$，只有大约 17 位在基值中起作用，而较高位仅统一影响 XOR 模式。 

这些限制迫使我们远离任何直接枚举$x$。 和$n$最多$10^5$和$x$潜在的巨大，甚至是计算能力$S(x)$为所有人$x$最多$r$是不可能的。 解决方案必须压缩所有的影响$x$成一个小的结构化表示。 

假设单调性会产生一个微妙的问题。 一个天真的猜测是$S(x)$顺利地改变与$x$，但异或破坏了单调性。 例如，对于单个元素$a = 5$，序列$a \oplus x$振荡：$5,4,7,6,1,0,3,2,\dots$，因此任何关于可达值形成连续区间的假设都是错误的。 

另一个陷阱是独立处理位，而不考虑 XOR 如何与阵列中的固定总体计数相互作用。 该函数在组合意义上是每比特线性的，但取决于有多少$a_i$设置每个位。 

## 方法

 蛮力方法会迭代所有可能的方法$x$在$[0, r]$, 计算$S(x)$，并收集不同的值。 每次评估费用$O(n)$，导致$O(nr)$最坏情况下的操作。 自从$r$可以大到$10^{18}$，这是不可行的。 

关键的观察是异或与固定$x$独立影响每一位。 对于固定位位置$k$， 无论$(a_i \oplus x)$有位$k$设置仅取决于$a_i$的一点$k$和$x_k$。 这使我们能够将总和重写为按位贡献的函数。 

对于每一位$k$， 让$c_k$是元素的数量，其中$k$第 位$a_i$是 1。那么对于给定的$x$, 位的贡献$k$总和仅取决于是否$x_k$是 0 或 1：

 如果$x_k = 0$，贡献是$c_k \cdot 2^k$。 如果$x_k = 1$，就变成$(n - c_k) \cdot 2^k$。 这将问题转化为每个比特独立地选择两个贡献之一。 

因此，每一个$S(x)$对应于为每一位选择两个值之一。 所有可能和的集合正是这些跨相关位的独立选择的所有组合的集合。 由于最多有 17 位，所以我们最多得到$2^{17} = 131072$可能的金额。 

我们可以使用简单的 DFS 或位迭代构造生成所有总和，对它们进行排序，然后回答范围查询$[l, r]$通过二分查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解结束$x$|$O(nr)$|$O(1)$| 太慢了 |
 | 比特DP超过贡献|$O(2^B + nB)$|$O(2^B)$| 已接受 |

 ## 算法演练

 我们压缩所有的效果$x$分成每比特独立的二进制选择。 

1. 计算每个位设置了多少个数组元素。 对于每个位位置$k$, 计算$c_k = |\{i : a_i \text{ has bit } k\}|$。 这隔离了 XOR 将如何全局影响该位。 
2. 对于每一位$k$，预先计算两个可能的贡献：当$x_k = 0$，该位贡献$c_k \cdot 2^k$，以及当$x_k = 1$，它贡献$(n - c_k) \cdot 2^k$。 这捕获了切换该位的全部效果$x$。 
3. 通过迭代位来构建所有可能的和。 从包含 0 的集合开始。对于每个位，通过将该位的两个贡献之一添加到每个现有的部分和来扩展当前集合。 这是有效的，因为比特可以累加且独立地做出贡献。 
4. 处理完所有相关位后（直到任意位中存在的最高位）$a_i$），我们获得了完整的可达值集$S(x)$。 对此列表进行排序。 
5. 使用二分查找来计算有多少个值$[l, r]$。 

### 为什么它有效

 每个位都是独立贡献的，因为异或运算按位进行，位置之间没有进位。 对于固定位，贡献仅取决于是否$x_k$是否翻转该位，并且该决定不与任何其他位相互作用。 因此每个有效的$x$对应于每比特一个贡献的唯一选择，并且每个这样的选择对应于一些$x$。 这在位选择和和之间建立了一对一的映射，确保完整性并且除了来自不同位模式的相同和之外没有重复。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, l, r = map(int, input().split())
    a = list(map(int, input().split()))

    MAXB = 0
    for v in a:
        MAXB = max(MAXB, v.bit_length())

    cnt = [0] * (MAXB + 1)

    for v in a:
        for b in range(MAXB + 1):
            if v >> b & 1:
                cnt[b] += 1

    # possible contributions per bit
    choices = []
    for b in range(MAXB + 1):
        bit_val = 1 << b
        c = cnt[b]
        choices.append((c * bit_val, (n - c) * bit_val))

    # build all possible sums
    sums = {0}
    for zero_val, one_val in choices:
        new = set()
        for s in sums:
            new.add(s + zero_val)
            new.add(s + one_val)
        sums = new

    arr = sorted(sums)

    # count in range [l, r]
    import bisect
    return bisect.bisect_right(arr, r) - bisect.bisect_left(arr, l)

print(solve())
```该实现首先将每个位压缩为二进制选择：要么处理$x_k = 0$或者$x_k = 1$。 集合构造通过用每个位的贡献扩展先前的部分和来迭代地累积所有可能的和。 最后的排序步骤可以实现高效的范围计数。 

一个微妙的点是我们从未明确构造$x$。 相反，我们直接构建所有可能的结果$S(x)$。 这避免了处理潜在的巨大价值$x$，这是无关紧要的，因为只有位模式才影响到最大位$a_i$。

 ## 边缘情况

 关键的边缘情况是数组中的每个数字的所有位都相同。 在这种情况下，翻转任何位$x$不会改变 1 和 0 之间的平衡，因此多个配置会崩溃为相同的总和。 该算法自然地处理这个问题，因为集合插入会删除重复项。 

另一种情况是当$l = r = 0$。 该算法正确地统计了生成集中是否出现 0。 由于在添加贡献之前初始状态始终为 0，因此这种情况总是会通过，无需特殊处理。 

第三种情况是只有一位位置有效时。 然后每个位都贡献一个简单的二值选择，并且最终集合的大小至多为 2。迭代构造仍然干净地应用，因为每个扩展都是独立的并且不假设多个活动位。
