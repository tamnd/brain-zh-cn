 ## 解决方案
 ## 问题理解

 我们得到一个值数组和一个固定的索引交换序列，每当金石“测试”该数组是否可以排序时，都会重复应用这些值。 重要的是，每次测试后他都会恢复阵列，因此交换序列始终在新副本上运行。 

毛毛每天都会独立地永久交换阵列中的两个位置。 修改后，Jinshi 再次运行相同的固定交换序列并检查结果数组是否按非降序排序。 

因此，每天的过程是：从当前数组开始，对索引应用已知的交换排列，并检查重新排序后的数组是否已排序。 

核心困难在于交换序列不依赖于值，它定义了固定的位置排列。 每天，我们通过交换两个位置来稍微修改底层数组，并且我们必须回答在应用固定的位置排列后，值是否最终排序。 

限制很大，最多有100000个仓位、100000个固定掉期和100000个查询。 任何重新计算交换序列的效果或重新模拟每个查询的完整过程的解决方案都将需要每个查询至少线性时间，这会导致大约 10^10 次操作，并且远远超出了时间限制。 

一种简单的方法是在每个查询后模拟 m 次交换，然后检查排序性，即每个查询的 O(n + m) 。 这立即在限制下被打破。 

更微妙的天真的尝试可能会尝试仅在每次查询后通过重建排列来重新计算，但最终仍然是 O(mq)。 

一种重要的边缘情况是交换序列很长但由于重复取消而实际上很小。 例如，交换位置 1 和 2 两次会返回同一性，但简单的模拟每次仍会将其视为两个操作，浪费工作。 

另一个棘手的情况是，当毛毛的交换影响最终排列结构中相距较远的位置时，这会使局部假设无效，除非我们仔细跟踪索引如何通过固定排列重新排列。 

## 方法

 关键的观察是 m 次交换的序列与值无关，因此它定义了索引的固定排列。 令此排列为 P，这意味着在运行所有交换后，最初位于位置 i 的值移动到位置 P(i)。 

我们可以通过从恒等位置开始并将每个交换应用为索引交换来计算此排列一次。 这为我们提供了如何重新排列位置的固定映射。 

现在考虑给定日期的阵列快照。 应用固定排列 P 后，我们可以根据固定索引顺序有效地对数组进行重新排序。 我们可以将 P 视为定义读取数组的新位置顺序，而不是考虑移动值。 

令 Q 为 P 的逆排列。那么应用交换序列后的最终数组为：

 我们按 Q(1)、Q(2)、...、Q(n) 的顺序读取位置。 

所以问题简化为：在原始数组中每天交换之后，序列是

 A[Q(1)], A[Q(2)], …, A[Q(n)] 非减？ 

这将问题转化为以固定顺序维护数组并支持底层数组中的点交换，同时检查重新排序的视图是否已排序。 

强力解决方案将在每次更新后重建序列 A[Q(i)] 并扫描它，每次查询的成本为 O(n)。 那还是太慢了。 

关键的改进是原始数组中的交换仅影响两个位置。 在 Q 有序视图中，这对应于恰好改变两个位置。 排序仅取决于相邻比较，因此仅需要检查这两个索引周围的局部邻域。

我们维护 Q 阶中相邻元素之间的“反转”数量。 每个查询仅更新固定数量的相邻对，因此我们可以在每个查询的 O(1) 内保持正确性。 

### 比较

 | 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 重新模拟掉期+每天检查| O(q(n + m)) | O(n) | 太慢了|
 | 每个查询重建重新排序的数组 | O(nq) | O(n) | 太慢了|
 | 维护 Q 序中的邻接违规 | O((n + m + q)) | O((n + m + q)) | O(n) | 已接受 |

 ## 算法演练

 ### 1. 构建由固定交换引起的排列

 我们从索引的恒等数组开始。 对于每个交换 (x, y)，我们交换此索引数组中的位置 x 和 y。 处理完所有 m 次交换后，该数组表示 P(i)，即交换序列之后索引 i 的最终目的地。 

### 2. 转换为逆序

 我们构造一个数组`rank[pos]`这样`rank[pos]`给出的位置`pos`在最终的排列顺序中。 这定义了我们必须读取原始数组以模拟所有交换效果的顺序。 

### 3. 构建初始有序序列

 我们在索引 1 到 n 上定义一个概念数组 S，其中：

 S[i] = A[ inverse_rank[i] ]。 

这表示应用固定排列后的数组。 

### 4. 计算 S 中的局部无序度

 我们计算有多少个索引 i 满足 S[i] > S[i+1]。 这个计数告诉我们序列是否已排序。 如果计数为零，则数组已排序。 

### 5. 处理每个查询（交换原始数组）

 对于每个查询交换 A 中的 l 和 r 位置：

 我们确定它们在 Q 阶中的位置，即rank[l]和rank[r]。 只有S中的这两个位置发生变化。 

我们暂时删除它们对邻接检查的贡献，交换 A 中的值，然后仅重新评估rank[l]和rank[r]周围的相邻对。 

### 6.输出结果

 如果更新后不存在相邻违规，则输出“Y”，否则输出“N”。 

### 为什么它有效

 交换序列引起的排列是固定的，因此最终检查中索引的相对顺序是恒定的。 唯一的动态成分是这些索引处的值。 

由于排序仅取决于固定顺序中的相邻比较，并且每个查询仅按该顺序修改两个位置，因此所有受影响的比较都是本地的。 在不影响相邻对的情况下，不会出现非局部反转，因此仅维护邻接违规就足够且准确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    a = list(map(int, input().split()))

    # Build permutation P from swaps
    p = list(range(n))
    for _ in range(m):
        x, y = map(int, input().split())
        x -= 1
        y -= 1
        p[x], p[y] = p[y], p[x]

    # inverse permutation: rank[position] = order index
    rank = [0] * n
    for i in range(n):
        rank[p[i]] = i

    # S[i] = a[p[i]] conceptually, but we store via inverse mapping
    S = [0] * n
    for i in range(n):
        S[i] = a[p[i]]

    def bad(i):
        return 1 if 0 <= i < n - 1 and S[i] > S[i + 1] else 0

    bad_cnt = 0
    for i in range(n - 1):
        bad_cnt += bad(i)

    def fix(pos):
        nonlocal bad_cnt
        for i in (pos - 1, pos):
            if 0 <= i < n - 1:
                bad_cnt -= bad(i)
        # recompute local position already updated outside

        for i in (pos - 1, pos):
            if 0 <= i < n - 1:
                bad_cnt += bad(i)

    out = []

    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1

        i1, i2 = rank[l], rank[r]

        # remove old contributions
        for i in (i1 - 1, i1, i2 - 1, i2):
            if 0 <= i < n - 1:
                bad_cnt -= bad(i)

        # swap in original array
        a[l], a[r] = a[r], a[l]

        # update S only at affected positions
        S[i1] = a[p[i1]]
        S[i2] = a[p[i2]]

        # add back contributions
        for i in (i1 - 1, i1, i2 - 1, i2):
            if 0 <= i < n - 1:
                bad_cnt += bad(i)

        out.append('Y' if bad_cnt == 0 else 'N')

    print(''.join(out))

if __name__ == "__main__":
    solve()
```该实现将静态排列结构与动态数组值分开。 数组`p`编码固定交换序列的效果，而`rank`让我们将原始索引映射到它们的最终顺序。 

数组`S`从未完全重建。 每个查询仅更新两个受影响的位置。 变量`bad_cnt`跟踪排列顺序中存在多少个相邻违规，并且仅围绕更改的索引进行调整。 

更新时一定要小心`bad_cnt`：每个查询最多触及每个更改索引的四个邻接位置，因此更新时间保持恒定。 周围缺少边界检查`i-1`和`i`是差一错误的常见来源。 

## 工作示例

 ### 示例 1 迹线

 我们只跟踪相关结构：有序数组 S 和邻接违规。 

| 步骤| 运营| 受影响的职位 | 坏对|
 | ---| ---| ---| ---|
 | 0 | 初始| 完整构建| 从 S | 计算
 | 1 | 交换 l=1, r=3 | 排名 1 和 3 | 更新本地|
 | 2 | 查询结果 | 无 | 检查 bad_cnt |

 第一次更新后，序列在固定排列下变得可排序，所以答案是 Y。 

随后的交换会破坏 Q 顺序中相邻元素之间必要的排序关系，因此 bad_cnt 变为非零，产生 N。 

### 示例 2 跟踪

 | 步骤| 运营| 受影响的职位 | 坏对|
 | ---| ---| ---| ---|
 | 0 | 初始| 完整构建| 计算|
 | 1 | 交换影响排序| 两个指数| 本地修复 |
 | 2 | 交换部分恢复秩序| 两个指数 | 本地修复 |

 这个例子表明，即使在中断之后，也可以在不进行全局重新计算的情况下恢复排序，因为只有局部邻接结构很重要。

 ## 边缘情况

 关键的边缘情况是查询中的两个交换位置映射到排列顺序中的相邻索引。 在这种情况下，两个位置都会影响 S 中的同一邻接对。该算法可以正确处理此问题，因为坏对在更新之前被删除并在更新后重新添加，从而防止重复计算。 

另一种情况是当 n 为 1 或 2 时。当 n = 1 时，无论交换如何，序列始终已排序。 对于 n = 2，只有一对邻接对，因此任何更新都只会切换一次比较，并且算法会正确地简化为跟踪一个布尔条件。
