 ## 解决方案
 ## 问题理解

 我们有一行$n$座位和$n$学生。 每个学生都有一个偏好的座位号，每个学生也有一个个人的不满意成本参数。 如果学生没有坐在他们想要的座位上，我们就会为该学生带来不满成本； 如果他们坐在自己喜欢的座位上，他们就不会做出任何贡献。 

任务是为每个学生分配一个独特的座位，形成座位排列，使总的不满总和尽可能小。 

所以输入定义了两个长度的数组$n$。 第一个数组告诉我们每个学生想要占据哪个座位。 第二个数组告诉我们“放错地方”该学生的代价有多大。 输出结果必须为学生分配座位以及由此产生的最小可能的总体不满意。 

约束条件$n \le 10^5$每次作业尝试立即排除任何超线性的情况。 三次甚至二次匹配方法会超出限制。 我们应该期待诸如排序或贪婪处理之类的东西$O(n \log n)$或者$O(n)$。 

一个天真的但常见的错误是假设我们可以将每个学生独立分配到他们喜欢的座位并任意解决冲突。 但这种做法失败了，因为当多名学生想要同一个座位时，谁获得座位的选择会根据他们的不满程度显着影响总成本。 

另一个微妙的边缘情况是，几个学生共享相同的首选座位，但不满意值却截然不同。 例如，如果三个学生都想要座位 1，但他们的成本是$1, 1000, 1000$，那么将高成本的学生送离他们的偏好是昂贵的，应该避免。 没有正确优先考虑这些冲突的贪婪者很容易选择次优的安排。 

最后，当所有偏好都不同时，答案几乎为零。 任何正确的解决方案都必须检测并保留此结构。 

## 方法

 一种强力解释是尝试为学生分配座位的所有排列。 对于每个排列，我们计算不在其首选座位上的学生的不满总和。 这是正确的，因为它会检查每个可能的分配。 然而，排列的数量是$n!$，甚至对于$n = 10$，这已经太大而无法计算。 

我们需要了解真正产生成本的因素。 只有当学生没有被分配到他们喜欢的座位时才会产生费用。 这重新定义了问题：我们希望在每个座位只能使用一次的约束下，最大限度地提高成功获得首选座位的学生的总“节省成本”。 

现在结构变得更加清晰了。 每个学生本质上都是对一个座位的请求，每个请求都有一个权重。 我们希望满足尽可能多的高权重请求，因为满足请求所产生的减少等于该学生的不满意成本。 

这成为学生和座位之间二分图中的最大权重匹配问题，但具有非常具体的结构：每个学生都有一个首选座位。 这将其简化为每个席位的贪婪冲突解决问题。 

关键思想是独立处理每个座位并最多分配一名学生。 如果多个学生想要同一个座位，我们应该将其提供给不满意成本最大的学生，因为满足该请求会产生最大的收益。 所有其他学生必须被分配到其他地方并支付费用。 

这将问题转化为按学生喜欢的座位对学生进行分组，并为每组选择一名代表。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n!)$|$O(n)$| 太慢了|
 | 最佳 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将每个座位视为一桶想要它的学生。 

1. 将所有学生按他们喜欢的座位分组。 对于每个座位，收集想要该座位的学生列表以及他们的不满意值。 
2. 对于每个座位组，选择不满意值最高的学生。 将该学生分配到这个座位。 这种选择确保我们在每个座位只能满足一个学生的情况下最大限度地满足高成本学生的利益。 
3. 将同一组中的所有其他学生标记为“未分配其首选座位”。 这些学生稍后将被安排在剩余的空闲座位上。 
4. 收集所有未接待所选学生的座位。 这些是免费座位。 
5. 收集所有未分配到其首选座位的学生。 这些是剩下的学生。 
6.将剩下的学生任意分配到空闲座位，因为这些分配无论如何都不符合他们的偏好，所以每个这样的学生无论他们得到哪个错误的座位，都会贡献他们全部的不满成本。 

关键的设计选择是第 2 步：在每个座位中，我们只保留最昂贵的错位学生。 原因是我们只有一次机会“拯救”这个席位，而我们希望尽可能拯救最昂贵的学生。 

### 为什么它有效

 该算法由每个座位的局部最优条件驱动。 每个座位最多可以匹配一名学生，因此在所有竞争该座位的学生中，只有一名学生可以避免付费。 选择成本最高的学生可以最大限度地减少该组的总惩罚，因为任何其他选择都会使成本较高的学生无法匹配，因此受到完全惩罚。 由于小组在席位上是独立的，因此这些局部最优决策结合成全局最优分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pref = list(map(int, input().split()))
    cost = list(map(int, input().split()))
    
    buckets = [[] for _ in range(n + 1)]
    
    for i in range(n):
        buckets[pref[i]].append(i)
    
    ans = [-1] * n
    used_seats = [False] * (n + 1)
    
    # assign best candidate per seat
    free_seats = []
    unassigned = []
    
    for seat in range(1, n + 1):
        if not buckets[seat]:
            free_seats.append(seat)
        else:
            best = buckets[seat][0]
            for i in buckets[seat]:
                if cost[i] > cost[best]:
                    best = i
            ans[best] = seat
            used_seats[seat] = True
            
            for i in buckets[seat]:
                if i != best:
                    unassigned.append(i)
    
    # assign remaining seats
    ptr = 0
    for i in unassigned:
        while ptr < len(free_seats) and ans[i] == -1:
            s = free_seats[ptr]
            ptr += 1
            ans[i] = s
    
    print(sum(cost[i] for i in range(n) if ans[i] != pref[i]))
    print(*ans)

def main():
    solve()

if __name__ == "__main__":
    main()
```该代码首先按学生喜欢的座位对他们进行分组。 然后，对于每个座位，它会识别出不满意成本最大的学生并直接分配给他们。 所有其他学生都被收集起来进行重新分配。 

空闲座位正是那些没有被选中的“最佳”学生的座位。 这些用于填补剩余的学生。 最终的成本计算只是将未安排在其首选座位的学生的成本相加。 

一个微妙的一点是，我们并不关心剩下的学生坐错了哪个座位。 成本仅取决于座位是否符合偏好，而不取决于其错误程度。

 ## 边缘情况

 一个关键的边缘情况是所有学生都喜欢同一个座位。 For input like:```
3
1 1 1
5 2 7
```该算法将成本为 7 的学生安排在座位 1 上，因为这样可以最大限度地节省成本。 剩下的两名学生被迫坐在其他座位上，引起了极大的不满。 分组步骤确保所有候选者都被一起考虑，最大成本选择保证最佳节省。 

另一种极端情况是一个座位没有感兴趣的学生。 这些座位就变成了空位，自然而然地吸纳了剩下的学生。 由于不满意仅取决于不匹配，因此这些空闲席位之间的任何放置都是等效的，因此分配顺序不会影响最优性。
