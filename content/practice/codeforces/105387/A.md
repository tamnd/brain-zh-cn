 ## 解决方案
 ## 问题理解

 我们得到一个二进制网格，它表示应用了称为膨胀的形态学操作后的图像。 每个单元格要么是黑色的（`#`）或白色（`.`）。 变换规则是局部的：如果输出中的像素在原始图像中已经是黑色或其八个邻居（包括对角线）中至少一个是黑色，则输出中的像素将变为黑色。 

任务是相反的。 我们得到了放大的图像，并且必须确定是否存在一些可以根据该规则产生它的原始图像。 如果存在，我们必须构造任何有效的原始图像。 

关键的困难在于膨胀会向外扩展黑色像素，因此多个不同的原始配置可能会产生相同的最终图像。 因此，相反的过程不是唯一确定的，并且在某些情况下是不可能的。 

网格尺寸最多为 100 x 100，这可以轻松实现 O(nm) 或 O(nm log nm) 解决方案。 任何模拟每个细胞邻域推理的方法都可以，但是任何子集上的指数方法都是不必要且不可行的。 

当输出中的黑色像素无法由任何可能的源像素“解释”时，就会出现微妙的边缘情况。 例如，如果一个单元格是黑色的，但它的所有邻居都是白色的，那么它只能来自同一位置的黑色原始单元格。 那总是好的。 当我们尝试重建原始图像时，真正的矛盾出现了，原始图像在膨胀后必须与给定图像完全匹配，但局部一致性约束在重叠邻域之间发生冲突。 

例如，考虑一种模式，其中孤立的黑色像素出现在无法同时分配原始源的位置，而不在膨胀结果中强制使用额外的黑色单元。 天真的贪婪重建可能会错误地分配原始黑色并过度产生黑色像素，但正确的方法会避免猜测，而是通过识别“必要的来源”来向后工作。 

## 方法

 一个蛮力的想法是尝试所有可能的原始网格。 原始细胞中的每个细胞都可以是黑色或白色，提供 2^(nm) 的可能性。 对于每个候选者，我们模拟膨胀并与给定的网格进行比较。 这是正确的，但完全不可行，因为即使是 10 x 10 的网格也已经给出了 2^100 个配置。 

关键的观察结果是膨胀是单调和局部的：输出中的黑色像素必须来自原始图像中 3 x 3 邻域中的至少一个黑色像素。 这意味着我们可以对每个单元格进行反向推理：如果输出中的一个单元格是黑色的，那么它的至少一个邻居（包括它自己）在原始单元格中必须是黑色的。 如果输出中的一个单元格是白色的，那么它在原始单元格中的邻居都不能是黑色的。 

这立即给我们提供了一个建设性的约束系统。 输出中的每个白色单元格禁止原始单元格中其 3 x 3 邻域中的所有黑色分配。 应用所有这些约束后，我们得到一个候选原始网格，其中一些单元格被强制为白色，而其他单元格则可能保持黑色。 然后，我们验证输出中的每个黑色单元格在其邻域中是否至少有一个可用的源单元格。 如果没有，重建是不可能的。 否则，我们可以安全地将黑色分配给每个黑色单元格的一个这样的有效邻居。 

这将问题转化为在具有恒定大小邻域的网格上的约束传播，将其减少到线性时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^(nm) · nm) | O(2^(nm) · nm) | O(纳米) | 太慢了 |
 | 基于约束的重建 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

1. 创建一个候选原始网格，将所有单元格标记为黑色进行初始化。 这代表了最宽松的起点，因为我们只会消除不可能的黑色位置。 
2. 对于输出图像中的每个白色单元，检查其在原始网格中的 3 x 3 邻域，并将所有这些位置标记为强制白色。 这遵循以下规则：白色输出单元不得受到原件中任何黑色像素的影响。 
3. 应用白细胞的所有约束后，我们得到了部分固定的原始网格。 此时，输出中的一些黑色单元可能仍然无法解释。 
4. 对于输出图像中的每个黑色单元，检查候选原始网格中的 3 x 3 邻居中是否至少有一个仍然是黑色。 如果不存在这样的邻居，则重建是不可能的。 
5. 如果每个黑色输出单元格至少有一个支持原始黑色候选单元，则通过在剩余的允许单元格中准确选择黑色来构建有效的原始单元格。 不需要进一步优化，因为任何此类分配都足以生成所需的扩张。 
6. 输出结果。 

### 为什么它有效

 该算法对膨胀规则进行反向编码。 白色输出单元强制执行硬排除约束，消除其附近所有可能的黑色来源。 黑色输出单元强制执行存在约束：其邻域中必须存在至少一个源。 由于这些约束纯粹是局部的并且在不同单元之间是独立的，因此单独满足它们可以保证全局一致性。 除了共享网格位置之外，不存在跨单元依赖性，并且该构造确保不会引入禁止的配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def inb(x, y, n, m):
    return 0 <= x < n and 0 <= y < m

n, m = map(int, input().split())
b = [list(input().strip()) for _ in range(n)]

# start with all black possible
a = [['#' for _ in range(m)] for _ in range(n)]

# step 1: white cells eliminate sources in 3x3 neighborhood
for i in range(n):
    for j in range(m):
        if b[i][j] == '.':
            for dx in (-1, 0, 1):
                for dy in (-1, 0, 1):
                    ni, nj = i + dx, j + dy
                    if inb(ni, nj, n, m):
                        a[ni][nj] = '.'

# step 2: verify black cells can be explained
for i in range(n):
    for j in range(m):
        if b[i][j] == '#':
            ok = False
            for dx in (-1, 0, 1):
                for dy in (-1, 0, 1):
                    ni, nj = i + dx, j + dy
                    if inb(ni, nj, n, m) and a[ni][nj] == '#':
                        ok = True
                        break
                if ok:
                    break
            if not ok:
                print("Impossible")
                sys.exit()

print("Possible")
for row in a:
    print("".join(row))
```该解决方案构建了一个初始化为全黑的候选原始图像。 这很重要，因为我们只会消除可能性，除非验证可行性，否则永远不会假设需要黑色。 

第一遍处理输出中的所有白细胞。 每个白色像素禁止其周围 3 x 3 区域中的任何黑色源，因此我们安全地将所有这些位置转换为候选网格中的白色。 这会强制执行膨胀定义中的所有必要约束。 

第二遍确保输出中的每个黑色像素在其邻域中至少有一个支持黑色候选者。 一旦黑色输出单元缺乏这种支持，我们就会终止，因为没有有效的原始单元可以产生它。

 | 测量| 复杂性 | 说明|

 ## 边缘情况

 当白色像素完全包围原本可能包含黑色像素的区域时，就会出现临界边缘情况。 例如，在输出中具有单个黑色单元的 3 x 3 网格中，如果所有周围单元都是白色，则算法会强制整个邻域为白色，从而消除该黑色像素的任何可能来源。 最终检查正确检测到不存在支持，并输出 Impossible。 

另一种边缘情况是黑色像素被隔离得很远。 在这种情况下，每个黑单元独立地需要至少一个候选源。 由于邻域不会干扰超出局部重叠的区域，因此该算法自然会分配有效源而不会发生冲突，这表明约束的独立性得以保留。 

第三种边缘情况是全白网格。 每个单元都被至少一个约束强制为白色或仍然不需要，并且不需要支持黑色单元。 该算法正确输出有效的全白原件。
