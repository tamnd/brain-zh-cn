 ## 解决方案
 ## 问题理解

 我们被要求计算有多少个长度的序列`n`可以使用红、绿、蓝三种颜色形成，其中序列中的每个位置都是一个立方体。 唯一的限制是连续块中不允许出现超过其自身限制的颜色：红色的最大条纹长度为`k_r`, 绿色有`k_g`，蓝色有`k_b`。 

因此，不要选择任意长度的序列`n`，我们正在计算大小为 3 的字母表上的所有有效字符串，其中每个字符在连续重复时都有其自己的最大允许运行长度。 

输出是此类有效序列的数量模`10^9 + 7`。 

这些约束使我们远离任何枚举序列的解决方案，甚至显式地在所有可能的运行长度上保持完整状态。 自从`n`可以大到一百万，任何甚至每个状态都是线性的或二次的方法`n`将会失败。 跟踪每种颜色的位置和最后运行长度的朴素动态编程会产生大致大小的状态空间`n * k`，当两者都能达到时就太大了`10^6`。 

第二个天真的尝试是仅跟踪最后的颜色和游程长度并直接迭代转换，但即使这样也会导致每个状态的转换取决于最多`k_c`以前的状态，再次产生$O(n \cdot k)$最坏的情况。 

一些边缘案例暴露了幼稚推理的陷阱。 如果全部`k_r = k_g = k_b = 1`，那么两个相邻的立方体不可能有相同的颜色。 答案就变成了`3 * 2^(n-1)`。 一个天真的 DP 意外地允许扩展长度为 1 的游程，即使对于像这样的小输入也会立即过度计数`n = 2`。 另一种边缘情况是当一个限制非常大时，例如`k_r = 10^6`其他的则很小。 盲目对称地限制每种颜色的运行的解决方案可能会错误地对红色过渡施加不必要的限制，而这些限制永远不会发生在相关范围内。 

核心困难在于每种颜色都有自己独立的“记忆窗口”，我们需要有效地强制执行该约束，同时仍然计算所有有效序列。 

## 方法

 暴力方法将构造每个可能的长度序列`n`并通过扫描连续运行来验证它。 每个步骤又分为三个选择，因此有$3^n$序列，甚至检查每个序列需要$O(n)$，导致不可能的$O(n \cdot 3^n)$运行时。 

更结构化的动态规划方法考虑`dp[i][c][l]`，长度序列的数量`i`以颜色结尾`c`与当前运行长度`l`。 这是正确的，因为它显式地强制执行运行约束，但它扩展到$O(n \cdot (k_r + k_g + k_b))$州，在最坏的情况下仍然太大。 

关键的观察是我们实际上不需要明确地跟踪精确的游程长度分布。 对于延长给定颜色的运行，重要的是该颜色的前一运行是否已经达到其允许的最大长度。 所有有效的转换都可以用每种颜色最近历史记录的累积计数来表示。 

这导致将状态压缩为三个序列：`dpR[i]`,`dpG[i]`， 和`dpB[i]`，其中每个表示长度有效序列的数量`i`以那种颜色结束。 延长色彩范围`c`只取决于最后一个`k_c`同一状态的贡献，可以使用嵌入在递归中的滑动窗口思想来维护。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举|$O(3^n \cdot n)$|$O(n)$| 太慢了|
 | 状态 DP 与游程长度 |$O(n \cdot k)$|$O(n \cdot k)$| 太慢了|
 | 具有滑动过渡的优化 DP |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们根据长度逐步构建答案`1`到`n`，维护三个数组`dpR`,`dpG`， 和`dpB`。 

1. 初始化长度序列的基本情况`1`。 每个立方体可以是任何颜色，并且每个立方体都是有效的，因为长度`1`永远不会违反任何约束。 所以`dpR[1] = dpG[1] = dpB[1] = 1`。 
2. 对于每个位置`i`从`2`到`n`，计算有多少个序列以每种颜色结束。 我们首先考虑红色。 任何有效的长度序列`i`以红色结尾的必须来自长度序列`i-1`然后附加一个红色立方体。 
3. 将红色扩展分为两类。 要么之前的立方体不是红色的，要么是红色的但红色运行仍然可以延长。 第一类贡献`dpG[i-1] + dpB[i-1]`。 
4. 对于现有红色运行的扩展，我们添加`dpR[i-1]`，但这会超出红色运行已达到其最大允许长度的序列。 这些无效的扩展名与红色运行位置的序列完全对应`i-1`已经有长度了`k_r`，可以使用移位项排除`dpR[i-1-k_r]`。 
5. 这产生了递归`dpR[i] = dpR[i-1] + dpG[i-1] + dpB[i-1] - dpR[i-1-k_r]`，如果索引超出范围，则省略减法。 
6. 使用各自的限制对绿色和蓝色对称应用相同的逻辑`k_g`和`k_b`。 
7. 填写完所有状态后`n`，将三个结束状态相加以获得有效序列的总数。 

关键的不变量是`dpC[i]`始终精确计算所有有效长度序列`i`最后一个立方体是颜色`C`，到目前为止所有运行约束均已满足。 递归保留了这个不变性，因为每个扩展要么从不同的颜色开始新的运行，要么继续有效的运行，并且减法精确地删除了运行超出其允许限制的那些序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, kr, kg, kb = map(int, input().split())

    dpR = [0] * (n + 1)
    dpG = [0] * (n + 1)
    dpB = [0] * (n + 1)

    dpR[1] = dpG[1] = dpB[1] = 1

    for i in range(2, n + 1):
        # Red
        val = (dpR[i - 1] + dpG[i - 1] + dpB[i - 1]) % MOD
        if i - 1 - kr >= 0:
            val = (val - dpR[i - 1 - kr]) % MOD
        dpR[i] = val

        # Green
        val = (dpR[i - 1] + dpG[i - 1] + dpB[i - 1]) % MOD
        if i - 1 - kg >= 0:
            val = (val - dpG[i - 1 - kg]) % MOD
        dpG[i] = val

        # Blue
        val = (dpR[i - 1] + dpG[i - 1] + dpB[i - 1]) % MOD
        if i - 1 - kb >= 0:
            val = (val - dpB[i - 1 - kb]) % MOD
        dpB[i] = val

    print((dpR[n] + dpG[n] + dpB[n]) % MOD)

if __name__ == "__main__":
    solve()
```该实现直接反映了重现。 每种颜色都使用相同的转换逻辑独立处理，但具有自己的约束长度。 减法步骤是唯一微妙的部分，因为它可以防止无效运行超出其允许的限制。 每一步都应用模运算，以将值保持在界限内并安全地处理减法后的负结果。 

实现此模式时的一个常见错误是忘记 Python 的负数模仍然需要标准化； 使用`% MOD`减法后确保正确性。 

## 工作示例

 考虑输入`5 2 2 2`。 所有颜色的表现都是对称的，并且每次运行的长度最多允许为 2。 

| 我| dpR[i] | dpR[i] | dpG[i] | dpG[i] dpB[i]|
 | ---| ---| ---| ---|
 | 1 | 1 | 1 | 1 |
 | 2 | 2 | 2 | 2 |
 | 3 | 6 - dpR[0] = 6 | 6 | 6 |
 | 4 | 18 - dpR[1] = 17 | 17 | 17 17 | 17
 | 5 | 类似地计算 | | |

 在每一步中，序列总数都像受约束的三色斐波那契式展开一样增长。 减法从长度开始变得重要`kr + 2`，确保排除超过 2 次的运行。 这证实了循环正确地执行局部运行限制，同时仍然计算所有颜色间的过渡。 

为了`6 1 1 3`，红色和绿色的行为类似于严格的交替约束，而蓝色则允许更长的运行时间。 

| 我| dpR[i] | dpR[i] | dpG[i] | dpG[i] dpB[i]|
 | ---| ---| ---| ---|
 | 1 | 1 | 1 | 1 |
 | 2 | 2 | 2 | 3 |
 | 3 | 4 | 4 | 8 |
 | 4 | 8 | 8 | 20 |
 | 5 | 16 | 16 16 | 16 48 | 48
 | 6 | 32 | 32 32 | 32 112 | 112

 该迹线显示了更严格的约束如何瓦解红色和绿色的状态增长，而蓝色主导组合扩展。 重复过程清楚地分离了这些效应，除了共享的前一步总数之外，没有任何交叉干扰。

 ## 边缘情况

 对于`n = 1`，重复不应尝试任何转换，因为没有先前的状态可以扩展。 初始化`dpR[1] = dpG[1] = dpB[1] = 1`直接覆盖这个案例，总共产生`3`，这符合以下事实：无论约束如何，任何单个多维数据集都是有效的。 

对于像这样的情况`3 1 1 1`，每次颜色变化必须严格交替。 该算法正确地减去任何超出长度的尝试`1`，有效地确保序列像`RR`或者`GG`永远不会被计算在内。 计算结果`12`匹配长度为三分之三的有效交替序列的精确计数。
