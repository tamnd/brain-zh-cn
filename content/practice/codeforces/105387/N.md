 ## 解决方案
 ## 问题理解

 我们得到了一个序列的部分损坏的描述，该序列最初来自一个简单的公式。 有一个未知的整数值$k$，并且对于每个索引$i$，通过除以得到预期值$k$经过$i$并使用标准数学舍入法舍入到最接近的整数，其中一半向上。 

所以每个位置$i$要么包含正确的舍入值$c_i$，或缺失并标记为问号。 目标是重建尽可能小的正整数$k$使得所有已知条目都符合此舍入规则。 

每个已知的约束关系$k$to a narrow range of values. 如果$c_i$已知，那么$k/i$必须位于周围的一个区间内$c_i$宽度一以$c_i$。 这将问题从“猜测$k$” 变成“找到位于许多区间交集的整数”。

 约束很严格，因为$n \le 1000$，所以任何检查所有可能的解决方案$k$达到很大的界限是不可行的。 直接搜索过去$k$需要考虑的值可能高达$10^9$或更高，具体取决于输入，这对于暴力迭代来说太大了。 

舍入有一个微妙之处：条件不是相等而是间隔约束。 另一个重要的一点是，缺失的条目没有任何限制，因此可以完全忽略它们。 

一个天真的错误是解释$c_i = k/i$as exact division or floor division. 例如，如果$k = 5$和$i = 2$， 然后$k/i = 2.5$，四舍五入为 3。将其视为 2 或 3 都是错误的，并且会错误地转移所有约束。 

## 方法

 一个蛮力的想法是尝试增加$k$从 1 向上检查是否每个已知$c_i$符合舍入规则。 对于每位候选人$k$，我们会计算$k/i$，四舍五入，并与所有已知位置进行比较。 每张支票费用$O(n)$，所以如果答案很大，这会变得非常慢。 

低效率来自于不使用约束结构。 Each fixed$c_i$不只是给出一个有效的$k$，它给出了一个连续的有效范围$k$价值观。 舍入条件转化为以下形式的不等式“$k/i$接近于$c_i$”，这变成了一个线性区间$k$。 

一旦每个约束成为一个区间，问题就简化为将所有这些区间相交并找到交集内的最小整数。 Instead of searching over$k$，我们维护一个全局可行区域并使用每个已知的观察来缩小它。 

唯一剩下的微妙之处是舍入会创建半整数边界，因此将所有内容缩放 2 更干净，并且完全避免浮点问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解结束$k$|$O(n \cdot k)$|$O(1)$| 太慢了|
 | 区间交集（缩放）|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们通过将所有内容加倍来消除分数界限。 Instead of working with$k$，我们与$T = 2k$。 每个约束变成一个简单的整数区间。 

1. 对于每个已知的对$(i, c_i)$，将舍入规则转换为上界$T = 2k$。 

条件$c_i - 0.5 \le \frac{k}{i} < c_i + 0.5$变成$2i c_i - i \le 2k < 2i c_i + i$。 

所以$T$必须位于整数区间内$[2ic_i - i, 2ic_i + i - 1]$。 
2. Maintain a global feasible interval$[L, R]$为了$T$。 将其初始化为非常宽，然后将其与每个约束间隔相交。 

After processing each known$c_i$， 更新$L = \max(L, 2ic_i - i)$和$R = \min(R, 2ic_i + i - 1)$。 
3.处理完所有约束后，有效$T$值正是这些整数$[L, R]$。 我们现在需要最小的有效$k$，对应于最小偶数$T$在这个范围内。 
4. 调整$T$向上到第一个不小于的偶数$L$。 如果$L$均匀，保留； 否则增加 1。 
5、输出$k = T / 2$。 

为什么它起作用与每个约束都是凸的这一事实有关$k$。 舍入规则为每个值创建一个连续的有效值区间$i$，并且与所有这些区间相交可以准确地保留可行解集。 由于所有约束都是充分必要条件，任何$k$在最终交叉路口之外将违反至少一个已知的$c_i$，以及任何$k$内部同时满足所有约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    arr = input().split()

    L = -10**30
    R = 10**30

    for i, val in enumerate(arr, start=1):
        if val == '?':
            continue

        c = int(val)

        left = 2 * i * c - i
        right = 2 * i * c + i - 1

        L = max(L, left)
        R = min(R, right)

    if L % 2 != 0:
        L += 1

    T = L
    k = T // 2
    print(k)

if __name__ == "__main__":
    main()
```实现的核心是将舍入约束转化为线性不等式$2k$。 每个已知值都会收紧可行范围，并且我们永远不需要考虑未知位置，因为它们没有任何约束。 最后一步确保我们选择最小的有效整数$k$，对应于最小偶数$T$。 

一个常见的错误是忘记在缩放之前上限是互斥的。 这就是为什么间隔结束于$2ic_i + i - 1$， 不是$2ic_i + i$。 

## 工作示例

 ### 示例 1

 输入：```
3
?
3
2
```我们一步步处理约束。 

| 我| c_i | 更新| R 更新 |
 | ---| ---| ---| ---|
 | 2 | 3 | L = 2·2·3 − 2 = 10 | R = 2·2·3 + 2 − 1 = 13 |
 | 3 | 2 | L = 最大值(10, 2·3·2 − 3 = 9) = 10 | R = min(13, 2·3·2 + 3 − 1 = 14) = 13 |

 所以$T \in [10, 13]$。 最小的甚至$T$是 10，给予$k = 5$。 

这显示了多个约束如何重叠以缩小有效范围，以及如何将最终答案选择为交集内的最小偶数。 

### 示例 2

 输入：```

?

```只有一个约束很重要。 

| 我| c_i | 左 | 右 |
 | ---| ---| ---| ---|
 | 2 | 4 | 2·2·4 − 2 = 14 | 2·2·4 + 2 − 1 = 17 |

 所以$T \in [14, 17]$。 最小的甚至$T$是 14，所以$k = 7$。 

这表明缺失值确实没有任何贡献，并且单个约束足以确定可行范围。

 ## 边缘情况

 一种重要的边缘情况是除一项之外的所有条目均丢失。 该算法简化为单个区间约束，并且答案完全来自该约束。 交集逻辑仍然有效，因为它从一个非常宽的范围开始并正确收缩。 

另一个微妙的情况是，当可行区间仅包含奇数值时$T$。 自从$T$必须是偶数，我们明确向上调整。 例如，如果$T \in [15, 15]$，我们跳过 15 并移至 16。这种转变保留了有效性，因为原始定义保证至少存在一个偶数解。 

另一种边缘情况是约束在单个边界值处重叠。 由于缩放后包含上限，因此区间端点是安全的，并且最终调整步骤不会意外排除有效的最小解。
