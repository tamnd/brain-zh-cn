 ## 解决方案
 ## 问题理解

 我们得到一个确定性的整数序列，该序列是通过将前一个值重复乘以固定因子然后取模而生成的。 该序列从给定的初始值开始并精确产生$n$数字。 下一个数字仅取决于前一个数字，因此一旦知道四个参数，就可以完全确定整行。 

任务不是为了计算整个序列本身，而是识别出现在该生成行中任何位置的五个最大值，并按升序输出它们。 

的约束条件为$n$达到$2 \cdot 10^7$，这意味着任何尝试存储所有值或执行比线性时间更差的操作的方法都将失败。 二次或偶数$O(n \log n)$基于排序的解决方案在实践中也太慢，因为生成序列已经需要数千万次操作，额外的开销会超出时间限制。 如果每个元素都保存在有开销的列表中，则内存限制还会阻止在 Python 中安全地存储完整数组。 

一个简单的解释是生成所有值，对它们进行排序，然后选择前五个。 这种方法在逻辑上是正确的，但在规模上立即失效。 为了$n = 20{,}000{,}000$，排序需要的顺序是$n \log n$，这远远超出了可行的范围。 

当人们尝试存储所有值以便稍后扫描时，就会出现一个更微妙的问题。 即使内存适合，缓存行为也会变得很差，Python 开销会主导执行时间。 

值得注意的一个极端情况是序列迅速崩溃为重复或零。 例如，如果$p = 1$，那么无论什么值，每个值都为零$a_1$和$k$。 在这种情况下，正确的输出只是五个零，但是假设可变性的粗心实现可能仍然会尝试不必要的处理或错误处理重复项。 

## 方法

 蛮力方法很简单：生成所有$n$使用递归的值，将它们存储在数组中，对数组进行排序，并获取最后五个元素。 这是可行的，因为排序正确地对所有值进行了排名，但它需要内存$n$整数和时间$O(n \log n)$。 和$n$高达两千万，连构建阵列所需的线性扫描都已经很昂贵了，排序完全占主导地位。 

关键的观察是我们实际上从来不需要所有值的完整排序。 我们只需要知道在流式传输序列时迄今为止看到的五个最大值。 这将问题从全局排序任务转换为流选择问题。 

序列本身是一次性生成的，在每一步中，我们都可以维护一个包含当前最佳五个候选者的小结构。 由于该结构永远不会超过大小五，因此所有更新都是恒定时间。 从对所有内容进行排序到维护固定大小的顶集的转变完全消除了对数因子。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（存储+排序）|$O(n \log n)$|$O(n)$| 太慢了|
 | 流媒体top-5维护|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算第一个值$a_1$并将其视为答案的第一个候选者。 这会初始化运行结构，因为在序列开始之前不可能进行比较。 
2. 维护一个最多包含五个数字的小列表，代表迄今为止看到的最大值。 将其按升序排列，以便始终易于访问其中最小的内容。 
3. 迭代从第二个元素到第一个元素的序列$n$-th 元素，使用生成每个值$a_i = (a_{i-1} \cdot k) \bmod p$。 此步骤纯粹是流式传输，因此不存储过去的值。 
4. 对于每个新生成的值，将其与当前前五个列表中的最小元素进行比较。 如果不大于，则立即丢弃，不会影响最终答案。 
5. 如果新值大于前五个列表中的最小元素，则将其插入列表中，删除最小元素以保持大小固定，并恢复排序顺序。 这可确保列表始终包含迄今为止看到的最佳候选者。 
6.全部处理完毕后$n$value，按升序输出五个存储的值。 

关键的不变量是在处理第一个之后$i$元素，维护的列表恰好包含其中的五个最大值$i$元素，已排序。 这最初成立，因为列表从第一个元素开始并增长到大小五，并且它被保留，因为每个新值要么无法进入前五名，要么替换当前前五名中最小的值而不干扰较大的元素。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, a1, k, p = map(int, input().split())

top = [a1]

cur = a1
for i in range(1, n):
    cur = (cur * k) % p

    if len(top) < 5:
        top.append(cur)
        top.sort()
    else:
        if cur > top[0]:
            top[0] = cur
            top.sort()

print(*top)
```该解决方案依赖于最小的数据结构：最多五个元素的列表。 每次更新都是持续的工作，因为对固定大小的数组进行排序实际上是恒定的时间。 迭代计算递归以避免重新计算或存储。 

一个微妙的点是我们从不存储完整的序列。 变量`cur`仅携带前一个值，足以生成下一个值。 这使内存使用保持恒定。 

另一个细节是条件替换逻辑。 只有当新值超过前五个当前最小值时，我们才会修改结构。 这可以防止不必要的排序操作，考虑到时间紧迫，这一点很重要。

 ## 边缘情况

 当$p = 1$，每次乘法后跟模立即产生零。 该算法仍然表现正常，因为序列生成步骤产生了一系列零，并且前五名结构逐渐填充零并稳定下来。 最终输出正确地变为五个零，无需任何特殊处理。 

什么时候$k = 1$，序列变为常数。 第一个值对所有位置重复，因此前五个结构简单地填充相同的值。 由于允许重复，因此没有替换逻辑会更改最终结果，并且该算法仍然返回相同数字的五个副本。 

什么时候$n = 5$，结构永远不会进入替换阶段。 每个元素都是直接插入的，不依赖维护逻辑，测试初始化​​的正确性和最终输出格式。
