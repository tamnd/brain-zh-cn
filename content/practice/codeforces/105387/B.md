 ## 解决方案
 ## 问题理解

 我们有一个非常大的网格，但只标记了其中的少量单元格。 在这种状态下，我们可以选择一个空单元格并选择四个基本方向之一。 在该选定的方向上，游戏会破坏严格位于该方向上同一行或同一列的每个标记单元，具体取决于我们选择水平还是垂直移动。 

目标是选择一个空的起始单元和一个方向，以使被破坏的标记单元的数量尽可能多，并且我们还需要输出一个达到这一最大值的有效起始位置。 

尽管网格大小在两个维度上都可以达到 10^9，但最多只能标记 200,000 个单元格。 这立即告诉我们解决方案的结构必须仅取决于标记的单元格，而不取决于任何完整的网格表示。 

强力解释是尝试每个空单元格，并为每个空单元格扫描其行和列以计算每个方向上有多少标记的单元格。 这是不可能的，因为空单元的数量实际上是 n·m 减去 k，这可能是巨大的。 

一个更微妙的约束来自这样一个事实：方向的选择只关心对齐。 一旦我们修复了一个单元格，只需要在其行或列中标记单元格即可。 这建议将问题压缩为行方向和列方向的频率信息。 

当人们假设最佳位置必须位于标记的单元格上或旁边时，就会出现天真的推理的常见失败情况。 这是错误的，因为将起点放置在行或列中的极值标记位置之外可以捕获该行中的所有标记单元格。 

例如，考虑具有标记的第 3 列和第 10 列的行。如果我们将起始点放置在同一行的第 1 列处，则整个右方向会破坏两个标记。 仅考虑与标记单元格相邻的位置的解决方案会错过这一点。 

当所有标记的单元格位于同一行或一列时，会出现另一种失败情况。 在这种情况下，选择不同行或列中的单元格可能仍然有效并且可能有必要，具体取决于该行是否完全填充。 

## 方法

 直接的强力方法将迭代每个空单元格并计算在四个方向上可到达的标记单元格的数量。 对于每个候选单元格，在最坏的情况下，使用哈希集或二分搜索扫描其行和列将花费 O(k) 每个单元格。 由于空单元的数量本质上是n·m，所以这种方法是完全不可行的。 

关键的观察结果是，除了按行和按列对标记单元进行分组之外，网格结构无关紧要。 一旦我们修复了一行，重要的是该行中存在多少个标记的单元格。 这同样适用于列。 

对于具有 t 个标记单元格的固定行，假设我们在该行中选择一个空单元格。 如果我们把它放在标记细胞所跨越的区间之外，我们就可以捕获一个方向上的所有 t 细胞。 如果我们把它放在区间内，我们可以将行分成左右两部分，但我们能做的最好的仍然是 t − 1。由于网格很大，除非行被完全填满，否则至少有一个空位置，所以我们总是可以选择标记范围之外的位置，给定值 t。 

因此，只要未完全填充，每一行都会贡献其标记单元格的数量。 相同的论点适用于列。 

这将任务简化为计算行频率和列频率并取其中的最大值，同时确保所选行或列中至少存在一个有效的空单元格。

 | ---| ---| ---| ---|
 | 蛮力 | O(n·m·k) | O(k) | 太慢了|
 | 最佳 | O(k) | O(k) | 已接受 |

 ## 算法演练

我们将问题分为行统计和列统计。 

1. 读取所有标记的单元格，并按行和列对它们进行分组。 我们为每一行存储其标记列的列表，并为每一列存储其标记行的列表。 这是我们需要的有关网格的唯一信息。 
2. 对于每一行，计算它包含多少个标记单元格。 如果计数等于 m，则该行没有空单元格，不能用作起始位置，因此我们忽略它。 
3. 对于每个可用行，计算其候选得分等于该行中标记单元格的数量。 我们还跟踪最佳行及其计数。 
4. 对色谱柱重复相同的过程。 如果某列的计数等于 n，则该列已被完全占用且无法使用。 否则，其候选分数是该列中标记单元格的数量。 
5. 比较最佳行得分和最佳列得分。 较大的那个就是答案。 如果两者都为零或不存在有效的行/列，则没有有效的移动，答案为零，坐标为 0 0。 
6. 为了重建最佳行的起始单元格，我们选择具有最大计数的行，然后查找该行中未标记的任何列索引。 由于该行最多有 k 个标记的单元格，因此我们可以尝试从 1 开始的列索引，直到找到集合中不存在的单元格为止。 同样的逻辑也适用于最佳列。 

它之所以有效，是基于这样一个事实：在任何至少有一个空单元格的行中，我们始终可以将起点放置在标记列段之外，并捕获一个方向上的所有标记单元格。 因此，该行中可实现的最佳水平破坏正是其中标记的单元格的数量。 同样的论点对于列对称地成立。 由于每个有效的移动都必须锚定在某些行或列中，因此全局最优值必须出现在这些每行最大值之间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())
    
    row = {}
    col = {}
    marked = set()

    for _ in range(k):
        x, y = map(int, input().split())
        marked.add((x, y))
        if x not in row:
            row[x] = []
        if y not in col:
            col[y] = []
        row[x].append(y)
        col[y].append(x)

    best = 0
    best_type = None  # 0 for row, 1 for col
    best_idx = None

    for x, ys in row.items():
        t = len(ys)
        if t < m:
            if t > best:
                best = t
                best_type = 0
                best_idx = x

    for y, xs in col.items():
        c = len(xs)
        if c < n:
            if c > best:
                best = c
                best_type = 1
                best_idx = y

    if best == 0:
        print(0)
        print("0 0")
        return

    if best_type == 0:
        x = best_idx
        forbidden = set(row[x])
        y = 1
        while y in forbidden:
            y += 1
        print(best)
        print(x, y)
    else:
        y = best_idx
        forbidden = set(col[y])
        x = 1
        while x in forbidden:
            x += 1
        print(best)
        print(x, y)

if __name__ == "__main__":
    solve()
```该代码首先构建行和列的邻接列表，同时跟踪所有标记的单元格以进行快速成员资格查询。 主选择循环计算最佳行和列候选，同时丢弃完全填充的行，因为这些行不能包含有效的起始单元格。 

重建步骤故意简单：它从 1 向上扫描，直到在所选行中找到未标记的坐标。 这是安全的，因为标记的单元格数量很少，因此在每条选定行的最坏情况下，扫描以 k 为界限。

 ## 边缘情况

 关键的边缘情况是所有标记的单元格占据整行。 在这种情况下，该行没有任何贡献，因为没有有效的空起始单元格。 该算法通过显式检查行计数是否等于 m 并跳过它来处理此问题。 

当最佳解决方案位于列而不是行时，会出现另一种边缘情况。 重建逻辑反映了行处理，并确保我们仍然通过扫描该列中第一个未标记的位置找到有效的空坐标。 

第三种边缘情况是 k 等于 0 时。 然后每个单元格都是空的，但任何移动都不会破坏任何东西。 该算法正确返回零，任何坐标都可以接受，这里表示为 0 0。
