 ## 解决方案
 ## 问题理解

 我们得到一组不同的正整数。 该集合必须精确地划分为`k`非空子集。 每个子集贡献其内部所有数字的 gcd，并且我们希望这些 gcd 值的最大可能总和。 

数字不同，每个值最多`10^6`，并且在所有测试用例中，每个测试用例中出现的最大值的总和最多为`10^6`。 最后一个条件才是真正的提示。 它强烈建议了一个大致有效的解决方案`O(M log M)`每个测试用例，其中`M`是集合中的最大值。 二次算法`n`当`n`本身可以达到`10^6`。 

棘手的部分是分区完全不受限制。 即使对于非常小的输入，直接搜索分区也是不可能的。 

一个常见的错误是关注 gcd 值本身并尝试贪婪地构建子集。 分区结构实际上比乍一看要简单得多。 

考虑集合`{3, 5, 6}`和`k = 2`。 

如果我们选择子集`{5}`和`{3, 6}`，总和为：`5 + gcd(3, 6) = 5 + 3 = 8`这是最佳的。 

第二个容易错过的情况是某些数字除以其他数字。 

为了`{5, 9, 10, 11, 13}`和`k = 4`, 投入`10`与...一起`5`是有益的，因为子集 gcd 变为`5`。 最优划分为：`{13}, {11}, {9}, {5, 10}`给予`13 + 11 + 9 + 5 = 38`总是隔离最大数字的幼稚策略并不能解释原因`10`应该合并而不是`5`。 

## 方法

 蛮力的想法很简单。 枚举将集合划分为的所有方法`k`非空子集，计算每个子集的 gcd，并保留最佳答案。 

这是正确的，因为它会检查每个有效分区。 问题在于分区数量呈超指数增长。 即使是几十个元素也已经是不可能的了。 

关键的观察来自于询问最佳分区内发生的情况。 

假设一个子集至少包含两个元素，其 gcd 为`g`。 选择任意元素`x`在该子集中，这不是最小的。 自从`g`将子集中的每个元素相除，我们有`g ≤ x`。 

如果我们删除`x`从子集并使其成为单例子集，贡献从`g`到`g + x`。 

总和至少增加`x - g ≥ 0`。 

这意味着，每当我们被允许创建更多子集时，分离元素永远不会有害。 

因为我们必须准确地结束`k`子集，最佳划分总是具有：`k - 1`单例子集和一个包含另一个子集的剩余子集`n - k + 1`元素。 

让`m = n - k + 1`。 

如果剩余的子集是`R`，那么答案就变成了`sum(all elements) - sum(R) + gcd(R)`。 

所有数字的总和是固定的，所以我们只需要最大化`gcd(R) - sum(R)`，

在哪里`|R| = m`。 

现在修复一个可能的 gcd 值`d`。 

如果`gcd(R) = d`, 的每个元素`R`必须能整除`d`。 

在所有能被整除的元素中`d`，最好的选择显然是`m`最小的，因为该术语`-sum(R)`应尽可能大。 

所以对于每个除数`d`，如果至少`m`集合中的数字可以整除`d`，我们计算`d - (sum of the m smallest present multiples of d)`。 

总体而言最好的价值`d`给出最佳修正项。 

约束的结构使得除数筛解变得自然。 对于每个`d`，我们按升序扫描它的倍数。 由于多次访问是按递增的数字顺序进行的，因此第一个`m`现在的倍数正是`m`最小的可整除数。

 ## 算法演练

 1.读取集合并计算总和`S`所有元素。 
2.让`m = n - k + 1`。 

这是最佳分区中单个非单一子集的大小。 
3. 为所有值构建一个存在数组，直到集合中的最大值。 
4. 对于每个可能的除数`d`从`1`到`max_value`:

 扫描所有的倍数`d`按递增顺序。 
5. 每当集合中存在倍数时，将其添加到运行总和中并增加计数器。 

因为倍数是按升序处理的，所以第一个`m`现在的倍数正是`m`最小的可分元素。 
6. 一旦计数器达到`m`, 计算`candidate = d - current_sum`。 

更新最佳值。 
7.最终答案是`S + best`。 

### 为什么它有效

 关键的性质是每个最优划分都可以精确地转化为一个`k - 1`单个子集和一个更大的子集，而不减少答案。 

转换之后，每个解决方案都由集合唯一地描述`R`大小的`m = n - k + 1`。 

其贡献是`S - sum(R) + gcd(R)`。 

对于固定的 gcd 值`d`, 的每个元素`R`必须能整除`d`。 最大化`d - sum(R)`,

 我们必须尽量减少`sum(R)`，这意味着选择`m`能被整除的最小可用数`d`。 

检查每个除数`d`涵盖了所有可能的gcd。 找到的最佳值恰好是可实现的最大值`gcd(R) - sum(R)`。 

因此，算法总是返回最佳答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    ans = []

    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))

        S = sum(a)
        m = n - k + 1
        mx = max(a)

        present = [0] * (mx + 1)
        for x in a:
            present[x] = 1

        best = -10**18

        for d in range(1, mx + 1):
            cnt = 0
            cur_sum = 0

            for multiple in range(d, mx + 1, d):
                if present[multiple]:
                    cnt += 1
                    cur_sum += multiple

                    if cnt == m:
                        value = d - cur_sum
                        if value > best:
                            best = value
                        break

        ans.append(str(S + best))

    sys.stdout.write("\n".join(ans))

if __name__ == "__main__":
    solve()
```第一部分计算固定数量`S`，所有元素的总和。 

价值`m = n - k + 1`直接来自结构证明，即最佳划分由一个较大的子集组成，并且`k - 1`单例子集。 

存在数组允许恒定时间检查某个值是否属于该集合。 由于所有值都是不同的，因此布尔数组就足够了。 

对于每个除数`d`，我们遍历它的倍数。 遍历顺序是递增的，所以第一个`m`现在的倍数自动为`m`最小的可整除数。 无需排序。 

当我们到达的那一刻`m`元素，我们已经知道该除数的最佳子集和，因此我们可以停止扫描更大的倍数`d`。 

所有算术都适合 Python 整数。

 ## 边缘情况

 考虑：```
1
3 2
3 5 6
```一种诱人的方法是隔离最大元素并假设其余元素贡献其最小元素。 剩余子集`{3, 6}`贡献`3`，这不是偶然的最小值，而是因为它的 gcd 恰好是`3`。 该算法检查除数`3`, 找到前两个可整除的元素`{3, 6}`，并得到正确答案`8`。 

现在考虑：```
1
5 4
13 11 10 9 5
```最好的解决方案不是通过将两个最小的数字分组来获得的。 最优剩余子集是`{5, 10}`因为它的gcd是`5`。 当算法处理除数时`5`，它立即找到恰好两个当前倍数，给出值`5 - (5 + 10) = -10`，这比其他所有除数都要好。 

最后：```
1
4 2
2 3 4 5
```最优答案是`6`。 算法选择`R = {2, 3, 4}`因为在所有大小为 3 的子集中，它最大化`gcd(R) - sum(R)`。 得到的分区是`{5}`和`{2, 3, 4}`, 给予`5 + 1 = 6`。 这证实了剩余子集的 gcd 不需要大于`1`。
