 ## 问题理解

 我们得到一个非负整数列表，每个整数代表分配给一组中的一个人的值。 任务是计算“群体凝聚力”的全局度量，定义为所有无序成对的人的值按位与的总和。 

具体来说，对于每对不同的索引$i < j$，我们取$a_i \& a_j$，然后我们将所有对的这些值相加。 输出是一个整数。 

的约束条件为$n$最大可达150,000，每个值可以大到$10^8$。 直接的成对计算需要大约$n^2 / 2$操作，在这个规模上大约是$10^{10}$，远远超出任何合理的时间限制。 这立即排除了任何二次解。 

这些值的边界为$10^8$，适合 27 位。 这意味着任何按位分解方法都可以安全地迭代最多 30 个位位置，从而使逐位计数策略变得合理。 

朴素方法的一个微妙的失败模式是整数溢出或错误地多次重新计算相同的贡献。 另一个常见的错误是尝试通过排序进行优化，这不会保留成对的位关系。 

例如，如果所有数字都相同，则说$[7,7,7]$, 每对都贡献$7$，所以答案是$3 \cdot 7 = 21$。 任何试图在不考虑对计数的情况下“压缩重复项”的不正确方法很容易错误地计算多重性。 

## 方法

 蛮力解决方案迭代所有对$(i, j)$并计算$a_i \& a_j$。 这是直接且正确的，因为它直接遵循问题的定义。 然而，其成本呈二次方增长$n$，并且有 15 万个元素，这会导致超过百亿次操作，这是不可行的。 

关键的观察结果是按位与运算在位之间是独立的。 仅当在一对数字中都设置了某个位时，该位才会对最终总和有所贡献。 这表明我们要扭转观点：我们不再逐对迭代，而是一点一点地计算贡献。 

固定一点位置$b$。 我们计算有多少个数字设置了这个位，比如说$c_b$。 任何一对这样的数字都有贡献$2^b$到最终的总和，这样的对的数量是$\binom{c_b}{2}$。 对所有位求和即可给出答案。 

这将问题从对枚举转变为对位的频率计数，从而显着降低了复杂性。

 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 位计数|$O(n \log A)$|$O(1)$| 已接受 |

 ## 算法演练

 我们处理该数组，同时维护每个位包含多少个数字的计数。 

1.初始化一个数组`cnt`大小为 31（足以满足高达$10^8$）。 每个位置存储到目前为止看到的有多少个数字设置了该位。 这种结构使我们能够增量地跟踪比特频率。 
2. 迭代每个数字$x$在输入中。 对于每个位位置$b$从0到30，检查是否位$b$设置于$x$。 如果是，则增加`cnt[b]`。 这确保在处理完所有数字后，`cnt[b]`等于贡献该位的元素总数。 
3. 计数后，通过迭代所有位位置来计算答案。 对于每一位$b$， 如果`cnt[b] >= 2`，那么有$\binom{cnt[b]}{2}$对贡献这一点。 
4.添加$\binom{cnt[b]}{2} \cdot 2^b$到最终的答案。 这重建了共享该位的所有对的总贡献。 
5. 输出累计和。 

关键步骤是按位分离贡献，这可以避免重复计算并确保每对每位被准确地考虑一次。 

### 为什么它有效

 每对$(i, j)$对于每个数字都有 1 的位，贡献是独立的。对于固定位，贡献仅取决于有多少数字包含该位，而不取决于它们的身份。 因此，对每个位组内的索引对进行计数可以准确地重建总和，而无需枚举对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    cnt = [0] * 31
    
    for x in a:
        b = 0
        while x:
            if x & 1:
                cnt[b] += 1
            x >>= 1
            b += 1
    
    ans = 0
    for b in range(31):
        c = cnt[b]
        if c >= 2:
            ans += (c * (c - 1) // 2) * (1 << b)
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现对每个数字使用直接位扫描，而不是使用固定循环检查所有 31 位。 这使操作保持紧凑，并避免对零尾部进行不必要的位检查。 

主要的微妙之处是确保每个位位置与其索引正确对齐`cnt`。 变量`b`当我们移动数字时跟踪位索引。

 ## 边缘情况

 对于单个元素输入，例如`1\n5`，该算法设置该数字的位数，但从未达到至少 2 的计数。 每一个`cnt[b]`至多为 1，因此所有二项式贡献都消失，产生 0，这与不存在对的事实相匹配。 

对于相同的元素，例如`3\n7 7 7`，7 的所有位都被计数 3 次。 每一位贡献$\binom{3}{2} \cdot 2^b$。 对位求和可在三对中精确地重建 7 的三个副本，直接匹配成对定义。 

对于具有不相交位集的数字，例如`1 2 4`，每一位都恰好出现在一个数字中，所以所有`cnt[b] ≤ 1`，这意味着没有一对共享任何位。 该算法正确返回 0，这与所有对中每个 AND 都为零的事实相符。
