 ## 解决方案
 ## 问题理解

 我们有一棵树$n$顶点，其中顶点 1 是起点和顶点$n$是目的地。 每条边连接两个顶点，但与标准加权树不同，每条边在某种意义上都是有向的：如果我们从$u$到$v$，我们获得一个值，如果我们从$v$到$u$，我们获得另一个值。 这两个值可以不同，甚至可以为负值。 

我们想要从节点 1 步行到节点$n$最大化沿途边缘值的总和。 允许步行重新访问边缘，但边缘的每个方向最多可以使用一次。 此限制很重要，因为它可以防止任意循环多次从同一有向边累积利润。 

输入大小达到$10^5$节点，因此任何二次解甚至依赖于路径上的重复重新计算的解都是不可能的。 对树结构的线性或近线性遍历是唯一可行的方向，可能与单个 DFS 或 DP 遍历相结合。 

一个微妙的点是，虽然允许重新访问节点，但不允许重新访问同一方向的边。 这消除了无限正循环的可能性，但仍然允许使用先前已向前使用的边缘的相反方向的临时绕道。 

天真的思维的一个典型失败案例是将每条边视为具有固定权重，例如始终使用$\max(p_i, q_i)$。 这是不正确的，因为方向取决于实际路径，而不是本地选择。 

例如，如果一条路径迫使我们首先遍历其低值方向的边缘，以便稍后到达更好的结构，那么贪婪的局部决策就会失败。 

## 方法

 暴力方法会尝试探索从节点 1 到节点的所有可能的路径$n$，跟踪已使用的有向边。 每个步骤都会分支到所有关联边，我们要么遍历未使用的方向，要么跳过它。 即使进行剪枝，这也会很快变得指数级，因为在每个节点，我们可能会以不同的事件边使用状态多次重新访问它。 由于每个边缘方向都会引入状态，因此配置数量呈指数增长$n$，使得这种方法不可行。 

关键的结构观察是图是一棵树。 在树中，任何两个节点之间都只有一条简单路径。 任何离开该路径的额外移动都必须包含绕道进入最终返回的子树。 这意味着问题可以重新定义为，对于每个子树，如果我们暂时离开主路线并返回，我们可以获取多少额外利润。 

关键思想是以节点 1 为树的根，并计算每个节点的 DP 值，该值表示如果我们从其父树进入该子树，则可以从其子树获得的最佳额外增益。 对于每条边，我们决定是从父节点遍历到子节点还是从子节点遍历到父节点，具体取决于在考虑最佳路线中的行进方向时哪一条对最终答案的贡献更大。 

这将问题转化为树 DP，其中每条边都会贡献固定的基线成本（在一个方向上使用它的成本），并且反转它的可能性会贡献相对于该基线的额外增益或惩罚。 我们积累贡献，同时确保遍历方向与根到目标路径结构一致。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 树DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将树的根设在节点 1 处，并假设我们要将值传播到节点$n$。 

我们对待每一个边缘$(u, v, p, q)$一旦我们确定了 DFS 方向，就有了自然的方向。 如果我们从父级遍历到子级，我们会根据该方向采用相应的定向权重。 关键是最终的答案仅取决于我们如何相对于根定位遍历。 

我们计算一个 DP 值$dp[v]$表示从节点开始的最佳可实现总和$v$向下走向节点$n$在它的子树内部，假设我们已经进入$v$。 

我们还坚持这样一个事实，即我们只需要考虑最终到达的路径$n$，因此不包含的子树$n$只有当他们提供可以在继续向上之前返还的利润时才做出贡献。 

1. 以节点1为树根并定位节点$n$在DFS期间。 我们需要这个，因为只有子树结构导致$n$重要的是主要道路，而其他人只是绕道而行。 
2. 执行 DFS，计算每个节点的子树是否包含节点$n$。 这确定了哪条子路径位于通往目标的主路径上。 
3.在DFS期间，对于节点之间的每条边$u$和孩子$v$，根据子树是否计算该边的贡献$v$包含$n$。 
4.如果$v$的子树包含$n$，那么最优路线最终必须进入$v$然后返回或继续向下$n$。 在这种情况下，我们将边缘视为主路径的一部分，并采取与移动方向一致的方向$n$。 
5. 如果$v$的子树不包含$n$，然后任何遍历到$v$必须返回$u$。 在这种情况下，我们可以选择使用两个方向中较好的一个进行往返，然后返回，从而获得增益$\max(p, q)$对于正向旅行，然后根据需要计算相应的返回成本，但由于返回是强制性的，因此我们将其建模为净收益$\max(p, q) - \text{cost back in opposite direction}$，这简化了选择最佳可用方向的过程。 
6、向上传播的同时，积累所有孩子的贡献。 DP确保每个子树独立贡献其最优绕行利润。 
7. 最终的答案是从1到1的路径上的累加值$n$，包括一切有益的弯路。 

关键的不变量是对于每个节点，$dp[v]$正确地表示从其子树中可获得的最大额外利润，而不违反“每个有向边最多使用一次”规则，因为每个子树都是独立求解的，并且仅通过树强制执行的单入口、单出口结构进行组合。 

这是有效的，因为树中的任何行走都可以分解为进入子树、执行内部遍历并通过同一边退出的段，除非路径继续向$n$。 这种分解确保子树之间的交错不会破坏最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    g = [[] for _ in range(n + 1)]
    
    for _ in range(n - 1):
        u, v, p, q = map(int, input().split())
        g[u].append((v, p, q))
        g[v].append((u, q, p))
    
    parent = [-1] * (n + 1)
    contains = [False] * (n + 1)
    dp = [0] * (n + 1)
    
    def dfs(u):
        contains[u] = (u == n)
        for v, p, q in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            dfs(v)
            if contains[v]:
                contains[u] = True
        
        for v, p, q in g[u]:
            if v == parent[u]:
                continue
            if contains[v]:
                dp[u] += p
            else:
                dp[u] += max(p, q)
    
    parent[1] = 0
    dfs(1)
    
    print(dp[1])

if __name__ == "__main__":
    solve()
```邻接表显式存储两个方向，以便我们可以将树视为无向树，但仍然通过交换来保留方向权重$(p, q)$当反转边缘时。 

DFS 首先计算哪些节点位于通往$n$，这对于区分强制边缘和可选绕道至关重要。 然后第二个累积步骤分配贡献：通向的边$n$沿着从父母到孩子的强制方向，而侧分支中的边缘通过选择更好的方向来做出最佳贡献，因为它们必须以往返方式遍历。 

微妙的部分是，我们依靠树结构来保证每个侧分支在使用时只进入和退出一次，因此$\max(p, q)$正确地模拟该绕道的最佳可能的单一遍历方向。 

## 工作示例

 考虑一个小链：$1 - 2 - 3$我们想要从 1 到 3。 

| 节点| 包含子树 | 边缘的贡献 |
 | ---| ---| ---|
 | 2 | 真实| 边 (1,2)：p |
 | 1 | 真实| 边 (2,3)：p |

 DFS 将两个节点都标记为包含目标，因此两条边都被迫朝节点 3 的前向方向移动。累积的结果是沿链的前向权重之和。 

这表明当图退化为路径时，算法简化为简单的方向和。 

现在考虑一个分支：$1 - 2 - 3$有一片额外的叶子$2 - 4$，并且只有节点 3 是目标。 

| 节点| 包含子树 | 边缘决策|
 | ---| ---| ---|
 | 4 | 假 | 在 (2,4) | 上取 max(p,q) |
 | 2 | 真实| (2,3) 强制，(2,4) 可选 |
 | 1 | 真实| (1,2) 被迫趋向 3 |

 包含节点 4 的子树并不位于通往 3 的路径上，因此我们只能将其用作绕道。 该算法正确地从边缘（2,4）提取最佳方向增益，而不影响主路径。 

这证明了强制路径结构和独立子树优化之间的分离。

 ## 边缘情况

 关键的边缘情况是目标节点直接连接到根。 在这种情况下，所有其他分支纯粹是可选的绕道，并且不得干扰主边缘。 该算法可以正确处理这个问题，因为只有子树包含目标的子节点才会贡献强制方向； 所有其他人均独立评估$\max(p, q)$。 

另一种情况是所有边权重均为负值。 一个幼稚的策略可能会完全避免边缘，但由于从 1 到$n$是强制性的，算法仍然在每一步选择破坏性最小的方向。 由于每条边都只处理一次，因此即使所有贡献均为负，结果仍保持最佳。 

最后一个微妙的情况发生在当有益循环似乎可能通过沿着分支返回时。 树结构可以防止重复利用，因为每个有向边最多被消耗一次，因此 DP 正确地将每个绕行限制为单个增益。
