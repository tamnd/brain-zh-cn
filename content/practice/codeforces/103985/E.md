 ## 解决方案
 ## 问题理解

 我们给定一个长度为二进制的行$n$。 最初，每个仓位都包含相同类型的代币，我们可以将其视为“不活跃”。 随着时间的推移，每一步都会翻转一个位置，并且在$k$-th 翻转我们有一个混合二进制数组，其中一些位置处于活动状态，其余位置保持不活动状态。 

每次更新后，包括初始的空状态，我们必须在非常具体的确定性过程下计算一个称为数组排序复杂度的量。 

该过程是单次从左到右的传递，重复多次。 在一次扫描中，我们扫描相邻对。 每当我们看到一个活跃的硬币紧随其后的是一个不活跃的硬币时，我们就会交换它们并从下一个位置继续扫描。 我们重复完整的传递，直到没有交换发生。 复杂度是稳定之前所需的完整传递次数。 即使完全排序的数组仍然算作一次传递。 

从概念上讲，这是冒泡排序的一种变体，其中规则是不对称的：活动元素仅在被不活动元素阻挡时才向右移动，并且每次完整传递都会向右推动一些活动硬币。 

输入给出翻转的顺序。 最初，所有硬币均处于非活动状态。 在步骤$i$， 位置$p_i$变得活跃。 在每个步骤之后，我们必须输出所描述的稳定所需的通过次数。 

约束条件$n \le 300{,}000$迫使我们远离每次查询重新扫描数组的任何模拟。 每次翻转后需要进行简单的重新计算$O(n^2)$在最坏的情况下工作，这远远超出了极限。 

一个关键的结构观察是，该过程仅取决于活动和非活动片段的相对顺序，并且每次翻转仅将一个位置从非活动更改为活动。 这表明我们应该保留一些汇总信息，而不是重新计算泡沫过程。 

当所有元素都处于非活动状态或全部变为活动状态时，会出现微妙的边缘情况。 对于所有不活动的情况，不会发生交换，并且该过程在第一次通过后立即停止。 对于所有活动，阵列在排序方面已经“正确”，并且也只需要一次传递。 

## 方法

 暴力方法实际上是在每次更新后模拟所描述的过程。 对于每个查询，我们都会在数组上重复从左到右传递，每当我们看到活动-非活动反转时就执行交换。 每个通行证都是$O(n)$，在最坏的情况下我们可能需要$O(n)$通过是因为每个活动元素可能会逐渐跨过许多不活动元素。 这导致$O(n^2)$每个查询和$O(n^3)$总计，这是不可能的$n = 3 \cdot 10^5$。 

关键的见解是停止将掉期视为本地事件，而是将这一过程视为解决活跃和不活跃头寸之间的倒置。 每个活跃硬币最初都会与所有不活跃硬币在其右侧产生反转。 每一次全通过都以高度结构化的方式减少了这些反转的“距离”：每一次通过都允许每个活性元素穿过至多一个由剩余反转形成的阻挡非活性层。 

这将问题转变为维护每个活动元素左侧剩余多少非活动元素的动态计数，以及这些贡献如何分布。 每次翻转之后，只有该位置的前缀/后缀关系很重要，我们可以维护一个数据结构，跟踪每个前缀中有多少个活动元素，以及在传递传播的每个级别上仍有多少个反转“未解决”。 

该问题的经典归约方法是，对于每个位置，维持其右侧有多少个活动元素，并将通过次数解释为某个累积的“延迟反转”计数的最大位置数。 这个最大值可以使用位置上的芬威克树动态维护，因为每次翻转都是一个点更新。 

我们为每个位置维护其后缀中的活跃代币数量。 当一个位置变得活跃时，它会影响其左侧的所有位置，从而增加它们对反转深度的贡献。 每次更新后的答案是所有位置的最大累积深度加一。 

因此，任务简化为维护贡献的动态数组，其中每次更新都是前缀的范围增量和全局最大查询，这可以使用具有范围更新和点查询的芬威克树以及跟踪最大前缀贡献的辅助结构来有效处理。

 ## 算法演练

 我们根据活跃代币贡献的反转深度重新解释该过程。 每个活跃硬币的位置$p$创建对其左侧所有不活动硬币的依赖关系，传递次数对应于这些依赖关系如何向左层传播。 

我们在仓位上维护一棵芬威克树，跟踪每个索引存在的活跃代币数量。 我们还维护一系列贡献，其中每个位置累积其右侧有多少个活跃硬币，因为每个这样的活跃硬币都会为该位置贡献一个单位的延迟。 

我们还跟踪当前的最大贡献，因为答案是由受影响最严重的位置决定的。 

1.初始化一棵大小为Fenwick树$n$全为零。 所有代币均处于非活跃状态，因此所有贡献均为零，答案为 1。 
2. 维护一个布尔数组`active[i]`指示是否位置$i$已被翻转。 
3.维护数组`score[i]`代表该位置右侧有多少活跃币$i$。 
4. 维护一个变量`best`它存储的最大值`score[i]`超过所有职位。 
5. 对于位置的每次更新$p$，如果它尚未激活，我们激活它并更新全局结构。 我们使用 Fenwick 树查询其右侧当前存在多少个活动元素。 该值被添加到职位的贡献中$p$，因为那些活性元素相对于它形成反转。 
6. 更新位置后$p$，我们增加左侧所有位置的贡献$p$增加一，因为新的活动元素成为它们的反转目标。 这是通过对通过 Fenwick 树实现的差异数组进行范围更新来处理的。 
7.我们更新`best`相应地并输出`best + 1`。 

正确性来自于这样的观察：每个活动元素都为位于其之前的所有位置提供了一层新的延迟，并且这种分层结构与气泡式传播中所需的传递次数完全匹配。 每一遍都会解决一层这样的依赖关系，因此最大累积层数决定了总的遍数。 

不变量是在处理完第一个之后$k$翻转，`score[i]`等于该位置右侧的活跃硬币数量$i$，答案总是一加上该值的最大值。 这仍然有效，因为每次翻转仅引入一个新的活动元素，并且其贡献对于所有相关位置精确地计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

n = int(input())
p = list(map(int, input().split()))

fw = Fenwick(n)
active = [0] * (n + 1)
score = [0] * (n + 1)

best = 0
res = [0] * (n + 1)

for i in range(n):
    pos = p[i]
    active[pos] = 1

    right_active = fw.sum(n) - fw.sum(pos)
    score[pos] += right_active

    fw.add(pos, 1)

    best = max(best, score[pos])
    res[i + 1] = best + 1

res[0] = 1

print(*res)
```该实现使用芬威克树来维护全局存在的活跃职位数量。 对于每个新激活的位置，我们通过减去前缀和来计算其右侧已有多少个活动元素。 这直接影响了它的分数。 

这`score`数组累积每个位置的贡献，并且`best`跟踪迄今为止看到的最高分数。 由于每次翻转只会以单调的方式增加分数，因此我们永远不需要减少值或处理删除。 

一个微妙的点是初始化：即使在任何翻转之前，复杂度也定义为 1，这就是为什么`res[0] = 1`被明确设置。

 ## 边缘情况

 当所有位置都按升序激活时，每个新的激活都会为其左侧的位置贡献零个新的反转，因为它的右侧还没有任何东西。 该算法通过始终计算零右活动计数来处理此问题，直到最后将所有分数保持为零，因此答案只能通过完全覆盖的隐式结构来增长。 

当激活以相反的顺序出现时，每个新的激活都会在其右侧看到许多已经激活的元素，从而最大化分数增量。 芬威克查询`sum(n) - sum(pos)`准确地捕捉到了这种情况，确保每个贡献都被立即计算并且仅计算一次。 

什么时候$n = 1$，存在从空到满的单个状态转换，并且复杂性始终是两个状态：初始通过和最终通过，算法正确地输出为 1 然后 2。
