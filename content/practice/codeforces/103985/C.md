 ## 解决方案
 ## 问题理解

 我们有一组王子和一组公主。 每位公主都会带着固定数量的嫁妆和两位她愿意嫁给的特定王子抵达。 如果最终的安排使用她，她必须与这两位王子中的一位恰好匹配。 每个王子最多可以搭配一位公主，每个公主最多可以使用一次。 我们可以自由地忽略任何王子或公主。 

目标是选择有效的王子-公主配对的子集，以便没有王子或公主被多次使用，并且所选公主的嫁妆总和最大化。 

这不是与单位权重的标准二分匹配。 关键的区别在于，每个公主都贡献一个权重，并且她的两个可能的边中的每一个都同样有效。 我们在一般图中选择一个匹配，其中右侧的每个节点（公主）的度数恰好为 2，并且我们想要最大权重匹配。 

约束条件允许最多 200,000 个王子和 200,000 个公主，因此任何三次甚至二次方法都立即不可能。 即使 O(nm) 的想法也会失败，因为 m 已经很大了。 这推动我们走向线性或近线性图处理，可能是 O(n + m) 或 O((n + m) log n)。 

天真的解释可能建议尝试所有公主子集或运行最大权重二分匹配。 在这种规模下这是不可能的。 

当多个公主争夺同一个王子时，就会出现一个微妙的问题。 例如，如果两个高价值公主的选项中都包含王子 1，则贪婪的选择可能会阻止全局更好的配置：

 输入：

 n = 1，m = 2

 公主1：(1, 1)无效，因为ai≠bi，所以调整：

 公主 1: (1, 2, 100)

 公主2：（1,2,1）

 如果我们在不考虑全局结构的朴素贪婪方案中先选择较小的一个，我们可能会失去值100的最佳匹配。冲突的结构是全局的，必须系统地处理。 

## 方法

 如果我们考虑暴力，每个公主都可以与她的第一个王子、第二个王子匹配，或者跳过。 这已经表明每个节点的分支因子最多为 3，从而导致指数级的复杂性。 即使限制于有效匹配，我们本质上也是在枚举通用图中的所有匹配，该通用图中随着边的数量呈指数增长。 

更结构化的强力方法是将其视为一般图中的最大权重匹配问题。 这可以用开花算法解决，但这里的图很特殊：每个公主的度数恰好为 2，王子形成二分结构的一侧，但只有一侧有约束。 

关键的观察是，王子形成的图表不是任意的。 每个公主恰好连接两个王子，因此每个公主可以看作两个王子节点之间的边，承载权重 w。 我们选择这些边的子集，这样没有王子会与多个选定的边相关联，从而最大化总权重。 这正是一个以王子为顶点、以公主为边的一般图上的最大权匹配问题。 

然而，出现了更深层次的结构：每个顶点（王子）都连接到边缘（公主），并且我们正在该图上选择匹配。 关键的简化是我们可以逐个组件地处理图形。 该图中的每个连通分量都是由王子和公主交替形成的，但由于每个公主的度为 2，而王子可以有任意度，因此该结构是一个通用图，但仍然具有一个关键属性：边是对顶点的独立选择，约束仅是顶点不相交。

这是一个经典的最大权重匹配问题，但在这里它变得容易处理，因为该图在最坏情况的对抗意义上不是任意的； 它可以使用权重贪婪过程结合联合查找或基于 DSU 的收缩思想来解决：以降序处理边缘，如果两个端点仍然空闲，则贪婪地获取边缘。 

我们的见解是，因为每个顶点（王子）只能使用一次，所以一旦我们选择一条高权重的边，它就会阻止与这些顶点相关的未来边。 按降序处理边可确保我们始终首先优先考虑较重的贡献，并且该结构不需要重新考虑，因为后面的边无法在不违反匹配约束的情况下改进先前选择的重边。 

这将问题简化为图上的经典最大权重匹配，其中贪婪排序是最佳的，因为缺乏会增加权重的交替路径改进，这在这个特定的 2 度左结构中成立。

 ## 算法演练

 我们将每位公主视为两个王子之间的边缘，其重量等于她的嫁妆。 我们想要选择一组不相交的边来最大化总重量。 

1. 将每个公主转换为一条边（ai，bi，wi）。 这将问题重新定义为选择加权边而不共享端点。 这种转换至关重要，因为它将叙述问题转变为图形优化问题。 
2. 按权重降序对所有边进行排序。 这确保了每当我们考虑一条边时，所有较重的边都已被决定。 直觉是，如果高值边可行，则应在任何低值冲突边之前获取它。 
3. 为每个王子维护一个布尔数组used[v]，指示他是否已经匹配。 最初一切都是假的。 
4. 按排序顺序迭代边。 对于每条边（u、v、w），检查两个端点是否未使用。 如果两者都是空闲的，请选择该边并将两个端点标记为已使用。 否则跳过它。 
5. 累加所有选定边的权重之和并输出。 

### 为什么它有效

 正确性来自于关于边排序的交换论证。 考虑与贪婪解决方案不同的任何最佳解决方案。 查看贪婪算法选择的最重边缘，而不是最佳解决方案选择的最重边缘。 该边必须通过其端点之一与另一条边匹配而在最佳解决方案中被阻止。 由于排序，该阻塞边的权重必须小于或等于当前边。 用当前边替换阻塞边不会减少总权重并保留可行性。 重复这个变换表明，贪婪解可以无损失地变换为最优解，因此它的权重是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    edges = []
    
    for _ in range(m):
        a, b, w = map(int, input().split())
        edges.append((w, a, b))
    
    edges.sort(reverse=True)
    
    used = [False] * (n + 1)
    ans = 0
    
    for w, a, b in edges:
        if not used[a] and not used[b]:
            used[a] = True
            used[b] = True
            ans += w
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将所有公主读取为加权边。 按权重排序是强制执行全局优先级的关键结构步骤。 这`used`数组强制每个王子最多参与一条选定的边的匹配约束。 

一个微妙的实现细节是，我们首先将权重存储在元组中，因此 Python 的默认排序直接按降序排列。 另一个重要的一点是我们永远不会重新审视边缘； 一旦使用端点，我们就会永久排除它，这与贪婪不变量一致。

 ## 边缘情况

 一个重要的边缘情况是多个高权重边缘共享一个王子。 例如，如果许多公主都包括王子1，则在与自由伙伴配对时，只有其中体重最高的才可能被选择。 贪婪排序确保首先考虑最重的边并锁定该王子，防止以后进行任何选择。 

另一种边缘情况是当权重稍低的边缘间接启用两个中等边缘时。 即使在这种情况下，延迟重边缘的任何优势都需要将其替换为涉及同一端点的多个边缘，这是不可能的，因为每个王子最多参与一个边缘。 该算法自然会处理这个问题，因为一旦消耗了一个顶点，未来的配置就无法重用它，因此不存在延迟增益。
