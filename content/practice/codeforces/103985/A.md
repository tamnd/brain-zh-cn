 ## 解决方案
 ## 问题理解

 我们有一张长方形的画，宽度为$w$和身高$h$。 两个相同的方形光源放置在这个矩形的左右垂直边上。 每个光源精确地照亮 90 度角区域，并且每个光的中心方向是水平的，这意味着每个光束围绕水平轴以 45 度角对称地上下传播。 

左侧灯置于高处$y_1$左边框，右灯置于高处$y_2$在右边界。 如果矩形内的每个点至少位于两个 90 度光锥之一内，则该点被视为点亮。 任务是确定矩形的每个点是否都被两个灯光中的至少一个覆盖。 

关键的困难在于每个灯不仅仅照亮垂直或水平的条带。 相反，每个区域都会创建一个三角形影响区域，当我们水平远离其源时，其垂直覆盖范围会线性扩展。 

输入尺寸限制很小，$w, h \le 10^6$，这会立即排除网格的任何离散化或对所有点的强力扫描。 任何解决方案都必须将几何形状减少为恒定数量的检查或导出封闭形式条件。 

一种简单的方法可能会尝试模拟沿每个垂直切片的覆盖范围$x$，保持照明间隔$y$- 坐标并检查它们的并集是否覆盖整个段$[0, h]$。 这会失败，因为它需要$O(w)$切片和区间合并，速度太慢。 

一个更微妙的问题是覆盖范围不断变化$x$。 照明区域的形状是分段线性的，因此如果不了解关键转变发生的位置，仅检查几个代表性位置显然不安全。 

## 方法

 如果我们固定一个垂直坐标$x$，左灯中心位于$(0, y_1)$照亮所有垂直坐标满足的点$$|y - y_1| \le x.$$所以在位置$x$，它贡献了间隔$[y_1 - x, y_1 + x]$。 

同样，右灯位于$(w, y_2)$照亮满足点$$|y - y_2| \le w - x,$$给出间隔$[y_2 - (w - x), y_2 + (w - x)]$。 

在每个$x$，这两个区间的并集必须覆盖整个段$[0, h]$。 同样，任何位置都不能有垂直间隙$x$。 如果最高下端点高于最低上端点，则存在间隙。 

所以我们定义：$$L(x) = \max(y_1 - x, \; y_2 - w + x),$$

$$R(x) = \min(y_1 + x, \; y_2 + w - x).$$矩形被完全覆盖当且仅当对于所有$x \in [0, w]$，我们有$L(x) \le R(x)$。 

现在观察这些函数的结构。 两个都$L(x)$和$R(x)$每个由两个线性函数组成。 两条线的最大值是V形函数，两条线的最小值是倒V形函数。 这意味着条件$L(x) \le R(x)$只能在活动线发生变化的点处切换，即 max 或 min 内的两个表达式相交的地方。 

这可以将问题从无穷多个中减少$x$值仅限于恒定数量的候选点。 

关键的$x$- 坐标是：

 端点$x = 0$和$x = w$，以及交点，其中：$$y_1 - x = y_2 - w + x \quad \Rightarrow \quad x = \frac{y_1 - y_2 + w}{2},$$

$$y_1 + x = y_2 + w - x \quad \Rightarrow \quad x = \frac{y_2 - y_1 + w}{2}.$$检查不等式$L(x) \le R(x)$在这些位置就足够了，因为在任何两个连续的临界点之间，$L(x)$和$R(x)$是线性的，因此它们的差异也是线性的，并且在端点处不过零的情况下不能改变符号。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力胜过一切$x$|$O(w)$|$O(1)$| 太慢了 |
 | 仅检查关键点 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算四个候选位置$x = 0$,$x = w$,$x_1 = (y_1 - y_2 + w)/2$， 和$x_2 = (y_2 - y_1 + w)/2$。 这些是覆盖范围结构可以改变的唯一点。 
2.忽略任何候选人$x$位于该段之外$[0, w]$，因为这些位置不对应于矩形的有效垂直切片。 
3. 对于每位有效候选人$x$，计算尽可能低的照明边界$L(x)$和最高可能的照明边界$R(x)$。 这给出了该切片的垂直覆盖间隔。 
4. 如果在任何候选点我们发现$L(x) > R(x)$，则该切片处存在垂直间隙。 这意味着矩形内有一个未被覆盖的区域，所以答案立即是“否”。 
5. 如果所有候选点都满足$L(x) \le R(x)$，那么它们之间不能形成任何间隙，因此矩形被完全覆盖，答案为“是”。 

### 为什么它有效

 固定覆盖范围$x$由每个光源的两个线性约束确定。 它们的包络线形成分段线性函数，每对交集至多有一个结构变化。 由于下包络线和上包络线都是由两条线构建的，因此每条线的所有可能的转换都由这些线的端点和交点确定。 在这些点之间，活动约束的顺序不会改变，因此任何覆盖范围的违反必须已经在采样位置之一可见。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(w, h, y1, y2):
    def eval_ok(x):
        # coverage interval at vertical slice x
        L = max(y1 - x, y2 - w + x)
        R = min(y1 + x, y2 + w - x)
        return L <= R

    candidates = [
        0,
        w,
        (y1 - y2 + w) / 2,
        (y2 - y1 + w) / 2
    ]

    for x in candidates:
        if 0 <= x <= w:
            if not eval_ok(x):
                return False
    return True

def main():
    w, h, y1, y2 = map(int, input().split())
    print("Yes" if check(w, h, y1, y2) else "No")

if __name__ == "__main__":
    main()
```实施直接遵循减少到恒定临界点。 辅助函数评估位置处是否存在垂直切片$x$下照明边界和上照明边界之间有间隙。 

唯一的微妙之处是对候选交点使用浮点运算。 这在这里是安全的，因为我们仅在将值插入线性表达式后比较值，并且精度要求完全在浮点精度范围内，值高达$10^6$。

import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    w, h, y1, y2 = map(int, input().split())

    def ok(x):
        L = max(y1 - x, y2 - w + x)
        R = min(y1 + x, y2 + w - x)
        return L <= R

    cand = [0, w, (y1 - y2 + w) / 2, (y2 - y1 + w) / 2]
    for x in cand:
        if 0 <= x <= w and not ok(x):
            print("No")
            return
    print("Yes")

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from contextlib import redirect_stdout
    import io as sio
    out = sio.StringIO()
    with redirect_stdout(out):
        solve()
    return out.getvalue().strip()

# provided samples
assert run("5 2 1 1\n") == "Yes"

 ## 边缘情况

 当两个灯处于相同高度时，会出现一种微妙的情况。 在这种情况下，包络线的交点会塌陷到对称位置，唯一可能的失败将发生在中点。 该算法显式地评估两个候选交点，在这种情况下它们是重合的，因此检查仍然有效。 

当一盏灯明显高于另一盏灯时，就会出现另一种边缘情况。 临界点严格位于内部$(0, w)$，并且单纯的仅端点检查会错过未覆盖的区域。 评估交叉点可以捕捉到这种精确的转变，其中较低的包络线将主导地位从一种光切换到另一种光。
