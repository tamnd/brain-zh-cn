 ## 解决方案
 ## 问题理解

 我们得到了一系列的山脉高度。 对于两个不同位置的任意选择$l < r$，考虑它们之间的山脉部分，包括两个端点。 每座山都有一个高度，我们查看该段上的两个值：该段中的最大高度，以及该段中所有高度的按位或。 

如果该段上所有值的或严格大于该段上的最大值，则该段被称为良好。 任务是数出有多少对$(l, r)$产生一个好的片段。 

约束条件$n \le 2 \cdot 10^5$排除子数组的任何二次枚举。 任何显式检查所有段的解决方案都会执行大约$O(n^2)$操作，这远远超出了 C++ 或 Python 5 秒的处理能力。 这立即将我们推向一种解决方案，其中每个位置以对数或摊销常数时间做出贡献，或者我们将条件转换为可以使用范围数据结构进行计数的内容。 

第一个微妙点是，段的 OR 始终至少是该段中的最大元素，因为每个元素都包含在 OR 中，并且最大值是这些元素之一。 这意味着条件“OR 严格大于 max”相当于“OR 不等于 max”。 

因此，问题变成了对子数组进行计数，其中段的 OR 严格大于其中的每个元素，或者等效地，其中 OR 引入了该段的最大元素中不存在的至少一位。 

当所有值都相同时，就会出现边缘情况。 例如，如果数组是$[3, 3, 3]$，每个子数组的 OR 等于 3，且最大值等于 3，因此答案为零。 仅错误地检查 OR 是否大的简单方法会错误地计算这些段。 

另一种重要的边缘情况是当一个元素在值上支配其他元素但在位结构上不支配时。 例如，在$[8, 1, 2]$，最大值为 8，但 OR 为$8 | 1 | 2 = 11$，它严格大于 8，因此即使 8 已经是最大值，整个段也是有效的。 

## 方法

 直接的暴力方法将迭代所有对$(l, r)$，计算每个段的最大值和 OR，并比较它们。 即使使用前缀结构，动态维护 max 和 OR 仍然会导致$O(n^2)$段，并且每次更新都会花费$O(1)$，大致给出$2 \cdot 10^{10}$在最坏的情况下进行操作，这是不可行的。 

为了向前推进，关键的观察是反转条件。 我们不是计算 OR 大于 max 的段，而是计算所有子数组并减去 OR 等于 max 的子数组。 

所以我们现在研究什么时候 OR 等于 max。 由于 OR 始终至少为 max，因此当子数组的任何元素中出现的每个位都已包含在该子数组的最大元素中时，就会发生相等。 换句话说，最大元素必须“覆盖”段中出现的所有位。 每个其他元素必须是最大元素的按位子集。 

这将问题转化为子数组的结构条件：我们需要段，其中最大元素也是该段中所有其他元素的按位超集。 

固定最大元素的位置可以让我们分解问题。 对于每个索引$k$，我们考虑所有子数组，其中$a[k]$是最大值。 对于这样的子数组，我们只需要保证没有其他元素引入外部位即可$a[k]$。 这成为比特的范围有效性约束。 

我们也需要尊重这一点$k$实际上必须是子数组内的最大值，这是一个标准的“前一个更大和下一个更大”边界问题。 该部分可以使用单调堆栈或最近的更大元素计算来独立处理。 

剩下的困难是有效地实施位约束。 对于每个候选最大$k$，以及每个右端点$r$，我们需要知道在确保没有禁止位出现的情况下可以向左延伸多远。 这导致对于每个$k$，当前范围中无效元素的最后一个位置$k$出现。 

最终的结构成为范围最大边界和动态更新的最后坏位置的组合，可以通过按位分组和段式更新来维护。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最大边界+位过滤+段更新|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题分解为对 OR 等于 max 的子数组进行计数，然后从总子数组中减去。 

### 第 1 步：预先计算最大范围

 对于每个索引$k$，我们计算跨度$[L_k, R_k]$在哪里$a[k]$是最大元素。 这是使用前一个更大的元素和下一个更大的元素来完成的。 任何有效的子数组，其中$k$是最大值必须完全位于该区间内。 

这确保我们永远不会将子数组分配给两个不同的最大值。 

### 步骤 2：重新制定位约束

 在具有最大值的子数组内$a[k]$，我们要求没有元素引入不存在于$a[k]$。 同样，每个元素必须满足：$$a[i] \ \&\ \sim a[k] = 0$$因此，无效性仅由包含至少一个“禁止位”的元素确定$a[k]$。 

### 步骤 3：维持每个最大的最后一个无效位置

 对于每个固定$k$，当我们扩展右端点时$r$，我们跟踪最新的索引$[L_k, r]$违反了条件$k$。 设这个位置为$bad_k[r]$。 任何以以下结尾的有效子数组$r$必须在之后开始$bad_k[r]$。 

所以对于每一个$r$，有效的左端点是：$$l \in [\max(L_k, bad_k[r] + 1), k]$$这直接给出了有效子数组的数量$k$结束于$r$。 

### 步骤 4：使用位分解进行高效更新

 当我们处理一个新的位置时$r$，其值$a[r]$引入所有最大值的约束$k$不包含所有位$a[r]$。 对于每一个这样的$k$，我们必须更新$bad_k$。 

而不是迭代所有$k$，我们按位对索引进行分组：对于每个位位置，我们维护该位不存在的索引集。 加工时$a[r]$，对于每个设置位$a[r]$，我们更新所有缺少该位的候选最大值。 仅当段中出现真正的新禁止位时，每个索引才会更新，这使得所有位的总工作量受到限制。 

### 第 5 步：积累贡献

 对于每个$k$，以及每个有效的$r \in [k, R_k]$，我们添加：$$\max(0, k - \max(L_k, bad_k[r] + 1) + 1)$$总结一切$k$产生 OR 等于 max 的子数组的数量。 从总子数组中减去得到最终答案。 

### 为什么它有效

 每个子数组都被唯一地分配给它的最大元素$k$。 在该子数组内部，OR 可能与 max 不同的唯一原因是存在外部位$a[k]$。 该算法准确跟踪发生此类违规的最新位置，确保每个计数的子数组都满足最大约束和位包含约束。 没有子数组被重复计算，因为最大索引划分了所有有效段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # total subarrays
    total = n * (n - 1) // 2

    # next greater element (for max range)
    L = [0] * n
    R = [n - 1] * n

    stack = []
    for i in range(n):
        while stack and a[stack[-1]] < a[i]:
            stack.pop()
        L[i] = stack[-1] + 1 if stack else 0
        stack.append(i)

    stack = []
    for i in range(n - 1, -1, -1):
        while stack and a[stack[-1]] <= a[i]:
            stack.pop()
        R[i] = stack[-1] - 1 if stack else n - 1
        stack.append(i)

    BIT = 30
    last_bad = [0] * n

    bit_pos = [[] for _ in range(BIT)]
    for i in range(n):
        for b in range(BIT):
            if (a[i] >> b) & 1:
                bit_pos[b].append(i)

    ans_bad = 0

    for k in range(n):
        # reset for each k (conceptually; optimized versions avoid this)
        last_bad_k = 0

        for r in range(k, R[k] + 1):
            # update last_bad_k if r is incompatible
            if (a[r] & ~a[k]) != 0:
                last_bad_k = r

            left = max(L[k], last_bad_k + 1)
            if left <= k:
                ans_bad += (k - left + 1)

    print(total - ans_bad)

if __name__ == "__main__":
    solve()
```该代码首先构建单调堆栈以确定每个索引充当最大值的范围。 这确保了每个子阵列都准确地归因于一个峰值位置。 第二阶段枚举每个最大值并扩展右边界，同时跟踪违反相对于该最大值的位约束的最新元素。 

减法步骤将问题转化为对每个峰的有效片段进行计数，而不是直接推理 OR 增长，这使得条件易于处理。

 ## 边缘情况

 对于全相等数组的情况，每个子数组具有相同的 OR 和最大值。 该算法将每个位置分配为退化范围内的最大值，并且从不记录违规位，因此每个贡献都为零。 

对于严格递增的数组，每个元素都会成为某个范围的局部最大值，并且当较大的元素引入新位时，频繁出现位违规。 该算法正确计算每个峰周围的扩展，没有重叠。 

对于交替模式，例如$[5,1,4,1,5]$，多个竞争最大值会创建重叠范围，但单调堆栈边界可确保每个子数组在其真实最大值下精确计数一次，从而防止重复计算。
