 ## 解决方案
 ## 问题理解

 我们在数轴上得到了一系列必须按固定顺序访问的交付点。 两名快递员从同一条线上的已知位置出发。 在每一步中，都会选择一名快递员前往下一个送货点，而另一名快递员则留在当前位置。 交货后，所选的快递员将到达该交货点，并且该过程将继续到下一个客户。 

分配决策是唯一的自由：对于每次按顺序的交付，我们决定是 Petya 还是 Vasya 执行它。 一旦分配了信使，他们就会从当前位置移动到所需的地点，并且他们的位置会永久更新，直到再次被选择为止。 

我们关心的数量是每个送货步骤后两个快递员之间的距离。 我们希望选择分配，以便整个过程中这些距离的最大值尽可能小。 

输入大小最多允许十万个投递点，坐标最大为10^9。 这立即排除了任何尝试显式模拟或比较所有可能分配的解决方案。 任何在每一步都对两种选择进行分支的方法都会导致 2^n 种可能性，这远远超出了可行的范围。 即使是跟踪两个信使在每一步的精确状态的动态编程也需要 O(n^2) 个状态（以简单的公式表示），这太大了。 

一个微妙的困难是，系统的状态不仅取决于当前的投递点，还取决于不动的快递员最后发送的位置。 这就产生了对历史的依赖，这使得天真的贪婪想法乍一看令人怀疑。 

一些边缘情况值得隔离。 

如果所有投递点都距离一位初始快递员非常近，但又远离另一位，则最佳策略可能会完全忽略一位快递员，因为移动两者只会不必要地增加距离。 例如，如果 s1 = 0，s2 = 1000，并且所有 xi 都接近 0，则最佳解决方案使 Petya 执行所有操作，并且答案由初始距离或从 Vasya 到路径的演化距离主导。 

另一个棘手的情况是当序列围绕初始位置左右交替时。 在这种情况下，频繁更换快递员会显着缩短最大距离，天真的“一个快递员处理大部分工作”策略就会失败。 

最后，如果最佳分配恰好切换快递一次或两次，则任何在不考虑未来几何形状的情况下提前提交的贪婪规则都可能是错误的。 

这些观察结果表明，核心困难在于平衡我们让一名快递员闲置的时间与其他快递员远离他们的距离。 

## 方法

 最直接的方法是尝试两个快递员之间所有可能的投递分配。 对于每个任务，我们模拟该过程并在每一步后跟踪两个位置，更新最大距离。 这是正确的，因为它完全遵循过程规则，但它需要检查 2^n 次分配，并且每次模拟都是 O(n)，导致指数时间。 

一个自然的改进是对交付前缀的动态编程。 在步骤 i，系统状态由每个快递员当前所在的位置确定，这取决于他们每个人执行的最后一次投递。 这导致由每个信使处理的最后一个索引定义的状态。 转换对应于将下一次交货分配给任一快递员。 虽然正确，但此公式创建了 O(n^2) 状态，因为每个快递员最后都可能访问过任何先前的递送或初始位置，并且对于 n 高达 100000 的情况，所有此类对之间的转换变得不可行。

简化问题的关键观察是，在任何步骤中，只有一名信使移动，而另一名则保持固定。 步骤 i 处的距离就是当前投递点与不活动快递员的最后位置之间的距离。 这意味着有关不活动信使的唯一相关信息是其当前位置，而不是其到达那里的完整历史记录。 

这导致了贪婪的观点。 在每个交货点，我们决定哪个快递员应该搬到那里。 该决定仅通过该快递员的新位置影响未来，而当前成本仅取决于其他快递员当前的距离。 系统始终由线上的两个活动点组成，并且每一步都会更新其中一个。 

这种结构允许一个简单的策略：始终将当前交货分配给当前位置更接近该交货点的快递员。 直觉是，移动更近的信使可以最小化信使之间的直接距离，并且因为每一步只有一个端点发生变化，所以延迟大的移动并不能在以后提供补偿好处。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分配模拟| O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 较上一位置满 DP | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 太慢了 |
 | 贪婪的最近快递分配 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维持两位快递员目前的立场。 最初是 s1 和 s2。 我们按顺序遍历交付点。 

1. 从代表信使当前位置的两个变量开始，初始化为 s1 和 s2。 初始距离已经是最大值的一部分，因为该问题考虑了整个过程。 
2. 对于每个投递点 xi，计算 xi 到每个快递员当前位置的距离。 这告诉我们哪个信使可以以较小的移动到达 xi ，也就是在这一步对系统干扰较小的信使。 
3.将xi分配给当前位置距离xi较近的快递员。 该快递员移动到 xi，因此我们将该快递员的位置更新为 xi。 其他快递保持不变。 
4. 更新位置后，计算两个快递员之间的距离，并用迄今为止看到的最大值更新答案。 
5. 继续，直到处理完所有交付点，然后输出记录的最大距离。 

它为何有效取决于国家的演变方式。 在任何时刻，系统都可以由一条线上的两点来完整描述。 当我们将一个信使移动到 xi 时，影响未来距离的唯一方法是更改​​这一对的一个端点。 选择较近的信使可确保新创建的配置使当前交付位置周围的点对尽可能紧密，从而防止过早形成并向前传播不必要的大间距。 任何替代选择都会用较远的端点替换较近的端点，这只会增加当前距离，并且不会为未来的步骤创建补偿结构优势，因为未来的决策仅取决于更新的端点位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, s1, s2 = map(int, input().split())
    x = list(map(int, input().split()))

    a = s1
    b = s2

    ans = abs(a - b)

    for xi in x:
        if abs(xi - a) <= abs(xi - b):
            a = xi
        else:
            b = xi

        if abs(a - b) > ans:
            ans = abs(a - b)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现为快递员的当前位置保留了两个变量。 在每一步中，它都会比较哪个快递员距离下一个送货点更近，并移动该快递员。 每次移动后都会更新答案，包括初始配置。 

一个微妙的点是，平局会被任意打破，有利于第一个信使，但任何一致的平局打破都是有效的，因为相等的距离意味着任一选择都保留相同的即时配置质量。 另一个细节是，在任何移动之前都包含初始距离，因为快递员开始分开，并且这种分开已经达到了最大值。

 ## 边缘情况

 一个重要的边缘情况是两个快递员到下一个投递点的距离相等。 在这种情况下，任一选择都会产生与递送点相同的直接距离，并且算法始终将其分配给第一个快递员。 由于状态在交换信使身份方面保持对称，因此这不会影响最终的最大距离。 

另一种边缘情况是所有交货点都位于两个起始位置的一侧。 在这种情况下，重复分配给最近的快递员实际上可以减少由一名快递员处理大部分或全部递送的情况，并且最大距离取决于不活动的快递员与移动中的快递员的距离。 贪心规则自然收敛于这种行为，无需特殊处理。 

当交付点在极左和极右交替时，就会出现最后一种边缘情况。 这里，算法会轮换快递员，确保没有一个快递员累积过多的位移。 每一步仅更新一个端点，并且通过始终保持更新的端点尽可能靠近新位置来维持距离界限。
