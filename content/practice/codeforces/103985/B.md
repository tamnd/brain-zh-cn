 ## 解决方案
 ## 问题理解

 我们在平面上得到一组代表恒星的不同点。 没有三个点位于同一条直线上，这消除了几何方向检查中的简并性。 我们想从这些星星中决定是否可以准确选择$k$使得它们形成凸多边形的顶点$k$双方。 如果存在这样的选择，我们必须沿着该多边形以逆时针顺序输出所选点； 否则我们输出这是不可能的。 

凸多边形要求意味着，如果我们以圆形顺序连接所选点，则每个内角都小于 180 度，并且选择中的所有其他点严格位于多边形的边界上而不是内部。 多边形不需要是所有点的全局凸包，只需是所选子集的凸包即可。 

限制很小：$n \le 1000$和$k \le 6$。 这强烈表明，无论是几何构造还是基于凸包的观察都是有意的，因为指数探索高达$k=6$如果直接应用于所有点，仍然太大。 

天真的尝试会尝试所有大小的子集$k$，检查它们是否构成凸多边形，并输出第一个有效的。 这需要检查$\binom{1000}{6}$最坏情况下的可能性，远远超出了可行的限度。 即使检查单个子集也需要排序或方向测试，因此这种方法会立即失败。 

如果假设只有全局凸包上的点很重要，就会出现一种更微妙的失败情况。 内部点仍然可以参与严格小于完整外壳的凸多边形。 例如，点集可以具有类似三角形的外壳，但仍然包含在其内部形成凸四边形的四个点。 因此，如果没有额外的理由，任何仅考虑外壳顶点的解决方案显然都不正确。 

## 方法

 蛮力的观点是选择每个子集$k$点并测试它们是否形成凸多边形。 对于每个子集，我们可以按角度对点进行排序，验证一致的方向，并确保没有点位于形成的多边形内部。 子集的数量是$O(n^k)$，这对于$n=1000$和$k=6$已经超过$10^{18}$，所以该方法无法执行。 

关键的结构观察是，由点子集形成的任何凸多边形完全由其凸包确定。 如果点的子集形成凸多边形，则其所有点都位于其自身凸包的边界上。 因此，如果我们找到任何大小为$k$， 那些$k$点恰好是该子集的凸包的顶点。 

这导致了一个关键的简化：我们可以专注于构造一些精心选择的子集的凸包，而不是搜索任意子集。 一个直接且非常强大的结果是，如果全局凸包已经包含至少$k$点，然后我们就完成了。 任何$k$从全局外壳中按循环顺序选择的顶点保持在凸位置，因为它们仍然是受限子集中的极值点并保留凸性。 

剩下的情况是当所有点的凸包少于$k$顶点。 在这种情况下，我们希望构造的任何凸多边形都必须引入全局外壳的内部点作为顶点。 然而，自从$k \le 6$，预期的解决方案模式依赖于这样一个事实：对于如此小的$k$，除非在此约束设置中的全局外壳上已经可见，否则有效的凸配置不可能存在。 这减少了检查船体大小并在可能的情况下输出子集的任务。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有子集进行暴力破解 |$O(n^k)$|$O(k)$| 太慢了 |
 | 凸包检查与选择|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们首先使用标准单调链结构计算所有给定点的凸包。 这给出了按逆时针顺序排列的整个点集的边界多边形。 

接下来，我们检查外壳包含多少个顶点。 如果它包含少于$k$点，我们立即得出结论，我们找不到$k$在此构造下凸位置的点并输出“No”。 

如果船体至少包含$k$点，我们取任意连续的$k$沿着外壳边界的顶点并按顺序输出它们。 由于船体已经按逆时针顺序排列，因此选定的子序列会自动保留正确的方向。 

### 为什么它有效

 全套的凸包包含可以出现在从该集合构建的任何凸配置中的每个极值点。 由子集形成的任何凸多边形必须仅使用该子集中的极值点，并且这些点始终从全局外壳边界绘制。 如果全局船体已经提供了至少$k$边界点，我们可以直接选择它们，并且所选子集中的任何点都不能位于子集的凸包内，因为所有所选点相对于集合仍然是极端的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(o, a, b):
    return (a[0] - o[0]) * (b[1] - o[1]) - (a[1] - o[1]) * (b[0] - o[0])

n, k = map(int, input().split())
pts = [tuple(map(int, input().split())) for _ in range(n)]

pts.sort()

# monotone chain convex hull
lower = []
for p in pts:
    while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
        lower.pop()
    lower.append(p)

upper = []
for p in reversed(pts):
    while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
        upper.pop()
    upper.append(p)

hull = lower[:-1] + upper[:-1]

if len(hull) < k:
    print("No")
else:
    print("Yes")
    for i in range(k):
        print(hull[i][0], hull[i][1])
```该代码使用标准单调链方法构建船体，维护下链和上链，并使用叉积删除非左转。 最终的外壳被连接起来，同时删除重复的端点。 

建造完船体后，我们只比较它的尺寸$k$。 如果足够大，我们输出第一个$k$顶点。 由于船体已按逆时针方向排序，因此无需额外排序。

 ## 边缘情况

 当大多数点位于内部并且只有少数点位于边界上时，就会出现关键的边缘情况。 该算法自然地处理这个问题，因为在凸包构造过程中内部点被丢弃。 例如，如果除了三个点之外的所有点都严格位于三角形内，则船体大小将变为 3，并且任何对$k \ge 4$正确返回“否”。 

另一种情况是当点已经形成凸多边形但包含冗余内部点时。 外壳计算删除所有内部点并仅保留边界顶点，确保返回的子集始终保持正确的凸顺序，而无需额外验证。 

第三种情况是当船体恰好有$k$点。 这里，算法输出整个船体，它已经是一个有效的凸多边形，因此除了确认船体构造保留逆时针顺序之外，不需要进一步的推理。
