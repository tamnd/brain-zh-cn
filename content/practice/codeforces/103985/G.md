 ## 解决方案
 ## 问题理解

 我们得到了最终可见的糖果包装纸序列，这些糖果包装纸按照吃糖果的顺序排列。 每个包装纸要么是红色的，要么是蓝色的，要么是未知的。 未知符号意味着我们无法看出吃掉的糖果是红色还是蓝色。 重要的隐藏机制是在吃之前如何选择糖果。 

每走一步，都会出现一个盒子，里面装有一定数量的红色和蓝色糖果。 食用者总是从当前数量较多的颜色中挑选。 如果两种颜色的数量相同，则选择红色。 吃完一颗糖果后，它的包装纸就会被添加到最终的序列中。 我们仅观察带有部分信息的最终序列，并且我们想要计算盒子的多少种不同的初始配置可以产生该序列的某种完成。 

配置仅取决于最初存在的红色和蓝色糖果的数量。 如果这些计数不同，则两个盒子是不同的。 我们被要求计算可以分配多少对非负整数，以便存在一种有效的方法来解释所有未知的包装器并在贪婪规则下重现观察到的序列。 

限制 n 高达 200000 立即排除任何试图模拟所有可能的初始状态或所有问号分配的方法。 如果我们直接枚举配置，即使每个候选配置的 O(n²) 或 O(n log n) 也太慢了。 该解决方案必须将问题减少到少量候选状态或通过线性扫描计算公式。 

当序列与贪婪规则不一致时，就会出现微妙的边缘情况。 例如，如果在某个前缀处观察到的颜色迫使贪婪规则选择与所需的主导条件相矛盾的颜色，则答案必须为零。 另一个边缘情况是当所有字符都未知时，因为可能适合多个初始状态，但由于确定性打破平局规则总是更喜欢红色，因此并非所有比率都有效。 

第二种不明显的情况是当序列完全确定时（没有“？”），其中只有一个初始配置可能起作用或根本不起作用，并且贪婪规则可以唯一地重建前缀差异所需的不等式。 

## 方法

 蛮力的想法是尝试所有可能的红色和蓝色糖果的初始计数。 对于每一对（R，B），我们逐步模拟该过程。 在每一步中，我们都会根据贪心规则检查必须选择哪种颜色，并验证它是否与观察到的字符匹配，或者如果是“？”则可以分配。 如果我们遇到矛盾，则该初始对无效。 

这是可行的，因为一旦初始计数固定，该过程就是确定性的。 然而，可能的（R，B）对的范围并没有以有用的方式受到限制。 原则上，R和B可以与n一样大，因此有O(n²)个候选状态。 每次模拟的成本为 O(n)，总复杂度为 O(n3)，这对于 n 高达 200000 的情况是完全不可行的。 

关键的观察结果是，该过程仅取决于演化过程中红色和蓝色计数之间的差异。 每一步都会通过从所选颜色中减去一个来更新差异。 贪心规则仅取决于此差异的符号，并具有固定的打破平局规则。 我们可以将过程视为受观察序列约束的差异变量的遍历，而不是跟踪完整计数。 每个有效的初始配置对应于一个一致的初始差值，该初始差值从不违反贪婪选择条件。

这将问题简化为检查哪些初始差异与序列约束兼容。 每个位置都会对当前差异施加不等式，并且这些不等式线性传播。 整个序列成为对单个整数参数的一组约束，我们计算有多少整数值同时满足所有约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（尝试所有初始计数）| O(n3) | O(1) | O(1) | 太慢了 |
 | 差异的约束传播 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们在每个步骤之前的任何时刻根据差值 d = R - B 重新构建该过程。 如果 d > 0，则必须选择红色。 如果 d < 0，则必须选择蓝色。 如果 d = 0，则由于平局决胜而选择红色。 

每一步消耗一颗糖果，因此如果选择红色，则 d 减少 1，如果选择蓝色，则增加 1。 

我们不知道哪些初始 d 值是有效的，但我们知道在处理完所有步骤后，d 已完全确定。 问题变成计算有多少起始值导致与所有观察到的约束一致的序列。 

我们维护两个界限，L 和 R，描述处理序列前缀后可能的当前差异的范围。 

### 步骤

 1. 对可能的初始差 d₀ 初始化 L = -∞ 和 R = +∞。 

这表示在应用任何约束之前，任何整数差都是可能的。 
2. 从左到右扫描序列，在每一步后保持可能的电流差异范围。 

在每个位置，我们将观察结果转化为对当前差异的约束。 
3. 如果字符是“r”，那么此时我们必须有 d ≥ 0，因为当 red 不严格小于时会选择 red。 

选择红色后，新的差值变为 d - 1，因此我们一致地向前传播约束。 
4. 如果字符是“b”，那么我们必须让 d < 0，因为只有当蓝色严格较多或红色不占主导地位时才会选择蓝色。 

选择蓝色后，新的差值变为d+1。 
5. 如果字符是“?”，则两种转换都可能，具体取决于 d。 这又分为两种情况：

 当 d ≥ 0 时，我们的行为类似于“r”，当 d < 0 时，我们的行为类似于“b”。 我们将两个结果约束合并到一个统一的区间中。 
6. 处理完所有位置后，最终的区间[L,R]代表所有有效的初始差值d₀。 答案是该区间内的整数个数，即 max(0, R - L + 1)。 

### 为什么它有效

 在每一步中，贪心规则都会根据当前差值的符号将整数行划分为两个区域。 每个观察到的字符要么强制执行一个区域，要么允许两个区域。 由于 d 的更新是线性且单调的，因此每一步之后可行的初始值集合都保持连续的区间。 不会出现不相交的可行区域，因为每个步骤仅应用具有单个阈值分割的仿射变换。 这保证了仅跟踪最小和最大可行起始差异就足够了，并且不会遗漏或重复计算有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    # We track possible current difference intervals [lo, hi]
    lo, hi = 0, 0  # current difference after 0 steps starts at 0 shift baseline

    for ch in s:
        if ch == 'r':
            # must have d >= 0 before choosing red
            new_lo = max(lo - 1, 0)
            new_hi = hi - 1
        elif ch == 'b':
            # must have d < 0 before choosing blue
            new_lo = lo + 1
            new_hi = min(hi + 1, -1)
        else:
            # '?' can be either
            # combine both transitions
            # red case
            r_lo = max(lo - 1, 0)
            r_hi = hi - 1
            # blue case
            b_lo = lo + 1
            b_hi = min(hi + 1, -1)

            new_lo = min(r_lo, b_lo)
            new_hi = max(r_hi, b_hi)

        lo, hi = new_lo, new_hi
        if lo > hi:
            print(0)
            return

    print(max(0, hi - lo + 1))

if __name__ == "__main__":
    solve()
```该代码在每个前缀之后维护一个可行的当前差异间隔。 对于红色观察，它强制步骤之前的差值必须为非负数，然后通过减一进行移动。 对于蓝色，它会通过加 1 来强化负性并发生变化。 对于未知值，它合并了两种可能性。 间隔更新对贪婪选择规则和单次线性传递中的差异演化进行编码。 

一个微妙的实现问题是在强制执行符号约束时处理边界。 必须在移位之前应用零处的分割，否则变换会混合不兼容的状态。 另一个微妙之处是，在合并“?”时，必须仔细计算可达区间的并集； 未能完全采用两个分支可能会低估有效的初始配置。

 ## 边缘情况

 一个关键的边缘情况是当序列立即与贪婪规则相矛盾时。 为了`"rr"`，在第一个红色之后，状态会强制第二个红色出现负差或无效差，从而消除所有可能性。 该算法通过处理第二步后间隔变空来检测这一点。 

另一种情况是一个长序列`"?"`人物。 该间隔会扩大一段时间，但仍然是连续的，因为每一步仅在零附近移动和反映。 该算法正确地累积了所有可能的初始差异，而无需显式枚举它们。 

最后一种情况是交替强制选择，例如`"rbrb..."`，其中每一步都会收紧间隔，但不允许分支。 该算法维持单个演化的可行范围，并且最终计数精确对应于与由交替引起的所有奇偶校验约束一致的整数初始差异的数量。
