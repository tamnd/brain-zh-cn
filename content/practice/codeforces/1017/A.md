 ## 解决方案
 ## 问题理解

 我们有一个班级的学生，每个班级都由一个从 1 开始的唯一整数 id 标识。每个学生都有四个对应于不同科目的考试成绩。 任务是根据四个科目的总分对所有学生进行排名。 总分越高意味着排名越好。 当两个学生的总分相同时，通过给予 ID 较小的学生较高的排名来打破平局。 

我们需要的输出是 id 为 1 的学生 Thomas 的排名。 排名定义为从 1 开始的排序顺序中的位置。 

限制很小，最多1000名学生。 这立即告诉我们，O(n²) 方法完全没问题，甚至直接基于排序的 O(n log n) 解决方案也有些过大，但干净且安全。 除了简单的聚合和排序之外，无需进行优化。 

这个问题的主要微妙之处在于平局打破规则。 如果我们忘记了当总和相等时较小的 id 获胜，我们可能会错误地将托马斯放在得分相同的人后面。 另一个微妙的点是，排名取决于最终排序列表中的位置，而不是计算有多少人仅在错误排序后才获得严格更高的分数。 仅严格计算较高总数而不正确处理平局的幼稚实现将会失败。 

当托马斯与多个学生的分数相同时，就会出现一个具体的失败案例。 假设托马斯和学生 5 的总数都是 400，而学生 2 的总数是 401。正确的排名是学生 2 排在第一位，然后托马斯排在学生 5 之前，因为 id 较小。 如果我们忽略 id 平局打破，Thomas 的排名可能会被错误计算为 3，而不是 2。 

## 方法

 思考这个问题最简单的方法是计算每个学生的总分，然后根据描述的规则对所有学生进行排序。 排序后，我们找到了托马斯并读出了他的位置。 

一种暴力方法是反复扫描所有学生并选择下一个尚未选择的最佳候选人。 这模仿了选择排序行为。 每个选择步骤都需要 O(n) 扫描，我们执行 n 次，从而产生 O(n²) 操作。 当n达到1000时，这大约是一百万次比较，这仍然可以接受但没有必要。 

更直接的方法是计算所有总计，用 id 存储它们，然后使用 Python 的内置排序进行排序。 关键的见解是排名顺序完全由单个键确定：负总分（用于降序），然后是 id（用于升序）。 这将问题转换为标准的排序任务。 

排序后，托马斯的排名就是他在排序列表中的索引加一。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力选择 | O(n²) | O(n) | 已接受 |
 | 按键排序| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 读取学生人数并存储他们的分数。 
2. 对于每个学生，计算四门科目成绩的总和。 这将每个学生减少到一个可比较的值。 
3. 将每个学生存储为由负和和 id 组成的一对。 使用负数以便按升序排序会产生降序分数顺序。 
4. 对学生列表进行排序。 排序会自动应用辅助规则，因为 id 包含在元组中。 
5. 扫描排序列表并找到 id 1 对应的条目。 
6. 输出其从 1 开始的位置作为排名。 

它之所以有效，是基于这样一个事实：按元组排序会强制执行字典顺序。 元组 (-sum, id) 确保较大的总和排在前面，而在相等的总和中，较小的 id 排在前面。 由于每个学生只代表一次，并且排序是完整且一致的，因此排序后的列表正是所需的排名。 因此，托马斯在此列表中的位置是他的正确排名。

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    students = []

    for i in range(1, n + 1):
        a, b, c, d = map(int, input().split())
        total = a + b + c + d
        students.append((-total, i))

    students.sort()

    for idx, (_, sid) in enumerate(students, start=1):
        if sid == 1:
            print(idx)
            return

if __name__ == "__main__":
    main()
```该解决方案构建一个元组列表，其中每个元组对两个排名标准进行编码。 排序步骤处理所有排序逻辑，从而避免手动比较错误。 最后的循环仅搜索 id 1，考虑到较小的约束，这是安全的。 

一个常见的错误是忘记在排序键中包含 id。 如果没有它，相等的分数可能会以任意顺序出现，这将破坏排名定义。

 ## 边缘情况

 当所有学生的总成绩相同时，排名完全取决于id排序。 例如，三个学生的得分均为 40，则排序顺序必须为 id 1, 2, 3。Thomas 始终是同等水平中的第一名，因此他的排名为 1。算法可以正确处理此问题，因为 id 是元组中的辅助键。 

当托马斯得分最低时，他应该出现在排序列表的末尾。 例如，如果 Thomas 的总数为 10，而其他人的总数为 20 和 30，则排序顺序变为 30, 20, 10，将他置于排名 3。排序键自然地确保了这一点，无需特殊处理，因为 -10 大于 -30 和 -20，将他推到后面的顺序。
