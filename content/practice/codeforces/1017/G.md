 ## 解决方案
 ## 问题理解

 我们正在处理一棵有根树，其中顶点 1 是根，每个节点最初都是白色的。 随着时间的推移，我们应用三种操作，要么翻转颜色，重置树的部分，要么询问节点的当前颜色。 

第一个操作是最不寻常的：当我们尝试将顶点着色为黑色时，只有当顶点当前为白色时才会成功。 如果已经是黑色的，我们不会停止或报告失败。 相反，我们将相同的操作重定向到其所有子级，并在那里重复相同的规则。 这会创建一种级联行为，将工作向下推动，直到找到白色节点。 

第二个操作就比较简单了。 它迫使整个子树变成白色，擦除其中所有以前的黑色标记。 

第三个操作是点查询，询问给定顶点当前是黑色还是白色。 

关键的困难在于操作可以从单个节点扩展到许多递归尝试，并且这种扩展取决于树的当前状态，该状态由于重置和先前的操作而动态变化。 

约束允许最多 100,000 个节点和 100,000 个查询。 这立即排除了在最坏情况下明确地逐个操作树的任何解决方案。 在操作中重复重定向的单链可能会退化为接触几乎每个节点，并且在查询中重复，这会变成二次方。 

基于 DFS 的简单模拟也会失败，因为子树重置与先前的颜色状态交互，从而使重新计算无效。 

当对已经全黑的子树调用重复操作时，会出现微妙的边缘情况。 例如，如果我们有一条链 1 → 2 → 3，并且都是黑色的，那么在节点 1 处应用操作 1 将继续向下推动该操作，直到它到达叶子或用完子节点。 幼稚的实现可能会过早停止或错误地重新访问节点。 

频繁的子树重置会出现另一个极端情况。 如果我们重置子树，然后立即对其应用级联操作，我们必须确保不依赖于辅助结构中存储的陈旧“黑色”状态。 

## 方法

 直接模拟维护每个节点的颜色并通过遍历树来执行每个操作。 操作二可以通过子树上的 DFS 来完成。 操作一可以通过重复检查节点并在需要时下降到子节点来完成。 

这工作正常，但成本是问题。 在最坏的情况下，单个操作一可能会遍历一条长链甚至许多分支，而操作二可能会触及大型子树中的所有节点。 对于 100,000 个查询，这会导致最坏情况的行为约为 10^10 次操作，这远远超出了限制。 

关键的观察是我们需要一个有效支持两种想法的结构。 首先，我们需要支持“激活节点，或者如果已经激活，则向下推动激活”。 其次，我们需要支持子树重置，使之前的激活快速失效。 

这种组合强烈建议将树的欧拉游览表示与支持范围分配和点查询的数据结构相结合。 但是，操作一不是标准更新，因为它根据当前状态有条件地下降。 

关键的见解是将黑色节点视为动态集，并为每个节点保持在我们尝试激活它时快速找到其子树路径中的下一个白色节点的能力。 这自然是使用欧拉阶上的线段树来处理的，该线段树跟踪节点当前是白色还是黑色，并支持对白色的范围分配和对黑色的点更新。

为了有效地处理“推送到孩子”行为，我们发现我们不需要显式遍历所有孩子。 相反，当我们尝试激活黑色节点时，我们会重复定位子树中的第一个白色节点。 这可以通过维护一棵线段树来支持，该线段树存储节点是否为白色，并支持在某个范围内查找值为 1 的节点。 

因此，操作一变成重复的“在子树中找到下一个白色节点，将其标记为黑色”，并在相关路径中不存在时停止。 

这将级联 DFS 转换为重复的对数查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力DFS模拟| 最坏情况 O(nq) | O(n) | 太慢了|
 | 欧拉之旅 + 具有下白查询的线段树 | O(q log n) | O(q log n) | O(n) | 已接受 |

 ## 算法演练

 我们以树为根并计算欧拉之旅，以便每个子树对应于一个连续的段。 

然后我们在这个欧拉数组上构建一棵线段树，其中每个位置存储对应的节点当前是否为白色。 

我们还维护一个将欧拉索引映射回节点标签的数组。 

### 步骤

 1. 从根开始执行 DFS 来计算进入时间`tin[v]`和退出时间`tout[v]`。 

这确保每个子树都成为连续的段`[tin[v], tout[v]]`。 
2. 构建线段树`[1, n]`初始化为全 1，意味着所有节点都是白色的。 
3. 要处理操作类型3（查询），只需检查位置的当前值`tin[v]`。 如果为1，则该节点为白色，否则为黑色。 
4. 要处理操作类型 2（将子树涂成白色），请应用 1 的范围分配`[tin[v], tout[v]]`。 

这会在对数时间内将子树中的所有节点恢复为白色。 
5. 要处理操作类型 1，请重复尝试以下操作：

 在以 为根的子树中查找任意白色节点`v`。 如果不存在，则停止。 

否则，将该节点标记为黑色并重复。 

我们搜索白色节点而不是遍历子节点的原因是，“如果是黑色，则转到子节点”规则实际上意味着我们跳过已经激活的节点并继续深入，直到找到有效的候选节点。 
6. 每次我们在欧拉索引处找到一个白节点`i`，我们将其设置为黑色并继续该过程。 我们总是将搜索限制在子树区间内，以确保正确性。 

### 为什么它有效

 关键的不变量是线段树总是准确地反映欧拉顺序中节点的当前着色。 每个子树操作对应一个连续的段更新，每个查询对应一个直接查找。 

对于操作一，我们永远不会跳过子树区间中的有效白色节点，因为线段树总是返回正确的存在查询。 一旦节点被标记为黑色，它将永远不会再次返回，直到子树重置将其恢复为白色。 这与重复下降的语义完全匹配：每个黑色节点都被永久绕过，我们只选择在处理时仍然是白色的节点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

class SegTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.tree = [0] * (2 * self.size)

    def build(self):
        for i in range(self.size):
            self.tree[self.size + i] = 1
        for i in range(self.size - 1, 0, -1):
            self.tree[i] = self.tree[2*i] + self.tree[2*i+1]

    def point_set(self, i, val):
        i += self.size
        self.tree[i] = val
        i //= 2
        while i:
            self.tree[i] = self.tree[2*i] + self.tree[2*i+1]
            i //= 2

    def range_set(self, l, r):
        # set all to 1 in range via naive segment tree recursion
        self._range_set(1, 0, self.size - 1, l, r)

    def _range_set(self, v, tl, tr, l, r):
        if l > r:
            return
        if l == tl and r == tr:
            self.tree[v] = (tr - tl + 1)
            return
        tm = (tl + tr) // 2
        self._range_set(v*2, tl, tm, l, min(r, tm))
        self._range_set(v*2+1, tm+1, tr, max(l, tm+1), r)
        self.tree[v] = self.tree[v*2] + self.tree[v*2+1]

    def find_white(self, l, r):
        return self._find(1, 0, self.size - 1, l, r)

    def _find(self, v, tl, tr, l, r):
        if l > r or self.tree[v] == 0:
            return -1
        if tl == tr:
            return tl
        tm = (tl + tr) // 2
        res = -1
        if l <= tm:
            res = self._find(v*2, tl, tm, l, min(r, tm))
        if res == -1 and r > tm:
            res = self._find(v*2+1, tm+1, tr, max(l, tm+1), r)
        return res

n, q = map(int, input().split())
g = [[] for _ in range(n+1)]
parent = list(map(int, input().split()))
for i, p in enumerate(parent, start=2):
    g[p].append(i)

tin = [0]*(n+1)
tout = [0]*(n+1)
timer = 0

def dfs(v):
    global timer
    tin[v] = timer
    timer += 1
    for to in g[v]:
        dfs(to)
    tout[v] = timer - 1

dfs(1)

st = SegTree(n)
st.build()

for _ in range(q):
    t, v = map(int, input().split())
    if t == 1:
        while True:
            idx = st.find_white(tin[v], tout[v])
            if idx == -1:
                break
            st.point_set(idx, 0)
    elif t == 2:
        st.range_set(tin[v], tout[v])
    else:
        print("black" if st.tree[st.size + tin[v]] == 0 else "white")
```DFS 部分构造了欧拉排序，因此子树查询变成了区间操作。 线段树为每个节点存储一个二进制状态，其中 1 表示白色，0 表示黑色。 

操作一重复搜索子树区间中的任何白色节点并将其翻转为黑色。 这直接模拟了“跳过黑色节点，转到子节点”规则，而无需显式遍历邻接表。 

操作二执行范围分配以恢复白色。 这是通过简单的递归线段树更新来实现的。 

操作三读取该节点对应的叶子值。 

一个微妙的点是，这里的线段树更多地被用作动态存在结构，而不是惰性传播系统。 正确性依赖于始终查询和更新一致的欧拉索引，而不是直接依赖树结构。

 ## 边缘情况

 一个关键的边缘情况是一条长链，其中每个节点在应用操作一之前都已经是黑色的。 在这种情况下，算法会重复查询线段树，但找不到白色节点，因此会立即终止，而不循环遍历节点。 这可以防止不必要的遍历。 

另一种边缘情况是在重叠区域上重复子树重置。 由于每次重置都会将线段树范围覆盖回 1，因此之前的任何黑色状态都将被完全丢弃，后续查询仅依赖于最新的分配。 

第三种边缘情况是没有子节点的节点，其中在黑色时应用操作一。 搜索在单例间隔上立即失败，并且不会发生无效下降，这符合“没有孩子意味着停止”的预期行为。
