 ## 解决方案
 ## 问题理解

 我们得到一个数字$n$，我们必须构造一个整数的排列$1$到$n$。 在所有这些排列中，我们被要求最小化排列上定义的数量。 

该数量是两个经典子序列测量的总和。 一是最长递增子序列的长度，意味着值在遵守索引顺序的同时严格递增的位置的最长序列。 另一个是最长递减子序列的长度，其定义类似，但具有严格递减的值。 

任务不是计算给定排列的这些值，而是构造一个排列，使它们的总和尽可能小。 

约束条件允许$n$最多$10^5$，这立即排除了任何尝试评估所有排列的 LIS 和 LDS 的方法。 即使是单个 LIS 计算$O(n \log n)$，并且排列的数量是阶乘的，因此穷举搜索是不可能的。 任何解决方案都必须直接在线性时间内或接近线性时间构造排列。 

一个微妙的陷阱是假设最小化 LIS 自动帮助 LDS，反之亦然。 例如，身份排列$[1, 2, 3, \dots, n]$有LIS$n$和摩门教$1$，而反向有 LIS$1$和摩门教$n$。 两者都产生了大笔资金$n+1$，所以极端的排序并不是最佳的。 

一个较小的例子更清楚地说明了这个问题。 为了$n = 4$，排列$[1,2,3,4]$给出总和$5$，但是重新排列的结构如$[3,4,1,2]$给出 LIS$2$和摩门教$2$，总计$4$。 这表明平衡结构比优化一个方向更重要。 

## 方法

 一个蛮力的想法是枚举所有排列并计算每个排列的 LIS 和 LDS。 每次评估费用$O(n \log n)$，并且有$n!$排列，所以即使对于$n = 10$。 

稍微更明智的暴力方法可能会尝试随机排列并保持最佳结果，但不能保证最优性，并且搜索空间仍然巨大。 

关键的观察结果是，当排列具有长单调结构时，LIS 和 LDS 都会变大。 严格递增数组使 LIS 最大化，严格递减数组使 LDS 最大化。 目标是避免任一方向上的长单调片段。 

最佳构造来自于强制排列为同时限制递增和递减子序列的块结构。 如果我们将数组分成两半，先放置后半部分，再放置前半部分，我们就会创建一个模式，其中值是局部排序的，但全局“混合”足以防止形成长单调链。 

更一般地，将元素排列在交替的高低块中可确保任何递增子序列最多可以从每个块中选取一个元素，并且相同的限制对称地适用于递减子序列。 这将 LIS 和 LDS 限制为大约$\lceil n/2 \rceil$，这是最小的舍入效果。 

这导致了一个简单的贪心构造：首先按升序放置较大的数字，然后按升序放置较小的数字。 这产生了 LIS 和 LDS 都受到严格约束和平衡的排列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n! \cdot n \log n)$|$O(n)$| 太慢了|
 | 最佳施工|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 拆分数字$1$到$n$分成两个连续的两半。 

目的是分离值，以便在任何单调子序列中强制进行大的跳跃。 
2. 按升序先输出后半部分的所有元素。 

这确保了较小的值出现在排列中的较晚位置，从而打破了长的递增链。 
3. 接下来按升序输出前半部分的所有元素。 

这保留了内部顺序，但阻止它扩展先前的递增结构。 
4. 返回结果序列作为排列。 

关键思想是每一半内的排序是单调的，但两半之间的全局过渡会造成值范围的不连续性。 任何增加的子序列在由于值差距而被迫重置其增长之前最多只能采用一个连续的值块。 

### 为什么它有效

 该构造强制执行值分离：后半部分中的每个元素都大于前半部分中的每个元素，但出现在序列中的较早位置。 值顺序和位置顺序之间的这种反转可以防止长单调子序列跨越两半。 因此，LIS 和 LDS 的大小大致为单个一半的大小，这是单个分割结构下可实现的最小规模。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())

left = list(range(1, n // 2 + 1))
right = list(range(n // 2 + 1, n + 1))

# print right half first, then left half
res = right + left

print(*res)
```这段代码直接实现了上面描述的构造。 分割点是$n // 2$，并且两半都保持递增顺序。 首先打印右半部分，以确保较大的值在排列中出现较早，这是限制单调子序列的关键结构反转。 

结构故意简单，因为困难不在于计算，而在于认识到单个分割足以同时最小化 LIS 和 LDS。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n = int(input())
    left = list(range(1, n // 2 + 1))
    right = list(range(n // 2 + 1, n + 1))
    res = right + left
    return " ".join(map(str, res)).strip()

 ## 边缘情况

 对于$n = 1$，该构造产生单元素排列，因此 LIS 和 LDS 均为 1，并且总和非常小。 

为了$n = 2$，分裂得到左 = [1]，右 = [2]，产生 [2,1]。 LIS 为 1，因为按顺序不存在递增对，而 LDS 为 2，因为整个序列是递减的。 任何排列的总和必须为 3，因此这是最优的。 

对于小奇数$n$，分割会产生不均匀的两半，但同样的原理成立。 为了$n = 3$，我们得到 [2,3,1]，其中 LIS 为 2，LDS 为 2。这种不平衡不会影响正确性，因为结构断裂仍然会阻止全长单调子序列。 

对于较大的$n$，相同的推理尺度：任何跨越边界扩展递增子序列的尝试都会失败，因为后半值放置得更早，迫使单调增长重置。
