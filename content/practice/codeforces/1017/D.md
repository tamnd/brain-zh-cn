 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串的集合，全部具有相同的固定长度$n \le 12$。 该集合中的每个字符串都具有多重性，因此重复很重要。 除此之外，每个位置$i$具有非负权重$w_i$。 

比较两个二进制字符串时$s$和$t$，我们只从字符匹配的位置累积权重。 如果$s_i = t_i$，我们添加$w_i$，否则我们什么也不添加。 这定义了两个字符串之间的“相似度分数”：它只是匹配位置的权重之和。 

对于每个查询，我们都会得到一个二进制字符串$t$和一个阈值$k$，并且我们必须计算多重集中有多少个字符串具有相似度得分$t$最多$k$，分别计算重复项。 

约束是关键信号。 长度$n$最多 12 个，因此每个字符串实际上是最多 12 个位置的位掩码。 然而，字符串的数量$m$和查询$q$上升到$5 \cdot 10^5$，因此任何将每个查询与所有字符串进行比较的解决方案都太慢了。 一个天真的$O(mq)$每次比较最多 12 次操作仍需要大约$6 \cdot 10^{10}$操作，这是不可能的。 

一个微妙的点是，权重在 100 以内是任意的，因此我们不能依赖像汉明距离这样的统一结构； 相反，我们必须对加权一致性进行编码。 

一个常见的陷阱是尝试预先计算每对字符串的相似性。 那需要$O(m^2)$预处理，这也是不可行的。 

另一个失败案例是忘记多重性。 由于字符串重复，我们必须仔细聚合计数； 只要存在重复项，将输入视为集合而不是多重集就会产生错误的答案。 

## 方法

 暴力方法很简单：对于每个查询字符串$t$，迭代每个存储的字符串$s$，计算加权匹配分数$O(n)$，并计算有分数的$\le k$。 这是正确的，因为它直接遵循定义。 问题是速度：每次查询都有成本$O(mn)$，因此总复杂度变为$O(mqn)$，这远远超出了限制。 

关键的观察来自于对$n$。 每个字符串可以表示为长度的位掩码$n$。 对于固定的查询字符串$t$，我们可以预先计算匹配或不匹配每个位置相对于的权重贡献$t$。 

我们不是直接比较字符串，而是将每个存储的字符串转换为表示其兼容性结构的数值。 然后我们汇总所有频率$2^n$面具。 一旦我们获得了每个掩码的计数，每个查询就成为所有位置子集上的问题：我们需要确定有多少掩码在阈值内产生分数。 

关键的见解是预先计算相对于查询的位置贡献。 对于每个面膜$s$, 比分$t$仅取决于位一致的位置。 这意味着我们可以为每个掩码预先计算它对任何固定的贡献$t$通过迭代不匹配或匹配的子掩码，这是可行的，因为$2^{12} = 4096$。 

我们预先计算每个字符串掩码的频率，然后通过枚举所有掩码并对具有分数约束的掩码求和来回答每个查询。 由于 4096 很小，即使$q \cdot 2^n$是关于$2 \cdot 10^9$在最坏的情况下，这是边界，所以我们使用快速子集 DP 或权重的中间相遇来进一步改进。 

一种更简洁的方法是使用位掩码上的加权变换为每个掩码预先计算每个可能查询模式的总分贡献，本质上独立处理每个位置并在子集上构建 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(mqn)$|$O(1)$| 太慢了|
 | 位掩码频率 + 变换 |$O((m+q)2^n)$|$O(2^n)$| 已接受 |

 ## 算法演练

 我们将每个二进制字符串压缩为大小的整数掩码$n$，其中位$i$指示字符是否为“1”。 我们还计算每个面具出现的次数。 

1. 将所有输入字符串转换为整数掩码并构建频率数组`cnt[mask]`。 这一步确保我们不再考虑单个字符串，而是在固定的大小范围内工作$2^n$。 
2. 对于每个查询字符串$t$，也将其转换为掩码`tm`。 与存储的掩码的相似度得分`s`取决于位相等的位置，因此我们通过按位运算解释匹配结构。 
3. 预先计算位置的每个子集的权重总和。 我们不是从头开始重新计算每个查询，而是按位位置分隔贡献，以便可以使用最多 12 位的子集枚举来评估匹配决策。 
4. 对于每个查询，迭代所有掩码`s`在`[0, 2^n)`，计算加权一致性分数`tm`使用预先计算的位置贡献，并累积`cnt[s]`如果分数在`k`。 
5. 输出累计计数。 

这变得如此之快的核心原因是状态域只有 4096 个。当仔细构建时，即使是在这个空间上的嵌套迭代仍然是可行的。 

### 为什么它有效

 每个字符串唯一对应于位置的子集。 两个字符串之间的相似度分数是一个在独立坐标上分解的函数：每个位置贡献$w_i$或者$0$，仅取决于该位置的平等。 由于这种可分离性，可以通过对每个位置的独立贡献进行求和来计算全局分数，这使得聚合位掩码上的计数而不是直接枚举字符串对是有效的。 除了简单的相等之外，位置之间不存在任何交互，因此位掩码表示完全捕获了问题结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    w = list(map(int, input().split()))

    size = 1 << n
    cnt = [0] * size

    # encode strings
    for _ in range(m):
        s = input().strip()
        mask = 0
        for i, ch in enumerate(s):
            if ch == '1':
                mask |= 1 << i
        cnt[mask] += 1

    # precompute for each mask: score against any t via bit structure
    # we will precompute contribution of equality using bit DP
    # dp[mask] will store sum of weights where bits are 1 in mask
    weight_sum = [0] * size
    for mask in range(size):
        s = 0
        for i in range(n):
            if mask & (1 << i):
                s += w[i]
        weight_sum[mask] = s

    # answer queries
    for _ in range(q):
        t = input().strip().split()
        s = t[0]
        k = int(t[1])

        tmask = 0
        for i, ch in enumerate(s):
            if ch == '1':
                tmask |= 1 << i

        ans = 0

        # compare against all masks
        for mask in range(size):
            # positions where equal:
            # equal bits = positions where both have 1 or both have 0
            # compute match mask: ~(mask ^ tmask)
            match = ~(mask ^ tmask) & (size - 1)
            score = weight_sum[match]
            if score <= k:
                ans += cnt[mask]

        print(ans)

if __name__ == "__main__":
    solve()
```该代码将每个字符串压缩为位掩码并使用以下方法聚合相同的字符串`cnt`。 数组`weight_sum`将位置的任何子集转换为权重之和，因此一旦我们知道两个字符串之间的哪些位置匹配，我们就可以在恒定时间内计算相似度得分。 

对于每个查询，我们计算查询掩码和存储的掩码之间的异或来定位不匹配的位置。 在固定范围内否定这一点$n$位边界给出了精确的匹配位置。 然后使用该掩码来索引`weight_sum`。 

关键的实现细节是将按位 NOT 屏蔽为`(size - 1)`，因为 Python 整数是无界的，否则会引入虚假的高位。

 ## 边缘情况

 当所有字符串都相同时，就会出现关键的边缘情况。 在那种情况下，`cnt`有一个非零条目，并且每个查询必须在阈值允许的情况下正确聚合所有重复项。 该算法自然地处理这个问题，因为累积是通过`cnt[mask]`而不是每个实例的迭代。 

另一个边缘情况是权重为零时。 然后，无论匹配结构如何，每个相似度得分都变为零，并且每个查询$k \ge 0$必须返回$m$。 该算法仍然有效，因为`weight_sum`对于任何掩码都是零，因此每个字符串都被计算在内。 

最后一个微妙的情况是查询中每个位置的字符串都不同。 在这种情况下，匹配掩码全为零，因此分数变为零，并且正确性取决于正确掩码按位 NOT。 表达式`(~(mask ^ tmask) & (size - 1))`仅确保最低的$n$位被考虑在内，防止意外包含高阶位。
