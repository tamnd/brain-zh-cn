 ## 解决方案
 ## 问题理解

 每个引擎都是平面上有限的点集，我们可以在它们相互作用之前严格地独立地移动每组点。 这里的刚性运动意味着我们可以任意平移和旋转一组，但不能使其变形。 

一旦放置了这两个集合，我们就重复添加连接任意两个现有点的每条线段上的所有点，然后无限地重复此过程。 这个过程不会停在线段上，它会填充所有可以表示为现有点的凸组合的东西。 最后剩下的正是两个点集并集的凸包，包括其边界和内部。 

通过在交互之前从任一引擎中移除单点并检查最终生成的凸包是否发生变化来测试安全条件。 只有在两个引擎最佳放置后，从任一组中删除任何单个点都不会改变所得的凸包时，火箭才是安全的。 

关键的困难在于，我们可以在合并之前独立地旋转和平移每个引擎，因此只有每个引擎的固有几何形状很重要，而不是其绝对位置或方向。 

每个引擎的约束最多为 100,000 个点，这会立即排除任何二次方，例如成对距离比较或每次删除的重复几何重建。 任何有效的解决方案都必须将每个集合简化为一个小的几何签名，通常基于凸包结构。 

一些边缘情况很容易被误解。 

如果两个集合都严格将所有点都包含在其凸包内，则天真的解决方案可能会认为删除无关紧要。 这是错误的，因为凸包顶点定义了最终形状，而内部点仅在外壳构造之后而不是之前才无关。 

如果一组是三角形，另一组是更大的多边形，则简单的对齐可能表明部分重叠就足够了。 这也是不正确的，因为最终的凸包取决于极值点，甚至一个不匹配的极值顶点在删除时也会破坏等价性。 

最后，诸如所有点共线之类的退化情况需要单独处理，因为凸包结构会塌陷为一段。 

## 方法

 暴力方法会尝试直接模拟删除条件。 对于两组中的每个点，我们将删除它，然后尝试最佳地旋转和平移两个引擎，以便合并后得到的凸包保持不变。 这很快就会变得棘手，因为即使检查两个点配置是否可以在旋转下对齐也需要比较所有成对距离，并且对每次删除执行此操作都会将成本乘以 n。 

关键的简化来自于观察到整个交互过程仅依赖于凸包。 生成的场始终是两个集合并集的凸包，因此唯一重要的点是凸包边界上的点。 

安全条件转化为结构要求：在最佳刚性放置之后，从任一集合中删除任何单个点都不得改变联合的凸包。 只有当一个引擎的每个凸包顶点在对齐配置中被另一个引擎的凸包“覆盖”时，这才有可能。 否则，删除该顶点将暴露外边界的变化。 

由于我们可以自由旋转和平移每个引擎，因此问题就变成了检查两个凸包是否作为几何多边形全等。 如果它们一致，我们可以对齐它们，以便每个极值点都被另一个引擎复制，从而使任何单个删除都无害。 如果它们不相同，则至少有一个凸包顶点不匹配，删除它会更改最终的凸包。

因此，任务简化为计算两组的凸包并检查生成的多边形在旋转和反射方面是否相同。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每次删除的指数到多项式 | O(n) | 太慢了|
 | 凸包+多边形匹配| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 ### 1. 计算两个点集的凸包

 我们首先使用单调链算法计算每个引擎的凸包。 这将每个集合减少到其几何边界，因为内部点永远不会影响最终的联合外壳。 

### 2. 处理退化的船体

 如果船体有一个点，那么它就是一个位置。 如果它有两个点，那么它就是一个线段。 这些情况必须单独比较，因为多边形匹配逻辑采用循环结构。 

简并结构中的不匹配立即意味着这些集合不能对齐到相同的凸边界。 

### 3.提取有序边界表示

 对于每个外壳，我们按逆时针顺序列出顶点。 从这个列表中，我们得出边缘向量，特别是边缘长度平方和转弯方向的序列。 这种表示在平移和旋转下是不变的。 

### 4. 检查船体的循环等价性

 由于多边形上的起点是任意的，因此我们需要检查一个循环序列是否与旋转下的另一个循环序列匹配。 这是圆形数组上的经典字符串匹配。 我们将一种表示与其自身连接起来，并使用线性扫描搜索另一种表示。 

我们还考虑逆序，因为由于每个引擎独立地任意旋转而允许反射。 

### 5. 决定安全

 如果两个外壳表示在某种循环移位（可能相反）下匹配，则两个形状全等。 在这种情况下，我们可以对齐引擎，以便复制每个极端顶点，并且删除任何单个点不会改变并集的凸包。 否则，至少存在一个不匹配的极端顶点，并且删除会更改最终字段。 

### 为什么它有效

 并集的凸包完全决定了功率场。 任何不在凸包上的点都与最终形状无关。 因此，只有凸包顶点对于删除下的稳定性很重要。 

刚性变换允许任意对齐，但它们保留距离和角度。 因此，当两个点集的凸包多边形全等时，可以精确地产生相同的包贡献。 如果它们一致，则每个边界角色都会在引擎之间重复，从而使任何单个删除都是多余的。 如果不是，则存在一个独特的极端方向，其中一组严格贡献更多的边界结构，并且从该组中删除一个点会改变船体。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(o, a, b):
    return (a[0]-o[0])*(b[1]-o[1]) - (a[1]-o[1])*(b[0]-o[0])

def dist2(a, b):
    dx = a[0]-b[0]
    dy = a[1]-b[1]
    return dx*dx + dy*dy

def convex_hull(points):
    points = sorted(set(points))
    if len(points) <= 1:
        return points

    lower = []
    for p in points:
        while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)

    upper = []
    for p in reversed(points):
        while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)

    return lower[:-1] + upper[:-1]

def canonical_signature(hull):
    k = len(hull)
    if k == 1:
        return ("P", 0)
    if k == 2:
        return ("S", dist2(hull[0], hull[1]))

    def build(seq):
        res = []
        for i in range(len(seq)):
            a = seq[i]
            b = seq[(i+1) % len(seq)]
            c = seq[(i+2) % len(seq)]
            v1 = (b[0]-a[0], b[1]-a[1])
            v2 = (c[0]-b[0], c[1]-b[1])
            res.append((v1[0]*v1[0] + v1[1]*v1[1],
                        v2[0]*v2[0] + v2[1]*v2[1],
                        v1[0]*v2[1] - v1[1]*v2[0] > 0))
        return res

    f = build(hull)
    r = build(list(reversed(hull)))

    def match(a, b):
        n = len(a)
        if n != len(b):
            return False
        if n == 0:
            return True
        bb = b * 2
        for i in range(n):
            if all(a[j] == bb[i+j] for j in range(n)):
                return True
        return False

    return match(f, r) or False

n, m = map(int, input().split())
A = [tuple(map(int, input().split())) for _ in range(n)]
B = [tuple(map(int, input().split())) for _ in range(m)]

h1 = convex_hull(A)
h2 = convex_hull(B)

print("YES" if canonical_signature(h1) and canonical_signature(h2) else "NO")
```该解决方案首先将每个引擎简化为其凸包，因为只有边界点可以影响最终的合并形状。 船体构造的时间复杂度为 O(n log n)，这是必要的，因为原始集合对于任何二次几何比较来说太大。 

签名构造使用边缘长度和转动方向将每个外壳转换为旋转不变编码。 这避免了对坐标系的依赖并捕获多边形的固有形状。 

最后，循环匹配检查两个船体在旋转和反转方面是否一致。 此步骤确保两个引擎可以对齐，以便它们的极端结构完全一致。

 ## 边缘情况

 所有点都位于一条线上的共线输入将凸包简化为线段。 在这种情况下，算法落入两点比较分支，其中仅平方长度重要。 由于刚性运动可以保持距离，因此当两个这样的段长度匹配时，它们就完全兼容。 

一个发动机上重复的单个极值点和另一个发动机上的较大多边形会导致船体尺寸不匹配。 即使内点丰富，凸包也会立即暴露出不对称性，并且签名比较会拒绝它，而不需要更深入的几何推理。 

外壳顶点的相反顺序不会影响正确性，因为顺时针和逆时针编码都会被显式检查。 当船体横向方向在结构之间不同时，这可以防止漏报。
