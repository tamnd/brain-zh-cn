 ## 解决方案
 ## 问题理解

 我们得到一个网格，其中每个单元格已经具有固定的颜色，或者尚未确定。 最终目标是为所有未决定的单元格分配颜色，以便生成的全彩色网格包含尽可能多的有效 2 × 2 “检查器”块。 

如果 2 × 2 块的四个单元形成完美的交替图案，则该块有助于得分，这意味着对角共享相同的颜色，相邻单元不同，就像棋盘一样。 恰好有两个有效方向，它们只是彼此的全局颜色交换。 

关键的限制是一些单元格已经固定并且无法更改，因此我们不能自由选择全局棋盘着色。 相反，我们尝试将部分约束扩展到完整着色，从而最大化与棋盘格图案匹配的 2 × 2 子网格数量。 

每个测试用例的网格大小最多为 100 × 100，但所有测试用例的单元格总和最多为 10^6。 这立即排除了在所有分配上使用重状态或每平方模拟独立重新计算每 2 × 2 平方最佳选择的任何情况，因为任何指数或每平方组合推理都会在多个测试用例下崩溃。 

当固定单元与全局棋盘发生冲突时，就会出现微妙的边缘情况。 例如，如果两个相邻的固定单元已经匹配相同的颜色，则局部破坏包含它们的任何 2 × 2 的两个棋盘方向之一。 分配单个全局模式并在每个网格翻转一次的简单方法可能会严重失败：

 输入：```
2 2
W W
W ?
```如果我们选择一个从 (1,1) 处的 W 开始的棋盘，我们会强制 (1,2)=B，这与固定的 W 相矛盾。如果我们选择相反的模式，我们同样会打破其他地方的约束。 正确的解决方案必须局部适应，而不是全局提交。 

另一个失败案例是贪婪地独立分配每个“？”。 由于每个单元格参与最多四个不同的 2 × 2 块，因此局部贪婪选择可以立即减少多个未来检查器的贡献，如果我们一次只考虑一个方格，这是不可见的。 

## 方法

 强力解释将尝试“？”单元格的所有可能分配。 每个赋值都定义了一个完整的着色，我们计算有多少个 2 × 2 块满足检查器属性。 对于 k 个未着色的单元，这是 2^k 种可能性，并且在最差布局中每次测试 k 可以是 10^4。 即使是单个测试用例也会使这完全不可行。 

关键的结构观察是，每个 2 × 2 方格条件仅取决于奇偶校验：如果我们将颜色视为 0 和 1，则有效的方格方块会强制对角相等且相邻角不同。 这相当于说 (i, j) 处的值是由 (i + j) 奇偶校验加上全局翻转确定的，只是由于固定约束，我们可以违反某些平方。 

我们不是独立地决定每个单元格，而是转换视角：只有两种基于全局奇偶校验的着色，并且任何有效的局部棋盘格方块都必须与其中之一一致。 因此，对于两个可能的棋盘模板中的每一个，我们可以测量有多少单元已经与固定约束发生冲突。 一旦选择了模板，我们就会相应地填充所有剩余的单元格，从而最大限度地提高与该模板的一致性。 

现在关键的简化是：由完整的棋盘着色引起的 2 × 2 方格的数量是确定性的和最大的。 与完美棋盘的任何偏差都会减少局部有效 2 × 2 块的数量，因此最佳策略是选择与固定单元最匹配的两个全局棋盘着色中较好的一个，然后贪婪地完成其余部分以保留该结构。 

因此，我们只评估两个候选完全填充，一个从 (0,0) 处的 W 开始，另一个从 (0,0) 处的 B 开始，尊重固定单元格。 对于每个，我们计算有多少固定细胞一致。 我们选择更好的一个并构建最终的网格。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有任务 | O(2^k·n·m) | O(2^k·n·m) | O(n·m) | 太慢了 |
 | 两种模式评估| O(n·m) | O(n·m) | 已接受 |

 ## 算法演练

 我们将 W 和 B 视为两个二元状态并评估两个候选棋盘模式。 

1. 假设一个模式，其中单元格 (0,0) 为 W。如果 (i + j) 为偶数，则每个单元格 (i, j) 都强制为 W，否则为 B。 这定义了一个完整的网格，没有歧义。 这是一个一致的棋盘。 
2. 计算有多少固定单元已经匹配此模式。 对于每个不是“？”的单元格，我们检查其给定的颜色是否与计算的颜色一致。 我们累积代表兼容性的分数。 
3. 对第二个图案重复相同的过程，其中 (0,0) 是 B。这会翻转所有颜色，但保持相同的奇偶校验结构。 
4. 选择兼容性分数较高的模式。 这确保了我们最大化与固定约束的一致，这间接最大化了可实现的检查器 2 × 2 块，因为任何分歧都会迫使至少一个局部违规。 
5. 根据所选模式填充每个单元格，不留下“？”单元格，构建最终网格。 

它之所以有效，是因为任何有效的 2 × 2 棋盘格图案都必须与网格上一致的奇偶校验分配对齐。 每个 2 × 2 块都强制执行全局传播的奇偶校验约束。 因此，最佳解决方案必须对应于两个奇偶校验分配之一，唯一的自由是选择使用哪个全局相位。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        g = [list(input().strip()) for _ in range(n)]

        def score(start):
            s = 0
            for i in range(n):
                for j in range(m):
                    if g[i][j] == '?':
                        continue
                    expected = start if (i + j) % 2 == 0 else ('B' if start == 'W' else 'W')
                    if g[i][j] == expected:
                        s += 1
            return s

        s1 = score('W')
        s2 = score('B')

        start = 'W' if s1 >= s2 else 'B'

        for i in range(n):
            row = []
            for j in range(m):
                if (i + j) % 2 == 0:
                    row.append(start)
                else:
                    row.append('B' if start == 'W' else 'W')
            print("".join(row))

solve()
```该解决方案通过每个测试用例扫描网格两次来评估两个全局棋盘阶段，这在总 10^6 单元约束下是安全的。 

然后，构建阶段根据奇偶校验简单地分配每个单元。 重要的实现细节是避免重新计算或存储两个候选者的中间网格，因为这会使内存加倍而没有任何好处。 相反，我们动态计算分数，然后重建一次。 

一个常见的错误是尝试通过强制局部修复来“尊重”固定单元。 这会破坏全局一致性，并会大幅减少有效 2 × 2 方格的数量。

 ## 边缘情况

 具有冲突局部模式的完全约束网格测试评分步骤是否正确地选择了更好的全局阶段。 例如：```
2 2
W B
B W
```两种模式得分相同，但任一输出都是有效的，因为两者都保留所有固定约束。 该算法确定性地选择一个，并且重建仍然会产生有效的棋盘。 

单行或单列网格根本没有 2 × 2 子网格。 该算法仍然会产生一致的颜色，并且答案始终是零模式，但正确性取决于维持固定的单元格。 奇偶校验结构自然满足了这一点，因为它永远不会违反约束，除非受到评分选择的强制。 

所有单元格都固定的网格测试评分是否正确处理完全一致的情况。 如果输入已经与棋盘格匹配，则所选模式将与所有单元格匹配，并且重建会保留最大可能数量的有效 2 × 2 块。
