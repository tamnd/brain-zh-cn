 ## 解决方案
 ## 问题理解

 我们有一组地下出口，每个出口都将洞穴与外界连接起来，并有自己的冰箱旅行费用。 大量蚂蚁必须每只执行一次袭击：它们通过某个出口离开洞穴，走到冰箱前，立即偷窃，然后可能通过另一个出口返回。 

蚂蚁每次通过出口时，都会在通道上停留 1 秒，在这一秒内它会在洞穴外。 退出后，它会根据所选出口行驶固定时间，然后返回并再次在出口处停留一段时间。 当一只蚂蚁从一个洞离开或进入时，该洞不能在同一秒内同时被另一只蚂蚁使用，也不能在同一时刻混合离开和进入操作。 

目标不是最小化每只蚂蚁的时间，而是最小化至少一只蚂蚁在洞穴外的总持续时间。 换句话说，我们正在最小化所有蚂蚁的所有“外部间隔”的并集长度。 

关键的难点来自于三个结构的相互作用。 首先，一旦选定的两个洞被固定，每只蚂蚁都会贡献固定的外部持续时间。 其次，所有蚂蚁都共享出口的容量限制，这会序列化每个孔的离开和进入操作。 第三，目标仅取决于外部活动的全球重叠，而不取决于个人的完成时间。 

输入大小使得暴力配对变得不可能。 蚂蚁的数量可以大到10^14，所以我们不能单独对待蚂蚁。 我们只控制每个洞有多少只蚂蚁以及它们如何配对。 孔的数量最多为 10^5，因此任何解决方案都必须减少聚合孔统计信息的问题，例如最小负载或总负载。 

一个幼稚的策略会尝试为每只蚂蚁分配一对孔并模拟调度。 这会立即失败，因为即使存储所有蚂蚁也是不可能的，更重要的是，调度和配对之间的交互创建了一个耦合系统。 

当所有孔具有相同的距离时，会出现微妙的边缘情况。 在这种情况下，分配孔的任何不平衡都会导致出口子集的拥塞，从而增加活跃期，即使所有蚂蚁都有相同的旅行成本。 另一种边缘情况是当一个洞比所有其他洞好得多时； 那么集中流量可能会降低旅行成本，但会增加序列化成本，并且如果没有全局推理，这种权衡就变得不明显。 

## 方法

 一种直接的方法是为每只蚂蚁分配一对洞，然后逐秒模拟事件：每次离开和进入操作都会占用一个洞一秒钟，然后我们跟踪蚂蚁何时在外面。 这在原则上是正确的，因为它尊重所有约束，但在计算上是不可能的。 蚂蚁数量最多为 10^14 只，每只蚂蚁至少贡献两次洞操作，因此即使对每只蚂蚁进行线性处理也是不可行的。 

中心观察是问题分为两个独立的部分。 第一个组成部分是我们处理所有跨洞离开和进入操作的速度。 每个孔每秒最多可以处理一次操作，因此总处理时间由负载最多的孔决定。 第二个组成部分是蚂蚁由于行进距离而在室外停留的时间，一旦离开时间固定，这与日程安排无关。 

这将问题转化为平衡孔之间的负载，同时还选择最小化最大行程贡献的孔对。 调度部分变成了分布在n台机器上的2m个单元操作的负载平衡问题。 行进部分仅取决于每只蚂蚁所选择的孔对，并且我们只关心最坏情况的完成扩展，因为它决定了所有蚂蚁返回内部的最终时间。

事实证明，最佳结构将所有蚂蚁集中在距离最小的孔上以进行离开和进入，因为这样可以最大限度地减少最大行程尾部，同时在所有孔中尽可能均匀地分配离开和进入事件，以最大程度地减少拥堵。

 | ---| ---| ---| ---|
 | 每只蚂蚁的显式模拟 | O(米) | O(米) | 太慢了 |
 | 负载均衡+最优配对| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们定义两个独立的量。 一是每只蚂蚁的最小可能最大行程贡献，这是通过使用离开和进入距离最小的洞来实现的。 另一个是在容量限制下处理所有进出操作所需的最短时间。 

1. 确定所有孔之间的最小距离。 该值决定了任何蚂蚁的最佳可能旅行成本，因为任何其他选择只会增加去程和回程旅行。 
2. 观察每只蚂蚁恰好生成两个洞操作，一只离开，一只进入。 在 m 只蚂蚁中，这将创建 2m 个单元操作，这些操作必须在 n 个孔中执行。 
3. 每个孔每秒最多执行一次操作，因此如果为一个孔分配 k 个操作，则需要 k 秒的处理时间。 这意味着所有操作所需的总时间由负载最多的孔决定。 
4. 为了最大限度地减少最大负载，请尽可能均匀地在所有孔中分配操作。 由于有 2m 操作和 n 个孔，因此最佳可实现的最大负载为 ceil(2m / n)。 
5. 一旦安排了操作，最后一只开始其旅程的蚂蚁将不迟于时间 ceil(2m / n) 减一，因为操作是连续打包的。 
6. 然后，每只蚂蚁添加一个行程尾部，其中包括 1 秒离开、a_i 向外行程、a_j 返回行程和 1 秒进入。 通过最佳配对，a_i 和 a_j 都是最小距离值。 
7. 最终答案是通过将最大计划开始时间和固定行程尾部相加得到的，总持续时间等于 ceil(2m / n) + 2 * min(a) + 1。 

### 为什么它有效

 调度过程完全由每个孔的容量决定，因此任何有效的策略都会产生一个负载向量，其最大分量下限是所有操作完成之前的时间。 均匀分布可以实现这一界限。 独立地，两个选定洞中的行进时间都是单调的，因此最小化每只蚂蚁的两个端点可以最小化最差的完成时间。 由于目标是所有外部间隔的并集，最后一只完成蚂蚁主导了总持续时间，并且最小化其完成时间足以实现最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))
        
        mn = min(a)
        
        # ceil(2m / n)
        base = (2 * m + n - 1) // n
        
        # final formula
        ans = base + 2 * mn + 1
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先提取最小距离，因为这决定了最佳可能的行驶配置。 然后，它计算在每孔单位容量下安排所有离开和进入操作所需的秒数。 最后的表达式添加了由首次出口、去程、回程和最终进入所带来的不可避免的旅行开销。 

唯一棘手的部分是处理 n 上 2m 的天花板划分，因为离开和进入操作都必须计算在内。 一旦识别出调度和旅行的分解，其余的计算就直接进行了。

 输入：```
1
3 4
2 5 7
```这里，最小距离为 2。操作总数为 8，并且有 3 个孔时，调度负载为 ceil(8/3) = 3。 

我们跟踪导出的数量：

 | 数量 | 价值|
 | ---| ---|
 | 分钟（一）| 2 |
 | 2米| 8 |
 | 天花板(2m/n) | 3 |
 | 最终答案| 3 + 2*2 + 1 = 8 |

 这表明，尽管蚂蚁有不同的可用洞，但拥塞仍占主导地位，并且只有极少的出行贡献。 

现在考虑一个高度倾斜的情况：

 输入：```
1
1 5
10
```只有一个洞存在，因此所有操作都是串行的。 负载总共为 10 次操作（5 次离开 + 5 次进入），给出 ceil(10/1) = 10。最小距离为 10。 

| 数量 | 价值|
 | ---| ---|

 这个案例表明，当没有并行性时，答案就变成了纯粹的串行加旅行。

 ## 边缘情况

 当只有一个洞时，所有2m操作都必须经过一个瓶颈。 该算法简化为计算 2m + 2·a_min + 1，这符合不可能并行性并且每个蚂蚁都通过相同资源严格串行的事实。 

当所有孔具有相同的距离时，最小选择不依赖于结构，但拥塞仍然主导结果。 该算法仍然将操作均匀地分布在孔之间，确保没有单个孔成为超出 ceil(2m/n) 的瓶颈，这证实了对称性不会破坏正确性。 

当 m 相对于 n 极大时，调度项占主导地位，行程差异在比例上变得可以忽略不计。 该公式通过将线性载荷项与恒定行程项分开来正确地捕获这一点。
