 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，都有一个整数数组，我们需要计算有多少对位置$(i, j)$和$i < j$满足涉及所选元素的乘法和加法的特定不等式。 

对于一对值$x = a_i$和$y = a_j$，条件是：$$x \cdot y < x + y$$我们实际上被要求，对于每个数组，计算有多少无序元素对使这个不等式成立。 

重写条件是第一个有用的步骤。 将所有内容移至一侧：$$xy - x - y < 0$$添加 1：$$(x - 1)(y - 1) < 1$$这种形式比原始表达方式更加结构化，并且是后续所有内容的关键。 

输入大小总的来说很大，总和$n$跨测试用例最多$10^6$。 这排除了任何$O(n^2)$每个测试用例立即接近，因为即使是单个密集测试用例的大小$10^5$已经暗示$10^{10}$配对检查。 

每个测试用例都需要一个线性算术或线性解决方案。 由于总输入量很大，即使$O(n \log n)$仅当仔细实施并且每个元素不过度重复时才是可接受的。 

有一些边缘情况打破了天真的推理：

 如果所有数字都是正数且很大，则说$a_i \ge 2$， 然后$x+y \le xy$总是成立，所以答案应该是零。 天真的编码员可能仍然尝试通过一些启发式和超算来对对进行计数。 

如果数组包含许多负值，则不等式可能会以不直观的方式翻转行为。 例如，与$x = -1$和$y = 2$，我们得到$-2 < 1$，这是正确的，即使一个值为负值，一个值为正值。 

如果两个值都为零，$0 \cdot 0 = 0$和$0 + 0 = 0$，因此不等式失败，因为它是严格的。 

这些角点相互作用正是在尝试任何计数策略之前必须改变不等式的原因。 

## 方法

 蛮力的想法很简单。 我们检查每一对$(i, j)$, 计算$a_i a_j$和$a_i + a_j$，如果不等式成立，则增加答案。 这是正确的，因为它直接评估所述条件，而不会丢失任何对。 

然而，这需要$\frac{n(n-1)}{2}$每个测试用例的操作。 和$n = 10^5$，这是关于$5 \cdot 10^9$比较，这远远超出了可行的范围。 

关键的观察结果来自于重写不等式：$$(x - 1)(y - 1) < 1$$现在结构只取决于距离有多近$x$和$y$是 1，而不是它们的绝对大小。 这表明对实线上的相对位置进行排序和推理。 

我们根据 的符号来分割数字$x - 1$。 价值观$x \le 1$制作$x - 1 \le 0$，而值$x \ge 2$制作$x - 1 \ge 1$。 这将数组划分为非正变换值和正变换值。 

现在考虑以下情况：

 如果两者都$x - 1$和$y - 1$是正数，那么它们的乘积至少为 1，因此不等式成立。 因此，两个值都大于 1 的配对不会做出贡献。 

如果一个为正，一个为非正，则乘积始终为非正，因此始终小于 1，因此所有此类交叉对都是有效的。 

如果两者都是非正数，则意味着两个原始值都是$\le 1$， 然后$(x - 1)(y - 1) \ge 0$。 仅当乘积恰好为 0 时，此类对才有效，这意味着其中至少有一个等于 1。 

所以整个问题归结为计数：

 至少包含一个元素的对$\le 1$与另一个元素$> 1$，加上涉及值 1 的对，仔细处理重复项。 

这种结构导致按频率计数而不是对计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 频率+案例分割|$O(n)$每个测试用例|$O(n)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 将数组分为三个概念组：等于 1 的值、小于 1 的值和大于 1 的值。这种分类直接对应于$x - 1$，它控制了不平等。 
2. 计算有多少个元素等于 1。这些元素很特殊，因为它们使得$(x - 1) = 0$，当与另一个 1 配对时，这会强制产品条件恰好变为 0。 
3. 计算有多少个元素小于 1。这些元素将与大于 1 的元素自由交互，因为它们的转换值分别为非正值和正值，保证乘积 < 1。 
4. 计算有多少个元素大于 1。这些元素之间不能配对，因为两个变换后的值都至少为 1，产生的乘积至少为 1。 
5. 计算贡献：

 - 任何涉及一个元素的对$\le 1$和一个元素$> 1$是有效的。 
- 任何至少一个元素为 1 并且另一个元素也为 1 的对$\le 1$仅在有限情况下有效，但可简化为涉及 1 和非正值的组合。 
6. 最终答案是通过使用组合计数而不是迭代对有效的跨组贡献求和来获得的。 

为什么它有效：

 将条件重写为$(x - 1)(y - 1) < 1$，问题就完全变成了两个移位值是否产生低于 1 的乘积。由于所有整数都通过它们是否低于、等于或高于 1 来分隔，因此每个类别与其他类别都有固定的交互规则。 这些交互规则是确定性的并且独立于排序，这意味着通过频率对对进行计数可以保持精确的正确性，而不会丢失或重复计算任何有效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        
        cnt1 = 0
        cnt_le1 = 0
        cnt_gt1 = 0
        
        for x in arr:
            if x == 1:
                cnt1 += 1
                cnt_le1 += 1
            elif x < 1:
                cnt_le1 += 1
            else:
                cnt_gt1 += 1
        
        ans = 0
        
        ans += cnt1 * (cnt1 - 1) // 2
        
        ans += cnt1 * (cnt_le1 - cnt1)
        
        ans += cnt_le1 * cnt_gt1
        
        out.append(str(ans))
    
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码遵循之前导出的精确分组逻辑。 关键的实现细节是维护元素的计数和元素的总数$\le 1$，因为涉及 1 的对取决于两个量。 最终表达式完全由组合配对构建，避免任何显式的配对迭代。 

必须注意每个元素都只被分类一次，因为之间的错误分类$=1$,$<1$， 和$>1$直接破坏了派生恒等式。 

## 工作示例

 考虑数组$[2, 3, 0, 1]$。 

我们计算计数：$cnt1 = 1$,$cnt\_le1 = 2$（0 和 1），$cnt\_gt1 = 2$（2和3）。 

| 步骤| cnt1 | cnt_le1 | cnt_gt1 | 部分答案 |
 | ---| ---| ---| ---| ---|
 | 初始化| 0 | 0 | 0 | 0 |
 | 扫描后| 1 | 2 | 2 | 0 |
 | 1 对 | 1 | 2 | 2 | 0 |
 | 1 与 le1 | 1 | 2 | 2 | 1 |
 | le1 与 gt1 | 1 | 2 | 2 | 5 |

 最终答案是5。 

现在考虑$[2, 2, 3]$。 

这里$cnt1 = 0$,$cnt\_le1 = 0$,$cnt\_gt1 = 3$。 每对都是无效的，因为两个值都大于 1。 

答案是0，与事实相符$xy \ge x + y$对于该范围内的所有对都成立。 

这些例子证实，只有跨区域的互动才能发挥作用。

 ## 边缘情况

 一种微妙的情况是当所有元素都等于 1 时。不等式变为$1 \cdot 1 < 2$，这总是正确的，所以每一对都应该被计算在内。 该算法计算$cnt1 = n$,$cnt\_le1 = n$,$cnt\_gt1 = 0$，给予：$$\frac{n(n-1)}{2}$$与完整的配对组完全匹配。 

另一种情况是所有元素都等于 2。那么每个乘积至少为 4，而和最多为 4，因此不存在严格的不等式。 该算法将所有元素分类为$cnt\_gt1$，产生零贡献。 

混合边缘情况，例如$[1, 0, 2]$展示了跨项逻辑。 唯一有效的对是$(1,0)$和$(1,2)$，取决于评估，计数公式通过$cnt1$基于和跨组的术语，无需重复计算。
