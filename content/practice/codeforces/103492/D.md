 ## 解决方案
 ## 问题理解

 我们给定一个正整数 x，并使用它周围的素数构造一个值。 首先，我们将f(x)定义为严格大于x的最小素数，因此它是x之后的下一个素数。 然后我们再次将相同的想法应用于 f(x)，给出 f(f(x))，它只是下一个素数之后的素数。 

从这两个素数中，我们通过取平均值形成一个数 g(x)，这意味着我们将它们相加并除以 2。任务是确定所得数 g(x) 本身是否是素数。 

输入由最多 100,000 个独立查询组成，每个 x 可以大到 10^18。 这立即排除了任何试图测试素数或为每个查询单独搜索 x 周围素数的方法。 任何解决方案都必须避免在每个测试用例中实际计算大素数。 

值非常小时会出现微妙的边缘情况。 对于 x = 1，下一个素数是 2 和 3，因此 g(x) 变为 2，即素数。 对于 x = 2，下一个素数是 3 和 5，因此 g(x) 变为 4，这不是素数。 这些微小的输入已经表明答案可能仅取决于前几个素数的结构而不是 x 的大小。 

主要困难在于 x 可能非常大，因此任何推理都必须依赖于连续素数的属性而不是它们的显式值。 

## 方法

 计算g(x)最直接的方法是通过向上搜索直到找到素数来计算f(x)，然后以同样的方式计算f(f(x))，最后检查它们的平均值是否是素数。 从定义上来说这是正确的，但它立即遇到了计算限制。 对于 x 高达 10^18，找到下一个素数需要大量素数测试并可能扫描大间隙，而对于 100,000 个查询执行此操作变得不可行。 

关键的观察是，我们实际上并不关心素数的数值，只关心它们的结构关系。 令 p1 = f(x) 且 p2 = f(f(x))。 这些是连续的素数。 值 g(x) 只是两个连续素数之间的中点。 

现在我们问一个更简单的结构问题：两个连续素数的中点本身可以​​是素数吗？ 如果这样的中点 m 是素数，那么我们将有 p1 < m < p2，这意味着两个连续素数之间严格存在一个素数。 这与连续性的定义相矛盾，除非间隔非常小。 唯一的例外发生在素数序列的最开始处，其中连续的素数 2 和 3 的中点为 2。 

因此，整个问题归结为检查这对连续素数是否为 (2, 3)，只有当 x 足够小以至于下一个素数为 2 时才会发生这种情况。

 ## 算法演练

 1. 对于每个 x，确定 x 之后的下一个素数是否是 2。只有当 x = 1 时才会发生这种情况，因为 2 是最小的素数，并且小于 2 的正整数没有不同的下一个素数。 
2. 如果x = 1，我们立即将连续的素数识别为2和3，因此中点为2，即素数。 
3. 对于任意x≥2，下一个素数f(x)至少为3，下一个素数f(f(x))至少为5，形成连续的奇数素数。 
4. 大于 (2, 3) 的任意两个连续奇素数的中点不能是素数，因为它严格位于两个连续素数之间，除非等于其中之一，否则这是不可能的。 
5. 仅当 x = 1 时输出 YES，否则输出 NO。 

### 为什么它有效

令 p1 = f(x) 且 p2 = f(f(x))。 这些是连续的素数。 假设 g(x) = (p1 + p2) / 2 是素数。 那么 g(x) 严格位于 p1 和 p2 之间。 这意味着两个连续素数之间存在一个素数，这与它们的定义相矛盾。 这种推理被打破的唯一情况是当间隔从 2 和 3 开始时，由于可能的间隙最小，中点在行为上与一个端点重合。 因此只有 x = 1 幸存。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        x = int(input())
        if x == 1:
            out.append("YES")
        else:
            out.append("NO")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现反映了整个主要推理崩溃为单一结构条件。 函数 f(x) 和第二个素数 f(f(x)) 永远不会被显式计算，因为它们唯一相关的属性是它们形成连续的素数。 这个决定简化为检查我们是否处于这些素数为 (2, 3) 的唯一情况，它恰好对应于 x = 1。 

超出此条件不需要边缘处理，因为所有 x ≥ 2 都会产生相同的结构结果。 

## 工作示例

 ### 示例 1：x = 1

 | 步骤| f(x) | f(x) | f(f(x)) | f(f(x)) | g(x) | g(x) |
 | --- | --- | --- | --- |
 | 1 | 2 | 3 | (2 + 3) / 2 = 2 |

 中点等于 2，即质数，因此输出为 YES。 这是连续素数为 (2, 3) 的唯一配置，允许中点为素数。 

### 示例 2：x = 2

 | 步骤| f(x) | f(x) | f(f(x)) | f(f(x)) | g(x) | g(x) |
 | --- | --- | --- | --- |
 | 1 | 3 | 5 | (3 + 5) / 2 = 4 |

 结果是4，是合数，所以输出是NO。 这反映了这样一个事实：一旦素数从 3 开始，连续素数之间的中点就不可能是素数。

 ## 边缘情况

 唯一有意义的边缘情况是当 x = 1 时，其中素数链以 2 和 3 开始。对于 x = 1，算法直接输出 YES，与计算的中点 (2 + 3) / 2 = 2 匹配。 

当 x ≥ 2 时，执行总是落入 NO 分支。 例如，当 x = 2、f(x) = 3 且 f(f(x)) = 5 时，给出中点 4，该值被正确拒绝。 

不会出现进一步的边界问题，因为决策不依赖于大素数的算术属性，仅依赖于最小可能素数配置的标识。
