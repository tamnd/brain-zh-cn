 ## 解决方案
 ## 问题理解

 我们有一棵树，其中每个节点都有不同的权重。 从任何起始节点，如果目标节点是两个节点之间唯一简单路径上的最大权重节点，则允许猴子跳转到另一个节点。 每次跳转后，该过程都会从新节点继续，我们想知道从每个可能的起始节点开始可以访问多少个不同的节点。 

关键的困难在于，是否允许跳跃取决于路径上的全局信息，而不仅仅是局部邻接性。 直接的解释是，从节点 u 可以跳转到任何节点 v，这样 u 到 v 的路径上没有节点的权重大于 a[v]，这意味着 a[v] 主导该路径。 

输入是一棵树，因此每对节点都有唯一的路径。 我们必须计算，对于每个节点 k，从 k 开始的重复有效跳转下可以到达多少个节点。 

约束很大：所有测试用例的节点总数最多为 8 × 10^5。 这立即排除了任何尝试使用路径查询独立模拟每个起始节点跳转的解决方案。 每个测试用例的任何二次方，甚至每个节点的 O(n log n) 都会太慢。 我们需要一个近线性或线性算数的全局构造。 

一个微妙的问题是可达性不是对称的，也不是明显的单调。 具有高权重的节点可以将许多节点“拉”向它，但如果中间最大值介入，路径条件仍然会阻止移动。 

一个天真的错误是假设从一个节点总是可以跳转到邻居或任何更高权重的节点。 例如，考虑权重为 3, 1, 2 的线 1-2-3。从节点 2 无法跳转到 3，因为沿着路径 2-1-3 的最大值为 3，但路径 2-1-3 在树意义上并不简单，除非考虑正确的结构，即使这样，条件也取决于完整路径最大值，而不是邻接或排序。 这表明局部贪婪推理失败了。 

## 方法

 蛮力解释很简单。 对于每个起始节点 k，我们尝试模拟所有可达的跳跃。 从当前节点 u 开始，我们考虑每个可能的目标 v 并检查 u 到 v 的路径上的最大值是否等于 a[v]。 如果是，则 v 可达。 我们继续从新到达的节点扩展，直到无法添加更多节点。 

检查每对 (u, v) 的路径最大值可以使用 O(log n) 中的 LCA 完成，因此在最坏情况下，单个可达性扩展已经是 O(n^2 log n)。 由于这种扩展对每个起始节点都会重复，因此在密集探索场景中，总数变为 O(n^3 log n)，这在 n 达到 10^5 时完全不可行。 

关键的结构见解是，条件“a[v] 是路径 u 到 v 上的最大值”相当于说，如果我们将每条边从较低权重定向到较高权重，那么运动就会受到路径的限制，其中路径上的最大值就像障碍一样。 我们不再考虑任意路径查询，而是将过程重新构建为探索一种结构，其中较高节点“控制”较低节点之间的连接。 

实现这一点的标准方法是按权重递增的顺序处理节点并维护一个动态森林，在其中我们逐渐添加节点并通过边将它们连接到已经活动的组件。 每次激活一个节点时，我们都会将其与已激活的邻居合并。 这构建了一个结构，其中连接性由增加的最大约束来控制。 

为了回答可达性问题，我们反过来看：对于一个固定的节点k，这个过程相当于计算有多少个节点可以与k关联，作为动态构建的森林中连接路径上的最高权重节点。 这减少了在单调激活过程下维护组件的情况，可以使用类似 DSU 的结构进行跟踪。

关键的观察结果是，当节点按权重递增的顺序激活时，每个节点都会附加到已经激活的邻居，并且生成的结构形成一棵合并树，其中每个组件都有明确定义的最大值。 节点的答案对应于在此过程中可以到达它的最高权重节点“拥有”的组件的大小。 

这导致了联合查找风格的解决方案，但需要仔细排序：我们模拟激活并维护组件大小。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每个测试用例 O(n^2 log n) | O(n) | 太慢了 |
 | 激活订单上的 DSU | O(n α(n)) | O(n α(n)) | O(n) | 已接受 |

 ## 算法演练

 我们按照节点权重的递增顺序处理节点。 在此过程中，我们逐渐“激活”节点并将它们连接到已经活跃的邻居。 

1. 将所有节点按权重升序排序。 这定义了节点可用于移动的顺序。 这种顺序的原因是任何有效的路径约束都是由最大值确定的，因此必须首先引入较低的权重，然后才能将较高的权重用作端点。 
2. 在节点上维护不相交的集合并集结构，最初每个节点均处于非活动状态并位于其自己的集合中。 
3. 按权重排序迭代节点。 当处理节点u时，将其标记为活动的。 此时，u 成为任何涉及 u 的未来路径的当前最大候选者。 
4. 对于 u 的每个邻居 v，如果 v 已经处于活动状态，则合并包含 u 和 v 的 DSU 集。这会连接现在相互可达的区域，而不会违反路径最大约束，因为所有中间节点都≤当前权重级别。 
5. 将u与所有活动邻居合并后，我们记录u所属的连通分量的大小。 该大小表示通过 u 作为某些有效跳转序列中的控制最大值可以到达的节点数。 
6. 将此组件大小存储为节点 u 的答案。 

### 为什么它有效

 在激活过程中的任何时刻，所有活跃节点的权重都小于或等于当前节点。 完全在活动节点内的任何路径的最大值等于这些节点之一，并且由于我们按升序处理，因此 DSU 组件表示不需要未处理的较高权重节点来满足路径约束的最大区域。 每个并集完全对应于可达性的有效扩展，而不会引入更高的中间最大值，否则会阻止跳转。 由于权重是不同的并且激活是单调的，因此每个组件的大小在处理节点时是固定的，并且后来的较高节点无法追溯更改较低节点的可达性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        adj = [[] for _ in range(n)]
        for _ in range(n - 1):
            u, v = map(int, input().split())
            u -= 1
            v -= 1
            adj[u].append(v)
            adj[v].append(u)

        a = list(map(int, input().split()))

        order = sorted(range(n), key=lambda x: a[x])

        parent = list(range(n))
        size = [1] * n
        active = [False] * n
        ans = [0] * n

        def find(x):
            while parent[x] != x:
                parent[x] = parent[parent[x]]
                x = parent[x]
            return x

        def union(x, y):
            x = find(x)
            y = find(y)
            if x == y:
                return
            if size[x] < size[y]:
                x, y = y, x
            parent[y] = x
            size[x] += size[y]

        for u in order:
            active[u] = True
            for v in adj[u]:
                if active[v]:
                    union(u, v)
            ans[u] = size[find(u)]

        sys.stdout.write("\n".join(map(str, ans)) + "\n")

if __name__ == "__main__":
    solve()
```代码直接遵循激活思路。 邻接表存储树结构。 数组`order`确保按权重递增的顺序处理节点。 DSU 仅维护活动节点之间的连接组件。 

这`active`数组是必不可少的，因为我们不能将节点连接到单调构造中尚不可用的邻居。 如果没有这个，我们将通过当前最大有界结构中不允许的节点错误地合并组件。 

每个节点的答案是在其变为活动状态时使用 DSU 组件大小获取的。 该时刻准确地捕获了可以到达的完整节点集，该节点充当所有允许路径上的有效最大值。 

## 工作示例

 考虑一棵小树：

 输入：```
1
4
1 2
2 3
2 4
4 1 3 2
```我们按权重递增的顺序处理节点：节点 0（权重 1）、节点 3（权重 2）、节点 2（权重 3）、节点 1（权重 4）。 

| 步骤| 激活节点| DSU 合并 | 元件尺寸| 答案已记录|
 | --- | --- | --- | --- | --- |
 | 1 | 0 | 无 | {0:1} | 答案[0]=1 |
 | 2 | 3 | 无 | {3:1} | 答案[3]=1 |
 | 3 | 2 | 无 | {2:1} | 答案[2]=1 |
 | 4 | 1 | 合并 (1-2), (1-3) | {1,2,3,0} 尺寸 4 | 答案[1]=4 |

 这表明只有最高权重的节点才能统一整个结构，而较小的节点在激活时保持孤立。 

现在考虑一条链：```
1
3
1 2
2 3
3 1 2
```顺序是 0、1、2。 

| 步骤| 激活节点| 合并| 元件尺寸| 回答 |
 | --- | --- | --- | --- | --- |
 | 1 | 0 | 无 | 1 | 1 |
 | 2 | 1 | (1-0) | 2 | 2 |
 | 3 | 2 | (2-1) | 3 | 3 |

 每次激活都会扩展一个生长组件，这与增加权重逐渐解锁树的想法相匹配。 

这些痕迹证实 DSU 组件单调演化，并且答案对应于激活时的最终可到达区域。

 ## 边缘情况

 关键的边缘情况是最高权重的节点位于树的中心。 例如：

 输入：```
1
5
1 2
2 3
3 4
4 5
3 1 5 2 4
```这里节点 2（值 5）最后激活并通过链连接所有先前激活的节点。 在激活过程中，较小的节点仅形成较小的组件，但当最大节点激活时，它会合并所有内容。 DSU 正确捕获了这一点，因为仅与已经活动的邻居执行联合，确保不会引入非法路径。 

另一个边缘情况是以低权重节点为中心的星形。 由于所有叶子在之后或之前以不同的顺序激活，因此叶子最初保持隔离。 只有当中心激活时，它才会与多个组件合并，从而导致组件大小突然跳跃。 该算法自然地处理这个问题，因为合并是对称的并且仅依赖于激活状态，而不依赖于结构假设。
