 ## 解决方案
 ## 问题理解

 我们有一个带有 n 个图块的圆形板，当我们踩到它时，每个图块贡献一个固定的整数值。 从第一个图块之前开始，我们从图块 1 开始，确定性地移动到图块 2，然后是图块 3，依此类推，在 n 之后回绕到图块 1。 每次我们落在一块图块上时，我们都会将其值添加到运行分数中，该分数可以上升或下降。 

每个查询都要求所需的最小步数，以便访问过的图块的累积总和恰好等于目标值 x。 “一步”意味着落在一块瓷砖上，因此在 k 步之后，我们在这个无限周期行走中访问了长度为 k 的前缀。 

一个关键的观察结果是，在 n 个步骤之后，我们恰好完成了数组的一个完整循环，因此每个完整循环贡献相同的总和 S = a1 + a2 + ... + an。 经过 k 步后，分数是完整循环的总和加上下一个循环的前缀。 

约束条件很大：每个测试用例的 n 和 m 最多为 100,000，总和最多为 500,000。 这排除了每个查询模拟的步骤或朴素的前缀扫描。 在最坏的情况下，任何每次查询都是线性的解决方案都会超过大约 10^10 次操作，这远远超出了限制。 我们需要对每个查询进行线性或近线性时间和对数或恒定时间的预处理。 

当总周期和 S 为零时，会出现微妙的边缘情况。 在这种情况下，重复循环不会改变分数，因此只有前缀和很重要。 如果 S 不为零，我们可以通过选择要进行多少个完整周期来调整分数，但余数必须与前缀和完全匹配。 另一个边缘情况是负目标，因为值可以减少，所以前缀和不是单调的，除非我们明确地跟踪它们。 

## 方法

 蛮力策略很简单。 对于每个查询 x，我们模拟沿着数组的无限重复逐步行走，保持运行总和，直到达到 x 或超出合理范围。 在最坏的情况下，答案可能需要许多完整周期，尤其是当 S 很小或为零时，因此这可能会退化为每个查询扫描最多 O(n * 答案周期)。 当 m 达到 10^5 时，这显然是不可行的。 

关键的结构见解是无限行走中的每个位置都可以写成 k 个完整循环加上基本数组的前缀。 如果我们定义前缀和 P[i] = a1 + ... + ai 和总和 S = P[n]，那么在 t 步骤之后我们有：

 分数(t) = k * S + P[r]，其中 t = k * n + r 且 0 ≤ r < n。 

因此，问题简化为询问 x 是否可以写成这种形式，并在所有有效分解中找到最小的 t。 

对于每个前缀值 P[r]，我们检查是否可以选择一个整数 k，使得 x - P[r] 可以被 S 整除，并且商 k 是非负的。 如果 S = 0，则条件退化为 x = P[r]，因为循环不会改变总和。 

我们预先计算所有前缀和，并可选择存储每个前缀值的最小索引 r。 然后每个查询减少为检查所有候选者 r 并评估可行性。 由于 n 很大，每次查询迭代所有 r 太慢，因此我们改为对前缀和进行分组，并仅保留每个值最早出现的值。 这是有效的，因为对于相同的周期计数，较早的 r 总是给出较小的 t。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(m · n · 循环) | O(1) | O(1) | 太慢了 |
 | 前缀+模分解| O(n + m · d) 其中 d = 不同的前缀状态 | O(n) | 已接受 |

 真正的优化是认识到只有每个前缀和的第一次出现很重要，并在预处理前缀值的哈希映射后将每个查询减少为常数时间算术。 

## 算法演练

 ## 预处理

 1. 计算前缀和 P[i]，同时从左到右扫描一次数组。 

我们还记录总和 S = P[n]。

此步骤对所有可能的部分循环贡献进行编码。 
2. 构建一个字典 bestPos，其中 bestPos[v] 存储满足 P[i] = v 的最小索引 i。 

我们只保留最早的索引，因为它为该前缀值产生最少的步数。 

## 查询处理

 1. 如果 S 为零，则重复循环不会改变分数。 对于查询 x，我们检查 x 是否存在于 bestPos 中。 如果是的话，答案是bestPos[x]，否则不可能。 
2.如果S非零，我们迭代bestPos中所有存储的前缀和v。 
3. 对于每个前缀和 v，计算差值 delta = x - v。 
4. 检查 delta 是否能被 S 整除。如果不能，则该前缀不能导致 x。 
5. 如果可整除，则计算 k = delta // S。 
6. 仅考虑 k ≥ 0，因为我们不能采用负数的完整周期。 
7. 候选答案是 k * n + bestPos[v]。 我们跟踪所有有效候选人的最小值。 

## 为什么它有效

 每个可达状态完全对应于一定数量的完整周期加上下一个周期的前缀。 前缀和唯一地确定了一个周期内的部分贡献，而完整周期的数量决定了我们在 n 步的倍数中前进了多远。 因为每个前缀总和都是用其最早的索引记录的，所以我们总是选择该前缀组件的最小步数。 可除性条件确保了目标和周期贡献之间的一致性，并且将 k 限制为非负数确保我们只在时间上前进。 这耗尽了有效行走的所有可能分解，因此所有有效候选者中的最小值就是正确答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))

    prefix = [0] * (n + 1)
    best = {}

    for i in range(1, n + 1):
        prefix[i] = prefix[i - 1] + a[i - 1]
        if prefix[i] not in best:
            best[prefix[i]] = i

    S = prefix[n]

    for _ in range(m):
        x = int(input())

        if S == 0:
            if x in best:
                print(best[x])
            else:
                print(-1)
            continue

        ans = 10**30

        for v, idx in best.items():
            delta = x - v
            if delta % S != 0:
                continue
            k = delta // S
            if k < 0:
                continue
            ans = min(ans, k * n + idx)

        print(-1 if ans == 10**30 else ans)

if __name__ == "__main__":
    solve()
```该代码首先构建前缀和并仅存储每个值的第一次出现。 这种减少可以防止二次行为。 提取总和S来判断循环重复是否影响可达性。 

每个查询都是通过测试所有不同的前缀总和值来处理的。 算术检查强制目标和循环结构之间的兼容性，最小步表达式直接反映分解为完整循环和前缀。 

一个常见的陷阱是忘记 S = 0 的情况。 如果不将其分开，划分和模块化逻辑就会崩溃。 另一个微妙之处是使用每个前缀和的第一次出现而不是所有出现，这确保了最少的步数。 

## 工作示例

 考虑一个数组 [1, -3, 4]。 前缀和为 P = [1, -2, 2]，总和 S = 2。 

查询 x = 3：

 | 前缀 v | idx | δ = x - v | 增量 % S | k | 步骤 = k·n + idx |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 1 | 2 | 0 | 1 | 1·3 + 1 = 4 |
 | -2 | 2 | 5 | 1 | - | 无效|
 | 2 | 3 | 1 | 1 | - | 无效|

 最少 4 步。 

这显示了不同的前缀对齐如何对应于不同的循环计数，并且只有一种组合才能产生有效的分解。 

现在考虑 x = 1：

 | 前缀 v | idx | 三角洲 | 增量 % S | k | 步骤|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 1 | 0 | 0 | 0 | 1 |
 | -2 | 2 | 3 | 1 | - | 无效|
 | 2 | 3 | -1 | -1 模 2 | - | 无效|

 答案是 1 步，在第一个图块处立即实现。 

这些例子证实了分解正确地匹配了早期贡献和循环贡献。

 ## 边缘情况

 当所有值都为零时，无论经过多少步，分数都不会改变。 对于任何查询 x，在步骤 0 处只能到达 x = 0，所有其他目标都是不可能的。 该算法通过检测 S = 0 并检查最佳前缀是否存在来正确处理此问题。 

当数组仅包含负值时，前缀和严格减少，但循环重复仍然允许根据结构达到更多负甚至正目标。 S 的模块化条件确保只接受一致的分解，并且最早的前缀索引保证最少的步骤。 

当 n = 1 时，前缀结构会折叠为单个重复值。 该算法简化为检查 x 是否是该值的倍数，步数等于乘数，这与直接模拟相匹配。 

当 x 等于完整循环总和时，正确答案恰好是 n 步，算法通过 v = 0（第一个元素之前的隐式前缀）与 k = 1 相结合来捕获这一点。
