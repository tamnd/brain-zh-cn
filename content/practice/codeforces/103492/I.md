 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每条边代表两个城市之间的运输路线。 每条路线都有一个基本成本和一个折扣参数。 旅行者从城市 1 出发，可以沿着任何定向路径到达其他城市。 路径的成本不仅仅是边成本的总和，因为第一个边之后的每个边可能会根据其成本与之前使用的边相比如何进行折扣。 

更准确地说，路径中的第一条边始终贡献其全部基本成本。 对于每个后续边，我们将其基本成本与路径中前一条边的基本成本进行比较。 如果当前边的基本成本严格大于前一条边的基本成本，则从中减去其折扣值； 否则我们将支付其全部基本费用。 总路径成本是这些调整后的边成本之和。 我们需要从城市 1 到每个城市的尽可能最低的成本。 

关键的困难在于，一条边的成本取决于所选路径中的前一条边，因此这不是标准的最短路径问题。 搜索的状态必须以某种方式记住最后使用的边成本。 

约束很大：每个测试用例最多 10^5 个城市和 2×10^5 条边，最多 10^4 个测试用例。 任何简单地扩展每个边缘转换的状态或独立处理每个路径的解决方案都将太慢。 我们正在寻找每个测试用例总体上接近 O(m log m) 或 O(m log n) 的东西。 

当折扣连锁店出现时，就会出现一种微妙的边缘情况。 考虑一条路径，其中边缘成本严格增加数倍，允许重复折扣：

 输入：

 1 → 2（成本 1）

 2 → 3（成本 2，折扣 1）

 3 → 4（成本 3，折扣 1）

 最佳路径受益于第一次增加后每一步的折扣。 当应用折扣时，不跟踪先前边缘权重的朴素最短路径算法无法正确建模，因为可以使用不同的“最后边缘成本”到达同一节点，从而导致不同的未来结果。 

另一种边缘情况发生在较早采取稍贵一点的边缘，以便稍后获得更大的折扣时。 基于局部边缘成本或节点上经典 Dijkstra 的贪婪直觉会失败，因为转移规则依赖于历史。 

## 方法

 暴力方法会将问题视为由（节点，last_edge_cost）定义的状态上的最短路径。 从一个状态开始，我们尝试每个传出边缘，并根据新边缘成本是否大于上一个边缘来计算下一个成本。 这是正确的，因为它直接模拟了问题的定义。 

然而，状态的数量就成为问题。 每个节点都可以通过许多可能的先前边缘成本到达，每个传入路径可能有一个。 在最坏的情况下，这会导致 O(m^2) 行为，因为每次转换都可能生成一个新的不同状态，并且每个状态沿着传出边缘扩展。 这已经远远超出了限制。 

关键的观察是，转换仅取决于当前的边缘成本是否大于前一个边缘成本，而不取决于完整的历史记录。 这表明我们实际上并不需要存储所有可能的先前成本，只需要存储到达节点的最佳已知方式，同时在最后的边值上保持有意义的结构。 

我们将这个过程重新表述为按成本排序的边上的分层最短路径问题。 关键思想是以成本递增的顺序处理边，同时在两种情况下保持每个节点的最佳距离：在使用给定成本阈值的边后到达它。 这使得我们只有在从较小成本边缘转向较大成本边缘时才能正确应用折扣，这与不断增加的加工订单完美契合。

为了有效地支持转换，我们为每个节点维护一个最佳距离，以及一个允许在处理按成本分组的边时进行更新的结构。 在固定成本水平内，我们放宽边缘，而不会触发未来边缘的折扣，从而确保仅与先前处理的较小成本进行比较的正确性。 

这将问题转化为多阶段松弛，类似于修改后的 Dijkstra 或边缘权重的扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对（节点，最后成本）状态进行暴力破解 | 最坏 O(m^2) | O(m^2) | 太慢了 |
 | 具有类 Dijkstra 传播的排序边分层松弛 | O(m log m) | O(n + m) | 已接受 |

 ## 算法演练

 我们处理按基本成本分组的边，并按升序排序。 在处理过程中，我们维护一个距离数组，表示在完全处理具有较小基本成本的所有边之后每个节点的最佳已知成本。 

1. 按基本成本对所有边进行排序。 这确保了当我们处理一批具有相同成本的边时，所有较小成本的边都已被充分考虑。 
2. 维护一个初始化为无穷大的距离数组，但距离[1] = 0 除外，因为我们从城市 1 开始。这表示截至当前处理成本水平的已知成本。 
3. 以相等的基本成本分组处理边。 对于每个组，我们首先考虑仅使用先前确定的距离进行松弛。 
4. 对于具有成本 a 和折扣 b 的每条边 u → v，假设前一条边成本严格小于 a，我们尝试从 u 到 v 的转换。 由于所有较小成本的边都已被处理，distance[u] 表示以最后的边成本 < a 到达 u 的最佳方式。 
5. 我们将候选成本计算为 distance[u] + (a - b) 并将其推入该成本组的临时缓冲区中。 
6. 处理完组中的所有边后，我们使用缓冲区中的最佳值更新距离[v]。 这种分离确保具有相同成本的边不会错误地相互折扣。 
7. 继续，直到处理完所有边组。 
8. 最后输出距离数组，将不可达节点替换为-1。 

正确性取决于这样一个事实：折扣仅适用于前一个边缘严格较小的情况。 通过按升序处理边，我们保证在处理成本 a 时，所有较小的有效“先前边”已经完全合并。 

### 为什么它有效

 该算法保持了这样一个不变量：在处理完所有成本严格小于 x 的边后，distance[u] 表示使用最后一条边成本至多为 x 的路径到达 u 的最小成本。 当处理成本为 x 的边时，应触发折扣的每个有效转换完全对应于来自使用较小边构建的状态，该状态已编码在 distance[u] 中。 这确保了原始定义中的比较条件纯粹通过排序来强制执行，从而无需显式存储最后的边缘成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**30

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        edges = []
        for _ in range(m):
            u, v, a, b = map(int, input().split())
            edges.append((a, u, v, b))

        edges.sort()

        dist = [INF] * (n + 1)
        dist[1] = 0

        i = 0
        while i < m:
            j = i
            group = []
            cur_a = edges[i][0]
            while j < m and edges[j][0] == cur_a:
                group.append(edges[j])
                j += 1

            updates = []

            for a, u, v, b in group:
                if dist[u] < INF:
                    updates.append((v, dist[u] + a - b))

            for v, val in updates:
                if val < dist[v]:
                    dist[v] = val

            i = j

        res = []
        for k in range(1, n + 1):
            res.append(str(dist[k] if dist[k] < INF else -1))
        print(" ".join(res))

if __name__ == "__main__":
    solve()
```该代码首先按成本对所有边进行排序，这是排序参数的支柱。 距离数组跟踪迄今为止到达每个城市的最佳成本。 具有相同成本的每组边被一起处理，以便相同权重的边不会通过中间更新错误地相互影响。 

更新阶段特意分为收集和应用。 这可以防止在同一组内使用新更新的距离，这将打破严格的“前一个边缘必须更小”条件。

 ## 边缘情况

 一种重要的情况是多条边具有相同的成本。 该算法依赖于将它们作为一个组进行处理，以便在相同的成本水平内，没有一条边可以从另一条同等成本的边中受益。 例如，如果两条边的成本均为 5，我们不得允许使用一条边的路径在同一迭代中立即折扣另一条边。 分组更新步骤强制执行这种分离。 

另一种情况是，最佳路径需要延迟使用廉价优势，以便稍后获得更好的折扣。 由于无论超出成本分组的边缘排序如何，该算法始终传播已知距离，因此这种延迟优化自然可以通过跨增加成本层的重复松弛来处理。 

最后，无法到达的节点保持无穷大，并在输出中正确转换为 -1。
