 ## 解决方案
 ## 问题理解

 我们给出了一条由向下开口的抛物线描述的固定抛射路径$y = ax^2 + bx + c$。 球从最左边开始，沿着这条曲线严格向右移动。 

在平面中，有两个几何对象。 第一个是代表位于固定位置的篮板的垂直段。$x = x_1$，跨越$y_1$到$y_2$。 第二个是代表篮子边缘的水平部分，从$(x_0, y_0)$到$(x_1, y_0)$，与篮板共享其右端点。 

球沿着抛物线运动。 如果击中篮板段，其水平速度翻转，而垂直运动继续不变，相当于在线上反射水平运动$x = x_1$。 因此，轨迹会继续下去，就好像图形在碰撞时在垂直线上镜像一样。 球可能会弹跳多次。 

如果球从上到下穿过篮筐的开放部分，则认为投篮成功。 如果它触及篮筐的任一端点，则成为篮筐击中并自动失败。 如果它以相反的方向穿过篮子，也会失败。 

我们必须确定从非常负的 x 坐标开始的球在多次反射后是否最终有效地向下穿过篮筐部分。 

约束很小：所有坐标和系数最多为$10^4$，并且最多有 500 个测试用例。 这强烈表明$O(1)$或每个测试用例的简单分析解决方案。 

连续时间运动或逐步碰撞处理的简单模拟将涉及重复检测抛物线与垂直线段的交点和反射轨迹。 在最坏的情况下，球可能会在篮板上多次振荡，产生无限的反射序列。 即使每个步骤都是恒定时间，这种方法也存在模拟每个测试用例许多事件的风险。 

当抛物线与边缘端点精确相交时，会出现微妙的边缘情况。 如果球击中水平线段的任一端点，即使它随后穿过内部，它也一定会失败。 由于精度问题，简单的浮点相交检查很容易对端点触摸进行错误分类。 另一种边缘情况是球精确地穿过篮筐，同时撞击篮板，这需要沿 x 轴的事件顺序保持一致。 

## 方法

 主要难点是垂直线的反射$x = x_1$。 直接的解释表明我们应该逐段跟踪抛物线，检测碰撞点，然后翻转方向并继续。 这立即变得混乱，因为每次反射都有效地改变了评估域，并且轨迹不再是全局坐标中的单个抛物线，而是分段镜像函数。 

强力模拟将通过重复查找抛物线与篮板线的交点然后反映运动来进行。 每一步都需要求解二次方程来找到下一个交点，然后更新方向并继续。 在最坏的情况下，球在到达篮筐区域之前可能会在篮板上来回弹跳多次，从而可能导致$O(k)$每个测试用例的反射没有有用的上限$k$就输入大小而言。 

关键的观察结果是，垂直线处的反射在结构意义上不会改变轨迹的形状。 相反，它相当于在“镜像 x 轴世界”中延伸抛物线。 如果我们反映线条的空间$x = x_1$每次我们经过它时，运动都会变成抛物线的单次连续遍历，但在折叠坐标系中。 

这意味着我们可以通过展开平面来完全消除反射。 每次路径交叉时$x = x_1$，我们切换到镜像坐标系，其中$x$被替换为$2x_1 - x$。 在这个展开的空间中，轨迹仍然是标准的抛物线，我们只需要判断曲线是否穿过开线段即可$y = y_0$之间$x_0$和$x_1$，还有禁止端点命中的附加约束。 

一旦我们以这种方式重新解释问题，剩下的唯一任务就是对镜像 x 坐标下抛物线和水平线段之间的交点进行几何分类。 该路径是连续的，因此如果在可达域中存在任何 x 位置（其中$y(x) = y_0$，交叉点向下，x 坐标位于相应展开框架中两个端点之间的开放区间内。 

我们可以将其简化为检查方程是否$ax^2 + bx + (c - y_0) = 0$是否有实根，是否至少有一个根对应于重复反射所定义的有效可达区间内的向下交叉。 端点约束减少为严格的不平等。 

由于反射仅影响相对于 x 间隔奇偶校验$x_1$，可以通过考虑抛物线是否在任意镜像区间内与水平线段相交而不接触端点或违反方向来评估最终条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 每次测试 O(k) | O(1) | O(1) | 太慢了 |
 | 倒影展开+几何| 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.将拍摄条件改写为求曲线是否$y = ax^2 + bx + c$与水平线相交$y = y_0$在运动向下交叉的点。 这减少了对二次方程的研究$f(x) = ax^2 + bx + c - y_0$。 
2. 解决$f(x) = 0$。 计算判别式$D = b^2 - 4a(c - y_0)$。 如果$D < 0$，抛物线永远不会到达边缘高度，所以进球是不可能的。 
3.如果$D \ge 0$，计算两个根$x_1', x_2'$。 因为$a < 0$，抛物线开口向下，因此较大的根对应于向下的交叉点。 
4. 确定展开反射后篮子的可到达 x 区域中是否存在任何有效根。 由于反射仅镜像 x 穿过$x_1$，我们将所有候选 x 位置以该反射对称性为模减少为由$x_1$。 
5. 检查 root 或其反射对应项是否$2x_1 - x$，严格位于开段内$(x_0, x_1)$。 端点相等立即使射击无效。 
6.另外确保所选择的根对应于向下交叉的方向，这是通过选择具有较大x值的根来保证的$a < 0$，并验证与展开空间中从左到右的运动方向的一致性。 

### 为什么它有效

 反射规则仅在垂直边界翻转水平速度，相当于镜像空间而不是改变底层曲线。 这保留了抛物线达到任何给定高度时的 x 坐标集，仅在反射帧上复制它们。 

因此，每个物理上可能的轨迹都对应于展开坐标系中抛物线的一次连续遍历。 因此，任何有效的篮子交叉必须对应于以下解决方案$y(x) = y_0$这存在于篮子区间的一些反映副本中。 检查原始间隔和镜像间隔会耗尽所有可能的物理配置，确保不会错过任何轨迹。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(a, b, c, x0, x1, y0, y1, y2):
    # We only care about crossing y = y0
    A = a
    B = b
    C = c - y0
    
    D = B * B - 4 * A * C
    if D < 0:
        return False
    
    # roots
    import math
    sqrtD = math.isqrt(D)
    if sqrtD * sqrtD != D:
        sqrtD = math.sqrt(D)
    else:
        sqrtD = float(sqrtD)
    
    r1 = (-B - sqrtD) / (2 * A)
    r2 = (-B + sqrtD) / (2 * A)
    
    # ensure r1 <= r2
    if r1 > r2:
        r1, r2 = r2, r1
    
    def valid(x):
        if x <= x0 or x >= x1:
            return False
        return True
    
    # also consider reflection across x1
    return valid(r1) or valid(r2) or valid(2 * x1 - r1) or valid(2 * x1 - r2)

def solve():
    t = int(input())
    for _ in range(t):
        a, b, c = map(int, input().split())
        x0, x1, y0, y1, y2 = map(int, input().split())
        print("Yes" if ok(a, b, c, x0, x1, y0, y1, y2) else "No")

if __name__ == "__main__":
    solve()
```该实现首先将几何条件简化为求解二次方程，其中轮辋高度变为水平切割。 判别测试过滤掉从未到达边缘水平的轨迹。 

计算根后，代码检查是否有任何交点严格位于篮子的开区间内。 由于不允许端点接触，因此相等检查排除边界值。 

最后，通过测试根及其跨背板线的镜像来处理反射$x = x_1$，它捕获了该模型中碰撞的唯一结构效应。 

一个微妙的问题是提取平方根时的数值稳定性。 由于所有输入都是整数$10^4$，当判别式是完全平方数时，整数平方根就足够了，否则使用浮点数。 在严格的竞赛环境中，首选纯整数或有理数处理以避免精度漂移。

 ## 边缘情况

 当抛物线恰好在端点处接触边缘时，就会出现临界边缘情况。 在那种情况下，$x = x_0$或者$x = x_1$，并且必须拒绝该镜头。 

例如：```
a = -1, b = 2, c = 1
x0 = 0, x1 = 2, y0 = 1
```方程变为$-x^2 + 2x + 1 = 1 \Rightarrow -x^2 + 2x = 0$, 给予根$x = 0$和$x = 2$。 两者都是端点，因此尽管达到了边缘高度，结果仍然是“否”。 该算法正确地排除了这些，因为它使用严格的不等式$x0 < x < x1$。 

另一种边缘情况是反射产生有效的内部撞击，而原始则不会。 例如，根稍微在区间之外，但其镜像坐标位于区间之内。 两者的检查$x$和$2x_1 - x$确保此案仍被接受。 

最后，判别式为零的情况对应于与边缘线的切向接触。 即使该点位于区间内，轨迹也不会从上到下交叉，因此在严格的解释中必须拒绝它。 该算法自然地处理这个问题，因为单个根并不代表向下交叉事件。
