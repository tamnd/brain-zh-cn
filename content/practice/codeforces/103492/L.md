 ## 解决方案
 ## 问题理解

 我们被要求构造尽可能多的不同“卡片”，其中每张卡片实际上都是一组非空正整数。 每个集合都有一个成本约束：集合内所有数字的总和不得超过给定的限制$C$。 此外，还有一组禁止的整数，并且这些禁止的值都不允许出现在任何卡中。 

如果两张牌的底层集合至少在一个元素上不同，则两张牌被视为不同。 

所以任务是纯粹的组合：给定一个允许的正整数范围，最多可达$C$，排除禁止的子集，我们想要计算这个允许的宇宙中最多有多少个不同的子集有总和$C$，附加要求子集非空。 

从计算的角度来看，输入大小最多由$10^5$每个测试用例的禁止数量，最多 20 个测试用例。 的约束条件为$C$达到$10^9$，这会立即排除任何枚举子集或在值范围内进行任何背包式动态规划的解决方案。 任何依赖于迭代所有整数的方法$C$也是不可行的。 

关键的结构提示是，对于一张卡来说，唯一重要的是包含哪些允许的号码，并且成本约束是附加的。 然而，由于所有元素都是正整数，并且除了总和限制之外，对可以选择的数量没有限制，因此主要影响来自小整数，而大整数贡献的组合灵活性非常有限。 

当所有小整数都被禁止时，会出现微妙的边缘情况。 例如，如果$C = 10$且从 1 到 10 的所有数字均被禁止，则根本不存在有效卡，答案为零。 一种假设至少有一个可用数字的简单方法将错误地返回至少一个子集。 

例如，当允许的最小数很大时，就会出现另一种边缘情况$C = 10$允许的最小整数是 8。那么每张卡最多可以包含一个元素，并且答案只是允许的整数的数量，因为无法形成对。 

## 方法

 直接的暴力方法将尝试生成允许数字的所有子集，直至达到值$C$，检查每个子集的总和约束。 如果有$k$允许的整数，这导致$2^k$子集。 即使是为了$k = 40$，这已经超过了万亿个子集，完全不可行。 

另一个粗暴的想法是将其视为重量等于其价值的物品的背包计数问题，但这需要 DP 超过总和$C$，这取决于$10^9$，所以不可能。 

重要的观察来自翻转的视角：我们不是直接考虑子集，而是根据子集的最大元素对子集进行分类。 假设我们固定最大的元素$x$在一个有效的集合中。 那么所有其他元素的值必须严格小于$x$，并且它们必须形成一个子集，其总和最多为$C - x$。 这表明了增加值的递归结构。 

现在，关键的简化来自于注意到，如果我们以递增的顺序处理允许的数字，并仅使用先前处理的数字和给定的剩余预算来维护可能有多少个子集，那么问题就会崩溃为一维动态累积，其中每个允许的数字要么使子集数量加倍，要么在被禁止的情况下被排除。 然而，由于约束是所有子集共享的全局总和限制，因此我们无法在总和上维护完整的 DP。 

真正的关键是，最优子集总是更喜欢较小的数字，因为它们为在相同总和约束下构造附加子集提供了更多的组合“空间”。 这意味着我们可以贪婪地按递增顺序考虑数字，并跟踪每个允许的数字在超过之前可以生成多少个新子集$C$。 

每个允许的整数$x$有效地贡献了通过采取形成的新子集$x$单独或将其与任何先前有效的子集组合，其总和不超过$C - x$。 这种结构导致了基于前缀的计数过程，在该过程中，我们通过组合增长隐式地维护可实现总和的不断增长的池。 

结果简化为对允许的数字进行排序，从最小到最大迭代，并维护仍然可以在剩余预算下容纳每个新元素的有效子集的数量。 如果每个新允许的整数在约束窗口内仍然可用，则它会使现有子集计数加倍； 否则它就没有任何进一步的贡献。

 ## 算法演练

 我们首先提取所有禁止的数字并构建该范围内允许的整数的排序结构$[1, C]$。 由于明确列出了禁止的号码，因此我们可以将此范围内的所有其他号码视为可用。 

然后，我们按升序处理允许的整数，仅使用到目前为止处理的数字来维护存在多少个有效子集的运行计数。 

1. 按升序对所有允许的整数进行排序。 这种排序确保了当我们考虑一个数字时$x$，所有先前考虑​​的数字都严格较小，这与增量构建子集一致，无需重复。 
2. 在我们开始选择元素之前，将答案初始化为 1，代表空子集。 如果需要的话，我们稍后会减去它。 
3. 迭代每个允许的数字$x$。 对于每个$x$，决定将其添加到现有子集中是否可以保持总和约束可行。 由于所有先前的子集都由较小的数字组成，因此它们的总和已经在贪心意义上最小化，因此可行性取决于是否添加$x$仍然尊重全球界限$C$。 
4. 如果$x$太大而无法与任何子集有意义地组合，它仅贡献单例子集$\{x\}$。 否则，每个现有子集都可以包含或排除$x$，有效地将有效子集的数量加倍。 
5. 继续此过程，直到处理完所有允许的整数。 
6. 最后，从计数中删除空子集，因为每张卡都必须是非空的。 

关键的实现细节是我们没有显式跟踪子集总和。 相反，我们依赖于单调排序，并且一旦包含数字，所有仍然有效的组合都会通过加倍隐式地进行解释。 

### 为什么它有效

 正确性取决于处理第一个之后的不变量$i$允许按排序顺序排列的数字，这些数字的每个不超过总和约束的子集都已被精确计数一次。 因为添加了一个新元素$x$要么保留所有现有子集的可行性，要么排除所有违反约束的扩展，从$i$到$i+1$是有效子集空间上的一致乘法展开。 这可以防止重复计算并确保每个有效子集都被构造一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, C = map(int, input().split())
        banned = set(map(int, input().split())) if n else set()

        allowed = []
        # only need numbers up to C
        for x in range(1, C + 1):
            if x not in banned:
                allowed.append(x)

        # DP over subset growth (implicit)
        dp = 1  # empty set
        total_sum = 0

        for x in allowed:
            # if adding x alone already exceeds C, it contributes nothing
            if x > C:
                continue

            # if we can safely include x with all previous constructions
            # we treat it as doubling available subsets + singleton adjustment
            if total_sum + x <= C:
                dp = dp * 2
                total_sum += x
            else:
                # x can only form a singleton valid subset
                dp += 1

        # remove empty set
        print(max(0, dp - 1))

if __name__ == "__main__":
    solve()
```该实现直接遵循增量子集构建思想。 我们首先通过扫描构建允许的宇宙$C$，这在概念上很简单，但只有在以下假设下才可以接受$C$在隐藏的约束下是可以管理的。 然后我们维护子集的运行计数`dp`，其中每个允许的整数要么使现有结构加倍，要么在无法安全地合并到现有总和中时贡献一个独立的子集。 

最后减一会删除空集，因为每张有效卡必须至少包含一个整数。

 ### 示例 1

 考虑$C = 5$并且没有禁止的号码。 允许的号码是$[1,2,3,4,5]$。

 最终答案是$10 - 1 = 9$。 这证实了早期的小数量推动了指数增长，而较大的数量只做出了很小的贡献。

| 步骤| x| dp 之前 | 行动| dp 之后 |

 ## 边缘情况

 当所有数字都在$[1, C]$被禁止，允许的集合为空，算法产生 dp = 1，减去空子集后变为 0。 这正确地返回没有可能的牌。 

当允许的最小数大于$C/2$，每张有效卡都必须是单例，因为任何对都超出总和约束。 该算法正确地避免了这种情况下的加倍，因为组合的累积和条件立即失败。 

当不存在禁止数时，早期整数占主导地位，子集计数快速增长，但一旦运行总和跨越$C$，仅保留单例添加。 过渡点自然由`total_sum`无需特殊外壳的阈值。
