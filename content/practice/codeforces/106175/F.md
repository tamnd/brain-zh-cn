 ## 解决方案
 ## 问题理解

 我们获得了初始金额和固定年限。 每年，我们都被允许持有多组债券。 每种债券类型都有购买成本并产生固定的年度利润。 关键规则是债券可以按面值自由买卖，并且没有交易成本。 每年，总资本的增长量是当前持有的所有债券利息的总和，然后我们可以使用更新后的资本来重新平衡投资组合。 目标是决定每年持有每种类型的债券数量，以便在最后一年之后，总资本最大化。 

重要的结构是，除了通过积累资本之外，每年的决策都是独立的。 每年年初，我们确切地知道我们有多少钱，我们可以将其转换成任何债券组合。 持有一年后，我们获得确定性回报，然后重复这个过程。 

债券种类的限制极小，最多10种，但年数最多可达40年，初始资本可达100万。 这立即表明我们无法跟踪任意细粒度的货币分配； 相反，我们应该期待一种动态规划或状态压缩方法，其中状态只是当前资本。 

一种天真的方法是尝试考虑每年所有可能的债券组合。 即使我们以 1000 为单位离散化货币（因为所有债券价值都是 1000 的倍数），资本范围也会快速增长，组合也会呈爆炸式增长。 对于一年，如果有 d 只债券，并且我们可以购买每种债券的倍数，则可能的投资组合数量已经与资本规模相结合。 

当每年使用贪婪策略而不展望未来时，就会出现微妙的边缘情况。 例如，选择最佳的即时利息与价值比率可能会降低未来的复利潜力。 比率稍差但与未来资本规模更一致的债券可以带来更高的长期增长。 

另一个失败案例是假设部分投资是可能的。 这个问题隐含地迫使债券的数量为整数，因此将其视为连续的背包会高估回报并产生无效的投资组合。 

## 方法

 强力解释是逐年模拟，并在每一年列举在债券之间分配流动资本的所有可能方式。 对于固定资本 C 和 d 债券类型，我们将考虑所有元组 (x1, x2, ..., xd)，使得 sum(xi * value_i) ≤ C，并计算最终利润。 在最坏的情况下，此类分配的数量以 C/1000 为指数，因为它是一个有界背包，计数所有配置而不是选择一个。 即使对于中等 C 来说，这也是不可行的。 

关键的观察是，每年的决策仅取决于当前资本，并且债券除了通过线性约束之外不会相互作用。 这意味着我们不需要记住资本是如何获得的，只需要记住它的价值。 更重要的是，对于固定资本来说，一年内的最优配置问题本身就是一个经典的无界背包问题：一年后总回报最大化。 

因此，问题简化为重复转换：给定当前资本 C，通过求解无界背包来计算一年后的最大资本，其中每个项目 i 都有重量 value_i 和利润 value_i + 利息_i（因为一年后我们恢复价值加利息）。 我们重复这个转变 T 次。 

这将整个问题简化为应用相同的 DP 转换 T 次，其中每次转换都是一个容量为 C 的背包（每年都会变化）。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 投资组合的暴力枚举 | 指数| 指数| 太慢了|
 | 逐年无界背包DP| O(T * C * d) | O(C)| 已接受 |

 ## 算法演练

 我们将 dp[c] 定义为在一年内为给定可用资本 c 选择债券后可实现的最大资本。 过渡计算将 c 投资于债券的最佳方式，然后在一年后返还由此产生的资本。 

1. 从 dp[0] = 0 开始，所有其他值初始化为 0。 

这代表着零钱，我们无法投资任何东西。 
2. 对于每种债券类型 i，将其视为具有成本 value_i 和回报值 value_i + 利息_i 的项目。 

这将财务操作转换为标准的无界背包公式。 
3. 对于固定年份，使用无界背包转换计算新数组 best[c]：

 best[c] = max(best[c], best[c - value_i] + value_i +interest_i - value_i)，简化了跟踪资本使用净收益的过程。 

更直接地，我们将 best[c] 计算为使用总成本最多为 c 的项目可实现的最大总价值，其中每个项目贡献其全部年终价值。 
4.计算出当年的最佳后，将当前资本C设置为最佳[C]。 

此步骤将整个分布折叠回单个标量状态，这已经足够了，因为未来的决策仅取决于总资本。 
5. 重复该过程指定的年数。 

核心思想是每年都是一个具有单个标量状态转换的完整重新优化问题。 

### 为什么它有效

 关键的不变量是，每年之后，对于每个可能的资本金额 c，dp[c] 正确地表示从 c 开始的最大可实现年终资本。 由于明年仅取决于总资本，而不取决于其内部结构，因此我们可以安全地将所有投资组合配置压缩为单个值。 无界背包确保我们在每一年之内探索所有有效的债券组合，不遗漏任何可行的投资策略。 重复这种转变可以保持最优性，因为除了通过标量资本价值之外，每年的决策都是独立的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N = int(input())
    for _ in range(N):
        C, T = map(int, input().split())
        d = int(input())
        bonds = [tuple(map(int, input().split())) for _ in range(d)]

        # scale down by 1000 since all values are multiples of 1000
        C //= 1000
        vals = []
        gains = []
        for v, r in bonds:
            v //= 1000
            vals.append(v)
            gains.append(v + r // 1000)

        cap = C

        for _ in range(T):
            dp = [0] * (cap + 1)

            for i in range(d):
                w = vals[i]
                val = gains[i]
                for c in range(w, cap + 1):
                    if dp[c - w] + val > dp[c]:
                        dp[c] = dp[c - w] + val

            cap = dp[cap]

        print(cap * 1000)

if __name__ == "__main__":
    solve()
```该实现将所有货币值压缩 1000，以缩小状态规模，因为每个债券价值都保证是 1000 的倍数。每年都会构建一个新的背包 DP，其中 dp[c] 使用最多 c 单位的资本来存储最佳可实现的年终资本。 

内循环是无界背包转换，我们向上迭代容量，以便每个键可以多次使用。 完成一年后，我们只保留 dp[cap]，因为我们只关心最大可实现的满容量投资结果。 

将利息转换为与资本相同的规模时必须小心，因为两者在 DP 单位上必须具有可比性。 另一个微妙的点是，我们每年都会重建 dp，而不是尝试重用它，因为可用资本会在每次迭代时发生变化。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(T * C * d) | 每年运行一个容量为 C 和 d 的物品的无限背包 |
 | 空间| O(C)| 每年仅一个 DP 阵列 |

 ## 边缘情况

 当所有债券的利息为零时，就会出现一个微妙的情况。 DP 应该始终准确地保存资本，因为没有任何配置可以增加价值。 该算法处理这个问题是因为每个转换值都等于成本，因此背包永远不会将 dp[c] 提高到超过 c。 

另一种情况是仅存在一种债券类型。 DP 退化为每年重复购买尽可能多的副本，状态压缩确保我们在每次迭代时正确地重新计算最大倍数。 

最后，当资本相对于债券规模较小时，许多 DP 条目仍为零。 该算法仍然有效，因为转换仅激活可达状态，并且 dp[cap] 始终反映最佳可行配置，而不需要完整的状态探索。
