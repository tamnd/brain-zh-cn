 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的字符串，并且我们最多可以从中删除一个字符。 经过这个可选的删除之后，我们获得一个新字符串，并且在所有可能的结果（包括什么都不做）中，我们想要字典顺序最小的一个。 

这里的字典顺序的行为类似于字典比较：两个字符串从头开始逐个字符进行比较，它们不同的第一个位置决定哪个较小。 如果一个字符串是另一个字符串的前缀，则较短的字符串较小。 

该约束允许字符串长度最大为 200,000。 这立即排除了任何独立尝试每个删除位置并天真地比较字符串的解决方案，因为这在最坏的情况下会导致 O(n²) 行为。 在严格的限制下，即使每个位置重复进行线性扫描也会太慢。 

一些边缘情况很重要。 

如果字符串在字典意义上已经是非递减的，那么当将其视为早期字符已经尽可能小的序列时，删除任何内容可能都没有帮助。 例如，在`"aaa"`，每个字符都是相同的，因此所有删除都会得到相同的结果。 

一个更微妙的情况是字符串中的第一个“drop”非常晚。 例如，在`"abdc"`，删除任一`'d'`或者`'c'`可以影响词典编排结果，并且最佳选择取决于第一次违反单调性的位置。 

天真的推理的另一个失败案例是假设我们应该始终删除比下一个字符大的第一个字符。 为了`"bcaa"`，去除`'b'`给出`"caa"`，但删除`'c'`给出`"baa"`，较小。 

核心困难在于决策不是本地的。 字符串中较早的删除可能会影响稍后获得的任何好处。 

## 方法

 暴力方法很简单：尝试删除每个位置一次（加上不删除任何内容的选项），构建结果字符串，然后比较所有结果以找到最小值。 每个候选构建成本为 O(n)，并且有 O(n) 个候选，从而导致 O(n²) 时间。 当n达到200,000时，这是完全不可行的。 

关键的观察是我们实际上不需要比较完整的候选字符串。 我们只需要决定删除一个字符将在哪里改善字典序分歧的第一个点。 这意味着我们正在寻找字符串停止“局部最优”的最早位置。 

更准确地说，我们从左到右扫描并查找第一个字符大于下一个字符的索引。 此时，保留当前字符会使前缀变得比必要的更糟糕，因为我们可以跳过它并立即暴露字符串中较早的较小字符。 一旦我们删除该字符，任何后续决定都不再影响字典顺序最小值，因为字典顺序比较已经由最早的差异决定。 

如果不存在这样的位置，则字符串是非递减的，这意味着每个字符都小于或等于下一个字符。 在这种情况下，删除最后一个字符会产生最小的结果，因为它会减少长度而不会恶化字典顺序。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 从第一个字符到倒数第二个字符遍历字符串。 

我们将每个字符与下一个字符进行比较，因为只有当较大的字符位于较小的字符之前时，词典编排的改进才有可能。 
2.找到第一个索引`i`这样`s[i] > s[i+1]`。 

这是字符串在前缀顺序中不再是最佳的第一个点。 
3. 如果存在这样的索引，则通过删除来构造答案`s[i]`。 

删除此字符会更早地暴露一个较小的字符，这保证了字典顺序上最小的可能前缀。 
4. 如果不存在这样的索引，则该字符串是非递减的。 

在这种情况下，删除任何内部字符都无法改善字典顺序，因此我们删除最后一个字符以最小化长度。 

### 为什么它有效

 该算法依赖于字典顺序比较是由第一个不同位置确定的事实。 最早的索引在哪里`s[i] > s[i+1]`是提高这一决定性地位的第一个机会。 删除该索引之前的任何字符只会使前缀变得更糟或保持不变，而删除其之后的任何字符不会影响已经确定的前缀比较。 这创建了一个贪婪的最优子结构：第一个“删除”完全确定了最佳可能的删除点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    s = input().strip()

    drop = -1
    for i in range(n - 1):
        if s[i] > s[i + 1]:
            drop = i
            break

    if drop == -1:
        print(s[:-1])
    else:
        print(s[:drop] + s[drop + 1:])

if __name__ == "__main__":
    solve()
```该解决方案读取字符串并扫描一次，以找到第一个字符大于下一个字符的位置。 该索引存储为`drop`。 如果不存在这样的索引，我们通过切片删除最后一个字符`s[:-1]`。 

一个微妙的点是，我们在第一次违规时立即中断。 这是至关重要的，因为一旦字典顺序关键前缀被修复，以后的违规行为就无关紧要了。

### 示例 2

 输入：`"abdc"`| 我| s[i] | s[i+1] | s[i+1] 下降|

 我们停在索引 2 处，因为`d > c`。 去除`d`给出`"abc"`。

这表明第一个减少点决定了最佳删除。

 ## 边缘情况

 对于像这样的完全递增的字符串`"abcde"`，该算法永远找不到一个位置`s[i] > s[i+1]`， 所以`drop`停留`-1`。 该代码删除最后一个字符并返回`"abcd"`，这是正确的，因为任何内部删除都会产生按字典顺序排列的比简单缩短字符串更大的前缀。 

对于具有早期反转的字符串，例如`"za"`，循环立即检测到`z > a`在索引 0 处。删除索引 0 会产生`"a"`，这是最小的可能结果，因为任何其他操作都会保留`'z'`作为前缀并使字典顺序恶化。 

对于具有重复字符的字符串，例如`"bbbbbb"`，不存在反转，因此算法删除最后一个字符，产生`"bbbbb"`。 每个可能的删除都会产生相同的词典编排值，并且这种选择在保持一致的同时保留了正确性。
