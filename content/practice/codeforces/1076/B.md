 ## 解决方案
 ## 问题理解

 我们给定一个大整数，然后我们重复应用总是减少它的确定性过程。 在每一步中，我们都会检查当前数字，找到其最小质因数，从该数字中减去该因数，然后继续直到该值变为零。 任务是确定在进程终止之前发生多少个这样的减法步骤。 

输入大小达到$10^{10}$，这立即排除了任何尝试使用朴素试除法在每一步从头开始分解数字的方法$n$。 线性扫描可达$n$将远远超出可行的限制，并且如果执行太多次，即使每次迭代重复进行平方根因式分解也会变得临界。 

关键的微妙之处在于迭代次数不受任何小东西的限制，例如$\log n$。 在最坏的情况下，小素数的重复减法会产生长链。 然而，每一步都严格减少$n$，更重要的是，一旦$n$变为偶数时，其最小素因数稳定在 2，这会创建一个长的确定性减半序列。 

当每一步低效地重新计算素数或最小素因数时，就会出现天真的错误。 例如，如果我们通过检查所有数字来重新计算除数$n$，甚至输入$n = 10^{10}$将无法处理。 另一种微妙的失败模式是忘记减去奇素数后，数字可能会变成偶数，从而极大地改变最小除数。 

## 方法

 暴力模拟直接遵循规则：在每次迭代时，从 2 向上扫描，直到找到当前数字的最小除数，将其减去，然后重复。 这是正确的，因为它准确地反映了该过程。 然而，通过扫描找到最小除数$\sqrt{n}$每一步都会导致最坏情况的复杂性$O(n \sqrt{n})$如果$n$下降缓慢，对于$10^{10}$。 

关键的观察是我们实际上并不需要在每一步都进行完全分解。 我们只需要最小的素因数。 一旦我们检测到一个数是否为偶数，我们立即知道答案是2。否则，我们只需要检查奇数除数$\sqrt{n}$。 自从$n$在奇数情况下至少减少 2，在偶数情况下减少一半，但在实践中昂贵的因式分解的数量仍然很小。 这将问题变成了素数式检查的受控序列，而不是完全重复的分解。 

该结构变得高效，因为大多数长序列发生在数字变得更早的时候，然后我们重复减去 2 而无需重新计算任何昂贵的东西。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（每一步扫描除数）|$O(n \sqrt{n})$|$O(1)$| 太慢了 |
 | 通过快速最小素数检查优化模拟 |$O(\sqrt{n} + \text{steps})$|$O(1)$| 已接受 |

 ## 算法演练

 1. 从给定的数字开始$n$。 将操作计数器初始化为零。 每个操作对应于当前最小素因数的一次减法。 
2. 虽然$n > 0$，确定最小素因数$n$。 从 2 开始测试可能的最小候选因子，但我们可以在找到的第一个除数处停止，因为任何较小的因子都必须已经是素数。 
3.一次最小除数$d$被识别，减去它$n$。 将操作计数器加一。 这准确地反映了所描述的过程的一种应用。 
4. 重复直到$n$变为零。 

一个关键的效率点是，一旦我们检测到$n$是偶数，我们立即知道$d = 2$无需进一步检查。 只有奇数需要由奇数候选人进行试除，最多$\sqrt{n}$。 

### 为什么它有效

 在每一步，算法都会复制定义过程的确切规则。 唯一的优化是如何找到最小素因数，但这不会改变减法的顺序，只会改变识别它们的速度。 由于最小素因数是唯一的，并且每次迭代总是减去一次，因此该过程及其计数与原始定义保持相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def smallest_prime_divisor(x: int) -> int:
    if x % 2 == 0:
        return 2
    i = 3
    while i * i <= x:
        if x % i == 0:
            return i
        i += 2
    return x  # x is prime

n = int(input().strip())
cnt = 0

while n > 0:
    d = smallest_prime_divisor(n)
    n -= d
    cnt += 1

print(cnt)
```核心函数隔离最小素因数。 偶数情况会立即处理，因为 2 始终是可能的最小质因数。 对于奇数，我们仅测试奇数候选数直到平方根，因为任何合数都必须具有该范围内的因子。 

主循环直接实现该过程。 每次迭代都会减少$n$严格保证终止。

 ## 边缘情况

 对于$n = 2$，算法立即将 2 识别为最小素因数，并将其减去一次，以零终止。 这确认了最小输入大小的正确处理。 

对于质数，例如$n = 11$，最小的除数是数字本身，因此该过程一步结束。 这会检查算法是否不会错误地搜索较小的不存在因素。 

对于像这样的偶数$n = 10$，第一次减法后变成8，然后继续重复减2，直到零。 这验证了算法正确稳定在除数 2 上，没有重新计算错误。
