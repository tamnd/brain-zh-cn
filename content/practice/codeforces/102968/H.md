 ## 解决方案
 ## 问题理解

 我们得到了一个整数序列，并且我们想要对子序列进行计数（因此我们按升序选择索引，不一定是连续的），并对我们选择的值具有特殊属性。 

如果所有选择的值都是不同的，并且一旦查看其最小值和最大值，整个区间中的每个整数都会出现在子序列中，则子序列被认为是有效的。 换句话说，对所选值进行排序后，它们必须形成一个没有间隙的连续整数块。 

因此，如果我们选择值，它们的行为就像一个集合而不是一个序列。 索引的顺序仅与我们可以选择这些值的方式有关，但值条件仅取决于集合。 

例如，选择 7、8、9 等值是有效的，因为范围是连续的。 选择 7 和 9 而没有 8 是无效的，因为区间中缺少 8。 重复也是被禁止的，因为子序列必须由不同的值组成。 

输入大小最高可达 100000，值也最高可达 100000。这排除了直接尝试所有子序列或所有值对的任何情况。 在该值范围内的二次解已经达到了 10^10 次运算，这远远超出了任何时间限制。 

一个微妙的极端情况是原始序列中完全缺失所选区间中的值。 例如，如果我们尝试形成一个涵盖值 3 到 6 的有效集合，但值 5 从未出现在数组中，则不可能构造这样的子序列，因此该区间贡献了零个有效子序列。 另一个特殊情况是数组中的重复值：重复项不会破坏有效性，但会增加选择索引的方式数量。 

## 方法

 暴力视图是枚举每个可能的子序列，计算其最小值和最大值，并验证它们之间的所有整数是否在所选集合中恰好出现一次。 即使我们忽略排序，也有 2^N 个子序列，所以这立即是不可行的。 

一种更结构化的尝试是根据我们可以选择的不同价值集进行思考。 任何有效的子序列都对应于选择一个值区间 [L, R]，然后为该区间中的每个值精确选取一次出现。 如果数组中没有出现某个值，则该间隔是不可能的。 如果某个值出现 cnt[v] 次，则在固定间隔内，形成子序列的方式数是 cnt[v] 与 [L, R] 中所有 v 的乘积。 

因此，问题归结为对所有值区间 [L, R] 求和，即区间内频率的乘积。 关键的难点是有O(V^2)个区间，太大了。 

节省的观察结果是，我们可以通过固定正确的端点来增量计算这个总和。 让 dp[r] 表示以值 r 结束的所有有效区间的总贡献。 任何这样的区间要么是单值区间 [r, r]，要么是某个以 r - 1 结尾的区间的扩展。如果我们已经知道以 r - 1 结尾的区间的乘积总和，那么将它们全部扩展 r 会将它们的贡献乘以 cnt[r]。 

这给出了将二次结构折叠成单遍的线性递归。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子序列的暴力破解 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了|
 | DP | 值区间 O(M)| O(M)| 已接受 |

 这里M是数组中的最大值。 

## 算法演练

1. 计算每个值在数组中出现的次数。 将其存储在数组 cnt 中，其中 cnt[x] 是值 x 的频率。 这将问题从索引空间转换为值空间。 
2. 将 dp[r] 定义为有效 kompakt 子序列的总数，这些子序列的值完全位于以值 r 结束的区间内，并且其最大值恰好为 r。 每个这样的子序列对应于在值空间中选择一个非空区间 [l, r]，然后在该区间中为每个值选择一次出现。 
3. 使用递推式 dp[r] = cnt[r] + cnt[r] * dp[r - 1] 计算 dp[r]。 项 cnt[r] 对应于区间 [r, r]，我们在其中选取任意出现的 r。 项 cnt[r] * dp[r - 1] 对应于通过包含值 r 来扩展以 r - 1 结尾的每个有效区间，该值将所有先前的选择乘以 cnt[r]。 
4. 通过对所有 r 求和 dp[r] 来累加答案。 
5. 由于产品可以快速增长，因此对所有运算进行模 1e9 + 7。 

整个计算是频率预处理后对值域的单次传递。 

### 为什么它有效

 每个有效子序列都由其值集唯一标识，该值集必须形成连续的区间 [l, r]。 对于固定间隔，不同值之间的选择是独立的，因为为值 v 选择索引不会影响其他值的选择。 这种独立性将计数变成了频率的乘积。 递归避免了重复计算，因为每个区间都由其右端点 r 生成一次，并且其左边界隐式表示在 dp[r - 1] 内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def main():
    n = int(input())
    arr = list(map(int, input().split()))

    if n == 0:
        print(0)
        return

    maxv = max(arr)
    cnt = [0] * (maxv + 1)

    for x in arr:
        cnt[x] += 1

    dp_prev = 0
    ans = 0

    for r in range(1, maxv + 1):
        if cnt[r] == 0:
            dp_r = 0
        else:
            dp_r = cnt[r] * (1 + dp_prev) % MOD

        ans = (ans + dp_r) % MOD
        dp_prev = dp_r

    print(ans)

if __name__ == "__main__":
    main()
```该实现将 DP 压缩为两个变量而不是完整数组。`dp_prev`存储以前一个值结束的间隔的总贡献。 当当前值频率为零时，任何间隔都不能在此结束，因此 DP 重置为零。 

一个常见的错误是忘记缺失值会破坏跨越它们的所有间隔。 这是自然处理的，因为`cnt[r] = 0`力量`dp_r = 0`，这可以防止任何跨越该值的区间产生影响。 

## 工作示例

 考虑数组`[1, 2, 2, 3]`。 

我们计算频率：cnt[1]=1，cnt[2]=2，cnt[3]=1。 

| r | 碳纳米管[r] | dp_prev | dp[r] 计算 | dp[r] | dp[r] |
 | --- | --- | --- | --- | --- |
 | 1 | 1 | 0 | 1 * (1 + 0) | 1 * (1 + 0) | 1 |
 | 2 | 2 | 1 | 2 * (1 + 1) | 2 * (1 + 1) | 4 |
 | 3 | 1 | 4 | 1 * (1 + 4) | 1 * (1 + 4) | 5 |

 最终答案是1+4+5=10。 

该跟踪显示了以每个值结束的间隔如何累积每个值独立选择索引的所有方式。 

现在考虑`[1, 3, 3]`。 

频率：cnt[1]=1，cnt[2]=0，cnt[3]=2。 

| r | 碳纳米管[r] | dp_prev | dp[r] | dp[r] |
 | ---| ---| ---| ---|
 | 1 | 1 | 0 | 1 |
 | 2 | 0 | 1 | 0 |
 | 3 | 2 | 0 | 2 |

 答案是3。 

值 2 处的零打破了原本跨越它的所有间隔，确保没有无效的紧凑间隔起作用。

 ## 边缘情况

 一个关键的边缘情况是数组的值空间中包含间隙。 例如，`[1, 2, 4]`没有跨越 1 到 4 的有效 kompakt 子序列，因为缺少值 3。 在 DP 期间，在 r = 3 时，我们得到 cnt[3] = 0，这会强制 dp[3] = 0。这会重置所有会超过 3 的间隔，因此以 4 结尾的间隔不能包含 1 或 2，这与定义匹配。 

另一个边缘情况是大量重复，例如`[2, 2, 2, 2]`。 每个区间只是 [2, 2]，但是有很多方法可以选择单个索引。 r = 2 处的 DP 变为 cnt[2] * (1 + dp[1]) = 4，它正确地计算了所有单元素子序列并且没有更大的间隔。
