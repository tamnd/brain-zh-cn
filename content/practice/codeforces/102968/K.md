 ## 解决方案
 ## 问题理解

 我们有一个非常大的方形网格$N \times N$， 在哪里$N$始终是二的幂。 只有次对角线上方的单元格是相关的，这意味着所有单元格$(X, Y)$这样$X + Y \le N$。 这形成了一个三角形区域。 

我们必须使用方形图块完全覆盖该区域，其中每个图块的边长也是 2 的幂。 瓷砖必须与网格对齐放置，并且它们不得重叠或延伸到允许的区域之外。 每个有效的单元都必须被覆盖，并且在所有有效的覆盖中，我们希望最小化所使用的图块数量。 

在构建这样的最佳平铺（不一定是唯一的）之后，我们会被要求进行多个查询。 每个查询给出一个单元格$(X, Y)$，并且我们必须以某种最佳结构输出覆盖该单元的瓷砖的边长。 如果存在多个最佳结构并且它们为该单元分配不同的图块大小，我们必须输出$-1$。 

这些限制促使我们寻求一种避免任何显式构造的解决方案。$N$可以大到$2 \cdot 10^{18}$，所以即使$O(N)$或者$O(N \log N)$每个查询都是不可能的。 相反，我们必须依赖三角形区域的结构分解。 

一个关键的微妙边缘情况是当查询的单元恰好位于两个同样最优的分解之间的边界上时。 例如，在小情况下，例如$N = 4$，不同的递归分割可以在对称位置产生不同的图块分配，并且答案必须反映模糊性而不是致力于一个图块分配。 

## 方法

 如果我们尝试直接解决这个问题，我们就不得不考虑将尽可能大的正方形瓷砖放入三角形区域中。 一个自然的暴力想法是模拟构造：在每一步中，取完全适合剩余未覆盖区域的最大的二次幂正方形，放置它，删除它的区域，然后重复，直到所有东西都被填满。 这种贪婪打包让人想起四叉树分解。 

然而，这立即变得不可行。 操作的数量取决于需要多少个图块，在最坏的情况下，该区域会退化为许多较小的未覆盖片段。 即使每个放置都是有效的，在以下情况下迭代所有单元甚至所有图块也是不可能的：$N$取决于$10^{18}$。 

关键的观察结果是，该区域具有非常严格的自相似结构，因为网格大小和图块大小都是 2 的幂。 任何最佳的平铺都必须遵循这种结构：唯一有意义的分割发生在中点，并且三角形递归地分解为更小的相似三角形和矩形。 这与分治覆盖和四叉树结构中出现的机制相同。 

我们没有构建平铺，而是将问题重新解释为三角形区域的递归划分。 在每个尺度上$2^k$，该区域要么完全属于一个大的方形图块，要么被分成恒定数量的大小子区域$2^{k-1}$。 这使得可以通过降低高度为的递归树来确定覆盖点的图块$O(\log N)$。 

当单元位于可以通过多种对称递归方式解析的边界上时，就会出现模糊条件。 在这些情况下，存在多个最佳分解，因此答案不是唯一确定的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力平铺模拟 | 实践中呈指数增长 | O(N²) 隐式 | 太慢了|
 | 递归二次幂分解 |$O(\log N)$每个查询 |$O(1)$| 已接受 |

 ## 算法演练

 我们将三角形区域解释为递归定义的对象。 

任何规模$N = 2^k$，我们将正方形分成四个大小象限$2^{k-1}$。 次对角线以可预测的方式穿过这些象限，因此允许的区域分解为以下组合：

 一个象限中的较小的完整三角形，以及其他象限中的矩形或三角形片段。 

最小化正方形瓷砖数量的结构总是更喜欢使用与这些象限边界对齐的尽可能大的正方形。 这会导致除边界线外的确定性分解模式。 

### 步骤

 1.如果$N = 1$，该区域包含单个单元格且唯一的图块大小为 1。返回 1。 
2. 求两个尺度的最大幂$S = 2^k = N$并考虑查询点是否位于可以大规模解析的区域$S/2$。 我们对位置进行分类$(X, Y)$相对于中点。 
3. 如果两者都$X$和$Y$位于左上象限（即$X \le S/2$和$Y \le S/2$和$X + Y \le S/2$），那么问题就简化为较小三角形中的相同查询$S/2$。 我们递归地继续。 
4. 如果该点位于其他象限之一，则它对应于反射三角形或在此比例下被单个大图块完全覆盖的矩形区域。 在这种情况下，我们确定当前图块大小$S/2$就是答案。 
5. 如果该点恰好位于分解边界上，这意味着它满足对称条件，可以属于两个等效的递归分支，我们返回$-1$因为存在多个最佳平铺，可以为该单元格分配不同的平铺大小。 
6. 重复此操作，直到我们以特定比例解析单元格或达到基本情况。 

### 为什么它有效

 在每个级别，允许区域的分解都是由二次方几何强制的。 任何最佳的平铺都必须与这些中点对齐，因为放置较大或未对齐的正方形会违反边界或增加平铺数量。 这会产生独特的递归结构，但在对称边界处除外，其中两个子问题的成本无法区分。 这些边界单元完全对应于不明确的查询，因为递归不强制执行唯一的分支选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(n, x, y):
    if n == 1:
        return 1

    ans = None
    cur = n

    while cur > 1:
        half = cur // 2

        in_top_left = (x <= half and y <= half and x + y <= half)

        if in_top_left:
            cur = half
            continue

        # outside main triangle quadrant structure
        # determine if on boundary causing ambiguity
        # boundary occurs when symmetric wrt diagonal split
        if x <= half and y > half:
            return cur // 2
        if x > half and y <= half:
            return cur // 2

        # bottom-right region relative to current block
        # this region is fully covered at this level
        # but may be ambiguous if exactly on split line
        if x + y == cur + 1:
            return -1

        return cur // 2

    return 1

def main():
    q = int(input())
    out = []
    for _ in range(q):
        n, x, y = map(int, input().split())
        out.append(str(solve_one(n, x, y)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现反复将当前区域大小减半。 变量`cur`表示包含查询点的活动子问题的当前边长。 如果该点完全位于左上角的递归三角形内，我们将继续下降。 

如果它进入另一个象限，我们立即识别出以这个比例覆盖它的图块的尺寸一定是`cur // 2`。 这对应于递归停止扩展的第一级。 

条件`x + y == cur + 1`用作歧义检测器。 它捕获了次对角线与递归分割对齐的精确边界，这意味着可以进行两个对称平铺。 

该解决方案避免了任何显式的网格构造，并且仅使用对数下降。

 ## 边缘情况

 一种重要的边缘情况是查询的单元格恰好位于递归分割边界上。 例如，当$X + Y = N + 1$，该单元格恰好位于当前子问题的次对角线上。 在这种情况下，两个对称分解是有效的，因为可以在该级别以两种等效方式对区域进行分区，从而导致不同的图块分配。 

对于这种情况，算法会触发歧义条件并返回$-1$而不是承诺瓷砖尺寸。 这反映了这样一个事实：没有任何一个最佳平铺能够唯一地确定单元的覆盖范围。 

当查询位于左上角递归三角形深处时，会发生另一种微妙的情况。 该算法反复减少$N$不会触发象限退出条件。 这是安全的，因为每次递归都会严格将问题大小减少一半，并且终止条件$N = 1$保证最终的解决没有歧义。
