 ## 解决方案
 ## 问题理解

 我们有一个城市集合，每个城市都有一个代表其人口的整数值。 我们被允许在两座城市之间修建道路。 如果我们将两个城市的人口连接起来`a`和`b`，这条路的“好处”是`gcd(a, b)`。 

结构要求是，修建道路后，每个城市都必须能够从其他城市到达，因此所选择的道路必须在所有节点上形成连通图。 由于目标是在保持连通性的同时使用尽可能少的道路，因此任何有效的解决方案都必须准确使用`N - 1`道路，这意味着我们正在有效地构建一棵生成树。 在所有生成树中，我们想要一棵能够最大化边权重之和的生成树，其中每个边权重是两个端点的 gcd。 

因此，该任务是完整图上的最大生成树问题，其中边权重`i`和`j`是`gcd(xi, xj)`。 问题是该图是隐式且密集的，因此枚举所有边对于`N = 10^5`。 

这些限制意味着计算预算紧张。 一个天真的`O(N^2 log N)`甚至`O(N^2)`方法将涉及最多`10^10`候选边缘，这是完全遥不可及的。 即使存储所有边缘也是不可能的。 这立即迫使我们避免显式的成对比较，而是利用 gcd 的数论结构。 

如果假设贪婪的本地连接，例如“将每个数字连接到其最佳 gcd 伙伴”，则会出现微妙的失败情况。 例如，使用值`[6, 10, 15]`，最好的局部 gcd 选择可能会产生误导：连接 6-10 给出 2、6-15 给出 3、10-15 给出 5。在不考虑全局连接约束的情况下贪婪地选取边可能会产生循环或错过更好的全局结构，因为高权重边可能会浪费在组件内部而不是链接组件。 

另一种失败模式来自于仅考虑素数：gcd 结构不是独立地考虑素因数，而是考虑跨多个节点的共享除数，并且这些重叠在整个集合中相互作用。 

## 方法

 蛮力的想法很简单。 我们计算所有成对的 gcd 值，按权重对边进行排序，并运行 Kruskal 算法以获得最大生成树。 这是正确的，因为 MST 属性保证了任何加权图的最优性。 然而，该图有`N(N-1)/2`边，大约是`5 * 10^9`什么时候`N = 10^5`。 即使生成边缘也已经太慢了，仅进行 gcd 计算将是灾难性的。 

关键的观察结果是边权重是高度结构化的。 gcd值大`d`只能出现在都是 的倍数的数字之间`d`。 我们不考虑边缘，而是反转视角：对于每个可能的值`d`，我们将所有可被整除的数字分组`d`。 任意具有权重的生成树边`d`必须来自该群体内部。 

现在我们可以按照降序思考`d`。 我们尝试使用权重边缘连接组件`d`在转向较小的值之前。 面临的挑战是如何有效地知道我们可以使用可被整除的数字合并多少个组件`d`。 这成为除数上的类似筛子的过程，类似于反向除数聚合。 

我们不是迭代对，而是迭代值`d`从`1`到`maxA`，收集所有倍数，并将它们合并。 当我们处理一个值时`d`，我们将可被整除的数字的代表元素连接起来`d`。 每个成功的工会都做出贡献`d`答案，因为我们在权重级别上有效地添加了一个生成森林`d`。 

这是可行的，因为 MST 中的每条边都可以被记入可以连接其端点的最高除数，并且我们确保首先考虑更高的 gcd 边。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（完整图上的克鲁斯卡尔）| O(N² log N) | O(N² log N) | O(N²) | 太慢了 |
 | 基于除数的 DSU（筛联合体）| O(A log A + N α(N)) | O(A log A + N α(N)) | O(A + N) | 已接受 |

 ## 算法演练

 我们在城市上维持一个不相交的联合结构。 我们还准备了桶，以便对于每个值`v`，我们知道哪些指数有人口`v`。 

我们从大到小处理候选 gcd 值。 

1. 按值对索引进行分组，以便我们可以快速检索人口可被给定整除的所有城市`d`。 这避免了重复扫描所有节点。 
2. 迭代`d`从`max_value`下降到`1`。 对于每个`d`, 收集所有是 的倍数的数字`d`通过扫描`d, 2d, 3d, ...`。 
3. 对于每个这样的倍数值，迭代具有该值的所有索引并尝试将它们与该除数的第一个代表结合起来。 每个成功的联合对应于添加一条带权重的边`d`。 
4. 每次联合成功时，添加`d`总答案，因为这条边是生成树的一部分，以尽可能高的权重级别贪婪地构建。 
5. 继续，直到处理完所有除数级别。 DSU 确保我们永远不会创建循环，因此我们总是得到准确的结果`N - 1`当图连接时联合成功。 

### 为什么它有效

 该算法有效地模拟了 Kruskal 的过程，而无需枚举边缘。 每个潜在的边缘`(i, j)`第一次遇到最大的`d`使得两者`xi`和`xj`可以被整除`d`，这正是`gcd(xi, xj)`如果我们向下处理除数。 DSU 确保一旦两个组件以较高权重连接，它们就永远不会以较低权重重新连接，从而保留最大生成树属性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    maxa = max(a)

    pos = [[] for _ in range(maxa + 1)]
    for i, v in enumerate(a):
        pos[v].append(i)

    parent = list(range(n))
    size = [1] * n

    def find(x):
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    def union(a, b):
        ra, rb = find(a), find(b)
        if ra == rb:
            return False
        if size[ra] < size[rb]:
            ra, rb = rb, ra
        parent[rb] = ra
        size[ra] += size[rb]
        return True

    ans = 0

    for d in range(maxa, 0, -1):
        first = -1
        for m in range(d, maxa + 1, d):
            for idx in pos[m]:
                if first == -1:
                    first = idx
                else:
                    if union(first, idx):
                        ans += d

    print(ans)

if __name__ == "__main__":
    solve()
```该代码在城市上构建 DSU，并在除数值上使用反向筛选。 这`pos`数组按精确值对索引进行分组，允许在迭代多个值时快速访问`d`。 

这`find`和`union`函数按大小实现路径压缩和联合，确保每个操作的摊余复杂度接近恒定。 键循环迭代所有除数`d`从大到小，对于每个除数，它扫描所有倍数并联合其相应的索引。 

变量`first`至关重要，因为它避免了在共享除数的所有节点之间建立完整的派系。 相反，我们只将它们连接成树结构，这足以跨越连接。 

## 工作示例

 考虑输入：```
N = 4
values = [6, 10, 15, 30]
```我们跟踪每个除数级别的联合。 

在`d = 30`，仅存在索引 3，不会发生并集。 

在`d = 15`，处理索引 2 和 3。 

| d | 所见指数 | 第一 | 工会行动| 添加 |
 | ---| ---| ---| ---| ---|
 | 15 | 15 15、30 | 2 | (2,3) | 15 | 15

 在`d = 10`，索引 1 和 3 存在，但 3 已连接。 

| d | 所见指数 | 第一 | 工会行动| 添加 |
 | ---| ---| ---| ---| ---|
 | 10 | 10 10、30 | 1 | (1, 根) 好 | 10 | 10

 在`d = 6`，索引 0 和 3 存在。 

| d | 所见指数 | 第一 | 工会行动| 添加 |
 | ---| ---| ---| ---| ---|
 | 6 | 6、30 | 0 | （0，根）好的 | 6 |

 在`d = 5`，没有出现有用的联合。 

此跟踪显示了如何优先考虑较高的 gcd 边缘，以及 DSU 如何在仍确保连接的同时防止冗余连接。 

第二个例子：```
N = 3
values = [2, 4, 8]
```在`d = 8`，仅节点 2。 

在`d = 4`，节点 1 和 2 与权重 4 连接。 

在`d = 2`，节点 0 连接到权重为 2 的现有组件。 

这证明了通过减少除数来形成链，这与最佳生成树相匹配。

 ## 边缘情况

 单个城市的情况虽然微不足道，但如果 DSU 逻辑假设至少有一个联合，则很容易处理不当。 用于输入`n = 1`，该算法不执行除数并并正确返回`0`，因为不存在道路。 

例如，当所有值都相同时`[6, 6, 6, 6]`，每对都有gcd 6。算法处理`d = 6`首先，使用以下方法将所有节点连接成一棵树：`n - 1`工会，每个贡献 6，产生`18`。 下除数什么也不做，因为所有节点都已经统一，这表明 DSU 正确地防止了过度计数。 

对于稀疏共享除数，例如`[2, 3, 5, 7]`，在更高的位置没有出现并集`d > 1`，并且在`d = 1`该算法将权重为 1 的组件连接起来，直到形成生成树。 这证实了即使 gcd 结构没有提供更强的边缘，也可以正确处理后备连接。
