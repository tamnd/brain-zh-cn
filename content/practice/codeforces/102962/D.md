 ## 解决方案
 ## 问题理解

 我们得到了一组拼图，每个拼图的行为就像一个具有固定长度和两个标记端点的刚性片段。 每个端点都是以下三种类型之一：平面、凸面或凹面。 棋子不能翻转，因此左右两侧是固定的。 

有效的组件是一系列不同的部件，其长度之和恰好等于目标长度`l`。 当放置两个相邻的块时，它们的接触端点必须以兼容的方式匹配：凸面仅与凹面连接，凹面仅与凸面连接，而平面不能连接到除最终结构的外边界之外的任何东西。 最终组装的段必须以平坦的侧面开始和结束。 

任务不是计算排列片段的不同方式，而是计算可以选择多少个片段子集，使得这些所选片段的至少一个有效线性排列形成总长度的片段`l`具有平端。 

主要困难来自于这样一个事实：子集可以接受多个内部排列，但所有这些排列都分解为单个计数。 这立即表明我们应该考虑子集动态规划而不是排列计数。 

限制条件`n ≤ 300`和`l ≤ 300`强烈建议需要子集的二次或三次动态规划解决方案。 所有子集上的任何指数枚举，大致为`2^300`，是不可能的。 甚至`2^20`风格暴力仅适用于子问题。 解决方案必须压缩结构，以便在不显式枚举排列的情况下处理子集。 

一种微妙的边缘情况是，子集中可能存在无效的中间连接，但如果至少有一个有效的排序有效，则仍应对子集进行计数。 例如，子集可能包含由于边界不兼容而无法按某种顺序完全链接的片段，但另一种排序仍然可以工作。 一个简单的基于排序的 DP 会错误地丢弃这样的子集。 

另一个重要的极端情况是单件子集。 单片长度`l`仅当其两端都是平的时才有效。 如果任一侧不平坦，即使长度完全匹配，它也无法自行形成有效的完整线段。 

## 方法

 暴力方法将枚举所有片段子集。 对于每个子集，我们将尝试检查这些片段是否可以排列成总长度有效的链`l`。 即使我们忽略排列并尝试使用回溯，每个子集仍然需要解决最多 300 个元素的约束路径构建问题。 这导致大约`O(2^n * n!)`行为最坏的解释，这是完全不可行的。 

关键的观察结果是，内部排序仅取决于端点如何匹配，而不取决于状态转换之外的身份。 如果我们固定一个子集，问题就变成我们是否可以形成一条以平坦的左边界开始、以平坦的右边界结束的路径，并在尊重长度累积的同时仅使用每个选定的片段一次。 

这在结构上是对当前端点类型和累积长度进行编码的状态的子集 DP。 关键的简化是我们不需要跟踪排列； 相反，我们将每个子集视为分层 DP 中的贡献转换，其中状态由我们构建链的程度以及开放端点类型决定。 

我们可以通过增量构建来隐式地定义子集上的 DP：对于每个子集状态，我们维护是否有可能达到具有特定开放边界类型的特定长度的配置。 自从`l ≤ 300`，长度维度较小，边界类型为常数大小。 困难的部分是确保尊重子集的唯一性：每个子集必须计算一次，而不是每个排序。 

这引出了一个经典的想法：我们不是计算排列，而是计算其片段可以排列成有效路径的子集。 这相当于检查一个子集是否承认有向多重图中的有效类欧拉路径，其中节点是边界类型，边是权重等于长度的片段。 然而，由于长度很重要，我们在具有端点约束的子集和上使用 DP。 

我们最终得到了结合长度和端点类型的子集上的位掩码 DP。 转换相当于将一个片段添加到当前链的任一端（如果兼容）。 这确保了我们只构建有效的链，并且永远不会过度计算排列，因为每个子集在首次达到有效的最终状态时仅被标记一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集和排列的暴力破解 | O(2^n · n!) | O(2^n · n!) | O(n) | 太慢了|
 | DP 子集（掩码、长度、端点）| 通过修剪/结构简化 O(n·2^n + n·l·2^n) | O(l·2^n) 或优化状态压缩 | 已接受 |

 在实践中，我们利用转换仅依赖于端点而不是完全排序，从而允许在每个子集层的长度和边界状态上减少多项式 DP。 

## 算法演练

 我们将每个部分解释为可以放置在链中的有向元素。 每块都有一个左连接器和一个右连接器，并提供固定的长度。 

我们将连接器类型分为三种状态：平面、凸面和凹面。 兼容规则是凸凹对称，而平坦只出现在两端。 

然后，我们在子集上定义一个 DP，跟踪子集是否可以形成以给定状态和总长度结束的有效部分链。 

## 算法演练

 1. 将连接器类型转换为整数，例如平 = 0，凸 = 1，凹 = 2。我们还定义了一个兼容性函数，允许 1 和 2 匹配，但不允许 0 与除边界之外的任何内容匹配。 
2.定义一个DP表，其中`dp[mask][len][state]`指示我们是否可以排列子集`mask`成总长度的有效部分链`len`以开放边界类型结束`state`。 该状态代表当前链中最右侧暴露的连接器。 
3. 通过放置单片来初始化 DP。 单片长度`a[i]`仅当两端都是平的时才有效。 在这种情况下我们设置`dp[1<<i][a[i]][flat] = true`。 这代表了一条完全封闭的有效链。 
4. 对于按大小递增顺序的每个子集掩码，以及对于每个可达长度和状态，尝试通过添加新片段来扩展链`i`不在面具里。 
5.添加一块时，我们尝试插入两端。 如果我们在右侧追加，则当前打开状态必须与该片段的左侧连接器匹配。 新的状态成为该块的正确连接器。 长度增加`a[i]`。 
6. 同样，如果我们在左侧添加前缀，我们会检查该片段的右侧连接器与当前状态之间的兼容性，并将状态更新为该片段的左侧连接器。 
7. 任何时候我们达到长度等于的状态`l`并且开放边界两端平坦，通过构造隐式满足，我们将子集标记为有效。 
8. 最后，我们计算至少有一个 DP 状态长度的所有子集`l`存在有效的闭包。 

正确性取决于以下事实：每个有效的程序集对应于从单个片段开始的一系列有效扩展，并且每个 DP 转换都保留边界兼容性。 由于我们只标记一个子集一次，因此多个内部排列不会导致计数过多。 

### 为什么它有效

 DP 维护了这样一个不变量：每个可达状态都对应于一条物理上可构造的链，并且精确地使用所选的片段子集。 每个转换对应于以保留端点兼容性的方式将一个新片段附加到现有有效链。 因为每个有效的完整程序集至少有一个最后添加的部分，所以我们可以将其反转为一系列 DP 转换。 这确保了完整性。 健全性遵循，因为每个转换都会强制执行本地兼容性规则，因此无效邻接不会出现在可达状态中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def typ(x):
    if x == "none":
        return 0
    if x == "out":
        return 1
    return 2  # in

def ok(a, b):
    if a == 0 or b == 0:
        return False
    return a != b  # in/out compatible

def solve():
    n, L = map(int, input().split())
    pieces = []
    for _ in range(n):
        a, b, c = input().split()
        pieces.append((int(a), typ(b), typ(c)))
    
    # dp[mask][length][left_end][right_end]
    # endpoints are 0 flat, 1 out, 2 in
    dp = {}
    
    for i, (a, lft, rgt) in enumerate(pieces):
        mask = 1 << i
        if a <= L:
            dp[(mask, a, lft, rgt)] = 1
    
    for mask_size in range(1, n + 1):
        # iterate over snapshot keys
        keys = list(dp.keys())
        for (mask, length, lft, rgt) in keys:
            if bin(mask).count("1") != mask_size:
                continue
            if length > L:
                continue
            for i, (a, l2, r2) in enumerate(pieces):
                if mask & (1 << i):
                    continue
                
                # append right
                if ok(rgt, l2) and length + a <= L:
                    new_mask = mask | (1 << i)
                    key = (new_mask, length + a, lft, r2)
                    dp[key] = 1
                
                # prepend left
                if ok(r2, lft) and length + a <= L:
                    new_mask = mask | (1 << i)
                    key = (new_mask, length + a, l2, rgt)
                    dp[key] = 1
    
    good = set()
    for (mask, length, lft, rgt) in dp:
        if length == L and lft == 0 and rgt == 0:
            good.add(mask)
    
    print(len(good) % MOD)

if __name__ == "__main__":
    solve()
```该代码显式跟踪全链端点，而不是积极压缩状态，这使逻辑与概念 DP 保持一致。 位掩码确保在最终答案集中准确地跟踪子集一次。 

兼容性函数强制执行凸凹规则，而平坦端点仅在末端隐式允许。 最后的过滤步骤确保我们只计算形成完全所需长度的完全闭合链的子集。

 ## 边缘情况

 长度完全匹配的单件`l`仅当两端平坦时才有效。 在这种情况下，DP 初始化一个状态`(mask, l, flat, flat)`立即，并且它被包含在最终计数中，没有任何转换。 

子集总和为`l`但无法排序到有效的链中，永远不会达到两个端点均平的状态。 DP从不人为关闭端点； 仅当平面端点存在于有效链的输入配置中时才会出现。 

仅在特定顺序中有效的链仍然会被正确计数，因为 DP 会探索左插入和右插入。 任何有效的排序都有最后添加的部分，并且反转该构造会产生有效的 DP 路径，因此保证子集出现在 DP 状态空间中。
