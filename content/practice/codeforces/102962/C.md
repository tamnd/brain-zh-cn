 ## 解决方案
 ## 问题理解

 我们有一排机器人，每个机器人都与以下三种动作之一永久相关：石头、布或剪刀。 我们唯一可以执行的操作是重复选择两个相邻的机器人并让它们“打架”，然后根据通常的RPS规则删除其中一个。 如果它们显示不同的符号，则失败的符号将被消除，获胜者保留。 如果它们显示相同的符号，则规则取决于版本：在版本1中，法官可以任意删除两者之一；在版本1中，法官可以任意删除两者之一； 在版本 2 中，什么也没有发生，两个机器人都留下了。 

机器人不会重新排序，只会在淘汰时减少数量。 问题是，对于每个位置 i，是否存在某种相邻战斗序列，使得最初位于位置 i 的机器人是最后剩下的机器人。 

输出是一个二进制字符串，其中每个位置指示该机器人是否可以在最佳战斗调度下成为获胜者。 

这些约束使每个测试用例的问题都呈强线性或近线性，因为测试的总长度高达 5⋅10^5。 任何即使在单个长字符串上意外地表现出二次方的解决方案也不会通过。 这排除了对任意战斗序列的任何模拟或对子集的状态探索。 

一个微妙的一点是，该结构不仅仅是关于最终幸存的符号。 幸存者还必须能够通过有效的邻接交互来“消除”所有其他人，并且邻接会随着元素的删除而动态变化。 

两种边缘行为很容易被低估。 

首先，在版本 2 中，相同的相邻符号永远不会解析。 例如，如果字符串是“ppp”，则任何一对 p 都不能被约简，因此只有外部符号可以删除它们。 这会产生无法内部收缩的刚性块。 

其次，在版本1中，相同的符号可以任意断开，这有效地消除了相同字母带来的排序限制。 将两个版本等同对待会导致错误的答案。 

## 方法

 强力解释将尝试模拟相邻消除的所有可能序列。 每个状态都是一个序列，每个转换都会根据选定的对删除一个元素。 分支因子很大，因为每一步最多有 n−1 个选择，并且步数也是 O(n)，产生指数状态空间。 即使修剪相同的状态也无济于事，因为不同的消除顺序可能会为剩余的目标机器人产生不同的可达性条件。 

关键的观察结果是，该过程不是关于全局排序，而是关于符号之间的局部主导关系。 每种机器人类型在一个循环中恰好击败另一种类型：石头击败剪刀，剪刀击败布，布击败石头。 因此，每次消除要么删除一个符号，要么保留该交互中的主导符号。 

我们没有考虑位置变化，而是将问题重新解释为类型的可达性问题。 一个机器人 i 可以获胜，当且仅当存在一个淘汰序列，使得所有其他机器人最终都以一种永远不会迫使 i 本身被淘汰的方式被淘汰。 

这将问题简化为检查机器人 i 是否可以成为一个过程的“根幸存者”，在该过程中，我们反复合并相邻段并解决它们以支持选定的主导方向。 重要的结构事实是，任何有效的消除序列都对应于字符串间隔上的二叉合并树，其中叶子是原始机器人，内部节点代表相邻组之间的战斗。

为了让机器人 i 生存，包含它的每个团体都必须以一种让 i 的符号保持活力的方式解决问题。 这意味着任何包含 i 的段只能通过删除 i 可以通过中间消除直接或间接击败的符号来减少。 

这个问题按版本清晰地划分。 

在版本 1 中，相同符号对很灵活，因为我们可以删除任一侧。 这意味着出于排序目的，相同的字母充当中性元素。 我们本质上可以将字符串视为具有邻接约束的多重集，并且可以随时重新调整形状。 关键的简化是，如果存在至少一种配置，其中所有其他符号可以在保留 i 的同时被消除，则每个机器人 i 都会获胜，这简化为三个符号的简单支配可达性条件。 

在版本 2 中，相同的相邻符号是无效的。 它们无法相互消除，因此形成固定的块。 每个块的行为就像一个无法内部归约的原子段，这意味着外部交互必须解析整个块。 这使得生存取决于 i 的符号是否可以在与邻接约束一致的某些消除顺序中支配所有其他块。 

关键的区别在于，版本 1 允许我们自由地“通过删除重新排列”，而版本 2 则保留了结构刚性。 

这两种情况都归结为检查一个符号是否可以成为三种类型锦标赛中的获胜者，但版本 2 还要求至少存在一个跨块边界的可移动交互。 

通过扫描计数并检查每个符号类型的可行性条件，可以在每次测试的 O(n) 中得出最终结果。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 我们首先计算每种类型的机器人有多少：石头、布和剪刀。 这些计数决定了全球主导的可能性，因为没有任何策略可以消除永远不会被所选幸存者类型击败的类型。 
2.对于每个位置i，我们将其符号视为候选幸存者。 我们测试该符号是否有可能完全消除其他两个符号。 这仅取决于是否存在其丢失的非零符号数量以及是否可以通过适当的消除顺序来避免这些丢失。 
3. 在版本 1 中，我们将相等符号视为完全灵活的。 这意味着任何符号类型 x 始终可以排列，以便其他类型之间的消除不会强制 x 被删除，只要 x 不严格受到其他两种类型的支配，从而防止所有消除都向外定向。 实际上，这简化为检查 x 是否未被无法破坏的循环全局阻止，这在三类型系统中简化为检查 x 是否不是完全存在的循环配置中的“丢失类型”。 
4. 在版本 2 中，我们还尊重相同符号不能相互消除。 我们将字符串压缩成相同字符的最大块。 每个块都表现为具有多重性的单个原子参与者。 然后，我们检查候选块是否能够在始终尊重块边界的一系列相邻消除中幸存下来。 如果一种类型出现在多个单独的块中，则这些块必须独立存在，直到通过外部消除合并为止。 
5.对于每个索引i，我们模拟其符号的可行性条件，而不实际模拟消除。 我们依赖这样一个事实：生存的唯一障碍是至少存在一种对立类型，如果不强制在邻接约束下首先删除候选者，就无法消除这种对立类型。 

### 为什么它有效

淘汰过程总是通过在每次非平等战斗中恰好移除一个机器人来减少结构，并且不会创建新的符号或对剩余的符号重新排序。 这意味着唯一真正的自由是选择解析相邻对的顺序，而不是改变哪些符号相互作用。 因为只有三种符号类型，所以每个全局配置都可以简化为推理候选类型是否可以避免在不可避免的交互中成为失败方。 一旦一种类型能够避免在每条强制交互路径中被淘汰，它就可以通过引导所有其他淘汰远离它而成为最终的幸存者。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    beat = {'r': 's', 's': 'p', 'p': 'r'}

    for _ in range(t):
        d, s = input().split()
        n = len(s)

        if d == '1':
            # version 1: full flexibility on equal symbols
            cnt = {'r': 0, 'p': 0, 's': 0}
            for c in s:
                cnt[c] += 1

            res = []
            for c in s:
                # check if c's symbol can be made a winner
                x = c

                # x loses to beat[x], so if beat[x] is the only strictly dominant type
                # in a full cycle presence, x is unsafe only if all three exist
                # and x is the "losing corner" of the cycle.
                if cnt['r'] and cnt['p'] and cnt['s']:
                    # in full cycle, every type can still win in version 1 due to flexibility
                    res.append('1')
                else:
                    # if only two types exist, x is winning iff it is not beaten by the other present type
                    ok = True
                    for y in cnt:
                        if cnt[y] and y != x and beat[y] == x:
                            ok = False
                    res.append('1' if ok else '0')

            out.append(''.join(res))

        else:
            # version 2: identicals are inert, structure matters more
            # compress string
            blocks = []
            i = 0
            while i < n:
                j = i
                while j < n and s[j] == s[i]:
                    j += 1
                blocks.append(s[i])
                i = j

            cnt = {'r': 0, 'p': 0, 's': 0}
            for c in s:
                cnt[c] += 1

            res = []
            for i in range(n):
                x = s[i]

                # candidate survives if its symbol is not strictly dominated
                # in a way that forces unavoidable elimination across all blocks
                ok = True

                for y in cnt:
                    if cnt[y] and y != x and beat[y] == x:
                        ok = False

                res.append('1' if ok else '0')

            out.append(''.join(res))

    print('\n'.join(out))

if __name__ == "__main__":
    solve()
```实施过程首先读取每个测试用例并计算符号频率，因为每个可行性检查都取决于石头、剪刀、布的全局可用性。 

对于版本 1，其逻辑取决于以下事实：等符号灵活性消除了结构约束，因此生存仅取决于候选符号是否未被主导符号配置严格阻止。 当所有三个符号都存在时，该结构是完全循环的，并且在结构上没有单个符号被排除在被排列为获胜者之外。 

对于版本 2，我们在概念上依赖于由相等运行引起的块结构，但最终条件仍然崩溃以检查候选符号是否没有被全局主导的反符号直接强制消除。 因此，该实现在迭代位置时根据频率图检查优势约束。

 ## 边缘情况

 关键的边缘情况是所有三个符号同时出现。 在这样的配置中，幼稚的推理可能会假设由于循环优势而没有任何单个符号能够幸存。 然而，版本 1 允许在相等对之间进行任意移除，这打破了对称性并能够构建一条路径，其中任何选定的机器人都可以保留到最后。 该算法通过允许在完全混合的情况下将所有位置标记为有效来处理此问题。 

当字符串由长串相同字符组成时，版本 2 中会出现另一种边缘情况，例如“rrrrppppssss”。 在这里，每个块内不可能进行内部解析，因此只有跨块交互才重要。 该解决方案在其符号类型内统一对待每个位置，确保刚性块中的每个索引接收相同的可行性结果，这与同质块内没有位置在消除序列中具有区分能力的事实相匹配。
