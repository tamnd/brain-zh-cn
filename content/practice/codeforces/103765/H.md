 ## 解决方案
 ## 问题理解

 我们正在研究一个由整数格点组成的网格，形成一个$(N+1)\times(N+1)$正方形。 其中一些网格点被标记为补给站。 每个仓库被选择的可能性相同。 独立地，Tanya 的起始位置也是一个均匀随机的网格点。 做出这两个选择后，她使用曼哈顿距离前往所选的站点，这意味着距离是绝对水平和垂直坐标差的总和。 

任务是计算随机网格点和随机供应点之间的预期曼哈顿距离，其中期望取自由一个任意网格点和一个给定供应点组成的所有对，两侧的概率一致。 

输出不是期望的原始理性值。 相反，我们必须计算该期望在模 998244353 下的模表示。具体来说，如果期望简化为分数$x/y$，我们必须输出模逆形式$y$次$x$，这相当于$x \cdot y^{-1} \bmod 998244353$。 

约束条件非常大，无论是$N$和$M$高达两百万。 迭代每个供应点的所有网格点的简单方法将需要以下顺序：$O(N^2 M)$，这是完全不可行的。 即使任何明确触及所有网格点的事情也是不可能的。 该解决方案必须避免枚举网格，而是利用曼哈顿距离的可分离性和坐标贡献的对称性。 

当只有一个供应点时，就会出现一种微妙的边缘情况$(0,0)$和$N=2$。 该网格包含 9 个点，平均距离不是根据供应点之间的距离计算的，而是根据所有网格点到该单个仓库的距离计算的。 一个常见的错误是意外地对电源到电源的距离而不是电网到电源的距离进行平均，这会错误地得出零。 正确的解释始终包括所有网格点作为起始位置。 

另一种边缘情况是当所有供应点都位于一条线上时，例如$y=0$。 简单的分解可能会独立处理 x 和 y，而没有根据点数进行适当的加权，从而导致不正确的缩放。 独立性是有效的，但只有在正确求和整个网格的贡献之后才有效。 

## 方法

 蛮力解释很简单：对于每个供应点，计算从该点到每个网格点的曼哈顿距离总和，然后对所有网格点和所有供应点进行平均。 这直接符合定义。 然而，计算从单个点到所有点的距离$(N+1)^2$网格点成本$O(N^2)$，并这样做是为了$M$供应点导致$O(MN^2)$，这远远超出了可行的限度。 

关键的结构观察是曼哈顿距离分裂成独立的 x 和 y 贡献。 对于固定供应点$(x_i, y_i)$，所有网格点的总和分解为所有 x 坐标的总和加上所有 y 坐标的总和。 具体来说，我们可以将总贡献重写为：$$\sum_{x=0}^{N}\sum_{y=0}^{N} (|x-x_i| + |y-y_i|)$$分为：$$\sum_{x=0}^{N}(N+1)\cdot |x-x_i| + \sum_{y=0}^{N}(N+1)\cdot |y-y_i|$$所以每个维度都可以独立处理，乘以$(N+1)$，然后合并。 

对于每个供应坐标值，这将问题简化为计算$a$, 总和$\sum_{t=0}^N |t-a|$。 这可以使用前缀算术在常数时间内计算出来：

 左边的值$a$贡献一个三角和，并值权$a$贡献另一个。 

一旦我们可以计算出每个供应点$O(1)$，完整的解决方案运行在$O(M)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(MN^2)$|$O(1)$| 太慢了 |
 | 最佳 |$O(M)$|$O(1)$| 已接受 |

 ## 算法演练

 我们首先对常数进行预处理$(N+1)$因为每个网格坐标计数都取决于它。 主要任务是评估统一整数范围内的绝对差之和。 

对于每个供应点$(x_i, y_i)$，我们计算两个独立的贡献，一个用于 x，一个用于 y，因为曼哈顿距离在维度上是相加的，并且网格是笛卡尔积。 

1. 对于坐标值$a$, 分割范围$[0, N]$分为两部分：左侧$[0, a]$和右侧$[a, N]$。 这种分离是必要的，因为绝对值改变了定义$a$。 
2. 计算左侧贡献的总和$a - t$为所有人$t \le a$。 这是一个算术级数，其闭合形式为$\frac{a(a+1)}{2}$。 这表示距左侧或等于以下点的总距离$a$。 
3. 计算正确贡献的总和$t - a$为所有人$t \ge a$。 这也是一个算术级数并且等于$\frac{(N-a)(N-a+1)}{2}$。 
4. 将两个部分相加以获得坐标的总一维贡献$a$。 
5. 将此一维结果乘以$(N+1)$因为对于每个 x 位置有$(N+1)$独立的 y 选择，反之亦然。 
6. 对每个供应点的 x 和 y 坐标重复此操作，并累积所有供应点的结果。 
7.除以网格点总数$(N+1)^2$乘以 998244353 下的模逆。 

### 为什么它有效

 正确性来自于曼哈顿距离求和的线性性和可分性。 每个网格点恰好贡献一个 x 项和一个 y 项，并且这些贡献不相互作用。 网格结构确保每个坐标值的均匀重数，因此一维分解准确地捕获完整的二维和。 除了均匀平均之外，不使用任何近似或概率论证； 代数分解保留了精确的等式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    N, M = map(int, input().split())
    total = 0
    size = N + 1

    for _ in range(M):
        x, y = map(int, input().split())

        lx = x * (x + 1) // 2
        rx = (N - x) * (N - x + 1) // 2
        ly = y * (y + 1) // 2
        ry = (N - y) * (N - y + 1) // 2

        sx = lx + rx
        sy = ly + ry

        # each coordinate repeats (N+1) times across the other axis
        total += (sx + sy) * size

    total %= MOD

    denom = (size * size) % MOD
    ans = total * modinv(denom) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```代码首先读取网格大小和供电点数量。 对于每个供应点，它使用算术级数公式独立计算 x 和 y 距离的一维总和。 将它们相结合并乘以$N+1$考虑正交轴上的完整网格复制。 累积总和表示归一化之前所有网格点和供给点对的总距离。 

最后，除以$(N+1)^2$使用模逆来执行，因为期望是大素数模下的有理数。 请注意仅在累加后应用模数以避免精度问题。 

## 工作示例

 考虑样本$N=2$和一个单一供应点$(0,0)$。 

对于这一点，x 贡献为$\sum_{t=0}^2 |t-0| = 0 + 1 + 2 = 3$。 y 也同样如此。 每个都乘以$N+1=3$，因此总贡献变为$3 \cdot 3 + 3 \cdot 3 = 18$。 网格点总数为9，所以期望为$18/9 = 2$。 

| 步骤| x| y | SX | sy | 贡献 |
 | ---| ---| ---| ---| ---| ---|
 | 计算| 0 | 0 | 3 | 3 | 18 | 18

 这证实了分解正确捕获了所有 9 个网格距离，而无需枚举它们。 

现在考虑$N=3$供应点位于$(2,1)$。 

对于 x，距离为$|0-2|+|1-2|+|2-2|+|3-2| = 2+1+0+1 = 4$。 对于 y，距离为$|0-1|+|1-1|+|2-1|+|3-1| = 1+0+1+2 = 4$。 每个都乘以$4$，所以总计是$32$。 除以 16 个网格点给出期望值$2$。 

| 步骤| x| y | SX | sy | 贡献 |
 | ---| ---| ---| ---| ---| ---|
 | 计算| 2 | 1 | 4 | 4 | 32 | 32

 这证明了对称性：由于均匀的网格结构，即使是非原点也会产生平衡的贡献。

 ## 边缘情况

 对于$N=1$单一供应点位于$(0,0)$，网格有四个点。 距离是$0,1,1,2$总和为 4，因此期望为 1。算法计算$sx = 1$,$sy = 1$, 乘以$N+1=2$，总计为 4，然后除以 4，正好得到 1。 这证实了最小网格的正确处理，其中算术级数公式简化为单项。 

对于边界补给点$(N, N)$，所有左侧贡献在“右公式对称”中消失，并且算术级数正确地折叠为从 0 到 N 的完整三角和。该算法没有特殊情况边界，但仍然产生正确的结果，因为闭合形式公式在端点仍然有效。
