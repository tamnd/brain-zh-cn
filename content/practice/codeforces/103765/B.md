 ## 解决方案
 ## 问题理解

 我们有一个用单串小写字母玩的游戏。 两名玩家轮流轮流，从第一个玩家开始。 移动包括选取当前字符串中任何相邻的相同字符对并删除它们，然后将剩余部分缝合在一起。 如果玩家在自己的回合中找不到这样的对子，则该玩家立即失败。 

任务是确定对于每个给定的起始字符串，假设两个玩家都发挥最佳，第一个玩家是否有获胜策略。 

约束足够严格，所有测试用例的总长度可以达到一百万。 这排除了任何以每次移动线性扫描的简单方式重复模拟字符串上移动的方法，因为在最坏的情况下，每次删除都会将字符串减少两个字符，并且可能存在 O(n) 这样的移动，从而导致总体行为 O(n²) 。 

一个微妙的点是，游戏不是关于任意删除，而只是关于删除相邻的相等对。 这使得结构高度局部化，但在位置之间不独立，因为删除可以创建最初不存在的新的相邻对。 

重要的边缘情况包括根本没有相邻相等对的字符串、所有字符都相同的字符串以及删除级联的字符串。 例如，在`"abccba"`，去掉中间的`"cc"`创造了一个新的机会结构，但不一定是第一个玩家的胜利，因为可用动作的均等比局部贪婪更重要。 

另一个重要的极端情况是存在多个不相交对时。 天真的直觉可能会认为每一对都是一个独立的移动，但这是不正确的，因为删除一对可能会破坏或创建其他对。 

## 方法

 暴力破解的解释很简单：模拟游戏状态。 对于每一个可能的移动，删除一个相邻的相等对，递归地评估结果字符串，并决定当前玩家是否可以强制获胜。 这是一款经典的弦乐DP游戏。 

然而，不同状态的数量呈爆炸式增长。 即使我们通过字符串配置进行记忆，每次删除都会在全局范围内更改邻接关系，并且在最坏的情况下可达状态的数量是指数级的。 For a string like`"aaaaa...."`，每次移动都会减少长度，但会创建新的邻接配置，仍然会产生巨大的分支结构。 这立即超出了任何可行的复杂性界限。 

关键的观察结果是，每次移动都会恰好删除两个字符，并且不会改变剩余字符的相对顺序。 更重要的是，每一步都会将长度减少两倍，因此游戏始终按照偶数步的严格递减顺序进行。 这意味着整个游戏实际上是由可能的走法决定的，而不是由选择的走法决定的。 

如果我们从堆栈过程的角度思考，每次删除都对应于弹出一对相同的相邻字符。 如果我们使用堆栈从左到右贪婪地处理，则重复删除相邻相等对的过程是确定性的：每当当前字符等于堆栈顶部时，我们就会删除顶部而不是推送。 

这种贪婪归约计算最终的归约字符串并隐式计算发生了多少次删除。 每次删除都是游戏中的一步。 由于每次移动都会将长度减少 2，因此无论顺序如何，移动总数都是固定的。 这使游戏陷入奇偶校验问题：如果可能的删除次数是奇数，则第一个玩家获胜，否则第二个玩家获胜。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 初始化一个空堆栈，表示字符串的当前简化形式。 该堆栈模拟了其中没有两个相邻字符相等的不变量。 
2. 从左到右逐个字符扫描字符串。 对于每个字符，如果堆栈不为空，则将其与堆栈顶部进行比较。 
3. 如果堆栈不为空且顶部元素等于当前字符，则弹出堆栈而不是压入。 这表示执行相邻相等对的删除。 这是正确的原因是任何相邻的相等对最终都必须被删除，并且延迟其删除不会影响删除的总数。 
4. 否则，将当前字符压入堆栈，扩展当前的缩减结构。 
5. 维护执行删除次数的计数器。 与匹配对相对应的每次弹出都会使该计数器加一。 
6. 处理完整个字符串后，检查删除计数的奇偶校验。 如果是奇数，则输出“Yes”，因为第一个玩家做出了最后的有效动作。 如果是偶数，则输出“No”。 

### 为什么它有效

 关键的不变量是，在使用堆栈规则处理字符串的每个前缀之后，堆栈表示该前缀在所有可能的最佳播放序列下的完全简化形式，并且无论有效移动的顺序如何，执行的删除次数都是固定的。 这是因为删除仅消除相邻的相等对，而不会在总计数中产生歧义：每个有效的游戏序列都对应于以某种顺序删除不相交的对，但可删除对的总数在重新排序下是不变的，因为每次删除严格地将长度减少两倍，并且不会以改变总计数的方式影响剩余结构的奇偶性。 

因此，游戏简化为计算堆栈过程中发生了多少次此类取消，并通过奇偶性决定获胜者。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []
    
    for _ in range(T):
        s = input().strip()
        stack = []
        moves = 0
        
        for ch in s:
            if stack and stack[-1] == ch:
                stack.pop()
                moves += 1
            else:
                stack.append(ch)
        
        if moves % 2 == 1:
            out.append("Yes")
        else:
            out.append("No")
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案使用堆栈来模拟相邻相等字符的取消。 每次取消完全对应于游戏中的一个有效动作，因此变量`moves`跟踪最佳游戏下可用的移动总数。 

关键的细节是我们永远不需要重新考虑之前的决定。 一旦删除了一对，它就不会影响将来的删除，除非暴露新的邻接关系，并且堆栈自然会处理该问题。 

最终决定纯粹基于奇偶性，这就是为什么不需要进一步的博弈论推理。

 ## 边缘情况

 对于像这样的单个字符`"a"`，堆栈从不执行弹出操作，因此`moves = 0`输出是`"No"`。 该算法正确地识别出不存在有效的移动。 

为了`"aaaa"`，该过程是连续的：前两个角色取消，然后剩余两个角色取消，总共产生两步。 堆栈恰好记录了两次弹出操作，因此奇偶校验为偶数，结果为`"No"`。 

对于交替字符串，例如`"ababab"`，不会形成相邻的相等对，因此堆栈只会增长而不会弹出。 移动计数保持为零，正确生成`"No"`因为在任何阶段都不存在任何合法的举动。
