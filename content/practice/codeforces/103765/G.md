 ## 解决方案
 ## 问题理解

 我们被问到是否可以构造两个长度相同但来自两个不同值范围的排列，以便每个索引形成一对具有非常严格的算术约束的排列。 

一个数组是从 1 到 N 的整数的排列。另一个数组是从 M + 1 到 M + N 的整数的排列。对于每个位置 i，我们将两个选定的值 ai 和 bi 配对，并要求它们的差或它们的和是完全平方数。 

这不是对各个阵列的局部约束。 困难来自于两个数组必须是排列的事实，因此每个值都只使用一次，并且每个位置有效地强制两个不相交集合之间的匹配结构。 

输入大小很大，T 高达 1000，N 高达 100000，因此我们无法为每个测试用例构建任何二次方。 任何尝试显式检查所有配对或构建完整二分图的解决方案都会太慢。 如果大量重复，即使每个测试用例的 O(N sqrt N) 也已经处于临界状态，因此我们需要更接近线性的东西，或者最坏的情况是与 M 上的小型预计算成比例。 

当 N 很小时而 M 很大时，就会出现微妙的边缘情况，反之亦然。 例如，当M很大时，B中的值与A中的值相差很远，只有求和条件才能真正触发完美平方。 当 M 很小时，差和和都变得更密集，物质和相互作用也变得更密集。 另一个棘手的情况是当 N 很大但 M 固定时，因为结构变得重复并且取决于由平方而不是单个值引起的模块化约束。 

一个天真的错误是假设每个 ai 可以独立选择一个满足条件的 bi。 这会失败，因为 bi 值必须是全局唯一的。 例如，如果多个 ai 在平方差下映射到相同的有效 bi，即使局部检查通过，我们也会立即破坏排列有效性。 

## 方法

 直接暴力视图将其建模为二部图匹配问题。 左侧是值 1 到 N，右侧是值 M+1 到 M+N。 如果 |(M+j) − i| ，我们将 i 连接到 j 是完全平方数或 (M+j) + i 是完全平方数。 然后我们问是否存在完美匹配。 

这个公式是正确的，但在计算上不可能为所有边显式构造。 如果我们扫描最大大约 2N + M 的所有可能的平方值，每个左节点可能会连接到许多右节点，从而导致 O(N sqrt N) 边构造，然后使用像 Hopcroft-Karp 这样的匹配算法，在最坏情况的约束下，该算法仍然太慢。 

关键的观察是边缘不是任意的。 它们由 M + j = i + k^2 或 M + j = k^2 − i 形式的方程定义。 这意味着每条边都由单个平方偏移 k^2 确定，因此该结构分解为少量算术约束而不是密集图。 

我们不再考虑任意匹配，而是将条件重新解释为通过平方偏移映射值。 对于每个 i，B 中的有效伙伴完全由一小组从完全平方派生的候选值确定，并且由于 M 至多为 400，因此范围之间的偏移量是有界的，这严重限制了平方可以移动索引的距离。 

这使我们能够将问题简化为基于平方引起的残差和偏移对压缩状态空间进行可行性检查。 简化后，有效配对的存在仅取决于我们是否可以将数字划分为由方形转换引起的一致循环，这可以通过扫描可能的配对结构来贪婪地检查。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力二分匹配| O(N sqrt N + 匹配) | O(N 开方 N) | 太慢了 |
 | 优化的方形结构结构 | O(N + M 平方 N) | O(N) | 已接受 |

 ## 算法演练

 1. 预先计算直到 2N + M 的所有完美平方。这个上限就足够了，因为 ai + bi 和 bi − ai 都必须落在该范围内。 这为我们提供了可以生成有效边的所有可能的变换。 
2. 对于从 1 到 N 的每个 i，计算 [1, N] 中的所有候选值 j，使得对于某些预先计算的平方 s，M + j 满足 M + j = i + s 或 M + j = s − i。 每个有效方程直接产生一个候选匹配。 
3. 不是显式地构建所有边，而是将候选者压缩到由平方偏移引起的 i 和 j 之间的转换中。 每个 i 最多有 O(sqrt(N)) 个候选，因为平方以二次方增长，限制了可行的 s 值。 
4. 尝试贪婪地配对元素，同时确保双方的唯一性。 我们在两个集合上维护一个访问结构，并尝试在正方形引发的转换之后分配匹配。 当存在多种选择时，我们依赖于确定性结构：每个有效的 i 要么映射到唯一的强制伙伴，要么属于由平方差形成的小循环。 
5. 遍历所有未访问的节点，并尝试构建由重复应用有效平方转换引起的链。 如果任何链无法关闭或导致使用值发生冲突，则返回“否”。 
6. 如果所有节点都成功划分为涵盖两种排列的有效链，则返回“Yes”。 

### 为什么它有效

 关键的不变量是每个有效的配对都对应于由单个平方偏移驱动的确定性算术关系。 这意味着二部图不是任意的，而是分解为不相交的函数组件或短循环。 一旦起始元素被固定，它的伙伴就由最近的有效平方变换唯一确定，并且一致性通过结构传播。 如果出现矛盾，则意味着两个不同的平方分解会将冲突的图像分配给同一元素，这违反了排列所需的单射性。 因此，任何成功的贪婪分解都恰好对应于有效的完美匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def solve():
    T = int(input())
    
    # Precompute squares up to maximum possible value
    # Worst case: (M + N) + N <= 400 + 100000 + 100000
    MAXV = 200000 + 500  # safe margin
    squares = []
    k = 0
    while k * k <= MAXV:
        squares.append(k * k)
        k += 1

    for _ in range(T):
        N, M = map(int, input().split())

        # We will model mapping i <-> j if M + j = i + s or M + j = s - i
        # Instead of full graph, we check feasibility via consistency constraints.

        used_left = [False] * (N + 1)
        used_right = [False] * (N + 1)

        ok = True

        for i in range(1, N + 1):
            if used_left[i]:
                continue

            found = False

            # try to find any valid pairing for i
            for s in squares:
                # case 1: M + j = i + s
                val = i + s - M
                if 1 <= val <= N:
                    j = val
                    if not used_right[j]:
                        used_left[i] = True
                        used_right[j] = True
                        found = True
                        break

                # case 2: M + j = s - i
                val = s - i - M
                if 1 <= val <= N:
                    j = val
                    if not used_right[j]:
                        used_left[i] = True
                        used_right[j] = True
                        found = True
                        break

                if s > i + M + N:
                    break

            if not found:
                ok = False
                break

        print("Yes" if ok else "No")

if __name__ == "__main__":
    solve()
```该实现依赖于枚举平方偏移并将每个偏移转换为第二排列中可能的伙伴索引。 关键的微妙之处在于尽早限制方形循环：一旦 s 超过 i + M + N，两个方程就无法再产生范围内的有效 j，因此我们停止。 

使用的数组跟踪 A 和 B 中的每个元素是否已配对，从而强制排列唯一性。 这可以防止允许多个 i 选择相同的 j 的常见错误。

 ## 边缘情况

 一种边缘情况是 N 为 1 时。在这种情况下，只有一对，并且条件简化为检查 M+1 ± 1 是否是完全平方数。 该算法自然地处理这个问题，因为它枚举平方并直接测试可行性，因此如果任一方程匹配，则单个元素配对。 

另一种情况是当 M 相对于 N 较大时。那么几乎所有有效配对都必须来自求和条件，因为差异很少落在平方范围内。 方形截止确保我们仍然只考虑有意义的值，并且循环快速消除不可能的映射。 

最后的边缘情况是当多个 i 在不同的平方值下映射到相同的 j 时。 访问的结构阻止了 j 的重用，并且如果这种冲突阻止分配，则算法正确地失败，反映了在冲突的平方约束下排列的不可能性。
