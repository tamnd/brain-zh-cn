 ## 解决方案
 ## 问题理解

 我们得到了几个独立的橘子园场景。 每个场景中都有$n$树木，以及$i$-th 树包含$a_i$橙子。 从每棵树上，我们可以挑选任意整数个橙子$0$和$a_i$，独立于其他树。 对所有树木进行选择后，我们获得了采摘橙子的总数。 

也有固定的数量$m$，代表队友。 有效的采摘计划是指采摘的橙子总数可以平均分配给所有队友，这意味着所有选定值的总和可以除以$m$。 任务是计算有多少个不同的采摘计划满足此可分性条件，其中如果至少一棵树贡献了不同数量的采摘橙子，则两个计划不同。 

每棵树的行为就像一个有界变量，对总和做出贡献，我们正在计算有多少种方法来选择这些变量，以便总和位于特定同余类模中$m$。 

约束条件意味着$n$可以达到$10^5$每个测试套件和$m \le 50$。 测试用例数量较多，最多可达$10^3$，但总共$n$在所有测试中也受$10^5$。 这强烈表明了一个接近线性的解决方案$n$每次测试，附加因素取决于$m$，而任何平方$n$是不可能的。 

对所有选择的天真枚举会考虑$\prod (a_i + 1)$配置，即使对于中等输入来说也是天文数字。 即使存储或迭代所有的总和也是不可行的。 

当所有的情况出现时，会出现更微妙的边缘情况$a_i$很大并且$m = 1$。 在这种情况下，每个选择都是有效的，所以答案应该是$\prod (a_i + 1)$模数$998244353$。 任何仅跟踪残基而不考虑总计数的方法仍必须保留这种行为。 

另一个极端情况是当所有$a_i = 0$。 只有一种方法什么都不选，总和是$0$，它总是可以被整除$m$。 答案应该永远是$1$，独立于$m$。 

## 方法

 一种直接的方法是将每棵树视为贡献一个小的多项式：$i$，我们可以选择$0,1,\dots,a_i$，所以它对生成函数的贡献是$x^0 + x^1 + \dots + x^{a_i}$。 有效方式的总数是其指数可被整除的所有单项式的系数和$m$在所有这些多项式的乘积中。 

直接扩展该产品是不可能的，因为度数会增长到$10^5$每项和项数以组合方式爆炸。 

关键的观察是我们不关心确切的总和，只关心它的模值$m$。 这可能会导致状态空间崩溃$10^5 \cdot n$可能的总和仅$m \le 50$残留类。 这立即表明了对残差的动态规划。 

我们维护一个 DP 数组，其中`dp[r]`计算有多少种方法可以达到总和等于$r \bmod m$。 最初，`dp[0] = 1`。 对于每棵树，我们计算它对每个残基类有多少种贡献，然后将其与全局 DP 进行卷积。 

对于一棵有价值的树$a$， 定义`cnt[t]`作为整数的个数$[0,a]$是一致的$t \bmod m$。 这可以使用算术级数计数在每个残基的恒定时间内计算出来。 然后树在残差上贡献一个小的多项式，我们用模卷积更新 DP$m$。 

每棵树的更新成本$O(m^2)$，给出总计$O(n m^2)$，这是可以接受的，因为$m \le 50$和总计$n \le 10^5$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举|$O(\prod (a_i+1))$|$O(n)$| 太慢了 |
 | DP 残留量 |$O(n m^2)$|$O(m)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1.初始化一个大小为DP的数组$m$，所有值都为零，除了`dp[0] = 1`。 这表示在选择任何树之前，唯一可实现的总和是 0。 
2. 对于每棵树$i$，计算中有多少个整数$[0, a_i]$落入每个残基类模$m$。 我们通过首先编写来做到这一点$a_i = q \cdot m + r$。 每个残基都准确地出现$q$次数和残留量$0$通过$r$额外出现一次。 这给出了频率数组`cnt`。 

此步骤很重要，因为它将大范围压缩为残差的紧凑表示，而不会丢失组合信息。 
3.创建新的DP阵列`ndp`初始化为零。 在处理这棵树后，我们将计算从先前的总和到新的总和的转换。 
4. 对于每个残留物$j$在当前的 DP 状态下，以及每个残基$k$从当前的树贡献中，添加：$$ndp[(j+k)\bmod m] += dp[j] \cdot cnt[k]$$这表示选择带有余数的部分和$j$，然后从当前树中添加一个带有残差的选择$k$。 
5. 填充后`ndp`， 代替`dp`和`ndp`。 
6. 当所有的树都处理完之后，答案是`dp[0]`，因为我们只想要可被整除的总和$m$。 

正确性取决于跟踪按残差分组的所有部分和。 步步，`dp[r]`精确计算获得与以下内容一致的总和的方法的数量$r$。 转换保留了这个不变量，因为有效部分配置的每个扩展都被计算一次，并且残差加法 mod$m$正确更新类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))

        dp = [0] * m
        dp[0] = 1

        for x in a:
            q, r = divmod(x, m)

            cnt = [q] * m
            for i in range(r + 1):
                cnt[i] += 1

            ndp = [0] * m
            for j in range(m):
                if dp[j] == 0:
                    continue
                for k in range(m):
                    if cnt[k] == 0:
                        continue
                    ndp[(j + k) % m] = (ndp[(j + k) % m] + dp[j] * cnt[k]) % MOD

            dp = ndp

        print(dp[0] % MOD)

if __name__ == "__main__":
    solve()
```DP 阵列`dp`存储可达到的残留物的计数。 对于每棵树，数组`cnt`将选择范围压缩为剩余频率。 嵌套循环实现模类上的卷积。 在转换期间应用模数以避免溢出并匹配所需的输出格式。 

一个微妙的点是处理`cnt`: 总和为$a_i + 1$，因此在转换过程中保留了总的路数。 模运算仅应用于最终的残差索引，而不应用于选择计数结构本身。 

## 工作示例

 考虑输入：```
n = 3, m = 2
a = [1, 1, 1]
```每棵树都允许选取 0 或 1。因此，每棵树都会贡献一个 0 和一个 1。 

在第一棵树之后，DP 为：

 | 残留物 | DP |
 | ---| ---|
 | 0 | 1 |
 | 1 | 1 |

 第二棵树之后：

 | j（旧 dp）| 碳纳米管| 贡献|
 | ---| ---| ---|
 | 0 | {0:1,1:1} | 0 和 1 相加 |
 | 1 | {0:1,1:1} | 1 和 0 相加 |

 结果 dp:

 | 残留物 | DP |
 | ---| ---|
 | 0 | 2 |
 | 1 | 2 |

 在第三棵树之后，应用相同的转换，产生：

 | 残留物 | DP |
 | ---| ---|
 | 0 | 4 |
 | 1 | 4 |

 所以答案是`dp[0] = 4`。 

该迹线表明 DP 保持对称，因为每棵树都贡献相同的残基结构，并且保留了 dp 代表选择上的全卷积的不变式。 

现在考虑：```
n = 1, m = 3
a = [4]
```我们可以选择 0 到 4，给出残数：

 0,1,2,0,1。 所以计数是：

 | 残留物 | 碳纳米管|
 | ---| ---|
 | 0 | 2 |
 | 1 | 2 |
 | 2 | 1 |

 从 dp = [1,0,0] 开始，经过卷积我们得到 dp = [2,2,1]。 答案是2，对应采摘0个或3个橙子。 

这证实了残差压缩正确地解释了当$a_i$不是的倍数$m$。

 ## 边缘情况

 当$m = 1$，每个总和都可以被整除$m$，因此 DP 应该有效地计算所有可能的选择。 该算法处理这个问题是因为`cnt`总和总是$a_i + 1$， 和`dp`只有一种状态，因此它可以正确地将所有贡献相乘，而不会产生任何残留混合。 

当所有$a_i = 0$， 每个`cnt`数组有`cnt[0] = 1`和所有其他条目为零。 DP 永远不会改变其初始状态，因此`dp[0]`仍然是 1，这与只有一种方法不选择任何内容的事实相匹配。 

什么时候$n$很大但是全部$a_i < m$, 里面的分布`cnt`变得高度稀疏，但卷积仍然可以正常工作，因为它显式地迭代所有残基并保留零条目。
