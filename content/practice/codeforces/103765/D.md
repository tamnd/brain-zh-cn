 ## 解决方案
 ## 问题理解

 我们给出了一个场景，其中有$n$相同的香蕉和$m$猴子。 每只猴子必须至少收到一根香蕉，所以我们真的在分发$n$进入$m$正整数$a_1, a_2, \dots, a_m$总金额固定。 

对于任何此类分布，某些香蕉计数值可能会在猴子中多次出现。 我们关心的数量是收到完全相同数量的香蕉的最大猴子数量。 由于可能有不同的分布，我们假设以最有利的方式选择分布以避免重复，并且我们问仍然不可避免的问题：这个“最大重复计数”的最小可能值是多少。 

换句话说，我们要求的是最小的整数$k$这样无论我们如何巧妙地选择$m$正整数总和为$n$，总会存在至少出现一定数量的香蕉$k$次。 

限制条件非常大：$n$可以达到$10^{18}$和$m$最多$10^9$，最多有 1000 个测试用例。 这立即排除了任何显式构造分布或迭代可能值的方法。 每个测试用例的任何解决方案都必须是对数或常数。 

出现微妙的边缘情况时$n$刚好大到足以强制重复。 例如，当$m=3$,$n=3$，唯一可能的分布是$(1,1,1)$，所以答案是$3$。 什么时候$n=6$,$m=3$，我们可以使用$(1,2,3)$，所以答案就变成了$1$。 这表明答案并不完全取决于$m$，还取决于“分散”总和的程度$n$允许。 

主要困难是我们没有计算固定数组中的重复次数； 我们正在对总和约束下所有可能的整数分区进行推理，并要求实现最佳的平衡。 

## 方法

 暴力的观点是枚举所有可能的分裂方式$n$进入$m$正整数并计算每个分区的任何值的最大频率。 这在概念上是正确的，但完全不可行，因为分区的数量随着$n$。 

为了避免这种情况，我们改变视角。 我们不考虑个体分布，而是考虑“避免重复”意味着什么。 如果我们不希望任何值出现超过$k$次，那么我们就限制了有多少只猴子可以分享相同的香蕉数量。 该限制强制采用一种结构模式：值必须分组，并且每个组最多有大小$k$。 问题是我们是否可以将香蕉数量分配给这些组，以便所有$m$猴子受到保护，同时尊重总金额$n$。 

关键的观察是，为了使重复尽可能少，我们应该使用尽可能多的不同值，并将它们尽可能均匀地包装在上限下$k$。 对于固定的情况来说最难满足的情况$k$是当我们尝试最小化总和同时仍然使用$m$猴子并尊重没有价值被使用超过$k$次。 如果即使这个最小的建筑超过$n$， 然后$k$是不可能的； 否则它是可以实现的。 

这将问题简化为寻找最小的$k$使得在此约束下的最小可能总和最多为$n$。

 ## 算法演练

 我们二分查找答案$k$，任何香蕉计数的最大允许频率。 

对于固定候选人$k$，我们计算分配时可实现的最小可能总和$m$将猴子分成几组，每组代表共享相同香蕉数量且最大尺寸的猴子$k$。 

1. 计算有多少个完整的大小组$k$我们可以形成：$t = \lfloor m / k \rfloor$，以及余数$r = m \bmod k$。 
2. 为了最小化总和，我们将最小的香蕉数分配给最大的组。 这意味着我们在概念上使用大小按值递增的顺序填充组$k, k, \dots, k, r$。 
3. 该结构的最小可实现总和是通过赋值获得的$1, 2, 3, \dots$给这些群体。 总计变为加权总和：$$\text{cost}(k) = k(1 + 2 + \dots + t) + r \cdot (t+1)$$4. 如果$\text{cost}(k) \le n$，那么可以实现一个分布，其中没有值出现超过$k$次。 
5.我们二分查找最小的$k$满足可行性。 

### 为什么它有效

 结构很紧凑，因为任何具有最大频率的有效分配$k$必须将猴子分成最多大小的组$k$。 为了最小化总和，我们总是将尽可能小的香蕉值分配给最大的组； 否则交换值只会增加总和而不会提高可行性。 这种贪婪的分层产生了在约束下可实现的绝对最小总和，因此与$n$正确判断是否$k$就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def feasible(n, m, k):
    t = m // k
    r = m % k

    # cost = k * (1 + 2 + ... + t) + r * (t + 1)
    cost = k * t * (t + 1) // 2 + r * (t + 1)
    return cost <= n

def solve_case(n, m):
    lo, hi = 1, m
    while lo < hi:
        mid = (lo + hi) // 2
        if feasible(n, m, mid):
            hi = mid
        else:
            lo = mid + 1
    return lo

def main():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        print(solve_case(n, m))

if __name__ == "__main__":
    main()
```这`feasible`函数计算假设香蕉数量不超过的最小可能总和`k`次。 然后二分查找找到最小的这样的`k`。 关键的实现细节是使用 64 位安全算术进行三角和，因为$n$可以大到$10^{18}$。 

## 工作示例

 考虑$n = 6, m = 3$。 

我们测试$k = 1$。 然后$t = 3, r = 0$，所以成本是$1 + 2 + 3 = 6$。 这是可行的，所以$k = 1$作品。 

现在考虑$n = 3, m = 3$。 

为了$k = 1$，成本又是$6$，这超过了$n$， 所以$k=1$是不可能的。 我们尝试$k=2$： 现在$t=1, r=1$，成本为$2 \cdot 1 + 1 \cdot 2 = 4$，仍然太大。 为了$k=3$:$t=1, r=0$，成本为$3$，匹配$n$。 所以答案是$3$。 

这些示例显示了收紧频率限制如何迫使施工分成更少的组，从而增加可实现的最小总和。

 ## 边缘情况

 当$n$非常接近$m$，算法往往返回大$k$，因为几乎没有空间将价值观分开。 在这种情况下，成本函数急剧增长，二分搜索很快收敛到$k=m$，这意味着所有猴子在极端压缩下必须共享相同的值。 

什么时候$n$相比而言较大$m$，成本条件立即满足$k=1$，反映出我们可以分配严格递增的值而不强制重复。 可行性检查正确地捕捉到了这一点，因为成本的三角结构相对于$n$。 

这两个极端证实了模型在“一切相同”和“完全不同”之间平稳过渡，无需特殊的外壳。
