 **求解时间：** 1m 13s
 **已验证：** 是的

 ## 解决方案
 ## 问题理解

 我们得到了一系列以固定的从左到右顺序放置的矩形图块。 每个图块都有两个可能的方向：我们可以将其保持为宽高，也可以将其旋转 90 度，从而交换两个值。 为每个瓷砖独立选择方向后，我们只查看最终的高度。 要求是这些高度必须形成从左到右不递增的序列。 

任务是确定是否存在这样的方向选择。 

关键约束是$n \le 10^5$。 这立即排除了对方向的任何指数探索，因为每个矩形都有两个状态，并且完整的强力搜索将涉及$2^n$的可能性。 即使对矩形对进行二次方法也会太慢。 因此，我们正在寻找一种贪婪或线性时间结构，可以从左到右处理一次矩形。 

当两个方向在局部都有效但只有一个方向导致可行的全局延续时，就会出现一种微妙的边缘情况。 例如，如果一个矩形是$(5, 4)$，并且之前选择的高度为 5，我们可能会倾向于选择仍然适合的较大可能方向。 然而，这种贪婪的“最大化当前高度”策略可能会破坏未来的可行性。 

考虑这个输入：```
2
5 4
4 6
```在第一个矩形中，选择高度 5 就可以了。 在第二个矩形中，6 和 4 都不能以非递增序列跟随 5，所以答案是否定的。 但如果我们在其他情况下选择不同，我们可能会保留可行性，这意味着必须仔细定义局部最优。 

## 方法

 暴力方法会尝试每个矩形的所有可能的方向选择。 对于每个配置，我们检查所得的高度序列是否不增加。 这很简单：对于每个$2^n$配置，我们扫描$n$元素，给予$O(n \cdot 2^n)$。 和$n = 10^5$，这是完全不可行的。 

问题的结构允许简化：在每一步中，我们只关心当前矩形仍然可以使用的最大允许高度，由之前的选择决定。 每个矩形为我们提供了两个候选高度，我们希望选择不超过之前选择的高度的最大的一个。 这种贪婪的选择之所以有效，是因为只要有可能就留下更大的有效高度，可以为未来保留更多的灵活性，同时仍然保持局部的可行性。 

我们将问题转化为一次传递：维护一个变量`cur`表示当前位置允许的最大高度。 对于每个矩形，选择最大可能的方向 ≤`cur`。 如果两个方向都不适合，则该过程失败。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \cdot 2^n)$|$O(n)$| 太慢了|
 | 贪心扫描|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们从左到右处理矩形，同时保持当前位置的最大允许高度。 

1. 初始化`cur`作为无穷大。 这表示第一矩形没有上限。 
2. 对于每个有边的矩形$(w, h)$，考虑两种可能的高度：$h$和$w$。 这两个值的顺序最初并不重要。 
3. 选择两个值中小于或等于的较大者`cur`。 如果两个值都有效，我们就采用较大的值，因为它为后面的矩形保留了更多空间。 
4. 如果两个值都不小于或等于`cur`，我们立即得出结论，序列不能变为非增并停止。 
5. 更新`cur`到选定的高度并继续到下一个矩形。 
6. 如果所有矩形均处理成功，则输出YES。 

### 为什么它有效

 在每一步中，我们都保持不变式`cur`是考虑到所有先前选择，我们可以放置在当前位置的最大可能高度。 在当前矩形的所有有效方向中，选择最大可行高度是安全的，因为任何较小的选择只会进一步限制未来的步骤，而不会解锁任何其他有效配置。 由于可行性仅取决于保持在之前的高度以下，因此保留最大可行值可确保我们永远不会放弃以后可能成功的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    cur = float('inf')

    for _ in range(n):
        a, b = map(int, input().split())
        if a > b:
            a, b = b, a

        if a <= cur and b <= cur:
            cur = b
        elif b <= cur:
            cur = b
        elif a <= cur:
            cur = a
        else:
            print("NO")
            return

    print("YES")

if __name__ == "__main__":
    solve()
```该代码直接反映了贪婪策略。 对于每个矩形，我们通过对它们进行排序来标准化其两个候选高度`a <= b`。 这使得推理变得更简单：`b`始终是该矩形的最佳可能高度，并且`a`是后备。 

条件结构确保我们始终选择最大的有效高度。 如果两者都适合`cur`，我们取`b`。 如果只有一个合适，我们就选择那个。 如果两者都不适合，我们会提前终止。 

一个常见的实现陷阱是忘记标准化每个矩形或错误更新`cur`即使不存在有效的选择。

 ## 边缘情况

 一种关键的边缘情况是，由于先前的限制，矩形只能放置在一个方向上。 例如：```
2
10 1
5 6
```在第一个矩形之后，`cur = 10`。 第二个矩形有候选 5 和 6，两者都有效，因此我们选择 6。该序列仍然有效。 

现在考虑：```
2
10 1
7 8
```第一步之后，`cur = 10`。 对于第二个矩形，7 和 8 都有效，因此我们选择 8。该序列有效并以 8 ≤ 10 结束。 

最后：```
2
5 1
6 7
```在第一个矩形之后，`cur = 5`。 第二个矩形有候选 6 和 7，两者都不≤ 5，因此算法正确输出“否”。 故障是立即发生的，并且以后的调整无法修复它，因为此步骤中不存在有效的方向。
