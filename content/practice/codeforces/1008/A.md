 ## 解决方案
 ## 问题理解

 我们会得到一个小写单词，并被要求验证它是否遵循特定的语音规则。 该规则限制辅音和元音如何按顺序出现。 

每个辅音后面都必须紧跟着一个元音，但有一个例外：辅音`'n'`允许后跟任何字母或字符串末尾。 元音固定为`'a'`,`'e'`,`'i'`,`'o'`， 和`'u'`。 任何其他字母都被视为辅音。 

任务只是确定整个字符串是否在其应用的所有地方都遵守此约束。 

输入长度最多为100，这意味着直接扫描字符串就足够了。 甚至一个$O(n^2)$此处的方法很简单，但结构表明单个从左到右的验证就足够了。 

在非正式地思考规则时，很容易忽略一些边缘情况。 第一个是出现在字符串末尾的辅音。 例如，`"abc"`结束于`'c'`，这是一个辅音，因此它立即违反了规则，因为没有后续元音。 

另一种情况是连续的辅音。 例如，`"king"`失败是因为`'k'`接下来是`'i'`这很好，但是`'n'`接下来是`'g'`， 和`'g'`是不是辅音`'n'`，所以后面需要一个元音，但没有任何保证。 

最后，`'n'`本身很特殊，但仅限于本地。 像这样的字符串`"nbo"`是有效的，因为`'n'`后面可以跟辅音，但是如果它是非辅音，则不会为下一个字符提供安全性`n`辅音。 

## 方法

 强力解释将检查每个辅音位置并向前扫描，直到找到下一个字符以验证它是否是元音或辅音是否是`'n'`。 这工作正常，但在最坏的情况下，它会重复重新扫描后缀，导致二次行为。 

由于这里的字符串很短，即使这样也会通过，但条件的结构表明了一个更清晰的不变量：每个位置的有效性仅取决于当前字符和下一个字符。 不存在远程依赖性，因此我们不需要任意向前看。 

这将问题简化为单次线性传递。 在每个位置，我们只需要验证当前字符是否是违反规则的辅音即可。 如果是元音，则不需要检查任何内容。 如果是的话`'n'`，总是安全的。 否则，下一个字符必须存在并且必须是元音。 

这一观察结果消除了所有冗余检查，并使验证纯粹是本地的。

 ## 算法演练

 我们从左到右处理字符串，根据每个字符是否是元音来验证它，`'n'`，或另一个辅音。 

1. 定义一组元音`{a, e, i, o, u}`所以会员资格检查的时间是恒定的。 这避免了重复的字符串比较。 
2. 迭代每个索引`i`字符串的。 
3. 如果当前字符是元音，则不受限制地继续，因为元音对后续字母没有限制。 
4. 如果当前字符是`'n'`，也继续因为`'n'`不受“后面必须跟元音”规则的约束。 
5. 如果当前字符是任何其他辅音，我们必须确保有下一个字符。 如果`i + 1`超出范围，该字符串立即无效。 
6. 如果存在下一个字符，请验证它是元音。 如果没有，则违反规则，我们可以提前终止。 

### 为什么它有效

 关键的不变量是，每个无效配置都可以在非非配置的确切位置检测到。`n`辅音出现。 这样的辅音对其直接后继有严格的本地要求。 如果违反了该要求，则以后不可能进行更正，因为该规则不是关于全局结构而是关于邻接性。 因此，仅检查下一个字符就足以保证正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    vowels = set("aeiou")
    
    n = len(s)
    for i, c in enumerate(s):
        if c in vowels:
            continue
        if c == 'n':
            continue
        
        # consonant that must be followed by vowel
        if i + 1 >= n or s[i + 1] not in vowels:
            print("NO")
            return
    
    print("YES")

if __name__ == "__main__":
    solve()
```代码直接反映了算法。 元音集用于快速成员资格检查，确保每个字符的恒定时间验证。 失败时提前退出可避免检测到违规后进行不必要的扫描。 

一个微妙的点是边界检查`i + 1 >= n`。 没有它，访问`s[i + 1]`当辅音出现在字符串末尾时会引发错误。 这正是使像这样的词无效的情况`"king"`。

 ## 边缘情况

 像这样的单字符字符串`"n"`是有效的，因为`'n'`明确免除需要后面的元音。 循环处理它一次，将其分类为特殊情况，并接受它，没有任何边界问题。 

以非结尾的字符串`n`辅音如`"abc"`在最后一个字符处失败。 什么时候`i`达到最终指标，条件`i + 1 >= n`立即触发拒绝。 这抓住了核心规则，即每个非特殊辅音后面必须跟有另一个字符，并且该字符必须是元音。 

一系列辅音，例如`"bcd"`第一个字符已经失败，因为`'b'`接下来是`'c'`，它不是元音，也不能免除。 该算法提前终止，表明在最早可能的位置检测到违规行为，而无需完全遍历。
