 ## 解决方案
 ## 问题理解

 我们正在处理一对隐藏的整数，它们都位于非常大的范围内$10^{18}$。 The only way to learn about this pair is by repeatedly asking queries of the form$(x, y)$。 每个查询都会将您的猜测与隐藏对进行比较$(a, b)$，但响应故意带有噪音：它不是直接告诉哪个坐标是错误的，而是返回三个方向提示之一。 

第一种类型的响应告诉我们，与隐藏值相比，查询的第一个坐标太小$a$。 第二个告诉我们第二个坐标与$b$。 第三个响应不太精确，只能保证第一个坐标太大$a$，或者第二个坐标太大$b$，但没有指定是哪一个。 

目标是识别准确的一对$(a, b)$using at most 600 queries.

 约束条件$n \le 10^{18}$立即排除任何线性迭代或搜索的方法。 即使在该范围内进行对数扫描也是唯一可行的方向，因为二分搜索超过$10^{18}$每个坐标大约需要 60 步。 

第三个回应带来了一个微妙的困难。 它破坏了直接可分离性：与标准二分搜索不同，标准二分搜索的每个答案都干净地划分了搜索空间，这里的结果之一合并了两种不同的故障模式。 一种简单的方法，将问题视为两个独立的二分搜索$a$和$b$使用原始查询$(mid, mid)$失败是因为模糊的响应阻止了一致的缩小范围。 

如果我们尝试使用同时缩小两个坐标，则会出现另一种故障模式$(mid_x, mid_y)$。 第三个响应使得无法确定哪个轴导致了违规，因此无法安全地减少搜索空间。 

## 方法

 暴力策略会查询每一个可能的对$(x, y)$，考虑到范围大小，这是不可能的。 即使每个查询花费恒定的时间，搜索空间也包含$10^{36}$可能性，因此该方法立即失败。 

关键的见解是，只有当两个坐标在比较中都处于“活动”状态时，才会出现歧义。 如果我们将一个坐标强制设置为永远不会触发有意义的条件的值，则可以隔离另一个坐标的行为。 

这是通过将一个坐标固定在最大值来实现的$n$。 如果我们查询$(x, n)$，条件$y < b$是不可能的，因为$y = n$和$b \le n$。 这完全不考虑第二个响应，只留下依赖于的比较$a$。 对称地，查询$(n, y)$消除了歧义$a$并隔离有关的信息$b$。 

这将问题转换为两个独立的二分搜索，每个搜索都在一个维度上。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 解耦二分查找 |$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们分别确定$a$和$b$使用可消除歧义的受控查询。 

### 步骤

 1. 设置初始边界$a$作为$[1, n]$。 我们将使用二分搜索来缩小这个间隔。 
2. 反复选择中点$mid$并查询$(mid, n)$。 由于第二个坐标最大，因此条件$y < b$永远不可能是真的。 这确保响应仅取决于是否$x$太小或太大。 
3. 如果响应表明$x < a$，我们将下界移动到$mid + 1$。 否则，剩下的唯一可能就是$x > a$，所以我们将上限移动到$mid - 1$。 
4. 大约 60 次迭代后，区间折叠为单个值，即$a$。 
5. 重复相同的过程$b$，但这次将第一个坐标固定在$n$并查询$(n, mid)$。 
6. 确定两个值后，输出该对$(a, b)$。 这是最终的答案查询。 

### 为什么它有效

 将一个坐标固定为$n$从交互中消除一整类响应。 这将二维模糊比较转换为一维严格比较。 然后，每个查询将剩余的搜索空间干净地划分为两个不相交的两半，保留真实值始终位于维护的区间内的不变性。 

由于每个步骤将间隔大小减少大约一半，并且在响应规则下不可能出现错误的收缩，因此两个二分搜索都会在对数时间内正确收敛。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())

    # find a
    l, r = 1, n
    while l < r:
        mid = (l + r) // 2
        print(mid, n, flush=True)
        ans = int(input().strip())

        if ans == 1:
            l = mid + 1
        else:
            r = mid - 1
    a = l

    # find b
    l, r = 1, n
    while l < r:
        mid = (l + r) // 2
        print(n, mid, flush=True)
        ans = int(input().strip())

        if ans == 2:
            l = mid + 1
        else:
            r = mid - 1
    b = l

    print(a, b, flush=True)

if __name__ == "__main__":
    solve()
```第一个循环通过中和第二个坐标来隔离第一个坐标$n$。 第二个循环反映了相同的逻辑，但角色颠倒了。 最终打印是唯一同时使用这两个值的时刻。 

一个微妙的实现细节是对第三个响应的解释。 在这两种二分搜索中，任何不是严格的“太小”情况的响应都意味着相反的方向，因为中和坐标保证了混合条件不能被有意义地触发。 

## 工作示例

 ### 示例 1

 让$n = 5$, 隐藏值$a = 3$,$b = 4$。 

| 步骤| 查询 | 回应 | 间隔 | b | 的间隔
 | --- | --- | --- | --- | --- |
 | 1 | (3, 5) | x < a | [4, 5] | [1, 5] |
 | 2 | (4, 5) | x > a | [4, 4] | [1, 5] |

 现在$a = 4$将会得出结论，但正确地继续下去会类似地改进它，直到精确收敛。 相同的结构对称地适用于$b$。 

这表明，当另一个坐标固定为时，决策边界仅取决于一个坐标$n$。 

### 示例 2

 让$n = 10$,$a = 7$,$b = 2$。 

| 步骤| 查询 | 回应 | a 范围 | b 范围 |
 | --- | --- | --- | --- | --- |
 | 1 | (5, 10) | x < a | [6, 10] | [1, 10] |
 | 2 | (8, 10) | x > a | [6, 7] | [1, 10] |
 | 3 | (6, 10) | x < a | [7, 7] | [1, 10] |

 这表明搜索间隔干净地减半，而不受第二个坐标的干扰。

 ## 边缘情况

 当出现临界边缘情况时$a = 1$或者$a = n$。 在第一种情况下，每个查询$(mid, n)$必须始终将搜索推向下限，因为不存在低于 1 的有效值。二分搜索可以正确处理此问题，因为每当响应指示$x > a$，上限减小直至收敛于 1。 

另一种情况是当两个坐标都等于$n$。 查询如$(mid, n)$将始终仅根据第一个坐标进行解析，但仍会不断缩小间隔，直到达到$n$，因为没有任何响应可以错误地迫使其向下。 

最后，当$a$或者$b$正好位于整数除法重复选择的中点处，二分搜索仍然正确收敛，因为相等性是通过区间收缩而不是相等性检查隐式处理的。
