 ## 解决方案
 ## 问题理解

 该问题要求我们反复用数字之和替换一个数字，直到该数字变成单位数。 输入的是一个数字`n`它可能非常大，最多可达 1000 万位数字。 输出是达到一位数所需的变换次数。 

输入大小立即排除存储`n`作为大多数编程语言中的标准整数，因为具有数百万位的整数超出了本机数据类型。 我们需要一种可以与`n`作为字符串或利用数字和的属性，而不完全具体化每个中间数字。 

关键的边缘情况发生在以下情况：`n`已经是个位数了。 例如，如果`n = 0`或者`n = 7`，不需要转换，所以输出应该是`0`。 自动执行求和运算的简单实现将错误地返回`1`在这种情况下。 另一个微妙之处是非常大的数字，例如`n = "9999999999999999999999"`。 正确的算法不应尝试将其直接转换为整数。 

## 方法

 最简单的方法是暴力破解：重复计算数字之和，直到数字只有一位数字。 这是正确的，因为每次转换都会严格减少数字的大小，保证最终终止。 我们可以遍历数字`n`并对它们求和。 这种方法的问题在于，对于具有数百万位数字的数字，如果简单地实现，重复对数字求和可能会很慢，特别是当您在类型之间来回转换数字时。 

最佳方法观察到变换的次数是由我们通过数字和减少多位数的次数决定的。 我们可以不用担心大整数`n`作为字符串。 在第一次迭代中，我们检查是否`n`超过一位数字。 如果是，则对数字求和（单独解析字符）。 然后继续，直到达到个位数。 每次迭代只需要遍历表示的字符串`n`或第一次通过后的小整数。 

这是有效的，因为在第一次转换之后，数字变得足够小，可以作为普通整数进行管理。 不会发生整数溢出，并且我们从不尝试以中间整数形式同时保存超过 1000 万个数字。

 | ---| ---| ---| ---|
 | 蛮力 | O(L * log n) | O(L * log n) | O(L) | 对于大 L 正确但效率低下 |
 | 最佳| O(L) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取号码`n`作为字符串。 这可以避免极大数字的任何整数溢出。 
2. 检查是否`n`已经是个位数了。 如果是，立即返回`0`。 这处理了不需要转换的边缘情况。 
3. 初始化计数器`count`为零。 这将跟踪转换的数量。 
4. 虽然`n`有多于一位数字，将每个字符转换为整数并对所有数字求和。 代替`n`有了这笔钱。 增量`count`一个。 
5.一次`n`是一位数，输出`count`。 

为什么它有效：不变的是每次迭代之后，`n`就位数或其数值而言，变得严格更小。 该过程终止时`n`恰好有一位数字，这保证计数准确地反映了执行了多少次转换。 该算法甚至适用于非常大的初始输入，因为第一个求和将数字字符串减少为可管理的整数，并且后续求和对足够小的数字进行运算以适合标准整数类型。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = input().strip()

if len(n) == 1:
    print(0)
else:
    count = 0
    current = n
    while len(current) > 1:
        total = sum(int(ch) for ch in current)
        current = str(total)
        count += 1
    print(count)
```该解决方案首先将输入读取为字符串以处理非常大的数字。 我们立即检查输入是否只有一位数字来处理这种简单的情况。 在循环内部，`sum(int(ch) for ch in current)`有效计算所有数字的总和，无需将整个字符串转换为整数，从而避免溢出。 求和后，结果将转换回字符串，以便循环条件可以检查位数。 计数器确保转换次数正确。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(L) | 每个数字在第一次迭代中最多被访问一次； 随后的迭代对一个小整数进行操作。 |
 | 空间| O(1) | O(1) | 无论输入大小如何，我们只存储当前的总和和计数器。 |

 即使对于最大 1000 万位数字的输入，该算法也是高效的。 第一次迭代与输入长度呈线性关系，后续迭代可以忽略不计，因为数字很快减少到一位或两位数字。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

    if len(n) == 1:

    count = 0
    current = n
    while len(current) > 1:
        total = sum(int(ch) for ch in current)
        current = str(total)
        count += 1
    return str(count)

 ## 边缘情况

 该算法通过返回正确处理单位数字输入`0`立即地。 对于非常大的数字，例如`"99999999999999999999"`，它以字符串形式有效计算总和，并将其减少为两位数，然后再次求和以达到一位数结果。 输出`2`反映了转换的确切数量。 字符串和整数之间的转换经过仔细控制，以防止任何溢出，确保高达 1000 万位输入的正确性。
