 ## 解决方案
 ## 问题理解

 我们得到了排列成一条线的数字序列，我们想把这条线切成三个连续的段。 第一个片段从开头开始，第二个片段位于中间，第三个片段在最后一个元素处结束。 其中一些片段允许为空，这意味着可以在边界处进行剪切，而不需要任何元素。 

重要的是第一段内的值的总和以及第三段内的值的总和。 我们希望这两个和相等，并且在所有这些有效的数组切割方法中，我们希望该公共和的最大可能值。 

输入大小可以达到二十万个元素，这立即排除了任何独立尝试所有三个段的所有可能切割位置的解决方案。 三重嵌套枚举意味着 n 次方的可能性，这是远远超出可行的。 仅当每个查询都以恒定或对数时间处理时，即使是在切点上的双循环也是可接受的，因此我们应该期望线性或接近线性的解决方案。 

一个微妙的点是允许空段。 这引入了边缘情况，其中一个或两个匹配总和可能为零。 例如，如果所有数字都是正数，则将两个外部段都清空的简单解决方案总是有效，给出答案零，但可能存在更好的分割。 当最佳配置使用非常小的前缀和后缀段时，会出现另一种边缘情况，因此仅在中心附近搜索或假设平衡分割的解决方案会失败。 

在类似的情况下，幼稚的方法常常会失败`[1,2,3,2,1]`。 最佳答案是 4，但是尝试逐步匹配前缀和后缀的贪婪方法可能会过早地提交较小的前缀和并错过较大的匹配后缀。 

## 方法

 暴力解决方案会选择两个切割点`a`和`b`，将第一段定义为`[0..a]`，第二个为`[a+1..b]`，第三个为`[b+1..n-1]`。 对于每一对，我们计算第一段和第三段的总和并检查相等性。 这需要每对进行 O(n) 工作，并且有 O(n²) 对，因此需要 O(n³) 时间。 即使前缀和将段和计算减少到 O(1)，双循环仍然会导致 O(n²)，这对于 200,000 个元素来说太慢了。 

关键的观察结果是，除了确保连续性之外，中间部分是不相关的。 一旦我们选择了一个前缀和，我们只关心是否存在与它不相交的具有相同和的后缀。 这将问题转化为寻找前缀和和后缀和中不重叠的最大值。 

我们可以预先计算所有前缀和。 然后我们从右侧扫描，同时保持后缀和。 在每个位置，我们检查当前后缀和是否作为严格在后缀开始之前结束的前缀和出现。 为了有效地确保正确性，我们使用两指针式遍历：一个指针从左侧移动累积前缀和候选，另一个指针从右侧移动累积后缀和。 然后，我们尝试贪婪地匹配它们，同时保留排序约束。 

这是可行的，因为前缀和和后缀和在累加中都是单调变化的，并且我们正在寻找累加值的相等性而不是任意子数组和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（全部剪切）| O(n2) 到 O(n3) | O(1) | O(1) | 太慢了 |
 | 关于前缀/后缀和的两个指针 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护两个指针，一个从左开始，一个从右开始，以及两个表示当前前缀和和后缀和的运行和。 

1. 初始化`i = 0`,`j = n - 1`,`left_sum = 0`,`right_sum = 0`， 和`answer = 0`。 

指针表示前缀和后缀段的候选边界。 
2. 虽然`i <= j`，延长较小总和的边。 

如果`left_sum`小于或等于，则相加`d[i]`到`left_sum`并移动`i`向前。 否则，添加`d[j]`到`right_sum`并移动`j`落后。 

这种平衡确保我们探索所有可能的均衡点，而不会跳过有效的对齐。 
3. 每次扩展后，检查是否`left_sum == right_sum`。 

如果相等则更新`answer`与这个值。 这将捕获前缀和后缀匹配的有效分割。 
4. 继续直到指针交叉。 

那时，所有可行的不相交前缀后缀对都已以单调方式进行了探索。 

关键思想是我们只增加总和，从不减少总和，因此平等事件在发生时就被完全捕获。 

### 为什么它有效

 该算法保持不变式`left_sum`始终是前缀段和`right_sum`始终是后缀段的总和，并且这些段从不重叠。 每个可能的有效解决方案都对应于指针将数组划分为三个连续部分的某个时刻。 因为我们总是扩展较小的一边，所以我们按照累积的升序模拟所有可行的前缀后缀和对。 不能跳过任何有效的等式，因为任何错过的等式都需要减少一侧，而该过程永远不会这样做。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    i, j = 0, n - 1
    left_sum, right_sum = 0, 0
    ans = 0

    while i <= j:
        if left_sum <= right_sum:
            left_sum += a[i]
            i += 1
        else:
            right_sum += a[j]
            j -= 1

        if left_sum == right_sum:
            ans = left_sum

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案读取数组并使用两个指针来模拟不断增长的前缀和后缀段。 中间段隐式是之间的剩余部分`i`和`j`。 支票`left_sum == right_sum`当两端累计的总数相同时，捕获有效的分割。 

总是延长较小总和的一方的决定保证了我们不会错过潜在的比赛。 如果我们总是任意扩展一侧，我们可以跳过另一侧较早发生相等的配置。

这表明相等可以出现多次并且算法保持最大值。

 ## 边缘情况

 对于像这样的输入`1 2 3`，算法开始扩展两端，但除了零之外从未实现相等，因为后缀和前缀和立即发散。 指针的移动确保两端最终都耗尽，而不会错误地宣布匹配。 

为了`2 2 2 2`，当双方对称增长时，算法会反复达到相等。 每个等式都会更新答案，最终结果反映了可实现的最大分裂。 

为了`1`，算法执行单个步骤，其中 left_sum 变为 1，而 right_sum 保持为 0，导致不相等并正确返回 0。
