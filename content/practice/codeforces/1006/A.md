 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，每个整数都在一个非常大的范围内，直到$10^9$。 Mishka 重复应用全局转换规则，该规则独立作用于每个值：每对连续整数$(1,2)$,$(3,4)$,$(5,6)$等，就地交换，但所有交换都作为单个概念操作同时应用。 

关键细节是该过程不是有意义的迭代。 在完全应用所有交换后，再次重复相同的操作会产生与第一次相同的结果。 这表明变换实际上是对每个元素应用一次的固定排列。 

输入尺寸$n \le 1000$很小，因此任何每个元素以线性甚至恒定时间运行的解决方案就足够了。 重要的是认识到转换的结构，而不是在大范围内逐步模拟它$10^9$，这是不可能的。 

一个天真的误解是尝试模拟所有交换$1$到$10^9$，或重复应用该过程直至稳定。 两者都是不必要的，并且在计算上也是不可行的。 

当考虑从未出现在数组中但位于已处理对之间的数字时，会出现微妙的边缘情况。 例如，如果我们尝试模拟部分映射或仅模拟观察到的值，我们可能会错过变换是全局定义的，并且是按数字独立定义的，而不是按数组内容定义的。 

## 方法

 蛮力解释是从字面上应用所描述的过程：对于每一对$(2i-1, 2i)$，扫描整个数组并交换出现的位置。 所有对的每次完整传递都达到$10^9 / 2$操作，即使我们将自己限制为仅出现在数组中的值，重复的传递仍然是不必要的。 

更具体的暴力简化是：对于数组中的每个值，重复检查它是否属于一对并交换它。 即使这也会导致$O(n \cdot 10^9)$如果在整个域上逐字实现，那么风格的概念复杂性是不可接受的。 

关键的观察结果是该操作纯粹是局部的值空间。 每个整数$x$独立映射：

 - 如果$x$很奇怪，它与$x+1$。 
- 如果$x$是偶数，它与$x-1$。 

因此每个元素的最终值是由单个算术规则决定的，而不是模拟。 整个过程相当于翻转二进制数的最低有效位，因为奇偶配对恰好是在每个连续对内切换奇偶校验。 

因此每个值$x$变换为：

 -$x \rightarrow x-1$如果$x$是偶数
 -$x \rightarrow x+1$如果$x$很奇怪

 这给出了一个$O(n)$直接映射解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·10^9) 概念 | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取整数数组。 不需要预处理，因为每个元素都是独立的。 
2. 对于每个元素$a_i$，判断是奇数还是偶数。 这种分类完全决定了它的最终值，因为变换是连续整数的固定配对。 
3. 如果值为奇数，则替换为$a_i + 1$。 这反映了每个奇数都与下一个整数交换。 
4. 如果值为偶数，则替换为$a_i - 1$。 这反映了每个偶数都与前一个整数交换。 
5. 按照相同的顺序输出变换后的数组。 

重要的结构点是没有元素影响任何其他元素。 没有级联或依赖关系，因此一次传递就足够了。 

### 为什么它有效

 该变换定义了正整数集合的排列，其中每个相邻对恰好交换一次。 每个数字恰好属于一对并映射到其伙伴。 由于这些对划分了整个整数行，因此映射是完全且确定的。 数组结构无关； 只有个人价值观才重要。 应用该规则一次会产生最终状态，再次应用该规则将返回原始配置，确认它是对合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    res = []
    for x in a:
        if x % 2 == 0:
            res.append(x - 1)
        else:
            res.append(x + 1)
    
    print(*res)

if __name__ == "__main__":
    solve()
```该解决方案读取输入一次并独立处理每个值。 奇偶校验直接对交换配对进行编码，避免在大整数范围内进行任何模拟。 由于转换不涉及重新排序，因此输出保留原始顺序。

 ## 边缘情况

 一种边缘情况是具有单个奇值的最小可能数组。 用于输入`1`，变换将其映射到`2`。 由于没有与其他元素交互，因此不会出现隐藏的依赖关系。 

另一个边缘情况是最大值$10^9$。 用于输入`[1000000000]`，它是偶数，所以它映射到`999999999`。 该规则统一适用于允许范围的边界，并且不需要溢出或特殊大小写，因为 Python 整数安全地处理减法并且结果保持在约束范围内。
