 ## 问题理解

 我们得到了一个哲学家列表，每个哲学家都与一个代表他们的“统治力量”的正整数值相关联。 如果一个哲学家的价值可以被另一个哲学家的价值整除，则可以说他统治另一个哲学家，这意味着较大的值必须是较小的值的精确倍数。 

任务是确定是否存在一位哲学家主导列表中的所有其他哲学家。 如果多个候选者满足此条件，我们选择索引最小的一个。 

输入只是哲学家的数量及其价值观。 输出是有效“国王”的从 1 开始的索引，如果不存在这样的哲学家，则输出为 -1。 

约束允许最多 10^9 个值和最多 2·10^5 个哲学家。 这立即排除了任何直接检查每对哲学家的方法，因为这将是 O(n^2)，在最坏的情况下可能达到 4·10^10 次操作，并且明显超出时间限制。 

当多个哲学家共享相同的价值观时，就会出现微妙的边缘情况。 由于任何数字都可以自分，相同的值会形成相互支配，如果该值也支配所有其他值，那么其中最早的索引就成为答案。 例如，如果所有值都是 224，则每个哲学家都支配其他哲学家，因此答案为 1。 

另一个特殊情况是没有一个值可以整除所有其他值，即使它非常大。 例如，在数组 [6, 10, 15] 中，没有数字可以整除其他数字，因此尽管存在多个“大”值，但正确的输出是 -1。 

## 方法

 蛮力方法尝试将每个哲学家作为候选国王，并检查他们的值是否可以被所有其他值整除。 对于每个候选 i，我们扫描所有 j 并验证 Ai % Aj == 0。这总​​共需要 O(n^2) 整除性检查。 每次检查都是恒定时间，但是当 n 达到 2·10^5 时，这变得不可行。 

关键的观察是，有效的王必须是数组中的最大值，因为如果 Ai 支配所有其他值，则 Ai 必须可被每个 Aj 整除，这意味着对于所有 j，Ai ≥ Aj。 因此任何候选者都必须来自最大值集合。 

这大大减少了搜索空间。 我们不检查所有索引，而是只检查那些保持最大值的索引。 由于所有相等的最大值在显性方面表现相同，因此我们只需要测试其中一个，最好是第一次出现。 

然后我们验证这个最大值是否可以整除数组中的每个元素。 如果是的话，那就是一位有效的国王。 否则，就不存在候选人。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 最大候选人检查 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为测试单个值是否可以整除所有其他值。 

1. 扫描一次数组，找到最大值及其第一个索引。 原因是任何有效的国王必须至少与所有其他国王一样大，因此只有最大值才符合条件。 
2. 再次遍历数组，检查每个元素是否能被最大值整除。 如果我们发现任何不可整除的元素，我们可以立即拒绝这个候选元素，因为支配性要求涵盖每个哲学家。 
3. 如果所有检查都通过，则返回第一个最大元素的存储索引。 
4. 如果任何检查失败，则返回-1。 

关键的决定是将注意力限制在最大元素上，从而消除了成对比较的需要。 

### 为什么它有效

如果值 A_k 支配所有其他值，则对于每个 i，对于某个整数 m，我们必须有 A_k = m · A_i，这意味着对于所有 i，A_k ≥ A_i。 这迫使 A_k 成为全局最大值。 在相等的最大值中，整除性是相同的，因此选择最小的索引可以确保平局规则下的正确性。 因此，该算法直接测试唯一可能的结构候选并验证所需的整除条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    mx = max(a)
    idx = -1
    
    for i in range(n):
        if a[i] == mx:
            idx = i + 1
            break
    
    for x in a:
        if mx % x != 0:
            print(-1)
            return
    
    print(idx)

if __name__ == "__main__":
    solve()
```第一遍计算最大值并记录其最早位置。 这是必要的，因为输出取决于索引，而不仅仅是值。 

第二遍通过检查候选最大值到每个元素的整除性来强制执行支配条件。 条件 mx % x == 0 是 mx 是每个 x 的倍数这一要求的直接翻译。 

失败时尽早退出可以防止不必要的检查，从而保持解决方案的线性。

 ## 边缘情况

 当所有值都相同时，最大值会出现多次。 该算法选择第一个出现的位置，并且仍然通过整除性检查，因为每个数字都可以整除自己。 例如，[224, 224, 224] 生成 mx = 224 并且第二遍中没有失败，因此返回索引 1。 

当最大值出现多次但其他值破坏整除性时，结果仍然是-1。 例如，[10, 10, 3] 的 mx = 10，但 10 % 3 失败，因此没有候选者生存。 

当数组包含 1 时，除非所有值都是 1，否则它不会自动成为答案，因为 1 只占主导地位。 例如，[1, 2, 3] 失败，因为 1 % 2 在主导的相反方向上无效。
