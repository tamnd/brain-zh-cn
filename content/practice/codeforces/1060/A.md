 ## 解决方案
 ## 问题理解

 我们有一组多组数字卡，每张卡包含一个从 0 到 9 的字符。我们希望从这些卡中组合出尽可能多的有效电话号码。 有效的电话号码具有固定的结构：它由 11 位数字组成，并且第一位数字必须为 8，其余 10 位可以填写任意数字。 

每张卡最多可以使用一次，并且只要我们有足够的卡来独立构建每个副本，我们就可以在概念上多次重复使用相同的电话号码模式。 

任务是确定可以从可用数字组成的以 8 开头的 11 位字符串的最大数量。 

约束 n ≤ 100 足够小，即使是二次或简单的贪婪计数方法也是安全的。 这立即排除了对复杂数据结构或组合的任何需要。 我们纯粹处理频率会计。 

一个关键的观察结果是，唯一的特殊要求是前导 8。每个电话号码恰好消耗一次数字 8，总共 11 张卡。 其他一切都只是一堆填充数字。 

当根本没有 8 时，就会出现边缘情况。 在这种情况下，无论存在多少其他数字，都无法形成有效的电话号码。 例如，输入`11111111111`产量 0。 

另一个微妙的情况是总位数足够但 8 不足。 例如，`888`重复使用太少的其他数字仍然会严格限制可用 8 的数量。 

最后，如果8很多，但总位数太少，则总容量占主导地位：即使每个电话号码需要8，但还需要10位数，所以剩下的8在没有数字支持的情况下毫无用处。 

## 方法

 思考这个问题的一种蛮力方法是尝试一个接一个地构造电话号码。 我们重复扫描当前的多组数字，检查是否可以选择一个 8 和任何其他 10 个数字，然后删除这些数字并继续。 每次尝试都会花费 O(11) 扫描或簿记成本，并且我们可能会重复 O(n) 次，从而导致 O(n²) 行为。 当 n ≤ 100 时，这已经很好了，但没有必要。 

问题的结构揭示了一个更简单的约束系统。 每个电话号码正好消耗 11 张卡，而这 11 张卡中，必须有一张是 8。因此，如果我们将 8 的数量表示为 c8，将总位数表示为 n，则电话号码 k 的数量必须同时满足 k ≤ c8 和 11k ≤ n。 第二个条件来自于卡消费总额，第一个条件来自于每个号码一个8的固定要求。 

因此，最佳解决方案只是这两个限制因素中的最小值。 这将问题简化为计算频率和采取简单的界限。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力建造| O(n²) | O(1) | O(1) | 已接受 |
 | 计数+最小约束| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算数字 8 在输入中出现的次数。 从所需的前导数字的角度来看，这表示电话号码的最大可能数量。 
2. 计算总共有多少张完整的 11 张牌组，即 n // 11。这表示可用牌总数强加的绝对上限。 
3. 答案是这两个值中较小的一个，因为每个有效的电话号码必须同时满足这两个约束。 
4. 直接输出该值。 

### 为什么它有效

 每个有效的电话号码恰好消耗 11 张不同的卡片，并且其中一张必须是 8。任何有效的构造都可以映射到电话号码和 11 张卡片的不相交组之间的配对，每个组在其第一个位置至少包含一个 8。 因此，任何解决方案都不能超过 8 的总数或不相交的 11 卡块的数量。 在不违反这些约束的情况下贪婪地使用卡片的构造总是达到最小界限，因为除了计数之外没有其他结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()
    
    c8 = s.count('8')
    
    print(min(c8, n // 11))

if __name__ == "__main__":
    solve()
```该实现首先读取卡片的数量和数字字符串。 然后，它会计算“8”的出现次数，这会直接控制由于强制前导数字限制而可以形成多少个电话号码。 第二个约束来自总长度：每个电话号码占用 11 位数字，因此 n // 11 是完整电话号码的最大可能数量，而与数字组成无关。 最终的答案是这两个极限中的最小值。 

不需要排序问题或贪婪选择，因为问题不依赖于排列，只依赖于计数。 

## 工作示例

 ### 示例 1

 输入：```
11
00000000008
```我们计算 c8 = 1 和 n // 11 = 1。 

| 步骤| c8 | // 11 | n // 11 | 当前答案 |
 | --- | --- | --- | --- |
 | 初始| 1 | 1 | 最小值(1, 1) = 1 |

 结果是 1，这意味着我们可以使用所有数字恰好形成一个有效的电话号码。 

这证实了即使大多数数字都是零，当总长度仅支持一组时，单个 8 就足够了。 

### 示例 2

 输入：```
22
8888888888888888888888
```这里 c8 = 22 且 n // 11 = 2。 

| 步骤| c8 | // 11 | n // 11 | 当前答案 |
 | --- | --- | --- | --- |
 | 初始| 22 | 22 2 | 分钟(22, 2) = 2 |

 尽管有很多 8，但总长度限制我们只能使用 2 个电话号码。 这证明了分组约束的主导地位。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    
    n = int(sys.stdin.readline().strip())
    s = sys.stdin.readline().strip()
    
    return str(min(s.count('8'), n // 11))

# provided sample
assert run("11\n00000000008\n") == "1", "sample 1"

# no 8 at all
assert run("11\n11111111111\n") == "0", "no valid phone numbers"

# exactly one full valid set
assert run("11\n81234567890\n") == "1", "single possible number"

# too few digits for even one number
assert run("10\n8888888888\n") == "0", "insufficient length"

# many 8s but limited by length
assert run("22\n8888888888888888888888\n") == "2", "length constraint dominates"
```| 测试输入| 预期产出 | 它验证了什么 |
 | --- | --- | --- |
 | 11 位数字，无 8 | 0 | 缺少所需的前导数字 |
 | 混合有效集 | 1 | 正常施工|
 | 10 位数字 8 | 0 | 长度限制阻止形成 |
 | 许多 8 | 2 | min(n//11, c8) 行为 |

 ## 边缘情况

 对于没有数字8的输入，例如`11111111111`，算法计算 c8 = 0 并立即返回 0。 这符合以下事实：甚至无法以有效的电话号码开头。 

对于 n < 11 的输入，例如`8888888888`, n // 11 是 0，所以无论有多少个 8，结果都是 0。 这正确地捕获了形成任何完整电话号码的可能性。 

对于像这样的输入`8888888888888888888888`其中 n = 22、c8 = 22 且 n // 11 = 2，因此输出为 2。该算法正确地限制了总可用分组容量，而不是过度使用数字 8。
