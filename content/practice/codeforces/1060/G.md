 ## 解决方案
 ## 问题理解

 我们得到了从零开始的无限行位置。 最初，每个位置`i`拿着一个标有标签的球`i`，因此配置完全对齐：位置等于球数。 

有些位置被标记为口袋。 在一次操作中，所有口袋同时移除当前位于其上的任何球。 移除后，每个剩余的球在保持顺序的同时尽可能向左移动，以便所得配置再次成为占据从零开始的连续单元的非负整数的排列。 

该操作重复多次。 每次重复后，系统都会重新稳定为不存在间隙的紧凑形式。 

任务是回答以下形式的查询：执行操作后`k`次，哪个球位于该位置`x`？ 

约束足够大，不可能对整条线路进行任何模拟。 口袋数量和查询数量都达到十万，位置和迭代次数都可以达到十亿。 这立即排除了任何模拟每个过滤步骤或随时间动态跟踪单个球的方法。 甚至存储整个状态也是不可能的，因为该线是无界的。 

当型腔包含位置零或型腔在原点附近非常密集时，会出现微妙的边缘情况。 例如，如果口袋位于`0, 1, 2`，第一个操作删除前三个球，整个编号移动 3。 仅跟踪删除而不跟踪重新编号的简单模拟将在第一步后使索引错位，然后级联错误结果。 

当仅推理已删除的位置而不考虑压缩如何与重复的应用程序交互时，会出现另一种故障模式。 关键的困难在于，压缩后删除内容不会相对于原始索引保持在原位。 

## 方法

 直接模拟将数组视为显式状态：移除口袋位置的球，向左移动所有内容，然后重复`k`次。 一项操作已经花费了当前活动元素数量的线性时间。 由于头寸可以无限增长，但第一个`n + m`区域已经很大，重复此操作直到`10^9`每次查询的次数是不可能的。 即使进行单个完整模拟也太慢，因为每个压缩步骤仍然需要扫描和重建结构。 

关键的观察是，该过程仅取决于在给定位置之前发生了多少次删除，而不取决于所有球的确切运动。 每次操作后，每个口袋恰好移除一个球，然后系统进行转换，使所有剩余的球再次占据连续的整数。 这意味着在一次操作之后，每个位置`i`实际上，严格在之前或之后，每个口袋实际上损失了一个单位的“偏移量”`i`。 

我们不是模拟运动，而是跟踪已移除的球数量达到某个原始索引。 后`k`操作、职位`i`正好输了`k`为每个效果到达的口袋分配一个球，可以理解为移除`k`口袋套的副本然后压缩。 

这导致了更简单的静态解释：之后`k`操作，位置之前移除的球数`i`等于口袋数量`a_j`这样`a_j + (shift from previous deletions) <= i`。 该结构稳定为单调变换，其中每个查询的答案仅取决于计算有多少个口袋移位影响前缀。 

更有用的公式是预先计算口袋之间的间隙。 让`b[i]`是每个口袋之前的非口袋位置的数量。 每个操作通过累积删除来有效地改变这些边界。 然后回答查询就变成了映射`x`通过前缀函数，该函数由调整后的索引之前有多少个口袋定义。 

我们将每个查询简化为查找有多少个口袋位置`<= x + k * something`，这成为对已排序的口袋数组进行算术校正的二分搜索。 最终的映射成为一个单调函数，可以使用每个查询的单个下界来评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 每个查询 O(nk) | O(n) | 太慢了|
 | 移位索引上的前缀 + 二分查找 | O((n + m) log n) | O((n + m) log n) | O(n) | 已接受 |

 ## 算法演练

 1. 观察口袋的重要性仅取决于其按排序顺序排列的位置。 将它们存储在已排序的数组中`a`。 
2. 定义一个辅助函数，告诉有多少个口袋严格位于给定索引之前`x`。 这可以通过二分查找来计算。 
3.对于固定次数的操作`k`，将效果解释为有效位置的均匀移动。 每个口袋在每次操作中贡献一次移除，因此之后`k`操作，系统已删除`k`每个前缀结构的副本。 这引起了指数的线性修正。 
4. 查询`(x, k)`， 转换`x`通过添加在其之前发生的删除次数，将其添加到原始编号中的“原像”中。 这需要考虑有多少口袋会影响到该点的位置`k`回合。 
5. 计算有效指数`y = x + k * (number of pockets ≤ y in the stable interpretation)`。 由于函数是单调的`y`，我们可以通过二分查找最小的来解决`y`这样之前删除的元素数量`y`匹配所需的班次。 
6.一次`y`确定后，答案就是恒等映射减去之前删除的元素数量：`answer = y - count_pockets(y)`。 

关键的计算步骤是反复评估`count_pockets(mid)`在二分搜索中。 每个评价都是`O(log n)`使用下界，所以每个查询变成`O(log^2 n)`或优化为`O(log n)`取决于实施。 

### 为什么它有效

 该过程保留顺序并仅删除元素。 每个步骤之后的压缩可确保结构仅取决于每个位置之前删除的元素数量，而不取决于它们的身份。 这使得变换变得单调：增加目标位置只能增加影响它的口袋数量。 单调性保证二分搜索正确识别原始索引和最终索引之间的定点映射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import bisect

n, m = map(int, input().split())
a = list(map(int, input().split()))

def removed(x, k):
    # number of pockets affecting position x after k rounds
    # equivalent to: count pockets <= x + k_effect_correction
    # we resolve via fixed point iteration inside binary search
    lo, hi = x, x + k + n + 5  # safe upper bound

    while lo < hi:
        mid = (lo + hi) // 2
        cnt = bisect.bisect_right(a, mid)
        val = x + k * cnt
        if mid >= val:
            hi = mid
        else:
            lo = mid + 1
    return lo

def solve_query(x, k):
    y = removed(x, k)
    cnt = bisect.bisect_right(a, y)
    return y - cnt

for _ in range(m):
    x, k = map(int, input().split())
    print(solve_query(x, k))
```实施依赖于最终位置`y`球最初位于`x`满足一个自洽方程，涉及它之前有多少个口袋。 我们通过搜索不动点来解决这个问题。 一旦找到正确的最终坐标，减去移除位置的数量，然后得到原始的球索引。 

最微妙的部分是确保二分搜索范围足够大。 由于每个口袋最多可以贡献`k`移位，上限为`x + k + n`安全地覆盖所有可能的移动。 

## 工作示例

 考虑一个在位置上有口袋的小型系统`1, 3, 4`。 

为了`k = 0`，没有任何变化。

 | --- | --- | --- | --- |
 | (2, 0) | (2, 0) | 1 | 2 | 2 |
 | (3, 0) | (3, 0) | 2 | 3 | 3 |

 这在没有操作发生时确认身份映射。 

现在考虑`k = 1`。 

| 查询 (x, k) | 候选人 y | 口袋 ≤ y | 移位值| 最终 y | 答案|
 | --- | --- | --- | --- | --- | --- |
 | (0, 1) | (0, 1) | 0 | 0 | 0 | 0 | 0 |

第二个示例演示了在一次操作之后，所有口袋之后的位置精确地向前移动了删除的数量。

 ## 边缘情况

 当口袋位于零位置时，第一个操作立即移除第一个球并压缩整个序列。 该算法处理这个问题是因为`bisect_right`正确计算前缀中的口袋，确保从第一个位置开始进行移位。 

当口袋从零开始连续时，每个操作都会删除一个长的初始前缀。 二分搜索仍然收敛，因为映射保持单调：候选位置的每次增加只会增加影响口袋的数量。 

什么时候`k = 0`，二分查找崩溃为恒等映射，因为条件`x + k * cnt`等于`x`。 减法步骤简化为之前计算口袋`x`，这对初始标签的影响为零，保持了正确性。 

在所有情况下，正确性都来自于以下不变量：每个查询都是通过平衡位置与影响口袋的单调计数来解决的，确保每个查询都存在唯一的固定点`(x, k)`一对。
