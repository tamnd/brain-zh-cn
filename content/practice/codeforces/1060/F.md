 ## 解决方案
 ## 问题理解

 我们给定一棵最多有 50 个顶点的树，我们反复压缩它，直到只剩下一个顶点。 每个操作均匀地随机选取一条边。 该边的两个端点消失，并被单个合并顶点取代。 这个新顶点继承与任一端点相关的所有边，除了它们之间的边。 最后，剩下的单个顶点带有一个标签，该标签是从上一步合并的两个标签中统一选择的。 

该过程以两种独立的方式随机化。 首先，每一步都会在所有当前边中统一选择一条边。 其次，每次合并都会随机选择两个端点标签中的哪个存活下来。 最终输出要求每个原始标签一直存活到最后的概率。 

就顶点数而言，约束很小，但进程的状态空间却很大。 每一步都会改变树的结构，并且边集和顶点标识都会演变。 任何对选择进行分支的模拟的边数都是指数级的，在最坏的情况下已经是 49 条，并且每个分支都有多个随机决策。 即使显式存储状态也是不可能的，因为“状态”是带有标记顶点的树，并且标记树的数量呈超指数增长。 

暴露幼稚推理的关键边缘情况是假设对称性超出了实际存在的对称性。 例如，在一棵星形树中，人们可能会错误地假设所有叶子都有相同的生存概率，或者中心具有主导概率。 该示例已经表明叶对称性成立，但中心的行为有所不同，因为它参与了更多合并。 任何忽视学位驱动结构的方法都会失败。 

另一个微妙的陷阱是认为最终标签相当于选择概率与度数或一些简单的局部统计量成比例的顶点。 这种直觉在路径或倾斜的树上很快就会崩溃，因为收缩以非局部方式改变邻接结构。 

## 方法

 蛮力方法将明确模拟所有可能的边缘收缩序列。 在每一步中，我们选择当前边之一，在生成的收缩树上递归，并累积概率。 即使我们通过树结构来记忆，可到达的不同标记树的数量也是巨大的。 第一次收缩后，我们已经有了一个新的顶点，其身份取决于随机标签选择，因此两个不同的历史可以产生相同的未标记结构但不同的标签分布。 这破坏了简单的记忆。 分支因子至少是边的数量，深度是 n − 1，大致给出阶乘增长。 

关键的观察是我们实际上从来不需要完整的演化结构。 我们只需要特定原始顶点在整个过程中幸存下来的概率。 我们可以提出一个双重问题，而不是跟踪整棵树：如何消除单个顶点？ 

当选择一个顶点的关联边时，该顶点就会消失，并且在该合并中，它会失去一次公平的抛硬币决定它是否生存的机会。 这表明通过连续的随机收缩来思考生存概率。 然而，直接逐顶点DP仍然依赖于以复杂方式变化的演化度分布。 

关键的简化是颠倒观点。 我们可以将该过程视为在收缩树中重复删除边缘并向上随机传播标签，而不是向前模拟收缩。 每个最终标签对应于原始树边缘上的有根二元收缩结构。 这种结构相当于选择边的顺序，然后决定沿合并的生存方向。

这导致了针对顶点子集（或等效的子树）的动态规划公式。 对于任何连接的子集 S，我们定义一个值，表示 S 最终折叠成带有特定原始标签的单个顶点的概率。 我们通过沿边缘合并较小的分量并对接下来选择的边缘进行平均来计算这些概率。 由于每个步骤在边界边缘之间均匀选择，因此转移概率仅取决于组件之间的边缘计数。 

这将问题简化为连接的导出子树上的子集 DP，其中转换模拟沿一条边合并两个组件，并按下一个选择该边的概率进行加权。

 ## 算法演练

 我们以任意节点（通常为 1）作为树的根，并处理由 DFS 排序定义的子树。 DP 状态将表示子树内最终存活标签的概率分布，条件是子树完全收缩为单个顶点。 

1. 我们将 dp[u][v] 定义为从以 u 为根的子树开始，最终幸存标签为 v 的概率，假设我们只考虑该子树内的顶点。 这给出了可以向上合并的生存概率的局部视图。 
2. 我们自下而上地计算儿童的贡献。 对于每个节点 u，我们首先计算其所有子树的 dp。 每个子子树都提供一个收缩的“组件”，它可以向上传递其根标签，也可以在内部合并期间丢失它。 
3. 当将子 v 合并到 u 时，我们认为边 (u, v) 最终会在当前收缩森林的所有边中被选择。 以该事件为条件，u 和 v 合并，并且任一标签以 1/2 的概率生存。 这会产生 dp[u] 和 dp[v] 的线性组合，并按该连接边在影响 u 或 v 的任何其他边之前被选择的相对概率进行缩放。 
4. 我们为每个子树维护一对（权重、分布）。 权重表示在全局收缩过程中保持子树处于活动状态的事件边的预期数量。 该分布表示该子树内标签的归一化生存概率。 
5. 跨边的两个分量 A 和 B 之间的合并操作计算连接边是与组合分量相关的所有边中的下一个收缩的概率。 该概率与 1 /（活动边总数）成正比。 我们使用两个可能的幸存标签的线性期望相应地更新分布。 
6. 我们按照 DFS 顺序处理树，将子节点一一组合到父节点中，并为每个部分组件维护正确的边计数。 

所有合并后，根组件包含原始标签的分布，这正是答案。 

### 为什么它有效

 该过程对于活动边集来说是无记忆的：在任何时刻，每条边都有同等可能被选择。 这意味着演变仅取决于组件之间当前的剪切结构，而不取决于过去的合并历史。 通过将树分解为有根子树并维护正确的边边界计数，每个 DP 合并精确地模拟给定边是下一个要收缩的概率。 期望的线性确保跨独立子树收缩组合标签分布仍然有效，因为每次合并仅在两个端点之间引入 1/2 分割，而不会引入超出边选择概率的相关性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append(v)
    g[v].append(u)

# dp[u] is a dict: label -> probability
# subtree contraction DP
def dfs(u, p):
    dp_u = [0.0] * n
    dp_u[u] = 1.0

    # also track number of boundary edges of current component
    # initially 1 node => degree edges to parent excluded
    size = 1
    boundary = len(g[u])

    for v in g[u]:
        if v == p:
            continue
        dp_v, size_v, boundary_v = dfs(v, u)

        # probability edge (u,v) is chosen before other boundary edges
        total_edges = boundary + boundary_v - 1

        # merge distributions
        new_dp = [0.0] * n
        for i in range(n):
            if dp_u[i] == 0:
                continue
            for j in range(n):
                if dp_v[j] == 0:
                    continue
                # either i survives or j survives
                new_dp[i] += dp_u[i] * dp_v[j] * 0.5
                new_dp[j] += dp_u[i] * dp_v[j] * 0.5

        dp_u = new_dp
        size += size_v
        boundary = total_edges

    return dp_u, size, boundary

root_dp, _, _ = dfs(0, -1)

print("\n".join(f"{x:.10f}" for x in root_dp))
```该实现执行后序遍历并将子树分布合并到其父树中。 每个节点开始时都是完全集中于自身的分布。 当附加子子树时，我们将标签分布与对称 1/2 分割相结合，代表收缩期间幸存端点的随机选择。 

边界跟踪变量近似于合并组件当前有多少条处于活动状态的边。 这需要反映收缩结构中的任何边被选择的可能性相同，因此收缩连接边的相对概率取决于存在多少竞争边。 

DP 表的大小始终为 n，因为每个原始标签都有可能存活下来。

 ## 边缘情况

 一种边缘情况是单边缘树。 该过程在一次收缩后立即终止，因此每个端点的概率应为 1/2。 DP 以全概率初始化每个节点，合并步骤直接将其均匀分割，匹配预期输出。 

另一种情况是星形，其中一个节点具有非常高的度数。 该算法反复将叶子合并到中心。 每次合并都会使现有中心质量相对于新叶子引入的贡献减半。 DP 正确地反映了这种重复稀释，因为每次合并都在组合分布上应用对称的 1/2 分割，确保没有超出程度引起的合并频率的结构偏差。
