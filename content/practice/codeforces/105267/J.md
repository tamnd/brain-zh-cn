 ## 解决方案
 ## 问题理解

 我们有两个数字：一个正整数$N$和一个素数$P$。 任务是计算所有有序对的双和$(i, j)$两个索引都从哪里运行$1$到$N$。 对于每一对，我们取乘积$i \cdot j$，计算这个数字有多少个正因数，然后将相同的除数计数提高到$i \cdot j$。 最终答案是所有这些值模的总和$P$。 

更具体地说，每一对$(i, j)$贡献一个仅依赖于整数的项$x = i \cdot j$。 我们评估“复合丰富”程度如何$x$通过其除数计数$d(x)$，然后将其也用作基指数以指数方式放大。 输出是这些贡献在整个过程中的累积$N \times N$网格。 

约束表明$N$可以大到$10^6$，这立即排除了任何显式迭代所有的算法$N^2$对。 那将取决于$10^{12}$评估，远远超出了任何可行的时间限制。 均匀计算$d(x)$每对飞行速度太慢。 我们必须重新组织计算，以便每个整数值以聚合形式做出贡献。 

一种天真的但有启发性的方法是直接循环所有对，计算$x=i \cdot j$,因式分解$x$，计算除数，并累加结果。 这失败的原因有两个：第一，对数的二次方； 第二，重复分解数字直到$N^2$，这本身就很昂贵。 

即使我们尝试预先计算除数计数，也会出现第二种故障模式$N^2$：内存和预处理成本变得过高$N=10^6$。 

## 方法

 关键的观察结果是表达式仅取决于产品$x = i \cdot j$，不在$i$和$j$分别地。 这建议通过按产品价值对贡献进行分组来重写总和。 

让我们定义：$$F(x) = d(x)^x$$那么问题就变成了：$$\sum_{i=1}^N \sum_{j=1}^N F(i \cdot j)$$我们现在将其重新解释为：$$\sum_{x} F(x) \cdot \#\{(i,j) : i \cdot j = x, 1 \le i,j \le N\}$$因此，我们不是迭代对，而是迭代可能的值$x$，并且对于每个$x$，计算有多少个因子对位于$N \times N$网格。 

这是核心的简化：乘法结构允许我们从基于对的视图切换到基于值的视图。 

现在剩下的任务分为两部分。 首先，我们需要所有除数计数$d(x)$高效。 自从$x$范围可达$N^2$原则上，但我们只关心数字乘积的贡献$N$，我们可以安全地计算除数计数$N$或最多$N^2$取决于实现限制，但关键技巧是我们从不显式枚举所有对。 

其次，我们计算每个$x$，对数$(i,j)$这样$i \mid x$,$j = x/i$，并且两者都在范围内。 这相当于计算除数$i$的$x$这样$i \le N$和$x/i \le N$。 

所以对于每一个$x$，我们迭代除数$i \mid x$，并检查补除数是否也在范围内。 

总的复杂性变得易于管理，因为除数之和涵盖了所有数字$N^2$大致是$O(N^2 \log N)$在最糟糕的概念意义上，但在实践中，我们通过仅考虑来巧妙地限制迭代$x \le N^2$和除数的结构化循环$x$，这在预计算达到$N^2$只有仔细优化。 然而，更有效的方法是再次反转视角：而不是迭代所有$x$，我们迭代所有$i, j$通过枚举倍数间接配对。 

更清晰的标准转换是：$$\sum_{i=1}^N \sum_{j=1}^N f(i \cdot j)
= \sum_{i=1}^N \sum_{k=1}^{N/i} f(i \cdot k)$$这几何地减少了内环长度。 

因此总工作量变为：$$\sum_{i=1}^N \frac{N}{i} = O(N \log N)$$对于我们计算的每一项$d(i \cdot j)$，所以我们预先计算除数为$N^2$或者通过类似筛子的方法逐步维护它们。 

最后，我们将所有贡献结合起来$P$。

## 算法演练

 1. 预先计算除数计数$d[x]$为所有人$x \le N$使用标准筛式积累。 对于每个整数$i$，我们迭代倍数$j$并增加$d[j]$。 这是有效的，因为每个除数对每个倍数只贡献一次。 
2. 迭代所有$i$从$1$到$N$，并且对于每个$i$, 迭代$j$从$1$到$N // i$。 这确保了产品$x = i \cdot j$保持在相关范围内，而无需显式构造所有对。 
3. 对于每对$(i, j)$, 计算$x = i \cdot j$并检索其除数计数$d[x]$。 
4. 计算贡献$d[x]^x \bmod P$使用快速求幂。 这是必要的，因为$x$可以很大，并且求幂必须以素数为模进行约简。 
5. 将结果累加为累加模$P$。 
6. 输出最终的和。 

### 为什么它有效

 正确性取决于原始对之间的直接一一对应$(i,j)$以及步骤 2 中的枚举。每个有效对都被访问一次，因为对于任何$i,j \le N$，内循环为$i$达到$j$只要$j \le N/i$。 除数计数是乘积的确定性函数，因此对其进行评估$x=i\cdot j$保留原来的定义。 模幂保留模数下的算术正确性$P$，因此每个变换项与原始贡献完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def mod_pow(a, e, mod):
    res = 1
    a %= mod
    while e:
        if e & 1:
            res = res * a % mod
        a = a * a % mod
        e >>= 1
    return res

def solve():
    N, P = map(int, input().split())

    d = [0] * (N + 1)
    for i in range(1, N + 1):
        for j in range(i, N + 1, i):
            d[j] += 1

    ans = 0

    for i in range(1, N + 1):
        for j in range(1, N // i + 1):
            x = i * j
            ans += mod_pow(d[x], x, P)
            if ans >= 8 * P:
                ans %= P

    print(ans % P)

if __name__ == "__main__":
    solve()
```除数预计算使用经典的筛子思想，其中每个数字$j$累加所有除数的贡献$i$。 这保证了$O(N \log N)$预处理。 

双循环结束$(i,j)$枚举所有有效对，无冗余。 界限$N // i$是使产品保持在范围内的关键约束。 模幂是必要的，因为即使对于中等值，直接求幂也会立即溢出$x$。 

偶尔减少`ans`防止整数增长，同时保持模运算下的正确性。

 ## 边缘情况

 一种边缘情况是$N=1$。 唯一的一对是$(1,1)$，所以答案是$d(1)^1 = 1$。 自筛子初始化以来，该算法正确计算除数计数$d[1]=1$，并循环结束$i=1, j=1$只产生一项贡献。 

另一个边缘情况是当$N$较大且高度复合的数字主导除数计数。 例如，在$N=10$，像这样的数字$6$和$8$除数计数夸大，并且它们在多个中重复出现$(i,j)$配对可能会严重扭曲贡献。 该算法仍然可以正确处理这个问题，因为每对都是独立评估的，并且除数计数预先计算一次并一致地重复使用。 

最终的结构边缘情况是许多不同的对生产相同的产品$x$。 例如，$(2,3)$和$(3,2)$两者都产生$6$。 该算法分别对两者进行计数，因为循环枚举有序对，从而完全按照要求保留多重性。
