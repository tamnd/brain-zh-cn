 ## 问题理解

 我们正在模拟四队足球队的最后一轮比赛。 三场比赛已经确定：中国队已经和泰国队打了两场比赛，现在只剩最后一轮了，包括中国队对阵韩国队、泰国队对阵新加坡队。 唯一的未知数是这两场比赛的最终比分，以四个整数给出：韩国得分为a，中国得分为b，泰国得分为c，新加坡得分为d。 

根据这两个结果，我们必须重建所有四支球队的最终小组积分榜。 每队以标准方式累积积分：胜三分，平一分，负零分。 排名首先根据总积分，然后是净胜球数，最后是总进球数来确定。 如果此后球队仍然平局，则会应用面对面的子排名，但问题保证我们永远不需要比最后一步更进一步。 

关键的难点在于，中泰两国在此前的交锋中已经部分打平，最终的排名可能需要通过详细的决胜规则来解决平局。 经过所有计算，我们必须决定中国队是否能进入小组前两名。 

约束非常小，所有分数都在 0 到 10 之间。这消除了性能方面的任何优化需求。 即使对所有可能的结果进行暴力模拟也是可行的。 真正的挑战是正确执行排名逻辑，尤其是中国和泰国之间的抢七，这取决于有限的比赛子集。 

当中国队和泰国队在积分、净胜球和总进球数上打成平手时，就会出现一个微妙的边缘情况，迫使双方进行正面比较。 在这种情况下，只有涉及这两支球队的比赛才重要，忽略这一限制会导致排名不正确。 

## 方法

 解决这个问题的直接方法是应用上一轮结果后完全模拟小组积分榜。 我们使用已知的之前结果加上两场决赛来计算所有四支球队的得分、进球数和失球数。 一旦我们有了完整的统计数据，我们就会根据排名规则对球队进行排序。 

当两个或更多团队在所有主要标准上都相同时，就会出现复杂情况。 在这种情况下，我们不能简单地依赖全球统计数据； 我们必须重新计算一个迷你表，只考虑平局球队之间进行的比赛。 在这个问题上，唯一能影响中国队出线的有意义的平局是中国队和泰国队之间。 

暴力方法可以通过重复对平局的球队进行分组并重新计算子表统计数据直到解决所有平局来明确模拟平局决胜过程。 由于只有四个团队，这在计算上仍然微不足道，但它不必要地复杂。 

关键的观察是我们不需要完整的递归平局解析系统。 我们只需要正确计算：

 1. 所有四支球队的全球排名。 
2. 如果中国和泰国在全球标准上打成平手，请比较他们的面对面迷你表。 

这将问题简化为简单的模拟加上一个条件决胜局评估。

 ## 算法演练

 我们首先明确地重建比赛结果，然后评估排名。 

1. 将所有四支球队的积分、进球数和失球数初始化为 0。 这为聚合设置了一个干净的基线。 
2. 添加已知的过去结果：

中国队与泰国队有两场比赛，一场2:1胜，一场1:1平。 根据这些数据，我们更新两支球队的得分和进球统计数据。 该步骤是固定的，不依赖于输入。 
3. 应用输入的最后一轮结果：

 韩国对中国的比赛为韩国贡献 a 进球，为中国贡献 b 进球，根据胜利、失败或平局更新两支球队的进球数、失球数和积分。 

泰国与新加坡对 c 和 d 的贡献类似。 
4. 计算每个团队：

 总积分、净胜球和总进球数。 这些是主要的排名关键。 
5. 使用这些键按字典顺序对四支球队进行排序：首先是得分，然后是净胜球数，然后是进球数。 
6、确定本次排名前两名的球队为临时预选赛球队。 
7. 如果中国已经进入前两名，则立即输出YES。 
8. 如果不是，请检查中国和泰国在所有全球标准（积分、净胜球、进球数）上是否持平。 如果不相等，则输出NO。 
9. 如果它们在全球范围内打成平手，则仅使用两个已知结果（2:1 和 1:1）不加任何其他内容来构建中国和泰国之间的比赛的交锋表，因为它们之间不存在其他比赛。 
10. 使用相同的排名规则在此迷你表中比较中国和泰国。 如果中国排名较高，输出YES； 否则输出NO。 

正确性取决于当全球排名不足以区分中国和泰国时正确检测，然后切换到限制比较集。 

它之所以有效，是因为只有当全球统计数据无法区分球队时，国际足联式的平局决胜法才能将面对面的结果分开。 由于只有中国和泰国可能需要此配置中的第二阶段，因此我们永远不需要模拟该对之外的任意子集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def add(team, gf, ga, pts, g, a):
    team["gf"] += gf
    team["ga"] += ga
    team["pts"] += pts

def result_points(x, y):
    if x > y:
        return 3, 0
    if x == y:
        return 1, 1
    return 0, 3

def key(t):
    return (t["pts"], t["gf"] - t["ga"], t["gf"])

def better(a, b):
    if key(a) != key(b):
        return key(a) > key(b)
    return False

china = {"pts": 0, "gf": 0, "ga": 0}
korea = {"pts": 0, "gf": 0, "ga": 0}
thailand = {"pts": 0, "gf": 0, "ga": 0}
singapore = {"pts": 0, "gf": 0, "ga": 0}

# known matches China vs Thailand
def apply(a, b, x, y):
    pa, pb = result_points(x, y)
    add(a, x, y, pa, 0, 0)
    add(b, y, x, pb, 0, 0)

apply(china, thailand, 2, 1)
apply(china, thailand, 1, 1)

a, b, c, d = map(int, input().split())

apply(korea, china, a, b)
apply(thailand, singapore, c, d)

teams = [china, korea, thailand, singapore]
teams.sort(key=lambda t: (t["pts"], t["gf"] - t["ga"], t["gf"]), reverse=True)

top2 = teams[:2]

if china in top2:
    print("YES")
    sys.exit()

def equal(a, b):
    return key(a) == key(b)

if not equal(china, thailand):
    print("NO")
    sys.exit()

apply_ct_pts = [(2, 1), (1, 1)]
c_pts = 0
t_pts = 0

for x, y in apply_ct_pts:
    if x > y:
        c_pts += 3
    elif x == y:
        c_pts += 1
        t_pts += 1
    else:
        t_pts += 3

if c_pts > t_pts:
    print("YES")
else:
    print("NO")
```该解决方案首先维护四个用于团队统计数据的字典。 每场比赛都会更新进球数、失球数和得分。 帮手`result_points`直接对评分规则进行编码，确保胜平负处理中没有歧义。 

处理完所有比赛后，球队将按标准词典排名键进行排序。 我们立即检查中国是否处于前两名，这解决了大多数情况，无需进一步的逻辑。 

只有当中国没有进入前两名时，我们才会考虑平局。 功能`equal`检测中国和泰国在全球排名规则下是否无法区分。 如果不是，答案立即是否定的，因为在这种配置下，任何抢七局都无法让中国队超越泰国队。 

最后一个块仅使用两场固定的历史比赛来计算中国和泰国之间的正面交锋贡献。 由于这些结果是恒定的，我们直接重新计算它们的迷你表点并进行比较。

 ## 边缘情况

 一个重要的边缘情况是，除了中国和泰国之间的正面交锋差异之外，所有球队都以相同的全球统计数据结束。 在这种情况下，全局排序无法将它们分开，因此必须触发回退比较。 该算法通过在应用头对头规则之前显式检查排名键的相等性来处理此问题。 

另一个边缘情况是中国在全球排序后已经进入前两名。 在这种情况下，决胜局逻辑不应干扰。 排序后尽早退出可确保不会不必要地应用头对头比较，从而保持正确性，即使额外的比较可能会得出相反的结果。
