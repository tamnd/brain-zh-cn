 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中每个节点都以初始正整数值开始。 时间以离散的步骤演变，每个节点都带有一个根据涉及其父节点的本地规则每天变化的值。 

在第 0 天，每个节点 i 都有一个值 a[i]。 日复一日，该值要么减少一，要么保持不变。 根总是每天减少一，直到达到零。 任何其他节点将自己与其当天的父节点进行比较：如果它严格大于其父节点，则它也减一； 否则当天保持不变。 

我们想要的每个节点的数量是所有天的总累计值，这意味着我们从零天开始对其值进行求和，直到它最终变为零并保持在那里。 

关键的难点在于节点的演化不是独立的。 是否减少取决于其父级的当前值，并且父级也在同时发生变化。 这会沿着每条从根到叶的路径创建一个耦合系统。 

这些约束意味着树可能很大，所有测试用例最多可达一百万个节点，因此任何解决方案在输入大小上都必须基本上是线性的。 随着时间的推移进行二次模拟是不可能的，因为值可以从 10^9 开始，这将使简单的日常模拟变得太慢。 

当子级开始时比其父级小时，就会出现微妙的边缘情况。 在这种情况下，它可能最初会停止下降，而父级继续下降，最终扭转不平等并触发两者开始一起下降的后期阶段。 这种切换行为是天真的直觉经常被打破的地方。 

## 方法

 直接模拟将明确地逐日跟踪值。 每天我们都会扫描所有节点并根据规则更新它们。 这是正确的，但立即变得不可行，因为所有值达到零之前的天数可能与初始值一样大，并且每个更新步骤都会涉及所有节点。 在最坏的情况下，这会导致大约 O(n * max(a[i])) 次操作，这完全超出了范围。 

流程的结构比时间轴更重要。 每个节点的值始终不增加，并且以非常有限的方式变化：它要么遵循与其父节点相同的斜率，要么在其父节点继续减小时暂停。 唯一的交互是沿着边缘进行的，因此我们可以关注单个父子对并了解他们的相对行为。 

考虑一个节点及其父节点。 如果子级开始时严格大于父级，则两者将始终一起减小，因为条件“子级 > 父级”永远保持为真，直到两者最终一起达到零。 在这种情况下，子级的行为就像一个孤立的递减序列，独立于父级。 

相反，如果子级开始小于或等于父级，则最初它不会减少，而父级继续下降。 这会导致父母和孩子之间的差距缩小并最终翻转符号。 在那一刻之后，孩子变得严格更大，并且两者开始永远同步减小。 

这意味着每个节点最多有两个阶段：冻结的等待阶段和与独立线性下降相同的同步衰减阶段。 这一观察结果消除了随着时间的推移进行模拟的任何需要； 我们只需要确定每个节点在开始最终递减运行之前保持冻结状态多长时间。 

由于每条边仅影响其子节点一次，因此可以在线性时间内从根向下处理整个树。

 ## 算法演练

我们以自上而下的方式处理树，这样当我们处理一个节点时，它的父节点的总体行为就已经被理解了。 

1、从根源开始。 根总是每天减少一，直到达到零，因此它的贡献是一个基于其初始值的简单三角序列。 这建立了一个独立于任何父母的基线。 
2. 对于每个非根节点，将其初始值与其父节点的初始值进行比较。 该比较确定节点是立即表现得像独立的衰减序列还是进入延迟阶段。 
3. 如果节点的初始值严格大于其父节点的初始值，则两个值从一开始就同步减小。 它们之间的差异保持恒定且为正，因此子级永远不会比父级更早停止下降。 在这种情况下，节点的完整行为与独立的递减序列相同。 
4. 如果节点的初始值小于或等于其父节点的初始值，则子节点保持冻结状态，而父节点减小。 我们计算父项的值需要多少天才能严格降至子项的值以下。 在那一刻之前，节点每天都会贡献一个恒定的值。 
5. 一旦不平等翻转，两个节点就会进入同步衰减，这意味着它们每天都会一起减少，直到达到零。 从那时起，节点的行为就像从其当前值开始的标准线性序列一样。 
6. 结合冻结阶段和递减阶段的贡献以获得节点的总和。 

正确性取决于节点与其父节点之间的交互仅改变一次行为的事实。 要么孩子总是在父母之上，要么从下面开始并只交叉一次，之后他们永远一起移动。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        a = list(map(int, input().split()))
        p = list(map(int, input().split()))

        parent = [0] * n
        for i in range(1, n):
            parent[i] = p[i - 1] - 1

        ans = [0] * n

        def tri(x):
            return x * (x + 1) // 2

        # root
        ans[0] = tri(a[0])

        for i in range(1, n):
            ai = a[i]
            ap = a[parent[i]]

            if ai > ap:
                ans[i] = tri(ai)
            else:
                t0 = ap - ai + 1
                ans[i] = ai * (t0 - 1) + tri(ai)

        print(*ans)

if __name__ == "__main__":
    solve()
```该实现依赖于在知道其父节点的初始值后独立计算每个节点。 树结构仅用于访问父值，因此除了确保父索引已经可用之外，我们不需要完整的遍历顺序。 

三角函数对纯递减序列的和进行编码。 唯一重要的部分是计算当节点在其父节点下方启动时开始减少之前的等待时间。 

一个常见的错误是在合并冻结阶段和腐烂阶段时重复计算第一个减少日。 该公式通过在转变点之前严格分隔恒定段来避免这种情况。

 ## 边缘情况

 具有单个节点的最小树执行仅根规则。 节点每天都会衰减，答案是一个三角形数字，在没有任何父交互的情况下确认基本情况。 

子节点大于父节点的两节点链证实了“总是一起递减”的机制。 孩子永远不会经历冻结阶段，因此任何错误地引入等待的逻辑都会产生不正确的结果。 

子节点小于父节点的两节点链测试转换行为。 在加入腐烂过程之前，孩子必须保持一段时间的稳定。 这是通常出现相差一误差的地方，尤其是在计算不等式翻转的确切时刻时。
