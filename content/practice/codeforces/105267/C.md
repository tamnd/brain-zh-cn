 ## 解决方案
 ## 问题理解

 我们在平面上得到一组点，需要计算有多少个不同的五个点选择可以形成一个非常具体的几何配置，称为“菱形”。 

有效的菱形不仅仅是任何五元组。 必须对这五个点进行排列，以便其中四个点的行为类似于具有特定图案的平行边和等长对角线的高度约束的四边形，而第五个点必须位于相对于其中一条边对称的位置。 这些条件混合了等距离等度量约束、平行线段和叉积符号等方向约束以及强制形状方向一致的角度排序约束。 

用更结构性的术语来说，这些条件迫使一个严格的模板：一旦固定了几个点，剩下的点基本上就取决于输入集中的存在性检查。 这是关键提示，问题不是搜索任意 5 点子集，而是计算点集中固定几何图案的嵌入。 

限制很小，最多300分。 这立即排除了任何枚举所有 5 点子集的方法，因为这将是以下顺序：$\binom{300}{5}$，太大了。 即使枚举所有三元组并天真地验证完整的几何结构也将是边缘性的，除非内部检查通过散列或预处理是恒定时间的。 

这种几何问题中的一个微妙问题是，如果我们不小心排序，相同五个点的多个排列可以代表同一个菱形。 该声明明确表示，如果任何点坐标不同，两个菱形就会不同，因此我们正在计算标记的选择，但我们仍然必须避免在不同的角色分配下对同一组进行过度计算。 

## 方法

 蛮力方法很简单：选择 5 个点的每个子集，将它们分配给角色$A, B, C, D, E$以所有可能的方式，并检查所有几何约束是否成立。 这是正确的，因为它直接测试定义。 然而，代价是巨大的。 有大约$3 \times 10^9$大小为五的子集，每个子​​集都需要检查排列和几何，这使得它不可行。 

关键的结构洞察力是约束迫使点之间存在强依赖性。 特别是，一旦我们确定了顺序$B, C, D$, 剩下的点$A$和$E$不再免费：

 一对线段相等且平行的条件迫使点之间存在平移关系，这意味着一个点由一个固定向量与另一个点确定。 这将几何约束转变为搜索集合内重复的向量差异。 

类似地，一个点与其他两个点等距的条件意味着该点位于线段的垂直平分线上，这进一步减少了搜索空间，因为它是对对而不是任意三元组的约束。 

这将问题从“选择 5 个点并验证”转换为“选择一个小的结构化核心（通常是 2 或 3 个点），使用向量关系导出剩余点的候选点，并计算输入集中存在多少个候选点”。 

因此，我们转向修复少量锚点并使用哈希集计算完成情况以进行 O(1) 存在检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 5 点子集上的暴力破解 |$O(n^5)$|$O(1)$| 太慢了 |
 | 锚点+向量重建|$O(n^3)$或者$O(n^3 \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 几何约束意味着一种刚性结构，其中配置的一部分通过向量相等决定其余部分。 

我们通过首先修复中间结构并重建剩余点来利用这一点。 

### 步骤

 1. 将所有点存储在哈希集中，以进行 O(1) 成员资格检查。 这是至关重要的，因为算法会重复构造候选点并需要验证它们是否存在于输入中。 
2. 迭代所有有序点对$(C, D)$。 这两点充当锚段。 向量$\overrightarrow{CD}$将为形状的其他部分定义方向和长度约束。 
3. 迭代所有点$E$并检查是否$E$等距于$C$和$D$， 意义$EC = ED$。 这个条件保证了$E$位于线段的垂直平分线上$CD$，这是钻石定义所要求的。 如果失败，请跳过。 
4.一次$(C, D, E)$固定后，重建剩余两点$A$和$B$使用向量约束。 关键的观察结果是平行和等长约束强制平移关系：$$\overrightarrow{BA} = \overrightarrow{DC}$$这意味着一旦$B$被选择，$A$唯一确定为：$$A = B + (D - C)$$5. 迭代所有选择$B$其余的点中。 对于每个$B$, 计算$A$使用上面的公式并检查是否$A$存在于点集中。 
6. 在计算配置之前，请确保所有五个点都不同。 这可以避免重建意外重用锚点之一的退化情况。 
7. 累积所有选择的有效配置数量。 

### 为什么它有效

 整个减少取决于菱形的四边形部分完全由单个平移向量确定的事实$D - C$。 一旦该向量被固定，任何有效的对$(B, A)$必须与该向量完全不同，这意味着所有有效的完成都对应于坐标空间中固定移位下的匹配对。 

等距条件为$E$独立限制哪些锚定段$(C, D)$是有效的，确保仅使用几何一致的基础。 由于每个有效的菱形在这种分解下都会产生一个这样的结构，因此计算所有有效的完成$(C, D, E)$在有序角色下精确枚举每个配置一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dist2(a, b):
    return (a[0] - b[0]) ** 2 + (a[1] - b[1]) ** 2

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    S = set(pts)

    ans = 0

    for i in range(n):
        C = pts[i]
        for j in range(n):
            D = pts[j]
            if i == j:
                continue

            cd2 = dist2(C, D)

            for k in range(n):
                E = pts[k]
                if dist2(E, C) != dist2(E, D):
                    continue

                for B in pts:
                    A = (B[0] + D[0] - C[0], B[1] + D[1] - C[1])

                    if A not in S:
                        continue

                    if A == B or A == C or A == D or A == E:
                        continue
                    if B == C or B == D or B == E:
                        continue

                    ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接应用了向量重构的思想。 关键操作是翻译$A = B + (D - C)$，它同时编码并行性和等长约束。 

等距检查使用平方距离来避免浮点错误。 这确保了$E$位于垂直平分线上$CD$无需明确计算角度。 

哈希集`S`使成员资格检查的时间恒定，这很重要，因为最里面的循环依赖于重复的存在查询。

 ## 边缘情况

 一种重要的边缘情况是当多个点满足同一线段的等距条件时$CD$。 在这种情况下，算法将正确地迭代所有此类$E$，但每个有效的菱形仍然是唯一表示的，因为$E$是正在计数的元组的一部分。 

当计算点$A$与其中之一一致$B, C, D, E$。 显式的独特性检查可防止无效的简并配置，其中重建的结构会塌陷为少于五个点。 

第三种情况是当多个不同的$(C, D)$排序对应于相同的底层段。 由于该算法处理$(C, D)$按照顺序，相同的几何基可以在不同的方向上计数两次，这与计数有序角色而不是无序集合一致。
