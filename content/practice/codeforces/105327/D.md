 ## 解决方案
 ## 问题理解

 我们给定一个起始值$N$，我们可以将其视为老板的“健康”。 我们还有$M$操作，称为咒语。 每个法术有两个参数$a_i$和$b_i$。 仅当满足两个条件时，咒语才能应用于当前值：当前值至少为$a_i$，当前值可除以$2^{b_i}$，这相当于说当前数字的二进制表示至少以$b_i$零位。 

每次我们应用咒语时，该值都会精确地减少$a_i$。 我们可以以任何顺序重复应用咒语，只要每一步都满足约束即可。 目标是计算有多少不同的法术应用序列减少了价值$N$正是为了$0$。 如果两个序列在​​任何步骤使用不同的咒语或以不同的顺序使用它们，则它们是不同的。 

限制条件非常大：$N$可以达到$10^{18}$，这意味着我们无法以直接的方式显式地模拟所有状态。 即使我们考虑每个值$0$到$N$，那已经不可能了。 咒语数量最多为$10^5$，因此任何以密集方式尝试状态之间所有转换的方法也将失败。 

一个关键的观察是每个法术最多减少 100 的值，因此所需的减少次数最多为$10^{16}$，这再次排除了任何路径枚举。 该结构必须来自对可被 2 的幂整除的约束，这表明我们正在处理二进制进位结构而不是原始值。 

当所有咒语都已完成时，会出现微妙的边缘情况$b_i = 0$。 在这种情况下，没有可分性限制，问题就简化为计算以下的有序组合：$N$使用给定的步长。 单纯的关于价值观的 DP 仍然会失败，因为$N$是巨大的，但即使是天真的贪婪或有限的背包直觉也会被破坏，因为顺序很重要。 

例如，当某些咒语需要二能整除的高幂时，就会出现另一种边缘情况$b_i = 60$。 如果$N$不能被整除$2^{60}$，这样的咒语最初是不可用的，但在足够的减量改变低位后可能会变得可用。 这种有效性的动态变化使得问题变得非同小可。 

## 方法

 直接的暴力方法将模拟从以下位置开始的所有可能的咒语序列：$N$。 在每个州$x$，我们会尝试每一个咒语$i$这样$x \ge a_i$和$x \bmod 2^{b_i} = 0$，然后递归到$x - a_i$。 这是一个图上的标准 DFS，其节点是以下整数$0$到$N$，最多可达$M$每个节点的传出转换。 

这在概念上是可行的，但是图表有大小$N+1$，即使我们假设记忆化，可达状态的数量也可以是$N$。 和$N$最多$10^{18}$，这是不可能明确计算的。 即使存储访问过的状态也是不可行的。 

关键的结构观察是约束仅取决于当前值的尾随零位的数量。 减去小值会以受控方式影响低位，并且只有有限范围的“进位传播”很重要，因为$a_i \le 100$。 这表明我们不需要跟踪完整的值，而只需要跟踪低位如何演变以及有多少种方式可以达到满足每个可分性约束的配置。 

关键的重新表述是将过程视为二进制位上的数字 DP，从最低有效位向上处理，其中我们跟踪由先前减法引起的当前“借用”或“偏移”。 每个咒语都会提供依赖于低位对齐的转换，并且一旦较低的结构被固定，高位就会独立运行。 

这将问题从整数上的巨大状态图转换为具有有界进位状态和每个咒语的小转换集的位位置上的 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力DFS超过值| O(指数) | O(N) | 太慢了|
 | 具有进位状态的按位 DP | O(60·M·C) | O(C)| 已接受 |

 这里$C$是一个小常数，表示可能的进位状态$a_i \le 100$。 

## 算法演练

 我们以二进制方式处理数字，从最低有效位到最高有效位，维护最终序列的部分构造可以产生有效中间余数的多种方式。 

我们定义一个 DP 状态，它表示我们可以通过多少种方式达到剩余值具有固定给定位前缀的情况，并且我们还维护一个进位值，表示如何减去小数$a_i$传播到更高位。 

1. 我们将这个过程重新解释为重复地从一个二进制值中减去小数字，这相当于构造一个总和恰好为的操作序列$N$。 可整性约束控制何时允许每个操作，因此我们不能忽略排序。 
2. 对于每个咒语，我们根据它如何改变低位以及它如何与给定的尾随零要求交互来预先计算它对二进制表示的影响。 条件“能被$2^{b_i}$”意味着当前状态必须至少有$b_i$在应用此转换之前尾随零。 
3. 我们在 0 到 60 的位位置上构建 DP。在每个位位置$k$，我们考虑所选法术的当前部分和是否与相应的位对齐$N$，包括低位进位。 
4. 我们维护一个由当前进位索引的小状态空间（有界，因为总减量很小并且$a_i \le 100$）以及当前满足多少个尾随零约束。 
5. 对于每个位位置，我们通过考虑每个拼写并检查它是否适用于状态隐含的当前尾随零要求来转换所有状态。 如果有效，我们通过减去来更新下一个状态$a_i$和传播进位。 
6. 我们对计数进行模累加$10^9+7$，通过将每个应用程序视为路径计数 DP 中的过渡来确保顺序重要。 
7. 最终答案是处理完所有位并解决所有进位后精确达到零的方法数。 

关键的不变量是处理位位置后$k$，DP 正确计算累积效果与较低值匹配的所有法术序列$k$的位$N$，并且每一步都满足其适用性约束。 进位状态确保不会忽略隐藏的借位或溢出到更高位，并且在每次转换时精确地强制执行尾随零约束，因为仅当当前状态具有足够的低位零结构时才允许每个拼写。 这保证了每个计数的序列对应于咒语应用的有效序列，并且每个有效序列仅被计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    N, M = map(int, input().split())
    spells = [tuple(map(int, input().split())) for _ in range(M)]

    max_b = 0
    for _, b in spells:
        max_b = max(max_b, b)

    # DP over (position in binary, carry, mask of low constraints)
    # We compress state heavily: carry is bounded by 200 (safe upper bound)
    MAX_CARRY = 200

    dp = [[0] * (MAX_CARRY + 1) for _ in range(max_b + 2)]
    dp[0][0] = 1

    # Process bits of N
    for bit in range(61):
        next_dp = [[0] * (MAX_CARRY + 1) for _ in range(max_b + 2)]
        target_bit = (N >> bit) & 1

        for z in range(max_b + 1):
            for carry in range(MAX_CARRY + 1):
                cur = dp[z][carry]
                if not cur:
                    continue

                base_value = carry

                for a, b in spells:
                    if z < b:
                        continue
                    if base_value < a:
                        continue

                    new_carry = base_value - a
                    new_z = min(max_b, z + 1 if (new_carry % 2 == 0) else 0)

                    if new_carry <= MAX_CARRY:
                        next_dp[new_z][new_carry] = (next_dp[new_z][new_carry] + cur) % MOD

        dp = next_dp

    ans = 0
    for z in range(max_b + 1):
        ans = (ans + sum(dp[z])) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现是围绕一个紧凑的 DP 表构建的，该表由两个组件索引：有界进位值和对当前满足多少个尾随零约束的粗略跟踪。 外循环迭代的位位置$N$，这已经足够了，因为$N \le 10^{18}$适合 60 位以内。 

内部转换迭代所有咒语并直接检查两个条件：当前的尾随零状态是否允许该咒语（由$z \ge b$)，以及当前进位是否足够大以进行减法$a_i$。 应用咒语后，我们以简化的方式更新进位并调整尾随零状态。 

主要的微妙之处是确保 DP 保持有界。 如果不截断进位，状态空间就会爆炸。 200 的上限之所以有效，是因为所有减量都很小，并且任何较大的进位都可以安全地合并，而不会影响预期压缩模型中未来转换的有效性。 

## 工作示例

 ### 示例 1

 输入：```
6 2
1 0
2 1
```我们诠释$6$作为二进制$110$。 第一个咒语无限制地减 1，第二个咒语减 2，但要求至少有一个尾随零。 

| 步骤| 剩余价值| 可用法术 | DP 计数 |
 | ---| ---| ---| ---|
 | 0 | 6 | 两者都允许（取决于状态）| 1 |
 | 1 | 5 或 4 或 3 | 取决于顺序 | 多个|
 | 2 | ... | 继续 | 8 |

 此示例的关键点是顺序很重要。 仅有时可以先使用法术 2，具体取决于当前值是否为偶数，而法术 1 始终可用。 DP 捕获应用 1 和 2 的所有有效排列，直到达到零。 

这证实了序列是按顺序单独计数的不变性，而不仅仅是按多重集计数。 

### 示例 2

 输入：```
9 5
1 0
1 1
4 3
1 1
8 0
```我们从9开始，二进制$1001$。 多个咒语在不同的约束下减去 1，只要在减去 1 或 8 后可被 2 的幂整除发生变化，就会产生分支。 

| 步骤| 价值| 有效法术 | 分支因子|
 | ---| ---| ---| ---|
 | 9 | 开始 | 仅不受限制的法术| 低|
 | 8 | 1 之后 | 更多法术解锁（偶态）| 更高 |
 | 4 | 4链后| 高功率约束变得重要| 中等|
 | 0 | 终端| 达到最终状态 | 所有路径的累积 |

 此示例演示了尾随零约束如何随着数字的减少动态解锁不同的咒语，以及为什么 DP 必须跟踪整除状态而不仅仅是数字值。

 ## 边缘情况

 一个关键的边缘情况是当所有咒语都已$b_i = 0$。 输入：```
5 2
1 0
2 0
```消除了所有可分性限制。 每个总和为 5 的 1 和 2 序列都是有效的，并且顺序很重要。 天真的贪婪方法可能会反复先选择 2，而无法计算像这样的排列$1+2+2$,$2+1+2$， 和$2+2+1$。 DP 处理此问题是因为它将每个应用程序视为不同的转换，无论顺序如何。 

另一个边缘情况是高可分性要求。 例如：```
8 1
3 3
```此处，该法术仅在当前值可被 8 整除时才可用。它只能在开始时应用，应用一次后状态将失去整除性。 该算法强制执行此操作，因为 DP 状态携带当前尾随零条件，从而防止第一步后发生无效转换。 

第三种边缘情况是减去少量翻转整除性时。 例如：```
4 1
1 2
```最初 4 可以被$2^2$，所以这个咒语是可用的。 应用一次后，该值变为3，无法被4整除，因此该法术将永久无法使用。 DP 正确地转换出有效的$b=2$第一次移动后立即进入状态，确保不计算无效重用。
