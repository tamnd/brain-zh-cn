 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并且允许我们重复应用在同一位置的两个数字之间交换各个位的操作。 如果我们选择两个索引和一个位位置，我们可以在两个数字之间交换该位是 0 还是 1。 

关键的结果是位不再与其原始数字相关联。 每个位的位置$k$整个数组中的元素可以任意重新分配$N$数字，只要每个位位置上 1 的总数保持不变。 

任务是使用任意数量的交换来重新排列这些位，以便生成的数组在字典顺序上是最大的。 这意味着我们首先最大化$a_1$，然后在所有这些配置中最大化$a_2$， 等等。 

限制条件$N \le 10^5$和$a_i \le 10^9$意味着每个数字最多可以容纳 30 位。 任何尝试模拟交换或考虑每个数字配置的解决方案都会失败，因为潜在配置的总数呈指数增长。 我们需要一种能够在线性或近线性时间内针对每个位位置和每个元素工作的方法。 

一个微妙的陷阱是假设我们可以通过尽可能将每个数字的位设置为 1 来独立地贪婪地最大化每个数字。 但这会失败，因为位是共享的全局资源。 

例如，如果我们有价值观`[8, 4, 2, 1]`，所有位都是不同的。 如果我们贪婪地尝试通过获取所有大位来最大化第一个元素，我们可能会忽略同一位不能在多个数字之间重复使用。 正确的输出变成`[15, 0, 0, 0]`，其中所有位都集中到第一个数字中。 

当较高位稀缺时，会出现另一种边缘情况。 为了`[12, 15, 1, 20]`，在不考虑字典结构的情况下贪婪地分配位可能会导致非最佳排序，例如将大位分配给后面的位置。 正确的解决方案是`[31, 13, 4, 0]`，其中最高位首先分配给较早的索引。 

核心挑战是认识到问题不在于交换数字，而在于跨位置重新分配固定的多重位集。 

## 方法

 暴力方法会尝试直接模拟操作。 每个操作都会在两个数字之间交换一点，因此可以想象重复选择有益的交换，直到字典顺序无法改进为止。 这很快就会变得棘手。 每次交换都会更改配置，并且有$O(N \cdot \log A)$位，导致状态数量达到天文数字。 即使是贪婪的局部交换也不能保证收敛到全局最优，因为改进后面的元素可能会恶化前面的元素。 

关键的见解是按位位置来分离问题。 由于交换仅交换同一位置的位，因此每个位位置彼此独立。 对于每一位$k$，我们确切地知道数组中存在多少个，并且这个计数永远不会改变。 

因此，对于每个位位置，我们实际上被要求将固定数量的位分配给$N$插槽，每个数组位置每个位一个，以最大化字典顺序的方式。 字典顺序意味着数组中索引较高的元素比索引较低的元素更重要，因此我们应该始终优先考虑在较早的位置设置较高值的位。 

这简化为按价值贡献对所有位贡献进行排序，但按位进行：对于从最高到最低的每个位，我们将其分配到仍然有利于全局顺序的最早可用位置。 具体来说，我们贪婪地将每个位的 1 分配给仍然可以容纳它们的最小索引，从而逐步构建数字。 

蛮力会失败，因为它将位视为可局部移动而无需结构。 每个位位置都是独立的观察结果将问题转化为跨位置的贪婪分配。

 ## 算法演练

 我们独立处理每个位位置，从最高位到最低位。 

1. 计算所有数字的每个位位置上存在多少个。 这为我们提供了每比特的固定预算。 这样做的原因是交换在全局范围内保留位计数，因此这些计数是不变的。 
2. 对于从最高到最低的每个位位置，决定哪些数组位置将在该位中接收 1。 由于较早的指数按字典顺序更重要，因此我们总是尝试将可用的指数分配给最早受益的位置。 
3. 保持每个的当前值$a_i$，最初为零，然后一点一点地构建它。 对于每个位，按顺序迭代索引，如果该位仍有剩余索引，则分配 1。 分配时减少剩余计数。 
4. 处理完一位后，移至下一个较低位。 这确保了在较低位细化排序之前较高位主导词典结构。 
5. 通过组合分配的位来构造最终值。 

为什么这有效与贪婪的支配论有关。 严格来说，较高位对字典顺序的影响比任何较低位的组合都大。 通过首先分配较高位并始终将它们放置在较早的索引中，我们确保以后的重新分配不能在不违反位守恒的情况下改进较早的前缀。 每个比特位置实际上是一个单独的资源，分配用于最大化前缀加权词典价值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    MAXB = 30
    cnt = [0] * MAXB

    for x in a:
        for b in range(MAXB):
            if x >> b & 1:
                cnt[b] += 1

    res = [0] * n

    for b in range(MAXB - 1, -1, -1):
        if cnt[b] == 0:
            continue
        for i in range(n):
            if cnt[b] == 0:
                break
            res[i] |= (1 << b)
            cnt[b] -= 1

    print(*res)

if __name__ == "__main__":
    solve()
```该实现首先提取全局位计数，这是安全的，因为交换保留了每位总数。 然后从最高位向下贪婪地构造结果。 

关键的实现细节是从左到右迭代每个位的索引。 这会正确强制执行字典优先级：首先填充较早的索引以获取较高位，这决定了整个数组的顺序。 

另一个微妙之处是 30 位的固定界限，它可以安全地覆盖$10^9$。 超出这个范围的迭代是不必要的，只会浪费时间。

 ## 边缘情况

 一种边缘情况是所有数字都为零。 该算法对各处的零位进行计数，因此不会发生任何分配，并且输出仍然全为零。 用于输入`3 / 0 0 0`，每个位计数器都为零，因此构造循环不会分配任何内容。 

另一个边缘情况是单元素数组。 由于所有位都属于一个数字，因此重新分配不会改变任何内容。 用于输入`1 / 7`，位计数被保留并分配给唯一的索引，产生`7`。 

最后的边缘情况是位的最大分散，例如`[1, 2, 4, 8, 16]`。 该算法按照位重要性降序将所有位收集到第一个位置。 每一位都是从最高位开始分配的，结果是`[31, 0, 0, 0, 0]`。 这表明词典编目最大化自然地将独立的比特源折叠成早期索引。
