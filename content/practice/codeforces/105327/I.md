 ## 解决方案
 ## 问题理解

 每个食品都标有一个数字，该数字应被视为质因数的多集。 如果一种食物的价值为12，它实际上意味着它贡献了成分2、2和3。因此，每种食物都对应于一组素数“成分”，可能有重复，但重复对于过敏规则并不重要，因为限制只是关于素数是否出现。 

客户还有一个代表过敏的数字，我们再次通过其质因数分解来解释它。 如果素数除以顾客的数量，则该顾客不能食用任何含有该素数作为因子的食物的菜肴。 菜肴是可用食物的任何子集，包括空子集。 

任务是，对于每个顾客，计算可以选择 N 种食物的多少个子集，使得划分顾客数量的素数不会出现在任何选定的食物中。 

如果直接解释，这些约束会迫使搜索空间非常大。 食物多达100000种，因此子集数量为2^N，这是一个天文数字。 即使我们只尝试独立处理每个查询，分解并检查每个查询的所有食物也会导致 10^10 规模行为，这在 1 秒内是不可能的。 

隐藏的结构是限制是乘法的，但每个素数都是独立的，并且值的范围以 10^6 为界，这强烈建议对除数或素数进行预计算。 

一个微妙的边缘情况是食品价值 1 的存在。由于 1 没有素因数，因此它永远不会被禁止，并且对每个人来说总是安全的。 如果解决方案错误地忽略 1 或将其视为无效，则它会在每个查询中计数不足。 

另一个边缘情况是重复相同的食物值。 即使多种食物共享相同的主要因素，它们仍然是子集中的不同选择，因此它们会增加计数而不是折叠计数。 

## 方法

 直接的强力解决方案将迭代每个查询的所有食物子集，并检查任何选定的食物是否与查询的过敏数共享一个素因数。 对于 N 个食物，即 2^N 个子集，甚至单个查询都变得不可行。 

一个稍微不那么幼稚的改进是预先计算每种食物和每个查询编号的素数因子，然后为每个查询标记所有禁止的素数并扫描所有食物以计算哪些是有效的。 这已经减少了对每个子集的检查，但在最坏的情况下仍然留下 O(NQ) 行为，即大约 10^10 次操作。 

观察的关键是扭转观点。 我们不是询问一个人允许吃哪些食物，而是根据食物包含的素数集对食物进行分类。 如果一种食物与查询的禁止素数集至少共享一个素数，则该食物被禁止。 因此，对于查询而言，唯一重要的是包含任何禁用素数的所有食物的并集。 

这建议按“坏质数”对食物进行分组，并使用包含性除以质数来划分食物价值。 由于值最多为 10^6，因此我们可以对每个值进行因式分解并将其与其素因数相关联。 然后对于每个素数 p，我们维护有多少食物可以被 p 整除。 在查询中对素数子集使用包含-排除给出禁止食物的数量，从而给出允许食物的数量。 一旦我们知道允许的食物数量，有效菜肴的数量就是 2^(allowed_count)，因为允许食物的每个子集独立地形成有效菜肴。 

这将问题简化为高效因式分解和快速求幂。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 | O(Q·2^N) | O(Q·2^N) | O(1) | O(1) | 太慢了 |
 | 按查询扫描 | O(NQ) + 因式分解 | O(1) | O(1) | 太慢了 |
 | Prime + 包含计数 | O((N + Q) √V + Q · 2^k) | O(V) | 已接受 |

这里 k 是查询中不同质因数的数量，它很小，因为数字 ≤ 10^6。 

## 算法演练

 1. 使用筛子预先计算 10^6 以内的每个整数的最小质因数。 这允许在对数时间内快速分解每种食物和查询值。 
2. 对于每个食物值 Vi，将其分解为其不同的素因数。 我们只关心素数是否出现，而不关心它的指数，因为任何素数的存在都会导致过敏者禁止食用该食物。 
3. 维护一个频率数组cnt[p]，存储有多少个食物可以被素数p整除。 每种食物在其因式分解中对每个不同素数贡献一次。 这告诉我们，对于任何禁用的素数，有多少食物变得无效。 
4. 将食物子集的总数预先计算为 2^N 模 MOD。 这是当一个人没有过敏时的答案，因为不排除任何食物。 
5. 对于每个查询值 Xi，将其分解为其不同的素因数。 这些素数代表禁用成分。 
6. 对这些素数使用包含-排除来计算有多少食物是“坏”的，这意味着它们至少包含一个禁止的素数。 对于质数的每个子集，计算有多少食物可以被子集中的所有质数整除，并替换符号以避免过度计数重叠。 
7、用N减去不良食品的数量，得到安全食品的数量。 
8. 查询的答案是 2^(safe food) modulo MOD，因为安全食品的每个子集都是有效的菜肴，并且选择是独立的。 

需要包含-排除的关键原因是食物可以包含多个禁止素数，因此 cnt[p] 双重计数的简单求和会重叠。 

### 为什么它有效

 每道菜的定义仅取决于所包含的食物。 当且仅当每种选择的食物都避免了所有禁止的素数时，菜肴才是有效的。 一旦我们确定了一组安全食品，其中的任何子集都是有效的，因为食品之间没有进一步的限制。 因此，问题简化为对过滤集的子集进行计数，并且包含-排除可确保过滤步骤正确地删除与禁止素数集恰好相交一次的所有食物。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
MAXV = 10**6

# smallest prime factor sieve
spf = list(range(MAXV + 1))
for i in range(2, int(MAXV ** 0.5) + 1):
    if spf[i] == i:
        for j in range(i * i, MAXV + 1, i):
            if spf[j] == j:
                spf[j] = i

def factor_distinct(x):
    primes = []
    while x > 1:
        p = spf[x]
        primes.append(p)
        while x % p == 0:
            x //= p
    return primes

N = int(input())
V = list(map(int, input().split()))

cnt = {}
for v in V:
    if v == 1:
        continue
    ps = factor_distinct(v)
    for p in ps:
        cnt[p] = cnt.get(p, 0) + 1

# total subsets
pow2 = [1] * (N + 1)
for i in range(1, N + 1):
    pow2[i] = pow2[i - 1] * 2 % MOD

Q = int(input())

for _ in range(Q):
    x = int(input())
    if x == 1:
        print(pow2[N])
        continue

    ps = factor_distinct(x)
    ps = list(set(ps))

    m = len(ps)
    bad = 0

    # inclusion-exclusion over primes of x
    for mask in range(1, 1 << m):
        mult = 1
        bits = 0
        for i in range(m):
            if mask & (1 << i):
                mult *= ps[i]
                bits += 1
        c = cnt.get(mult, 0)
        if bits % 2 == 1:
            bad += c
        else:
            bad -= c

    bad %= MOD
    if bad < 0:
        bad += MOD

    safe = N - bad
    print(pow2[safe])
```筛子构建了一个最小质因数表，以便食物和查询的因式分解快速且统一。 cnt 字典存储有多少食物可以被每个素数整除，这已经足够了，因为包含-排除会重建查询内素数之间的重叠。 

对于每个查询，我们提取不同的素数并迭代它们的所有子集。 由于 10^6 以内的数字至多有大约 7 个不同的质因数，因此子集循环很小。 对于每个子集，我们计算有多少食物可以被这些素数的乘积整除，这恰好对应于同时包含所有这些素数的食物。 

最后，我们计算安全食品的数量，并将 2 指数化为该值。 

## 工作示例

 考虑示例输入：```
N = 6
V = [1, 2, 3, 4, 5, 6]
Q = 4
queries = [1, 2, 4, 6]
```我们首先计算cnt：

 2 出现在 {2,4,6} 中，因此 cnt[2] = 3

 3 出现在 {3,6} 中，因此 cnt[3] = 2

 5 出现在 {5} 中，因此 cnt[5] = 1

 现在跟踪查询。 

### 查询 1：x = 1

 | 步骤| 价值|
 | --- | --- |
 | 素数 | []|
 | 安全食品| 6 |
 | 答案| 2^6 = 64 | 2^6 = 64

 没有禁止任何素数，因此所有食物都是安全的。 

### 查询 2：x = 2

 | 步骤| 价值|
 | --- | --- |
 | 素数 | [2] |
 | 子集 {2} | cnt[2] = 3 |
 | 坏食物| 3 |
 | 安全食品| 3 |
 | 答案| 2^3 = 8 | 2^3 = 8

 含有 2 的食物被排除在外：2、4、6。 

### 查询 3：x = 4

 虽然4 = 2^2，素数集仍然是[2]。 

| 步骤| 价值|
 | --- | --- |
 | 素数 | [2] |
 | 坏食物| 3 |
 | 安全食品| 3 |
 | 答案| 8 |

 这证实了指数重数并不重要。 

### 查询 4：x = 6

 | 步骤| 价值|
 | --- | --- |
 | 素数 | [2, 3] |
 | 子集 {2} | 3 |
 | 子集 {3} | 2 |
 | 子集 {2,3} | 1 |
 | 坏食物| 3 + 2 − 1 = 4 |
 | 安全食品| 2 |
 | 答案| 2^2 = 4 | 2^2 = 4

 排除的食物是那些能被 2 或 3 整除的食物，通过包含-排除来纠正重叠。

 ## 边缘情况

 一个关键的边缘情况是食物值为 1 时。对于输入：```
N = 3
V = [1, 2, 3]
X = 2
```食物 1 始终是安全的。 该算法从不将 1 插入任何素数集中，因此 cnt 会忽略它。 对于查询 2，我们将安全食品计算为 2（食品 1 和食品 3），因此答案为 2^2 = 4，对应于子集 {}、{1}、{3}、{1,3}。 这证实了中性元素的正确处理。 

另一种边缘情况是查询数字中重复素数，例如 X = 8 = 2^3。 因式分解步骤仅返回 [2]，因此包含-排除仍然正确并且不会重复计算。 因此，该算法一致地处理所有幂并避免过度计算禁用素数。
