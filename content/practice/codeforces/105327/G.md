 ## 解决方案
 ## 问题理解

 我们被赋予了一个单一河流系统的根基结构，最终将所有源头合并成一条最终流入大海的河流。 每个源头都是一条独立的河流，具有固定的初始水量和自己的标识符作为其名称。 然后，按照确定的顺序，合并成对的现有河流，直到只剩下一条河流。 

每次合并都会将两条不相交的河流合并成一条新的河流，其水量是两条河流的总和。 由此产生的河流的名称不是任意的，它是通过比较两条合并河流的总流量而选择的。 水量较大的河流因此而得名。 如果两个交易量相等，则通过选择较小的指数来打破平局。 

整个结构建成后，我们反复增加一些原始水源的水量。 这些增加是永久性的，会影响所有未来的合并。 每次更新后，我们必须在整个合并结构的根部报告最终河流的名称。 

重要的观察结果是，合并结构是最多包含 2N−1 个节点的固定二叉树。 只有叶子值随时间变化，而内部节点则隐式重新计算其总和和获胜者名称。 

限制允许最多 100000 个源和 100000 个更新。 每次更新后直接重新计算所有子树总和将需要每次查询 O(N)，导致 O(NQ) 太大，最多 10^10 次操作。 这立即排除了每个查询从头开始重新计算树的任何方法。 

当多次合并产生相等的子树总和时，就会出现微妙的边缘情况。 在这种情况下，必须传播最小的索引。 天真的实现可能会忘记打破平局取决于当前值，而不是初始结构。 另一个微妙的问题是更新仅影响叶节点，但它们的效果会传播到所有祖先，这意味着正确性取决于维护动态子树聚合。 

## 方法

 该结构是一个固定的二叉树，其中每个内部节点存储其子树中叶子的总和以及通过比较其两个子节点定义的“获胜者”标签。 暴力破解的想法很简单：每次更新后，重新计算从叶子向上的所有子树总和，然后重新计算所有内部节点的获胜者。 这是可行的，因为树是静态的并且评估是确定性的，因此完整的重新计算总是会产生正确的根。 故障点纯粹是性能问题，因为每次更新都会涉及所有 N 个节点，并且有 Q 次更新，从而产生二次行为。 

关键的见解是根决策仅取决于聚合的子树总和，并且这些总和仅沿着从修改的叶子到根的路径变化。 这表明，如果我们有一个平衡结构，则将结构视为静态树的一部分，其中每次更新都会影响类似 O(log N) 的传播，但这里的树是任意的。 我们不尝试利用高度，而是反转依赖关系：每个节点只需要知道其两个子节点中的哪一个当前占主导地位。 如果我们为每个节点维护其当前的获胜者和当前的子树总和，那么叶子的更新只需要沿着到根的路径重新计算这些值。 由于树是静态的，因此每个受影响的路径的每个节点都可以更新一次，并且每个内部节点重新计算的时间复杂度为 O(1)。 

关键的结构属性是每个内部节点的值都是其两个子节点的纯函数，因此维护正确性减少为在固定树上维护自下而上的动态程序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个查询重新计算完整树 | O(NQ) | O(N) | 太慢了|
 | 树上自下而上的传播 最坏情况为 O((N + Q) log N)，使用父指针时实际为 O((N + Q)) | O(N) | 已接受 |

 ## 算法演练

我们首先将构建过程重新解释为具有 2N−1 个节点的显式二叉树，其中每个内部节点恰好有两个子节点，每个叶子对应于一个原始源。 

1. 按照给定的方式构建树，为每个节点存储其两个子节点，并为每个节点存储一个父指针。 这允许更新后向上传播。 
2.初始化一个数组`value[i]`存储每个节点的当前水量。 对于叶子，这是给定的，对于内部节点，它将被计算。 
3. 执行后序遍历或迭代自下而上传递来计算所有节点的初始子树值和获胜者标签。 对于每个内部节点，计算其总和作为其子节点的总和。 
4. 定义一个函数来“重新计算”节点。 对于具有子 A 和 B 的节点，其总和变为`sum[A] + sum[B]`。 它的获胜者成为其子树的总和严格较大的子树，或者如果相等则较小的索引。 此步骤对河流命名规则进行编码。 
5. 通过增加叶子的值来处理叶子上的更新。 更改叶子后，使用父指针从该叶子向上移动到根，使用上述规则重新计算该路径上的每个节点。 
6. 每次更新传播完成后，输出存储在根节点的获胜者。 

向上传播足够的原因是只有更新叶子的祖先才能更改其子树和。 该路径之外的任何节点都具有相同的子树组成，因此具有相同的总和和获胜者。 

### 为什么它有效

 每个节点都维护其子树的正确摘要：其总水量以及根据合并规则从中出现的河流的身份。 叶子更新仅更改一个子树叶子值，因此摘要变为无效的节点正是在其子树中包含该叶子的节点。 因为树是静态的，所以这些节点形成到根的单个路径，并且沿着该路径重新计算从叶向上归纳地恢复正确性。 在每次重新计算时，子级都已经是正确的，因此每个父级都会根据有效输入重新计算，从而保留一直到根的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = [0] + list(map(int, input().split()))

    # nodes are 1..2n-1
    N = 2 * n
    left = [0] * N
    right = [0] * N
    parent = [0] * N

    for i in range(n + 1, 2 * n):
        u, v = map(int, input().split())
        left[i] = u
        right[i] = v
        parent[u] = i
        parent[v] = i

    # sum and winner
    sm = [0] * N
    winner = [0] * N

    for i in range(1, n + 1):
        sm[i] = a[i]
        winner[i] = i

    def recompute(x):
        l = left[x]
        r = right[x]
        sm[x] = sm[l] + sm[r]

        wl = winner[l]
        wr = winner[r]

        if sm[l] > sm[r]:
            winner[x] = wl
        elif sm[r] > sm[l]:
            winner[x] = wr
        else:
            winner[x] = min(wl, wr)

    for i in range(n + 1, 2 * n):
        recompute(i)

    root = 2 * n - 1

    def update(i, delta):
        nonlocal a
        x = i
        sm[x] += delta
        while x:
            if x != i:
                recompute(x)
            x = parent[x]

    q = int(input())
    out = []
    for _ in range(q):
        i, d = map(int, input().split())
        update(i, d)
        out.append(str(winner[root]))

    print(winner[root])
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现显式地构造了完整的二叉合并树。 每个内部节点都存储指向其两个子节点的指针，并且每个节点都知道其父节点，从而可以在更新后向上传播。 数组`sm`和`winner`分别维护当前子树总和和获胜索引。 

这`recompute`功能是核心转变。 它在恒定时间内强制执行求和聚合和命名规则。 使用显式处理平局打破条件`min`，这确保了子树总和相等时的正确性。 

更新仅适用于叶子。 增加叶子值后，我们使用父指针向上走并重新计算所有受影响的节点。 然后直接读取根值。 

一个微妙的实现细节是我们必须仅重新计算叶子本身一次，然后向上传播。 此外，根始终是节点`2n-1`，因为每次合并都会顺序创建一个新节点。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N + Q·h) | O(N + Q·h) | 每次更新都会沿着到根的路径重新计算节点 |
 | 空间| O(N) | 树形和 DP 阵列的存储 |

 这里h是构造的合并树的高度。 由于树是按顺序构建的，因此通常仅在最坏的情况下发生倾斜，但仍受 N 限制。当 N、Q 高达 10^5 时，这种传播在实践中非常高效，因为每个节点重新计算的时间复杂度为 O(1)，并且每次更新仅涉及一个叶子的祖先。 

由于所有阵列的节点数量都是线性的，因此内存使用量完全在限制范围内。

 ## 边缘情况

 一种重要的边缘情况是两个子树的总体积完全相同。 在这种情况下，获胜者一定是较小的索引，而不一定是具有较高叶值的索引。 例如，如果子树 A 的叶子总和为 5，而子树 B 的总和也为 5，则决策完全取决于当前获胜者的索引。 重新计算函数显式检查相等性并应用`min(wl, wr)`，无论内部结构如何，它都保持正确性。 

另一种情况是同一叶子上的重复更新。 由于更新是累加性的，因此叶值可以任意增大，但所有计算都保持在 Python 的整数范围内。 传播仍然只影响祖先节点，因此重复更新不会降低正确性，只会影响每次更新的树高。 

最后的结构情况是当树由于输入构造顺序而高度不平衡时。 即使在这种情况下，父指针传播仍然正确地重新计算所有受影响的祖先，因为每个内部节点都存储显式父链接。 该算法不假设平衡，因此即使根很深，正确性也不会受到影响。
