 ## 解决方案
 ## 问题理解

 我们得到了一个递归定义的分数族，它是由“一除以一加上类似的东西”形式的重复嵌套模式构建的。 该构造从一个简单的基值开始，然后重复地将先前的结果包装在相同结构的新层中。 

更具体地说，每个级别都会产生一个有理数。 第一个重要级别是通过取 1 并形成分子为 1、分母为 1 加上先前值的分数而获得的。 每个下一个级别都会重复完全相同的转换，始终对前一个分数应用相同的“包装”。 

任务不是输出分数本身，而是仅输出 N 层完全约简分数的分子。即使表达式增长为深度嵌套的连分数，但最终结果会简化为约简分数 a/b，并且要求我们仅打印 a。 

约束 N ≤ 40 意味着深度非常小，因此任何在每个级别执行恒定或线性工作的解决方案都足够了。 即使是显式操作大整数或分数的算法也是安全的，因为值像斐波那契数一样增长，并且根据竞争性编程标准保持很小。 

一个微妙的点是，表达式不是从左到右计算的，也不是简单地作为字符串展开的。 如果不仔细保留分数结构，则幼稚的符号展开很容易误导实现。 另一个常见的陷阱是未能在每一步减少分数，尽管在这个特定的递归中，如果构造正确，分数会自动保持在最低项。 

边缘情况很少。 当 N = 1 时，表达式只是 1/(1+1) = 1/2，因此分子为 1。在较高值时，结构很快类似于斐波那契增长，并且不正确的实现通常会交换分子和分母更新，从而产生移位序列。 

## 方法

 计算该值的直接方法是逐级显式构建连分数。 在每一步，我们存储当前分数 p = a/b 并应用变换

 p → 1 / (1 + p)。 

如果我们代入 p = a/b，则

 1 + p = (b + a) / b，因此

 1 / (1 + p) = b / (a + b)。 

这说明每一步都将(a,b)变换为(b,a+b)，这正是斐波那契递推结构。 分子和分母像连续的斐波那契数一样演变，只是顺序相反。 

从 p₁ = 1/2 开始，得到 (a, b) = (1, 2)。 重复应用更新会生成对 (1,2)、(2,3)、(3,5)、(5,8) 等。 因此，N 层的分子是该索引约定下的第 N 个斐波那契数。 

强力替代方案将使用有理算术或符号树在每个级别显式构造嵌套分数。 这是正确的，但从概念上讲效率很低，因为每个级别都会重复并嵌套整个先前的结构，如果扩展，则会导致指数表达式大小。 尽管 Python 分数仍然可以处理 N ≤ 40，但与线性递归相比，它是不必要的间接。 

关键的见解是连分数经过精心设计，以便每个包装步骤都将分数转换为分子和分母的简单线性变换。 这会将深度嵌套的表达式折叠成类似斐波那契的序列。

 | ---| ---| ---| ---|
 | 暴力破解 | O(2^N) 符号增长 | O(2^N) | O(2^N) | 太慢了|
 | 线性递归（斐波那契）| O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们希望在每个级别之后跟踪分数的分子和分母，而无需扩展嵌套表达式。

1. 从基本分数 p₁ = 1/2 开始，因此分子 a = 1，分母 b = 2。这是构造开始后第一个明确给定的级别。 
2. 对于从 2 到 N 的每个下一个级别，将当前分数 p = a/b 替换为 1 / (1 + p)。 我们不直接将其计算为十进制或嵌套表达式，因为这会破坏结构。 
3. 将 1 + p 重写为 (a + b) / b。 这一步至关重要，因为它将嵌套分数转换为可以干净地反转的简单有理表达式。 
4. 取倒数得到新的分数 b / (a + b)。 这直接将 (a, b) 对更新为 (b, a + b)。 这种变换保留了正确性，因为它纯粹源自分数的代数运算。 
5. 重复此更新 N − 1 次，因为初始状态已对应于级别 1。 
6. 完成所有转换后，输出a，即最终分数的分子。 

### 为什么它有效

 在每一步中，我们都保持不变式（a，b）代表当前水平的减少分数。 从 (a, b) 到 (b, a + b) 的变换正是应用 p → 1 / (1 + p) 的代数简化。 由于分子和分母都使用精确算术进行更新，并且变换保留了互质性，因此分数在整个过程中保持最低项。 这保证了在 N − 1 次应用之后，a 正是 p_N 的分子。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N = int(input().strip())

    a, b = 1, 2  # p1 = 1/2

    for _ in range(N - 1):
        a, b = b, a + b

    print(a)

if __name__ == "__main__":
    solve()
```该实现仅存储分子和分母并就地更新它们。 初始状态完全对应于第一个定义的分数，并且每次循环迭代应用导出的递推 (a, b) → (b, a + b)。 迭代次数为 N − 1，因为级别 1 已经初始化。 

关键的实现细节是同时交换和添加更新。 如果将其写为两个单独的赋值而不进行元组解包，则必须小心不要过早覆盖值。

 ## 边缘情况

 当 N = 1 时，算法初始化 (a, b) = (1, 2) 并执行零次迭代。 输出立即为 a = 1，与基本分数 1/2 匹配。 

对于 N = 2，我们执行一次更新。 从 (1,2) 开始，变换得到 (2,3)。 打印分子 2，与显式计算的二级分数相匹配。 

对于较大的 N，例如 N = 40，重复应用相同的不变性保留变换可以保证正确性。 在每一步中，分数都会减少，因为连续的斐波那契数是互质的，因此除了递归本身之外，不需要隐藏的简化。
