 ## 解决方案
 ## 问题理解

 我们得到两个一起传输的二进制字符串，但某些位置可能已被破坏为通配符`*`。 一个字符串代表一个大的二进制数，另一个字符串代表一个小得多的二进制数。 原始传输具有以下属性：第一个字符串编码的值可被第二个字符串编码的值整除。 

我们的任务是重建第一个数字的任何有效原始二进制字符串，与接收到的部分信息一致，使得第二个字符串至少存在一个完成，使整除条件为真。 一些位置固定为`0`或者`1`，还有一些是未知的。 尽管第一个字符串本身可以长达 500 位，但两个字符串中的未知位总数都很小，最多 16 个。 

这种结构意味着强烈的不对称性。 大量的数字可能有许多未知的位置，但总共只存在极少数。 第二个数字很短，最多 16 位，因此它的数值总是足够小，可以对其进行模运算。 可整除条件表明我们需要推理余数而不是完整整数。 

主要困难在于，两个字符串的简单重构会导致最多 16 个未知位的指数分支。 那最多就是$2^{16} = 65536$可能性，虽然是临界点，但如果每个候选者都得到有效检查，仍然是可行的。 

当第二个数字非常小时，特别是当它计算为 1 或 2 时，就会出现微妙的边缘情况。在这些情况下，整除性约束变得微不足道或退化，并且假设可逆性或非零模行为的粗心实现可能会失败。 另一个边缘情况是当通配符分布严重偏向大字符串时，迫使大多数分支发生在位置贡献中，而不是直接发生在模数中。 

## 方法

 暴力方法会尝试所有方法来替换每一个`*`在两个字符串中`0`或者`1`，计算结果整数，并检查整除性。 由于总共最多有 16 个未知位，因此这会产生最多$2^{16}$候选人。 每个候选者都需要构造潜在的 500 位整数，但这可以通过增量二进制评估或模块化算术来处理。 这是正确的，但如果重复直接在大整数上实现，就会变得脆弱，特别是如果我们每次都重新计算完整值。 

关键的观察是，我们永远不需要大数的完整值，只需要其对构造的小数取模的余数。 一旦确定了第二个数字的候选者，我们就可以计算它的值$N$，然后将大字符串解释为二进制数模$N$在一次通过中。 这将每次检查减少到大字符串长度的线性时间。 

因此，问题的结构变成了两级搜索。 首先，我们枚举小字符串的所有有效补全，因为它最多包含 16 个未知位并且无论如何都很短。 然后，对于每个这样的候选模数，我们计算是否存在与其一致的大字符串的补全，从而产生余数零。 由于未知位是全局有限的，因此我们可以将所有未知位置视为单个二元决策池，并以确保一致性的方式将它们分布在位置之间。 

蛮力之所以有效，是因为未知空间很小，但如果我们重复重新计算完整整数或将大字符串视为需要完整枚举，则会失败。 模块化算术压缩大字符串评估的观察结果允许每个候选分配在线性时间内得到验证。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过完全重新计算对所有作业进行暴力破解 |$O(2^{16} \cdot n)$具有大常数|$O(1)$| 太慢/脆弱|
 | 枚举小字符串赋值 + 模块化检查 |$O(2^{k} \cdot n)$,$k \le 16$|$O(1)$| 已接受 |

 ## 算法演练

 我们将未知结构分为两个阶段：选择控制号和验证消息。 

1. 枚举所有可能的替换`*`在第二个字符串中。 每个未知位要么是 0，要么是 1，因此我们生成除数的每个候选二进制数。 这是可行的，因为未知位的总数最多为 16，因此即使全部都在该字符串中，总组合仍然是有界的。 
2. 对于每个候选第二字符串，计算其整数值$N$。 如果$N = 0$，跳过它，因为整除性是未定义的，而且该问题保证至少有一个`1`存在，因此有效的候选人总是存在。 
3.对于固定的$N$，尝试构造第一个字符串的有效完成。 我们将第一个字符串视为二进制数并对其求模$N$。 我们不是构建完整的整数，而是从左到右扫描并保留余数。 在每个位置，如果该位是固定的，我们就使用它； 如果是的话`*`，我们尝试这两种可能性，但我们通过记住总分支在全球范围内是有限的，以受控的方式做到这一点。 
4. 为了避免大字符串的指数爆炸，我们利用了两个字符串中的未知位都是全局限制的事实。 我们将决策分配给这些未知位置作为单个状态空间。 每个完整的作业同时修复两个字符串。 
5. 对于每个完整赋值，我们在线性时间内计算两个字符串的整数值，并检查是否$M \bmod N = 0$。 如果成立，我们输出构建的$M$立即地。 
6. 如果没有适合特定选择的作业$N$，我们继续寻找下一个候选，直到找到有效的对。 

正确性取决于这样一个事实：接收到的字符串的每一种可能的完成都恰好对应于最多 16 个通配符位的一个分配，因此枚举所有分配可以保证覆盖所有有效的原始传输。 

### 为什么它有效

 不确定性的状态空间完全包含在通配符位置中。 每个这样的位置恰好贡献一个二元决策。 该算法枚举所有此类决策，并对每个决策评估归纳对是否满足整除性约束。 由于每个有效的原始传输都对应于这些分配之一，因此搜索不会错过解决方案。 同时，无效的补全不能通过模数检查，因为算术是针对每个候选者精确计算的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_candidates(s):
    pos = [i for i, c in enumerate(s) if c == '*']
    k = len(pos)
    res = []

    for mask in range(1 << k):
        arr = list(s)
        for i in range(k):
            arr[pos[i]] = '1' if (mask >> i) & 1 else '0'
        res.append(''.join(arr))
    return res

def to_int(bin_str):
    val = 0
    for c in bin_str:
        val = (val << 1) + (c == '1')
    return val

def check(M, N):
    if N == 0:
        return False
    val = 0
    for c in M:
        val = (val << 1) + (c == '1')
        val %= N
    return val == 0

def main():
    M_raw = input().strip()
    N_raw = input().strip()

    N_candidates = build_candidates(N_raw)
    M_candidates = build_candidates(M_raw)

    for N_str in N_candidates:
        N_val = to_int(N_str)
        if N_val == 0:
            continue

        for M_str in M_candidates:
            if check(M_str, N_val):
                print(M_str)
                return

if __name__ == "__main__":
    main()
```该解决方案首先扩展两个字符串的所有通配符补全。 这是安全的，因为通配符的总数最多为 16 个，因此组合搜索空间仍然是有限的。 每个候选除数都被转换为整数，然后通过计算除数的余数来测试每个候选消息。 

关键的实现细节是大型二进制字符串的模块化评估。 该代码不是构造大整数，而是使用移位来维护滚动余数，这确保了恒定的内存使用量和每次检查的线性时间。 

一个常见的错误是为每对重复重新计算整数值，这是不必要的，但在约束下仍然可以接受； 然而，使用模块化累积可以使解决方案保持稳定和快速。

 ## 边缘情况

 当第二个数字非常小时，就会出现一种边缘情况，例如`1`或者`10`。 如果是的话`1`，每个数字都是可整除的，因此第一个字符串的任何完成都是有效的。 该算法仍然有效，因为它测试所有分配并立即返回第一个有效分配。 

另一个边缘情况是所有通配符都位于第二个字符串中。 在这种情况下，外循环首先枚举所有除数。 对于每个除数，第一个字符串按原样进行测试。 这避免了两个字符串之间的任何依赖关系，并确保即使除数在候选之间发生巨大变化时也能确保正确性。 

最后一种边缘情况是重构字符串中出现前导零。 由于二进制解释允许前导零而不改变有效性，因此该算法不会拒绝它们。 任何此类完成都与整除性检查保持一致，因此返回任何有效字符串就足够了。
