 ## 解决方案
 ## 问题理解

 我们被要求为一个虚构的行星设计一个简化的“闰年系统”，该行星的一年长度不完全是当地天数的整数。 

从物理学角度来看，输入提供了足够的信息来计算行星完成绕恒星运行一周所需的时间。 假设轨道是圆形的，因此总行进距离与周长成正比，除以轨道速度即可得出轨道周期（以小时为单位）。 将其转换为当地时间（因为一天以小时为单位）会产生一个实数值，我们可以将其视为以“行星日”衡量的真正回归年。 

任务是使用类似于公历的日历规则来近似这个实数。 首先，我们通过将回归年四舍五入到最接近的整数来选择每年的基本整数天数，并向上舍入。 然后，我们引入由三个整数 n1、n2、n3 定义的周期性校正系统，其中 n1 除以 n2，n2 除以 n3，n3 最多为 1000。这些规则在长度为 n3 年的周期内创建闰年的重复模式，其中某些年份根据整除性添加或减去额外的一天。 

目标是选择 n1、n2、n3，以使所得的平均日历年长度尽可能接近真正的回归年。 

组合部分的约束很小，因为 n3 的上限为 1000。这是关键的结构限制：它允许枚举候选循环结构。 大值只出现在回归年的物理计算中，但这部分是O(1)。 

简单的解释可能会建议连续优化或数论反演，但 n3 上的离散界限迫使对除数链进行搜索。 

一个微妙的边缘情况在于舍入步骤。 如果回归年正好是 x.5，我们必须向上舍入。 这里的错误会使 d 改变 1，这会翻转所有校正项的方向并导致完全不同的最佳结构。 另一个陷阱是忘记了闰年添加或减去一天的闰年规则会根据基数舍入是向下还是向上舍入而变化。 

## 方法

 蛮力的想法是计算回归年 T，然后尝试所有有效的三元组 (n1, n2, n3)。 对于每个三元组，我们在长度为 n3 的完整周期上模拟日历规则，计算插入了多少额外或缺失的天数，并得出平均年长度。 

对于每个候选者，此模拟花费 O(n3) 时间，因为我们在周期中每年进行检查。 由于 n3 可以达到 1000，并且我们可能会尝试所有除数链，因此这很快就会变得太慢。 在最坏的情况下，n1 大约有 1000 个选择，n2 大约 1000 个，n3 大约 1000 个，最多给出 10^9 个状态，每个状态的成本为 O(1000)，远远超出了限制。 

关键的观察是，日历规则的结构使得平均值完全由整除计数而不是模拟来确定。 在长度为 n3 的完整周期中，可被 n1、n2 和 n3 整除的年数可以直接使用底除法来计算。 由于嵌套的可分性约束，包含-排除折叠成简单的封闭形式。 这将每个候选评估减少到 O(1)，使得枚举变得可行。 

因此，我们不是模拟年份，而是分析计算每个周期的预期修正，并将其与真实的回归年进行比较。

 ## 算法演练

 ### 1.计算轨道周期

 我们使用圆周运动以小时为单位计算回归年。 轨道长度与半径成正比，所以我们使用$2\pi r$作为周长近似值，除以速度 s 即可得到每个轨道的小时数，然后除以 h 即可转换为当地时间。 这给出了实数 T。 

### 2.转换为日历基准年

 我们通过将 T 舍入到最接近的整数来计算 d，其中 0.5 向上舍入。 这定义了基准年长度。 

这很重要的原因是所有校正逻辑都与该整数锚相关。 

### 3.确定校正方向

 如果 T 向下舍入，则闰年会增加 +1 天。 如果 T 向上舍入，则闰年减去 1 天。 

该选择决定了校正项的符号。 

### 4. 枚举除数链

 我们迭代所有有效的 n1 直到 1000。对于每个 n1，我们迭代所有倍数 n2。 对于每个 n2，我们迭代 n3 的所有倍数，直到 1000。 

该结构遵循约束 n1 | n2| n3。 

### 5.计算周期修正

 对于长度为 n3 的循环：

 我们使用整数除法计算每个约束可以整除多少年。 

我们定义：

 - 计数（n1）= n3 // n1
 - 计数（n2）= n3 // n2
 - 计数（n3）= n3 // n3 = 1

 由于嵌套，实际闰年的数量等于：

 计数 (n1) - 计数 (n2) + 计数 (n3)

 每个这样的年份根据舍入方向贡献+1 或-1。 

因此，整个周期的总修正量为：

 校正 = 符号 × (计数(n1) - 计数(n2) + 计数(n3)) / n3

 ### 6. 与目标比较

 我们计算候选人的年长：

 cand = d + 修正

 我们最小化绝对误差 |can 和 − T|。 

### 为什么它有效

 不变量是每个有效日历都是周期性的，周期为 n3，并且在该周期内，闰规则的贡献仅取决于整除计数。 由于规则是通过构造 (n1 | n2 | n3) 嵌套的，因此包含-排除是精确的，并且不存在重叠歧义。 因此，每年的平均修正量完全由n1、n2、n3的简单算术确定，并且枚举涵盖了所有可能性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    r, s, h = map(int, input().split())

    # tropical year in days
    # orbital period (hours) = 2πr / s
    # convert to planet days: divide by h
    T = (2.0 * 3.141592653589793 * r) / (s * h)

    # rounding with .5 up
    d = int(T + 0.5)

    # direction of leap adjustment
    # if rounded down => add leap days
    # if rounded up   => subtract leap days
    if T >= d:
        sign = 1.0   # rounded down or exact integer
    else:
        sign = -1.0  # rounded up

    best = float("inf")
    ans = (1, 2, 3)

    for n1 in range(2, 1001):
        for n2 in range(n1 * 2, 1001, n1):
            for n3 in range(n2 * 2, 1001, n2):
                cnt1 = n3 // n1
                cnt2 = n3 // n2
                cnt3 = 1

                leaps = cnt1 - cnt2 + cnt3

                avg = d + sign * (leaps / n3)

                err = abs(avg - T)

                if err < best:
                    best = err
                    ans = (n1, n2, n3)

    print(ans[0], ans[1], ans[2])

if __name__ == "__main__":
    solve()
```T的计算直接编码轨道模型。 舍入步骤是谨慎实施的，并倾向于向上舍入，使用`+0.5`。 符号处理将闰年增加或减少天数的两种情况分开，这一点至关重要，因为它翻转了优化方向。 

三重循环通过以 n1 的倍数步进 n2 和以 n2 的倍数步进 n3 自然地强制执行整除性约束。 这避免了显式的 gcd 检查并保证了所有候选人的有效性。 

修正公式用封闭式计数取代了对年度行为的任何模拟，这使得解决方案足够快。

 ## 边缘情况

 一种极端情况是回归年非常接近整数加 0.5。 在这种情况下，舍入行为变得不稳定。 该算法可以正确处理它，因为比较 T >= d 一致地定义了符号，确保校正方向符合舍入规则。 

另一种情况是当 n1 = 2 是最佳的并且 n2、n3 增长为大倍数时。 枚举仍然涵盖了它，因为 n2 和 n3 是作为倍数生成的，没有密度限制。 

最后一种情况是 T 恰好是整数。 然后 d 等于 T，符号分支将其视为非负，并且最优结构自然地向最小化校正幅度崩溃，枚举正确地捕获了该校正幅度。
