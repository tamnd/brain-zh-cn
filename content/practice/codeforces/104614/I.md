 ## 解决方案
 ## 问题理解

 该城市被建模为无向加权图。 每个交叉点都是一个顶点，每条道路都是一条边，其权重为其长度。 市长总是从十字路口行驶`a`到交叉路口`b`，而帕特只想铺设属于这两个位置之间至少一条最短路径的道路。 

任务不是找到一条最短路径。 如果存在使用某条道路的最短路径，则应铺设该道路。 剩下的每条路都是不必要的，我们必须输出它们的长度之和。 

该图最多包含 100 个顶点，而语句中的路数不受限制，但永远不能超过简单无向图中的边数，即`n(n-1)/2`。 即使最稠密的图也只有不到 5000 条边。 这些限制足够小，多次运行 Dijkstra 算法是完全可行的。 不需要更复杂的最短路径数据结构。 

主要困难是可能存在几条不同的最短路线。 仅重建一条最短路径的解决方案将错过属于另一条同样短的路线的道路。 

考虑下图。```
4 4 1 4
1 2 1
2 4 1
1 3 1
3 4 1
```正确的输出是```
0
```两条路线均从`1`到`4`有长度`2`，所以每条路都属于某个最短路径。 在 Dijkstra 期间仅存储一个父级的解决方案将错误地仅标记一个路由和输出`2`。 

另一个微妙的情况是当直接边与更长的替代边同时存在时。```
3 3 1 3
1 2 2
2 3 2
1 3 4
```正确的输出是```
0
```两条路径都有总长度`4`，所以每条边都必须铺好。 仅寻找更短的替代道路会错误地排除直接道路。 

当道路不可能出现在任何最短路径上时，就会出现最后的边缘情况。```
3 3 1 3
1 2 1
2 3 1
1 3 5
```正确的输出是```
5
```长的直接边从来没有用，因此它的长度有助于答案。 

## 方法

 一个简单的想法是独立测试每条道路。 移除道路，计算最短路径，然后强制道路进入一条路径，并检查所得距离是否等于全局最优距离。 这是可行的，因为每条边都是单独检查的，但它需要为每条道路运行最短路径算法。 大约有 5000 条边，每一条 Dijkstra 都取`O(m log n)`，总功变为`O(m² log n)`，即使对于这些约束，这也是不必要的。 

关键的观察是，一条边是否属于某个最短路径仅取决于距两个端点的距离。 

假设一条边连接`u`和`v`有重量`w`。 如果我们已经知道距离的最短距离`a`到每个顶点以及从每个顶点到`b`，那么我们可以立即测试两个可能的方向。 

边缘出现在最短路径上，如果`distA[u] + w + distB[v] == distA[b]`或者`distA[v] + w + distB[u] == distA[b]`。 

第一个方程对应于达到`u`，遍历边到`v`，然后继续`b`。 第二个处理相反的方向，因为图是无向的。 

这将问题简化为两次 Dijkstra 运行，然后扫描一次边缘列表。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |`O(m² log n)`|`O(n + m)`| 太慢了 |
 | 最佳|`O(m log n)`|`O(n + m)`| 已接受 |

 ## 算法演练

 1. 为无向加权图构建邻接表，同时将每条边存储在单独的列表中。 Dijkstra 使用邻接列表，而最终扫描需要边列表。 
2.从市长家开始运行Dijkstra`a`。 记录最短距离`a`到数组中的每个顶点`distA`。 
3.从市长办公室开始再次运行Dijkstra`b`。 记录最短距离`b`到数组中的每个顶点`distB`。 
4.让`best = distA[b]`，这是两个位置之间每条最短路线的长度。 
5.检查每条道路`(u, v, w)`一次。 检查是否`distA[u] + w + distB[v]`或者`distA[v] + w + distB[u]`等于`best`。 
6. 如果两个等式都不成立，则没有最短路径可以包含这条路，因此将其长度添加到答案中。 
7. 打印累计金额。 

### 为什么它有效

 第一个 Dijkstra 计算从一开始就到达每个顶点的最小可能成本。 第二个计算从每个顶点到目的地的最小可能剩余成本。 

假设一条边属于最短路径。 穿过该路径以最佳方式到达一个端点，穿过边缘，然后以最佳方式到达目的地。 否则路径可能会缩短。 因此，两个等式之一必须成立。 

相反，如果其中一个等式成立，则连接到第一端点的最短路径、边缘本身以及从第二端点开始的最短路径会产生总长度等于全局最优的路径。 因此，边缘位于至少一条最短路径上。 

该测试既是必要的也是充分的，因此每条道路都被正确分类。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def dijkstra(start, graph, n):
    INF = 10 ** 18
    dist = [INF] * (n + 1)
    dist[start] = 0
    pq = [(0, start)]

    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

def solve():
    n, m, a, b = map(int, input().split())

    graph = [[] for _ in range(n + 1)]
    edges = []

    for _ in range(m):
        u, v, w = map(int, input().split())
        graph[u].append((v, w))
        graph[v].append((u, w))
        edges.append((u, v, w))

    distA = dijkstra(a, graph, n)
    distB = dijkstra(b, graph, n)

    shortest = distA[b]
    ans = 0

    for u, v, w in edges:
        if (distA[u] + w + distB[v] == shortest or
                distA[v] + w + distB[u] == shortest):
            continue
        ans += w

    print(ans)

if __name__ == "__main__":
    solve()
```解决方案直接遵循算法。 该图被存储两次，一次作为邻接列表，用于高效的最短路径计算，一次作为普通边列表，因此每条道路都可以精确检查一次。 

Dijkstra 实现使用标准惰性优先级队列技术。 每当从堆中删除过时的条目时，都会通过将其距离与当前最佳值进行比较来忽略它。 

最终扫描对每个无向边执行两次相等测试。 检查两个方向至关重要，因为最短路径可能沿任一方向穿过边缘。 由于 Python 整数具有任意精度，因此即使对多个距离求和也不存在溢出风险。

import io
import heapq

def solve():

    n, m, a, b = map(int, input().split())

    graph = [[] for _ in range(n + 1)]
    edges = []

    for _ in range(m):
        u, v, w = map(int, input().split())
        graph[u].append((v, w))
        graph[v].append((u, w))
        edges.append((u, v, w))

    def dijkstra(s):
        INF = 10 ** 18
        dist = [INF] * (n + 1)
        dist[s] = 0
        pq = [(0, s)]
        while pq:
            d, u = heapq.heappop(pq)
            if d != dist[u]:
                continue
            for v, w in graph[u]:
                nd = d + w
                if nd < dist[v]:
                    dist[v] = nd
                    heapq.heappush(pq, (nd, v))

    db = dijkstra(b)

        ans += w
    print(ans)

    sys.stdout = io.StringIO()
    solve()

 ## 边缘情况

 第一个不明显的情况是多条最短路径。```
4 4 1 4
1 2 1
2 4 1
1 3 1
3 4 1
```最短距离是`2`。 对于每条边，两个相等检查之一成功，因此每条边都被分类为属于最短路径。 算法输出`0`，而基于父级的重建会错误地丢弃一半边缘。 

第二种情况是两条完全不同的路线总长度相同。```
3 3 1 3
1 2 2
2 3 2
1 3 4
```最短距离是`4`。 直接边和双边路线均满足相等性检验，因此所有三条路均已铺设，答案为`0`。 

最后一个案例包含一条比所有替代方案都更糟糕的道路。```
3 3 1 3
1 2 1
2 3 1
1 3 5
```最短距离是`2`。 长边产生`0 + 5 + 0 = 5`，它大于最优值，因此两个等式都不成立。 它的长度被添加到答案中，产生正确的输出`5`。
