 ## 解决方案
 ## 问题理解

 我们有一个矩形迷宫和两个机器人，它们被放置在具有初始方向的不同单元上。 迷宫是一个网格，其中的运动被内墙和外边界阻挡，除了位于一个特定单元的南边界的单个出口之外。 

两个机器人同时收到完全相同的命令。 每个命令要么是向前移动，要么是旋转，并且旋转是免费的。 当发出前进命令时，两个机器人都会尝试朝当前方向移动一步。 如果机器人面对墙壁，它不会移动，而是会记录到碰撞。 如果它在面朝南的情况下通过指定的出口单元离开网格，它就会离开迷宫并且不再受到未来命令的影响。 

目标是发送一系列前进命令，最终让两个机器人都出去。 我们首先最小化前向命令的数量，并且在所有最优解决方案中，我们最小化碰撞的总数。 一个关键的限制是机器人在退出之前的任何时候都不允许占据同一个单元。 

网格最多为 50 x 50，因此最多有 2500 个单元格。 对于两个机器人，初始关节位置空间已经达到约 600 万个状态，并且当包括方向时，它会变得更大。 这立即排除了对命令序列的任何指数模拟。 

一个微妙的边缘情况来自于共享的移动规则。 如果两个机器人都被前向命令推入同一个单元，即使它们从不同方向到达，该状态也是无效的，必须避免。 另一个边缘情况是退出行为：机器人只有在指定的退出单元上并向南移动时才退出； 仅仅站在出口单元上是不够的。 

## 方法

 直接模拟将尝试搜索命令序列。 每个状态都包括机器人位置和方向，每个步骤都会分支三个可能的命令。 由于前向命令以非平凡的方式交互（阻塞、碰撞和可能的退出），因此这很快就会变成一个大的分支过程。 即使忽略方向，探索所有长度序列直到答案也会导致指数爆炸。 

关键的简化来自于旋转命令是免费的这一事实。 因为两个机器人总是一起旋转，所以我们永远不会被迫永久“承诺”一个方向。 在执行任何前向命令之前，我们可以以零成本任意旋转系统，这意味着每个前向步骤都可以解释为从四个罗盘方向中选择一个方向并立即应用它。 

这消除了跟踪方向历史记录的需要。 相反，每个状态只需要记住每个机器人当前所在的位置（或者是否已经退出）。 从任何状态，我们都可以尝试所有四种可能的前进方向。 

这将问题转化为最短路径搜索，其中每个状态转换的每个前向命令的成本为 1，次要成本等于该移动引起的颠簸数量。 由于所有边都具有非负权重，并且我们需要字典顺序上的最小距离，因此联合状态空间上的 Dijkstra 算法是合适的。 

状态空间由成对的位置组成，每个机器人都有一个附加的“出”标记。 这最多产生大约 (2501 × 2501) 个状态，虽然很大，但通过修剪和高效散列或索引仍然可行。 每个状态最多有四个传出转换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力命令搜索 | 指数| 指数| 太慢了 |
 | Dijkstra 谈联合立场 | O(V log V + E log V)，其中 V ≈ 6.25e6 | O(V) | 已接受 |

 ## 算法演练

我们将迷宫转换为可以快速回答运动查询的结构：对于每个单元和方向，我们知道运动是否被墙壁或边界阻挡。 

然后，我们对组合的机器人配置运行最短路径算法。 

### 步骤

 1. 将每个机器人的状态表示为网格单元或“退出”标记。 完整的状态是这两个。 这会随时压缩有关系统的所有相关信息。 
2. 建立墙查找表，以便我们可以从任何单元格检查向北、向南、向东或向西移动是否被阻止。 出口侧经过特殊处理：出口单元的南边仅对该单元开放。 
3. 为 Dijkstra 算法初始化一个优先级队列。 初始状态是两个机器人的给定起始位置，前进命令为零，碰撞为零。 
4. 对于每个弹出状态，尝试所有四个可能的前进方向。 对于每个方向，同时模拟两个机器人：

 如果机器人已经出局，它就会继续出局。 

如果机器人在迷宫中并且可以朝那个方向移动，那么它就会移动。 

如果它由于墙壁而无法移动，它将保留下来并算作碰撞。 

如果它位于出口单元并尝试向南移动，它会退出而不是停留。 
5. 如果两个机器人在移动后占据同一个单元并且都没有出去，则放弃此转换。 这可以防止非法重叠配置。 
6. 对于每个有效的结果状态，更新其成本对。 主键是转发命令的数量，辅助键是总碰撞次数。 如果该状态改善了其最著名的对，则推入优先级队列。 
7. 当第一次达到两个机器人都出局的状态时停止，因为 Dijkstra 保证了最优性。 

### 为什么它有效

 在任何时候，算法都保持优先级队列按（前向命令、碰撞）递增的字典顺序探索状态的不变性。 每一次转换都完全对应于一致应用于两个机器人的一个全局前向命令。 由于旋转是自由的，因此忽略方向历史记录不会排除任何有效行为。 两个机器人第一次被标记为出局时，我们已经找到了全局最优序列，因为任何后续发现都将需要至少同样多的前进步骤，并且已经通过碰撞将平局最小化。 

## Python 解决方案```python
import sys
import heapq
input = sys.stdin.readline

# directions: N, E, S, W
dx = [0, 1, 0, -1]
dy = [1, 0, -1, 0]

def solve():
    c, r, e = map(int, input().split())

    c1, r1, d1, c2, r2, d2 = input().split()
    c1, r1 = int(c1)-1, int(r1)-1
    c2, r2 = int(c2)-1, int(r2)-1

    dir_map = {'N':0,'E':1,'S':2,'W':3}

    # walls
    horiz = [[False]*r for _ in range(c)]   # between (x,y) and (x,y+1)
    vert = [[False]*r for _ in range(c)]    # between (x,y) and (x+1,y)

    parts = list(map(int, input().split()))
    n = parts[0]
    idx = 1
    for _ in range(n):
        x = parts[idx]-1
        y = parts[idx+1]-1
        idx += 2
        horiz[x][y] = True

    parts = list(map(int, input().split()))
    n = parts[0]
    idx = 1
    for _ in range(n):
        x = parts[idx]-1
        y = parts[idx+1]-1
        idx += 2
        vert[x][y] = True

    exit_x = e - 1
    exit_y = 0  # row 1

    def move(x, y, d):
        if x == -1:
            return (-1, 0, 0)  # already out

        nx, ny = x + dx[d], y + dy[d]

        # exiting condition
        if x == exit_x and y == exit_y and d == 2:
            return (-1, 0, 0)

        bump = 0

        # boundary / wall checks
        if nx < 0 or nx >= c or ny < 0 or ny >= r:
            return (x, y, 1)

        # horizontal walls
        if d == 0 and horiz[x][y]:
            return (x, y, 1)
        if d == 2 and horiz[x][y-1]:
            return (x, y, 1)

        # vertical walls
        if d == 1 and vert[x][y]:
            return (x, y, 1)
        if d == 3 and vert[x-1][y]:
            return (x, y, 1)

        return (nx, ny, 0)

    def encode(x1, y1, x2, y2):
        return ((x1+1)*(r+1)+y1) * ((c+1)*(r+1)) + (x2+1)*(r+1)+y2

    INF = (10**18, 10**18)
    dist = {}

    start = (c1, r1, c2, r2)
    pq = [(0, 0, c1, r1, c2, r2)]
    dist[start] = (0, 0)

    while pq:
        f, b, x1, y1, x2, y2 = heapq.heappop(pq)

        if dist.get((x1,y1,x2,y2), INF) != (f, b):
            continue

        if x1 == -1 and x2 == -1:
            print(f, b)
            return

        for d in range(4):
            nx1, ny1, b1 = move(x1, y1, d)
            nx2, ny2, b2 = move(x2, y2, d)

            if nx1 != -1 and nx2 != -1 and nx1 == nx2 and ny1 == ny2:
                continue

            nf = f + 1
            nb = b + b1 + b2

            state = (nx1, ny1, nx2, ny2)
            if dist.get(state, INF) > (nf, nb):
                dist[state] = (nf, nb)
                heapq.heappush(pq, (nf, nb, nx1, ny1, nx2, ny2))

solve()
```实现的核心是`move`函数，它将一个全局前向命令应用于单个机器人。 它仔细区分了三种情况：退出、撞到墙壁或边界以及成功移动。 在进行一般移动之前必须检查出口箱，因为从出口单元向南走会完全移除机器人。 

Dijkstra 循环将每对机器人位置视为一个节点。 每条边对应于选择四个方向之一并同时应用它。 优先级队列中的字典顺序确保首先最小化前向命令，然后最小化碰撞，而不需要单独的松弛结构。 

移动后的碰撞检查可防止两个机器人占据同一单元的非法中间状态，这是明确禁止的。

 ## 边缘情况

 当两个机器人都靠近出口单元但只有一个机器人可以在不阻挡另一个机器人的情况下退出时，就会出现一种边缘情况。 该算法可以处理此问题，因为退出被视为独立于其他机器人位置的状态转换。 

另一种边缘情况是循环地反复撞击墙壁。 由于每个状态都由位置和累积成本决定，因此会忽略重新访问具有更差成本的相同配置，从而防止无限循环。 

最后一个微妙的情况是当一个机器人退出而另一个机器人仍在里面时。 退出的机器人转换为终端状态，不再参与碰撞或运动，这自然可以通过将其视为固定的“退出”标记来处理。
