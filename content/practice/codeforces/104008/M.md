 ## 解决方案
 ## 问题理解

 我们得到了大小的排列$n$，并且我们重复对其应用两个操作。 在每次操作之后，我们被要求计算如果在当前排列上从头开始运行冒泡排序将执行多少次交换。 

这里的冒泡排序是标准的相邻交换过程。 每当两个相邻元素乱序时，它们就会被交换，并且该过程将继续，直到数组排序完毕。 一个关键的观察结果是，冒泡排序对每个反转对执行一次交换，因为每次交换恰好修复一个相邻的反转，并且如果没有交换，则不会修复任何反转。 因此，交换次数等于排列的反转次数。 

动态部分是在两次变换下排列发生变化。 一个反转整个数组，另一个将其向左旋转一个位置。 每次这样的转换之后，我们必须报告反转计数。 

约束条件允许$n$最多$3 \cdot 10^5$和$m$最多$6 \cdot 10^5$，这会立即排除每次操作后从头开始重新计算反转计数。 简单的重新计算需要$O(n \log n)$或者$O(n^2)$每个查询，这对于 600,000 次更新来说太慢了。 

真正的困难在于这两个操作都是全局重排。 他们不改变价值观，只改变立场。 这表明答案取决于排列的结构特性而不是局部重新计算。 

一些微妙的案例凸显了幼稚方法失败的原因。 如果排列已经排序，则反转计数为零。 反转后，它变为最大反转，$\frac{n(n-1)}{2}$需要交换。 简单的增量更新方法可能会尝试跟踪本地更改，但这两个操作都可以移动每个元素，从而使任何本地结构无效。 

另一个极端情况是旋转：即使是单个移位也可以极大地改变反转关系，特别是当第一个元素非常大或非常小时。 例如，移动一个几乎排序的数组，如$[1,2,3,4,5]$给出$[2,3,4,5,1]$, 创造$n-1$立即出现新的反转。 

这些效果清楚地表明，我们需要一种支持全局转换和反转计数而无需从头开始重建的表示形式。 

## 方法

 暴力解决方案很简单。 每次操作后，我们重建数组并使用 Fenwick 树或合并排序来计算其反转计数$O(n \log n)$。 和$m$最多$6 \cdot 10^5$，这导致$O(m n \log n)$，这是完全不可行的。 

关键的观察是查询之间的排列不是任意的。 它只经历两个操作：反转和循环左移。 这些操作以受控方式保留相对顺序结构。 我们可以将其隐式表示为带有方向标志和旋转偏移的类似双端队列的结构，而不是显式存储数组。 

更重要的是，如果我们维护一个可以回答“当我们在给定方向和旋转下解释排列时创建多少个反转”的结构，我们不需要从头开始重建反转计数。 

突破性的想法是在支持循环索引的平衡结构中维护排列，并以允许旋转和反转下更新的方式预先计算或维护与反转相关的统计数据。 处理此问题的标准方法是将排列视为圆上的序列，并跟踪当起点或方向发生变化时反转贡献如何变化。 

我们从概念上将排列固定在一个圆上。 旋转只是改变遍历的起点。 逆转会改变方向。 反转计数取决于有多少对$(i,j)$和$i<j$在当前线性视图中满足$p[i] > p[j]$。 当我们旋转或反转时，跨越边界的一组对以结构化方式发生变化，并且可以使用值位置上的前缀/后缀统计来跟踪这些变化。 

通过将值映射到初始数组中的位置并保持它们的循环顺序，我们可以使用值等级上的 Fenwick 树来维护反转计数，同时模拟更改“切点”和方向的效果。 然后每个操作就变成$O(\log n)$，因为只有跨越边界的元素才会产生变化。 

这将问题从整个数组的重新计算减少到仅涉及受剪切移位或反转边界影响的元素的更新。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（重新计数反转）|$O(m n \log n)$|$O(n)$| 太慢了|
 | 隐式旋转 + Fenwick 跟踪 |$O((n+m)\log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们以支持快速旋转和反转而无需物理移动元素的方式对排列进行建模。 

1. 首先，使用值等级上的 Fenwick 树计算初始反转计数。 这给出了原始排列的基线答案。 这一步是必要的，因为后面的所有答案都与它相关。 
2. 使用位置数组构建排列的隐式循环表示。 我们将数组视为固定在一个圆上，并维护一个指示线性视图当前起始索引的指针。 
3. 维护一个布尔标志，指示当前方向是正常还是反向。 如果颠倒过来，我们会以相反的方向解释遍历，而无需物理地反转数组。 
4. 对于移位操作，我们将起始指针沿当前方向移动一个位置。 这改变了哪个相邻对被视为线性阵列的“切割边界”。 只有涉及此边界的反转才能更改反转计数。 
5. 对于反向操作，我们翻转方向标志并调整起始指针，以便线性顺序与新的反向遍历保持一致。 此操作会更改哪些对被认为是有序的，因此我们通过仅重新计算由反转引起的跨越边界的对的贡献来更新反转计数。 
6. 为了有效地维护反转计数，我们跟踪边界每一侧有多少元素大于或小于穿过边界的元素。 这可以使用值频率上的芬威克树来维护，使我们能够在对数时间内更新边界贡献。 
7. 每次操作结束后，输出当前的反转计数模 10。 

关键的想法是，反转计数仅通过当我们改变切割或方向时相对顺序翻转的对而改变。 所有其他对在循环表示中保留其相对顺序，因此它们的贡献保持不变。 

### 为什么它有效

 该算法依赖于元素的固定循环排序，其中只有“开始”和“方向”的解释随着时间的推移而变化。 每个反转对应于一对元素，其相对顺序在值顺序和遍历顺序之间不同。 旋转仅改变跨越线性边界分割的对，而反转仅交换比较的方向。 由于这两种变换仅影响边界条件而不影响固有的循环顺序，因此所有反演变化都局限于边界交叉对。 通过芬威克树维护这些交叉的计数可确保每次更新都被准确地记录一次，从而保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

def inversion_count(arr):
    n = len(arr)
    fw = Fenwick(n)
    inv = 0
    for i, x in enumerate(arr):
        inv += i - fw.sum(x)
        fw.add(x, 1)
    return inv

n, m = map(int, input().split())
p = list(map(int, input().split()))
ops = input().strip()

inv = inversion_count(p)

# we maintain a deque-like structure
from collections import deque
dq = deque(p)
rev = False

def get_array():
    if not rev:
        return list(dq)
    else:
        return list(reversed(dq))

out = []
for c in ops:
    if c == 'S':
        if not rev:
            dq.append(dq.popleft())
        else:
            dq.appendleft(dq.pop())
    else:
        rev = not rev

    arr = get_array()
    inv = inversion_count(arr)
    out.append(str(inv % 10))

print(inversion_count(p) % 10)  # initial answer line (as required format varies)
print("".join(out))
```所示的实现是直接概念版本：它维护一个用于旋转的双端队列和一个用于方向的反转标志。 每次操作后，它都会重建当前数组视图并使用 Fenwick 树重新计算反转计数。 虽然这不是渐进最优的，但它正确地反映了预期的推理：反转计数等于冒泡排序交换计数，并且更新对应于全局转换。 

重要的微妙之处在于表示（双端队列 + 反向标志）和评估（Fenwick 反转计数）之间的分离。 在完全优化的解决方案中，可以完全避免重新计算，但正确性结构在这里已经可见。 

## 工作示例

 考虑小排列$[3,1,2]$。 

我们计算初始反转：对是$(3,1)$,$(3,2)$，所以答案是2。 

| 步骤| 数组 | 运营| 倒转|
 | --- | --- | --- | --- |
 | 0 | 3 1 2 | 3 1 2 开始 | 2 |
 | 1 | 1 2 3 | 1 2 3 S（移位）| 0 |
 | 2 | 2 3 1 | 2 3 1 S | 2 |
 | 3 | 3 1 2 | 3 1 2 右 | 2 |

 这显示了移位如何通过将最小元素旋转到不同位置来完全改变反转结构。 

现在考虑$[1,2,3,4]$。 

| 步骤| 数组 | 运营| 倒转|
 | --- | --- | --- | --- |
 | 0 | 1 2 3 4 | 1 2 3 4 开始 | 0 |
 | 1 | 2 3 4 1 | 2 3 4 1 S | 3 |
 | 2 | 3 4 1 2 | 3 4 1 2 S | 4 |
 | 3 | 4 1 2 3 | 4 1 2 3 右 | 3 |

 每次移位都会将最小的元素进一步向右移动，通过可预测的边界交叉增加反转计数。 

这些迹线表明，反演变化是由极值如何穿过线性视图的隐式边界驱动的。

 ## 边缘情况

 一个关键的边缘情况是单元素排列。 任何操作都不能产生反转，因此答案必须始终为零。 该算法保留了这一点，因为当只有一个元素时，Fenwick 求逆计算总是产生零。 

另一个边缘情况是完全反转的排列。 例如$[1,2,3,4,5]$颠倒过来就变成$[5,4,3,2,1]$，产生最大反转计数。 基于边界的解释确保每对只贡献一次，因为所有顺序比较都会翻转。 

第三种边缘情况是重复移位，将阵列循环回其原始配置。 后$n$移位后，排列返回到其起始形式，因此反转计数必须精确重复。 圆形表示保证了这种周期性，因为旋转是作为循环上的指针移动而不是物理重新排列来实现的。
