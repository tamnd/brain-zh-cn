 ## 解决方案
 ## 问题理解

 我们得到了一个一维的细胞带。 每个单元格要么是空的，要么包含一朵百合花，由字符串表示，其中`L`表示百合已经种下并且`.`意味着细胞是空的土壤。 

我们可以将猫粮放在一些空的单元格上，将它们标记为`C`。 但是，将猫粮放置在`i`施加严格的安全约束：单元本身及其直接邻居`i-1`和`i+1`不得含有百合花。 换句话说，猫粮的放置会“阻止”其周围半径 1 范围内的百合花。 

任务是选择猫粮的位置，以最大化猫粮的数量`C`我们放置单元格时要尊重这一限制。 任何能够实现最大猫粮放置数量的有效配置都是可以接受的。 

约束条件`n ≤ 1000`意味着即使二次或轻度三次解也可以，但问题的结构强烈表明贪婪或线性扫描就足够了。 由于每个决策仅影响相邻小区，因此不需要全局搜索或DP。 

当百合花非常茂密时，会出现微妙的边缘情况。 例如，如果字符串是`LLL`，任何地方都不能放置猫粮，因为与百合相邻的每个空单元格都会立即违反约束。 另一个边缘情况是交替模式，例如`L.L.L`，天真的贪婪放置可能会错误地尝试放置`C`距离百合太近而没有检查两个邻居。 

如果一个粗心的方法经常会失败，因为只要一个相邻的单元是安全的，就放置猫粮，而不是验证两侧。 例如，在`..L..`，将猫粮放置在位置 2 会阻止位置 3 被使用，并且贪婪的从左到右放置必须考虑到这种传播。 

## 方法

 暴力策略将尝试空单元格的每个子集并检查有效性。 对于每个子集，我们将扫描字符串并验证没有选择`C`违反了附近百合花的规则。 最多有`n`位置，所以子集的数量是`2^n`，每张支票的费用`O(n)`，导致`O(n·2^n)`运营。 即使对于小公司来说，这也是不可能的`n`。 

关键的观察结果是，每个猫粮放置位置仅限制大小为 3 的局部邻域。这个局部性意味着如果我们从左到右扫描，我们永远不需要重新考虑之前的决定。 一旦我们放置了一个`C`在位置`i`, 职位`i-1`和`i+1`如果它们包含百合花，它们就会被有效禁止，我们自然会在贪婪的构造中跳过它们。 

最佳结构是遍历字符串并放置一个`C`每当当前位置为空且其邻居都不包含违反规则的百合花时。 放置一个后`C`，我们跳过下一个单元格以避免未来放置中意外的邻接冲突。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了|
 | 贪婪扫描 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理字符串并逐步构建答案。 

1. 从索引 0 开始并维护网格的可变副本。 

我们需要可变性，因为放置一个`C`改变未来允许的职位。 
2. 各位置`i`，检查单元格是否为空。 

如果它已经包含`L`，我们不能在那里放置任何东西，所以我们继续前进。 
3. 如果电池是`.`，我们检查是否放置了`C`这里就违反了规则。 

这需要确保既不`i-1`,`i`，也不`i+1`包含`L`。 
4. 如果条件满足，我们放置一个`C`在位置`i`。 

一旦我们这样做了，我们会立即对其进行标记，以便将来的决策看到其效果。 
5. 放置后`C`，我们通过递增跳过下一个位置`i`2 而不是 1。 

这可以防止意外放置在受限邻域中造成重叠冲突。 
6.如果我们不放置`C`，我们只需移至下一个索引。 

### 为什么它有效

 关键的不变量是每次我们放置一个`C`，所有禁止的交互都是严格局部的，并且永远不会传播到距离 1 之外。这意味着先前做出的决策不会因后来的选择而无效。 由于我们只放置一个`C`当它的邻域是干净的时，以后的放置不会引起涉及该单元的冲突，因为任何未来`C`至少有两步之遥。 因此，贪心选择是安全且局部最优的，并且局部最优意味着全局最优，因为每个放置恰好消耗了最小禁止区域。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = list(input().strip())
    n = len(s)

    i = 0
    while i < n:
        if s[i] == '.':
            left_ok = (i == 0 or s[i - 1] != 'L')
            right_ok = (i == n - 1 or s[i + 1] != 'L')

            if left_ok and right_ok:
                s[i] = 'C'
                i += 2
                continue
        i += 1

    print(''.join(s))

if __name__ == "__main__":
    solve()
```该实现反映了贪婪扫描。 我们将字符串转换为列表以允许就地更新，因为 Python 字符串是不可变的。 两个边界检查干净地处理边缘单元，没有索引错误。 放置一个后`C`，跳过下一个索引是一种实用的优化，它还可以防止重新评估邻域刚刚被“使用”的位置。 

一个常见的陷阱是在放置之前忘记检查两个邻居`C`。 另一个是未能正确处理边界，这会导致不正确的拒绝或索引错误`i = 0`或者`i = n - 1`。

 ## 边缘情况

 对于像这样的单单元格字符串`.`该算法立即放置一个`C`因为两个邻居检查都没有通过。 不存在边界风险`i == 0`和`i == n-1`都是真的。 

对于完全阻塞的字符串，例如`LLL`，每个位置都会被跳过，因为初始检查`s[i] == '.'`到处都会失败，因此不会尝试无效的放置。 

对于边界密集的图案，例如`.L`或者`L.`，边缘检查正确地将缺失的邻居视为安全。 例如在`.L`，索引 0 为空，但其右邻居为`L`，所以没有发生放置，产生`.L`。
