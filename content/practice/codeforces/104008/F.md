 ## 解决方案
 ## 问题理解

 我们在平面上给出了几个几何区域。 每个区域都是一个圆形扇区：由中心点、半径（由从中心到两个边界点的距离隐式给出）以及从中心开始的两条射线之间的角跨度定义的圆盘的一部分。 该扇区始终是从射线逆时针扫描$OA$射线$OB$，且角度保证不超过180度。 

任务是计算所有这些扇区的并集所覆盖的总面积。 重叠区域只能计算一次，因此对各个扇区面积进行简单求和是无效的。 

这些约束足够严格，迫使我们远离任何简单的几何叠加。 和$n \le 1000$，二次或近二次方法是边缘方法，但如果每次交互都是对数的或使用仔细的事件处理，则可能可以接受。 然而，由于坐标范围高达$10^4$以及需要$10^{-6}$精确。 

一个微妙的困难是扇区可以以复杂的曲线方式重叠。 与多边形不同，它们的边界包括圆弧，因此标准多边形并集技术是不够的，除非扩展为显式处理圆弧。 

一些边缘情况很重要：

 一个问题是退化角跨度接近于零。 A、O、B 几乎共线的扇区会产生极薄的楔形。 由于浮动精度，天真的角度扫描可能会错误地丢弃它。 

另一个问题是完整的半圆。 由于角度允许正好为 180 度，因此一个扇区可以成为半磁盘。 如果两个这样的半盘重叠，它们的相交边界是弯曲的并且不能简化为多边形裁剪。 

第三个问题是具有相同中心但不同半径和角度的嵌套扇区。 如果完全覆盖，完全位于较大扇区内部的较小扇区不会产生任何影响，但部分重叠需要正确的弧积分。 

主要的概念挑战是每个扇区都可以分解为一个减去缺失的互补扇区的圆盘，但这种转换并不能直接简化并集。 

## 方法

 强力的想法是将每个扇区边界近似为密集的多边形链：用许多小线段替换每个弧，然后运行多边形联合算法，例如平面扫描或基于库的多边形裁剪。 这在原则上是正确的，因为增加分割会收敛到真实区域。 

然而，如果每个扇区近似为$k$段，我们得到$O(nk)$边缘。 即使是中等精度也需要$k \approx 1000$，生产$10^6$边缘。 该规模的多边形联合至少​​变为$O(m \log m)$和$m = 10^6$，这太慢并且占用大量内存。 更糟糕的是，精度要求迫使在小角度附近进行更精细的离散化。 

关键的见解是避免完全近似弧，而是围绕每个扇区中心径向分解问题。 扇区自然地在极坐标中定义：它是一组点$(r, \theta)$和$0 \le r \le R(\theta)$为了$\theta$在一个区间内。 这表明将每个扇区视为角度的函数：在每个角度，它都贡献一个半径间隔。 

如果我们固定一个角度$\theta$，每个扇区要么不贡献任何内容，要么贡献一个径向区间$[0, R_i]$如果$\theta$位于其角跨度内。 因此，角度联合$\theta$简直就是$[0, \max R_i(\theta)]$。 总面积成为积分$\theta$的$\frac{1}{2} R_{\max}(\theta)^2$。 

因此，问题简化为在扇区边界引起的所有角度间隔上寻找活动扇区之间的最大半径。 每个扇区在单位圆上贡献一个间隔和一个常数值（其半径）。 我们可以将其视为对角度事件的扫描，维护跟踪活动半径并支持最大查询的结构。 

我们对扇区的所有起始角度和结束角度进行排序并进行扫描$2\pi$。 在连续的事件角度之间，活动扇区组是固定的，因此最大半径是恒定的，并且该角度切片上的面积贡献很简单$\frac{1}{2} \Delta \theta \cdot R_{\max}^2$。 

我们使用带有延迟删除的多重集或堆来维护活动半径。 每个扇区以其起始角度进入并以其结束角度离开。 主要的技术精妙之处在于通过环绕正确计算角度顺序并处理精密关系。 

这将几何图形简化为具有动态最大查询的角度上的扫描线。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 多边形近似 |$O(nk \log (nk))$|$O(nk)$| 太慢而且不精确|
 | 最大半径的角扫掠 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 将每个扇形转换为极参数：计算其半径$R$作为距中心的距离$O$到任一端点$A$或者$B$，并计算角度$\alpha = \angle OA$,$\beta = \angle OB$在标准中$[0, 2\pi)$形式。 这确保每个扇区都成为明确定义的角度间隔。 
2.如果$\alpha > \beta$，将扇区解释为环绕$2\pi$并将其分成两个区间$[\alpha, 2\pi)$和$[0, \beta]$。 这避免了清扫期间的不连续性。 
3. 对于每个间隔端点，创建两个事件：起始角度处的插入事件，值为$R$，以及结束角度处的删除事件，值为$R$。 这些事件表示扇区在角度扫描中何时变得活动或不活动。 
4. 按角度对所有事件进行排序。 如果多个事件共享相同的角度，请在插入之前处理删除。 这种排序可确保当一个扇区结束而另一个扇区在同一边界处开始时，零宽度区域不会被过度计数。 
5. 扫描已排序的事件，同时维护多组活动半径。 任意时刻，该集合的最大元素表示$R_{\max}(\theta)$为当前的角度间隔。 
6. 连续事件角度之间$\theta_i$和$\theta_{i+1}$，计算角宽度$\Delta \theta = \theta_{i+1} - \theta_i$。 添加贡献$\frac{1}{2} \cdot \Delta \theta \cdot (R_{\max})^2$到答案。 
7.根据所有事件更新多重集$\theta_{i+1}$，根据需要插入或删除半径，然后继续扫描。 
8. 输出累计面积。 

### 为什么它有效

 任意固定角度$\theta$，每个扇区要么贡献从原点到其半径的整个径向段，要么根本不贡献。 因此该角度的并集始终是单个区间$[0, R_{\max}(\theta)]$。 这消除了跟踪扇区之间成对重叠的任何需要，因为所有重叠都折叠成径向空间中的最大操作。 扫描将圆划分为活动扇区集不改变的间隔，确保$R_{\max}$在每个区间内是恒定的，因此对角度进行积分可以精确地重建总面积，而无需近似。 

## Python 解决方案```python
import sys
import math
input = sys.stdin.readline

def angle(x, y):
    a = math.atan2(y, x)
    if a < 0:
        a += 2 * math.pi
    return a

n = int(input())
events = []

for _ in range(n):
    xo, yo, xa, ya, xb, yb = map(int, input().split())
    
    ra = math.hypot(xa - xo, ya - yo)
    rb = math.hypot(xb - xo, yb - yo)
    r = ra  # guaranteed equal

    a1 = angle(xa - xo, ya - yo)
    a2 = angle(xb - xo, yb - yo)

    # ensure CCW from a1 to a2, may wrap
    if a1 <= a2:
        events.append((a1, 1, r))
        events.append((a2, -1, r))
    else:
        events.append((a1, 1, r))
        events.append((2 * math.pi, -1, r))
        events.append((0.0, 1, r))
        events.append((a2, -1, r))

events.sort()

import bisect
active = []

def add(x):
    bisect.insort(active, x)

def remove(x):
    i = bisect.bisect_left(active, x)
    active.pop(i)

ans = 0.0
prev = 0.0

def current_max():
    return active[-1] if active else 0.0

for ang, typ, r in events:
    if ang != prev:
        if active:
            rmax = current_max()
            ans += 0.5 * (ang - prev) * rmax * rmax
        prev = ang

    if typ == 1:
        add(r)
    else:
        remove(r)

# no need to close circle because endpoints already cover [0, 2pi)
print("{:.10f}".format(ans))
```执行直接跟随扫描。 每个扇区被分解为角度事件，并且环绕扇区被分割为$2\pi$以保持连续性。 排序列表维护活动半径，允许随时提取最大值。 

最微妙的部分是事件排序。 该代码确保在更新每个边界角度的活动集之前计算面积，因此每个角度间隔都使用正确的配置。 使用排序列表可以进行删除$O(n)$，这是可以接受的$n \le 1000$，尽管在生产解决方案中，具有延迟删除功能的堆会更好。

 ## 边缘情况

 关键的边缘情况是多个扇区以相同角度开始或结束时。 该算法在为下一个间隔应用贡献之前一致地处理边界处的所有事件。 这可以防止共享端点处的重复计算。 

另一种情况是交叉的环绕扇区$2\pi$。 通过将它们分成两个间​​隔，扫描永远不会出现不连续的跳跃，并且跨边界的并集保持正确$2\pi$和$0$。 

最后，当只有一个扇区保持活动状态时，该算法将正确地简化为标准扇区面积公式。 扫描自然地处理这个问题，因为多重集恰好包含一个半径，并且积分变成单个连续的弧段。
