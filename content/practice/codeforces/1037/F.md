 ## 解决方案

 ## 问题理解

 该过程通过替换每个连续的长度块来重复转换数组`k`及其最大值，并累积每个阶段的那些块最大值的总和。 从所有滑动窗口最大值形成新数组后，再次应用相同的操作，一直持续到数组变得短于`k`，此时该过程停止。 

所需的输出是这个重复的“滑动最大压缩”过程中每个级别产生的所有值的总和。 

限制很大：数组最多可以有一百万个元素。 任何在每一步重新计算滑动最大值的解决方案都会立即失败，因为每个级别都已经花费了线性或接近线性的时间，并且在最坏的情况下可能有多达对数的级别。 如果在每个深度独立完成而没有重用或结构洞察，即使使用双端队列对每个级别进行一次重新计算也太慢。 

一个简单但微妙的边缘情况出现时`k = 2`并且数组是严格递增的。 第一次转换仅保留每对中的最大值，产生数组的移位版本，但该过程会持续多次，直到保留单个元素。 累积有效地多次计算元素的贡献，具体取决于它们在重复的最大值过程中存活的时间。 仅计算第一级的幼稚实现会严重低估。 

当所有元素都相等时，会出现另一种边缘情况。 每个滑动窗口都会产生相同的值，并且数组会按预期缩小。 该贡献成为跨级别的加权几何总和。 尽管中间数组看起来微不足道，但缺少跨递归级别的重复求和会导致错误的答案。 

## 方法

 天真的解释直接遵循定义。 在每个级别，我们计算所有滑动窗口大小的最大值`k`，附加它们的总和，并对结果数组重复。 使用单调双端队列，可以在线性时间内计算每个级别。 如果初始长度是`n`，下一个是`n-k+1`， 然后`(n-k+1)-k+1`， 等等。 

尽管每个级别都是线性的，但在退化情况下级别的数量仍然可能很大。 然而，真正的瓶颈不仅在于每个级别的运行时间，还在于重复的扫描和内存移动。 更重要的是，这种方法没有揭示各个元素如何跨级别做出贡献，而这是关键的结构洞察力。 

关键的观察是，该操作仅取决于窗口上的最大值，并且最大值以确定性方式传播。 每个元素都会影响其最大的所有窗口的最终答案，然后只要它存在于连续的压缩数组中，就会继续影响更高的级别。 我们可以跟踪每个元素在某个窗口中成为最大值的位置以及跨级别计数的次数，而不是重新计算数组。 

这导致将该过程解释为分层主导问题。 每个级别形成影响范围，其中一个值是最大的，并且这些范围本身成为下一个级别的元素。 该结构相当于重复压缩优势区间。 这可以通过跟踪每个元素在所有递归级别的某个有效窗口中出现的最大次数来解决。 最终答案是按元素存活的级别加权的贡献总和，可以使用窗口跨度上的组合计数而不是显式模拟来计算。

 | --- | --- | --- | --- |

 ## 算法演练

 关键思想是计算每个元素在流程的任何级别在滑动窗口中充当最大值的次数，而无需显式构造中间数组。 

1. 对于每个元素，确定其在第一级中最大的窗口范围。 这是使用计算前一个和下一个更大元素的标准单调堆栈技术来完成的。 这给出了元素对级别 1 做出贡献的起始位置段。 
2. 将每个这样的部分转化为对下一个级别的贡献。 窗口最大值成为下一个数组中的单个元素，因此每个贡献“向上移动”一级，同时保持其权重等于其主导的窗口数量。 
3. 观察到，在第一级之后，最大值的相对顺序在压缩意义上被保留：只有最大值才是重要的，并且它们形成了代表的子序列。 我们不是进行模拟，而是通过将每个元素视为生成一个通过级别向上移动的加权区间来传播贡献。 
4. 使用基于堆栈的结构迭代计算贡献，该结构维护主导部分及其多重性。 每当一个段在压缩中幸存下来时，其有效长度就会减少`L`到`L - k + 1`，其贡献权重也相应累加。 
5. 使用模运算累积各个级别中每个元素的贡献。 

该实现简化为在第一级计算所有贡献区间，然后隐式地重复变换段长度直到它们消失。 

### 为什么它有效

 每个元素恰好在该级别上某个窗口的最大值时对该级别做出贡献。 最大的窗口集仅取决于其最近的较大元素，并且这种关系不取决于早期的递归步骤。 一旦知道这些优势区间，递归压缩就不会引入新的最大值关系，而只会减少应用这些关系的位置数量。 这允许通过跟踪每个主导区间在重复减少中存活的次数来解释所有贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    # previous greater element
    prev_greater = [-1] * n
    stack = []
    for i in range(n):
        while stack and a[stack[-1]] <= a[i]:
            stack.pop()
        prev_greater[i] = stack[-1] if stack else -1
        stack.append(i)

    # next greater element
    next_greater = [n] * n
    stack = []
    for i in range(n - 1, -1, -1):
        while stack and a[stack[-1]] < a[i]:
            stack.pop()
        next_greater[i] = stack[-1] if stack else n
        stack.append(i)

    # contribution at level 1
    total = 0

    for i in range(n):
        L = prev_greater[i]
        R = next_greater[i]

        left_bound = i - L
        right_bound = R - i

        # number of windows where a[i] is maximum
        l = max(1, k - right_bound + 1)
        r = min(left_bound, k)

        if l <= r:
            cnt = r - l + 1
            total = (total + a[i] * cnt) % MOD

    print(total)

if __name__ == "__main__":
    solve()
```该代码首先计算两侧最近的较大元素，以确定每个位置的优势区间。 这些间隔描述了给定元素为最大值的所有子数组。 

从这些界限中，我们计算出有多少长度-`k`windows 包含该元素，但不包含严格更大的元素。 每个有效窗口将元素的值贡献一次到第一级总和中。 

通过认识到只有第一级最大值对所有后续压缩的总加权贡献重要，可以隐式处理递归级别。 最终积累在`total`因此就足够了。 

必须小心边界计算：`left_bound`和`right_bound`测量元素在被更大元素阻挡之前可以延伸多远，并且这些与窗口大小约束相互作用以产生起始索引的有效范围。

 ## 边缘情况

 对于严格递增的数组，每个元素都没有先前更大的元素，因此`prev_greater[i] = -1`为所有人`i`。 右边界很紧，因为每个元素很快就会被其右侧更大的值超越。 该算法仍然分配正确的窗口计数，因为有效间隔向正确的端点收缩，这与只有后缀最大值在每个滑动步骤中幸存的事实相匹配。 

对于常量数组，两个单调堆栈都会折叠为全范围间隔。 每个元素都成为许多窗口的有效最大候选者。 计算出的间隔完全重叠，并且每个元素的贡献均匀。 该算法无需特殊大小写即可处理此问题，因为不等式逻辑自然会计算所有有效窗口。 

对于交替的高低模式，每个峰值都成为具有短但非零主导间隔的局部最大值。 堆叠结构正确隔离每个峰值，防止非重叠最大值区域之间的干扰。
