 ## 解决方案

 ## 问题理解

 我们被赋予了一个动态的友谊系统$n$人们。 最初它们之间没有边缘。 每天早上，都会在两个先前未连接的顶点之间添加一条新的无向边。 每次添加之后，我们需要考虑当前的图并回答有关晚上旅行可以选择的最大可能顶点组的问题。 

有效的行程组是具有局部密度约束的顶点子集：对于每个选定的顶点，要么将其从组中排除，要么必须至少具有$k$它的图邻居也包含在该组中。 目标是在每次边缘插入后最大化此类子集的大小。 

关键对象不是连通性，而是导出子图内的度数。 我们正在有效地寻找最小度至少为的最大归纳子图$k$，通常被描述为$k$-图的核心。 

限制条件$n, m \le 2 \cdot 10^5$排除每天从头开始重新计算任何内容。 一个简单的重新计算$k$-每条边之后的核心需要反复修剪低度顶点，成本高昂$O(n + m)$在最坏的情况下每个查询，导致$O(m(n+m))$，太大了。 

当图早期稀疏但后来变得密集时，就会出现微妙的边缘情况。 即使后来存在一个大的有效组，早期的答案也可能为零。 例如，如果$k=2$并且图形仅在第 3 天形成三角形，然后第 1 天和第 2 天必须输出 0，即使最终结构很小但并不平凡。 任何假设每个顶点包含的答案单调增长而不重新检查度数的方法都会失败。 

## 方法

 蛮力的想法很简单。 在每个新边之后，我们构建当前图并重复删除剩余集合中度数小于的所有顶点$k$。 这是标准的剥皮工艺$k$-core：我们维护一个度数小于的顶点队列$k$，删除它们，并更新邻居直到稳定。 剩余的顶点形成有效集合，其大小就是答案。 

这是正确的，因为任何违反度数约束的顶点都不能属于任何有效的解决方案，并且删除它只能减少邻居度数，可能会触发进一步的删除。 问题在于，在每个边缘之后从头开始重复此过程会多次重新计算基本相同的剥离过程。 由于每次运行都是$O(n + m)$，总复杂度变为$O(m(n+m))$，这是不可接受的$2 \cdot 10^5$。 

关键的观察结果是边缘仅被添加，而从未被删除。 这意味着度数只会随着时间的推移而增加，因此一旦顶点对$k$-core，以后永远不会失效。 然而，困难在于有效性取决于核心中存在的其他顶点，而不仅仅是原始度数。 

关键的转变是逆转时间。 我们不是向前添加边缘，而是向后处理它们，同时保持动态$k$-核。 最初，在某个时间$m$，所有边都存在。 我们计算最终的$k$-核心一次。 然后我们以相反的顺序一一删除边。 当删除边时，某些顶点可能会下降到度数以下$k$并且必须从核心中移除。 这将创建一个级联删除过程。 每个顶点最多被删除一次，并且每条边在删除过程中被处理恒定的次数，从而给出线性的总复杂度。 

为了支持快速更新，我们维护仅限于当前核心的邻接列表和当前度数。 我们还维护一个顶点队列，这些顶点在每次边缘删除后都会变得无效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每天重新计算 k-core |$O(m(n+m))$|$O(n+m)$| 太慢了 |
 | 动态k-core反向删除 |$O(n + m)$|$O(n + m)$| 已接受 |

 ## 算法演练

 我们通过反向处理边序列并跟踪每个顶点何时不再属于有效顶点来解决该问题$k$-核。 

1. 使用所有边构建完整图，并计算每个顶点的初始度数。 

这代表白天的状态$m$，所有友谊都存在的地方。 
2. 计算初始值$k$-core 使用度数小于的所有顶点的队列$k$。 

我们反复删除这些顶点并降低其仍处于活动状态的邻居的度数。 

这给出了当天的最终稳定集$m$。 
3. 维护一个布尔数组，标记每个顶点当前是否在核心中。 

还维护所有边的邻接列表。 
4. 将所有边存储在一个数组中，并以相反的顺序处理它们$m$下降到$1$。 
5. 对于每条边$(u, v)$，从当前状态中移除其效果。 

如果两个端点仍然位于核心中，则减少它们当前的度数。 
6. 每当一个顶点的度数下降到以下时$k$，将其推入删除队列。 

该顶点在当前状态下不再有效，必须被剥离。 
7. 当队列非空时，重复删除顶点：

 将它们标记为不活动，并减少其邻居仍然活动的程度。 

如果任何邻居跌倒在下面$k$，它被添加到队列中。 
8、充分处理掉边缘导致的删除级联后$i$，记录当前核心的大小作为当天的答案$i-1$。 

### 为什么它有效

 在逆向处理的任何时刻，活动顶点的集合正是$k$- 由边后缀形成的图的核心。 剥离过程保持每个活动顶点至少具有度数的不变性$k$在活动集中。 当边缘被移除时，唯一可能的违规来自于其端点处的度数减小，并且级联确保解决所有由此产生的违规。 由于每个顶点被删除一次并且永远不会重新插入，因此结构在所有步骤中保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m, k = map(int, input().split())
    
    edges = []
    adj = [[] for _ in range(n)]
    
    for i in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        edges.append((u, v))
        adj[u].append((v, i))
        adj[v].append((u, i))
    
    deg = [len(adj[i]) for i in range(n)]
    alive = [True] * n
    removed = [False] * n
    
    q = deque()
    
    for i in range(n):
        if deg[i] < k:
            q.append(i)
            removed[i] = True
            alive[i] = False
    
    while q:
        u = q.popleft()
        for v, _ in adj[u]:
            if alive[v]:
                deg[v] -= 1
                if not removed[v] and deg[v] < k:
                    removed[v] = True
                    alive[v] = False
                    q.append(v)
    
    ans = [0] * m
    cur_alive = sum(alive)
    
    for i in range(m - 1, -1, -1):
        ans[i] = cur_alive
        
        u, v = edges[i]
        if alive[u] and alive[v]:
            deg[u] -= 1
            deg[v] -= 1
            
            for x in (u, v):
                if alive[x] and deg[x] < k:
                    alive[x] = False
                    q.append(x)
        
        while q:
            u = q.popleft()
            for v, _ in adj[u]:
                if alive[v]:
                    deg[v] -= 1
                    if alive[v] and deg[v] < k:
                        alive[v] = False
                        q.append(v)
        
        cur_alive = sum(alive)
    
    print("\n".join(map(str, ans)))

if __name__ == "__main__":
    solve()
```该代码首先构建邻接表并将度数初始化为全图度数。 然后它执行初始剥离来计算最终的$k$- 对应天的核心$m$。 此步骤至关重要，因为逆向处理假设有一个有效的起始核心。 

在反向迭代期间，每条边都从活动结构中删除。 如果两个端点当前都处于活动状态，则它们的度数会降低，这可能会违反$k$-核心条件。 BFS 队列处理级联删除，确保恢复不变量。 

一个微妙的实现细节是顶点在删除后永远不会重新插入。 这`alive`数组保证单调删除，这就是线性复杂性的原因。

 ## 边缘情况

 一种边缘情况是顶点仅在级联后才勉强达到阈值。 例如，在一个三角形中$k=2$，在所有三个边都存在之前，没有顶点存在。 该算法处理这个问题是因为初始剥离会删除所有内容，而反向插入仅在两个邻居都处于活动状态时恢复顶点，确保所有三个顶点在其中任何一个变得稳定之前同时存在。 

另一种边缘情况是单个边缘移除触发大型级联。 考虑一个密集组件，其中每个顶点都有精确的度数$k$。 删除一条边会使两个端点下降到阈值以下，这可以广泛传播。 BFS 队列确保在进入下一步之前完全解决此传播，即使在最坏情况的级联删除下也能保持正确性。
