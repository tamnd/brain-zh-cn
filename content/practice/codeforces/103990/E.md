 ## 解决方案
 ## 问题理解

 我们给定一个大整数$k$，我们想要表达的值$k^2$作为两个特殊值的总和。 每个值都来自由整数索引的固定集合$x$在范围内$1 \le x \le 2125$，以及与索引相关的值$x$是$x^1$，这只是$x$本身。 

因此任务减少为选择两个不同的索引$x < y$这样$$x + y = k^2$$以及两者$x$和$y$都在允许的范围之内。 

在所有有效对中，我们必须返回最小化的那对$x + y$。 由于每个有效对都已经满足$x + y = k^2$，这个条件实际上在所有解决方案中都是恒定的。 因此，一旦满足可行性，区分解决方案的真正约束是存在，而不是优化。 

唯一的困难是限制很大$k$, 直至$4 \cdot 10^{18}$，这使得$k^2$巨大，远远超出 64 位范围。 然而，索引范围很小，最多只有 2125，因此任何有效的总和也必须位于一个非常小的区间内。 

关键的结构含义是，我们正在一个小有界域中搜索一对整数，其总和等于潜在的巨大目标。 这立即表明大多数输入都是不可能的，并且只有非常具体的输入$k$值可以产生有效的分解。 

一个幼稚的错误是尝试迭代所有对$(x, y)$并检查是否$x + y = k^2$。 这是正确的，但没有必要。 另一个错误是尝试计算$k^2$直接使用标准整数类型，这在没有大整数的语言中会溢出。 Python 避免了这种情况，但推理仍然必须依赖于边界而不是实际计算。 

一个微妙的边缘情况是不存在对的情况。 例如，如果$k^2 < 3$，该范围内没有两个不同的正整数之和可以等于它。 另一个边缘情况是当$k^2$超过$2125 + 2124 = 4249$，在这种情况下也不存在有效对，因为允许域中的最大可能总和是有界的。 

## 方法

 蛮力方法很简单。 我们迭代所有对$(x, y)$和$1 \le x < y \le 2125$，计算它们的总和，并检查它是否等于$k^2$。 由于有大约$\frac{2125^2}{2} \approx 2.2 \cdot 10^6$对，这在孤立的情况下已经是可行的。 但是，如果问题有多个查询或范围较大，则这种方法将变得低效。 它的正确性微不足道，因为它直接枚举整个解空间。 

关键的观察是我们实际上不需要搜索这两个变量。 一次$x$被选择，$y$完全确定为$k^2 - x$。 这将问题转化为一次传递$x$，检查是否隐含$y$有效且在范围内。 

这将搜索空间从二次减少到线性。 我们还利用了这样一个事实：我们希望最小化该对$x + y$，但由于所有有效对共享相同的总和，因此我们通过扫描隐式最小化$x$按升序排列并返回第一个有效对。

 ## 算法演练

 ### 关键思想

 我们固定该端点对中的一个端点并推导另一个端点。 我们只接受在允许的索引范围内的对。 

### 步骤

 1. 计算目标值$S = k^2$。 

我们不依赖于固定宽度的整数运算； 从概念上讲，这是一个任意精度值。 
2. 迭代$x$从 1 到 2125。 

我们按升序扫描，因为第一个有效对自动具有最小的可能$x$，这与最小化一致$x + y$。 
3. 对于每个$x$, 计算$y = S - x$。 

这是唯一可以配对的候选值$x$达到目标总和。 
4. 检查是否$y$是一个有效的索引，意思是$1 \le y \le 2125$。 

如果没有，则丢弃此$x$。 这确保两个元素都来自可用集合。 
5. 确保$x < y$。 

这避免了重复对并保证了独特性。 如果$x = y$，它将违反两个不同球体的要求。 
6. 如果找到有效的对，则输出$x$和$y$，然后终止。 

提前退出是正确的，因为从小到大扫描$x$保证最低限度$x$，因此最小$x + y$可行的解决方案之中。 
7. 如果循环后没有找到对，则输出-1。 

### 为什么它有效

 该算法依赖于每个有效解恰好对应于一次迭代的不变量，其中$x$是该对中较小的元素。 对于任何可行的分解$S = x + y$，当循环到达特定的$x$，计算出的$y$将匹配并通过范围检查。 既然我们列举了所有可能的$x$，我们不能错过有效的一对。 该排序确保第一场比赛在所需的平局规则下是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

k = int(input().strip())
S = k * k

LIMIT = 2125

for x in range(1, LIMIT + 1):
    y = S - x
    if 1 <= y <= LIMIT and x < y:
        print(x, y)
        break
else:
    print(-1)
```代码直接实现了重写求和条件导出的一维搜索。 循环结构很重要：`for ... else`图案确保我们只打印`-1`如果没有找到有效的分解。 

一个微妙的实现点是使用 Python 的任意精度整数$k^2$。 在较低级语言中，这需要仔细处理以避免溢出，但在这里我们可以安全地直接计算它。 

条件`x < y`强制唯一性并防止重复的对称解决方案。 由于我们按升序扫描$x$，第一个有效对自动按字典顺序最小，因此在平局打破规则下是最佳的。

 ## 边缘情况

 ### 非常小$k$为了$k = 1$,$S = 1$。 循环检查$x = 1$, 给予$y = 0$，这是无效的。 不存在其他候选者，因此输出为-1。 当总和超出可行区间时，该算法会正确拒绝不可能的分解。 

＃＃＃ 大的$k$为了$k = 4 \cdot 10^{18}$,$S$是一个天文数字。 对于每一个$x \in [1, 2125]$,$y = S - x$远远超出了允许的范围。 该算法拒绝所有候选者并输出-1。 这表明该方法不依赖于$k$，仅在诱导和上。 

### 假设有效案例

 如果存在一对$(x, y)$这样$x + y = k^2$，那么当循环达到$x$，计算出的$y$完全匹配并超越界限。 该算法在此时立即终止，通过解决方案的建设性恢复来确认正确性。
