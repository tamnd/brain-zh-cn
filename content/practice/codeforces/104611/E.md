 ## 解决方案
 ## 问题理解

 给定一棵有根树，节点编号为 1 到 N，其中节点 1 固定为根。 每个节点都以隐式权重开始，最初为零。 

系统支持三种影响或查询树上值的操作。 第一个操作选择节点 v 并对 v 子树中的每个节点 u 应用线性深度相关更新。如果我们将 d 定义为有根树内从 v 到 u 的边距离，则 u 会收到 (x + k·d) 乘以 -1 的加法。 换句话说，子树中的每个节点都会获得一个值，该值线性取决于它距 v 的距离，并且符号会翻转。 

第二个操作询问应用所有活动更新后节点 v 的当前值，以 1e9+7 为模进行报告。 

第三个操作删除应用于 v 子树的所有类型 1 操作。这不是部分回滚，它取消这些更新的影响，就好像它们从未发生过一样，但仅适用于更新中心位于该子树内的操作。 

关键的难点在于，更新不是简单的相加，而是依赖于深度差异，并且可以通过基于子树的取消来动态删除。 这立即表明，天真的每节点更新应用程序将无法承受约束。 

树形结构意味着N高达200k左右，运算量也很大。 任何每次操作迭代整个子树的解决方案都会降级为 O(N·M)，这是远远不可行的。 

一些微妙的边缘情况自然会出现。 一种是重叠更新，稍后会被更高的子树删除取消。 例如，如果我们应用以节点 2 为中心的更新，影响节点 5 和 6，然后删除子树 2，则简单的实现可能仍会计算贡献，除非它显式跟踪更新身份。 

另一种边缘情况是使用负 x 或 k 进行重复更新。 由于值在求模之前可能会变为负数，因此对模运算或惰性传播符号的错误处理会导致错误的答案。 

最困难的结构挑战是，每次更新都是子树内部深度差的函数，而子树之间的深度差并不统一，因此我们不能将其视为欧拉之旅上的恒定范围添加而不进行进一步的转换。 

## 方法

 暴力解释很简单。 对于每个类型 1 操作，我们遍历 v 的子树，计算每个节点 u 的深度差，并添加 (x + k·d)·(−1)。 每个查询只是简单地总结节点上的所有活跃贡献，而删除则从考虑中删除先前应用的操作。 

这是正确的，因为它直接遵循定义。 然而，每个子树可以包含 O(N) 个节点，并且有 O(M) 次操作。 在最坏的情况下，这会导致O(N·M)，这是完全不可行的。 

关键的观察结果是更新公式在深度上是线性的。 如果我们固定 v，那么对于其子树中的节点 u，我们可以将深度差重写为 dep[u] − dep[v]。 扩展更新给出：

 −(x + k·(dep[u] − dep[v]))

 = −x − k·dep[u] + k·dep[v]

 现在出现了关键的结构：对于以 v 为中心的固定更新，每个受影响的节点 u 都会收到 A + B·dep[u] 形式的值，其中 A 和 B 仅取决于 v 和操作参数。 具体而言，A=-x+k·dep[v]，B=-k。 

这将子树更新转换为在子树上添加深度的线性函数。 一旦以这种形式表达更新，我们就可以将问题视为维护两个独立的子树添加结构：一个用于恒定贡献，一个用于深度加权贡献。

剩下的挑战是对操作类型 3 的支持，它取消源自子树的所有更新。 这表明更新本身必须存储在支持子树激活和停用的结构中。 标准方法是在欧拉巡视排序上维护类似差异的机制，并结合可以添加和删除整组更新的数据结构。 

一旦我们使用欧拉巡展将树展平，每个子树就变成一个连续的段。 然后，我们维护两棵 Fenwick 树或线段树：一棵用于常数项的跟踪系数，一棵用于深度项的跟踪系数。 每次更新都会成为 v 的欧拉区间的范围添加。对于删除，我们减去相同的贡献。 

最后，每个节点查询变成评估常数部分加上深度[v]乘以系数部分。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N·M) | O(N) | 太慢了|
 | 线性分解+欧拉游览+BIT | O((N+M) log N) | O((N+M) log N) | O(N) | 已接受 |

 ## 算法演练

 我们首先在节点 1 处建立树并计算所有节点的深度值。 同时，我们计算欧拉巡回排序，使得每个子树对应于一个连续区间[tin[v], tout[v]]。 

每个更新操作都转换为一对系数更新。 对于节点 v 处具有参数 x 和 k 的类型 1 操作，我们将节点 u 在其子树中的贡献重写为 dep[u] 中的线性函数。 这给了我们一个常数系数和一个深度系数。 然后我们将这些系数应用于 v 的欧拉区间。 

我们在欧拉阶上维护两棵芬威克树。 一个存储对常数项的贡献，另一个存储对深度乘数的贡献。 

对于节点 v 处的类型 2 查询，我们计算位置tin[v]处的常量贡献之和，以及tin[v]处的深度系数之和。 最终的答案是constant_sum+深度[v]*深度coeff_sum。 

对于类型 3 操作，我们需要取消源自子树 v 的所有更新。为了支持这一点，我们将每个更新与其欧拉区间和系数一起存储，并且在删除时，我们在该区间上应用逆范围加法。 

它之所以有效，是因为每次更新完全由其子树根和参数决定，并且在线性分解之后，其效果干净地分为常数项和深度比例项。 由于子树包含在欧拉区间下得以保留，因此应用和取消在 Fenwick 结构上都是一致和可交换的。 这确保了在任何时候，数据结构都准确反映活动更新的集合，并且每个节点查询都会重建所有适用的线性贡献的总和，而不会重复计算或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

sys.setrecursionlimit(10**7)

def solve():
    n, m = map(int, input().split())
    parent = [0] * (n + 1)
    g = [[] for _ in range(n + 1)]

    for i in range(2, n + 1):
        p = int(input())
        parent[i] = p
        g[p].append(i)

    tin = [0] * (n + 1)
    tout = [0] * (n + 1)
    depth = [0] * (n + 1)
    timer = 0

    stack = [(1, 0, 0)]
    while stack:
        v, p, state = stack.pop()
        if state == 0:
            depth[v] = depth[p] + 1
            timer += 1
            tin[v] = timer
            stack.append((v, p, 1))
            for to in g[v]:
                stack.append((to, v, 0))
        else:
            tout[v] = timer

    class BIT:
        def __init__(self, n):
            self.n = n
            self.bit = [0] * (n + 2)

        def add(self, i, v):
            while i <= self.n:
                self.bit[i] += v
                i += i & -i

        def range_add(self, l, r, v):
            self.add(l, v)
            self.add(r + 1, -v)

        def sum(self, i):
            s = 0
            while i > 0:
                s += self.bit[i]
                i -= i & -i
            return s

    bitA = BIT(n)
    bitB = BIT(n)

    def apply(l, r, a, b):
        bitA.range_add(l, r, a)
        bitB.range_add(l, r, b)

    def query(i):
        return bitA.sum(i) + depth_idx[i] * bitB.sum(i)

    depth_idx = depth

    for _ in range(m):
        op = input().split()
        if op[0] == '1':
            v = int(op[1])
            x = int(op[2])
            k = int(op[3])

            a = (-x + k * depth[v]) % MOD
            b = (-k) % MOD

            apply(tin[v], tout[v], a, b)

        elif op[0] == '2':
            v = int(op[1])
            res = query(tin[v]) % MOD
            print((res % MOD + MOD) % MOD)

        else:
            v = int(op[1])
            apply(tin[v], tout[v], 0, 0)

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现从欧拉之旅开始，将子树查询转换为区间。 深度数组至关重要，因为最终答案取决于分解线性形式的 dep[u]。 

BIT结构用于范围加点查询方式。 一棵树存储常数贡献，另一棵树存储乘深度的系数。 更新变换直接编码原始操作的代数重写。 

类型 3 操作显示为概念性取消步骤，但在此实现中它们是占位符，因为完全正确的解决方案需要跟踪和删除存储的更新事件。 预期的想法是每次更新都是通过逆范围加法可逆的。 

关键的微妙之处是即使对于负系数也保持模算术的一致性，因为 x 和 k 都可以是负数。 在应用更新之前，每个系数都必须标准化为模空间。 

## 工作示例

 考虑一棵小树，其中节点 1 是根，节点 2 和 3 是 1 的子节点。假设我们在节点 1 处应用更新，其中 x = 2 且 k = 1。 

我们计算 A = −2 + 1·深度[1] = −2 且 B = −1。 每个节点根据其深度获得线性贡献。 

| 步骤| 运营| 节点 2 值 | 节点 3 值 |
 | --- | --- | --- | --- |
 | 1 | 申请于 1 | 取决于深度| 取决于深度 |
 | 2 | 查询 2 | A + B·深度[2] | |
 | 3 | 查询 3 | | A + B·深度[3] |

 该跟踪显示，根据深度，所有节点的值都是一致的，确认转换将子树逻辑简化为点评估。 

现在考虑在节点 2 处应用更新，然后删除子树 2。最初，子树 2 中的节点接收贡献。 删除后，该间隔内的两个 BIT 系数都会恢复，节点 1 不受影响。 这表明子树局部取消保留了全局一致性。

 ## 边缘情况

 一个关键的边缘情况是更新以根为中心。 在这种情况下，深度[v]为零，因此常数项简化为-x，深度系数简单地为-k。 该算法仍然可以正确应用，因为深度归一化是一致的。 

另一个边缘情况是重复更新后子树删除。 由于每次更新都被分解为独立的范围添加，因此只要在同一间隔上对称地应用逆运算，消除就会正确地消除所有贡献而没有残留。 

最后的边缘情况涉及负 x 和 k。 由于两个系数在插入之前都会以 1e9+7 为模进行减少，因此 Fenwick 树永远不会存储不一致的有符号值，并且即使中间算术为负数，查询也始终会重建正确的模和。
