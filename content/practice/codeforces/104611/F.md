 ## 解决方案
 ## 问题理解

 我们有两个圆形宝石序列，长度均为 n。 每个位置都有一个宝石类型，并且序列被认为是循环的，这意味着我们可以选择任何起点并绕圆以固定方向遍历它们。 

宝石类型之间存在有向的转化规则。 每条规则都规定我们可以花费一定的成本将 a 类型的宝石更改为 b 类型的宝石，但除非明确给出，否则反向操作不一定可用。 

任务是选择第一个序列的起始旋转和方向（顺时针或逆时针），然后将两个圆圈逐个位置对齐。 对于每对对齐的宝石，我们可以应用一系列变换以使两颗宝石相等，并支付这些变换的成本总和。 我们希望起始位置和方向的所有选择的总成本尽可能最小。 如果没有办法使所有对齐的对相等，我们输出−1。 

这些限制意味着宝石类型的数量很少，最多 400 种，而规则的数量却很大。 序列长度 n 足够大，每次转换尝试的任何 O(n²) 或更差的情况都必须小心处理，但超过 400 个状态的密集三次解是可行的。 该结构建议将问题分为两部分：计算所有宝石类型之间的最短转换成本，然后有效评估两个序列之间的所有循环对齐。 

当转换是间接的时，就会出现微妙的失败案例。 例如，如果仅存在a→b和b→c，则a→c是可能的，但必须通过中间步骤来发现。 贪婪或仅直接边缘的方法会低估可行性或高估成本。 

另一个边缘情况是方向处理。 如果我们只考虑第二个序列的一个方向，我们可能会错过需要反转遍历顺序的有效最佳解决方案。 

## 方法

 直接的蛮力策略将尝试第一个循环中的每个可能的起始索引和每个可能的方向。 对于每个对齐，它将使用宝石类型之间的最短路径查询来计算独立转换每个配对宝石的成本。 

这已经表明了一种分解：如果我们知道将任何类型 a 转换为任何类型 b 的最小成本，那么每个对齐成本就变成了位置的简单总和。 然后，强力检查 2n 个对齐，每个成本为 O(n)，每个转换查找为 O(1)，预处理后每个测试用例的成本为 O(n²)。 瓶颈不在于比对数量，而在于正确计算转换成本。 

关键的观察结果是宝石变换形成最多 400 个节点的有向加权图。 将一种类型转换为另一种类型的最便宜的方法是该图上的最短路径问题。 一旦计算出所有对的最短路径，任意两种宝石类型之间的转换成本就变成了直接查找。 

在此减少之后，剩下的任务是有效地评估所有循环移位。 对于固定的移位和方向，成本是预先计算的类型到类型距离的位置之和。 从结构上讲，这是序列上的循环相关问题。 由于 n 很大，从头开始重新计算每个移位会太慢，但由于所有计算都是简单的累加贡献，因此我们可以直接以 O(n²) 总数评估移位，这在 n 高达几千的典型限制下是可以接受的。 

我们还必须评估两个方向：正向对齐和反向对齐，因为该问题允许选择遍历方向。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 无需预处理的暴力破解 | O(n3) | O(1) | O(1) | 太慢了 |
 | 弗洛伊德 + 检查所有班次 | O(V3 + n2) | O(V²) | 已接受 |

 ## 算法演练

 ### 第 1 步：建立最短的转型成本

 我们将宝石类型建模为有向图中的节点。 每个具有成本 c 的规则 a → b 成为有向边。 我们使用 Floyd-Warshall 计算 400 种可能类型的所有节点对之间的最小成本。 

此步骤是必要的，因为转换可以链接中间类型，并且我们需要每对的真正最小成本。 

### 步骤 2：准备两个遍历方向

 我们分别考虑两种情况。 在前向情况下，我们保持第二个序列不变。 在相反的情况下，我们在概念上反转一个圆的遍历方向，这对应于在匹配时反转一个序列的顺序。 

### 步骤 3：枚举所有循环移位

 对于第一个序列中每个可能的起始索引 k，我们将 s[i + k] 与 t[i] (mod n) 对齐。 这代表固定一个起点并一起走两个圈。 

### 步骤 4：计算固定班次的成本

 对于给定的班次 k，我们通过对所有 i 求和 dist[s[(i+k) mod n]][t[i]] 来计算总成本。 这直接使用预先计算的最短路径表。 

### 步骤 5：跟踪所有配置的最小值

 我们对两个方向重复进行平移评估，并在所有情况下保持最小值。 如果每种配置都会产生无限成本，我们输出 -1。 

### 为什么它有效

 预处理后，宝石类型之间的每次转换都与位置无关。 这意味着剩下的唯一结构是对齐，而不是转换复杂性。 由于每个对齐都分解为独立的每个位置成本，因此总成本恰好是最佳局部变换的总和。 彻底检查所有旋转和两个方向可确保我们考虑两个圆的每个有效循环配对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

def solve():
    n, m = map(int, input().split())
    s = list(map(int, input().split()))
    t = list(map(int, input().split()))

    MAXV = 400
    dist = [[INF] * (MAXV + 1) for _ in range(MAXV + 1)]

    for i in range(1, MAXV + 1):
        dist[i][i] = 0

    for _ in range(m):
        a, b, c = map(int, input().split())
        if c < dist[a][b]:
            dist[a][b] = c

    for k in range(1, MAXV + 1):
        dk = dist[k]
        for i in range(1, MAXV + 1):
            di = dist[i]
            dik = di[k]
            if dik == INF:
                continue
            for j in range(1, MAXV + 1):
                if dk[j] == INF:
                    continue
                nd = dik + dk[j]
                if nd < di[j]:
                    di[j] = nd

    def calc(arr_t):
        best = INF
        for shift in range(n):
            total = 0
            ok = True
            for i in range(n):
                a = s[(i + shift) % n]
                b = arr_t[i]
                d = dist[a][b]
                if d == INF:
                    ok = False
                    break
                total += d
            if ok:
                best = min(best, total)
        return best

    ans1 = calc(t)
    ans2 = calc(t[::-1])

    ans = min(ans1, ans2)
    if ans >= INF:
        print(-1)
    else:
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个 400 x 400 距离表，并使用直接转换成本对其进行初始化。 Floyd-Warshall 用于传播间接变换，以便 dist[a][b] 成为将 a 转换为 b 的最小可能成本。 

函数 calc 计算第二序列的一个固定方向。 对于每个可能的旋转，它都会以 n 为模对齐索引，并使用预先计算的 dist 表累积转换成本。 如果任何位置无法转换，则该配置将被丢弃。 

最后，我们对原始第二个序列运行此过程一次，对其相反序列运行一次，取最小值。 

## 工作示例

 考虑一个 n = 4 的小例子，我们用简单的变换规则比较两个循环序列。 

令 s = [1, 2, 3, 4]，t = [1, 5, 3, 4]，并假设变换允许 2 → 5 和 5 → 3 具有一定的成本。 

### 前进方向，shift = 0

 | 我| s[i] | t[i] | t[i] | 成本|
 | ---| ---| ---| ---|
 | 0 | 1 | 1 | 0 |
 | 1 | 2 | 5 | 距离(2,5) |
 | 2 | 3 | 3 | 0 |
 | 3 | 4 | 4 | 0 |

 总成本完全通过将 2 转换为 5 来确定。 

该跟踪显示了如何只有不匹配的位置才会产生成本，以及最短路径预处理如何使每个查找立即进行。 

### 反向，shift = 1

 | 我| s[(i+1)%4] | s[(i+1)%4] t[i] | t[i] | 成本|
 | ---| ---| ---| ---|
 | 0 | 2 | 4 | 距离(2,4) |
 | 1 | 3 | 1 | 距离(3,1) |
 | 2 | 4 | 5 | 距离(4,5) |
 | 3 | 1 | 3 | 距离(1,3) |

 此配置演示了反转为何重要：最佳对齐可能仅在翻转遍历方向后出现，因为循环结构在反转下是对称的。

 ## 边缘情况

 一种重要的边缘情况是不存在直接变换但存在多步路径。 Floyd 预处理可确保此类情况仍能得到正确处理。 例如，如果仅存在 1 → 2 和 2 → 3，则对齐期间的 1 到 3 转换将正确使用两条边。 

另一种边缘情况是当一个对齐方向不可能但相反方向有效时。 该算法显式评估第二个序列的两个方向，因此它不依赖于对称性。 

当所有轮换都有效但只有一个轮换产生最佳成本时，会出现最后一种情况。 由于每次轮班都是独立枚举的，并且成本都经过完全重新计算，因此不存在因局部选择而错过全局最优轮换的风险。
