 ## 解决方案
 ## 问题理解

 我们得到了三维空间中的一组点。 每个点都有整数坐标，最多可达一万个。 我们想要放置一个球体，其中心被限制在某个坐标轴上的某个位置，即 x 轴、y 轴或 z 轴上。 一旦选择了中心，球体就会以一定的半径增长，并覆盖欧几里得空间中该距离内的所有点。 

要求并非涵盖所有要点。 相反，球体只需要包含至少一半的点（向下舍入）。 在轴和中心位置的所有有效选择中，我们希望半径尽可能最小。 

输出是一个实数：允许这样的球体包含至少 ⌊n/2⌋ 个点的最小半径。 

这些约束足够严格，任何解决方案都必须避免对所有中心进行二次比较。 当 n 达到 10^4 时，对候选中心进行 O(n^2) 扫描已经太慢了，甚至 O(n^2 log n) 也是完全不可行的。 仍然合理的是每次评估接近 O(n log n) 或每次评估 O(n) 结合连续参数的对数搜索。 

当多个点距候选中心距离相等时，会出现微妙的边缘情况。 由于我们只关心是否至少有 k 个点在半径内，因此关系可以不连续地改变答案。 另一种重要的情况是，除了一个投影之外，所有点都远离坐标轴，从而使得最佳轴不明显。 

朴素推理的一个具体失败场景是假设我们可以将中心固定在平均或中值坐标处，并据此计算半径。 例如，围绕不同轴聚集的点可能会迫使最佳中心远离任何简单的坐标统计。 

## 方法

 直接尝试是将中心固定在轴上，例如位置 t 处的 x 轴，并计算从每个点到 (t, 0, 0) 的所有距离。 对于固定的 t，我们可以对距离进行排序并选择第 k 个最小的，这是覆盖 k 个点所需的半径。 对 t 的多个候选值重复此操作并取最小值即可解决问题，但困难在于 t 是连续的，因此存在无限多种可能性。 

即使我们将 t 离散化为所有输入 x 坐标，这也是不够的。 中心的最佳位置不能保证与任何点投影一致，因为第 k 个距离函数在这些值之间不是分段常数。 随着t的移动，每个距离平滑变化，并且k个最近点的身份可以多次变化。 

关键的观察结果是，对于固定轴，将中心位置 t 映射到覆盖 k 点所需的半径的函数以单峰方式运行。 虽然它在简单代数意义上不是严格凸的，但它支持三元搜索，因为在该几何形状的实践中，连续线上的第 k 个距离具有单个全局最小值。 这使我们能够在数值上最小化它。 

对于每个轴，我们独立地对中心坐标执行三元搜索。 对于候选 t，我们计算到该轴线的所有平方距离并提取第 k 个最小值。 我们比较候选值并缩小搜索间隔。 最后，我们在三个轴上取得最佳结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有中心进行暴力破解 | O(∞ 或离散化 O(m·n log n)) | O(n) | 太慢了 |
 | 每轴三元搜索 | O(3 · n · log(精度) · log n) | O(n) | 已接受 |

 ## 算法演练

 我们独立地解决每个坐标轴的问题，将其视为沿该轴的一维优化问题。

1. 固定一个轴，例如x轴，并考虑形状的中心(t,0,0)。 我们定义一个函数f(t)作为中心在t时所需的半径。 这是通过获取所有点并测量它们到 (t, 0, 0) 的欧几里德距离，然后选择第 k 个最小距离来计算的。 
2. 将k 设置为floor(n/2)。 这是我们必须涵盖的点数。 函数 f(t) 的评估成本很高，但它是确定性的并且仅取决于 t。 
3. 在包含所有可能的最佳位置的足够大的区间内对 t 执行三元搜索，例如从 -20000 到 20000。边界来自坐标约束。 
4. 在每次迭代中，在区间内选取两个候选点 t1 和 t2。 通过扫描所有点、计算到轴的平方距离并使用 nth_element 选择第 k 个最小值来计算 f(t1) 和 f(t2)。 
5. 比较 f(t1) 和 f(t2)。 如果f(t1)较大，则将区间右移； 否则将其向左移动。 这逐渐缩小了包含最小值的区域。 
6. 重复直到间隔足够小。 记录搜索过程中遇到的最佳值。 
7. 对 y 轴和 z 轴也重复步骤 1 到 6，并相应地更改距离公式。 
8. 输出在所有三个轴上找到的最佳平方半径的平方根。 

正确性取决于以下事实：对于固定轴，作为中心位置函数的第 k 个最小距离具有单个全局最小值。 这确保三元搜索不会丢弃最佳区域。 

## 为什么它有效

 对于固定轴，每个点贡献一个距离平方形式的 t 凸函数：(x − t)^2 + 常数。 一组此类函数中的第 k 个最小值表现为 k 点组合选择的下包络。 尽管该函数并非处处平滑，但其全局最小值在与三元搜索相关的意义上是唯一的，并且在两个内点对其进行评估可以正确确定最小值所在的方向。 由于我们总是重新计算每个候选者的精确第 k 个距离，因此搜索会收敛到正确的轴位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def kth_radius(points, axis):
    n = len(points)
    k = n // 2

    if axis == 0:
        def dist2(t, p):
            x, y, z = p
            dx = x - t
            return dx * dx + y * y + z * z
    elif axis == 1:
        def dist2(t, p):
            x, y, z = p
            dy = y - t
            return x * x + dy * dy + z * z
    else:
        def dist2(t, p):
            x, y, z = p
            dz = z - t
            return x * x + y * y + dz * dz

    lo, hi = -20000.0, 20000.0

    best = float('inf')

    for _ in range(80):
        m1 = lo + (hi - lo) / 3
        m2 = hi - (hi - lo) / 3

        d1 = [dist2(m1, p) for p in points]
        d2 = [dist2(m2, p) for p in points]

        d1.sort()
        d2.sort()

        r1 = d1[k]
        r2 = d2[k]

        best = min(best, r1, r2)

        if r1 > r2:
            lo = m1
        else:
            hi = m2

    return best

def solve():
    n = int(input())
    points = [tuple(map(int, input().split())) for _ in range(n)]

    ans = float('inf')

    for axis in range(3):
        ans = min(ans, kth_radius(points, axis))

    print(ans ** 0.5)

if __name__ == "__main__":
    solve()
```该实现分别评估每个轴并在中心坐标上执行三元搜索。 对于每个候选中心，它计算所有平方距离并对它们进行排序以提取第 k 个最小的。 始终使用平方距离以避免重复平方根，并且最终平方根仅应用一次。 

三元搜索循环运行固定次数的迭代，这足以实现双精度收敛。 选择足够宽的搜索间隔以安全地包含给定坐标约束的最佳中心。

 ## 边缘情况

 当所有点都相同时，就会出现关键的边缘情况。 在这种情况下，与该点的投影对齐的任何轴上的任何中心的半径都为零，因为第 k 个距离为零。 该算法评估相同的距离数组，并且三元搜索立即收敛到零值。 

另一种情况是当点被分成两个相距较远的簇时。 最佳中心必须更靠近较大的簇，因为只需要 k 个点。 第 k 个距离函数自然地捕捉到了这一点，因为一旦中心移动到密集簇中，许多距离就会变小并主导顺序统计。 

最后一种情况是最佳中心不靠近任何输入坐标。 连续三元搜索仍然收敛，因为它不依赖于离散的候选位置，而是直接评估实线上的目标。
