 ## 问题理解

 两名参赛者正在记录他们随着时间的推移解决了多少编程问题。 他们每个人都已经解决了一些问题，然后他们继续以固定的每日速度解决问题。 任务是在固定的天数后确定谁的总计数更多，或者他们是否平局。 

具体来说，一个人从初始计数开始`a`并将其增加`x`每天为`c`天。 另一个开始于`b`并将其增加`y`每天的天数相同。 我们被要求比较最终总数：

 第一个值变为`a + x * c`，第二个就变成了`b + y * c`，然后我们输出较大的那个。 

约束非常小，所有输入都在 1 到 10 之间。这立即消除了对常数时间算术之外的性能考虑的任何需要。 甚至是在循环中重新计算总数的方法`c`days 在这里微不足道，因为最大迭代次数为 10。 

唯一微妙的问题是确保在正确累积两个线性级数之后进行比较。 一个典型的错误是比较每日费率`x`和`y`不考虑初始值，或者意外地错误地相乘，例如`(a + x) * c`而不是`a + x * c`。 

在溢出或大输入大小方面不存在隐藏的边缘情况，但从概念上讲，边界情况是两个总数相等时。 例如，如果`a = 3, x = 2, b = 5, y = 1, c = 2`，那么双方都以 7 结束，输出必定是平局。 

## 方法

 暴力解释模拟了日复一日的过程。 开始于`a`和`b`，我们反复添加`x`和`y`为了`c`迭代。 这很简单，并且准确地反映了故事。 每天之后，我们都会更新两个计数器。 后`c`天，我们比较结果。 

这是正确的，因为它直接对所描述的过程进行建模。 然而，尽管它对于这个问题已经很有效，但它并不是结构的最清晰的表示。 关键的观察结果是，两个序列都是具有恒定步长的算术级数。 我们可以使用单个乘法以封闭形式计算最终值，而不是模拟所有步骤。 

这种简化之所以有效，是因为常数值的重复加法相当于乘法。 正在做`x`额外`c`时间正好是`x * c`，类似地对于`y`。

 ## 算法演练

 我们计算两个最终结果并直接比较它们。 

1.读取五个整数`a, x, b, y, c`。 这些定义了初始分数、每日增长率以及增长发生的天数。 
2. 计算第一人的最终得分为`a + x * c`。 这代表从`a`并添加`x`每天一次`c`天。 
3. 计算第二人的最终得分：`b + y * c`，应用相同的逻辑。 
4. 比较两个最终值。 如果第一个更大，则第一个人获胜。 如果第二个更大，则第二个人获胜。 否则，它们是平等的。 
5.输出对应的字符串`"Feefa"`,`"Foofa"`， 或者`"Draw"`。 

### 为什么它有效

 每个人的进步都是线性的，每天不断增加。 之后总计`c`天数仅取决于初始值和相同增量的总和。 由于加法具有结合性和交换性，因此将每日增量分组为单个乘法不会改变结果。 因此，最终总数的比较相当于比较完整的每日模拟结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a = int(input())
x = int(input())
b = int(input())
y = int(input())
c = int(input())

first = a + x * c
second = b + y * c

if first > second:
    print("Feefa")
elif second > first:
    print("Foofa")
else:
    print("Draw")
```该代码遵循直接公式方法。 根据语句格式的要求单独读取每个输入。 每个参与者执行一次乘法，确保恒定的时间计算。 

唯一微妙的一点是确保正确的运算符优先级和分组。 写作`a + x * c`是正确的，因为乘法是在加法之前计算的，但为了清楚起见，可以添加括号而不改变含义。

 ## 边缘情况

 一个关键的边缘情况是两个参与者的总分相同。 例如，`a = 3, x = 2, b = 5, y = 1, c = 2`导致两者都以 7 结尾。该算法独立计算两个表达式并正确识别相等性，产生`"Draw"`。 

另一个微妙的情况是，一名参与者开始落后，但日利率更高。 例如，`a = 1, x = 3, b = 2, y = 1, c = 4`最终值的结果`13`和`6`。 该公式正确捕获了累积优势，因为它汇总了所有每日增量，而不是仅比较初始值。 

尽管问题规模很小，但这些案例强调解决方案必须比较完整的线性增长，而不是中间或每日快照。
