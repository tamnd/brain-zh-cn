 ## 解决方案
 ## 问题理解

 我们得到了一套帽子，每顶帽子都由四个数字描述。 这些数字定义了买方和卖方之间的结构化讨价还价过程。 买方从想要的价格开始，逐步提高价格，而卖方从初始价格开始，逐步降低价格。 两个移动都以相同步长的算术级数发生，因此它们的序列彼此接近，直到触发多个停止条件之一。 

对于每一顶帽子，这种谈判要么以最终商定的价格成功达成交易，要么如果卖方拒绝低于生产成本而买方仍低于该阈值则失败。 如果达成交易，我们还需要知道在此过程中总共说了多少次话语。 

任务是准确选择一顶可以成功购买的帽子，从而最小化最终支付的价格。 如果多顶帽子的最低价格相同，则任何一顶帽子都可以接受。 

输入大小高达 200,000 个帽子，因此任何模拟每个帽子的完整对话的解决方案都太慢。 每次协商都是一系列线性步骤，并且可以持续很多轮，因此，在最坏的情况下，简单的模拟会导致二次行为。 我们需要一种封闭形式的方法来计算每顶帽子的最终结果。 

一个关键的微妙之处在于，谈判可以以三种不同的方式结束，并且只有其中两种对应于有效的购买。 如果只检查序列是否相交，粗心的方法会错误分类卖方因成本限制而拒绝的情况。 

当买方和卖方序列重叠但卖方只接受低于成本的价格时，就会出现一个简单的失败场景。 例如，如果序列在 17 处相遇，但成本为 18，则即使存在交集，也必须拒绝交易。 

另一个棘手的情况是买家的下一个报价跳过了卖家当前的要价。 在这种情况下，买家首先停止，而不是卖家，这会改变最终价格和步骤数。 

最后，还有一种直接接受的情况，即第一个买家的报价已经超过了卖家的初始价格。 在这里，答案是立即的，并且交换的次数是最少的。 缺少此快捷方式会导致边缘模拟中的步数计数不正确。 

## 方法

 强力模拟将显式地为每个帽子生成算术级数并交替转动，直到触发一个停止条件。 每个步骤都会更新当前的买方报价和卖方需求，检查停止条件，然后继续。 这是简单且正确的，因为它完全遵循问题的规则。 

然而，每次协商的持续时间与步骤数成正比，直到序列交叉，如果参数是对抗性的，在最坏的情况下，每帽子大约 10^6。 对于多达 2 × 10^5 帽子，这远远超出了可行的限制。 

关键的观察结果是，两个序列都是具有相等步长的算术级数。 这意味着我们正在有效地寻找两个线性函数在交替采样规则下相遇的第一个点。 我们可以使用整数算术计算精确的相交行为，而不是逐步模拟。 

我们跟踪买家序列为

 B_k = d + k·s

 和卖方顺序为

 S_k = c − k·s

 通过以交替步骤比较这两个序列来控制相互作用。 当一个序列交叉或成本限制破坏可行性时，该过程结束。 由于两个序列以相同的斜率大小线性移动，因此它们的相对顺序单调变化，因此我们可以直接求解第一个交汇点。

我们将每一顶帽子简化为计算其中一个停止条件触发的最小 k，然后验证该停止状态是否对应于有效销售或拒绝。

 | 每顶帽子的算术/闭合形式 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立加工每一顶帽子。 

1. 将买方报价定义为从 d 开始并每轮增加 s 的序列。 将卖方询问定义为从 c 开始并每轮递减 s 的序列。 该过程在概念上是交替的，但两个序列都是单调算术级数，因此我们没有显式地模拟回合。 
2. 计算买方报价不再严格低于下一个卖方报价的最早点。 这对应于解决两个算术序列的重叠问题。 因为两者都以相同的步长移动，所以这减少了比较初始偏移和匝数奇偶校验的情况。 
3. 计算买方和卖方可能相交的第一个潜在会议索引 k。 这实际上是最小的 k，使得 d + k·s ≥ c − k·s。 重新排列得到 2k·s ≥ c − d，因此 k = ceil((c − d) / (2s))。 
4.确定候选人会议价格。 此时，两个序列已经收敛到相同的值，因此按照规则，由哪一方先触发接受决定交易价格。 这相当于评估该 k 处的两个序列并检查问题定义的停止条件。 
5. 根据成本检查可行性。 如果会议价格低于 m，即使序列重叠，卖方也会拒绝。 在这种情况下，结果是失败，我们丢弃这顶帽子。 
6. 否则计算最终价格和交易次数。 口语行数与终止指数成正比：每一步贡献一个买方或卖方行动，因此在典型的交叉情况下总数为 2k + 1，对立即接受情况进行特殊处理。 
7. 通过最小化最终价格来跟踪最佳有效帽子。 如果多顶帽子的价格相同，则任何指数都可以接受。 

### 为什么它有效

 买方和卖方价值均以相同的步幅但方向相反的方式线性演变。 这创建了一个单调的收敛过程，其中卖方和买方之间的差异每整轮缩小恰好 2 秒。 因此，买方出价首次达到或超过卖方要价的时间是唯一确定的，并且可以以封闭形式计算。 成本约束仅作为收敛后的阈值检查，因此不影响交集的位置，仅影响交集是否有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    best_price = None
    best_idx = -1
    best_ops = 0

    for i in range(1, n + 1):
        c, d, m, s = map(int, input().split())

        # immediate acceptance case
        if d >= c:
            price = d
            ops = 1
            if m <= price:
                if best_price is None or price < best_price:
                    best_price = price
                    best_idx = i
                    best_ops = ops
            continue

        # compute meeting step
        diff = c - d
        k = (diff + 2 * s - 1) // (2 * s)

        price_buyer = d + k * s
        price_seller = c - k * s

        price = price_buyer  # they meet or cross here

        # validity check
        if price < m:
            continue

        ops = 2 * k + 1

        if best_price is None or price < best_price:
            best_price = price
            best_idx = i
            best_ops = ops

    if best_price is None:
        print(-1)
    else:
        print(best_idx, best_price, best_ops)

if __name__ == "__main__":
    solve()
```该解决方案独立处理每个帽子并计算结果，无需模拟。 特殊情况`d >= c`处理立即接受，买方已经至少提供了卖方的起始价格。 

核心计算将线性协商减少为第一个交集索引的单个算术表达式。 上限划分确保我们在完全平等之前正确处理买方超过卖方的情况。 

成本检查`price < m`收敛后强制执行拒绝条件，匹配如果最终商定价格低于生产成本则卖方拒绝的规则。 

## 工作示例

 考虑一个有两顶帽子的小场景。

10 2 1 3
20 5 10 4
```对于第一顶帽子，买方从 2 开始，卖方从 10 开始。差值为 8，步长为 3，因此 k = ceil(8 / 6) = 2。买方价格为 2 + 2·3 = 8，卖方价格为 10 − 2·3 = 4。它们交叉，最终价格为 8，由于 m = 1，因此有效。 

对于第二顶帽子，买方从 5 开始，卖方从 20 开始。差值为 15，步长为 4，因此 k = ceil(15 / 8) = 2。买方价格为 13，卖方价格为 12，因此最终价格为 13。

 | 2 | 2 | 13 | 12 | 12 13 | 是的 |

 两者都有效，所以我们选择帽子 1。 

该跟踪显示了两次协商如何崩溃为计算单个交集索引而不是模拟交替对话。

 ## 边缘情况

 一种极端情况是买家已经至少提供了卖家的初始价格。 在这种情况下，谈判立即结束。 该算法通过检查来处理这个问题`d >= c`在进行任何算术运算之前，请确保运算次数为 1 并且不执行除法。 

当计算的会议步骤由于算术比较开始时相等而恰好为零时，会出现另一种边缘情况。 在这种情况下，k 变为零，并且两个价格都等于其初始值，正确地表示立即收敛。 

当计算的交叉口价格低于成本阈值时，就会出现第三种边缘情况。 即使发生算术收敛，算法也会在计算 k 后丢弃帽子，符合成本阻止销售的规则，无论谈判行为如何。
