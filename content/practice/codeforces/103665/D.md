 ## 解决方案
 ## 问题理解

 我们得到一行$n$石头，每块颜色为黑色或白色。 目标是确定通过重复应用允许的局部重新着色操作是否可以转换整条线，使所有宝石最终具有相同的颜色。 

操作受到高度限制并且仅依赖于邻接性。 对于内部位置，只有当两个相邻棋子都存在并且已经共享相同颜色时才可以重新着色，在这种情况下，棋子被迫采用该共享颜色。 对于端点，可以将石头重新着色以匹配其唯一的邻居。 该过程不是选择任意翻转，而是通过阵列传播现有的统一模式。 

输入大小达到$n = 100{,}000$，这会立即排除任何明确逐步应用操作的模拟。 即使每个步骤执行一次操作也可能导致一系列操作$O(n^2)$如果天真地实施，或更糟糕的行为。 该解决方案必须对配置的可达性或不变量进行推理。 

当根本无法进行任何操作时，就会出现关键边缘情况。 例如，在像 010101 这样的完全交替的字符串中，没有内部三元组具有相同的邻居，并且端点也不能更改任何有用的内容。 在这种情况下，输出立即为“否”，因为配置已冻结。 

另一个微妙的边缘情况是，存在被相反颜色包围的单个相同颜色块，例如 000111000。很容易假设这样的块总是可以扩展，但如果周围的结构阻止传播一致地到达两侧，则该过程可能会停止。 正确性条件必须捕捉某种颜色是否最终能够主导整个结构，而不是在某个点是否存在局部移动。 

## 方法

 暴力解释将直接模拟该过程。 在每一步中，我们都会扫描数组，找到所有有效位置，应用所有可能的重新着色，然后重复直到没有发生变化。 从概念上讲，这是可行的，因为每一步都会严格减少颜色不一致的数量或扩展统一的段，因此可以保证终止。 

然而，每次完整扫描都是$O(n)$，并且在最坏的情况下，每次迭代仅改变一两个位置。 这可能导致$O(n^2)$行为，这对于$n = 10^5$。 瓶颈不在于正确性，而在于无法压缩传播过程。 

关键的观察结果是，这些操作只会向外扩展现有的单色段。 他们从不创建新的颜色区域； 它们只允许统一块之间的边界向内移动。 这意味着这个过程相当于询问至少一种颜色是否具有最终可以扩展穿过所有障碍的“种子结构”。 

关键的结构见解是，当且仅当存在至少一对相邻的相等对或允许传播开始的边界位置时，该配置才可解。 如果不存在相邻的相等对，则每个内部三元组都具有交替的颜色，并且端点除了复制单个邻居而不创建级联之外不能触发任何更改。 在这种情况下，配置将永远卡住。 

因此，问题简化为一个简单的检查：字符串是否包含至少一对相等的相邻字符。 如果是，则该颜色可以逐步向外传播，直到它主导整个阵列。 如果不是，则每一步都是无效的并且配置是不可变的。

 | --- | --- | --- | --- |
 | 暴力模拟|$O(n^2)$|$O(n)$| 太慢了 |
 | 邻接观察|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将问题简化为检测是否存在两个连续的石头具有相同颜色的地方。 

1. 从左到右扫描数组，检查每对相邻位置。 这确定是否存在任何稳定的局部区域。 
2.如果在任何位置$i$，我们发现$a[i] = a[i+1]$，我们立即得出结论，答案是“是”。 原因是这样的一对形成了种子，重复应用允许的操作可以向外扩展均匀性。 
3. 如果扫描完成后不存在这样的对，则输出“No”。 这对应于完全交替的配置，其中没有操作可以创建传播的起点。 

这已经足够了，因为任何有效的转换过程都必须从一个可以实际触发更改的区域开始。 如果没有初始相邻的相等对，则不会激活任何内部规则，并且端点本身无法生成稳定的扩展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
s = input().strip()

for i in range(n - 1):
    if s[i] == s[i + 1]:
        print("Yes")
        break
else:
    print("No")
```该解决方案对字符串执行一次线性扫描。 该循环检查相邻字符，这直接对应于搜索可行的传播种子。 这`else`仅当未发生中断（即未找到有效的相邻对）时，循环才会执行。 

一个常见的实现错误是忘记答案仅取决于邻接性，而不取决于全局结构。 另一个微妙的陷阱是对 Python 中循环-else 构造的错误处理，这对于干净的终止逻辑至关重要。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n = int(input().strip())
    s = input().strip()

    for i in range(n - 1):
        if s[i] == s[i + 1]:
            return "Yes"

 ## 边缘情况

 对于像这样的单块石头`0`，不存在相邻对，因此算法立即返回“否”，与不存在任何操作来更改任何内容的事实相匹配。 

对于两颗宝石的相同配置，例如`11`，扫描在索引 0 处找到相邻对并返回“Yes”。 这反映出该系统已经是统一的并且基本满足了目标。 

对于完全交替的较长字符串，例如`0101010`，每次邻接检查都会失败。 该算法正确返回“否”，并且过程解释确认不存在启动任何传播链的触发器。 

对于完全统一的字符串，例如`000000`，第一次比较立即成功。 这说明目标已经实现，无需任何操作。
