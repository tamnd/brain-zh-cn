 ## 解决方案
 ## 问题理解

 我们得到一组相同的金条，每条金条都可以以三种有效方式之一定向，贡献的高度为`a`,`b`， 或者`c`。 所有的金条必须只使用一次，我们将它们堆叠成一个塔。 对于一个配置来说，唯一重要的是它的总高度，它是所有高度的贡献之和。`n`酒吧。 

不同的方向选择可能会产生相同的总高度。 任务是计算可以达到多少个不同的总高度。 

输入给出`n`、条数和三个整数`a`,`b`， 和`c`，这是每个条形根据方向可能的高度贡献。 输出是通过选择可以形成的不同总和的数量，对于每个`n`位置，三个值之一。 

限制因素很重要。 条形的数量可以多达 1200 个，这排除了任何枚举所有条形的方法`3^n`配置。 即使对于中等水平的人来说，这个空间也是天文数字般的大。`n`。 然而，价值观`a`,`b`， 和`c`很大，高达`10^9`，所以我们也不能依赖重量范围较小的有界背包。 

一个关键的观察结果是，问题中唯一的结构是可加的：每个条形独立贡献，而我们只关心总和。 

当值相等时会出现边缘情况。 如果`a = b = c`，那么每个配置都会产生相同的总高度，答案是 1。如果两个值相等，比如说`a = b != c`，那么实际上我们每个柱只有两个选择，但仍然需要正确处理不同总和的计数。 

一个天真的错误是假设配置的数量等于总和的数量，这是错误的，因为许多不同的分配会合并为相同的总数。 

## 方法

 蛮力的想法很简单：尝试所有方法来分配每个`n`条形图值之一`a`,`b`， 或者`c`，计算结果总和，并将其存储在集合中。 这是正确的，因为它显式地构造了每种可能的配置。 然而，它需要迭代`3^n`状态，即使对于`n = 30`，更不用说`n = 1200`。 

关键的结构是，最终的总和仅取决于分配给每个值的条数，而不取决于它们的位置。 如果我们让`x`,`y`， 和`z`是分配的条数`a`,`b`， 和`c`，则总高度为`x*a + y*b + z*c`， 和`x + y + z = n`。 因此，问题简化为计算线性形式在受简单单纯形约束的所有整数三元组上的不同值。 

我们不是枚举分配，而是枚举计数分布。 只有`O(n^2)`有效的三元组`(x, y, z)`因为有一次`x`和`y`是固定的，`z`已确定。 每个三元组恰好产生一个总和，因此我们可以将它们插入到一个集合中并计算不同的结果。 

这将指数问题转化为二次问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解分配| O(3^n) | O(3^n) | O(3^n) | O(3^n) | 太慢了 |
 | 枚举计数三元组 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们将观点从序列转向计数。 

1. 迭代所有可能的值`x`，贡献的条数`a`， 从`0`到`n`。 这修复了多少次`a`被使用。 
2.对于每个固定`x`, 迭代`y`，贡献的条数`b`， 从`0`到`n - x`。 其余`z`被迫成为`n - x - y`，所以我们从不明确地循环它。 
3. 对于每个三元组`(x, y, z)`，计算总高度`x*a + y*b + z*c`。 将此值插入到哈希集中。 该集合会自动删除由于不同而导致的重复项`(x, y, z)`组合产生相同的总和。 
4. 在所有迭代之后，集合的大小是不同的可达到的高度的数量。 

该枚举有效的原因是每个有效配置都唯一对应于一个三元组`(x, y, z)`，并且每个三元组对应于至少一种配置。 

### 为什么它有效

 该算法依赖于这样一个不变量：条形的任何排列都完全由每个值被选择的次数计数决定，直至排列。 由于顺序不影响总和，因此两个配置当且仅当它们共享相同的配置时才等效`(x, y, z)`。 嵌套循环将所有可能的此类三元组枚举一次，因此会考虑每个可能的总和，并且不会遗漏任何有效的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, a, b, c = map(int, input().split())
    seen = set()

    for x in range(n + 1):
        for y in range(n - x + 1):
            z = n - x - y
            seen.add(x * a + y * b + z * c)

    print(len(seen))

if __name__ == "__main__":
    solve()
```实现直接遵循三重计数枚举。 外循环修复了多少次`a`使用了，内循环修复了多少次`b`被使用。 剩余计数为`c`是隐式计算的，这避免了不必要的第三次循环。 

使用集合是必要的，因为不同的`(x, y, z)`组合可以产生相同的总和，特别是当`a`,`b`， 和`c`具有算术关系。 

## 工作示例

 考虑输入`n = 2`,`a = 1`,`b = 2`,`c = 3`。 

我们枚举所有三元组：

 | 1 | 0 | 1 | 4 |

 总和的集合是`{2, 3, 4, 5, 6}`，所以答案是`5`。 

此跟踪显示不同的计数分布可能会在相同的总和上发生冲突，例如`(x=0,y=2,z=0)`和`(x=1,y=0,z=1)`两者都生产`4`。 

现在考虑`n = 3`,`a = b = 1`,`c = 10`。

这表明了如何平等的价值观`a`和`b`创建许多折叠配置，但基于集合的枚举仍然正确捕获唯一性。

 ## 边缘情况

 例如，当所有三个值都相等时`n = 1000, a = b = c = 5`，每种配置产生相同的总数`5000`。 该算法迭代所有`(x, y)`对，但每个计算出的总和都是相同的，因此该集合仅包含一个元素。 最终输出正确`1`。 

当两个值相等时，例如`a = b = 1`和`c = 100`, 许多不同的`(x, y, z)`三元组分解为相同的总和。 例如，`(x=2,y=1,z=0)`和`(x=3,y=0,z=0)`两者的贡献不同，但仍然产生重叠的范围。 该集可确保合并重复项，并且不需要手动重复数据删除逻辑。 

当值很大时，例如`10^9`，在 Python 中不存在溢出风险，但在固定宽度语言中这需要 64 位整数。 该算法永远不会乘以超过`n * max(a,b,c)`，它安全地适合 64 位有符号整数。
