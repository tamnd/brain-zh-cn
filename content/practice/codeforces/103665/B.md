 ## 解决方案
 ## 问题理解

 我们得到的一个单词恰好属于两个外星字母表之一。 一个字母表仅使用字母`A`和`B`，而另一个仅使用数字`0`和`1`。 任务是确定输入单词属于哪个字母表，然后使用固定的逐字符映射将其转换为另一个字母表。 

该映射是完全确定性的。 每一个`A`变成`0`， 每一个`B`变成`1`，反之每一个`0`变成`A`， 每一个`1`变成`B`。 除了独立的字符替换之外没有任何结构，因此单词长度保持不变，并且每个位置都可以在没有上下文的情况下进行处理。 

输入大小限制较小，字长最多为 100 个字符。 这立即意味着，即使是最直接的线性扫描也相当快，因为​​我们最多执行 100 个字符检查和替换。 任何达到 O(n²) 的值仍然没问题，但没有理由超过 O(n)。 

边缘情况很少，但有一些微妙的情况值得明确说明。 字长可以为1，例如输入`A`，这应该成为`0`。 相似地，`1`应该成为`B`。 另一种情况是当整个字符串已经是二进制形式时，例如`0101`，必须完全转换为字母。 粗心的实现可能会在没有检查的情况下错误地假设一个字母表，但保证说输入完全属于一个字母表，因此一次扫描就足以做出决定。 

## 方法

 强力解释会将其视为一般翻译问题：首先尝试检测字母表，然后在多次扫描时重复应用替换或构建中间字符串。 人们可以想象使用重复的字符串操作或使用重复扫描和替换字符的嵌套循环来替换字符，直到没有任何更改为止。 这仍然有效，因为转换是简单且本地的，但这是不必要的开销。 

关键的观察是输入保证是同质的。 要么所有字符都来自`{A, B}`或者全部都来自`{0, 1}`。 这意味着除了在扫描时检查字符之外，我们不需要任何复杂的检测或验证逻辑。 一次传递就足够了：我们可以将每个字符直接映射到其对应字符，并在一次扫描中构建结果。 

这将问题简化为直接字符替换问题，其中每个位置在 O(1) 中独立转换，给出总 O(n) 解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（重复扫描/重建）| O(n²) | O(n) | 已接受但没有必要 |
 | 最佳单通道映射 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1.读取整数`n`，它告诉我们单词中有多少个字符。 我们实际上并不需要它来进行处理，但它可以确保输入的一致性。 
2.读取字符串`s`长度`n`。 
3. 检查字符串的第一个字符以确定翻译方向。 如果是的话`A`或者`B`，然后我们将字母表转换为二进制。 否则，必须是`0`或者`1`，这意味着我们从二进制转换为字母。 这是可行的，因为该问题保证了字符串完全一致。 
4. 创建一个空列表或缓冲区以有效地构建输出。 使用列表可以避免重复的字符串连接，这在 Python 中会变慢。 
5. 迭代字符串中的每个字符。 对于每个字符，应用映射：

 如果我们处于字母到二进制模式，请替换`A → 0`和`B → 1`。 

如果我们处于二进制到字母模式，则替换`0 → A`和`1 → B`。 

将转换后的字符附加到输出缓冲区。 
6. 将缓冲区连接成最终字符串并打印它。 

### 为什么它有效

 每个字符都独立于所有其他字符，翻译规则是两个字母表之间的双射。 这意味着每个有效的输入字符串都准确地映射到一个有效的输出字符串，没有歧义。 由于我们确定一次模式并应用固定的每个字符转换，因此该算法可以逐个位置地保持正确性。 没有任何角色依赖于上下文，因此不会出现级联错误或状态损坏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    # detect alphabet from first character
    if s[0] in "AB":
        # Dobos -> Femos
        mp = {'A': '0', 'B': '1'}
    else:
        # Femos -> Dobos
        mp = {'0': 'A', '1': 'B'}

    res = []
    for c in s:
        res.append(mp[c])

    print("".join(res))

if __name__ == "__main__":
    solve()
```实现完全遵循算法。 字典`mp`对翻译规则进行编码，因此每个字符查找的时间都是恒定的。 结果累积在列表中以避免字符串连接产生二次行为。 关于方向的决定是根据第一个字符一次做出的，由于保证整个字符串属于一个字母表，所以这是安全的。

 ## 边缘情况

 对于单字符输入，该算法仍然可以正确识别该字符的字母表并应用一次映射。 例如，输入`A`设置映射`{A → 0}`，循环运行一次产生`0`。 

用于二进制单字符输入`1`，映射为`{1 → B}`，输出是`B`。 不存在歧义，因为检测步骤仅依赖于固定字母表中的成员资格，并且这些字母表是不相交的。 

对于交替字符串，例如`ABAB`，该算法不假设结构，它只是应用每个索引的映射。 每个位置都是独立的，因此位置之间的相互作用不会影响正确性。
