 ## 解决方案
 ## 问题理解

 我们被要求计算有多少个大小相同的二进制矩阵$n \times n$满足两个全局约束，这两个约束以非常不对称的方式施加在行和列上。 

每行都有一个关于其所有条目的 AND 的条件。 如果行的 AND$i$为 1，强制整行填充 1。 如果为0，则仅表示该行未完全被1填满，因此该行中的某个位置必须至少存在一个零。 

每列都有一个关于其所有条目的 OR 的条件。 如果列的 OR$j$为 0，则整列必须全为零。 如果为 1，则该列中的某处必须至少包含一个 1。 

任务是计算有多少个矩阵同时满足两组约束。 

这些约束立即表明行和列之间存在紧密的交互，因为单个单元格同时促成一行 AND 约束和一列 OR 约束。 和$n$最多$10^5$，任何尝试显式考虑矩阵甚至迭代所有单元格的方法都是不可能的。 该解决方案必须将结构压缩为行和列聚合。 

当行需要全 1 而列需要全 0 时，就会出现关键边缘情况。 例如，如果某行的 AND 等于 1，某列的 OR 等于 0，则交集单元格将被迫同时为 1 和 0，这是不可能的。 在这种情况下，答案必须立即为零。 

当一侧强制“完全刚性”时，例如一行必须全为 1 或一列必须全为 0，就会出现另一种微妙的情况。 这些强制线可以消除矩阵中的整个自由度并破坏计数结构，因此独立处理每一行或每一列会导致不正确的过度计数。 

## 方法

 暴力方法会尝试分配每个$n^2$单元格为 0 或 1，然后验证所有行 AND 和列 OR 是否与给定字符串匹配。 这需要检查$2^{n^2}$矩阵，即使对于$n = 20$。 即使尝试生成逐行配置也会给出$2^n$每行的选择，仍然远远超出限制。 

关键的观察结果是，约束不是每个单元格的局部约束，而是每个行和列的全局约束，并且每个约束都会迫使整个行或列进入极端状态。 AND 等于 1 的行是完全固定的。 OR 等于 0 的列也完全固定。 一旦识别出这些强制结构，剩余的自由度就会减少到在较小的子网格上计算“无全行”和“无全零列”类型约束下的二进制矩阵。 

蛮力之所以有效，是因为它探索了每一项任务，但失败了，因为它没有利用大多数行或列变得僵化的事实。 行和列分为强制组和灵活组的观察结果允许问题简化为在结构化约束上包含排除的组合计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^{n^2})$|$O(n^2)$| 太慢了|
 | 最佳 |$O(n)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 我们首先将约束转化为行和列的结构规则。 

### 步骤 1：将行和列约束转换为强制模式

 如果一行$i$有$a_i = 1$，该行中的每个条目都必须为 1。如果一列$j$有$b_j = 0$，该列中的每个条目都必须为 0。这些是绝对约束。 

这立即创建了一个矛盾条件：如果存在任何行$a_i = 1$和任何列$b_j = 0$，那么交叉单元将被迫同时为 1 和 0，从而使构造变得不可能。 

### 第 2 步：提前拒绝

 如果存在这样的冲突对，则返回 0。 

### 步骤3：按是否存在强制全1行进行分割

 让$k$是索引的数量$a_i = 1$。 

我们对待两种根本不同的制度取决于是否$k$为零或正数。 

### 步骤 4：至少一行完全固定为 1 的情况

 如果$k \ge 1$，那么每一列自动包含至少一个来自这些全一行的 1。 这意味着只要不存在我们已经排除的强制置零列，所有列 OR 约束都已满足。 

现在唯一剩下的自由就是与$a_i = 0$。 每个这样的行必须​​避免成为全行，但在其他方面不受限制。 

这些行中的每一行都可以是除全一字符串之外的任何二进制字符串，给出$2^n - 1$每行的选择。 由于行是独立的，因此贡献为：$$(2^n - 1)^{\#\{i : a_i = 0\}}$$### 步骤 5：所有行都是零与行的情况

 如果$k = 0$, every row must contain at least one zero. 没有行被强制为全1。 

此时，只有列 OR 约束很重要，并且列具有$b_j = 0$完全固定为零。 我们忽略它们并专注于具有以下内容的列$b_j = 1$，说有$m$这样的专栏。 

现在我们计算大小矩阵$n \times m$这样：

 没有行全为 1，也没有列全为 0。 

这是经典的对称包含-排除结构。 我们考虑强制行的子集变为全 1 或列的子集变为全 0。 任何同时包含强制行和强制零列的配置都是不可能的，因为它们的交集需要矛盾。 

因此，包含-排除干净地分成两个独立的系列：行约束和列约束。 

每个家庭贡献：$$(2^m - 1)^n$$由于基于行和基于列的包含都贡献相同的表达式，因此我们得到：$$2(2^m - 1)^n - 2^{nm}$$### 第 6 步：合并案例

 在验证一致性后，我们根据是否存在强制单行返回适当的公式。 

### 为什么它有效

 不变的是，每个约束都简化为固定整行、固定整列或在独立行或列上强制执行“并非全部相等”条件。 一旦强制结构被分离，剩余的选择就变得独立于每行，并且对称“全行”和“全零列”事件的包含-排除因素干净地没有交互项。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = input().strip()
        b = input().strip()

        has_a1 = any(ch == '1' for ch in a)
        has_b0 = any(ch == '0' for ch in b)

        if has_a1 and has_b0:
            print(0)
            continue

        pow2n = modpow(2, n)
        base = (pow2n - 1) % MOD

        if has_a1:
            k0 = a.count('0')
            print(pow(base, k0, MOD))
        else:
            m = b.count('1')
            term = pow((modpow(2, m) - 1) % MOD, n, MOD)
            full = modpow(2, n * m % (MOD - 1))
            ans = (2 * term - full) % MOD
            print(ans)

if __name__ == "__main__":
    solve()
```该代码首先检查强制为 1 的行和强制为 0 的列之间的矛盾。 然后它分为两种结构体系：至少有一个完全一排，或者没有完全一排。 在第一种机制中，每个剩余行作为任何非全一二进制字符串独立贡献$n$列。 在第二种情况下，矩阵简化为$n \times m$行和列“非简并”约束都是对称的系统，导致直接包含-排除公式$2^m$。 

一个微妙的实现细节是在模运算下仔细处理 2 的幂。 表达式$2^{nm}$必须使用模幂和指数减少来计算，因为直接乘以$n \cdot m$可以溢出标准整数限制，并且还必须遵守费马约简。 

## 工作示例

 例如，考虑一个小情况，其中没有行被强制为全 1，也没有列被强制为全 0$n = 2$,$a = 00$,$b = 11$。 这里两列都处于活动状态，所以$m = 2$。 公式变为：$$2(2^2 - 1)^2 - 2^{4} = 2 \cdot 3^2 - 16 = 18 - 16 = 2$$| 步骤| 活跃专栏 | 中间的$(2^m - 1)^n$| 最终价值|
 | --- | --- | --- | --- |
 | 计算 m | 2 | - | - |
 | 计算项 | - | 9 | - |
 | 计算满| - | - | 16 | 16
 | 结果 | - | - | 2 |

 这显示了过度计数的仅行约束和仅列约束之间的包含排除取消。 

现在考虑一种至少有一行的情况，例如$a = 10$,$b = 11$,$n = 2$。 这里第 1 行固定为所有 1。 每列自动包含一个 1，因此只有第 2 行提供自由度。 第 2 行必须避免全为 1，给出 3 个选择，所以答案是 3。 

| 步骤| 固定行| 自由行| 每个自由行的选择 |
 | --- | --- | --- | --- |
 | 识别结构 | 第 1 行固定 | 第 2 排免费 | 3 |
 | 计算结果| - | - | 3 |

 这证实了一旦存在单个完整行，列约束就会消失。

 ## 边缘情况

 当一行被强制为全 1，而一列被强制为全 0 时，矛盾会立即出现在它们的交叉单元格处。 例如，$a = 1$,$b = 0$,$n = 1$导致单个单元格必须同时为 1 和 0，从而产生强制零答案。 该算法通过检查中是否有任何 1 来直接检测到这一点$a$和任意 0$b$。 

当所有行都是零与行时，系统纯粹是为了防止全为行，同时满足列或约束。 减少活动列可确保固定为零的列不会错误地促成有效配置。
