 ## 解决方案
 ## 问题理解

 我们给出了一个在一系列位置上的购物场景，其中每个位置代表路径上的一个点，并且其中一些位置是特殊的。 任务是推理旅行者在与这些特殊点交互时如何沿着这条线移动，并计算与旅程相关的最终量，通常是从起始位置移动到目的地时访问或经过所需点的最小或总成本。 

关键思想是运动沿着线性结构发生，因此每个决策仅取决于排序，而不取决于分支。 每个查询或测试都描述了位置的配置，并且我们必须针对存在许多位置或查询的潜在大输入有效地计算结果。 

这些限制意味着任何沿着线一步步重复模拟运动的解决方案都会太慢。 如果位置数量约为 10^5 或更多，则我们重复向前扫描每个决策的 O(n^2) 方法将导致大约 10^10 次操作，这远远超出了典型限制。 这迫使解决方案在每个测试用例中以线性或接近线性的时间处理数组，或者使用前缀信息或贪婪扫描等预处理。 

此类问题中的一个微妙问题是处理不存在特殊点的段。 例如，如果我们需要确保访问标记的位置，但有很长一段未标记的延伸，天真的贪婪运动可能会假设我们可以“稍后调整”，但实际上，排序约束强制固定的遍历成本。 当所有特殊点聚集在线路的一端时，就会出现另一种边缘情况，使得一个方向变得微不足道，而相反的方向则强制完全遍历。 

## 方法

 暴力解释很简单：沿着线路一步步模拟旅程。 在每个位置，我们检查它是否与购物需求相关并相应地更新当前状态。 如果我们需要在向左或向右移动之间做出决定，我们会递归或迭代地尝试两种可能性，累积成本并跟踪访问的状态。 

这是可行的，因为状态空间在概念上很简单，位置加上访问的要求。 然而，州的数量却在迅速增长。 如果有 n 个位置并且我们考虑所有可能的路径，那么最坏的情况就像在 n 个步骤中探索分支因子为 2 的决策树，这会导致指数行为。 即使是从每个位置重新启动扫描的简化模拟也会变成 O(n^2)，这对于大 n 来说已经太慢了。 

关键的观察是结构是线性的，决策仅取决于特殊点的相对顺序。 我们可以将问题压缩为相关位置之间的间隔，而不是模拟运动。 一旦我们确定了最左边和最右边的所需点，该段内的路径就是确定性的。 任何最佳路径都不会重新访问不必要的区域，因为这样做会严格增加成本，而不会提高可行性。 

这将问题简化为计算边界位置以及对连续所需点之间或端点与最近的所需区域之间的距离进行求和。 一旦相关点被提取并排序，答案就变成了对相邻差异的简单扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |
 | 间隔压缩+扫描| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1. 阅读所有职位并确定哪些职位与购物相关。 我们将它们分成一个列表，因为只有它们的顺序对移动成本很重要。 
2. 对相关职位列表进行排序。 排序是必要的，因为移动成本取决于沿线的邻接情况，而不是输入顺序。 
3. 如果没有相关位置，则返回零，因为不需要移动来满足购物约束。 
4. 计算最左边和最右边相关位置之间的距离。 这定义了必须至少覆盖一次的最小跨度。 
5. 如果问题需要在此范围之外的特定端点开始或结束，请通过添加从起点到相关线段最近端点的距离来扩展答案。 
6. 通过将连续相关位置之间的差距相加来累积总成本。 每个间隙都代表着购物环节中不可避免的遍历。 
7. 返回最终累加值作为答案。 

第 6 步背后的原因是，一旦我们承诺覆盖一条线上的所有必需点，最便宜的方法就是按排序顺序遍历它们而不回溯。 任何偏差都会带来额外的距离，但不会减少所需的覆盖范围。 

### 为什么它有效

 该算法依赖于以下不变量：在每一步中，我们都维护所需位置的连续访问段。 向外扩展此段总是会增加覆盖范围，而无需重新访问内部点。 由于该线具有全序，因此任何最佳路径都可以转换为在已排序的所需位置上的单调行走，而不会增加成本。 这种转换消除了循环，并确保成本恰好是排序点上诱导路径中边缘间隙的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # interpret non-zero (or marked) positions as required points
    req = [i for i, x in enumerate(a) if x != 0]

    if not req:
        print(0)
        return

    req.sort()

    ans = 0
    for i in range(1, len(req)):
        ans += req[i] - req[i - 1]

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先读取数组并提取对旅程重要的索引。 它不是模拟运动，而是将问题简化为计算连续重要索引之间的距离。 

即使输入位置未按升序给出，排序也可确保正确性。 相邻对上的循环直接编码了这样的想法：我们从一个所需点遍历到下一个点而无需绕道。 

忽略所有非必需位置的决定是核心优化。 这些位置不会影响成本结构，因为它们永远不会改变访问端点的顺序或必要性。 

## 工作示例

 ### 示例 1

 输入：```
n = 5
a = [0, 1, 0, 1, 0]
```我们提取所需的职位：`[1, 3]`。 
| 步骤| 请求 | 当前差距| 和 |
 | --- | --- | --- | --- |
 | 开始 | [1, 3] | - | 0 |
 | 我 = 1 | 1 → 3 | 2 | 2 |
 输出是`2`。 

这表明只有两个所需位置之间的距离很重要，中间的非必需点无关紧要。 

### 示例 2

 输入：```
n = 7
a = [1, 0, 0, 1, 0, 1, 0]
```所需职位：`[0, 3, 5]`。 
| 步骤| 请求 | 当前差距| 和 |
 | --- | --- | --- | --- |
 | 开始 | [0, 3, 5] | - | 0 |
 | 我 = 1 | 0 → 3 | 3 | 3 |
 | 我 = 2 | 3 → 5 | 2 | 5 |
 输出是`5`。 

这表明最优路径在排序的所需位置上是严格单调的。

 ## 边缘情况

 一种边缘情况是根本没有所需位置。 在这种情况下，过滤后的列表将变为空，算法必须直接返回零。 例如：

 输入：```
n = 5
a = [0, 0, 0, 0, 0]
```该算法产生一个空的`req`list 并立即返回 0，这符合不需要移动的事实。 

另一种边缘情况是只需要一个位置的情况。 例如：

 输入：```
n = 4
a = [0, 0, 1, 0]
```过滤后的列表为`[2]`。 由于没有连续的对，因此跳过循环并且答案保持为 0。这是正确的，因为访问单个点不需要在不同的所需位置之间移动。 

第三种边缘情况是需要所有位置时。 在这种情况下，算法会将所有相邻差异相加，有效地产生`n - 1`。 为了：```
a = [1, 1, 1, 1]
```计算结果`1 + 1 + 1 = 3`，相当于走完整段一次而不回溯。
