 ## 解决方案
 ## 问题理解

 我们给出了带有长度的圆形日历的一年模型`k`，其中每一天都会重复`k`步骤。 有`n`法老，每个法老都被分配一个区域，分为两个阶段：种植需要`p[i]`时间单位和收获时间`c[i]`时间单位。 两个阶段完成后，该法老的一个节日恰好在同一时间发生`p[i] + c[i]`。 

因为时间是有周期循环的`k`，对于调度来说重要的不是绝对时间，而是一年中的日期位置，即`(p[i] + c[i]) mod k`。 

我们被告知每个节日必须在第一天的某一天举行`n`某年的某天，这意味着该年的某天必须在该范围内`[0, n-1]`。 由于整年的转变只会增加数倍`k`，它不会改变余数模`k`，所以每个法老的节日完全由`t[i] = (p[i] + c[i]) mod k`。 

两个节日不得在同一日历日举行，即使它们位于不同的年份。 这消除了使用年移来分离碰撞的任何自由，因为碰撞仅取决于残差类模`k`。 

所以任务减少到检查这些是否`n`残基都是不同的并且每个都位于`[0, n-1]`。 

从约束的角度来看，`n`上升到`10^6`，因此任何解都必须是线性或近线性的。 排序很好，散列也很好，但是任何二次方都是不可能的。 关键点是，一旦问题被正确地简化为模余数，就不再需要组合搜索了。 

如果不仔细减少，就会出现一些微妙的边缘情况：

 如果两个法老生出同样的东西`(p[i] + c[i]) % k`，即使它们的总和相差很大，它们也会发生碰撞，例如`k = 5`,`t = [1, 6]`两者都映射到`1`，强行发生冲突。 

如果余数有效模`k`但超过`n-1`， 例如`n = 3`,`k = 10`，余数等于`7`，那么即使它是一年中的有效日期，它也违反了“前 n 天”的要求。 

最后，如果错误地尝试分配不同的年份以避免冲突，则会失败，因为年轮变化不会改变一年中的日期。 

## 方法

 强力解释将尝试将每个法老分配到某一年的特定日期，并明确模拟放置，检查所有可能的年轮变化中的所有碰撞。 这很快就变得不可行，因为每个选择都会与所有其他选择相互作用，导致对放置的指数或至少二次推理。 

结构简化的关键是时间是周期性的且模数固定`k`，所以每个事件都会折叠成一个不可变的残基`(p[i] + c[i]) mod k`。 一旦观察到这一点，所有自由都会消失：我们只是检查这些残基是否形成到集合中的单射映射`{0, 1, ..., n-1}`。 

这将问题简化为简单的频率检查和范围验证。

 ## 算法演练

 ### 1. 计算有效节日日

 对于每个法老，计算`t[i] = (p[i] + c[i]) % k`。 这是一年中唯一可能发生节日的一天，因为按整年移动不会改变该值。 

### 2. 验证日期范围约束

 检查是否每个`t[i]`在于`[0, n-1]`。 如果任何值大于或等于`n`，配置是不可能的，因为该节日不能放置在允许的第一个`n`天。 

### 3.检查唯一性

 插入每个`t[i]`到一个哈希集中。 如果已经存在任何值，则两个节日共享同一日历日，从而使时间表无效。 

### 4.返回结果

 如果这两个条件对于所有法老都成立，则输出`"S"`，否则输出`"N"`。 

### 为什么它有效

 每个法老的节日时间都是固定的模数`k`，因此多年来的调度自由度不会影响一年内的订购。 唯一重要的限制是这些固定残基是否可以注入到可用的`n`插槽无冲突。 该算法直接验证允许域的单射性，因此任何失败都完全对应于禁止的重叠或超出范围的分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    p = list(map(int, input().split()))
    c = list(map(int, input().split()))
    
    seen = set()
    
    for i in range(n):
        t = (p[i] + c[i]) % k
        if t >= n:
            print("N")
            return
        if t in seen:
            print("N")
            return
        seen.add(t)
    
    print("S")

if __name__ == "__main__":
    solve()
```代码完全遵循归约。 计算`(p[i] + c[i]) % k`是规范的时间位置。 这`t >= n`检查强制执行“前 n 天”限制。 该集合强制规定没有两个法老共享同一天。 

一个常见的实施陷阱是忘记必须独立检查两个约束：如果值超出允许的日期范围，则仅唯一性是不够的。

 ## 边缘情况

 一种重要的边缘情况是所有值都不同模`k`但有一个超出了允许的范围。 例如，如果`n = 3`,`k = 10`，计算出的残差是`7`，仍然无效，因为前三天内无法放置。 该算法立即通过以下方式拒绝这一点：`t >= n`任何唯一性推理问题之前的条件。 

另一个边缘情况是模数缩减下的完全碰撞。 例如，如果多个法老的意见不同`(p[i] + c[i])`但共享相同的余数模`k`，无论对跨年安排的任何解释如何，它们都不可避免地发生冲突。 基于集合的检查在迭代期间直接捕获这一点。 

最后一个微妙的情况是`k < n`由于限制是不可能的（`n ≤ k`），但即便如此，残差仍然可能发生冲突，因为模约简会将较大的整数空间折叠成较小的循环域。
