 ## 解决方案
 ## 问题理解

 我们得到了多组小写字母。 这些字母最初被排列成具有两个属性的字符串。 首先，该字符串是一个回文，因此它的左半部分通过对称性决定了它的右半部分。 其次，在所有可以用这些字母组成的回文中，原始的回文在字典顺序上是最小的。 

打乱后，我们只看到了一袋字母，看不到它们的顺序。 任务是重建满足这两个条件的唯一回文。 

输出不仅仅是使用相同字符的任何回文，而是所有有效回文中按字典顺序排列的最小回文。 

长度限制达到 100000。任何尝试枚举或回溯排列的解决方案都会立即被排除，因为阶乘增长是不可能的。 即使排序加上线性构造也是可以接受的，因为 O(n log n) 排序和 O(n) 重建都在时间上很合适。 

一个微妙的边缘情况来自这样一个事实：保证存在一个回文排列。 这意味着字符计数已经满足必要的回文可行性条件：如果 n 为奇数，则最多有一个字符具有奇数频率；如果 n 为偶数，则最多没有一个字符。 尝试“修复”无效发行版的幼稚方法是不必要的，并且在这种保证下是错误的。 

如果试图从两端贪婪地构建回文而不首先进行排序，则会出现另一种失败模式。 例如，如果字母按输入顺序而不是字典顺序使用，则生成的回文可能是有效的，但不是最小的。 

## 方法

 强力解决方案将生成字符串的所有排列，过滤那些回文，并选择字典顺序最小的。 这在概念上是有效的，因为它直接遵循定义，但需要检查 n! 排列。 即使检查每个排列的成本也为 O(n)，使得总复杂度为 O(n · n!)，即使 n 约为 20，这也远远超出了可行的范围。 

关键的观察是回文完全由其左半部分和可能的中心字符决定。 我们不需要排列位置，只需决定每个字符中有多少个位于左侧。 一旦左半部分被固定，右半部分就会被对称地强制。 

为了实现字典顺序最小的顺序，左半部分本身必须是所有有效多重集中字典顺序最小的。 这减少了按排序顺序分配字符的问题，始终将尽可能多的较小字母放入较早的位置，同时尊重剩余计数仍可以形成回文的要求。 

这将问题转化为确定性构造：对字符进行排序，半贪婪地构建，如果需要则分配中间字符，然后进行镜像。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n·n!) | O(n·n!) | O(n) | 太慢了 |
 | 最佳| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1.统计每个字符出现的频率。 这给出了可用字母的完整摘要，而不考虑顺序。 
2. 找出可能占据回文中心的字符。 这是唯一允许出现奇数频率的字符。 如果存在多个，我们通常会产生矛盾，但问题保证了有效性。 
3. 对于每个字符，通过将其频率除以 2 来计算有多少个副本到达左半部分。此步骤将问题从全字符串构造简化为半字符串构造，因为右半部分是自动确定的。 
4. 通过按字母顺序递增的迭代字符并附加计算出的计数来构建左半部分。 这确保了字典顺序上最小的结构，因为较早的字符会尽可能早地放置在最终字符串中。 
5. 通过连接左半部分、所选的中间字符（如果有）和左半部分的反面来构建完整答案。 

### 为什么它有效

 字典顺序上最小的回文完全是通过最小化最早的不同位置来确定的。 任何有效的回文在其左半部分都必须具有相同的多重集分布，因此最小化字典顺序减少为在固定计数下构造最小的可能左半部分。 按排序字符顺序进行贪婪分配可确保在每个位置都使用最小的可用字符，并且未来的决策无法追溯创建更小的前缀，因为一旦计数固定，剩余字符是独立的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    cnt = [0] * 26

    for ch in s:
        cnt[ord(ch) - 97] += 1

    mid = ""
    left_parts = []

    for i in range(26):
        if cnt[i] % 2 == 1:
            mid = chr(i + 97)
        left_parts.append((chr(i + 97), cnt[i] // 2))

    left = []
    for ch, c in left_parts:
        if c > 0:
            left.append(ch * c)

    left = "".join(left)
    right = left[::-1]

    print(left + mid + right)

if __name__ == "__main__":
    solve()
```该实现从计算频率开始，以便我们完全消除对原始顺序的依赖。 每当看到奇数计数时，中点字符就会更新； 由于有效性得到保证，因此该字符要么保持为空，要么最终成为正确的单个奇数频率字符。 

左半部分是通过从“a”到“z”迭代构建的，确保字典优先级。 每个字符恰好贡献了其出现次数的一半。 最后，反转左半部分构造右半部分，保持回文对称性。 

这里的一个常见错误是尝试根据频率将中间字符放置在更早或更晚的位置。 它的位置是固定的：只有一个字符可以占据中心，并且不影响外部结构的字典顺序。 

## 工作示例

 ### 示例 1：赛车

 输入字符串是`racecar`。 频率计数为 r:2、a:2、c:2、e:1。 唯一奇怪的字符是`e`，所以它成为中心。 

左半部分由半计数形成：a:1、c:1、r:1。 按字典顺序排序给出 a、c、r。 

| 步骤| 行动| 左半边|
 | ---| ---| ---|
 | 开始| 计算频率 | a:2 c:2 e:1 r:2 |
 | 中| 选择奇数字符 | 电子|
 | 构建 | 数到一半 | 一个，c，r |
 | 决赛| 镜子| acr + e + RCA |

 输出变为`acrerca`。 

这表明字典顺序如何主导原始排列。 

### 示例 2：aabbccdde

 计数为 a:2、b:2、c:2、d:2、e:1。 奇数字符是e，所以它又是中心。 

半计数为 a:1、b:1、c:1、d:1。 

| 步骤| 行动| 左半边|
 | ---| ---| ---|
 | 开始| 计算频率 | a:2 b:2 c:2 d:2 e:1 |
 | 中| 选奇数| 电子|
 | 构建 | 半数 | a b c d |
 | 决赛| 镜子| abcd + e + dcba |

 输出是`abcdedcba`。 

这证实了当除一个字符之外的所有字符都平衡时，解决方案简化为简单的排序镜像。

 ## 边缘情况

 一种边缘情况是单个字符串，例如`x`。 频率数组有一个奇数，所以`x`成为中间人物。 左半部分是空的，所以输出很简单`x`。 该算法自然会处理这个问题，因为不会发生半构造。 

另一种情况是所有字符都相同，例如`aaaaaa`。 这里不存在奇怪的字符，所以中间仍然是空的。 半数产生三`a`左边的字符，镜像产生`aaaaaa`。 该算法不会尝试分配中间字符，这是正确的，因为偶数长度的回文没有中心。 

一个更结构化的案例，例如`cbaabc`显示排序效果。 频率为 a:2、b:2、c:2，因此左半部分变为`abc`排序后。 即使输入已经是回文，该构造也强制执行字典顺序最小化，从而产生`abccba`，它比许多有效的重新排列要小，例如`cbaabc`。
