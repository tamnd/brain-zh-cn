 ## 解决方案
 ## 问题理解

 我们得到了放置在数轴上的鸡蛋的位置，每个鸡蛋都有一个不同的整数坐标。 我们最多允许安装`k`加热灯，并且每个灯也可以放置在同一条线上的整数坐标处。 每个鸡蛋都被认为是由其最近的灯加热，并且配置的成本是从任何鸡蛋到其最近的灯的最大距离。 

任务是选择灯的位置，以使最大距离尽可能小。 

解释这一点的一种有用方法是，每个灯“覆盖”线路的一段，并且每个鸡蛋必须距离这些覆盖区域中的至少一个足够近。 我们并不是试图最小化总距离，只是最小化所有鸡蛋中最坏的情况。 

这些限制足够大，任何试图测试所有灯放置甚至所有鸡蛋子集的解决方案都是不可能的。 高达`5 × 10^5`职位，一个`O(n^2)`方法已经超出了可行的限制许多数量级，甚至`O(nk)`在最坏的情况下太大。 

关键的结构约束是一切都在一维线上。 这消除了所有几何复杂性，并将问题转化为区间覆盖。 

一些边缘情况值得明确思考。 如果`k = n`，我们可以在每个鸡蛋上放一盏灯，给出答案`0`。 如果`k = 1`，最优灯放置在类中值最优覆盖点附近，答案就变成以单个中心覆盖所有点所需的最小半径。 如果除了一个较远的异常值外，所有鸡蛋都聚集在一起，则答案取决于该异常值与最近聚类中心的距离，而不是平均间距。 

一个天真的错误是假设将灯放置在固定位置（例如阵列的端点或中点）总是有效的。 例如，当点间隔不均匀时，这种方法就会失败`[0, 1, 2, 100]`和`k = 2`，其中最优分组是`{0,1,2}`和`{100}`，不是对称分裂。 

## 方法

 蛮力视角会尝试选择`k`灯在所有可能的整数坐标中的位置并计算所得的最大距离。 即使将灯限制在鸡蛋位置，这也会成为具有指数选择的组合选择问题，并评估每种配置成本`O(n)`。 这太慢了，因为选择灯位置的方法数量随着增加`n choose k`。 

关键的观察来自于翻转视角。 我们不直接选择灯位置，而是提出一个决策问题：是否确定最大允许距离`R`，我们最多可以放置`k`灯让每个鸡蛋都在里面`R`一些灯？ 

一次`R`是固定的，每个灯放置在位置`x`覆盖区间内的所有鸡蛋`[x - R, x + R]`。 这将问题转化为用最小数量的长度间隔覆盖所有点`2R + 1`以整数位置为中心。 在排序线上，可以贪婪地解决这个问题：从最左边的未覆盖的鸡蛋开始，在仍然覆盖它的情况下尽可能向右放置一盏灯，并跳过它覆盖的所有鸡蛋。 

这将原来的优化问题变成了单调决策问题`R`。 如果一个半径`R`有效，任何更大的半径也有效。 这种单调性允许对答案进行二分搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 二分查找+贪心 | O(n 对数范围) | O(n) | 已接受 |

 ## 算法演练

 ### 1.对鸡蛋位置进行排序

 我们首先对所有坐标进行排序，以便我们可以从左到右处理它们。 这种排序至关重要，因为覆盖决策仅取决于沿线的相对位置。 

### 2. 定义固定半径的可行性检查`R`我们测试是否可以使用最多覆盖所有鸡蛋`k`灯，每个覆盖距离`R`。 

### 3. 从最左边未覆盖的鸡蛋开始贪婪放置

 我们从左到右扫描。 当我们遇到第一个尚未覆盖的鸡蛋时，我们将灯放置在最佳位置，以最大程度地覆盖后续鸡蛋。 

### 4. 起始鸡蛋的最佳灯位置

 如果第一个未覆盖的鸡蛋位于位置`a[i]`，将灯置于`a[i] + R`是最优的。 这最大化了我们可以将覆盖范围扩大到多远，同时仍然确保起始鸡蛋在距离之内`R`。 

然后，该灯将所有鸡蛋覆盖到位置`a[i] + 2R`。 

### 5. 跳过所有覆盖的鸡蛋

 我们将指针移过覆盖范围内的所有鸡蛋并重复，直到覆盖所有鸡蛋或超过`k`灯。 

### 6.二分查找最小有效值`R`我们搜索遍`R`从`0`最多`max(a) - min(a)`。 对于每个候选人，我们都会进行可行性检查。 

### 为什么它有效

 正确性依赖于两个属性。 首先，对于固定半径，贪婪策略最大限度地减少了灯的数量，因为每个位置都覆盖了从当前未覆盖的鸡蛋开始的最大可能后缀。 其次，可行性单调`R`：增加半径绝不会增加所需灯的数量。 这保证了二分搜索收敛到最小有效半径，而不会跳过最佳点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(a, k, R):
    n = len(a)
    i = 0
    used = 0

    while i < n:
        used += 1
        if used > k:
            return False

        start = a[i]
        cover_end = start + 2 * R

        # place lamp at start + R, cover up to start + 2R
        while i < n and a[i] <= cover_end:
            i += 1

    return True

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    a.sort()

    lo, hi = 0, a[-1] - a[0]

    while lo < hi:
        mid = (lo + hi) // 2
        if can(a, k, mid):
            hi = mid
        else:
            lo = mid + 1

    print(lo)

if __name__ == "__main__":
    solve()
```该解决方案首先对鸡蛋位置进行排序，以便贪婪覆盖可以线性运行。 这`can`函数执行关键的可行性检查：它重复选择最左边未覆盖的鸡蛋，假设为其放置了最佳的灯，并跳过落在其覆盖范围内的所有鸡蛋。 然后二分搜索缩小最小半径，仍然允许使用最多覆盖所有鸡蛋`k`灯。 

一个微妙的点是使用`start + 2 * R`作为覆盖边界。 这是因为将灯放置在`start + R`，使最左边的鸡蛋恰好距离`R`并向右对称扩展覆盖范围。 

## 工作示例

 ### 示例 1

 输入：```
3 2
0 5 7
```我们对已经排序的数组进行排序。 

我们二分查找`R`。 

| 步骤| 右 | 灯数 | 覆盖间隔|
 | ---| ---| ---| ---|
 | 检查 | 0 | 3 | [0]、[5]、[7] |
 | 检查 | 1 | 2 | [0,1],[5,7] |
 | 检查 | 1 | 有效 | ≤ 2 盏灯 |

 最小有效半径为`1`。 

这显示了集群的重要性：`{5,7}`当半径允许重叠时，可由一盏灯处理。 

### 示例 2

 输入：```
5 2
-2 -1 0 1 4
```排序后的数组是`[-2, -1, 0, 1, 4]`。 

| 步骤| 右 | 使用的灯具 | 覆盖范围|
 | ---| ---| ---| ---|
 | 检查 | 1 | 3 | 失败|
 | 检查 | 2 | 2 | 成功|

 在`R = 2`，第一盏灯罩来自`-2`继续直至`0`，放置第二个灯用于覆盖剩余区域`1`和`4`。 

这证实了贪婪行为可以正确处理不均匀的间距。

 ## 边缘情况

 对于单个鸡蛋，贪婪过程立即放置一盏灯覆盖它，产生半径`0`。 该算法不会尝试不必要的额外放置。 

为了`k = n`，每个鸡蛋都被单独覆盖，因为每个未覆盖的鸡蛋都会触发一盏灯，但条件`used > k`永远不会失败，所以可行性成立`R = 0`，二分查找正确找到。 

对于相距较远的点，例如`[0, 1000000000]`和`k = 1`，算法放置一盏灯覆盖两端，计算出的半径变为有效距离的一半，由贪婪覆盖范围精确捕获。
