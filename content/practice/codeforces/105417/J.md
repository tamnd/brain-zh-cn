 ## 解决方案
 ## 问题理解

 我们在网格上有几个点，每个点代表一个鸡蛋。 农场在任何时刻的“紧凑性”都是所有无序鸡蛋对的曼哈顿距离之和。 换句话说，每对鸡蛋都贡献水平距离加上它们之间的垂直距离，并且我们将其汇总到所有对上。 

时间被分成几天。 当天$i$， 有$s_i$秒，在每一秒中，我们可以将一个鸡蛋沿四个方向之一移动一个单位，或者选择什么都不做。 每天结束后，我们都会评估当前配置的紧凑性。 目标是最小化这些每日紧凑度值的总和。 

关键的困难在于，移动的数量不是免费的，它们分布在几天内，并且成本函数取决于所有成对的曼哈顿距离，该距离在点之间高度耦合。 

限制条件$n, d \le 2 \cdot 10^5$排除任何模拟运动或每天从头开始重新计算成对距离的方法。 即使动态地维持所有成对距离也会太昂贵，因为有$O(n^2)$对。 任何有效的解决方案都必须将系统简化为聚合统计数据。 

当所有鸡蛋开始聚集或某些天鸡蛋非常大时，就会出现微妙的边缘情况$s_i$。 一个幼稚的策略可能会尝试每天独立地贪婪地“压缩”鸡蛋，但这会失败，因为成本取决于全局成对结构，而不是个体距离。 

例如，考虑两个鸡蛋$(0,0)$和$(100,0)$。 将一个鸡蛋移向另一个鸡蛋会线性降低成本，但如果移动分布在几天内，早期部分移动仍然会降低所有未来几天的成本。 忽略未来贡献的贪婪的每日优化将错误地评估这种权衡。 

另一种失败情况发生在鸡蛋沿一个轴对齐时。 水平和垂直贡献的行为独立，但许多幼稚的实现错误地将它们耦合在一起，导致错误的更新。 

## 方法

 主要障碍是目标在每天之后对紧凑性求和，而紧凑性本身取决于所有成对的曼哈顿距离。 直接跟踪成对距离是不可行的。 

第一个观察结果是曼哈顿距离分为独立的 x 和 y 分量。 总成本是 x 坐标的成对绝对差值加上 y 坐标的成对绝对差值之和。 这使我们能够独立处理 x 和 y 并在最后添加结果。 

我们现在只关注一个维度。 假设我们只考虑 x 坐标。 紧凑性成为所有对的总和$|x_i - x_j|$，可以使用排序顺序重写为前缀和的线性函数。 这减少了状态$O(n^2)$相互作用$O(n)$有序结构。 

第二个关键洞察是时间性的：每一个举动都会影响未来的所有日子。 如果我们将鸡蛋向左移动 1 个单位，则会将所有未来的紧凑性值减少其参与的对的数量。因此，每个单位的移动在剩余的时间内都会带来持续的好处。 

我们不是模拟位置，而是将每个鸡蛋的运动解释为贡献边际改进。 我们可以沿每个轴对鸡蛋进行分类，并考虑减少该轴上的散布可降低总成本多少。 最优策略总是“首先将极端向内拉”，因为减少最大差距会产生成对距离总和的最大边际收益。 

这导致了一种结构，其中每个运动单位都会减少特定顺序统计差距的贡献。 问题变成分配固定的总移动次数（所有移动的总和）$s_i$）跨天，其中每天的成本取决于剩余的“未减少”点差。 

我们使用前缀和计算初始总 x 紧致性和 y 紧致性。 然后我们模拟每天可以应用多少次“有效收缩操作”，首先贪婪地减少最大的贡献。 由于每个单位的缩减都会影响未来的日子，因此我们维持贡献的连续递减顺序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每一步的暴力模拟 |$O(d \cdot s_i \cdot n)$|$O(n)$| 太慢了 |
 | 聚合前缀和+贪婪减少贡献|$O(n \log n + d \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们独立地减少 x 和 y 坐标的问题，然后对结果求和。 

1. 对 x 坐标进行排序。 需要排序，以便可以使用前缀和重写绝对差，这将成对距离转换为相邻顺序结构的线性计算。 
2. 使用前缀和计算初始 x 紧致性。 对于每个位置，我们使用累积和将其贡献计算为到所有先前点和所有后续点的距离。 
3. 对 y 坐标执行相同的操作。 
4. 计算可用操作总数$S = \sum s_i$。 这是全天单位移动的总数。 
5. 观察到每个单位移动都可以在 x 或 y 维度上将紧凑度降低 1，但好处取决于应用的位置。 最好的用途始终是减少最大的剩余边际贡献。 
6. 维护一个贡献结构，表示每个“价差单位”对总成本的贡献程度。 每次我们应用一步，我们都会减少最大的剩余贡献。 这相当于始终以排序坐标中的当前最大间隙为目标。 
7.我们模拟应用的效果$S$减少。 每次减少都会使总紧凑度降低当前最高边际贡献，并且当我们反复减少它时，该贡献会缩小。 
8. 最终的答案是初始紧致度减去所有应用的总减少量$S$移动，取模$998244353$。 

### 为什么它有效

 关键的不变量是，在任何时刻，一维的紧凑性完全由连续排序坐标之间的间隙多重集决定。 每个单位移动可以被视为从这些间隙之一中减少一个单位，并且减少较大的间隙总是比减少较小的间隙产生更大或相同的总成对总和的减少。 由于成本函数在这些差距中是线性的，因此始终选择最大的可用边际减少量会随着时间的推移产生最佳分配。 这种贪婪的支配确保了没有重新安排动作可以在几天内产生更小的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, d = map(int, input().split())
    xs = []
    ys = []
    for _ in range(n):
        x, y = map(int, input().split())
        xs.append(x)
        ys.append(y)

    s = list(map(int, input().split()))
    S = sum(s)

    xs.sort()
    ys.sort()

    def initial_cost(arr):
        pref = 0
        res = 0
        for i, v in enumerate(arr):
            res += v * i - pref
            pref += v
        return res

    base_x = initial_cost(xs)
    base_y = initial_cost(ys)
    base = base_x + base_y

    # collect gaps between consecutive points
    gaps = []
    for arr in (xs, ys):
        for i in range(1, len(arr)):
            gaps.append(arr[i] - arr[i - 1])

    gaps.sort(reverse=True)

    # each move reduces largest remaining gap contribution by 1
    # each gap contributes linearly, so we just subtract S from largest gaps greedily
    i = 0
    reduction = 0

    for _ in range(S):
        if i >= len(gaps):
            break
        reduction += gaps[i]
        gaps[i] -= 1
        if gaps[i] == 0:
            i += 1
        else:
            j = i
            while j + 1 < len(gaps) and gaps[j] < gaps[j + 1]:
                gaps[j], gaps[j + 1] = gaps[j + 1], gaps[j]
                j += 1

    answer = (base - reduction) % MOD
    print(answer)

if __name__ == "__main__":
    solve()
```该代码首先将坐标分为 x 和 y 列表，因为曼哈顿距离干净地分成两个独立的一维问题。 然后，它使用前缀累加计算每个维度中的初始成对距离总和，这避免了任何二次配对。 

接下来，它按排序顺序构造连续间隙的列表。 这些差距编码压缩配置产生最高边际效益的地方。 

模拟循环将每个单位移动到当前最大的间隙，减少间隙并维持顺序。 这个贪婪过程模型重复应用最有利的压缩操作。 

最后，从初始紧致度中减去总减少量。 

一个微妙的实现细节是在每次递减之后维护间隙的排序顺序。 原生堆会更干净，但当前结构使用本地交换； 正确性取决于每次更新后始终恢复排序不变式。

 ## 边缘情况

 一种边缘情况是所有鸡蛋共享相同位置。 在这种情况下，所有间隙都为零，因此不可能减少。 该算法立即计算零紧致性，并且永远不会进入归约循环，从而正确地产生零。 

另一种边缘情况是只有一个鸡蛋时。 两两和为空，因此初始紧凑性为零。 即使有很多动作可用，也没有间隙可以缩小，所以答案仍然是零。 

第三种边缘情况是所有移动预算与间隙相比非常大。 贪婪缩减首先耗尽所有正间隙，然后停止，因为不再进一步缩减会提高成本。 这符合这样的事实：一旦所有坐标都相等，额外的移动就不能进一步减少成对距离。
