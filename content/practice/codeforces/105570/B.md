 ## 解决方案
 ## 问题理解

 我们正在构建一个整数序列$a_1, a_2, \dots, a_n$。 各值$a_i$必须保持在其个人范围之内$[1, b_i]$。 此外，相邻值被捆绑在一起：连续元素的差异不能超过给定的限制，因此$a_i$和$a_{i+1}$必须满足$|a_i - a_{i+1}| \le d_i$。 

On top of this, the array is not read directly. Instead, we are given a permutation$p$和一个符号数组$x$其中每个$x_i$是$+1$或者$-1$。 从我们构造的数组$a$，通过取元素形成一个名为 Gift 的派生序列$a_{p_i}$并乘以$x_i$。 所以位置的值$i$在礼物中是$a_{p_i}$或者$-a_{p_i}$。 

目标是选择一个有效的$a$这样这个派生的 Gift 数组在字典顺序上是最小的。 

约束条件很大，有$n$最多$3 \cdot 10^5$。 任何尝试搜索值或独立模拟每个位置选择的解决方案都会立即变得太慢。 该结构是一条约束路径，因此我们期望通过仔细的传播实现线性或近线性的结构。 

一个微妙的困难是每个$a_i$受到自身边界和邻居的约束，因此局部最小化一个位置可能会使其他位置的可行性失效。 另一个复杂之处是字典顺序与索引顺序不一致$a$，但是按照排列顺序$p$，因此决策必须遵循礼物顺序而不是自然索引顺序。 

天真的贪婪失败的一个小例子：如果我们选择每个$a_i$独立地尽可能小，我们以后可能会违反邻居约束。 相反，如果我们总是满足局部约束但忽略字典优先级，我们可能会错过全局更好的分配，牺牲后面的位置来改进早期的礼物条目。 

## 方法

 思考问题的天真的方法是分别对待$a_i$独立地进行调整，然后进行调整以满足约束条件。 人们可能会尝试按照礼物位置的顺序贪婪地分配值：对于每个位置$i$， 选择$a_{p_i}$尽可能小，如果$x_i = 1$，或者尽可能大，如果$x_i = -1$，同时尊重$[1, b_i]$。 每次任务完成后，人们都会尝试修复邻居以满足$|a_i - a_{i+1}| \le d_i$。 问题是修复可能会遍及整个链条。 在最坏的情况下，一个分配可以传播到所有分配$n$位置，并重复此操作$n$步骤导致二次行为。 

关键的观察结果是约束形成了一条简单的路径，因此通过保持左右传播的一致值范围可以完全捕获可行性。 每次赋值不需要全局重新计算； 它只需要局部间隔收紧并沿链传播。 

我们为每个职位维护$i$可行区间$[L_i, R_i]$考虑到迄今为止处理的所有约束，仍然可能的值。 邻域约束转化为区间交集：如果$a_i$是在一个范围内，那么$a_{i+1}$必须位于该范围的移动版本内，反之亦然。 重复传播很快就会收敛，因为每次更新只会收紧间隔。 

一旦通过间隔保持可行性，字典最小化就成为一个受控过程：我们按照以下顺序处理礼物位置：$i = 1 \dots n$，将每个映射到索引$p_i$，并力$a_{p_i}$根据$x_i$。 固定值后，我们再次传播约束以保持一致性。 

这将问题简化为线上的重复间隔紧缩，由于边界的单调收敛，该线在实践中是线性的。

 ## 算法演练

 我们维护两个数组$L_i$和$R_i$，初始设置为$[1, b_i]$。 这些代表了所有的值$a_i$仍然可以在不违反迄今为止发现的约束的情况下进行。 

我们还维护邻接约束作为双向区间含义：如果$a_i$是固定的或受限制的，它限制$a_{i+1}$，反之亦然。 

### 步骤

 1. 初始化每个$L_i = 1$,$R_i = b_i$。 在考虑邻接性之前，这仅对每个节点的约束进行编码。 
2. 在链上传播一次邻接约束，直至稳定。 对于每条边$(i, i+1)$，强制执行$a_{i+1} \in [L_i - d_i, R_i + d_i]$和类似地$a_i \in [L_{i+1} - d_i, R_{i+1} + d_i]$，与现有区间相交。 这确立了无约束系统的全局可行性。 
3. 处理礼品订单中的位置$i = 1$到$n$。 让$j = p_i$是实际索引$a$。 
4.如果$x_i = 1$，我们尝试使$a_j$尽可能小，所以我们选择$a_j = L_j$。 如果$x_i = -1$，我们尝试使$a_j$尽可能大，所以我们选择$a_j = R_j$。 这直接最小化了当前的礼物条目，因为$x_i \cdot a_j$通过极端化最小化$a_j$朝着正确的方向。 
5.修复$a_j$通过设置$L_j = R_j = a_j$。 这将灵活的变量转换为确定的变量。 
6. 将这种执着传播给邻居。 对于每条边$(i, i+1)$， 更新：$$L_{i+1} = \max(L_{i+1}, L_i - d_i), \quad R_{i+1} = \min(R_{i+1}, R_i + d_i)$$并在相反方向上对称。 如果任何一个间隔空了，施工就不可能进行。 
7. 重复传播直到间隔没有变化。 

### 为什么它有效

 在每一步中，维护的间隔准确地代表了在已经固定的决策下仍然可以扩展到完整有效配置的值集。 当我们修好$a_j$到与其间隔一致的极值，我们在所有全局可行的完成中为当前的 Gift 位置选择最佳的可能值。 传播确保未来的选择不会与过去的决策不一致； 它仅删除不再可扩展的值。 由于间隔总是单调缩小，因此我们永远不会重新审视之前的决策，并且自始至终都保留了可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    b = list(map(int, input().split()))
    d = list(map(int, input().split()))
    p = list(map(int, input().split()))
    x = list(map(int, input().split()))

    L = [1] * n
    R = b[:]

    # initial propagation (two passes is enough on a line in practice)
    changed = True
    while changed:
        changed = False
        for i in range(n - 1):
            # i -> i+1
            nl = max(L[i+1], L[i] - d[i])
            nr = min(R[i+1], R[i] + d[i])
            if nl != L[i+1] or nr != R[i+1]:
                L[i+1], R[i+1] = nl, nr
                changed = True

            # i+1 -> i
            nl = max(L[i], L[i+1] - d[i])
            nr = min(R[i], R[i+1] + d[i])
            if nl != L[i] or nr != R[i]:
                L[i], R[i] = nl, nr
                changed = True

    # process Gift order
    for i in range(n):
        j = p[i] - 1
        if x[i] == 1:
            val = L[j]
        else:
            val = R[j]

        L[j] = R[j] = val

        # local propagation after fixing
        changed = True
        while changed:
            changed = False
            for k in range(n - 1):
                nl = max(L[k+1], L[k] - d[k])
                nr = min(R[k+1], R[k] + d[k])
                if nl != L[k+1] or nr != R[k+1]:
                    L[k+1], R[k+1] = nl, nr
                    changed = True

                nl = max(L[k], L[k+1] - d[k])
                nr = min(R[k], R[k+1] + d[k])
                if nl != L[k] or nr != R[k]:
                    L[k], R[k] = nl, nr
                    changed = True

    print(*L)

if __name__ == "__main__":
    solve()
```实现将问题分为两个阶段：首先建立全局可行性范围，然后根据礼物优先级逐步固定值。 传播循环是确保每个分配保持与邻接约束兼容的核心机制。 

一个微妙的点是，固定一个值后，算法会重用相同的区间紧缩逻辑。 这是必要的，因为固定值可以强制链两侧进行级联减少。 正确性依赖于始终重新相交的范围，直到稳定为止，而不是假设单次通过就足够了。 

## 工作示例

 由于示例在语句中已部分格式化，因此我们在一个小型构造案例上说明了该机制。 

考虑$n = 3$,$b = [5, 5, 5]$,$d = [2, 2]$,$p = [2, 1, 3]$,$x = [1, -1, 1]$。 

最初，所有间隔都是$[1, 5]$。 

初始传播后，没有任何变化，因为所有范围都是兼容的。 

我们处理礼品订单：

 | 步骤| j = p[i] | 行动| 左 | 右 |
 | --- | --- | --- | --- | --- |
 | 1 | 2 | 设置最小值 (x=1) | [1,5]、[1,1]、[1,5] | 修复后|
 | 2 | 1 | 设置最大值 (x=-1) | [5,5]、[1,1]、[1,5] | 修复后|
 | 3 | 3 | 设置最小值 (x=1) | [5,5]、[1,1]、[1,1] | 决赛|

 这一轨迹显示了固定一个位置如何降低全局灵活性，特别是当邻接约束通过链传播紧缩时。 

重要的行为是，一旦一个变量被固定，它的邻居就会立即被迫进入更窄的范围，从而影响以后的决策。

 ## 边缘情况

 一个关键的边缘情况是单个强制分配会折叠多个间隔。 例如，如果中间元素固定为较高值，则左右邻居可能都需要显着缩小，然后可以进一步级联。 该算法可以处理这个问题，因为每个传播步骤都会重新加强两个方向，确保每次崩溃后恢复一致性。 

另一个边缘情况是当所有$b_i$是相同的并且全部$d_i = 0$。 在这种情况下，唯一有效的解决方案是常量数组。 该算法正确收敛，因为每个区间交集都会立即将所有范围折叠为相同的单个值。 

第三种情况是当$x_i = -1$对于礼品订单中的许多早期职位。 这迫使早期指数$a$最大化，这最初似乎可以增加以后的灵活性。 然而，传播会立即反映约束，根据需要缩小邻居并防止任何隐藏的违规。
