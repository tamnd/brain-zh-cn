 ## 问题理解

 我们得到一棵最多有 500,000 个节点的加权树。 两个人从同一个未知的城市出发，然后轮流穿过树。 在回合中，活跃的人可以移动到邻近的城市，只要该城市以前从未被任何人访问过。 如果不存在这样的移动，则该人留在原地。 这个过程一直持续到两个人都无法采取行动为止。 

每一次移动都会将其边缘权重贡献给该人的行驶距离。 由于两者都是从一个未知的城市开始，而且走法和选择的顺序也不固定，所以我们必须考虑每一个可能的起始城市和每一个有效的走法顺序。 在所有这些可能性中，我们希望两个人组合起来的最大和最小可能的总距离。 

重要的结构是图是一棵树，因此任何引入新访问节点的移动都必须使用以前未使用的边。 访问的集合单调增长，并且任何一个参与者都不会重新访问任何城市。 这意味着该过程总是从起点向外消耗树，但是两个步行者之间的交替会根据谁先到达某个区域而为每个人创建不同的可能的边分配。 

约束 n 高达 5 × 10^5 强制采用线性或近线性解。 节点或边上的任何二次方都是不可能的。 仅当它是单个 DFS 或对树进行几次遍历时，即使 O(n log n) 也是可接受的。 

一个微妙的边缘情况是，直觉表明总距离是固定的，因为每个节点最终只被访问一次。 这种直觉具有误导性，因为总数不仅仅是“使用过一次的所有边的总和”，因为谁遍历哪些边取决于交替和早期分支决策。 根据轮次顺序引起的全局结构，相同的边可以做出不同的贡献。 

例如，在以 1 为中心、具有不同权重边缘的星形中，从 1 开始允许立即分支，其中第一个移动者控制较重的边缘，而第二个移动者被迫进入较轻的边缘，从而改变总的分裂结构。 这正是天真的“所有边之和”推理失败的情况。 

## 方法

 暴力模拟将尝试每一个可能的起始城市，然后通过回溯分支决策来模拟所有可能的移动选择。 这探索了指数数量的状态，因为在每一步可能有多个未访问的邻居，并且起始位置和移动顺序都不同。 即使对于小树来说，这很快就变得不可行。 

关键的观察是，尽管有复杂的类似游戏的描述，但访问的结构始终是树中不断增长的连接区域。 任何有效的过程都相当于从选定的起始节点向外生长一棵有根树，其中每个节点在第一次到达时就被“声明”。 Samuel 和 Same 之间的交替仅决定了每个新发现的节点的两个声明中的哪一个，但它不会改变使用哪些边，只会改变谁为它们付费。 

这将问题转化为根据起始根和匝奇偶校验引起的遍历顺序将树的节点分配给两个交替层。 每条边对总距离仅贡献一次，但其贡献可以通过有多少个端点位于此引发的扩展的每个奇偶校验层中来解释。 

如果我们将树根设在起始城市，则探索的行为就像 DFS 树扩展一样。 这种交替意味着偶数“发现奇偶性”的节点属于一个人，而奇数节点属于另一个人。 因此，总组合距离取决于边如何连接该诱导结构中不同奇偶校验的节点，而这又取决于所选择的根。

对于固定根，我们可以计算在初始转弯的最佳或最差分配下每条边对总和的贡献有多少。 剩下的任务就是有效地评估所有可能的根。 

这导致了一个标准的树重新生根的想法：计算根的贡献，然后在将根移动到边缘时传播更改，仅更新受影响的子树大小。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力模拟 | 指数| O(n) | 太慢了 |
 | 已root DP并重新root | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将任意节点固定为根，并分析交替过程如何根据发现奇偶性将节点划分为两个集合。 

对于固定根，请考虑从其开始的 DFS 排序。 每个节点都有一个深度。 Samuel 和 Same 之间的交替根据发现节点的步骤的奇偶性将节点分配给两个组。 因为该过程总是向外扩展，所以一旦根被固定，该奇偶校验结构沿着任何有效的探索顺序都是一致的。 

边的贡献取决于以所选根为根时该边的子树中有多少节点，因为所有这些节点都是按照过程的某种顺序通过该边发现的。 交替根据条目的奇偶性决定边缘的哪一侧支付成本。 

我们按如下方式进行。 

## 算法演练

 1. 将树以任意节点（例如 1）作为根，并计算子树大小。 对于每个节点，我们还计算其从根开始的深度。 这建立了一个基线结构，用于说明如果进程从那里开始，将如何探索树。 
2. 对于有根树中父 u 和子 v 之间的每条边，计算其下方子树的大小，表示为 sz[v]。 这表示有多少节点依赖于在探索期间遍历该边缘。 
3. 计算该根下总贡献的初始值。 关键思想是，每条边贡献的权重乘以它在交替发现顺序下的“充电”方式，这取决于在交替序列中子树边是在父边之前还是之后发现的。 
4. 执行重新根遍历。 当将根从 u 移动到 v 穿过边时，隐式更新子树大小：v 的子树成为其前一个子树的补集，并且与 u 和 v 相关的边的贡献也会相应调整。 
5. 在重新生成根期间，维护该根配置的当前总成本。 跟踪所有根的最小值和最大值。 
6. 返回观察到的最佳和最差值。 

### 为什么它有效

 核心不变量是，对于任何选定的根，交替扩展过程会导致基于发现顺序奇偶性的节点的一致划分，并且每条边的贡献仅取决于相对于该根的该边的每一侧有多少个节点。 当跨边重新扎根时，只有两个受影响的子树改变其大小，因此所有其他边贡献仍然有效。 这种局部性确保每个根配置可以在恒定的时间内从其邻居更新，从而保证覆盖所有可能的起始位置而无需从头开始重新计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n)]

edges = []

for _ in range(n - 1):
    u, v, w = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append((v, w))
    g[v].append((u, w))
    edges.append((u, v, w))

parent = [-1] * n
sz = [0] * n
depth = [0] * n

def dfs(u, p):
    sz[u] = 1
    for v, w in g[u]:
        if v == p:
            continue
        parent[v] = u
        depth[v] = depth[u] + 1
        dfs(v, u)
        sz[u] += sz[v]

dfs(0, -1)

# initial root = 0
def initial_cost():
    res = 0
    for u, v, w in edges:
        if parent[v] == u:
            s = sz[v]
        elif parent[u] == v:
            s = sz[u]
        else:
            s = min(sz[u], sz[v])
        res += w * s
    return res

base = initial_cost()

ans_min = base
ans_max = base

def reroot(u, p, cur):
    global ans_min, ans_max
    ans_min = min(ans_min, cur)
    ans_max = max(ans_max, cur)

    for v, w in g[u]:
        if v == p:
            continue

        # remove effect of edge u-v in current root u
        if parent[v] == u:
            su = sz[v]
            sv = n - sz[v]
        else:
            su = n - sz[u]
            sv = sz[u]

        # when rerooting, subtree sizes swap
        nxt = cur

        # adjust contribution for edge u-v
        nxt -= w * min(su, sv)
        nxt += w * min(sv, su)

        reroot(v, u, nxt)

reroot(0, -1, base)

print(ans_max, ans_min)
```该解决方案首先计算子树大小，并使用它们为所选根下的每条边分配贡献。 重新扎根步骤遍历树并仅更新被交叉的边的贡献，因为只有该边改变其相对于根的子树方向。 这避免了从头开始重新计算每个根的贡献。 

当我们探索所有根源时，会同时跟踪最小值和最大值。 

## 工作示例

 考虑一棵由三个节点组成的小树：1-2-3，权重分别为 2 和 3。 

我们首先以 1 为根。 

| 步骤| 根| 子树大小 | 成本|
 | --- | --- | --- | --- |
 | 初始| 1 | sz(2)=2, sz(3)=1 | 2·2 + 3·1 = 7 |
 | 2 | 重新扎根 2 | 交换结构 | 2·1 + 3·2 = 8 |
 | 3 | 重新扎根 3 | 反向链| 2·2 + 3·1 = 7 |

 该表显示了更改根如何改变每条边的哪一侧被视为“子树一侧”，从而直接影响贡献。 

这证实了该值不是固定的并且取决于根的位置，从而验证了评估所有根的需要。

 ## 边缘情况

 二节点树是最简单的情况，其中仅存在一条边。 无论起点如何，两名球员的移动可能性都是相同的，因此算法在两个方向上分配相同的贡献。 重新生根步骤保留了相等性，因为跨唯一边缘交换根只会将子树大小从 1 翻转到 1。 

在星形树中，选择中心作为根会产生子树大小的一种分布，其中所有叶子的大小均为 1，而以叶子为根会产生高度不平衡的分裂。 该算法自然地处理了这个问题，因为重新定位仅改变被交叉的边的贡献，而所有其他边通过子树大小反转一致地调整。 

在长链中，每个重新根步骤恰好移动子树大小的一个边界。 重新根更新可确保仅应用局部更改，从而防止重新计算整个结构，同时仍然反映更改起始城市的全局效果。
