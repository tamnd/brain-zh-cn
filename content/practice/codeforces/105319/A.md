 ## 解决方案
 ## 问题理解

 我们有一组盘子，每个盘子都有重量和宽度。 从这些板中，我们可以选择一些子集，然后将所选子集分成两个不相交的组，代表杠铃的左侧和右侧。 未选择的板块将被忽略。 

配置的“平衡”仅取决于宽度。 对于选定的左右组划分，我们计算左侧宽度总和与右侧宽度总和之间的绝对差。 然后教练提出多个查询：对于每个查询间隔$[L, R]$，我们想知道所选板材的最大可能总重量，以便生成的平衡值落在该区间内。 

关键的困难在于我们没有被要求构建一个固定的分割。 每个查询都是独立的，我们必须决定采用哪些板块以及如何分割它们以满足平衡约束，同时最大化总重量。 

这些约束已经告诉我们，对所有子集进行暴力破解是不可能的。 高达$10^5$每个测试用例的板数，任何指数子集计数都会立即被排除。 即使是对板和平衡值的二次动态规划也需要小心处理，因为总宽度总和仅可达$10^5$，这表明背包式状态空间可能是可用的。 

当没有配置可以在查询范围内实现任何平衡值时，就会出现微妙的边缘情况。 在这种情况下，答案必须为零，而不是“最佳接近值”或空的选择解释。 另一个重要的情况是，当所有板的宽度相同时，平衡结构就会崩溃，并且许多查询变得要么微不足道地实现，要么不可能实现，具体取决于奇偶校验。 

## 方法

 蛮力的想法很简单：选择任何盘子子集，以所有可能的方式将其分成左右两部分，计算平衡并跟踪总重量。 这是正确的，但立即爆炸。 即使我们忽略分割并考虑将每个板分配到左侧、右侧或未使用的位置，我们也已经为每个板提供了三种选择，从而导致$3^n$配置。 这远远超出了任何计算极限。 

第一个简化是分离关注点。 总重量仅取决于选择的板，而不取决于它们的分割方式。 一旦子集被固定，我们总是包含它的全部重量，无论板向左还是向右。 分割的唯一作用是确定是否可以满足宽度差约束。 

这将问题转化为可达性问题。 我们想知道通过将符号分配给选定的宽度可以产生哪些平衡值。 每个选定的板贡献$+b_i$或者$-b_i$到余额差。 如果我们将所有选定宽度的总和表示为$S$，分配给一侧的宽度总和为$x$，则余额变为$S - 2x$。 这意味着每个可实现的平衡完全由宽度的子集和决定。 

现在是关键的观察。 由于权重始终为正且与分割无关，因此一旦选择了板的子集，我们总是更愿意将其完全包含在内。 除了能否实现平衡值之外，重量和平衡可行性之间不存在任何权衡。 因此，查询的最佳答案要么是所有权重的总和（如果该范围内至少存在一个有效的分割），要么为零。 

因此，问题简化为计算所有可实现的宽度子集总和，并从中导出所有可实现的平衡值。 这是宽度数组上的经典背包可达性问题。 

我们在可能的宽度子集和上计算布尔 DP。 对于每个可达到的总和$x$，我们可以得出一个平衡值$S - 2x$， 在哪里$S$是所有宽度的总和。 我们将所有此类平衡值标记为可实现。 最后，我们构建一个结构，允许回答询问是否存在任何可实现的平衡的查询$[L, R]$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集和分割的暴力破解 | 指数| 指数| 太慢了|
 | 宽度上的子集 DP + 派生余额 |$O(n \cdot \sum b)$|$O(\sum b)$| 已接受 |

 ## 算法演练

 ### ## 算法演练

 1. 计算所有宽度的总和$S$。 该值决定子集总和如何转换为最终余额值。 
2. 在宽度上运行背包式动态规划，我们计算数组的所有可实现的子集和$b$。 我们维护一个布尔数组`dp[x]`意味着板的某些子集完全具有选定的总宽度$x$。 过渡是标准子集和：对于每个板宽度$b_i$，我们以相反的顺序更新可达总和，因此每个板块最多使用一次。 
3.计算所有可达子集和后$x$，使用转换将它们转换为平衡值$d = S - 2x$。 每个可达子集总和产生一个可实现的平衡值。 
4. 将所有可达到的余额值存储在布尔数组中`can[d]`。 这为每个可能的余额提供了直接的可达性信息。 
5. 构建前缀结构`can`这样对于任何查询间隔$[L, R]$，我们可以快速确定该范围内是否存在至少一个可实现的平衡。 
6. 对于每个查询，使用前缀结构扫描区间端点。 如果有任何值$[L, R]$可以实现，输出所有板的重量总和。 否则输出零。 

### 为什么它有效

 每个有效的配置完全对应于选择板的子集并将其分成两侧，这相当于将符号分配给选定的宽度。 该符号分配完全由宽度的子集和捕获。 DP 枚举所有可能的子集和，因此每个可行的平衡值在派生集中仅表示一次。 由于重量与分割无关并且仅取决于选择的板，因此任何可行的平衡都允许采用该子集中的所有板，从而最大化总重量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, q = map(int, input().split())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))

        S = sum(b)
        total_weight = sum(a)

        # subset sum DP for widths
        dp = [False] * (S + 1)
        dp[0] = True

        for w in b:
            for s in range(S, w - 1, -1):
                if dp[s - w]:
                    dp[s] = True

        # reachable balance values
        can = set()
        for x in range(S + 1):
            if dp[x]:
                can.add(S - 2 * x)

        # convert to sorted list for range queries
        can = sorted(can)

        # answer queries by binary search
        import bisect
        for _ in range(q):
            L, R = map(int, input().split())
            i = bisect.bisect_left(can, L)
            if i < len(can) and can[i] <= R:
                print(total_weight)
            else:
                print(0)

if __name__ == "__main__":
    solve()
```DP 部分是宽度为标准 0/1 背包。 我们向后迭代，因此每个板块贡献一次。 之后，我们使用有符号和与子集和之间的代数关系将子集和转换为可实现的平衡值。 

查询处理依赖于对所有可实现的余额进行排序并使用二分搜索来检查是否有任何值位于查询间隔内。 

## 工作示例

 考虑一个有宽度的小箱子$[1, 2]$和重量$[5, 3]$。 宽度的子集和是$0, 1, 2, 3$。 这些产生的平衡值计算为$S - 2x = 3 - 2x$，给出 {3, 1, -1, -3}。 排序后，可达到的余额为$[-3, -1, 1, 3]$。 

| 步骤| 选定的子集总和$x$| 平衡$S - 2x$| 可达集|
 | --- | --- | --- | --- |
 | 0 | 0 | 3 | {3} |
 | 1 | 1 | 1 | {3, 1} |
 | 2 | 2 | -1 | {3, 1, -1} |
 | 3 | 3 | -3 | {3, 1, -1, -3} |

 像这样的查询$[0, 2]$在区间内找到值 1，因此答案是总重量$8$。 像这样的查询$[4, 10]$没有找到，所以答案是 0。 

该跟踪证实我们不再推理单个分配，而是推理诱导的平衡值集。

 ## 边缘情况

 关键的边缘情况是当查询间隔内没有子集总和产生任何平衡时。 例如，如果所有宽度都等于 1 并且只能达到偶数余额，则要求奇数间隔的查询将始终失败。 DP 正确地反映了这一点，因为子集和确定了结果平衡集中的奇偶校验约束，并且二分搜索`can`会发现没有交集。 

另一种情况是当查询间隔很宽的时候，比如$[1, 10^5]$。 在这种情况下，只要存在至少一个子集和，则通常至少有一个余额会落入该区间，并且答案将成为权重的完整总和。 该算法自然地处理这个问题，因为它不依赖于间隔大小，只依赖于存在。
