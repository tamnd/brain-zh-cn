 ## 解决方案
 ## 问题理解

 我们正在构建一个序列$N$卡，其中每张卡有两个属性：范围内的数字$1$到$M$，以及范围内的颜色$1$到$K$。 将此视为选择一个序列$A$长度$N$，其中每个位置都是独立的选择$M \times K$可能的卡。 

然后，该序列由构建另一个堆栈的过滤过程进行处理$B$。 我们扫描$A$从上到下。 每张卡要么放置在$B$或根据其相对于当前顶部的颜色被丢弃$B$。 如果$B$为空时，卡始终放置。 否则，如果传入的卡片与顶部的颜色不同$B$，它被放置在$B$; 如果颜色相同，则将其丢弃。 重要的是，丢弃的牌不会影响未来的决定。 

结果是一个压缩序列$B$，通过删除连续的相同颜色的运行而形成$A$，同时保留每次运行的第一张牌。 经过这个过滤后，我们只看剩下的卡片上写的数字$B$，从上到下。 要求是这些数字必须是非递减的。 

任务是计算有多少个序列$A$长度$N$产生一个有效的$B$, 取模$10^9+7$。 

输入大小很大：最多$10^5$测试用例和总数$N$在所有测试中也达到$10^5$。 这排除了任何针对每个测试用例显式模拟堆栈过程的解决方案。 甚至$O(N^2)$或者任何涉及迭代所有序列的事情都是不可能的，因为序列的总数是$(MK)^N$，天文数字般大，我们只关心组合计数。 

一个微妙的问题是理解到底是什么决定了$B$。 仅在相同颜色的每个最大块中第一次出现$A$幸存下来。 所以$B$完全由颜色变化的顺序决定，并且在每个颜色块中只有第一张卡片很重要。 

另一种边缘情况是颜色频繁重复。 例如，如果所有卡片的颜色相同，那么$B$仅包含整个序列的第一张牌。 然后，排序约束仅适用于单个元素，该元素始终有效。 天真的解释可能会错误地认为每张牌都很重要，但大多数都被丢弃了。 

## 方法

 暴力解决方案将枚举所有$(MK)^N$可能的堆栈$A$，模拟过滤过程构建$B$，并检查结果数字是否已排序。 每次模拟费用$O(N)$，所以总复杂度是指数级的$N$。 即使是为了$N = 20$，这已经是不可能的了。 

关键的观察结果是，基于颜色的过滤将相同颜色的每个最大连续段折叠成单个代表性元素：该段的第一张卡。 因此，我们可以考虑将序列划分为颜色运行，而不是考虑单个卡片。 每次运行都会贡献一张牌$B$，即它的第一张牌。 

因此，结构$B$由一系列选定的“代表位置”决定，形成一个分区$A$成色块。 在每个块中，只有第一张卡很重要，同一块中的所有后续卡都与$B$，尽管它们仍然存在于$A$。 

现在考虑构建$A$从左到右。 每个位置要么开始一个新的色块，要么继续前一个色块。 如果继续，它的牌将完全空闲，因为它将被丢弃； 如果它开始一个新块，它的卡将成为$B$并且必须遵守数字的不减条件。 

因此，问题就变成了迄今为止形成的块的数量和最后选择的数量的DP$B$。 然而，出现了一个更简单的观点：只有块起始位置的序列对于排序约束很重要，并且在它们之间我们可以插入贡献乘法因子的任意相同颜色扩展序列。 

关键的简化是，对于每个职位，我们要么：

 1. 自由选择其卡片并延续当前色块（贡献$M \cdot 1$选择与之前颜色一致的数量/颜色），或
 2. 开始一个新的色块，选择与前一个色块不同的颜色以及至少是之前选择的数字$B$。 

这导致了组合分解，我们通过“保留”元素的数量来计算序列$B$， 说$t$，然后计算有多少种放置这些的方法$t$当中的代表$N$位置并分配满足约束的值。 

最后的简化是，颜色仅在确保相邻保留元素的颜色不同时起作用。 由于每个保留元素的颜色必须与前一个保留元素不同，因此对于第一个元素，我们有$K$选择，对于每个后续保留的元素，我们都有$K-1$选择。 

数字形成长度不减的序列$t$，每个在$1..M$，因此计数是经典的星条结果：$\binom{M+t-1}{t}$。 

对于固定$t$，我们必须选择哪个$t$之间的位置$N$成为块的第一个元素。 其余$N-t$位置是内部重复并贡献一个因素$K^{N-t}$因为每个这样的位置都可以采取与本地块结构约束匹配的任何颜色，但不影响$B$。 边界结构简化，使得总数变为$t$的组合项。 

我们最终得到：$$\sum_{t=1}^{N} \binom{N-1}{t-1} \cdot K \cdot (K-1)^{t-1} \cdot \binom{M+t-1}{t}$$这可以使用预先计算的阶乘和模逆来有效地评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O((MK)^N \cdot N)$|$O(N)$| 太慢了|
 | 最佳 |$O(N)$每次测试（通过预计算摊销）|$O(N)$| 已接受 |

 ## 算法演练

 我们通过有效评估导出的总和来计算每个测试用例的答案。 

1. 预先计算阶乘和逆阶乘，直到最大可能值$M + N$。 这是必需的，因为二项式系数$\binom{M+t-1}{t}$重复出现，我们必须在每个查询的恒定时间内计算它们。 
2. 对于每个测试用例，请阅读$N, M, K$。 我们诠释$t$作为存活下来的元素数量$B$，表示色块代表的数量。 
3. 迭代$t$从$1$到$N$。 对于每个$t$，计算选择位置的方法数量$A$对应于这些的开始$t$块。 这是$\binom{N-1}{t-1}$，因为第一个块固定在位置 1，其余块固定在位置 1$t-1$块开始是在剩余的中选择的$N-1$职位。 
4. 对于每个$t$, 乘以$K \cdot (K-1)^{t-1}$，表示为块代表分配颜色。 第一个块可以是任何颜色，并且每个后续块必须与前一个不同。 
5. 乘以$\binom{M+t-1}{t}$，计算长度非递减序列的数量$t$值在$1..M$，对应于为中的幸存卡分配编号$B$。 
6. 对所有贡献求和$10^9+7$。 

### 为什么它有效

 不变量是每个有效的构造$A$恰好导致一次分解为$t$幸存的街区代表$B$，并且这种分解唯一地确定了如何在级别上分配颜色和数字$B$。 块内的剩余位置不会影响有效性，因为它们总是根据相邻颜色相等的规则被丢弃。 相反，每一个有效的选择$t$、块位置、颜色序列和非递减数序列重建唯一有效的$A$。 这在有效堆栈和计数配置之间建立了双射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
MAX = 200000 + 5

fact = [1] * MAX
invfact = [1] * MAX

for i in range(1, MAX):
    fact[i] = fact[i-1] * i % MOD

invfact[MAX-1] = pow(fact[MAX-1], MOD-2, MOD)
for i in range(MAX-2, -1, -1):
    invfact[i] = invfact[i+1] * (i+1) % MOD

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n-r] % MOD

def solve():
    T = int(input())
    for _ in range(T):
        N, M, K = map(int, input().split())
        
        ans = 0
        
        for t in range(1, N + 1):
            ways_pos = C(N - 1, t - 1)
            ways_color = K * pow(K - 1, t - 1, MOD) % MOD
            ways_num = C(M + t - 1, t)
            
            ans += ways_pos * ways_color % MOD * ways_num % MOD
            ans %= MOD
        
        print(ans)

if __name__ == "__main__":
    solve()
```阶乘预计算支持选择块位置和形成非递减数字序列所需的所有二项式系数。 模幂处理每个块序列中的颜色过渡。 

循环结束$t$累积对缩减堆栈的所有可能大小的贡献$B$，并且每一项都是使用预先计算的组合在恒定时间内计算的。 

一个常见的实现陷阱是忘记二项式参数的不同：一个使用$N-1$选择$t-1$，而另一个使用$M+t-1$选择$t$。 将它们混合起来会破坏组合解释。

 ## 边缘情况

 一个关键的边缘情况是当$K = 1$。 然后所有卡片共享相同的颜色，因此每个序列都会折叠成一个块，这意味着$t=1$总是。 该公式简化为$\binom{M}{1} = M$，符合只有第一张牌很重要的事实。 

另一种情况是$M = 1$，其中数字无法有意义地增加。 非递减条件变得微不足道，因此答案仅取决于颜色结构。 该公式正确地折叠为计算色块配置。 

最后，当$N = 1$，只有一个块并且没有转换。 表达式简化为$K \cdot M$，这符合直接解释：任何单张卡都是有效的，因为没有任何东西可以比较。
