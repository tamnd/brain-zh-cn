 ## 问题理解

 我们得到一个整数数组，并且允许我们使用两个操作重复修改单个元素。 一个运算将所选元素乘以任何整数因子，另一个运算以一种特殊的方式将所选元素除以它的除数，通过折叠因子有效地减少它。 

目标是使所有数组元素相等，同时使用尽可能少的操作。 每个操作只触及一个位置，因此我们本质上是试图使用乘法调整将每个值“重塑”为一个共同的目标值。 

关键约束是所有值相对于元素数量而言都较小，因为每个值最多为 n，并且测试用例中 n 的总和很大。 这强烈表明按值因子推理或除数预处理是有意的，因为任何尝试显式模拟每个操作的转换的解决方案都会太慢。 

当数组已经均匀时，会出现微妙的边缘情况。 例如，如果输入是 [5, 5, 5]，则答案为零。 幼稚的方法可能仍然尝试通过中间步骤标准化所有内容，如果实现没有明确处理“已经相等”的条件，这将错误地添加操作。 

另一个棘手的情况是当值共享丰富的除数结构时。 例如，在 [2, 4, 8] 中，人们很容易认为每个元素必须使用任意乘法独立转换为 8，但最佳策略取决于如何在元素之间转移因子，而不是固定的每个元素转换成本。 

## 方法

 乍一看，人们可能会尝试选择一个目标值 T，并为每个元素 ai 计算使用乘法和特殊除法运算将 ai 转换为 T 所需的最少操作数。 这成为整数上的最短路径样式转换问题，其中每个数字可以移动到倍数或某些除数。 

这种强力解释原则上是正确的，因为每个有效的操作序列都对应于整数状态图中的一条路径。 然而，即使值以 n 为界，图形也很大，并且探索每个元素和每个可能目标的转换会导致重复工作。 在最坏的情况下，对于 n 个元素中的每一个，我们将探索最多 O(n) 个状态，从而导致每个测试用例的行为为 O(n²)，这对于总共 n 高达 10⁶ 的情况是不可行的。 

关键的观察是这两种操作都保留和操作素因子结构而不是任意数字结构。 乘以 k 会增加 ai 中素数的重数，而除法运算则有效地减少或重新分配它们。 唯一重要的不变量是每个人工智能与最终目标共享公共因子结构的距离有多远。 

我们不再考虑绝对值，而是转变视角：每个数字都可以分解为对对齐很重要的“核心形式”，而操作实际上是调整每个元素与共享结构的距离。 问题归结为对不匹配的因子分量进行分组和计数，而不是模拟转换。 

一旦重新构建，最佳解决方案就减少到识别整个数组中最“兼容”的结构并计算已经适合它的元素数量，因为已经对齐的元素需要更少或零操作，而其他元素则需要有限的调整。

 | --- | --- | --- | --- |
 | 蛮力 | 每次测试 O(n²) | O(n) | 太慢了|
 | 最佳 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1. 对于数组中的每个数字，将其分解为其素数结构或等效的压缩表示。 这是必要的，因为两个允许的操作仅影响乘法结构，因此原始值的信息量不够。 
2. 根据这些表示构建频率图。 目的是确定哪个结构出现最频繁，因为将所有内容与最常见的结构对齐可以最大限度地减少所需的修改数量。 
3. 计算已共享相同核心结构的最大元素组的大小。 该组代表不需要转换的元素。 
4. 答案是元素总数减去该最大组的大小，因为该组之外的每个元素必须至少修改一次才能与所选目标结构保持一致。 

为什么这种方法有效与以下事实有关：每个有效的最终配置都对应于选择一个规范结构并将所有其他元素转换为它。 任何转换序列都可以重新排列，以便每个元素独立地进入目标结构，因此每个元素的成本可相加分解。 这使得问题相当于最大化已经与单个选定代表匹配的元素数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        freq = {}
        for x in a:
            freq[x] = freq.get(x, 0) + 1

        best = max(freq.values())
        print(n - best)

if __name__ == "__main__":
    solve()
```该实现依赖于对值的频率进行计数作为其有效的“规范状态”。 我们没有明确地模拟乘法和除法，而是利用这样一个事实，即通过选择最常见的目标形式并将其他所有内容转换为它来实现最佳游戏。 字典在线性时间内累积出现次数，并取最大值给出最大的已对齐子集。 

减法`n - best`直接表示必须更改的元素数量。 不会出现订购问题，因为一旦目标确定，每个元素的成本都是独立的。

 ### 示例 1

```我们跟踪频率：

 | 步骤| 价值| 频率图 | 最佳|
 | --- | --- | --- | --- |

这表明我们不会尝试归一化为最大值 (4)，而是选择最频繁的结构。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        for x in a:

 ## 边缘情况

 对于像 [7, 7, 7, 7] 这样的相同元素的数组，频率映射包含值为 4 的单个键。算法选择它作为最佳组，产生 4 − 4 = 0，正确地表明不需要任何操作。 

对于像 [1, 2, 3, 4] 这样的完全多样化的数组，每个值都出现一次，因此最佳频率是 1。答案变为 3，这意味着我们必须更改三个元素以匹配所选的代表。 该算法自然地处理这个问题，无需特殊分支，因为即使所有计数相等，最大频率仍然有效。
