 ## 解决方案
 ## 问题理解

 我们得到一个由十进制数字组成的字符串。 唯一允许的操作是选择一个位置并将其数字向上或向下移动一位，保持在 0 到 9 的范围内。重复此操作任意次数，我们希望转换字符串，使其向前和向后读取相同的值。 成本是在所有位置上执行的单步增量或减量的总数，任务是最小化此成本。 

输入的结构以一种简单但严格的方式起作用：每个位置仅与其来自字符串另一端的镜像位置交互。 所有测试用例的总长度最多为 1000 的约束意味着任何解决方案即使每个测试用例执行二次量的额外工作也已经是安全的，而任何涉及嵌套重计算的问题仍然会通过，但没有必要。 

如果试图一步一步地模拟操作，就会发生一种天真的误解。 例如，如果我们采取一对像`1`和`9`，我们可能会重复增加较小的数字，直到它与较大的数字匹配，这会花费 8 次操作。 这在本地是正确的，但是对每个可能的中间目标依次执行会浪费推理时间并导致实现过于复杂。 

如果假设两个数字必须独立地朝固定方向改变，则会出现另一种微妙的故障模式。 例如，将两个数字都转向 0 或 9 可能看起来对称，但并不总是最佳的。 最佳会面点在全球范围内并不固定。 

## 方法

 蛮力视角独立地对待每个位置，但模拟所有可能的方式来使镜像角色相等。 对于一对数字`a`和`b`，可以尝试选择一个目标数字`d`从 0 到 9 并计算将双方转换为`d`。 这给出了成本`|a - d| + |b - d|`，并取所有最小值`d`产生正确的答案。 这是正确的，因为任何最后相等的数字都必须位于 0 到 9 之间的整数行上的某个位置。 

然而，这会为每对引入超过 10 个可能目标的不必要的循环，从而导致粗略的`O(10n)`每个测试用例的工作。 虽然仍然是线性的，但它隐藏了表达式具有封闭形式简化的事实。 

关键的观察是几何的而不是程序的。 在数轴上，当交点位于两个点之间时，将两个点移动到公共交点的可能性最小。 在这种情况下，总距离会缩减为端点之间的直接距离。 因此，我们可以直接将成本计算为两个数字之间的绝对差，而不是搜索会议数字。 

这将整个问题简化为对镜像位置上的这些成对差异求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（尝试每对的所有目标）| O(n·10) | O(1) | O(1) | 已接受 |
 | 最优（直接配对差异）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例并计算该字符串的最小成本。 

1. 读取字符串并将每个字符解释为数字。 字符串长度定义存在多少个镜像对。 
2. 将累加器初始化为总成本为零。 这将收集所有对称对所需的最少操作。 
3. 对于每个索引`i`从字符串的开头到中点，将其与索引配对`n - i - 1`。 这确保每个位置都精确匹配一次。 
4. 计算该对中两个数字之间的绝对差。 该值表示使它们相等所需的最小单步操作数。 
5. 将该差值添加到累加器中。 每对都是独立的，因此这些成本直接相加。 
6. 处理完所有对后，输出累加和。 

### 为什么它有效

 每个镜像对都独立于所有其他镜像对，因为一个位置上的操作不会影响另一个位置。 对于任何两个数字，使它们相等的最小成本是通过沿着整数线向彼此移动来实现的，并且该过程中所经过的总距离恰好是它们之间的绝对差。 由于每个有效回文必须在所有镜像对上强制执行相等，因此全局最优值是独立最优对成本的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        s = input().strip()
        n = len(s)
        ans = 0

        for i in range(n // 2):
            a = ord(s[i]) - ord('0')
            b = ord(s[n - i - 1]) - ord('0')
            ans += abs(a - b)

        print(ans)

if __name__ == "__main__":
    solve()
```该代码遵循配对对称索引并累积绝对差的结构。 转换使用`ord`尽管是直接的，但避免了重复的整数解析开销`int()`考虑到限制，也是可以接受的。 

循环仅运行到`n // 2`，确保每对都被精确计数一次。 对于奇数长度的字符串不需要特殊处理，因为中间字符本身已经满足回文条件并且贡献零成本。

 ## 边缘情况

 单字符字符串已经是回文，因为没有镜像约束需要满足。 该算法处理零对，因此累积成本保持为零，与预期结果匹配。 

对于像这样的输入`19`，算法将两个数字配对并计算`|1 - 9| = 8`。 任何通过中间数字进行路由的尝试仍然会累积恰好八个单位步骤，并且直接计算反映了无需模拟的最小路径。 

对于已经是回文的字符串，例如`1221`，每个镜像对都由相同的数字组成，每对产生零贡献。 该算法自然会保留这一点，因为每个绝对差值均为零。 

对于奇数长度的字符串，例如`12321`，中心数字永远不会与任何东西配对。 循环故意停在一半长度，确保中心不会影响结果，并且不会发生错误配对。
