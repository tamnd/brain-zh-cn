 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供一棵树。 乔治和穆罕默德两个玩家轮流在这棵树上行动。 在每一回合中，玩家执行两个操作中的一个：要么删除树上所有当前的叶子，要么选择一个叶子来保留并删除所有其他叶子。 

叶子是当前度数最多为 1 的任何节点，因此叶子集合在每次删除后都会发生变化。 球员们交替进行，乔治总是先行动。 当玩家无法执行有效操作时游戏结束，该玩家失败。 

关键的困难在于，树正在以一种不平凡的方式不断缩小，因为“保留一片叶子”可以保留结构，而删除其他叶子可能会立即崩溃多个层。 这使得游戏不再关乎局部动作，而是关乎树可以承受多少次“叶子剥落”。 

输入大小很大：所有测试用例最多有 100,000 个节点。 这排除了任何通过对整棵树进行线性或对数扫描，以幼稚的方式重复重新计算叶子的模拟。 解决方案必须在每个测试用例的线性时间内有效地推理树结构。 

当树极小或高度倾斜时，就会出现微妙的边缘情况。 

例如，如果树只有一个节点：

 输入：```
1
1
```就改变任何有意义的举动所需的意义上来说，没有叶子。 乔治无法进行任何操作，所以乔治立即失败。 任何解决方案都必须明确处理这种情况。 

对于两个节点的链：```
1
2
1 2
```两个节点最初都是叶子，但在任何操作后树都会立即消失，乔治获胜，因为穆罕默德在第一次完整叶子移除后没有响应。 忽略同时去除叶子的天真的推理可能会错误地评估这种情况。 

这些例子表明，真正的状态变化不是每个节点，而是每个“叶子层”。 

## 方法

 暴力模拟将显式地维护树，重复重新计算叶子集，并模拟每个玩家的移动。 每个操作都需要扫描所有节点以识别叶子，然后更新度。 由于每个回合可能会删除大部分节点，但游戏中的总转换数量仍可能高达 O(n)，因此在最坏的情况下这会导致 O(n^2) 行为。 对于 100,000 个节点，这太慢了。 

关键的见解是，游戏并不取决于每层内的确切分支结构，而仅取决于树倒塌之前可能进行多少轮叶子移除。 每一步都会通过剥去一层叶子来有效地减少树的“叶子深度结构”，并可选择保留单个分支。 

这将问题转化为分析我们可以迭代地从树上移除叶子多少次，直到它变得微不足道。 这个数字本质上是在剥叶轮中测量的树的高度，这与树的直径密切相关。 事实上，在最坏的情况下，每个完整的“删除所有叶子”操作都会将树高减少两层，并且保留一个叶子的选项只会改变奇偶性，而不是幅度。 

因此，结果简化为关于完全消除树所需的有效剥皮轮数的类似平价决策。 一旦计算出这个数字，获胜者将取决于乔治（第一位玩家）面临的决定性回合数是奇数还是偶数。 

我们可以使用两次 BFS 遍历来计算树直径，然后将其转换为叶子去除层的数量。 答案取决于该层数是奇数还是偶数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n²) | O(n) | 太慢了|
 | 基于直径的缩减 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将游戏简化为树的结构属性：它的直径。 

1. 构建树的邻接表表示。 这允许在每个测试用例的线性时间内有效地遍历。 
2. 从任意节点（通常是节点 1）运行 BFS 来查找最远的节点 A。这是可行的，因为在一棵树中，来自任何节点的 BFS 都会到达直径的一个端点。 
3. 从节点 A 开始运行第二次 BFS，找到最远的节点 B，并记录 A 和 B 之间的距离。这个距离就是树的直径。 
4. 将直径转换为所需的剥叶轮数。 每轮对应从两端剥一层，所以有效轮数为`(diameter + 1) // 2`。 
5. 根据平价决定胜负。 如果回合数为奇数，则乔治（第一个玩家）获胜； 否则穆罕默德获胜。 

这种转换背后的直觉是，每个“全叶移除阶段”都会从最长路径的两端对称地向内折叠树。 可选的“保存一片叶子”移动仅影响哪个端点暂时幸存，但它不会改变从根本上需要多少次此类崩溃。 

### 为什么它有效

 树在反复去除叶子的情况下的演化取决于其最长路径上存在的顶点层数。 每个操作最多将这种分层结构减少一个有意义的单元。 直径捕获任意两个叶子之间的最大可能深度，因此控制交替折叠步骤的数量，直到树变空或成为单个节点。 由于玩家只影响最后一层是对称还是稍微偏移，所以游戏结果完全取决于塌陷层总数是奇数还是偶数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def bfs(start, adj):
    n = len(adj) - 1
    dist = [-1] * (n + 1)
    q = deque([start])
    dist[start] = 0
    far = start

    while q:
        u = q.popleft()
        for v in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                q.append(v)
                if dist[v] > dist[far]:
                    far = v
    return far, dist[far]

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        adj = [[] for _ in range(n + 1)]

        for _ in range(n - 1):
            u, v = map(int, input().split())
            adj[u].append(v)
            adj[v].append(u)

        if n == 1:
            out.append("Neodoomer")
            continue

        a, _ = bfs(1, adj)
        b, diameter = bfs(a, adj)

        rounds = (diameter + 1) // 2

        if rounds % 2 == 1:
            out.append("Go8")
        else:
            out.append("Neodoomer")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先为每个测试用例构建邻接表。 对于单节点树，它立即输出乔治的失败条件，因为不可能移动。 

BFS 函数使用两次来计算直径。 第一个 BFS 识别远端点，第二个 BFS 测量距该端点的最大距离。 

转换`(diameter + 1) // 2`捕获有效的叶子去除轮数。 最终的奇偶校验决定了获胜者。 

唯一微妙的实现细节是确保 BFS 重置每个测试用例的距离，并且为每棵树重建邻接列表，因为多个测试用例是按顺序处理的。

    for _ in range(t):

 ## 边缘情况

 对于单节点树，算法在尝试任何 BFS 之前显式返回“Neodoomer”。 这避免了对空结构的不正确的直径计算，并且符合不存在移动的事实。 

对于二节点树，BFS 发现直径为 1，导致`(1 + 1)//2 = 1`圆形的。 奇数奇偶性正确地将胜利分配给了乔治，反映出第一步立即使树倒塌。 

对于星形树，无论中心的高度如何，BFS 仍然计算直径 2。 该算法完全忽略分支，这与所有叶子在规则下都是等效的并且只有深度很重要的事实相匹配。 

对于长链，直径等于 n − 1，产生许多轮。 奇偶性为`(n // 2)`确定获胜者，表明交替的叶层是通过直径压缩而不是显式模拟完全捕获的。
