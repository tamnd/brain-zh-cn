 ## 解决方案
 ## 问题理解

 我们得到了一组具有唯一名称的文件。 其中一些当前放置在称为舞台的特殊区域中，而其余的则放置在工作区中。 我们还获得了一组目标文件，这些文件最后必须完全位于 Stage 中，而所有其他文件则位于 Stage 之外。 

我们允许进行四种类型的操作：在舞台和工作区之间以任一方向移动单个文件，或者在单个操作中将所有文件从一侧移动到另一侧。 每个操作都需要一次移动，我们希望最大限度地减少将初始阶段配置转换为所需的最终配置所需的移动总数。 

关键的困难在于批量操作可能会同时修复许多不匹配，但它们也具有破坏性，因为它们会移动所有内容，无论它是否正确。 解决方案必须决定何时值得单独修复元件以及何时完全重置是有益的。 

从结构意义上来说，约束很小。 每次测试的文件总数最多为100个，并且名称是唯一的。 这立即排除了对大量组合优化或图搜索的任何需要。 每次测试以线性或二次时间检查初始集和目标集之间的所有关系的解决方案就足够了。 

当初始阶段已经与目标完全匹配时，会出现微妙的边缘情况。 简单的解决方案可能仍会执行不必​​要的操作，但正确的答案是零。 另一个特殊情况是阶段和目标完全不相交。 在这种情况下，我们必须决定是单独移动所有内容还是执行完全重置，然后选择性插入。 

## 方法

 蛮力视角是从状态的角度来思考。 每个文件位于 Stage 或 Workspace 中，因此最多有 2^n 个配置。 从任何配置中，我们都可以应用四种操作之一并转换到另一种状态。 最短路径搜索（例如 BFS）将找到最少的操作数。 这是正确的，因为每个操作都有相同的成本，并且状态空间是有限的。 

然而，即使n很小，2^100也是完全不可行的。 甚至存储状态也变得不可能。 操作的结构表明，大多数状态空间是无关的，因为我们不关心中间配置，只关心文件最终是否相对于目标正确放置。 

关键的观察结果是，在比较初始阶段成员资格和所需的最终阶段成员资格时，每个文件都属于四个类别之一。 文件可能已经是正确的，这意味着它开始并应在 Stage 中结束，或者开始并应在 Workspace 中结束。 它也可能不正确，这意味着它当前位于 Stage 中但不应该位于 Stage 中，或者当前位于 Workspace 中但应该位于 Stage 中。 重要的操作正是那些修复这些不匹配的操作。 

单个文件移动仅修复一个不匹配，而完整移动会重置整个配置，并且可能会立即减少许多不匹配，但代价是也会撤消正确的放置。 这会产生一个权衡：要么直接修复每个不正确的文件，要么执行全局重置，然后重建正确的阶段。 

该解决方案减少为计算初始配置中有多少文件已经正确以及有多少文件是错误的，然后决定全局重置是否有益。 由于完整移动会完全清空或填满舞台，因此在这样的操作之后，我们可以通过单独插入所需的文件来重建目标。 

事实证明，最佳策略仅取决于最初已正确放置在 Stage 中的文件数量。 如果许多目标文件已在 Stage 中，我们会避免全局移动，仅在本地调整不匹配。 如果很少或没有正确的，首先重置阶段会变得更便宜。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 状态 BFS 在所有配置上 | O(2^n·n) | O(2^n·n) | O(2^n) | O(2^n) | 太慢了 |
 | 设置交叉点计数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 将当前阶段列表转换为集合，以便成员资格检查的时间恒定。 这使我们能够快速确定文件当前是否位于 Stage 内。 
2. 将目标列表转换为另一组。 问题归结为比较这两组文件的完整范围。 
3. 计算初始阶段集中已存在多少目标文件。 这表示如果我们不执行破坏性的完整操作，则不需要执行任何操作的文件。 
4. 计算Stage 中丢失了多少个目标文件。 这些是我们最终必须使用单文件操作插入的文件，除非我们决定首先重置所有内容。 
5. 计算当前有多少额外文件位于 Stage 中但不在目标集中。 除非使用完全重置操作，否则必须将其删除。 
6. 评估两种策略。 第一个策略是仅使用单文件移动：必须将每个丢失的目标文件插入到 Stage 中，并且必须删除每个额外的文件。 该策略的成本正是初始阶段和目标阶段之间不匹配的数量。 
7. 第二种策略是执行一次完整的 Stage 重置，将所有内容移至工作区，然后单独插入所有所需的目标文件。 这需要一次操作加上 k 次插入。 
8. 返回这两个计算成本中的最小值。 

### 为什么它有效

 关键的不变量是，在决定是否使用完全重置后，每个剩余的操作对于每个文件都是独立的。 单个文件移动恰好解决了一个成员资格不匹配问题，并且从不与其他文件交互。 完全重置会消除所有先前的正确性信息，因此只有当不匹配的数量足够大以至于从头开始重建比单独修复它们更便宜时，它才有用。 由于不存在在规模上部分保留正确性的中间操作，因此这两种策略完全覆盖了所有最优路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n, m, k = map(int, input().split())
    s = input().split()
    stage = set(input().split())
    target = set(input().split())

    # count mismatches relative to final target
    missing_in_stage = 0
    extra_in_stage = 0

    for x in target:
        if x not in stage:
            missing_in_stage += 1

    for x in stage:
        if x not in target:
            extra_in_stage += 1

    # cost if we only do single-file moves
    cost_direct = missing_in_stage + extra_in_stage

    # cost if we reset stage then build target
    cost_reset = 1 + k

    print(min(cost_direct, cost_reset))
```该代码首先读取每个测试用例并将阶段和目标集合转换为集合，这允许 O(1) 成员资格检查。 然后，它会计算 Stage 中缺少的目标文件数量以及 Stage 中当前存在的无关文件数量。 这两个量充分描述了仅使用单文件操作修复配置的成本。 

另一种成本假设我们执行完全重置，这需要一次操作，然后单独插入每个所需的目标文件，这需要 k 次操作。 取这两种策略之间的最小值可以保证最优性。 

一个微妙的实现点是计算中不需要所有文件的初始列表。 尽管它出现在输入中，但它不会影响成本结构，因为操作仅依赖于阶段成员资格转换。 

## 工作示例

 ### 示例 1

 输入：

 阶段 = {dp}

 目标={贪婪，实现}

 我们计算缺失的和额外的集合。 

| 步骤| 舞台| 目标| 失踪| 额外 | 直接成本| 重置成本 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 初始化| {dp} | {g，i} | 2 | 1 | 3 | 3 |

 直接策略删除dp并添加两个目标文件，花费3次操作。 重置策略需要 1 + 2 = 3 次操作。 两者都是平局，所以两者都是最佳的。 

这显示了全局重置同样好但并非严格更好的情况。 

### 示例 2

 输入：

 阶段 = {地理、树木、数学}

 目标={数学，BS}

 | 步骤| 舞台| 目标| 失踪| 额外 | 直接成本| 重置成本 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 初始化| {g,t,m} | {米，废话} | 1 | 2 | 3 | 3 |

 直接成本删除两个额外文件并插入一个丢失的文件，总共 3 个。重置成本为 1 + 2 = 3。这两种策略再次一致。 

这凸显了相等情况很常见，因为 k 相对于不匹配来说很小，并且两种策略都崩溃为类似的线性表达式。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | 每次测试 O(n) | 我们只扫描阶段和目标集一次 |
 | 空间| O(n) | 将存储文件名设置为最大大小 n |

 这些约束将测试中的总 n 限制为 100，因此这个线性解决方案很容易足够快。 内存使用量可以忽略不计，因为我们只存储几小组字符串。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n, m, k = map(int, input().split())
        s = input().split()
        stage = set(input().split())
        target = set(input().split())

        missing = sum(1 for x in target if x not in stage)
        extra = sum(1 for x in stage if x not in target)

        out.append(str(min(missing + extra, 1 + k)))

    return "\n".join(out)

 ## 边缘情况

 一种边缘情况是阶段已经等于目标。 对于像 Stage = {a, b} 和 Target = {a, b} 这样的输入，缺失计数和额外计数都为零，因此直接成本为零，重置成本为 1 + k，严格来说更大。 该算法无需执行任何操作即可正确返回零。 

另一个边缘情况是阶段和目标不相交。 对于 Stage = {a, b} 和 Target = {c, d}，missing 为 2，extra 为 2，因此直接成本为 4。重置成本为 1 + 2 = 3，因此算法正确地倾向于重置，这反映出单个全局擦除比单独纠正所有不匹配更便宜。 

当 Stage 为空且 Target 也为空时，会出现最后一种边缘情况。 两个成本分别评估为零和一，因此答案为零，与不需要任何操作的事实相匹配。
