 ## 解决方案
 ## 问题理解

 我们从一个数组开始，该数组最初包含按排序顺序从 1 到 n 的整数。 该过程恰好重复 n 次，每次重复都包含对当前数组执行的两个操作。 

首先，我们删除数组的中值元素。 中位数定义为当前数组大小为 m 时位置 ⌈m/2⌉ 处的元素。 由于数组在概念上始终按照剩余值的相同顺序排序，因此中位数始终是剩余集合的中间元素。 

其次，删除中位数后，我们计算剩余数组的 MEX 并将其添加到运行分数中。 这里的MEX是当前数组中没有出现的最小正整数。 

关键输出是所有 n 次移除累积的总分。 

约束条件很大，n 最多为 10^9，测试用例最多为 10^5。 这立即排除了任何阵列模拟或重复中值提取。 即使保持平衡的结构也太慢了，因为我们会对每个测试用例进行 n 次删除。 

一个微妙的边缘情况来自于对收缩前缀的 MEX 的理解。 例如数组为[1]，去掉中位数留下一个空数组，空的MEX为1。另一个小情况是n = 2：开始[1,2]，中位数为1，剩余[2]，MEX为1，然后中位数2为空，MEX又为1。 最终得分为 2。任何关于 MEX 删除后如何演变的错误直觉都将在这里破裂。 

真正的困难在于中位数去除是全局性的，但 MEX 仅取决于最小的缺失整数，这与早期小数字的去除方式密切相关。 

## 方法

 暴力模拟将按排序顺序维护数组。 每一步都会删除中值元素，然后从 1 向上扫描以找到 MEX。 如果在一个简单的数组中完成，移除中位数的时间复杂度为 O(n)，并且在最坏的情况下找到 MEX 的时间复杂度也为 O(n)。 重复此 n 次会导致每个测试用例的复杂度为 O(n^2)，即使对于单个测试，n 达到 10^9 也是不可能的。 

即使使用平衡 BST 或顺序统计树，中值去除也可以减少到 O(log n)，但计算 MEX 仍然需要跟踪最小的缺失正整数。 如果简单地完成，MEX 查询仍然很昂贵，除非我们维护一个单独的结构。 即便如此，更深层次的问题仍然是我们实际上不需要模拟所有移除。 

关键的观察结果是，该过程是确定性的，并且仅取决于删除的相对顺序，而不取决于数组的完整结构。 由于我们总是删除剩余集合 {1, 2, ..., k} 的中位数，因此删除元素的序列是通过重复取缩小的连续集合的中间来完全确定的。 这会产生将原始数字按删除顺序进行可预测的划分。 

同时，MEX 在任何时刻都取决于有多少个最小整数已被删除。 如果我们跟踪每个整数在被删除之前存活了多长时间，我们就可以准确确定它何时对 MEX 值积累做出贡献。 

关键的简化是颠倒视角。 我们不是模拟删除，而是确定 [1, n] 中的每个整数 i 在哪一步被删除。 一旦我们知道了每个值的移除时间，我们就可以重构 MEX 的演化方式：在步骤 t，MEX 是移除时间≥ t 的最小数。 

这将问题简化为理解从连续段中重复获取中位数的消除顺序，这是经典的分而治之模式。

 ## 算法演练

我们在集合 {1, 2, ..., n} 上对移除过程进行建模。 在每一步中，我们都会删除当前段的中值。 这相当于在索引上构建一棵二叉树，其中根是中位数，左子树在左半部分是相同的过程，右子树在右半部分是相同的过程。 移除顺序正是这个隐式线段树的前序遍历。 

1. 我们定义一个函数，通过模拟间隔上的分而治之的分割来为每个值分配其删除步骤。 对于区间 [l, r]，我们计算 mid = (l + r) // 2 （对中间位置进行适当的 ceil 处理）并为其分配下一个可用的移除时间。 
2. 我们递归地处理左区间 [l, mid - 1] 和右区间 [mid + 1, r]，不断增加深度。 这构建了与重复中值删除一致的完整删除顺序。 
3. 一旦我们知道每个值的移除时间，我们就按它们的移除时间对值进行排序。 这给出了元素从数组中消失的确切顺序。 
4. 我们现在计算 MEX 随着时间的推移的贡献。 我们扫描从 1 到 n 的时间步长，并维护一个指针 mx 来跟踪尚未删除的最小数字。 在每个步骤 t，我们将 mx 添加到答案中，如果元素 mx 在时间 t 被删除，我们向前递增 mx 直到找到仍然存在的值。 
5. 我们将这些贡献累积到最终得分中。 

关键思想是时间 t 的 MEX 仅取决于时间 t 尚未删除的最小索引。 由于删除时间已知，因此检查某个值是否保留下来是一个简单的比较。 

### 为什么它有效

 中值移除过程以始终选择任何活动区间的中间元素的方式对数组进行分区。 这保证了每个元素的删除时间仅由其在递归中值分裂树中的位置决定。 任何步骤的 MEX 仅取决于从 1 开始的值的前缀生存，因此跟踪最早的尚未删除的整数就足够了。 由于移除时间定义了严格的排序，因此 MEX 会单调演化，除非当前最小的存活数被移除（这会使 MEX 向上移动）。 这确保了一旦移除时间已知，就可以在恒定的摊销时间内计算每个步骤。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def build(l, r, t, order):
    if l > r:
        return
    mid = (l + r) // 2
    order[mid] = t
    build(l, mid - 1, t + 1, order)
    build(l, mid + 1, r, t + 1, order)

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())

        order = [0] * (n + 1)
        build(1, n, 1, order)

        # bucket elements by removal time
        # we will invert into time -> value mapping
        pos = [[] for _ in range(n + 2)]
        for i in range(1, n + 1):
            pos[order[i]].append(i)

        removed = [False] * (n + 2)
        mex = 1
        ans = 0

        for time in range(1, n + 1):
            for v in pos[time]:
                removed[v] = True

            while removed[mex]:
                mex += 1

            ans += mex

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```递归函数`build`根据中值分割结构为每个数字分配一个移除时间。 每个节点都在其子节点之前分配，这对中位数被删除的确切顺序进行编码。 

这`pos`数组反转此映射，以便我们可以有效地处理给定时间步发生的所有删除。 这避免了每次迭代扫描所有元素。 

这`mex`指针是单调维护的。 它只向前移动，因此即使循环运行 n 次，每个值最多访问一次，这使每个测试用例的总工作保持线性。 

一个微妙的点是递归深度对应于 O(log n)，因为每次分割都会将间隔减半。 尽管 n 可能很大，但除了存储删除时间之外，我们永远不会在内存中构建该大小的显式数组。 

## 工作示例

 ### 示例 1

 考虑 n = 5。 

我们首先计算中值分割的移除时间：

 | 价值| 1 | 2 | 3 | 4 | 5 |
 | --- | --- | --- | --- | --- | --- |
 | 移除时间 | 3 | 2 | 1 | 2 | 3 |

 现在我们模拟：

 | 时间 | 已删除 | 墨西哥 | 分数 |
 | --- | --- | --- | --- |
 | 1 | 3 | 1 | 1 |
 | 2 | 2,4 | 1 | 2 |
 | 3 | 1,5 | 1,5 | 1 | 3 |
 | 4 | - | 1 | 4 |
 | 5 | - | 1 | 5 |

 最终得分为5分。 

这表明，在足够早地移除最小元素后，MEX 会迅速稳定并保持恒定。

 ## 边缘情况

 ### n = 1

 输入：```
1
1
```唯一的元素将立即被删除。 数组变空，MEX 为 1，因此答案为 1。算法将移除时间 1 分配给值 1，mex 从 1 开始，并且对单步保持有效。 

### n = 2

 输入：```
1
2
```根据中值约定，值 1 首先或第二被删除，但删除时间映射可确保正确的排序。 在时间 1 时，mex 为 1。在时间 2 时，删除剩余元素，mex 仍为 1。总计为 2。 

### n = 5

 递归拆分分配结构化删除时间，确保正确的预排序删除顺序。 只有当 1, 2, 3... 被删除时，mex 指针才会前进，与定义完全匹配。
