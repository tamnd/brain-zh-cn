 ## 解决方案
 ## 问题理解

 我们得到一个数组$A$我们想要将其转换为非递减序列。 除此之外，我们还有一套单独的$B$的备用值。 跨越的每一个元素$A$和$B$是独特的。 

唯一允许的操作是从中获取值$B$并覆盖任意位置$A$与它，以及每个元素$B$最多可以使用一次。 目标不是重新安排$A$直接，但策略性地替换一些条目，以便最终数组按非降序排序，同时最大限度地减少使用的替换次数。 如果没有任何替换序列可以实现有序数组，则答案是$-1$。 

一个关键的结构性限制是替代品不能随意重复使用。 中的每个值$B$是一次性工具，因此问题不仅仅是“有多少位置错误”，而是“可以用哪些可用值修复哪些位置”。 

约束条件允许$N, M$最多$5 \cdot 10^5$，这会立即排除任何尝试模拟替换或尝试子集的解决方案。 任何平方项$N$或者$M$太慢了。 有效的方法必须依赖于预处理后的排序和线性或近线性扫描。 

一个微妙的一点是我们没有被要求做$A$通过重新排列或插入元素排序，但严格通过替换排序。 这意味着每个最终值必须来自原始数组或来自$B$，我们正在有效地选择要“修复”哪些位置以及要分配哪些值。 

打破朴素推理的边缘情况很容易被忽略。 例如，如果$A = [2, 6, 13, 10]$和$B = [5]$，人们可能会认为用 5 替换 13 可以修复 10 的反转，但如果我们稍后需要更大的值，它可能会破坏左侧的排序约束。 这正是局部修复失败的原因：替换必须与最终的单调结构全局一致。 

另一个边缘情况是当所有元素都在$B$要么太小，要么太大，无法修复必要的约束。 例如，如果$A = [3, 1, 2]$和$B = [0]$，没有办法修复 3 和 1 之间的反转，同时保持其余部分的可行性。 

## 方法

 蛮力策略会尝试所有位置的子集$A$进行替换，并为每个子集检查我们是否可以分配不同的值$B$以便对结果数组进行排序。 对于大小的固定子集$k$，我们还需要选择哪个$k$值来自$B$使用它们并将其与位置相匹配。 即使可行性检查是线性的，子集的数量也是$2^N$，并且赋值会增加阶乘复杂性。 这在非常小的情况下是完全不可行的$N$。 

关键的观察是，一旦最终的数组被排序，结构就变得严格了：我们必须匹配一个目标非递减序列，并且每个位置都会对最终的值施加限制。 我们不再考虑任意替换，而是转而考虑如何从左到右构建有效的排序数组，同时最大限度地减少被迫使用的频率$B$。 

重新构建问题的一个有用方法是将最终排序的数组视为一些必须与原始“兼容”的非递减序列$A$。 如果我们处理$A$从左到右，我们希望保留尽可能多的原始值，但前提是它们仍然能够适应全球增长的结构。 每当一个元素破坏了可行性，我们可以替换它，但我们应该使用最小的可用值$B$可以保持一致性。 一旦我们了解可行性仅取决于排序约束，而不取决于仓位的同一性，这自然就表明了贪婪策略。 

更深入的见解是，我们可以将问题视为构造最长的前缀$A$可以保持不变，同时仍然可以使用可用的替换扩展到排序数组。 任何无法保留的元素都会迫使我们使用替代品，而替代品的选择必须尊重未来的排序，这促使我们对两者进行排序$A$和$B$和贪婪匹配约束。 

最终的解决方案变成了一次扫描，我们在最终数组中维护最后选择的值并决定是否保留$A[i]$或者用尽可能小的替换它$B$仍保持非递减顺序的值。 如果两者都不可能，则配置是不可能的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集+验证|$O(2^N)$|$O(N)$| 太慢了|
 | 贪心排序数组 |$O((N+M)\log(N+M))$|$O(N+M)$| 已接受 |

 ## 算法演练

 ## 算法演练

 1. 对两个数组进行排序$A$和$B$。 排序$B$使我们能够在需要时始终选择最小的可用替代品，这一点至关重要，因为更大的替代品只会使未来的可行性变得更加困难。 
2. 初始化一个指针$j = 0$为了$B$，和一个变量$last = -\infty$表示放置在构造的最终数组中的最后一个值。 这跟踪单调约束。 
3. 扫描每个元素$A[i]$从左到右，决定是保留还是替换。 这个顺序很重要，因为早期的决定限制了所有未来的头寸。 
4. 如果$A[i] \ge last$，我们可以安全地保留$A[i]$，我们设置$last = A[i]$。 这是最优的，因为保留原始值可以减少替换次数。 
5. 否则，我们尝试使用以下方法修复违规行为$B$。 使指针前进$j$直到我们找到最小的$B[j] \ge last$。 如果这样的元素存在，我们使用它，增加替换计数，然后更新$last = B[j]$，然后移动$j$向前。 
6. 如果两者都不是$A[i]$也没有任何剩余价值$B$可以满足$last$，我们得出结论该过程是不可能的并返回$-1$。 这对应于可用值中的差距，在不破坏单调性的情况下无法弥合该差距。 
7. 继续，直到所有元素$A$已处理，并返回已使用的替换数。 

### 为什么它有效

 该算法在处理索引后保持了不变式$i$，构造的序列是使用某些替换子集可实现的字典顺序上最小的可能非递减序列。 排序$B$确保每当我们必须插入替换时，使用最小的可行值可以为未来的位置保留最大的灵活性。 任何更大的选择只会收紧未来的约束，而不会提高当前的可行性，因为唯一的要求是非递减顺序，而不是接近原始值。 这种贪婪的优势确保如果存在解决方案，算法永远不会不必要地阻塞自己。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    a.sort()
    b.sort()
    
    j = 0
    last = -10**18
    used = 0
    
    for x in a:
        if x >= last:
            last = x
        else:
            while j < m and b[j] < last:
                j += 1
            if j == m:
                print(-1)
                return
            last = b[j]
            j += 1
            used += 1
    
    print(used)

if __name__ == "__main__":
    solve()
```该实现直接反映了贪婪构造。 对两个数组进行排序是将问题转化为单调匹配过程的结构步骤。 指针指向上方$B$确保每个替换最多使用一次并且始终选择最少。 

一个微妙的实现细节是使用一个大的负初始值$last$，这避免了第一个元素的特殊情况。 另一个是我们从不重新考虑之前的选择，因此排序后扫描是严格线性的。

 ## 边缘情况

 一个关键的边缘情况是数组已经排序。 该算法永远不会进入替换分支，因为每个$A[i]$满足单调条件，因此答案仍然是零替换。 

另一个边缘情况是当所有元素都在$B$小于当前约束。 在这种情况下，指针指向$B$在没有找到有效候选者的情况下耗尽，算法正确返回$-1$。 这对应于没有替代品可以弥补单调需求的差距。 

第三种情况是贪婪的选择可能看起来有风险，例如选择最小的有效值$B$价值。 该跟踪显示，早期使用较大的值可能会阻碍以后的可行性。 该算法通过始终消耗最小的可行替换，为未来元素保留最大的余量来避免这种情况。
