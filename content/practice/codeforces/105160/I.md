 ## 解决方案
 ## 问题理解

 我们在平面上有两个圆。 每个圆都由其中心坐标和半径定义。 对于每个测试用例，我们需要计算存在多少条不同的直线，使得该线同时与两个圆相切。 如果有无限多条这样的线，我们必须输出-1。 

从几何学上讲，如果一条线恰好在一个点处接触每个圆，而不穿过内部，则该线是有效的。 相同的几何线仅被计数一次，即使它可以被视为来自不同的“切线配置”。 

就测试用例而言，约束很大，最多 100000 个，而坐标和半径最多 10000 个。这会立即排除任何显式枚举候选线或对所有可能的切线执行相交检查的每次测试几何构造。 该解决方案必须将每个测试用例的算术量减少到恒定量。 

当两个圆重合时，就会出现微妙的边缘情况。 如果圆心和半径相同，则该圆的每条切线同时与两者相切，从而产生无限多条有效线。 当圆嵌套或者一个圆严格位于另一个圆内部而不接触时，会出现另一种极端情况； 在这种情况下，不存在公共切线。 仅检查中心距与半径总和的简单方法通常无法正确区分外部和内部相切配置。 

## 方法

 强力解释将尝试构造第一个圆的所有切线，然后针对第二个圆测试每条线。 单个圆有无限多条切线，因此在实践中，人们会离散方向或尝试通过角度采样或斜率参数化来枚举切线。 即使我们将自己限制为由接触点定义的切线，这仍然涉及连续参数和求解每个候选方向的二次方程。 对于每条候选线，验证与两个圆的相切需要距离计算和根检查，导致对于某些大 K 的每个测试用例至少需要 O(K) 工作。这在 100000 个测试用例下基本上是不可行的。 

关键的观察是，两个圆的任何公共切线必须是四种经典几何类型之一：它可以是外部公切线（接触线段连接中心同一侧的两个圆）或内部公切线（穿过线段连接中心，接触相对侧）。 每种类型都简化为圆心和半径之间的距离条件。 

设中心为 A 和 B，半径为 r1 和 r2，设 d 为中心之间的距离。 切线的存在仅取决于将 d 与 r1 + r2 和 |r1 − r2| 进行比较。 这是通过沿切线的法线方向扩大或缩小另一个圆来将一个圆转变为一点而实现的。 每个有效配置对应于特定的几何不等式，有效切线的数量由这些配置的可能数量决定。 

当圆重合（无限多条切线）、内部或外部接触（某些切线类型折叠成一条线）或一个圆完全位于另一个圆内部（无切线）时，会发生特殊的简并性。 

因此，整个问题简化为仅使用三个量对两个圆的相对位置进行分类：中心距离、半径之和以及半径的绝对差。

 ## 算法演练

对于每个测试用例，我们计算中心之间的平方距离以避免浮点错误。 然后我们将其与从半径导出的平方阈值进行比较。 

1. 读取两个圆(x1,y1,r1)和(x2,y2,r2)。 如果圆的圆心和半径相同，我们输出 -1，因为单个圆的每条切线对两者都有效。 
2. 计算 dx = x1 − x2 和 dy = y1 − y2，并定义 d2 = dx·dx + dy·dy。 这表示中心之间的平方距离。 
3. 计算 sumR = r1 + r2 和 diffR = |r1 − r2|。 这些值定义外部和内部相切配置的阈值。 
4. 如果 d2 为零但圆不相同，则圆具有相同的中心但具有不同的半径。 在这种情况下，没有一条线可以与两者相切，因为一个圆的任何切线都完全位于另一个同心圆之外。 我们输出0。 
5. 如果 d2 < diffR²，则一个圆严格位于另一个圆内部而不接触。 没有切线可以同时触及两个边界，因此我们输出 0。 
6. 如果 d2 等于 diffR²，则圆内切。 恰好存在一条内公切线。 
7. 如果 d2 > diffR² 且 d2 < sumR²，则圆重叠。 在该区域中，恰好存在两条外公切线。 
8. 如果 d2 等于 sumR²，则圆外切。 在这种情况下，存在三种不同的公切线：两条外切线和一条简并共享切线。 
9. 如果 d2 > sumR²，则圆是不相交的。 所有四个经典公切线都存在。 

### 为什么它有效

 每条公共切线均由垂直于切点处半径的方向矢量确定。 对于固定线，沿其法线方向移动可以减少比较从中心到线的有符号距离的问题。 相切施加的约束转化为该有符号距离的线性方程。 对两个圆同时求解这些问题正好得出涉及 d、r1 和 r2 的条件。 解的数量对应于可能有多少个一致的符号分配，这正是上面的案例分析所列举的。 由于分类仅取决于相对距离，因此不需要超出这些比较的几何构造。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        x1, y1, r1 = map(int, input().split())
        x2, y2, r2 = map(int, input().split())

        dx = x1 - x2
        dy = y1 - y2
        d2 = dx * dx + dy * dy

        if x1 == x2 and y1 == y2 and r1 == r2:
            out.append("-1")
            continue

        sumR = r1 + r2
        diffR = abs(r1 - r2)

        if d2 == 0:
            out.append("0")
        elif d2 < diffR * diffR:
            out.append("0")
        elif d2 == diffR * diffR:
            out.append("1")
        elif d2 < sumR * sumR:
            out.append("2")
        elif d2 == sumR * sumR:
            out.append("3")
        else:
            out.append("4")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现直接反映了分类。 第一个守卫处理无限相同圆的情况。 始终使用平方距离以避免精度问题。 每个后续分支完全对应于由圆是否嵌套、相切、相交或不相交定义的几何体系。 检查的顺序很重要，因为必须在严格的不平等之前将相等的情况分开，以避免错误分类。 

## 工作示例

 ### 示例 1

 输入：```
1
0 0 1
0 3 1
```这里 dx = 0，dy = −3，所以 d2 = 9。我们有 sumR = 2 且 diffR = 0，所以 sumR² = 4 且 diffR² = 0。 

| 步骤| d2 | 差异R² | 总和R² | 决定|
 | ---| ---| ---| ---| ---|
 | 计算距离 | 9 | 0 | 4 | 不相交的情况 |

 由于 d2 > sumR²，圆被分开并且所有四个切线都存在，所以答案是 4。 

这证实了充分分离的圆允许内部和外部切线的解释。 

### 示例 2

 输入：```
1
0 0 2
3 0 1
```这里 dx = 3，dy = 0，所以 d2 = 9。我们有 sumR = 3，diffR = 1，所以 sumR² = 9 且 diffR² = 1。 

| 步骤| d2 | 差异R² | 总和R² | 决定|
 | ---| ---| ---| ---| ---|
 | 比较 | 9 | 1 | 9 | 外切线|

 由于 d2 == sumR²，圆是外切线，产生 3 条公切线。 

此案例显示了相等处理的重要性：缺少相等分支会错误地将其分类为不相交或重叠。

 ## 边缘情况

 当圆相同时，算法触发第一个条件并立即输出-1。 例如，(0,0,1) 和 (0,0,1) 在任何距离计算之前就被捕获，正确地表示无限多个共享切线。 

对于不同半径的同心圆，例如(0,0,5)和(0,0,2)，距离的平方为零，而diffR²为正。 条件 d2 < diffR² 被触发，产生 0。这符合一个圆的任何切线不能接触另一个同心圆的几何事实。 

对于像 (0,0,2) 和 (3,0,1) 这样的外切圆，我们得到 d2 == sumR²。 相等分支确保我们返回 3。严格的仅不等式实现会将其错误分类为不相交，从而错误地输出 4。 

对于不相交的圆，例如 (0,0,1) 和 (10,0,1)，距离条件超过 sumR²，将配置置于具有 4 条切线的最终情况中，反映完整的外部和内部切线集。
