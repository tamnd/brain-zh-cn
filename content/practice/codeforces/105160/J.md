 ## 解决方案
 ## 问题理解

 我们得到一棵树，其节点标记为 1 到 n，再加上一个额外的节点 0。节点 0 连接到节点 1，因此节点 0 实际上就像附加在原始树之上的根。 每隔一条边将 n 个学生位置连接成一棵树。 

每个学生都生活在一个独特的节点i。 巴士路线是一条从不重复边缘且始终在节点 0 结束的路径。学生只能在自己的节点登上巴士，一旦上车，他们就会沿着该巴士乘坐，直到到达节点 0。 

因为图是一棵树，所以任何结束于节点 0 的边简单路径的行为就像一条从下到上的通向根的路径：它不能分支，也不能重新访问边，因此它实际上对应于选择一个起始节点并沿着通向节点 0 的唯一简单路径行走。 

一个关键的结果是，当且仅当所有这些学生都位于单个根向路径上时，单个总线才能为多个学生提供服务，这意味着它们的节点排列在有根树（以节点 0 为根）中的一个祖先链上。 

我们被要求计算有多少种方法可以选择 3 名学生，使得他们不会都由一辆公共汽车服务。 换句话说，我们计算所有节点三元组，除了三个节点位于单个根到叶路径上的节点之外。 

约束 n 最大为 2 × 10^5 意味着任何解决方案都必须接近线性或 O(n log n)。 任何对或三对节点的二次方都会太慢。 即使是 O(n^{4/3}) 式的解决方案在这里也是不必要的，因为结构是一棵树，并且建议在预处理后进行简单的组合计数。 

一个微妙的边缘情况源于对“同一辆巴士”含义的误解。 它与原始树中的连通性无关，而是与位于一个祖先链上有关。 例如，在以节点 1 为中心的星形中，节点 2、3、4 不在一条路径上，因此涉及它们的任何三元组都是有效的。 相反，在链 4-3-2-1-0 中，该链上的任何三个节点都是无效的，因为从 4 到 0 的一条总线经过所有这些节点。 

## 方法

 直接方法将枚举所有节点三元组并检查它们是否位于单个根到 0 的路径上。 检查这一条件需要验证一个节点是否是其中最深的节点以及其他两个节点是否是它的祖先。 这可以使用父指针或 O(1) 或 O(log n) 的 LCA 查询来完成，但枚举所有三元组的成本为 O(n^3)，这对于 n = 2 × 10^5 来说是不可能的。 

关键的观察是翻转视角。 我们不检查三元组，而是计算坏的三元组。 当所有三个节点都位于有根树中的单个祖先链上时，三元组是不好的。 

现在将节点 v 固定为该三元组中的最深节点。 其他两个节点必须在 v 的祖先中选择（排除 v 本身或根据公式仔细包括它）。 一旦我们正确定义了深度，每个坏三元组都有一个唯一的最深节点，这可以避免重复计算。 

在使用来自节点 0 的单个 DFS 或 BFS 计算深度后，这会将问题简化为节点上的纯粹组合总和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解三倍 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了 |
 | 树的深度计数| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以节点 0 为树根并计算每个节点的深度，其中深度[v] 是从 v 到节点 0 的路径上的学生节点数，包括 v 本身。

1. 建立树的邻接表，其中节点0与节点1相连。 
2. 从节点 0 运行 BFS 或 DFS 来计算父关系和深度。 我们设置深度[0] = 0，并且对于v的任何子节点u，深度[u] =深度[v] + 1。这确保深度[v]计算出从v到根侧的路径上有多少个学生节点（不包括节点0）。 
3. 预先计算学生三元组的总数为 C(n, 3)。 这代表所有可能的选择。 
4. 计算坏三元组的数量。 对于每个节点 v，将 v 视为三元组中最深的节点。 那么另外两个节点必须在v的祖先节点中选择，它们也是学生节点，其中有深度[v] - 1。 

因此，v 的贡献为 C(深度[v] - 1, 2)，前提是深度[v] ≥ 3。 
5. 对所有节点 v 求和该值。 
6. 答案是总三元数减去坏三元数。 

这种分解有效的原因是每个有效的“同一总线”三元组恰好对应于树顺序中的一个最深节点。 一旦该节点被固定，剩下的两个节点就被唯一地约束在它上面。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
adj = [[] for _ in range(n + 1)]

for _ in range(n - 1):
    u, v = map(int, input().split())
    adj[u].append(v)
    adj[v].append(u)

# include node 0 connected to 1
adj.append([])  # index n+1 unused (just safety, not needed)
adj[0] = [1]
adj[1].append(0)

parent = [-1] * (n + 1)
depth = [0] * (n + 1)

stack = [0]
parent[0] = 0

while stack:
    v = stack.pop()
    for u in adj[v]:
        if u == parent[v]:
            continue
        parent[u] = v
        depth[u] = depth[v] + 1
        stack.append(u)

def C2(x):
    return x * (x - 1) // 2

total = n * (n - 1) * (n - 2) // 6
bad = 0

for v in range(1, n + 1):
    if depth[v] >= 3:
        bad += C2(depth[v] - 1)

print(total - bad)
```该代码首先构建树并显式添加节点 0 和节点 1 之间的连接。DFS 计算以节点 0 为根的父数组和深度数组。 

函数 C2(x) 有效地计算对的二项式系数。 对于每个节点v，我们将深度[v] - 1解释为v之上的学生节点中有效祖先的数量。选择其中的任何两个与v一起形成一个坏三元组。 

最后，我们从三元组总数中减去这些。 

一个常见的陷阱是错误定义深度：如果深度从节点 1 而不是节点 0 开始，则组合公式就会失效。 另一个问题是忘记节点 0 不是学生并且不能包含在组合中。 

## 工作示例

 ### 示例 1

 考虑一个简单的链：3-2-1-0。 

深度为：

 节点 1: 1，节点 2: 2，节点 3: 3。 

| v | 深度[v] | 不良贡献 C(深度[v]-1,2) |
 | ---| ---| ---|
 | 1 | 1 | 0 |
 | 2 | 2 | 0 |
 | 3 | 3 | 1 |

 三元组总数为 C(3,3) = 1。 

坏三元组 = 1（三元组 {1,2,3}）。 

答案 = 0。 

这证实了链中的所有三元组都是无效的，因为它们位于一条路径上。 

### 示例 2

 考虑一个星形：节点 1 连接到 0，节点 2、3、4 连接到 1。 

深度：

 节点 1 = 1，节点 2、3、4 = 2。 

| v | 深度[v] | 贡献 |
 | ---| ---| ---|
 | 1 | 1 | 0 |
 | 2 | 2 | 0 |
 | 3 | 2 | 0 |
 | 4 | 2 | 0 |

 三元组总数 = C(4,3) = 4。 

坏三元组 = 0。 

答案=4。 

这表明在分支结构中，没有三个节点位于单个祖先链上。
