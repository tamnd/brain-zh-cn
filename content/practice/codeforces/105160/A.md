 ## 解决方案
 ## 问题理解

 我们得到了一个问题难度列表，其中每个问题还有一个由其在输入中的位置给出的隐式标识符。 任务是根据难度从小到大对问题索引重新排序。 当两个问题的难度相同时，原索引较小的问题一定先出现。 

输出的不是排序后的难度本身，而是按此规则排序后的原始索引序列。 

输入量很小，最多有1000个问题。 这立即允许任何围绕二次时间或更好的方法。 执行大约一百万个基本操作的解决方案是完全安全的，而任何涉及在另一个循环内重复进行完整扫描的操作仍然可能会通过，但没有必要。 

一个常见的错误是仅对难度值进行排序而不跟踪索引，这会丢失所需的输出。 另一个错误是对索引进行排序但忘记了平局规则，这会导致出现相同值时排序不正确。 例如，如果输入是`3 1 1`，正确的输出是`2 3 1`，因为两个`1`保持原来的顺序。 

当所有值都相等时，会出现另一种微妙的边缘情况。 如果`a = [5, 5, 5]`，输出必须是`1 2 3`，完全保留原始顺序。 

## 方法

 思考这个问题的强力方法是重复搜索剩余的最小难度，将其标记为已使用，并将其索引附加到结果中。 每个选择都需要扫描整个数组以找到下一个尚未使用的最小值。 这本质上是重复 n 次的选择过程，每次扫描成本为 O(n)，从而导致总操作量为 O(n²)。 

这种方法是正确的，因为它直接遵循排序规则：始终选择最小的可用元素，并且如果仔细执行，扫描会通过在相等值中选择最小索引来自然地解决平局问题。 然而，它在概念上变得低效，因为它多次重复地重新检查已经处理的元素。 

关键的观察结果是，如果我们将每个元素与其索引打包在一起，则排序规则正是标准排序算法已经提供的规则。 如果我们对对进行排序`(ai, i)`，排序比较首先使用难度，当难度相等时自动回退到索引顺序。 这将问题转化为对 n 个项目的单一排序。 

改进来自于认识到结构是静态的。 处理过程中没有任何变化，因此我们不需要增量选择。 一次性全局订购就足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力选择 | O(n²) | O(n) | 已接受但没有必要 |
 | 配对排序 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1.读取整数n和困难数组。 每个位置隐式代表一个从 1 开始的问题标识符。 
2. 构建一个对列表，其中每个元素都存储难度及其原始索引。 这样可以保留每个问题的同一性，同时可以进行比较。 
3. 对列表进行排序。 排序规则首先比较难度，如果两个难度相等，则比较索引。 这确保了在单个操作中强制执行所需的排序。 
4. 遍历排序列表并仅提取索引，产生最终的输出顺序。 

### 为什么它有效

 排序强加了与问题所需的排序关系一致的全局顺序。 比较函数定义了总排序，其中`(ai, i)`确保难度为主键，索引为决胜负。 排序后，任意两个元素都以正确的相对顺序出现，排序的传递性保证了整个序列的一致性，无需进一步调整。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
a = list(map(int, input().split()))

pairs = []
for i in range(n):
    pairs.append((a[i], i + 1))

pairs.sort()

res = [str(idx) for _, idx in pairs]
print(" ".join(res))
```实施的核心思想是构建`(value, index)`对。 该索引转换为从 1 开始，因为输出预计问题编号从 1 开始。 

Python 中的内置排序按字典顺序比较元组，这意味着它会自动比较第一个元素，并且只有当它们相等时才会比较第二个元素。 这直接编码了所需的平局打破规则，无需额外的逻辑。 

最后的循环仅按顺序提取索引，将它们转换为字符串以进行快速连接。 使用`sys.stdin.readline`确保输入处理高效，尽管对于 n 高达 1000 来说这并不是绝对必要的。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n = int(input().strip())
    a = list(map(int, input().split()))
    pairs = [(a[i], i + 1) for i in range(n)]
    pairs.sort()
    return " ".join(str(idx) for _, idx in pairs)

# provided sample style case
assert run("5\n3 1 1 4 2\n") == "2 3 5 1 4"

# all equal

# already sorted

# reverse order
assert run("4\n4 3 2 1\n") == "4 3 2 1"

# minimum size
assert run("1\n7\n") == "1"

 ## 边缘情况

 当所有困难都相同时，例如输入`5 5 5 5`，该算法在所有对中构造相同的第一个元素。 然后排序完全依靠索引，保持自然顺序。 输出变为`1 2 3 4 5`，符合按原位置解决关系的要求。 

当数组严格递增时，比如`1 2 3 4`，排序根本不会改变对的顺序。 该算法仍然运行相同的步骤，但输出按顺序与输入索引相同。 

当数组严格递减时，例如`4 3 2 1`，排序完全颠倒了结构。 每个元素根据值移动到正确的位置，并且由于值是不同的，因此索引不会影响排序。
