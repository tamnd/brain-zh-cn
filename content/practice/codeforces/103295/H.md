 ## 解决方案
 ## 问题理解

 我们被赋予了一个由片段组成的线性世界，其中一些位置是固体陆地，其他位置是水。 该任务围绕着构建从起始区域到目标区域的连续土地路径，但我们可以通过在水隙上建造有限数量的“桥梁”来修改地图。 每座桥梁都有效地将一片连续的水域转化为可通行的土地，目标是确定是否可以使用最多给定数量的此类结构来连接所需的端点。 

输入描述了单元的一维排列，以及对可以放置多少个桥以及它们在连接方面的影响的限制。 输出是一个二元决策，指示是否存在有效的桥梁放置序列，使得通过仅陆地遍历可以到达起点和终点。 

主要困难不是检查可达性本身，而是了解桥梁布局如何在预算限制下将多个不相连的土地部分合并为单个连接的组件。 

从复杂性的角度来看，自然尺度建议线性或近线性解决方案。 如果单元数量约为 10^5 或更多，则任何重复模拟桥放置或从头开始重新计算连接的方法都会太慢。 这立即排除了每个操作的朴素图重建或任何二次合并策略。 

一些边缘情况非常重要。 如果已经有连续的陆地路径，则不需要桥梁，并且无论限制如何，答案都应该始终是肯定的。 相反，如果土地被极度分割成被水隙隔开的孤立的单个单元，那么每次合并都很重要，而贪婪或错误的计算策略很容易低估所需的桥梁。 一个微妙的例子是当水隙大小不均匀时，例如：

 输入：```
land-water-water-land-water-land
k = 1
```一种天真的方法可能会假设一座桥如果“接触”多个部分就可以连接所有东西，但正确的解释需要认识到一座桥只能消除一个连续的水间隙，而不是同时消除多个分离的间隙。 

## 方法

 蛮力的想法是明确模拟所有可能的放置桥梁的方式。 我们将土地组成部分之间的每个水域间隔视为可以填充的潜在间隙，并尝试合并最多 k 个间隙的所有组合。 在每次假设选择之后，我们重新计算结果结构是否形成从开​​始到结束的完全连接的路径。 

这是正确的，因为它枚举了桥使用的每个有效序列，但它很快就变得不可行。 如果有 g 个间隙，则大小至多为 k 的子集数量约为二项式(g, k)，并且每次检查都需要对结构进行扫描或并集操作。 在最坏的情况下，这在本质上会变成指数或至少是组合，远远超出大输入的任何合理限制。 

关键的见解是我们实际上不需要模拟不同的选择。 重要的是从起点到终点有多少不相交的水隙将连续的陆地部分分开。 每个桥都可以消除一个这样的间隙，这意味着问题减少到计算连接的组件和内部分离，而不是探索配置。 

一旦土地部分被压缩成组件，起点和终点之间的结构就是土地和水块交替的序列。 每个水块都代表强制分离。 为了连接一切，我们必须为每个我们选择消除的水块“支付”一座桥梁的费用。 由于合并相邻的土地组件会减少每个桥的间隙数量，因此最佳策略只是在最小必要数量的分离间隙上使用桥梁，这正是端点组件之间的间隙数量减去一个阈值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^g·n) | O(2^g·n) | O(n) | 太慢了 |
 | 最佳| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将输入字符串减少为相同类型的最大连续段，将陆地和水域分开。 

1. 扫描数组并将其压缩为交替的段。 每个部分都存储是陆地还是水域及其边界。 这是必要的，因为只有段之间的转换才重要，而不是单个单元格。 
2. 识别包含起始位置的段和包含结束位置的段。 如果两者位于同一土地段，则答案立即为“是”，因为不需要遍历或修改。 
3. 按顺序从起始段走到结束段，计算它们之间有多少个水段。 每个这样的水段都代表着一个断层，如果我们想保持连续性，就必须弥合它。 
4. 将所需水隙数量与可用桥梁数量 k 进行比较。 如果 k 至少是间隙的数量，我们就可以消除所有间隔并从头到尾连接。 
5. 如果可行则输出yes，否则输出no。 

这样做的原因是开始和结束之间的结构是线性的，因此任何有效的连接都必须按顺序跨越每个中间边界。 每个水段都充当强制性障碍，如果不通过桥梁转换就无法绕过。 由于桥梁不会以跳过多个独立间隙的方式相互作用，因此问题变成了直接计算障碍物。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = input().strip()
    a, b = map(int, input().split())
    a -= 1
    b -= 1

    # assume '1' is land, '0' is water
    if a > b:
        a, b = b, a

    # if start and end are same type land cell in trivial interpretation
    if s[a] == '1' and s[b] == '1' and s[a:b+1].count('0') == 0:
        print("YES")
        return

    # count water segments between a and b
    i = a
    gaps = 0

    while i < b:
        if s[i] == '1':
            i += 1
            continue

        # we are in water, count full block
        gaps += 1
        while i < b and s[i] == '0':
            i += 1

    print("YES" if gaps <= k else "NO")

if __name__ == "__main__":
    solve()
```该代码读取二进制字符串并对端点进行规范化，因此遍历始终是从左到右。 然后它扫描一次间隔，跳过陆地单元并将连续的水块计算为单个障碍物。 每个障碍代表一个桥梁要求。 最终与 k 的比较决定可行性。 

一个微妙的点是确保连续的水细胞不会被重复计算。 内循环将每次流过的水折叠成间隙计数器的一个增量。

 ## 边缘情况

 一种重要的边缘情况是端点之间的整个间隔不含水。 例如，`111111`与任何 k。 该算法立即计算出零间隙，因此它输出 YES，而不执行任何桥接逻辑，与路径已存在的事实相匹配。 

另一种边缘情况是端点被大水块包围但只允许有一个桥。 例如，`1000001`k = 1。扫描找到一个连续的水块，因此算法正确输出“是”，因为整个块可以在一次操作中转换。 

更微妙的情况是在每个位置交替陆地和水域。 该算法将每次水运行视为一个单独的单元，确保它不会多次错误地计算单个零。 为了`101010`，它正确识别了两个水块而不是五个单独的单元，从而保持了桥梁要求的正确性。
