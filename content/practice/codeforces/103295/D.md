 ## 解决方案
 ## 问题理解

 我们有一个代表玉米地的网格，其中每个单元格要么被玉米遮挡，要么是空的。 任务是计算可以形成多少个不同的三角形，使得三角形的所有三个顶点都位于包含玉米的单元格上。 

三角形有严格的几何结构：它必须是直角三角形，并且它的两条边必须与网格轴对齐。 这意味着三角形的边是网格上的水平和垂直部分，并且在选定的玉米单元之一处形成直角。 因此，通过选择满足该轴对齐直角三角形条件的三个玉米单元来完全确定每个有效三角形。 

网格大小最大为 500 x 500，这意味着最多 250,000 个单元格。 对所有三元组进行三次枚举是完全不可行的。 即使是对所有玉米细胞对进行二次方法，在最坏的情况下也需要进行 10^10 次检查，这远远超出了时间限制。 这立即表明该解决方案必须避免显式枚举点对，而是依赖于沿行和列的计数结构。 

当玉米密度非常高或非常低时，会出现微妙的边缘情况。 如果网格全部为空，则答案几乎为零，并且任何假设每行或每列至少有一个有效顶点的计数逻辑都必须干净地处理这一问题。 在另一个极端，如果网格充满了玉米，天真的组合计数很容易多次对三角形进行计数，除非每个三角形都与一个唯一的“锚”单元相关联，从而防止重复。 

## 方法

 蛮力的想法很简单。 我们尝试每三个玉米细胞，并检查它们是否形成直角三角形，其腿与网格轴对齐。 这需要检查三个点的所有组合，即 O(K^3)，其中 K 是玉米细胞的数量。 在密集网格中，K 可以是 250,000，使得这种方法甚至无法在约束内概念化。 即使限制为对并导出第三点也没有多大帮助，因为验证存在性仍然需要昂贵的查找而不进行预处理。 

关键的观察是每个有效的三角形都可以通过其直角顶点唯一地描述。 一旦我们将一个玉米单元固定为直角角，其他两个顶点必须分别位于同一行和同一列。 这意味着我们不需要轴对齐之外的几何图形； 我们只需要每行和每列的玉米细胞计数。 

假设在一个固定的单元格中，我们知道其行中有多少个玉米单元格，其列中有多少个玉米单元格。 如果我们选择同一行和同一列中的另一个玉米单元格，我们会以当前单元格为直角形成一个有效的直角三角形。 此类选择的数量是这两个计数的乘积，但我们必须小心不要将当前单元格本身包含在这些计数中。 所以我们在相乘之前从每个方向减一。 

这减少了从推理整个网格中的点对到维护两个频率阵列的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解三倍以上 | O(K^3) | O(K^3) | O(1) | O(1) | 太慢了 |
 | 每行/列计数 | O(纳米) | O(n + m) | 已接受 |

 ## 算法演练

1. 读取网格并识别所有玉米细胞。 执行此操作时，维护两个数组，一个计算每行中出现的玉米细胞数量，一个计算每列中出现的玉米细胞数量。 此预处理将所有几何信息压缩为简单的频率计数。 
2. 再次迭代网格中的每个单元格。 当我们遇到玉米细胞时，将其视为三角形的潜在直角顶点。 
3. 对于位置 (i, j) 处的每个此类单元，计算同一行上的第二个顶点存在多少个有效选择。 这是 row_count[i] 减一，因为我们排除了当前单元格。 
4. 同样计算同一列上第二条腿存在多少个有效选择。 这是 col_count[j] 减一。 
5. 将这两个值相乘。 该乘积计算所有在 (i, j) 处具有直角的三角形。 
6. 将网格中所有玉米单元格的值相加。 

乘法步骤是正确的，因为在行中选择一个不同的玉米单元固定三角形的一条边，在列中选择一个不同的玉米单元独立地固定另一条边。 

### 为什么它有效

 每个有效三角形都恰好有一个直角所在的单元格。 该单元是唯一定义的，因为它是唯一与其他两个顶点共享水平和垂直边的顶点。 通过将计数锚定在该顶点，每个三角形都会被精确计数一次。 行选择和列选择之间的独立性保证了所有组合都对应于不同的几何三角形，并且不会引入无效形状，因为轴对齐会自动强制垂直。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    row = [0] * n
    col = [0] * m

    # count stars
    for i in range(n):
        for j in range(m):
            if grid[i][j] == '*':
                row[i] += 1
                col[j] += 1

    ans = 0

    for i in range(n):
        for j in range(m):
            if grid[i][j] == '*':
                ans += (row[i] - 1) * (col[j] - 1)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现将预处理与计数分开，这避免了重复重新计算行或列统计信息。 两个方向减一是必不可少的，因为当前单元格同时计入行和列总计中。 

在 Python 中，最终的累加循环在溢出方面是安全的，但在像 C++ 这样的强类型语言中，由于密集网格中的二次增长，结果可能需要 64 位整数。

 ## 边缘情况

 对于包含星形的单单元格网格，该算法会正确返回零，因为行数或列数变为 1，从而使两个因子在相减后都为零。 这可以防止退化几何中的假三角形。 

在完全填充的网格中，每个单元格充当直角顶点。 该算法正确地计算了所有组合，因为每个三角形都唯一锚定在其角上，并且行和列选择的乘法捕获所有有效对而不会重复。 

在星星被隔离的非常稀疏的网格中，行或列计数保持为一，确保不会形成无效的三角形，因为无法构造三角形的一条腿。
