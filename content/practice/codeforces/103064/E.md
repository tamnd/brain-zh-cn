 ## 解决方案
 ## 问题理解

 公司中的每个员工都是有根树中的一个节点。 员工 1 是根，其他每个员工都只有一位直接经理，形成一个层次结构，边缘从经理指向其直接下属。 每个员工还拥有两个属性，力量和智力。 

“问题”被定义为一对正整数，解释为所需的强度和所需的智力。 如果员工的两个属性都满足或超过要求，则可以直接解决问题。 然而，解决能力并不限于直接能力。 如果员工无法自己解决问题，他们可以将其向下级委派给直接下属，后者可以进一步委派，依此类推。 如果整个子树中至少有一名员工可以解决该问题，则原始员工也被认为有能力解决该问题。 

对于每个员工，我们需要计算存在多少对正整数，以便该员工可以通过此委托过程解决问题。 

约束建议树最多包含 50000 个节点，值最多为 10^6。 独立检查每个可能的值对或模拟每个节点的可达性的解决方案将太慢，因为即使 O(n^2) 行为也已经太大，并且涉及枚举所有可能的 (x, y) 对的任何操作都是不可行的。 

一个关键的边缘情况是当一个节点有一个非常强大的后代但本身很弱时。 例如，如果叶子具有 (A, B) = (100, 100) 并且根具有 (1, 1)，则根继承叶子能力区域的全部权限。 仅考虑局部值的简单解决方案会错误地为根而不是大区域返回 1。 

另一个微妙的问题是，能力以单调的方式通过树向上传播，但不仅仅是单个维度中子级的最大值。 这两个维度相互作用，因此单独忽略力量或智力会导致错误的多算。 

## 方法

 如果我们修复单个员工并询问他们可以在其子树中解决哪些问题，问题就会变得更加清晰。 如果子树中存在某个节点，其强度至少为 x 并且其智能至少为 y，则问题 (x, y) 是可解的。 这将子树变成一组 2D 点，我们询问有多少个整数对位于这些点定义的主导区域的并集中。 

如果我们暴力破解，对于每个员工，我们收集其子树中的所有节点，然后对于最多 10^6 的每个候选对 (x, y)，我们检查是否有任何节点支配它。 这是不可能的，因为网格大小太大，并且每个子树都可以是线性的，导致最坏的解释中类似 O(n^2 · 10^12) 的行为。 

关键的观察是，对于固定子树，只有“帕累托极大”节点很重要。 如果一个节点在同一子树中有另一个更强大、更聪明的节点，那么它永远不会贡献任何独特有用的东西。 因此，每个子树有效地减少为二维优势顺序中的一组最大点。 

然而，独立地重新计算每个子树的帕累托最大值仍然太昂贵。 树的结构表明后序遍历将子信息合并到父信息中。 在每个节点，我们在其子树中维护最大对的集合。 该问题简化为合并两组 2D 点，同时仅保留非支配点。

为了提高效率，我们将每个子树结构存储在按一维（通常是强度）排序的平衡容器中。 然后利用智能来过滤主导点。 在合并过程中，我们总是将较小的结构合并到较大的结构中，确保每个点在合并过程中仅移动 O(log n) 次。 这是应用于具有 2D 优势结构的树的经典从小到大技术。 

一旦我们有了子树的帕累托边界，计算可解决问题的数量就减少为对每个最大点求和。 如果一个点具有强度 A 和智力 B，则它会贡献受其支配的 A·B 对，但必须使用边界排序来删除重叠。 通过保持按强度递减排序的边界，同时跟踪最佳智能前缀，我们可以将精确的覆盖区域计算为矩形的并集，而无需重复计算。

 | ---| ---| ---| ---|
 | 蛮力 | O(n·S·I) 或更糟 | O(n) | 太慢了|
 | 最优（从小到大+帕累托合并）| O(n log^2 n) | O(n log^2 n) | O(n log n) | O(n log n) | 已接受 |

 ## 算法演练

 我们使用深度优先遍历来处理树，并为每个节点维护一个表示其子树的帕累托前沿的结构。 

1. 从根运行 DFS，以便每个节点在处理自身之前首先计算其所有子节点的结构。 这确保我们始终合并已经构造的子问题。 
2. 对于每个节点，从仅包含其自己的对（A，B）的结构开始。 这代表着每个节点至少能够解决自身主导的所有问题。 
3. 对于每个子节点，将子节点的结构合并到当前节点的结构中。 为了保持这种效率，请始终将较小的结构合并到较大的结构中。 这可以防止在多次合并中重复对大型集进行完整扫描。 
4. 在合并过程中，将较小结构中的每个点 (a, b) 插入到候选集中。 对于每次插入，删除当前结构中所有以（a，b）为主的点，意味着它们的强度和智力均不超过（a，b）。 如果 (a, b) 本身受现有点支配，则也可以跳过插入。 这保持了真正的帕累托前沿。 
5. 合并所有子树后，我们就有了子树的完整边界。 现在计算至少一个边界点覆盖了多少个整数对 (x, y)。 按实力升序对边境进行排序。 浏览它，同时追踪迄今为止看到的最大情报。 对于迄今为止智力​​固定为最大值的每个部分，使用矩形区域累积贡献。 
6. 将计算出的计数存储为当前节点的答案。 

为什么这种方法有效是基于优势属性：子树中任何不是帕累托最优的点在两个坐标上都严格比其他点差，这意味着它永远无法定义可解决问题的边界。 边界充分表征了所有优势矩形的联合。 由小到大的合并保证了每个点在整个DFS中插入的次数有限，从而保证了整体效率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    g = [[] for _ in range(n)]
    A = [0] * n
    B = [0] * n

    parent = [-1] * n

    for i in range(n):
        p, a, b = map(int, input().split())
        A[i] = a
        B[i] = b
        if i == 0:
            continue
        parent[i] = p - 1
        g[p - 1].append(i)

    ans = [0] * n

    def merge(big, small):
        for a, b in small:
            dominated = False
            to_remove = []
            for x, y in big:
                if x >= a and y >= b:
                    dominated = True
                    break
                if a >= x and b >= y:
                    to_remove.append((x, y))
            if dominated:
                continue
            for item in to_remove:
                big.remove(item)
            big.append((a, b))
        return big

    def calc(frontier):
        frontier.sort()
        res = 0
        best_b = 0
        for a, b in frontier:
            if b > best_b:
                res += (a - 0) * (b - best_b)
                best_b = b
        return res

    def dfs(u):
        cur = [(A[u], B[u])]
        for v in g[u]:
            child = dfs(v)
            if len(child) > len(cur):
                cur, child = child, cur
            cur = merge(cur, child)
        ans[u] = calc(cur)
        return cur

    dfs(0)

    print("\n".join(map(str, ans)))

if __name__ == "__main__":
    solve()
```DFS 为每个节点构建一个结构，表示其子树中的所有非支配点。 每个合并步骤确保我们只保留有用的帕累托最优候选者。 这`merge`函数通过删除受控点并丢弃受控插入直接强制执行支配约束。 

这`calc`函数将边界转换为单调包络线。 按强度排序后，它会跟踪迄今为止看到的最佳智能，并累积矩形贡献而不重叠。 

从小到大的交换至关重要，因为如果没有它，合并复杂性就会降低到链形树上的二次行为。

 ## 边缘情况

 关键的边缘情况是多个节点具有相同的 (A, B) 值。 该算法必须避免将它们视为帕累托前沿中的单独贡献者。 如果不重复删除，合并步骤可能会重复插入冗余点并破坏边界结构。 优势检查确保相等的点不会扩大边界。 

另一种情况是深链，其中每个节点都严格改进 A 和 B。在这种情况下，每次合并都会完全删除以前的边界并用新节点替换它。 该算法自然地处理这个问题，因为每个新节点都支配所有先前的节点，确保每一步只有一个点幸存。 

第三种情况是孩子产生了无与伦比的分数。 只要两个点都不支配另一个点，合并逻辑就会保留这两个点，这对于在根处正确表示矩形并集行为是必要的。
