 ## 解决方案
 ## 问题理解

 我们正在使用一个由整数除法构建的函数，应用了两次。 对于固定数量$N$, 每个候选值$X$在范围内$[1, N]$通过第一次计算测试$\lfloor N / X \rfloor$，然后使用该结果作为除数再次应用相同的运算。 一个值$X$如果这个两步过程准确返回，则被认为是有效的$X$再次。 

所以结构是对称的：从$X$，将其映射到一个商，然后使用该商映射回来，并检查我们是否返回到原始值。 任务是计算有多少个整数$[1, N]$对于每个给定满足这个定点条件$N$，最多有 10 个查询和值$N$一样大$10^{18}$。 

蛮力方法将测试每一个$X$，计算两个楼层划分，并比较结果。 那是$O(N)$每个查询，这是不可能的$N$达到$10^{18}$。 甚至$N = 10^9$已经远远超出了可行的限度。 

主要的边缘情况来自这样一个事实：楼层划分会产生较大的恒定值间隔。 例如，如果$N = 10$， 然后$\lfloor 10 / X \rfloor$对于像这样的范围是恒定的$X = 1$给 10,$X = 2$到$3$给予 5，依此类推。 幼稚的按值检查会多次重新计算相同的商，从而丢失此结构。 

解决问题的关键是停止个人思维$X$值，而是使用商保持恒定的范围。 

## 方法

 暴力法直接检查每个整数$X$，计算$k = \lfloor N / X \rfloor$，然后计算$\lfloor N / k \rfloor$并将其与$X$。 根据定义，这是正确的，但每个查询都需要线性工作，这会完全失败$N$。 

结构观察是函数$\lfloor N / X \rfloor$是分段常数。 所有值$X$产生相同的商形成一个连续的线段。 我们可以迭代这些片段，而不是迭代单个值。 

在一个段内，其中$k = \lfloor N / X \rfloor$是固定的，每一个$X$共享相同的第一个转换。 第二个变换变成$\lfloor N / k \rfloor$，对于该段来说也是恒定的。 这意味着我们只需要测试每个段的单个候选值：$X' = \lfloor N / k \rfloor$。 如果该候选者位于产生的段内$k$，那么它就是一个有效的解。 

这减少了迭代所有整数的问题$N$迭代$O(\sqrt{N})$商段。

 | ---| ---| ---| ---|
 | 蛮力 |$O(N)$每个查询 |$O(1)$| 太慢了|
 | 最佳（基于细分）|$O(\sqrt{N})$每个查询 |$O(1)$| 已接受 |

 ## 算法演练

 我们处理每个值$N$独立。 

1. 从$X = 1$，并使用标准的楼层划分分组技术来识别最大间隔$X$在哪里$k = \lfloor N / X \rfloor$保持不变。 这是有效的，因为商仅在以下情况下改变：$X$跨越除数边界$N$。 
2. 对于起始于$X = l$, 计算$k = \lfloor N / l \rfloor$。 这就是每个人第一次转型的价值$X$在这一段中。 
3.确定正确的边界$r$该段作为最大值，使得所有$X \in [l, r]$满足$\lfloor N / X \rfloor = k$。 
4.对于这个固定$k$，计算第二个变换值$X' = \lfloor N / k \rfloor$。 
5. 检查是否$X'$位于当前段内$[l, r]$。 如果是的话，那么这个$X'$是有效的并且对答案的贡献恰好为 1。 
6. 通过设置移动到下一段$l = r + 1$并重复直到$l > N$。 

### 为什么它有效

 在任何段内$[l, r]$，值$\lfloor N / X \rfloor$对于所有人来说都是相同的$X$。 这意味着第一次转换完全由段索引决定。 一旦我们修好$k$，第二个变换变得独立于$X$并产生单个候选值$X'$。 两步映射可以返回到原始值的唯一方法是该候选值是否位于生成相同值的同一区域中$k$，保证两个楼层划分方向的一致性。 由于每个有效$X$由其段和候选检查唯一确定，不计算重复项。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(n: int) -> int:
    ans = 0
    x = 1

    while x <= n:
        k = n // x
        r = n // k

        x_candidate = n // k
        if x <= x_candidate <= r:
            ans += 1

        x = r + 1

    return ans

def main():
    q = int(input())
    for _ in range(q):
        n = int(input())
        print(solve_one(n))

if __name__ == "__main__":
    main()
```该实现直接反映了基于段的推理。 循环变量`x`表示每个商段的左边界。 价值`k = n // x`定义线段上的常数商，并且`r = n // k`计算该段的完整范围。 

关键的微妙之处在于我们不会迭代个体$X$，仅超过段边界。 对于每个细分，我们只计算一个候选者$X = \lfloor N / k \rfloor$，并检查它是否位于段内。 这保证了正确性，同时避免了多余的工作。

 ## 边缘情况

 对于$N = 1$，唯一的值是$X = 1$，并且两个除法都立即返回 1，因此答案为 1。该算法处理此问题是因为第一个段是$[1,1]$和$k = 1$，候选者也是 1，位于段内。 

对于小$N$像 2 或 3 一样，段很短并且大多是单例。 每个段正确地贡献了一个有效值，因为候选者$X = \lfloor N / k \rfloor$始终与段边界匹配。 

对于大型$N$，段数变小，因为$\lfloor N / X \rfloor$迅速减少。 该算法从不迭代个体$X$，所以即使$N = 10^{18}$仍处于可控状态。 

在所有情况下，正确性源于每个段都有唯一的商这一事实$k$，以及任何有效的$X$必须是由该商导出的唯一不动点。
