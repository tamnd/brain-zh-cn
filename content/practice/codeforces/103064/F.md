 ## 解决方案
 ## 问题理解

 我们得到了一组候选朋友组的集合，其中每个组都有已知的大小。 另外，我们给出了几个“倒数”程序，其中每个程序都由多个步骤定义，并且选择是在一圈人中完成的。 

对于任何选定的团体，想象一下将其成员从组织者开始围成一个圆圈。 然后我们绕圈每人重复数一个单词，直到数完为止。 计票结束的人被选为展示项目的人。 组织者也始终是圈子的一部分，因此实际上圈子的大小取决于团体和组织者。 

每个计数数字的关键问题是确定最小索引组，以便在使用该组时组织者不是选定的发言者。 如果不存在这样的群，我们输出−1。 

重要的隐藏结构是，对于大小为A的组，最终选择的位置仅取决于计数模A的余数加上来自起始点的固定偏移量。 当计数回到组织者身上时，组织者就“坏”了，这对应于相对于组大小的计数长度的特定模块化条件。 

约束非常大，最多有 10^5 个组和最多 10^6 个查询，因此不可能对组进行任何每次查询扫描。 即使是对组进行排序加上二分搜索也需要仔细构建，因为每个查询都必须独立且快速地得到回答。 这促使我们采取一种预处理策略，将每个组减少到一个简单的“坏值”阈值。 

一种简单的方法是为每对组和查询模拟循环，这将需要 O(NQ) 操作，在最坏的情况下最多需要 10^11 步，这远远超出了可行性。 

一个不太天真的想法是使用 O(1) 中的模算术计算每对在圆中的最终位置，但仍然迭代每个查询的所有组仍然太慢。 

关键的观察是，对于固定的组大小 A，当且仅当计数长度 B 满足简单的基于整除性的条件时，才会选择组织者：实际上，B mod A 等于 0（或者等效地，B 是 A 的倍数，具体取决于索引约定）。 因此，每个组都会贡献一组“坏 B 值”，并且对于每个查询 B，我们需要第一个大小不能整除 B 的组。 

边缘情况来自小团体和小 B。当 A = 1 时，无论 B 如何，组织者总是被选择，因为圈子退化了。 这意味着第一组永远都不好。 另一个边缘情况是所有组大小都除 B 时，在这种情况下答案为 -1。 

## 方法

 暴力解决方案很简单：对于每个查询 B，按升序迭代所有组，并检查 B 是否满足该组大小的“坏”条件。 第一个不导致组织者被选中的组就是答案。 这是可行的，因为我们直接测试每对的条件定义，但它需要每个查询检查最多 N 个组，从而导致 O(NQ) 复杂性。 

由于 N 和 Q 可能同时很大，因此出现了瓶颈。 关键的结构见解是，每个组仅取决于其大小，并且条件仅取决于 B 和 A 之间的整除性。这将问题转化为经典的“在数组中找到第一个 A，使得 A 不能整除 B”。

我们可以按排序顺序预处理组大小，并利用可整性在倍数上单调表现的事实，而不是检查每个查询的所有组。 然而，由于排序 A 中的整除性不是单调的，因此我们相反：我们维护组大小的频率并使用 B 的预先计算的除数。对于每个查询 B，我们枚举 B 的除数并将所有除 B 的组大小标记为无效； 那么答案是大小不在该除数集中的最小索引。 

由于 B ≤ 10^7，枚举除数的成本约为 O(√B)，并且所有查询的总和保持在限制范围内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(NQ) | O(1) | O(1) | 太慢了|
 | 基于除数的预处理 | O(Q√B + N) | O(N + 最大 A) | 已接受 |

 ## 算法演练

 1. 读取所有组大小并构建从大小到出现的最小索引的映射。 这很重要，因为我们总是想要最早的有效组。 
2. 对于每个查询值 B，通过迭代直至 √B 有效计算 B 的所有除数。 对于每个除数 d，d 和 B/d 都是有效除数。 
3. 将每个除数 d 视为可以选择组织者的组大小，因此这些组大小对于此查询来说是“坏”的。 
4. 在输入的所有组大小中，我们想要最小的索引，其大小对于该 B 来说没有被标记为坏。为了有效地做到这一点，我们维护候选组大小的全局结构并检查坏集中的成员资格。 
5.如果每个组大小都出现在坏集合中，则输出-1。 

关键的计算技巧是除数枚举将所有组的完整扫描压缩为从算术结构派生的小得多的集合。 

### 为什么它有效

 对于大小为 A 的组，当计数长度 B 与 A 引起的周期长度一致时，组织者就会被准确选择，从而简化为可除条件。 因此，所有失败的组正是那些大小整除 B 的组。每个有效组都必须避免这种情况，并且由于我们返回最小索引有效组，因此检查除数集中的成员资格足以保证正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def divisors(x):
    small = []
    large = []
    i = 1
    while i * i <= x:
        if x % i == 0:
            small.append(i)
            if i * i != x:
                large.append(x // i)
        i += 1
    return small + large

def main():
    n = int(input())
    a = list(map(int, input().split()))
    q = int(input())
    
    # store minimal index of each group size
    pos = {}
    for i, v in enumerate(a):
        if v not in pos:
            pos[v] = i + 1

    # all unique group sizes sorted by index
    groups = sorted(pos.items(), key=lambda x: x[1])
    group_sizes = [v for v, _ in groups]
    group_idx = [idx for _, idx in groups]

    for _ in range(q):
        b = int(input())
        bad = set(divisors(b))

        ans = float('inf')

        # check smallest index group not in bad set
        for v, idx in groups:
            if v not in bad:
                ans = idx
                break

        print(-1 if ans == float('inf') else ans)

if __name__ == "__main__":
    main()
```该代码首先将组大小压缩到其最早出现的索引中，因为后来的重复项无法改进答案。 然后，对于每个查询，它计算 B 的所有除数并将它们存储在哈希集中以进行 O(1) 成员资格检查。 

组上的循环按索引排序，因此遇到的第一个有效大小立即给出答案。 微妙的一点是确保我们只保留每个大小的最早的索引； 否则重复会扭曲“最低索引”的要求。

 ## 边缘情况

 对于 B 等于 1 的情况，每个组大小都将其除以，因此所有组都无效，输出必须为 -1。 除数枚举正确生成 {1}，因此只有大小 1 被标记为错误，但由于所有有效答案都需要不可分割的大小，因此如果仅存在大小 1 的组，则不存在任何答案。 

当所有组大小均为 1 时，每个查询都会立即生成 -1，因为 1 整除所有 B。该算法会处理此问题，因为坏集始终包含 1。 

当 B 为素数时，仅存在约数 1 和 B。 这使得除了等于 1 或 B 的组大小之外的几乎所有组大小都有效，并且通过按索引顺序扫描可以正确选择大小与这些不同的最小索引组。
