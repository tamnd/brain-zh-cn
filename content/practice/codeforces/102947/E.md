 ## 解决方案
 ## 问题理解

 我们给定一个大小为的方阵$n \times n$，其中每个条目描述了如果分配给特定食物类型，特定幸存者会贡献多少价值。 每个幸存者必须准确地分配给一种食物类型，并且每种食物类型必须准确地接收一个幸存者，因此分配是行和列之间的完美匹配。 

目标是选择这种一对一的分配，以使所选矩阵值的总和尽可能大。 

尽管该声明谈论了幸存者和食物类型，但该结构纯粹是一个分配问题：从每一行和每一列中精确选择一个元素，使总和最大化。 

约束条件$n \le 10$是关键信号。 一般分配问题$n$最多 500 个需要匈牙利算法或最小成本最大流技术，但这里的搜索空间足够小，可以枚举所有排列。 

唯一有意义的边缘情况来自于理解每行必须恰好使用一次并且每列必须恰好使用一次。 天真的贪婪方法会立即失败，因为局部最佳选择可能会阻碍全局最佳配对。 

例如，考虑：```
n = 3
matrix:
10 1 1
1 10 1
1 1 10
```每行或每列的贪婪算法在这里有效，但稍作修改：```
n = 3
matrix:
10 9 1
9 10 1
1 1 100
```独立选择每行中的最佳条目可能会将最后一行陷入低值列，从而失去最佳配对结构。 

因此，问题本质上是关于排列而不是独立选择。 

## 方法

 蛮力的想法是尝试将幸存者分配给食物类型的所有可能的分配。 由于每个分配对应于列的排列$n$行，我们可以枚举所有$n!$排列并计算每个排列的总和。 

这是正确的，因为每个有效的赋值都恰好对应一个排列，并且每个排列都代表一个有效的赋值。 

然而，成本成倍增长。 即使是为了$n = 10$，我们得到$10! = 3,628,800$排列。 每次评估费用$O(n)$，大致给出$3.6 \times 10^7$操作，这是边界，但只有在仔细实现的情况下，在 Python 中仍然可以接受。 该结构表明，与尝试大量优化暴力相比，我们可以在清晰度上做得更好。 

关键的观察是不存在单调性或贪婪可行性等附加结构，因此最简单的正确解决方案是使用回溯或的纯排列枚举`itertools.permutations`。 

这将问题简化为对列顺序进行彻底的彻底搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n! \cdot n)$|$O(n)$| 接受为$n \le 10$|
 | DP / 匈牙利语（这里太过分了）|$O(n^3)$|$O(n^2)$| 已接受但没有必要 |

 ## 算法演练

 1.读取尺寸矩阵$n \times n$，其中条目$(i, j)$是分配幸存者的值$i$食物类型$j$。 这完全定义了任何任务的成本。 
2. 生成数字的所有排列$0$到$n-1$。 每个排列代表一个选择，其中行$i$被分配给列`perm[i]`。 这保证了每列仅使用一次。 
3. 对于每个排列，通过迭代所有行并添加来计算总和`matrix[i][perm[i]]`。 这评估了该任务的价值。 
4. 跟踪所有排列中看到的最大总和。 这代表了最好的分配。 
5. 检查所有排列后输出最大值。 

唯一微妙的一点是确保我们将排列正确解释为列分配。 行索引和列索引之间的任何不匹配都会破坏正确性。 

### 为什么它有效

 幸存者对食物类型的每个有效分配都对应于行和列之间的双射，并且每个双射恰好是列索引的排列。 由于我们评估所有排列，因此不会错过任何有效的分配。 因为我们精确计算每个排列的总和，所以找到的最大值就是最优解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from itertools import permutations

n = int(input())
a = [list(map(int, input().split())) for _ in range(n)]

best = 0

for perm in permutations(range(n)):
    total = 0
    for i in range(n):
        total += a[i][perm[i]]
    if total > best:
        best = total

print(best)
```该解决方案直接反映了赋值解释。 矩阵按原样存储，每个排列被视为每行的列选择。 运行最大值保持所见的最佳分配。 

这里的一个常见错误是尝试贪婪的逐行选择，但这种选择会失败，因为早期的选择限制了未来列的可用性。 排列方法通过从一开始就强制执行全局一致性来避免这种情况。

 ### 示例 1

 输入：```
2

0 2
```我们枚举列的排列$[0,1]$。 

| 排列| 第 0 行选择 | 第 1 行选择 | 总计 |
 | ---| ---| ---| ---|
 | (0, 1) | (0, 1) | 3 | 2 | 5 |

 最好是 5，通过将第 0 行分配给第 0 列，将第 1 行分配给第 1 列来实现。 

这显示了不同的排列如何对应于不同的有效匹配。

 输入：```

## 测试用例```python
import sys, io
from itertools import permutations

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(input())
    a = [list(map(int, input().split())) for _ in range(n)]
    best = 0
    for p in permutations(range(n)):
        total = 0
        for i in range(n):
            total += a[i][p[i]]
        best = max(best, total)
    return str(best)

# provided sample
assert run("2\n3 4\n0 2\n") == "5", "sample 1"

# custom: all equal values
assert run("2\n1 1\n1 1\n") == "2", "uniform matrix"

# custom: diagonal dominant
assert run("3\n10 1 1\n1 10 1\n1 1 10\n") == "30", "diagonal best"

# custom: forcing permutation choice
assert run("3\n1 2 3\n3 2 1\n2 3 1\n") == "8", "mixed case"

# custom: minimum size
assert run("1\n7\n") == "7", "single element"
```| 测试输入| 预期产出 | 它验证了什么 |
 | ---| ---| ---|
 | 2x2 均匀矩阵 | 2 | 对称和平等选择|
 | 对角矩阵| 30| 最佳对角线的正确匹配 |
 | 混合 3x3 | 8 | 非平凡的排列选择 |
 | 单元素| 7 | 基本情况处理 |

 ## 边缘情况

 对于单个幸存者，算法立即返回唯一可用的值，因为唯一的排列为空，并且总和减少到单个矩阵条目。 

对于所有条目都相等的均匀矩阵，每个排列都会产生相同的总和。 该算法仍然探索所有排列，但始终跟踪相同的最大值，从而确认选择顺序中不存在偏差。 

对于最优分配需要非贪婪交换的矩阵，例如当高值分散时，排列枚举可以保证正确性。 每个可能的组合都会被明确地评估，因此即使中间选择看起来不是最优的，也不会错过任何分配。
