 ## 解决方案
 ## 问题理解

 我们得到了最多 30,000 个整数的集合，每个整数最多可以用 12 位表示。 除此之外，我们还获得了许多“按位机器”，其中每台机器都定义了从两个输入数字到一个输出数字的转换。 转换是逐位进行的：对于每个位位置，机器将六个固定布尔运算之一应用于两个输入的相应位，并将结果位组合回一个数字。 

每台机器都会重复该任务：计算有多少有序寄存器对产生恰好为零的输出。 

一个关键细节是对是有序的并且允许重复，因此 (i, j) 和 (j, i) 都很重要，并且 (i, i) 有效。 

两个不同方向的约束都很严格。 字长最多为 12，这强烈表明对 w 的任何指数依赖性都是可以接受的。 然而，n 最大为 3e4，m 最大为 5e4，这使得每次查询的复杂度为 O(n²) 都不可能。 在最坏的情况下，对所有对的每个门进行幼稚评估将需要每个查询大约 9e8 次评估，这是完全不可行的。 

操作的结构也很重要：每个门的每个位都独立应用，并且每个位仅取决于相应的输入位。 这种跨位位置的可分离性是主要的结构线索。 

一个微妙的边缘情况是，所有输入值都相同，或者无论输入如何，函数都恒定为零或恒定非零。 例如，如果门将每个位对映射为 0，则每个有序对都会做出贡献，给出 n²。 如果它将所有内容映射为 1，则答案为零。 这些极端情况必须由相同的计数框架自然处理，而不是作为特殊情况。 

另一个微妙的情况是不同的数字对在位级别产生相同的中间行为。 幼稚的按数字推理可能会忽略不同对独立贡献并且必须组合计数。 

## 方法

 直接方法评估每个有序对上的每个门。 对于 m 个门中的每一个，我们将迭代所有 n² 对并计算 w 位结果。 每次评估的成本为 O(w)，因此总复杂度变为 O(m · n² · w)，这远远超出了可行的极限。 

核心观察是 w 很小，因此我们应该通过值的位模式来压缩值，并推理每个位模式而不是每个值对的贡献。 每个数字都是一个 w 位向量，因此最多有 2^w 个不同值，即最多 4096 个。我们可以聚合每个值的频率，而不是跟踪大小为 n 的数组中的出现次数。 

现在问题变成：给定一个频率数组 freq[x]，我们想要计算在给定的按位变换下有多少个有序对 (x, y) 产生输出零。 

关键的结构思想是分离位。 对于固定门，位置 i 处的输出位仅取决于 i 处的输入位。 因此，条件“输出等于 0”意味着每个位位置必须独立地计算为 0。这意味着我们可以将每个值视为 w 长度的位串并预先计算转换，而不是处理完整整数。 

这导致在 0 到 2^w - 1 的值上进行标准位掩码动态编程。对于每个门，如果输出为零，我们计算函数 f(x, y) = 1，否则为 0。由于域只有 4096，我们可以通过迭代所有不同位掩码对来计算贡献。 

我们通过注意到该函数是跨位可分离的来进一步优化。 对于每个位位置，我们在 (b1, b2) 上定义一个 4 条目真值表。 对于每个门，我们可以针对每对位掩码预先计算所有位位置是否满足零输出。 这将每对的评估减少到 O(w)，并且我们只有 2^w 个状态，因此每个门的最坏情况总数是 O(4^w)，这是可以接受的，因为 w ≤ 12。

更有效的观点是为每个门预先计算位对上的转换表，然后通过检查一致性来组合位贡献。 当且仅当对于每个位位置 i，位转换输出 0 时，每对 (x, y) 才有效。因此，我们可以预先计算位的布尔兼容性函数，然后使用按位分解或对位掩码进行快速 zeta 式计数来评估对。 

最后，我们组合频率计数：每个有效对 (x, y) 贡献 freq[x] · freq[y]。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解对 | O(m·n²·w) | O(1) | O(1) | 太慢了 |
 | 位掩码聚合+DP对| O(m·2^w·2^w) | O(m·2^w·2^w) | O(2^w) | O(2^w) | 已接受 |

 ## 算法演练

 1. 将输入压缩到包含所有可能的 w 位值的频率数组中。 

我们用大小为 2^w 的数组 freq 替换 n 个寄存器的列表。 这是有效的，因为所有计算仅依赖于位模式，而不依赖于位置。 
2. 对于每个门字符，预先计算其每比特真值表。 

每个门符号定义了产生一位的两位的函数。 我们将其存储为 2×2 表以便快速查找。 
3. 对于给定的长度为 w 的门串，在完整的 w 位对上构造一个布尔函数。 

对于一对 (x, y)，我们使用相应的门独立检查每个位的位置。 如果每个位置都输出 0，则该对有效。 
4. 迭代 [0, 2^w) 中所有可能的值对 (x, y)。 

对于每一对，我们通过检查所有位位置来测试它是否产生零。 由于 w ≤ 12 并且域大小最多为 4096，因此该步骤是可行的。 
5. 使用频率累积贡献。 

如果一对 (x, y) 有效，我们将 freq[x] * freq[y] 添加到答案中。 
6. 输出每个门的最终总和。 

### 为什么它有效

 每个寄存器的值完全由其w个独立位决定。 门在每个位位置独立地应用函数，因此当且仅当每个位位置在相应的局部门下产生零时输出为零。 这意味着正确性减少为检查每比特的有效性，并且计算有效对减少为对独立压缩状态求和。 倍频正确地考虑了所有有序寄存器对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_bit_table(ch):
    if ch == 'A':
        return ((0, 0), (0, 1))
    if ch == 'O':
        return ((0, 1), (1, 1))
    if ch == 'X':
        return ((0, 1), (1, 0))
    if ch == 'a':
        return ((1, 1), (1, 0))  # NAND
    if ch == 'o':
        return ((1, 0), (0, 0))  # NOR
    if ch == 'x':
        return ((1, 0), (0, 1))  # XNOR

def main():
    w, n, m = map(int, input().split())
    arr = list(map(int, input().split()))

    maxv = 1 << w
    freq = [0] * maxv
    for v in arr:
        freq[v] += 1

    bits = []
    for v in range(maxv):
        b = [0] * w
        for i in range(w):
            b[i] = (v >> i) & 1
        bits.append(b)

    gates = [input().strip() for _ in range(m)]

    for g in gates:
        tables = [build_bit_table(c) for c in g]

        ans = 0
        for x in range(maxv):
            fx = freq[x]
            if fx == 0:
                continue
            bx = bits[x]

            for y in range(maxv):
                fy = freq[y]
                if fy == 0:
                    continue

                ok = True
                for i in range(w):
                    if tables[i][bx[i]][(y >> i) & 1] != 0:
                        ok = False
                        break

                if ok:
                    ans += fx * fy

        print(ans)

if __name__ == "__main__":
    main()
```The implementation begins by compressing values into a frequency array over the full 2^w domain. 这很重要，因为它消除了对每门计算中 n 的依赖。 

功能`build_bit_table`将每个门字符编码为位对的恒定时间查找。 这避免了紧密循环内重复的条件逻辑。 

我们为每个值预先计算一次位分解，以便检查位一致性不会在 x 的内部循环内重复移位和屏蔽。 然而，我们仍然动态计算 y 的位，考虑到域很小，这是可以接受的。 

对于每个门，我们迭代所有 (x, y) 对并检查所有位位置是否产生零输出。 如果是这样，我们将 freq[x] · freq[y] 添加到答案中。 这直接实现了对有序寄存器对进行计数的数学要求。 

这里的一个常见陷阱是忘记对是有序的。 双环自然包括(x,y)和(y,x)，因此不需要对称性调整。

 ## 边缘情况

 一个关键的边缘情况是门的所有位总是产生零。 在这种情况下，每对都应该做出贡献，给出 n²。 该算法自然地处理这个问题，因为每个 (x, y) 都通过按位检查，因此包含每个频率乘积。 

另一个边缘情况是当
