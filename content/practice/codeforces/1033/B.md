 ## 解决方案
 ## 问题理解

 我们从一块边长的大方形布开始$a$。 从一个角开始，形成一个较小的正方形边$b$被剪掉了。 剩下的布是一个L形区域，其面积就是大正方形的面积减去被移除的正方形的面积，所以$a^2 - b^2$。 

任务不是直接计算这个面积以进行输出，而是确定这个结果数是否是素数。 

每个测试用例给出一对$(a, b)$，我们必须回答是否$a^2 - b^2$是素数。 

限制条件非常大：$a$和$b$可以达到$10^{11}$，这意味着$a^2$可以达到$10^{22}$。 这立即排除了任何尝试通过试除法将数字因子分解为值本身，甚至为每个测试用例以天真的方式分解为平方根的方法。 甚至$O(\sqrt{n})$如果重复执行，每个测试用例就会变得临界，因为$\sqrt{10^{22}} = 10^{11}$，太大了。 

这里的关键结构是我们没有得到任意的数字。 该值始终是两个平方之差，具有很强的代数性质，限制了其因式分解行为。 

出现微妙的边缘情况时$a$和$b$非常接近。 例如，$a=34, b=33$给出$34^2 - 33^2 = 67$，这是素数。 但对于稍大的间隙，例如$16, 13$，我们得到$87$，即使数字很小，它也是合数。 另一个边缘情况是当表达式变得偶数时； 任何大于 2 的偶数都自动成为非素数，这已经排除了许多无需计算的情况。 

## 方法

 暴力方法将计算$a^2 - b^2$然后使用试除法来测试素数$\sqrt{a^2 - b^2}$。 虽然正确，但这是不可行的，因为差异可能大到$10^{22}$，使得素性测试需要高达$10^{11}$最坏情况下每个测试用例的迭代次数。 

关键的见解来自于重写表达式：$$a^2 - b^2 = (a-b)(a+b)$$这个分解是决定性的。 该数字始终是两个大于零的整数的乘积，并且其结构受到极大限制。 

自从$a > b$，两个因子都是正整数。 还，$a+b > a-b$，并且两者都至少为 1。两个整数的乘积是素数的唯一方式是当其中一个等于 1 时。 

所以我们问：什么时候可以$(a-b)(a+b)$等于素数？ 

质数恰好有两个正因数：1 和它本身。 因此，其中一个因素必须是 1。$a+b \ge 2$，不可能是1。所以唯一的可能是：$$a - b = 1$$如果$a-b = 1$，则表达式变为：$$(1)(2b+1) = 2b+1$$所以结果是素数当且仅当$2b+1$是素数。 

这将问题简化为对最多的数字进行单一素性测试$2 \cdot 10^{11}$，这是可以通过确定性试除来管理的$\sqrt{n}$，或者仅使用奇数除数进行更快的检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(\sqrt{n} \cdot t)$和$n \approx 10^{22}$|$O(1)$| 太慢了 |
 | 最佳|$O(\sqrt{b})$每个有效案例（通常由于早期拒绝而保持不变）|$O(1)$| 已接受 |

 ## 算法演练

 ### 关键思想设置

 1. 计算$d = a - b$。 这隔离了唯一可能将因式分解简化为类似素数的部分。 
2.如果$d \neq 1$，立即返回“NO”。 这是因为该数字成为两个严格大于 1 的整数的乘积，因此它是合数。 
3.如果$d = 1$, 计算$x = a^2 - b^2 = (a-b)(a+b) = 2b + 1$。 
4. 检查是否$x$使用试除法是素数$\sqrt{x}$。 
5. 如果是素数，则返回“YES”，否则返回“NO”。 

关键的观察结果是，所有复杂性都会崩溃为单个特殊情况，其中结构退化为候选素数。 

### 为什么它有效

 不变的是每个有效值$a^2 - b^2$因素完全一样$(a-b)(a+b)$。 自从$a+b > a-b$并且两者都是大于或等于 1 的整数，除非有一个因子为 1，否则该表达式是合数。满足这一点的唯一可能方法是$a-b = 1$，这独特地将问题简化为测试单个派生数的素数$2b+1$。 没有其他结构可以产生素数，因为任何非平凡因式分解都会立即违反素数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_prime(n: int) -> bool:
    if n < 2:
        return False
    if n % 2 == 0:
        return n == 2
    i = 3
    while i * i <= n:
        if n % i == 0:
            return False
        i += 2
    return True

t = int(input())
for _ in range(t):
    a, b = map(int, input().split())
    
    if a - b != 1:
        print("NO")
    else:
        val = 2 * b + 1
        print("YES" if is_prime(val) else "NO")
```该实现依赖于分离琐碎的拒绝案例$a-b \neq 1$来自唯一有意义的案例。 这避免了任何大量的算术运算$a^2$直接，防止溢出问题和不必要的计算。 

素性检查通过在处理 2 后跳过偶数来优化，考虑到结构如下$2b+1$，总是奇怪的。

 ## 边缘情况

 ### 案例：$a-b > 1$用于输入$a=10, b=7$，我们得到$a-b=3$。 该算法立即返回“NO”。 这是正确的，因为$a^2-b^2 = (a-b)(a+b) = 3 \cdot 17 = 51$，这是复合的。 

计算永远不会达到素性测试，这避免了不必要的工作。 

### 案例：最小差异

 对于$a=3, b=2$，我们得到$a-b=1$，所以我们计算$2b+1=5$。 素性测试确认 5 是素数，因此输出为“YES”。 这是逻辑激活第二个分支的最小的非平凡情况。 

### 案例：大值

 对于$a=10^{11}, b=10^{11}-1$，算法计算$2b+1 \approx 2 \cdot 10^{11}$。 素数检查运行在$O(\sqrt{b})$，但这仍然很容易拟合，因为只需要几十万次迭代，并且$t \le 5$。 

这证实了即使在上限，解决方案也能清晰地缩放。
