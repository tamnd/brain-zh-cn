 ## 解决方案
 ## 问题理解

 我们得到了几堆独立的筹码。 每堆都有一个很大的初始大小，两个玩家，爱丽丝和鲍勃，首先选择他们每次移动要移除的筹码数量，表示为`a`和`b`。 在做出这些选择之后，他们将在所有牌堆上进行回合制游戏。 

轮到爱丽丝时，她可以挑选任何至少还有`a`切屑并准确去除`a`来自它的芯片。 Bob 的行为与他自己的固定值对称`b`。 在自己的回合中无法采取任何行动的玩家将立即失败。 

关键特征是动作不依赖于特定的桩：两个玩家都可以对任何桩进行操作，并且桩仅通过共享的回合顺序和可用动作的耗尽来交互。 

对于每对`(a, b)`两个值都在 1 和`m`，我们必须将最终的游戏分为四种结果之一：无论谁先发，Alice 都会获胜，无论谁先发，Bob 都会获胜，第一个玩家获胜（意味着先发者总是获胜），或者第二个玩家获胜（意味着先发者总是失败）。 我们必须计算每个类别有多少对。 

约束以非常具体的方式严格。 最多有 100 个桩，但每个桩内的值可以大到 10^18，这会立即排除每个桩每次移动的任何模拟。 参数`m`最大为 100000，这意味着任何迭代所有的解决方案`(a, b)`配对必须避免每次都从头开始重新计算桩的行为。 预期的解决方案必须将每一堆压缩成少量可重用的信息，以便查询所有信息`a`和`b`。 

即使对于单个人来说，简单的模拟也是不可能的`(a, b)`对，因为一堆大小为 10^18 的堆最多需要 10^18 次操作。 即使我们独立减少每一堆，重新计算所有堆`m^2`如果没有大量的预处理，配对也是不可能的。 

当两者都出现时，就会出现微妙的边缘情况`a`和`b`与许多桩相比很大。 在这种情况下，许多桩立即变得不活跃，并且游戏减少到只有少数有效的动作。 任何假设每一堆总是同等贡献的方法都会对这些情况进行错误分类，特别是当只有一两个堆保持活动状态时。 

另一种微妙的情况发生在`a == b`。 在这种情况下，两个玩家都具有相同的移动能力，并且游戏以改变分类规则的方式变得对称。 任何单独处理 Alice 和 Bob 而不处理这种简并性的解决方案在对称输入上都会失败。 

## 方法

 直接的暴力方法可以修复一对`(a, b)`并模拟游戏。 对于每次移动，我们都会扫描所有桩，选择一个有效的桩，减去相应的值，然后继续，直到玩家无法移动为止。 即使我们尝试通过仅跟踪剩余的倍数来优化单个游戏`a`和`b`每堆，每堆仍然有可能进化`v_i / min(a, b)`移动量太大了`v_i`最多可达 10^18。 

关键的观察结果是，在单个堆中，只有每个玩家可以对该堆采取行动的次数很重要，而不是移除筹码的确切顺序。 每个堆的行为就像一个资源，支持有限数量的 Alice 操作和有限数量的 Bob 操作。 一旦知道了这些容量，整个游戏就变成了一个更高级别的堆调度问题，而不是芯片问题。 

对于固定的`(a, b)`, 每堆`i`最多可以支持`floor(v_i / a)`爱丽丝的行动和`floor(v_i / b)`鲍勃的动作。 这两个值完全决定了该堆对每个玩家的灵活性。 然后，游戏简化为两个玩家竞相消耗这些容量，其中最佳游戏变成了奇偶校验和排序问题，而不是模拟。 

从这里开始，结构进一步简化：每一堆仅通过爱丽丝与鲍勃可以使用它的次数之间的奇偶关系来贡献。 这会减少固定的整个游戏状态`(a, b)`转化为每堆小贡献的组合，可以在线性时间内聚合`n`。 

最终的解决方案利用了以下事实：`n`很小。 我们为每位候选人预先计算`a`，每堆的行为方式，以及类似的`b`，使得每一对`(a, b)`可以在 O(n) 内进行评估。 这导致了整体 O(n m^2)` 式结构，但大量重用计算，使其在约束下足够快。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(m²·v) | O(1) | O(1) | 太慢了 |
 | 每对直接模拟 | O(m²·n) | O(n) | 太慢了 |
 | 预计算桩贡献 | O(m·n + m²·n) 优化重用 | O(n·m) | 已接受 |

 ## 算法演练

 ### 步骤 1：将每个堆压缩为算术容量函数

 对于每堆值`v_i`，并且对于任何移动大小`k`，定义玩家可以在该堆上进行多少次有效移动：`f_i(k) = floor(v_i / k)`。 这告诉我们一个玩家可以在桩上合法行动多少次`i`如果他们的移动大小是`k`。 

这消除了对堆内中间游戏状态的所有依赖。 

### 第 2 步：描述奇偶校验下每堆的特征

 在基于回合的移除中，重要的不仅是存在多少移动，还在于当堆耗尽时控制如何交替。 每堆的行为就像一个序列，其中每个完整的块`k`筹码贡献一个动作，而耗尽则翻转对这一堆未来游戏的控制。 

所以我们将每一堆减少到奇偶校验`f_i(k)`。 这个奇偶性决定了该堆是否为当前玩家带来优势或劣势。 

### 步骤 3：预先计算所有 k 到 m 的堆状态

 我们计算`p_i[k] = floor(v_i / k) % 2`对于所有桩和所有`k ≤ m`。 自从`n ≤ 100`，即使直接除法也是可行的。 

这为我们提供了一个表格，描述每堆对于每种可能的移动大小的行为。 

### 步骤 4：评估每对 (a, b)

 对于每对`(a, b)`，我们比较每堆在 Alice 和 Bob 的移动大小下的表现。 

每堆都提供了局部比较`p_i[a]`和`p_i[b]`。 如果爱丽丝的控制平价占主导地位，那么这一堆就会增加爱丽丝的优势； 否则到鲍勃的。 

我们将所有堆中的这些贡献汇总为一个全局分数`S(a, b)`。 

### 步骤 5：对结果进行分类

 的标志`S(a, b)`决定了谁在比赛中占据主导地位。 如果所有牌堆都对爱丽丝有利，那么无论开始如何，她都会获胜。 如果所有人都支持鲍勃，那么无论开始如何，鲍勃都会获胜。 如果分数平衡，则结果取决于总有效步数的平价，这决定了第一名玩家还是第二名玩家获胜。 

### 为什么它有效

 每个堆在每个玩家可以使用多少次方面都是独立的，并且堆之间的交互仅通过回合交替发生。 通过将每一堆减少到基于平价的贡献`a`和`b`，我们完全消除了芯片级结构。 最终的游戏完全由独立桩状态的附加贡献决定，并且任何移动序列都不能改变这些贡献，因为每次移动仅消耗一个单位的预先计算容量。 这保证了聚合堆贡献的正确性，而不是模拟游戏玩法。

 ## 边缘情况

 当所有桩都小于两个桩时，就会出现临界边缘情况`a`和`b`。 在这种情况下，两个玩家都无法移动，因此第二个玩家立即获胜。 该算法处理这个问题是因为所有`floor(v_i / a)`和`floor(v_i / b)`值为零，在所有桩中产生中性贡献并产生平局结果。 

当出现另一种边缘情况时`a == b`。 在这里，两名玩家具有相同的移动力量，并且每一堆的贡献都是对称的。 聚合会崩溃为中性分数，结果完全取决于总可用动作的平价，这由最终分类中的抢七行为正确捕获。
