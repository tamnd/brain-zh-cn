 ## 解决方案
 ## 问题理解

 我们得到一组整数，每个整数都被认为是“几乎素数结构”，因为它的除数数量非常小，在 3 到 5 之间。我们将所有这些数字相乘成一个巨大的数字，并被要求计算最终产品有多少个除数，模 998244353。 

关键对象不是值本身，而是它们的素因数分解。 一个数的约数仅取决于其质因数分解中的指数。 如果一个数字写成$x = p_1^{e_1} p_2^{e_2} \cdots$，那么除数计数为$(e_1 + 1)(e_2 + 1)\cdots$。 因此，该任务相当于重建所有输入数字中出现的所有素数的组合指数结构，然后应用该公式。 

约束条件$n \le 500$很小，但每个数字可以大到$2 \cdot 10^{18}$，这使得直接因式分解变得非常重要。 然而，除数限制极其严格。 具有 3 到 5 个约数的数字只能有非常有限的形式：要么是素数平方$p^2$（3 个约数），两个不同素数的乘积$pq$（4 个约数），或素数立方$p^3$或者$p^4$（4 或 5 个约数，具体取决于指数结构），或四次方质数$p^4$（5 个除数）。 这种结构确保了每个$a_i$至多有两个不同的质因数。 

一种简单的方法，尝试使用通用方法（例如试除法）对每个数字进行因式分解$\sqrt{a_i}$在最坏的情况下会太慢，因为$\sqrt{10^{18}} = 10^9$。 即使重复 500 次也是不可能的。 

第二个微妙的问题是，某些数字可能在不同的数字之间共享素数$a_i$，因此除非我们正确地全局聚合指数，否则每个数字的局部推理是不够的。 

当数字是单个素数的幂时，就会出现边缘情况。 例如，如果$a_i = p^4$，那么它将指数 4 贡献给一个素数，而忽略这一点会导致除数计数不正确。 另一个问题是错误分类$p^2$相对$pq$，两者的大小看起来相似，但指数结构不同。 

## 方法

 蛮力的想法是分解每一个$a_i$独立地使用试除法或 Pollard-Rho，收集所有素数指数，并将指数贡献乘以全局字典。 原则上这是正确的，因为除数计数仅取决于指数，但一般整数分解在这里是多余的。 

结构性约束彻底改变了情况。 由于每个数最多有 5 个约数，因此它的质因数分解必须非常小。 这意味着每个$a_i$是单个素数的幂或至多两个素数的乘积。 这意味着我们只需要检测每个数字最多两个素数。 

我们可以利用数字之间的 gcd 关系，而不是完全分解。 如果两个数字共享一个质因数，它们的 gcd 将直接暴露它。 一旦我们检测到一个素因数，我们就把它完全除掉。 由于每个数字在不同素数方面都很小，因此重复的 gcd 提取可以完全分解所有数字。 

这将问题从重整数分解简化为受控的基于 gcd 的剥离过程，其中每个步骤都确定性地提取质因数。

 ## 算法演练

 我们维护剩余值的多重集和素数指数字典。 

1. 对于每个数字$a_i$，我们尝试通过使用 gcd 将其与其他数字进行比较来提取其素数结构。 这个想法是任何共享的素因数都将显示为一个不平凡的 gcd。 
2. 对于每个$a_i$, 选择另一个$a_j$。 计算$g = gcd(a_i, a_j)$。 如果$g > 1$， 然后$g$必须包含至少一个完整的质因数$a_i$。 
3.如果$g$既不为 1 也不等于$a_i$，我们发现了一个合适的因子。 我们对待$g$作为主要功率元件并将其从$a_i$, 更新$a_i \leftarrow a_i / g$。 此步骤是安全的，因为除数结构保证除了单层素数之外没有隐藏的复合干扰。 
4. 重复 gcd 提取，直到每个$a_i$减少到 1 或素数幂。 任何大于 1 的剩余值本身就是素数幂。 
5. 一旦所有数字都分解为素数幂，我们就聚合每个素数的指数计数。 每次出现质因数都会增加其指数和。 
6. 最后，计算除数计数作为素数的乘积$(e_p + 1)$，模 998244353。 

### 为什么它有效

 每个数字至多有两个不同的质因数。 这严重限制了 gcd 交互的行为方式：任何重要的 gcd 都必须对应于数字之间共享的整个素数功率因数。 这可以防止一般整数中存在的不明确的部分重叠。 因此，重复的 gcd 剥离干净地隔离了所有素数贡献，而不需要显式分解。 不变的是，在每个提取步骤之后，所有移除的组件都保证是完整的主要功率贡献并且不会混合复合材料。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

MOD = 998244353

def solve():
    n = int(input())
    a = [int(input()) for _ in range(n)]

    primes = {}

    for i in range(n):
        x = a[i]
        if x == 1:
            continue

        for j in range(n):
            if i == j:
                continue
            if x == 1:
                break
            g = gcd(x, a[j])
            if g != 1 and g != x:
                # extract factor
                while x % g == 0:
                    x //= g
                # record factor g contribution
                y = g
                primes[y] = primes.get(y, 0) + 1

        if x > 1:
            primes[x] = primes.get(x, 0) + 1

    ans = 1
    for e in primes.values():
        ans = ans * (e + 1) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该代码迭代每个数字，并尝试使用 gcd 与其他数字剥离共享因子。 每当发现重要的 gcd 时，它就会被视为完整的主功率分量，并从当前值中完全删除。 所有交互作用后的任何剩余值都被假定为剩余素幂并直接记录。 

最后的除数计算是乘法除数公式在素数指数上的直接应用。 

一个微妙的细节是，我们将所有出现的 gcd 因子从当前数字中完全除掉，确保不留下任何部分残留。 这避免了重复计算相同的主要贡献。

 ## 边缘情况

 一个关键的边缘情况是所有数字都是不同素数的幂。 在这种情况下，所有 gcd 检查都会返回 1，并且算法必须回退到将每个数字视为完整素数幂。 最后的乘法为$(e+1)$仍然正确地计算指数 1 或更高。 

另一个边缘情况是重复的权力，例如$p^4$。 即使只有一个素数，其他数字的 gcd 可能永远不会触发，因此算法必须正确保留约简后的完整数字。 最后一步确保任何剩余的$x > 1$没有丢失。 

最后一个微妙的情况是两个数字共享一个素数但具有不同的指数。 gcd 提取仍然隔离共享基素数结构，因为任何 gcd 都将包含最小指数幂，而全除法将其从两个数字中干净地删除。
