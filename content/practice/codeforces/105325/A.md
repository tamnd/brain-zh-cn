 ## 解决方案
 ## 问题理解

 我们被要求为完整图的每条边分配权重$n$标记的城市。 有$\frac{n(n-1)}{2}$无向边，我们必须将每个整数放在$1$到$\frac{n(n-1)}{2}$正好一次。 

扭曲是这些权重引起的度量的一致性条件。 如果我们将权重解释为边长，并将两个城市之间的距离定义为图中的最短路径距离，那么对于尽可能多的对$(u,v)$尽可能地，最短路径距离必须等于之间的直接边权重$u$和$v$。 换句话说，我们希望大多数边的行为就像“度量边”一样，永远不会通过中间顶点的路由来改进。 

简单的解读表明，我们正在尝试使图表表现得像树度量，同时仍然完整并使用所有不同的权重。 这在全球范围内是不可能的，因为在任何循环中，如果权重是任意的，三角不等式都会迫使至少一条边可以改进。 评分函数证实了这一点：我们获得的奖励与满足条件的对数成比例，因此构造不需要完美，只需构建最大化直接边最优性的结构即可。 

约束很小，$n \le 52$，所以任意$O(n^2)$或者$O(n^3)$施工是微不足道的。 真正的挑战纯粹是组合：决定权重的排序，使最直接的边仍然是最短路径。 

天真的思维的一个微妙的失败案例是任意分配权重，例如逐行填充边缘或随机排列标签。 在这样的结构中，几乎每个三角形$(i,j,k)$将具有较重的直接边，通过第三个顶点变得非最佳。 例如，如果$w(i,j)$都很大$w(i,k)$和$w(k,j)$都很小，那么之间的最短路径$i$和$j$避免直接边缘，打破条件。 

因此，我们的目标是构建权重，使大多数三角形都是“紧”的，即直接边被迫成为最小路线。 

## 方法

 完整的图表建议考虑按“重要性”对边进行排序。 关键的障碍是任何三角形都会在其三个边之间产生竞争。 如果我们将较小的权重放置在已经由小边良好连接的图形区域中，那么这些边可能会保持最佳状态，而稍后放置的较大权重更容易被绕过。 

一个蛮力的想法是尝试所有边权重的排列，并计算每对的直接边是否最短。 这需要为每个排列运行全对最短路径，这是不可行的，因为有$(n^2)!$排列，甚至评估一个候选人都需要$O(n^3)$通过弗洛伊德·沃歇尔。 

关键的见解是避免全局推理最短路径，而是强制执行保证大部分边缘子集最优性的局部结构。 在具有不同权重的完整图中实现此目的的标准方法是以仔细排序的方式构造边，以便当连接新顶点时，其关联边以受控方式大于所有先前放置的边。 这确保了较早的边不能通过较晚的顶点被绕过，因为较晚的顶点只会引入更大的弯路。 

我们本质上是模拟一个增长过程：从边缘较小的小核心开始，然后逐步附加权重越来越大的新顶点。 在此结构中，较早顶点内的边受​​到保护，因为通过较新顶点的任何替代路径都使用较大的权重。 涉及较新顶点的边可能会失去最优性，但这些边相对较少，并且该构造最大化了保留边的数量。 

这将问题简化为在增加“层深度”时决定边的一致顺序，以便图的行为类似于嵌套层次结构。

 ## 算法演练

 我们以递增的顺序构造边权重，将它们逐行填充到上三角邻接矩阵中。 

1.初始化一个计数器$cur = 1$。 这代表下一个未使用的边权重。 
2. 迭代第一个顶点$i$从$1$到$n-1$。 对于每个$i$，我们为边分配权重$(i, j)$为所有人$j > i$。 
3. 对于每一对这样的$(i, j)$， 分配$w(i, j) = cur$，然后递增$cur$。 
4. 按照这个确切的顺序输出上三角矩阵的所有行。 

关键思想是，较早的顶点会获得带有较小标签的边到较晚的顶点，并且当我们向下移动时$i$，边缘逐渐变大。 

### 为什么它有效

 考虑任何边缘$(i, j)$和$i < j$。 任何替代路径$i$到$j$必须经过一些中间顶点$k$。 如果$k < i$，然后两条边$(k,i)$和$(k,j)$被分配得更早或更小，但该结构确保绕道通过这样的$k$相对于直接边缘不会降低成本，因为直接边缘$(i,j)$在与涉及的较早行相关的所有边之后分配$i$已经按受控递增顺序排列。 

如果$k > i$，然后是边$(i,k)$和$(k,j)$权重严格大于或等于行中的边$i$，使任何两步路径通过$k$一旦顺序确定，严格来说比直接边缘更昂贵。 

这种排序强制执行单调结构：靠近矩阵左上角的边缘主导潜在的绕行，因此对于大部分对来说，最短路径与直接边缘一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    cur = 1

    for i in range(n - 1):
        row = []
        for j in range(i + 1, n):
            row.append(str(cur))
            cur += 1
        print(" ".join(row))

if __name__ == "__main__":
    solve()
```该实现直接遵循上三角邻接矩阵的构造。 唯一的状态是全局计数器`cur`，这保证了每个整数$1$到$\frac{n(n-1)}{2}$仅使用一次。 

循环结构与输出格式完全匹配：行$i$印刷$n-i-1$对应于边的值$(i, j)$为所有人$j > i$。 无需存储完整的矩阵，因为值会立即生成并打印。

 ## 边缘情况

 最小的非平凡情况是$n=3$。 该算法分配$(1,2)=1$,$(1,3)=2$,$(2,3)=3$。 输出是：```
1 2
3
```排序没有歧义，并且每条边都只使用一次。 由于没有第三个顶点替代方案可以将任何边减少到其直接权重以下（只有一个三角形），因此该构造基本上满足了预期的结构。 

在上限$n=52$，该算法仍然只执行 1326 次分配。 输出保持严格的三角形，并且不会发生行分配不匹配，因为循环结构直接镜像所需的格式。
