 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$棋盘和主教放置在起始方格上。 主教可以在一次移动中移动任意数量的单元格，但只能沿对角线移动。 任务不仅仅是决定主教是否可以到达目标方格，而是计算从起点到目的地存在多少条不同的最短移动路线。 

这里的最短路径是指使用尽可能少的对角线移动的路径。 由于主教移动是直线对角线滑动，因此如果我们已经从目的地开始，则唯一可能的路径长度是零移动；如果两个方格位于同一条对角线上，则唯一可能的路径长度是一移动；如果主教必须绕道通过与两个端点对角对齐的中间方格，则唯一可能的路径长度是两移动。 

电路板尺寸可以大到$10^9$，这会立即排除网格上的任何模拟。 任何解决方案都必须纯粹使用坐标几何而不是遍历。 每个决定仅取决于坐标之间的算术关系。 

第一个微妙之处是主教受到颜色平价的限制。 在棋盘上，象总是停留在具有相同值的格子上$x + y \bmod 2$。 如果起始方格和结束方格的奇偶性不同，则无论棋盘大小如何，都无法将它们连接起来。 

第二种边缘情况是起点和终点相同。 正确的答案正是一条长度为零的路径，即使没有执行任何移动。 

经常导致错误的第三种情况是两种不同的两步分解似乎是可能的。 由于主教移动是完整的对角线段，因此最多可能有两个不同的中间方块连接起始对角线和目标对角线。 

## 方法

 直接模拟将尝试探索从起始方格开始的所有可能的主教移动，并跟踪到目的地的所有最短路径。 从任何位置来看，主教最多有四个对角线方向，每个方向最多允许$O(n)$着陆位置。 即使我们将自己限制在最短路径上，状态空间也会变成一个图$O(n^2)$节点和密集的对角连接。 BFS 从概念上讲是可行的，但是$n$可以是$10^9$，因此节点或边的枚举是不可能的。 

关键的观察结果是主教运动结构将整个问题分解为一小组几何情况。 目的地要么位于起点的两条对角线上，要么不在。 如果是，则恰好存在一条最短路径。 如果不是，但方块共享奇偶性，则最短路径必须恰好是两次移动，并且通过选择同时位于起点对角线上和终点对角线上的方块来完全确定任何有效路径。 

这将问题简化为寻找两条对角线的交点。 每个主教的举动对应于选择线$x - y = c$或者$x + y = c$，因此中间的正方形是从起点开始的一条对角线与从终点开始的一条对角线的交点。 这样的交点最多有两个，我们只需要统计棋盘内有多少个有效的整数点即可。

 ## 算法演练

 1. 检查起始方格和结束方格是否相同。 如果是，则主教无需移动就已满足要求，因此只有一条有效路径。 
2. 使用以下方法计算两个方块的颜色奇偶性$(x + y) \bmod 2$。 如果这些不同，则返回零，因为主教移动序列不能跨越颜色类别。 
3. 检查两个正方形是否位于同一条对角线上。 如果发生这种情况$x_1 - y_1 = x_2 - y_2$或者$x_1 + y_1 = x_2 + y_2$。 如果成立，则象可以一招直接走，而且这样的路只有一条。 
4. 否则，最少移动次数为两次。 任何有效路径都必须经过一个中间方格，该中间方格在一次对角线移动中可从两个端点到达。 
5. 通过对角线相交构造最多两个候选中间正方形：

 第一个候选来自解决$x - y = x_1 - y_1$和$x + y = x_2 + y_2$。 

第二个候选人来自解决$x + y = x_1 + y_1$和$x - y = x_2 - y_2$。 
6. 对于每个候选人，计算$x = \frac{(sum \pm diff)}{2}$,$y = sum - x$，并检查两个坐标是否均为整数且位于棋盘边界内。 
7. 计算这些候选中间方块中有多少个是有效的。 每个有效中点恰好对应于一条不同的最短路径。 

### 为什么它有效

 每条最短的双步路径必须包括从起点到其对角线上某个方格的第一次移动，以及从该方格到目标的第二次移动。 在这两个步骤中唯一可到达的正方形正是由端点定义的对角线的交点。 由于每对对角线最多相交于一个点，因此候选中间方格的总数至多为两个，并且每个对角线对应于唯一的路径分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    x1, y1 = map(int, input().split())
    x2, y2 = map(int, input().split())

    if x1 == x2 and y1 == y2:
        print(1)
        return

    if (x1 + y1) % 2 != (x2 + y2) % 2:
        print(0)
        return

    if x1 - y1 == x2 - y2 or x1 + y1 == x2 + y2:
        print(1)
        return

    def valid(x, y):
        return 1 <= x <= n and 1 <= y <= n

    count = 0

    s1 = x1 + y1
    d1 = x1 - y1
    s2 = x2 + y2
    d2 = x2 - y2

    if (s2 + d1) % 2 == 0:
        x = (s2 + d1) // 2
        y = s2 - x
        if valid(x, y):
            count += 1

    if (s1 + d2) % 2 == 0:
        x = (s1 + d2) // 2
        y = s1 - x
        if valid(x, y):
            count += 1

    print(count)

if __name__ == "__main__":
    solve()
```该解决方案是围绕零移动和一移动情况的早期退出构建的，因为这些在坐标对齐时占主导地位。 奇偶校验在任何几何构造之前删除所有不可能的情况，避免不必要的算术。 

两个候选中点直接来自对对角方程组的求解。 整数检查通过偶和验证是隐式的，因为对角线交叉需要一致的奇偶校验。 然后根据板边界验证每个候选者，因为代数交集可能位于板边界之外$n \times n$网格，即使无限对角线在那里相交。

 ## 边缘情况

 当起点和终点重合时，算法在进行任何几何推理之前立即返回 1。 这可以避免错误地计算对角线交点，否则会建议中间的正方形。 

当奇偶校验不同时，计算会提前停止，因为所有后续的代数交集仍然代表颜色不正确的正方形，使得无论棋盘几何形状如何，任何路径都是不可能的。 

当两个正方形位于同一条对角线上时，两个候选中点公式都可能产生端点或无效点。 早期的对角线检查可以防止计算虚假的中间结构，并正确地将答案固定为一个。
