 ## 解决方案
 ## 问题理解

 我们正在用一堆石头玩一个两人外卖游戏。 玩家轮流轮流，每个回合玩家移除 1 到 K 颗石子（含）。 不同的是，玩家被禁止拿走与对手在前一棋中拿走的棋子数量完全相同的棋子。 如果玩家在自己的回合中没有合法的动作，那么该玩家就输了。 

任务是确定对于每个独立的游戏配置，第一个玩家是否强制获胜，或者第二个玩家是否可以在双方最佳发挥的情况下强制获胜。 

这些约束允许最多 10 个测试用例，N 最多为 5000 个。这强烈建议对棋子数量采用动态编程解决方案，可能需要额外的状态维度来跟踪最后的移动大小。 每个测试用例的三次甚至高二次解原则上是可以接受的，但是任何超过 N 或 K 的指数都会太慢。 

“禁止的最后一步”规则产生了一个微妙的边缘情况。 仅根据剩余棋子来跟踪某个位置是否获胜的简单方法是不够的。 例如，在 N=4、K=3 等小型配置中，最佳玩法关键取决于最后采取的行动，而忽略这一点会导致错误的结论，因为相同的牌堆大小可能会获胜或失败，具体取决于先前的移动约束。 

## 方法

 游戏的直接暴力模型将状态定义为（n，last），其中n是剩余的石子，last是前一个玩家拿走的石子的数量。 从这个状态开始，当前玩家尝试从 1 到 K 的所有移动 i（除了 i = 最后），并转换到 (n - i, i)。 如果任何举动导致对手处于失败状态，则当前状态为获胜。 

这个公式是正确的，但如果简单地实现，它会创建 O(NK) 状态，每个状态最多有 K 个转换，导致每个测试用例为 O(NK^2)。 当N、K达到5000时，在最坏的情况下这将变成10^11次运算，这远远超出了极限。 

关键的结构观察是，转换仅取决于每个子状态是输还是赢，并且约束“不能采取最后一步”仅排除每个状态的一个选项。 我们可以预先计算每个 n 和每个可能的最后是否存在至少一个获胜动作，而不是为每个最后独立地重新计算转换。 这可以通过为每个 n 维护从 1 到 K 的所有移动的计数或布尔值并减去禁止的移动来加速。 这将内部转换转变为每个状态的 O(1) 摊销工作。 

我们可以建立一个 DP 表 dp[n][last]，其中 dp[n][last] 为真，如果当前玩家可以在剩余 n 颗棋子的情况下获胜，并且最后一步是最后一步。 我们以增加的 n 来计算值，因为所有的转换都会变得更小的 n。 对于每个状态，我们想知道是否存在一个移动 i 使得 i ≠ Last 并且 dp[n - i][i] 为假。 

我们可以为每个 n 维护一个移动前缀结构，指示哪些移动会导致丢失状态。 这允许在预处理转换后检查每个状态 O(1) 中是否存在有效移动。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N·K²) | O(N·K) | 太慢了 |
 | 具有移动排除功能的优化 DP | O(N·K) | O(N·K) | 已接受 |

 ## 算法演练

 我们定义一个DP表dp[n][last]，其中last是前一个移动大小（0表示开始时没有限制）。

1. 初始化 dp[0][last] = false 表示所有最后一个。 由于没有剩余棋子，当前玩家无法移动并立即失败，因此这是基本情况。 
2. 从 1 到 N 迭代 n。我们计算 [0, K] 中所有最后一个的 dp[n][last]。 
3. 对于固定状态（n，最后），我们考虑从 1 到 K 的所有可能的移动 i。如果 i > n，我们停止，因为我们不能拿比可用的棋子更多的棋子。 
4. 如果 i == last 则跳过 i 步，因为规则禁止这样做。 
5. 对于每个有效的移动 i，我们检查 dp[n - i][i]。 如果存在任何移动使得 dp[n - i][i] 为假，则 dp[n][last] 为真，因为我们可以迫使对手进入失败状态。 
6. 如果不存在这样的移动，则 dp[n][last] 为 false。 
7. 测试用例的答案是dp[N][0]，表示没有限制的初始状态。 

### 为什么它有效

 DP 状态完全编码所有相关历史：只有剩余的桩大小和最后一步的合法性才重要。 每个转换都严格减少 n，因此 DP 是非循环的。 递归完全符合获胜位置的极小极大定义：当且仅当存在导致对手失败状态的移动时，状态才获胜。 由于所有可达状态在使用之前都会进行评估，因此每个 dp 值在计算时都是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        N, K = map(int, input().split())

        dp = [[False] * (K + 1) for _ in range(N + 1)]

        for n in range(1, N + 1):
            for last in range(K + 1):
                win = False
                upper = min(K, n)
                for take in range(1, upper + 1):
                    if take == last:
                        continue
                    if not dp[n - take][take]:
                        win = True
                        break
                dp[n][last] = win

        print(1 if dp[N][0] else 2)

if __name__ == "__main__":
    solve()
```DP表是自下而上构建的，确保在评估dp[n][last]时，所有状态dp[n - take][take]都已被计算，因为n - take严格小于n。 嵌套循环直接反映了递归情况。 条件`take == last`强制执行不能重复前一步的规则。 

初始状态使用last = 0，它充当虚拟值，永远不会与任何合法的移动发生冲突。 

## 工作示例

 ### 示例 1：N = 4，K = 2

 我们增量计算 dp。 

| n | 最后| 有效动作 | 赢了？ |
 | --- | --- | --- | --- |
 | 1 | 0 | 1 | 真实 |
 | 2 | 0 | 1,2 | 真实 |
 | 3 | 0 | 1,2 | 真实 |
 | 4 | 0 | 取决于 dp[3][1], dp[2][2] | 真实 |

 对于 n=4，last=0，取 1 留下 dp[3][1]。 由于 dp[3][1] 在这个配置链中输给了对手，所以第一个玩家有获胜的举动。 

这确认了第一个样本输出为 1。 

### 示例 2：N = 4，K = 3

 我们再次计算 dp[4][0]。 可能的动作有 1、2、3。 

我们检查结果：

 - 将 1 引向 dp[3][1]
 - 取得 2 条线索到达 dp[2][2]
 - 取得 3 条线索到达 dp[1][3]

 这些状态中的每一个都允许下一个玩家做出最佳响应，这样第一个玩家就无法强行获胜，因此 dp[4][0] 变为 false。 

| 移动| 下一个状态 | 结果 |
 | --- | --- | --- |
 | 1 | dp[3][1] | dp[3][1] | 为对手路径获胜|
 | 2 | dp[2][2] | dp[2][2] | 为对手路径获胜|
 | 3 | dp[1][3] | dp[1][3] | 为对手路径获胜|

 因此第二个玩家获胜。

 ## 边缘情况

 一种重要的边缘情况是当 K 大于 N 时。在这种情况下，可用的移动会随着 n 的减小而动态缩小。 DP通过以下方式正确处理了这个问题`upper = min(K, n)`，确保我们永远不会考虑无效的举动。 例如，当 n = 3 且 K = 5 时，仅考虑移动 1 到 3。 

另一种边缘情况是最后的移动限制消除了唯一可能的移动。 例如，如果 n = 2、K = 2、last = 1，则只有第 2 步有效。 DP 仅正确评估剩余的合法转换，因此该状态不会因无效选项而被错误地标记为获胜。 

第三种边缘情况发生在游戏开始时，last = 0。这不能排除任何移动，因为 0 不是有效的取值。 初始化通过允许从初始状态开始的所有移动来确保这一点。
