 ## 解决方案
 ## 问题理解

 我们正在使用长度正确平衡的括号字符串$2n$。 这样的字符串可以被认为是一个序列$n$左括号和$n$右括号的排列方式使得在每个前缀处，开括号的数量永远不会超过闭括号的数量，并且在末尾计数匹配。 

从所有这些平衡字符串中，我们被要求仅计算那些即使在删除两个中间字符（特别是位置）后仍保持平衡的特殊字符串$n$和$n+1$在 1 索引的字符串中。 删除会产生长度较短的字符串$2n-2$，并且我们要求这个结果字符串仍然是正确的括号序列。 

因此，约束不仅仅是全局平衡，而是以序列中间为中心的结构稳定条件。 我们正在有效地计算在固定内部删除操作下仍然有效的加泰罗尼亚结构。 

约束条件$n \le 30$足够小，以至于涉及动态规划的解决方案超过$O(n^3)$甚至$O(n^4)$状态是可以接受的，因为总规模在几万次操作以下。 这强烈表明问题在于结构计数而不是贪婪或模拟。 

一种简单的方法将生成所有加泰罗尼亚语序列，其增长为$C_n$，并测试删除条件。 这已经是可行的$n = 30$，但本质上仍然是指数级的。 更糟糕的是，删除后验证每个序列会增加另一个线性因素，使其不切实际。 

天真的推理的一个微妙的失败案例是假设如果原始是平衡的，则删除中间总是保持平衡。 例如，`((()))`删除位置 3 和 4 后仍然有效，但是`(()())`在其结构的任意重新排列下并不总是表现相似。 删除的效果取决于中间区域的嵌套深度，而不仅仅是全局平衡。 

关键的困难在于，删除将序列分成两部分，其相互作用取决于切割边界处的前缀后缀平衡对齐。 

## 方法

 暴力策略枚举所有正确长度的括号序列$2n$，然后删除位置$n$和$n+1$，并检查结果序列是否仍然有效。 正确性很简单，因为我们直接测试条件。 平衡序列的个数就是加泰罗尼亚数$C_n$，这对于$n=30$是关于$10^8$，太大了。 即使进行剪枝，生成所有序列也是指数级的，并且无法在时间限制内完成。 

为了避免枚举完整的结构，我们转而利用平衡序列在其中间的行为方式。 删除操作从中点边界的每一侧删除了一个字符，这表明该结构可以通过耦合条件分为左半部分和右半部分。 

我们将正确的括号序列重新解释为一条永远不会低于零的路径，使用$+1$对于“(”和$-1$为了 '）'。 中间的删除从每一侧删除一个步骤，以受限的方式有效地删除一对匹配的贡献。 问题变成了计算前缀到位置的路径$n-1$，与从以下位置开始的后缀组合$n+2$，重新加入后仍形成有效的 Dyck 路径。 

关键的见解是在中间之前和中间之后修复平衡。 让前缀达到位置$n-1$在某个高度结束$h$。 位置上的字符$n$贡献+1或-1，但由于它必须是与有效性一致的左括号或右括号，因此状态转换受到限制。 同样对于位置$n+1$。 

删除两者后，串联必须仍然保持非负性。 这将问题简化为计算将戴克路径分成两部分的方法，这两个部分的边界高度在移除局部对的情况下对齐。 

我们可以使用前缀长度和当前余额上的 DP 来形式化这一点，跟踪我们是在删除的段之前还是之后。 我们考虑三个区域：前缀之前$n$、中间的两个字符以及后面的后缀$n+1$。 中间对可以被视为连接两个 DP 状态的局部转换，同时强制删除不会违反有效性。 

这导致了对位置和平衡的动态规划，并带有额外的维度编码，无论我们是在左段、中段还是右段。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举|$O(C_n \cdot n)$|$O(n)$| 太慢了 |
 | DP 位置和平衡状态 |$O(n^3)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们定义 DP 状态来跟踪某个位置存在多少个有效的部分结构，同时保持括号平衡。 

1.我们定义一个DP表，其中$dp[i][b]$表示构造长度前缀的方式数$i$以平衡结束$b$，在我们仍在形成潜在解序列的有效前缀的约束下。 这是标准 Dyck 路径构造规则，确保$b \ge 0$。 
2.我们为前缀区域单独运行这个DP直到位置$n-1$。 在此阶段，我们记录所有可能的端点余额$b$。 这很重要，因为中间删除将取决于我们在删除区域之前达到的平衡。 
3.我们类似地计算后缀DP，但相反，从位置开始$2n$下降到$n+2$。 对于每个可能的起始余额，我们计算后缀在保持有效的同时可以返回到余额零的方式有多少种。 这反映了前缀计算，但时间相反。 
4. 我们明确考虑位置上的两个中间字符$n$和$n+1$。 我们枚举了它们可能的“(”和“)”分配，但强制在每一步中余额都不会变成负数。 对于每个可行的对，我们确定它们如何转换前缀和后缀之间的边界平衡。 
5. 对于每个可行的分裂状态，我们将前缀 DP 和后缀 DP 中与导出边界条件相匹配的路数相乘。 对所有有效组合求和得出最终答案。 

基本思想是，中间删除引起局部约束，该约束仅取决于进入和离开中间区域的平衡，而不取决于完整的结构。 

### 为什么它有效

 正确的括号序列可以在任何点分解为前缀和后缀，其接口完全由单个整数描述：当前余额。 DP 利用 Dyck 路径的马尔可夫特性。 

删除操作恰好删除了两个相邻位置，因此它仅影响跨固定大小窗口的转换。 该窗口之外的所有约束保持不变。 因此，结果序列的正确性仅取决于如何通过该局部修改来转换平衡。 DP枚举所有全局有效的结构，同时确保本地修改保留非负性，因此不会统计无效配置，也不会遗漏任何有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    N = 2 * n

    # dp_left[i][b] = ways for prefix of length i with balance b
    dp_left = [[0] * (n + 2) for _ in range(n + 1)]
    dp_left[0][0] = 1

    for i in range(n):
        for b in range(n + 1):
            if dp_left[i][b] == 0:
                continue
            # add '('
            dp_left[i + 1][b + 1] += dp_left[i][b]
            # add ')'
            if b > 0:
                dp_left[i + 1][b - 1] += dp_left[i][b]

    # dp_right[i][b] for suffix from position i to end
    dp_right = [[0] * (n + 2) for _ in range(N + 2)]
    dp_right[N + 1][0] = 1

    for i in range(N, n + 1, -1):
        for b in range(n + 1):
            if dp_right[i][b] == 0:
                continue
            # if we place '(' at i
            dp_right[i - 1][b + 1] += dp_right[i][b]
            # if we place ')' at i
            if b > 0:
                dp_right[i - 1][b - 1] += dp_right[i][b]

    ans = 0

    # middle positions are n and n+1
    for b in range(n + 1):
        for mid1 in [1, -1]:  # '(' or ')'
            for mid2 in [1, -1]:
                b1 = b + mid1
                if b1 < 0:
                    continue
                b2 = b1 + mid2
                if b2 < 0:
                    continue

                # after removing both, balance returns to b
                ans += dp_left[n - 1][b] * dp_right[n + 2][b2]

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案围绕两个 DP 表构建。 第一个构建直到第一个删除位置的所有有效前缀，明确跟踪余额。 第二个从末尾到第二个删除位置反向构建有效后缀。 

中间的转换循环尝试所有可能的局部括号配置以获取位置$n$和$n+1$，过滤掉那些会在任何中间步骤打破平衡约束的内容。 对于每个有效的中间配置，我们将前缀结束余额与由中间效果调整的后缀起始余额连接起来。 

在索引后缀 DP 时需要小心，因为它是根据位置而不是长度定义的。 关键的不变量是两个 DP 表始终表示有效的 Dyck 路径段，因此当它们的边界平衡匹配时，乘法是安全的。

 ## 边缘情况

 对于$n=1$，只有一个括号序列`"()"`。 删除中间的两个位置会删除整个字符串，留下一个空字符串，这是有效的。 DP 正确地处理了这个问题，因为前缀 DP 和后缀 DP 都还原为单个基本状态，并且中间循环没有无效转换。 

为了$n=2$，有效序列是`"()()"`和`"(())"`。 去掉位置2和位置3后，仅`"()"`对于其中一个仍然有效，它与 DP 的单个有效余额路径相匹配。 该算法正确区分了嵌套结构和顺序结构，因为它们的中间平衡不同，并且 DP 显式地分离了这些状态。
