 ## 解决方案
 ## 问题理解

 我们被要求构建一个输出整数对的生成器$(a, b)$满意的$1 \le a \le b \le k$，并且每当我们请求新值时，每个有效对都必须以相同的概率生成。 

关键要求不仅是生成有效对，还要确保整个有效对集的一致性。 这意味着如果有$T$总共有效对，每一对都必须有准确的概率$1/T$。 

输入由单个上限组成$k$，它定义了每对的两个分量可以采用的值的范围，以及一个整数$n$，这是我们必须生成的对的数量。 输出很简单$n$独立生成对。 

有效对的数量在结构上是三角形的。 对于每个固定$a$，值$b$范围可以从$a$到$k$, 给予$k - a + 1$对。 总结一切$a$，总对数为$k(k+1)/2$。 和$k$最多$10^9$，这个数字太大而无法显式枚举或存储，因此任何解决方案都必须使用公式或隐式索引。 

显式构建所有对的简单方法在时间和内存上都会立即失败。 即使生成完整列表也需要$O(k^2)$在最坏的情况下，这是不可能的。 

如果我们尝试选择，就会出现更微妙的失败案例$a$和$b$统一独立于$[1, k]$然后拒绝无效对，其中$a > b$。 这会产生一个有偏差的分布，因为配对较小$a$在拒绝后的有效结果中比例过高，因为拒绝概率在正方形的所有区域中并不均匀。 

核心困难是确保对嵌入正方形内的三角形域进行均匀采样。 

## 方法

 蛮力的想法是显式列出所有有效对$(a, b)$，将它们存储在一个数组中，并从该数组中统一选取。 这在概念上是有效的，因为每对都只表示一次。 然而，对的数量是$k(k+1)/2$，即使对于中等强度的情况来说，它也会变得天文数字般大$k$。 仅内存要求是不可能的，并且构建时间是$k$。 

关键的观察是我们不需要完整的对集合，只需要一种对它们进行统一索引的方法。 如果我们可以将每一对映射到一个唯一的整数$[1, T]$并有效地反转该映射，然后生成一对简化为生成一个统一整数并对其进行解码。 

我们可以按字典顺序对对进行排序$a$，然后由$b$。 对于固定的$a$，正好有$k - a + 1$对。 这给出了累积结构：所有对从较小的开始$a$形成连续的块。 如果我们定义一个前缀函数$$P(a) = \sum_{i=1}^{a} (k - i + 1),$$然后$P(a)$告诉我们有多少对存在直到并包括块$a$。 一旦我们采样了一个随机整数$x$在$[1, T]$，我们可以找到哪个$a$包含它，然后计算该块内的偏移量以恢复$b$。 

唯一剩下的挑战是找到$a$高效。 自从$P(a)$是一个二次函数，我们可以使用二分查找或直接使用二次公式来求逆它，这给出$O(1)$每个样本的计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举|$O(k^2)$|$O(k^2)$| 太慢了 |
 | 索引映射与反转 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们假设可以访问一个可以产生大范围值的均匀随机整数生成器。 

### 步骤

 1. 计算有效对的总数$T = k(k+1)/2$。 这定义了均匀采样空间。 
2. 对于每一个$n$所需的输出，生成一个随机整数$x$均匀地在$[1, T]$。 这表示字典顺序中一对的索引。 
3.确定最小的$a$使得对的数量达到阻止$a$，即$P(a)$，至少是$x$。 此步骤识别该对属于哪个第一坐标。 
4.一次$a$已知，计算距离$x$位于块内$a$。 该偏移量对应于选择$b$在范围内$[a, k]$， 所以$b = a + (x - P(a-1) - 1)$以从 0 开始的偏移量形式。 
5、输出$(a, b)$。 

唯一重要的部分是步骤 3。我们不是线性扫描，而是反转二次形式$P(a)$。 自从$P(a)$是单调递增的，反演是明确定义的并且产生独特的$a$。 

### 为什么它有效

 该结构通过固定的方式将整个有效对集合划分为连续的块$a$。 每个块的大小都是精确的$k-a+1$，因此索引空间与这些块大小完全对齐。 因为随机索引是均匀的$[1, T]$，每个块位置的可能性均等，并且在每个块内每个$b$是同样有可能的。 这种两级均匀性保证了每对$(a,b)$被精确地选择概率$1/T$，因此所有有效对的分布是均匀的。 

## Python 解决方案```python
import sys
import random
import math

input = sys.stdin.readline

def find_a(k, x):
    # Solve P(a) >= x where P(a) = a*k - a*(a-1)/2
    # Rearranged quadratic: a^2 - (2k+1)a + 2x <= 0
    # Take smaller root upper bound

    A = 2 * k + 1
    D = A * A - 8 * x
    s = int(math.isqrt(D))

    a = (A - s) // 2
    if a < 1:
        a = 1

    # fix boundary if needed
    while a * k - a * (a - 1) // 2 < x:
        a += 1
    while (a - 1) * k - (a - 1) * (a - 2) // 2 >= x:
        a -= 1

    return a

def prefix(k, a):
    return a * k - a * (a - 1) // 2

def solve():
    k = int(input())
    n = int(input())

    T = k * (k + 1) // 2

    for _ in range(n):
        x = random.randint(1, T)

        a = find_a(k, x)
        prev = prefix(k, a - 1)
        offset = x - prev - 1
        b = a + offset

        print(a, b)

if __name__ == "__main__":
    solve()
```功能`find_a`使用二次方程的封闭式近似来执行前缀和的反演，然后通过最多几次调整来对其进行校正，以确保其落在精确的边界上。 这避免了完全二分搜索，同时在整数舍入下保持安全。 

这`prefix`函数对给定的累积计数进行编码$a$，它既用于反转校正又用于计算块内的偏移。 

主循环生成一个统一索引并使用此逆映射将其转换为一对。 使用`random.randint`确保整个指标范围的一致性。 

## 工作示例

 考虑一个小案例，其中$k = 4$。 有效的对按顺序是：$(1,1), (1,2), (1,3), (1,4), (2,2), (2,3), (2,4), (3,3), (3,4), (4,4)$所以$T = 10$。 

### 示例 1

 假设$x = 6$。 

| 步骤| 价值|
 | --- | --- |
 | 总 T | 10 | 10
 | 随机 x | 6 |
 | 找到一个 | 2 |
 | 前缀 P(1) | 4 |
 | 偏移| 6 - 4 - 1 = 1 | 6 - 4 - 1 = 1
 | 乙| 2 + 1 = 3 |

 输出对是$(2,3)$。 

这证实了第二个块内的索引$(2,2),(2,3),(2,4)$已正确映射。 

### 示例 2

 假设$x = 1$。 

| 步骤| 价值|
 | --- | --- |
 | 总 T | 10 | 10
 | 随机 x | 1 |
 | 找到一个 | 1 |
 | 前缀 P(0) | 0 |
 | 偏移| 1 - 0 - 1 = 0 |
 | 乙| 1 |

 输出对是$(1,1)$。 

这验证了第一个块开始的边界的正确性。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import random

    # deterministic seed for reproducibility in tests
    random.seed(1)

    k = int(sys.stdin.readline())
    n = int(sys.stdin.readline())

    T = k * (k + 1) // 2

    def find_a(k, x):
        A = 2 * k + 1
        D = A * A - 8 * x
        s = int(D ** 0.5)
        a = (A - s) // 2
        a = max(1, min(k, a))
        return a

    def prefix(k, a):
        return a * k - a * (a - 1) // 2

    out = []
    for _ in range(n):
        x = random.randint(1, T)
        a = find_a(k, x)
        b = a + (x - prefix(k, a - 1) - 1)
        out.append(f"{a} {b}")

    return "\n".join(out)

# provided sample-like sanity check (structure only)
assert run("5\n3\n") is not None

# minimum size
assert run("2\n5\n") is not None

# uniform generation sanity (non-deterministic structure check)
assert len(run("10\n10\n").splitlines()) == 10

 ## 边缘情况

 当出现临界边缘情况时$x$正好落在两者之间的边界上$a$- 块。 例如，当$k = 4$，之间的过渡$a = 1$和$a = 2$发生在$x = 4$。 前缀和必须将此值精确映射到第一个块的最后一个元素，$(1,4)$，不是第二个块的第一个元素。 

修正步骤在`find_a`通过调整近似二次解来处理这个问题，直到完全满足前缀不等式。 这确保了即使存在整数舍入误差，所选的$a$始终对应于正确的块。 

另一个边缘情况是当$x = 1$。 该公式可能会产生$a = 0$或由于舍入而出现下冲，但夹紧和校正立即将其提升到$a = 1$，之后$b = 1$直接从零偏移开始。
