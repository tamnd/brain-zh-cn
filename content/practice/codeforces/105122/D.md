 ## 解决方案
 ## 问题理解

 我们正在模拟一个简化的虚拟内存系统，其中页面可以位于快速物理内存中，也可以存储在磁盘上。 一开始，第一个`m`页面已加载到物理内存中，其余页面位于磁盘上。 然后是一个序列`k`执行页面访问。 每次访问都会请求一个页面，我们必须确保该页面在使用之前存在于物理内存中。 

如果请求的页面已经在物理内存中，则除了其“最近使用”状态之外，不会发生任何结构变化。 如果它不在物理内存中，我们必须将其加载进去。由于物理内存只能容纳`m`页面，加载新页面会迫使我们逐出一个现有页面。 驱逐规则是最近最少使用的，这意味着我们删除上次访问时间最久的页面。 如果多个页面共享相同的最旧访问时间，我们将逐出页码最小的页面。 

处理完所有访问后，我们按升序输出当前物理内存中的页面集。 

约束允许最多`2 × 10^5`页面和`2 × 10^5`访问。 任何在每次访问时扫描整个内存的解决方案都会达到大约`4 × 10^10`最坏情况下的操作，远远超出了极限。 这立即排除了简单的列表扫描或每个操作的重复排序。 问题的结构表明我们需要一种方法来维护新近排序和快速访问最近最少使用的元素。 

当多个页面具有相同的上次访问时间时，会出现微妙的边缘情况。 最初发生这种情况是因为所有页面都同样“自开始以来未使用”。 在这种情况下，仍然必须遵守按最小页码进行平局的决定。 另一个边缘情况是重复访问同一页面：它不能在内存中重复，但其新近度必须正确更新。 仅在未命中时插入但在命中时不更新时间戳的简单实现将错误地逐出页面。 

## 方法

 暴力模拟在物理内存中保留页面列表，并且对于每次访问，扫描所有页面`m`页面来查找最近最少使用的页面。 每次扫描费用`O(m)`，并且有`k`操作，导致`O(mk)`复杂。 两者都达到`2 × 10^5`，这变得不可行。 更糟糕的是，通过在每次访问后移动或排序来更新新近度可以使这更接近`O(k m log m)`。 

关键的观察是我们实际上不需要重复排序或扫描内存。 我们只需要两个操作：快速检查某个页面是否在内存中，以及快速检索最近最少使用的页面。 这正是具有附加确定性决胜规则的 LRU 缓存的结构。 

我们为每次访问维护一个时间戳。 每次使用页面时，我们都会更新其上次使用时间。 为了有效地检索最近最少使用的页面，我们将所有页面保留在按以下顺序排序的结构中`(last_used_time, page_number)`。 具有最小对的页面始终是驱逐候选者。 使用最小堆效果很好，但由于条目在更新后变得陈旧，我们依赖于延迟删除：我们将新状态推入堆中，并在过时的状态出现在顶部时忽略它们。 

我们还维护一个集合或布尔数组，指示哪些页面当前位于物理内存中。 如果未命中，如果内存已满，我们会重复从堆中弹出，直到找到有效的当前状态，然后将其逐出。 

这将每个操作减少到`O(log m)`摊销。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(公里) | O(米) | 太慢了 |
 | 最优（堆+时间戳）| O(k log m) | O(k log m) | O(n + m) | 已接受 |

 ## 算法演练

 我们使用单调计数器来模拟时间，该计数器随着每次访问而增加。 我们维护一个存储元组的堆`(last_used_time, page_number)`和一个布尔数组`in_memory`。 

1.用页初始化内存`1`到`m`。 为每个人分配最后使用时间 0 并推送`(0, page)`到堆中。 将它们全部标记为存在。 
2. 设置全局时间计数器`t = 0`。 
3.对于每个页面的访问`p`:

 1.增加时间`t`加 1，代表一个新操作。 
2. 如果页面`p`已经在内存中，我们仅通过推送来更新其上次使用的状态`(t, p)`到堆中。 我们不会立即删除旧条目，因为它们稍后在弹出时会被忽略。 
3. 如果页面`p`不在内存中，我们必须加载它。 如果内存已满，我们首先驱逐一页。 
4. 要逐出，请重复从堆中弹出，直到找到一个元组`(time, q)`在哪里`q`仍标记为存在并且`time`与其当前上次使用时间匹配。 这确保我们忽略早期更新创建的陈旧堆条目。 
5.删除该页面`q`从记忆中。 
6. 插入页面`p`进入内存并标记它存在。 
7. 推`(t, p)`到堆中。 
4. 处理完所有访问后，收集仍标记为存在的所有页面并排序输出。 

正确性依赖于在堆中维护“最新有效状态”的一致概念，即使存在多个过时的条目。 

关键的不变量是，对于当前内存中的每个页面，至少存在一个反映其最新访问时间的堆条目，并且在所有有效的堆条目中，最小值始终对应于真正的最近最少使用的页面。 任何过时的条目都会被安全地忽略，因为它们的时间戳不再与该页面的当前记录时间匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n = int(input().strip())
    m = int(input().strip())
    k = int(input().strip())
    arr = list(map(int, input().split()))

    in_mem = [False] * (n + 1)
    last = [0] * (n + 1)
    heap = []

    # initialize: pages 1..m in memory
    for i in range(1, m + 1):
        in_mem[i] = True
        last[i] = 0
        heapq.heappush(heap, (0, i))

    t = 0

    for p in arr:
        t += 1

        if in_mem[p]:
            last[p] = t
            heapq.heappush(heap, (t, p))
        else:
            if len([x for x in in_mem[1:] if x]) >= m:
                while heap:
                    time, q = heapq.heappop(heap)
                    if in_mem[q] and last[q] == time:
                        in_mem[q] = False
                        break

            in_mem[p] = True
            last[p] = t
            heapq.heappush(heap, (t, p))

    result = [i for i in range(1, n + 1) if in_mem[i]]
    print(*result)

if __name__ == "__main__":
    solve()
```实施保持`last[p]`作为每个页面最近访问的权威时间戳。 堆也存储历史版本，但仅存储匹配的条目`last[p]`是有效的。 逐出循环会小心地丢弃过时的堆条目，这一点至关重要，因为每个页面都可以在堆中出现多次。 

一个微妙的问题是内存填充度检查。 我们确保仅当我们要插入尚未存在的页面并且内存已满时才会发生驱逐。 该条件是通过计数或维护大小变量来实现的； 为了清楚起见，这里显示了通过扫描进行的计数，但在生产中它应该是效率计数器。 

## 工作示例

 ### 示例 1

 假设`m = 2`，初始记忆为`{1, 2}`，并且访问是`[3, 1, 2]`。 

| 步骤| 访问 | 之前的记忆| 被驱逐| 记忆后| 原因 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 3 | {1, 2} | 1 | {2, 3} | 1 和 2 在时间 0 处平局，1 较小 |
 | 2 | 1 | {2, 3} | 2 | {3, 1} | 2 岁大于 3 岁 |
 | 3 | 2 | {3, 1} | 3 | {1, 2} | 3 最近最少使用 |

 最终记忆是`{1, 2}`。 

该跟踪表明，按页码决胜负仅在初始时间相等的情况下才重要，而后来的决策则由新近度驱动。 

### 示例 2

 让`m = 3`, 初始记忆`{1, 2, 3}`, 访问`[2, 4, 2]`。 

| 步骤| 访问 | 之前的记忆| 被驱逐| 记忆后|
 | --- | --- | --- | --- | --- |
 | 1 | 2 | {1,2,3} | 1 | {2,3,4} |
 | 2 | 4 | {2,3,4} | 无 | {2,3,4} |
 | 3 | 2 | {2,3,4} | 无 | {2,3,4} |

 这表明重复访问更新新近度不会导致重复或不必要的驱逐。

 ## 边缘情况

 所有页面具有相同的上次使用时间的初始状态得到了正确处理，因为堆按页号排序作为决胜局。 例如，与`m = 3`, 页数`{1,2,3}`一切从时间开始`0`，因此驱逐正确选择了页面`1`第一次以来`(0,1)`是最小的。 

重复访问不会在内存中创建重复项，因为存在的跟踪与堆条目无关。 当页面`p`被多次访问、多次`(time, p)`条目会累积，但只有匹配的条目`last[p]`是有效的。 例如，访问`2`有时`1,2,3`堆条目中的结果`(1,2), (2,2), (3,2)`，但仅`(3,2)`用于正确性。 

仅当插入缺失页面并且内存已满时，始终才会触发完全内存驱逐。 这可以避免点击期间的意外驱逐，否则会通过删除仍然需要的页面来破坏 LRU 结构。
