 ## 解决方案
 ## 问题理解

 我们有一组固定的学生注册号，每个号由六位数字组成（允许使用前导零，因此像这样的数字`000123`有效且不同于`123000`）。 在此预处理步骤之后，我们收到多个查询，其中每个查询都是代表管理员请求的另一个六位数字。 

对于每个查询号码，我们必须从给定的集合中选择一个学生号码，以最大化称为数字模和的特殊数字运算。 此操作是针对每个数字位置独立定义的：对于六个位置中的每一个，我们将学号和查询编号的数字相加，然后将结果模 10。得到的六位数字形成一个新数字，其值就是我们想要最大化的分数。 

因此，对于每个查询，任务是找到产生字典顺序最佳（或数值最大）按数字求和模 10 结果的学生编号。 

限制很严格：最多 100,000 个学生编号和 100,000 个查询，每次比较涉及六位数字的位置。 一个简单的解决方案是检查每个学生的每个查询，在最坏的情况下将执行大约 10^10 次比较，这远远超出了一秒的限制。 这立即排除了任何 O(NQ) 方法。 

一个微妙但重要的边缘情况是由前导零引起的。 由于数字是长度为 6 的固定宽度字符串，因此将它们视为整数会破坏位置意义。 例如，比较`000999`和`999000`数字在这里没有意义，因为操作是按数字进行的。 任何正确的解决方案都必须将它们视为字符串或数字数组。 

另一个重要的极端情况是，该操作在通常的数字意义上不是单调的。 对于给定的查询，正常整数值较大的学生编号不一定会产生较大的数字模和结果。 例如，使用查询`999999`，最佳匹配是`000000`因为每个数字和变成 9，而“大”学生喜欢`123456`产生像这样的数字和`0,1,2,...`，这更糟糕。 

## 方法

 暴力方法很简单：对于每个查询，迭代所有学生编号，计算所有六个位置的数字总和模 10，并跟踪最佳结果。 每次评估的成本为 O(6)，因此总复杂度为 O(NQ)。 如果两个维度均为 100,000，则每个查询批次大约需要 6 亿位数字操作，总共会导致数百亿次，这太慢了。 

关键的观察是每个数字位置在贡献方面是独立的，但限制是我们必须选择一个学生编号，其组合数字向量同时在所有六个位置上都是最佳的。 这是一个经典的“多维数字变换”优化问题。 

由于每个号码只有 6 位数字长，因此我们可以将每个学号视为 6 级数字空间中的一条路径。 关键的见解是针对每种可能的查询模式预先计算学生的最佳响应。 因为每个数字都在`[0..9]`，理论上只有 10^6 个可能的查询，但这太大了，无法直接存储。 然而，我们可以利用数字 DP 风格的想法：我们可以构建一个学生编号字典树，并使用逐位传播预先计算每个数字配置的最佳候选者。 

更有效的观点是考虑对于固定的查询数字，每个学生数字独立地对结果数字做出贡献。 因此，对于每个查询，我们希望最大化一小组候选者的向量得分，这建议维护一个可以有效回答“数字评分下的最佳匹配”的结构。 实现此目的的一种实用方法是，对于每个位置的每个可能的查询数字，预先计算最佳学生数字贡献，从而有效地将搜索减少为结构化查找而不是全面扫描。 

我们可以使用学生编号字典有效地实现这一点，其中每个节点聚合从该子树可到达的最佳候选者。 然后，每个查询都会逐位处理，始终遵循最大化结果数字总和的转换，有效地将每个查询变成 O(6 * 10) 探索而不是 O(N)。

 | ---| ---| ---| ---|
 | 蛮力 | O(NQ) | O(N) | 太慢了 |
 | Trie + 贪心数字 DP | O((N + Q) · 6 · 10) | O(N·6) | 已接受 |

 ## 算法演练

 我们根据所有学生编号构建一个数字特里树，然后用它来贪婪地回答查询，同时尊重数字最大化。 

1. 将每个学生编号插入一个字典树中，其中每个级别对应于从最重要到最不重要的一位数字。 每个节点存储对其子节点的引用以及有关最佳可达值的可选元数据。 这种结构确保我们可以按数字顺序遍历所有候选者。 
2. 对于每个查询，从 trie 的根部开始，从左到右处理数字。 在位置 i，我们查看查询数字 d 并评估从 0 到 9 到子数字 k 的所有可能转换。 
3. 对于每个候选数字 k，我们计算所得的数字贡献`(d + k) % 10`。 我们希望最大化这个值，同时也确保路径存在于 trie 中。 所以我们只考虑有效的孩子。 
4. 在所有有效的孩子中，我们选择产生最高的数字 k`(d + k) % 10`。 如果多个数字产生相同的值，我们会使用预先计算的规则来打破平局，例如字典中最大的剩余后缀或存储的最佳叶子。 
5. 移动到选定的子节点并将 k 附加到生成的学生编号中。 
6. 重复此过程，直到处理完所有六位数字，生成一个完整的候选学生编号，该编号可最大化按数字求模和。 

### 为什么它有效

 在每个数字位置，除了完成有效学生编号的可行性之外，对最终分数的贡献与未来的数字无关。 trie 保证了可行性，并且贪婪选择是有效的，因为目标函数分解为每个数字有界贡献的总和，没有进位交互。 由于每个数字位置在所有全局有效候选者中进行局部优化，因此任何后续决策都不能追溯性地改进先前数字的模块化贡献，从而使得贪婪选择在特里树约束下全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Node:
    __slots__ = ("child",)
    def __init__(self):
        self.child = {}

def insert(root, s):
    node = root
    for ch in s:
        if ch not in node.child:
            node.child[ch] = Node()
        node = node.child[ch]

def best_match(root, q):
    node = root
    res = []
    for ch in q:
        dq = ord(ch) - 48

        best_digit = None
        best_val = -1

        for k in node.child.keys():
            dk = ord(k) - 48
            val = (dq + dk) % 10
            if val > best_val or (val == best_val and k > best_digit):
                best_val = val
                best_digit = k

        res.append(best_digit)
        node = node.child[best_digit]

    return "".join(res)

def main():
    n = int(input())
    root = Node()

    for _ in range(n):
        s = input().strip()
        insert(root, s)

    q = int(input())
    out = []

    for _ in range(q):
        query = input().strip()
        out.append(best_match(root, query))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```trie 结构逐位存储所有学生编号。 每个查询都通过遍历 trie 并根据模和规则在每个级别本地选择最佳数字来解决。 当多个候选人产生相同的数字分数时，平局决胜可确保确定性输出。 

一个微妙的实现细节是我们只迭代每个节点的现有子节点，这可以保持转换的效率。 另一个是我们依靠字符串数字而不是整数来保留前导零和位置正确性。

 ## 边缘情况

 一个关键的边缘情况是正确处理前导零。 例如，如果一个学生是`000999`，将其视为整数 999 会错误地对齐数字并破坏 trie 结构。 该算法存储原始字符，因此路径`0 → 0 → 0 → 9 → 9 → 9`被准确地保留，并且查询按数字对齐而不会失真。 

另一个边缘情况是当多个数字产生相同的模和时打破平局。 例如，如果查询数字为 5，则学生数字 5 和 15 mod 10 等值不可能都存在，但数字 5 和 15 不存在，因此必须一致地解决有效数字之间的联系。 当值相等时，特里结构通过优先选择较大的数字键来确保确定性选择。 

最后，当所有学生共享一个前缀时，trie 路径对于前面的数字变得线性。 该算法正确地继续而不分支，直到出现分歧，确保在遍历过程中不会人为丢失候选者。
