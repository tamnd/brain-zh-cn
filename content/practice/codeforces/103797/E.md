 ## 解决方案
 ## 问题理解

 任务是模拟一组学生如何在公共汽车上占据座位，并累计直到每个人都就座为止所花费的总时间。 公交车可以看作是一个有 N 排的网格，每排包含四个固定座位：两个靠窗座位和两个靠过道座位。 学生按照字符串给出的固定顺序一一进入，每个学生要么表现正常，要么表现为内向的人，座位偏好略有不同。 

物理布局的限制很重要：学生总是尽量坐在离入口尽可能远的地方。 这意味着每个决定都是从从最后一排向第一排扫描并选择满足座位条件的第一排开始的。 一旦选择了一行，确切的座位类型决定了额外的就座时间，而穿过行则贡献与行索引成比例的行进时间。 

由于 N 最多为 30，而学生人数最多为 120，因此对行和学生进行直接模拟很容易足够快。 在这些限制下，任何 O(N·|S|) 甚至 O(N²·|S|) 的方法仍然可以忽略不计。 

主要的微妙之处在于，“内向”的学生一般并不只是简单地喜欢靠窗的座位。 他们专门寻找仍然有空闲靠窗座位的最远一排，即使该排在其他位置也有可用的过道，或者即使较近的一排具有更好的整体配置。 只有当任何地方都没有靠窗的座位时，它们才会恢复正常行为。 

当将内向者视为对任何靠窗座位的全局贪婪而不动态地重新评估行选择时，就会出现天真的错误。 另一个常见的失败是忘记，内向的人坐在靠窗的座位后，同一排靠过道的座位对后来的学生仍然有效，这可能会以不明显的方式改变未来的决定。 

例如，假设有两排，并且第 1 排中只剩下一个靠窗的座位。如果内向的人在所有排都部分坐满后进入，则即使第 2 排仍然有靠过道的座位，他们也必须选择第 1 排。 粗心的实施可能会选择最远的一排有任何座位，这对于内向的人来说是错误的。 

## 方法

 暴力模拟是思考问题最直接的方法。 我们维护每排每个座位的状态，对于每个即将入学的学生，从最远的排向入口扫描，根据他们的规则找到有效的排。 对于普通学生来说，我们选择第一排有空闲座位的，然后我们选择靠窗的，然后再选择过道的。 对于性格内向的人，我们首先尝试找一排有空位的靠窗座位； 只有当不存在时我们才会回到正常规则。 

这是正确的，因为它完全反映了问题中描述的过程。 然而，如果扩大规模，效率就会变得很低：每个学生可能需要扫描所有 N 行，并且每次扫描都可能涉及检查座位可用性。 这导致大约 O(|S|·N) 操作，这在这里仍然没问题，但如果在没有结构的情况下实现，概念上的暴力很容易降级为 O(|S|·N·seat_checks)。 

关键的观察结果是 N 非常小并且固定，因此除了干净的状态表示之外我们不需要任何优化。 每排都可以使用简单的计数器来表示剩余的靠窗和过道座位。 这消除了明确跟踪各个座位的需要，并简化了行选择逻辑。 

因此，该解决方案简化为简单的贪婪模拟，并仔细定义了行选择和座位分配的优先级规则。

 ## 算法演练

我们为每排维护两个值：还有多少个靠窗座位，以及还有多少个过道座位。 最初，每行都是 2。 

每个学生都按顺序处理，我们根据他们最终使用的排和座位类型计算他们的步行时间和座位时间。 

### 步骤

 1. 读出行数和学生的顺序。 初始化每排两个靠窗座位和两个过道座位。 
2. 按顺序对每个学生判断他们是正常的还是内向的。 
3. 如果学生性格内向，则从最远到最近扫描各行，找到仍然至少有一个靠窗座位的第一排。 如果存在这样的一排，请为他们分配一个靠窗的座位。 否则，就将他们视为普通学生。 
4. 如果学生是正常人（或性格内向，没有可用的窗户），则从最远到最近扫描各行，找到有可用座位的第一排。 
5. 在所选行内，分配一个靠窗的座位（如果有）； 否则分配一个靠过道的座位。 
6. 将穿过入口的每一行的行进时间计算为 2 秒，与 (行索引 − 1) 成正比。 
7.增加就座时间：靠窗座位10秒，过道座位5秒。 
8. 更新该排的剩余座位数并累计总时间。 

### 为什么它有效

 在每一步中，该算法都遵循登机规则所描述的严格优先级顺序。 最远的行偏好是通过从末尾扫描来强制执行的，而座位类型偏好是在所选行内本地强制执行的。 由于每个决策仅取决于当前状态，并且不需要重新考虑之前的分配，因此模拟保留了正确性。 州代表保证席位一旦被占用就不能重复使用，并且每个学生独立地应用相同的确定性选择规则。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    # each row: [window_left, aisle_left]
    rows = [[2, 2] for _ in range(n)]
    total = 0

    for ch in s:
        chosen_row = -1
        seat_type = None  # 'W' or 'A'

        if ch == 'I':
            # try to find farthest row with a window seat
            for i in range(n - 1, -1, -1):
                if rows[i][0] > 0:
                    chosen_row = i
                    seat_type = 'W'
                    break

            # fallback to normal if no window seats exist
            if chosen_row == -1:
                for i in range(n - 1, -1, -1):
                    if rows[i][0] + rows[i][1] > 0:
                        chosen_row = i
                        if rows[i][0] > 0:
                            seat_type = 'W'
                        else:
                            seat_type = 'A'
                        break

        else:
            # normal student: farthest row with any seat
            for i in range(n - 1, -1, -1):
                if rows[i][0] + rows[i][1] > 0:
                    chosen_row = i
                    if rows[i][0] > 0:
                        seat_type = 'W'
                    else:
                        seat_type = 'A'
                    break

        # update state and time
        if seat_type == 'W':
            rows[chosen_row][0] -= 1
            total += (chosen_row) * 2 + 10
        else:
            rows[chosen_row][1] -= 1
            total += (chosen_row) * 2 + 5

    print(total)

if __name__ == "__main__":
    solve()
```该实现仅使用每行计数器来保持总线状态紧凑。 行选择是通过反向扫描来完成的，它直接编码“最远行优先”规则，无需任何额外的数据结构。 时间计算使用行索引作为距入口的距离，其中第 0 行与入口相邻，因此步行成本变为`2 * row_index`。 

一个微妙的点是内向者的后退：一旦任何地方都没有靠窗的座位，他们的行为就会变得与正常学生相同。 这是明确实现的，以便窗口耗尽不会破坏选择逻辑。

 ## 边缘情况

 一种重要的边缘情况是，当所有靠窗的座位都空了之后，内向的人就会出现。 在这种情况下，算法必须立即切换到正常行为，而不尝试分配靠窗的座位。 例如，在具有输入的单行总线中`WWII`（对于这个抽象，将 W 解释为 E 等价），一旦两个靠窗的座位都被占用，剩下的内向者必须占据过道的座位，并且后备逻辑确保他们不再尝试无效的窗口选择。 

另一个边缘情况是，早期内向的人部分占据了不同排的靠窗座位。 由于每次选择总是从最远的行开始扫描，因此即使前面的行已经包含混合座位，后来的学生可能仍然更喜欢不同的行。 基于状态的扫描保证每个决策反映当前的全局配置而不是任何本地历史记录。
