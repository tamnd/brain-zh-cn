 ## 解决方案
 ## 问题理解

 我们得到一个长度的数组$N$，最初用零填充，代表每个学生当前拥有多少子弹。 在线执行两种类型的操作。 

第一个操作是范围更新：给定一个段$[l, r]$和一个值$x$，该部分的每个学生都会收到$x$额外的子弹。 重要的是，这不是替代而是增量，并且随着时间的推移会累积多次更新。 

第二个操作是点查询：对于给定的位置$p$，我们被问到该位置当前的子弹数量是否至少为$x$。 如果是，我们会积极地从概念上做出回应，学生会立即准确地使用$x$子弹，减少他的储存量$x$。 否则我们会做出消极的反应，状态也不会改变。 

限制条件$N, Q \le 10^5$迫使我们高效地支持范围添加和点查询。 任何为每个操作重新计算一定范围内的值的解决方案都会导致$O(NQ)$，这太慢了。 即使维护一个简单的数组也是不够的，因为每次操作重复的范围更新仍然是线性的。 

一个微妙的边缘情况是成功的查询会修改数组。 例如，如果一个学生恰好有$x$子弹并且被查询两次，第二次查询必须返回负数，因为第一次查询消耗了子弹。 不保持递减的天真的“仅范围和”方法会默默地过度计算。 

## 方法

 暴力破解的想法很简单：显式维护数组。 如需更新$[l, r, x]$，我们迭代该段中的所有索引并添加$x$。 对于位置的查询$p$，我们只需检查该值并减去$x$如果可能的话。 

这是正确的，但每次更新都会花费$O(r-l+1)$，这变成$O(N)$在最坏的情况下。 和$10^5$操作，总复杂度退化为$10^{10}$，这是不可行的。 

关键的观察是我们不需要明确地知道数组的每个元素。 我们只需要两个功能：在范围内添加值和查询单个索引。 这是适用具有惰性传播的 Fenwick 树或线段树的经典设置。 

然而，我们可以进一步简化。 可以使用差异数组来处理具有点查询结构的范围添加：我们维护一个数组$diff$这样添加$x$到$[l, r]$翻译为$diff[l] += x$和$diff[r+1] -= x$。 然后是position处的值$p$前缀总和为$p$。 这将范围更新减少到$O(1)$并将查询指向前缀和。 

唯一剩下的复杂之处是查询不是纯粹的读取。 当查询成功时，我们必须减去$x$永久。 这也可以建模为底层数组上的点更新，这会转换为差异数组上的两次更新。 

因此，最终的解决方案将差异数组与芬威克树结合起来，以有效地支持前缀和和点修改。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(NQ)$|$O(N)$| 太慢了 |
 | Fenwick Tree（差异数组+更新）|$O(Q \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们在子弹计数的差异数组表示上维护一个芬威克树。 

1.初始化一棵大小为Fenwick Tree的$N+1$，最初全为零。 这表示一个差异数组，其中实际值来自前缀和。 
2. 处理更新$!\ l\ r\ x$，我们添加$x$在位置$l$并减去$x$在位置$r+1$如果它在范围内。 这确保了每个前缀总和超出$l$包括增量，以及超出的一切$r$将其取消。 
3. 回答该位置的查询$p$，我们计算前缀和$p$，给出该学生当前的子弹数。 
4. 如果该值至少为$x$，我们输出 success 并立即减去$x$从位置$p$。 这是 Fenwick 树中的点更新，实现为添加$-x$在$p$和$+x$在$p+1$。 
5. 如果该值小于$x$，我们输出失败并且不修改任何内容。 

关键的设计选择是范围更新和点修改都以相同的原语表示：差异数组更新，芬威克树可以有效地处理这一点。 

### 为什么它有效

 差异数组确保原始数组的每个元素都表示为前缀和。 任何范围添加都以受控方式精确影响前缀结构：它引入了一个步骤$l$并下降到$r+1$，因此每个指标都反映了正确的累计值。 

点减法的工作原理是对称的：将单个位置减少$x$相当于在该索引处引入负单位步骤并在之后恢复它，这保留了所有其他位置的正确性。 由于所有操作都是前缀贡献的线性组合，因此芬威克树不变量保证每次更新后的一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 2)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

n, q = map(int, input().split())
fw = Fenwick(n)

for _ in range(q):
    tmp = input().split()
    if tmp[0] == '?':
        p = int(tmp[1])
        x = int(tmp[2])
        cur = fw.sum(p)
        if cur >= x:
            print("yes sir")
            fw.add(p, -x)
        else:
            print("negative")
    else:
        l = int(tmp[1])
        r = int(tmp[2])
        x = int(tmp[3])
        fw.add(l, x)
        if r + 1 <= n:
            fw.add(r + 1, -x)
```Fenwick 树用作差异数组上的前缀和引擎。 这`add`函数通过将范围更新和点减少映射到局部增量来实现范围更新和点减少。 

对于范围更新，我们应用标准差异技巧：在左端点处递增，并在右端点之后递减。 这可确保该范围内的任何前缀和仅反映一次添加值。 

对于查询，我们计算位置处的前缀和`p`。 如果足够，我们立即通过执行另一对 Fenwick 更新来减少该点的值，有效地将其视为长度为 1 的局部范围更新。 

必须注意边界`r + 1`，不得超过 Fenwick 尺寸。 此外，所有索引都是从 1 开始的，以完全匹配 Fenwick 的实现。

 ## 边缘情况

 一个关键的边缘情况是在同一位置上重复成功的查询。 假设一名学生收到 5 颗子弹，然后被要求两次提供 3 颗子弹。 第一个查询成功并将值减少到 2，而第二个查询失败。 基于 Fenwick 的实现正确地反映了这一点，因为减法是作为点更新立即应用的，而不仅仅是逻辑上假设的。 

另一个边缘情况是范围更新以最后一个索引结束。 例如，申请$[1, N]$不得尝试更新$N+1$在芬威克树中。 该实现通过边界检查明确地保护这一点，确保不会发生超出范围的更新，同时保持前缀和的正确性。
