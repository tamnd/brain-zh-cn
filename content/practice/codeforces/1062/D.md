 ## 解决方案
 ## 问题理解

 我们得到一组绝对值从 2 到 n 范围内的整数，每个数字也有其负数对应项。 将此范围内的每个整数 i 视为图中的一个节点，包括 i 和 -i。 

如果一个节点是另一个节点的非平凡整数倍，则允许在两个节点 a 和 b 之间移动。 具体来说，必须存在绝对值至少为2的整数x，使得a乘以x得到b，或者b乘以x得到a。 每个这样的移动都有一个等于 x 绝对值的权重，它恰好是两个数字之间的比率。 

我们可以从任何节点开始，沿着这些允许的移动走，但是一对节点之间的每个移动最多只能使用一次。 目标是选择一系列这样的动作，使收集到的总重量最大化。 

约束允许 n 最大为 100000，这会立即排除任何检查所有数字对的解决方案。 尝试考虑所有对或动态模拟路径的简单方法将需要 n 平方检查或更糟的数量级，这远远超出了两秒内可以运行的范围。 问题的结构表明我们必须利用除数和倍数之间的算术规律，而不是将其视为通用图。 

当 n 非常小时，会出现微妙的边缘情况。 如果 n 等于 2，则不存在绝对值大于 1 的有效乘数来将值保持在允许的范围内，因此不存在移动并且答案必须为 0。当 n 为素数时，会出现另一种极端情况。 尽管存在数字，但有效的倍数很少，任何错误的假设（例如“每个节点至少贡献一条边”）都会被过度计算。 

## 方法

 该问题的强力解释构建了一个图表，其中检查每对数字 a 和 b，如果一个数字除另一个数字的绝对值至少为 2，则添加一条边。 然后我们尝试找到一条边不被重用的最大权重路径。 这已经很昂贵了，因为仅构建图就需要检查所有 O(n^2) 对。 

即使图已构建，找到最佳路径也不是标准的最短路径或 MST 问题。 它类似于在一般图中找到最大权重轨迹，这需要指数探索或对所用边的状态进行复杂的 DP。 随着 n 的增长，这很快变得不可行。 

关键的观察是该图不是任意的。 每条边都来自除数关系。 对于固定的 i，边仅将 i 连接到其倍数 2i、3i、4i 等。 这意味着整个结构可以通过除数干净地分解，而不是成为一个密集的网络。 

第二个结构简化来自正数和负数之间的对称性。 i 和 ki 之间的每条有效边也存在于 -i 和 -ki 之间，具有相同的权重。 这会复制整个结构，而不会在正组件和负组件之间产生相互作用。 所以总的答案恰好是正方贡献的两倍。 

现在只关注正整数。 对于每个 i，每个多个 ki 都会贡献权重为 k 的边的定向选择。 至关重要的是，这些边不会干扰不同的 i 值，因为每条边都是由 (i, ki) 对唯一确定的。 所以我们不再解决路径问题，而是简单地对所有有效边求和。 

对于固定的 i，有效的 k 值从 2 开始一直到 Floor(n / i)。 每个这样的 k 都对总数贡献 k。 对所有 i 求和给出全部正贡献，加倍则给出负贡献。

 | --- | --- | --- | --- |
 | 蛮力 | O(n² + 复杂轨迹搜索) | O(n²) | 太慢了 |
 | 最佳| O(n log n) | O(n log n) | O(1) | O(1) | 已接受 |

## 算法演练

 1. 迭代从 2 到 n 的每个整数 i。 每个 i 充当基值，其倍数定义有效的转换。 这确保我们从较小的端点开始考虑每个可能的除数关系一次。 
2. 对于每个 i，计算在 n 之前的范围内存在多少个 i 的倍数。 令 m = 楼层(n / i)。 这些对应于可能的乘数 k，使得 i * k 有效。 
3. 忽略 k = 1，因为转换需要 |x| > 1，因此只有从 2 到 m 的 k 是有效的。 每个这样的 k 代表从 i 到 i * k 的有效边。 
4. 通过对所有有效乘数求和来计算 i 的贡献：2 + 3 + ... + m。 这是一个算术级数，因此我们使用公式 sum(1..m) minus 1 在 O(1) 中计算它。 
5. 将此贡献添加到所有 i 的运行总计中。 这会累积正数图中的所有边。 
6. 处理完所有 i 后，将总数乘以 2 以考虑对称负数图。 

### 为什么它有效

 每个有效的移动都唯一对应于 k ≥ 2 的一对 (i, ki)。不同 i 值之间不共享边，因此计算每个 i 的贡献不会重复计算或错过任何转换。 负侧是正侧的同构副本，因此加倍是精确的。 由于在此计数视图中每条边都是独立的，因此所有边权重的总和就是可实现的最大分数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    if n < 2:
        print(0)
        return

    total = 0

    for i in range(2, n + 1):
        m = n // i
        if m >= 2:
            # sum of k from 2..m = (1..m) - 1
            total += (m * (m + 1)) // 2 - 1

    print(total * 2)

if __name__ == "__main__":
    solve()
```该实现直接反映了基于除数的分解。 i 上的循环确保考虑每个基数。 除法 n // i 计算倍数延伸的距离。 算术级数公式避免显式迭代 k 值，这对于效率至关重要。 最后，乘以 2 即可得到镜像负图。 

一个常见的错误是忘记排除 k = 1，这会错误地计算无效的自变换。 另一个是忘记正数和负数之间的对称性，这将产生正确答案的一半。

 ## 边缘情况

 对于 n = 2，不存在有效的 k ≥ 2 使得 i * k 保持在界限内。 该算法正确地生成零，因为每个 m = 1 并且没有添加任何贡献。 

对于像 n = 3 或 n = 5 这样的小素数，每个 i 都没有有效的倍数，因此循环没有任何贡献。 该结构确保这些节点不会错误地贡献边缘。 

对于像 n = 100000 这样的高度复合数，内部计算在每个 i 中保持恒定时间，避免任何超过倍数的扩展并保持运行时稳定。
