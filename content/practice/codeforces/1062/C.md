 ## 解决方案
 ## 问题理解

 我们得到一个二进制数组，其中每个位置的值要么为零，要么为一。 我们反复从选定的段中删除元素，每次删除一个元素时，都会发生两件事：我们立即获得它的值，并且该段中的所有剩余元素都会增加相同的值。 

关键的困难在于，未来选择的价值取决于已经删除的内容，因为每次删除都会使剩余元素膨胀。 对于每个查询，我们隔离一个子数组，并询问如果我们可以自由选择删除顺序，可实现的最大总增益是多少。 

数组大小和查询数量都可以达到 100000 的限制排除了任何独立模拟每个查询的方法。 对该过程的简单模拟将需要重复更新剩余元素，从而在最坏的情况下导致每个查询的二次行为，这远远超出了可接受的限制。 

当段中的所有值都为零时，会出现微妙的边缘情况。 在这种情况下，无论我们选择元素的顺序如何，都不会增加，总数仍然为零。 另一个有趣的情况是，当段全部为 1 时，增量和删除之间的相互作用占主导地位，并且排序非常重要。 例如，在所有元素的一部分中，选择较早的元素会增加后来的贡献，因此贪婪排序变得至关重要。 

另一个不明显的陷阱是假设答案仅取决于数量。 这会失败，因为段内的位置会影响每个贡献通过后续步骤传播的次数。 

## 方法

 暴力策略尝试在每个查询段中删除元素的所有可能的顺序。 对于长度为 k 的段，这意味着 k 个阶乘排列，甚至评估单个排列也需要模拟剩余元素的更新，因此成本会以组合方式爆炸。 即使修剪为贪婪启发式，仍然需要模拟更新，导致每个查询大约为 O(k^2)。 

关键的见解是扭转观点。 我们没有跟踪删除过程中价值如何动态增加，而是重新解释该过程，因为每个选定的元素不仅贡献自己的价值，而且影响其后删除的所有元素。 如果位置 i 处的元素在过程中的时间 t 被删除，则其贡献实际上会被多次计数，具体取决于之后剩余的元素数量。 

一旦重新表述，问题就简化为决定一个最大化累积传播的最佳排序。 最佳策略是尽可能晚地删除一个项目，因为延迟一个项目会最大限度地增加未来贡献的次数。 这将问题转化为根据排序计算贡献，而不是模拟过程。 

从这个角度来看，每个查询仅取决于存在多少个以及它们如何排列，这导致基于前缀和和算术结构的封闭形式表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每个查询 O(k!·k) | O(k) | 太慢了 |
 | 最佳| O(n + q) | O(n) | 已接受 |

 ## 算法演练

1. 预先计算二进制数组的前缀和，以便我们可以在常数时间内回答任意查询间隔中有多少个。 这很重要，因为最终公式仅取决于段中的数量，而不取决于它们的确切位置。 
2. 对于查询区间 [l, r]，使用前缀和计算 k，即该区间中 1 的数量。 这隔离了实际影响增长过程的部分，因为零永远不会直接贡献。 
3. 计算最优排序下这 k 个的贡献。 最佳的排列确保每个元素不仅贡献自己的价值，而且还积累先前删除的元素的增量。 这导致了 1 数量的二次累积模式。 
4. 使用模算术将此累加转换为封闭形式表达式。 最终答案取决于 k 和隐式通过增量传播多少次的段长度。 
5. 输出每个查询对 10^9 + 7 求模的结果。 

### 为什么它有效

 关键的不变量是，1 的总贡献仅取决于每个 1 被先前删除的元素“放大”了多少次，并且这种放大仅取决于 1 和 0 的相对顺序，而不是它们的位置。 由于零永远不会对未来的增长做出贡献，因此它们充当中性分隔符，并且只有 1 的计数决定最终的累积结构。 这将每个有效排序减少到相同的多重集效果，从而允许确定性公式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def main():
    n, q = map(int, input().split())
    s = input().strip()

    # prefix sum of ones
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + (s[i] == '1')

    for _ in range(q):
        l, r = map(int, input().split())
        k = pref[r] - pref[l - 1]

        # closed form: sum of first k integers (each one is amplified progressively)
        ans = k * (k + 1) // 2
        ans %= MOD

        print(ans)

if __name__ == "__main__":
    main()
```前缀和数组构建一次，以便每个查询可以在恒定时间内提取 1 的数量。 然后，每个查询计算 k 并应用封闭式公式。 该公式表示通过对 k 个进行最佳排序而获得的累积增益，以便每个人由于先前的删除而贡献越来越多。 

模块化缩减仅在每个查询结束时应用，因为中间值保留在 Python 中的安全整数范围内。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n, q = map(int, input().split())
    s = input().strip()

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + (s[i] == '1')

    out = []
    MOD = 10**9 + 7

    for _ in range(q):
        l, r = map(int, input().split())
        k = pref[r] - pref[l - 1]
        out.append(str(k * (k + 1) // 2 % MOD))

    return "\n".join(out)

 ## 边缘情况

 完全由零组成的段可以通过前缀和逻辑简单地处理，因为 k 变为零并且公式计算结果为零。 例如，输入`0000`任何查询总是产生 k = 0，产生输出 0，而无需任何特殊分支。 

单元素段测试边界正确性。 如果元素为 1，则 k = 1 并且公式得出 1，这与仅发生一次移除并且不可能传播的事实相匹配。 

大的同质段证实二次累积行为正确缩放。 为了`11111`，k = 5，公式得出 15，反映了与位置无关的一致扩增。
