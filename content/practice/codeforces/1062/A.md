 ## 解决方案
 ## 问题理解

 我们得到一个严格递增的整数数组，全部在 1 到 1000 之间。允许有人恰好擦除一个连续的元素块，在序列中留下一个间隙。 其余元素保留在原来的位置。 

擦除后，仅使用剩余值以及原始数组严格以 1 和 1000 为界的值递增这一事实，丢失的段仍然必须是唯一可恢复的。 

任务是找到可以删除的最长连续段，同时仍然保证擦除的值是唯一确定的。 

关键的约束是重建的唯一性。 如果多个不同的递增补全适合该间隙，则该删除无效。 

由于 n 至多为 100，因此对可能的间隔进行二次甚至三次检查是很容易实现的。 任何超出 O(n^3) 的事情都是不必要的，但仍然可能是安全的，但我们可以通过推理模糊性是如何产生的来做得更好。 

当剩余的边界值为多个严格递增的整数序列留下太多的数字空间时，就会出现微妙的失败情况。 

例如，如果我们删除中间段，但左边界值和右边界值之间的间隙足够大，足以容纳多个严格增加的缺失长度填充，则重建不是唯一的。 

另一种边缘情况发生在边界处。 如果我们从头或尾擦除，则只有一侧限制缺失值，因此可行性仅取决于全局边界 1 和 1000。 

一个天真的错误是假设只要价值持续增长，任何差距都是可以重建的。 这忽略了大数值区间内的组合自由度。 

## 方法

 一个蛮力的想法是尝试每一个可能的子数组来擦除并检查擦除的段是否可以唯一地重建。 对于每个区间，我们模拟是否存在恰好一个符合两个边界约束之间的严格递增序列。 

对于从 l 到 r 的段，我们查看 a[l-1] 和 a[r+1] （如果存在），并计算有多少种方法可以在它们之间选择 r-l+1 严格递增整​​数。 这可以通过组合数学或 DP 对 [1, 1000] 中可能的值来完成。 

对于每个区间，从头开始计算有效性为 O(1000) 或更差，并且有 O(n^2) 个区间，因此暴力破解变为 O(n^2 * 1000)。 对于 n = 100 来说这已经很好了，但是有点矫枉过正了。 

关键的观察是，当至少有两种有效的方法来填补空白时，唯一性就会失败。 当边界之间的可用整数数量严格大于要填充的位置数量时，就会发生这种情况。 如果开区间中有 k 个位置且至少有 k+1 个可用整数，我们总是可以选择多个递增序列。 

因此，对于固定间隔，条件简化为使用边界值的简单算术检查，而无需枚举序列。 这将每次检查变成 O(1)，总共允许 O(n^2)。

 ## 算法演练

 我们尝试每一个可能的连续段作为擦除块并测试重建是否唯一。

1. 修复擦除段的左端点l。 这代表第一个被删除的元素。 
2. 将右端点r从l延伸到n。 这定义了已擦除块[l,r]。 擦除段的长度为r - l + 1。 
3. 确定边界约束。 如果 l > 1，则该段之前的值为 a[l-1]；如果 r < n，则该段之后的值为 a[r+1]。 这两个值限制了可以放置在擦除区域中的内容。 
4. 计算有多少个整数严格位于两个边界值之间。 如果两个边界都存在，则为 a[r+1] - a[l-1] - 1。如果一侧缺失，我们将其视为该侧无界，但仍受 1 和 1000 约束。 
5. 将可用值的数量与要填充的位置数量进行比较。 如果可用整数超过位置数，则有多种方法可以选择严格递增序列，因此重建不是唯一的。 
6. 如果可用整数恰好等于位置数，则只有一种赋值方式，因此该段有效。 
7. 跟踪任何有效段的最大长度。 

重要的细节是，严格的单调性迫使擦除的值对应于固定间隔内严格增加的选择，因此问题减少到计算组合而不是构造序列。 

### 为什么它有效

 在两个固定边界值之间，任何有效的完成都对应于从该区间的整数中选择与擦除段长度相同的严格递增序列。 如果区间包含的元素多于位置，则存在多个子集，因此唯一性失效。 如果它包含与位置一样多的元素，则恰好存在一个可能的子集，从而强制进行一次重建。 这种等价性确保了贪婪计数条件完全符合唯一性要求。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    a = list(map(int, input().split()))

    ans = 0

    for l in range(n):
        for r in range(l, n):
            k = r - l + 1

            left = a[l - 1] if l > 0 else 0
            right = a[r + 1] if r + 1 < n else 1001

            available = right - left - 1

            if available == k:
                ans = max(ans, k)

    print(ans)

if __name__ == "__main__":
    main()
```嵌套循环枚举所有可能的已擦除段。 对于每个线段，我们计算其长度及其相邻线段之间的有效数值间隔。 哨兵值 0 和 1001 扩展了边界，以便统一处理边缘处的删除。 

关键的实现细节是条件`available == k`。 这编码了唯一性：必须存在足够多的整数来强制执行单个严格递增的选择。 如果存在的整数多于位置，则会出现歧义。 

## 工作示例

 考虑数组`[1, 3, 4, 5, 6, 9]`。 

我们测试该段`[4, 5]`，对应于 l = 2，r = 3（0 索引）。 边界为左侧 3 个，右侧 5 个。 它们之间严格可用的整数为{4}，因此available = 1。段长度也是2？ 实际上这个段是无效的，因为长度2需要正好两个3到5之间的整数，这是不可能的，所以它被拒绝。 

现在考虑`[3, 4]`其中 l = 1，r = 2。边界为 1 和 5。可用整数为 {2, 3, 4}？ 实际上严格在 1 和 5 之间给出 {2, 3, 4}，因此 available = 3。段长度为 2，因此 available > k，这意味着存在多个填充且重建不唯一，因此它是无效的。 

最后考虑一个有效的最佳片​​段，其中间隙大小与所需的位置数量完全匹配。 这是唯一保持唯一性的情况，算法通过相等来捕获它。 

该迹线表明太小和太大的间隔都会被拒绝，只有精确拟合才能生存。

 ## 边缘情况

 一个边缘情况是从一开始就删除。 对于像这样的输入`[5, 6, 7]`，删除前缀意味着左边界实际上为 1。该算法将缺失的左边界视为 0，因此间隔变得正确有界，并且唯一性仅取决于右约束，而右约束的处理方式是一致的。 

另一种边缘情况是删除整个数组（可能除了一个元素）。 由于两个边界都变为 0 和 1001，因此可用范围是最大的。 条件`available == k`仍然正确地确定是否存在唯一的严格递增序列，并在剩余太多数字自由度时防止模棱两可的完成。
