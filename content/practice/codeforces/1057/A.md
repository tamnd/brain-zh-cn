 ## 解决方案
 ## 问题理解

 我们得到一个不断增长的路由器网络，形成一棵以路由器 1 为根的树。除了第一个路由器之外的每个路由器都是逐个添加的，并且每个新路由器都直接连接到恰好一个较早的路由器。 这意味着每个路由器都有一个父路由器，路由器 1 是没有父路由器的原始根。 

对于从 2 到 n 的每个路由器，输入给出了创建该路由器时所附加的路由器的索引。 根据这些信息，我们可以重建树结构，但任务并不要求整个结构。 相反，我们只对一条特定路由感兴趣：从路由器 1 到路由器 n 的唯一路径。 

由于每个路由器只有一个父路由器，因此该树中从节点 1 到节点 n 仅有一条简单路径。 输出必须按顺序列出该路径上的节点。 

约束 n 高达 200000 意味着任何解都必须是线性或近线性的。 重复搜索或重建路径的二次方法会太慢，因为在最坏的情况下它可能执行 10^10 次运算。 需要单遍重构或简单遍历。 

如果一个人试图在不存储父母的情况下“模拟向上行走”，就会出现一个微妙的问题。 如果我们不显式记录父指针，我们可能会重复重新计算祖先，这会导致重复扫描并变得低效。 另一个潜在的错误是构建邻接表并运行从 1 到 n 的 DFS，而不确保路径重建的正确性，这可能会意外返回完整的遍历路径而不是唯一的简单路径。 

## 方法

 解决该问题的一种强力方法是使用邻接表构建完整树，然后从节点 1 运行深度优先搜索，直到找到节点 n。 在 DFS 期间，我们维护当前路径并在到达 n 时返回它。 这是可行的，因为图是一棵树，因此任何两个节点之间都只有一条路径。 

然而，这种方法存在对大型子树进行不必要的探索的风险。 在最坏的情况下，DFS 可能会在找到节点 n 之前遍历几乎所有边，从而导致 O(n) 遍历。 虽然 O(n) 实际上是可以接受的，但实现比需要的更复杂，并且如果回溯处理不当，很容易编写错误。 

关键的观察是我们根本不需要搜索。 每个节点显式存储其父节点。 这意味着我们可以通过重复跟踪父指针来重建从 n 回到 1 的路径。 一旦到达节点 1，我们就反转收集到的序列以获得正确方向的路径。 

这将问题简化为一个简单的指针追踪过程，保证路径长度（最多为 n）呈线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 从 1 到 n 的 DFS 搜索 | O(n) | O(n) | 已接受但没有必要 |
 | 母链重构| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们利用这样一个事实：除了节点 1 之外，每个节点都只有一个父节点。 

1. 读取父数组，其中 p[i] 给出节点 i 的父节点。 我们在概念上定义 p[1] = 0，因为它没有父级。 
2. 从节点 n 开始，重复移动到其父节点，将每个访问过的节点存储在列表中。 这将以相反的顺序构建路径。 
3. 当到达节点 1 时停止，因为它是树的根。 
4. 反转收集到的列表，使其从 1 开始，到 n 结束。 
5. 输出结果序列。 

每一步都是安全的，因为父关系总是严格导致较小的索引，因此我们不能无限地循环或循环。 

### 为什么它有效

父指针定义了一个有根树结构。 在树中，每个节点都有一条到根的简单路径。 从任何节点跟随父链接最终都必须到达根，因为索引沿着边缘严格递减 (p[i] < i)。 这保证了终止。 由于从 n 向上只有一条路径，因此我们收集的序列正是从 n 到 1 的唯一路径，将其反转会产生所需的从 1 到 n 的路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    p = [0] * (n + 1)

    vals = list(map(int, input().split()))
    for i in range(2, n + 1):
        p[i] = vals[i - 2]

    path = []
    cur = n

    while cur != 0:
        path.append(cur)
        cur = p[cur]

    path.reverse()
    print(*path)

if __name__ == "__main__":
    solve()
```该解决方案首先以直接索引形式重建父数组以进行常量时间访问。 然后它从节点 n 开始并跟随父指针直到到达根。 收集的列表在最后是相反的，因为遍历自然是从子级到父级，而所需的输出是从根到目标。 

一个常见的错误是忘记完全停止在节点 1 或错误地初始化节点 1 的父节点，这将导致无效查找。 另一个问题是以错误的顺序附加节点并忘记在最后反转。

 ## 边缘情况

 一个关键的边缘情况是当树是从 1 到 n 的直链时。 在这种情况下，每个节点都指向前一个节点，因此算法必须遍历最大可能的深度。 从n开始，我们重复跟踪父级：n → n−1 → … → 1。收集到的反向路径正确输出从1到n的整个序列，没有任何分支歧义。 

另一个边缘情况是星形树，其中每个节点都直接连接到 1。在这种情况下，n 的父节点是 1，因此路径只是 n → 1 反转为 1 → n。 该算法无需任何特殊大小写即可处理此问题，因为它不依赖于结构，仅依赖于父指针。 

第三个微妙的情况是当 n 很小时，例如 n = 2。这里循环仍然正确执行一次，收集 [2, 1]，并将其反转为 [1, 2]，表明终止条件干净地处理了根。
