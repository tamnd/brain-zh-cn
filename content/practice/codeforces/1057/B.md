 ## 解决方案
 ## 问题理解

 我们的时间线分为几秒。 每一秒，我们都知道有多少请求到达服务器。 这为我们提供了一个数组，其中每个位置代表该秒的请求量。 

我们需要找到最长的连续时间间隔，使得该时间间隔内的请求总数严格大于该时间间隔长度的 100 倍。 换句话说，如果我们选择一段长度`t`，其中请求的总和超过`100 * t`，该段被视为攻击期，并且我们希望该段的最大可能长度。 

输入大小最多可达 5000 秒，每个值也最多可达 5000。这立即表明 O(n^2) 解决方案是可以接受的，因为 n^2 约为 2500 万次操作，这在带有前缀和的 Python 中完全适合 2 秒。 

一个关键的观察结果是，该条件仅取决于时间间隔内的总和。 这使得前缀求和成为一个自然的工具，因为它允许恒定时间范围求和查询。 

主要困难不是计算总和，而是有效检查所有候选区间并提取最大有效长度。 

当没有区间满足条件时，就会出现微妙的边缘情况。 例如，如果所有值均为 0，则每个区间的总和为 0，且永远不会大于`100 * t`。 在这种情况下，答案必须是 0，而不是负数或 -1。 

另一种边缘情况是只有单个元素符合条件。 例如，如果`r[i] = 101`，则长度为 1 的间隔有效，但更长的间隔可能会失败。 该解决方案仍然必须正确检测和比较所有有效长度。 

最后，请注意条件是严格的：`sum > 100 * length`。 使用`>=`相反，会错误地包含边界情况，例如`100 * t`。 

## 方法

 直接方法检查每个可能的子数组。 对于每个起始索引`i`，我们扩展到每个结束索引`j`，计算总和`r[i..j]`，并检查是否超过`100 * (j - i + 1)`。 这是正确的，因为它明确地评估每个可能的间隔。 

然而，如果天真地完成，从头开始为每个间隔重新计算总和会导致 O(n^3) 复杂性。 即使部分重用，具有增量总和更新的双循环也会给出 O(n^2)，这是我们在暴力形式中所能期望的最好结果。 

关键的改进是预先计算前缀和，以便可以在 O(1) 内计算任何区间和。 这减少了对每对的检查`(i, j)`至恒定时间。 我们仍然枚举所有对，但现在每张支票都很便宜。 

问题的结构不允许比 O(n^2) 更好，因为我们明确要求最长的有效间隔，而没有单调约束，这将启用两个指针或二分搜索。 该条件在间隔长度上不是单调的，因此滑动窗口技术不适用。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（嵌套+重新计算）| O(n^3) | O(n^3) | O(1) | O(1) | 太慢了 |
 | 前缀和 + O(n^2) 枚举 | O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 1. 构建一个前缀和数组，其中`pref[i]`存储第一个的总和`i`元素。 这允许稍后快速计算任何段总和。 
2. 迭代所有可能的左端点`l`从 0 到 n-1。 
3. 对于每个`l`，延伸右端点`r`从`l`到 n - 1。 
4. 计算线段之和`[l, r]`使用`pref[r+1] - pref[l]`。 
5. 计算线段的长度为`r - l + 1`。 
6. 检查是否`segment_sum > 100 * length`。 
7. 如果条件成立，则用迄今为止看到的最大长度更新答案。 

我们扫描所有内容的原因`(l, r)`对的一点是任何有效的区间都必须出现在这个枚举中的某个地方。 Prefix sums ensure that we do not recompute overlapping sums repeatedly.

 ### 为什么它有效

 该算法对每个可能的连续间隔仅评估一次。 对于每个间隔，检查的正确性仅取决于其总和与基于其长度的线性阈值之间的精确算术比较。 由于前缀和给出了精确的区间和，因此不涉及近似或启发式。 因此，将检测任何满足条件的间隔，并跟踪其中的最大长度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    r = list(map(int, input().split()))

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + r[i]

    ans = 0

    for l in range(n):
        for rgt in range(l, n):
            total = pref[rgt + 1] - pref[l]
            length = rgt - l + 1
            if total > 100 * length:
                if length > ans:
                    ans = length

    print(ans)

if __name__ == "__main__":
    solve()
```
