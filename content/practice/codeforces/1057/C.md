 ## 问题理解

 我们得到一排糖果盒，每个糖果盒位于一个整数坐标处。 每个盒子包含固定数量的糖果，所有糖果颜色相同，每个盒子要么是红色，要么是绿色，要么是蓝色。 我们从一个特定的盒子开始。 从那里，我们可以向左或向右走到相邻的盒子，每次移动花费一秒，或者我们可以立即消耗当前盒子中的所有糖果。 

吃糖果的过程受到两个条件的限制，这两个条件适用于吃糖果的盒子的顺序。 首先，连续吃过的盒子必须有不同的颜色。 其次，每个新吃过的盒子里的糖果数量必须严格超过之前吃过的盒子。 我们想要选择满足这些约束的一系列被吃掉的盒子，使得收集到的糖果总数至少为 k，同时最小化总步行时间。 

关键的结构是，吃东西在时间上是免费的，但运动却不是，所以成本完全取决于我们如何导航路线以到达有效的递增、交替颜色的盒子子序列。 

约束 n ≤ 50 和 k ≤ 2000 强烈表明我们可以考虑在子集或位置上相当密集的动态规划，但不能对没有结构的完整序列进行指数探索。 对所有被吃掉的盒子序列进行简单的指数搜索会增长 3^n 或更糟，考虑到强大的排序约束，这是不必要的。

 当最佳解决方案需要跳过附近“好看”的盒子时，就会出现微妙的边缘情况。 例如，如果一个装有许多糖果的盒子的颜色与上次吃掉的颜色相匹配或者其糖果数量违反了严格增加，则该盒子可能会提前无法使用。 总是选择最近的有效框的贪婪方法可能会失败。 

另一种失败模式是假设一旦我们选择了有效的递增序列的框，我们就可以在它们之间贪婪地行走，而无需重新考虑之前的决策。 由于移动成本取决于全球定位，而不仅仅是序列有效性，因此本地决策可能会导致旅行路线不理想。 

## 方法

 暴力解释是想象选择每一个可能的盒子序列来吃东西，检查它是否满足颜色交替和严格的递增约束，然后计算从 s 开始按顺序访问它们所需的最小步行成本。 即使我们固定了一个序列，计算最小旅行成本也很重要，因为我们可以自由地左右移动，并且可能会穿过已经访问过的盒子。 

有效序列的数量以 n 为指数。 即使我们将自己限制在子序列中，我们仍然面临着巨大的状态空间。 这种方法在 n = 50 左右就变得不可行。 

关键的观察是，“状态”唯一有意义的结构是最后吃掉的盒子。 一旦我们知道了我们吃的最后一个盒子，所有未来的决定都只取决于它的位置、颜色和糖果数量。 这表明了端点上的动态规划公式。 

我们定义了盒子对 i→j 之间的 dp 转换，其中 j 是 i 之后的一个有效的下一个被吃掉的盒子，如果它的颜色不同并且它的糖果数量更大。 转移的成本是当前位置和j之间的最短步行距离。 然而，我们还必须考虑到这样一个事实：在 i 处吃完饭后，我们的位置变成了 i，所以运动就是 |i − j|。 

这将问题简化为在有向图中寻找最小成本路径，其中节点是盒子，边尊重增加的糖果数量和颜色交替，并且边权重是索引之间的绝对距离。 我们还需要允许从初始位置 s 开始，它可以自由移动到第一个选择的框。

因为我们还跟踪收集的糖果总数，所以我们需要一个额外的维度：到目前为止已经吃掉了多少糖果。 然而，由于 k ≤ 2000 并且每个盒子最多有 50 颗糖果，因此背包式尺寸是可行的。 

我们将 dp[i][t] 定义为在盒子 i 恰好吃掉 t 颗糖果时结束的最短时间。 转换考虑所有先前的盒子 j 具有较小的糖果数量和不同的颜色。 从 dp[j][t − r[i]] 我们用成本 dp[j][t − r[i]] + |i − j| 更新 dp[i][t]。 初始状态允许从 s 开始，成本为 |s − i| 且 t = r[i]。 

这会在盒子和总糖果上产生干净的分层 DP。

 ## 算法演练

 1. 我们将每个盒子视为增加有效饮食序列的潜在终点。 对于每个盒子 i，我们记录它的糖果数量和颜色。 这将建立将发生转换的对象。 
2. 我们创建一个 DP 表，其中 dp[i][c] 表示在吃完 c 个糖果后到达第 i 个盒子所需的最短步行时间。 我们将所有值初始化为无穷大，因为我们正在最小化。 
3.对于每个框i，我们初始化直接从起始位置s开始序列的状态。 如果我们先吃第 i 盒，时间成本为 |s − i| 糖果数量变为 r[i]。 这捕获了所有有效的第一步。
