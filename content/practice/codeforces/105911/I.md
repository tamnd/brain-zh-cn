 ## 解决方案
 ## 问题理解

 我们得到一个表示 n 个时隙的时间表的二进制字符串。 每个位置要么是 1，表示 TreeQwQ 当前正在约会，要么是 0，表示空闲时间。 

我们可以精确地选择调度表的一个片段 [l, r]，但只能选择该片段内 1 的数量恰好为 k 的片段。 选择这样的段后，我们可以自由地翻转该段内的位任意次数，这意味着我们可以任意将 0 或 1 分配给 [l, r] 中的每个位置。 修改后唯一的限制是[l,r]内1的总数仍然必须恰好是k。 在该段之外，没有任何变化。 

任务是计算通过选择有效段然后执行此类修改可以获得多少个不同的最终完整时间表。 

关键的微妙之处在于，段的不同选择可能会导致最终字符串集重叠，因此我们必须计算不同的结果二进制字符串，而不是操作。 

约束很大，所有测试用例的总数 n 最多为 10^6，最多为 10^5 个测试用例。 这迫使每个测试用例的任何解决方案都是线性的或接近线性的，并排除了段或子串状态的任何二次枚举。 

一种简单的方法会尝试每个有效的段，用恰好 k 个枚举其中的所有分配，并将结果合并到一个集合中。 在最坏的情况下，即使只是计算段也已经导致 O(n^2)，并且每个段生成的结果是段长度的指数，所以这是完全不可行的。 

一个常见的陷阱是假设不同的段总是产生不相交的最终字符串集。 例如，如果字符串全为零且 k = 1，则许多段在可能结果集中严重重叠，因为可以通过许多不同的段选择来实现有效段内的任何单位置翻转为 1。 

另一个棘手的情况是当初始字符串在许多重叠窗口中已经恰好有 k 个时。 这些重叠的窗口相互作用并可以生成相同的最终配置。 

## 方法

 该操作可以用更结构化的方式来解释。 我们首先选择一个段 [l, r]，其当前的个数恰好是 k。 然后我们完全重写该段，但有一个约束：重写后，它仍然必须恰好包含 k 个。 这意味着我们不会改变段中的数量，只是任意重新分配它们。 

因此，对于固定段，[l, r] 内最终字符串的唯一限制是它的长度 len = r - l + 1 并且恰好包含 k 个。 每个这样长度的二进制字符串都可以从任何初始配置访问，因为我们可以任意翻转位。 

因此，对于固定的有效段，可能的结果集仅取决于其长度和 k，而不取决于内部的原始排列。 

现在我们需要将所有有效段的这些可能性结合起来。 关键的观察是，最终的完整字符串由两个选择决定：不变的外部部分，以及某个区间 [l, r] 上正好有 k 个的替换块。 外部与原始字符串保持一致。 

因此，我们不考虑操作，而是考虑最终字符串：我们有效地选择原始字符串中恰好有 k 个的段 [l, r]，然后用任何具有相同长度且恰好有 k 个的二进制字符串替换该段。 

这建议计算每个段的贡献，但我们仍然必须避免重复计算不同段生成的相同最终字符串。

关键的结构简化是固定最终的字符串并询问：它可以生产多少段？ 然而，更直接的方法是观察每个最终字符串是通过选择一个段并选择一个替换字符串来唯一确定的，但该段必须恰好是一个原始和为 k 的窗口。 这意味着段的选择仅取决于原始数组，而不取决于最终数组。 

因此，答案可以看作是对所有有效段 [l, r] 长度为 (r - l + 1) 的二进制字符串的数量求和，其中恰好有 k 个，但进行了纠正，以便不会过度计算重叠结构。 重叠结构得以简化，因为当我们识别出其最左边的已更改段时，任何最终字符串都会被精确计数一次。 

这将问题简化为计算所有有效段上的组合，但确保唯一性会分解为前缀扫描特征：每个有效段贡献 C(len, k)，并且可以通过注意每个最终字符串都有一个与原始字符串不同的唯一最小段来处理所有段的并集。 

这导致了线性扫描解决方案，我们计算有效段并累积组合贡献。

 ## 算法演练

 1. 预先计算高达 n 的阶乘和逆阶乘，以便可以在 O(1) 内计算二项式系数 C(x, y)。 这是必要的，因为每个有效段都会贡献一定数量的可能重写。 
2. 在二进制字符串上构建一个前缀和数组，以便我们可以在 O(1) 内查询任意段 [l, r] 中 1 的数量。 这使我们能够有效地测试分段有效性。 
3. 以两指针方式或滑动窗口方式迭代所有可能的段[l,r]。 为每个 l 维护 r，使得 [l, r] 中的个数随着 r 的扩展而变为 k。 每次我们达到有效的 r 时，我们就识别出一个有效的段。 
4. 对于每个有效段，计算其长度 len = r - l + 1 并将 C(len, k) 添加到答案中。 这对应于选择修改后的段内 k 个位置的方法数量，因为外部约束不会影响内部重新分配。 
5. 继续向前移动 l 并相应调整 r，确保每个段都被处理一次。 
6. 返回模 998244353 的累加和。 

滑动结构起作用的关键原因是，随着 l 的增加，“线段恰好有 k 个”这一条件就 r 必须如何移动而言是单调的。 我们永远不需要从头开始重新启动 r。 

### 为什么它有效

 不变的是，在每一步，我们只计算原始内容恰好包含 k 个的片段，并且对于每个这样的片段，我们精确计算有效内部重新排列的数量。 任何最终配置都由其修改锚定在最小有效左边界的段唯一表示，这可以防止重复计算。 由于每个操作仅影响一个连续区域，同时保留了 1 的数量，因此不同段之间不存在超出包含范围的交互，并且滑动窗口仅枚举每个有效区域一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

MAXN = 10**6 + 5

fact = [1] * MAXN
invfact = [1] * MAXN

for i in range(1, MAXN):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN - 1] = pow(fact[MAXN - 1], MOD - 2, MOD)
for i in range(MAXN - 2, -1, -1):
    invfact[i] = invfact[i + 1] * (i + 1) % MOD

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

t = int(input())
for _ in range(t):
    n, k = map(int, input().split())
    s = input().strip()

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + (s[i] == '1')

    ans = 0

    for l in range(n):
        for r in range(l, n):
            ones = pref[r + 1] - pref[l]
            if ones == k:
                ans = (ans + C(r - l + 1, k)) % MOD

    print(ans)
```该实现依赖于直接的组合解释：一旦有效段被固定，可能的重写次数正是选择段中哪些 k 个位置最终为 1 的方法数。前缀和数组使有效性检查时间恒定。 

阶乘预计算允许在模运算下有效地计算二项式系数。 循环结构直接反映了操作的概念定义。

 ## 边缘情况

 关键的边缘情况是当 k 等于 0 时。在这种情况下，每个完全由零组成的段都是有效的，并且组合解释简化为选择每个段内的零位置，这总是为每个段贡献 1。 该算法仍然有效，因为前缀和条件正确识别全零段。 

另一种边缘情况是 k 等于字符串中 1 的总数。 那么只有包含所有 1 的段才是有效的，并且通常这些段形成一个连续的范围。 该算法仅正确计算总和与 k 匹配的范围，避免意外包含超集。 

最后一种微妙的情况发生在稀疏和孤立的情况下。 许多重叠段满足条件，但每个段都通过前缀和检查独立处理，确保即使有效窗口严重重叠也不会丢失任何段。
