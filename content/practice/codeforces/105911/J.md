 ## 解决方案
 ## 问题理解

 我们在无限网格上给出了几个点。 每个点代表一个放置在特定坐标上的辣椒，并且每个辣椒都有固定的类型。 一种类型沿着其列相互作用，这意味着它“关心”共享相同 x 坐标的其他辣椒。 另一种类型沿着其行交互，这意味着它“关心”共享相同 y 坐标的辣椒。 

要求是每个现有的胡椒必须由至少一个具有正确交互类型的其他胡椒“支持”。 具体来说，垂直型胡椒（沿列相互作用的胡椒）需要在同一列但不同行中至少有一个其他垂直型胡椒。 水平型胡椒（沿行相互作用的胡椒）需要在同一行但不同列中至少有一个其他水平型胡椒。 如果辣椒还没有这样的伙伴，我们可以在网格上的任何位置添加新辣椒，选择它们的位置和类型，并且我们希望尽量减少添加的数量。 

约束允许所有测试用例中最多有 2×10^5 个辣椒，坐标最多为 10^9。 这立即排除了任何试图模拟所有点之间成对交互或直接搜索网格的方法。 k 中的任何二次方都会太慢，如果天真地完成，甚至每对对数因子也会处于临界状态。 

当结构几乎完成但尚未完全完成时，就会出现微妙的边缘情况。 例如，如果一列中恰好有一个垂直胡椒，则在同一列中不添加至少一个垂直胡椒就不可能满足要求。 仅检查全局计数或仅检查行/列是否存在的简单方法会错过这一点。 

同样，考虑一个带有两个垂直辣椒的柱子。 两人都已经很满足了，因为各自都有了同一纵队的搭档。 如果一个粗心的贪婪策略没有认识到双方的满足已经存在，它可能仍然会尝试添加不必要的额外辣椒。 

## 方法

 一个蛮力的想法是明确模拟每个辣椒的满意度。 对于每个辣椒，我们将扫描所有其他辣椒并检查同一行或同一列中是否存在具有正确类型的有效伙伴。 这直接遵循条件，但在最坏的情况下每个测试用例的成本为 O(k^2)，当 k 达到 2×10^5 时，这变得完全不可行。 

关键的观察是，辣椒的满意度并不取决于全局配置，只取决于特定等价类中是否至少存在另一个辣椒。 垂直辣椒按 x 坐标分区，水平辣椒按 y 坐标分区。 在每个这样的组中，唯一重要的是该类型辣椒的数量。 

一旦我们隔离了这个结构，问题就变成局部的了。 对于每个 x 坐标，我们只关心那里存在多少个垂直的辣椒。 对于每个 y 坐标，我们只关心那里存在多少个水平辣椒。 交互要求简化为一个简单的阈值条件：如果组的大小为零或至少为 2，则该组有效。 一组大小为 1 的组是唯一有问题的情况，因为它迫使我们再添加一个元素来修复它。 

不需要跨组互动，因为添加的辣椒始终可以独立放置到需要的行或列中。 这消除了不同坐标之间的任何耦合。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(k^2) | O(k^2) | O(k) | 太慢了 |
 | 按行/列计数 | O(k) | O(k) | 已接受 |

 ## 算法演练

 我们将问题分为两个独立的计数任务：一个用于按列分组的垂直辣椒，一个用于按行分组的水平辣椒。

1.我们遍历所有给定的辣椒并维护两个哈希图。 一张地图计算每列 x 中出现的垂直辣椒数量。 另一个计算每行 y 中出现了多少个水平辣椒。 此步骤将所有几何信息压缩为频率统计。 
2. 处理完所有辣椒后，我们检查垂直图中的每一列。 如果一列恰好包含一个垂直胡椒，则该胡椒当前在其列中没有伙伴。 为了解决这个问题，我们必须在同一列的某处添加至少一个额外的垂直胡椒。 如果计数为两个或更多，则无需执行任何操作，因为该列中的每个辣椒都已经有至少一个同伴。 
3. 我们对水平图中的行重复相同的推理。 任何一行恰好包含一个水平胡椒粉，都贡献一个所需的添加物。 
4. 我们对两张图中的所有此类单例案例进行求和。 该总数是所需添加辣椒的最少数量。 

这里重要的设计选择是每个有问题的组都是独立修复的。 添加新的胡椒来修复单个列不会干扰其他列，因为它被放置在选定的坐标处并且仅影响该特定组。 

### 为什么它有效

 不变的是，每个有效的配置必须确保在每列中，垂直辣椒被分组为至少两个大小的组件，并且在每行中，水平辣椒被类似地分组。 任何规模为一的群体都是孤立的，无法在内部得到满足，因此它必然迫使至少一个外部添加。 相反，向每个单组添加一个新的胡椒总是足够的，因为它在该组内创建了一个有效的对，而不会影响其他地方的正确性。 由于组不会跨坐标交互，因此对独立校正求和会产生全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        k = int(input())
        
        col_v = defaultdict(int)
        row_h = defaultdict(int)

        for _ in range(k):
            x, y, w = map(int, input().split())
            if w == 0:
                col_v[x] += 1
            else:
                row_h[y] += 1

        ans = 0

        for x, c in col_v.items():
            if c == 1:
                ans += 1

        for y, c in row_h.items():
            if c == 1:
                ans += 1

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现将网格压缩为两个频率图，一张由 x 键控表示垂直辣椒，另一张由 y 键控表示水平辣椒。 每个映射条目直接代表一个约束组。 最后的循环只是计算这些组中有多少个大小恰好为 1，这对应于强制添加的数量。 

这里的一个常见实施错误是尝试模拟迭代地添加辣椒。 这将需要重复更新结构，并且可能意外地导致重复计算或级联更新。 计数方法完全避免了这种情况，因为它直接从初始分布计算最终状态。

 | 扫描行| - | d | 单例 +1 |

 这表明只有孤立的群体才重要，而不是更大的群体。

 ## 边缘情况

 临界边缘情况是坐标组的大小恰好为 1。 例如：

 输入：```
1
1
5 5 0
```该算法将 x=5 列分类为具有一个垂直胡椒。 在处理过程中，它将答案加一。 不存在行贡献，因此最终输出为 1。正确性来自于单个节点如果不在其列中引入至少一个附加节点就无法满足要求。 

另一种情况是当多个单例组存在于不同的列或行时：

 输入：```
1
3
1 1 0
2 2 0
3 3 1
```这里每个组都是独立的。 该算法计算三个单例结构并返回 3。每个增量对应于固定一个断开连接的需求，并且由于添加可以独立放置，因此不存在重叠或重复计算。 

第三种微妙的情况是当一个组已经有两个元素但分布在不同位置时。 即使坐标不同，只要它们共享相同的x（垂直）或相同的y（水平），它们就已经满足彼此。 该算法自然会处理这个问题，因为它只检查计数，而不检查组内的空间分布。
