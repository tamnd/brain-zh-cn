 ## 解决方案
 ## 问题理解

 我们从$n$相同的硬币，最初都显示正面。 另一位玩家偷偷翻转准确$k$其中，我们不知道选择了哪个子集。 之后，我们必须为每枚硬币分配一个来自四个可能动作的指令：将其放入 A 堆而不翻转，将其放入 A 堆并翻转，将其放入 B 堆而不翻转，或将其放入 B 堆并翻转。 

应用所有分配后，每枚硬币都恰好属于两堆硬币中的一个，可能会根据我们在分配期间是否翻转它来改变其方向。 目标是，无论哪一个$k$最初翻转硬币，A 堆中最终正面的数量等于 B 堆中最终正面的数量。允许其中一堆为空。 

输出不是直接的桩的配置而是长度$n$字母表 {1,2,3,4} 上的字符串，编码分配给每个硬币的操作。 

关键约束是$n \le 10^4$，因此任何构造都必须是线性或接近线性的。 尝试枚举翻转硬币的子集是指数级的，并且立即不可行。 甚至推理一切$\binom{n}{k}$可能性太大，因此解决方案必须将不确定性压缩为少量聚合条件。 

出现微妙的边缘情况时$k=0$。 在这种情况下，最初不会掷硬币，因此无论分配如何，我们都必须保证正面计数相等。 另一个极端是$k=n$，所有硬币都被翻转； 即使初始状态完全颠倒，构造仍然必须有效。 

一种幼稚的方法可能会尝试贪婪地分配硬币来平衡预期的贡献，但期望是无关紧要的，因为该要求是所有大小子集的最坏情况$k$。 另一个常见的失败是对称地处理 A 堆和 B 堆，而不控制翻转与堆分配的交互方式，这在翻转硬币的对抗性选择下会失败。 

## 方法

 直接的暴力策略将为每个硬币分配四种操作之一，然后模拟所有可能的选择$k$硬币最初被翻转。 对于每个分配，我们将检查大小的每个子集是否$k$导致两堆最终的头数相等。 这需要评估$\binom{n}{k}$每个作业的案例，并且有$4^n$任务，使其成为天文数字。 

瓶颈是两个组合选择之间的相互作用：对手选择翻转硬币，我们选择分配。 该结构表明我们应该避免完全跟踪单个代币，而是进行总体推理。 

关键的观察结果是，每枚硬币都会对硬币堆之间的平衡产生确定性的符号效应，如果最初翻转硬币，还会加上修正项。 问题简化为确保 A 堆和 B 堆之间的总贡献差在任何选择下都不变$k$翻转硬币。 这迫使结构中和每个硬币的敏感性，这意味着每个硬币必须以一种内部抵消或与另一个硬币配对的方式做出贡献。 

这自然会导致配对硬币。 如果我们将硬币分成对，并在每对内分配对称操作，我们可以确保翻转一对内的任何一枚硬币都不会改变全局平衡。 这将问题从全局不确定性减少到对上的局部不变量。 

最终的构建取决于是否$n-k$是均匀的。 如果是奇数，则会留下一枚未配对的硬币，其贡献无法抵消，从而使任务变得不可能。 如果是偶数，我们可以将硬币分成对，并在每对中分配互补操作，这样无论翻转如何，每对都会在堆之间贡献固定的净零差异。

 | --- | --- | --- | --- |
 | 暴力枚举| O(4^n · C(n,k)) | O(4^n · C(n,k)) | O(n) | 太慢了 |
 | 成对不变构造 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过在硬币对上强制执行局部不变性来构建分配。 

1.首先检查是否$n$很奇怪。 如果是，立即输出-1。 这是因为任何有效的构造都需要将硬币分组为不相交的对，以消除翻转引起的不平衡。 
2.如果$n$是偶数，继续处理连续对的硬币$(i, i+1)$。 
3. 对于每一对，分配第一个硬币操作 1（堆 A，不翻转）和第二个硬币操作 4（堆 B，翻转）。 这创建了一个对称结构，其中一枚硬币始终以原始方向进入 A 堆，而另一枚硬币进入 B 堆但被翻转。 
4. 从概念上验证每对硬币都为每一堆贡献了一个有效的正面，无论其中一枚硬币最初是否被翻转过。 翻转要么保持对内的对称性，要么交换角色，但不会改变桩之间的差异。 
5. 输出构造好的字符串。 

关键的想法是，每一对都充当一个独立的小工具，其对最终头差的净贡献始终为零，与对手的初始翻转无关。 

### 为什么它有效

 我们在每对的层面上维持一个不变量：无论这对中的哪枚硬币最初被翻转，由该对贡献的 A 堆和 B 堆之间的头数差异始终为零。 由于每枚硬币都属于这样的一对，因此全局差异是成对差异的总和，该总和始终为零。 对手的选择$k$翻转的硬币仅重新分配对内的翻转，但不能破坏此取消属性。 因此，任何有效的初始配置都会导致两个堆中的头数相等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())

if n % 2 == 1:
    print(-1)
    sys.exit()

res = []
for i in range(n // 2):
    res.append("1")
    res.append("4")

print("".join(res))
```代码首先检查奇偶校验$n$，它直接编码配对的可行性。 如果配对不可能，则无需进一步推理，我们返回 -1。 

否则，我们通过重复固定的两个字符块来构建字符串。 操作“1”和“4”的选择是有意的：它们将硬币发送到相反的堆，同时确保对称的翻转行为。 由于每枚硬币在其在货币对中的角色中都受到相同的对待，因此不需要依赖于指数的逻辑。 

该结构避免了任何依赖$k$，这很重要：因为$k$我们不知道并且是敌对的选择，任何正确的解决方案都必须独立于它。

 ## 边缘情况

 对于$n=1$，算法立即输出 -1，因为不可能进行配对。 这是正确的，因为无论操作选择如何，单个硬币总是会在硬币堆之间引入不可约的不对称性。 

为了$n=2$，构造输出“14”。 如果对手翻转一枚硬币或同时翻转两枚硬币，则堆分配之间的对称性可确保每一堆在该对内取消后恰好收到一个有效的头部贡献。 

为了$n=3$，算法再次输出-1。 虽然$k$可能是 0 或 3，未配对的硬币无法与配对结构中和，并且任何分配操作的尝试都会在至少一种情况下留下残余不平衡。
