 ## 解决方案
 ## 问题理解

 我们给出了一系列的日子。 每天我们选择一个非负实数$r_i$，解释为热量摄入，但限制在固定区间内$[L, R]$。 有些日子已经固定了：$k$的位置，价值$r_i$是预先确定的，不能更改。 

除此之外，还有另一个序列$c_i$，这不是自由选择的。 它使用线性递归从前几天确定性地演变：$$c_i = p \cdot c_{i-1} + (1 - p)\cdot r_{i-1}.$$所以每个$c_i$是先前代谢状态和先前摄入量的加权混合。 

目标是全局的：选择所有未固定的$r_i$值，以便所有天数的总和$c_i - r_i$被最大化。 

关键的困难在于每一个决定$r_i$影响一切未来$c_j$通过因子重复传播的值$p$。 所以问题不是局部可分的。 

约束条件意味着$n$可以达到$2 \cdot 10^5$每个测试套件，最多$10^4$测试用例。 这立即排除了任何尝试模拟不同分配或对所有状态进行动态编程的解决方案。 任何二次或偶数$O(n \log n)$仅当在所有测试中仔细摊销时，每个测试用例才是可接受的。 

固定日期打破连续性会产生微妙的边缘情况。 如果有人试图将问题视为简单的对头寸的贪婪，那么它会失败，因为当天的决定$i$影响所有后来的贡献，包括由固定约束分隔的部分。 

例如，如果$p$接近 1，早期决策传播得远； 如果$p$很小，只有附近的邻居才重要。 一个天真的本地贪婪者，总是选择$R$或者$L$依赖于即时增益会忽略这种远程影响，并且在固定位置和自由位置交替等情况下会失败。 

## 方法

 如果我们忽略重复性，目标就显得微不足道：最大化$\sum (c_i - r_i)$只是意味着推动每个$r_i$根据符号到一个边界。 但复发将一切联系在一起。 

暴力解释将尝试所有可能的分配给未固定的$r_i$。 由于每个空闲日都有一个连续的范围$[L, R]$，即使离散化为几个候选也会导致指数组合。 即使我们每个空闲日只考虑两个选择，状态空间也会变成$2^{n-k}$，这是不可能的。 

关键的观察结果是递归是线性的。 每一个$r_j$为许多未来做出贡献$c_i$值，但其影响力是倍增的$p$。 这意味着最终目标可以重写为所有的线性函数$r_i$：$$\sum_{i=1}^n (c_i - r_i) = \sum_{i=1}^n w_i r_i + \text{constant},$$对于某些系数$w_i$只取决于$p$以及职位结构。 

一旦认识到这种减少，问题就变成了经典的约束线性优化：每个$r_i$除了盒子约束之外是独立的$[L, R]$，并且每个人都独立地为目标做出贡献。 因此，每个$r_i$应该被推到任一$L$或者$R$，取决于其系数的符号，同时尊重固定约束。 

剩下的任务是有效地计算这些系数。 直接展开表明每个$r_j$为所有人做出贡献$c_i$为了$i > j$有重量$(1-p)p^{i-j-1}(1-p)$式几何衰变。 对所有后缀求和给出一个几何级数，因此每个位置都有一个封闭形式系数，仅取决于$p$和$n-j$。 这可以使用后缀传播技巧在线性时间内计算出来。 

固定位置只是覆盖选择并被视为常量。

 ## 算法演练

 我们重写每个人的贡献$r_i$达到最终目标。 

###第一步：扩大重复影响力

 我们扩展$c_i$反复：$$c_i = p^{i-1}c_0 + \sum_{j=1}^{i-1} (1-p)p^{i-j-1} r_j.$$这显示了每一个$r_j$影响一切未来$c_i$。 

### 第 2 步：交换求和

 我们重写目标：$$\sum_{i=1}^n c_i - \sum_{i=1}^n r_i.$$第一项成为所有对的双倍和$(j, i)$和$j < i$，加上初始贡献$c_0$。 重新排序对所有术语进行分组$r_j$，产生每个位置的系数。 

### 步骤 3：计算贡献权重

 对于固定的$j$，其总贡献为：$$r_j \cdot (1-p)\sum_{i=j+1}^n p^{i-j-1} = r_j \cdot (1-p)\frac{1-p^{n-j}}{1-p} = r_j (1 - p^{n-j}).$$然后直接减去$-r_j$术语，给予：$$w_j = (1 - p^{n-j}) - 1 = -p^{n-j}.$$所以每$r_j$与系数呈线性关系$-p^{n-j}$，固定常数来自$c_0$。 

### 步骤 4：处理约束

 各未固定$r_j$被选择于$[L, R]$。 由于目标是线性的$r_j$，最优选择为：

 如果$w_j \ge 0$， 选择$R$，否则选择$L$。 

### 步骤 5：累积最终答案

 我们预先计算的幂$p$从后面避免重新计算和求和贡献。 

### 为什么它有效

 整个系统总体呈线性$r_i$。 递推不会引入不同决策变量之间的非线性相互作用； 它只是通过几何衰减随时间重新分配权重。 一旦重写为独立线性项的和，可行区域就变成了区间的乘积。 区间乘积上的线性目标总是在每个区间的边界独立地达到其最优，因此每个变量决策都是局部最优和全局一致的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, k = map(int, input().split())
        r0, c0, p, L, R = map(float, input().split())

        fixed = {}
        for _ in range(k):
            pos, val = input().split()
            pos = int(pos)
            val = float(val)
            fixed[pos] = val

        # suffix powers of p
        powp = [1.0] * (n + 2)
        for i in range(n, 0, -1):
            powp[i] = powp[i + 1] * p

        ans = 0.0

        # contribution from initial c0
        ans += c0 * powp[1]

        for i in range(1, n + 1):
            coeff = -powp[i]

            if i in fixed:
                ans += coeff * fixed[i]
            else:
                if coeff >= 0:
                    ans += coeff * R
                else:
                    ans += coeff * L

        print(f"{ans:.10f}")

if __name__ == "__main__":
    solve()
```该实现计算的幂$p$从后面这样$p^{n-i}$可用于$O(1)$。 每个位置都是独立处理的。 固定值会覆盖边界选择，而自由变量则被推至使线性贡献最大化的端点。 

一个微妙的点是浮动精度：因为所有系数都是$p$，值衰减很快，但仍然需要双精度累加来维持$10^{-6}$准确性。

 ## 边缘情况

 一种边缘情况是所有位置都固定。 在这种情况下，算法绝不能尝试边界替换。 系数逻辑仍然有效，因为固定覆盖路径完全绕过决策，因此结果变成线性形式的直接评估。 

另一种边缘情况发生在$p$极其接近 1。那么$p^{n-i}$衰减缓慢，早期位置仍然对总数产生显着影响。 该算法无需修改即可处理此问题，因为它直接使用浮点幂，而不是截断或近似级数。 

最后一种情况是当$p$非常小。 然后，除了最后一个系数之外的所有系数都接近于零，因此只有最后几天才重要。 该实现仍然评估每一项，但数值贡献自然集中在最后，与递归的分析行为相匹配。
