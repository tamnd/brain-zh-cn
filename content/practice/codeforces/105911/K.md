 ## 解决方案
 ## 问题理解

 我们有一排雕像，每个雕像最初指向循环排列的四个方向之一：前、右、后、左，然后在完整旋转后再次回到前。 我们可以执行全局顺时针旋转所有雕像的操作，但不同的是，每个操作都是通过按下特定雕像或按下我们自己来触发的，这两种操作最终都会产生相同的全局效果：每个雕像顺时针旋转 90 度。 

两个允许的操作之间的唯一区别是概念上的，而不是几何上的。 每次按下时，每个雕像都会在循环中精确旋转一步。 由于该操作永远不会单独影响雕像，因此整个系统只能通过所有值模 4 的重复全局增量来演化。 

目标是选择一系列的按压顺序，以便在所有旋转之后，每个雕像最终都指向前方。 每个雕像都以 {0, 1, 2, 3} 中的值开始，其中 0 表示前面，1、2、3 对应于右、后、左。 

约束 n 可以大到 10^6，这会立即排除任何尝试对每个雕像进行不同按压顺序或探索操作的指数组合的模拟。 任何有效的解决方案都必须将问题减少到最坏的 O(n) 或 O(n log n)，最好是 O(n)。 

如果一个人试图局部思考，就会出现一个微妙的问题。 例如，人们可能会想象按下不同的雕像可能会产生不同的效果，但实际上这两种操作都会对整个阵列产生相同的转换。 这消除了任何对每个雕像的控制，并将问题简化为纯粹的全局偏移系统。 

当所有雕像都已对齐、所有雕像都相同但未对齐、以及方向的分布产生冲突的需求时，就会出现边缘情况。 例如，输入`0 1 2 3`显然不能通过每个雕像的部分推理来修复，因为每次按下都会平等地影响所有值。 正确的输出仅取决于需要多少次全局旋转，而不取决于任何操作顺序。 

## 方法

 暴力破解的想法是从模拟按压顺序开始的。 由于每次按下都会将所有雕像旋转一步，因此我们可以尝试应用 k 次按下并检查所有值是否都以 4 为模变为零。这很简单：对于每个 k，计算变换后的值并验证它们是否都为零。 问题是 k 原则上是无界的，甚至将每个雕像限制为 [0, 3] 中的 k 也无济于事，因为操作不会隔离各个位置。 

一种稍微结构化的蛮力方法是观察，因为每个操作都是相同的，所以唯一重要的是按 4 模的按下总数。我们可以尝试从 0 到 3 的 k，每次模拟整个数组，并检查有效性。 这可行，但仍然需要 O(4n)，这很好，但隐藏了更深的结构。 

关键的见解是系统只有一个自由度：全局旋转偏移。 如果我们执行 k 次操作，每个雕像都会增加 k modulo 4。因此，对于从 ai 开始的雕像，其最终值变为 (ai + k) mod 4。对于所有以 0 结束的雕像，我们需要对每个 i 进行 (ai + k) mod 4 = 0。 这意味着对于所有 i，k == -ai mod 4，这意味着所有 ai 在移位 k 后必须同意模 4 的相同值。 仅当所有 ai 都等于模 4，并且 k 由该值唯一确定时，这才有可能。 

因此，问题简化为检查一致性和计算单个全局偏移量。

 | --- | --- | --- | --- |
 | k 的强力模拟 | O(4n) | O(4n) | O(1) | O(1) | 已接受但没有必要 |
 | 全球偏移观察| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取方向数组并将每个值解释为模 4 算术中的整数。 这是必要的，因为每个操作都保留模块化结构。 
2. 观察到按下 k 后，每个元素都变成 (ai + k) mod 4，因此我们正在搜索使所有值同时为零的单个 k。 
3. 将条件 (ai + k) mod 4 = 0 转换为 k ≡ -ai mod 4。这意味着所有 ai 必须产生相同的所需 k 值。 
4. 根据第一个元素计算 k 个候选值 k = (4 - a0) mod 4。 
5. 通过检查对于每个 i，(ai + k) mod 4 是否等于 0 来验证一致性。如果任何元素违反此规定，则在统一全局旋转假设下不存在解。 
6. 如果一致，则返回 k 作为最小按压次数，因为任何额外的 4 次按压的完整循环都是多余的。 

这样做的原因是系统没有空间独立性。 每个操作对每个雕像应用相同的变换，因此整个状态空间崩溃为表示全局旋转的单个循环变量。 该算法通过将整个数组简化为具有一个未知数的单个模方程组来利用这一不变量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    k = (4 - a[0]) % 4
    
    for x in a:
        if (x + k) % 4 != 0:
            print(-1)
            return
    
    print(k)

if __name__ == "__main__":
    solve()
```代码读取输入并立即将问题简化为找到从第一个雕像导出的单个全局旋转偏移 k。 该选择确定了最终的方向必须是：如果第一个雕像需要 k 次旋转才能到达前方，那么所有其他雕像也必须在相同的 k 下到达前方。 

该循环验证了这个假设。 如果任何雕像在相同的 k 下没有对齐，则证明没有任何操作序列可以修复该配置。 此检查至关重要，因为它强制执行而不是假设全局一致性条件。 

一个微妙的实现细节是 k 的模计算。 使用 (4 - a[0]) % 4 可确保 k 保持在 [0, 3] 范围内，避免负值，同时保持正确性。 

## 工作示例

 ### 示例 1

 输入：```
4
0 1 2 3
```我们从第一个元素计算 k。 

| 步骤| 一个[我] | (a[i] + k) % 4 | 有效 |
 | --- | --- | --- | --- |
 | 我=0 | 0 | (0+0)%4 = 0 | 是的 |
 | 我=1 | 1 | (1+0)%4 = 1 | 没有|

 由于 k = 0 立即失败，我们尝试 k = 3 （因为 (4 - 0) % 4 = 0，但整个数组的一致性失败），并且我们观察到没有单个 k 满足所有约束。 输出为-1。 

这表明仅使用全局旋转无法对齐混合方向。 

### 示例 2

 输入：```
3
2 2 2
```计算 k = (4 - 2) % 4 = 2。 

| 步骤| 一个[我] | (a[i] + k) % 4 | 有效 |
 | --- | --- | --- | --- |
 | 我=0 | 2 | 0 | 是的 |
 | 我=1 | 2 | 0 | 是的 |
 | 我=2 | 2 | 0 | 是的 |

 所有雕像在旋转 2 次后对齐，确认计算的偏移的正确性。 

这证实了统一数组总是可以干净地减少到单个全局旋转要求。

 ## 边缘情况

 一种重要的边缘情况是 n = 1 时。对于输入`3`，算法计算 k = (4 - 3) % 4 = 1，验证循环轻松通过，产生输出 1。这是正确的，因为单个雕像在固定旋转次数后始终与自身匹配。 

另一种情况是所有值都相同。 为了`2 2 2 2`，k 变为 2，并且每个检查都通过，因为系统简化为均匀移位。 

失败的情况是混合奇偶校验，例如`0 2 0 2`。 该算法选择 k = 0，但第二个元素立即违反了条件，因为 (2 + 0) % 4 = 2，因此它正确地拒绝了配置。
