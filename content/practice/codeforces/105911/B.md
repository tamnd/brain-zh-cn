 ## 解决方案
 ## 问题理解

 我们得到一个网格，其中一些单元格被标记为已占用，并且如果我们向四个方向移动，则所有已占用的单元格形成一个连通区域。 任务是使用独立放置在每一层上的小乐高积木，将每个占用的单元覆盖两次，一次在上层，一次在下层。 

每一层都是已占用单元格的完整平铺。 层中的图块必须是以下三种形状之一：单个单元、覆盖两个侧相邻单元的多米诺骨牌或覆盖形成直角的三个单元的 L 形三联骨牌。 空单元格在两层中都必须保持为空。 

额外的约束是全局的：如果我们将每个砖块视为一个节点，那么只要两个砖块在至少一个网格单元上重叠但属于不同的层，它们就会被连接。 换句话说，每个单元在顶层覆盖它的砖块和底层覆盖它的砖块之间创建一条边缘。 最终的要求是，跨两层的所有砖块必须在这些边缘下形成单个连接的组件。 

网格大小可达 1000 x 1000，因此单元总数最多可达 100 万个。 因此，任何解决方案都必须在占用单元数呈线性或接近线性的时间内运行。 网格尺寸中的任何二次方都是不可能的。 

一种简单的方法会尝试同时搜索两层中的所有平铺。 即使忽略图块放置的巨大分支因子，连接条件也会耦合两层，从而使状态空间在单元数量上有效呈指数增长。 

如果尝试独立地平铺每一层，则会出现更微妙的故障模式。 独立平铺确实满足局部有效性，但它们不能保证二分“砖块重叠图”的全局连通性，它很容易分裂成许多断开连接的组件。 

这种失败的一个小例子是 2 x 2 完整网格。 如果两层都使用两个水平多米诺骨牌平铺，则每个单元仅在其列内连接一对砖块，从而产生两个断开的组件。 

## 方法

 关键的困难在于连接不是在层内，而是通过共享单元跨层。 这表明需要分离关注点：一层应保证结构，而另一层应保证跨该结构的连接性。 

暴力破解的想法是枚举两个层的平铺并检查连接性。 从概念上讲，这是可行的，因为任何有效对都可以在线性时间内验证，但即使对于小网格，网格的平铺数量也会呈指数增长。 除了微小的输入之外，这会立即失败。 

关键的观察是我们不需要两个层来承载结构。 一层形成精心设计的分解，将所有细胞“粘合”在一起就足够了，而另一层可以非常简单。 

我们选择非常不对称的结构。 在底层，我们构建一个连接的平铺，其中每个砖块覆盖网格邻接图中一小部分连接的单元格。 这些砖块将作为连接整个结构的支柱。 

在顶层，我们做了一些琐碎的事情：每个占用的单元格都是它自己的 1 x 1 砖块。 这使得顶层最大限度地分散，但这实际上是有用的，因为顶层和底层之间的连接现在完全由底层如何将单元分组在一起来控制。 

现在，连接条件简化为更简单的语句。 每个底层砖块都连接它重叠的所有单层顶层砖块。 因此，如果底层砖块在网格上共同形成一个连接结构，那么所有顶层砖块都会通过它们连接起来。

因此，问题简化为使用允许的形状构建连接网格的平铺，使得由网格中的共享边引起的这些平铺的邻接图被连接。 由于网格区域本身是连接的，因此我们可以构建一棵生成树并用它来引导分解成大小最多为三的小连接块。 

这就是L形变得至关重要的地方。 具有两个子节点的树节点可以与两个子节点一起分组为单个 L 形砖块，确保一块砖块可以连接树的多个分支。 这使我们能够将生成树压缩为连接的块，而不会破坏覆盖范围。

 ## 算法演练

 我们仅处理值为 1 的单元格集合，并将它们视为具有四个方向邻居之间的边的图的顶点。 

1. 我们首先使用深度优先搜索构建该区域的生成树。 这是可能的，因为该区域是连接的，并且它为我们提供了一种无需循环即可捕获邻接的结构。 
2. 我们通过为每个单元分配自己的程序块 ID 来构建顶层。 该层没有内部结构，但每个细胞都成为潜在的连接点。 
3. 现在我们使用生成树构建底层。 目标是将树顶点划分为组，其中每个组在树中产生一个连通的子图，并且大小最多为 3。 
4. 我们以后序方式处理树。 对于每个节点，我们查看有多少子子树保持“未配对”状态。 如果子子树尚未放入积木中，则它会向上贡献一个节点。 
5. 每当一个节点至少有两个未配对的子节点时，我们将当前节点与其中两个子节点分组，并形成覆盖这三个顶点的 L 形砖块。 这是有效的，因为在树中，每个子节点都与父节点相邻，因此所有三个单元都在网格中连接。 
6. 如果恰好有一个未配对的子节点，我们将其与当前节点配对成多米诺骨牌。 由于树相邻性，这再次是有效的。 
7. 如果没有子节点未配对，则该节点将成为底层的单例砖块。 
8. 这个底层分区确保每个单元格都被覆盖一次，并且每个砖块都是允许的形状之一，因为它总是由一个节点、树中的一条边或一个父节点以及两个与其相邻的子节点组成。 
9. 最后，我们为两层中的所有砖块分配唯一的 ID，并输出两个网格。 

### 为什么它有效

 生成树保证我们形成的每个分组都由网格相邻的单元格组成。 后序构造确保每个节点都包含在一个组中，因为节点要么在分组期间被父节点消耗，要么在无法分组时保持为单例。 

对于连通性，请观察由跨层砖块形成的二分图的结构。 每个底层砖块都会接触与其单元相对应的所有顶层单块砖块。 由于底层砖块通过生成树中的共享单元连接，因此这些重叠会产生跨越所有砖块的连接链。 因此，任何砖块的子集都不会被孤立，因为任何分离都会与生成树的连通性或始终保持至少一块砖内父子结构完整的分组规则相矛盾。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n, m = map(int, input().split())
g = [list(input().strip()) for _ in range(n)]

idx = [[-1]*m for _ in range(n)]
cells = []
for i in range(n):
    for j in range(m):
        if g[i][j] == '1':
            idx[i][j] = len(cells)
            cells.append((i, j))

N = len(cells)
adj = [[] for _ in range(N)]

dirs = [(1,0),(-1,0),(0,1),(0,-1)]
for i, (x, y) in enumerate(cells):
    for dx, dy in dirs:
        nx, ny = x + dx, y + dy
        if 0 <= nx < n and 0 <= ny < m and idx[nx][ny] != -1:
            adj[i].append(idx[nx][ny])

parent = [-1]*N
tree = [[] for _ in range(N)]
visited = [False]*N

def dfs(u):
    visited[u] = True
    for v in adj[u]:
        if not visited[v]:
            parent[v] = u
            tree[u].append(v)
            dfs(v)

dfs(0)

top_id = [[0]*m for _ in range(n)]
bid = 1
for i in range(n):
    for j in range(m):
        if g[i][j] == '1':
            top_id[i][j] = bid
            bid += 1

bot_id = [[0]*m for _ in range(n)]
bid2 = 1

def build(u):
    global bid2
    leftovers = []
    for v in tree[u]:
        build(v)
        leftovers.append(v)

    i, j = cells[u]

    while len(leftovers) >= 2:
        a = leftovers.pop()
        b = leftovers.pop()
        ua, va = cells[a]
        ub, vb = cells[b]
        bot_id[i][j] = bot_id[ua][va] = bot_id[ub][vb] = bid2
        bid2 += 1

    if len(leftovers) == 1:
        v = leftovers.pop()
        uv, vv = cells[v]
        bot_id[i][j] = bot_id[uv][vv] = bid2
        bid2 += 1

    if bot_id[i][j] == 0:
        bot_id[i][j] = bid2
        bid2 += 1

build(0)

for row in top_id:
    print(*row)
for row in bot_id:
    print(*row)
```该实现首先构建占用区域的邻接图，然后提取 DFS 树。 顶层为每个单元分配一个唯一的标识符，使每个单元成为一个单例砖块。 

底层构造是在树上递归的。 每个节点都会收集子节点的结果，并尽可能将两个子节点与父节点贪婪地组合成 L 形砖块。 如果只剩下一个孩子，就会形成多米诺骨牌。 否则该节点将成为单例砖。 每个分配都直接在网格上完成，因此输出保持 O(nm)。 

必须注意仅组合 DFS 树中连接的节点，因为树边对应于有效的网格邻接。 

## 工作示例

 考虑一个简单的 1 x 3 行。 

| 步骤| 节点| 剩菜| 行动| 砖成型|
 | --- | --- | --- | --- | --- |
 | 0 | 中| [左，右] | 带两个孩子| L 型（父母 + 2 名儿童）|

 底层根据遍历顺序形成一个 L 形或三单元结构，而顶层仍然是三个单例。 这证实了一个底部砖块连接了所有顶部砖块。 

现在考虑一个 T 形区域。 

| 步骤| 节点| 剩菜| 行动| 砖成型 |
 | --- | --- | --- | --- | --- |
 | 0 | 中心 | [上、左、右] | 两个孩子组 | L 型 |
 | 1 | 中心 | [右] | 与母链配对| 多米诺骨牌|
 | 2 | 叶| []| 单例| 1x1 | 1x1

 这演示了如何使用 L 形状吸收分支，确保没有子子树断开连接。

 ## 边缘情况

 通过立即在两层中分配单例砖来处理单个单元区域，因此连接性很简单。 

细线单元在 DFS 树中没有分支，因此该算法仅生成多米诺骨牌或单例，而不需要 L 形。 

高度分支的区域由 L 形分组步骤处理，这确保父级可以将两个子级吸收到一块砖中，从而防止树分裂成断开的组件。
