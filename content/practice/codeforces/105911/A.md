 ## 解决方案
 ## 问题理解

 我们得到了三个独立的事件，代表哪吒在海上引起骚乱花费了多长时间。 每个事件的持续时间以小时为单位，并且在每个骚扰小时内，都会产生固定数量的投诉。 

具体来说，有三个持续时间，对应于三个单独的事件。 第一个持续 A 小时，第二个持续 B 小时，第三个持续 C 小时。 在这些事件中花费的每一个小时都会产生 D 类投诉。 任务是计算所有三个事件产生的投诉总数。 

问题的结构意味着扰动海洋所花费的总时间只是三个持续时间的总和。 由于每个小时独立贡献相同的固定成本，因此最终答案只是总时间乘以每小时的投诉率。 

约束非常小，所有值都限制在 1 到 100 之间。这立即排除了对典型整数类型溢出的任何担忧，并确认即使是最简单的算术方法在时间限制内也足够了。 

不存在涉及排序、重叠或部分贡献的隐藏边缘情况。 每个事件都是完全独立的，并且对结果有线性影响。 

一个常见的错误是意外地相乘而不是求和持续时间，例如计算 A × B × C × D，这在问题上下文中没有任何有意义的解释。 另一个错误是对每个事件而不是每小时应用 D，导致 (A + B + C) × D 被误解为 A × D + B × D + C × D，但在更复杂的语言中由于缺少括号或不正确的分组而错误地实现。 在 Python 中，这风险较小，但仍然值得明确。 

## 方法

 思考问题最直接的方法是模拟每个事件期间发生的情况。 我们可以从概念上迭代三个事件中每个事件的每小时，并每小时添加 D 个投诉。 这看起来就像循环 A 次，然后循环 B 次，然后循环 C 次，每次都会增加一个计数器。 这是正确的，因为它准确地反映了定义。 

然而，这种方法是不必要的，因为它执行重复的相同操作。 迭代总数为 A + B + C，最多 300 次。尽管这已经很小，但它揭示了底层结构：每次迭代都执行完全相同的工作，添加 D。 

一旦我们认识到每个小时的贡献都是独立且均匀的，我们就可以将模拟分解为单个算术表达式。 我们不是迭代，而是首先计算总时间，然后乘以 D 一次。这将整个计算减少到常数时间。 

关键的观察结果是线性的：投诉生成过程随着时间的推移而累加，并且每单位小时都是均匀的。 这允许在应用速率之前聚合所有时间间隔。

 ## 算法演练

 我们想要计算所有干扰中产生的投诉总数。 

1. 从输入中读取四个整数 A、B、C 和 D。 它们代表三个独立的持续时间和每小时统一的投诉率。 
2. 通过对三个持续时间求和来计算在海洋扰动中花费的总时间：total_time = A + B + C。此聚合有效，因为每个事件对总投诉计数都有独立贡献。 
3. 将总时间乘以 D 以获得最终的投诉数量：answer =total_time × D。此步骤将统一的每小时成本应用于合并的持续时间。 
4. 输出计算结果。 

### 为什么它有效

每个小时的活动对投诉的贡献是无法区分的。 由于比率 D 在所有时间和所有事件中都是恒定的，因此投诉总数仅取决于所花费的总小时数。 由于加法分布于乘法，(A×D)+(B×D)+(C×D)正好等于(A+B+C)×D，保证了先聚合后乘法一次的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    A, B, C, D = map(int, input().split())
    total_time = A + B + C
    print(total_time * D)

if __name__ == "__main__":
    main()
```直接按照推导公式实现。 输入被一行读取并解压缩为四个整数。 A、B 和 C 的和被显式存储以使计算结构清晰，然后乘以 D。 

使用total_time的中间变量对于正确性来说并不是必需的，但它减少了误读表达式的机会并反映了推导中使用的逻辑分解。 

所有算术都是使用 Python 整数完成的，它可以安全地处理最大可能值 (300 × 100 = 30000)。
