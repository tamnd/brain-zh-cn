 ## 解决方案
 ## 问题理解

 我们有一个长方形的花园$n$行和$m$列。 每个单元格都包含一个整数，描述那里生长的花朵的类型。 我们观察花园两次：一次在改造过程之前，一次在改造过程之后。 

假设转换以一种非常结构化的方式进行。 每种花类型都精确地映射到一种结果类型，并且这种映射在整个网格中是一致的。 这意味着如果两个单元格之前具有相同的值，那么它们之后仍然必须变为相同的值。 相反，如果两个单元格之前具有不同的值，则允许它们变得相同或保持不同，但关键的约束是转换是从旧值到新值的明确定义的函数。 

所以任务就是验证一个函数是否存在$f$这样对于每个细胞$(i, j)$，后值等于$f(\text{before}[i][j])$，并且此函数在相同前值的所有出现中都是一致的。 

限制因素$n, m \le 1000$暗示直到$10^6$每个网格的单元格。 如果单元数量是二次方，则比较每对单元或尝试显式构建成对一致性检查的解决方案将太慢。 我们应该期望对网格进行线性扫描是必要的。 

当相同的初始值映射到不同位置的两个不同输出时，会出现微妙的故障情况。 例如，如果值 5 出现在初始网格的两个单元格中，但在一个位置变为 1，在另一个位置变为 2，则立即违反假设。 另一个失败的情况是当我们意外地假定单射性或双射性时，而该语句并不要求这样做。 它仅与每个源值的一致性有关。 

## 方法

 思考该问题的一种强力方法是显式强制执行映射规则。 对于初始网格中的每个值，我们可以存储它映射到最终网格中的值集。 当我们迭代所有单元时，我们更新这个结构。 最后，每个键都必须映射到一个唯一的值。 这种方法已经接近最佳，因为每个单元格都被处理一次，但简单的实现可能会使用嵌套检查或重复扫描每个值，从而导致$O((nm)^2)$如果实施不当，最坏情况下的行为。 

关键的观察结果是，除了配对相应的单元之外，网格结构是无关紧要的。 我们从来不需要空间推理；我们不需要空间推理。 我们只需要确保对的一致性$(a_{ij}, b_{ij})$。 因此，整个问题简化为检查从整数到整数的函数是否基于观察到的样本被明确定义。 

因此，我们维护一个字典，将每个原始值映射到为其分配的转换值。 当我们遇到一对时，如果映射是新的，我们就存储它； 如果它已经存在，我们验证一致性。 任何矛盾都会立即使假设无效。

 ## 算法演练

 1. 并行读取两个网格，逐个单元，以便每个位置贡献一对$(x, y)$代表之前和之后的值。 这种配对是问题中唯一有意义的结构。 
2.维护字典`mp`记录从原始值到转换值所需的映射。 关键思想是每个原始值必须始终映射到相同的结果。 
3. 对于每个单元对$(x, y)$，检查是否`x`已经存在于`mp`。 
4.如果`x`不存在，分配`mp[x] = y`。 这会根据该值第一次出现时修复该值的转换规则。 
5. 如果`x`已经存在，请验证`mp[x] == y`。 如果不是，则立即得出结论：假设是错误的，因为单个输入值会产生多个输出。 
6. 如果所有对都通过了一致性检查，则输出“YES”。 否则，输出“NO”。 

### 为什么它有效

 该算法保持了处理任何单元格前缀后的不变量，`mp`描述与所有观察到的对一致的偏函数。 如果出现矛盾，则直接对应于违反函数定义。 相反，如果不出现矛盾，则每个输入值在其所有出现中都恰好有一个分配的输出值，这正是所需的条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    before = []
    for _ in range(n):
        before.append(list(map(int, input().split())))
    
    after = []
    for _ in range(n):
        after.append(list(map(int, input().split())))
    
    mp = {}
    
    for i in range(n):
        for j in range(m):
            x = before[i][j]
            y = after[i][j]
            
            if x in mp:
                if mp[x] != y:
                    print("NO")
                    return
            else:
                mp[x] = y
    
    print("YES")

if __name__ == "__main__":
    solve()
```该代码首先完全读取两个矩阵，然后迭代对齐的位置。 字典`mp`存储每种原始花类型的首次看到的映射。 在随后的遭遇中，我们强制执行一致性。 尽早退出矛盾很重要，因为一旦条件被打破，它可以避免不必要的扫描。

 ## 边缘情况

 一种边缘情况是某个值出现多次但只有一次出现不一致。 例如，如果值 10 出现在 1000 个单元格中，并且其中只有一个单元格的映射不同，则算法会在处理该冲突对时立即捕获它，因为字典查找的时间恒定且不依赖于频率。 

另一种情况是当值很大时（最多$10^5$），但分布稀疏。 该字典仍然有效，因为它通过实际观察到的值而不是索引位置进行键控，因此内存使用量取决于不同的值，而不是网格大小。 

最后一个微妙的情况是不同的输入产生相同的输出值。 例如，1→5和2→5是完全有效的。 该算法自然地允许这样做，因为它从不强制单射性，只强制每个键的一致性。
