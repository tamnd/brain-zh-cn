 ## 解决方案
 ## 问题理解

 我们被要求计算有多少个严格小于给定数字的正整数`n`在削尖过程中可能会出现在特殊铅笔上。 

关键约束是所有有效数字必须是素数并且不能包含数字零。 这支铅笔原本在其全长上写有一些未知的素数。 当它被锐化时，数字从最重要的一侧被一个接一个地删除，产生一个数字序列，并且每个这样的中间数字也是质数并且不包含零。 当没有剩余时，该过程停止。 我们没有得到原始数字或序列，只有最终的界限`n`，我们必须计算有多少个整数小于`n`可能出现在这样一个有效的锐化序列中的某个地方。 

所以任务不仅仅是“数出n以下的素数”。 我们必须使用附加约束来计算素数：它们的十进制表示不包含零。 锐化过程本身不会引入超出有效数字前缀的新数字，但由于任何有效前缀本身必须是不带零的素数，因此问题简化为枚举下面的所有素数`n`避免数字零。 

约束条件`n ≤ 10^9`立即排除任何尝试对所有数字进行分解或天真测试素性的方法`n`在一个简单的循环中，因为单独检查每个数字将需要多达十亿次素性检查。 甚至一个`O(√n)`每个数字的素性测试会太慢。 

一个微妙的边缘情况是包含零的数字必须被排除，即使它们是质数。 例如，`101`是素数但无效，因为它包含零。 另一个边缘情况是个位数素数，例如`2`,`3`,`5`,`7`是有效的，但是`10`即使很小也是无效的。 

主要的隐藏结构是我们实际上不需要考虑直到`n`，但仅限素数，并且我们还需要确保不出现数字零。 

## 方法

 直接的暴力解决方案将迭代所有整数`1`到`n-1`，检查每个数字是否包含零数字，然后测试它是否是素数。 数字检查很便宜，但素性测试占主导地位。 即使经过优化`O(√x)`素性测试，最坏情况成本大致变为：`sum_{x=1}^{10^9} √x ≈ 10^13`这是完全不可行的。 

问题的结构提出了不同的观点。 而不是检查每个数字最多`n`，我们只生成可能满足约束条件的候选者。 由于有效数字不能包含零，因此每个数字都来自`1`到`9`。 由于数字必须是素数，因此我们还需要将自己限制为仅素数。 

这建议结合两个想法：在允许的数字字母表上基于数字的生成`{1..9}`并仅对生成的候选者进行素数检查。 然而，生成最多 10 位的所有无零数字仍然很大（`9^10`规模），所以我们需要修剪：我们只保留素数，当数字超过时我们可以提前终止`n`。 

更实用的结构是数字 DP 样式生成或对通过附加数字形成的数字进行 BFS`1..9`，在每一步检查素数。 自从`n ≤ 10^9`，我们只探索最多 10 位数字，并通过剪枝`n`在实践中保持搜索较小。 仍然需要素性检查，但它们仅适用于生成的候选者，这些候选者是稀疏的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n√n) | O(n√n) | O(1) | O(1) | 太慢了 |
 | 数字生成+素性检查| O(k·候选者·√n)（修剪）| O(k) | 已接受 |

 这里`k`是生成的候选数，由于数字限制和修剪，它仍然是可管理的`n`。 

## 算法演练

 我们使用从空前缀开始的 DFS 构造所有不包含零数字的数字，重复附加来自`1`到`9`。 在构建过程中，我们维护当前的数值。 

1. 从一个空数开始，并将计数器初始化为零。 这代表我们的数字字符串搜索树的根。 
2. 对于每个当前数字，尝试附加来自`1`到`9`来组成一个新的号码。 我们明确避免`0`，它在构造时强制执行数字约束，而不是稍后过滤。 
3. 如果新的数字大于或等于`n`，我们停止进一步扩展它。 任何进一步的扩展只会增加该值，因此任何后代都不会有效。 
4. 如果新的数量至少为`2`，我们使用试除法直到其平方根来测试它是否为素数。 如果它是素数，我们就增加答案计数器。 
5. 从这个新数开始递归地继续DFS。 

递归自然地按长度递增的顺序探索所有有效的数字字符串，但通过修剪`n`防止爆炸。 

### 为什么它有效

 我们生成的每个数字都是仅由数字组成的正整数`1`到`9`，因此它自动满足“无零位”约束。 DFS 确保每个这样的数字都小于`n`只被访问一次。 我们只计算那些通过素数测试的素数，因此我们精确计算下面的素数`n`没有零数字。 由于任何有效数字都必须作为该构造树中的节点出现，并且我们永远不会跳过下面的任何有效构造路径`n`，完整性和正确性如下。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def is_prime(x: int) -> bool:
    if x < 2:
        return False
    if x % 2 == 0:
        return x == 2
    r = int(math.isqrt(x))
    d = 3
    while d <= r:
        if x % d == 0:
            return False
        d += 2
    return True

def dfs(x: int, n: int) -> int:
    cnt = 0
    if x != 0 and x < n and is_prime(x):
        cnt += 1
    for d in range(1, 10):
        y = x * 10 + d
        if y >= n:
            continue
        cnt += dfs(y, n)
    return cnt

def main():
    n = int(input().strip())
    print(dfs(0, n))

if __name__ == "__main__":
    main()
```该解决方案使用 DFS 增量构建数字。 初始调用从`0`，它被视为空前缀。 每一步都附加数字`1`通过`9`，确保不会出现零。 

仅对生成的候选者执行素性测试。 支票`x < n`确保我们永远不会考虑超出所需范围的值。 特殊条件`x != 0`防止对初始空状态进行计数。 

一个微妙的点是修剪：我们仅在下一个值超过或等于时停止递归`n`。 这可以防止对大型分支进行不必要的探索。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(k·√n) | O(k·√n) | 每个生成的候选者都会经过高达 √n | 的素性测试
 | 空间| O(d) | 递归深度受位数限制 (≤ 10) |

 ## 边缘情况

 一个重要的边缘情况是当`n`非常小。 为了`n = 2`，不存在小于 2 的素数，所以答案为零。 DFS仍然从`0`，但没有有效节点满足`x < n`和原始性同时存在，所以什么都不算数。 

另一种边缘情况发生在个位数素数的边界处。 为了`n = 10`， 仅有的`2, 3, 5, 7`是有效的。 DFS 从根开始准确生成这些值，并且所有值都被计数。 数字如`10`永远不会生成，因为数字`0`施工时禁止。 

对于较大的`n`，比如上面的`100`，该算法正确地探索两位数和三位数分支，但修剪任何达到或超过的分支`n`。 由于剪枝是在递归之前进行的，因此不会发生无效探索，并且限制下的所有有效素数仍然可以通过至少一条构造路径到达。
