 ## 解决方案
 ## 问题理解

 我们在 2D 平面上得到一组不同的点，每个点代表墙上的一个钉子。 任何一对钉子都可以用直绳连接起来，形成线段。 任务是选择两个钉子，使它们之间的线段尽可能水平，这意味着线段的绝对坡度最小化。 由于斜率对应于与水平轴的角度的正切，因此最小化角度相当于最小化斜率的绝对值。 

输入大小最多可达五万点。 每对的直接比较将涉及以下顺序：$n^2$运营，最坏的情况约为25亿。 这远远超出了 Python 中一秒钟所能容纳的范围，因此任何解决方案都必须避免显式检查所有对。 

当多个线段具有相同的最小斜率时，会出现一个微妙的问题。 在这种情况下，任何正确的配对都是可以接受的，因此该算法不需要超出一致性的平局打破。 

如果仅比较坐标差异而不考虑绝对斜率，简单的方法也可能会失败。 例如，仅独立比较垂直或水平差异会错过对角线近水平段，其中 x 和 y 差异都很大但比率很小。 

## 方法

 蛮力方法检查每对点，计算它们之间的斜率，并保留具有最小绝对斜率的对。 这是简单且正确的，因为每个候选者都被明确评估，因此不会错过任何最佳对。 然而，随着$n = 5 \cdot 10^4$，对的数量大致为$1.25 \cdot 10^9$，并且每个都涉及算术运算，使其不可行。 

关键的观察是最小化绝对斜率$|\frac{y_2 - y_1}{x_2 - x_1}|$相当于最小化$|y_2 - y_1| / |x_2 - x_1|$。 我们可以利用以下事实：在适当处理方向后，最佳线段将出现在按 x 坐标排序的“相邻”点之间，而不是全局搜索所有对。 

如果我们按 x 坐标对点进行排序，则对于固定点，绝对值最小斜率最接近的候选点可能来自此排序中的附近点。 直观上，大的水平间隙往往会增加分母，但也会引入更大的 y 变化，并且产生最小比率的极值权衡往往发生在 x 中接近的点之间。 这极大地减少了搜索空间：我们只需要按排序顺序检查相邻对，而不是所有对。 

这将排序后的问题转化为线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 排序+相邻检查| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们继续将候选线段集减少为由按 x 排序的连续点形成的线段集。 

1.读取所有点并将它们成对存储$(x, y)$。 该结构保留了完整的几何信息而无需转换。 
2. 按 x 坐标对点进行排序。 如果两个点共享 x（这在这个问题中不会发生，但在防御上是安全的），则任意打破平局，因为斜率将是无限的； 然而，该声明保证了不同的点，因此 x 关系并不重要。 
3. 初始化一个变量来跟踪迄今为止找到的最佳斜率和相应的点对。 我们使用交叉乘法来表示斜率比较，以避免浮点精度问题。 
4. 迭代已排序数组中的相邻对。 对于每个连续的对$(x_i, y_i)$和$(x_{i+1}, y_{i+1})$，计算绝对斜率分子和分母：$$|y_{i+1} - y_i|,\quad |x_{i+1} - x_i|$$不用除法，而是使用交叉乘法将分数与当前最佳分数进行比较。 
5. 每当当前段产生较小的绝对斜率时，更新最佳对。 
6. 输出找到的最佳对的端点。 

关键的归约步骤是对按 x 顺序排序的相邻对的限制。 这是通过斜率最小化的结构来证明的：任何不相邻的对都可以分解为中间点，这些中间点将在此排序参数下产生没有更差斜率的候选点。 

### 为什么它有效

 按 x 排序后，任何跳过 x 顺序中间点的线段都可以“收紧”，而无需增加斜率大小。 直观上，引入中间点会减少水平跨度，同时不会增加垂直不平衡而导致比率恶化。 这意味着最优对必须按排序顺序出现在连续的邻居中，因此仅扫描这些候选者就足以保证全局最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    pts = []
    for _ in range(n):
        x, y = map(int, input().split())
        pts.append((x, y))

    pts.sort()

    best_i = 0
    best_num = abs(pts[1][1] - pts[0][1])
    best_den = abs(pts[1][0] - pts[0][0])

    for i in range(1, n - 1):
        x1, y1 = pts[i]
        x2, y2 = pts[i + 1]

        num = abs(y2 - y1)
        den = abs(x2 - x1)

        if num * best_den < best_num * den:
            best_num = num
            best_den = den
            best_i = i

    x1, y1 = pts[best_i]
    x2, y2 = pts[best_i + 1]

    print(x1, y1, x2, y2)

if __name__ == "__main__":
    main()
```该实现依赖于按 x 字典顺序对点进行排序（并且 y 隐式作为辅助键）。 初始最佳对被视为第一个相邻对以确保有效的初始化。 

该比较通过使用交叉乘法来避免浮点运算，从而在整数运算下精确地保留分数的顺序。 这很重要，因为坐标最多可达$10^4$，中间产品安全地适合 64 位整数范围。 

## 工作示例

 ### 示例 1

 输入：```
3
0 2
6 4
8 2
```排序点：

 | 我| x| y |
 | ---| ---| ---|
 | 0 | 0 | 2 |
 | 1 | 6 | 4 |
 | 2 | 8 | 2 |

 我们评估相邻对。 

| 配对| Δy | Δx| 坡度|
 | ---| ---| ---| ---|
 | (0,2)-(6,4) | 2 | 6 | 1/3 | 1/3
 | (6,4)-(8,2) | 2 | 2 | 1 |

 第一对更好，所以我们输出 (0,2) 和 (6,4)。 

这证实了只有相邻对才重要，并且斜率比较纯粹基于比率。 

### 示例 2

 输入：```
3
2 4
8 2
8 6
```排序点：

 | 我| x| y |
 | ---| ---| ---|
 | 0 | 2 | 4 |
 | 1 | 8 | 2 |
 | 2 | 8 | 6 |

 相邻对：

 | 配对| Δy | Δx| 坡度|
 | ---| ---| ---| ---|
 | (2,4)-(8,2) | 2 | 6 | 1/3 | 1/3
 | (8,2)-(8,6) | (8,2)-(8,6) | 4 | 0 | 未定义（垂直）|

 垂直段在实践中被忽略，因为 Δx = 0 意味着无限斜率。 该算法有效地避免了选择它，因为任何有限的斜率都更好。 结果是 (2,4)-(8,2)，匹配最佳水平对齐。

 ## 边缘情况

 关键的边缘情况是两个点共享相同的 x 坐标。 在这种情况下，该线段是垂直的并且具有无限斜率，因此它永远不可能是最佳的。 该算法自然会忽略这些，因为 Δx = 0 会产生与任何有限斜率的无效比较，并且不会发生更新。 

另一种边缘情况是最佳对位于排序数组的边界时。 由于我们明确考虑每个相邻对，包括第一个和最后一个可能的对，因此总是评估边界最优段。 

最后，隐式处理多对具有相同最小斜率的情况，因为算法仅在严格改进时更新。 关系中的任何有效对仍然是可接受的，并且一致地返回第一个遇到的最小段。
