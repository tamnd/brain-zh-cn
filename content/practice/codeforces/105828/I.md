 ## 解决方案
 ## 问题理解

 我们给出了一组$2n$学生和不能被放入同一学习小组的配对列表。 任务是将所有学生分成两组，每组恰好包含$n$人，这样就不会出现被禁止的一对最终出现在一个群体中的情况。 我们不会被要求构建分割，只是计算存在多少个有效分割，结果取模$10^9 + 7$。 仅通过交换两个组的名称而不同的两个分割被认为是相同的。 

从图的角度来看，每个学生都是一个顶点，每个禁止对都是一条无向边。 有效分区是一种将每个顶点分配给两侧之一的方法，使得每条边都连接不同侧的顶点，并且两侧具有相同的大小$n$。 

这些约束意味着非常具体的计算结构。 顶点数最多为$2000$，而边数可以达到$10^5$。 这排除了对学生子集的任何指数枚举。 即使对所有分区进行二次检查也是不可能的，因为可供选择的方法数量太多$n$出于$2n$已经是天文数字了。 该解决方案必须依赖于图形结构和对聚合组件的动态编程。 

当图不是二分图时，会出现微妙的边缘情况。 例如，如果学生之间存在相互冲突的三角关系$1, 2, 3$，不可能将所有边放置在两个组之间，因此答案必须为零。 仅平衡组大小而不检查二分性的幼稚方法会错误地计算此类情况。 

另一个重要的边缘情况是当图是二分图但不连接时。 每个连接的组件可以独立决定哪种颜色属于 A 组。但是，这些选择会影响 A 组的最终大小，因此组件翻转的某些组合可能会超过或低于$n$。 每个组件的天真贪婪分配在这里失败，因为局部决策影响全局约束。 

## 方法

 直接的暴力解决方案会尝试所有方法来选择$n$学生出$2n$，然后检查是否没有禁止边完全位于所选组或其补集内。 这需要迭代$\binom{2n}{n}$子集并检查所有$m$每个子集的边。 即使是为了$2n = 20$，这变得不可行，并且在实际约束下$2n = 2000$，这是完全不可能的。 

关键的观察是禁止边仅施加二分约束。 如果图形包含奇数环，则根本不存在有效分配。 否则，每个连接的组件在结构方面都是独立的：一旦我们修复了一个顶点的一侧，整个组件就被确定为全局翻转。 

对于每个连接的组件，我们可以使用 BFS 或 DFS 计算二分。 假设在组件的一种任意有效着色中，着色为 0 的顶点数为$a_i$，元件尺寸为$s_i$。 如果我们翻转该组件的颜色，则该组件对 A 组的贡献变为$s_i - a_i$。 因此，每个组件都贡献一个二元选择，影响 A 组的总大小。 

这将问题简化为为每个组件选择两个值之一，以便所选值的总和恰好等于$n$。 这是一个经典的组件背包式动态规划问题。 唯一剩下的微妙之处是最终的着色被计算两次，每次两组的全局交换计算一次，所以我们必须将最终的答案除以 2。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 |$O(\binom{2n}{n} \cdot m)$|$O(n)$| 太慢了 |
 | 组件+DP背包|$O(n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 我们首先将学生视为图的顶点，并从禁止对中构建邻接表。 

1. 我们对每个未访问的顶点运行 BFS 或 DFS，以探索其连接的组件。 在此遍历过程中，我们为每个顶点分配二分颜色 0 或 1。 如果我们遇到一条边连接两个相同颜色的顶点的冲突，则该图不是二分图，答案立即为零。 此步骤确保任何禁止对都不能仅通过结构强制进入一组。 
2. 对于每个连通分量，我们计算有多少个顶点接收到颜色 0。令该值为$a_i$，并令组件尺寸为$s_i$。 我们将这一对隐式存储为两个可能的贡献：$a_i$为 A 组做出贡献，或$s_i - a_i$确实如此，取决于我们是否翻转组件。 
3. 我们初始化一个动态编程数组，其中$dp[x]$表示处理某些组件前缀的方法数量，以便准确地$x$学生被分配到 A 组。最初，$dp[0] = 1$。 
4. 对于每个组件，我们构建一个新的 DP 数组。 从现有的每笔金额中$x$，我们过渡到$x + a_i$和$x + (s_i - a_i)$，相应地添加路数。 此步骤编码是否翻转组件的决定。 
5. 处理完所有组件后，得到的值$dp[n]$给出有效颜色分配的数量，其中 A 组恰好有$n$学生。 
6. 由于交换两个组不会创建新的有效分区，因此每个有效分区在此 DP 中被计数两次，每次全局颜色交换一次。 我们将最终答案乘以 2 的模逆。 

### 为什么它有效

 二分着色步骤保证每个禁止边都位于每个组件内的相反颜色之间，因此无论组件如何翻转，都不会违反条件。 每个连接的组件在结构上是独立的，但为 A 组贡献了两种可能大小的固定选择。DP 枚举了这些独立选择的所有一致组合，最终的约束确保了全局平衡。 唯一的多算来自于交换两组的对称性，这会均匀地影响每个解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
INV2 = (MOD + 1) // 2

n, m = map(int, input().split())
N = 2 * n

g = [[] for _ in range(N + 1)]
for _ in range(m):
    a, b = map(int, input().split())
    g[a].append(b)
    g[b].append(a)

color = [-1] * (N + 1)
components = []

from collections import deque

for i in range(1, N + 1):
    if color[i] != -1:
        continue
    q = deque([i])
    color[i] = 0
    cnt0 = 0
    size = 0
    ok = True

    while q:
        v = q.popleft()
        size += 1
        cnt0 += (color[v] == 0)

        for to in g[v]:
            if color[to] == -1:
                color[to] = color[v] ^ 1
                q.append(to)
            elif color[to] == color[v]:
                ok = False

    if not ok:
        print(0)
        sys.exit(0)

    cnt1 = size - cnt0
    components.append((cnt0, cnt1))

dp = [0] * (n + 1)
dp[0] = 1

for a, b in components:
    ndp = [0] * (n + 1)
    for i in range(n + 1):
        if dp[i] == 0:
            continue
        if i + a <= n:
            ndp[i + a] = (ndp[i + a] + dp[i]) % MOD
        if i + b <= n:
            ndp[i + b] = (ndp[i + b] + dp[i]) % MOD
    dp = ndp

ans = dp[n] * INV2 % MOD
print(ans)
```图构建步骤构建邻接列表，以便二分检查在边数上变得线性。 BFS 部分既检查可行性，又同时计算每个组件的大小划分。 动态规划数组的边界为$n$， 不是$2n$，因为我们只跟踪一组的大小。 

最终乘以$INV2$纠正交换两组标签产生相同分区的固有对称性。

 ## 边缘情况

 一个关键的边缘情况是没有禁止对时。 在这种情况下，该图由以下部分组成$2n$孤立的顶点，每个顶点充当大小为 1 的组件。每个顶点根据其翻转选择向 A 组贡献 0 或 1，因此 DP 变成二项式系数计算。 为了$n=2$， 有$\binom{4}{2} = 6$分配，除以 2 产生 3 个有效分区，与预期的组合结果匹配。 

另一种边缘情况是单个奇数循环，例如三角形。 在 BFS 着色过程中，算法最终会为一条边的两个端点分配相同的颜色，从而立即触发拒绝。 这可以防止任何 DP 计算，这是必要的，因为 DP 假定二分结构。 

最后一个微妙的情况是，当所有组件都是循环或树时，但它们的大小贡献使得不可能准确地达到$n$。 在这种情况下，DP 数组仅以以下结尾$dp[n] = 0$，正确地表明即使局部二分性成立，也没有全局分配满足大小约束。
