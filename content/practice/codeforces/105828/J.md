 ## 解决方案
 ## 问题理解

 我们得到了几个独立的矩形，它们代表了棋盘的可能形状。 对于每块板，我们需要计算有多少种方法可以将轴对齐的矩形“窗口”放置在其中，以使窗口具有固定的区域`s`，并且它不接触板的任何边界。 

安置由两个选择决定。 首先，我们选择整数边长`(x, y)`这样`x * y = s`。 其次，我们选择一个位置`x × y`内的矩形`a × b`木板。 窗口必须严格位于内部，这意味着窗口和棋盘的每个边界之间必须至少有一个空单元格。 所以如果窗口有大小`x × y`，有效放置的数量为`(a - x - 1) * (b - y - 1)`前提是这两个术语都是积极的。 不允许旋转板子，但可以交换`x`和`y`对应于选择不同的因式分解，因此两个方向都很重要。 

我们重复这个计算最多`2 · 10^5`板，以及`s`可以大到`10^15`。 这立即排除了任何每次查询的因式分解`s`直到其平方根。 相反，我们需要对所有除数进行预先计算`s`，这是可行的，因为约数的数量最多约为`10^5`在最坏的情况下，通常要小得多。 

当窗口至少在一个维度上太大时，就会出现微妙的失败情况。 如果`x >= a - 1`或者`y >= b - 1`，则没有有效的放置，因为窗口触及或超出了所需的一个单元格边框边距。 另一个边缘情况是当`s = 1`，其中唯一的窗口是`1 × 1`，并且该公式简化为计算板的内部单元格，它仍然必须遵守“一个单元格边框”限制。 

## 方法

 直接方法将独立处理每个板。 对于固定板`(a, b)`，我们可以枚举所有因子对`(x, y)`这样`x * y = s`，并为每对计算放置的数量。 这是正确的，因为每个有效窗口都必须对应于某些因式分解`s`。 然而，通过扫描最多来枚举除数`sqrt(s)`每个查询是不可能的，当`s`达到`10^15`并且有`2 · 10^5`查询，大致导致`10^10`最坏情况下的操作。 

关键的观察是`s`在所有查询之间共享。 我们可以将它分解一次，生成所有除数对`(x, y)`，并将它们重复用于每个板。 每个查询的剩余工作仅与除数的数量成正比`s`，而不是其大小。 

对于每个除数`x`的`s`，我们定义`y = s / x`。 每对订购的`(x, y)`和`(y, x)`对应于窗口的潜在不同方向。 对于每个方向，我们独立计算放置数量并累积结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个查询的暴力破解 | O(t √s) | O(t √s) | O(1) | O(1) | 太慢了 |
 | 预计算 s | 的除数 O(√s + t · d(s)) | O(√s + t · d(s)) | O(d(s)) | O(d(s)) | 已接受 |

 这里`d(s)`是的约数`s`。 

## 算法演练

 我们首先预先计算所有除数`s`。 自从`s ≤ 10^15`，迭代至`sqrt(s)`一次是可行的。 

1. 迭代`i`从`1`到`⌊√s⌋`。 对于每个`i`， 如果`s % i == 0`, 记录两者`i`和`s / i`作为除数对。 这会产生所有因子对`(x, y)`这样`x * y = s`。 
2.对于每个查询板`(a, b)`,初始化一个累加器`ans = 0`。 
3. 对于每个除数`x`的`s`, 计算`y = s / x`。 
4. 治疗`(x, y)`作为窗口方向。 如果`x < a - 1`和`y < b - 1`，那么有效放置的数量是`(a - x - 1) * (b - y - 1)`。 将此值添加到`ans`。 
5. 对交换的方向重复相同的检查`(y, x)`如果`x != y`，因为它代表一个不同的矩形，除非它是正方形。 
6. 输出`ans mod (10^9 + 7)`。 

关键的设计选择是严格遵守不等式`a - x - 1`和`b - y - 1`。 这直接编码了窗口每一侧必须保留至少一个单元格的要求。 

### 为什么它有效

 每个有效的配置都唯一对应于因式分解的选择`s`进入`(x, y)`以及放置一个`x × y`板内至少有一个单元格填充的矩形。 计数`(a - x - 1) * (b - y - 1)`枚举此类窗口的所有有效左上角位置，因为左上角必须位于行中`1`通过`a - x - 1`和列`1`通过`b - y - 1`。 对所有因式分解求和在每个方向上恰好覆盖了所有可能的窗口形状，并且不包括无效放置，因为任何违反边界约束的行为都会使公式中至少有一个因子为非正数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

t, s = map(int, input().split())

divs = []
i = 1
while i * i <= s:
    if s % i == 0:
        j = s // i
        divs.append((i, j))
        if i != j:
            divs.append((j, i))
    i += 1

for _ in range(t):
    a, b = map(int, input().split())
    ans = 0

    for x, y in divs:
        if x < a - 1 and y < b - 1:
            ans += (a - x - 1) * (b - y - 1)
        if x != y and y < a - 1 and x < b - 1:
            ans += (a - y - 1) * (b - x - 1)

    print(ans % MOD)
```除数预计算在处理查询之前完成一次。 每个查询仅迭代除数列表。 条件`x < a - 1`确保至少有一个有效行可供窗口启动，因为窗口必须在底部留出至少一个单元格的边距。 同样的推理对称地适用于列。 

我们明确处理这两个方向。 对于非平方因式分解，`(x, y)`和`(y, x)`不同，所以两者都必须计算在内。 对于正方形，我们避免重复计算。

 考虑第一个样本：

 输入板有`(5, 6)`,`(5, 5)`,`(6, 5)`和`s = 4`。 除数是`(1, 4)`,`(2, 2)`,`(4, 1)`。 

板用`(5, 6)`，我们测试每个形状：

 | x| y | 5×6 有效 | 贡献 |

 | 4 | 1 | 是的 | (0)*(4)=0 |

 总计为`9`。 

为了`(5, 5)`， 仅有的`(2, 2)`有意义，因为`(1,4)`和`(4,1)`至少在一个维度上违反边界约束。 

| x| y | 5×5 有效 | 贡献 |
 | ---| ---| ---| ---|
 | 2 | 2 | 是的 | (2)*(2)=4 |

 总计为`4`。 

为了`(6, 5)`, 对称性`(5, 6)`给出相同的总数`9`。 

这些痕迹表明，每个因式分解都是独立贡献的，并且无效方向自然会被不等式检查过滤掉。

def run(inp: str) -> str:

    divs = []
    i = 1
    while i * i <= s:

            j = s // i

                divs.append((j, i))

        ans = 0
        for x, y in divs:
            if x < a - 1 and y < b - 1:
                ans += (a - x - 1) * (b - y - 1)
            if x != y and y < a - 1 and x < b - 1:
                ans += (a - y - 1) * (b - x - 1)
        out.append(str(ans % MOD))

    return "\n".join(out)

 ## 边缘情况

 当棋盘相对于任何除数而言太小时`s`，每一项都被条件过滤掉`x < a - 1`。 例如，与`s = 4`和板`2 × 10`， 虽然`(1,4)`存在，`1 < 1`失败，所以贡献为零。 该算法自然地处理这个问题，无需特殊的外壳。 

什么时候`s = 1`，唯一的除数是`(1,1)`。 条件简化为`(a - 2) * (b - 2)`，这符合窗口必须完全位于内部并具有一个单元格边距的要求。 

什么时候`s`是完全平方数，需要注意避免重复计算`(x, x)`。 代码检查`x != y`在添加交换的方向之前，确保正确计数而不重复。
