 ## 解决方案
 ## 问题理解

 我们有一行单元格，每个单元格包含一个小写字母。 令牌从第一个单元格开始，并根据确定性规则重复移动。 从当前位置开始，如果整行中至少存在一个其他单元格包含相同的字母，则令牌跳转到其中一个单元格； 否则向右移动一步。 仅当令牌移过最后一个单元格时，该过程才会停止。 

问题不是模拟随机性，而是确定是否存在任何方法可以让该过程永远持续下去，这意味着令牌永远不会超出最后一个位置，因为它不断陷入由重复字母创建的循环中。 

输入大小最多可达 100000 个字符，这排除了任何逐步探索多个步骤的转换的模拟。 任何可以在没有检测的情况下重复访问状态的方法都会面临指数或无限运行时间的风险。 正确的解决方案必须将过程简化为字符串的结构属性。 

当重复的字母形成循环通过前向索引时，会出现微妙的失败情况。 例如，在`aaa`，从位置 1 可以转到位置 2，从 2 到位置 3，再从 3 回到位置 1，从而产生一个永远无法到达终点的循环。 天真的只向前的直觉会忽略由于“任何其他事件”而可能发生的向后跳跃。 

另一个边缘情况是像这样的字符串`abca`。 即使字母重复，该结构也不一定会形成阻塞出口的循环，因为运动最终仍然可以到达不存在前向捕获循环的位置。 

## 方法

 暴力解释将每个位置视为一个状态并重复应用转换规则。 从给定的索引 i 开始，我们查看所有出现的 s[i]，选择其中一个，然后移动到那里； 否则我们转到 i + 1。这自然定义了一个位置上的有向图。 我们可以尝试模拟所有可能的路径或运行从状态 1 到状态 n + 1 的可达性分析。 

问题是每个节点可能有许多其他节点的传出边，而这些边可能会产生循环。 在最坏的情况下，对可达状态的全面探索是指数级的，因为每个字母组在其出现之间引起密集的连接。 

关键的观察结果是随机性是无关的。 唯一重要的是强制转移的诱导有向图中是否存在循环。 如果从某个索引我们可以到达左侧具有相同字母的另一个索引，那么我们可以在该字母类的出现之间循环，并且永远不会前进到右边界。 

这将问题简化为检测是否存在在位置 i < j < k 中至少出现两次的字母，以便移动可以在同一字母类内向后返回。 事实上，该结构进一步简化：如果任何字母至少出现两次，那么从该字母的第一次出现开始，该过程可以跳转到后来的出现，并且从那里唯一可能的重复跳转使我们保持在同一组索引内，从而防止有保证的逃逸。 

因此，问题归结为检查字符串是否有重复字符。 如果每个字符恰好出现一次，则令牌永远不会向后跳转或循环，因此它严格向右移动并最终退出。 如果任意字符重复，则存在可达循环。

 ## 算法演练

 1. 读取字符串并从左到右扫描时计算每个字符的频率。 此步骤捕获任何字母是否创建多个可能的目的地。 
2. 如果任意字符出现多次，则立即断定该进程可能陷入循环并输出NO。 原因是重复的字母引入了替代移动，可以将令牌从严格增加的索引中重定向出来。 
3. 如果所有字符都不同，则得出结论，不可能有向后或横向循环，因此每一步要么严格移动到唯一的未访问位置，要么退出数组，从而保证终止。 

### 为什么它有效

 仅当某个位置的字母再次出现时，该过程才会创建非转发行为。 该条件相当于存在重复字符。 一旦存在重复项，系统将在由相同字母引起的索引上包含至少一个强连接组件，从而允许重新访问并防止有保证的逃逸。 如果不存在重复项，则每个位置都只有一种前进方式，并且没有替代转换，从而强制执行严格递增的路径，直到终止。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
cnt = [0] * 26

for ch in s:
    cnt[ord(ch) - 97] += 1

for x in cnt:
    if x > 1:
        print("NO")
        break
else:
    print("YES")
```该解决方案在 26 个小写字母上维护一个固定大小的频率数组，这已经足够了，因为字母表是固定的。 每个字符都会以恒定的时间递增其计数器。 

最后一个循环检查是否有任何字符重复。 这`else`循环上的子句确保我们只打印`YES`如果没有发现重复。 这避免了额外的状态跟踪并保持逻辑线性和干净。 

## 工作示例

 ### 示例 1：`abc`| 步骤| 人物 | 更新频率 | 发现重复 |

 | 3 | c | c:1 | 没有 |

 由于没有字母重复，因此算法输出 YES。 这反映出每一步都是被迫向前的，并且令牌最终必须离开阵列。 

### 示例 2：`aaa`| 步骤| 人物 | 更新频率 | 发现重复 |
 | ---| ---| ---| ---|
 | 1 | 一个 | 一个：1 | 没有 |
 | 2 | 一个 | 一个：2 | 是的 |
 | 3 | 一个 | 一个：3 | 是的 |

 立即检测到重复项，因此输出为“否”。 这对应于在出现的情况之间跳转的能力`a`并保持在一个闭环内。

 ## 边缘情况

 对于像这样的单字符字符串`a`，算法计算一次并没有发现重复项，因此输出 YES。 该进程简单地向右移动一次并立即退出。 

对于像这样的字符串`abac`, 字符`a`出现多次。 频率阵列标记重复，触发 NO。 这与具有相同字母的多个位置的存在相匹配，这使得能够重新访问较早的索引并在转换图中形成循环。
