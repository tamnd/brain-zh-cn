 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中都有一组卡片，每张卡片上都写有两个数字。 卡片正常放置时可见一位数字，翻转卡片后出现另一位数字。 

我们可以将所有卡片以任意顺序排列成一行，并且对于每张卡片，我们还选择哪一面最初朝上。 从左到右读取，可见数字形成一个十进制数，我们称之为 m1。 然后将所有卡片翻转，因此每张卡片都显示其相反的数字，再次读取会给出另一个数字 m2。 这两个数字都不允许以零开头。 

任务是构建每张卡片的顺序和方向，使得绝对差 |m1 − m2| 尽可能小。 我们必须输出结果数字对，而不仅仅是最小值。 

关键的结构细节是每张卡片都提供一对数字，这些数字在翻转后总是交换。 如果一张卡片在 m1 中显示 xi，则在 m2 中显示 yi，反之亦然。 

限制很大：测试用例中总共最多有 2·10^5 个卡。 这迫使每个测试用例都有一个线性或近线性的解决方案。 尝试所有排列或使用指数搜索的任何解决方案都是不可能的。 

一个微妙的约束是前导零限制。 如果 m1 或 m2 的第一个数字变为零，则即使数值差在其他情况下是最佳的，构造也是无效的。 这主要影响最重要的位置，需要特殊处理。 

当一个人试图独立优化每个位置而不考虑位置权重时，就会出现一个幼稚的错误。 例如，通过最小化局部数字差异来贪婪地处理每张牌可能会失败，因为高位置的微小变化支配了所有较低位置。 

另一种失败模式是忽略翻转对称地交换数字。 如果一张牌是 (0, 9)，选择方向会影响相反方向的两个数字，因此局部选择会传播到全局不平衡。 

## 方法

 暴力策略将尝试卡片的所有排列以及每张卡片的两个方向。 这会产生 n! · 2^n 个配置，计算每个配置的 m1 和 m2 需要 O(n)，即使 n = 10 也会导致不可能的爆炸。 

为了简化结构，请观察这两个数字的不同之处仅在于我们如何为每张卡分配方向。 一旦固定，每个位置都会贡献一对数字（a，b），其中相反的数字始终是（b，a）。 因此，每张卡都会贡献一个仅取决于其所选方向的有符号数字差异，并且这种效果会通过位置权重 10^k 放大。 

这将问题转化为将每张牌分配到一个位置，并选择一个方向，在该位置权重上添加或减去其数字差异。 由于十进制位置结构，高位支配低位，因此最小化第一个不同的高位比微调低位重要得多。 

关键的见解是分别控制两件事。 首先，我们决定哪些牌放在更重要的位置，确保内部差异较大的不稳定牌不会影响高位数字。 其次，我们在构造数字时贪婪地选择方向，以使偏差尽可能接近于零。

这导致按照 |xi − yi| 的大小对卡片进行排序。 内部不对称性较小的牌在高位置更安全，而内部不对称性较大的牌应推到右侧，其效果会被 10 次方打折。 

在此排序之上，我们从左到右构建数字，保持前缀之间的当前差异并选择每个方向以使不断变化的差异的绝对值保持最小。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n!·2^n) | O(n!·2^n) | O(n) | 太慢了 |
 | 排序贪婪 + 构造 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 ### 1. 按数字差异对卡片进行分组

 对于每张卡，计算 d = |xi − yi|。 这衡量了卡片对最终差异的影响程度。 d 较小的牌更稳定，更适合早期位置。 

### 2.慎重选择第一个位置

 我们必须确保两个数字都不以零开头。 任何 xi 和 yi 均非零的卡都是安全的，因为任一方向都保持前导数字有效。 其中，我们选择一个放在最重要的位置。 

有效的领先卡牌的选择对于最优性来说并不重要，但是在这里放置一张稳定的卡牌可以避免无效的构造。 

### 3. 对剩余的卡片进行排序

 所有剩余的卡片均按非降序 |xi − yi| 排序。 这确保了具有较大内部不平衡的卡片被推向较低重要性的位置。 

这种排序降低了单一大不平衡主导最终差异的风险。 

### 4. 从左到右构建序列

 我们同时构造 m1 和 m2，同时保持它们的前缀之间的运行差异。 

假设我们在每个位置放置一张卡片。 我们通过比较两种可能的结果来确定其方向：显示 xi 与 yi。 我们计算每个选择在将前一个前缀移动一位小数（乘以 10）后如何改变当前差异，并选择产生新差异的较小绝对值的方向。 

这种贪婪的选择之所以有效，是因为未来头寸的影响总是小至少 10 倍。 

### 5.直接记录数字

 我们不是以算术方式构造数字，而是将 m1 和 m2 的数字存储为字符串或数组。 这可以避免溢出并保持构造的明确性。 

### 为什么它有效

 该过程保持了不变式，即在放置 i 张卡片后，在给定固定顺序的情况下，构造的 m1 和 m2 前缀是为这些 i 位置分配方向的所有方法中可能最好的。 由于每个新位置的权重严格低于所有先前位置的权重总和，因此较高位置的任何次优选择都无法在以后纠正。 贪婪的方向选择确保在每一步，前缀差异最小化，这直接限制了最终的绝对差异。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        cards = []

        good_first = []

        for _ in range(n):
            x, y = map(int, input().split())
            cards.append((x, y))
            if x != 0 and y != 0:
                good_first.append((abs(x - y), x, y))

        # choose leading card
        if good_first:
            good_first.sort()
            _, fx, fy = good_first[0]
            cards.remove((fx, fy))
            first = (fx, fy)
        else:
            # fallback (problem guarantees at least one good card exists)
            first = cards.pop()

        # sort remaining by |x-y|
        cards.sort(key=lambda c: abs(c[0] - c[1]))

        order = [first] + cards

        m1 = []
        m2 = []
        diff = 0  # current prefix difference (m1 - m2)

        for i, (x, y) in enumerate(order):
            if i == 0:
                # fix orientation to avoid leading zero
                if x == 0:
                    m1.append(str(y))
                    m2.append(str(x))
                    diff = diff * 10 + (y - x)
                else:
                    m1.append(str(x))
                    m2.append(str(y))
                    diff = diff * 10 + (x - y)
                continue

            # option 1: x on m1, y on m2
            d1 = diff * 10 + (x - y)
            # option 2: y on m1, x on m2
            d2 = diff * 10 + (y - x)

            if abs(d1) <= abs(d2):
                m1.append(str(x))
                m2.append(str(y))
                diff = d1
            else:
                m1.append(str(y))
                m2.append(str(x))
                diff = d2

        out.append(str(int("".join(m1))) + " " + str(int("".join(m2))))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先分隔一张安全的前导卡，其中两个数字均非零，以避免无效的前导零。 这张卡放在第一位。 

所有剩余的卡片均按其内部数字差异进行排序，以便不稳定的卡片被推向不太重要的位置。 

然后我们同时构造两个数字。 变量`diff`当我们从左到右添加数字时，跟踪当前的前缀差异 m1 - m2。 每个新数字都会将之前的差异移动 10 倍，我们测试当前卡片的两种可能方向，选择使运行差异的绝对值最小的方向。 

最终数字构建为字符串并转换为整数，仅用于输出格式，确保不存在前导零问题。

 ## 边缘情况

 一个关键的边缘情况是只有一张卡的两位数字都非零。 这张卡必须放在第一位，否则在两种解释之一中，构造的数字可能以零开头。 该算法在对其余卡片进行排序之前显式提取这样的卡片。 

当许多牌具有相同的数字时会出现另一种边缘情况，例如 (7,7)。 无论放置或方向如何，这些卡的贡献都为零。 算法自然地将它们放置在任何地方，而不影响运行差异，贪婪步骤离开`diff`不变。 

第三种情况涉及交替的极端值，例如 (0,9) 和 (9,0)。 在这里，方向选择非常重要。 贪婪更新规则确保选择产生较小直接前缀差异的方向，从而防止早期发散锁定较大的最终间隙。
