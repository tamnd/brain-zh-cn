 ## 解决方案
 ## 问题理解

 我们得到了一个二进制字符串的集合，它们的长度都相同。 每个字符串都是一种语言中的一个单词，我们只能通过删除其右端的字符来缩短单词。 在缩短所有单词之后，我们希望保持一个属性：对于每个单词，其缩短版本仍然足以在整个集合中唯一地确定原始单词。 

形式上，我们为每个单词选择一个前缀，并且该前缀不能是缩写后任何其他单词的前缀。 同样，不允许两个单词共享相同的缩短表示，并且对于每个单词，我们希望保留最短的前缀，以将其与所有其他单词区分开来。 

输出是每个单词的新版本，其中每个单词都被其最小区分前缀替换。 

这里的限制非常重要。 最多可以有 80,000 个单词，每个单词的长度最多为 32。这立即排除了 n 的任何二次方。 比较每对字符串或重复扫描每个前缀的所有单词的解决方案的行为类似于 O(n²k)，这太慢了。 k 的小值表明我们应该将每个单词视为固定大小的位对象并利用按位结构或尝试。 

当许多单词共享长前缀时，就会出现一个微妙的问题。 例如，如果我们有像 110000、110001、110010 这样的单词，那么区分前缀可能会很长，并且多个单词仅在最后位出现分歧。 独立地停止在每个单词的第一个不同字符处的天真想法是不正确的，因为唯一性取决于整个集合，而不是孤立的成对比较。 

另一种边缘情况是所有字仅在最后一位不同。 那么每个单词必须保持全长减去一个前缀。 任何仅查看浅层前缀的直接邻居或本地频率的方法都会低估所需的前缀长度。 

## 方法

 直接的暴力方法是，对于每个单词，尝试将前缀长度从 1 增加到 k 并检查该前缀是否出现在任何其他单词中。 检查前缀需要扫描所有单词或使用散列，这在最坏的情况下仍然会导致 O(n²k)。 对于 80,000 个单词，即使每个前缀长度的线性扫描也远远超出了限制。 

关键的观察是，我们只关心单词之间的区别，两个单词之间的区别点是它们不同的第一个位置。 如果我们可以组织所有单词，以便将相似的前缀分组在一起，我们就可以本地化比较。 这自然表明了比特上的二进制字典树。 

当将所有单词插入到特里树中时，每个节点代表一个前缀，我们可以存储有多少个单词通过它。 当与前缀对应的节点的计数为 1 时，前缀足以准确地区分单词。也就是说，没有其他单词共享它。 

因此，我们不需要将一个单词与所有其他单词进行比较，只需在 trie 中遍历它的路径并找到子树大小变为 1 的第一个节点。该深度是该单词的最短唯一前缀长度。 

这减少了从全局比较到局部子树计数的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（检查所有单词的所有前缀）| O(n²k) | O(n²k) | O(1) 或 O(nk) | 太慢了 |
 | 具有子树计数的 Trie | O(nk) | O(nk) | O(nk) | O(nk) | 已接受 |

 ## 算法演练

 我们使用二进制 trie 分两次处理所有单词。

1. 构建一个二进制 trie，其中每个节点对应一个位前缀。 对于每个单词，我们从根开始，沿着每个位的边缘，如果需要的话创建节点。 在每个节点，我们都会增加一个计数器，用于存储通过该节点的单词数量。 该计数器表示有多少单词共享该前缀。 
2. trie构建完成后，我们再次独立处理每个单词。 我们沿着特里树从根开始沿着它的各个部分走下去。 
3. 在遍历单词时，我们跟踪深度。 在每个节点，我们检查存储的计数器。 该计数器第一个变为 1 的位置是该前缀在所有单词中最早唯一的点。 
4. 我们立即在该深度处停下来，并将其记录为该单词所需的缩短长度。 
5. 每个单词的输出只是其到该深度的前缀。 

我们可以立即停止的原因是，一旦前缀是唯一的，任何较长的前缀也是唯一的，但较短的前缀则不然，因此第一次出现是最佳的。 

### 为什么它有效

 每个 trie 节点精确对应一个二进制前缀。 节点处存储的计数器等于共享该前缀的输入单词的数量。 如果一个节点的计数器为 1，则恰好有一个单词到达该节点，因此前缀在整个集合中唯一标识该单词。 如果计数器大于 1，则至少有一个其他单词共享相同的前缀，因此尚不能使用。 因此，最小区分前缀恰好是单词路径上计数器为 1 的第一个节点。这保证了最小性，因为我们按长度递增的顺序遍历前缀。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Node:
    __slots__ = ("child", "cnt")
    def __init__(self):
        self.child = [-1, -1]
        self.cnt = 0

def solve():
    n, k = map(int, input().split())
    words = [input().strip() for _ in range(n)]

    trie = [Node()]

    def add(word):
        v = 0
        trie[v].cnt += 1
        for ch in word:
            b = ord(ch) - 48
            if trie[v].child[b] == -1:
                trie[v].child[b] = len(trie)
                trie.append(Node())
            v = trie[v].child[b]
            trie[v].cnt += 1

    for w in words:
        add(w)

    out = []

    for w in words:
        v = 0
        prefix = []
        ans_len = k
        for i, ch in enumerate(w):
            b = ord(ch) - 48
            v = trie[v].child[b]
            prefix.append(ch)
            if trie[v].cnt == 1:
                ans_len = i + 1
                break
        out.append("".join(prefix[:ans_len]))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现使用一个简单的基于数组的 trie，其中每个节点存储两个子节点（位 0 和 1），以及一个计数器，用于记录有多少单词通过它。 第一遍构建结构并填充子树计数。 第二遍遍历每个单词，直到到达计数为 1 的节点。 

一个常见的错误是忘记增加根计数器，这是必要的，以便计数保持一致并且每个节点反映通过它的正确单词数。 另一个微妙之处是确保我们在第一次出现 cnt == 1 时停止而不是继续，因为继续仍然是正确的，但不会产生最小前缀。 

## 工作示例

 考虑一小组单词：110、111、100。 

构建特里树后，根分裂为 1 和 0。在 1 下，我们有 10 和 11 个分支。 前缀100对应的节点早期是唯一的，而110和111只在最后一位分开。

 ## 边缘情况

 一个重要的边缘情况是所有单词直到最后一个字符都相同。 例如，0000、0001、0010、0011。 trie 将具有较大的共享前缀，并且唯一性仅出现在深度 3 或 4 处，具体取决于分支。 该算法通过确保计数器在最终分歧之前保持大于 1 来处理此问题，因此不会过早切掉单词。 

另一个边缘情况是单词仅在最后一个字符处变得唯一。 在这种情况下，遍历到达最后一个节点，但仍然没有看到之前的 cnt == 1，因此 ans_len 变为 k。 这保留了正确性，因为没有更短的前缀可以区分它。 

第三种情况是字在第一位立即发散。 然后，第一个 trie 级别的每个分支已经有 cnt == 1，并且每个单词都减少到长度 1。该算法自然地捕获了这一点，无需特殊处理，因为根子级立即计数拆分。
