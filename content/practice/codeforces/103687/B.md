 ## 解决方案
 ## 问题理解

 我们得到一个仅由小写英文字母组成的字符串。 任务是从左到右扫描该字符串，并且每当连续字符形成子字符串时`"cjb"`，我们必须在该事件之后立即插入一个逗号。 字符串的其余部分保持不变，并且必须以尊重输出从左到右结构的方式处理重叠或重复出现的情况。 

输入大小可以高达 100000 个字符，这意味着任何重复构建子字符串或对字符串执行嵌套扫描的解决方案都会太慢。 需要一种与字符串长度呈线性关系的算法，因为在 Python 中，大约 100000 次操作是 1 秒时间限制内唯一合适的预算。 

检查每个位置并通过切片重建字符串的简单方法可能会以微妙的方式失败。 一个问题是由于重复的字符串连接而导致性能下降，在最坏的情况下会变成二次。 另一个问题是，如果在迭代时尝试就地修改字符串，则会错误地处理重叠模式。 

例如，考虑输入`"cjbjb"`。 子串`"cjb"`出现在开头。 插入逗号后，我们得到`"cjb,jb"`。 不正确地移动索引的粗心方法可能会跳过字符或尝试在修改的区域内重新匹配，从而产生重复的逗号或丢失有效匹配。 

另一种边缘情况是没有出现的字符串，例如`"abcdef"`，其中输出必须与输入相同。 由于错误的边界检查，不正确的方法仍可能引入分隔符。 

## 方法

 蛮力的想法很简单：迭代每个索引`i`在字符串中，并对于每个位置检查子字符串是否从`i`比赛`"cjb"`。 如果有，请追加`"cjb,"`到结果并适当地向前跳。 否则，追加当前字符。 

这种方法是正确的，因为它直接遵循任务的定义。 问题在于如何构建结果以及如何推进指数的效率。 如果使用重复的字符串连接来实现，则每个追加操作的成本可能为 O(n)，最坏的情况为 O(n²)。 对于 n 高达 100000 的情况，这太慢了。 

关键的观察是，我们永远不需要重新访问以前的角色，并且我们永远不需要在看到的最后几个角色之外维持任何复杂的状态。 这使我们能够一次性处理字符串，同时维护最近字符的小型滚动缓冲区。 一旦缓冲区匹配`"cjb"`，我们发出它并立即插入一个逗号。 

这将问题变成了流式构建任务：我们增量构建输出，每一步仅检查最后三个字符。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们逐个字符地处理字符串，同时保持不断增长的输出缓冲区。 

1.初始化一个空列表`out`有效地存储输出字符。 使用列表而不是字符串来避免二次串联成本。 
2. 迭代每个字符`ch`在输入字符串中，将其附加到`out`。 追加的时间复杂度为 O(1)。 
3. 每次追加后，检查最后三个字符是否在`out`形式`"cjb"`。 此检查仅在当前长度至少为 3 时有效。 
4. 如果找到匹配项，请在后面添加逗号`out`。 我们不删除字符或回溯，因为模式是固定长度的，并且我们希望保留原始字符加上插入的逗号。 
5. 继续直到字符串末尾。 
6. 加入列表`out`转换成最终的字符串并输出。 

第 3 步安全的原因是我们只需要检测以当前位置结束的模式。 由于我们从左到右处理并且从不重新访问输出的早期部分，因此任何有效的出现都必须以当前索引结束。 

### 为什么它有效

 在每一步中，构造的前缀为`out`正是输入字符串的相应前缀根据规则“在每次出现后插入一个逗号”的转换版本`cjb`到目前为止看到的”。该算法永远不会删除或重新排列字符，只会添加新字符。任何出现`"cjb"`输入中的逗号也连续出现在输出中，并且检查确保逗号每次出现时恰好插入一次，在第一时刻它在流末尾变得可见。 这可以防止丢失匹配和重复插入。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

out = []

for ch in s:
    out.append(ch)
    if len(out) >= 3 and out[-3] == 'c' and out[-2] == 'j' and out[-1] == 'b':
        out.append(',')

print(''.join(out))
```该解决方案依赖于使用列表的增量构造，这避免了重复的字符串复制。 模式检查仅对最后三个字符进行，从而使其每次迭代的时间恒定。 

一个微妙的实现细节是使用列表而不是字符串累加器。 使用`ans += ch`由于重复分配，重复会显着降低性能。 另一个重要的细节是在索引之前检查长度`out[-3]`，这可以避免短前缀的运行时错误。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    s = sys.stdin.readline().strip()
    out = []
    for ch in s:
        out.append(ch)
        if len(out) >= 3 and out[-3] == 'c' and out[-2] == 'j' and out[-1] == 'b':
            out.append(',')

 ## 边缘情况

 用于输入`"cjb"`，算法附加`c`,`j`,`b`依次。 在第三个字符处，缓冲区以`"cjb"`，因此立即添加一个逗号，产生`"cjb,"`。 不再存在任何字符，因此该过程干净地结束。 

用于输入`"ccjb"`，缓冲区演变为`"c" → "cc" → "ccj" → "ccjb"`。 仅在最后一步才匹配后缀，产生`"ccjb,"`。 这证实了该算法不需要固定位置的模式对齐，只需要后缀匹配。 

用于输入`"cjcjb"`，该算法确保了重叠结构的正确处理。 加工后`"cjc"`，没有匹配发生。 什么时候`"cjb"`形成于末尾，只有最后出现才会触发逗号。 这表明较早的部分前缀不会干扰后面的有效匹配。
