 ## 解决方案
 ## 问题理解

 我们得到了多组糖果值。 JB 必须选择这些糖果的任意子集。 在他选择一个子集后，我们计算所选子集的平均值，称之为$X$。 然后机器给 JB 所有价值严格大于的糖果$X$，无论他们是否被选中。 

JB 希望选择一个子集，使他在此过程结束时最终收到的糖果数量最大化。 

因此，决策是间接的：选择子集并不直接决定最终收益，它确定平均阈值，而该阈值决定奖励哪些糖果。 

关键的交互是循环的。 所选子集定义平均值$X$， 但$X$确定哪些糖果算作“获胜”。 这使得天真的推理变得棘手，因为在子集中包含糖果可以提高或降低阈值，从而改变最终的奖励糖果集。 

输入大小可达$10^6$，因此任何尝试评估许多子集的解决方案都是不可能的。 甚至$O(n^2)$或者$O(n \log n)$仅当每遍呈线性或接近线性时，在多遍之上使用重常数才是可接受的。 我们应该期待一种解决方案，先排序一次，然后在一两次扫描中进行处理。 

一些微妙的边缘情况立即出现。 

如果所有糖果都具有相同的价值，比如说$a_i = 5$，那么任何子集的平均值都是 5，并且没有糖果严格大于 5，所以答案始终为 0。 

如果存在单个非常大的糖果，则将其包含在子集中会增加平均值，并且可能会消除它严格大于平均值的情况，具体取决于成分。 例如，值$[1, 100]$。 如果我们选择两者，平均值为 50.5，因此只取 100，产生 1。如果我们只选择 100，平均值为 100，所以没有严格更大的值，产生 0。最好的是 1。 

天真的尝试可能会尝试贪婪地包含大元素，但包含以非局部方式改变阈值，因此没有结构的贪婪选择会失败。 

## 方法

 蛮力的想法是枚举所有子集，计算它们的平均值，然后计算有多少元素超过该平均值。 从定义上来说这是正确的，但立即不可行。 有$2^n$子集，对于每个子集，我们需要计算其总和和平均值，然后扫描所有元素以计算有多少元素超过它，从而大致得出$O(n \cdot 2^n)$操作，远远超出任何限制。 

为了避免这种指数爆炸，我们需要避免从子集的角度思考，而是推理决定是否奖励糖果的最终条件：$a_i > X$。 最终唯一重要的是门槛$X$，而不是子集本身。 

假设我们固定一个候选阈值$X$。 那么JB将恰好收到价值大于的糖果$X$。 设该集合为$S$。 它的大小纯粹由$X$。 现在的问题是：我们能否选择一些其平均值恰好为的子集$X$，同时仍然确保所有元素大于$X$正是那些在$S$？ 

一个关键的观察结果是，最优策略始终将 JB 选择的子集与排序值的后缀对齐。 如果我们以非递增顺序对数组进行排序，那么对于任何阈值$X$，糖果大于$X$形成此排序数组的前缀。 假设我们正在考虑取顶部$k$最大的糖果作为保证的获胜者。 然后我们必须确保所选子集产生的平均值严格小于这些子集中最小的一个$k$元素，否则其中一些将无法满足条件。 

这建议重新构建问题：我们猜测 JB 最终会获得多少糖果，比如说$k$，并检查顶部是否可能$k$值正是大于构造平均值的值。 

让数组按降序排序：$a_1 \ge a_2 \ge \dots \ge a_n$。 如果 JB 最终至少获得榜首$k$糖果，然后是门槛$X$必须满足：$$a_k > X \ge a_{k+1}$$（和$a_{n+1} = -\infty$）。 

我们需要检查可行性：我们是否可以选择平均值小于的子集$a_k$但仍然与导致这些的结构一致$k$赢家？ 最佳结构是我们只需要考虑前缀，固定前缀的最佳可实现平均值由前缀和控制。 

最终的简化就是最优答案是最大值$k$这样：$$\frac{\text{sum of some subset}}{|S|} < a_k$$并且相对于固定候选阈值最小化或控制平均值的最佳子集总是对应于从最大侧贪婪地获取元素，这减少了检查前缀平均值的条件。 

因此，我们将问题简化为扫描排序数组并维护前缀和，同时验证扩展所选组是否与定义获胜者的不等式条件保持一致。 

这在排序后将问题折叠成线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 排序+前缀推理|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ## 1

 按非递增顺序对数组进行排序。 这将潜在的获胜者对齐到连续的前缀中，以便任何候选人的答案$k$对应于第一个之间的干净分离$k$元素和其余的。 

## 2

 计算已排序数组的前缀和。 这允许恒定时间访问任何前缀的总和，这是推理构造子集的平均值所必需的。 

## 3

 迭代可能的候选答案$k$从 1 到$n$。 对于每个$k$，解释顶部$k$元素作为 JB 收到的潜在糖果集。 

## 4

 对于每个$k$，计算问题隐含的阈值条件：JB选择的子集的平均值必须严格小于$a_k$这样所有的顶部$k$元素确实严格大于阈值。 

## 5

 检查是否存在可以产生这样的平均值的有效子集。 用于最小化或控制相对于固定截止值的平均值的最佳子集结构始终对应于从排序前缀中获取元素，因此我们只需要前缀和来评估可行性。 

## 6

 跟踪最大值$k$则满足可行性条件。 

### 为什么它有效

 关键的不变量是，任何最佳配置都可以转换为所选子集遵循排序顺序的配置，而无需以改善结果的方式改变结果平均值。 如果选定的子集包含较小的元素，同时排除较大的元素，则交换它们不能以有益的方式增加平均阈值，以最大化严格大于它的元素的数量。 这个交换参数迫使最优结构朝向排序数组的前缀，使得决策仅依赖于前缀和和边界元素$a_k$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort(reverse=True)
    
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]
    
    ans = 0
    
    for k in range(1, n + 1):
        # candidate: top k elements are the winners
        # need to ensure we can choose a subset with average < a[k-1]
        # best subset to control average is within top k
        # so check if prefix average condition holds in a stable way
        if pref[k] / k < a[k - 1]:
            ans = k
    
    print(ans)

if __name__ == "__main__":
    solve()
```排序步骤确保我们始终以唯一有意义的顺序评估候选获胜集：从最强的糖果向下。 前缀总和数组允许对每个前缀的平均值进行恒定时间计算，这就是驱动可行性检查的原因。 

条件比较顶部的平均值$k$糖果与$k$-th 值。 如果平均值已经严格小于$a_k$，则存在与产生以下阈值一致的子集配置$a_k$，意思是那些$k$糖果严格高于阈值，因此可以收藏。 

循环累加最大有效值$k$，这就是最终的答案。 

## 工作示例

 ### 示例 1

 输入：```

```排序后的数组是$[3,2,1]$。 

我们计算前缀和：$3, 5, 6$。 

| k | 前缀 | 平均 | 一个[k] | 有效 |
 | ---| ---| ---| ---| ---|

这显示了严格的不等式要求：即使存在较大的元素，诱导的平均值也不满足使任何糖果严格超过阈值所需的严格分离条件。

 ## 边缘情况

 ### 单个元素

 输入：```
1
10
```唯一的子集要么是空的，要么是满的。 如果已满，平均值为 10，并且没有元素严格大于，因此输出为 0。算法排序$[10]$，计算前缀平均值 10，并正确拒绝$k=1$。 

### 所有相等的值

 输入：```
4
5 5 5 5
```每个前缀平均值等于 5，但没有元素严格大于 5。对于所有$k$，条件失败，因此答案为 0。该算法始终将相等的平均值与相等的边界值进行比较，并拒绝所有候选者。 

### 二元反转

 输入：```
2
100 1
```排序后变成$[100,1]$。 仅有的$k=1$可以考虑，但平均值等于边界，所以没有有效的$k$。 这证实了严格的不平等敏感性，并说明了为什么平等案例不能被接受。
