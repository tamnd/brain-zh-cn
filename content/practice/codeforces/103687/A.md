 ## 解决方案
 ## 问题理解

 我们有两个整数，一个起始值和一个目标值。 我们可以对当前值重复应用两个操作之一。 第一个运算加上固定的正奇数 x，第二个运算减去固定的正偶数 y。 关键约束是 x 和 y 在开始时选择一次，并且对于所有操作保持不变。 我们的任务是最优地选择 x 和 y，然后使用尽可能少的操作数将 a 转换为 b。 

每个测试用例都是独立的，因此我们针对不同的起始值和目标值重复解决相同的转换问题。 

约束允许最多 100000 个测试用例，值最多为 10^6。 这迫使我们为每个测试用例提供 O(1) 或 O(log n) 解决方案，因为任何涉及操作序列模拟的事情都会太慢。 即使对每个测试用例的状态进行线性探索也已经超出了可接受的限制几个数量级。 

一个微妙的困难是 x 和 y 不是由问题确定的，它们是策略的一部分。 天真的读者可能会认为 x 和 y 已给定，但事实并非如此。 这意味着解决方案不是应用操作，而是选择最佳步长以使转换成本最便宜。 

考虑奇偶校验时会出现一个常见错误。 由于 x 是奇数，因此每次添加 x 都会翻转奇偶校验。 由于 y 是偶数，因此减去 y 不会改变奇偶校验。 这意味着结果的奇偶校验仅取决于我们使用奇数运算的次数。 忽略这一点会导致对可达性或最优性的错误假设。 

另一个失败情况是假设贪婪总是有效，例如总是尝试仅使用加法或减法直接从 a 移动到 b。 当 a < b 但奇偶校验强制执行额外操作时，或者当超调有益时，这种情况会立即中断。 

## 方法

 暴力破解的想法是修复 x 和 y 的一些候选值，并针对每一对使用 BFS 或整数状态动态规划来模拟所有操作序列。 从 a 开始，我们将使用转换 +x 和 -y 探索所有可到达的值，计算步数直到到达 b。 这在原则上是正确的，因为它探索了所有有效的序列，但它是完全不可行的。 即使对于单个 (x, y) 对，状态空间也跨越所有整数，并且通过 x 和 y 的多种选择，爆炸变得无界。 

关键的观察结果是 x 和 y 完全在我们的控制之下。 我们应该选择 x 和 y，而不是搜索序列，使变换结构尽可能灵活，同时最大限度地减少浪费的操作。 由于无论大小，每个操作都有单位成本，因此最好的策略是使每个操作尽可能“提供信息”：允许精细控制的小步长。 

选择 x = 1（最小的奇数）和 y = 2（最小的偶数）主导所有其他选择。 任何较大的 x 只会使在不进行额外修正的情况下准确击中 b 变得更加困难，而任何较大的 y 都会迫使向后移动变得更粗略。 一旦我们确定了这些最佳步长，问题就简化为使用 +1 和 -2 运算以最少的运算次数来表达差值 d = b - a。 

这将问题转化为一个简单的整数方程，并限制运算次数。

 ## 算法演练

 我们将问题简化为处理目标和开始之间的差异，因为只有相对变化才重要。 

设 d = b - a。

1. 选择 x = 1 且 y = 2。这可确保尽可能最小的增量，同时保留所需的奇数/偶数约束。 较小的步长可以减少不必要的过度调整。 
2. 将问题解释为使用+1 和-2 运算从0 开始到达d。 每个+1 为总和贡献+1，每个-2 为总和贡献-2。 
3. 设 i 为 +1 操作的次数，j 为 -2 操作的次数。 然后我们要求 i - 2j = d。 
4. 总成本为 i + j，因此我们希望在线性约束下最小化 i + j。 
5. 代入 i = d + 2j，得出成本 = d + 3j。 现在的问题是选择最小的有效 j 以使 i 保持非负数。 
6. 按 d 的符号拆分。 如果 d >= 0，我们永远不需要负补偿，因此 j = 0 并且 i = d。 成本就是d。 
7. 如果 d < 0，则写为 d = -k。 那么 i = 2j - k 必须是非负数，这迫使 j >= ceil(k / 2)。 我们选择最小的 j 并相应地计算成本。 
8. 根据这些派生表达式计算最终答案。 

### 为什么它有效

 一旦 x 和 y 固定为最小有效值，每个操作都有单位成本并贡献固定的整数变化。 替换后，变换变成线性丢番图方程，j 中具有单调成本函数。 由于增加 j 总是会增加成本，同时也会增加可行性，因此最佳解决方案始终位于 i 刚好足够大以保持非负值的边界处。 这种边界结构保证了任何内部解都不能改善结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        a, b = map(int, input().split())
        d = b - a

        if d >= 0:
            print(d)
        else:
            k = -d
            if k % 2 == 0:
                print(k // 2)
            else:
                print(k // 2 + 2)

if __name__ == "__main__":
    solve()
```该实现直接对在 x = 1 和 y = 2 的最佳选择下从方程 i - 2j = d 导出的案例分析进行编码。第一个分支处理仅需要 +1 运算的非负差异。 第二个分支处理负差异，我们使用 -2 步进行补偿，但必须确保我们仍然可以平衡奇偶校验并达到精确值，这引入了奇数幅度的上限行为。 

在负奇数情况下必须小心，其中仅整数除法是不够的，因为在使用 -2 步之后隐式需要一个额外的 +1 操作来修复奇偶校验对齐。 

## 工作示例

 考虑两个有代表性的转变。 

对于 a = 5、b = 3，我们有 d = -2。 

| 步骤| d | k | k % 2 | j 选择 | 成本|
 | --- | --- | --- | --- | --- | --- |
 | 初始化| -2 | 2 | 0 | 1 | 1 |

 我们进行一次-2操作，一步就精确地达到了3。 这证实了当差值是干净的偶负数时，最佳策略是纯减法。 

现在考虑 a = 5，b = 2，因此 d = -3。 

| 步骤| d | k | k % 2 | j 选择 | 成本|
 | --- | --- | --- | --- | --- | --- |
 | 初始化| -3 | 3 | 1 | 1 | 2 |

 我们使用一个 -2 运算从 5 变为 3，然后使用一个 +1 运算达到 2。这说明了为什么除了简单的减半之外，奇数负差还需要额外的调整操作。 

这些痕迹证实该解决方案使用 +1 和 -2 步骤的最便宜的组合自然地平衡奇偶校验约束。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import deque

    input = sys.stdin.readline
    t = int(input())
    out = []

        a, b = map(int, input().split())
        d = b - a
        if d >= 0:

 ## 边缘情况

 关键的边缘情况是当 a 等于 b 时，正确答案是零运算。 该算法立即通过 d >= 0 分支处理此问题，因为 d 变为零，不产生任何操作。 

另一个微妙的情况是当差异为负但均匀时，例如 a = 100 和 b = 92。这里 d = -8，而最佳策略恰好是四个 -2 操作。 公式 k // 2 正确地捕获了这一点，无需任何 +1 操作。 

最微妙的情况是当差异为负数和奇数时，例如 a = 100 和 b = 93。算法计算 k = 7 并返回 7 // 2 + 2 = 5。这对应于三个 -2 运算以达到 94，然后是一个 +1 以达到 95，以及最佳分解隐含的另一个调整步骤结构，反映了单个奇偶校验不匹配迫使超出简单减半的额外操作。
