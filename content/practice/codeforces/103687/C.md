 ## 解决方案
 ## 问题理解

 在类似股票市场的系统中，我们有两组参与者。 一组包含想要购买股票的人，每个人都指定了他们愿意支付的最高价格。 另一组包含想要出售股票的人，每个人都指定了他们愿意接受的最低价格。 

单一最终价格$x$是系统固定的。 如果买家的报价至少为$x$，因为他们愿意支付那么多或更多。 如果卖家的报价最高为$x$，因为他们愿意接受该价格或更低的价格。 任务是计算两个组中总共有多少人满足这些条件。 

输入大小可达$10^5$买家和$10^5$卖家。 任何检查每个元素一次的解决方案都是可以接受的，但任何涉及排序或嵌套扫描的解决方案仍然可以。 然而，任何二次或涉及每个元素重复完整扫描的事情都会太慢，因为它会达到$10^{10}$最坏情况下的操作。 

此类问题中的一个常见错误是误解了不平等的方向。 买家符合条件时$a_i \ge x$， 不是$a_i \le x$。 卖家符合条件时$b_i \le x$， 不是$b_i \ge x$。 交换这些条件会导致计数完全反转。 

当所有值都等于时，会出现另一种微妙的边缘情况$x$。 在这种情况下，两组中的每个参与者都应该被计算在内。 任何偏离一阈值的错误都会立即失败。 

## 方法

 蛮力方法很简单。 我们扫描所有买家并检查每个价值是否至少$x$。 然后我们扫描所有卖家并检查每个值是否最多$x$。 每当满足条件时，我们就会增加计数器。 这是有效的，因为每个决定都是独立的，最终的答案就是满意的参与者总数。 

这种方法在渐近结构方面已经是最优的，因为该问题不需要排序或配对。 它可能被认为是“暴力”的唯一原因是它明确地一一检查每个元素。 

操作总数为$n + m$，这最多是$2 \cdot 10^5$。 即使在 Python 中，这也很容易受到限制。 任何排序或预处理的尝试都是不必要的开销，并且不会提高正确性或性能。 

关键的观察结果是，最终价格充当固定阈值，将参与者分为两个独立的过滤器。 买家按下限过滤，卖家按上限过滤。 各组之间没有交互，因此问题分解为两个简单的计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n + m) | O(1) | O(1) | 已接受 |
 | 最佳 | O(n + m) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取整数$n, m, x$。 这些定义了两组的规模和阈值价格。 
2. 将计数器初始化为零。 这将累积所有有效参与者。 
3. 阅读买家价格清单。 对于每个值$a_i$，检查是否$a_i \ge x$。 如果是这样，则增加计数器。 这反映出买家愿意以系统价格或更高价格进行交易。 
4.阅读卖家价格表。 对于每个值$b_i$，检查是否$b_i \le x$。 如果是这样，则增加计数器。 这反映出卖家愿意接受系统价格或更低。 
5. 输出最终计数器。 

单独循环背后的原因是每个组都有不同的不平等方向，混合它们会引入不必要的复杂性并带来不正确比较的风险。 

### 为什么它有效

 每个参与者独立地满足一个仅取决于他们自己的价格和固定阈值的条件$x$。 一个参与者的决定永远不会影响另一个参与者。 由于这种独立性，单独统计有效买家和卖家并将结果相加就相当于评估整个系统。 该算法本质上是计算在输入数组上定义的两个不相交条件集的并集的大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m, x = map(int, input().split())

ans = 0

a = list(map(int, input().split()))
for v in a:
    if v >= x:
        ans += 1

b = list(map(int, input().split()))
for v in b:
    if v <= x:
        ans += 1

print(ans)
```代码直接反映了算法。 我们首先读取参数，然后独立处理买家和卖家。 柜台`ans`仅当值满足其特定于组的条件时才会更新。 

一个常见的实施错误是混合严格不等式和非严格不等式。 在这里，两种情况都包括平等：买家接受$a_i = x$, 卖家接受$b_i = x$。 这是有意为之，并且与声明相符。

 ## 边缘情况

 临界边缘情况是所有值都等于阈值时$x$。 例如：```
3 2 5
5 5 5
5 5
```该算法独立处理每个买家和卖家。 每个比较都评估为真，因为两者$a_i \ge x$和$b_i \le x$相等时保持。 计数器每个元素增加一次，总共产生 5。任何使用严格不等式的实现都会错误地返回 0。 

另一种极端情况是没有人符合资格：```
3 3 10
1 2 3
11 12 13
```在这里，所有买家都失败了$a_i \ge x$所有卖家都失败了$b_i \le x$。 计数器始终保持为零，确认算法可以正确处理空选择集，而无需特殊的大小写。
