 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，然后我们重复地对其应用非常具体的转换。 在一个转换步骤中，每个位置都会同时更新，以便每个元素成为数组中所有其他元素（不包括其自身）的总和。 

准确执行此转换后$k$有时，我们被要求计算最终数组值的分布情况，以任意两个元素之间的最大绝对差来衡量。 

重新解释该操作的一个关键方法是关注数组的总和。 令当前总和为$S$。 一次操作后，每个元素$a_i$变成$S - a_i$。 这意味着每个值都以非常结构化的方式反映在当前总和周围，这表明数组的演变是高度规则的而不是混乱的。 

这些限制使得这一观察变得至关重要。 操作次数$k$可以非常大，最多$10^9$，因此逐步模拟该过程是不可能的。 当对所有测试用例进行求和时，即使每次操作一次通过也已经超出了时间限制。 这立即排除了任何依赖于重复显式应用转换的方法。 

另一个重要的限制是所有测试用例的元素总数最多可达$2 \cdot 10^5$。 这表明我们的目标应该是$O(n)$或者$O(n \log n)$每个测试用例的解决方案，理想情况下在预处理后每个测试进行恒定时间处理。 

出现微妙的边缘情况时$n = 1$。 在这种情况下，该操作将用所有其他元素的总和（为零）替换唯一的元素。 但由于不存在“其他元素”，因此总和为零，并且永远保持为零。 这意味着答案始终为零，无论$k$。 任何解决方案都必须显式处理这种情况，以避免不必要的计算或不正确的转换。 

第二个重要的边缘情况是所有元素都相等。 经过一次操作后，每个元素变成$(n-1)x$，因此数组保持均匀。 最大值和最小值之间的差异保持为零，并且对于所有进一步的操作仍然如此。 

有趣的行为只有在值不同时才会出现，关键的挑战是理解元素之间的差异在重复转换下如何演变。 

## 方法

 暴力模拟直接应用变换$k$次。 每个操作都会重新计算总和并重建数组$O(n)$，导致总复杂度为$O(nk)$。 和$k$最多$10^9$，这是完全不可行的。 

关键的见解是跟踪各个元素如何相互关联，而不是它们的绝对值。 如果我们扩展一个操作，每个元素就变成$S - a_i$，这相当于$(n-1)\cdot \text{mean of array} - (a_i - \text{mean})$直至线性变换。 更重要的是，对最终答案重要的不是绝对值，而是两个位置之间的差异。 

让我们考虑两个要素$a_i$和$a_j$。 经过一次操作后，它们的区别就变成了：$$(S - a_i) - (S - a_j) = a_j - a_i$$因此，差异翻转了符号，但保留了大小。 

两次运算后，差值再次翻转并返回到原来的符号。 这意味着成对差异的结构是周期性的，周期为 2。整个数组在两种状态之间交替：原始数组及其转换版本，其中每个值都是$S - a_i$。 

因此，问题简化为评估最多两个状态，具体取决于是否$k$是偶数还是奇数。 

我们计算：

 - 如果$k$为偶数时，数组结构与原来等效。 
- 如果$k$是奇数，每个元素变成$S - a_i$， 在哪里$S$是初始总和。 

从这里开始，计算最大差值就很简单了：我们只需取相应状态下 max 和 min 之间的差值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(nk)$|$O(n)$| 太慢了|
 | 宇称观测（2态系统）|$O(n)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 1.读取数组并计算其总和$S$，以及它的最小值和最大值。 这三个值完全决定了系统的两种可能状态，因为变换仅取决于$S$和个别元素。 
2.如果$n = 1$，立即输出0。对于一个元素，变换总是产生0，并且没有变化。 
3.如果$k$是偶数，我们知道阵列有效地保持其原始配置。 最大的差异很简单$\max(a) - \min(a)$，因为结构变化不会影响相对排序。 
4. 如果$k$是奇数，每个元素变成$S - a_i$。 在这个变换后的数组中，最大元素对应于$S - \min(a)$，最小值对应于$S - \max(a)$。 发生这种反转是因为减法颠倒了顺序。 
5. 计算变换后状态的差异：$$(S - \min(a)) - (S - \max(a)) = \max(a) - \min(a)$$因此差异在数值上保持不变。 
6. 输出计算出的差值。 

它起作用的根源在于每个操作都对数组值应用仿射变换。 虽然值本身发生显着变化，但所有成对差异仅交替符号且从不改变大小。 由于该问题仅要求最大绝对差，因此系统崩溃为两个交替状态，使得进一步的迭代变得无关紧要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        arr = list(map(int, input().split()))

        if n == 1:
            print(0)
            continue

        total = sum(arr)
        mn = min(arr)
        mx = max(arr)

        if k % 2 == 0:
            print(mx - mn)
        else:
            # transformed array would be S - a[i]
            # min becomes S - mx, max becomes S - mn
            print((total - mn) - (total - mx))

if __name__ == "__main__":
    solve()
```实现遵循我们从不显式构造中间数组的观察。 我们只跟踪汇总统计数据。 唯一微妙的一点是处理奇偶校验$k$，因为它决定了我们是评估原始数组还是其转换后的版本。 

表达式`(total - mn) - (total - mx)`仔细排序以反映正确的变换后的最大值减去变换后的最小值。 用代数展开它表明它简化回`mx - mn`，但保持这种形式可以使逻辑明确，并避免在将想法适应问题的变体时出现错误。

 ## 边缘情况

 对于$n = 1$，算法立即输出零，因为没有一对元素可以形成差异。 该变换退化为常数零序列，因此不需要进一步推理。 

对于所有元素都相等的数组，每次转换都会生成另一个按比例缩放的统一数组$n-1$，保持零价差。 计算出的最大最小值保持为零，匹配偶数和奇数的正确答案$k$。 

对于大型$k$，该解决方案从不模拟操作。 相反，它完全依赖于奇偶校验，因此诸如$10^9$不影响运行时间或正确性。
