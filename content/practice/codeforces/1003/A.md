[CF 1003A - 波利卡普的口袋](https://codeforces.com/problemset/problem/1003/A)

 **求解时间：** 1m 15s

 ## 解决方案
 ## 问题理解

 我们被赋予了多组硬币值，我们需要将每枚硬币放入一个“口袋”集合中，但有一个简单的限制：在任何单个口袋内，不允许任何值重复。 除了价值之外，硬币是无法区分的，因此任务纯粹是关于每个价值存在多少个副本以及如何将它们跨组分离。 

每个口袋的行为就像一组值，而不是多重集。 如果一个值在输入中多次出现，则这些副本必须分布在不同的口袋中。 目标是尽量减少容纳所有硬币所需的口袋数量。 

约束范围非常小，最多 100 个硬币，并且值也以 100 为界。这立即告诉我们，从二次时间向下的任何解决方案都很容易足够，甚至直接计数方法也是安全的，无需考虑性能。 

关键边缘行为来自重复项。 如果所有值都不同，那么一个口袋就足够了。 如果所有值都相同，则每个硬币都会强制有一个单独的口袋，因为没有口袋可以包含两个相同的值。 一个微妙的情况是，多个值以不同的频率重复，因为限制因素不是硬币的总数，而是最常见的单个值。 

例如，如果我们输入类似`1 1 2 2 3`，值`1`出现两次并且`2`也出现两次，迫使至少有两个口袋，即使总尺寸大于两个。 任何仅考虑总计数或唯一计数的幼稚尝试都会在这里失败。 

## 方法

 强力解释将尝试实际模拟打包过程。 人们可以通过贪婪地放置硬币来重复构建口袋，同时确保口袋内没有重复的硬币，取出用过的硬币，并重复直到放置完所有硬币。 这是正确的，因为每一步都会产生有效的分组，并且最终所有硬币都被分配。 

问题是效率。 每次我们建造一个口袋时，我们都会扫描剩余的硬币并尝试容纳尽可能多的不同值。 在最坏的情况下，例如当所有硬币都相同或存在许多重复项时，我们会重复处理几乎完整的列表。 这会导致大约 O(n²) 或更糟糕的行为，具体取决于实现细节，尽管 n 在这里很小，但这种方法在概念上是不必要的。 

关键的观察是，频率为 f 的每个不同值恰好贡献了 f 个“要求”，必须将其放入不同的口袋中。 每个口袋最多可以容纳每个值出现一次，因此如果一个值出现f次，我们至少需要f个不同的口袋来放置其所有副本。 这意味着答案必须至少是任何值的最大频率。 

这个界限也是足够的。 如果我们创建与最大频率一样多的口袋，我们可以以简单的循环方式将每个值的出现分布在不同的口袋中。 由于没有值出现的次数多于口袋的数量，因此每次出现都可以无冲突地放置。 

这将问题简化为频率计数任务。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力打包 | O(n²) | O(n) | 已接受但没有必要 |
 | 频率计数 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算每个硬币值在数组中出现的次数。 这以紧凑的形式捕获了所有约束，因为仅重复冲突的问题。 
2. 跟踪所有值中最大的频率。 该值表示所需的最小口袋数，以便即使是最频繁的值也可以跨不同的口袋分开。 
3. 输出这个最大频率作为答案。 所有其他值都会自动放入这些口袋中，因为它们中没有一个比所选容量出现的频率更高。 

### 为什么它有效

 每个口袋最多可容纳一枚给定价值的硬币。 因此，如果某个值出现 f 次，我们至少需要 f 个不同的块来放置所有出现的值而不重复。 相反，如果我们有 k 个口袋，其中 k 等于最大频率，我们可以将每个值的第 i 次出现分配给口袋 i。 这保证了不会有两个相同的值在同一个口袋中发生冲突，并且每个硬币都被分配一次。 

该算法是正确的，因为它将打包约束转换为每个值的容量约束，并在所有值上使用最紧密的可能界限。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

freq = [0] * 101

for x in a:
    freq[x] += 1

ans = max(freq)
print(ans)
```该解决方案完全依赖于频率计数。 数组`freq`大小为 101，因为值保证在 1 到 100 之间。每个硬币都会增加其相应的桶，最终的答案就是最大的桶值。 

无需排序或模拟。 该逻辑取决于这样一个事实：除了共享口袋之外，约束在各个值之间是独立的，唯一的共享限制是相同的值必须分开。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(input())
    a = list(map(int, input().split()))
    freq = [0] * 101
    for x in a:
        freq[x] += 1
    return str(max(freq))

# provided sample
assert run("6\n1 2 4 3 3 2\n") == "2"

# all distinct
assert run("4\n1 2 3 4\n") == "1"

# all same
assert run("5\n7 7 7 7 7\n") == "5"

# mixed frequencies
assert run("7\n1 1 2 2 2 3 4\n") == "3"

# single element
assert run("1\n10\n") == "1"

# two high-frequency values
assert run("6\n5 5 6 6 6 5\n") == "3"
```| 测试输入| 预期产出 | 它验证了什么 |
 | ---| ---| ---|
 | 1 2 3 4 | 1 2 3 4 1 | 所有不同的值 |
 | 7 7 7 7 7 | 7 7 7 7 7 5 | 最大重复情况|
 | 1 1 2 2 2 3 4 | 1 1 2 2 2 3 4 3 | 混频结构|
 | 10 | 10 1 | 最小尺寸输入|

 ## 边缘情况

 当所有硬币都有唯一的值时，每个频率都是1，所以最大值是1，只需要一个口袋。 该算法将所有频率初始化为零并每次递增一次，将最大值保留为 1，从而正确生成单个口袋。 

当所有硬币都相同时，每个增量都会进入同一个桶。 对于像这样的输入`7 7 7 7`，7 的出现频率变为 4，算法输出 4。这与由于限制而导致每个相同硬币必须占据单独口袋的事实相匹配。 

当多个值共享相同的最大频率时，例如`1 1 2 2`，两个值都达到频率 2。最大值仍为 2，两个口袋就足够了。 每次出现 1 和 2 都可以在两个口袋中分开而不会发生冲突，从而确认平局不会将结果更改为超出最大值本身。
