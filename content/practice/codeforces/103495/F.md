 ## 解决方案
 ## 问题理解

 我们有一棵树，其中每个节点都带有一个数值。 从任何起始节点开始，猴子都可以沿着边缘行走而无需重新访问节点，因此每个有效的行走都对应于树中的一条简单路径。 沿着这样的路径，我们忽略最终度量的遍历顺序，而是按照节点值出现在路径上的顺序查看节点值的序列。 

对于每个起始节点$i$，我们考虑所有可能的简单路径$i$。 对于每个这样的路径，我们获取沿路径的值并计算该值序列的最长递增子序列 (LIS) 的长度。 在所有起始路径中$i$，我们想要最大可能的 LIS 长度。 

因此，任务不是选择单个路径，而是对路径的选择及其内部的子序列选择进行优化。 每个节点的输出都是一个数字：您可以在从该节点开始的所有简单路径中获得的最佳 LIS。 

这些限制意味着我们必须处理最多$2 \cdot 10^5$所有测试用例的节点总数。 二次或偶数$O(n \log^2 n)$每个测试用例方法将无法生存。 树结构强烈表明，在最终解决方案中，每条边只应处理恒定或对数次数，并且需要跨节点全局重用计算。 

一个关键的微妙之处在于 LIS 是沿着路径序列计算的，而不是通过树结构本身计算的。 这意味着我们正在优化由路径引起的排列，而不是祖先关系。 这里的一个天真的混淆是将其视为“有根树 DP 上的 LIS”，这并不直接有效，因为路径可以向上和向下移动。 

天真的思维的一个失败案例是一棵星形树。 假设中心的值为 100，叶子的值递增。 从一片叶子开始，人们可能认为答案只是“更大邻居的数量”，但最佳路径可能是叶子→中心→另一片叶子，其中中心根据方向打破单调性约束，不同的子序列完全跳过它。 这表明路径结构和子序列选择以一种不平凡的方式相互作用。 

## 方法

 暴力解决方案将枚举从每个节点开始的每条简单路径，生成其值序列，并使用标准耐心排序方法计算 LIS。 由于有$O(n)$起始节点，每个节点有$O(n)$树中可能的路径，在最坏的情况下，这很快就会退化为指数数量的路径。 即使我们将自己限制在简单的路径上，单个节点也可以参与$O(n)$路径，导致不可行的$O(n^2)$或更差的要评估的序列数。 每个 LIS 计算成本$O(n \log n)$，使得总数显然不可能。 

关键的观察是我们实际上不需要明确地考虑所有路径。 重要的是，树中的任何简单路径都是通过选择起始节点然后向外行走来唯一确定的。 从一个固定的起点开始，任何最优解都对应于在每个分支点选择一个方向并形成一条路径。 沿着这样的路径，我们只对不断增加的值子序列感兴趣，这意味着我们可以自由地跳过不会改善子序列的节点。 

这将问题转化为更接近于“在由值约束引起的隐式有向结构中沿着任何根到节点行走找到最长的递增序列”。 处理这个问题的经典方法是按节点值的递增顺序处理节点，并使用类似于树上的 LIS 的动态编程思想：当我们处理节点时，我们维护以该节点结束的最佳可实现 LIS，并通过邻接传播改进，同时尊重单调性。 

然而，因为我们可以从任何节点开始，所以我们必须将每个节点视为潜在的起点，这与通常的 LIS DP 方向相反。 正确的重新解释是，对于每个节点，我们希望从该节点开始，在树中可以到达最长的严格递增的值链，其中移动不受限制，但必须保持在简单路径上。 这可以通过按值对节点进行排序并对由从较小值到较大值的边引起的 DAG 执行 DP 来解决。 

树结构保证在这个有向版本中没有循环，并且每条边都在一致的方向上松弛。 节点的最终答案是从该节点开始的最佳链长度，可以通过反转转换或按升序计算 DP 并存储最佳可达扩展来计算。

 | 树上的值排序DP | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题重新解释为在树中找到最长的严格递增链，其中允许沿边缘进行转换，但只能从较小的值节点到较大的值节点。 

1. 按值升序对所有节点进行排序。 这确保了在处理节点时，已经考虑了递增序列中所有可能的前驱节点。 这种排序至关重要，因为它强制我们只能向前扩展序列的值。 
2.维护一个DP数组，其中$dp[u]$表示以节点结束的递增序列的最大长度$u$。 最初，每个节点都有$dp[u] = 1$，因为每个节点单独形成一个有效的序列。 
3. 按值递增的顺序遍历节点。 对于每个节点$u$，考虑所有邻居$v$这样$a[v] > a[u]$。 对于每个这样的边，我们尝试从$u$到$v$, 更新$dp[v] = \max(dp[v], dp[u] + 1)$。 此步骤是有效的，因为值顺序保证沿路径严格增加。 
4.处理完所有节点后，得到每个起始节点的答案$u$不简单$dp[u]$， 因为$dp$描述以节点结束的序列。 我们需要从节点开始的序列。 为了转换这一点，我们反转视角：我们不是向前传播，而是按降序计算第二个 DP，或者等效地，我们通过按相反值顺序处理节点并向较小的邻居放松来计算从每个节点开始的最佳链。 
5. 以相反的顺序，对于每个节点$u$，我们看看邻居$v$和$a[v] < a[u]$，并更新$ans[v] = \max(ans[v], ans[u] + 1)$。 这确保每个节点都从它开始积累最佳的可能链。 
6.最终数组$ans$包含每个节点所需的答案。 

其核心思想是，树中增加的路径可以被视为从较小值到较大值的有向边，并且可以使用两次值排序 DP 来计算此 DAG 中的最长路径，以处理链的两端。 

### 为什么它有效

 不变的是，当按排序顺序处理节点时，每次我们扩展一条链时，我们只从一个节点移动到一个严格更大的值节点，从而确保归纳图中的无环性。 每个有效的递增序列对应于该 DAG 中的一条路径，并且当其边缘按顺序松弛时，每条这样的路径都被恰好考虑一次。 由于 DP 始终记录每个节点从有效前驱节点可实现的最大链长度，因此不会错过任何最佳序列，也不会引入无效序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    adj = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        adj[u].append(v)
        adj[v].append(u)
    
    nodes = list(range(n))
    nodes.sort(key=lambda x: a[x])
    
    dp = [1] * n
    
    for u in nodes:
        for v in adj[u]:
            if a[v] > a[u]:
                if dp[v] < dp[u] + 1:
                    dp[v] = dp[u] + 1
    
    ans = [1] * n
    
    for u in reversed(nodes):
        for v in adj[u]:
            if a[v] < a[u]:
                if ans[v] < ans[u] + 1:
                    ans[v] = ans[u] + 1
    
    sys.stdout.write("\n".join(map(str, ans)) + "\n")

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```该解决方案为每个测试用例构建邻接列表，并按值排序处理节点。 第一个 DP 传递计算以节点结束的最佳递增链，而第二个反向传递将其转换为从节点开始的最佳链。 邻接检查确保我们仅根据通道沿着严格增加或减少的值边缘移动。 

微妙的一点是，我们永远不会以违反单调性的方式重新访问节点，因为排序强制执行全局顺序。 使用两次传递可以避免复杂的 LCA 或质心分解。 

## 工作示例

 ### 示例 1

 考虑一棵小树：

 节点：1-2-3 排成一行

 值：[1,3,2]

 我们按值顺序处理节点：1、3、2。 

| 步骤| 节点| dp 更新 | 状态 dp |
 | --- | --- | --- | --- |
 | 1 | 1 | 邻居 3 更大，dp[3]=2 | [1,2,1]|
 | 2 | 2 | 没有更大的邻居| [1,2,1]|
 | 3 | 3 | 没有更大的邻居| [1,2,1]|

 反向传球：

 | 步骤| 节点| 更新 | 状态 ans |
 | --- | --- | --- | --- |
 | 1 | 3 | 3→2 给出 ans[2]=2 | [1,1,2]|
 | 2 | 2 | 2→1 无效，2→3 有效 | [1,3,2]|
 | 3 | 1 | 1→2 有效 | [2,3,2]|

 这表明答案如何取决于起始位置而不是结束位置。 

### 示例 2

 树：以 1 为中心的星形

 值：[5,1,2,3]

 排序顺序：按值计算为 2,3,4,1。 

DP 捕捉到从一片叶子开始，只有当它改善了递增链时，我们才能穿过中心。 反向传递确保从中心开始产生尽可能长的扩展到更高价值的叶子。

 ## 边缘情况

 单节点情况很简单，因为不存在转换，因此答案必须为 1。该算法将 dp 和 ans 初始化为 1，因此两次传递都保持不变。 

严格递增路径测试前向传播是否正确地沿着一条线累积链长度而没有分支干扰。 由于每个节点按值顺序只有一个有效的前向邻居，因此 DP 干净地构建了一条链。 

严格递减路径测试反向 DP 通过。 由于没有边允许按值顺序向上传播，因此第一遍不执行任何操作，并且正确性完全取决于捕获有效起始链的反向传递。
