 ## 解决方案
 ## 问题理解

 我们给出了几个独立的测试用例，其中每个测试用例由两条等长的“字符”行组成。 每个字符都表示为一个短字符串，以编码其音调的数字结尾。 该数字决定该字符是属于平声类还是仄声类。 

任务是检查两条线是否满足纯粹基于这些声调类的非常严格的结构关系。 首先，第一行中的每个位置必须在第二行中具有补声类：平调必须与斜调匹配，斜调必须与平调匹配。 其次，还有一个额外的边界约束：第一行的最后一个字符必须是倾斜的，如果满足每个位置规则，这会立即强制第二行的最后一个字符保持水平。 

约束很小，最多 100 个测试用例，行长度最多 20 个。这立即告诉我们，任何解决方案，即使使用简单的嵌套循环，也很容易足够快，因为字符比较的总数在最坏的情况下也只有几千个。 

主要的边缘情况来自于对色调映射的误解或忘记这两个条件必须同时成立。 一个常见的错误是只检查位置反转而忘记最后一个字符约束，反之亦然。 

一个具体的失败例子是当所有位置都满足反转但最后一个字符违反规则时。 例如，如果第一行的最后一个音调是水平的，则即使所有其他位置都完美匹配，正确的输出也是“否”。 另一个微妙的情况是行长度不匹配，但该问题保证了相等的长度，因此不需要额外的处理。 

## 方法

 天真的方法是从字面上解释问题：对于每个测试用例，比较每个相应的字符对，确定它们的声调类别，并验证它们是否相反。 然后分别检查最后一个字符的情况。 这直接模拟了规则并且是正确的，因为我们没有丢失任何隐藏结构。 

这种方法的成本与处理的字符总数成正比。 在最多 100 个测试用例和每行 20 个字符的情况下，我们最多执行 2000 次比较，这是可以忽略不计的。 

除了简化分类步骤之外，没有任何有意义的优化。 关键的见解是整个问题简化为每个字符的恒定时间映射和每个测试用例的线性扫描。 一旦我们将音调映射到两个布尔类中，剩下的就是直接的一致性检查。

 ## 算法演练

1. 对于每个测试用例，读取整数n和两行token，因为每个测试用例都是独立的，必须单独验证。 
2. 将每个标记的最后一个字符转换为表示其音调类别的布尔值。 我们将数字 1 和数字 2 视为平声，数字 3 和数字 4 视为仄声。 此规范化步骤很重要，因为它将问题从字符串处理简化为简单的布尔比较。 
3. 迭代从 0 到 n − 1 的所有位置，并对每个位置比较第一行和第二行的音调类别。 该规则要求它们在每个位置都相反，因此布尔值相等对于所有对都必须失败。 
4.具体检查对于每个索引i，toneA[i]不等于toneB[i]。 如果任何索引违反了这一点，我们可以立即断定该对联无效。 
5. 单独验证第一行最后一个字符倾斜的边界条件。 如果不是，则无论所有其他匹配如何，该对联都是无效的。 
6. 如果每个位置条件和最后一个字符条件都成立，则输出 YES，否则输出 NO。 

### 为什么它有效

 该算法将每个字符简化为二进制状态（水平或倾斜），并强制第二行在每个索引处都是第一行的按位补码。 正确性源于以下事实：问题约束纯粹是局部的：除了明确检查的最终字符规则之外，每个位置都是独立的。 不存在全局依赖性，因此满足所有局部约束可以保证有效性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def tone_is_level(token: str) -> bool:
    # last character is digit 1-4
    d = token[-1]
    return d == '1' or d == '2'

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n = int(input().strip())
        a = input().split()
        b = input().split()

        ok = True

        # check inversion condition
        for i in range(n):
            if tone_is_level(a[i]) == tone_is_level(b[i]):
                ok = False
                break

        # check last character constraint
        if tone_is_level(a[-1]):
            ok = False

        out.append("YES" if ok else "NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现首先在比较中隐式地将每个标记转换为二进制分类。 功能`tone_is_level`隔离解析逻辑，确保我们永远不会将字符串解析与条件检查混合在一起。 

当发现不匹配时，循环会提前中断，这是可选的，但可以保持实现干净并避免不必要的检查。 最后一个字符规则在循环之后应用，因为它独立于每个位置的验证。 

一个微妙的细节是我们不需要显式检查第二行的最后一个字符。 如果第一行的最后一个字符是倾斜的并且所有位置都正确反转，则第二行的最后一个字符会自动变为水平，因此仅通过检查一侧即可完全强制执行约束。

 ## 边缘情况

 一种边缘情况是所有位置都满足反转但违反了最后一个字符规则。 该算法显式检查第一行的最后一个标记并立即拒绝，以确保正确性。 

另一个边缘情况是 n = 1。在这种情况下，两个条件都合并为一个检查：单个对的音调必须相反，第一个必须是倾斜的。 该算法自然地处理这个问题，因为循环运行一次并且无条件应用最终检查。 

最后一个微妙的情况是语气解析被误解。 通过仅提取最后一个字符并将其映射为布尔值，该算法避免了对拼写前缀的任何依赖，该前缀的长度可以任意变化。
