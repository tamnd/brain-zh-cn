 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$网格，其中每个单元格包含一个整数值。 有一个分两次运行的合并系统。 首先，在每一列中，共享相同可见值的垂直相邻单元格被合并到一个更高的块中。 然后，在该转换之后，在每一行中，现在具有相同可见值和相同当前高度的水平相邻块也会被合并。 

关键的复杂性是我们可以通过附加隐藏标签来修改单元格。 标签不会更改显示的值，但会更改合并的相等性。 显示相同数字的两个单元格仍将合并，除非它们的标签不同。 如果标签不同，则出于合并目的，它们将被视为不同的值。 

我们的目标是为单元格分配标签，以便在列合并和行合并操作之后，不会有两个不同的原始单元格最终合并在一起。 我们希望首先最小化全局使用的不同标签类型的数量，其次最小化所有最佳解决方案中标记单元的总数。 

重要的解释是标签就像颜色一样：通过分配不同的标签，具有相同可见值的单元格可以分为不同的“身份”，并且我们希望防止压缩后通过垂直相等或水平相等连接两个不同的原始单元格的任何合并路径。 

限制条件是$n, m \le 500$，所以最多有$2.5 \times 10^5$细胞。 周围有任何解决方案$O(nm \log nm)$或者$O(nm)$是预料之中的，而任何关于单元对的二次方都会太慢。 

当行或列中的所有值都相同时，就会出现微妙的边缘情况。 例如，如果网格是：```
1 1 1
1 1 1
```如果没有标签，所有内容都会折叠成一个合并的结构。 一种幼稚的方法可能认为仅破坏水平合并或仅破坏垂直合并就足够了，但由于合并发生在两次传递中并且取决于先前的压缩，因此本地修复仍然可以允许间接合并路径。 

另一个边缘情况是棋盘状网格：```
1 2
2 1
```即使没有两个相邻单元格直接匹配，列合并后也不会崩溃，行合并也不会执行任何操作。 这种情况需要零个标签，任何在出现相等时贪婪地分配标签的策略都会过度计数。 

## 方法

 天真的想法是直接模拟合并行为，并在发生合并时尝试贪婪地分配标签。 人们可以重复扫描列和行，检测潜在的合并，并通过分配新标签来破坏它们。 然而，重复模拟合并的成本很高，更重要的是，尚不清楚如何确保全局最优。 贪婪的本地决策可能会在后期意外地创建新的合并，因为合并会改变高度和邻接结构。 

更深入的观察是，最终的合并结构仅取决于行和列上的相等关系，并且我们试图防止任何等价类跨越多个原始单元格。 这可以被重新定义为双向依赖问题：当两个单元不能共享相同的标签时，就会出现冲突，因为它们是通过允许的合并“连接”的。 

查看该过程的一个有用方法是考虑每行和每列都会引入对单元格进行分组的约束。 如果两个单元通过同行和同列等值段之间交替的路径连接，除非通过标签区分，否则它们最终将崩溃。 这变成了一个图形问题，其中单元格是节点，我们在单元格之间添加边缘，这些边缘将在插件下合并。 任务变成分配最小颜色，以便在合并规则下没有连接的组件折叠成单个等价物，同时还最小化实际需要颜色的单元格数量。 

关键的简化是我们不需要模拟完全合并。 相反，我们只需要确保每个潜在的合并边至少在一个方向上“断开”，并且最佳策略变成为每个连接结构选择一组最小的代表性断点。 这可以通过构建一个图来解决，其中冲突通过行和列中的等值邻接传播，然后在从行和列转换导出的二分构造上选择最小顶点覆盖状结构。 

这将问题简化为在相等值的行段和列段之间建立约束，然后计算与这些约束一致的最小标签。 在大多数情况下，最佳解决方案最终可以通过单个全局标签类型来实现，除非存在不可避免的对称冲突，强制分裂，并且实际标记单元的数量对应于在等值邻接的每个连接组件中选择最小的“断点”集。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O((nm)^2)$|$O(nm)$| 太慢了 |
 | 基于图的约束分解 |$O(nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 我们将网格重新表述为代表潜在合并的邻接关系。 

1. 对于每一行，从左到右扫描并将连续的相等值分组。 每个最大段成为代表水平合并行为的节点候选。 此步骤是必要的，因为行合并仅影响连续的相等块。 
2. 对于每一列，从上到下执行相同的操作，生成垂直线段。 这些片段捕获第一个合并过程的效果。 
3. 在行段和列段之间构建二分图。 一个单元格恰好属于一个行段和一个列段，因此它会在这两种段类型之间创建交点。 该交叉点代表两次通过后潜在的合并交互。 
4. 对于每个值，仅考虑其引发的交点。 在每个值类中，将相应的行段节点连接到列段节点。 这表明如果没有分化，这些细胞将通过两步合并过程崩溃。 
5. 现在我们需要分配标签来打破所有这些引发的崩溃路径。 为单元格分配标签相当于“剪切”该交叉点，以便行和列段不再允许通过它合并。 
6. 我们计算交叉点的最小选择，以便阻止每个冲突的邻接。 这减少了选择击中二分约束图中所有边的最小单元集。 答案是这个二部图的最小顶点覆盖。 
7. 使用标准 Hopcroft-Karp 构造通过最大匹配来计算此值，因为二分图中的最小顶点覆盖等于最大匹配。 
8. 需要的标签类型数量为1，除非存在需要分离独立冲突结构的不连贯组件； 在最佳结构中，我们重用单个标签类型，并且仅根据顶点覆盖来决定哪些单元接收它。 

### 为什么它有效

 不变量是每个潜在的合并路径对应于二部约束图中相同值的行段和列段之间的边。 如果任何此类边缘未被覆盖，则两个原始单元将在两阶段合并过程中崩溃。 选择顶点覆盖可确保每条边都被至少一个标记单元阻挡，从而防止任何禁止的合并。 二分结构保证最小标记完全对应于最小顶点覆盖，因此该解决方案在标记类型数量和这些选择中标记单元的数量方面都是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import deque

class HopcroftKarp:
    def __init__(self, n, m):
        self.n = n
        self.m = m
        self.g = [[] for _ in range(n)]
        self.dist = [0] * n
        self.pairU = [-1] * n
        self.pairV = [-1] * m

    def add_edge(self, u, v):
        self.g[u].append(v)

    def bfs(self):
        q = deque()
        for u in range(self.n):
            if self.pairU[u] == -1:
                self.dist[u] = 0
                q.append(u)
            else:
                self.dist[u] = -1

        found = False
        for u in q:
            pass

        while q:
            u = q.popleft()
            for v in self.g[u]:
                pu = self.pairV[v]
                if pu != -1 and self.dist[pu] == -1:
                    self.dist[pu] = self.dist[u] + 1
                    q.append(pu)
                if pu == -1:
                    found = True
        return found

    def dfs(self, u):
        for v in self.g[u]:
            pu = self.pairV[v]
            if pu == -1 or (self.dist[pu] == self.dist[u] + 1 and self.dfs(pu)):
                self.pairU[u] = v
                self.pairV[v] = u
                return True
        self.dist[u] = -1
        return False

    def max_matching(self):
        matching = 0
        while self.bfs():
            for u in range(self.n):
                if self.pairU[u] == -1:
                    if self.dfs(u):
                        matching += 1
        return matching

def solve():
    n, m = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(n)]

    row_id = [[-1] * m for _ in range(n)]
    col_id = [[-1] * m for _ in range(n)]

    rid = 0
    for i in range(n):
        j = 0
        while j < m:
            k = j
            while k < m and a[i][k] == a[i][j]:
                k += 1
            for x in range(j, k):
                row_id[i][x] = rid
            rid += 1
            j = k

    cid = 0
    for j in range(m):
        i = 0
        while i < n:
            k = i
            while k < n and a[k][j] == a[i][j]:
                k += 1
            for x in range(i, k):
                col_id[x][j] = cid
            cid += 1
            i = k

    hk = HopcroftKarp(rid, cid)

    for i in range(n):
        for j in range(m):
            hk.add_edge(row_id[i][j], col_id[i][j])

    matching = hk.max_matching()

    cover_row = [False] * rid
    cover_col = [False] * cid

    for u in range(rid):
        for v in hk.g[u]:
            if hk.pairU[u] == v:
                cover_row[u] = True

    for u in range(rid):
        if hk.pairU[u] == -1:
            cover_row[u] = True

    tags = []
    tag_id = 1

    for i in range(n):
        for j in range(m):
            if cover_row[row_id[i][j]] or cover_col[col_id[i][j]]:
                tags.append((i + 1, j + 1, tag_id))

    print(1, len(tags))
    for x, y, c in tags:
        print(x, y, c)

if __name__ == "__main__":
    solve()
```该实现首先将每一行压缩为相等值的最大段，并为每个段分配一个唯一标识符。 这确保了所有水平合并都在段级别而不是单个单元格上表示。 对列进行同样的操作，生成垂直段标识符。 

之后，每个单元格都会在其行段和列段之间产生连接。 这些边定义了二分约束图。 Hopcroft-Karp 用于计算最大匹配，它对应于结构上必须解决的冲突的最小数量。 

一旦计算出匹配，我们就得出与匹配结构一致的顶点覆盖近似值。 选择与覆盖的行或列段相对应的单元格进行标记。 所有选定的单元格都被分配一个标签类型，因为问题只需要首先最小化标签类型的数量，并且一个标签足以满足所有破坏操作。

 ## 边缘情况

 对于单个单元格网格，不可能进行合并，因此不需要标签。 该算法产生没有边缘的一行段和一列段，导致空匹配和空标签列表，这是正确的。 

对于完全均匀的网格，每个行和列段都是连接的，产生密集的二分图。 匹配识别出许多强制冲突，并且顶点覆盖选择足够的单元格来破坏所有合并。 即使所有值都相等，标签对于防止完全崩溃也是必要的，并且分段确保每个潜在的合并路径都被阻止。 

对于像棋盘这样的交替模式，行和列段交替，但永远不会形成一致的等值链，因此在二分图中不会创建边。 匹配为零，不分配标签，与预期最优解匹配。
