 ## 解决方案
 ## 问题理解

 我们得到一个仅由小写英文字母组成的字符串。 在一次移动中，我们可以选择字符串的连续部分并将其反转，或者我们可以选择保持字符串不变。 目标是获得按字典顺序最多可以使用一次这样的反转生成的最小字符串。 

这里的字典顺序的行为类似于字典顺序：我们从左到右比较两个字符串，它们不同的第一个位置决定哪个较小，较早的字母字符较小。 

输入包含多个测试用例，跨测试用例的所有字符串总长度很大，可达约 150 万个字符。 这立即排除了任何显式尝试每种可能反转的方法，因为枚举所有子字符串对于每个测试用例都是二次的，并且会超出时间限制几个数量级。 

一个幼稚的错误来自于只考虑交换或局部改进。 例如，在输入时`ba`，反转整个字符串会产生`ab`，这是最优的，但是在类似的事情上`abac`，在没有全局策略的情况下反转任意片段可能会忽略将一个小角色向左移动很远有时是有益的，即使它会干扰中间。 

当局部良好的反转不是全局最优时，就会出现另一种微妙的失败情况。 例如，在`cab`, 仅反转`ca`给出`acb`，这是最优的，但在较长的字符串中，早期改进的贪婪选择可能会阻碍以后更好的改进，除非该策略保证全局最小性。 

核心难点在于单次反转同时影响三个区域：段之前的前缀、反转的段本身以及其之后的后缀。 后缀保持不变，但反转的段会在内部更改顺序，这使得对所有子字符串进行暴力推理的成本很高。 

## 方法

 暴力解决方案会尝试所有对`(l, r)`并在反转该段后计算结果字符串，然后选择最小值。 有$O(n^2)$这样的货币对和每次逆转成本$O(n)$实现，因此总复杂度变为$O(n^3)$如果从字面上或至少是这样做的话$O(n^2)$通过仔细的模拟。 和$n$最多$10^5$，这是不可能的。 

即使我们仅通过比较候选者而不完全重建字符串来进行优化，我们仍然面临$O(n^2)$候选者，并且在最坏的情况下每次比较可能需要线性时间。 因此，我们需要一种方法来完全避免枚举段。 

关键的观察是字典顺序是尽早确定的。 我们只关心改进字符串可以变小的第一个位置。 这表明我们应该关注最早的指数，通过一次逆转就可以带来更好的性格。 

如果我们固定一个位置`i`，唯一有用的操作是从后缀中带一些字符`[i+1, n-1]`定位`i`通过反转以该字符结尾的段。 反转后，所选字符将成为该位置的新值`i`。 

所以策略就变成了：对于每个位置，询问字符串后面是否存在更小的字符。 如果是，我们想要最小的这样的字符，并且在它的出现中我们更喜欢最右边的一个，因为反转到最右边的出现可以最大限度地提高早期位置的字典顺序，而不限制未来的选择。 

这将问题简化为跟踪每个后缀、最小字符及其出现的位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（尝试所有逆转）| O(n3) 或 O(n2) | O(n) | 太慢了 |
 | 后缀-min + 单最优反转 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过决定是否应该执行逆转以及如果执行逆转则在何处执行来构建答案。 

1. 预先计算每个位置`i`后缀中最小的字符开始于`i`，以及这个最小字符出现的最后位置。 这是从右到左完成的，以便我们可以维护最小字符和代表性索引。 如果我们在位置上开始逆转，这使我们能够立即获得最好的角色`i`。 
2. 从左到右扫描字符串。 在位置`i`， 比较`s[i]`后缀中最小的字符开始于`i + 1`。 如果后缀包含严格较小的字符，则位置`i`可以通过将较小的角色向前推进来改进。 
3. 一旦我们找到第一个位置`i`如果可以进行改进，我们会确定最佳字符串必须与从此位置开始的原始字符串不同。 这是因为任何按字典顺序排列的较小结果都必须减少最早可能的不匹配点。 
4.设后缀中最好的字符为`c`，并让`j`是最后一次出现`c`。 我们反转子串`i`到`j`。 这个动作`c`定位`i`并将一步之间的所有内容以相反的顺序向右移动。 
5. 执行单次反转后输出结果字符串。 如果没有位置`i`可以改进一下，我们输出原始字符串。 

为什么它有效可以归结为控制第一个不同的位置。 任何有效的最佳答案必须与原始字符串的某个位置一致`i`然后要么保留`s[i]`或者用更小的东西替换它。 因此，最早可以进行替换的索引是唯一有意义的决策点。 一旦我们选择提高位置`i`，我们想要可以占据它的尽可能最小的字符，并且从最右边的出现位置开始放置它可以确保我们不会意外地阻止字符串中更早的更好的配置。 任何改变较早索引或使用更大字符的替代反转都不能产生比此结构按字典顺序更小的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve(s: str) -> str:
    n = len(s)
    if n <= 1:
        return s

    suf_min = [''] * n
    suf_pos = [0] * n

    suf_min[-1] = s[-1]
    suf_pos[-1] = n - 1

    for i in range(n - 2, -1, -1):
        if s[i] < suf_min[i + 1]:
            suf_min[i] = s[i]
            suf_pos[i] = i
        elif s[i] > suf_min[i + 1]:
            suf_min[i] = suf_min[i + 1]
            suf_pos[i] = suf_pos[i + 1]
        else:
            suf_min[i] = s[i]
            suf_pos[i] = suf_pos[i + 1]

    s = list(s)

    for i in range(n - 1):
        if suf_min[i + 1] < s[i]:
            j = suf_pos[i + 1]
            s[i:j + 1] = reversed(s[i:j + 1])
            return ''.join(s)

    return ''.join(s)

def main():
    t = int(input())
    for _ in range(t):
        s = input().strip()
        print(solve(s))

if __name__ == "__main__":
    main()
```该解决方案构建了一个后缀结构，用于跟踪每个位置右侧可用的最小字符及其出现的位置。 这避免了对每个索引重新计算扫描。 

对第一个改进点的扫描至关重要，因为词典最小化取决于我们可以减少字符的最早位置。 一旦发现，我们立即应用反转并停止，因为任何后续修改都会使更大的前缀保持不变，从而变得更糟。 

逆转本身使用了列表上的 Python 切片，这足够有效，因为每个测试用例最多发生一次。 

## 工作示例

 ### 示例 1：`abbcabaac`我们计算后缀最小值并从左到右扫描，直到找到第一个可改进的位置。 

| 我| s[i] | i | 之后的后缀 min 决定|
 | --- | --- | --- | --- |
 | 0 | 一个 | 乙| 没有变化|
 | 1 | 乙| 一个 | 发现改进|

 在索引 1 处，我们看到稍后存在一个较小的字符，即`a`。 最后一次出现的`a`后缀中确定反转边界，即最远`a`我们可以使用。 

反转该段后，字符串变为`aaabacbbc`。 

这表明该算法不仅选择最近的较小字符，还将反转延伸到最后一次出现，确保第一个决策点的最大改进。 

### 示例 2：`cbad`我们跟踪这个过程：

 | 我| s[i] | i | 之后的后缀 min 决定|
 | --- | --- | --- | --- |
 | 0 | c | 一个 | 发现改进|

 最小的后缀字符是`a`，所以我们从索引 0 反转到最后一个`a`，即索引 2。 反转`cba`给出`abc`，生产`abdc`添加未更改的后缀后。 

这演示了算法如何优先考虑尽早引入全局最小的可达字符。

 ## 边缘情况

 单字符字符串，例如`a`永远不会触发反转条件，因为没有后缀可以改进。 该算法立即返回它，因为后缀最小结构很简单并且位置上的循环为空。 

在像这样的字符串中`ba`，索引 0 处的后缀最小值为`a`，小于`b`。 该算法选择片段`[0, 1]`，反转它，并产生`ab`。 后缀跟踪可确保即使是最简单的情况也能与较大的输入保持一致的处理。 

对于完全递减的字符串，例如`cba`，每个位置都可以立即得到提升。 算法选择第一个索引`0`, 识别`a`作为最小的后缀字符，并反转整个字符串。 这会产生`abc`，这与全局最优匹配，因为任何部分反转都会使更大的前缀保持不变，并导致更糟糕的词典结果。
