 ## 解决方案
 ## 问题理解

 我们看到一座怪物塔，从下到上排成一排。 每层都有一个固定强度的怪物。 玩家开始时有一定的初始力量$x$。 在任何时刻，玩家都只能看最低的$k$剩余楼层，并选择其中一只强度不超过当前强度的怪物。 当这样的怪物被杀死时，玩家会获得其力量，并且该楼层会消失，导致所有较高楼层向下移动一层。 

任务是确定最小的初始强度$x$这样玩家最终可以根据这些规则移除塔中的所有怪物。 

关键在于移除怪物会改变塔的结构，因此哪些怪物位于“最低”$k$” 窗口是动态的。这个过程没有固定的顺序；这取决于我们决定移除哪些怪物。

 约束允许最多$2 \times 10^5$测试用例中的总数是怪物，因此任何比每个测试用例的线性运算更糟糕的解决方案都将陷入困境。 重复扫描最低值的二次模拟$k$每次拆除后的楼层最多需要$O(n^2)$最坏情况下的操作，速度太慢$n$达到$2 \times 10^5$。 

天真的推理的一个微妙的失败案例是假设我们应该总是在当前窗口中获取最小的可到达的怪物。 这可能会让玩家陷入延迟大收益的境地。 

例如，考虑一座小塔：```
n = 3, k = 2
a = [5, 1, 10]
```如果我们总是选择最小的可用怪物，我们可能会先拿 1 个，慢慢增加强度，但最佳策略可能需要先拿 5 个才能更早达到 10 个。 排序约束使得贪婪的局部选择变得非常重要。 

另一个隐藏的问题是忽略了移除一层后，新的怪物进入可到达的窗口。 解决第一个问题的解决方案$k$因为静态集是不正确的。 

## 方法

 暴力方法可以模拟整个过程。 我们尝试候选人的初始强度$x$，维持当前怪物数组，重复扫描最低的$k$元素来寻找可杀死的怪物。 每次移除都需要移动结构并重新扫描$k$元素。 在最坏的情况下，我们执行$n$清除量，每次成本计算$O(k)$，导致$O(nk)$，退化为$O(n^2)$。 

这太慢了，因为每次操作后状态都会发生变化，但唯一真正的困难是跟踪当前可用的最低怪物$k$考虑到目前的实力，哪些是可以杀死的。 

关键的观察是我们实际上不需要显式地模拟动态结构。 相反，我们可以从可行性方面考虑：对于固定的起始强度$x$，我们想知道是否存在一个删除序列，在每一步我们可以在第一个元素中选择任何元素的约束下消耗所有元素$k$剩余的存活元素。 

重要的结构见解是，我们可以与之交互的候选者集仅取决于我们已经删除的元素数量，而不是它们的确切身份。 这表明贪婪的可行性检查：始终考虑最小的可到达的未处理元素，并在保持约束的同时模拟强度的最佳可能增长。 

为了有效地决定可行性，我们可以对当前可到达的元素使用优先级结构，并维护第一个的滑动边界$k$活着的怪物。 当我们移除怪物时，新的怪物就会进入窗口。 在每一步中，我们总是选择我们可以杀死的最小的怪物，因为如果存在更小的有效选项，那么选择更大的怪物永远不会提高未来的可行性； 它只会降低灵活性。 

这将问题变成了单调的可行性检查：如果给定$x$作品，任何更大的$x$也有效。 因此，我们可以二分查找$x$。 

每张支票都运行在$O(n \log n)$使用堆（或者甚至$O(n)$通过仔细的两指针管理进行摊销），使整个解决方案变得高效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(n^2)$|$O(n)$| 太慢了|
 | 贪婪可行性+二分查找|$O(n \log n \log V)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题视为决策问题：给定初始强度$x$，我们可以清理塔吗？ 

1. 固定一个值$x$并初始化一个代表当前最低值的结构体$k$可用的怪物。 我们还维护一个指针来跟踪有多少怪物进入了窗口。 
2.插入第一个$k$怪物进入以力量为关键的最小结构。 这代表当前可以到达的所有怪物。 
3. 当仍有怪物残留或建筑物内时，反复尝试选择我们可以杀死的怪物。 如果结构中最小的可用怪物的强度大于当前强度，则不存在有效的移动，并且此$x$失败。 
4. 否则，移除该怪物，将其强度添加至当前强度，并将其标记为已删除。 移除一个怪物后，将下一个看不见的怪物推入结构中，以便窗口始终包含最低的怪物$k$活着的怪物。 
5. 继续直到所有怪物都被移除。 如果我们成功的话，最初的$x$就足够了。 
6. 使用二分查找$x$，因为可行性是单调的：增加初始强度永远不会减少可用的动作。 

为什么这有效来自于约束的结构。 在任何时候，唯一的限制是我们可以从最低的中选择$k$剩余元素。 堆正好代表该集合。 其中，选择最小的可杀死怪物是安全的，因为尽早增强力量只会扩大未来的选择。 任何跳过可杀死的较小怪物而选择较大怪物的策略都无法解锁以前无法获得的新选项； 它只会消耗更多的力量而没有任何好处，因此最佳策略总是倾向于最小的可行去除。 

二分搜索是合理的，因为清除塔的能力在初始强度上是单调的。 如果某个$x$允许完整的有效删除序列，然后任何$x' > x$可以遵循相同的顺序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

def can(x, a, k):
    n = len(a)
    cur = x
    i = 0
    heap = []

    # initial window
    while i < n and i < k:
        heapq.heappush(heap, a[i])
        i += 1

    # process
    used = k
    while heap:
        # ensure we can take something
        if heap[0] > cur:
            return False

        val = heapq.heappop(heap)
        cur += val

        if i < n:
            heapq.heappush(heap, a[i])
            i += 1

    return True

def solve():
    T = int(input())
    for _ in range(T):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))

        lo, hi = 0, sum(a)
        ans = hi

        while lo <= hi:
            mid = (lo + hi) // 2
            if can(mid, a, k):
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1

        print(ans)

if __name__ == "__main__":
    solve()
```这`can`函数模拟固定起始强度的可行性。 堆总是恰好包含下一个$k$按数组中出现的顺序排列的可到达的怪物，这与动态“最低”相匹配$k$删除后“地板”，因为删除的元素永远不会重新插入，并且新的元素随着先前的元素消失而进入。 

二分搜索在答案范围内运行，其边界为$0$以及所有怪物力量的总和，因为这对于任何可能的增长来说都是一个微不足道的上限。 

一个微妙的点是正确维护滑动窗口：当一个元素被删除时，我们只推送下一个看不见的元素，确保堆大小反映当前可到达的段。

这表明，尽管 10 很大并且最初无法访问，但移动窗口允许它在早期删除后变得可访问。 

### 示例 2

 输入：```
n = 3, k = 1
a = [5, 1, 10]
```尝试$x = 5$。 

| 步骤| 当前实力| 堆| 行动|

这个例子说明了为什么初始强度至关重要：早期的无能会阻塞整个链条，即使后来的增长就足够了。

 ## 边缘情况

 当出现以下情况时会出现极端情况：$k = 1$。 在这种情况下，结构退化为严格的线，其中只有当前最上面的怪物可用。 该算法自然地处理这一问题，因为堆始终只包含一个元素，如果该元素超过当前强度，则会立即发生故障。 对于输入：```
n = 3, k = 1
a = [5, 1, 10], x = 5
```模拟逐步进行：先取5，强度变为10，然后取1，然后取10。 任何较小的初始值在第一次比较时都会立即失败。 

另一个边缘情况是，与任何合理的初始强度相比，所有怪物都已经很小。 在：```
n = 5, k = 3
a = [1, 1, 1, 1, 1]
```甚至$x = 1$成功了，基于堆的进程在重新填充结构的同时不断地耗尽结构。 堆总是反映下一个的不变量$k$活的元素确保在移动过程中不会跳过或丢失任何怪物。 

第三种情况是当一个非常大的怪物被深埋时：```
n = 4, k = 2
a = [1, 1, 100, 1]
```即使最初无法达到 100，重复消耗前两个小怪物也会增加足够的强度，最终将其纳入活动窗口。 滑动插入机制保证它最终在正确的时间进入堆，并且模拟不会过早地丢弃它。
