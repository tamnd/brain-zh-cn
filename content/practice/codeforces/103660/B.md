 ## 解决方案
 ## 问题理解

 我们拥有一种伤害技能，其输出取决于其蓄力时间。 伤害呈线性增长，但一旦充电时间超过阈值，增长速度就会发生变化$k$。 之前或时$k$秒，每一秒都有贡献$x$损害。 后$k$，额外的秒数贡献$y$反而造成伤害。 

对于每个测试用例，我们还会获得敌人生命值的列表。 该技能仅使用一次，并对所有敌人造成相同的伤害。 目标是选择充电时间最小的整数$t$这样造成的伤害至少是所有敌人中的最大生命值，因为每个敌人都会受到相同的伤害，并且最弱的约束是最强的敌人。 

输入规模较小：最多 100 个测试用例，每个测试最多 100 个敌人，除了健康值外，所有参数都以 100 为界，健康值最多可达$10^9$。 这立即排除了任何以天真的方式尝试所有可能的充电时间直至最大健康值的方法。 线性扫描可达$10^9$每个测试用例都会太慢。 

边缘情况中唯一的微妙之处来自于损坏的分段定义。 如果$x < y$，然后充电更长时间$k$变得更加高效，因此最优解可能会超调$k$显著地。 如果$x \ge y$，然后充电超过$k$要么更糟，要么毫无用处，所以最好的策略总是在第一段达到目标时立即停止，或者最多恰好在$k$。 

一个幼稚的错误是假设具有单一斜率的单调行为或完全忽略第二段。 例如，如果$k = 2, x = 10, y = 1$，敌人的生命值为 25，停在$t = 3$给出$20 + 1 = 21$，这比提前停止更糟糕，但无效，因为损坏仍然会增加，但速度要慢得多。 正确的答案必须明确考虑这两个阶段。 

## 方法

 暴力策略会尝试所有可能的充电时间$t$从 1 开始，计算每个的损伤函数$t$，并检查是否足以杀死所有敌人。 对于每位候选人$t$，计算损伤为$O(1)$，因此每个测试用例的成本与答案值成正比。 

问题是答案可以大到$10^9$，因为敌人的生命值达到了这个范围并且$x, y$可以小到1。在最坏的情况下，我们可能需要为每个测试用例模拟数十亿个时间步，这是不可行的。 

关键的观察结果是损伤函数是单调的$t$。 它线性增加，首先是斜率$x$，然后用斜率$y$。 因为单调，一次一次$t$就足够了，任何更大的时间也足够了。 这允许对答案进行二分搜索。 

对于任何固定的$t$，我们可以使用分段公式计算常数时间内的伤害，将其与最大敌人生命值进行比较，并使用二分搜索找到最小的有效值$t$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(T \cdot ans)$|$O(1)$| 太慢了 |
 | 二分查找 |$O(T \cdot \log ans)$|$O(1)$| 已接受 |

 ## 算法演练

 我们减少每个测试用例以找到最小值$t$使得伤害（t）≥最大生命值。 

1.读取所有敌人的生命值并计算$H = \max a_i$。 这是唯一重要的值，因为所有敌人都会受到相同的伤害。 
2. 定义函数$f(t)$计算伤害：

 如果$t \le k$， 返回$t \cdot x$。 

否则返回$k \cdot x + (t - k) \cdot y$。 

这直接符合分段计费规则。 
3. 设置二分查找范围。 答案至少是 0，至多是一个安全上限，例如$10^9$或者$H + k$，因为即使斜率为 1，所需的时间也不能超过健康范围。 
4. 执行二分查找$t$。 对于每个中点$m$, 计算$f(m)$。 如果$f(m) \ge H$， 然后$m$就足够了，我们尝试更小的值。 否则我们需要更多时间。 
5.二分查找后，输出最小的$t$即满足条件。 

关键的想法是我们永远不会一步一步地模拟时间； 我们只评估封闭形式的损伤函数。 

### 为什么它有效

 功能$f(t)$不减于$t$因为两个段都有非负斜率$x$和$y$。 一旦我们穿越$k$，斜率可能会改变，但永远不会变成负值。 这保证了一个单调谓词：如果一个时间$t$足以杀死所有敌人，那么任何更大的时间也足够了。 二分搜索之所以有效，正是因为这种单调性成立，确保每一步都可以安全地将搜索空间减半。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def damage(t, k, x, y):
    if t <= k:
        return t * x
    return k * x + (t - k) * y

def solve():
    T = int(input())
    for _ in range(T):
        n, k, x, y = map(int, input().split())
        a = list(map(int, input().split()))
        H = max(a)

        lo, hi = 0, 10**18

        while lo < hi:
            mid = (lo + hi) // 2
            if damage(mid, k, x, y) >= H:
                hi = mid
            else:
                lo = mid + 1

        print(lo)

if __name__ == "__main__":
    solve()
```功能`damage`完全按照问题中的描述对分段线性行为进行编码。 二分搜索在整数时间域上运行，缩小范围，直到找到最小的足够值。 

一个微妙的实现细节是上限的选择。 使用像这样的大常数$10^{18}$是安全的，因为它轻松地超过了给定限制下任何可能所需的时间。 另一个重要的一点是仅计算最大生命值，而不是在该步骤之后单独求和或处理所有敌人。

 ## 边缘情况

 一个重要的边缘情况是当$y < x$，意味着该技能在之后变得效率较低$k$。 例如：

 输入：```
1
1 3 10 1
50
```为了$t \le 3$，伤害为$10t$。 所以在$t=3$，伤害为 30。之后，生长速度急剧减慢。 二分查找正确找到$t=5$自从$50$仅需要通过第一段达到 50。 

跟踪搜索：

 在$t=5$，伤害为$3*10 + 2*1 = 32$，仍然不够，所以我们需要更大的值，但由于斜率下降，达到 50 可能需要更大的 t，二分搜索仍然可以正确处理。 

这证实了该算法不假设增加斜率质量，仅假设单调性。 

另一个极端情况是所有敌人的生命值都非常低。 答案应该折叠到最小$t$， 经常$t=1$，二分搜索正确地找到了它，因为 Damage(1) 已经满足条件并且搜索立即收敛到下界。
