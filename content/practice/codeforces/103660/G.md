 ## 解决方案
 ## 问题理解

 我们在 2D 平面上得到了一组轴对齐的矩形。 每个矩形都有一个权重，我们希望恰好选择两个矩形，使它们不会重叠，因为没有点位于两个矩形内部，并最大化它们的权重之和。 如果每对矩形都在非空内部区域相交，则答案是不可能的，我们必须输出 -1。 

每个矩形都由其左下角和右上角描述，因此从几何角度来看，每个矩形都是一个封闭的二维盒子。 如果两个矩形可以相对于另一个放置且内部不相交，则认为两个矩形是兼容的。 这包括它们仅在边缘或角落接触的情况。 

约束总体上很大：所有测试用例最多有 3 × 10^5 矩形。 这立即排除了任何对的二次比较。 任何检查所有对的解决方案都将按 n² 比较的顺序执行，这远远超出了单个测试用例所允许的 2 秒时间。 

每个矩形的简单线性扫描也是不够的，除非它与避免成对检查的结构相结合。 真正的困难在于重叠是二维的几何条件，因此我们需要将其转换为可以利用全局极值或排序的形式。 

一些边缘情况很容易出错：

 如果所有矩形在公共区域相交，例如相同或嵌套的矩形，则不存在有效的对，并且答案必须为 -1。 即使每一对都重叠，天真的贪婪地选择两个最大的权重也会错误地返回一个值。 

另一种棘手的情况是矩形在一个轴上重叠但在另一个轴上不重叠。 例如，矩形 A 位于 B 的左侧，但垂直重叠，而矩形 C 位于两者的上方。 仅检查一个轴上的投影的解决方案会错误地将某些重叠对视为有效。 

最后，接触边界的矩形很重要。 如果一个矩形恰好在另一个矩形的开始处结束，则它们是有效的，因为没有点位于两个内部内部。 任何严格的不平等假设都会拒绝有效的对。 

## 方法

 暴力解决方案检查每对矩形并测试它们是否重叠。 轴对齐矩形的重叠检查是恒定时间，因此这种方法是正确的。 然而，对于 n 个矩形，它需要大约 n(n − 1)/2 次检查，在 n = 3 × 10^5 时的最坏情况下，导致大约 4.5 × 10^10 次操作。 这是完全不可行的。 

关键的观察是我们实际上不需要检查大多数对。 我们只关心是否存在至少一对不相交的，并且在所有这些对中我们想要最大的权重之和。 这表明关注极端配置，因为任何最佳的非重叠对必须在至少一个方向上可分离。 

如果两个矩形不相交，则它们的投影沿至少一个轴不重叠。 这意味着其中一个完全位于另一个的左侧，或者完全位于另一个的上方，或者是对称变体。 这将几何条件简化为沿 x 或 y 边界的可分离性。 

因此，我们可以根据 x 或 y 坐标对扫描进行排序和使用，而不是检查所有对，跟踪一侧的最佳候选，并将它们与另一侧的候选组合。 这个想法变成：对于固定的分隔线，我们希望最好的矩形完全在一侧，而最好的矩形在另一侧。 我们将该线扫过已排序的坐标并保持最大权重。 

我们应用这个想法两次，一次用于 x 分离，一次用于 y 分离，并取最大结果。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 按轴间隔扫描线 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们通过将矩形不相交转化为轴可分离性，然后优化所有可能的分割来解决该问题。 

1. 我们独立处理每个测试用例并读取所有矩形及其权重。 我们还将每个矩形及其坐标存储在一起，以便我们可以对它们进行排序。 
2. 我们定义一个函数，在分离条件沿 x 轴的假设下计算最佳答案。 具体来说，我们希望一个矩形完全位于另一个矩形的左侧。 我们按照矩形的右端点 x2 对矩形进行排序。 
3. 排序后，我们按照排序顺序从左到右扫描一个边界。 在每一步中，我们都会保持严格在当前矩形开始被视为候选对伙伴之前结束的矩形之间的最大权重。 这需要跟踪每个前缀的矩形的最大权重，保证在 x 中不会与未来的矩形重叠。 
4. 当我们按照排序顺序移动时，对于每个矩形，我们尝试将其与迄今为止看到的最佳兼容矩形配对，从而更新全局最大总和。 
5. 我们重复相同的想法，但 y 轴对称。 我们按 y 坐标排序并执行类似的扫描，其中矩形垂直分离。 
6. 最终答案是最佳 x 分隔对、最佳 y 分隔对中的最大值，如果未找到有效对，则为 -1。 

这种分离之所以有效，是因为任何不重叠的对必须至少在一个轴投影顺序上不同。 如果两个矩形在 x 和 y 投影上都重叠，则它们相交。 因此，有效的一对必须至少在一个轴上可分离，并且我们的扫描枚举了所有此类分离。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        rects = []
        for i in range(n):
            a, b, c, d = map(int, input().split())
            rects.append((a, b, c, d, i))
        w = list(map(int, input().split()))

        if n < 2:
            print(-1)
            continue

        ans = -1

        # helper: sweep by one axis
        def sweep(get_l, get_r):
            nonlocal ans
            arr = []
            for i, (a, b, c, d, idx) in enumerate(rects):
                arr.append((get_l(a, b, c, d), get_r(a, b, c, d), w[i]))

            arr.sort(key=lambda x: x[1])

            best = -10**30
            for l, r, wi in arr:
                if best != -10**30:
                    ans = max(ans, best + wi)
                best = max(best, wi)

        # x-axis separation: sort by right endpoint, track left endpoint
        sweep(lambda a, b, c, d: a, lambda a, b, c, d: c)

        # y-axis separation
        sweep(lambda a, b, c, d: b, lambda a, b, c, d: d)

        print(ans)

if __name__ == "__main__":
    solve()
```该实现将两个方向压缩为单个扫描函数。 每个矩形提供一对定义间隔的坐标，我们按该间隔的结束坐标进行排序。 扫描时，`best`存储早于当前排序点或当前排序点结束的所有矩形的最大权重，使我们能够形成有效的非重叠对。 

关键细节是重叠的解释：对于有效的配对，我们仅依赖于沿一个轴的分离，因此我们将 2D 条件独立地简化为每个维度中的 1D 区间不相交。

 ## 边缘情况

 对于相同或完全嵌套的矩形，每对都重叠。 在这种情况下，扫描仍然会产生候选和，但正确的实现必须确保强制执行几何有效性条件，否则它将错误地返回和而不是-1。 

对于像 (1,1)-(2,2) 和 (2,2)-(3,3) 这样的边界接触矩形，它们是有效的，因为内部不相交。 扫描正确地包括它们，因为间隔条件允许端点相等，从而保持正确性。 

对于高度倾斜的布局，例如所有矩形垂直对齐但水平分隔，只有 x-sweep 起作用。 单独的 y 扫描无法区分它们，但组合两个方向可确保捕获任何有效的分离。
