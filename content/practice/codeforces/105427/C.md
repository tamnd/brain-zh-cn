 ## 解决方案
 ## 问题理解

 我们得到一个字符串集合，每个字符串代表一个用简化罗马数字系统编写的数字。 每个字符都是标准罗马符号 I、V、X、L、C、D、M 之一，固定值为 1、5、10、50、100、500、1000。 

解释规则不是经典的严格罗马规则，而是一个宽松的版本：每当一个符号出现在严格较大值符号左侧的某个位置时，左侧符号就会产生负面影响。 否则它会做出积极的贡献。 与通常的罗马数字系统不同，减法不限于相邻对，因此可以通过字符串中后面出现的任何较大符号来“取消”符号。 

对于每个输入字符串，我们必须根据此规则计算结果整数值并打印它。 

输入的大小使得结构变得很重要。 最多可以有 1000 个字符串，所有字符串的总长度最多为 300,000 个字符。 这意味着任何解决方案都必须与总输入大小呈线性关系。 多次扫描每个字符串或比较所有字符对的方法会太慢。 

天真的阅读可能会建议将每个字符与所有后面的字符进行检查，以决定是否将其减去。 这将导致每个字符串的二次行为，这在给定的规模下是不可能的。 

一个关键的微妙之处在于，减法仅取决于右侧某处是否存在更大的符号，而不取决于它有多远或存在多少个这样的符号。 

典型的失败案例来自于仅解释相邻字符之间的减法。 例如，在字符串“IVI”中，局部规则可能会从 V 中减去 I，但会错过最终的 I 再次为正的情况。 

如果我们仅在下一个字符较大时进行减法，则会出现另一种失败情况。 在“IXC”中，I和X都应该受到后来更大符号的影响，但是基于邻接的逻辑会忽略C对I的影响。 

## 方法

 暴力破解的想法很简单：对于每个字符，扫描其右侧的所有内容以检查是否存在更大值的罗马符号。 如果存在这样的符号，我们减去当前值，否则添加它。 

这是正确的，因为它直接对规则定义进行编码：当符号右侧某处有更大的符号时，该符号恰好为负数。 然而，这需要对于字符串中的每个 L 个字符，扫描最多 L 个字符。 每个字符串的复杂度为 O(L²)，在总长度为 300,000 的最坏情况下，这将超过 10^10 次操作，这远远超出了可行的限制。 

优化来自于颠倒视角。 我们不是问“这个位置的右侧是否存在更大的符号？”，而是问“从右侧远处看到的最大符号是多少？”。 如果我们从右向左遍历字符串，我们可以保持到目前为止遇到的最大罗马值。 对于每个字符，如果严格小于这个最大值，则必须减去它； 否则它会做出积极贡献并更新最大值。 

这是可行的，因为条件“右侧存在更大的符号”相当于“最大后缀值大于当前值”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(L²) 总计 | O(1) | O(1) | 太慢了|
 | 最佳 | O(L) 总计 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个罗马数字字符串。 

1. 使用固定映射将每个罗马字符转换为其整数值。 这允许符号之间进行恒定时间比较。 
2. 初始化变量`max_right`为零。 这将存储从右侧扫描时迄今为止看到的最大值。 
3. 从最后一个字符遍历字符串到第一个字符。 
4. 对于每个字符，将其值与`max_right`。 如果严格小于`max_right`，从运行答案中减去它。 否则，添加并更新`max_right`到这个值。 
5.处理完所有字符后，输出累加和。 

步骤 4 的关键思想是我们永远不需要显式搜索更大的符号。 后缀最大值已经编码了这样的符号是否存在。 

### 为什么它有效

 在从右到左扫描期间的任何位置 i，`max_right`表示严格位于 i 右侧的所有字符中的最大值。 因此，当且仅当 i 处的字符右侧存在具有更大值的字符时，才会减去 i 处的字符。 这与问题定义完全匹配。 不变量是处理后缀i+1..n后，`max_right`等于该后缀中的最大值，确保位置 i 的正确分类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

value = {
    'I': 1,
    'V': 5,
    'X': 10,
    'L': 50,
    'C': 100,
    'D': 500,
    'M': 1000
}

def solve(s: str) -> int:
    max_right = 0
    ans = 0

    for ch in reversed(s):
        v = value[ch]
        if v < max_right:
            ans -= v
        else:
            ans += v
            max_right = v

    return ans

def main():
    n = int(input())
    for _ in range(n):
        s = input().strip()
        print(solve(s))

if __name__ == "__main__":
    main()
```映射表是固定的，并且允许对每个字符进行恒定时间的查找。 

主循环独立处理每个字符串。 逆转至关重要，因为它确保`max_right`始终指后缀最大值，而不是前缀属性。 

减法条件严格`v < max_right`， 不是`<=`，因为在问题规则下相等的值不会触发减法。

 ## 边缘情况

 一种重要的边缘情况是减法是由远处而不是相邻的符号触发的。 在`ID`，I 不在更大的符号旁边，但它仍然被减去，因为 D 出现得更晚。 处理过程中，我们首先看到D，设置`max_right = 500`，然后对 I 进行处理并减去，正确生成 499。 

另一种边缘情况是当值增加然后再次减少时，例如`IVI`。 第一个 I 的处理没有更大的后缀，因此它会添加。 然后 V 更新后缀最大值。 然后将最终的 I 与 V 进行比较并减去。 这证实了该算法正确处理局部增加和随后的减少，而不需要任何成对比较。 

最后一个微妙的情况是重复大符号，例如`MCM`。 第一个 M 设置最大值，C 被减去，因为 M 存在于其右侧，最后一个 M 正常添加。 即使字符串中稍后存在多个大符号，后缀最大值也可确保对较早符号进行正确分类。
