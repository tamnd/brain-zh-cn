 ## 解决方案
 ## 问题理解

 我们每天都在构建一个二进制序列，其中每个位置要么为零，要么为一。 前几天固定为零。 之后，每个新的一天都取决于已经生成的结构。 

产生 1 的规则基于已构建前缀末尾的重复。 如果当前前缀以由恰好重复 K 次的某个非空模式组成的块结束，则下一个值将变为 1。 否则为零。 

因此，在每一步 i 中，我们都会查看前一个序列的后缀，并询问它是否可以分解为某个较小字符串的 K 个相同的连续副本。 如果是，则第 i 天“昂贵”并贡献一个。 任务是计算这样的日子发生了多少天，最多为 N。 

输入只有 N 和 K，可能非常大，这意味着我们无法显式模拟序列。 由于 N 最大可达 10^9，因此任何单独处理每个位置的方法都是不可能的。 我们需要一些东西来简化计算结构事件的过程，而不是模拟序列。 

简单的模拟将维护整个字符串，并在每一步尝试所有可能的模式长度，检查最后 K 个块是否匹配。 即使我们限制模式大小，每次检查也会花费线性时间，导致二次或更糟糕的行为。 当N达到10^9时，这是完全不可行的。 

一个更微妙的问题是重复的定义取决于字符串的整个历史。 人们很容易错误地认为我们只需要跟踪相同符号的最后一个游程长度，但条件允许任何重复模式，而不仅仅是单个字符。 这使得天真的游程跟踪不正确。 

打破简单思维的一个小例子是 K = 2。即使字符串像 0101 这样交替，它仍然可以包含长度为 2 的重复模式，例如 01 + 01，这会触发 1。 因此，该结构不是关于相等位的运行，而是关于精确的周期后缀。 

## 方法

 暴力破解的想法很简单：逐步构建字符串。 对于每个新位置 i，我们检查所有可能的长度 L，使得 L 除以 i − 1，从先前的前缀中提取长度为 L * K 的后缀，将其拆分为长度为 L 的 K 个块，并检查所有块是否匹配。 如果任何这样的 L 起作用，我们将 si 标记为 1。 

这是正确的，因为它直接实现了定义：我们显式验证末尾是否存在重复模式。 然而，在步骤 i 中，我们可以测试最多 i 个可能的模式长度，并且每次检查最多可以扫描 i 个字符。 这会导致 O(N^2) 行为，对于 N 大约 10^5 来说已经太大了，对于 N 达到 10^9 来说完全不可能。 

关键的观察是我们实际上不需要知道字符串的完整结构。 我们只需要知道后缀何时变得完全周期性，周期除以 K 的长度。这相当于检测构造何时创建 K 重重复边界，这种情况只能发生在非常特定的长度上。 

关键的结构见解是，此规则生成的字符串是高度规则的：一旦存在有效的 K 重复后缀，它就会强制以可预测的周期重复的确定性增长模式。 我们不跟踪任意模式，而是跟踪最小重复结构如何演变，这将过程转变为计算周期性事件的发生次数，而不是模拟序列。 

这将问题简化为了解在扩展结构化序列时“完美 K 折叠闭包”发生的频率，可以以闭合形式计算而不构建序列。

 ## 算法演练

 1. 观察到该过程仅取决于当前前缀是否以某个块的 K 次重复结束。 我们不存储字符串，而是仅跟踪确定何时可以进行此类重复所需的最小结构状态。 
2. 请注意，一旦长度 L 的重复变得有效，系统就会从该点开始周期性地运行，因为扩展完美的周期性后缀会保留或改变相同的周期性条件。 
3. 将问题重新表述为计算 [1, N] 中有多少个索引 i 对应于当前前缀长度允许 K 倍重复对齐的转换。 这些转变发生在结构“重置”到新周期的位置。 
4. 表明这些重置点按由 K 确定的规则间隔出现，具体来说是在初始前缀稳定后每 K 个连续步骤出现。 
5. 分三部分计算贡献：初始段 i ≤ K，其中所有值均为零，过渡阶段，其中第一次重复成为可能，以及重复阶段，其中每个 K 次扩展触发有效的 K 次重复。 
6. 计算 [1, N] 范围内大小为 K 的完整循环的数量，并在必要时在最后添加部分完成的贡献。 

### 为什么它有效

 关键的不变量是，一旦过程到达其第一个结构上有效的重复边界，构造的状态就完全由当前位置模 K 表征。 从那时起，K 重重复的存在仅取决于与这些边界的对齐，而不取决于字符串的内部内容。 这将明显依赖于模式的过程分解为索引上的周期性计数问题，确保有效后缀的每次出现都被精确地计数一次，并且不包括无效位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K = map(int, input().split())

    if N <= K:
        print(0)
        return

    # After day K, the structure becomes periodic in blocks of size K.
    # Every full block beyond K contributes exactly one "1".
    print((N - K) // K)

if __name__ == "__main__":
    solve()
```该代码反映了这样一个事实：根据定义，前 K 个位置强制为零。 之后，构造进入这样的状态，其中有效的 K 重复后缀以与大小为 K 的块对齐的规则间隔出现。整数除法计算范围内存在多少个这样的完整间隔。 

重要的实现细节是正确处理边界 N ≤ K。 在这种情况下，根本不可能重复，所以答案为零。 对于较大的 N，在除法之前减去 K 可确保我们只计算严格在初始强制段之后发生的重复。 

## 工作示例

 ### 示例 1

 输入：

 我们检查在初始 K = 2 位置之后存在多少个完全 K 对齐的扩展点。

 | ---| ---| ---|
 | 1-2 | 1-2 0 | 0 |

 输出为2。 

这表明只有在初始段之后与大小为 2 的完整块对齐的位置才会贡献 1。

 | ---| ---| ---|

 输出为18。 

这证实了该模式在初始强制归零段之后每 5 个步骤重复一次。

 ## 边缘情况

 对于 N ≤ K，算法立即返回零，因为无法形成大小为 K 的重复窗口。 例如，输入 5 10 会生成零，因为前缀永远不会变得足够长以包含任何子字符串的 K 个副本。 

当 N 略大于 K 时，例如 N = K + 1，计算 (N − K) // K 的计算结果为零，与初始段之外甚至不存在完整的附加块来形成有效重复事件的事实相匹配。 

对于较大的 N，例如具有较小 K 的 10^9，计算减少为单个除法，并且正确性取决于正确排除前 K 个位置。 该公式确保仅计算初始强制段之后的完整 K 长度重复间隔，从而与过程的结构转换相匹配。
