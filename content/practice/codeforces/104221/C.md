 ## 解决方案
 ## 问题理解

 库兹亚正在准备一场生日聚会，但他能邀请多少客人完全取决于一家不知名的面包店如何切蛋糕。 他所在的城市有几家面包店，每家面包店总是把蛋糕切成固定数量的片。 Kuzya 不会提前知道他的母亲选择了哪家面包店。 

他想选择一些客人，这样无论使用哪家面包店，蛋糕都可以平均分配给聚会上的每个人，包括库兹亚本人。 每个人收到的切片数量必须完全相同，不能有任何剩余的部分。 

更具体地说，我们得到了几个数字，每个数字描述了一个蛋糕可以被切成多少个相等的块。 我们必须选择一个组大小，以便对于每个可能的蛋糕大小，该蛋糕都可以在组中均匀分配。 

输出是在此约束下的最大可能客人数量。 由于库兹亚本人也参加，所以总人数比我们输出的人数多了一位。 

最多 100,000 个蛋糕尺寸最大为 10^9 的面包店的约束立即排除了任何尝试所有可能的组大小并以简单的方式单独检查它们的解决方案。 在最坏的情况下，对每个候选者进行直接整除性检查将导致大约 10^14 次操作，这远远超出了可行的限制。 

当数字没有有意义的共同结构时，就会出现常见的微妙失败情况。 例如，如果面包店生产 2、3 和 5 个切片，则无法选择大于 0 个客人的组大小，因为没有大于 1 的整数可以整除所有切片。 另一个棘手的情况是当所有值都相同时，答案只是该值减一，因为整个蛋糕大小本身就是限制因素。 

## 方法

 蛮力方法是尝试聚会中所有可能的人数，并验证该人数是否能整除所有蛋糕大小。 具体来说，我们将迭代候选组大小，从 1 到数组中的最小值，并针对每个候选检查所有面包店的整除性。 这是正确的，因为它直接强制执行这样的条件：每个蛋糕都可以在选定的人数中平均分配。 

然而，这种方法太慢了。 在最坏的情况下，我们的值可能高达 10^9 和 100,000 个面包店。 即使将候选限制为最小元素，我们仍然面临最多 10^9 次检查，每次检查扫描 100,000 个元素，导致大约 10^14 次操作。 

关键的观察结果是“团体规模适用于所有面包店”这一条件意味着人数必须除以每个 a_i。 这正是整个数组的公约数的定义。 我们不需要测试所有候选者，而只需要整除所有元素的最大整数，即最大公约数。 

一旦我们计算出所有值的 gcd，任何有效的组大小都必须除以这个 gcd，并且最大可能的选择就是 gcd 本身。 由于 Kuzya 本人也被计算在内，因此客人人数比团体人数少一位。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| --- | --- |
 | 蛮力 | O(n·min(a_i)) | O(n·min(a_i)) | O(1) | O(1) | 太慢了|
 | 最优（GCD）| O(n log A) | O(n log A) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为找到所有蛋糕尺寸的最大公约数。 

1. 首先以第一家面包店的蛋糕尺寸作为初始候选 gcd 值。 
2. 迭代所有剩余的面包店值，用每个新数字更新 gcd。

每次更新都会用 gcd(gcd, a_i) 替换当前的 gcd，缩小或保持不变。 
3.处理完所有值后，最终的gcd代表可以均匀分布在所有面包店的最大蛋糕片数。 
4. 将其减一转换为客人数量，因为 gcd 代表总人数（客人加上 Kuzya）。 

我们减一的原因是结构性的：如果蛋糕可以分成 d 等份，那么聚会中正好有 d 个人，所以客人是 d 减一。 

### 为什么它有效

 在每一步中，运行的 gcd 都是迄今为止除以所有已处理元素的最大整数。 当我们包含一个新元素时，使用 gcd 更新会准确保留公约数集。 这个不变量确保在最后一个元素之后，结果是整除每个 a_i 的最大数字，因此不存在更大的有效组大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

n = int(input())
a = list(map(int, input().split()))

g = a[0]
for x in a[1:]:
    g = gcd(g, x)

print(g - 1)
```该解决方案读取所有面包店蛋糕尺寸并维护正在运行的 gcd。 这`math.gcd`函数在每次调用的对数时间内有效地计算最大公约数，考虑到界限很大，这一点很重要。 

最后减一直接反映了gcd统计的是总参与者，而不仅仅是嘉宾。 

一个常见的实现陷阱是忘记用第一个元素初始化 gcd，而是从零开始。 虽然在数学上 gcd(0, x) = x，这是安全的，但不太明确； 初始化为`a[0]`避免混淆并更清晰地匹配不变的描述。

import sys, io
from math import gcd

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))

    g = a[0]
    for x in a[1:]:
        g = gcd(g, x)

 ## 边缘情况

 当只有一家面包店时，gcd 只是该值，因此答案变为 a_i - 1。该算法自然地处理此问题，因为跳过了剩余元素的循环，使初始值保持不变。 

对于像 7 这样的输入，运行的 gcd 是 7，因此输出变为 6，这意味着六个客人加上 Kuzya。 

当所有值都相等时，例如 10 10 10，gcd 始终保持 10。 这意味着聚会总共可以有 10 个人，即 9 位客人。 该算法保留了这一点，因为 gcd(a, a) 始终是 a。 

当数组包含1时，gcd立即变为1，并且没有进一步的更新可以改变它。 这正确地将答案强制为零客人，因为不能形成超过一个人的团体。
