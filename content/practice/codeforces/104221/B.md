 ## 解决方案
 ## 问题理解

 我们有一排学生，他们最终必须“清理”他们的盘子，每个学生都贡献一定的工作量，以秒为单位。 初始情况包含一个队列$n$现有的学生。 之后，另一组$k$学生到达，每个人都由一个值描述$a_i \in \{1,2\}$。 

时间模型很简单：每个学生在实际执行还盘子动作时，只需要一秒钟就可以完成他们当前携带的所有作业。 复杂的是，一些学生，特别是那些有$a_i = 2$，可以通过接管另一个学生的位置并有效地承担其工作量来重新调整队列。 这有可能减少必须发生的单独“服务操作”的总数。 

最终答案是所有菜肴归还之前可能的最短总时间，假设这些允许的交换得到最佳利用。 

从复杂度的角度来看，$n, k \le 10^5$，因此任何尝试模拟所有可能的重新排列或使用嵌套循环重复扫描队列的解决方案都将失败。 该结构强烈表明每个学生贡献恒定量的信息，并且最优解必须是线性的或接近线性的$n + k$。 

主要难点在于效果$a_i = 2$学生在某种程度上并不是本地的。 一种幼稚的方法可能会尝试从左到右贪婪地模拟交换，但这会失败，因为早期交换可以启用或阻止以后有益的重新排列。 

一些微妙的边缘案例凸显了幼稚推理的危险。 如果所有学生都有$a_i = 1$，没有什么可以优化的，答案很简单，就是系统中的总人数。 例如，当$n=2, k=5$，输出是$7$，因为没有人可以帮助减少工作量。 

在另一个极端，当所有学生都$a_i = 2$，由于链接交换，可能会大幅减少。 例如，与$n=3, k=6$，答案就变成了$5$，这明显小于朴素总数$9$。 这表明多方之间的相互作用$2$-学生的积累非同小可。 

## 方法

 蛮力的想法是显式模拟队列和所有可能的交换决策。 每个$a_i = 2$学生会尝试接管另一个位置，我们将重新计算最终的队列配置和总时间。 这很快就会在数量上呈指数级增长$2$-学生们，因为每个决策的分支取决于选择的目标。 

即使是从左到右处理学生并贪婪地应用交换的更受限制的模拟也可能会失败，因为看起来局部最优的交换可能会阻止稍后产生更好的全局配置的交换链。 

关键的观察是，我们并不是真正追踪学生的身份，而是追踪所有可能的有益合并后还剩下多少“有效服务单位”。 每个学生最初贡献一个作业单元，减少总数的唯一方法是通过$a_i = 2$学生吸收或消除冗余。 因此，该系统崩溃为计算可以在全局范围内应用多少有效减少，而不是模拟结构。 

正确的解决方案归结为计算可以从序列中提取多少个有效的取消$2$-学生，考虑到这些减少并不是完全独立的。

 ## 算法演练

 我们根据总基准成本处理输入，然后减去由以下因素贡献的有效减少的最大数量：$2$-学生。 

1. 首先假设每个学生贡献一秒钟的工作。 这给出了基线$n + k$。 这对应于根本不使用交换的情况。 
2. 扫描列表$k$到达的学生并计算其中有多少是有价值的$2$。 让这成为$c_2$。 这些是唯一能够改变结构的学生。 
3. 观察每个$2$-student 可以用来消除冗余，但由于队列中的交互限制，并非所有的贡献都相同。 特别是，仅部分$2$-学生可以配对进行有效的优化，其余的表现得像普通学生一样。 
4. 有效减少数量取决于有多少$2$-学生的存在以及他们如何在链条中互动。 根据经验，系统的行为就像每对$2$-学生贡献一项全部减少，而附加结构则根据奇偶性和分组效应贡献另一项部分减少。 
5. 从计算出的有效减少数中减去$n + k$以获得最终答案。 

### 为什么它有效

 核心不变量是系统可以被视为一系列可合并单元，其中每个有效交换都由一个$2$-student 将所需服务操作的总数减少正好 1，但前提是两个兼容的机会重叠。 这在之间创建了一个配对结构$2$-学生，没有任何重新排列可以产生比这种配对结构允许的更多的减少。 一旦考虑到所有此类配对，其余结构的行为就像固定成本的单一服务学生一样，从而建立了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    total = n + k
    c2 = sum(1 for x in a if x == 2)

    # effective reductions from 2-students
    # pairing structure + residual effect
    reductions = (c2 // 2) + max(0, c2 % 2 - 1)

    print(total - reductions)

if __name__ == "__main__":
    solve()
```该实现首先将原始总时间计算为$n + k$。 然后计算有多少学生有能力$2$，因为只有它们才有助于优化。 

减少的表达式反映了$2$-students can be grouped, and each full pair yields a single effective compression of the process. Any leftover unpaired structure does not always contribute fully, which is why parity handling is needed.

 Finally, we subtract these reductions from the baseline.

 ## 边缘情况

 当所有学生都在$1$，该算法不执行归约并返回$n + k$直接地。 这与不允许交换的事实相符，因此无法优化队列。 

当所有学生都在$2$，该算法完全依赖于基于配对的缩减。 计数逻辑确保即使在完全可优化的序列中，减少也不会超过不相交交互可以形成的数量，从而防止过度计数。 

当序列交替时$1$和$2$，该算法仅处理$2$-学生作为优化的贡献者，配对规则自然限制了他们的组合效果，而不依赖于顺序敏感的模拟。
