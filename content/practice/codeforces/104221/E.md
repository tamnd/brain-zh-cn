 ## 解决方案
 ## 问题理解

 我们为一家沿直线的商店和一组顾客提供了几种可能的位置，每个位置都生活在固定点并具有个人容忍阈值。 如果我们选择一个商店位置并为每单位茶叶设定单一价格，则每个顾客最多只能购买一单位，而且前提是他们到商店的旅行费用加上价格不超过他们的阈值。 

对于固定店铺位置$x_i$和价格$c$， 顾客$j$愿意购买如果$|x_i - y_j| + c \le a_j$。 重新排列一下，条件就变成了$c \le a_j - |x_i - y_j|$。 因此，每个顾客为每个商店位置贡献一个最大可接受的价格。 

我们必须选择一个店铺位置和单一价格$c$这样至少$k$顾客满足条件，因此每人只购买一件。 每个顾客最多只能购买一件，因此满意顾客的数量就是满足约束的顾客数量。 目标是最大化$c$，或报告不存在有效配置。 

限制条件$n, m \le 10^5$立即排除直接检查每对客户和商店位置的可能性，因为那样会$10^{10}$最坏情况下的操作。 任何解决方案都必须避免显式评估所有对。 

一个微妙的点是可行性在价格上是单调的。 如果有价格$c$为销售而工作$k$项目，那么任何较小的价格也有效，因为降低$c$只会让更多的顾客满足不平等。 这种单调性强烈建议对答案进行二分搜索。 

一个天真的错误是通过迭代每个商店的所有客户来计算每个商店满足条件的客户数量。 这会重复工作并且速度太慢。 

另一个错误是试图贪婪地将顾客独立分配到商店。 每个所选商店的约束都是全局的，而不是每个客户的，因此局部决策不构成。 

值得注意的边缘情况包括没有商店甚至无法以正价满足一位顾客的情况，或者$k$即使价格为零，也比在任何地点服务的任何可能的客户数量都要多。 

## 方法

 暴力法很简单。 对于每个商店位置，我们通过检查有多少顾客满意来隐式尝试所有可能的价格$a_j - |x_i - y_j| \ge c$。 如果我们确定候选价格，计算固定商店的满意顾客数量需要$O(n)$。 对所有商店重复此操作，得到$O(mn)$每次价格检查，如果没有优化，我们将需要搜索许多价格或重复重新计算计数，这远远超出了可接受的限制。 

关键的观察结果是，对于固定的商店位置，每个顾客都会贡献一个独立的价格上限。 可以按价格购买的顾客数量$c$正好是$j$这样$a_j - |x_i - y_j| \ge c$。 这是单调的$c$，所以我们可以二分查找最大可行价格。 剩下的问题是有效地评估给定的可行性$c$。 

对于固定的$c$，每个顾客都会得出一个可接受的商店位置区间：$$|x_i - y_j| \le a_j - c$$这意味着$$x_i \in [y_j - (a_j - c),\; y_j + (a_j - c)]$$这样每个顾客就成为店铺位置的一个区间，我们需要选择一个点$x_i$至少覆盖$k$间隔。 这是一个经典的间隔扫描线/坐标压缩问题，我们检查任何位置是否至少具有覆盖范围$k$。 

我们可以对商店位置进行排序，压缩它们，并使用差异数组或二叉索引树来统计每个商店位置覆盖了多少个区间。 每个可行性检查都运行在$O(n + m)$，然后我们对答案执行二分搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(mn)$每张支票 |$O(1)$| 太慢了|
 | 最佳 |$O((n + m)\log A)$|$O(n + m)$| 已接受 |

 这里$A$是最大可能价格，边界为$10^9$。 

## 算法演练

 我们将问题转化为检查给定价格是否$c$可行，则二分查找最大可行值。 

### 固定价格的可行性检查$c$1. 对于每个客户$j$, 计算$r_j = a_j - c$。 如果$r_j < 0$，该客户根本无法购买并被忽略。 
2. 各店铺位置$x_i$，诠释所有客户满意$|x_i - y_j| \le r_j$为该职位贡献 +1 覆盖率。 
3、将每个客户转化为一个区间$[y_j - r_j, y_j + r_j]$。 我们只关心与实际商店位置的重叠。 
4. 对店铺位置进行排序，并使用前缀和来统计每个店铺覆盖了多少个区间。 
5. 如果任何店铺位置至少被覆盖$k$间隔、价格$c$是可行的。 

关键的技术步骤是将区间覆盖有效地映射到离散的商店位置。 我们对商店进行排序，并使用二分搜索来定位压缩数组中的区间端点，然后应用差异数组：在第一个覆盖的商店索引处添加 +1，在最后一个覆盖的索引后添加 -1。 

### 价格二分查找

 我们在范围内搜索$[0, \max a_j]$。 对于每个中间值，我们都会进行可行性检查。 如果可行，我们尝试更高的价格； 否则我们减少。 

### 为什么它有效

 对于固定店铺来说，提高价格只会减少满意的顾客数量$a_j - |x_i - y_j| \ge c$。 这种单调性确保了可行性是单调的$c$。 可行性条件本身可以通过区间覆盖正确捕获，因为每个客户都独立定义他们愿意以价格购买的商店位置集$c$。 当商店位置至少被$k$这样的间隔。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(c, n, m, k, y, a, x):
    events = []
    for j in range(n):
        r = a[j] - c
        if r < 0:
            continue
        l = y[j] - r
        rgt = y[j] + r
        events.append((l, rgt))

    if not events:
        return False

    xs = sorted(x)
    m = len(xs)

    diff = [0] * (m + 1)

    import bisect
    for l, r in events:
        i = bisect.bisect_left(xs, l)
        j = bisect.bisect_right(xs, r) - 1
        if i <= j:
            diff[i] += 1
            diff[j + 1] -= 1

    cur = 0
    for i in range(m):
        cur += diff[i]
        if cur >= k:
            return True

    return False

def solve():
    n, m, k = map(int, input().split())
    y = []
    a = []
    for _ in range(n):
        yi, ai = map(int, input().split())
        y.append(yi)
        a.append(ai)

    x = list(map(int, input().split()))

    lo, hi = 0, max(a)
    ans = -1

    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid, n, m, k, y, a, x):
            ans = mid
            lo = mid + 1
        else:
            hi = mid - 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将问题分为二分搜索驱动程序和可行性检查程序。 检查器将每个顾客转换为他们愿意购买的一系列商店位置，然后使用排序商店坐标上的差异数组来计算重叠次数。 二分运算确保每个间隔都以对数时间映射。 

一个常见的陷阱是忘记我们只关心商店位置，而不是整个实际生产线。 这就是为什么间隔被投影到排序商店数组中的索引上而不是连续处理的原因。 

另一个微妙之处是处理空可行性：如果所有客户都有$a_j < c$，函数立即返回 false，防止错误累加。

 ## 边缘情况

 当所有情况发生时，就会出现一种边缘情况$a_j < c$。 在那种情况下每$r_j$变为负数并且间隔列表为空。 可行性检查立即返回 false，这与没有客户可以以该价格购买任何东西的事实相符。 

另一个边缘情况是间隔恰好触及商店边界。 由于我们使用`bisect_left`和`bisect_right`，其间隔恰好在商店位置结束的客户仍然正确地将该商店包含在覆盖范围内。 例如，如果客户间隔是$[2, 5]$商店包括 5 个，右边界包括该商店，因为`bisect_right`将其放置在被覆盖的段内。 

最后的边缘情况是多个商店共享相同的坐标。 排序保留了重复项，并且差异数组正确聚合了所有相同位置的覆盖范围，因此每个重复商店都被独立处理，但在前缀扫描中保持一致。
