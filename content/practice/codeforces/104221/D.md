 ## 问题理解

 我们得到一个无向简单图$n$交叉路口和$m$道路。 每条道路连接两个不同的交叉路口，并且任意一对交叉路口之间最多有一条道路。 任务是为每条道路分配一个方向，以便在定向后，不可能从某个交叉路口开始并在一个循环中永远沿着有方向的道路行驶。 

用图的术语来说，我们必须将无向图转换为没有有向环的有向图。 我们可以自由地独立选择每条边的方向，但最终的有向结构必须是非循环的。 如果存在多个有效方向，则可以打印其中任何一个。 

约束条件很大：最多$2 \cdot 10^5$顶点和边。 这立即排除了任何二次方，例如尝试所有顶点排列或重复模拟循环形成。 我们需要一个$O(n + m)$或者$O(m \log n)$建造。 

一个微妙的点是输入图本身可能包含循环，甚至可能包含许多循环。 “我们必须首先打破循环”的天真假设会导致不必要的复杂性。 真正的任务不是删除边缘或修改结构，而是纯粹指定方向。 

典型的失败案例来自于在没有全局规则的情况下尝试局部贪婪方向。 例如，在一个三角形中$1-2-3-1$，如果我们定向$1 \to 2$,$2 \to 3$， 和$3 \to 1$，我们立即创建一个有向循环。 如果不强制执行全局排序，仅基于局部一致性的贪心规则很容易陷入此类矛盾。 

另一个陷阱是假设我们需要首先检测无向图中的循环。 这是无关紧要的，因为即使是充满循环的图仍然可以定向为 DAG。 

## 方法

 蛮力思维方式是尝试将边逐一定向，并检查每次分配后是否出现有向循环。 每次检查都需要在有向图中进行循环检测，即$O(n + m)$，我们就这么做了$m$次，导致$O(m(n + m))$。 和$m$最多$2 \cdot 10^5$，这太慢了。 

关键的结构见解是，当存在顶点的全局排序使得每个边都按该顺序指向前方时，有向图是非循环的。 这将问题从“动态避免循环”转换为“选择一致的顶点排序”。 

一旦我们承认我们只需要一个全序，那么构建就变得微不足道了。 我们为顶点分配严格的顺序，例如它们的标签$1 < 2 < \dots < n$。 然后对于每个无向边$(u, v)$，我们将其从较小的标签定向到较大的标签。 任何有向循环都意味着严格增加的标签序列最终返回到起点，这是不可能的。 

无论原始图有多密集或多循环，这都有效，因为方向忽略结构并且仅依赖于全局排序。

 ## 算法演练

 我们通过强制执行一致的顶点排序来构造有向无环图。 

1. 为每个顶点分配一个固定的等级，该等级等于其索引$1$到$n$。 这对所有顶点创建了严格的总顺序。 引入这种顺序的原因是任何非循环方向都必须与某种顺序兼容，因此我们直接构建一个。 
2. 迭代输入中给出的所有边。 对于每条边$(u, v)$，比较排名$u$和$v$。 
3.如果$u < v$，将边缘定向为$u \to v$。 否则将其定向为$v \to u$。 这确保了每条边都从排名较小的顶点定向到排名较大的顶点。 
4. 按照读取的顺序输出所有定向边。 

### 为什么它有效

 假设定向后存在有向循环，这是矛盾的。 沿着这个循环，由于构造规则，每条边都必须从较小的索引到较大的索引。 这意味着当我们遍历循环时，顶点索引在每一步都会严格增加。 严格增加不能返回到起始顶点，因为这需要起始索引在循环中同时为最小和最大。 因此不存在有向循环，并且该图是 DAG。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m = map(int, input().split())
    edges = []
    for _ in range(m):
        u, v = map(int, input().split())
        if u < v:
            edges.append((u, v))
        else:
            edges.append((v, u))

    out = []
    for u, v in edges:
        out.append(f"{u} {v}")

    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现直接对排序思想进行编码。 每条边都会立即标准化，以便其方向遵循顶点的自然数字顺序。 不需要邻接表、DFS 或循环检测，因为构造本身保证了非循环性。 

一个常见的错误是存储边缘并随后不一致地决定方向。 重要的属性是相同的全局排序统一应用于每条边，没有例外。

 ## 边缘情况

 密集循环图（例如完整图）可以正确处理，因为每条边仍然根据相同的全局数字顺序进行定向。 例如，在完全连接的三元组中，每条边都与$1 < 2 < 3$，产生严格的层次结构而不是循环。 

单边图的工作原理很简单，因为比较会立即指定方向，而不依赖于其他边。 即使在这种最小情况下，所有边遵循相同顺序的不变性仍然有效。 

以任意顺序给出输入边的图不会影响正确性，因为方向独立于输入序列。 每条边都被单独处理，但始终遵循相同的全局规则。
