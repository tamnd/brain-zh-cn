 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中顶点 1 是根，其他每个顶点都有一个固定的父节点。 在这棵树上，我们考虑顶点的排列，这意味着从 1 到 n 的每个顶点标签按某种顺序恰好出现一次。 

如果每个位置 i 都被分配了一个与顶点 i 至少以三种方式之一“兼容”的值 p[i]，则该排列被称为有效。 或者该值等于 i 本身，或者该值对应于从 i 到根的路径上的顶点（因此它是 i 的祖先），或者该值位于 i 的子树中（因此它是 i 的后代）。 该约束在结构上是对称的，但在解释上是不对称的，因为祖先和后代关系取决于以 1 为根的树。 

任务是计算存在多少个这样的有效排列，以 998244353 为模。 

输入大小总体而言很小，所有测试用例的 n 总和最多为 5000。这立即排除了比每个测试用例大致二次更糟糕的情况，并强烈建议解决方案应基于树结构的动态规划或子树的组合分解。 即使 n 约为 20，任何排列的阶乘枚举也是不可能的，因此条件必须转换为关于来自不同子树的元素如何交互的结构化约束。 

一个微妙的边缘情况来自这样一个事实：祖先集和后代集沿着根路径严重重叠。 例如，在链树中，每对顶点都是可比较的，因此条件几乎变得空洞，并且每个排列都是有效的。 在另一个极端，在星形树中，除了通过根之外，大多数对都是无法比较的，这会产生更强的结构约束。 任何独立处理祖先和后代约束的幼稚尝试都会错误地计算排列，因为同一顶点可以满足不同位置的多个角色。 

## 方法

 一种强力方法会枚举所有顶点排列，并根据条件检查每个排列。 对于每个位置 i，我们将验证 p[i] 是否等于 i，或者 p[i] 是否位于 i 的祖先链或其子树中。 预先计算祖先和后代关系使得每次检查的时间复杂度为 O(1)，但每个测试用例生成排列已经是 O(n!)，即使 n 约为 12，这也是不可行的。 

关键的观察结果是，条件不依赖于绝对标签，而仅依赖于树中的结构关系。 对于固定顶点i，允许的值p[i]正好形成由其子树中的所有节点联合其祖先组成的集合，从树序角度来看，这是一个连通结构。 这表明分配受到排列如何尊重子树分解的限制。 

关键的结构洞察是自下而上地处理树，并计算如何通过合并子树的排列来形成排列。 每个子树的行为就像一个块，可以保持内部一致或允许通过祖先连接进行某些“交叉分配”。 因为每个节点仅沿着根路径向上连接，所以不同子子树之间的交互受到高度限制：子树仅通过其最低公共祖先耦合，这使得树 DP 的状态跟踪子树中有多少元素通过祖先边缘分配到其外部。 

这将全局排列问题简化为组合独立子树贡献，同时维护每个节点的较小状态空间，而不是跟踪完整排列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!·n) | O(n!·n) | O(n) | 太慢了|
 | 树DP合并子树| 每个测试用例的 O(n²) | O(n²) | 已接受 |

 ## 算法演练

我们将树的根设为 1 并按后序处理它。 对于每个节点 u，我们计算一个 DP 表，该表描述了在不同数量的“外部放置”下可以将排列分配给 u 的子树的多种方式，这意味着该子树中的元素被分配到子树外部的位置，但通过祖先关系仍然有效。 

我们将 dp[u][k] 定义为处理 u 子树的方法数量，使得 subtree(u) 中的 k 个元素不放置在 subtree(u) 内，而是用于 u 上方的祖先位置。 

我们用 dp[u][0] = 1 在叶子处初始化，因为单个节点只能映射到其自身并且没有可以与之交互的子树结构。 

对于内部节点 u，我们将子节点一一合并。 假设我们已经有了 u 的部分 DP，并且合并了一个子 v。我们通过决定 v 的子树中有多少元素向上导出来组合配置。 如果 v 导出 x 元素，则这些 x 元素必须容纳在相对于 u 的可用祖先位置中，并且其余 subtree(v) 元素保留在内部。 

这个合并步骤是组合的：我们选择每个子树导出多少元素，然后乘以二项式系数，该二项式系数表示我们如何在不同子树之间交错分配，同时尊重排列的唯一性。 树结构保证来自不同子元素的导出元素除了通过计数之外不会干扰，因为它们的祖先路径仅在 u 处相交。 

处理完所有子节点后，我们相应地更新 dp[u] 并继续向上。 从根本上来说，没有祖先，因此有效的配置必须具有零个导出元素。 最终答案是dp[1][0]。 

### 为什么它有效

 DP 不变量是 dp[u][k] 正确地计算了 subtree(u) 的所有有效部分排列，这些排列与祖先后代约束一致，并且正好有 k 个元素“向上推”到 u 的祖先。 树结构确保任何有效的全局排列都可以唯一地分解为独立的子树贡献以及导出元素沿祖先链放置的位置的选择。 由于祖先链不分支，因此除了最低共同祖先之外，不同子树之间不会发生冲突，而这正是 DP 状态所捕获的。 每个合并步骤都保持一致性，因为它只考虑子树元素保留在内部或沿着允许的祖先路径向上移动的有效位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        parents = list(map(int, input().split()))
        
        g = [[] for _ in range(n + 1)]
        for i, p in enumerate(parents, start=2):
            g[p].append(i)

        # dp[u] = list where dp[u][k] = ways with k exported nodes
        dp = [None] * (n + 1)

        def dfs(u):
            dp[u] = [0]
            dp[u][0] = 1
            size = 1

            for v in g[u]:
                dfs(v)

                ndp = [0] * (size + len(dp[v]))
                for i in range(len(dp[u])):
                    if dp[u][i] == 0:
                        continue
                    for j in range(len(dp[v])):
                        val = dp[v][j] * dp[u][i] % MOD
                        ndp[i + j] = (ndp[i + j] + val) % MOD
                dp[u] = ndp
                size += len(dp[v]) - 1

            return

        dfs(1)

        print(dp[1][0] % MOD)

if __name__ == "__main__":
    solve()
```该解决方案从父数组构建有根树并运行后序 DFS。 每个节点维护一个 DP 数组，该数组通过其子树中被视为“向上导出”的顶点数量进行索引。 合并步骤是来自子树的 DP 数组的卷积，累积所有可能的方式来跨子树分布导出的元素。 

最终答案是在导出元素为零的根处获取的，因为根没有祖先来吸收任何向上的分配。 

该实现依赖于重复的多项式卷积。 内部嵌套循环直接实现这一点，正确性取决于组合两个子树的状态时正确对齐索引 i + j。 大小簿记确保数组保持足够大以表示所有可能的导出计数。 

## 工作示例

 ### 示例 1

 考虑一个简单的链：1 → 2 → 3。 

| 节点| dp 状态（k 导出）|
 | --- | --- |
 | 1 | [1] |
 | 2 | [1] |
 | 3 | [1] |

 每个节点没有有意义的分支，因此不会强制导出。 每个子树都会简单地合并。 

这表明DP并没有在线性结构中引入人为的限制，因为祖先-后代关系已经允许充分的灵活性。 

### 示例 2

 考虑一颗星星：1 和孩子 2、3。 

| 步骤| 处理节点 | dp[1] 状态 |
 | --- | --- | --- |
 | 初始化| 1 | [1] |
 | 添加 2 | 合并叶| [1, 0] |
 | 添加 3 | 合并叶| [1, 0, 0] |

 合并两个子项后，只有根目录下没有导出的配置才会起作用。 

这演示了组合独立子树时导出如何累积以及为什么根约束强制过滤到 k = 0。

 ## 边缘情况

 对于二节点树，顶点 1 是 2 的父节点。DP 以 dp[1] = [1]、dp[2] = [1] 开始，并且合并不会产生额外的状态。 根约束强制 dp[1][0] = 1，与只有身份排列同时满足两个节点的祖先-后代一致性的事实相匹配。 

对于星形树来说，所有叶子都是根下的独立子树。 每个叶子贡献一个微不足道的 DP，合并表明任何交叉分配都需要无法在根处吸收的导出，因此只有平衡的内部配置仍然有效。 该算法自然地通过导出计数约束过滤无效排列，最终的 dp[1][0] 准确捕获有效的全局分配。
