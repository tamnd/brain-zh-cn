 ## 解决方案
 ## 问题理解

 我们得到一个加权网格，其中每个单元格都有一个非负的难度值。 相邻单元格允许在四个方向上移动。 对于每个查询，我们都会得到一个起始位置和一个最多包含 100 个单元格的轴对齐小矩形。 任务是从给定的起始单元格开始在网格上移动，并确保该矩形内的每个单元格至少被访问一次。 

路线成本的定义方式有点不寻常：它是整个步行过程中访问的所有单元格中最大的单元格难度。 由于允许重新访问单元格并且顺序是任意的，因此唯一重要的是哪些单元格被踩过，而不是路径结构本身。 

因此，每个查询都要求最小的可能值 X，以便存在从给定单元格开始的步行，可以到达所有所需的矩形单元格，同时不会踩到难度大于 X 的任何单元格。 

网格大小可以大到 500 x 500，最多可以有 100000 个查询。 关键限制是每个查询的矩形最多包含 100 个单元格，这是问题易于处理的唯一原因。 

一个天真的想法是独立处理每个查询并从一开始就运行最短路径或洪水填充，但即使是在 500 x 500 网格上进行一次这样的搜索也已经很昂贵，并且执行 100000 次远远超出了可行的限制。 

当起始单元本身不在矩形内部时，会出现更微妙的边缘情况。 在这种情况下，路径必须首先通过潜在的高价值地形到达矩形。 例如，如果起点位于低值区域，但所有连接走廊都经过单个非常高的像元，则即使矩形本身很便宜，答案也必须至少为该瓶颈值。 

另一种失败模式来自于假设答案仅取决于矩形内的最大值。 这是错误的，因为起点和矩形之间的连接可能需要走出它。 

## 方法

 蛮力解释很简单。 对于每个查询，我们可以假设一个阈值 X，将自己限制为值最多为 X 的所有单元格，并检查起始点是否可以到达该导出图中的每个矩形单元格。 如果是，则X是可行的。 我们可以二分搜索 X 并每次重复一次 BFS 或 DFS。 

这在逻辑上是可行的，但是太慢了。 每个 BFS 的时间复杂度为 O(nm)，并且具有多达 100000 个查询和大约 30 个二分搜索步骤，这变得完全不可行。 

关键的结构观察是，对于任何固定阈值 X，我们只关心由值不超过 X 的单元组成的子图中的连通性。在该子图中，每个查询询问一小组节点（起始点加上最多 100 个矩形单元）是否位于与起始点相同的连通分量中。 

这表明要扭转观点。 我们可以随着阈值的增加建立连接，而不是检查猜测的阈值的连接。 如果我们按难度对所有单元进行排序并逐渐激活它们，则仅当添加新单元时连接才会发生变化。 这正是不断增长的图上的联合查找场景。 

一旦构建了这个结构，每个查询就变成了寻找最小激活点的问题，其中所有必需的节点都位于与开始相同的组件中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 二分搜索 + 每个查询的 BFS | O(q log V · nm) | O(q log V · nm) | O(纳米) | 太慢了|
 | 已排序单元格的离线 DSU | O(nm log nm + q·k log V) | O(nm log nm + q·k log V) | O(纳米) | 已接受 |

 ## 算法演练

 我们将网格转换为图，其中每个单元格都是一个节点，边连接相邻的单元格。 每个节点的权重等于其难度。

我们按照难度递增的顺序处理节点，使用并查找结构来维护已激活节点的连接组件。 

对于每个查询，我们跟踪由起始单元格及其矩形中的所有单元格组成的集合。 

1. 按难度升序对所有网格单元进行排序。 
2. 初始化一个并查找结构，其中还没有任何单元处于活动状态。 
3. 维持每个单元的激活状态，最初全部处于非活动状态。 
4. 浏览已排序的单元格列表。 处理单元时，将其标记为活动并将其与任何已经活动的邻居合并。 这逐渐构建了与阈值水平相对应的组件。 
5. 对于每个查询，我们从概念上问：在此激活顺序中的哪个点，所有必需的单元都连接到起始单元？ 
6. 为了有效地回答这个问题，我们对排序的激活顺序执行二分搜索。 对于中点位置，我们激活所有单元格直至该难度，并检查所有查询单元格是否与开始共享相同的并查根。 
7. 每个查询的答案是满足该条件的最小阈值。 

昂贵的部分是检查，它验证最多 101 个节点是否属于同一个连接组件。 由于矩形很小，因此每次查询此检查都足够便宜。 

### 为什么它有效

 在任何固定阈值 X 处，激活的单元正是那些难度最大为 X 的单元。这些单元上的并查找正好代表了阈值限制图的连接结构。 当且仅当所有必需的节点位于与该结构中的起点相同的连接组件中时，查询才是可行的。 由于组件仅随着 X 的增加而增长，因此 X 中的可行性是单调的，这保证了二分搜索返回最小的有效阈值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1

def solve():
    n, m, q = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(n)]

    cells = []
    for i in range(n):
        for j in range(m):
            cells.append((grid[i][j], i, j))
    cells.sort()

    idx = lambda x, y: x * m + y

    dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

    pos = { (i, j): idx(i, j) for i in range(n) for j in range(m) }

    queries = []
    for _ in range(q):
        r, c, r1, c1, r2, c2 = map(int, input().split())
        r -= 1
        c -= 1
        r1 -= 1
        c1 -= 1
        r2 -= 1
        c2 -= 1

        nodes = [idx(r, c)]
        for i in range(r1, r2 + 1):
            for j in range(c1, c2 + 1):
                nodes.append(idx(i, j))

        queries.append(nodes)

    def check(limit):
        dsu = DSU(n * m)
        active = [False] * (n * m)

        for k in range(limit + 1):
            _, i, j = cells[k]
            v = idx(i, j)
            active[v] = True
            for di, dj in dirs:
                ni, nj = i + di, j + dj
                if 0 <= ni < n and 0 <= nj < m:
                    u = idx(ni, nj)
                    if active[u]:
                        dsu.union(v, u)

        res = []
        for nodes in queries:
            root = dsu.find(nodes[0])
            ok = True
            for v in nodes:
                if dsu.find(v) != root:
                    ok = False
                    break
            res.append(ok)
        return res

    lo = 0
    hi = n * m - 1
    ans = [0] * q

    # binary search per query
    for i in range(q):
        l, r = 0, n * m - 1
        while l < r:
            mid = (l + r) // 2
            if check(mid)[i]:
                r = mid
            else:
                l = mid + 1
        ans[i] = cells[l][0]

    print("\n".join(map(str, ans)))

if __name__ == "__main__":
    solve()
```该解决方案按难度构建所有单元格的全局排序，并将其用作阈值的代理。 检查功能内的 DSU 重建此排序的给定前缀的连接性。 

二分搜索隔离最小的前缀，其中起始和所有必需的矩形单元属于单个连接的组件。 返回的值是该前缀中最后一个激活单元的难度，它恰好对应于最小可行的最大难度。 

一个微妙的实现细节是索引：展平网格可确保 DSU 操作是 O(1) 数组访问。 另一个重要的一点是，每个查询都会将矩形扩展为显式节点列表，这是可以接受的，因为每个节点最多包含 100 个单元。

 ## 边缘情况

 当起始单元位于矩形之外时，算法会正确考虑连接路径，因为并查找连接是在整个网格上构建的，而不仅仅是目标区域。 即使矩形单元是内部连接的，它们仍然无法到达，直到激活阈值包括至少一个来自起始组件的连接走廊。 

当所有矩形单元都被高值墙隔离时，只有当这些墙被激活时才会出现连通性。 二分搜索自然会识别最小的此类墙值，因为 DSU 组件恰好在此时合并。
