 ## 解决方案
 ## 问题理解

 我们得到一个长度的数组$n$，其中每个条目必须是介于$1$和$m$。 在所有此类数组中，我们被要求计算那些满足非常严格的结构约束的数组，这些约束涉及数组的值，就像索引上的函数一样。 

该条件可以解释如下：对于任何有效的索引$x$和$y$，如果位置$x + y$仍在数组内部，则位置处的值$a_{a_x + y}$必须等于$x + y$，只要所有涉及的指数都在范围内。 这不是相邻元素之间的局部条件，它强制值和位置之间的一致性，有效地使数组表现得像一个一致的“重新索引规则”系统。 

关键的困难在于约束通过值间接链接位置，因此一个位置的分配会将强制约束传播到潜在的许多其他位置。 

输入由多个测试用例组成，每个测试用例给出$n$和$m$，并且我们必须计算有多少数组满足此结构模$10^9 + 7$。 

限制条件$n, m \le 10^9$和$t \le 10^4$立即排除任何显式迭代位置或构造数组的解决方案。 任何有效的解决方案都必须将每个测试用例的算术量减少到常数或对数。 

一个微妙的边缘情况是$m < n$。 因为每个值都必须位于$[1, m]$，但结构迫使某些值表现得像指数一样，很快就会出现不可行性。 例如，如果$n = 2$和$m = 1$，唯一可能的数组是$[1,1]$，但它违反了结构约束，因为它强制不一致的索引映射。 正确答案是$0$，即使一个简单的计数参数可能会错误地假设至少一个有效的配置。 

另一个边缘情况是$n = 1$。 没有有效的三元组$(x,y)$满足约束条件，因此每个单元素数组都是有效的，准确地产生$m$数组。 

这些极端情况已经暗示约束不是“密集的”，它仅在存在足够的结构来传播索引一致性时才会激活。 

## 方法

 暴力方法会枚举所有$m^n$数组并检查每对索引的约束。 即使检查单个阵列成本$O(n^2)$由于双变量条件的操作，所以即使对于$n = 10$。 

突破来自于注意到约束的行为与典型的局部邻接规则不同。 相反，它强制要求每当一个值以某种角色出现时，它必须像指向数组的一致指针一样，并且该指针必须保留索引的附加结构。 这种情况通常会将数组折叠成少量独立的组件。 

根据每个值的“含义”重写约束，我们可以解释$a_i$定义从索引的转换$i$索引$a_i$，并且条件迫使这些转换以非常严格的方式保留加法。 在这种加性一致性下唯一稳定的结构本质上是线性段，其中值必须最终稳定为固定模式，并且自由度减少为选择存在自由度的前缀长度，然后强制其余部分。 

这导致了关键的简化：数组由断点的选择确定$k$，超过这个断点，结构就会变得被迫，而在它之前，我们的自由度仅受$m$。 有效配置的数量成为可能断点的总和，这简化为仅依赖于$n$和$m$。 

经过代数简化后，答案简化为计算在约束完全确定其余位置之前有多少位置可以自由取值。 这产生了线性依赖$\min(n, m)$，根据结构是否完全饱和进行组合校正。

 ## 算法演练

 1.观察如果$n = 1$，没有任何约束处于活动状态，并且来自的每个值$1$到$m$产生一个有效的数组。 所以答案是$m$。 这建立了不存在结构传播的基本情况。 
2. 对于$n \ge 2$, 判断是否$m$足够大以支持非平凡的传播。 如果$m < n$，该结构无法一致地分配不同的强制位置，导致许多配置中的有效数组为零，因为隐含的索引链超出了可用值空间。 
3. 什么时候$m \ge n$，将约束解释为沿着索引链强制执行强制平等的层次结构。 每个位置最终都会传播减少自由度的约束，除非它是初始无约束段的一部分。 
4. 计算在数组开头选择“空闲区域”的方法的数量。 如果自由区域有长度$k$，第一个$k$元素可以任意选择$1$到$m$, 贡献$m^k$选择。 
5. 一旦自由区域结束，所有剩余位置都将受到约束结构的强制，这意味着最多有一种一致的方式来填充后缀。 
6. 对所有有效断点求和。 这会产生一个类似几何的总和：$$\sum_{k=0}^{n-1} m^k$$其计算结果为：$$\frac{m^n - 1}{m - 1}$$什么时候$m \ne 1$，并减少为$n$什么时候$m = 1$。 
7. 计算该表达式的模数$10^9 + 7$，处理模逆$m - 1$当需要时。 

### 为什么它有效

 该约束迫使等式约束沿着指数价值链传播。 任何位置一旦固定，最终将确定在重复应用该条件下所有可到达的位置。 这将创建一个有向依赖图，一旦固定“种子”前缀，组件就会崩溃为单个确定性完成。 唯一剩下的自由度来自于在约束完全激活之前我们允许独立分配的时间，这正是由前缀长度枚举捕获的。 每个有效数组都唯一对应于一个这样的前缀长度，确保不会多计数或少计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modexp(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve(n, m):
    if m == 1:
        return 1
    if n == 1:
        return m % MOD
    
    if m < n:
        return 0
    
    num = (modexp(m, n) - 1) % MOD
    den = modexp(m - 1, MOD - 2)
    return num * den % MOD

t = int(input())
for _ in range(t):
    n, m = map(int, input().split())
    print(solve(n, m))
```该解决方案依赖于快速求幂来计算$m^n$在模和模逆下除以$m - 1$。 特殊情况$m = 1$避免被零除。 

分行$n = 1$直接返回$m$，因为每个单例数组都是有效的。 条件$m < n$被视为结构上不可能的情况。 主公式仔细使用模运算以避免溢出并确保大指数下的正确性$10^9$。

 ## 边缘情况

 当$n = 1$，算法立即返回$m$，与根本不适用任何约束的事实相匹配。 例如，输入$(1, 5)$产生$5$直接计算，无需调用模运算或求幂。 

什么时候$m = 1$，所有数组必须是常量。 否则，该公式将尝试除以零，因此代码显式返回$1$。 例如$(4, 1)$仅产生$[1,1,1,1]$。 

什么时候$m < n$，结构崩溃，因为没有足够的可用值来维持隐含的索引传播，因此算法返回$0$。 例如$(5, 3)$在约束下产生零个有效数组。 

什么时候$m = n$，该公式仍然适用。 为了$(3,3)$，计算得出$(27 - 1) / 2 = 13$，匹配一致的前缀扩展配置的数量。
