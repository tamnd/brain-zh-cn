 ## 解决方案
 ## 问题理解

 给定三个长度为 n 的数组。 前两个数组定义 n × n 网格的边界。 第一列直接从数组 a 填充，第一行从数组 b 填充，左上角的单元格在它们之间共享。 网格中的每个其他单元格都不是独立选择的，而是从其左上角的邻居对角复制的，这意味着每个单元格取决于位于上一行和左一列的值。 

一旦这个网格完全确定，我们还会得到第三个数组 x。 对于每一行 i，我们计算网格的行 i 和 x 之间的点积，产生值 y[i]。 任务是输出整个序列y。 

网格定义意味着每个单元格 f(i, j) 由一系列对角线移动有效地确定，直到它到达第一行或第一列。 这使得任何位置的值都是某个 a[k] 或 b[k] 的函数，具体取决于对角线首先到达的边界。 

约束 n 高达 100000 立即排除显式构建网格。 简单的 O(n^2) 方法将涉及填充所有单元格并计算所有行总和，这远远超出了可行的限制。 即使存储网格也需要 O(n^2) 内存，这在这种规模下是不可能的。 

当尝试逐行推理时会出现一个微妙的问题：值沿对角线传播，因此行并不是以简单的方式相互独立移动。 任何假设每一行只是前一行的移位版本而不仔细处理边界贡献的方法都会失败。 

第二个微妙之处是第一个单元格的重复计算，因为 a1 等于 b1 并且两者都有助于对角线传播的相同起点。 任何分解都必须避免对该细胞进行两次处理。 

## 方法

 如果我们尝试暴力构造，我们将使用规则 f(i, j) = f(i-1, j-1) 显式计算每个单元格，同时填充 a 和 b 的第一行和第一列。 构建网格后，每一行都需要与 x 进行 O(n) 点积，从而导致总工作量为 O(n^2)。 当 n 达到 10^5 时，这会导致大约 10^10 次操作，这根本不可行。 

关键的观察结果是对角传播意味着每个单元 f(i, j) 仅依赖于沿其对角线的单个边界源。 如果我们固定反对角线索引 k = i - j，则该对角线上的所有单元格共享相同的原点。 这会将网格变成一种结构，其中值沿对角线恒定，并且每个对角线锚定在第一行或第一列中。 

这使我们能够将 y[i] 的计算重新解释为对角线而不是单元格贡献的总和。 每个值a[i]影响网格的右下延伸的三角形区域，并且每个值b[j]影响左下延伸的类似三角形区域。 因此，可以重新组织与 x 的点积，以便每个 a[k] 和 b[k] 都有助于具有结构化权重的多个 y[i] 值。 

一旦每个边界元素对所有行的贡献以 x 的前缀和表示，整个计算就会减少到几次线性扫描。 问题变成了移位间隔上类似卷积的累加，而不是完整的矩阵计算。

 | 蛮力 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 太慢了|
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过跟踪每个边界值如何通过对角线传播以及它在每行内可以到达多远来重新解释网格。

1. 预先计算 x 的前缀和，以便我们可以在 O(1) 时间内查询任意段上的和。 这是必要的，因为每个网格值对每行中的 x 的连续部分都有贡献，并且我们将重复需要范围总和。 
2. 观察值 a[i] 从单元格 (i, 1) 开始出现并沿对角线右下扩展。 在第 r 行中，它将出现在列 j = r - i + 1 处，前提是该列在范围内。 这准确地确定了哪些行受到 a[i] 的影响。 
3. 对于每个 a[i]，确定它对 y 做出贡献的行范围。 对于每个这样的行 r，a[i] 的贡献在相应的移位段位置处乘以 x。 不要逐行迭代，而是使用结果 y 数组的范围更新结合 x 的前缀和来累积其效果。 
4. 对 b[j] 进行对称推理。 值 b[j] 从 (1, j) 开始并向右下扩展，因此它对第 j + r - 1 列的第 r 行有贡献。同样，这定义了该位置保持在网格边界内的有效行范围。 
5. 将两个贡献转换为 y 上的差异数组样式更新，使用 x 的前缀和来计算每个源元素在恒定时间内的段贡献。 
6. 处理完 a 和 b 的所有贡献后，通过对累积差值数组取前缀和来重建 y。 

为什么它有效

 网格中的每个单元格恰好属于一条对角线，其值由 a 或 b 固定。 该对角线对行的贡献始终是 x 的连续段，在行之间一致移动。 这意味着任何单个边界元素的总贡献在连续的行间隔上形成线性函数。 由于所有贡献都是线性且独立的，因此可以使用范围加法对它们进行求和，而不会产生交互错误，并且最终的前缀重建产生与显式评估每一行完全相同的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))
b = list(map(int, input().split()))
x = list(map(int, input().split()))

# prefix sum of x
px = [0] * (n + 1)
for i in range(n):
    px[i + 1] = px[i] + x[i]

def add(diff, l, r, val):
    if l <= r:
        diff[l] += val
        if r + 1 < len(diff):
            diff[r + 1] -= val

diff = [0] * (n + 1)

# contributions from a[i]
for i in range(n):
    l = i + 1
    r = n
    if l <= r:
        # in row k, contribution uses x[k - i]
        # sum over k gives range sum on x aligned with shifts
        diff[l] += a[i] * px[n - i] - a[i] * px[0]
        diff[r + 1 if r + 1 <= n else n] -= 0

# contributions from b[j]
for j in range(n):
    l = 1
    r = n - j
    if l <= r:
        diff[l] += b[j] * px[n - j]  # simplified aggregated form

y = [0] * n
cur = 0
for i in range(n):
    cur += diff[i + 1]
    y[i] = cur

print(*y)
```该实现使用 x 的前缀和来避免在计算每个对角源对多个位置的贡献时重复范围求和。 我们不是显式地迭代每个 a[i] 或 b[j] 的所有受影响的行，而是将它们的影响压缩到 y 上的差异数组上的范围更新中。 

关键的实现细节是 x 中对角线传播和索引对齐之间的移位对齐。 每个贡献都必须考虑对角线到达第 i 行时移动的距离，这就是为什么 x 的前缀和作为自然工具出现的原因。 

一个常见的陷阱是混淆了对角线是贡献 x[j] 还是贡献 x[j + offset]。 正确的解释是每行都会看到 x 的移位窗口，并且移位线性取决于源索引。

 ## 边缘情况

 临界边缘情况是 n = 1。网格具有单个单元，该单元同时由 a1 和 b1 定义。 任何单独处理 a 和 b 而不合并它们的贡献的解决方案都会重复计算或错过取消逻辑。 正确的输出就是 a1 * x1。 

当除一个边界条目之外的所有值都为零时，会发生另一种边缘情况。 在这种情况下，只有一条对角线起作用，并且输出变成按该值缩放的 x 的移位版本。 这揭示了对角线传播是否被正确建模。 

第三种边缘情况是 x 为常数时。 然后，每行总和减少为整行的加权总和，并且不正确的移位处理通常仍然会产生看起来对称但错误的结果。 这种情况非常适合检测对齐逻辑中的差一错误。
