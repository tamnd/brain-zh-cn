 ## 解决方案
 ## 问题理解

 我们给出了一系列放置在一条线上的整数坐标处的垂直极点，其中极点`i`坐在位置`x = i`并且有高度`h[i]`。 从左侧开始，光沿矢量描述的固定方向传播`(tx, ty)`和`tx > 0`，因此光线总是向右移动，同时可能向上或向下移动，具体取决于`ty`。 

每根杆子都可能阻挡其右侧所有物体的光线。 该问题没有考虑单个光线，而是定义了一个“阴影包络线”，它是通过将每个极点向左移动、从其顶部向给定方向发射光线并考虑每个 x 位置达到的最大高度而形成的上边界。 如果在到达时其顶部严格高于该封套，则认为杆被照亮。 如果它恰好位于信封上，则视为已阻塞。 

任务是计算有多少根灯杆被照亮。 

总量约束`n`跨测试用例的数量最多为 500,000，这立即排除了任何将每个极点与所有先前极点进行比较的二次方法。 解决方案必须以线性或接近线性的时间处理每个测试用例。 

微妙的困难在于，阴影包络线不仅仅是迄今为止的最大高度。 它是许多斜线的最大值，因此包络线的斜率会随着时间而变化。 任何仅跟踪最大高度或与单行进行比较的幼稚方法都会失败。 

一些边缘案例暴露了典型的错误。 如果所有高度都相等并且`ty = 0`，每条光线都是水平的，并且所有东西都完全重叠，因此只有第一个极点被照亮。 天真的“严格大于之前的最大高度”方法会错误地将更多杆标记为可见。 

如果`ty < 0`，光线向下倾斜，这意味着阴影的下降幅度较小，并且即使高度较小，后来的极点也可以被照亮。 任何假设单调增加包络线高度的方法都会在这里立即破裂。 

如果`ty > 0`，光线向上倾斜，因此较早的高杆占主导地位更长，并且可以在右侧产生阴影。 这是最难的情况，因为包络线变成了变相的凸上壳问题。 

## 方法

 蛮力的想法很简单：对于每个极点`i`，模拟所有先前的极点`j < i`，计算从`(j, h[j])`方向`(tx, ty)`, 评估其在位置处的高度`i`，并取最大值。 如果`h[i]`严格大于该最大值，该杆被照亮。 

每次评估都是恒定时间，因此每个`i`成本`O(n)`工作、给予`O(n^2)`每个测试用例。 和`n = 5 * 10^5`总的测试来看，这太慢了。 

关键的观察结果是每个极点在平面上定义一条线。 我们希望在每个 x 坐标处，在整数点处评估的一组不断增长的线性函数中的最大值。 这正是一个动态凸包/线容器问题，但在稍微变换的坐标系中。 

每极`(i, h[i])`定义一条线：$$y = h[i] + (x - i)\cdot \frac{t_y}{t_x}$$重写：$$y = \frac{t_y}{t_x} x + \left(h[i] - i \cdot \frac{t_y}{t_x}\right)$$所有线共享相同的斜率分量`x`，所以我们可以分解出`t_y / t_x`并将线之间的比较减少为比较类似截距的值。 我们没有明确地维护实值线，而是比较离散位置的贡献。 

一种更简洁的方法是完全避免浮点斜率。 将所有内容乘以`tx`:$$y \cdot tx = h[i]\cdot tx + (x - i)\cdot ty$$在位置`x = j`, 来自极点的贡献`i`变成：$$h[i]\cdot tx + (j - i)\cdot ty$$所以对于每一个`j`，我们想要：$$\max_{i < j} (h[i]\cdot tx - i \cdot ty) + j \cdot ty$$该术语取决于`i`独立于`j`，所以我们保持运行最大值：$$best = \max(h[i]\cdot tx - i \cdot ty)$$那么包络值在`j`是：$$best + j \cdot ty$$如果满足以下条件，杆就会被照亮：$$h[j]\cdot tx > best + j \cdot ty$$这将整个问题简化为保持最大值的单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 变换 + 前缀最大 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 重写极点的条件`j`通过将几何比较的两边都乘以`tx`，删除分数并将阴影包络转换为线性表达式`j`。 
2. 观察之前极点的所有贡献`i < j`可以分成一个术语，仅取决于`i`和一个仅取决于的术语`j`。 这允许预先计算运行最大值`i`- 依赖部分。 
3. 维护一个变量`best`，最初非常负，代表最大值`h[i] * tx - i * ty`在所有处理过的杆上。 
4. 对于每个极点`j`，计算其“亮度分数”`h[j] * tx`并将其与`best + j * ty`。 
5. 如果分数严格更大，则增加答案，因为极点位于阴影包络线上方。 平等不算是阐明的，符合问题的严格条件。 
6. 更新`best`使用当前极点的贡献`h[j] * tx - j * ty`。 

### 为什么它有效

 每个先前的极点都贡献一个线性函数`j`投影到方向向量上后。 包络线是这些函数的逐点最大值。 由于每个函数仅在变换空间中的截距不同，因此每一步的最大值仅取决于迄今为止的最佳截距。 该算法准确地维护了该最大值，因此在每个索引处`j`它重建真实的包络值，而不显式存储所有行。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        h = list(map(int, input().split()))
        tx, ty = map(int, input().split())

        best = -10**30
        ans = 0

        for i in range(n):
            cur_score = h[i] * tx
            envelope = best + (i + 1) * ty

            if cur_score > envelope:
                ans += 1

            cand = h[i] * tx - (i + 1) * ty
            if cand > best:
                best = cand

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案在一次从左到右的扫描中处理每个测试用例。 关键的实现细节是使用`i + 1`一致，因为位置在几何中是 1 索引的。 比较是严格的，因此相等不被算作照亮。 

所有算术都是使用 Python 整数完成的，它可以轻松处理高达约`1e18`或更大，无需担心溢出。

 ## 边缘情况

 一个关键的边缘情况是所有杆具有相同的高度并且`ty = 0`。 信封变成一条恒定的水平线。 用于输入`n = 5, h = [3,3,3,3,3], tx = 2, ty = 0`，第一个极点被照亮，因为它位于初始空包络线之上，但随后的每个极点都恰好位于第一个极点形成的包络线上。 该算法计算`best = h[i]*tx`，并且每个新极点都进行比较`h[j]*tx`和`best`，由于严格的不平等而产生平等并正确拒绝照明。 

另一种情况是负斜率。 和`h = [1,2,3], tx = 1, ty = -5`，包络急剧减小。 该算法维持`best = h[i] - i*(-5)`，增加得很快，但减去`j*ty`有效地提高了包络项，允许后面的极点通过检查。 这说明了为什么仅跟踪最大高度会失败，而变换后的线性形式仍然正确。 

最后的边缘情况是，一个非常大的早期极点支配了所有后来的极点。 为了`h = [10^9, 1, 1, 1], tx = 1, ty = 1`，第一极设置了一个巨大的`best`，并且所有后续比较都无法满足严格的不等式。 扫描正确地将答案保持为 1，而无需重新计算或溢出风险。
