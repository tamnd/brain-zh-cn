 ## 解决方案
 ## 问题理解

 该问题描述了一艘带有两个独立激光器的宇宙飞船。 每个激光都有一个功率值和一个重新加载时间。 激光立即开始充电，每次射击后都需要重新装填时间才能再次射击。 如果两个激光器同时准备好，它们就可以一起发射并组合它们的功率。 

敌舰有耐久值和护盾值。 单次激光射击的威力减去护盾。 组合射击仅处理一次两次能量之和减去护盾值，因此同时发射比分别发射两束激光精确一个护盾值要好。 

任务是找到最早可以消灭敌人的时间。 

输入包含两束激光的功率和装弹时间，其次是敌人的耐久度和护盾。 输出是最后一击将敌人耐久值降低到零或以下的最短时间。 

耐久度最高5000，装弹次数可达$10^{12}$。 这个组合告诉我们，答案可能非常大，因此模拟每一秒或每个重新加载事件是不可能的。 同时，所需的损坏量足够小，我们可以负担得起取决于耐用性的算法，但我们需要避免迭代较大的时间值。 

一些边缘情况需要注意。 如果一束激光较弱但速度更快，则仅使用该激光可以避免等待组合射击。 例如：```
10 1
5000 100000
25 9
```第一个激光仅交易$10-9=1$每次射击造成伤害，但每秒都能射击。 答案是25，因为等待第二束激光会比较慢。 

另一种情况是激光器始终同步。 例如：```
5 10
4 10
16 1
```两个激光器分别在 10、20 等时间准备就绪。 联合射击交易$5+4-1=8$伤害，所以两次射击就足够了，答案是 20。如果只考虑单独射击的粗心解决方案会低估同步的优势。 

当重新加载时间没有常见的共同点时，就会出现第三种微妙的情况：```
100 7
100 11
50 1
```激光很少结合在一起，因此计算每一对可能的发射是浪费的。 关于组合射击的唯一相关信息是重装时间表的频率。 

## 方法

 直接方法将模拟时间线。 我们可以存储每个激光可用的下一时刻，重复选择最佳动作，并继续，直到造成足够的伤害。 这是正确的，因为每次射击事件只能在激光充电时发生。 然而，装填时间可能会非常长，因此即使是少量有用的射击也可能会在巨大的间隙后发生。 基于时间进展或事件处理的模拟是不合适的。 

关键的观察是我们实际上不需要知道镜头的顺序。 考虑固定的完成时间$T$。 此时，激光器1已经准确地完成了$\lfloor T/t_1 \rfloor$开火的机会，而激光 2 恰好有$\lfloor T/t_2 \rfloor$机会。 我们应该始终使用所有可用的射击，因为所有伤害都是积极的。 

如果两个激光同时可用，那么同时发射它们会产生额外的效果$s$与单独射击相比造成的伤害。 公共开火时刻恰好是两个装填时间的最小公倍数的正倍数。 因此，对于给定的时间$T$，最大伤害为：$$\left\lfloor\frac{T}{t_1}\right\rfloor(p_1-s)
+
\left\lfloor\frac{T}{t_2}\right\rfloor(p_2-s)
+
\left\lfloor\frac{T}{\operatorname{lcm}(t_1,t_2)}\right\rfloor s$$现在的问题是检查给定的时间是否足够。 这个谓词是单调的：如果时间$T$就够了，每次更大的时间也够了。 这允许对答案进行二分搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(答案/最短重新加载时间) | O(1) | O(1) | 太慢了 |
 | 最佳| O(log(答案)) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算单独发射激光 1、单独发射激光 2 的伤害值，以及组合这两种激光所获得的额外伤害。 单个激光贡献$p_i-s$，而组合镜头相当于两个单独的镜头加上一个额外的镜头$s$。 
2. 创建一个函数来检查给定时间是否$T$就足够了。 该功能按时间统计每个激光器可以发射多少次$T$，然后添加两个重新加载计划满足时的同步奖励。 
3. 找到两个重新加载时间的最小公倍数。 该值的每一个正倍数都是两个激光器同时准备就绪的时刻。 
4. 二分查找答案。 从一个小的右边界开始，然后反复将其加倍，直到检查功能表明可以在该时间内消灭敌人。 然后搜索区间，找到最小的有效时间。 
5. 第一次通过检查返回。 

为什么有效：对于每个选定的结束时间，该公式都会计算该时刻可能造成的所有伤害。 它包括每一个可能的单镜头和每一个可能的同步镜头。 由于延迟射击无法在同一截止日期之前造成更多伤害，因此这是当时可能的最大伤害。 检查函数是单调的，因此二分查找第一次发现敌人变得可摧毁。 

## Python 解决方案```python
import sys
import math

input = sys.stdin.readline

p1, t1 = map(int, input().split())
p2, t2 = map(int, input().split())
h, s = map(int, input().split())

d1 = p1 - s
d2 = p2 - s

g = math.gcd(t1, t2)
lcm = t1 // g * t2

def can(time):
    damage = (time // t1) * d1
    damage += (time // t2) * d2
    damage += (time // lcm) * s
    return damage >= h

left = 0
right = 1

while not can(right):
    right *= 2

while left < right:
    mid = (left + right) // 2
    if can(mid):
        right = mid
    else:
        left = mid + 1

print(left)
```第一部分计算每个激光的有效伤害。 每次射击都会减去护盾，因此公式中使用的伤害值已经是真实伤害值。 

这`can`函数是问题的整个数学模型。 两个楼层的分区计算个人射击机会。 最不常见的倍数计算的是两束激光可以合并为一次更强攻击的时刻，以及额外的倍数$s$占节省的护盾减少量。 

初始二分搜索上限未被猜测。 它通过加倍扩展直到有效，这保证了即使是由重新加载时间引起的非常大的答案$10^{12}$被处理。 Python 整数避免了最小公倍数计算和损害计算期间的溢出。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(log(答案)) | 每个二分搜索步骤仅执行恒定时间算术。 |
 | 空间| O(1) | O(1) | 该算法仅存储几个整数变量。 |

 二分查找的迭代次数很少，因为答案是通过将一个区间反复减半来找到的，而该区间的上界是通过加倍得到的。 该解决方案不依赖于重新加载时间的大小，因此值约为$10^{12}$很容易处理。

import sys
import io
import math

def solve(data):
    it = iter(data.strip().split())
    p1 = int(next(it))
    t1 = int(next(it))
    p2 = int(next(it))
    t2 = int(next(it))
    h = int(next(it))
    s = int(next(it))

    d1 = p1 - s
    d2 = p2 - s
    lcm = t1 // math.gcd(t1, t2) * t2

    def can(t):
        return (t // t1) * d1 + (t // t2) * d2 + (t // lcm) * s >= h

    l, r = 0, 1
    while not can(r):
        r *= 2

    while l < r:
        m = (l + r) // 2
        if can(m):
            r = m
        else:
            l = m + 1
    return str(l)

assert solve("10 1 5000 100000 25 9") == "25", "sample 2"
assert solve("5 10 4 10 16 1") == "20", "equal reload times"

 ## 边缘情况

 当一束激光比另一束激光快得多时，算法不会强制组合射击。 为了：```
10 1
5000 100000
25 9
```时间 25 处的检查计算出第一束激光发射了 25 次，第二束激光发射了 0 次，同步奖励为零。 伤害正好是25，所以二分查找这次接受了。 

当两个激光器始终同时准备就绪时，同步项就变得至关重要。 为了：```
5 10
4 10
16 1
```最小公倍数为 10。在时间 20 时，每个激光有两次发射机会，并且有两个组合时刻。 伤害计算得出：$$2 \cdot 4 + 2 \cdot 3 + 2 \cdot 1 = 16$$所以算法返回 20。 

当最小公倍数非常大时，该算法仍然有效，因为它从不枚举同步时刻。 它仅使用整数除法计算截止日期之前存在多少个。 为了：```
100 7
100 11
50 1
```前几次二分查找检查只是简单地评估算术公式，无需模拟任何镜头即可找到答案。
