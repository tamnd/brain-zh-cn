 ## 解决方案
 ## 问题理解

 我们得到一组现有的酒店，它们放置在沿着无限数轴的整数点上。 我们想在某个整数坐标处再开一家酒店。 要求是离这家新酒店最近的现有酒店的距离必须准确$d$。 换句话说，如果我们查看从新位置到每个现有酒店的所有距离，这些距离的最小值必须等于$d$。 

输入为我们提供了现有酒店的数量及其坐标，已按升序排序。 任务是计算新酒店有多少个整数位置满足这个精确的距离条件。 

约束条件$n \le 100$足够小，可以毫无顾虑地对候选进行二次扫描。 即使我们考虑检查每个潜在的候选职位并针对所有现有酒店进行验证，运营总数仍将保持在几百万之内。 这立即排除了对高级数据结构或对数查询优化的需要。 简单的几何检查方法就足够了。 

在这个问题中经常出现的一个天真的误解是认为我们只需要精确地检查距离上的点$d$来自某些酒店，未经进一步验证。 这还不够，因为一个点可能位于远处$d$距离一家酒店较近$d$到另一个。 

例如，考虑位置 0 和 10 的酒店$d = 5$。 点 5 与两家酒店的距离均为 5，因此它是有效的。 但像 4 这样的点距离 0 为 4，距离 10 为 6，因此它的最小距离是 4，而不是 5，因此不能计算在内。 类似地，像 15 这样的点距 10 的距离为 5，但距 10 的距离仅为 5，距 0 的距离仅为 15，因此它是有效的； 但是，如果附近有另一家酒店，则该点可能无法满足条件。 这表明围绕单个酒店的局部推理是不够的； 需要全局验证。 

另一个微妙的边缘情况是候选人职位的重复。 不同的酒店可能会生成相同的候选坐标$x_i + d$或者$x_i - d$，多次计数会高估答案。 

## 方法

 蛮力的想法是迭代合理范围内的所有整数坐标，例如从最小酒店位置减去$d$到最大加$d$，并检查每个点到任意酒店的最小距离是否恰好是$d$。 原则上这是正确的，但范围可以大到$10^9$双向，无法直接扫描。 

关键的观察是有效点必须恰好位于半径为圆的边界上$d$以某酒店为中心。 如果某个点有效，则其最近的酒店的距离必须准确$d$，这意味着它必须满足$|y - x_i| = d$至少一个$i$。 这立即将搜索空间减少到每个酒店只有两个候选者：$x_i - d$和$x_i + d$。 

然而，产生候选人还不够。 每位候选人必须针对所有酒店进行验证，以确保没有一家酒店比距离更近$d$。 这种验证很便宜，因为$n \le 100$，因此检查每个候选成本的所有距离$O(n)$，而我们只有$2n$候选人。 

因此，最终的解决方案是针对小型候选集的生成和验证方法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解坐标 |$O(R \cdot n)$在哪里$R$可以是$10^9$|$O(1)$| 太慢了 |
 | 从每家酒店生成候选人|$O(n^2)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 对于每个酒店位置$x_i$，构造两个候选位置：$x_i - d$和$x_i + d$。 这些是唯一可能具有精确最小距离的点$d$，因为任何有效点都必须恰好是$d$远离至少一家酒店。 
2. 对于每个候选人职位$y$，计算其到每家酒店的距离。 
3. 在检查距离时，记录与$y$到任何一家酒店。 这决定是否$y$满足条件。 
4. 如果最小距离正好等于$d$，将此候选人视为有效。 
5. 使用集合来避免计算不同酒店产生的重复候选职位。 

关键思想是我们永远不需要考虑不远处的点$d$来自某个酒店，因为任何有效的解决方案都必须触及至少一个以酒店为中心的“半径边界”。 

### 为什么它有效

 任何有效位置$y$必须满足$\min_i |y - x_i| = d$。 这意味着至少存在一个索引$i$这样$|y - x_i| = d$，以及所有其他指数$j$,$|y - x_j| \ge d$。 因此，每个有效点必须出现在由以下内容构建的候选集中：$x_i \pm d$。 验证步骤可确保我们排除距离其他酒店太近的候选者，从而保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d = map(int, input().split())
    x = list(map(int, input().split()))

    candidates = set()

    for xi in x:
        candidates.add(xi - d)
        candidates.add(xi + d)

    ans = 0

    for y in candidates:
        best = float('inf')
        for xi in x:
            dist = abs(y - xi)
            if dist < best:
                best = dist
                if best < d:
                    break
        if best == d:
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先在每家酒店周围构建所有可能的候选边界。 该集合至关重要，因为多个酒店可以生成相同的坐标，并且重复项不得夸大答案。 

在验证过程中，我们计算每个候选者到任何酒店的最近距离。 一旦我们检测到距离小于$d$，因为这会立即取消候选人的资格。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n, d = map(int, input().split())
    x = list(map(int, input().split()))

    candidates = set()
    for xi in x:
        candidates.add(xi - d)
        candidates.add(xi + d)

    ans = 0
    for y in candidates:
        best = float('inf')
        for xi in x:
            best = min(best, abs(y - xi))
        if best == d:
            ans += 1

    return str(ans)

 ## 边缘情况

 当只有一家酒店时，每个有效位置正好是两点：$x_1 - d$和$x_1 + d$。 该算法准确地生成了这两个候选者，并且都通过了验证，因为没有其他酒店可以违反距离条件。 

当酒店距离很近时，一些候选边界会失败，因为另一家酒店比其他酒店更近$d$。 验证循环通过检查所有距离正确地消除了这些情况。 

当多个酒店生成相同的候选坐标时，该集合确保它仅被计数一次。 如果没有重复数据删除，相同的有效位置可能会被过度计数，尤其是在存在对称配置时。
