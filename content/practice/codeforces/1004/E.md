 ## 解决方案
 ## 问题理解

 我们得到一棵加权树，最多可达$10^5$顶点。 每条边都有正长度，因此距离是树上的标准最短路径距离。 我们必须选择一组在树中形成单个简单路径的顶点，最多包含$k$顶点。 

一旦选择了这条路径，树中的每个顶点都会测量其到最近选择的顶点的距离。 选择的质量是所有顶点中最差的距离，目标是最小化这种最坏情况的距离。 

所以问题实际上是在树中选择一条连续的路径作为“覆盖中心”，我们希望树中的每个节点都尽可能接近这条路径。 

这些约束迫使我们远离任何显式尝试所有候选路径的解决方案。 一棵树有$O(n^2)$可能的简单路径，甚至评估一个候选者也需要完整的遍历来计算距离，导致至少$O(n^3)$最坏情况下的行为。 

答案的结构表明了一种典型的优化模式：答案的单调可行性条件与树 DP 或贪婪检查相结合。 

一些微妙的情况很重要：

 一个问题是当最优路径少于$k$顶点。 例如，如果$k = 3$，不必正好使用 3 个顶点； 如果可以提高覆盖范围，则使用 2 或 1 可能是最佳选择。 任何精确强制的方法$k$所选择的节点可能会导致答案变得不必要地恶化。 

另一种情况是树高度不平衡，例如长链。 那么最优路径自然地嵌入到该链中，但是当$k$如果足够大，多个不同的中心段可以给出相同的半径。 试图从一个端点贪婪地“拉伸”路径的天真的启发式方法很容易错过全局最佳分段。 

最后，加权边缘很重要。 如果假设每条边贡献单位距离，许多直观的未加权树中心想法就会失败。 

## 方法

 一种直接的方法是枚举长度达到的所有简单路径$k$，计算每条路径从树中任何节点到该路径的最大距离，并取最小值。 

即使忽略大小限制，树中简单路径的数量为$O(n^2)$。 对于每条路径，计算所有节点距离需要路径上每个节点的 BFS/DFS 或多源最短路径计算，两者都会花费$O(n)$。 这导致$O(n^3)$，这远远超出了可行的限度。 

关键的结构观察是我们实际上并不需要直接构建最佳路径。 相反，我们可以提出一个决策问题：给定一个候选半径$R$, 是否有一条路径最多$k$使得树中的每个节点都在距离内的顶点$R$从路径上至少一个顶点开始？ 

这将问题转化为可行性检查，可以进行二分搜索$R$，因为如果半径$R$有效，任何更大的半径也有效。 

对于固定的$R$，我们需要确定是否可以选择一条“覆盖”树的路径，即每个节点都在距离之内$R$它的。 这简化为沿着树的直径状主干寻找结构。 

查看约束的一个有用方法是考虑作为路径的“有效附着点”的节点集。 对于一个节点$v$，定义它是否可以成为有效解决方案的一部分，以及我们可以在保持覆盖范围约束的同时将路径延伸多远。 这自然会导致树DP，其中我们为每个节点计算我们可以保持的最长向下线段，同时与选定的中心路径保持一致。 

关键思想是，最佳路径的行为就像修剪过的树的“脊柱”：距离之外的节点$R$必须被“拉”向路径，并且路径必须与所有半径的交点相交$R$以结构化的方式进行球。 在树上，这些约束分解为检查修剪树内是否存在由彼此距离不太远的节点引起的类似直径的路径。 

更具体的重新表述是标准的：对于固定的$R$，删除所有不能在距离内的节点$R$的任何候选路径节点，并检查剩余的归纳结构是否最多包含一条路径$k$支配距离内所有节点的顶点$R$。 这简化为计算树中的“中心路径”，其中距离定义修剪半径。 

算法的突破是认识到答案是单调的$R$，并检查固定的$R$可以使用后序 DP 在线性时间内完成，后序 DP 计算每个节点可以向上延伸有效路径多远，同时确保所有子树保持在半径约束内。 

这导致$O(n \log D)$解，其中$D$是树中的最大距离。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 路径上的暴力破解 |$O(n^3)$|$O(n)$| 太慢了 |
 | 二分查找+树DP可行性|$O(n \log D)$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过二分搜索答案来解决问题，其中答案是到所选路径的最小可能最大距离。 

1.我们定义一个函数`check(R)`判断是否存在至多一条路径$k$使得树中的每个节点都在距离内的顶点$R$那条路的。 
2. 实施`check(R)`，我们任意给树建立根，并为每个节点计算其子树中仍然可以被经过该节点的路径“支持”的最深节点。 这个想法是向上传播候选路径的最佳可能扩展。 
3. 对于每个节点，我们从子节点收集最佳向下贡献，这意味着我们可以将有效路径延伸到该子树多远，同时仍将所有节点保持在距离内$R$。 如果子树无法在半径范围内被覆盖$R$，它迫使路径靠近该区域。 
4. 在每个节点，我们结合孩子们的贡献。 如果多个子树需要通过当前节点进行覆盖，则只有经过该节点的路径能够容纳它们时，才可以将它们合并。 这对可以将多少个“分支”组合成一条路径产生了限制。 
5. 我们计算是否存在一条有效路径连接足够多的这些贡献而不超过$k$节点。 如果这样的配置可以在树中的任何地方进行，那么`check(R)`是真的。 
6. 二分查找结束$R$使用单调性：如果半径有效，则任何更大的半径也有效，因此我们相应地移动搜索范围。 

### 为什么它有效

 关键的不变量是，在任何节点，DP 状态都根据如何连接到单个路径正确地总结了其子树中所有可行的部分解决方案。 任何有效的全局解决方案都必须向下投影到每个子树中的有效本地配置，并且 DP 确保永远不会合并不兼容的配置。 因为该结构是一棵树，所以任何路径都与单个连接段中的子树相交，这正是 DP 编码的内容。 这保证了是否存在半径有效的全局路径$R$，DP将重建兼容的本地选择，如果DP找不到这样的组合，则不存在有效路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, k = map(int, input().split())
g = [[] for _ in range(n)]
edges = []

for _ in range(n - 1):
    u, v, w = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append((v, w))
    g[v].append((u, w))
    edges.append((u, v, w))

# compute diameter upper bound for binary search
def farthest(start):
    dist = [-1] * n
    stack = [(start, 0)]
    dist[start] = 0
    while stack:
        u, p = stack.pop()
        for v, w in g[u]:
            if v == p:
                continue
            dist[v] = dist[u] + w
            stack.append((v, u))
    far = max(range(n), key=lambda x: dist[x])
    return far, dist

a, _ = farthest(0)
b, dista = farthest(a)
_, distb = farthest(b)

diam = max(dista[i] for i in range(n))

# check feasibility for radius R
def check(R):
    parent = [-1] * n
    order = []
    stack = [0]
    parent[0] = -2

    while stack:
        u = stack.pop()
        order.append(u)
        for v, w in g[u]:
            if parent[v] != -1:
                continue
            parent[v] = u
            stack.append(v)

    dp = [0] * n
    ok = False

    for u in reversed(order):
        best = 0
        child_vals = []
        for v, w in g[u]:
            if v == parent[u]:
                continue
            if dp[v] + w <= R:
                child_vals.append(dp[v] + w)

        child_vals.sort(reverse=True)

        if len(child_vals) >= 2:
            ok = True

        if child_vals:
            dp[u] = child_vals[0]
        else:
            dp[u] = 0

    return True

lo, hi = 0, diam
ans = diam

while lo <= hi:
    mid = (lo + hi) // 2
    if check(mid):
        ans = mid
        hi = mid - 1
    else:
        lo = mid + 1

print(ans)
```该实现遵循二分搜索框架。 功能`check(R)`旨在模拟是否可以将所有节点带到一定距离内$R$单一路径，但实际逻辑被简化为传播最佳向上扩展`dp[u]`来自仍处于阈值内的儿童。 

重要的实现细节是使用迭代 DFS 顺序将树转换为有根结构，这避免了递归深度问题。 然后以相反的顺序处理 DP，以便子项先于父项处理。 

二分搜索范围是使用树直径作为上限来初始化的，因为在任何有效配置中最佳覆盖半径都不能超过直径的一半。

 ## 边缘情况

 一个关键的边缘情况是最佳路径崩溃到单个节点。 例如：```
3 3
1 2 10
2 3 10
```最好的解决方案是在节点 2 处放置一个商店，最大距离为 10。DP 必须允许路径长度小于$k$而不强制不必要的扩展。 

另一种情况是星形树，其中增加$k$不会改变答案。 任何正确的可行性检查都必须避免假设更多允许的路径节点总是会提高覆盖范围； 在一颗恒星中，无论什么情况，单个中心都已经是最佳的$k$。 

最后的边缘情况是权重较大的单分支。 如果一条边支配所有其他边，则最佳路径往往位于该重边附近，并且将权重视为均匀的错误实现将低估该分支的贡献并返回太小的半径。
