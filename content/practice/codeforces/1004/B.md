 ## 解决方案
 ## 问题理解

 我们得到一排位置，每个位置都必须填充两种可能类型中的一种。 您可以将其视为构造一个长度为二进制的字符串$n$，其中每个位置要么是类型 A，要么是类型 B。 

有多个访问者，每个访问者都会查看该行的连续部分。 对于给定的细分市场，其得分由其混合程度定义：具体来说，如果我们计算该细分市场中有多少位置属于类型 0，有多少位置属于类型 1，则该细分市场的贡献就是这两个计数的乘积。 

总分是所有给定细分中这些产品的总和，目标是分配 0 和 1 以使该总和最大化。 

限制条件$n, m \le 10^3$暗示着一个$O(n^2 m)$甚至$O(n^3)$在最坏的情况下，解决方案已经太慢了，因为这会达到大约$10^9$运营。 我们应该期待$O(nm)$或者$O(n^2)$构建，可能涉及计算每个位置对目标的贡献，而不是从头开始重新计算分段分数。 

一个关键的微妙之处在于，细分市场的贡献在位置上不是线性的。 从 0 到 1 的单次翻转会以非线性方式更改包含该位置的所有段，因此单个段上的贪婪局部推理是不可靠的，除非我们将问题转换为每个位置影响模型。 

## 方法

 一个蛮力的想法是尝试所有$2^n$0 和 1 的分配。 对于每个作业，我们通过计算每个部分中的 0 和 1 来计算其分数。 每段计算量为$O(n)$，因此评估一项作业成本$O(mn)$，总复杂度变为$O(2^n m n)$，即使对于$n = 20$。 

更结构化的暴力破解通过注意到段分数仅取决于 0 和 1 的计数来改善这一点。 我们可以预先计算前缀和并评估每个段$O(1)$，将一项作业的评估减少到$O(n + m)$。 仍然，$2^n (n+m)$是不可能的。 

关键的结构性转变是停止从细分角度思考，而是从成对职位角度思考。 展开每个分段内的乘积表明，同一分段内的每对位置如果颜色不同，都会对总分做出贡献。 这将目标转换为指数对的加权和。 每对贡献的权重等于包含两个位置的段数。 现在问题变成：分配0/1以最大化相对标签之间的切割边的权重之和。 这正是完整图的最大切割，其权重源自线段重叠。 

一旦被视为切割问题，结构就会变得对称：每个位置通过非负权重与其他位置相互作用。 此设置中的标准技巧是通过累积每个位置的带符号影响并选择局部增加目标的符号来贪婪地分配标签，因为该问题相当于最大化具有非负系数的二次形式。 

这将问题简化为计算每个位置与段内所有其他位置的连接强度，然后根据相对于全局平衡它是否更喜欢处于 0 组或 1 组中来分配它。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举 |$O(2^n (n+m))$|$O(n)$| 太慢了 |
 | 成对权重+贪心赋值|$O(n^2 + m)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们首先计算每对位置的重要性。 这是通过计算有多少段来完成的$[l, r]$包含两个索引$i$和$j$。 如果我们修复$i \le j$，这个计数是左端点最多的线段数$i$右端点至少是$j$。 使用区间端点上的二维差分数组进行预计算，得到一个矩阵$w[i][j]$在$O(n^2 + m)$。 
接下来我们解释一下目标。 两个位置的$i$和$j$接收不同的值，我们获得$w[i][j]$。 如果它们相等，我们就不会从这对中得到任何好处。 因此，目标是将索引分为两组，从而最大化交叉对的总权重。 

1.我们构建一个2D数组$w$在哪里$w[i][j]$计算有多少个段包含两者$i$和$j$。 这是通过使用段端点上的前缀累积来完成的。 
2. 我们用全零初始化一个赋值数组。 
3. 我们计算每个位置与先前决定的位置的总交互作用。 如果我们翻转它的值，这就是增益差。 
4.我们对职位进行一一处理。 对于职位$i$，我们计算已分配的零的权重总和减去一的总和。 如果该值为正，则设置$i$为 1 会增加切割权重，否则我们将其设置为 0。 
5. 我们输出生成的二进制字符串。 

每一步的决策都是正确的，因为节点的贡献仅取决于它与已经固定的节点的交互，并且未来的决策将对称地考虑相同的成对权重，因此我们不会不一致地重复计算。 

### 为什么它有效

 总分可以重写为无序索引对的总和，其中每对都贡献$w[i][j]$当且仅当两个端点具有不同的标签时。 这使得目标成为二元变量的二次形式。 当我们按顺序分配变量时，为位置分配值的边际收益$i$仅取决于先前分配的变量加上独立于未来分配的常量偏移量。 由于每对在分配其第二个端点时都被考虑一次，因此贪心选择优化了本地贡献，而不影响未来贡献的一致性。 这保证了结构在固定的加工顺序下保持最佳状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    segs = [tuple(map(int, input().split())) for _ in range(m)]

    # count coverage of segments in a 2D sense using endpoint sweeps
    add = [[0] * (n + 2) for _ in range(n + 2)]

    for l, r in segs:
        add[l][r] += 1

    # prefix sum over l dimension
    for i in range(1, n + 1):
        for j in range(1, n + 1):
            add[i][j] += add[i - 1][j]

    # suffix sum over r dimension
    for i in range(n, 0, -1):
        for j in range(n, 0, -1):
            add[i][j] += add[i][j + 1]

    w = [[0] * (n + 1) for _ in range(n + 1)]

    # w[i][j] = number of segments covering both i and j
    for i in range(1, n + 1):
        for j in range(i, n + 1):
            w[i][j] = add[i][j]

    # greedy assignment
    res = [0] * (n + 1)
    used0 = [0] * (n + 1)
    used1 = [0] * (n + 1)

    for i in range(1, n + 1):
        gain0 = 0
        gain1 = 0
        for j in range(1, i):
            if res[j] == 0:
                gain1 += w[j][i]
            else:
                gain0 += w[j][i]

        if gain1 > gain0:
            res[i] = 1
        else:
            res[i] = 0

    print("".join(str(res[i]) for i in range(1, n + 1)))

if __name__ == "__main__":
    solve()
```该实现首先将段信息转换为成对交互表。 二维前缀构造是关键的技术步骤：它聚合了每对覆盖的间隔数$(i, j)$。 索引被小心地转移以避免边界问题，通过使用$n+2$网格。 

贪心循环从左到右处理索引。 对于每个位置，它会比较相对于已固定位置分配 1 与 0 的好处。 仅考虑之前处理过的索引，这确保每对在决策时只贡献一次，从而防止重复计算。 

最终字符串是直接从赋值数组构造的。

 ## 边缘情况

 一个最小的案例$n = 1$不包含任何段或一个简单的段，并且任何分配都是最佳的。 该算法处理唯一的索引，发现没有先前的贡献，并分配 0，产生有效的输出。 

完全覆盖的部分，例如$[1, n]$在所有对之间创建统一的交互权重。 在这种情况下，每个位置都会看到相同的增益，并且算法始终分配 0，除非出现严格更好的增益，从而在许多最佳增益中产生一个有效分区。 

当嵌套结构中的段严重重叠时，例如$[1, 10], [2, 9], [3, 8]$，内部位置累积较大的成对权重。 贪婪扫描自然会首先分配早期的高影响力位置，然后对齐后期位置以最大化交叉贡献，从而匹配目标的预期结构。
