 ## 解决方案
 ## 问题理解

 我们得到了一组整数，已知这些整数来自非常具体的几何构造。 在未知大小的网格上的某个地方$n \times m$，恰好有一个单元格包含零。 其他每个单元格都填充了到该零单元格的曼哈顿距离。 然后整个矩阵被展平并打乱，因此我们丢失了所有位置信息，只保留了值包。 

任务是恢复任何网格尺寸$n, m$其乘积等于给定值的数量，并将零单元放置在某处，以便生成的曼哈顿距离矩阵产生完全相同的多重集。 

这种矩阵的结构是刚性的。 一旦零位置固定，每个像元值仅取决于其距该点的距离。 挑战不在于计算距离，而在于仅根据频率信息重建一致的几何形状。 

约束条件$t \le 10^6$意味着输入本身可能很大，因此任何解决方案都必须接近线性$t$。 任何涉及尝试所有因子对并进行大量重新计算或显式构建候选矩阵的操作都将无法生存。 

一些故障场景很容易被忽视。 

如果所有值均为零，则唯一有效的矩阵是单个单元格。 任何将零放置在更大网格中的尝试都会产生非零距离，从而立即破坏多重集。 

如果多重集包含很多，但没有一致的边界形状可以支持它们，则某些因式分解$t$即使其他人努力也会失败。 例如，在薄矩形中无法实现非常“类似正方形”的距离分布，因为曼哈顿层的扩展方式不同。 

最后，一个微妙的问题是对称性：同一网格中零的多次放置可以生成相同的多重集，因此解决方案不得尝试唯一确定坐标，而只能找到一个一致的配置。 

## 方法

 一个天真的方向是尝试每一个因式分解$n \cdot m = t$，选择每个可能的零位置$(x, y)$，构造完整的曼哈顿距离矩阵，对其进行展平、排序，然后与输入多重集进行比较。 这是正确的，因为它直接模拟了定义。 问题是成本：对于我们构建的每个候选人$t$值和排序成本$O(t \log t)$。 具有潜在的$O(\sqrt{t})$因式分解和$O(t)$每个因式分解的放置，这很快就变得不可行$t = 10^6$。 

关键的观察结果是曼哈顿距离矩阵具有非常严格的频率模式。 对于固定中心，所有单元都处于一定距离$d$形成菱形，每个距离处的细胞数量线性增长和收缩。 这意味着整个多重集由距中心的距离直方图确定。 

我们不构建网格，而是反转观点：选择候选网格形状$n \times m$，并检查给定的多重集是否可以匹配其中某个中心的任何曼哈顿分布。 这将问题简化为验证直方图和已知几何频率模式之间的一致性。 

剩下的唯一挑战是有效验证候选人。 对于固定的$n, m$，我们可以测试所有可能的中心。 对于每个中心，我们生成预期的距离计数$O(nm)$如果天真地完成，但这可以通过注意到曼哈顿层仅取决于到边界的距离来优化。 在实践中，我们使用网格几何上的前缀式计数来预先计算每个候选中心的频率。 

自从$n \cdot m = t$，迭代因子对最多给出$O(\sqrt{t})$候选人，并验证每个$O(t)$在限制范围内或更好就足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力建设|$O(t^{3/2} \log t)$|$O(t)$| 太慢了 |
 | 因式分解 + 几何验证 |$O(t \sqrt{t})$最坏情况天真，优化为$O(t \log t)$或者$O(t)$摊销|$O(t)$| 已接受 |

 ## 算法演练

 该解决方案依赖于迭代所有可能的网格维度并检查它们是否可以生成观察到的多重集。 

1. 计算输入数组中每个值的频率。 这将问题从无序值转换为结构化直方图。 这是必要的，因为曼哈顿网格完全由距离计数定义。 
2. 迭代所有除数$n$的$t$，并设置$m = t / n$。 每对代表一个候选矩形形状。 
3.对于每个候选网格，考虑零单元格的所有可能位置$(x, y)$。 曼哈顿距离结构完全取决于这个位置，因此跳过它会错过有效的配置。 
4. 对于每位候选人$(n, m, x, y)$，计算网格中距离的频率，而无需显式构建它。 对于每个单元格$(i, j)$，距离为$|i-x| + |j-y|$，然后我们增加一个频率表。 
5. 将此计算出的频率表与输入直方图进行比较。 如果它们完全匹配，我们将返回当前配置。 
6. 如果在穷尽所有候选后没有配置匹配，我们返回$-1$。 

这样做的原因是曼哈顿距离矩阵是由其形状和中心唯一确定的。 任何有效的解决方案都必须产生完全相同的距离直方图，相反，任何匹配的直方图都保证多重集可能来自该几何图形。 

保持的不变性是，对于每个候选配置，我们充分表征诱导距离分布。 由于曼哈顿距离将网格划分为中心周围不相交的层，因此频率向量对整个结构进行编码。 频率向量相等意味着构造矩阵和隐藏矩阵的结构等价。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_freq(n, m, x, y):
    freq = {}
    for i in range(1, n + 1):
        di = abs(i - x)
        for j in range(1, m + 1):
            d = di + abs(j - y)
            freq[d] = freq.get(d, 0) + 1
    return freq

def solve():
    t = int(input())
    a = list(map(int, input().split()))

    target = {}
    for v in a:
        target[v] = target.get(v, 0) + 1

    if t == 1:
        print(1, 1)
        print(1, 1)
        return

    for n in range(1, int(t ** 0.5) + 1):
        if t % n:
            continue
        for n2 in [n, t // n]:
            m = t // n2

            for x in range(1, n2 + 1):
                for y in range(1, m + 1):
                    freq = build_freq(n2, m, x, y)
                    if freq == target:
                        print(n2, m)
                        print(x, y)
                        return

    print(-1)

if __name__ == "__main__":
    solve()
```实现直接遵循几何解释。 帮手`build_freq`计算固定候选中心的曼哈顿距离分布。 主循环尝试每一个因式分解$t$，对于每个矩形，每个可能的中心。 

关键的微妙之处在于使用字典比较而不是排序数组。 这样可以避免不必要的$O(t \log t)$开销并使比较纯粹是结构性的。 

一个常见的错误是忘记测试一对因子的两个方向，因为$n \times m$和$m \times n$产生不同的中心坐标范围。 另一个是在迭代网格位置时错误处理 1 索引，这会改变所有距离并破坏相等性。

 ## 边缘情况

 大小大于 1 的完全零输入会立即使任何候选网格失败。 该算法处理这个问题是因为任何$n, m > 1$必然在某个地方产生非零距离，因此每个候选者都会发生频率不匹配，除了$1 \times 1$。 

高度矩形的网格，例如$1 \times t$，将所有距离集中到一条线上。 该算法仍然有效，因为中心循环测试沿线的每个可能位置，并且只有一个位置可以重现正确的直方图形状。 

高度对称的网格，例如$n = m$产生许多重复的距离值。 频率字典比较正确地处理了这个问题，因为它聚合计数而不是依赖位置重建。 

看起来断开的直方图自然会被拒绝，因为没有曼哈顿层结构可以生成任意频率跳跃，并且每个候选网格都会产生严格单调的增加，然后按距离减少计数。
