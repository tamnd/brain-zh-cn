 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 从这个数组中，我们在概念上形成了每个有序的不同索引对之间的所有成对差异。 对于每对位置$i \neq j$，我们计算$|a_i - a_j|$。 这会产生尺寸的多重集$n(n-1)$，因为两个方向都被计算在内。 

任务不是显式构造这个多重集，而是找到其最大的和$m$价值观。 

困难来自于这样一个事实：$n$可能足够大以至于不可能枚举所有对。 即使是为了$n = 2 \cdot 10^5$，对的数量约为$4 \cdot 10^{10}$，这远远超出了任何可行的计算。 解决方案必须依赖于这些差异如何分布的结构。 

一个简单的实现将显式计算所有成对差异，对它们进行排序，并对顶部求和$m$。 这在时间和记忆上都立即失败了。 

当推理对称性时，会出现一个更微妙的问题。 每对在多重集中贡献两个值：$|a_i - a_j|$和$|a_j - a_i|$，它们是相等的。 因此，每个不同的无序对贡献的值恰好是相同值的两倍。 忘记这种重复会导致计数和求和中出现两倍的错误。 

当所有值都相等时，就会出现边缘情况。 在这种情况下，每个差异都为零，因此无论什么，答案都必须为零$m$。 任何依赖于排序或分区的解决方案都必须处理所有贡献都崩溃为单个值的退化情况。 

## 方法

 蛮力方法很简单。 对于每对索引$i$和$j$, 计算$|a_i - a_j|$，将其存储在列表中，将列表按降序排序，并取第一个$m$元素。 这是正确的，因为它明确地构造了问题中描述的多重集。 然而，它需要生成$n(n-1)$值，其大小是二次方。 为了$n = 2 \cdot 10^5$，这在时间和记忆上都是不可行的。 

关键的观察结果是绝对差异仅取决于顺序，而不取决于位置。 如果我们对数组进行排序，较大的差异对应于排序顺序中相距较远的对。 具体来说，对于两个元素$a_i \le a_j$，差异是$a_j - a_i$。 最大化该值意味着将小元素与大元素配对。 

我们可以考虑排序数组以及在按降序排列所有差异时每个成对距离可以被“选择”多少次，而不是生成所有对。 其结构是索引空间中固定距离引起的差异对应于值空间中的单调段。 这使我们能够聚合距离层而不是单个对的贡献。 

标准转换是对数组进行排序并考虑对称分区之间的差异。 最大的差异来自极端：$a_1$和$a_n$， 然后$a_1$和$a_{n-1}$和$a_2$和$a_n$， 等等。 这表明我们采用双指针结构，其中我们重复获取最大的剩余间隙，但贪婪地这样做仍然需要仔细计数，因为每对都会贡献两次。 

更可靠的思考方式是组合排名。 对于排序数组中的每个固定差异位置，我们可以计算有多少对至少在给定阈值上产生差异，然后使用它来累积顶部$m$贡献。 这将问题变成了对排序差异进行计数+前缀总和，而不是显式枚举。

 | ---| ---| ---| ---|

 ## 算法演练

 我们对数组进行排序，以便所有差异可以表示为$a[j] - a[i]$和$j > i$。 然后我们的想法是每对贡献两次。 

1. 对数组进行非降序排序。 这确保了每个差异都可以被视为非负前向差异，从而消除了绝对值的复杂性。 
2. 计算共有多少个无序对，即$n(n-1)/2$，并记住每个人在多重集中贡献两次。 这让我们能够一致地根据有序或无序对进行推理。 
3.我们想要最大的$m$所有有序差异之间的值。 我们不去生成它们，而是考虑按价值降序选择差异。 
4. 对于固定索引$i$, 所有对$(i, j)$和$j > i$产生价值$a[j] - a[i]$，形成递减序列为$i$增加并且$j$减少。 这种结构意味着我们可以通过固定端点并计算它们生成的对数来枚举贡献。 
5. 我们模拟使用贪婪两指针累加选择最大差异。 我们总是比较排序数组左右两端的下一个最佳候选间隙，因为极端对主导所有内部对。 
6. 每次我们都会挑选一对$(i, j)$，我们将其贡献两次添加到答案中并减少$m$因此。 我们移动仍然允许生成下一个最大未使用差异的指针。 
7. 我们继续直到$m$已考虑有序差异。 

### 为什么它有效

 排序后，每个差异对应一对唯一的索引$(i, j)$。 一对的值在两个方向上都是单调的：递增$j$增大差异，减小$i$增加它。 因此，成对差异的全局排序与数组极值附近的对对齐。 贪心选择总是选择最大剩余可用差值，因为任何非极端对都受到尚未使用的某些极端配对的限制。 这确保了在每一步中，我们都选择多重集的下一个最大元素，而无需显式维护所有候选元素。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort()

        i, j = 0, n - 1
        ans = 0

        # We process pairs in a greedy manner from the ends.
        # Each pair contributes twice to the multiset.
        while i < j and m > 0:
            left_gain = a[j] - a[i]
            
            # number of pairs available using this boundary
            cnt = (j - i) * 2

            take = min(m, cnt)

            ans += take * left_gain

            m -= take

            # move the pointer that reduces future maximum span
            # shrinking from the side with fewer remaining contributions
            if (j - i) <= (n - 1 - j + i):
                j -= 1
            else:
                i += 1

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码首先对每个测试用例数组进行排序，以使差异变得有方向性。 两个指针`i`和`j`代表我们正在压缩的当前极值区间。 表达式`a[j] - a[i]`是仍然跨越当前段的所有对之间的最大可能差异。 

变量`cnt`考虑到每个无序对在多重集中出现两次，因此我们可以消耗最多`2 * (j - i)`该值的有序差异。 我们减去`m`因此。 

指针移动启发式旨在从贡献较少剩余对的一侧缩小段，确保我们在耗尽较大的差异水平后系统地暴露较小的差异水平。

 ## 边缘情况

 当所有元素相等时，每对差异为零。 该算法立即产生`diff = 0`每一步，所以即使`m`很大时，累积的答案仍然为零。 指针移动并不重要，因为缩小间隔不会改变任何对的值。 

什么时候`n = 2`，恰好有一个唯一差异在多重集中重复两次。 算法集`i = 0`,`j = 1`，计算一次差异，并消耗最多两个有序贡献。 这符合两者的要求$(1,2)$和$(2,1)$存在。 

什么时候`m`很大，接近$n(n-1)$，算法最终完全缩小了间隔，并且已经考虑了所有可能的贡献。 任何剩余的`m`将对应于零值贡献，这不会影响总和，即使在`m`等于完整的多重集大小。
