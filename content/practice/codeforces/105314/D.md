 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 每个测试用例都有一个项目数组。 每个项目有两个属性：一个值`a[i]`和一个类似权重的参数`b[i]`。 对于每对订购的商品`(i, j)`，我们定义一个贡献，该贡献取决于它们的按位或`a`值，乘以`b[j]`。 任务是计算每个固定的`i`, 总贡献`j`。 

所以对于每个索引`i`，我们必须评估表达式$$\sum_{j=1}^{n} (a_i \,|\, a_j)\cdot b_j$$并输出它。 

关键的难点是表达式不对称`i`和`j`因为乘数是`b[j]`， 不是`b[i]`。 这意味着我们无法在不仔细重新组织计算的情况下预先计算对称矩阵或直接重用对贡献。 

这些限制迫使我们远离任何二次推理。 总计`n`所有测试用例都可以达到 1e6，因此任何尝试显式检查所有对的方法都会失败。 一个天真的双循环就可以了$10^{12}$最坏情况下的操作，远远超出1秒的限制。 

微妙的边缘情况来自按位或结构。 因为值高达$2^{15}$，所有行为都限制在 15 位以内。 这暗示解决方案必须利用按位分解，而不是将数字视为不透明整数。 

一个常见的错误是尝试预先计算诸如前缀和之类的东西`a`或者`b`独立。 这会失败，因为 OR 不是可加的，并且不会以保留线性结构的方式分配总和。 

## 方法

 暴力解决方案直接评估定义。 对于每一个`i`，它循环所有`j`，计算`(a[i] | a[j]) * b[j]`，并将其累积起来。 这是正确的，因为它遵循字面定义。 问题是性能。 和`n`最多$2 \cdot 10^5$，每个测试用例已经需要$4 \cdot 10^{10}$在最坏的情况下进行操作，并且在测试用例中这变得不可行。 

关键的结构观察来自于在位级别上扩展“或”。 对于每个位位置，`(a[i] | a[j])`如果在其中之一中设置了该位，则贡献该位`a[i]`或者`a[j]`。 这建议将总和分成每个比特的贡献。 我们可以计算每个位出现的次数，而不是为每对重新计算 OR。`a[j]`加权为`b[j]`，并单独处理该位是否已经存在于`a[i]`。 

对于固定位`k`，该位的贡献`(a[i] | a[j])`是：

 - 总是`2^k`如果位`k`设置于`a[i]`- 否则`2^k`如果位`k`设置于`a[j]`这将问题分解为可预先计算的全局统计数据`j`。 

我们对每一位进行预先计算`k`，所有的总和`b[j]`在哪里`a[j]`已设置该位。 这让我们可以快速评估第二种情况的贡献。 第一种情况仅取决于`i`以及所有的总和`b[j]`。 

所以对于每一个`i`，我们结合：

 - 位已存在于`a[i]`，贡献全额`b[j]`每个这样的位
 - 缺少的位`a[i]`，仅贡献子集`b[j]`该位存在于哪里`a[j]`这将每个测试用例减少到 O(n * B)，其中 B 为 15。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 位分解| O(n·15) | O(15) | 已接受 |

 ## 算法演练

 1. 预先计算所有的总和`b[j]`。 这将被每个索引重用，因为每对`(i, j)`包括`b[j]`以完全相同的方式。 
2. 对于每个位位置`k`从 0 到 14，计算`sum_bit[k]`，所有的总和`b[j]`使得第 k 位`a[j]`已设置。 这隔离了每个位在所有项目中贡献的重量。 
3. 对于每个索引`i`，将答案初始化为零。 
4. 对于每一位`k`,检查是否位`k`设置于`a[i]`。 如果设置了，那么`(a[i] | a[j])`总会有这一点，无论`a[j]`，所以每一个`b[j]`做出贡献。 因此我们添加`2^k * total_b`。 
5.如果有位`k`没有设置在`a[i]`，那么 OR 只能从这些中获得这一位`j`在哪里`a[j]`已经有了。 在这种情况下我们添加`2^k * sum_bit[k]`。 
6. 输出每个的计算值`i`。 

正确性取决于独立处理每个位并将 OR 重建为独立二进制贡献的总和，加权为`b[j]`。 

### 为什么它有效

 OR 运算相当于对每个位置取按位最大值。 每个位对最终值的贡献都是独立的，其贡献仅取决于它是否出现在任一操作数中。 由于乘以`b[j]`不干扰位独立性，我们可以将总和重新分配给位。 对于每一位、每对`(i, j)`要么总是贡献该位，要么仅根据条件有条件地贡献它`a[j]`。 这消除了不同之间的任何依赖关系`j`值并确保线性聚合有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))
        
        total_b = sum(b)
        
        sum_bit = [0] * 15
        
        for i in range(n):
            x = a[i]
            bi = b[i]
            bit = 0
            while x:
                if x & 1:
                    sum_bit[bit] += bi
                x >>= 1
                bit += 1
        
        res = [0] * n
        
        for i in range(n):
            ai = a[i]
            ans = 0
            for k in range(15):
                if ai & (1 << k):
                    ans += (1 << k) * total_b
                else:
                    ans += (1 << k) * sum_bit[k]
            res[i] = ans
        
        print(*res)

if __name__ == "__main__":
    solve()
```该实现将预处理与查询构造分开。 这`sum_bit`数组累积每个位对所有位的加权贡献`j`。 第二个循环通过检查位存在来构造每个答案`a[i]`。 

一个微妙的点是我们从不直接重新计算 OR。 每次出现`(a[i] | a[j])`被独立比特贡献的总和所取代，这完全避免了二次复杂度。

 | 3 | 3 | 1 | 0,1 | 位 0 += 1，位 1 += 1 |

 对于`a[1]=1`:

 | 比特| 设置在a[i]中？ | 贡献 |

 ## 边缘情况

 对于单元素数组，表达式简化为`(a[i] | a[i]) * b[i]`，即`a[i] * b[i]`。 该算法处理这个问题是因为`total_b = b[i]`和`sum_bit`仅计算该元素中的位，因此每个设置的位都准确贡献`bit * b[i]`。 

对于所有输入`a[i]`每个人都拥有相同的价值`sum_bit[k]`要么等于`total_b`或零，具体取决于该位是否被设置。 然后，每个索引都会收到相同的贡献，并且算法会正确生成一个常量数组。 

当值具有不相交的位时，每个`sum_bit[k]`聚合不相交的索引子集。 该算法清楚地分离了这些贡献，因为每个位都是独立处理的，确保不相关位之间不会发生干扰。
