 ## 解决方案
 ## 问题理解

 我们给出了几个独立的场景。 每个场景中都有$n$电影，每部电影都有固定的票价。 目标是把所有电影都看完一次。 通常，观看电影需要支付单独的门票费用。 

但是，还有另一种购买选项：您可以支付固定金额$k$并收到$m$免费电影票。 每张这样的门票都可以用于任何电影，有效地让您“跳过”支付该电影的个人费用。 您可以多次购买此捆绑包，任何未使用的门票都可以结转。 

对于每个测试用例，任务是决定购买多少个捆绑包以及使用免费门票观看哪些电影，以便最大限度地减少总花费。 

关键的矛盾在于直接支付昂贵的电影费用和最佳使用捆绑票之间的关系。 每个捆绑包都有固定的成本和固定的容量，因此决策不是针对单个电影的本地决策，而是整个排序成本结构的全局决策。 

这些约束意味着效率要求。 所有测试用例的电影总数可以达到$10^6$，因此任何解必须接近线性或$O(n \log n)$。 任何平方项$n$会太慢，因为即使是单个测试用例也可以达到$2 \cdot 10^5$电影。 

天真的尝试会尝试所有可能的捆绑数量并模拟最昂贵电影的门票分配。 这立即变得太慢，因为每次模拟都涉及重复排序或扫描大型数组。 

一些边缘情况很重要：

 如果所有电影的成本都很小，那么购买捆绑包可能会完全浪费，因为$k$可能会超过最便宜的总和$m$电影。 

如果一部电影与其他电影相比非常昂贵，那么使用捆绑票几乎总是最佳选择，因为可以节省大量费用$a_i$证明支出合理$k$遍布$m$用途。 

如果$m = n$，一包有可能取代所有电影，所以我们比较$k$与所有成本的总和。 

朴素推理失败的一个小例子是：

 输入：$n=4, m=2, k=10$，成本：$[1, 2, 100, 101]$如果我们贪婪地购买 2 个捆绑包并自由分配，我们可能会浪费小价值的门票。 最佳解决方案显然使用 100 和 101 的票，而不是 1 和 2 的票。 

这表明分配必须始终在全球范围内优先考虑昂贵的电影。 

## 方法

 直接暴力的方法会考虑购买$x$束，其中$x$范围从 0 到$n$。 对于每一个选择$x$，我们将模拟使用$x \cdot m$购买最昂贵电影的门票，这样可以最大限度地节省费用。 我们对数组进行排序，然后将剩余成本计算为未覆盖元素的总和加上$x \cdot k$。 

这是有效的，因为一旦免费门票的数量固定，最佳策略始终是将它们应用到最大值。 然而，尝试一切$x$太慢了。 对于每个$x$，我们会重新计算前缀/后缀总和或重复重新扫描数据，导致$O(n^2)$或者$O(n^2 \log n)$行为。 

关键的观察是我们实际上不需要尝试所有数量的捆绑包。 每个捆绑包提供$m$“免费插槽”成本计算$k$。 因此，我们正在决定用每组大小的固定成本替换多少个最大的元素$m$。 该结构建议先排序一次，然后贪婪地决定应该替换多少元素。 

我们可以考虑首先在最大的元素上使用免费门票，而不是考虑捆绑。 如果我们决定使用$t$免费门票总计，那么我们至少必须支付$\lceil t / m \rceil \cdot k$，并且这些门票应始终涵盖$t$最大的电影成本。 

所以问题归结为决定最优$t$，相当于从大到小扫描排序数组可能的前缀并动态计算成本。 

我们按降序对数组进行排序。 让前缀和代表第一个的总成本$i$最大的电影。 如果我们使用$i$门票，我们支付剩余费用$n - i$直接看电影，加上$\lceil i / m \rceil \cdot k$。 

我们评估所有$i$从$0$到$n$，排序后是线性的，给出了有效的解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解捆绑 |$O(n^2)$|$O(n)$| 太慢了|
 | 排序+前缀评估|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 按降序对电影费用进行排序，以便我们始终首先考虑使用最昂贵的电影的免费门票。 与其他位置相比，这保证了门票的任何使用都会立即产生最大的节省。 
2. 计算排序数组的前缀和，以便我们可以快速评估最昂贵电影的任何子集的总成本。 
3. 迭代使用的免费门票数量，从 0 到$n$。 每个值代表一个决定：我们恰好使用那么多的门票来替换昂贵的电影。 
4.对于固定数量$i$使用过的门票，计算需要多少个捆绑包，即$\lceil i / m \rceil$。 将此乘以$k$以获得购买这些门票的成本。 
5. 计算正常支付剩余费用的成本$n - i$电影，即总和减去电影的总和$i$最大的元素。 
6. 将两个部分结合起来以获得此配置的总成本，并跟踪所有成本中的最小值$i$。 
7. 输出找到的最小值。 

这一枚举足够的原因是，任何最佳解决方案都可以转换为使用的票证始终对应于最大元素的解决方案，因为将票证从较小的元素交换到较大的元素不会增加成本，也不会破坏可行性。 

### 为什么它有效

 在任何时候，我们只关心免费门票覆盖了多少部电影，而不关心它们来自哪个捆绑包。 由于每张门票具有相同的价值，并且每部电影都是独立的，因此最佳策略始终首先将门票分配给最大的可用成本。 这创建了一个单调结构：随着使用的门票数量的增加，所覆盖的电影集在排序数组的前缀中增长。 由于捆绑成本仅取决于有多少组的大小$m$需要时，该前缀上的成本函数是明确定义的，并且可以通过检查所有前缀长度来最小化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m, k = map(int, input().split())
        a = list(map(int, input().split()))
        
        a.sort(reverse=True)
        
        prefix = [0] * (n + 1)
        for i in range(n):
            prefix[i + 1] = prefix[i] + a[i]
        
        total = prefix[n]
        ans = total
        
        for i in range(n + 1):
            bundles = (i + m - 1) // m
            cost = bundles * k + (total - prefix[i])
            if cost < ans:
                ans = cost
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对数组进行排序，以便任何前缀都对应于最昂贵的电影。 前缀和数组允许恒定时间计算顶部的总值$i$电影。 

对于每个可能的门票使用次数$i$，我们通过将门票分组为大小的块来计算需要多少个捆绑包$m$。 这就是上限划分出现的地方，因为部分使用捆绑包仍然需要支付整个捆绑包的费用。 

剩余成本就是所有未受影响的电影的总和，通过减去前缀和获得。 总体上最小的$i$就是答案。 

一个微妙的点是我们包括$i = 0$，这相当于根本不购买捆绑包，以确保捆绑包永远不会带来好处时的正确性。

 ## 边缘情况

 当$m = n$，该算法有效地将使用捆绑包与直接购买所有东西进行比较。 在迭代中，$i=n$给出一束和零直接成本，所以答案变成$\min(\text{sum}, k)$，这符合直觉。 

什么时候$k$非常大，每个计算的捆绑成本在表达式中占主导地位。 循环仍然包括$i=0$，因此算法正确地选择正常支付所有费用。 

当所有值都相同时，排序不会改变结构，但前缀评估仍然正确地探索捆绑是否值得。 成本函数变为线性$i$，并且最小值正确地出现在任一边界处，具体取决于$k$相对$m \cdot a_i$。 

什么时候$n$为 1 时，仅存在两种状态，不购买任何商品或购买一组票$m=1$。 该循环对两者进行评估，因此不需要特殊的外壳。
