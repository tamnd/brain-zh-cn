 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，都有一系列排列成行的单词序列。 任务是计算有多少对位置$(i, j)$和$i < j$满足一个简单的视觉条件：如果这些位置上的两个单词以相同的字母开头，那么它们“使艾哈迈德着迷”。 

因此，对于每个测试用例，我们本质上是扫描字符串列表并计算有多少对共享相同的第一个字符。 

约束允许最多$10^5$所有测试中每个测试用例的总单词数，并且每个单词都很短，最多长度 13。这意味着直接比较每对单词的任何解决方案在最坏的情况下都会太慢，因为它需要大约$n^2$每个测试用例的比较，大约是$10^{10}$操作时$n = 10^5$。 这远远超出了一秒钟的运行速度。 

一个微妙的边缘情况是所有单词都以相同字符开头。 例如，如果输入是：```
1
5
aaa aab aac aad aae
```每对都是有效的，所以答案是$\binom{5}{2} = 10$。 简单的解决方案可能仍然在逻辑上有效，但必须避免由于不正确的对迭代逻辑而导致重复计数或丢失对。 

另一个边缘情况是所有单词都以不同的字母开头：```
1
4
apple banana cat dog
```这里答案是零，任何分组逻辑都必须正确处理单例组。 

## 方法

 蛮力的想法很简单。 我们检查每对索引$(i, j)$, 比较第一个字符$w_i$和$w_j$，如果匹配则增加答案。 这是正确的，因为它直接评估问题中陈述的条件。 

问题在于规模。 为了$n = 10^5$, 比较次数约为$5 \times 10^9$，即使对于较小但仍然较大的输入，它也会变得太慢。 

关键的观察是只有每个单词的第一个字符很重要。 我们可以按单词的起始字母对单词进行分组，而不是成对比较单词。 如果出现一个字母$k$次作为第一个字符，那么它的贡献正好$\frac{k(k-1)}{2}$有效对。 这将问题从对枚举转换为频率计数。 

我们将问题从二次比较简化为单个线性扫描加上对 26 个可能字母的持续工作。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 频率计数 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 将大小为 26 的频率数组初始化为零，每个小写字母一个槽。 该数组将存储每个字母开头的单词数。 
2. 对于输入中的每个字，读取其第一个字符并递增相应的频率计数器。 此步骤将字符串中的所有相关信息压缩为单个整数更新。 
3. 处理完所有单词后，迭代 26 个字母。 对于每个频率$f$，使用公式计算该字母贡献的有效对的数量$f \times (f - 1) / 2$，并将其添加到答案中。 
4. 输出测试用例的累计和。 

### 为什么它有效

 该算法依赖于根据第一个字符将索引划分为不相交的组。 每个有效对必须恰好来自一个这样的组，并且组内的每个对都是有效的。 由于组不重叠，因此对每个组内的组合求和只会对每个有效对进行一次计数，并且不会包含任何无效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        words = input().split()
        
        freq = [0] * 26
        
        for w in words:
            freq[ord(w[0]) - 97] += 1
        
        ans = 0
        for f in freq:
            ans += f * (f - 1) // 2
        
        print(ans)

if __name__ == "__main__":
    solve()
```代码独立处理每个测试用例。 关键的实现细节是仅访问第一个字符，因此我们避免了不必要的字符串处理。 每个测试用例都会重置频率数组，以避免用例之间的泄漏。 

读取所有输入后应用配对计数公式，这确保我们不会意外计数部分信息。 

## 工作示例

 ### 示例 1

 输入：```
1
5
apple ant axe bat ball
```我们通过第一个字母跟踪频率：

 | 步骤| 词| 第一个字符 | 频率阵列（部分） |
 | ---| ---| ---| ---|
 | 1 | 苹果| 一个 | a = 1 |
 | 2 | 蚂蚁| 一个 | a = 2 |
 | 3 | 斧头| 一个 | a = 3 |
 | 4 | 蝙蝠 | 乙| a = 3，b = 1 |
 | 5 | 球 | 乙| a = 3，b = 2 |

 现在计算对：

 - 对于“a”：$3 \cdot 2 / 2 = 3$- 对于“b”：$2 \cdot 1 / 2 = 1$总计 = 4。 

这展示了分组如何将配对计数转换为简单的组合。 

### 示例 2

 输入：```
1
4
cat dog fish goat
```| 步骤| 词| 第一个字符 | 频率阵列|
 | ---| ---| ---| ---|
 | 1 | 猫 | c | c = 1 |
 | 2 | 狗| d | c = 1，d = 1 |
 | 3 | 鱼 | f | c=1, d=1, f=1 |
 | 4 | 山羊 | 克| c=1, d=1, f=1, g=1 |

 所有频率均为 1，因此每个贡献都为零。 

这证实了单例组不会产生配对。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n)$| 每个单词都会处理一次，并且仅使用其第一个字符 |
 | 空间|$O(1)$| 频率数组的大小固定为 26，无论输入如何 |

 该解决方案很容易满足限制，因为总$n \le 10^5$，使工作整体呈线性。

 ## 边缘情况

 对于像这样的单个单词测试用例`1 a`，频率数组包含一个非零项 1。组合公式给出$1 \cdot 0 / 2 = 0$，所以输出是正确的。 

对于所有单词都具有相同首字母的情况，例如`a aa aaa aaaa`，“a”的频率变为 4。 该算法计算$4 \cdot 3 / 2 = 6$，匹配无序索引对的数量。 

对于完全多样化的集合，例如`a b c d e`，每个频率都是 1，因此所有贡献都为零，输出保持为 0，正确地避免了意外的交叉字母配对。
