 ## 解决方案
 ## 问题理解

 我们给出了长度的排列$n$，我们解释任何一对索引$(l, r)$作为与段中的值集相对应的“碎片”$a_l, a_{l+1}, \dots, a_r$如果$l \le r$。 如果$l > r$，该碎片代表一个空集。 

我们可以选择两个碎片，取两个结果集的并集，如果这个并集形成整数的“完美区间”，这意味着它恰好包含从最小值到最大值的所有整数，没有间隙，没有重复，那么这个并集被称为超级碎片。 任务是计算有多少个不同的集合$C$可以作为两个片段的并集来获得。 

因为数组是一个排列，所以每个值$1$到$n$恰好出现一次，因此每个碎片都对应于选择由子数组引起的这些值的子集。 

关键输出不是关于段对，而是关于不同的结果集$C$。 如果两个不同的对产生相同的集合，则它们被视为相同的答案。 

约束总和$n$所有测试用例最多是$10^4$，而每个$n$取决于$5000$。 这强烈表明$O(n^2)$或者每个测试用例稍微超二次的解决方案是可以接受的，但是$O(n^3)$或者所有段对的简单枚举都不是。 

天真的解释会考虑所有$O(n^2)$段和所有段对，导致$O(n^4)$组合，这是不可能的。 

一个更微妙的问题是重复计算：多个段对可以生成相同的并集，因此任何正确的方法都必须计算有效并集的规范表示，而不是对本身。 

边缘情况出现在以下情况：

 1. 一个分片为空。 那么答案就简化为所有有效的单段“完美间隔”。 

例子：$a = [1,2,3]$。 任何子数组都已经是连续的区间，因此所有单个段都是有效的输出。 
2. 段在值中重叠，但在索引中不重叠，这在集合的排列意义上是不可能的，但在推理中很重要：重叠的索引段并不意味着重叠的值间隔。 
3. 即使两个段在值空间中都不连续，两个段也可以组合形成有效区间，因此简单的“好段”分类会失败。 

## 方法

 蛮力策略很简单。 我们枚举所有子数组$A$和所有子数组$B$，计算它们的值集，并取它们的并集。 然后我们检查结果集是否形成连续区间。 每个子数组可以表示为$O(1)$如果我们维护频率数组或集合，则更新，但合并两个集合仍然是$O(n)$。 这导致大约$O(n^4)$最坏情况下每个测试用例的总工作量，因为有$O(n^2)$子数组并组合每对是线性的。 

这太慢了，但它揭示了问题的结构：对于一个集合来说，重要的只是它的最小值和最大值以及它是否完整。 因此，我们不想跟踪完整的集合，而是想根据值的间隔进行推理。 

关键的观察是任何有效的超级碎片都对应于选择一个值区间$[x, y]$，我们只需要检查是否可以使用最多两个不相交的索引段覆盖该区间内的所有值。 由于值是一种排列，因此$x, x+1, \dots, y$都是不同的，我们问的是它们是否可以被最多两个连续的索引块覆盖。 

这将问题简化为值位置上的区间几何。 

我们映射每个值$v$到它的位置$pos[v]$。 对于候选值区间$[l, r]$，我们看一下位置集$\{pos[l], \dots, pos[r]\}$。 这些位置在一条线上形成一个集合，我们询问它们是否可以被最多两个按索引顺序的连续线段覆盖。 这相当于检查这组位置在排序时是否最多有两个“间隙”。 

现在问题变成：统计所有值区间$[l, r]$使得这些值的位置最多形成两个连续的块。 

我们可以修复$l$并扩展$r$，维持最小和最大位置并跟踪这些位置形成了多少个不相交的段。 自从$n$足够小，我们可以维护一个在摊余常数时间内更新段计数的结构。

 ## 算法演练

 我们完全在价值空间中重新构建问题。 

### 1.构建位置数组

 我们计算$pos[v]$，索引值所在$v$出现在排列中。 这让我们可以用位置集替换值区间。 

这是必要的，因为必须通过索引空间中的结构来检查值空间中的连续性。 

### 2. 迭代值区间的左端点

 我们固定一个起始值$l$并逐步延伸$r$从$l$到$n$。 

在每一步中，我们都会考虑值集$\{l, l+1, \dots, r\}$。 

### 3. 维持一组仓位

 当我们添加新值时$r$，我们插入$pos[r]$成动态有序结构（使用平衡结构或带有邻接检查的排序列表实现）。 

我们维护由这些位置形成的连续段的数量。 

新职位可以是：

 - 如果两个邻居都不存在，则开始一个新段，
 - 如果两个邻居都存在，则合并两个段，
 - 如果恰好存在一个邻居，则扩展一个段。 

我们保留一个分段计数器。 

### 4. 检查有效性条件

 对于每个间隔$[l, r]$，我们检查位置是否形成最多两个连续的线段。 

如果是，则该值区间可以形成为两个分片的并集。 

我们算一下。 

### 5. 累积答案

 我们总结所有有效的$(l, r)$所有的间隔$l$。 

### 为什么它有效

 任何有效的超碎片都对应于某个值区间$[l, r]$，因为联合在价值空间中必须是无间隙的。 由于值是一种排列，每个值都出现一次，因此值空间中的任何间隙都对应于缺失的元素，这将违反“无间隙”条件。 

因此，整个问题简化为检查这些值的位置是否可以被最多两个连续的索引段覆盖。 动态维护段计数可确保我们正确跟踪两个碎片是否足以覆盖该集合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    pos = [0] * (n + 1)
    for i, v in enumerate(a):
        pos[v] = i

    ans = 0

    for l in range(1, n + 1):
        used = [False] * n
        segs = 0

        for r in range(l, n + 1):
            p = pos[r]
            used[p] = True

            left = p > 0 and used[p - 1]
            right = p + 1 < n and used[p + 1]

            if left and right:
                segs -= 1
            elif not left and not right:
                segs += 1

            if segs <= 2:
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现遵循扩展值区间的思想$[l, r]$并跟踪相应位置形成了多少个连续块。 

数组`used`标记当前包含哪些位置。 插入新位置时，我们仅检查其直接邻居以确定段结构如何变化，因为一维数组中的连通性仅取决于邻接性。 

变量`segs`跟踪在诱导的位置集中存在多少个连接的组件。 如果添加一个新点连接两个组件，我们就会减少计数； 如果它创建了一个新的隔离组件，我们就会增加它。

 ## 边缘情况

 对于$n = 1$，算法初始化单个位置并立即将其计为有效间隔。 段数为1，满足约束。 

对于单调排列，例如$[1,2,3,\dots,n]$，每个值区间对应一个连续的索引块，因此段计数永远不会超过1。该算法对所有$\frac{n(n+1)}{2}$间隔正确。 

对于高度交错的排列，例如$[1,n,2,n-1,\dots]$，位置振荡，但基于邻接的段更新仍然正确地合并和分割组件，确保只有结构上有效的间隔对答案有贡献。
