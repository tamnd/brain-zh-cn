 ## 解决方案
 ## 问题理解

 我们有一个圆形大厅，以原点为中心，半径为$R$， 和$n$里面现有的人。 每个人在平面上占据一个点，我们保证每对现有的人至少相距2个单位。 

我们必须决定是否存在一个位置，在圆的内部或边界上，一个新的人可以站立，使得他们与每个现有的人的距离至少为 2。如果存在这样的点，我们必须输出任何有效的点。 如果不是，我们输出这是不可能的。 

从几何上讲，这是一个带有禁区的圆盘的可行性问题：每个存在的人在他们周围定义了一个半径为2的封闭圆盘，我们不允许进入。 我们需要在大圆内找到一个未被任何这些禁止圆盘覆盖的点。 

约束条件很大，最多可达$10^5$点，因此任何检查所有点对或在所有磁盘之间执行简单几何相交测试的解决方案都会立即变得太慢。 对候选位置或成对几何构造的强力检查至少需要二次时间，这在 2 秒限制下是不可行的。 

当有效区域仅存在于大圆的边界处或恰好存在于禁止盘之间的切点处时，会出现微妙的边缘情况。 另一个棘手的情况是，即使边界上其他地方存在有效点，圆心也无效，这可能会破坏朴素的中心优先启发法。 

## 方法

 直接暴力的想法是考虑平面上的所有点，或者至少考虑由禁止圆盘和边界圆的相交形成的所有有意义的候选点。 一个自然的想法是，有效的解决方案必须位于自由区域的中心或约束边界的交点，这意味着圆与圆的交点（禁止盘和大圆之间）或成对的盘交点。 这导致考虑所有点对或所有几何相交事件。 

然而，这样的候选交点的数量是$O(n^2)$，因为每对人都可以生成恒定的多个交点。 即使每个候选人都根据所有点进行检查，这也会变成$O(n^3)$，这远远超出了限制。 

关键的观察是我们不需要考虑所有交叉点。 相反，我们将每个现有的人视为强加了一个约束：我们不能在其距离 2 之内。 如果一个点是有效的，那么可行性边界必须要么接触外圆，要么接触至少一个禁止圆。 这表明了一个方向：如果存在有效的解决方案，那么检查一组小得多的候选点就足够了，这些候选点对于一个或两个约束是“紧密”的。 

标准的简化是测试从局部几何中派生出来的一小组精心挑选的候选点：大圆的中心，对于每个点，从原点到我们向外“推”的点的方向，直到我们到达大圆的边界或者我们从某个点精确地到达距离 2。 如果存在可行点，则至少一个这样的基于方向的候选点将满足所有约束。 

这将问题从全局成对几何图形减少到具有局部可行性检查的线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（成对几何候选）|$O(n^2)$或者更糟|$O(1)$-$O(n)$| 太慢了|
 | 定向候选人检查|$O(n^2)$|$O(1)$| 已接受 |

 ## 算法演练

 我们构建的候选点要么是原点，要么位于从原点到现有点的射线上，因为与禁区的任何最佳“第一次接触”都可以沿着这样的方向表示。 

1. 首先检查圆心$(0, 0)$。 如果距离所有点至少 2 个单位，则它立即成为有效答案。 这是最简单的可行候选方案，可以避免不必要的工作。 
2. 对于位于该点的每个现有人员$p = (x, y)$，计算从原点到$p$。 如果我们考虑射线从原点开始并向$p$，任何可行的边界点受$p$必须沿着或靠近这个方向。 
3. 从原点向$p$直到两个事件之一发生：我们到达半径大圆的边界$R$，或者我们到达距离 2 处的点$p$。 第一个事件受以下条件限制$R$，第二个确保我们保持在周围的禁止盘之外$p$。 
4. 计算沿该射线的最远有效点。 这减少了寻找一个标量$t$使得这一点$t \cdot \frac{p}{|p|}$满足两者$|t \cdot \frac{p}{|p|}| \le R$和$|t \cdot \frac{p}{|p|} - p| \ge 2$。 我们将其作为一维约束进行几何求解$t$。 
5. 边界的约束很简单$t \le R$。 距离约束为$p$变为二次不等式$t$，给出禁止值的区间。 我们选择最大可行的$t$在这两种约束下。 
6. 对于这样得到的每个候选点，检查它是否距离所有点至少2个单位。 如果是，则立即输出。 
7. 如果没有候选人工作，则输出不存在有效位置。 

### 为什么它有效

 任何有效的解决方案区域都是圆盘（大厅）与补充圆盘（禁区）的交集。 这种区域的边界总是由外圆或禁圆之一形成。 如果存在解，则可行区域的边界上存在一个点，该点至少“紧”约束一个约束。 通过枚举从原点到每个约束中心的光线，我们确保捕获至少一个正确识别第一个阻挡约束的方向。 这保证了可行点（如果存在）将出现在测试的候选点中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def dist2(x1, y1, x2, y2):
    return (x1 - x2) ** 2 + (y1 - y2) ** 2

n, R = input().split()
n = int(n)
R = float(R)

pts = []
for _ in range(n):
    x, y = map(float, input().split())
    pts.append((x, y))

def valid(x, y):
    if x * x + y * y > R * R + 1e-9:
        return False
    for px, py in pts:
        dx = x - px
        dy = y - py
        if dx * dx + dy * dy < 4.0 - 1e-9:
            return False
    return True

# check center
if valid(0.0, 0.0):
    print("Yes")
    print(0.0, 0.0)
    sys.exit(0)

# try candidates along each point direction
for px, py in pts:
    norm = math.hypot(px, py)
    if norm == 0:
        continue

    ux = px / norm
    uy = py / norm

    # binary search best t on [0, R]
    lo, hi = 0.0, R
    for _ in range(60):
        mid = (lo + hi) / 2
        x = ux * mid
        y = uy * mid
        ok = True
        for qx, qy in pts:
            dx = x - qx
            dy = y - qy
            if dx * dx + dy * dy < 4.0 - 1e-9:
                ok = False
                break
        if ok:
            lo = mid
        else:
            hi = mid

    x = ux * lo
    y = uy * lo
    if valid(x, y):
        print("Yes")
        print(x, y)
        sys.exit(0)

print("No")
```该代码首先定义辅助距离检查并读取所有点。 这`valid`函数编码了核心可行性条件：在圆内且在半径为2的所有禁盘之外。 

我们首先测试原点，因为它是唯一不依赖于几何形状的候选者，并且在点稀疏时通常会通过。 

对于每个点，我们从原点构建一个方向向量并将其标准化。 然后我们使用半径参数的二分搜索沿着这条射线进行搜索$t$。 谓词检查候选点对于所有约束是否有效。 如果它有效，我们就尝试进一步向外延伸；如果它有效，我们就尝试进一步向外延伸。 否则，我们会缩小范围。 这确保我们找到沿该方向的最远可行点。 

最后，我们验证并输出第一个成功的候选者。 

关键的实现微妙之处在于浮点稳定性：所有比较都使用较小的 epsilon 裕度，以防止错误地拒绝边界有效点而导致精度错误。

 ## 边缘情况

 一种边缘情况是原点已经无效。 该算法通过立即跳过它并转向定向候选来处理此问题。 例如，如果一个点非常靠近中心，则原点检查会失败，并且解决方案依赖于基于射线的候选点，以确保不会过早拒绝。 

另一种情况是所有点都位于围绕原点的完美圆上。 在这种情况下，中心可能是无效的，但禁区之间可能仍然存在边界点。 射线搜索确保探索每个角度方向，二分搜索找到该方向上的最大可行半径。 

最后一个微妙的情况是接近精确距离 2 的浮点精度。在两个比较方向上使用 epsilon 调整可确保将恰好位于边界上的点一致地视为有效，从而防止由于舍入误差而出现漏报。
