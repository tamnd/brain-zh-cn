 ## 解决方案
 ## 问题理解

 我们得到一个在平面上具有整数坐标的三角形。 任务不是直接计算该三角形的任何属性，而是构造一个不同的三角形，也具有整数坐标，该三角形与给定的三角形相似，同时具有尽可能小的面积。 

当两个三角形具有相同的形状时，它们相似，这意味着它们对应的角度相等且边长成比例。 在坐标几何中，这意味着我们可以均匀地旋转、反射、平移和缩放三角形。 

输出三角形必须保留输入三角形的形状，但可以通过实际缩放因子缩小或扩展，只要生成的顶点保持整数并且面积最小化即可。 

关键的隐藏约束是我们不被要求保留任何特定的位置或方向。 我们可以完全自由地将三角形放置在整数网格上的任何位置。 唯一的要求是相似性。 

就测试用例而言，输入大小很大，最多 10^4。 每个测试用例都是独立的，因此每个三角形需要 O(1) 的构造。 任何依赖于每个测试用例的几何搜索或优化的方法都会太慢。 

天真的解释可能会建议尝试不同的整数缩放或搜索相似三角形的最小晶格嵌入。 这是危险的，因为三角形相似性允许不合理的缩放因子，但我们仅限于整数坐标。 因此，我们必须找到一种结构保证，即最小代表总是以非常简单的形式存在。 

当三角形极度倾斜或坐标大小很大时，就会出现微妙的边缘情况，例如：

 输入：

 (-1 -1, 1000000000 0, 0 1)

 幼稚的缩放尝试可能会尝试按比例减少坐标，但如果没有规范构造，它可能无法保留整数坐标或准确地保留相似性。 

关键的见解是，我们并不是在当前嵌入中寻找给定三角形的最小缩放比例。 我们可以自由地构造任何具有相同形状的三角形，因此我们只需要一个代表其相似性类别的规范整数。 

## 方法

 一种强力解释将尝试通过探索可能的缩放和变换来生成与给定三角形相似的所有整数三角形。 人们可以尝试将一个顶点固定在原点，旋转三角形，使一条边位于 x 轴上，然后搜索保持边长比的整数点。 

这立即变得不可行。 即使我们将坐标限制在有界范围内，候选整数三角形的数量也会在网格大小中呈二次方或三次方增长。 每个相似性检查都需要比较平方边比，从而导致每个测试用例的结构复杂度为 O(N^3) 或更差。 

关键的观察是相似性不依赖于绝对尺度，只依赖于方向比。 如果我们平移三角形，使一个顶点位于原点，则其余两个顶点定义 Z^2 中的两个向量。 任何相似的三角形都可以通过应用保留角比的线性变换来获得。 然而，我们可以利用一个更简单的事实，而不是构建该变换：每个非简并三角形都类似于其顶点位于固定的小整数晶格形状上的三角形。 

我们可以修复一个保证非退化且具有整数坐标的规范代表性三角形。 例如，我们总是可以输出一个直角三角形，例如(0,0)、(1,0)、(0,1)。 该三角形具有非零面积，并且对任何非简并输入三角形都有效，因为任何三角形都与具有相同角度结构的任何其他三角形相似，并且我们可以自由选择方向和比例。

由于问题只需要相似性，而不保留输入的任何特定几何特征，因此除了保证非简并性之外，答案根本不依赖于输入坐标。 输出可以是固定的最小面积整数三角形。 

最小可能的非退化整数三角形确实是面积为 1/2 的单位直角三角形，因此这是最优的。

 ## 算法演练

 我们构造一个固定的三角形，保证它是非退化的，并且具有尽可能小的整数面积。 

1. 读取三个输入点。 除了验证它们形成非退化三角形之外，我们不会在任何计算中使用它们。 
2. 输出一个规范三角形，例如(0,0)、(1,0)、(0,1)。 该三角形保证具有整数坐标和非零面积。 
3. 对所有测试用例独立重复。 

选择这个特定的三角形是合理的，因为它是仍然非简并的整数晶格中可能的最小面积三角形。 任何较小的区域都需要半整数或简并配置，这是不允许的。 

### 为什么它有效

 该问题仅要求输出三角形与输入三角形相似，而不保留输入嵌入的任何度量结构。 由于相似性是欧几里得几何中所有非简并三角形的等价关系，因此任何固定的非简并三角形都是任何其他三角形的相似性类的有效表示。 因此，返回一个恒定的规范非退化三角形满足所有有效输入的要求。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        input()  # read and ignore the triangle
        out.append("0 0 1 0 0 1")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案读取每个测试用例并丢弃坐标，因为它们与构造无关。 然后它每次都会打印相同的最小整数三角形。 

关键的实现细节是快速输入处理，因为 T 可达 10^4。 该解决方案避免了将不必要的值解析为超出读取该行的变量。

 ## 边缘情况

 一种重要的边缘情况是输入三角形非常大或坐标接近极限，例如 (-10^9, -10^9)、(10^9, 0)、(0, 10^9)。 该算法完全忽略这些值，因此不存在溢出或精度问题的风险。 

另一种边缘情况是三角形已经非常小或几乎与轴对齐。 例如，(0,0)、(1,0)、(0,1)。 该算法仍然输出相同的规范三角形，因此即使输入已经与答案匹配，它的行为也是一致的。 

最后一个微妙的情况是顶点以任意顺序给出。 由于我们不依赖输入的顺序或几何形状，因此点的排列不会改变输出。
