 ## 解决方案
 ## 问题理解

 我们在整数网格上有两个点，每个点充当圆形影响的中心。 在每个中心周围，给定欧几里得半径内的每个晶格点的值都通过乘以 -1 进行翻转。 每个网格点最初都包含一些正数，但唯一重要的是它是否被翻转奇数次或偶数次。 在两次循环操作之后，当单元恰好位于两个圆盘之一内时，该单元恰好为负值。 

因此，任务简化为一个几何问题：找到两个圆之一内的任何整数格点，或者报告不存在这样的点。 答案坐标也必须位于最大 3×10⁹ 的绝对值范围内。 

约束很大，坐标和半径高达 10⁹。 这排除了任何枚举圆内网格点或迭代边界框中所有整数点的方法，因为即使单个圆也可以覆盖大约 r² 网格点，这远远超出了可行的限制。 

当两个圆圈几乎相同或一个圆圈完全包含在另一个圆圈内时，就会出现一种微妙的故障模式。 在这种情况下，可能不存在奇数覆盖范围的点。 

例如，如果两个中心重合且半径相等，则每个点都会翻转两次或零次，因此不存在有效点，答案必须为 -1 -1。 总是尝试在边界附近选取一个点而不检查重叠结构的幼稚策略会错误地输出圆内的内容。 

另一种边缘情况是近乎完全包含：如果一个圆严格位于另一个圆内部，则受较小圆影响的每个点也会受到较大圆的影响，因此同样不存在单独覆盖的区域。 

## 方法

 一个蛮力的想法是迭代包含两个圆的边界框中的所有整数点，检查每个点是否正好位于一个圆内，然后返回第一个有效的点。 原则上这是正确的，因为每个有效答案都必须位于该区域内，但边界框的边长最大为 2×10⁹，给出大约 10^8 个候选点。 即使我们限制为圆，半径为 r 的圆内的整数点数也是 θ(r²)，这对于直接检查来说太大了。 

关键的观察是我们不需要任何特定的点，只需要两个圆盘的对称差中存在一个点。 两个凸区域的对称差要么是空的，要么包含边界相邻结构。 我们可以只搜索从几何导出的有限候选点集，而不是在区域内搜索。 

关键的简化是，如果存在有效点，则必须在至少一个圆的边界上或非常靠近边界存在一个有效点。 更准确地说，覆盖范围的任何变化仅在跨越圆边界时发生，因此可以在圆心附近的点和整数方向上的一些极值偏移之间找到有效答案。 

这导致了一种建设性的方法：我们尝试从中心导出的一小组候选点，并且对于每个候选点，我们验证它是否恰好位于一个圆中。 如果都不起作用，我们可以放心地得出对称差为空的结论，这种情况只有当一个圆包含在另一个圆中或者它们相同时才会发生。 

因为圆之间的包含可以通过中心和半径之间的距离来检查，所以我们可以明确地检测空的情况，否则通过从一个中心向另一个中心移动半径差来构建边界见证点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(R²) | O(1) | O(1) | 太慢了|
 | 几何构造 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

我们将每个圆视为欧几里得空间中的圆心和半径。 设圆为 C1 和 C2。 

1. 计算中心之间的平方距离。 这避免了浮点精度问题并将所有内容保持为整数。 平方形式就足够了，因为所有比较都是单调的。 
2. 检查一个圆是否完全包含在另一个圆中。 如果距离(center1, center2) + r1 ≤ r2，则圆 C1 在 C2 内部。 如果这在任一方向上都成立，则较小圆覆盖的每个点也会被较大圆覆盖，因此没有点具有奇数覆盖范围。 
3. 如果包含在任一方向上成立并且圆圈相同或嵌套，则输出 -1 -1。 这可以处理对称差为空的所有情况。 
4. 否则，我们知道圆不是嵌套的，因此它们的对称差非空。 我们现在构造一个候选点。 我们采用从 center1 到 center2 的向量，在概念上对其进行归一化，然后从 center1 向 center2 移动距离 r1 + 1 或整数近似中的 r1。 目标是降落在圆 1 之外，但仍足够接近以可能保持在圆 2 之外，反之亦然，具体取决于不对称性。 

一个更简单的确定性构造是尝试围绕每个中心的一组固定的偏移量，例如曼哈顿方向上的点或半径边界处的轴对齐方向上的点，并针对两个圆进行测试。 

1. 对于每个候选点，检查其到每个中心的距离平方是否≤r²。 如果正好在一个圆内，则立即返回。 
2. 如果没有候选有效（这只发生在已排除的简并对称情况下），则输出 -1 -1。 

### 为什么它有效

 关键的不变量是，具有不同覆盖范围的任何点都必须位于两个磁盘的对称差异中，并且如果磁盘不是嵌套的，则该区域是非空的，并且必须与一组恒定大小的边界相邻整数候选相交。 由于圆成员资格仅在跨越边界时发生变化，并且边界是凸的且连续的，因此离散网格必须在中心之间的极值方向附近包含见证人。 遏制检查消除了由于完全重叠而使推理失败的所有情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def inside(x, y, cx, cy, r2):
    dx = x - cx
    dy = y - cy
    return dx*dx + dy*dy <= r2

def solve():
    x1, y1, r1 = map(int, input().split())
    x2, y2, r2 = map(int, input().split())

    d1 = (x1 - x2) ** 2 + (y1 - y2) ** 2

    # containment checks
    if (x1 == x2 and y1 == y2 and r1 == r2):
        print(-1, -1)
        return

    if d1 <= (r1 - r2) ** 2 and r1 >= r2:
        print(-1, -1)
        return
    if d1 <= (r2 - r1) ** 2 and r2 >= r1:
        print(-1, -1)
        return

    candidates = [
        (x1 + r1, y1),
        (x1 - r1, y1),
        (x1, y1 + r1),
        (x1, y1 - r1),
        (x2 + r2, y2),
        (x2 - r2, y2),
        (x2, y2 + r2),
        (x2, y2 - r2),
    ]

    for x, y in candidates:
        c1 = inside(x, y, x1, y1, r1 * r1)
        c2 = inside(x, y, x2, y2, r2 * r2)
        if c1 ^ c2:
            print(x, y)
            return

    print(-1, -1)

if __name__ == "__main__":
    solve()
```该解决方案首先通过平方距离对圆隶属度进行编码，以避免浮点错误。 包含检查明确消除了不存在对称差异的情况，这对于正确性至关重要。 

候选生成步骤使用每个圆周围的轴对齐边界点。 这是可行的，因为如果一个点恰好存在于一个圆盘中，则从中心起至少一个边界方向必须通向独占区域，并且这些极值点捕获这些转变。 

异或条件`c1 ^ c2`直接编码“恰好在一个圆内”。

 ## 边缘情况

 当两个圆完全重合时，每个点都会翻转两次或零次。 该算法通过对中心和半径的相等检查来尽早捕获这一点，确保不会尝试进行候选搜索。 

当一个圆严格位于另一个圆内时，距离平方与半径差的比较可识别包含情况。 该算法立即输出 -1 -1，防止错误的边界猜测，否则会返回仍在两个圆内的点。 

当圆相距较远时，包含失败，并且任一中心周围的候选点立即位于另一个圆之外，从而产生有效的不对称点，而无需探索任何内部区域。
