 ## 解决方案
 ## 问题理解

 我们有$n$足球运动员。 最初，每个玩家$i$有一件编号的衬衫$i$，根据序列做标签$1,2,\dots,n$。 

变更后球衣号码$1$被替换为$n+1$，因此可用的衬衫号码集变为$2,3,\dots,n+1$。 这些$n$新数字必须作为排列分配$p$超过头寸$1..n$，其中位置$i$对应原来拥有号码的玩家$i$。 

一名球员$i$如果他的新球衣号码接受新任务$p[i]$可以被他的旧号码整除$i$。 任务是计算有多少种排列$2..n+1$每个位置都满足这个整除条件。 

约束允许最多$10^4$测试用例和总数$n$最多$10^5$。 这立即排除了任何显式构造或搜索排列的方法。 甚至$O(n \sqrt n)$在最坏的情况下，每个测试用例都会太慢。 该解决方案必须将每个测试用例简化为类似的内容$O(1)$或者至多$O(\log n)$。 

一个微妙的一点是，大多数职位都受到很大的限制。 对于大型$i$, 的倍数$i$里面$2..n+1$很少。 天真的贪婪分配会失败，因为一个位置的本地选择可能会删除另一个位置的唯一有效值。 

边缘情况出现时$n$很小，或者当可分结构很紧时。 例如，当$n=1$，唯一可能的分配是微不足道的。 什么时候$n=3$, 位置之间的相互作用$1,2,3$和价值观$2,3,4$已经表明可以存在多个有效的排列，但仍然以非常结构化的方式存在。 

## 方法

 蛮力方法会尝试所有排列$2..n+1$，检查每个位置是否$i$接收一个可除以的值$i$。 这涉及到$n!$排列，以及每次检查成本$O(n)$，导致$O(n \cdot n!)$，即使对于$n=10$。 

更结构化的蛮力会尝试通过修剪进行回溯。 对于每个位置$i$，我们尝试所有未使用的倍数$i$。 虽然这在一定程度上减少了分支，但可分图在小索引附近仍然很密集，尤其是在位置$1$，它可以接受所有值。 在最坏的情况下，这仍然会导致指数行为。 

关键的观察结果是，除了一个位置之外，所有位置的行为几乎都是严格的。 对于每一个$i > 1$，值$i$本身总是有效的，因为$i \mid i$，因此所有位置的身份分配始终一致$i \ge 2$。 破坏这种结构的唯一因素是价值$1$被删除并替换为$n+1$，这恰好引入了一个额外的“灵活”元素。 

这意味着整个系统的行为就像一个稳定的身份排列$2..n$，只有一个涉及位置的交互作用$1$和额外的价值$n+1$。 由此，所有有效的配置都归结为选择如何$n+1$与一个位置交换一起使用，其他一切保持不变。 

问题归结为计算有多少个约数$n+1$可以作为该交换点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n \cdot n!)$|$O(n)$| 太慢了|
 | 基于除数的计数 |$O(\sqrt{n})$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 核心思想是描述所有有效排列的特征，而不需要显式地构造它们。 

### 1. 修复除 1 之外的所有位置的结构

 对于每一个$i \ge 2$, 分配$p[i] = i$始终有效，因为每个数字都可以整除自己。 这形成了一个基线配置，它已经满足除了可能在位置之外的所有约束$1$。 

该基线使用所有值$2..n$正确，只留下价值$n+1$未使用。 

### 2.了解位置1的作用

 职位$1$除了“一切都可以被 1 整除”之外，没有任何整除性限制，因此它可以取任何剩余值。 这是唯一可以吸收通过替换而引入的灵活性的位置$1$和$n+1$。 

### 3. 考虑将位置 1 与另一个索引交换

 如果我们分配$p[1] = k$，然后值$k$被从其自然位置移开。 保持有效性而又不会过多干扰其他位置的唯一方法是放置$n+1$就位$k$， IE。$p[k] = n+1$，同时保持所有其他位置固定。 

这在之间创建了一个干净的2个周期$1$和$k$，而其他一切仍然是同一性。 

### 4. 检查此交换的可行性

 为了使分配在该位置有效$k$，我们需要：$$k \mid (n+1)$$因为$p[k] = n+1$。 

因此，每个有效的配置都与选择除数完全对应$k$的$n+1$和$1 \le k \le n+1$。 

### 5. 计算所有有效选项

 每个除数$k$的$n+1$给出了一种有效的配置，但是$k=1$对应于放$1$就位$1$，这是不可能的，因为价值$1$不可用。 

所以答案是：$$\text{number of divisors of } (n+1) - 1$$### 为什么它有效

 所有职位$i \ge 2$已经满足身份分配下的约束。 使用引入的唯一干扰$k \neq n+1$本地化：交换$1$和$k$仅影响位置$k$，并且可行性简化为单一可分条件$k \mid (n+1)$。 无法形成更大的循环，因为除此之外没有第二个“额外自由度”$n+1$，因此系统无法支持跨多个指数的独立调整。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXN = 100000 + 5

# smallest prime factor sieve
spf = list(range(MAXN))
for i in range(2, int(MAXN ** 0.5) + 1):
    if spf[i] == i:
        for j in range(i * i, MAXN, i):
            if spf[j] == j:
                spf[j] = i

def count_divisors(x):
    res = 1
    while x > 1:
        p = spf[x]
        cnt = 0
        while x % p == 0:
            x //= p
            cnt += 1
        res *= (cnt + 1)
    return res

t = int(input())
for _ in range(t):
    n = int(input())
    x = n + 1
    print(count_divisors(x) - 1)
```该解决方案依靠预先计算的最小质因数筛来快速分解数字。 对于每个测试用例，我们计算$n+1$使用其质因数分解并减一以排除无效除数$1$。 

减法是必不可少的：它消除了不可能的情况$p[1] = 1$，由于值$1$这套新衬衫中不存在。

import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    MAXN = 100000 + 5
    spf = list(range(MAXN))
    for i in range(2, int(MAXN ** 0.5) + 1):
        if spf[i] == i:
            for j in range(i * i, MAXN, i):
                if spf[j] == j:
                    spf[j] = i

    def count_divisors(x):
        res = 1
        while x > 1:
            p = spf[x]
            cnt = 0
            while x % p == 0:
                x //= p
                cnt += 1
            res *= (cnt + 1)
        return res

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append(str(count_divisors(n + 1) - 1))
    return "\n".join(out)

 ## 边缘情况

 当$n=1$，系统退化到具有设定值的单个位置$\{2\}$。 唯一有效的分配是微不足道的，对应于$n+1=2$排除后只有一个有效的除数选择$1$。 

什么时候$n$是素数减一，比如说$n+1 = p$，约数的数量恰好是$2$。 移除后$1$，只剩下一种配置，它对应于类身份分配。 该算法自然地处理这个问题，因为除数计数公式产生$2 - 1 = 1$。 

什么时候$n+1$是高度复合的，有效配置的数量增加，但每个仍然对应于涉及位置的单个交换$1$，因此选择之间不会发生干扰。
