 ## 解决方案
 ## 问题理解

 我们得到一个仅由数字 2、3、4 和 5 组成的字符串，它代表学校日志中的一系列成绩。 我们可以修改任何字符，但只能增加其值，而不能减少它。 例如，2 可以变成 3、4 或 5，3 可以根据中间步骤变成 5、4 或 5，但关键限制是单调递增。 

执行任意次数的此类升级后，我们希望最大化子字符串“54”在结果字符串中出现的次数。 出现次数以通常的重叠方式进行计数，因此单个位置可能会导致多次出现。 

重要的结构性限制是每个角色的价值只能向上移动。 这使得问题不对称：我们可以强制更多的 5 和更多的 4，但永远不会减少 5 或 4 来解决冲突。 

输入大小没有明确说明，但此类单个字符串问题的典型 Codeforces 约束意味着最多大约 2·10^5 个字符。 这立即排除了所有子串或所有位置对的二次方法。 

天真的读者可能会尝试考虑为每个位置分配最终值的所有方法，但每个位置最多有 4 个选择，导致 4^n 种状态，这是完全不可行的。 

一个更微妙的问题是重叠。 如果我们贪婪地将每 4 个转换为 5 个，每 5 个转换为 4 个，我们可能会无意中破坏潜在的“54”对，因为升级会干扰邻近的结构。 

天真的贪婪的一个小失败说明：

 输入：```
454
```如果我们独立地尝试最大化 5 和 4，我们可能会将所有内容都转换为 5：```
555
```这会产生 0 次出现“54”，尽管更好的计划是：```
554
```产生 1 次出现。 

因此每个角色的局部决策是不够的； 邻接很重要。 

## 方法

 关键的观察是每个“54”仅取决于相邻对，并且每个字符只能增加。 这表明我们应该考虑构建最终值，以便尽可能多的相邻对成为 (5,4)。 

因为我们只能增加数字，所以只有当原始数字是 4 或 3 或 2 时才可能产生 4，但产生 5 总是可能的。 这种不对称性就是问题的整个结构。 

我们可以将每个位置重新解释为具有两个潜在的角色：如果它变成5，它可以充当“54”的左侧，如果它变成4，它可以充当“54”的右侧。但是，单个位置不能以不兼容的方式同时充当这两个角色，并且相邻的决策会相互作用。 

最佳策略来自于从左到右扫描并决定是否“花费”一个位置以形成以该位置结尾的 54。 如果我们想要在位置 i-1 和 i 处出现“54”，我们必须确保位置 i 变为 4，位置 i-1 变为 5。由于两者都可以通过增加来实现，唯一的问题是我们是否通过强制这对来获得收益。 

微妙之处在于，一旦我们决定形成以 i 结尾的“54”，我们就不应该允许位置 i-1 用作另一对的右侧，因为它被固定为 5。这自然会导致贪婪的 DP，我们跟踪前一个位置是否被用作对中的 5。 

蛮力会尝试我们放置“54”的相邻边的所有子集，确保分配中不会发生冲突。 这相当于在具有兼容性约束的路径中选择边的子集，这本质上是一条线上的最大匹配式结构，但由于升级自由，所有边都是独立可行的。 

这将问题简化为选择一组不相交或重叠的机会，其中如果我们将 i 设置为 5，将 i+1 设置为 4，则每条边 (i, i+1) 可以变为“54”。由于任何位置都可以提升到 4 或 5，因此除了分配一致性之外，不存在成本交互。 

因此，每个相邻对都可以独立地转换为“54”，而最佳答案就是相邻对的数量，即n-1。 然而，只有当我们始终能够同时在所有边上一致地分配值时，这才是正确的，这需要检查跨链的可行性约束。 

事实上，一致性总是可以实现的：如果我们决定每个位置 i 作为左端点时为 5，作为右端点时为 4，则仅当要求节点同时为 4 和 5 时才会出现冲突。当我们尝试使用两条边 (i-1, i) 和 (i, i+1) 时，就会发生这种情况。 因此，每个位置最多可以作为右端点参与一个“54”，但只有通过优先级解决冲突，它仍然可以多次作为左端点。 

这成为路径上经典的贪婪选择边，我们在没有邻接冲突的情况下选择尽可能多的边，从而产生简单的线性DP。

 ## 算法演练

1. 我们从左到右扫描字符串，并将每个相邻对 (i, i+1) 视为形成“54”的候选者。 这是有效的，因为如果需要，任何数字都可以增加到 5 或 4。 
2. 对于每个位置，我们决定是否将其用作“54”的右端（意味着它变成4）或左端（变成5）。 由于一个位置不能同时满足不兼容的分配，因此我们跟踪前一个位置是否已被用作形成对的一部分。 
3. 当我们在位置 i 时，如果前一个位置没有用作右端点，我们尝试使用 (i, i+1) 形成“54”，这会出现一次，并将 i 标记为 5，将 i+1 标记为 4。 
4. 如果我们无法在 i 处形成一对，因为它会与已分配的角色冲突，我们会跳过它并继续。 
5. 我们继续这个过程，直到字符串末尾，累积有效“54”形态的最大数量。 

### 为什么它有效

 关键的不变量是，每次我们选择一条边（i，i+1）时，我们都将两个端点完全委托给固定的角色，并且这些角色永远不会与之前固定的分配发生冲突。 由于图是一条路径，任何有效的非冲突边集都对应于类似匹配的结构，并且贪婪的从左到右选择总是在单调分配可行性下产生最大数量的非重叠可用约束。 后面的选择无法在不破坏先前强制分配的情况下增加有效对的数量，因此贪婪构造是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
n = len(s)

used = [False] * n
ans = 0

for i in range(n - 1):
    if not used[i] and not used[i + 1]:
        ans += 1
        used[i] = True
        used[i + 1] = True

print(ans)
```该解决方案标记了已经提交给“54”对的位置。 每次我们看到一个空闲的相邻对时，我们都会贪婪地获取它。 这完全对应于在路径上选择最大匹配，这是最优的。 

数组`used`确保我们永远不会为一个职位分配矛盾的角色。 一旦某个仓位参与配对，它就会被锁定。

 ## 边缘情况

 最小边缘情况是长度为 2 的字符串。对于输入“54”，算法立即获取该对并生成 1，匹配唯一可能出现的情况。 

对于“555”，每个相邻的对最初都是有效的，但是一旦第一对被占用，中间位置就会被锁定，从而阻止第二对。 贪婪算法产生 1，任何尝试获取两对的结果都会迫使中间的字符出现矛盾的分配。 

对于“2222”，每个位置都可以提高，算法选择 (0,1) 和 (2,3)，产生 2。任何选择 (1,2) 的尝试都不会增加不相交对的总数，从而确认了最优性。
