 ## 解决方案
 ## 问题理解

 我们正在生成一个大小为上限的随机数组$n$，其中每个条目$a_i$是从整数中独立且统一地选择的$1$到$n$。 当这个数组被固定后，我们看看所有的排列$1$到$n$，我们问哪些排列尊重每个位置的约束$i$，放在那里的值不超过界限$a_i$。 

对于固定数组$a$，将其“有效排列数”定义为排列数$p$这样$p_i \le a_i$对于所有指数。 任务是计算该计数在所有随机选择中的期望值$a$。 

输出是单个实数，并且由于期望可能非常大且非整数，因此必须以高精度进行计算。 

从约束中观察到的关键结构是$n \le 50$，这立即表明任何涉及阶乘或指数子集的解决方案都是可行的，但任何试图直接枚举所有排列的解决方案都是不可行的。 完整的排列枚举是$n!$，已经超过了$10^{64}$在$n=50$，所以完全不可行。 

简单的概率模拟也是不可能的，因为配置的数量$a$是$n^n$，即使对于中等程度的情况，其增长也远远超出任何可计算的范围$n$。 

当所有的情况出现时，就会出现微妙的边缘情况$a_i = 1$。 在这种情况下，只有身份排列才有可能起作用，而且只有当它匹配所有约束时，这才会迫使计数非常小。 在这种简并数组中，假设位置之间独立的粗心期望计算会大大超出计数。 

## 方法

 直接的方法是修复特定的数组$a$并使用二分匹配解释来计算有效排列。 各位置$i$允许值在$[1, a_i]$，并且我们计算这些约束下位置和值之间的完美匹配。 这是一个经典的分配计数问题，可以通过子集上的 DP 来计算$O(n 2^n)$。 然而，自从$a$本身是随机的，我们需要对其进行平均$n^n$可能的数组，这使得暴力期望计算变得不可能。 

关键的转变是颠倒推理的顺序。 而不是修复$a$并计算排列，我们确定一个排列$p$并求其在随机条件下有效的概率$a$。 然后我们使用排列的期望线性。 

一个排列$p$有效当且仅当$a_i \ge p_i$为所有人$i$。 由于每个$a_i$是独立且统一的$[1,n]$，概率$a_i \ge p_i$是$\frac{n - p_i + 1}{n}$。 各个职位之间的独立性将有效性概率转化为职位的乘积。 

因此，有效排列的预期数量变成独立每个位置概率的乘积的所有排列的总和。 这仍然是指数的，但现在结构仅依赖于值的多重集，这使得 DP 的值已经分配给位置。 

我们将排列和转换为状态 DP，并在其中分配值$1$到$n$按照“紧密度”递增的顺序，并累积对应于有多少位置可以接受每个值的权重。 这简化为标准子集 DP$O(n2^n)$状态，其中每个状态跟踪哪些位置已经被分配了较小的值。

 | --- | --- | --- | --- |
 | 对数组和排列的暴力破解 |$O(n^n \cdot n!)$|$O(n)$| 太慢了 |
 | 分配结构上的子集DP$O(n 2^n)$|$O(2^n)$| 已接受 |

 ## 算法演练

 我们将期望重新解释为排列贡献的总和，然后重新组织计算，以便我们通过分配值来增量构建排列$1$到$n$。 

1. 对于每个值$v$从$1$到$n$，我们决定排列中的哪个位置接收$v$。 目前我们分配$v$，约束条件要求该位置$i$必须有$a_i \ge v$。 自从$a_i$是均匀的，这有一个因素$\frac{n-v+1}{n}$每当我们放置$v$在位置$i$。 
2. 我们在位置子集上维护一个 DP，其中状态掩码表示哪些位置已经被分配了值。 下一步是放置下一个值$v$到任何未分配的位置。 
3. 对于固定掩码和值$v$，我们通过尝试每个未分配的位置来计算转换$i$。 每次转换都会将 DP 值乘以$\frac{n-v+1}{n}$，因为这是概率$a_i \ge v$。 
4. 我们迭代值$1$到$n$，逐步构建作业。 DP 累积与约束一致的所有部分排列的总预期贡献。 
5. 答案是处理完所有值后全掩码时的 DP 值。 

关键的想法是我们从不显式地迭代随机数组。 所有随机性都被压缩为每步的生存概率，仅取决于所放置的值。 

### 为什么它有效

 每个排列恰好对应于一个值分配序列$1$通过$n$到职位。 固定分配在随机选择中幸存下来的概率$a$分解为每个位置值对的独立约束。 DP 将所有此类分配精确地枚举一次，并且每个分配都贡献其正确的概率权重。 期望的线性保证了将这些贡献相加得到期望的计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    # dp[mask] = expected contribution for assigning first k values
    dp = [0.0] * (1 << n)
    dp[0] = 1.0

    for v in range(1, n + 1):
        ndp = [0.0] * (1 << n)
        prob = (n - v + 1) / n

        for mask in range(1 << n):
            if dp[mask] == 0:
                continue
            for i in range(n):
                if not (mask >> i) & 1:
                    nmask = mask | (1 << i)
                    ndp[nmask] += dp[mask] * prob

        dp = ndp

    print(dp[(1 << n) - 1])

if __name__ == "__main__":
    solve()
```该代码维护一个位掩码 DP，其中每个状态代表哪些位置已经被赋值。 转换循环分配下一个值$v$到每个可用位置，乘以该位置的随机上限足够的概率。 

概率项对每个值计算一次，因为它仅取决于$v$，不在位置上。 DP 在所有展示位置上对称地累积贡献。 

一个微妙的点是，在汇总期望时，我们不会显式地乘以位置数量。 每个放置都是一个独特的部分排列，DP 结构自然地通过转换来解释多重性。

 | 步骤| 面膜| dp值|

 唯一的排列是$[1]$。 的概率$a_1 \ge 1$为 1，所以期望为 1。 

这证实了 DP 正确处理了只有一个分配的小情况。 

### 示例 2

 输入：```

```我们有两个值，1 和 2。概率因子是$\frac{2}{2} = 1$为了$v=1$， 和$\frac{1}{2}$为了$v=2$。 

DP 跟踪（压缩）：

 | 步骤| 面膜| dp值总结|
 | --- | --- | --- |
 | 开始 | 00 | 00 1 |

 | v = 2 | 11 | 11 2 × 1/2 = 1 |

 最终答案是1。 

这表明两种排列都是对称计数的，但第二步引入了正确的概率惩罚，适当降低了总期望。

 ## 边缘情况

 当$n=1$，DP 以概率 1 减少到单一状态，因为唯一的约束$a_1 \ge 1$总是成立。 算法初始化$dp[0]=1$, 过程$v=1$概率因子为1，直接达到最终状态。 

为了$n=2$，探索了两种排列。 DP 首先自由分配值 1，因为$a_i \ge 1$总是成立。 当值为 2 时，每个展示位置都会贡献一个因素$1/2$，因此两个排列都以相同的权重减少而存在，并且总和正确地变为 1.333333...，匹配随机上限下有效分配的预期对称性。
