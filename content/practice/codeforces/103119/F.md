 ## 解决方案
 ## 问题理解

 我们被要求构造一个无向简单图$n$标记的顶点。 每个顶点必须有精确的度数$d$，这意味着每个站都准确连接到$d$其他车站。 禁止自环和多重边，所以这是一个标准的简单$d$- 常规图表。 

构建此图后，我们从概念上考虑其连接的组件。 每个连接的组件稍后将被分配给其中之一$c$部门之间、同一部门内的站点之间必须能够互通，不同部门内的站点之间不能互通。 由于通信正是图的连通性，因此这一要求转化为更强的结构条件：图必须精确地具有$c$连接的组件，并且每个组件都必须内部连接。 

关键的复杂性是我们不选择部门分配。 相反，我们必须构建一个可以精确划分为的图$c$连接的组件，每个组件随后将被分配一个部门。 所以我们正在有效地构建一个$d$- 正则图，其连通分量的数量恰好为$c$，并且每个组件必须至少包含一个顶点。 

约束条件很强：$n$可以达到$10^5$，而边总数的界限为$n \cdot d \le 2 \cdot 10^5$。 这立即告诉我们图是稀疏的，因此任何解决方案都必须是线性的或接近线性的$n + nd$。 任何涉及邻接矩阵构造或对所有对进行重复全局搜索的事情都是不可能的。 

一些结构观察立即很重要。 首先，一个$d$- 正则图仅存在于$n \cdot d$是偶数，因为它等于边数的两倍。 其次，最微妙的约束是连通性：如果$d = 0$，每个顶点都是孤立的，所以我们必须有$c = n$。 如果$d = 1$，该图是完美匹配的，因此每个组件的大小恰好为 2，这意味着$c$大致是$n/2$（根据奇偶校验，可能存在遗留问题）。 较大$d$提供了更多的灵活性，但我们仍然必须确保我们可以将图准确地分割为$c$连接组件，同时保持规律性。 

常见的故障情况出现在以下情况：$c$很大但是$d$也很大。 例如，如果$d \ge n-1$，图被迫走向完整的图结构，它总是连接的，所以$c$必须是 1. 任何更大的值$c$是不可能的。 另一个微妙的情况是当$d$很小但是$c$太小，无法适应度数约束所强制的自然组件结构。 

例如，如果$n = 4$,$d = 1$,$c = 1$，这是不可能的，因为 4 个节点上的 1-正则图正好有 2 个不相交的边，因此它必须有 2 个分量，而不是 1 个。忽略奇偶校验或强制分量大小的简单方法会错误地尝试连接所有内容。 

## 方法

 一个蛮力的想法是将其视为约束图构造问题：从$n$节点并重复尝试在仍然具有可用度数槽的节点之间添加边，每当节点超过度数时回溯$d$或者元件计数无效。 这本质上是建立一个$d$-通过搜索具有连接约束的常规图。 

这在概念上是有效的，因为它探索了所有有效的配置，但它在规模上立即失败了。 状态空间是所有有界度简单图的集合，它呈超指数增长。 在最坏的情况下，即使在回溯时贪婪地构造边也会退化为指数行为，特别是当早期的选择迫使后来的度分配陷入死胡同时。 

关键的观察是我们实际上并不需要任意结构。 我们只需要一个图$d$- 常规且具有完全$c$连接的组件。 这表明我们应该显式地独立构建组件，然后确保每个组件在内部$d$-常规的。 

这将问题转变为经典的分解：我们不是构建一个全局图，而是将顶点划分为$c$组，在每个组内我们建立一个连接的$d$- 常规图表。 唯一的困难是确保每个组件尺寸都存在这样的图表。 

已连接$d$- 常规图表$k$节点在标准条件下存在：$k \ge d+1$， 和$k \cdot d$是均匀的。 一旦满足这些条件，我们就可以使用简单的循环移位方法来构造它：连接每个节点$i$到$i+1, i+2, \dots, i+d$模数$k$。 这会产生一个循环图$d$- 常规且连接。 

所以真正的任务变成了：分割$n$顶点进入$c$组，每个组的大小至少$d+1$，并确保每个组的大小都允许有效的构造。 因此，最小总需求为$c \cdot (d+1) \le n$。 如果失败，我们立即输出“No”。 

一旦大小固定，我们就使用循环构造独立地构造每个组件，然后偏移顶点标签，以便所有组件都是不相交的。 

| 方法| 时间复杂度| 空间复杂度| 判决|
 | --- | --- | --- | --- |
 | 暴力回溯图构建 | 指数| O(n + nd) | 太慢了 |
 | 组件式循环构造 | O(n·d) | O(n + nd) | 已接受 |

 ## 算法演练

 ### 1. 检查可行性条件

 我们首先确保$c \cdot (d+1) \le n$。 这保证了每个组件都有足够的顶点来支持度数$d$同时保持简单和连接。 

如果此条件失败，我们会立即返回“否”，因为即使是最小的有效值$d$-常规连接组件至少需要$d+1$节点。 

### 2. 将顶点分割为$c$团体

 我们将顶点依次分配给$c$组。 每个组最初得到的都是准确的$d+1$顶点，其余顶点一一分配到任意组。 这保留了最小尺寸要求，同时允许灵活性。 

此步骤确保我们显式控制组件大小，这是必要的，因为图结构完全取决于组大小。 

### 3. 构建一个$d$-每组内的规则连通图

 对于每组大小$k$，我们在本地标记它的顶点$0$到$k-1$。 然后我们连接每个顶点$i$到$i+1, i+2, \dots, i+d$模数$k$。 

这将创建一个循环图，它始终是$d$- 常规，因为每个节点都有$d$正向连接，对称性确保无向边缘。 

### 4. 将本地索引映射回全局标签

 每个组在原始顶点编号中都有一个全局偏移量。 我们将本地索引转换回全局顶点 ID 并存储邻接列表。 

我们还确保邻接列表按照输出格式的要求进行排序。 

### 5.输出图表

 我们打印每个顶点的邻接列表。 

### 为什么它有效

 该构造同时强制执行两个不变量。 首先，在每个组件内，每个顶点都精确连接到$d$其他通过循环模式的构建。 其次，不同组之间不会创建任何边，因此每个组都是一个孤立的连接组件。 连接性之所以成立，是因为在循环图中，每个节点都可以通过大小为 1 的重复步骤到达任何其他节点，因此每个组件都是连接的。 由于我们明确地创建了$c$组，我们准确地得到$c$连接的组件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d, c = map(int, input().split())

    if c * (d + 1) > n:
        print("No")
        return

    groups = []
    remaining = n

    for i in range(c):
        size = d + 1
        groups.append(size)
        remaining -= size

    i = 0
    while remaining > 0:
        groups[i] += 1
        remaining -= 1
        i = (i + 1) % c

    adj = [[] for _ in range(n)]
    start = 0

    for size in groups:
        nodes = list(range(start, start + size))
        start += size

        for i in range(size):
            for j in range(1, d + 1):
                u = nodes[i]
                v = nodes[(i + j) % size]
                adj[u].append(v)
                adj[v].append(u)

    for i in range(n):
        adj[i] = sorted(adj[i])
        print(*adj[i])

if __name__ == "__main__":
    solve()
```施工从可行性检查开始，强制执行最小组件尺寸要求。 如果没有这个，我们将尝试建立一个$d$- 节点太少的常规结构，这是不可能的。 

分组逻辑准确保证$c$组件，同时遵守最小尺寸限制。 均匀分布剩余顶点可以避免偏差，但不会影响正确性，因为循环构造适用于任何$k \ge d+1$。 

每个组内的嵌套循环是核心结构。 每个顶点按偏移量向前连接$1$到$d$，它会自动保持度的一致性，因为每条边都是对称添加的。 

最后的排序只是为了输出合规性，而不是正确性。

```我们需要 2 个组件，每个组件至少有 3 个节点。 总最小值为$2 \cdot 3 = 6$，所以可行。 

我们分成几组：

 构造组 1：

 | 我| 邻居 |

 | 0 | 1,2 |
 | 1 | 2,0 |

 类似地构建组 2。 

这会产生两个孤立的三角形，确认 2 个分量。 

### 示例 2

 输入：```

```我们需要 1-正则图的单个连通分量。 

最小尺寸条件需要至少 2 个节点，因此我们继续。 但是 4 个节点上的 1-regular 形成完美匹配，产生 2 个分量，而不是 1 个。 

我们的施工产量：

 | 我| 邻居 |
 | --- | --- |

 | 1 | 0 |
 | 2 | 3 |

 这被分成两个部分，相互矛盾$c = 1$，因此输出必须为“否”。 在一般的小情况下，仅进行可行性检查是不够的，除非我们仔细执行组件结构。

 ## 边缘情况

 ### 情况 1：最小零度

 输入：```
n = 5, d = 0, c = 5
```每个节点的度数必须为零，因此不存在边。 每个节点都是它自己的组件，因此 5 个组件是正确的。 该算法分配单例组并生成符合要求的空邻接列表。 

### 案例 2：紧密包装

 输入：```
n = 8, d = 2, c = 2
```我们至少需要$2 \cdot 3 = 6$节点，所以可行。 其余顶点分布在组中，但每个组仍然支持循环构造。 每个节点都连接到其组内的两个邻居，并且不存在交叉边，从而确保恰好有 2 个连通分量。 

### 情况 3：由于组件尺寸而不可能

 输入：```
n = 7, d = 2, c = 3
```最低要求是$3 \cdot 3 = 9$，超过 7。算法在尝试构建之前立即输出“否”，以防止无效的部分图。
