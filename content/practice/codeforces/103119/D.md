 ## 解决方案
 ## 问题理解

 我们获得了五件神器物品，每个装备槽各一件。 每个神器都贡献了五个统计数据线，并且在所有神器中我们只关心四个统计数据：固定攻击力、攻击力百分比、暴击率和暴击伤害。 

最终的角色模型以固定的基本统计数据开始：1500 ATK、5% 暴击率和 50% 暴击伤害。 工件根据统计数据类型以加法或乘法方式修改这些值。 平直ATK值直接相加。 ATK 百分比值累积为应用于基础 ATK 的单个乘数。 暴击率和暴击伤害也会在神器上累加。 

聚合后，ATK 的计算方式为 1500 乘以（1 加总 ATK 百分比）加上总固定 ATK。 暴击率受到限制，以便任何高于 100% 的值都恰好变为 100%。 暴击伤害没有这样的上限。 

最后，预期伤害被定义为正常伤害和致命伤害的混合：

 E = ATK乘以非暴击概率加上ATK乘以暴击倍数乘以暴击概率。 这简化为正常结果和关键结果之间的加权平均值。 

输入是纯文本的，因此主要困难是解析和正确累积具有混合百分比格式和浮点精度的值。 

限制很小：只有 5 个工件，每个工件有 5 条线，因此总共 25 条线。 这排除了 O(1) 解析和算术之外的任何优化需求。 真正的风险是数字错误：误解百分号、忘记 5% 的基本暴击率，或者错误地将暴击率限制在 100%。 

当暴击率在求和后超过 100% 时，会出现微妙的边缘情况。 例如，如果神器的暴击率总计为 150%，则必须将有效值精确限制为 100%，否则预期伤害将会被高估。 

另一个常见的陷阱是混合百分比缩放。 ATK 比率以百分比形式给出，但作为乘数应用（x 百分比变为 x 除以 100）。 将其误读为直接加法会导致输出下降 100 倍。 

## 方法

 强力解释会尝试枚举每个槽的工件选择的组合，但这里每个槽只有一个工件并且所有工件都被选择，因此不存在组合搜索。 唯一的工作就是聚合。 

核心观察是，每个统计数据都独立且线性地贡献，直到最终评估步骤。 这使我们能够将问题简化为解析 25 个字符串并对四个运行总计求和。 

一旦计算出总数，我们就直接应用 ATK 和预期伤害的公式。 唯一的非线性行为是暴击率上限为 1.0，必须在求和之后和计算期望之前应用。 

因为在最终公式之前一切都是相加的，所以除了四个浮点累加器之外，我们不需要任何数据结构。 

| 方法| 时间复杂度| 空间复杂度| 判决|
 | --- | --- | --- | --- |
 | 暴力枚举| O(1) | O(1) | O(1) | O(1) | 已接受 |
 | 解析和聚合 | O(25) | O(1) | O(1) | 已接受 |

 ## 算法演练

1.初始化四个累加器：总平攻击、总攻击百分比、总暴击率、总暴击伤害。 这些从零开始，因为基本统计数据在最终公式中单独处理。 
2. 读取所有 25 行并通过在加号处拆分来解析每个统计字符串。 左侧标识统计类型，右侧提供可能包含也可能不包含百分号的数值。 
3.如果统计的是攻击力，则直接将数值与总平攻击力相加。 
4.如果统计的是攻击率，则将百分比除以100转换为小数，然后将其添加到总攻击力百分比中。 
5. 如果统计数据是暴击率，则转换为小数并将其添加到总暴击率中。 
6. 如果统计数据为暴击伤害，则转换为小数并将其添加到总暴击伤害中。 
7. 处理完所有工件后，将最终 ATK 计算为 1500 乘以（1 加总 ATK 百分比）加上总固定 ATK。 这将基本缩放与附加固定奖金分开。 
8.将暴击率限制在最多1.0。 这强制执行了概率不能超过确定性的规则。 
9. 使用加权公式计算预期伤害：ATK乘以（1减去暴击率）加上ATK乘以（1加暴击伤害）乘以暴击率。 
10. 输出足够精度的结果以满足误差约束。 

### 为什么它有效

 所有统计贡献都是独立且相加的，直到最终的期望公式。 这意味着我们可以安全地聚合每个类别，而不必担心工件之间的顺序或交互影响。 唯一的非线性变换是暴击率上限，它在求和之后应用，保持正确性，因为概率饱和仅取决于总和，而不取决于单个源。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

BASE_ATK = 1500.0
BASE_CRIT_RATE = 0.05
BASE_CRIT_DMG = 0.50

flat_atk = 0.0
atk_rate = 0.0
crit_rate = 0.0
crit_dmg = 0.0

for _ in range(25):
    s = input().strip()
    typ, val = s.split('+')

    if val.endswith('%'):
        x = float(val[:-1]) / 100.0
    else:
        x = float(val)

    if typ == "ATK":
        flat_atk += x
    elif typ == "ATK Rate":
        atk_rate += x
    elif typ == "Crit Rate":
        crit_rate += x
    elif typ == "Crit DMG Rate":
        crit_dmg += x

atk = BASE_ATK * (1.0 + atk_rate) + flat_atk

crit_rate = BASE_CRIT_RATE + crit_rate
crit_dmg = BASE_CRIT_DMG + crit_dmg

if crit_rate > 1.0:
    crit_rate = 1.0

expected = atk * (1.0 - crit_rate) + atk * (1.0 + crit_dmg) * crit_rate

print(expected)
```该解决方案围绕四个独立的累加器构建，每个相关统计数据一个。 对所有行统一进行解析，并简单检查百分比值。 一个常见的实施错误是忘记百分比统计数据在聚合之前必须除以 100。 

ATK 公式仅在收集了所有贡献后才应用，因为 ATK 百分比修改基础值而不是充当直接累加项。 在添加基础 5% 后应用暴击率限制，因为基础贡献和神器贡献在强制执行概率上限之前结合在一起。 

## 工作示例

 我们使用输入的最小子集构建简化的跟踪。 

### 示例 1

 输入：

 攻击力+10

 攻击力+10%

 暴击率+10%

 暴击伤害率+10%

 | 步骤| 平攻击力 | 攻击率 | 暴击率| 暴击伤害 |
 | --- | --- | --- | --- | --- |
 | 解析后| 10 | 10 0.10 | 0.10 0.10 | 0.10 0.10 | 0.10

 最终计算如下：

 攻击力=1500×1.1+10=1660

 暴击率 = 0.05 + 0.10 = 0.15

 暴击伤害 = 0.50 + 0.10 = 0.60

 预期 = 1660 × (0.85 + 1.60 × 0.15) = 1660 × 1.09 = 1809.4

 这证明了基本统计数据和附加修饰符的正确分离。 

### 示例 2

 输入：

 暴击率+120%

 攻击力+0

 | 步骤| 平攻击力| 攻击率 | 暴击率| 暴击伤害 |
 | --- | --- | --- | --- | --- |
 | 解析后| 0 | 0.0 | 0.0 1.20 | 1.20 0.0 | 0.0

 添加碱基后：

 暴击率 = 0.05 + 1.20 = 1.25，然后固定为 1.0

 预期变为确定性：

 攻击力×（1+暴击伤害）=1500×1.5=2250

 这证实了概率饱和的正确处理。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import math

    BASE_ATK = 1500.0
    BASE_CRIT_RATE = 0.05
    BASE_CRIT_DMG = 0.50

    flat_atk = 0.0
    atk_rate = 0.0
    crit_rate = 0.0
    crit_dmg = 0.0

    for _ in range(25):
        s = sys.stdin.readline().strip()
        typ, val = s.split('+')
        if val.endswith('%'):
            x = float(val[:-1]) / 100.0
        else:
            x = float(val)

        if typ == "ATK":
            flat_atk += x
        elif typ == "ATK Rate":
            atk_rate += x
        elif typ == "Crit Rate":
            crit_rate += x
        elif typ == "Crit DMG Rate":
            crit_dmg += x

    atk = BASE_ATK * (1.0 + atk_rate) + flat_atk
    crit_rate = BASE_CRIT_RATE + crit_rate
    crit_dmg = BASE_CRIT_DMG + crit_dmg
    crit_rate = min(1.0, crit_rate)

    return str(atk * (1.0 - crit_rate) + atk * (1.0 + crit_dmg) * crit_rate)

inp = "\n".join(["ATK+10"] * 25)
assert float(run(inp)) > 1500

# crit cap test
inp = "\n".join(["Crit Rate+50%"] * 3 + ["ATK+0"] * 22)

# all damage stats

 ## 边缘情况

 一个关键的边缘情况是暴击率溢出。 如果在汇总所有工件后总暴击率超过 1.0，则正确的行为是限制它。 该算法通过在聚合后应用最小操作来处理此问题，确保概率不超过 100%。 

另一种情况是混合百分比格式。 由于某些值包含百分号而其他值则不包含，因此未能通过除以 100 进行归一化将使 ATK 率或暴击率增加 100 倍。解析步骤显式检查百分号并进行一致转换。 

最后一个微妙的情况是浮点精度。 由于要求的容错能力是1e-6，所以使用Python浮点运算就足够了，但是不带格式控制的打印也是可以接受的，因为判断允许相对误差。
