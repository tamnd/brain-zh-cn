 ## 解决方案
 ## 问题理解

 我们得到了一系列硬币面额，其中包括从 1 到 n 的每个整数值，并且我们可以使用任意数量的任意这些值的硬币。 任务是使用尽可能少的硬币来表示目标总和S，其中允许无限制地重复相同的硬币值。 

这本质上是一个资源最小化问题：我们希望将 S 分解为 [1, n] 范围内的整数多重集，从而最小化该多重集的大小。 每枚硬币都为总价值贡献了自己的价值，我们希望恰好达到 S 并且不超过它。 

限制因素在这里非常重要。 n的值可以大到100000，而S可以大到1e9。 这立即排除了任何尝试在 S 上模拟组合或动态编程的方法，因为任何依赖于 S 的状态空间都会太大。 即使是 O(S) 解决方案也是不可能的。 我们被迫采用一种解决方案，依靠结构而不是枚举，以恒定时间或对数时间计算每个测试用例的答案。 

一个天真的但常见的错误是假设贪婪地使用最大的硬币总是可以通过重复获取 min(n, S) 直到 S 变为零来实现。 当拿走大硬币后的余数很小但在不增加硬币总数的情况下无法有效表达时，这种方法会以微妙的方式失败。 另一个不正确的直觉是将其视为背包问题并尝试直接优化组合； 由于 S 的规模，这也失败了。 

## 方法

 蛮力的想法很简单：尝试每一个可能的多重集合，其值在 [1, n] 中且总和为 S，并计算最小基数。 这在原则上是正确的，因为它探索了所有有效的分解。 然而，此类多重集的数量随着 S 增长得非常快。即使限制为有界硬币值，在一般设置中，整数的分区数量也以 S 的平方根呈指数增长，使得这种方法在达到给定约束之前很久就不可行。 

关键的观察结果是，硬币集的结构使问题的行为类似于整数分区的约束版本，其中大值总是更可取的。 如果我们使用 k 个硬币，则通过取 n 的 k 个副本，得到 k·n，我们可以用 k 个硬币实现的最佳总和。 这意味着任何使用 k 个硬币的解决方案只能覆盖最多 k·n 的总和。 相反，如果我们想达到 S，我们至少需要足够的硬币，使得它们的最大可实现总和达到 S。 

这立即表明了 ceil(S / n) 的下限。 更深层次的问题是这个界限是否总是可以实现的。 事实证明是这样的，因为我们总是可以用较小的硬币值来表示任何余数：一旦我们取了尽可能多的n个硬币，剩下的就严格小于n，并且该余数总是可以用单个硬币来表示（或者在组合调整中最多增加一个硬币，但结构保证了可行性）。 这使得下限变得紧张。 

因此，最优策略简化为一个简单的除法论证：使用尽可能多的n值硬币，如果有剩余，可以用一枚额外的硬币覆盖。

 ## 算法演练

 该解决方案依赖于将目标总和 S 与大小为 n 的硬币的最大贡献进行比较。

1. 计算 S 中有多少个完整的 n 大小的硬币。这是使用整数除法将 S 除以 n。 这代表了我们在不超过S的情况下可以使用的大币的最大数量。 
2. 取出n个硬币后，计算余数。 这是 S 模 n。 余数代表满n个硬币无法覆盖的剩余金额。 
3. 如果余数为零，则答案正是完整硬币的数量，因为 S 可以被 n 整除。 
4. 如果余数不为零，我们需要额外一枚硬币来覆盖剩下的部分。 这是因为从 1 到 n 的任何值都可用，因此任何严格小于 n 的余数始终可以使用一枚硬币形成。 

推理取决于硬币价值 1 始终存在的事实，因此任何剩余的都可以始终表示，而不会进一步增加复杂性。 

### 为什么它有效

 关键的不变量是，在获得价值为 n 的最低（S / n）枚硬币后，剩余价值严格小于 n，因此始终可以使用可用的硬币集来表示。 由于我们总是喜欢较大的硬币以尽量减少数量，因此任何最佳解决方案都必须使用尽可能多的 n 个硬币； 用较小的硬币替换 n 个硬币永远不会减少所需的硬币数量。 这确保了贪婪分解为 n 大小的块加上余数是最优的并且无法改进。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, s = map(int, input().split())
    ans = s // n
    if s % n != 0:
        ans += 1
    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了除法推理。 整数除法计算 S 中适合多少个完整的 n 个硬币。条件增量处理需要额外一枚硬币的任何剩余余数。 

一个微妙的点是我们不需要构建实际的硬币集。 该问题只要求最少的硬币数量，而不要求其组成，所以算术就足够了。

### 示例 2

 输入：n = 6，S = 16

 | 步骤| 完整硬币（S//n）| 余数 (S%n) | 回答 |

 两枚价值 6 的硬币得到 12，剩下 4。另外一枚硬币可以覆盖剩余的部分。 

这些痕迹表明，该算法有效地将 S 分解为大小为 n 的最大块，并独立处理剩余部分，从而确认了上限划分行为的正确性。

 ## 边缘情况

 一种边缘情况是 S 小于 n。 在这种情况下，S // n 变为零，并且 S % n 等于 S。该算法正确返回 1，因为 n 以内的任何值都可以用单个硬币表示。 例如，当 n = 10 且 S = 7 时，我们取零个完整硬币和一枚价值 7 的硬币。 

另一个边缘情况是 S 可以被 n 整除。 例如，n = 4 且 S = 12 给出 S // n = 3 且余数为 0，因此无需添加额外的硬币。 这反映了仅使用最大价值硬币的最佳策略。 

最后的边缘情况是当 n = 1 时。这里每个硬币都是 1，因此形成 S 的唯一方法是使用 S 个硬币。 该公式给出 S // 1 = S 且余数为 0，与正确答案完全匹配。
