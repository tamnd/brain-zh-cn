 ## 解决方案
 ## 问题理解

 我们得到了几堆并排放置的垂直块。 各位置$i$有一个堆栈的高度$a_i$。 从上面来看，相机只关心某个位置是被占用还是空的，因此每一列只要至少有一个块就贡献一个可见单元。 从侧面，相机看到堆栈的轮廓，因此在每个位置，它都会看到该列中剩余的最高块。 

任务是在保持两个视图不变的情况下删除尽可能多的块。 这意味着每列必须保持非空，并且每列的高度必须与最初完全相同，因为列的任何减少都会缩小侧面轮廓。 

输入大小达到$n = 10^5$，因此任何比线性或线性时间更差的解决方案都会陷入困境。 反复尝试从每列中删除块的二次方法已经太慢了，因为它意味着以下顺序：$10^{10}$最坏情况下的操作。 

当在没有协调的情况下贪婪地思考每一列时，就会出现一个微妙的问题。 如果我们尝试独立地减少每个堆栈，同时保留其高度，我们可能会假设我们可以删除除每列一个块之外的所有内容，但这忽略了侧视图仅取决于每列的最大高度而不是内部结构的约束。 另一个错误是尝试在堆栈之间重新分配块，这是明确不允许的。 

一个关键的边缘情况是所有堆栈都相等，例如$a = [3, 3, 3]$。 正确的答案不是简单地“删除除顶层之外的所有内容”，因为顶层仍然必须保留两个视图，这意味着我们不能删除它下面的整个列。 这迫使我们思考哪些区块是根本必要的，而不是我们可以自由删除多少区块。 

## 方法

 暴力策略将模拟从任何堆栈中逐一删除块，并检查两个投影是否保持不变。 每次移除都需要重新计算顶视图和侧视图，这取决于扫描所有堆栈以验证约束。 即使仔细记账，这至少会导致$O(n)$每次尝试移除的工作量，最多$O(n \cdot m)$最坏情况下的操作，因为每列最多可以有$m$块。 这远远超出了可行的限度。 

一旦我们解释了必须保持固定的内容，问题的结构就会简化。 顶视图需要每个位置$i$为了保持非空，所以每个堆栈必须至少保留一个块。 侧视图要求每列的最大高度保持等于原始高度$a_i$，这意味着至少一列必须保持其完整高度，并且每一列必须保留足够的块以独立维持其自身的最大值。 由于块不在堆栈之间共享，因此除了全局要求非空之外，每个堆栈都是独立的。 

这导致了一个直接的观察：在每个堆栈中，我们必须保留至少一个块，但是只要我们不减少该堆栈的最大高度，我们就可以自由地删除所有其他块。 由于最大高度恰好是$a_i$，我们必须至少保留$a_i$堆栈中的块$i$如果我们严格地将侧视图解释为保留完整的柱高。 然而，关键的简化是，其他堆栈中的任何内容都不依赖于给定堆栈超出其高度的内部结构，因此我们只需要为顶视图保留每个堆栈的一个代表性“支撑”，并确保最高的块在侧视图中仍然存在。 这会折叠到在顶层中保留每个堆栈恰好一个块，并保留完整的原始高度结构，这实际上意味着每个堆栈在其可移动内部之外恰好贡献了一个不可避免的块。 

更清晰的观察方式是翻转视角：每一叠高度$a_i$必须保留至少一个方块以保持上方的可见性，并且所有剩余的$a_i - 1$可以在不影响任一视图的情况下移除块，因为侧视图仅取决于高度的存在$a_i$，而不是它下面有多少块。 

因此，在每个堆栈中，恰好$a_i - 1$块是可移动的，对所有堆栈求和即可给出答案。

 | 最佳 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 首先独立考虑每个堆栈，因为没有操作在堆栈之间移动块。 
2.对于堆栈的高度$a_i$，确定哪些块是保留两个视图所必需的。 
3. 认识到顶视图只需要至少一个块即可在该位置保持可见。 
4. 认识到最顶层幸存结构下方的任何块都不会影响任一投影。 
5. 得出结论，每个堆栈必须仅保留一个块，并且可以删除所有其他块。 
6. 通过求和计算总的可移动块数$a_i - 1$跨所有堆栈。 

### 为什么它有效

 不变的是，在整个过程中每一列都保持非空，并且每一列在正确的位置仍然包含至少一个块以保留两个投影。 由于侧视图仅取决于每根柱中最大高度的存在，而不取决于中间结构，因此删除不是柱的最后剩余代表的任何块不会改变任何投影。 由于堆栈是独立的并且没有重新分配，因此对每列移除求和即可得出全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    ans = 0
    for x in a:
        ans += x - 1
    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接应用每列推理。 每个堆栈都贡献$a_i - 1$可移动块，所以我们累积这个值。 变量`m`没有使用，因为约束$a_i \le m$只限制输入的有效性，不影响计算。 

唯一微妙的实现细节是避免差一错误：我们计算的是可移动块，而不是剩余块。 每个堆栈只保留一个块，因此减一是必不可少的。

 ## 边缘情况

 对于像这样的单个堆栈`1`，算法计算$1 - 1 = 0$，意味着无法删除任何块。 这符合堆栈必须在两个视图中保持可见的要求，并且删除其唯一的块会破坏顶视图。 

对于最大堆栈，例如`1000000000`，算法返回$999999999$。 循环处理该值一次，Python 的整数算术可以安全地处理总和，而无需担心溢出。 

对于像这样的混合配置`1 2 1`，中间的堆栈恰好贡献了一个可移动块，而其他堆栈贡献了零。 这证实了每个堆栈都是独立处理的，并且没有跨堆栈约束改变计算。
