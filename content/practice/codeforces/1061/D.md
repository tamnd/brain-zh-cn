 ## 解决方案
 ## 问题理解

 我们得到了一组时间间隔，每个时间间隔代表一个占据连续分钟范围的电视节目。 关键的限制是一台电视不能用于观看时间间隔重叠的两个节目。 如果一个节目从第 2 分钟运行到第 5 分钟，另一个节目从第 4 分钟运行到第 7 分钟，则它们无法分配到同一台电视，因为第 4 分钟和第 5 分钟发生冲突。 

每台电视的租用时间必须连续，必须涵盖分配给它的所有节目。 如果从一分钟开始使用电视`a`直到分钟`b`，成本由固定租赁费加上每分钟与持续时间成比例的延期费组成：`x + y * (b - a)`。 

任务是将所有节目分配给电视，将不重叠的间隔分组到同一台电视上，以最小化总租赁成本的方式。 每台电视对应一组不相交的间隔，必须按时间递增顺序观看，并且成本仅取决于从该电视使用的第一分钟到使用的最后一分钟的时间跨度。 

约束足够大，以至于任何比较所有可能的分配或尝试枚举间隔分区的解决方案都是不可能的。 最多 100,000 个间隔，任何比粗略更糟糕的情况`O(n log n)`会挣扎。 这立即排除了对子集的强力分组或动态编程。 

一些边缘案例暴露了常见的陷阱。 如果所有间隔都是不相交的，天真的方法可能仍然会尝试错误地合并它们并低估电视的数量。 例如，间隔`[1,2]`和`[3,4]`在一台电视上兼容，但是`[1,3]`和`[2,4]`即使它们的端点很接近，也不是。 另一个微妙的情况是当间隔形成重叠链时，例如`[1,10], [2,3], [4,5], [6,7]`。 贪婪的“如果端点看起来是分离的，则将所有内容都打包到一台电视中”的方法在这里失败了，因为长间隔阻碍了重用。 

真正的挑战是平衡两个相反的成本：开设新电视的成本`x`，同时将现有电视扩展到更长的跨度成本`y`每分钟。 自从`y < x`，将间隔合并为更长的跨度通常是有益的，但前提是重叠结构允许。 

## 方法

 强力解决方案将尝试将每个间隔分配给电视并维护所有可能的分组。 每台电视形成一系列不重叠的间隔，对于每个任务，我们根据每组中的最短和最长时间计算成本。 间隔分区的数量呈指数增长，使得这种方法即使对于`n = 30`，更不用说`10^5`。 

关键的观察结果是，电视的成本仅取决于其分配的间隔的联合跨度，而不取决于其内部包含多少个间隔。 如果我们按开始时间对间隔进行排序，我们就可以按顺序处理它们，并推断出在任何时刻有多少台电视同时处于活动状态。 

在任何分钟，重叠间隔的数量决定了至少需要多少台电视。 然而，尽早打开电视并让它在间隙中保持运行可能比反复打开新电视更便宜，因为每台新电视都要支付固定成本`x`。 

这导致了扫线视角：随着时间的推移，间隔开始和结束，并且我们维护有多少是活跃的。 每当一个间隔开始超出所有当前活动的链时，我们要么扩展现有电视，要么启动一台新电视，具体取决于成本权衡。 当按排序顺序处理间隔并将其分配给最早的兼容电视时，最佳策略会变得贪婪，从而最大限度地减少时间跨度的碎片。 

我们有效地维护一组活动的“电视片段”，并始终将新的间隔附加到最后一端最接近但仍然有效的电视，因为这可以最大限度地减少浪费的空闲时间。 这减少了不必要的新电视打开，同时尊重重叠限制。

 ## 算法演练

 1. 按开始时间对所有间隔进行排序。 这确保我们按时间顺序处理节目，因此决策反映了真实的重叠结构而不是任意的输入顺序。 
2. 维护跟踪可用电视的数据结构，并按分配给该电视的最后一个节目的最后结束时间进行索引。 最小堆在这里效果很好，因为我们希望重用最早可用的电视。 
3. 迭代每个区间`[l, r]`按排序顺序。 
4. 对于当前间隔，检查是否存在最后分配的间隔在之前或之前结束的电视`l`。 如果存在这样的电视，请重复使用结束时间最短的电视，因为这样可以为将来的任务提供更大的灵活性。 
5. 如果不存在这样的电视，则为此时间间隔打开一个新电视。 
6. 为电视分配时间间隔时，更新成本贡献。 如果这是电视的第一个间隔，则费用包括从`l`。 否则，我们扩展现有的跨度，这只会增加终点。 
7. 跟踪每台电视的当前状态`[start, end]`跨度，因为最终成本仅取决于这些边界，而不取决于内部的各个区间。 
8. 处理完所有间隔后，将使用的所有电视的成本相加`x + y * (end - start)`对于每个。 

### 为什么它有效

 总是重复使用最早可用的电视的贪婪选择保留了未来的灵活性。 任何使用不同兼容电视的替代分配都无法在不增加其他地方碎片的情况下减少所选电视的跨度。 由于成本随跨度线性增长并且具有固定的开放费用，因此最大限度地减少电视数量和总空闲扩展相当于将间隔打包到具有最小跨度的非重叠段链中。 基于堆的最早适应策略确保每个链在时间上尽可能紧密，防止不必要的间隙增加`(end - start)`。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline
MOD = 10**9 + 7

def solve():
    n, x, y = map(int, input().split())
    segs = [tuple(map(int, input().split())) for _ in range(n)]
    segs.sort()

    # each TV: (current_end, start_time)
    # we track active TVs by end time
    heap = []

    # store final segments for cost computation
    tvs = []

    for l, r in segs:
        if heap and heap[0][0] < l:
            end, start = heapq.heappop(heap)
            # extend this TV
            start = min(start, l)
            end = r
            heapq.heappush(heap, (end, start))
        else:
            heapq.heappush(heap, (r, l))

    # compute cost
    res = 0
    while heap:
        end, start = heapq.heappop(heap)
        length = end - start
        res += x + y * length
        res %= MOD

    print(res % MOD)

if __name__ == "__main__":
    solve()
```其实现直接遵循贪婪链思想。 间隔已排序，以便当我们处理时`[l, r]`，所有较早的间隔都已放入某个链中。 该堆存储当前的电视连锁店，并以结束时间为键。 如果最早收尾的电视可以容纳当前节目，我们就延长； 否则我们开始一个新的链。 

一个微妙的点是我们只需要`(start, end)`每台电视，因为中间结构与成本无关。 另一个关键细节是使用堆而不是扫描所有电视，这确保了算法保持不变`O(n log n)`。

 ## 边缘情况

 一种边缘情况是间隔恰好接触边界。 例如`[1,2]`和`[3,4]`应该是可合并的，因为在第 3 分钟将它们与第 2 分钟分开时没有重叠。该算法可以正确处理此问题，因为我们允许在以下情况下重用：`end < l`，确保邻接有效。 

另一个边缘情况是严格的重叠链，例如`[1,10], [2,3], [3,4]`。 长间隔迫使分离，尽管短间隔似乎可以相互合并。 堆确保长间隔占据自己的链，而短间隔仍可以根据顺序进行组合。 

最后一个边缘情况是差距很大，打开一台新电视可能看起来有益，但扩展现有电视更便宜，因为`y < x`。 该算法自然会尽可能地扩展跨度，因为重用总是会减少固定成本积累。
