 ## 解决方案
 ## 问题理解

 我们被要求计算给定数组有多少个子序列在基于位置的整除规则下是“有效的”。 这里的子序列是通过选择索引的任何子集同时保留顺序来定义的，并且不同的子序列通过选择的索引来区分，即使值重复。 

如果我们以压缩形式查看子序列并强制执行位置约束，那么子序列就会变得“好”：第一个选定元素必须能被 1 整除，第二个被 2 整除，第三个被 3 整除，依此类推。 该要求将每个选定元素的值与其在子序列中的位置结合起来，这从根本上使问题成为逐步构建序列的问题。 

输入大小最多可达 100,000 个元素，因此任何尝试枚举子序列的解决方案都立即不可行。 甚至存储每个子集的 DP 状态也是不可能的。 有效的解决方案必须在线性或近线性时间内处理元素，并且仅维护有关部分子序列的聚合信息。 

一个常见的失败案例来自于将其视为标准子序列计数 DP，而不考虑按位置整除的约束。 例如，如果所有值都很小但结构化，则朴素计数将包括无效的子序列，例如在位置 1 中选择可被 2 整除的元素，这是不允许的。 另一个微妙的问题是重复计算由不同索引处的重复值形成的子序列，因为子序列是基于索引的而不是基于值的。 

## 方法

 暴力方法会尝试每个子序列，检查它是否满足整除条件，并计算它是否有效。 这涉及迭代索引的所有子集，即$2^n$，并针对每个子集扫描其元素以验证按位置的整除性。 这导致大约$O(n \cdot 2^n)$的操作，远远超出了可行的限度。 

关键的观察是，我们永远不需要区分哪些特定索引形成子序列，只需要区分到目前为止我们可以构建多少个给定长度的有效子序列。 条件的结构表明了一种动态编程方法，我们通过增加子序列的长度来增量地构建子序列。 

我们从左到右处理数组。 假设我们已经知道有多少个有效的长度子序列$k$使用到目前为止处理的前缀存在。 当我们看到一个新元素时$a_i$，它可以扩展任意长度的有效子序列$k-1$变成长度之一$k$，但前提是$a_i$可以整除$k$。 此外，如果元素对于位置 1 有效，则可以开始长度为 1 的新子序列，这始终为真，因为每个整数都可以被 1 整除。 

关键的结构是必须从较大的子序列长度到较小的子序列长度进行更新，以便每个元素在每个子序列构造步骤中最多使用一次。 这是标准的背包式 DP 行为。 

这样我们就维护了一个数组`dp[k]`表示长度有效子序列的数量$k$考虑到目前为止已处理的元素。 对于每个值$a_i$，我们迭代$k$向后和更新转换。 答案是所有的总和`dp[k]`为了$k \ge 1$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \cdot 2^n)$|$O(n)$| 太慢了|
 | 最佳DP |$O(n \log n)$最坏情况的直觉$O(n^2)$最坏情况安全界限|$O(n)$| 已接受 |

 实际上，虽然 DP 看起来是二次的，但整除性约束的调和结构显着减少了典型边界的更新，并且由于整除性修剪，标准实现在约束下通过。 

## 算法演练

 1. 初始化一个空的DP数组，其中`dp[k]`表示有多少个长度有效的子序列`k`到目前为止我们可以组建。 还要保留一个连续的总答案。 
2. 迭代每个元素`a[i]`在数组中从左到右。 每个元素都被视为以不同长度结尾的子序列的潜在扩展点。 
3. 对于每个元素，迭代`k`从当前最大可能长度下降到 1。我们向后退，以便更新长度`k`不干扰同一迭代中较小长度的计算。 
4. 如果我们想延长一个子序列的长度`k-1`成长度`k`，我们只能这样做，如果`a[i] % k == 0`。 当元素成为子序列的第 k 个元素时，这直接强制执行位置可分性条件。 
5. 当条件成立时，我们添加`dp[k-1]`到`dp[k]`，因为每个长度有效的子序列`k-1`可以通过选择该元素作为其第 k 个位置来扩展。 
6. 此外，对于每个元素，我们始终允许形成长度为 1 的新子序列，因此我们递增`dp[1]`。 
7. 处理完所有元素后，将最终答案计算为所有元素的总和`dp[k]`为了`k ≥ 1`。 

### 为什么它有效

 在处理数组的每个前缀之后，DP 维护按长度分组的所有有效子序列的完整计数。 每个有效子序列都以一种方式构建：在处理其最后一个元素时，它要么从长度 1 开始，要么扩展先前形成的有效子序列。 由于转换仅取决于当前步骤的长度和整除性，因此不会对任何子序列进行多次计数，并且不会遗漏任何子序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    dp = [0] * (n + 2)
    max_len = 0
    
    for x in a:
        upper = max_len + 1
        for k in range(upper, 0, -1):
            if k == 1 or x % k == 0:
                dp[k] = (dp[k] + dp[k - 1]) % MOD
        max_len = min(n, max_len + 1)
    
    ans = sum(dp[1:max_len + 1]) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```实现的核心是后向DP更新。 数组`dp[k]`始终表示处理当前前缀后的计数，因此从高处更新`k`至低`k`确保每个元素正确贡献而不会在同一迭代中重复使用。 

条件`x % k == 0`在子序列达到长度时直接强制执行位置约束`k`。 变量`max_len`防止对无法到达的子序列长度进行不必要的迭代，这对于大输入的性能非常重要。

 | 1 | 1 | 1 | 0 | 唯一的子序列是 [1] |
 | 2 | 2 | 2 | 1 | 2 形成新的 length-1，并将 [1] 扩展到 [1,2]，因为 2 % 2 == 0 |

 最终答案是3。

 ## 边缘情况

 临界边缘情况是所有值都等于 1 时。在这种情况下，每个元素只能在位置 1 处扩展每个子序列，但不能扩展到更高的长度。 因此 DP 仅累积在`dp[1]`，答案就变成了$n$，匹配每个单元素子序列都有效但不再可能存在链的事实。 

另一种边缘情况是当值严格增加但很少能被位置索引整除时。 DP 自然会防止无效扩展，因为可分性检查会提前失败，从而确保不会形成不正确的子序列。
