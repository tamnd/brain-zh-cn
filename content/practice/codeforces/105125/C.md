 ## 解决方案
 ## 问题理解

 我们得到了一组代表字母的整数，其中从 1 到 NM 的每个值只是全序字母表中的一个符号。 任务是将所有这些符号分成 N 个单词，每个单词恰好有 M 个字母，每次出现恰好一次。 

形成单词后，我们按字典顺序对单词进行排序，得到序列 s1, s2, …, sₙ。 然而，我们并没有被要求自己构造单词。 相反，对于每个位置 k，我们必须确定此排序顺序中第 k 个单词可以具有的最小可能值，假设我们可以自由地为每个 k 不同地重新划分多重集。 

关键的微妙之处在于每个 k 都是独立的。 我们并不是构建一个单一的最优分区；而是构建一个最优分区。 相反，对于每个 k，我们想象一个专门最小化 sₖ 的最佳对手分区。 

约束允许 NM 高达 10⁶，因此任何解决方案都必须接近线性或 n log n。 任何涉及重复排序或重复模拟分组的事情都太慢了。 即使是 O((NM)²) 的分区推理也是完全不可能的。 

当人们假设单个贪婪分区足以满足所有 k 时，就会出现天真的陷阱。 它不是。 不同的 k 值可能需要完全不同的字母全局分布。 例如，最小化第一个单词会鼓励尽早集中少量数字，而最小化中间单词可能需要平衡较早的单词，以便字典顺序发生变化。 

另一个微妙的边缘情况是频率高度倾斜时。 如果一个数字出现很多次，天真的分组可能会将其困在早期的单词中，人为地夸大后来的词典位置，除非我们明确地推理分配能力。 

## 方法

 强力策略是枚举所有可能的方法，将多重集划分为 N 个大小为 M 的组，然后对每个划分进行排序并记录第 k 个单词。 这在原则上是正确的，因为它探索了所有有效的配置，但分区的数量以组合方式增长。 即使对于适度的输入，分配的数量也是 (NM) 的量级！ / (M!)^N，这远远超出了任何可行的计算。 

关键的观察是我们实际上从来不需要单词的完整结构。 我们只关心排序列表中的一个特定位置。 这意味着我们可以考虑有多少单词可以在字典顺序上被迫小于候选单词。 

我们不是构造单词，而是以产生尽可能小的第 k 个元素的方式模拟贪婪地形成单词的效果。 如果我们固定一个阈值 x 并询问是否有可能至少 k−1 个单词严格小于以 x 开头的候选单词，则结构变得单调并且可以通过贪婪包装进行测试。 

这引出了一个建设性的想法：为了最小化第 k 个单词，我们希望在被迫构建第 k 个单词之前“花费”最小的可用符号来构建尽可能多的完全小单词。 一旦考虑到那些较早的单词，第 k 个单词就会在最佳包装下有效地强制消耗按字典顺序排列的下一个可用符号。 

一个关键的简化是，从全局角度来看，最优构造总是表现得贪婪。 我们对所有符号进行排序，然后按字典顺序从左到右重复填充单词，确保较早的单词消耗尽可能小的可用元素。 这最大限度地延长了我们可以延迟较大元素出现在早期词典位置的时间。 

然后，通过模拟这种贪婪打包直至形成 k 个单词的点来确定第 k 个单词，但这样做的方式要尊重剩余频率引起的容量限制。

 ## 算法演练

 我们将所有值视为排序符号数组上的频率。 

我们维护的结构始终允许我们首先获取最小的可用符号。 

1. 计算多重集中每个值的频率，并按升序对不同值进行排序。 
2. 构建表示所有剩余符号的全局多重集结构（概念上是最小堆或排序频率列表）。 这确保我们在构造单词时始终访问最小的未使用符号。 
3. 对于从 1 到 N 的每一个 k，我们模拟以贪婪的方式构造 k−1 个完整单词。 通过重复采用最小的可用符号来填充每个单词，直到选择 M 个为止。 这确保了这些单词在字典顺序上是最少的，因此尽可能小。 
4. 构造完 k−1 个单词后，我们开始构造第 k 个单词。 我们再次重复从剩余池中取出最小的可用符号，直到我们有 M 个元素。 该序列是最小可能的 sₖ。 
5. 输出构造出的第k个单词。 

这个模拟对于每个 k 独立有效的原因是我们总是假设一个最小化 sₖ 的最佳对手分区。 任何使前面的单词变大的偏差只能使后面的小元素可用，这无助于减少第 k 个单词。 

### 为什么它有效

 该过程保持贪婪最优不变量：在任何时候，在所有未使用的符号中，将最小的可用符号放置到任何单词中最早可能的位置不会损害最小化未来目标单词的能力。 这是因为字典顺序是从第一个位置开始比较的，因此延迟小符号只会增加较早单词的值，这在独立优化特定的第 k 个位置时是无关紧要的。 因此，总是消耗最小可用元素的构造首先为每个 k 生成按字典顺序最小的可行前缀配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import Counter
import heapq

def build_sorted_list(freq):
    heap = []
    for v, c in freq.items():
        heapq.heappush(heap, v)
    return heap

def extract_min(heap, freq):
    while heap:
        x = heap[0]
        if freq[x] > 0:
            return x
        heapq.heappop(heap)
    return None

def solve():
    N, M = map(int, input().split())
    arr = list(map(int, input().split()))
    
    freq = Counter(arr)
    heap = build_sorted_list(freq)
    
    # we maintain a working copy for simulation
    base_freq = dict(freq)
    
    def simulate(k):
        local = dict(base_freq)
        h = list(heap)
        heapq.heapify(h)
        
        def take():
            while h:
                x = h[0]
                if local[x] > 0:
                    local[x] -= 1
                    if local[x] == 0:
                        heapq.heappop(h)
                    return x
                else:
                    heapq.heappop(h)
            return None
        
        # build k-1 words
        for _ in range(k - 1):
            for _ in range(M):
                take()
        
        # build k-th word
        res = []
        for _ in range(M):
            res.append(take())
        
        return res
    
    for k in range(1, N + 1):
        ans = simulate(k)
        print(*ans)

if __name__ == "__main__":
    solve()
```该实现维护频率图和可用符号的最小堆。 这`simulate(k)`函数重建状态的本地副本，以便每个查询都是独立的，因为每个 k 允许不同的最佳分区。 

这`take()`函数是关键部分：它始终通过延迟丢弃耗尽的值来确保堆顶有效。 这避免了重复的重新堆化。 每次调用仅删除一次出现，符合贪婪构造规则。 

外循环只是打印每个 k 构造的第 k 个单词。 

一个微妙的点是每次模拟都是独立的，因此我们每次都复制频率状态。 这是必要的，因为跨 k 的重用会错误地耦合不同的最佳结构。

 我们考虑将多重集分割成 N = 4 个大小为 M = 3 的单词。 

| 步骤| 剩余最小元素| 形成的词 | 当前第 k 个构造 |

 ## 边缘情况

 当所有值都相同时，就会出现关键的边缘情况。 在这种情况下，无论分区如何，每个单词都是相同的，因此每个 sₖ 都是相同的序列。 该算法正确地以任何顺序使用相同的元素，因为当所有键都相等时，堆排序无关紧要。 

另一个边缘情况是当 M = 1 时。每个单词都是单个元素，因此对单词进行排序相当于对数组本身进行排序。 该算法退化为重复提取最小的剩余元素，从而直接产生正确的第k个最小值序列。 

当小值仅仅不足以填充早期单词时，就会出现更微妙的情况。 贪婪模拟确保一旦小值用完，后面的单词立即转移到更大的值，从而保留正确的词典结构，而不需要任何回溯或全局优化。
