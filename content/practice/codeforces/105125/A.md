 ## 解决方案
 ## 问题理解

 每个子句都是三个变量的乘积，其中每个变量都是`0`或者`1`。 整个表达式是所有子句值的总和。 我们被问到是否存在变量赋值使得该和为奇数。 如果存在这样的分配，我们还必须输出一个有效的分配。 

由于只有最终总和的奇偶性很重要，因此每次计算都会以模数进行`2`。 加法变成了 XOR，而乘法仍然是普通的乘法，因为操作数已经是位了。 因此，该表达式是一个多项式`GF(2)`仅由三次单项式组成。 

所有测试用例的变量和子句总数最多为`10^5`。 这立即排除了尝试每项作业，因为`2^100000`是完全不可行的。 即使算法仅对一半变量呈指数增长，也远远超出了限制。 我们需要一种与输入大小基本呈线性的算法。 

有几种情况很容易处理不当。 

假设同一个子句出现两次。```
1
3 2
1 2 3
1 2 3
```正确答案是`NO`。 两个子句总是评估为相同的值，因此它们的贡献是`0+0=0`或者`1+1=2`，两者均等。 一种仅检查某些子句是否可以成为的解决方案`1`会错误地回答`YES`。 

子句内的重复变量也需要小心。```
1
1 1
1 1 1
```该子句等于`x1³`，但自从`x1`是`0`或者`1`，这简直就是`x1`。 环境`x1=1`使表达式变得奇怪，所以正确答案是`YES`。 假设每个子句包含三个不同变量的任何论证在这里都会失败。 

另一个微妙的情况是每个子句都包含一个特定的变量。```
1
3 2
1 2 3
1 1 2
```环境`x1=0`立即迫使整个表达式变为零。 该算法必须正确地推理分配而不是计算子句的出现次数。 

## 方法

 蛮力方法会尝试每一种方法`2^n`作业。 对于每个作业，它都会评估所有`m`子句并检查所得总和是否为奇数。 这是正确的，因为它显式地测试了每个可能的分配，但其最坏情况的复杂性是`O(m·2^n)`，即使对于`n=40`，更不用说`100000`。 

关键的观察来自于将表达式视为多项式`GF(2)`。 

单项式贡献`1`仅当其中出现的每个变量都等于`1`。 假设我们重复选择仍然出现在任何地方的最大索引变量。 每个包含该变量的剩余单项式都将其作为其最大变量，因为子句索引满足`a ≤ b ≤ c`。 

将最大变量为的所有单项式分组在一起`xk`。 

他们的贡献有以下形式```
xk · P(previous variables)
```在哪里`P`是一个仅涉及索引小于的变量的多项式`k`。 

如果`P`永远可以成为`1`，我们只需选择`xk=1`并让整个团队做出贡献`1`。 如果`P`总是零，那么每个单项式包含`xk`是无关紧要的，可以被丢弃。 

这就给出了一个递归消除过程。 每个不同的单项式都被处理一次。 当没有变量剩余时，递归触底。 此时多项式要么全为零，要么全一。 

我们不需要显式地操作符号多项式，只需维护单项式列表并按最大变量递归地对它们进行分区即可。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |`O(m·2^n)`|`O(n)`| 太慢了 |
 | 最佳|`O(n+m)`|`O(n+m)`| 已接受 |

 ## 算法演练

 1. 用三个索引代表每个子句。 
2. 通过递归函数从最大索引到最小索引处理变量。 
3.对于当前最大的变量`xk`，将所有单项式分成最大变量为`k`以及那些不包含`k`作为他们最大的变量。 
4. 删除`k`来自第一组中的每个单项式。 单项式例如`(2,5,5)`变成`(2,5)`删除最大变量的一次出现后。 如果所有副本都消失，结果就变成常数单项式。 
5. 递归地确定约简多项式是否可以计算为`1`。 
6. 如果约简多项式可满足，则赋值`xk=1`并继续剩下的单项式。 
7. 否则分配`xk=0`。 每个单项式包含`xk`消失了，所以只剩下第二组。 
8. 继续，直到处理完所有变量。 
9. 在基本情况下，当多项式包含常数项时，多项式完全可满足`1`。 

### 为什么它有效

 每个包含当前最大变量的单项式的形式为`xk·M`， 在哪里`M`仅包含较小的变量。 如果`M`可以成为`1`， 环境`xk=1`精确激活由这些约简单项式表示的奇偶校验。 如果`M`始终为零，不分配`xk`可以使这些单项式有所贡献，因此可以安全地忽略它们。 由于每次递归调用都会从考虑中删除一个变量，因此最终每个单项式都会变成常数`1`或消失。 这保留了每一步多项式的奇偶性，因此只要原始表达式存在，最终的赋值就满足原始表达式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(1 << 25)

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n, m = map(int, input().split())

        poly = []
        for _ in range(m):
            poly.append(tuple(map(int, input().split())))

        assign = [0] * (n + 1)

        def dfs(var, terms):
            if var == 0:
                parity = 0
                for term in terms:
                    if len(term) == 0:
                        parity ^= 1
                return parity == 1

            with_var = []
            without_var = []

            for term in terms:
                if term and term[-1] == var:
                    with_var.append(term[:-1])
                else:
                    without_var.append(term)

            if dfs(var - 1, with_var):
                assign[var] = 1
                merged = without_var + with_var
                return dfs(var - 1, merged)

            assign[var] = 0
            return dfs(var - 1, without_var)

        if dfs(n, poly):
            out.append("YES")
            out.append(" ".join(map(str, assign[1:])))
        else:
            out.append("NO")

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```递归函数完全遵循前面描述的消除过程。 每个级别处理一个变量，根据单项式是否以该变量结尾对单项式进行分区，并在必要时删除一次出现的情况。 

将每个单项式表示为元组很方便，因为删除最大变量变得简单`term[:-1]`。 由于索引已经排序，因此最大的变量始终是最后一个元素。 

基本情况值得特别关注。 只有空元组代表常数项。 它们的奇偶性决定了剩余多项式是等于一还是零。 

递归深度最多为`n`，因此递归限制相应增加。

 ## 边缘情况

 考虑重复子句的示例。```
1
3 2
1 2 3
1 2 3
```两个子句是相同的，所以结束`GF(2)`他们互相抵消。 在消除过程中，它们一直保持配对直至恒定水平，其中两个常数项也相互抵消。 算法正确输出`NO`。 

现在考虑重复变量。```
1
1 1
1 1 1
```递归消除删除变量的一份副本`1`直到单项式成为常数项。 由于该常量仅出现一次，因此基本情况报告成功，并生成分配`x1=1`。 

最后，考虑一个被一个变量强制为零的子句。```
1
3 1
1 2 3
```如果递归检查确定激活最高变量是有益的，则它设置`x3=1`。 否则它设置`x3=0`，立即将该条款从考虑中删除。 这与乘以二进制变量的代数行为完全匹配，因此每个分支都保持正确性。
