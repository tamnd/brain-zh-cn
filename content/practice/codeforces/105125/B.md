 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例描述一组射击车道，其中车道`i`包含`A_i`目标排列成一行。 蒂姆将进行一系列射击，每个射击都被分配到一个车道上。 

射击机制很不寻常，因为射击的结果取决于它是整体的第一、第三、第五等射击还是第二、第四、第六等射击。 在奇数次射击中，所选车道上的第一个剩余目标将被摧毁。 在偶数次射击中，第一个剩余目标将被跳过，第二个剩余目标将被摧毁（如果存在）。 如果此时该泳道不包含第二个目标，则该射击无效，因为它会错过所有目标，并且禁止此类射击。 

任务是确定是否存在所有射击的一系列车道选择，可以精确地摧毁每个目标一次，而不会做出无效的移动。 如果存在，我们还必须输出一个有效的序列。 

限制很大：在所有测试用例中，车道总数达到三十万，目标总数达到五十万。 这排除了任何通过重复扫描车道来模拟镜头的策略，或者维护依赖于线性搜索的昂贵的每次镜头状态更新的策略。 任何解决方案都必须在目标数量基本呈线性的时间内运行。 

当局部思考时，会出现一个微妙的问题。 一个天真的想法是总是在任何仍然有目标的车道上射击。 这会立即失败，因为偶数射击的行为不同，并且可能会跳过车道中唯一剩余的目标，从而有效地使某些配置无法完成，即使目标仍然存在。 

在小情况下会出现另一种故障模式。 例如，如果一条车道上有两个目标，人们可能期望两次射击总是有效，但第二次射击变成偶数，并尝试击中第一次移除后不存在的第二个目标，即使目标总数为偶数，也会导致失败。 

第三个重要的边缘情况是所有车道都只有一个目标。 在第一次射击移除一个目标后，第二次射击必须在仍然至少有两个剩余目标的车道上进行，这是不可能的，因此对于分布在大小为 1 的车道上的多个目标，答案立即是否定的。 

这些例子表明，答案不仅取决于总计数，还取决于我们是否能够在整个过程中维持一系列有效的均匀射击。 

## 方法

 暴力策略将模拟所有可能的车道选择序列。 在每一步中，我们都会选择在当前全局镜头索引奇偶性的情况下仍然允许有效镜头的任何通道。 我们将维护每个通道中剩余目标的列表，并递归地或通过状态的 BFS 尝试所有可能的选择。 

这种方法原则上是正确的，因为它直接探索配置的状态空间。 然而，分支因子与非空通道的数量成正比，并且每个转换都会修改通道结构。 即使我们使用高效的数据结构，状态数量也会随着镜头数量呈指数级增长，因为每次镜头都会以影响未来有效性的方式改变一条通道的结构。 目标总数高达五十万，这是完全不可行的。 

关键的观察结果是奇偶校验约束创建了严格的配对结构。 每个车道不是独立的； 相反，它的目标被消耗的模式是奇数射击消除第一个目标，偶数射击消除第二个剩余目标。 这意味着每个通道的行为就像一系列成对的删除，除了可能的剩余元素。 只有当我们能够安排射击时，系统才会保持一致，以便每当我们在一条车道上进行均匀射击时，该车道必须已经有至少两个剩余目标。

这导致了一个必要的全局条件：在奇数次射击后的每一时刻，我们必须确保保留足够的结构，以便下一次偶数射击始终可以在有效的地方执行。 保证这一点的唯一方法是以一致的交替消耗模式对车道内的目标进行配对，这减少了对计数的简单可行性条件的检查。 

事实上，每个车道贡献一定数量的“配对机会”，等于楼层（A_i / 2）。 这些对应于安全的均匀消耗。 奇数车道中剩余的单个目标必须仅由奇数射击消耗，并且一旦我们确定了序列长度，奇数射击的总数就固定了。 当我们可以安排所有配对消耗而不会用完可以支持均匀射击的通道时，该系统正是可行的。 

这将问题简化为过度计数的贪婪构造，保持当前有多少车道有可用的第二目标，而不是没有可用的第二目标。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 指数| O(N + A) | 太慢了 |
 | 具有计数的贪婪构造 | O(A) | O(N) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 计算每个车道是否至少有两个目标。 我们维护两组：具有一个目标的车道和具有至少两个目标的车道。 这种区别很重要，因为只有第二组可以支持偶数镜头。 
2. 我们通过迭代从 1 到 A 的总镜头来模拟构建镜头序列，其中 A 是目标总数。 我们总是决定当前镜头的车道。 
3.如果当前射击索引为奇数，我们可以摧毁任何仍有剩余目标的车道上的第一个目标。 我们选择当前至少有一个目标剩余的任何车道。 如果可能的话，我们更喜欢至少有两个目标的车道，因为保留单目标车道中的结构可以防止以后出现死胡同。 如果没有车道有目标，施工就会失败。 
4. 如果当前射击指数为偶数，则必须在某个车道上执行第二目标去除。 仅当存在至少一条剩余尺寸至少为 2 的车道时，这才有效。 我们选择这样一条车道并删除其第二个剩余目标。 如果不存在这样的车道，则无法进行施工。 
5. 每次删除后，我们都会更新车道的剩余计数，并根据需要在“至少两个”和“一个或零”类别之间移动。 
6. 我们记录每个镜头所选择的车道索引，构建最终序列。 

核心思想是奇数步是灵活的消耗步，而偶数步是需要车道结构深度的约束操作。 贪婪策略总是优先考虑保持深度。 

它的工作原理与车道状态的不变量有关。 在任何时间点，每条车道都处于两种状态之一：它要么至少有两个剩余目标，要么最多有一个。 即使是投篮也只能由第一状态提供。 该算法确保每当需要均匀射击时，我们都不会在第一个状态下耗尽所有车道。 这是有保证的，因为我们总是避免在所有偶数操作得到满足之前将所有具有剩余目标的通道折叠成单例。 使用更深通道的贪婪偏好首先保留了未来偶数步骤的可行性，并且由于每个通道单调地从较大计数过渡到较小计数，因此之前恢复的能力不会丢失或重新生成，从而使可行性条件单调。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        total = sum(a)
        if total == 0:
            print("NO")
            continue
        
        # We maintain two lists of lanes by remaining count
        # store (remaining, index)
        from collections import deque
        
        big = deque()
        small = deque()
        
        for i, x in enumerate(a):
            if x >= 2:
                big.append([x, i])
            elif x == 1:
                small.append([x, i])
        
        res = []
        
        # helper: get any valid lane for odd/even
        def pop_big():
            while big and big[0][0] <= 1:
                small.append(big.popleft())
            if not big:
                return None
            x, i = big[0]
            big[0][0] -= 1
            return i
        
        def pop_any():
            while big and big[0][0] == 0:
                big.popleft()
            if big:
                big[0][0] -= 1
                return big[0][1]
            while small:
                if small[0][0] == 0:
                    small.popleft()
                else:
                    small[0][0] -= 1
                    return small[0][1]
            return None
        
        for shot in range(1, total + 1):
            if shot % 2 == 0:
                lane = pop_big()
                if lane is None:
                    print("NO")
                    break
                res.append(lane + 1)
            else:
                lane = pop_any()
                if lane is None:
                    print("NO")
                    break
                res.append(lane + 1)
        else:
            print("YES")
            print(*res)

for _ in range(1):
    solve()
```该实现维护两个双端队列，根据通道当前是否有足够的深度来支持均匀射击来分隔通道。 这`pop_big`函数强制即使射击也总是从至少有两个剩余目标的车道消耗，而`pop_any`用于可以消耗任何可用目标的奇怪射击。 

一个微妙的点是双端队列存储可变的剩余计数，因此当计数降至零时我们依赖于延迟清理。 这避免了重复扫描或重新平衡结构。 正确性依赖于每个通道的剩余计数只会减少的事实，因此可以安全地跳过过时的条目。 

奇数和偶数镜头处理之间的顺序至关重要。 如果甚至允许从任意车道消耗子弹，那么在只剩下单例车道的情况下，构造将立即中断。

 ## 边缘情况

 一种关键的边缘情况是所有车道都只有一个目标。 该算法立即将所有奇数射击路由到单例消耗中，但第一个偶数射击变得不可能，因为没有车道包含第二个目标。 该结构正确地识别了这一点，因为`pop_big`在第一个偶数步失败。 

另一个边缘情况是单车道有两个目标。 第一次射击会创建一个有效状态，但第二次射击是偶数，并且必须消耗不再存在的第二个目标。 该算法捕获了这一点，因为在第一次删除后车道从“大”过渡到“小”，没有为第二次操作留下有效的源。 

例如，当目标总数足够多但分布不均匀时，就会出现较大的结构边缘情况`[3, 3, 1]`。 局部贪婪消耗可能会在满足所有偶数步骤之前意外地将所有通道减小为一。 该算法通过始终优先选择更深的通道进行消耗来避免这种情况，并在最后步骤之前保留至少一个有效的偶数通道。
