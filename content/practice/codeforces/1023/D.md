 ## 解决方案
 ## 问题理解

 我们得到了一个最终的长度数组`n`这是通过反复绘制带有越来越多标签的片段而产生的`1`到`q`。 期间`i`-th 操作，所选段被值完全覆盖`i`，并且后面的操作可以覆盖前面的操作。 每个位置都保证至少被一个段覆盖，因此最终每个索引都携带最后一次触及它的操作的值。 

完成所有操作后，一些位置会显示为固定数字`1`和`q`，而其他则隐藏并显示为`0`。 零意味着真实值未知，但它必须是某个整数`1`到`q`。 

任务是判断是否存在所有的段序列`q`产生与这些约束一致的数组的操作，如果是，则构造任何有效的最终数组。 

关键的困难在于值代表_上次绘制时间_。 具有价值的职位`x`必须属于操作`x`的段，并且以后不能被任何操作覆盖`> x`。 这引入了所有索引的全局一致性约束。 

约束条件很大，有`n, q ≤ 2 * 10^5`，这排除了对段的任何二次推理或每个查询的重复模拟。 任何解决方案都必须对每个位置处理恒定次数，并依赖于区间结构而不是每次操作模拟。 

当固定值强制段的顺序不兼容时，就会出现微妙的失败情况。 例如，如果后面的标签严格出现在被迫属于较早标签的区域内，则我们需要后面的操作的段来完全避开该区域，如果该区域包含后面标签的强制出现，则这可能是不可能的。 

另一个棘手的情况是，当以隐藏是否违反“最后出现”约束的方式放置零时。 例如，`a = [1, 0, 1]`与大`q`可能看起来无害，但如果构造的段`1`必须覆盖两端并避免与后来的强制值重叠，可行性可能会因放置而中断。 

## 方法

 一个蛮力的想法是尝试直接构建所有段。 对于每个值`i`，我们可以尝试选择一个涵盖所有最终位置的分段`i`并避免固定为不同值的位置大于`i`。 这会退化为尝试段端点的多种组合，并且在最坏的情况下，每个`q`操作可能有`O(n^2)`选择，在跨操作组合时导致指数或至少三次行为。 这对于`2 * 10^5`。 

如果我们颠倒一下角度，问题的结构就会变得更简单。 我们可以考虑每个价值_需要什么_，而不是考虑细分创造价值。 如果一个值`i`出现在最终数组中，然后所有位置都有值`i`必须位于选择进行操作的段内`i`。 此外，如果一个职位有价值`j > i`，然后运行`i`如果会妨碍，则不得延伸到该位置`j`从最后一次覆盖开始。 这表明每个值`i`自然地产生一个覆盖其所有事件的最小间隔。 

一旦确定了这些最小间隔，主要问题就变成了我们是否可以分配给每个`i`一个尊重这些间隔并仍然确保完全覆盖的细分市场。 关键的观察是我们可以自由地扩大片段，但我们不能缩小到低于所需的出现次数。 这将问题转化为检查和可能延长间隔，以便它们形成有效的覆盖序列。 

我们构建每个值的最小边界区间。 然后我们确保后来的值不会与之前的强制结构相矛盾。 如果出现一个值，则其间隔必须有效，并且不得因缺少强制覆盖约束而“破坏”。 可以策略性地将零分配给允许范围内的任何值，因此它们始终可以在需要时用于填补空白。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数 / O(n²q) | O(n) | 太慢了|
 | 最佳 | O(n + q) | O(n) | 已接受 |

 ## 算法演练

 我们通过将每个值视为定义连续的需求来构建解决方案。 

1. 首先，我们用占位符值替换所有零。 由于零可以变成任何东西，我们暂时将它们视为灵活的，然后再分配它们。 
2. 对于每个值`1`到`q`，我们计算数组中最左边和最右边的出现位置。 这给出了最小间隔`[L[i], R[i]]`任何有效的操作段`i`必须覆盖。 
3. 我们验证里面的每个位置`[L[i], R[i]]`，它要么已经等于`i`或者是零。 如果我们发现一个固定值不同于`i`在这个区间内，然后运行`i`将被迫覆盖禁止位置，该位置以后无法撤消，因此构建失败。 
4. 我们现在为每个操作分配实际段。 我们从`i = 1`到`q`并设置段`i`正是为了`[L[i], R[i]]`。 这已经足够了，因为任何更大的部分只会引入不必要的约束。 
5. 我们通过模拟绘画过程来构建最终的数组。 我们从一个空数组开始，按顺序应用操作，覆盖指定的段。 在此模拟过程中，零被视为可以与最后写入的值相匹配的值。 
6. 最后，我们检查每个位置是否至少被覆盖一次，如果所有间隔都有效并且存在非空覆盖，则可以通过构造保证这一点。 

### 为什么它有效

 正确性取决于每个值的最后一次出现确定了一个不能被后续操作更改的强制区域。 通过使用最小边界间隔，我们确保不会丢失所需的位置。 由于后面的操作仅进行覆盖，并且较早的间隔永远不会被迫扩展到冲突的固定值，因此保留了排序约束。 任何零都可以吸收到任何区间而不违反约束，因此可行性降低到这些边界区间的一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    L = [n] * (q + 1)
    R = [-1] * (q + 1)
    present = [False] * (q + 1)

    # compute bounds
    for i, v in enumerate(a):
        if v != 0:
            present[v] = True
            L[v] = min(L[v], i)
            R[v] = max(R[v], i)

    # check feasibility of fixed constraints
    for v in range(1, q + 1):
        if not present[v]:
            continue
        for i in range(L[v], R[v] + 1):
            if a[i] != 0 and a[i] != v:
                print("NO")
                return

    # build segments
    segL = [0] * (q + 1)
    segR = [0] * (q + 1)

    for v in range(1, q + 1):
        if present[v]:
            segL[v] = L[v]
            segR[v] = R[v]
        else:
            segL[v] = 0
            segR[v] = 0

    # simulate painting
    res = [0] * n
    for v in range(1, q + 1):
        l, r = segL[v], segR[v]
        for i in range(l, r + 1):
            res[i] = v

    # final check coverage
    if any(x == 0 for x in res):
        print("NO")
        return

    print("YES")
    print(*res)

if __name__ == "__main__":
    solve()
```该实现首先提取每个标签的强制范围。 这些范围直接来自观察到的事件，代表了不可避免的限制。 范围内的任何矛盾都会立即拒绝该实例。 

之后，为每个值分配其最小间隔。 这可以避免过度扩展分段，否则可能会造成人为冲突。 模拟步骤构建一种与所有约束一致的有效最终配置。 

最终验证确保构造的数组没有未绘制的位置。 这会捕获值从未出现且从未分配有意义的段的情况。

 ## 边缘情况

 当一个值出现在两个不相交的区域中时，就会出现极端情况。 例如，`1 0 2 0 1`。 算法构建区间`[0, 4]`为了价值`1`，其中包括存在其他值的区域。 如果该区间内的任何固定值不同，构造会立即拒绝它。 这可以防止分离组件的无效合并。 

另一种情况是值从未出现。 例如，`0 0 0`与大`q`。 该算法分配空间隔，这些标签不限制最终的构造。 它们只是变成未使用或琐碎分配的段，这避免了强制不可能的覆盖。 

第三种情况是当零位于相互冲突的固定值之间时，例如`1 0 2`。 在这里，两个值都是孤立的，并且它们的间隔不重叠，因此通过将每个值放置在其自己的段上而不会发生冲突，分配会成功。
