 ## 解决方案
 ## 问题理解

 我们给出了一系列完全规则的玩具价格：玩具的价格从 1 到 n，每个整数价格只出现一次。 我们想要计算有多少对不同的无序玩具的总价格恰好等于 k。 每对由两个不同的整数 a 和 b 确定，其中 1 ≤ a < b ≤ n 且 a + b = k。 

输出只是此类有效对的数量。 

约束是极端的：n 和 k 都可以大到 10^14。 这立即排除了任何迭代玩具范围的方法。 最多 n 的线性扫描需要最多 10^14 次操作，这远远超出了任何可行的时间限制。 甚至对数或 n 多项式方法也是不必要的； 问题的结构表明我们应该避免完全迭代域，而是对有效对进行代数推理。 

这里最常见的微妙故障案例来自边界处理。 一个天真的尝试可能会计算一个有效的伙伴 b = k - a 并计算 [1, n] 中的所有 a，这样 b 也在 [1, n] 中。 当忘记对必须满足 a < b 时，就会出现错误，否则对将被重复计算。 另一个常见错误发生在允许 a = b 时，这种情况仅在 k 为偶数且 a = k/2 时发生。 即使它位于范围内，也必须将其排除。 

例如，如果 n = 8 且 k = 5，则有效对为 (1, 4) 和 (2, 3)。 如果粗心地实现对所有有效 a 进行计数而不强制执行 a < b，则会同时对 (1, 4) 和 (4, 1) 进行计数，从而使答案加倍。 类似地，如果 n = 10 且 k = 10，则除非明确排除，否则 (5, 5) 对将被错误地包括在内。 

## 方法

 蛮力的想法很简单：尝试从 1 到 n 的所有可能的第一个玩具 a，计算 b = k - a，并检查 b 是否位于 [a + 1, n] 中。 这是正确的，因为它直接强制执行所有约束。 然而，它需要迭代所有n个值，导致时间复杂度为O(n)，当n达到10^14时，这变得不可能。 

关键的观察是，一旦 a 固定，b 就完全确定，因此问题实际上是计算同时满足两个不等式的有效整数 a：b = k - a 必须位于 [1, n] 中，并且 a < b 也必须成立。 这些条件将问题转换为相交的整数区间。 

从 b = k - a，约束 1 ≤ b ≤ n 变为 1 ≤ k - a ≤ n，重新排列为 k - n ≤ a ≤ k - 1。约束 a < b 变为 a < k - a，或等效的 2a < k，这意味着 a ≤ (k - 1) // 2。最后，根据有效玩具的定义，我们已经得到 1 ≤ a ≤ n。 

所以 a 必须位于三个区间的交点处：

 1≤a≤n

 k - n ≤ a ≤ k - 1

 1 ≤ a ≤ (k - 1) // 2

 一旦知道了这个交集，答案就是其中的整数个数，即 max(0, R - L + 1)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n) | O(1) | O(1) | 太慢了|
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算有效 a 值的下限为 L = max(1, k - n)。 这确保了 b = k - a 不超过 n。 
2. 计算有效 a 值的上限，即 R = min(n, k - 1)。 这确保 b 至少为 1。 
3. 通过将上限收紧至 R = min(R, (k - 1) // 2) 来强制执行独特性条件 a < b。 
4. 如果 L > R，则不存在满足所有约束的有效整数 a，因此答案为 0。 
5. 否则，有效a的个数为R - L + 1。 

这些步骤以这种方式排序的原因是每个约束独立地限制 a 的可能值。 取所有有效间隔的交集可以保证我们准确地计算那些同时满足所有条件的对。 

### 为什么它有效

每个有效对 (a, b) 恰好对应于一个整数 a，并且 a 必须满足所有导出的不等式。 相反，最终区间中的每个整数 a 都会生成唯一有效的 b = k - a，该 b = k - a 位于范围内并且严格大于 a。 该转换将原始组合计数问题简化为对单个区间内的整数点进行计数，并且在此映射中不会丢失或重复任何有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())

L = max(1, k - n)
R = min(n, k - 1)
R = min(R, (k - 1) // 2)

ans = max(0, R - L + 1)
print(ans)
```代码直接实现了算法中导出的区间交集。 第一个界限强制第二个玩具存在于最高价格 n 内。 第二个确保积极性，并且 a 和 b 在排序形式上是不同的。 第三个强制 a < b，删除对称重复项和无效的中点情况。 

一个微妙的实现细节是应用约束的顺序：虽然交集在数学上是可交换的，但最后应用对称约束可以避免意外计算无效的中点值，例如当 k 为偶数时的 k/2。 

## 工作示例

 ### 示例 1

 输入：n = 8，k = 5

 | 步骤| 左 | R（对称前）| R（对称后）|
 | --- | --- | --- | --- |
 | 初始| 1 | 4 | 2 |

 我们计算 L = max(1, 5 - 8) = 1。R 开始为 min(8, 4) = 4。然后我们强制对称性：(k - 1) // 2 = 2，因此 R 变为 2。最终有效的 a 值为 {1, 2}，生成对 (1, 4) 和 (2, 3)。 

这证实了间隔方法正确地避免了对反向或无效对的计数，同时仍然捕获所有有效对。 

### 示例 2

 输入：n = 10，k = 10

 | 步骤| 左 | R（对称前）| R（对称后）|
 | --- | --- | --- | --- |
 | 初始| 1 | 9 | 4 |

 这里 L = max(1, 10 - 10) = 1。对称之前的 R 是 min(10, 9) = 9。应用对称之后，R = 4。有效的 a 值为 {1, 2, 3, 4}，对应于对 (1,9), (2,8), (3,7), (4,6)。 无效的中点 (5,5) 将被自动排除。 

这演示了条件 a < b 是如何纯粹通过边界而不是显式检查来强制执行的。

 ## 边缘情况

 一个关键的边缘情况是 k 很小时，具体来说 k ≤ 2。在这种情况下，k - n 总是 ≤ 0，因此 L 变为 1，但 R 变为 min(n, k - 1)，最多为 1。如果 k = 1 或 k = 2，R ≤ 1，并且在强制 a < b 后，R 变为 0，产生空区间。 算法正确返回 0。 

另一个重要的情况是当 k 相对于 n 非常大时，例如 n = 5 和 k = 100。那么 k - n = 95，因此 L = 95，而 R = min(5, 99) = 5。由于 L > R，区间为空，正确地产生 0。这反映出 [1, n] 中没有两个数字之和可以达到如此大的 k。 

最后一个微妙的情况是 k 是偶数且 k/2 位于 [1, n] 范围内。 例如，n = 10，k = 10 给出中点 5。区间构造最初包括 a = 5，但强制 R ≤ (k - 1) // 2 将删除它。 这保证了我们永远不会计算无效的自对 (5, 5)，从而在没有特殊情况分支的情况下保持正确性。
