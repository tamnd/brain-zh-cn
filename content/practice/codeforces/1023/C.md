 ## 解决方案
 ## 问题理解

 我们得到了一个正确平衡的括号序列，这意味着字符串的每个前缀的右括号永远不会多于左括号，并且总体计数完美匹配。 从这个长序列中，我们被要求提取一个较短的固定偶数长度的序列，保持字符的原始顺序，这样得到的序列仍然是一个有效的平衡括号序列。 

我们允许执行的操作是删除：我们从原始字符串中删除一些字符，但不允许我们重新排序任何内容。 在删除字符的所有可能方法中，我们必须生成长度恰好为 k 的任何有效平衡序列。 

约束条件很大，n 高达 200,000。 任何尝试探索组合或重新计算许多候选子序列有效性的解决方案都会太慢。 这立即表明我们需要一次线性扫描或非常接近它的东西，因为 O(n log n) 或更糟的情况是可以接受的，但任何二次的都不能接受。 

经常出现的一个天真的错误是试图贪婪地选择括号，而只检查本地平衡，而不考虑以后的选择可能会导致失败。 例如，在不考虑结构的情况下，始终采用最早可用的“(”，直到中途，然后采用“)”，可能会在微妙的情况下破坏有效性，在这种情况下，早期消耗“(”会阻止正确完成更深层次的嵌套。另一种失败模式是尝试维护所有可能子序列的堆栈并稍后进行修剪，这会导致组合爆炸。

 关键的困难不仅在于选择 k 个字符，还在于确保剩余的前缀结构仍然允许有效的平衡形成。 

## 方法

 强力的观点是考虑长度为 k 的所有子序列并测试每个子序列是否是有效的括号序列。 检查有效性需要 O(k)，并且子序列的数量是组合的，因此即使对于小输入，这也很快变得不可能。 即使限制于贪婪生成，如果我们尝试通过探索两种可能性（采用或跳过）来选择每个下一个字符，我们仍然面临指数分支。 

常规括号序列的结构给出了一个关键的简化：在任何时候，如果我们在构建子序列时保持平衡，我们只需要确保我们永远不会关闭比打开的更多，并且我们仍然可以完成序列。 由于我们知道最终长度是固定且均匀的，因此我们可以考虑按顺序精确选择 k/2 个左括号和 k/2 个右括号。 

关键的观察是，我们不需要保留原始序列的所有有效前缀，只需保留有效的子序列。 这使我们能够在维持单个余额计数器的同时贪婪地构建答案。 当从左到右选择字符时，我们确保永远不会超过 k 个字符，并且永远不会违反有效序列的前缀条件。 

贪婪的想法变成：我们扫描原始字符串并决定是否取每个字符。 我们维护已经选择的字符数量并确保不超过 k。 我们还保持平衡，以便在任何时候我们都不会相对于空位使用太多的右括号。 由于原始字符串已经有效，因此我们始终保证前面存在足够的结构来完成平衡序列。

 ## 算法演练

 我们在构建答案时从左到右处理字符串。

1. 初始化一个空结果字符串和两个计数器：到目前为止我们已经取了多少个字符，以及构造序列中左括号和右括号的当前余额。 
2. 对于输入字符串中的每个字符，决定是否将其包含在结果中。 仅当我们仍未达到长度 k 时，我们才考虑采用它。 这确保我们永远不会超过所需的输出大小。 
3. 如果字符是左括号，只要我们仍然需要字符，我们就可以安全地使用它。 我们追加它并增加余额。 这是安全的，因为添加左括号永远不会违反有效性。 
4. 如果该字符是右括号，则仅在不破坏有效性的情况下才采用它，这意味着当前余额为正。 这确保我们永远不会创建右括号超过左括号的前缀。 
5. 继续，直到我们恰好选择了 k 个字符。 由于 k 是偶数并且保证存在有效答案，因此我们将在扫描结束之前或恰好在结束时成功。 

这种贪婪选择背后的原因是我们只强制执行局部有效性约束。 我们绝不承诺做出无法保持平衡的选择。 

### 为什么它有效

 构造的序列始终保持其前缀是有效的部分括号序列的不变性。 每次添加 ')' 时，我们都会确保有一个不匹配的 '(' 可用，因此余额永远不会变为负值。每个 '(' 都会增加未来 ')' 的可用容量。由于我们恰好在长度 k 处停止并且原始序列是平衡的，因此我们永远不会被迫陷入少于 k 个字符可以形成有效前缀的死胡同。解决方案存在的保证确保了这种贪婪的选择永远不会阻止完成。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = input().strip()

    res = []
    balance = 0

    for ch in s:
        if len(res) == k:
            break

        if ch == '(':
            res.append(ch)
            balance += 1
        else:
            if balance > 0:
                res.append(ch)
                balance -= 1

    print("".join(res))

if __name__ == "__main__":
    solve()
```代码维护了答案的运行结构。 这`res`list 存储选择的字符，一旦它的大小达到 k，我们立即停止。 平衡变量确保我们永远不会附加右括号，除非它与之前选择的左括号匹配。 

微妙的一点是，我们从未明确跟踪还需要多少个左括号或右括号。 该约束由原始序列的有效性以及我们只接受有效前缀步骤的事实隐式处理。

 ## 边缘情况

 一种边缘情况是原始序列已经恰好达到所需的长度 k。 该算法只是简单地获取遇到的每个有效括号，同时保持平衡，并且由于在 k 之前没有触发提前停止，因此完整字符串不变地返回。 

另一种情况是最佳子序列需要跳过早期的有效对以保留后面的结构。 例如，在一个长序列中`"()(())()"`当 k 小于 n 时，当余额为零时，算法可能会跳过一些有效的右括号，但仍然保留足够的结构来达到 k 个字符。 平衡永远不会变为负数的不变性确保我们永远不会将自己锁定在无效的前缀中，即使平衡条件隐式地做出了跳过决策。
