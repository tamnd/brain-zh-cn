 ## 解决方案
 ## 问题理解

 我们被赋予一个未知数$n \times n$网格中的每个单元格要么是开放的，要么是阻塞的。 仅允许从单元格移动到其右侧邻居或其底部邻居，并且仅当目标单元格打开时才允许移动。 目标是准确地使用确定从左上角单元格到右下角单元格的任何有效路径$n-1$向下移动并且$n-1$向右移动。 

问题是我们无法直接看到网格。 相反，我们可以查询矩形可达性：给定两个单元格$(r_1, c_1)$和$(r_2, c_2)$，我们被告知网格内是否存在从第一个到第二个的单调路径（仅向右/向下移动）。 但是，查询受到限制：查询端点之间的曼哈顿距离必须至少为$n-1$，这会阻止检查任意小的子问题。 

该问题保证至少有一条有效路径$(1,1)$到$(n,n)$存在。 我们必须使用最多重建任何这样的路径$4n$查询。 

一个天真的误解是将此视为查询后具有完全可见性的最短路径问题。 但这失败了，因为我们无法局部探测小区域； 例如，试图决定是向右走还是向下走$(1,1)$通过检查$(2,2)$是非法的，当$n > 3$，因为曼哈顿距离约束阻止了这种细粒度的查询。 

核心困难在于局部决策无法直接观察； 每个查询都必须跨越很长的距离，因此我们必须进行全局推理并减少结构化步骤中的不确定性。 

## 方法

 一个蛮力的想法是重建每个单元对的可达性，或者通过重复查询连接来模拟未知空闲单元上的 BFS。 这是不可能的，因为即使是一次可达性检查也是昂贵且受限的，而且我们需要$\Theta(n^2)$州，远远超过$4n$查询限制。 

关键的结构见解是我们不需要完全重建网格，甚至不需要知道所有可达状态。 我们只需要一条从左上角到右下角的单调路径。 任何有效路径均由以下部分组成$n-1$向右移动并且$n-1$向下移动，那么问题就变成了决定每一步的方向，同时确保前缀仍然可以扩展到目的地。 

我们不检查局部可行性，而是在固定数量的步骤后维护可到达单元格的边界。 在任何时候，所有有效路径都位于一个“层”上，其中$r + c$是恒定的。 我们可以使用大矩形查询来测试是否存在从候选下一个单元格到目的地的路径，由于曼哈顿约束，这是允许的。 这让我们决定选择一个方向是否可以保持全局可达性。 

我们贪婪地构建路径$(1,1)$到$(n,n)$，并且在每一步中，我们尝试向右或向下移动，并验证哪个选项仍然允许到达目的地。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（全局重建）|$O(n^2)$查询 |$O(n^2)$| 太慢了|
 | 贪婪的可达性查询 |$O(n)$查询 |$O(1)$| 已接受 |

 ## 算法演练

 该解决方案逐步构建路径，始终保持在到达右下角的有效单调路线内。 

1. 从单元格开始$(1,1)$并维护一个字符串$S$为答案路径。 我们将准确附加$2n-2$移动。 
2. 各位置$(r,c)$，考虑两种可能的下一步行动：$(r, c+1)$并下降到$(r+1, c)$，只要它们留在网格内。 
3. 为了在它们之间做出决定，我们测试向右移动是否仍然允许到达$(n,n)$。 我们询问是否存在一条从$(r, c+1)$到$(n,n)$网格内。 这是使用单个矩形查询完成的。 
4. 如果正确的举动是有效的（答案是“是”），我们就采取它。 否则，我们就必须下去。 存在完整路径的保证确保两个方向中的至少一个始终保持端点可达。 
5. 将选择的动作附加到$S$，更新当前单元格，重复直到达到$(n,n)$。 
6. 输出构造好的字符串。 

关键的想法是每一步都保持全球可行性。 我们永远不会选择将当前位置与目的地断开的移动。 

### 为什么它有效

 在任何时候，假设我们位于一个位于某个有效单调路径上的单元格$(1,1)$到$(n,n)$。 该路径必须从当前单元格向右或向下继续。 如果我们测试这两种可能性，至少其中一种可以保证目的地可达。 大矩形上的可达性查询可以正确检测是否存在这样的延续，因此贪婪选择永远不会消除所有有效的完成。 这一不变量确保我们始终保持在至少一条有效路径上直到最后。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
print = sys.stdout.write

def ask(r1, c1, r2, c2):
    sys.stdout.write(f"? {r1} {c1} {r2} {c2}\n")
    sys.stdout.flush()
    return sys.stdin.readline().strip()

def solve():
    n = int(input().strip())
    
    r, c = 1, 1
    path = []

    for _ in range(2 * n - 2):
        if r < n:
            res = ask(r + 1, c, n, n)
            if res == "YES":
                path.append('D')
                r += 1
                continue

        path.append('R')
        c += 1

    sys.stdout.write("! " + "".join(path) + "\n")
    sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该代码维护当前位置并增量构建答案字符串。 功能`ask`包装交互处理并确保每次查询后刷新，这在交互问题中至关重要。 

决策逻辑始终优先检查向下移动是否保持可达性$(n,n)$。 如果没有，则默认向右移动。 这个顺序是任意的； 只要保持一致性，交换方向也有效。 

循环完全运行$2n-2$步骤，匹配所需的路径长度。 每个步骤都会减少剩余的行距离或列距离，从而保证终止。

 ## 边缘情况

 一个边缘情况是第一行大部分被阻塞，除了强制进入第二行之外。 该算法会处理此问题，因为每次测试向下移动时，查询都会立即返回 NO，直到到达强制列，强制向右移动，直到唯一可行的转换。 

另一种情况是最佳路径频繁改变方向。 由于每个决策都是独立的并且基于最终单元格的可达性，因此该算法永远不会做出局部最优但全局无效的选择； 它只检查是否存在延续，从而保留了可行性。 

最后一种情况是，这两种举措在早期阶段都是可能的。 贪心规则任意选择一个（在此实现中向下）。 即使这导致更长的水平绕行，不变式也保证完整路径仍然存在，因为可达性查询仅拒绝会隔离目的地的移动。
