 ## 问题理解

 我们正在处理一个最初为空或填充零的一维数组，并且要求我们处理两种操作。 一个操作通过在其位置上添加算术级数来更新连续段。 另一个操作要求在应用所有先前的更新后从数组派生的值或聚合信息。 

算术级数更新意味着当我们从位置选择一个段时`l`到`r`，我们不是添加一个常量值，而是添加一个从某个值开始的序列`l`当我们向右移动时，会增加一个固定的差异。 所以位置`l`得到第一项，`l+1`得到第一项加上公差，依此类推，直到`r`。 

挑战在于索引范围和操作数量都可能很大，这排除了为每次更新显式重新计算数组的情况。 简单的模拟需要为每次更新将值写入潜在的大段中，这会在最坏的情况下（当许多更新与长范围重叠时）导致二次行为。 

这些约束意味着我们必须在大致对数时间或更好的时间内处理每个操作。 一旦操作数量达到大约 100,000 或更多，任何每次更新都会涉及每个元素的解决方案都将失败。 

一些边缘情况使幼稚的推理变得脆弱。 如果算术级数具有负公差，则该段上的值会减小，这可能会破坏假设单调性的实现。 如果`l == r`，更新退化为单点加法，并且假设级数中至少有两个元素的解决方案可能会错误地应用该公式。 另一个微妙的情况是重叠更新：应用于重叠段的两个算术级数不会合并到另一个简单级数中，因此任何为每个段存储单个“当前级数”的尝试都会失败。 

## 方法

 暴力解决方案通过迭代直接应用每个更新`l`到`r`并将相应的等差数列项添加到每个位置上。 每次更新费用`O(r - l + 1)`，因此在更新跨越整个数组的最坏情况下，单个操作是`O(n)`。 高达`m`操作，这变成了`O(nm)`，当两者都很大时，这太慢了。 

关键的观察是算术级数是指数的线性函数。 在位置`i`，附加值的形式为`a + (i - l) * d`，可以重写为`(a - l*d) + i*d`。 这将更新分为两个独立的部分：一个是整个范围内的恒定贡献，另一个是与指数成比例的贡献。 

这种分解使我们能够将问题转化为维护两个独立的范围相加结构：一个用于常数，一个用于系数`i`。 一旦我们能够有效地支持常量的范围加法和有效的线性系数的范围加法，每次更新都会变成两个标准范围更新的组合。 然后查询将两个结构的贡献结合起来。 

为了有效地支持这一点，我们使用差异数组样式的二叉索引树（Fenwick Tree），维护两个 BIT，以便我们可以在范围更新后重建前缀和。

 ## 算法演练

 ### 1.重写等差数列

 每次更新于`[l, r]`具有起始值`a`和差异`d`被重写为索引的函数`i`:`a + (i - l) * d = (a - l*d) + i*d`。 

这将更新分为常数部分和系数`i`，这是结构简化的关键。 

### 2. 维护两个独立的范围添加结构

 我们维护两棵芬威克树。 一个跟踪常量的范围添加，另一个跟踪应用于索引的系数的范围添加。 这种分离允许我们通过组合两种效果来重建任意位置的最终值。 

### 3. 将每次更新应用为两个范围操作

 对于每次更新`[l, r]`，我们执行：

 一个范围添加`(a - l*d)`到常数 BIT，并添加一个范围`d`为同一时间间隔内的系数 BIT。 

这样做的原因是，一旦将两个贡献结合起来，范围中的每个位置都会收到完全正确的线性变换。 

### 4. 通过重建值来回答查询

 获取位置处的值`i`，我们计算：`constant_sum(i) + i * coefficient_sum(i)`。 

这两个组件都是通过对其各自的 BIT 进行前缀查询来获取的。 

### 5.在线处理多项操作

 所有更新和查询均按顺序处理，确保每个查询都能看到所有先前更新的完整效果。 

### 为什么它有效

 核心不变量是，在任何时候，该结构都会将所有应用的算术级数的正确分解存储为两个加性字段：常数字段和线性索引字段。 每次更新都准确地保留了这种分解，因为每个算术级数都可以唯一地表示为常数项和与指数成比例的项之和。 由于芬威克树在范围更新和点查询下正确维护前缀聚合，因此每个索引的重建始终与所有更新的累积贡献相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 2)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_add(self, l, r, v):
        self.add(l, v)
        self.add(r + 1, -v)

n, q = map(int, input().split())

bit_const = Fenwick(n)
bit_coef = Fenwick(n)

for _ in range(q):
    tmp = input().split()
    if not tmp:
        continue
    t = int(tmp[0])

    if t == 1:
        l, r, a, d = map(int, tmp[1:])
        bit_const.range_add(l, r, a - l * d)
        bit_coef.range_add(l, r, d)

    else:
        i = int(tmp[1])
        res = bit_const.sum(i) + i * bit_coef.sum(i)
        print(res)
```Fenwick 结构通过差异数组用于范围添加、点查询模式。 每个算术级数更新被分为常数贡献和线性系数贡献。 恒定的BIT累加`(a - l*d)`超出范围，同时系数 BIT 累加`d`。 

对于索引处的查询`i`，两个 BIT 均在前缀处查询`i`。 常数贡献直接相加，而系数贡献乘以`i`，重建原始线性表达式。 

必须注意 1 索引，因为该公式直接取决于位置索引。 使用 0 索引而不进行调整会错误地移动所有算术级数值。

 ## 边缘情况

 一种边缘情况是单元素更新，其中`l == r`。 在这种情况下，算术级数会分解为单个值`a`。 这种转变仍然有效，因为`(a - l*d)`恰好变成`a`自从`d`当不存在第二项时，则无关紧要。 除了该单点之外，系数更新没有任何贡献。 

另一种情况是零差级数。 这里每个元素都接收相同的值。 分解将所有贡献分配给常数 BIT，而不将任何贡献分配给系数 BIT，这正确地模拟了均匀范围加法。 

最后一种情况是多次重叠更新。 每个更新独立地分解为线性分量，并且由于两个芬威克树都支持累加累积，因此重叠的片段自然地总结它们的效果而不会产生干扰。
