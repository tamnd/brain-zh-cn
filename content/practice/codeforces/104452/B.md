 ## 解决方案
 ## 问题理解

 我们得到了沿线的植物高度列表和一组查询。 每个查询给出一个切割高度$L$，并且我们必须计算在该切割之上还剩下多少材料。 

对于每个灌木丛的高度$a_i$，仅上面的部分$L$贡献，而且只有当它是积极的。 所以每个灌木丛都有贡献$\max(a_i - L, 0)$。 任务是对每个查询的所有灌木丛的贡献求和。 

因此每个查询都是独立的：我们想象在高度处进行水平切割$L$，丢弃其下方的所有内容，并对剩余的垂直长度求和。 

限制条件$N, K \le 10^5$迫使我们避免每个查询从头开始重新计算总和。 直接模拟将花费$O(NK)$，这大约是$10^{10}$在最坏的情况下，操作速度太慢。 这立即建议对高度进行预处理，以便可以在对数或恒定时间内回答每个查询。 

这里经常出现的一个天真的错误是仅对元素进行重新计算$a_i > L$无需预处理。 例如，如果高度是$[10^9, 10^9, \dots]$并且查询有许多小值，每个查询仍然扫描整个数组，从而导致超时，即使大多数值在查询中的行为相似。 

另一个微妙的陷阱是忘记贡献取决于有多少元素超过$L$以及超出了多少。 将其视为上面元素的计数$L$丢失震级信息并产生错误的答案。 

## 方法

 暴力方法独立评估每个查询。 对于给定的$L$，它遍历所有$a_i$, 累积$a_i - L$什么时候$a_i > L$，并输出总和。 这是正确的，因为它直接遵循定义，但它重复相同的扫描$K$次，导致$O(NK)$时间。 

关键的观察是对于固定的$L$，只有大于的元素$L$物质，并且它们的贡献可以以将计数与求和分开的方式重写。 如果我们对数组进行排序并维护前缀和，我们可以快速计算有多少元素超过$L$以及它们的总和。 这将每个查询减少为二分搜索加算术，从而实现高效评估。 

我们对表达式进行变换：$$\sum \max(a_i - L, 0)
= \sum_{a_i > L} a_i - L \cdot (\text{count of } a_i > L)$$一旦数组被排序，就可以通过单个二分搜索和前缀和查找来获得这两个项。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(NK)$|$O(1)$| 太慢了|
 | 排序+前缀和|$O(N \log N + K \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 1. 按非降序对高度数组进行排序。 这允许我们使用二分搜索来隔离大于给定查询值的所有元素。 
2. 构建一个前缀和数组，其中`pref[i]`存储第一个的总和`i`已排序数组中的元素。 这允许恒定时间范围求和查询。 
3.对于每个查询值$L$，使用二分查找查找第一个索引`pos`这样`a[pos] > L`。 一切都来自`pos`到$N-1$对答案有贡献。 
4. 计算所有大于的元素的总和$L$作为`total = pref[n] - pref[pos]`。 
5. 计算有多少元素超出$L$作为`cnt = n - pos`。 
6.减去损失的基础高度$L \cdot cnt$从`total`以获得最终答案。 
7. 独立输出每个查询的结果。 

关键思想是，一旦阈值分割数组，贡献就变成剩余元素数量的线性函数，因此所需的统计数据都直接来自预处理。 

### 为什么它有效

 排序可确保所有有效贡献者形成数组的连续后缀。 前缀和对累积总数进行编码，因此可以在恒定时间内计算任何后缀和。 由于转换将原始非线性最大表达式重写为该后缀上两个线性量的差，因此每个查询都减少为单个区间计算。 不会丢失任何信息，因为每个术语在前缀或排除的后缀中都只计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    k = int(input())
    
    a.sort()
    
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]
    
    from bisect import bisect_right
    
    for _ in range(k):
        L = int(input())
        pos = bisect_right(a, L)
        cnt = n - pos
        total = pref[n] - pref[pos]
        ans = total - L * cnt
        print(ans)

if __name__ == "__main__":
    solve()
```排序后，我们可以快速隔离出超出每个查询阈值的元素后缀。 前缀求和数组将该后缀转换为恒定时间求和查询。 二分搜索定位贡献元素和非贡献元素之间的边界。 然后使用直接从原始定义派生的固定算术表达式来评估每个查询。 

一个常见的实施错误是使用`bisect_left`而不是`bisect_right`，其中错误地包含等于的元素$L$。 另一个微妙的问题是忘记必须在计算完整后缀和之后应用减法，而不是每个元素，否则整数运算会退化为不必要的循环。

 ## 边缘情况

 关键的边缘情况是所有元素都小于或等于$L$。 在这种情况下，后缀为空，答案必须为零。 该算法处理这个问题是因为`pos = n`，所以后缀 sum 和 count 都为零，公式自然返回零。 

另一种情况是当$L = 0$。 那么每个元素都充分贡献，答案就成为数组的总和。 The binary search returns`pos = 0`，所以后缀是整个数组，减法什么也不做。 

当所有元素都相等时，每个查询要么保留所有内容，要么一步删除所有内容。 该公式正确地捕获了这种突然的转变，因为后缀和和计数一起缩放，从而在以下情况下保留了精确的取消：$L$超过该值。
