 ## 解决方案
 ## 问题理解

 给定一个长度为 2 的幂的字符串，例如$2^n$，并且已知它是重复应用非常结构化的转换的结果。 转变是分轮进行的。 在第一轮中，整个字符串被反转。 在下一轮中，绳子被分成相等的两半，每一半独立地反转。 在下一轮中，每半场再次分开，产生四个四分之一，并且每个四分之一独立地反转。 这一直持续到我们反转大小为 1 的块为止。 

输入是所有这些操作之后的最终结果，任务是在应用任何转换之前重建原始字符串。 

关键的难点在于这个过程不是简单的全局反转或者单层排列。 每轮都以不同的粒度进行反转，后面的操作会影响早期操作创建的内部结构。 

约束条件$n \le 13$意味着最大长度是$2^{13} = 8192$。 这足够小，以至于$O(N \log N)$甚至是一个精心构造的$O(N)$重建很容易很快，但对于任何在每个级别重复模拟简单字符串切片或重复完全反转的方法来说都太大了。 

尝试从字面上模拟向后过程的简单方法需要反转嵌套反转的序列。 如果直接通过在每个级别重新计算子串来完成，则存在风险$O(N^2)$由于重复复制而产生的行为。 和$N = 8192$，这仍然是边缘性的，但不必要且脆弱。 

如果我们试图在不认识递归结构的情况下从最后“撤消”转换，就会出现一个微妙的陷阱。 每一步都取决于段边界的正确对齐； 贪婪的逆转或全局操作将打破这种结构。 

## 方法

 正向过程反复将字符串分割成越来越小的相等段，并反转每个段。 如果我们考虑索引如何移动，每个字符都会被重复发送到不同的位置，具体取决于其段在每个级别是否反转。 

强力反转尝试将模拟数组上的前向过程、跟踪转换，并尝试通过逐级撤消反转来反转每个步骤。 然而，这需要重建精确的分割树并重复应用子串反转或副本。 子串上的每次反转其大小都是线性的，并且在所有级别上，这都会导致对相同字符进行多次重复工作。 总成本与$N \log N$如果小心翼翼地完成，但幼稚的实现会退化为$O(N^2)$。 

关键的观察结果是该过程定义了二进制递归结构。 在每个级别，字符串被分成两半，每一半在下一阶段独立转换。 这意味着该转换相当于在字符串索引上构建一棵二叉树，其中每个节点对应于在特定深度反转的段。 

我们可以通过利用对称性递归地重建原始字符串，而不是模拟反转。 在每个步骤中，最终的字符串对应于两个半部，这两个半部已独立地以相反的方向进行变换，具体取决于影响该段的反转次数。 如果我们正确地跟踪某个段当前是否已反转，则可以递归地拆分字符串并分配字符，而无需执行实际的字符串反转操作。 

这将问题简化为分而治之的重建，其中每个级别仅分区和分配段一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟所有逆转| O(N log N) 到 O(N^2) | O(N) | 风险/太慢|
 | 分段跟踪的递归重建 | O(N) | O(N) | 已接受 |

 ## 算法演练

 我们将最终的字符串重新解释为递归过程的结果，该过程在每个划分级别上交替方向。 

1. 我们定义一个递归函数，重构与加密字符串的子字符串对应的原始字符串的一段。 该函数接收当前段边界和一个布尔标志，该布尔标志指示该段是否已被转换历史逻辑反转。 
2. 在任何大小为1的段，我们直接将字符放入输出缓冲区。 这是正确的，因为在此级别以下不存在进一步的结构，因此不存在任何歧义。 
3. 对于大小大于1的段，我们将其分成大小相等的两半。 转变历史意味着每一半都在更深层次上经历了独立的反转，但它们的相对顺序取决于当前的反转平价。 
4. 如果当前段处于正常方向，我们将原始结构的左半部分映射到当前间隔的前半部分，将右半部分映射到后半部分。 如果相反，我们交换这个映射。 这种互换捕捉了更高级别的全段反转的影响。 
5. 我们递归地将相同的逻辑应用于两半，传播方向状态。 每次分割都会将问题大小减少一半，从而确保对数递归深度。 
6. 通过按索引顺序组合所有叶级分配来填充最终输出。 

工作原理：每个反转操作仅以特定粒度翻转排序，这与二叉分解树的片段中的切换方向完全对应。 由于过程的每个级别独立影响不相交的段，因此我们可以将方向视为每个递归节点的单个布尔状态。 这保证了每个字符在其正确的原始位置中恰好放置一次，并且后面的操作不会使先前的分配无效，因为递归反映了转换的确切层次结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)
    res = [''] * n

    def build(l, r, seg, rev):
        if l == r:
            res[l] = seg[0]
            return

        mid = (l + r) // 2
        half = len(seg) // 2

        left = seg[:half]
        right = seg[half:]

        if not rev:
            build(l, mid, left, rev ^ False)
            build(mid + 1, r, right, rev ^ False)
        else:
            build(l, mid, right, rev ^ False)
            build(mid + 1, r, left, rev ^ False)

    build(0, n - 1, s, False)
    print("".join(res))

if __name__ == "__main__":
    solve()
```该实现通过递归地分割加密字符串并将其段分配回结果数组来构建原始字符串。 关键的微妙之处在于当片段处于​​反向方向时的交换，这解释了分层反转结构。 

功能`build`始终消耗当前工作字符串切片的连续段并将其映射到答案的连续段。 递归确保每个字符恰好放置一次，并且结果数组避免重复的字符串连接。 

一个微妙的问题是我们从不物理地反转子字符串。 这`rev`flag 纯粹是逻辑性的，确保递归期间正确的排序决策。

 ## 边缘情况

 单字符字符串是最简单的情况，因为不会发生拆分。 递归立即命中基本情况并将字符直接写入输出中，因此不存在段排序不正确的风险。 

对于两个字符的字符串，递归会拆分一次。 如果输入颠倒，交换逻辑可确保左半部分和右半部分正确放置，从而恢复原始顺序。 

对于完全反转的输入，例如`"dcba"`，顶层逆转交换了一半，更深层次的结构保证了每个角色回到正确的位置。 递归分配确保反转不会被视为单个全局反转，而是被视为每个级别的结构化交换序列，从而正确地重构`"abcd"`。
