 ## 解决方案
 ## 问题理解

 我们得到一个整数列表，我们需要根据取决于列表中的最小值和最大值的规则来选择单个元素。 

我们首先计算一个参考值，即最小和最大元素的算术平均值，而不是对整个数组进行排序并采用经典的中位数。 然后我们在数组中寻找最接近该参考值的元素。 距离是用通常的绝对差值来测量的。 如果多个元素同样接近，我们选择其中最小值。 

因此，任务不是关于排序数组中的位置排序，而是关于接近极值的中点。 

输入大小最多可达 100,000 个元素，其值的大小约为 2·10^9。 这立即排除了重新计算每个候选者任何昂贵的东西或尝试所有对的解决方案。 任何二次方的操作，例如将每个元素与其他元素进行比较或在嵌套循环内排序，都会太慢。 经过简单预处理后的线性扫描是唯一可行的方向。 

一些边缘情况在这里很重要。 

如果所有元素都相等，则最小值和最大值重合，因此参考值正是该数字。 每个元素都同样接近，因此答案必须是相同的值。 任何错误处理浮点中点计算或关系的解决方案仍然可能会破坏这种情况。 

如果数组包含围绕中点对称放置的值，例如 [1, 3] 或 [1, 2, 3]，则多个元素可能会在距离上相连。 平局打破规则迫使我们选择这些候选者中最小的一个，因此我们必须明确跟踪平局，而不仅仅是保留任何最接近的值。 

最后，由于值可能很大，如果不小心，将 mid = (min + max) / 2 作为浮点数计算可能会引入精度问题。 但是，如果我们以代数方式比较距离，则根本不需要浮点数。 

## 方法

 直接解释建议计算数组的最小值和最大值，然后检查每个元素以查看其与中点的接近程度。 蛮力的想法很简单：计算最小值和最大值，计算中点，然后迭代所有元素并跟踪与该中点绝对差最小的元素。 

这已经给出了 O(n) 解决方案，因此不需要排序或嵌套比较。 唯一的微妙之处在于我们如何在不引入浮点错误的情况下比较距离以及我们如何一致地处理平局。 

关键的观察结果是，一旦 min 和 max 已知，中点就固定了，因此问题简化为单遍选择问题：选择最小化 |a[i] - (min + max)/2| 的元素。 由于与常量阈值的比较可以在不进行除法的情况下重写，因此我们可以通过将 2 * a[i] 与 (min + max) 进行比较来完全避免浮动。 这将所有内容保留为整数并保留精确的顺序。 

暴力心理模型之所以有效，是因为我们只需要全局极值和一次遍历数组。 仅当使用朴素浮点中点计算实现或未显式处理平局打破时，它才会失败。

 ## 算法演练

1. 读取所有数字并计算数组中的最小值和最大值。 这将建立定义目标中点的参考区间端点。 
2. 计算这些端点的总和，即中点的两倍。 我们避免除法，因为我们只需要比较，而不是实际的中点值。 
3. 初始化一个变量来存储最佳候选者，另一个变量来存储迄今为止看到的最佳距离。 The best distance starts as infinity.
 4. Iterate through every element in the array. 对于每个元素，以缩放整数形式计算其距中点的绝对距离，比较 |2 * a[i] - (min + max)|。 这测量的顺序与到真实中点的距离相同。 
5. 如果该距离小于目前为止的最佳距离，则更新最佳距离和最佳候选值。 
6. 如果距离等于最佳距离，则仅当当前值更小时才更新候选值。 This enforces the tie-breaking rule.
 7.处理完所有元素后，输出存储的候选。 

### 为什么它有效

 从全局最小值和最大值导出的中点是独立于数组结构的固定常数。 每个元素纯粹根据其与该常数的距离来评估，因此问题简化为在确定性函数上选择全局 argmin。 转换为双倍值可以保留顺序，因为乘以 2 是单调的并且不会影响比较。 在扫描过程中明确执行平局打破规则，确保在等距离候选者中进行确定性选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    mn = min(a)
    mx = max(a)
    target_sum = mn + mx

    best_val = None
    best_dist = None

    for x in a:
        dist = abs(2 * x - target_sum)
        if best_dist is None or dist < best_dist or (dist == best_dist and x < best_val):
            best_dist = dist
            best_val = x

    print(best_val)

if __name__ == "__main__":
    solve()
```该解决方案首先提取全局极值，该极值隐式定义参考中点。 它不是直接计算中点，而是使用加倍表达式来保持所有算术的完整性和精确性。 该循环维护运行中的最佳候选者，仅当找到严格更好的距离或当平局被较小的值打破时才更新它。 

一个常见的实施错误是计算`(mn + mx) / 2`使用浮点运算，然后使用浮点数比较距离。 当值很大时，这可能会引入舍入误差。 另一个微妙的错误是未能严格执行平局打破规则，这会导致当多个值等距时输出不正确。

 ## 边缘情况

 当所有值都相同时，最小值等于最大值，并且目标中点会折叠到相同的值。 算法集`target_sum = 2 * x`，因此每个元素产生的距离为零。 由于平局打破规则更喜欢较小的值并且所有值都相等，因此第一个遇到的元素仍然有效，并且最终答案是重复的值。 

当有两个元素到中点的距离相等时，例如`[1, 3]`，两者都产生距中点 2 的距离 2。算法先比较 1，然后比较 3。处理 1 后，best_val 变为 1。处理 3 时，距离相等，但 3 不小于 1，因此被忽略。 输出仍为 1，符合所需规则。
