 ## 解决方案
 ## 问题理解

 我们得到一条水平直线道路，建模为 x 轴。 蒂莫菲在这条线上的两个不同位置放置了金属探测器。 首先，他站在原点并将探测器范围设置为值 r₁，然后移动到位置 x = a 并将范围设置为 r2。 每次，探测器都会亮起，这意味着隐藏的金币恰好位于距探测器的欧几里得距离处。 

从几何角度来说，这意味着硬币位于以 (0, 0) 为中心、半径为 r1 的圆上，也位于以 (a, 0) 为中心、半径为 r2 的圆上。 任务是恢复这两个圆的交点的整数坐标，附加条件是 y 坐标必须非负。 

约束允许 a、r₁、r2 最大为 10⁹，这会立即排除任何密集搜索或坐标枚举。 任何解决方案都必须是每个测试用例的恒定时间，并且完全依赖于圆几何的代数推导。 

一个微妙的边缘情况是当圆相切或几乎相切时。 使用浮点平方根进行简单的几何计算可能会引入精度误差并产生非整数结果或 y 的轻微负值。 由于该问题保证整数答案，因此必须避免任何舍入方法以支持精确算术。 

另一种边缘情况是当交点恰好位于 x 轴上时，即 y = 0。在这种情况下，两个圆在一个点处接触，并且粗心的平方根处理可能会因浮动误差而产生较小的负值。 

## 方法

 强力解释会尝试搜索整数点 (x, y)，使其到 (0, 0) 和 (a, 0) 的距离与 r₁ 和 r2 匹配。 这意味着检查大范围内的所有整数 x 并从每个约束导出 y，验证两个方程是否成立。 即使我们将 x 限制为 ±10⁹，这也会导致候选数量不可行，大约为 10⁹ 检查。 

关键观察点是该点位于两个圆的交点处，且圆心位于 x 轴上。 这种对称性迫使解由一个线性关系唯一确定。 

将两个圆方程相减，消除 y 和 x² 中的二次项，留下 x 中的线性方程。 这直接确定了交点的 x 坐标。 一旦 x 已知，代回任一圆方程即可得到 y²，并且由于硬币位于轴上方或轴上，因此我们取非负平方根。 

这将整个问题简化为常数时间代数。

 ## 算法演练

 我们依赖于两个圆形成的方程组。 

1. 从两个距离约束开始：未知点 (x, y) 满足 x2 + y2 = r22 和 (x − a)2 + y2 = r22。 它们对两个探测器读数进行编码。 
2. 将第二个方程展开为 x2 − 2ax + a2 + y2 = r22。 此步骤是必要的，因为它暴露了与第一个方程相比将抵消的项。 
3. 用第二个方程减去第一个方程。 x² 和 y² 项完全抵消，留下 −2ax + a² = r²² − r₁²。 这是几何体折叠成线性约束的关键简化。 
4. 求解 x，得到 x = (a² + r₁² − r²²) / (2a)。 这唯一地确定了水平坐标，因为两个圆最多相交于两个对称点，并且 y ≥ 0 条件确定了正确的点。 
5. 将 x 代入 x² + y² = r₁² 以计算 y² = r₁² − x²。 这将恢复距 x 轴的垂直距离。 
6. 取 y = sqrt(y²)。 由于该问题保证整数坐标，因此 y² 是完全平方数，y 是整数。 
7. 输出（x，y）。 通过构造，y 值是非负的。

### 为什么它有效

 正确性来自于通过减去两个圆方程来完全消除 y。 这将非线性几何问题简化为 x 的线性约束，该约束对于圆的任何交点都必须成立。 一旦 x 被固定，剩下的方程就唯一地定义 y²，并且条件 y ≥ 0 消除了 x 轴上方和下方的两个对称交点之间的模糊性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def solve():
    a = int(input().strip())
    r1 = int(input().strip())
    r2 = int(input().strip())

    # x-coordinate from linear equation
    x = (a*a + r1*r1 - r2*r2) // (2*a)

    # y^2 from circle equation
    y2 = r1*r1 - x*x
    y = int(math.isqrt(y2))

    print(x)
    print(y)

if __name__ == "__main__":
    solve()
```该实现直接反映了代数推导。 除法是精确的，因为该问题保证整数坐标，因此 (a² + r₁² − r²²) 可以被 2a 整除。 使用整数运算完全避免了精度问题。 

平方根的计算方法为`math.isqrt`，它返回精确的整数平方根，没有浮点错误。 这很重要，因为浮点 sqrt 对于较大的值可能会产生轻微的误差。

 ## 边缘情况

 一种边缘情况是圆在 x 轴上精确相交。 在这种情况下 y 变为零，并且计算出的 y² 也恰好为零。 该算法自然地处理这个问题，因为`isqrt(0)`返回 0，没有任何特殊大小写。 

另一种情况是当交点非常接近切线时。 代数上 y² 变成一个完美的平方，但任何浮点实现都会引入小的负值，如 −1e−9。 使用整数算术可以完全避免这种情况，因为所有中间值都保持精确。 

最后一种情况是 x 计算产生等于 0 或 a 的边界值。 这对应于检测器位置之一处的交叉点。 该公式仍然直接适用，因为它来自圆方程的精确消去，并且在问题保证整数输出的情况下除法仍然有效。
