 ## 解决方案
 ## 问题理解

 我们被要求统计 1 到 n 范围内同时满足两个条件的整数。 首先，数字必须是偶数。 其次，当我们以二进制形式写入该数字时，设置的位数（即二进制表示形式中的位数）也必须是偶数。 

因此，我们通过组合奇偶校验约束来过滤最多 n 的自然数前缀：值本身的奇偶校验和其 popcount 的奇偶校验必须都为零。 

约束 n 最大为 10^9，这个值足够小，二进制表示最多有 30 位。 这立即表明我们可以在固定数量的位置上使用按位数字 DP。 对于 n 约为 10^5 的情况，暴力扫描最多 n 也是可行的，但在 10^9 时则完全不可能。 

一种简单的方法是迭代从 1 到 n 的所有数字并计算每个数字的 popcount。 这是正确的，但在最坏的情况下太慢，因为它执行 O(n log n) 位操作，在上限处达到大约 10^9 次操作。 这超出了时间限制。 

一个微妙的边缘情况来自奇偶校验交互。 例如，像 1、3、5 这样的数字是奇数，因此无论 popcount 如何，它们都会被排除。 但像 2 (10 2) 或 6 (110 2) 这样的数字取决于这两个约束，缺少一个约束会导致计数过多。 

另一个微妙之处是 popcount 奇偶校验在数值上不是单调的。 由于进位，相邻数字可以翻转最后一位和多个较高位，因此任何基于前缀的推理都必须显式编码状态。 

## 方法

 蛮力策略很简单：迭代从 1 到 n 的每个整数 x，检查 x 是否为偶数，然后计算其二进制表示形式中 1 的数量并测试该计数是否为偶数。 这是正确的，因为它直接应用了定义。 

成本来自于重复计算每个数字的 popcount。 每个 popcount 的时间复杂度为 O(log n)，并且有 n 个数字，因此 O(n log n)。 对于 n = 10^9，这远远超出了可行的限制。 

关键的观察是，我们对数字的确切值并不感兴趣，只对它们的二进制结构和奇偶校验属性感兴趣。 这建议将数字视为位串，并在跟踪约束的同时逐位构建它们。 由于 n 最多有 30 位，因此我们可以使用标准数字 DP 代替二进制表示。 

我们从最高有效位到最低有效位处理位，维护三个状态：我们是否仍然严格遵守 n 的前缀，当前构造的数字是否已经是偶数或奇数（或者更直接地，我们放置的最后一位确定奇偶校验），以及到目前为止看到的个数的奇偶校验。 

数字本身的奇偶校验仅取决于最后一位，因此我们不需要值奇偶校验的完整状态，只需确保最后最低有效位为零。 在 DP 中，通过仅考虑有效的完成来自然地强制执行这一点。 

该问题简化为计算最后一位为 0 的二进制数 ≤ n 甚至 popcount。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n log n) | O(n log n) | O(1) | O(1) | 太慢了|
 | 数字DP | O(log n) | O(log n) | 已接受 |

 ## 算法演练

 我们使用数字 DP 代替 n 的二进制表示来解决这个问题。

1. 将 n 转换为其二进制表示形式，即从最高有效位到最低有效位的位列表。 这给出了一个固定长度的表示，我们将通过前缀来匹配前缀。 
2. 定义一个 DP 函数 dp(pos,ight,ones_parity,last_bit)，其中 pos 是当前位索引，tight 指示到目前为止的前缀是否完全等于 n 的前缀，ones_parity 跟踪到目前为止使用的 1 的数量是偶数还是奇数，last_bit 跟踪最后选择的位，以便我们可以在最后强制执行偶数。 
3. 在每个位置，尝试放置 0 或 1。如果ight 为 true，则我们不能超过 n 中的相应位。 否则，我们可以自由选择0或1。 
4. 当放置一个位时，如果该位为 1，则通过切换来更新 Ones_parity。同时将 last_bit 更新为所选值。 
5. 当我们到达最终位置时，只有当last_bit为0（确保偶数）并且ones_parity为偶数时，我们才接受该数字。 
6. 从最高有效位开始对所有有效完成求和。 

工作原理：在每个前缀处，DP 准确存储决定未来有效性所需的奇偶校验相关信息。 共享相同 pos、紧密状态、奇偶校验和最后一位的两个前缀可以互换，因为其余决策仅取决于这些值，而不取决于到目前为止构建的确切数值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input().strip())
    bits = list(map(int, bin(n)[2:]))
    L = len(bits)

    from functools import lru_cache

    @lru_cache(None)
    def dp(pos, tight, ones_parity, last_bit, started):
        if pos == L:
            if not started:
                return 0
            return int(last_bit == 0 and ones_parity == 0)

        limit = bits[pos] if tight else 1
        res = 0

        for b in range(limit + 1):
            ntight = tight and (b == limit)
            nstarted = started or (b == 1)

            if not nstarted:
                # still leading zeros
                res += dp(pos + 1, ntight, 0, 0, nstarted)
            else:
                npar = ones_parity ^ (b == 1)
                res += dp(pos + 1, ntight, npar, b, nstarted)

        return res

    print(dp(0, 1, 0, 0, 0))

if __name__ == "__main__":
    solve()
```该解决方案使用位位置的记忆递归。 这`tight`flag 强制执行上限 n。 这`ones_parity`变量使用 XOR 累加来跟踪设置位的奇偶校验。 这`started`标志处理前导零，以便我们不会错误地将未使用的前缀位视为对弹出计数或奇偶校验约束有贡献。 

这`last_bit`跟踪以强制执行数字必须为偶数的要求，这相当于确保任何有效非零数字中最低有效构造位为零。

 ## 边缘情况

 一种边缘情况是 n = 1。唯一的数字是 1，它是奇数，因此不能计算在内。 DP 会处理此问题，因为任何以 last_bit = 1 结尾的构造数字在最终状态都会被拒绝。 

另一个边缘情况是像 2 这样的小偶数。二进制为 102，popcount 为 1，这是奇数，因此被排除在外。 DP 正确地传播 ones_parity 并在叶子处拒绝它。 

进一步的情况是二的幂。 这些总是有一个设置位，因此 popcount 奇偶校验总是奇数，这意味着除了 0 之外没有 2 的幂符合条件。 DP 自然地对此进行编码，因为对于任何单热位模式，ones_parity 都会变为 1，并且在没有附加位的情况下永远不会返回到 0，这将超出紧密模式中的界限。
