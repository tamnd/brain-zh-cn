 ## 解决方案
 ## 问题理解

 我们得到一棵最多有一百万个节点的树，每对节点都隐式地生成一个旅行请求：旅行者从编号较小的节点出发，沿着树中唯一的简单路径前往编号较大的节点。 每条这样的路径都会为其使用的每条边贡献一次遍历计数。 

任务是确定在所有边上，这些有向对使用最多的边以及有多少边达到最大使用计数。 

输入是一个树结构。 由于一棵树在任意两个节点之间只有一条简单路径，因此每对节点都确定性地对该路径上的每条边的负载做出贡献。 输出是两个数字：所有边缘上的最大负载，以及有多少边缘达到该最大值。 

最多一百万个节点的约束排除了任何枚举节点对或单独模拟路径的方法。 即使 O(N^2) 推理也是完全不可能的，因为这意味着大约 10^12 对。 即使 O(N log N) 仅在每个节点的工作量极小时（通常是单个 DFS 或线性聚合）才可行。 

有向条件 u < v 产生了一个微妙的问题。简单的无向对计数方法会忽略排序，但这里方向仅与如何选择对有关，而不与如何遍历路径有关。 每个无序对 {u, v} 仍然只贡献一次，因为只有一个方向满足 u < v。 

一个常见的错误是假设对称性并尝试计算子树大小而不考虑每条边接收的全局贡献。 另一个微妙的陷阱是尝试根树并使用朴素子树大小来计算贡献，而不仔细地将其连接到有多少对穿过边缘。 

## 方法

 强力解释将考虑每对节点 u 和 v，其中 u < v，使用 BFS 或 LCA 计算它们之间的路径，并沿该路径的边缘递增计数器。 这在概念上是正确的，因为它直接遵循定义。 然而，有 θ(N^2) 个这样的对，并且在最坏的情况下每条路径可以采用 O(N)，从而导致退化链树中的 O(N^3) 行为。 即使进行了 LCA 的预处理，每对计算 LCA 的成本仍然是 O(1)，但仍然需要推理对边的贡献，而无需额外的结构，每个边不能直接累加。 

关键的观察是我们不需要单独的路径。 一旦删除，每条边都会将树分成两个部分。 端点位于不同组件中的任何一对节点都必须穿过该边。 因此，一条边的使用次数仅取决于该边两侧的大小。 这将问题从路径枚举转变为子树大小计算。 

一旦我们任意地给树建立根，每条边都会将一个节点连接到其父节点。 删除这样的边会将树分成大小为 s 的子树和大小为 N − s 的剩余部分。 穿过该边的无序对的数量为 s × (N − s)。 由于每一对恰好对应于其路径上的边集的一次遍历，因此该公式给出了精确的边使用计数。 

我们使用 DFS 计算子树大小，评估每条边的该值，并跟踪最大值以及有多少条边达到该值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力配对模拟 | O(N^3) 最坏情况 | O(N) | 太慢了 |
 | 子树大小计数 | O(N) | O(N) | 已接受 |

 ## 算法演练

1. 以节点 1 为树根，构建邻接表进行遍历。 这给出了在不改变树结构的情况下定义父子关系的方向。 
2. 从根运行 DFS 来计算每个子树的大小。 对于节点 u，其子树大小为 1 加上所有子节点的子树大小之和。 
3. 在 DFS 期间，对于节点 u 及其子节点 v 之间的每条边，将该边的贡献计算为 sz[v] × (N − sz[v])。 这表示必须通过该边的所有对，因为一个端点位于子树内部，另一个端点位于子树外部。 
4. 在处理边缘时维护两个变量：到目前为止看到的最大边缘负载以及有多少边缘达到该最大值。 在计算每条边的贡献后更新它们。 
5. DFS完成后，输出最大值及其频率。 

关键点是，当我们知道其下方子树的大小时，每条边都会被计算一次，从而确保线性工作。 

### 为什么它有效

 每对节点都恰好形成一条路径，并且该路径恰好穿过将两个端点分成不同组件的边。 对于任何固定边，其删除将树划分为两个不相交的节点集。 当且仅当一对端点位于不同的集合中时，该对才对该边做出贡献。 这些对的数量恰好是两个分量大小的乘积，因此每条边的贡献完全由子树大小决定，与全局结构无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    N = int(input())
    g = [[] for _ in range(N + 1)]

    for _ in range(N - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    parent = [0] * (N + 1)
    sz = [0] * (N + 1)

    max_val = 0
    count = 0

    def dfs(u):
        nonlocal max_val, count
        sz[u] = 1
        for v in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            dfs(v)
            sz[u] += sz[v]

            contrib = sz[v] * (N - sz[v])
            if contrib > max_val:
                max_val = contrib
                count = 1
            elif contrib == max_val:
                count += 1

    dfs(1)

    print(max_val, count)

if __name__ == "__main__":
    solve()
```该解决方案使用单个 DFS 遍历。 父数组可防止重新访问无向邻接列表中的父数组。 子树大小的累加是自下而上进行的。 每次我们完成对子项的探索时，我们都会立即知道相应子项侧组件的大小，并可以计算边缘贡献。 

一个微妙的实现细节是递归深度。 当N达到10^6时，Python默认的递归限制是不够的，因此必须大幅增加。 另一个重要的细节是，我们在 DFS 期间计算边缘贡献，而不是存储子树大小并稍后迭代，这避免了额外的内存开销并保持一切 O(N)。

 ## 边缘情况

 在二节点树中，DFS 立即将一条边贡献计算为 1 × 1 = 1，并且它变得最大且唯一，从而确认对最小树的正确处理。 

在星形树中，每个叶边的子树大小为 1，给出相同的贡献 1 × (N − 1)。 该算法在 DFS 期间访问每个叶边缘一次，并一致地递增等频计数器，确保正确计算连接数。 

在长链中，子树大小沿着遍历单调减小。 从递归返回时，每条边的贡献仅计算一次，并且中心边自然会产生最高的乘积，从而验证基于 DFS 的分解正确捕获全局对分布，而无需显式枚举对。
