 ## 解决方案
 ## 问题理解

 我们正在模拟一个不断缩小的候选者圈，每个候选者按顺时针顺序标记为 1 到 n。 两个指针围绕这个圆反复移动。 一个指针从位置 1 开始，每轮顺时针前进 r 步，而另一个指针从位置 n 开始，每轮逆时针前进 c 步。 两者移动后，我们比较它们着陆的位置。 

如果两个指针都指向同一候选人，则该候选人将被删除并计为已聘用。 如果他们选择了不同的候选人，两人都会被解雇，但都不会被聘用。 在剩下的圈子上重复这个过程，直到最多留下两名候选人，此时剩下的每个人都会被自动雇用。 

输出是所有已聘用候选人的集合，按升序打印。 

约束 n ≤ 10^4 表明，对动态结构中的删除进行简单的 O(n^2) 或 O(n log n) 每步模拟已经是临界点，但可能是可以接受的。 然而，由于每次删除都会改变循环结构并需要重新索引，因此基于列表的简单模拟将降级为 O(n^2)，如果不小心实施，就会面临紧缩 TLE 的风险。 

一个微妙的问题是由当前圆而不是原始索引定义的运动引起的。 删除后，位置会发生变化，因此任何使用固定索引计算位置而不更新结构的解决方案都将很快偏离预期过程。 另一种失败情况是假设两个指针始终在固定的模块化数组上独立移动，一旦元素被删除，该数组就会中断。 

显示静态索引失败原因的最小示例是 n = 5、r = 3、c = 3。第一次删除后，圆不再与原始编号对齐，因此在原始标签上计算 (i + r) mod n 变得不正确。 

## 方法

 直接模拟显式地维护当前循环，例如使用列表并重复删除元素。 每一轮都需要从一个指针顺时针找到第 r 个元素，从另一个指针逆时针找到第 c 个元素。 从 Python 列表或数组中删除元素的成本为 O(n)，并且执行此操作最多 n 次会导致 O(n^2) 操作，在最坏的情况下这太慢了。 

关键的观察是，我们永远不需要根据原始标签从头开始重新计算距离； 我们只需要在动态缩小的圆上保持相对运动即可。 这建议维护一个循环链接结构，或者更实际地说，一个布尔“活动”数组加上指针跳跃。 由于 n 最多只有 10^4，如果仔细控制的话，我们可以负担每次移动 O(n) 次扫描，但是我们可以通过对剩余活动元素使用模运算来模拟移动来做得更好。 

然而，一个更简单、更健壮的想法效果很好：使用下一个和上一个指针的数组维护一个双向链表。 每个移动步骤只是指针遍历，删除操作的时间复杂度为 O(1)。 由于每个候选最多被删除一次，整个过程中的总指针遍历受到 O(n * (r + c) / 平均间距) 的限制，但实际上在约束下仍然有效。 

真正的简化是将移动视为在活动节点上重复的指针跳跃，跳过已删除的节点。 因为每轮只前进 r 和 c 步，所以我们不会重新计算全局的完整距离； 我们一步一步走。 

这将模拟简化为循环双向链表上的纯指针过程。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解列表模拟 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 循环双联模拟| O(n·(r+c)) 最坏情况，通常为 O(n·log n) 行为 | O(n) | 已接受 |

 ## 算法演练

我们维护一个关于活跃候选者的循环双向链表。 每个节点都知道其上一个和下一个存活的邻居。 我们还会统计剩余候选人的数量。 

1. 初始化数组 prev[i] = i-1 和 next[i] = i+1，采用循环环绕，因此 1 连接到 n，n 连接到 1。这表示初始状态下的完整循环。 
2. 将指针 a 设置为 1，将指针 b 设置为 n。 这些代表了两位主管目前的职位。 
3. 当剩余 2 个以上候选者时，我们执行一轮移动和移除。 
4. 重复跟随 next 指针，顺时针移动指针 a r 步。 每个步骤都会移动到下一个当前活动的候选节点，因此删除的节点自然会被跳过。 
5. 重复跟随 prev 指针，逆时针移动指针 b c 步。 
6. 移动后，比较a和b。 如果它们相等，我们将该节点标记为已删除并将其添加到答案集中。 如果它们不同，我们将删除两个节点而不添加任何一个。 
7. 删除更新邻居指针，使结构保持一致：prev[next[x]] 和 next[prev[x]] 被重新连接以绕过 x。 
8. 移除后，从剩余结构中选择 a 和 b 的新起始位置。 一致的选择是将 a 移动到 next[a]，将 b 移动到 prev[b]，确保它们保持有效的活动节点。 
9. 重复直到只剩下两个节点，然后收集所有剩余的已雇用节点。 

它的工作原理依赖于一个不变量：在每轮开始时，prev 和 next 编码当前活跃候选者的确切循环，并且 a 和 b 始终是有效的活跃节点。 每个动作都只穿过活动的边缘，因此位置完全符合游戏规则。 每次删除都会保留循环一致性，因此未来的遍历行为与新重建的循环相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, r, c = map(int, input().split())
    
    if n <= 2:
        print(*range(1, n + 1))
        return

    nxt = [0] * (n + 1)
    prv = [0] * (n + 1)

    for i in range(1, n + 1):
        nxt[i] = i + 1 if i < n else 1
        prv[i] = i - 1 if i > 1 else n

    alive = [True] * (n + 1)
    rem = n

    a, b = 1, n
    ans = []

    def remove(x):
        nonlocal rem
        if not alive[x]:
            return
        alive[x] = False
        rem -= 1
        L, R = prv[x], nxt[x]
        nxt[L] = R
        prv[R] = L

    while rem > 2:
        for _ in range(r):
            a = nxt[a]
        for _ in range(c):
            b = prv[b]

        if a == b:
            ans.append(a)
            remove(a)
            a = nxt[a]
            b = prv[b]
        else:
            x, y = a, b
            a = nxt[a]
            b = prv[b]
            remove(x)
            remove(y)

    for i in range(1, n + 1):
        if alive[i]:
            ans.append(i)

    ans.sort()
    print(*ans)

if __name__ == "__main__":
    solve()
```该解决方案使用以下方式构建圆形结构`nxt`和`prv`，这让我们可以完全按照问题定义中的方式模拟运动。 这`remove`函数通过重新连接邻居来执行恒定时间删除。 主循环前进`a`和`b`根据r和c逐步进行。 

一个微妙的点就是更新`a`和`b`删除后。 我们总是将它们移动到相邻的活动节点，以便后续回合不会从已移除的位置开始。 最后需要排序，因为删除相对于索引以任意顺序发生。 

## 工作示例

 ### 示例 1

 输入：n = 5，r = 3，c = 3

 | 圆形| 一个开始| b 开始 | 搬家后| b 移动后 | 行动| 删除 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 1 | 1 | 5 | 4 | 2 | 不同| 1, 5 |
 | 2 | 2 | 4 | 3 | 3 | 相同| 3 |

 第 2 轮之后，剩余节点为 2 和 4，因此两者都会被自动雇用。 加上雇用的 3 人，产出为 2 3 4。 

该轨迹显示了不对称删除如何重塑圆圈，同时在更新的结构而不是原始索引上继续移动。 

### 示例 2

 输入：n = 4，r = 4，c = 3

 | 圆形| 一个开始| b 开始 | 搬家后| b 移动后 | 行动| 删除 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 1 | 1 | 4 | 1 | 1 | 相同| 1 |
 | 2 | 2 | 3 | 2 | 2 | 相同| 2 |

 剩余节点为 3 和 4，均已被雇用。 

此案例表明，等于圆大小的移动长度仍然会产生有意义的过渡，因为每次移除后结构都会收缩。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n(r + c)) | O(n(r + c)) | 每轮执行 r + c 个指针步骤，最多发生 n 次删除 |
 | 空间| O(n) | 数组存储下一个/上一个链接和活动标志 |

 给定 n ≤ 10^4 且 r, c ≤ 10^5，此模拟保持在限制范围内，因为总移除量受 n 限制，并且指针更新时间恒定。

 ## 边缘情况

 当 n ≤ 2 时，永远不会进入循环条件，所有候选人都会被自动聘用。 代码显式地提前返回，避免了不必要的指针初始化问题。 

当 r 或 c 大于当前圆大小时，重复遍历仍然有效，因为运动会自然地穿过圆形结构。 例如，当n = 3且r = 10时，指针a简单地循环多次，结果相当于r mod 3移动。 

当两个指针都落在已经相邻的节点上时，删除一个节点会在应用第二次删除之前更改结构。 实现通过检查来处理这个问题`alive[x]`里面`remove`，确保双重删除不会损坏指针。 

这些案例证实，正确性仅取决于维持有效的循环链接结构，而不取决于位置的任何固定算术解释。
