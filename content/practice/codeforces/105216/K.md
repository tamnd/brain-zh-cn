 ## 解决方案
 ## 问题理解

 我们正在模拟计算机每周到达一台的过程。 每台计算机都被分配一个随机“名称”，它只是从范围中统一选择的整数$1$到$N$。 何塞最多只能保留$k$随时都有电脑。 当一台新计算机到达并且房子已满时，最旧的计算机首先被移除，因此该结构的行为就像一个大小为 的 FIFO 队列$k$。 

这个过程无限期地持续下去，我们被要求计算直到房子里的两台计算机第一次共享相同名称的预期周数。 该事件被视为失败，当它发生时我们立即停止计数。 

所以状态最多以最后一个滑动窗口的形式演化$k$生成的随机值。 我们正在等待该窗口内的第一个重复项。 

这些约束立即排除了任何模拟。 的价值$N$可以大到$10^{12}$，所以我们无法跟踪数组中的频率。 的价值$k$可以达到$10^6$，对于二次碰撞跟踪或每步重新计算来说太大。 答案是对随机过程的期望，因此我们需要一个封闭形式的表达式或快速组合推导。 

一个微妙的边缘情况是$k$超过$N$。 在这种情况下，一旦我们有超过$N$窗口中的项目，但由于窗口大小上限为$k$，该过程实际上表现不同。 如果$k > N$，事件最终得到保证，但预期时间取决于滑动窗口填充和强制重叠的速度。 天真的“k 项生日悖论”方法是错误的，因为窗口不是静态的。 

另一个失败案例来自于将每周视为一个独立的“选择 k 项并检查重复项”实验。 这忽略了旧元素被删除，因此如果将它们推出窗口，早期的碰撞可能会消失。 

## 方法

 暴力模拟很简单。 我们一一生成随机数，维护一个最多size的队列$k$，以及当前队列中值的频率图。 每一步我们都会插入新值，如果需要的话逐出最旧的值，并检查插入的值是否已存在于映射中。 如果是这样，我们就停下来。 

这是正确的，但它本质上是马尔可夫过程中停止时间的蒙特卡罗模拟。 即使每一步都是$O(1)$，预期停止时间大约是滚动窗口的碰撞时间，对于大的情况来说可能非常大$N$。 更重要的是，我们被要求提供准确的期望，而不是经验估计。 

关键的观察结果是该过程仅取决于当前窗口中的活动元素集。 当引入新元素时，创建“悲伤计算机”的唯一方法是该值是否已存在于当前元素中$k$活跃的名字。 所以问题归结为分析概率$i$- 第一次抽奖是之前任何一次抽奖的重复$\min(k, i-1)$积极的价值观。 

我们转变视角：我们不是模拟时间，而是计算过程中第一次碰撞的预期等待时间，其中每个步骤都会引入一个新的随机值，并在窗口满后删除一个旧值。 关键的简化在于，从固定传入值的角度来看，活动值集的行为类似于大小一致的随机集，最大可达$k$，因为窗口中的每个位置对应于一个独立的随机抽取。 

因此，在窗口填满后的任何时间，活动状态本质上是大小的多重集$k$取自$N$，如果新的绘制与其中任何一个匹配，则它会发生碰撞$k$价值观。 如果我们忽略删除引入的相关性，则下一次抽奖安全的概率大约为$1 - \frac{k}{N}$，只要所有活动值都是不同的。 当碰撞发生时，该过程就会停止。 

这将问题变成了一个经典的预期等待时间问题：我们重复执行试验，每次试验成功（无冲突）的概率与窗口中当前有多少个不同值成正比。 然而，由于停止事件发生在第一次重复时，我们可以跟踪预期时间，直到流中的第一次重复绘制，其中只有最后一个$k$元素很重要。 

导出它的一种更简洁的方法是考虑该过程至少存活的概率$t$步骤。 这需要所有的第一$t$每个大小的滑动窗口内的值都是不同的$k$。 为了$t \le k$，这就是全部$t$值不同，给出标准下降因子概率。 为了$t > k$, 每个窗口大小$k$必须是不同的，这会强制形成相当于 de Bruijn 式回避条件的结构化组合约束。 我们不是直接枚举有效序列，而是使用生存概率的线性来计算期望值，这会根据每一步剩余的新鲜选择数量来产生伸缩产品。 

关键的简化是，该过程的行为类似于具有“记忆”的广义生日问题$k$”，以及步骤中的生存概率$i$仅取决于窗口中剩余多少个不同的值，这始终是精确的$\min(i-1, k)$生存之下。 这使我们能够计算：$$P(\text{no collision at step } i) = \frac{N - \min(i-1, k)}{N}$$预期停止时间成为生存概率的总和，该总和可分解为有限总和，最多可达$k+1$，后跟具有恒定碰撞概率的几何尾部$\frac{k}{N}$。 

这会产生一个可以计算的表达式$O(k)$时间采用模运算，具有几何尾封闭形式。

 | --- | --- | --- | --- |
 | 暴力模拟|$O(\mathbb{E}[T])$|$O(k)$| 太慢了 |
 | 最优组合期望|$O(k)$|$O(1)$| 已接受 |

 ## 算法演练

 我们计算第一个重复项出现在大小为 的滑动窗口中之前的预期时间$k$，在均匀随机抽取下$1$到$N$。 

1.我们对每个前缀长度的生存概率进行建模$i$，假设尚未发生重复。 对于第一个$k$步骤，所有先前看到的值都是不同的，所以在步骤$i$，正好有$i-1$禁止值。 这给出了生存概率因子$\frac{N-(i-1)}{N}$。 
2. 我们将这个推理扩展到步骤$k+1$，窗口现在已满。 在生存状态下，窗口包含$k$不同的值，因此下一个值安全的概率变为$\frac{N-k}{N}$。 该值对于所有后续步骤保持稳定。 
3. 我们迭代地计算生存概率。 让$S_i$是到目前为止没有发生碰撞的概率$i$。 我们初始化$S_0 = 1$，并使用上面导出的逐步生存因子进行更新。 
4. 使用恒等式计算预期停止时间$E[T] = \sum_{i \ge 0} S_i$，它将停止时间转换为生存概率的总和，而不是直接枚举停止事件。 
5. 我们将总和分为两部分：有限前缀$i \le k$，其中生存概率随着可用选择的减少而乘法下降，并且无限尾部从$k+1$，它成为一个具有比率的几何级数$\frac{N-k}{N}$。 
6. 我们使用以下的运行积来计算有限前缀$\frac{N-i}{N}$，然后添加从$S_k$。 
7. 所有计算均以模进行$10^9+7$，使用模逆除以$N$。 

### 为什么它有效

 关键的不变量是，只要没有发生冲突，活动窗口总是由不同的元素组成。 这迫使下一次绘制的禁止值的数量仅取决于当前窗口大小，而不是确切的历史记录。 一旦窗口达到大小$k$，禁止值的数量正好稳定在$k$，从那时起使该过程无记忆。 这将问题转换为变化概率的前缀，后面跟着一个固定的几何尾部，它唯一地确定了期望。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    N, k = map(int, input().split())
    
    # If k >= N, collision happens quickly in expectation,
    # but formula still works if handled carefully.
    
    invN = modinv(N % MOD)
    
    # S = survival probability up to current step
    S = 1
    
    # expected value
    ans = 0
    
    # prefix up to k steps
    for i in range(1, k + 1):
        ans = (ans + S) % MOD
        S = S * (N - (i - 1)) % MOD * invN % MOD
    
    # tail geometric part: from step k+1 onward
    # survival ratio becomes (N-k)/N
    if N > k:
        ratio = (N - k) % MOD * invN % MOD
        
        # expected remaining contribution = S / (1 - ratio)
        # = S * N / k
        inv_one_minus = modinv((k % MOD))
        tail = S * N % MOD * inv_one_minus % MOD
        ans = (ans + tail) % MOD
    else:
        # if k >= N, collision guaranteed soon; treat tail carefully
        ans = ans % MOD
    
    print(ans % MOD)

if __name__ == "__main__":
    solve()
```代码维护了运行生存概率`S`，它表示在“到目前为止所有不同”假设下，在当前前缀长度内没有发生重复的概率。 每次迭代更新`S`使用可用的新鲜选择的数量。 前缀贡献累积预期生存质量。 

一旦我们达到尺寸`k`，该过程进入稳定状态，其中每次新绘制都避免与概率发生碰撞`(N-k)/N`。 这创建了一个几何尾部，它使用模逆以闭合形式求和`k`。 

一个微妙的实现细节是干净地处理模块化划分。 表达式`S * N / k`模算术中的运算必须使用模逆来编写，并且乘法的顺序很重要，以避免溢出并保持正确性。

 ## 边缘情况

 当$N = 1$，每台计算机都有相同的名称。 第一次插入总是安全的，第二次插入立即引起冲突。 该算法处理这个问题是因为在第一步之后生存概率下降到零，使得期望值崩溃到一个小常数。 

什么时候$k = 1$，窗户上永远不会容纳多于一台计算机。 每台新计算机仅与前一台计算机进行比较。 这退化为一个简单的几何过程，当两个连续的抽奖匹配时会发生碰撞。 前缀计算自然地简化为单步生存模型，并且尾部公式在第一步之后立即变得精确。 

什么时候$k \ge N$，如果早期没有发生冲突，窗口足够大，最终可以包含所有可能的名称。 前缀计算准确地捕获了不断缩小的可用性，直到系统转变为不存在新值的状态，迫使几何尾部消失。
