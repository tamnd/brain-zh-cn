 ## 解决方案
 ## 问题理解

 我们给出了几个独立的场景。 每个场景中都有$N$布谷鸟钟，每个时钟都有自己固定的周期行为。 钟$i$有时会发出声音$1, 1+i, 1+2i, 1+3i,\dots$。 所有时钟最初都已对齐，以便第一个声音在某个时间发生$1$，然后他们继续重复各自的周期。 

我们被问到：在特定时间$T$，有多少个时钟会在那一刻准确地发出声音。 

所以对于固定时钟$i$，我们需要检查是否$T$属于等差数列，起始于$1$有步骤$i$。 这相当于检查是否$T-1$可以整除$i$。 因此，每个测试用例的答案是整数的数量$i$在范围内$[1, N]$这样$i \mid (T-1)$。 

约束条件达到$N, T \le 10^9$以及最多 100 个测试用例。 这立即排除了任何迭代每个测试用例的所有时钟的方法，因为这将是$O(NQ)$，在最坏的情况下达到$10^{11}$运营。 

当出现微妙的边缘情况时$T = 1$。 在这种情况下，每个时钟都会在时间 1 响起，因为所有级数都从 1 开始。公式$T-1 = 0$意味着每一个$i$除以 0，所以答案应该是$N$。 任何忘记此特殊结构的实现都可能意外返回 0 或错误处理除法检查。 

另一种需要注意的情况是大$T$在哪里$T-1 > N$。 一种朴素的除数计数方法，假设除数始终小于$N$可能会忽略这样一个事实：我们只计算除数$N$，并非所有因数$T-1$。 

## 方法

 直接模拟方法很简单。 对于每个时钟$i$从 1 到$N$，我们检查是否$T-1$可以整除$i$。 这是根据时钟的观察正确得出的$i$准时响起$1 + k i$，所以我们只需要测试成员资格$T$按照这个顺序。 瓶颈很明显：对于每个测试用例，我们可能需要扫描最多$10^9$值，远远超出了可行的范围。 

关键的结构见解是，问题实际上并不是所有独立的时钟，而是单个数字的除数$T-1$。 每个有效的时钟索引恰好是以下的除数$T-1$，我们只关心那些不超过的除数$N$。 这将问题转化为有界除数计数问题。 

而不是迭代所有可能的$i$，我们枚举除数$T-1$通过扫描有效地$\sqrt{T-1}$。 对于每个除数$d$，我们同时考虑$d$和$(T-1)/d$，如果它们在范围内，则对它们进行计数$[1, N]$。 这将每个测试用例减少到$O(\sqrt{T})$，这足够快了，因为$\sqrt{10^9} \approx 31623$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N)$每个测试用例|$O(1)$| 太慢了|
 | 最佳 |$O(\sqrt{T})$每个测试用例|$O(1)$| 已接受 |

 ## 算法演练

 我们以直接连接时钟和除数的形式重写条件。 一个时钟$i$准时响起$T$当且仅当$T = 1 + k i$对于某个整数$k$，这相当于$T-1$可以被整除$i$。 

我们表示$x = T - 1$。 任务变成计算有多少个整数$i$满足$1 \le i \le N$和$i \mid x$。 

### 步骤

 1. 计算$x = T - 1$。 这将算术级数条件转换为干净的整除条件。 问题的结构完全被捕获$x$。 
2. 将应答计数器初始化为零。 这将累积有效除数$x$也尊重上限$N$。 
3. 迭代$i$从 1 到$\lfloor \sqrt{x} \rfloor$。 对于每个$i$，检查是否$i$划分$x$。 这是有效的，因为除数在平方根周围成对对称。 
4. 每当$i \mid x$，同时考虑$i$和$x / i$作为潜在的时钟索引。 每个代表一个有效的除数对。 
5. 对于每个候选除数，仅当它最多时才将其添加到答案中$N$。 这确保我们只计算系统中存在的时钟。 
6.如果$i = x / i$，避免重复计算，因为它是一个完美的方形案例。 

### 为什么它有效

 每个有效的时钟索引恰好是以下的除数$x = T-1$，以及每个除数$x$对应于按时响铃的时钟$T$。 枚举结束$\sqrt{x}$保证我们找到所有除数对而没有遗漏，因为任何除数大于$\sqrt{x}$与较小的配对。 过滤方式为$N$确保我们只计算存在的时钟，因此该算法精确计算除数之间的交集$x$和范围$[1, N]$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q = int(input())
    for _ in range(q):
        n, t = map(int, input().split())
        x = t - 1
        
        ans = 0
        
        i = 1
        while i * i <= x:
            if x % i == 0:
                if i <= n:
                    ans += 1
                j = x // i
                if j != i and j <= n:
                    ans += 1
            i += 1
        
        print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循除数对枚举策略。 转变$x = T-1$每个测试用例应用一次。 循环至$\sqrt{x}$确保我们在最坏的情况下只执行大约 3 万次迭代。 支票`j != i`防止重复计算$x$是一个完美的正方形。 附加条件`<= n`强制执行时钟范围约束。 

这里一个常见的错误是忘记了$i$和$x//i$必须进行测试$N$独立。 另一种是错误地从 0 开始循环或错误处理案例$x = 0$，其中每个整数直到$N$是有效的。

 ## 边缘情况

 当$T = 1$，该算法有效地设置$x = 0$。 除数条件退化，因为每个整数都除零。 正确的行为是所有$N$时钟在时间 1 响铃。实现在测试包装器中显式处理此问题，因为基于循环的除数逻辑如果不加以保护，将无限地过度计数或行为不正确。 

什么时候$x$是一个完全平方数，例如$T = 10$给予$x = 9$，除数对$(3,3)$出现一次。 支票`j != i`确保只计算一次，保持正确性。 

什么时候$N$小于$\sqrt{x}$，许多除数成对出现，但只有一个子集有效。 有条件的`<= n`确保我们只计算实际存在的时钟，无论有多少个数学除数$x$有。
