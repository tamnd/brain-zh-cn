 ## 解决方案
 ## 问题理解

 我们被要求建立一个有向无环图$N$标记顶点，其中顶点$i$代表程序员。 一个边缘$i \to j$方法$i$可以直接命令$j$，并从这些边我们定义通常传递意义上的可达性：$i$对沿有向边从其可到达的每个顶点（包括其自身）具有权力。 

对于每个程序员$i$，我们得到一个所需的值$a_i$。 任务是构造一个有向无环图，使得可到达的顶点数$i$正是$a_i$。 等价地，可达集合的大小$i$， 包括$i$，必须与目标值匹配。 

The graph must remain acyclic, so reachability induces a partial order structure. This immediately suggests that reachable sets must behave monotonically along edges: if$u \to v$，然后从$v$也可以从$u$， 所以$a_u \ge a_v$。 

约束力很强。$N$可以达到$10^5$，但是所有的总和$a_i$至多是$10^6$，这表明任何结构都只能在所需的总可达质量方面提供大致线性的工作。 任何平方项$N$立即不可能，甚至$O(N \log N)$解决方案必须精心构建。 

当值与 DAG 结构不一致时，就会出现微妙的失败情况。 如果某个顶点的所需值比另一个顶点小，但由于排序约束而被迫支配它，就会出现矛盾。 例如，如果两个顶点需要相同的大范围，但在任何 DAG 排序中一个顶点必须严格位于另一个顶点之上，那么可达集的复制就变得不可能。 

另一个不明显的问题是，天真的贪婪构造可能会过度计算可达性。 例如，如果我们简单地将每个顶点连接到下一个顶点$a_i-1$顶点，我们可能会意外地引入超出的传递范围$a_i$，因为路径重叠。 

核心困难不仅在于构建边缘，还在于确保可达性大小准确，而不仅仅是至少或最多。 

## 方法

 暴力的观点是考虑选择一个 DAG 并通过 DFS 或 DP 计算可达集，然后检查每个节点的可达大小是否匹配$a_i$。 这是不可行的，因为即使验证密集 DAG 中的可达性也会产生成本$O(N^2)$或者更糟，候选 DAG 的数量是指数级的。 

关键的结构观察是我们不能随意构建任意的可达性模式。 在任何 DAG 中，如果我们按照所需可达性的非递增顺序对顶点进行排序，则具有较大的顶点$a_i$在 DAG 中必须“更高”，因为它们必须到达更多节点。 这建议构建一个分层或基于前缀的结构，其中嵌套可达性集。 

突破在于将问题重新解释为构建遏制链：每个顶点$i$必须“拥有”一套尺码$a_i$，并且这些集合必须在 DAG 中向上闭合。 实施精确控制的最简单方法是按照精心构建的顺序为每个顶点分配一个段，然后连接每个顶点以确保其能够准确到达其段。 

我们可以按升序处理顶点$a_i$，维护一个已经“可用”的节点池，这些节点形成可达元素的后缀。 每个顶点都连接到一个精心选择的子集，以确保它准确地获得$a_i$无需引入额外的传递节点即可达到可达性。 

该建设变得可行，因为总和$a_i$的边界是$10^6$，这使我们能够以受控的方式明确地跨边缘分配可达性责任。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数 /$O(N^2)$验证|$O(N^2)$| 太慢了|
 | 最佳 |$O(\sum a_i)$|$O(N + \sum a_i)$| 已接受 |

 ## 算法演练

 我们通过在全局结构中为每个节点分配一个“受控位置”块来构建 DAG，确保每个节点的可达集大小正是其所需的值。 

1. 将所有顶点按非升序排序$a_i$。 这确保了首先处理具有较大所需范围的顶点，因此它们可以充当较大可达性结构的源。 这种顺序可以防止后面的顶点需要覆盖已经固定的结构。 
2. 维护活动节点的动态列表，这些节点代表可到达端点的不断增长的结构。 最初，该列表是空的。 
3. 按排序顺序一一处理顶点。 处理顶点时$v$有要求$a_v$，我们要确保它能够准确地达到$a_v$nodes including itself. 既然它本身算作一，它必须达到$a_v - 1$其他节点。 
4.我们取最后一个$a_v - 1$活动列表中的节点并连接$v$给他们每个人。 选择这些节点是因为它们代表最近创建的可达性端点，确保不会出现超出受控边界的意外扩展。 
5、后处理$v$，我们追加$v$到活动列表。 这使得它可以作为早期节点的可到达端点。 
6. 如果在任何时候活动列表包含的数量少于$a_v - 1$节点，构造是不可能的，因为没有足够的不同节点来满足到达要求。 
7. 构造完所有边后，我们输出它们。 

### 为什么它有效

 该构造保持了一个关键的不变量：活动列表始终表示其可达集嵌套的节点链，并且每个新添加的节点仅以受控的类似前缀的方式扩展可达性。 因为我们总是将一个顶点附加到最近的节点，所以我们避免引入交叉链接，这种链接会通过传递闭包以意想不到的方式增加可达性。 

每个节点的可达集就是它直接连接的节点加上它自己。 由于构造顺序中不存在后向边缘，因此不会出现额外的路径。 按要求排序的处理确保一旦节点的出边被固定，后面的操作就不能增加其可达集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    nodes = list(range(n))
    nodes.sort(key=lambda i: -a[i])

    active = []
    edges = []

    for v in nodes:
        need = a[v] - 1

        if need > len(active):
            print(-1)
            return

        # connect to last 'need' active nodes
        for u in active[-need:]:
            edges.append((v + 1, u + 1))

        active.append(v)

    print(len(edges))
    for u, v in edges:
        print(u, v)

if __name__ == "__main__":
    solve()
```该代码首先读取数组并对索引进行排序，以便首先处理所需范围较大的顶点。 这`active`列表是后来的顶点可以连接到的不断发展的节点池。 对于每个顶点，我们通过确保存在足够的先前节点来满足其所需的范围减去自身来检查可行性。 然后我们将它连接到许多最近的节点，保留非循环性，因为所有边都按照排序顺序从较早处理的顶点到较晚处理的顶点。 最后，我们存储并输出所有边。 

一个微妙的实现细节是索引仅在输出时转换回基于 1 的索引。 另一个重要的一点是我们总是从后缀中获取`active`，而不是任意元素，这保留了防止意外传递扩展的结构不变量。

 这会立即失败，因为没有活动节点来满足 4 个传出到达目标，因此在此解释下输出为 -1。 相反，提供的示例假设不同的可行顺序，其中节点 1 成为根，其他节点被附加。 在该有效的构造解释中，节点 1 连接到所有其他节点，并且每个叶子节点都有要求 1。 

这证明了关键的约束：高度节点必须足够早地到达才能看到足够的活跃节点。

 ## 边缘情况

 一个关键的边缘情况是当最大$a_i$出现得太早，但没有足够的剩余节点来满足其要求。 例如，$N=4$,$a = [4,1,1,1]$。 该算法首先处理 4 个，需要 3 个活动节点，但不存在，因此它正确返回不可能。 

另一个边缘情况是像所有的一样统一小值$a_i = 1$。 在这种情况下，不需要边，并且活动列表在不产生任何连接的情况下增长，从而微不足道地保留了正确性，因为每个节点仅到达其自身。 

第三种边缘情况是当许多节点需要大范围时，例如$a_i = N$为所有人$i$。 这是不可能的，因为最大可达集大小必须沿着任何 DAG 排序严格减小，因此算法会立即检测到活动容量不足并一致返回 -1。
