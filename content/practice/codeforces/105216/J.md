 ## 解决方案
 ## 问题理解

 我们有一群武士和一些已经互相尊重的对。 尊重是对称的，因此信息可以被视为无向图，其中顶点是武士，边是现有的尊重关系。 

我们可以“引入”对，以便添加任何缺失的边，这意味着我们可以将任何非边变成边，但此类添加的数量必须受到限制。 执行这些添加后，我们必须将所有武士分为两个非空组，以便每个组内的每个武士都尊重同一组中的所有其他武士。 用图的术语来说，在我们添加边之后，两个群体都必须产生派系。 

因此，任务不仅是决定这样的划分是否可行，而且还要显式构造一个有效的划分和一组添加的边，使两个部分完全连接。 

关键的结构要求是每个组必须形成一个完整的子图。 必须显式添加组内任何缺失的边。 组之间的优势根本不重要，因为候选人只在自己的组内进行检查。 

对允许引入数量的限制迫使我们要小心：如果我们选择一个非常不平衡的分区，一组就会变得很大，我们可能需要在其中添加几乎所有缺失的边。 在空图的极端情况下，将几乎所有顶点放入一组将需要添加 θ(N²)，这可能会违反限制。 

唯一重要的边缘情况是当 N = 1 时。在这种情况下，我们无法分成两个非空组，因此答案立即不可能。 

## 方法

 蛮力的想法是尝试将所有可能的顶点划分为两个非空集合，并计算每个集合内有多少条缺失的边。 对于固定分区，我们将精确添加那些缺失的边，并检查添加的数量是否在允许的范围内。 这是正确的，因为任何有效的解决方案都对应于某个分区。 然而，可能的分区有 2^N 种，甚至评估一个分区也需要扫描所有对，导致 O(N²·2^N)，这对于 N 高达 1000 来说是完全不可行的。 

关键的观察是我们实际上不需要寻找巧妙的分区。 任何分区只要我们控制其平衡就可以发挥作用。 我们付出的成本是两组内缺失边的数量。 当图根本没有边时，此成本最大化，因为每一对都是缺失的边。 在最坏的情况下，所需添加的数量仅取决于两个组的大小，而不取决于图的结构。 

这将问题简化为选择一个保证内部对安全上限的分区。 最好的策略是尽可能均匀地分割顶点。 那么即使在最坏的情况下，内部对的数量也受到大约 N²/4 的限制，这与限制相符。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解分区 | O(2^N·N²) | O(2^N·N²) | O(N) | 太慢了 |
 | 平衡分区+添加缺失边| O(N²) | O(N²) | 已接受 |

 ## 算法演练

 我们仅基于索引构建固定分区。 

1. 将顶点集分成两组：前 ⌊N/2⌋ 顶点前往 S1，其余顶点前往 S2。 这确保了当 N ≥ 2 时两个组都非空。 
2. 构建一个结构（例如邻接矩阵）来表示哪些对已经具有尊重关系。 这使我们能够快速测试边缘是否存在。 
3. 对于S1 内的每对顶点，检查是否已存在边。 如果没有，我们将添加它并记录为简介。 
4. 对 S2 内的每一对重复相同的过程。 
5. 输出所有记录的添加边。

我们不考虑跨组边的原因是这个问题从来不需要 S1 和 S2 之间的连接。 只有内部完整性才重要。 

### 为什么它有效

 该结构保证 S1 和 S2 都成为派系，因为每个缺失的内部边缘都被显式添加。 唯一剩下的问题是添加的边数是否超过限制。 

最坏的情况发生在初始图没有边时。 那么S1和S2内的所有对都丢失了，所以加法的数量是C(|S1|,2) + C(|S2|,2)。 当分割尽可能平衡时，这个表达式会最大化，这正是我们构建分区的方式。 在这种情况下，总数最多为 N(N−1)/4，这符合约束条件。 输入中的任何现有边只会减少所需添加的数量，而不会增加它，因此界限始终成立。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M = map(int, input().split())
    if N == 1:
        print("NO")
        return

    adj = [[False] * N for _ in range(N)]

    for _ in range(M):
        a, b = map(int, input().split())
        a -= 1
        b -= 1
        adj[a][b] = adj[b][a] = True

    S1 = list(range(N // 2))
    S2 = list(range(N // 2, N))

    ans = []

    def add_edges(group):
        g = group
        for i in range(len(g)):
            for j in range(i + 1, len(g)):
                u, v = g[i], g[j]
                if not adj[u][v]:
                    adj[u][v] = adj[v][u] = True
                    ans.append((u + 1, v + 1))

    add_edges(S1)
    add_edges(S2)

    print("YES")
    print(len(ans))
    for u, v in ans:
        print(u, v)

if __name__ == "__main__":
    solve()
```该解决方案首先拒绝唯一不可能的情况 N = 1，因为无法形成两个非空子集。 

然后我们构建一个邻接矩阵，以便我们可以在恒定时间内测试缺失的边。 这很重要，因为我们可能需要检查最多 O(N²) 对。 

分区通过索引拆分确定性地固定，这避免了任何搜索或优化。 

最后，我们迭代每个组内的所有对并精确添加缺失的边。 邻接矩阵会随着我们的操作而更新，以避免重复添加。

 ## 边缘情况

 对于 N = 1，算法立即拒绝这种情况，因为不存在划分为两个非空集合的情况。 

对于空图，必须添加所有内部对。 平衡分割确保添加的边数保持在允许的范围内，并且算法将显式枚举每个组内的所有缺失对。 

对于全连接图，根本不进行任何添加。 该算法仍然输出有效的分区，并且两个组仍然是派系，没有任何修改。 

对于混合图，现有边仅减少所需插入的数量。 由于我们只添加缺失的内部边缘，因此我们永远不会违反正确性或超出约束。
