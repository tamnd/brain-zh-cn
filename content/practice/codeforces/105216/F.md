 ## 解决方案
 ## 问题理解

 我们得到一行奖品值，每个奖品都有一个正整数值。 约翰有分数限制$p$，且只允许挑选价值不超过的奖品$p$。 在所有有效奖品中，他想要价值最大的一个。 

简单来说，我们扫描整数列表，并且只允许考虑那些最多为给定阈值的整数。 任务是找到这些符合条件的整数中的最大值。 

限制条件很小：奖品数量$n$和所有值，包括$p$，最多 1000。这立即告诉我们，即使是对阵列进行简单的线性扫描也足够快。 一个$O(n)$甚至$O(n^2)$方法在限制内仍然可以舒适地运行，但超出限制的任何东西都是不必要的复杂性。 

输入的结构也很简单：一行给出了大小和限制，第二行给出了奖品值列表。 

即使在如此简单的问题中，一些边缘情况也很重要。 首先，所有值可能等于$p$，这意味着每个项目都是有效的，答案就是该值。 其次，最大有效值可能出现在列表的末尾，因此如果盲目应用任何假定排序顺序的早期停止策略都将失败。 第三，值小于$p$可能出现在任何地方，我们必须确保我们不会意外地选择第一个有效元素而不是最大有效元素。 

## 方法

 暴力破解的想法与最终的解决方案几乎相同。 我们扫描每一个奖品并记录我们被允许获得的最佳价值。 对于每个元素，如果小于或等于$p$，我们将其与当前的最佳答案进行比较，如果更大则进行更新。 

这是有效的，因为每个有效的候选者都必须至少检查一次，并且没有顺序保证可以让我们跳过元素。 操作数正好是$n$与阈值和最多的比较$n$更新最大变量，所以总体而言$O(n)$。 

除此之外没有任何有意义的优化。 排序会产生成本$O(n \log n)$而不改进逻辑。 对于静态数组上的单个查询来说，堆或段树等数据结构是不必要的开销。 关键的观察结果是，问题纯粹是静态列表上的约束最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力扫描|$O(n)$|$O(1)$| 已接受 |
 | Sorting then filtering |$O(n \log n)$|$O(1)$或者$O(n)$| 已接受但没有必要 |
 | 最佳扫描|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们处理该数组一次，同时维护一个变量，该变量存储迄今为止看到的最佳有效奖品。 

1. 初始化变量`best`其值保证小于任何有效答案，例如 0。这是有效的，因为所有奖品值都至少为 1，因此 0 不会意外成为答案。 
2. 迭代每个奖品值$v_i$在数组中从左到右。 
3. 对于每个值，检查它是否满足$v_i \le p$。 这个条件确保我们只考虑允许约翰获得的奖品。 
4. 如果该值有效，则将其与`best`。 如果$v_i > best$， 更新`best`到$v_i$。 此步骤确保我们始终保留迄今为止看到的最大有效值。 
5.处理完所有值后，输出`best`作为最终答案。 

这个过程起作用的原因是每个可行的候选者都被检查一次，并且算法在处理第一个候选者后保持不变$k$元素，`best`存储该前缀中所有有效元素中的最大值。 由于最终答案仅取决于有效值中的全局最大值，因此将此不变量扩展到整个数组可以保证正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, p = map(int, input().split())
    arr = list(map(int, input().split()))
    
    best = 0
    for v in arr:
        if v <= p and v > best:
            best = v
    
    print(best)

if __name__ == "__main__":
    solve()
```解决方案首先读取列表的大小和阈值。 然后它将所有奖品值解析到一个数组中。 

变量`best`初始化为0，因为所有值都是正数，所以这是约束下最大跟踪的安全单位元素$v_i \le p$。 

该循环在一次传递中执行两项检查：首先验证关于$p$，然后更新最大值。 此顺序可防止无效值导致的错误更新。 

最后，直接打印计算出的最大值。

 ## 边缘情况

 一种边缘情况是所有元素都大于$p$。 例如，如果$p = 2$数组是$[5, 6, 7]$，没有符合条件的值。 算法初始化`best`为 0 并且从不更新它，产生 0。由于该问题保证至少存在一个有效选择，因此这种情况不会出现在官方输入中，但实现仍然可以安全地处理它。 

另一种边缘情况是最大有效值出现在第一个位置时。 用于输入$p = 10$， 大批$[9, 1, 8]$，算法正确设置`best`立即变为 9，并且不会受到后来更小的值的影响。 

最后的边缘情况是最大有效值出现在末尾时。 用于输入$p = 10$， 大批$[1, 2, 10]$，算法逐步更新并正确捕获最后一个元素为最佳元素。 这证实了正确性不需要任何排序假设。
