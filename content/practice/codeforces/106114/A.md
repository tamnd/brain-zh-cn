 ## 解决方案
 ## 问题理解

 我们给出了 n 行 m 列的矩形排列的单元格。 每行 i 最初包含一块装在左侧的 ai 石头，因此在第 i 行中，第 1 列到 ai 都被填充，其余单元格为空。 所描述的操作是按列应用的类似重力的变换：所有石头垂直向下落，直到它们到达各自列中可能的最低空位置。 

下落过程稳定后，每根柱子都会从底部填充该柱子中原来存在的尽可能多的石头，而上面的任何空白空间都会保留。 任务是计算在重新配置之后，每行最终有多少颗石头。 

关键的困难在于，初始状态是以压缩的行方式给出的，但下降的过程是列驱动的，所以我们必须在心里转置结构。 

约束 n, m ≤ 10^4 意味着在最坏的情况下，在显式网格上进行 O(nm) 模拟在时间和内存上都太大。 完整的模拟最多需要 10^8 个单元，这在严格限制下的 Python 中是边界或不安全的。 这促使我们对计数进行推理，而不是明确的网格。 

当行具有非常不同的 ai 值时，就会出现微妙的边缘情况。 例如，如果 n = 3、m = 5 且 a = [5, 1, 1]，则第一行在所有列中贡献许多石子，而较低的行仅在第一列中贡献。 掉落后，列的高度差异很大，并且天真的逐行重新分配会放错石头，除非我们明确考虑按列累积。 

另一种失败模式来自于错误地假设行独立地保留其计数。 例如，认为每一行都有一颗石头是错误的，因为掉落会在行之间重新分配石头。 

## 方法

 蛮力解释构建了一个明确的网格，标记所有左对齐的棋子，然后逐列重复模拟重力或反复将棋子向下推直到稳定。 这是正确的，因为它确实遵循流程定义。 然而，网格最多可以包含 n × m 个单元，并且每个模拟步骤可能需要多次扫描列或执行每个单元的移动。 在最坏的情况下，每个单元都会被处理多次，从而导致大约 O(nm) 的内存和可能的 O(nm) 或更糟糕的时间，具体取决于实现开销。 当 n 和 m 达到 10^4 时，这变得太大了。 

关键的观察结果是，最终的配置仅取决于列总数，而不取决于石子最初在行内的分布方式。 每行 i 贡献分布在前 a_i 列的 a_i 石头。 如果我们反转观点，每列 j 恰好接收行 i 的数量，使得 a_i ≥ j。 一旦我们知道列的总和，引力就会简单地将这些总和排序为行：每行 i 最终接收高度至少达到 i 的列的总数。 

因此，我们不是模拟运动，而是计算列高上的频率数组，然后使用后缀累加将其转换为行计数。

 | 蛮力 | O(纳米) | O(纳米) | 太慢了 |
 | 最佳| O(n + m) | O(米) | 已接受 |

 ## 算法演练

 1. 构建一个大小为 m 的数组 cnt，其中 cnt[j] 计算有多少行 ai ≥ j + 1。这对应于在落下之前在第 j + 1 列中出现的石子数量。 此步骤用计数视角取代显式网格构建。 
2. 对于从 1 到 n 的每一行，我们想知道有多少列至少有 i 个石头。 该数字正是 cnt[i - 1]。 这是有效的，因为当且仅当一列的高度至少为 i 时，该列在掉落后才对第 i 行做出贡献。 
3. 输出 cnt[0], cnt[1], ..., cnt[n - 1] 作为最终行计数。

这种转换纯粹是观点的改变：最终配置中的行由列高决定，列高由原始 ai 值的前缀覆盖率决定。 

### 为什么它有效

 每个原始行仅向列的前缀贡献石子，因此每列 j 累积的行数恰好等于延伸到至少 j 的行数。 倒塌后，柱子成为独立的堆垛，其高度是固定的。 最后一行 i 正好占据堆叠高度至少为 i 的那些列，因为重力从底部向上填充，没有间隙。 这在“最终行占用”和“高度≥i的列数”之间建立了直接等价，这保证了计数变换的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
a = list(map(int, input().split()))

cnt = [0] * (m + 2)

for x in a:
    cnt[1] += 1
    if x + 1 <= m:
        cnt[x + 1] -= 1

for i in range(1, m + 1):
    cnt[i] += cnt[i - 1]

ans = []
for i in range(1, n + 1):
    ans.append(str(cnt[i]))

print(" ".join(ans))
```该代码避免构建任何网格。 相反，它使用列索引上的差异数组来计算延伸到每列的行数。 前缀总和将其转换为实际的列高。 然后通过读取有多少列达到至少高度 i 来直接回答每一行 i 。 

一个常见的陷阱是混淆了行索引和列索引。 数组 cnt 按列高度阈值而不是按行索引。 另一个微妙之处是在应用差异数组时确保边界，因为超过 m 的列不得更新。 

## 工作示例

 ### 示例 1

 输入：

 n = 3，m = 4

 a = [3, 1, 2]

 我们计算列覆盖率。 

| 步骤| 一个[我] | 更新对cnt的影响 | 前缀 | 后的 cnt
 | --- | --- | --- | --- |
 | 1 | 3 | + 1, - 4 | [0,1,1,1] |
 | 2 | 1 | + 1, - 2 | [0,2,1,1] |
 | 3 | 2 | + 1, - 3 | [0,3,2,1] |

 现在前缀后的 cnt 意思是：

 第 1 列有 3 个石头，第 2 列有 2 个，第 3 列有 1 个，第 4 列有 0 个。 

最后几行：

 第 1 行 = 高度 ≥ 1 = 3 的列

 第 2 行 = 高度 ≥ 2 = 2 的列

 第 3 行 = 高度 ≥ 3 的列 = 1

 所以输出是[3,2,1]。 

这证实了每一行纯粹是通过阈值列高来确定的。 

### 示例 2

 输入：

 n = 4，m = 5

 a = [5, 3, 3, 1]

 柱高变为：

 [4,3,2,1,1]

 现在我们计算行输出：

 第 1 行 = 5 列

 第 2 行 = 3 列

 第 3 行 = 3 列

 第 4 行 = 1 列

 输出：[5,3,3,1]

 这表明，即使使用重复的值，该方法也可以正确聚合重叠的前缀，而无需显式模拟。

    import sys

 ## 边缘情况

 一个关键的边缘情况是一行主导所有列，例如 n = 3、m = 5、a = [5, 1, 1]。 列高度变得严重倾斜：第 1 列有 3 个石子，而第 2 列到第 5 列只有 1 个石子。该算法可以正确处理此问题，因为 cnt[j] 直接计算每列的行覆盖率，因此不需要有关均匀性的假设。 

另一个边缘情况是当所有 ai 都很小时，例如 a = [1, 1, 1, 1]。 那么只有第一列非零，前缀累加后我们得到列高[4,0,0,...]。 行输出变为[1,0,0,0]，这与下降后只有底部行接收单个占用列的事实相匹配。 

最后的边缘情况是最大饱和度，其中所有 ai = m。 在这种情况下，每列的高度为 n，因此每行都会收到 m 个石头。 该算法为所有 i 生成 cnt[i] = m，这正确反映了完整的矩形填充。
