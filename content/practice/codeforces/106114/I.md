 ## 解决方案
 ## 问题理解

 我们得到一个数字$n$和一个界限$R$。 对于每个基地$k$从 2 到$R$，我们写$n$在基地$k$，然后计算其数字之和。 在所有这些基数中，我们想要最小可能的数字和。 

所以任务不是转换$n$一次，但在不同的位置系统下重复地重新解释相同的整数，并衡量其表示在数字和方面的“轻量级”程度。 

约束是不对称的。 数量$n$可以大到$10^{12}$，所以任何线性依赖于的方法$n$已经太大了。 查询数量最多为 100 个，因此我们可以承受每个测试用例进行几百万次操作，但无法通过昂贵的分解重复重新处理所有基数。 

一个关键的观察范围$n$是那个$n \le 10^{12}$大约可容纳 40 位，因此二进制表示形式很小。 这暗示着对于大基数来说，跨基数的数字和是以结构化的方式表现的，而不是保持任意的。 

天真的思维的一个微妙的失败案例是假设数字总和总是随着基数的增加而减少。 例如，小基数可以与大基数不同地压缩结构，并且最小值不一定是最大的$R$或最小基数 2。 

另一个边缘情况是当$n$本身小于基数$k$。 在这种情况下，表示形式是一位数字，等于$n$，所以数字和等于$n$。 在假定多个数字始终存在的暴力实现中，这通常会被错误地跳过。 

## 方法

 最直接的方法就是尝试每一个基地$k$从 2 到$R$， 转变$n$进入基地$k$，计算数字和，并跟踪最小值。 将数字转换为基数$k$需要$O(\log_k n)$部门，所以在所有基地，这大致变成$$\sum_{k=2}^{R} O(\log n) \approx O(R \log n)$$这太慢了$R$很大，特别是如果$R$可以达到$10^{12}$。 即使对于中等$R$，重复除法是昂贵的。 

关键结构是当基数相对于$\sqrt{n}$，表示为$n$变得很短。 具体来说，如果$k > \sqrt{n}$， 然后$n$在基地$k$最多有两位数：$$n = a_0 + a_1 k$$和$a_1 \in \{0,1\}$。 事实上，当$k > n$，我们有一个数字，并且当$\sqrt{n} < k \le n$，我们有一个两位数的表示。 

这会压缩搜索空间：我们可以枚举可能的数字对，而不是单独迭代所有大碱基$(a_1, a_0)$并得出哪些碱基会产生它们。 这种对的数量大约为$\sqrt{n}$， 自从$a_1 k \le n$意味着$a_1 \le \sqrt{n}$。 

适用于小型基地$k \le \sqrt{n}$，我们可以直接模拟数字提取。 对于大基数，我们反转表示逻辑并枚举候选分解$n$化为一位或两位数。 

这将问题分为两种情况：小基数的直接模拟和大基数的组合枚举。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有基地 |$O(R \log n)$|$O(1)$| 太慢了|
 | 拆分+枚举最多$\sqrt{n}$|$O(\sqrt{n} \log n)$|$O(1)$| 已接受 |

 ## 算法演练

 ### 最优策略

 1. 将答案初始设置为$n$以基数 2 为单位。这保证了有效的上限，因为基数 2 生成明确定义的表示形式并且始终在范围内。 
2. 迭代所有基础$k$从 2 到$\min(R, \lfloor \sqrt{n} \rfloor)$。 对于每个碱基，重复划分$n$经过$k$并积累余数。 这直接计算该基数中的数字和。 

我们停下来的原因$\sqrt{n}$超过这个点，表示最多会缩小到两位数，因此与直接构造相比，模拟变得浪费。 
3. 在处理这些小基数时，用遇到的最小数字和更新答案。 
4. 现在处理大型基地$k > \sqrt{n}$。 在这个政权下，$n$只能表示为：$$n = a_0 + a_1 k$$在哪里$a_1 \ge 1$意味着$k \le n$。 所以我们只考虑碱基$R \cap [\sqrt{n}, n]$。 
5.而不是迭代$k$，迭代可能的值$a_1$。 对于每个$a_1$, 计算$a_0 = n \bmod a_1$-式分解约束，但更具体地说，观察：$$k = \frac{n - a_0}{a_1}$$在哪里$a_0 < k$。 这限制了有效$k$到由结构化表达式的除数引起的范围，因此我们枚举了$n$固定为两位数$a_1$和计算$k$。 
6. 对于每个有效分解，计算数字和$a_0 + a_1$，并更新答案。 
7. 在所有情况下取最小值。 

### 为什么它有效

 该算法依赖于结构不变量：每个表示$n$在基地$k > \sqrt{n}$最多有两位数。 这意味着数字和完全由一对决定$(a_1, a_0)$和$n = a_1 k + a_0$。 我们不是搜索基数，而是搜索可行的数字分解$n$。 每个有效的基数都对应于一个这样的分解，因此不会遗漏任何候选数，并且不会包含无效的数字和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def digit_sum_base(n, k):
    s = 0
    while n:
        s += n % k
        n //= k
    return s

def solve():
    n, R = map(int, input().split())

    # base 2 as initial bound
    ans = digit_sum_base(n, 2)

    lim = int(n ** 0.5)

    # small bases: direct simulation
    for k in range(2, min(R, lim) + 1):
        ans = min(ans, digit_sum_base(n, k))

    # large bases: at most two digits
    # n = a0 + a1 * k
    for a1 in range(1, lim + 1):
        k = n // a1
        if k > R or k < 2:
            continue

        a0 = n - a1 * k
        if 0 <= a0 < k:
            ans = min(ans, a0 + a1)

    print(ans)

if __name__ == "__main__":
    solve()
```功能`digit_sum_base`使用重复除法执行标准基数转换，这是有效的，因为每个数字恰好是每个步骤的余数。 

第一个循环覆盖了表示形式可能具有多个数字的所有基数。 截止时间为$\sqrt{n}$确保我们不会太频繁地进行这种昂贵的操作。 

第二个循环枚举可能的高位系数$a_1$。 对于每个，我们得出一个候选库$k$，并通过重构余数来验证是否一致$a_0$。 这避免了显式迭代所有大基数。

 ## 边缘情况

 ### 情况：基数等于 n

 输入：```
n = 100, R = 100
```什么时候$k = n$，表示为“1 0”，数字和为 1。该算法在大基数逻辑中捕获了这一点：$a_1 = 1$,$a_0 = 0$，给出总和 1。 

### 案例：非常小的 R

 输入：```
n = 50, R = 2
```仅基数 2 有效。 小基数循环确保我们仍然可以正确计算数字和，而不需要大基数枚举。 

### 情况：n 刚好低于平方根阈值

 输入：```
n = 1000000, R = 1000000
```在这里，两种制度有重叠。 小基数循环处理 1000 以内的所有基数，而大基数枚举确保 1000 以上的基数仍通过两位数分解进行检查。 不会遗漏任何基数，因为高于阈值的每个表示都必须是两位数。
