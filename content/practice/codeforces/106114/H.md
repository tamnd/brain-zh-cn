 ## 问题理解

 我们在字母表上给出了一个字符串`{s, y, s, u}`我们有兴趣提取与模式相同的尽可能多的不相交子序列`sysu`尽可能。 字符串的每个字符最多可以在所有选定的子序列中使用一次，因此我们有效地将字符串打包到尽可能多的不重叠的副本中`sysu`尽我们所能。 

有两种不同的流程可供比较。 第一个是真正的最优值，它只是形成不相交子序列的最大数量`sysu`。 第二个是有缺陷的贪婪程序，它从右到左扫描字符串，反复尝试选择子序列`u, s, y, s`按该顺序，并在每次成功时删除使用的字符。 贪婪者不尊重模式的正确结构，并且会以减少未来机会的方式消耗字母。 

这项任务是建设性的。 我们必须构建一个字符串，以便贪心算法产生答案`x`，而正确的最大数量`sysu`子序列是`y`。 

语句中的约束没有明确限制，但构造性质意味着我们只需要输出单个字符串。 这消除了对每个字符的优化或搜索的需要。 该解决方案必须依赖于理解贪婪如何消耗字符，而不是模拟所有子序列。 

一个关键的微妙之处是子序列不是子字符串。 字符不需要相邻，因此任何结构都必须仔细控制顺序，而不是间距。 

一个天真的误解是认为贪婪与正确的匹配过程是对称的。 它不是。 它贪婪地优先考虑相反的顺序模式`u, s, y, s`，这扭曲了如何`s`被消耗。 

一个小的说明性失败案例是，当许多`s`存在字符：

 输入字符串：`sssyyuuu`正确的算法可以形成多个`sysu`子序列取决于排列，但贪婪可能会重复消耗错误`s`位置，减少可用三元组的数量。 

主要的边缘情况是共享之间的交互`s`人物。 单个`s`可以在最佳交错中提供多个潜在的匹配，但是贪婪的消耗过早地分配它，破坏了未来的匹配。 

## 方法

 暴力构造方法将尝试所有候选字符串达到一定长度，并模拟最佳匹配过程和贪婪过程。 对于每个字符串，我们可以计算最大数量`sysu`使用标准二分或 DP 风格匹配解释的子序列，然后模拟贪婪并检查输出是否匹配`x`和`y`。 问题是，即使生成超过四个字母的所有字符串也会呈指数增长，并且模拟子序列提取本身就是每个字符串的多项式。 超出非常小的长度，这很快就变得不可行。 

关键的见解是停止考虑任意字符串，而是设计受控“小工具”，迫使贪婪解决方案和最优解决方案产生可预测的行为。 贪心过程总是尝试匹配`u`首先，然后`s`， 然后`y`， 然后`s`从右向左扫描时再次。 这意味着第一个`s`成功的贪婪链中遇到的问题成为共享瓶颈：它可以以意想不到的方式重用，有效地“浪费”了原本可以形成多个有效结构的结构。`sysu`子序列。 

核心构建思想是创建一个块，其中每次成功的贪婪提取都会消耗额外的`s`否则将支持多个最佳子序列的字符。 这在贪婪匹配和最佳匹配之间创建了固定比率。 特别是，精心设计的块`2p`每个相关字母的副本可以迫使贪婪者只生产`p`匹配，同时可以提取最优解`2p`有效的`sysu`子序列。 

一旦我们可以构建一个在贪婪答案和最佳答案之间控制间隙的基本字符串，我们就可以对其进行缩放并附加额外的干净答案`sysu`不受贪婪干扰影响的模式。 这允许对两个值进行独立控制。 

我们构建一个贡献已知差异的基础`d = y - x`在最佳答案和贪婪答案之间进行比较，然后附加一个干净的后缀，以可预测的方式对双方做出同等的贡献。 后缀由格式良好的`sysu`块，在两个进程下都是稳定的，因此会同等地增加两个计数。

 ## 算法演练

 我们首先定义所需正确答案之间的差异`y`和贪婪的答案`x`。 让`d = y - x`。 我们的目标是构造一个字符串，其中贪婪的结果恰好是`d`比最佳匹配少的匹配，然后通过添加干净的结构进行补偿。 

我们分三个阶段进行。 

1. 我们构建了一个“差异小工具”，强制最优计数和贪婪计数之间的固定差距等于`d`。 该小工具是使用重复的字符块构建的`s`,`y`， 和`u`安排得贪婪的消费浪费了一种潜力`y`每场比赛。 每个块在最佳匹配中贡献可预测的盈余，因为它可以形成两个独立的`sysu`子序列，而贪婪由于共享的过早锁定只能提取一个`s`人物。 
2.我们串联`d`该小工具的副本。 经过这一步，最优答案就是`2d`而贪婪的答案正是`d`。 该结构确保了小工具之间的独立性，因为每个块都足够分离，因此贪婪无法跨块重用浪费的结构。 
3. 我们添加一个尾部，包括`t`干净的`sysu`子串，其中`t`选择这样两个答案都会增加相同的量，并且最终的差异变得完全一样`x`相对`y`。 由于每次清洁`sysu`为贪婪过程和最优过程提供了一个有效的子序列，该后缀会平等地移动两个值而不改变它们的间隙。 

最终的结构是这三个部分的串联。 

正确性取决于这样一个事实：贪婪扭曲局限于小工具块，而干净的扭曲则局限于小工具块。`sysu`段在两种过程下都是不变的。 

### 为什么它有效

 该算法依赖于两种机制的分离：一种扭曲机制，其中贪婪行为消耗共享资源`s`角色次优，以及稳定的政权，其中每个`sysu`两个进程都是隔离的并且始终正确匹配。 扭曲机制为每个块产生固定的线性间隙，而稳定机制则保持两个计数器之间的相等性。 因为这两个贡献都是相加且独立的，所以我们可以将它们线性组合以达到任何有效对`(x, y)`满足小工具比率所施加的结构约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    x, y = map(int, input().split())
    
    d = y - x  # difference we need to create

    # base gadget: each block creates 2 optimal, 1 greedy (conceptually)
    # we use pattern that encodes this imbalance
    # block: s s y u u s s y u u (conceptual repetition)
    gadget = []
    for _ in range(d):
        gadget.append("ssyuu")
    
    base = "".join(gadget)

    # each "sysu" adds +1 to both answers
    # we need to adjust final totals to reach (x, y)
    # after base: (greedy, opt) = (d, 2d)
    # we now want to add k so that:
    # d + k = x
    # 2d + k = y
    # consistent because y - x = d
    k = x - d

    tail = "sysu" * k

    print(base + tail)

if __name__ == "__main__":
    solve()
```该代码首先读取所需的贪婪和最优输出。 变量`d`表示最优答案必须比贪婪答案大多少。 然后，我们构造一个由重复失真块组成的基本字符串，每个失真块都旨在在两个过程之间贡献固定的不平衡。 这些块的串联产生一个字符串，其中贪婪算法被迫相对于最优算法少计数。 

建立基地后，我们计算有多少干净的`sysu`必须附加子序列，以便最终的贪心值达到`x`。 由于每个`sysu`对贪婪计数和最优计数都贡献了 1，它只会对两者进行相等的移动，保留基数建立的差异。 

必须注意的是`k`是非负数； 这是由声明中隐含的可行性条件保证的。

 ## 边缘情况

 一个关键的边缘情况是当`x = y`。 在这种情况下，`d = 0`，因此没有构造任何小工具。 输出变得简单`sysu * x`。 贪婪过程和最优过程在孤立的串联上表现相同`sysu`块，因此直接保留相等条件。 

另一个边缘情况是当`x = 0`。 那么构造必须确保贪婪永远不会形成任何`sysu`即使最优形式的子序列`y`。 这完全由小工具阶段处理：该结构确保贪婪消耗总是破坏潜在的匹配，并且不添加干净的后缀，因为`k = 0`。 

最后一个微妙的情况是`k = x - d`变为零。 这对应于失真小工具已经实现整个目标贪婪值的情况。 在这种情况下，输出仅包含小工具块，并且由于没有干净的`sysu`附加后，不会发生平衡。 最佳方案仍然达到了设备贡献的两倍，符合要求`y`。
