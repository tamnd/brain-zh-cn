 ## 解决方案
 ## 问题理解

 我们得到一个字符串和三个大系数，它们定义两个选定子字符串的评分函数。 我们必须在字符串内选择两个不重叠的段。 第一个段必须匹配字符串的前缀，第二个段必须匹配字符串的后缀。 这些片段必须严格排序且不相交，它们对分数的贡献取决于它们的长度加上它们之间的间隙。 

任务是最大化一个线性表达式，该表达式奖励两个段的长度，并通过第三个系数奖励或惩罚它们之间的距离。 关键约束是两个子字符串不是任意的：第一个子字符串必须等于整个字符串的前缀，第二个子字符串必须等于整个字符串的后缀。 

约束允许字符串长度达到一百万，这立即排除了任何二次甚至$O(n \log n)$子串边界上的双循环。 任何解决方案本质上都必须在线性或接近线性的时间内处理字符串。 由于两个选定的子字符串都必须匹配前缀和后缀模式，因此问题从根本上讲是关于预先计算前缀匹配从每个位置延伸多远以及后缀匹配向后延伸到字符串中多远。 

一个天真的方法会尝试所有四元组$(l_1, r_1, l_2, r_2)$通过直接比较来验证前缀和后缀条件。 即使限制端点，由于子串匹配，这仍然是三次或更糟，并且显然会失败$n = 10^6$。 

更微妙的失败模式来自于错误地独立处理前缀和后缀匹配。 例如，子字符串可能与前缀匹配，但与候选后缀无效重叠，反之亦然。 另一个常见的错误是假设一旦知道前缀匹配长度，任何起始位置都可以重用它而无需验证边界约束。 

## 方法

 强力解释确定了选择两个片段的想法。 我们迭代第一个段的所有有效选择，确保它与字符串的前缀匹配，然后迭代与后缀匹配的所有有效第二段，并直接计算表达式。 即使通过哈希优化子串相等性，我们仍然面临$O(n^2)$候选对，这变得不可能$10^6$。 

关键的观察是有效段的结构受到极大限制。 等于前缀的子字符串完全由其结束位置决定。 类似地，等于后缀的子字符串完全由其起始位置决定。 这将问题从四个自由变量减少到两个有意义的边界：前缀匹配段的末尾和后缀匹配段的开头。 

一旦我们用这两个索引来表达一切，目标就变成了它们的位置和可预先计算的前缀后缀匹配长度的函数。 剩下的挑战是有效评估它们之间的最佳分割点，同时尊重排序约束$r_1 < l_2$。 这自然会导致预先计算每个位置的前缀匹配长度和每个位置的后缀匹配长度，然后扫描可能的分割点，同时保持最佳可实现的前缀贡献和后缀贡献。 

然后，问题变成具有前缀最大维护的线性扫描，而不是对段边界的组合搜索。

 ## 算法演练

 我们首先计算每个位置与字符串前缀的匹配程度。 我们定义一个数组`pref[i]`作为从位置开始的最长前缀的长度`i`。 这可以使用 Z 函数或 KMP 式预处理在线性时间内进行计算。 

我们类似地计算`suff[i]`，以位置结束的最长后缀匹配`i`，再次使用反向字符串处理。 

接下来我们重新解释两个选定的子字符串。 第一段在某个位置结束`r1`，并且它的最佳开始受到它与前缀的匹配程度的限制。 第二段从某个位置开始`l2`，其最佳可能的结尾受到后缀匹配的限制。 

然后，我们迭代两个段之间的所有有效分割边界，保持左侧的最佳可实现贡献，并将它们与右侧的贡献相结合。 

可以重新组织表达式，以便依赖于第一段的项仅依赖于其结束位置，而依赖于第二段的项仅依赖于其起始位置，加上它们之间间隙的线性惩罚或奖励。 

当我们扫描时，我们维持可能的第一段值的运行最大值`r1`从左到右。 对于每个潜力`l2`，我们将它与之前最好的结合起来`r1`满足排序约束。 

### 为什么它有效

 每个有效的解决方案都是由端点唯一确定的$r_1$和$l_2$。 一旦这些问题得到解决，最有效的$l_1$和$r_2$受到前缀和后缀匹配约束的强制，因此不需要独立优化。 扫描确保在评估候选分割时，我们已经考虑了所有有效的左段，并且维护的最大值确保我们永远不会错过更好的前缀对齐配置。 这保证了每个可行的配置在其规范表示中只被考虑一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def z_function(s):
    n = len(s)
    z = [0] * n
    l = r = 0
    for i in range(1, n):
        if i <= r:
            z[i] = min(r - i + 1, z[i - l])
        while i + z[i] < n and s[z[i]] == s[i + z[i]]:
            z[i] += 1
        if i + z[i] - 1 > r:
            l, r = i, i + z[i] - 1
    return z

def solve():
    n, x, y, zc = map(int, input().split())
    s = input().strip()

    # prefix matches
    z = z_function(s)
    pref = [0] * n
    pref[0] = n
    for i in range(1, n):
        pref[i] = min(z[i], n - i)

    rs = s[::-1]
    zr = z_function(rs)
    suff = [0] * n
    suff[n - 1] = n
    for i in range(n - 1):
        suff[i] = min(zr[n - 1 - i], i + 1)

    NEG = -10**30

    best = NEG
    best_left = NEG

    for r1 in range(n):
        if pref[r1] > 0:
            val_left = (r1) * x + pref[r1] * x
            best_left = max(best_left, val_left)

        l2 = r1 + 1
        if l2 < n and suff[l2] > 0:
            val_right = (n - l2 - 1) * y + suff[l2] * y
            best = max(best, best_left + val_right + (l2 - r1) * zc + x + y - zc)

    if best == NEG:
        print(0)
    else:
        print(best)

if __name__ == "__main__":
    solve()
```该解决方案从 Z 函数计算开始，以测量字符串的每个后缀与前缀的匹配程度。 这可以直接访问前缀相等约束，而无需子字符串比较。 

然后，我们对反转字符串计算镜像 Z 函数，以获得原始字符串中每个位置的后缀匹配长度。 这种对称性避免了构建后缀自动机或基于哈希的检查。 

横扫过去`r1`保持任何有效的第一段结束于或之前的最佳贡献`r1`。 每次前进时，我们都会使用该端点处的前缀匹配长度来更新此最佳值。 

对于每个潜在的分割，我们评估紧随其后开始的最佳有效第二段`r1`。 该组合使用预先计算的贡献和线性间隙项。 所有分割的最大值给出了最终答案。 

边界处理至关重要：前缀或后缀匹配长度为零的位置不得起作用，并且不存在有效对的情况必须返回零。

 ## 边缘情况

 一种重要的边缘情况是，除了完整字符串之外，没有任何子字符串与任何位置的前缀匹配。 在这种情况下，`best_left`永远不会变得有效，并且算法正确返回零，因为无法形成组合。 

另一个边缘情况是包含所有相同字符的字符串。 在这里，每个位置对于前缀和后缀匹配都有效。 扫描仍必须遵守顺序，并且间隙项将占主导地位。 该算法自然地处理这个问题，因为每个位置都会更新左右贡献。 

当最佳分割将两个段相邻时，会出现最后一个微妙的情况。 该公式仍然适用，因为间隙项变为零，并且预先计算的最佳左值和右值与邻接约束保持一致。
