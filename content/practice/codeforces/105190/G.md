 ## 解决方案
 ## 问题理解

 我们正在拍摄一系列$n$篮球尝试，每次尝试要么成功，要么失败。 关键的难点在于得分的概率不是固定的：它只取决于上一次投篮的结果，所以这个过程有一个简单的“一步记忆”。 

一开始，第一次射击有一个固定的成功概率，以分数形式给出$\frac{p_0}{q_0}$。 之后，系统的演变如下。 如果上一次射击成功，则下一次射击有概率成功$\frac{p_1}{q_1}$。 如果上一次射击未命中，则下一次射击有概率成功$\frac{p_2}{q_2}$。 

任务是计算所有成功射击的预期数量$n$尝试。 由于每次射击的概率取决于之前的结果，因此事件是相关的，我们不能独立地对待它们。 输出必须以模数形式给出$998244353$，这意味着每个分数都必须使用模逆来解释为模算术。 

这些约束是关于期望什么样的解决方案的主要提示。 测试用例的数量可以大到$5 \cdot 10^4$， 和$n$可以达到$10^{12}$。 这立即排除了任何模拟或每步动态编程。 甚至$O(n)$每个测试用例都是不可能的，甚至$O(\log n)$每一步都太慢了。 预期的解决方案必须将整个过程减少到每个测试用例的恒定操作数，通常是通过找到封闭形式的递归或将系统减少到小的线性变换。 

经常打破天真的方法的一个微妙问题是独立处理每个镜头或仅跟踪“当前概率”而不记住转变取决于先前的结果分布。 另一个陷阱是试图模拟随时间变化的概率：对于大的$n$，甚至重复单个线性 DP 步骤$10^{12}$次是不可行的。 

例如，如果有人错误地假设所有镜头都有概率$\frac{p_1}{q_1}$，答案就变成了$n \cdot \frac{p_1}{q_1}$，这显然是错误的，因为第一次射击和失误状态会影响接下来的一切。 

## 方法

 直接暴力方法很容易描述。 我们逐个镜头进行模拟。 我们跟踪前一个镜头是成功还是失败，并且在每一步中我们都会计算以该状态为条件的成功概率。 我们还维护状态的概率分布。 后$n$步骤，我们将每一步成功的概率相加以获得期望。 

这是正确的，因为它尊重马尔可夫依赖性：每一步仅以前一步为条件。 然而，成本是线性的$n$。 对于单个测试用例$n = 10^{12}$，这是完全不可行的。 

关键的观察结果是该过程只有两个相关状态：上一个镜头成功或失败。 这些状态之间的转移概率在先前的分布中是线性的。 这使我们能够将整个系统压缩为一维递归，以获得当前射击成功的概率。 一旦建立了递归式，期望值就变成了线性递归式的总和，它具有封闭的形式。 

这将问题从模拟长随机过程转变为评估模算术中的几何类型级数。

 ## 算法演练

 让$a = \frac{p_1}{q_1}$,$b = \frac{p_2}{q_2}$， 和$s_i$是概率$i$第-次射击成功。 

我们还定义$E_i$作为第一批中成功射击的预期数量$i$。 

1. 计算$s_1 = \frac{p_0}{q_0}$。 这是基本情况，因为第一次射击不依赖于任何先前的结果。 
2. 表达过渡$s_i$为了$i \ge 2$。 上一次射击有概率成功$s_{i-1}$，或有概率失败$1 - s_{i-1}$。 所以：$$s_i = s_{i-1} \cdot a + (1 - s_{i-1}) \cdot b$$3. 将递推式简化为仿射形式：$$s_i = b + (a - b)s_{i-1}$$这是一阶线性递推。 
4. 定义常量$d = a - b$和$c = b$， 所以：$$s_i = c + d s_{i-1}$$5.解决这个重复问题。 如果$d \ne 1$，它有闭合形式：$$s_i = s_* + (s_1 - s_*) d^{i-1}$$在哪里$s_* = \frac{c}{1-d}$是不动点。 
6. 计算期望值：$$E_n = \sum_{i=1}^{n} s_i$$代入封闭形式并分成两个总和：

 常数部分和几何级数部分。 
7. 使用模运算计算两个总和：

 几何和和算术几何组合。 这将一切简化为一些模幂和逆运算。 

如果$d = 1$，递推式变为：$$s_i = s_{i-1} + c$$所以$s_i$是线性的$i$，总和就变成了一个简单的算术级数。 

### 为什么它有效

 核心不变量是进程的整个历史记录完全由单个标量捕获$s_i$，前一次射击成功的概率。 All future probabilities depend only on this value, and the update rule is linear. 一旦系统变成线性递归，长期行为就会崩溃为封闭式表达式，并且对$n$简化为标准几何级数恒等式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n = int(input())
        p0, q0, p1, q1, p2, q2 = map(int, input().split())

        s1 = p0 * modinv(q0) % MOD
        a = p1 * modinv(q1) % MOD
        b = p2 * modinv(q2) % MOD

        d = (a - b) % MOD
        c = b

        if n == 1:
            out.append(str(s1))
            continue

        if d == 1:
            # s_i = s1 + (i-1)*c
            # sum = n*s1 + c*(n*(n-1)/2)
            n_mod = n % MOD
            sum_n = n_mod * (n_mod - 1) % MOD * modinv(2) % MOD
            ans = (n_mod * s1 + c * sum_n) % MOD
            out.append(str(ans))
            continue

        # fixed point s*
        denom = (1 - d) % MOD
        inv_denom = modinv(denom)
        s_star = c * inv_denom % MOD

        dn = pow(d, n, MOD)
        dn1 = pow(d, n - 1, MOD)

        # sum s_i = n*s* + (s1 - s*) * sum d^(i-1)
        geom = (dn - 1) % MOD * modinv(d - 1) % MOD

        ans = (n % MOD * s_star % MOD + (s1 - s_star) % MOD * geom) % MOD
        ans %= MOD

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现首先使用模逆将所有概率转换为模形式。 这是必要的，因为所有概率都是有理数，并且必须在模算术下进行评估。 

然后我们将转换规则简化为两个常数$a$和$b$，这完全确定了过程。 From there, the recurrence is simplified into a single linear form with multiplier$d = a - b$。 

解决方案仅分支一次：是否$d = 1$。 这种情况对应于简并递归，其中概率线性增长而不是几何增长，并且必须单独处理以避免在几何级数公式中被零除。 

其他一切都直接来自几何级数与线性项相结合的封闭式求和。

 ## 边缘情况

 一个微妙的情况是当$d = 1$， 意义$a = b + 1$在模算术中。 这使得递归是线性的而不是几何的。 例如，如果$a = 1$和$b = 0$，那么每一步都会使成功概率增加一个常数。 该算法显式地分离这种情况并计算算术级数和，避免在几何公式中被零除。 

另一种情况是$n = 1$。 递归公式假设至少有一次转换，因此答案必须直接返回$s_1$。 该实现在任何求幂之前处理这个问题。 

最后一个极端情况是当概率崩溃到相同的值时，$a = b$。 然后$d = 0$，并且序列在第一步之后变得恒定。 几何公式可以清楚地处理这个问题，因为$d^k = 0$为了$k \ge 1$，并且总和正确地减少到一个常数级数。
