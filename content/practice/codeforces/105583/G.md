 ## 解决方案
 ## 问题理解

 我们得到了一组整数。 两名选手，Kolya 和 Mitya，交替从这组中挑选号码，Kolya 首先开始。 取出所有数字后，每个玩家计算他们收集的数字的按位异或。 异或值较大的玩家获胜； 如果两个 XOR 值相等，则结果是平局。 

关键的结构是游戏本身不涉及游戏过程中的计分，仅涉及将数组最终划分为两个固定大小的子集。 唯一重要的是数字如何在两个玩家之间分配，因为选择的顺序不会影响最终的异或，只会影响每个玩家最终得到的子集。 

约束最多允许 2000 个数字，每个数字最多 10^18。 这立即表明，任何涉及子集枚举或模拟所有游戏状态的解决方案都是不可能的，因为分区的数量在 N 中呈指数级增长。甚至子集上的动态编程也是不可行的。 

重要的隐藏边缘情况是贪婪挑选策略并不明显有效。 例如，对于 1、2、3 这样的数字，本地选择并不能清楚地确定谁在 XOR 空间中受益，因为 XOR 是非单调的。 像“总是取最大的数字”这样的天真的想法失败了：

 输入：`1 2 3 4`这种贪婪策略可能会给第一个玩家 4 和 3，第二个玩家得到 2 和 1，但异或比较与总和排序不一致，因此总和的直觉会被打破。 

真正的困难在于，游戏简化为具有交替选择约束的组合划分问题，但最终评估仅取决于异或结构，而不是排序。 

## 方法

 暴力解释将游戏视为对状态的完整极小极大搜索：每个状态由剩余的数字和轮到谁来定义。 从每个状态，玩家选择一个数字并转换到下一个状态，在最终状态我们计算异或差。 

这种暴力方法原则上是正确的，因为它探索了每一个可能的游戏序列。 然而，子集的状态数量已经是 2^N，并且转换乘以 N，导致复杂度大约为 O(N·2^N)，这远远超出了 N 高达 2000 的任何可行限制。 

关键的观察是移动的顺序并不重要。 由于玩家交替选择并且两者都恰好采用了下限（N/2）或上限（N/2）元素，因此游戏相当于将多重集划分为两个固定大小的子集：Kolya 获得 ceil(N/2)，Mitya 获得下限（N/2）。 唯一剩下的问题是如何分配元素以使 Kolya 相对于 Mitya 的最终异或最大化。 

现在关键的结构见解是子集上的 XOR 仅取决于 GF(2) 上的线性结构。 每个数字都可以被视为一个位向量，XOR 是该空间中的向量加法。 这将问题转化为关于 GF(2) 上向量跨度的推理，其中只有线性独立性很重要。 

我们将问题简化为决定 Kolya 是否可以强制执行严格更大的 XOR、相等的 XOR，或者必须失败，因为 Kolya 有效地选择了 ceil(N/2) 元素，而 Mitya 获得了其余的元素。 这成为向量空间中的奇偶约束选择问题，其中唯一相关的量是整个集合的异或基础以及相对于选择大小存在多少自由度。 

结果取决于所有元素的总异或和线性基的等级是否与 N 的奇偶性以及第一个玩家控制基分量奇偶性的能力相互作用。 将集合简化为 GF(2) 的线性基础后，我们检查第一个玩家是否可以在最终的 XOR 差异中强制执行非零优势。 根据基础等级是否等于 N 或者更小，以及 N 是偶数还是奇数，这会分解为较小的恒定数量的情况。

 ## 算法演练

 我们将每个数字建模为 60 位向量并构建线性 XOR 基础。

1. 从所有数字构建线性基，将每个数字从最高位插入到最低位。 这会将集合压缩为最多 60 个独立向量。 目的是隔离 XOR 空间中的真实自由度，因为相关向量不会影响可实现的 XOR 结果。 
2. 计算基的秩r。 这告诉我们输入的 XOR 空间中存在多少个独立方向。 
3. 设 N 为元素数量。 将 N 与 r 进行比较以了解冗余。 如果 N > r，则 XOR 空间中存在重复项，这意味着某些元素线性依赖于其他元素。 
4. 如果 N - r 至少为 2，则结构变得足够灵活，第二个参与者可以镜像策略，从而迫使结果平衡。 在这种情况下，双方都无法在异或空间中强行取得严格的优势，因此结果会变成平局。 
5. 如果 N - r 等于 1，则只有一个从属元素。 这造成了选择中的单一奇偶性不平衡。 因为 Kolya 首先移动，所以他可以利用这个额外的元素来偏置 XOR 结果，从而获得严格的胜利。 
6. 如果 N 等于 r，则所有元素都是线性无关的。 在这种情况下，每次选择都会删除一个唯一的基础方向。 游戏就相当于在交替回合下选择一个基本元素分配，N的奇偶性决定了异或差的最终控制。 如果 N 是奇数，Kolya 会获得一个额外的元素，这给了他一个更好的异或结果； 如果 N 是偶数，则对称性完美，结果是平局。 
7. 在这种结构中，第二个玩家永远无法强行获胜，因为任何获胜策略都意味着 XOR 空间奇偶校验的受控反转，这在线性独立约束下是不可能的。 

### 为什么它有效

 任何子集的 XOR 仅取决于包含哪些基向量。 该游戏简化为在固定回合顺序下在两个玩家之间分配基础贡献，这固定了子集大小。 唯一的自由是相关向量的分配，这不会改变跨度。 因此，结果完全取决于独立方向的奇偶性以及是否存在相关向量形式的松弛。 线性基完全捕获所有可达的 XOR 值，因此在归约过程中不会丢失任何信息，并且最终的比较归结为关于秩和奇偶校验的有限情况分析。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    B = [0] * 61
    
    def insert(x):
        for i in range(60, -1, -1):
            if (x >> i) & 1:
                if not B[i]:
                    B[i] = x
                    return
                x ^= B[i]
    
    for x in arr:
        insert(x)
    
    rank = sum(1 for x in B if x)
    
    if n - rank >= 2:
        print("Draw")
    elif n - rank == 1:
        print("First")
    else:
        # n == rank
        if n % 2 == 1:
            print("First")
        else:
            print("Draw")

if __name__ == "__main__":
    solve()
```该实现使用从高位到低位的标准贪婪插入构建 60 位 XOR 基础。 每个数字都会根据现有的基本向量进行减少，确保仅保留独立的分量。 

排名计算只是对非零基向量进行计数。 然后决策逻辑遵循之前得出的案例分析。 微妙之处在于区分存在冗余和完全独立的情况，因为单一差异会改变是否存在奇偶优势。 

## 工作示例

 ### 示例 1

 输入：`1 2 3 4`我们构建 XOR 基础：

 | 步骤| 已插入 | 基态（非零向量） | 排名|
 | --- | --- | --- | --- |
 | 1 | 1 | {1} | 1 |
 | 2 | 2 | {1, 2} | 2 |
 | 3 | 3 | {1,2,3}| 2 |
 | 4 | 4 | {1,2,4}| 3 |

 这里rank=3，N=4，所以N-rank=1，属于“单依赖元素”的情况。 

输出是`First`。 

这显示了一个冗余 XOR 关系如何为第一个参与者提供足够的灵活性来偏置最终的 XOR 分割。 

### 示例 2

 输入：`1 1 1`基础建设：

 | 步骤| 已插入 | 基础状态| 排名|
 | --- | --- | --- | --- |
 | 1 | 1 | {1} | 1 |
 | 2 | 1 | {1} | 1 |
 | 3 | 1 | {1} | 1 |

 这里N=3，rank=1，所以N-rank=2，直接导致强制平局。 

这证明了完全冗余的情况，即双方都没有获得结构性优势。

 ## 边缘情况

 一种重要的边缘情况是所有数字都相同。 用于输入`5 5 5 5`，当 N 较大时，XOR 基础的秩为 1，因此 N - 秩较大，算法将其分类为平局。 逐步运行，第一个之后的每个插入都不会执行任何操作，因此基础仍然存在`{5}`最终的决定正确地识别出完全冗余。 

另一种情况是当所有数字都是 2 的幂时，例如`1 2 4 8 16 32`。 这里每个元素都会增加基础等级，给出 N = 等级。 该算法属于奇偶校验分支。 对于偶数 N，结果是平局，因为两个玩家都获得对独立 XOR 方向的对称控制。 

一个微妙的情况是其他独立元素之间的单个重复，例如`1 2 3 4 1`。 重复项将排名精确地减少了 1，产生 N - 排名 = 1。该算法将其置于第一玩家获胜类别中，并且跟踪插入确认仅发生一次基本碰撞，从而创建了 Kolya 可以利用的一个依赖自由度。
