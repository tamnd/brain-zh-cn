 ## 解决方案
 ## 问题理解

 我们给出了单位圆边界上的点。 每个点都由一个角度指定，这些点代表圆内直线切割的可用端点。 

我们必须准确执行$K-1$直线切割，每次切割都是连接两个给定边界点的弦。 这些剪切不是独立的：第一个剪切之后的每个剪切都必须从前一个剪切结束的端点开始，因此所有剪切形成一个连续的连接和弦链。 该链将磁盘准确地划分为$K$地区。 

每个区域部分由圆弧界定，部分由该链的弦界定。 目标是选择切割链，以便最小的结果区域具有最大可能的面积，并且我们必须输出该面积。 

关键的结构性限制是$N \le 30$，因此我们显然不会尝试任意的几何构造或对所有可能的切割进行持续优化。 相反，解决方案必须将问题简化为选择和排序一小部分边界点。 

一个微妙的边缘情况来自于切割必须形成一条链的事实。 天真的解释可能允许独立的和弦，但这是不可能的，因为每个新的剪辑都必须从前一个剪辑的结束处开始。 这迫使所有选定的点都位于围绕圆的单个循环顺序中。 

另一个重要的边缘情况是几何形状的圆形性质。 例如，角度接近$0^\circ$和$359^\circ$在圆上相邻，但在数值上相距较远。 任何线性化角度而不包裹的解决方案在以下情况下都会失败：

 输入：```
3 2
0 180 359
```正确的结构取决于将圆视为连续的而不是线段。 

最后，目标取决于圆弧段的面积。 一个常见的陷阱是尝试从头开始独立计算每个区域的面积。 正确的公式将一切都简化为按循环顺序连续选择的顶点之间的角度间隙。 

## 方法

 暴力策略会尝试选择一个顺序$K$顶点出$N$，然后将它们连接成链并以几何方式计算所有结果区域面积。 多种选择和订购方式$K$点大致是$P(N, K)$，在最坏的情况下$N = 30$是一个天文数字。 即使限制到子集也给出$\binom{30}{15} \approx 1.5 \times 10^8$，并且每个候选者都需要对所有切片进行几何重建，这使得它不可行。 

关键的观察是，一旦我们固定了沿圆访问的边界点的顺序，分区结构就完全由连续点之间的角度间隙决定。 每个切片对应于这些间隙之一。 因此问题归结为选择$K$围绕一个圆的点，使得连续选定点之间的最小循环间隙最大化。 

如果我们将角间隙表示为$\theta$，切片的面积是圆弧段的面积：$$A(\theta) = \frac{1}{2}(\theta - \sin \theta)$$该函数严格递增$\theta \in (0, \pi]$，因此最大化最小面积相当于最大化最小角间隙。 

这就把问题变成了一个经典的圆形间距可行性问题：确定最大的$d$这样我们就可以选择$K$按循环顺序在圆上的点，其中每个连续间隙至少$d$。 一次$d$已知，则直接根据相应的切片面积计算答案。 

我们可以测试候选人的可行性$d$贪婪地。 对于每个起点，我们至少重复选择下一个可用点$d$向前度数，绕一圈，检查我们是否可以选择$K$同时也满足了回到起点的差距。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对有序子集的暴力破解 |$O(N! / (N-K)!)$|$O(K)$| 太慢了 |
 | 二分查找+贪心检查|$O(N^2 \log 360)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 将给定的角度转换为排序列表，并通过将每个角度加上 360 度来扩展它。 这使我们能够模拟线性阵列中的圆形包裹。 此步骤避免了贪婪选择过程中特殊的套管模运算。 
2. 定义一个函数，检查候选最小间隙是否为$d$是可以实现的。 该函数尝试构建一个有效的链$K$点。 
3. 对于每个可能的起始索引$i$，尝试通过重复选择角度至少为先前选择的角度加上的第一个点来贪婪地构建序列$d$。 这种贪婪是正确的，因为选择最早的有效下一点可以为未来的选择留下最大的灵活性。 
4.选择后$K$点，验证循环闭合条件。 从最后选择的点回到起点（通过 +360 环绕）的间隙也必须至少为$d$。 如果没有这种检查，构造将错误地接受不符合循环一致性的序列。 
5. 如果任何起始索引成功，则候选者$d$是可行的。 
6. 二分查找$d$在范围内$[0, 360]$具有足够的精度，反复测试可行性。 
7. 一旦最大可行$d$找到后，使用以弧度为单位的切片面积公式计算答案：$$A = \frac{1}{2}(\theta - \sin \theta), \quad \theta = d \cdot \pi / 180$$### 为什么它有效

 任何有效的解决方案都对应于选择$K$循环顺序的边界点。 链约束迫使分区仅依赖于连续的角度差。 由于每个切片面积随其角度单调增加，因此最大化最小切片面积相当于最大化最小间隙。 贪婪可行性检查正确地捕获是否可以对所有间隙执行统一的下界，并且二分搜索找到最佳的这样的界限。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def can(d, ang, n, k):
    for start in range(n):
        cnt = 1
        last = ang[start]
        i = start
        while cnt < k:
            j = i + 1
            while j < start + n and ang[j] < last + d:
                j += 1
            if j >= start + n:
                break
            last = ang[j]
            i = j
            cnt += 1

        if cnt == k:
            if ang[start] + 360 - last >= d:
                return True
    return False

def main():
    n, k = map(int, input().split())
    a = list(map(float, input().split()))
    a.sort()

    ang = a + [x + 360 for x in a]

    lo, hi = 0.0, 360.0
    for _ in range(60):
        mid = (lo + hi) / 2
        if can(mid, ang, n, k):
            lo = mid
        else:
            hi = mid

    theta = lo * math.pi / 180.0
    ans = 0.5 * (theta - math.sin(theta))
    print("{:.10f}".format(ans))

if __name__ == "__main__":
    main()
```该代码首先构造一个双角数组来自然地处理圆形包裹。 可行性函数会尝试每个可能的起点，因为最佳配置可能从圆上的任何位置开始。 内部贪婪扫描向前推进指针，直到找到符合最小间隙约束的下一个有效点。 

二分搜索使用固定的迭代计数而不是 epsilon 检查，这对于此设置中的浮点比较更稳定。 最后，从角度到弧度的转换是必要的，因为圆弧面积的几何公式​​是以弧度定义的。 

一个常见的微妙错误是忘记了最终的环绕间隙检查，这会错误地接受局部有效但在圆上全局不一致的链。

 输入：```

0 60 270
```我们测试二分搜索期间候选间隙的可行性。 认为$d = 60$。 

| 步骤| 开始| 已选择 | 最后一个角度| 下一个选择 | 有效的？ |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 0 | 0 | 0 | 60| 是的 |
 | 2 | 0 | 0, 60 | 60| 检查包装| 270-60 = 210 | 270-60 = 210

 此配置成功$d = 60$，但较高的值会失败，因为最小的弧将受到最紧密的间距的限制。 二分搜索收敛到最大可行间隙，这决定了最终的区域。 

### 示例 2

 输入：```
4 4
85 90 265 270
```这里我们必须选取所有点，因此所有间隙都由几何形状固定。 

| 步骤| 开始| 已选择 | 差距|
 | --- | --- | --- | --- |
 | 85 | 85 85 → 90 → 265 → 270 | 所有点| 5, 175, 5, 175 |

 最小间隙为$5^\circ$，并且每个切片区域都由该结构确定。 任何增加间距的尝试都会立即失败，因为无法以更大的最小间距形成子集。 

这个例子强调了当$K = N$，解完全由原始循环排序决定。

 ## 边缘情况

 当除了一个大间隙之外的点都聚集在一起时，就会出现一种重要的边缘情况。 例如，用角度$0, 1, 2, 3, 300$，最优选择避免密集区域并倾向于在大间隙周围扩散。 贪婪可行性检查可以正确处理这个问题，因为它总是前进到最早的有效下一个点，为剩余的选择保留最大的灵活性。 

另一个边缘情况是当$K = 2$。 该解决方案简化为选择任意一对点来最大化它们的圆距离。 该算法仍然有效，因为它有效地检查所有起点并在约束下选择最远可到达的第二个点$d$，确保正确的二分搜索收敛。 

最后一个边缘情况是$K = N$，其中没有选择自由。 该算法自然会强制按循环顺序选择所有点，并且最小间隙恰好是连续输入角度（包括环绕）之间的最小距离。
