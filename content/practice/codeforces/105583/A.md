 ## 解决方案
 ## 问题理解

 我们正在建造一个由重复的模块化部件组成的垂直结构。 每个高度单位就是一层，每层消耗固定数量的相同砖块。 层被分组为块，每个块都有一种颜色。 块循环重复颜色，因此在使用所有可用颜色后，我们从颜色一重新开始。 

塔应该有总高度$H$层。 每一层都需要$P$砖块，所以如果我们从头开始建造完整的塔，它将恰好包含$H \cdot P$砖头。 然而，塔的一部分已经建成，我们被告知第一个$N$砖块（从下往上数）已经按照相同的颜色图案正确放置。 任务是确定完成塔楼还需要多少每种颜色的砖块。 

重要的结构是颜色不是按层分配而是按块分配。 每个块恰好包含$K$层，除了最上面的块，其高度可能不完整。 由于每层包含$P$砖块，一个完整的块包含$K \cdot P$砖头。 块的颜色仅取决于其在序列中的位置：块 1 是颜色 1，块 2 是颜色 2，依此类推循环取模$C$。 

挑战是计算从位置到片段中出现的每种颜色的砖块数量$N+1$到$H \cdot P$在这个周期性的块色结构中。 

约束足够大，任何模拟砖块建造的解决方案都可以达到$H \cdot P$是不可能的。 自从$H$最多可达 100,000 个，每层最多包含 10,000 个砖块，总大小可达$10^9$，这排除了线性模拟。 任何可接受的解决方案都必须及时与颜色数量成正比或最多与高度的对数因子成正比。 

两端的部分块会产生微妙的边缘情况。 已经构建的前缀$N$砖块可能会终止于一层的中间，甚至可能终止于块内。 类似地，剩余的后缀可能在部分完成的块内结束。 仅计算完整块或仅计算完整层的简单方法会在这些边界区域中错误计数。 

例如，如果$P = 4$,$K = 2$，那么每块都是2层，8块砖。 如果$N = 7$，那么构建的前缀在第一个块的第二层的中间结束。 任何从下一个完整块开始计算颜色的解决方案都会错误地忽略部分消耗的层，从而改变所有颜色分配。 

## 方法

 暴力策略将逐层重建整个塔，逐块分配颜色，并为每块砖增加计数器。 从概念上讲，这是可行的，因为结构是确定性的：一旦我们知道了砖块的位置，我们就可以得出它的块索引，从而得出它的颜色。 然而，这需要迭代所有$H \cdot P$砖块，在最坏的情况下是周围$10^9$操作，远远超出时间限制。 

关键的观察是我们实际上不需要明确地建造塔。 每个砖块位置直接映射到一个块索引，并且块重复每个$C$。 因此，我们可以通过在块范围内工作并处理部分边界来计算给定间隔内有多少每种颜色的砖块，而不是迭代砖块。 

从块级别来看，结构变得更加简单。 每个块都有固定的大小$S = K \cdot P$除了可能的最后一个块之外，每个连续的砖块都可以分解为最多两个部分块以及中间的几个完整块。 完整的块完全贡献一种颜色，而部分块则贡献一系列颜色相同的砖块。 

因此问题就简化为：

 首先确定每个块属于哪种颜色，然后计算每个颜色的后缀中有多少个完整块和部分块$[N+1, H \cdot P]$。 

这将问题从总砖块的线性减少为每种颜色的恒定工作加上一些算术运算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(H·P) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(C)| O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算整个塔中的砖块总数为$T = H \cdot P$。 该任务简化为计算区间内的颜色$[N+1, T]$。 
2.定义一个辅助结构：每个块都有大小$S = K \cdot P$。 砖块位置的块索引$x$是$(x - 1) // S$，将其颜色确定为$(block\_index \bmod C) + 1$。 
3.分割区间$[N+1, T]$分为三部分：直到下一个块边界的前缀、中间的完整块以及最后一个完整块边界之后的后缀。 这种分解将不规则的部分块与均匀的片段隔离开来。 
4.确定第一个位置$L = N+1$并计算它距块的开头有多远。 余数确定该块中已经消耗了多少块砖，以及部分起始块中剩余多少块。 
5. 如果$L$未与块边界对齐，通过计算直到该块末尾剩余多少块来处理部分起始块，并将所有块分配给其相应的颜色。 
6. 在起始部分块之后，移动到下一个块边界。 计算在到达结束位置之前有多少个完整块完全适合该间隔$T$。 
7. 对于完整块，不要逐块处理，而是使用索引算术计算该范围内出现的每种颜色的块数。 由于颜色重复$C$块，每个完整的循环对所有颜色的贡献相等。 
8. 最后，处理结束部分块 if$T$位于一个块内。 计算最后一个方块中有多少块砖落入该区间，并将它们分配给正确的颜色。 

### 为什么它有效

 正确性依赖于这样一个事实：每块砖都唯一属于一个块，并且每个块都有单一的固定颜色。 一旦间隔被分解为最多两个部分块和一系列完整块，每个块都会独立地对最终计数做出贡献。 基于模数的着色确保块颜色以固定周期重复$C$，因此计算完整周期会均匀分布，而无需跟踪各个块。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    H, P, C, K, N = map(int, input().split())

    total = H * P
    block_size = K * P

    # result per color (1-indexed)
    ans = [0] * C

    def add_block(block_idx, cnt):
        color = block_idx % C
        ans[color] += cnt

    # process range [L, R]
    L, R = N + 1, total

    if L > R:
        print(*ans)
        return

    # starting partial block
    start_block = (L - 1) // block_size
    start_block_start = start_block * block_size
    start_block_end = start_block_start + block_size - 1

    if L <= start_block_end:
        take = min(R, start_block_end) - L + 1
        add_block(start_block, take)
        L += take

    # full blocks
    if L <= R:
        first_full_block = (L - 1) // block_size
        last_full_block = (R - 1) // block_size

        if first_full_block <= last_full_block:
            for b in range(first_full_block, last_full_block + 1):
                bs = b * block_size
                be = bs + block_size - 1

                if bs < L:
                    left = L - bs
                else:
                    left = 0

                if be > R:
                    right = R - bs + 1
                else:
                    right = block_size

                add_block(b, max(0, right - left))

    print(*ans)

if __name__ == "__main__":
    solve()
```实现直接遵循区间分解。 关键部分是通过整数除以计算块索引`block_size`。 每个块仅贡献一种颜色，由下式确定`block_idx % C`，所以我们累积每个颜色数组的计数。 

边界算术必须小心。 首先处理起始部分块，以便后续计算对块对齐索引进行操作。 如果没有这种调整，余数逻辑将重复计数或使范围错位。 块上的循环在约束下是安全的，因为块计数最多是$H / K$，这是可以管理的$H \le 10^5$，尽管在完全优化的版本中，它可以通过周期计数进一步减少。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(C + 范围内的块数) | 每种颜色都通过块分组和部分处理进行更新 |
 | 空间| O(C)| 我们为每种颜色存放一个柜台|

 该算法很容易满足限制，因为$C \le H \le 10^5$，块级迭代受块段数量而不是单个砖块的数量限制。 内存占用仅与颜色数量呈线性关系。

 ## 边缘情况

 当剪切位置恰好位于块边界时，就会发生临界边缘情况。 例如，如果$K = 2$,$P = 3$，则块大小为 6。如果$N = 6$，前缀恰好在第一个块的末尾结束，因此不应发生部分处理。 

输入：```
H = 3, P = 3, C = 2, K = 2, N = 6
```这里$T = 9$，我们从 7 数到 9。由于 7 在块 2 内，因此我们直接从部分块开始，但没有块 1 的剩余部分。 

该算法计算：

 - 起始块= 1
 - 开始块开始= 6
 - 开始块结束= 11

 因此，它正确地识别出没有保留部分前缀，并且仅处理块 2 内的后缀。生成的分布与直接手动计数相匹配，确认边界相等不会产生相差一错误。 

另一个边缘情况是当$N = T - 1$。 那么只剩下一块砖了。 该算法将间隔缩小到单个位置，并且只有相应的块为其颜色贡献恰好一个单位。
