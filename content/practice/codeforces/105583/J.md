 ## 解决方案
 ## 问题理解

 我们被要求构造一个固定长度的字符串$N$，使用小写英文字母，使其“包含”尽可能多的不同给定模式单词。 如果模式在构造的字符串中的任何位置出现为连续子字符串，则认为该模式存在。 每个模式最多贡献一分，即使它出现多次。 

所以任务不是计算出现次数，而是选择单个长度 -$N$最大化多少个的字符串$K$给定的简短单词出现在其中的某处。 

结构上的约束非常小：$N \le 14$,$K \le 10$，并且每个模式的长度最多为 7。这立即表明答案空间呈指数分布$N$，但对于位掩码动态编程来说仍然足够小。 超过 26 个字母的长度为 14 的字符串是一个天文数字，因此不可能对整个字符串进行任何直接搜索。 然而，关键的观察结果是，只有长度最多为 7 的子串之间的交互才重要，因此我们不需要考虑最近字符的滑动窗口之外的任何内容。 

天真的方法会尝试所有$26^N$字符串并检查每个字符串中出现哪些模式。 这已经是不可能的事了$N=14$， 自从$26^{14}$远远超出任何可行的计算。 

第二个天真的想法是逐步构建字符串并贪婪地选择最大化直接模式匹配的字符。 这会失败，因为模式可能只有在选择了几个未来字符后才变得可匹配，并且本地决策可能会永久阻止它。 

当重叠模式竞争时，会出现更微妙的失败情况。 例如，选择尽早完成一种模式的角色可能会阻止以后形成更长的重叠模式，即使两者都对分数有所贡献。 

因此，正确的解决方案必须跟踪有关哪些模式将来可能完全匹配以及哪些模式已经匹配的部分信息。 

## 方法

 这里的中心结构是，我们正在构建一个短字符串，并关心其中是否出现小的禁止模式。 由于所有模式的长度最多为 7，因此当我们追加一个字符时，只有最后最多 7 个字符可以贡献新的匹配项。 

这表明动态编程状态会记住构造字符串的短后缀，而不是整个字符串。 

蛮力观点是考虑所有长度的字符串$N$。 对于每个字符串，我们扫描所有$K$模式并检查每个是否显示为子字符串，成本大致$O(N \cdot K \cdot 7)$。 这对于每个字符串来说已经是可以管理的，但是字符串的数量使得它不可行。 

关键的改进是从左到右构建字符串，并仅保留评估未来模式匹配所需的内容。 在任何时候，关于前缀唯一重要的是：

 最后最多 7 个字符，以及哪些模式已经见过。 

这将问题简化为对由后缀字符串和匹配模式的位掩码组成的状态进行图遍历。 

每一步都通过附加一个字符、更新后缀（仅保留最后 7 个字符）以及更新新后缀窗口中出现的模式来进行转换。 

自从$N \le 14$，DP步数很少，并且由于后缀长度最多为7，因此状态空间的边界为$26^7 \cdot 2^{10}$，但实际上只有可达状态才重要。 

我们对步骤和后缀使用动态编程，存储每个状态可实现的最佳掩码。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解字符串 |$O(26^N \cdot N \cdot K)$|$O(1)$| 太慢了 |
 | DP 后缀 + 掩码 |$O(N \cdot 26^7 \cdot K)$（可到达的状态少得多） |$O(26^7 \cdot K)$| 已接受 |

 ## 算法演练

 我们通过构建的字符串中的当前位置、字符串的最后最多 7 个字符以及指示哪些模式已经出现的位掩码来表示每个 DP 状态。 

我们还预先计算每个可能的小字符串后缀包含哪些模式，因为重复检查模式匹配的成本会很高。 

### 步骤

 1. 初始化一个 DP 表，其中初始状态为空字符串，没有匹配的模式。 后缀为空且位掩码为零。 这代表选择任何字符之前的开始。 
2.对于从0到的每个位置$N-1$，我们考虑通过将一个字符从 'a' 附加到 'z' 来扩展所有可达状态。 这对应于增量构建字符串。 
3. 附加字符时，我们通过连接它并修剪到最后 7 个字符来更新后缀。 这已经足够了，因为没有模式超过 7，因此任何较早的模式都无法形成新的匹配。 
4. 对于新后缀，我们确定哪些模式在其中显示为子字符串。 我们将这些模式或到当前位掩码中，因为一旦模式出现在任何地方，它就会永远被计数。 
5. 我们为每对（位置、后缀）存储可实现的最佳位掩码。 如果两种方式达到相同的后缀，我们只保留具有更好掩码的一种。 
6.全部处理完毕后$N$位置，我们扫描所有最终状态并选择最大位掩码值，即与最多模式匹配的状态。 
7. 我们通过在转换期间存储父指针来重建导致该状态的任何有效字符串。 

### 为什么它有效

 正确性依赖于以下事实：任何模式的出现完全由长度最多为 7 且结束于其完成位置的窗口决定。 因此，一旦我们维护了最后 7 个字符，我们就不会丢失检测未来匹配项所需的信息。 DP 不依赖于完整的历史，仅依赖于这个有界后缀。 由于每次转换都会保留后缀和累积的模式集，因此未来的扩展无法改变过去的正确性，以及深度上的最佳掩模$N$完全对应于最佳结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K = map(int, input().split())
    patterns = [input().strip() for _ in range(K)]
    
    # Precompute pattern matching in a string
    def calc_mask(s):
        mask = 0
        for i, p in enumerate(patterns):
            if p in s:
                mask |= (1 << i)
        return mask
    
    from collections import defaultdict
    
    dp = [defaultdict(lambda: -1) for _ in range(N + 1)]
    parent = [{} for _ in range(N + 1)]
    
    dp[0][""] = 0
    
    for i in range(N):
        for suf, mask in dp[i].items():
            for c in "abcdefghijklmnopqrstuvwxyz":
                ns = (suf + c)[-7:]
                nmask = mask | calc_mask(ns)
                
                if dp[i + 1][ns] < nmask:
                    dp[i + 1][ns] = nmask
                    parent[i + 1][ns] = (suf, c)
    
    best_mask = -1
    best_state = None
    
    for suf, mask in dp[N].items():
        if mask > best_mask:
            best_mask = mask
            best_state = suf
    
    # reconstruct
    res = []
    cur = best_state
    for i in range(N, 0, -1):
        prev, ch = parent[i][cur]
        res.append(ch)
        cur = prev
    
    print("".join(reversed(res)))

if __name__ == "__main__":
    solve()
```DP表`dp[i]`详细映射每个可能的后缀`i`匹配模式的最佳位掩码。 转换步骤会迭代所有字母，追加一个字符，并仅保留后缀的最后 7 个字符。 帮手`calc_mask`检查新后缀中出现哪些模式并更新掩码。 

这`parent`结构体存储如何达到每个状态，以便我们可以在完成 DP 后重建实际的字符串。 这是必要的，因为问题需要字符串本身，而不仅仅是最大值。 

一个微妙的点是我们只比较相同长度的相同后缀的掩码。 这可以防止丢失到达具有不同模式集的相同后缀的最佳路径。

 ## 边缘情况

 一种边缘情况是所有模式严重重叠，例如重复的前缀。 DP 正确地累积了多个匹配，因为一旦某个模式出现在最后 7 个字符内，它就会永久记录在掩码中并且不会丢失。 

另一种边缘情况是模式仅出现在一长串字符之后。 由于后缀始终被修剪为长度 7，因此任何超过该长度的模式都是不可能的，但约束明确阻止了这种情况。 因此，任何有效模式都不会被错误地“切断”。 

最后的边缘情况是多个不同的构造路径导致相同的后缀。 DP 通过仅保留最大掩码来合并它们，这是安全的，因为任何未来的扩展仅取决于后缀和掩码，而不取决于后缀是如何到达的。
