 ## 解决方案
 ## 问题理解

 我们得到一个有 n 个顶点和一些现有无向边的图。 除此之外，我们还得到了顶点的排列，它表示 DFS 从该排列中的第一个顶点开始的确切顺序，最初发现的节点。 

任务是添加最少数量的新边，以便存在一种方法来运行标准递归 DFS（我们按某种顺序迭代邻居）并准确地重现此发现顺序。 

重要的解释转变是我们并不是试图“任意地按此顺序访问节点”。 我们试图确保 DFS 没有选择点会迫使它偏离序列中所需的下一个顶点。 换句话说，当我们位于 DFS 堆栈中的一个节点时，排列中的下一个未访问节点必须能够以 DFS 可以合法选择下一个节点的方式进行访问，而不必被迫先探索其他节点。 

约束大小 n、m 最大为 10^5 意味着任何解决方案都必须接近线性或 O(n log n)。 任何关于边或顶点的二次方都会立即失败，因为 m 已经足够密集，使得 O(n^2) 不可行。 

当现有图已经“几乎”支持 DFS 顺序但由于排列的连续段之间缺少连接而失败时，就会出现微妙的边缘情况。 例如，如果排列为 1, 2, 3, 4，但边仅连接 (1, 3) 和 (3, 4)，则即使图是连通的，在不破坏所需顺序的情况下，从 1 开始的 DFS 也无法达到 2。 这表明仅连接性是不够的，排序约束占主导地位。 

当给定边中存在多个组件但排列交错来自不同组件的节点时，会出现另一种边缘情况。 在这种情况下，我们被迫添加边缘以将组件缝合到精确的 DFS 一致结构中。 

## 方法

 一个蛮力的想法是通过尝试显式强制排列来模拟 DFS 顺序构造。 可以想象，对于排列中的每个位置 i，检查当前图中是否存在从当前 DFS 边界到下一个节点的不违反 DFS 约束的路径。 如果没有，我们添加一条边并继续。 

然而，直接模拟 DFS 一致性的成本很高。 每次检查可能需要图遍历，如果对所有 n 个位置重复进行，最坏的情况将变为 O(n^2) 或更糟。 当 n 达到 10^5 时，这是不可行的。 

关键的观察结果是，DFS 并不直接关心全局结构，它只关心通过类似堆栈的进程的可达性。 如果我们按照给定的顺序处理节点，那么我们被迫向图“添加能量”的唯一时间是当排列中的下一个节点无法从现有结构下的当前 DFS 活动组件到达时。 

这导致了一个建设性的解释：我们维护一个代表当前 DFS 可到达的“边界”的结构。 每当排列中的下一个节点未使用现有边连接到该边界时，我们必须添加一条边来连接它，从而沿着排列有效地缝合 DFS 树。 

更深入的见解是，最优结构本质上是一棵 DFS 树，其预序正是给定的排列，因此我们正在重建与该预序一致的树，并计算有多少邻接约束已经与其对齐。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每步强力 DFS 模拟 | O(n²) | O(n + m) | 太慢了|
 | 构造性排列拼接| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

1. 将给定的排列解释为 DFS 树所需的前序。 排列中的第一个节点是该树的根。 这就确定了我们建设的起点。 
2. 在现有边缘上维护联合结构或连接跟踪机制。 目的是了解当前图中是否已经连接了排列的连续“段”。 
3. 从左到右扫描排列，将每个位置视为 DFS 必须发现的下一个节点。 
4. 对于每个下一个节点，检查是否可以使用现有边从当前 DFS 活动区域到达该节点。 如果可达，我们不需要添加任何东西，我们在概念上继续 DFS。 
5. 如果它不可达，我们必须添加一条边将其连接到当前活动 DFS 区域中的某个节点。 贪婪的选择是将其连接到排列中的前一个节点，因为这可以保持 DFS 一致性，而不会引入不必要的分支。 
6. 将每个此类强制连接计为一个添加的边。 
7. 继续，直到处理完所有节点，确保构建的图承认完全遵循排列的 DFS。 

### 为什么它有效

 核心不变量是，在处理排列的前缀 i 后，该前缀中的所有节点都位于单个 DFS 一致的可达结构中，这意味着存在与排列前缀一致的 DFS 树。 每当我们检测到从该结构无法到达下一个节点时，这意味着 DFS 排序不能在不添加新边的情况下将其包含在下一个节点中，因为 DFS 无法在断开连接的组件或无法到达的区域之间“传送”。 因此，每条添加的边都是由当前图中的结构不可能性所强制的，并且添加它最低限度地恢复了 DFS 的可行性，而不会影响早期的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n+1))
        self.r = [0]*(n+1)

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1

def solve():
    n, m = map(int, input().split())
    perm = list(map(int, input().split()))
    
    dsu = DSU(n)
    
    for _ in range(m):
        u, v = map(int, input().split())
        dsu.union(u, v)

    ans = 0

    for i in range(1, n):
        if dsu.find(perm[i]) != dsu.find(perm[i-1]):
            ans += 1
            dsu.union(perm[i], perm[i-1])

    print(ans)

if __name__ == "__main__":
    solve()
```DSU 用于跟踪当前图的连通性。 每个现有的边都会合并组件，然后我们按顺序扫描排列。 每当两个连续的顶点位于不同的组件中时，我们就必须添加一条边来连接它们，因为 DFS 无法在没有边的情况下在断开的组件之间移动。 

微妙之处在于我们只关心排列中连续元素之间的连接性。 如果它们已经连接，则可以安排 DFS 遵守该顺序； 如果没有，我们就被迫引入一座桥。

 ## 边缘情况

 一种重要的边缘情况是当图已经连接但排列在遥远的节点之间跳跃时。 在这种情况下，不需要边，因为可以命令 DFS 遵循排列而无需结构更改。 DSU 通过显示所有连续节点都在同一组件中（产生零操作）来确认这一点。 

另一种边缘情况是完全断开的图，其排列在组件之间交替。 该算法在组件之间的每个交换机上正确插入一条边，从而有效地构建了一条强制 DFS 连续性的链。 

第三种边缘情况是 n = 1，其中不存在边缘并且不需要任何操作。 循环永远不会运行，并且答案仍然为零，与 DFS 的简单情况匹配。
