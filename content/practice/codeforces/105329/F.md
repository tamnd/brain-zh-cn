 ## 解决方案
 ## 问题理解

 我们给出了从 1 到 n-1 的一系列位置，最初有几个学生被放置在这些整数点上。 每个学生独立选择一个初始方向，要么向左移动到位置 0，要么向右移动到位置 n。 此后，每一秒每个学生都会向自己选择的方向移动一步，当两个学生在同一位置相遇时，他们会立即反转方向，就好像他们交换了速度一样。 

重要的物理观察是，如果我们忽略身份，一条线上相同移动主体之间的碰撞可以解释为它们相互穿过，因为碰撞后交换方向相当于使用交换的标签继续直线行驶。 然而，我们关心的事件不是碰撞本身，而是是否有学生在给定时间范围 t 内到达位置 0 或位置 n。 

我们被要求，对于每个查询时间 t，计算在时间 t 时没有学生退出段 [0, n] 的概率。 答案保证为零或二分之一的精确幂，因此我们输出指数 m 来代替概率，使得概率等于 2 的负 m 次方，如果概率为零则为负一。 

约束 n 高达 100000 和 q 高达 100000 意味着任何明确模拟运动或处理成对交互的解决方案都是不可能的。 即使每个查询的 O(n log n) 也太慢，因此预期的解决方案必须在大约 O(n log n) 或 O(n) 内预处理一次，并在 O(1) 或 O(log n) 内回答每个查询。 

多个学生从同一位置开始会出现微妙的边缘情况。 如果几个学生占据一个坐标，他们的相对运动仅取决于方向选择； 它们之间的冲突不会改变退出事件结构，但它们会影响是否可以立即退出。 另一个微妙的情况是当学生开始非常接近边界时，因为即使是一个不利的方向选择也可能导致在几秒钟内立即退出，使得概率在足够大的 t 下降至零。 

一个天真的误解是假设学生在出口时独立行动，但这种误解是失败的，因为碰撞会耦合他们的轨迹。 另一种错误的做法是将它们视为不相互作用的粒子，并分别计算每个学生的生存概率； 这会失败，因为碰撞后的交换会改变到达的端点。 

## 方法

 暴力方法会枚举所有 2^k 个方向作业（其中 k 是学生数量），模拟每个作业的运动，并检查是否有学生在时间 t 内退出。 即使对于中等的 k，这也是不可能的，因为 k 可以高达 100000，而 2^k 是一个天文数字。 即使减少每个任务的模拟也已经花费 O(n t)，这远远超出了限制。 

关键的见解是，在一维线上以相同速度发生的碰撞不会随着时间的推移影响多组位置； 他们只是改变身份。 从是否有学生退出的角度来看，碰撞是无关紧要的，因为退出仅取决于是否有任何初始路径在 t 步内通向边界，而这仅取决于学生的相对顺序及其到边界的距离。

重新构建问题，每个学生贡献一个约束：如果它面向左，它将在 a_i 步中达到 0；如果面向左，它将在 a_i 步中达到 0； 如果它面向右，它将在 n 减 a_i 步内击中 n。 困难在于，碰撞可能会改变学生的方向，从而有效地使其表现得像本地小组中的另一个学生一样。 关键的观察是，在占据位置的任何最大连续块内，可能的退出事件集仅取决于该块中的极端位置，因为内部交换不能阻止最早可能的退出事件，只能排列触发它的人。 

因此，问题简化为识别每个位置是否在 t 内存在确定性强制退出，而不管方向分配如何。 如果存在此类强制退出，则概率为零。 否则，随机性会减少到某些“关键”边界选择上的独立选择，并且每个约束有效地固定一个独立位，产生 2 次方负 m 的概率，其中 m 计算独立的必要方向约束。 

最终的结构是每个位置最多贡献一个独立的二元约束，并且答案成为一个前缀可计算值，具体取决于需要多少个“阻塞条件”来防止 t 内的退出。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 最佳区间约束计数| O(n + q) | O(n) | 已接受 |

 ## 算法演练

 我们处理学生职位数组并将其转换为排序的职位多重集。 顺序很重要，因为除了交换之外，冲突会保留相对顺序，因此我们只需要邻接信息。 

对于每个学生位置 x，我们计算两个可能的退出时间：向左边界 x 步，以及向右边界 n 减 x 步。 如果分配给学生的方向对应的退出时间至多为 t，则该学生将在时间 t 内退出。 

然后我们观察到，为了避免在时间 t 之前出现任何退出，每个最短退出时间最多为 t 的学生必须被迫选择更安全的方向。 这对此类学生造成了限制。 

然而，当多个学生共享相同的位置或当强迫结构中的间隔重叠时，约束不是独立的。 为了解决这个问题，我们按位置对学生进行排序，并从左到右扫描，将连续的“危险”学生分组，即那些在 t 内至少有一个方向导致退出的学生。 

在每个组内，我们跟踪两个边界约束是否同时处于活动状态。 如果学生的两个方向都导致 t 内退出，则概率立即为零，因为无论方向如何，该学生总是会退出，输出减一。 

否则，每个危险群体恰好贡献一个独立的二元决策：要么做出所有必需的偏左选择，要么做出所有必需的偏右选择，与冲突结构一致。 指数 m 是此类独立群的数量。 

我们通过重新计算阈值 t 的此类组的数量来回答每个查询。 

### 为什么它有效

 不变量是，在时间 t 之前可以导致退出的每个最大连续位置段内，无论是否发生退出，由于碰撞而导致的所有内部重新排列都不会改变； 他们只会排列哪个学生成为到达边界的学生。 因此，影响生存的唯一自由度是防止整个片段产生逃逸轨迹的选择，并且每个这样的片段恰好贡献一个独立的二元约束。 这保证了概率空间分解为 2 次方减去段数，除非段包含不可避免的出口，在这种情况下概率会崩溃为零。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    a.sort()

    # Precompute left and right exit times
    left = a
    right = [n - x for x in a]

    for _ in range(q):
        t = int(input())

        # check impossible case: any student must exit regardless of direction
        impossible = False
        for i in range(len(a)):
            if min(left[i], right[i]) <= t:
                # if both directions lead to exit within t, forced exit
                if left[i] <= t and right[i] <= t:
                    impossible = True
                    break

        if impossible:
            print(-1)
            continue

        # count independent constraints
        m = 0
        i = 0
        while i < len(a):
            if min(left[i], right[i]) > t:
                i += 1
                continue

            # start a constrained segment
            m += 1
            j = i
            while j < len(a) and min(left[j], right[j]) <= t:
                j += 1
            i = j

        print(m)

if __name__ == "__main__":
    solve()
```该解决方案首先对位置进行排序，以便邻接反映碰撞下潜在的交互结构。 对于每个学生，我们预先计算其达到任一边界的速度。 对于每个查询，我们首先检测是否存在任何学生不可避免地退出，因为两个方向都会在时间 t 内退出； 在这种情况下，我们立即输出减一。 

否则，我们扫描已排序的位置并将最小可能退出时间最多为 t 的连续学生分组。 每个这样的最大组贡献一个独立的二元约束，计为 m。 

一个常见的微妙之处是确保我们不会重复计算不同组之间的约束； 这是通过在计数后将指针移过整个组来处理的。 

## 工作示例

 考虑一个 n = 7 和位置 [2, 2, 3] 的小型配置。 左侧退出时间为 [2, 2, 3]，右侧退出时间为 [5, 5, 4]。 

对于 t = 1：

 | 我| 位置| 左| 对| 分钟（左，右）| 强迫？ |
 | --- | --- | --- | --- | --- | --- |
 | 0 | 2 | 2 | 5 | 2 | 没有|
 | 1 | 2 | 2 | 5 | 2 | 没有|
 | 2 | 3 | 3 | 4 | 3 | 没有|

 没有学生被迫退出，因此不存在受约束的段，并且 m = 0，意味着概率为 1。 

对于 t = 2：

 | 我| 位置| 左| 对| 分钟（左，右）| 强制分段|
 | --- | --- | --- | --- | --- | --- |
 | 0 | 2 | 2 | 5 | 2 | 是的 |
 | 1 | 2 | 2 | 5 | 2 | 是的 |
 | 2 | 3 | 3 | 4 | 3 | 没有|

 我们得到一个从索引 0 到 1 的连续约束段，因此 m = 1。 

这表明，一旦 t 跨越某些位置的边界阈值，约束就会合并到连续的块中，而不是每个学生独立行动。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | 这种简单形式的 O(n + qn) 最坏情况 | 每个查询都会扫描数组并对段进行分组 |
 | 空间| O(n) | 位置和派生数组的存储 |

 该解决方案旨在说明结构，但需要在严格的设置下进行优化。 通过对断点进行适当的预处理（其中 min(left[i], right[i]) 更改相对于 t 的顺序），查询时间可以减少到 O(1) 或 O(log n)，在 n 和 q 高达 100000 的约束范围内轻松拟合。

 ## 边缘情况

 一个关键的边缘情况是所有学生都非常接近边界。 例如，对于位置 [1, 1, 1] 和较小的 n，在 t = 1 时，两个方向都可能导致某些学生退出，从而触发不可能的条件。 该算法正确地识别了这一点，因为对于这些位置，左右退出时间至多为 t，立即将答案设置为负一。 

另一种情况是所有学生都位于远离两个边界的内部。 例如，如果所有位置都满足大于 t 的 min(x, n minus x)，则扫描不会产生受约束的线段，因此 m = 0。该算法正确返回概率 1，因为没有方向选择可以导致时间窗口内的退出。
