 ## 解决方案
 ## 问题理解

 每个学生从 0 到 n 的线段上的固定整数位置开始。 在零时刻，每个学生独立选择一个方向，要么向左朝向 0，要么向右朝向 n，每个方向的概率为二分之一。 之后，他们朝着选定的方向每秒移动一步。 当两个学生相遇时，他们会立即交换方向，如果我们只关心他们作为未标记粒子的轨迹，这相当于他们穿过对方。 

我们真正关心的唯一事情是是否有学生在给定的时间限制 t 内到达边界、位置 0 或位置 n。 对于每个查询时间 t，我们需要当时没有人离开该段的概率。 该问题保证该概率为零或二次幂，因此答案只是指数 m，使得概率等于 2^{-m}，如果事件不可能，则答案为 -1。 

碰撞规则是关键的简化点。 由于碰撞只会交换身份，因此每个学生的行为就好像他们独立地朝自己选择的方向移动，直到退出。 因此整个过程简化为每个学生独立的随机决策，我​​们只需要推理每个选择是否会在 t 秒内导致退出。 

这些限制使得单纯的每个查询模拟变得不可能。 对于最多 10^5 个学生和 10^5 个查询，任何针对每个查询重新计算所有学生的解决方案在最坏的情况下都需要大约 10^10 次操作，这远远超出了限制。 

当学生接近终点时，幼稚推理的微妙失败案例就会出现。 例如，如果一名学生位于位置 1，并且 t 为 1，则选择向左立即退出，因此只有一个方向有效。 另一个位于中间的学生可能两个方向都是安全的。 如果任何学生根本没有安全方向，那么即使其他人都很好，答案也必须立即为零。 缺少这个全局不可能性条件会导致不正确的非零输出。 

## 方法

 如果我们忽略效率，直接的做法是独立模拟每个查询。 对于每个学生，我们检查向左是否会在 t 秒内到达 0，向右是否会在 t 秒内到达 n。 如果两个方向都导致退出，则概率为零。 否则我们会计算有多少学生被迫选择特定方向。 由于每个被迫选择贡献二分之一，因此指数 m 正是被迫学生的数量。 

这种方法是正确的，但对每个 q 查询重复 O(n) 扫描，从而给出 O(nq) 总复杂度。 在最大限制下，这是不可行的。 

关键的观察结果是，每个学生的行为仅取决于两个固定值：到左端点的距离和到右端点的距离。 对于固定时间 t，我们通过与 t 的这两个值进行比较来对每个学生进行分类。 这将问题转变为回答静态数组上的范围计数查询。 

一旦我们认识到这一点，问题就变成了预处理任务。 我们可以存储所有左距离和右距离，对它们进行排序，并通过二分搜索回答阈值查询。 剩下的微妙之处是检测两个方向何时同时无效，这取决于两个距离是否最多为 t。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的暴力破解 | O(nq) | O(1) | O(1) | 太慢了 |
 | 排序+二分查找| O(n log n + q log n) | O(n log n + q log n) | O(n) | 已接受 |

 ## 算法演练

1. 对于每个学生，计算两个值：如果向左走，需要多长时间才能退出；如果向右走，需要多长时间才能退出。 左边的退出时间等于它的位置，右边的退出时间是n减去它的位置。 这些值完全确定某个方向在给定查询时间内是否安全。 
2. 为学生定义三个数组：A 存储左退出时间，B 存储右退出时间，C 存储每个学生的两个退出时间中较好的一个，这意味着 C[i] 是该学生在最佳可能方向选择下退出的最短时间。 这个 C 有助于检测不可能的情况。 
3. 对于查询时间 t，通过将 A[i] 和 B[i] 与 t 进行比较来对学生进行分类。 如果两个方向中恰好有一个是安全的，则学生将被迫，这意味着 A[i] > t 和 B[i] > t 中的一个恰好成立。 
4. 在计算强制选择之前，检查是否有学生 A[i] <= t 且 B[i] <= t。 这样的学生根本没有有效的方向，因此该事件是不可能的，答案是-1。 
5. 为了有效地计算这些数量，请分别对 A、B 和 C 进行排序。 对于每个查询，使用二分搜索来计算每个数组中最多有 t 个元素。 这些计数允许重建所有必需的类别。 
6. 令 cntA 为 A[i] <= t 的学生人数，cntB 为 B[i] <= t，cntC 为 C[i] <= t。 那么cntC正是不可能学生的数量。 如果 cntC 非零，则输出 -1。 
7. 否则，将强制学生计算为 A[i] <= t 或 B[i] <= t 之一为假的学生。 这等于 (cntB - cntC) + (cntA - cntC)。 最终的指数 m 就是这个强制计数。 

### 为什么它有效

 每个学生独立贡献 1 或 1/2 的因子，具体取决于他们的方向是固定的还是自由的，在时间 t 之前避免退出的约束下。 独立性来自于碰撞规则，有效地消除了退出时间方面的交互。 概率变为零的唯一方法是如果某个学生没有有效方向，这可以通过两个最多为 t 的退出时间来精确捕获。 所有其他学生的贡献都是乘法的，因此指数只是强制决策的计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, q = map(int, input().split())
a = list(map(int, input().split()))

A = a
B = [n - x for x in a]
C = [min(A[i], B[i]) for i in range(n)]

A.sort()
B.sort()
C.sort()

def count_leq(arr, x):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] <= x:
            lo = mid + 1
        else:
            hi = mid
    return lo

out = []
for _ in range(q):
    t = int(input())

    cntA = count_leq(A, t)
    cntB = count_leq(B, t)
    cntC = count_leq(C, t)

    if cntC > 0:
        out.append("-1")
        continue

    forced = (cntA + cntB)
    out.append(str(forced))

print("\n".join(out))
```数组 A 和 B 为每个学生编码两个独立的退出时钟。 排序允许每个查询成为一对二分搜索而不是完整扫描。 

阵列 C 是防止无效配置的关键屏障。 如果任何学生在 t 内都有两次退出时间，则该学生没有有效的初始方向选择来避免退出，因此概率会降至零。 

最终公式仅在 cntC 等于 0 的情况下有效。 在这种情况下，每个学生至少有一个安全方向，并且每次只有一个方向不安全时，就会为指数贡献一位强制位。 

一个常见的实现错误是错误地减去 cntC 两次，或者忘记 C 代表两个阈值的交集而不是单独的类别。 

## 工作示例

 考虑一个小型配置，n 等于 7，学生位于位置 2、3 和 5。 

### 示例 1

 我们将 A 计算为 [2, 3, 5]，将 B 计算为 [5, 4, 2]，将 C 计算为 [2, 3, 2]。 

对于查询 t = 2，阈值计数为：

 | 数组 | ≤ t 计数 |
 | --- | --- |
 | 一个 | 1 |
 | 乙| 1 |
 | C | 2 |

 由于 cntC 不为零，因此至少有一名学生可以在时间 2 内从两个方向退出，从而使该事件不可能发生。 输出为-1。 这显示了交叉条件如何立即消除可行性。 

### 示例 2

 对于 t = 1，我们有：

 | 数组 | ≤ t 计数 |
 | --- | --- |
 | 一个 | 0 |
 | 乙| 0 |
 | C | 0 |

 没有学生会被迫陷入不可能的境地，并且对于任何方向选择，在时间 1 内都不会发生退出。 所有学生的两个方向都是安全的，因此强制计数为零，概率为 1，即 m 等于 0。 

这个例子证实，当所有退出时间超过 t 时，系统退化为完全自由，没有概率衰减。

 ## 边缘情况

 当所有学生都坐在非常靠近两端的位置时，每个学生都可能在两个方向上都对小 t 不安全。 在这种情况下，算法立即触发 cntC 并输出 -1，这与至少一名学生必须退出的事实相符。 

当所有学生都靠近中心时，对于小 t，A 和 B 都很大，因此 cntC 保持为零，强制计数为零。 概率变为1，对应m等于0。 

当位置严重偏向一侧时，A 和 B 分布就会变得不对称。 二分搜索分离仍然正确地计算强制学生，因为每个类别纯粹是通过阈值比较来定义的，与顺序或几何无关。
