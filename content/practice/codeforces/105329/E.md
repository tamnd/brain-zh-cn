 ## 解决方案
 ## 问题理解

 我们得到了“石头、剪刀、布”游戏的循环版本。 每个玩家每轮不会独立选择动作； 相反，它们每个都有固定的长度重复模式`n`对于一名玩家和`m`对于另一个。 移动顺序无限重复，游戏持续进行`k`回合。 回合中`i`，每个玩家的移动是通过使用索引到他们的模式来确定的`i mod n`和`i mod m`。 

每轮的结果遵循标准规则：石头胜剪刀，剪刀胜布，布胜石头。 我们提前被告知一名玩家的完整模式，而另一名玩家可以自由选择自己的模式。 任务是构建第二个玩家的重复策略，以便第一个玩家在整个游戏中赢得尽可能多的回合。`k`回合。 

关键的结构约束是两种模式周期性重复，因此一轮的联合状态仅取决于索引对`(i mod n, i mod m)`。 这立即表明结果的序列是周期性的`lcm(n, m)`，因为经过多轮比赛后，两名球员都会回到同一对位置。 

因为`k`可以非常大（最多$10^{12}$），我们无法单独模拟所有回合。 甚至迭代长度可达的单个完整循环$10^5$是可行的，但任何与`k`是不可能的。 

出现微妙的边缘情况时`k`小于整个周期或当`n`和`m`不是互质的。 在这些情况下，并非所有对`(i mod n, j mod m)`均匀地实现，因此任何解决方案都必须考虑每对出现在第一个对中的频率`k`轮数而不是假设均匀分布。 

另一个失败案例来自假设模式中每个位置的独立优化。 如果我们贪婪地分配响应以最大化已知字符串的每个索引的胜利，而不考虑每个索引对全局出现的频率，我们可能会夸大收益。 例如，局部最佳响应对于频繁发生的对齐可能是最佳的，但如果该对齐很少发生，则全局最佳响应可能是次优的。 

## 方法

 蛮力视角从观察对手的策略开始`m`串起来`{R, P, S}`。 有$3^m$可能的策略，对于每一个我们都可以模拟所有`k`回合并计算已知玩家的获胜次数。 即使我们使用周期性来压缩模拟，评估每个候选策略也是指数级的`m`，这立即是不可行的。 

一个更有用的观点是逆向优化。 我们不是构建完整的策略并对其进行评估，而是询问在第二个玩家周期的特定位置选择特定符号可以得到什么。 固定位置`j`按照对手的模式。 每当比赛进行到一轮时`i`这样`i mod m = j`，对手的动作总是一样的。 在所有这些轮次中，唯一的变化是指数`i mod n`已知玩家的。 

这意味着每个位置`j`独立贡献多组配对`(s[i], chosen_move)`，分别加权多少次`i mod n`与此同时发生`j`超过第一个`k`回合。 一旦我们计算出这些频率，就可以选择最佳的位置移动`j`成为三个固定选项的局部最大化。 

关键的见解是，通过计算每对的次数来完全捕获位置之间的相互作用`(i, j)`出现在第一个`k`步骤。 一旦知道了这个频率表，优化就完全解耦了`j`。 

周期性结构意味着我们可以计算每对在一个完整的长度周期中出现的频率`lcm(n, m)`然后按适合的完整周期数进行缩放`k`，加上部分余数。 这将问题减少到在单个可管理的时期内计算贡献，而不是在`k`。

 | --- | --- | --- | --- |
 | 暴力战胜所有策略|$O(3^m \cdot k)$|$O(m)$| 太慢了 |
 | 周期 + 配对计数优化 |$O(nm)$或者$O(n + m)$取决于实施|$O(nm)$或者$O(n + m)$| 已接受 |

 ## 算法演练

 1. 计算周期`L = lcm(n, m)`。 索引的联合模式每次都会重复`L`轮，因此分析一个完整的周期足以理解重复结构。 
2. 对于每个位置`j`在第二个玩家的循环中，确定它在第一个玩家中使用了多少次`L`回合。 这很简单，因为`i mod m`在一个完整周期内统一循环。 
3. 对于每对`(i mod n, j mod m)`在循环内，计算它发生的次数`L`步骤。 这构建了一个频率表，描述已知弦位置和对手位置之间每次交互发生的频率。 
4. 将问题简化为独立决策`j`。 对于固定的`j`，考虑所有指标`i`已知字符串并累加每个符号的次数`s[i]`将在位置上面对对手的移动`j`。 
5. 对于每个`j`，尝试三种可能的动作。 每一步都有一个确定性的回报`R`,`P`， 和`S`，因此计算通过选择该位置的移动所贡献的总胜利`j`。 
6. 选择每个动作总贡献最大的动作`j`独立并对所有贡献求和。 
7. 缩放结果，如果`k`大于`L`，因为完整的循环完全重复。 

### 为什么它有效

 正确性基于以下事实：任何一轮的状态完全由下式决定：`(i mod n, i mod m)`。 这将无限的轮序列划分为索引对的独立等价类。 无论其在时间上的位置如何，一对的每次出现都贡献相同的回报，因此总分是对频率的线性总和。 一旦以这种方式表达，对手在每个位置的决策仅影响涉及该位置的项，并且不同位置之间不存在耦合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def beat(a, b):
    return (a == 'R' and b == 'S') or (a == 'S' and b == 'P') or (a == 'P' and b == 'R')

def score(move, s):
    total = 0
    for ch in s:
        if beat(ch, move):
            total += 1
    return total

n, m, k = map(int, input().split())
s = input().strip()

# Build frequency of each index i mod n across k rounds grouped by j mod m
# We compute how many times each pair (i, j) appears in first k rounds.

cnt_i = [0] * n
cnt_j = [0] * m

for i in range(k):
    cnt_i[i % n] += 1
    cnt_j[i % m] += 1

ans = 0

for j in range(m):
    best = 0
    for move in "RPS":
        cur = 0
        for i in range(n):
            if beat(s[i], move):
                cur += cnt_i[i] * cnt_j[j]
        best = max(best, cur)
    ans += best

print(ans)
```该代码将问题直接分解为索引频率。 数组`cnt_i`和`cnt_j`表示每个残基类别在第一个残基类别中使用的频率`k`回合。 嵌套循环结束`i`和`j`通过独立频率的乘法隐式重建每对的贡献。 

最微妙的部分是理解我们从未明确构建完整的`k × k`交互网格。 相反，我们依赖于模块化结构的可分离性：每一轮都独立地贡献于`(i mod n)`和`(i mod m)`。 

## 工作示例

 ### 示例 1

 考虑`n = 2`,`m = 2`,`k = 4`， 和`s = "RS"`。 

我们首先计算每个索引在前 4 轮中出现的频率。 

| 我| 我mod 2 |
 | --- | --- |
 | 0 | 0 |
 | 1 | 1 |
 | 2 | 0 |
 | 3 | 1 |

 所以`cnt_i = [2, 2]`。 这同样适用于`cnt_j`。 

现在我们评估第二名球员的每个位置。 对于固定的`j`，我们尝试每一步并计算它产生了多少胜利`s`。 

例如，如果我们选择`R`，它仅击败`S`，所以只有位置`i = 1`做出贡献。 

| 移动| 贡献 |
 | --- | --- |
 | 右 | 2 * cnt_j[j] | 2 * cnt_j[j] |
 | 普 | 取决于 R 匹配 |
 | S | 取决于 P 匹配 |

 总结最佳选择给出最终答案。 

该迹线表明只有频率结构重要，而不是顺序。 

### 示例 2

 让`n = 3`,`m = 2`,`k = 6`,`s = "RPS"`。 

我们计算`cnt_i = [2, 2, 2]`和`cnt_j = [3, 3]`。 

每个`j`独立优化。 对于每一步棋，我们都会评估其中有多少个符号`s`它的节拍按频率加权。 这里的对称性证明了相同的`cnt_i`值会导致相同的每个位置优化。

 ## 边缘情况

 当出现一种边缘情况时`n = 1`，这意味着已知玩家总是使用相同的符号。 在这种情况下，每个决策都简化为选择对手每个位置的最佳响应，因为所有交互都崩溃为频率表中的一行。 该算法自然会处理这个问题，因为`cnt_i[0] = k`所有其他条目均为零，因此无需特殊大小写即可正确计算贡献。 

另一种情况是当`m = 1`，其中对手在其循环中只有一个位置。 然后，所有回合都取决于单一选择，并且算法正确地将所有可能的收益聚合到该位置的一个优化步骤中。 

最后一个微妙的情况发生在`k < n`或者`k < m`。 这里，有些索引从未出现在第一个中`k`轮，产生零条目`cnt_i`或者`cnt_j`。 由于未使用的位置没有任何贡献，因此算法仍然为它们选择任意移动，而不影响结果，与预期行为相匹配。
