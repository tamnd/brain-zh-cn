 ## 解决方案
 ## 问题理解

 我们给出了一个大小为的矩形网格$n \times m$，最初为空。 移动按固定顺序一一到达，每一次移动都会将之前未绘制的单元格涂成黑色。 每次移动之后，我们需要决定是否允许放置该移动或是否应该跳过该移动。 

使一步棋无效的规则很微妙。 当一组黑色单元格在 4 邻居邻接图中包含一个简单循环，并且该循环将至少一个网格单元严格包围在其边界内时，该过程就会停止，并且不会应用该移动。 否则，移动被接受并且单元格保持黑色。 

所以从概念上讲，我们正在动态构建网格图的子图，并且我们必须检测第一次出现“几何封闭”循环。 

约束条件很大：最多$3 \cdot 10^5$细胞可以整体存在，一举一动都截然不同。 这排除了每个步骤之后对整个网格或图表的任何重新计算。 任何尝试每次移动运行 BFS 或 DFS 的解决方案都会立即失败，因为这会导致$O(k(nm))$最坏情况下的行为。 

关键的困难在于周期本身是不够的。 网格图中的循环并不总是“有效的游戏结束”：只有包含非空内部物质的循环。 必须仔细区分触及边界或在空白空间周围退化的小局部循环。 

一个常见的陷阱是将其视为“检测增量图中的任何循环”。 这会失败，因为环可能存在而没有封闭区域，而且还因为条件与平面结构相关，而不仅仅是图结构。 

另一种失败情况是假设一旦出现循环，只需在最新节点周围本地检测到它就足够了。 循环可能通过远处的旧组件形成。 

## 方法

 一种直接的方法是维护当前的黑色图，并且对于每个新的移动，从新节点运行 DFS 或 BFS，以检查是否已经存在返回自身的路径，该路径与内部形成循环。 如果发现循环，我们将拒绝该移动。 

这原则上是正确的，但太慢了。 每个BFS/DFS最多可以遍历$O(nm)$节点，我们可以这样做$O(k)$次，给予$O(k \cdot nm)$，这远远超出了限制。 

结构洞察力来自于重新构建条件。 我们并不是真正被要求检测任意循环，而是检测当前的黑色区域是否变得“平面上非简单连接”，这意味着它不再是平面意义上的森林状区域。 在网格图中，可以使用不相交集并集 (DSU) 增量跟踪，但标准 DSU 仅跟踪连接性，而不跟踪周期。 

关键的观察结果是，在平面网格图中，每个单元的度数最多为 4，并且每次边缘插入要么合并两个组件，要么在连接的组件内创建一个新循环。 在已经具有足够结构来包围区域的组件内部创建循环的那一刻，它对应于我们第一次“在 2D 空间中闭合循环”。 

为了准确起见，我们在单元上维护一个 DSU。 此外，我们还跟踪每个连接组件中的边数。 每次我们连接两个单元时，如果它们已经在同一个 DSU 组件中，我们就会添加一条冗余边，从而形成一个循环候选。 关键点是，并非每个冗余边都是致命的，但任何组件中的第一个冗余边都是循环出现的时刻。 在网格图中，组件中的第一个循环恰好对应于一个闭环，此时它必须包围至少一个单元，因为网格循环是平面的并且不能退化为零面积循环。 

因此，游戏在我们第一次尝试合并两个已连接的节点时结束。 

因此，我们通过循环检测将问题简化为动态连接。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每一步强力 DFS/BFS |$O(k \cdot nm)$|$O(nm)$| 太慢了|
 | DSU周期检测|$O(k \alpha(nm))$|$O(nm)$| 已接受 |

 ## 算法演练

 我们将每个单元格视为图中的一个节点。 我们在所有单元上维护 DSU 结构，最初每个单元都是隔离且不活动的。 

我们还维护一个布尔网格，标记单元是否已被激活（涂成黑色）。 

### 步骤

 1. 初始化一个DSU$n \cdot m$节点。 每个节点都以自己的集合开始。 我们还为所有单元维护一个最初为 false 的已访问或活动数组。 
2. 流程按顺序移动。 对于搬家$i$，考虑细胞$(r_i, c_i)$。 
3. 将单元格标记为活动。 这意味着它成为图表的一部分。 
4. 对于该小区的四个邻居中的每一个，检查该邻居是否处于活动状态。 如果它不活动，请忽略它，因为它不是当前图形的一部分。 
5. 如果邻居处于活动状态，则尝试将当前小区与 DSU 中的邻居合并。 在执行联合之前，请检查它们是否已属于同一 DSU 组件。 
6. 如果我们发现至少一个邻居已位于同一 DSU 组件中，则意味着添加此单元会在该组件中引入一个循环。 在这种情况下，我们立即为此移动输出 0，并且不会进一步激活或合并它。 
7. 如果没有发生此类冲突，我们将该单元与所有活动邻居合并并输出 1。 

微妙的顺序很重要：我们必须在合并之前检测循环，否则我们将无法正确检测此移动的“已连接”状态。 

### 为什么它有效

 DSU 将黑色单元的连接组件维持在 4 邻接状态下。 当我们尝试连接两个已连接的节点时，我们引入了一条不属于该组件的任何生成树的边。 这会在平面网格图中创建一个循环。 在网格中，任何这样的循环都对应于一个闭合的多边形链，其嵌入必须包围至少一个单位正方形，因为边缘是网格中心之间的轴对齐线段。 因此，第一个冗余边恰好是第一次满足游戏结束条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        return True

def solve():
    n, m, k = map(int, input().split())
    idx = lambda r, c: (r - 1) * m + (c - 1)

    dsu = DSU(n * m)
    active = [False] * (n * m)

    out = []

    for _ in range(k):
        r, c = map(int, input().split())
        v = idx(r, c)

        if active[v]:
            out.append('0')
            continue

        active[v] = True
        cycle = False

        for dr, dc in ((1, 0), (-1, 0), (0, 1), (0, -1)):
            nr, nc = r + dr, c + dc
            if 1 <= nr <= n and 1 <= nc <= m:
                u = idx(nr, nc)
                if active[u]:
                    if dsu.find(u) == dsu.find(v):
                        cycle = True
                    else:
                        dsu.union(u, v)

        if cycle:
            out.append('0')
            active[v] = False
        else:
            out.append('1')

    print(''.join(out))

if __name__ == "__main__":
    solve()
```DSU 存储活动小区之间的连接性。 这`active`数组确保我们只连接已经绘制的方块。 

当我们在同一 DSU 组件中看到已经处于活动状态的邻居时，就会准确地触发循环检测。 这表明新边在该组件中是多余的。 

我们还通过立即拒绝重复输入位置的情况来处理它们，因为重新添加节点会在增量构造中微不足道地引入不一致。 

## 工作示例

 考虑一个简单的$2 \times 2$网格移动形成一个正方形。

1 2

```我们在每一步之后跟踪 DSU 根。

这证实了第一冗余边对应于平面嵌入中的循环形成的不变式。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(k \alpha(nm))$| 每次移动都会对最多 4 个邻居执行恒定数量的 DSU 查找和并集 |
 | 空间|$O(nm)$| DSU 阵列和每个单元的激活状态 |

 这些约束允许最多 300,000 个单元，并且具有路径压缩的 DSU 基本上以每次操作的恒定摊销时间运行。 即使对于完整的输入大小，这也完全符合限制。

 ## 边缘情况

 对已激活单元格的重复移动会立即被拒绝，因为`active[v]`已经是真的了。 如果没有此保护，DSU 将错误地尝试再次将节点与其自身或其邻居合并，从而可能产生错误的循环信号。 

沿着外边缘形成循环的边界行走被正确处理，因为循环检测纯粹是结构性的：当最后一个边缘闭合循环时，DSU 将已经在一个组件中拥有所有顶点，并且最终的并集尝试将检测冗余。 

带有分支的“树状”扩展永远不会触发拒绝，因为每个联合都连接以前单独的组件。 仅当一条边连接同一组件中已有的两个顶点时，算法才会拒绝，匹配平面循环完成的时刻。
