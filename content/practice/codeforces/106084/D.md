 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供一个字符串，我们想要测量它距离回文有多远，其中“距离”是允许插入、删除和替换的标准编辑距离。 

目标不是将字符串转换为另一个任意字符串，而是限制为任何回文。 因此，对于每个输入单词，我们正在寻找将其转换为向前和向后读取相同的字符串所需的最少编辑次数。 目标回文不是预先固定的，我们可以选择最好的回文。 

关键的困难在于，即使在本地，最接近的回文也可能并不明显。 末端不匹配不会强制替换，因为我们可以删除或插入字符并移位对齐以减少未来的成本。 这使得贪心推理变得不可靠。 

输入约束的总长度很小，所有字符串长度的总和最多为 3000。这立即表明每个测试用例的 O(n²) 动态编程解决方案是可以接受的，而任何立方体或涉及字符串对上的大状态空间也仍然会通过，但没有必要。 

一种幼稚的方法会尝试隐式地将字符串与所有回文进行比较。 即使限制为相同长度的回文也已经为每个位置提供了 26 个选择，当与改变长度的编辑操作相结合时，该选择将呈指数级增长。 即使对于中等的弦，这也很快变得不可行。 

边缘情况大多是结构性的。 

像“x”这样的单字符字符串应该已经是回文，所以答案是零。 除了一个字符之外几乎对称的字符串（例如“bababac”）通常可以通过一端的单个替换来修复，但如果粗心的算法强制对称位置严格匹配而不考虑删除，那么粗心的算法可能会过多计算编辑次数。 

另一个微妙的情况是，最佳对齐需要删除字符而不是直接替换不匹配的字符，例如“abcda”。 直接替换两端并不总是最佳的； 删除中间结构可以减少编辑次数。 

## 方法

 蛮力视角是想象生成每个回文并计算从输入字符串到每个候选者的编辑距离，然后取最小值。 即使我们将回文长度限制为最多，例如输入长度的两倍，回文的数量也会随着长度呈指数增长。 通过标准编辑距离，每次比较本身的成本为 O(n²)，因此这种方法远远超出了任何可行的限制。 

更结构化的观点来自于观察回文完全由其左半部分决定。 然而，即使我们尝试优化所有可能的一半，我们仍然面临着巨大的搜索空间。 

关键的转变是停止明确地思考目标字符串。 相反，我们在动态规划期间直接在原始字符串的间隔上强制执行回文约束。 我们问一个更简单的问题：将子字符串 w[l..r] 转换为回文所需的最少编辑次数是多少？ 这完全消除了枚举目标的需要。 

一旦我们以这种方式构建问题，结构就变得局部了。 在任意区间[l,r]，如果字符已经匹配，我们只需要求解内部区间。 如果它们不匹配，我们可以通过替换来修复它们，或者删除一侧以更好地在内部进行对齐。 

这导致经典区间 DP 在每个不匹配处具有三个有意义的选择：捕获替换、从左侧删除和从右侧删除。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 回文上的暴力破解 | 指数| O(n) | 太慢了|
 | 间隔 DP | 每个字符串 O(n²) | O(n²) | 已接受 |

 ## 算法演练

我们将 dp[l][r] 定义为将子字符串 w[l..r] 转换为回文所需的最少编辑操作数。 

1. 将所有单个字符和空间隔的 dp 初始化为零。 任何长度为 0 或 1 的字符串都已经是回文，因此不需要编辑。 
2. 按长度递增顺序考虑子字符串，从长度 2 开始直到整个字符串。 首先处理小区间可确保每当我们计算 dp[l][r] 时，所有较小的子问题都是已知的。 
3. 对于每个区间 [l, r]，比较字符 w[l] 和 w[r]。 如果它们相等，我们不需要直接修改它们，所以成本正好是 dp[l+1][r-1]。 这对应于将两端配对并独立求解内部。 
4. 如果 w[l] 和 w[r] 不同，我们考虑三种方法来解决不匹配问题。 我们可以替换一个端点来匹配另一个端点，这需要 1 加 dp[l+1][r-1]。 我们可以删除左边的字符，这需要花费 1 加 dp[l+1][r]，有效地将间隔向右移动。 我们可以删除右边的字符，这需要 1 加 dp[l][r-1]，将间隔向左移动。 
5. 取这些选项中的最小值并将其存储在 dp[l][r] 中。 这捕获了最终强制对称的所有有效编辑序列。 
6. 每个测试用例的答案是 dp[0][n-1]。 

这样做的原因是，字符串上的每个最佳编辑操作序列都可以解释为逐渐解决某个时间间隔内最外层未解决的不匹配问题。 任何最佳解决方案都必须将当前端对齐在一起或在继续之前移除其中一个端。 这保证了递归探索所有结构上不同的最佳变换，而不会丢失任何有效的对齐路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(s: str) -> int:
    n = len(s)
    if n <= 1:
        return 0

    dp = [[0] * n for _ in range(n)]

    for length in range(2, n + 1):
        for l in range(n - length + 1):
            r = l + length - 1

            if s[l] == s[r]:
                dp[l][r] = dp[l + 1][r - 1]
            else:
                dp[l][r] = min(
                    dp[l + 1][r - 1] + 1,
                    dp[l + 1][r] + 1,
                    dp[l][r - 1] + 1
                )

    return dp[0][n - 1]

def main():
    t = int(input())
    out = []
    for _ in range(t):
        s = input().strip()
        out.append(str(solve_one(s)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案使用二维 DP 表，其中每个条目代表一个子串区间。 长度上的嵌套循环确保在需要时已经计算出所有依赖关系，因为每个转换仅引用严格较小的间隔。 

三种情况的最小值直接编码替换与删除决策。 替换情况对应于同时固定两端，而删除情况对应于移动对齐以便将来的匹配成为可能。 这种结构允许将编辑距离行为嵌入到回文约束中。

这证实了最佳解决方案可能更喜欢结构编辑而不是立即的字符修复。 

### 示例 2：“bababac”

这表明算法可以自然地识别单个边界修复何时解析整个结构。

 ## 边缘情况

 像“x”这样的单字符字符串作为长度为 1 的间隔进入 DP，其中基本规则立即应用，产生零编辑，因为不可能出现不匹配。 该算法从不尝试长度为 1 的转换，因此不会出现意外的过度计数。 

像“ab”这样的两个字符的字符串恰好创建一个区间 [0,1]，其中不匹配会触发最小的替换或删除。 两条路径正确地花费 1，并且没有访问无效的中间间隔。 

已经是回文的字符串（例如“abba”）在每个递归级别始终满足 s[l] == s[r]，因此 DP 会反复减少到内部间隔，直到达到基本情况，从而始终累积零成本。
