 ## 解决方案
 ## 问题理解

 我们收到了几个独立的查询。 每个查询包含两个整数，我们需要判断这两个数字是否构成一对孪生素数。 这里的孪生素数概念非常具体：两个数字只有当它们都是素数并且它们的差恰好是二时才符合资格。 

因此，对于每一对，任务不是分解或计算任何超出素性的复杂内容以及两个值之间的简单关系检查。 每个测试用例的输出是一个简单的二元决策：如果两个数字都是质数并且相差正好 2，则打印肯定响应，否则打印否定响应。 

约束足够小，我们可以对每个数字进行直接的素性检查。 每个值最多为 1,000,000，最多有 10 个测试用例。 这立即排除了诸如每个查询的繁重预处理之类的任何事情，但可以轻松地允许简单的素性测试$O(\sqrt{n})$或预先计算的筛子$O(N \log \log N)$。 即使是较慢的每次查询方法也很容易受到限制，因为素性检查的总数很小。 

唯一微妙的边缘情况来自几乎满足条件的输入。 数字对可能相差 2，但包括复合数，如 (12, 14)，或者两个数字可能单独为素数，但相差 2，如实际上有效的 (11, 13)，或也有效的 (5, 7)。 另一种边缘情况是一个数字是素数而另一个不是素数，例如 (3, 5) 有效，但 (2, 4) 失败，因为 4 不是素数，即使差异是正确的。 这些情况很重要，因为幼稚的实现可能只检查差异并完全忘记素性。 

## 方法

 蛮力的想法很简单。 对于每个测试用例，我们独立检查两个数字是否为素数。 最简单的素性检查尝试将数字除以从 2 到其平方根的每个整数。 如果两个数字都没有除数，则仅当它们的差值恰好为 2 时，我们才接受该对。 

这是正确的，因为素数是直接定义的，不需要结构技巧。 当我们考虑最坏的情况时，就会出现效率低下的情况：检查 100 万以内的数字的素性需要大约 1,000 次除法，因此单个测试用例大约需要 2,000 次检查。 对于 10 个测试用例，这大约只有 20,000 次操作，这已经是微不足道的了，因此即使这种暴力方法在约束下也是可以接受的。 

尽管如此，我们仍然可以通过使用筛子预先计算一百万以内的所有素数来进一步简化推理。 一旦构建了该数组，每个查询就会变成恒定时间：两次查找和一次减法检查。 筛子将工作从查询时间转移到预处理时间，这对于此约束大小可以忽略不计。 

关键的观察结果是，问题纯粹是关于素数集合中的成员资格，并且该集合对于所有查询都是固定的。 这使得预处理在简单性和重复查询方面更加严格。

 ## 算法演练

 我们用筛子标记一百万以内的所有素数。 

1. 构建布尔数组`is_prime`大小为 1,000,001，最初假设除 0 和 1 之外的所有数字都是质数。这建立了一个基线，我们仅删除已知的复合数。 
2. 从 2 向上过筛。 对于仍标记为质数的每个数字，将其所有倍数标记为非质数。 此步骤确保每个合数在每个最小质因数中精确消除一次。 
3. 预处理后，读取每个查询对(a,b)。 
4. 对于每一对，首先检查是否`b - a == 2`。 这是必要的，因为孪生素数必须恰好相距两个。 
5. 如果差异条件不成立，则立即输出N，因为不需要进行素性测试。 
6. 如果差异条件成立，检查两者是否成立`a`和`b`在筛表中被标记为素数。 
7. 仅当两者都是素数时才输出Y，否则输出N。 

### 为什么它有效

 筛子保证每个整数的正确分类，直到极限。 由于孪生素数纯粹由两个独立的素性条件加上固定的差异约束来定义，因此检查预先计算的素数集中的成员资格相当于执行完整的素性测试。 该算法从不依赖于部分或启发式检查，因此每个接受的对都完全满足两个定义属性，并且每个拒绝的对至少违反其中一个。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXN = 10**6

# Sieve of Eratosthenes
is_prime = [True] * (MAXN + 1)
is_prime[0] = is_prime[1] = False

p = 2
while p * p <= MAXN:
    if is_prime[p]:
        step = p
        start = p * p
        for x in range(start, MAXN + 1, step):
            is_prime[x] = False
    p += 1

t = int(input())
for _ in range(t):
    a, b = map(int, input().split())
    if b - a == 2 and is_prime[a] and is_prime[b]:
        print("Y")
    else:
        print("N")
```在处理任何查询之前，筛选器会构建一次。 这确保了以后的每次检查都是简单的数组查找。 条件`b - a == 2`首先检查以避免明显失败时不必要的素数表访问，尽管考虑到较小的约束，这主要是风格上的。 

核心思想是素性信息被完全缓存，因此每次查询都减少为常数时间验证。

 ## 边缘情况

 一种重要的边缘情况是满足差异条件但素性失败时。 例如，输入 (12, 14) 通过间隙检查，但两个数字都是合数。 筛子将两者标记为 false，因此算法正确输出 N。 

另一种情况是，两个数都是素数，但差距不正好是二，例如 (11, 13) 有效，但 (11, 17) 尽管都是素数，但仍被拒绝。 该算法首先检查差异并立即拒绝不匹配的间隙，从而防止不必要的查找工作。 

最后一种情况是最小素数。 对于 (2, 3)，差值为 1，因此即使两者都是素数，它也会被拒绝。 对于 (3, 5)，两个条件都成立，并且算法正确地将其识别为孪生素数对。
