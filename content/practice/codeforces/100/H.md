 ## 解决方案
 ## 问题理解

 我们有几块 10 × 10 战舰板。 每个单元格要么是空的，要么被船的一部分占据。 任务是验证每个占用的单元是否属于有效的队列配置。 

有效的战列舰舰队恰好包含一艘长度为 4 的船、两艘长度为 3 的船、三艘长度为 2 的船和四艘长度为 1 的船。每艘船必须是直的水平或垂直线段。 船只不能重叠、不能弯曲、不能相互接触，甚至是对角接触。 

该板很小，只有 100 个单元。 即使有多达 10 块电路板，我们总共也最多只能处理 1000 个电池。 这立即排除了性能作为主要挑战的可能性。 困难纯粹在于正确验证所有几何条件。 

最危险的错误来自于接受无效的船舶形状或忘记对角邻接检查。 

考虑这个板片段：```
**00000000
*000000000
0000000000
```三个占据的单元格形成L形。 粗心的 DFS 只计算连接的组件，会看到尺寸为 3 的组件，并可能错误地将其分类为长度为 3 的有效船舶。正确答案是否定的，因为船舶必须是直的。 

另一个常见的陷阱是对角线接触：```
*000000000
0*00000000
0000000000
```这是两艘对角接触的单细胞飞船。 战舰规则禁止这样做，所以该棋盘无效。 仅检查 4 方向邻接性的解决方案将错过这一点。 

第三个微妙的情况是合并船舶：```
***0000000
00*0000000
0000000000
```垂直单元与水平船正交，形成弯曲组件。 董事会必须立即拒绝。 

最后一类错误来自船舶计数。 即使每个连接的组件都是一条直线，车队的组成也必须与所需的数量完全匹配。 例如，拥有两艘长度为 4 的船只且没有长度为 3 的船只是无效的。 

## 方法

 蛮力的想法很简单。 我们可以尝试独立识别每个可能的船舶放置位置，并从棋盘上删除匹配的船舶。 由于棋盘只有 10 × 10，即使是相当笨拙的枚举仍然可以快速运行。 

一种幼稚的策略是重复扫描棋盘上的水平或垂直部分，将它们标记为船舶，并希望剩余的单元格也形成有效的船舶。 问题是含糊不清。 畸形的形状可能会意外分解为多艘合法的船只。 例如，L 形可以解释为长度为 2 的船加上单单元船，即使组件本身无效。 

思考棋盘的正确方法是将其视为图形问题。 每个占用的单元都属于 4 方向邻接下的一个连通分量。 由于船舶不能正交接触，因此每个连接的组件必须恰好对应于一艘船。 

这个观察简化了一切：

 1. 对每个未访问的占用单元运行 DFS 或 BFS。 
2. 提取整个连通分量。 
3. 验证所有单元格是否位于一行或一列上。 
4. 验证单元格是否形成连续的段。 
5. 验证没有单元与另一艘船对角接触。 
6. 计算船长。 

由于电路板只有 100 个单元，因此整个电路板的 DFS 实际上是恒定时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(100立方) | O(100) | 接受但尴尬|
 | 最佳 | O(100) | O(100) | 已接受 |

 ## 算法演练

 1. 将 10 × 10 的棋盘读成网格。 
2. 维护一个访问过的数组，以便每个占用的单元格只被处理一次。 
3. 迭代板上的每个单元格。 
4. 当无人访问时`'*'`找到单元格后，从它开始 DFS 或 BFS，仅使用 4 方向移动来收集其整个连接组件。 

使用 4 个方向是正确的，因为船舶仅水平或垂直连接。 对角线单元绝不能属于同一艘船。 
5. 对于组件内的每个单元，检查所有四个对角邻居。 

如果任何对角邻居包含`'*'`，棋盘立即无效，因为船只不允许对角接触。 
6. 收集组件后，验证其形状。 

如果组件的尺寸为 1，则它自动成为有效的单单元飞船。 

否则，要么：

 - 所有单元格必须共享同一行，或者
 - 所有单元格必须共享同一列。 

如果这两个条件都不成立，船就会弯曲，板子就失效了。 
7. 检查连续性。 

对于水平船，对列索引进行排序并验证它们是否形成连续整数。 

对于垂直船，对行索引进行排序并验证它们是否形成连续的整数。 

这可以防止形状断开，例如：```
*0*
```8. 记录船长。 
9. 处理完所有组件后，验证确切的机队组成：

 - 一艘 4 号船，
 - 两艘 3 号船，
 - 三艘 2 号船，
 - 四艘 1 号船。 
10. 如果所有检查都成功，则打印 YES，否则打印 NO。 

### 为什么它有效

 每个占用的单元都属于 4 方向邻接下的一个连通分量。 由于船舶不能重叠或正交接触，因此每个组件必须代表一艘船舶。 

该算法验证合法船舶的所有必要属性：

 - 对角线检查防止接触船舶，
 - 行/列对齐可防止弯曲，
 - 连续性检查可防止出现间隙，
 - 船队计数强制执行所需的确切船舶库存。 

由于每个组件都是独立检查的，并且每个占用的单元都只处理一次，因此任何无效配置都无法逃脱检测。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

DIRS = [(-1, 0), (1, 0), (0, -1), (0, 1)]
DIAG = [(-1, -1), (-1, 1), (1, -1), (1, 1)]

def solve_board(grid):
    visited = [[False] * 10 for _ in range(10)]
    counts = [0] * 5

    def inside(r, c):
        return 0 <= r < 10 and 0 <= c < 10

    def dfs(sr, sc):
        stack = [(sr, sc)]
        visited[sr][sc] = True
        cells = []

        while stack:
            r, c = stack.pop()
            cells.append((r, c))

            for dr, dc in DIRS:
                nr, nc = r + dr, c + dc

                if inside(nr, nc) and not visited[nr][nc] and grid[nr][nc] == '*':
                    visited[nr][nc] = True
                    stack.append((nr, nc))

        return cells

    for r in range(10):
        for c in range(10):
            if grid[r][c] == '*' and not visited[r][c]:
                cells = dfs(r, c)

                # diagonal touching check
                for x, y in cells:
                    for dx, dy in DIAG:
                        nx, ny = x + dx, y + dy

                        if inside(nx, ny) and grid[nx][ny] == '*':
                            return False

                size = len(cells)

                if size > 4:
                    return False

                rows = set(x for x, _ in cells)
                cols = set(y for _, y in cells)

                if len(rows) != 1 and len(cols) != 1:
                    return False

                if len(rows) == 1:
                    vals = sorted(y for _, y in cells)
                else:
                    vals = sorted(x for x, _ in cells)

                for i in range(1, len(vals)):
                    if vals[i] != vals[i - 1] + 1:
                        return False

                counts[size] += 1

    return counts[1] == 4 and counts[2] == 3 and counts[3] == 2 and counts[4] == 1

def main():
    t = int(input())
    ans = []

    for _ in range(t):
        grid = []

        while len(grid) < 10:
            line = input().strip()

            if line:
                grid.append(line)

        ans.append("YES" if solve_board(grid) else "NO")

    print("\n".join(ans))

if __name__ == "__main__":
    main()
```该解决方案将验证分成独立的检查，这使得调试更加容易。 

DFS 一次只收集一个连接的组件。 因为我们只在四个方向上移动，所以对角线接触的船只仍然是独立的组件，这正是我们想要的。 

对角线验证是在成分提取后有意执行的。 此时我们已经知道哪些单元格属于当前船舶，因此任何对角线`'*'`必须属于另一艘船并立即违反规则。 

行和列集检查是核心形状验证。 如果两组的尺寸均大于 1，则组件会弯曲。 这捕获了所有 L 形和 T 形。 

连续性检查虽然微妙但必要。 像这样的组件：```
*0*
```不能出现在有效的 4 向连接下，但检查连续性仍然使逻辑稳健且独立。 

最终的舰队数量验证保证了准确的战舰库存。 如果计数与官方规则不符，即使形状完美的船只也会被拒绝。

 输入：```
