 ## 解决方案
 ## 问题理解

 我们有一个正方形的房间，边长`n`，所以总面积是`n × n`。 我们还有`k`方形地毯，每块地毯都有边长`n1`。 每块地毯始终保持轴对齐，因为禁止旋转，但由于地毯是正方形，旋转实际上不会改变任何东西。 

问题是这些地毯能否完全覆盖房间。 地毯可能会重叠，因此允许使用额外的区域。 唯一重要的是房间的每个点是否都能被至少一张地毯覆盖。 

限制非常小。 边长最多为 12，地毯的数量最多为 10。即使是详尽的几何搜索在技术上也是合适的，但问题隐藏了更简单的数学观察。 由于可以通过一些算术运算来决定答案，因此没有理由模拟放置。 

主要的陷阱是误解重叠的含义。 一个天真的想法是将地毯总面积与房间面积进行比较，并在地毯提供足够面积时回答“是”。 这还不够，因为几何形状仍然很重要。 

考虑这个例子：```
10 1 10
```单块地毯与房间完全匹配，所以答案是肯定的。 

现在看：```
10 4 4
```地毯总面积为`4 × 4 × 4 = 64`，但房间面积为`100`。 答案显然是否定的，因为没有足够的可用总面积。 

更有趣的案例是：```
10 4 6
```地毯总面积为`144`，大于房间面积。 答案是肯定的。 四`6 × 6`地毯可以通过稍微重叠来覆盖整个房间。 

粗心的实施还可能错误地认为地毯必须完美地铺满房间，没有重叠。 例如：```
10 4 6
```平铺解释失败是因为`10`不能被整除`6`，但正确答案仍然是 YES，因为允许重叠。 

真实情况是房间一侧需要多少块地毯。 由于每块地毯的跨度`n1`水平和垂直单位，我们需要足够的地毯来覆盖长度`n`在两个方向上。 

## 方法

 蛮力思维是尝试房间里所有可能的地毯放置位置。 由于坐标很小，我们可以离散房间并递归地放置地毯，同时检查所有单元格是否都被覆盖。 

这种方法之所以有效，是因为限制非常小，但展示位置的数量却呈爆炸式增长。 即使每块地毯只有大约 100 个可能的位置，尝试所有配置也将涉及大约`100^10`说是最坏的情况，这是完全不现实的。 

关键的观察是只有一维覆盖很重要。 

假设我们将地毯放在网格中。 沿着房间的一侧，每块地毯的贡献最大`n1`长度。 覆盖一段长度`n`，我们至少需要：$$\left\lceil \frac{n}{n1} \right\rceil$$沿着那个维度的地毯。 

因为房间是二维的，所以我们需要水平方向上的许多地毯和垂直方向上的相同数量的地毯。 因此所需地毯的最少数量为：$$\left\lceil \frac{n}{n1} \right\rceil^2$$如果我们已经有至少那么多地毯，我们可以将它们排列成网格并完全覆盖房间，可能在边缘附近重叠。 

这将整个问题转化为单个算术比较。

 ## 算法演练

 1. 读取数值`n`,`k`， 和`n1`。 
2. 计算房间一侧需要多少块地毯。 

由于每块地毯都覆盖`n1`一个方向上的单位，计数为：$$need = \left\lceil \frac{n}{n1} \right\rceil$$整数上限除法可以计算为：```
need = (n + n1 - 1) // n1
```3. 计算完整网格覆盖所需的地毯总数。```
required = need * need
```我们需要`need`地毯横跨和`need`地毯下来了，所以总数就是他们的产品。 
4. 比较`k`和`required`。 

如果`k >= required`， 打印`YES`。 否则打印`NO`。 

### 为什么它有效

 每块地毯覆盖侧面的一个正方形区域`n1`。 沿着单一维度，覆盖长度`n`至少需要`ceil(n / n1)`地毯，因为每块地毯的贡献最大`n1`单位。 

由于房间是正方形的，我们独立需要那么多行和列的地毯。 一个`need × need`即使某些地毯延伸到边界之外或与其他地毯重叠，布置也始终覆盖房间。 

该算法准确地计算了这个最小可行计数，因此答案是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k, n1 = map(int, input().split())

need = (n + n1 - 1) // n1

if k >= need * need:
    print("YES")
else:
    print("NO")
```第一步直接从标准输入读取三个整数。 

表达式：```
(n + n1 - 1) // n1
```使用整数运算实现上限除法。 这比浮点除法更安全，因为它避免了精度问题。 

乘法：```
need * need
```表示方格排列中的地毯数量。 由于约束很小，因此在 Python 中不可能发生溢出，但即使在具有固定大小整数的语言中，值仍然非常小。 

最终比较检查可用的地毯是否足以构建覆盖网格。

import sys, io

def solve():
    import sys
    input = sys.stdin.readline

    n, k, n1 = map(int, input().split())

    need = (n + n1 - 1) // n1

    if k >= need * need:
        print("YES")
    else:
        print("NO")

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout

    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()

    solve()

    out = sys.stdout.getvalue()

    sys.stdin = backup_stdin
    sys.stdout = backup_stdout

    return out

 ## 边缘情况

 考虑输入：```
10 4 6
```该算法计算：```
need = ceil(10 / 6) = 2
required = 2 × 2 = 4
```自从`k = 4`，答案是YES。 

这个案例推翻了地毯必须完美地铺满房间的错误假设。 一个`6 × 6`地毯不能均匀划分房间尺寸，但重叠可以完全覆盖。 

现在考虑：```
10 4 4
```该算法计算：```
need = ceil(10 / 4) = 3
required = 9
```由于只有 4 块地毯可用，所以答案是否定的。 

这捕获了仅比较总面积的解决方案。 尽管每块地毯都相当大，但四块地毯无法在两个维度上横跨整个房间。 

最后，考虑：```
10 1 10
```计算变为：```
need = 1
required = 1
```该算法正确地打印“是”，因为一张地毯已经完全覆盖了整个房间。
