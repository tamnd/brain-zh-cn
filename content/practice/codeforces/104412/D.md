 ## 解决方案
 ## 问题理解

 我们有两个整数数组。 第一个数组，称为模式数组，有长度$M$。 第二个数组，称为源数组，有长度$N$。 我们想要计算通过向该段的每个元素添加相同的常量值，可以将源数组的多少个连续段转换为模式数组。 

换句话说，一段$N$比赛$M$如果值的形状相同，则最多垂直移动。 绝对值并不重要，重要的是值如何从一个位置变化到下一个位置。 

关键的限制是两个数组都可以非常大，最多可达$10^6$元素。 这排除了每个子数组之间的任何二次比较$N$和$M$，因为这需要粗略检查$N \cdot M$最坏情况下的元素，远远超出了$10^6$运营。 

一种天真的方法会比较每个长度-$M$的子数组$N$直接针对$M$，但即使是一次比较也会花费$O(M)$，导致$O(NM)$总体而言，最多可达$10^{12}$最坏情况下的操作。 

当所有元素都在时，就会出现微妙的边缘情况$M$是相同的。 在这种情况下，每个有效的匹配都需要选择的子数组$N$也包含相同的值，但移位版本仍然可以匹配。 如果错误地比较原始值而不是相对差异，则将错过相同形状的情况。 例如，$M = [1,1,1]$和一个子数组$[5,5,5]$应该匹配，但直接比较会失败。 

另一个极端情况是当$M$有长度$1$。 每个单个元素子数组$N$有效，因为任何值都可以移动以匹配单个数字。 答案很简单$N$，如果该方法始终假设存在差异，则很容易忽视这一点。 

## 方法

 中心观察是“等于恒定变化”消除了绝对值，只留下相对结构。 如果我们减去连续的元素，常数移位就会完全消失。 

对于任意数组$A$，定义其差异表示：$$D_A[i] = A[i+1] - A[i]$$当且仅当两个数组的差异数组相同时，两个数组才是移位等效的。 这将问题转化为模式匹配任务：我们不再匹配值，而是匹配差异序列。 

蛮力解决方案将迭代每个子数组$N$长度$M$，计算其差异序列，并将其与$M$。 每次比较的成本$O(M)$，并且有$O(N)$候选人，所以总复杂度是$O(NM)$。 当两个数组都到达时这太慢了$10^6$。 

一旦问题简化为匹配两个序列，它就变成了标准的子字符串搜索问题：我们需要计算一个数组在另一个数组中出现的次数。 这可以使用滚动哈希或前缀函数 (KMP) 在线性时间内解决。 结构很重要：我们在大文本中匹配固定模式，而字母表是任意整数，因此散列是很自然的选择。 

将两个数组转换为其差异形式后，我们在文本差异数组内搜索模式差异数组的出现情况。 每个匹配对应于原始数组中的一个有效子数组。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(NM)$|$O(1)$额外 | 太慢了|
 | 差异+KMP/哈希|$O(N)$|$O(N)$| 已接受 |

 ## 算法演练

 1.如果$M = 1$， 返回$N$。 任何单个元素子数组都可以移动以匹配单个值，因此每个位置都是有效的。 
2. 构建差异数组$M$，其中每个条目是$m[i+1] - m[i]$。 这对其结构进行编码，与绝对值无关。 此步骤将图案尺寸减小到$M-1$。 
3. 构建差异数组$N$,同样计算$n[i+1] - n[i]$。 这将问题转化为查找一个序列在另一个序列中出现的情况。 
4.如果$M > N$， 返回$0$。 较长的模式无法放入较短的序列中。 
5. 使用线性时间模式匹配算法（例如 KMP）来计算模式差异数组在文本差异数组中出现的次数。 每个匹配对应于原始数组中的有效起始索引。 
6. 返回找到的匹配项数。 

### 为什么它有效

 恒定移位对所有元素的影响相同，因此在取相邻差异时会抵消。 这意味着任何两个仅相差一个常数的数组都会产生相同的差异数组。 相反，如果两个数组具有相同的差异数组，则它们的值必须相差恒定的偏移量，因为从固定的起始值重建会产生相同的相对结构。 这样，匹配差异数组就完全等价于原来的相似条件。 差异空间中的每个有效子数组匹配都与原始数组中的有效移位匹配一一对应。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_diff(arr):
    return [arr[i+1] - arr[i] for i in range(len(arr) - 1)]

def kmp_prefix(p):
    pi = [0] * len(p)
    j = 0
    for i in range(1, len(p)):
        while j > 0 and p[i] != p[j]:
            j = pi[j - 1]
        if p[i] == p[j]:
            j += 1
            pi[i] = j
    return pi

def kmp_count(text, pat):
    if not pat:
        return len(text) + 1

    pi = kmp_prefix(pat)
    j = 0
    count = 0

    for i in range(len(text)):
        while j > 0 and text[i] != pat[j]:
            j = pi[j - 1]
        if text[i] == pat[j]:
            j += 1
        if j == len(pat):
            count += 1
            j = pi[j - 1]

    return count

def solve():
    M, N = map(int, input().split())
    m = list(map(int, input().split()))
    n = list(map(int, input().split()))

    if M == 1:
        print(N)
        return

    if M > N:
        print(0)
        return

    dm = build_diff(m)
    dn = build_diff(n)

    print(kmp_count(dn, dm))

if __name__ == "__main__":
    solve()
```该解决方案首先处理模式长度为 1 或大于文本的退化情况。 转换步骤计算差异数组，将匹配问题缩小为一个超长度问题$M-1$和$N-1$。 

KMP 例程用于有效地计算线性时间内的模式出现次数。 前缀数组对发生不匹配时我们可以安全回退的程度进行编码，从而防止重新检查已经验证的结构。 每当匹配完整模式时，匹配循环就会递增计数器，然后从最长的有效边界继续。 

一个微妙的点是，我们在预处理后根本不比较原始数组。 所有正确性完全取决于差异数组在翻译下保持等价性的事实。

 ## 边缘情况

 对于本案$M = 1$，算法直接返回$N$。 这与差异数组变为空的事实相一致，并且每个位置都简单地匹配一个空模式。 例如，输入$M=[7]$,$N=[1,2,3]$产生 3 个匹配，因为每个单个元素子数组都可以移动到 7。 

对于均匀数组，例如$M=[5,5,5]$和$N=[1,1,1,1]$，差异数组完全为零。 KMP 过程在零文本中重复匹配零模式，产生$N-M+1$有效匹配。 这证实了该算法正确地处理了差分后值信息完全消失的情况。 

对于不匹配的结构，例如$M=[1,2,3]$和$N=[10,20,30]$，差异数组的不同之处在于$[1,1]$相对$[10,10]$，因此不会发生子字符串匹配，从而按预期产生零。
