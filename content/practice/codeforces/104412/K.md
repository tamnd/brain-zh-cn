 ## 解决方案
 ## 问题理解

 我们有一个大小为的正方形网格$N \times N$，其中每个单元格包含一个从 0 到 9 的数字，表示地形类型。 我们也给出了固定的大小$K$，我们需要检查每一个可能的$K \times K$网格内的子正方形。 

如果子正方形的所有四个角单元都包含相同的值，则该子正方形被视为有效。 任务是数出有多少这样的$K \times K$网格中存在子方格。 

输入约束使网格可能很大，最多可达$1000 \times 1000$。 这立即排除了任何重新检查每个的解决方案$K \times K$在最坏的情况下逐个单元地平方。 每平方的天真检查将检查$K^2$细胞，导致$O(N^2 K^2)$，可以达到$10^{12}$操作时$N = K = 1000$，远远超出了可行的限度。 

一个更微妙的问题是重叠计算。 邻近的$K \times K$窗户共享其大部分内部空间，但天真的方法会从头开始重新计算所有内容。 

一些边缘情况值得关注。 

如果$K = N$，只有一个方格需要检查，即整个网格。 答案是 1 或 0，具体取决于四个角是否匹配。 

如果网格中的所有值都相同，则每个$K \times K$子方有效，所以答案是$(N-K+1)^2$。 

如果角上只出现一个数字，但内部不同，则错误地检查所有单元格而不是仅角点的天真的解决方案仍然可以在逻辑上工作，但会浪费时间； 然而，有缺陷的解决方案可能会意外地包含内部约束，这是错误的，因为只有角落很重要。 

## 方法

 蛮力方法很简单。 我们迭代每个可能的左上角$(i, j)$的一个$K \times K$子广场。 对于每一个，我们明确地比较所有$K^2$细胞或至少穿过边界。 即使我们稍微优化并只检查四个角，蛮力仍然需要这样做$(N-K+1)^2$的位置，最多是$10^6$。 那部分很好。 仅当我们错误地验证完整方块时，真实成本才会出现。 

因此，关键的观察结果是，条件仅取决于四个位置：左上、右上、左下、右下。 广场内的其他一切都无关紧要。 这将问题分解为每个位置的恒定时间检查。 

因此，我们不需要扫描网格或维护前缀结构，而是简单地迭代所有有效的起点并比较四个单元格。 

该问题从 2D 几何聚合任务简化为每个窗口工作量恒定的简单滑动枚举问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（全方格检查）|$O(N^2 K^2)$|$O(1)$| 太慢了|
 | 仅角球检查 |$O(N^2)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 阅读$N$和$K$，然后将网格存储在二维数组中。 这是必要的，因为我们在检查角点时需要随机访问任何单元格。 
2. 迭代 a 的所有可能的左上角位置$K \times K$正方形。 这些职位是$(i, j)$在哪里$0 \le i \le N-K$和$0 \le j \le N-K$。 这些边界确保正方形保持在网格内。 
3. 对于每个位置$(i, j)$，确定其四个角：$(i, j)$,$(i, j+K-1)$,$(i+K-1, j)$， 和$(i+K-1, j+K-1)$。 这些是唯一对有效性重要的单元格。 
4. 检查四个角值是否相等。 如果是，则增加答案。 
5. 处理完所有位置后，输出累计计数。 

关键的设计选择是将检查仅限于角落。 这是有效的，因为条件从不引用内部单元格，因此任何额外的计算都是多余的。 

### 为什么它有效

 每个有效的位置完全由四个固定位置的本地条件定义。 每个$K \times K$square 由其左上角唯一确定，有效性谓词仅取决于距该锚点固定偏移处的值。 由于我们对每个锚点只评估一次并应用定义有效性的精确条件，因此不会错过任何有效方块，也不会计算无效方块。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(N)]
    
    ans = 0
    
    for i in range(N - K + 1):
        for j in range(N - K + 1):
            a = grid[i][j]
            b = grid[i][j + K - 1]
            c = grid[i + K - 1][j]
            d = grid[i + K - 1][j + K - 1]
            if a == b == c == d:
                ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 嵌套循环枚举所有有效的左上角。 四个角落访问是恒定时间数组查找，确保内部循环保持不变$O(1)$。 

主要的微妙之处在于索引：右下角位于$i + K - 1, j + K - 1$， 不是$i + K, j + K$。 这里的相差一错误是最常见的错误来源。

 ## 边缘情况

 当$K = N$，恰好有一个候选方格。 该算法评估单次迭代$(i, j) = (0, 0)$。 它直接比较整个网格的四个角，只有匹配时才返回 1。 不会发生越界访问，因为$i + K - 1 = N - 1$，这是有效的索引。 

当所有网格值都相同时，每次迭代都满足条件。 The loops run over$(N-K+1)^2$positions, and each comparison returns true. 最终答案等于可能放置的数量，与子方块的组合计数相匹配。 

当只有内部单元不同但角匹配时，算法仍然可以正确计数，因为它从不检查内部位置。 For example, in a$3 \times 3$正方形：```
1 1 1
1 9 1
1 1 1
```一个$3 \times 3$窗口具有匹配的角点（全部为 1），因此无论中心 9 是什么，它都会被计数一次。这证实了限制对角点的关注对于正确性来说既是充分的也是必要的。
