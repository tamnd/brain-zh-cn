 ## 解决方案
 ## 问题理解

 我们得到一个很长的数字字符串，我们可以将它分成几个不相交的连续片段。 每个选定的部分都被解释为十进制数，并且仅当该数字是素数或完全平方数时才被视为有效。 我们不需要覆盖整个字符串。 选择棋子后未使用的任何数字将被完全忽略。 目标是挑选尽可能多的有效片段而不让它们重叠。 

因此，任务不是将整个字符串划分为有效的块，而是在字符串内的任何位置选择最大数量的不重叠的“好”子字符串。 

输入大小立即改变了我们的想法。 该字符串的长度最多可达一百万位，因此任何尝试枚举所有子字符串的解决方案都是不可能的。 检查每个子串的简单方法将产生大约 K² 候选，在最坏的情况下大约是 10^2 次操作。 即使仅检查子字符串的子集仍然太慢，除非对有效子字符串的大小有严格的限制。 

一个关键的结构含义是我们只关心实际上可以是素数或完美平方的子串。 这迫使有效子串在任何实际解决方案中都相对较短，因为对极大整数进行素性测试和平方检查在竞赛环境中是不可行的。 这导致了预期的减少：只有小于固定长度的子串才是相关的候选者。 

有一些边缘情况打破了天真的推理。 

第一个陷阱是假设我们必须对整个字符串进行分区。 例如，在`687`，获取所有数字是不可能的，但只能选择`7`是有效且最优的。 

第二个陷阱是假设贪婪的从左到右选择总是有效。 例如，在`10067`, 采摘`1`,`0`,`0`贪婪地在局部给出更多的棋子，但最优解是`100 | 67`，只有当我们考虑更长的线段时才可见。 

第三个陷阱是忘记存在重叠的候选者。 字符串`10067`包含两者`10`和`100`从相同的位置开始，只有对选择进行全局优化才能产生正确的答案。 

## 方法

 暴力策略是考虑每个子串，检查其数值是否为素数或完全平方数，收集所有有效间隔，然后运行最大组的不重叠间隔。 这在原则上是正确的，因为每个有效答案都是由这样的区间组成的。 然而，枚举所有子串的成本已经是 O(K²)，并且当 K 高达 10⁶ 时，这变得完全不可行。 即使存储间隔也会超出内存。 

关键的观察结果是，有效数字必须来自极少数候选人。 如果我们预先计算所有素数和完全平方数达到固定上限（通常是所有适合少量数字的值），则每个有效子字符串的长度必须最多为该上限。 这将问题从任意长度子字符串检查转变为有界窗口扫描。 

一旦子串长度受到常数 L（通常约为 6 到 7 位）的限制，该结构就变得易于管理。 对于每个起始位置，我们仅尝试扩展到 L 个字符，计算数字，并检查预先计算的有效素数和平方集合中的成员资格。 

生成所有有效间隔后，问题就变成选择一条线上不重叠间隔的最大数量，其中每个间隔具有相等的权重。 这可以通过从左到右的简单动态编程扫描来解决，在每个位置我们决定是跳过还是采用从那里开始的有效子字符串。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子串+区间调度| O(K²) | O(K²) | 太慢了|
 | 具有预先计算的有效数的有界长度 DP | O(K·L) | O(K) | 已接受 |

 ## 算法演练

 ### 预计算步骤

 1. 预先计算所有完全平方数，直到所选数字限制 L 的最大可表示值。这是通过迭代 i² 并将结果存储在哈希集中来完成的。 这允许稍后进行恒定时间平方检查。 
2. 使用筛子预先计算所有素数，使其达到相同的数值范围。 将它们存储在同一个哈希集或单独的集合中，然后合并两个源。 这确保了任何候选子串都可以在 O(1) 内得到验证。 

限制数字范围的原因是永远不会考虑任何长于 L 位数字的子字符串，因此永远不需要测试更大的值。 

### 提取有效区间

 1. 对于字符串中的每个索引 i，开始逐字符扩展数字直至长度 L，形成子字符串 N[i..j]。 
2. 将每个子字符串增量地转换为整数，而不是从头开始重新解析。 这避免了重复的开销。 
3. 如果形成的数字存在于预先计算的有效集合中，则将区间 (i, j) 存储为有效选择。 

此步骤将字符串转换为候选片段的集合，其中每个片段都是独立有效的。 

### 动态规划选择

 1. 定义 dp[i] 为我们可以从位置 i 开始选取的最大有效段数。 
2. 从右向左遍历字符串。 在每个位置 i，首先假设我们跳过它，因此 dp[i] = dp[i+1]。 
3. 对于从 i 开始的每个有效间隔，用 dp[j+1] + 1 更新 dp[i]，其中 j 是间隔的结尾。 

这确保我们要么忽略当前字符，要么采用从这里开始的有效段之一。 

### 为什么它有效

 该算法依赖于这样一个事实：每个决策点仅取决于未来的位置。 一旦我们修复了从 i 开始的有效段，问题的其余部分就变得与所选段无关。 因为所有段都具有相同的权重，所以我们永远不需要选择一个有效的子字符串而不是另一个，除非通过最大化计数。 DP 保证非重叠有效间隔的每种可能组合在转换过程中都被恰好考虑一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXL = 6
LIMIT = 10**6  # enough for primes/squares in typical intended range

# Sieve primes up to LIMIT
is_prime = [True] * (LIMIT + 1)
is_prime[0] = is_prime[1] = False
for i in range(2, int(LIMIT ** 0.5) + 1):
    if is_prime[i]:
        step = i
        start = i * i
        for j in range(start, LIMIT + 1, step):
            is_prime[j] = False

valid = set()

# squares
i = 1
while i * i <= LIMIT:
    valid.add(i * i)
    i += 1

# primes
for i in range(LIMIT + 1):
    if is_prime[i]:
        valid.add(i)

K = int(input().strip())
s = input().strip()

n = len(s)

# dp[i] = best answer from i to end
dp = [0] * (n + 1)

for i in range(n - 1, -1, -1):
    best = dp[i + 1]
    num = 0
    for j in range(i, min(n, i + MAXL)):
        num = num * 10 + (ord(s[j]) - 48)
        if num in valid:
            best = max(best, 1 + dp[j + 1])
    dp[i] = best

print(dp[0])
```该解决方案首先为所有有效的数字构建一个快速查找结构。 这从主循环中删除了素数和平方检查，并用恒定时间隶属度测试代替。 

DP 是从右到左进行的，因此在评估 dp[i] 时 dp[j+1] 是已知的。 每个位置都会尝试跳过或采用从那里开始的任何有效子字符串。 内部循环以 MAXL 为界，因此总工作与字符串大小保持线性关系。 

一个微妙的细节是增量构建`num`。 从切片重新计算整数每个子字符串的成本为 O(L)，但逐位累加则每个扩展的成本为 O(1)。 

## 工作示例

 ### 示例 1：`687`我们计算长度最大为 6 的有效子串。 

| 我| 尝试过子字符串 | 有效的选择| dp[i] | dp[i] |
 | --- | --- | --- | --- |
 | 0 | 6、68、687 | 无 | dp[1] | dp[1] |
 | 1 | 8, 87 | 8, 87 无 | dp[2] | dp[2] |
 | 2 | 7 | 7 | 1 |

 在索引 2 处，我们可以取`7`，给出 dp[2] = 1。在较早的位置，没有有效的段开始，因此最佳值仍为 1。 

这表明，在出现有效的隔离段之前，跳过字符可能是最佳选择。 

### 示例 2：`10067`| 我| 尝试过子字符串 | 有效的选择| dp[i] | dp[i] |
 | --- | --- | --- | --- |
 | 0 | 1、10、100 | 100 | 100 1 + dp[3] | 1 + dp[3] |
 | 3 | 6, 67 | 6, 67 67 | 67 1 + dp[5] | 1 + dp[5] |
 | 5 | 7 | 7 | 1 |

 从索引 3 中我们取`67`，从索引 0 开始我们取`100`。 DP 将这两个不重叠的段链接起来。 

这表明即使存在较小的分割，较长的分割也如何主导较小的分割。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(K·L) | 每个位置扩展至 L 位 |
 | 空间| O(K + 极限) | dp 数组加上预先计算的素数/平方 |

 约束条件允许 K 高达一百万，因此具有较小常数因子的线性扫描就足够了。 预处理成本与 K 无关并且在限制范围内，因为它仅取决于用于有效数字的数字界限。

 ## 边缘情况

 对于像这样的单位数字输入`7`，算法立即找到从索引 0 开始的有效子字符串，并将 dp[0] 设置为 1。没有进一步的转换，因此结果是正确的。 

对于像这样的字符串`1234`，根据预先计算的集合，只有少数短子串可能是有效的。 DP 确保即使多个有效子串重叠，也仅采用最佳的非重叠选择，因为 dp[i] 始终比较跳过与采用有效间隔。 

为了`100100`, 子串`100`以不相交的位置出现两次。 DP 独立选择这两个事件，因为在获取第一个事件之后`100`，它从正确的下一个索引继续，自动保持不重叠。 

这些案例证实状态定义仅取决于后缀最优性，因此早期的选择永远不会使后来的最优段失效。
