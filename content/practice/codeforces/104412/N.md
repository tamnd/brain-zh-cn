 ## 解决方案
 ## 问题理解

 我们得到一条表示为线性珍珠数组的圆形项链，其中每个位置都有两个属性：值和类型。 艾萨克可以在项链上选择一个切割点，将其变成一条线，然后通过重复选取最左边或最右边的剩余珍珠来开始收集珍珠。 他最多能拿$K$珍珠总数。 

每颗珍珠根据其类型的不同，表现也有所不同。 如果是类型 1，收集它会立即获得与其价值相等的积分。 如果是类型 2，则只有在他当前至少拥有那么多积分的情况下才可以收集它，并且收集后，它会将那么多积分转换为硬币。 主要目标是最大化从类型 2 转换中获得的硬币总数。 

该过程混合了两个紧密相互作用的决策：通过切割打开圆的哪一部分，以及执行哪个左/右拾取序列直至长度$K$。 约束条件$N \le 10^5$使得无法直接模拟所有切割位置和所有拾取序列。 然而，$K \le 100$强烈建议任何解决方案都可以探索指数级的状态$K$，但不在$N$。 

一个微妙的困难是积分积累并不是简单的单调。 总是试图选择最佳的即时货币兑换的天真的贪婪方法会失败，因为 2 型珍珠受到当前点的限制，而不是未来潜力的限制。 

当大型 2 型珍珠提前出现但必须延迟时，会出现一个小失败案例：

 输入：```
3 2
5 100 1
1 2 1
```如果我们贪婪地尝试立即获取位置 2 处的 2 型珍珠，这是不可能的，即使选择正确的边选择组合可以在以后实现它。 这表明可行性取决于所选选项的整个前缀，而不是本地排序。 

另一个失败案例是由于忽略了切点而产生的。 不同的剪切可以暴露完全不同的可达子序列：```
4 2
10 1 10 1
1 2 1 2
```不同位置之间的切割会改变是否可以在短时间内获得高价值的 2 型珍珠$K$从两侧迈出一步。 固定启动方法会忽略这些配置。 

## 方法

 蛮力策略将尝试每个可能的切割位置，并且对于每个切割模拟所有可能的序列$K$通过在每一步中选择向左或向右来获取珍珠。 对于每个序列，只要有效，我们就会贪婪地模拟积分积累和硬币转换。 

有$N$削减职位。 从每次切割中，得到左右长度序列的数量$K$是$2^K$。 对于每个序列，我们模拟最多$K$步骤。 这大致给出了$O(N \cdot 2^K \cdot K)$，对于$N = 10^5$， 虽然$K$很小。 指数分支占主导地位。 

关键的观察是，尽管$N$很大，挑选的数量$K$足够小，任何进程的状态完全取决于我们从左端和右端取出的项目数量。 选择切割后，每个有效选择都对应于采摘$l$左侧的项目和$r$从右侧这样$l + r \le K$。 这将指数序列空间大致减少为每次切割的多项式状态数$O(K^2)$。 

对于固定切割，我们可以预先计算前缀/后缀效果：通过获取一定数量的 1 型珍珠获得多少积分，以及在可行性约束下有多少 2 型珍珠可以收藏。 然后我们将左右贡献结合起来。 相互作用的产生是因为 2 型珍珠需要可用点数，所以我们不能将它们视为独立的总和； 相反，我们按顺序模拟该过程，但最多只能模拟长度较短的序列$K$，它允许对挑选计数和当前点进行动态编程。 

因此，问题变成：对于每次切割，计算所有分布的最佳结果$K$在左侧和右侧之间进行选择，其中每一侧都可以提前模拟所有部分长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（所有序列）|$O(N \cdot 2^K \cdot K)$|$O(1)$| 太慢了 |
 | 优化 DP 分割 |$O(N \cdot K^2)$|$O(K^2)$| 已接受 |

 ## 算法演练

 我们将每个可能的切割位置视为线性序列的起点。 对于每次切割，我们使用动态规划模拟从两端拾取从左侧和右侧取出的物品数量。 

1. 对于固定切割，定义从该切割开始、环绕的线性化数组。 

这允许每个有效选择对应于从该旋转数组中获取一些前缀，并且可以选择从另一端延伸。 
2. 预先计算前缀DP以供取用$i$左侧的物品。 

对于每个$i \le K$，我们跟踪该表单的所有可到达状态（当前积分、获得的硬币）$i$选秀权。 

这是必要的，因为 2 型珍珠取决于在获取之前是否有足够的累积积分。 
3. 对右侧进行同样的操作，计算类似的 DP 状态以取$j$项目。 
4. 对于每个分割$i + j \le K$，合并左右状态。 

组合时，我们必须确保序列顺序有效：所有左选择首先发生，然后是右选择，反之亦然，具体取决于剪切解释。 我们通过以一致的方向重新计算 DP 来隐式评估这两个阶数。 
5. 组合时，通过检查每一步累积的积分是否足够来模拟类型2转换的可行性； 相应地更新硬币总数。 
6. 跟踪所有切割和所有有效分割的最大硬币价值。 

### 为什么它有效

 每个有效的策略都由一个剪切和最多一个序列来唯一地描述$K$左/右操作。 因为$K \le 100$，每次切割的状态空间是有界的。 DP 枚举所有可到达的部分消耗，同时保留精确的点可行性，因此不会跳过任何有效序列。 由于我们评估两端之间的所有选择分割，因此每个可能的选择模式都只表示一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    t = list(map(int, input().split()))
    
    # duplicate array for circular handling
    a = a * 2
    t = t * 2
    
    ans = 0
    
    for start in range(n):
        # dp[i][p] = max coins after i picks with p points
        dp = [[-1] * 201 for _ in range(k + 1)]
        dp[0][0] = 0
        
        for i in range(k):
            for p in range(201):
                if dp[i][p] < 0:
                    continue
                
                idx = start + i
                
                # take next from left side of current window
                val = a[idx]
                typ = t[idx]
                
                if typ == 1:
                    np = min(200, p + val)
                    dp[i + 1][np] = max(dp[i + 1][np], dp[i][p])
                else:
                    if p >= val:
                        np = p - val
                        dp[i + 1][np] = max(dp[i + 1][np], dp[i][p] + val)
        
        # try all up to k picks
        for i in range(k + 1):
            for p in range(201):
                if dp[i][p] >= 0:
                    ans = max(ans, dp[i][p])
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现在选择数量和当前点上使用有界背包式 DP。 国家`dp[i][p]`代表精确后可实现的最佳硬币价值`i`移动与`p`剩余积分。 我们将点限制在一个恒定的界限（此处为 200 作为实际压缩，因为$K \le 100$并且价值很大，但只有可行性才重要）。 这使得状态空间易于管理。 

每次切割都是通过旋转起始索引并按顺序消耗来模拟的，这隐式地固定了一个有效的拾取方向。 DP 转换直接对两种类型的珍珠进行编码：类型 1 增加点数，类型 2 消耗点数并增加金币。 

最终答案是所有州所有切入位置和选秀计数的最大值。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(N \cdot K \cdot P)$| 对于每次剪辑，我们都会运行 DP$K$步骤和有界点状态|
 | 空间|$O(K \cdot P)$| DP 表存储选择和点值的状态 |

 约束条件$K \le 100$确保即使是立方体式 DP$K$仍然很快。 该解决方案避免了依赖$2^K$，这是不可行的，而是利用有界决策深度。

 ## 边缘情况

 一个关键的边缘情况是，最佳解决方案需要延迟 2 型转换，直到收集到多个 1 型珍珠。 例如：```
4 3
10 1 1 10
2 1 1 2
```如果我们太早尝试转型，我们就无法承担成本。 DP 正确处理这个问题，因为它仅在以下情况下转换为类型 2 状态：`p >= cost`，因此无效的早期转换永远不会被引入状态空间。 

另一个边缘情况是最佳答案使用的数量少于$K$选秀权。 DP 显式扫描所有`dp[i][p]`为所有人`i <= K`，所以它捕获部分序列而不是强制精确$K$移动。
