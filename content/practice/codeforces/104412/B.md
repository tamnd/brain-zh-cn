 ## 解决方案
 ## 问题理解

 所描述的过程是一个随机排序过程，它反复对数组进行洗牌，直到它恰好被排序。 一次“成功迭代”意味着在一次随机洗牌之后，所得的排列已经排序。 

一个关键的观察是，每次洗牌都会产生多重值集的均匀随机排列，而不仅仅是索引的不同排列。 如果我们想要直接落在排序排列中的概率，我们实际上是在问：在数组元素的所有不同排列中，哪个分数对应于排序顺序。 

如果所有元素都不同，则恰好存在一种排序排列$N!$，所以概率是$1/N!$。 当存在重复项时，许多索引排列表示相同的值排列。 如果一个值$x$出现$f_x$次，则不同排列的数量为$$\frac{N!}{\prod f_x!}$$仅对这些排列之一进行排序，因此概率变为$$\frac{1}{N! / \prod f_x!} = \frac{\prod f_x!}{N!}.$$任务是输出这个值模$10^9+7$，然后在一个数组位置改变值的点更新下维护它。 每次更新仅更改两个频率：一个值丢失一次出现，另一个值增加一次，因此可以增量更新概率。 

约束条件允许两者$N$和$K$最多$10^6$，这会立即排除每个查询从头开始重新计算阶乘积。 任何每次更新扫描所有值的解决方案都会太慢，因为这会导致$O(NK)$最坏情况下的行为，远远超出了可行的限度。 

一个微妙的问题是值可以达到$10^9$，因此直接按值索引是不可能的。 除非值被压缩或存储在哈希映射中，否则任何依赖于按值键控的数组的方法都会默默地失败。 

当所有元素都相同时，就会出现边缘情况。 那么概率应该是$1$，因为每个排列都是相同的并且已经排序。 另一个极端情况是当重复更新交换值时导致频率振荡； 正确性取决于维护基于频率的贡献，而不是明确地跟踪排列。 

## 方法

 强力解释将通过重建频率图并评估每次更新后重新计算完整概率$$\frac{\prod f_x!}{N!}.$$计算每个查询的频率阶乘仍然是可以接受的，但是重新计算频率本身需要在每次更新后扫描整个数组，即$O(N)$每个查询。 高达$10^6$更新，这变成$10^{12}$在最坏的情况下进行操作，这是不可行的。 

关键的见解是概率仅取决于多重集频率分布。 当单个元素变化时，只有两个频率变化±1，全局乘积$\prod f_x!$可以在本地更新。 这将问题转化为在点更新下维护动态乘积，如果预先计算阶乘，则可以在每次更新的恒定时间内完成。 

我们还预先计算阶乘和模逆，以便除以$N!$全局处理一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 重新计算每个查询 |$O(NK)$|$O(N)$| 太慢了|
 | 维护频率产品|$O(N + K)$|$O(N)$| 已接受 |

 ## 算法演练

 我们将概率重写为：$$\text{answer} = \left(\prod_{v} f_v!\right) \cdot (N!)^{-1}.$$我们动态地维护这个表达式。 

### 步骤

 1. 预先计算阶乘和逆阶乘$N + K$。 这是必要的，因为任何频率都可以增长到$N$，并且阶乘值必须在 O(1) 时间内可用。 
2. 构建初始数组值的频率图。 对于每个值，增加其计数。 
3. 计算所有值的初始乘积$f_v!$。 这代表概率公式的分子。 
4. 计算常数分母因子$(N!)^{-1}$一旦使用预先计算的逆阶乘。 
5. 对于每次更新，从乘积中删除旧值频率的贡献，更新频率，然后重新插入新的阶乘贡献。 产品始终与当前频率保持一致。 
6、每次更新后，输出：$$\text{product} \cdot (N!)^{-1} \bmod (10^9+7).$$### 为什么它有效

 该算法保持了精确值$\prod f_v!$随时。 由于更新仅改变两个频率，因此产品调整是本地且精确的。 分母$N!$保持不变，因为数组大小不会改变。 这确保了维护的表达式始终等于从排列计数参数导出的概率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    N, K = map(int, input().split())
    arr = list(map(int, input().split()))

    maxn = N + K + 5

    fact = [1] * (maxn)
    invfact = [1] * (maxn)

    for i in range(1, maxn):
        fact[i] = fact[i - 1] * i % MOD

    invfact[maxn - 1] = pow(fact[maxn - 1], MOD - 2, MOD)
    for i in range(maxn - 2, -1, -1):
        invfact[i] = invfact[i + 1] * (i + 1) % MOD

    freq = {}
    prod = 1

    for x in arr:
        if x not in freq:
            freq[x] = 0
        freq[x] += 1

    for v in freq.values():
        prod = prod * fact[v] % MOD

    inv_n_fact = invfact[N]

    out = []

    def remove(val):
        nonlocal prod
        c = freq[val]
        prod = prod * invfact[c] % MOD
        c -= 1
        freq[val] = c
        prod = prod * fact[c] % MOD

    def add(val):
        nonlocal prod
        c = freq.get(val, 0)
        if val not in freq:
            freq[val] = 0
        prod = prod * invfact[c] % MOD
        c += 1
        freq[val] = c
        prod = prod * fact[c] % MOD

    for _ in range(K):
        A, B = map(int, input().split())
        A -= 1
        old = arr[A]

        remove(old)
        arr[A] = B
        add(B)

        out.append(str(prod * inv_n_fact % MOD))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案的核心是维护代表所有因子贡献的单一全局产品。 删除和添加助手确保对称地应用每个频率变化：在递减之前删除旧的阶乘项，并在调整后插入新的阶乘项。 这种排序可以防止过时的贡献留在产品中。 

阶乘和逆阶乘表保证所有更新都是 O(1)，并且模逆$N!$被预先计算一次$N$永远不会改变。

 ## 边缘情况

 当所有元素相同时，频率图包含一个带有值的键$N$。 产品变成$N!$，并乘以$(N!)^{-1}$产生 1。该算法自然地处理这个问题，因为删除和添加操作总是使阶乘积与当前频率保持一致。 

当更新反复来回交换两个值时，每个值的频率在两个状态之间振荡。 每个转换都应用对称删除和添加操作，因此全局产品中不会累积任何漂移。 

当一个值完全消失时，它的频率变为零。 删除步骤通过除掉最后一个阶乘贡献来减少乘积，并且因为$0! = 1$，产品中不存在无效阶乘项。
