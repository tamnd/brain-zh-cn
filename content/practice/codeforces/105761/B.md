 ## 解决方案
 ## 问题理解

 我们得到一对起始的个位数，其中至少有一个非零。 从这对中，我们生成一个序列，其中每个下一个值的形成与斐波那契完全相同：前两个值的总和。 唯一的区别是我们不允许数字超过一位数，因此每当总和达到 10 或更多时，我们只保留最后一位数字。 

这创建了一个从初始有序对演变为有序对序列的确定性过程。 在每一步中，系统的状态都由最后两个数字完整描述。 任务是确定序列运行多长时间，直到我们在连续位置再次遇到原始起始对，这意味着系统已返回到其初始状态。 

尽管序列看起来是数字，但真正的结构是对 (a, b) 上的转换系统，其中每个步骤移动到 (b, (a + b) mod 10)。 由于只有 100 种可能的状态，因此可以很快保证重复。 

这些约束足够小，任何探索多达 100 种可能性的完整状态空间的方法都可以在限制内轻松运行。 即使是跟踪所有访问状态的简单模拟也是有效的恒定时间。 

一个微妙的边缘情况是序列立即以非常短的循环循环。 例如，从 (1, 0) 开始，接下来的状态是 (0, 1)、(1, 1)、(1, 2) 等，与直觉相比，返回可能会相对较晚发生。 另一个边缘情况是对称开始，如 (5, 5)，其中递归很快稳定为短周期。 仅检查单个值而不是对的重复的天真的尝试将会失败，因为问题明确在于与原始状态匹配的连续对。 

## 方法

 理解该过程的直接方法是完全按照定义对其进行模拟。 我们从给定的对开始，重复计算下一个值作为总和的最后一位数字。 在每一步中，我们都会向前移动该对并检查是否已返回到初始配置。 

这种强力模拟是正确的，因为它反映了序列的定义而无需任何转换。 然而，序列是根据状态而不是个体数字而演变的。 每个组件只有 10 个可能的值，因此总共只存在 100 个状态。 这意味着最多 100 次转换后，某些状态必须重复。 因此，暴力方法最多探索一个恒定大小的图。 

关键的观察结果是，我们正在处理的不是无界的斐波那契式增长问题。 相反，我们正在处理一个有限状态机。 一旦我们认识到递归仅取决于最后两个值模 10，问题就简化为找到从初始状态开始的循环长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 每次测试 O(100) | O(1) | O(1) | 已接受 |

 两种描述都导致相同的实现，因为状态空间很小，不需要进一步优化。 

## 算法演练

1. 读取起始对 (a, b)，它定义了系统的初始状态。 这种状态很重要，因为我们在连续位置返回到该状态时进行测量。 
2. 将初始状态对存储为我们要再次检测的目标状态。 
3. 初始化一个计数器来跟踪到目前为止序列中出现了多少个数字。 我们从已经存在的两个数字开始，因此初始计数为 2。 
4. 将下一个值计算为 (a + b) mod 10，然后将该对向前移动到 (b, next_value)。 此步骤对递归规则进行编码，同时强制执行单位数约束。 
5. 生成每个新数字后递增计数器，因为每个新状态都会向序列贡献一个附加元素。 
6. 继续重复转换，直到当前对与原始起始对匹配。 当这种情况发生时，序列就完成了一个完整的循环。 
7. 输出计数器，该计数器代表序列的长度，直到并且包括初始对的第一次出现。 

### 为什么它有效

 该过程完全通过有序对的转换而演变，并且每次转换都是确定性的。 这意味着系统定义了一个有向图，其中每个节点只有一个出边。 从这样一个有限图中的任何节点开始都保证了最终的重复。 因为我们仅在初始节点重新出现时才明确停止，所以该过程恰好测量从该节点开始的一个完整周期长度。 中间重复不能提前终止，因为我们比较完整的对，而不是单个值，从而保持状态完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b = map(int, input().split())
    start = (a, b)

    count = 2
    while True:
        c = (a + b) % 10
        a, b = b, c
        count += 1
        if (a, b) == start:
            print(count)
            return

if __name__ == "__main__":
    solve()
```该实现仅保留当前对和计数器。 过渡线`(a + b) % 10`直接执行 Fiborooji 规则。 循环继续，直到状态再次与初始对匹配，此时我们已经完成了一个完整的循环。 计数器从 2 开始，因为在任何转换发生之前输入已经贡献了两个元素。 

这里的一个常见错误是从 0 或 1 开始计数器，这会将最终答案错误地偏移初始状态大小。 另一个微妙的点是我们必须比较成对的值，而不是单个值，因为循环条件同时取决于前面的两个数字。

### 示例 2：输入 0 6

 从 (0, 6) 开始，演化的主导因素是零在序列早期充当稳定器，但系统稍后仍会进入完整的循环。 应用相同的状态机逻辑，并且序列最终返回到 (0, 6)，从而确认了递归的循环性质。

 ## 边缘情况

 对于像 (0, 6) 这样的开始，序列最初的行为就像一个移位的斐波那契链，其中有许多零影响早期的转换。 该算法仍然跟踪完整的对，因此即使各个值提前重复，在精确的对 (0, 6) 重新出现之前也不会发生终止。 循环正确地继续，直到完全状态匹配。 

对于像 (5, 5) 这样的对称开始，下一个状态变成 (5, 0)，这很快就会破坏对称性。 即使两个起始值相同，状态机也会立即发散，并且仅在完成其循环的完整遍历后才返回到初始对。 该实现自然地处理了这个问题，因为它总是比较有序对而不是独立地比较值。
