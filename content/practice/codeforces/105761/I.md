 ## 解决方案
 ## 问题理解

 我们给出一个整数序列和一个阈值`k`。 从序列中，我们想要选择一个子序列，这意味着我们保留一些元素而不改变它们的顺序，并且我们尝试使其尽可能长。 约束是针对连续选择的元素：每当我们在子序列中选择两个相邻元素时，它们的绝对差必须至少为`k`。 

所以如果我们选择值`x1, x2, x3, ...`，那么对于每个`i`，我们必须有`|xi - x(i+1)| >= k`。 任务是计算这样一个子序列的最大可能长度。 

输入大小最多可达 300,000 个元素，值最大可达 10^9。 这立即排除了对索引的任何二次方法，因为在最坏的情况下 O(n^2) 约为 9e10 次操作，这远远超出了限制。 即使 O(n sqrt n) 也太慢了。 我们应该期望更接近 O(n log n) 或 O(n) 的结果。 

该结构在索引空间中不是局部的，而是取决于值差异，这暗示我们需要以允许快速查询的方式维护先前看到的值的全局视图，例如“我们可以用这个值扩展的最佳子序列是什么，同时尊重 k-gap 条件”。 

一些很容易被忽视的边缘情况：

 如果所有值都相同并且`k > 0`，则不能成对选择两个相邻元素，因此即使数组很长，答案也是 1。 例如，输入`5 10`带数组`7 7 7 7 7`应该输出`1`。 

如果`k = 0`，约束消失，所以答案总是`n`。 仍然强制执行严格不平等或使用不正确的比较逻辑的幼稚解决方案可能会失败。 

如果值的间隔刚好在阈值附近，则贪婪决策或局部决策可能会失败。 例如，早期选择一个大值可能会阻止许多未来的选择，即使较小的起始选择会导致更长的子序列。 

## 方法

 暴力解决方案尝试计算在每个位置结束的最长有效子序列。 对于每个索引`i`，我们扫描所有之前的索引`j < i`并检查是否`|a[i] - a[j]| >= k`, 更新`dp[i] = max(dp[j] + 1)`。 这是正确的，因为它明确测试了所有有效的前任。 然而，这是 O(n^2)，并且当 n = 300,000 时，它会导致大约 9e10 次比较，这是不可行的。 

瓶颈在于，对于每个元素，我们都会重复扫描所有先前的元素，即使我们只关心那些值“兼容”的元素中的最佳 dp 值。 转换条件仅取决于值差异，而不取决于索引结构。 这表明我们应该按价值重新组织信息。 

关键的观察是对于当前值`x`，有效的先前元素是那些值最多的元素`x - k`或者至少`x + k`。 我们希望在这两个值范围内的所有先前处理的元素中获得最佳的 dp 值。 

这将问题转变为维护支持前缀最大查询和后缀最大查询的值的动态结构。 由于值很大（最多 1e9），我们对其进行压缩并在排序的唯一值上使用线段树或 Fenwick 树。 每个位置都存储该精确值所达到的最大 dp 值。 然后，对于每个新元素，我们查询两个范围内的最大值，将它们组合起来，并更新其值位置。 

这减少了从扫描索引到每个元素两个范围最大查询的转换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对指数的暴力 DP | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 协调压缩DP与线段树| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们从左到右处理数组，维护一个数据结构，为每个可能的值存储以该值结尾的最佳子序列长度。 

1. 首先，收集数组中的所有值并对它们进行唯一排序。 这允许我们将每个值映射到压缩坐标系中的索引。 此步骤是必要的，因为原始值对于直接索引来说太大。 
2. 在这些压缩索引上构建线段树，其中每个节点存储迄今为止与该压缩值一起出现的任何数字的最大 dp 值。 最初，所有值均为零。 
3. 对于每个元素`a[i]`,求其压缩位置`p`。 
4. 我们需要可以扩展的最佳子序列`a[i]`。 这来自两个不相交的组：值 ≤`a[i] - k`，且值≥`a[i] + k`。 我们使用对排序的唯一数组进行二分搜索将它们转换为索引范围。 
5. 在线段树中查询两个范围内的最大 dp 值。 最好的前驱结果是这两个结果中的最大值。 
6. 设置`dp = best_prev + 1`。 这表示采用最佳有效子序列并附加当前元素。 
7.更新位置的线段树`p`和`dp`，因为未来的元素可以从此值扩展。 
8. 跟踪所有元素的最大 dp。 

最终的答案就是看到的最大dp值。 

为什么它有效：在每一步，线段树都包含以已处理元素中的任何值结尾的最佳子序列长度。 处理新值时`x`，任何有效的前驱必须完全位于约束定义的两个安全值区间之一内。 由于 dp 已经存储了所有先前前缀的最佳解决方案，因此在这些间隔上取最大值给出了最佳扩展`x`。 未来元素不依赖于索引排序，仅依赖于值的可行性，因此该结构归纳地保留了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class SegTree:
    def __init__(self, n):
        self.n = n
        self.seg = [0] * (4 * n)

    def update(self, idx, val, node=1, nl=0, nr=None):
        if nr is None:
            nr = self.n - 1
        if nl == nr:
            if val > self.seg[node]:
                self.seg[node] = val
            return
        mid = (nl + nr) // 2
        if idx <= mid:
            self.update(idx, val, node * 2, nl, mid)
        else:
            self.update(idx, val, node * 2 + 1, mid + 1, nr)
        self.seg[node] = max(self.seg[node * 2], self.seg[node * 2 + 1])

    def query(self, l, r, node=1, nl=0, nr=None):
        if nr is None:
            nr = self.n - 1
        if r < nl or nr < l:
            return 0
        if l <= nl and nr <= r:
            return self.seg[node]
        mid = (nl + nr) // 2
        return max(
            self.query(l, r, node * 2, nl, mid),
            self.query(l, r, node * 2 + 1, mid + 1, nr)
        )

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    vals = sorted(set(a))
    mp = {v: i for i, v in enumerate(vals)}

    st = SegTree(len(vals))

    ans = 1

    for x in a:
        idx = mp[x]

        left_val = x - k
        right_val = x + k

        import bisect
        best = 0

        li = bisect.bisect_right(vals, left_val) - 1
        if li >= 0:
            best = max(best, st.query(0, li))

        ri = bisect.bisect_left(vals, right_val)
        if ri < len(vals):
            best = max(best, st.query(ri, len(vals) - 1))

        cur = best + 1
        st.update(idx, cur)
        if cur > ans:
            ans = cur

    print(ans)

if __name__ == "__main__":
    solve()
```线段树维护每个压缩值的最大子序列长度。 如果较长的子序列在那里结束，则每次更新只会改进一个位置。 查询分为两个值范围，因为有效转换需要在禁止间隔之外`(x - k, x + k)`。 

一个微妙的实现细节是我们从不直接将 dp[i] 用作索引，而仅用作值。 这是有效的，因为相同值的多次出现共享相同的状态，并且取最大值就足够了，因为后面的出现仅受益于可能最好的先前链。 

另一个重要的细节是处理空查询范围，其中平分结果可能会产生无效的间隔。 这些被安全地视为零贡献。

 ## 边缘情况

 对于像这样的输入`5 10`和`7 7 7 7 7`，该算法将所有值映射到单个压缩索引。 每个查询都返回零，因为两个有效范围都不包括唯一的值。 每个 dp 都会变成 1，并且每次更新都会强化该状态。 最终答案为 1，符合当 k > 0 时有效子序列中不能有两个相等值相邻的约束。 

对于像这样的情况`k = 0`， 例如`1 2 3 4`，左右查询范围始终包含所有先前的值，因为这两个条件`x - k`和`x + k`塌陷到同一点。 每一步都会找到迄今为止的完整最大 dp，产生严格递增的 dp 序列`1,2,3,4`。 

对于 k 周围紧密间隔的值，例如`1 100 2 99 3 98`，结构正确地在小值和大值之间交替。 在每一步中，总是在相反的范围内找到有效的前驱，并且线段树保留迄今为止最好的链，从而充分利用所有元素。
