 ## 解决方案
 ## 问题理解

 我们得到一个仅由小写英文字母组成的字符串。 任务是根据每个不同字母出现的次数对该字符串进行分类。 

如果一个字符串中出现的每个字符都出现奇数次，则该字符串称为奇数字符串。 即使其中出现的每个字符出现偶数次，也会调用字符串。 如果字符串包含不同字符的奇数和偶数频率的混合，则它不属于这两个类别。 

因此，输出是三向分类。 如果字符串是奇数，则打印 1；如果是偶数，则打印 0；如果是混合，则打印 0/1。 

字符串长度最多为 60。这立即意味着任何达到大约 O(n) 甚至 O(26·n) 的解决方案在限制内运行都是微不足道的。 除了单次通过频率计数之外，不需要高级数据结构或优化。 

边缘情况仅在解释时很微妙。 一个幼稚的错误是将“奇数字符串”解释为“总长度为奇数”，将“偶数字符串”解释为“总长度为偶数”。 这是不正确的，因为条件适用于每个字符，而不是全局。 

例如，考虑字符串`aabb`。 总长度为 4，为偶数，但字符数为`a:2`,`b:2`，所以从问题的意义上来说也是如此，并且应该输出`0`。 

另一个例子是`aab`。 总长度为 3，但计数为`a:2`,`b:1`。 这既不是奇数也不是偶数，所以输出是`0/1`。 这里的错误是将其分类为奇数，因为长度是奇数，但偶数频率的存在打破了条件。 

最后，像这样的字符串`abc`所有计数都等于 1，因此它是奇数并且应该输出`1`。 

关键的观察是，我们只需要检查每个字符的频率奇偶性，然后汇总所有字符是否都通过两个统一条件之一。 

## 方法

 解决该问题的直接方法是计算每个字符的出现次数，然后显式检查条件。 由于字母大小固定为 26 个小写字母，因此我们可以维护一个大小为 26 的频率数组并扫描一次字符串。 

对于每个不同的字符，暴力版本将扫描整个字符串以计算其出现次数。 这将花费 O(26·n)，虽然在这里仍然很小，但在概念上是不必要的。 如果字母表更大，效率低下就会变得更加明显，因为重复重新扫描字符串重复项是有效的。 

改进来自于认识到频率计算可以在单次传递中增量完成。 一旦我们有了所有计数，分类就是一个简单的检查：我们测试所有非零计数是否为奇数，或者所有非零计数是否为偶数。 我们还必须确保不会混淆零计数字母，因为它们不应该影响条件。 

这将问题简化为超过 26 个值的恒定时间后处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（重新扫描每个字符）| O(26·n) | O(26·n) | O(1) | O(1) | 已接受但多余 |
 | 最佳（单程频率）| O(n + 26) | O(26) | 已接受 |

 ## 算法演练

 1.初始化一个数组`freq`大小为 26，带零。 每个索引对应一个字母`a`到`z`。 该结构将存储每个字符在输入字符串中出现的次数。 
2. 迭代字符串中的每个字符并增加其相应的频率`freq`。 这将在一次传递中构建完整的分布。 
3. 检查所有非零频率是否都是奇数。 这需要扫描频率阵列并验证每个`freq[i] != 0`满足`freq[i] % 2 == 1`。 
4. 如果步骤 3 成功，则该字符串被分类为奇数，并返回`1`。 
5. 否则，通过验证来检查所有非零频率是否为偶数`freq[i] % 2 == 0`为所有人`freq[i] != 0`。 
6. 如果第 5 步成功，则该字符串被分类为偶数并返回`0`。 
7. 如果两个条件都不成立，则返回`0/1`，表示混合奇偶校验配置。 

### 为什么它有效

 该算法将问题的定义显式编码到频率空间中。 每个字符对最终分类都有独立的贡献，因此唯一相关的属性是每个单独的计数是否满足奇偶校验约束。 由于频率数组捕获精确的计数并且没有操作混合字母之间的计数，因此检查所有条目可以保证正确性。 如果任何单个字符违反统一奇偶校验条件，则整个字符串无法满足该类别。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

freq = [0] * 26

for ch in s:
    freq[ord(ch) - 97] += 1

all_odd = True
all_even = True

for f in freq:
    if f == 0:
        continue
    if f % 2 == 1:
        all_even = False
    else:
        all_odd = False

if all_odd:
    print("1")
elif all_even:
    print("0")
else:
    print("0/1")
```该解决方案首先在线性时间内构造一个频率表。 然后，它对固定大小的字母表执行一次扫描，以确定字符串是否满足奇数条件或偶数条件。 两个布尔标志`all_odd`和`all_even`同时保持以避免重复通过。 零频率将被忽略，因为缺失的字符不会影响定义。 

一个常见的实施陷阱是未能忽略零计数。 将零视为偶数会错误地强制所有字符串为“偶数”，这不是有意的。 检查显式跳过零条目以避免此问题。

## 测试用例```python
import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    s = input().strip()

    freq = [0] * 26
    for ch in s:
        freq[ord(ch) - 97] += 1

    all_odd = True
    all_even = True

    for f in freq:
        if f == 0:
            continue
        if f % 2 == 1:
            all_even = False
        else:
            all_odd = False

    if all_odd:
        return "1"
    elif all_even:
        return "0"
    else:
        return "0/1"

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    return solve()

# provided samples (interpreted)
assert run("geekkeeg\n") == "0/1"
assert run("funnyn\n") == "0/1"
assert run("zztop\n") == "0/1"

# custom cases
assert run("a\n") == "1", "single char is odd"
assert run("aa\n") == "0", "single char even count"
assert run("abcabc\n") == "0", "all even frequencies"
assert run("abc\n") == "1", "all odd frequencies"
assert run("aabbc\n") == "0/1", "mixed parity"
```| 测试输入| 预期产出 | 它验证了什么 |

 |`aa`|`0`| 最小尺寸连壳|
 |`abcabc`|`0`| 均匀偶数频率|
 |`abc`|`1`| 均匀奇数频率|

 ## 边缘情况

 一个关键的边缘情况是某些字母缺失。 例如，在`abc`，频率数组中的大多数条目为零。 该算法在检查期间显式忽略零，因此它们不会错误地强制字符串进入偶数类别。 扫描只能看到`a:1, b:1, c:1`，都是奇数，所以结果是`1`。 

另一种情况是单字符字符串，例如`z`。 频率数组有一个条目等于 1，其余条目为零。 由于 1 是奇数，`all_odd`始终保持正确，输出为`1`。 

最后，像这样的混合情况`aab`证明了维护这两个标志的必要性。 加工后`aab`，频率是`a:2, b:1`。 偶数和奇数的存在都会翻转两者`all_odd`和`all_even`以假乱真，正确生产`0/1`。
