 ## 解决方案
 ## 问题理解

 我们得到了一个类似天际线的城市，由放置在 x 轴顶部的轴对齐的矩形建筑物组成。 每个建筑物从某个 x 坐标开始，向右延伸固定宽度，然后垂直上升到给定高度。 建筑物在区域上永远不会重叠，但它们可能在边缘处接触。 

毛毛虫从地线 y = 0 上的原点出发，必须到达点 (100, 0)。 它只沿着地面、垂直的建筑墙壁和水平的屋顶爬行。 每当它遇到建筑物时，它可能会爬上其左墙，穿过其屋顶，然后沿其右墙下降，当它从左向右移动时，有效地遵循所有建筑物的联合轮廓。 

任务是计算沿着这条步行的总距离，包括水平和垂直移动。 

每个建筑物由起始 x 位置 s、宽度 w 和高度 h 定义。 该建筑物沿 x 轴占据区间 [s, s + w]，从 y = 0 到 y = h。 

关键的观察结果是，毛毛虫的路径恰好是由所有建筑物形成的天际线的上层，加上不存在建筑物的初始和最终地面部分。 因此，问题简化为计算沿 x 轴和天际线绘制的边界曲线的总长度。 

限制很小，最多 50 栋建筑物。 这立即排除了对高级空间数据结构或进行大量优化的任何需要。 即使是 O(n²) 或 O(n² log n) 推理也是非常安全的。 主要挑战不是性能，而是正确合并重叠的水平部分并考虑垂直过渡。 

一些边缘情况很重要：

 如果两座建筑物在 x 范围内重叠，则只有最大高度对屋顶有贡献，但两者都可能对天际线上升或下降的垂直墙壁产生影响。 例如，考虑 (0, 10, 2) 和 (1, 9, 5)。 天际线在 x = 1 处从 0 上升到 5，然后在 x = 10 处从 5 下降到 0。独立添加每个建筑物的简单方法会重复计算内部结构并产生错误的路径。 

如果建筑物在边缘处精确接触，例如一个建筑物在 x = 10 处结束，另一座建筑物从 x = 10 处开始，则该边界处不存在垂直移动。 粗心的实现可能会错误地添加零宽度“间隙过渡”或重复计算垂直段。 

最后，当某个区域不存在建筑物时，毛毛虫沿着 y = 0 行走，这仅贡献水平距离，但不得引入垂直分量。 

## 方法

 一个直接但正确的思考问题的方法是明确地模拟天际线构造。 我们可以离散 x 轴，并针对每个小步骤，计算覆盖该部分的最大建筑物高度。 那么路径长度就是水平步长加上相邻段之间的垂直跳跃的总和。 这在概念上很简单：我们构建一个高度剖面函数 h(x)，然后计算其类似周长的遍历成本。 

然而，如果坐标很大，则精细分辨率下的离散化是不必要的，并且可能效率低下。 在这个问题中，坐标足够小，即使是简单的离散化也能通过，但它掩盖了结构。 

更原则性的方法是认识到天际线仅在建筑物边界处发生变化：在每个 s 和 s + w 处。 在这些点之间，高度是恒定的。 这意味着我们可以将 x 轴压缩为关键点的排序列表，计算每个区间的最大高度，然后测量转换。 

蛮力的想法是：对于每个小 x，计算覆盖 x 的所有建筑物的最大高度，然后求和转换。 这是 O(U·n)，其中 U 是最大 100 的坐标范围。在最坏的概括中，这变得太慢，但在这里它是微不足道的。

优化推理以区间推理代替连续扫描。 我们收集所有线段端点，对它们进行排序，并评估每个间隔的高度。 由于 n 最多为 50，因此每个时间间隔检查所有建筑物都是一项持续的工作。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力扫描 x = 0..99 | O(100·n) | O(100·n) | O(1) | O(1) | 已接受 |
 | 区间压缩| O(n² log n) | O(n² log n) | O(n) | 已接受 |

 ## 算法演练

 我们通过关注所有有趣的 x 坐标之间的线段来逐步构建天际线。 

1. 收集所有建筑物端点 s 和 s + w。 这些是天际线唯一可以改变高度的地方。 连续端点之间的任何间隔都具有恒定的高度。 
2. 对这些坐标进行排序并删除重复项以形成压缩的坐标列表。 这为我们提供了不相交的区间 [x[i], x[i+1]]，在该区间内天际线高度是固定的。 
3. 对于每个区间，计算覆盖该区间的任何建筑物的最大高度。 如果建筑物的水平跨度与该间隔相交，则建筑物覆盖该间隔。 我们取最大值，因为只有最高的可见结构才能确定顶部边界。 
4. 现在计算步行距离。 对于每个间隔，我们在地面上添加等于 x[i+1] − x[i] 的水平距离，加上可能由垂直过渡隐式处理的附加结构。 
5. 为了考虑垂直运动，我们跟踪连续间隔之间的高度变化。 如果高度增加，毛毛虫就会爬上这个差值； 如果它减少，它就会下降。 
6. 总答案是所有水平间隔长度加上连续天际线高度之间的所有绝对差之和。 

一个关键的微妙之处是初始化：我们从高度 0 开始，第一个非零高度会导致从地面垂直爬升。 

它之所以起作用是基于这样一个事实：毛毛虫的路径恰好是沿 x 投影时矩形并集的周长。 天际线函数对上边界进行编码，每次高度的变化都对应于边界的垂直段​​。 水平线段与间隔宽度完全对应。 由于建筑物在面积上不重叠，因此不会重复计算或遗漏任何部分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    buildings = []
    xs = set()

    for _ in range(n):
        s, w, h = map(int, input().split())
        buildings.append((s, s + w, h))
        xs.add(s)
        xs.add(s + w)

    xs = sorted(xs)

    def height_at_interval(l, r):
        best = 0
        for s, e, h in buildings:
            if not (e <= l or s >= r):
                best = max(best, h)
        return best

    prev_h = 0
    ans = 0

    for i in range(len(xs) - 1):
        l, r = xs[i], xs[i + 1]
        h = height_at_interval(l, r)

        ans += (r - l)
        ans += abs(h - prev_h)
        prev_h = h

    ans += prev_h
    print(ans)

if __name__ == "__main__":
    solve()
```其实现直接遵循区间压缩思想。 这组坐标确保我们只评估可能发生变化的区间。 对于每个间隔，我们通过检查所有建筑物来重新计算最大高度，考虑到较小的约束，这已经足够了。 

变量`prev_h`跟踪前一个间隔的高度，以便任何变化都会引入垂直贡献。 最后添加的是`prev_h`说明最终下降到地面。 

一个微妙的点是，无论高度如何，水平移动总是按照每个间隔进行计数，因为毛毛虫总是沿着地面或屋顶精确地穿过每个段的 x 跨度一次。 这就是为什么`(r - l)`总是被添加。

 ## 边缘情况

 一个关键的边缘情况是仅在边界处接触的建筑物。 例如：

 输入：```
2
0 10 3
10 10 4
```当 x = 10 时，建筑物之间没有水平间隙，也没有共享垂直墙。 该算法正确处理高度为 3 的 [0,10] 和高度为 4 的 [10,20]，在边界处产生单个垂直上升 1。 不存在重复计数，因为间隔分割可确保仅捕获一次转换。 

另一个边缘情况是一座建筑完全包含在一座更高的建筑内：

 输入：```
2
0 20 5
5 5 2
```内部建筑完全不影响天际线。 高度函数在所有区间内保持为 5，因此重叠内部不会发生垂直过渡。 该算法正确地忽略了内部结构，因为它总是采用每个间隔的最大高度而不是求和贡献。
