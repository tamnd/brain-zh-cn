 ## 解决方案
 ## 问题理解

 我们有一条从位置 0 到位置 L 的直线路径。沿着这条线，有一些特殊的点，称为增压站。 每当我们到达这些站点之一时，我们的速度立即增加一个固定的常数值 c，我们在骑行开始时选择一次，然后必须在任何地方使用。 

在加速站之间，运动受简单的线性衰减控制：如果我们进入速度为 v 的一段，那么 t 秒后速度变为 v − t，并且在该段中行驶的距离是速度随时间的积分。 如果我们在下一站之前用完了速度，我们就会停下来，骑行就会失败。 

目标是选择尽可能小的助推值c，以便从位置0开始以初始速度c（因为第一次助推为0），我们可以到达最终位置L，按顺序穿过所有助推站，并在最多T的总时间内完成。 

约束的结构使得升压站的数量 n 最多为 100，而 L 和 T 最多可达 10^9。 这立即排除了任何时间步长或细粒度离散化的模拟。 任何解决方案都必须在恒定时间内独立处理每个段，因此每次检查 O(n log answer) 或 O(n) 是唯一可行的方向。 

浮点精度产生了一个微妙的问题。 距离公式涉及时间的二次表达式，因此不小心比较可能会引入错误。 另一个重要的边缘情况是当速度达到零时所需的速度几乎不足以到达站点。 在这种情况下，该段恰好在可行性边界处结束，并且如果处理不一致，连续时间内的离一式推理会导致错误的二分搜索决策。 

例如，如果两个站点非常接近并且 c 被稍微低估，则由于浮动误差，计算出的行进时间可能仍然显得足够，但实际上速度会过早达到零，我们永远不会达到下一个加速。 

## 方法

 蛮力方法会尝试 c 的候选值并模拟整个骑行过程。 对于每个分段，我们求解一个二次方程，以确定到达下一次提升的距离需要多长时间。 我们累积时间并检查是否可以在 T 内完成。此模拟是正确的，因为给定进入速度时每个段都是独立的，并且运动公式精确地确定了行进距离。 

然而，连续尝试 c 的值是不可能的，因为 c 是实值并且可能是无界的。 即使我们以小步长离散化 c，所需的精度约为 1e-6，这在合理的搜索范围内需要大约 10^7 到 10^8 个候选，并且每个候选的成本为 O(n)。 这变得太慢了。 

关键的观察是单调性。 如果给定的提升值 c 允许我们在时间 T 内完成，那么任何更大的 c 也可以工作。 增加 c 会严格增加每个站点的速度，这只会减少每个路段所需的行驶时间。 这种单调行为允许在 c 上进行二分搜索。 

每次可行性检查都是对所有部分的前向模拟。 主要的技术挑战是计算，对于长度为 d、初始速度为 v 的线段，在线性衰减下需要多长时间才能覆盖它。 求解 v t − t^2/2 = d 给出一个二次方程，我们取较小的正根。 

这将整个问题变成了标准的“使用 O(n) 检查对答案进行二分搜索”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 离散化 c 上的强力模拟 | O(K·n) | O(K·n) | O(1) | O(1) | 太慢了 |
 | 使用 O(n) 模拟对 c 进行二分搜索 | O(n 对数精度) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们二分查找使骑行可行的最小提升值 c。

1. 我们定义一个函数 check(c) 来确定固定的增压值是否允许在时间 T 内完成。该函数逐段模拟骑行。 
2. 开始时，速度为 c，因为我们立即在位置 0 处收到加速。我们还将总时间初始化为 0。 
3. 对于连续提升位置之间的每个段，我们知道所需的距离 d。 我们还知道进入该段时的当前速度 v。 
4. 对于每个路段，我们计算在减速运动下行进距离 d 所需的最短时间 t。 我们求解方程 v t − t^2 / 2 = d。 有效根为 t = v − sqrt(v^2 − 2d)，假设 v^2 ≥ 2d。 
5. 如果 v^2 < 2d，则该路段是不可能的，因为在到达下一站之前速度已达到零。 在这种情况下，check(c) 立即失败。 
6. 否则，我们将 t 添加到总时间中，并将段末尾的速度更新为 v − t，等于 sqrt(v^2 − 2d)。 
7. 当我们到达加速站时，我们将速度增加 c，因此 v 变为 v + c，并继续。 
8.处理完所有段后，检查总时间是否≤T。如果是，则c可行。 

然后，我们在足够大的范围（通常为 [0, 1e9] 或更高）中对 c 进行二分搜索，并返回最小的可行值。 

### 为什么它有效

 该模拟保留了严格的物理不变量：在每个段的开始处，速度完全捕获了所有过去的影响，并且二次距离关系唯一地确定了行进时间。 由于速度仅随着 c 的增大而增加，并且路段行驶时间是入口速度的递减函数，因此 c 中的可行性是单调的。 这保证了二分查找的正确性，因为谓词 check(c) 沿实数线从 false 转换为 true 一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def check(c, L, n, tlim, b):
    v = c
    time = 0.0

    for i in range(1, n):
        d = b[i] - b[i - 1]

        if v * v < 2 * d:
            return False

        nv = math.sqrt(v * v - 2 * d)
        dt = v - nv

        time += dt
        v = nv + c

        if time > tlim:
            return False

    d = L - b[-1]
    if v * v < 2 * d:
        return False

    nv = math.sqrt(v * v - 2 * d)
    dt = v - nv

    time += dt

    return time <= tlim

def solve():
    L, n, tlim = map(int, input().split())
    b = list(map(int, input().split()))

    lo, hi = 0.0, 1.0
    while not check(hi, L, n, tlim, b):
        hi *= 2

    for _ in range(80):
        mid = (lo + hi) / 2
        if check(mid, L, n, tlim, b):
            hi = mid
        else:
            lo = mid

    print(hi)

if __name__ == "__main__":
    solve()
```实现的核心是检查函数，它以 O(n) 的速度计算可行性。 每个段都使用二次运动方程的封闭形式解，而不是模拟逐秒的速度衰减。 

表达式 sqrt(v^2 − 2d) 是经过距离 d 后的剩余速度，v − sqrt(v^2 − 2d) 是在该段中花费的时间。 这避免了任何数值积分。 

二分搜索以指数方式扩展上限，直到找到可行的解决方案，这保证了正确性，而无需猜测极限。 

最终循环使用固定次数的迭代 (80)，因为双精度浮点精度足以满足 1e-6 精度。

 ## 边缘情况

 一个关键的边缘情况是两个升压站重合或非常接近。 在这种情况下，所需的距离接近于零，因此该段几乎不贡献任何时间，并且速度基本上只增加 c。 该算法可以正确处理此问题，因为 d = 0 使得 v^2 ≥ 0 始终为真，nv = v 且 dt = 0，因此仅保留 boost 更新。 

另一种情况是速度完全足够时，即 v^2 = 2d。 然后 nv 变为零，dt 等于 v。模拟仍然有效，因为它完全以零速度降落在空间站并立即应用升压，从而保留了可行性。 

更微妙的情况是精度在可行性边界附近浮动。 由于检查使用平方根和比较，微小的数值错误可能会改变可行性。 二分搜索通过仅要求 1e-6 内的一致性来缓解这一问题，并使用足够大的固定迭代计数来确保稳定的收敛。
