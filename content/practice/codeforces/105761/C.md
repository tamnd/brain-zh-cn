 ## 解决方案
 ## 问题理解

 我们给出了五个整数来描述一支足球队的赛季统计数据，但这些整数的顺序是打乱的。 我们知道它们对应于比赛场次、胜利、平局、失利和总积分，但我们不知道哪个数字是哪个。 

足球比赛的计分规则是固定的：胜得3分，平得1分，负得0分。 此外，每场比赛都会产生其中一个结果，因此总比赛数是胜、平和负的总和。 总积分按三胜加平局计算。 

任务是按未知顺序获取五个数字并重建唯一的有效解释，然后按规范顺序打印它们：MP、W、D、L、Pts。 

尽管所有值都很小（最多 300），但关键的困难在于我们必须同时在两个约束下正确分配角色：一个来自匹配计数的线性约束，另一个来自评分的线性约束。 任意分配角色的天真的尝试将会失败，因为存在多种排列，但只有一种排列同时满足两个方程。 

边缘情况主要围绕重复值。 例如，像这样的输入`0 0 1 1 0`包含重复项，这使得天真的贪婪赋值变得不明确。 另一个微妙的情况是，当平局和亏损或胜利和平局具有相同的值时，即使有效的解决方案仍然是唯一的，这也可能会误导基于排序的启发式方法。 

由于只有五个数字，因此在常数时间内对所有排列进行暴力破解是可行的。 这是主要的结构观察。 

## 方法

 一个直接的想法是尝试将这五个数字分配给角色 MP、W、D、L 和 Pts。 对于每个作业，我们检查它是否满足两个条件：第一，MP 必须等于 W + D + L，第二，Pts 必须等于 3W + D。如果两者都成立，我们就找到了正确的解释。 

只有 5 个值，因此排列数为 5 阶乘，即 120。对于每个排列，我们都会进行恒定的工作，因此这种方法非常快。 

这里不需要更多“基于思想”的优化，因为约束很小。 即使我们试图通过推理哪个数字最大或最小来进一步减少它，我们仍然需要验证一致性，并且排列检查已经简单且安全。 

蛮力方法之所以有效，是因为搜索空间很小，并且约束完全确定了唯一的映射。 经观察发现只有5个！ 作业的存在将看似推理的问题简化为简单的验证循环。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(5!) | O(1) | O(1) | 已接受 |
 | 最优（与蛮力相同）| O(120) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将五个输入数字视为多重集，并尝试对五个角色的所有可能分配。 

1. 将五个整数读入数组。 这些代表 MP、W、D、L 和 Pts 的未知排列。 
2. 生成这五个值的所有排列。 每个排列对应一个假设的分配：第一个值是 MP，第二个值是 W，第三个值是 D，第四个值是 L，第五个值是 Pts。 我们不假设任何顺序，因为输入没有给出任何提示。 
3. 对于每个排列，计算它是否满足足球统计的结构约束。 我们检查两个方程：MP 必须等于 W + D + L，Pts 必须等于 3W + D。这一步是唯一的过滤条件，它强制执行评分系统的规则。 
4. 该问题保证存在一个有效的赋值，因此一旦我们找到满足这两个条件的排列，我们就以所需的顺序输出它并终止。 

这是足够的原因是约束完全定义了系统。 任何不正确的分配都会违反至少一个方程，因此无法通过检查。 

### 为什么它有效

 每个有效的足球表行完全由两个方程 MP = W + D + L 和 Pts = 3W + D 确定。由于我们正在测试输入数字到这些角色的所有可能分配，因此排列中必须出现正确的配置。 唯一性保证确保没有其他排列可以同时满足这两个约束，因此我们找到的第一个有效匹配是唯一的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from itertools import permutations

nums = list(map(int, input().split()))

for mp, w, d, l, pts in permutations(nums):
    if mp == w + d + l and pts == 3 * w + d:
        print(mp, w, d, l, pts)
        break
```该解决方案直接编码了详尽的分配检查的思想。 我们依赖Python`itertools.permutations`枚举所有可能的角色分配。 

关键细节是我们不会尝试优化选择顺序或积极修剪。 只有 120 个案例，开销可以忽略不计。 早期中断确保我们在找到有效配置后立即停止，这与唯一性保证一致。 

一个微妙的一点是，即使值重复，我们也会平等地对待所有排列。 这很好，因为正确性取决于值分配，而不是索引标识。 重复的值可能会产生重复的排列，但它们不会以任何有意义的方式影响正确性或性能。 

## 工作示例

 ### 示例 1

 输入：`19 11 2 4 5`我们测试排列，直到找到有效的映射。 

| 国会议员 | 西 | d | 左 | 积分 | 国会议员检查| 点检查 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 11 | 11 5 | 4 | 2 | 19 | 19 11 ≠ 11 | 19 = 19 | 19 = 19

 这里，MP = 11，W = 5，D = 4，L = 2，Pts = 19。 

我们验证 MP = W + D + L → 11 = 5 + 4 + 2 = 11 且 Pts = 3W + D → 19 = 15 + 4 = 19。 

这确认找到了正确的分配。 

### 示例 2

 输入：`0 0 1 1 0`一种有效的排列是：

 | 国会议员 | 西 | d | 左 | 积分 | 国会议员检查| 点检查 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 1 | 0 | 0 | 1 | 0 | 1 = 1 | 0 = 0 |

 这里 MP = 1, W = 0, D = 0, L = 1, Pts = 0 完全满足这两个约束。 

这种情况说明了为什么重复值很重要：存在多种排列，但只有一种排列同时满足两个方程。

## 测试用例```python
import sys, io
from itertools import permutations

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    nums = list(map(int, sys.stdin.readline().split()))
    for mp, w, d, l, pts in permutations(nums):
        if mp == w + d + l and pts == 3 * w + d:
            return f"{mp} {w} {d} {l} {pts}"
    return ""

# provided samples

 ## 边缘情况

 一种重要的边缘情况是当多个值相同时，这会增加无法区分的排列的数量。 对于像这样的输入`0 0 1 1 0`，该算法仍然有效，因为正确性与值的唯一性无关，而是与约束的满足有关。 置换循环最终会尝试`MP=1, W=0, D=0, L=1, Pts=0`，它通过了两项检查，而所有其他排列要么违反 MP 一致性，要么违反评分一致性。 

另一个边缘情况是胜利为零时。 对于像这样的输入`1 0 0 1 0`，除了结构之外，有效的解释没有来自胜利或平局的得分贡献。 检查仍然成功，因为 MP = 0 + 0 + 1 和 Pts = 0 完全匹配。 

第三种边缘情况是平局为零时，得分就会变成纯粹的胜利倍数。 例如，如果输入是`6 2 0 4 6`，只有正确的排列才能同时满足 MP 和 Pts 方程。 该算法不会特殊对待零，它只是统一验证方程，这避免了任何特殊情况的错误。
