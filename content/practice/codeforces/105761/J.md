 ## 解决方案
 ## 问题理解

 我们正在研究一个非常大的整数网格，其中每个点都有四个相邻的移动：上、下、左、右。 有些网格点因为正在施工而被封锁，这些被封锁的点无法访问。 

每个查询都会询问从起点到终点的有效最短路径的数量，其中“最短”意味着曼哈顿最短，因此每一步都必须严格减少到目的地的曼哈顿距离。 这一限制消除了所有的徘徊：每一次移动都必须在正确的水平方向或正确的垂直方向上，永远不能偏离目标。 

如果没有阻塞的单元格，则每条路径$(x_1, y_1)$到$(x_2, y_2)$将由固定数量的水平移动和任意顺序的垂直移动组成，因此答案将是二项式系数。 困难在于有些格点是禁止的，并且必须排除穿过它们的路径。 

网格坐标在两个维度上都达到 500,000，因此网格太大而无法明确表示。 阻塞点的数量最多为10个，这是关键的结构约束。 

一个微妙但重要的一点是，路径只允许沿着单调最短路线移动。 这意味着如果目的地是东北，我们只会向北或向东移动。 因此，每个有效路径都位于起点和终点之间的轴对齐矩形内。 

边缘情况来自于障碍物如何与这种单调结构相互作用。 简单的最短路径计数很容易过多计算被阻挡在远处但仍在矩形内的路径，另一个常见的错误是忘记即使矩形内的单个障碍物也可以通过分割区域来完全改变组合数学。 

例如，如果开始是$(0,0)$，结束是$(2,2)$，并且有一个块位于$(1,1)$，那么所有经过的路径$(1,1)$均无效。 正确答案是总路径减去经过该点的路径：

 总计为 6，通过 (1,1) 为$\binom{2}{1} \cdot \binom{2}{1} = 4$，所以答案是 2。忽略中间阻塞点的简单方法将输出 6。 

这些约束意味着我们不能对每个查询执行网格 DP 或 BFS。 对于多达 10,000 个查询，任何解决方案都比大约更糟糕$O(1)$或者$O(n^2)$每个查询只有在以下情况下才是可接受的：$n$很小，确实如此。 关键是要利用只有 10 个障碍的事实。 

## 方法

 如果没有障碍，每个查询都会简化为组合问题。 让$\Delta x = |x_2 - x_1|$和$\Delta y = |y_2 - y_1|$。 最短路径的数量为$\binom{\Delta x + \Delta y}{\Delta x}$，因为我们在所有移动中选择水平移动的位置。 

对于障碍物，暴力破解的想法是将每个查询视为限制在矩形内的网格 DP 或 BFS。 这会立即失败，因为坐标范围高达 500,000，因此即使每个查询只遍历一次也是不可能的。 

更精确的暴力破解方法是仅使用未被遮挡的坐标在矩形上运行 DP。 但矩形仍然可能很大，所以这仍然不可行。 

关键的观察结果是障碍物的数量很少。 这表明将障碍视为特殊的中间点并对其使用包含-排除。 我们不再考虑网格状态，而是考虑一小组点之间的路径。 

我们将障碍与开始和结束一起排序。 然后我们计算任意两点之间忽略障碍物的单调路径的数量。 之后，我们减去穿过至少一个障碍物的路径的贡献，但以一种结构化的方式：对于每个点，计算从开始到它的有效路径的数量，排除穿过较早障碍物的路径。 

这成为每个查询最多 12 个点的经典 DP，因为我们包括起点和终点以及最多 10 个障碍。 

对于每个查询，我们：

 计算从起点到所有点，然后从每个点到终点的路径，同时确保在两点之间移动时，我们只考虑单调可行的过渡（仅当坐标适当增加时）。 然后，我们通过按排序顺序处理点并减去经过中间阻挡点的路径来应用障碍物包含。 

这将每个查询减少到$O(n^2)$在哪里$n \le 12$，这对于 10,000 次查询来说足够快了。

 ## 算法演练

 我们首先修复单个查询。 我们收集所有相关点：开始、结束和所有被阻止的单元格。 

我们只关心位于由开始和结束定义的单调矩形内的点。 如果障碍物位于此矩形之外，则它不能位于任何最短路径上，并且可以在此查询中忽略。 

接下来，我们通过增加 x 对所有相关点进行排序，如果 x 相等，则增加 y。 这种排序遵循单调移动约束，因为任何有效路径都必须在两个维度上从较小的坐标移动到较大的坐标。 

我们为每个点定义一个 DP 值，表示从起点到该点的单调路径数量，同时避开之前处理过的障碍物。 

我们在起始点将 DP 初始化为 1。 

对于排序顺序中的每个点，我们通过首先计算从开始到它的所有单调路径（忽略障碍物）来计算其 DP 值。 这是一个组合值。 然后我们减去来自可以达到它的早期点的贡献。 

因此，我们按照以下结构化方式进行。 

1. 建立一个由起点、终点和所有障碍物组成的点列表。 

这确保了所有潜在的“块交互”都明确表示为小型 DAG 中的节点。 
2. 过滤不在查询矩形范围内的点。 

任何这样的点都不能位于最短单调路径上，因此包含它只会增加不必要的状态。 
3. 按 x 然后按 y 对点进行排序。 

这确保了当我们处理一个点时，有效单调路径中所有可能的前驱点都已被处理。 
4. 预先计算高达 1,000,000 的二项式系数或阶乘，或使用具有预先计算的阶乘和逆阶乘的快速组合函数。 

这是必需的，因为两点之间的每个边权重都是二项式系数。 
5. 将 dp[i] 定义为从起点到点 i 的有效路径数，忽略到目前为止的障碍物。 
6. 对于按排序顺序的每个点 i：

 计算从 start 到 i 的基本路径为 Comb(dx + dy, dx)。 

然后减去经过任何较早点 j 的路径：

 如果 j 可以单调到达 i，则减去 dp[j] *ways(j → i)。 
7.答案是dp[end]。 

减法步骤本质上是嵌入在 DP 形式中的包含-排除。 每个 dp[j] 已经表示在 j 之前避开障碍物的路径，因此乘以从 j 到 i 的路径就可以计算出所有首先到达 j 然后继续的路径。 

### 为什么它有效

 从开始到结束的每条单调路径都可以由其第一个障碍（或没有）唯一地分解。 如果一条路径没有穿过任何障碍物，则将其计入终点的直接组合项中。 如果它穿过至少一个障碍物，则令 j 为按排序顺序访问的第一个障碍物。 start到j的前缀已经计入dp[j]中，j到end的后缀独立计数。 DP 减法确保每个此类分解恰好计数一次，并且按排序顺序进行处理可保证没有循环依赖或重复计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

MAXN = 500000

fact = [1] * (MAXN + 1)
invfact = [1] * (MAXN + 1)

for i in range(1, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN] = modinv(fact[MAXN])
for i in range(MAXN, 0, -1):
    invfact[i - 1] = invfact[i] * i % MOD

def comb(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

def solve():
    n = int(input())
    obstacles = [tuple(map(int, input().split())) for _ in range(n)]
    t = int(input())

    for _ in range(t):
        x1, y1, x2, y2 = map(int, input().split())

        if x1 <= x2:
            xs, ys = x1, y1
            xt, yt = x2, y2
        else:
            xs, ys = x2, y2
            xt, yt = x1, y1

        pts = [(xs, ys), (xt, yt)]
        for x, y in obstacles:
            if xs <= x <= xt and ys <= y <= yt:
                pts.append((x, y))

        pts = list(set(pts))
        pts.sort()

        k = len(pts)
        dp = [0] * k
        dp[0] = 1

        for i in range(k):
            xi, yi = pts[i]
            ways = comb(xi - xs + yi - ys, xi - xs)
            dp[i] = ways

            for j in range(i):
                xj, yj = pts[j]
                if xj <= xi and yj <= yi:
                    dx = xi - xj
                    dy = yi - yj
                    dp[i] = (dp[i] - dp[j] * comb(dx + dy, dx)) % MOD

        end_index = pts.index((xt, yt))
        print(dp[end_index] % MOD)

solve()
```该代码首先预先计算阶乘和逆阶乘直到最大坐标，从而允许恒定时间二项式系数查询。 这是至关重要的，因为两点之间的每个转换都是组合评估的。 

对于每个查询，我们规范化方向，以便始终从左下角移动到右上角，确保单调性。 然后，我们收集边界矩形内的所有障碍物并将它们添加到点集中。 

排序可确保在计算 dp[i] 时，所有可能的前驱都已计算完毕。 dp 值从一开始的总组合路径开始，我们减去所有早期可到达点的贡献，有效地消除“穿过”障碍物的路径。 

最终的答案就是目标点的dp值。 

一个微妙的实现细节是在减法之后取模以避免负值。 另一个是确保只考虑单调的前辈； 否则无效的向后转换会破坏计数。

 ## 边缘情况

 关键的边缘情况是障碍物位于查询的边界矩形之外。 在这种情况下，必须完全忽略它，因为没有单调最短路径可以到达它。 该算法通过在构建 DP 集之前使用坐标边界过滤障碍物来处理此问题。 

另一种情况是多个障碍物共享相同的 x 或 y 排序约束，这使得它们在路径顺序上无法比较。 按 x 然后 y 排序可确保单调图的有效拓扑顺序，因此每个点仅取决于较早的可到达点。 

最后，当起点和终点相邻时，DP 退化为小尺寸的单个二项式系数。 该算法仍然有效，因为集合中没有中间点，因此不会发生减法，并且直接返回基本组合计数。
