 ## 解决方案
 ## 问题理解

 我们得到一个整数$b$，我们在概念上迭代每个正整数$a$。 对于每个$a$，我们计算从最小公倍数导出的值$a$和$b$， 具体来说$\frac{\mathrm{lcm}(a,b)}{a}$。 伊万将这个值写在黑板上，然后我们被问到，当$a$范围涵盖所有正整数。 

关键的一点是，虽然$a$范围可达$10^{18}$，表达式仅取决于如何$a$与基本结构相互作用$b$。 输出不是求和或最大化任何东西，而是计算该比率可以产生多少种不同的乘法结果。 

约束条件$b \le 10^{10}$建议仅依赖于素因式分解的解决方案$b$是可行的，因为因式分解$10^{10}$在确定性方面是可管理的$O(\sqrt{b})$时间。 任何迭代所有方法$a \le 10^{18}$立刻就不可能了。 

一个天真的尝试会尝试评估许多表达式$a$，但这在两个方面失败了。 首先，域$a$实际上是无限的。 二、多种不同$a$产生相同的结果，并且重复结构是高度规则的，但在没有数论的情况下并不明显。 

出现微妙的边缘情况时$b = 1$。 然后$\mathrm{lcm}(a,1) = a$，所以表达式总是$1$，只给出一个值。 另一个边缘情况是当$b$是素数，其中结构根据被该素数整除而仅分解为两种可能的结果。 

## 方法

 从表达式的定义开始。 使用身份$$\mathrm{lcm}(a,b) = \frac{ab}{\gcd(a,b)},$$我们将表达式重写为$$\frac{\mathrm{lcm}(a,b)}{a} = \frac{b}{\gcd(a,b)}.$$这将问题从有关最小公倍数的问题转变为纯粹有关 gcd 值的问题。 现在表达式仅取决于$\gcd(a,b)$，它必须是除数$b$。 这立即减少了无限域$a$转化为一组可能的 gcd 值。 

让$g = \gcd(a,b)$。 然后$g \mid b$，表达式变为$b/g$。 因此，黑板上写的每个值都与选择的除数完全对应$g$的$b$，前提是这样的 gcd 实际上可以通过某些方法实现$a$。 每个除数$g$的$b$可以通过采取$a = g \cdot k$在哪里$k$与 互质$b/g$，因此不排除任何除数。 

因此，写入的不同值的集合正是：$$\left\{ \frac{b}{d} \mid d \mid b \right\}.$$不同的除数$d$产生不同的值$b/d$，所以不同结果的数量正好是除数的数量$b$。 

问题归结为计算除数$b$，这是通过质因数分解完成的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力结束$a$|$O(10^{18})$|$O(1)$| 太慢了|
 | 因式分解 + 除数计数 |$O(\sqrt{b})$|$O(1)$| 已接受 |

 ## 算法演练

 1. 因式分解$b$进入其主要权力。 我们扫描整数$\sqrt{b}$，提取每个素数整除的次数$b$。 这已经足够了，因为任何复合因子必须有一个不超过的素因子$\sqrt{b}$。 
2. 对于每个素数$p$, 计算其指数$e$在因式分解中$b$。 这一步很重要，因为除数计数仅取决于这些指数，而不取决于实际值$b$。 
3. 使用公式计算除数总数$$\tau(b) = \prod (e_i + 1),$$其中每个$e_i$是一个不同质因数的指数。 
4. 输出该乘积作为答案，因为每个除数恰好对应于一个不同的值$\frac{b}{\gcd(a,b)}$。 

### 为什么它有效

 表达式的每个可能值均由以下除数唯一确定$b$，因为表达式简化为$b/g$在哪里$g \mid b$。 相反，每个除数$g$可以通过构造来实现为 gcd$a$具有适当的共享素数并避免会增加 GCD 的额外素数。 这在除数之间建立了一对一的对应关系$b$以及写在黑板上的不同价值观。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    b = int(input().strip())
    x = b
    ans = 1

    p = 2
    while p * p <= x:
        if x % p == 0:
            cnt = 0
            while x % p == 0:
                x //= p
                cnt += 1
            ans *= (cnt + 1)
        p += 1

    if x > 1:
        ans *= 2

    print(ans)

if __name__ == "__main__":
    solve()
```代码首先分解$b$通过审判庭。 每次找到质因数时，它都会计算其重数并更新除数乘积。 如果循环后剩余大于 1 的余数，则该余数是影响指数 1 的质因数，使除数计数加倍。 

微妙的一点是我们从不迭代$a$。 整个问题归结为内部结构$b$，并且 gcd 重新表述消除了对巨大上限的任何依赖$a$。 

## 工作示例

 考虑$b = 2$。 除数是$1, 2$。 表达式值为$2/1 = 2$和$2/2 = 1$。 

| 步骤| 除数$d$|$b/d$|
 | --- | --- | --- |
 | 1 | 1 | 2 |
 | 2 | 2 | 1 |

 这显示了两个不同的值，与除数计数相匹配。 

现在考虑$b = 12 = 2^2 \cdot 3^1$。 它的除数由指数选择决定：$2^2$，选择指数0到2； 为了$3^1$，选择指数0到1。 

| 总理| 指数| 贡献 |
 | --- | --- | --- |
 | 2 | 2 | 3 个选择 |
 | 3 | 1 | 2 个选择 |

 总除数$= 3 \cdot 2 = 6$。 

这演示了独立素数贡献如何乘法组合，反映基于 gcd 的值的结构。

 ## 边缘情况

 当$b = 1$，因式分解不会产生素数，因此除数计数仍为 1。该算法无需进入循环体即可正确返回 1。 

什么时候$b$是素数，说$13$，循环找不到因素，剩下的$x > 1$触发乘以 2，恰好产生两个除数$1$和$13$，匹配表达式的预期两个不同值。 

什么时候$b$是一个完美的力量，比如$2^k$，循环重复除掉相同的素数并累加指数$k$，生产$k+1$除数，它与可实现的 gcd 值的数量完全对应，从而与原始表达式的不同输出相对应。
