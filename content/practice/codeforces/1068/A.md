 ## 解决方案
 ## 问题理解

 我们正在处理一个由不同硬币类型组成的固定宇宙，其中有`N`总共可能有不同的硬币。 伊万已经拥有`K`与这个宇宙不同的硬币。 现在`M`朋友们会各自给他礼物，每个朋友必须贡献相同数量的金币。 

所有赠送的硬币在所有朋友中都必须是不同的，这意味着任何硬币类型在整套礼物中只能出现一次。 由于朋友们不知道伊万已经拥有哪些硬币，因此在伊万未知的收藏的最坏情况下，必须满足涉及“新硬币”的要求。 

关键的要求是，在所有赠送的硬币中，至少`L`其中肯定是伊万以前没有拥有过的硬币。 由于我们必须在不了解伊万收藏的情况下保证这一点，因此我们必须确保无论他的藏品如何`K`选择硬币后，礼品套装中至少包含`L`新硬币。 

输出要求最小数量`x`这样每个朋友都准确地给出`x`不同的硬币，并且可以满足所有约束。 如果没有这样的`x`存在，我们返回`-1`。 

约束条件非常大，最多可达`10^18`，这会立即排除任何模拟或迭代构造。 任何解决方案都必须是纯粹的算术，基于对重叠和最坏情况交叉点的推理。 

一些微妙的边缘情况立即出现。 

如果`N < M`，那么即使每个朋友只给出一枚硬币，所有朋友也不可能给出不同的硬币，因为我们至少需要`M`每个朋友的第一枚硬币都有不同的硬币类型。 

如果`L > N - K`，那么即使我们赠送伊万收藏之外的所有硬币，宇宙中也没有足够的“新”硬币来满足要求。 

当出现更微妙的失败情况时，`M * x > N`，因为所有赠送的硬币必须在全球范围内是独一无二的。 即使每个朋友给出的数字很小，总池数也不能超过`N`。 

## 方法

 一种幼稚的方法是尝试增加`x`开始于`1`，并为每个候选人模拟是否可以分配`M * x`不同的硬币，同时保证至少`L`伊万未知之外的硬币`K`-放。 检查需要推理最坏情况的重叠：假设 Ivan 的`K`硬币的选择是为了最大限度地与礼品套装相交。 

对于固定的`x`，我们赠送`M * x`总共不同的硬币。 在最坏的情况下，伊万`K`硬币与这些礼物尽可能重叠，因此保证“新”硬币的数量最小化。 新硬币的最小数量是：`max(0, M * x - K)`我们需要这个至少是`L`。 所以我们需要：`M * x - K >= L`，这简化为`M * x >= K + L`。 

然而，我们还需要宇宙总大小的可行性：`M * x <= N`。 

所以暴力结束`x`会反复检查这些不平等。 但自从`x`可以达到`10^18`，从最坏的解释来看，这在概念上仍然是 O(N) 并且不可行。 

关键的观察是可行性完全由两个线性约束决定：`M * x >= K + L`和`M * x <= N`。 

这些约束定义了一个简单的区间`x`，我们被要求提供最小的整数`x`在这个区间内。 

因此我们计算：

 下限：`ceil((K + L) / M)`上限：`floor(N / M)`如果下限超过上限，则无有效解。 

这将整个问题简化为整数运算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N/M) | O(1) | O(1) | 太慢了|
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算保证至少需要的最少赠送币总数`L`在最坏的情况下新硬币。 这个要求就变成了`M * x >= K + L`。 
2. 将其转换为下界`x`使用带上限的整数除法：`(K + L + M - 1) // M`。 
3. 计算宇宙约束允许的最大可能赠金总数`M * x <= N`，给出上限`N // M`。 
4. 如果下限超过上限，则得出不存在有效分配的结论并输出`-1`。 
5. 否则，输出下界作为每个朋友的最小可行硬币数量。 

### 为什么它有效

 关键的不变量是任何有效的配置仅取决于赠送硬币的总数`T = M * x`，而不是他们在朋友之间的分配。 独特性迫使一切`T`赠品硬币在全球范围内是唯一的，因此结构简化为选择大小的子集`T`从`N`。 

最坏的情况与伊万的未知重叠`K`-子集总是最大化，因此新币的最小保证数量恰好是`max(0, T - K)`。 这使得可行性条件完全线性`T`，因此线性于`x`。 一旦进行此还原，就不会保留任何隐藏结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M, K, L = map(int, input().split())

    # upper bound: cannot exceed total available coins
    hi = N // M

    # lower bound: ensure enough guaranteed new coins
    need = K + L
    lo = (need + M - 1) // M  # ceil division

    if lo > hi:
        print(-1)
    else:
        print(lo)

if __name__ == "__main__":
    solve()
```该实现直接镜像派生边界。 唯一的微妙之处是下界的正确上限除法，这可以避免浮点错误并确保高达以下大值的正确性`10^18`。 

第二个约束`M * x <= N`使用楼层划分来处理。 两种计算都必须使用 Python 天然支持的 64 位安全整数算术。

def run(inp: str) -> str:

 ## 边缘情况

 一个关键的边缘情况是当`M > N`。 例如：

 输入：`N = 5, M = 10, K = 1, L = 1`这里`hi = 5 // 10 = 0`，这意味着即使给每个朋友 1 个硬币也是不可能的。 算法立即返回`-1`因为没有有效的`x`满足`M * x <= N`。 这符合现实，因为当硬币类型少于朋友时，我们甚至无法为每个朋友分配一枚不同的硬币。 

另一种情况是当`K + L`相对于来说非常大`N`。 例如：

 输入：`N = 100, M = 10, K = 90, L = 20`这里`need = 110`， 所以`lo = ceil(110 / 10) = 11`， 但`hi = 10`，所以不可行。 这表明伊万现有的收藏和所需的新颖性之间的相互作用可以迫使不可能，即使`N >= M`。
