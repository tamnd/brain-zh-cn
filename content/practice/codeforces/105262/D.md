 ## 解决方案
 ## 问题理解

 我们试图选择一个数字基数来表示给定的整数，以便其表示形式至少包含一次数字 4。 在给定范围内的所有此类碱基中，我们想要最大的一个。 

具体来说，对于每个测试用例，我们都会收到一个数字$n$。 我们想象写作$n$在基地$b$， 在哪里$b$必须至少为 2，最多$2 \cdot 10^{12}$。 在该表示中，我们扫描它的数字。 如果至少一位数字等于 4，则该基数被视为有效。 我们的任务是找到最大有效碱基，或者报告不存在。 

约束条件$n \le 10^{12}$表示的位数$n$任何合理的基数都很小，在基数 2 中最多约为 40，对于较大的基数甚至更小。 这限制了任何表示形式中可以存在的数字位置数量。 测试用例的数量最多为 10 个，因此算法大致为$O(n^{1/2})$或者$O(n^{1/3})$每个案例还是可以的。 

天真的尝试会尝试每一个基地$b$从 2 到$2 \cdot 10^{12}$， 转变$n$进入基地$b$，并检查一个数字是否等于 4。这种方法会立即失败，因为即使测试单个底数也是对数的$n$，因此总工作量将远远超出任何限制。 

当人们只检查是否$n \% b = 4$。 这只验证最后一位数字并错过所有更高的数字位置。 例如，$n = 24$以 5 为基数是$44_5$，有 4 但不满足$24 \% 5 = 4$单独作为所有有效碱基的充分条件。 

真正的困难在于数字4可以出现在任何位置，并且每个位置对$b$。 

## 方法

 蛮力结构很简单。 对于每个候选人基地$b$，我们反复划分$n$经过$b$并检查余数是否有数字等于 4。这是正确的，因为基数转换实际上是重复除法。 问题是范围$b$是巨大的，并且迭代所有它大致给出$2 \cdot 10^{12}$候选人，每个成本$O(\log n)$，这是完全不可行的。 

关键的观察是数字条件可以用代数表达。 如果位置处出现数字4$i$，然后在基数$b$这个数字可以写成$$n = x \cdot b^{i+1} + 4 \cdot b^i + y$$和$0 \le y < b^i$。 这将问题转变为寻找该方程的整数解，而不是模拟基数。 

我们根据数字 4 的位置来区分情况。当 4 位于最低有效数字时，方程简化为$n = q b + 4$， 所以$b \mid (n - 4)$。 这意味着在这种情况下所有有效基数都是$n - 4$，可以有效地枚举。 

当 4 处于较高位置时，比如说$i \ge 1$， 期限$b^{i+1}$成长很快。 这立即限制$b$因为$b^{i+1} \le n$。 因此，对于每个固定位置，可能的碱基范围为$n^{1/(i+1)}$，它变得非常小，因为$i$增加。 这使我们能够分别尝试每个位置的所有候选碱基，并通过重建碱基表示来验证有效性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力攻打基地 |$O(n \log n)$|$O(1)$| 太慢了|
 | 基于位置的枚举 |$O(\sum n^{1/k} \log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们根据数字 4 在基本表示中可能出现的位置来划分搜索。 

1、首先处理数字4在个位的情况。 在这种情况下我们需要$n = q b + 4$。 重新排列给出$n - 4 = q b$，所以每个有效的基数都必须整除$n - 4$。 我们枚举所有的约数$n - 4$，检查底数是否至少为 2，并验证转换$n$这个基数确实包含数字 4。此步骤是必要的，因为整除性本身并不能保证更高的数字也不会违反约束。 
2.接下来考虑数字4位于位置的情况$i \ge 1$。 我们将数字重写为$n = a b^{i+1} + 4 b^i + r$， 在哪里$r < b^i$。 这确保了位置上的数字$i$正好是 4，并且不会发生进位或溢出。 
3.对于固定位置$i$，我们利用这样一个事实$b^{i+1} \le n$。 这给出了一个上限$b$， 具体来说$b \le n^{1/(i+1)}$。 我们迭代所有此类候选库。 
4. 对于每个候选碱基，我们显式构造数字$n$在该基数中检查是否有任何数字等于 4。这种验证是安全的，因为候选集很小。 
5. 我们跟踪所有职位的所有有效候选人的最大基数。 

正确性来自于每个有效碱基的某个位置必须有其数字 4$i$。 我们枚举所有可能的位置，并且对于每个位置，我们只考虑可能支持该数字位置而不会溢出的基数。 每个有效的解决方案都恰好出现在其中一种情况下，因此不会错过任何候选方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def has_four(n, b):
    while n > 0:
        if n % b == 4:
            return True
        n //= b
    return False

def divisors(x):
    res = []
    i = 1
    while i * i <= x:
        if x % i == 0:
            res.append(i)
            if i * i != x:
                res.append(x // i)
        i += 1
    return res

T = int(input())
for _ in range(T):
    n = int(input())
    best = -1

    if n - 4 > 0:
        for d in divisors(n - 4):
            if d >= 2 and has_four(n, d):
                best = max(best, d)

    i = 1
    while True:
        pow_b = 1
        ok = False

        max_b = int((n) ** (1 / (i + 1)))
        if max_b < 2:
            break

        for b in range(2, max_b + 1):
            if has_four(n, b):
                best = max(best, b)

        i += 1
        if max_b <= 2:
            break

    print(best)
```除数步骤处理当 4 是最低有效数字时的特殊结构，此时问题会分解为干净的整除条件。 第二个循环尝试增加数字位置并限制基数，以便位置贡献$b^{i+1}$不超过$n$。 辅助函数`has_four`进行基数转换并直接检查数字，这是安全的，因为数字的个数是对数$n$。 

增加的停止条件$i$来自这样一个事实：一旦$b^{i+1} > n$，更高的数字位置不能包含 4。

 ## 边缘情况

 一种微妙的情况发生在$n = 4$。 这里$n - 4 = 0$，因此除数逻辑会错误地尝试枚举零除数。 正确的行为是立即拒绝，因为 4 的基数表示都没有以有效的方式包含数字 4（表示形式只是“4”，但基数必须严格大于 4，这使得该数字在基数约束中无效）。 

另一种情况是当最佳碱基恰好是$n - 4$。 为了$n = 20$，我们得到$n - 4 = 16$，基数 16 产生表示$14_{16}$，其中包含单个数字 4。除数枚举正确包含该候选者，并且验证可确保其被接受。 

对于大型企业来说，会出现第三种边缘情况$n$有效碱基稀疏，只有较高位置的数字放置才起作用。 有界搜索结束$b^{i+1} \le n$确保这些仍然可以到达，因为对于小$i$，基数范围仍然足够大以包括所有候选者，并且对于大$i$，边界迅速崩溃，防止错过解决方案。
