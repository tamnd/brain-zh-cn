 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并要求选择一个连续的段并计算其修改后的总和。 在所选段内，添加第一个元素，减去第二个元素，再次添加第三个元素，这种交替持续到段末尾。 目标是找到所有子数组上这个交替和的最大可能值。 

一个关键细节是，符号模式始终从所选子数组的开头开始为正，无论它在原始数组中的何处开始。 因此，问题不在于固定的全局符号模式，而在于选择片段及其起始位置，以使交替行为达到最佳。 

这些约束意味着数组可以很大，每个测试用例最多可达一百万个元素，最多可达一万个测试用例。 这立即排除了任何显式尝试所有子数组的解决方案，因为子数组的数量是大小的二次方。 即使每个测试用例使用 O(n^2) 方法也会远远超出可接受的限制。 因此，每个测试用例的解决方案必须是线性的或接近线性的，单次处理元素。 

朴素推理的一个微妙的失败案例来自于假设最佳段始终与全局最大子数组和相关。 例如，在像这样的数组中`[5, 100, 5]`，标准最大子数组和是整个数组，但交替符号使中间元素减重很大，因此最佳交替段可以完全避免它。 另一个陷阱是假设我们可以全局修复奇偶校验并运行单个类似 Kadane 的进程； 最佳答案取决于所选子数组是从偶数位置还是奇数位置开始，因此必须显式跟踪奇偶校验。 

## 方法

 暴力方法是枚举每个可能的子数组，直接计算其交替和，并跟踪最大值。 对于每个子数组，计算交替和在最坏的情况下需要线性时间，因此如果简单地完成，总复杂度变为 O(n^3)，如果使用前缀预处理，则总复杂度变为 O(n^2)。 由于测试中 n 高达 10^6，这是完全不可行的。 

关键的观察结果是交替结构仅取决于子阵列内的位置奇偶性。 一旦我们确定了子数组的开始位置，符号模式就变得确定了。 这使我们能够将问题转化为类似于 Kadane 算法的动态规划问题，但有两个状态跟踪当前位置是否预计会被添加或减去。 

我们不是重新计算所有子数组的总和，而是在两种可能性下维护以每个索引结尾的最佳交替总和：一种是添加当前元素，另一种是减去当前元素。 这允许我们扩展以前的子数组或在当前位置重新启动，确保线性时间处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) 到 O(n^3) | O(1) | O(1) | 太慢了 |
 | 最佳DP | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时维护两个值来描述以当前索引结尾的最佳交替子数组。 

1. 我们在每个位置定义两种状态：一种表示在此结束的子数组的最佳交替和，其中该元素被视为交替模式中的正位置，另一种表示该元素被视为负位置。 这种区别是必要的，因为符号取决于所选子数组的长度，而不是全局索引。 
2. 处理新元素时，我们计算结束子数组的最佳方式，并在此索引处做出正贡献。 要么我们在这里开始一个新的子数组，要么我们扩展以前的子数组，其中该元素将处于负位置，将其翻转回正位置。 这捕获了重新启动和持续的情况。 
3. 同样，我们计算结束子数组并减去该元素的最佳方式。 这是通过扩展先前为正值的状态，然后减去当前值，或者在需要时重新开始。 
4. 在每个索引处，我们使用所有子数组中的最佳值更新全局答案，这些子数组以正的最后贡献结束，因为有效的交替子数组总是以正号开头。 
5. 我们迭代整个数组一次，以恒定的时间更新每个元素的这两个状态。 

关键的不变量是，在每个位置，两个状态完全代表以该索引结束的所有可能的交替子数组，并根据最后一个位置在子数组内是奇数还是偶数进行划分。 任何以索引 i 结尾的有效子数组都必须恰好属于这两个类别之一，并且两种可能性都考虑扩展或重新启动，以确保不会错过任何候选解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        # dp_even: best subarray ending here where current position is even in subarray (i.e. '-')
        # dp_odd: best subarray ending here where current position is odd in subarray (i.e. '+')
        
        dp_odd = 0
        dp_even = 0
        ans = 0
        
        for x in a:
            new_odd = max(x, dp_even + x)
            new_even = max(-x, dp_odd - x)
            
            dp_odd = new_odd
            dp_even = new_even
            
            ans = max(ans, dp_odd)
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码维护两个滚动值。`dp_odd`表示以添加元素的当前位置结束的最佳交替子数组，这意味着子数组长度为奇数。 这对应于在当前元素处开始一个新的子数组或扩展先前的偶数位置状态。 表达式`dp_even + x`捕获连续性，同时`x`捕获重新启动。 

相似地，`dp_even`表示以减去元素的当前位置结束的子数组。 它可以来自扩展先前的`dp_odd`陈述或重新开始`-x`。 

答案总是取自`dp_odd`因为有效的交替子数组必须以正号开头。

该跟踪显示了在新索引处重新启动如何与扩展竞争，以及如何通过分离状态来保留交替奇偶校验。

 ## 边缘情况

 单元素数组是最简单的情况，其中答案必须等于该元素。 该算法自然地处理这个问题，因为`dp_odd`初始化为零并立即替换为`x`第一步，产生正确的结果。 

对于大值被较小值包围的数组，算法正确地决定是将大值包含在减法位置还是在其周围重新开始。 例如，在`[1, 100, 1]`，延伸到中间会导致减法，但重新启动允许捕获`100`积极地。 

当所有元素相等时，交替减法会减少增益，最优策略变为选择短段。 DP 在扩展和重新启动之间正确交替，确保它不会过度承诺长交替链而减少总数。
