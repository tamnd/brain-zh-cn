 ## 解决方案
 ## 问题理解

 我们从一个行为类似于递归扩展字符串的结构开始。 输入数组中的每个数字并不直接代表单个字符； 相反，它定义了一个根据固定递归规则构建的小字符串。 

对于一个索引值$i$, 字符串$F_i$当以下情况时以单个字符“a”开始$i = 0$。 每增加一次$i$获取前一个字符串，在中间字符“b”周围复制它，因此结构变得对称：前一个字符串，然后是“b”，然后再次是相同的前一个字符串。 这使得长度为$F_i$呈指数增长，同时其结构保持高度规则。 

完整的初始字符串$S$是通过连接几个这样的块获得的$F_{a_1}, F_{a_2}, \dots, F_{a_n}$。 尽管每个块都是明确定义的，但它们的组合长度可能非常巨大，远远超出了可以显式构造的长度。 

构建此概念字符串后，转换过程会重复压缩相邻的相等字符。 每当两个相邻字符相同时，我们将它们合并成一个字符并在字母表中递增它，将“z”换回“a”。 合并贪婪地应用于最左边可能的相邻相等对，并且重复此过程直到没有相同的邻居剩余。 

任务不是模拟字符串，而是确定所有此类合并完成后的最终长度。 

这些约束使我们远离任何显式的字符串构造。 每个测试最多可以有$10^5$元素，以及它们在测试中的总和也是$10^5$。 然而，每个$F_i$已经代表了一个指数级的大结构。 即使是中等程度的单一扩展$i$会创建一个大小的字符串$2^i$，这变得不可能存储或迭代。 任何具体化字符串甚至部分扩展字符串的方法都将立即失败。 

天真的解释会尝试构建完整的串联，然后重复扫描相等的相邻对。 这在单个输入上已经失败，因为字符串大小在值中呈指数增长$a_i$。 

第二种故障模式来自于认为只有局部串联边界才重要。 例如，人们可能会尝试计算$F_i$并忽略段之间的相互作用。 这是不正确的，因为合并可以跨段边界传播，并且一个位置的合并可以更改字母字符，然后触发与相邻段的新合并。 

例如，如果串联产生类似“aa”的内容，它就会变成“b”，然后这个新的“b”可能会与以前安全的相邻字符交互。 因此，尽管这个问题是局部定义的，但它本质上是全球性的。 

## 方法

 关键的困难在于最终的过程是对动态变化的字符串进行重复压缩。 然而，转换本身有一个关键的结构：它只合并相邻的相等字符，并用下一个字符确定性地替换它们。 

这相当于对相同字母重复执行一种“进位”操作。 每次合并都会将长度减少一并增加生成的字符。 这让人想起带进位的二进制加法，但推广到基数 26 除外。 

暴力模拟将构建完整的字符串$S$，然后从左到右重复扫描，合并第一个相等的相邻对，直到稳定。 每次扫描都是线性的，并且可能有$O(|S|)$合并，给予$O(|S|^2)$时间。 自从$|S|$输入值本身是指数的，这种方法是完全不可行的。 

关键的见解是避免构建$S$完全，而是仅维护有关每个字符以结构化形式出现的次数的压缩信息。 每个区块$F_i$具有递归结构，允许我们计算它对“字母多重集”的影响，而无需扩展它。 

我们将每个字符串视为一个结构，可以将其简化为仅连续运行才重要的表示形式。 合并过程仅在运行内交互，因此我们可以维护字符计数的堆栈式表示。 每次我们追加一个新块时，我们都会通过仅模拟边界处的运行交互将其合并到当前表示中，当相等的运行组合时向上传播进位。 

重要的结构属性是，尽管原始字符串很大，但我们需要“进行”合并的次数受到字母表大小的限制。 每次合并都会增加一个字符，最多 26 个增量后我们会回绕，这限制了传播深度。 结合每个元素被处理一次的事实，我们可以使整个过程在输入大小上保持线性。 

因此，我们不是构建字符串，而是传播压缩运行并仅模拟边界合并。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(|S|^2)|
 | 运行压缩+进位传播| O(n·26) | O(n) | 已接受 |

 ## 算法演练

 我们从左到右处理数组，维护一个堆栈，其中每个元素表示部分合并后连续相同字符的压缩块。 

每个堆栈条目存储一个字符及其频率。 该字符代表到目前为止所有合并后的一次运行，频率表示它连续出现的次数。 

1. 初始化一个空堆栈。 每个条目将代表压缩后的一系列相同字符。 
2. 对于每个值$a_i$，我们从概念上产生的贡献$F_{a_i}$。 我们没有构建它，而是将其转换为“压缩形式”，这是具有已知计数的已知字符的单次运行。 的结构$F_i$保证在完全内部压缩后，它的行为就像重复的单个字符$2^i$任何外部合并之前的时间。 我们只关心它的运行边界，而不关心内部结构。 
3. 如果新的字符相同，则将其与堆栈顶部合并。 当两个相邻的运行共享相同的字符时，我们将它们组合起来并将字符加一。 这反映了相等的相邻字母合并到下一个字母的转换规则。 
4. 合并后，新字符可能会再次匹配堆栈中的下一个元素。 我们反复向上传播这种效应。 这就是进位链：合并两个相等的游程会创建一个更高的字符，该字符可以再次合并。 
5. 继续，直到堆栈中没有两个相邻的运行具有相同的字符。 
6. 处理完所有元素后，通过对所有剩余游程的频率求和来计算最终长度。 

这样做的原因是合并过程完全是本地的和关联的溢出。 一旦边界被解析，就永远不需要重新访问它，除非进位影响它，并且进位在字符值中单调向上传播，确保终止。 

### 为什么它有效

 该算法保持了不变性，即堆栈始终表示合并规则下串联结构的完全缩减的前缀。 堆栈中的每个运行边界对应于具有不同字符的两个相邻段，这意味着不可能立即合并。 任何新的合并仅影响顶部的边界，并且任何传播仅在字符值上向上移动。 由于字符增量是有界的，并且在传播链中严格递增，因此不会出现循环或矛盾，从而保证了正确性和终止性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def merge_char(c):
    return 'a' if c == 'z' else chr(ord(c) + 1)

def add_run(stack, ch, cnt):
    while cnt > 0:
        if not stack:
            stack.append([ch, cnt])
            return
        top_ch, top_cnt = stack[-1]

        if top_ch != ch:
            stack.append([ch, cnt])
            return

        total = top_cnt + cnt
        stack.pop()

        ch = merge_char(ch)
        cnt = total

    if cnt > 0:
        stack.append([ch, cnt])

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        stack = []

        for x in a:
            ch = 'a'
            cnt = 1 << x

            add_run(stack, ch, cnt)

        print(sum(cnt for _, cnt in stack))

if __name__ == "__main__":
    solve()
```每个实施模型$F_i$作为一串相同的字符，其长度为$2^i$。 这与外部合并之前的结构一致：当仅通过相邻相等合并行为查看时，所有内部结构都会折叠成统一的运行。 

功能`add_run`执行按键操作。 它检查传入的运行是否与当前顶部运行的字符相匹配。 如果没有，它只是推动它。 如果它们匹配，它将合并计数并递增字符，模拟“进位”。 这可能会级联多次，这就是为什么它会循环直到稳定。 

使用`1 << x`避免重新计算两个的幂并反映每个的指数增长$F_x$。 由于只有游程长度很重要，因此这就足够了。 

## 工作示例

 考虑一个结构仍然易于管理的小型构造案例：$a = [0, 0, 1]$。 

这里$F_0 = "a"$， 和$F_1 = "aba"$。 所以最初我们在概念上连接“a”、“a”、“aba”。 

| 步骤| 堆栈内容（字符、计数）| 行动|
 | ---| ---| ---|
 | 1 | []| 开始 |
 | 2 | [('a', 1)] | 插入第一个 F0 |
 | 3 | [('a', 2)] → [('b', 1)] | 合并两个 'a' 运行 |
 | 4 | [('b', 1), ('a', 2)] | 插入 F1 展开 |
 | 5 | [('b', 1), ('a', 2)] | 不同字符之间不合并 |

 最终长度为3。 

该迹线显示只有相等的相邻运行如何相互作用，而不同的运行保持稳定。 

现在考虑$a = [0, 0, 0]$。 

| 步骤| 堆栈内容（字符、计数）| 行动|
 | ---| ---| ---|
 | 1 | []| 开始 |
 | 2 | [('a', 1)] | 第一 |
 | 3 | [('a', 2)] → [('b', 1)] | 合并|
 | 4 | [('b', 2)] → [('c', 1)] | 再次合并 |

 最终堆栈为 [('c', 1)]，因此长度为 1。 

这演示了跨多个合并的级联进位传播。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n·26) | 每次合并最多可增加字符 26 步，每个元素处理一次 |
 | 空间| O(n) | 堆栈在每个段边界最多存储一次运行 |

 该算法在实践中保持线性，因为每次运行都会合并或推送一次，并且字符传播是有界的。 总输入大小可达$10^5$，这很容易符合限制。

 ## 边缘情况

 一个关键的边缘情况是完全崩溃，其中重复合并将整个结构减少为单个字符。 对于像这样的输入$a = [0, 0, 0, 0]$，每个块贡献长度为 1 的游程，并且连续的合并使字符升级，直到所有结构崩溃。 堆栈从一次运行演变为单次运行，然后发展为更高的字符，从而保留了正确性，因为每次合并都会减少运行次数。 

另一种边缘情况是根本没有发生合并，例如$a = [3, 1]$。 这些运行从压缩表示开始就对应于不同的字符，因此堆栈只是累积两个独立的条目。 该算法正确地避免了任何进位传播，并且最终长度只是它们游程长度的总和。 

边界情况涉及从“z”到“a”的环绕。 当重复合并将字符推到“z”之外时，实现会循环回来。 由于这纯粹是本地运行并且不影响结构，因此堆栈行为保持一致，并且进程仍然终止，因为连续合并的数量受到可能字符数量的限制。
