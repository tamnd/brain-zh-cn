 ## 解决方案
 ## 问题理解

 网格描述了一个城市，其中每个单元格都有一个代表其类型的字母标签。 城市内的移动受到一种特殊的限制：任意单元格之间不允许直接移动。 相反，运动总是分为两个阶段。 首先，从你当前的牢房，你可以“传送”到与你当前牢房具有相同字母的任何其他牢房，而无需支付任何费用。 完成此传送后，您可以选择乘坐地铁前往网格中的任何单元格，并支付乘坐起点和目的地之间的曼哈顿距离。 

一次旅行包括访问由字符串给出的一系列所需类型。 在第 i 天，您必须以某个类型与计划的第 i 个字符匹配的单元格结束。 您可以选择要结束的确切单元格，并且可以在每次付费移动之前利用传送。 

关键的困难在于，传送可以让你在支付地铁搬家费用之前在连接的“类型类别”中任意重新定位自己。 这意味着旅行者的有效状态不是单个单元格而是字母类型，连续两天之间的成本取决于相应类型的最佳代表单元对。 

限制非常大：网格每个测试用例最多可以有一百万个单元，并且计划也可以最多一百万个步骤，最多一万个测试用例。 任何试图直接考虑所有字母出现之间的转换或模拟网格上每步移动的解决方案都是不可能的。 唯一可行的方法必须将每个字母简化为一个小的摘要结构。 

当一个字母多次出现在网格中时，就会出现微妙的边缘情况。 一种幼稚的方法可能假设我们只需要每个字母一个代表位置，例如第一次出现。 这会失败，因为两个字母之间的最佳移动取决于在两个集合中选择最佳的出现对。 另一种失败情况是当同一个字母出现在相距很远的多个簇中时； 选择一个代表可能会严重高估距离。 

## 方法

 如果我们忽略隐形传输，问题将减少为每一步选择一个网格单元并在连续选择之间支付曼哈顿距离。 这已经很昂贵了，因为每一步都涉及在潜在的 n·m 个候选者中进行选择。 

强力解释将尝试计算每对连续的所需字母，第一个字母的任何出现与第二个字母的任何出现之间的最小曼哈顿距离。 原则上这是正确的，因为远距传送允许我们在当前字母出现的任何地方结束每一天。 然而，计算两个字母所有位置之间的成对距离太慢：如果一个字母出现 O(nm) 次，将其与另一个频繁出现的字母进行比较将导致每对的二次行为，并且最多可以有 26 个字母，但位置仍然太多。 

关键的观察结果是，一旦我们隔离了每个字母的所有位置，问题就变成了曼哈顿距离下网格中的点集之间的转换序列。 曼哈顿距离的结构允许经典的简化：可以使用坐标变换来计算两组点之间的最小距离，将曼哈顿距离转变为我们只需要极值的形式。

对于两个点 (x1, y1) 和 (x2, y2)，曼哈顿距离为 |x1 − x2| + |y1 − y2|。 展开绝对值可得到四种线性形式：(x + y)、(x − y)、(−x + y)、(−x − y)。 对于任何固定对的集合 A 和 B，最小曼哈顿距离是通过匹配这些变换坐标中的极值投影来实现的。 因此，对于每个字母，预先计算所有出现的 x+y 和 x−y 的最小值和最大值就足够了。 这四个数字充分描述了该字母的几何形状，用于距离查询。 

一旦每个字母被压缩为四个极值，计算字母之间的转换成本就变成了 O(1)。 最终的答案只是计划中连续字符之间的成本之和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 所有事件中的强力成对距离 | O(k·(nm)^2) | O(k·(nm)^2) | O(纳米) | 太慢了|
 | 预先计算每个字母的极值投影 | O(nm + k) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 扫描一次网格并收集每个字母的所有坐标。 在此阶段，我们正在构建完整的位置图，因为远距离传送使字母的每次出现都相当于起点或终点。 
2. 对于每个字母，计算四个值：x + y 的最小值和最大值，以及所有出现的 x − y 的最小值和最大值。 这些总结了字母在两个对角线方向上的分布，这是与曼哈顿距离分解相关的唯一方向。 
3. 预先计算任意两个字母 a 和 b 之间的成本函数。 对于四个转换后的指标中的每一个，通过组合极端值来考虑 a 和 b 之间的最坏情况配对。 最佳曼哈顿距离是从极值导出的这四个候选差异中的最小值。 
4. 遍历计划字符串并对连续字母之间的成本求和。 每个转换代表一天在前一个字母的最佳出现处结束，然后选择下一个字母的最佳起点。 

这样做的原因是，传送消除了在支付移动费用之前您所站立的字母的任何限制。 因此，每个字母实际上都是一组点，只有两组点之间的最小曼哈顿距离才重要。 极值变换保证捕获所有对的最小值，而无需枚举它们。 

保持的不变性是，在处理计划中的每个字符之后，我们在概念上以该字母的最佳出现结束，从而最小化未来的转换成本，即使我们从未明确跟踪该位置。 所有可能的位置都通过极值投影隐式表示。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m, k = map(int, input().split())
        
        INF = 10**30
        min_xp = {}
        max_xp = {}
        min_xm = {}
        max_xm = {}
        
        for i in range(n):
            row = input().strip()
            for j, ch in enumerate(row):
                x, y = i, j
                xp = x + y
                xm = x - y
                
                if ch not in min_xp:
                    min_xp[ch] = INF
                    max_xp[ch] = -INF
                    min_xm[ch] = INF
                    max_xm[ch] = -INF
                
                min_xp[ch] = min(min_xp[ch], xp)
                max_xp[ch] = max(max_xp[ch], xp)
                min_xm[ch] = min(min_xm[ch], xm)
                max_xm[ch] = max(max_xm[ch], xm)
        
        P = input().strip()
        
        def dist(a, b):
            # compute min Manhattan distance between any occurrence of a and b
            candidates = []
            
            # using x+y projection
            candidates.append(max(abs(min_xp[a] - max_xp[b]), abs(max_xp[a] - min_xp[b])))
            # using x-y projection
            candidates.append(max(abs(min_xm[a] - max_xm[b]), abs(max_xm[a] - min_xm[b])))
            
            return min(candidates)
        
        ans = 0
        for i in range(1, k):
            ans += dist(P[i-1], P[i])
        
        print(ans)

if __name__ == "__main__":
    solve()
```网格扫描构建了四个按字符键入的极值字典。 这避免了显式存储超出聚合所需的所有位置。 每个单元的每次更新都是恒定时间。 

过渡函数`dist`实现关键的几何缩减。 它使用两个字母的极值投影来比较它们。 仅需要两个候选表达式，因为在考虑跨集合的最小到最大配对时，四个曼哈顿变换对称地崩溃。 

最后的循环累积计划中连续字符的成本。 不需要前一个字符之外的状态，因为一旦字母被压缩，每个步骤都是独立的。

作为第二个示例，考虑一个退化网格，其中所有单元格都是相同的字母。 然后所有极值都重合，并且每个距离都变为零。 该算法正确地将所有转换折叠到零成本，符合仅传送就足够的直觉。

 ## 边缘情况

 一个字母出现在两个相距较远的区域的网格是主要的压力测试。 假设有一封信`a`出现在 (0,0) 和 (100,100) 处，而`b`出现在 (0,100) 和 (100,0) 处。 天真的代表选择可能会选择错误的配对并高估距离。 极值方法通过 x+y 和 x−y 捕获两个对角线方向，确保始终表示最小配对。 

在像“aba”这样的计划中，算法首先计算 dist(a,b)，然后计算 dist(b,a)。 由于极值公式中保持对称性，因此两种转变都是一致的，并且不依赖于隐式选择的发生。 

最后一个极端情况是单字符计划。 转换循环永远不会运行，因此答案仍然为零，这与不需要地铁移动的事实相匹配。
