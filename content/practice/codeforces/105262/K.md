 ## 解决方案
 ## 问题理解

 我们得到了几个独立的实验。 每个实验都有一个固定的正整数阈值$w$，这是未知的，但在同一测试用例的所有实验中都是一致的。 每个实验提供了一个正整数数组，代表从左到右放置的番茄重量。 从第一个番茄开始，我们模拟按顺序进食：我们保留吃掉的总重量，并继续，只要添加下一个番茄不超过$w$。 一旦下一个番茄的总数严格大于$w$，我们永久停止该实验。 实验的输出是未吃掉的西红柿数量。 

所以每个实验都描述了一个前缀和约束：如果第一个实验的前缀和$k$西红柿是$\le w$，它们被吃掉； 否则该过程将停止在前缀超出的第一个位置$w$。 

经过多次实验，结果相同$w$必须解释所有观察到的“剩余计数”。 任务是确定最小的这样$w$，或得出结论：没有值$w$可以满足每一个实验。 

约束很大：所有实验的总数组长度可达$10^6$，并且最多有$10^4$测试用例。 这排除了尝试所有候选值的任何解决方案$w$或对每个猜测重复进行模拟。 任何有效的解决方案都必须以线性时间处理每个实验，并结合每个实验基本恒定或对数时间的约束。 

一种简单的重建方法是尝试以下候选值$w$，模拟每个实验，并检查所得的剩余计数是否与给定的匹配$m$。 由于权重上升到$10^9$并且前缀和可以达到$10^{15}$或更多，搜索空间$w$是巨大的，并且每次模拟都是$O(n)$，使得这不可行。 

当实验相互矛盾时，就会出现微妙的边缘情况。 例如，一项实验可能会迫使$w \ge 10$而另一种力量$w \le 5$。 另一种故障模式是假设停止位置仅取决于总和而不取决于前缀结构； 总权重相同但前缀不同的两个数组可能意味着不同的有效值$w$。 

## 方法

 关键的观察是每个实验都没有描述$w$直接，但描述了在前缀和交叉之前我们可以移动多远$w$。 让$k$是实验中吃掉的西红柿的数量。 然后$k = n - m$。 该过程意味着首先$k$西红柿完全消耗掉，或者$k = n$或$(k+1)$第一个番茄是第一个不能拿走的。 

这转化为两种类型的约束：

 如果$k = n$，那么所有的西红柿都被吃掉了，意味着数组的总和最多为$w$。 

如果$k < n$，然后我们正好吃到了第一个$k$西红柿，所以前缀和$S_k$必须满足$S_k \le w$，以及下一个番茄重量$a_{k+1}$使得无法继续进行，所以$S_k + a_{k+1} > w$。 这给出了严格的上限：$w < S_k + a_{k+1}$。 

所以每个实验都会产生一个下限$w \ge S_k$或约束区间$S_k \le w < S_k + a_{k+1}$，取决于进程是提前结束还是耗尽了所有内容。 

因此，所有实验都简化为对单个整数变量的相交约束$w$。 最终的答案是所有有效区间的交集中的最小整数。 如果交集为空，则没有可行的$w$。 

蛮力方法会枚举所有可能的$w$直到最大前缀总和并测试每个实验，成本$O(U \cdot n)$， 在哪里$U$可以达到$10^{15}$，这是不可能的。 

相反，我们只维护一个全局可行区间$[L, R)$。 每个实验都使用线性时间内计算的前缀和来收紧此间隔。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解结束$w$|$O(U \cdot n)$|$O(1)$| 太慢了|
 | 通过前缀和进行区间交集 |$O(\sum n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个实验，并不断完善全球可行范围$w$。 

1. 初始化可行范围为$L = 0$,$R = 10^{18} + 1$。 这个范围代表了所有可能的值$w$与迄今为止处理的实验一致。 
2. 对于每个实验，计算$k = n - m$，吃西红柿的数量。 
3. 构建前缀和，同时扫描数组直到位置$k+1$，因为只有这个边界对于约束很重要。 
4. 如果$k = n$，那么所有的西红柿都被吃掉了。 这意味着总和$S_n$必须满足$w \ge S_n$。 更新$L = \max(L, S_n)$。 原因是任何较小的$w$本来会早点停下来的。 
5.如果$k < n$, 计算$S_k$并检查下一个元素$a_{k+1}$。 恰好停止的条件$k$是$S_k \le w < S_k + a_{k+1}$。 通过相交更新可行区间：$L = \max(L, S_k)$,$R = \min(R, S_k + a_{k+1})$。 这捕获了“足够的容量来吃 k 个项目”和“不足以吃下一个项目”。 
6. 处理完所有实验后，检查是否$L \ge R$。 如果是，则没有整数$w$满足所有约束，因此输出“False Hypothesis”。 
7.否则输出$L$，这是可行区间中的最小整数。 

它起作用的原因是每个实验都定义了一个凸约束$w$，半线或有界区间。 吃饭的过程是单调的$w$: 增加$w$只能增加或保留吃西红柿的数量。 因此，每个实验都会转化为一组连续的有效数据$w$价值观。 将这些集合相交可以保持正确性，并且交集中的最小值是最小可行值$w$贯穿所有实验。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        e = int(input())
        L, R = 0, 10**18 + 1

        for _ in range(e):
            n, m = map(int, input().split())
            a = list(map(int, input().split()))

            k = n - m

            if k == 0:
                # nothing eaten, so w < a[0]
                R = min(R, a[0])
                continue

            if k == n:
                # all eaten: w >= total sum
                s = 0
                for x in a:
                    s += x
                L = max(L, s)
                continue

            s = 0
            for i in range(k):
                s += a[i]
            next_w = a[k]

            L = max(L, s)
            R = min(R, s + next_w)

        if L >= R:
            print("False Hypothesis")
        else:
            print(L)

if __name__ == "__main__":
    solve()
```该代码维护一个全局可行区间并根据实验对其进行细化。 前缀和仅计算到停止边界$k$，这可以避免在不需要约束时对整个数组进行不必要的工作。 

一个微妙的点是处理$k = 0$，这意味着他们不能吃任何西红柿。 在那种情况下，$w$必须严格小于第一个元素，这成为约束$w < a_1$，实施为$R = \min(R, a_1)$。 

整数安全性很重要，因为总和可以达到$10^{15}$，但 Python 整数自然可以处理这个问题。 上限$10^{18}$确保最终答案范围是安全的。

 ## 边缘情况

 一个关键的边缘情况是实验中没有吃番茄。 对于输入：```
1
1
3 1
5 1 1
```我们有$k = 0$。 算法集$R = a_1 = 5$， 意义$w < 5$。 任何假设仅前缀和重要的不正确方法都会错过严格的不等式并允许$w = 5$，这是无效的，因为它允许吃第一个番茄。 

另一种情况是当所有西红柿都吃完时：```
1
1
3 0
1 2 3
```这里$w \ge 6$。 仅使用停止点的解决方案会错过这一点并错误地允许更小的$w$。

2

```力量$w < 5$和$w \ge 6$，不产生交集。 该算法通过以下方式检测到这一点$L \ge R$并正确输出故障。
