 ## 解决方案
 ## 问题理解

 我们得到一个网格，代表一个城市，分为 n × m 块。 每个街区可能包含一家咖啡店，提供卡布奇诺、热巧克力、两种饮料或不提供任何饮料。 两个人从该网格的对角开始：一个从左上角的单元格 (1, 1) 开始，另一个从右下角的单元格 (n, m) 开始。 

两者都可以在四个基本方向上一次移动一步。 每次移动通常需要花费 1 个单位的努力，但是有一个特殊的规则可以根据当前单元格上的咖啡店类型来降低此成本。 如果一个人站在提供他们喜欢的饮料的牢房上，那么他们可以移动到任何相邻的牢房，而无需花费精力。 否则，搬家费用为1。 

第一个人更喜欢热巧克力，因此含有 2 型或 3 型的细胞充当他们的“自由移动来源”。 第二个人更喜欢卡布奇诺，因此包含 1 型或 3 型的细胞是他们的自由移动来源。 

目标是计算两个人必须花费的最小总努力，以便他们能够到达网格中某处的同一单元格。 

所有测试用例的约束 n · m ≤ 10^6 意味着所有输入的网格单元总数在大小上是线性的，因此任何解决方案都必须接近 O(nm) 或 O(nm log nm)。 任何类似对两个人同时进行的成对状态探索（规模为 (nm)^2 或更糟）在一定范围内都是不可能的。 

一个天真的但诱人的想法是一起模拟两个动作，跟踪两个人的位置。 这导致状态空间的大小为 nm × nm，这远远超出了可行的范围。 

如果一个人试图贪婪地在几何上“看起来最近”的单元格中相遇，就会出现更微妙的失败情况。 例如，如果由于附近缺乏自由移动区域而导致两个单元之一无法以低廉的价格到达中心附近的单元，则该单元可能不是最佳的。 最佳交汇点取决于最短路径结构，而不是曼哈顿距离。 

## 方法

 该问题的强力解释将其视为二代理最短路径问题。 我们将跟踪状态（x1，y1，x2，y2），并且每一步都会根据同步规则移动一个或两个玩家。 即使我们将移动简化为独立路径，尝试所有交汇点仍然需要计算从两个起点到每个单元的最短路径。 如果没有优化，每个单元或每对运行最短路径搜索在最坏的情况下会导致大约 O((nm)^2) 行为，因为每个 BFS 或 Dijkstra 扩展都会重新访问大部分网格。 

关键的观察结果是，两个人直到最后的会面点才进行互动。 它们的路径是独立的，并且两者的成本结构在形式上是相同的，区别仅在于哪些单元提供零成本转换。 这意味着我们可以分别预先计算从每个起点到达每个单元格的最小成本。 一旦我们有了这两个距离网格，答案就是它们的所有单元格之和的最小值。 

成本函数的结构至关重要。 移出单元格要么免费，要么花费 1，仅取决于单元格类型。 这正是边权重为 {0, 1} 的网格图上的最短路径问题，可以使用 0-1 BFS 有效解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 联合国家搜索| O((纳米)^2) | O((纳米)^2) | 太慢了|
 | 两次独立的 0-1 BFS 运行 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们通过计算两个独立的距离网格（每个人一个）来解决这个问题。

1. 从(1, 1)开始为第一个人建立一个距离网格。 将所有距离初始化为一个大值并将起始单元格设置为 0。 
2. 从 (1, 1) 运行 0-1 BFS。 从单元格扩展时，确定移动到邻居的成本是 0 还是 1。成本仅取决于当前单元格是否包含热巧克力（类型 2 或 3）。 如果是，所有出局动作都是免费的； 否则，它们的成本为 1。使用双端队列更新距离：对于成本 0 的边推到前面，对于成本 1 的边推到后面。 
3. 对从 (n, m) 开始的第二个人重复相同的过程，但现在自由移动条件取决于卡布奇诺的可用性（类型 1 或 3）。 
4. 计算完两个距离网格后，迭代网格中的每个单元格并计算 distA[i][j] + distB[i][j]。 跟踪最小总和。 
5. 返回该最小值作为答案。 

唯一的微妙之处在于，“自由移动”条件适用于您要离开的牢房，而不是您要进入的牢房。 这使得图的边仅依赖于源节点，这正是 0-1 BFS 能够干净地工作的原因。 

### 为什么它有效

 BFS 中的每个状态都代表处于特定的单元，并且具有已知的到达该单元的最低成本。 由于边权重仅为 0 或 1，并且仅取决于当前单元类型，因此在标准 0-1 BFS 排序下保留了最短路径属性。 双端队列确保以成本 0 达到的任何状态都会在以成本 1 达到的状态之前处理，从而保持正确的非递减探索顺序。 由于两个代理独立计算到每个单元的真实最短路径成本，因此任何交汇点都结合了两条最佳独立路径，并且在所有单元上取最小值保证了全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

INF = 10**18

def bfs(grid, n, m, sx, sy, good_set):
    dist = [[INF] * m for _ in range(n)]
    dq = deque()
    dist[sx][sy] = 0
    dq.append((sx, sy))

    while dq:
        x, y = dq.popleft()
        d = dist[x][y]

        # determine cost of moving out of (x, y)
        w = 0 if grid[x][y] in good_set else 1

        for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < m:
                nd = d + w
                if nd < dist[nx][ny]:
                    dist[nx][ny] = nd
                    if w == 0:
                        dq.appendleft((nx, ny))
                    else:
                        dq.append((nx, ny))
    return dist

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        grid = [list(map(int, input().split())) for _ in range(n)]

        dist1 = bfs(grid, n, m, 0, 0, {2, 3})
        dist2 = bfs(grid, n, m, n - 1, m - 1, {1, 3})

        ans = INF
        for i in range(n):
            row1 = dist1[i]
            row2 = dist2[i]
            for j in range(m):
                ans = min(ans, row1[j] + row2[j])

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将两个旅行者完全分开，并将问题简化为两个最短路径计算。 每个 BFS 使用双端队列来维护 0-1 BFS 排序，确保网格边数的线性复杂度。 

一个常见的实施陷阱是错误地将饮料条件应用到目标单元格而不是源单元格。 这会改变图模型并破坏 0-1 BFS 正确性，因为每个传出状态的边权重将不再一致。 

另一个微妙的点是内存布局：存储两个距离的完整网格是必要的，但仍然安全，因为测试中的单元总数以 10^6 为界。 

## 工作示例

 考虑一个小网格，其中一个或两个单元提供自由移动的机会。 

### 示例 1

 网格：```
2 0
0 1
```在这里，第一个人受益于热巧克力细胞（2或3），第二人受益于卡布奇诺细胞（1或3）。 

我们计算两个距离图。 

| 步骤| 细胞| Dist1（从 1,1）| Dist2（从 2,2）|
 | ---| ---| ---| ---|
 | 初始化| (1,1) / (2,2) | 0 / 信息 | 信息 / 0 |
 | BFS1 | 展开 (1,1) | 邻居更新 | |
 | BFS2 | 展开 (2,2) | | 邻居更新 |

 两次 BFS 运行后，我们检查所有单元格并选择最小总和 dist1 + dist2。 交汇点自然地出现为双方都可以廉价到达的单元，而不一定是几何中心。 

这表明独立的最短路径就足够了，不需要同步逻辑。 

### 示例 2

 网格：```
3 3 3
0 0 0
1 0 2
```最上面一排最终允许两名旅客自由移动（因为 3 人对双方都有帮助），从而创建了一条零成本过渡的走廊。 

来自每个角落的 BFS 以零成本扩展沿着该走廊快速传播，这证实了一旦到达一个好的单元格，网格的大部分就变得有效地廉价遍历。 

最终的最小值出现在通过重叠的低成本区域可到达的小区处。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | 每次测试 O(nm) | 每个单元在两次 0-1 BFS 运行中被处理恒定次数 |
 | 空间| O(纳米) | 两个距离网格加上双端队列存储 |

 由于所有测试用例的单元总数最多为 10^6，因此该解决方案可以轻松满足时间和内存限制。

 ## 边缘情况

 单细胞网格是最干净的边界情况。 两个人在同一个细胞上开始和结束，因此无论细胞类型如何，答案都必须为零。 BFS 正确初始化起始单元，因此两个距离数组在该位置都包含零，并且最小和为零。 

没有特殊单元（全零）的网格将问题简化为具有统一成本边缘的两个独立的最短路径计算。 BFS 退化为标准的多步最短路径，两个距离都反映了依赖于障碍物的类似曼哈顿的移动成本，确保了正确性而不依赖于任何自由转换。 

充满 3 型细胞的网格可创造最大的自由度。 每个单元都允许零成本移动，因此两次 BFS 运行都会将成本为零的起始单元传播到整个网格。 每个单元格都成为总成本为零的有效交汇点，并且算法通过评估所有可能的交汇单元格正确地返回零。
