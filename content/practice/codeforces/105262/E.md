 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，节点 1 固定为根。 两名玩家交替行动，从秘密伙伴开始。 移动包括选择根以外的任何节点并删除该节点及其子树中的每个节点。 做出最后有效移动的玩家获胜，这意味着面临没有非根节点的位置的玩家失败。 

从更结构化的角度来看，每一步都会折叠整个有根子树，并且根本身永远不允许被选择或删除。 一旦除根之外的每个节点都被直接或通过早期删除间接删除，游戏就会结束。 

这些约束允许所有测试用例最多有 4 · 10^5 个节点，这排除了任何尝试在每次移动后模拟游戏状态的策略。 即使是重新计算子树或更新每次移动的动态结构的单一模拟也已经太慢了，因为在最坏的情况下操作数量可能呈二次方增长。 预期的解决方案必须将游戏简化为每个测试用例可在线性时间内计算的属性，理想情况下仅以非常粗略的方式依赖于树的结构。 

当局部思考移动时，会出现一个微妙的问题。 树深处的节点仅删除其自己的子树，而较高的节点则删除树的更大部分。 采取“大”动作是否有利，或者最佳游戏是否更喜欢最小的移除，这并不明显。 这个选择的分辨率决定了整个游戏的结果。 

## 方法

 思考博弈的一个直接方法是将其视为树状态上的标准公正博弈。 每个位置都是一个以 1 为根的剩余森林，每次移动都会删除整个子树。 人们可以尝试为每个可能的有根森林分配一个 Sprague-Grundy 值，但状态空间是巨大的，因为删除子树会改变全局结构，并且不会干净地分解为独立的子博弈。 任何显式计算 Grundy 值的尝试都需要在最坏的情况下探索指数级的许多配置，而这立即变得不可行。 

关键的简化来自于观察行动中真正重要的因素。 每一步都会删除至少一个节点，并且根据所选的顶点，它可能会删除更多节点。 关键的结构性观察是，删除不必要的内容永远不会产生额外的未来动作，它只会删除选项。 任何消失的节点本身就是一个潜在的举动，因此大量删除严格减少了双方未来可用决策的数量。 

这将问题变成了一个单调的过程：唯一消耗的资源是可移动节点的集合，并且每次移动至少消耗其中一个。 如果两个玩家都采取最佳行动，那么任何一方都没有动力去移动不必要的东西，因为这样做只会缩短游戏时间并减少他们自己稍后移动的次数。 

这导致了集中减少：在最佳游戏中，可以假设每一步都恰好删除一个节点。 一旦这一点被接受，整个游戏的长度就变得确定了。 由于正好有 n − 1 个可移动节点（除根之外的所有节点），因此无论树的形状如何，游戏都会持续 n − 1 次移动。 因此，获胜者纯粹由 n − 1 的奇偶性决定。

 ## 算法演练

 该解决方案根本不需要模拟动作； 它只提取树的大小。

1. 读取测试用例的节点数 n。 这完全确定了答案，因为一旦我们接受最佳游戏行为，树结构就变得无关紧要。 
2. 观察每一步移动至少删除一个节点，在最佳游戏中，我们可以将注意力限制在仅删除一个节点的移动上。 理由是删除额外的节点只会消除未来的移动，否则会延长游戏时间。 
3. 由于游戏从 n − 1 个非根节点开始，并且每次移动都可以被视为恰好删除其中一个，因此任何最佳游戏序列中的移动总数固定为 n − 1。 
4. 通过检查 n − 1 的奇偶性来确定获胜者。如果 n − 1 是奇数，则第一个玩家（秘密伙伴）做出最后一步并获胜。 否则，第二个玩家（艾德）获胜。 

### 为什么它有效

 该解决方案背后的不变量是唯一相关的数量是剩余非根节点的计数。 每个合法的移动都严格地使该计数至少减少 1，并且任何将其减少超过 1 的移动都可以被这些已删除节点上的一系列移动所取代，而不会改变最佳结果。 这种等价性意味着所有最佳玩法都可以转换为每回合仅删除一个节点的序列，保留回合顺序和获胜者，同时最大化移动次数。 由于最终状态是固定的并且与策略无关，因此可移动节点的初始计数的奇偶性决定了获胜者。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        # read and discard edges
        for _ in range(n - 1):
            input()
        # game reduces to parity of n - 1 moves
        if (n - 1) % 2 == 1:
            out.append("The Secret Partner")
        else:
            out.append("Eddard")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码直接反映了减少。 所有边都是只读的，以消耗输入； 它们在计算中不起任何作用。 每个测试用例唯一有意义的值是 n，并且决策是对 n − 1 进行简单的奇偶校验。 

这里的一个常见错误是尝试构建邻接列表或计算子树大小，这是不必要的开销，并且在完全约束下存在 TLE 风险。 

## 工作示例

 考虑一棵 n = 4 的小树。有 3 个非根节点，因此游戏长度为 3 步。 自秘密伙伴开始后，他们进行了第 1 步和第 3 步，因此他们获胜。 

| 转 | 剩余非根节点| 动作解读|
 | ---| ---| ---|
 | 1（秘密伙伴）| 2 | 删除一个节点 |
 | 2（艾德）| 1 | 删除一个节点 |
 | 3（秘密伙伴）| 0 | 删除最后一个节点 |

 这证实了当 n − 1 为奇数时，第一个玩家将采取最后一步。 

现在考虑 n = 5。有 4 个非根节点，因此发生 4 次移动。 

| 转 | 剩余非根节点| 动作解读|
 | ---| ---| ---|
 | 1（秘密伙伴）| 3 | 删除一个节点 |
 | 2（艾德）| 2 | 删除一个节点 |
 | 3（秘密伙伴）| 1 | 删除一个节点 |
 | 4（艾德）| 0 | 删除最后一个节点 |

 在这里，第二个玩家做出了最后一步，所以他们获胜。 

这些痕迹表明，树的结构不会影响序列长度，只有节点的初始计数会影响序列长度。

 ## 边缘情况

 单节点情况突出了不存在移动的边界。 当 n = 1 时，没有有效的选择，因此秘密伙伴立即失败。 该公式给出 n − 1 = 0，这是偶数，正确地产生第二个玩家获胜。 

对于 n = 2，恰好存在一步棋。 秘密伙伴删除唯一的非根节点，留下根并结束游戏，因此他们获胜。 奇偶校验规则给出 n − 1 = 1，奇数，与此结果匹配。 

在较大的树（例如星形树）中，所有节点都是根的直接子节点这一事实不会改变任何东西。 尽管原则上移动可以删除不同数量的节点，但最佳游戏减少为单节点删除，并且结果仍然仅取决于可移动节点总数是奇数还是偶数。
