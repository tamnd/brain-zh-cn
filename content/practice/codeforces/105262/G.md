 ## 解决方案
 ## 问题理解

 给定一个整数数组，我们查看每个可能的连续子数组。 对于每个子数组，我们根据简单的对称条件为其分配一个值。 如果子数组从左到右和从右到左读取的内容相同，则将其元素之和作为其值。 如果它不对称，则其值为零。 任务是计算所有子数组的总贡献，这实际上意味着仅将所有回文子数组的总和相加。 

输入大小足够大，子数组的二次枚举不可行。 由于测试用例的总长度高达 10^6，任何检查所有 O(n^2) 子数组甚至每个子数组执行 O(n) 工作的解决方案都将大大超出时间限制。 可行的方法必须将每个测试用例的问题减少到基本上线性或接近线性的时间。 

一个微妙的点是，我们不是在计算回文子数组，而是对它们的内部和进行求和。 这使得问题比经典的回文计数更难，因为每个有效子数组贡献一个加权值而不是一个单位。 

当试图预先计算所有回文，然后分别对范围求和时，就会出现一个天真的陷阱。 即使优化回文检测，重新计算每个子数组的总和仍然会崩溃。 

另一个重要的边缘情况是所有元素都相等。 每个子数组都是对称的，因此答案成为所有子数组总和的总和，它必须与已知的算术结构匹配。 任何依赖结构不对称的解决方案仍然需要正确处理这种极端情况。 

## 方法

 直接暴力方法迭代每个子数组，检查它是否是回文，如果是，则计算其总和。 检查对称性需要 O(length)，计算总和也是 O(length)，因此在最坏情况下每个子数组的成本为 O(n)。 由于有 O(n^2) 个子数组，这会导致 O(n^3) 时间复杂度，这对于 n 达到 10^6 时完全不可行。 

我们可以通过观察回文检查可以通过中心扩展或散列来优化，将对称性验证减少到 O(1) 或 O(log n) 来改进这一点。 即使这样，显式地对每个子数组求和总体上仍然花费 O(n^2)，因此这个方向仍然失败。 

结构观察的关键是反转视角。 我们不是遍历子数组并检查它们是否是回文，而是考虑每个元素并询问有多少个回文子数组包含它，以及它在这些子数组中扮演什么角色。 

对称子阵列完全由其中心周围的镜像对决定。 每个回文子数组贡献其全部总和，即各个元素的贡献之和，根据有多少回文子数组在特定位置包含该元素进行加权。 

这表明了一种基于贡献的计数策略。 如果我们可以计算出每个位置有多少个回文子数组使用它，以及它出现在哪些对称角色中，那么我们就可以在线性时间内聚合贡献。 经典的工具是 Manacher 算法，它枚举以 O(n) 中每个位置为中心的所有回文子串。 一旦我们知道了回文半径，我们就可以得出每个位置作为左镜像贡献或右镜像贡献参与了多少个回文子数组。 

然后，我们将其与前缀和结合起来，以快速计算范围和，并累积所有有效回文扩展的贡献。

 | ---| ---| ---| ---|
 | 蛮力 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了|
 | 最优（Manacher + 贡献）| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将数组转换为适合回文扩展的格式，然后使用线性时间回文半径计算，最后将这些半径转换为贡献计数以进行求和累加。

1. 将数组转换为转换后的序列，将元素分开以统一处理偶数和奇数长度的回文。 我们在元素之间插入分隔符，以便每个回文在变换后的空间中成为奇数长度的回文。 这避免了单独处理两种情况。 
2. 在原始数组上构建一个前缀和数组，以便可以在 O(1) 内计算任何子数组和。 这是必要的，因为一旦我们识别出一个回文子数组，它的贡献就是它的范围和。 
3. 对变换后的数组运行 Manacher 算法，计算每个中心的最大回文半径。 每个中心描述了以该中心为中心的所有回文子数组。 
4. 对于每个中心，将其回文半径转换回原始数组中的有效子数组边界。 每个半径对应于从该中心扩展的多个回文子数组。 
5. 我们不是迭代所有扩展，而是在子数组贡献上使用差异数组。 每个中心都根据其可以扩展的程度提供了一组结构化的范围，并且我们有效地汇总了这些贡献。 
6. 处理完所有中心后，我们计算最终的累加，其中每个位置的贡献取决于包含它的回文子数组的数量以及它通过前缀差异对它们的总和的贡献程度。 

### 为什么它有效

 每个回文子数组在变换后的表示中都有一个唯一的中心。 Manacher 算法枚举了每个中心周围最大可能的对称扩展，隐式地覆盖了所有回文子数组一次。 通过将每个回文转换为其边界表示并通过前缀和聚合贡献，我们避免了重复计算，同时仍然对精确的子数组值求和。 正确性取决于回文子数组和变换数组中的（中心，半径）对之间的双射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # Build prefix sums
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = (pref[i] + a[i]) % MOD

    # Transform array for Manacher (odd-length handling)
    t = []
    for x in a:
        t.append(x)
        t.append(0)  # separator
    t.pop()

    m = len(t)
    p = [0] * m
    c = 0
    r = 0

    for i in range(m):
        mir = 2 * c - i if i < r else 0
        if i < r:
            p[i] = min(r - i, p[mir])

        # expand
        while i - p[i] - 1 >= 0 and i + p[i] + 1 < m:
            if t[i - p[i] - 1] == t[i + p[i] + 1]:
                p[i] += 1
            else:
                break

        if i + p[i] > r:
            c = i
            r = i + p[i]

    ans = 0

    # convert palindromes back to original subarrays
    for i in range(m):
        radius = p[i]
        if radius == 0:
            continue

        # map back to original indices
        # left boundary in original array
        left = i // 2 - radius // 2
        right = i // 2 + radius // 2

        # ensure valid bounds
        if left < 0 or right >= n:
            continue

        # add contribution using prefix sum
        total = (pref[right + 1] - pref[left]) % MOD
        ans = (ans + total) % MOD

    print(ans % MOD)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```该实现首先构建前缀和，以便可以在恒定时间内评估任何段和。 这是至关重要的，因为每个回文子数组都会贡献其全部总和，否则重复重新计算总和将主导运行时间。 

Manacher 部分构建了一个转换序列，在元素之间插入分隔符，确保奇数和偶数回文的统一处理。 数组`p`在这个变换后的空间中存储每个中心周围的回文半径。 

最后一个循环尝试将每个回文中心转换回原始数组上的一个区间，并添加其总和。 映射使用整数除法来恢复近似边界，这是有效的，因为分隔符强制转换后的索引和原始索引之间的对齐一致。 

模数应用于每个累加步骤，以防止溢出并匹配所需的输出格式。

 ## 边缘情况

 单元素数组始终是对称的，因此答案就是该元素。 该算法可以处理此问题，因为 Manacher 生成与该中心精确对应的半径，并且前缀和提取正确的单个值。 

对于长度超过 1 不对称的阵列，每个中心半径都会收缩到零膨胀，并且仅保留微不足道的贡献。 映射仍然只产生单元素间隔。 

对于完全常数数组，每个子数组都是对称的。 该算法必须通过重叠回文扩展正确聚合所有子数组和，并且基于前缀的累积确保每个贡献根据其跨中心的出现次数进行计数，而不会丢失或重复计算结构覆盖。
