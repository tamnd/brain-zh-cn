 ## 解决方案
 ## 问题理解

 每个参赛者面临一小组最多 12 个问题，并且有固定的时间。 他们以随机的方式与问题交互：他们不是提前选择顺序，而是在剩余未使用的问题中重复统一选择。 选择一个问题后，他们要么解决它，要么失败，这取决于他们的评分与问题的难度相比是否足够高。 无论哪种方式，都会消耗时间，并且参赛者永远不会重新审视问题。 

因此，随机过程是问题的均匀随机排列，但每个步骤的时间成本取决于问题以及问题是否解决或失败。 一旦时间耗尽，该过程立即停止，只有在时限内完全完成问题才有助于答案。 

对于每个参赛者来说，任务是计算在这个随机过程下解决问题的预期数量，并以一个大素数为模输出。 

约束是关键的结构提示。 问题的数量很少，最多 12 个，这允许子集上的指数状态。 参赛者的数量很大，多达十万，因此任何为每个参赛者重新计算动态程序的解决方案都会立即变得太慢。 时间很小，最多 300，这表明随时间变化的第二个 DP 维度是可行的。 因此，子集状态和时间之间的相互作用必须预先计算一次并为所有参赛者重复使用。 

一个微妙的边缘情况来自“在剩余问题中均匀随机选择”规则。 它意味着完全随机排列，而不是独立选择。 另一个重要的细节是，如果参赛者没有足够的剩余时间来完成所选问题的成本，则即使选择已经发生，该过程也会立即停止，不再进行任何进一步的尝试。 

一个幼稚的错误是独立地处理每个问题并乘以概率。 这会失败，因为顺序不固定并且时间截断将所有事件耦合在一起。 另一个常见的失败是重新计算每个参赛者的 DP，这无法扩展到 100k 查询。 

## 方法

 强力模拟将生成问题的所有排列，并模拟每个排列的过程，并对结果进行平均。 这在概念上很简单：枚举所有 n 个阶乘，模拟每个阶乘的时间演化，并计算平均值。 然而，即使n等于12，排列数量也已经达到4.79亿，并且每次模拟最多处理12步。 这远远超出了可行的限度。 

关键的观察结果是，该过程完全由两件事决定：剩余问题集和剩余时间。 实际顺序仅通过统一随机选择起作用，该选择将过程转换为子集上的马尔可夫链。 这允许对问题子集和时间预算进行动态编程。 

一旦我们接受了这一点，问题就变成了计算函数 dp[S][t]，表示从剩余集合 S 开始还剩 t 分钟的已解决问题的预期数量。 每个转换从 S 中统一选择一个元素，根据参赛者是否可以解决该问题应用适当的成本，然后移动到较小的子集。 

唯一的复杂之处在于，每个参赛者都定义了哪些问题是可以解决的，但这仅取决于比较评分和难度。 这意味着每个参赛者对应一个“好”问题的子集，并且可以为每个这样的子集预先计算 DP。 由于n最多为12，子集数量仅为4096，这使得完全预计算成为可能。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 全排列模拟 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 具有时间维度的子集 DP | O(2^n · 2^n · t · n) 预计算，每个查询 O(1) | O(2^n·t) | O(2^n·t) | 已接受 |

 ## 算法演练

 我们预先计算参赛者可能能够解决的每个可能的问题子集的答案，然后将这些结果重复用于所有参赛者。 

1. 为每个参赛者构建一个位掩码`good_mask`如果参赛者的评分至少是问题 i 的难度，则位 i 为 1。 这将问题分为该参赛者可解决和不可解决的类别。 
2.定义DP表`dp[mask][time]`， 在哪里`mask`表示剩余未使用问题的集合，并且`time`是剩余分钟数。 该值存储该状态中已解决问题的预期数量。 
3. 按大小递增的顺序处理掩码，以便始终已计算出到较小掩码的转换。 这种顺序确保当我们计算一个状态时，所有接下来的状态都可用。 
4. 对于每个州`(mask, time)`，通过迭代每个问题来计算总贡献`i`在`mask`。 每个问题都是按概率选择的`1 / |mask|`。 
5.如果出现问题`i`对于此掩码类型无法求解，它会消耗`tf[i]`如果时间允许，解决问题的时间为零； 否则该过程将停止。 
6.如果出现问题`i`是可解的，它消耗`ts[i]`时间并贡献一个已解决的问题（如果完全完成）； 否则它贡献为零并停止。 
7. 如果减去成本后剩余时间为负，则该分支立即终止并贡献为零。 
8. 否则，过渡到`dp[mask without i][new_time]`，加 1 表示成功解决。 
9. 将累计总和除以`|mask|`使用模逆来解释均匀随机选择。 

### 为什么它有效

 该过程是马尔可夫决策系统，其中下一个状态仅取决于当前剩余集合和剩余时间。 每个有效的执行路径恰好对应于一个选定问题的序列，并且每个这样的序列的概率等于每个步骤的统一选择的乘积。 DP 通过以第一个选择为条件并对所有可能性求和来计算期望，这保留了期望的线性度。 由于每个状态都分解为更小的子集，并且所有转换都严格减小掩码大小，因此不存在循环，并且 DP 是有根据的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, m, t = map(int, input().split())
    d = list(map(int, input().split()))
    ts = list(map(int, input().split()))
    tf = list(map(int, input().split()))
    ratings = list(map(int, input().split()))

    # dp[mask][time]
    size = 1 << n
    inv = [1] * (n + 1)
    for i in range(1, n + 1):
        inv[i] = pow(i, MOD - 2, MOD)

    # precompute dp for every possible "good mask"
    # dp_good[good_mask][mask][time] is too big,
    # instead we compute dp per good_mask

    # store results
    res = {}

    # iterate all good masks
    for good in range(size):
        dp = [[0] * (t + 1) for _ in range(size)]
        # process by increasing mask size
        for mask in range(size):
            for tm in range(t + 1):
                if mask == 0:
                    continue
                total = 0
                cnt = bin(mask).count("1")

                for i in range(n):
                    if not (mask >> i) & 1:
                        continue

                    if good >> i & 1:
                        cost = ts[i]
                        if tm < cost:
                            val = 0
                        else:
                            val = 1 + dp[mask ^ (1 << i)][tm - cost]
                    else:
                        cost = tf[i]
                        if tm < cost:
                            val = 0
                        else:
                            val = dp[mask ^ (1 << i)][tm - cost]

                    total += val

                dp[mask][tm] = total * inv[cnt] % MOD

        res[good] = dp[(1 << n) - 1][t]

    out = []
    for r in ratings:
        good = 0
        for i in range(n):
            if r >= d[i]:
                good |= (1 << i)
        out.append(str(res[good]))

    print(" ".join(out))

if __name__ == "__main__":
    solve()
```DP表`dp[mask][tm]`表示特定剩余组数和剩余时间的预期得分。 掩码和时间上的嵌套循环是有序的，以便当我们计算时`dp[mask][tm]`，所有较小的掩模都已经计算出来。 该转换明确枚举下一个选择的问题，根据当前良好掩模分类下是否可以解决该问题来应用正确的时间成本。 

最后的映射步骤将每个参赛者转换为可解决问题的位掩码，并直接检索预先计算的答案。

 ## 边缘情况

 关键的边缘情况是无法在剩余时间内完成所选问题。 在这种情况下，该过程会立即停止，并且不会做出部分贡献。 例如，如果剩余时间为 5，问题成本为 10，则 DP 分支直接贡献为零，并且不会进一步转换。 

当时间与成本完全匹配时，就会出现另一种微妙的情况。 在这种情况下，问题仍然算作已解决，并且该过程随后终止。 DP 通过在以下情况下允许转换来处理此问题：`tm >= cost`并消耗完全相同的数量，留下零时间，但仍在终止前计算解决方案。 

最后的边缘情况是剩余集合变空时。 无论剩余时间如何，DP 都会正确返回零，因为无法选择进一步的问题并且不可能做出进一步的贡献。
