 ## 解决方案
 ## 问题理解

 我们给定一个正整数$K$。 我们查看数字的所有二进制表示形式$N$，但不是通常的最小形式。 相反，每个数字都精确地使用$K$位，包括前导零。 所以每$N$对应长度固定的二进制字符串$K$。 

一个数字$N$被称为$K$- 二进制重复，如果这样$K$位串可以表示为一些较短的二进制模式的重复。 也就是说，存在一个非空的二进制字符串$S$，比长度短$K$，这样重复$S$多次给出准确的$K$位表示$N$。 

任务是，对于每个查询$K$, 计算有多少个整数$N$在范围内$[0, 2^K - 1]$产生一个$K$- 位表示在这个意义上是周期性的，并输出计数模$10^9 + 7$。 

关键的微妙之处在于，我们不是要求整数意义上的周期性，而是要求包括前导零的固定长度二进制字符串的周期性。 这意味着像这样的字符串`0011`是重复模式的有效候选者，即使整数本身通常会以不同的方式写入。 

这些限制非常重要。 我们最多有$10^5$测试用例和$K$最多$10^6$，所以任何处理每个的解决方案$K$在线性时间内独立是不可能的。 甚至$O(K)$每个查询会导致$10^{11}$在最坏的情况下进行操作，这远远超出了限制。 这会强制进行预计算或封闭形式的组合观察，从而将每个查询减少到$O(1)$。 

一些边缘案例澄清了定义。 

什么时候$K = 1$，只有两个字符串：`0`和`1`。 两者都不能通过重复较短的非空字符串来形成，所以答案是$0$。 

什么时候$K = 2$，字符串是`00`,`01`,`10`,`11`。 这里`00`是重复`0`， 和`11`是重复`1`，所以有 2 个有效数字。 

一个常见的错误是只处理“干净”的整数表示，而忽略前导零。 这会错误地拒绝类似的情况`00`，这对于周期结构至关重要。 

另一个错误是假设任何具有重复字符的字符串都是有效的。 例如`0101`是有效的，但是`0110`不是，即使两者都包含重复位，因为只有第一个具有一致的周期来划分整个长度。 

## 方法

 蛮力方法很简单：对于每个$K$- 位字符串，尝试每个可能的周期长度$d$划分$K$。 对于每个除数$d < K$,检查长度前缀是否$d$重复$K/d$times 重建完整的字符串。 如果任何除数起作用，则该字符串被计数。 

这是有效的，因为周期性字符串正是那些以适当的周期除以整个长度的字符串。 然而，这是灾难性的昂贵。 对于每个$2^K$字符串，我们可以检查$O(K)$除数并执行$O(K)$比较、给予$O(K \cdot 2^K)$，即使对于非常小的情况也是不可行的$K$，更不用说达到$10^6$。 

关键的观察是我们不是使用固定的字符串，而是计算长度的字符串$K$是周期性的。 这将问题从每个字符串检查转变为组合计数问题。 

长度为二进制的字符串$K$完全由其最小周期决定$p$，一旦我们修复$p$，字符串是通过选择第一个来确定的$p$位，然后重复它们。 但是，由于带句点的字符串，会发生过度计数$p$也计算所有倍数$p$。 这正是除数晶格结构$K$。 

因此，我们不是直接计算周期字符串，而是计算所有字符串并减去那些“原始”字符串，这意味着它们没有更小的周期。 所有字符串的数量是$2^K$。 原始字符串的数量可以使用除数的莫比乌斯求逆来计算$K$。 一旦我们知道了所有除数的原始计数，我们就可以有效地导出周期计数。 

这导致了除数上的标准数论 DP：我们预先计算$2^K$然后使用包含-排除减去较小除数的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(2^K \cdot K)$|$O(1)$| 太慢了|
 | 除数 + 莫比乌斯 / DP |$O(K \log K)$预计算，$O(1)$每个查询 |$O(K)$| 已接受 |

 ## 算法演练

 1. 预先计算最大的 2 的幂$K$在所有查询中，因为$2^K$表示所有长度的二进制字符串$K$。 这给出了每个案例的基数。 
2. 对于每个值$K$，在过滤掉原始字符串之前，初始化所有字符串都是有效候选字符串的想法。 重复的结构仅取决于除数$K$，所以我们关注除数关系。 
3. 迭代所有可能的除数$d$每个的$K$按升序排列，治疗$d$作为潜在的经期长度。 带句点的字符串$d$由重复大小的块形成$d$，所以有$2^d$在考虑过度计数之前这样的块。 
4. 减去已经代表更基本模式的较小除数的贡献。 此步骤确保每个周期结构在其最小周期处精确计数一次。 减法遵循除数格上的包含-排除。 
5. 计算出所有除数的原始字符串的数量后，导出周期字符串的数量，即总数减去全长的原始字符串。 这准确地隔离了那些具有适当重复结构的字符串。 
6. 回答每个问题$O(1)$使用预先计算的数组。 

### 为什么它有效

 每个$K$-bit 字符串具有唯一的最小周期，这是通过重复生成它的最小前缀。 这导致所有字符串按其最小周期进行划分。 暴力计数重叠，因为周期最小的字符串$d$也与每个倍数兼容$d$，但只能归因一次。 莫比乌斯反演通过系统地取消较小除数的贡献来纠正这种多算现象，确保每个字符串都准确地分配给一个最小周期类。 这保证了最终基于减法的计数的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

MAX_K = 10**6

# precompute powers of 2
pow2 = [1] * (MAX_K + 1)
for i in range(1, MAX_K + 1):
    pow2[i] = (pow2[i - 1] * 2) % MOD

# mobius function via linear sieve
mu = [1] * (MAX_K + 1)
is_prime = [True] * (MAX_K + 1)
primes = []

for i in range(2, MAX_K + 1):
    if is_prime[i]:
        primes.append(i)
        mu[i] = -1
    for p in primes:
        if i * p > MAX_K:
            break
        is_prime[i * p] = False
        if i % p == 0:
            mu[i * p] = 0
            break
        else:
            mu[i * p] = -mu[i]

# precompute periodic counts using divisor sums
periodic = [0] * (MAX_K + 1)

for i in range(1, MAX_K + 1):
    # strings with period dividing i
    total = 0
    j = 1
    while j * j <= i:
        if i % j == 0:
            d1 = j
            d2 = i // j
            total = (total + mu[i // d1] * pow2[d1]) % MOD
            if d1 != d2:
                total = (total + mu[i // d2] * pow2[d2]) % MOD
        j += 1
    periodic[i] = (pow2[i] - total) % MOD

t = int(input())
for _ in range(t):
    k = int(input())
    print(periodic[k] % MOD)
```该代码将问题分为两个概念层。 首先，它预先计算 2 的幂，因为每个二进制字符串都对应于自由选择的位。 其次，它使用基于莫比乌斯的除数包含排除来隔离原始字符串，这些字符串正是不是通过重复较小的块形成的。 最终的答案是从总空间中减去原始计数。 

一个常见的实现陷阱是忘记前导零是字符串空间的一部分，这就是为什么所有计数都结束的原因$2^K$，而不是简化表示中的整数。 

另一个微妙之处是有效处理除数枚举。 预计算莫比乌斯避免了每个查询重新计算除数结构，否则会导致超时$10^5$查询。

 ## 边缘情况

 对于$K = 1$，算法正确返回 0，因为没有合适的除数来形成重复，因此包含-排除不会留下周期性结构。 

对于总理$K$，仅存在重复长度为 1 模式或全长模式的字符串，并且减法正确地删除了所有仅全长结构，仅留下常量字符串。 

对于 2 的幂，除数格是密集的，并且莫比乌斯消除可确保像周期 4 内的周期 2 这样的嵌套重复不会被重复计算，从而保持所有分层重复的正确性。
