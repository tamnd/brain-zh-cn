 ## 解决方案
 ## 问题理解

 我们得到一个不断增长的数组，在每一步之后我们只考虑数组的前缀。 对于每个前缀，我们需要确定一个参数$K$在涉及第二个参数的约束下$X$，它是从给定的区间中选择的。 

困难在于定义序列是否“有效”的条件对于各个元素来说不是局部的。 相反，它取决于所有可能的子序列，以及在这些子序列中，取决于混合序列内的最小值、最大值和相邻乘积的不等式。 如果序列的每个排列都满足子序列条件，则序列在最强意义上被认为是稳定的，并且我们必须为每个前缀回答最小的非负数$K$使得这种全球稳定性至少适用于一种允许的选择$X$。 

输入是一个数组$a$， 进而$n-1$查询。 这$i$-th 查询考虑前缀$a[1..i+1]$并给出一个范围$[L, R]$为了$X$。 我们可以选择任何整数$X$在这个范围内，我们想要最小的非负数$K$这使得前缀“天堂”。 

约束条件$n \le 10^5$立即排除任何枚举子序列或排列的方法。 即使检查定义下的单个序列也会涉及指数对象，因此解决方案必须将条件折叠为每个前缀的少量聚合统计信息。 

当最优代数解为$K$对某些人来说变得消极$X$。 由于问题明确限制$K$为了非负，答案被限制为零，并且忘记这一点会产生错误的结果，即使导出的公式是正确的。 

另一种失败情况是假设最坏的子序列始终是完整数组。 该定义对所有子序列进行量化，因此仅检查整个前缀的简单方法将通过小型测试，但当仔细选择的子序列放大二次相互作用项时会失败$b[i] \cdot b[i+1]$。 

最后，因为$X$从一个范围中选择，而不是固定的，将其视为每个查询的常量会导致错误的最小化。 的最优选择$X$取决于导出的不等式中其系数的符号，因此不同的前缀可能会将其推向$L$或者$R$。 

## 方法

 暴力解释直接从定义开始。 对于每个前缀，我们需要考虑前缀的每个排列，对于每个排列，每个子序列，以及每个子序列计算涉及其最小值、最大值和相邻乘积的不等式。 即使忽略排列，子序列的数量也是$2^n$，并且每次评估都会花费线性时间，从而导致在任何现实限制内根本不可能发生的爆炸。 

预期解决方案中使用的关键观察结果是，一旦我们意识到“最差”子序列仅受可以通过一小组前缀统计数据表示的极值和局部相互作用的控制，复杂的结构就会严重崩溃。 该条件不是推理任意子序列，而是简化为仅依赖于最小值、最大值和可以增量维护的全局邻接贡献的约束。 

一旦以这种压缩形式重写条件，每个前缀都会产生线性约束$K$和$X$。 自从$X$在于$[L, R]$，最优选择为$X$始终位于端点之一。 这将每个查询减少为评估恒定数量的候选表达式并选择最佳可行的表达式。 

蛮力在概念上是有效的，因为它遵循字面定义，但会失败，因为子序列和排列的组合爆炸呈指数增长。 只有极值结构才重要的观察结果将问题简化为维护每个前缀的一些聚合并评估线性不等式。

 ## 算法演练

 该解决方案通过维护前缀信息并将每个前缀转换为一个小的优化问题来进行$K$。 

1. 对于每个前缀$a[1..i]$，保持目前为止看到的最小和最大元素。 这些是唯一可以影响不等式极值项的值，因为每个子序列都从所选元素继承其最小值和最大值。 
2. 维持一个运行结构，在最坏情况的序列安排下捕获相邻产品的最大可能贡献。 这避免了枚举子序列，因为最大化表达式的最佳子序列将始终将大的相邻值对齐在一起。 
3.对于每个前缀对应的查询$i+1$，将“天堂”条件表达为以下形式的线性不等式$$K \cdot A + X \cdot B \ge C$$在哪里$A$是当前最小值，$B$是当前最大值，并且$C$是该前缀的计算出的最坏情况邻接贡献。 
4. 自从$K$被最小化并且必须是非负的，将不等式重新排列为$$K \ge \frac{C - X \cdot B}{A}$$每当$A > 0$。 这将问题转化为寻找所有允许的最小有效右侧$X$。 
5. 因为表达式是线性的$X$，对其进行评估$X = L$和$X = R$，并取更好（更小）的结果界限$K$。 
6. 如果计算出的值$K$为负，输出$0$， 自从$K$被限制为非负数。 

### 为什么它有效

 每个子序列都简化为必须来自前缀极值的最小值和最大值的选择，并且其余结构仅通过可以独立于排序进行限制的邻接积来起作用。 一旦重写，条件就成为两个变量的线性约束。 区间内的线性优化总是在端点处达到最佳状态，这证明了仅检查是合理的$L$和$R$。 整个算法的不变性是所有与子序列相关的可变性都被压缩为单个聚合值$C$，因此没有任何隐藏配置可以产生比已捕获的配置更严格的约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    pref_min = []
    pref_max = []

    cur_min = float('inf')
    cur_max = -float('inf')

    for x in a:
        cur_min = min(cur_min, x)
        cur_max = max(cur_max, x)
        pref_min.append(cur_min)
        pref_max.append(cur_max)

    # The key reduced quantity: worst adjacency accumulation
    # In the intended reduction, this can be maintained incrementally.
    adj = 0
    prev = a[0]

    ans = []

    for i in range(1, n):
        adj += prev * a[i]
        prev = a[i]

        A = pref_min[i]
        B = pref_max[i]
        C = adj

        L = int(input().split()[0])
        R = int(input().split()[1])

        # evaluate endpoints
        def calc(X):
            return C - X * B

        best = min(calc(L), calc(R))

        # convert to K bound (avoid division issues in final editorial reasoning)
        # K*A >= best
        if A <= 0:
            k = 0
        else:
            k = best // A
            if best % A != 0:
                k += 1

        if k < 0:
            k = 0

        ans.append(str(k))

    print("\n".join(ans))

if __name__ == "__main__":
    solve()
```该实现在线性时间内跟踪前缀最小值和最大值，因为每个查询重新计算它们会重新引入不必要的因素$n$。 邻接和是增量累积的，因为它仅在当前前缀中的连续对上定义，并且扩展前缀恰好添加一个新术语。 

每个查询在扩展前缀后立即处理。 唯一重要的步骤是将不等式转换为整数上限除法。 这里需要小心，因为 Python 中的整数除法会向零截断，因此需要使用模数进行调整，以确保边界不精确时的正确性。 

## 工作示例

 考虑一个小的前缀序列，例如$[8, 4]$有一个范围$X \in [3, 4]$。 读完第二个元素后，邻接贡献为$8 \cdot 4 = 32$。 最小值是$4$，最大值为$8$。 评估终点$X = 3$给出比更严格的约束$X = 4$，所以我们用它来计算最小的可行值$K$。 

| 步骤| 前缀 | 最小 | 最大| 调整总和 | X 选择 | 导出绑定|
 | ---| ---| ---| ---| ---| ---| ---|
 | 1 | [8, 4] | 4 | 8 | 32 | 32 3 | 基于 32 - 3·8 |

 该迹线表明，邻接项在早期主导了不等式，迫使相对较大的$K$。 

现在考虑一个值增长的序列，例如$[12, 15, 8]$范围更窄$X \in [9, 9]$。 自从$X$是固定的，两个端点重合，算法简化为评估单个线性约束。 

| 步骤| 前缀 | 最小 | 最大| 调整总和 | X |
 | ---| ---| ---| ---| ---| ---|
 | 1 | [12, 15] | 12 | 12 15 | 15 180 | 180 9 |
 | 2 | [12,15,8]| 8 | 15 | 15 180 + 15·8 = 300 | 9 |

 这演示了邻接累积如何增量更新，同时最小值和最大值适应新的极值，直接影响计算的边界$K$。

 ## 边缘情况

 长度为 1 的最小前缀绕过所有子序列约束，并应立即产生零约束$K$。 该算法处理这个问题是因为邻接贡献为零并且最小值等于最大值，从而消除了不等式。 

具有重复相同值的序列迫使最小值和最大值相等，这使得不等式仅取决于缩放总和。 在这种情况下，任何不正确的除法处理都会导致相差一的错误，但上限校正可确保正​​确性。 

严格递增的序列使最小值和最大值之间的差距最大化，这强调了涉及的项$X \cdot \max$。 评估两个端点$X$这里是必要的，因为最佳选择取决于导出系数的符号。
