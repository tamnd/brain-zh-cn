 ## 解决方案
 ## 问题理解

 我们得到一个由小单位三角形构建的条形区域。 形状随参数增长$n$，其总大小与$n$。 任务是计算有多少种方法可以使用相同的“T 形”或“多米诺骨牌”块完全覆盖该区域，每个块覆盖固定的少量三角形，并允许旋转。 

重要的结构特性是该区域不是任意的：它通过每次附加一个恒定宽度的“层”来增长$n$增加一。 这使得这个问题从根本上来说是连续的而不是全局的。 每一块瓷砖的尺寸$n$可以看作是扩展尺寸的平铺$n-1$，但扩展不是唯一的，因为最后两层之间的边界可以用不止一种一致的方式覆盖。 

输入由多个测试用例组成，每个测试用例给出一个整数$n$最多$10^9$。 我们必须输出每个的有效平铺数量$n$。 结果适合 64 位有符号整数。 

大的约束立即排除任何指数甚至线性的$n$每个测试用例的 DP。 甚至$O(n)$每个查询都是不可能的。 自从$t$可能很大，我们需要一个封闭的形式或一个恒定时间的递归评估。 

小尺寸时会出现微妙的边缘情况$n$。 为了$n = 1$，该区域是最小的并且通常只允许一个平铺。 为了$n = 2$，多个本地配置已经出现。 任何不正确的解决方案通常都会在这里失败，因为它隐含地假设重复开始得太早。 

另一个常见的失败是假定条带各段之间是独立的。 简单的分解将独立地处理每个“列”，但三角形几何形状通过共享边界边缘在相邻列之间创建依赖关系。 

## 方法

 蛮力的想法很简单：为固定的区域构建区域$n$，枚举所有的图块放置位置，并使用 DFS 或回溯来尝试每个覆盖。 这是正确的，因为每个有效的平铺都是显式生成的。 

然而，递归中的每一步都必须考虑当前未覆盖边界处的多个放置，并且状态数量随着$n$。 即使在边界轮廓上进行记忆，状态的数量也取决于带的宽度，该宽度保持不变，但会导致转换图的大小在深度上仍然呈指数关系。 即使对于$n \approx 40$。 

关键的观察结果是，部分填充区域的边界仅具有恒定数量的有意义的配置。 当将条带延伸一层时，只能出现一小组边界态，并且它们之间的过渡是固定的。 这将问题简化为列上的有限状态自动机。 

一旦状态图被识别，系统就会崩溃为线性递归。 在这个特定问题中，状态数量减少为单个有效递归：$$f(n) = f(n-1) + f(n-2)$$根据前两层的几何形状仔细确定基础值。 

发生这种情况是因为每个有效的长度平铺$n$可以根据最后一列的填充方式进行分类：它要么唯一地扩展先前的配置，要么以完全相同的一种替代方式与前一列配对。 不存在其他结构选择。 

因此，问题简化为计算大范围的斐波那契式序列$n$，这对于预计算或快速加倍来说是微不足道的。

 ## 算法演练

 1.观察答案仅取决于$n$，因此我们的目标是预先计算或导出递归，而不是模拟几何。 
2. 直接从小配置计算基值。 该地区为$n=1$恰好有一个平铺，并且$n=2$有两块瓷砖。 这些来自明确枚举如何覆盖第一扩展层。 
3. 确定尺寸的有效平铺方式$n$可以结束。 当我们查看最右边的扩展时，它要么形成尺寸平铺的延续$n-1$，或者它与前一层配对，恰好删除一个额外的自由度，对应于大小的平铺$n-2$。 
4. 将这种结构分裂转化为递归：$$f(n) = f(n-1) + f(n-2)$$这不是一个假设，而是一个分类：每个平铺都唯一地属于这两种扩展类型之一。 
5. 预先计算斐波那契值直至最大值$n$所有测试用例都需要，或者使用对数时间快速加倍来计算每个查询。 
6. 输出$f(n)$对于每个查询。 

### 为什么它有效

 正确性来自于三角形条带的已处理部分和未处理部分之间的边界具有恒定的复杂性这一事实。 任何平铺都只能通过该边界与下一层交互，并且只有两个有效的边界延续可以保留可平铺性。 这强制执行确定性的两项递归，其中每个大小的平铺$n$从较小的有效平铺中仅生成一次。 

任何配置都无法“跳过”状态或创建额外的独立分支，因为在不立即修复相邻放置的情况下，图块无法延伸到一层之外。 局部刚性是将问题归结为斐波那契结构的原因。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def fib(n):
    # fast doubling: returns (F(n), F(n+1))
    if n == 0:
        return (0, 1)
    a, b = fib(n >> 1)
    c = a * (2*b - a)
    d = a*a + b*b
    if n & 1:
        return (d, c + d)
    return (c, d)

t = int(input())
for _ in range(t):
    n = int(input())
    # based on geometry: f(1)=1, f(2)=2 => shifted Fibonacci
    if n == 1:
        print(1)
        continue
    fn, _ = fib(n)
    print(fn + 1)
```该代码使用快速加倍来在每个查询的对数时间内计算斐波那契数。 递归本身嵌入在条带的结构中，唯一的微妙之处是对齐索引，以便几何基本情况与数学斐波那契定义相匹配。 

一个常见的错误是两者之间的对齐不一致$f(1), f(2)$和标准斐波那契$F(0), F(1)$。 必须根据手动验证的小型案例来检查班次。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 对于$n=1$，算法直接返回基值而不调用递归。 这可以防止对斐波那契索引的无效访问，并确保移位的定义不会在零处中断。 

为了$n=2$，两个递归分支都存在，因此计算不得将它们折叠为单个状态。 快速加倍方法可以干净地处理这个问题，因为它本质上计算了两者$F(n)$和$F(n+1)$，即使在很小的边界上也能保持正确性。
