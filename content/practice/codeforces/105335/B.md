 ## 解决方案
 ## 问题理解

 输入是从 2 到 9 的数字序列，来自旧的多次点击电话键盘。 每个数字对应一组字母，连续多次按下该数字即可产生一个字母。 例如，按 2 一次给出一个字母，按 2 两次给出组中的下一个字母，依此类推。 

记录数据中关键的缺失部分是字母之间的停顿。 当输入一个单词时，说话者会在不同的字母之间暂停，但录制的字符串会将所有内容合并成一个连续的按键流。 任务是恢复哪些字母序列可以产生观察到的按键流。 

键盘的行为类似于标准电话映射：2 映射到 abc，3 映射到 def，4 映射到 ghi，5 映射到 jkl，6 映射到 mno，7 映射到 pqrs，8 映射到 tuv，9 映射到 wxyz。 重复按下相同的数字会循环显示其字母，但这个问题保证了打字员永远不会超过按键上的字母数量，因此不会发生回绕。 

输出是任何有效的解码字符串，但在所有有效的解释中，我们必须选择最短的解释，如果仍然存在歧义，则选择字典顺序最小的解释。 

根据约束条件，输入长度最多为 200，这意味着任何达到 O(n) 或 O(n log n) 的解决方案都是微不足道的。 即使 O(n²) 也可能会通过，但该结构强烈建议线性扫描。 

天真的解释可能会尝试任意插入暂停并测试字符串的所有分段。 这立即变成指数，因为每对数字之间都有一个二元决定：是否中断。 对于长度 200，这已经是 2199 种可能性，这是不可行的。 

一个更微妙的错误是假设单个数字总是对应于单个字母。 这在重复数字等情况下会失败，因为需要多次按下才能到达映射中后面的字母。 

当数字重复多次时就会出现边缘情况。 例如，像 7777 这样的段必须使用 7 的完整映射来映射到 4 个字母的单词。另一种情况是像 2323 这样的交替数字，其中每个数字的变化都会强制产生一个新的字母边界。 

## 方法

 暴力破解的想法是考虑所有可能的方法将数字字符串分割成多个片段，其中每个片段代表单个键的重复按下。 对于每个分段，我们会根据其长度和相应的键盘映射将每个分段转换为字母。 这是有效的，因为它直接模拟带有显式暂停的打字过程。 

故障点是组合爆炸。 对于长度为 n 的字符串，有 n−1 个间隙，每个间隙可以是分割，也可以不是分割。 这导致 2^(n−1) 分割。 即使对于 n = 50，这也已经太大了，并且约束最多允许 200。 

关键的观察是，一旦我们强制执行打字模型，停顿在有效的解释中就不是任意的。 如果相同的数字连续重复，则这些按键必须属于同一个字母。 暂停只能在数字改变时发生，因为否则它会代表在仍然按下同一键时重置，这将与连续按下的解释相矛盾。 这会强制进行独特的分段：相同数字的最大连续块。 

一旦字符串被划分为多个串，每个串都会通过将其长度解释为按键次数来独立地映射到一个字母。 

因此，最短的可能输出总是通过这种最大运行解释来实现，因为引入额外的分割只会增加字母的数量。 字典顺序最小也是固定的，因为没有其他分段可以保持有效性。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分割 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 游程解码| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们一次性处理该字符串，将连续的相同数字分组为块。 

1. 从左到右扫描字符串，同时保留当前数字及其运行长度。 这会将最大连续相同数字分组在一起，因为数字的变化是新字母的唯一有效边界。 
2. 对于长度为 k 的数字 d 的每个完整运行，通过使用位置 k 索引到 d 的映射，将其转换为字母。 这直接模拟按键k次。 
3. 将生成的字母附加到输出字符串。 
4. 继续处理整个字符串，确保最终的运行也已转换。 

正确性来自于这样的事实：每次运行对应于同一按钮上连续不间断的按键序列。 由于该问题保证不会超出密钥字母计数的溢出，因此每个游程长度始终映射到有效字母而不会产生歧义。 

### 为什么它有效

 基本的不变量是，在扫描中的每一点，当前运行恰好代表原始打字过程中的一个字母。 任何在运行中进行拆分的尝试都意味着在按下同一键的同时插入暂停，这不能产生不同的字母，只会增加输出中的字符数。 因此，最大游程分解产生最小长度有效解码。 由于每次运行都确定性地映射到单个字母，因此没有替代解码可以产生按字典顺序更小或更短的字符串。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

KEYS = {
    '2': "abc",
    '3': "def",
    '4': "ghi",
    '5': "jkl",
    '6': "mno",
    '7': "pqrs",
    '8': "tuv",
    '9': "wxyz"
}

def solve():
    s = input().strip()
    n = len(s)
    
    res = []
    i = 0
    
    while i < n:
        j = i
        while j < n and s[j] == s[i]:
            j += 1
        
        digit = s[i]
        length = j - i
        
        res.append(KEYS[digit][length - 1])
        
        i = j
    
    print("".join(res))

if __name__ == "__main__":
    solve()
```该实现直接反映了运行分解。 内循环扩展相同数字的每个最大块。 表达式`length - 1`至关重要，因为键盘字符串的索引是从零开始的，而按下次数是从一开始的。 

一个常见的错误是尝试累积计数，但在数字发生变化时没有正确重置。 基于指针的方法通过在开始下一个段之前显式关闭每个段来避免这种情况。

 ## 边缘情况

 关键的边缘情况是数字串与其键盘组的长度完全匹配。 对于映射到四个字母的数字 7，像 7777 这样的输入必须映射到最后一个字母 's'。 该算法自然地处理这个问题，因为游程长度直接用作映射的索引。 

另一个边缘情况是单位数输入。 由于只有一次运行，算法立即返回该键映射的第一个字母，并且不会触发边界逻辑。 

交替的数字（例如 232323）形成多个单长度序列。 每次运行都是独立处理的，确保数字更改不会发生意外合并。
