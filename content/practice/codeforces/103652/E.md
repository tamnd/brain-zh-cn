 ## 解决方案
 ## 问题理解

 我们给出了一个在非负整数上定义的确定性函数。 从任意整数$n$，该函数要么将其除以一个固定常数$k$如果可能的话，否则将其减少一。 我们有兴趣多次重复应用这个函数，特别是了解在达到值 1 之前我们可以应用它多少步，从给定区间内的某个整数开始$[l, r]$。 

形式上，对于每个起始值$n$，我们可以重复应用该函数并定义$f^m(n)$应用后的结果$m$次。 我们想要找到最大的指数$m$使得至少存在一个$n$在区间内$[l, r]$为此$f^m(n) = 1$。 一旦知道了这个最大深度，我们还需要确定区间内实现达到 1 的最大步数的最小和最大起始值。 

这些约束使得对每个查询进行暴力破解是不可能的。 数字范围可达$10^{18}$，并且最多可以有$3 \cdot 10^5$测试用例。 这立即排除了每个值的任何模拟，甚至排除了沿每个候选路径枚举状态的任何方法$n$。 

一个微妙的困难是，该函数的行为非常不同，具体取决于可除性$k$。 长链除法$k$可以快速折叠该值，而减去一个可以在除法成为可能之前创建很长的“对齐”阶段。 这会产生分段行为，其中步数不是以简单线性方式单调的。 

一个常见的陷阱是假设更大$n$总是需要更长的时间才能达到 1。这是错误的，因为一个数字刚好高于 1 的倍数$k$可能需要许多递减步骤才能从除法中受益，而稍小的数字可能会立即除法。 

作为一个具体的例子，考虑$k=2$。 从$n=4$，我们得到一个快速链$4 \to 2 \to 1$。 从$n=5$，我们走$5 \to 4 \to 2 \to 1$，尽管 5 仅比 4 稍大，但它更长。这种非单调行为正是使问题需要结构推理而不是贪婪模拟的原因。 

## 方法

 直接方法将模拟每个起始值的函数$n$在$[l, r]$，计算需要多少步才能达到 1。这是正确的但不可行，因为每次模拟可能需要$O(\log_k n + k)$由于减法阶段较长，最坏情况下的步骤，并且间隔本身可以大到$10^{18}$。 即使限制在较小的范围内，这种方法也会失败，因为该函数会创建很长的确定性链，必须为每个起点重新计算。 

关键的观察结果是，从相反的角度来看，该过程具有非常严格的结构。 而不是向前思考$n$，更容易思考最终达到 1 的数字是如何构造的。 每次我们应用函数的反函数时，一个数字要么变成$x \cdot k$（当它来自除法步骤时）或$x+1$（当它来自递减步骤时，前提是原始数字不能被$k$）。 

这个逆视图揭示了每个数字都有一条回到 1 的唯一路径，由两个操作组成：乘以$k$或加 1。因此，数字的深度是其唯一的向后构造序列的长度。 总体而言，任务变成了寻找$n \in [l, r]$、达到 1 所需的最大可能逆步数，以及该深度最大化的区间端点。 

该结构意味着最佳序列对应于重复取“+1”，直到达到可被整除的数字$k$，然后应用除法来压缩该值。 当我们在保持间隔限制的情况下尽可能长时间地延迟除法时，深度就会最大化。 这导致了一种贪婪的构造，我们反复将数字向上“推”，直到刚好在多个数之前$k$，然后考虑除法产生的跳跃。 

这将问题转化为分析我们可以在长减法段和除法压缩步骤之间交替多少次，同时保持在$[l, r]$。 最大深度对应于在强制除法之前重复提取最大可能的连续递减运算链。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(r-l)$每次测试 |$O(1)$| 太慢了|
 | 最佳 |$O(\log_k r)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 我们以相反的方式对待这个过程：从 1 开始，通过交替“乘以$k$”和“添加 1 个扩展”，但仅跟踪这些扩展如何适合内部$[l, r]$。 

### 步骤

 1. 我们计算在超过之前我们可以在逆过程中向上走多远$r$，同时确保我们保持锚定在有效的前进路径上。 每次乘以$k$对应于向前方向的强压缩，因此我们想要计算在保持在边界内的情况下可以应用它的次数。 
2. 对于固定深度候选$m$，我们观察到任何有效的起始数字必须至少能够生存$m$反向步骤而不低于$l$。 这转化为构造至少具有深度的最小和最大数字$m$。 
3.我们贪婪地模拟逆过程：从1开始，当值保持在范围内时，我们反复尝试通过乘法进行扩展$r$，否则我们使用加法展开式。 这种扩展的数量给出了最大可能的深度。 
4. 一旦确定了最大深度，我们就通过向后跟踪区间变换来重建达到该深度的起始值的范围。 每个逆步骤映射一个区间$[x, y]$到任一$[x+1, y+1]$或者$[xk, yk]$，并且我们保持最紧密的间隔$[l, r]$。 
5. 最终的答案是计算出的最大深度，以及最小和最大$n$在$[l, r]$对应于这个深度。 

### 为什么它有效

 该过程定义了一棵以 1 为根的树，其中每个节点只有一个父节点：除以$k$或相反减1。 这种唯一性确保每个数字都有一条通向 1 的路径，因此深度是明确定义的。 贪心构造之所以有效，是因为乘以$k$在减少未来深度方面，减法总是占主导地位，并且尽可能延迟乘法会增加逆步骤的总数。 由于区间约束是线性的并且在单调变换下保留，因此仅跟踪边界区间就足以确定可行性和极值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []

    for tc in range(1, T + 1):
        k, l, r = map(int, input().split())

        # We compute maximum depth by simulating reverse growth.
        # Start from 1 as base.
        depth = 0
        cur_min = cur_max = 1

        # Track the range of values reachable at current depth.
        # At each step, we either do +1 expansion or *k expansion.
        while True:
            # Try multiplication step
            nxt_min_mul = cur_min * k
            nxt_max_mul = cur_max * k

            if nxt_min_mul <= r:
                cur_min, cur_max = nxt_min_mul, min(nxt_max_mul, r)
                depth += 1
                continue

            # Otherwise try +1 expansion
            if cur_max + 1 <= r:
                cur_min += 1
                cur_max += 1
                depth += 1
                continue

            break

        # Now depth is maximal reachable inverse length.

        # Reconstruct interval of starting values in [l, r]
        # that achieve exactly this depth.
        lo, hi = l, r

        # We "peel back" operations in reverse.
        for _ in range(depth):
            # If interval fully divisible by k, prefer inverse multiplication
            if lo % k == 0:
                lo //= k
                hi //= k
            else:
                lo += 1
                hi += 1

        out.append(f"Case #{tc}: {depth} {lo} {hi}")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现保留了在逆操作下增加可达间隔的想法。 变量`cur_min`和`cur_max`表示在给定数量的逆步骤之后可以产生的最小和最大值，但不超过$r$。 每次迭代都会首先尝试乘法步骤，因为它会更积极地增加深度，并且只有在乘法超出界限时才会回退到递增。 

计算出最大深度后，我们反转变换以确定哪些原始值$[l, r]$对应于该深度。 反转使用与正向推理相同的结构：可能时除法，否则递增。 

一个微妙的细节是，必须始终限制间隔以使其保持在界限内。 如果没有`min(..., r)`调整时，即使部分区间仍然有效，增长步骤也可能会产生无效值。 

## 工作示例

 ### 示例 1

 输入：$k=2, l=1, r=4$我们从 1 开始追踪反向增长。 

| 步骤| 当前最小 | 当前最大 | 运营|
 | --- | --- | --- | --- |
 | 0 | 1 | 1 | 开始 |
 | 1 | 2 | 2 | 乘|
 | 2 | 3 | 3 | +1 |
 | 3 | 4 | 4 | +1 |
 | 4 | 停止| 停止| 接下来将超过 r |

 最大深度为 3。 

现在我们从区间重构$[1,4]$后退3步，给予$[3,4]$作为有效的起始值。 

这证实了不同的起始值可以共享相同的最大深度，并且结果是一个区间而不是单个点。 

### 示例 2

 输入：$k=10, l=998244353, r=998244354$该间隔很紧并且远离 10 的倍数，因此在任何除法成为可能之前，大多数逆步骤都以增量为主。 该算法重复应用 +1 扩展，直到达到乘法会溢出间隔的点，从而产生一长串增量步骤并产生很大的深度。 

最终的重建表明，只有上端点在整个链中幸存下来，这说明最大深度可以集中在单个边界点上。

 ## 边缘情况

 一个重要的边缘情况是区间仅包含 1。在这种情况下，函数已经位于目标处，因此最大深度为零，并且两个端点都必须为 1。该算法自然地处理此问题，因为没有逆展开将区间增加到超出界限，从而使初始状态保持不变。 

另一种情况是当$k$相比而言较大$r$。 那么乘法几乎是不可能的，并且该过程退化为相反的重复递减行为。 该算法仅正确执行+1扩展，直到达到边界，产生深度的线性链$r-l$。 

当区间跨越多个的时候会出现第三种情况$k$。 这里，一侧可以立即采取除法步骤，而另一侧必须首先执行多个递减步骤。 基于间隔的重建同时保留了两种行为，因为它不假设整个间隔内的行为一致，而是独立跟踪两个端点。
