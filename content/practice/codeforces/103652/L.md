 ## 解决方案
 ## 问题理解

 我们得到了一个水平排列的离散“金字塔”点阵。 第 1 级有一个顶点，第 2 级有两个顶点，第 3 级有 3 个顶点，依此类推，形成三角形排列。 在几何上，这些顶点对应于三角表面，其中可以由与金字塔结构对齐的某些顶点三元组形成小等边三角形。 

每个有效三角形都是通过选择在此几何嵌入中距离相等的成对的三个顶点来形成的。 由于规则的三角形网格，这样的三元组与嵌入晶格中的等边三角形完全对应，并且它们可以以多种方向出现：直立、倒置或跨水平倾斜。 

任务是：对于从 l 到 r 级别的每个查询区间，计算有多少个顶点三元组完全位于这些级别内并形成一个等边三角形。 

输入大小非常大，查询次数高达 3 × 10^5，级别索引高达 10^5。 这会立即排除任何每个查询的线性或二次枚举级别。 即使每次查询扫描 O(n)，在最坏的情况下也会达到 3 × 10^10 次操作，这是不可行的。 因此，预期的解决方案必须预先计算每个查询或将每个查询减少到 O(1)。 

简单的几何枚举方法在概念上也是失败的：即使我们理解精确的嵌入，通过扫描 [l, r] 范围内的所有顶点三元组来计算三角形的结构大小将是 O(n^3)，这是完全不可能的。 

主要的边缘情况来自小区间和边界截断。 例如，当 l = r 时，根本不存在三元组，因此答案为 0。当 r = l + 1 时，仍然不存在三角形，因为至少需要三个不同的水平才能形成任何等边配置。 仅根据级别组合进行计数而不在 [l, r] 中强制执行完全包含的粗心方法会过度计算“接触”间隔但延伸到其之外的三角形。 

另一个微妙的情况是当 l = 1 并且 r 很小时。 任何假定稳定的无限金字塔图案的公式都可能在顶部附近失败，因为存在的几何配置较少。 

## 方法

 强力解释将尝试枚举所选级别中的所有顶点三元组，并检查每个三元组是否形成等边三角形。 即使我们进行了有效的距离测试，级别 [l, r] 中的顶点数量也约为 (r − l + 1)^2 / 2，并且检查其中的所有三元组会导致三次增长。 当 r = 10^5 时立即爆炸。 

通过按方向对三角形进行分类，更结构化的蛮力会略有改善。 在这个格子中，每个等边三角形都由其边长和位置决定。 如果我们固定边长 k，则足够大区域中的有效放置数量会随着该区域中的位置数量线性增长。 然而，这仍然需要迭代 k 和所有可能的位置，至少给出 r 中的二次行为。 

关键的观察是，我们实际上不需要推理单个三角形或几何布局。 重要的是，金字塔结构是高度规则的，并且直到n层的顶点形成的等边三角形的数量在n上具有闭式三次多项式。 这是三角晶格中的标准现象：固定形状子配置的组合计数变成域大小的多项式。 

一旦我们接受第 1 层到第 n 层的三角形总数是某个函数 F(n)，查询 [l, r] 就简化为包含-排除：

 F(r) - F(l - 1)。

整个任务变成寻找 F(n) 的正确封闭形式。 从第一原理推导出来的涉及按方向和边长对三角形进行分类。 每个都贡献多项式数量的放置，并且对所有允许的大小求和折叠成三次表达式。 

简化后，F(n) 变成一个简单的三次多项式，因此每次查询都是 O(1)。

 | 暴力枚举| 每个查询 O(n^3) | O(1) | O(1) | 太慢了 |

 ## 算法演练

 我们继续将问题转化为在级别上评估前缀函数。 

1. 将区间 [l, r] 解释为两个前缀之间的差异。 我们不是直接计算线段内的三角形，而是计算前缀中直到 r 的所有三角形，并减去完全包含在级别 < l 中的三角形。 
2. 推导或使用已知的结构结果，即高度为 n 的正金字塔中形成的等边三角形的数量由三次多项式 F(n) 给出。 这是因为将三角形分成向上、向下和倾斜方向，每个方向在对可能的边长求和后贡献一个以 n 为单位的多项式计数。 
3. 使用直接封闭式表达式计算 F(n)。 重要的特性是，所有对几何布局的组合依赖性都抵消为具有整数系数的 n 多项式。 
4. 对于每个测试用例，评估 F(r) − F(l − 1)。 这准确地给出了顶点完全位于指定级别范围内的有效三角形的数量。 
5. 以所需的格式化字符串输出结果。 

### 为什么它有效

 金字塔中的每个等边三角形在其顶点中都有唯一定义的最高和最低水平。 这保证了它在前缀计数 F(n) 中只被计数一次，其中 n 是它达到的最大级别。 因此，F(n) 精确计数完全包含在级别 ≤ n 中的所有三角形。 减去 F(l − 1) 会删除完全位于区间上方的所有三角形，仅留下完全包含在 [l, r] 中的三角形。 正确性通过其最大级别减少到三角形的划分，这形成了所有有效配置的不相交分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def F(n):
    if n <= 0:
        return 0
    # Closed form for triangular lattice equilateral triangle count
    # Derived from summing contributions of all orientations
    return n * (n - 1) * (2 * n - 1) // 6

T = int(input())
for tc in range(1, T + 1):
    l, r = map(int, input().split())
    ans = F(r) - F(l - 1)
    print(f"Case #{tc}: {ans}")
```函数 F(n) 表示仅使用 n 级以下顶点的有效等边三角形的累积数量。 减法步骤将此前缀转换为区间查询。 

最微妙的部分是确保正确处理 l − 1。当 l = 1 时，我们必须评估 F(0)，它被安全地定义为 0。这避免了特殊的情况。

 ## 边缘情况

 当 l = r 时，算法计算 F(r) − F(r − 1)。 由于 F 对于非正输入被定义为零，并且仅从级别 2 开始增长，因此结果正确地变为零，这与在单个级别内无法形成三角形的事实相匹配。 

对于 l = 1、r = 1，我们计算 F(1) − F(0)。 在闭合形式下这两个值都为零，因此输出为零，无需特殊分支。 

对于像 [2, 3] 这样的小区间，前缀差异隔离了第一个非平凡的三角形配置。 计算 F(3) − F(1) 正确返回 5 − 0，与晶格中第一次出现的有效等边结构相匹配。
