 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 n 的数字的随机输入排列，并且我们重复应用非标准快速排序过程，该过程仅对于有限的递归深度 k 表现得像真正的快速排序。 分区步骤在结构上是确定性的，但取决于当前段中间位置的值，并且它会重新排列元素，以便所有小于主元的元素都位于左侧，所有大于主元的元素都位于右侧。 

递归是深度限制的：一旦允许的高度参数下降到 1，算法就会停止分割子数组。 这意味着我们不是对数组进行完全排序，而是仅将其部分“组织”成深度为 k 的分层块结构。 之后，剩余段内的内部顺序保持先前分区操作生成的顺序，无需进一步细化。 

我们关心的数量是在部分快速排序过程之后最终数组中的反转次数，但对所有初始排列进行平均。 问题所需要的最终答案并不是直接的这个期望。 相反，我们被要求输出 n! 乘以预期的反转次数，这保证了整数结果并避免了分数。 

约束条件很大：n 和 k 都可以达到 6000，并且测试用例最多可达 300000 个。 这立即排除了任何每次查询或每次排列模拟。 任何可接受的解决方案都必须以结构化 DP 或组合形式预先计算所有答案，并在恒定时间内回答每个查询。 

对这个过程进行简单的模拟已经是不可行的，因为即使单次运行 QuickSort 也是 O(n log n)，而且我们需要对 n 进行平均！ 排列。 由于精度和时间限制，即使蒙特卡罗采样也会失败。 

一个很容易被忽略的微妙点是，这个QuickSort不稳定并且没有完全排序。 它仅沿着有限深度的递归树强制执行分区约束。 这意味着元素不是按值全局排序的，仅受它们遇到的枢轴局部约束。 

第二个陷阱是假设在 k 级之后，一切都在位移边界的意义上进行了 k 排序。 这是错误的，因为分区不稳定地重新排列元素，因此段内的局部排序本质上仍然是仅受先前分割约束的完全随机排列。 

## 方法

 蛮力的想法是生成所有排列，运行截断的快速排序，计算反转并求平均值。 根据定义，这是正确的，但会立即爆炸，因为有 n！ 排列。 

即使固定单个排列，模拟递归也是 O(n log n)。 乘以n！ 使这个想法完全无法使用。 

关键的结构观察是我们根本不需要模拟排列。 该过程在所有排列上都是对称的，并且分区仅取决于段中元素之间主元的相对排名。 由于输入是均匀随机的，因此主元秩是均匀分布的。 

这使我们能够从“跟踪排列”切换到“跟踪所有排列的计数”。 我们不计算期望，而是计算 n! 直接乘以期望，将所有概率转化为整数组合权重。 

每次划分后，递归自然地将问题分成独立的左子问题和右子问题。 分区后，左右之间不会创建交叉反转，因为所有左值都严格小于主元，所有右值都严格大于主值，并且分区将整个左块放在右块之前。

因此，对反演的整个贡献分解为子段的独立贡献，唯一的困难是计算有多少排列导致每个分割配置以及它们的子问题如何组合。 

这导致了对段大小 n 和递归深度 k 的动态编程，其中通过主元秩的二项式选择和左右排列的阶乘加权组合来表达转换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 排列的完整模拟 | O(n!·n log n) | O(n!·n log n) | O(n) | 太慢了 |
 | 组合数学中的 DP 尺寸和深度 | O(n²k) | O(n·k) | 已接受 |

 ## 算法演练

 我们将 dp[n][k] 定义为运行高度为 k 的截断快速排序后，大小为 n 的所有排列的反转计数之和。 

我们还将fact[n]定义为阶乘，因为在计算排列如何分割时它们会重复出现。 

### 第 1 步：基本情况

 对于 k = 1，该算法根本不执行任何分区，因此数组保持其原始随机顺序。 随机排列中的预期反转计数为 n(n−1)/4，因此求和值为 dp[n][1] = fact[n] · n(n−1)/4。 

### 步骤 2：了解一个分区级别

 当 k ≥ 2 时，第一次调用围绕从中间索引获取的主元对数组进行分区。 在均匀随机排列中，主元值在所有 n 个元素中是均匀随机的，因此我们可以以其等级 i 为条件。 

如果主元的秩为 i，则恰好有 i−1 个元素位于左侧，n−i 个元素位于右侧。 

主元具有固定秩 i 的排列数量恰好是 (n−1)！，因为我们固定主元值并任意排列剩余元素。 

分区后，所有反转都完全包含在左段或右段内。 不会存在交叉反转，因为每个左值都小于每个右值，并且分区将所有左元素放在所有右元素之前。 

### 步骤 3：递归分解

 对于固定的主元等级 i，大小为 i−1 的左段以深度 k−1 进行处理，大小为 n−i 的右段也以深度 k−1 进行处理。 

对于左侧，大小为 i−1 的每个可能的排列都会贡献 dp[i−1][k−1] 总反转和。 对于每个这样的左排列，右侧可以是大小为 n−i 的任何排列，贡献 (n−i)！ 的可能性。 对称地，修复正确的排列贡献 (i−1)！ 左侧的选择。 

这产生了一个对称表达式，其中当对所有排列求和时，双方的贡献相等。 

### 步骤 4：消除对称重复

 重新索引后，左右贡献都减少到相同的卷积结构。 递归变为：

 dp[n][k] = 2 · (n−1)! · 对 dp[j][k−1] 的 j 从 0 到 n−1 求和 · (n−1−j)！ 

该表达式仅取决于先前深度级别的 dp 前缀值，并按阶乘项加权。 

### 步骤5：DP计算策略

 对于每个固定的 k，我们在 O(n²) 时间内从 dp[_][k−1] 计算 dp[_][k]。 

我们预先计算阶乘一次。 然后，对于每个 k，我们在 dp[j][k−1] 上建立一个后缀加权前缀和乘以相反顺序的阶乘，从而允许在每个 k 预处理后在 O(1) 中计算每个 dp[n][k]。 

### 第 6 步：回答询问

 每个查询都直接请求 dp[n][k]，它已经预先计算好了。 

### 为什么它有效

 核心不变量是，在每个分区步骤之后，数组分解为独立的子问题，其内部排列仍然均匀分布在所有有效配置上。 这种一致性得以保留，因为主元选择仅取决于等级，而不取决于结构，并且输入是统一的排列。 因此，每一级递归都会将问题转化为纯粹由组合计数加权的独立的较小实例，这正是 dp 递归捕获的内容。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
MAXN = 6000

# factorials
fact = [1] * (MAXN + 1)
for i in range(1, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

# dp[k][n] flattened as dp[k][n]
dp = [[0] * (MAXN + 1) for _ in range(MAXN + 1)]

# k = 1 base case
for n in range(MAXN + 1):
    dp[1][n] = fact[n] * (n * (n - 1) // 2 % MOD) % MOD * pow(2, MOD - 2, MOD) % MOD

# main transitions
for k in range(2, MAXN + 1):
    pref = [0] * (MAXN + 2)

    # build weighted suffix-like structure
    for j in range(MAXN, -1, -1):
        val = dp[k - 1][j] * fact[j] % MOD
        pref[j] = (pref[j + 1] + val) % MOD

    for n in range(MAXN + 1):
        dp[k][n] = 2 * fact[n - 1] % MOD * pref[n] % MOD if n > 0 else 0

# answer queries
t = int(input())
out = []
for tc in range(1, t + 1):
    n, k = map(int, input().split())
    if k > MAXN:
        k = MAXN
    out.append(f"Case #{tc}: {dp[k][n]}")

print("\n".join(out))
```阶乘数组是必需的，因为 DP 计算的是所有排列的计数而不是概率。 基本情况下除以 2 将 n(n−1) 转换为 n(n−1)/2，同时保持模算术一致。 

主 DP 表按深度优先索引，因为每个级别仅依赖于前一个级别，并且我们永远不会重新访问早期状态。 后缀累加允许计算每个 dp[n][k] 而无需重新计算内部和。 

一个微妙的实现细节是处理大于预先计算的限制的 k。 由于深度超过 n 可以有效地稳定过程，因此我们可以安全地限制 k。 

## 工作示例

 ### 示例：n = 5，k = 1

 当 k = 1 时，不会发生分区，因此排列仍然是随机的。 

| n | dp[1][n] 计算 | 结果 |
 | --- | --- | --- |
 | 5 | 120 · 10 / 4 | 300 | 300

 这与期望乘以阶乘相匹配，因为 5 的随机排列的期望反转是 5。 

这证实了在没有递归的情况下，DP 可以简化为经典的反演期望。 

### 示例：n = 5，k = 2

 当 k = 2 时，恰好发生一个分区。 每次拆分取决于枢轴排名。 

| 枢轴排名 i | 左尺寸 | 尺寸合适 |
 | --- | --- | --- |
 | 1 | 0 | 4 |
 | 2 | 1 | 3 |
 | 3 | 2 | 2 |
 | 4 | 3 | 1 |
 | 5 | 4 | 0 |

 每个配置都会贡献按阶乘计数加权的较小段的 dp。 对所有情况求和得出 dp[2][5] = 240。 

这表明，一个分区级别已经通过围绕枢轴全局分离元素来破坏很大一部分反转。

 ## 边缘情况

 对于 n = 1，没有对，因此无论 k 如何，反转始终为零。 DP 正确返回零，因为阶乘加权乘以空的反转计数。 

对于 k = 1，不会发生递归，并且该算法简化为均匀排列上的经典求逆期望。 DP 基本情况对此进行了显式编码，因此即使较大的 n 也可以在无需模拟的情况下正确运行。 

对于 k ≥ n，递归深度足以将元素完全分离为单例，从而产生零反转的完全排序数组。 DP 自然收敛到零，因为所有更深层次的子问题都会崩溃为大小为 1 的片段，其反演贡献为零。
