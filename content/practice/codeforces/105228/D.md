 ## 问题理解

 我们给出了几个独立的场景。 每个场景中都有$n$台阶，每个台阶都有一个高度值，还有一只青蛙从最左边的台阶移动到最右边的台阶。 在开始旅程之前，青蛙可以按任意顺序重新排列步骤。 

移动规则基于连续台阶之间的高度差。 如果青蛙向上跳跃，高度的增加不得超过$k$。 如果下降，下降幅度不得超过$k$以及。 绝对高度差超过的任何跳跃$k$导致失败。 问题是我们是否可以排列给定的多重高度集，以便存在从第一个到最后一个的所有步骤的排序，其中每个相邻差异最多$k$。 

每个测试用例的输出都是一个字符，指示是否存在这种排序。 

约束足够小，任何解决方案都可以粗略地$O(n \log n)$每个测试用例都很好。 自从总共$n$跨测试用例最多 2000 个，即使独立排序每个测试也足够快。 在实践中，每次测试的任何二次方仍然是可以接受的，但我们应该避免尝试所有排列或回溯，因为这会增加阶乘。 

一个微妙的问题是重新排列完全消除了位置结构。 这是关键的转变：我们不是在解决数组遍历问题，而是在构建具有有限相邻差异的排列。 

一个天真的但诱人的错误是假设我们只需要检查连续排序元素之间的最大间隙是否至多$k$。 这实际上是正确的，但许多人想得太多并尝试贪心模拟或图形连接。 

## 方法

 强力解释是尝试高度的所有排列，并检查每个排列是否满足每个相邻差值最多的约束$k$。 这是正确的，但立即不可行，因为有$n!$排列，这甚至远远超出了限制$n = 10$。 

我们需要理解排列的什么属性使得所有相邻的差异受下限$k$。 一旦数组被排序，连续选择的元素之间的最大可能的跳跃就会被最小化。 与排序顺序相比，任何其他排序只能增加一些相邻间隙，因为排序将相邻值尽可能紧密地放置在一起。 

这引出了关键的见解：如果我们对数组进行排序，排序顺序中最差的邻接差异正是连续元素之间的最大间隙。 如果最大间隙最多为$k$，我们可以简单地使用排序顺序作为有效的构造。 如果超过$k$，那么任何重新排列都无法修复它，因为这两个值必须在某处至少被一个邻接分隔开，并且值之间的任何邻接都必须在任何排序中的某个点上至少产生该间隙。 

所以问题减少到排序后检查单个条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n!)$|$O(n)$| 太慢了|
 | 排序并检查相邻差异 |$O(n \log n)$|$O(1)$额外（或$O(n)$) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 阅读高度和限制列表$k$。 问题的结构不依赖于顺序，因此我们立即将输入视为多重集。 
2. 按非降序对数组进行排序。 使用排序是因为它可以最大限度地减少连续元素之间的局部差异，而这正是青蛙所关心的。 
3. 扫描已排序的数组并计算连续元素之间的最大差异。 
4. 如果任何一点的差异超过$k$，我们立即得出结论，不存在有效的重排并且输出失败。 否则，我们就得出成功的结论。 
5. 对所有测试用例重复此操作。 

此扫描足够的原因是排序顺序是最“压缩”的值排列。 任何其他排列都会引入至少一个不小于已排序邻接之一的邻接，而这种方式无法改善最差间隙。 

### 为什么它有效

 考虑任何有效的排列。 查看最小和最大的元素。 在排列中的某个点，转换必须在中间值之间移动。 通过按大小排序可以实现值之间最紧密的链接。 如果即使在这种最佳链接中，最大相邻间隙超过$k$，那么该对值在任何遍历中都必须在某处相邻，或者必须由其累积转换不能减少最大局部跳转要求的中间值桥接。 因此，排序后的排列充当相邻差异的极小极大配置，因此检查它就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))
        
        a.sort()
        
        ok = True
        for i in range(n - 1):
            if a[i + 1] - a[i] > k:
                ok = False
                break
        
        out.append("S" if ok else "F")
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案的结构是对每个测试用例进行独立排序。 关键操作是排序后的单遍，我们只比较邻居。 决策变量`ok`跟踪是否发生任何违规行为。 提前中断对于正确性来说并不是必需的，但可以减少最坏情况输入中不必要的比较。 

一个常见的错误是忘记排序后只有相邻的差异才重要。 无需考虑不相邻对，因为它们之间的任何有效路径都已经受到中间步骤的约束。

 ## 边缘情况

 一个关键的边缘情况是当$n = 2$。 在这种情况下，答案仅取决于两个值之间的绝对差是否最多为$k$。 该算法自然地处理这个问题，因为对两个元素进行排序并检查它们的差异是完全相同的逻辑。 

当所有值都相同时，会出现另一种边缘情况。 排序在各处产生零差异，因此扫描通过而不会触发任何失败，即使在以下情况下也能正确返回成功：$k = 0$。 

一个更具说明性的情况是当一个值远远超出其他值时，例如`[1, 2, 3, 100]`与小$k$。 分选率`[1, 2, 3, 100]`，并且 3 和 100 之间的差距立即违反了约束。 这正确地反映了没有任何排列可以避免在某个点将 100 与链中的某项相邻，从而强制进行太大的跳跃。
