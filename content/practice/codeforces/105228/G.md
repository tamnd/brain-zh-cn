 ## 解决方案
 ## 问题理解

 我们得到了多个独立的查询。 每个查询使用三个点描述平面中的一个三角形。 垂直光源从上方照射，三角形的每个点都会在地面上向下投射垂直投影。 任务是计算位于该投影阴影下方的地面区域的总面积，包括三角形本身正下方的面积。 

重新表述几何对象的一个​​有用方法是：对于三角形覆盖的每个 x 坐标，考虑具有该 x 值的三角形的所有点。 其中，取y坐标最大的。 这定义了 x 上的分段线性“屋顶”函数，所需的答案是该屋顶下直至 y = 0 的面积。 

测试用例数量适中，最多为 1000 个，坐标为绝对值在 1000 左右的小整数。 这立即排除了任何在更高维度的精细网格上进行大量逐点模拟的方法。 对所有整数 x 坐标进行简单扫描仍然是可行的，因为整个 x 范围最多约为 2000，但是每个测试用例中每个像素或每个成对边缘交互重新计算几何形状的任何操作都是安全的，但没有必要。 

当一个顶点按照 x 顺序位于相对边缘的垂直投影“内部”时，就会出现一种微妙的边缘情况。 在这种情况下，上边界不是由所有三个边按顺序形成的，而是仅由三角形凸包的两段形成。 如果我们错误地假设所有三个顶点始终对顶部边界有贡献，则在中间顶点位于连接其他两个顶点的线段下方的情况下，我们将过度计算面积。 

例如，如果点是 A(0,0)、B(1,1)、C(2,0)，则上边界是 A 到 B 到 C。但如果 B 是 (1,0)，则正确的上边界是单个线段 A 到 C，并且将 B 视为边界的一部分会错误地创建“凹痕”并减少面积。 

## 方法

 一个蛮力的想法是直接在精细的 x 网格上模拟阴影。 对于最小和最大 x 坐标之间的每个整数 x，我们计算三角形在该 x 处与垂直线相交的位置，然后取交点之间的最大 y 值并使用小梯形累加面积。 由于坐标范围仅为 2000 左右，因此每个测试用例大约进行 2000 次评估，并且每次评估都需要检查三角形边缘，从而给出大约数百万次操作的安全上限。 它会过去，但它隐藏了几何结构并且更脆弱。 

关键的观察结果是三角形是凸的，因此对于任何固定的 x，三角形内部的点集形成一个垂直线段，并且其上端点沿边缘线性变化。 因此，当投影到 x 轴上时，阴影的上边界就是三角形的上壳。 

一旦我们意识到我们只需要上层外壳，问题就减少到选择三角形的哪些边形成该外壳。 按x坐标对顶点排序后，只有两种可能。 中间顶点位于连接最左边和最右边顶点的线段上方，在这种情况下，上边界从左到中到右。 或者它位于该线段之上或之下，在这种情况下，上边界只是从最左到最右的直线段。 

一旦边界已知，其下方的面积就是连续船体顶点之间的梯形之和，直接使用线性段的标准公式计算。

## 算法演练

 ### 1. 按 x 坐标对三角形顶点进行排序

 我们标记这些点，使得 x₁ ≤ x2 ≤ x₃。 这种顺序决定了水平轴上的最左边、中间和最右边的点，这一点很重要，因为阴影边界在 x 上是单调的。 

### 2.判断中点是否对上边界有贡献

 我们检查中间点是否位于连接最左边和最右边点的线段之上。 这是通过将中点的 y 值与线段在其 x 坐标处的 y 值进行比较来完成的。 

如果在上面，则上边界必须经过中点。 如果不在上面，则从最左到最右的直接段在最大 y 方面已经主导了所有内容，因此中间点与上包络线无关。 

### 3.构造上边界折线

 如果使用中点，则边界是两条线段的折线：从左到中和从中到右。 否则，它是从左到右的单个线段。 

### 4.计算边界下的面积

 对于边界上的每对连续点，计算它们与 x 轴之间的梯形面积。 如果一个线段将 (xₐ, yₐ) 连接到 (x_b, y_b)，则其贡献为 (x_b − xₐ) × (yₐ + y_b) / 2。 

我们将这些贡献相加以获得总阴影面积。 

### 为什么它有效

 该三角形是凸三角形，因此任何垂直线都与它相交成一条线段。 这保证了 x 上的最大 y 值是由三角形的上凸包精确定义的凹分段线性函数。 由于三角形只有三个顶点，因此它的上壳最多可以包含三个点，唯一不明确的是中间的 x 顶点是位于基段的上方还是下方。 一旦正确解决了该选择，被积分的函数在每个区间上都是完全线性的，因此梯形积分是精确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def upper_area(p):
    p.sort()  # sort by x

    (x1, y1), (x2, y2), (x3, y3) = p

    def on_or_below(xa, ya, xb, yb, xc, yc):
        # check if B is above AC using cross product (A->C) x (A->B)
        return (xb - xa) * (yc - ya) - (yb - ya) * (xc - xa) <= 0

    # if middle point is not above segment x1-x3, use direct segment
    if on_or_below(x1, y1, x3, y3, x2, y2):
        hull = [(x1, y1), (x3, y3)]
    else:
        hull = [(x1, y1), (x2, y2), (x3, y3)]

    area = 0
    for i in range(len(hull) - 1):
        xA, yA = hull[i]
        xB, yB = hull[i + 1]
        area += (xB - xA) * (yA + yB) / 2

    return area

def main():
    t = int(input())
    out = []
    for _ in range(t):
        arr = list(map(int, input().split()))
        pts = [(arr[0], arr[1]), (arr[2], arr[3]), (arr[4], arr[5])]
        out.append(str(int(upper_area(pts))))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```解决方案首先对点进行排序，以便三角形的水平结构变得明确。 功能`on_or_below`使用叉积测试来确定中间点是否位于连接外部点的线上方。 这避免了浮点运算并将所有内容保留在整数空间中。 

然后，面积计算是沿上边界段的梯形积分的直接应用。 在这里使用浮点除法是安全的，因为问题保证了整数输出，但在内部，由于整数端点，值保持精确。

 ## 边缘情况

 当排序后三角形的最高点不是中间的 x 顶点时，就会出现临界边缘情况。 例如，如果最左边或最右边的顶点也是最高点，该算法仍然有效，因为船体结构自然减少到两个点，并且梯形公式正确地折叠成一个大线段。 

另一个微妙的情况是，中点恰好位于其他两个点之间的线段上。 在这种情况下，叉积测试返回零，并且算法正确地丢弃中间点，避免不必要的分割，否则会引入冗余但无害的分割。
