 ## 解决方案
 ## 问题理解

 我们正在研究一个由围成一圈的人玩的连续游戏。 每个玩家都有一个固定的“最喜欢的号码”。 面从 1 到 m 的单个骰子会重复滚动，但滚动不会全局共享。 相反，玩家按顺序轮流：首先玩家 1 掷骰，然后玩家 2，依此类推，在玩家 n 之后循环回到玩家 1。 

当第一次掷骰结果是当前玩家最喜欢的数字的约数时，游戏就会停止。 该玩家立即获胜，并且不会再进行掷骰。 每个卷都是独立的，并且在 1 到 m 的整数上均匀随机。 

任务是计算每个玩家在轮到时第一个触发停止条件的概率。 

重要的结构是，该过程是独立试验的单一演变序列，但每个试验都有不同的“成功条件”，具体取决于轮到谁。 如果所有先前的玩家在自己的回合中反复失败，然后他们在自己的一个回合中成功，则玩家获胜。 

这些限制非常重要。 当 n 达到 100000，m 达到 10^13 时，我们无法简单地模拟该过程或枚举每个数字的除数。 如果分解到 m，即使独立计算每个玩家的除数也会失败。 我们需要一个公式，在预处理后，每个玩家的评估时间接近 O(1) 或 O(log m)。 

当尝试独立计算每个玩家的概率时，会出现一个幼稚的错误，就好像它们是孤立的伯努利过程一样。 这忽略了循环依赖：如果玩家 i 在回合中失败，则过程将移至 i+1，并且成功概率会跨周期累积。 

另一个微妙的问题是假设玩家获胜赛事之间的独立性。 它们不是独立的，因为只有一个玩家获胜并且概率质量流经循环。 

一个小的说明性边缘情况是 n=2、m=2、a=[1,1]。 如果两个玩家都掷出 1，那么他们总是会成功，因此玩家 1 以 1/2 的概率获胜，而玩家 2 以 1/4 的概率获胜。 天真的“每个人都有平等的机会”或独立计算将会失败。 

## 方法

 暴力解释逐步模拟该过程。 每回合，我们都会在 [1, m] 中滚动一个数字，检查它是否整除当前玩家的 a[i]，然后停止或前进。 该模拟需要对所有可能的滚动序列进行采样或枚举。 状态空间的长度是无限的，因为游戏可以无限地继续，甚至在合理的深度截断也会导致指数分支。 每个步骤都有 m 个可能性，经过 k 个回合我们已经有 m^k 个结果，即使对于很小的 k 也是不可能的。 

关键的观察是每个回合都是独立的并且仅取决于当前玩家。 所以我们不关心确切的序列，只关心玩家之间的转换概率。 对于任何玩家 i，游戏在 i 结束的概率为 p_i，并且移动到 i+1 的概率为 (1 - p_i)。 

这将问题转化为具有吸收状态的循环上的马尔可夫链：每个状态都有一个自吸收概率 p_i，否则确定性地转换到下​​一个状态。 该结构在圆上是线性的，因此我们可以通过用围绕循环的失效概率的乘积来表示最终的吸收概率。 

设 f_i 为玩家 i 在其回合中失败的概率，p_i = 1 - f_i 为他们单次访问的成功概率。 那么该过程完成一个完整循环而没有人获胜的概率就是所有 f_i 的乘积。 这会在循环中产生几何重复，并且每个玩家的最终获胜概率成为他们在一个循环中的贡献，该贡献按早期循环全部失败的概率进行缩放。

剩下的任务是有效地计算 p_i：我们需要 [1, m] 中的随机数除以 a_i 的概率，这只是 a_i ≤ m 除以 m 的除数的数量。 由于 a_i ≤ m，a_i 的所有除数都是有效候选数，因此 p_i = d(a_i)/m，其中 d(a_i) 是除数数。 

因此，核心减少到计算最多 10^5 个数字的除数计数，然后将它们组合成循环概率积。

 ## 算法演练

 我们首先计算每个玩家轮到他们时成功的概率。 

1. 对于每个值 a[i]，计算其除数 d[i] 的数量。 这是通过对 a[i] 进行质因数分解并使用指数规则来完成的。 此步骤是必要的，因为成功概率仅取决于 [1, m] 中有多少个值除以 a[i]，这等于除数计数，因为 a[i] ≤ m。 
2. 将每个除数计数转换为概率 p[i] = d[i] / m modulo MOD。 这代表轮到玩家 i 时立即获胜的机会。 
3. 定义 f[i] = 1 - p[i]，玩家 i 在自己的回合中没有获胜的概率。 这是游戏继续经过玩家 i 的概率。 
4. 计算玩家完整循环没有获胜者的总概率，即 F = 所有 f[i] 的乘积。 这捕获了这样的事件：在一轮完整的比赛之后，没有人获胜并且系统重置为相同的状态。 
5. 计算 f[i] 绕圆的前缀积。 对于每个玩家 i，我们想要在给定周期达到 i 之前循环中所有先前玩家失败的概率。 
6. 组合贡献：如果我们处于某个周期 k 中，其中所有先前的 k-1 个周期都没有获胜者（因子 F^(k-1)），则玩家 i 获胜，该周期中 i 之前的所有玩家在周期 k 中失败，而 i 在周期 k 中成功。 这形成了一个几何级数，使用 (1 - F) 的模逆函数将其折叠成封闭形式。 
7. 将 i 之前的前缀故障乘积乘以 p[i]，然后按 (1 - F) 的倒数缩放以考虑重复循环。 

关键的不变量是，在没有人获胜的每个完整周期之后，系统将返回到具有独立的未来掷骰的相同状态。 这使得该过程在循环边界处无记忆，从而允许几何级数崩溃。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def factorize(x):
    i = 2
    res = {}
    while i * i <= x:
        while x % i == 0:
            res[i] = res.get(i, 0) + 1
            x //= i
        i += 1
    if x > 1:
        res[x] = res.get(x, 0) + 1
    return res

def divisor_count(x):
    f = factorize(x)
    cnt = 1
    for e in f.values():
        cnt *= (e + 1)
    return cnt

n, m = map(int, input().split())
a = list(map(int, input().split()))

p = []
for x in a:
    d = divisor_count(x)
    p.append(d * modinv(m) % MOD)

f = [(1 - x) % MOD for x in p]

pref = [1] * n
for i in range(n):
    pref[i] = f[i] * (pref[i - 1] if i else 1) % MOD

F = pref[-1]

inv_cycle = modinv((1 - F) % MOD)

ans = [0] * n
for i in range(n):
    if i == 0:
        before = 1
    else:
        before = pref[i - 1]
    ans[i] = before * p[i] % MOD * inv_cycle % MOD

print(*ans)
```该代码首先计算每个玩家号码的模逆和基于因式分解的除数计数。 这是唯一昂贵的算术步骤，但它仍然可行，因为每个数字都是独立分解的。 

每个概率都被仔细地转换为模形式，并且补数被计算为 1 减去模算术中的该值。 前缀乘积数组对一个周期中所有早期玩家失败的概率进行编码。 

跨周期的几何重复由术语 inv_cycle 处理，它对应于对全周期故障的无限几何级数求和。 最终答案乘以三个部分：所有先前玩家在一个周期中失败的概率、当前玩家成功的概率以及重复周期的标准化。 

一个微妙的点是（1 - F）的模块化处理。 由于在简并情况下 F 可以为 1，因此模逆仅在循环具有非零终止概率时才有效，这一点成立是因为每个参与者至少有一个有效除数。

 ## 边缘情况

 一个关键的边缘情况是所有玩家都有相同的成功概率。 在这种情况下，前缀产品仍然因位置而异，因此较早的玩家占主导地位。 例如，如果所有 a[i]=1，则每个 p[i]=1/m，并且该算法简化为纯循环几何过程，其中仅排序决定最终概率。 

另一个边缘情况是 m=1。 每次掷骰都是 1，因此每个玩家在第一回合都会立即成功。 前缀结构确保只有玩家 1 获得概率 1，因为该过程永远不会到达其他玩家。 该算法捕获了这一点，因为 p[1]=1 且 f[1]=0，使得循环在第一个玩家处立即终止。
