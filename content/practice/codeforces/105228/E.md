 ## 解决方案
 ## 问题理解

 我们获得了一系列鸽子品种，其中每个品种都有有限的可供购买的鸽子供应。 我们的目标是建造尽可能多的相同的鸽舍。 每栋房子都必须包含准确的`k`鸽子，同一房屋内的两只鸽子不能属于同一物种。 然而，同一物种可以出现在多个不同的鸡舍中，只要我们不超过该物种可用的鸽子数量。 

我们不需要显式地构造作业。 我们只需要确定在这些限制下可以形成的最大房屋数量。 

关键的困难在于，选择更多的鸡舍限制了每个物种重复使用的频率，因为单个物种最多只能为每个鸡舍贡献一只鸽子。 如果我们尝试构建`H`房屋，然后是每个物种`i`最多可以贡献`min(p_i, H)`共有鸽子。 

这立即暗示了一个可行性条件：如果我们确定候选房屋数量`H`，所有物种的可用鸽子总数变为`sum(min(p_i, H))`。 因为每个房子都需要`k`鸽子，我们需要`sum(min(p_i, H)) >= H * k`。 

约束条件达到`n = 10^5`和`p_i ≤ 10^9`，因此任何尝试所有可能的分配或模拟每个房子的分配的解决方案都会太慢。 对固定的天真检查`H`很好，但我们必须能够有效地测试许多值，这表明单调可行性结构和二分搜索。 

当一个物种的体型非常大时，就会出现微妙的边缘情况。`p_i`。 天真的解释可能会错误地认为这样的物种总是可以对所有房屋做出完全的贡献，但由于“每间房屋一个”的限制，它仍然受到房屋数量的限制。 例如，如果`k = 2`,`p = [1000]`，那么即使我们有很多鸽子，我们也永远不能建造超过 0 栋房子，因为我们没有足够的不同物种来填满一个房子。 

另一个边缘情况出现时`k > n`。 即使鸽子总数很多，单个鸡舍也需要`k`不同的物种，如果我们的物种少于`k`物种。 那么正确答案总是零。 

## 方法

 一种直接的尝试是一栋一栋地建造房屋。 对于每个房屋，我们会反复挑选仍然有剩余鸽子的物种，并确保每个房屋的每个物种最多挑选一只鸽子。 填满每个房子后，我们减少计数并继续。 这很快就会变得昂贵，因为每个选择步骤可能会扫描许多物种，并且我们可能会重复此过程最多`10^9`推理空间最坏情况下的时间。 即使使用贪婪堆，结构也不稳定，因为约束是跨房屋的全局约束，而不是局部约束。 

关键的观察是我们不需要模拟施工。 相反，我们只需要检查是否有给定数量的房屋`H`是可行的。 对于每个物种`i`，最多可以贡献`H`鸽子（每户一只），但也不能超过`p_i`。 所以它的真正贡献是有限的`min(p_i, H)`。 

如果我们将这些贡献相加，我们就得到了所有房屋中可以填充的鸽子位置总数。 因为每间房子都需要准确的`k`对于鸽子，可行性减少到一个不等式。 

这将问题转化为单调决策问题：如果`H`房子是可能的，那么任何`H' < H`也是可以的。 这种单调性允许二分查找`H`。 对于每个中点，我们计算可行性`O(n)`使用排序数组或前缀和。 排序一次可以快速计算`sum(min(p_i, H))`将物种分为以下几类`H`以及以上那些。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每栋房子的贪婪模拟 | O(H·n) | O(n) | 太慢了|
 | 具有可行性检查的二分搜索 | O(n log n + n log max_p) | O(n log n + n log max_p) | O(n) | 已接受 |

 ## 算法演练

 ### 1. 对可用鸽子进行排序

 我们对数组进行排序`p`按非递减顺序。 这使我们能够快速分离出完全可用的物种（`p_i < H`）来自那些上限为`H`。 

### 2. 定义固定数量房屋的可行性函数`H`对于每个物种，我们计算它可以贡献多少只鸽子：

 如果`p_i < H`，它贡献`p_i`。 

不然就贡献了`H`。 

我们将所有物种的数据相加，得到可用鸽子总数。 

此步骤很重要，因为它隐式编码每个房屋的限制，而无需构造任何分配。 

### 3.检查是否`H`房子可以填满

 我们将可用鸽子总数与所需鸽子进行比较`H * k`。 如果可用的鸽子至少有这么大，那么我们就可以将鸽子分配到各个房屋中。 

### 4.二分查找最大可行`H`我们搜索从`0`到`sum(p_i) // k`。 对于每个中点，我们使用上面的函数测试可行性。 如果可行，我们向右移动； 否则，我们向左移动。 

答案是最大可行`H`。 

### 为什么它有效

 关键不变量是对于任何固定的`H`，表达式`sum(min(p_i, H))`是可以合法分配的鸽子位置的确切最大数量`H`房屋。 “每个房屋每个物种一个”的限制被完全体现在`min(p_i, H)`上限，并且不存在进一步的结构限制，因为房屋是可互换且对称的。 因此，可行性仅取决于总容量是否覆盖总需求，单调性保证了二分查找的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def feasible(p, k, H):
    total = 0
    for x in p:
        if x >= H:
            total += H
        else:
            total += x
        if total >= H * k:
            return True
    return total >= H * k

def solve():
    n, k = map(int, input().split())
    p = list(map(int, input().split()))
    
    if k > n:
        print(0)
        return

    p.sort()

    lo, hi = 0, sum(p) // k
    ans = 0

    while lo <= hi:
        mid = (lo + hi) // 2
        if feasible(p, k, mid):
            ans = mid
            lo = mid + 1
        else:
            hi = mid - 1

    print(ans)

if __name__ == "__main__":
    solve()
```这`feasible`函数计算可用鸽子的总数，假设每个房屋中任何物种都不能使用一次以上。 提前退出时`total >= H * k`防止不必要的求和`H`很大。 

二分查找范围是安全的，因为每栋房子的消耗量都是一样的`k`鸽子，因此房屋的最大数量不能超过鸽子总数除以`k`。

 ## 边缘情况

 当`k > n`，算法立即返回零，因为没有足够多的不同物种可以形成房屋。 即使`p_i`规模很大，每栋房子的多样性限制阻碍了任何建设。 

对于非常大的`p_i`， 例如`p = [10^9, 10^9, ..., 10^9]`，二分查找将推入`H`高，但可行性将正确限制增长，因为每个物种仅贡献`H`每栋房屋，防止过度计数。 

当所有`p_i`小且相同，二分搜索紧密收敛，一旦达到阈值，每个可行性检查都会提前终止`H*k`达到了，即使在最坏情况下均匀分布也能确保效率。
