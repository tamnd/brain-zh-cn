 ## 解决方案
 ## 问题理解

 我们有一组任务，每个任务 i 都有一个时间成本 Ai 和一个价值 Bi。 一个人的总时间预算为 M，并且可以选择任务的子集来完成，每个任务最多一次，可以任意顺序，只要花费的总时间不超过 M。 

不同的是，奖励不是以通常的方式累加的。 任务完成后，并不直接给予固定奖励。 相反，每次任务完成时，该人都会回忆起迄今为止完成的所有任务，并再次获得他们的 Bi 值。 因此，完成的奖励取决于已经完成的任务数量。 

如果我们从排序角度考虑，当序列中的第 k 个任务完成时，它会贡献 Bi 与所有先前完成的任务的总和，不包括此时的自身。 

目标是选择任务的顺序和子集，以在时间限制下最大化总累积记忆奖励。 

任务数量的限制很小，N ≤ 30，但时间预算 M 非常大，高达 10^9，成本 Ai 高达 10^9。 这立即排除了任何追踪时间作为维度的 DP。 经典的背包超过M是不可能的。 

相反，该结构建议对任务进行子集选择，因为 N 对于指数子集来说足够小。 

如果我们忽略排序，就会出现微妙的失败情况。 例如，假设存在两个任务：

 输入：

 N = 2，M = 10

 A = [6, 6]

 B = [100, 1]

 如果我们两者都选择，那么顺序很重要。 如果我们首先执行高 B 任务，它会给出 0，然后低 B 任务会给出 100。总计 100。如果颠倒过来，总计为 1。忽略排序的简单子集和式解决方案将错过这种区别。 

当我们假设每个任务的贡献是其后已完成任务数量的 B 倍时，就会出现另一种失败情况。 这是正确的，但前提是谨慎处理订购。 

## 方法

 关键的困难在于奖励取决于相对顺序：较早完成的任务对较晚完成的任务有利，而较晚完成的任务则不会从未来的任务中受益。 

强力方法是尝试所有子集和每个子集的所有排列，计算总时间和奖励，并采用最佳有效配置。 对于大小为 k 的子集，排列贡献 k!，对所有子集求和得出的总复杂度约为 Σ k! C(N, k)，即使 N = 30，其增长也远远超出了可行性。 

关键的观察结果是，对于任何固定子集，最佳排序完全是通过按 Bi 对任务进行排序来确定的。 如果我们提前完成具有较大 Bi 的任务，我们就会增加应用于该任务的未来乘数的数量。 然而，我们需要仔细推理：每个已完成的任务都会将其 Bi 贡献给所有未来已完成的任务，这意味着任务的 Bi 就像一个权重，根据其后的任务数量而被多次计算。 

如果我们反转视角，不去思考“每次完成都会收集过去的 Bi”，我们可以这样想：i 在 j 之前的每对 (i, j) 都会为 j 的奖励积累贡献 Bi。 因此，对于按所选顺序执行的每个 Bi 任务，每个 Bi 都会被精确计数一次。 

因此，如果选定的子集 S 排序为 s1、s2、...、sk，则总奖励变为：

 B[s1]·(k-1) + B[s2]·(k-2) + ... + B[sk]·0

 当较大的 Bi 较早出现时，该值会最大化，因为它们会乘以较大的系数。 

因此，对于固定子集，我们通过递减 Bi 进行排序并计算：

 对位置 i 求和： B[i] × (k - 1 - i)

 现在问题简化为选择一个总时间 ≤ M 的子集来最大化该值。 

这是 N ≤ 30 上的经典中间相遇背包。我们将任务分成大小为 15 的两半。对于每一半，我们枚举所有子集，计算（时间，贡献）并存储它们。 然后我们随着时间的推移使用排序和优势修剪进行合并。

然后，我们组合左子集和右子集：如果左子集有时间 tL，右子集有 tR，我们要求 tL + tR ≤ M。对于每个右子集，我们使用已排序左状态的前缀最大值来维护剩余容量的最佳可实现值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(N!·2^N) | O(N!·2^N) | O(1) | O(1) | 太慢了 |
 | 中间相遇子集枚举 | O(2^(N/2) log 2^(N/2)) | O(2^(N/2) log 2^(N/2)) | O(2^(N/2)) | O(2^(N/2)) | 已接受 |

 ## 算法演练

 1. 将任务分成两半。 这使得枚举易于管理，因为每一半最多有 15 个元素，使得 2^15 状态可行。 
2. 对于每一半，使用位掩码枚举所有子集。 对于每个子集，假设该子集中的最佳排序，计算总时间并计算奖励。 
3. 要计算子集的奖励，请收集子集中的所有 Bi，按降序对它们进行排序，然后应用加权和，其中较早的元素获得更高的重数。 
4. 将每个子集状态存储为一对（时间、值）。 
5. 将左半部分的所有状态按时间排序，并计算值的前缀最大值。 这允许在时间限制下快速查询最佳左子集。 
6. 对于每个右子集，计算剩余容量 M - tR，并在左数组中进行二分搜索以找到最佳兼容子集。 
7.跟踪左子集和右子集的所有组合的最大值。 

为什么它有效：任何最佳解决方案都会唯一地分为左子集和右子集。 在每个子集中，通过对 Bi 进行递减排序，排序是局部最优的。 中间相遇合并会检查所有可行的分割而不会丢失组合，并且前缀最大值确保我们始终为每个右侧状态选择最佳兼容的左侧状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def gen_half(A, B):
    n = len(A)
    states = []
    for mask in range(1 << n):
        t = 0
        vals = []
        for i in range(n):
            if mask & (1 << i):
                t += A[i]
                vals.append(B[i])
        if t > 0 or mask == 0:
            vals.sort(reverse=True)
            k = len(vals)
            score = 0
            for i, v in enumerate(vals):
                score += v * (k - 1 - i)
            states.append((t, score))
    return states

def solve():
    n, m = map(int, input().split())
    A = list(map(int, input().split()))
    B = list(map(int, input().split()))

    mid = n // 2
    leftA, rightA = A[:mid], A[mid:]
    leftB, rightB = B[:mid], B[mid:]

    left = gen_half(leftA, leftB)
    right = gen_half(rightA, rightB)

    left.sort()
    # prune dominated by time
    filtered_left = []
    best = -1
    for t, v in left:
        if v > best:
            best = v
        filtered_left.append((t, best))

    times = [t for t, _ in filtered_left]
    vals = [v for _, v in filtered_left]

    import bisect

    ans = 0
    for tR, vR in right:
        if tR > m:
            continue
        rem = m - tR
        idx = bisect.bisect_right(times, rem) - 1
        if idx >= 0:
            ans = max(ans, vR + vals[idx])
        else:
            ans = max(ans, vR)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将数组分成两半，以便仅对大小为 15 的组启用指数枚举。 子集生成器计算每个子集的时间成本和奖励值。 奖励计算显式地按降序对选定的 Bi 值进行排序，并应用从成对贡献解释导出的位置权重。 

生成左右状态后，左侧被处理成单调结构：我们按时间排序并保持可实现奖励的运行最大值。 这消除了较长时间不能提供更好奖励的主导状态。 

然后使用二分搜索将每个右子集与最兼容的左子集配对。 这确保我们尊重总时间限制，同时最大化组合奖励。 

一个微妙的实现点是我们必须正确包含空子集。 它贡献零时间和零奖励，并充当有效的基线。 

## 工作示例

 考虑一个小例子：

 N = 4，M = 10

 A = [3,4,5,2]

 B = [8, 1, 7, 6]

 分为 [0,1] 和 [2,3]。 

左子集：

 | 面膜| 时间 | B 选择 | 排序 B | 分数 |
 | --- | --- | --- | --- | --- |
 | 00 | 00 0 | []| []| 0 |
 | 01 | 3 | [8] | [8] | 0 |
 | 10 | 10 4 | [1] | [1] | 0 |
 | 11 | 11 7 | [8,1]| [8,1]| 8 |

 右子集：

 | 面膜| 时间 | B 选择 | 排序 B | 分数 |
 | --- | --- | --- | --- | --- |
 | 00 | 00 0 | []| []| 0 |
 | 01 | 5 | [7] | [7] | 0 |
 | 10 | 10 2 | [6] | [6] | 0 |
 | 11 | 11 7 | [7,6]| [7,6]| 7 |

 现在组合起来，例如右掩码 11 的时间为 7，因此左掩码必须适合时间 ≤ 3。只有左掩码 00 和 01 适合，给出最佳值 0。总计为 7。 

对于右掩码 10（时间 2），我们可以使用左掩码 11（时间 7），总时间 9，值 8 + 0 = 8。 

此跟踪显示时间过滤如何与子集值聚合交互。

 ## 边缘情况

 一种重要的边缘情况是，由于 Ai 值较大，因此不选择最佳任务。 在这种情况下，除了空子集之外，所有子集时间都超过M。 该算法自然地将空状态包含在两半中，因此将它们组合起来会产生 (0, 0)，这是正确考虑的。 

另一个边缘情况是所有 Bi 都相同。 在这种情况下，排序并不重要，奖励会减少为子集大小的函数。 子集评分仍然会产生一致的结果，因为每个排列都会产生相同的总成对贡献。 

最后一种情况是 M 足够大以包含所有任务。 然后，中间相遇合并完全选择两半，并且通过减少 Bi 计算的排序产生全局最优值，匹配导出的成对结构。
