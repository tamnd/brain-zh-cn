 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中每个节点代表扩散过程中的一个状态。 从根开始，令牌向下移动，直到到达叶子。 在每个节点，如果没有做任何特殊的事情，则下一步不受控制：该过程可能会移动到当前节点的任何子节点，这意味着每个传出边缘都是可能的延续。 

有些叶子被标记为理想的结果。 目标是保证，无论随机选择在下降过程中的表现如何，该过程始终以这些标记的叶子之一结束。 

我们还获得了一组可选的“控件”。 每个控件都与特定节点 u 和 u 的特定子节点 v 相关联。 如果我们激活这个控制，那么每当令牌位于 u 时，它就不再随机分支：它被迫确定性地转到 v。 

我们可以选择这些控件的任何子集，但有一个优先级规则：如果多个选定的控件影响同一节点，则只有输入顺序中最早的一个有效，其余的将被忽略。 

任务是找到我们需要激活的最小控制数量，以便保证进程到达标记的叶子。 如果这是不可能的，我们输出-1。 

测试用例中的树大小最多可达 5×10^5，因此任何解决方案都必须是线性或接近线性的。 任何涉及对所有子节点重复进行每节点重新计算或对控件进行任何子集枚举的操作都会立即变得太慢。 

一个微妙的问题是失败是如何发生的。 如果在任何节点我们让分支行为处于活动状态，那么子选择的对抗序列可以故意引导该过程远离所有好的叶子。 这意味着正确性与概率无关，而是与最坏情况的保证有关。 

一个常见的错误是假设每个节点至少有一个“好”子子树就足够了。 这是不够的，因为即使存在好孩子，随机分支仍然可以选择坏孩子。 

## 方法

 一个蛮力的想法是尝试所有控件子集。 对于每个子集，我们模拟最坏情况分支下的过程：在每个非强制节点，我们假设对手选择最坏的可能子节点。 如果我们仍然可以保证到达标记的叶子，我们就会更新答案。 

这立刻就爆炸了，因为使用 M 个控件就有 2^M 个子集，而 M 可以和 N 一样大。即使检查一个子集也需要遍历树并推理所有路径，因此这种方法是完全不可行的。 

关键的观察是，问题不在于独立探索许多路径，而在于消除不确定性。 当一个节点仍然有可能导致不安全子树的分支选择时，它就是危险的。 消除这种危险的唯一方法是确保所有子节点都是安全的，或者强制节点成为单个安全子节点。 

这导致了自下而上的视图。 如果从该节点开始，无论分支如何解析（或我们如何应用允许的控制），我们仍然可以保证以标记的叶子结束，则每个节点都可以归类为“安全”。 一旦定义了安全性，最小化控制就成为一个局部决策：要么我们不支付任何费用并依赖所有孩子的安全，要么我们支付一个控制来强制一个单一的安全方向并完全绕过所有其他孩子。 

这将全局优化问题简化为树DP，其中每个节点在“此处无控制”和“此处仅激活一个控制”之间做出决定。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对控制子集的暴力破解 | O(2^M·N) | O(2^M·N) | O(N) | 太慢了 |
 | 具有强制转换的树DP | O(N + M) | O(N + M) | 已接受 |

 ## 算法演练

 我们从叶子向上处理树，计算每个节点是否可以使其安全以及所需的最小控制数量是多少。

1. 首先，如果每个叶子是目标叶子之一，则将其标记为安全。 任何不在目标集中的叶子都会立即不安全，因为达到它违反了要求。 这定义了递归的基础。 
2. 对于每个内部节点，我们考虑两种根本不同的模式。 在第一种模式中，我们不在该节点激活控件，因此进程在所有子节点之间自由分支。 在这种情况下，安全性要求每个子子树都已经是安全的，因为在最坏的情况下可以选择任何子树。 
3. 如果所有子节点都是安全的，那么该节点也可以被认为是安全的，而无需在该节点激活任何控制。 在这种情况下，成本只是其子级成本的总和，因为他们的决定是独立的。 
4. 如果即使有一个子节点不安全，则让该节点不受控制也是不可接受的，因为该进程可能会移动到该不安全的子树中。 在这种情况下，我们必须考虑激活该节点上的控件。 
5. 当在节点 u 处激活控件时，它会强制移动到特定的子节点 v。这会删除 u 处的所有分支，因此所有其他子节点都变得无关紧要。 当我们可以选择一个控制 u → v（其中 v 是安全的）时，该节点就变得安全。 该选项的成本是 1 加上使 v 安全的成本。 
6. 我们将每个节点的答案计算为非受控选项（如果有效）和所有有效受控选项之间的最小值。 如果两个选项都不存在，则该节点不安全。 
7. 最终答案是在根处计算的成本，除非根不安全，在这种情况下答案为-1。 

### 为什么它有效

 关键的不变量是，当且仅当存在选择控制的策略时，节点才被标记为安全，使得从该节点开始，过程的每个可能的演化都被迫在目标叶中结束。 不受控制的案例模拟了对抗性分支，只有当所有孩子都安全时才是安全的。 受控案例将所有分支折叠成单个选定的子项，因此安全性完全取决于该选定的子项。 由于每个控制要么消除节点上的所有不确定性，要么使其完全暴露，因此这是安全性向上传播的唯一两种方式，并且DP在没有遗漏或重复计算的情况下耗尽了这两种可能性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

INF = 10**18

def solve():
    T = int(input())
    for _ in range(T):
        N, M, K = map(int, input().split())
        
        children = [[] for _ in range(N + 1)]
        parent = [0] * (N + 1)
        
        for i in range(1, N + 1):
            arr = list(map(int, input().split()))
            ki = arr[0]
            for v in arr[1:]:
                children[i].append(v)
                parent[v] = i
        
        cand = [[] for _ in range(N + 1)]
        for _ in range(M):
            u, v = map(int, input().split())
            cand[u].append(v)
        
        target = set(map(int, input().split())) if K > 0 else set()
        
        # order nodes by reverse BFS / topo from leaves
        order = []
        stack = [1]
        while stack:
            u = stack.pop()
            order.append(u)
            for v in children[u]:
                stack.append(v)
        
        # postorder by reversing is not guaranteed safe due to reuse,
        # so we compute explicit order via DFS iterative
        order = []
        stack = [(1, 0)]
        while stack:
            u, state = stack.pop()
            if state == 0:
                stack.append((u, 1))
                for v in children[u]:
                    stack.append((v, 0))
            else:
                order.append(u)
        
        safe = [False] * (N + 1)
        cost = [0] * (N + 1)
        
        for u in order:
            if not children[u]:  # leaf
                safe[u] = (u in target)
                cost[u] = 0
                continue
            
            all_children_safe = True
            sum_cost = 0
            for v in children[u]:
                if not safe[v]:
                    all_children_safe = False
                sum_cost += cost[v]
            
            best = INF
            if all_children_safe:
                safe[u] = True
                best = min(best, sum_cost)
            
            # try forcing at u
            for v in cand[u]:
                if safe[v]:
                    safe[u] = True
                    best = min(best, 1 + cost[v])
            
            cost[u] = best if best < INF else 0
            if best < INF:
                safe[u] = True
        
        print(cost[1] if safe[1] else -1)

if __name__ == "__main__":
    solve()
```该代码首先构建树并存储按起始节点分组的候选控件。 然后，它按后序处理节点，以便每个子节点都在其父节点之前计算。 

对于每个节点，它检查所有子节点是否安全。 如果是，则非受控模式有效并贡献子成本总和。 然后，它评估从该节点开始的所有控件，每个控件都会将该节点折叠为单个子状态，并将成本加一。 

一个微妙的细节是安全性和成本是一起跟踪的。 仅当存在至少一个有效构造时，节点才被认为是安全的，否则其成本无关紧要，并且该节点将作为不安全的传播。

 ## 边缘情况

 一个关键的边缘情况是一个节点有多个子节点，但所有子节点都已经是安全的。 在这种情况下，即使存在分支，也不需要控制，因为每个可能的分支都是可接受的。 该算法通过“所有子节点安全”条件来处理此问题，这允许节点保持不受控制。 

另一种边缘情况是当一个节点同时具有安全和不安全的子节点，但该节点不存在控制时。 在这种情况下，非受控模式会失败，因为可以选择不安全的子级，而受控模式则不可能，因为无法重定向。 该节点被正确标记为不安全，并将故障向上传播，最终导致根无法生成。 

最后一个重要的边缘情况是 K 为空时。 那么没有叶子是可接受的，因此所有叶子节点都是不安全的，并且这种情况向上传播，使得根也变得不安全。
