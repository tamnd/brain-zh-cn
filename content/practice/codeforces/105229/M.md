 ## 解决方案
 ## 问题理解

 我们得到一行$n$睡莲从左到右标记。 我们需要在这些位置上设计两个独立的定向连接系统。 

第一个系统属于青蛙。 它由某些位置的定向跳跃组成$a_i$到$b_i$，总是与$a_i < b_i$，并且每个位置最多使用一次作为开始，最多使用一次作为结束。 由于这一限制，青蛙的移动形成了线上不相交的有向边的集合。 

第二个系统属于鸟类。 它以相同的方式使用对来定义$x_i \to y_i$，并且所有开始和结束都不同。 

一旦这两个系统被固定，就可以通过重复地跟随有向边来移动，因此青蛙和鸟都可以沿着各自有向图中的路径移动。 

一对位置$(i, j)$如果青蛙能够到达就会变得危险$j$从$i$使用它的跳跃系统，同时鸟也可以到达$j$从$i$使用其飞行系统。 即使有一对这样的存在，鸟最终也能抓住青蛙。 

任务是以不存在此类重叠可达性对的方式构建两个系统，同时最大化两个系统之间的边总数。 

关键的限制是每个系统的结构非常有限：因为所有源都是不同的，所有目标都是不同的，所以每个系统都是从左到右的部分匹配。 然而，可达性是可传递的，因此即使边缘在本地看起来很简单，链接也可能会产生远程可达性并可能产生禁止的重叠。 

和$n \le 10^4$，我们可以负担得起$O(n)$或者$O(n \log n)$建造。 任何涉及全局成对推理的事物$O(n^2)$成对是不可能的。 

当一个系统形成一条链时，就会出现微妙的故障情况。 例如，如果我们将青蛙边缘构建为$1 \to 2, 2 \to 3, 3 \to 4$，那么青蛙可以从每个较早的节点到达每个较晚的节点。 在这种情况下，鸟儿必须完全避开每一对可以到达的鸟，这会使其结构几乎崩溃。 这表明长链是危险的，因为它们会通过传递性破坏可达性。 

## 方法

 一个蛮力的想法是尝试所有可能的方法，在唯一性约束下将有向边分配给青蛙和鸟，计算两个图中的可达性，并检查两个图中是否有任何对是可达的。 这很快就变得不可行，因为即使构建一个图的可达性也是如此$O(n^2)$在最坏的情况下，配置的数量是组合的。 

结构上的见解是，只有当链形成时，可达性才会变大。 如果图不包含长度大于 1 的有向路径，则其可达性关系正是其边集。 当每个节点最多出现一条边作为源并且最多出现一条边作为目的地时，就会发生这种情况，而且重要的是，没有顶点同时用作两条边之间的中间桥梁。 换句话说，我们将每个系统保持为匹配，因此传递闭包不会扩展任何内容。 

一旦两个系统都匹配，条件就会大大简化：当且仅当一对系统直接是一条边时，它才是可达的。 约束“在两个系统中都无法到达任何对”变成“两个系统之间没有共享有向边”。 

这减少了将尽可能多的不相交有向边打包到线路上的两个匹配中的任务。 由于每个匹配最多可以使用每个顶点一次作为起点，最多一次作为终点，因此每个匹配最多可以包含$\lfloor n/2 \rfloor$边，并且我们可以以交错模式同时构造两者，以便它们一起使用几乎所有相邻对而不会发生冲突。 

清洁施工达到总量$n-1$边缘。

 | --- | --- | --- | --- |
 | 暴力破解图 + 可达性 | 指数 /$O(n^3)$|$O(n^2)$| 太慢了 |
 | 交错匹配结构 |$O(n)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 我们在路径上建立两个定向匹配$1, 2, \dots, n$，小心交错，以免重复使用定向对。 

1. 通过配对从 1 开始的连续节点来构造青蛙边：$1 \to 2, 3 \to 4, 5 \to 6,$等等。 

这确保了青蛙图是一组独立的边，路径不超过一步，因此可达性与边集相同。 
2. 通过将配对移动一位来构建鸟类边缘：$2 \to 3, 4 \to 5, 6 \to 7,$等等。 

这也形成了匹配，再次防止了任何多步可达性。 
3. 当用完有效对时停止每个系统。 如果$n$为奇数时，最后一个节点在一个或两个系统中保持未使用状态，这是由于匹配约束而不可避免的。 
4. 先输出所有青蛙边缘，然后输出所有鸟边缘。 

该结构保证了每条边在系统中都是唯一的，因为青蛙边总是连接奇偶对，而鸟边连接移位一位的奇偶对。 这种结构偏移可以防止重叠。 

为什么它有效可以归结为可达性崩溃。 由于两个图都是匹配的，因此没有顶点充当连接两条边的交汇点，因此每个可达对恰好是一条边。 因此，发生禁止条件的唯一方法是两个系统都包含相同的有向对。 交替结构完全防止了这种情况，同时最大限度地增加了每个系统可以容纳的边缘数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    frog = []
    bird = []
    
    # frog: (1,2), (3,4), ...
    for i in range(1, n, 2):
        if i + 1 <= n:
            frog.append((i, i + 1))
    
    # bird: (2,3), (4,5), ...
    for i in range(2, n, 2):
        if i + 1 <= n:
            bird.append((i, i + 1))
    
    m = len(frog) + len(bird)
    print(m)
    
    for a, b in frog:
        print(a, b)
    
    for a, b in bird:
        print(a, b)

if __name__ == "__main__":
    solve()
```青蛙边缘是通过逐步执行奇数索引来生成的，确保每条边缘都从奇数位置开始并在下一个偶数位置结束。 鸟的边缘是对称生成的，但从偶数索引开始偏移。 这保证了两个列表中都不会出现有向对。 

一个常见的陷阱是试图在不控制结构的情况下最大化边缘，这会意外地创建像这样的链$1 \to 2 \to 3$并导致大量的传递闭包。 在这里，我们通过确保每个节点最多参与每个图的一条边来明确避免这种情况，从而消除所有多步骤可达性。 

## 工作示例

 ### 示例 1

 让$n = 6$。 

青蛙边缘为：$1 \to 2, 3 \to 4, 5 \to 6$鸟的边缘是：$2 \to 3, 4 \to 5$| 步骤| 青蛙边缘| 鸟的边缘| 总计 |
 | --- | --- | --- | --- |
 | 1 | (1,2) | - | 1 |
 | 2 | (1,2),(3,4) | - | 2 |
 | 3 | (1,2),(3,4),(5,6) | - | 3 |
 | 4 | 相同| (2,3) | 4 |
 | 5 | 相同| (2,3),(4,5) | 5 |

 这表明交替图案填充了几乎所有相邻对而没有重叠。 

观察到的不变量是青蛙边缘总是从奇数索引开始，而鸟边缘总是从偶数索引开始，因此没有边缘重合。 

### 示例 2

 让$n = 5$。 

青蛙边缘：$1 \to 2, 3 \to 4$鸟的边缘：$2 \to 3, 4 \to 5$| 步骤| 青蛙| 鸟 | 总计 |
 | --- | --- | --- | --- |
 | 1 | (1,2) | - | 1 |
 | 2 | (1,2),(3,4) | - | 2 |
 | 3 | 相同| (2,3) | 3 |
 | 4 | 相同| (2,3),(4,5) | 4 |

 同样，不会发生重叠，并且所有边都是直接的，因此可达性不会超出单个边。

 ## 边缘情况

 对于$n = 2$，该结构产生单个青蛙边缘$1 \to 2$并且没有鸟的边缘。 不存在重叠的可能性，因为只有一对存在。 

为了$n = 3$, 青蛙得到$1 \to 2$, 鸟得到$2 \to 3$。 这些边缘之外不存在传递可达性，因此不会出现隐藏的重叠。 

对于较大的奇数$n$，最后一个节点在一个或两个系统中均未使用。 由于匹配约束，这是不可避免的，算法自然地将其隔离，而不影响正确性。
