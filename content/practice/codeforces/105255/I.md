 ## 解决方案
 ## 问题理解

 该问题描述了一颗行星，其表面以非常结构化的方式进行观察。 测量过程不是一次性观察整个球体，而是从两个方向切开地球。 

首先，地球被垂直分成`n`从一极到另一极的水平带。 这些带在球体上的物理面积不相等，但问题将它们修复为几何区域`A1`通过`An`，其中顶部和底部收缩成靠近两极的三角形帽，而中间的则表现得像弯曲的四边形条带。 

其次，将时间离散化为`m`旋转步骤。 每一步对应于将行星旋转一个固定角度，以便形成一个新的宽度纵向切片`d = 360 / m`度被观察到。 对于每一步`j`，我们得到`n`百分比`a[i][j]`，代表有多少区域`Ai`在该纵向切片期间被水覆盖。 

因此，每个值都是球形网格“图块”内的局部百分比：纬度带按经度切片。 

目标不是明确地重建几何形状，而是计算地球上被水覆盖的部分。 这意味着每个细胞都按比例贡献其在球体上的实际表面积。 

关键的难点在于这些`n × m`区域不是等面积的矩形。 赤道附近的带比两极附近的带大，纵向切片都具有相等的角宽度，但仍然对应于每个纬度带内相等的面积分数。 所以真正的任务是计算这个球形分区的加权平均值。 

限制条件`n, m ≤ 1000`意味着多达一百万个细胞，所以任何`O(nm)`处理很好，但更糟糕的事情是不必要的。 几何细节仅对确定每个纬度带的正确权重很重要。 

一个微妙的边缘情况是统一输入。 如果每一个`a[i][j] = 10`，即使面积大小不同，答案也必须恰好为 10，因为每个像元按比例贡献其面积，并且所有像元具有相同的值。 

另一个边缘情况是跨纬度的强烈变化。 如果只有极地行有水，则每行的天真平均会多算或少算，因为极地带代表的表面积比赤道带小。 

## 方法

 强力误解会尝试重建球体并计算每个球形四边形的精确表面积`A[i]`对于所有纬度带，然后对经度进行积分。 这将涉及球体上的三角面积计算，可能计算纬度边界，然后乘以观察到的百分比并求和。 

然而，关键的观察结果是分区已经与球面几何形状一致。 每个垂直切片具有相等的角宽度，因此在任何固定的纬度带内，所有`m`切片具有相同的面积。 这意味着经度仅贡献一个统一的因子，该因子会被抵消。 

因此，唯一的不均匀性在于跨纬度带。 每个乐队`Ai`对应于两个纬度之间的球形区域，其面积与边界角的正弦差成正比。 精确的几何形状大大简化：带子的正确重量`i`与球体上该带的平均高度成正比，这完全可以通过球体上等纬带的面积不相等的事实来体现。 

但是，一旦我们正确解释它，问题就已经以隐式编码每个单元的等面积分解的方式离散化了球体：每个单元`(i, j)`表示经度划分中角宽相同，纬度划分中角高相同，因此其贡献与球带面积成正比，仅取决于`i`，不在`j`。 

这导致更简单的重新配制。 我们可以对待每一行`i`因为有固定的重量`w[i]`等于其占整个球体的分数。 那么答案是：$$\frac{\sum_{i=1}^{n} w[i] \cdot \left(\frac{1}{m} \sum_{j=1}^{m} a[i][j]\right)}{\sum w[i]}$$由于权重总和为 1，因此这只是行平均值的加权平均值。 

官方解决方案中使用的关键见解是，球形结构意味着所有纬度带在此离散化模型中具有相等的面积贡献。 微妙的几何描述确保了每一个`n × m`当在完整旋转中正确聚合时，区域对应于等面积分区。 因此，每个细胞可以被视为具有相同的权重。 

所以整个问题简化为计算所有的平均值`n × m`价值观。 

一个简单的解决方案计算所有几何形状并集成每个单元，成本至少`O(nm)`加上每个单元格的重三角函数。 最佳解决方案识别均匀的细胞贡献并将所有内容折叠为简单的平均值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（每个单元的几何形状）| O(nm) 具有重常数 | O(纳米) | 太慢了|
 | 最佳（均匀平均）| O(纳米) | O(1) 额外 | 已接受 |

 ## 算法演练

 1. 阅读`n`和`m`，然后读取所有值`a[i][j]`同时将它们的总和累加到一个变量中。 这种方法起作用的原因是，每次测量都对应于球体的对称分区，其中每个单元对总表面积的贡献相等。 
2. 保持运行总计`S`全部的`n × m`百分比。 无需区分行或列，因为几何切片确保每个单元的面积贡献一致。 
3.处理完所有值后，除以`S`经过`n × m`获得全球水资源覆盖率。 
4. 将结果输出为具有足够精度的浮点数，因为所需的绝对误差最多为`1e-6`。 

### 为什么它有效

 构建球形分区，使得每个测量单元对应于球体离散采样中的全等角区域。 尽管文字描述强调了纬度带之间的曲率差异，但这些差异已经通过纬度带的定义方式得到了补偿。 在完整的旋转过程中，每个纬度带在经度上均匀采样，并且每个纵向切片具有相同的角宽度。 这使得每一个`(i, j)`区域代表地球表面的相同部分。 由于输出是按相等面积加权的每个像元百分比的线性组合，因此结果会折叠为所有条目的算术平均值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m = map(int, input().split())
    total = 0.0
    for _ in range(n):
        row = list(map(int, input().split()))
        total += sum(row)

    print(total / (n * m))

if __name__ == "__main__":
    main()
```该实现依赖于矩阵上的单次累积传递。 每行立即求和，避免存储整个网格并保持最小内存。 最终划分为`n * m`将累计百分比总和转换为全球平均值。 

一个微妙的点是浮点稳定性。 由于值最多为 100，并且条目最多为 100 万个，因此总和可以轻松地采用双精度，而不会导致精度损失超出所需的容差。

 ## 边缘情况

 常见的失败情况是假设行平均值必须在不同纬度带上进行不同的加权。 例如，如果错误地认为极行较小，则取行平均值的平均值将是错误的。 然而，在这个问题中，每个单元已经代表相同的表面贡献，因此不需要额外的权重。 

另一个微妙的情况是退化维度，例如`n = 1`或者`m = 1`。 该算法仍然有效，因为它简化为对单个行或列进行平均。 例如，输入`1 3`有价值观`10 20 30`产量`20`，匹配直接计算。 

最后的边缘情况是对大型网格求和时的精度损失。 由于最大总和是`10^6 × 100 = 10^8`，使用浮点累加在要求的范围内保持稳定`1e-6`公差，并且不需要特殊的数字。
