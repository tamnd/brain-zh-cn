 ## 解决方案
 ## 问题理解

 我们有六个骰子，每个骰子在掷骰后在其顶面上显示一个符号，但在内部每个骰子有六个可能的符号，所有符号的可能性都是相等的。 除此之外，我们还得到了一本允许单词的字典，每个单词每个骰子只有一个字母，因此一个单词对应于所有骰子位置的符号的完整分配。 

单次游戏包括滚动所有骰子，查看生成的符号元组，然后决定保留哪些骰子以及重新滚动哪些骰子。 保持骰子保留其当前的面，而重新掷骰子则独立地重新采样其六个面之一。 这种情况一直持续到掷骰后的某个时刻，所有骰子上的可见符号与字典中的一个单词完全匹配。 

目标是最小化单个骰子的预期总数，而不是轮数。 每掷出一个骰子，就会贡献一个单位的成本。 

关键的困难在于，我们可以根据当前对任何单词的部分进度自适应地决定重掷哪些骰子。 这将问题转变为每个单词的状态空间非常小的受控随机过程，但具有大量候选目标单词。 

这些限制以非常具体的方式发挥作用。 骰子的数量最多为六个，这使得任何指数 in-d 方法都可行。 字数可能高达 200,000 个，因此任何在字长上以线性时间处理每个字或对每个字执行大量模拟的解决方案都会太慢。 对于每个单词的每个骰子子集执行恒定因子工作的解决方案是可以接受的，因为只有$2^6 = 64$子集。 

一种简单的方法会动态模拟骰子值的所有可能配置的最佳策略，但状态空间是$26^d$从字母表的意义上来说，而且太大了。 即使在不利用少量骰子的情况下独立模拟每个单词的所有策略也会超时。 

当由于某些骰子从未包含所需符号而根本无法形成单词时，就会出现微妙的边缘情况。 例如，如果骰子有面`GHI234`，它永远不能产生`A`，所以任何需要的词`A`无论策略如何，处于这个位置都是不可能的。 在这种情况下，必须完全忽略该词。 

## 方法

 一个蛮力的想法是将这个过程视为一个完整的马尔可夫决策过程，针对所有可能的骰子面配置以及我们可能选择冻结的所有骰子子集。 在任何配置中，我们都会枚举要重新滚动的骰子的所有选择，并递归地计算期望值。 配置的数量与骰子值的数量呈指数关系，并且每个转换分支为许多重掷结果。 即使与$d \le 6$，全面分配的分支使得这是不可行的。 

关键的简化来自于将注意力集中在单个候选词上。 如果我们承诺一个词，唯一相关的问题就是我们能以多快的速度将当前骰子状态转换为确切的目标配置。 一旦骰子显示出目标单词的正确符号，就没有理由再次重新掷骰子，因为重新掷骰子只能延迟完成。 这意味着该过程变得单调，因为正确匹配的位置集只会增长。 

对于固定字，我们可以使用子集 DP 来建模该过程，其中状态表示哪些骰子已经锁定到其正确的符号。 来自一个州$S$, 所有骰子都在$S$被冻结，剩余的骰子同时重新掷骰子。 每个重新滚动的骰子都有一个独立的$1/6$在那一轮中成为正确的机会，因此下一个状态是通过独立地将成功的骰子添加到$S$。 

这将问题转化为每个单词最多 64 个状态的小概率 DP。 每个状态的预期成本取决于该步骤中掷骰子的数量加上最终状态分布的预期值。 

最后，我们计算每个单词的期望值，并取所有有效单词的最小值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 完整状态 MDP 配置 面孔呈指数增长 | 巨大 | 太慢了 |
 | 骰子状态上的每个字子集 DP |$O(w \cdot 2^d)$|$O(2^d)$| 已接受 |

 ## 算法演练

 修正一个单词并假设它是我们想要达到的目标配置。 

1. 首先，通过验证对于每个位置，该位置的骰子在其表面的某处包含所需的符号来检查可行性。 如果任何位置未通过此测试，则该单词是不可能的并且可以跳过。 
2. 通过位掩码定义DP状态$S$，其中位$i$表示死亡$i$目前已经显示了目标单词的正确符号，一旦出现我们将永远保留它。 
3. 对于一个州$S$， 让$U$是一组骰子不在$S$。 在一轮中，我们将每个骰子投入$U$一次。 每个这样的骰子都以概率独立地匹配其所需的符号$1/6$。 
4. 本轮费用为$|U|$，因为每个人都死在$U$恰好滚动一次。 
5. 对于每个子集$T \subseteq U$，骰子恰好出现在$T$本轮正确的是$$\prod_{i \in T} \frac{1}{6} \cdot \prod_{i \in U \setminus T} \frac{5}{6}.$$下一个状态变成$S \cup T$。 
6. 写一个递归式：$$E[S] = |U| + \sum_{T \subseteq U} P(T) \cdot E[S \cup T].$$7. 按子集大小的降序计算 DP 值，使得所有超集$S$计算时已经知道$E[S]$。 基本情况是$E[\text{all bits}] = 0$。 
8.这个词的答案是$E[\emptyset]$。 对所有单词重复并取最小值。 

该 DP 有效的原因是该过程仅取决于哪些骰子已经正确，而不取决于它们如何变得正确。 一旦骰子与其目标符号相匹配，冻结它就不会恶化未来的预期，因为它消除了随机性来源而不影响最终的成功条件。 这保证了仅由锁定位置集定义的状态形成固定目标词的最优策略的完整马尔可夫表示。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    d, w = map(int, input().split())
    dice = [input().strip() for _ in range(d)]
    
    # precompute face sets
    face_set = [set(s) for s in dice]
    
    words = []
    for _ in range(w):
        words.append(input().strip())
    
    INF = 1e100
    ans = INF
    
    # iterate words
    for word in words:
        ok = True
        for i in range(d):
            if word[i] not in face_set[i]:
                ok = False
                break
        if not ok:
            continue
        
        size = 1 << d
        dp = [0.0] * size
        
        # process subsets in reverse by number of bits set
        for mask in range(size - 1, -1, -1):
            if mask == size - 1:
                dp[mask] = 0.0
                continue
            
            U = []
            for i in range(d):
                if not (mask >> i) & 1:
                    U.append(i)
            
            k = len(U)
            cost = k
            
            # enumerate all subsets of U
            exp_val = cost
            
            # iterate subsets via bitmask
            for sub in range(1 << k):
                prob = 1.0
                new_mask = mask
                for j in range(k):
                    i = U[j]
                    if (sub >> j) & 1:
                        prob *= (1.0 / 6.0)
                        new_mask |= (1 << i)
                    else:
                        prob *= (5.0 / 6.0)
                
                exp_val += prob * dp[new_mask]
            
            dp[mask] = exp_val
        
        ans = min(ans, dp[0])
    
    if ans == INF:
        print("impossible")
    else:
        print(f"{ans:.10f}")

if __name__ == "__main__":
    solve()
```该实现直接在骰子子集上镜像 DP。 每个掩码代表哪些骰子已经与目标单词匹配。 对于每个状态，代码枚举仍然不匹配的骰子的所有子集来计算转换分布。 每个状态增加的成本正是该步骤中掷骰子的数量。 

从完整掩码向下的排序确保所有转换都进入具有更多已计算的位集的状态。 这避免了递归并保持计算严格自下而上。 

一个微妙的点是浮点稳定性。 概率涉及最多六项的乘积$1/6$或者$5/6$，所以双精度就足够了。 

## 工作示例

 ### 示例 1

 我们只追踪单个单词的前几个 DP 状态，例如`PEACE`有五个骰子。 

令骰子计数为 5，并假设可行性成立。 

| 国家面具| 掷骰子| 成本| 主要过渡思路|
 | ---| ---| ---| ---|
 | 00000 | 5 个骰子 | 5 | 新正确匹配的所有子集 |
 | 00001 | 4 个骰子 | 4 | 所需卷数更少 |
 | 11111 | 11111 0 个骰子 | 0 | 终端|

 从初始状态开始，每一轮都会滚动所有五个骰子，直到某个子集与目标字母匹配。 随着越来越多的骰子锁定，每轮需要的骰子数量越来越少，预期值也会相应降低。 DP 准确地捕捉到了这种递减的成本结构。 

这证实了以下不变量：只有锁定的正确位置集对未来才重要。 

### 示例 2

 考虑两个骰子和单词的情况`AB`，其中模具 2 不能产生`A`或者`B`。 

| 检查步骤 | 结果 |
 | ---| ---|
 | 模具 1 包含`A`或者`B`| 是的 |
 | 骰子 2 包含`A`或者`B`| 没有|

 由于骰子上不可能有一个所需的符号，因此该单词会立即被丢弃。 DP 永远不会运行，这可以防止错误地将无法到达的状态视为具有有限期望。 

这说明了为什么在运行子集 DP 之前可行性过滤至关重要。

 ## 边缘情况

 一种关键的边缘情况是当一个单词在结构上是不可能的，因为至少一个骰子从未包含所需的符号。 在这种情况下，如果强制的话，DP 仍然会在数学上产生一个值，但它会对应于一条不存在的成功路径。 可行性检查确保这些词永远不会进入 DP，因此最终的最小值始终取代真正可实现的目标。 

另一个微妙的情况是所有骰子都已经与初始掷骰中的单词匹配。 在该状态下，DP 分配零成本，因为吸收状态$S = \text{all}$已达到，无需进一步滚动。 

最后，多个单词共享重叠结构的情况可以自然地处理，因为每个单词都会产生一个独立的 DP，并且所有单词的最小值可以正确捕获切换策略，而无需动态建模单词切换。
