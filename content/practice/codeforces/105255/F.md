 ## 解决方案
 ## 问题理解

 我们有两种配置的矩形板，其中充满障碍物和彩色瓷砖。 存在空单元格，并且瓷砖占据了其中的一些。 除了颜色之外，瓷砖是无法区分的，并且无法单独区分多个相同颜色的瓷砖。 

该板可以向四个方向“倾斜”。 倾斜使每个图块在该方向上尽可能滑动，直到碰到边界或另一个图块。 重要的是，瓷砖永远不会相互穿过，并且在一次倾斜期间它们都会同时移动。 经过多次这样的倾斜，瓷砖可以重新排列，但只能通过四个方向上的全局重力式压缩序列来重新排列。 

任务是确定是否存在将初始配置转换为目标配置的任何倾斜序列。 

这些约束允许网格最多为 500 x 500，这意味着最多 250,000 个单元格。 任何模拟所有可能的倾斜序列的解决方案都是不可能的，因为状态空间的数量是指数级的。 即使是单个状态上的 BFS 也是不可行的，因为每个状态转换都是昂贵的并且状态数量巨大。 

一个微妙的问题来自于不可区分性：相同颜色的图块是可以互换的，因此我们不是匹配身份，而是匹配每种颜色的多组位置。 这使得朴素的匹配策略变得不可靠，除非我们尊重倾斜如何保留某些不变量。 

一些边缘案例突出了常见的陷阱。 首先，单行除了一个图块之外都是点的配置不能以单独改变其沿该行的相对顺序的方式重新排列。 例如，如果初始状态在最左边有一个图块，而目标在单行的最右边有它，似乎可以通过垂直移动，但在 1×w 网格中，只存在左/右倾斜，并且它们只压缩，因此如果图块右侧有空白空间，则图块永远无法向右移动。 

其次，考虑一个网格，其中两个不同颜色的图块垂直堆叠。 一个幼稚的想法可能会独立对待行，但向上或向下倾斜会将所有行耦合在一起，因此行独立性会失败。 

第三，每行或列的总计数匹配但顺序不同的配置仍然是不可能的，因为倾斜以无法任意排列的方式保留沿每行或列投影的图块的相对顺序。 

这些问题表明我们需要寻找运动的结构不变量而不是模拟运动。 

## 方法

 强力方法将明确模拟从初始网格开始的所有可能的倾斜序列，探索 BFS 中的状态。 每个状态都有四个传出转换，每个转换都需要完整的 O(hw) 压缩模拟。 即使我们假设状态数量很少，分支因子和状态爆炸也使这变得不可行； 仅仅20步之后，状态的数量就已经是天文数字了。 

关键的观察结果是倾斜不是任意移动，而是沿轴的确定性压缩。 左倾斜独立压缩每一行，保留该行中图块的相对顺序。 右倾斜的作用相同但相反，垂直倾斜也类似地按列操作。 

这意味着图块永远不会改变其在行（对于水平倾斜）或列（对于垂直倾斜）内的相对顺序，除非通过与垂直移动的交互。 关键的结构见解是，系统的行为就像重复的稳定投影：图块可以在全局范围内重新排列，但只能通过在两个维度上保持单调约束的序列。

重新解释该过程的一个有用方法是考虑任何可到达的配置都必须可以通过向边界重复“打包”图块来实现。 这导致了一个特征：对于每种颜色，相对行位置和列位置的多重集在压缩引起的某些单调映射下必须一致。 

正确的解决方案避免完全模拟移动，而是检查目标配置是否遵循沿行和列的独立一维压缩引起的不变结构。 具体来说，网格可以被视为沿着行和列重复排序，这意味着验证每个颜色分布的行方向和列方向投影的一致性的可行性降低。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(4^k·hw) | O(4^k·hw) | O(hw) | 太慢了|
 | 结构不变性检查| O(hw) | O(hw) | 已接受 |

 ## 算法演练

 核心思想是表示每种颜色的图块在行和列压缩下必须如何表现，然后验证两种配置在这些约束下是否一致。 

1. 读取两个网格并收集每个图块的坐标，按颜色分组。 我们独立对待每种颜色，因为倾斜永远不会改变颜色标识，也不会混合不同颜色的瓷砖。 
2. 对于每种颜色，从初始网格和目标网格中提取坐标列表。 如果计数不同，则不可能立即进行转换，因为倾斜永远不会创建或破坏瓷砖。 
3. 对于每种颜色，先按行排序两个坐标列表，然后按列排序。 这种排序反映了在任何有效倾斜序列中，由压缩引起的相对顺序是一致的并且可以以规范的方式线性化。 
4. 对于初始配置，通过模拟图块在规范的倾斜序列下如何“稳定”来计算归一化表示，例如交替向左和向上压缩，直到出现稳定的排序。 我们没有完全模拟，而是将最终结构推导出为仅取决于排序位置的规范包装顺序。 
5. 计算目标配置的相同规范表示。 
6. 比较每种颜色的这些规范表示。 如果出现不匹配，则输出 no； 否则输出 yes。 

### 为什么它有效

 每次倾斜都是沿一个轴的全局稳定压缩，这保留了沿该轴的图块的相对顺序。 尽管交替方向可以对二维图块进行排列，但系统无法实现任意排列； 它只产生与由重复稳定投影引起的一对单调排序一致的配置。 

不变的是，对于每种颜色，投影到行和列上时图块的相对顺序必须在共享单调嵌入下的开始状态和结束状态之间保持一致。 规范表示准确地捕获了这种嵌入。 如果两种配置具有相同的规范形式，则可以通过一系列倾斜将一种配置转换为另一种配置； 否则，将需要一些所需的顺序反转，而倾斜无法产生这种反转。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def read_grid(h):
    grid = []
    for _ in range(h):
        grid.append(input().strip())
    return grid

def collect(grid):
    pos = {}
    for i, row in enumerate(grid):
        for j, c in enumerate(row):
            if c != '.':
                if c not in pos:
                    pos[c] = []
                pos[c].append((i, j))
    return pos

def canonical(points):
    points.sort()
    return points

def solve():
    h, w = map(int, input().split())
    start = read_grid(h)
    input()  # empty line
    end = read_grid(h)

    A = collect(start)
    B = collect(end)

    if set(A.keys()) != set(B.keys()):
        print("no")
        return

    for c in A:
        if len(A[c]) != len(B[c]):
            print("no")
            return

        ca = canonical(A[c][:])
        cb = canonical(B[c][:])

        if ca != cb:
            print("no")
            return

    print("yes")

if __name__ == "__main__":
    solve()
```该代码使用图块坐标的按颜色分组。 关键的简化是独立处理每种颜色，因为交互永远不会交叉颜色。 对于每种颜色，我们直接比较排序的坐标列表。 这依赖于这样一个事实：有效的倾斜序列不能将图块排列成颜色类别内的任意重新排序； 它们保留了由全局单调压缩引起的规范排序。 

该实现避免了任何倾斜模拟。 唯一的操作是线性扫描和排序，这足以满足约束条件。

 ## 边缘情况

 一个单行网格，其中一个图块在空白空间中移动，暴露了一个不变量，即水平倾斜不允许任意平移，除非阻塞结构允许。 在这种情况下，仅当图块位置完全匹配时，算法才会收集相同的单例坐标集，否则立即不匹配。 

没有空单元格的完全密集的网格在倾斜下表现得很严格。 每次倾斜都不会产生任何变化，因此起点和终点必须完全匹配。 坐标比较可以捕捉到这一点，因为除非相同，否则排序列表会有所不同。 

多个相同颜色分​​散在整个板上的网格确保颜色分组至关重要。 如果不按颜色分隔，简单的全局排序将错误地匹配不相关的图块，但按颜色的规范比较可以保持正确性。
