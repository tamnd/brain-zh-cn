 ## 解决方案
 ## 问题理解

 我们有一个加权网格，其中的每个单元格$n \times m$board 包含代表“收益率”的正值。 任务是在该网格上放置三个完全相同的矩形图。 每个图必须覆盖一个连续的轴对齐矩形，并且每个图都有固定的尺寸$a \times b$，但它可以旋转，所以任一方向$a \times b$或者$b \times a$每个地块独立允许。 

每个图贡献其覆盖的所有单元格值的总和，并且单元格中的图不允许重叠。 目标是选择三个位置以使收集到的总金额最大化。 

网格大小达到$1000 \times 1000$，因此任何直接尝试所有展示位置的解决方案都会立即变得太慢。 即使枚举所有可能的位置也已经达到了大约$10^6$，并在其中选择三元组将爆炸为$10^{18}$的组合，远远超出了可行的限度。 这迫使我们有效地预先计算矩形值，然后仔细构建选择过程，以便重叠约束不需要显式检查所有对或三元组。 

当矩形至少在一个方向上不适合时，就会出现一种微妙的边缘情况。 例如，如果$a > n$和$b > n$同时对于某些维度（对于列也类似），则不存在放置并且答案为零。 另一种边缘情况是，即使存在矩形，也只能放置一两个位置，例如非常小的网格，例如$3 \times 3$与一个$2 \times 2$长方形。 正确的输出仍然为零，因为无法形成三个不重叠的位置。 

天真的贪婪方法也很容易失败。 例如，如果网格有两个非常靠近的高值区域，则首先贪婪地选择最佳矩形可能会阻止唯一允许两个额外大矩形的配置，即使稍差的第一选择会导致更好的全局总和。 

## 方法

 蛮力的想法很简单：枚举矩形的每个有效位置，计算其总和，然后尝试不重叠的每个三元组位置。 这是正确的，因为它会检查所有配置，但这是完全不可行的。 安置数量的顺序是$O(nm)$，并且检查三元组导致$O(n^3 m^3)$行为最糟糕的解释，这远远超出了任何实际限制。 

关键的观察是，困难不在于计算矩形总和，而在于强制三个选定矩形之间不重叠。 一旦知道矩形和，剩下的任务就变成选择三个不相交的高权重矩形。 

一个关键的结构特性简化了问题：任何两个不重叠的轴对齐矩形都可以由垂直线或水平线分隔。 这是因为，如果两个矩形不相交，则它们在 x 轴上的投影不重叠，或者它们在 y 轴上的投影不重叠。 这一事实使我们能够使用网格分区来推理独立性。 

我们将这个想法扩展到三个矩形。 三个不重叠的矩形的任何最佳配置可以通过将网格切割成最多三个垂直条带或三个水平条带使得每个矩形完全位于一个条带内来表示。 这将问题简化为考虑基于分区的 DP 而不是任意空间交互。 

一旦网格被划分，每个条带就变得独立，我们只需要知道一个区域内的最佳矩形（或最佳矩形组合）。 这促使预先计算每个的总和$a \times b$和$b \times a$矩形并在这些值上构建 2D 前缀结构，以便我们可以有效地查询任何子网格内的最佳矩形。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有三元组 |$O((nm)^3)$|$O(nm)$| 太慢了|
 | 前缀和+分区DP|$O(nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 我们分别处理矩形的两个方向，并将它们的贡献计算为统一的结构。 

## 算法演练

 1. 计算网格的二维前缀和，以便可以在恒定时间内计算任何矩形和。 这是必要的，因为我们将查询数百万个候选矩形。 
2. 枚举矩形在两个方向上的所有有效位置。 对于每个左上角位置，使用前缀和计算其总和。 将这些总和存储在两个辅助网格中，每个网格对应一个方向。 
3. 在这些网格上构建 2D 前缀最大表。 这使我们能够在恒定时间内回答“完全包含在给定左上角位置子网格中的最佳矩形是什么”形式的查询。 
4. 要处理两个不重叠的矩形，请使用不相交的矩形可以用垂直线或水平线分隔的结构属性。 对于列处的垂直分割$c$，第一个矩形必须完全位于列中$\le c$，第二个完全在列中$> c$。 我们通过尝试所有分割并组合左右区域的前缀最大值来计算最佳对。 我们对水平分割重复相同的逻辑。 
5. 将相同的划分思想扩展到三个矩形。 我们尝试放置两个垂直切口或两个水平切口，将网格分成三个独立的区域。 对于每个区域，我们使用预先计算的最大结构查询最佳矩形并组合结果。 
6. 取所有垂直和水平三重分区中的最大值。 如果不存在有效放置，则结果保持为零。 

基本思想是，我们不考虑三个相互作用的形状，而是将问题转化为扫描全局分隔符，从而完全消除重叠约束。 

### 为什么它有效

 正确性来自于以下事实：任何最佳的非重叠矩形集都可以按其投影的 x 坐标或 y 坐标进行排序。 这些排序中的至少一种允许将矩形一致地单调分离成连续的条带。 一旦存在这样的排序，在连续矩形之间的边界处放置切口就保留了可行性，并且每个矩形都被限制在不同的区域内。 这保证了搜索所有可能的切割位置不会错过最佳配置。

def build_prefix(a):
    n, m = len(a), len(a[0])
    ps = [[0]*(m+1) for _ in range(n+1)]
    for i in range(n):
        row = 0
        for j in range(m):
            row += a[i][j]
            ps[i+1][j+1] = ps[i][j+1] + row
    return ps

def rect_sum(ps, x1, y1, x2, y2):
    return ps[x2][y2] - ps[x1][y2] - ps[x2][y1] + ps[x1][y1]

def build_best(grid, h, w):
    n, m = len(grid), len(grid[0])
    ps = build_prefix(grid)

    return best, pref

在完整的解决方案中，剩下的步骤是迭代一个或两个分隔符的所有可能的垂直和水平剪切位置，并组合最多三个区域查询。 重要的实现细节是，区域中的矩形有效性仅取决于左上角的可行范围，而不是完整的几何形状，这就是为什么放置网格上的前缀最大值就足够了。 

一个常见的陷阱是忘记调整边界：高度的矩形$h$从行开始$i$必须满足$i+h \le n$，因此左上角位置的有效区域缩小了$h-1$。 色谱柱也需要类似的护理。

 ## 边缘情况

 一个关键的边缘情况是矩形至少在一个维度上太大。 在这种情况下，放置网格将变为空，并且必须忽略每个方向。 该算法通过跳过无效的来处理这个问题$(h, w)$早配对。 

另一种情况是，即使存在许多单个放置，也只能容纳两个矩形。 分区逻辑自然会防止不正确的过度计数，因为任何有效的配置都必须将每个矩形分配给不同的区域，并且如果少于三个区域包含可行的放置，则全局最大值保持为零。 

最后一个微妙的情况是，最佳矩形在密集区域中严重重叠。 前缀最大方法确保一旦选择了一个区域，我们只考虑完全不相交的左上角可行区域，防止由于忽略矩形范围而导致意外重叠。
