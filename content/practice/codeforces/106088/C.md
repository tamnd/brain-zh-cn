 ## 解决方案
 ## 问题理解

 给定两个长度为 n 的序列，代表写在两个手镯上的数字。 目标是使两个多组数字相同，这意味着重新排序后，两个手链包含完全相同的值和相同的重数。 

我们可以对任一手镯进行单一操作：选择两个不同的值 a 和 b，并同时将所有出现的 a 与 b 交换。 此操作在手环上是全局的，因此一步操作中每个 a 都会变成 b，每个 b 都会变成 a。 我们最多可以执行一次这个操作，我们想知道是否有可能使两个多重集相等，或者它们是否已经相等，或者是否不可能。 

关键的观察结果是该操作不会改变频率，它只会排列一个数组中值的标签。 因此，我们不是在编辑单个元素，而是在全局重命名两个值。 

输入大小 n 高达 100000，因此任何尝试所有交换或模拟每对值的转换的解决方案都太慢。 对值或频率对的二次方法是不可行的。 我们必须将问题简化为频率比较和最多恒定数量的候选变换。 

当存在多个不匹配的值时，会出现微妙的失败情况。 例如，如果 Alice 有 (1,1,2) 而 Bob 有 (2,2,1)，则即使位置不同，它们作为多重集已经相等。 另一种情况是，当存在三个或更多不同的不匹配值时，单个交换无法修复两个以上的标签，即使总数表面上匹配，也无法得到答案。 

## 方法

 如果我们忽略对操作的约束，自然的强力想法是尝试一个手镯中每一个可能的值对（a，b），应用交换，重新计算多重集，并检查与另一个手镯的相等性。 构造频率图进行相等性检查 O(U)，其中 U 是不同值的数量，但尝试所有对会得到 O(U²·U)，当 U 可达 n 时，该值太大了。 

瓶颈是操作一次只影响两个值，但数组之间的不匹配可能涉及很多值。 关键的结构见解是交换 a 和 b 仅交换它们的频率，因此我们有效地排列了一个数组的频率表中的条目。 这意味着唯一重要的是我们是否可以通过不执行任何操作或在一侧恰好交换两个标签来匹配频率多重集。 

这将问题简化为比较频率字典并检查差异是否可以通过至多一个标签换位来解决。 

我们计算两个阵列的频率图。 如果它们已经相同，则答案为 0。否则，我们将查看频率不同的所有值。 如果有两个以上这样的值，则单个交换无法修复两个以上的标签，因此答案为 -1。 

如果恰好有两个值 x 和 y 存在差异，我们检查交换一个数组中的 x 和 y 是否会使频率完全匹配。 这种交换有效地交换了它们的计数，因此我们模拟了对频率差异的影响。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有价值对的暴力交换 | O(U3)| O(U) | 太慢了|
 | 最多一次交换模拟的频率比较 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们依赖频率图，因为操作从不依赖于顺序，只依赖于计数。

1. 为两个手环建立频率字典fx和fy。 
2. 如果fx等于fy，则立即输出0。 这捕获了仅存在排列差异的情况。 
3. 计算所有值 v 的集合，其中 fx[v] != fy[v]。 这些是唯一受任何可能的修正影响的值。 
4. 如果此类值的数量超过 2，则输出 -1，因为一次交换只能影响两个标签，并且无法修复更复杂的不匹配。 
5. 如果 x 和 y 恰好有 2 个值，则测试交换 fx 中的 x 和 y 是否与 fy 对齐。 这意味着检查 fx[x] 是否变为 fx[y] 且 fx[y] 是否变为 fx[x]，而所有其他频率保持不变。 
6. 如果条件成立，则输出1，否则输出-1。 
7. 如果在步骤2之后没有不匹配的值，我们已经返回0，因此不需要进一步的情况。 

重要的部分是步骤 5。交换操作不会移动元素，而是重命名它们。 因此，唯一可能的纠正是完全交换两个频率桶。 如果不匹配不能表示为单个标签交换，则不存在一个操作的序列。 

### 为什么它有效

 该操作会导致值空间上的排列，即恒等或转置。 由于每次操作仅在全局范围内交换两个标签，因此在一次移动后，一个手环的任何可到达配置都与交换其直方图中的两个频率条目完全对应。 因此，问题简化为检查两个频率多重集是否因值域中的转置而不同或根本没有不同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    from collections import Counter
    fa = Counter(a)
    fb = Counter(b)

    if fa == fb:
        print(0)
        return

    keys = set(fa.keys()) | set(fb.keys())
    diff = []
    for k in keys:
        if fa[k] != fb[k]:
            diff.append(k)

    if len(diff) > 2:
        print(-1)
        return

    if len(diff) == 0:
        print(0)
        return

    if len(diff) == 2:
        x, y = diff

        def check():
            fx = fa.copy()
            fx[x], fx[y] = fx[y], fx[x]
            return fx == fb

        if check():
            print(1)
        else:
            print(-1)
        return

    print(-1)

if __name__ == "__main__":
    solve()
```实现的重点是对发生次数进行计数。 一开始的平等检查避免了不必要的工作。 diff 提取步骤准确地隔离了两个直方图不同的标签，这直接限制了搜索空间。 

仅当恰好存在两个有问题的值时才进行交换模拟，因为任何其他情况都无法在一次操作中修复。 复制计数器是安全的，因为不同值的数量受 n 限制，并且只复制一份。

 | 2 | 1 | 2 |

 不匹配值为 {2, 4}。 

我们模拟交换 fa 中的 2 和 4：

 | 步骤| 发[2] | 发[4] | 结果|
 | --- | --- | --- | --- |
 | 原创| 1 | 2 | 不匹配|
 | 交换 | 2 | 1 | 匹配 Facebook |

 由于一次交换后相等成立，因此输出为 1。 

这证实了单个标签转置可以修复这种不平衡。

```频率表：

 | 价值| 发| 脸书 |
 | --- | --- | --- |
 | 4 | 1 | 2 |

交换 fa 中的 4 和 5 得出：

 | 价值| 交换后的fa | 脸书 |

import sys, io
from collections import Counter

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    fa = Counter(a)
    fb = Counter(b)

    if fa == fb:
        return "0"

    keys = set(fa) | set(fb)
    diff = [k for k in keys if fa[k] != fb[k]]

    if len(diff) > 2:
        return "-1"

    if len(diff) == 0:
        return "0"

    if len(diff) == 2:
        x, y = diff
        fx = fa.copy()
        fx[x], fx[y] = fx[y], fx[x]
        return "1" if fx == fb else "-1"

    return "-1"

 ## 边缘情况

 一种微妙的情况是数组作为多重集已经相等，但顺序不同。 对于像这样的输入`[2, 1, 2]`和`[1, 2, 2]`，频率图相同，算法立即返回 0。 比较位置而不是频率的简单方法会错误地尝试不必要的操作。 

另一种情况是当两个值的频率恰好不同但交换不能解析总数时，如`[1,2,3,4,5]`相对`[1,2,3,4,4]`。 该算法正确识别了两个不匹配的值，但在模拟后拒绝，因为交换标签无法改变聚合多重性不平衡。 

第三种边缘情况是只有一个不匹配的值。 在有效的频率守恒下，这种情况不会发生，除非两个数组的该值的总分布不一致。 该算法正确输出 -1，因为单个标签无法在不影响至少两个值的情况下转换为另一个标签或从另一个标签转换而来，因此仅一个不匹配就表明结构上的不可能性。
