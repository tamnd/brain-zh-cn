 ## 解决方案
 ## 问题理解

 我们给出了一个大小为的矩形网格$n \times m$。 从这个网格中，我们选择一个与网格线对齐的子矩形，这意味着我们选择一个连续的行和列块。 矩形仅由其尺寸、高度和宽度定义，因为其位置并不重要，并且仅当两个矩形尺寸不同时才认为它们相同。 

对于每个选定的大小的矩形$a \times b$，我们概念性地绘制了在其内部形成网格线的所有单元段，包括其边界。 相邻网格点之间的每个单位段的绘制时间为一秒。 我们有时间限制$t$，我们需要计算至少需要多少个不同的矩形尺寸$t$要绘制的线段。 

关键输出不是网格内的放置数量，而是有效维度对的数量$(a, b)$使得该矩形的绘制成本至少为$t$，有约束条件$1 \le a \le n$,$1 \le b \le m$。 

一个中的段数$a \times b$矩形来自网格几何。 有$(a+1)b$垂直段和$(b+1)a$水平段，因此总数为：$$(a+1)b + (b+1)a = 2ab + a + b$$我们必须数对$(a, b)$满意：$$2ab + a + b \ge t$$限制因素很重要：$n \le 10^3$， 但$m, t \le 10^9$。 这立即表明我们无法迭代所有$a, b$配对$O(nm)$，因为这将取决于$10^{12}$运营。 甚至$O(n \log m)$是可以接受的，因为$n$很小，我们可以利用单调性$b$。 

边缘情况出现时$t$非常小或非常大。 

如果$t = 1$，每个矩形都是有效的，因为即使是$1 \times 1$矩形有$2\cdot1\cdot1 + 1 + 1 = 4$段。 

如果$t$非常大，例如$t = 10^9$，那么连最大矩形$n \times m$可能不满足条件，因此答案为零。 

一个常见的错误是忽略边界段并仅使用$2ab$，这低估了成本并导致不正确的比较，特别是对于小矩形。 

## 方法

 天真的想法很简单：枚举每个矩形的大小$a \times b$, 计算$2ab + a + b$，并检查是否至少是$t$。 这是正确的，因为该公式直接测量单位段的数量。 然而，这需要检查$n \times m$对，在最坏的情况下是$10^3 \times 10^9$，完全不可行。 

不平等的结构改变了一切。 定影$a$，表达式$2ab + a + b$线性增长$b$。 这意味着对于固定高度$a$，有一个阈值$b$之后所有较大的宽度都有效。 这种单调行为使我们能够求解最小有效值$b$并计算还有多少个选择$O(1)$。 

重新排列不等式：$$2ab + a + b \ge t$$

$$b(2a + 1) \ge t - a$$对于每个$a$，我们可以计算最小整数$b$满足：$$b \ge \left\lceil \frac{t - a}{2a + 1} \right\rceil$$然后全部$b$从该值到$m$只要下限在范围内，都是有效的。 

这将二维计数问题转化为一维扫描问题$a$，每步具有恒定时间算术。

 | 蛮力 |$O(nm)$|$O(1)$| 太慢了|
 | 最佳|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们处理每个可能的高度$a$独立地计算每个宽度的宽度$b$是有效的。 

1. 固定一个值$a$从 1 到$n$。 这隔离了一个维度，因此我们可以推断单调性$b$。 
2. 重写不等式$2ab + a + b \ge t$进入$b(2a+1) \ge t - a$。 这隔离了$b$因为它允许我们解决直接阈值条件。 
3. 计算最小整数$b$则满足不等式。 如果$t \le a$，那么甚至$b = 1$有效，因为右侧变为非正值，并且该高度的每个矩形都是有效的。 
4. 否则计算：$$b_{\min} = \left\lceil \frac{t - a}{2a + 1} \right\rceil$$此步骤确保我们找到第一个有效宽度。 
5. 如果$b_{\min} \le m$，然后所有宽度从$b_{\min}$到$m$是有效的，有贡献$m - b_{\min} + 1$矩形。 如果$b_{\min} > m$，这个高度没有任何贡献。 
6. 总捐款额$a$。 

### 为什么它有效

 对于固定$a$，函数$f(b) = 2ab + a + b$严格递增于$b$。 这保证了单一阈值行为：一次$b$足够大以满足不等式，所有较大的值也满足它。 因此计数有效$b$简化为找到不等式成立的第一个点。 由于每个$a$是独立且详尽地处理的，每个有效矩形都被精确计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m, t = map(int, input().split())

ans = 0

for a in range(1, n + 1):
    # inequality: 2ab + a + b >= t
    # b(2a+1) >= t - a

    if t <= a:
        # even b = 1 works
        ans += m
        continue

    numerator = t - a
    denom = 2 * a + 1

    b_min = (numerator + denom - 1) // denom  # ceil division

    if b_min <= m:
        ans += m - b_min + 1

print(ans)
```核心实现直接遵循代数变换。 重要的细节是案件的处理$t \le a$，其中分子变为非正数，并且朴素的上限除法会错误地产生零或负值。 单独处理它可以避免微妙的一对一问题。 

上限除法采用整数运算实现，保证正确性，不存在浮点精度风险。 

## 工作示例

 ### 示例 1：`2 3 15`我们迭代高度$a = 1, 2$。 

为了$a = 1$，阈值是：$$2b + 1 + b = 3b + 1 \ge 15 \Rightarrow b \ge 5$$自从$m = 3$, 无效$b$。 

为了$a = 2$:$$4b + 2 + b = 5b + 2 \ge 15 \Rightarrow b \ge 3$$仅有的$b = 3$作品。 

| 一个 | 状况 | b_min | 有效 b 计数 |
 | --- | --- | --- | --- |
 | 1 | 3b+1 ≥ 15 | 5 | 0 |
 | 2 | 5b+2 ≥ 15 | 3 | 1 |

 最终答案是1。 

这显示了较大的高度如何使宽度效率的阈值向下移动。 

### 示例 2：`3 3 15`我们检查所有$a \in [1,3]$。 

| 一个 | 不平等 | b_min | 贡献 |
 | --- | --- | --- | --- |
 | 1 | 3b+1 ≥ 15 | 5 | 0 |
 | 2 | 5b+2 ≥ 15 | 3 | 1 |
 | 3 | 7b+3 ≥ 15 | 2 | 2 |

 总计为$3$。 

这个例子强调了多个高度可以贡献重叠的有效宽度，并且算法独立地累积它们。

 ## 边缘情况

 一种边缘情况是当$t \le a$，其中代数重排表明分子为非正数。 例如，与$n = 5, m = 5, t = 2$， 为了$a = 3$我们得到$t - a = -1$。 正确的行为是每一个$b \in [1, m]$作品。 代码显式检查此条件并分配全部贡献$m$，防止错误的天花板划分。 

另一个边缘情况是当计算的$b_{\min}$超过$m$。 例如，$n = 3, m = 3, t = 100$。 对于所有人$a$，阈值太大了$b_{\min} > 3$，并且贡献必须为零。 该算法通过检查来确保这一点$b_{\min} \le m$在添加任何内容之前，避免出现负计数或环绕错误。
