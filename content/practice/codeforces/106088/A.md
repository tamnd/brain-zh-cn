 ## 解决方案
 ## 问题理解

 我们有一系列带有持续时间的剧集`t1, t2, ..., tn`。 Kolya 按顺序观看它们并准确地花费`m`每天的时间单位，除了可能在最后一天，系列赛可能会在他到达之前结束`m`。 

如果一天的观看时间长度正好满足要求，则称一天为成功`m`在某个剧集的精确结尾处结束。 如果会议在剧集中间结束，那么这一天就不成功。 最后一天比较特殊：如果少于`m`观看单位仍然存在，即使该系列恰好在剧集结束时结束，这一天也不计算在内。 

输入给出剧集持续时间和每日观看限制。 我们必须计算有多少成功的日子。 

关键的观察结果是观看进度仅取决于总观看时间。 经过一整天的时间，Kolya 观看了`m`单位，两天后`2m`，整整三天后`3m`， 等等。 当这些累计观看时间之一与一集的结束重合时，这一天就算成功。 

就以下方面而言，限制很小`n`，最多只有 1000，但剧集长度和`m`可以达到`10^9`。 这立即排除了任何一次推进一个时间单位的模拟。 我们需要一种基于累积总和而不是单个时间单位的方法。 

一个微妙的点是最后部分日的处理。 假设所有剧集的总持续时间是`S`。 完整观看天数仅对应时间`m, 2m, 3m, ...`严格在观看过程中。 如果某一集在某个时间结束`S`和`S`一整天后未达到长度`m`，一定不能算在内。 

考虑：```
1 5
3
```整个系列仅持续 3 个单元。 科利亚花了一天的时间看完了它。 正确答案是`0`， 不是`1`。 

另一个容易犯的错误是计算一天中发生的剧集结局，而不是精确地计算当天边界处的剧集结局。 

例子：```
2 3
1 2
```第一集在 1 个单位后结束，但第一天在 3 个单位后结束。 正确答案是`1`因为时间 3 的日期边界与第 2 集的结尾重合。计算每个情节的结尾将是错误的。 

当总持续时间恰好是以下的倍数时，就会出现第三种边缘情况`m`。 

例子：```
2 2
1 3
```总持续时间为 4。有整整两天。 第 1 天在时间 2 结束，即第 2 集的中间。第 2 天在时间 4 结束，恰好在系列结束时，也是剧集结束时。 答案是`1`。 

最后一整天仍然很重要，因为科利亚准确地观看了`m`当天的单位。 

## 方法

 思考这个问题的一个蛮力方法是模拟 Kolya 每天的观看情况。 每天我们都会确定他停在哪里，可能会经历几集。 自从`n`只有1000，即使反复晋级也是可以接受的。 我们可以保留当前的剧集和其中的位置，并每天进行模拟，直到系列结束。 

暴力模拟是正确的，因为它完全遵循观看过程。 问题是它不必要地复杂。 进程的实际状态完全由总观看时间决定。 

关键的观察结果是，情节结束发生在情节持续时间的累积总和上。 如果我们定义```
pref[i] = t1 + t2 + ... + ti
```然后情节`i`正好在时间结束`pref[i]`。 

当完整观看日的结束与剧集结束重合时，即为成功的一天。 完整观看日有时会结束```
m, 2m, 3m, ...
```直至系列总持续时间。 

所以我们只需要计算剧集结束时间的倍数`m`。 

还有一个条件。 最后一天不计算在内。 一集在某个时间结束`pref[i]`仅当一整天结束时达到该值时才会做出贡献。 这意味着`pref[i]`必须能整除`m`。 

别的都无所谓。 我们不需要模拟暂停或情节转换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n + 天数) | O(1) | O(1) | 已被接受但不必要的复杂 |
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 阅读`n`和`m`。 
2.初始化一个运行累计时长`prefix = 0`。 
3. 初始化答案`ans = 0`。 
4. 按顺序处理剧集。 
5. 将当前剧集时长添加到`prefix`。 
6. 检查是否`prefix % m == 0`。 

如果累积持续时间可除以`m`，则本集的结束与某个完整观看日的结束一致。 
7. 如果条件成立，则增加`ans`。 
8.所有episode处理完毕后，输出`ans`。 

为什么这会自动处理最后的部分日期？ 因为不完整的一天永远不会在以下时间结束：`m`。 只有完整观看天数才对应累积观看次数`k·m`。 因此以倍数计算剧集结局`m`准确计算成功的天数并忽略任何最后不完整的天数。 

### 为什么它有效

 让`pref[i]`是情节的时刻`i`完成。 

后`k`完整观看日，Kolya 完全观看了`k·m`时间单位。 当一天的结束时刻与剧集的结束时间重合时，这一天就是成功的。 这意味着一定存在一个情节`i`这样```
pref[i] = k·m
```对于某个整数`k`。 

这个条件相当于```
pref[i] % m = 0
```每个成功的一天恰好对应于一个情节结束时间，可除以`m`，并且每个这样的剧集结束时间都对应于成功的一天。 该算法精确地计算了这些时刻，因此结果是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
t = list(map(int, input().split()))

prefix = 0
ans = 0

for x in t:
    prefix += x
    if prefix % m == 0:
        ans += 1

print(ans)
```该解决方案维护迄今为止处理的所有事件的累积持续时间。 

每当累积持续时间可以被整除时`m`，这意味着当前剧集恰好在某个完整观看日结束时结束。 我们立即增加答案。 

不需要额外的数据结构。 累积总和可以变得如此之大`1000 × 10^9 = 10^12`，它很容易适合 Python 整数。 

一个常见的实施错误是试图推断个别日期和剧集位置。 可分性条件已经捕获了整个过程。 另一个错误是担心最后的部分时间。 因为只有倍数`m`计算时，不完整的天数将被自动排除。

 ## 边缘情况

 考虑输入```
1 5
3
```累计剧集结束时间为 3。`3 % 5 != 0`，算法不计算并返回`0`。 这符合最后部分日不成功的规则。 

现在考虑```
2 3
1 2
```累加和为 1 和 3。只有 3 能被 3 整除，所以答案是`1`。 

痕迹：```
prefix = 1 -> not counted
prefix = 3 -> counted
```该算法计算日期边界，而不仅仅是情节结束的存在。 

最后，考虑```
2 2
1 3
```累计和为 1 和 4。```
1 % 2 = 1
4 % 2 = 0
```只有第二集有贡献，产生答案`1`。 第一天在第 2 集中结束，而第二天则在该系列结束时结束。 该算法通过累积情节结束时间的整除自动捕获这种区别。
