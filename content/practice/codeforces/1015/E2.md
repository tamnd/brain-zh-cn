 ## 解决方案
 ## 问题理解

 网格包含两种单元格：填充的和空的。 我们可以放置称为星星的几何物体，其中每颗星星由一个中心填充单元和四个沿四个基本方向延伸的直臂组成。 所有四个臂的长度必须相同，并且该长度必须至少为一。 明星的尺寸`s`因此正好占据中心单元加上`s`沿同一行和列的每个方向上的单元格。 

任务是确定网格中的整个填充单元集是否可以解释为放置在网格内任何位置的此类星星的集合的并集。 星星可以任意重叠，因此唯一的要求是输入中的每个填充单元格至少被一颗星星覆盖，并且没有星星延伸到网格之外。 

输出不需要最小化任何东西。 任何有效分解最多为`n·m`星星是可以接受的。 

约束条件`n, m ≤ 1000`意味着多达一百万个细胞。 这立即排除了任何试图从每个中心明确测试每个可能的恒星并在四个方向上重复扫描的解决方案。 在最坏的情况下，对所有大小的所有恒星进行简单的枚举将达到立方行为，这是不可接受的。 

一个更微妙的困难是恒星可以重叠，这消除了局部结构之间通常的独立性。 在不了解最大可能臂长的情况下提前提交的贪婪布局很容易失败。 

一种典型的失败案例是多位明星共用手臂时：```
..*..
.***.
*****
.***.
..*..
```如果我们贪婪地在每个中心只放置大小为 1 的星星，我们就无法捕获可以降低表示复杂性的更大结构，即使问题不需要最小化。 真正的问题不是优化，而是正确性：我们必须确保每个所需的单元都属于某个有效的星形配置，并且每个星形在内部与满臂一致。 

当存在星形中心但一个方向被破坏时，会出现另一种边缘情况：```
..*..
.**..
*****
.....
.....
```这里存在中心，但缺少向上的臂，因此即使局部密度看起来很高，在该中心也无法形成有效的恒星。 

这些例子表明，正确性取决于验证候选中心周围完整的对称臂结构。 

## 方法

 一个蛮力的想法是对待每一个`'*'`细胞作为潜在的中心，并尝试向所有四个方向向外延伸手臂。 对于每个中心，我们尝试增加尺寸，直到一个方向断裂。 这给了我们该中心可能的最大恒星尺寸。 检查一个中心的费用为`O(min(n, m))`，并且有`O(nm)`中心，因此总复杂度变为`O(nm·min(n, m))`，达到约`10^9`最坏情况下的操作。 这太慢了。 

关键的观察结果是网格不需要用不相交的结构来解释。 相反，我们只需要确保覆盖范围。 这使我们能够从最大可能的有效中心向下贪婪地构建恒星。 

对于每个单元，我们可以使用预先计算的连续的计数来计算最大可能的恒星大小`'*'`细胞在四个方向。 如果细胞有臂长`k`，那么它可以成为大小恒星的中心`1`通过`k`。 一旦我们确定了所有这样的中心，我们就可以获取所有最大的星星（那些具有最大有效尺寸的星星），并且这些大星星已经覆盖了网格的很大一部分。 任何剩余的未覆盖的`'*'`细胞必须是较小恒星的中心或已经被较大恒星覆盖的臂的一部分。 

这自然会导致我们贪婪地将最大可能尺寸的恒星放置在每个有效中心，并依靠重叠来覆盖剩余结构的结构。 

我们预先计算四个 DP 数组：左、右、上、下，其中每个条目存储连续的`'*'`很重要。 这允许恒定时间计算每个细胞的最大恒星尺寸：```
size = min(up, down, left, right) - 1
```如果该值为正，则细胞可以容纳一颗恒星。 

生成所有恒星后，我们在单独的网格中标记它们的覆盖范围。 如果有的话`'*'`如果未覆盖，则配置无效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(nm·min(n,m)) | O(1) | O(1) | 太慢了 |
 | 最佳| O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们使用定向预处理和贪婪放置以结构化方式构建解决方案。 

1.计算四个辅助网格`up`,`down`,`left`， 和`right`。 每个单元格存储多少个连续的`'*'`细胞从它向那个方向延伸，包括它自己。 这使我们无需重新扫描网格即可了解本地臂容量。 
2. 对于每个细胞`(i, j)`其中包含`'*'`，计算其最大可能的恒星大小为`k = min(up[i][j], down[i][j], left[i][j], right[i][j]) - 1`。 减法将中心单元从臂长约束中移除。 
3.如果`k > 0`，记录一个以`(i, j)`有尺寸`k`。 这代表锚定在该位置的最大有效星。 
4. 维护一个布尔覆盖网格，最初全部为 false。 对于每个记录的恒星，标记它覆盖的所有单元：中心加上所有四个臂，直到其大小。 此步骤至关重要，因为即使单个恒星不能解释网格，重叠的恒星也可以共同解释网格。 
5. 放置完所有最大星星后，扫描网格。 如果有的话`'*'`细胞没有被覆盖，答案是不可能的。 
6.输出所有记录的星星。 

重要的想法是，每个可以为有效星形中心做出贡献的单元格首先被最大化。 任何可能存在的较小恒星都会通过与这些最大结构的重叠来隐式处理。 

### 为什么它有效

 正确性基于以下事实：每个有效的恒星必须完全位于`'*'`所有四个方向的细胞。 DP 阵列精确捕获这些片段。 如果一个单元格不能支持大小为 1 的星体，则它不可能成为任何有效星体中心的一部分，因此排除它是安全的。 

当我们放置最大的星星时，该区域的所有潜在贡献都被耗尽。 任何可以放置在那里的较小的恒星都完全包含在较大或相等的恒星的覆盖范围内，因此不会丢失有效的配置。 因此，如果有的话`'*'`仍然未被发现，它不能用任何有效星星的组合来解释。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
g = [list(input().strip()) for _ in range(n)]

up = [[0]*m for _ in range(n)]
down = [[0]*m for _ in range(n)]
left = [[0]*m for _ in range(n)]
right = [[0]*m for _ in range(n)]

for i in range(n):
    for j in range(m):
        if g[i][j] == '*':
            left[i][j] = (left[i][j-1] if j > 0 else 0) + 1

for i in range(n):
    for j in range(m-1, -1, -1):
        if g[i][j] == '*':
            right[i][j] = (right[i][j+1] if j+1 < m else 0) + 1

for j in range(m):
    for i in range(n):
        if g[i][j] == '*':
            up[i][j] = (up[i-1][j] if i > 0 else 0) + 1

for j in range(m):
    for i in range(n-1, -1, -1):
        if g[i][j] == '*':
            down[i][j] = (down[i+1][j] if i+1 < n else 0) + 1

stars = []
covered = [[False]*m for _ in range(n)]

for i in range(n):
    for j in range(m):
        if g[i][j] == '*':
            k = min(up[i][j], down[i][j], left[i][j], right[i][j]) - 1
            if k > 0:
                stars.append((i, j, k))
                for t in range(k+1):
                    covered[i][j+t] = True
                    covered[i][j-t] = True
                    covered[i+t][j] = True
                    covered[i-t][j] = True

ok = True
for i in range(n):
    for j in range(m):
        if g[i][j] == '*' and not covered[i][j]:
            ok = False

if not ok:
    print(-1)
else:
    print(len(stars))
    for x, y, s in stars:
        print(x+1, y+1, s)
```该解决方案首先将方向信息压缩为四个DP通道，这避免了在评估恒星大小时的重复扫描。 关键的微妙之处在于覆盖标记必须对称地包括中心和所有四个臂； 这里任何一点点的错误都会破坏正确性，因为星星的大小`k`精确跨度`k`细胞向外，不`k+1`。 

最后的验证步骤确保不会出现任何偏差`'*'`细胞不受任何恒星影响。

 ## 边缘情况

 临界边缘情况是当潜在中心具有部分对称性时。 考虑一个网格，其中存在左臂和右臂，但一个垂直方向立即断裂。 DP阵列正确分配`up`或者`down`作为1，使`min(...) - 1 = 0`，这可以防止在该位置非法创建恒星。 

另一个边缘情况是细星链，需要重叠才能覆盖所有细胞。 由于该算法始终检查最终覆盖范围而不是假设完整性，因此即使放置了许多小星星，它也能正确检测到缺失的覆盖范围。 

最后的边缘情况是完全空的网格。 在这种情况下，不会生成任何星形，并且输出只是零，这与星形是可选的并且仅需要覆盖填充单元的要求是一致的。
