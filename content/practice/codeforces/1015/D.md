 ## 解决方案
 ## 问题理解

 我们站在一长串编号从 1 到 1 的房子中最左边的房子$n$。 我们必须准确执行$k$移动，每次移动都包括从我们当前的房子跳到任何不同的房子。 每次跳跃所贡献的成本等于房屋指数的绝对差，并且所有移动的总成本必须恰好为$s$。 

任务不是找到最短路径或最大化距离。 相反，我们必须构造任意序列$k$访问过的房屋，使得连续位置不同并且总累积行驶距离匹配$s$。 我们被允许重新参观房屋，因此该结构是步行而不是简单的路径，但我们不能留在原地。 

关键的难点在于序列长度是固定的，并且每一步贡献一个非负的量，其边界为$1$到$n-1$。 这会产生一个约束组合问题：我们正在精确求和$k$绝对差异，每个差异都是通过选择位置间接选择的。 

这些约束已经暗示了贪婪或建设性的解决方案。 的价值$n$可以非常大，最多可达$10^9$，因此任何依赖于迭代房屋的方法都是不可能的。 移动次数$k$取决于$2 \cdot 10^5$，允许线性结构。 总距离$s$可以大到$10^{18}$，这会立即排除任何每步搜索或距离上的 DP。 

一种简单的方法会尝试模拟所有可能的长度序列$k$，也许通过回溯或 BFS 对由位置和剩余距离定义的状态进行。 这会立即失败，因为状态空间是$O(nk)$原则上，即使限制位置也无济于事，因为距离是连续的$n$。 

当尝试贪婪的“总是走极端”策略而不计划均等时，会出现一个更微妙的问题。 例如，在 1 和 1 之间反复跳动$n$最大化每次移动的距离，但不保证每个中间总和都是可实现的。 过度调整的贪婪策略无法恢复，因为所有动作都是积极的。 

## 方法

 蛮力观点将每次移动视为选择下一个房子并累积距离。 从起始位置开始，我们分支到所有可能的下一个房屋，递归地继续直到$k$步骤被选择。 这是正确的，因为它枚举了所有有效序列，但分支因子是$n-1$，深度为$k$，导致指数复杂性。 即使通过剩余距离进行修剪仍然留下太多状态，因为$s$是大而连续的。 

关键的观察结果是，问题不在于个人位置，而在于每次移动可以贡献多少距离。 最大可能的单次移动是$n-1$，通过在端点之间跳转来实现。 这表明大多数动作应该是最大跳跃，只有少数动作需要调整以微调总和。 

我们可以考虑构建一个序列，在两个极端之间反复反弹以产生较大的贡献，然后调整一些步骤以小的受控量减少或增加总数。 由于每个动作都独立地贡献绝对差异，因此我们可以从左到右贪婪地设计动作，确保我们永远不会陷入低于或高于目标的困境。 

最终的构造依赖于这样的想法：我们总是可以通过将最大跳跃分成更小的绕路来减少最大跳跃，而不打破邻接约束。

 ## 算法演练

 我们从 1 号房子开始，一步步构建序列。 

1. 计算每次移动的最大可能贡献，即$n-1$。 如果$s > k \cdot (n-1)$，没有解决方案，因为即使是最大可能的跳跃也无法达到目标。 这给出了立即不可能的条件。 
2. 将当前位置初始化为 1 并记录剩余距离$rem = s$。 
3. 对于每一个$k$移动，贪婪地决定下一个位置。 在每一步中，我们都尝试使用尽可能大的跳跃，这并不妨碍我们在剩余的动作中完成剩余的距离。 这意味着我们确保：$$rem - (n-1) \cdot (k - i - 1) \le \text{chosen move distance} \le n-1$$此约束确保了未来步骤的可行性。 
4. 为了实现选定的距离，我们在适当的方向上选择尽可能远的房子。 如果我们当前处于位置$x$，我们可以移动到 1 或$n$。 我们选择在考虑未来最大贡献后能够实现所需剩余金额的端点。 
5. 选择下一个位置后，我们更新剩余距离并继续。 

核心思想是，我们始终保持在剩余目标的可达到范围内，同时利用极端位置来最大限度地提高灵活性。 

### 为什么它有效

 该结构保持了每次移动后的不变量，可以使用剩余的移动次数来实现剩余距离，因为我们从不会在当前预算中花费超过必要的费用$n-1$每一步。 由于每一步都将剩余总和保持在可行的区间内，因此我们永远不会陷入死胡同。 端点充当通用调整点，允许在需要时通过切换方向来实现每个所需的差异。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, s = map(int, input().split())

    if s < k or s > k * (n - 1):
        print("NO")
        return

    print("YES")

    cur = 1
    rem = s
    res = []

    for i in range(k):
        left_moves = k - i - 1

        # We choose next position among [1, n]
        # Try both endpoints and pick a valid one
        # that keeps feasibility for remaining steps

        # candidate 1: go to 1
        d1 = abs(cur - 1)
        if rem - d1 <= left_moves * (n - 1) and rem - d1 >= left_moves:
            res.append(1)
            rem -= d1
            cur = 1
            continue

        # candidate 2: go to n
        d2 = abs(cur - n)
        res.append(n)
        rem -= d2
        cur = n

    print(*res)

if __name__ == "__main__":
    solve()
```该实现在每一步都保持贪婪的可行性检查。 关键部分是验证选择移动后，剩余总和仍然位于最小可能剩余距离（所有步移动至少 1）和最大可能剩余距离（所有步移动至少 1）之间。$n-1$）。 

决定先尝试第 1 号房屋，然后再返回第 1 号房屋$n$是任意的； 两个端点都是对称的，并且可行性保证至少一种选择有效。 的更新为`rem`确保我们跟踪准确的剩余预算，同时`cur`跟踪计算绝对差所需的当前位置。 

一个常见的错误是忘记下界约束$rem \ge left\_moves$，这强制每个剩余的移动必须贡献至少 1 个距离单位。

 ## 边缘情况

 一个重要的边缘情况是当$s$恰好处于最小可能值$k$。 这迫使每次移动都精确地贡献 1 个距离单位，这意味着我们必须在相邻的房屋之间交替。 该算法自然地处理了这个问题，因为可行性检查强制选择每个剩余步骤至少保留一个单元的移动，从而防止任何会降低未来灵活性的跳跃。 

另一个边缘情况是当$s = k \cdot (n-1)$，其中每次移动都必须是端点之间的全长跳跃。 在这种情况下，构造确定性地在 1 和$n$。 贪心选择始终有效，因为任何步骤都不需要减少。 

第三种微妙的情况发生在$n=2$。 每一步都被迫精确地贡献 1。该算法简化为 1 和 2 之间的简单交替，并且可行性边界崩溃为单个有效序列而不会产生歧义。
