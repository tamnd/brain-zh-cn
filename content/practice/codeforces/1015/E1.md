 ## 解决方案
 ## 问题理解

 我们给出一个矩形网格，其中填充了两种类型的单元格，占用的单元格标记为`*`和空单元格标记为`.`。 任务是将网格解释为称为星形的几何对象的叠加。 星形的定义是选择一个中心单元，然后在四个基本方向上延伸四个等长的臂。 所有臂都必须具有正长度，并且沿着这些臂的每个单元格（包括中心）都必须位于网格内，并且必须是`*`。 

输出并不要求我们重建唯一的分解。 相反，任何有效星星的集合都是可以接受的，只要每个`*`网格中至少有一个由所有恒星联合产生的星细胞所覆盖。 允许重叠，因此多个星星可能会绘制同一个单元格。 

关键的困难在于星形是受约束的物体：有效的星形迫使对称的十字形状，并且中心必须是`*`具有连续的`*`四个方向上的链条。 任何`*`不属于这种结构的一部分必须可以解释为被其他恒星的臂或中心覆盖。 

约束很小，两个维度都最多为 100。这立即排除了每个测试用例超出大致三次或轻微二次工作的任何内容。 对每个单元中所有可能的恒星进行完整枚举已经是可行的，因为每个单元最多可以支持 100 个可能的半径。 

一种幼稚的方法可能会尝试对待每一个`*`独立且贪婪地将其分配给某个恒星，而不确保全局一致性。 当一个单元似乎是多个重叠候选恒星的一部分但只有一种配置实际上涵盖了所有所需的方向时，这种情况就会失败。 

当一个细胞位于手臂上但它本身并不是一个有效的中心时，就会出现更微妙的失败情况。 例如，考虑一个长水平段`*`没有垂直支撑。 贪心方法可能会尝试将恒星沿其居中，但这些中心都没有有效的垂直臂，因此分解变得不可能，即使该部分可以通过在其他地方重叠较小的恒星来解释。 

另一个问题是重复计算中心：如果我们独立标记每个有效中心并假设覆盖范围是自动的，我们可能无法确保所有中心`*`s 实际上至少可以通过一个正长度的星形臂到达。 

## 方法

 暴力的观点是将每个单元视为潜在的中心并尝试所有可能的半径。 对于每个候选人，我们检查所有四个臂是否完全由`*`。 如果有效，我们可以放置该星星。 生成所有星星后，我们需要验证每个`*`单元格被该组中至少一颗星星覆盖。 这种蛮力已经接近最优，因为每个中心最多有`O(min(n, m))`半径，检查一颗恒星需要花费其半径的线性时间。 

然而，这仍然做了多余的工作：许多细胞不可能成为中心，因为它们没有`*`在所有四个方向。 关键的观察是，有效的星中心完全由最小连续长度确定`*`四个方向的分段。 一旦我们计算出每个单元的这些长度，我们就立即知道每个位置的最大可能的恒星大小。 

这将问题变成了建设性提取问题：我们计算所有有效的恒星，然后验证覆盖范围。 由于允许重叠，我们不需要选择最小集或避免冗余； 我们只需要正确性。 任何适合的星星都可以包括在内。 

最终的解决方案是预先计算定向运行`*`，提取所有最大的星星，然后在网格上模拟它们的覆盖范围，以确保每个`*`已解释。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| O(n²m + nm·min(n,m)) | O(纳米) | 最坏情况下太慢|
 | 定向 DP + 构造 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 1. 对于每个单元格，计算有多少个连续的单元格`*`s 向上、向下、向左、向右延伸，包括其自身。 这可以通过四次 DP 经过网格来完成。 这样做的原因是每个方向仅依赖于该方向上的前一个单元，因此局部信息以线性时间全局传播。 
2. 对于每个像元，将其四个方向值中的最小值减一确定为最大可能的恒星大小。 需要减一，因为一颗星的大小`s`需要`s`距离中心一步之遥，意味着中心本身贡献一个单位。 
3. 收集计算出的大小至少为 1 的所有像元。每个这样的像元代表一个有效的星形中心。 
4. 将这些星星记录在输出列表中。 我们不会尝试优化星星的数量，因为该问题不需要极小值。 
5. 构造一个单独的布尔网格，初始化为 false。 对于每颗星星，标记其覆盖的所有单元格：中心和所有四个臂直至其半径。 
6. 标记所有星星后，验证每个原件`*`细胞被覆盖。 如果有的话`*`未被覆盖，网格无法表示，因此输出-1。 

关键的想法是我们从不试图决定哪颗恒星“拥有”一个细胞。 相反，我们只确保所有有效的星星都存在，然后验证它们的并集是否解释了网格。 

### 为什么它有效

 我们构建的每颗星都相对于其中心的方向约束而言是最大的。 这意味着如果一个单元可能是一个更大的有效恒星的中心，我们的 DP 已经准确地捕获了该半径。 任何分解中的每个有效恒星都必须遵守相同的方向限制，因此将我们限制在这些候选者中并不会删除任何必要的结构。 

由于星星可以任意重叠，因此我们只需要一个有效星星的超集，其并集覆盖网格。 DP生成的集合就是这样的超集，最终的验证步骤保证了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [list(input().strip()) for _ in range(n)]

    up = [[0]*m for _ in range(n)]
    down = [[0]*m for _ in range(n)]
    left = [[0]*m for _ in range(n)]
    right = [[0]*m for _ in range(n)]

    for i in range(n):
        for j in range(m):
            if g[i][j] == '*':
                left[i][j] = (left[i][j-1] + 1) if j > 0 else 1

    for i in range(n):
        for j in range(m-1, -1, -1):
            if g[i][j] == '*':
                right[i][j] = (right[i][j+1] + 1) if j + 1 < m else 1

    for j in range(m):
        for i in range(n):
            if g[i][j] == '*':
                up[i][j] = (up[i-1][j] + 1) if i > 0 else 1

    for j in range(m):
        for i in range(n-1, -1, -1):
            if g[i][j] == '*':
                down[i][j] = (down[i+1][j] + 1) if i + 1 < n else 1

    stars = []
    cover = [[False]*m for _ in range(n)]

    for i in range(n):
        for j in range(m):
            if g[i][j] == '*':
                s = min(up[i][j], down[i][j], left[i][j], right[i][j]) - 1
                if s > 0:
                    stars.append((i, j, s))

    for i, j, s in stars:
        cover[i][j] = True
        for d in range(1, s+1):
            cover[i+d][j] = True
            cover[i-d][j] = True
            cover[i][j+d] = True
            cover[i][j-d] = True

    for i in range(n):
        for j in range(m):
            if g[i][j] == '*' and not cover[i][j]:
                print(-1)
                return

    print(len(stars))
    for i, j, s in stars:
        print(i+1, j+1, s)

if __name__ == "__main__":
    solve()
```该实现从四个定向 DP 数组开始，将所有几何约束压缩为每个单元的恒定时间查询。 计算时减一`s`至关重要，因为 DP 计数包括中心本身，但恒星大小仅根据臂长定义。 

覆盖模拟显式地扩展了每颗星，这是安全的，因为星的总数受以下限制：`n*m`，并且每次扩展最多花费`O(n+m)`最坏情况下每颗星。 在约束为 100 的情况下，这仍然有效。 

最终的验证循环确保不会`*`单元格无法解释，这是正确性所需的唯一全局条件。

 ## 边缘情况

 当出现以下情况时，就会出现极端情况：`*`单元没有垂直或水平连续性。 在这种情况下，所有方向 DP 值都会崩溃，计算出的大小为零，因此该像元永远不会被选为星形中心。 如果这样的单元未被任何其他星覆盖，则最终验证会检测到它并拒绝配置。 

另一个边缘情况是一条长水平线`*`没有垂直支撑。 每个细胞都有左右连续性，但上下连续性等于1，所以不能形成恒星。 该算法正确地没有产生星星，并且如果有星星则验证失败`*`存在。 

最后一种情况是有效星星严重重叠，其中覆盖范围是多余的。 该算法仍然有效，因为它不依赖于唯一性或最小性，仅依赖于有效扩展的存在，并且最终的覆盖检查吸收了所有冗余。
