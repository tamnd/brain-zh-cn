 ## 解决方案
 ## 问题理解

 给定一条从 1 到 m 的一维数轴，并在其上放置几个闭区间。 每个区间覆盖其端点之间的每个整数点，包括两端。 由于间隔可能重叠，因此某些点可以被多次覆盖，而另一些点可能根本不会被覆盖。 

任务是找出 1 到 m 范围内所有未被任何区间覆盖的整数点。 换句话说，我们正在绘制一条线上的线段，并询问哪些位置仍未绘制。 

约束非常小：线段数量和坐标范围最多为 100。这立即告诉我们，即使是二次或三次方法也是安全的，并且我们不需要高级数据结构。 对所有点进行直接模拟就已经足够了。 

主要的微妙之处来自于正确处理重叠和端点。 由于线段是包含性的，因此必须将完全等于 l 或 r 的点视为被覆盖。 一个常见的错误是将段视为半开或忘记标记端点。 

当段完全覆盖整个范围时，会出现第二种边缘情况。 例如，如果我们有 [1, m]，则每个点都被覆盖，并且答案必须为 0，没有第二行。 总是打印第二行的粗心实现可能会达不到格式要求。 

另一个极端情况是重叠的单点段。 例如，[2,2]、[2,2]、[2,3]。 第2点仍然必须被视为覆盖一次，并且重复不得影响正确性。 

## 方法

 解决该问题最直接的方法是检查从 1 到 m 的每个点，并测试它是否位于至少一个线段内。 对于每个点 x，我们扫描所有线段并检查是否存在满足 li ≤ x ≤ ri 的 i。 如果不存在这样的段，我们将 x 添加到答案中。 

这是可行的，因为约束很小：最多 100 个点和 100 个线段，导致最多 10,000 次检查，这是微不足道的。 

然而，暴力破解结构存在冗余。 尽管可以更有效地预先计算覆盖范围，但每个点都会独立地重新检查所有段。 由于坐标范围很小，我们可以维护一个布尔数组 mark[1..m]，最初为 false，并在单次遍历段中标记任何段覆盖的每个点。 处理完所有段后，我们只需收集标记仍然为假的索引。 这消除了重复扫描并使解决方案更干净、更直接。 

关键的观察结果是我们不需要在线回答查询；我们需要在网上回答问题。 我们只需要最终的报道。 这使我们能够将范围标记转换为简单的数组扫描。

 | 蛮力 | 每点扫描 O(n·m) → O(n·m²) | O(1) | O(1) | 已接受但多余 |

 ## 算法演练

 我们用一个布尔数组来记录每个位置是否被覆盖。 

1. 初始化一个大小为 m+1 的数组 cover，并将所有值设置为 false。 这表示最初没有点被任何线段覆盖。 
2. 迭代每个段 [l, r]。 对于从 l 到 r 的每个整数 x，设置 Covered[x] = true。 这直接编码了覆盖范围的定义。 
3.处理完所有段后，从1迭代到m。 收集所有索引 x，使得 Covered[x] 仍然为 false。 这些正是未被覆盖的点。 
4. 设 k 为未覆盖点的数量。 打印 k，如果 k > 0，则以任意顺序打印收集的点，通常是递增顺序。 

我们可以安全地做到这一点的原因是每个段独立贡献覆盖范围，并且标记是幂等的。 如果多个线段覆盖同一点，重复分配不会改变结果。 

### 为什么它有效

在每一步中，当且仅当存在至少一个包含 x 的已处理段时，covered[x] 才为 true。 由于我们只对所有分段进行一次处理，并且仅在存在覆盖时将值设置为 true，因此不会引入误报。 同样，任何不在任何段内的点都不会被标记，因此它仍然是错误的。 这在数组状态和所有段的并集之间创建了精确的等价。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

covered = [False] * (m + 1)

for _ in range(n):
    l, r = map(int, input().split())
    for x in range(l, r + 1):
        covered[x] = True

ans = []
for x in range(1, m + 1):
    if not covered[x]:
        ans.append(x)

print(len(ans))
if ans:
    print(*ans)
```该解决方案使用直接布尔标记数组。 每个段上的内部循环显式地扩展了覆盖范围，由于约束较小，这是安全的。 最后一遍按升序收集所有未标记的点。 

一个微妙的实现细节是 range(l, r + 1) 的使用，它正确地包含了右端点。 缺少 +1 将错误地排除线段端点并产生错误的答案。 

输出格式条件也很重要：当没有未覆盖的点时，我们不能打印空的第二行。

 ## 边缘情况

 一个关键的边缘情况是所有段合并成一个连续的块。 例如：```
3 5
1 3
2 5
1 5
```处理标记从 1 到 5 的每个点。覆盖的数组变为全部 true。 最终扫描没有发现错误条目，因此 ans 为空，我们正确输出 0。任何无条件打印第二行的实现都将违反此处所需的格式。 

另一种情况是当段仅覆盖端点时：```
2 5
1 1
5 5
```只有位置 2、3 和 4 仍然为假。 标记循环确保仅填充精确的包含范围，因此端点不会将覆盖范围泄漏到相邻小区中。
