 ## 解决方案
 ## 问题理解

 给定两个长度相等的字符串，其中一个字符串表示字符的起始排列，另一个字符串表示目标排列。 唯一允许的操作是交换起始字符串中的两个相邻字符。 每次交换都会将字符向左或向右移动一个位置，因此通过多次交换，我们可以逐渐对字符串重新排序。 

任务是确定起始字符串是否可以使用这些相邻的交换转换为目标字符串，如果可以，则输出长度最多为 10000 的任何有效交换序列。 

关键的结构观察是相邻交换允许我们实现字符的任何排列，但仅限于字符串的多重集约束内。 如果两个字符串没有相同的字符频率，则任何交换序列都无济于事，因为交换只会对字符进行重新排序，而不会更改字符串中存在的内容。 

约束 n ≤ 50 足够小，我们可以在构建答案时直接在字符串上模拟操作。 即使是重复扫描和移位的二次过程也是可以接受的，因为在最坏的情况下，总工作量受到每相约 2500 次交换的限制，完全在限制范围内。 

当一个人试图在不检查可行性的情况下贪婪地匹配字符时，就会出现一种微妙的失败情况。 例如，如果我们忽略频率不匹配，我们可能会尝试“寻找”剩余后缀中不存在的字符，从而导致无限或无效的构造。 另一个边缘情况是贪婪策略找到匹配字符但没有正确地逐步移动它，从而导致交换后索引不一致。 

## 方法

 一个蛮力的想法是将其视为字符串排列空间中的最短路径问题。 每个状态都是一个排列，每次移动都会交换相邻的字符。 这形成了一个带有 n! 的图。 状态，BFS 会找到从 s 到 t 的一系列交换。 这是正确的，但即使对于 n = 10 也是完全不可行的，因为 10！ 已经超过 300 万个状态，每个状态最多有 n 个转换。 

关键的见解是我们不需要最短路径，只需要任何有效路径。 这允许一种建设性的贪婪方法：我们从左到右对齐字符串，一次固定一个位置。 当处理位置 i 时，我们确保使用相邻交换将 t[i] 的正确字符放入位置 i。 每次交换都会使角色靠近一步，然后我们重复直到它就位。 

这是有效的，因为一旦前缀被固定，以后的操作就不需要打扰它。 我们只移动不固定后缀内的字符。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 排列上的 BFS | O(n!) | O(n!) | 太慢了 |
 | 贪婪相邻交换 | O(n^3) | O(n^3) | O(n) | 已接受 |

 ## 算法演练

 我们直接在可变字符列表上模拟转换。 

1. 检查s和t中的多组字符是否相同。 如果不是，则没有任何交换序列可以将一种转换为另一种，因此我们立即返回 -1。 这是必要的，因为交换会保留字符计数。 
2. 将字符串 s 转换为列表，以便我们可以在 O(1) 时间内交换字符。 
3. 从 0 到 n - 1 迭代每个位置 i，将其视为 t[i] 的最终位置。 
4. 对于每个位置 i，从 i 向前扫描，直到找到索引 j，使得 s[j] 等于 t[i]。 这确保我们选择一个可以移动到位的有效事件。 
5. 一旦找到 j，就重复将 s[j] 与 s[j - 1] 交换，每次递减 j 直到 j 等于 i。 每次交换都被记录为一次操作。 
6. 对所有职位继续此过程。 最后，s 变得与 t 相同。

这种逐步移动起作用的原因是交换相邻元素允许我们向左“冒泡”字符。 每次交换都会将其到目标位置的距离恰好减少一，并且前缀保持固定，因为在 i 完成后我们永远不会移动 i 左边的字符。 

### 为什么它有效

 不变的是，处理完位置 i 后，前缀 s[0..i] 与 t[0..i] 完全匹配，并且任何进一步的操作都不会更改此前缀。 每个步骤仅交换从 i 开始的后缀内的元素，因此较早的位置保持不变。 因为我们总是从后缀中选择匹配的字符，所以我们永远不会破坏后面位置的可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    s = list(input().strip())
    t = list(input().strip())

    if sorted(s) != sorted(t):
        print(-1)
        return

    ops = []

    for i in range(n):
        j = i
        while j < n and s[j] != t[i]:
            j += 1

        while j > i:
            s[j], s[j - 1] = s[j - 1], s[j]
            ops.append(j)  # 1-based position of swap
            j -= 1

    print(len(ops))
    if ops:
        print(*ops)

if __name__ == "__main__":
    solve()
```该解决方案首先使用排序进行可行性检查，这已经足够了，因为排列完全由字符计数定义。 主循环从左到右构造目标字符串。 内部搜索找到所需字符的有效出现，随后的交换阶段将其移动到位。 

一个常见的错误是忘记每次互换后指数都会发生变化。 此实现通过物理修改列表并在交换循环期间跟踪字符的当前位置来避免这种情况，从而确保正确性而无需重新计算。 

另一个微妙之处是输出索引。 该问题需要从 1 开始的交换位置，因此我们在根据 j-1 和 j 之间的交换位置递减 j 之前输出 j。

 ## 边缘情况

 一种重要的边缘情况是目标字符串需要在多个相同字符之间移动字符。 例如，将“aabb”转换为“bbaa”需要仔细选择每个字符的正确出现位置。 该算法可以正确处理此问题，因为它始终从当前位置向前搜索，确保它选择后缀中仍然存在的字符，而不是重复使用错误的出现位置。 

另一种情况是正确的字符已经位于位置 i 时。 在这种情况下，内部交换循环将被完全跳过，并且前缀保持不变。 这可以防止不必要的操作并确保我们保持在 10000 次移动限制之内。 

最后一种情况是所需字符仅存在于字符串末尾。 算法仍然找到它，然后将它一步一步冒泡离开。 每次交换都会严格缩短其到目标位置的距离，保证终止时不会超出或破坏较早的位置。
