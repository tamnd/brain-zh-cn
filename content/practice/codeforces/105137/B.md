 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串和两种修改它的方法，每种方法都有一定的成本。 目标是将字符串转换为“良好”形式，其中相邻字符对没有不同。 换句话说，最终的字符串必须完全由 0 或完全由 1 组成，因为任何混合都必然包含转换 01 或 10。 

每个测试用例给出字符串长度和两个操作成本。 一个操作翻转一位。 另一个操作选择两个位置并用它们的 XOR 值替换这两个字符，这实际上使两个选定的位置等于 0（如果它们相同），或者等于 0 和 1（具体取决于它们的值）。 解释二进制位上的异或，第二个操作仅在应用于一对不同位时才有用，因为如果两个位相等，则异或使它们保持相等并且无助于减少不平衡。 

输出是使字符串均匀所需的最小成本。 

约束很小，n 和成本都最多 1000 个，最多 10 个测试用例。 这立即允许解为 n 的二次甚至三次，但运算的结构表明存在更简单的计数参数。 

一种简单的方法可能会尝试所有操作序列，但这很快就会变得指数级，因为每个操作都会改变状态空间。 稍微更合理的蛮力会尝试基于对 0 和 1 的计数进行动态规划来计算最小成本，但即使这样也太过分了，因为只有 0 和 1 的总数很重要。 

误解第二个操作会产生微妙的边缘情况。 例如，对两个相等的位应用异或没有任何用处，但天真的求解器可能会错误地认为它总是有助于降低成本。 另一个边缘情况是字符串已经统一，无论操作成本如何，答案都应该为零。 

## 方法

 关键的观察是最终字符串必须全为零或全一。 因此，问题归结为决定哪个目标更便宜：将所有内容都转换为 0 或将所有内容转换为 1。 

让字符串包含 z 个零和 o 个一。 为了使一切归零，我们必须消除一切。 为了使一切都为1，我们必须消除所有的零。 

第一个操作以成本 a 翻转一位，因此如果单独使用，转换不匹配的位总是每位成本 a。 第二个操作作用于两个不同的位。 如果我们取一个 0 和一个 1，应用 XOR 会使两者相等，有效地将两者变成 0，这意味着我们消除了一个 1，但也不会在该对中引入新的 1。 然而，从净效果来看，此操作将 1 的数量减少了 1，同时也没有在其他地方增加 1 的数量。 对称地，如果从相反的目标角度来看，它会将零减少 1。 

因此，每个操作 2 一起消耗 1 0 和 1 1，成本为 b。 它本质上是相反位之间的配对操作。 

因此，为了转换为全零，我们只关心删除那些。 我们可以单独翻转每个 1，或者将每个 1 与 0 配对并使用操作 2。最佳策略变成配对和翻转之间的贪婪成本比较。 

每对 (0,1) 都可以通过采用两次翻转与一次异或运算中更便宜的方法以及可能的额外调整来最佳解决。 由于每个 XOR 以成本 b 去除一个 1，同时还消耗一个 0，因此关键数量变为我们可以形成多少对：min(z, o)。 

相同的逻辑对称地适用于转换为全一。 

蛮力方法将模拟字符串上的所有操作，每个状态扩展的时间复杂度为 O(n²) 或更糟，并且在概念上无法扩展。 只计算物质的观察将问题简化为每个测试用例的恒定时间算术。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(2ⁿ) 或 O(n³) | O(n) | 太慢了 |
 | 基于计数的优化 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们计算字符串中零和一的数量。 然后我们评估两个候选答案：使所有字符为零的成本和使所有字符为一的成本。 

1. 计算字符串中零 z 和一 o 的数量。 这是必要的，因为只有分布才重要，而不是位置。 
2. 要将一切都归零，我们必须消除所有的1。 只要有利，我们就会考虑使用操作 2 将 1 与 0 配对。 
3. 令 k 为可能对的数量，即 min(z, o)。 每对都可以使用一次异或运算或两次翻转来解析。 我们比较成本并为每对采取更便宜的策略。 
4. 配对后，任何剩余的（如果 z < o）必须使用操作 1 单独翻转。 
5. 计算全为零的总成本。 
6. 重复对称推理以生成所有 1：将 0 与 1 配对并翻转剩余的 0。 
7. 输出两个计算成本中的最小值。 

为什么有效：每个操作都会删除或转换不匹配的位，并且任何操作都不能影响两个以上的位置。 任何最佳解决方案都可以重新排列，以便异或运算仅作用于相反的位，而翻转仅用于剩余的不匹配位。 这种范式确保解决方案仅取决于零和一的计数，而不取决于排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, a, b = map(int, input().split())
        s = input().strip()

        z = s.count('0')
        o = n - z

        # cost to make all zeros
        pair = min(z, o)
        cost_to_zero = pair * b + (o - pair) * a

        # cost to make all ones
        cost_to_one = pair * b + (z - pair) * a

        print(min(cost_to_zero, cost_to_one))

if __name__ == "__main__":
    solve()
```该实现首先提取零和一的计数，因为在识别操作仅作用于值类型之后位置信息是无关紧要的。 变量`pair`表示可以形成多少个相反的位对。 

为了转换为零，必须删除每个 1。 我们首先尝试使用操作 2 将尽可能多的 1 与 0 配对，然后使用操作 1 支付剩余的不匹配的 1。对称地重复使用相同的结构来转换为 1。 

最终答案是两种策略中的最小值。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    def solve():
        t = int(input())
        out = []
        for _ in range(t):
            n, a, b = map(int, input().split())
            s = input().strip()
            z = s.count('0')
            o = n - z
            pair = min(z, o)
            cost0 = pair * b + (o - pair) * a
            cost1 = pair * b + (z - pair) * a
            out.append(str(min(cost0, cost1)))
        return "\n".join(out)

    return solve()

 ## 边缘情况

 一种边缘情况是字符串已经统一。 用于输入`0000`，算法计算`z=4`,`o=0`， 所以`pair=0`。 两种成本均为零，这正确地反映了不需要任何操作。 

当一项操作比另一项操作便宜得多时，就会出现另一种边缘情况。 如果`b < 2a`，如果可能的话，配对总是更可取的。 例如`1010`和`a=5`,`b=1`产生`pair=2`, 给予成本`2`，而单独翻转会花费`20`。 该公式自然地捕获了这一点，而无需分支。 

最后一个边缘情况是极端不平衡，例如`1110000`。 该算法仍然有效，因为配对仅消耗最少的计数，而多余的计数则需要独立处理。
