 ## 解决方案
 ## 问题理解

 我们被要求建造一个$n \times n$精确使用二进制网格$k$那些和$n^2 - k$零。 网格的唯一要求是结构性的：当我们将所有行视为二进制字符串并将所有列视为二进制字符串时，不同行的数量加上不同列的数量必须最多为 5。 

因此，约束不是关于局部模式或邻接性，而是关于压缩多样性。 只要网格最终只有很少的不同行类型和列类型组合，我们就可以将它们放置在任何地方。 

关键的难点在于$n$可以大到 1000，因此任何构造都必须是线性或接近线性的$n^2$。 然而，输出本身已经$n^2$， 所以$O(n^2)$建设是不可避免的，也是预料之中的。 

主要的微妙之处在于我们必须同时控制行多样性和列多样性。 独立构造行的简单方法很容易创建太多不同的列模式。 

如果我们尝试粗心的构造，一些失败的情况会立即出现。 例如，逐行填充，直到耗尽$k$往往会产生许多不同的列，因为填充行和未填充行之间的边界创建了“阶梯”模式。 

另一个有问题的想法是交替行，例如$1010...$和$0101...$。 即使行多样性很小，列多样性也会爆炸$n$，违反了等级界限。 

挑战在于设计一个网格，其中行和列都高度重复，并且它们的组合不同计数保持有界。 

## 方法

 蛮力思维方式是将其视为约束满足问题：尝试为每个单元格分配一个值并维护一组不同的行和列，每当数字超过 5 时就回溯。这显然是指数级的$n^2$因为每个单元格的选择都可能会更改行和列签名，并且没有保证提前终止的修剪结构。 即使是为了$n = 20$，这已经是不可能的了。 

关键的观察是我们不需要优化任何内容或搜索。 我们只需要存在一个非常简单的结构。 如果我们可以确保所有行至多是几个固定模式之一，并且所有列也来自一个小的固定集合，则条件自动满足。 

强制行和列多样性保持恒定的最简单方法是使网格几乎均匀，所有变化都集中在一个矩形中。 如果除了可能的一或两行之外所有行都相同，并且除了可能的一或两列之外所有列都相同，则行类型和列类型都保持有界。 

一种干净的结构是将网格分成两个大的统一块：一个块填充 1，其余填充零。 这会产生最多两种不同的行类型和最多两种不同的列类型，从而给出最多 4 的总超级等级，这满足了要求。 

然后我们调整全 1 矩形的大小，使其恰好包含$k$那些。 

构造变为：选择形成左上矩形的行和列的前缀，其面积精确或稍微调整以匹配$k$。 因为我们允许任意排列，所以我们可以简单地在前缀区域内逐行贪婪地填充矩形前缀，并将其他所有内容保持为零。 这仍然保留了有限数量的不同行和列模式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（回溯网格构建）| 指数在$n^2$|$O(n^2)$| 太慢了 |
 | 结构化矩形结构|$O(n^2)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 我们构建一个网格，其中所有网格都放置在单个连续的左上角区域中。 

1. 初始化一个$n \times n$网格充满零。 这保证了修改之前所有行和列的单一基线模式。 
2. 从左上角的单元格开始填充，从左到右逐行扫描。 
3. 放置直到恰好$k$已经写了。 这确保了满足计数约束，而不需要复杂的平衡。 
4、一次立即停止$k$已放置。 所有剩余的单元格保持为零。 
5. 输出网格。 

这一过程的重要结构效果是，行集最多只能在一个转换点发生变化：最后部分填充的行上方的行充满了直到某个前缀的“1”，而下方的所有行都是“0”。 类似地，列的行为是对称的：最后部分填充的列之前的列在所有填充的行中都包含 1，而其之后的列全部为零。 

### 为什么它有效

 生成的网格最多有两种行类型：完全为零的行，以及可能包含 1 前缀的行。 不会出现其他行图案，因为填充是在单个线性扫描中进行的。 类似地，最多有两种列类型：完全为零的列，以及可能具有部分前缀为 1 的列。 因此，不同的行加列的总数最多为 4，满足最多 5 的要求。构造不依赖于$k$除了填充进行的程度之外，因此它适用于所有有效输入。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    
    grid = [["0"] * n for _ in range(n)]
    
    for i in range(n):
        for j in range(n):
            if k == 0:
                break
            grid[i][j] = "1"
            k -= 1
        if k == 0:
            break
    
    for row in grid:
        print("".join(row))

if __name__ == "__main__":
    solve()
```该解决方案将网格构建为字符数组列表，以实现高效突变。 双循环执行简单的左上角填充。 提前停止条件确保我们一次不会溢出$k$已放置。 

关键的实现细节是在以下情况下正确打破两个循环：$k$达到零。 如果不打破外循环，输出仍然是正确的，但会浪费时间； 在某些变体中，未能正确中断内部循环可能会在完成后错误地覆盖零。

 ## 边缘情况

 一个关键的边缘情况是当$k = 0$。 该算法从不进入填充循环，直接输出全零网格。 这仅保留一种行类型和一种列类型，使超级等级保持最小。 

什么时候$k = n^2$，循环填充整个网格。 每个单元格都变为 1，产生单个行类型和单个列类型。 这是最统一的可能配置并且基本满足约束。 

什么时候$k$正好停在行边界处，填充区域形成一个完美的矩形，恰好产生两种行类型：全一行和全零行。 列多样性同样受到限制。 

什么时候$k$停在一行的中间，只有一行成为混合前缀行。 这是引入第三个候选模式的唯一行，但仍然使不同行和列的总数远低于 5。
