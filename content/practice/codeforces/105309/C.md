 ## 解决方案
 ## 问题理解

 网格可以看作是二进制图块的棋盘。 每个单元格包含一个 0 或一个 1，我们可以翻转一个单元格，将 0 变成 1 或 1 变成 0。目标是修改网格，以便水平或垂直方向上的两个相邻单元格不共享相同的值。 换句话说，相邻单元之间的每条边在变换后必须连接不同的值。 

有效的最终配置正是棋盘图案，因为这是保证每个相邻对都不同的唯一结构。 任务是选择最接近原始网格的配置，最大限度地减少翻转单元的数量。 

这些约束允许最多 10^4 个测试用例，所有网格中总共最多有 10^6 个单元格。 这意味着解决方案必须与每个测试用例的网格大小呈线性关系，因为每个测试用例的任何二次甚至重常数因子重复处理都会太慢。 如果我们能够以简单的方式表达目标结构，则对每个网格进行一次遍历就足够了。 

当网格非常小时，就会出现微妙的边缘情况。 对于 1 x 1 网格，没有需要满足的邻接约束，因此无论值如何都不需要翻转。 对于 1 x m 或 n x 1 网格，棋盘条件简化为沿一条线交替值，并且相同的两种模式逻辑仍然适用。 

另一个潜在的陷阱是试图通过贪婪翻转来局部修复违规行为。 例如，如果我们看到两个相等的邻居，翻转其中一个可能会在其他地方产生新的违规。 这使得任何本地策略都不可靠。 

## 方法

 强力策略将尝试为每个单元格分配值，同时动态强制执行邻接约束。 人们可以想象对每个单元尝试所有可能的 0 和 1 分配，但这会导致 2^(n·m) 配置，即使对于小网格，这也是完全不可行的。 

一种稍微结构化的暴力方法是逐个单元做出决定，在保持与邻居的有效性的同时，在是否翻转的问题上进行分支。 即使进行了修剪，状态的数量也会呈指数级增长，因为每个决策都会影响未来的约束。 操作数大约增长2^(n·m)，远远超出了限制。 

关键的观察是任何有效的最终网格都必须像棋盘一样交替值。 一旦认识到这一点，问题就简化为将原始网格与仅两个可能的目标网格进行比较。 一种模式将值 0 分配给 (i + j) 为偶数的所有单元格，否则为 1，而另一种模式为其补码。 我们只需计算匹配每个模式需要翻转多少次并取最小值。 

这将问题从对所有网格的指数搜索减少到对输入进行恒定数量的遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^(n·m)) | O(2^(n·m)) | O(n·m) | 太慢了 |
 | 最佳 | O(n·m) | O(1) 额外 | 已接受 |

 ## 算法演练

 ### 最优策略

1. 请注意，任何有效的配置都必须在相邻单元格之间交替值，因此只能使用棋盘图案。 这将搜索空间减少到两个候选者。 
2. 定义第一个候选模式，其中如果 (i + j) 为偶数，则位置 (i, j) 处的单元格预计为 0，否则为 1。 这创建了一个固定的交替结构。 
3. 将第二个候选模式定义为第一个候选模式的精确反转，其中偶校验单元为 1，奇校验单元为 0。这覆盖了唯一剩余的有效棋盘。 
4. 对于网格中的每个单元格，将其当前值与两个候选模式进行比较，并分别计算不匹配的数量。 不匹配对应于所需的翻转。 
5. 每个模式所需的翻转次数就是该模式的总不匹配计数。 
6. 网格的答案是两个不匹配计数中的最小值。 

### 为什么它有效

 任何有效的配置都必须为相邻单元格分配不同的值，这会强制网格图采用严格的二分着色。 网格是二分的，具有基于坐标奇偶性的唯一分区，因此每个有效的解决方案完全取决于分配给一个奇偶性类的值的选择。 一旦这个选择被确定，所有其他细胞都将被迫。 这就是为什么恰好有两个有效模式，以及为什么与这两个模式进行比较可以保证最少的翻转次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n, m = map(int, input().split())
        grid = [input().strip() for _ in range(n)]
        
        cost0 = 0
        cost1 = 0
        
        for i in range(n):
            row = grid[i]
            for j in range(m):
                bit = row[j]
                
                expected0 = '0' if (i + j) % 2 == 0 else '1'
                expected1 = '1' if (i + j) % 2 == 0 else '0'
                
                if bit != expected0:
                    cost0 += 1
                if bit != expected1:
                    cost1 += 1
        
        out.append(str(min(cost0, cost1)))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现独立地读取每个测试用例并维护与两个棋盘模式相对应的两个计数器。 对于每个单元格，它使用索引奇偶校验来计算该单元格在每个模式下应具有的值。 每个不匹配都会增加相应的成本。 

正确的常见实现细节是避免重新计算网格转换。 该解决方案不是构建完整的预期网格，而是使用奇偶校验动态计算预期值。 这使得内存使用量最小化并确保线性时间处理。 

另一个微妙之处是确保两个成本计数器独立维护。 混合逻辑或尝试重用单个计数器会导致不正确的结果，因为这两种模式逐个单元地不同。

 ## 边缘情况

 1 x 1 网格没有邻接约束。 该算法评估两种模式，并找到至少一种模式的零不匹配，因为两种模式都会减少到单个期望值，并且两种选择都可以通过构建两种模式的最小值来匹配。 

统一的网格（例如 2 x 2 块中的全零）强调了比较两个棋盘方向的需要。 一种模式正确对齐一半的单元格，导致恰好两次不匹配，而另一种模式的表现类似，并且最小的正确捕获最佳翻转。 

已经与棋盘图案匹配的完全交替网格对于两个奇偶校验分配之一产生零不匹配。 该算法无需任何特殊大小写即可正确识别这一点，因为匹配配置的不匹配计数器保持为零。
