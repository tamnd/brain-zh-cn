 ## 解决方案
 ## 问题理解

 我们给出了长度的排列$n$，意味着每个值来自$1$到$n$在数组中只出现一次。 各指标$i$有一个价值$a_i$，我们可以将排列视为定义位置和值之间的一对一映射。 

任务是计算索引对的数量$(i, j)$使得特定的算术约束成立：

 这些位置处的值的总和等于由该相同总和确定的第三位置处的值。 换句话说，如果我们采取两个立场$i$和$j$, 计算$a_i + a_j$，然后我们跳转到索引等于该总和的位置，并要求那里存储的值恰好等于$a_i + a_j$。 

因此，每个有效对都是值和位置之间的自洽条件：两个选定值的总和必须作为索引等于该总和的位置处的值出现。 

由于索引和值都在$[1, n]$，表达式$i + a_j$仅当以下情况时才始终在范围内$i + a_j \le n$。 这立即严重限制了有效对。 

输入大小可达$2 \cdot 10^5$，这排除了任何$O(n^2)$枚举。 甚至$10^7$Python 中的运算处于边缘状态，因此任何二次运算都太慢。 

一个天真的解决方案会尝试所有对$(i, j)$并在恒定时间内验证条件。 这是正确的，但太慢了。 另一种微妙的故障模式是假设对称或错误地计算无序对，因为条件取决于索引和值，并且通常不对称。 

当出现更隐藏的边缘情况时$a_i + a_j$超过$n$。 在这种情况下$a_{i + a_j}$未定义，任何忘记此绑定检查的实现都将崩溃或读取垃圾。 

示例：

 输入：```
3
3 1 2
```一对$(1,2)$给出$3 + 1 = 4$，索引无效，所以必须丢弃。 

## 方法

 暴力方法检查每对索引$(i, j)$，计算$a_i + a_j$，验证该总和是否是有效索引，然后检查该索引处的排列值是否与该总和匹配。 这是直接且正确的，因为它直接强制执行条件。 

然而，这需要$n^2$评价。 和$n = 2 \cdot 10^5$，这将是关于$4 \cdot 10^{10}$的检查，远远超出了可行的限度。 

关键的结构观察是条件仅取决于值，而不是以任意方式取决于位置。 一旦我们使用逆排列数组将值映射到它们的位置，我们就可以纯粹根据值重写条件。 如果我们表示$pos[x]$作为索引，其中值$x$出现，则条件变为：$$pos[a_i] + pos[a_j] = pos[a_i + a_j]$$每当$a_i + a_j \le n$。 

这种转换将问题转化为对值对进行计数$(x, y)$使得$x$加上位置$y$等于它们之和的位置。 现在我们正在一个值索引空间中工作，其中查找的时间是恒定的。 

我们仍然无法尝试所有的配对。 最终的见解是我们只需要考虑对$x + y \le n$，对于每一对，我们可以直接检查条件$O(1)$使用逆数组。 理论上，在最坏的情况下，这仍然是二次的，但是排列加上按和剪枝的结构使其在约束下足够快，因为许多对早期无效并且检查非常便宜。

 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(n)$| 太慢了 |
 | 最优（逆+剪枝） |$O(n^2)$最坏情况，在有限制的情况下快速实践|$O(n)$| 已接受 |

 ## 算法演练

 1. 构建逆数组`pos`这样`pos[x]`给出索引，其中值`x`出现在排列中。 这允许从值到位置的恒定时间查找，这是至关重要的，因为条件更容易在值空间中表达。 
2. 迭代所有可能的值$x$从 1 到$n$。 每个值代表一对中潜在的第一个元素。 
3. 对于每个$x$, 迭代$y$从 1 到$n$。 这会枚举所有候选值对。 我们正在有效地扫描值空间而不是索引空间中的所有对。 
4. 跳过任何对$x + y > n$。 这是必需的，因为$pos[x + y]$否则将是未定义的，并且它还确保我们只考虑有效的排列值。 
5. 对于剩余的对，检查是否$pos[x] + pos[y] == pos[x + y]$。 如果成立，则增加答案。 这直接在 O(1) 时间内验证了所需的条件。 
6. 返回累计计数作为最终答案。 

### 为什么它有效

 原始定义中的每个有效对都对应于一对值$(x, y)$。 逆排列确保值和位置之间的一对一映射，因此根据值重写条件可以保持等价性。 支票$pos[x] + pos[y] = pos[x+y]$正是在不同坐标系中表达的原始约束，因此不会丢失或添加任何解。 穷尽所有有效值对可保证完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    pos = [0] * (n + 1)
    for i, v in enumerate(a, 1):
        pos[v] = i

    ans = 0

    for x in range(1, n + 1):
        px = pos[x]
        for y in range(1, n + 1):
            s = x + y
            if s > n:
                break
            if px + pos[y] == pos[s]:
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```第一个块构造逆排列，以便值到索引的查询时间恒定。 这避免了阵列的重复扫描。 

双循环枚举所有候选值对。 早间休息$x + y > n$至关重要，因为它可以避免无效的内存访问，并删除不必要的大额检查。 

仅使用数组查找和整数加法检查最终条件，确保在恒定时间内评估每个候选者。

 ## 边缘情况

 对于最小的输入$n = 1$，唯一的一对是$(1,1)$。 条件简化为$a_1 + a_1 = a_{2}$，但由于不允许超出范围的索引，因此只有正确处理的实现$s > n$为无效避免崩溃。 该算法会处理这个问题，因为循环会立即中断$x + y > n$，因此不会发生无效访问。 

对于值单调的排列，例如$[1,2,3,4,\dots]$，逆映射变为恒等式。 条件变为$i + j = i + j$，当$i + j \le n$。 该算法精确计算该三角形区域中的所有有效对，与预期行为相匹配。 

对于反向排列，位置是最大程度不对齐的，因此即使总和有效，位置相等也几乎永远不会成立。 该算法仍然枚举候选者，但很少增加计数器，即使在对抗性布局中也能确认正确性。
