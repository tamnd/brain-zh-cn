 ## 解决方案
 ## 问题理解

 我们有一棵无根的树$n$节点及其节点的目标排列。 最终目标是通过一系列全局“重新扎根洗牌”将树转换为有根线形树，以便从根开始的 DFS 完全按照给定的顺序访问节点。 

单个洗牌不是局部边缘修改。 相反，我们选择一个节点$V$并在概念上将树分割为$V$，生成多个连接组件。 然后通过选择其中的根来递归地“重新扎根”每个组件，最后将所有生成的扎根组件附加回$V$。 重要的效果是，每次洗牌都会在选定的顶点处重新定义整个结构的根，同时保留树的连接性，但允许在该根下任意重组子树。 

经过所有的洗牌之后，最终的结构必须是一条路径（每个节点的度数最多为 2），并且从根开始的 DFS 顺序必须与给定的排列完全匹配。 我们被要求尽量减少洗牌次数。 

约束条件$n \le 10^5$每次测试和总计$2 \cdot 10^5$强烈排除任何尝试显式模拟洗牌或重复重建树的解决方案。 任何可接受的解决方案都必须将问题减少到每个测试用例的少量结构观察，每个测试可能是线性时间。 

一个关键的微妙约束是最终的 DFS 顺序是完全固定的。 这意味着不仅邻接结构很重要，而且确切的遍历顺序也很重要。 这消除了自由：任何有效的最终树本质上都被迫表现得像与排列一致的路径。 

将洗牌解释为标准根更改时会出现一个幼稚的错误。 它更强大：它允许在选定的根下任意重新排列整个子树。 这通常会误导人们认为一个操作总是可以强加所需的结构，当初始树已经具有与目标排序不兼容的约束时，这是错误的。 

例如，考虑以 1 为中心的星形排列$1,2,3,4$。 以 1 为根的单个洗牌已经产生有效的路径排序，因此答案是 1。但是如果排列是$2,3,4,1$，一个天真的方法可能仍然以 1 为根并期望 DFS 灵活性，但 DFS 顺序是由邻接固定的，因此我们必须通过操作仔细控制边缘方向。 

另一种边缘情况是，排列已经是树的 DFS 顺序，但不是从当前结构下的有效根开始。 即使如此，至少需要一次洗牌，因为树最初是无根的，所以我们必须显式选择一个根。 

## 方法

 蛮力视角会尝试模拟洗牌：选择一个节点，重建有根结构，并检查生成的 DFS 顺序是否可以更接近目标排列。 由于每次洗牌都可以全局重组子树，因此模拟需要考虑指数级的许多根选择和子树重建。 甚至代表所有中间有根树也已经$O(n^2)$在最坏的情况下，每次操作以及操作顺序都会使该成本倍增，远远超出了可行性。 

关键的简化来自于重新解释 shuffle 的实际作用：每次 shuffle 有效地围绕选定的根“重新定向”树，并允许我们从该根强制执行 DFS 一致的排序。 由于最终的结构必须是其 DFS 与固定排列匹配的路径，因此最终的树不是任意的； 它正是按照排列顺序的哈密顿路径。 

这意味着整个问题简化为询问我们必须“重置”根多少次，以便排列与路径上的 DFS 遍历兼容。 单个根选择为我们提供了一种与 DFS 兼容的排序。 如果排列可以与树的某个有根版本的 DFS 对齐，那么我们只需一次操作即可完成。 否则，我们至少需要一个额外的重组点，而事实证明，最小操作数对应于排列在原始树结构中强制“回溯中断”的次数。 

结构见解是，排列中的邻接必须对应于单个 DFS 兼容根的树中的边。 每当$p_i$和$p_{i+1}$由于没有通过边连接，我们被迫执行至少一次额外的洗牌来重新调整子树结构，以便这种邻接在 DFS 顺序中变得可执行。 每次这样的违规都会有效地增加所需的洗牌段的数量。 

因此，解决方案简化为计算排列如何分解为已经与原始树结构中的 DFS 邻接一致的最大段。

 ## 算法演练

 我们关注排列并检查它如何与树的邻接结构对齐。 

1. 为树构建邻接集，以便我们可以测试两个连续的排列节点是否通过边连接$O(1)$平均时间。 这很重要，因为整个解决方案依赖于检测 DFS 连续性被破坏的位置。 
2. 从左到右遍历排列并比较每一对$(p_i, p_{i+1})$。 只要树中它们之间没有边缘，就标记一个中断。 中断意味着任何以任何节点为根的 DFS 都无法在不进行重组的情况下将两个元素保留为连续访问。 
3. 计算此类中断的次数。 每次中断都表明必须将排列分割成一个新段，因为树上的 DFS 无法在不重新访问结构的情况下跳过非相邻节点，而这在单根遍历中是不可能的。 
4. 答案是这些断点形成的线段数。 由于每次洗牌只能对一个这样的段结构强制执行 DFS 一致性，因此每个段对应一个所需的洗牌过程。 
5. 通过在每个检测到的中断处对排列进行切片来输出片段本身。 每个段都作为一个洗牌过程打印。 

### 为什么它有效

 树上的 DFS 顺序必须遵守父子邻接转换。 如果排列中的两个连续节点不是直接连接的，则当前结构的求根不能使它们在 DFS 中连续，而无需重新配置子树边界。 每次洗牌都允许我们围绕选定的根重建树，从而有效地修复一个连续的 DFS 一致段。 因此，排列唯一地分解为最大DFS相邻块，并且每个块对应于一次洗牌。 这保证了最小性和正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        adj = [set() for _ in range(n + 1)]
        for _ in range(n - 1):
            u, v = map(int, input().split())
            adj[u].add(v)
            adj[v].add(u)

        p = list(map(int, input().split()))

        # identify breaks where consecutive permutation elements are not adjacent in tree
        cuts = [0]
        for i in range(n - 1):
            if p[i + 1] not in adj[p[i]]:
                cuts.append(i + 1)
        cuts.append(n)

        k = len(cuts) - 1
        print(k)
        for i in range(k):
            segment = p[cuts[i]:cuts[i + 1]]
            print(*segment)

if __name__ == "__main__":
    solve()
```该实现直接对分段逻辑进行编码。 邻接结构存储为一组，用于连续排列元素之间的常量时间成员资格检查。 

唯一微妙的部分是正确索引切割：我们存储新洗牌必须开始的位置，从索引 0 开始到 n 结束，因此切片会产生精确的片段，而不会重叠或遗漏。

 ## 边缘情况

 退化线树突出了边界行为。 考虑一条路径$1-2-3-4$与排列$1,2,3,4$。 该算法找不到不相邻的连续对，因此它会生成单个段。 这与节点 1 的 DFS 已经产生排列的事实相匹配。 

星形树暴露出最大的分枝。 如果排列在叶子之间交替，则除了涉及中心的步骤之外的每个步骤都会产生中断，从而强制许多片段。 该算法在每个非边缘邻接处正确分裂，反映出 DFS 无法在不重新访问中心的情况下在叶子之间跳转，这违反了所需的顺序。 

树上的完全随机排列通常会产生大量中断，并且每个中断对应于不可避免的重新扎根要求，这与每个片段是在单次洗牌下可实现的最大 DFS 一致前缀的解释相匹配。
