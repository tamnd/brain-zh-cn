 ## 解决方案
 ## 问题理解

 我们有一圈煎饼店。 每个商店都有一定数量的煎饼，所有商店都按一个循环排列，这样经过最后一个商店就会回到第一个商店。 

两个玩家与这个圈子互动。 首先，Lura 选择一家起始商店，然后 Oscar 选择另一家起始商店。 此后，双方都扩大了自己的领地，但都受到对方已经占领的店铺的限制。 移动总是沿着循环顺序进行，并且不允许他们通过对手已经访问过的商店。 

由于移动仅被其他玩家访问过的部分阻挡，因此两个起始选择有效地决定了如何将圆分割成两个连续的弧线。 每个玩家最终都会在这些弧线之一中收集所有煎饼，并且弧线边界完全由所选的两个起始位置决定。 

从卢拉的角度来看，她先选择起始商店，然后奥斯卡做出最佳反应。 奥斯卡的目标是选择他的起点，以便卢拉最终收集到的煎饼尽可能小，而卢拉则希望最大化她所保证的结果。 

输入给出了多个测试用例。 每个测试用例描述了商店的数量以及每个商店的煎饼。 输出是在双方最佳发挥下，卢拉能保证自己的最大煎饼总数。 

所有测试用例的 n 总和至多为 2×10^5 的约束意味着我们不能承受比每个测试用例大致线性或近线性更糟糕的情况。 每个测试用例的 O(n²) 策略会太慢，因为在最坏的情况下它会达到大约 10^10 次操作。 即使所有测试的 O(n²) 显然也会失败。 这将我们推向循环数组上的前缀和和二分搜索或双指针技术。 

当对间隔进行局部推理而不考虑循环环绕时，会出现微妙的失败情况。 例如，如果高价值商店跨越 n 和 1 之间的边界进行分割，则线性区间模型就会破裂：

 如果 p = [100, 1, 1, 100]，选择相反的点可以产生像 [100,1] 和 [1,100] 这样的弧，并且简单的线性分割会忽略这些点实际上在圆上相邻。 任何正确的解决方案都必须将数组视为循环数组。 

另一个陷阱是假设 Lura 只是对任意一对点取两个固定弧中较大的一个。 正确的值取决于遍历的最佳方向，该方向隐含在循环结构中，不可忽略。 

## 方法

 暴力策略尝试了卢拉（Lura）的每一种可能的起始商店和奥斯卡（Oscar）的每一种可能的反应。 对于一对固定的起点 i 和 j，圆被分成两条圆弧。 我们计算两条弧上的煎饼总和，并将约束引起的两个可能方向中较好的分配给 Lura。 这需要每对进行 O(n) 工作来计算弧和，如果简单地完成，则总计为 O(n³)，如果每对使用前缀和，则为 O(n²)。 

对于 n 达到 10^5 的情况，即使 O(n²) 也太慢了。 关键的观察是，对于固定的起始位置 i，Oscar 并没有选择任意复杂的策略。 他对 j 的选择仅决定如何将圆切割成两条互补的弧。 Lura 的最终增益成为这两个弧和的最大值，相当于在 i 和 j 处分割圆时较大部分的大小。 

如果总和为 T，且一条弧的总和为 x，另一条弧的总和为 T − x，因此 Lura 得到 max(x, T − x)。 最小化这个值相当于使 x 尽可能接近 T/2。 因此，Oscar 尝试选择 j，以便从 i 到 j 的弧和尽可能平衡。

因此，对于每个 i，问题简化为：在圆上的所有 j 中，找到一个割点，使从 i 开始的前缀和最接近总数的一半。 在奥斯卡做出最佳反应后，卢拉选择 i 来最大化可实现的最佳平衡。 

这将问题转化为经典的“与目标最接近的前缀和”查询，对具有前缀和的双倍数组进行查询，可以使用二分搜索来解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 前缀+二分查找| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将圆展开成一个重复两次的线性数组，以便任何圆弧段都成为正常区间。 我们在这个扩展数组上构建前缀和。 

1. 计算所有存储的总和 T，并在长度为 2n 的数组上构建前缀和，其中后半部分重复前半部分。 这允许从 i 开始的任何圆弧段都可以表示为标准子数组和。 
2. 对于原始范围中的每个起始位置 i，我们仅考虑区间 (i+1, i+n−1) 中的端点 j。 这一限制确保我们不会在一个完整的圆之外进行完整的环绕，从而保持分段有意义。 
3. 对于固定的 i 和 j，定义 x = 从 i 到 j 向前移动的煎饼之和。 计算方式为 prefix[j] - prefix[i]。 互补弧的值为 T − x。 
4. Lura 收到的该对的值是 max(x, T − x)。 当 x 尽可能接近 T/2 时，这种情况会最小化，因为这会使两条弧尽可能平衡。 
5. 因此，对于每个 i，我们搜索 j，使得 prefix[j] 最接近 prefix[i] + T/2。 我们对排序后的前缀结构使用二分搜索来找到最佳候选 j。 
6. 我们评估每个 i 的最佳 j，计算结果值 max(x, T − x)，并保留所有 i 的最大值。 

最终的答案是卢拉可以通过选择最佳起始位置来强制实现的最佳保证结果。 

### 为什么它有效

 关键的不变量是，对于任何一对起点，圆都被精确地划分为两个互补的弧，而 Lura 的增益仅取决于较大弧的大小。 一旦进行了这种减少，对抗结构就消失了：奥斯卡的最佳响应只是前缀和的几何平衡问题。 由于每个有效的分割恰好对应于前缀和中的一个差异，因此搜索最接近 T/2 的值可以捕获所有最佳响应，而不会丢失任何配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        p = list(map(int, input().split()))
        
        T = sum(p)
        a = p + p
        
        pref = [0] * (2 * n + 1)
        for i in range(2 * n):
            pref[i + 1] = pref[i] + a[i]
        
        best = 0
        
        from bisect import bisect_left
        
        for i in range(n):
            base = pref[i]
            target = base + T / 2
            
            l = i + 1
            r = i + n - 1
            
            # binary search on pref to find closest pref[j]
            # we search in indices [l, r]
            
            # convert to values
            # we binary search on pref[l..r]
            lo, hi = l, r
            
            while lo <= hi:
                mid = (lo + hi) // 2
                if pref[mid] < target:
                    lo = mid + 1
                else:
                    hi = mid - 1
            
            candidates = []
            if l <= lo <= r:
                candidates.append(lo)
            if l <= lo - 1 <= r:
                candidates.append(lo - 1)
            
            for j in candidates:
                x = pref[j] - base
                val = max(x, T - x)
                if val > best:
                    best = val
        
        print(best)

if __name__ == "__main__":
    solve()
```该实现在双倍数组上构建前缀和，以便循环间隔变成线性范围。 对于每个起始索引 i，它搜索一个分割点 j，使弧和最接近总数的一半。 二分搜索的目标是前缀值而不是原始索引，这就是为什么必须检查搜索位置周围的候选者的原因。 

必须注意仅对目标使用浮点除法； 使用 2_T 和 2_x 的整数算术可以避免严格实现中的精度问题。 当确切目标不存在时，围绕 lo 和 lo−1 的候选处理可确保正确性。 

## 工作示例

 考虑一个小配置，其中圆为 [4, 9, 3, 5]。 总和是21。 

对于固定起点 i = 0（值 4），我们检查圆周围可能的分割点 j。 目标是找到从 4 开始且最接近 10.5 的前缀和。 

| j | 弧和 x | T − x | 最大 (x, T − x) |
 | --- | --- | --- | --- |
 | 1 | 13 | 8 | 13 |
 | 2 | 16 | 16 5 | 16 | 16
 | 3 | 21 | 21 0 | 21 | 21

 从 i = 0 开始的最佳切割是 j = 1，给出值 13。 

现在考虑 i = 1（值 9）：

 | j | 弧和 x | T − x | 最大 (x, T − x) |
 | --- | --- | --- | --- |
 | 2 | 12 | 12 9| 12 | 12
 | 3 | 17 | 17 4 | 17 | 17
 | 4 | 21 | 21 0 | 21 | 21

 最佳切割是 j = 2，值为 12。 

这显示了不同的起点如何导致不同的最佳平衡，并且卢拉选择了最大化她最坏情况结果的起始位置。

 ## 边缘情况

 当所有煎饼都集中在一个小的连续区域时，就会出现一个关键的边缘情况。 对于像 [100, 1, 1, 1, 1] 这样的输入，任何避免分裂 100 个簇的切割都会产生高度不均匀的弧。 该算法可以正确处理此问题，因为最接近一半搜索仍然返回与大块相邻的边界，从而最大化可实现的弧。 

另一种情况是最佳分割正好位于中间，但没有明确表示为前缀和。 例如 [2, 2, 2, 2]。 目标是 4，但没有任何前缀从每个起点都精确等于 4。 二分搜索正确地选择两侧最近的前缀，确保所选的弧尽可能平衡。 

最后一个微妙的情况是环绕优势，其中最佳弧线穿过 n 对 1 边界。 重复的数组构造确保每个这样的段都表示为连续的间隔，因此除了扩展前缀和数组之外不需要特殊处理。
