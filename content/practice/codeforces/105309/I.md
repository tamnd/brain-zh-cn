 ## 解决方案
 ## 问题理解

 我们得到两个长度相等的字符串，并且允许我们使用作用于连续段的操作来修改第一个字符串。 每个操作都会选择一个段，并将两个固定转换之一应用于该段中的每个字符。 

第一个转换围绕字母表中点翻转字符，映射位置，以便开头附近的字母转到结尾附近的字母。 第二个转换将字符在字母表中旋转 13 个位置，形成对称配对结构，应用两次该结构将返回原始字符。 

任务是确定将初始字符串转换为目标字符串所需的最少段操作数，或者确定根本无法完成。 

重要的限制是操作适用于整个区间，这立即表明我们没有解决独立的每个位置的变换。 相反，我们正在处理一系列所需的字符级转换，当连续位置需要相同的操作类型时，可以合并这些转换。 

输入大小最多可达一百万个字符，这迫使任何解决方案都以线性时间运行。 任何显式检查所有可能的段或模拟间隔操作的方法都会失败，因为即使对段进行二次扫描也会超出运行时间几个数量级。 

当源中的字符在任一转换下都无法到达目标时，就会出现微妙的边缘情况。 例如，如果我们取一个字母，其反面与反面不同，但都不等于目标字符，则任何操作序列都无法修复该位置。 由于操作在段上是统一的，因此我们无法在不影响其邻居的情况下独立修复单个字符，因此不可能性会在全局范围内传播。 

另一种边缘情况是当所需的变换在位置之间交替时，例如需要重复在位置 i 处反向并在 i+1 处相反。 天真的贪婪者可能会尝试为每个字符开始和结束段，但这会超出计数，因为当连续位置共享相同的所需转换时，可以扩展段。 

## 方法

 一个蛮力的想法是直接模拟操作。 我们可以尝试所有可能的段，应用任一转换，并对所有字符串状态进行 BFS。 这在原则上是正确的，因为每次移动都会在有效字符串之间转换，但状态空间是巨大的。 每个字符串都有 n 个位置，每个操作选择取决于一对段端点和一个转换类型，每个状态给出 O(n^2) 次转换。 即使是受限的 BFS 也会立即爆炸，因为可能的字符串数量为 26^n。 

通过关注每个位置只有两个相关转换的事实，更加结构化的蛮力略有改进。 对于每个索引，我们可以计算反向或相反（或两者的序列）是否可以将 s[i] 转换为 t[i]。 然后，我们将问题简化为为每个位置分配几个标签之一，然后分组为片段。 即便如此，在最坏的情况下，重新计算段分区的成本仍然是 O(n^2)。 

关键的观察是我们永远不需要推理实际的中间字符串。 每个操作只是一个段上变换状态的统一切换。 因此，我们不考虑字符串，而是考虑派生数组，其中每个位置编码将 s[i] 固定为 t[i] 所需的转换（如果可能）。

一旦每个位置都映射到所需的状态，问题就变成：用最少数量的段覆盖这个数组，其中每个段应用一致的动作，以固定的方式改变状态。 该结构分解为对统一区域之间的转换进行计数，因为具有相同要求的任何最大连续块都可以通过单个操作来处​​理。 

这将问题简化为具有状态压缩的线性扫描。

 | --- | --- | --- | --- |

 ## 算法演练

 我们首先将每个字符对 (s[i], t[i]) 转换为一小组描述需要进行何种转换的可能性。 由于反向和相反是固定的对合，因此每个字母只有恒定数量的可达结果。 如果任何操作或任何有效组合都无法将 s[i] 映射到 t[i]，我们立即得出不可能的结论。 

接下来，我们将每个位置标准化为目标操作标签。 关键思想是我们不关心操作的确切顺序，只关心可以实现所需的每个位置转换的最小数量的连续段。 

然后，我们从左到右扫描数组，并将需要相同操作标签的连续位置分组。 每次标签更改时，我们都被迫开始一个新的段，因为应用于段的单个操作不能混合不兼容的转换要求。 

最后，我们计算存在多少个这样的段，这直接给出了最小移动次数。 

### 为什么它有效

 每个操作统一作用于连续范围，因此在任何选定的段内，所有位置都必须经历相同的转换。 这导致了一个约束，即该解决方案只能将数组划分为连续的块，其中每个块对应于单个一致的转换要求。 任何合并两个相邻不同要求的尝试都会导致至少一个位置不匹配，从而使其无效。 因此，最小操作数正是可行性过滤后相同变换标签的最大连续运行数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# map letter to 0..25
def val(c):
    return ord(c) - 97

def rev(x):
    return 25 - x

def opp(x):
    return (x + 13) % 26

def solve():
    n = int(input().strip())
    s = input().strip()
    t = input().strip()

    ops = []

    for i in range(n):
        a = val(s[i])
        b = val(t[i])

        ok = False

        # try no-op (should match directly)
        if a == b:
            ops.append(0)
            continue

        # try reverse
        if rev(a) == b:
            ops.append(1)
            continue

        # try opposite
        if opp(a) == b:
            ops.append(2)
            continue

        # try combinations (reverse then opposite or opposite then reverse)
        if opp(rev(a)) == b or rev(opp(a)) == b:
            ops.append(3)
            continue

        print(-1)
        return

    ans = 0
    prev = -1

    for x in ops:
        if x != prev:
            ans += 1
            prev = x

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将每个字符转换压缩为少量符号状态之一。 这些检查明确测试是否可以使用允许的转换到达目标字母，包括按顺序应用这两个操作产生正确结果的可能性。 

构建此数组后，第二个循环计算需要多少个连续段。 变量`prev`跟踪之前所需的状态，每次更改都会增加答案。 第一个元素总是开始一个新的段。 

一个微妙的实现点是在计算段之前必须检查可行性。 如果任何位置无法到达，继续将产生误导性的段计数。

 ## 边缘情况

 一种边缘情况是每个位置都可以单独到达，但在不同的所需操作之间交替。 对于像 s = "aaaa" 和 t = "znzn" 这样的输入，算法会分配一个类似 [reverse,opposite,reverse,opposite] 的序列。 扫描产生四个段，正确反映没有两个相邻位置共享兼容的操作。 

另一个边缘情况是完全一致的转换。 如果每个位置都通过相反的方式映射，则 ops 数组是恒定的。 扫描产生单个段，这意味着对整个范围应用一次操作就足够了。 

最后的边缘情况是不可能的。 对于任何反向或相反（或其组合）都不会产生目标的位置，算法会立即停止。 例如，如果 s[i] = 'a' 并且 t[i] 是不在集合 {a, z, n} 中的字母，则该位置无法在任何允许的序列下固定，并且无论其他位置如何，整个字符串都是不可能的。
