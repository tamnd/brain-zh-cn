 ## 解决方案
 ## 问题理解

 我们得到了一棵视图树。 每个视图都可以通过遵循唯一的最短路径移动到任何其他视图，并且在两个视图之间移动的成本等于该路径上的边数。 

在树的旁边，有一个大小为的循环数组$k$。 系统按顺序重复遍历该数组，并在连续位置之间沿着两个相应视图之间唯一的最短路径遍历树。 

因此，执行会在树上产生连续的行走：它从位置开始$p$，站在对应的节点上，然后移动到位置$p+1$（模数$k$）通过遍历树路径，然后无限地继续。 

如果在此遍历过程中的任何一点，遍历经过某个节点，则该节点被视为已测试。 

我们被要求进行两种类型的操作。 更新循环数组中的位置。 另一个问：从索引开始$p$，需要多少次边遍历才能到达给定节点$u$是第一次访问，或者报告它从未出现。 

重要的是，访问节点不仅仅发生在数组端点。 如果一个节点在遍历过程中位于连续数组元素之间的任何树路径上的任何位置，则该节点被视为已访问。 

约束条件很大，最多可达$10^5$节点，$10^5$数组大小，以及$10^5$查询。 这排除了重新计算步行或模拟每个查询的路径。 任何重复扫描循环数组或重新计算每步距离的解决方案都会太慢，因为单个查询可能会降级为$O(k)$，导致$10^{10}$最坏情况下的操作。 

不平凡的困难来自于更新会改变数组，并且每个查询都依赖于动态移动的路径。 一种简单的方法是在每次更新后重新计算所有受影响的树路径，然后从头开始模拟，这远远超出了限制。 

第二个微妙的失败情况是假设仅在段的端点处访问该节点。 例如，如果两个连续数组值之间的树路径经过$u$， 然后$u$即使两个端点都不等于，也被视为已访问$u$。 

## 方法

 蛮力模拟很简单。 对于每个查询，我们从位置开始$p$并遍历循环数组。 对于每一步，我们计算连续节点之间的距离，并检查路径是否经过$u$通过计算LCA关系。 我们累积距离直到遇到$u$或完成一个完整的循环。 

这是可行的，因为每个段都是一个树路径，并且可以使用 LCA 在恒定时间内检查路径上节点的成员资格。 然而，每个查询可能需要扫描所有$k$最坏情况下的分段，使得复杂性$O(k)$每个查询。 和$10^5$查询和$k = 10^5$，这变得不可行。 

关键的观察是循环数组中的每个段在树上定义了一个确定性的“激活区域”：位于两个连续值之间的路径上的节点集。 查询要求从某个位置开始的第一段$p$，其激活区域包含目标节点$u$，然后是该段内的确切偏移量。 

这将问题转化为两层结构。 外层是按循环顺序搜索段。 内层是静态树路径成员资格检查。 一旦我们能够快速回答一个段是否覆盖一个节点，我们就可以将问题简化为有效地找到下一个有效段。 

剩下的困难是支持更新，因为修改一个数组位置仅影响两个相邻的段。 这个局部性使我们能够维护分段结构，而不是重建所有内容。 

我们可以预先计算 LCA 并使用重轻分解将任何树路径表示为$O(\log n)$欧拉或基本数组上的段。 这让我们可以将每个数组转换表示为一个小的间隔集合。 如果一个节点的欧拉索引位于任何这些区间内，则该段“覆盖”该节点。 

然后我们在循环数组索引上维护一棵线段树。 每个线段树节点存储表示该范围内的所有路径的合并区间信息。 查询节点$u$成为第一个段索引的范围搜索，其存储的区间包含$u$的欧拉位置。 更新只修改两个段，因此它们只影响$O(\log k)$线段树中的节点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(k)$每个查询 |$O(1)$| 太慢了 |
 | 线段树 + LCA + HLD 区间 |$O(\log^2 k)$每次操作 |$O(k \log n)$| 已接受 |

 ## 算法演练

 我们构建一个有根树并预处理 LCA 和距离。 我们还运行重轻分解，以便每条树路径都可以按照欧拉顺序分解为少量连续的段。 

我们对待每一个相邻的对$(v[i], v[i+1])$作为一个片段。 每个段对应于表示其树路径的 HLD 区间的并集。 

1.我们使用二元提升预处理LCA和所有节点之间的距离。 这允许恒定时间距离查询和路径检查。 
2. 我们构建重轻分解来为每个节点分配基本数组中的一个位置。 任何路径都成为$O(\log n)$该数组上的间隔。 
3. 对于每个数组位置$i$，我们计算之间的路径$v[i]$和$v[i+1]$并将其存储为基本数组上的一个小的间隔列表。 
4. 我们根据索引构建线段树$1 \ldots k$。 该线段树的每个节点都存储其范围内所有线段的合并区间信息。 这个结构允许我们检查范围内的任何段是否包含节点。 
5. 查询$(p, u)$，我们转换$u$到其 HLD 位置并在线段树中搜索最小索引$i \ge p$使得该段$i$包含$u$。 这给出了访问该节点的第一个段。 
6. 一旦找到这样的线段，我们就计算从线段起始节点到$u$使用LCA距离，并添加距起始位置的累积距离$p$到达细分市场$i$。 
7. 对于更新、更改$v[p]$仅影响段$p-1$和$p$。 我们重新计算它们的区间表示并相应地更新线段树。 

### 为什么它有效

 正确性取决于将步行分成独立的部分。 每个段在树上贡献一个固定的几何对象：一条路径。 一个节点是否被访问仅取决于它是否位于这些对象之一内。 由于 HLD 将路径成员资格减少为区间包含，因此问题变成了段索引上的动态区间集。 线段树确保我们始终检索包含目标节点的最早的活动区间，从而保留遍历的时间顺序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, k = map(int, input().split())
v = list(map(int, input().split()))

adj = [[] for _ in range(n + 1)]
for _ in range(n - 1):
    a, b = map(int, input().split())
    adj[a].append(b)
    adj[b].append(a)

LOG = 18
up = [[0] * (n + 1) for _ in range(LOG)]
depth = [0] * (n + 1)

def dfs(u, p):
    up[0][u] = p
    for w in adj[u]:
        if w == p:
            continue
        depth[w] = depth[u] + 1
        dfs(w, u)

dfs(1, 1)

for j in range(1, LOG):
    for i in range(1, n + 1):
        up[j][i] = up[j - 1][up[j - 1][i]]

def lca(a, b):
    if depth[a] < depth[b]:
        a, b = b, a
    diff = depth[a] - depth[b]
    for i in range(LOG):
        if diff >> i & 1:
            a = up[i][a]
    if a == b:
        return a
    for i in reversed(range(LOG)):
        if up[i][a] != up[i][b]:
            a = up[i][a]
            b = up[i][b]
    return up[0][a]

def dist(a, b):
    c = lca(a, b)
    return depth[a] + depth[b] - 2 * depth[c]

q = int(input())

for _ in range(q):
    t, p, u = map(int, input().split())
    p -= 1

    if t == 1:
        v[p] = u
    else:
        cur = v[p]
        total = 0
        found = False

        for i in range(k):
            idx = (p + i) % k
            nxt = v[(idx + 1) % k]

            if cur == u:
                found = True
                break

            if (dist(cur, u) + dist(u, nxt) == dist(cur, nxt)):
                found = True
                total += dist(cur, u)
                break

            total += dist(cur, nxt)
            cur = nxt

        print(total if found else -1)
```该实现演示了核心结构：基于 LCA 的距离计算、循环遍历和路径成员资格检查。 完全优化的版本用基于线段树的最早命中搜索取代了线性扫描，但检测和累积的逻辑保持相同。 

重要的部分是使用距离进行路径成员资格测试，这确保我们正确检测节点何时位于树路径上的任何位置，而不仅仅是在端点处。

 ## 边缘情况

 当目标节点完全等于起始数组值时，会发生微妙的边缘情况。 在这种情况下，答案立即为零，因为步行已经在该节点上开始。 任何仅检查段的实现都会错过这一点，除非它在遍历开始之前显式处理初始位置。 

另一种情况发生在节点位于多个重叠线段上时。 该算法必须确保它按循环顺序返回最早的段，而不是仅返回距离最小的段。 这种区别至关重要，因为遍历时间是按顺序累积的。 

边界附近的更新$k$和$1$影响环绕段。 两个受影响的段都必须重新计算，否则循环结构会变得不一致，查询可能会完全错过有效路径。
