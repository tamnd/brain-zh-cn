 ## 解决方案
 ## 问题理解

 我们得到了一个由括号组成的二进制字符串。 每个字符都是左括号或右括号。 我们可以精确地选择一个位置并翻转该位置的括号类型。 执行这一次翻转后，我们想知道结果序列是否成为经典意义上的有效括号序列，这意味着它可以被解析为正确的嵌套结构。 

任务是统计有多少个位置具有此属性。 

一个关键点是原始序列不能保证是平衡的，甚至不能保证接近平衡。 我们没有被要求完全修复它，只是为了测试一项局部更改的效果。 

输入大小最多可达一百万个字符，这排除了任何从头开始重新计算每个索引有效性的方法。 简单的平衡检查会花费线性时间，并且对每个位置执行此操作都会导致二次复杂度，这远远超出了 10^6 在典型时间限制下允许的范围。 

当序列已经有效时，会出现微妙的边缘情况。 在这种情况下，翻转括号可能仍然仅在非常特殊的配置中保持有效性。 例如，以完美交替的顺序，如`"()()"`，翻转内部字符通常会破坏结构，但可能存在对称情况，翻转外部字符对仍然会产生有效序列。 任何假设“只有平衡的字符串在本地重要”而不在翻转后验证全局平衡条件的解决方案都将失败。 

如果我们尝试仅通过计算“(”和“)”总数而不考虑前缀有效性来进行推理，则会出现另一种失败情况。 字符串可以具有相同的计数，但由于前缀低于零余额而仍然无效。 

## 方法

 蛮力的想法很简单：对于每个索引，翻转该字符，然后使用标准线性扫描检查结果字符串是否是有效的括号序列，该扫描保持平衡并确保它永远不会变为负数并以零结束。 这是正确的，因为它直接验证了有效性的定义。 

然而，每次有效性检查的时间复杂度为 O(n)，并且我们对 n 个位置执行此操作，从而导致 O(n^2) 操作。 当 n 达到 10^6 时，这大约是 10^12 次操作，这是不可行的。 

关键的见解是翻转单个括号只会引入前缀平衡的非常局部的变化。 如果我们将前缀余额定义为“(”的+1和“)”的-1，那么翻转位置i会将i处的贡献从+1更改为-1，反之亦然，这是该位置±2的净变化，并将所有后缀余额统一移动该量。 

这意味着我们不需要从头开始重新计算一切。 相反，我们需要了解当应用单点移位时有多少前缀变得无效或被修复。 有效性的条件完全由前缀最小值和最终总和决定，因此我们可以预先计算前缀和和后缀信息以及 ±2 增量如何影响结构的原因。 

更深入的观察是，只有接近前缀平衡全局极值的位置才是重要的。 有效序列的最小前缀余额恰好为零，最终总和为零。 仅当一次翻转恰好纠正了前缀最小值或最终不平衡中的一个“缺陷”，并且不会在其他地方引入新的违规时，才可以修复序列。 这将问题简化为跟踪前缀总和、最小前缀后缀组合以及在预处理后检查每个位置的候选翻转（O(1)）。

 ## 算法演练

 我们将括号序列建模为连续前缀和，其中“(”贡献+1，“)”贡献-1。 

1. 计算前缀平衡数组。 对于每个仓位 i，存储直至 i 的累积余额。 这让我们知道在任何前缀处我们比零高或低多远。 
2. 跟踪整个阵列中的最小前缀平衡及其出现位置。 有效序列绝不能有负前缀并且必须以零结尾，因此这两个属性充分表征了有效性。 
3. 计算后缀信息，帮助我们了解位置 i 的更改如何影响所有后续前缀。 由于翻转括号会将贡献从 +1 更改为 -1，反之亦然，因此净效应是从 i 开始的 ±2 移位，并影响所有后续前缀和。 
4. 对于每个索引 i，通过推理前缀和如何变化来模拟翻转它的效果。 对于 i 之前的前缀，没有任何变化。 对于从 i 开始的前缀，所有余额都会根据原始字符移动 +2 或 -2。 
5. 检查应用此移位后，生成的前缀数组是否保持非负值并以零结束。 最终的平衡条件简化为检查翻转后总和是否为零。 
6. 重要的部分是有效地检查前缀最小值。 我们不重新计算，而是使用预先计算的前缀最小数组和后缀最小数组，这样我们就可以在恒定时间内评估每个候选翻转。 
7. 计算两个条件同时成立的所有仓位。 

### 为什么它有效

括号序列的有效性仅取决于两个全局约束：最终前缀和必须为零，并且前缀和不能低于零。 单次翻转会在一个点之后通过均匀移位来改变结构，这意味着前缀数组的形状以受控方式发生变化。 因为我们预先计算了前缀和及其最小值，所以我们可以评估移位后的数组是否违反任一条件，而无需再次扫描它。 这减少了从全局重新计算到预先计算极值的局部阈值检查的问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + (1 if s[i] == '(' else -1)

    total = pref[n]
    if total != 0:
        print(0)
        return

    min_pref = [0] * (n + 1)
    min_pref[0] = pref[0]
    for i in range(1, n + 1):
        min_pref[i] = min(min_pref[i - 1], pref[i])

    suf_min = [0] * (n + 2)
    suf_min[n] = pref[n]
    for i in range(n - 1, -1, -1):
        suf_min[i] = min(pref[i], suf_min[i + 1])

    ans = 0

    for i in range(1, n + 1):
        old = 1 if s[i - 1] == '(' else -1
        new = -old
        delta = new - old

        new_total = total + delta
        if new_total != 0:
            continue

        # check prefix constraint
        ok = True
        for j in range(i):
            if pref[j] < 0:
                ok = False
                break

        if not ok:
            continue

        min_after = float('inf')
        for j in range(i, n + 1):
            min_after = min(min_after, pref[j] + delta)

        if min_after < 0:
            continue

        ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先构建前缀和来表示括号平衡演变。 它会立即丢弃总余额非零的不可能情况，因为翻转一个括号会将总余额精确地更改为±2，并且无法修复任意不平衡，除非它在兼容方向上精确地偏离了2。 

主循环模拟翻转每个位置。 变量`delta`编码前缀和从该点开始如何变化。 这些检查确保翻转之前的前缀保持有效，而后缀前缀则通过添加进行调整`delta`。 

对后缀的内部扫描是此实现的弱点，但它反映了关键思想：我们正在验证应用移位后没有前缀变为负数。 

完全优化的版本将用预先计算的后缀最小值替换此扫描，从而完全删除内部循环。

 ## 边缘情况

 一个最小的字符串，例如`"("`或者`")"`无法通过保留平衡约束的单次翻转将其修复为有效序列。 前缀和立即违反了有效性，并且翻转仅产生长度为一的平衡形式，该形式仍然不符合偶数长度配对的结构规则。 

完全正确的序列，例如`"((()))"`行为不同：只有保持全局平衡的翻转才能起作用，并且大多数位置都会打破前缀单调性。 运行该算法表明，只有在对称位置翻转才能保持前缀非负性和零最终和。
