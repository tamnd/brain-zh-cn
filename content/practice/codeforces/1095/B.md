 ## 解决方案
 ## 问题理解

 我们得到了一个代表数组的数字列表，并且允许我们恰好删除一个元素。 删除它后，我们看看剩余数字是如何“分散”的，定义为最大和最小剩余值之间的差。 

任务是选择要删除哪个单个元素，以使该分布变得尽可能小，然后输出可实现的最小分布。 

关键限制是数组可以很大，最多可包含 100,000 个元素。 这立即排除了任何尝试删除每个元素并在每次删除的线性时间内从头开始重新计算最小值和最大值的方法，因为这会导致粗略的结果$O(n^2)$在最坏的情况下工作。 相反，我们需要计算一次有关全局结构的内容并重用它。 

当极值多次出现时，就会出现微妙的边缘情况。 例如，如果最小值出现多次，则仅删除一次出现不会更改最小值。 这同样适用于最大值。 这很重要，因为最佳去除通常针对极端情况之一，但并非总是如此。 

考虑数组`[1, 100, 100, 100]`。 去除`1`提供传播`100 - 100 = 0`。 去除`100`提供传播`100 - 1 = 99`。 如果没有正确考虑极值的重复项，则仅考虑删除排序顺序中的第一个或最后一个元素的简单方法将会失败。 

当所有值都相等时，会出现另一种边缘情况。 然后任何删除仍然留下所有相等的值，答案为零。 

## 方法

 暴力破解的想法很简单：对于每个索引，模拟删除该元素，计算剩余数组的最小值和最大值，并跟踪最小的结果差异。 计算每次移除成本后的最小值和最大值$O(n)$，并为所有人做这件事$n$头寸收益率$O(n^2)$。 和$n = 10^5$，这远远超出了可行的限度。 

问题的结构使得更有效的策略成为可能。 不稳定性仅取决于剩余元素的最小值和最大值。 当我们删除一个元素时，只有两件事可以改变答案：要么我们删除一些不是极值的东西，而最小/最大保持不变，要么我们删除一个极值，并可能将最小或最大移动到第二小或第二大元素。 

这表明我们只需要知道最小和第二小的值，以及最大和第二大的值。 如果我们删除中间的元素，则 min 和 max 保持不变，因此结果不变。 如果我们删除当前的最小值，新的最小值将成为第二个最小值。 如果我们删除当前的最大值，新的最大值将成为第二个最大值。 

因此，最佳答案必须仅来自两个候选：删除最小值或删除最大值。 任何其他删除都无法改善范围，因为它会使两个极值保持完整。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 扫描数组一次，确定最小值、次小值、最大值、次大值。 这四个数字充分描述了去除极端元素后范围如何变化。 
2. 将原始范围计算为`max_value - min_value`。 这代表了我们删除非极端元素或实际上没有改进任何东西的情况。 
3.考虑删除最小元素。 移除后，最小的剩余值变为第二小的值，因此结果范围变为`max_value - second_min`。 
4.考虑删除最大元素。 移除后，最大的剩余值变为第二大，因此结果范围变为`second_max - min_value`。 
5. 取这三个值中的最小值：原始范围、删除最小值和删除最大值。 这是移除一个元素后可实现的最佳不稳定性。 

### 为什么它有效

 阵列的不稳定性仅取决于其极值。 删除非极端元素无法更改范围的任一端点，因此无法改善结果。 删除极值仅更改范围的一侧，并且该侧的下一个候选值由第二小或第二大元素唯一确定。 由于每个有效的删除都属于这些类别之一，因此彻底检查这些案例涵盖了所有可能性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

INF = 10**18

min1 = min2 = INF
max1 = max2 = -INF

for x in a:
    # update smallest and second smallest
    if x < min1:
        min2 = min1
        min1 = x
    elif x < min2:
        min2 = x

    # update largest and second largest
    if x > max1:
        max2 = max1
        max1 = x
    elif x > max2:
        max2 = x

original = max1 - min1
remove_min = max1 - min2
remove_max = max2 - min1

print(min(original, remove_min, remove_max))
```该实现对数组执行一次传递，维护四个运行值。 更新逻辑仔细地保留每个极端的最佳和次佳候选者，确保当假定删除极端时，正确的替换值是已知的。 

最后一步评估三个有意义的配置。 没有明确考虑其他删除​​，因为它们不能改变范围的任何边界。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))

    INF = 10**18

    min1 = min2 = INF
    max1 = max2 = -INF

    for x in a:
        if x < min1:
            min2 = min1
            min1 = x
        elif x < min2:
            min2 = x

        if x > max1:
            max2 = max1
            max1 = x
        elif x > max2:
            max2 = x

    return str(min(max1 - min1, max1 - min2, max2 - min1))

# provided sample
assert run("4\n1 3 3 7\n") == "2"

# all equal
assert run("5\n5 5 5 5 5\n") == "0"

# remove min best case
assert run("4\n1 100 100 100\n") == "0"

# remove max best case
assert run("4\n1 2 3 4\n") == "2"

# minimal size
assert run("2\n1 10\n") == "0"

 ## 边缘情况

 一种重要的情况是最小值或最大值出现多次。 用于输入`[1, 1, 5, 10]`，去掉一个`1`不改变最小值，所以第二个最小值仍然是`1`。 该算法处理这个问题是因为`min2`也将是`1`， 所以`max - min2`正确反映范围没有变化。 

另一种情况是，最佳走法并不明显来自位置，而是来自价值重复。 在`[1, 2, 3, 4]`，去除`2`或者`3`没有任何作用来提高范围，同时删除`1`或者`4`给出较小的范围。 该算法仅通过评估极端移除来捕获这一点，确保永远不需要显式测试非极端移除。 

最后，对于大小为 2 的数组，删除一个元素会留下一个值，从而使不稳定性为零。 该公式正确减少，因为任一极端删除都会导致比较相同的剩余端点。
