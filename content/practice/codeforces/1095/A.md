 ## 解决方案
 ## 问题理解

 我们得到一个字符串，该字符串是通过应用于某些隐藏的原始字符串的非常特定的“拉伸重复”规则生成的。 原始字符串很短，长度最多为10，但我们不知道。 我们得到的是每个字符重复与其在原始字符串中的位置相同的次数后的最终扩展字符串。 

具体来说，如果隐藏字符串是`s = s1 s2 ... sm`，然后是第一个字符`s1`加密字符串中出现一次，第二个字符`s2`连续出现两次，第三个字符出现三次，依此类推。 因此，加密字符串是长度按 1, 2, 3, ..., m 增长的块的串联。 

任务是从这个扩展形式恢复原始字符串。 

约束很小：加密字符串长度最多为 55，而原始字符串隐式长度最多为 10，因为 1 + 2 + ... + 10 = 55。这立即告诉我们，任何在线性时间内扫描或模拟的解决方案都足够了。 即使是最多 10 层的嵌套循环在性能方面也微不足道，但没有必要。 

主要的边缘情况是结构性的而不是计算性的。 由于块大小不断增加，错误的解决方案通常会在假设段大小相等或尝试贪婪地分割而不遵守严格的 1、2、3、... 增长模式时失败。 另一个微妙的问题是离一索引：第 k 个字符属于大小为 k 的块，而不是 k-1 或 k+1，这会默默地破坏重建。 

## 方法

 暴力破解的视角首先想象我们尝试猜测原始字符串并模拟加密。 由于原始长度最多为 10，因此每个位置有 26 种可能性，因此有 26^10 个潜在候选者。 对于每个候选者，我们可以在 O(m^2) 时间内构建其加密形式（因为我们为位置 i 附加 i 个副本），从而提供了天文数字般的搜索空间。 即使进行了修剪，考虑到结构，这也是完全没有必要的。 

关键的观察结果是，加密过程以非常结构化的方式是确定性和单射的：加密的字符串只是已知长度块的串联。 第一个块的大小为 1，第二个块的大小为 2，依此类推。 因此，我们不必猜测原始字符串，而是可以通过遍历加密字符串并切出大小不断增加的块来直接对其进行解码。 

这将问题变成了一个简单的解析任务：维护一个指向字符串的指针，提取 1 个字符，然后提取 2 个字符，然后提取 3 个字符，依此类推，并记录每个块的第一个字符。

 由于需要猜测字符串，在最坏的情况下，暴力方法的效率呈指数级增长。 最佳方法与加密字符串的长度呈线性关系。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(26^m · 平方米) | O(米) | 太慢了 |
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们通过利用密码的固定块结构来重建原始字符串。 

1. 初始化一个指针`i = 0`跟踪我们在加密字符串中的位置。 我们还维护一个块大小计数器`k = 1`。 
2. 虽然`i`小于字符串的长度，我们知道原始编码中的下一个块有大小`k`，所以我们取从开始的子串`i`长度`k`。 
3. 生成该块的原始字符必须是该子字符串的第一个字符，因为该块中的所有字符在结构上都是相同的。 
4. 将该字符附加到答案中。 
5. 将指针向前移动`k`，并增加`k`加 1 以匹配递增重复规则。
 6. 继续，直到绳子完全耗尽。 

关键思想是我们永远不需要验证超出一致性的块内容，因为问题保证了输入是有效的。 

### 为什么它有效

 在每个步骤中，加密过程都会生成一个由相同字符组成的连续块，其长度完全由其在原始字符串中的位置决定。 由于块是不重叠的并且按大小递增严格排序，因此块之间的边界由序列 1, 2, 3, ..., m 唯一确定。 因此，在解码时，块边界是固定且明确的，选择每个块的第一个字符可以准确地重建原始字符串。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
t = input().strip()

res = []
i = 0
k = 1

while i < n:
    res.append(t[i])
    i += k
    k += 1

print("".join(res))
```该解决方案读取加密的字符串并使用两个变量遍历它：`i`，当前位置，以及`k`，预期的块长度。 At each iteration, we extract the character at position`i`作为当前块的代表。 然后我们向前跳`k`字符，因为加密保证这一整个段对应于一个重复的字符。 递增`k`确保我们尊重 1、2、3……增长模式。 

不需要子字符串验证，因为保证输入是有效的编码。

 | 测量| 复杂性 | 说明|

 | 时间 | O(n) | Each character is visited exactly once as we jump through blocks |
 | 空间| O(1) | O(1) | 除了输出存储之外，只使用了几个计数器 |

 这些约束将输入大小限制为 55，因此即使效率较低的线性扫描也足够了。 该解决方案是最优的并且在限制范围内舒适。

 ## 边缘情况

 微妙的边缘情况是字符串长度为 1 的最小可能输入。在这种情况下，算法从`k = 1`, reads exactly one character, and terminates immediately. 不存在越界的风险，因为指针增量与剩余长度完全匹配。 

另一种情况是所有字符都相同。 例如，`t = "aaaaa..."`。 该算法仍然有效，因为它从不依赖于字符差异，仅依赖于块长度。 每个步骤都会消耗一个不断增长的段，并且输出只是为每个块重复相同的字符。 

最后一种情况是最大长度为 55。由于 1 + 2 + ... + 10 = 55，算法将在 10 次迭代后自然终止。 不存在部分块歧义，因为块大小的总和与字符串长度完全匹配。
