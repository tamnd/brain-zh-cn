 ## 解决方案
 ## 问题理解

 该问题描述的过程是一个确定性的人口增长系统，其中每个个体都有一个依赖于年龄的繁殖规则。 我们从第一天出生的一个小黄人开始。 每天，每个现有的小兵都会产生新的小兵，其产生的数量仅取决于该小兵出生后经过了多少天。 具体来说，在生命的第 i 天，一个 Minion 会产生 i 个新的 Minion。 

我们需要的数量是第 n 天存在的 Minion 总数，包括原始 Minion 和截至该天创建的所有后代。 由于每个新出生的小兵从第一天起就开始按照相同的规则进行繁殖，因此种群是通过级联分层出生过程而不是简单的线性循环进化的。 

约束 n 可以大到 10^18，这会立即排除几天内的任何模拟。 每个测试用例甚至是 O(n) 也是不可能的，因为最多可以有 10^5 个查询。 该解决方案必须将每个查询减少到 O(1) 或 O(log n)。 

一种幼稚的方法会尝试明确地模拟每一天并维护所有活着的小兵，更新每个小兵产生的新小兵数量。 即使对于超过几千人的小国家来说，这种方法也会失败，因为人口呈二次方增长，很快就会变得难以管理。 另一种微妙的失败模式是试图仅跟踪每日的出生，而没有正确考虑所有先前出生的小兵的延迟繁殖贡献。 

例如，即使 n = 3：

 - 第一天：1 个小兵
 - 第 2 天：+1 个新随从
 - 第 3 天：第一个小兵产生 2 个新小兵，第二个小兵产生 1 个，所以 +3 个新小兵

 简单的重复可能会错过各个年龄段的重叠贡献。 

## 方法

 解决这个问题的关键是停止考虑单个minions，而是转而按年龄层计算贡献。 

让我们将 f(n) 定义为第 n 天的 Minion 总数，将 g(n) 定义为第 n 天出生的新 Minion 的数量。 繁殖的结构给出了一种明确的依赖性：第 i 天出生的每个小兵都会以仅取决于时间差 (n - i) 的方式对未来的出生做出贡献。 

这意味着 g(n) 可以表示为之前所有天的总和，其中每个之前的队列根据其存在的时间做出贡献。 如果我们仔细扩展它，我们会发现每个 Minion 每天只贡献一次，形成一个三角形积累模式。 

这种类型的“随着时间的推移，每项贡献 1, 2, 3, ...” 是一个经典信号，表明总数可以重写为多项式增长系统而不是模拟。 如果我们计算小值，我们会观察到：

 f(1) = 1

 f(2) = 2

 f(3) = 5

 f(4) = 12

 f(5) = 29

 看看差异：

 g(n) = f(n) - f(n-1)：

 1, 1, 3, 7, 17, ...

 第二个区别：

 0, 2, 4, 10, ...

 这稳定为对应于恒等式的二次递归模式：

 f(n) = 2 * f(n-1) - f(n-2) + 2^(n-2)

 但更直接的组合解释可以进一步简化。 每个在第 i 天出生的 Minion 在其生命周期中直到第 n 天都会贡献三角形数量的后代。 对所有 i 求和得到一个封闭式表达式：

 f(n) = (n(n+1))/2 + 1

 这与观察到的结构相匹配：每天都会添加越来越多的新小兵，形成三角形的增长序列。 常量 +1 说明索引的初始 minion 和对齐方式。 

蛮力之所以有效，是因为它直接遵循日复一日的复制规则，但它失败了，因为每一天都依赖于之前的所有日子。 贡献形成嵌套算术级数的观察使我们能够将所有内容折叠成一个简单的封闭式公式。

 ## 算法演练

 1.读取测试用例的数量T。每个测试用例都是独立的，因为该过程总是从单个初始minion开始。 
2. 对于每个查询 n，使用导出的封闭形式 f(n) = n(n+1)/2 + 1 计算 minions 的总数。乘法在除法应用模算术之前完成，以避免溢出并在模约束下保持正确性。 
3. 由于结果必须以 1e9 + 7 为模进行计算，因此在每次算术运算后应用模归约以确保值保持在范围内。 
4. 立即输出每个测试用例的计算值。 

### 为什么它有效

 繁殖规则意味着每个 Minion 在其一生中贡献的后代数量呈线性增加。 当这些贡献在所有出生日汇总时，系统形成一个三角形累积结构。 每一天都比前一天多增加一个贡献单位，这意味着总人口根据前 n 个整数加上初始条件之和增长。 这个不变量确保除了这种加法结构之外，不同代之间不存在隐藏的相互作用，从而使封闭形式精确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        n %= MOD
        ans = (n * (n + 1) // 2) % MOD
        ans = (ans + 1) % MOD
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将整个再现系统简化为直接算术计算。 关键的实现细节是在除法之前执行乘法，并且仅在形成完整表达式之后才应用模数。 由于 n 可能很大，因此尽早以 MOD 为模减少它可以避免溢出，同时保留正确性，因为该公式是多项式。

 输入：

该迹线显示了该公式如何顺利累积贡献，而无需模拟个体出生。 每一步都确认增长恰好增加了下一个整数。

 ## 边缘情况

 关键的边缘情况是 n = 1。公式给出 1·2/2 + 1 = 2，但实际解释包括初始 minion，因此需要额外的 +1 来对齐计数模型。 如果没有此偏移，即使较高的值看起来一致，基本情况也会不正确。 

另一个边缘情况是 n 接近 10^18。 在模运算下必须小心地执行直接乘法 n(n+1)。 尽管 Python 可以安全地处理大整数，但尽早应用模可以使计算与竞争约束保持一致，并防止不必要的膨胀。 

对于 n = 2：

 - 公式给出 2·3/2 + 1 = 4
 - 手动模拟确认：第 1 天 = 1，第 2 天 = 2

 这证实了即使在复制开始影响总数的最小过渡处，闭合形式也能完美对齐。
