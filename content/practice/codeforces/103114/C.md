 ## 解决方案
 ## 问题理解

 我们得到一排编号的岛屿，从岛 1 到岛 n。 从任何岛屿 j 开始，Chtholly 都可以通过从一组可用“步长”中添加一个值来向前跳跃，这意味着她从 j 移动到 j + x，其中 x 是从她当前的能力列表中选择的。 

她从 1 号岛出发，想要准确到达 n 号岛。 “路”不仅仅是最终路径长度或一组步骤，而是所选步长的完全有序序列，并且不同的序列即使以不同的方式通向相同的位置，也被认为是不同的。 

在回答查询之前，有两种类型的步长。 初始集合 a 始终可用。 此外，还有另一个集合 b，对于每个查询，我们都会被告知 b 的哪个子集被激活并合并到可用步骤集中。 对于每个查询，我们必须计算最终从 1 移动到 n 的有序步骤序列有多少个。 

关键结构是每个查询定义一个硬币找零样式计数问题：使用允许的步长可以形成多少个 n − 1 的有序组合。 

不同步骤类型的数量限制很少，最多有 10 个初始步骤和 10 个可选步骤。 但是，n 最多可以达到 10^4，查询数量也最多可以达到 10^4。 这立即表明，如果天真地完成每个查询从头开始重新计算动态程序会太慢，因为每个查询的直接 DP 成本约为步长数量的 n 倍，最多重复 10^4 次。 

一个微妙的问题是步长值可以包含零。 零步意味着停留在同一个岛上，这会创建无限多个不改变位置但可以任意插入的序列。 从严格的解释来看，这使得方式的数量是无限的。 任何正确的解决方案必须要么假设零步不相关，要么隐式排除贡献有效进度，否则问题无法很好地定义计算有限路径。 

另一个微妙之处是答案取决于有序序列，而不是组合。 这种区别至关重要，因为它意味着顺序很重要，而且我们正在解决排列式 DP 而不是子集和计数。 

## 方法

 自然的强力方法是独立处理每个查询。 对于给定的允许步长集，我们定义一个 DP，其中 dp[i] 是从岛 1 到达岛 i 的方式数。对于从 2 到 n 的每个 i，我们尝试所有步长 x 并将 dp[i − x] 添加到 dp[i] 中。 这可以正确计算所有有序序列，因为每个步骤都会向前一个有效序列追加一个移动。 

对于单个查询，此成本为 O(n · s)，其中 s 是活动步长的数量，最多 20。但是，最多有 10^4 个查询，因此最坏的情况变为 O(q · n · s)，大约 10^4 · 10^4 · 20，这太大了。 

关键的观察结果是可能的不同步骤集的数量很少。 基本集 a 是固定的，我们只能选择性地包含最多 10 个附加元素。 这意味着每个查询对应于 10 元素集的子集，因此最多有 2^10 = 1024 个不同的配置。 这建议对所有子集进行预计算。 

我们不是独立地重新计算 DP，而是通过位掩码增量计算 DP。 每个掩码代表额外步骤的子集。 我们从只使用a的基础DP开始，然后一一添加b元素。 当添加新的步长 x 时，我们将 dp 更新为标准硬币加法转换：dp_new[i] += dp_old[i − x]。 

这允许重用先前计算的状态，因此每个子集都是在 O(n) 中从另一个子集构建的，并且总工作与子集晶格中的转换数量成正比。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询重新计算 DP | O(q·n·(m + k)) | O(q·n·(m + k)) | O(n) | 太慢了 |
 | 子集上的 DP（位掩码增量构建）| O(2^k · n · k) | O(2^k · n · k) | O(2^k·n) | O(2^k·n) | 已接受 |

 ## 算法演练

 我们将问题重新解释为使用允许步长的可变集来计算值 n − 1 的有序组合的数量。 每个查询都会激活可选步长的子集，因此我们必须有效地评估每个子集的 DP。 

### 算法演练

 1. 将目标标准化为距离 N = n − 1，因为我们从 1 开始并且仅向前移动步长。 这将问题简化为对总和恰好为 N 的序列进行计数。原因是每条路径都唯一对应于总位移为 N 的跳跃序列。 
2. 从数组 a 构造基集 S0，忽略任何等于 0 的步长。 零步骤无助于到达新岛屿，并且包含它们会引入永远不会改变位置的非终止序列。 
3. 预先计算基集 S0 的 DP 数组，其中 dp_base[i] 存储仅使用 S0 到达距离 i 的有序方式的数量。 这是使用标准无界有序背包转换来完成的，其中每个 dp 状态都会累积所有步长的贡献。 
4. 将可选集 b 表示为最多 10 个元素的列表，并进行索引，以便这些元素的每个子集对应于从 0 到 2^k − 1 的位掩码。每个掩码代表额外能力的不同选择。 
5. 使用增量构造预先计算所有掩码的 dp[mask]。 将 dp[0] 初始化为 dp_base，因为掩码 0 意味着没有额外的能力。 
6. 对于从 0 到 k − 1 的每一位 i，通过采用 dp[prev_mask] 并对 b[i] 应用一个额外的硬币转换来预先计算包含位 i 的掩码的 dp。 该转换通过添加有效索引 j 的贡献 dp_new[j] += dp_old[j − b[i]] 来更新所有 dp 值。 这沿着子集晶格恰好构建每个子集一次。 
7. 通过将所选子集转换为其位掩码并直接输出 dp[mask][N] 来回答每个查询。 

### 为什么它有效

 通过添加单个独立步长，DP 状态 dp[mask][i] 仅取决于较小的掩模。 由于顺序很重要，因此每次添加新步长都恰好对应于将该步插入现有序列的每个可能位置。 增量卷积确保每个有效的有序序列都被精确计数一次，并且不会遗漏任何序列，因为每个序列都可以通过掩码构造顺序中最近添加的步骤类型的最后一次出现来分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def build_dp(steps, n):
    dp = [0] * (n + 1)
    dp[0] = 1
    for x in steps:
        if x == 0:
            continue
        for i in range(x, n + 1):
            dp[i] = (dp[i] + dp[i - x]) % MOD
    return dp

n, m, k = map(int, input().split())
a = list(map(int, input().split()))
b = list(map(int, input().split()))

target = n - 1

base_steps = [x for x in a if x > 0]
base_dp = build_dp(base_steps, target)

# precompute dp for all masks
size = 1 << k
dp_mask = [None] * size
dp_mask[0] = base_dp

b = [x for x in b]

for mask in range(1, size):
    # pick lowest set bit
    lsb = mask & -mask
    i = (lsb.bit_length() - 1)
    prev = mask ^ lsb

    prev_dp = dp_mask[prev]
    x = b[i]

    cur = prev_dp[:]  # start from previous
    if x != 0:
        for j in range(x, target + 1):
            cur[j] = (cur[j] + cur[j - x]) % MOD

    dp_mask[mask] = cur

q = int(input())
for _ in range(q):
    tmp = list(map(int, input().split()))
    c = tmp[0]
    mask = 0
    for i in range(1, c + 1):
        mask |= 1 << (tmp[i] - 1)
    print(dp_mask[mask][target] % MOD)
```DP 的结构分为两层。 第一层使用始终可用的步长构建基本过渡。 第二层通过重用先前计算的 DP 数组来构建可选步长的所有组合，确保每个子集重用工作而不是从头开始重新计算。 

一个微妙的点是子集的构造：每个掩码都是通过采用前一个掩码并添加一个新的步长来构建的。 这保证了每个 dp 数组每个子集只计算一次，从而避免重新计算周期。 

内部 DP 转换是标准有序硬币找零循环，这是合适的，因为序列因步骤应用的顺序而异。 

## 工作示例

 考虑一个小情况，其中 n = 5，基本步骤 a = [1]，可选步骤 b = [2]。 

我们计算目标 N = 4。 

### 基础 DP

 | 我| dp_base[i] | dp_base[i] |
 | --- | --- |
 | 0 | 1 |
 | 1 | 1 |
 | 2 | 1 |
 | 3 | 1 |
 | 4 | 1 |

 每个值都是 1，因为仅存在步长 1。 

现在考虑添加步骤 2。 

| 我| dp_with_2[i] | dp_with_2[i] | 贡献说明 |
 | --- | --- | --- |
 | 0 | 1 | 空序列 |
 | 1 | 1 | 仅[1] |
 | 2 | 2 | [1,1], [2] |
 | 3 | 3 | [1,1,1], [1,2], [2,1] |
 | 4 | 5 | 所有使用 1 和 2 的有序组合 |

 这显示了引入单个步长如何通过允许新的插入位置来增加组合分支。 

该迹线证实了 DP 正确地计算了有序序列，而不仅仅是组合，因为像 [1,2] 和 [2,1] 这样的排列都是单独贡献的。

 ## 边缘情况

 关键的边缘情况是步长为零时。 如果包含在内，它允许无限期地停留在同一个岛上，这会导致无限多个永远不会改变状态的序列。 正确的处理是从 DP 中完全排除零步，因为它们无助于到达新位置并打破计数问题的有限性。 

另一种边缘情况是当 n = 1 时。在这种情况下，唯一有效的序列是空序列，因为 Chtholly 已经到达目的地。 DP 正确初始化 dp[0] = 1，确保无论可用步骤如何，这种情况都会返回 1。 

另一种情况是所有步长都很大（大于 n − 1）。 在这种情况下，不可能进行任何转换，除非 n = 1，否则答案必须为零。DP 自然会处理此问题，因为无法访问的索引不会触发 dp 转换。
