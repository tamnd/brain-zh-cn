 ## 解决方案
 ## 问题理解

 该任务描述了一组城市的交通网络，其中每个城市包含四种内部“存在模式”：学校、火车站、机场和比赛场地。 在一个城市内，从这些位置中的任何一个移动到任何其他位置都需要花费固定金额`x`并需要固定的时间`y`。 

城市之间有两种类型的双向连接：铁路和航空。 每条这样的边都连接两个城市，并有自己的成本和时间。 你只能使用这些边缘在城市之间穿梭，但在城市内部你可以在四个内部角色之间自由切换，每次切换都要支付固定的城内转换成本。 

旅程始终从城市 1 的学校开始，并必须在城市 n 的比赛地点结束。 在所有有效路线中，我们首先必须最小化总货币成本。 只有当多条路线共享相同的最低成本时，我们才能最大限度地减少总时间。 

从根本上来说，这是一个最短路径问题，但不是简单的图中的问题：每个城市都会扩展到具有额外过渡边的多个内部状态。 

这些约束清楚地表明，任何解决方案在每个测试用例中都必须几乎呈线性表现。 由于城市和边的总数最多为 10^6，因此需要 O((n + m + l) log n) Dijkstra 式解决方案。 任何尝试构建一个每个城市有四个节点和密集的城市内连接的显式完全扩展图仍然没问题，但任何比对数线性图遍历更糟糕的事情都会失败。 

一个微妙的点是，城市内的移动不是在特定的固定对之间，而是有效地允许四个角色中的任何两个角色之间的移动。 这意味着将其天真地建模为每个城市 6 个边是正确的，但我们必须确保最短路径逻辑自然地处理重复的转换。 

另一个重要的细节是词典优化：成本是首要的，时间是次要的。 单纯考虑成本的 Dijkstra 将无法按时执行正确的决胜局，而单纯的时间独立优化将是错误的，因为它忽略了成本优先级。 

当底层城市间图中城市 1 和城市 n 之间没有路径时，即使存在城市内转移，也会出现边缘情况。 在这种情况下，输出必须为-1。 另一个边缘情况是，最佳路线需要多次城内转换，例如在乘坐铁路之前从学校切换到火车站，然后在到达后再次切换。 

## 方法

 暴力解释是明确构建一个图，其中每个城市扩展到代表四个角色的四个节点。 在每个城市内部，我们将这四个节点中的每一对与成本边缘连接起来`x`和时间`y`。 对于城市之间的每条铁路或航空路线`u`和`v`，我们将每个角色联系起来`u`到每一个角色`v`，因为一旦您到达交通枢纽，城市之间的旅行并不取决于内部位置类型。 这将创建一个图表`4n`节点和粗略`6n + 16(m + l)`边缘。 

在这张图上，我们需要在（成本，时间）字典序最小化的情况下，从“城市1的学校”节点到“城市n的比赛场地”节点的最短路径。 优先级队列存储的简单 Dijkstra`(cost, time, node)`工作正常，因为排序自然会强制执行成本优先，然后是时间。 

蛮力是正确的，但其效率低下是因为推动了太多的状态转换并反复放松了密集的城市内边缘集。 虽然在技术上边缘仍然是线性的，但常数因子变得很大，更重要的是它掩盖了一个更简单的结构：城市内的运动是均匀的，并且不取决于我们处于四个角色中的哪一个，因此我们不需要以完全对称的方式将它们视为不同的逻辑状态。 

关键的观察是，城市内的四种角色对于离开城市的目的是等效的。 每当我们身处城市之中，重要的是我们是否愿意付出城市内的成本，将自己“重新定位”到一个方便的状态，然后再取得优势。 这意味着我们可以在概念上将每个城市视为一个节点，但允许代表付费的自我转换`x, y`“重置”城市内部的状态。 通过这种压缩，我们将图缩小为`n`具有原始边加上自环的节点。 

这使我们能够直接在城市上运行标准词典 Dijkstra，其中每条边代表一条运输路线或城市内的移动。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 扩展 4 状态图 + Dijkstra | O((n + m + l) log n) | O((n + m + l) log n) | O(n + m + l) | O(n + m + l) | 已接受 |
 | 压缩城市图 + Dijkstra | O((n + m + l) log n) | O((n + m + l) log n) | O(n + m + l) | O(n + m + l) | 已接受 |

 实际上，两者渐近相似，但压缩解释更简单并且避免了不必要的状态扩展。 

## 算法演练

 我们将每个城市建模为单个节点，并维护成对的最短路径`(cost, time)`。 

1. 初始化一个距离数组，其中每个条目存储`(infinite_cost, infinite_time)`。 将起始城市 1 设置为`(0, 0)`因为我们从那里的学校开始。 
2. 使用按字典顺序排序的优先级队列`(cost, time, node)`。 这确保了每当我们提取一个州时，它都是当前到达该城市的最著名的方式。 
3. 插入`(0, 0, 1)`进入优先队列。 
4. 当队列不为空时，提取状态`(c, t, u)`。 如果此状态比已记录的最佳状态更差`u`，跳过它。 这可以避免处理过时的路径。 
5. 对于每条铁路或航空路线`(u, v, cost_e, time_e)`，尝试放松`v`使用`(c + cost_e, t + time_e)`。 如果该对按字典顺序比存储的值更好`v`，更新它并将其推入队列。 
6. 此外，允许城市内转机：从城市`u`，我们可以支付`(x, y)`在城市内部“重新配置”并留在`u`。 如果`(c + x, t + y)`提高储值`u`，推一下。 该模型在从边缘到达之前或到达之后内部角色之间的切换进行建模。 
7. 继续，直到队列耗尽。 

答案是 city 中存储的对`n`。 如果仍为无穷大，则输出`-1`。 

正确性取决于这样一个事实：原始问题中的每一个有意义的操作都可以简化为穿越运输边缘或支付重新定位的城市内成本。 原始四层结构中的任何运动序列都与这些松弛的序列完全对应。 

不变的是，每当一个城市从优先级队列中弹出时，存储的`(cost, time)`是到达该城市的所有路径中按字典顺序可实现的最小路径对。 由于所有边缘松弛都保持非负性，因此即使在字典顺序下，Dijkstra 的贪心提取仍然有效。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline
INF = (10**30, 10**30)

def better(a, b):
    return a[0] < b[0] or (a[0] == b[0] and a[1] < b[1])

def solve():
    T = int(input())
    for _ in range(T):
        n, m, l, x, y = map(int, input().split())

        g = [[] for _ in range(n + 1)]

        for _ in range(m):
            u, v, a, b = map(int, input().split())
            g[u].append((v, a, b))
            g[v].append((u, a, b))

        for _ in range(l):
            u, v, a, b = map(int, input().split())
            g[u].append((v, a, b))
            g[v].append((u, a, b))

        dist = [INF] * (n + 1)
        dist[1] = (0, 0)

        pq = [(0, 0, 1)]

        while pq:
            c, t, u = heapq.heappop(pq)
            if (c, t) != dist[u]:
                continue

            if c + x < dist[u][0] or (c + x == dist[u][0] and t + y < dist[u][1]):
                dist[u] = (c + x, t + y)
                heapq.heappush(pq, (c + x, t + y, u))

            for v, a, b in g[u]:
                nc, nt = c + a, t + b
                if nc < dist[v][0] or (nc == dist[v][0] and nt < dist[v][1]):
                    dist[v] = (nc, nt)
                    heapq.heappush(pq, (nc, nt, v))

        if dist[n] == INF:
            print(-1)
        else:
            print(dist[n][0], dist[n][1])

if __name__ == "__main__":
    solve()
```该实现将每个城市视为一个节点，并使用以成本和时间为关键字的优先级队列。 这`dist`数组存储每个城市最知名的词典对。 

微妙的部分是城市内部的转变。 我们没有明确地建模四个内部节点，而是注入了一种放松，允许在添加的情况下留在同一个城市`(x, y)`。 这体现了我们在乘坐任何交通工具之前始终可以在城市内重新定位的想法。 

优先级队列检查`(c, t) != dist[u]`至关重要，因为堆中可以存在同一城市的多个州。 如果没有这种保护，过时的状态将导致多余的放松并可能降低性能。

 ## 边缘情况

 一个关键的边缘情况是城市 1 没有外出运输边缘。 该算法仍然有效，因为仅城市内的转换无法到达其他城市，因此所有松弛都保留在节点 1 内，并且堆最终会清空。 城市 n 的距离保持无穷大，正确生成 -1。 

另一种情况是，最优路径需要连续多次支付同城成本才能占据优势。 例如，如果铁路成本昂贵，但只有在切换状态后才具有时间效率，重复`(x, y)`放松可以连锁。 该算法可以处理这个问题，因为每个城市内松弛都被视为 Dijkstra 中的标准边缘，并且按照字典成本时间对增加的顺序自然地探索重复的改进。 

最后一个微妙的情况是具有不同权衡的相同城市之间的多个平行边。 由于每条边都是独立松弛的，并且 Dijkstra 保留了最佳的词典对，因此算法会自动选择正确的边，无需进行特殊处理。
