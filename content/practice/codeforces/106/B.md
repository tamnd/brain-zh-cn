[CF 106B - 选择笔记本电脑](https://codeforces.com/problemset/problem/106/B)

 **评分：** 1000
 **标签：** 暴力破解、实施
 **求解时间：** 2m 3s
 **已验证：** 是的

 ## 解决方案
 ## 问题理解

 每台笔记本电脑都有四个值：处理器速度、RAM 大小、HDD 大小和价格。 如果存在另一台笔记本电脑在所有三个技术特性上同时严格更好，则该笔记本电脑被认为是过时的。 价格对于决定一台笔记本电脑是否过时并不重要。 

移除所有过时的笔记本电脑后，我们必须选择剩余成本最低的笔记本电脑并打印其原始索引。 

限制很小。 最多有 100 台笔记本电脑，因此即使是比较每对笔记本电脑的算法也是完全安全的。 完整的成对比较最多执行$100^2 = 10{,}000$检查，这在 2 秒的时间限制内是微不足道的。 

主要的困难不是表现，而是正确解释主导条件。 只有当另一台笔记本电脑的速度严格更大、RAM 严格更大并且硬盘驱动器同时严格更大时，一台笔记本电脑才会过时。 

一个容易犯的错误就是认为“大于或等于”就足够了。 考虑这个输入：```
2
2000 1024 100 300
2000 2048 200 250
```第二台笔记本电脑的 RAM 和 HDD 更好，但处理器速度相同，而不是严格更快。 这两款笔记本电脑都没有过时。 正确答案是：```
2
```粗心的实现使用`>=`会错误地移除第一台笔记本电脑。 

另一个常见的错误是在检查过时的笔记本电脑时忘记价格无关紧要。 考虑：```
2
3000 2048 300 900
2500 1024 200 100
```尽管第二台笔记本电脑便宜得多，但它已经过时了。 正确答案是：```
1
```一些实现错误地保留了更便宜的笔记本电脑，而不管规格如何。 

还有一个微妙的情况是几台笔记本电脑没有可比性。 例如：```
3
3000 1024 500 400
2500 4096 300 350
3200 2048 200 300
```没有一个能够支配另一个，因为每个都失去了至少一个属性。 这三者仍然是候选者，因此我们只选择最便宜的一台，即笔记本电脑 3。 

## 方法

 最直接的方法是检查每台笔记本电脑，并询问其他笔记本电脑是否占主导地位。 笔记本电脑用`i`，我们迭代所有笔记本电脑`j`。 如果笔记本电脑`j`具有比笔记本电脑更高的速度、RAM 和 HDD`i`已经过时，可以丢弃。 

在识别出所有未过时的笔记本电脑后，我们对其进行扫描并选择价格最低的一台。 

这种蛮力方法已经足够快了。 最多 100 台笔记本电脑，总比较次数只有 10,000 次。 每次比较都会检查三个整数，因此运行时间实际上是瞬时的。 

不需要排序、高级数据结构或优化技巧，因为约束故意很小。 关键的观察结果是，该问题询问了恰好三个属性上的成对优势。 由于每台笔记本电脑都可能与其他笔记本电脑交互，因此完整的比较是最简单、最清晰的解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 已接受 |
 | 最佳| O(n²) | O(1) | O(1) | 已接受 |

 对于这个问题，暴力解法也是最优的实际解法。 

## 算法演练

 1. 读取所有笔记本电脑并将其属性与其原始的从 1 开始的索引一起存储。 
2. 对于每台笔记本电脑`i`，最初假设它还没有过时。 
3. 比较笔记本电脑`i`与其他所有笔记本电脑相比`j`。 
4.如果是笔记本电脑`j`具有严格更高的速度、严格更大的 RAM 和严格更大的 HDD，标记笔记本电脑`i`已经过时了。 

这与问题陈述中的确切定义相匹配。 相同的值不算数。 
5. 经过所有比较后，仅保留从未标记为过时的笔记本电脑。 
6. 在剩下的笔记本电脑中，选择成本最低的一台。 
7. 打印其原始索引。 

### 为什么它有效

 当且仅当没有其他笔记本电脑在所有三个技术规格中占主导地位时，笔记本电脑才是有效的。 该算法针对每台笔记本电脑彻底检查此条件。 由于每个可能的主导对都经过测试，因此任何过时的笔记本电脑都无法通过过滤步骤。 

经过筛选后，问题就简化为在所有有效候选者中选择成本最低的笔记本电脑。 由于该算法对每台幸存的笔记本电脑都扫描一次，因此它总是能找到正确的最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    laptops = []

    for i in range(n):
        speed, ram, hdd, cost = map(int, input().split())
        laptops.append((speed, ram, hdd, cost, i + 1))

    best_cost = float('inf')
    best_index = -1

    for i in range(n):
        s1, r1, h1, c1, idx1 = laptops[i]

        outdated = False

        for j in range(n):
            if i == j:
                continue

            s2, r2, h2, c2, idx2 = laptops[j]

            if s2 > s1 and r2 > r1 and h2 > h1:
                outdated = True
                break

        if not outdated and c1 < best_cost:
            best_cost = c1
            best_index = idx1

    print(best_index)

if __name__ == "__main__":
    solve()
```该解决方案将每台笔记本电脑及其原始索引存储在一起，因为输出需要输入中的位置，而不是笔记本电脑数据本身。 

嵌套循环直接实现优势检查。 对于每台笔记本电脑，我们都会寻找另一台在所有三个特性方面都更好的笔记本电脑。 这`break`很重要，因为一旦知道笔记本电脑已经过时，就没有必要进行进一步的比较。 

严格的比较运算符是最重要的实现细节。 更换`>`和`>=`当属性相等时，会改变问题的含义并产生错误的结果。 

最终选择步骤仅考虑通过过时测试的笔记本电脑。 由于所有价格都不同，因此成本最低的笔记本电脑是独一无二的。

 ## 边缘情况

 再次考虑相等的情况：```
2
2000 1024 100 300
2000 2048 200 250
```Laptop 2 具有更好的 RAM 和 HDD，但处理器速度相同。 该算法检查：```
2000 > 2000
```这是错误的，因此笔记本电脑 1 并未标记为过时。 两台笔记本电脑均幸存下来，并选择了较便宜的一台笔记本电脑 2。 输出是：```
2
```现在考虑一台便宜但过时的笔记本电脑：```
2
3000 2048 300 900
2500 1024 200 100
```将笔记本电脑 2 与笔记本电脑 1 进行比较时：```
3000 > 2500
2048 > 1024
300 > 200
```这三个条件都成立，因此尽管笔记本电脑 2 更便宜，但它已经过时了。 只有笔记本电脑 1 仍然有效，因此答案是：```
1
```最后，考虑无与伦比的笔记本电脑：```
3
3000 1024 500 400
2500 4096 300 350
3200 2048 200 300
```在每次比较中，每台笔记本电脑都至少在一个类别中失败。 没有一个被标记为过时。 然后，该算法只需在所有三个候选者中选择最低价格，即笔记本电脑 3。
