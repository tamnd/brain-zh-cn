 ## 解决方案
 ## 问题理解

 我们被要求在三维空间中放置一个太空救援站，使其与任何给定行星的最大距离最小化。 每个行星都由其坐标表示$(x, y, z)$。 输入由一个整数组成$n$代表行星的数量$n$坐标线。 输出应该是一个点$(x_0, y_0, z_0)$最大限度地减少到任何行星的最远距离。 如果距离与最佳距离相差最多，则接受答案$10^{-6}$。 

限制很小：$n \le 100$坐标在$[-10^4, 10^4]$。 由于行星数量很少，我们可以负担得起对行星进行三次甚至四次迭代的算法，但空间的连续性阻止了纯粹离散的蛮力。 一种简单的方法会尝试网格上所有可能的点，但精度要求使得这是不可能的。 

不明显的边缘情况包括行星对齐或对称的配置。 例如，如果行星位于$(1, 0, 0)$和$(-1, 0, 0)$，最优点为$(0, 0, 0)$。 只将行星本身视为候选点的粗心算法可能会错误地选择其中一颗行星而不是中心。 

## 方法

 蛮力的想法是检查 3D 边界框中的所有点并计算到行星的最大距离。 对于尺寸的网格$M^3$，每次检查都需要$O(n)$距离计算。 和$M = 20000$（覆盖所有整数坐标）和$n = 100$，这需要$20000^3 \cdot 100$操作，显然不可行。 

关键的观察是最大距离函数$f(x, y, z) = \max_i \text{dist}((x, y, z), (x_i, y_i, z_i))$在 3D 中是凸的。 直观上，当我们将候选点移近最远行星的中心时，最大距离会减小。 凸性允许我们使用迭代几何优化方法，例如三元搜索或梯度下降。 自从$n$很小，一种将候选点移向最远行星的随机爬山方法可以有效地发挥作用。 

这个想法是从任意点（例如，第一个行星）开始，识别距离当前点最远的行星，然后稍微向它移动。 重复此过程逐渐收敛到最小包围球中心。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力网格| O(M^3 * n) | O(M^3 * n) | O(n) | 太慢了 |
 | 迭代爬山/三元搜索 | O(n * 迭代) | O(n) | 已接受 |

 ## 算法演练

 1. 初始化任意行星上的候选点，例如第一个行星。 这是合理的，因为最佳点通常位于行星凸包内的某个位置。 
2. 将初始步长设置为任意两个行星之间最大距离的一半。 这确保了最初的移动覆盖了很大的空间。 
3. 重复直到步长变得非常小：

 1. 计算当前候选点到所有行星的距离。 
2. 确定距离当前候选点最远的行星。 
3. 将候选点向最远的行星移动一小部分步长。 
4. 稍微减小步长，例如乘以 0.99，以进行更精细的调整。 
4.经过足够的迭代后，输出当前候选点。 该过程收敛到一个点，使到行星的最大距离最小化。 

为什么有效：在每一步中，我们都会通过向最远的行星移动来减少最大距离。 最大距离函数的凸性确保没有局部最小值捕获该过程。 逐渐减小步长使我们能够以任意精度细化位置。 

## Python 解决方案```python
import sys
import math
input = sys.stdin.readline

def distance(p1, p2):
    return math.sqrt((p1[0]-p2[0])**2 + (p1[1]-p2[1])**2 + (p1[2]-p2[2])**2)

def main():
    n = int(input())
    planets = [tuple(map(int, input().split())) for _ in range(n)]
    
    # Initialize at the first planet
    x, y, z = planets[0]
    
    # Step size: half the max distance between any two planets
    step = 0.0
    for i in range(n):
        for j in range(i+1, n):
            step = max(step, distance(planets[i], planets[j]))
    step /= 2
    
    # Iterative hill climb
    for _ in range(10000):
        farthest = max(planets, key=lambda p: distance((x, y, z), p))
        dx, dy, dz = farthest[0]-x, farthest[1]-y, farthest[2]-z
        d = math.sqrt(dx*dx + dy*dy + dz*dz)
        if d == 0:
            break
        x += dx/d * step
        y += dy/d * step
        z += dz/d * step
        step *= 0.995
    
    print(f"{x:.10f} {y:.10f} {z:.10f}")

if __name__ == "__main__":
    main()
```该解决方案从行星开始，以确保候选点位于行星的凸包内。 计算最大成对距离给出合理的初始步长。 在每次迭代中，我们都会向最远的行星移动步长的一小部分，并逐渐减小步长以提高精度。 我们处理零距离情况以防止被零除。 选择足够高的迭代次数以实现所需的$10^{-6}$精确。
