 ## 解决方案
 ## 问题理解

 我们给出了一系列的日期，每天梅尔都会收到一个标记为 1 到 5 的蛋糕层成分。梅尔正在制作婚礼蛋糕，每个完整的蛋糕必须按照严格的顺序从第 1 层到第 5 层组装。每种成分可以在到达的当天立即使用，但如果当天没有使用，它就会永远丢失。 

不同的是，梅尔可以同时制作多个蛋糕。 这意味着在任何时候，他都可能有几个未完成的蛋糕，每个蛋糕都处于没有分层和距离完成只有一步之遥的某个阶段。 每天的成分要么可以将一个现有的部分蛋糕从第 k 层推进到第 k+1 层，要么被忽略。 

任务是确定全天可以完成的完整 5 层蛋糕的最大数量。 

输入大小可以达到一百万天，这立即排除了任何尝试显式跟踪每个蛋糕的所有部分状态或模拟所有可能分配的方法。 任何解决方案每天都必须以恒定时间或摊销恒定时间进行处理。 

当层以“错误”的顺序到达时，就会出现微妙的边缘情况，例如许多 5 早在 1 之后。 总是试图首先完成最先进的蛋糕的天真贪婪可能会失败，因为它阻止了启动新的有效链的能力。 另一种失败情况是当中间层丰富但 1s 不足时，过度分配早期层会导致部分死饼永远无法完成。 

## 方法

 强力模拟将维护所有部分蛋糕的列表，并且对于每个传入层，尝试它可以扩展的每个可能的蛋糕。 在最坏的情况下，k 天后可能会有 O(k) 部分蛋糕，因此每个新的一天都可能扫描所有蛋糕，当 N 很大时，会导致 O(N^2) 行为。 这在 10^5 次左右的操作中已经变得不可行，而这里 N 高达 10^6。 

关键的观察是我们永远不需要区分各个蛋糕。 重要的是当前有多少蛋糕正在等待下一个所需的层。 正在进行中的蛋糕可以纯粹通过其当前阶段（从 1 到 4）来描述，因为阶段 5 意味着完成并且不再需要跟踪。 

这将问题减少到管理五个计数器：有多少个部分蛋糕正在等待第 1 层、第 2 层、第 3 层、第 4 层，以及已经形成了多少个完整蛋糕。 每天我们都贪婪地尝试使用传入层来推进需要它的蛋糕。 如果存在多种选择，我们总是更喜欢推进可以使用它的最高级阶段，因为这为早期层保留了灵活性，而这些层以后很难匹配。 

这种贪婪之所以有效，是因为延迟较高阶段的完成而支持较低阶段永远不会增加未来的可能性，而做相反的事情会永久阻止链的形成。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛋糕的暴力模拟| O(N^2) | O(N^2) | O(N) | 太慢了 |
 | 阶段计数器贪婪模拟| O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护一个数组`cnt[1..4]`在哪里`cnt[i]`是当前处于第 i 阶段的部分蛋糕的数量，这意味着它们正在等待第 i+1 层。 我们还设有一个成品蛋糕柜台。 

每天处理一个值`a[i]`。 

1.如果`a[i] == 1`，我们在第 1 阶段开始一个新的部分蛋糕，所以我们增加`cnt[1]`。 这是唯一可以创建新链的层。 
2.如果`a[i] == 2`，我们尝试推进等待第 2 层的蛋糕，即处于第 1 阶段的蛋糕。如果`cnt[1] > 0`，我们递减`cnt[1]`并增加`cnt[2]`。 如果不存在这样的蛋糕，我们就丢弃这种成分。 
3.如果`a[i] == 3`，如果可能的话，我们首先尝试扩展第 2 阶段的蛋糕，因为这样可以保持早期阶段可用。 所以如果`cnt[2] > 0`，我们从`cnt[2]`到`cnt[3]`，否则我们忽略该层。 
4.如果`a[i] == 4`，我们尝试首先将阶段 3 扩展到阶段 4。如果`cnt[3] > 0`，我们向前推进。 否则我们就丢弃它。 
5. 如果`a[i] == 5`，如果我们有任何第 4 阶段的部分蛋糕，这就完成了一个蛋糕。 如果`cnt[4] > 0`，我们减少它并增加答案。 

每个步骤内的顺序很重要，因为我们总是更喜欢扩展最高级的可能阶段。 

它为何有效的原因在于，在任何时候，多组部分蛋糕都完全由其阶段来表征，并且任何有效的最终时间表都可以转换为早期阶段的进展永远不会阻碍后期阶段完成的不变量。 任何时候我们选择跳过较高级蛋糕的可能进展而选择较不高级的蛋糕，我们只会延迟进展，而不会增加可完成的总数，而相反的选择可能会破坏原本会完成的链。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    cnt1 = cnt2 = cnt3 = cnt4 = 0
    ans = 0
    
    for x in arr:
        if x == 1:
            cnt1 += 1
        elif x == 2:
            if cnt1:
                cnt1 -= 1
                cnt2 += 1
        elif x == 3:
            if cnt2:
                cnt2 -= 1
                cnt3 += 1
        elif x == 4:
            if cnt3:
                cnt3 -= 1
                cnt4 += 1
        else:  # x == 5
            if cnt4:
                cnt4 -= 1
                ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了前面描述的stage machine。 每个变量完全对应于不完整蛋糕的一个阶段，并且转换是局部恒定时间更新。 仅当第 4 阶段蛋糕成功接收到第 5 层时，最终答案才会累积。 

一个常见的实施陷阱是试图“公平”并在所有可能的蛋糕上分配层。 这是不必要的，也是有害的，因为该结构保证只计算物质，而不计算身份。

 ## 边缘情况

 当所有早期层出现较晚时，例如许多 5 后面跟着 1，算法会正确地生成零，因为当 5 到达时，第 4 阶段的蛋糕永远不存在。 试图“存储”5 的天真解释会错误地夸大结果。 

当各层以完美的顺序重复到达 1 至 5 时，每一步恰好前进一个链，并且不会发生干扰。 计数器确保没有层被浪费。 

当中间层丰富但 1 稀缺时，只有起始点的数量限制了答案。 该算法反映了这一点，因为 cnt1 是瓶颈，任何后续阶段都无法超越它。
