 ## 解决方案
 ## 问题理解

 我们得到偶数个索引，每个索引代表一个字符比喻。 在每对有序的转义之间，我们都有一个有向值，该值可能是负值，意味着这种关系是禁止的，否则是非零模值。 

任务是计算一个非常大的组合结构。 首先，我们将所有比喻分成两个相等的组，我们可以将其视为为每个索引分配一个二进制标签。 然后每个节点精确地选择一个传出目标，约束条件是第一组中的节点仅指向第二组中的节点，第二组中的节点仅指向第一组中的节点。 此外，这些选择在两个方向上都是单射的，因此每个节点都恰好有一个传出边和一个传入边。 这迫使传出指针的结构成为所有节点的排列。 

每个有向边贡献一个乘法因子。 如果源节点属于第一组，则其贡献乘以分子，如果源节点属于第二组，则其贡献乘以分母。 配置的总价值是所有这些贡献的乘积，解释为以素数为模的有理数。 

最终答案是所有有效组分配和所有有效指针结构的该值的总和。 

约束 n ≤ 21 立即排除任何显式枚举排列的方法。 即使 O(n!) 也太大了，甚至 O(2^n n^2) 也必须小心处理。 这是一个经典信号，表明解决方案必须将排列压缩为循环级贡献，并使用子集动态规划或排列的结构转换。 

微妙的边缘情况来自值 -1。 由于它代表不可能的边，因此必须排除使用它的任何配置。 在模算术下将 -1 视为正常值的简单解决方案将错误地包含无效匹配，从而产生非零贡献，而正确答案应完全排除这些排列。 

另一个微妙的情况是部门结构。 由于一半节点参与分母，因此尝试直接使用浮点或整数除法计算表达式的粗心实现将在模运算下立即中断，特别是当循环交互且取消不是本地时。 

## 方法

 直接暴力方法首先选择将节点划分为两组，然后选择与二分约束一致的所有有效排列，最后计算每个配置的值。 即使忽略分区，枚举排列也已经是 n!，而有了分区，它就变成了 n! 乘以 2^n，即使 n 约为 10，这也是完全不可行的。 

关键的结构见解是指针配置不是任意的。 每个节点恰好有一个出边和一个入边，因此该结构是 n 个元素的排列。 每个排列都分解为不相交的循环，并且二分约束迫使每个循环在两组之间交替。 这立即意味着每个周期的长度必须均匀。 

一旦认识到这一点，分区就可以被吸收到循环结构中。 对于固定排列，每个循环恰好允许两个一致的二分标记，通过选择循环中的哪个奇偶校验被视为“第一组”来获得。 这两个选择对循环贡献的权重相互影响。 因此，每个循环都会贡献一个 X + X^{-1} 形式的对称表达式，其中 X 是沿循环具有固定交替约定的边权重的乘积。

这将整个问题简化为仅限于偶数长度循环的排列总和，其中每个循环仅根据其内部排序贡献局部权重。 全球答案成为这些本地贡献周期的产物。 

剩下的挑战是计算所有此类排列的总和，而无需明确枚举它们。 这是通过子集上的动态编程来处理的，我们从最小的未使用元素开始迭代地构建循环，确保每个循环只构建一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解分区和排列 | O(n!·2^n) | O(n!·2^n) | O(n) | 太慢了|
 | 在子集上循环 DP | O(n^2 2^n) | O(n^2 2^n) | O(2^n) | O(2^n) | 已接受 |

 ## 算法演练

 我们将排列表示为不相交循环的集合，并通过在一组未使用的节点上逐个选择循环来构造答案。 

1.修复当前子集中最小的未使用节点。 该节点必须属于下一个循环，否则同一个循环在不同的旋转下会被计数多次。 
2.构建在此固定节点开始和结束的所有可能的有向循环，并覆盖剩余未使用节点的一些子集。 我们只允许偶数长度的循环，因为奇数循环不能在两组之间一致地交替。 
3. 对于选定的周期排序，通过沿周期分配符号模式来计算其交替权重。 如果循环被写为 v0 → v1 → … → vk−1 → v0，我们分配交替指数，以便偶数位置的边以乘法贡献，奇数位置以相反的方式贡献。 这产生了循环的基值X。 
4. 通过贡献 X + X^{-1} 添加循环的两个可能的标签。 这些对应于选择哪个交替奇偶被视为周期的“男性”一侧。 
5. 从当前掩码中删除该循环的节点，并递归计算剩余节点的贡献。 将循环贡献与剩余子集的结果相乘。 
6. 对为固定起始节点选择的所有有效循环求和，确保每个子集恰好被划分为循环一次。 

### 为什么它有效

 每个有效配置都唯一地分解为不相交的循环，并且每个循环必须在两组之间交替。 一旦循环结构固定，二分标记每个循环就有两个有效选择，从而产生相互贡献。 DP 通过将每个循环锚定在其最小元素处来精确枚举一次，从而防止由于旋转而导致计数过多。 由于排列分解为循环是唯一的，DP 恰好覆盖每个有效配置一次，并为其分配正确的乘法权重。

## 工作示例

 ### 示例 1

 输入：```
2
1 1
1 1
```这里每条边的值为 1，因此无论结构如何，每个有效配置都会贡献 1。 唯一的自由度是节点如何在循环中配对。 由于循环和相互贡献都归结为 1，因此每个有效配置都贡献 1，并且组标签恰好有两个对称分配，产生总和为 2。 

| 步骤| 结构| 骑行重量| 贡献|
 | ---| ---| ---| ---|
 | 中/F隔断 | 中频或调频| - | 1 |
 | 排列| 单2周期| 1 + 1 | 共 2 条 |

 这证实了当所有权重都是中性时，答案纯粹是计算标签分配中的对称性。 

### 示例 2

 输入：```
2
4 1 2 -1
1 1 2 -1
-1 4 1 1
-1 -1 1 4
```这里无效的边立即修剪许多排列。 DP 通过将涉及 -1 的转换视为零权重边，有效地忽略了它们。 其余结构迫使循环仅使用有效的转换，并且由于不对称值，每个循环的贡献在很大程度上取决于方向。 

| 步骤| 选择的周期 | 转发产品 X | 周期贡献|
 | ---| ---| ---| ---|
 | 从 0 开始 | 0 → 1 → 2 → 3 → 0 | 计算交替积 | X + X^{-1} |

 这个例子表明，即使结构上存在许多排列，-1 约束消除了状态空间的大部分，并且 DP 自然地通过零权重传播避免了无效循环。

 ## 边缘情况

 当循环包含 -1 条目时，就会出现临界边缘情况。 在这种情况下，任何使用该边的尝试遍历都会在 DP 状态中产生零贡献。 这确保了此类循环永远不会被计算在内，满足禁止关系完全使配置无效的要求。 

另一个边缘情况是单个 2 周期。 这是最简单的有效排列循环，它直接产生 a_{i,j} * a_{j,i} 加上其倒数形式的贡献。 DP 将其作为循环构造的基本情况来干净地处理，确保不会发生重复计算，因为循环锚定在其最小元素上。 

最后，在整个周期中交替奇偶校验一致性保证了只产生偶数长度的周期。 任何尝试的奇数长度构造都无法在 DP 状态下关闭，从而防止无效的双向分配泄漏到最终总和中。
