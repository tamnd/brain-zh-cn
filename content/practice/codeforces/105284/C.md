 ## 解决方案
 ## 问题理解

 我们得到一个路径图，其中节点编号为 1 到 n，其中每个节点都与其直接邻居相连。 然后每个节点 i 以概率 1/i 独立保留并以概率 1 − 1/i 删除。 全部删除后，剩余的节点形成原链的一个子图，这个子图可能会分裂成多个连通的组件。 

此设置中的连接组件只是保留节点的最大连续块。 每当我们看到删除的节点产生间隙时，链就会分裂。 所以最终的答案是这种连续保留段的预期数量。 

输入给出了许多独立的 n 值，对于每个值，我们必须计算该期望模 10^9+7。 

约束允许 n 最多为 10^6，测试用例总数最多为 2×10^5，因此任何解决方案都必须本质上是线性的或每个测试用例都更好。 在最坏的情况下，即使每个查询的 O(n) 也太慢，除非我们预先计算。 该结构强烈建议对 n 进行前缀计算。 

一种简单的方法是模拟保留/删除节点的所有子集并计算组件。 这是 2^n 个配置，对于 n 超过 20 这样的小值来说已经不可能了。即使蒙特卡罗也不会给出精确的模块化期望。 

如果尝试直接模拟或 DP 测试每个测试用例的所有配置，则会出现一个更微妙的问题：概率模型取决于 i，因此每个位置的贡献不同，并且每个查询从头开始重新计算将多次重复相同的工作。 

## 方法

 关键的观察是保留/删除节点的二进制序列中的连接组件可以在本地进行计数。 如果保留节点 i 并且删除 i−1 或 i−1 不存在，则新组件恰好从位置 i 开始。 

因此，如果我们为“i 启动一个新组件”定义一个指标，那么期望就变成了本地事件的概率之和。 

令K_i 为节点i 被保留的事件。 当 K_i 为 true 并且 i = 1 或 K_{i−1} 为 false 时，节点 i 贡献一个新组件。 通过期望的线性，我们可以在 i 上独立地对这些概率求和。 

蛮力思想将枚举节点的所有子集，计算 O(n) 中每个子集的连通分量，并对所有概率求平均值。 这会失败，因为子集的数量是指数级的，并且概率权重不均匀。 

关键的简化是我们永远不需要超出相邻节点的相关性。 每一项仅取决于 K_i 和 K_{i−1}，并且节点状态的独立性使其完全局部化。 

我们计算：

 对于 i = 1，组件启动的概率为 P(K_1) = 1。 

对于 i > 1，当节点 i 被保留并且节点 i−1 不被保留时，新组件开始：

 P = P(K_i) × P(不是 K_{i−1}) = (1/i) × (1 − 1/(i−1))。 

这将整个问题简化为 i 上的前缀和。 

剩下的唯一需要注意的是模运算和单独处理特殊情况 i = 1。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 局部期望总和| O(n) 预处理，O(1) 每个查询 | O(n) | 已接受 |

 ## 算法演练

1. 跨测试用例预先计算最大 n 以内的所有整数的模逆。 这是必要的，因为概率涉及质数模下的 1/i，因此必须用模逆乘法来代替除法。 
2. 构建一个数组 dp，其中 dp[i] 存储长度为 i 的链的预期组件数量。 我们从 1 到 max n 递增地计算 dp。 
3. 初始化 dp[1] = 1，因为始终保留单个节点并形成一个组件。 
4. 对于从 2 到 max n 的每个 i，计算位置 i 的贡献作为保留 i 和不保留 i−1 的概率。 这是 (1/i) × (1 − 1/(i−1))。 我们将其添加到 dp[i−1] 以获得 dp[i]。 
5. 通过打印 dp[n] 回答每个查询。 

这种增量形式起作用的原因是，每个新位置要么扩展现有组件，要么启动一个新组件，只有第二种情况会影响组件数量。 

### 为什么它有效

 组件的预期数量等于组件的预期启动数量。 当节点 i 存在但节点 i−1 不存在时，开始恰好发生在位置 i 处。 这些事件仅取决于独立的伯努利变量，因此它们的概率会干净地倍增。 期望的线性允许对这些每个位置的概率求和，而无需考虑不同起点之间的相互作用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    t = int(input())
    ns = []
    max_n = 0

    for _ in range(t):
        n = int(input())
        ns.append(n)
        if n > max_n:
            max_n = n

    if max_n == 0:
        return

    inv = [0] * (max_n + 2)
    for i in range(1, max_n + 2):
        inv[i] = modinv(i)

    dp = [0] * (max_n + 2)
    dp[1] = 1

    for i in range(2, max_n + 1):
        p_keep = inv[i]
        p_prev_keep = inv[i - 1]
        p_prev_not = (1 - p_prev_keep) % MOD
        dp[i] = (dp[i - 1] + p_keep * p_prev_not) % MOD

    out = []
    for n in ns:
        out.append(str(dp[n]))
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```实现首先读取所有测试用例以确定最大 n，因为我们需要全局预计算。 模逆是使用费马定理计算的。 

dp 数组存储每个前缀长度的期望值。 每个转换仅使用 i 和 i−1，因此计算是线性的。 

一个微妙的点是正确计算 (1 − 1/(i−1)) 模 MOD。 我们在减法后明确应用模以避免负值。 

## 工作示例

 考虑 n = 2 的样本。 

| 我| P(保留 i) | P（之前没有）| 贡献| dp[i] | dp[i] |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | - | 1 | 1 |
 | 2 | 1/2 | 1/2 1 − 1/1 = 0 | 0 | 1 |

 对于 n = 2，节点 1 始终存在，因此无论节点 2 如何，始终只有一个组件与 dp[2] = 1 匹配。 

现在考虑 n = 4。 

| 我| P(保留 i) | P（之前没有）| 贡献| dp[i] | dp[i] |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | - | 1 | 1 |
 | 2 | 1/2 | 1/2 0 | 0 | 1 |
 | 3 | 1/3 | 1/3 1 − 1/2 = 1/2 | 1/6 | 1/6 1 + 1/6 |
 | 4 | 1/4 | 1/4 1 − 1/3 = 2/3 | 1 − 1/3 = 2/3 | 1/6 | 1/6 1 + 1/6 + 1/6 | 1 + 1/6 + 1/6 |

 这会产生 4/3，与预期的模块化输出相匹配。 

该跟踪显示，仅当保留的节点前面有缺失的节点时，组件才会出现，并且贡献独立累积。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(最大 n + t) | 预先计算 dp 一次，每次在 O(1) 内回答查询 |
 | 空间| O(最大 n) | 存储 dp 和逆数组 |

 高达 10^6 的预处理完全符合时间限制，并且每个查询都是恒定时间，使得该解决方案适用于最多 2×10^5 的测试用例。 

## 测试用例```python
import sys, io

MOD = 10**9 + 7

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    ns = []
    max_n = 0
    for _ in range(t):
        n = int(input())
        ns.append(n)
        max_n = max(max_n, n)

    inv = [0] * (max_n + 2)
    for i in range(1, max_n + 2):
        inv[i] = pow(i, MOD - 2, MOD)

    dp = [0] * (max_n + 2)
    if max_n >= 1:
        dp[1] = 1

    for i in range(2, max_n + 1):
        dp[i] = (dp[i - 1] + inv[i] * (1 - inv[i - 1]) % MOD) % MOD

    return "\n".join(str(dp[n]) for n in ns)

# provided samples

assert run("1\n2\n") == "1", "two nodes"

 ## 边缘情况

 当 n = 1 时，算法直接设置 dp[1] = 1，因为单个保证保留节点恰好形成一个组件。 不依赖 dp[0]，这避免了未定义的行为。 

对于 n = 2，转换使用 P(prev not) = 1 − 1/1 = 0，正确捕获节点 1 始终存在，因此节点 2 永远无法启动新组件。 

对于较大的 n，每一项仅取决于 i 和 i−1，因此即使在最大 n = 10^6 时，计算也保持稳定和线性，没有隐藏递归或状态爆炸。
