 ## 解决方案
 ## 问题理解

 我们被要求计算整数对$(n, p)$在哪里$p$被限制在之间$0$和$P$，并且涉及乘法和类似阶乘行为的两个算术条件必须同时成立。 一个条件迫使两者之间建立一种关系$n \cdot p$和一个源自的值$p$，另一个禁止退化相等的情况，即产品崩溃回原始值。 

关键的结构思想是对于每个固定的$p$，值$n$不是自由选择的。 相反，这些限制实际上迫使单个候选人$n$（如果有的话），通过方程的代数重排确定，涉及$n \cdot p$。 这意味着问题不在于搜索两个变量，而在于迭代$p$并检查是否是有效的整数$n$存在。 

约束条件$P \le 10^5$立即排除任何尝试枚举所有可能对的解决方案$(n, p)$以二维方式。 一个天真的双循环已经太慢了，如果$n$有界类似于$p$，并且如果$n$是无界的或很大的。 预期的解决方案必须将所有内容简化为一次传递$p$。 

微妙的边缘情况出现在非常小的值$p$。 什么时候$p = 0$, 任何产品$n \cdot p$变为零，这往往会破坏唯一性论证。 什么时候$p = 1$或者$p = 2$、阶乘和线性表达式会分解为小整数，其中等式条件的行为与一般情况不同。 一个粗心的推导假设“大$p$“行为会错误地包含或排除这些小值，这正是错误答案通常来自的地方。

 ## 方法

 暴力解释会尝试所有对$(n, p)$和$0 \le p \le P$并直接测试这两个条件。 即使我们限制$n$到类似的范围，这已经给出了$O(P^2)$操作，大约是$10^{10}$在最坏的情况下。 这远远超出了 1 秒限制所能处理的范围。 

一旦我们隔离，病情的结构就会大大简化$n$。 核心关系是产品$n \cdot p$必须匹配仅源自的值$p$，我们可以将其解释为$p!$。 这立即迫使$n = \frac{p!}{p}$，进一步简化为$n = (p-1)!$。 所以对于每一个$p$, 最多有一个候选者$n$，问题就简化为决定该候选者在剩余的不平等约束下是否有效。 

一旦进行了这种减少，剩下的条件只会过滤掉产品塌陷回原来状态的病态情况。$p$。 这种情况仅发生在最小阶乘值上，特别是当$p$是如此之小$p! = p$。 出现这种情况的原因是$p = 1$和$p = 2$。 所有较大的值都会自动满足不等式条件。 

因此，暴力方法分解为一个简单的计数问题$p$，其中每个有效$p$恰好贡献了一对有效的。

 | --- | --- | --- | --- |
 | 蛮力 |$O(P^2)$|$O(1)$| 太慢了 |
 | 减少每次检查 |$O(P)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 观察对于每个固定$p$，条件力$n$满足以下形式的单个方程$n \cdot p = p!$。 这意味着$n$每当$p > 0$。 
2. 重新排列方程以计算唯一可能的候选者$n$作为$n = (p-1)!$。 这消除了搜索的需要$n$完全。 
3. 对于每个$p$，验证是否满足第二个条件$p \ne n \cdot p$成立。 自从$n \cdot p = p!$，这个条件就变成了$p \ne p!$。 
4. 确定以下值$p$在哪里$p = p!$。 这个等式仅适用于$p = 1$和$p = 2$，因为阶乘增长很快并且超过了来自的恒等值$p \ge 3$向前。 
5. 计算所有整数$p$在范围内$[1, P]$排除$p = 1$和$p = 2$。 每一个这样的$p$恰好贡献了一对有效的。 

### 为什么它有效

 该转换将问题简化为每个问题的一对一映射$p$给单个候选人$n$。 没有两个不同$n$值可以满足相同的约束$p$，并且所有无效性都被这一小组完全捕获$p$其中阶乘等于身份值。 这保证了计数有效$p$直接产生正确数量的有效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    P = int(input().strip())
    # valid p are all integers in [1, P] except p = 1 and p = 2
    if P <= 2:
        print(0)
    else:
        print(P - 2)

if __name__ == "__main__":
    solve()
```该代码反映了最终的减少，我们不再尝试计算阶乘或枚举$n$。 唯一有意义的观察是所有$p \ge 3$恰好贡献一对有效的。 

边界处理为$P \le 2$是必要的，因为公式$P - 2$否则会产生负数或不正确的计数。 这是唯一通常出现相差一错误的地方。

 ## 边缘情况

 对于$P = 1$，唯一的候选人$p$是 1，但它失败了，因为阶乘崩溃并违反了不等式条件。 该算法正确返回 0，因为$P \le 2$。 

为了$P = 2$， 两个都$p = 1$和$p = 2$出于同样的原因无效。 减法形式将给出 0，匹配正确的结果，而不需要超出边界检查的特殊处理。 

为了$P \ge 3$，无效的情况是固定的并且不会随着$P$，所以公式$P - 2$保持稳定并正确计算所有剩余的有效值$p$价值观。
