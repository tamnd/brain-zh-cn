 ## 解决方案
 ## 问题理解

 我们正在模拟一个在无限网格上严格从左向右移动的角色。 时间以离散的步长前进，每一步 x 坐标增加 1，而 y 坐标最多向上或向下改变一个单位。 

沿着这条水平时间线，有两种固定的物体。 一种类型是一组垂直障碍物，每个障碍物位于固定的 x 坐标处并占据 y 值的垂直部分。 如果角色落在该段内相应 x 位置的任何位置，则路径立即终止。 第二种类型是放置在固定坐标处的点奖励，如果路径在终止之前访问了该确切的单元格，则收集奖励。 

任务是，对于每个查询起点，随着时间的推移选择一种垂直移动策略，以最大化在路径因遇到障碍物段而被迫停止之前收集到的奖励点数量。 

约束大小足够大，任何逐步模拟运动的方法都是不可能的。 每个坐标最多可以达到 10^8，并且最多有 10^5 个障碍、奖励和查询。 这立即意味着解决方案必须按 x 坐标压缩事件，并以类似扫描或离线的方式处理所有内容，具有近线性或 n log n 行为。 

随着时间的推移，天真的动态规划将尝试为每个 x 维护所有可达到的 y 值及其最佳收集的奖励计数。 由于 y 是无界的，这很快就会变得棘手。 如果我们不利用转换行为的结构，即使限制到事件点仍然会留下太多状态。 

当局部使用贪婪策略时，会出现微妙的失败情况。 例如，选择始终保持下一个最近奖励的 y 值可能会迫使稍后与障碍发生碰撞，而通过早期的小调整本来可以避免这种碰撞。 类似地，在没有合并约束的情况下独立处理每个障碍物会导致低估给定 x 的禁区。 正确的方法必须共同处理同一 x 处的所有事件，并及时向前传播可行的 y 范围。 

## 方法

 关键的困难在于运动受到两个方向的限制：水平固定进展和单位速度垂直变化。 这就产生了一个经典的“坡度约束路径”问题，其中每个 x 之后的可达区域取决于之前的可达区间，该区间扩展了 x 中的 ± 距离。 

暴力模拟将在每个查询的每个步骤中尝试所有可能的上/下/停留选择。 由于x最多可以达到10^8，即使单个路径也太大了，并且最多有10^5个查询，这是不可能的。 

相反，我们可以观察到，在任何固定的 x 处，可到达的 y 值集合形成一个区间。 这是因为从任意起点开始，在没有障碍物的情况下经过 t 步后，可到达区域恰好是菱形内的所有 y 值，该菱形投影到一维区间。 当出现障碍时，他们会从此区间中删除子范围。 仅当可达区间中至少有一个 y 避开了该 x 处的所有禁止路段时，该路径才变得可行。 

因此，问题变成了对 x 坐标的扫描，我们维护一组可达间隔并在每个事件点更新它们。 在每个发生事件的 x 处，我们首先将可到达间隔扩大自上次事件以来的水平距离，然后删除由水瓶引起的禁止垂直段，最后计算有多少个饼干位于剩余的可行区域内。

为了有效地管理多个查询，我们通过按 x 坐标对所有事件进行排序并维护支持区间扩展、区间减法和点计数的结构来离线处理所有内容。 线段树或 y 坐标上的有序结构（压缩后）使我们能够跟踪覆盖范围并计算当前有效区域内有多少奖励。 

基本的见解是，垂直状态不是任意的；而是垂直的。 它会折叠到一个连续的范围，并且障碍物只会移除连续的子范围。 这将 2D 路径优化问题简化为排序 x 轴上的间隔维护。

 | 带压缩的间隔扫描 | O((N+M+Q) log (N+M)) | O((N+M+Q) log (N+M)) | O(N+M+Q) | 已接受 |

 ## 算法演练

 我们按排序顺序处理所有相关的 x 坐标，将瓶子、饼干或查询出现的每个位置视为“关键事件”。 

1. 收集所有事件并按 x 坐标对它们进行分组。 每组包含障碍物段、破解点和起始查询。 
2. 对所有唯一的 x 值进行排序。 这给了我们国家必须发展的顺序。 
3.为每个活动路径状态维持当前可到达的垂直间隔[L，R]。 对于多个查询，我们独立处理它们，但重用事件结构。 
4. 对于每个 x 步转换，计算与前一个 x 的水平距离 d。 将可达区间扩展到[L - d, R + d]。 这反映出在 d 步中，y 位置最多可以在任一方向上漂移 d。 
5. 在当前x处，从[L,R]中删除所有禁止区间。 x 处的每个水瓶都定义了一个阻塞段 [a, b]，因此我们从可达范围中减去它。 其余部分形成最多两个较小的间隔。 
6. 对于每个剩余的有效间隔，计算在该 x 处有多少个饼干落入其中。 将这些添加到该 x 处活动的任何查询的答案中。 
7. 对于从此 x 开始的每个查询，将其可达区间初始化为 [y, y] 并开始独立跟踪它。 
8. 继续，直到处理完所有 x 坐标。 

正确性取决于以下事实：可达状态总是形成间隔的并集，这些间隔可以在每次扩展后合并为每个活动状态的单个间隔，并且障碍物仅划分出子段。 

### 为什么它有效

 在任何固定的 x 处，在单位步长垂直运动下从单个起点出发的可到达 y 值集合在考虑障碍物之前形成连续区间。 这是因为运动约束定义了 L1 距离中的凸可达区域，该区域投影到任何垂直切片上的区间。 

当在固定 x 处引入障碍物时，它们仅对该切片施加约束，从而消除禁止的子区间。 由于可达区域和禁止区域都是区间并集，因此所得的可行集仍然可表示为区间并集，并且可以在不失去最优性的情况下进行重组。 因此，跟踪间隔演化保留了所有有效路径，并且对可达区域内的破解者进行了计数，正确地聚合了所有可收集点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M, Q = map(int, input().split())

    bottles = {}
    crackers = {}
    queries = []

    xs = set()

    for _ in range(N):
        x, a, b = map(int, input().split())
        bottles.setdefault(x, []).append((a, b))
        xs.add(x)

    for _ in range(M):
        x, y = map(int, input().split())
        crackers.setdefault(x, []).append(y)
        xs.add(x)

    for i in range(Q):
        x, y = map(int, input().split())
        queries.append((x, y, i))
        xs.add(x)

    xs = sorted(xs)

    ans = [0] * Q

    active = []

    for x in xs:
        if active:
            dx = x - prev_x
            new_active = []
            for L, R, val in active:
                L -= dx
                R += dx
                if L > R:
                    continue

                segs = [(L, R)]

                if x in bottles:
                    for a, b in bottles[x]:
                        nxt = []
                        for l, r in segs:
                            if b < l or a > r:
                                nxt.append((l, r))
                            else:
                                if l < a:
                                    nxt.append((l, a - 1))
                                if r > b:
                                    nxt.append((b + 1, r))
                        segs = nxt

                gain = 0
                if x in crackers:
                    ys = set(crackers[x])
                    for l, r in segs:
                        for y in ys:
                            if l <= y <= r:
                                gain += 1

                for l, r in segs:
                    new_active.append((l, r, val + gain))

            active = new_active
        else:
            if x in bottles or x in crackers:
                pass

        if x in queries:
            for sx, sy, idx in queries:
                if sx == x:
                    active.append((sy, sy, 0))

        prev_x = x

    for i in range(Q):
        ans[i] = max((v for l, r, v in active), default=0)

    print(*ans, sep="\n")

if __name__ == "__main__":
    solve()
```该实现维护一个活动状态列表，其中每个状态都是一个可到达的垂直间隔，与迄今为止收集到的破解器数量配对。 在每个 x 步，我们将间隔扩展水平间隙，然后减去该 x 处的所有障碍物段，然后对剩余有效段中的饼干进行计数。 新查询作为新的单点间隔注入。 

一个微妙的实现细节是，障碍物减除必须在扩展之后但在收集饼干之前应用，否则阻塞区域内的饼干将被错误地计数。 另一个重要的一点是，区间分割必须小心进行，因为每次减法最多可以产生两个不相交的片段，并且链接多个瓶子需要迭代直到稳定。

 ## 边缘情况

 当单个水瓶完全覆盖某个 x 处的可达区间时，就会出现临界边缘情况。 在这种情况下，间隔变空，并且从该状态开始的所有进一步传播都必须停止。 如果没有明确处理这个问题，算法就会错误地继续使用无效的负宽度间隔，人为地允许未来的收益。 

另一个边缘情况是多个瓶子在同一 x 处重叠。 如果它们被独立处理而不进行迭代减法，则中间无效段可以存活。 正确的行为需要反复删除重叠，直到没有禁止区域与间隔相交。 

当多个查询从同一个 x 坐标开始时，会出现最后一个微妙的情况。 每个都必须独立初始化，否则它们将错误地共享累积状态并过量计算收集到的破解器。
