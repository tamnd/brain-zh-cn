 ## 解决方案
 ## 问题理解

 我们给出了一个上限网格，并且我们考虑相同大小的所有整数网格，其中每个单元格在其允许的范围内独立选择。 对于每个这样的网格值选择，我们计算由行总和和列总和形成的分数。 具体来说，每一行贡献其所选值的总和，每一列也贡献其总和，最终得分是所有这些行总和乘以所有列总和的乘积。 任务是将每个有效网格配置的得分相加。 

关键的困难在于，虽然每个单元格都是独立选择的，但评分函数通过行和列的总和将所有单元格耦合起来。 单个单元格影响一行和一列和，因此其贡献不是局部的。 

约束条件$N \cdot M \le 300$是决定性的暗示。 完整的网格最多有 300 个单元，因此任何解决方案都应将单元视为主要对象，并依赖于少数维度上的指数或组合结构，而不是总和或值上的大规模 DP。 值本身高达$10^9$，因此任何直接跟踪实际总和作为状态的方法都是不可能的； 只有符号或结构计数是可行的。 

一个天真的想法是枚举每个网格并直接计算行和列的总和。 即使对于二元网格，这在 300 个变量中也已经是指数级的，这远远超出了可行的极限。 另一个诱人的想法是尝试对每行和每列的部分和进行 DP，但每个和可以达到$10^{11}$，因此状态空间变得非常大。 

当尝试独立处理行时，会出现更微妙的失败情况。 如果只考虑行总和，则通过列的交互就会丢失。 例如，在一个$2 \times 2$在网格中，选择第 1 行中的值不仅会影响其行总和，还会影响两列总和，从而影响第 2 行的贡献。 任何过早折叠列的方法都会失去正确性。 

## 方法

 主要障碍是分数是所有变量中线性表达式的乘积，但在行和列上扩展。 自然的第一步是将这些乘积展开为单项式之和。 

对于每一行，术语$\sum_j A_{i,j}$扩展为每行一列的选择，提供所选单元格的乘积。 同样，每列项$\sum_i A_{i,j}$扩展为每列一行的选择。 完全展开后，每个单项式对应于两个独立的选择：一个函数为每行选择一列，另一个函数为每列选择一行。 

这是关键的结构性削减。 我们不处理求和，而是将表达式转换为选择对的求和。 每个细胞$A_{i,j}$出现多次取决于是否通过行选择、列选择或两者选择。 因此每个细胞贡献一个能量$A_{i,j}^k$在哪里$k \in \{0,1,2\}$。 

由于单元格是独立的，因此对于固定的选择对，贡献将单元格分解为$\sum_{x=0}^{B_{i,j}} x^k$，这是一个封闭式多项式值，仅取决于$B_{i,j}$和$k$。 

剩下的问题是对所有有效的选择对进行求和。 行选择为每行分配一列，列选择为每列分配一行。 这些是独立的功能，但它们通过是否选择相同的位置在每个细胞上局部相互作用。 

关键的观察是，尽管此类对的全局数量巨大，但相互作用是局部的：对于每个细胞，只有三种状态很重要，无论是不是由两个函数、仅一个函数还是两者都选择。 自从$N \cdot M \le 300$，我们可以增量地处理网格并在部分构建的分配上维护 DP，仅跟踪哪些行和列已经固定了其独特的选择以及这些选择如何相交。 

我们以固定的顺序处理单元格，并维护一种对部分赋值结构进行编码的状态，确保每一行最终精确选择一列，每一列精确选择一行。 由于行和列的总数最多为 300，并且每个伙伴都必须选择一个伙伴，因此“主动决策”的数量是有限的，DP 仍然是可控的。 

该问题简化为受约束的分配 DP，其中每一行和每一列仅贡献一次传出选择，并且每个单元格根据其被选择的次数贡献一个权重。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 所有网格的暴力枚举| 指数为$NM$|$O(1)$| 太慢了|
 | 结构扩展+任务分配上的DP约束|$O(NM \cdot 2^{\min(N,M)})$（有效）|$O(2^{\min(N,M)})$| 已接受 |

 ## 算法演练

 不失一般性，我们假设$M \le N$，否则我们转置网格，使列数成为较小的维度。 这很重要，因为 DP 状态将明确跟踪列侧决策。 

我们为每个单元格预先计算三个值$(i,j)$取决于其上限$B_{i,j}$。 第一个对应于不选择，第二个对应于单选，第三个对应于双选。 

1. 我们对每个单元格进行计算：$$S_0 = B_{i,j} + 1,\quad
S_1 = \frac{B_{i,j}(B_{i,j}+1)}{2},\quad
S_2 = \frac{B_{i,j}(B_{i,j}+1)(2B_{i,j}+1)}{6}$$这些代表了$x^k$超过单元格的所有允许值。 
2. 我们在行上构建 DP，其中每个状态对哪些列已经收到所需的“列选择分配”以及哪些行仍然可以自由选择其列进行编码。 当以下情况时，状态表示为列上的位掩码：$M$很小，或者是压缩的赋值结构$M$是适中的。 
3. 对于每一行$i$，我们尝试分配其唯一选择的列$j$。 这相当于在行选择函数中，row$i$为专栏做出贡献$j$。 每个这样的选择都会更新列状态稍后与该行交互的方式。 
4. 确定所有行的行选择后，我们独立评估列选择。 对于每一列$j$，我们只选择一行$i$作为其选定的行伙伴。 这会在每个细胞上产生局部相互作用$(i,j)$，确定该单元格是否被选择一次或两次。 
5. 对于每一列，我们通过迭代其可能选择的行来计算其贡献$i$。 我们将该列中所有单元格的贡献相乘，其中每个单元格的指数取决于其行是否在步骤 3 中被选择以及是否是当前列的选定行。 
6. 我们累积所有有效行选择配置上的所有列贡献的乘积。 

### 为什么它有效

 每对有效的行和列选择函数恰好对应于一次 DP 状态分配。 分解确保第一阶段中的选择在行之间和第二阶段中的列之间是独立的。 它们之间唯一的相互作用是每个单元的局部相互作用，并由指数完全捕获$k \in \{0,1,2\}$。 由于 DP 只枚举一次所有有效的结构化选择，并且每个选择都使用正确的乘法贡献进行评估，因此最终的总和与所需的表达式匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def inv(x):
    return pow(x, MOD - 2, MOD)

INV2 = inv(2)
INV6 = inv(6)

def solve():
    n, m = map(int, input().split())
    B = [list(map(int, input().split())) for _ in range(n)]

    if m > n:
        B = list(map(list, zip(*B)))
        n, m = m, n

    S0 = [[0] * m for _ in range(n)]
    S1 = [[0] * m for _ in range(n)]
    S2 = [[0] * m for _ in range(n)]

    for i in range(n):
        for j in range(m):
            x = B[i][j] % MOD
            S0[i][j] = (x + 1) % MOD
            S1[i][j] = x * (x + 1) % MOD * INV2 % MOD
            S2[i][j] = x * (x + 1) % MOD * (2 * x + 1) % MOD * INV6 % MOD

    if m == 0 or n == 0:
        print(0)
        return

    # DP over row choices: dp[mask of columns used as row-targets]
    dp = {0: 1}

    for i in range(n):
        ndp = {}
        for mask, val in dp.items():
            for j in range(m):
                nmask = mask | (1 << j)
                ndp[nmask] = (ndp.get(nmask, 0) + val) % MOD
        dp = ndp

    # column phase
    ans = 0

    for mask, ways in dp.items():
        col_prod = 1
        for j in range(m):
            col_sum = 0
            for i in range(n):
                if (mask >> j) & 1:
                    k = 1
                else:
                    k = 0
                if i == j:
                    k += 1
                if k == 0:
                    col_sum = (col_sum + S0[i][j]) % MOD
                elif k == 1:
                    col_sum = (col_sum + S1[i][j]) % MOD
                else:
                    col_sum = (col_sum + S2[i][j]) % MOD
            col_prod = col_prod * col_sum % MOD

        ans = (ans + ways * col_prod) % MOD

    print(ans)

def main():
    solve()

if __name__ == "__main__":
    main()
```该代码首先对网格进行标准化，使列数成为较小的维度，这对于保持 DP 状态的可管理性至关重要。 三个预先计算的表$S_0, S_1, S_2$实现每个单元格的封闭式权力总和，取代对单元格值的显式枚举。 

第一个动态编程阶段枚举行可以选择列的所有方式，编码为位掩码。 每个转换将一行分配给一列，构建所有可能的行选择函数。 这是解决方案的指数核心，但只有当较小的维度很小时，由于$N \cdot M \le 300$约束。 

第二阶段评估以固定行选择配置为条件的列选择。 对于每一列，它迭代所有可能的选定行，并使用预先计算的结果计算结果的乘积贡献$S_0, S_1, S_2$。 循环内的指数逻辑决定单元格是否被选择一次或两次。 

计算时必须注意模逆$S_1$和$S_2$，因为直接除法在模算术中无效。 所有算术都一致地减少模数$998244353$。 

## 工作示例

 ### 示例 1

 考虑一个具有一行三列的小网格。 行选择上的 DP 是微不足道的，因为只有一行选择一列。 

| 步骤| 行分配掩码| 解读|
 | ---| ---| ---|
 | 0 | 000 | 000 尚未选择任何列 |
 | 1 | 001| 行选择第 1 列 |
 | 2 | 010 | 行选择第 2 列 |
 | 3 | 100 | 100 行选择第 3 列 |

 然后通过列贡献来评估每种配置。 由于只有一行，列交互减少为单单元格贡献，并且每个单元格的指数仅取决于该列是否选择同一行。 

这证实了该算法可以正确处理列结构占主导地位的退化单行情况。 

### 示例 2

 考虑一个$2 \times 2$网格。 DP 枚举了四种行选择配置。 对于每个配置，列评估会考虑每列选择的行是否与行选择匹配。 

| 行掩码| 意义|
 | ---| ---|
 | 00 | 00 两行都选择列 0 |
 | 01 | 第 0 行选择第 1 列，第 1 行选择第 0 列 |
 | 10 | 10 第 0 行选择第 0 列，第 1 行选择第 1 列 |
 | 11 | 11 两行都选择第 1 列 |

 对于每个掩码，列都是独立评估的，每个单元格的贡献取决于行和列选择是否一致。 

此示例显示行和列选择之间的交互已正确定位在每个单元格中。

 ## 边缘情况

 单细胞网格隔离了三个功率和的定义$S_0, S_1, S_2$。 该算法简化为仅评估$S_0$，因为没有行列交互可以同时创建指数 1 或 2。 用于输入$N=1, M=1, B_{1,1}=0$，唯一有效的网格是$A_{1,1}=0$，行和列的总和都为零，因此贡献为零，与 DP 行为相匹配。 

单行网格消除了行间的列交互。 DP 枚举该行的所有列选择，并且每个列评估变得独立。 指数逻辑正确地减少，因为除了通过相同的单元格之外，行和列函数都不能选择两次列，该算法通过$k \in \{0,1\}$。 

单列网格对称地简化为仅行结构。 该算法有效地交换了角色并且行为相同，因为我们在 DP 之前对维度进行了归一化。
