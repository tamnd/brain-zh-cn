 ## 解决方案
 ## 问题理解

 我们得到一个数组和三个不同的值，$X$,$Y$， 和$K$，与订购$Y < K < X$。 对于每个测试用例，我们需要计算同时满足两个极值条件的子数组：在子数组内，最大值必须恰好为$X$，最小值必须恰好是$Y$，和值$K$其内部任何地方均被禁止。 

仅当每个元素都位于封闭范围内时，子数组才对答案做出贡献$[Y, X]$，两个端点实际上都是在子数组内“实现”的，这意味着至少出现一次$X$并且出现一次$Y$，并且没有出现$K$出现。 

这些约束意味着所有测试用例的总数组长度最多为$5 \cdot 10^5$，这排除了任何二次甚至接近二次的每次测试方法。 任何解决方案在总输入大小上都必须是线性或接近线性的。 显式检查所有子数组的解决方案需要$O(N^2)$每个测试用例的时间，这将超过$10^{10}$在最坏的情况下操作是不可行的。 

当数组不包含有效的值时，会出现微妙的边缘情况$X$或者$Y$出现在禁止元素之间的段中。 例如，如果$A = [5, 4, 6]$,$X=6$,$Y=4$,$K=5$，子数组$[4,6]$有效，但任何包含 5 的段都是无效的，无论其他元素如何。 一个简单的滑动窗口，仅跟踪最小值和最大值而不显式处理$K$会错误地包含包含以下内容的无效子数组$K$。 

另一种故障模式是处理“max is$X$”和“最小是$Y$” 独立。例如，一个段可能包含$X$和$Y$，但也包含比$X$或小于$Y$，如果不仔细执行边界，就会破坏正确性。 

## 方法

 暴力解决方案枚举每个子数组并计算其最小值、最大值，并检查是否存在$K$。 对于每个子数组$[l, r]$，扫描范围成本$O(r-l+1)$，所以总复杂度是$O(N^3)$在最字面的实现中，或者$O(N^2)$增量维护。 和$N = 5 \cdot 10^5$， 甚至$O(N^2)$远远超出了可行的限度。 

关键的观察是$K$充当硬分离器。 任何有效的子数组都不能跨越索引，其中$A[i] = K$。 这将数组分成独立的段。 在每个细分市场中，我们只关心其中的值$[Y, X]$，因为任何超出此范围的值都会立即违反最小值或最大值要求。 

在一个无段内$K$，我们将问题简化为计算其元素位于的子数组$[Y, X]$并且其中至少包含一个$X$以及至少一个$Y$。 这转化为一个经典的两指针计数问题：我们维护一个滑动窗口，跟踪$X$和$Y$，并确保所有元素都在边界内。 

我们不是直接动态地强制执行最小和最大条件，而是反转计数逻辑。 我们计算有效区域内的所有子数组，并减去那些未能包含的子数组$X$或未能包含$Y$。 跟踪最后看到的头寸使我们能够在恒定时间内计算每个索引的有效贡献。

 | ---| ---| ---| ---|
 | 蛮力 |$O(N^2)$|$O(1)$| 太慢了|
 | 最佳 |$O(N)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例，从左到右扫描数组，同时维护段边界和出现信息。 

1. 我们维护一个标记当前有效段开始的指针，每当遇到$K$，因为没有子数组可以包含它。 这确保我们永远不会考虑无效的跨度。 
2. 在一个段内，我们跟踪最后一次看到的索引$X$和$Y$。 这些位置告诉我们以当前索引结尾的子数组是否可以满足这两个约束。 
3. 对于每个索引$r$，我们首先验证是否$A[r]$是在$[Y, X]$。 如果它在外面，我们将段开始重置为$r+1$因为任何跨越该点的子数组都会违反最小/最大约束。 
4.如果$A[r] = X$，我们更新最后看到的位置$X$。 如果$A[r] = Y$，我们更新最后看到的位置$Y$。 
5. 对于每个端点$r$，我们计算有多少个有效的起始位置$l$存在使得两者$X$和$Y$出现在$[l, r]$和$l$是在最后一次出现之后$K$以及任何超出范围的元素。 结束于的有效子数组的数量$r$是$\max(0, \min(lastX, lastY) - start)$。 

每个有效子数组结束于$r$必须不晚于最后出现的最早的$X$和$Y$，否则其中之一就会失踪。 

### 为什么它有效

 在每个位置$r$，该算法隐式地描述了所有以$r$有两个约束：它们必须完全位于当前的干净段内（没有$K$或超出范围的元素），并且它们的左端点必须定位，以便两个必需的值都出现在子数组内。 最后出现的索引完全确定子数组是否包含至少一个$X$和一个$Y$，因为在这些最后出现的最小值之前开始的任何子数组都将包含两者，而任何较晚的开始将排除其中之一。 这个不变量确保每个计数的子数组恰好满足所有约束一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, X, Y, K = map(int, input().split())
        a = list(map(int, input().split()))
        
        last_x = -1
        last_y = -1
        start = 0
        ans = 0
        
        for i, v in enumerate(a):
            if v == K:
                start = i + 1
                last_x = -1
                last_y = -1
                continue
            
            if v < Y or v > X:
                start = i + 1
                last_x = -1
                last_y = -1
                continue
            
            if v == X:
                last_x = i
            if v == Y:
                last_y = i
            
            if last_x != -1 and last_y != -1:
                ans += max(0, min(last_x, last_y) - start + 1)
    
        print(ans)

if __name__ == "__main__":
    solve()
```该实现保持一个移动的左边界，称为`start`，每当我们遇到禁止值时，它就会被强制向前$K$或任何超出允许范围的值$[Y, X]$。 这保证了每个考虑的子数组对于边界和排除规则都是有效的。 

变量`last_x`和`last_y`存储最近的位置$X$和$Y$在当前段内。 当两者都定义时，来自的每个索引`start`最多`min(last_x, last_y)`可以作为左端点，生成以以下位置结尾的子数组`i`包含两个必需的值。 

表达式`min(last_x, last_y) - start + 1`计算存在多少个这样的起始位置。 这`+1`至关重要，因为两个端点都是包容性的，缺少它会导致单长度有效扩展的系统性低估。

 ## 边缘情况

 一个关键的边缘情况是当$K$重复出现，将数组分成许多微小的部分。 在这种情况下，算法会重置`start`,`last_x`， 和`last_y`每次，确保没有子数组跨越禁止值。 对于像这样的输入$A = [7, 2, 4, 2, 7]$，有效子数组仅限于$[7]$,$[4]$， 和$[7]$段，并且算法永远不会错误地合并它们。 

另一个边缘情况是当$X$或者$Y$永远不会出现在段内。 例如，$A = [3, 3, 3]$和$X=3, Y=1$不会产生有效的子数组贡献，因为`last_y`永远不会变得有效。 该算法正确地产生零，因为条件`last_x != -1 and last_y != -1`永远不成立。 

最后一个微妙的情况发生在$X$和$Y$以相反的顺序出现。 为了$A = [1, 5]$和$X=5, Y=1$，贡献只有在两者都被看到后才开始累积，并且该公式自然地处理排序，因为`min(last_x, last_y)`捕获其中最早的需求。
