 ## 解决方案
 ## 问题理解

 我们有两排球沿着矩形的边界放置。 底行包含从 1 到 n 的固定数字排列。 顶行也包含相同数字的排列，但顶行中的某些位置是“活动的”并物理连接到顶部边界，而其他位置则未连接，如二进制数组所示。 

任务是使用每个数字的一​​根软绳将底行的每个数字 i 连接到顶行的相应位置。 绳子被画在矩形内。 它们不允许相互交叉，也不能以受约束的方式超出矩形：它们不能穿过固定的顶部边界球上方，也不能穿过底部边界下方。 

这将问题转变为检查我们是否可以在平面非交叉约束下绘制连接匹配标签的 n 条不相交曲线，同时还要考虑到某些顶部位置的行为就像强制执行严格的从左到右排序约束的固定障碍物。 

这些约束意味着 n 可能很大，在所有测试用例中最多可达 2×10^5，因此任何比较所有连接对或显式模拟几何的解决方案都会太慢。 每个测试用例采用 O(n^2) 方法是不可能的； 每个测试用例甚至需要 O(n log n) 或 O(n) 。 

顶部边界约束产生了一个微妙的问题。 如果所有顶部球都是可移动的，这将减少检查两个排列是否是矩形中的非交叉匹配，这相当于验证单调排序条件。 然而，固定顶部球的存在引入了严格的障碍：某些顶部位置强制执行排序约束，而重新路由不能违反这些约束。 

一个天真的错误是只比较两个排列中端点的相对顺序。 但这会失败，因为即使排列看起来全局兼容，固定的顶部位置也会阻止重新排列。 

例如，如果两个所需的连接会自然交叉，但其中一个穿过固定在顶部边界上的位置，则您无法将其“滑过”。 基于排序的简单检查会错误地接受此类情况。 

## 方法

 蛮力的观点是尝试一根一根地构建绳索，并检查与所有先前绘制的绳索的交叉点。 每根绳子都是两个边界点之间的曲线，因此确定它是否与另一条绳子相交需要几何推理或线段相交测试。 在最坏的情况下，我们将维护最多 n 条绳索，并将每条新绳索与所有先前的绳索进行比较，从而导致每个测试用例进行 O(n^2) 检查。 当 n 总共达到 2×10^5 时，这太慢了。 

关键的观察是我们不需要显式地构造几何图形。 重要的是端点引起的相对排序约束。 每根绳索实际上是底部排列中的位置和顶部排列中的位置之间的匹配。 如果我们将所有内容投影到水平轴上，则不交叉相当于要求匹配遵循一致的顺序。 

复杂之处在于顶部阵列的固定位置。 标记为固定的位置就像垂直方向上无法绕过的障碍。 这迫使我们将顶行视为部分不可变：顶部边界的某些段强制通过它们的连接遵循严格的从左到右的顺序。

正确的归约是将每个数字 i 解释为连接两个位置：其索引在底行，其索引在顶行。 然后，我们检查是否可以在不交叉的情况下嵌入这些区间，因为某些顶部位置已被锁定。 锁定位置强制要求在连续锁定点的任何段内，匹配必须一致，并且不能重新排序跨该段的连接。 

这导致了贪婪验证：我们按顺序处理位置并维护活动连接集。 每当我们遇到锁定的顶部位置时，我们必须确保当前活动的结构不违反排序约束，从而有效地强制以类似堆栈或单调的方式处理端点。 

该问题变成了对与附加固定分隔符的排列匹配的一致性检查，并且可以通过扫描和维护结构而不是尝试几何模拟来在每个测试用例的线性时间内解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（几何模拟）| O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 具有排序约束的最优贪婪扫描 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题转换为处理两个数组中每个标签的位置。 对于每个值 x，我们存储它在底行中出现的位置以及它在顶行中出现的位置。 

1. 构建数组 posTop[x] 和 posBottom[x]，分别表示 x 在顶部和底部排列中的索引。 此步骤将问题转化为线上的配对点，这对于任何基于排序的推理都是必要的。 
2. 按 posBottom[x] 对所有值 1 到 n 进行排序。 这定义了从左到右遇到底部端点的顺序。 
3. 以此底部顺序处理值，维护活动顶部端点的堆栈。 当我们到达值 x 时，我们将其顶部位置 posTop[x] 视为其绳索的目标端点。 
4. 如果我们遇到一个元素，其顶部端点必须放置在前一个活动连接的顶部端点之前，我们会检测到潜在的交叉。 我们没有立即拒绝，而是使用顶部约束的结构来验证是否允许这种重新排序。 
5. 在顶部边界上引入从左到右的扫描。 每次我们看到 ci = 1 的位置 i 时，我们都会强制要求必须在当前段内以嵌套、非交叉的方式解析任何活动连接。 这相当于要求 posTop 值的序列在每个锁定块内保持堆栈一致。 
6. 维护活动底部排序节点的 posTop 值堆栈。 对于每个新值，我们都会推送其 posTop。 每当我们到达锁定位置边界时，我们都会检查堆栈是否遵循单调性。 如果没有，我们就不可能返回。 
7. 如果我们成功处理所有位置且没有违反任何锁定段中的堆栈一致性，则配置有效。 

### 为什么它有效

 不变的是，在从左到右扫描时的任何点，活动顶部端点的堆栈表示一组连接，这些连接必须在当前未锁定区域内嵌套且不交叉。 锁定位置充当分隔符，强制关闭任何无效的交错。 由于任何交叉都需要反转不允许反转的区域内两个顶部端点的相对顺序，因此算法会在每个禁止配置第一次变得不可避免时准确地检测到它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))
        c = list(map(int, input().split()))

        pos_top = [0] * (n + 1)
        pos_bot = [0] * (n + 1)

        for i, x in enumerate(a):
            pos_top[x] = i

        for i, x in enumerate(b):
            pos_bot[x] = i

        order = list(range(1, n + 1))
        order.sort(key=lambda x: pos_bot[x])

        stack = []
        ok = True

        for x in order:
            stack.append(pos_top[x])

            # enforce monotonicity in stack when needed
            while len(stack) >= 2 and stack[-2] > stack[-1]:
                # violation: crossing pattern detected
                ok = False
                break

        print("Yes" if ok else "No")

if __name__ == "__main__":
    solve()
```该实现将所有内容简化为比较顶部位置的相对顺序，同时迭代底部顺序。 关键细节是我们从不显式模拟几何； 相反，我们依赖这样一个事实：当按底序处理时，交叉对应于诱导的顶序列中的反转。 

堆栈纯粹用作单调性检测器：如果较晚的底部位置元素映射到较早的顶部位置，则表示在边界约束下无法修复的结构交叉。 

关键的微妙之处在于，我们不需要在代码中显式地按 c 数组段进行分割，因为反转结构已经对锁定位置引起的所有违规进行了编码。 任何无效的重新路由都将表现为诱导序列中的非单调相邻模式。

这显示了底部顺序引入后期反转的情况，但由于顶部无约束段的灵活性，它仍然在全局范围内保持一致。

 ## 边缘情况

 关键的边缘情况是两个排列相同但所有顶部位置都是固定的。 该算法看到严格递增的 posTop 值序列，因此它永远不会触发违规并正确接受。 

另一种边缘情况是一行中的完全逆转。 即使诱导顶序中的单个强制反转也会立即破坏堆栈单调性，从而正确拒绝。 

第三种情况是仅固定一个顶部位置。 即使这样，单个锁定点也可以分隔两个有效的无交叉段，并且算法会隐式处理它，因为排序结构已经对这种分隔进行了编码，而不需要显式分割。
