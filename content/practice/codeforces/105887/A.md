 ## 解决方案
 ## 问题理解

 给定一个长度为 的循环数组$n$，其中每个位置保存一个表示某个位掩码的整数值。 该阵列以离散的轮次演变。 在$i$第轮，每个位置$j$通过与移动的另一个位置进行按位“或”，同时更新其值$i$在圆圈上向前迈出一步。 正式地，每个$A_j$变成$A_j \,\mathrm{OR}\, A_{(j+i)\bmod n}$。 

一轮中的所有更新都是同时进行的，后面的轮次对已更新的数组进行操作。 

按顺序执行几个这样的轮次后，值开始通过不同偏移量的重复“或”合并在循环中传播。 该过程逐渐将信息（位）传播到各个位置。 任务是确定最小的轮数$T$之后数组中的所有元素都变得相等。 

输出不是结果数组本身，而只是每个位置包含相同值所需的最小轮数。 

约束条件允许$n \le 10^5$，这立即排除了任何显式执行的模拟$O(n)$每轮工作的所有可能$T$，因为即使$O(n^2)$总的行为会太慢。 任何正确的解决方案都必须将过程简化为每个测试用例的封闭形式或接近恒定的计算。 

出现微妙的边缘情况时$n$非常小。 例如，如果$n = 2$，一轮已经迫使两个位置相互或，使它们立即相等。 为了$n = 3$，行为有所不同：值传播得更慢，因为每轮仅引入固定的步长，并且重叠需要多轮。 这表明答案取决于信息在整个周期中传播的速度，而不是值本身。 

另一个常见的错误是假设由于 OR 是幂等且可交换的，因此单轮或少量固定轮数就足够了。 当$n$很大并且每轮的传播距离受到限制。 

## 方法

 一种幼稚的解释是从字面上模拟该过程。 在每一轮中$i$，我们通过迭代所有位置来计算一个新数组$j$, 采取$A_j$或者$A_{j+i}$，指数模$n$。 重复此操作$T$回合费用$O(nT)$。 在最坏的情况下，如果$T$的顺序是$n$，这变成了$O(n^2)$，这远远超出了可接受的限度$n = 10^5$。 

关键的观察结果是数组的值不会影响进程的结构。 只有“可达性”的传播很重要：一旦来自位置的值$x$到达位置$y$，由于 OR 的单调性质，它可以永久贡献所有位。 

因此，我们不是跟踪值，而是跟踪哪些位置影响其他位置$T$回合。 

从传播的角度思考之后，这个过程就变成了一个循环上的图可达性问题。 回合中$i$，我们有效地添加长度的边$i$。 后$T$回合中，任何位置都可以影响通过从以下位置选择的步长总和可到达的位置$1$到$T$，因为重复的轮次构成了这些转变。 

因此，可达距离的集合是所有元素的总和$\{1,2,\dots,T\}$，意味着任何距离$$1 + 2 + \dots + T = \frac{T(T+1)}{2}.$$所以之后$T$每回合，每个位置最多可以收集距离内所有位置的信息$\frac{T(T+1)}{2}$沿着循环。 一旦这个跨度覆盖了整个周期，每个位置都会收到所有其他位置的贡献，并且所有值都变得与数组的全局 OR 相同。 

所以问题就简化为寻找最小的$T$这样：$$\frac{T(T+1)}{2} \ge n - 1.$$这是一个简单的二次阈值。

 ## 算法演练

 我们计算所需的最小轮数，以便传播距离覆盖整个周期。 

1. 阅读$n$对于每个测试用例。 数组值与最终答案无关，因为 OR 只积累信息，不影响传播速度。 
2. 将此过程解释为循环上的扩展半径问题。 后$T$每轮中，信息可以传播的最大距离等于每轮中引入的步长之和。 
3. 计算三角数$S(T) = \frac{T(T+1)}{2}$，表示经过之后可以到达的最远距离$T$回合。 
4. 找到最小的$T$这样$S(T) \ge n-1$。 这确保了每个位置最多可以到达其他所有位置$T$回合。 
5. 直接使用二次公式求解不等式。 我们需要：$$T^2 + T - 2(n-1) \ge 0.$$正根给出：$$T = \left\lceil \frac{-1 + \sqrt{1 + 8(n-1)}}{2} \right\rceil.$$6. 为每个测试用例输出该值。 

### 为什么它有效

 该过程仅通过循环移位来扩展信息，并且每一轮都会向可能的运动集合添加新的固定步长。 回合的组成对应于对这些步长进行求和。 因为 OR 是单调且不可逆的，一旦一个位置接收到一个位，它就永远不会丢失它，所以唯一的限制因素是可达移位图是否完全连接。 三角形界限准确地表征了累积移位范围何时覆盖整个周期，保证每个节点都收到了所有其他节点的贡献，从而强制平等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        
        # We only need n
        if n <= 1:
            print(0)
            continue
        
        target = n - 1
        disc = 1 + 8 * target
        root = int(math.isqrt(disc))
        if root * root < disc:
            root += 1
        
        T = (-1 + root) // 2
        if T * (T + 1) // 2 < target:
            T += 1
        
        print(T)

if __name__ == "__main__":
    solve()
```该实现完全忽略数组值，因为最终同步时间仅取决于位置交换信息的速度，而不取决于它们携带的信息。 

计算使用整数平方根以避免浮点精度问题。 估计后$T$根据二次公式，我们会根据需要对其进行修正，以确保完全满足三角条件。

 ## 边缘情况

 对于$n = 2$，传播图在一轮后就已经完全连接了。 该算法计算$n-1 = 1$，以及条件$T(T+1)/2 \ge 1$产量$T = 1$，匹配直接模拟。 

为了$n = 3$，我们需要$n-1 = 2$。 达到 2 的最小三角形数是$3$，对应于$T = 2$。 这反映出一轮仅提供邻接，而完全混合需要第二层传播。 

对于大型$n$， 例如$n = 10^5$，计算出的$T$就在附近$\sqrt{2n}$，与$n$，确保算法保持高效并避免任何线性模拟陷阱。
