 ## 解决方案
 ## 问题理解

 每个测试都描述了三个值之间的简单比较游戏：爱丽丝的力量`a`, 鲍勃的力量`b`，以及Candy的实力`c`。 输出是通过对这些比较应用优先级规则而选择的字符串。 

决策过程是连续的。 首先，Alice 尝试确定她是否直接击败了 Bob，这对应于检查是否`a > b`。 如果成立，则答案固定为`"Win"`并且不需要进一步的推理。 

如果爱丽丝没有直接击败鲍勃，游戏就会转为间接解释：如果糖果比鲍勃强，鲍勃就被认为较弱，这意味着`c > b`。 在这种情况下，输出变为`"WIN"`。 

如果两个条件都不成立，意味着 Alice 没有击败 Bob，Candy 也没有击败 Bob，结果是`"nowin"`。 

输入由多个独立的测试用例组成。 每个测试用例包含三个整数，并且每个整数都必须在相同的逻辑下进行评估。 约束足够小，可以在恒定时间内处理每个测试。 即使最多有 100 个测试用例，任何 O(T) 或 O(T log T) 解决方案都足够了，并且不需要预处理或数据结构。 

主要的微妙之处是条件的严格排序。 一个常见的错误是检查`c > b`前`a > b`，这会错误地对 Alice 已经直接获胜的情况进行分类。 另一个边缘情况是当值相等时，尤其是`a == b`或者`c == b`，这两者都必须被视为各自严格不等式的失败。 

例如，当`a = 6, b = 6, c = 100`，Alice 没有打败 Bob，但 Candy 打败了，所以正确的输出是`"WIN"`。 如果`a = 10, b = 5, c = 0`，正确的输出是`"Win"`尽管 Candy 无关紧要。 如果`a = 5, b = 10, c = 7`，这两个条件都不成立，因此输出为`"nowin"`。 

## 方法

 强力解释会尝试从字面上对叙述进行建模：模拟比较，甚至可能推理角色之间的传递性“胜利”。 人们可能会尝试构建一个关系图，其中如果一个节点击败另一个节点或通过 Candy 间接变得更强，则该节点被视为获胜。 这样的模型是不必要的，因为规则仅分解为两个直接比较。 即使实现了，这种模拟仍然会在每次测试中评估恒定数量的边缘，但它会带来复杂性而没有任何好处。 

关键的简化是认识到整个游戏是由固定优先级决策树定义的。 第一个条件`a > b`完全确定结果为真时，这意味着任何进一步的推理在该分支中都是无关紧要的。 只有当它失败时我们才检查第二个条件`c > b`。 如果两者都失败，我们默认为最终状态。 

这将问题从任何形式的关系推理减少为严格有序的条件检查。 该结构本质上是一个两级决策函数。

 ## 算法演练

 ## 算法演练

 1.读取测试用例数量`T`。 每个测试用例都是独立的，因此迭代中不会携带任何状态。 
2. 对于每个测试用例，读取三个整数`a`,`b`， 和`c`。 这些代表固定的强度，并且不对它们应用任何转换。 
3. 检查是否`a > b`。 如果该条件成立，立即输出`"Win"`。 这一步首先出现是因为问题将其定义为最高优先级规则。 
4.如果`a > b`为 false，检查是否`c > b`。 如果该条件成立，则输出`"WIN"`。 这代表了 Candy 决定 Bob 损失的次要解释。 
5. 如果两个条件都不满足，则输出`"nowin"`。 这对应于 Bob 没有被 Alice 或 Candy 击败的剩余区域。 

### 为什么它有效

 逻辑划分整个空间`(a, b, c)`分成由严格不平等定义的三个不相交的区域。 第一个条件完全捕获了 Alice 支配 Bob 的所有情况，并且首先检查它，以便没有替代规则覆盖它。 第二个条件仅在 Alice 失败时激活，并且仅依赖于 Candy 与 Bob 的比较。 由于这两项检查在执行顺序上是互斥的，并且在与最终回退结合时共同详尽无遗，因此每个输入都准确地映射到一个输出，而不会产生歧义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        a, b, c = map(int, input().split())
        if a > b:
            print("Win")
        elif c > b:
            print("WIN")
        else:
            print("nowin")

if __name__ == "__main__":
    solve()
```该解决方案读取所有测试用例一次并独立处理每个测试用例。 分支结构直接镜像决策树，确保条件的优先级完全按照指定保留。 

关键的实现细节是使用`elif`。 这保证了一旦`a > b`满足时，程序不会错误地评估第二个条件。 如果没有这个结构，就需要一个单独的`if`为了`c > b`可能会错误地覆盖正确的输出。

 ## 边缘情况

 当所有值都相等时，就会出现一种边缘情况。 用于输入`a = b = c = 10`，算法检查`a > b`失败，然后检查`c > b`这也失败了，产生`"nowin"`。 这符合规则结构，因为 Alice 和 Candy 都没有严格超过 Bob。 

另一种情况是只有平等才能区分结果，例如`a = 5, b = 5, c = 6`。 这里第一个条件由于相等而失败，但第二个条件成功，因为 Candy 严格超过 Bob，产生`"WIN"`。 这证实了相等不会触发任何获胜条件。 

最后一种情况是，无论Candy如何，Alice都严格更强，例如`a = 100, b = 1, c = 0`。 该算法在第一次检查时立即停止并返回`"Win"`，证明 Candy 从未在该分支中被评估。
