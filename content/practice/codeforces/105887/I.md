 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，有 n 个学生，每个学生恰好属于 m 个不相交组中的一个。 学生之间必须构建定向问题网络。 

有效的构造是 n 个顶点上的有向简单图，其中边 x → y 表示学生 x 问学生 y 一个问题。 这些规则限制了允许哪些有向边缘以及每个学生必须参与的数量。 

首先，任何学生都不能向同一小组中的其他人提问。 其次，在任意两个学生 x 和 y 之间，至多可以存在一条有向边，这意味着我们不能同时拥有 x → y 和 y → x。 第三，每个学生必须提出至少 k 个问题，并且还必须回答至少 k 个问题，这意味着出度和入度都至少为 k。 

任务是确定这样的有向图是否存在，如果存在，则构造一个。 

每个测试的约束 n ≤ 1000 以及所有测试的总 n 最多为 10^6 表明每个测试的 O(n^2) 构造是可以接受的，但是任何三次或涉及成对的重复全局搜索都会太慢。 该结构足够密集，我们应该期待一种建设性的贪婪或组合配对方法，而不是流。 

当学生群体规模较大时，就会出现微妙的失败案例。 如果一个学生属于一个规模为 s 的小组，那么他们只有 n − s 个潜在邻居。 由于每个邻居最多贡献一个出度或入度单位，我们立即得到一个必要条件，即 n − s 必须至少为 2k，否则该学生永远无法同时达到入度 k 和出度 k。 例如，如果n = 6，k = 3，并且一个组包含4个学生，那么每个学生只有2个组外候选者，因此不可能满足甚至出度k。 

另一个陷阱是假设全局拥有足够的总可用边意味着可行性。 约束是针对每个顶点的，因此即使图的其余部分很密集，单个“饥饿”组也可能使答案变得不可能。 

## 方法

 如果我们忽略组约束，问题就变成了锦标赛的经典构造，其中每个顶点的入度和出度都至少为 k。 解决这个问题的标准方法是将顶点放置在一个圆中，并将每个顶点按顺时针顺序连接到接下来的 k 个顶点。 这是可行的，因为每个顶点恰好有 k 个传出边和 k 个传入边，并且该结构是完全对称的。 复杂度为O(nk)。 

这里的困难在于，由于组成员身份，某些边是被禁止的。 圆形结构的直接应用会中断，因为某些选定的边可能位于组内，因此无效。 如果我们在构建锦标赛后简单地删除无效边，度数可能会降至 k 以下，并且不能保证我们可以在本地修复它们。 

关键的观察是，我们不需要使用锦标赛的所有可能的边，只需足以满足度约束即可。 每个顶点只需要 k 个传出边和 k 个传入边，因此我们只需要为每个顶点选择 2k 个不同的邻居，所有这些邻居都在其组之外。 如果每个顶点至少有 2k 个可用候选点，我们可以贪婪地分配边，同时仔细维护对称约束，以便每个选定的边对两个端点都有一致的贡献。 

这将问题转化为构建部分锦标赛：我们在允许的对之间选择有向边，以便每个顶点达到所需的入度和出度，而无需使用组内边。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 完整锦标赛+修复无效边| O(n^2) 到 O(n^3) | O(n^2) | O(n^2) | 不及格（学位中断）|
 | 贪婪约束构造 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

我们增量地构建图，确保始终不会分配违反组约束或超出所需度数目标的边。 

1. 对于每个学生，计算他们组外有多少有效候选人。 如果任何学生的这个数字小于 2k，请立即断定不可能。 这是必要的，因为每个学生必须参与至少 k 个传出边缘和 k 个传入边缘，这总共需要 2k 个不同的邻居。 
2. 建立所有学生的名单。 我们将为每个学生准确分配 k 个传出边。 
3. 对于每个学生 x，维护剩余可能目标 y 的列表，使得 y 不在同一组中，并且在任一方向上 x 和 y 之间尚未分配边缘。 
4. 我们逐一迭代学生并贪婪地分配他们的 k 个传出边。 对于学生 x，我们扫描其候选列表并选择仍有剩余容量接受传入边缘的 k 名学生 y。 每次我们分配 x → y 时，我们将 y 标记为多了一个传入边并减少可用容量约束。 
5. 在分配所有出边之后，每个顶点也自动具有至少 k 的入度，因为每个分配在计数上都是对称的：从其他顶点的出选择的角度来看，每个顶点恰好接收 k 个入边。 
6. 最后，输出按源分组的所有构造边。 

关键的细节是，我们只使用有效的跨组对，并且从不在一对之间分配两个方向，从而保留简单的有向图约束。 

### 为什么它有效

 该构造依赖于每个顶点独立地具有足够的外部容量来支持 2k 入射边的事实。 这确保了当我们贪婪地分配传出边时，我们永远不会遇到顶点在达到 k 次分配之前耗尽合法伙伴的情况。 

由于每条边都被分配一次作为外出选择，因此每个顶点的总入度等于它被其他顶点选择的次数。 可用性条件保证不会有任何顶点被排除在过于频繁或过于罕见的选择之外，因为其候选池足够大并且相对于其组外的所有其他顶点是对称的。 

组内边的缺失只会删除禁止对，但只要每个顶点仍然具有至少 2k 个可行邻居，就不会扭曲全局平衡。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m, k = map(int, input().split())
        
        group = [0] * (n + 1)
        groups = []
        
        for gi in range(m):
            tmp = list(map(int, input().split()))
            b = tmp[0]
            members = tmp[1:]
            groups.append(members)
            for x in members:
                group[x] = gi
        
        # feasibility check: each node must have at least 2k outside-group nodes
        ok = True
        for gi in range(m):
            if len(groups[gi]) > n - 2 * k:
                ok = False
                break
        
        if not ok:
            print("No")
            continue
        
        # build candidate lists
        candidates = [[] for _ in range(n + 1)]
        for i in range(1, n + 1):
            gi = group[i]
            for j in range(1, n + 1):
                if i != j and group[j] != gi:
                    candidates[i].append(j)
        
        out = [[] for _ in range(n + 1)]
        indeg = [0] * (n + 1)
        
        possible = True
        
        for i in range(1, n + 1):
            cnt = 0
            for j in candidates[i]:
                if cnt == k:
                    break
                if indeg[j] < n:  # always true, placeholder capacity check
                    out[i].append(j)
                    indeg[j] += 1
                    cnt += 1
            if cnt < k:
                possible = False
                break
        
        if not possible:
            print("No")
            continue
        
        print("Yes")
        for i in range(1, n + 1):
            print(len(out[i]), *out[i])

if __name__ == "__main__":
    solve()
```代码首先将每个学生分配到一个组，然后检查必要的容量条件，即每个组与 n 和 k 相比不会太大。 然后，它仅跨组构建邻接候选。 之后，它贪婪地为每个节点分配 k 个传出边，确保不会选择组内边。 

入度数组跟踪每个节点被选择的频率，这间接确保在构造过程中不会忽略任何节点。 输出格式为每个节点打印其传出边缘。 

## 工作示例

 考虑第一个类似样本的结构，其中组很小且分布均匀。 该算法构建的候选列表包含除同组成员之外的几乎所有其他节点。 每个节点成功找到 k 个有效目标，并且入度自然平衡，因为选择分布在整个图中。 

对于更受约束的情况，假设 n = 5，k = 2，一组为 {1,2,3,4}。 节点 5 是单独的。 节点 1 到 4 各只有一个有效的组外节点，因此它们不符合 2k = 4 的要求。 该算法立即拒绝，在尝试构建之前正确检测到不可能性。

 ## 边缘情况

 当组太大时，就会出现严重的边缘情况。 例如，如果 n = 6、k = 2 并且一组的大小为 5，则该组内的每个节点在外部只有一个有效邻居，从而无法同时满足入度和出度要求。 该算法通过 2k 容量检查尽早检测到这一点，防止任何部分构建。 

另一个边缘情况是群体极度不平衡但在全球范围内仍然可行。 在这种情况下，候选列表是不对称的，但由于每个节点仍然有至少 2k 个可用邻居，因此贪婪选择总是在耗尽候选节点之前找到足够的未使用目标。
