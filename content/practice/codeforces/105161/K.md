 ## 问题理解

 我们得到一个多重集或整数数组。 游戏中两名玩家交替轮流。 在每一回合中，允许玩家移除结构中存在的当前最大值的一次出现。 删除它后，玩家还可以附加严格小于该最大值的任何元素子集，从声明中描述的一些可用的“类似前缀”池中选择，包括不附加任何内容的可能性。 

游戏继续进行，直到没有有效的移动，不能移动的玩家就输了。 任务是根据初始配置确定第一个玩家是否强制获胜，或者第二个玩家是否始终可以做出最佳响应以获胜。 

尽管该操作听起来很丰富，但关键的隐藏结构是唯一的全局相关量是最大值最初出现的次数。 

从复杂性的角度来看，输入大小与元素数量成线性关系。 这立即排除了对博弈树的任何模拟。 任何尝试在每次删除后对状态进行建模或跟踪所有可能的附加配置的方法都会发生组合爆炸，因为每次移动都会分支成许多可能的子集。 

唯一可行的解​​决方案必须将游戏简化为在线性时间内计算的单个聚合统计数据。 

一个微妙的边缘情况是所有元素都相等。 在这种情况下，每一步都会简单地删除一个最大值，并且不存在有意义的“较小前缀”。 天真的读者可能会认为可选的附加操作可能会影响结果，但它永远不会改变游戏完全由重复删除最大元素决定的事实。 

例如，如果数组是`[5, 5, 5]`，每一步都会移除一个`5`。 游戏在三步后结束，因此第一个玩家获胜。 如果是的话`[5, 5]`，第二个玩家获胜。 这已经暗示了一种奇偶现象。 

## 方法

 暴力方法将明确地模拟游戏。 每个状态将存储当前的多重集，并且每个转换将删除一个最大值并分支到要添加的较小元素的所有可能子集。 这很快就变得不可行，因为较小元素的子集数量可能呈指数级增长，并且游戏状态的数量呈爆炸性增长。 即使对于中等`n`，这种方法无法超越非常小的实例。 

关键的观察是附加元素永远不会影响最大值的身份，除非它们与当前最大值匹配，这是禁止的，因为只能添加较小的元素。 因此，仅当当前最大值的所有出现都已被删除时，多重集中的最大值才会严格减少。 低于最大值的内部结构永远不会影响最大删除的计数。 

这将游戏简化为纯粹的计数过程：最大值的每次出现对应于控制交替的一个基本动作。 可选操作不影响去除最大值的次数的奇偶性，完全决定了最优玩法下的胜负。 

因此，问题归结为计算有多少元素等于全局最大值并检查该计数是奇数还是偶数。

 ## 算法演练

 我们现在以直接顺序描述最优推理过程。 

1. 读取输入数组并确定存在的最大值。 这是决定游戏中强制结构的唯一值，因为所有操作都围绕着删除最大值。 
2. 计算该最大值在数组中出现的次数。 该计数代表游戏中基本移动的总数，因为每个移动恰好删除一个最大值。 
3. 根据本次计数的奇偶性确定获胜者。 如果最大元素的数量是奇数，则第一个玩家走最后一步并获胜。 如果相等，则第二个玩家走最后一步并获胜。 

关键思想是，无论玩家如何使用可选插入较小元素，都永远无法增加剩余最大元素的数量，也无法改变每个最大值最终都必须被一一移除的事实。 

### 为什么它有效

 不变量是最大元素的多重集每次移动恰好减少一个，与任何二次操作无关。 所有附加插入都涉及严格较小的值，因此它们不会影响当前的最大层。 

由于这种分离，游戏分解为固定长度的序列`k`强制移动，其中`k`是最大元素的数量。 玩家交替这些强制动作，无法跳过或合并它们。 因此，获胜者完全取决于是否`k`是奇数还是偶数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = list(map(int, input().split()))
    if not data:
        return
    n = data[0]
    arr = data[1:1+n]

    mx = max(arr)
    cnt = 0
    for x in arr:
        if x == mx:
            cnt += 1

    print("First" if cnt % 2 == 1 else "Second")

if __name__ == "__main__":
    solve()
```该解决方案首先一次性解析输入。 它使用 Python 的内置函数计算最大值，然后使用简单的循环计算其出现次数。 这避免了任何不必要的数据结构。 

一个常见的实现陷阱是忘记输入格式可能会导致`n`与数组值在同一行，因此必须谨慎处理直接拆分。 另一个微妙之处是确保只计算等于最大值的元素，而不计算大于或等于比较的元素，这些元素会意外地在转换后的数据中包含重复项。

 ## 边缘情况

 对于像这样的单元素数组`[x]`，最大值出现一次，第一个玩家通过删除它立即获胜。 算法正确识别计数`1`并返回第一玩家获胜。 

对于所有元素都相同的数组，例如`[7, 7, 7, 7]`，每个元素都是最大值。 计数为 4，因此第二个玩家获胜。 模拟视图与此完全匹配，因为每次移动都会删除一个元素，直到耗尽。 

对于具有唯一最大值和许多较小元素的数组，例如`[10, 1, 1, 1]`，最大计数为 1，保证先手者获胜。 任何使用“附加较小前缀”操作的尝试都不会改变只存在一个决定性着法的事实。
