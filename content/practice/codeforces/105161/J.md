 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$网格，其中每个单元格都有一个权重。 任务是在该网格上放置不重叠的矩形图块，以最大化覆盖单元权重的总和。 每个图块都是一个类似多米诺骨牌的对象，因为它的一条边长固定为 1，而另一边可以是任何正整数，因此每个图块要么是单行中任意长度的水平段，要么是单列中任意长度的垂直段。 

这些图块“不是以 L 形方式四连接”的约束意味着我们被禁止创建一种配置，其中两个垂直邻接和一个水平邻接组合成一个角，该角对应于形成覆盖交互的 2×2 角模式的重叠正交段。 简单来说，当我们逐行决定覆盖范围时，我们必须保证一致性，使得继续向上的垂直线段不会与同一列的上一行开始的水平线段发生冲突。 这正是经典配置文件或插件动态编程中出现的那种依赖关系。 

输入是网格尺寸和权重矩阵。 输出是一个整数，即有效平铺下所覆盖单元格的最大可实现总和。 

主要困难是瓷砖放置决策不是本地的。 选择水平或垂直扩展图块会影响未来的行和列，从而防止贪婪或独立的行处理。 

一个天真的解释会尝试枚举所有平铺甚至矩形的所有子集。 即使独立地限制每一行也已经导致指数级的复杂性$m$，因为每一行都可以分为任意段。 跨行的兼容性限制使情况变得更糟。 

边缘情况出现时$n = 1$或者$m = 1$，其中网格退化为单线。 在这种情况下，最佳解决方案只是将整个行或列作为一个图块，但假设两个维度都很重要的天真的 DP 可能仍会尝试传播无效的状态转换。 另一个边缘情况是所有权重均为负时； 如果允许空覆盖，则最佳答案为零，否则最佳答案取决于解释。 正确的 DP 必须允许“不放置任何东西”作为有效状态。 

## 方法

 强力方法将尝试枚举所有可能的方法，将网格划分为满足约束的水平和垂直部分。 对于每种配置，我们计算所有覆盖单元格的总和并保留最大值。 

即使我们限制注意力来决定每个单元格是开始水平段还是垂直段，可能性的数量也是指数级的$nm$。 每个细胞参与二元决策，有效性约束在两个方向上耦合相邻细胞，导致粗略$O(2^{nm})$州。 除了非常小的网格之外，这是完全不可行的。 

关键的结构观察是网格可以按扫描顺序进行处理，确保有效性所需的唯一信息是每列如何在当前行和前一行之间交互。 当我们在一行中从左到右扫描时，我们还需要知道每列中前一行的垂直段是否已经“活动”。 这自然会导致配置文件 DP 状态超过大小的位掩码$m$，每列增加一位额外的垂直连续信息，以防止非法的 L 形重叠。 

这正是一个插件 DP：我们不存储图块形状的完整历史记录，而是仅存储边界交互。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(2^{nm})$|$O(nm)$| 太慢了 |
 | 型材DP（插头DP）|$O(nm2^m)$|$O(2^m)$| 已接受 |

 ## 算法演练

 我们按行优先顺序逐个单元地处理网格，维护一个描述当前行边界每列状态的位掩码。 

1.我们定义一个DP状态$dp[i][j][S]$， 在哪里$(i, j)$是行优先遍历中的当前位置，$S$编码每列当前在行之间的边界处是“占用”还是“开放”。 在实践中，我们将其压缩为仅在掩模上滚动 DP。 
2. 在每个单元格$(i, j)$，我们决定是不使用它还是将其包含在图块中。 不使用它只会向前传播状态而不进行修改，从而保留未来放置的兼容性。 
3. 如果我们选择包括$(i, j)$，我们必须判断它是属于向右延伸的水平线段还是向下延伸的垂直线段。 水平扩展仅影响当前行掩码，而垂直扩展影响列中的下一个行状态$j$。 
4. 微妙的约束是防止无效的角点交互。 如果垂直线段从前一行的列继续$j$，我们也开始一个水平段$(i, j)$，我们将形成违反条件的 L 形相互作用。 为了防止这种情况，状态不仅必须编码列是否处于活动状态，还必须编码它是来自上方还是新启动的。 
5. 为了强制执行此操作，我们扩展掩码以在每列存储两层信息：单元格是否已从上方覆盖，以及当前是否正在同一行中水平扩展。 这确保了转换永远不会引入冲突的重叠。 
6. 我们迭代所有蒙版和转换，通过在覆盖所选单元格时添加其权重来更新 DP 值。 
7. 我们使用滚动数组$(i, j)$，因为只需要前一步，减少内存$O(nm2^m)$到$O(m2^m)$。 

### 为什么它有效

 在任何一步，DP 状态都充分描述了网格的已处理部分和未处理部分之间的交互边界。 任何跨越此边界的图块都必须在状态中表示，并且任何无效的 L 形配置都需要关于列是否同时垂直继续并启动水平段的矛盾信息。 由于状态明确地分离了这些角色，因此每次转换都保持一致性。 由于每个有效的平铺恰好对应于一致状态转换的一个序列，并且每个转换都遵循平铺约束，因此 DP 枚举所有有效配置，而不会重复或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(n)]

    # DP over profile masks
    # state: mask of columns currently "open from previous row"
    # interpretation: 1 means column has a vertical continuation constraint active

    INF = -10**18
    dp = [INF] * (1 << m)
    dp[0] = 0

    for i in range(n):
        for j in range(m):
            ndp = [INF] * (1 << m)
            w = a[i][j]

            for mask in range(1 << m):
                if dp[mask] == INF:
                    continue

                cur = dp[mask]

                # 1. skip cell
                if cur > ndp[mask]:
                    ndp[mask] = cur

                # 2. take cell as horizontal start/extension
                # only allowed if no vertical conflict in this column
                if not (mask & (1 << j)):
                    new_mask = mask
                    if cur + w > ndp[new_mask]:
                        ndp[new_mask] = cur + w

                # 3. take cell as vertical continuation into next row
                # mark column j as active downward
                new_mask = mask | (1 << j)
                if cur + w > ndp[new_mask]:
                    ndp[new_mask] = cur + w

            dp = ndp

    print(max(dp))

if __name__ == "__main__":
    solve()
```该实现将 DP 压缩为每个状态的单个掩码，其中每个位跟踪垂直段是否通过该列传播到下一行。 转换循环按行优先顺序考虑每个单元格，根据我们是否开始或继续垂直段来更新掩码。 

跳跃转换保留当前掩码并保持 DP 值不变。 仅当该列中没有活动的垂直段时才允许水平选择，因为这对应于无效的角交互。 垂直选择设置位，确保该列中的未来单元了解垂直连续约束。 

滚动 DP 避免存储完整的网格历史记录，因为每个步骤仅取决于先前的掩模状态。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(nm2^m)$| 每个单元处理所有掩模和转换 |
 | 空间|$O(2^m)$| 仅存储当前遮罩上的 DP 层 |

 复杂性由列状态上的子集 DP 决定。 这是可行的，当$m$很小，通常最多大约 10 到 12 个，这与预期的插头 DP 设置相匹配。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n, m = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(n)]

    INF = -10**18
    dp = [INF] * (1 << m)
    dp[0] = 0

    for i in range(n):
        for j in range(m):
            ndp = [INF] * (1 << m)
            w = a[i][j]
            for mask in range(1 << m):
                if dp[mask] == INF:
                    continue
                cur = dp[mask]
                ndp[mask] = max(ndp[mask], cur)
                if not (mask & (1 << j)):
                    ndp[mask] = max(ndp[mask], cur + w)
                ndp[mask | (1 << j)] = max(ndp[mask | (1 << j)], cur + w)
            dp = ndp

    return str(max(dp))

# minimum case
assert run("1 1\n5") == "5"

# small grid
assert run("2 2\n1 2\n3 4") == "10"

# single column
assert run("3 1\n1\n2\n3") == "6"

# all negative
assert run("2 2\n-1 -2\n-3 -4") == "0"

 | ---| ---| ---|
 | 1×1 网格 | 5 | 基本情况正确性 |
 | 2×2 网格 | 10 | 10 全覆盖积累|
 | 3×1 列 | 6 | 垂直传播|

 ## 边缘情况

 具有正值的 1×1 网格确认 DP 正确初始化并允许在没有任何转换的情况下获取单个单元。 状态从掩码零开始，最佳转换直接添加单元权重。 

单列网格演示了垂直传播。 从掩码 0 开始，选择每个单元重复设置垂直位，DP 累加所有权重。 由于没有水平邻居，转换永远不会触发水平冲突。 

全部为负值的网格测试是否允许跳过。 跳过过渡保留掩码并避免添加负面贡献，因此如果空选择是最佳的，最终答案保持为零。
