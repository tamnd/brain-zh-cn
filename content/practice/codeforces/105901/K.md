 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中都有多个赌场和多个现有玩家。 对于每个赌场，每个现有玩家都已经投入了非负数量的骰子。 我们是最后一个玩家，必须选择在每个赌场中放置多少个骰子。 

一旦赌场的所有选择都确定下来，游戏就不会简单地选择最大值。 相反，首先进行过滤步骤：至少两个不同玩家出现的任何骰子计数值都将完全从该赌场的考虑范围中删除。 删除所有重复计数后，比较剩余的未删除值，剩余数字最高的玩家赢得该赌场。 如果删除重复项后没有留下任何内容，则该赌场不会产生赢家。 

我们的决定是一个价值向量，每个赌场都有一个。 对于每个赌场，我们可以选择 10^9 以内的任何非负整数。 所有赌场经过独立评估后，每个玩家都有自己赢得的赌场数量。 我们的目标是最大化我们赢得的赌场数量，但对所有现有玩家有一个平局规则：我们的获胜次数必须至少与其他玩家的获胜次数一样多。 在所有这些有效策略中，我们希望最小化我们放置的骰子总数。 

重要的结构性事实是，除了对玩家获胜次数的全局限制之外，每个赌场都是独立的。 输出只是我们选择的骰子值数组。 

约束很小：n 和 m 最多为 50，因此每个测试用例最多有 2500 个值 ai,j。 这强烈表明我们可以为每个测试用例进行 O(nm) 或 O(nm log m) 推理，但任何 n 或 m 的立方体仍然是安全的； 对每个赌场的选择进行指数搜索是不可能的，因为每个 bi 的上限为 10^9。 

如果我们忽略重复删除规则，就会出现微妙的失败情况。 例如，如果在赌场中玩家的值为 [5, 5, 4]，则两个 5 都会消失，只剩下 4。 天真的“击败最大值”策略会选择 6，但这是不必要的，有时在全局范围内并不是最优的，因为我们只需要击败最强的幸存唯一值，而不是原始最大值。 

另一个微妙的问题是玩家之间的对称性：胜利是按每个玩家计算的，并且多个玩家可以平局获胜计数。 我们的策略必须确保我们不比任何一个对手更差，而不仅仅是最强的对手。 

## 方法

 暴力观点会尝试为每个赌场决定我们想要的结果：要么我们赢了，我们让别人赢了，或者没有赢家，然后相应地分配 bi。 然而，重复删除和最大选择之间的相互作用使得直接枚举变得复杂。 即使我们为每个赌场确定了一个目标获胜者，选择强制执行它的 bi 也需要仔细的约束：我们必须严格大于该赌场最终生存的最大值才能获得预期的胜利，或者如果我们希望其他人获胜，则严格避免所有关键值。 

关键的观察是扭转视角。 我们不是直接决定 bi，而是研究如何才能赢得单个赌场的胜利。 删除重复项后，只有玩家之间唯一的值才重要。 因此，在每个赌场中，唯一相关的对手值是那些在该行中恰好出现一次的 ai,j。 在这些独特的价值中，最大的一个是我们想要赢得那个赌场就必须超越的门槛。 任何重复的值都是无关紧要的，因为无论我们如何选择，它都会被删除。 

因此，每个赌场都简化为一个简单的结构：计算对手之间恰好出现一次的一组值； 如果该集合为空，则除非我们自己引入独特的值，否则没有人会获胜，但我们选择的任何值都将变得独特，因为我们是唯一放置它的玩家。 所以在这种情况下，我们选择0总能获胜。

如果唯一对手值的集合非空，则令 mx 为其最大值。 为了赢得赌场，我们必须选择bi严格大于mx。 这样的最小选择是 mx + 1。如果我们不想赢得赌场，选择 bi = 0 就足够了，因为 0 总是安全的：要么它是重复的（如果任何对手有 0 并且我们引入了第二个 0），要么如果我们不试图击败任何东西，那么它是无关紧要的。 

现在，全局约束通过每个对手的获胜次数来耦合赌场。 对于每个对手j，我们必须保证我们的获胜次数至少是我们选择后j为获胜者的赌场的数量。 但由于每个赌场获胜者仅取决于本地阈值，因此我们可以独立对待每个对手：我们希望确保赢得所有赌场，否则除非我们干预，否则给定对手将成为唯一获胜者。 

这导致了贪婪对齐：对于每个赌场，我们检查是否存在一个对手，其唯一最大值严格大于所有其他对手的唯一最大值。 如果是这样，除非我们击败他们，否则对手就会获胜，而保持所有玩家可行性的最便宜的方法就是尽可能控制此类赌场。 

因为n很小，我们可以直接计算每个赌场的候选阈值，并独立决定是否有利于赢得它或让它被其他人拿走，但受到我们的获胜计数必须主导每个玩家的最终获胜计数的要求的限制。 最佳结构最终是：我们必须赢得每个赌场，否则某些对手将获得无法在其他地方补偿的“关键”获胜优势，并且在这些强制获胜中，我们使用最小 bi = mx + 1。 

因此，最终的解决方案减少到每个赌场计算唯一值最大值并选择 0 或 mx+1，同时确保此选择产生的获胜计数至少等于每个对手的获胜计数。 因为选择更多的胜利只会增加成本，所以我们永远不会自愿赢得赌场，除非有必要满足支配性约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过全局模拟对每个赌场进行强力分配 | n 中的指数 | O(纳米) | 太慢了 |
 | 最优唯一值减少 + 每个赌场的贪婪决策 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 对于每个赌场，计算每个值在 m 个现有玩家中出现的次数。 这让我们能够识别该赌场中哪些值是唯一的。 此步骤是必要的，因为只有唯一值才能在删除阶段幸存下来。 
2. 对于每个赌场，收集仅出现一次的所有值。 这些是过滤步骤后唯一可能确定现有玩家中获胜者的值。 
3. 如果赌场中没有唯一值，则设置 bi = 0。原因是所有对手的贡献在重复删除后消失，因此除非我们引入唯一的数字，否则任何现有玩家都无法获得获胜值。 选择 0 不会为我们创造获胜的局面，它会保持赌场中立。 
4. 否则计算 mx，即该赌场中唯一值中的最大值。 这代表了幸存对手的最强贡献。 
5. 设置bi = mx + 1。这保证了我们的价值严格大于每个幸存对手的价值，因此在重复删除后我们成为该赌场的赢家。 
6.记录在这个构造下我们赢了哪些赌场，并验证对于每个对手j，他们赢的赌场数量不超过我们的。 如果任何对手超过我们，则通过将获胜赌场的最小可能子集变为 bi = 0 进行调整，优先考虑 mx 最小的赌场，因为这些是牺牲的最便宜的胜利。 

贪心调整是安全的，因为减少赢钱只会影响单个赌场，而且我们总是以最受控的方式同时降低成本和赢钱数量。 

### 为什么它有效

删除重复项后，每个赌场都会减少到一组不同的有效对手优势。 获胜者仅由该组的最大值加上我们引入的任何值决定。 因此，每个赌场的行为都像一个阈值问题：要么我们保持在阈值以下并输，要么超过阈值并赢。 由于各个赌场的阈值是独立的，因此任何可行的全局解决方案都可以转化为我们只选择最小获胜值的解决方案，否则选择 0。 这种转变永远不会增加成本并保留获胜关系，这保证了对必要胜利的贪婪选择是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n, m = map(int, input().split())
        a = [list(map(int, input().split())) for _ in range(n)]

        res = []
        win_threshold = []

        # compute per-casino threshold
        for i in range(n):
            freq = {}
            for j in range(m):
                v = a[i][j]
                freq[v] = freq.get(v, 0) + 1

            uniq_max = 0
            has_uniq = False

            for v, c in freq.items():
                if c == 1:
                    has_uniq = True
                    if v > uniq_max:
                        uniq_max = v

            if not has_uniq:
                res.append(0)
                win_threshold.append(-1)
            else:
                res.append(uniq_max + 1)
                win_threshold.append(uniq_max)

        # compute opponent wins under this construction
        opp_wins = [0] * m
        my_wins = 0

        for i in range(n):
            if res[i] == 0:
                continue

            freq = {}
            for j in range(m):
                v = a[i][j]
                freq[v] = freq.get(v, 0) + 1

            uniq_max = -1
            for v, c in freq.items():
                if c == 1:
                    uniq_max = max(uniq_max, v)

            # we win this casino if res[i] > uniq_max
            if uniq_max == -1 or res[i] > uniq_max:
                my_wins += 1

        # compute opponents wins
        for i in range(n):
            freq = {}
            for j in range(m):
                v = a[i][j]
                freq[v] = freq.get(v, 0) + 1

            uniq_vals = {v for v, c in freq.items() if c == 1}
            if not uniq_vals:
                continue

            mx = max(uniq_vals)

            # winner is max unique value holder
            for j in range(m):
                if a[i][j] == mx:
                    opp_wins[j] += 1
                    break

        # adjust if needed (greedy reduction of wins)
        for i in range(n):
            if my_wins >= max(opp_wins):
                break
            if res[i] > 0:
                res[i] = 0
                my_wins -= 1

        out.append(" ".join(map(str, res)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先将每个赌场压缩成频率图，以检测哪些值能够避免重复消除。 然后，它根据是否存在任何唯一值来分配最小获胜值 mx+1 或零。 之后，它会明确计算我们和所有对手的获胜计数，并最终在任何对手超过我们的获胜计数时执行受控的减少步骤。 

一个微妙的实现细节是我们多次重新计算频率图。 给定 n，m ≤ 50，这是可以接受的，并且使代码比维护共享预计算结构更简单。 调整阶段只会让赢钱的赌场变成输钱的赌场； 它永远不需要增加任何值，从而保持了正确性。

 ## 边缘情况

 一种边缘情况是赌场中所有对手的值都重复。 对于像 [5, 5, 5] 这样的输入，每个值都会在过滤阶段被删除。 该算法正确分配 bi = 0，因为没有有意义的阈值。 任何正值仍然会使我们成为唯一幸存的条目，但 0 是最佳的，因为我们正在最小化总和。 

另一种情况是只有一个唯一值并且该值非常大。 例如 [100000000, 100000000, 1]。 只有 1 个幸存下来，因此 mx = 1，我们选择 2。简单的方法可能会错误地与 100000000 进行比较，并选择一个围绕该范围的值，这是不必要的。 

最后一个微妙的情况是多个赌场具有相同的阈值。 由于每个赌场都是独立的，因此选择最小获胜值不会产生隐藏的依赖性。 即使两个赌场都要求我们超过值 10，在两个赌场中分配 11 也是安全的，并且不会跨赌场交互。
