 ## 解决方案
 ## 问题理解

 我们得到了几个整数数组，对于每个数组，我们希望构建一个具有非常具体的结构属性的子序列。 

选取任何选定的子序列并对其进行排序。 令最小元素为左端点，最大元素为右端点，中位数为排序顺序中位于 ceil(k/2) 位置的元素。 如果中位数恰好是最小值和最大值的算术平均值，则该子序列被称为良好。 

换句话说，如果排序后的子序列是D，那么我们要求2·D[ceil(k/2)]等于D[1] + D[k]。 该条件耦合了所选集合的三个不同阶统计量，这立即表明任意子序列受到严格约束。 

任务是为每个测试用例找到从原始数组中提取的良好子序列的最大可能长度，同时保留相对顺序。 

约束总体上很小，所有测试用例的 n 总和不超过 3000。这消除了对 n log n 或 n log^2 n 优化的任何需要，并强烈建议采用 O(n^2) 甚至 O(n^2 log n) 策略。 

如果尝试贪婪或局部推理，一个微妙的问题就会立即出现。 该条件涉及中位数，这取决于所选子序列的大小，因此删除或添加单个元素可以更改哪个元素成为中位数并完全改变有效性。 例如，某个区间可能在包含其所有元素的情况下不满足条件，但在删除一些内部值后变得有效。 这会扼杀天真的“取一个值范围并检查一次”的想法，除非结构保证最优性。 

## 方法

 一种直接的暴力方法会枚举所有子序列，通过排序检查每个子序列，并验证中值是否等于最小值和最大值的平均值。 这是正确的，但不可行。 有 2^n 个子序列，即使检查一个子序列也会花费 O(n log n)，因此最坏的情况会远远超出限制。 

关键的结构观察是条件仅取决于排序子序列的三个值：最小值、最大值和中值。 如果我们固定最小 x 和最大 y，则中位数将强制为 (x + y) / 2，前提是该值是整数并且出现在数组中。 这将问题从任意子序列减少到选择端点对。 

一旦 x 和 y 固定，任何候选子序列都必须完全位于值范围 [x, y] 内，否则最小值或最大值将会改变。 在该范围内的所有元素中，我们希望选择仍保持 (x + y) / 2 作为中位数的最大可能子序列。 

这将问题转换为检查每对 (x, y) 中 [x, y] 中元素的完整多重集是否满足中值约束，如果满足，我们将采用所有元素。 尽管原则上我们可以删除元素，但这样做只会减少长度，并且一旦整个区间已经满足中值条件就无济于事。 

因此，我们开始协调压缩和前缀频率。 对于每对压缩值 i ≤ j，我们将 U[i] 视为 x，将 U[j] 视为 y。 我们计算中点 m = (x + y) / 2，检查它是一个整数并且存在于数组中，然后评估区间中有多少个元素小于 m、等于 m 和大于 m。 这些计数完全确定了中位位置条件。 

所有对的暴力破解都是 O(n^2)，并且使用前缀和的每次检查都是 O(1)，这完全符合总约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子序列的暴力破解 | O(2^n · n log n) | O(2^n · n log n) | O(n) | 太慢了 |
 | 将端点与前缀计数配对 | O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 ### ## 算法演练

1. 将数组中的所有值压缩到一个排序的唯一列表 U 中，并在 U 上构建一个频率数组。 

这允许使用前缀和在恒定时间内回答对值的范围查询。 
2. 在频率数组上构建前缀和，以便我们可以快速计算有多少元素落入任意值区间 [U[i], U[j]]。 
3. 迭代所有可能的左端点 i，将 U[i] 视为候选最小值。 
4. 对于每个 i，迭代所有右端点 j ≥ i，将 U[j] 视为候选最大值。 
5. 计算 x = U[i] 和 y = U[j]。 如果 (x + y) 是奇数，则跳过该对，因为没有整数中位数可以满足条件。 
6. 计算 m = (x + y) / 2 并将其放入压缩数组中。 如果 m 不存在，则跳过这一对。 
7. 使用前缀和，计算 L 作为 A 中具有 [x, m) 值的元素数量，E 作为等于 m 的元素数量，R 作为 (m, y] 中的元素数量。
 8. 令 k = L + E + R 且 t = ceil(k / 2)。 当且仅当 L < t 且 t ≤ L + E 时，子序列才有效。 
9. 如果有效，则用 k 更新答案，因为采用完整间隔给出了该 (x, y) 的最大可能子序列。 

### 为什么它有效

 固定最小值和最大值会强制每个有效子序列位于闭值区间内。 在这样的区间内，在不改变端点的情况下影响中位数的唯一方法是删除内部元素，但删除元素只会缩小 k 并移动中位数位置，而这种方式无法将可行性扩展到超出整个区间已经为固定 (x, y, m) 提供的范围。 因此，对于每个端点对，要么整个区间已经满足中值约束，要么不存在具有这些端点的更大的有效子序列。 

前缀计数条件直接根据有多少元素位于中值下方和中值处来编码中值位置的定义，因此有效性检查是精确的而不是启发式的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        a = list(map(int, input().split()))

        vals = sorted(set(a))
        idx = {v: i for i, v in enumerate(vals)}
        m = len(vals)

        freq = [0] * m
        for v in a:
            freq[idx[v]] += 1

        pref = [0] * (m + 1)
        for i in range(m):
            pref[i + 1] = pref[i] + freq[i]

        def range_sum(l, r):
            if l > r:
                return 0
            return pref[r + 1] - pref[l]

        ans = 1

        for i in range(m):
            for j in range(i, m):
                x = vals[i]
                y = vals[j]

                s = x + y
                if s % 2:
                    continue
                mid = s // 2
                if mid not in idx:
                    continue

                k = idx[mid]
                if k < i or k > j:
                    continue

                L = range_sum(i, k - 1)
                E = freq[k]
                R = range_sum(k + 1, j)

                total = L + E + R
                if total == 0:
                    continue

                t = (total + 1) // 2

                if L < t <= L + E:
                    ans = max(ans, total)

        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先压缩值，以便可以快速评估值上的每个区间。 嵌套循环枚举所有可能的最小值和最大值选择。 对于每一对，计算并验证中点值，因为中值必须完全等于该中点。 

前缀和允许在常数时间内计算 L 和 R，因此每对检查的复杂度为 O(1)。 条件 L < t ≤ L + E 是中值位置落在排序区间中等于中值的块内这一要求的直接转换。 

最终答案是满足此约束的任何区间的最大大小。 

## 工作示例

 考虑数组 A = [9, 8, 2, 11, 5]。 

我们将值压缩为 [2, 5, 8, 9, 11]。 假设我们选择 x = 2 和 y = 8。然后 m = 5。我们计算 [2, 8] 上的计数：L = 1（值 2），E = 1（值 5），R = 1（值 8）。 因此总计 k = 3，t = 2。由于 L = 1 < 2 ≤ 2 = L + E，因此该区间有效并有助于答案 3。

 ## 边缘情况

 关键的边缘情况是中点值存在但位于所选值区间之外。 例如，如果我们选择 x = 2 且 y = 8，但没有等于 5 的元素，则即使算术中点有效，条件也必定失败。 该算法显式检查压缩图中 m 的存在，防止无效的中值假设。 

另一种边缘情况是当区间包含许多等于中位数的元素时。 对于 A = [5, 5, 5, 1, 9]，选择 x = 1 和 y = 9 给出 m = 5，且 E = 3。对于多个可能的子序列长度满足中值条件，并且检查 L < t ≤ L + E 正确捕获所有这些子序列，因为任何中值位置都落在相等值的块内。 

最后一个微妙的情况是偶数长度的子序列。 由于中位数是使用 ceil(k/2) 定义的，因此与标准“下中位数”定义相比，中位数位置稍微向左移动。 条件 t = ceil(k/2) 确保始终一致地处理中位数，并且不等式 L < t ≤ L + E 可以处理奇数 k 和偶数 k，而无需特殊大小写。
