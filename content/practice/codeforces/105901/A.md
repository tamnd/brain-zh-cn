 ## 解决方案
 ## 问题理解

 我们得到一个数字属性列表，其中每个属性都以初始值开头。 除此之外，还有一些限制。 每个约束都针对特定的属性索引，并要求修改后该索引处的值必须位于给定的包含区间内。 

我们可以通过重复增加或减少 1 来调整任何属性值，并且每个单位的更改花费 1 个时间单位。 任务是修改初始数组以满足每个约束，同时最小化所有属性的总成本。 如果不可能同时满足所有约束，则答案必须报告为不可能。 

从输入格式中观察到的关键是，除了共享相同的属性索引之外，约束是独立的。 这意味着约束不会在不同位置之间相互作用； 它们仅在同一索引内相交。 

约束的数量很少，每个测试用例最多 100 个属性和 100 个约束。 这立即排除了尝试所有可能的最终数组或对值进行全局搜索之类的任何事情。 即使迭代 10^9 范围内的所有可能值也是完全不可行的。 

尝试独立调整每个属性同时重复检查所有约束的简单方法仍然是可以接受的，但是任何模拟逐步单位更改的方法在概念上都太慢，即使经过优化也是如此。 

当对同一索引的约束相互矛盾但又不明显时，就会出现微妙的失败情况。 例如，如果某个属性根据一个约束必须位于 [1, 5] 中，根据另一个约束必须位于 [6, 10] 中，则没有值可以同时满足这两个约束。 仅应用最后看到的约束的简单求解器会错误地假设可行性。 

另一个极端情况是属性根本没有约束。 在这种情况下，它不应该贡献任何成本，因为无论其初始值如何，它都可以保持不变。 

## 方法

 暴力视角会尝试通过在合理范围内尝试所有可能的最终值来调整每个属性值，例如从 1 到 10^9 扫描每个索引并检查它是否满足所有约束。 对于每个候选值，我们将计算其与原始值的距离并选择最小的有效值。 原则上这是可行的，因为我们明确验证了可行性，但复杂性变得令人望而却步。 即使对于单个属性，扫描最多 10^9 个值也是不可能的，而将其乘以最多 100 个属性则使其完全不可行。 

结构简化来自于认识到对固定索引的约束仅限制该索引位于区间的交集内。 我们不需要考虑许多候选值，而只需要影响该索引的所有范围的交集。 一旦我们有了这个交集，该属性的问题就减少为选择线段中最接近固定数字的点，这是直线上的标准投影问题。 

这将全局优化转化为每个索引的独立一维问题。 一旦知道其可行区间，每个属性就会独立地对总成本做出贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·10^9) | O(n·10^9) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(nq) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。

1. 对于每个属性索引，我们首先假设它没有限制。 我们将其可行区间表示为无限宽，这意味着最初允许任何值。 
2. 我们迭代所有约束。 对于每个约束（p，l，r），我们通过将属性p与[l，r]相交来限制属性p的可行区间。 这意味着我们将下限更新为 max(当前下限, l)，将上限更新为 min(当前上限, r)。 此步骤确保我们在不丢失任何约束的情况下累积所有需求。 
3. 处理完所有约束后，我们检查每个属性区间。 如果对于任何索引，下限大于上限，则交集为空，并且没有值可以满足所有约束。 在这种情况下，答案立即是-1。 
4. 如果该区间有效，我们计算将初始值移动到该区间的最小成本。 如果初始值已经在里面，则成本为零。 如果它低于该区间，我们将其上移至下限。 如果高于，我们将其向下移动到上限。 我们在所有属性上累积此成本。 
5. 最后，我们输出总累计成本。 

核心思想是，每个属性都被简化为选择线段中的一个点，使到固定值的距离最小化。 

### 为什么它有效

 对于每个索引，所有约束仅限制区间内的成员资格，多个区间内的成员资格相当于其交集内的成员资格。 任何有效的最终值都必须位于该交集内，并且其中的每个值对于约束都同样有效。 因此，最小化成本简化为闭区间上的最近点问题。 由于直线上的距离是凸的，因此最近的有效点始终是原始值（如果在内部）或区间边界之一。 这保证了每个属性独立的最优性，并且跨属性的独立性确保将这些局部最优值相加得到全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

t = int(input())
for _ in range(t):
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    L = [-INF] * n
    R = [INF] * n

    for _ in range(q):
        p, l, r = map(int, input().split())
        p -= 1
        if l > L[p]:
            L[p] = l
        if r < R[p]:
            R[p] = r

    ok = True
    for i in range(n):
        if L[i] == -INF:
            L[i] = 1
            R[i] = 10**9

        if L[i] > R[i]:
            ok = False
            break

    if not ok:
        print(-1)
        continue

    ans = 0
    for i in range(n):
        if a[i] < L[i]:
            ans += L[i] - a[i]
        elif a[i] > R[i]:
            ans += a[i] - R[i]

    print(ans)
```该实现维护两个数组，表示每个属性的可行范围。 通过收紧这些范围来施加约束。 一个微妙的点是没有限制地处理属性； 通过将它们初始化为较宽的间隔，然后将未触及的域转换为完全允许的域，将它们视为完全免费。 

成本计算是在确认可行性后进行的，因为在检查交叉点有效性之前计算距离可能会浪费工作或隐藏不可能的情况。 

## 工作示例

 输入：```

constraints:

2: (1, 10, 15)

```我们跟踪每个索引的间隔。 

| 索引 | 初始间隔| 约束 1 之后 | 约束 2 之后 | 约束 3 之后 | 最后间隔 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | [-inf，inf] | 不变| [10, 15] | 不变| [10, 15] |
 | 2 | [-inf，inf] | 不变| 不变| 不变| [1, 1e9] |

 现在计算成本：

 对于索引 1，20 高于 [10, 15]，因此成本为 5。 

对于索引 2，25 位于自由范围内，成本为 0。 

对于索引 3, 4 低于 [5, 6]，因此成本为 1。

该轨迹显示约束被压缩为间隔，并且最终的调整是纯粹的几何距离。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    INF = 10**18

    t = int(input())
    out = []
    for _ in range(t):
        n, q = map(int, input().split())
        a = list(map(int, input().split()))

        L = [-INF] * n
        R = [INF] * n

        for _ in range(q):
            p, l, r = map(int, input().split())
            p -= 1
            L[p] = max(L[p], l)
            R[p] = min(R[p], r)

        ok = True
        for i in range(n):
            if L[i] == -INF:
                L[i], R[i] = 1, 10**9
            if L[i] > R[i]:
                ok = False
                break

        if not ok:
            out.append("-1")
            continue

        ans = 0
        for i in range(n):
            if a[i] < L[i]:
                ans += L[i] - a[i]
            elif a[i] > R[i]:
                ans += a[i] - R[i]

        out.append(str(ans))

    return "\n".join(out)

# minimal feasible
assert run("""1
1 0
10
""") == "0"

# contradiction
assert run("""1
1 2
10
1 1 5
1 6 8
""") == "-1"

# already satisfied
assert run("""1
2 2
5 7
1 1 10
2 6 8
""") == "0"

# needs adjustment both sides
assert run("""1
2 2
0 20
1 5 10
2 0 15
""") == "15"
```| 测试输入| 预期产出 | 它验证了什么 |
 | --- | --- | --- |
 | 单一属性，无约束 | 0 | 自由变量无需任何成本 |
 | 相互冲突的约束| -1 | 不可行的交叉点检测|
 | 已经满足的间隔 | 0 | 没有不必要的动作|
 | 混合调整| 15 | 15 正确的距离到间隔逻辑 |

 ## 边缘情况

 当属性根本没有约束时，算法会为其分配一个完整的有效域。 由于初始值始终位于该域内，因此计算的成本保持为零，这与没有约束强制进行任何更改的事实相匹配。 

当约束相互矛盾时，例如同一索引上的间隔不相交，交集在处理过程中会立即崩溃。 该算法纯粹通过区间界限来检测这一点，无需模拟值，并正确输出 -1。 

当初始值恰好位于最终区间的边界上时，两个不等式分支都避免增加成本，因为仅当严格位于外部时才需要移动。 这确保了边界相邻的情况不会在距离计算中引入相差一的误差。
