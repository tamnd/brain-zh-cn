 ## 解决方案
 ## 问题理解

 我们给出一个网格，其中每个单元格都包含一个整数值，并且我们考虑从左上角到右下角的所有单调路径，其中每次移动都是向右或向下。 每条路径都会收集其访问的单元格的值，但通过将它们视为一组来删除重复项。 路径的值是沿该路径出现的不同整数的数量。 任务是对每个有效路径上的该值求和。 

因此，我们不是以直接的方式单独评估路径，而是实际上要求计算每个不同值对所有路径上的“不同计数”有多少次贡献。 

这些限制在结构意义上是严格的，而不是在原始尺寸上。 每个测试用例在所有用例中总共有多达 100,000 个单元格，并且网格尺寸很大但产品稀疏。 这排除了任何枚举路径的解决方案，因为单调路径的数量可以是 n 和 m 的指数。 如果跟踪集合或位掩码，即使是路径上的动态编程也是不可能的，因为值的范围最大为 n × m，最大可达 100,000。 

一种简单的方法是尝试模拟所有路径、收集集合并计算大小。 即使是单个路径也是 O(n + m)，并且存在二项式 (n + m, n) 路径，即使对于像 20 × 20 这样的中等网格，也会立即爆炸。这清楚地表明，解决方案必须完全避免枚举路径，而是在全局范围内计算每个值的贡献。 

当所有值都相同时，会出现微妙的边缘情况。 例如，如果每个单元格均为 7，则每条路径的值为 1，因为集合为 {7}。 答案就是路径的数量。 任何错误地计算每个单元格出现次数而不是每个路径每个不同值的出现次数的方法都会在此处过度计数。 

另一种边缘情况是当值沿某些结构化对角线唯一时，这可能会误导假设行或列之间独立的解决方案。 

## 方法

 暴力的观点是枚举从 (1, 1) 到 (n, m) 的每条单调路径，收集路径上的所有值，对它们进行重复删除，并计算集合大小。 这是正确的，因为它直接遵循定义。 然而，此类路径的数量是组合的，大致为 C(n + m − 2, n − 1)，即使对于中等网格，其数量也会变得天文数字。 对于 30 × 30 的网格，这已经超过了 10^16 条路径，使其不可行。 

关键的转变是颠倒视角。 我们不是迭代路径和计算值，而是迭代值并计算有多少路径至少“看到”该值一次。 当且仅当路径经过至少一个包含该值的单元格时，每个值都会对路径贡献 +1。 因此，问题变成了与网格单元组的路径交叉点的计数问题。 

对于固定值 v，它的所有出现形成网格单元的子集。 我们需要计算有多少单调路径至少经过这些单元格之一。 这相当于总路径减去避免所有出现 v 的路径。 

现在问题变成：给定一组阻塞单元（所有位置都包含 v），计算避开它们的单调路径。 如果我们按坐标按排序顺序处理 v 的单元格并在前缀约束上应用 DP，则可以使用包含在排序结构中来处理此问题。 

关键的观察是，对于每个值 v，我们只需要计算经过至少一次出现的路径数。 由于总值最多为 n × m，因此如果每个单元格处理固定次数，则对所有值求和仍然可行。

我们预先计算从开始到结束的 DP 路径计数，然后使用标准的“格子路径上的禁止点”技巧。 对于每个值 v，我们对其出现进行排序，并使用链上的包含来计算至少经历一次出现的路径：我们考虑从开始到第一次出现、连续出现之间（确保单调顺序）以及从最后一次出现到结束的转换。 

这减少了每个值对组合路径计数的可管理序列的贡献，而不是指数枚举。

 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(纳米) | 太慢了 |
 | 出现次数上的价值型 DP | O(nm log nm) | O(纳米) | 已接受 |

 ## 算法演练

 1. 预先计算高达 n × m 的阶乘和逆阶乘，因为所有路径计数都是二项式系数。 这是必要的，因为两个单元之间的每个单调路径计数仅取决于坐标差。 
2. 预计算函数`ways(x1, y1, x2, y2)`使用组合返回从一个单元格到另一个单元格的单调路径数。 这提供了一种直接计算受约束路径段的方法。 
3. 按值对所有网格单元进行分组。 每组代表特定整数出现的所有位置。 这是计算贡献的单位。 
4. 对于每个值组，先按行排序，然后按列排序。 这确保了任何经历多次出现的有效路径都遵循单调移动，因为路径不能向上或向左移动。 
5. 通过动态链接计算至少出现一次该值的路径数。 首先计算从 (1, 1) 到每次出现的路径。 然后按升序连接出现的事件，对经过选定的值“首次命中”的路径的贡献进行求和。 
6. 最后，将最后一次出现的贡献添加到 (n, m)，完成包含至少一次该值出现的完整路径。 
7. 将此贡献与所有不同值相加，得到最终答案。 

关键思想是，每个路径都被分配给每个值的一个“第一次访问的出现”，这可以防止重复计数。 

### 为什么它有效

 固定值 v 并考虑任何有效路径。 如果路径包含 v，则沿 v 出现的路径存在唯一定义的第一个单元格。 v 的每个贡献都可以计入第一次出现。 这将包含 v 的所有有效路径划分为由 v 的出现索引的不相交集合。可以使用连续点之间的标准单调路径 DP 按排序顺序对每个集合进行计数，确保正确性且不会过度计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

MAX = 100000 + 5

fact = [1] * (MAX)
invfact = [1] * (MAX)

def modpow(a, e):
    r = 1
    while e:
        if e & 1:
            r = r * a % MOD
        a = a * a % MOD
        e >>= 1
    return r

def init():
    for i in range(1, MAX):
        fact[i] = fact[i - 1] * i % MOD
    invfact[MAX - 1] = modpow(fact[MAX - 1], MOD - 2)
    for i in range(MAX - 2, -1, -1):
        invfact[i] = invfact[i + 1] * (i + 1) % MOD

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

def ways(x1, y1, x2, y2):
    dx = x2 - x1
    dy = y2 - y1
    if dx < 0 or dy < 0:
        return 0
    return C(dx + dy, dx)

init()

T = int(input())
for _ in range(T):
    n, m = map(int, input().split())
    vals = {}
    for i in range(n):
        row = list(map(int, input().split()))
        for j, v in enumerate(row):
            vals.setdefault(v, []).append((i, j))

    ans = 0

    for v, pts in vals.items():
        pts.sort()
        k = len(pts)

        dp = [0] * k

        for i in range(k):
            x, y = pts[i]
            dp[i] = ways(0, 0, x, y)
            for j in range(i):
                x2, y2 = pts[j]
                if x2 <= x and y2 <= y:
                    dp[i] = (dp[i] - dp[j] * ways(x2, y2, x, y)) % MOD

        contrib = 0
        for i in range(k):
            x, y = pts[i]
            contrib = (contrib + dp[i] * ways(x, y, n - 1, m - 1)) % MOD

        ans = (ans + contrib) % MOD

    print(ans % MOD)
```该实现首先预先计算阶乘和逆阶乘，以允许恒定时间二项式系数查询。 这`ways`函数将坐标差转换为晶格路径计数。 

对于每个值，我们计算`dp[i]`作为从一开始到达第 i 次出现的路径数，同时避免早期出现相同值。 这是通过按排序顺序对先前处理的点使用包含-排除来完成的。 然后，每次出现都会贡献持续到末尾的路径，从而完成至少一次包含该值的完整有效路径。 

## 工作示例

 ### 示例 1

 输入：```
2 3
5 2 1
1 5 5
```我们列出职位：

 | 价值| 职位 |
 | --- | --- |
 | 1 | (0,2), (1,0) | (0,2), (1,0) |
 | 2 | (0,1)|
 | 5 | (0,0), (1,1), (1,2) | (0,0), (1,1), (1,2) |

 对于值 2，仅存在一个单元格，因此通过该单元格的所有路径都直接计为从开始到 (0,1) 的路径乘以从 (0,1) 到结束的路径。 

对于值 5，存在多次出现。 我们计算 dp：

 | 我| 细胞| dp[i] | dp[i] |
 | --- | --- | --- |
 | 0 | (0,0) | (0,0) | 方式(开始→(0,0)) |
 | 1 | (1,1) | ways(start→(1,1)) 减去经过 (0,0) | 的路径
 | 2 | (1,2) | ways(start→(1,2)) 减去之前的路径 |

 然后每个dp[i]乘以方式结束。 

这演示了减法如何防止重复计算多次出现的路径。 

### 示例 2

 输入：```
2 3

3 3 3
```所有值都是相同的。 每条路径对值 3 的贡献恰好为 1。 

| 价值| 贡献|

 由于所有路径都包含 3，因此 dp 会合并为总路径数。 这证实了统一网格的正确性，否则如果每个单元格独立计数，就会发生过度计数。

 ## 边缘情况

 关键的边缘情况是所有单元格都包含相同的值。 该算法将所有出现的情况视为单个值组。 dp 结构确保每条路径对该值只贡献一次，因为沿每条路径的第一次出现是由单调运动唯一确定的。 

另一个边缘情况是值的稀疏重复。 例如：```
1 0 2
3 0 4
0 0 5
```这里每个值出现一次，因此 dp 简化为端点之间的简单路径计数。 减法循环不执行任何操作，在不存在重叠时确认正确性。 

最后的边缘情况涉及出现在严格增加的对角链中的值。 该排序保证只有前向可到达的点进行交互，因此无效的转换永远不会错误地提供负校正，从而保持 DAG 结构上包含-排除的正确性。
