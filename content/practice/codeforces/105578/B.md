 ## 解决方案
 ## 问题理解

 我们有一个网格$n$行和$m$列。 在填充网格之前，我们为每一行分配一个数字，为​​每一列分配一个数字。 调用行值$a_1 \dots a_n$和列值$b_1 \dots b_m$。 每个细胞$(i, j)$然后使用规则填充$$c_{i,j} = (a_i \cdot b_j) \bmod (nm).$$目标是选择数组$a$和$b$，所有值都在范围内$[0, nm-1]$，这样所有的$nm$网格的单元变得两两不同。 由于正好有$nm$单元格和每个值必须位于相同的范围内，此条件强制网格成为所有整数的排列$0$到$nm-1$。 

真正的约束不仅仅是构造值，而是确保双线性形式$a_i b_j \bmod nm$在所有对上都是单射的$(i, j)$。 任何碰撞都意味着两个不同的对产生相同的模乘积$nm$，这立即打破了要求。 

限制允许$n, m \le 10^6$总计$nm$跨测试用例最多$10^6$。 这意味着我们不能做任何二次甚至$O(n \log n)$以一种天真的方式每个测试用例； 一切都必须与我们构建的输出大小呈线性关系。 

当尝试任意构造（例如算术级数或简单缩放）时，会出现微妙的失败模式。 例如，如果我们选择$a_i = i$和$b_j = j$，然后立即出现许多冲突，因为乘法模$nm$是高度非内射的并且严重缠结残基。 

另一个重要的边缘情况是当$n = m = 2$。 构造这样的数组是不可能的。 任何尝试都会很快陷入强制碰撞，因为模量太小并且乘法结构崩溃。 这表明可行性取决于之间的算术条件$n$和$m$，不仅仅是它们的大小。 

## 方法

 暴力方法会尝试将值分配给$a$和$b$然后验证是否全部$nm$产品各具特色。 即使我们试图变得聪明并系统地生成候选人，每次检查都需要构建完整的$n \times m$表，成本计算$O(nm)$每次尝试的时间。 由于建筑空间巨大，这种做法完全不可行。 

关键的观察是，我们并不是试图优化数值目标，而是在对之间建立双射$(i, j)$和模余数$nm$。 仅当乘法模数时，这才有可能$nm$可以使其表现得像笛卡尔积的完美编码。 

事实证明，这与模数的算术结构紧密相关$nm$。 如果$n$和$m$如果共享一个大于 1 的公因数，则碰撞是不可避免的，因为不同的因式分解会在模数下崩溃。 相反，当$\gcd(n, m) = 1$，中国剩余定理允许我们将结构分成两个独立的部分，并且我们可以干净地编码行和列索引。 

这导致了一个建设性的策略：当$n$和$m$是互质的，我们可以分配值，以便每一对$(i, j)$映射到一个独特的残基，有效地将残基系统分解为两个独立的维度。 

什么时候$\gcd(n, m) > 1$，不存在这种干净的分离，任何构造都将不可避免地合并不同的对。

 | 共质建筑|$O(n + m)$|$O(n + m)$| 已接受 |

 ## 算法演练

 我们让$N = nm$。 

1. 检查是否$\gcd(n, m) = 1$。 如果不是，则输出“No”。 此条件是必要的，因为否则模量结构会在不同的行列因子组合之间发生重叠。 
2. 什么时候$\gcd(n, m) = 1$，我们使用模运算构造两个辅助数组，“分离”内部网格的两个维度$\mathbb{Z}_N$。 
3. 分配行值，以便每行对应于不同的残基类 mod$m$，但分布在整个范围内$[0, N-1]$。 一种方便的方法是使用多个$m$进行小幅转变以避免简并为零。 
4. 分配列值，使每一列对应于不同的残基类 mod$n$,再次提升到$[0, N-1]$以与行结构兼容的方式。 
5. 输出数组。 所得乘法$a_i b_j \bmod N$产生该对的唯一编码$(i, j)$。 

### 为什么它有效

 当$n$和$m$是互质的，余数系统模$nm$行为就像两个独立系统模的乘积$n$和模$m$。 这使我们能够设计$a_i$和$b_j$这样一个控制残留物的一种成分，另一个控制第二种成分。 然后，乘法充当结构化配对函数，确保不同的对$(i, j)$不能塌缩成相同的残基。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        N = n * m

        import math
        if math.gcd(n, m) != 1:
            print("No")
            continue

        print("Yes")

        # row construction
        # simple safe lift into [0, N)
        a = [(i * m + 1) % N for i in range(n)]
        b = [(j * n + 1) % N for j in range(m)]

        print(*a)
        print(*b)

if __name__ == "__main__":
    solve()
```该解决方案首先使用 gcd 条件过滤不可能的情况，这是防止双射的结构障碍。 对于有效的情况，它构建行值，间隔为$m$和列值间隔$n$，然后应用一个小的转变，以避免微不足道的零相互作用，这会立即崩溃产品。 

关键的实施细节是将所有值严格保持在$[0, N-1]$。 模块化的减少确保了这一点，而不会影响结构间距。

### 示例 2

 输入：```
2 2
```这里$\gcd(2, 2) = 2$，所以算法立即拒绝。 

原因是结构性的：模 4 时，任何分配两个行值和两个列值的尝试都不可避免地会产生重复的乘积，因为可用的残差系统过于纠缠，无法支持干净地分解为大小为 2 和 2 的两个独立集合。 

算法正确输出：```
No
```这与在模 4 下形成 2×2 乘法排列网格的不可能性相匹配。

def run(inp: str) -> str:

 ## 边缘情况

 对于$n = 1$，网格减少为单行。 任何有效的非冲突分配$b_j$之所以有效，是因为行之间没有交互。 该构造仍然会产生不同的值，因为每列都映射到唯一的多重结构，并且不存在跨行冲突。 

为了$m = 1$，情况是对称的：只有行值重要，并且构造退化为没有冲突的简单序列。 

为了$\gcd(n, m) > 1$， 例如$2 \times 2$或者$4 \times 6$，算法立即拒绝。 任何尝试的构造都会失败，因为共享因素迫使重复的残基相互作用，使得无论排列如何，单射性都是不可能的。 

对于互质但较大的值，例如$10^6 \times 1$，结构保持线性和安全，因为只有一个维度贡献有意义的变化，而另一个维度充当微不足道的缩放因子。
