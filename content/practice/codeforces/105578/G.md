 ## 解决方案
 ## 问题理解

 给定一个简单的多边形，其顶点都是 1000 x 1000 网格内的整数点，但顶点以完全打乱的顺序呈现，因此我们无法直接恢复边或邻接。 我们可以在 [0, 1000] 范围内查询 p/q 形式的任何有理 x 坐标处的垂直线，而不是与顶点交互。 对于这样的一条线，交互器返回位于该垂直线上的多边形所有线段的总长度，这意味着多边形与线 x = p/q 之间的交点长度之和。 

最终任务不是重建多边形本身，而只是将其面积精确地计算为约简分数。 

约束足够严格，我们无法承担任何繁重的几何重建。 最多有 1000 个测试用例，所有用例的顶点总数以 1000 为界。每个测试最多允许 n − 2 个查询，因此任何策略都必须使用很少的精心选择的测量来提取区域。 

一个微妙的困难是我们查询的函数不是简单的线性函数。 每当垂直线经过顶点或交点的组合结构发生变化时，给定 x 处的多边形的横截面长度就会发生变化。 函数是全局线性的天真假设在凹多边形上会失败，在凹多边形上会出现多个不相交的间隔并合并或分裂。 

第二个微妙之处是我们不知道多边形顺序，因此不能直接使用标准多边形面积公式（例如鞋带公式）。 任何方法都必须完全依赖于形状作为集合的几何属性。 

## 方法

 暴力的想法会首先尝试重建多边形结构。 人们可以想象查询许多 x 值，尝试推断存在多少个相交线段以及它们如何连接，然后重建边。 这从根本上来说是困难的，因为多边形没有以任何一致的顺序给出，并且横截面信息失去了顶点之间的连接性。 即使我们查询每个整数 x 坐标，我们也只能知道总垂直切片长度，而不知道这些切片如何在相邻 x 位置之间对应。 这使得显式重建变得不稳定并且比所需的复杂得多。 

关键的观察是我们实际上并不需要多边形结构。 形状的面积可以从其垂直截面函数 f(x) 恢复，其中 f(x) 是与 x 处垂直线相交的总长度。 面积正是该函数对 x 的积分。 这将整个问题简化为我们可以查询的函数的数值积分。 

关键的结构特性是，对于具有整数坐标的多边形，函数 f(x) 仅在与多边形顶点对应的 x 坐标处改变其斜率。 在两个连续整数 x 值之间，没有顶点位于间隔内，因此相交的组合结构不会以改变该线段内线性的方式发生变化。 因此，f(x) 表现为分段线性函数，其断点位于整数 x 值上。 

这意味着每个单位区间 [i, i + 1] 上的积分可以使用端点 f(i) 和 f(i + 1) 的梯形规则精确计算。 对所有整数间隔求和即可得出精确的面积。 

因此，问题简化为在整数位置查询 f(x) 并对梯形求和。

 ## 算法演练

我们将多边形视为位于沿 x 轴的已知边界区间 [0, 1000] 内。 我们评估整数 x 坐标处的横截面函数并将其积分。 

1. 对于每个测试用例，初始化一个变量，该变量将面积累加为有理数或浮点表示，稍后可以对其进行标准化。 该累加器表示单位 x 间隔上的梯形之和。 
2. 在从 0 到 1000 的每个整数 x 处查询交互器的 f(x)。每个查询给出该 x 坐标处多边形的总垂直长度。 这直接对应于横截面的高度。 
3. 对于每个区间 [x, x + 1]，计算对面积的贡献为 (f(x) + f(x + 1)) / 2。这是在单位区间上对线性函数进行积分的标准梯形规则。 
4. 将 x = 0 到 999 范围内的所有区间贡献相加。这将生成精确的多边形面积。 
5. 将所得面积作为约简分数输出。 由于所有输入都是有理数且坐标是整数，因此可以通过 gcd 完成归一化。 

工作原理：函数 f(x) 对每个 x 处多边形的总垂直测量进行编码。 由于多边形边是整数顶点之间的直线，因此 f(x) 的变化只能发生在整数 x 坐标处。 在每个单位间隔内，每条边在 x 上呈线性行为，并且所有相交线段的并集仅通过线性插值变化。 这保证了 f(x) 在每个整数区间上仿射，因此梯形规则是精确的而不是近似的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(x):
    print(f"? {x} 1")
    sys.stdout.flush()
    r, s = map(int, input().split())
    return r, s

def add_frac(a_num, a_den, b_num, b_den):
    return a_num * b_den + b_num * a_den, a_den * b_den

def reduce_frac(n, d):
    from math import gcd
    g = gcd(n, d)
    return n // g, d // g

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())

        # We ignore vertices since they are shuffled and unused in this strategy.
        for _ in range(n):
            input()

        # Store f(x) as fractions
        fx = []

        for x in range(1001):
            r, s = ask(x)
            fx.append((r, s))

        num = 0
        den = 1

        for i in range(1000):
            r1, s1 = fx[i]
            r2, s2 = fx[i + 1]

            # (r1/s1 + r2/s2) / 2
            cur_num = r1 * s2 + r2 * s1
            cur_den = 2 * s1 * s2

            num = num * cur_den + cur_num * den
            den = den * cur_den

            from math import gcd
            g = gcd(num, den)
            num //= g
            den //= g

        num, den = reduce_frac(num, den)
        print(f"! {num} {den}")
        sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该实现完全侧重于查询整数 x 坐标并累积梯形贡献。 每个查询都会立即刷新，因为问题是交互式的。 分数被显式维护，因为每个查询返回一个有理数，并且中间结果必须保持准确性。 

一个常见的陷阱是假设浮点运算就足够了。 这是不安全的，因为中间值可以累积大分母，因此一切都通过 gcd 约简保持有理形式。 

## 工作示例

 考虑一个小多边形，其中已知整数 x 处的横截面长度可形成简单的线性图案。 

### 示例 1

 假设查询返回：

 | x| f(x) | f(x) |
 | ---| ---|
 | 0 | 0 |
 | 1 | 2 |
 | 2 | 4 |

 我们计算梯形：

 | 间隔 | 计算| 贡献|
 | ---| ---| ---|
 | [0,1]| (0 + 2)/2 | (0 + 2)/2 | 1 |
 | [1,2]| (2 + 4)/2 | 3 |

 总面积为4。 

这表明横截面的线性增长完全整合到几何面积中。 

### 示例 2

 假设：

 | x| f(x) | f(x) |
 | ---| ---|
 | 0 | 3 |
 | 1 | 3 |
 | 2 | 3 |

 每个间隔贡献 (3 + 3)/2 = 3，因此两个单元的总面积为 6。 

这证实了该方法可以正确处理与矩形相对应的恒定横截面。

 ## 边缘情况

 关键的边缘情况是多边形退化为多个顶点共享相同 x 坐标的形状。 在这种情况下，横截面函数可以在多个区间内保持恒定，并且梯形规则仍然适用而无需修改，因为重复的相同值会产生斜率变化的精确抵消。 

当多边形跨越整个范围 [0, 1000] 但具有稀疏结构时，会出现另一种边缘情况，从而导致 f(x) 发生急剧变化。 由于这些变化仅发生在整数边界处，因此对所有整数位置进行采样仍然可以准确捕获每个断点，并且不会丢失任何中间行为。 

最后一种情况是多边形在给定 x 处产生多个不相交的垂直间隔。 函数 f(x) 已将这些间隔求和为单个长度值，因此积分过程仍然有效，无需区分各个分量。
