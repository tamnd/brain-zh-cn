 ## 解决方案
 ## 问题理解

 我们得到了一个完整的图表$n$城市，但仅$n-2$它的边缘被标记为风景区。 这些风景边形成了一种几乎连接的结构，从某种意义上说，如果允许我们再添加一条边，风景图将变得完全连接。 

这种结构条件迫使景观边缘形成两个相连的组件，每个组件都是一棵树。 如果有三个或更多组件，添加一条边无法连接所有组件。 如果它已经是一个组件，则至少需要$n-1$边缘。 因此，风景网络恰好是两棵树，其顶点集划分了所有城市。 

在这个底层的风景结构之上，我们考虑一个任意的“指南地图”，它是选定的道路子集（完整图的边缘）。 小Q从城市1开始执行确定性遍历。从他当前的城市开始，他总是尝试去引导地图中出现的编号最小的未访问邻居。 如果不存在，他会沿着先前使用的引导地图边缘回溯，直到找到有效的移动或返回城市 1 并停止。 

这种行为正是在引导图上执行的深度优先搜索，其中邻居按增加标签排序。 

要求是，在这次穿越期间，他必须设法访问所有风景区边缘，并且在这样做时，他最多允许穿越一条不同的非风景区道路。 由于风景区边缘是我们有义务“覆盖”的唯一边缘，因此引导地图必须能够到达两个风景区组成部分，但不得依赖它们之外的多个额外连接。 

任务是计算有多少不同的引导图满足此约束。 

约束条件达到$2 \cdot 10^5$，因此任何解都必须是线性或近线性的$n$。 任何涉及枚举线性结构之外的图或边的事情都是立即不可能的。 

解释遍历规则会产生一个微妙的问题。 天真的阅读可能会建议我们需要在许多候选图上模拟 DFS，但风景图的结构迫使底层组合条件变得简单得多。 

一个潜在的困惑是在遍历过程中是否可以使用多个非场景边缘。 然而，由于遍历遵循单个图，因此引导地图中包含的任何非风景边缘都是可用的。 “最多一条不同的路没有风景”的限制意味着在整个旅程中，只有一条这样的边可以成为遍历路径的一部分。 这强烈地限制了导游地图的行为，就像一棵建在风景森林顶部的树一样。 

## 方法

 如果我们尝试直接从遍历过程中进行推理，我们将需要考虑所有可能的图$n$节点并模拟尊重字典顺序的 DFS，同时还跟踪曾经使用过多少非场景边。 这是没有希望的：即使限制在树木上也已经给出了$n^{n-2}$可能性，并添加额外的边缘使空间变得更大。 

关键的简化来自于对场景结构的理解。 比如说，风景的边缘已经形成了两棵不相交的树$A$和$B$。 由于小 Q 从城市 1 开始，如果 1 位于一个组件中，则没有通过风景边缘到达另一组件的路径。 

因此，任何有效的引导地图必须至少包含一个连接这两个组件的非风景边缘，否则 DFS 根本无法到达第二个组件，更不用说访问其风景边缘了。 

现在考虑对非风景旅游的限制。 如果引导地图包含多个非风景边，则 DFS 必须遍历至少两个不同的此类边，以便连接或探索图的某些部分，因为非风景边所起的唯一作用是在两棵风景树之间架起桥梁。 这违反了约束，因此引导图只能在两个组件之间引入恰好一条连接边。 

一旦这一点被接受，任何有效的导游地图的结构就会变得极其严格。 它由所有场景边缘加上一条附加边缘组成，该边缘的端点位于不同的场景组件中。 任何这样的边都会生成一个连通图，并且由于原始风景边已经覆盖了所有顶点，因此该图是一棵树。 

DFS规则并没有进一步限制选择哪条连接边，因为一旦图连接起来，遍历就完全确定但始终有效。 每一个这样的选择都会产生不同的引导图，并且不同的边缘显然会产生不同的图形。 

因此，问题简化为计算我们可以选择第一个场景组件中的一个端点和第二个场景组件中的一个端点的有多少种方式。 如果元件尺寸为$s$和$n-s$，答案是$s \cdot (n-s)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解引导图 + DFS 模拟 | 指数| O(n) | 太慢了|
 | 成分分解+交叉边计数| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 从场景边缘构建一个图并找到其连接的组件。 由于正好有$n-2$边并且结构保证在添加一条边后变得连通，我们将恰好找到两个分量。 
2. 使用 DFS 或 BFS 计算每个组件的大小。 设这些尺寸为$s$和$n-s$。 遍历很简单，因为风景图已经是一片森林了。 
3. 认识到任何有效的引导图必须用一条附加边将这两个组件连接起来，并且可以在第一个组件中的任何顶点和第二个组件中的任何顶点之间任意选择该边。 
4. 将两个分量的大小相乘以获得有效引导图的数量。 

### 为什么它有效

 风景边将图恰好划分为两棵树。 任何有效的遍历都必须能够在它们之间移动，这只能通过引导地图中包含的非风景边缘进行。 由于该约束最多允许使用一个这样的边，因此该边必须是两个组件之间的唯一连接器。 一旦该边被固定，引导图就是一棵包含两个组件的树，并且跨分区的端点的每个选择都会产生一个独特的有效结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    g = [[] for _ in range(n + 1)]

    for _ in range(n - 2):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    visited = [False] * (n + 1)
    comp_sizes = []

    def dfs(start):
        stack = [start]
        visited[start] = True
        size = 0

        while stack:
            u = stack.pop()
            size += 1
            for v in g[u]:
                if not visited[v]:
                    visited[v] = True
                    stack.append(v)

        return size

    for i in range(1, n + 1):
        if not visited[i]:
            comp_sizes.append(dfs(i))

    if len(comp_sizes) != 2:
        print(0)
        return

    ans = comp_sizes[0] * comp_sizes[1]
    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先重建风景森林并计算其连接的组件。 由于输入保证结构在一条附加边之后变为连接状态，因此我们期望恰好有两个组件，但代码仍然可以防止不一致。 

每次 DFS 运行都会测量一个场景组件的大小。 最终答案将两个尺寸相乘，对应于选取单个所需连接边的端点的方法数量。 

该实现使用迭代 DFS 来避免递归深度问题，因为$n$可以很大。 

## 工作示例

 考虑一个场景：$n=4$风景边缘是$(1,4)$和$(2,3)$。 风景图分为两个部分：$\{1,4\}$和$\{2,3\}$。 

| 步骤| 堆栈/节点| 访问过的组件 | 迄今为止的元件尺寸|
 | ---| ---| ---| ---|
 | 从 1 开始 DFS | [1] | {1,4} | 2 |
 | 从 2 | 开始 DFS [2] | {2,3} | 2 |

 该算法检测大小为 2 和 2 的两个分量，产生$2 \cdot 2 = 4$。 

这证实了两个组件之间的每次配对都会产生有效的引导图。 

现在考虑一个较小的情况$n=3$，那里只有一个风景边缘，比如说$(1,2)$。 组成部分是$\{1,2\}$和$\{3\}$。 答案就变成了$2 \cdot 1 = 2$，对应于选择$(1,3)$或者$(2,3)$作为单个连接边。

 ## 边缘情况

 最小情况$n=2$这是唯一不存在风景边缘的情况。 该算法仅在仔细解释后才将其视为两个单顶点分量：由于存在零边，因此两个顶点都是孤立分量，给出大小 1 和 1 并产生答案 1。DFS 正确找到两个分量并将它们相乘。 

在像链条这样的情况下$1-2-3$如果只有一条边是风景优美的，则图表会分成多个部分$\{1,2\}$和$\{3\}$。 从 1 开始的 DFS 访问两个节点，第二个 DFS 访问最后一个节点，产生大小 2 和 1。乘法正确地计算了两个可能的连接边，$(1,3)$和$(2,3)$，它符合任何跨组件连接都会产生有效引导图的要求。
