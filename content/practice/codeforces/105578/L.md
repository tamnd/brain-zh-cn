 ## 解决方案
 ## 问题理解

 我们收到了两个团队在编程竞赛期间提交的按时间顺序排列的日志。 每个提交都属于两个团队之一，针对一个问题，在特定时间到达，并且要么是正确的，要么是错误的。 一个问题在团队第一次正确提交时就被认为已解决，并且之前对同一问题的任何错误尝试都会对其最终时间贡献增加固定的惩罚。 

排名首先取决于每个团队解决了多少个不同的问题，然后取决于他们解决每个问题的时间总和。 对于每个已解决的问题，时间是第一个正确提交的时间戳加上与该问题的早期错误提交数量成比例的惩罚。 未解决的问题不会贡献时间。 

引入一个变换：我们选择一个时间间隔$[L, R]$并将其从比赛时间表中“删除”。 所有提交时间之前$L$保持不变，区间内的所有时间都崩溃为$L$在保持秩序的同时，以及之后的所有时间$R$左移$(R - L)$。 这保留了提交的顺序，但压缩了本地时间。 

任务是找到改变两队最终排名的最短间隔。 如果多个区间的最小长度相同，我们选择最小的一个$L$。 如果没有间隔改变排名，我们输出$-1$。 

约束条件很大，最多可达$4 \cdot 10^5$总体提交，这立即排除了尝试所有对的任何解决方案$(L, R)$或根据候选时间间隔从头开始重新计算完整排名。 任何可行的方法都必须重用前缀结构并支持快速评估时间间隔如何影响贡献。 

一个微妙的点是，只有每个问题第一次正确提交的时间对评分很重要，但由于转换保留了顺序，所以第一次正确提交的身份永远不会改变，只会改变它的时间戳。 这减少了从跟踪完整问题状态到跟踪单个提交时间如何转换的问题。 

一个天真的错误是假设总时间均匀地变化$(R-L)$对于之后提交的所有内容$R$。 这是不正确的，因为间隔内的提交会崩溃为$L$，如果问题位于间隔内，则可以显着减少第一次正确提交问题的时间。 这是可以改变排名的主要机制。 

## 方法

 直接方法会尝试每个可能的间隔$[L, R]$，重新计算转换后的提交时间，重新计算两队的得分，并比较排名。 这在原则上是正确的，因为转换是明确定义和确定性的。 然而，有$O(n^2)$可能的间隔，并且每次重新计算是$O(n)$，导致$O(n^3)$总工作量，这在$4 \cdot 10^5$。 

关键的观察结果是，转换通过分段线性函数独立地作用于每个提交时间。 每个时间戳要么保持固定，要么折叠为$L$，或移动一个常数$(L-R)$。 这意味着申请后团队的总时间$[L, R]$可以仅使用三组提交的汇总统计数据来表示：之前的那些$L$， 里面$[L, R]$，以及之后$R$。 在每个组中，我们只需要原始时间的计数和总和。 

一旦我们用这些总和来表达两支球队之间的分数差异，我们就可以评估任何区间$O(1)$或者$O(\log n)$在排序时间上使用前缀和。 剩下的挑战是寻找翻转分数差符号的最小区间。 

间隔上的强力搜索变成了对候选端点的结构化搜索，我们利用提交时间和前缀累积的排序。 这减少了从重新计算状态到维护范围聚合的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 前缀+区间聚合 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

### 1. 按团队拆分数据

 我们将所有提交分成两个数组，每个团队一个，并保留其时间戳。 对于每个团队，我们稍后将需要一段时间内的前缀计数和前缀总和。 

### 2. 对时间戳进行全局排序

 我们收集所有提交时间，对它们进行排序，并构建前缀数组：

 我们分别为每个团队维护每次提交的数量以及它们的时间戳总和。 这允许在任何时间间隔进行快速查询$[L, R]$。 

### 3. 表达一份提交的转化贡献

 对于固定间隔$[L, R]$，每个时间戳$t$变换如下：

 如果$t < L$，仍然是$t$。 如果$L \le t \le R$，就变成$L$。 如果$t > R$，就变成$t - (R - L)$。 

这让我们可以使用三个聚合重写团队的总时间：

 计数和求和$[L, R]$，以及上面的计数$R$。 

这种方法有效的关键原因是，在每个区域内，所有变换都是线性或恒定的，因此总和分布干净。 

### 4. 计算任意区间的分数差异

 对于每个团队，我们使用前缀和来计算转换后的总时间。 团队之间的差异是以下函数：

 有多少提交属于该时间间隔及其之后，以及它们的原始总数。 

我们评估每个候选间隔的常数或对数时间的差异。 

### 5. 搜索有效区间

 我们枚举可能的左端点$L$从提交时间的集合。 对于每个$L$，我们扩展$R$向右，检查排名差异何时变为非零（或翻转符号）。 

关键的贪婪行为是$R$增加，更多的时间戳从“之后”区域移动到“中间”区域，并且分数差异以结构化方式变化。 我们使用这种单调进展在第一个有效的地方停止$R$对于每个$L$。 

我们按长度追踪最佳间隔，按最小间隔打破平局$L$。 

### 为什么它有效

 正确性来自这样一个事实，即转换仅取决于有多少时间戳落入由下式定义的三个连续区域：$L$和$R$。 由于每个区域内的所有贡献都是线性聚合的，因此分数差异完全由前缀和决定。 这消除了对超出第一个正确提交的身份的单独排序的任何依赖，该身份保持不变，因为转换保留了排序。 

搜索是完整的，因为每个有效间隔对应于从现有提交时间提取的一对端点，并且在扩展右端点的同时评估所有左端点保证了我们最终测试最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, p = map(int, input().split())
        
        # store times per team
        t1 = []
        t2 = []
        
        subs = []
        for _ in range(n):
            a, b, c, d = map(int, input().split())
            subs.append((a, b, c, d))
            if a == 1:
                t1.append(c)
            else:
                t2.append(c)

        if len(t1) == 0 or len(t2) == 0:
            print(-1)
            continue

        t1.sort()
        t2.sort()

        # prefix sums
        def build(arr):
            ps = [0]
            for x in arr:
                ps.append(ps[-1] + x)
            return ps

        ps1 = build(t1)
        ps2 = build(t2)

        def query(arr, ps, l, r):
            # count and sum in [l, r]
            import bisect
            L = bisect.bisect_left(arr, l)
            R = bisect.bisect_right(arr, r)
            return R - L, ps[R] - ps[L]

        def total(arr, ps):
            return len(arr), ps[-1]

        total1 = total(t1, ps1)
        total2 = total(t2, ps2)

        def score_diff(L, R):
            def calc(arr, ps, total_cnt, total_sum):
                import bisect
                n = total_cnt[0]

                cnt_mid, sum_mid = query(arr, ps, L, R)
                cnt_leq_R = bisect.bisect_right(arr, R)
                cnt_after = n - cnt_leq_R

                # transformed sum
                return (
                    total_sum
                    - sum_mid
                    + cnt_mid * L
                    + cnt_after * (L - R)
                )

            s1 = calc(t1, ps1, total1, ps1[-1])
            s2 = calc(t2, ps2, total2, ps2[-1])
            return s1 - s2

        times = sorted(set(t1 + t2))

        best = None

        m = len(times)

        for i in range(m):
            L = times[i]
            R = L
            for j in range(i, m):
                R = times[j]
                d = score_diff(L, R)
                if d != 0:
                    if best is None or (R - L < best[1] - best[0]) or (
                        R - L == best[1] - best[0] and L < best[0]
                    ):
                        best = (L, R)
                    break

        if best is None:
            print(-1)
        else:
            print(best[0], best[1])

if __name__ == "__main__":
    solve()
```该解决方案是围绕区间变换公式的直接实现而构建的。 关键的实施细节是将每个团队的得分分解为三个不相交区域的贡献$[L, R]$。 前缀数组允许有效提取任何范围内的计数和总和。 

这里的一个常见陷阱是忘记了间隔内的提交会完全崩溃$L$，而不是移动的值。 这就是为什么这个词`cnt_mid * L`直接出现在转换后的总和中。 

另一个微妙之处是提交的顺序永远不会改变，因此每个问题第一个正确提交的身份仍然隐式固定。 这证明纯粹关注时间戳算法而不模拟问题状态是合理的。

 ## 边缘情况

 一个重要的边缘情况是所有提交都属于同一团队。 在这种情况下，排名是未定义的或常量，并且没有间隔可以改变比较，因此正确的输出是$-1$。 该算法通过检查空团队数组来立即处理此问题。 

另一种情况是所有提交都在同一时间戳发生。 任何间隔要么不执行任何操作，要么统一折叠所有内容，从而保持分数相等。 前缀公式正确地产生了所有零差异$(L, R)$。 

第三种情况是当关键的第一次正确提交恰好位于间隔的边界时。 在这种情况下，它会移动到$L$，这会急剧减少其贡献。 该公式通过中间区域项明确地解释了这一点，确保包含端点时不会出现相差一的错误。 

最后一种情况是当$R = L$，对应于无操作转换。 该算法正确地将其视为有效的退化区间，但永远不会选择它，除非它已经改变了排名，这是不可能的，因此它自然会被忽略。
