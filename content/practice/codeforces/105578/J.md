 ## 解决方案
 ## 问题理解

 我们有一个固定的八队单败淘汰赛分组，已经按照四分之一决赛从上到下的顺序排列。 每个团队都有唯一的名称和唯一的整数强度。 比赛以标准的淘汰赛方式进行：括号中的相邻配对进行比赛，较强的球队总是晋级，较弱的球队被淘汰。 这种情况一直持续到四分之一决赛、半决赛，然后是决赛，最终产生了一名冠军和一名亚军。 

任务不是模拟任意锦标赛结构，而是遵循这个精确的固定括号布局。 这意味着第一轮比赛由1对2、3对4、5对6和7对8球队之间的比赛组成。这些比赛的获胜者将再次按顺序配对进入半决赛，然后决赛将在两个半决赛获胜者之间进行。 

尽管故事背景是关于电子竞技的，但计算问题纯粹是对固定比较二叉树的确定性简化。 

输入大小恒定为八支球队，因此不需要渐近优化。 在所有情况下，即使是直接模拟甚至重新计算也是微不足道的。 主要要求是括号处理的正确性，因为配对结构很容易被误读或错误地实现。 

由于限制，边缘情况很少，但在简单的实现中仍然存在微妙的故障模式：

 一个常见的错误是按实力对所有球队进行排序并取前两名。 这是不正确的，因为锦标赛结构很重要。 例如，如果最强的两支球队在四分之一决赛中相遇，其中一支球队会提前被淘汰，无法获得亚军，即使它比其他球队都强。 

另一个错误是跨轮配对不正确，例如根据原始指数而不是获胜者在括号中的位置重新配对。 这打破了淘汰赛结构。 

## 方法

 暴力解释将明确模拟整个锦标赛。 我们维护当前一轮参与者的列表，然后重复将相邻条目配对，比较优势，并构建下一轮，直到剩下一个团队。 每场比赛都是持续不断的工作，八支球队总共最多进行七次比较。 这已经是微不足道的事情了，但结构概括得很清楚。 

关键的观察结果是括号是具有固定顺序的完美二叉树。 每场比赛将两支球队合并为一支，获胜者将向上传播，无需重新排序。 因为只有三轮，所以我们可以逐级模拟，没有任何开销。 

在这里，暴力方法和最优方法实际上是相同的，因为结构是固定的并且很小。 唯一的区别在于我们是明确地对回合进行建模还是尝试以更抽象的方式得出答案。

 ## 算法演练

 我们直接使用（强度，名称）对列表来模拟淘汰赛。

1. 按顺序读取八支球队，并将每支球队存储为一对，其中包含实力和名称。 我们首先保持实力，因为比较是数字化且频繁的。 
2. 按括号顺序初始化本轮名单为这八支球队。 排序至关重要，因为匹配总是在相邻对之间进行。 
3. 对于每一轮，为获胜者创建一个空列表。 
4. 以两步迭代当前列表。 对于每对相邻的球队，比较实力并将较强的球队添加到下一轮名单中。 这直接编码了实力强者获胜的规则。 
5. 处理完所有配对后，将当前列表替换为获胜者列表。 
6. 重复，直到只剩下一组。 那支球队是冠军。 
7. 亚军是在决赛中输掉比赛的球队。 我们可以通过在比较之前存储两个决赛选手来跟踪最后一轮的情况。 

一种稍微更明确的方法是在半决赛后停止，我们将四支球队减少为两支决赛队伍，然后进行最后一次比较并记录胜者和败者。 

### 为什么它有效

 每一轮都精确地模拟完整二元锦标赛树的一个级别。 每支球队每轮只参加一场比赛，直至淘汰。 由于配对是固定且确定的，因此每场比赛的获胜者都是通过比较实力来唯一确定的。 该结构保证了在锦标赛约束下最终存活的节点是全球冠军，而最后被淘汰的节点因为是最终比赛的失败者而成为亚军。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

teams = []

for _ in range(8):
    name, t = input().split()
    teams.append((int(t), name))

while len(teams) > 2:
    nxt = []
    for i in range(0, len(teams), 2):
        if teams[i][0] > teams[i + 1][0]:
            nxt.append(teams[i])
        else:
            nxt.append(teams[i + 1])
    teams = nxt

# final match
a, b = teams[0], teams[1]
if a[0] > b[0]:
    print(f"{a[1]} beats {b[1]}")
else:
    print(f"{b[1]} beats {a[1]}")
```该代码读取八支球队并将它们存储为（实力，名称）对，以便比较是简单的整数比较。 该循环通过配对相邻元素并将获胜者向前推进来处理回合，这与固定括号结构完全匹配。 

最终剩下的两支队伍代表决赛入围。 最后的比较直接决定冠军和亚军，因为决赛的失败者是唯一确定的。 

一个常见的实施陷阱是在模拟比赛之前意外地对球队进行排序。 这会破坏支架结构。 另一个微妙的问题是在构建下一轮时混合索引； 步长为 2 是强制性的，因为每场比赛恰好消耗两个参与者。

 ## 边缘情况

 一种边缘情况是最强的团队被放置在一个分支中，而该分支很早就遇到了第二强的团队。 例如：

 输入：```
A 100
B 99
C 1
D 2
E 98
F 3
G 97
H 4
```四分之一决赛根据配对产生B或A，但如果A和B在半决赛或更早的时间相遇，即使A和B在全球范围内都很强大，也会被淘汰。 该算法可以正确处理此问题，因为它从不比较全局排名，仅比较相邻的括号对。 

另一个边缘情况是亚军不是全球第二强的球队。 在决赛中，失败者只是最后一次比较失败的决赛选手。 该代码显式存储两个决赛入围者并对它们进行一次比较，确保无论全球排名如何，都能正确捕获亚军。 

最后，所有值都是唯一的，保证比较中不会出现歧义，因此不需要平局逻辑。
