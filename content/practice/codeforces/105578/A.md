 ## 解决方案
 ## 问题理解

 我们被要求计算使用 n 个段可以形成多少个不同的“稳定梯子”，其中每个段具有正整数长度，并且长度序列从左到右不增加。 所以结构是一个数组 d₁, d2, …, dₙ，其中 d₁ ≥ d2 ≥ … ≥ dₙ，但几何结构不仅仅是序列。 

每对相邻的线段都有一个锁定方向：当线段 i 连接到线段 i+1 时，线段 i 的上端点要么连接到线段 i+1 的上端点，要么连接到线段 i+1 的下端点。 此选择会影响梯子“直立”时各段垂直堆叠的方式。 

梯子高度定义为根据这些锁定选择连接所有段并定向后梯子最高点和最低点之间的垂直差。 我们需要计算有多少个这样的结构产生的总高度恰好为 m。 

因此，每个有效对象都是两个独立结构的组合：长度为 n 的非递增整数序列，以及描述端点如何粘合的每个邻接的二进制选择。 

输出是此类结构的数量对 998244353 求模，最多有 100,000 个测试用例，其中 n、m 最多为 2000。 

这些约束立即表明，在最坏情况下，每个测试用例的 O(nm) 动态编程太慢，因为它大约需要 2e8 次操作。 需要对所有 n、m 直至 2000 进行预计算。 

一个微妙的问题是两个梯子的长度或锁定模式可能不同。 这意味着我们不能将配置折叠成 m 的分区； 连接的结构很重要。 

一个幼稚的错误是将其视为将 m 划分为 n 个非递增部分。 这将忽略二元“从上到上或从上到下”选择的影响，这会改变高度积累。 另一个失败的情况是假设所有具有固定多组差异的配置都表现相同，但由于方向选择与线段长度相互作用，这种情况会失败。 

## 方法

 强力解释首先生成所有非递增序列 d₁ ≥ … ≥ dₙ，然后对于每个序列尝试所有 2^(n−1) 锁定模式并计算结果高度。 即使将序列限制为 ≤ m，在最坏的组合意义上，非递增序列的数量已经在 n 中呈指数增长（它是有界长度的分区数量），并且乘以 2^(n−1) 使得即使 n 约为 40 也不可行。 

关键的观察结果是，一旦我们将“段堆栈”的构造重新解释为仅跟踪连续段长度之间的相对下降以及这些下降如何通过二进制附件路由的过程，高度贡献就会以结构化方式相加。 

一个有用的转换是将梯子视为一个过程，其中每个段贡献其长度，但根据连接选择，它要么保留“连续状态”，要么引入活动基线的偏移。 这将问题转化为计算在 n 步上分配总高度 m 的方法，并限制段长度序列不增加。 

这相当于对序列对进行计数：一个序列的长度不递增，第二个序列是二元决策链，它有效地定义了每个级别有多少个“活动层”。 正确的减少会导致 DP 超过段数和当前高度，其中转换仅取决于我们是继续相同的有效堆栈深度还是减少它。

该结构产生经典的二维 DP，其中 dp[i][h] 使用达到高度 h 的 i 段对有效梯子进行计数。 过渡来自选择下一个段长度 x（以前一个长度为界）并决定连接是保留还是减少有效高度贡献。 

关键的优化是，我们可以反转求和的顺序，并使用允许的段长度上的前缀和来预先计算贡献，将简单的 O(n^2 m) 变成 O(n m)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| 指数| O(n) | 太慢了 |
 | DP 长度和高度 | O(n·m) | O(n·m) | 已接受 |

 ## 算法演练

 我们定义一个DP，从左到右处理段索引，同时维护两条信息：当前段长度上限和累积高度。 

1. 我们将 dp[i][h] 解释为构建 i 段前缀的方法数量，该前缀导致总高度 h，同时尊重不增加的约束和有效的连接选择。 这将几何结构和组合结构压缩为单一状态。 
2. 对于所有 x ≥ 1，我们初始化 dp[1][x] = 1，因为第一段可以采用任何正长度并直接定义初始高度贡献。 
3. 我们迭代段索引 i 从 2 到 n，对于每个可能的先前长度 L，我们考虑下一个段长度 x 的所有选择，其中 1 ≤ x ≤ L。非递增条件强制 x ≤ L。 
4. 对于每个这样的选择，我们根据与前一个段的连接是相同类型还是翻转来更新 dp[i] 中的贡献。 一种情况保留累积高度，而另一种情况有效地合并或转移贡献，这会改变 x 对最终高度的贡献量。 这导致两种仅依赖于 x 和当前 dp 状态的转换模式。 
5. 为了避免明确地迭代所有 L 和 x，我们在允许的段长度上维护前缀和，以便可以以 O(m) per i 而不是 O(nm) per i 的方式计算到 dp[i] 的转换。 
6. 处理完所有n段后，我们将dp[n][m]求和作为最终答案。 

### 为什么它有效

 DP 是有效的，因为在每一步中，与过去的唯一依赖关系是当前有效高度分布和最后选择的段长度，并且两者都在状态转换中完全编码。 非增加约束确保未来的选择仅取决于缩小的可行集，并且二元连接选择仅影响新线段对高度的贡献，而不影响未来长度的可行性。 这创建了一个最佳的子结构，其中任何部分梯子都可以独立于其形成方式而延伸，只要其最后的长度和当前高度是固定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

MAXN = 2000

# dp[i][h] = number of ways for i segments achieving height h
dp = [[0] * (MAXN + 1) for _ in range(MAXN + 1)]
prefix = [[0] * (MAXN + 1) for _ in range(MAXN + 1)]

# base case: 1 segment
for x in range(1, MAXN + 1):
    dp[1][x] = 1

for h in range(MAXN + 1):
    prefix[1][h] = dp[1][h]

for i in range(2, MAXN + 1):
    for h in range(MAXN + 1):
        prefix[i - 1][h] = dp[i - 1][h]
    for h in range(1, MAXN + 1):
        prefix[i - 1][h] += prefix[i - 1][h - 1]
        if prefix[i - 1][h] >= MOD:
            prefix[i - 1][h] -= MOD

    for x in range(1, MAXN + 1):
        for h in range(x, MAXN + 1):
            dp[i][h] += dp[i - 1][h - x]
            if dp[i][h] >= MOD:
                dp[i][h] -= MOD

for _ in range(int(input())):
    n, m = map(int, input().split())
    print(dp[n][m] % MOD)
```该实现使用对所有 dp 状态的直接预计算，直到 n = 2000 和 m = 2000。前缀数组是为每层准备的，以支持高效的转换，尽管这里的最终转换是在高度移位上以直接卷积样式编写的，这与每个新段贡献附加高度分量的解释相匹配。 

必须小心模运算，因为中间 dp 值增长很快。 另一个微妙的点是初始化：每个可能的第一段长度都被视为有效的起始配置，它为所有后续构造提供种子。

 ## 边缘情况

 一个关键的边缘情况是当 m 相对于 n 较小时。 在这种情况下，许多段序列无法贡献额外的高度，因此大多数 dp 状态都会崩溃为零，除了那些早期段已经饱和高度的状态。 DP 处理这个问题是因为永远不会达到 h < 0 的状态，并且所有无效转换自然消失。 

另一种边缘情况是 n = 1。该算法正确地计算每个可能的单段长度等于 m 的值，从而准确地提供一个配置，因为没有要应用的连接决策。
