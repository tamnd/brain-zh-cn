 ## 解决方案
 ## 问题理解

 我们有无限的卡片集合，其中每张卡片都由固定整数的幂标记$n$。 第一张卡有价值$n^0$，第二个$n^1$，第三个$n^2$， 等等。 从这副牌中，通过选择纸牌的任何有限子集来形成“移动”，并且移动的值只是所选纸牌值的总和。 

所以每一步都对应着选择一些指数$i$，并求和$n^i$。 由于每张牌都可以选择或不选择，因此每一步操作本质上都是以下幂的子集和：$n$。 

然后将所有可能的移动按其结果总和排序，并要求我们提供$k$- 此排序中的最小总和。 

限制很大：两者$n$和$k$可以达到$10^9$。 这立即排除了任何显式生成子集甚至枚举值的尝试$k$通过模拟。 子集的数量随着考虑的卡片数量呈指数增长，因此即使对于适度的截止，简单的方法也会爆炸。 

一个微妙的点是了解是否包含空子集。 这些例子阐明了事实并非如此，因为序列从 1 开始，而不是从 0 开始。这意味着我们只考虑非空子集。 

典型的失败案例来自尝试贪婪地生成总和或使用子集上的优先级队列。 例如，即使只有权力$1, n, n^2$，子集的数量已经是8，并且扩展这个结构达到$k = 10^9$是不可能的。 

真正的困难是按排序顺序识别这些和的结构。 

## 方法

 暴力方法将尝试生成索引的所有子集并计算它们的总和，然后对它们进行排序。 即使我们尝试增量生成它们，子集的数量也会增长$2^m$在哪里$m$是考虑的权力的数量。 达到周围的价值观$k$，我们需要$m$足够大$2^m \ge k$， 意义$m \approx 30$已经足够了$k \le 10^9$。 然而，在最坏的情况下，显式枚举子集并对它们进行排序仍然需要存储大约十亿个值，这是不可能的。 

关键的见解是停止将子集视为组合对象，而是重新解释总和结构。 每个子集对应选择是否包含每个幂$n^i$。 这正是每个索引的二进制选择，因此每一步都对应于一个二进制字符串，并且该值是该二进制字符串在基数中的评估$n$仅使用数字 0 和 1。 

现在关键的观察是，按数值对这些值进行排序相当于对解释为具有位置权重的数字的二进制表示进行排序$n^i$。 由于较高的权力支配较低的权力（因为$n \ge 2$），这种排序与以标准 2 进制位置意义解释的二进制整数的自然排序完全一致。 

这意味着$k$-第一个最小值与书写完全对应$k$以二进制表示，并将每一位解释为是否包含相应的幂$n$。 这$i$第 位贡献$n^i$到最后的总和。 

所以我们不生成子集，而是直接解码$k$成二进制。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举|$O(2^m \log 2^m)$|$O(2^m)$| 太慢了|
 | 二进制表示映射 |$O(\log k)$|$O(1)$| 已接受 |

 ## 算法演练

 我们想要计算由二进制形式表示的值$k$，其中每一位对应于选择的幂$n$。 

1. 阅读$n$和$k$。 我们对待$k$作为子集和的排序列表中的排名。 
2. 转换$k$进入其二进制表示形式。 各位位置$i$告诉我们是否$n^i$包含在子集中。 
3、初始化运行答案为0，当前功率值为1，代表$n^0$。 
4. 迭代$k$从最不重要到最重要。 对于每一位：

 如果该位为1，则添加当前的功率$n$到答案。 
5. 处理完每一位后，将当前功率乘以$n$（模数$10^9+7$) 移动到下一个指数。 
6. 输出最终累加模$10^9+7$。 

原因乘以$n$有效的是每一步都让我们从$n^i$到$n^{i+1}$，与甲板的结构相匹配。 

### 为什么它有效

 每个子集唯一对应一个二进制字符串，其值是一个以基数为基数的位置数系统$n$并且数字限制为 0 或 1。$n \ge 2$，较高的索引始终支配较低的索引，确保按值排序与按二进制掩码的整数值排序相同。 因此子集和的排序顺序与整数的自然顺序相匹配$k = 1, 2, 3, \dots$，以及二进制表示$k$直接编码所需的子集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, k = map(int, input().split())

    ans = 0
    power = 1

    while k > 0:
        if k & 1:
            ans = (ans + power) % MOD
        power = (power * n) % MOD
        k >>= 1

    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了二值分解的思想。 变量`power`曲目$n^i$当我们扫描一些位时$k$。 每次遇到设置位时，我们都会将相应的幂添加到结果中。 每一步都应用模数以避免溢出。 

一个微妙的实现细节是，我们从不显式地预先计算 30 或 31 以内的所有幂。 相反，我们迭代地构建它们，这已经足够了，因为$k \le 10^9$意味着最多 30 位。

    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 一种边缘情况是当$k$是二的幂。 例如，$k = 8$对应于二进制`1000`，仅表示$n^3$被选中。 该算法自然地处理这个问题，因为只有一位起作用，并且所有较低的功率都被忽略。 

另一种情况是当所有低位都被设置时，例如$k = 7$给予`111`。 这里总和变成$1 + n + n^2$。 迭代累积可以正确构建此过程，无需任何特殊处理。 

最后一个微妙的案例很大$n$。 由于所有算术都是模数$10^9+7$，中间幂被安全地减少，并且乘法永远不会溢出 Python 中的典型整数限制，但模数减少仍然保持值有界和一致。
