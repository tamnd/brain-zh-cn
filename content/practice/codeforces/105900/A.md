 ## 解决方案
 ## 问题理解

 我们得到一组从 1 到 N 标记的山脉，其中每个标签也代表其难度级别。 存在 A 先于 B 形式的 M 个优先规则，这意味着 Isa 无法攀登 B 山，除非她已经攀登过 A 山。由于 A < B 始终得到保证，因此每个依赖项仅从编号较低的山指向编号较高的山。 

任务是构建一个顺序，让 Isa 只爬一次所有山峰，同时尊重所有依赖性。 在所有有效订单中，我们必须输出字典顺序最小的订单，这意味着我们总是倾向于尽早放置数量较少的山脉，前提是我们不违反任何先决条件限制。 如果不存在有效的订购，我们必须报告不可能。 

该结构是顶点 1 到 N 上的有向图，每个约束都是有向边 A → B。目标是拓扑排序，但不仅仅是任何拓扑排序，而是字典顺序中最小的拓扑排序。 

约束最多为 2 · 10^5 个顶点和边。 这立即排除了任何 O(NM) 或 O(N^2) 模拟策略。 即使是 O(NM) 的图遍历也太慢了。 唯一可接受的解决方案接近 O((N + M) log N) 或 O(N + M)。 

一个关键的结构约束是边缘只能从较小的标签到较大的标签。 这保证了该图在构造上是非循环的，因为您永远无法返回到较小的索引，因此循环不可能存在。 然而，我们仍然需要尊重排序约束并优化字典输出。 

一个天真的但重要的陷阱是认为简单地对邻接列表进行排序或贪婪地选择最小的可用节点而不进行仔细的依赖性跟踪就足够了。 例如，如果我们总是全局选择最小的未访问节点，我们可能会选择一个先决条件尚未满足的节点。 

另一个微妙的边缘情况是多个节点在不同时间变得可用。 如果我们不维护当前有效节点的全局结构，我们可能会错过正确的字典顺序最小延续。 

## 方法

 暴力的想法是重复扫描所有节点并选择尚未使用且先决条件已满足的最小节点。 选择它后，我们将其标记为已使用并重复。 为了每次检查有效性，我们要么重新计算入度，要么扫描影响它的所有边。 

这是合乎逻辑的，因为我们在每一步都确保遵守约束，但成本很高。 对于 N 个步骤中的每一个，扫描所有节点的成本为 O(N)，验证先决条件的成本总计为 O(M)。 这会导致 O(N^2 + M) 行为，当 N 为 2 · 10^5 时，这远远超出了限制。 

关键的观察是，这是一个经典的拓扑排序问题，唯一的区别是我们必须始终在入度为零的节点中选择最小的可用节点。 这意味着维持一组动态的当前有效候选人。 

一旦我们在一个总是给出最小元素的结构中维护具有零传入边的所有节点，该过程就会变得简单。 我们重复提取最小的可用节点，将其附加到答案中，并删除其传出边缘，可能会激活新节点。 最小堆或优先级队列是实现此目的的自然工具。 

条件 A < B 确保图已经是非循环的，因此除了验证我们是否在最后处理了所有节点之外，我们不需要循环检测逻辑。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力扫描每一步| O(N^2 + M) | O(N^2 + M) | O(N + M) | 太慢了|
 | 最小堆字典序拓扑排序 | O((N + M) log N) | O((N + M) log N) | O(N + M) | 已接受 |

 ## 算法演练

我们将问题建模为有向图并计算每个节点的入度。 入度为零的节点是那些先决条件已经满足的节点。 

1. 为所有边 A → B 构建邻接表，并计算每条边的入度[B]。 这准确地编码了每座山还需要多少先决条件。 
2. 初始化一个最小堆（优先级队列）并插入每个节点 i，使得 in Degree[i] = 0。这些山峰可以在不违反约束的情况下立即攀爬。 
3. 反复从堆中取出最小的元素。 这确保了在当前所有可用的山脉中，我们始终选择字典顺序最小的选项。 
4. 将所选节点附加到答案序列中。 
5. 对于所选节点 u 的每个邻居 v，将 in Degree[v] 减 1。如果 in Degree [v] 变为零，则将 v 推入堆中。 这表示 v 的所有先决条件现已满足，因此可供选择。 
6. 继续，直到堆为空。 
7. 如果结果序列的长度为N，则输出它。 否则输出-1。 

始终将新解锁的节点立即推入堆的正确性确保我们永远不会延迟可以按字典顺序更早选择的有效候选者。 

### 为什么它有效

 在每一步中，堆都准确地包含先前选择的节点已满足其先决条件的节点集。 任何不在堆中的节点都有剩余依赖，因此选择它是无效的。 在有效的候选者中，选择最小的标签是最佳的，因为任何更大的选择都会使序列按字典顺序恶化，而不会在以后产生任何优势。 依赖结构保证一旦节点变得可用，延迟它只能将其进一步推向正确的顺序，而永远不会提高可行性或字典顺序。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n + 1)]
    indeg = [0] * (n + 1)

    for _ in range(m):
        a, b = map(int, input().split())
        adj[a].append(b)
        indeg[b] += 1

    heap = []
    for i in range(1, n + 1):
        if indeg[i] == 0:
            heapq.heappush(heap, i)

    res = []

    while heap:
        u = heapq.heappop(heap)
        res.append(u)

        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                heapq.heappush(heap, v)

    if len(res) != n:
        print(-1)
    else:
        print(*res)

if __name__ == "__main__":
    solve()
```邻接列表存储所有依赖边，而入度数组则跟踪每个节点仍有多少个先决条件未满足。 堆是通过始终选择最小的可用节点来强制字典编排最优性的关键结构。 

一个微妙的实现细节是，只有当节点的入度恰好为零时，节点才会被推入堆中。 提前推送会违反正确性，因为该节点尚未有效使用。 

最终检查确保所有节点均已处理。 即使输入保证 A < B，此检查仍然是拓扑排序问题的标准安全条件。

 ## 边缘情况

 一种重要的边缘情况是不存在依赖关系。 在这种情况下，每个节点最初的入度都为零，因此堆包含所有节点。 然后，该算法只需按顺序输出 1 到 N，因为堆在每一步都强制执行字典顺序选择。 

另一种情况是依赖关系形成严格的链。 例如，1 → 2 → 3 → 4。堆始终仅包含一个元素，因此该算法的行为就像一种简单的遍历。 排序没有歧义，堆开销也不会改变正确性。 

更微妙的情况是节点较晚可用但小于已可用的节点。 例如，如果节点 3 在节点 5 之后变得可用，则堆可确保在插入时立即选择节点 3，从而保持字典最小性。 这正是天真的“每次扫描所有候选人”方法会处理不当或实施效率低下的情况。
