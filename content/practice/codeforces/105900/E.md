 ## 解决方案
 ## 问题理解

 我们正在寻找长度的字符串`n`由26个小写英文字母组成，但我们不关心实际使用的字母。 相反，如果可以通过一致重命名字母将一个字符串转换为另一个字符串，则认为两个字符串是相同的，其中每个字母必须映射到不同的字母。 

这意味着重复的结构才是重要的。 例如，在长度为 3 的字符串中，字符串`aba`,`cbc`， 和`xyx`结构上都是相同的，因为第一个和最后一个位置匹配，而中间位置不同。 另一方面，`aab`和`aba`之所以不同，是因为职位之间的平等关系不同。 

所以任务是计算有多少个不同的长度“相等模式”`n`假设我们有一个足够大的字母表（26 个字母），但只要我们不超过可用的不同字母，就允许重复使用。 

平等模式可以被视为位置的划分`1..n`分成组，同一组中的位置必须共享相同的字母，不同组必须使用不同的字母。 单射映射条件强制不同的组不能折叠成一个字母。 

约束条件`n ≤ 10^6`立即排除分区上的任何组合枚举、斯特林数或子集上的 DP。 任何东西甚至是二次方`n`是不可能的。 我们需要一个封闭形式的组合恒等式或快速递归。 

出现微妙的边缘情况时`n > 26`。 那么并非所有分区都是有效的，因为我们不能分配超过 26 个不同的字母。 因此必须排除超过 26 个块的分区。 对于小`n ≤ 26`，每个分区都是有效的。 对于较大的`n`，我们只允许最多 26 个块的分区。 

## 方法

 如果我们忽略效率，直接的解释是生成所有长度的字符串`n`，通过按首次出现的顺序重新标记字母来将每个字母简化为其规范形式，并计算不同的规范形式。 这一下子就炸了：有`26^n`字符串，甚至将它们分组也是不可能的。 

重新定义问题是关键一步。 我们考虑的不是字符串，而是从左到右扫描时出现了多少个新字母。 每个位置要么重复使用现有符号，要么引入新符号。 该结构完全取决于“新的不同字母”何时出现以及后面的位置如何引用前面的位置。 

这导致了一个经典的解释：答案是一个集合的划分的数量。`n`将元素排序序列放入标记块中，其中块对应于字母，但标签与重命名无关。 这正是贝尔号码`B(n)`当字母表无界时。 

但是，我们的字母上限为 26 个，因此我们需要一个字母的分区数`n`元素设置为最多 26 个块。 那是：$$\sum_{k=1}^{\min(n, 26)} S(n, k)$$在哪里`S(n, k)`是第二类斯特林数。 

我们无法计算完整的 DP`n = 10^6`在二维表中。 解锁解决方案的观察结果是我们只需要`k ≤ 26`，一个恒定的上限。 这使我们能够使用 1D DP 迭代计算斯特林数`k`，对每个进行更新`n`。 

复发：$$S(n, k) = S(n-1, k-1) + k \cdot S(n-1, k)$$让我们逐行构建，但最多只保留 26 列。

 ## 算法演练

 我们动态构建斯特林数，同时仅保留最后计算的行。 

1.初始化一个DP数组`dp[k]`， 在哪里`dp[k]`表示将当前前缀划分为多少种方式`k`块。 为了`n = 0`，我们设置`dp[0] = 1`因为空结构只有一个微不足道的分区。 
2. 迭代`i`从`1`到`n`。 在每一步中，我们都会更新一个新数组`ndp`代表大小的分区`i`。 
3. 对于每个`k`从`1`到`min(i, 26)`, 计算`ndp[k] = dp[k-1] + k * dp[k]`。 第一项对应于使新元素成为自己的块，第二项对应于将其插入到任何现有块中。 
4. 灌装后`ndp`， 代替`dp`和`ndp`。 
5.处理完所有位置后，求和`dp[k]`为了`k = 1`到`min(n, 26)`以获得最终答案。 

对 26 的限制确保我们永远不会分配或计算超出恒定宽度，从而使过程保持线性`n`。 

### 为什么它有效

 每一步`i`，DP对前缀的所有有效分区进行编码`1..i`。 每个新元素要么形成一个新的等价类，要么加入一个现有的等价类。 这完全符合斯特林数的递推定义。 因为我们不允许超过 26 个类，所以通过截断 DP 维度自然会排除无效状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input().strip())
    
    K = 26
    dp = [0] * (K + 1)
    dp[0] = 1
    
    for i in range(1, n + 1):
        ndp = [0] * (K + 1)
        lim = min(i, K)
        for k in range(1, lim + 1):
            ndp[k] = (dp[k - 1] + k * dp[k]) % MOD
        dp = ndp
    
    ans = sum(dp) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```DP 数组通过将其大小固定为 26 来保持较小。转换直接对斯特林递推进行编码。 我们小心地将循环限制在`min(i, 26)`所以我们永远不会访问无效状态。 最终总和汇总了所有有效分区，最多包含 26 个块。

 ## 边缘情况

 对于`n = 1`，DP 从一个状态开始，并立即产生一个有效模式，对应于任何单个字母。 循环从不尝试无效的转换，因为`dp[0]`是唯一的贡献基地。 

为了`n ≤ 26`，所有斯特林数都有贡献并且不会发生截断。 DP 的行为与标准贝尔数计算完全相同，仅限于`n`元素，因此字母限制不会影响结果。 

为了`n > 26`，永远不会创建超过 26 个区块的状态，因为`lim = min(i, 26)`上限转换。 这可确保自动排除需要更多不同字母的分区，而不是稍后过滤，从而避免任何后处理的需要。
