 ## 解决方案
 ## 问题理解

 给定三个小整数$A$,$B$， 和$C$，然后是最多 1000 个查询值的列表$X_1, X_2, \dots, X_Q$。 对于每个查询值，我们必须计算一个固定的三次表达式，取其绝对值，最后使用按位异或组合所有结果。 

该函数是一个类似多项式的表达式，混合了三次项、二次项和线性项，再加上一个额外的乘积项：$$f(x) = |(A + C)x^3 - Bx^2 + BCx + (x + C)(x - A)|$$计算后$f(X_i)$对于每个查询，输出是所有这些值的异或。 

约束条件非常小：所有系数和输入都位于$[-100, 100]$，最多有 1000 条评价。 这立即意味着任何直接计算每个值的方法都已经足够快了，因为每次评估都是持续的工作，而且我们只执行 1000 次。 

一个微妙的点是，在取绝对值之前，表达式可能会变成负值，即使输入很小，中间结果也可能超出各个变量的范围。 例如，当$x = 100$，三次项可以达到大约$10^6$，所以我们必须依赖Python的任意精度整数或其他语言中仔细的整数处理。 

另一个潜在的陷阱是误解 XOR 聚合。 这不是求和，也不是模运算。 每个计算值独立地贡献于最终的二进制异或累加器，因此处理的顺序并不重要。 

边缘情况主要与标志处理和取消有关：

 如果$A = B = C = 0$，则表达式简化为$|x(x)| = x^2$。 忘记绝对值的简单实现在许多对称情况下仍然会产生正确的 XOR，但只要中间表达式为负数就会失败。 

如果$x = 0$，则表达式简化为$|C \cdot (-A)| = | -AC |$，一般来说不为零。 假设立方占主导地位的粗心简化可能会错误地删除低阶项。 

## 方法

 暴力方法直接计算每个的表达式$x$，应用绝对值，并对结果进行异或。 由于每次评估仅涉及恒定数量的算术运算，因此总工作量为$O(Q)$，在约束条件下已经是最优的了。 

简单实现中唯一潜在的问题是固定宽度整数语言中的溢出，但这里即使是最坏情况的大小也保持在 64 位整数的安全范围内：$100^3 = 10^6$，因此多项式仍然很小。 

除了直接评估之外，没有任何隐藏结构可供利用。 关键的观察结果是，该问题不要求对查询进行优化，只要求重复评估固定表达式。 一旦认识到这一点，解决方案就变得纯粹以实施为中心。

 ## 算法演练

 1. 阅读$A$,$B$， 和$C$。 这些在所有评估过程中保持不变，因此可以直接在循环内重用它们，而无需重新计算或存储。 
2.读取查询数$Q$，后跟值列表$X_i$。 这些值中的每一个都将被独立处理，因此不需要预处理。 
3. 初始化累加器变量`ans = 0`。 该变量存储所有评估结果的运行异或。 XOR 具有关联性和交换性，因此我们不需要保留顺序或存储中间结果。 
4.对于每个查询值$x$，计算多项式表达式：$$(A + C)x^3 - Bx^2 + BCx + (x + C)(x - A)$$乘法是直接使用整数运算完成的。 我们完全避免预扩展，因为可读性和正确性比这种规模的微优化更重要。 
5. 取计算结果的绝对值。 这一步至关重要，因为函数的定义显式地丢弃了符号信息。 异或必须应用于这些绝对量值，而不是原始多项式输出。 
6. 更新累加器：`ans ^= value`。 这会将当前结果集成到全局 XOR 中，而无需存储以前的值。 
7.处理完所有查询后，输出`ans`。 

### 为什么它有效

 每个查询恰好贡献一个从其输入值确定性导出的整数。 由于 XOR 是对每个整数独立应用的按位运算，因此最终结果仅取决于计算值的多重集，而不取决于它们的顺序。 该算法仅计算每个值一次，并使用 XOR 将它们组合起来，因此不会丢失或重复信息。 绝对值步骤确保计算出的序列与函数定义完全匹配，从而通过构造使累加正确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

A, B, C = map(int, input().split())
Q = int(input())
xs = list(map(int, input().split()))

ans = 0

for x in xs:
    val = (A + C) * x * x * x - B * x * x + B * C * x + (x + C) * (x - A)
    if val < 0:
        val = -val
    ans ^= val

print(ans)
```实现几乎逐字遵循该算法。 三次项的计算明确为`x * x * x`，考虑到较小的限制，这是安全的。 使用比较手动处理绝对值，而不是调用`abs`，尽管两者都有效。 

一个微妙的实现细节是中间乘法顺序只影响性能，而不影响正确性。 Python 处理大整数，因此不会出现溢出问题。 XOR 累加器会根据查询立即更新，从而避免任何存储需求。

 ## 边缘情况

 考虑$A = B = C = 0$。 那么表达式就变成了$|x^2|$，它始终是非负的。 用于输入$x = [-2, 0, 3]$，评估值为$4, 0, 9$，异或为$4 \oplus 0 \oplus 9 = 13$。 该算法自然地处理这个问题，因为它总是应用绝对值和 XOR 聚合而不假设符号。 

现在考虑具有负系数的情况，例如$A = -10, B = 5, C = -5$和$x = 1$。 由于项之间的线性相互作用占主导地位，原始多项式很容易变为负值。 该算法计算完整表达式，应用绝对值，然后对其进行异或。 任何试图在没有仔细跟踪符号的情况下进行代数简化的方法都有可能降低绝对值效应，这将完全改变结果。 

最后，为了$x = 0$，表达式简化为$|C \cdot (-A)|$。 该算法仍然评估所有项，并且所有三次和二次部分自然消失。 这证实了该实现可以正确处理简并输入，而无需特殊的大小写。
