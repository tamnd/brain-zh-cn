 ## 解决方案
 ## 问题理解

 我们正在比较两个装满冰淇淋的容器：一个圆柱形瓶子和一个顶部有额外圆顶的圆锥体。 

瓶子是一个带有半径的简单圆柱体`rL`和身高`hL`，完全充满。 它的容量只是普通气缸的容积。 

锥体侧面由一个半径为 的直圆锥组成`rC`和身高`hC`，也完全填充，然后是相同半径的实心半球`rC`上面还放着冰淇淋。 因此，锥形容器实际上有两个部分构成体积。 

任务是确定瓶子是否比圆锥半球结构严格容纳更多的冰淇淋。 如果是，我们输出`Injusto`，否则我们输出`Justo`。 

输入大小非常小，所有维度最多为 100。这立即排除了任何高级优化的需要。 直接的常数时间计算就足够了，甚至浮点运算在技术上也可以工作，尽管它引入了不必要的精度风险。 

唯一微妙的问题是体积涉及 π，但由于两边都包含 π 作为因子，所以如果我们进行代数比较，它就完全抵消了。 保留 π 并使用浮点比较的粗心实现仍然可以通过，但它是可以避免的并且不太健壮。 

边缘情况主要是关于平等和边界比较。 例如，当两个容器的容量相同时，答案必须是`Justo`因为条件要求瓶子严格要大一些。 

## 方法

 强力解释将使用几何公式直接计算两个体积。 瓶子很简单：圆柱体体积`π rL^2 hL`。 圆锥是圆锥体积的总和`(1/3)π rC^2 hC`和半球体积`(2/3)π rC^3`。 

这种方法已经是O(1)了，所以不存在算法瓶颈。 唯一潜在的低效率是浮点运算和 π 的不必要使用，这在比较缩放后应该是精确整数的值时引入了舍入问题。 

关键的观察结果是 π 对于每个项都是通用的。 如果我们将所有内容乘以 3/π，我们会将所有体积转换为整数：

 瓶子变成`3 rL^2 hL`，圆锥体就变成了`rC^2 hC + 2 rC^3`。 经过此转换后，比较是纯粹基于整数的且精确的。 

这将问题简化为单个整数比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 带有浮点的直接几何 | O(1) | O(1) | O(1) | O(1) | 可行，但精度有风险
 | 缩放整数比较 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 最佳方法

 1.读取四个整数`rL, hL, rC, hC`。 它们定义了两个几何实体，我们将比较其体积。 
2. 计算缩放后的瓶子体积：`Vb = 3 * rL * rL * hL`。 这是通过将圆柱体体积乘以 3 并除去 π 得出的。 
3. 计算缩放后的圆锥加半球体积：`Vc = rC * rC * hC + 2 * rC * rC * rC`。 第一项对应于圆锥体，第二项对应于缩放后的半球体。 
4. 比较`Vb`和`Vc`。 如果`Vb`严格更大，输出`Injusto`。 否则输出`Justo`。 

缩放背后的原因是它在消除无理常量的同时准确地保留顺序。 

### 为什么它有效

 两个原始体积都与 π 成正比。 由于 π 在两个表达式中都是正数且相同，因此乘或除 π 不会改变顺序。 同样，将所有内容乘以 3 可以清除圆锥公式中的分数。 经过此转换后，两个表达式都变成了精确表示比例体积的整数，因此比较它们在数学上相当于比较原始体积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    rL, hL, rC, hC = map(int, input().split())

    bottle = 3 * rL * rL * hL
    cone = rC * rC * hC + 2 * rC * rC * rC

    if bottle > cone:
        print("Injusto")
    else:
        print("Justo")

if __name__ == "__main__":
    solve()
```实现直接遵循代数简化。 唯一需要注意的是整数运算的一致使用。 所有中间值都适合 32 位整数，因为最大幅度约为`3 * 100^3 = 3e6`。 

决策步骤按照声明的要求使用严格的不平等。 任何平局默认为`Justo`。

 ## 边缘情况

 一种重要的边缘情况是缩放后相等。 例如，`1 4 1 10`产生相同比例的体积。 该算法准确地计算了两个表达式并且正确地落入`Justo`分支，因为条件很严格。 

另一种情况是锥体半径较大但高度较小时。 即使半球项按立方体增长，如果其半径或高度足够大，瓶子仍然可以占据主导地位。 整数比较处理这个问题没有任何不稳定。 

最终边缘场景的最大值约为 100。即便如此，最大中间值约为`3 * 100^3 = 3,000,000`，完全在整数限制内，因此不会出现溢出或精度问题。
