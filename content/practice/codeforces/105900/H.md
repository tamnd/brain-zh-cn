 ## 解决方案
 ## 问题理解

 在 2D 平面上给出两个移动物体，每个物体都表示为一个刚性三角形。 每个三角形由零时刻的三个固定顶点描述，然后整个三角形以恒定速度矢量平移。 形状不会旋转或变形，只是随时间线性变化。 

任务是确定这两个移动三角形是否在任何时刻重叠，如果重叠，则计算它们第一次接触或相交的最早时间。 如果它们在任何时候都不相交，我们必须输出−1。 

思考该问题的一个有用方法是，每个三角形都成为时空中连续的 3D 对象：在时间 t 时，每个点都线性移动，因此三角形扫出一个棱镜。 当这两个棱镜在某个时间片 t ≥ 0 相交时，就会发生碰撞。 

对坐标的约束在数量级上很大，但足够小，以至于所有几何计算都可以使用浮点或精确算术通过仔细处理来完成。 关键的挑战不是输入的大小，而是时间的连续性，这可以防止暴力破解时间步长。 

简单的离散化会尝试检查许多时间点，但由于速度可能很大并且相交可能发生在非常精确的分数时间，因此这种方法从根本上来说是不正确的。 

当三角形在一个点或沿着一条边完全接触时，就会出现微妙的边缘情况。 例如，如果在时间 t = 0.5 时它们恰好接触到一个顶点，则仍算作碰撞。 另一种边缘情况是三角形平行移动并且从不相交，即使它们的投影在某些时间间隔内重叠。 

最重要的结构问题是交集条件在时间上不是单调的。 两个三角形可能不相交，然后相交，然后再次不相交，具体取决于相对运动。 这排除了在没有更深入推理的情况下按时间进行简单二分搜索的情况。 

## 方法

 蛮力的想法是连续或以非常小的步长模拟时间，并检查三角形是否在每个采样时刻相交。 对于每个时间 t，我们都会平移两个三角形并运行多边形相交测试，这会花费恒定的时间。 然而，即使我们只考虑 10^6 个时间样本，我们也很容易错过准确的碰撞时刻，因为它可能发生在由线相交方程确定的无理时间或高精度有理时间。 这使得离散化变得太慢而且不正确。 

关键的见解是将视角从绝对运动转变为相对运动。 我们不移动两个三角形，而是固定一个三角形，让另一个三角形以相对速度差移动。 现在问题变成：固定三角形是否与移动三角形相交？ 

这将问题简化为凸多边形之间的经典连续碰撞检测问题。 由于三角形是凸的，我们可以使用分离轴定理。 对于两个凸多边形，当且仅当没有分离轴时它们相交，并且每个轴约束在时间上成为线性不等式。 

每个潜在的分离方向由任一三角形的边法线定义。 对于固定方向，两个三角形在该轴上的投影变成随时间线性移动的区间。 该轴上的交点变为 a + bt ≤ c + dt 形式的简单不等式。 每个轴贡献一个可能重叠的时间间隔，并且总体碰撞间隔是所有这些间隔的交集。 

因此，问题简化为计算由边缘法线引起的所有有效时间间隔，将它们相交，并在相交处取最小的非负时间。

 ## 算法演练

 我们首先将问题转化为相对运动。 如果三角形 A 的速度为 vA，三角形 B 的速度为 vB，则我们固定 A 并以速度 v = vB − vA 移动 B。 这不会改变碰撞事件的相对几何形状，只是简化了描述。 

接下来，我们考虑候选分离轴。 对于凸多边形，任何分离轴必须垂直于任一多边形的某些边。 由于这两个形状都是三角形，因此我们总共只有六个边，因此有六个候选方向。 

对于每个轴，我们将两个三角形投影到该轴上。 三角形投影是一个区间，其端点是时间的线性函数。 具体来说，每个顶点贡献一个线性函数 x(t) = x0 + v · t，因此投影间隔的每个端点在时间上也是线性的。 

然后，我们为每个轴计算两个投影间隔重叠的时间间隔。 这成为一对线性不等式，我们将其求解为有效 t 的单个区间。 

我们将所有这些间隔在所有轴上相交。 最终的可行时间集是实线上最多六个区间的交集。 如果在 t ≥ 0 时交集为空，则不存在碰撞。 否则，交集中最小的 t 就是答案。 

### 为什么它有效

 对于凸多边形，不相交相当于存在分离轴。 由于运动是线性的，因此任何固定轴上的投影间隙随时间线性变化。 因此，该轴上重叠的条件简化为时间上的线性可行性约束。 由于碰撞要求所有轴同时重叠，因此与这些约束相交可以准确捕获碰撞时间集。 在不满足所有轴约束的情况下，任何其他几何配置都不会导致相交。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dot(ax, ay, bx, by):
    return ax * bx + ay * by

def proj_interval(poly, vx, vy):
    mn = float('inf')
    mx = -float('inf')
    for x, y in poly:
        val = x * vx + y * vy
        mn = min(mn, val)
        mx = max(mx, val)
    return mn, mx

def get_axes(poly1, poly2):
    axes = []
    polys = [poly1, poly2]
    for poly in polys:
        for i in range(3):
            x1, y1 = poly[i]
            x2, y2 = poly[(i + 1) % 3]
            dx, dy = x2 - x1, y2 - y1
            axes.append((-dy, dx))
    return axes

def solve():
    p1 = [tuple(map(int, input().split())) for _ in range(3)]
    v1 = tuple(map(int, input().split()))
    p2 = [tuple(map(int, input().split())) for _ in range(3)]
    v2 = tuple(map(int, input().split()))

    vx = v2[0] - v1[0]
    vy = v2[1] - v1[1]

    axes = get_axes(p1, p2)

    lo = 0.0
    hi = float('inf')

    for ax, ay in axes:
        mn1, mx1 = proj_interval(p1, ax, ay)

        shifted = []
        for x, y in p2:
            shifted.append((x, y))

        mn2, mx2 = proj_interval(shifted, ax, ay)

        # projection of p2 moves linearly: shift both endpoints by t*(vx,vy)
        vproj = vx * ax + vy * ay

        if vproj == 0:
            if mx2 < mn1 or mx1 < mn2:
                print(-1)
                return
            continue

        # solve overlap: mn2 + t*vproj <= mx1 and mx2 + t*vproj >= mn1
        t1 = (mn1 - mx2) / vproj
        t2 = (mx1 - mn2) / vproj

        if t1 > t2:
            t1, t2 = t2, t1

        lo = max(lo, t1)
        hi = min(hi, t2)

        if lo > hi:
            print(-1)
            return

    if hi < 0:
        print(-1)
        return

    lo = max(lo, 0.0)
    print(lo)

if __name__ == "__main__":
    solve()
```解决方案首先将第二个三角形转换为相对运动系统，因此只有一个速度矢量重要。 这是至关重要的，因为所有后续计算都依赖于单个对象相对于静态参考的线性移动。 

每个轴均源自三角形边，因为只有这些方向才能定义分离超平面。 投影步骤将 2D 凸交集问题简化为 1D 区间重叠问题。 

对于每个轴，关键操作是将间隔重叠转化为时间上的线性不等式。 变量`vproj`是投影间隔沿该轴移动的速度。 如果它为零，我们只检查时间间隔是否在零时已经重叠，因为它们永远不会改变相对位置。 

否则，我们计算移动区间与固定区间相交的时间范围。 这些边界通过交集跨所有轴合并，任何空交集立即证明不存在碰撞。 

一个微妙的点是除以时保持正确的顺序`vproj`，因为它的符号翻转了不等式的方向。 交换`t1`和`t2`确保我们始终将它们视为一个间隔。

 ## 边缘情况

 临界边缘情况是轴上的投影速度为零时。 在这种情况下，间隔的相对顺序不会随时间改变。 该算法通过仅在零时间检查重叠来正确处理此问题。 如果它们最初是不相交的，那么未来的时间就无法解决这个问题。 

当碰撞正好发生在 t = 0 时，会出现另一种边缘情况。该算法将最终答案限制为`lo = max(lo, 0.0)`，确保不会错过直接交叉路口。 

最后的边缘情况是两个三角形在一个点相接触。 在投影空间中，这对应于至少一个轴间隔边界上的相等。 由于间隔计算中包含不等式，因此这种情况会产生有效的零宽度交叉间隔，该间隔在该确切时间被正确报告为碰撞。
