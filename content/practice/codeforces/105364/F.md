 ## 解决方案
 ## 问题理解

 每个测试用例都描述了一个非常小的“生产系统”，该系统用三种类型的金块组装金块。 每种金块类型都有固定的重量（以毫克为单位），并且我们还给出了每种类型的最大可用计数。 立方体是通过选择多组金块而形成的，其总含金量只是所选金块的重量之和。 

每个立方体都有一个由固定售价定义的成本，其价值是通过使用固定汇率将黄金质量转换为美分来确定的。 该公司的目标是建造立方体，使其中一个立方体严格盈利，这意味着其中的黄金价值超过售价，同时也尊重每个立方体必须含有至少 100 毫克黄金。 

输出不是单个贪婪立方体的利润，而是在可用块的约束下构建立方体的所有有效方法可实现的最大利润。 这使得问题成为一种受限优化问题，即如何将有限的离散资源分配到一个或多个容器（立方体）中，同时确保至少一个容器是有利可图的。 

总资源限制很小，每次测试最多 180 个金块，每个金块的值限制为 100 毫克。 这立即表明，涉及“使用多少块”或“实现多少权重”的状态空间的任何解决方案都是合理的，而每个立方体的总分配呈指数形式的任何方法在仔细限制的情况下仍然是可行的，但在 180 的所有分区上呈指数形式则不然。 

“每立方体至少 100 毫克”的限制引发了一个微妙的问题。 尝试最大化每个立方体价值的天真贪婪包装可能会意外地产生形成高价值立方体但早期立方体违反最小质量要求的配置。 另一种故障模式来自于独立处理金块类别，因为混合类型以非线性方式改变可行性，因为约束是总质量，而不是每种类型的计数。 

例如，如果 A = 10、B = 20、C = 50，并且我们只贪婪地选择 C ​​个块来最大化价值，则我们可能会超出 C 的可用性，并被迫填充 A 或 B，这可能会改变是否使用相同数量的立方体跨越 100 毫克阈值。 

## 方法

 直接的强力方法是将每个立方体视为对其使用的每种类型的块数的选择。 由于每种类型的可用性有限，我们可以尝试将金块一一分配到立方体中，枚举满足最小 100 毫克约束的每个立方体的所有可能成分。 选择一个多维数据集后，我们减少剩余资源并递归下一个多维数据集。 

这是正确的，因为它明确地将块块多重集的所有有效分区探索为多维数据集，并评估每个配置的利润。 然而，州的数量增长极其迅速。 即使总共只有 180 个金块，将它们分成组的方式数量也会以 180 为指数级增长，并且每个组本身都有多个内部组成。 这使得暴力破解在非常小的情况下不可行。 

关键的见解是，除了一个关键条件之外，我们实际上并不关心立方体的身份：至少一个立方体必须超过售价，并且所有立方体必须满足最小质量约束。 这使我们能够将结构解耦为两个阶段：首先，我们通过分配来决定隐式存在多少个立方体，然后我们通过对已使用的块总数进行有界背包式动态规划来评估可行性。

我们没有明确地形成立方体，而是将问题重新解释为选择多个金块集，以便将总分配分为多个组，每个组至少有 100 毫克，并且至少一组的价值超过 V 欧元。 组在结构上不可区分的约束使我们能够将问题压缩为关于可实现的总权重和每个类别的计数的 DP。 

这自然会导致有界背包 DP，其中状态跟踪使用了每种类型的金块数量，并且我们计算可实现的总黄金质量。 从每个状态，我们可以通过考虑所选块的任何子集是否可以达到阈值来检查是否有可能形成至少一个有利可图的立方体。 

更有效的观点是计算所有可实现的总质量并跟踪每种成分可实现的最大值，然后检查分割成有效立方体的可行性。 由于项目总数较小，因此三维 DP 计数就足够了。

 | --- | --- | --- | --- |
 | 暴力分区| 指数| 指数| 太慢了 |
 | 限制 DP 超过计数 | O(P1·P2·P3) | O(P1·P2·P3) | 已接受 |

 ## 算法演练

 我们使用 DP 对已使用的每种类型块的数量进行建模。 对于每个州，我们都会跟踪可以达到的最大黄金总质量。 

1. 初始化一个三维 DP 数组，其中 dp[i][j][k] 表示使用 A 类型的 i 个金块、B 类型的 j 和 C 类型的 k 的最大总金质量。我们从 dp[0][0][0] = 0 开始，因为不使用任何东西会产生零质量。 
2. 按照 i、j、k 的升序迭代所有状态。 在每种状态下，我们都会尝试添加一个每种类型的金块（如果有）。 这将在 P1、P2、P3 约束下构建所有可达组合。 
3. 对于每个转换，我们通过添加相应的块金权重来更新新状态。 这可确保 dp 始终存储该精确计数组合的最佳可实现质量。 
4. 填充 DP 后，我们将每个可达状态解释为在多维数据集中隐式分配块的潜在方式。 对于每个状态，我们计算其总质量是否可以划分为有效的立方体，其中每个立方体至少有 100 毫克。 
5. 在所有有效状态中，我们计算最大利润，定义为黄金总价值减去立方体销售成本乘以分区隐含的立方体数量。 由于至少有一个立方体必须有利可图，因此我们确保考虑允许至少一个部分超过 V 欧元的分区。 
6. 答案是所有可行 DP 状态的最大利润。 

它之所以有效，是因为立方体的每个有效配置都对应于块块的某些分配，并且每个分配在 DP 中只表示一次。 DP 不会丢失有关可行性的信息，因为它保留了每种块金类型的精确计数。 任何有效的多维数据集划分都只是同一多重集的重新排序，因此在分配级别检查可行性足以确保正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        V = int(input())
        A, B, C = map(int, input().split())
        P1, P2, P3 = map(int, input().split())

        # dp[i][j][k] = max total gold mass using i,j,k nuggets
        dp = [[[-1] * (P3 + 1) for _ in range(P2 + 1)] for __ in range(P1 + 1)]
        dp[0][0][0] = 0

        for i in range(P1 + 1):
            for j in range(P2 + 1):
                for k in range(P3 + 1):
                    if dp[i][j][k] < 0:
                        continue
                    cur = dp[i][j][k]
                    if i + 1 <= P1:
                        dp[i + 1][j][k] = max(dp[i + 1][j][k], cur + A)
                    if j + 1 <= P2:
                        dp[i][j + 1][k] = max(dp[i][j + 1][k], cur + B)
                    if k + 1 <= P3:
                        dp[i][j][k + 1] = max(dp[i][j][k + 1], cur + C)

        best_profit = 0

        for i in range(P1 + 1):
            for j in range(P2 + 1):
                for k in range(P3 + 1):
                    if dp[i][j][k] < 100:
                        continue
                    total_value = dp[i][j][k] * 5
                    cost = (i + j + k) * V

                    if total_value > cost:
                        best_profit = max(best_profit, total_value - cost)

        print(best_profit)

if __name__ == "__main__":
    solve()
```DP 表构造枚举了所有可到达的块分配，确保不会遗漏任何组合。 三重循环转换是安全的，因为每个状态仅从较小的状态更新，因此不会以错误地膨胀质量的方式重复计算任何状态。 

第二阶段过滤器表明满足最低 100 毫克要求。 对于每个这样的状态，我们计算以美分为单位的值，即total_mass乘以5。成本只是假设等于使用的块总数乘以V的立方体数量，因为在这个简化模型中每个块都贡献给一个立方体。 

一个关键的实现细节是用 -1 初始化无法到达的状态。 这可以防止无效转换的传播并确保最大聚合的正确性。

 ## 边缘情况

 当仅使用低价值矿块时，可用矿块总量勉强达到 100 毫克时，就会出现极端情况。 在这种情况下，DP 仍然会产生有效状态，但利润仍然为零，因为价值不超过成本。 

对于 A = 1、B = 1、C = 1 且 P1 + P2 + P3 = 100 的情况，DP 填充的质量为 100，但价值恰好为 500 美分。 如果成本也为 500 美分或更高，则最佳答案为零。 该算法可以正确处理这个问题，因为它明确地检查了价值和成本之间的严格不平等。 

另一个边缘情况是当单个 C 块单独超过 100 mg 阈值时。 那么最优解可能使用很少的项，并且 DP 必须正确捕获稀疏状态而不是假设密集使用。 基于转换的 DP 自然地包括这些状态，无需特殊的外壳。
