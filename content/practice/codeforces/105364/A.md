 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每一个中，我们收到一个偶数长度的整数列表，并且我们必须决定是否可以将这些数字分成对，以使每对具有相同的总和。 每个元素必须仅使用一次，并且所有形成的对必须共享一个共同的目标总和。 

思考该任务的另一种方式是，我们尝试将数组重新排列成不相交的对，并且每对必须“平衡”到相同的值。 挑战不是明确地构建配对，而只是确定这样的配对是否存在。 

约束足够大，任何尝试显式测试配对的方法都会失败。 高达$5 \cdot 10^5$由于所有测试用例中的数字都相同，即使考虑每个用例的二次配对数量的解决方案也是不可能的。 这迫使我们采取一种策略，其中每个测试用例通常在线性或接近线性的时间内处理$O(n \log n)$或更好。 

当所有值都相同或值重复分布不均匀时，就会出现一个微妙的问题。 例如，如果所有数字都相同，则始终可以配对，因为每对都有相同的总和。 另一方面，如果多重集是倾斜的，使得只有一个候选和可以适用于一对，但不能一致地适用于所有对，则天真的贪婪配对可能会错误地假设可行性或失败，具体取决于配对顺序。 

考虑一个误导性的案例，例如：```
4
1 2 3 4
```如果我们贪婪地将相邻元素配对，我们会得到和 3 和 7，这会立即失败。 但即使是像 (1,4) 和 (2,3) 这样的替代配对也能得到和 5 和 5，所以这种情况实际上是有效的。 这表明配对顺序对于朴素策略很重要，我们需要全局不变量而不是局部决策。 

## 方法

 暴力策略将尝试形成数组的所有可能的配对，并检查任何完整配对是否在所有配对中产生相等的总和。 这相当于探索完整图中的所有完美匹配$n$顶点，呈超指数增长。 即使进行修剪，配对的数量也约为$(n-1)!!$，即使对于$n = 20$。 

关键的观察是，如果存在这样的配对，则目标总和必须是唯一确定的。 假设我们对数组进行排序。 如果我们将最小的元素与最大的元素配对，那么就已经确定了所需的总和。 其他所有对都必须符合相同的总和，这迫使采用严格的结构：第二小的对与第二大的对，依此类推。 如果发生任何不匹配，则任何替代配对都无法挽救这种情况，因为更改任何配对都会破坏全局总和一致性。 

这减少了从搜索所有配对到在排序后验证单个确定性配对模式的问题。

 ## 算法演练

 我们独立处理每个测试用例。 

1.读取数组并按非降序排序。 排序是必要的，因为最佳配对结构取决于匹配的极值，而排序干净地暴露了这些极值。 
2. 使用第一个和最后一个元素计算候选目标和，具体来说$a[0] + a[n-1]$。 如果存在有效的解决方案，则强制使用此值，因为最小和最大元素必须在任何有效配置中配对； 否则，用任何其他配对替换它们只会减少范围并破坏一致性。 
3. 使用两个指针，一个从开头开始，一个在结尾，并向内移动。 在每一步中，从当前最小的剩余元素和最大的剩余元素形成一个概念对。 
4. 对于每个这样的对，验证它们的总和等于先前计算的目标总和。 如果任何配对出现偏差，请立即断定不存在有效配对。 
5. 如果所有对都与目标总和匹配，则存在有效分区。 

关键思想是，一旦最外面的元素被固定，剩余的结构就完全确定了。 如果不破坏一致性，就没有重新安排的灵活性。 

### 为什么它有效

 排序后，假设存在有效的解决方案。 让最小元素与除最大值之外的某个元素配对。 如果发生这种情况，最大值必须与小于或等于第二大值的值配对。 这将迫使第二对的总和超过或低于原始目标（具体取决于排序），这与所有对共享相同总和的要求相矛盾。 

这创建了一个严格的配对结构：排序后的数组必须从两端向内对称配对。 由于目标和是由极端元素固定的，所以每一对都是强制的，任何偏差都会使配置无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        a.sort()

        target = a[0] + a[-1]

        ok = True
        i, j = 0, n - 1

        while i < j:
            if a[i] + a[j] != target:
                ok = False
                break
            i += 1
            j -= 1

        out.append("SI" if ok else "NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先对数组进行排序，使最小值和最大值成为自然的第一个候选对。 目标总和从这些极端值开始立即固定，因为任何有效的解决方案都必须包括与该值一致的一些配对结构。 

两指针循环强制每个对称对匹配相同的和。 循环只运行$n/2$迭代，因此验证步骤是线性的。 唯一微妙的要求是必须在循环开始之前计算目标； 动态重新计算它是不正确的，因为它会导致不一致的局部调整。

 ## 边缘情况

 一种重要的边缘情况是所有值都相同。 对于输入：```
6
7 7 7 7 7 7
```排序给出相同的数组，目标是 14。每对总和为 14，因此算法接受。 指针扫描永远不会发现不匹配，从而确认简并均匀分布的正确性。 

另一种边缘情况涉及负数，如果不排序，最佳配对并不明显：```
4
-3 -1 2 4
```排序给出`[-3, -1, 2, 4]`，目标 = 1。这些对是 (-3,4) 和 (-1,2)，总和均为 1。即使简单的相邻配对会错误地表明失败，该算法也能正确识别可行性。 

第三种边缘情况是只有一对违反约束：```
6
0 1 2 3 4 10
```排序后的数组通向目标 10，但配对很快就显示出不匹配。 该算法立即拒绝而不探索替代配对，这对于大输入的效率至关重要。
