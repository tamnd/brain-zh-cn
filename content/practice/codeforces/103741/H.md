 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 n 的数字排列，但我们不知道它的顺序。 我们不是直接构建它，而是在位置之间给予约束。 每个约束的形式为“位置 x 处的值小于位置 y 处的值”。 

任务是计算有多少个长度为 n 的排列同时满足所有这些不等式，并返回对 998244353 取模的答案。 

重新解释这一点的一个有用方法是考虑将排列值 1 到 n 分配给位置 1 到 n。 每个约束都会比较两个位置并强制在此处分配的值之间进行排序。 因此，每个约束实际上都是一条有向边，表示一个位置必须接收比另一个位置更小的标签。 

关于约束的关键观察是每个位置最多作为 xi 出现一次。 这意味着每个节点最多有一个出局约束，因此由约束形成的有向图每个节点最多有一个出度。 因此，该结构是有向链和有向循环的集合，但循环是立即不可能的，因为它们在循环周围强制执行严格的不平等。 

因此，任何有效实例的行为都必须像有向链森林一样。 

约束尺度很重要：n 最大可达 2⋅10^6，因此任何 O(n log n) 或 O(n) 解决方案都是可以接受的，但任何涉及阶乘式枚举或子集上的图 DP 的情况都是不可能的。 我们必须将问题简化为可以在线性时间或接近线性时间处理的结构。 

当约束形成循环时，就会出现微妙的边缘情况。 例如：

 n = 3

 约束：1 < 2、2 < 3、3 < 1

 这迫使 p1 < p2 < p3 < p1，这是不可能的，因此答案必须为 0。忽略循环检测的简单计数方法会错误地计算排列。 

另一种边缘情况是根本没有限制。 那么每个排列都是有效的，所以答案是n!。 在这种退化情况下，任何解决方案都必须简化为阶乘。 

## 方法

 如果我们尝试直接解决问题，我们可能会考虑分配值 1 到 n 并检查每个排列的约束。 这给出了 n! 排列，检查每一个排列的成本为 O(m)，因此总复杂度为 O(n!⋅m)，即使 n 小到 20，这也是完全不可行的。 

一种更结构化的强力方法是将其视为拓扑排序问题：约束定义偏序，并且我们想要计算该顺序的线性扩展。 计算通用 DAG 的线性扩展是 #P-complete 的，因此我们不期望子集上的通用 DP 能够扩展。 

关键的简化来自于特殊的结构：每个位置最多有一个输出约束。 这意味着图分解为独立的链，并且不存在节点必须小于其他多个节点的分支约束。 这一限制极大地降低了线性扩展计数的复杂性。 

每个链在其节点之间强制执行严格的排序，但在不同的链之间，除了相对交错之外没有任何约束。 问题变成了计算我们可以在尊重内部顺序的同时交错这些链的方式有多少种。 

现在观察更强大的东西：由于每个节点至多有一个传出边缘，因此每个连接的组件要么是一条简单的有向路径，要么是一个循环。 循环使实例无效，因此我们只处理路径。 每条路径都已在内部固定，这意味着其上的所有节点必须根据路径方向严格按递增顺序出现。 

因此，每条路径的行为就像一个“块”，其内部排列是固定的，我们只选择如何在保留内部顺序的同时为这些块分配排列等级。

最终结果简化为对链长度进行多项式计数，从而简化为阶乘除以链大小的阶乘的乘积。 然而，因为每个节点都有自己的位置，并且约束仅比较位置，所以正确的简化最终会更加清晰：每个有效结构为每个拓扑排列贡献恰好 1 种方式，答案变成 n 的阶乘减去强制比较的修正，最终减少到计算阶乘 n 并除以独立链的大小。 

这导致使用 DSU 或简单遍历来计算链长度的 O(n) 解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!·m) | O(n!·m) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 构建一个有向图，其中每个位置 x 至多有一个到 y 的出边。 这直接对每个约束 p[x] < p[y] 进行编码。 该结构确保我们使用不相交的链而不是任意的图。 
2. 通过检查有向图中的环来检测无效性。 由于每个节点的出度最多为 1，因此如果我们跟踪来自任何节点的指针并在同一遍历中重新访问先前见过的节点，则存在循环。 如果找到循环，答案立即为零，因为它迫使矛盾的不平等。 
3. 对于每个没有入边的节点，开始沿着出边向前遍历，计算其链的长度。 每次遍历都会标记访问过的节点，因此不会对任何节点进行两次处理。 
4. 收集所有链条长度。 这些代表独立的段，其中内部顺序是固定的，但链之间的相对顺序是自由的。 
5. 计算有效排列的数量，即 n 的阶乘除以所有链长度的阶乘的乘积。 这种划分说明了这样一个事实：在每个链内，位置不再可以自由排列。 
6. 使用阶乘值的模逆返回模 998244353 的结果。 

关键的想法是，每个链都强制执行严格的内部排序，因此排列自由仅存在于我们如何在全球范围内交错链中。 由于所有元素都是不同的并且约束永远不会分支，因此这种交错可以干净地分解为多项式系数。 

### 为什么它有效

 每个链代表位置的完全有序子集，必须在任何有效排列中保留相对顺序。 一旦链被固定，构造一个有效的排列就相当于选择所有元素的全局排序，使得每个链都按排序顺序出现。 这正是计算一个偏序集的线性扩展，其连接组件是链。 由于组件不相互作用，因此扩展的总数是链大小的多项式系数，从而产生阶乘除法公式。 链间约束不会引入额外的限制，因此保留了独立性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, m = map(int, input().split())
    
    nxt = [-1] * (n + 1)
    indeg = [0] * (n + 1)
    
    for _ in range(m):
        x, y = map(int, input().split())
        if nxt[x] != -1 and nxt[x] != y:
            print(0)
            return
        nxt[x] = y
        indeg[y] += 1

    # cycle detection + chain length computation
    vis = [0] * (n + 1)
    chain_lens = []

    def dfs(start):
        cur = start
        length = 0
        path = []
        while cur != -1:
            if vis[cur] == 1:
                # already processed
                return 0
            if vis[cur] == 2:
                break
            vis[cur] = 1
            path.append(cur)
            nxt_node = nxt[cur]
            cur = nxt_node
            length += 1
        
        for v in path:
            vis[v] = 2
        
        return length

    for i in range(1, n + 1):
        if indeg[i] == 0 and vis[i] == 0:
            length = dfs(i)
            if length:
                chain_lens.append(length)

    # remaining unvisited nodes are cycles or isolated leftovers
    for i in range(1, n + 1):
        if vis[i] == 0:
            length = dfs(i)
            if length:
                chain_lens.append(length)

    # precompute factorial
    fact = [1] * (n + 1)
    for i in range(1, n + 1):
        fact[i] = fact[i - 1] * i % MOD

    inv_fact = [1] * (n + 1)
    inv_fact[n] = pow(fact[n], MOD - 2, MOD)
    for i in range(n, 0, -1):
        inv_fact[i - 1] = inv_fact[i] * i % MOD

    # check if any cycle implied (invalid traversal)
    # if we ever encountered incomplete chain coverage, treat as invalid
    if sum(chain_lens) != n:
        print(0)
        return

    ans = fact[n]
    for c in chain_lens:
        ans = ans * inv_fact[c] % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先将约束编码为函数图，因为每个节点最多有一个传出边。 这保证了链式结构并允许线性遍历而无需复杂的邻接表。 

DFS 式的遍历是迭代的，以避免 n 高达 2⋅10^6 时的递归深度问题。 它还标记三态系统中的节点，以检测循环并避免重新访问已处理的节点。 

阶乘和逆阶乘被预先计算至 n，以支持快速多项式计算。 最后的乘法/除法步骤实现了导出的组合公式。 

一个微妙的实现细节是确保每个节点都属于一个链或周期。 如果任何节点仍未被访问或遍历长度未覆盖所有节点，则该结构无效或循环，并且我们正确地返回零。

 ## 边缘情况

 由约束形成的循环是最关键的失败情况。 例如，具有约束 1 < 2 和 2 < 1 的输入 2 应立即返回零。 当遍历重新访问活动节点时，算法会检测到这一点，从而在功能图中标记一个循环。 

当位置完全没有约束时，就会出现孤立节点情况。 在这种情况下，它形成长度为 1 的链，并提供阶乘因子 1，除了全局阶乘缩放之外，总体答案保持不变。 

完全约束的链将答案减少到 1，因为只有一种方法可以使所有元素与约束一致地排序。 遍历将所有节点合并成一条链，多项式除法将阶乘 n 折叠为阶乘 n。
