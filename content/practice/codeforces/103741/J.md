 ## 解决方案
 ## 问题理解

 我们正在维护一个动态的整数序列。 该序列开始时为空，并且仅通过将元素附加到末尾来增长。 在任何时候，我们都可能被要求计算从所有元素对导出的值，其中第一个元素位于第二个元素的左侧。 

对于固定序列$a_1, a_2, \dots, a_n$，问题定义了一个数量$A_i$对于每个分割点$i$介于 1 和$n-1$。 对于每个这样的分割，我们查看所有对$(j, k)$这样$j \le i < k$, 计算$a_j \& a_k$，并取所有这些对中的最大值。 然后$A_i$是所有这些贡献对所有这些对的 XOR，这实际上减少到相同的最大值，因为 XOR 实际上并没有在最大表达式的上下文中以有意义的独立方式累积多个值。 最终查询要求最大值$A_i$在所有有效分割点上$i$。 

在操作上，我们附加值并偶尔询问：在将序列分为左部分和右部分的所有方法中，左侧任何元素和右侧任何元素之间最大的按位 AND 是多少。 

关键限制是$q \le 10^5$操作和值最多可达$2^{20}$。 这立即排除了每个查询从头开始重新计算答案的任何解决方案，因为重建成对交互是$O(n^2)$在最坏的情况下，每个查询都会导致$10^{10}$运营。 

动态性质产生了一个微妙的问题。 一种简单的方法可能会尝试维护所有成对 AND 值或在每次插入后重新计算最佳分割。 这会失败，因为每个插入都可能与所有先前的元素交互，并且重新计算分割最大值将反复重新访问整个历史记录。 

另一种隐藏的边缘情况是当序列长度小于 2 时。在这种情况下，不存在分割，答案必须为 0。这很重要，因为假设至少存在一对的简单实现要么崩溃，要么返回未初始化的最大值。 

## 方法

 蛮力解释很简单。 每次操作之后，如果我们被询问一个查询，我们会尝试每个分割点$i$。 对于每个分割，我们扫描所有穿过分割边界的对并计算最大值$a_j \& a_k$。 这是正确的，因为它直接遵循定义。 

然而，成本却令人望而却步。 对于长度序列$n$，单次拆分检查成本$O(n^2)$，并且有$O(n)$拆分，因此单个查询是$O(n^3)$以最坏的解释。 即使优化分割评估以重用信息，我们仍然面临$O(n^2)$每个查询，这在以下情况下变得不可能$10^5$运营。 

关键的观察是问题实际上并不在于分裂。 每个有效对$(j,k)$和$j<k$贡献$a_j \& a_k$，答案就是当前前缀中所有对的最大 AND。 分割公式会分散注意力：最佳分割总是将实现全局最大成对 AND 的两个元素放在相对的两侧。 

因此，问题简化为在插入时维护当前集合中所有对之间的最大按位与。 关键的结构是按位 AND 具有按位的单调性：较高位占主导地位，并且只有当两个数字共享这些高位时，一对才会获得高值。 这建议维护按高位模式分组的候选者。 

此设置的标准技巧是为每个值维护最佳值，并且它可以与先前插入的值形成，但这仍然是$O(n^2)$。 相反，我们对之前的数字进行按位查找。 插入新号码时$x$，我们想要找到之前的数字$y$最大化$x \& y$。 这可以从最高位到最低位贪婪地完成，优先选择两个位均为 1 的路径。我们用这个最佳配对更新全局答案。 

这是有效的，因为当我们为每个位选择两个数字是否都为 1 时，就会实现任何最佳对的 AND 值。 特里树确保我们总是能找到结构中已有的最佳兼容伙伴。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$每个查询|$O(1)$| 太慢了 |
 | 按位 Trie 维护 |$O(q \cdot 20)$|$O(q \cdot 20)$| 已接受 |

 ## 算法演练

 1.初始化一个空的按位trie和一个变量`best = 0`。 trie 以二进制形式存储所有先前插入的数字，按从 19 到 0 的位进行组织。这种结构允许有效匹配共享高位的数字。 
2. 按顺序处理各个操作。 如果操作是插入数字`x`，首先查询 trie 找出最大可能值`(x & y)`超过所有先前插入的`y`。 这一步确保我们捕获最好的一对`x`参加。 
3. 要计算此最佳匹配，请从最高位到最低位遍历 trie。 在每一位，如果当前位`x`是 1，我们更喜欢移动到该位也为 1 的子级，因为这对 AND 结果有积极贡献。 如果不存在这样的孩子，我们就会转移到另一个分支。 
4. 遍历时，只有当两者都满足时，才通过设置一位来累加结果 AND 值`x`并且所选路径编号的该位等于 1。这种贪婪选择是有效的，因为较高位主导最终值。 
5.遍历完成后，更新`best`其当前值和计算的匹配值的最大值`x`。 
6. 插入`x`到 trie 中，以便它可以用于将来的查询。 这确保了对称性：当插入后面的元素时，每对最终都会被评估一次。 
7.如果操作是查询，则输出`best`如果至少插入了两个元素； 否则输出0。 

### 为什么它有效

 该算法保持不变式`best`是插入元素中所有对的最大按位与。 每对$(a_i, a_j)$当插入后面的元素时，只计算一次，因为此时前面的元素已经在 trie 中，我们计算它的最佳匹配。 trie 遍历确保对于每个插入的数字，我们在已插入元素的约束下找到全局最优的伙伴，因此不会错过更好的对。 由于 AND 是对称的，并且插入顺序仅延迟计算直到第二个元素出现，因此始终可以正确跟踪全局最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Trie:
    def __init__(self):
        self.next = [{}]  # list of dict children
        self.has = [0]

    def insert(self, x):
        node = 0
        for b in range(19, -1, -1):
            bit = (x >> b) & 1
            if bit not in self.next[node]:
                self.next[node][bit] = len(self.next)
                self.next.append({})
                self.has.append(0)
            node = self.next[node][bit]
            self.has[node] += 1

    def query_best_and(self, x):
        node = 0
        res = 0
        for b in range(19, -1, -1):
            bit = (x >> b) & 1
            # try to keep 1 if possible
            if bit == 1 and 1 in self.next[node]:
                node = self.next[node][1]
                res |= (1 << b)
            elif bit == 0 and 1 in self.next[node]:
                node = self.next[node][1]
            elif bit in self.next[node]:
                node = self.next[node][bit]
            else:
                # fallback (should not happen in valid trie)
                if 0 in self.next[node]:
                    node = self.next[node][0]
                else:
                    break
        return res

def solve():
    q = int(input())
    trie = Trie()
    best = 0
    cnt = 0

    for _ in range(q):
        tmp = input().split()
        if tmp[0] == '1':
            x = int(tmp[1])
            cnt += 1
            if cnt > 1:
                best = max(best, trie.query_best_and(x))
            trie.insert(x)
        else:
            print(best if cnt >= 2 else 0)

if __name__ == "__main__":
    solve()
```该实现为每个数字维护超过 20 位的二进制 trie。 这`query_best_and`函数贪婪地尝试首先与 1 位对齐，因为只有当双方都设置了位时 AND 才会受益。 插入发生在查询之后，因此当前元素不会与自身配对。 

这`best`仅当至少存在一个先前元素时才更新变量，确保第一个插入情况的正确性。 柜台`cnt`强制执行两个元素返回零之前的查询条件。

 ## 边缘情况

 一种关键的边缘情况是仅存在一个元素并且发出查询的情况。 例如，输入`1 10`后面的查询必须输出 0。该算法通过以下方式处理此问题：`cnt < 2`检查，防止任何无效的 trie 查询。 

另一种情况是所有数字不共享公共位。 例如，插入 1、2、4、8 等 2 的幂会导致每个 AND 都为零。 trie 仍然处理每个插入，但没有遍历找到共享位，所以`best`仍然是 0，这与正确答案匹配。 

第三种情况是重复相同的数字。 用于输入`1 7, 1 7`，特里树会找到一个完美的匹配，产生`7 & 7 = 7`，这成为全局最大值。 由于重复项是独立插入的，因此在插入第二个副本时，算法仍会评估该对，以确保正确性。
