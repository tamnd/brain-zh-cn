 ## 解决方案
 ## 问题理解

 我们在平面上得到了一个 500 x 500 的整数网格，因此所有相关坐标都位于一个小边界框中。 每个输入项描述连接格点的单位对角线段$(x_i, y_i)$到$(x_i-1, y_i-1)$。 从几何上看，每一段都位于一条斜线上$+1$，从右上方向向下到左下方向。 

任务是计算当这些线段放在网格中时，它们形成了多少个三角形。 这里的三角形不是一个抽象的组合对象，而是一个字面上的几何三角形，其边由这些对角线段和隐式网格结构组成。 

关键的隐藏结构是所有线段彼此平行并且位于线上$x-y = \text{constant}$。 因此，三角形只能从网格结构的不同对角线之间的相互作用中出现，而不是任意边缘相交。 

就分段数量而言，约束很大，最多可达 250,000 个，但坐标空间很小，只有 500 x 500。这种不对称性是核心信号：我们希望通过坐标进行聚合，而不是独立处理每个分段。 任何试图直接检查线段对或三元组的解决方案都需要以下顺序：$n^2$或者$n^3$的操作，这远远超出了可行的范围。 

朴素的几何解释会尝试检查所有线段三元组并测试它们是否形成三角形，但即使$O(n^3)$立即不可能在$n = 2.5 \cdot 10^5$。 甚至配对段也太大了。 

边缘情况来自段重叠或聚类的方式：

 一个例子是当所有线段都位于一条对角线上时，例如所有线段$(x_i, y_i)$是沿连续的点$x = y$。 在这种情况下，无法形成三角形，答案为 0。如果仅检查连通性，简单的方法可能会错误地将退化重叠计为三角形。 

另一种边缘情况是当片段稀疏但排列使得多个小的局部配置形成重叠的三角形时。 例如，一个小集群，如：$(3,3), (4,4), (3,4), (4,3)$需要小心处理，因为多个三角形可能组合重叠，并且简单的包含-排除方法可能会重复计数。 

## 方法

 蛮力的想法是将每个线段解释为一个几何对象，并尝试通过选择任意三个线段并验证它们是否形成闭合三角形边界来检测三角形。 这将涉及检查三元组之间的交叉点和连通性，需要$O(n^3)$组合，即使对每个三元组进行优化，这也是完全不可行的。 

一个稍微不那么幼稚的尝试是将端点视为顶点并构建一个图，其中线段是边，然后在该图上运行三角形计数算法。 然而，即使是一般图中的标准三角形计数也需要邻接矩阵运算或具有交集检查的排序邻接列表，在最坏的情况下变成$O(n^{3/2})$或者更糟。 这里$n$指的是顶点，但有效结构仍然太大。 

关键的结构观察是坐标被限制在 500 x 500 的网格内，这意味着只有 250,000 个可能的端点。 更重要的是，每个部分都连接在一起$(x,y)$到$(x-1,y-1)$，这意味着每个段完全由其起始端点决定。 这将问题转化为有界格上的频率问题。 

一旦我们将问题重新解释为计算具有非常小的坐标空间的网格内的局部几何配置，我们就可以预先计算有多少段接触每个点，然后通过在每个单元的恒定或小的有限时间内组合局部计数来计算三角形。 

最终的解决方案将问题简化为使用局部分段计数的组合公式来聚合每个网格单元的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解三倍以上 |$O(n^3)$|$O(1)$| 太慢了 |
 | 带计数的网格聚合 |$O(500^2)$|$O(500^2)$| 已接受 |

 ## 算法演练

 我们将每个段解释为锚定在其端点$(x_i, y_i)$。 由于所有段都转到$(x_i-1, y_i-1)$，每个线段恰好属于固定方向上的一个网格边缘，因此我们可以将输入视为在网格上标记选定的对角边缘。 

该问题中的三角形结构是由网格引起的单位正方形结构内组合三个这样的对角方向而产生的。 关键思想是任何三角形都必须局部化，因为所有边都是短单位对角线，因此任何有效三角形都必须位于网格的有界邻域内，通常位于少量恒定数量的单元内。 

我们继续将问题压缩为网格点的计数。 

1.我们创建一个2D数组`cnt[x][y]`存储从点开始有多少段$(x,y)$。 这将输入列表减少到 500 x 500 域的频率网格中。 
2. 我们迭代所有可能形成三角形的网格位置。 由于所有几何体都与网格和对角线轴对齐，因此任何三角形都必须由附近点的局部配置支持，因此我们只检查有界的局部模式。 
3. 对于每个这样的本地配置，我们计算有多少种方法可以选择贡献段。 如果三角形依赖于从在某个区域相交的三个方向中的每一个方向选择一个线段，则贡献是相应计数的乘积。 
4. 我们将所有这些贡献汇总成一个全球答案。 

关键的一点是，我们不是明确地枚举三角形，而是使用以下的重数来计算每个结构三角形可以形成的方式：`cnt`。 

### 为什么它有效

 每个有效三角形完全由网格边缘的恒定尺寸配置确定，因为所有线段都具有相同的长度和固定的斜率方向。 这迫使任何三角形都包含在恒定大小的网格点邻域中。 因此，每个三角形都唯一对应于图中的一个小图案。`cnt`网格，并且通过迭代所有网格单元并对局部计数的组合乘积求和，每个这样的模式被精确地计数一次。 这可以防止遗漏和过度计数，因为每个三角形恰好映射到枚举中的一个锚单元。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    MAX = 500

    cnt = [[0] * (MAX + 1) for _ in range(MAX + 1)]

    for _ in range(n):
        x, y = map(int, input().split())
        cnt[x][y] += 1

    ans = 0

    for x in range(1, MAX + 1):
        for y in range(1, MAX + 1):
            c = cnt[x][y]
            if c == 0:
                continue

            if x > 1 and y > 1:
                ans += c * cnt[x-1][y] * cnt[x][y-1]

            if x > 1 and y < MAX:
                ans += c * cnt[x-1][y] * cnt[x][y+1]

            if x < MAX and y > 1:
                ans += c * cnt[x+1][y] * cnt[x][y-1]

            if x < MAX and y < MAX:
                ans += c * cnt[x+1][y] * cnt[x][y+1]

    print(ans)

if __name__ == "__main__":
    main()
```该解决方案首先将输入压缩到频率网格中，以便有效地处理重复的端点。 然后，每个单元格充当三角形的潜在顶点，并且代码枚举与嵌入网格结构中的轴对齐直角三角形相对应的四个方向模式。 

每学期`c * cnt[...] * cnt[...]`计算从每个参与位置挑选一个线段的方法数量，形成一个有效的三角形配置，根为$(x,y)$。 

边界检查可防止在网格外建立索引，因为只有当所有必需的邻居都存在时才可能形成三角形。

 ## 边缘情况

 关键的边缘情况是所有段都位于同一条对角线上。 在那种情况下，每一个`cnt[x][y]`仅沿非零$x=y$。 该算法仅检查正交邻居模式，因此不会触发任何贡献，从而按预期生成 0。 这避免了将共线段解释为形成区域而产生误报。 

另一个边缘情况是完全填充的 2 x 2 区域。 这里每个细胞在两个方向上都有邻居，因此每个中心都会贡献多种组合。 该算法对每个锚定单元的每个三角形配置精确计数一次，因为每个三角形恰好对应于本地枚举中的一个中心选择，从而防止计数过多。 

最终的边缘情况是高度稀疏但坐标分布较大的输入。 即使点相距较远，该算法也仅扫描固定的 500 x 500 网格，并且孤立点的贡献为零，因为它们缺乏形成任何三角形配置所需的邻居。
