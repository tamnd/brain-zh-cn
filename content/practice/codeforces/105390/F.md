 ## 解决方案
 ## 问题理解

 我们得到一棵树，其中每个节点都携带两个独立的信息：颜色（红色或蓝色）和正权重。 如果我们查看整棵树，我们可以计算一个称为其美丽的分数，定义为所有红色节点的总权重减去所有蓝色节点的总权重。 

只有当一棵树包含至少一个红色节点和至少一个蓝色节点时，它才被称为有效的或“完美的”，并且它的美丽是非负的。 如果我们拿一棵树开始删除边缘，我们就把它分成一片森林。 仅当森林的所有连接组件都不是完美的树时，森林才被认为是坏的，这意味着每个组件要么只有一种颜色，要么具有负面美感，或两者兼而有之。 

任务是删除尽可能少的边，以便删除后，生成的森林中没有完美的连接组件。 

n 最大为 8000 的约束意味着我们可以承受大约 n 平方或 n log 平方的方法，但是任何立方或枚举所有边子集都是不可能的。 由于该结构是一棵树，任何直接尝试所有边缘删除的解决方案都会呈指数爆炸，因为删除 k 个边缘已经创建了 2^k 种可能性。 

混合组件会产生一个微妙的问题：组件可能会以三种不同的方式变得无效。 它可能会变得单色，可能会具有负面美感，或者两者兼而有之。 仅检查全局美感或仅检查连接性的幼稚方法会失败，因为分裂既可以以非局部方式破坏又可以创建完美的组件。 

作为一个具体的失败案例，考虑一棵树，它的整个结构具有积极的美感，但有一个子树已经是完美的。 删除随机边缘可能会隔离该子树并保持其完美性，而另一次切割可能会通过以不同方式混合蓝色和红色分布来破坏它。 所以我们不能单独对待边缘。 

## 方法

 暴力的想法会尝试删除所有边缘子集，计算所有结果组件，并检查任何组件是否完美。 即使我们只尝试切割 k 个边，我们仍然需要枚举由这些切割引起的树的分区。 这很快就会变成 n 的指数，因为每个边缘决策都会改变全局的连接性。 

关键的观察是“完美性”是连接子树的一个属性，删除边只会分裂树。 因此，我们不必考虑任意森林，而是可以考虑切割边缘，以便每个剩余组件避免同时满足两个条件：具有两种颜色和非负总重量差异。 

这导致了标准的重新制定：我们希望确保最终森林中的每个连接组件要么变成单色，要么具有负美感。 由于切割边缘只会限制连接性，因此我们试图“摧毁”所有可能完美的子树。 

这提出了树DP的观点：对于每个子树，我们想要了解它在经过一些切割后是否可以作为有效组件存在，以及需要多少成本（切割次数）来确保它不完美。 红色和蓝色计数与总和符号之间的相互作用自然表明维持子树中的聚合贡献并决定是否切割边缘以破坏正配置。 

最佳解决方案的工作原理是对树进行生根并针对每个节点计算其子树是否可以形成或有助于形成完美的组件。 每当子树有可能形成完美的配置时，我们可能会被迫切断它与父树的连接，而目标就是最小化这种强制切断。 该问题简化为识别最大“危险”子树，否则其红蓝组合贡献将保持非负值，同时仍包含两种颜色。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解边缘子集 | 指数| O(n) | 太慢了 |
 | 对有效子树进行强制切割的树DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以节点 1 为树根，并使用后序 DFS 计算子树信息。 

1. 对于每个节点，计算其子树的两个聚合值：红色节点和蓝色节点之间的总权重差，以及子树是否包含至少一个红色节点和至少一个蓝色节点。 如果将子树视为独立组件，这两个值完全确定子树本身是否是完美树。 
2. 在 DFS 期间，假设我们最初保留所有边。 对于每个子树，我们检查以该子树为根的子树在附加到当前节点时是否可以自行形成完美的组件。 如果是，我们有一个选择：要么向上合并，要么切断。 
3. 如果将子子树合并到当前组件中可以保持颜色和非负整体美感，那么合并后的结构将是“危险的”，因为它成为完美的候选者。 为了防止这种情况，我们更喜欢剪掉子边缘。 每次这样的削减都会使答案增加一。 
4. 如果合并不能创建完美的结构，我们通过将子树的贡献添加到当前节点的聚合值来安全地向上合并子树。 
5. 处理完所有子树后，我们将当前子树的聚合状态返回到其父树，这代表了可实现的最佳配置，而无需在下面引入完美的组件。 
6. 最终答案是DFS期间强制切割的总数。 

### 为什么它有效

 核心不变量是，处理完一个节点后，我们返回到其父节点的子树保证不会包含仍然在内部连接的完美组件。 任何可以创建完美组件的配置都可以立即通过切割边缘来分离。 由于每次切割仅在子树有助于形成有效的完美结构时进行，因此我们绝不会进行不必要的切割，并且每个剩余的连接组件在构造上都是安全的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    s = input().strip()

    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    # color: 0 red, 1 blue
    color = [0 if c == '0' else 1 for c in s]

    # we maintain:
    # sumDiff = sum(red a) - sum(blue a)
    # hasRed, hasBlue
    sys.setrecursionlimit(10**7)

    ans = 0

    def dfs(u, p):
        nonlocal ans
        sum_diff = a[u] if color[u] == 0 else -a[u]
        has_red = (color[u] == 0)
        has_blue = (color[u] == 1)

        for v in g[u]:
            if v == p:
                continue
            c_diff, c_red, c_blue = dfs(v, u)

            new_diff = sum_diff + c_diff
            new_red = has_red or c_red
            new_blue = has_blue or c_blue

            # if merging would make a perfect component, cut
            if new_red and new_blue and new_diff >= 0:
                ans += 1
                continue

            sum_diff = new_diff
            has_red = new_red
            has_blue = new_blue

        return sum_diff, has_red, has_blue

    dfs(0, -1)
    print(ans)

if __name__ == "__main__":
    solve()
```该实现在树上执行单个 DFS。 每个节点在向上冒泡信息的同时维护三个值：有符号和贡献，以及红色和蓝色是否都出现在累积子树中。 关键决策点发生在将子子树与当前节点组合时。 如果合并状态已经满足完美树的条件，则删除该边而不是合并。 

微妙的一点是我们以后不会重新考虑切割子树。 这是有效的，因为一旦切割了一条边，该子树就会变得孤立，并且不会影响任何祖先组件。 

## 工作示例

 ### 示例 1

 考虑一棵小树，其中合并所有内容会产生一个完美的结构。 

| 节点| 处理过的孩子 | 总和差异 | 已红 | 有_蓝色 | 行动|
 | ---| ---| ---| ---| ---| ---|
 | 叶子1 | - | +2 | T | F | 开始 |
 | 叶子2 | 叶子1 | +2+(-3) | T | T | 保持|
 | 根 | 叶子2 | 有效合并| T | T | 根据标志剪切或保留|

 DFS 尝试向上合并子树，但是当合并状态变为双色且总和非负时，边缘将被切断。 这演示了该算法如何阻止局部而不是全局形成完美组件。 

### 示例 2

 需要多次切割的情况：

 | 步骤| 子树| 合并状态| 决定|
 | ---| ---| ---| ---|
 | 1 | 孩子A | 完美| 切|
 | 2 | 孩子B | 完美| 切|
 | 3 | 根 | 隔离组件| 完成 |

 这表明独立的危险子树是独立处理的，并且每个危险子树都强制剪切一次。

 ## 边缘情况

 一个关键的边缘情况是每个节点具有相同的颜色。 在这种情况下，没有任何组件可以满足“两种颜色都存在”的要求，因此答案必须为零。 DFS 永远不会触发削减，因为`has_red and has_blue`总是假的。 

当每个节点沿权重较大的链交替颜色时，会出现另一种边缘情况。 在这里，合并逐渐增加了满足颜色和非负和的机会，因此切割可能发生在多个级别。 每当合并状态变得完美时，算法就会正确隔离子树，确保没有隐藏的完美组件在结构的更深处存活下来。
