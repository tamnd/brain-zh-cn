 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$矩阵，其中值按行优先顺序填充。 行中的数字$i$， 柱子$j$简直就是$(i-1)\cdot n + j$，因此每一行都是一个连续的整数块，并且每一列以固定偏移量从每个块中选取一个元素。 

我们必须从每行中精确选择两个单元格，从每列中精确选择两个单元格，并且在所有此类有效选择中，我们希望所选值的总和最小。 

结构约束很严格：每一行恰好贡献两次选择，因此我们总共选择$2n$细胞。 列约束强制实现全局平衡：每列也必须恰好选择两次，这意味着不能逐行独立地进行选择。 

约束条件$n \le 1000$排除任何依赖于指数或基于流量的公式$n^3$或者更糟。 三次或二次方法仍然是可以接受的，但是任何涉及一般分配或具有大常数的最小成本流的方法都有太慢的风险。 这已经暗示该结构必须简化为直接公式或非常简单的组合结构。 

当贪婪地思考每行时，会出现微妙的边缘情况。 如果我们简单地选择每行中的两个最小的数字，我们总是选择每行中的前两列，这使得第 1 列和第 2 列被选中$n$次，违反了每列被恰好选择两次的要求。 因此，天真的贪婪方法完全打破了列约束，即使它是局部最优的。 

另一个陷阱是在选择最少的行对后尝试均匀分配列的使用。 这种结构过于僵化：一旦一行选择了两列，就会迫使这些列吸收部分全局配额。 在不显着增加成本的情况下，小型的局部修复无法解决全球失衡问题。 

## 方法

 A brute-force interpretation would be to choose, for each row, a pair of columns, and then check whether the resulting multiset of column counts is exactly 2 for every column. 有$\binom{n}{2}$每行的选择数，因此分配的总数是$\binom{n}{2}^n$，即使对于$n = 10$。 这显然是不可行的。 

关键的观察结果是矩阵是完全有序且可分离的：值仅取决于每行内的列索引，并且所有行在附加偏移量之前都是相同的$(i-1)n$。 这意味着在行中选择一列会带来固定的每列成本加上恒定的行移位。 由于每行恰好贡献两个元素，因此行移位部分对于任何有效的解决方案都是固定的，并且不会影响优化。 

因此，真正的问题归结为为每行选择两个列索引，使得每列恰好使用两次，同时最小化所有行中选择的列索引的总和。 行偏移贡献一个常数：$$\sum_{i=1}^n 2(i-1)n$$这与选择无关。 

现在我们只关心列索引。 每列必须在所有行中恰好出现两次，我们选择$2n$总列分配分为$n$每行对。 

这相当于分区$2n$将每列相同的标记放入每行两个大小的行中，但更重要的是，由于除了索引成本之外所有列都是对称的，因此我们希望尽可能频繁地使用较小的列。 但是，每列的容量固定为 2，因此可用的“槽位”总数正好是$2n$，匹配所需选择的数量。 这意味着每列必须在全局范围内使用两次，而列的多重性则没有自由度。 

所以列部分的成本也是固定的：$$2 \cdot (1 + 2 + \dots + n)$$剩下的唯一选择是如何在行中配对这些列用法。 由于配对不会影响成本，因此任何有效的配对都有效，并且最小总和完全由这些固定贡献决定。 

因此，答案纯粹是算术。

 ## 算法演练

 1.观察行中的每个值$i$可以写成$(i-1)n + j$，分隔行相关部分和列相关部分。 这种分解至关重要，因为它让我们能够隔离出真正可以改变的内容。 
2. 计算每行对总和的贡献次数。 每行恰好贡献两个选定元素，因此只要满足行约束，行贡献就固定。 
3. 将总和表示为行偏移量和所选列索引的组合。 行偏移总和是一个与选择无关的常数，因此可以直接计算它们。 
4. 分析列约束：整个矩阵中的每一列必须恰好选择两次。 由于有$n$列，这恰好迫使$2n$总列选择，与所选单元格的数量相匹配。 
5. 将列贡献计算为固定多重集总和，其中每个列索引恰好出现两次。 
6. 将行和列贡献组合成最终的闭合形式表达式。 

### 为什么它有效

 关键的不变量是两个维度都被约束完全饱和。 每行恰好贡献两次选择，固定总行贡献结构，而每列被迫恰好出现两次，完全固定列多重集。 由于两个自由度都受到完全约束，因此不存在可以改变总和的优化决策，因此每个有效配置都会产生相同的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    row_part = 0
    for i in range(1, n + 1):
        row_part += 2 * (i - 1) * n
    
    col_part = 2 * (n * (n + 1) // 2)
    
    print(row_part + col_part)

if __name__ == "__main__":
    solve()
```该代码直接实现将总和分解为行相关部分和列相关部分。 该循环计算来自的总贡献$(i-1)n$偏移量，乘以 2，因为每行贡献两个元素。 列贡献使用以下事实：每个列索引从 1 到$n$正好出现两次。 

一个常见的实现错误是忘记了两个组件中的因子 2：每行贡献两次选择，每列也全局使用两次。 另一个微妙之处是确保求和时使用整数除法$1 + \dots + n$。

 ## 边缘情况

 对于$n = 2$，矩阵为：$$\begin{bmatrix}
1 & 2 \\
3 & 4
\end{bmatrix}$$该算法将行部分计算为零，因为只有第二行有贡献$(i-1)n$, 给予$2 \cdot 1 \cdot 2 = 4$。 列部分是$2(1+2)=6$。 但是，由于每个有效选择必须使用所有列两次，因此该结构强制采用固定分布，并且计算出的总数在所有有效选择中保持不变。 

这证实，即使在配对灵活性最小的最小配置中，约束也已经完全确定了解空间，并且该公式不依赖于如何在行内选择配对。
