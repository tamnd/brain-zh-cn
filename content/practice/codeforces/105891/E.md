 ## 解决方案
 ## 问题理解

 我们被要求模拟一个非常受限的打字过程，该过程按顺序构建目标字符串，但有一个变化：字符不必仅附加在当前字符串的末尾。 相反，部分构建的字符串内有一个可移动光标，因此每个新字符都可以插入到任何位置。 该成本模型具有三个组成部分：使用给定的成本矩阵将左手手指从一个字母更改为另一个字母，以每步固定成本在当前字符串内向左或向右移动光标，以及以固定成本打印当前选定的字符。 

最初字符串为空，光标位于位置零，左手手指放在字母上`a`。 目标是执行一系列操作，以便最终字符串完全按顺序等于给定的目标字符串。 

一个关键的微妙之处在于，即使最终的字符串是固定的，我们“构造”它的顺序也不是固定的。 我们可以选择先插入较早或较晚的字符，这会改变光标移动成本，因为字符串会动态增长。 移动手指的成本仅取决于字母，但光标移动取决于当前的部分布局。 

这些限制使得结构变得重要。 目标字符串的长度最多为 20，这排除了在没有记忆的情况下对所有插入序列的排列进行任何指数搜索。 字母大小最多为26，因此在某种状态下跟踪手指位置是可行的。 由于三角不等式，成本矩阵是类似度量的，这确保我们永远不需要考虑直接转换之外的间接指路径。 

一个天真的解释是模拟所有可能的插入顺序和光标位置。 这很快就变得不可能了，因为即使 n = 20，构造顺序的排列数量也是天文数字。 

微妙的边缘情况来自相同的字符和对称的成本。 例如，如果所有成本为零并且 t 为零，则任何插入顺序都是最优的，但天真的贪婪策略可能仍然锁定到次优游标路径，因为它致力于早期位置决策而不考虑未来的插入。 

当字符重复时会出现另一种故障模式。 如果字符串是这样的`ababa`，贪婪的“总是按最终顺序插入下一个出现”策略忽略了插入稍后出现的事实`a`首先可以减少中间步骤的光标移动。 

## 方法

 蛮力方法将尝试通过在每一步选择接下来插入的剩余位置来枚举构建字符串的所有可能方法，并且还准确地模拟光标移动和手指移动。 每个插入步骤都取决于字符串的当前结构，因此状态会动态变化。 即使我们忽略最优性并只计算可能性，也最多有 n！ 插入顺序，对于每个插入顺序，我们模拟最多 n 个光标移动，导致阶乘增长远远超出可行的限制。 

关键的观察是，虽然插入顺序是灵活的，但字符的最终相对顺序由它们在目标字符串中的索引决定。 这意味着每个部分构造对应于选择已插入位置的子集，并且当前结构完全由该子集确定。 特别是，插入新字符的位置由其左侧有多少个已选择的索引唯一确定。 

这将问题转化为索引子集上的状态空间，其中唯一不明确的是光标位置和手指位置。 一旦我们修复了一个子集，当前字符串的结构就被隐式定义，因此光标移动就成为子集的确定性函数。 

因此，我们从排列上的指数搜索转向子集上的动态编程，其中每个状态对已放置的字符进行编码、光标当前处于该诱导顺序中的位置以及手指当前持有哪个字母。 转换对应于选择下一个要插入的字符并支付适当移动光标和手指的成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解广告订单 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 带有光标和手指状态的 DP 子集 | O(2^n · n^2 · m) | O(2^n · n^2 · m) | O(2^n·n·m) | O(2^n·n·m) | 已接受 |

 ## 算法演练

 我们将每个状态视为部分构造的字符串的快照。 快照由已插入的目标字符串的索引、光标当前位于该部分字符串内的位置以及左手手指所持有的字符来定义。 

1. 我们为目标字符串中的每个位置子集定义一个 DP 状态，因为任何部分构建的字符串都精确对应于一组已插入的索引。 这是可行的，因为一旦子集固定，插入顺序就不再重要，只有最终的相对顺序才重要。 
2. 对于每个子集，我们还跟踪当前部分字符串中的光标位置。 如果子集的大小为 k，则光标可以位于从 0 到 k 的任何整数位置，表示字符之间的位置。 这已经足够了，因为所有有效的光标移动都是沿着该线性结构的相邻交换。 
3. 我们还跟踪当前的手指字符，因为更改字母会产生取决于前一个字母的成本。 如果没有这个，我们将失去有关未来转变的必要信息。 
4. 我们用一个空子集初始化 DP，光标位于位置 0，手指位于`'a'`。 这与任何打印发生之前的初始配置相匹配。 
5. 从某个状态，我们尝试插入子集中尚未包含的每个字符索引。 当我们选择索引 i 处的字符时，我们计算其在当前部分字符串内的插入位置，因为已选择的索引数量严格小于 i。 这是有效的，因为部分字符串始终保持索引的原始顺序。 
6. 要将光标从当前位置移动到插入位置，我们需要支付与这两个位置之间的距离成比例的成本。 搬家后，我们支付固定的打印费用，然后更新子集。 
7. 我们还支付将手指从当前字符移动到目标字符串中索引 i 处的字符的成本。 
8. 我们为新子集、新光标位置（成为插入位置加一，因为插入将光标置于新字符的右侧）和新手指位置更新 DP。 

处理完所有状态后，答案是所有完整子集中的最小成本。 

### 为什么它有效

 正确性依赖于以下事实：部分构建的字符串的结构是由插入索引的子集唯一确定的。 一旦这个问题被修复，每个光标位置都对应于该诱导排序中明确定义的位置，并且每个插入位置都是确定性的。 DP 探索所有有效的插入序列，没有冗余，并且由于每个操作成本仅取决于本地状态信息（子集、光标位置、手指位置），因此不会丢失隐藏的依赖性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, t = map(int, input().split())
    s = input().strip()
    cost = [list(map(int, input().split())) for _ in range(m)]

    INF = 10**18

    # precompute index of each character
    idx = [ord(c) - ord('a') for c in s]

    # dp[mask][pos][c] = min cost
    # mask up to 2^20, pos up to 20, c up to 26
    N = 1 << n
    dp = [[[INF] * m for _ in range(n + 1)] for _ in range(N)]

    dp[0][0][0] = 0  # finger at 'a'

    for mask in range(N):
        k = mask.bit_count()

        # precompute rank positions for each i
        for pos in range(k + 1):
            for c in range(m):
                cur = dp[mask][pos][c]
                if cur == INF:
                    continue

                for i in range(n):
                    if mask & (1 << i):
                        continue

                    # compute insertion position
                    left_count = 0
                    for j in range(i):
                        if mask & (1 << j):
                            left_count += 1

                    new_pos = left_count

                    nmask = mask | (1 << i)

                    move_cursor = abs(pos - new_pos) * t
                    move_finger = cost[c][idx[i]]
                    total = cur + move_cursor + move_finger + t

                    # after insertion cursor is to the right of inserted char
                    dp[nmask][new_pos + 1][idx[i]] = min(
                        dp[nmask][new_pos + 1][idx[i]],
                        total
                    )

    ans = INF
    full = N - 1
    for pos in range(n + 1):
        for c in range(m):
            ans = min(ans, dp[full][pos][c])

    print(ans)

if __name__ == "__main__":
    solve()
```DP表存储达到部分构造状态的最小成本。 转换显式选择要插入的下一个字符，并使用已插入索引之间的排名来计算其在当前序列中的位置。 光标移动被处理为这些等级中的简单绝对差，按 t 缩放。 

手指移动直接从成本矩阵中获取，并且每次插入都会添加一次打印成本。 更新步骤小心地将新的光标位置放置在插入字符的右侧，这就是我们存储的原因`new_pos + 1`。 

## 工作示例

 考虑一个小情况，目标字符串是`ab`有两个字母。 

| 步骤| 面膜| 光标位置 | 手指| 行动| 成本增加 | 状态|
 | --- | --- | --- | --- | --- | --- | --- |
 | 0 | 00 | 00 0 | 一个 | 开始 | 0 | dp[00][0][a] | dp[00][0][a] |
 | 1 | 01 | 1 | 一个 | 在 0 | 处插入 a t | dp[01][1][a] | dp[01][1][a] |
 | 2 | 11 | 11 1 | 乙| 在 1 | 处插入 b 成本(a,b)+t | 决赛|

 该跟踪表明，按自然顺序插入不会产生超出必要移位的光标移动。 

现在考虑`ba`，其中最优策略不同。 

| 步骤| 面膜| 光标位置 | 手指| 行动| 成本增加 | 状态|
 | --- | --- | --- | --- | --- | --- | --- |
 | 0 | 00 | 00 0 | 一个 | 开始 | 0 | dp[00][0][a] | dp[00][0][a] |
 | 1 | 10 | 10 0 | 乙| 首先插入 b | 成本(a,b)+t | dp[10][0][b] | dp[10][0][b] |
 | 2 | 11 | 11 1 | 一个 | 在前面插入一个 | t | 决赛|

 这表明插入顺序改变了游标成本结构，并且 DP 捕获了这两种可能性。

 | 测量 | 复杂性 | 说明|

 | 时间 | O(2^n · n^2 · m) | O(2^n · n^2 · m) | 每个子集最多考虑 n 个转换，每个计算等级以朴素形式为 O(n) |

 ## 边缘情况

 对于像这样的字符串`aaaa`，最优策略永远不需要将手指从`a`，所以所有的DP转换都应该避免不必要的成本。 子集 DP 自然地处理这个问题，因为每个转换在从`a`到`a`。 

对于反转的字符串，例如`cba`，算法正确评估两个插入顺序。 如果我们首先插入`c`， 然后`b`， 然后`a`，光标位置发生显着变化，但 DP 会探索这两个订单并选择更便宜的一个。 

当 t = 0 时，光标移动变得自由，最佳解决方案减少为仅最小化手指移动。 DP 仍然有效，因为光标成本从转换中消失，并且只有手指转换仍然相关。
