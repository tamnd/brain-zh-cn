 ## 解决方案
 ## 问题理解

 我们有一排学生，每个学生都有固定的分数。 教师重复选择一个学生并检查该学生是否可以从该行中删除。 是否学生`i`消失仅取决于左右两侧学生的相对值`i`那一刻。 

如果学生周围的序列以特定方式“混合”，那么他们就会被移除。 具体来说，如果有证据表明较高和较低的分数以两种对称模式中的任何一种分布在左侧和右侧，则该学生就会消失。 一种模式要求左侧有一个不小于的值，右侧有一个不大于的值，另一种模式则交换这些角色。 

教师可以按任意顺序选择移除，移除后剩余的学生会轮班，因此会动态评估条件。 目标是最大化可以删除的学生数量，或者等效地最小化无论我们如何巧妙地选择删除顺序都永远无法删除的学生数量。 

约束允许最多`2 × 10^5`学生们，因此任何通过扫描数组来重复尝试所有删除或重新计算条件的解决方案都会立即变得太慢。 对去除量进行二次甚至接近二次的动态模拟是行不通的。 我们需要表征哪些元素在所有有效的删除顺序下本质上是“稳定的”。 

当数组是单调的时，会出现微妙的边缘情况。 在严格递增的数组中，例如`1 2 3 4 5`，每个中间元素在左侧都有较小的元素，在右侧都有较大的元素，因此其中许多元素都是可移动的。 但是，端点的行为有所不同，因为一侧是空的，这会阻止满足条件。 任何正确的解决方案都必须正确处理边界位置。 

另一个棘手的情况是当值重复时，例如`3 1 3 2 3`。 仅进行本地比较是不够的； 重要的是在一个位置的相对两侧是否至少存在一个严格较大的元素和一个严格较小的元素。 

## 方法

 直接模拟将重复扫描阵列，找到所有当前可移动的学生，删除一个，然后重复，直到无法再删除为止。 学生的每张支票都需要扫描两面，成本`O(n)`，并且这种情况有可能发生`O(n)`次，导致`O(n^2)`或者更糟。 和`n`最多`2 × 10^5`，这远远超出了可行的限度。 

关键的观察结果是，去除条件不取决于邻接或局部结构，而仅取决于每一侧是否存在某些不平等。 一旦我们根据前缀和后缀信息重新表述所有内容，我们就可以确定某个位置是否可以删除，而无需模拟删除。 

只有当学生的值以分裂的方式“夹在中间”时，学生才变得可移动：一侧的值至少一样大，另一侧的值至多一样大（或对称情况）。 这意味着可移除性取决于数组是否包含分布在相对于位置的较大值和较小值`i`。 

永远无法消除的学生正是那些在至少一种方向意义上保持极值的学生：前缀最大、前缀最小、后缀最大或后缀最小。 任何不受这些极值属性中的至少一个保护的位置最终都可以通过选择合适的删除顺序来暴露和删除。 

因此，我们不是模拟删除，而是预先计算四个集合：前缀最大值、前缀最小值、后缀最大值和后缀最小值。 至少属于这些集合之一的任何索引都是安全的，并将保留在最终配置中。 答案是他们工会的规模。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解模拟删除| O(n²) | O(n) | 太慢了 |
 | 前缀/后缀极值分类 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 从左到右扫描时计算前缀最大值，标记迄今为止看到的最大值的每个位置。 这捕获了无法从左侧控制的元素。 
2. 在同一扫描中计算前缀最小值，标记迄今为止看到的最小的位置。 这些元素不能从左侧“向下超越”。 
3. 从右向左扫描时计算后缀最大值，标记后缀中最大值的位置。 这些不能由右翼主导。 
4. 从右到左类似地计算后缀最小值。 
5. 标记出现在四个类别中任意一个的每个索引。 最终的答案是标记索引的数量。 

每个类别都很重要的原因是，去除需要在冲突的方向上同时存在较大和较小的值。 如果某个位置是前缀最大值，则左侧没有任何内容更大，这会阻止触发可移除性的两种方式之一。 同样的想法对称地适用于其他三种情况。 

### 为什么它有效

 可移除性条件要求同时见证两个值不小于`a[i]`一侧且值不大于`a[i]`在另一侧，采用某种分体式配置。 如果一个位置在四个方向意义上的任何一个上都是极端的，那么至少有一侧无法提供“夹住”它所需的不平等结构。 因此，这样的位置永远不会被任何删除序列完全暴露。 相反，任何不受这些极端约束保护的位置都可以通过首先消除阻挡元件来移除，这最终形成所需的分体配置。 这将问题简化为识别至少在一个方向上处于极值的所有指数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

pref_max = [False] * n
pref_min = [False] * n
suf_max = [False] * n
suf_min = [False] * n

cur = -10**18
for i in range(n):
    if a[i] >= cur:
        pref_max[i] = True
        cur = a[i]

cur = 10**18
for i in range(n):
    if a[i] <= cur:
        pref_min[i] = True
        cur = a[i]

cur = -10**18
for i in range(n - 1, -1, -1):
    if a[i] >= cur:
        suf_max[i] = True
        cur = a[i]

cur = 10**18
for i in range(n - 1, -1, -1):
    if a[i] <= cur:
        suf_min[i] = True
        cur = a[i]

ans = 0
for i in range(n):
    if pref_max[i] or pref_min[i] or suf_max[i] or suf_min[i]:
        ans += 1

print(ans)
```该代码计算阵列上的四次线性扫描。 每次扫描都会保持迄今为止在相应方向上看到的最佳值，并标记达到新极值的位置。 

最后一个循环仅计算属于至少一个极值类别的索引。 这避免了任何删除模拟，一旦结构表征已知，这种模拟就会太慢而且也是不必要的。

 ## 边缘情况

 对于像这样的单元素数组`10`，所有四次扫描立即将唯一位置标记为各个方向上的极值。 没有左侧或右侧，因此算法正确地将其计为幸存者。 

对于像这样的严格递增序列`1 2 3 4 5`，每个位置都是一个前缀最大值。 在前缀扫描期间，每个元素都会成为新的运行最大值，因此所有位置都会被标记，从而产生完整的幸存者集。 

对于具有相等值的序列，例如`3 3 3 3`，每个元素在不同时间同时是前缀最大值和前缀最小值，具体取决于相等处理。 扫描标记所有位置，最终的并集正确包含每个索引。
