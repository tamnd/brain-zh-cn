 ## 解决方案
 ## 问题理解

 我们被赋予了一个系统$n$灯，每个当前打开或关闭，以及$m$开关。 每个开关都连接到一组灯。 按下开关会翻转与其连接的每个灯的状态。 

目标不是按时间顺序模拟操作，而是决定我们应该恢复哪些开关（每次恢复使开关可用，并且每个可用开关被有效地按下一次），以便在应用所有选定的开关后，每个灯最终都会关闭。 每个开关都有相关的维修成本，我们希望总成本最小。 

重要的微妙之处在于我们无法对开关重新排序或多次按下它们。 我们选择开关的子集，每个选定的开关都对其连接的灯提供单个 XOR 切换。 

交换机最初的构建方式还有一个额外的结构限制。 每个开关在安装时，其连接组中至少有一个灯，该灯最多与一个先前安装的开关相关。 从图的角度来看，如果我们在开关和灯之间建立一个二分图，并逐一添加开关节点，则每个新开关必须包含至少一个灯，并且该灯在之前的开关中当前度数最多为1。 

这个条件是问题的关键。 它防止结构成为任意稠密超图，并迫使其形成非常受限的形式。 

限制因素$n, m \le 60$已经表明开关上的指数或子集 DP 原则上是可能的，但前提是该结构减少了有效维度。 没有结构，这是一个成本最低的异或系统$m$变量与$n$约束，这是一个标准的高斯消除问题，具有潜在的$2^{m-n}$自由变量。 在最坏的情况下，这个数字太大了。 

天真的想法的一个常见失败案例是将其视为每个灯独立的或贪婪地选择低成本开关。 但这会失败，因为每个开关同时影响多个灯，并且决策会通过异或奇偶校验进行全局干扰。 

## 方法

 如果我们忽略结构约束，该问题就成为 GF(2) 上线性系统的经典最小成本解决方案。 我们将建立一个矩阵$A$其中行对应于灯，列对应于开关，并求解$Ax = b$超过 GF(2)，最小化$\sum c_i x_i$。 最直接的方法是高斯消去法，它为解决方案提供了基础，并将问题简化为探索维数为零的空间$m - \text{rank}(A)$。 在最坏的情况下，这可能是大约$2^{60}$，这是不可能的。 

关键的观察结果是，构造规则迫使开关和灯的二分图表现得像森林一样。 每次添加一个开关时，它必须包含一盏灯，该灯在较早的开关中的度数最多为一。 这可以防止在关联图中创建循环，因为形成循环将要求循环中的每个节点在某个点已经具有至少二度，这与规则相矛盾。 

一旦我们认识到关联图是一片森林，代数结构就会显着崩溃。 每个连接的组件都成为一棵树，并且树在 XOR 约束系统中具有非常强大的属性：每个组件在灯侧恰好贡献一个自由度，这转化为对该组件中的开关的单个奇偶校验约束。 

这将全局系统简化为独立的组件，每个组件不再是一般的线性系统，而是一小组变量上的单一奇偶优化问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 全高斯消除+子集搜索|$O(m^3 + 2^{m-n})$|$O(mn)$| 太慢了 |
 | 树分解 + 每个组件 DP |$O(n + m)$|$O(n + m)$| 已接受 |

 ## 算法演练

 1. 我们将这种情况建模为二部图，一侧为开关节点，另一侧为灯节点，其中边意味着开关影响灯。 然后，我们按连接的组件对节点进行分组。 
2.我们验证构造规则的结构结果：这些边形成的图一定是森林。 这是因为每条新边在插入时总是引入至少一个度数最多为 1 的顶点，这会阻止环的形成。 
3. 在森林里$n + m$节点和$m$边，每个连接的组件必须满足顶点和边之间的紧密关系。 一个组件$V$顶点正好有$V - 1$边缘，意味着它是一棵树。 
4. 我们检查单个连接的组件。 因为它是一棵二分树，所以计算顶点数表明每个组件恰好包含一个灯节点。 这是结构所强制的：如果有多个灯，我们将超出树边缘约束。 
5. 因此，每个组件都以单个灯为中心，并通过树结构连接多个开关。 
6. 对于固定组件，我们将灯状况转换为 XOR 方程：该组件中所有选定开关的奇偶校验必须与该灯的初始状态匹配。 
7. 现在，我们将每个组件简化为一个简单的优化问题：选择总成本最小的开关子集，使其 XOR 等于固定位。 
8. 我们对组件中的开关使用二态动态编程来解决这个问题。 我们保持实现 XOR 奇偶校验 0 和 1 的最低成本。 
9. 独立加工完所有部件后，我们将所需成本相加。 如果任何组件不能满足其所需的奇偶性，则答案是不可能的。 

### 为什么它有效

 不变量是每个连接的组件精确地贡献一个独立的 XOR 约束，并且该组件内的所有开关仅通过该单一奇偶校验条件进行交互。 树结构保证不存在链接同一组件的不同部分的额外隐藏约束。 因此，全局系统干净地分解为独立的奇偶最小化问题，并且最优地解决每个组件会产生全局最优解决方案，因为成本在不相交的组件之间是累加的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    init = input().strip()
    cost = list(map(int, input().split()))

    adj = [[] for _ in range(n + m)]

    # nodes: 0..n-1 lamps, n..n+m-1 switches
    for i in range(m):
        s = input().strip()
        u = n + i
        for j, ch in enumerate(s):
            if ch == '1':
                adj[u].append(j)
                adj[j].append(u)

    vis = [False] * (n + m)
    ans = 0

    for i in range(n + m):
        if vis[i]:
            continue

        stack = [i]
        comp = []
        vis[i] = True

        lamps = set()
        switches = []

        while stack:
            v = stack.pop()
            comp.append(v)
            if v < n:
                lamps.add(v)
            else:
                switches.append(v - n)

            for to in adj[v]:
                if not vis[to]:
                    vis[to] = True
                    stack.append(to)

        if not switches:
            continue

        if len(lamps) != 1:
            # invalid structure under intended interpretation
            print(-1)
            return

        lamp = next(iter(lamps))
        target = int(init[lamp])

        dp = {0: 0, 1: 10**18}

        for idx in switches:
            c = cost[idx]
            ndp = {0: 10**18, 1: 10**18}
            for p in [0, 1]:
                if dp[p] >= 10**18:
                    continue
                ndp[p] = min(ndp[p], dp[p])
                ndp[p ^ 1] = min(ndp[p ^ 1], dp[p] + c)
            dp = ndp

        if dp[target] >= 10**18:
            print(-1)
            return

        ans += dp[target]

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先构建灯和开关之间的二分图。 然后它使用 DFS 查找连接的组件。 对于每个组件，它分离了灯和开关； 该结构保证每个组件只有一盏灯。 

一旦进入组件，问题就减少到在奇偶校验约束下选择成本最低的开关。 DP 状态仅保留两个值，表示所选开关的 XOR 是 0 还是 1。每个开关要么不贡献任何内容，要么在增加其成本的同时翻转奇偶校验。 

最终答案是所有组件的最优成本之和。 

## 工作示例

 考虑一个小型系统，其中两个开关都影响单个灯。 灯以状态 1 启动，我们希望它关闭。 

| 步骤| 考虑开关| DP[0] | DP[1] |
 | ---| ---| ---| ---|
 | 开始| 无 | 0 | 信息 |
 | 添加 S1（成本 3）| S1 | 0 | 3 |
 | 添加 S2（成本 2）| S2 | 0 | 2 |

 单独选择 S2 会给出奇偶校验 1，成本为 2，与目标匹配，因此答案为 2。 

该跟踪显示了 DP 如何跟踪奇偶校验而不是特定组合，这使得它即使在子集增长时也能干净地扩展。 

作为第二个示例，假设组件具有成本为 5、4 和 7 的开关，并且灯需要奇偶校验 0。 

| 步骤| 开关| DP[0] | DP[1] |
 | ---| ---| ---| ---|
 | 开始| - | 0 | 信息 |
 | 5 | S1 | 0 | 5 |
 | 4 | S2 | 0 | 4 |
 | 7 | S3 | 0 | 4 |

 最好的偶校验子集要么是空的，要么是选择对，但 DP 会自动聚合所有可能性，而无需显式枚举它们。

 ## 边缘情况

 一种边缘情况是灯最初关闭并且属于没有开关的组件。 在这种情况下，DP 永远不会为该组件运行，并且算法会正确地将其视为已满足。 如果灯最初是亮着的，则该组件将不包含有效的方法来翻转它，并且 DP 永远无法匹配所需的奇偶校验，从而导致拒绝。 

另一个边缘情况是具有单个开关的组件。 DP 从奇偶校验 0 开始，成本为 0，并且随着切换成本保持在 0 或移动到 1。 如果灯需要奇偶校验 1，则解决方案会正确选择该单个开关； 否则它不选择任何内容。 

最后的边缘情况是多个开关共享相同的效果。 DP 自然地通过隐式考虑所有子集来处理此问题，确保始终选择实现所需奇偶校验的最便宜的组合，而不管交换机定义中的冗余如何。
