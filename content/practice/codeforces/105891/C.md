 ## 解决方案
 ## 问题理解

 我们有两个非常大的正整数，`a`和`b`，对于每个测试用例，我们必须确定是否存在另一个正整数`x`（以$10^{18}$）同时满足两个条件。 

第一个条件力`x`不与任何素因数共享`a`， 意义`x`必须互质于`a`。 第二个条件需要`x`至少共享一个素因数`b`，表示最大公约数`x`和`b`严格大于一。 如果这样一个`x`存在，我们可以输出任何有效的。 

约束足够大，我们无法尝试任何形式的枚举`x`。 当有多达$10^4$测试用例和值达到$10^{18}$。 任何可行的解决方案都必须将问题简化为推理`a`和`b`而不是操纵候选值`x`。 

出现微妙的失败情况时`b`没有缺少的质因数`a`。 例如，如果`a = 6`和`b = 3`, 每个数字都可以被`b`能被 3 整除，但因为 3 能整除`a`，任何这样的`x`将违反互质条件。 仅检查整除关系而不考虑共享素数结构的幼稚方法可能会错误地假设存在解决方案。 

另一个边缘情况是当`b = 1`。 在这种情况下，`gcd(b, x)`始终为 1，因此第二个条件永远无法满足，使得无论如何，答案始终不可能`a`。 

## 方法

 强力解释将尝试构建候选者`x`并直接测试两个gcd条件。 对于每个测试用例，人们可能会尝试迭代可能的除数`b`，甚至通过素数的倍数除法`b`，检查他们是否避免了所有素因数`a`。 这在概念上是正确的，因为任何有效的`x`必须至少包含一个素因数`b`，但这种方法在输入规模下崩溃了。 当值达到以下值时，即使重复分解数字或测试许多候选者也是不可行的$10^{18}$。 

关键的观察是，唯一重要的是素因子集`a`和`b`。 满足`gcd(a, x) = 1`，我们必须确保没有任何素数可以相除`x`出现在`a`。 满足`gcd(b, x) > 1`，我们至少需要一个素数除法`b`也分`x`。 只有存在至少一个质因数，这两个约束才能满足`b`不分裂`a`。 

一旦这个想法到位，构建就变得简单了。 我们考虑因素`b`分解为不同的素数因子，并且对于每个这样的素数`p`，我们检查是否`p`划分`a`。 如果存在这样的`p`不分裂`a`，我们可以简单地选择`x = p`。 这自动满足两个条件：它与`b`并且与`a`。 

如果每个素因数`b`也分`a`，那么任意数与`b`必然与以下因素共享一个因素`a`，使得条件不相容。

 | ---| ---| ---| ---|
 | 蛮力 |$O(b)$每次测试或更差 |$O(1)$| 太慢了 |
 | 最佳 |$O(\sqrt{b})$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1.如果`b = 1`,立即输出`-1`。 这是因为没有整数`x`可以使`gcd(b, x)`大于 1 时`b`没有素因数。 
2. 因式分解`b`分解为其不同的素因数。 我们通过试分割来做到这一点$\sqrt{b}$，提取每个质因数一次并跳过重复项。 
3. 对于每个素因数`p`的`b`，检查是否`a % p != 0`。 如果这成立，我们就找到了一个属于`b`但不`a`，所以我们可以安全地选择`x = p`并立即输出。 
4. 如果我们扫描完所有素因数`b`每个人都划分`a`，那么不存在有效的构造，我们输出`-1`。 

选择背后的理由`x = p`是它是保证的最小可能结构`gcd(b, x) > 1`。 我们不需要构建复合值，因为添加更多因素只会增加与`a`。 

### 为什么它有效

 该算法依赖于一个简单的结构不变量：每个有效的`x`必须包含至少一个素因数`b`，并且必须避免所有素因数`a`。 因此，可行性仅取决于素因子集是否`b`不是素因数的子集`a`。 如果存在素数`b`缺席于`a`，单独选择该素数即可满足这两个约束。 如果不存在这样的素数，则任何与以下素数共享因子的数`b`自动与以下人员共享一个因素`a`，使条件互斥。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(a, b):
    if b == 1:
        return -1

    x = b
    i = 2
    found = False

    # try to extract prime factors of b
    while i * i <= x:
        if x % i == 0:
            if a % i != 0:
                return i
            while x % i == 0:
                x //= i
        i += 1

    if x > 1:
        if a % x != 0:
            return x

    return -1

def main():
    t = int(input())
    out = []
    for _ in range(t):
        a, b = map(int, input().split())
        out.append(str(solve_case(a, b)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```实现直接遵循因式分解逻辑。 我们减少`b`在扫描除数时，确保每个素因数被恰好考虑一次。 支票`a % i != 0`编码所选择的素数不得出现在`a`。 最后剩下的`x > 1`处理以下情况`b`本身就是素数。 

一个常见的陷阱是忘记在除掉小因素后，剩下的`x`是 1 或素数。 这对于正确性至关重要`b`包含超出平方根范围的大质因数。

 ## 边缘情况

 一个重要的边缘情况是当`b = 1`。 自从`b`没有质因数，因此无法构造任何质因数`x`和`gcd(b, x) > 1`。 该算法在分解开始之前立即拒绝这种情况。 

另一种情况是当`b`是素数。 该算法要么返回`b`本身，如果它不分裂`a`， 或者`-1`如果是的话。 这直接对应于素数的单因子结构，其中没有替代候选。 

最后一种情况是当`b`与共享其所有主要因素`a`，即使两个数字都很大并且是合数。 因式分解循环确保检测到每个此类共享因子，并且仅当至少一个因子逃脱此交集时，算法才会成功。
