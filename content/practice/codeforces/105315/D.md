 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，都有一个字符串列表，我们希望对不同索引的有序对进行计数，这样当我们将第一个字符串与第二个字符串连接时，得到的字符串向前和向后读取相同的值。 

输出不是关于构造回文或修改字符串。 它纯粹是计算有多少个不同单词的有序对在连接后形成回文。 

这些限制立即使我们远离总弦长度的二次方。 在所有测试用例中，字符串长度的总和以 4 · 10^5 为界，因此任何重复检查每对字符串字符的解决方案都将失败。 简单的 O(n²·L) 方法（其中 L 是平均字符串长度）在最坏的情况下会退化为大约 10^10 个字符比较，这远远超出了限制。 

当字符串可以在不同的角色中重用时，会出现微妙的边缘情况，因为字符串对是有序的并且索引必须不同。 例如，如果所有字符串均为空，则每个有序的不同索引对都是有效的，因为“”+“”仍然是回文。 基于字符串值而不是索引的粗心重复数据删除会导致计数不足。 

另一种边缘情况涉及单独已经是回文的字符串。 一个天真的想法可能是计算回文字符串并任意组合它们，但这忽略了两个字符串之间的边界很重要的结构约束。 例如，“ab”和“ba”形成有效对，但都不是单独的回文。 

## 方法

 暴力解决方案将尝试每个有序对 (i, j)，连接 si 和 sj，并检查结果是否是回文。 正确性很简单，因为它直接验证定义。 然而，每次回文检查的成本为 O(|si| + |sj|)，并且有 O(n²) 对。 在最坏的情况下，这会变成 O(n² · L)，当 n 达到 10^5 时，即使字符串平均很短，这也太慢了。 

关键的观察是我们不需要构建串联。 当 sj 等于 si 的倒数时，连接的字符串 si + sj 就是回文，但有一个更细微的约束：边界必须对齐，以便反转整个连接会交换两个部分。 这意味着 sj 必须是 si 的反转，并且除了反转字符串的精确匹配之外不需要其他结构。 

这将问题简化为计算每个字符串出现的次数，并将其与其相反的出现次数进行匹配。 唯一的复杂之处是避免将字符串与自身配对，除非它在不同的索引中多次出现。 因此，对于每个不同的字符串x，我们将其频率与reverse(x)的频率相匹配，并仔细确保我们不会重复计算有序对。 

因此，我们可以构建一个频率图，并且对于每个字符串 x，查找其反转形式 rx。 贡献取决于 x 是否等于 rx。 

如果 x != rx，则每次出现的 x 都可以与每次出现的 rx 配对，从而形成 freq[x] · freq[rx] 有序对。 当仔细迭代时，这会计算两个方向，因此我们需要确保只计算每个无序对一次或显式控制排序。 

如果 x == rx，则字符串本身就是回文，并且任何不同出现的有序对都会提供有效的串联，给出 freq[x] · (freq[x] - 1)。

 ## 算法演练

1. 读取测试用例中的所有字符串，并构建一个以字符串本身为键的频率字典。 这压缩了重复的工作，因为只有不同的字符串值对配对逻辑很重要。 
2. 对于每个不同的字符串 x，计算其反转版本 rx。 这是唯一重要的转换，因为跨串联的回文结构会迫使边界对称。 
3. 如果 x 在字典顺序上或结构上按迭代顺序“小于”rx，则仅计算对 (x, rx) 的贡献一次。 贡献是 freq[x] · freq[rx]，因为每次出现的 x 都可以与每次出现的 rx 配对。 
4. 如果 x 等于 rx，意味着字符串本身就是回文，则将 freq[x] · (freq[x] - 1) 添加到答案中。 这会计算同一回文值存储桶中所有有序的不同索引对。 
5. 通过强制执行一致的排序规则（例如仅在字典顺序中的 x <= rx 时进行处理），确保反向对不会重复计算。 

### 为什么它有效

 每个有效的串联 si + sj 必须围绕其中心完美镜像。 该约束迫使 sj 与 si 相反，因为回文数的后半部分反转为前半部分。 因此，问题简化为将每个字符串与其相反的对应字符串进行匹配。 对相同的字符串进行分组可确保我们正确计算所有索引级对，并且排序规则可防止对称对之间的重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        arr = input().split()

        freq = {}
        for s in arr:
            freq[s] = freq.get(s, 0) + 1

        ans = 0
        used = set()

        for s in freq:
            rs = s[::-1]

            if s in used:
                continue

            if s == rs:
                c = freq[s]
                ans += c * (c - 1)

            else:
                c1 = freq[s]
                c2 = freq.get(rs, 0)
                ans += c1 * c2

            used.add(s)
            used.add(rs)

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将输入压缩为频率图，以便重复的字符串不会导致重复的工作。 反转操作仅应用于唯一字符串级别，这使复杂性与总输入大小保持线性关系。 

这`used`set 可以防止字符串与其反向字符串之间的重复计算。 一旦处理了一对（s，反向（s）），两者都会被标记，以确保以后不会重新访问对称性。 

特殊情况`s == rs`通过计算同一组中不同出现次数的所有有序对来正确处理回文字符串。 

一个常见的实现陷阱是忘记顺序很重要：(i, j) 和 (j, i) 是不同的对，除非分别考虑两个方向。 公式`c * (c - 1)`已经包括两个方向。

 ## 边缘情况

 对于单字符重复字符串，例如`["a", "a", "a"]`，算法进入自回文分支，因为“a”等于其倒数。 频率为 3，因此贡献变为 3 · 2 = 6。这完全对应于所有有序索引对 (i, j)，i ≠ j。 

对于混合反向链，例如`["ab", "bc", "ba", "cb"]`，每个字符串仅与其相反的字符串配对。 该算法确保每个对组都被处理一次，因为`used`设置，防止重复计数，同时仍然正确累积所有交叉对组合。
