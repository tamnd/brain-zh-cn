 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 每个测试用例由几对整数组成。 对于每对$(a, b)$，我们计算一个定义为的值$a^b$。 该问题并不要求正常数字形式的值本身； 相反，它考虑无限小数展开$a^b$并关注小数点后的数字，将这种分数展开视为无限序列。 

如果两对对应值的无限小数部分被认为是等效的$a^b$完全相同。 任务是统计有多少个无序索引对$(i, j)$， 和$i < j$，产生相同的无限小数数字序列。 

短语“仅当等于无限多个数字时才等于”消除了有限精度引起的歧义。 我们通过完整的十进制扩展来比较两个实数，这意味着即使任何数字上的微小差异，无论多远，都会使它们不同。 

约束条件很大：最多$2 \cdot 10^6$总共对。 这立即排除了任何在每次比较时重新计算或模拟数值的方法。 甚至$O(n^2)$每个测试用例都是不可能的，甚至$O(n \log n)$必须避免大量的每个元素计算，例如大幂运算或任意精度的小数模拟。 

一个天真的解释会尝试计算$a^b$作为浮点或高精度数字并比较小数部分。 这在两个方面是失败的。 首先，用精确的分数展开式计算大幂是不可行的。 其次，浮点运算完全打破了无限展开的等式。 

第二个微妙的边缘情况是，由于对数表示的结构周期性，不同的整数对可以产生相同的小数部分。 例如，如果两个表达式仅相差整数幂差异（在分数表示中抵消），则简单的数字比较会错误地将它们视为不同的。 

## 方法

 暴力解决方案将计算$a^b$以高精度（或非常精细地近似），提取小数部分的长前缀，并比较字符串。 即使我们只比较长度的前缀$k$，选择$k$如果没有深入的数论洞察力，足够大以保证所有情况下的正确性是不可能的，并且每对的任何此类计算都太慢了$n$最多$2 \cdot 10^6$。 

关键的观察是我们实际上不需要数值$a^b$。 我们只需要它的分数行为的表示，以在所有对中唯一地识别它。 这促使我们将表达式转换为规范形式。 

我们使用对数。 写$$a^b = e^{b \ln a}.$$现在分解$b \ln a$分为整数部分和小数部分：$$b \ln a = \lfloor b \ln a \rfloor + \{b \ln a\}.$$然后$$a^b = e^{\lfloor b \ln a \rfloor} \cdot e^{\{b \ln a\}}.$$整数部分仅贡献整数缩放因子，而小数部分的行为完全由小数部分决定$\{b \ln a\}$。 由于乘以 的整数次方$e$改变幅度但不影响无限十进制序列下的分数展开模式的重复结构，当两个值的分数对数分量匹配时，两个值完全匹配。 

因此，每一对都可以用密钥表示：$$key(a,b) = b \ln a - \lfloor b \ln a \rfloor.$$现在的问题是计算所有对中该键的相等值。 我们计算这个浮点值，仔细地将其标准化，并使用哈希图来计算频率。 

一个微妙的点是浮动精度。 直接比较双精度数是不安全的，因此我们通过将小数部分缩放到足够大的精度来离散它（例如$10^{-12}$或更好）并将其转换为整数键。 这确保了稳定的分组。

 ## 算法演练

 1. 对于每对$(a, b)$, 计算$x = b \cdot \ln(a)$。 这将求幂转换为对数空间中的线性表达式，避免溢出和巨大的数字。 
2. 提取小数部分$x$作为$f = x - \lfloor x \rfloor$。 这隔离了决定无限十进制表示的重复结构的部分。 
3. 转换$f$例如，通过使用固定的大常数对其进行缩放，将其转换为稳定的整数密钥$10^{12}$，并将其四舍五入。 此步骤确保仅超出数值精度的值会合并到同一个存储桶中。 
4. 使用哈希映射来计算每个键在所有对中出现的次数。 
5. 对于每个频率$c$， 添加$c(c-1)/2$到答案。 这对共享相同小数签名的所有索引对进行计数。 

缩放起作用的原因是小数部分位于$[0,1)$，并且足够大的固定精度可确保对数学上相同的所有值进行确定性分组。 

### 为什么它有效

 每对$(a,b)$被映射成一个实数$b \ln a$，并且产生相同无限小数行为的所有对必须共享该对数值的相同小数部分。 整数部分仅提供乘法缩放，不影响相等条件。 因此，原始问题的等式简化为单个归一化分数不变量的等式。 由于我们通过该不变量的确定性离散化来存储桶，因此每个有效的等价类都只计算一次，并且两个不同的类不会合并，除非它们的差异超出了可表示的精度，这是由足够高的缩放控制的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math
from collections import defaultdict

SCALE = 10**12

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        freq = defaultdict(int)

        for _ in range(n):
            a, b = map(int, input().split())
            x = b * math.log(a)
            f = x - math.floor(x)
            key = int(f * SCALE + 0.5)
            freq[key] += 1

        ans = 0
        for c in freq.values():
            ans += c * (c - 1) // 2

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案依赖于将每对转换为对数空间表示，然后隔离小数部分。 乘以固定比例将连续小数值变成离散的桶索引。 舍入步骤是必不可少的； 如果没有它，浮点表示噪声会将相同的数学值分割成单独的键。 

最后的计数步骤使用标准的对组合公式，这避免了任何显式的对枚举。

## 测试用例```python
import sys, io
import math
from collections import defaultdict

SCALE = 10**12

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    out = []
    t = int(input())
    for _ in range(t):
        n = int(input())
        freq = defaultdict(int)
        for _ in range(n):
            a, b = map(int, input().split())
            x = b * math.log(a)
            f = x - math.floor(x)
            key = int(f * SCALE + 0.5)
            freq[key] += 1
        ans = 0
        for c in freq.values():
            ans += c * (c - 1) // 2
        out.append(str(ans))
    return "\n".join(out)

# small no-collision case
assert solve("1\n3\n2 3\n5 2\n4 5\n") == "0"

# duplicate groups
assert solve("1\n4\n2 2\n4 1\n8 2\n16 1\n") == "2"

# all equal
assert solve("1\n3\n2 2\n2 2\n2 2\n") == "3"

# minimum case
assert solve("1\n1\n10 10\n") == "0"

 ## 边缘情况

 一种边缘情况是所有对都相同，例如$(2,2)$重复了很多次。 该算法将每一对映射到相同的对数密钥，生成大小为单个频率的桶$n$。 对计数变为$n(n-1)/2$，由组合公式正确处理。 

另一种边缘情况是当小数部分由于浮点舍入而非常接近时，例如仅在深精度级别上不同的对。 缩放步骤将它们压缩为相同的整数键，舍入确保一致的存储桶分配而不是拆分组。 

最后一个边缘情况是$a = 1$。 在那种情况下$1^b = 1$， 所以$b \ln 1 = 0$为所有人$b$。 每个这样的对都映射到零，并且所有这些对都被正确分组为单个等价类，从而在该子集中产生对的完整二次计数。
