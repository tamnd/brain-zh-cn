 ## 解决方案
 ## 问题理解

 我们给出一排从 0 到 n 编号的图块，一只猫从第 0 个图块开始。从任何图块 i 开始，猫可以向前跳到任何图块 j，只要 j 严格领先且不超过 k 步，即 1 ≤ j − i ≤ k。 这个问题问的是，如果猫重复进行这样的跳跃，有多少种不同的方式可以到达第 n 块。 

每条路都是从 0 开始到 n 结束的位置序列，其中每一步最多向前移动 k 个格子。 不同的序列被认为是不同的方式，即使它们共享中间位置，但跳跃的选择不同。 

输入包含多个测试用例，每个测试用例都是独立的。 对于每一对 (n, k)，我们必须计算有效跳转序列的数量模 1e9 + 7。 

约束很大：每个测试用例 n 最多可达 1e6，并且测试用例中所有 n 的总和最多可达 2e6。 这立即排除了任何在 O(k) 时间内从头开始重新计算每个位置的转换的解决方案，因为在最坏的情况下 k 也可能很大，并且总工作量将超过 1e12 次操作。 

简单的动态编程方法将 dp[i] 定义为到达图块 i 的方式数量，并通过将 dp[i − 1] 到 dp[i − k] 求和来计算 dp[i]。 这是正确的，但太慢了，因为每个状态可能需要最多 k 个添加，每个测试用例的复杂度为 O(nk)。 

当 k 很大时，例如 k ≥ n，就会出现第二个微妙的问题。 在这种情况下，每个先前的位置都可以直接转换到 i，因此 dp[i] 成为所有先前 dp 值的前缀和。 即使结构完全简化，仍然迭代 k 的简单实现会做不必要的工作和 TLE。 

## 方法

 暴力动态规划的解释很简单。 从每个图块 i 开始，我们尝试所有可能的下一个跳转到 i + 1、i + 2、直到 i + k，并累积贡献。 这是有效的，因为每个有效路径都是通过扩展较短的有效路径来唯一构造的，因此对所有前趋求和可以准确地计算所有序列。 

效率低下源于重新计算重叠的总和。 计算 dp[i] 时，我们重复对已用于 dp[i + 1]、dp[i + 2] 等的相同 dp 值求和。 连续窗口之间的重叠建议维护运行窗口总和，而不是每次都重新计算。 

关键的观察结果是 dp[i] 仅取决于先前 k 值的滑动范围。 我们不是从头开始重新计算这个范围总和，而是维护最后 k dp 值的滚动总和。 当我们从 i − 1 移动到 i 时，我们将 dp[i − 1] 添加到窗口中，并删除 dp[i − k − 1]（如果存在）。 这将每次转换减少到 O(1)，将整个 DP 变成线性时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力DP | O(nk) | O(nk) | O(n) | 太慢了 |
 | 滑动窗口 DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将 dp[i] 定义为到达图块 i 的方式数。 过渡来自距离 k 内的所有先前图块。

1. 初始化 dp[0] = 1，因为只有一种方法可以从原点开始而不进行任何跳转。 
2. 维护一个表示 dp[i − 1] + dp[i − 2] + ... + dp[i − k] 的运行窗口总和，但仅限于有效索引。 这个总和代表了在一次额外跳跃中到达当前位置的所有方法。 
3. 对于从 1 到 n 的每个位置 i，将 dp[i] 设置为等于当前窗口总和。 这是有效的，因为每个有效的最后跳转必须来自之前的 k 个位置之一，并且每个这样的 dp 值都为 i 贡献了一个扩展。 
4. 分配 dp[i] 后，通过添加 dp[i] 来更新窗口总和，因为它可能对未来状态有贡献。 
5. 如果 i ≥ k，则从窗口总和中删除 dp[i − k]，因为它不再位于未来位置的距离 k 内。 这保持了窗口总和始终准确反映最后 k 个 dp 值的不变性。 

中心思想是每个 dp[i] 只计算一次，然后贡献给接下来的 k 个状态，之后它就会过期。 

### 为什么它有效

 在每个索引 i 处，窗口总和等于到达任何图块 j 的方式总数，使得 i − k ≤ j ≤ i − 1。以 i 结尾的每条有效路径必须恰好来自这样的一个 j，然后是从 j 到 i 的单次跳转。 由于 dp[j] 已经计算了到达 j 的所有路径，因此对所有有效 j 求和将所有路径根据其最后的跳转原点划分为不相交的组。 这可确保不会遗漏任何路径，并且不会重复计算任何路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve_case(n, k):
    dp = [0] * (n + 1)
    dp[0] = 1

    window_sum = 1

    for i in range(1, n + 1):
        dp[i] = window_sum % MOD
        window_sum = (window_sum + dp[i]) % MOD

        if i >= k:
            window_sum = (window_sum - dp[i - k]) % MOD

    return dp[n]

def main():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        print(solve_case(n, k))

if __name__ == "__main__":
    main()
```其实现完全遵循滑动窗口DP。 为了清晰起见，保留了 dp 数组，但实际上只需要最后的 k 值来保持正确性。 window_sum 变量是避免重复重新计算总和的关键优化。 

当 i ≥ k 时的减法步骤可以防止窗口无限增长。 如果没有它，算法将错误地累积不再允许作为跳跃起点的位置的贡献。

 ## 边缘情况

 当k = 1时，算法退化为单路径链。 对于 n = 5，每个 dp[i] 仅取决于 dp[i − 1]，因此窗口总和始终等于 dp[i − 1]。 更新规则仍然有效，因为我们添加 dp[i] 并立即删除 dp[i − 1]，留下稳定的值传播。 

当 k ≥ n 时，每个 dp[i] 变为包括 dp[0] 在内的所有先前 dp 值的总和。 对于 n = 3、k = 10，窗口永远不会删除任何内容，因为所有步骤的 i < k。 窗口总和累加所有 dp 值，产生指数增长，这与每个较早的图块可以直接跳转到任何较晚的图块这一事实相一致。
