 ## 解决方案
 ## 问题理解

 我们得到了几个独立的实验。 每个实验都有一个参考整数$D$代表特定猫的 DNA，以及由整数表示的其他猫的列表$a_i$。 每个整数将遗传特征编码为位。 

如果存在至少一个位位置，则认为一只猫与参考猫相关。$a_i$和$D$有一个 1。按位计算，这意味着两个数字的按位与非零。 任务只是计算每个测试用例中有多少个值满足此条件。 

每个测试用例最多可以包含$2 \cdot 10^5$数字，并且在所有测试用例中最多$10^6$。 这立即排除了任何二次方$n$，因为即使每个测试用例的线性扫描也已经是唯一可行的结构。 我们对每个元素执行的操作必须是恒定时间。 

出现微妙的边缘情况时$a_i = 0$。 自从$0 \& D = 0$，这些元素永远不会被计算在内，无论$D$。 另一种边缘情况发生在$D = 0$。 在这种情况下，没有数字可以与$D$，所以答案始终为零，即使所有$a_i$均非零。 如果假设每个正数自动共享一位，那么这些情况很容易处理不当。 

## 方法

 最直接的方法是单独检查每只猫。 对于每个$a_i$, 计算$a_i \& D$如果结果非零，则增加计数器。 这是正确的，因为相关性的定义正是这个条件。 

这种方法的成本与所有测试用例的整数总数呈线性关系。 对于 32 位或 64 位整数，每个按位 AND 都是常数时间，因此总复杂度为$O(\sum n)$，这最多是$10^6$。 如果使用快速输入来实现，那么在 Python 中这在一秒的限制内是相当快的。 

没有更深层次的结构可供利用，因为每个元素都是独立的。 任何对值进行预处理或分组的尝试都不会进一步减少工作量，因为条件取决于固定的位掩码$D$分别应用于每个值。 关键的认识是问题已经很小了：它是一个基于位条件的流过滤器。

 | 暴力破解（每个元素并检查）| 每个测试用例 O(n) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 每种情况的目标是计算有多少值与以下内容共享至少一个设置位：$D$。 

1. 阅读$n$和$D$对于测试用例。 这定义了我们将与之比较的固定位掩码。 
2. 将计数器初始化为零。 这将累积所有符合条件的猫。 
3. 迭代每个值$a_i$在列表中。 
4. 计算$a_i \& D$。 如果结果不为零，则递增计数器。 该检查直接编码两个位掩码是否相交。 
5. 处理完所有值后，输出计数器。 

每个步骤都是必要的，因为无法从一个元素推断另一个元素的结果； 该条件对于每对来说都是纯粹局部的$(a_i, D)$。 

### 为什么它有效

 正确性取决于按位与隔离共享设置位的事实。 如果两个数字共享至少一位均为 1 的位，则它们的 AND 必须非零。 相反，如果 AND 为零，则不共享任何位位置，这意味着它们与问题定义无关。 由于我们对每个元素都测试一次，并且条件既是必要的又是充分的，所以最终的计数是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, D = map(int, input().split())
        arr = list(map(int, input().split()))
        
        ans = 0
        for x in arr:
            if x & D:
                ans += 1
        
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案的结构是测试用例上的简单流式循环。 快速输入通过`sys.stdin.readline`是必不可少的，因为整数总数可以达到一百万。 表达式`x & D`是核心运营； 它直接对关系条件进行编码，无需任何额外的计算。 

除了当前测试用例之外，不需要额外的预处理或存储，这使得内存使用量相对于输入大小保持恒定。

 ## 边缘情况

 一个重要的边缘情况是当$D = 0$。 对于输入：```
1
4 0
1 2 3 4
```该算法计算$x \& 0 = 0$对于每个元素。 循环仍然遍历所有元素，但计数器永远不会增加。 输出为 0，这与定义相符，因为不可能出现位重叠。 

另一个边缘情况是当所有$a_i = 0$:```
1
3 5
0 0 0
```每次迭代都会评估`0 & 5 = 0`，因此计数器始终保持为零。 该算法自然地处理这个问题，无需特殊的外壳。 

最后一个微妙的情况是$D$仅设置了一位，例如$D = 1$。 只应计算奇数：```
1
5 1
2 3 4 5 6
```该算法仅正确计算 3 和 5，因为只有它们具有最低有效位集。
