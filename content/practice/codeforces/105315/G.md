 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供一个有向或无向加权图，以及指定的起始节点和目标节点。 路线是连接起点和目标的任何边序列，只要顶点和边在图中形成有效的行走，它们就可以重复。 此类路线的值定义为沿该路线的所有边权重的按位与。 任务是在从起点到目标的所有可能有效路线中最大化该值。 

重要的结构细节是按位与在路径上的表现方式。 一旦沿路线使用的任何边权重中的某个位变为零，整个路线值的该位就会永久丢失。 因此，较长的路线只会限制最终值，但它们可能仍然有用，因为它们允许我们到达保留更多高位的边缘。 

这些限制意味着我们无法枚举路径。 每个测试用例最多有 10^5 个节点和 3×10^5 条边，任何依赖于探索所有路径甚至所有简单路径的解决方案都是不可能的。 即使是子集上的最短路径式动态规划也是遥不可及的。 唯一可行的方向是独立处理位，并将问题减少到边缘过滤下的重复可达性检查。 

如果我们假设答案只是沿某些最短路径或任何最短路径的最小边权重，则会出现微妙的失败情况。 例如，考虑一个图，其中存在具有低权重边的从 s 到 e 的直接路径，但较长的周期允许仅使用高权重边的不同路径。 即使存在更好的保留 AND 的路线，贪婪的最短路径方法也会错误地选择直接边缘并产生较小的 AND 值。 

另一个陷阱是假设路径长度或边权重的单调性。 添加边并不一定会以可预测的方式减少 AND，除非我们对允许的位施加限制。 这就是为什么问题自然会变成位过滤和连接问题而不是经典的最短路径问题。 

## 方法

 蛮力的想法是考虑从 s 到 e 的所有可能的行走，并计算每个行走的权重的按位与。 这是正确的，因为它明确评估每条有效路线。 然而，由于循环可以任意多次遍历，因此行走次数呈指数增长。 即使限制为简单路径也无济于事，因为在最坏的情况下，在密集图中枚举它们仍然是指数级的。 

关键的观察是翻转视角。 我们不构建路径，而是决定要在最终答案中保留哪些位。 假设我们猜测某个位掩码 X 是可以实现的。 这意味着存在一条从 s 到 e 的路径，该路径上的每条边都设置了 X 的所有位。 换句话说，路径上的所有边都属于通过过滤权重包含 X 作为比特子集的边形成的子图。 因此，可行性简化为过滤图中的连接性检查。 

这导致从最高位向下进行贪婪位构造。 我们从答案 = 0 开始。对于从最高有效位到最低有效位的每一位，我们尝试设置它并仅使用权重包含所有当前所需位的边来测试 s 和 e 是否仍然连接。 如果连通性成立，我们保留该位； 否则我们会丢弃它。 

可以使用 DFS、BFS 或 DSU 在过滤的边缘上完成连接测试。 由于我们重复此检查最多 60 位，因此总体复杂性仍然是可控的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（各行各业）| 指数| O(n + m) | 太慢了|
 | 按位贪婪 + 连接性检查 | O(60 × (n + m)) | O(60 × (n + m)) | O(n + m) | 已接受 |

 ## 算法演练

 我们维护一个候选答案掩码，从高到低一点一点地构建。

1. 将答案掩码初始化为 0。这表示我们最初允许所有路径，因为尚未强制执行位约束。 
2.从59到0迭代位。我们考虑每个位是否可以包含在最终答案中。 
3. 对于给定的位 b，临时构造一个等于当前答案加上该位的测试掩码。 
4. 构建或遍历仅由边组成的子图，其权重包含测试掩码中的所有位。 此过滤可确保在此图中找到的任何路径都将保留按位 AND 下掩码中的所有位。 
5. 从 s 运行连通性搜索（通常是 BFS 或 DFS）。 如果我们可以到达 e，则存在一条保留测试掩码中所有位的有效路径，因此我们在答案中永久设置该位。 
6. 如果 e 不可达，则丢弃该位并保持之前的答案不变。 

这种贪婪选择有效的原因是在二进制表示中较高位支配较低位。 一旦发现较高位不可行，以后的操作就无法恢复它，因此我们可以安全地丢弃它。 相反，如果一个位是可行的，保留它永远不会阻止我们稍后添加较低位，因为较低位只会进一步收紧约束。 

### 为什么它有效

 在任何步骤中，当前掩码表示一组与已选择的每个位都兼容的边缘。 连通性检查确保至少存在一条完全包含在该约束子图中的路径。 因此，不变量是在处理位 b 之后，存在一条从 s 到 e 的路径，该路径仅使用其权重包含所有选定位的边。 由于位决策仅限制图，因此一旦确认可行性，它就与所有先前接受的位保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m, s, e = map(int, input().split())
        
        edges = []
        adj_all = [[] for _ in range(n + 1)]
        
        for _ in range(m):
            a, b, w = map(int, input().split())
            edges.append((a, b, w))
            adj_all[a].append((b, w))
            adj_all[b].append((a, w))
        
        def can(mask):
            from collections import deque
            vis = [False] * (n + 1)
            dq = deque([s])
            vis[s] = True
            
            while dq:
                u = dq.popleft()
                if u == e:
                    return True
                for v, w in adj_all[u]:
                    if not vis[v] and (w & mask) == mask:
                        vis[v] = True
                        dq.append(v)
            return False
        
        ans = 0
        for bit in range(59, -1, -1):
            if can(ans | (1 << bit)):
                ans |= (1 << bit)
        
        print(ans)

if __name__ == "__main__":
    solve()
```该实现保留完整的邻接列表并为每个位尝试执行 BFS。 关键检查`(w & mask) == mask`强制使用的每条边都保留答案中当前所需的所有位。 

BFS 对每一位都重新启动，这是可以接受的，因为对位的约束很小并且图大小适中。 正确性取决于不在不同掩码之间重用访问状态，因为可达性会随着每个添加的约束而变化。

 ## 边缘情况

 当唯一可用路径需要放松高位以维持连接时，就会出现一种边缘情况。 假设一个图，其中 s 通过低权重桥连接到 e，但高权重边形成与 e 断开的环。 该算法将正确拒绝高位，因为受约束掩码下的 BFS 无法到达目标，即使完整图中存在视觉上更长的路径。 

另一种情况是存在多个组件且连接性取决于确切的掩码。 例如，边可能将 s 连接到掩码 X 下的中间组件，但 e 仅在不同的边子集下。 BFS 重新计算每个掩码的最新可达性，确保过时的可达性假设不会在位试验之间泄漏。
