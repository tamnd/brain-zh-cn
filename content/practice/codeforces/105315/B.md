 ## 解决方案
 ## 问题理解

 我们得到了许多独立的查询。 每个查询提供一个整数n，我们必须输出一个整数a，其中a是最接近n的素数。 如果两个素数同样接近，则必须选择较小的一个。 

该任务本质上是重复回答：“哪个素数最接近数轴上的给定值？”

 输入大小清楚地表明 n 最多可以达到 1,000,000，并且最多可以有 100,000 个查询。 如果每个查询重复繁重的工作，尝试独立测试每个 n 的素性的简单方法已经很困难了。 约束的关键含义是我们需要对主要信息进行一次预处理并有效地重用它。 

当 n 本身为素数时，会出现微妙的边缘情况。 在这种情况下，答案很简单，因为到自身的距离为零。 当两个素数等距时，例如 n = 10 左右，其中 7 和 11 的距离都是 3，就会出现另一种边缘情况。该问题明确要求选择较小的素数，因此 7 是正确的。 

另一种可能破坏朴素逻辑的情况是在没有适当界限的情况下从 n 向外搜索。 如果我们只向下或只向上搜索，我们可能会错过另一边最接近的素数。 

## 方法

 蛮力的想法很简单。 对于每个查询 n，我们检查从 n 向外的每个整数：首先检查 n 是否为素数，然后检查 n−1 和 n+1，然后 n−2 和 n+2，依此类推，直到找到两侧的素数。 每次素数检查至少花费 O(√n)，并且在最坏的情况下，我们可能会在两边都达到素数之前探索一个很大的范围。 超过 100,000 个查询，这很快就会变得太慢。 

效率低下的原因是在不同查询中重复对重叠数字进行素性检查。 许多查询询问附近的值，并且数字的素数在查询之间不会改变。 这建议将所有素数预处理一次，直到最大可能的 n 次，然后在恒定时间内回答每个查询。 

一旦我们预先计算了一个布尔数组来指示 1,000,000 以内的每个数字是否是素数，问题就简化为静态集中的最近邻搜索。 对于每个 n，我们从 n 同时在两个方向上向外扩展，直到我们在每一侧找到至少一个素数或确认一个方向获胜。 由于搜索范围受到预先计算的筛法的限制，因此在实践中这变得足够快。 

一种更结构化的思考方式是，我们将重复的素数查询转换为单个全局筛选结构，然后进行局部最近素数查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(t · √n · k) 最坏情况展开 | O(1) | O(1) | 太慢了|
 | 筛选+搜索| O(N log log N + t·d) | O(N log log N + t · d) | O(N) | 已接受 |

 这里 d 是到最近素数的平均距离，对于 10^6 以下的数字来说，该距离很小。 

## 算法演练

 我们首先使用筛子预先计算 1,000,000 以内的所有数字的素数。 这确保了以后的任何素性检查都是 O(1)，这是必要的，因为我们将多次查询此信息。 

接下来，我们迭代每个查询值 n 并确定最接近的素数。

1. 预先计算一个大小为 1,000,001 的数组 is_prime，初始化为 true，然后将 0 和 1 标记为非素数。 这建立了一个基线，其中每个数字都被假定为素数，除非另有证明。 
2. 从 2 向上进行筛选，每当我们找到素数 p 时，将 p 的所有倍数标记为非素数。 此步骤通过消除结构化批次中的合数而不是单独测试每个数字，有效地为整个范围构建完整的素性结构。 
3. 对于每个查询值n，首先检查is_prime[n]是否为true。 如果是这样，我们立即输出 n，因为它与自身的距离为零，使其成为最接近的素数。 
4. 如果 n 不是质数，我们从 1 开始向外扩展，增加距离 d。在每一步，我们检查 n − d 和 n + d，前提是它们保持在边界内。 一旦我们在这些候选者中找到至少一个素数，我们就停止。 
5. 如果 n − d 和 n + d 在相同距离处都是素数，则我们输出 n − d，因为该问题需要在平局的情况下选择较小的素数。 

为什么这有效与搜索过程中的对称性有关。 我们探索距离 n 越来越远的数字，所以当我们第一次遇到任何素数时，它一定是最接近的素数。 因为我们在增加 d 之前检查每个距离级别的两侧，所以我们保证不会跳过更近的素数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXN = 10**6

is_prime = [True] * (MAXN + 1)
is_prime[0] = is_prime[1] = False

p = 2
while p * p <= MAXN:
    if is_prime[p]:
        step = p
        start = p * p
        for x in range(start, MAXN + 1, step):
            is_prime[x] = False
    p += 1

def solve(n):
    if is_prime[n]:
        return n
    d = 1
    while True:
        left = n - d if n - d >= 0 else None
        right = n + d if n + d <= MAXN else None

        if left is not None and is_prime[left]:
            return left
        if right is not None and is_prime[right]:
            return right
        d += 1

t = int(input())
out = []
for _ in range(t):
    n = int(input())
    out.append(str(solve(n)))

print("\n".join(out))
```筛子部分一次性构建全局素性知识。 循环结构确保从正方形开始有效地消除复合材料，从而避免了多余的工作。 

查询功能完全依赖于预先计算的数组。 尽早返回素数 n 很重要，因为它可以避免不必要的扩展。 双向扩展保证了正确性，因为距离单调增加。 

一个微妙的细节是边界检查。 如果不确保 n − d 和 n + d 保持在数组边界内，我们可能会访问无效索引或错过边缘处的有效素数。 该实现明确保护两个方向。 

## 工作示例

 ### 示例 1

 输入：

 n = 10

 我们向外扩展：

 | d | n-d | 主要的？ | n+d | 主要的？ | 决定|
 | ---| ---| ---| ---| ---| ---|
 | 1 | 9 | 没有| 11 | 11 是的 | 返回 11 |

 我们在 d = 1 处停止，因为 11 是素数，而 9 不是素数。 这表明向上和向下搜索都是必要的，因为最接近的素数可能只位于一侧。 

### 示例 2

 输入：

 n = 10（平局案例演示）

 我们再次检查距离：

 | d | n-d | 主要的？ | n+d | 主要的？ | 决定|
 | ---| ---| ---| ---| ---| ---|
 | 1 | 9 | 没有| 11 | 11 是的 | 返回 11 |
 | 2 | 8 | 没有| 12 | 12 没有| 继续 |

 这证实了算法总是选择在最小距离处第一个遇到的素数，并且当两侧在某个 d 处匹配时，选择左侧。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N log log N + t·d) | O(N log log N + t · d) | sieve 构建素数，每个查询向外扩展直到最接近的素数 |
 | 空间| O(N) | 素数高达 1e6 的布尔数组 |

 筛子在预处理中占主导地位，但很容易满足 N = 1e6 的约束。 每个查询通常需要非常小的向外移动，因为素数在此范围内很密集，使得解决方案对于 100,000 个查询来说非常快速。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    MAXN = 10**6
    is_prime = [True] * (MAXN + 1)
    is_prime[0] = is_prime[1] = False

    p = 2
    while p * p <= MAXN:
        if is_prime[p]:
            for x in range(p * p, MAXN + 1, p):
                is_prime[x] = False
        p += 1

    def solve(n):
        if is_prime[n]:
            return n
        d = 1
        while True:
            if n - d >= 0 and is_prime[n - d]:
                return n - d
            if n + d <= MAXN and is_prime[n + d]:
                return n + d
            d += 1

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append(str(solve(n)))
    return "\n".join(out)

 ## 边缘情况

 对于 n = 1，该算法从 d = 1 开始，并检查 n − d = 0 和 n + d = 2。由于 0 无效且 2 是质数，因此它会立即正确返回 2。 

对于 n = 2，早期素数检查返回 2，而不进入搜索循环。 这避免了不必要的边界检查并确保最小有效素数的正确性。 

对于像 n = 999983（已经是上限附近的素数）这样的大值，筛子会正确标记它，并且算法会立即返回它而不进行扩展。 这表明早期退出条件对于密集素数区域附近的性能稳定性至关重要。
