 ## 解决方案
 ## 问题理解

 我们得到了几种花卉类型的集合，其中每个集合（称为花束）本质上是从大小最多为 60 的宇宙中抽取的一组允许的花卉标签。对于每个测试用例，Osama 恰好选择一个花束，然后构建一个长度为 1 的花园$m$。 花园中的每个位置都是通过从所选花束中挑选任何花型来独立填充的，并且允许重复。 

所以如果一束花包含$k$不同的花型，那么该花束就可以准确地生成$k^m$不同的花园，因为每个$m$职位可以是以下任何一个$k$独立类型。 

复杂的是，我们被要求提供所有花束中不同花园的数量，这意味着如果两个不同的花束可以产生相同的序列，则该序列只能计算一次。 当花束排版重叠时，这种情况自然会发生。 

约束驱动解决方案结构。 花束的数量最多为 19 个，这立即表明对$n$， 例如$2^n$，是可以接受的。 价值$m$可以大到$10^9$，因此任何迭代花园位置的方法都是不可能的。 然而，由于每个花束都贡献了以下形式的术语$|S|^m$，我们只需要快速求幂，而不需要长度迭代。 

最重要的结构观察是花类型的范围只有 60 个大小，因此每个花束可以表示为超过 60 个元素的位掩码。 这使得交叉口操作快速而准确。 

一个微妙的边缘情况是花束的不同子集产生相同的交集。 例如，如果花束 A 和花束 B 相同，则两者贡献相同$k^m$，天真的求和会重复计算。 另一个问题是，必须对花束子集使用包含-排除，否则无法纠正重叠。 

天真的想法的一个具体失败案例是：

 如果花束 1 是$\{1,2\}$花束 2 是$\{2,1\}$， 和$m=2$，然后两者都会生成相同的序列集，并且只需添加$2^2 + 2^2$会多算的。 正确答案是$4$， 不是$8$。 

## 方法

 蛮力的想法是独立处理每束花，枚举所有花束$k^m$从概念上对序列进行排序，然后尝试跨花束进行重复数据删除。 这会立即失败，因为即使在以下情况下存储或迭代所有序列也是不可能的：$m$很大。 每束花束的序列数量呈指数增长$m$, so any explicit construction is infeasible.

 关键的结构见解是每个花束定义了一个集合的完整笛卡尔幂。 这些权力的联合可以使用花束上的包含-排除来处理。 对于任何花束子集，它们的交集对应于仅使用该子集中所有花束共有的花类型即可形成的序列。 If that intersection has size$x$，那么它正好贡献$x^m$序列。 

对所有非空子集使用包含-排除可确保每个序列都被精确计数一次，因为每个有效序列都有一组明确定义的花束，其中包含其所有使用的花类型，并且包含-排除隔离了负责它的最大交集。 

自从$n \le 19$，迭代所有$2^n$子集是可行的。 每个子集都需要计算掩码的交集，然后评估快速模幂。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| 指数为$m$| 巨大| 太慢了|
 | 子集上的包含-排除 |$O(2^n \cdot 60 + 2^n \log m)$|$O(60)$| 已接受 |

 ## 算法演练

 1. 将每个花束表示为 60 位掩码。 每个位都指示该花束中是否存在某种花型。 这允许使用按位与来计算交集，这是常数时间。 
2. 使用位掩码迭代花束的所有非空子集$1$到$(1 \ll n) - 1$。 每个子集代表一组花束，我们将分析其交集。 
3. 对于每个子集，通过对子集中的所有花束进行 AND 运算来计算交集掩码。 该掩码的大小是子集中所有花束共有的花朵类型的数量。 这是唯一一组可以同时出现在对所有这些花束有效的序列中的类型。 
4. 计算$x^m \bmod (10^9+7)$， 在哪里$x$是相交掩模的大小。 这表示可以使用这些常见类型形成的序列数。 
5. 应用包含-排除：如果子集大小为奇数，则将此值添加到答案中，否则减去它。 这种交替纠正了由于不同花束组之间的重叠而导致的计数过多。 
6. 返回最终累加结果取模$10^9+7$。 

### 为什么它有效

 每个有效的花园都是一个长度-$m$对某些花类型的排序。 该套装必须至少包含在一个花束中，但可以包含在多个花束中。 花束子集上的包含-排除可确保每个序列针对仍包含其所有所选花类型的花束的最大子集计数一次。 由于交集大小唯一地确定了每个子集中序列的可用字母表，因此贡献$x^m$精确计算该子集支持的所有序列，并且交替符号消除了较大重叠子集的过度计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modpow(a, e):
    res = 1
    a %= MOD
    while e > 0:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        masks = []
        for _ in range(n):
            arr = list(map(int, input().split()))
            k = arr[0]
            vals = arr[1:]
            mask = 0
            for v in vals:
                mask |= 1 << (v - 1)
            masks.append(mask)

        ans = 0

        for s in range(1, 1 << n):
            inter = (1 << 60) - 1
            for i in range(n):
                if s >> i & 1:
                    inter &= masks[i]
                    if inter == 0:
                        break

            if inter == 0:
                continue

            cnt = inter.bit_count()
            val = modpow(cnt, m)

            if bin(s).count("1") % 2 == 1:
                ans = (ans + val) % MOD
            else:
                ans = (ans - val) % MOD

        print(ans % MOD)

if __name__ == "__main__":
    solve()
```实现直接遵循子集枚举。 每个花束都被压缩为一个位掩码，因此交集减少为单个 AND 运算。 交集从所有 60 位集开始，即使在某些花束稀疏的情况下也能确保正确性。 

求幂是用二进制求幂处理的，因为$m$可以大到$10^9$。 子集奇偶校验确定是添加还是减去贡献，从而完全按照需要实现包含-排除。 

一个小但重要的细节是当交叉点变为零时提前停止。 一旦没有常见的花类型，该子集就没有任何贡献，并且会跳过进一步的处理。

 ## 边缘情况

 所有花束都相同的情况会强调重复的子集贡献。 对于每个花束所在的输入`{1,2,3}`和$m=5$，每个子集具有相同的交集大小 3。包含-排除和变为$(1 - (n choose 1) + (n choose 2) - \dots) \cdot 3^5$，其精确计算结果为$3^5$，确认重复项正确折叠。 

当花束没有共同元素时，就会出现空交集的情况。 例如`{1}`,`{2}`,`{3}`与任何$m$。 每个大小至少为 2 的子集的交集为零并且没有任何贡献，只留下单例，因此结果是$1^m + 1^m + 1^m = 3$，这符合每个花束仅生成一个恒定序列的事实。 

更大的重叠结构，例如`{1,2}`,`{2,3}`,`{1,3}`，检查成对重叠是否已通过三重交集进行校正。 该算法为单元素 2、1 和 3 分配正确的权重，而不会重复，因为每个元素都显示为特定子集族的交集。
