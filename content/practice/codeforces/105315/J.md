 ## 解决方案
 ## 问题理解

 每个测试用例描述一组菜肴，其中每道菜都属于特定的餐厅并具有数字味道值。 关键的限制是你不能混用餐馆。 您必须选择一家餐厅并拿走属于该餐厅的所有菜肴。 一家餐厅的价值是由其中最弱的菜肴决定的，即所有菜肴中味道价值最低的。 你的任务是找出哪家餐厅的最弱菜品尽可能地强。 

更具体地说，对于每个餐厅标签，您将其所有相关的口味值分组，计算该组内的最小值，然后在所有这些最小值中选择最大值。 

所有测试用例的输入大小总计高达 500,000 道菜肴，这立即排除了任何重复扫描每个查询列表或从头开始重新计算每个餐厅选择最小值的方法。 盘子数量呈二次方的解决方案会太慢，因为即使每个测试用例重复 10^3 次 10^5 操作也已经导致不可接受的运行时间。 

当多个菜肴属于同一家餐厅但以任意顺序到达时，就会出现微妙的边缘情况。 例如，如果一家餐厅的菜肴值为 10、2 和 7，则无论顺序如何，该餐厅的答案贡献都是 2。 在这种情况下，如果最小值出现较早并被覆盖，仅跟踪每个餐厅最后看到的值的简单方法将错误地输出 7。 

当每家餐厅都只有一种菜时，就会出现另一种极端情况。 那么答案就是最大的 di，因为每个餐厅的最小值是其唯一的值。 任何意外合并餐馆的错误分组逻辑都会在这里失败。 

## 方法

 蛮力的方法是独立对待每个餐厅。 对于每个餐厅标签，我们收集其所有菜肴，并通过扫描分配给它的完整菜肴列表来明确计算最小值。 如果有 n 个菜肴，并且可能有 n 个不同的餐厅，那么在最坏的情况下，我们会重复重新扫描每个餐厅的几乎整个数组。 这导致最坏情况下的操作计数约为 n 平方，在最大限制下大约为 10^10 次操作，远远超出 2 秒限制可以处理的范围。 

关键的观察是我们实际上不需要重新计算任何东西。 每道菜都为一个餐厅组贡献一个值，而每个组唯一重要的信息就是其最小值。 这表明当我们读取输入一次时，保持每家餐厅的运行最小值。 我们没有重新访问数据，而是更新以餐厅 id 为键的字典，存储迄今为止看到的最小值。 

这将问题转化为单通道聚合问题。 每次更新都是常数时间，处理完所有菜肴后，我们只需要扫描收集到的最小值即可找到最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例，构建从餐厅 ID 到当前最小菜肴值的映射。

1. 初始化一个空字典，为每个餐厅存储迄今为止看到的最小味道值。 这种结构是必要的，因为我们需要每道菜不断更新。 
2. 逐一阅读每道菜。 对于属于 a 餐厅且值为 d 的菜肴，检查 a 是否以前见过。 如果没有，则直接将 d 存储为当前最小值。 
3. 如果餐厅已经存在于字典中，则仅当新菜肴的值较小时才更新其存储的值。 这确保我们永远不会忘记该餐厅中最差的菜肴。 
4. 处理完所有菜肴后，迭代所有存储的最小值并计算其中的最大值。 最后一步根据所需标准选择最佳的餐厅。 
5. 输出测试用例的最大值。 

这样做的原因是，每家餐厅的贡献完全由其最小值决定，并且可以逐步维持最小值，而无需再次提供完整的菜肴列表。 

### 为什么它有效

 在处理过程中的任何时刻，字典都会存储每家餐厅所有已处理菜肴的最少数量。 这是一个不变量，仅在出现较小的值时替换存储的值。 由于每道菜都只处理一次，因此在输入结束时，每个餐厅的存储值恰好是该餐厅全套菜品的全局最小值。 将最大值与这些正确的最小值相比较，得出最佳的餐厅选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        best = {}
        for _ in range(n):
            a, d = map(int, input().split())
            if a in best:
                if d < best[a]:
                    best[a] = d
            else:
                best[a] = d

        ans = 0
        for v in best.values():
            if v > ans:
                ans = v
        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现的核心是字典`best`，它存储每个餐厅的当前最小值。 更新步骤经过精心编写，以避免不必要的函数调用或合并。 最后的循环是分开的，因为将聚合与读取混合在一起会使正确性推理变得复杂，并且有可能错过所有餐厅的最终比较。 

一个微妙的点是初始化`ans`。 从 0 开始是安全的，因为根据约束条件，所有菜品值都是正值。 如果值可能为负数，则需要将初始化调整为负无穷大或第一个字典值。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        best = {}
        for _ in range(n):
            a, d = map(int, input().split())
            if a in best:
                best[a] = min(best[a], d)
            else:
                best[a] = d

        ans = max(best.values())
        out.append(str(ans))

    return "\n".join(out)

 ## 边缘情况

 一个重要的边缘情况是，一家餐厅多次出现，其价值随着时间的推移而下降。 例如，输入：```
1
4
1 10
1 5
1 7
1 3
```该算法逐步更新存储的值：

 首先是10，然后是5，然后在7之后仍然是5，然后变成3。存储的值始终是迄今为止看到的最小值的不变性保证了正确性，最终输出是3。 

另一种情况是多家餐厅竞争激烈时：```
1
5
1 10
2 9
3 8
4 7
5 6
```每个餐厅都有一个值，因此字典存储所有值不变。 最终的最大值正确返回 10，这表明当不需要分组细化时，该算法简化为简单的最大值选择。
