 ## 解决方案
 ## 问题理解

 我们正在维护一组动态整数，其中元素一次插入和删除一个操作。 每次更新后，我们需要计算在数轴上查看时该集合的“碎片化”程度。 

更准确地说，如果我们对当前元素进行排序并将连续整数分组为最大连续块，则每个这样的块贡献一个段。 单个孤立的数字也算作一个段。 我们要求维护的函数正是这些连续段的数量。 

例如，如果集合是`{1, 2, 4, 5, 6, 8}`，排序的顺序已经很清楚了。 数字`{1, 2}`形成一段，`{4, 5, 6}`形成另一个，并且`{8}`单独存在，总共分为三个部分。 

每个操作要么向集合中插入一个数字，要么删除一个保证存在的数字。 每次修改后，我们必须输出当前此类段的数量。 

操作数的限制最多为 100 万，值可以大到 10^18。 这立即排除了在每次更新后扫描或排序整个集合的任何解决方案。 即使是对数结构也必须谨慎使用，因为每次操作我们都需要摊销 O(log n) 或更好的结果。 

围绕邻接出现了一个微妙的问题。 定义完全取决于是否`x-1`和`x+1`存在于集合中。 任何正确的解决方案都必须有效地跟踪这些关系，因为每次更改后从头开始重新计算它们会太慢。 

更新桥接或分割段时会出现边缘情况。 

如果我们插入一个连接两个现有线段的数字，例如插入`5`进入`{1, 2}`和`{6, 7}`，段数减少 1。 简单地在插入时增加计数器的简单方法将会失败。 

如果我们删除连接两个邻居的号码，例如删除`5`从`{4, 5, 6}`，段数加一。 任何在删除时不考虑邻接性的解决方案都将在这里崩溃。 

最后，围绕边界重复插入和删除，例如`x = 0`或者非常大的值在逻辑上仍然表现相同，因为只有邻接性很重要，而不是大小。 

## 方法

 强力策略将维护整个集合并在每次操作后重新计算段数。 每次更新后，我们可以对集合进行排序并线性扫描它，计算连续差异大于 1 的次数。这是正确的，因为它直接遵循连续间隔的定义。 

然而，由于排序，每次查询的重新计算成本为 O(n log n)，如果我们维护排序结构但仍扫描它，则成本为 O(n)。 由于操作次数高达 10^6，并且集合大小可能也很大，这很快就变得不可行。 

关键的观察结果是，答案仅取决于更新元素周围的局部结构。 插入或取出时`x`，仅存在`x-1`和`x+1`很重要。 这两个邻居决定是否`x`开始一个新段、合并两个段或扩展现有段。 

这将全局结构问题简化为局部邻接维护问题。 哈希集提供 O(1) 平均成员资格检查，允许我们在每次操作的恒定时间内更新段计数。 

我们维护一个变量`segments`代表当前连续运行的数量。 对于插入，我们检查是否`x-1`和`x+1`存在。 对于删除，我们基于相同的邻居执行逆调整。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(q·n) | O(q·n) | O(n) | 太慢了 |
 | 最优（哈希集）| O(q) 平均 | O(n) | 已接受 |

 ## 算法演练

 我们维护一个集合`S`和一个整数`segments`。 

1. 初始化`S`作为空和`segments = 0`。 没有元素意味着没有段。 
2. 对于插入`+ x`，检查是否`x`已经缺席了。 让`left = (x-1 in S)`和`right = (x+1 in S)`。 
3. 如果两者都`left`和`right`是假的，那么`x`形成一个全新的孤立段，因此增加`segments`1. 这种情况发生在`x`不连接到任何现有的东西。 
4. 如果两者都`left`和`right`为真，插入`x`将两个先前独立的段连接成一个更大的段，因此减少`segments`1. 这是两次合并同时发生的唯一情况。 
5. 如果恰好是其中之一`left`或者`right`是真的，那么`x`扩展现有段而不更改总计数。 
6. 插入`x`进入`S`。 
7. 删除`- x`,再次计算`left = (x-1 in S)`和`right = (x+1 in S)`移除之前。 
8. 如果两者都`left`和`right`是假的，删除`x`删除一个独立的段，所以减少`segments`1.
 9. 如果两者都`left`和`right`是真的，删除`x`将一段分成两段，因此增加`segments`1.
 10. 如果恰好存在一个邻居，则删除仅缩小一个段，而不改变段计数。 
11. 删除`x`从`S`。 

### 为什么它有效

 集合中的每个整数恰好是一个最大连续块的一部分。 块数量改变的唯一方式是当在影响块之间邻接的位置插入或删除元素时`x-1`,`x`， 和`x+1`。 集合中的所有其他元素保持相同的相对顺序和连接性。 由于每个操作仅修改一个点，因此组件的唯一结构更改是涉及最多两个相邻连接的局部合并或拆分。 更新规则完全枚举所有可能的本地配置，确保每次操作后段计数保持准确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    q = int(input())
    S = set()
    segments = 0
    out = []

    for _ in range(q):
        c, xs = input().split()
        x = int(xs)

        if c == '+':
            left = (x - 1) in S
            right = (x + 1) in S

            if not left and not right:
                segments += 1
            elif left and right:
                segments -= 1

            S.add(x)

        else:
            left = (x - 1) in S
            right = (x + 1) in S

            if not left and not right:
                segments -= 1
            elif left and right:
                segments += 1

            S.remove(x)

        out.append(str(segments))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    main()
```该实现依赖于Python`set`支持平均 O(1) 成员资格查询`x-1`和`x+1`。 关键的微妙之处在于，在修改集合之前执行邻居检查。 如果我们先插入或删除，我们将丢失正确的邻接信息并对转换进行错误分类。 

这`segments`变量仅在两种极端情况下更新：孤立元素或双连接。 中间情况隐式保留计数，这就是为什么不需要显式处理的原因。

 ## 边缘情况

 一种微妙的情况是当插入连接两个独立的段时。 例如，从`{1}`和`{3}`, 插入`2`必须减少段数。 

输入：```
+ 1
+ 3
+ 2
```插入前`2`,`S = {1, 3}`段数为2。处理时`2`, 两个邻居`1`和`3`存在。 该规则触发减少，给出段 = 1，这正确反映了`{1,2,3}`作为一个区间。 

另一种情况是删除桥元素。 

输入：```
+ 4
+ 5
+ 6
- 5
```在`{4,5,6}`，段数为1。删除`5`看到两个邻居都存在，因此段增加到 2，产生`{4}`和`{6}`。 该算法正确地捕获了分割，因为它在删除之前评估了邻接性。
