 ## 解决方案
 ## 问题理解

 我们得到一个描述另一个隐藏数组的累积 gcd 信息的序列。 对于每个位置$i$，值$b_i$等于第一个的 gcd$i$未知数组的元素$a$。 我们的任务是重建任何有效的数组$a$可以产生给定的序列$b$，或者确定不存在这样的数组。 

关键的困难在于 gcd 是累积的并且一般来说是不可逆的。 每个$b_i$同时约束所有先前的元素，而不仅仅是$a_i$。 因此，重建必须尊重前缀之间的全局一致性条件。 

约束允许最多$10^6$测试用例中的总元素，这强烈表明$O(n)$或者$O(n \log n)$每个测试解决方案。 任何涉及成对一致性检查或对候选人进行回溯的方法都会太慢。 

当 gcd 序列不是单调非递增时，就会出现微妙的失败情况。 由于前缀 gcd 只能保持不变或减少，因此任何增加都会立即使序列变得不可能。 例如，如果$b = [6, 10]$，这是无效的，因为较大前缀的 gcd 不能变得大于前一个前缀的 gcd。 

另一个常见的错误是假设设置$a_i = b_i$总是有效。 例如，如果$b = [6, 3]$， 环境$a = [6, 3]$产生前缀 gcd$6, 3$，这很好，但如果后来的约束通过可分结构强制矛盾，则幼稚的分配可能会在更复杂的序列中失败。 

## 方法

 暴力重建会尝试猜测每个$a_i$并验证生成的前缀 gcd 是否与给定的匹配$b$。 这涉及到为每个候选数组重复重新计算 gcd，成本$O(n)$每次验证，并且由于每个位置有无限多个候选选择，该方法甚至没有被明确定义为有效的搜索空间。 

更结构化的蛮力将尝试构建$a$增量地同时保持所有前缀 gcd 约束。 在步骤$i$，我们会尝试值$a_i$这样$\gcd(b_{i-1}, a_i) = b_i$，但测试所有有效倍数$b_i$最多$10^9$导致搜索不可行。 

关键的见解是扭转观点。 而不是治疗$b_i$作为对未知前缀行为的约束，我们构造一个候选数组$a$_强制_前缀 gcd 演化完全匹配。 我们观察到前缀 gcd 从$b_{i-1}$到$b_i$，新元素必须引入正确的 gcd 缩减，并且足以确保：$$\gcd(b_{i-1}, a_i) = b_i$$这将问题简化为在简单的 gcd 约束下独立构造每个元素。 

我们可以通过选择来强制执行此操作：$$a_i = b_i$$每当$b_i$划分$b_{i-1}$， 自从：$$\gcd(b_{i-1}, b_i) = b_i$$然而，仅此还不足以确保完整前缀演化的正确性，因为我们还必须确保先前的元素与所有未来的 gcd 值保持一致。 

更深入的观察是，一旦前缀 gcd 是$b_i$，每个较早的前缀的 gcd 必须与其对应的完全相等$b_j$。 这迫使形成一个可分链：$$b_{i-1} \bmod b_i = 0$$必须为所有人保留$i$。 如果这在任何地方失败，则不存在解决方案。 

一旦链有效，我们可以通过设置构造一个简单的有效数组：$$a_i = b_i$$这是可行的，因为前缀 gcd 计算变为：$$\gcd(b_1, b_2, \dots, b_i) = b_i$$只要每一步都是除数转换。

 | 最佳 | O(n) | O(1) 额外 | 已接受 |

 ## 算法演练

 1. 对于每个测试用例，读取数组$b$。 我们将其视为必须由某个数组精确实现的所需前缀 gcd 序列$a$。 
2.验证每个前缀gcd不得增加的必要条件。 具体来说，检查每个$i > 1$,$b_{i-1} \ge b_i$。 如果失败，立即输出-1，因为添加更多元素时 gcd 值无法增加。 
3. 通过检查来验证整个序列的整除性一致性$b_{i-1} \bmod b_i = 0$为所有人$i > 1$。 这确保了每个步骤都可以作为来自前一个前缀的有效 gcd 转换而获得。 
4. 如果所有检查都通过，则通过设置构造答案$a_i = b_i$对于所有职位。 此选择保证每个前缀 gcd 与给定序列完全匹配。 
5. 输出构造好的数组。 

选择这种结构的原因是它直接将所需的前缀结构嵌入到数组本身中，从而无需任何隐藏的调整或辅助值。 

### 为什么它有效

 每一步$i$，我们认为构造数组的前缀 gcd 等于$b_i$。 归纳开始于$i = 1$， 在哪里$a_1 = b_1$。 假设前缀 gcd 达到$i-1$等于$b_{i-1}$，添加$a_i = b_i$结果是：$$\gcd(b_{i-1}, b_i) = b_i$$因为可分性条件。 这确保了前缀 gcd 转换在每一步都与给定序列完全匹配，因此构造的数组是有效的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        b = list(map(int, input().split()))

        ok = True
        for i in range(1, n):
            if b[i-1] < b[i] or b[i-1] % b[i] != 0:
                ok = False
                break

        if not ok:
            out.append("-1")
        else:
            out.append(" ".join(map(str, b)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现直接强制执行算法中导出的结构条件。 第一个循环检查单调不增加行为和整除性，这是有效性所需的唯一约束。 验证通过后，重复使用$b$因为输出数组就足够了。 

一个常见的实现陷阱是忘记了可分性检查而只强制执行单调性。 这会错误地接受像这样的序列$[6, 4]$，它不能由任何数组产生，因为没有数字可以将 gcd 从 6 减少到 4。

 ## 边缘情况

 一个关键的边缘情况是序列减少但不是按整除性减少。 用于输入$b = [12, 8]$，算法检查$12 \bmod 8 \neq 0$，立即拒绝。 这正确地反映了不可能性，因为没有数组可以有前缀 gcd 12 后跟 8。 

另一个边缘情况是恒定序列，例如$b = [7, 7, 7]$。 该算法简单地验证单调性和整除性并输出$a = [7, 7, 7]$，保留所有前缀的 gcd。 

第三种边缘情况是长度为一的序列。 任何单个值都是有效的，因为它直接定义数组并且不存在一致性约束。
