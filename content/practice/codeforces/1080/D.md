 ## 解决方案
 ## 问题理解

 我们从边长为 2 的幂的正方形网格开始，具体来说$2^n \times 2^n$。 唯一允许的操作是取任何现有的边长方形$a$并将其分成四个相等的边长较小的正方形$a/2$。 这个过程可以重复，但是一旦我们到达单位正方形，就不可能进一步分裂。 

这个过程不是任意的：我们必须准确地执行$k$总共这样的分裂，可能会在不同的时间选择不同的方格进行分裂。 完成所有操作后，我们看看最终将大正方形分割成各种大小的等轴对齐的较小正方形。 

关键的约束是最终配置的结构性约束。 我们关注一些固定边长的正方形$a$。 在这些方格中，整个棋盘的左下角和右上角的方格必须通过一条仅通过边的相邻方格移动的路径连接$a$，其中邻接意味着共享完整的一面。 

任务是决定是否可以准确地执行$k$分裂，以便这种连接条件适用于某些选择$a$。 如果可能的话，我们还必须输出使用的比例，表示为$\log_2(a)$。 

限制条件非常大：$n$可以达到$10^9$和$k$最多$10^{18}$，这排除了任何网格模拟或显式构造。 任何解决方案都必须将问题简化为关于分割如何在完整二进制分解的级别之间分布的算术推理。 

一种简单的方法是模拟分裂，维护正方形的优先级结构并尝试所有操作的分布。 这会立即失败，因为在几个级别之后甚至存储网格也变得不可能：$n$完全分割，单位方块的数量是$4^n$，这远远超出了任何限制，即使对于小$n$。 甚至对每个操作进行推理也是不可能的，因为$k$太大了。 

一个微妙的边缘情况是$k$与完全递归分解中的可用分割数相比，它很小。 例如，与$n = 1$，我们有一个$2 \times 2$网格。 一次分割产生四个单位方块，并且已经固定了所有结构。 如果$k$在达到单位大小之前超过所有方块的最大可能分割数，该过程将变得不可能，因为无法再应用任何操作。 

另一个微妙的情况是，不同的值$a$对应于分解的不同“级别”。 选择$a = 2^d$意味着我们正在看一个网格$2^{n-d} \times 2^{n-d}$块，而连接性要求完全取决于该级别上是否有足够的块可以保持完整，同时将分割分布到其他地方。 

## 方法

 暴力破解的想法是模拟所有可能的分裂操作序列。 每个状态都是一组正方形大小，每个操作都会选择一个正方形并用四个较小的正方形替换它。 这就形成了一个巨大的分支过程。 即使我们通过对称性进行修剪，状态数量也会呈指数增长$k$，因为每次分裂都会引入未来分裂可能发生的多个新选择。 状态空间很快就会变得棘手。 

关键的观察是，我们实际上不需要网格的几何形状，只需要每个尺度上存在多少个正方形。 每次分割都会使方格总数恰好增加 3，因为一个方格变成了四个。 所以之后$k$运算时，方格总数是固定的：$1 + 3k$。 问题归结为我们是否可以以某种方式将这些方块分布在各个级别上，从而保留从左下角到右上角的某种选定比例的连接“走廊”。 

现在考虑确定目标规模$a = 2^d$。 在这种规模下，董事会在概念上是一个$2^{n-d} \times 2^{n-d}$块网格。 为了使路径存在，我们需要在对角之间建立一个由该规模的块组成的连接链。 这迫使完整块的最小“主干”，而所有额外的分裂只能在不破坏连接的情况下细化其他区域。 

关键的见解是连接性要求强制采用非常特定的结构量：在水平上$d$，网格有$2^{n-d}$在每个方向上都有步骤，因此任何路径至少需要$2(2^{n-d}-1)+1$如果我们用曼哈顿的术语来思考，那么街区的数量是固定的，但更重要的是，在这个规模和以上的街区数量必须沿着路径保持不分割。 这限制了全局可以执行的总拆分次数。 

反转视角使事情变得更简单。 想象一下，我们不是从一个正方形开始构建，而是将所有东西完全分割成单位正方形。 这对应于最大$4^n - 1$分裂，但我们只关心中间可达状态。 该过程相当于选择一个截止水平$d$并决定该级别以下的所有内容都完全展开，而一些较高级别的方块保持未扩展以形成骨干。 每一个选择$d$在完整的二叉四叉树上引入特定所需的分裂数量，并且该数量是唯一确定的。 

所以问题就变成了检查是否存在一个整数$d \in [0, n]$使得在水平面上建立连接走廊所需的分割数量$d$正是$k$。 这个所需的数字结果形成了一个简单的单调函数$d$，所以我们可以计算它并测试所有可能的级别。

 | --- | --- | --- | --- |
 | 蛮力模拟 | 指数在$k$| 指数| 太慢了|
 | 基于级别的计数 |$O(n)$或者$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们根据四叉树的级别重新解释该过程。 

1.我们认为选择一个值$d$意味着我们希望路径使用边的平方$2^d$。 这将初始网格划分为$2^{n-d} \times 2^{n-d}$大细胞。 
2.对于固定的$d$，如果我们完全细化所有单元格（除了那些形成这些大单元格从左下角到右上角的最小连接路径的单元格），我们计算需要多少次分割。 
3. 水平上的每个单元格$i$对应于一个可以保持完整或在水平面上分成四个的正方形$i-1$。 从水平完全扩展单元所贡献的分裂总数$i$下降到水平$0$是$4^i - 1$。 
4. 我们在级别上构建最小路径$d$。 该路径恰好包含$2^{n-d+1}-1$曼哈顿路径结构中对角之间的单元格。 
5.所有不在这条路径上的单元格都完全展开到级别0，而路径单元格保持在级别$d$。 这给出了确定性的分割计数：

 分割总数是完全扩展总数减去沿路径保存的扩展数。 
6. 我们通过对所有级别的贡献求和并减去路径的贡献来有效地计算该值，这避免了完全枚举。 
7.我们迭代所有可能的$d$，如果有的话正好产生$k$，我们输出 YES 并且$d$。 

### 为什么它有效

 关键的不变量是每个有效的最终配置都可以通过选择一个级别来唯一地描述$d$其中存在未分割的正方形的连接主干，而其他一切都完全扩展。 与此结构的任何偏差要么会断开所需的路径，要么会引入冗余的部分扩展，这些扩展可以在树上向下或向上推，而无需更改连接性，但会更改拆分数量。 这种标准化将所有有效配置折叠为单个参数$d$，使每个选择的操作计数具有确定性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_ops(n, d):
    # number of cells per side at level d
    # grid is 2^(n-d)
    m = n - d
    # total cells at level d grid
    total_cells = 1 << (2 * m) if m < 60 else None

    # If m is large, we avoid direct computation by noting overflow cases
    # We instead compute using geometric series reasoning:
    # total full expansion cost from level n to 0 is (4^n - 1)/3 in split-units
    # but we only need relative comparison, so we compute difference form.

    # compute full expansion cost for a single root down to level d
    def full(x):
        return (1 << (2 * x)) - 1

    full_all = full(n)
    full_cell = full(d)

    # number of cells on path at level d
    path_len = (1 << (m + 1)) - 1

    # each path cell saves full(d) expansion
    saved = path_len * full_cell
    return full_all - saved

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())

        # small n safety: try all d
        ans = -1
        for d in range(n + 1):
            if count_ops(n, d) == k:
                ans = d
                break

        if ans == -1:
            print("NO")
        else:
            print("YES", ans)

if __name__ == "__main__":
    solve()
```代码将问题视为选择级别$d$最终路径方块。 功能`full(x)`编码从侧面完全扩展正方形需要多少分割$2^x$缩小到单位大小，即$4^x - 1$。 这个想法是，配置相当于完全扩展除了连接的骨干网之外的所有内容$d$，所以我们减去所有骨干细胞的扩张成本。 

迭代结束$d$检查路径的每个可能的比例。 一旦找到匹配的分割计数，该比例就有效。 

一个微妙的实现细节是直接求幂$4^n$对于大的来说是不可能的$n$，因此所有计算都依赖于位移位和仔细的减法结构。 Python 整数处理较大的值，但实际上，人们会逐步限制或比较，以避免不必要的溢出计算。

 ## 边缘情况

 对于$n = 0$，网格是单个单元格，无法进行任何操作。 任何$k > 0$必须立即失败，算法会处理这个问题，因为没有有效的$d$产生积极的分裂。 

对于非常大的$k$， 例如$10^{18}$，计算值随着减小而增长得非常快$d$，因此大多数情况很快就会失败，只有非常粗略的级别仍然是可行的候选者。 这确保了搜索$d$在实践中提前终止。
