 ## 解决方案
 ## 问题理解

 Petya 正在为一个聚会准备请柬，每张请柬都会消耗固定数量的彩色纸张。 每张邀请函需要 2 张红纸、5 张绿纸和 8 张蓝纸。 如果有$n$朋友们，那么$n$必须制作邀请函，因此总需求与$n$: 红色需求是$2n$，绿色需求为$5n$，蓝色需求为$8n$。 

本店不出售单张。 相反，每次购买都是单一颜色的笔记本，并且每个笔记本都包含准确的内容$k$床单。 笔记本不能在购买时拆分成多种颜色或部分使用，因此决定纯粹是必须购买每种颜色的多少本完整笔记本，以便总页数足够。 

输出是所有三种颜色的笔记本的最小总数，保证有足够的纸张用于所有邀请。 

约束条件允许两者$n$和$k$一样大$10^8$，这会立即排除任何在笔记本或工作表上迭代的模拟。 任何方法都必须在恒定时间内计算出答案。 该结构是纯算术的，因此对数或线性解是不必要的，并且会显得过大。 

当某种颜色的总张数不能被整除时，会出现微妙的边缘情况$k$。 例如，如果$n = 1$,$k = 5$，并且红色需求为 2，则红色需求一台笔记本就足够了，但如果需求为 6，则即使 6 接近 5，也需要两台笔记本。这种上限行为是错误的主要来源，特别是在使用整数除法而不向上舍入的情况下。 

## 方法

 一种幼稚的策略是模拟为每种颜色一张一张地购买笔记本，直到达到所需的张数。 对于每种颜色，我们可以重复添加$k$并计算需要添加多少张才能达到$2n$,$5n$， 和$8n$。 这是正确的，因为它模仿了购买笔记本电脑的物理过程，但在最坏的情况下$n = 10^8$，所需的总数约为$10^9$床单。 如果$k = 1$，这会导致每种颜色进行数十亿次迭代，这远远超出了时间限制。 

关键的观察是每种颜色都是独立的。 红色所需的笔记本数量不影响绿色或蓝色。 一旦我们计算出一种颜色所需的总张数，问题就简化为标准的上限划分：有多少组尺寸$k$需要满足总需求。 

这将整个问题简化为三个独立的整数上限除法，然后进行求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(n \cdot \max(2,5,8))$最差的图纸级建模 |$O(1)$| 太慢了|
 | 最佳上限算法 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算每种颜色所需的总张数：红色为$2n$, 绿色是$5n$，蓝色是$8n$。 此步骤隔离需求，以便可以独立处理每种颜色。 
2. 对于每种颜色，确定需要多少个笔记本。 由于每个笔记本都贡献了准确的$k$张数，所需数量为大于或等于需求除以的最小整数$k$。 这是一个天花板划分。 
3. 使用整数算术计算上限除法：$(x + k - 1) // k$。 这避免了浮点运算并正确处理以下情况$x$正好可以整除$k$。 
4. 将三个结果相加，输出笔记本总数。 

### 为什么它有效

 每种颜色都表现为一个单独的包装问题：我们使用固定尺寸的容器来满足固定的需求。 任何解决方案都必须至少满足需求，因此对于每种颜色，笔记本电脑的数量必须满足$ans \cdot k \ge x$。 满足这个不等式的最小整数正是上限除法。 由于颜色不相互作用，因此将这些最小独立解决方案相加即可得出全局最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())

    red = 2 * n
    green = 5 * n
    blue = 8 * n

    def need(x):
        return (x + k - 1) // k

    print(need(red) + need(green) + need(blue))

if __name__ == "__main__":
    solve()
```该代码直接将数学结构转换为三个独立的计算。 辅助函数`need`使用整数算术安全地实现上限除法，这避免了浮点精度问题并确保边界处的正确性`x`是的倍数`k`。

 ## 边缘情况

 当$k = 1$，每张纸必须来自单独的笔记本。 用于输入$n = 1, k = 1$，需求为 2、5 和 8，因此结果是 15 台笔记本。 该算法可以正确处理这个问题，因为$(x + 0) // 1 = x$，保留精确计数。 

什么时候$k$非常大，比如$k \ge 8n$，所有需求都适合每种颜色的一个笔记本。 为了$n = 10, k = 100$、红色、绿色和蓝色各需要 1 个笔记本，产生输出 3。上限公式正确折叠，因为每个需求除以$k$小于 1 但向上舍入为 1。 

当$k$精确划分所有需求，例如$n = 2, k = 2$，红色为 4，绿色为 10，蓝色为 16，均可被 2 整除。该公式简化为精确整数除法，无需舍入，避免了计数过多。
