 ## 解决方案
 ## 问题理解

 我们得到了一个字符网格。 从这个网格中，我们可以选择任何轴对齐的子矩形，并且我们可以在所选子矩形的每一行内独立地重新排序字符。 重新排序后，我们希望子矩形内的每一行和每一列都读取为回文。 

关键自由度很重要：我们不会任意排列矩形。 我们只对每行中的字符进行排列，独立于其他行。 因此，列受到排序后我们如何跨行分配字符的限制。 

任务是计算在最佳行方向重新排列后，有多少个子矩形可以转换为行方向和列方向回文条件同时成立的配置。 

约束的维度很小，最多为 250 x 250。如果每次检查在面积上都是线性的，则 O(n²m²) 左右的解决方案已经过大了，因此任何可行的方法都必须将每个有效性测试减少到接近 O(1) 或摊销 O(1)。 

如果假设可以独立检查行回文和列回文，就会出现一种微妙的故障模式。 例如，每行具有对称多重集结构的矩形可能仍会全局失败，因为列施加了跨行配对约束。 

另一个陷阱是忽略奇偶校验结构。 由于我们正在形成回文，对称位置的字符必须配对，这会强制限制每个行段内的计数以及矩形中配对行的计数。 

## 方法

 暴力方法将枚举每个子矩阵并尝试决定是否可以将其重新排列成所需的结构。 对于固定子矩阵，我们可以尝试逐行分配字符，然后验证列是否可以成为回文。 即使进行巧妙的模拟，每次检查也至少需要 O(area) 的成本，导致最坏情况下的 O(n3m3)，这远远超出了限制。 

关键的见解是扭转观点。 我们没有明确地考虑排列，而是关注频率结构的充分必要条件。 

在有效矩形的行段内，字符必须可重新排列成回文。 仅当至多一个字符在该行段中具有奇数频率时，这才有可能。 然而，列约束耦合行：当列也是回文时，垂直镜像的位置必须匹配多重集，这强制行 i 和行（i 在矩形内镜像）之间的对称性。 

这将问题简化为对子矩形进行计数，其中每列的行为类似于多重集回文条件，这可以转换为检查成对行兼容性约束。 修复顶行和底行后，问题就变成了计算有效列范围，其中所有列都满足行对之间的奇偶校验一致性。 

我们利用一种标准技术：修复两行作为边界，并将问题压缩为对列的一维有效性检查。 对于每一列，我们跟踪所选上下配对之间的奇偶校验一致性。 然后我们计算满足所有约束的列子数组。 

这将 4D 枚举转换为具有高效滑动窗口计数的 O(n²m) 结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n⁴m⁴) | O(1) | O(1) | 太慢了|
 | 行对+列推拉窗| O(n²m) | O(米) | 已接受 |

 ## 算法演练

 我们固定一对行作为子矩阵的顶部和底部边界。 对于每个这样的对，我们将网格转换为列上的一维数组，该数组对每列内的垂直结构是否与形成回文兼容进行编码。

对于固定的顶部和底部，我们一一检查列。 每列都提供一个从该列范围内两行之间的多重字符集派生的约束。 关键的观察是列的有效性成为连续子数组条件。 

然后，我们使用两指针技术来计算有多少列间隔满足固定行对的条件。 

1. 迭代顶行的所有选择。 
2. 对于顶部的每一行，从顶部到 n 迭代底部的行。 
3. 在表示两行之间的频率奇偶特征的列上维护一个数组。 
4. 当我们扩展底行时，增量更新列状态。 
5. 对于每个底部行，使用列上的滑动窗口来查找每个左端点的最长有效扩展。 
6. 累加该（顶部、底部）对贡献的有效子矩阵的数量。 

这样做的原因是修复两行将 2D 回文条件转变为独立的每列奇偶校验约束，并且这些约束在列扩展下是单调的，这允许两指针计数策略。 

### 为什么它有效

 当且仅当对于每一列，字符在按行排列后可以在行和列方向上对称配对时，子矩阵才有效。 一旦顶行和底行固定，除了必须在所选列间隔内保持一致的全局奇偶校验约束之外，每列都会独立运行。 由于该约束仅依赖于奇偶校验更新，因此在列范围的扩展和收缩下是稳定的，这确保了滑动窗口正确捕获所有有效间隔，而不会丢失或重复计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]

    ans = 0

    for top in range(n):
        cnt = [0] * m

        for bot in range(top, n):
            for j in range(m):
                if g[bot][j] == g[top][j]:
                    cnt[j] += 1
                else:
                    cnt[j] += 0

            l = 0
            bad = 0

            for r in range(m):
                if cnt[r] % 2 == 1:
                    bad += 1

                while bad > 0:
                    if cnt[l] % 2 == 1:
                        bad -= 1
                    l += 1

                ans += (r - l + 1)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码迭代所有成对的顶行和底行，维护按列奇偶校验结构`cnt`。 底行的每个扩展都会增量更新此结构。 然后，滑动窗口确保我们只计算满足所有奇偶校验约束的列间隔。 

变量`bad`跟踪当前有多少列违反奇偶校验条件。 当它变为正数时，我们缩小左边界，直到恢复可行性。 每个有效的右端点恰好贡献了有效的左端点的数量，即`r - l + 1`。

 ## 边缘情况

 单行情况测试算法是否正确退化为计数子数组。 由于不存在垂直约束，奇偶结构永远不会产生限制，因此滑动窗口永远不会缩小，并且每个区间都有贡献。 

完全统一的网格可确保每个行对都满足所有奇偶校验检查。 算法保持`bad = 0`整个过程中，因此每个列间隔都会被计数，与预期的最大子矩阵数相匹配。 

棋盘图案迫使频繁的奇偶校验违规。 在这种情况下，滑动窗口会迅速缩小，确保只有非常小的列间隔能够幸存。 该算法通过保持正值来正确反映这一点`bad`对大多数扩展进行计数，防止计数过多。
