 ## 解决方案
 ## 问题理解

 我们得到一个非常大的无限序列，其中每个位置都有一个仅基于其索引的确定性值。 该值交替符号并线性增长：位置 1 贡献 -1，位置 2 贡献 +2，位置 3 贡献 -3，依此类推。 一般来说，位于位置的元素$i$是$i$乘以每一步都会翻转的符号。 

每个查询都要求连续段中的值的总和$[l, r]$。 由于查询最多可达 1000 个，索引最多可达 10 亿个，因此我们无法直接构造数组或迭代范围。 任何涉及查询范围中每个元素的解决方案都会降级为$10^9$在最坏的情况下进行操作，这在时间限制内是不可能的。 

关键的挑战是，虽然序列很大，但其结构极其规则，这表明存在封闭形式的前缀和。 

当范围仅跨越奇数或偶数位置时，会出现微妙的边缘情况。 例如，考虑$[1, 1]$,$[2, 2]$， 或者$[1, 2]$。 如果幼稚的实现依赖于增量模拟而没有仔细的索引，则可能会错误地处理符号交替。 另一种故障模式是符号模式中的相差一错误：是否$i$Even 是正数还是负数必须在前缀和范围计算中保持一致。 

## 方法

 暴力方法将通过迭代来独立计算每个查询$l$到$r$并求和$a_i$。 这是正确的，因为它直接遵循定义。 但是，每个查询最多可以跨越$10^9$元素，因此即使是单个最坏情况的查询也已经太大了，并且最多可达$10^3$查询该方法是完全不可行的。 

序列的结构是关键的观察。 这些值交替符号，同时幅度线性增加。 这表明将序列分为偶数和奇数位置，其中每个子序列成为具有一致符号的简单算术级数。 

奇数位置贡献负值：at 位置$2k-1$，值为$-(2k-1)$。 即使职位也能贡献正值：在职位上$2k$，值为$+2k$。 每个子序列都有一个已知的算术级数前缀和公式，它允许我们计算总和$n$在恒定的时间内。 一旦前缀和可用，每个查询都会减少为减法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每个查询 O(r − l + 1) | O(1) | O(1) | 太慢了|
 | 最优前缀公式 | 每个查询 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 通过分离奇数和偶数索引的贡献来重写序列。 这很重要，因为每个部分都成为标准的算术级数而不是交替模式。 
2. 将前缀和导出到索引$n$。 我们计算最多存在多少个偶数和奇数项$n$，因为它们的计数因奇偶校验而略有不同。 
3. 计算偶数索引的总和。 连职位都是$2, 4, 6, \dots, 2k$。 他们的总和是：$$2(1 + 2 + \dots + k) = k(k+1)$$在哪里$k = \lfloor n/2 \rfloor$。 
4. 计算奇数索引的总和。 奇数位置是$1, 3, 5, \dots, (2k-1)$。 他们的贡献是负的：$$-(1 + 3 + 5 + \dots + (2k-1)) = -k^2$$在哪里$k = \lceil n/2 \rceil$。 
5. 将两部分组合起来得到前缀和$S(n)$。 然后每次查询$[l, r]$回答为：$$S(r) - S(l-1)$$这样做的原因是序列分解为两个具有不相交索引的独立算术序列。 前缀和尊重线性，因此我们可以单独对每个子序列求和并重新组合它们而无需交互。 

### 为什么它有效

 关键的不变量是每个索引恰好属于两个不相交的集合之一：偶数或奇数。 在每个集合上，价值函数变成具有固定步长和固定符号的纯算术级数。 由于前缀和在不相交的并集上是线性的，因此总前缀和恰好是两个独立前缀和的总和。 这保证了减去前缀值可以正确隔离任何段$[l, r]$没有重新计算或重叠错误。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def pref(n):
    if n <= 0:
        return 0

    k_even = n // 2
    even_sum = k_even * (k_even + 1)

    k_odd = (n + 1) // 2
    odd_sum = k_odd * k_odd

    return even_sum - odd_sum

q = int(input())
for _ in range(q):
    l, r = map(int, input().split())
    print(pref(r) - pref(l - 1))
```功能`pref(n)`通过分成偶数和奇数贡献来计算从索引 1 到 n 的序列之和。 对于偶数指数，我们使用第一个指数之和的公式$k$偶数。 对于奇数索引，我们计算第一个索引的总和$k$奇数并将其取负。 

每个查询都使用标准的前缀减法技巧，这是安全的，因为前缀函数是明确定义的$n = 0$通过基本情况。 

这里的一个常见错误是混淆了奇数和偶数位置的计数。 另一个是忘记了奇数位置会产生负值，因此必须在求和之后而不是之前应用符号。

 ## 边缘情况

 关键的边缘情况是范围从 1 开始，因为前缀逻辑取决于处理$l - 1 = 0$。 例如，在$[1, 1]$，计算变为$S(1) - S(0)$。 该实现显式返回 0$S(0)$，确保正确性，无需特殊的大小写查询。 

另一个边缘情况是当$l = r$。 该算法通过减去相邻前缀来简化为单值查找，并且算术公式仍然正确运行，因为偶数和奇数贡献都是跨边界连续的。 

第三种情况是当$l$和$r$有不同的奇偶校验，例如$[2, 3]$。 这里，该段从正偶数项交叉到负奇数项。 前缀函数不关心这个边界，因为两个贡献都已经全局编码在$S(n)$。
