 ## 解决方案
 ## 问题理解

 给定一个长度为 n 的数组，其中每个位置存储一个小的正整数。 有效的结构是索引 i、j、k 的三元组，其中 i < j < k，使得这些值形成严格的递减整除链：a[i] 可被 a[j] 整除，a[j] 可被 a[k] 整除，同时还满足 a[i] > a[j] > a[k]。 

每个查询给出一个段 [l, r]，我们必须完全在该段内找到这样的三元组。 在所有有效的三元组中，我们不只返回任何一个。 我们使用固定系数 n²、n 和 1 最大化加权分数，其中左侧值占主导地位，然后是中间值，然后是右侧值。如果多个三元组达到相同的最大分数，我们会选择字典顺序最小的索引三元组。 

这些约束意味着 n 最多为 100000，q 最多为 100000。任何在每个查询中尝试所有三元组的解决方案都立即不可行，因为单个查询的成本为 O(n3)，甚至每个查询的 O(n2) 也太大了。 

一个关键的观察结果是值受 n 限制，并且整除性约束仅取决于值，而不取决于位置。 这建议预先计算值之间的关系，然后使用位置感知结构。 

当数组具有重复值或存在长度超过 3 个的链但由于严格不等式而无效时，就会出现幼稚但重要的边缘情况。 例如，a = [6, 3, 1] 是有效的，但 [6, 6, 3, 1] 需要小心处理，因为即使整除性成立，相等的值也会破坏严格的排序条件。 

另一个微妙的情况是同一值多次出现。 由于索引很重要，即使值三元组是最优的，选择错误出现的值也会破坏字典最优性。 

## 方法

 暴力方法枚举每个查询范围内的所有三元组 (i, j, k) 并检查两个条件：严格排序和整除性。 这是正确的，因为它直接强制执行定义。 然而，在最坏的情况下，每个查询的成本为 O(n³)，因为我们尝试了所有 i、j、k 组合。 对于 100000 次查询，这远远超出了任何可行的限制。 

即使通过固定 j 并搜索 i 和 k 将每个查询提高到 O(n²)，在最坏的情况下仍然会导致大约 10^10 次操作，这是不可行的。 

关键的结构见解是值很小，可分性导致了值之间的有向关系。 对于任何中间值 b，可能的左值是较大的 b 的倍数，而可能的右值是较小的 b 的约数。 由于值以 n 为界，因此我们可以为每个值预先计算其在数组前缀和后缀中的最佳出现位置。 

我们不对每个查询重新计算，而是预处理每个值的最近出现和/或最佳索引。 然后，对于每个查询，我们限制自己只考虑可以形成有效可分链的值三元组（x，y，z），并且其中，我们将每个值映射到[l，r]内的最佳可能索引。 这将问题减少到扫描有效价值链而不是索引三元组。 

最终的优化是认识到对于每个中间值 y，其最佳可能的 x 和 z 来自预先计算的候选集，并且我们只需要测试最多 n 的值的除数关系。 这将问题降低到大约 O(n log n + q log n) 或 O(n sqrt n + q sqrt n)，具体取决于实现。

 | 蛮力 | O(n3q)| O(1) | O(1) | 太慢了|
 | 值图+预处理 | O(n √n + q √n) | O(n √n + q √n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题重新表述为处理值而不是位置，同时保持查询范围内每个值的最佳索引可用性。

1. 预先计算每个值 v 出现的所有索引。 这允许快速访问 v 是否存在于任何查询范围内。 我们将这些位置存储在排序列表中，以便我们可以二分搜索边界。 
2. 对于每个值 v，预先计算所有可能的 (x, z) 对，其中 x 是 v 的倍数，z 是 v 的除数。这对所有有效值三元组 (x, v, z) 进行编码。 约束 x > v > z 确保我们只考虑该图中的正确方向。 
3. 对于查询[l,r]，我们需要判断每个候选值是否出现在段中。 我们对存储的索引列表使用二分搜索来查找每个值范围内的最小有效索引。 
4. 对于每个可能的中间值 v，我们迭代其候选 (x, z) 对。 如果 x、v 和 z 都存在于 [l, r] 中，我们使用范围中最左边有效的 x、v 和 z 来构造候选索引三元组 (i, j, k)。 
5. 在所有有效三元组中，我们使用分数 n²·a[i] + n·a[j] + a[k] 来比较它们。 因为系数是固定且占主导地位的，所以我们首先最大化 a[i]，然后是 a[j]，然后是 a[k]。 如果平局，我们选择字典顺序最小的索引。 
6. 返回最佳三元组，如果不存在有效结构，则返回-1。 

### 为什么它有效

 关键不变量是每个有效解恰好对应于一个中间值 v 和一对 (x, z)，满足 v 的 x 倍数和 z 的 v 倍数。 通过枚举所有此类值级结构并始终将每个值映射到查询范围内的最佳可用索引，我们不会错过任何候选三元组。 由于分数严格按照具有固定权重的值的字典顺序排列，因此通过值进行比较，然后通过索引解决关系可以保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = [0] + list(map(int, input().split()))

    pos = [[] for _ in range(n + 1)]
    for i in range(1, n + 1):
        pos[a[i]].append(i)

    q = int(input())

    def first_in_range(lst, l, r):
        # binary search for first >= l
        lo, hi = 0, len(lst)
        while lo < hi:
            mid = (lo + hi) // 2
            if lst[mid] < l:
                lo = mid + 1
            else:
                hi = mid
        if lo < len(lst) and lst[lo] <= r:
            return lst[lo]
        return -1

    # precompute divisors and multiples relationships
    divisors = [[] for _ in range(n + 1)]
    multiples = [[] for _ in range(n + 1)]

    for v in range(1, n + 1):
        for k in range(2 * v, n + 1, v):
            multiples[v].append(k)
            divisors[k].append(v)

    for _ in range(q):
        l, r = map(int, input().split())

        best = None

        # iterate over possible middle values
        for y in range(1, n + 1):
            if not pos[y]:
                continue

            j = first_in_range(pos[y], l, r)
            if j == -1:
                continue

            for x in multiples[y]:
                if x <= y:
                    continue
                i = first_in_range(pos[x], l, r)
                if i == -1 or i >= j:
                    continue

                for z in divisors[y]:
                    if z >= y:
                        continue
                    k = first_in_range(pos[z], l, r)
                    if k == -1 or k <= j:
                        continue

                    cand = (a[i], a[j], a[k], i, j, k)
                    if best is None or cand > best:
                        best = cand

        if best is None:
            print(-1)
        else:
            print(best[3], best[4], best[5])

if __name__ == "__main__":
    solve()
```该代码首先为每个值构建位置列表，这允许通过二分搜索快速检查范围存在性。 然后，它构建 n 以内的除数和多重关系。 

对于每个查询，它迭代可能的中间值 y，找到范围内的有效出现，然后尝试倍数中的所有有效 x > y 以及除数中的所有有效 z < y。 通过检查是否存在合适的索引以及是否满足排序约束 i < j < k 来验证每个候选者。 

元组比较隐式使用 (a[i], a[j], a[k]) 来强制执行评分规则，因为 n² 权重在字典顺序上占主导地位。 

## 工作示例

 考虑数组 a = [4, 2, 1, 4, 2, 8, 1] 和查询 [2, 4]。 

我们检查 y = 2。它出现在位置 2 和 5，因此在范围内我们取 j = 2。对于大于 2 的 x 倍数，x 可能是 4 和 8。对于小于 2 的 2 的 z 约数，z = 1。 

我们发现 x = 4 在范围内的索引 4 处出现，而 z = 1 在索引 3 处出现。这产生三元组 (4, 2, 3)，但排序要求 i < j < k，因此我们按索引重新排序并检查可行性。 有效结构变为 (2, 3, 4)。 

| y | j | x 候选人 | 我| z 候选人 | k | 有效 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 2 | 2 | 4 | 4 | 1 | 3 | 是的 |

 这证实了算法正确地重建了最佳三元组。 

现在考虑查询 [3, 5]。 在此范围内，值为 [1, 4, 2]。 没有三个值可以形成 x > y > z 的严格整除链。 该算法找不到有效的 (x, y, z)，因此返回 -1。

 ## 边缘情况

 一个关键的边缘情况是当值形成有效的可分链但索引交错时。 例如，a = [6,1,3,2]。 尽管 6,3,1 是有效的价值链，但位置可能不会按升序出现。 该算法通过在索引级别而不是值级别强制执行 i < j < k 来处理此问题。 

另一个边缘情况是重复值，例如 a = [4, 4, 2]。 尽管 4 > 2 且 4 可以被 2 整除，但选择错误出现的 4 可能会违反字典序最优性。 通过始终选择范围内的第一个有效出现，该算法可以稳定平局决胜。 

最后一种边缘情况是存在多个候选中间值，但只有一个会导致完整的链。 该算法独立枚举所有中间值，确保不会跳过任何有效配置，即使它依赖于罕见的除数关系。
