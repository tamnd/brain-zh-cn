 ## 解决方案
 ## 问题理解

 我们得到一棵树，其顶点标记为 1 到 n。 每个顶点就像一个火车站，沿着两个车站之间的任何一条边移动只需花费一张票。 不同的是，门票的价格并不是全球固定的。 相反，每个车站 i 以每张票的价格 ai 出售车票，当您到达该车站时，您可以以该价格购买任意数量的车票。 

Maxim 从站点 1 开始，希望沿着边缘移动到达站点 n。 由于每次边遍历都会消耗一张票，因此所需的票总数恰好是所选路径上的边数，但成本取决于这些票的购买地点。 他可以在旅途之前或旅途中策略性地在更便宜的车站购买车票并继续使用。 

任务是计算保证 Maxim 可以沿着树从 1 移动到 n 所需的最小总成本（以韩元为单位）。 

约束 n ≤ 2 × 10^5 意味着任何解必须接近线性或线性算数。 尝试考虑所有节点对、所有路径或重新计算每个状态的最短路径的解决方案将太慢。 由于该结构是一棵树，因此任何两个节点之间都有一条独特的简单路径，这简化了运动的几何形状，但没有简化在哪里买票的经济决策。 

解释“在 i 购买任意数量的门票”会产生一个微妙的问题。 天真的读者可能认为只有起始节点很重要，但最佳策略取决于沿路径前缀看到的最低价格，而不仅仅是端点。 

例如，如果 a1 很大，但路径上的节点 3 的价格很小，那么在节点 1 购买所有东西显然不是最优的。 当廉价节点离开主路径但仍然位于在不正确的贪婪方法遍历期间临时进入的子树上时，会出现另一种失败情况。 任何假设每个节点有一个本地决策而不考虑全局路径结构的方法都可能失败。 

## 方法

 直接的暴力方法会尝试模拟从 1 到 n 的路径上所有可能的购票方式。 由于树中的路径是唯一的，唯一剩下的选择是在每个访问的顶点购买多少张票。 如果我们考虑长度为 k 的路径，天真的状态将跟踪我们还有多少张剩余门票以及它们是在哪个节点购买的，然后在每一步决定购买。 这很快就会呈指数增长，因为在每个节点我们可以选择任何购买金额，并且状态数量随着路径长度和可能的票证分布而增长。 

即使我们简化并假设我们在需要时只购买一张门票，我们仍然错过了关键的组合结构：门票是可互换的，只有最便宜的购买点才重要。 暴力破解会失败，因为它没有利用以最低价格节点购买多张门票主导任何延期购买策略的优势。 

关键的观察是，边缘所需的任何票都可以在从 1 到该点的路径上迄今为止看到的最便宜的车站购买。 一旦我们到达一个节点，我们实际上就可以访问路径上所有先前的站点，因此我们应该始终“继承”迄今为止遇到的最低价格。 这将问题转化为从 1 走到 n，同时保持沿路径的最小 ai 并为遍历的每条边支付该最小值。 

这将问题简化为找到从 1 到 n 的唯一路径，并对每条边求和至该边的最小节点价格。

 ## 算法演练

 我们依赖这样一个事实：在一棵树中，节点 1 和节点 n 之间只有一条路径。

1. 使用邻接表构建树。 这允许高效的遍历而无需重新计算连接。 树结构确保我们可以无环遍历。 
2. 从节点 1 运行 BFS 或 DFS 来计算每个节点的父指针。 这对于重建从 n 回到 1 的唯一路径是必要的。我们存储parent[v]和可选的深度[v]。 此步骤将无向树转换为节点 1 处的有根结构。 
3. 从节点 n 开始，使用父指针重建返回节点 1 的路径。 我们反转这个列表以获得从 1 到 n 的正确顺序的路径。 此步骤隔离了问题中唯一感兴趣的序列。 
4. 遍历从 1 到 n 的路径，同时维护一个运行变量 best_price，初始化为 a1。 在路径上的每个节点 v 处，更新 best_price = min(best_price, a[v])。 然后将 best_price 添加到每个边缘转换的答案中。 这模拟了每一条边都需要一张票的想法，而且我们总是在迄今为止看到的最便宜的车站购买它。 
5. 输出累计成本。 

关键的细节是我们增加每条边的成本，而不是每节点的成本。 由于沿着边缘移动仅消耗一张票，因此每次转换都必须使用截至该点的最佳可用价格收费一次。 

### 为什么它有效

 在从 1 到 n 的路径上的任何一点，迄今为止访问过的所有车站都可以购买可结转的车票。 未来边缘所需的任何车票都可以在迄今为止访问过的价格最低的车站购买，因为相对于购买而言，车票的消费时间没有限制。 因此，对于路径上的第 i 条边，最便宜的可能购买价格恰好是路径前缀中的节点中的最小 ai。 该算法保持此前缀最小值，并在每条边上应用一次，从而满足可行性和最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import defaultdict, deque

n = int(input())
a = list(map(int, input().split()))

g = defaultdict(list)
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append(v)
    g[v].append(u)

parent = [-1] * n
depth = [0] * n

q = deque([0])
parent[0] = -2

while q:
    u = q.popleft()
    for v in g[u]:
        if parent[v] == -1:
            parent[v] = u
            depth[v] = depth[u] + 1
            q.append(v)

path = []
cur = n - 1
while cur != -2:
    path.append(cur)
    cur = parent[cur]

path.reverse()

ans = 0
best = a[path[0]]

for i in range(1, len(path)):
    best = min(best, a[path[i]])
    ans += best

print(ans)
```BFS 部分从节点 1 构建一棵有根树，以便每个节点都知道返回根的唯一路径上的前任节点。 父数组初始化为 -1 以标记未访问的节点，并将节点 1 设置为特殊标记 -2，以便我们可以干净地停止重建。 

路径重建步骤使用父指针从节点 n 向后行走。 这是可行的，因为在一棵树中，每个节点在 BFS 树中都只有一个父节点，从而保证单个有效链回到 1。 

成本累加循环是核心逻辑。 我们将最好的价格维持为迄今为止沿着重建路径看到的最低价格。 每条边贡献最佳一次，反映每次遍历消耗一张票。 

一个常见的错误是对边上的 min(a[u], a[v]) 求和。 这是不正确的，因为最优价格取决于整个前缀最小值，而不仅仅是每条边的端点。

 该表显示了一旦到达更便宜的站点，所有未来的边缘都会从中受益，即使后来的站点价格昂贵。

 树：1连接到2和3，2连接到4，3连接到5，目标是5。

| 步骤| 节点| 最优惠价格| 增加成本| 总计 |

 ## 边缘情况

 一种边缘情况是最便宜的节点不在路径的起点或终点，而是在中间的某个位置。 例如，如果 a1 = 100、a2 = 1、a3 = 100，则正确答案必须在到达节点 2 后对两条边使用价格 1。算法在节点 2 处正确更新并向前传播。 

另一种情况是倾斜树，其中从 1 到 n 的路径很长，并且所有其他分支都不相关。 重建步骤确保我们完全忽略所有非路径节点，从而防止 BFS 遍历产生错误聚合。 

第三种情况是所有价格相同。 该算法仍然有效，因为最佳值永远不会改变，并且结果减少到 (n − 1) × a1，与路径中的边数匹配。
