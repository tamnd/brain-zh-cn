 ## 解决方案
 ## 问题理解

 我们有两种类型的资源：一种类型代表单个“A”块，另一种代表“C”块。 使用这些，我们想要构建一个三角形结构，其中第 i 行恰好包含一个 A，后跟 i−1 个 C。 

因此结构逐行增长。 第一行需要 1 A 和 0 C。第二行需要 1 A 和 1 C。第三行需要 1 A 和 2 C，依此类推。 每增加一行都会增加所需 C 的数量，同时始终只消耗一个 A。 

任务是确定最大高度 h，以便可以构建从 1 到 h 的所有行，而不会超过可用的 x A 和 y C。 

约束 x, y ≤ 10^18 强制解决方案避免对行进行任何模拟。 在最坏的情况下，线性构造需要多达 10^18 次迭代，这在一秒钟内是不可能的。 即使每次测试的对数开销也可以，但任何直接依赖于 h 的开销就不行了。 

一个微妙的失败案例来自于没有计划的贪婪行构造。 例如，如果 x 很大但 y 很小，一种幼稚的方法可能会尝试构建直到每行用完 C，但错误地计算了累积的 C 使用量。 另一个问题是忘记了 A 的使用量在高度上是严格线性的，而 C 的使用量呈二次方增长。 

例如，如果 x = 10 且 y = 3，则简单的逐行方法可能会尝试 4 行，因为 A 允许这样做，但第 4 行总共需要 6 个 C，这是不可能的。 

## 方法

 直接模拟会逐行构建。 对于每一行 i，我们从可用池中减去一个 A 和 i−1 个 C。 这是有效的，因为结构是确定性的，所以我们可以贪婪地构建，直到资源耗尽。 然而，在最坏的情况下，行数可能高达 10^9 或更多，这使得这是不可行的。 

关键的观察结果是，资源消耗仅取决于高度 h，而不取决于中间选择。 构建 h 行后，A 的总使用量恰好为 h，C 的总使用量为 0 + 1 + 2 + ... + (h−1) 之和，等于 h(h−1)/2。 这将问题转化为检查单个 h 的可行性。 

现在的任务是找到满足两个约束条件的最大值 h：h ≤ x 和 h(h−1)/2 ≤ y。 第一个约束是线性的，第二个约束是二次的。 二次约束可以直接使用从不等式导出的封闭式表达式来求解，因此不需要二分查找。 

我们使用二次公式计算满足 h(h−1)/2 ≤ y 的最大 h，然后用 x 取最小值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(h) | O(1) | O(1) | 太慢了|
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算仅受 A 限制的最大可能高度。 这就是 x，因为每一行恰好消耗一个 A。 
2. 通过求解不等式 h(h−1)/2 ≤ y 计算 C 允许的最大高度。 这是通过对所有行所需的 C 求和得出的。 
3. 将不等式重新整理为 h² − h − 2y ≤ 0，这是一个标准的二次约束。 
4. 计算相应方程 h = (1 + sqrt(1 + 8y)) / 2 的正根并取其下限。 这给出了不违反 C 约束的最大整数 h。 
5. 最终答案是两个极限中较小的一个：一个来自 A，一个来自 C。 

### 为什么它有效

 该结构强制每一行独立地消耗一个 A，因此 A 的使用量在高度上是线性的。 C 要求确定性地累积为前缀和，因此任何有效高度都完全由单个标量不等式来表征。 由于 h 中的两个约束都是单调的，因此一旦高度无效，所有较大的高度也无效，从而确保计算出的最大值是全局最优的。 

## Python 解决方案```python
import sys
import math
input = sys.stdin.readline

def solve():
    x, y = map(int, input().split())
    
    # limit from A's
    h_a = x
    
    # limit from C's: h(h-1)/2 <= y
    # solve h^2 - h - 2y <= 0
    disc = 1 + 8 * y
    h_c = (1 + math.isqrt(disc)) // 2
    
    print(min(h_a, h_c))

if __name__ == "__main__":
    solve()
```该实现将两个约束完全分开。 A 约束是直接的，因为每一行仅消耗一个 A。C 约束使用整数平方根处理以避免浮点精度问题，因为 y 可以大到 10^18。 使用`math.isqrt`确保精确的整数运算。 

决赛`min`结合了这两个约束，因为有效的三角形必须同时满足这两个约束。 

## 工作示例

 ### 示例 1

 输入：x = 4，y = 6

 | 步骤| 从 A | h h 来自 C | 当前答案 |
 | ---| ---| ---| ---|
 | 计算极限| 4 | 求解 h(h−1)/2 ≤ 6 → h = 4 | 4 |

 这里，两个约束都允许高度为 4。检查 C 的使用情况：0+1+2+3 = 6，与可用的 C 完全匹配，因此可以实现全高。 

### 示例 2

 输入：x = 4，y = 5

 | 步骤| 从 A | h h 来自 C | 当前答案 |
 | ---| ---| ---| ---|
 | 计算极限| 4 | 求解 h(h−1)/2 ≤ 5 → h = 3 | 3 |

 尽管 A 允许 4 行，但 C 在高度 4 处断裂，因为它需要 6 个 C。 因此限制因素为C，答案变为3。

 ## 边缘情况

 一种边缘情况是 y = 0。在这种情况下，只有不需要 C 的行是可能的。 这意味着无论 x 如何，都只能存在第一行。 公式给出disc = 1，因此h_c = 1，最终答案变为min(x, 1)，这是正确的。 

另一种边缘情况是 x 远小于 C 限制高度。 例如 x = 2 并且 y 非常大。 C 公式可能会产生很大的 h_c，但最终答案正确地限制为 2，因为 A 会线性消耗并立即成为瓶颈。 

第三种情况是 y 与三角数完全匹配。 对于 y = 6，该公式准确地得出 h_c = 4，因为 1 + 2 + 3 = 6，确认处理等式边界时不会因整数平方根下限行为而出现差一错误。
