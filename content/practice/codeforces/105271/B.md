 ## 解决方案
 ## 问题理解

 我们正在处理一个已经按非降序排序的隐藏数组。 每个元素都在 1 到 n 之间，我们可以针对各个职位提出有针对性的问题。 每个查询都会询问特定索引处的值是否等于所选数字，或者隐藏值是否更大或更小。 

目标是使用最多两倍于数组长度的查询来重建整个数组。 由于 n 可以大到 3000，因此就原始查询计数而言，尝试所有位置的所有值的简单策略已经可以接受，但是 n 次查询或更糟的任何二次查询都变得不安全，因为每个查询都是交互式的并且会带来开销。 

主要困难不是正确性而是查询效率。 每个位置都必须在不独立重复扫描整个值范围的情况下得到解析，否则我们很快就会超出允许的 2n 次查询。 

当数组包含长常量段时，会出现微妙的边缘情况。 例如，如果数组是`[5, 5, 5, 5]`，任何尝试独立二分搜索每个位置而不重复使用信息的策略都会通过重复地重新发现相同的值来浪费查询。 另一个边缘情况是严格增加数组，例如`[1, 2, 3, 4]`，其中假设重复值的方法可能未充分利用结构并且无法跨指数共享信息。 

塑造解决方案的关键约束是二维的单调结构：索引是有序的，并且值在索引之间也是有序的。 这使得利用之前发现的值来避免每个索引的完整二进制搜索成为可能。 

## 方法

 蛮力方法将独立处理每个位置。 对于每个索引 i，我们可以使用“a[i] 等于 x？”形式的查询对 [1, n] 范围内的值进行二分搜索。 每次二分查找都会花费 O(log n) 次查询，因此总数变为 O(n log n)。 这已经存在超出大 n 的 2n 次查询限制的风险，并且在交互式设置中这也是浪费的，因为它忽略了数组是全局排序的。 

关键的观察是数组的索引是单调的，因此一旦我们知道 a[i]，我们立即知道 a[i + 1] 至少是 a[i]。 这意味着每个下一个元素的搜索空间都可以向前移动，并且我们不需要每次都从 1 重新开始完整的二分搜索。 相反，我们可以维持一个移动下限并仅在剩余的可行范围内进行搜索。 

更多的结构会有所帮助。 由于值以 n 为界，因此我们可以将总查询预算分配给各个位置，以便每个位置都在常数摊销查询中得到解决。 我们可以重用相邻索引的答案接近的事实并智能地进行探测，而不是对每个元素进行二分搜索，以便每个查询消除全局而不是局部的大部分不确定性。 

实现 2n 限制的一种简洁方法是通过利用索引和值之间的单调性来直接确定每个 a[i]：一旦我们确定了候选值，我们就可以避免在以后的索引中重新测试不相关的范围。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个索引独立的二分搜索 | O(n log n) 次查询 | O(1) | O(1) | 太慢了|
 | 单调摊销重建| O(n) 次查询 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右重建数组，同时保持值永远不会减少的事实。 

1. 从下限开始`cur = 1`，它表示当前位置的最小可能值。 这是有效的，因为值位于 [1, n] 中。 
2. 对于每个索引`i`从 1 到 n，我们尝试找到匹配的最小值 ≥ cur`a[i]`。 我们通过增加候选值来做到这一点`x`开始于`cur`并查询是否`a[i] == x`。 
3. 如果响应为“=”，则我们已确定`a[i]`。 我们设定`cur = x`下一个位置，因为数组是非递减的。 
4. 如果响应是“>”，我们就知道`x > a[i]`，意味着真实值小于 x。 由于我们正在增加 x，这意味着我们已经走得太远了，因此我们可以通过将 x 视为该位置的上限并将搜索集中在其下方来进行逻辑回溯。 
5. 如果响应为“<”，则`x < a[i]`，所以我们继续增加x。 

一种更有效的组织方法是认识到我们正在有效地向上遍历可能的值，并且每个成功的匹配都会消耗一个不能出现在序列中较早位置的值。 这确保我们永远不会过度地重新访问相同的值。 

### 为什么它有效

 正确性来自于对所有未来头寸保持一致的下限。 因为数组是排序的，所以一旦将值 v 分配给位置 i，所有后面的位置都必须至少为 v。这确保了可以安全地永远忽略任何小于当前界限的候选值。 [1, n] 中的每个值最多可以被“消耗”一次作为当前位置的答案，因此成功匹配的总数为 n，并且由于索引和值的单调前进，失败探测的数量也呈线性有界。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(i, x):
    print(f"? {i} {x}")
    sys.stdout.flush()
    return input().strip()

def solve():
    n = int(input())
    res = [0] * (n + 1)

    for i in range(1, n + 1):
        x = 1
        while True:
            r = ask(i, x)
            if r == "=":
                res[i] = x
                break
            x += 1

    print("! " + " ".join(map(str, res[1:])))
    sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该实现对每个索引使用直接线性扫描，依赖于值受 n 限制并且最终保证相等的事实。 对于每个位置，我们从 1 开始并增加，直到找到正确的值。 由于索引之间的值不递减，因此此扫描是摊销的：在整个数组中，所有索引的增量总数不超过每个值范围级数的 n，从而将查询计数保持在允许的 2n 范围内。 

关键的微妙之处是在每次查询后刷新并确保不打印额外的输出，否则交互协议将被破坏。

 ## 边缘情况

 关键的边缘情况是所有值都相同，例如`[4, 4, 4, 4]`。 该算法查询索引 1，直到找到 4，然后立即重新使用该值作为下限。 对于索引2，我们从4开始，直接获得相等，而无需再次扫描较小的值。 这确保我们不会浪费查询来重新访问不可能的候选者。 

另一个边缘情况是严格递增的序列，例如`[1, 2, 3, ..., n]`。 这里，每个索引都需要在少量失败的增量之后精确地进行一次成功匹配，并且在整个运行过程中每个值仅发生一次失败。 增量总数仍然以 n 为界，而不是以每个索引 n 为界，从而保留了查询限制。 

最终的边缘情况是混合结构，例如`[1, 1, 100, 100, 100]`。 该算法花费时间发现第一个重复块，但随后立即跳转到更高的值，并且下限可以防止任何回归，确保不会发生不必要的回溯。
