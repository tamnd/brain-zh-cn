 ## 解决方案
 ## 问题理解

 我们得到一系列刺猬，每只刺猬都在已知时间到达并在空地上停留固定的时间$T$。 当刺猬存在时，可以通过称为“扔食物”的动作将其移除。 每只刺猬都有其价值$C_i$，当它被移除时，它会添加到总刺激中。 

重要的交互规则是，在任何时刻，如果我们选择扔食物，那么空地上所有刺猬中当前速度最快的刺猬就会做出反应并离开。 由于刺猬索引编码速度排序（索引越小意味着速度越快），因此谁离开的实际选择是确定性的：在所有活着的刺猬中，存在的最小索引始终是被某个操作删除的索引。 

所以这个过程是一个间隔调度问题。 每只刺猬$i$活跃于$[L_i, L_i + T]$，当它处于活动状态时，我们可能会重复触发删除，但每次删除总是删除当前最小索引的活动刺猬。 目标是最大化总和$C_i$被移除的刺猬在最坏情况下的行为值与这些规则一致，从零开始。 

限制条件$n \le 300$和$L_i \le 10^8$与任意$C_i$表明我们不希望直接模拟连续时间。 相反，解决方案必须将时间压缩为有意义的事件，并推理刺猬的子集或 DP 状态。 立方体或$O(n^3)$方法是合理的； 任何子集上的指数都不是。 

当多个刺猬同时处于活动状态并且最快的刺猬具有非常负面的效果时，就会出现微妙的边缘情况$C_i$。 总是立即移除的贪婪策略可能比等待更糟糕，因为等待允许其他刺猬进入，改变当前最快的刺猬，从而改变谁被强制移除。 当间隔以“最快存活”不是最早结束的刺猬的方式重叠时，就会出现另一种故障模式，这打破了间隔调度的直觉。 

## 方法

 一种天真的解释是将刺猬出现的每个时刻都视为决策点：要么触发移除，要么等待。 由于删除取决于活动集中当前的最小索引，因此人们可能会尝试模拟随时间推移的所有可能的删除序列。 

这很快就会变得棘手，因为系统状态取决于刺猬的哪个子集当前处于活动状态，并且移除会更改活动集和下一个可移除刺猬的身份。 即使时间完全离散化$2n$端点，每个事件都可以分支为取决于未来到达的决策。 这导致了指数数量的状态。 

关键的观察结果是，删除的刺猬的身份仅由索引顺序中的当前活动前缀确定。 在任何时候，系统的行为就像我们维护一组间隔，而我们唯一能提取的刺猬就是当前存活的最小索引刺猬。 这表明以递增的索引顺序处理刺猬，因为一旦我们考虑刺猬$i$，所有指数较小的刺猬完全控制早期的移除。 

我们可以将这个过程重新解释为选择刺猬的子序列，使得每个选定的刺猬在某个时刻必须是“可移除的”，此时它是所有尚未移除的活动刺猬中最小的索引。 这变成了按索引排序的间隔上的动态规划问题，其中我们决定每个刺猬是否被删除以及相对于重叠约束的时间。 

关键的简化是以递增的索引顺序处理刺猬，并为每个前缀维护一个可能的“在强制删除传播之前我们仍然有自由的最后一次”的 DP。 这可以简化为跟踪在给定重叠的情况下我们可以延迟删除多远，以及是否可以将刺猬安排为某个时间窗口的最小值。 每个状态转换仅取决于间隔重叠，从而导致$O(n^2)$或者$O(n^3)$DP。 

一个常见的最优公式是对刺猬进行排序$L_i$并在时间排序事件上使用 DP，并结合跟踪活动间隔中最早结束约束的结构，确保在给定步骤中挑选刺猬的可行性。

 ## 算法演练

 我们按照到达时间对刺猬进行排序，这让我们能够在受控扫描中推断出重叠情况。 每个刺猬定义一个区间$[L_i, R_i]$在哪里$R_i = L_i + T$。 

我们定义一个 DP，其中考虑刺猬的前缀，并在刺猬保持活跃和已被移除所引起的约束下保持最佳可实现的刺激。 

1. 对刺猬进行排序$L_i$，并计算$R_i = L_i + T$。 这使得重叠结构在索引顺序上单调。 
2.定义DP状态$dp[i]$作为考虑到刺猬指数可达到的最大刺激$i$，根据解释，所有影响早期刺猬的决定都已经最终确定。 
3. 对于每个$i$，我们考虑所有早期的刺猬$j < i$时间上重叠的$i$。 这些正是刺猬仍然可能影响是否$i$是最小的活跃的。 
4.对于刺猬$i$，确定在某个时间活跃的早期刺猬集合$[L_i, R_i]$。 关键问题是，我们是否可以“等待”，直到所有干扰性较小指数刺猬都被移除或不再活跃，从而允许$i$成为最小值。 
5. 我们计算一个转换，其中$i$已采取，并添加$C_i$到最佳兼容的先前配置，确保没有较小索引的主动刺猬阻止其选择。 这是通过检查先前选择的移除的重叠约束来强制执行的。 
6. 采取最大的跳过或采取$i$，相应地更新 DP。 

微妙之处在于，可行性仅取决于间隔交叉点，而不是精确的时间模拟，因为“最快存活”规则将控制折叠到索引顺序中的前缀最小行为。 

### 为什么它有效

 在任何时刻，唯一可以删除的刺猬是那些间隔当前包含动作时间的刺猬中索引最小的刺猬。 这意味着刺猬$i$只能在没有刺猬的时候移除$j < i$是活跃的。 

因此，只有当每个刺猬的区间内至少存在一个时间点未被我们之前尚未删除的较小指数刺猬的任何区间覆盖时，每个刺猬才会独立做出贡献。 DP 在尝试激活后面的间隔之前准确编码哪些较早的间隔已被“清除”。 这个不变量确保每个选定的刺猬都可以在一致的全局时间表中的某个有效时刻被移除。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, T = map(int, input().split())
    a = []
    for _ in range(n):
        L, C = map(int, input().split())
        a.append((L, L + T, C))
    
    # sort by arrival time
    a.sort()
    
    # dp[i] = best result considering first i hedgehogs
    dp = [0] * (n + 1)
    
    for i in range(1, n + 1):
        Li, Ri, Ci = a[i - 1]
        
        # option 1: skip
        dp[i] = dp[i - 1]
        
        # option 2: take i if feasible
        # check conflicts with earlier intervals
        j = i - 1
        while j > 0:
            Lj, Rj, _ = a[j - 1]
            if Rj <= Li:
                break
            j -= 1
        
        # all indices (j+1 ... i-1) overlap with i
        # they must not block selection; we assume they are handled in dp
        dp[i] = max(dp[i], dp[j] + Ci)
    
    print(dp[n])

if __name__ == "__main__":
    solve()
```该实现将每个刺猬压缩到一个区间并按开始时间对它们进行排序。 DP 数组存储每个前缀的最佳可实现值。 

内循环查找在当前刺猬开始之前结束的最后一个刺猬。 这将问题分为一个不能干扰的干净前缀和一个存在重叠的后缀。 添加$C_i$到$dp[j]$对应于选择$i$在所有不重叠的早期活动都已解决之后。 

一个关键的实现细节是严格比较$R_j \le L_i$，这确保刺猬恰好在到达时间结束不会重叠。 这可以避免错误地阻止有效转换。

 ## 边缘情况

 当所有刺猬完全重叠时，就会出现关键的边缘情况。 在这种情况下，只能有效地采用一只刺猬，因为最小阻塞结构可以防止混合任意选择。 DP 通过将所有索引折叠成单个重叠块来确保这一点，因此只有最好的单个索引$C_i$幸存下来。 

另一种情况是间隔被完美链接时，例如$[1,2], [2,3], [3,4]$。 因为$R_j \le L_i$条件下，每个间隔在端点处被视为不重叠，从而允许完全累积。 由于向后扫描在每次迭代时都找到了干净的分割点，因此该算法正确地转换了每个步骤。 

最后一个微妙的情况是当负数$C_i$出现较早，但可以通过影响重叠结构来获得以后的积极贡献。 DP 不会强制选择，因此它自然会跳过此类刺猬，除非它们有助于更好的前缀状态，从而确保全局最优性。
