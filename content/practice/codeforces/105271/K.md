 ## 解决方案
 ## 问题理解

 我们有一个 n 行 m 列的座位网格。 每个单元格都包含一个成本，选择一个座位就意味着支付该成本。 AAlikhan 想要恰好选择 k + 1 个座位（他自己加上 k 个朋友），使得总成本不超过预算 B。 

所选的座位必须使用 4 向邻接形成单个连接的组件，因此从任何选定的单元格中，您可以通过仅在选定的单元格中向上、向下、向左或向右移动来到达任何其他选定的单元格。 

目标不是最小化成本或最大化座位数量，因为座位数量是固定的。 相反，在所有大小为 k + 1 且总成本最多为 B 的有效连接选择中，我们希望最小化距舞台最远座位的行索引。 由于第 1 行最接近舞台，第 n 行最远，这意味着我们希望最小化所选单元格中的最大行索引。 

因此，该问题是一个具有预算约束和最小最大深度目标的约束连通子图选择问题。 

约束 n、m ≤ 100 意味着最多 10,000 个单元格。 所选单元格的数量最多为 100 个，因为 k < 100。这强烈表明解决方案可以在本地探索网格状态，但无法从 10,000 个单元格中枚举大小为 k + 1 的所有子集，因为这在组合上将是巨大的。 

一个微妙的点是连接性与所选的细胞紧密耦合。 即使我们知道哪些细胞便宜，我们也不能随意挑选它们，因为它们必须形成一个连接的形状。 

如果试图贪婪地选择 k + 1 个最便宜的可达单元而不强制连接，就会出现一个幼稚的陷阱，即使成本有效，这也会产生断开的集合。 

另一个陷阱是假设我们可以独立选择行或列，因为连接性耦合了两个维度。 

## 方法

 直接的强力方法是尝试 k + 1 个单元的每个子集，检查它是否连接，计算其成本，并跟踪最小可能的最大行。 这在原则上是正确的，因为它探索了完整的搜索空间。 

然而，子集的数量为 C(10000, k+1)，即使 k = 5 也是不可行的。即使限制为连通子集也无济于事，因为网格中连通形状的数量随 k 呈指数增长。 每次连通性检查都是 O(k)，成本求和也是 O(k)，因此这种方法会立即失败。 

关键的结构观察是 k 小，而不是网格。 这表明我们应该从起始单元开始逐步增长解决方案，维护连接的组件，并仅探索小的扩展。 

另一个重要的观察是最大行的约束是单调的：如果我们可以形成一个有效的连通分量，其最高行是r，那么允许行≥r只会使问题变得更容易。 这表明对超过行限制 r 的答案进行二进制搜索。 

对于固定的 r，我们将自己限制在第 1 行到第 r 行的单元格中，并询问是否存在大小为 k + 1 且成本 ≤ B 的连通分量。这成为可行性检查。 

由于 k ≤ 99，我们可以将其视为有界状态扩展问题。 从任何起始单元开始，我们尝试通过重复添加相邻单元来构建连通组件，但我们只跟踪大小为 k + 1 的部分状态的最佳可能成本。因为 k 很小，所以我们可以对锚定在起始单元的子集使用 BFS 式扩展或 Dijkstra 式松弛，并按大小和成本进行修剪。 

我们对候选 r 值重复此可行性检查，并取最小可行 r。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 | O(2^(n·m)) | O(2^(n·m)) | O(k) | 太慢了 |
 | 二分查找+有界BFS扩展 | O(nm·k²·log n) | O(nm·k²·log n) | O(nm·k) | 已接受 |

 ## 算法演练

我们首先将答案视为行阈值，并检查是否可以保持在某些行前缀内。 

1. 我们对最小行限制 r 进行二分搜索，以便仅使用第 1 行到第 r 行中的单元格就存在有效的连通组。 这是有效的，因为如果 r 存在解，那么对于任何更大的 r 也存在解，因为有更多的单元格可用。 
2. 对于固定的 r，我们构建一个允许的单元格列表，这些单元格的行索引 ≤ r。 我们完全忽略所有其他人，因为他们被禁止进行此检查。 
3. 我们尝试将每个允许的单元作为连接组件的起点。 这是必要的，因为不能保证最佳组件包含任何特定的单元，例如最便宜的或最顶层的单元。 
4. 从起始单元开始，我们对组件当前边界定义的状态运行最佳优先扩展，跟踪我们已经选择了多少个单元以及总成本。 我们仅通过添加尚未包含的新的相邻允许单元格来进行扩展。 
5. 我们按照成本优先考虑部分结构，始终首先探索最便宜的部分连接组件。 当我们在预算 B 内达到 k + 1 大小时，我们立即成功实现该 r。 
6. 如果任何起始单元产生有效的结构，我们就为该 r 返回 true。 否则的话，在这个行限制之下是不可能的。 

### 为什么它有效

 正确性依赖于这样一个事实：任何有效的解决方案都可以植根于其一个单元格，然后向外生长。 每个连通集都有一个生成树结构，因此存在一种顺序，我们可以在其中将其单元逐一添加，同时保持连通性。 我们的扩展模拟从每个可能的根开始的所有此类顺序。 由于我们通过增加成本来探索状态，因此我们确保如果约束内存在任何有效的连接组件，则最终将在不跳过必要的中间部分集的情况下构造它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

def feasible(grid, n, m, k, B, r_limit):
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]
    
    cells = []
    for i in range(n):
        for j in range(m):
            if i < r_limit:
                cells.append((i, j))
    
    allowed = [[False]*m for _ in range(n)]
    for i in range(r_limit):
        for j in range(m):
            allowed[i][j] = True

    for si in range(r_limit):
        for sj in range(m):
            start_cost = grid[si][sj]
            if start_cost > B:
                continue

            # state: (cost, size, i, j, mask_set)
            # we encode visited locally via set per path (k small)
            pq = []
            heapq.heappush(pq, (start_cost, 1, si, sj, {(si, sj)}))

            while pq:
                cost, sz, i, j, used = heapq.heappop(pq)

                if sz == k + 1 and cost <= B:
                    return True

                if sz > k + 1 or cost > B:
                    continue

                for di, dj in dirs:
                    ni, nj = i + di, j + dj
                    if 0 <= ni < r_limit and 0 <= nj < m:
                        if not allowed[ni][nj]:
                            continue
                        if (ni, nj) in used:
                            continue
                        new_cost = cost + grid[ni][nj]
                        if new_cost > B:
                            continue
                        new_used = set(used)
                        new_used.add((ni, nj))
                        heapq.heappush(pq, (new_cost, sz + 1, ni, nj, new_used))
    return False

def solve():
    n, m, k = map(int, input().split())
    B = int(input())
    grid = [list(map(int, input().split())) for _ in range(n)]

    lo, hi = 1, n
    ans = n

    while lo <= hi:
        mid = (lo + hi) // 2
        if feasible(grid, n, m, k, B, mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先定义固定行限制的可行性检查。 在其中，我们将注意力限制在当前阈值以上的行。 然后，我们尝试从每个有效单元开始构建大小为 k + 1 的连接组件。 

状态表示显式跟踪当前单元、当前成本、所选单元的数量以及该部分构造中访问的单元的集合。 这是必要的，因为连接性要求我们不要重新访问同一部分路径中的单元。 

优先级队列确保我们始终首先扩展成本较低的部分构造，这可以改善存在有效配置时的提前终止。 

然后，外部二分搜索找到最小的可行行限制。

 ## 边缘情况

 一种边缘情况是 k = 0，这意味着只需要一个座位。 在这种情况下，答案只是最小的行 i，使得第 i 行中存在成本 ≤ B 的单元格。 该算法自然地处理了这个问题，因为可行性检查在大小为 1 时立即成功。 

另一种边缘情况是，由于成本原因，只有一个列可用，这迫使连接进入垂直链。 扩展过程仍然有效，因为邻接自然地沿着列构建了一条路径。 

第三种边缘情况是最佳解决方案完全位于最后一行。 二分搜索正确地达到 r = n，并且可行性总是成功，因为所有单元都被允许，即使没有早期前缀可以满足约束，也能确保正确性。
