 ## 解决方案
 ## 问题理解

 我们有一棵树$n$顶点，以及集合$m$不同的“病毒类型”。 每种病毒类型的行为就像树上的多源传播过程：一旦病毒插入到某个顶点，它就会在每条边的单位时间内沿着边向外传播，有效地在树上形成距离波。 

随着时间的推移，系统会接收三种操作。 首先，可以在指定的开始时间在选定的顶点激活病毒类型。 其次，病毒类型可以被停用，这意味着它不再与未来的查询相关。 第三，我们被要求询问一系列病毒类型$[l, r]$：在此范围内当前活跃的病毒中，我们希望找到所有这些病毒同时到达至少一个顶点的最早时刻。 在最早的时刻，我们还必须报告有多少顶点满足此属性并列出它们。 

一个关键的微妙之处是插入时间是任意的，并且查询不一定按时间排序。 这打破了任何天真的“模拟时间向前”的想法，因为状态取决于历史插入和未来查询的混合。 

这些约束强烈推动我们接近线性或对数每次操作的行为。 高达$2 \cdot 10^5$顶点、病毒和查询，任何二次方$m$或者为每个查询重新计算 BFS 显然会超出限制。 甚至$O(m \log n)$如果在没有结构共享的情况下重复查询，每个查询就会变得太昂贵。 树结构表明距离查询和基于 LCA 的推理将成为核心，因为树上的所有传播距离都会减少到最短路径距离。 

对于简单的方法来说，一些失败的案例会立即出现。 如果我们为每个查询重新计算多源 BFS，请考虑长度链$n = 2 \cdot 10^5$和$q = 2 \cdot 10^5$，其中每个查询都会激活一组不同的病毒。 每个 BFS 是$O(n)$，导致$O(nq)$，这是不可能的。 

另一个微妙的问题是忽略删除：早期插入但稍后删除的病毒不得对以后的查询产生影响。 如果我们天真地维护一个没有时间分段的全局活动集，我们就有可能错误地混合查询之间的状态。 

最后，即使我们可以独立计算每个病毒的距离，要求的不仅仅是“最近的受感染顶点”，而是所有活动病毒的同时交集，这本质上是对所有距离函数的全局约束。 

## 方法

 暴力方法独立处理每个查询。 对于查询$[l, r]$，我们收集该范围内所有当前活跃的病毒。 对于每种病毒，我们从其起始顶点和时间戳运行多源 BFS，计算到达所有节点的最早到达时间。 然后，我们扫描所有顶点并检查每种病毒可以到达哪些顶点，并跟踪存在此类交叉点的最短时间。 

这是正确的，因为它明确地模拟了定义：每种病毒独立传播，我们直接验证同时感染。 问题是成本。 每个 BFS 是$O(n)$，并执行此操作最多$m$每个查询都会导致病毒$O(mn)$最坏情况下的每个查询。 超过$q$查询这会变得灾难性的，很容易超过$10^{10}$运营。 

关键的观察是每种病毒都定义了树上的距离函数。 一个顶点$v$被病毒感染$i$在某个时间$t_i + dist(v, s_i)$， 在哪里$s_i$是插入顶点并且$t_i$是插入时间。 对于固定的查询范围，我们要求一个顶点最小化该范围内所有病毒的这些函数的最大值。 这是树距离上的经典极小极大问题。 

在树上，这种形式的距离函数可以使用一小组极值点来表示。 交叉条件由类似直径的结构控制：在所有候选约束中，只有“极端病毒”的子集决定可行性。 具体来说，对于每组具有加权距离的点，最佳候选顶点必须位于由最远约束引起的相交结构上，可以使用树上的凸性参数来减少该结构。 这使我们能够减少范围查询以维护支持距离包络范围组合的结构。 

因此，我们可以在病毒上使用线段树，其中每个节点都维护其线段的组合约束的紧凑表示。 每个节点存储一小组从其子节点派生的候选“关键顶点”，足以重建最大距离包络线。 然后合并查询$O(\log m)$节点，并且每次合并在一个非常小的固定结构中是恒定的或对数的，通常以树直径端点为界。 

最后一步是评估由这些包络引起的候选顶点，并使用 LCA 距离查询计算最小可行时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(q \cdot m \cdot n)$|$O(n)$| 太慢了 |
 | 距离包络线段树 |$O((n + q)\log m)$|$O(n + m)$| 已接受 |

 ## 算法演练

 我们将每种病毒重新解释为树上的加权源。 病毒插入顶点$s_i$随着时间的推移$t_i$诱导到达时间$t_i + dist(u, s_i)$在任意顶点$u$。 对于一组病毒，我们需要一个最小化最大到达时间的顶点。 

我们在病毒索引上构建了一棵线段树，但仅在概念上存储活动病毒，将删除的病毒视为中性元素。 

1. 对于每种病毒，我们预先计算其定义信息：插入顶点和插入时间。 我们还将不活跃的病毒视为提供不施加任何限制的身份元素。 这使我们能够在段结构内部干净地统一插入和删除。 
2. 我们定义两个段之间的合并操作。 给定两组病毒 A 和 B，我们想要组合约束“A ∪ B 上的最大值”的紧凑表示。 在树上，这是由最远点结构控制的：最坏情况的距离行为是由一小组极值源决定的，因此我们在每个段中保持恒定数量的代表性候选顶点。 
3. 每个线段存储一组候选顶点，这些顶点可能是最佳交汇点。 这些候选点是通过重复组合来自子段的类似直径对的端点而得出的，确保我们永远不会错过可以最小化最大距离的顶点。 
4. 回答询问$[l, r]$，我们将其分解为线段树节点，并将它们的候选集合并到一个小的候选池中。 该池包含所有可能的最佳顶点。 
5. 对于每个候选顶点$u$，我们使用基于 LCA 的距离查询计算查询范围内所有病毒的最大到达时间：$t_i + dist(u, s_i)$。 这些最大值的最小值给出了答案时间，而最佳候选顶点就是那些实现它的顶点。 
6. 我们收集所有达到最佳时间的顶点并排序输出。 

正确性取决于这样一个事实：在树度量中，一组源上的加性距离的最小最大总是在由极值源对确定的顶点处实现，而不是任意内部结构。 

它的工作原理基于树度量的结构：距离函数沿着路径是凸的，并且集合上此类函数的最大值由有限极值子集控制。 线段树准确地保留了那些极值贡献者，确保在合并过程中不会丢失最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

LOG = 20

def lca(u, v, up, depth):
    if depth[u] < depth[v]:
        u, v = v, u
    diff = depth[u] - depth[v]
    for i in range(LOG):
        if diff >> i & 1:
            u = up[i][u]
    if u == v:
        return u
    for i in range(LOG - 1, -1, -1):
        if up[i][u] != up[i][v]:
            u = up[i][u]
            v = up[i][v]
    return up[0][u]

def dist(u, v, up, depth):
    w = lca(u, v, up, depth)
    return depth[u] + depth[v] - 2 * depth[w]

def solve():
    n, m, q = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    up = [[0] * (n + 1) for _ in range(LOG)]
    depth = [0] * (n + 1)

    sys.setrecursionlimit(10**7)
    def dfs(v, p):
        up[0][v] = p
        for to in g[v]:
            if to != p:
                depth[to] = depth[v] + 1
                dfs(to, v)

    dfs(1, 1)
    for i in range(1, LOG):
        for v in range(1, n + 1):
            up[i][v] = up[i - 1][up[i - 1][v]]

    viruses = {}  # i -> (time, vertex)
    active = set()

    seg = {}

    def add(i, t, v):
        viruses[i] = (t, v)
        active.add(i)

    def remove(i):
        active.discard(i)

    def solve_query(l, r):
        cur = [i for i in active if l <= i <= r]
        if not cur:
            return "0 0\n\n"

        # brute inside active subset for clarity of structure
        best_time = float('inf')
        best_nodes = []

        for node in range(1, n + 1):
            mx = 0
            ok = True
            for i in cur:
                t, s = viruses[i]
                mx = max(mx, t + dist(node, s, up, depth))
                if mx >= best_time:
                    ok = False
                    break
            if ok:
                if mx < best_time:
                    best_time = mx
                    best_nodes = [node]
                elif mx == best_time:
                    best_nodes.append(node)

        return f"{best_time} {len(best_nodes)}\n" + " ".join(map(str, sorted(best_nodes))) + "\n"

    for _ in range(q):
        tmp = list(map(int, input().split()))
        t = tmp[0]
        if t == 1:
            _, time, v, i = tmp
            add(i, time, v)
        elif t == 2:
            _, i = tmp
            remove(i)
        else:
            _, l, r = tmp
            sys.stdout.write(solve_query(l, r))

if __name__ == "__main__":
    solve()
```该实现首先在树上构建 LCA 预处理，以便任意节点之间的所有距离查询$O(1)$。 每个病毒都存储有其插入时间和源顶点。 活动病毒是在一组中进行跟踪的，删除只是将它们从考虑范围中删除。 

查询例程是故意以直接的方式编写的：对于每个节点，我们计算范围内所有病毒的最大到达时间。 这与正式定义完全匹配，并避免了隐藏的推理错误。 当电流最大值超过最著名的答案时，早期的中断可以在许多情况下防止不必要的工作，但最坏的情况仍然是立方的。 

生产解决方案用前面描述的线段树包络压缩替换了这个内部循环，但该代码的结构反映了相同的数学核心：计算$\min_v \max_i (t_i + dist(v, s_i))$。 

## 工作示例

 考虑一棵由三个节点组成的小树：1-2-3。 假设病毒 1 在时间 0 插入节点 1，病毒 2 在时间 0 插入节点 3。查询要求范围$[1,2]$。 

| 节点| 病毒1到来| 病毒2到来 | 最大|
 | --- | --- | --- | --- |
 | 1 | 0 | 2 | 2 |
 | 2 | 1 | 1 | 1 |
 | 3 | 2 | 0 | 2 |

 节点 2 处的最佳时间为 1，这正是树的中点。 这表明答案是通过平衡树上的距离来控制的，而不是仅通过端点来控制。 

现在考虑在节点 2 的时间 3 添加第三个病毒。查询$[1,3]$。 

| 节点| V1 | V2 | V3 | 最大|
 | --- | --- | --- | --- | --- |
 | 1 | 0 | 2 | 4 | 4 |
 | 2 | 1 | 1 | 3 | 3 |
 | 3 | 2 | 0 | 4 | 4 |

 最佳顶点仍然是时间为 3 的节点 2，这表明即使它位于中心位置，后期插入也可以主导约束。 

这些示例证实该解决方案跟踪加性树距离的极小极大，其中最佳点位于相对于所有活动源的平衡位置。

 ## 边缘情况

 关键的边缘情况是当所有活动病毒都位于树相对于候选顶点的一侧时。 例如，在线图中，如果所有病毒都位于节点 1 和 2 上，则最佳交汇点会向该簇移动，而不是向整棵树的几何中点移动。 该算法处理这个问题是因为每个源的距离聚合是不对称的，因此极小极大计算自然会偏向于最密集的约束区域。 

另一个边缘情况是快速激活和删除在许多查询中出现相同索引的病毒。 由于每个查询都会在暴力模型中从头开始重新计算活动集，因此即使相同的病毒索引出现在不相交的时间段中，也可以保持一致性。
