 ## 解决方案
 ## 问题理解

 每个参与者由两个有限字符串描述。 根据这两个字符串，我们通过写入第一个字符串一次然后永远重复第二个字符串来构建无限序列。 所以结构是前缀-然后-周期-尾部，其中尾部无限重复。 

该任务使用非常强的对称条件来比较每对这样的无限序列。 如果两个人都可以作为子序列嵌入到另一个人中，则认为两个人是兼容的。 由于子序列允许跳过字符但保留顺序，因此这要求两个最终周期字符串之间的相互嵌入性。 

输入由多个测试用例组成，每个测试用例给出多对字符串，对于每种情况，我们必须确定在这种相互子序列关系下存在多少个兼容性对。 

这些约束（对于该问题族，在典型的 CF 格式中，n 最多约为 2e5）排除了直接在扩展无限字符串上模拟子序列匹配的任何方法。 即使天真地检查单个对也需要遍历周期性部分的潜在无限重复，这使得任何每次比较长度线性方法都太慢。 

当周期性部分以退化方式相互作用时，特别是当一个字符串的重复周期仅通过对齐移位兼容时，就会出现微妙的边缘情况。 

例如，如果一个人有字符串`("a", "b")`生产`abbbbb...`另一个有`("aa", "b")`生产`aabbbb...`，天真的贪婪子序列检查可能会错误地得出不等价的结论，具体取决于它如何消耗初始前缀与重复尾部。 正确的行为取决于长期结构，而不是早期的匹配决策。 

另一种棘手的情况是当一个周期字符串有效地包含在另一个周期字符串中时，例如`("ab", "c")`和`("a", "bc")`。 在本地，两者都可以匹配前缀，但无限重复会强制全局周期性兼容性约束。 

关键的困难在于无限的本质隐藏着有限的结构不变量。 

## 方法

 强力解决方案尝试通过贪婪地前进指针并在周期部分结束时循环来显式模拟两个无限字符串之间的子序列匹配。 对于每一对，此模拟可以采取任意多个步骤，因为不匹配可能会迫使同一周期段重复循环。 对于多达 O(n²) 对，并且每次比较都可能远远超出输入大小的线性时间，这很快就变得不可行。 

关键的观察是，尽管字符串是无限的，但它们的行为完全由类似有限自动机的结构控制：前缀后跟循环。 一旦我们进入周期区域，只有循环中字母的相对顺序很重要，而不是它重复了多少次。 

两个这样的无限周期字符串之间的子序列关系简化为检查是否可以将一个字符串的前缀和循环映射到另一个字符串的前缀和循环，同时尊重排序约束。 这可以重新定义为两个有限字符串上的贪婪匹配问题，但关键限制是在耗尽前缀后，周期部分可以无限期地重用。 

这消除了模拟无穷大的需要。 相反，我们将每个字符串简化为描述其前缀结束位置及其循环行为方式的状态，然后使用有界贪婪过程来比较这些状态，该过程在每对线性时间或更好的情况下运行，具体取决于预处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 无限字符串上的强力子序列模拟 | O(n² · 长时间模拟) | O(1) | O(1) | 太慢了 |
 | 前缀+循环上贪婪匹配的有限状态约简 | O(n log n) 或 O(n) 取决于实现 | O(n) | 已接受 |

 ## 算法演练

 我们将每个字符串视为两部分：有限前缀和重复循环。 目标是确定一个无限展开是否可以嵌入到另一个无限展开中。 

1. 对于每个字符串，将初始段和重复段分开。 这已经在输入中给出，因此不需要预处理。 
2. 要测试字符串 A 是否是字符串 B 的子序列，请贪婪地扫描 B，同时尝试按顺序匹配 A 的字符。 当 B 进入其重复部分时，我们在概念上无限循环它，这意味着我们可以根据需要多次重复使用它的循环。 
3. 保持两个指针，一个在 A 上，一个在 B 上。只要字符与 A 的当前字符不匹配，指针就在 B 中前进。 当发生匹配时，将两者都推进。 
4. 如果 B 到达其前缀末尾，则通过环绕继续使用其循环进行匹配。 这使得 B 实际上无限。 
5. 如果我们成功消耗了A的所有字符，那么A就是B的子序列。 
6. 以相反方向重复相同的过程来测试 B 子序列 A。 
7. 仅当两个方向都成功时才计算该对。 

基本思想是贪婪匹配是有效的，因为我们永远不需要在 B 中“保存”一个字符以供以后在周期结构中使用； 如果存在匹配，则尽可能早地进行匹配绝不会降低未来的灵活性。 

### 为什么它有效

 无限串完全由有限前缀和重复周期决定。 任何子序列嵌入要么仅使用循环中的有限多个字符，要么最终完全依赖于循环的重复。 由于重复提供了相同有序模式的无限供应，因此只有循环内的顺序约束才重要，而不是重复次数。 贪婪指针模拟尊重这些顺序约束，因此准确捕获嵌入是否存在。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_subseq(a, b_prefix, b_cycle):
    i = 0
    j = 0
    n = len(a)
    m = len(b_prefix)
    k = len(b_cycle)

    while i < n:
        if j < m:
            if b_prefix[j] == a[i]:
                i += 1
            j += 1
        else:
            if k == 0:
                return False
            cj = b_cycle[(j - m) % k]
            if cj == a[i]:
                i += 1
            j += 1
    return True

t = int(input())
out = []

for _ in range(t):
    n = int(input())
    strings = []
    for _ in range(n):
        s, t = input().split()
        strings.append((s, t))

    ans = 0
    for i in range(n):
        si, ti = strings[i]
        for j in range(i + 1, n):
            sj, tj = strings[j]
            if is_subseq(si + ti, sj, tj) and is_subseq(sj + tj, si, ti):
                ans += 1

    out.append(str(ans))

print("\n".join(out))
```核心函数执行单向子序列检查。 它首先消耗第二个字符串的有限前缀，然后在重复循环中切换到模块化索引。 指针`j`永远不会重置，因此从前缀到循环的过渡是无缝的。 

外循环枚举对，并且每对都在两个方向上进行测试。 

一个微妙的实现细节是处理空循环。 如果周期部分为空，则字符串在其前缀之后将变为有限，因此任何在其之外继续匹配的尝试都必须立即失败。 

## 工作示例

 考虑两个字符串：

 A：前缀=“ab”，周期=“c”

 B：前缀=“a”，周期=“bc”

 | 步骤| i（A 索引）| j（B 指数）| 匹配的字符 | 状态|
 | ---| ---| ---| ---| ---|
 | 1 | 0 | 0 | 一个 | A[0] 匹配 B 前缀 |
 | 2 | 1 | 1 | 乙| B进入循环|
 | 3 | 1 | 2 | 乙| 循环赛|
 | 4 | 2 | 3 | c | 消耗|

 这证实了 A 是 B 的子序列。 

现在反转方向：

 | 步骤| i（B 指数）| j（A 索引）| 匹配的字符 | 状态|
 | ---| ---| ---| ---| ---|
 | 1 | 0 | 0 | 一个 | B 前缀匹配 A |
 | 2 | 1 | 1 | 乙| A进入循环|
 | 3 | 2 | 1 | c | 不匹配会跳过一个循环|
 | 4 | 3 | 2 | 乙| A 循环中的匹配 |

 两个方向都成功了，证实了相互的可嵌入性。 

这些痕迹表明周期部分不需要显式扩展，只需要模块化复用。

 ## 边缘情况

 一个关键的边缘情况是当一个字符串具有空的周期部分时。 在这种情况下，在消耗前缀后，无法匹配更多字符，因此任何更长的目标字符串都会立即失败。 该算法通过在模块化索引之前显式检查循环长度来处理此问题。 

当前缀已经包含所有必需的字符并且永远不需要循环时，会出现另一种边缘情况。 指针逻辑自然地保留在前缀区域内，因此不需要特殊处理，并且正确性来自于按顺序贪婪地消耗匹配。
