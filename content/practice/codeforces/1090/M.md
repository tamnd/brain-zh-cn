 ## 解决方案
 ## 问题理解

 我们得到一排房屋，每栋房屋都涂有整数颜色。 我们想要选择这条线的连续线段，以便在所选线段内，没有两个相邻的房屋具有相同的颜色。 在所有这些有效的细分中，我们被要求最大化该细分中的房屋数量。 

输入只是一个颜色数组。 输出是最长连续子数组的长度，其中每个相邻元素对都不同。 

这些约束立即指导设计空间。 对于多达 100,000 个房屋，任何尝试所有子数组并从头开始检查有效性的解决方案都将涉及以下顺序：$n^2$细分市场和潜在的$O(n)$每个段的验证，这远远超出了一秒的限制。 因此，我们正在寻找线性扫描或两指针式解决方案，以恒定的次数处理每个元素。 

当数组完美交替时，会出现微妙的边缘情况，例如`1 2 1 2 1`。 正确答案是完整长度。 过于激进地重新启动的简单方法可能会错误地截断该段。 另一个边缘情况是完全均匀的数组，例如`5 5 5 5`，其中每个有效段的长度为 1。任何解决方案都必须在每次违规时正确重置。 

## 方法

 强力策略将枚举每个可能的起始位置并将段向右延伸，同时保持相邻元素不同的约束。 对于每个起始索引$i$，我们向前扫描直到找到索引$j$这样$a[j] = a[j-1]$，然后记录长度$j - i$。 

这是正确的，因为它直接构造所有最大有效段。 然而，在最坏的情况下，所有元素都是不同的或交替的，每个起始位置几乎延伸到末尾，大致给出$n + (n-1) + \dots + 1 = O(n^2)$运营。 和$n = 100000$，这的顺序是$10^{10}$，这是不可行的。 

关键的观察结果是有效性仅取决于局部邻接性。 一旦我们遇到一对相等的邻居，任何跨越该边界的线段都会变得无效。 这意味着数组自然地划分为最大块，其中相邻元素始终不同。 在每个块内部，整个块都是有效的，所以我们只需要最大块长度。 

这将问题简化为单个线性扫描，其中我们跟踪有效邻接的当前条纹长度，并在遇到相等时重置它。

 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护在每个位置结束的当前有效段的运行长度。 

1. 从第一栋房子开始。 当前有效段长度为 1，因为单个元素始终有效。 
2. 从索引 1 开始从左到右迭代数组。在每个位置，将当前颜色与前一个颜色进行比较。 
3.如果当前颜色与之前的颜色不同，我们可以将当前有效段扩展1。这是安全的，因为唯一的条件是邻接，并且我们刚刚验证了新邻接是有效的。 
4. 如果当前颜色与前一个颜色相同，则此时有效段被破坏。 我们将当前段长度重置为 1，因为新段可以从此位置开始。 
5. 记录扫描期间看到的最大段长度。 

答案就是这个运行长度达到的最大值。 

### 为什么它有效

 在任何位置，算法都会维护恰好在该位置结束的最长有效段的长度。 这是可行的，因为唯一的约束涉及相邻元素。 只要最后一个相邻对有效，当前窗口中所有较早的对在窗口扩展时就已经被验证。 当发生违规时，没有有效的段可以跨越它，因此在该索引处重新启动可以保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    best = 1
    cur = 1

    for i in range(1, n):
        if a[i] != a[i - 1]:
            cur += 1
        else:
            cur = 1
        if cur > best:
            best = cur

    print(best)

if __name__ == "__main__":
    solve()
```该解决方案读取数组并维护两个变量。 变量`cur`跟踪以索引结尾的当前有效段的长度`i`。 变量`best`存储所有位置的最大此类值。 

过渡很简单。 当连续元素不同时，我们扩展`cur`。 当它们匹配时，我们重置。 重要的实现细节是重置必须设置`cur`为 1，而不是 0，因为单个元素始终自行形成有效段。 

每一步之后都会更新答案，以确保我们捕获阵列中任何位置的最佳片段。

 ## 边缘情况

 完全均匀的数组，例如`5 5 5 5`导致每一步重置。 该算法将其处理为一系列独立的长度为 1 的段，并且始终更新`best`为 1 并且永远不会错误地跨越无效边界进行合并。 

交替数组，例如`1 2 1 2 1 2`永远不会触发重置。 运行长度连续增长到 6，确认重复的有效转换被正确累积，无需人为截断。 

具有集群重复项的案例，例如`1 2 2 3 4 4 5`表现出交替的生长和重置。 扫描产生段长度`2`，然后重置，然后`3`，表明该算法正确地隔离了最大有效延伸，而不需要显式的分段逻辑。
