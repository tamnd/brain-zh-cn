 ## 解决方案
 ## 问题理解

 有`t`学生和`n`讲座。 如果学生至少出席，则通过`k`讲座。 

讲座在两个礼堂之间轮流进行。 奇数索引的讲座在第一礼堂举行，最多可容纳`a`学生。 指数为偶数的讲座在第二礼堂举行，最多可容纳`b`学生。 

我们可以自由决定哪些学生参加每场讲座，只要不超过相应礼堂的容量。 学生可以参加任何讲座。 

任务是确定最终可以参加的最大学生人数`k`讲座。 

这些值可以大到$10^9$，这立即排除了对学生或讲座的任何模拟。 甚至一个$O(n)$算法是不可能的，因为`n`本身可能是$10^9$。 该解决方案必须仅使用恒定数量的算术运算。 

问题的微妙之处在于，听课并不事先与特定学生绑定。 我们可以完全重新安排各个讲座的出席情况。 重要的是每种礼堂类型的所有讲座的可用座位总数。 

### 不明显的边缘情况

 考虑：```
10 3 4 4 5
```只有三堂课，但通过需要参加五堂课。 没有一个学生能够通过。 

正确答案是：```
0
```仅计算可用座位总数的粗心解决方案可能会错误地产生肯定的答案。 

现在考虑：```
5 4 5 3 3
```每种礼堂类型都有两场讲座。 

可用座位总数：```
2 * 5 + 2 * 3 = 16
```每个通过考试的学生需要出席三次。 

自从$16 / 3 = 5$，五位同学全部都能通过。 

正确答案是：```
5
```这个例子表明，答案不仅仅由较小的礼堂决定。 

另一个重要案例是：```
100000 100000 100000 100000 1
```只需听一堂课即可通过。 

每个学生只能参加一场讲座并通过。 

答案是：```
100000
```学生数量本身就成为限制因素。 

## 方法

 暴力思维会尝试为学生分配讲座并跟踪每个学生参加了多少次讲座。 这很快就变得不可能了，因为学生人数和讲座数量都可能达到$10^9$。 

关键的观察是学生的身份并不重要。 我们只关心有多少出席名额。 

假设我们想要确切地`x`学生通过。 

这些学生中的每一位必须至少参加`k`讲座。 对于所有通过考试的学生，我们至少需要：$$x \cdot k$$出席时段。 

有多少个出席名额？ 

让$$o = \left\lceil \frac{n}{2} \right\rceil$$是奇数索引讲座的数量，$$e = \left\lfloor \frac{n}{2} \right\rfloor$$是偶数索引讲座的数量。 

整个球场的可用座位总数为$$S = o \cdot a + e \cdot b.$$如果我们想要`x`学生要通过，我们需要$$xk \le S.$$乍一看这表明$$x \le \left\lfloor \frac{S}{k} \right\rfloor.$$然而，还有一项限制。 单个学生不能参加超过所有可用讲座的课程。 

学生最多可参加`n`讲座。 如果`k > n`，通过是不可能的。 

如果`k ≤ n`，那么出勤名额总是可以分配给所选择的学生。 唯一剩下的限制是我们不能通过超过`t`学生。 

因此答案就变成了：$$\min\left(t,\left\lfloor\frac{S}{k}\right\rfloor\right).$$唯一的例外是当`k > n`，在这种情况下答案为零。

 ## 算法演练

 1. 计算第一礼堂举办的讲座数量：$$o = \frac{n+1}{2}$$使用整数除法。 
2. 计算第二礼堂举办的讲座数量：$$e = \frac{n}{2}.$$3.如果`k > n`， 输出`0`。 

任何学生不得参加超过`n`讲座。 
4. 计算出勤名额总数：$$S = o \cdot a + e \cdot b.$$5. 计算可以接收至少`k`出席人数：$$\left\lfloor \frac{S}{k} \right\rfloor.$$6. 最终答案是该值和`t`，因为只有`t`学生可用。 

### 为什么它有效

 每个及格学生至少消耗`k`出席时段。 由于正好有`S`整个课程期间可用的出席名额，不超过$\lfloor S/k \rfloor$学生可以通过。 

相反，当`k ≤ n`，任意集合`x`学生与$$xk \le S$$可以分配足够的讲座访问。 唯一消耗的资源是考勤容量，总容量正好是`S`。 由于学生最多可以参加所有`n`听课，不存在额外的瓶颈。 

因此通过学生的最大可行数量恰好是$$\min\left(t,\left\lfloor\frac{S}{k}\right\rfloor\right).$$## Python 解决方案```python
import sys
input = sys.stdin.readline

t, n, a, b, k = map(int, input().split())

if k > n:
    print(0)
else:
    odd_lectures = (n + 1) // 2
    even_lectures = n // 2

    total_slots = odd_lectures * a + even_lectures * b

    print(min(t, total_slots // k))
```实现直接遵循数学推导。 

第一个检查处理唯一不可能的情况，即需要比课程中存在的更多的讲座出席人数。 

奇数和偶数讲座的计数是用整数算术计算的。 对于奇数`n`，第一个礼堂会收到一个额外的讲座，因为讲座编号从 1 开始。 

所有计算都适合 64 位整数。 最大可能值大约是$$10^9 \cdot 10^9 = 10^{18},$$这在 Python 中仍然是安全的。

 ## 边缘情况

 考虑：```
10 3 4 4 5
```该算法首先检查`k > n`。 自从`5 > 3`，没有学生能够参加足够的讲座。 它立即返回：```

```现在考虑：```

```我们计算：$$o = 2,\quad e = 2$$和$$S = 2 \cdot 5 + 2 \cdot 3 = 16.$$然后：$$\left\lfloor \frac{16}{3} \right\rfloor = 5.$$由于正好有五个学生，所以答案是：```
5
```最后考虑：```
100000 100000 100000 100000 1
```每个出勤名额都可以产生一名及格学生，因为只需要一场讲座。 公式给出的值远大于学生人数，所以最终的结果`min(t, ...)`正确地将答案限制为：```
100000
```这说明了为什么学生人数必须包含在最终计算中。
