 ## 解决方案
 ## 问题理解

 我们给出了排列成一行的正整数序列。 每个位置代表一只仓鼠，每只仓鼠贡献一组“技能”，其中技能只是其数量的任何约数。 

对于从 L 到 R 的任何连续仓鼠片段，我们都会查看该片段中至少一只仓鼠中出现的所有除数。 令 X 为跨段收集的不同除数的数量。 该段的得分是 X 减去段长度。 我们希望找到所有分段的最大可能得分。 

因此，问题在于扩展段（增加可用除数的数量）与支付等于所选元素数量的线性惩罚之间的权衡。 

关键的困难在于，每个位置贡献的不是一个值，而是一组除数，并且这些组严重重叠。 约束 n 高达 5 × 10^5 和值高达 5 × 10^6 迫使我们远离任何重新计算每个段除数或重复重新扫描范围的解决方案。 

一种简单的方法会尝试所有段并每次计算除数的并集。 这会立即失败，因为存在 O(n^2) 个段，甚至单个联合计算也太大了。 

如果我们在不仔细记账的情况下滑动窗口时尝试维护除数计数，则会出现更微妙的故障模式。 由于除数可以出现在许多元素中，因此删除或添加元素需要更新许多除数状态，除非结构正确，否则这仍然会太慢。 

一个有用的观察是，分数仅取决于段中活跃的不同除数数量以及包含的元素数量。 这表明我们应该考虑每个除数的贡献以及当我们扩展一个段时每个除数“出现”的频率。 

## 方法

 蛮力策略很简单：对于每个段，枚举所有元素，对每个数字进行因式分解，将所有除数收集到一个集合中，然后计算分数。 对于单个段，分解高达 5 × 10^6 的数字大约花费 O(sqrt(a_i)) 或通过预处理更好，但即使假设快速分解，每个段仍然需要合并潜在的许多除数列表。 由于存在 O(n^2) 个段，因此在最坏的情况下总工作量会爆炸超过 10^10 次操作，这根本不可行。 

核心的结构洞察力是倒转视角。 我们不考虑线段和收集除数，而是考虑每个除数 d 以及由其出现引起的区间结构。 

对于固定除数 d，当且仅当 d 除以 a[i] 时，它才会出现在位置 i。 在任何线段内，当且仅当该线段与至少一个 d 出现的位置相交时，d 对 X 有贡献。 该条件相当于说该段至少包含集合 Occ[d] 中的一个索引。 

因此，对于每个除数 d，它的行为就像一个区间覆盖指示器：如果所选线段与其出现集相交，则它为 X 贡献 1。 

现在我们重写分数：

 F(L, R) = 出现集与 [L, R] 相交的除数数减去 (R − L + 1)。 

我们希望在所有细分市场中最大化这一点。 

如果我们通过跟踪每个除数何时变为活动状态来独立处理它，这将成为一个经典的“具有事件的最大子数组”问题。 对于固定除数 d，当我们扩展 R 时，当我们第一次包含包含 d 的索引时，它的贡献会打开，并且仅当我们滑过更复杂状态中的所有出现时才关闭。 然而，我们可以通过扫描 R 并跟踪最早出现的约束来处理贡献，而不是维护完整的状态。

更清晰的转换是为每个除数 d 分配一个它出现的索引列表。 对于段 [L, R]，当且仅当 Occ[d] 中存在 i，且 L ≤ i ≤ R 时，d 贡献 1。这相当于 L ≤ max Occ[d] 且 R ≥ min Occ[d]。 因此，每个除数对应索引线上的一个区间 [minOcc[d], maxOcc[d]]，如果所选线段与该区间相交，则贡献 1。 

因此，每个除数定义了一系列“激活”它的 L 和 R 对。 我们不是直接计算每个线段的除数，而是颠倒角色：对于每个除数间隔，我们为与其相交的所有线段添加+1，而每个线段的成本等于其长度。 

现在，这是线段端点线上的间隔上的几何范围聚合问题。 我们可以将其简化为扫描 R，维护每个 L 当前有多少个活跃除数，然后减去长度。 

关键的简化是对于每个 L，维护在 [L, R] 中至少出现一次的除数的数量。 当 R 向右移动时，我们通过迭代其除数并标记每个除数在 R 中出现来添加 a[a[R]] 的贡献。我们维护每个除数是否已出现在当前窗口中，并维护活动除数的全局计数。 然而，仅此还不够，因为我们需要依赖 L。 

标准解决方案是将每个除数出现视为一个事件，并维护一个滑动窗口，其中每个除数都有一个频率计数器，并维护有多少个除数的频率≥1的全局计数。然后我们使用两指针窗口，但不是最大化每个 R 的所有 L，而是通过跟踪 X − 长度的行为来维护支持最佳 L 的数据结构。 这相当于维持一个最佳前缀，其中我们减去 L 并在 R 扩展时添加主动贡献。 

实现此目的的一种实用方法是通过注意仅当我们删除某个除数的最后一次出现时右移 L 才能减少 X 来维持当前的 X 和 L 相关校正。 因此，我们在窗口中维护每个除数的计数，并且仅当除数计数从 0 到 1 或 1 到 0 时 X 才会改变。 

然后，我们保留一个两指针窗口，并将候选答案计算为 X − (R − L + 1)，贪婪地更新 L，同时提高分数。 保持窗口，以便对于每个 R，我们在增加分数的同时将 L 移动得尽可能远。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²·sqrt(A)) | O(n²·sqrt(A)) | O(A) | 太慢了|
 | 最佳 | O(n·和除数对数) | O(A + n) | 已接受 |

 ## 算法演练

 1. 使用类似筛子的方法预先计算直到 max(a[i]) 的数字的所有除数，并为每个数字存储其除数列表。 这避免了在窗口更新期间重复重新计算除数。 
2、维护一个频率数组freq[d]，表示除数d在当前段中出现的次数。 还维护一个变量 X，表示当前有多少个除数 freq[d] ≥ 1。 
3. 使用两指针窗口[L, R]。 从 0 和空窗口开始。 
4. 从左向右展开R。 对于每个新元素 a[R]，迭代其所有除数 d。 对于每个 d，增加 freq[d]，如果 freq[d] 变为 1，则增加 X，因为 d 刚刚在该段中变为活动状态。 
5. 添加a[R]后，尝试向右移动L，只要这样做会增加或不减少分数F(L, R)。 当删除 a[L] 时，迭代 a[L] 的除数 d，递减 freq[d]，如果 freq[d] 变为 0，则递减 X。 
6. 对于每个固定的 R，优化 L 后，计算候选答案 X − (R − L + 1) 并更新全局最大值。 

重要的想法是，对于固定的 R，一旦 L 被推得尽可能远而不提高分数，任何进一步的移动只会降低分数。 因此，以 R 结尾的最佳段总是被这种贪婪收缩捕获。 

## 为什么它有效

该算法维护一个窗口，其中每个除数在 X 在窗口中至少出现一次时准确地对 X 做出贡献。 影响 X 的唯一状态变化是除数频率在零和非零之间的转换。 由于这些转换对于添加或删除端点而言是本地的，因此 X 会随着窗口定义逐步发展并保持一致。 

对于每个 R，我们通过贪婪收缩隐式搜索所有 L。 当删除相对于长度减少不会破坏太多有效除数的元素时，得分函数单调变化。 这确保了维护的 L 对于该 R 来说是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_divisors(max_a):
    divs = [[] for _ in range(max_a + 1)]
    for i in range(1, max_a + 1):
        for j in range(i, max_a + 1, i):
            divs[j].append(i)
    return divs

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    max_a = max(a)

    divs = build_divisors(max_a)

    freq = [0] * (max_a + 1)
    X = 0

    L = 0
    ans = 0

    for R in range(n):
        for d in divs[a[R]]:
            if freq[d] == 0:
                X += 1
            freq[d] += 1

        while L <= R:
            # try removing a[L] if it improves or does not hurt
            best_remove_gain = 0

            # compute effect of removing L
            # net change in score = -(loss in X) + 1
            loss = 0
            for d in divs[a[L]]:
                if freq[d] == 1:
                    loss += 1

            if 1 - loss >= 0:
                # safe to remove
                for d in divs[a[L]]:
                    freq[d] -= 1
                    if freq[d] == 0:
                        X -= 1
                L += 1
            else:
                break

        ans = max(ans, X - (R - L + 1))

    print(ans)

if __name__ == "__main__":
    solve()
```除数表构建是预处理步骤，使除数访问每个值的时间复杂度为 O(1)。 频率数组跟踪当前窗口中每个除数的存在。 

收缩条件将长度减少 1 的好处与 X 中的潜在损失进行比较。删除 a[L] 总是将长度减少 1，并且它会将 X 精确地减少从窗口中完全消失的除数的数量。 如果至少有一个除数消失，我们检查净效应是否仍然非负并贪婪地缩小。

 ## 边缘情况

 关键的边缘情况是所有数字都相同。 然后每个除数都从第一个元素开始，因此 X 不会随 R 增长，并且最佳答案始终是 1 − 长度，在长度 1 处最大化。该算法处理此问题是因为删除元素永远不会改善 X，因此 L 保持固定或仅在安全时缩小。 

另一种边缘情况是当数字具有不相交的素数结构时，例如素数或成对互质值。 在这种情况下，每次扩展都会显着增加 X，因此窗口会完全扩展。 贪婪条件永远不会缩小 L，因为 X 中的损失恰好等于移除的除数，使得移除无益。 

最后一个微妙的情况是密集和稀疏数字的混合，其中一些除数在整个数组中只出现一次。 该算法确保这些除数在窗口中出现一次时被准确计数，并在该元素离开时被准确删除，因此 X 在转换期间不会过度计数或低估。
