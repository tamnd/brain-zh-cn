 ## 解决方案
 ## 问题理解

 我们得到一串排列成圆圈的数字，并且可以将其旋转任意次。 每次旋转都是向左或向右循环移位一次。 选择旋转后，我们查看生成的线性字符串并测量其“潜力”，即相等数字的最大连续线段的数量。 

例如，在像这样的字符串中`112223`，各段是`11 | 222 | 3`，因此势为 3。如果旋转减少了此段数，则认为旋转更好。 在实现尽可能少的段数的所有旋转中，我们必须选择需要与原始字符串进行最少循环移位的旋转，并输出该数字并带有指示方向的符号：左移为负，右移为正。 

关键的观察结果是，旋转不会改变相邻角色关系的多重集，它们只会改变我们“切割”圆圈以形成一条线的位置。 这意味着每次旋转都具有几乎相同的结构，除了一对因剪切而改变的邻接对之外。 

n 最大为 10^6 的约束迫使我们远离任何二次方。 任何尝试所有旋转并从头开始重新计算段计数的解决方案都需要 O(n^2)，这太慢了。 我们需要一种 O(n) 方法来隐式评估所有旋转。 

一个微妙的边缘情况是所有字符都相同。 在这种情况下，每次旋转都会产生一个块，因此每个答案都是同样最优的，我们必须返回零移位。 

另一个重要的情况是当存在多个有效的最佳切割位置时。 例如，如果字符串有多个位置相邻字符不同，则多次旋转会同等地减少潜在的可能性，并且我们必须选择在循环移位距离方面最接近原始位置的一个。 

## 方法

 蛮力方法会尝试每次旋转。 对于每次旋转，我们将构造旋转的字符串并计算相邻字符不同的次数。 这给出了段的数量。 这是正确的，因为段计数很容易从转换中计算出来，但每次旋转的成本为 O(n)，并且有 n 次旋转，导致总操作为 O(n^2)，这对于 n 高达 10^6 来说太大了。 

关键的结构见解是段数仅取决于我们切割圆绳的位置。 如果我们将绳子视为圆形，则每次旋转都对应于选择起始切割位置。 圆内所有相邻对保持不变； 只有旋转字符串的最后一个字符和第一个字符之间的边缘取决于剪切。 这意味着我们可以预先计算所有邻接不匹配一次，然后通过检查切割是否打破不匹配来在恒定时间内评估每次旋转。 

这将问题简化为扫描我们可能切割圆的所有位置，计算每个位置的结果线段计数（O(1)），并选择最佳的一个。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将每个位置 i 视为字符 i−1 和 i 之间可能的“剪切”（带有环绕）。 这里的切割意味着旋转的字符串从 i 开始。

1. 计算哪些相邻对在原始循环意义上不同。 我们实际上不需要显式地构建循环结构； 我们只需要知道每个位置 s[j] 和 s[j−1] 是否不同。 
2. 观察圆内不匹配的相邻对的总数是固定的。 设该值为T。 
3. 对于在位置 i 处选择的切割，唯一受影响的邻接是 s[i−1] 和 s[i] 之间的边。 在线性表示中，该边成为边界并且不被算作内部转换。 
4. 因此，如果 s[i−1] 和 s[i] 不同，则此处的剪切将从总数中删除一个转换，从而将段计数减少 1。 如果它们相等，则段计数保持不变。 
5. 对于每个 i，计算它是否是“有益的切割”，即 s[i−1] != s[i]。 对于每个这样的 i，它实现了尽可能少的段数。 
6. 在所有有益的剪切中，选择需要对原始字符串进行最少操作的剪切。 i 的旋转可以通过 i 次左移或 n−i 次右移来实现，因此我们选择这两个值中较小的一个。 如果相等，则任一方向都可以。 
7. 如果我们使用左移，则输出所选移位带有负号，如果使用右移则输出正号。 

### 为什么它有效

 任何旋转的线段计数仅取决于存在多少个相邻的不相等对，并且除了单个切割边缘之外，所有旋转共享相同的一组圆形邻接。 这意味着每次旋转与其他旋转的不同之处仅在于它是否恰好删除了一个转换。 因此，最佳值仅取决于我们是否在不同的边界上进行切割。 一旦这个问题得到解决，最小化操作就变成了一个循环上的简单距离问题，与段结构无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    if n == 1:
        print(0)
        return

    best_i = None

    for i in range(n):
        if s[i] != s[i - 1]:
            if best_i is None:
                best_i = i
            else:
                # choose closer rotation to 0
                def cost(x):
                    left = x
                    right = n - x
                    if left <= right:
                        return left, -left
                    return right, right

                c_best = cost(best_i)
                c_cur = cost(i)

                if c_cur < c_best:
                    best_i = i

    if best_i is None:
        print(0)
        return

    left = best_i
    right = n - best_i

    if left <= right:
        print(-left)
    else:
        print(right)

if __name__ == "__main__":
    solve()
```该代码首先识别相邻字符不同的所有有效剪切位置。 这些正是实现尽可能最少的段数的旋转。 然后，它通过比较左移和右移成本，在其中选择最接近原始配置的旋转。 如果不存在这样的切割，则字符串是统一的，并且任何操作都不会改变任何内容。 

一个微妙的点是索引：将 i 视为 i−1 和 i 之间的切割，自然会使用 Python 的负索引处理环绕，其中 s[i-1] 正确地引用当 i = 0 时的最后一个字符。

 ## 边缘情况

 当所有字符都相同时，相邻字符不存在不同的位置。 算法正确离开`best_i`未设置并直接输出零，因为每次旋转都会产生一个片段，并且没有旋转可以改善或区分结果。 

当存在多个有效剪切位置时，例如在交替或重复模式中，该算法独立评估每个候选剪切并将循环距离与原始位置进行比较。 由于成本函数在圆上是对称的，因此所选择的结果始终是最接近的有效切割，并且当左移不比右移差时，可以通过优先选择左移来自然地解决平局。 

当最佳切割位于位置 0（意味着不需要旋转）时，左侧和右侧成本都为零。 该算法输出零，正确反映原始字符串已经是最佳的。
