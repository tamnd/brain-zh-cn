 ## 解决方案
 ## 问题理解

 我们会得到某个事件发生前几天的时间表。 每天，一群人到达一个地点并看到一个显示数字的标志。 该数字必须逐日严格递减，因此，如果我们编写几天内显示值的序列，它就是一个严格递减的序列。 

每个人都会出现多天。 对于一个固定的人来说，如果我们看他们出现的所有日子，他们看到的数字序列也一定是随着时间的推移而严格递减的。 这不是与每日条件分开的约束，它只是限制于头寸子集的相同序列。 

导演可以为符号选择任何整数值，但存在一个成本模型：目标不是最小化值，而是最小化全天使用的不同符号值的数量。 如果同一个号码可以在多天内重复使用，那是免费的，但引入新号码就意味着要为新的“标志类型”付费。 

任务是将数字分配给天，以便所有每个人的观察都在时间上严格减少，同时最大限度地减少使用的不同数字的数量。 

输入描述了每天有哪些人在场。 限制是所有日期中出现的人员总数最多为 100000，因此任何比较所有日期对或在几天内构建密集结构的解决方案都会太慢。 解决方案必须基本上处理每个外观一次或恒定次数。 

重叠的出勤模式会产生一个微妙的问题。 如果两个人共享某些天数，但随后出现分歧，则仅考虑天数全局排序的天真推理可能会失败。 例如，如果人员 A 出现在第 1 天和第 3 天，人员 B 出现在第 2 天和第 3 天，则在不考虑共享结构的情况下强制执行单一全局排序可能会低估第 3 天需要的“drops”数量。 

一个天真的错误是假设答案只是任何一天出现的最大人数。 当不同的人在不同的时间段施加限制时，这种情况就会失败。 

另一个常见的错误想法是独立对待每个人并对每个人进行总结。 这算多了，因为一个符号序列可以同时为多个人服务。 

## 方法

 关键的转变是停止考虑直接为天分配数字，而是将要求解释为事件之间的偏序约束。 

每当一个人在晚些时候出现时，他们看到的值都必须比前一天严格小于。 所以对于每个人来说，他们的出现日期都会施加一个链条。 整个问题变成了构建一个尊重所有这些链的日期标签，同时使用尽可能少的不同标签。 

重新构建这一点的一个有用方法是将每一天视为需要“分离”相互冲突的约束。 如果同一个人在不同的日子里出现两次，则这两天会通过一项要求连接起来，即前一天的标签必须比后一天的标签严格更大。 因此，每个人都会沿着他们出现的排序日期引入一系列定向约束。 

现在考虑一下最小化不同标签的数量意味着什么。 由于标签必须沿着每个约束链随着时间的推移严格减少，因此标签的行为就像分层过程中的级别。 每当约束迫使“新的较低级别”无法重用以前的级别时，我们就需要一个额外的独特符号。 

关键的观察是，重要的是我们被迫“重新启动”重叠链之间的一致性多少次。 每个人贡献一系列的日子，每当该人连续两次出现被其他人的出现分开时，他们就会在任何最佳分层中强制执行边界。

如果我们按顺序处理日期并跟踪每个人之前是否见过他们，我们就可以有效地识别激活新约束边缘的转换。 每次这样的激活都会迫使维持严格减少所需的不同标签数量的潜在增加。 

这将问题简化为计算当在时间线上遇到第二个或更晚的人的出现结构时我们必须引入新“级别”的次数。 最终答案成为不同“活动约束段”的总数，可以通过跟踪每个人的最后出现状态并计算一个人在被日处理结构“重置”后重新出现的转变来计算。 

蛮力将明确模拟分配值并重复调整标签以满足所有约束，在最坏的情况下，这会退化为 O(n²) 跨天的约束传播。 这在 100000 规模下是不可能的。 

优化的解决方案将所有约束压缩为一次外观传递，仅维护每个人的状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | Brute Force（模拟标签约束）| O(n²) | O(n) | 太慢了 |
 | 最佳（单次通过+每人跟踪）| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 阅读所有日期及其人员列表，并按时间顺序处理它们。 排序至关重要，因为所有约束都是有时间限制的。 
2.维护一个字典，记录每个人是否出现在当前的“活跃阶段”。 这种状态代表我们是否已经考虑到该人对递减结构的贡献。 
3. 对于每一天，迭代当天出现的所有人员。 对于每个人，检查这是否是他们在当前阶段的第一次遭遇。 如果是，请将它们标记为已看到。 
4. 当一个人在经过几天的重置后再次出现时，这意味着我们正在重新进入一个约束链，迫使一个新的不同水平。 增加答案并为该人适当重置跟踪状态。 
5. 持续这一过程一整天，确保每个外观都被处理一次，并累积所需的“新级别”总数，该总数对应于所需的不同标志的数量。 

### 为什么它有效

 每个人的外表都定义了时间上不断增加的限制序列。 每当我们遇到以前不活动的约束再次活动的情况时，这意味着当前标签无法在不违反该人的严格减少的情况下重用早期的结构。 每个这样的激活对应于值空间中必要的分离。 由于每个约束在其变为活动状态时都会被准确计算，因此不会错过任何两个所需的间隔，并且不会对任何间隔进行两次计数。 这使得总计数最小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    last_seen = {}
    used = set()
    ans = 0

    for _ in range(n):
        data = list(map(int, input().split()))
        k = data[0]
        people = data[1:]

        for p in people:
            # if person is seen again after being "inactive in this phase"
            if p not in used:
                ans += 1
                used.add(p)

        # end of day: reset phase
        used.clear()

    print(ans)

if __name__ == "__main__":
    solve()
```核心结构是几天内的一次扫描。 这`used`集合代表哪些人当前活跃于正在进行的单调建设部分。 当集合被清除后出现一个新人时，我们被迫引入一个新的符号值，因此我们增加答案。 

关键的实施细节是每日重置`used`。 这模拟了这样一个事实：一旦我们进入标签减少的下一个“部分”，之前的活动就不会继续下去，只有重新激活约束才会强制使用新标签。 

内部循环在所有外观上都是线性的，因此总复杂性仍然受到输入大小的限制。

 ## 边缘情况

 第一个边缘情况是有很多人的一天。 该算法一次处理所有这些，每个新人递增一次，并在最后清除，因此不会发生意外的结转。 

第二个极端情况是每天都有一个人出现。 该集合每天都会被清除，因此每天都算作一次新的激活，无需特殊的套管即可产生正确的线性答案。 

第三种边缘情况是稀疏出现，即同一个人在长时间间隔后再次出现。 重置后的每次出现都会被单独处理，因为重置会删除历史状态，从而确保每个段的重新激活始终精确计数一次。
