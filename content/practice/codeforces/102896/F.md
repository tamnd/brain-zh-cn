 ## 解决方案
 ## 问题理解

 我们得到了二维整数网格上的点的集合。 任务是确定是否可以选择形成正方形顶点的四个点。 正方形不限于轴对齐，因此可以任意旋转，只要所有四个边都相等并且所有角度都是直角即可。 

输入可以理解为一组几何对象，其中每个对象都是平面上的一个点。 输出是一个简单的决定：是否存在这些点中的任何四元组可以充当有效正方形的角点。 

关键的困难在于正方形不是由输入中的局部邻接或顺序定义的。 任何四个点的子集可能会也可能不会形成正方形，并且方向不固定。 这立即排除了密集输入中的天真的组合检查：检查所有四元组将需要检查$O(n^4)$的候选人，这甚至远远超出了可行的范围$n$大约几千。 

第二个微妙的问题是，正方形的坐标可能会以不明显的方式重叠。 例如，两对不同的点可以通过不同的对角线定义相同的几何正方形。 任何假设点的固定顺序或仅依赖于轴对齐的解决方案在旋转配置（例如形成 45 度正方形的点）上都会失败。 

当多个点位于多个对角线重合的圆形配置上时，就会出现典型的边缘情况。 

例如，考虑以下几点：```
(0, 0), (1, 1), (1, 0), (0, 1)
```这显然形成了一个正方形，但是任何仅通过最小/最大坐标检查轴对齐矩形的方法在这里仍然可能成功。 然而，旋转相同的结构会打破这种天真的逻辑：```
(0, 0), (2, 1), (1, 2), (-1, 1)
```这仍然是一个有效的正方形，但轴对齐启发式完全失败。 

挑战在于在不显式枚举所有四元组的情况下检测正方形。 

## 方法

 暴力方法将迭代四个点的所有组合并直接检查它们是否形成正方形。 给定四个点，我们可以计算所有成对距离并验证我们是否有四个相等的边和两条相等的对角线。 这在逻辑上是简单且正确的，因为它直接使用几何定义。 

然而，四元数的个数是$\binom{n}{4}$，其增长顺序为$O(n^4)$。 为了$n = 2000$，这已经超过了$10^{12}$检查，每次检查都涉及恒定时间距离计算。 即使经过深度优化的代码也无法承受这种规模。 

关键的观察是，正方形是由其对角线而不是其边唯一确定的。 如果我们选取任何两个可能是正方形对角的点，则其他两点在几何上确定：它们必须位于距线段中点相等的距离，并且方向垂直。 

这导致了一个更加结构化的视图：我们选择两个点作为候选对角线，而不是选择四个点。 每对点都定义了一条潜在的对角线，我们可以计算它的中点和平方长度。 如果两个不同的对共享相同的中点和相同的平方距离，则它们形成正方形的两条对角线。 

这将问题简化为配对边而不是四元组。 我们存储按键分组的所有点对，其中包含：

 中点（以确保它们具有相同的中心）和平方距离（以确保对角线长度相等）。 任何包含至少两对的组都表示一个有效的正方形。 

这将问题从四个元素的组合爆炸转变为对的散列问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（4 点检查）|$O(n^4)$|$O(1)$| 太慢了 |
 | 对+对角线散列|$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 1. 将所有点读入数组。 每个点都被视为正方形的潜在顶点，但我们不假设除此之外的任何结构。 
2. 迭代所有无序点对。 每对都被视为正方形的候选对角线。 这是从四点推理到两点推理的关键转变。 
3. 对于每对$(p_i, p_j)$，将中点计算为一对双坐标$(x_i + x_j, y_i + y_j)$。 我们避免浮点除法，因为中点必须与同一个正方形的对角线完全匹配。 
4. 计算线段的平方距离：$(x_i - x_j)^2 + (y_i - y_j)^2$。 这标识了没有平方根的对角线长度，保留了整数运算。 
5. 使用由以下键控的哈希映射$((x_i + x_j, y_i + y_j), dist)$。 对于每一对，检查该密钥是否已存在。 如果确实如此，我们就找到了与相同正方形几何形状匹配的另一条对角线，这意味着存在形成正方形的四个点。 
6. 如果处理完所有对后没有找到匹配对，则断定不存在正方形。 

### 为什么它有效

 每个正方形都有两条对角线。 这些对角线共享相同的中点并且长度相等。 相反，如果两条线段共享中点和长度，并且它们的端点不同，则四个端点必须形成对角线相等的平行四边形，这迫使它成为正方形。 该算法利用了正方形和有效对角线对之间的双射，确保不会遗漏任何有效配置，也不接受任何无效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    
    seen = {}
    
    for i in range(n):
        x1, y1 = pts[i]
        for j in range(i + 1, n):
            x2, y2 = pts[j]
            
            mx = x1 + x2
            my = y1 + y2
            dx = x1 - x2
            dy = y1 - y2
            dist = dx * dx + dy * dy
            
            key = (mx, my, dist)
            if key in seen:
                print("YES")
                return
            seen[key] = 1
    
    print("NO")

if __name__ == "__main__":
    solve()
```该解决方案依赖于枚举所有点对并将每对编码为潜在方形对角线的规范表示。 该字典确保对先前见过的对角线进行恒定时间查找。 

一个微妙的实现细节是使用双倍中点坐标而不是实际中点。 这避免了浮点精度问题并确保仅使用整数进行精确匹配。 另一个重要的细节是必须使用平方距离而不是欧几里德距离，因为平方根会引入不必要的精度和性能成本。

该轨迹演示了算法如何通过对角线而不是边来识别结构。

 ## 边缘情况

 一种微妙的情况是多个方块重叠或共享点。 该算法自然地处理这个问题，因为它不假设唯一性； 它只需要数据集中任意位置的两条匹配的对角线。 

例如：```
4
0 0
1 1
1 0
0 1
```立即形成一个正方形，但在其他地方添加额外的点形成另一个正方形不会产生干扰，因为每一对都是独立处理的。 

另一种情况是当点重复或出现退化线段时。 重复点不会产生误报，因为相同的点对不会形成具有不同端点的有效对角线； 它们仅加强现有的键，而不引入新的几何结构。 

最后，具有许多相等距离但不同中点的情况不会发生冲突，因为中点编码将它们完全分开。
