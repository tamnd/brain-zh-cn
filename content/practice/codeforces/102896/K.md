 ## 解决方案
 ## 问题理解

 一家商店给了我们几包气球。 每个包裹的气球上都写有价格和多组数字。 凯特想要购买一个包裹，这样，使用该包裹内的数字，她可以形成年份“2021”，这意味着她至少需要两个零，两个一，并且一二，一二，一一已经计算在内，所以总共要求减少到数字2的两个副本，数字0的两个副本，数字1的一个副本是不够的，因为目标具体是“2 0 2 1”，所以确切的要求是数字计数：2出现两次，0 出现一次，1 出现一次。 

每个包都可以用作一袋数字，只要数字存在，我们就可以自由地从中选取数字。 任务是找到满足这些数字要求的最便宜的包索引，或者报告没有包可以满足它们。 

输入大小足够小，可以独立检查每个包。 由于最多有大约一千个包裹，每个数字串的长度最多为一百个，即使对每个包裹进行简单的频率扫描也很容易足够快。 任何比大约 10^8 原始操作更糟糕的方法仍然是安全的，但我们远远低于这个水平。 

主要的边缘情况来自对多重性的误解。 一个常见的错误是假设需要将“2021”视为子字符串，但数字可以任意重新排列。 另一个微妙的问题是忘记可能会出现多个零或一个，并且只计算物质。 最后，一个包可能包含所有必需的数字，但数量不足，例如只有一个“2”。 

一个失败的天真的想法的例子：如果我们只检查字符串是否至少包含字符“2”、“0”、“1”一次，我们就会错误地接受像“201”这样缺少第二个“2”的包。 

另一个边缘情况是当多个包裹满足条件时，我们必须选择最低价格，而不是第一个有效的价格。 

## 方法

 暴力方法几乎已经是最佳的。 对于每个包裹，我们统计数字0到9的出现次数，并验证计数是否满足形成“2021”的要求。 这是正确的，因为数字位置并不重要，重要的是频率。 

暴力破解的成本很简单：对于 n 个包中的每一个，我们扫描长度最多为 100 的字符串，因此总工作量约为 100,000 个字符检查。 这是可以忽略不计的。 

没有更深层次的组合结构可供利用，因为每个包都是独立的，并且一旦计算出频率，检查就是恒定时间的。 唯一的“优化”是在计数已经超过所需阈值时提前停止，但即使这样也没有必要。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（计算每个包的位数）| O(n·L) | O(n·L) | O(1) | O(1) | 已接受 |
 | 最优（与暴力相同）| O(n·L) | O(n·L) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取包裹数量并初始化一个变量，将最佳答案存储为“未找到”，将最佳价格存储为无穷大。 这确保我们可以统一比较所有候选人。 
2. 对于每个包裹，读取其价格和数字字符串，然后通过扫描该字符串一次来计算大小为 10 的频率数组。 这一步是必要的，因为我们需要精确的多重性，而不仅仅是存在。 
3. 检查包裹是否包含至少两个“2”、至少一个“0”、至少一个“1”。 这直接编码了我们是否可以从多重集中构造“2021”。 
4. 如果满足条件并且价格小于目前看到的最佳价格，则更新此套餐索引的最佳答案。 这维持了有效候选者的运行最小值。 
5.处理完所有包后，如果存在最好的索引，则输出，否则输出0。 

### 为什么它有效

每个包都是独立的，因此问题简化为对一组候选包的过滤和最小化模式。 频率检查既必要又充分，因为任何数字排列都是允许的。 维护有效包中的最小索引可以正确捕获所需的输出，因为我们总是与最知名的有效解决方案进行比较。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(cnt):
    return cnt[2] >= 2 and cnt[0] >= 1 and cnt[1] >= 1

n = int(input())
best_price = float('inf')
best_idx = 0

for i in range(1, n + 1):
    parts = input().split()
    p = int(parts[0])
    s = parts[1]

    cnt = [0] * 10
    for ch in s:
        cnt[ord(ch) - 48] += 1

    if ok(cnt):
        if p < best_price:
            best_price = p
            best_idx = i

print(best_idx)
```该解决方案是围绕直接数字频率表构建的。 辅助检查隔离了形成“2021”的条件，这避免了循环内的重复逻辑。 索引跟踪是仔细进行的，以尊重问题陈述中基于 1 的索引。 

一个常见的实施错误是忘记重置每个测试用例的频率数组或混淆 ASCII 转换。 使用`ord(ch) - 48`确保从字符到数字值的正确映射。

 ## 边缘情况

 像“202”这样的包即使在视觉上与目标相似，也会失败，因为它只包含一个“1”。 该算法正确地拒绝了它，因为频率检查严格执行所有必需的计数。 

像“2220011”这样的包包含足够的数字，但数字过多。 该算法接受它，因为多余的数字不会影响可行性，只有最小阈值很重要。 

具有相同价格的多个有效包裹的情况可以自然地处理，因为该算法保持第一个遇到的最小值，并且该问题允许任何有效索引处于平局。
