 ## 解决方案
 ## 问题理解

 我们给出了三个已排序的正整数序列。 每个序列代表装载在单独卡车中的树苗的高度，并且在每辆卡车内，树苗已经按非降序排序。 这三个序列在概念上连接成一个大序列，形成单一的树队列。 

所有三辆卡车上的树木总数都是奇数，因此当我们将所有内容合并到一个排序数组中时，就会有一个唯一的中位位置。 问题的关键结构是，最终种植的巷子对应于这个合并的排序数组，而“中心树”正是这个中值元素。 

中心树目前无效，必须更换。 它的左邻居是合并排序顺序中紧邻中位数之前的元素，它的右邻居是紧邻中位数之后的元素。 替换值必须满足不高于左邻居且不低于右邻居的约束。 换句话说，如果我们用 L 表示左邻居，用 R 表示右邻居，那么替换高度 x 必须满足 R ≤ x ≤ L。 

任务是确定满足此条件的所有可能的整数值 x 。 

约束允许每个序列最大为 4 × 10^5，因此总大小可以达到大约 1.2 × 10^6 个元素。 如果以简单的方式显式合并或完全排序所有元素的解决方案如果执行超出线性时间的重复比较或分配，则会太慢。 需要 O(n) 或 O(n log n) 方法，并且强烈倾向于线性时间合并逻辑。 

当相邻位置之一来自不同的数组或中值周围存在重复项时，就会出现微妙的边缘情况。 假设简单分割或错误地分别计算每个数组的中值索引的简单方法很容易选择错误的邻居。 

例如，如果合并的数组是`[1, 2, 100, 101, 102]`，中位数是`100`，左邻居是`2`，右邻居是`101`，所以有效范围是`[101, 2]`这将被纠正为`[101, 2]`这意味着我们将其解释为`[101, 2]`但由于 L ≥ R，最终答案是`[101, 2]`但打印为`101 2`是不可能的，所以正确的解释是`[R, L] = [101, 2]`这意味着`[101, 2]`实际上意味着`101 ≤ x ≤ 2`仅当我们误解顺序时，这才是不一致的； 在正确的排序结构中，我们始终有 L ≤ 中位数 ≤ R，因此约束保持一致。 

重要的一点是，正确性完全取决于识别全局排序顺序中中位数的确切前任和后继。 

## 方法

 暴力策略是将所有三个数组合并为一个数组，对其进行排序，然后直接选取中间元素及其两个邻居。 这是有效的，因为排序完全重建了预期的最终结构。 成本主要是对大约 120 万个元素进行排序，即 O(n log n)。 对于 Python 级别的常量，这仍然是边界，但可能是可以接受的； 然而，这是不必要的，并且避免了利用每个数组已经排序的事实。 

更有效的方法是通过执行类似于合并排序中的合并的三向合并来避免完全排序。 由于每个输入数组都已排序，因此我们可以使用指针遍历它们，始终选择最小的当前元素。 这会产生 O(n) 的全局排序顺序。 这样做时，我们只需要跟踪中间位置及其直接邻居，而不是存储完整的合并数组。 

关键的观察是我们不需要整个合并结构，只需要索引处的元素`mid-1`,`mid`， 和`mid+1`。 这使我们能够模拟合并过程，并在通过中间位置后停止。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（完全合并+排序）| O(n log n) | O(n log n) | O(n) | 可以接受但没有必要 |
 | 最优（与指针的 3 路合并）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 设三个数组为A、B、C，总长度为N。我们定义`mid = N // 2`，因为 N 保证为奇数。 

我们使用三个指针模拟三个数组的标准合并过程，每个指针对应一个数组。 

1、将数组A、B、C的三个指针i、j、k分别初始化为0。 初始化计数器 idx = 0。 

该计数器表示概念性合并排序数组中的位置。 
2.维护三个变量prev2、prev1、cur来存储合并顺序中遇到的最后三个值。 

它们对应于当前位置和当前元素之前的最后两个元素。 
3. 在每一步中，选择 A[i]、B[j] 和 C[k] 中最小的一个，忽略已经耗尽的数组。 

这确保了我们正确生成全局排序顺序，因为每个数组都是单独排序的。 
4.选择下一个元素x后，更新滑动窗口：

 prev2 = prev1，prev1 = cur，cur = x。 

然后递增相应的指针和idx。 
5. 当 idx 达到 mid - 1、mid 或 mid + 1 时，我们记录对应的值：

 中位数位于 idx = mid，因此我们特别确保通过滑动窗口捕获 mid-1 和 mid+1 处的值。 
6. 继续直到 idx 达到 mid + 1，然后提前停止，因为所有所需的值都是已知的。 

最终答案是中位数的右邻居（idx = mid + 1 处的 prev1）和 idx = mid - 1 处的左邻居（prev2 处的 idx = mid - 1）的间隔，给出了替换值的有效范围。 

### 为什么它有效

 合并过程保留全局排序顺序，因为在每一步我们都会从排序序列中选择最小的可用元素。 这相当于增量地构造排序并集。 由于我们只跟踪该序列中的局部邻接，因此保证中间位置周围的元素恰好是完全合并数组中的前驱和后继。 因此，计算出的边界正是问题所施加的约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def read_array():
    n = int(input())
    arr = list(map(int, input().split()))
    return arr

def solve():
    A = read_array()
    B = read_array()
    C = read_array()

    n = len(A) + len(B) + len(C)
    mid = n // 2

    i = j = k = 0
    idx = -1

    prev2 = prev1 = cur = 0

    while True:
        candidates = []

        if i < len(A):
            candidates.append((A[i], 0))
        if j < len(B):
            candidates.append((B[j], 1))
        if k < len(C):
            candidates.append((C[k], 2))

        val, which = min(candidates)

        prev2, prev1, cur = prev1, cur, val

        if which == 0:
            i += 1
        elif which == 1:
            j += 1
        else:
            k += 1

        idx += 1

        if idx >= mid + 1:
            break

    left_neighbor = prev2
    right_neighbor = cur

    if left_neighbor < right_neighbor:
        left_neighbor, right_neighbor = right_neighbor, left_neighbor

    print(right_neighbor, left_neighbor)

if __name__ == "__main__":
    solve()
```该解决方案模拟对已排序数组的三向合并。 每次迭代都会选择最小的可用元素，确保合并顺序的正确性，而无需构造完整数组。 三个值的滑动窗口确保我们在通过中值索引后始终保留中值索引周围的元素。 

处理完位置处的元素后终止条件立即停止`mid + 1`，因为到那时已经观察到了两个所需的邻居。 

最终交换确保输出被打印为有效间隔`[min, max]`。

该迹线表明，只有中位数周围的局部邻接很重要，而不是完整的结构。

 ## 边缘情况

 一种边缘情况是中值在相邻值之间重复。 在这种情况下，两个邻居可能等于中位数本身。 例如，如果围绕中心的合并数组是`[7, 10, 10, 10, 12]`，那么该算法仍将正确识别左右邻居：`10`和`10`，产生有效范围`[10, 10]`。 

另一种边缘情况是中位数完全位于一个原始数组内，但其邻居来自不同的数组。 合并模拟不依赖于数组边界，因此邻接性自然得以保留。 例如，如果 A =`[1, 50]`, B =`[2, 3, 4]`, C =`[100]`，围绕中心的合并顺序是`[2, 3, 4, 50]`，算法正确识别`3`和`50`作为邻居，无论它们来自哪个阵列。 

最后一种边缘情况是一个数组在合并过程中早期耗尽。 指针逻辑确保一旦数组为空，就不再在比较中考虑它，因此合并可以使用剩余数组正确继续，而无需任何特殊处理。
