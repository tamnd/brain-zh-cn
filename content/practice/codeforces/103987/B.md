 ## 解决方案
 ## 问题理解

 我们得到一个表示一维细胞线的二进制字符串。 每个单元格保存 0 或 1。我们被要求精确模拟称为规则 110 的元胞自动机的一个步骤。每个位置的下一个状态仅取决于其当前值及其左右邻居的值。 字符串外部的单元格被视为 0，因此从概念上讲，数组的两端都被固定的零包围。 

该任务纯粹是局部变换：每个位置 i 都会根据 (i−1, i, i+1) 形成的三元组生成一个新值。 所有更新同时发生，这意味着位置 i 处的新值必须仅根据原始字符串计算，而不是根据部分更新的结果计算。 

字符串长度最大为100,000。 这立即排除了比线性时间更糟糕的情况。 任何尝试重复移动或模拟每个单元串上的多次传递的方法都会向二次行为漂移并在极限下失败。 

一个常见的错误是就地更新字符串。 例如，如果我们在计算 s[i+1] 时覆盖 s[i]，邻居信息就会被损坏。 另一个微妙的问题是忘记了边界条件：最左边的单元格在其左侧使用虚拟 0，最右边的单元格在其右侧使用虚拟 0。 例如，如果输入是`1`，正确的邻域是`0 1 0`，所以我们仍然必须产生由该三元组确定的输出，而不是假设缺失的邻居在代码中隐式消失。 

## 方法

 直接模拟读取每个索引 i 并显式检查其左、中、右邻居。 由于规则是完全本地且固定的，因此我们可以在恒定时间内计算每个输出字符。 暴力版本已经正确地执行了这个想法，但唯一的低效率来自于粗心的实现，例如重新计算子字符串或为每个索引重复切片字符串，这会增加开销。 

关键的观察结果是，自动机规则在各个位置上是无状态的：每个输出单元仅依赖于三个输入值，仅此而已。 这意味着我们可以在一次传递中独立处理所有位置。 不存在需要迭代直至收敛的传播或依赖链。 整个下一代是从三元组到单个位的直接映射。 

这将问题简化为扫描一次字符串并对 8 个可能的三元组应用固定查找。 我们可以显式地对规则 110 真值表进行编码，也可以直接从语句中导出它。

 ## 算法演练

 我们通过独立评估每个位置来计算下一个状态字符串。 

1. 扩展字符串的思想，在两端添加虚拟零，以便每个索引都有明确定义的邻居。 这避免了主逻辑中特殊的外壳边界位置。 
2. 对于从 0 到 n−1 的每个位置 i，形成一个三元组 (a, b, c)，其中 a 是左邻居（如果 i 是 0，则为 0），b 是当前单元格，c 是右邻居（如果 i 是 n−1，则为 0）。 这个三元组完全决定了下一个值。 
3. 将规则 110 映射应用到三元组。 该规则是固定的，可以硬编码为小型查找或实现为条件检查。 由于只有八个可能的三元组，因此这是恒定时间的工作。 
4. 将计算值存储到新数组中，而不是覆盖原始字符串。 这种分离保证了所有决策都基于原始配置。 
5. 处理完所有索引后，输出构造的结果字符串。 

### 为什么它有效

 自动机被定义为同步局部更新规则：每个单元的下一个状态仅取决于其前一个状态的直接邻域。 由于从原始数组查看时，邻域在各个索引之间是独立的，因此每个计算都是隔离的。 使用单独的输出数组保留了这种独立性，确保计算值不会在同一步骤中影响另一个值。 这保证了结果字符串与同时应用的规则相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def rule110(a, b, c):
    # Rule 110 truth table:
    # 111 -> 0
    # 110 -> 1
    # 101 -> 1
    # 100 -> 0
    # 011 -> 1
    # 010 -> 1
    # 001 -> 1
    # 000 -> 0
    if a == '1' and b == '1' and c == '1':
        return '0'
    if a == '1' and b == '1' and c == '0':
        return '1'
    if a == '1' and b == '0' and c == '1':
        return '1'
    if a == '1' and b == '0' and c == '0':
        return '0'
    if a == '0' and b == '1' and c == '1':
        return '1'
    if a == '0' and b == '1' and c == '0':
        return '1'
    if a == '0' and b == '0' and c == '1':
        return '1'
    return '0'

n = int(input())
s = input().strip()

res = []

for i in range(n):
    a = s[i - 1] if i > 0 else '0'
    b = s[i]
    c = s[i + 1] if i + 1 < n else '0'
    res.append(rule110(a, b, c))

print("".join(res))
```该代码读取输入一次，并在列表中增量构造输出以提高效率。 辅助函数显式地对规则进行编码，避免任何可能掩盖正确性的位操作技巧。 

边界处理是内联完成的：索引 -1 和索引 n 被视为零，而不修改字符串。 这避免了额外的内存开销并保持所有位置的逻辑对称。

现在考虑单细胞输入`s = "1"`。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(sys.stdin.readline())
    s = sys.stdin.readline().strip()

    def rule(a, b, c):
        if a == '1' and b == '1' and c == '1': return '0'
        if a == '1' and b == '1' and c == '0': return '1'
        if a == '1' and b == '0' and c == '1': return '1'
        if a == '1' and b == '0' and c == '0': return '0'
        if a == '0' and b == '1' and c == '1': return '1'
        if a == '0' and b == '1' and c == '0': return '1'
        if a == '0' and b == '0' and c == '1': return '1'
        return '0'

    res = []
    for i in range(n):
        a = s[i-1] if i > 0 else '0'
        b = s[i]
        c = s[i+1] if i+1 < n else '0'
        res.append(rule(a,b,c))

    return "".join(res)

assert run("3\n010\n") == "110", "basic propagation"
assert run("5\n00000\n") == "00000", "all zeros"

 ## 边缘情况

 最小长度字符串对边界处理最敏感。 用于输入`s = "0"`，邻域是`(0,0,0)`，映射到`0`，所以输出保持`"0"`。 当索引超出范围时，该算法明确地将两个邻居视为零，因此单个单元格的两侧表现一致。 

为了`s = "1"`，邻域变为`(0,1,0)`。 该规则将这个三元组映射到`1`，并且代码通过直接查找反映了这一点，而不需要对大小为一的输入进行任何特殊分支。 

一串全 1 表明相邻单元格之间的相互作用。 为了`s = "111"`，每个位置看到不同的三元组：中心看到`111`这变成`0`，而边缘看到`011`和`110`，两者都映射到`1`。 输出变为`"101"`，并且该算法准确地再现了这一点，因为每个索引都是独立于原始字符串而不是部分更新的版本进行评估的。
