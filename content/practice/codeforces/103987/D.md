 ## 解决方案
 ## 问题理解

 每个任务给出一个起始索引$i$，并且所需的计算始终相同：将三个连续整数相加，以$i$， 具体来说$(i-1) + i + (i+1)$。 从代数上来说，这可以简化为$3i$，所以每个任务实际上都要求我们计算三的倍数。 

然而，限制不是关于算术正确性，而是关于小数加法行为。 中诺拒绝执行任何需要计算的任务$3i$以 10 为基数相加时，任何数字位置都需要进位。在逐列加法期间，当某些数字和达到 10 或更多时，就会发生进位，从而强制进位到下一个更高的数字。 

所以问题就变成了纯粹的数字问题：有多少个整数$i$在范围内$1$到$n$乘以 3 是否会产生一个在标准乘以 3 期间无需任何数字进位即可形成的数字。 

约束条件$n \le 10^{18}$排除任何直接迭代。 即使单独检查每个数字也是不可能的。 该解决方案必须仅依赖于数字结构和十进制表示的计数参数。 

一个微妙的问题是“无进位”取决于数字之间的相互作用$i$并乘以 3。即使$3i$全局很小，由于 4 或更多的数字，进位仍然可能在本地发生$i$， 自从$3 \cdot 4 = 12$。 

关键的隐藏边缘情况是进位传播。 例如，如果一个数字产生进位进入下一个位置，则该下一个位置也可能溢出，即使其直接乘积很小。 不跟踪进位传播的简单的逐位检查将会失败。 

## 方法

 蛮力解释很简单：迭代所有$i \in [1, n]$, 计算$3i$，模拟数字乘法或加法，并检查任何数字和是否产生进位。 这是正确的，但立即不可行，因为$n$可以达到$10^{18}$，使得迭代变得不可能。 

关键的观察结果是，无进位乘以 3 相当于数字 DP 的十进制表示形式$i$。 我们处理的数字$i$从最高有效位到最低有效位，保持前一个数字乘法中的进位当前是否处于活动状态。 每个数字转换仅取决于当前数字和传入进位，因此问题变成了标准数字动态编程计数$[1, n]$。 

暴力破解在概念上是有效的，因为它直接模拟操作的定义。 它失败了，因为状态空间是线性的$n$。 数字 DP 将状态空间减少到最多 20 个数字乘以 2 个进位状态，使其可行。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n \cdot \log n)$|$O(1)$| 太慢了|
 | 数字DP |$O(\log n)$|$O(\log n)$| 已接受 |

 ## 算法演练

 我们数一下里面有多少个数字$[0, n]$乘以 3 时不产生进位。 

1. 转换$n$转换为十进制数字数组，以便我们可以从最高有效数字到最低有效数字对其进行处理。 这使我们能够逐位强制执行上限约束。 
2.定义动态规划函数$dp(pos, tight, carry)$， 在哪里$pos$是当前的数字索引，$tight$指示前缀是否仍然等于$n$， 和$carry$指示先前的数字乘法是否在该位置产生了进位。 
3. 在每个位置，尝试所有可能的数字$d$从 0 到 9，但限制为$d \le n[pos]$如果我们处于紧张状态。 这确保我们永远不会超过$n$。 
4. 对于每个选定的数字$d$，在该数字处模拟乘以 3：计算$val = 3d + carry$。 如果$val \ge 10$，这个转换是无效的，因为它产生了一个进位，这违反了约束。 否则，下一个进位变为 0，因为我们不需要进位传播。 
5. 递归到具有更新状态的下一个数字。 对所有有效转换求和。 
6. 如果我们包含数字 0 但问题只从 1 开始计数，则最后减 1。 

关键是我们绝不允许进位存在。 因此 DP 实际上只接受每个数字都在集合中的数字序列$\{0,1,2,3\}$， 因为$3 \cdot d \le 9$必须持有。 

### 为什么它有效

 该算法在每个数字上局部强制执行乘法约束，同时通过状态传播保持全局正确性。 任何无效数字都必须在第一个数字处失败，其中$3d + carry \ge 10$，并且由于 DP 在严格约束下探索所有前缀，因此每个有效数字都只计算一次。 没有进位可确保数字之间的独立性，因此 DP 状态完全捕获所有必要的历史记录。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = input().strip()
    digits = list(map(int, n))

    from functools import lru_cache

    @lru_cache(None)
    def dp(pos, tight, carry):
        if pos == len(digits):
            return 1 if carry == 0 else 0

        limit = digits[pos] if tight else 9
        res = 0

        for d in range(limit + 1):
            if 3 * d + carry >= 10:
                continue
            nd = 0
            ntight = tight and (d == limit)
            res += dp(pos + 1, ntight, nd)

        return res

    ans = dp(0, 1, 0)
    print(ans - 1)

if __name__ == "__main__":
    solve()
```该代码在十进制表示上执行标准数字 DP$n$。 状态对位置、前缀限制和携带状态进行编码。 该转换明确拒绝任何乘以 3 时会产生进位的数字。最后的减法从计数中删除空数。 

一个微妙的点是我们只允许在下一个状态中进位为零。 这是正确的，因为任何产生进位的转换都是无效的并会立即被修剪。 

## 工作示例

 ### 示例 1

 让$n = 12$，数字是$[1, 2]$。 

我们在一个简化的表中跟踪状态，其中进位始终为 0，因为无效的转换被删除。 

| 邮政 | 紧| 选择的数字 | 有效的？ | 下一个状态 |
 | ---| ---| ---| ---| ---|
 | 0 | 1 | 1 | 是 (3) | 位置 1 |
 | 0 | 1 | 2 | 是 (6) | 位置 1 |
 | 1 | 1 | 0-4 | 是的 | 结束 |

 这表明任何位置都不允许使用大于 3 的数字，因为它们会创建$3d \ge 10$。 因此 12 以内的有效数字是仅由数字 0-3 组成的数字。 

这证实了约束纯粹是数字限制的。 

### 示例 2

 让$n = 30$, 数字$[3, 0]$。 

第一个数字，我们可以选择0,1,2,3。 

如果我们选择 3，我们将继续进行紧密传播。 在第二位数字中，只有数字 0-3 仍然有效。 

DP 统计 1 到 30 之间所有位数≤ 3 的数字，由于严格限制，不包括超过 30 的数字。 

这表明紧密标志正确地强制执行上限，而数字限制强制执行进位规则。

 ## 边缘情况

 一个关键的边缘情况是当$n$包含大于3的数字。例如$n = 49$。 该算法正确地拒绝任何包含数字 4-9 的数字，因为$3 \cdot d \ge 12$立即导致该数字进位。 

另一个边缘情况是当$n$其本身在结构上是有效的，但受到严格约束的排除。 例如，$n = 30$。 数字 30 在数字方面是有效的（3 和 0），但许多中间数字（如 31-39）由于数字限制而自动无效，并且 DP 确保它们不被计算在内。 

最后的边缘情况是$n = 0$，DP 将计算空数。 最后的减法确保我们排除这种人为的情况，并且只计算有效的正整数。
