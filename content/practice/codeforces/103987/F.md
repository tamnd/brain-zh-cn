 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例描述一组石堆，两个玩家从爱丽丝开始交替玩游戏。 

轮到爱丽丝时，她选择一堆石子并从中取出任意正数的石子。 她在这一步中移除的棋子数量成为爱丽丝在整个游戏中移除的棋子的总和。 

在鲍勃的回合中，他也选择了一堆，但他的移动受到限制：他移除的石子数量必须至少是爱丽丝在游戏中迄今为止移除的石子总数，而不仅仅是最后一步。 这使得鲍勃的合法动作随着游戏的进行而越来越受到限制。 

当玩家无法在自己的回合中做出有效的移动时，游戏结束，并且该玩家失败。 由于爱丽丝先移动，她试图强制鲍勃最终没有有效移动的位置。 

输入大小建议所有测试用例最多有 2·10^5 堆，因此每个测试用例的任何解决方案都必须大致呈线性或线性算数。 任何模拟移动或每回合重复更新堆的东西都是立即不可行的，因为在最坏情况推理中，游戏可能持续最多 O（总石子）移动，这远远超出了限制。 

鲍勃的约束如何演变时立即出现了一个微妙的边缘情况。 如果 Alice 进行了大量的初始删除，Bob 可能根本无法做出响应。 例如，如果最大的一堆严格大于所有其他堆的总和，从而允许爱丽丝一步完全拿走它，那么鲍勃可能没有足够大的一堆来满足他的最低要求。 

第二个不明显的边缘情况是存在多个大桩时。 然后鲍勃总是可以对爱丽丝最初的攻击性举动作出反应，并且游戏不再立即结束，从而导致双方玩家重复消耗大量堆的长时间交互。 

困难在于，整个游戏结构由爱丽丝的第一步主导，因为它决定了鲍勃在游戏其余部分的最小阈值。 

## 方法

 暴力方法将直接模拟游戏。 我们将迭代回合，跟踪爱丽丝移除的石子总数，并且对于鲍勃的每个移动，扫描所有堆以找到大小至少为该阈值的有效移动。 这在概念上是正确的，因为它完全遵循规则。 然而，每次移动可能需要扫描最多 O(n) 堆，并且移动次数可以与移除的石子总数成正比。 这导致最坏情况的复杂性远远超出可接受的限度。 

关键的观察结果是，鲍勃的约束仅取决于一个值：到目前为止，爱丽丝已经移除的石头总数。 这个值是单调的，完全由爱丽丝的第一步控制，因为爱丽丝的任何进一步的移动只会增加它，只会让鲍勃变得更弱。 

这意味着第一步决定了游戏的整个结构。 爱丽丝总是会选择她的第一步来最大程度地给鲍勃施加压力，而最佳选择简化为选择一堆并决定从中拿走多少。 任何部分删除都通过简单地获取整个选定的堆来控制，因为增加爱丽丝的总数只会进一步限制鲍勃。 

一旦爱丽丝选择了一堆并将其完全移除，鲍勃就会留下一个等于该堆大小的固定阈值。 从那一刻起，鲍勃只能玩至少那么大的牌堆，并且他的每一步棋都会使这样的牌堆减少至少该阈值。 

这将游戏分解为最大堆和剩余结构之间的比较。 如果最大的一堆是唯一占优的，爱丽丝可以立即迫使鲍勃进入一个没有合法动作的位置。 否则，鲍勃总是可以至少回答最初的动作，并且游戏以受控方式继续，鲍勃生存足够长的时间以最终耗尽爱丽丝的优势。

 ## 算法演练

 该解决方案减少为仅检查最大和第二大的堆。 

1. 找出数组中最大的桩值。 这代表了爱丽丝的最佳第一步，因为任何较小的选择只会削弱她的地位。 
2. 计算有多少堆达到该最大值。 这很重要，因为鲍勃的响应能力取决于另一堆是否可以匹配爱丽丝的阈值。 
3. 如果最大值恰好出现一次，则爱丽丝可以在第一步中拿走整个最大的一堆。 此举之后，没有其他堆可以满足鲍勃的要求，所以鲍勃立即失败。 
4. 如果最大值出现多次，Bob 总是可以通过选择另一个最大桩来响应 Alice 的初始动作。 这使得游戏在开场后保持活力，并防止爱丽丝立即获胜。 
5. 在这种情况下，交互在剩余的大堆上以对称的方式继续，鲍勃总是可以在足够长的时间内反映爱丽丝的压力，以避免立即被困住，从而导致鲍勃在最佳游戏下获胜。 

### 为什么它有效

 关键的不变量是鲍勃的约束完全由爱丽丝的第一步决定，而不取决于鲍勃的动作。 这使得游戏实际上是从爱丽丝的初始选择中选择阈值的游戏之一。 一旦阈值确定，Bob 的移动能力仅取决于每轮缩减后是否存在至少一个满足或超过该阈值的堆。 

如果最大值是唯一的，爱丽丝可以选择一个在她的移动后无法达到的阈值，立即瓦解鲍勃的合法移动集。 如果它不是唯一的，那么在鲍勃的回合开始时，阈值总是由至少另一堆支撑，防止立即崩溃并允许鲍勃通过最佳游戏保持生存能力。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        mx = max(a)
        cnt = 0
        for x in a:
            if x == mx:
                cnt += 1
        
        if cnt == 1:
            print("Alice")
        else:
            print("Bob")

if __name__ == "__main__":
    solve()
```该实现直接计算最大值并计算其出现次数。 推理完全基于第一步的结构，因此不需要模拟。 唯一的微妙之处是确保每个测试用例在 O(n) 时间内正确计算最大计数。

 ## 边缘情况

 一个关键的边缘情况是所有桩都相同。 在这种情况下，没有任何一堆牌可以让爱丽丝占据主导地位。 在爱丽丝拿走一整堆后，鲍勃总是有另一堆相同大小的东西来响应，以防止立即崩溃。 该算法正确地将其分类为鲍勃的胜利，因为最大值不是唯一的。 

另一种边缘情况是只有一堆的情况。 爱丽丝将其完全删除，而鲍勃没有合法的举动。 该算法正确输出 Alice，因为最大值恰好出现一次。 

最后一个微妙的情况是数组严重倾斜但顶部不唯一。 即使其中一堆非常大，而其他堆则小得多，只要该最大值是唯一的，爱丽丝就会立即获胜。 正确性仅取决于最大值的唯一性，而不取决于它与其他桩之和的关系。
