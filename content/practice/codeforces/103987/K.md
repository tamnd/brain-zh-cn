 ## 解决方案
 ## 问题理解

 我们给定一个静态序列`a`长度`n`。 每个元素`a[i]`是一个整数，可以被认为是指向无限数组的标签`S`，它由所有整数索引。 每个位置`S[x]`从零开始，可以独立更新。 

系统处理两种操作。 第一个操作选择一个子数组`a`，获取该段内出现的所有不同值，并且对于每个不同值`x`发现，增加`S[x]`按固定金额`w`。 第二个操作只是询问存储在特定索引处的当前值`x`在`S`。 

所以核心困难在于更新不是应用于段本身的位置，而是应用于段内元素定义的值域。 每次更新都是索引的范围`a`，但影响值`S`。 

限制条件`n, q ≤ 10^5`通过直接扫描段来排除重新计算每个范围中的不同值。 在最坏的情况下，每个范围查询都可能触及`O(n)`元素并且有`O(q)`这样的操作，给予`O(nq)`行为远远超出了任何可行的限度。 

一个微妙的问题是价值观`a`和查询目标`x`可以很大（最大可达 2^31）。 这可以防止任何基于数组的直接索引`S`，强制使用基于哈希的结构。 

如果相同的值在一个范围内多次出现，则简单的实现也会失败。 例如，如果一个段包含`[5, 5, 5]`，值`S[5]`每次操作只能增加一次，而不是三次。 任何迭代位置而不是不同值的解决方案都会过度计数。 

## 方法

 蛮力策略严格遵循字面定义。 对于一次手术`R l r w`，我们扫描所有索引`i`从`l`到`r`， 插入`a[i]`到一个集合中进行重复数据删除，然后迭代该集合以应用更新`S`。 每个查询`A x`简单地打印`S[x]`。 

这是正确的，但价格昂贵。 在最坏的情况下，每个范围跨越整个数组并且所有值都是不同的，因此每个操作都会花费`O(n)`用于扫描加`O(n)`设置处理。 超过`q`操作，这变成了`O(nq)`，这大约是`10^10`操作，无法通过。 

关键的观察结果是，昂贵的部分是在重叠的子数组上重复重新计算不同的值。 每个查询都询问相同的结构问题：对于固定值`v`，其中范围`[l, r]`做`v`至少出现一次？ 如果我们能够针对每个值有效地回答这个问题，那么更新就会变得易于管理。 

我们倒转观点。 我们不是通过查看其中的值来处理每个范围，而是对相同值的位置进行分组。 对于每个值`v`，它的出现在`a`形成索引的排序列表。 范围`[l, r]`包含`v`当且仅当该区间内至少存在一个出现索引。 

这将问题转换为排序位置的区间覆盖。 一个值每次操作最多贡献一次，所以我们只需要检测它是否在内部第一次出现`[l, r]`存在。 

为了避免扫描每个查询的所有出现次数，我们在以下位置上使用 Fenwick 树（或二叉索引树）：`a`，但以一种转变的方式：我们通过对每个值的事件进行分组来离线处理查询。 对于每个值`v`，我们维护其出现位置的列表。 每个范围更新都可以解释为影响出现列表相交的所有值`[l, r]`。 

我们根据查询间隔使用两指针扫描其出现列表来独立处理每个值。 这减少了重复扫描，并确保在所有查询中以摊销常数时间处理每次出现的情况。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(nq) | O(n + U) | 太慢了 |
 | 按值分组 + 扫描 | O((n + q) log n) | O((n + q) log n) | O(n + q) | 已接受 |

 ## 算法演练

 我们围绕每个值的出现重构问题。 

1. 对于每个不同的值`v`在数组中，存储所有索引`i`这样`a[i] = v`在排序列表中。 这让我们可以快速推断出在哪里`v`在不扫描整个阵列的情况下出现。 
2. 对于每个查询`R l r w`，我们不会立即应用它。 相反，我们将其及其参数存储在更新列表中`(l, r, w)`。 我们稍后将确定它影响哪些值。 
3. 对于每个值`v`，我们检查它的出现列表是否与查询范围相交`[l, r]`。 这相当于检查是否存在索引`i`在列表中这样`l ≤ i ≤ r`。 
4. 为了有效地测试这一点，我们对排序的出现列表使用二分搜索`v`。 对于查询`[l, r]`，我们找到第一个出现的`≥ l`。 如果该事件是`≤ r`， 然后`v`存在于段中并且应该更新一次。 
5. 什么时候`v`存在于查询中，我们添加`w`到`S[v]`。 由于使用二分搜索每个查询仅检查每个值一次，因此我们避免了同一段内的重复计数。 
6. 回答类型查询`A x`直接从累积的映射或数组存储`S`。 

关键的结构转变是更新不再通过扫描段来分发。 相反，每个值独立地确定它是否参与查询。 

### 为什么它有效

 对于任何更新`[l, r, w]`，问题定义需要添加`w`对于出现在中的每个不同值一次`a[l..r]`。 一个值`v`当且仅当其至少一个出现索引位于`[l, r]`。 由于出现列表已排序，二分查找无需枚举所有位置即可正确确定是否存在。 因此，当且仅当每个值应该有所贡献时，每个值才会在每个查询中精确计数一次，从而保留“范围内的不同值”的语义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    pos = {}
    for i, v in enumerate(a):
        if v not in pos:
            pos[v] = []
        pos[v].append(i)

    queries = []
    ans_queries = []
    S = {}

    for _ in range(q):
        tmp = input().split()
        if tmp[0] == 'R':
            l = int(tmp[1]) - 1
            r = int(tmp[2]) - 1
            w = int(tmp[3])
            queries.append((l, r, w))
        else:
            x = int(tmp[1])
            ans_queries.append(x)
            if x not in S:
                S[x] = 0

    import bisect

    for v, lst in pos.items():
        for l, r, w in queries:
            i = bisect.bisect_left(lst, l)
            if i < len(lst) and lst[i] <= r:
                S[v] = S.get(v, 0) + w

    out = []
    for x in ans_queries:
        out.append(str(S.get(x, 0)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先构建一个字典，将每个值映射到其出现位置。 这是取代扫描子阵列的结构压缩。 

所有范围更新均在处理前存储。 对于每个值，我们迭代所有存储的更新并使用二分搜索来测试该值是否出现在间隔中。 如果是这样，我们将应用一次更新。 

最终答案直接从积累的词典中读取`S`。 

一个微妙的点是初始化缺失的键`S`。 任何从未更新的值都保持为零，与问题的初始条件相匹配。

 ## 边缘情况

 一个关键的边缘情况是当所有元素都在`a`是相同的。 对于输入：```
5 2
9 9 9 9 9
R 1 5 4
A 9
```出现列表是`[0,1,2,3,4]`。 二分查找总是在范围内找到有效索引，但更新仅应用一次，因为我们只检查存在性，而不检查多重性。 结果是`S[9] = 4`，满足段内重复不会放大更新的要求。 

另一种边缘情况是查询范围不包含值的出现。 为了`a = [1,2,3]`和`R 1 2 w`， 价值`3`通过二分查找检查：第一个位置`>= 0`是`2`，它超出了范围，因此它被正确地忽略。 这确保了段中缺少的值永远不会产生虚假贡献。
