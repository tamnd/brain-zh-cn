 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，代表一行花栗鼠的饥饿程度。 索引为的特定花栗鼠`k`必须始终包含在内。 我们可以选择任何连续的段`[l, r]`这样它包含索引`k`。 选择这样一个段的成本是其中所有值的总和。 

每个有效的段都会产生一个成本，并且在所有这些段中，我们概念上按升序对这些总和进行排序。 任务是输出`m`- 其中最小的值。 

限制条件很大：`n`最多可达一百万，这会立即排除任何显式枚举所有段的解决方案。 即使只有包含固定索引的段`k`是允许的，他们的数量仍然是`k * (n - k + 1)`在最坏的情况下，它是二次的。 这迫使我们进入一种可以隐式生成或计算段总和而不是枚举它们的结构。 

一个微妙的困难来自负值。 自从`a[i]`可能为负，扩展一个段不会单调增加其总和。 这打破了基于滑动窗口或贪婪扩张的任何直觉。 

一个天真的错误是假设“最小的部分来自周围的最短范围”`k`”。例如，如果值交替频繁，则稍长的段的总和可能比较短的段小。因此必须在全局而不是本地处理排序。

 另一种失败情况是尝试预先计算所有前缀和，然后枚举所有对`(l, r)`和`l ≤ k ≤ r`。 这会产生高达一万亿次操作`n = 10^6`，这在时间和记忆上都是不可行的。 

关键的困难不是计算一段和，而是找到`m`- 以固定枢轴为中心的结构化段总和系列中的最小的。 

## 方法

 蛮力方法很简单。 修复每个`l ≤ k`和每一个`r ≥ k`，计算总和`a[l..r]`，存储所有结果，对它们进行排序，然后取`m`-th 元素。 正确性是直接的，因为它直接构造有效段的完整多重集。 

瓶颈在于枚举本身。 有`k`左端点的选择和`n - k + 1`选择正确的端点，产生`O(n^2)`最坏情况下的部分。 即使每个和都是使用前缀和在 O(1) 中计算的，仅仅存储和排序它们也是不可能的`n = 10^6`。 

关键的观察结果是所有段和共享结构。 如果我们修复`k`, 每段`[l, r]`可以写成左扩展的组合`k`和一个右扩展`k`。 更准确地说，我们可以重写：`sum(l, r) = (sum from l to k) + (sum from k to r) - a[k]`这将每个段分成左贡献和右贡献并进行恒定校正。 所以不要从二维的角度思考`(l, r)`，我们将问题简化为组合两个一维类前缀和族。 

现在定义：`L[i] = sum of a[i..k]`为了`i ≤ k`， 和`R[j] = sum of a[k..j]`为了`j ≥ k`。 

每个有效段总和变为`L[i] + R[j] - a[k]`。 

这将问题转化为选择`m`- 两个排序数组的最小对和`L`和`R`。 两个数组都可以使用线性时间内的前缀和来计算，并且两者在其自然方向上都是单调的：向左延伸减少`L`, 向右延伸增加`R`。 

现在任务变成：对所有对和进行计数和排序`L[i] + R[j]`，然后减去一个常数。 

这是一个经典的“排序行中第 k 个最小对和”问题，可以使用对答案的二分搜索结合双指针计数过程来解决。 

我们对候选值进行二分查找`X`并计算有多少对满足`L[i] + R[j] ≤ X + a[k]`。 由于两个数组都是单调的，因此我们可以使用双指针扫描来计算线性时间。 

这将整个问题从二次枚举简化为`O(n log V)`在哪里`V`是取值范围。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n² log n) | O(n² log n) | O(n²) | 太慢了|
 | 最佳 | O(n log V) | O(n) | 已接受 |

 ## 算法演练

 我们首先构建前缀和，以便可以在恒定时间内计算任何子数组和。 这使我们能够构建以`k`无需反复重新计算总和。 

1. 计算数组的前缀和。 这允许`sum(l, r)`O(1) 中的查询，这是必要的，因为我们将隐式导出许多段和。 
2.构造左贡献数组`L`。 对于每个索引`i ≤ k`，计算段的总和`[i, k]`。 我们将这些值存储在一个数组中。 这样做的原因是每个有效段都有一个左边界，并且它对总和的贡献完全由我们向左延伸的距离来捕获`k`。 
3.构建正确的贡献数组`R`。 对于每个索引`j ≥ k`，计算总和`[k, j]`。 这对称地捕获了所有正确的扩展。 
4. 对公式进行归一化，使每个有效段总和变为`L[i] + R[j] - a[k]`。 减法避免了重复计算中心元素`a[k]`。 
5. 排序`L`和`R`。 排序是必要的，以便我们可以使用单调两指针扫描有效地计算对和。 单调结构取代了显式枚举的需要。 
6. 对答案值进行二分查找`X`。 对于每位候选人`X`，我们计算有多少对满足`L[i] + R[j] - a[k] ≤ X`。 这相当于`L[i] + R[j] ≤ X + a[k]`。 
7. 为了有效地计算对数，固定一个指针`j`在最大有效索引处`R`并仅向下移动它。 对于每个`L[i]`，我们减少`j`直至满足总和条件。 有效对的数量`i`那么就是`j + 1`。 这是有效的，因为增加`i`增加`L[i]`， 所以`j`整体上只能朝一个方向移动。 
8. 使用二分查找中的计数功能来找到最小的`X`这样至少`m`对满足条件。 

### 为什么它有效

 每个有效段唯一对应于一对`(i, j)`代表我们向左和向右延伸多远`k`。 该转换将排序保持在恒定的偏移范围内，因此对段和进行排名相当于对对和进行排名。 

双指针计数是正确的，因为`L`和`R`是单调数组。 一次一对`(i, j)`有效，任何较小的`j' < j`也将同样有效`i`，以及任何更大的`i' > i`只会让情况变得更加困难。 这保证了单调边界，这正是在二分搜索中实现线性时间计数的原因。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_pairs(L, R, limit):
    n = len(L)
    m = len(R)
    j = m - 1
    total = 0

    for i in range(n):
        while j >= 0 and L[i] + R[j] > limit:
            j -= 1
        total += (j + 1)
    return total

def solve():
    n, k, m = map(int, input().split())
    a = list(map(int, input().split()))

    k -= 1

    prefix = [0] * (n + 1)
    for i in range(n):
        prefix[i + 1] = prefix[i] + a[i]

    L = []
    for i in range(k, -1, -1):
        L.append(prefix[k + 1] - prefix[i])

    R = []
    for j in range(k, n):
        R.append(prefix[j + 1] - prefix[k])

    L.sort()
    R.sort()

    base = a[k]

    def ok(x):
        return count_pairs(L, R, x + base) >= m

    lo = -10**18
    hi = 10**18

    while lo < hi:
        mid = (lo + hi) // 2
        if ok(mid):
            hi = mid
        else:
            lo = mid + 1

    print(lo)

if __name__ == "__main__":
    solve()
```该实现首先构建前缀和，以便可以在 O(1) 内计算左段和右段的和。 数组`L`和`R`存储周围扩展的所有贡献`k`分别向左和向右。 对它们进行排序可以实现高效计数所需的单调结构。 

功能`count_pairs`是重点优化。 它维护一个指针`j`只向左移动`i`增加。 这避免了重新计算并确保每个二分搜索步骤的计数以线性时间运行。 

二分查找对最终的段总和值进行操作。 由于我们将段和转换为对和加上一个常数，因此我们与`x + base`谓词内部。

 ## 边缘情况

 一种重要的边缘情况是所有值都相等。 在这种情况下，每个段都具有可预测的按长度线性排序，但算法仍必须正确处理重复项。 对和结构产生许多相同的值，并且二分搜索不能依赖于严格的不等式。 

另一种情况是所有值均为负数。 然后，延长一段几乎总是会减少总和，但并不均匀。 两指针计数仍然有效，因为它仅取决于顺序，而不取决于符号。 

最后一个边缘情况是`k`位于一个边界处。 如果`k = 1`， 然后`L`只有一个元素，问题就简化为右侧的前缀段。 如果`k = n`，它对称地减少。 的建设`L`和`R`自然正确退化，不需要特殊处理。
