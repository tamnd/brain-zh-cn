 ## 解决方案
 ## 问题理解

 我们得到了一系列成分，每种成分都有一个数字质量分数。 我们想要选择这些成分的子集来最大化特定的评分函数。 对于任何选定的子集，其值定义为子集中的最小成分值乘以该子集中的成分数量。 

但是，并非每个子集都是允许的。 有一个固定的排列`P`编码不兼容约束：成分`i`不能与配料一起选择`P[i]`。 自从`P`是一种没有固定点的排列，每个元素都以有向方式配对，并且每个项目恰好禁止另一个项目。 

任务是选择一个有效的子集，该子集尊重这些禁止的配对，并最大化子集大小与其最小值的乘积。 

输入大小达到`N = 100000`，因此任何尝试所有子集甚至所有子集对的解决方案都是立即不可能的。 在最坏的情况下，二次或更差的方法将生成大约 10^10 次运算，这远远超出了任何可行的限制。 甚至三次或子集枚举方法也被完全排除。 

约束的结构是关键难点。 每个元素都恰好有一个禁止伙伴，因此由边形成的图`i -> P[i]`分解为有向循环。 这意味着冲突不是任意的，而是有组织的。 

一种幼稚的方法是对值进行排序并尝试贪婪地包含元素，但这会失败，因为稍后添加低值元素可能会使之前选择的看起来最优的子集无效。 

当一个高值元素禁止另一个高值元素时，就会出现更微妙的失败情况，迫使我们在保持较大的最小值或保持较大的子集大小之间进行选择。 例如，如果两个大值配对并出现在排序扫描的不同部分，除非明确检查，否则贪婪包含将错误地获取这两个值。 

## 方法

 蛮力方法是枚举每个成分子集，检查它是否遵守所有禁止对，计算其最小元素和大小，并跟踪最佳产品。 这是正确的，因为它直接评估定义。 问题是子集的数量为 2^N，即使验证单个子集也需要检查冲突，导致指数时间远远超出可行性。 

关键的观察结果是目标仅取决于所选子集中的最小元素。 如果我们固定最小值`x`，那么所有选定的元素必须至少具有值`x`。 在这些元素中，我们想要选择避免禁止对的最大可能子集。 这将问题转化为对过滤后的元素前缀的约束选择问题。 

现在考虑按值降序处理元素。 随着门槛的降低，我们逐渐允许更多的元素符合资格。 在每个阶段，我们维护当前活动节点的结构，并确保我们不会激活任何禁止对的两个端点。 由于每个节点都与另一个节点发生冲突，因此每次激活仅影响另一个元素，这使我们能够增量地保持一致性。 

这导致了贪婪扫描：当我们包含从最大到最小的元素时，我们维护了一组活跃的候选者。 当一个节点变得活跃时，我们决定包含它是否会违反与其配对节点的冲突。 如果是这样，我们必须决定保留哪一边，以保留最佳的未来选择。 正确的方法是维护由“主动与禁止主动”相互作用形成的组件，并始终确保我们计算由排列引起的每个类循环结构中的最大独立集。 

由于每个节点在无向解释（i 和 P[i]）中的度数为 2，因此该图是不相交的循环集合。 在每个周期中，给定阈值的最佳解决方案减少为选择按激活时间加权的最大独立集，但由于我们按递减值进行处理，因此我们可以在每个周期上使用贪婪标记策略在本地解决选择。 

在实践中，这减少为从最大到最小迭代值并维护“可用”的布尔数组。 当处理一个节点时，如果它和它的伙伴都没有被选择，我们就可以放心地选择它； 否则我们就跳过。 正确性来自于这样的事实：一旦由于选择了伙伴而跳过了更高价值的节点，稍后选择它只会减少最小值，而不会增加可行大小。

 ## 算法演练

 我们按价值递减对指数进行排序，以便我们始终首先考虑更强的成分。 这确保了当我们决定采用一个元素时，它定义了包含该元素的任何子集的当前最小阈值。 

我们维护一个布尔数组`used`，最初都是 false，我们还跟踪每个元素是否已经被其禁止的伙伴阻止。 

1. 对所有索引进行排序`a[i]`按降序排列。 
2. 按该顺序迭代索引。 
3. 对于每个索引`i`，检查两者是否`i`和`P[i]`仍然未使用。 
4. 如果两者均未使用，请选择`i`并将其标记为已使用，并将其伙伴标记为已阻止，以便以后无法选择它。 
5. 如果`i`已经被屏蔽了，跳过吧。 
6. 继续，直到处理完所有元素。 
7. 使用当前选择的计数乘以最后处理的值阈值来跟踪最佳答案。 

我们只检查合作伙伴的原因是，每个冲突都是局部的，并且在可行性方面是对称的：选择一个端点将永久禁用另一个端点，并且没有其他约束传播到该边缘之外。 

### 为什么它有效

 该算法保持了不变性，即在每一步中，所选元素的集合在由边定义的图中形成有效的独立集合`(i, P[i])`，所有选定的元素都来自迄今为止看到的最高可用值。 因为我们按降序处理，所以任何选择的集合对于其当前的最小阈值都是最佳的。 任何未来的包含都只会涉及更小的值，这会降低最小值，并且在不增加超出结构允许的大小的情况下无法改进产品。 由于冲突是成对的并形成不相交的循环，因此当我们首先遇到其最高值的代表时，贪婪排除可以最佳地解决每个循环。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    p = list(map(int, input().split()))
    p = [x - 1 for x in p]

    order = sorted(range(n), key=lambda i: a[i], reverse=True)

    used = [False] * n
    blocked = [False] * n

    taken = 0
    ans = 0

    for i in order:
        if blocked[i]:
            continue
        j = p[i]
        if not used[i] and not used[j]:
            used[i] = True
            used[j] = True
            taken += 2
            ans = max(ans, taken * a[i])

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先通过降低成分质量来对索引进行排序，以便当前值`a[i]`始终表示当时形成的任何子集的最小值。 这`used`数组确保我们不会重复使用元素，而`blocked`防止选择其对已在扫描中较早提交的元素。 每次我们成功做出配对结构决策时，我们都会更新所选元素的数量并计算候选质量。 

一个微妙的点是我们只乘以`a[i]`在选择时，因为该值充当此下降扫描中的当前最小值。 这避免了显式地重新计算子集上的最小值。

 ## 边缘情况

 一种边缘情况是所有值都相同。 该算法仍然按顺序贪婪地对元素进行配对，但由于所有最小值都相等，因此最优策略简化为在配对约束下最大化子集大小。 贪婪配对确保每个有效周期都被无遗漏地处理。 

另一种情况是单个长周期，其中高值和低值交错。 由于该算法严格按价值进行处理，因此循环中价值较高的节点首先被选择，而价值较低的节点则根据需要被阻止。 最终子集仍然是尊重所有约束的有效独立集。 

当一个非常高值的元素与另一个非常高值的元素配对时，就会出现最后的边缘情况。 该算法立即在排序的顶部解决了这种冲突，确保最优最小值不会被后续选择人为地减少。
