 ## 解决方案
 ## 问题理解

 我们给出一个从未知整数位置开始的运动过程$X$。 从这个起点开始，执行一系列移动。 第一步向前移动一个单位，第二步向后移动三个单位，第三步向前移动四个单位，第四步向后移动八个单位，以此类推，每一步交替方向，每次步长加倍。 这个过程结束后，我们被告知两个值：总位移$D$，表示所有有符号移动的净总和，以及最终位置$Y$，这是进程停止的点。 

任务是判断是否存在整数起始位置$X$这样，遵循这种确定性的运动模式就可以准确地产生给定的最终位置和总位移。 如果这样一个$X$存在，我们必须输出它。 否则，我们会报告数据不一致。 

约束条件非常大，$D$最多$3 \cdot 10^{18}$和坐标高达$10^{18}$。 这立即排除了对过程的任何一步一步的模拟，因为达到这样的幅度所需的步骤数仅随着距离以对数方式增长，但如果在没有结构的情况下简单地接近，仍然远远超过安全迭代限制。 任何解决方案都必须依赖于对运动结构的封闭式观察。 

一个微妙的边缘情况是由于我们没有给出步骤数这一事实而产生的。 该交替加倍序列的不同前缀产生不同的部分和，并且必须考虑多个前缀长度。 使用完整无限模式的天真假设是不正确的，因为当到达陆地时，该过程恰好停止，这对应于未知的前缀长度。 

另一个边缘情况是重建中的模糊性$X$。 即使最后的位置$Y$是固定的，多个假设的步数可以产生相同的位移$D$，并且只有一种前缀长度和起始位置一致的组合有效。 

## 方法

 如果我们尝试直接模拟该过程，我们会从猜测开始$X$，应用大小移动$1, 3, 4, 8, 16, \dots$具有交替符号，并计算运行位移和最终位置。 对于每位候选人$X$，我们会尝试匹配观察到的$(D, Y)$。 然而，自从$X$它本身是未知的，并且在给定的约束内是无界的，这种强力方法甚至没有被明确定义为有限的搜索空间。 即使固定一个合理的范围，步长的指数增长也意味着达到有意义的距离需要大约$O(\log D)$每个模拟的步骤，并扫描可能的起点将使这完全不可行。 

关键的见解是运动模式与起点无关。 后总位移$k$步骤仅取决于几何序列的交替和：$$+1 - 3 + 4 - 8 + 16 - \dots$$如果我们将移动的前缀和定义为$S_k$，那么最终的位置就是$X + S_k$。 同时，位移$D$正是$S_k$。 这立即意味着如果序列长度$k$知道了，问题就变得微不足道了：$$X = Y - D$$整个困难在于确定一个有效的$k$存在使得交替几何前缀和等于$D$，并且该过程可以合法地停止在该步骤。 

序列的结构允许我们显式地计算前缀和。 第一步之后的每一步都遵循一个清晰的模式：幅度是从以下位置开始的 2 的幂$1, 3, 4, 8, 16, \dots$和符号交替出现。 这使得$S_k$可以用封闭形式表达，我们只需要检查是否有任何前缀匹配$D$。 由于步长呈指数增长，$k$对于以下所有有效值最多约为 60$10^{18}$。 

因此，问题简化为迭代所有可行的前缀长度，计算相应的位移，并检查与$D$。 一旦找到有效的前缀，我们就重建$X$直接地。

 ## 算法演练

 1. 观察整个过程后的净位移等于所有移动步骤的总和，与起始位置无关。 这使我们能够分离角色$X$从运动序列本身。 
2. 预先计算或迭代构建运动序列值。 第一步是$1$，并且每一个下一个幅度都会加倍，而符号从正开始交替。 这会生成确定性前缀和序列。 
3. 对于每个可能的前缀长度$k$，计算前缀和$S_k$。 这表示如果过程恰好在$k$移动。 
4. 比较每个计算结果$S_k$与给定的$D$。 如果没有前缀和匹配$D$，数据无法一致，因为没有有效的停止点产生所需的位移。 
5. 找到匹配的前缀长度后，使用以下关系计算起始位置$X = Y - D$，因为最终位置等于初始位置加上位移。 
6. 输出数据与计算结果一致$X$。 

### 为什么它有效

 运动序列是固定的并且与起始位置无关，因此每种可能的场景都唯一对应于选择前缀长度$k$。 每个这样的选择都定义了一个独特的位移$S_k$，最终位置始终是$X + S_k$。 因此一致性相当于检查是否$D$是序列的前缀和之一。 一旦成立，起始位置就通过从最终位置减去位移来唯一确定，不会留下任何歧义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_prefix_sums(limit):
    # sequence: +1, -3, +4, -8, +16, ...
    vals = []
    cur = 1
    sign = 1
    for _ in range(limit):
        vals.append(sign * cur)
        sign *= -1
        if len(vals) >= 2:
            cur *= 2
        else:
            cur = 3  # second term is special: 3 instead of 2
    pref = []
    s = 0
    for v in vals:
        s += v
        pref.append(s)
    return pref

def solve():
    D = int(input())
    Y = int(input())

    # precompute enough terms (log scale bound)
    pref = build_prefix_sums(70)

    ok = False
    for s in pref:
        if s == D:
            ok = True
            break

    if not ok:
        print("Nao")
        return

    X = Y - D
    print("Sim")
    print(X)

if __name__ == "__main__":
    solve()
```该实现显式地构建移动序列并构建前缀和。 序列长度上限为 70，因为值呈指数增长，因此范围内的任何有效表示$10^{18}$必须出现在这个范围内。 

一致性检查是对前缀和的简单线性扫描。 一旦找到，我们计算$X$直接使用身份$Y = X + D$。 唯一微妙的一点是确保序列生成与所描述的加倍和交替模式完全匹配，因为第二项中的任何差一都会破坏整个结构。 

## 工作示例

 考虑一个输入，其中位移序列前缀和包括目标$D$。 假设序列产生前缀和：

 | 步骤 k | 移动| 前缀和|
 | ---| ---| ---|
 | 1 | +1 | 1 |
 | 2 | -3 | -2 |
 | 3 | +4 | 2 |
 | 4 | -8 | -6 |

 如果$D = 2$和$Y = 5$，我们检测到$k = 3$是有效的。 

| 步骤 k | 前缀和| 比赛 D |
 | ---| ---| ---|
 | 1 | 1 | 没有 |
 | 2 | -2 | 没有 |
 | 3 | 2 | 是的 |
 | 4 | -6 | 没有 |

 这确认了一致性和产量$X = Y - D = 5 - 2 = 3$。 

现在考虑没有前缀和匹配的情况：

 | 步骤 k | 前缀和|
 | ---| ---|
 | 1 | 1 |
 | 2 | -2 |
 | 3 | 2 |

 如果$D = 3$，没有匹配，所以输出不一致。

 ## 边缘情况

 当位移与非常早的前缀匹配时，就会出现一种边缘情况。 例如，如果$D = 1$，该过程可以在第一次移动后立即停止。 该算法处理此问题是因为前缀列表包含第一项，因此它可以正确识别一致性，而无需进一步迭代。 

另一个边缘情况是负位移。 由于序列交替符号并且大小增长，早期的前缀很容易产生负和，例如$-2$。 该检查不会假设呈阳性，因此会正确接受这些情况。 

最后的边缘情况是在允许的范围内没有前缀和匹配。 由于值呈指数增长，因此丢失所有匹配项意味着不一致，并且算法在用尽少量固定步骤后终止。
