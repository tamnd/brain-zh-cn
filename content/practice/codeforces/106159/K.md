 ## 解决方案
 ## 问题理解

 我们得到了一组时间戳，表示消息从未来到达的时间。 每个时间戳只是一个非常大的数轴上的单个整数。 阅读完所有消息后，我们被要求回答多个查询，其中每个查询给出一个闭区间$[L, R]$我们必须计算有多少存储的时间戳落在该间隔内。 

因此，输入是静态的：首先是固定的值列表，然后是对该同一列表的一批范围计数查询。 不会发生更新，并且每个查询都是独立的。 每个查询都询问整数行子范围的频率问题。 

约束足够大，任何每次查询扫描整个列表的方法都会立即变得太慢。 和$N, Q \le 2 \cdot 10^5$,天真的$O(NQ)$解决方案的性能可达$4 \cdot 10^{10}$最坏情况下的比较，这远远超出了 Python 甚至 C++ 在典型限制下 4 秒的运行时间。 这推动我们走向预处理和对数查询处理。 

一个微妙的点是时间戳是任意的$[1, 10^9]$，所以我们不能依赖直接索引或频率数组。 域太大，无法构建直接直方图。 

边缘情况主要围绕边界处理。 例如，如果所有时间戳都相同并且查询完全匹配或错过该值，则正确性完全取决于包含间隔逻辑。 

考虑：

 输入：```
3 1
5 5 5
5 5
```输出应该是：```
3
```一个幼稚的错误是将间隔视为半开或错误处理相等性，这会错误地返回 0 或 1。 

另一个边缘情况：```
4 1
1 3 5 7
2 4
```正确的输出是 1。任何边界比较中的差一或不正确的计数策略都会在此处中断。 

## 方法

 蛮力的想法很简单。 对于每个查询$[L, R]$，我们迭代所有$N$时间戳并计算有多少个位于范围内。 这是正确的，因为它直接匹配查询的定义。 但是，它会对每个查询重复相同的完整扫描。 和$Q$查询，这会导致$N \cdot Q$比较。 

当两者$N$和$Q$都很大，这就成为瓶颈。 即使每次比较都是常数时间，总的工作量也太大了。 

关键的观察是我们不需要保留时间戳的原始顺序。 查询仅取决于值范围内的成员资格，而不取决于位置。 一旦我们对时间戳进行排序，每个查询都会减少到查找排序数组中两个位置之间有多少元素。 

排序后，所有元素$\le R$形成前缀，所有元素$< L$形成另一个前缀。 答案就是两个前缀计数之间的差。 这正是二分查找所提供的：我们可以定位$R$和边界$L$在对数时间内。 

因此，我们不扫描数组，而是预排序一次，然后通过两次二分搜索回答每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(NQ)$|$O(1)$| 太慢了|
 | 排序 + 二分查找 |$O(N \log N + Q \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们将问题转换为排序问题，其中范围计数变成前缀边界计数。 

1. 将所有时间戳读入数组。 我们存储它们是因为我们需要对它们进行排序以进行数据的结构组织。 排序是必要的，因为它将任意值转换为应用二分搜索的单调序列。 
2. 按非降序对数组进行排序。 这一步是关键的转换：一旦排序，所有小于或等于阈值的值形成一个连续的前缀，这允许我们用边界查找代替计数。 
3. 对于每个查询$[L, R]$，计算排序数组中的两个位置：值大于或等于的第一个索引$L$，以及值大于的第一个索引$R$。 这两个边界精确地隔离了有效段。 
4. 查询的答案是这两个位置之间的差异。 这是有效的，因为左边界之前的所有内容都太小，右边界之后的所有内容都太大，而中间的所有内容都是有效的。 
5. 立即输出每个计算出的差异或存储结果以供最终输出。 

### 为什么它有效

 排序将问题从任意隶属度计数问题转换为单调序列上的区间长度提取问题。 每个查询实际上都在询问排序数组和区间之间的交集的大小。 二分搜索边界是精确的分割点，将数组划分为三个不相交的区域：以下值$L$，里面的值$[L, R]$，以及上面的值$R$。 由于这些区域按排序顺序是连续的，因此可以在不扫描元素的情况下计算它们的大小，并且不会重复计算或遗漏任何元素。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from bisect import bisect_left, bisect_right

def main():
    n, q = map(int, input().split())
    arr = list(map(int, input().split()))
    arr.sort()

    out = []
    for _ in range(q):
        l, r = map(int, input().split())
        left = bisect_left(arr, l)
        right = bisect_right(arr, r)
        out.append(str(right - left))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先读取时间戳并对其进行排序。 排序完成一次，这确保了以后的所有操作都可以依赖二分查找结构。 

每个查询使用两个标准二分搜索。`bisect_left`找到第一个位置`l`可以在不破坏顺序的情况下插入，有效地计算有多少元素严格小于`l`。`bisect_right`找到第一个大于的位置`r`，计算有多少个元素小于或等于`r`。 它们的差异准确地给出了区间中的元素数量。 

一个常见的陷阱是交换这两个函数或仅使用其中之一。 另一个微妙的问题是忘记了区间两端都包含在内，这就是为什么`bisect_right`是必要的而不是另一个`bisect_left`。

 ## 边缘情况

 对于单个重复值，排序数组会将所有结构折叠成相同的元素。 该算法仍然有效，因为`bisect_left`和`bisect_right`正确地将整个重复块括起来。 例如，使用输入`[5,5,5]`并查询`[5,5]`，两个边界隔离索引`[0,3)`，产生 3。 

对于完全超出范围的查询，例如`[4,4]`在`[1,2,3,4,5]`，两个二分搜索返回相同的索引，产生零。 这证实了空间隔是自然处理的，无需特殊的外壳。 

对于全方位查询，例如`[1,10^9]`,`bisect_left`返回 0 和`bisect_right`回报$N$，给出完整计数。 这表明该算法正确处理极端边界，没有溢出或丢失元素。
