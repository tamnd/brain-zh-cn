 ## 解决方案
 ## 问题理解

 我们被要求计算需要多少钻石块才能建造一个阶梯金字塔的高度$N$。 结构分层：水平$i$（从顶部的1开始）是一个边长为的正方形$2i+1$，所以它包含$(2i+1)^2$块。 因此，块的总数是从第 1 层到第 1 层的这些奇数边长的平方和$N$，并且答案必须以模形式给出$10^9 + 7$。 

输入是单个整数$N$，可以大到$10^{18}$。 这立即排除了任何迭代级别的方法。 甚至一个$O(N)$解决方案是不可能的，因为它需要最多$10^{18}$运营。 我们必须将求和压缩为封闭式表达式或极少量的算术运算。 

如果尝试计算，溢出处理中会出现一个微妙的问题$(2i+1)^2$反复无模块化护理。 由于中间值呈二次方增长$i$， 和$i$本身可以达到$10^{18}$，64 位算术中的简单乘法甚至在求和之前就已经不安全了。 

边缘情况在结构上是最小的，但在数字上很重要。 什么时候$N=1$，答案很简单$3^2 = 9$。 朴素循环在这里不是性能问题，但仍然可以作为健全性检查。 在上限$N=10^{18}$，只有基于公式的解决方案才是可行的。 

## 方法

 对问题的直接解释建议迭代每个级别并求和$(2i+1)^2$。 这是正确的，因为它遵循字面定义。 展开这个表达式给出$4i^2 + 4i + 1$，因此总数变为：$$\sum_{i=1}^{N} (4i^2 + 4i + 1)
= 4\sum i^2 + 4\sum i + \sum 1$$此转换是关键的简化步骤。 我们不是对每个级别的二次表达式求和，而是将问题简化为已知求和：$$\sum_{i=1}^{N} i = \frac{N(N+1)}{2}, \quad \sum_{i=1}^{N} i^2 = \frac{N(N+1)(2N+1)}{6}$$将它们代入表达式中会产生可在常数时间内计算的封闭形式。 瓶颈不再是迭代，而是具有大中间值的模运算，因此我们依赖于 2 和 6 的模逆$10^9 + 7$。 

暴力方法失败是因为它执行$N$迭代，每次计算一个平方并将其相加，从而得到$O(N)$时间。 在$N = 10^{18}$，这是不可行的。 优化方法使用代数恒等式将总和分解为常数时间算术。

 ## 算法演练

 我们首先重写块总数：$$\sum_{i=1}^{N} (2i+1)^2$$将平方展开得到多项式$i$，这允许我们使用标准求和公式。 

### 步骤

 1. 展开每一项$(2i+1)^2$进入$4i^2 + 4i + 1$。 

此步骤将几何结构转换为可以独立求和的代数分量。 
2. 将总和分成三个单独的总和：$$4\sum i^2 + 4\sum i + \sum 1$$这种分离之所以有效，是因为求和是线性的。 
3. 将每个求和的已知公式代入：$$\sum i = \frac{N(N+1)}{2}, \quad \sum i^2 = \frac{N(N+1)(2N+1)}{6}, \quad \sum 1 = N$$4. 将所有内容组合成一个模数表达式：$$4 \cdot \frac{N(N+1)(2N+1)}{6} + 4 \cdot \frac{N(N+1)}{2} + N$$5. 计算 2 和 6 的模逆$10^9 + 7$，并使用模乘仔细计算表达式以避免溢出。 

关键的实现细节是减少中间值模$10^9 + 7$在每个乘法步骤中，因为大整数的直接乘法，例如$10^{18}$在某些语言中甚至可以溢出 128 位算术，并且是不必要的。 

### 为什么它有效

 正确性依赖于以下事实：原始和中的每一项都是二次多项式$i$。 连续范围内多项式的任何和都可以使用幂和公式精确表示。 因为我们用以下形式重写了整个表达式$\sum i^2$,$\sum i$， 和$\sum 1$，这些是所有整数的精确恒等式$N$，变换保持平等。 模运算步骤不会改变正确性，因为所有运算都是在素数模定义的域中执行，其中除以 2 和 6 通过模逆是有效的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

inv2 = modinv(2)
inv6 = modinv(6)

def solve():
    N = int(input().strip())

    n = N % MOD

    sum_i = n * (n + 1) % MOD * inv2 % MOD
    sum_i2 = n * (n + 1) % MOD * (2 * n + 1) % MOD * inv6 % MOD

    # 4 * sum_i2 + 4 * sum_i + N
    ans = (4 * sum_i2) % MOD
    ans = (ans + 4 * sum_i) % MOD
    ans = (ans + n) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该解计算 2 和 6 的模逆一次，因为除以这些常数出现在两个求和公式中。 输入$N$减少模数$10^9 + 7$早期因为所有代数表达式都是多项式$N$，并减少早期保留正确性，同时防止中间乘法溢出。 

每个求和都是独立计算的，以避免混合项，从而使模数简化变得复杂。 最终表达式直接反映导出公式，确保实现错误的风险较低。

 ## 边缘情况

 对于$N=1$，算法计算$n=1$，然后直接计算两个总和。 中间值为：$$\sum i = 1, \quad \sum i^2 = 1$$最终结果变成$4 + 4 + 1 = 9$，配套单层塔。 

对于非常大的$N$， 例如$10^{18}$，算法从不迭代。 它减少了$N$模数$10^9+7$并执行固定的乘法和逆运算序列。 尽管原始值很大，但公式的多项式性质保证了求值之前的模归约在有限域中产生正确的结果。 

对于中间值，例如$N=2$，该算法正确地组合了两层的贡献，没有任何差一误差，因为所有总和都是在包含范围内定义的$[1, N]$，它与问题的从 1 开始的级别索引相匹配。
