 ## 问题理解

 我们给出一个无向图，其中顶点代表卡片，边代表卡片对之间的相互信任关系。 每个顶点定义一个“循环大小”：对于每张卡，我们查看包含它的所有简单循环，并采用最小的循环长度。 如果根本没有循环包含该卡，则其循环大小定义为 1。 

最终分数是通过对所有顶点、顶点索引与其最小循环大小的乘积求和来计算的。 

因此，任务不是全局枚举所有循环，而是为每个顶点分配 1（如果它不属于任何循环），或者分配包含该顶点的最小循环的长度。 

图约束很小，最多有 100 个顶点。 这立即表明三次甚至中等超二次解是可以接受的，但任何指数循环枚举都是不必要的。 问题的结构暗示我们关心通过每个顶点的最短周期，这是未加权图中的经典最短路径样式问题，具有附加约束，即路径必须返回到同一节点而不重用第一条边。 

一个天真的解释是枚举所有循环，计算它们的长度，并更新它们包含的所有顶点。 这会失败，因为密集图中的循环数呈指数增长。 即使在大小为 100 的完整图中，循环枚举也是很棘手的。 

一个更微妙的失败案例来自于将“通过顶点的周期长度”简单地视为图的全局周长。 周长是任何地方最短的循环，但该循环可能不包括给定的顶点。 例如，如果顶点 1、2、3 之间存在一个三角形，而 4、5、6 之间存在另一个单独的三角形，则顶点 4 必须获得循环大小 3，即使全局最短循环的长度也为 3。这在这里可行，但在一般图中，最短循环避免了某些顶点，全局计算会错误地标记它们。 

另一个边缘情况是树中附加到循环组件的顶点。 这些顶点应该获得值 1，即使它们可能接近循环。 任何假设“到环路的距离”或距环路节点的 BFS 的方法都是不够的，因为与环路相邻并不会使顶点成为环路的一部分。 

## 方法

 暴力方法会尝试枚举图中的每个简单循环。 对于找到的每个循环，我们将用迄今为止看到的最小循环长度更新其中的所有顶点。 虽然正确性很简单，但密集图中的简单循环数量以 N 为指数，使得这种方法即使在 N = 100 时也无法使用。 

另一个方向是单独关注每个顶点并计算包含它的最短周期。 固定一个顶点 v，并尝试删除一个关联边或强制一条返回 v 的路径。这自然会产生最短路径计算思想：如果我们想要通过 v 的最短循环，我们可以考虑从 v 开始，走到某个邻居 u，然后找到从 u 回到 v 的最短路径，而不立即使用边 (u, v)。 这减少了从每个邻居或从具有状态跟踪的 v 运行 BFS 的问题。 

关键的见解是，通过顶点的未加权最短循环可以分解为两个最短的不相交段：v 中的一条边和避开该边返回 v 的最短路径。 对于 v 的每个起始邻居 u，我们计算图中从 u 回到 v 的最短路径，其中忽略边 (v, u)。 所有此类选择的最小值给出了 v 的答案。 

由于 N 最多为 100，因此对每个顶点运行 BFS 是可以接受的。 每个 BFS 都是 O(N + M)，执行 N 次大约会产生 O(N(N + M))，这完全在限制范围内。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（循环枚举）| 指数| O(N + M) | 太慢了|
 | 具有边缘排除的每顶点 BFS | O(N(N + M)) | O(N(N + M)) | O(N + M) | 已接受 |

 ## 算法演练

 我们独立计算每个顶点的答案。 

1. 对于每个顶点 v，我们假设其循环大小从无穷大开始。 我们将尝试找到包含 v 的最佳循环。 
2. 对于顶点 v，我们将 v 的每个邻居 u 视为经过 v 的循环中潜在的第一步。这确保我们计数的任何循环实际上都使用 v 并立即通过真实边离开它。 
3.对于每个这样的邻居u，我们从u开始运行BFS来找到返回v的最短路径，但我们禁止再次直接使用边（u，v）。 这避免了无效的琐碎的双边“循环”。 
4. 从 u 到 v 的 BFS 距离给出路径长度 k。 那么通过 v 经过 u 的循环长度是 k + 1，因为我们在开头添加了边 (v, u)。 
5. 我们对所有邻居 u 取最小的循环长度。 如果没有 BFS 可以到达 v，则 v 不属于任何循环，并且我们分配循环大小 1。 
6. 计算出 v 的周期大小后，我们将 v * Cycle_size 添加到全局答案中。 

### 为什么它有效

 任何包含 v 的循环都必须从从 v 移动到某个邻居 u 开始，因为循环很简单并且必须通过其关联边之一离开 v。 一旦我们修复了第一条边，循环的剩余部分就是从 u 回到 v 的路径，不会重用第一条边。 BFS 保证在所有此类路径中，我们找到边数最短的一条。 因此，最小化 u 的所有选择涵盖了通过 v 的所有可能的简单循环，并且为每个 u 选择最短路径确保不会错过更短的循环。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def bfs(n, adj, start, target, banned_edge):
    dist = [-1] * (n + 1)
    q = deque([start])
    dist[start] = 0

    while q:
        v = q.popleft()
        if v == target:
            return dist[v]
        for to in adj[v]:
            if (v == banned_edge[0] and to == banned_edge[1]) or (v == banned_edge[1] and to == banned_edge[0]):
                continue
            if dist[to] == -1:
                dist[to] = dist[v] + 1
                q.append(to)
    return -1

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n + 1)]
    for _ in range(m):
        a, b = map(int, input().split())
        adj[a].append(b)
        adj[b].append(a)

    ans = 0

    for v in range(1, n + 1):
        best = float('inf')
        for u in adj[v]:
            d = bfs(n, adj, u, v, (v, u))
            if d != -1:
                best = min(best, d + 1)

        if best == float('inf'):
            best = 1

        ans += best * v

    print(ans)

if __name__ == "__main__":
    solve()
```该实现构建一个邻接列表，然后分别评估每个顶点。 对于每个顶点 v，它会尝试每个邻居作为 v 的第一步，并运行 BFS 来查找到 v 的最短返回路径，同时排除直接后边。 BFS 使用初始化为 -1 的距离数组来标记未访问的节点。 

一个微妙的点是边缘排除。 我们必须在循环搜索开始时排除 (v, u) 的精确定向使用，否则 BFS 会立即一步返回 v 并错误地计算出长度为 2 的循环，这在无向图中不是有效的简单循环。

 ## 边缘情况

 对于树状结构，例如简单路径 1-2-3-4，算法正确地将 1 分配给每个顶点，因为来自任何邻居的 BFS 都无法在不违反排除边规则或遇到不存在的循环的情况下返回到起始顶点。 每个顶点独立地以 best = 无穷大结束并转换为 1。 

对于连接到单个三角形加上悬空叶子的顶点，例如带有额外边 1-4 的三角形 1-2-3，单独处理顶点 4。 它唯一的邻居是 1，但从 1 到 4 的 BFS 无法找到环，因此它仍然是 1。同时，顶点 1、2 和 3 通过基于邻居的 BFS 运行正确检测到三角形，确认与环的邻接不会被误解为环的一部分。
