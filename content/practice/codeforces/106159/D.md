 ## 解决方案
 ## 问题理解

 我们得到一个长数字序列和一个较短的模式序列。 我们可以选择 0 到 9999 之间的单个移位值。应用此移位意味着将其添加到长序列的每个元素，并对所有元素取模 10000。经过此转换，我们想要计算移位数组包含作为连续子数组的模式的次数。 我们的任务是选择使出现次数最大化的班次，如果多个班次达到相同的最大值，我们必须输出最小的此类班次。 

关键结构是这种转变是全球性的。 我们不匹配模式中的任意值，我们要求：对于固定移位，在对 A 的每个元素应用相同的附加偏移量后，有多少个位置 i 满足 A[i..i+M-1] == B。 

限制非常大：数组长度最多可达一百万。 这立即排除了任何尝试独立模拟每个班次或从头开始为每个候选班次重新计算模式匹配的解决方案。 必须避免任何 N 甚至 O(N * 10000) 的二次方，除非它具有非常小的常数和强大的过滤功能。 

微妙的边缘情况来自模块化环绕。 由于值以 10000 为模，因此减法和等式会循环运行。 例如，模式匹配可能需要像 9998 这样的移位，即使原始差异看起来为负，除非正确处理模算术。 另一个边缘情况是多个职位平等地投票支持不同的轮班； 答案一定是最小的班次，因此领带处理很重要。 

一个简单的实现将尝试所有班次，并且对于每个班次扫描整个数组检查匹配。 这会在大输入上默默地失败，因为它执行大约 10^4 × 10^6 比较，这是不可行的。 

## 方法

 蛮力的想法很简单。 修复 Shift s，从概念上构造移位数组，然后将图案滑到其上。 如果对于所有 j，A[i+j] + s ≠ B[j] (mod 10000)，则每个位置 i 都贡献一个匹配。 检查所有班次会使成本乘以 10000，检查所有位置会使成本乘以 N，在最坏的情况下大约需要 10^10 次操作，这远远超出了限制。 

关键的观察是，移位是由 A 和 B 的每个对齐独立确定的。如果我们固定起始位置 i，那么对于模式中的每个 j，我们必须有 A[i+j] + s == B[j]。 这意味着所有值 B[j] - A[i+j] (mod 10000) 必须相等。 因此，每个对齐要么贡献一个一致的移位值，要么不贡献任何内容。 

这将问题简化为从 A 中大小为 M 的每个窗口中提取单个候选班次，验证一致性，并计算有多少窗口为每个班次投票。 我们不是模拟变化，而是聚合由差异引起的约束。 

效率增益来自于每个窗口最多产生一个有效移位这一事实，因此我们可以简单地以 O(NM) 处理所有窗口，但这仍然太慢。 然后我们观察到，我们只需要检测窗口中的所有差异是否相同，这可以使用滑动窗口哈希或滚动一致性检查来增量维护。 一种实用的方法是跟踪窗口中的第一个隐含移位并确保所有后续位置与其匹配； 如果出现不匹配，则该窗口无效。 

因此，我们可以在 O(N) 中计算每个位置的单个班次候选，并聚合超过 10000 个可能班次的频率。

 ## 算法演练

 我们将每个对齐方式重新定义为对移位的约束。

1. 对于 A 中适合长度为 M 的完整窗口的每个索引 i，我们尝试确定是否存在移位 s，使得移位 A[i..i+M-1] 使其等于 B。我们从第一对中计算候选移位，s = (B[0] - A[i]) mod 10000。这定义了如果该窗口有效，移位必须是多少。 
2. 我们通过检查每个 j 是否 (A[i+j] + s) mod 10000 等于 B[j] 来验证整个窗口是否针对此候选移位。 如果任何位置失败，则该窗口不会贡献任何有效的转变。 这是必要的，因为单个不匹配意味着差异不一致，因此没有全局移位可以满足该模式。 
3. 如果窗口有效，我们将增加 Shift s 的频率计数器。 这汇总了该转变下出现的模式的次数。 
4.处理完所有窗口后，我们扫描从0到9999的所有移位并选择频率最大的移位。 如果多个班次相同，我们通过向上迭代选择最小的一个。 

关键的想法是，每个有效的比赛窗口只为一个轮班贡献一票。 

### 为什么它有效

 对于移位 s 下位置 i 处模式的任何有效匹配，所有等式 A[i+j] + s == B[j] 必须同时成立。 这意味着 s 由任何单个位置 j 唯一确定，并且所有 j 之间的一致性相当于匹配的有效性。 相反，任何通过检查的窗口都会在恰好一个班次下产生正确的模式出现。 因此，对每个班次的有效窗口进行计数与对转换后的数组中的出现次数进行计数完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10000

def main():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    freq = [0] * MOD

    for i in range(n - m + 1):
        s = (b[0] - a[i]) % MOD
        ok = True

        for j in range(m):
            if (a[i + j] + s) % MOD != b[j]:
                ok = False
                break

        if ok:
            freq[s] += 1

    best_s = 0
    best_cnt = freq[0]

    for s in range(1, MOD):
        if freq[s] > best_cnt:
            best_cnt = freq[s]
            best_s = s

    print(best_s, best_cnt)

if __name__ == "__main__":
    main()
```代码直接实现了滑动窗口验证的思想。 模运算确保了环绕下的正确性。 给定约束条件下，大小为 10000 的频率数组是安全的。 

一个微妙的实现细节是内部循环的早期中断。 如果没有它，即使早期无效，每个窗口也总是会执行 M 检查，这仍然是正确的，但速度较慢。 另一个重要的细节是将最佳移位初始化为 0，因此在向上扫描时，领带会自动选择较小的移位。

 ## 边缘情况

 一个重要的边缘情况是每个窗口都无效。 在这种情况下，所有移位计数保持为零，正确答案是移位 0，因为我们选择最小的最大化移位。 该算法自然地处理这个问题，因为 freq[0] 被初始化为零，并且没有移位超过它，因此 best_s 保持为 0。 

另一种情况是环绕算术，其中减法产生负值。 例如，A = [9999, 0] 且 B = [0, 1]。 计算得出的移位为 (0 - 9999) mod 10000 = 1，这可以在模加后正确对齐两个元素。 该算法在每次比较时都依赖于模运算，因此除了模运算之外不需要特殊处理。 

第三种边缘情况是重复相同的窗口，多次产生相同的移位。 每个出现次数都是独立计数的，因为每个起始索引单独影响频率，这与移位数组中出现次数的定义相匹配。
