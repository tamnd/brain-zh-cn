 ## 解决方案
 ## 问题理解

 我们得到一系列歌曲，每首歌曲都与一个可以是正值或负值的值相关联。 我们想要选择这些歌曲的子集来最大化所选值的总和，但是有一个间距限制：如果我们在位置选择一首歌曲`i`，那么接下来我们就不能再选择任何歌曲了`K - 1`位置，意味着指数严格介于`i`和`i + K`无法选择。 

任务是计算在此规则下的最佳可能总和，包括根本不选择歌曲的可能性，这会产生总和为零。 

约束允许最多`N = 10^5`元素。 尝试所有子集甚至所有有效组合的简单解决方案将呈指数级爆炸，甚至检查每个位置的所有先前候选者的动态编程也将面临二次行为的风险。 在 1 秒的限制下，我们应该以大致线性或接近线性的时间为目标，当然不会比`O(N log N)`。 

当所有值都为负时，会出现微妙的边缘情况。 贪婪或强制选择解释可能会错误地选择一些歌曲，但最佳策略是不选择任何内容并输出零。 另一个边缘情况是当`K = 1`，这消除了所有限制并将问题简化为简单地获取所有积极贡献。 

## 方法

 暴力方法会尝试歌曲的每个子集，并检查它是否遵守间距规则。 这是正确的，但不可行，因为有`2^N`子集，甚至验证每个子集都会花费线性时间，从而导致天文数字般的运行时间。 

通过在每个索引上做出决定，更加结构化的蛮力略有改进：要么选择歌曲`i`并跳转到`i + K`，或跳过它并移至`i + 1`。 这种递归正确地探索了所有有效的解决方案，但仍然存在大量分支，在最坏的情况下会导致指数行为，因为相同的后缀状态会被重复重新计算。 

关键的观察结果是该问题具有针对位置的最优子结构。 如果我们从索引开始定义最佳答案`i`，这仅取决于未来的选择，以及当时的决定`i`是本地的：要么包含它并向前跳`K`，或排除它并继续`i + 1`。 这自然会导致对索引的动态规划。 

我们定义`dp[i]`作为我们从位置开始可以获得的最大总和`i`。 然后我们以相反的顺序计算转换，以便未来的状态是已知的。 这将问题简化为线性 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^N) | O(2^N) | O(N) | 太慢了|
 | DP 指数 | O(N) | O(N) | 已接受 |

 ## 算法演练

 我们从右到左处理数组，这样当我们到达位置时`i`，我们已经知道了超出它的位置的最佳答案。 

1.定义一个DP数组`dp`尺寸的`N + K + 5`，用零初始化。 额外的空间可以避免向前跳跃时的边界检查。 
2. 迭代`i`从`N`下降到`1`。 在每个位置，我们计算两种可能性。 
3.第一个选项是跳过当前歌曲，这很有价值`dp[i + 1]`。 这代表不使用歌曲`i`根本不。 
4.第二个选择是拍摄歌曲`i`，这给出了值`A[i] + dp[i + K]`。 这是有效的，因为选择后`i`，下一个有效位置是`i + K`。 
5. 设置`dp[i]`为这两个值中的最大值。 
6.填表后，答案为`dp[1]`，但我们也将其与零进行比较以允许空选择。 

### 为什么它有效

 在每个索引处`i`，国家`dp[i]`仅使用索引获取最佳可实现总和`i`以及更远的地方。 任何有效的解决方案必须包括`i`或排除它。 如果排除的话`i`，正是`dp[i + 1]`。 如果它包括`i`，该限制迫使下一个选择从`i + K`，最好的延续是`dp[i + K]`。 由于这两种选择都枚举了所有有效的可能性而没有重叠，因此递归是完整的，并且从数组末尾归纳地取最大值保留了最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K = map(int, input().split())
    A = list(map(int, input().split()))
    
    # 1-index the array for clarity
    A = [0] + A
    
    dp = [0] * (N + K + 5)
    
    for i in range(N, 0, -1):
        take = A[i] + dp[i + K]
        skip = dp[i + 1]
        dp[i] = max(take, skip)
    
    print(max(0, dp[1]))

if __name__ == "__main__":
    solve()
```DP 数组的大小带有额外的填充，以便`i + K`和`i + 1`始终有效的访问，避免循环内的条件检查。 转换直接对每个索引处的两个可能的决策进行编码。 

决赛`max(0, dp[1])`是必不可少的，因为递归假设我们必须从位置开始`1`，但如果所有贡献均为负，则最佳策略可能不选择任何内容。 

## 工作示例

 ### 示例 1

 输入：`N = 3, K = 1, A = [1, 2, 3]`这里`K = 1`意味着选择之间实际上没有限制。 

| 我| A[i] | dp[i+1] | dp[i+1] A[i] + dp[i+K] | dp[i] | dp[i] |
 | --- | --- | --- | --- | --- |
 | 3 | 3 | 0 | 3 | 3 |
 | 2 | 2 | 3 | 5 | 5 |
 | 1 | 1 | 5 | 6 | 6 |

 最终结果是`6`，确认可以采用所有元素。 

### 示例 2

 输入：`N = 4, K = 4, A = [2, 3, 0, 2]`这里选择任何索引都会阻止所有其他索引。 

| 我| A[i] | dp[i+1] | dp[i+1] A[i] + dp[i+K] | dp[i] | dp[i] |
 | --- | --- | --- | --- | --- |
 | 4 | 2 | 0 | 2 | 2 |
 | 3 | 0 | 2 | 0 | 2 |
 | 2 | 3 | 2 | 3 | 3 |
 | 1 | 2 | 3 | 2 | 3 |

 最优答案是`3`，对应选择第二首歌曲。 

这些痕迹显示了 DP 如何自然地平衡本地增益与未来可用性。

 ## 边缘情况

 当所有值都为负时，DP 仍将计算可实现的最佳后缀和，但每次转换都会传播负值，除非我们明确限制结果。 决赛`max(0, dp[1])`确保即使每个选择都会减少总和，我们也更喜欢空选择。 

用于输入`N = 3, K = 2, A = [-1, -2, -3]`，DP 的演化如下。 在`i = 3`,`dp[3] = max(-3, 0) = 0`。 在`i = 2`, 接受给予`-2 + dp[4] = -2`，跳过给出`0`， 所以`dp[2] = 0`。 在`i = 1`, 接受给予`-1 + dp[3] = -1`，跳过给出`0`， 所以`dp[1] = 0`。 最终的答案是`0`，匹配空播放列表。 

什么时候`K = 1`，每个索引都是独立的。 为了`A = [1, 2, 3]`，每个状态比较取`A[i] + dp[i+1]`反对跳过`dp[i+1]`，这简化为简单地累加所有正值，并且 DP 正确地总结了所有内容，因为所有贡献都是有益的。
