 ## 解决方案
 ## 问题理解

 我们有一场漫长的比赛，包括`n`圈，每圈从 26 名可能的车手中选出一名获胜者，并用小写英文字母标记。 所以完整的比赛只是一串长度`n`超过大小为 26 的字母表。 

我们还得到了这场比赛的一小段片段，一个字符串`s`长度`m`，它肯定会作为整个比赛中的连续块出现在某个地方。 然而，我们不知道它出现在哪里，我们也不知道这个片段之外的种族的其余部分。 

任务是计算有多少个完整的比赛序列长度`n`可能使得子串`s`出现在其中的某个地方。 如果获胜者至少有一个位置不同，则两场比赛被视为不同。 

关键的难点在于`n`可以非常大，最多 10^18，因此我们无法显式构造或迭代完整字符串。 片段长度`m`很小，最多 100，这表明所有有意义的结构必须来自如何放置该子串以及重叠或多次出现如何相互作用。 

天真的解释可能会尝试选择一个起始位置`s`在完整的字符串中，然后计算其他地方的任意填充。 这已经暗示了一个复杂的情况：如果`s`可以出现多次，或者与自身重叠，那么不同的位置不是独立的。 

出现微妙的边缘情况时`s`具有内部周期性结构。 

例如，如果`s = "aaa"`， 和`n = 3`，那么子字符串只能从位置 1 开始出现，但其余字母是完全强制的。 粗心的解决方案可能会因单独处理展示位置而导致计算过多。 

另一个边缘情况是当`s`与自身重叠，例如`s = "ababa"`，其中模式的转变可以与有效匹配重合。 忽略重叠结构会导致配置的重复计算。 

因此，主要的挑战不仅在于放置，还在于多次出现的模式如何能够在整个字符串中一致共存。 

## 方法

 暴力方法是考虑每个可能的长度字符串`n`超过26个字母，并检查是否`s`显示为子字符串。 这显然是不可行的，因为字符串的数量是 26^n，即使对于小字符串来说，这也是一个天文数字。`n`。 

更结构化的蛮力改进是固定起始位置`i`对于发生`s`， 从`1`到`n - m + 1`。 对于每一个选择`i`，我们修复这些`m`字符，然后自由选择剩余的`n - m`职位。 这产生`(n - m + 1) * 26^(n - m)`字符串。 然而，当`s`可以出现在同一字符串中的多个位置，因为单个构造的字符串可能包含`s`不止一次。 

关键的观察结果是，问题不在于单个放置，而在于由出现的情况引起的一致性约束`s`在一个长度内`n`细绳。 每个安置`s`施加固定字母，并且重叠放置施加位置之间的相等约束。 这是一个经典的结构，我们将字符串视为约束系统。 

自从`m`很小，重叠结构`s`可以通过它的边界函数（前缀也是后缀）来充分理解。 这决定了如何多次出现`s`可以在更大的字符串内对齐。 

我们对展示位置进行建模`s`作为位置约束图中的节点，其中重叠副本合并索引。 结果是，每个有效的全局配置都是通过选择一组一致的放置来确定的，但一致性会降低自由度。 

关键的简化是，不用对所有的事情进行推理`n`，我们只跟踪长度的重叠情况`m`行为，以及有多少独立职位保持空闲。 最终答案变成 26 的幂，并根据结构所引入的强制相等数进行调整`s`，乘以长度内模式的有效移位对齐数`n`。 

因此，该解决方案简化为计算在考虑了嵌入的所有强制约束后剩余多少个有效的独立字符槽`s`长度中的任意位置`n`细绳。

 | 基于约束+前缀函数的计数 | O(米) | O(米) | 已接受 |

 ## 算法演练

 我们将问题解释为计算长度的字符串`n`超过 26 个字母包含`s`至少一次，但不是直接包含-排除`n`，我们转变视角：我们首先计算所有字符串并减去那些避免的字符串`s`。 

这将问题转化为长度模式上的经典自动机计数问题`m`。 

1.构造前缀函数`s`，它告诉我们，对于每个前缀结束位置，最长的边界也是前缀`s`。 这对扩展字符串时部分匹配转换的方式进行编码。 需要这样做的原因是，当从左到右扫描字符串时，部分匹配`s`可以重叠，我们需要一种确定性的方法来跟踪该状态。 
2. 构建自动机`m`州，其中州`i`表示我们当前已经匹配了一个长度的前缀`i`的`s`。 对于每个状态和角色`c`，使用前缀函数转换计算下一个状态。 如果我们达到状态`m`，这意味着我们完全匹配`s`。 
3.我们现在对字符串长度进行动态规划`n`，但自从`n`高达 10^18，我们无法线性迭代。 相反，我们将转换表示为大小矩阵`m × m`，其中条目`(i, j)`计算状态的转换`i`到`j`没有完成模式。 
4. 我们对这个转移矩阵使用快速求幂来计算有多少个字符串长度`n`避免达到最终状态。 应用矩阵求幂的原因是自动机转换在状态分布上形成线性递归。 
5.让`total = 26^n mod MOD`。 让`bad`是长度的字符串的数量`n`永远不会达到状态`m`。 那么答案就是`total - bad`。 
6.我们计算`26^n mod MOD`使用二进制求幂，并计算矩阵求幂结果`bad`。 

### 为什么它有效

 每个位置的自动机状态精确地编码了最长的前缀`s`匹配以该位置结尾的后缀。 这确保了每种可能发生的情况`s`在发生时进行精确跟踪，不会遗漏重叠或重复计算。 任何至少包含一次出现的字符串最终都会达到终止状态，并且任何避免出现的字符串`s`与从未到达它的路径完全对应。 因此，对有效字符串进行计数可以简化为对有限自动机中的路径进行计数，这可以通过矩阵求幂来完全捕获。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def build_prefix(s):
    m = len(s)
    pi = [0] * m
    for i in range(1, m):
        j = pi[i - 1]
        while j > 0 and s[i] != s[j]:
            j = pi[j - 1]
        if s[i] == s[j]:
            j += 1
        pi[i] = j
    return pi

def build_automaton(s, pi):
    m = len(s)
    nxt = [[0] * 26 for _ in range(m + 1)]
    for i in range(m + 1):
        for c in range(26):
            if i == m:
                nxt[i][c] = m
                continue
            j = i
            while j > 0 and chr(ord('a') + c) != s[j]:
                j = pi[j - 1]
            if chr(ord('a') + c) == s[j]:
                j += 1
            nxt[i][c] = j
    return nxt

def mat_mul(a, b):
    m = len(a)
    res = [[0] * m for _ in range(m)]
    for i in range(m):
        for k in range(m):
            if a[i][k]:
                for j in range(m):
                    res[i][j] = (res[i][j] + a[i][k] * b[k][j]) % MOD
    return res

def mat_pow(mat, exp):
    m = len(mat)
    res = [[0] * m for _ in range(m)]
    for i in range(m):
        res[i][i] = 1
    while exp:
        if exp & 1:
            res = mat_mul(res, mat)
        mat = mat_mul(mat, mat)
        exp >>= 1
    return res

def solve():
    n, m = map(int, input().split())
    s = input().strip()

    pi = build_prefix(s)
    nxt = build_automaton(s, pi)

    size = m
    trans = [[0] * size for _ in range(size)]

    for i in range(size):
        for c in range(26):
            j = nxt[i][c]
            if j < m:
                trans[i][j] += 1

    mat = mat_pow(trans, n)

    dp0 = [0] * m
    dp0[0] = 1

    bad = 0
    for i in range(m):
        bad = (bad + mat[0][i]) % MOD

    total = pow(26, n, MOD)
    print((total - bad) % MOD)

if __name__ == "__main__":
    solve()
```该实现首先构建前缀函数，当字符中断当前匹配时，该函数允许有效的回退。 自动机构造明确定义了部分匹配状态之间的转换，确保表示字符串的每个可能的扩展。 

转换矩阵仅包含非终结符转换，因为一旦模式匹配，我们就不想将那些字符串计入“avoiding s”DP 中。 矩阵求幂将这个转换系统延伸至长度`n`，有效地模拟该长度的所有可能的字符串，无需线性迭代。 

最后减去`26^n`将补码计数转换为至少包含一次出现的字符串数`s`。 

一个常见的微妙之处是正确处理终止状态：它必须吸收所有转换以避免重新进入 DP，否则会发生过度计数。

```我们在该模式上构建一个自动机`"ilprw"`。 DP 跟踪状态 0 到 5，其中 5 是终止状态。 

| 步骤| 意义|
 | --- | --- |
 | 总字符串 | 26^6 | 26^6

 自动机求幂对所有从未达到状态 5 的长度为 6 的字符串进行计数。从总数中减去得到答案。 

这证实我们正在计算子字符串的所有位置，包括从任何有效位置开始的位置。 

### 示例 2

 输入：```

ababa
```这里的模式有很强的重叠：`"ababa"`在前缀处与自身重叠`"aba"`。 自动机捕获 3 → 5 等过渡，然后通过边界结构返回。 

DP 确保重叠出现不会破坏正确性，因为前缀函数转换会正确合并它们。

 ## 边缘情况

 一个关键的边缘情况是模式是重复的，例如`"aaaa..."`。 在这种情况下，前缀函数创建一个长链，其中每个字符都不断扩展匹配。 自动机将许多转换折叠成一条路径，并且对最终状态的错误处理会导致严重少计或多计。 正确的构造确保一旦状态`m`达到后，它仍然很吸引人，因此任何包含该模式的字符串都会从“坏”计数中排除一次。 

另一个边缘情况是当`m = 1`。 那么该模式就是一个字母，自动机就简化为一个简单的二态系统。 矩阵求幂仍然有效，但从状态 0 初始化转换时的任何错误都会立即导致计数错误，因为每个字符都会直接触发接受。 

最后一个微妙的情况是`n = m`。 这里每个字符串要么等于模式，要么不等于。 自动机正确地减少了检查字符串是否恰好匹配一次，并从`26^n`产生正确的单次出现结构而无需重复计算。
