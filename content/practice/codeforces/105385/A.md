 ## 解决方案
 ## 问题理解

 我们拥有几台独立运行的印刷机，但有助于实现相同的共同目标：至少生产$k$文件的总份数。 每台打印机并不是以简单的线性方式以恒定的长期速率工作。 相反，它遵循一个循环。 在每个周期中，它每隔$t_i$固定突发长度秒$l_i$副本，然后关闭$w_i$重复相同行为之前的几秒钟。 

因此，每台打印机都会在活动阶段和冷却阶段之间交替。 在活动阶段，它输出$l_i$以稳定的速度复印一份$t_i$秒，采取$t_i \cdot l_i$总秒数。 之后，它暂停$w_i$秒，并且该模式重复。 

所有打印机从零时间开始同时运行。 目标是确定最小时间$T$这样所有打印机按时间生产的总份数$T$至少是$k$。 

这些限制使得简单的模拟变得不可能。 打印机数量最多100台，但所需份数$k$可以大到$10^9$，时间参数也可以达到$10^9$。 这立即排除了以幼稚的方式逐秒甚至逐事件进行模拟的可能性。 任何方法都必须对一段时间内的产量进行总体评估。 

一个常见的微妙故障案例来自于部分周期处理不当。 打印机有时可能正处于活动阶段或空闲阶段$T$，并且仅计算完整周期会导致计数不足。 

例如，如果打印机有$t=2, l=3, w=10$，那么在 7 秒内它还没有完成其第一个活动阶段（需要 6 秒），因此它生成了 3 个副本。 单纯的仅循环计算可能会错误地将时间 7 视为产生 0 个完整循环，因此产生 0 个副本，这是错误的。 

另一个失败案例来自假设线性速率。 由于停机，有效速度不是恒定的，因此使用$\frac{l_i}{t_i l_i + w_i}$乘以$T$只给出一个近似值，对于边界决策来说并不总是足够精确。 

## 方法

 一个直接的方法是模拟时间。 对于每台打印机，我们跟踪其循环位置并计算其截至时间生产的份数$T$。 对于固定的$T$，我们可以计算输出$O(n)$如果仔细完成的话，每台打印机都可以，但是我们需要尝试所有可能的次数。 自从$T$可能非常大，这变得不可行。 

关键的结构洞察力是单调性。 如果一个时间$T$足以产生至少$k$份，那么任何更长的时间也足够了。 这种单调属性使我们能够用二分搜索代替随时间推移的搜索。 

剩下的任务是计算，在固定的时间内$T$，每台打印机生产多少份。 每台打印机重复一个长度循环$c_i = t_i l_i + w_i$。 在每个完整的周期中，它准确地产生$l_i$副本。 我们可以计算有多少个完整的周期适合$T$, 乘以$l_i$，然后通过检查在时间用完之前可以完成多少活动阶段副本来处理剩余时间。 

这减少了对候选时间的评估，从潜在的复杂模拟到每台打印机的直接算术计算。

 | ---| ---| ---| ---|
 | 暴力模拟|$O(T \cdot n)$|$O(1)$| 太慢了|
 | 二分查找+计数|$O(n \log T)$|$O(1)$| 已接受 |

 ## 算法演练

 我们想要找到最小的时间$T$使得总制作份数至少达到$k$。 

1. 定义一个函数$f(T)$计算所有打印机及时生产的份数$T$。 对于每台打印机，我们首先计算其完整周期长度$c_i = t_i l_i + w_i$。 这捕获了工作时间和冷却时间。 
2. 对于每台打印机，计算有多少个完整周期适合$T$作为$q = T // c_i$。 每个完整周期都准确贡献$l_i$份，所以全周期贡献是$q \cdot l_i$。 
3. 计算剩余时间$r = T \% c_i$。 在此剩余段期间，打印机可能仍处于活动阶段。 活跃阶段包括产生最多$l_i$份，每份$t_i$秒。 因此，额外副本的数量为$\min(l_i, r // t_i)$。 
4. 将所有打印机的贡献相加得到$f(T)$。 这个函数是单调的$T$，这意味着它永远不会减少$T$增加。 
5. 随着时间的推移执行二分查找。 下限为 0。上限可以安全地设置为$\min\_time \cdot k$， 在哪里$\min\_time$是最小的$t_i$，因为即使是最快的打印机也可以在这段时间内打印出所有副本。 
6. 在每个中点$mid$， 评价$f(mid)$。 如果至少是$k$，将上限向左移动。 否则，将下限向右移动。 
7. 最终的下限是满足要求的最短时间。 

### 为什么它有效

 正确性基于以下事实：每台打印机的输出随时间的推移是一个逐步非递减函数。 它在活动阶段增加并在停机期间保持恒定，但永远不会减少。 对这些函数求和可以保持单调性。 这保证了谓词“至少可以产生$k$按时间复制$T$”随着时间的推移定义了一个连续的区域，允许二分搜索准确地找到边界。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(t, printers, k):
    total = 0
    for ti, li, wi in printers:
        cycle = ti * li + wi
        full = t // cycle
        total += full * li

        rem = t % cycle
        total += min(li, rem // ti)

        if total >= k:
            return True
    return total >= k

def solve():
    T = int(input())
    for _ in range(T):
        n, k = map(int, input().split())
        printers = [tuple(map(int, input().split())) for _ in range(n)]

        lo, hi = 0, 10**18

        while lo < hi:
            mid = (lo + hi) // 2
            if can(mid, printers, k):
                hi = mid
            else:
                lo = mid + 1

        print(lo)

if __name__ == "__main__":
    solve()
```功能`can`实行固定时间核心考核。 它仔细地将完整周期与部分周期分开。 完整周期是简单的乘法，而其余部分必须仅限于活动阶段。 提前退出可以提高速度$k$已达到。 

二分查找使用较大的上限$10^{18}$，它安全地涵盖了生产极其缓慢的最坏情况。 即使在这种松散的界限下，单调检查也能确保正确性。 

一个微妙的实现细节是对循环和剩余生产使用整数除法。 任何浮点方法都会引入精度问题，并且在大输入时会失败。

 ## 边缘情况

 一个重要的边缘情况是当$T$落在周期内但处于活动阶段之外。 认为$t=2, l=3, w=10$和$T=7$。 打印机在 16 秒内完成一个完整的循环，因此$T=7$它仍处于活跃阶段。 它已经产生了$3$自那时以来的副本$7 // 2 = 3$。 该算法使用余数逻辑正确地捕获了这一点。 

另一个边缘情况是当$T$正好位于循环边界上。 如果$T = k \cdot (t_i l_i + w_i)$，则余数为零，贡献恰好为$k \cdot l_i$。 该公式避免了重复计算或遗漏部分工作。 

最后一个边缘情况是$k$非常大，只有最快的打印机才重要。 二分搜索仍然可以正确收敛，因为可行性检查与总输出成线性比例，并且 Python 的大整数避免了溢出。
