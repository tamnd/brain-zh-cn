 ## 解决方案
 ## 问题理解

 给定一个字符串，我们可以将它循环向左旋转一定数量的位置。 左移一位$d$表示从position开始取子串$d$到最后并附加前缀$0 \dots d-1$在最后。 

每次旋转后，我们只查看旋转字符串的第一个和最后一个字符。 我们想要最小的旋转量$d \ge 0$使得这两个字符变得相等。 如果没有旋转使第一个和最后一个字符匹配，我们输出$-1$。 

重新构建这一点的一个有用方法是想象圆上的绳子。 每次旋转只选择一个起始索引$i$，结果字符串开始于$i$并结束于$i-1$模数$n$。 所以我们实际上要求的是最小的索引$i$这样$s[i] = s[i-1]$在循环意义上，我们返回$i$作为移位量。 

约束允许总字符串长度达到$5 \times 10^5$，这排除了尝试每次旋转并重新计算端点或每次轮班扫描字符串的任何解决方案。 任何与总输入大小成二次方的东西显然都会失败。 

当所有字符都不同时，就会出现微妙的边缘情况。 例如，输入`"abc"`即使在循环意义上也没有一对相邻的相等字符，因此任何旋转都不能使端点相等，答案必须是$-1$。 仅检查线性邻接而不考虑环绕的简单方法可能会错误地错过最后一个字符和第一个字符之间的过渡，或者相反可能会错误地假设移位后存在匹配。 

另一种边缘情况是字符串已经以相同的字符开始和结束。 为了`"abca"`，答案是$0$，并且任何算法都必须确保它不会仅仅因为存在其他匹配而意外返回更大的旋转。 

## 方法

 蛮力方法尝试每一个可能的转变$d$从$0$到$n-1$。 对于每次移位，我们从概念上旋转字符串并比较第一个和最后一个字符。 构建旋转字符串成本$O(n)$，即使我们避免构造并且只是索引模$n$，检查端点是$O(1)$，所以这部分很好。 真正的问题是我们仍然需要通过考虑所有因素来决定正确性$d$，即$O(n)$每个测试用例的检查。 

然而，关键的观察是旋转仅改变哪个索引成为第一个字符。 如果我们选择起始索引$i$，那么最后一个字符总是$s[i-1]$。 因此“第一个等于最后一个”的条件变成了纯粹的局部条件：$s[i] = s[i-1]$。 

这将整个问题分解为寻找最小索引$i$使得循环字符串中的相邻字符匹配。 我们可以扫描一次字符串，检查每一对$(i-1, i)$，还有环绕对$(n-1, 0)$。 最小的此类索引$i$就是答案。 如果不存在，我们输出$-1$。

 | ---| ---| ---| ---|
 | 暴力旋转 |$O(n^2)$|$O(1)$| 太慢了 |
 | 圆形邻接的线性扫描|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们想要确定最后一个字符和第一个字符之间的边界变得有效的最早旋转点。 

1. 将每个可能的旋转视为选择起始索引$i$。 旋转后，第一个字符是$s[i]$最后一个是$s[i-1]$在循环索引中。 这种重新表述消除了模拟旋转的任何需要。 
2.检查所有索引$i$从$0$到$n-1$。 对于每个$i$， 比较$s[i]$和$s[i-1]$，其中索引$-1$对应于$n-1$。 这一步直接测试是否旋转$i$产生一个“美丽”的字符串。 
3. 保留最小索引$i$则满足等式。 由于我们从左到右扫描，第一个有效的$i$自动最小。 
4. 如果没有索引满足条件，则返回$-1$。 

### 为什么它有效

 左移后$i$，旋转后的字符串正是循环重新标记，其中位置$0$对应原始索引$i$和位置$n-1$对应原始索引$i-1$。 因此，有效答案的条件仅取决于原始圆形排列中的单个邻接。 每个可能的旋转字符串恰好对应于一个这样的邻接检查，并且没有两个不同的移位会产生超出该边界对的不同比较。 这种一对一的映射确保扫描所有圆形邻接耗尽所有可能的旋转而不会遗漏或重复。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    if n == 1:
        print(0)
        return

    for i in range(n):
        if s[i] == s[i - 1]:
            print(i)
            return

    print(-1)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该解决方案依赖于扫描每个位置一次并以圆形形式检查其左邻居。 表达式`s[i - 1]`自然地处理环绕情况，因为 Python 负索引映射`-1`到最后一个字符，它与索引完全对应$n-1$。 

特殊情况$n=1$单独处理，因为单字符串对于旋转总是很漂亮$0$，否则循环仍会正常运行，但显式情况使意图清晰并避免不必要的迭代。

 ## 边缘情况

 对于像这样的字符串`"abab"`，每个字符都与其邻居不同，即使是圆形。 扫描检查所有索引：

 当 i = 0 时，比较`a`与最后`b`，没有匹配。 当 i = 1 时，`b`与`a`，没有匹配。 当 i = 2 时，`a`与`b`，没有匹配。 当 i = 3 时，`b`与`a`，没有匹配。 该算法在没有找到有效索引和输出的情况下完成$-1$，这是正确的，因为没有旋转可以对齐端点。 

为了`"aa"`，在 i = 0 时我们比较`a`和`a`（环绕），立即返回 0。这证实了圆形邻接正确捕获了旋转边界条件，而无需显式旋转模拟。
