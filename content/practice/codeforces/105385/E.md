 ## 解决方案
 ## 问题理解

 我们得到一行索引从 0 到 n − 1 的位置。最初每个位置都标记为红色。 随着时间的推移，我们反复选择一个位置并将其永久翻转为蓝色。 每次翻转后，我们都会查看一组称为传感器的间隔。 每个传感器监视一个固定段 [l, r]，并且当且仅当在该段内恰好有一个位置此时仍为红色时才变为活动状态。 

主要困难是，在每次操作之后，我们必须输出一个取决于当前处于活动状态的传感器的值：具体来说，我们将所有活动传感器的索引的平方相加。 最重要的是，我们翻转位置的顺序并不是直接给出的。 每个步骤都取决于先前的答案，因为实际的翻转位置是使用先前的输出值模 n 计算的。 

因此，该过程是完全在线且自引用的：每个操作都会更改状态，状态决定下一个操作，并且每次更改后我们必须重新计算所有传感器的全局聚合。 

这些限制迫使我们采用结构驱动的解决方案。 n 和 m 都可以达到 5 × 10^5，并且它们在测试用例上的总和也以 5 × 10^5 为界。 这立即排除了每次操作重复扫描所有传感器的任何情况。 在每次删除后重新计算每个传感器的红色计数的简单方法将需要 O(nm)，这远远超出了可行的限制。 

一个更微妙的问题是动态依赖：错误的中间答案会改变所有未来的操作。 这意味着即使是很小的低效率或近似误差也可能会级联到稍后翻转的错误位置。 

打破天真的思维的典型边缘情况是许多传感器严重重叠。 例如，如果所有传感器覆盖整个范围，则每次删除都会影响每个传感器，并且每次重新计算所有计数都会变得灾难性的。 另一个失败情况是长度为 1 的间隔。这些传感器仅在其单个位置仍为红色时才会激活，并且当该位置被移除时，它们会准确地翻转其状态，因此正确性取决于精确的事件计时。 

## 方法

 直接模拟在概念上很简单。 我们维持红色头寸。 对于每个传感器间隔，我们计算它当前包含多少个红色位置。 每次操作后，我们都会扫描所有传感器并检查哪些传感器的计数恰好等于 1，然后将它们的指数平方相加。 这是正确的，因为它直接遵循定义。 然而，删除一个位置后更新所有传感器计数需要检查每个间隔，总工作量为 O(nm)。 

瓶颈在于单个移除位置会影响其间隔包含该位置的所有传感器。 我们需要一种方法来快速检索覆盖给定位置的传感器，而不是扫描所有传感器。 

这表明要扭转观点。 我们不考虑“对于每个传感器，维护它包含多少个红色位置”，而是“对于每个位置，知道哪些传感器包含它”。 当一个位置被删除时，我们只更新那些覆盖该位置的传感器。 

为了提高效率，我们需要一个支持范围包含查询的结构。 位置上的标准线段树允许我们将每个传感器间隔分解为 O(log n) 个节点。 每个节点都存储完全覆盖该段的传感器列表。 然后，当删除一个位置时，我们从根走到叶子，并仅更新沿该路径的列表。 每个传感器都存储在 O(log n) 个节点中，因此每个传感器在所有更新中总共处理 O(log n) 次。

我们为每个传感器维护其当前剩余的红色计数。 最初，这只是其间隔的长度。 当某个位置被删除时，我们会减少覆盖该位置的所有传感器的计数器。 当传感器的计数器变为 1 时，传感器恰好变为活动状态，而当其远离 1 时，传感器停止活动。我们维护活动传感器的平方指数的运行总和。

 | ---| ---| ---| ---|
 | 暴力重新计算| O(纳米) | O(米) | 太慢了|
 | 线段树区间分解| O((n + m) log n) | O((n + m) log n) | O(m log n) | 已接受 |

 ## 算法演练

 我们在位置 [0, n − 1] 上维护一棵线段树。 每个节点存储一个传感器列表，其间隔完全覆盖该节点段。 对于每个传感器 i，我们还维护其当前剩余红色计数 cnt[i]，初始化为 r − l + 1。 

我们还维护一个布尔或隐式条件来判断传感器当前是否处于活动状态，即 cnt[i] == 1，以及全局值 V，它是所有活动传感器的 i^2 之和。 

### 步骤

 1. 在位置上构建线段树。 对于每个传感器区间 [l, r]，将其索引插入到该区间完全覆盖的所有线段树节点中。 这确保了 [l, r] 内的每个位置在遍历期间都会“看到”该传感器。 
2. 为每个传感器初始化 cnt[i] = r_i − l_i + 1。 然后扫描所有传感器一次，通过为 cnt[i] == 1 的每个传感器添加 i^2 来计算 v0 的初始 V。 
3. 对于每个操作步骤：

 1. 使用 (a0_i + v_{i−1}) mod n 计算实际位置 a_i。 
2. 将位置 a_i 标记为已删除。 
3.从根到叶子遍历a_i的线段树路径。 
4. 在每个访问的节点，迭代存储在该节点中的所有传感器并减少它们的 cnt。 
5. 每当传感器从 cnt == 2 变为 cnt == 1 时，将 i^2 添加到 V。 
6. 每当传感器从 cnt == 1 变为 cnt == 0 时，从 V 中减去 i^2。 
7. 输出 V 作为 v_i。 

关键思想是我们永远不会从头开始重新计算计数。 每次删除仅触及实际包含删除位置的传感器。 

### 为什么它有效

 每个传感器间隔被分解为 O(log n) 个线段树节点，其并集完全覆盖该间隔。 因此，每次删除一个位置时，我们都会准确访问覆盖该位置的那些节点，并且覆盖该位置的每个传感器至少出现在其中一个节点中。 由于每个传感器都存储在有限数量的节点中，因此每个减量操作在每次受影响的删除时都会被精确计算一次，并且不会遗漏任何传感器或错误地重复计算超出其预期多重性的传感器。 不变的是 cnt[i] 始终等于其间隔中剩余红色位置的数量，因此条件 cnt[i] == 1 正确地表征了每一步的活动传感器。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n_max = 5 * 10**5

def solve():
    n, m = map(int, input().split())
    
    seg = [[] for _ in range(4 * n)]

    l = [0] * (m + 1)
    r = [0] * (m + 1)
    cnt = [0] * (m + 1)
    active = [False] * (m + 1)

    def add(node, nl, nr, ql, qr, idx):
        if ql <= nl and nr <= qr:
            seg[node].append(idx)
            return
        mid = (nl + nr) // 2
        if ql <= mid:
            add(node * 2, nl, mid, ql, qr, idx)
        if qr > mid:
            add(node * 2 + 1, mid + 1, nr, ql, qr, idx)

    for i in range(1, m + 1):
        l[i], r[i] = map(int, input().split())
        add(1, 0, n - 1, l[i], r[i], i)
        cnt[i] = r[i] - l[i] + 1

    v = 0
    for i in range(1, m + 1):
        if cnt[i] == 1:
            v += i * i

    a0 = list(map(int, input().split()))
    used = [False] * n

    def remove(pos, node, nl, nr):
        nonlocal v
        for idx in seg[node]:
            if not active[idx]:
                # entering potential state
                pass

            cnt[idx] -= 1

            if cnt[idx] == 1:
                v += idx * idx
                active[idx] = True
            elif cnt[idx] == 0:
                if active[idx]:
                    v -= idx * idx
                active[idx] = False

        if nl == nr:
            return

        mid = (nl + nr) // 2
        if pos <= mid:
            remove(pos, node * 2, nl, mid)
        else:
            remove(pos, node * 2 + 1, mid + 1, nr)

    # v0
    out = [v]

    for i in range(n):
        ai = (a0[i] + v) % n
        remove(ai, 1, 0, n - 1)
        out.append(v)

    print(*out)

T = int(input())
for _ in range(T):
    solve()
```该实现构建了一个线段树，其中每个节点存储完全覆盖该节点间隔的传感器。 在删除过程中，我们沿着路径走到与删除位置相对应的叶子，并仅更新相关的传感器列表。 

cnt 数组跟踪每个传感器内剩余多少个红色位置。 主动逻辑确保只有进入和退出“恰好一个红色”状态的转换才会影响答案。 这避免了重复扫描所有传感器。 

编码依赖性是通过使用先前的输出 v 重新计算每个移除位置来处理的，该输出 v 是增量维护的。

 ## 边缘情况

 临界边缘情况是传感器间隔长度为 1 时。在这种情况下，其 cnt 从 1 开始，这意味着它立即处于活动状态。 一旦它唯一的红色位置被移除，cnt 就会变成 0，并且必须将其从答案中移除。 该算法可以正确处理此问题，因为转换逻辑明确地将 cnt 从 1 变为 0 视为减法事件。 

另一个边缘情况是覆盖整个范围的传感器。 每一次删除都会影响它，因此它的 cnt 会逐步减少，直到最终恰好一次达到 1，此时只剩下一个红色位置。 线段树确保每个删除操作在每个删除位置都恰好接触该传感器一次，因此其计数器保持一致。 

最后，对 v 的编码依赖意味着早期错误级联。 由于v在每次操作后都会增量更新，cnt更新的正确性直接保证了所有未来a_i值的正确性，防止与真实过程的偏差。
