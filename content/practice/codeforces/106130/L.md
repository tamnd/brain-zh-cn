 ## 解决方案
 ## 问题理解

 我们得到一个代表一行硬币的二进制字符串。 每个位置要么是 0，要么是 1，目标是将整个字符串转换为全 1。 

唯一允许的操作是选择两个相邻的位置并同时翻转两个硬币。 翻转意味着将 0 变为 1，将 1 变为 0。因此，每次移动恰好切换两个连续位。 

问题纯粹是关于可达性：从初始配置开始，我们可以使用任意数量的此类相邻双翻转操作来达到全一配置吗？ 

输入量极小，最多20个硬币。 这立即表明，如果需要的话，指数状态探索是可行的，而且我们应该能够导出一个干净的结构不变量，而不是依赖蛮力。 

一个微妙的点是操作的效果是重叠的。 位置 i 处的翻转会影响 i 和 i+1，因此移动会在本地交互，但可以在整个数组中传播更改。 这通常会导致基于奇偶校验的约束。 

值得指出的边缘情况来自非常小的字符串和已经解决的配置。 

例如，如果字符串已经像 input 一样全是`1111`，正确答案很简单`Yes`。 任何正确的方法都必须明确接受这一点，而不尝试不必要的转换。 

另一个例子是`10`。 对位置 1 和 2 进行一次操作即可翻转这两个位，从而翻转`10`进入`01`，然后没有进一步的移动可以修复它。 所以答案是`No`。 天真的贪婪方法可能会错误地假设本地修复总是可能的。 

一个稍微长一点的例子是`001`，其中操作可以以局部推理可能误导的方式进行交互。 正确的解决方案必须捕获全局不变量，而不是依赖局部改进。 

## 方法

 思考这个问题的一种强力方法是将每个二进制字符串视为图中的一个状态。 每个状态通过对任何索引 i 应用有效操作、翻转位置 i 和 i+1 来连接到其他状态。 由于有 n 位，因此每个状态有 n−1 种可能的移动。 

状态总数最多为 2^20，即一百万左右。 在这个状态空间上的 BFS 或 DFS 总是会发现所有状态是否可达。 每次转换都需要 O(n) 时间来构造或复制字符串，因此最坏情况的总复杂度约为 O(n · 2^n)，这是边界，但在严格限制下仍然可以接受。 

然而，这忽略了结构。 每个操作恰好翻转两位，这意味着它保留了零数量的奇偶校验。 如果我们更仔细地观察，每次移动都会根据本地配置将零的总数改变 0、2 或 -2，但更重要的是，当正确地将其视为 GF(2) 上的线性系统时，它保留了模 2 的位和的奇偶校验。 

这个问题在 GF(2) 上自然是线性的。 每个操作对应于将一个向量与位置 i 和 i+1 处的 1 相加。 我们被问到是否可以使用这些基本操作从初始向量获得目标向量（全一）。 这成为链结构上的线性方程组。 

一个关键的简化来自于从左到右贪婪地消除变量。 由于每个操作仅影响相邻位置，因此我们可以将问题视为强制沿数组保持一致性：一旦我们决定如何固定位置 i，它就会限制位置 i+1。 

这导致了一个简单的确定性模拟：我们贪婪地强制第一位变为 1，并向前传播必要的翻转。 由于每次移动仅影响相邻对，因此在给定先前决策的情况下，只有一种方法可以固定每个位置。 

蛮力探索所有状态，但该结构将系统压缩为单次通过检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力 BFS | O(n·2^n) | O(n·2^n) | O(2^n) | O(2^n) | 太慢了 |
 | 贪婪传播 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理字符串并模拟强制决策。 

1. 从第一个位置开始。 如果它已经是 1，我们什么都不做。 如果它是 0，修复它的唯一方法是在位置 0 和 1 上应用操作，这会翻转两者。 
2. 当我们在位置 i 处应用翻转时，我们同时切换 s[i] 和 s[i+1]。 通过就地修改字符串或跟踪操作的奇偶校验来隐式记录此操作。 
3.移动到位置i+1并重复相同的逻辑。 在每一步中，我们通过可能将当前位置与其右邻居一起翻转来确保当前位置变为 1。 
4. 继续这个过程直到位置n−2。 那时，所有之前的位置都保证为 1。 
5. 最后，检查最后的位置。 如果为1，则可以进行变换； 否则是不可能的。 

关键的设计选择是我们永远不会重新审视之前的立场。 一旦使用涉及（i，i+1）的操作固定了位置i，以后的操作就不会再触及i，因此它的值变得稳定。 

### 为什么它有效

 该算法强制执行从左到右的不变量：处理索引 i 后，所有位置 ≤ i 都等于 1。固定位置 i 的每个操作仅影响 i 和 i+1，并且后续操作不会再次修改 i，因为所有后续操作都涉及严格大于 i 的索引。

这会产生约束的强制传播。 如果在任何步骤中我们都无法使用操作来纠正位置 i 处的 0（例如在不存在伙伴的最后位置），则配置是不可能的。 否则，每个局部校正都是全局一致的，因为每个操作的效果仅限于大小为 2 的滑动窗口。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = list(input().strip())
    n = len(s)

    for i in range(n - 1):
        if s[i] == '0':
            # flip i and i+1
            s[i] = '1'
            s[i + 1] = '1' if s[i + 1] == '0' else '0'

    print("Yes" if s[-1] == '1' else "No")

if __name__ == "__main__":
    solve()
```该实现直接编码了贪婪传播思想。 该字符串被转换为可变列表，以便可以在恒定时间内应用翻转。 该循环最多只能运行 n−2，因为只有具有右邻居的位置才能直接固定。 

关键的微妙之处在于我们从不尝试直接修复最后一个位置。 相反，它是由早期的强制操作隐式确定的。

这些痕迹显示了早期强制决策如何传播以及最终位置如何充当一致性检查。

 ## 边缘情况

 单字符案例`1`由于不需要任何操作，因此行为很简单。 该算法将完全跳过循环并直接输出`Yes`。 

案例`0`展示了为什么邻接是必要的。 没有有效的操作，因此最后一个字符检查立即失败，因为它仍然存在`0`。 

为了`10`，第一个位置是`1`，因此索引 0 处不会触发翻转。第二个位置保持不变`0`，并且由于没有伙伴就无法纠正，因此算法正确输出`No`。 

为了`001`，第一个索引触发翻转产生`110`。 此后，无法进行进一步的纠正，并且最终状态未通过最后位置检查。 这表明早期的贪婪修复仍然会导致最终不可避免的不一致，这正是算法检测到的。
