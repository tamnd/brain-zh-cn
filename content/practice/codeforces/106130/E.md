 ## 解决方案
 ## 问题理解

 我们有两个按降序排列的马速度数组。 一份属于田忌，一份属于国王。 一场比赛是由双方配对一匹未使用的马组成，结果取决于严格的比较：只有当他选择的马严格比对手的马快时，田忌才能赚钱，在这种情况下，奖励等于对手的速度。 

我们的目标不是最大化胜利，而是最大化总奖励价值，这意味着我们更愿意尽可能赢得高价值的对手，因为击败更强的对手比击败弱的对手能获得更多的钱。 

两个数组都已按非递增顺序排序，这强烈限制了任何最佳策略的结构。 然而，配对并不是固定的，因此主要的困难是决定哪一个比赛子集应该用于盈利以及如何最佳地分配马匹。 

这些限制允许每侧最多 500,000 匹马，因此任何尝试所有配对或使用二次匹配的解决方案都是立即不可能的。 甚至需要 O(n log n) 或 O(n + m) 方法以及仔细的贪婪逻辑。 由于时间限制，任何涉及重新计算所有配对或回溯选择的策略都将失败。 

当许多马具有相同的速度时，就会出现微妙的边缘情况。 由于获胜需要严格的不平等，因此相同的速度就像失败一样，如果处理不当，可能会阻碍更好的比赛。 例如，将稍强的马与同等的对手配对会浪费其他地方的潜在胜利。 

## 方法

 暴力解释是考虑所有配对马匹的方法，并总结获胜配对的奖励。 从概念上讲，对于田忌这边的每一匹马，我们可以选择任何未使用的对手马，并递归地探索所有可能性。 这是正确的，但立即组合爆炸：匹配的数量是马数量的阶乘，甚至修剪也无济于事，因为每个配对决策都会改变未来的可用性。 

关键的观察结果是，奖励结构在对手的实力方面是单调的。 战胜较大的双子总是能获得至少与战胜较小的双子一样多的奖励，而无论配对选择如何，失败的收益都为零。 这表明，最佳比赛必须仔细分配田忌的更强马匹，以击败他们实际上可以击败的最强对手，但又不能将它们浪费在他们以后可以更有效地击败的对手上。 

这将问题转化为两个排序序列之间的贪婪匹配过程，在该过程中，我们尽可能匹配双方的大值，但有时会故意牺牲弱马以保留更强的马以获得更好的机会。 

表达这一点的标准方法是两点贪婪策略：我们比较剩余的最强马匹，并决定是否进行有利可图的匹配或丢弃无用的配对以重新定位剩余的候选者。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(n + m) | 太慢了 |
 | 贪心二指针 | O(n + m) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护两个指针，一个用于田忌的马，一个用于国王的马，两者都从最强的一端开始，因为数组是按降序排序的。

1. 将田忌数组的指针 i 初始化为 0，将国王数组的指针 j 初始化为 0，并将总奖励变量初始化为零。 
2. 当两个指针都在界限内时，比较 ai 和 bj。 
3. 如果 ai > bj，我们在这两匹马之间进行一场获胜比赛。 我们将 bj 添加到答案中，因为这是获胜的奖励，然后将两个指针向前移动。 这是最优的，因为使用最强的可用田忌马来击败最强的可能击败的对手可以最大化当前的收益，而不会损害未来的选择。 
4. 如果 ai ≤ bj，田忌无法通过 ai 对抗 bj 从该配对中赚钱。 然而，我们不是在 bj 上浪费 ai，而是通过向前移动 King 的指针来丢弃 bj。 这是一个战略性的移除：bj目前对于ai来说太强了，所以它也会阻挡较弱的田忌马，移除它可能会暴露出更合适的对手。 
5. 继续直至一侧耗尽。 

### 为什么它有效

 在任何时候，该算法都保持不变，即所有先前考虑的剩余对手如果被跳过，都严格强于或等于当前的田忌候选者，这意味着他们无法与该候选者或任何较弱的候选者进行有利可图的匹配。 只要有可能获胜，将当前最强的可行比赛配对是安全的，因为延迟获胜只会减少双方的可用选项，而不会增加奖励。 每当不可能获胜时，移除对手方可以减少未来的阻挡，而不会损失潜在的利润，因为在那个阶段无论如何都没有剩余的田忌马可以击败对手。 

这种贪婪的消除确保了每一个有利可图的匹配都在可用时准确地进行，并且不会为了更糟糕的结构性结果而跳过有利可图的配对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    i = 0
    j = 0
    ans = 0
    
    while i < n and j < m:
        if a[i] > b[j]:
            ans += b[j]
            i += 1
            j += 1
        else:
            j += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案完全依赖于线性扫描。 决策点是严格的不平等检查。 当田忌获胜时，由于两匹马都被消耗，所以两支指针都前进。 当他输了或平局时，只有国王的指针前进，有效地丢弃对手的一匹无用的马。 

一个常见的实施错误是在失败的情况下反转指针移动，这会错误地消耗田忌马并减少潜在的比赛。 另一个微妙的问题是忘记平等是一种损失，这必须与“无法获胜”分支的行为相同。

 ## 边缘情况

 当所有值都相等时，每次比较都会失败。 例如，`a = [5,5,5]`,`b = [5,5,5]`。 该算法不断地从对手一方丢弃，从不消耗田忌的马匹，并以零奖励结束。 这证实了损失相等规则的正确处理。 

当田忌整体严格更强时，比如`a = [10,9,8]`,`b = [7,6,5]`，每次比较都会产生胜利，因此两个指针同步前进，并且所有奖励都会累积。 该算法绝不会不必要地丢弃，因为不会发生阻塞。 

当对手强马多而田忌弱马少时，比如`a = [3,2,1]`,`b = [10,9,8,7,6]`，该算法反复丢弃对手的马，直到耗尽。 田忌从来不会在一场无法获胜的比赛中浪费一匹马，这对于正确性至关重要，因为过早的消耗会减少已经为零的增益，但仍然会扭曲后面状态中的指针对齐。
