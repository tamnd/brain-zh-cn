 ## 解决方案
 ## 问题理解

 我们得到一个小写字符串。 在游戏开始之前，我们可以按照我们喜欢的任何顺序重新排列角色。 

单次消除操作选择一个由相同字符组成的连续块，并且该块的长度必须至少为 3。整个块消失，字符串的其余部分被连接起来。 我们最多可以表演`k`此类操作。 

任务是找到字符串的最小可能最终长度。 

关键的细节是在发生任何消除之前可以自由地重新排列字符串。 一旦顺序不再固定，问题就不再是原始字符串内的位置了。 唯一重要的是每个角色出现的次数。 

字符串长度可以大到`10^5`，因此任何试图模拟字符串本身的重新排列或消除序列的算法都是不必要的。 字母表仅包含 26 个小写字母，这强烈表明解决方案应该基于字符频率而不是字符串结构。 

一些边缘情况很容易被误判。 

考虑：```
n = 2, k = 100
s = aa
```计数为`'a'`只有 2。无法形成长度至少为 3 的块，所以答案是`2`， 不是`0`。 

考虑：```
n = 6, k = 1
s = aaabbb
```粗心的做法可能会认为重排后只能去掉两组中的一组，这是正确的。 我们删除其中一个`"aaa"`或者`"bbb"`, 剩余长度`3`。 

考虑：```
n = 7, k = 5
s = aaaaabc
```虽然`k`很大，性格`'a'`可以在一次操作中完全消除，因为所有五个副本都可以放入一个块中`"aaaaa"`。 答案是`2`，而不是需要多次操作的东西。 

## 方法

 蛮力思维是考虑所有可能的重新排列和所有可能的消除顺序。 即使对于中等长度的字符串，排列的数量也是天文数字，并且每次消除后字符串都会再次发生变化。 这是完全不可行的。 

关键的观察是重新排列消除了所有位置限制。 

假设出现一个字符`c`次。 

如果`c < 3`，该角色永远无法形成可移动的块。 无论我们如何重新排列字符串，至少那些`c`字符必须保留。 

如果`c ≥ 3`，我们可以将所有副本放在一起放入一个连续的块中，并在单个操作中删除整个块。 对同一个字符进行多次操作永远不会有任何好处，因为一次操作已经删除了该字符的所有出现。 

这将问题转化为简单的资源分配问题。 

每个出现频率至少为 3 的字符都会成为一个项目：

 - 成本：1 次操作。 
- 增益：其频率。 

我们的预算最多为`k`操作并希望最大化删除的字符数。 最佳策略是首先删除频率最大的字符。 

由于只有26个小写字母，所以我们只需要统计频率，收集所有频率至少为3的，按降序排序，取最大的`k`。

 | --- | --- | --- | --- |
 | 蛮力 | 指数| 指数| 太慢了 |
 | 最佳| O(n + 26 log 26) | O(n + 26 log 26) | O(26) | 已接受 |

 ## 算法演练

 1. 统计每个小写字母出现的频率。 
2. 对于每个频率`c`，检查是否`c ≥ 3`。 

小于 3 的频率永远无法被消除，因此它对可移除池没有任何贡献。 
3. 将每个可移动频率存储在列表中。 

每个这样的频率代表一个可以通过一次操作完全删除的字符。 
4. 按降序对列表进行排序。 

由于每个可移动字符都花费相同数量的操作，即一次，因此我们应该首先在最大频率上进行操作。 
5. 采取第一个`min(k, len(list))`频率并对它们求和。 

这是可以删除的最大字符数。 
6. 输出：```
n - removed_characters
```### 为什么它有效

 重新排列后，出现的相同字符始终可以组合在一起。 

对于出现的角色`c ≥ 3`次，连续放置所有副本会创建一个可移动的长度块`c`，因此可以通过一次操作删除整个字符类。 对于出现次数少于3次的字符，根本不能形成可移动的块。 

因此，每个角色都独立地分为两类之一：完全可移动且成本为 1，或者不可移动。 问题变成最多选择`k`总频率尽可能大的可移除字符。 由于所有成本都是相等的，因此选择最大频率是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = input().strip()

    freq = [0] * 26
    for ch in s:
        freq[ord(ch) - ord('a')] += 1

    removable = [c for c in freq if c >= 3]
    removable.sort(reverse=True)

    removed = sum(removable[:k])
    print(n - removed)

solve()
```第一部分计算字符频率。 

名单`removable`恰好包含那些可以在一次操作中删除整个出现集的字符。 低于 3 的频率将被忽略，因为它们永远无法参与有效的消除。 

按降序排序后，第一个`k`条目对应于最有利可图的业务。 将它们相加即可得到删除字符的最大数量。 

最终的答案是原始长度减去删除的字符数。 

一个微妙的点是`k`可能比可移动字符的数量大得多。 Python 切片自然地处理这个问题，所以`removable[:k]`即使当`k`超过列表长度。

 ## 边缘情况

 考虑：```
2 100
aa
```的频率`'a'`是 2。可移除列表为空，因为 2 小于 3。```
removable = []
removed = 0
answer = 2
```即使无限的运营预算也无济于事，因为无法创建有效的区块。 

考虑：```
6 1
aaabbb
```频率是`[3, 3]`。 

排序后：```
[3, 3]
```只有一个操作可用，因此我们采用第一个频率：```
removed = 3
answer = 6 - 3 = 3
```这证实了重要的是操作次数，而不仅仅是可移动性。 

考虑：```
7 5
aaaaabc
```频率是：```
a -> 5
b -> 1
c -> 1
```可删除的列表是：```
[5]
```尽管有五种操作可用，但只有一种可移动的字符类型。```
removed = 5
answer = 2
```该算法正确地将频率 5 视为单个可移动块，而不需要多次操作。
