 ## 解决方案
 ## 问题理解

 我们给出一个概率值`p`，描述一次独立尝试获得所需结果的成功率。 一名学生恰好有 3 次独立尝试。 每次尝试要么成功的概率`p`或有概率失败`1 - p`。 一旦至少发生一次成功，该过程就不再相关，因为这已经保证了总体成功。 

任务是计算三次尝试中至少一次成功的概率。 

输入是介于之间的浮点数`0`和`1`。 输出是一个浮点数，表示三次尝试内成功的概率，容差为`1e-4`相对误差或绝对误差。 

尽管声明很短，但关键的模型选择是认识到这是一个经典的“重复独立伯努利试验中至少取得一次成功”的问题。 

不涉及大的约束，因此性能无关紧要，但数值稳定性稍微重要，因为我们正在处理浮点运算。 唯一微妙的边缘情况出现在边界处`p = 0`和`p = 1`。 

如果`p = 0`，每次尝试都会失败，所以答案一定是`0`。 像这样的简单计算`1 - (1 - p)^3`在这里仍然有效，但前提是浮点精度不会引入负零或微小噪声。 

如果`p = 1`，每次尝试都会成功，所以答案正是`1`。 

一个常见的错误是尝试手动枚举序列，并意外地重复计算发生多次成功的结果。 除非仔细构建，否则这会导致不正确的组合推理。 

## 方法

 强力解释将明确列举三个独立试验的所有可能结果。 每次试验都有两种状态，成功或失败，因此有`2^3 = 8`总结果模式。 我们可以计算每种模式的概率，并对至少包含一次成功的模式求和。 

这种方法是正确的，因为它直接遵循独立事件概率的定义。 然而，如果试验数量增加，它会变得不必要的冗长并且扩展性很差，因为`n`试验需要`2^n`组合。 

关键的观察结果是，通过其补语来计算“至少一次成功”会更容易。 我们不是对所有有利的情况求和，而是计算所有尝试失败的概率并从中减去`1`。 

单次尝试失败的概率为`(1 - p)`。 由于尝试是独立的，因此三个尝试都失败的概率为`(1 - p)^3`。 因此，期望的概率为：`1 - (1 - p)^3`这将问题从枚举结果简化为单个算术表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| O(2^3) | O(2^3) | O(1) | O(1) | 有效但没有必要 |
 | 补体公式 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取概率`p`从输入。 该值表示每次尝试的成功概率。 
2. 计算失效概率`q = 1 - p`。 这将问题转化为对单个“坏事件”而不是“好事件”的推理。 
3. 计算所有 3 次尝试都失败的概率为`q^3`。 独立性允许尝试的倍增。 
4. 减去该值`1`以获得至少一次尝试成功的概率。 
5. 将结果打印为浮点数。 

唯一真正的计算是恒定数量的算术运算，因此实现很简单。 

### 为什么它有效

 “至少一次成功”和“全部失败”事件是互补且互斥的，覆盖了整个概率空间。 由于独立性允许将“全部失败”事件精确计算为相同概率的乘积，因此补码给出了精确的目标，而无需超出浮点运算的近似值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

p = float(input().strip())

q = 1.0 - p
ans = 1.0 - q * q * q

print(ans)
```该代码将概率读取为浮点数，计算补码失败概率，并通过重复乘法将其提高到三次方。 这避免了调用的任何潜在开销`pow`并使计算保持明确。 

减去`1.0`最后完成以最小化浮点错误累积。 由于该表达式仅涉及少量运算，因此精度完全在所需的公差范围内。

 ## 边缘情况

 对于`p = 0`，我们得到`q = 1`， 所以`q^3 = 1`结果是`0`。 该算法正确地捕捉到每次尝试都会失败，因此不可能成功。 

为了`p = 1`，我们得到`q = 0`， 所以`q^3 = 0`结果是`1`。 这反映了第一次尝试就一定会成功，而后续的尝试则无关紧要。 

对于接近的极端浮点输入`0`或者`1`，减法保持稳定，因为该表达式仅涉及一次减法和两次乘法，保持在所需的误差容差范围内。
