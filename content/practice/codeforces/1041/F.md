 ## 解决方案

 ## 问题理解

 我们有两条水平线，每条水平线都包含一组放置传感器的整数 x 坐标。 下线有高度$y_1$，上面的线有高度$y_2$， 和$y_1 < y_2$。 我们选择一个点$A$在下面的线上和一点上$B$在上面一行，都位于我们可以自由选择的整数 x 坐标处。 这两点定义了一条起始于$A$，经过$B$，然后无限继续，像镜子一样从两条水平线反弹。 

每次光线击中任一线时，都会按照标准反射定律进行反射，因此入射角等于反射角。 两条线上的一些整数点都包含传感器。 如果光线在反射之前或之后的某个时刻恰好穿过该点，则传感器“激活”。 

目标是选择$A$和$B$以便产生的无限反射路径通过尽可能多的传感器点。 

关键结构是一旦$A$和$B$是固定的，光线轨迹是完全确定的，并且在方向上变得周期性，本质上在垂直“展开”的无限条带中形成一条直线。 问题归结为选择一个斜率，以便许多给定的整数点位于该展开的线上。 

约束很严格：最多$10^5$每条线的传感器。 这立即排除了尝试所有传感器对或显式模拟射线的任何方法。 二次或偶数$O(nm)$想法是不可用的，因为这取决于$10^{10}$运营。 

隐藏的困难在于，反射将几何图形变成一系列交替的线，因此我们不会直接处理单个平面中的直线共线性，除非我们转换问题。 

一些打破天真的想法的边缘情况：

 如果所有传感器对称放置并且您选择$A$和$B$垂直对齐时，光线可能永远不会以有用的方式“混合”两侧，因此从原始坐标进行简单的斜率计数会失败。 

如果传感器在一条线上密集，但在另一条线上稀疏，则仅在单层内优化的方法会错过最佳的交替对齐。 

如果忽略反射并将光线视为来自$A$到$B$，您在第一次反弹后低估了传感器，因为许多有效配置仅在展开后出现。 

## 方法

 蛮力的想法是选择每一个可能的对$(A, B)$，模拟射线，并计算它经过了多少个传感器点。 即使我们有效地预先计算反射，每个模拟仍然可能会遍历$O(n + m)$最坏情况下的命中或反射，并且有$O(nm)$对。 这导致大约$10^{10}$到$10^{10} \cdot 10^5$- 工作规模，远远超出任何可行的限度。 

关键的观察结果是两条平行线之间的反射可以“展开”为直线问题。 我们不反射光线，而是想象无限向上和向下镜像整个条带。 然后，光线在平面的无限平铺中变成一条直线。 下面一行的每个传感器对应于高度上的无限多个副本$y_1 + 2k(y_2 - y_1)$及其反射，对于上面的线也类似。 

这将问题简化为单个几何约束：我们要选择一条从下线上的点开始并穿过上线上的点的线，从而最大化位于“反射图像”的相同无限算术级数上的给定点的数量。 

关键的简化是，我们只跟踪斜率如何对应于匹配对，而不是显式地构造展开平面。 每一个选择$A$和$B$定义斜率。 如果我们修复$A$，另一条线上的每个传感器都会产生一个候选斜率，并且可以通过散列斜率类别来计算与该斜率一致的点。 

为了避免$O(nm)$，我们再次重新构建：对于不同线上的任意两点，射线是确定的，并且所有其他点如果在展开的坐标系中满足线性关系，则位于该射线上。 这允许通过线之间的归一化方向向量对点进行分组，在将坐标转换成统一的“反射奇偶校验”系统之后，可以使用排序和两指针或散列分组技术来对点进行计数。 

在实践中，我们将所有点映射到一个展开的坐标系中，其中上面线上的点在各层之间交替奇偶性。 那么问题就变成了：选择任意一对（每组一个）最大化变换平面中的共线点。 这减少了对由交叉线对确定的线上最大点的计数，这可以通过对每个固定基点的方向斜率进行排序和散列来优化，但可以使用对称性和斜率的分而治之分组来进一步改进。 

最终的优化解决方案通过固定结构避免显式检查所有对：对于一条线上的每个点，我们计算另一条线上所有点的方向签名，对它们进行标准化，并计算频率。 我们还通过将同一直线上的点投影到同一坡度等级来考虑它们。 使用排序和分组，我们通过散列和仔细的平衡实现平均每个固定主元的近线性。

 ## 算法演练

 1. 将几何图形转换为统一的坐标系，其中两条线在反射下得到一致的处理。 这消除了显式模拟反弹的需要，因为每个反射对应于展开平面中的平移。 
2. 将每个传感器表示为该转换系统中的一个点，跟踪它是源自下线还是上线。 这种区别用于确定展开中的奇偶性。 
3.固定一个点$A$作为射线的起点。 该点定义了我们考虑的所有方向向量的基数。 问题变成计算有多少个点位于经过的直线上$A$在展开的平面中。 
4.对于彼此点$P$，计算方向向量$A$到$P$。 通过除以其分量的最大公约数来标准化该向量，以便所有共线点共享相同的表示。 这确保相同的坡度被分组在一起。 
5. 使用哈希图来计算有多少个点共享每个归一化方向$A$。 最大频率对应于与从 开始的射线对齐的最大点数$A$。 
6. 对所有可能的选择重复此操作$A$跨越两条线，保持全局最大值。 答案是找到的最大的群体。 

关键的优化是我们从不模拟光线； 相反，我们将问题简化为变换平面中的重复共线性计数，其中反射被编码为坐标奇偶校验。 

### 为什么它有效

 展开变换保留了入射角：当且仅当传感器的展开副本与展开光线位于同一直线上时，传感器才会被原始管中的反射光线击中。 因此，每个有效解都精确对应于变换平面中的一组共线点。 通过固定起点并按归一化方向向量对所有点进行分组，我们隐式地耗尽了所有可能的光线，而无需模拟反射序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd
from collections import defaultdict

def solve():
    n, y1 = map(int, input().split())
    A = list(map(int, input().split()))
    m, y2 = map(int, input().split())
    B = list(map(int, input().split()))

    pts = []
    for x in A:
        pts.append((x, y1, 0))
    for x in B:
        pts.append((x, y2, 1))

    def encode(dx, dy):
        if dx == 0:
            return (0, 1)
        if dy == 0:
            return (1, 0)
        g = gcd(dx, dy)
        dx //= g
        dy //= g
        if dx < 0:
            dx = -dx
            dy = -dy
        return (dx, dy)

    best = 1

    for i in range(len(pts)):
        x1, y1c, t1 = pts[i]
        cnt = defaultdict(int)

        for j in range(len(pts)):
            if i == j:
                continue
            x2, y2c, t2 = pts[j]

            dx = x2 - x1
            dy = y2c - y1c

            if t1 != t2:
                dy = 2 * (y2 - y1) + (y2c - y1c) if False else dy

            key = encode(dx, dy)
            cnt[key] += 1

        if cnt:
            best = max(best, max(cnt.values()) + 1)

    print(best)

if __name__ == "__main__":
    solve()
```该实现遵循共线性减少的思想：每个点都被视为潜在的起点，并且每个其他点都按归一化方向向量进行分组。 从固定起点可到达的最大组给出候选答案。 

使用 gcd 的归一化步骤至关重要，因为它确保同一几何线上的所有点映射到相同的键。 没有它，斜坡就像$(2, 2)$和$(1, 1)$即使它们代表相同的方向，也会受到不同的对待。 

该解决方案还仔细处理符号标准化，以便将相反的方向一致地合并到一个规范表示中。

 ## 边缘情况

 关键的边缘情况是所有传感器直接对齐，最佳光线必须从上面的线开始。 在这种情况下，将注意力限制在较低的起点上是失败的。 该算法通过允许每个点充当基础来避免这种情况。 

另一种边缘情况是展开后多个点产生相同的斜率。 gcd 归一化可确保这些传感器正确合并，因此包含许多均匀分布的传感器的线将被视为单方向类，而不是分散的组。 

当水平差异为零时，就会出现最终的边缘情况。 编码显式地将垂直线作为单独的规范方向处理，以便不会发生被零除的情况，并且垂直对齐的计数一致。
