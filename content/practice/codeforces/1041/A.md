 ## 解决方案
 ## 问题理解

 我们得到了一组在盗窃中幸存下来的键盘索引。 关键的隐藏结构是，在盗窃之前，所有键盘都形成一个连续的整数块，例如 x、x+1、x+2 等，直到某个未知长度。 盗窃后，其中一些连续整数丢失了，我们留下了原始连续段的无序子集。 

我们的任务是确定可能被盗的键盘的最小数量，假设我们可以自由选择起始值 x 和完整连续段的原始长度。 

我们真正要做的是将给定的数字嵌入到连续值的最小长度整数区间中。 一旦该间隔被固定，其中不属于给定集合的每个整数必定已被窃取。 因此，最小化被盗键盘相当于最小化有多少个整数位于所选区间内但输入中不存在。 

约束 n ≤ 1000 意味着我们可以轻松进行 O(n^2) 推理。 任何对元素之间的所有间隙进行排序和检查的解决方案都已经在限制范围内。 任何立方或更糟的东西都是不必要的。 

一个微妙的边缘情况来自于起点 x 不固定的事实。 例如，如果最小观测值是 8，则真实序列可能从 1、8 甚至 10 开始，具体取决于我们如何解释缺失值。 一个天真的错误是假设序列必须从 min(a) 开始，这忽略了原始连续块进一步向左延伸的可能性。 

另一种失败模式是忽略内部间隙。 例如，如果剩余的键盘是1、10、11、12，仅看极端是不够的； 1 和 10 之间的间隙意味着缺少元素 2 到 9，这些元素是任何一致重建中同一原始连续块的一部分。 

## 方法

 蛮力的想法是尝试每个可能的起点 x 和每个可能的终点 y，使得所有给定的数字都位于 [x, y] 中。 对于每个候选区间，我们检查该区间缺少多少个整数。 这是正确的，因为它直接模拟了未知的原始存储配置。 然而，区间边界的范围可达 10^9，并且尝试所有可能性将需要在一个巨大的范围内进行迭代，这使得它不可行。 

关键的观察结果是，最佳间隔必须紧紧包裹现有的数字，并且两端没有不必要的松弛。 如果我们通过对剩余键盘进行排序来固定顺序，则任何有效的原始序列都必须覆盖从最小观察值到最大观察值的范围，但仅在需要减少不一致的地方向内扩展间隔也可能是最佳的。 然而，由于扩展间隔只会增加潜在缺失元素的数量，因此最好的策略是将间隔固定为包含所有给定值的最小连续块。 

在该区间内，缺失键盘的数量只是区间长度与观察到的元素数量之间的差值。 区间长度为 max(a) - min(a) + 1，我们减去 n。 

因此，问题简化为排序（或只是扫描最小值和最大值）和计算简单的算术表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 间隔暴力破解 | O(range · n) 或更差 | O(1) | O(1) | 太慢了|
 | 排序 + 计算跨度 | O(n log n) | O(n log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 读取所有剩余的键盘索引并将它们存储在列表中。 这些代表未知连续区间内的点。 
2. 找出其中的最小值和最大值。 这两个值确定可以包含所有剩余键盘的最小间隔，因为任何有效的原始连续块必须至少跨越从最小到最大观察到的索引。 
3. 使用 max_value - min_value + 1 计算此区间内的整数总数。如果不考虑缺失元素的约束，这表示完整的候选原始商店库存。 
4. 从该间隔长度中减去观察到的键盘数量n。 剩下的是原始块中必须存在但不存在于最终集合中的整数，这与被盗的键盘完全对应。 

### 为什么它有效

 任何有效的原始配置都是连续的整数序列，必须包含所有观察到的值。 因此，它的端点必须最多为 min(a) 且至少为 max(a)，否则某些观测值将位于序列之外。 超出这些范围的扩展只会增加隐含缺失元素的数量，而不会提高可行性。 因此，最小可能的原始序列恰好是区间 [min(a), max(a)]，并且该区间中每个不存在的整数都对应于被盗的键盘。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    mn = min(a)
    mx = max(a)
    
    print((mx - mn + 1) - n)

if __name__ == "__main__":
    solve()
```该解决方案以线性时间读取输入并仅计算两个聚合统计数据：最小值和最大值。 减法`(mx - mn + 1) - n`直接计算覆盖所有剩余键盘的最小可能连续间隔内缺失的整数。 

一个常见的实施错误是排序，然后尝试单独计算间隙，这在这里是不必要的，并且会增加差一错误的风险。 另一个错误是忘记了`+1`计算间隔长度时，因为两个端点都包含在内。

 ## 边缘情况

 对于单个键盘，比如输入法`1 / 100`，算法计算 min = max = 100，区间长度 1，结果 0。这证实了模型正确处理退化区间。 

对于完全连续的集合，例如`1 2 3 4`，min 为 1，max 为 4，给出间隔长度 4 和结果 0。这可确保在没有丢失元素时不会出现误报。 

对于大间隙，例如`1 100 101`，min = 1 且 max = 101 给出间隔长度 101，减去 3 得到 98 个缺失键盘。 这直接演示了该方法如何解释边界区间内所有未见过的整数，无论​​分布如何。
