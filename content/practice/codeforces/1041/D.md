 ## 解决方案

 ## 问题理解

 当飞机固定在高度 h 时，滑翔机在某个整数 x 坐标处从移动的飞机上释放。 一旦释放，滑翔机总是每秒向右移动一个单位。 同时，它的高度每秒减少一个单位，因此它本质上是沿着对角线下降的。 

地面上有特殊的水平间隔，存在向上的气流。 当滑翔机高于这些间隔之一时，它会在那一秒停止失去高度，同时仍继续以相同的速度向右移动。 在这些间隔之外，它会正常下降。 一旦高度达到零，滑翔机就会停止。 

任务是选择沿飞行路径的整数起始 x 位置，以使着陆前行进的总水平距离最大化。 

重要的相互作用是，每个空中间隔都会有效地“暂停”下降，延长滑翔机在空中停留的时间。 由于一旦起始点固定，运动就是确定性的，因此问题就变成了选择最佳起始位置，以便轨迹与尽可能多的有效“节省高度”的机会重叠。 

约束足够大，以至于不可能对每个起点进行任何模拟。 由于分段数量最多为 2 × 10^5，坐标最多为 10^9，任何重新计算每个候选起点的完整飞行的解决方案都会大大超出时间限制。 由于扫描间隔，即使单个模拟也是 O(n) 或更糟，因此尝试所有启动都是不可行的。 

主要的边缘情况源自间隔边缘处的边界行为。 如果滑翔机恰好降落在间隔结束的点，它就不再从中受益，因此逐一处理很重要。 另一个微妙的情况是当间隔相距很远时：天真的贪婪假设每个间隔总是有帮助是不正确的，因为如果滑翔机下降得太早，滑翔机可能无法在着陆之前到达它。 

## 方法

 如果我们固定起始位置，滑翔机的高度每走一步就会减少一个，这意味着它正好有 h 个“自由下降单位”可供使用。 每次在某个间隔内时，下降都会暂停，从而有效地在间隔内每秒节省一个高度单位。 

所以问题就变成了：对于给定的起点，路径与并集段重叠的总秒数是多少，以及最终的 x 位置延伸了多少？ 

强力方法会尝试感兴趣范围内的每一个可能的整数起始位置，并模拟每个位置的完整飞行。 每次模拟都会扫描间隔或在间隔上维护一个指针，每次启动的成本为 O(n)。 由于起始位置的跨度高达 10^9，即使限制为区间端点也会留下 O(n) 个候选点，导致总操作量为 O(n^2)，这太慢了。 

关键的观察结果是滑翔机的轨迹以结构化的方式与间隔相互作用。 如果我们固定一个起点，则路径是 x 轴上的一条线段，其长度取决于保留的高度。 每个间隔的贡献仅取决于与该段的重叠。 

我们不是每次开始都进行模拟，而是颠倒视角。 我们认为飞行会沿 x 轴消耗“高度预算”h，但每次经过一个间隔时，我们都会有效地扩展可用预算，因为那里没有消耗高度。 这导致了对间隔加间隙联合的滑动窗口风格解释。

我们通过双重表示来压缩结构：对于每个间隔，我们计算它如何改变可达距离以及如何改变有效高度消耗。 因为间隔不重叠，所以我们可以按顺序处理它们，并保持起始点可以从链接连续间隔中“受益”的程度。 

最佳解决方案简化为跟踪，对于与间隔边界对齐的每个潜在起始位置，滑翔机可以通过贪婪地消耗间隔外的高度并跳过间隔内的消耗来传播多远。 两指针或基于前缀的 DP 保持最大可到达端点，同时根据间隔调整累积的“保存高度”。 

重要的结构见解是，答案是在区间边界或新区间开始影响轨迹的点处实现的，因此我们只评估 O(n) 候选者而不是所有整数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n² + 范围) | O(1) | O(1) | 太慢了|
 | 扫描+区间链接（两个指针/贪婪）| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 将间隔排序并解释为一个预处理结构，我们可以快速知道滑翔机从任何起点在气流中花费了多长时间。 由于间隔已经按排序顺序和不相交给出，因此我们可以直接将它们用作序列。 
2. 考虑某个区间的左边界处的起点。 从这一点开始，我们模拟滑翔机在其有效高度达到零之前走了多远，但每当我们在一个区间内时，我们都会修改高度消耗。 
3. 在间隔上保持指针并跟踪两个量：当前可到达的 x 坐标和剩余的有效高度预算。 
4.当前位置在所有区间之外时，向右移动每步消耗一个单位高度，因此可到达的距离受到剩余高度的限制。 
5. 当我们进入一个区间时，我们立即停止降低高度，直到退出它。 这意味着我们可以“跳过”该段长度的消耗高度，从而有效地将最终到达范围延长了间隔长度减去该跨度中可能损失的长度。 
6. 继续按顺序移动间隔，将其贡献合并到总可达距离中，直到高度预算用完。 
7. 尝试将每个间隔起点作为候选起始位置，并使用上述过程计算最远可到达端点，更新全局最大值。 

这样做的原因是最优解总是从区间边界开始。 间隙内的任何起点都可以转移到下一个间隔起点，而不会减少可达距离，因为较早的起点只会增加不必要的下降，而不会获得额外的间隔覆盖范围。 这创建了一个单调结构，其中间隔开始足以代表所有可能的开始。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, h = map(int, input().split())
    seg = [tuple(map(int, input().split())) for _ in range(n)]

    ans = 0
    j = 0

    for i in range(n):
        start = seg[i][0]
        height = h
        pos = start

        j = i

        while j < n:
            l, r = seg[j]

            if pos < l:
                dist = l - pos
                if height <= dist:
                    ans = max(ans, pos + height)
                    break
                height -= dist
                pos = l

            if pos <= r:
                pos = r
                j += 1
            else:
                j += 1

        else:
            ans = max(ans, pos + height)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码在每个间隔上迭代作为潜在的起始锚点。 对于每次启动，它都会模拟向前移动，同时跟踪剩余高度。 当位置在区间外时，高度根据间隙长度而减小。 当位于某个区间内时，指针会跳过该区间而不降低高度，从而有效地将整个段视为自由移动。 嵌套指针确保每个间隔每次启动最多被访问一次。 

一个微妙的点是间隙中高度耗尽时的中断条件。 此时，滑翔机降落在间隙内，因此最终位置直接计算为 pos + height，而不尝试处理进一步的间隔。

 ## 边缘情况

 第一种边缘情况是滑翔机在某个区间内启动。 在这种情况下，它会立即受益于零高度损失，直到退出区间。 该算法处理这个问题是因为它检查`pos <= r`并直接跳到区间终点，不消耗高度。 

当滑翔机恰好在间隔边界处超出高度时，会发生另一种边缘情况。 如果高度恰好在位置 l 处变为零，则最终答案将计算为 pos + height，它恰好落在正确的 x 坐标处，而不会错误地进入下一个区间。 

最后的边缘情况是由于间隙耗尽而在某个点之后没有可到达的间隔。 中断条件确保一旦高度不足以到达下一个间隔，模拟就会立即终止，从而防止错误地包含后面的段。
