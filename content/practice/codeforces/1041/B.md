 ## 解决方案

 ## 问题理解

 我们正在寻找在两个同时约束下可能有多少种屏幕尺寸。 一、电视的宽度不能超过限制`a`，并且高度不得超过限制`b`。 其次，屏幕必须遵循固定的宽高比`x : y`，这意味着每个有效的屏幕都必须是该比例的整数缩放。 

因此，任何有效的屏幕都具有以下形式`(w, h) = (kx, ky)`对于某个正整数`k`。 任务是计算有多少个值`k`两者都保留`kx ≤ a`和`ky ≤ b`。 

因此，问题就简化为找出一个边界框内可以容纳多少个基本矩形的整数倍。 

约束最大为 10^18，这会立即排除迭代所有可能的宽度或高度。 即使是线性扫描也是不可能的。 任何有效的解决方案都必须将问题减少到恒定数量的算术运算。 

一个幼稚的错误是尝试检查所有`w`最多`a`并验证是否`w * y % x == 0`，然后计算`h = w * y / x`。 这太慢了，如果不小心处理乘法，还有溢出的风险。 

另一个微妙的问题是整数除法截断。 如果我们计算`a // x`或者`b // y`如果以派生形式不正确，我们可能会错误地计算一个约束比另一个约束更严格的边界情况。 

核心困难是认识到我们正在计算有效的缩放因子，而不是宽度和高度的独立选择。 

## 方法

 暴力解释会尝试每个可能的整数对`(w, h)`这样`w ≤ a`和`h ≤ b`，然后检查是否`w / h = x / y`。 对于每一对，我们将使用交叉乘法验证比率条件`w * y == h * x`。 

这种方法是正确的，但成本极高。 对的数量是`a × b`，在最坏的情况下是`10^36`。 甚至概念迭代也是不可能的。 

关键的观察结果是，比率条件强制所有有效对位于缩放矩形的单个算术级数上。 一旦我们修好`(x, y)`，每个有效屏幕都由缩放因子唯一确定`k`。 因此，我们不需要在 2D 网格上搜索，而只需要查找有多少个整数`k`满足两个界限。 

从`(kx ≤ a)`我们得到`k ≤ a / x`，并从`(ky ≤ b)`我们得到`k ≤ b / y`。 两者必须同时成立，因此限制因素是这两个上限中的最小值。 

因此答案很简单`min(a // x, b // y)`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 (w, h) | O(a·b) | O(a·b) | O(1) | O(1) | 太慢了|
 | 最佳缩放因子 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 将长宽比约束解释为缩放规则。 任何有效的屏幕都必须是`(w, h) = (k·x, k·y)`对于某个整数`k ≥ 1`。 这消除了宽度和高度的独立选择。 
2. 将宽度约束转化为不等式`k`。 自从`kx ≤ a`，我们推导出`k ≤ a // x`。 整数除法是安全的，因为我们只计算完整的有效缩放。 
3. 将高度约束转化为另一个不等式`k`。 自从`ky ≤ b`，我们推导出`k ≤ b // y`。 
4. 结合两个约束。 相同`k`必须满足两个上限，因此最大有效`k`是`min(a // x, b // y)`。 
5. 计算有效的解决方案。 每个整数`k`从`1`达到这个最大值正好产生一对有效的，所以答案就是那个最大值。 

### 为什么它有效

 每个有效的对必须精确满足比率，这迫使宽度和高度之间成比例。 这将搜索空间从二维网格折叠为单个参数`k`。 的限制`w`和`h`成为独立的上限`k`，并且这些约束的交集恰好是两个导出限制的最小值。 没有遗漏任何有效的对，因为每个可行的屏幕都对应一个唯一的整数`k`，并且不包含无效对，因为任何`k`超出任一界限都会违反至少一个约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, x, y = map(int, input().split())
    print(min(a // x, b // y))

if __name__ == "__main__":
    solve()
```该解决方案直接应用从几何到算术的导出变换。 关键的实现细节是使用整数除法，它会自动降低最大可行缩放因子。 不需要循环或进一步验证，因为比率约束已经通过将所有解决方案表示为倍数而在结构上强制执行`(x, y)`。

 ## 边缘情况

 当`a < x`或者`b < y`，该公式立即得出零，因为至少有一个除法变为零。 例如，输入`a = 4, b = 100, x = 5, y = 1`给出`4 // 5 = 0`，因此不存在有效的电视。 该算法自然地处理这个问题，无需特殊条件。 

当两个约束都很大但不均匀时，例如`a = 10^18, b = 10^18, x = 1, y = 10^18`，宽度允许多个值`k`，但高度将其限制为恰好 1。 计算`min(a // 1, b // 10^18)`正确地产生`1`，证明更紧密的维度总是通过最小运算占据主导地位。
