 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，并且允许我们使用第二个二进制字符串来修改它，该字符串具有完全相同的内容$m$那些。 第二个字符串有 1 的每个位置都会翻转原始字符串的相应位。 全部翻转后，我们得到一个新的字符串$T$。 对于每一个$m$，我们希望以最大化有多少个相邻对的方式来选择翻转位置$T$等于$11$。 

换句话说，我们可以精确切换$m$原始字符串的位置，切换后我们希望最大化索引的数量$i$使得两者$T_i$和$T_{i+1}$是 1. 我们必须计算每个可能的最优值$m$从 0 到$n$。 

约束非常严格：总和$n$所有测试用例最多是$10^6$。 这立即排除了每个测试用例的任何二次方。 甚至$O(n \log n)$每个测试都是临界的，因此每个测试的解决方案必须本质上是线性的，可能需要一些全局排序或前缀处理。 

一个天真的方法会尝试所有子集$m$要翻转的位置，重新计算结果字符串，并计算有效的相邻对。 这显然是不可能的，因为子集的数量是$\binom{n}{m}$，甚至评估单一配置成本$O(n)$，导致指数行为。 

一个稍微好一点但仍然不可能的想法是独立对待每个位置并为翻转它分配一个“好处”。 这会失败，因为翻转一个位置会改变其邻居的贡献，因此效果不是独立的。 

当两个相邻位置都是翻转的候选位置时，就会出现微妙的边缘情况。 例如，在这样的段中`101`，仅翻转中间位的行为与翻转两个端点的行为非常不同，因为邻接结构会非线性变化。 这种依赖性是核心困难。 

## 方法

 关键的转变是停止考虑单个翻转，而是考虑建立最终的一组位置，其中$T_i = 1$。 

各位置$i$原始字符串在最终字符串中有两种可能的状态：

 如果$S_i = 1$，我们可以将其保持为 1，而无需进行翻转。 

如果$S_i = 0$，使其变为 1 需要翻转一次。 

因此，每个位置在最终字符串中都有变为 1 的成本，并且我们选择位置的子集在预算为 1 的情况下变为 1$m$。 

现在的目标变成：选择一组$A$的位置设置为 1，最大化内部相邻对的数量$A$，受成本限制。 

这正是一个路径图，其中每个选定的顶点都对边做出贡献，并且邻居也被选定。 该值是由以下引起的边数$A$。 

我们观察局部结构，而不是推理全局子集：获得邻接的唯一方法是连接相邻的选定位置。 唯一可以改进结构的地方是原始字符串中现有 1 块之间的边界。 

在原始字符串中，所有位置$S_i = 1$已经是可供选择的“自由”候选人。 它们形成由零分隔的自然块。 重要的观察结果是，零是唯一昂贵的元素，每个零都可以被视为相邻 1 块之间的可选连接器。 

每个零都有明确定义的局部效应：

 如果未选择，则会破坏左右块之间的邻接关系。 如果选择它，它将连接它们并创建新的相邻对。 选择零的增益仅取决于原始字符串中它的直接邻居，这使得所有决策都是独立的。 

每个零都属于以下三种情况之一：

 如果两个邻居都是 1，则选择它会创建两个新的相邻对。 

如果恰好有一个邻居为 1，则选择它会创建一对新的相邻对。 

如果两个邻居都不为 1，那么选择它根本没有帮助，而且永远不是最优的。 

这减少了选择最多的问题$m$0 具有最高增益，而所有 1 总是免费获取。 一旦收益是独立的，对它们进行排序就会给出最优策略。

 ## 算法演练

 1. 扫描字符串并计算初始基线：假设我们始终保留每个`1`来自最终集中的原始字符串$A$。 这不需要任何成本，并且已经在每个连续的原始块内贡献了邻接性。 
2. 确定所有位置$S_i = 0$。 每个这样的职位都是一个潜在的翻转候选者，选择它需要花费 1 个单位的预算。 
3. 对于位置处的每个零$i$，计算其局部增益：

 如果原始字符串中两个邻居都是 1，则其增益为 2。 

如果恰好有一个邻居为 1，则其增益为 1。 

否则其增益为0。 

这是有效的，因为最初只有原始的在$A$，因此零的贡献仅取决于它是否连接现有的活动端点。 
4. 将所有积极的收获收集到一个列表中。 增益为 0 的零毫无用处，可以忽略。 
5. 按降序对增益进行排序。 此排序确保对于任何预算$m$，最好的选择是采取$m$最大的可用改进。 
6. 在排序后的增益上构建前缀和数组。 每个的答案$m$是顶部的总和$m$增益加上原始连续增益的基线邻接。 

### 为什么它有效

 关键的不变量是每个有益的零对目标都有独立的贡献。 选择零仅影响通过其直接邻居的邻接，并且因为这些邻居要么都是固定的，要么不受其他零选择的影响，所以没有两个零决策会以改变边际增益排序的方式进行干扰。 这就把问题变成了一个标准的“采取顶部$m$独立奖励”优化，其中通过减少增益进行贪婪选择是最优的。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input().strip())
        s = input().strip()

        # baseline: adjacency inside original 1-blocks
        base = 0
        for i in range(n - 1):
            if s[i] == '1' and s[i + 1] == '1':
                base += 1

        gains = []

        for i in range(n):
            if s[i] == '0':
                left = 1 if i > 0 and s[i - 1] == '1' else 0
                right = 1 if i < n - 1 and s[i + 1] == '1' else 0
                gain = left + right
                if gain > 0:
                    gains.append(gain)

        gains.sort(reverse=True)

        pref = [0]
        for g in gains:
            pref.append(pref[-1] + g)

        # output answers for all m
        res = []
        for m in range(n + 1):
            if m <= len(gains):
                res.append(str(base + pref[m]))
            else:
                res.append(str(base + pref[-1]))

        print(" ".join(res))

if __name__ == "__main__":
    solve()
```该实现将问题分为固定基线和独立改进列表。 基线计算原始邻接之间已经存在的邻接，而增益列表仅捕获通过翻转零可实现的改进。 

对增益进行排序可确保每个前缀对应于翻转预算的最佳分配。 前缀数组允许回答所有$m$排序后线性时间内的值。 

一个微妙的点是处理$m$大于有用零的数量。 在这种情况下，额外的翻转不会增加答案，因为它们对应于零增益操作。 

## 工作示例

 考虑一个结构可见的简单字符串：

 输入：```
1
5
10101
```由于不存在连续的基线，因此基线邻接值为 0。 

我们计算每零的增益：

 索引 0 是`1`， 跳过。 

索引 1 是`0`，邻居为 1 和 1，增益 = 2。 

索引 2 是`1`。 

索引 3 是`0`，邻居为 1 和 1，增益 = 2。 

索引 4 是`1`。 

因此，增益 = [2, 2]，排序后仍为 [2, 2]。 

现在我们构建前缀和：

 | 米 | 选择的收益| 总计 |
 | ---| ---| ---|
 | 0 | []| 0 |
 | 1 | [2] | 2 |
 | 2 | [2,2]| 4 |
 | 3 | [2,2]| 4 |
 | 4 | [2,2]| 4 |
 | 5 | [2,2]| 4 |

 这表明，一旦所有有用的连接器都被取出，额外的翻转并不能改善结构。 

现在考虑第二种情况：

 输入：```
1
4
1100
```基线邻接度为 1（在两个初始邻接度之间）。 

收益：

 索引 2 (`0`) 的左邻居为 1，右邻居为 0，增益 = 1。 

索引 3 (`0`) 的左邻居为 0，右邻居为 0，增益 = 0。 

所以收益 = [1]。 

前缀：

 | 米 | 价值|
 | ---| ---|
 | 0 | 1 |
 | 1 | 2 |
 | 2 | 2 |
 | 3 | 2 |
 | 4 | 2 |

 这证实了只有有用的零才会影响改进。

 ## 边缘情况

 完全为零的字符串是最微妙的情况。 每个位置都没有贡献的邻居，所以每个增益都是零。 该算法正确地生成了一个零增益列表，并且所有答案都保持为零，无论$m$，因为翻转隔离位永远不会创建相邻的 11 对。 

完整的一个字符串的行为有所不同：基线已经给出了最大邻接性$n-1$，并且没有零位置可以改进任何东西。 增益列表为空，因此所有答案保持不变，正确反映了在该模型下翻转只会破坏结构而不会创造新的收益。 

边界重的字符串，例如`10001`演示了边缘零点的作用。 中间的零有两个相邻的，贡献增益 2，而边缘零最多贡献 1 或 0。排序步骤确保始终首先选择中央连接器，这与组件的最佳合并相匹配。
