 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 一步中，我们可以选择两个不同的元素。 如果它们的总和是偶数，我们将两者都删除并将它们的总和添加回数组中，因此数组大小正好减少一。 

任务不是模拟任意选择，而是确定执行最多操作后最终能得到的最小可能数量的元素$k$此类操作。 

重要的是，每个操作都会将两个元素合并为一个，并且只有当两个所选元素具有相同的奇偶校验时才可能进行合并，因为只有这样它们的和才是偶数。 因此，每个操作本质上是将两个偶数或两个奇数组合成一个奇偶校验与同一组匹配的数字。 

输入规模较小，总共$n$超过所有测试用例$10^3$， 和$k$还高达$10^3$。 这排除了任何昂贵的每次操作或每对模拟成为瓶颈的可能性，但更重要的是，它表明答案取决于计数结构而不是构造序列。 

天真的解释会尝试重复模拟合并，但这立即引入了歧义：应该选择哪对来最小化最终大小？ 例如，合并大值或小值不会影响未来的可能性，只有奇偶校验才重要。 因此，任何跟踪实际值而不是奇偶结构的方法都有可能做不必要的工作。 

边缘情况出现在以下情况：

 例如，如果所有数字都有交替奇偶校验计数$[1,2,3,4]$，操作受到限制，因为合并仅发生在奇偶校验组内。 任何两个元素的贪婪配对而不跟踪奇偶校验组可能会错误地尝试无效合并。 

如果所有数字都具有一个奇偶校验，例如全部为偶数，则每次操作都会将数组大小恰好减少一，直到只剩下一个元素或耗尽$k$。 简单的模拟可能仍然会使这个问题变得过于复杂。 

如果正好有一个奇数，则它永远不能参与合并，因此它始终被保留，并且任何最佳策略都必须避免尝试包含它的浪费操作。 

## 方法

 蛮力的想法是模拟操作：扫描所有对，选择任何有效对，合并它们，然后重复$k$次。 这是正确的，因为每个有效的移动都会被考虑，但效率很低，因为每个操作都需要检查所有对，并且每次合并后数组仅缩小一个元素。 在最坏的情况下，这大约会导致$O(k \cdot n^2)$，当$n$和$k$都是$10^3$。 

关键的观察是，除了平等之外，价值观本身是无关紧要的。 每个操作都以可预测的方式保留奇偶校验结构：合并两个偶数给出一个偶数，合并两个奇数也给出一个偶数，但重要的是两个元素都被消耗为一个。 因此，真正重要的是每个奇偶校验类存在多少个元素。 

我们只需要计算偶数和奇数。 每个操作消耗同一奇偶校验组的两个元素，并根据解释生成该组或偶数组的一个元素，但为了最小化计数，唯一重要的事实是每个操作将总大小恰好减少一。 

然而，并非所有操作都始终可行。 只有当我们至少有某个奇偶组的两个元素时，我们才能执行移动。 因此，最大运算次数受到我们可以在偶数和奇数内部分别形成多少对的限制。 

于是答案就变成了：

 我们计算实际上有多少个操作是可能的$k$，并从中减去$n$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(k \cdot n^2)$|$O(n)$| 太慢了 |
 | 奇偶校验计数|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将问题简化为计算可以执行多少次有效合并。 

### 步骤

 1. 统计数组中有多少个偶数和多少个奇数。 

这是足够的原因是只有奇偶校验才能确定合并是否有效。 
2. 计算偶数之间可以组成多少对，即$\lfloor \frac{even}{2} \rfloor$。 

每个这样的对都可以合并一次。 
3. 计算赔率可以组成多少对，即$\lfloor \frac{odd}{2} \rfloor$。 
4. 将这些相加得到没有任何限制的可能操作总数：$maxOps = \lfloor even/2 \rfloor + \lfloor odd/2 \rfloor$。 
5. 因为我们最多被允许$k$操作，实际执行的操作数为$ops = \min(k, maxOps)$。 
6. 每一次操作都会使数组大小恰好减少一个元素，因此最终大小为$n - ops$。 

### 为什么它有效

 关键的不变量是，每个有效操作恰好消耗同一奇偶校验组中的两个元素，并将它们替换为一个元素，该元素除了减少该组中的计数之外，不会改变未来合并的可行性结构。 这意味着唯一不断变化的状态是每个奇偶校验类中的元素数量。 任何有效移动序列都可以重新排列，而不改变可能的操作数量，因为奇偶校验组内的配对选择是独立的。 所以最大化操作数相当于在每个奇偶校验类中贪婪地形成尽可能多的不相交对，直到极限$k$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))
        
        even = 0
        odd = 0
        
        for x in a:
            if x % 2 == 0:
                even += 1
            else:
                odd += 1
        
        max_ops = (even // 2) + (odd // 2)
        ops = min(k, max_ops)
        
        print(n - ops)

if __name__ == "__main__":
    solve()
```实现直接遵循推理。 我们首先通过奇偶校验来分隔元素，奇偶校验捕获与有效操作相关的所有结构。 然后我们计算每个奇偶校验组内独立合并的最大数量。 最后我们上限为$k$，因为我们不能超过允许的操作数。 

一个微妙的点是我们从不模拟合并后的数组。 这是安全的，因为每次合并都会将总计数恰好减少 1，并且不会以某种方式更改奇偶校验计数，从而使未来的配对潜力超出整数除法已计数的范围。 

## 工作示例

 ### 示例 1

 输入：```
n = 5, k = 2
a = [1, 2, 3, 4, 5]
```我们跟踪奇偶校验计数。 

| 步骤| 偶| 奇数| 最大行动 | 剩余 k | 行动|
 | --- | --- | --- | --- | --- | --- |
 | 开始| 2 | 3 | 1 + 1 = 2 | 2 | 计算对 |
 | 选择操作后| 2 | 3 | 2 | 0 | 最小（k，最大操作数）|

 我们最多可以执行2次操作。 最终尺寸为$5 - 2 = 3$。 

这表明，即使阵列很小，限制因素也是奇偶校验组内的配对。 

### 示例 2

 输入：```
n = 6, k = 10
a = [2, 4, 6, 1, 3, 5]
```| 步骤| 偶| 奇数| 最大行动 | 剩余 k | 行动|
 | --- | --- | --- | --- | --- | --- |
 | 开始| 3 | 3 | 1 + 1 = 2 | 10 | 10 计算对 |
 | 选择操作后| 3 | 3 | 2 | 8 | k 不具有约束力 |

 尽管我们只能执行 2 个操作$k$很大。 最终尺寸为$6 - 2 = 4$。 

这表明$k$只是一个上限，并不能保证减少。

 ## 边缘情况

 一种边缘情况是任一奇偶校验的元素少于两个。 例如，输入：```
n = 3, a = [2, 1, 3]
```这里偶数=1，奇数=2。只能形成一对奇数对，因此最多可以进行一次操作。 该算法计算$\lfloor odd/2 \rfloor = 1$，给出最终答案$3 - 1 = 2$，它匹配唯一有效的合并。 

另一种情况是当$k$非常大：```
n = 4, k = 100, a = [1, 3, 5, 7]
```偶数为零，奇数为四，因此 maxOps = 2。$k$允许许多操作，我们不能超过可用对，所以最终大小是$4 - 2 = 2$。 该算法正确地限制了操作$\min(k, maxOps)$，防止过度减法。 

最后一个微妙的情况是奇偶校验高度不平衡时：```
n = 5, a = [2, 4, 6, 8, 1]
```偶数 = 4，奇数 = 1。只能进行两种操作，都在偶数范围内。 这个奇怪的元素始终保持不变。 结果是$5 - 2 = 3$，表明孤立元素自然存在，无需特殊处理。
