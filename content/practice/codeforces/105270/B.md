 ## 解决方案
 ## 问题理解

 给定一个数组，我们可以查看它的任何连续部分。 对于每个这样的段，我们计算它的 MEX，即不出现在段内的最小非负整数。 在所有细分市场中，我们首先关心那些MEX尽可能大的细分市场。 确定可实现的最大 MEX 值后，我们会查看实现该值的所有段，并选择长度最小的段。 任务是仅输出最小长度。 

重新解释这一点的一个有用方法是思考一个细分市场拥有大型 MEX 意味着什么。 如果一个段的 MEX 等于 k，则从 0 到 k−1 的每个值必须在其中至少出现一次，而 k 本身必须不存在。 因此，最大化 MEX 意味着我们希望一个段包含尽可能多的从 0 开始的不同小整数，而不包括第一个缺失的整数。 

这些约束意味着我们不能尝试所有 O(n²) 段。 由于测试用例的总 n 高达 10⁵，即使 O(n√n) 也是不安全的。 每个测试用例我们都需要一些线性或近线性的东西。 

当存在重复时，幼稚思维的一个微妙的失败案例就会出现。 例如，如果数组是`[0, 1, 0, 1, 2]`，很容易认为较长的分段更好，因为它们“积累”了更多的值，但 MEX 仅取决于存在性，而不取决于频率。 任何错误跟踪计数或尝试贪婪扩展而不检查缺失值的方法都可能高估段质量。 

另一个棘手的情况是 0 完全缺失。 那么每个段都有 MEX 0，因此最大 MEX 为 0，答案就成为最短的可能段，即 1。 

## 方法

 强力解决方案将枚举每一对 (l, r)，计算该段的 MEX，并跟踪最佳结果。 如果通过一组扫描完成，计算 MEX 本身需要 O(r−l+1)，因此在最坏的情况下，完整的方法将变为 O(n3)。 即使使用频率数组和增量更新进行优化，我们仍然需要考虑 O(n²) 段，这对于高达 10⁵ 的 n 来说太大了。 

关键的观察结果是，段的 MEX 仅取决于从 0 到某个 k−1 的所有数字是否都出现在其中。 因此，我们不去考虑任意的​​部分，而是关注我们想要包含的价值观。 

令 M 为整个数组的全局 MEX。 任何段的 MEX 都不可能大于 M，因为 M 本身到处都不存在。 因此，最大可能的 MEX 正是 M。所以我们只需要包含中每个数字的段`[0, M−1]`。 

现在问题变成：在覆盖从 0 到 M−1 的每个值至少出现一次的所有线段中，找到最短的一个。 这是一个经典的“覆盖一组所需元素的最小窗口”问题。 我们可以使用带有频率计数的滑动窗口来解决它。 

我们维护一个窗口 [l, r] 并扩展 r，同时跟踪我们覆盖了多少个不同的所需值。 一旦窗口包含 0 到 M−1 的所有值，我们就会尝试缩小 l 以最小化长度，同时保留覆盖范围。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n3) | O(1) | O(1) | 太慢了 |
 | 滑动窗| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先计算数组的全局 MEX。 这是通过标记从 0 开始的哪些值出现，并在第一个缺失的值处停止来完成的。 设该值为M。 

然后我们解决集合上的覆盖问题`{0, 1, ..., M−1}`。 

1. 为当前窗口中的值构建频率数组或哈希图。 我们只关心价值观`[0, M−1]`，忽略更大的所有内容，因为它们不影响 MEX。 

原因是 MEX 仅取决于最小缺失值的缺失。 
2. 维护计数器`have`它跟踪窗口中当前存在多少个不同的所需值。 
3. 展开右侧指针`r`步步。 对于每个新元素，如果它在`[0, M−1]`，增加其频率。 如果它第一次出现，则增加`have`。 
4. 什么时候`have == M`，当前窗口包含所有需要的值，因此它是有效的并且 MEX 至少为 M。由于 M 是可能的最大值，因此该窗口在 MEX 方面是最优的。 
5、此时，缩小左指针`l`尽可能多的同时仍然保持`have == M`。 每次我们搬家`l`，我们更新频率并可能减少`have`。 
6. 在每个有效窗口期间，用所见的最小长度更新答案。 
7. 继续，直到右指针到达末尾。 

### 为什么它有效

 关键的不变量是在任何时刻，`have == M`当且仅当当前窗口包含至少一次出现的每个整数`[0, M−1]`。 这个条件完全等价于 MEX 等于 M 的窗口。由于没有段可以超过 MEX M，因此每个有效窗口都是候选解。 收缩步骤确保对于每个右端点，我们只考虑最小的有效左端点，因此不会错过最佳片段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        seen = [False] * (n + 2)
        for x in a:
            if x <= n + 1:
                seen[x] = True

        mex = 0
        while seen[mex]:
            mex += 1

        if mex == 0:
            print(1)
            continue

        freq = [0] * (mex + 1)
        have = 0
        ans = n
        l = 0

        for r in range(n):
            x = a[r]
            if 0 <= x < mex:
                if freq[x] == 0:
                    have += 1
                freq[x] += 1

            while have == mex:
                ans = min(ans, r - l + 1)
                y = a[l]
                if 0 <= y < mex:
                    freq[y] -= 1
                    if freq[y] == 0:
                        have -= 1
                l += 1

        print(ans)

if __name__ == "__main__":
    solve()
```第一部分通过标记出现的整数来计算全局 MEX。 这是必要的，因为它定义了目标覆盖范围集。 然后，滑动窗口仅在低于该 MEX 的值上运行，因为较大的值无关紧要。 

收缩环是最精致的部分。 条件`while have == mex`确保我们不断地最小化每个固定右边界的左边界，这就是保证最小段长度的原因。

 ## 边缘情况

 当数组不包含 0 时，计算出的 MEX 为 0，算法直接输出 1。对于输入`[5, 6, 7]`，不需要频率处理，因为没有段可以将 MEX 提高到 0 以上。 

当所有必需的值出现多次时，滑动窗口必须忽略重复项。 为了`[0,1,0,1,2]`，频率逻辑确保`have`只计算不同的存在，因此缩小不会破坏正确性。 

当所需值分散得很远时，窗口会扩大，直到包含最后一个所需值，然后急剧收缩。 这保证了即使最佳段开始较晚，当右指针到达其端点时仍然会被发现。
