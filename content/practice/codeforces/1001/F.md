 ## 解决方案
 ## 问题理解

 我们得到了一系列量子位，但重要的承诺比量子框架所暗示的要简单：系统保证处于两种经典计算基础状态之一。 每个基本状态由一个布尔数组描述，`bits0`和`bits1`，对应于两个不同长度的位串`N`。 

更具体地说，量子位不是任意叠加的。 整个寄存器已经处于干净的经典配置中，或者完全等于`bits0`或完全等于`bits1`。 任务是确定当前存在并返回哪一个`0`或者`1`因此。 

我们被允许使用的唯一工具是对所提供的量子位进行量子测量。 一旦测量，状态就会崩溃，但这无关紧要，因为我们只需要一个决定。 

提示中没有明确说明约束条件，但此类典型的交互式量子问题假设最大`N`（在测试中通常是 10^5 或更多）。 这排除了任何二次或涉及状态演化重复模拟的情况。 任何有效的解决方案都必须在量子位数量的线性时间内运行，因为每个量子位最多可以检查一次。 

一个微妙的边缘情况是`bits0`和`bits1`仅在一个位置上有所不同。 在这种情况下，尝试“尽早猜测”而不检查该位置的解决方案可能会失败。 例如，如果不同的索引位于末尾并且算法提前停止，则可能会错误地得出相等的结论。 

另一个边缘情况是忘记测量是破坏性的。 如果解决方案测量量子位，然后尝试以非本地方式将其重新用于比较逻辑，则存在对无效状态进行推理的风险。 在这个问题中，我们通过将测量视为完整位串的一次性提取来完全避免这种情况。 

## 方法

 强力解释是尝试直接推理量子态，模拟所有可能性或重复查询结构。 这会导致不必要的开销，并且在概念上与问题保证不一致。 由于该状态被承诺已经是计算基础状态，因此无需模拟分支演化。 任何将其视为一般量子判别问题的尝试都会退化为指数推理或至少是重复的测量策略，这是不必要的。 

关键的观察是，测量计算基础中的每个量子位会立即揭示整个隐藏的位串。 一旦我们有了观察到的字符串，问题就简化为一个简单的相等性检查`bits0`和`bits1`。 由于它们至少在一个位置上不同，因此一次比较将恰好匹配。 

这将任务从“量子态识别”减少为“读取和比较二进制数组”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（量子推理/重复检查）| O(2^N) 或更糟的概念开销 | O(N)-O(2^N) | O(N)-O(2^N) | 太慢了 |
 | 最佳（测量一次，比较）| O(N) | O(N) | 已接受 |

 ## 算法演练

 1. 测量计算基础中的每个量子位并将得到的经典位存储在数组中`obs`。 这一步直接提取隐藏的经典状态，因为输入保证已经是基状态。 
2. 比较`obs`和`bits0`。 如果每个位置都匹配，则状态完全正确`bits0`，所以返回`0`。 正确性来自于状态必须是两个给定字符串之一的承诺。 
3.否则返回`1`，因为唯一剩下的可能性是`bits1`。 保证`bits0`和`bits1`至少在一个位置上不同可确保不会产生歧义。 

### 为什么它有效

 不变量是测量后，`obs`与量子位寄存器的真实底层计算基础状态相同。 由于输入状态保证恰好是`bits0`或者确切地说`bits1`，观察到的字符串必须与其中之一完全匹配。 由于这两个候选者是不同的，因此等式比较可以唯一地识别出正确的候选者，而无需任何概率推理。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve(qs, bits0, bits1):
    obs = [False] * len(qs)
    
    for i in range(len(qs)):
        # In actual Q# this would be: M(qs[i])
        # Here we assume qs already encodes classical info in simulation form
        obs[i] = qs[i]
    
    if obs == bits0:
        return 0
    return 1
```该实现反映了这样一个事实：唯一有意义的操作是从每个量子位中提取经典信息。 在真实的 Q# 环境中，这对应于应用测量`M`在每个量子位上。 之后的逻辑就纯属经典了。 

比较步骤是安全的，因为该问题保证了排他性：两个位串之一恰好与测量结果匹配。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N) | 每个量子位测量一次并比较一次 |
 | 空间| O(N) | 观察到的比特串的存储 |

 线性复杂度是最佳的，因为每个量子位必须至少检查一次，以区分在任何位置可能不同的两个任意位串。 这非常适合量子模拟或测量问题的典型约束。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    
    # simple simulation wrapper
    data = list(map(int, sys.stdin.read().split()))
    n = data[0]
    bits0 = data[1:1+n]
    bits1 = data[1+n:1+2*n]
    qs = data[1+2*n:1+3*n]
    
    def solve(qs, bits0, bits1):
        return 0 if qs == bits0 else 1
    
    return str(solve(qs, bits0, bits1))

# differs in last bit

 ## 边缘情况

 一种边缘情况是当`bits0`和`bits1`仅在最后一个索引处有所不同。 该算法仍然有效，因为测量不依赖于提前终止。 在比较之前始终读取完整数组，因此永远不会跳过区分位。 

另一个边缘情况是`N = 1`。 这里的量子位是`0`或者`1`，并且两个候选者都是单位字符串。 测量产生一个单元素数组，并且相等比较仍然可以唯一地标识正确的状态，而不会产生歧义。 

最后的边缘情况是两个位串除了单个共享前缀之外都是互补的。 该算法不依赖于差异所在，只依赖于全字符串相等，因此位置偏差不会影响正确性。
