 ## 解决方案
 ## 问题理解

 我们得到了一个最多八个量子位的小寄存器，所有量子位最初都是在全零计算基础状态下准备的。 任务是将这个寄存器转换成一种称为 GHZ 状态的特定纠缠态，其中所有量子位都完全相关，即系统处于全零和全一配置的相等叠加。 

具体来说，期望的最终状态是两分支叠加：一个分支中每个量子位都是 0，另一个分支中每个量子位都是 1，且幅度相等。 挑战不是计算数值答案，而是应用一系列量子运算，从初始基态产生精确的变换。 

限制非常小，最多有八个量子位。 这立即告诉我们，任何与量子位数量呈线性的解决方案都是足够的，甚至常数因子开销也是无关紧要的。 真正的困难不是性能，而是了解从单个局部操作生成全局纠缠的量子门的正确结构。 

天真的误解常常来自于试图将量子位独立地“设置”为叠加态。 例如，将 Hadamard 门应用于每个量子位将在所有 2^N 位串上创建统一的叠加，而不是所需的受限两态叠加。 对于 N 等于 2，这会产生四个状态，而不是所需的两个相关状态，因此对于 GHZ 准备来说从根本上来说是不正确的。 

另一个微妙的错误是假设无需选择单个“控制”量子位即可实现纠缠。 如果没有参考量子位，就没有机制将所有量子位关联为相同的值，因此任何对称本地操作都无法强制全局一致性。 

## 方法

 考虑目标状态的一种强力方法是想象显式构建一个将 |0...0⟩ 映射到 (|0...0⟩ + |1...1⟩)/sqrt2 的量子电路。 人们可能会尝试直接推断振幅，枚举每个门如何影响所有 2^N 基本状态。 即使对于中等的 N，这也会很快变得难以管理，因为每个操作都会混合整个状态空间的幅度，并且跟踪正确性需要指数簿记。 

关键的结构观察是 GHZ 态具有非常简单的生成形式。 我们可以首先在单个量子位上创建叠加，然后使用受控操作将其值确定性地传播到所有其他量子位，而不是尝试立即构建所有相关性。 一旦第一个量子位处于 0 和 1 的平衡叠加状态，通过受控非门将其值复制到每个其他量子位中可确保所有量子位镜像其状态。 这恰好产生了两个全局配置，一个用于初始叠加的每个分支。 

暴力方法会失败，因为它试图将 GHZ 状态视为全局对象。 建设性方法之所以有效，是因为它将问题简化为创建单一随机源，然后通过纠缠进行分配。

 ## 算法演练

 我们使用单个 Hadamard 门和一系列受控非运算来构建 GHZ 状态。

1. 将 Hadamard 门应用于第一个量子位。 这会将其从确定的 0 状态转换为 0 和 1 的相等叠加。此步骤创建 GHZ 所需的分支结构。 
2. 对于从索引 1 到 N − 1 的每隔一个量子位，应用受控非门，其中第一个量子位作为控制，当前量子位作为目标。 这确保了每当第一个量子位为 1 时，所有其他量子位也会翻转为 1，而当它为 0 时，它们仍保持为 0。 
3. 将这种相关性传播到所有量子位后，寄存器恰好包含两个具有相等幅度的计算基础配置，即 GHZ 状态。 

### 为什么它有效

 正确性来自于在每个受控非运算之后维护一个简单的不变量：每个处理的量子位都与叠加的每个分支中的第一个量子位的值匹配。 Hadamard 门在第一个量子位上创建两个分支状态，每个受控非保留这些分支的结构，同时将其一致性扩展到其他量子位。 由于没有任何操作会混合两个分支或引入新的基础状态，因此最终状态准确地保持两项叠加，其中每一项中的所有量子位都是相同的。

 ## 边缘情况

 对于 N = 1，不需要纠缠操作，因为在应用 Hadamard 后 GHZ 状态退化为叠加的单个量子位。 该算法将 Hadamard 应用于唯一的量子位，并且不执行 CNOT，立即保持正确的状态。 

对于 N = 2，电路在哈达玛之后简化为单个纠缠门。 第一个量子位成为叠加，并且 CNOT 正确地产生贝尔态。 任何尝试应用额外的基于对称的门都会错误地重新引入不需要的基本状态，但该算法通过使用单个传播步骤完全避免了这种情况。
