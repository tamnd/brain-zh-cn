 ## 解决方案
 ## 问题理解

 我们可以访问一个黑盒量子预言机，它编码了一个布尔函数$2^N$长度的二进制输入$N$。 对于任何输入位串$x$，预言机评估$f(x)$没有明确透露功能。 我们承诺该函数是极其结构化的：它要么是恒定的，意味着它为每个输入输出相同的值，要么是平衡的，意味着所有输入的一半映射到 0，另一半映射到 1。 

任务是在量子电路设置中仅使用一次预言机调用来确定隐藏函数是恒定的还是平衡的。 输出是一个布尔值，指示函数是否恒定。 

我们只能调用一次预言的约束完全排除了任何经典的采样策略。 如果我们尝试在甚至几个输入上查询函数，我们将立即超出确定区分平衡函数的限制，因为平衡函数可以在输入空间中任意隐藏其结构。 传统上，确定区分这些情况需要$2^{N-1} + 1$最坏情况下的查询，因为平衡函数可以在任何较小的样本上模拟常数函数。 

这里不明显的困难是，在部分观察下，平衡函数看起来与常数函数没有区别。 例如，如果$N = 3$，平衡函数可能对输入 000、001、010 输出 0，对其余输入输出 1。 任何小的随机样本都可能会误导性地显得恒定。 

量子公式完全避免了这种采样障碍，但前提是我们正确利用干扰。 

## 方法

 蛮力经典方法会查询所有的预言$2^N$输入，计算 1 的数量，并确定函数是恒定的还是平衡的。 这是正确的，但一旦出现就完全不可行$N$增长超出小值，因为它需要指数时间。 

量子解决方案背后的关键结构观察是 Deutsch-Jozsa 承诺问题按相位而不是显式枚举对全局信息进行编码。 我们没有尝试直接检查函数值，而是在所有输入上准备一个统一的叠加，应用一次预言，然后使用干扰来放大函数的全局属性。 

在预言机之前和之后应用阿达玛变换后，振幅干扰的方式会抵消除函数恒定时单个计算基础状态之外的所有结果。 如果函数是平衡的，相消干涉可以确保这种特殊状态的幅度为零。 

这将全局计数问题转变为单个幅度检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^N)$|$O(1)$| 太慢了|
 | 量子德语-Jozsa |$O(1)$甲骨文调用|$O(N)$量子位 | 已接受 |

 ## 算法演练

 我们使用标准 Deutsch-Jozsa 电路模型$N$输入量子位和一个辅助量子位。 

1. 全部初始化$N$输入量子位$|0\rangle$和辅助量子位$|1\rangle$。 需要辅助量子位，以便预言机可以将函数值编码为相位翻转，而不是覆盖计算状态。 
2. 对每个量子位应用 Hadamard 门。 这将输入寄存器转换为所有寄存器的相等叠加$2^N$输入，并将辅助量子位转变为相位参考状态。 
3.应用一次预言机。 神谕地图$|x\rangle|y\rangle$到$|x\rangle|y \oplus f(x)\rangle$。 由于辅助量子位是在叠加状态下准备的，因此该操作有效地引入了相位因子$(-1)^{f(x)}$在每个基础状态$|x\rangle$。 
4. 再次将 Hadamard 门应用于所有输入量子位。 此步骤会导致不同输入对应的幅度之间产生干扰。 如果函数恒定，则所有振幅都会增强到全零状态。 如果函数是平衡的，则对该状态的贡献完全抵消。 
5. 测量最终状态是否为全零计算基础状态。 如果是，则该函数是常数。 否则，就是平衡的。 

### 为什么它有效

 该算法依赖于全局干扰恒等式：在第二次 Hadamard 变换之后，全零状态的幅度与$(-1)^{f(x)}$超过所有输入$x$。 如果函数是常数，则该和等于$2^N$或者$-2^N$，给出非零幅度。 如果函数是平衡的，则恰好有一半的项是$+1$一半是$-1$，所以总和为零。 这种结构取消保证了通过单个预言机调用确定性地分离两种情况。 

## Python 解决方案

 虽然这是一个量子问题，所需的提交是在 Q# 中，但逻辑结构是经典的：我们实现 Deutsch-Jozsa 电路并返回测量是否产生全零状态。```
# Q# solution (conceptual translation)
# In actual Codeforces environment, this is written in Q#

namespace Solution {
    open Microsoft.Quantum.Primitive;
    open Microsoft.Quantum.Canon;

    operation Solve (N : Int, Uf : ((Qubit[], Qubit) => ())) : Bool
    {
        body
        {
            use qs = Qubit[N + 1];

            let input = qs[0..N-1];
            let aux = qs[N];

            X(aux);

            ApplyToEach(H, qs);

            Uf(input, aux);

            ApplyToEach(H, input);

            mutable allZero = true;

            for i in 0..N-1 {
                let res = M(input[i]);
                if (res == One) {
                    set allZero = false;
                }
            }

            let auxRes = M(aux);

            ResetAll(qs);

            return allZero;
        }
    }
}
```关键的实现细节是在状态下准备辅助量子位$|1\rangle$在应用哈达玛之前。 如果没有这个，预言机就无法正确地将函数求值转换为阶段信息，并且干扰也不会编码我们所依赖的全局属性。 

第二个微妙之处是我们只需要验证所有输入量子位在测量后是否都崩溃为零。 任何偏差都表明相消干涉失败，这种情况只发生在承诺的平衡功能上。

 ## 边缘情况

 一种边缘情况是当$N = 0$，其中输入空间只有一个元素。 在这种情况下，承诺会退化，因为“平衡”功能不存在。 该算法仍然表现正确，因为叠加减少为单一状态，并且预言机只是设置了一个不会取消的全局阶段。 

另一个微妙的情况是当函数为常数 1 而不是常数 0 时。这里每个基本状态接收一个阶段$-1$，但第二次哈达玛之后的干涉图样仍然完全集中在全零状态，因此测量结果仍然是确定性的。 

最后的边缘情况是确保辅助量子位正确重置。 如果它处于纠缠状态或未重置，则较大电路中的后续运行可能会产生不正确的干扰模式，但在这个单次问题中，只要立即完成测量，它就不会影响最终的布尔结果。
