 ## 解决方案
 ## 问题理解

 我们得到一对量子位，它们是在四个特定的双量子位纠缠态之一中准备的。 这四种状态形成了贝尔基础，这意味着它们是最大纠缠的，并且仅在相对相位和位翻转方面有所不同，而不是在每个量子位上的局部经典信息方面有所不同。 

任务不是重建振幅或执行量子态断层扫描。 相反，我们必须应用固定的量子门序列，然后测量量子位，以便测量结果唯一地识别我们从四个贝尔态中的哪一个开始。 输出是 0 到 3 之间的单个整数，表示该身份。 

关键约束是概念上的而不是数字上的。 我们只允许在两个量子位上进行恒定数量的量子操作。 这排除了类似统计抽样或重复测量的任何情况。 任何正确的解决方案都必须一次性确定地将纠缠信息转换为计算基础信息。 

立即测量量子位是一个天真的错误。 这破坏了纠缠，只留下相关的随机性，使得所有四种贝尔态都无法区分。 例如，两者$|\Phi^+\rangle$和$|\Psi^+\rangle$如果直接测量，即使它们的相关性不同，也会产生均匀随机的位对。 另一个失败案例是尝试独立测量每个量子位并以经典方式解释结果，这会丢失区分正态和负态的相位信息。 

第二种微妙的边缘情况是仅应用 CNOT 门或仅应用 Hadamard 门。 其中每一个都部分地变换了基础，但没有将所有四个贝尔状态完全分离成正交计算基础状态。 缺少的转换是将相位差压缩为可测量的位翻转。 

## 方法

 蛮力量子策略将尝试通过重复准备相同的系统并估计测量结果的概率来区分状态。 这将依赖于分布的统计分离。 原则上，每个贝尔状态在随机门序列后都会在测量结果上产生不同的分布，但以高置信度区分它们需要多次重复。 由于我们只有一个状态副本，因此这种方法在问题设置中是不可能的。 

关键的观察是贝尔状态不是任意的。 它们正是将 Hadamard 门和 CNOT 门应用于计算基础状态的结果$|00\rangle, |01\rangle, |10\rangle, |11\rangle$。 这意味着可以通过反转该电路来“撤销”贝尔基础。 如果我们按照正确的顺序应用 CNOT，然后应用 Hadamard，我们会将每个贝尔状态映射回唯一的计算基础状态。 

一旦转换回计算基础形式，对两个量子位的一次测量就会产生两个直接编码原始状态索引的经典位。

 ## 算法演练

 我们假设第一个量子位是控制，第二个量子位是目标。 

1. 将 CNOT 门从第一个量子位应用到第二个量子位。 

此步骤消除了贝尔态的位翻转分量中的纠缠结构。 在此操作之后，四个贝尔状态的区别仅在于第一个量子位上的单量子位变换。 
2. 对第一个量子位应用 Hadamard 门。 

这将转换之间的相位差$|+\rangle$和$|-\rangle$转化为可测量的计算基础差异。 如果没有这个步骤，“正与负”贝尔态在测量后将仍然无法区分。 
3. 在计算基础上测量两个量子位。 

测量将状态折叠成经典的位串$b_0 b_1$。 
4. 将结果位解释为整数$2 \cdot b_0 + b_1$。 

该整数是原始贝尔状态的标识。 

### 为什么它有效

 正确性来自于以下事实：第一个量子位上的 Hadamard 后的 CNOT 恰好是标准贝尔态准备电路的逆。 由于贝尔状态形成由计算基的酉变换产生的正交基，因此应用逆酉将每个贝尔状态映射到唯一的计算基向量。 因此，在该基础上的测量对于初始状态的身份而言是确定性的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Quantum solution is conceptual; in actual Codeforces MQL environment,
# these operations are provided by the framework.
# We return the measurement result encoded as integer.

def Solve(qs):
    # Apply CNOT: qs[0] controls qs[1]
    CNOT(qs[0], qs[1])

    # Apply Hadamard to first qubit
    H(qs[0])

    # Measure both qubits
    b0 = M(qs[0])
    b1 = M(qs[1])

    return b0 * 2 + b1
```该结构遵循标准贝尔基反演电路。 首先应用 CNOT 门来消除计算组件之间的纠缠。 然后，第一个量子位上的哈达玛将相位模糊度解析为可测量的基差。 最后，测量产生两个直接编码原始贝尔索引的经典位。 

一个微妙的实现细节是整数转换中量子位的顺序。 第一个测量的量子位贡献高位。 即使量子变换本身是正确的，反转此映射也会排列输出并导致不正确的索引。

 ## 边缘情况

 常见的边缘情况是在应用完整逆电路之前进行测量。 例如，如果我们立即测量$|\Psi^-\rangle$状态，结果是 01 或 10 的概率相等，无法与其他贝尔状态区分开来。 该算法通过确保在任何测量之前应用 CNOT 和 Hadamard 来避免这种情况。 

另一个微妙的情况是交换 CNOT 中的量子位角色。 如果目标和控制颠倒，则变换不再与逆贝尔准备电路匹配，并且位串和贝尔索引之间的映射变得不一致。 固定的控制目标方向对于正确性至关重要，因为贝尔状态在任意局部反演下不是对称的。 

如果哈达玛应用于第二个量子位而不是第一个量子位，则会出现第三种情况。 这打破了相位信息和计算基础编码之间的一致性，导致混合输出，其中多个贝尔状态崩溃为相同的测量结果分布。
