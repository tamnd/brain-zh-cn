 ## 解决方案
 ## 问题理解

 我们得到了一个数组，据称该数组来自一个非常结构化的结构。 固定一个正整数$k$。 如果我们列出所有产品$i \cdot j$为了$1 \le i \le k$和$1 \le j \le k$，逐行，我们得到一个长度$k^2$顺序。 每行$i$包括$i, 2i, 3i, \dots, ki$，并且所有行都按递增顺序连接$i$。 

输入为我们提供了这样一个扁平乘法表的连续部分。 我们的任务是确定最小的可能$k$这样该段就可以出现在$k \times k$乘法表。 

关键的结构约束是段必须与行的全局排序保持一致。 我们匹配的不是一个集合，而是一个非常严格、重复的算术模式的子串。 

约束足够小$n \le 100$，因此任何每个测试用例都是三次的解决方案都可以，但我们仍然需要仔细推理每个候选者的有效性$k$，因为表增长为$k^2$结构取决于可分性模式。 

当值大量重复时，就会出现一个微妙的问题。 例如，像这样的序列$[2, 2, 2, 2]$或者$[1, 1, 1]$可以以多种方式出现在不同的领域$k$，并且朴素的贪婪匹配可以轻松地选择不一致的行解释。 另一个棘手的情况是当段跨越行边界时，因为展平不会插入分隔符； 有效的匹配可以从一行的中间开始并在另一行的中间结束。 

例如，与$k = 4$，该表开始为：$[1,2,3,4,2,4,6,8,3,6,\dots]$。 

像这样的一段$[4,6,8,10]$不对应于单行，而是跨越较大表中的第 2 行和第 5 行。 仅在行内进行检查的天真的想法会错过这种情况。 

## 方法

 蛮力方法是修复一个候选者$k$并明确构建整个$k \times k$展平表，然后检查给定数组是否显示为子字符串。 建表成本$O(k^2)$，并且扫描模式需要花费另一笔费用$O(k^2)$，所以每次检查都是$O(k^2)$。 总结一切$k$最多$K$大致给出$O(K^3)$，这里是可以接受的，因为$k$不是直接给出的，必须从数组中的值导出。 

然而，我们仍然需要一种方法来检查固定的可行性$k$没有完全构建表。 

关键的观察结果是表中的每个元素都是一个产品$i \cdot j$，因此数组的任何有效对齐都必须分配每个元素$a_t$对某些对$(i_t, j_t)$和$1 \le i_t, j_t \le k$。 此外，连续元素之间的转换受到限制：如果我们保持在一行内$i$，那么连续的值必须精确地相差$i$。 如果我们跳到下一行，我们会重置乘数索引$j$，但我们仍然按照全球秩序继续下去。 

这导致了动态可行性检查：对于固定的$k$，我们尝试表中所有可能的起始位置，并尝试模拟序列是否可以连续匹配。 因为$n \le 100$，我们可以进行相当强的状态探索，但关键的修剪是每个状态由当前行决定$i$和列索引$j$，因为该值固定为$i \cdot j$。 

我们不是构建整个表，而是模拟转换：$(i,j)$，下一个值是$(i, j+1)$如果$j < k$， 或者$(i+1, 1)$如果我们移动到下一行。 这意味着该表是在大小网格上的确定性遍历$k^2$，我们正在检查该序列是否是该遍历的子串。 

因此对于每个$k$，我们尝试每一种可能的起始状态$(i,j)$，遵循转换，并验证我们是否可以匹配所有$n$元素。 

因为$n \le 100$， 这是$O(k^2 \cdot n)$，并且自从$k$受数组中的值限制（最多$10^9$，但我们只需要考虑$\max(a)$为了可分割性的可行性），实际上我们可以在第一个有效的时候提前停止$k$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(k^3)$或者$O(k^2 \cdot n)$每张支票 |$O(1)$| 概念上太慢|
 | 最佳 |$O(n \sqrt{\max a})$或者$O(n \cdot k)$修剪|$O(1)$| 已接受 |

 ## 算法演练

 主要困难在于，如果数组来自乘法表，则需要对其必须满足的结构进行逆向工程。 

我们利用这样一个事实：每一行都是一个等差级数，其差值等于其行索引。 因此，如果两个连续元素属于同一行，则它们的差值是恒定的并且等于行号。 

我们不知道行边界，因此我们尝试为每个段分配一致的行索引。 关键思想是，连续段的任何有效表示都可以分解为留在一行内的最大运行，并且这些运行必须与增加的行索引一致。 

因此我们会检查候选人$k$尝试将序列解释为通过遍历无限扁平的行网格而生成的$1$到$k$，每一行是$i, 2i, \dots, ki$，按顺序概念性地重复。 

### 步骤

 1.对于固定候选人$k$，考虑概念网格，其中行$i$是序列$i, 2i, \dots, ki$。 我们没有明确地构建它，但我们依赖于这样一个事实：从任何位置$(i,j)$，下一个元素是唯一确定的。 
2. 尝试所有可能的起始单元格$(i,j)$和$1 \le i \le k$和$1 \le j \le k$。 起始值必须匹配$a_0$，所以我们只将候选人保留在$i \cdot j = a_0$。 
3. 对于每个有效的起始对，向前模拟序列。 在步骤$t$，我们在$(i_t, j_t)$，我们检查是否$i_t \cdot j_t = a_t$。 如果不是，则本次启动无效。 
4. 确定性地推进：如果$j_t < k$，移至$(i_t, j_t + 1)$，否则移至$(i_t + 1, 1)$。 如果$i_t > k$，我们离开餐桌后就很早就停下来了。 
5. 如果任何起始状态成功匹配所有元素，则当前状态$k$是可行的。 
6. 答案是最小值$k$可行性成立。 

### 为什么它有效

 展平表是对所有对的单一确定性遍历$(i,j)$按行和列的字典顺序。 任何连续子数组都精确对应于该遍历的连续段。 因此，每个有效的解决方案都必须对应于该遍历中的某个起始位置，并且一旦起始位置固定，序列就被强制。 这消除了歧义：正确性减少为检查所有可能的开始并验证一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(a, k):
    n = len(a)

    # try every starting position (i, j)
    for i in range(1, k + 1):
        for j in range(1, k + 1):
            if i * j != a[0]:
                continue

            x, y = i, j
            good = True

            for t in range(n):
                if x > k:
                    good = False
                    break
                if x * y != a[t]:
                    good = False
                    break

                # move to next cell in flattened table
                if y < k:
                    y += 1
                else:
                    x += 1
                    y = 1

            if good:
                return True

    return False

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        mx = max(a)

        # k must be at least sqrt(max value) in any valid table
        lo, hi = 1, int(mx ** 0.5) + 2
        ans = hi

        for k in range(lo, hi + 1):
            if ok(a, k):
                ans = k
                break

        print(ans)

if __name__ == "__main__":
    solve()
```实现的核心是`ok`函数，它模拟展平乘法表的确定性遍历。 转换规则与结构完全匹配：在一行中我们增加列索引，到达行末尾后我们移动到下一行。 

启动过滤器`i * j == a[0]`立即删除大多数无效状态。 如果没有这种修剪，模拟仍然是正确的，但速度会慢一些$k^2$。 

外循环增加$k$从小到大，并在第一个可行值处停止，因为我们需要最小有效表大小。 

## 工作示例

 考虑输入数组$[4, 6, 8, 10]$。 

我们测试增加$k$。 对于小$k$， 说$k = 3$，表中最大值为$9$， 所以$10$无法出现，使其立即无效。 为了$k = 5$，我们尝试从哪里开始$i \cdot j = 4$，即$(1,4), (2,2), (4,1)$。 

| 步骤| 开始 (i,j) | 当前值| 数组索引 | 比赛状态 |
 | --- | --- | --- | --- | --- |
 | 1 | (1,4) | 4 | 0 | 好的 |
 | 2 | (1,5) | 5 | 1 | 不匹配|
 | 1 | (2,2) | 4 | 0 | 好的 |
 | 2 | (2,3) | 6 | 1 | 好的 |
 | 3 | (2,4) | 8 | 2 | 好的 |
 | 4 | (2,5)->(3,1) | 10 | 10 3 | 好的 |

 这表明嵌入成功，因此$k = 5$。 

现在考虑$[1,2,2,4]$。 

为了$k = 2$，展平后的表为$[1,2,2,4]$，所以开始于$(1,1)$立即匹配：

 | 步骤| 职位| 价值| 数组值 |
 | --- | --- | --- | --- |
 | 1 | (1,1) | 1 | 1 |
 | 2 | (1,2) | 2 | 2 |
 | 3 | (2,1) | 2 | 2 |
 | 4 | (2,2) | 4 | 4 |

 匹配成功，无需更大$k$。 

这些痕迹证实该算法有效地搜索确定性遍历中的所有有效入口点。

 ## 边缘情况

 一种微妙的情况是所有值都相同，例如$[1,1,1,1]$。 该算法正确地处理了这个问题，因为对于任何$k \ge 1$，存在多个起始位置，其中$i \cdot j = 1$，特别是仅$(1,1)$。 模拟来自$(1,1)$确定性地通过$(1,2), (1,3), \dots$，产生长期不断增加的价值，而不是恒定的价值。 因此只有$k = 1$有效，并且算法正确地拒绝了更大的$k$。 

另一种边缘情况是当序列需要立即跨越多个行边界时，例如$[k, k+1, k+2]$对于某些候选人$k$。 转换规则强制在列之后增加行$k$，因此预期跳跃的任何不匹配都会导致提前开始无效。 模拟确保不可能出现无效的行跳过，因为行转换是确定性的且固定的。
