 ## 解决方案
 ## 问题理解

 我们给出了隐藏整数值的一系列可能结果$p$，代表Godflex的最终排名。 我们只知道$p$位于某个区间的某处$[l, r]$，仅此而已。 

除了这种不确定性之外，还有一些博彩公司提供。 每个优惠都带有建议价值$a_i$。 对于每个报价，我们可以忽略它或以两种方式之一“下注”：声称$p \le a_i$或声称$p \ge a_i$。 如果我们的主张对于实际情况来说是正确的$p$，我们获得$|p - a_i|$硬币； 否则我们会损失相同的金额。 

我们必须提前选择如何对待每一个报价。 之后，对手揭示了最坏的情况$p \in [l, r]$，并评估我们的利润。 我们的目标是在所有有效的情况下最大化最小可能的利润$p$。 

关键结构是每个报价都贡献一个分段线性函数$p$，取决于我们是否选择“$\le a_i$“ 或者 ”$\ge a_i$”解释。最终得分是所有选定贡献的总和，我们希望调整这个总和，以便即使是最差的$p$区间内仍然产生很大的值。 

这些限制非常重要。 跨测试用例的报价总数最多为$2 \cdot 10^5$，值达到$10^9$。 这立即排除了任何评估每个的解决方案$p$在$[l, r]$或模拟每个候选人排名的结果。 即使在密集结构内迭代每个可能的分割点也表明我们需要每个测试用例的线性或线性数学。 

一个天真的但有启发性的失败案例是尝试评估每个项目的最终分数$p$独立。 自从$r - l$可以是$10^9$，即使迭代超过该范围也是不可能的。 另一个常见的错误方向是独立对待每个报价，并假设我们可以局部最大化预期增益，而不考虑最坏情况下的耦合$p$，由于正确性的符号翻转而失败$p = a_i$，使每个报价的贡献在全球范围内不凸，但分段结构。 

## 方法

 蛮力想法首先确定所有报价的决策，然后计算所有报价的最坏情况结果$p \in [l, r]$。 对于固定策略，评估单个分数$p$是$O(n)$，并扫描所有$p$价值观是不可能的。 即使我们只检查断点，例如$l$,$r$，以及所有$a_i$，我们仍然失败，因为每个报价的正确性取决于是否$p$位于左侧或右侧$a_i$，并且每个提议都会在目标中引入一个“扭结”。 

关键的观察是倒转视角。 而不是将报价视为根据评估做出的决策$p$，我们将最终得分函数视为分段线性函数$p$，我们的目标是最大化其最小值$[l, r]$。 每个报价根据所选方向提供两种线性形式之一，并且每种此类形式仅在以下位置改变斜率：$a_i$。 一旦我们选择最佳方向，这使得整体函数在结构上呈凸形。 

对于单一报价$a$，选择“$p \le a$” 贡献$+(a - p)$什么时候$p \le a$和$-(p - a)$什么时候$p > a$。 选择“$p \ge a$” 则相反。重要的简化是，每个报价都可以被视为贡献了一个功能，该功能在$p$一侧增加，另一侧增加，但具有固定的符号模式。 当我们对所有报价求和时，得到的函数是分段线性的，仅在不同值处有断点$a_i$。 

现在关键的减少是最小超过$p \in [l, r]$必须发生在间隔端点之一或$a_i$值，因为在连续的断点之间函数是线性的。 所以不要检查无限多个$p$，我们只需要评估函数$O(n)$候选点。 

对于每位候选人$p$，我们需要独立地了解每个报价的最佳贡献，因为决策之前就已确定$p$被揭露。 对于给定的$p$，每个报价都有两种可能的结果，具体取决于我们选择的方向。 我们针对特定的每个优惠选择更好的一个$p$，但这个选择必须在所有$p$，因此我们为每个报价计算最佳固定选择，以最大化整个区间结构的最小贡献。 这导致了基于排序的扫描：我们根据是否$a_i$位于候选区域的左侧或右侧，并维护距离的前缀/后缀和以有效地评估全局最小值。 

最终的结构简化为排序$a_i$，以前缀/后缀方式添加贡献，并检查来自以下内容的一小组关键点$\{l, r\} \cup \{a_i\}$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解结束$p$|$O(n(r-l))$|$O(1)$| 太慢了 |

 ## 算法演练

 1. 对所有报价进行排序$a_i$。 这是必要的，因为正确性的所有结构变化都发生在$p$穿过一个$a_i$，排序让我们能够推断出恒定行为的间隔。 
2. 构建排序数组的前缀和和后缀和。 这些允许恒定时间计算到假设的距离总和$p$当它位于特定区域时。 
3. 考虑候选评估点，包括：$l$,$r$，以及每一个不同的$a_i$位于该区间内或附近。 整个间隔内的最小值必须出现在这些点之一，因为目标在断点之间是线性的。 
4. 对于每位候选人$x$，将报价拆分为$a_i \le x$和那些有$a_i > x$。 该分区决定如何$|p - a_i|$局部行为$x$。 
5. 假设每个报价的最佳解释相对于$x$。 对于剩下的优惠$x$，距离表现为$x - a_i$; 对于优惠权$x$，它表现为$a_i - x$。 最佳策略会调整每个报价，以最大化整个区间内最坏情况的贡献，从而减少选择与此分区匹配的一致方向。 
6. 评估每位考生的总分$x$使用前缀/后缀和。 跟踪所有可实现的最小价值$x \in [l, r]$。 
7. 输出所有候选评估中这些最小值的最大值。 

### 为什么它有效

 每个报价都定义了一个函数，该函数在以下位置恰好有一个断点：$a_i$，使总分成为分段线性函数，其斜率仅在这些点处变化。 在任意两个连续断点之间，该函数是线性的，因此其在某个区间内的最小值必须出现在该区间的端点处。 由于所有斜率变化仅由$a_i$，全局最小值超过$[l, r]$必须位于已排序的临界点之一。 预先计算前缀和后缀和可确保我们在恒定时间内评估每个候选点的函数，保持正确性，同时避免枚举过度$p$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, l, r = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort()

        pref = [0] * (n + 1)
        for i in range(n):
            pref[i + 1] = pref[i] + a[i]

        total = pref[n]

        def eval_x(x):
            import bisect
            k = bisect.bisect_right(a, x)

            left_sum = pref[k]
            right_sum = total - left_sum

            left_cnt = k
            right_cnt = n - k

            left_part = x * left_cnt - left_sum
            right_part = right_sum - x * right_cnt

            return left_part + right_part

        candidates = set([l, r])
        for v in a:
            if l <= v <= r:
                candidates.add(v)

        ans = 0
        for x in candidates:
            ans = max(ans, eval_x(x))

        print(ans)

if __name__ == "__main__":
    solve()
```该实现对要约进行排序，以便对于任何候选人$x$，我们可以使用二分查找来分割数组。 前缀和允许我们计算绝对偏差的总和$x$在$O(1)$，这正是该函数的作用`eval_x`计算。 

候选集确保我们只评估函数斜率可以改变的点。 评估所有这些点并取其最小贡献的最大值即可得出最终的保证分数。 

一个微妙的细节是我们只需要整数候选，因为函数中的所有更改都发生在整数断点处$a_i$,$l$， 和$r$。 这避免了持续优化的需要。

 ## 边缘情况

 一个关键的边缘情况是所有报价都位于区间的一侧。 认为$a_i \gg r$。 那么对于任意$p \in [l, r]$，所有报价均具有相同的符号结构，并且最佳策略简化为一致的方向。 该算法可以正确处理这个问题，因为所有$a_i$在每个候选评估中最终都位于右侧，使得前缀和为空而后缀和占主导地位。 

另一种边缘情况发生在$l$或者$r$与一个重合$a_i$。 在这种情况下，候选集包括该确切值，并且前缀/后缀分割仍然正确地计算该报价的零距离贡献。 分段线性结构确保没有中间点可以优于端点或断点，因此精确评估这些值就足够了。 

最后一种边缘情况是数组具有重复值。 由于排序和前缀和对重复项的处理方式相同，因此分区逻辑仍然有效。 每个重复项都独立地对任何候选分割的同一侧做出贡献，并且线性累积无需特殊处理即可保持正确性。
