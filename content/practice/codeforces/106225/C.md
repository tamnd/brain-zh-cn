 ## 解决方案
 ## 问题理解

 我们得到 n 个序列的集合，每个序列的长度为 m，每个序列对应于一个由隐藏构造过程生成的块。 隐藏过程从长度为 nm/2 的排列开始，将其与自身连接以获得长度为 nm 的数组，然后将这个双倍数组分割为 n 个长度为 m 的连续块，最后在显示它们之前对这些块进行任意排列。 

我们的任务是重建任何长度为 nm/2 的排列，这些排列可以在复制和洗牌后生成给定的块多重集。 

关键的结构事实是，最终构造中的每个值在全长 nm 数组中恰好出现两次，在排列的每个副本中出现一次。 分割成块后，每个数字正好出现在两个块中，因为分割不会破坏重复的结构。 

约束 n、m ≤ 70 意味着每个测试用例 nm ≤ 4900，因此总数据大小足以进行 O(nm log nm) 甚至 O(nm²) 推理。 这是一个强烈的暗示，表明我们应该利用全局频率结构，而不是对块的排列进行任何组合重建。 

块边界对齐会产生一个微妙的问题。 块的分割是固定在双倍数组上的，但我们不知道哪个块对应于原始排列的哪个段。 配对相同块或基于块相等进行重建的天真尝试会失败，因为不能保证块是不同的。 例如，两个相同的块可能来自排列中的不同位置，不一定来自重复的结构对齐。 

真正的挑战是我们没有重建块排序。 我们正在重建底层排列，其重复产生长度为 m 的块的多重集。 

## 方法

 强力解释将尝试通过猜测块如何与双倍数组中的位置对齐来重建排列。 人们可以想象将每个块分配给前半部分或后半部分，然后在重叠处强制执行一致性。 这很快就变成了组合：对于 n 个块，每个块的大小为 m，对 nm/2 位置的分配和对齐呈指数级增长，甚至对于 n, m 大约 70 也是不可行的。 

关键的观察是我们实际上不需要恢复块之间的对齐。 我们只需要恢复原始排列中的多重值集。 由于最终数组是排列的两个串联副本，因此隐藏排列中的每个值在完整的 nm 长度数组中恰好出现两次，因此在组合的所有块中恰好出现两次。 

因此，忽略结构并纯粹关注频率足以重建排列值。 从 1 到 nm/2 的每个整数在 nm 值的输入多重集中恰好出现两次。 这意味着我们可以简单地收集所有数字，计算频率，并只取每个数字一次。 

唯一剩下的微妙之处是确保我们输出长度为 nm/2 的有效排列。 由于输入保证解的存在，因此从 1 到 nm/2 的每个数字必须在 nm 条目中恰好出现两次。

 ## 算法演练

 我们将问题简化为所有条目的频率恢复。

1. 读取所有 n 个块并将它们展平为一个 nm 数字列表。 这是有效的，因为块只是双倍数组的段的排列，因此展平保留了完整的多重值集。 
2. 使用哈希映射或数组计算每个数字的出现次数。 由于值介于 1 到 nm/2 之间，因此我们可以使用大小为 nm/2 + 1 的数组进行直接索引。 
3. 通过迭代从 1 到 nm/2 的所有值并将每个值恰好附加一次来构造答案。 正确性来自于每个值在加倍排列中恰好出现两次，因此频率必须恰好为 2。 
4. 输出结果列表。 

重要的推理步骤是块结构永远不会影响全局频率。 尽管拆分可能会将相同值的部分放入不同的块中，但它不会改变每个值整体出现的次数。 

### 为什么它有效

 长度 L = nm/2 的隐藏排列 P 包含从 1 到 L 的每个整数恰好一次。 构造的数组是 P 与 P 串联，因此每个整数在整个数组中恰好出现两次。 分成块和排列块不会改变值的多重集。 因此，对所有块的频率进行计数可以恢复多重集，其中每个有效值恰好出现两次。 选择每个值一次即可重建与过程一致的有效排列 P。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    total = n * m
    arr = []
    
    for _ in range(n):
        arr.extend(map(int, input().split()))
    
    # values are in [1, total//2]
    L = total // 2
    freq = [0] * (L + 1)
    
    for x in arr:
        freq[x] += 1
    
    res = []
    for i in range(1, L + 1):
        if freq[i] >= 1:
            res.append(i)
    
    print(*res)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该代码首先将所有输入块展平为单个数组，因为块排序与频率推理无关。 然后它计算每个整数的出现次数。 最后，它通过选择每个值一次来构造排列。 

一个微妙的实现细节是假设值范围恰好为 1 到 nm/2。 这是根据该长度的排列和复制步骤的定义得出的。 我们不需要验证频率恰好等于二，因为问题保证了可解性。 

## 工作示例

 ### 示例 1

 输入：

 n = 2，m = 2

 块：

 [1, 2]

 [3, 4]

 展平数组：[1,2,3,4]

 | 步骤| 频率状态| 输出构造 |
 | --- | --- | --- |
 | 读完后| 1:1、2:1、3:1、4:1 | - |
 | 建筑答案| 每个值取一次| [1,2,3,4]|

 这对应于 L = 4，因此恢复的排列只是所有值 1 到 4。 

这证实了当排列在值空间中是连续的时，可以根据频率的存在立即进行重建。 

### 示例 2

 输入：

 n = 3，m = 2

 块：

 [5, 6]

 [1, 2]

 [3, 4]

 展平数组：[5,6,1,2,3,4]

 | 步骤| 频率状态| 输出构造 |
 | --- | --- | --- |
 | 读完后| 所有值都出现一次 | - |
 | 建筑答案| 选择每个值一次 | [1, 2, 3, 4, 5, 6] |

 尽管块排序是任意的，重构的排列仍然有效，因为只要排列的任何排序都包含所有值一次，就可以工作。 

这表明块排列对底层多重集的可重构性没有影响。

 ## 边缘情况

 一种特殊情况是块重复相同的序列。 在这种情况下，简单的重建可能会尝试将相同的块匹配为段的配对副本，但这是不必要的。 该算法只是简单地计算全局的出现次数，因此块级别的重复没有效果。 

另一种情况是，当值以没有块与原始排列段相似的方式跨块交错时。 即便如此，平坦化仍然保留了频率，并且重建仍然正确，因为只有多重性才重要。 

最后一种情况是 m = 1，其中每个块都是一个数字。 该解决方案仍然有效，因为展平会产生完整的多重集，并且选择每个数字一次会立即重建排列。
