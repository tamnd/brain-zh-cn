 ## 解决方案
 ## 问题理解

 我们得到一个数组，其中每个值代表一种怪物类型。 当剩余的每种类型出现的次数与其自身值完全相同时，该级别被认为是平衡的。 例如，如果输入`4`存在，那么它必须恰好出现四次。 完全删除的类型没有任何要求。 任务是删除尽可能少的元素，使得剩余的多重集满足这个规则。 

输入包括`n`怪物类型，每种类型之间`0`和`n`。 由于只允许删除，因此我们永远不会增加任何值的频率。 对于每种类型，我们要么保持其频率与其值完全相同，要么删除该类型的所有副本。 

界限`n ≤ 1000`很小，所以即使是二次解也能轻松拟合。 尽管如此，问题的结构允许在计算频率后得到线性解。 唯一重要的信息是每个值出现的次数，而不是数组的顺序。 

有几种情况很容易处理不当。 

假设一个值出现的次数少于它本身的值。```
3
2 2 1
```类型`2`出现两次，所以可以留下来。 类型`1`出现一次，所以也可以留下来。 不需要删除，答案是`0`。 删除计数不大于该值的任何值的粗心方法将错误地删除类型`1`。 

另一种情况是某个值出现的次数太少。```
2
3 3
```类型`3`仅出现两次，但需要保留三份副本。 由于不可能添加元素，因此必须删除两个副本。 答案是`2`。 

一个特殊的值是`0`。```
4
0 0 1 1
```如果输入`0`仍然存在，它需要恰好出现零次，这意味着它根本不可能存在。 必须删除两个零。 类型`1`appears twice, so one copy is removed and one is kept. 答案是`3`。 忘记零需要零出现会导致错误的答案。 

## 方法

 最直接的暴力想法是独立处理每个不同的值。 对于一个值`x`随频率`f`，我们可以尝试所有的可能性并确定保留一些副本是否合法。 由于唯一有效的剩余频率恰好是`x`，只有两个有意义的选择。 如果`f ≥ x`，我们可以保留`x`复制并删除`f - x`，否则我们必须删除所有`f`副本。 

组织性较差的暴力实现可能会重复扫描数组中的每个值，从而导致大约`n²`最坏情况下的操作。 和`n = 1000`，这仍然有效，但它做了不必要的工作。 

关键的观察是不同的价值观不会相互作用。 类型的决定`5`对类型的决定没有影响`7`。 一旦我们知道每个值的频率，每种类型的最小删除次数就可以独立确定。 

如果一个值`x`出现`f`有时会出现两种情况。 什么时候`f < x`，保持类型是不可能的，因为我们无法创建额外的副本，所以所有`f`出现的情况被删除。 什么时候`f ≥ x`，我们准确地保留`x`复制并删除多余的`f - x`。 

频率图提供了所需的所有信息，并且每个不同的值都被处理一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 已接受 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 读取数组并计算每个值出现的次数。 
2. 将答案初始化为零。 
3. 对于每个不同的值`x`随频率`f`，确定是否存在足够的副本来保留该类型。 
4. 如果`f < x`， 添加`f`答案，因为必须删除该值的所有副本。 

如果值少于`x`出现，并且不允许添加元素。 
5. 否则，添加`f - x`到答案，因为正是`x`应保留副本。 

超出所需数量的任何副本都是不必要的，必须删除。 
6. 输出累计答案。 

### 为什么它有效

 每个值都完全独立于其他值。 对于一个类型`x`，唯一合法的剩余频率是`x`。 如果当前频率`f`小于`x`，不存在包含该类型的有效配置，因此删除所有副本是不可避免的。 如果`f`至少是`x`，准确地保持`x`副本可以最大限度地减少删除，因为任何较小的正频率都是无效的，任何较大的频率都会违反条件。 将每个值的最优成本相加即可得到全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# solution
from collections import Counter

n = int(input())
a = list(map(int, input().split()))

cnt = Counter(a)

ans = 0

for x, f in cnt.items():
    if f < x:
        ans += f
    else:
        ans += f - x

print(ans)
```第一部分使用以下方法计算每种怪物类型的频率`Counter`。 由于数组的顺序无关，因此频率包含所有必要的信息。 

变量`ans`存储删除的总数。 对于每个值`x`，代码区分了算法中描述的两种情况。 

什么时候`f < x`，由于达到频率，每个出现的情况都被删除`x`是不可能的。 否则，正好`x`保留副本，多余的`f - x`副本被删除。 

价值`0`是自动处理的。 自从`f < 0`从来都不是真的，第二个分支有贡献`f - 0 = f`，意味着所有零都被删除，这与定义匹配。

# helper: run solution on input string, return output string
import sys, io
from collections import Counter

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))

    cnt = Counter(a)

    ans = 0
    for x, f in cnt.items():
        if f < x:
            ans += f
        else:
            ans += f - x

 ## 边缘情况

 考虑输入```
2
3 3
```频率表仅包含值`3`随频率`2`。 自从`2 < 3`, 保持类型`3`是不可能的。 该算法添加`2`到答案和输出`2`。 任何保留这些怪物的尝试都会留下频率`2`，这违反了要求。 

现在考虑```
4
0 0 1 1
```频率是`0 → 2`和`1 → 2`。 为了价值`0`，算法贡献`2 - 0 = 2`，删除所有零。 为了价值`1`，它贡献`2 - 1 = 1`，仅保留一份副本。 最终的答案是`3`。 

最后，考虑```
3
2 2 1
```类型`2`出现两次并输入`1`出现一次。 两个频率都已经与它们的值匹配，因此每个频率都贡献零缺失。 答案是`0`，确认该算法使已经平衡的配置保持不变。
