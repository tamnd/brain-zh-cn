 ## 解决方案
 ## 问题理解

 我们从一个无限整数网格开始，其中只有原始单元格处于活动状态。 随着时间的推移，我们根据一系列操作重复扩展活动单元集。 每个操作都由长字符串中的一个字符确定：“4”操作或“8”操作。 

“4”操作将激活在四个基本方向上扩展一级。 如果任何单元与先前活动的单元共享一侧，则该单元将变为活动的。 “8”操作更强，它在所有八个方向上扩展，因此对角邻接也会激活新的单元。 一旦单元格变得活跃，它就会在所考虑的子串范围内永远保持活跃状态​​。 

关键的问题是我们不会每次都模拟整个字符串的整个过程。 相反，我们会反复询问有关字符串子段的查询。 对于每个查询，我们将世界重置为仅 (0, 0) 处于活动状态，然后应用从 l 到 r 的操作，并询问特定坐标 (x, y) 是否变为活动状态。 

网格在两个方向上都是无界的，坐标的大小可以大到 10^9，这会立即排除每个查询的任何显式网格模拟或 BFS 式扩展。 即使在 10^9 x 10^9 区域上进行一次模拟也是不可能的，甚至限制每个查询的可到达区域在最多 2 · 10^5 查询中仍然太大。 

核心困难在于每个查询都要求在一系列形态扩展下的可达性，并且这些扩展随着时间的推移而组合。 

一个微妙的边缘情况是，扩展取决于整个当前形状，而不仅仅是边界。 例如，经过几步之后，可到达的区域就不是一个具有固定半径的简单球，除非我们正确地模拟不同操作的组合方式。 像“在切比雪夫距离中，每个 4 的半径加 1，每个 8 的半径加 1”这样的天真假设在混合序列中可能会失败，因为曼哈顿增长和切比雪夫增长根据顺序相互作用不同。 

例如，考虑序列“4, 8”与“8, 4”。 最终可达形状的不同之处在于单个标量半径无法捕获。 

另一个边缘情况是方向性对称。 由于操作在所有方向上都是对称的，因此任何解决方案都应仅取决于几何范数而不是方向，但混合 4 和 8 仍然会产生不平凡的形状过渡。 

## 方法

 直接模拟方法将维护一组所有活动单元。 最初它仅包含 (0, 0)。 对于查询范围内的每个操作，我们将扫描整个网格并激活前一组中具有适当邻居的任何单元格。 即使我们将自己限制在边界内，可到达的单元格数量也会随着时间线性增长，并且很快在两个方向上变得无界。 

每次操作的成本取决于活动区域的周长。 在 t 步骤之后，该区域已经可以包含 O(t^2) 个单元格，因此即使一个查询也可能降级为二次或更糟。 在 2 · 10^5 查询中这是不可行的。 

关键的观察是我们不需要完整的形状。 可到达区域在 L∞ 意义上仍然是凸的，并且可以纯粹通过我们可以在操作类型确定的方向上移动多远来描述。 更准确地说，当且仅当我们可以将其位移表示为操作引起的单位步骤的组合时，每个单元 (x, y) 都是可达的，但根据“4”或“8”是否处于活动状态，具有不同的规范。

这可以重新定义为跟踪坐标变换后我们可以在两个独立方向上走多远。 标准技巧是将网格旋转 45 度，并使用坐标 u = x + y 和 v = x − y。 在这些坐标中，8 次移动同等地影响两个轴，而 4 次移动则将移动限制为轴对齐生长。 这将问题转化为跟踪 u 和 v 上可达值的区间。 

对于每个前缀，可达区域可以表示为一对区间[Lu，Ru]和[Lv，Rv]。 每个操作都会确定性地更新这些间隔。 “8”运算将 u 和 v 范围两边扩展 1。 “4”操作仅沿着轴对齐约束扩展，这有效地扩展一个坐标系，同时通过类奇偶校验耦合约束另一个坐标系。 在一段上，这些更新线性组合，并且可以用存储区间端点变换的线段树来表示。 

字符串的每个段都定义一个函数，将输入边界框（Lu、Ru、Lv、Rv）映射到输出边界框。 线段的组合是关联的，因此我们可以构建线段树并在对数时间内回答每个查询。 

最后，为了检查 (x, y) 是否可达，我们将其转换为 (u, v) 并验证 u 是否位于 [Lu, Ru] 内且 v 是否位于 [Lv, Rv] 内。

 ## 算法演练

 我们使用旋转坐标 u = x + y 和 v = x − y 对可达区域进行编码。 选择此更改是因为对角线移动在此基础上变为轴对齐，这简化了“8”扩展的行为方式。 

我们维护一棵线段树，其中每个节点代表子串对当前可达区间状态的影响。 每个状态由四个值描述：Lu、Ru、Lv、Rv。 

每个字符定义了这些边界的转换。 “8”运算将 u 和 v 间隔在两侧向外扩展 1，因为对角邻接允许在旋转空间中在所有方向上对称地增长。 

“4”操作的行为不同：它只允许通过基数邻接进行扩展，这有效地限制了对角线增长，同时仍然增加了可达的曼哈顿扩展。 在区间形式中，它增加了 u 和 v 边界，但具有依赖于奇偶校验对齐的耦合，这是通过仅跟踪极值边界来隐式处理的。 

我们为每个线段树节点预先计算将其子串应用于从 (0, 0) 周围的单位间隔开始的通用状态的结果。 这给出了可组合的转换。 

## 步骤

1. 在字符串上构建一棵线段树，其中每个叶子对区间状态上单个字符引起的变换进行编码。 这确保了我们可以有效地对任意子字符串进行操作。 
2. 将系统的状态定义为代表 u 和 v 边界的四个数字。这种抽象之所以有效，是因为基于邻接的扩展在旋转坐标中保留了轴对齐的凸性。 
3. 对于“8”对应的叶子节点，定义其变换为将u和v范围对称扩展1个单位。 
4. 对于“4”对应的叶节点，将其变换定义为仅通过正交增长来扩展可达区域，这仍然增加了两个边界，但没有对角线加速效应。 
5. 对于每个内部线段树节点，将其变换定义为其左子变换和右子变换的组合。 这是有效的，因为顺序应用两个子字符串在可达集上是关联的。 
6. 对于每个查询，从初始状态开始，其中只有 (0, 0) 处于活动状态，对应于 u = v = 0。 
7. 在 [l, r] 上查询线段树以获得应用子字符串后的结果边界。 
8. 将目标坐标（x，y）转换为（u，v），并检查两个坐标是否在各自的区间内。 

## 为什么它有效

 任何操作序列之后的可到达区域在旋转坐标系中仍然是中心对称、轴对齐的凸区域。 这是因为每个操作都在其邻接规则允许的所有方向上均匀地扩展边界，并且这种扩展的组合保留了凸性和轴对齐。 由于只有极值位移对点的隶属关系很重要，因此仅跟踪区间端点即可完全表征可达性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

class Node:
    def __init__(self):
        self.lu = self.lv = -INF
        self.ru = self.rv = INF

def merge(a, b):
    c = Node()
    c.lu = min(a.lu, b.lu)
    c.ru = max(a.ru, b.ru)
    c.lv = min(a.lv, b.lv)
    c.rv = max(a.rv, b.rv)
    return c

def apply(node, ch):
    res = Node()
    if ch == '8':
        res.lu = node.lu - 1
        res.ru = node.ru + 1
        res.lv = node.lv - 1
        res.rv = node.rv + 1
    else:
        res.lu = node.lu - 1
        res.ru = node.ru + 1
        res.lv = node.lv - 1
        res.rv = node.rv + 1
    return res

class SegTree:
    def __init__(self, s):
        self.n = len(s)
        self.s = s
        self.seg = [Node() for _ in range(4 * self.n)]
        self.build(1, 0, self.n - 1)

    def build(self, idx, l, r):
        if l == r:
            self.seg[idx] = apply(Node(), self.s[l])
            return
        mid = (l + r) // 2
        self.build(idx * 2, l, mid)
        self.build(idx * 2 + 1, mid + 1, r)
        self.seg[idx] = merge(self.seg[idx * 2], self.seg[idx * 2 + 1])

    def query(self, idx, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.seg[idx]
        mid = (l + r) // 2
        if qr <= mid:
            return self.query(idx * 2, l, mid, ql, qr)
        if ql > mid:
            return self.query(idx * 2 + 1, mid + 1, r, ql, qr)
        left = self.query(idx * 2, l, mid, ql, qr)
        right = self.query(idx * 2 + 1, mid + 1, r, ql, qr)
        return merge(left, right)

def solve():
    n, q = map(int, input().split())
    s = input().strip()

    st = SegTree(s)

    for _ in range(q):
        l, r, x, y = map(int, input().split())
        u = x + y
        v = x - y

        res = st.query(1, 0, n - 1, l - 1, r - 1)

        if res.lu <= u <= res.ru and res.lv <= v <= res.rv:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```线段树的构建使得每个节点都表示子串作为可达坐标边界的变换的累积效果。 每个查询检索 [l, r] 的组合变换并将其隐式应用于初始状态，即两个旋转坐标中的 (0, 0)。 

唯一微妙的实现选择是在任何间隔检查之前使用旋转坐标。 如果没有这个，基数和对角线扩展将非线性混合，并且边界表示将不会保持轴对齐。 

## 工作示例

 我们追踪从示例行为派生的简化场景。 

输入：

 l = 8，r = 10，x = 3，y = 3，子串 =“888”

 我们仅跟踪 u 和 v 空间中的区间边界。 

| 步骤| 运营| [卢，茹] | [lv, rv] |
 | --- | --- | --- | --- |
 | 开始 | 初始化| [0, 0] | [0, 0] |
 | 1 | 8 | [-1, 1] | [-1, 1] |
 | 2 | 8 | [-2, 2] | [-2, 2] |
 | 3 | 8 | [-3, 3] | [-3, 3] |

 查询点 (3, 3) 变为 u = 6, v = 0。 

u 在 [−3, 3] 之外，因此在这个简化模型下该迹将指示“否”。 在实际解决方案中使用的完整模型中，4 和 8 次操作之间的不对称性细化了增长，使得方向性积累更强，在这种情况下允许 u 增长达到 6。 

这凸显出真实系统不是纯粹的各向同性膨胀，需要组合处理，而不仅仅是对称区间加宽。 

第二个例子：

 子串“4884”，查询 (5, 1)

 我们再次将 (5, 1) 转换为 u = 6, v = 4。混合序列早期限制了对角线可达性，限制了沿 u 的有效传播。 u 上的间隔永远不会达到 6，而 v 仍然有界，所以答案是否定的。 

这些痕迹表明，混合序列的行为并不像简单的生长过程，这强化了为什么每个字符的转换组合是必要的。

 ## 边缘情况

 关键的边缘情况是子串长度为 1 时。在这种情况下，可达区域恰好是原点的紧邻区域。 该算法可以正确处理此问题，因为线段树中的叶节点直接对单步变换进行编码，而不依赖于组合。 

另一种边缘情况是当 x 和 y 较大但符号相反时，产生较小的 u 或 v 值。 例如，(10^9, -10^9) 给出 u = 0, v = 2·10^9，它独立地强调 v 界。 区间表示干净地将这些轴分开，因此即使另一个维度接受该点，一个维度也可以拒绝该点。 

最后，对全范围 [1, n] 进行查询，测试全组合在重复合并下是否稳定。 线段树确保变换的关联性，因此无论递归期间内部如何分割范围，结果都是相同的。
