 ## 解决方案
 ## 问题理解

 我们有一个游戏，里面有一堆石头。 两名玩家轮流行动，爱丽丝先行动。 在每一回合中，玩家都会查看当前的石子数量（例如 x），并根据奇偶性规则移除石子。 

如果 x 是奇数，则玩家别无选择，必须恰好移走一颗石子，将石子堆减少到 x−1。 如果 x 是偶数，玩家有两个选择：移走一颗石子，或者移走恰好一半的石子，分别将游戏移至 x−1 或 x/2。 在回合开始时面对零子的玩家会立即失败，因为无法移动。 

任务是确定对于每个初始 n，假设两个玩家都发挥最佳状态，Alice 是否强制获胜。 

约束条件最多为一百万个测试用例，石头数最多为一百万个。 这立即排除了游戏的任何测试模拟。 即使每个测试用例进行线性扫描也太慢了。 任何可接受的解决方案都必须依赖于高达 10^6 的预计算或获胜状态的封闭形式表征。 

朴素推理的一个微妙的失败案例来自于假设偶数状态和奇数状态之间的对称性或假设“取一半”总是强的。 例如，当 n = 2 时，两次移动都导致 1，即下一个玩家获胜，因此 2 表示失败。 当 n = 4 时，取一半导致 2（输），因此 4 变为获胜，即使减去 1 导致 3（获胜）。 这表明，即使是头寸也需要同时考虑两种移动选项，而不仅仅是较大的减少。 

另一个陷阱是假设所有奇怪的位置都输了，因为它们只允许一次移动。 对于像 1 和 3 这样的小值来说这是错误的，因为强制链会导致对手陷入失败状态。 

## 方法

 解决这个游戏的直接方法是将每个整数x视为一个游戏状态并定义它是赢还是输。 从状态 x 开始，我们查看一步中可到达的所有状态，如果这些可到达状态中至少有一个失败，则将 x 标记为获胜。 

这自然会导致动态规划公式。 设 dp[x] 表示当前玩家是否有剩余 x 颗棋子的获胜策略。 根据规则，如果 x 是奇数，则唯一的转换是到 x−1。 如果 x 是偶数，则转移到 x−1 和 x/2。 这给出了 dp[x] 仅取决于较小索引的递归，因此我们可以计算从 0 到 n 的 dp。 

暴力 DP 简单且正确，但如果每次重新计算，其成本为每个测试用例的 O(n)，这对于 10^6 次查询来说是不可能的。 即使计算一次，回答查询也是 O(1)，所以真正的问题是确保我们预先计算一次直到最大 n。 

关键的结构观察是，在一个小的前缀之后，游戏稳定为一个简单的模式。 超过阈值的奇数状态会持续失败，因为它们唯一的举动会直接导致获胜的偶数状态。 即使超过阈值的国家也会持续获胜，因为它们的两次转变中至少有一次总是会导致失败。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的完整重新计算 | 每个查询 O(n) | O(1) | O(1) | 太慢了 |
 | DP 预计算至 max n | 总共 O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过计算一次从 0 到 1e6 的 dp 值来构建解决方案。

1. 将 dp[0] 初始化为失败，因为空堆中不存在任何移动。 
2. 对于从 1 到 max_n 的每个 x，根据先前计算的状态确定 dp[x]。 
3. 如果 x 是奇数，则唯一的移动是 x−1，因此 dp[x] 变为 dp[x−1] 的相反数。 这直接反映了当且仅当强制移动使对手处于失败位置时当前玩家获胜。 
4. 如果 x 是偶数，则评估选项 x−1 和 x/2。 如果其中任何一个导致失败状态，则将 dp[x] 标记为获胜状态。 这编码了最佳玩法，因为玩家选择了最佳的可用动作。 
5. 填充 dp 后，通过打印 dp[n] 是否为真来回答每个查询，时间复​​杂度为 O(1)。 

这样做的原因是 dp[x] 准确地编码了有限公平游戏中获胜位置的定义：如果一个状态至少有一次移动到失败状态，则该状态获胜；如果所有移动都导致获胜状态，则该状态失败。 由于每次转换都会减少 x，因此 DP 是有根据的并且涵盖了所有依赖关系。 

通过检查递归性，可以得出更深层次的结构简化：奇数状态从 x−1 传播值，而偶数状态则受到 x−1 和 x/2 的影响。 在小值之后，它很快就会崩溃为稳定的模式，但 DP 公式是保证正确性的最干净的方法。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXN = 10**6

dp = [False] * (MAXN + 1)
dp[0] = False

for x in range(1, MAXN + 1):
    if x % 2 == 1:
        dp[x] = not dp[x - 1]
    else:
        lose = False
        if not dp[x - 1]:
            lose = True
        if not dp[x // 2]:
            lose = True
        dp[x] = lose

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append("Alice" if dp[n] else "Bob")
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```DP 数组在全局范围内计算一次，因此每个测试用例都会在恒定时间内得到回答。 转换逻辑直接遵循移动规则：奇数状态有一个强制转换，而偶数状态检查两种可能的移动并选择是否有任何移动导致失败。 

一个常见的实现错误是在查询循环内重新计算 dp，这在 10^6 个输入下会立即超时。 另一个微妙的问题是忘记了如果任何一步都输了，而不是所有一步都赢了，则 dp[x] 应该为真。

### 示例 2：n = 6

 | x| 平价 | x−1 | x−1 | x/2 | 结果 |
 | --- | --- | --- | --- | --- |

这两个例子展示了如何通过减半移动来保存偶数状态，而奇数状态则严格由前一个位置决定。

 ## 边缘情况

 当n = 1时，该状态立即获胜，因为强制移动导致0，使对手无法移动。 

对于 n = 2，该状态正在失败，因为两种可能的举动都会导致 n = 1，即获胜，因此无法逃脱。 

当 n = 3 时，强制移动到 2，即失败，使得 3 获胜，尽管它是奇数。 

对于较大的奇数值，例如 n = 5，强制移动到 4，即获胜，因此位置会变成失败。 这表明一旦他们唯一的孩子变得稳定并获胜，奇怪的国家就不再受到青睐。 

对于诸如 n = 4 之类的偶数值，减半移动到 2，即失败，因此 4 变为获胜，即使减去 1 会导致获胜状态。
