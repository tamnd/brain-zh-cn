 ## 解决方案
 ## 问题理解

 我们被要求构建一个小“程序文件”，该文件由以下内容组成：$n$线。 每行都有非常严格的格式：它以大括号内写的数字开头，然后是一系列逗号。 逗号不仅仅是标点符号，它们还代表“节拍”，每个逗号贡献一个固定的量，具体取决于该行大括号内的数字。 

形式上，每行看起来像`{x},,,,,,`在哪里$x$是我们选择的整数。 的价值$x$必须是 2 的幂，所有$x$不同行之间的值必须不同，并且每一行必须至少包含一个逗号。 

如果一行使用值$x$并包含$c$逗号，那么该行贡献$c \cdot x$至总分。 对所有行求和必须准确给出$k$。 同时每行都有严格的长度限制$m$，所以文本表示`{x}`加逗号必须在该限制之内。 

因此，任务是同时决定两件事：将两个不同的权力分配给$n$行，以及每行应有多少个逗号，以便加权和等于$k$并且每条线都保持在长度限制内。 

限制很小：$n \le 20$和$k \le 100$。 这立即表明指数或 DP 式推理是可行的，因为任何构造都受到非常小的目标和的限制。 

一个关键的结构限制来自于每条线必须至少做出贡献$x$，因为每行至少有一个逗号。 这意味着选择的总和$x$值不能超过$k$，否则即使是最小可能的构造也已经超调了。 

第二个微妙的限制是所有$x$值必须是两个下的不同幂$2^{30}$，所以我们有效地从集合中选择不同的值$\{1,2,4,8,\dots\}$。 

如果我们过早地选择 2 的大幂，就会立即出现幼稚的失败案例。 例如，如果$k=10$我们选择$x=32$对于一行来说，即使是一个逗号也贡献了 32 个，这已经超过了目标总数。 任何忽略这个下限约束的方法都会立即崩溃。 

另一个失败案例来自于忽略字符串长度。 即使数字分配有效，`{x}`对于更大的 2 的幂会增加数字长度，并且长逗号序列可能会溢出每行限制$m$。 

## 方法

 一个蛮力的想法是尝试所有的分配$n$两个不同的幂，然后尝试所有方法来分配逗号计数以匹配$k$。 对于每一个选择$x_i$，我们将枚举所有整数组合$k$进入$n$加权部分，其中部分$i$贡献的倍数$x_i$。 发行版数量增长极快，甚至$k \le 100$，朴素的枚举变得昂贵，因为每行可以贡献$O(k)$可能的逗号计数，导致粗略$O(k^n)$行为。 

一旦我们修正了“两个唯一有用的幂都是小幂”这一观察结果，该结构就会变得更加易于管理。 由于每行至少贡献$x_i$，总和必须恰好是$k \le 100$， 任何$x_i > 100$永远不能出现。 这会将候选集折叠为最多七个值：$1,2,4,8,16,32,64$。 这减少了从组合选择二的任意幂到选择这些小值的子集的问题。 

一旦$x_i$值是固定的，剩下的任务就是将剩余预算分配到这些权重的倍数中。 这是一个容量很小的有界背包，里面的每件物品$i$可以选择多次，对应于额外的逗号。 

我们还需要强制每行的字符串长度约束，这限制了每个选定的可以放置多少个逗号$x_i$。 

这自然会导致对线和总和的动态规划公式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| 指数为$n$和$k$| O(1) | O(1) | 太慢了 |
 | DP 线和总和 |$O(n \cdot k \cdot m)$|$O(n \cdot k)$| 已接受 |

 ## 算法演练

 我们固定候选值$x$作为二的最小幂：$1,2,4,8,\dots$。 自从$k \le 100$，我们只保留那些$\le k$，并且我们准确地选择$n$与此列表中不同的。 如果我们不能选择$n$其最小可能贡献不超过的值$k$，我们立即失败。 

然后，我们根据已处理的行数以及构建的总节拍和来定义动态编程状态。 

1. 我们构建一个由 2 人开始的候选人权力列表$1$，一旦值超过就停止$k$，并且我们确保我们至少有$n$候选人。 如果不是，答案是不可能的。 
2.我们选择第一个$n$候选人作为$x_1, x_2, \dots, x_n$。 这种选择是安全的，因为任何更大的值只会使可行性变得更加困难。 
3. 对于每行$i$，我们计算它可以包含多少个逗号。 每行都有固定的字符开销`{`,`}`，以及数字$x_i$，所以我们计算允许的最大逗号：$$c_i \le m - (\text{length of } \{x_i\})$$还有$c_i \ge 1$。 
4. 我们定义一个DP，其中$dp[i][s]$表示是否可以构建第一个$i$具有总节拍总和的行$s$。 最初$dp[0][0] = true$。 
5. 对于每行$i$，我们尝试所有可能的逗号计数$c_i$在允许的范围内。 每一个选择都有贡献$c_i \cdot x_i$到总和，所以我们转换：$$dp[i][s + c_i x_i] = true$$6.填写DP后，我们检查是否$dp[n][k]$是可达的。 如果不是，输出-1。 
7. 如果可达，我们重建以下选择：$c_i$通过回溯 DP 表并以所需格式输出每一行。 

其有效的关键原因是，除了全局总和约束之外，每条线的行为都是独立的。 一次$x_i$是固定的，唯一的相互作用是总预算如何$k$跨线划分，这正是 DP 捕获的内容。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())

    # generate candidate powers of two
    xs = []
    v = 1
    while v <= k and len(xs) < n:
        xs.append(v)
        v *= 2

    if len(xs) < n:
        print(-1)
        return

    xs = xs[:n]

    # precompute max commas per line
    maxc = []
    base_len = []
    for x in xs:
        l = len(str(x)) + 2  # braces + digits
        base_len.append(l)
        maxc.append(m - l)

        if maxc[-1] < 1:
            print(-1)
            return

    # dp[i][s] = reachable
    dp = [[False] * (k + 1) for _ in range(n + 1)]
    par = [[None] * (k + 1) for _ in range(n + 1)]

    dp[0][0] = True

    for i in range(n):
        x = xs[i]
        for s in range(k + 1):
            if not dp[i][s]:
                continue
            for c in range(1, maxc[i] + 1):
                ns = s + c * x
                if ns <= k and not dp[i + 1][ns]:
                    dp[i + 1][ns] = True
                    par[i + 1][ns] = (s, c)

    if not dp[n][k]:
        print(-1)
        return

    res_c = [0] * n
    cur = k
    for i in range(n, 0, -1):
        ps, c = par[i][cur]
        res_c[i - 1] = c
        cur = ps

    for i in range(n):
        x = xs[i]
        print("{" + str(x) + "}" + "," * res_c[i])

if __name__ == "__main__":
    solve()
```实施首先将注意力限制在二的小幂上，因为由于强制的至少一个逗号规则，任何更大的值都会立即超出预算。 

DP表是逐行构建的，其中每一行对应一行，每次转换对应选择多个逗号。 父指针表存储先前的总和和选择的逗号计数，从而允许重建最终字符串。 

一个微妙的细节是每行 1 个逗号的下限。 这是通过从以下位置开始直接在 DP 循环中强制执行的：`c = 1`，因此不会引入无效的空行状态。

 ## 边缘情况

 一种边缘情况是当$n$相对于太大$k$。 例如，如果$n=20$和$k=3$，甚至选择$x_i=1$对于所有行，已经强制要求最小总和为 20，这超出了目标。 该算法很早就发现了这一点，因为 DP 永远不会达到$k$。 

另一种边缘情况是当一行的长度不足以放置一个逗号时。 如果`{x}`已经超过$m-1$， 然后`maxc[i] < 1`，算法立即拒绝该配置。 这可以防止违反格式限制的无效输出。 

最后的边缘情况是仅可以使用二的幂的子集。 例如，如果$k=5$，选择$x=1,2,4$是有效的，但添加$8$会立即破坏可行性。 该结构通过将候选值限制为不超过的值来避免这种情况$k$，确保不会发生不可避免的溢出。
