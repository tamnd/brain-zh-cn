 ## 解决方案
 ## 问题理解

 我们从一个二进制字符串开始，该序列仅由`0`和`1`。 允许两种本地编辑操作，每种操作在两个相邻位置之间插入一个额外的字符。 

第一个操作仅适用于一对相等的邻居。 如果我们看到`00`，我们可以插入一个`1`他们之间，如果我们看到`11`，我们可以插入一个`0`。 此操作依赖于具有“同质”边界，并且总是引入相反的字符，创建新的交替点。 

第二个操作更加宽松。 如果我们看到`01`或者`10`，我们可以插入`0`或者`1`它们之间，因此插入的字符不受约束。 

我们需要准确地执行`a`第一类操作和`b`第二种类型的操作，可以以任何顺序并且在它们适用的任何位置进行。 这样做之后，根据位置和插入位的选择，可以产生许多不同的最终字符串。 任务是计算可以获得多少个不同的最终字符串，取模`10^9 + 7`。 如果在某个时刻无法执行所需数量的操作，则答案为零。 

输入大小高达一百万，这会立即排除任何插入或动态字符串构造的模拟。 每个操作都会改变字符串的长度，简单的分支会呈指数级爆炸。 任何解决方案都必须仅适用于压缩的结构信息，几乎肯定基于游程或邻接结构，而不是显式字符串。 

一个微妙的失败案例来自于假设操作始终可以独立应用。 例如，从`00`，如果我们应用第一个操作，我们会得到`010`。 之后，原来的`00`邻接性消失，因此不能在同一位置应用另一个第一类操作。 这意味着可行性取决于不断发展的结构，而不仅仅是初始计数。 

另一种边缘情况是当字符串交替时，例如`0101`。 这里最初不存在相等的相邻对，因此零第一类型操作是不可能的，除非通过第二类型插入间接创建新的相等邻接。 将操作视为独立的天真计数将错误地允许`a > 0`在这种情况下。 

## 方法

 暴力解释将过程视为状态图。 每个状态都是一个二进制字符串，每个边对应于在允许的位置之一插入一个字符。 我们将递归地尝试所有可能的序列`a + b`操作并将结果字符串收集到一个集合中。 这在概念上是正确的，因为它精确地探索了定义的转换系统。 

然而，分支因子很大。 每个操作都可以应用于多个位置，并且每个应用都会更改可用的未来移动。 状态数量呈指数增长`n + a + b`，在最坏的情况下最多可进行 200 万次操作。 即使是一个很小的例子也是不可行的。 

关键的观察结果是，该过程并不真正依赖于精确的字符串几何形状，而仅依赖于邻接类型：相等对和不相等对。 插入仅影响局部邻接结构，唯一有意义的全局状态信息是每种类型存在多少个位置以及它们如何演化。 

更深入的简化是跟踪字符串中“块”的数量，其中块是相同字符的最大运行次数。 每个字符串都由其块结构完整描述。 第一个操作分割一个块，将块的数量增加一个。 第二个操作细化了边界，但不会以根本独立的方式改变块计数：它仅在策略性放置时才能创建新块，但其组合效果在边界之间是对称的。 

这将问题简化为计算我们可以在不断变化的边界上分配操作的方式，同时跟踪创建了多少新边界。 最终计数仅取决于每种操作类型引入了多少有效“选择”，这成为动态增加仓位的组合计数问题。

 ## 算法演练

 我们根据运行重新制定流程。 让初始字符串分解为最大的相等字符块。 让`k`是此类块的数量。 

一个关键的结构事实是，每个操作都在块之间或块内部的边界上进行本地操作，但从不需要超出存在多少个边界的全局意识。 

我们跟踪有效插入点的数量如何通过单个聚合参数隐式演变：“活动位置”的数量，最初是`n - 1`。 

### 步骤

 1. 计算字符串中的转换次数，即索引数`i`这样`S[i] != S[i+1]`。 调用这个值`t`。 

这给出了不同字符之间的初始边界数，这些边界正是可以立即应用第二类操作的地方。 
2. 计算相等相邻对的个数，即`(n - 1 - t)`。 打电话给这个`e`。 

这些对应于第一类操作适用的初始位置。 
3. 将第一类操作解释为增加边界数量。 每次插入`00 -> 010`或者`11 -> 101`恰好引入了一个新的转换边界，因此每个第一类型操作都会将可用异构位置的数量增加一个。 

这意味着执行后`a`第一类操作，可用边界位置的数量变为`t + a`。 
4、第二类操作选择任意类型的边界，自由插入字符。 每个此类操作都会增加长度，从而将潜在插入点的数量增加一，而不管字符选择如何。 因此，每个第二类型操作基于当前插入位置的数量贡献组合因子。 

插入位置的数量线性变化：

 初始位置 =`n - 1`，经过 k 次操作 =`n - 1 + k`。 
5. 关键的组合结构是，每个操作都相当于在一个增长集中选择一个位置：

 第一次操作：`n - 1`选择

 第二次操作：`n`选择

 第三次操作：`n + 1`选择

 等等，但只有通过可行性来强制执行有效性约束`a`相对于初始相等对。 
6. 可行性条件：我们必须确保当不存在相等对时我们永远不需要第一类操作。 由于第一类操作只能应用于存在相等邻接的情况，并且每个此类操作都会创建一个新操作，因此唯一的要求是`a`不得超过初始相等邻接的数量加上先前第一类操作创建的数量，这始终是满足的。 唯一真正的限制是，如果满足以下条件，则必须存在至少一个有效的初始相等对：`a > 0`。 
7. 总路数变为线性因子的乘积：

 对于每个`a + b`操作时，我们乘以当时可用插入位置的数量。 
8. 因此答案是：$$\prod_{i=0}^{a+b-1} (n - 1 + i)$$乘以考虑由操作类型引起的难以区分的选择的调整，其分解为第一和第二操作之间的二项式系数分裂：$$\binom{a+b}{a}$$### 为什么它有效

 不变的是，在任何操作序列之后，字符串的组合自由度仅取决于其长度，而不取决于字符的精确排列。 每次插入都会将长度增加一并在下一步中恰好贡献一个新的潜在插入位置。 相等邻接和不相等邻接之间的区别仅与启动第一类操作的可行性有关，但是一旦第一类操作成为可能，它就会传播新的相等邻接，确保未来不会纯粹由结构引起死锁。 

这将系统折叠成一个过程，其中每个有效序列都唯一对应于以下顺序：`a`难以区分的“1 类触发器”和`b`难以区分的“自由插入”，每个插入都会在稳定增长的时间间隔内提供乘法选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n, a, b = map(int, input().split())
    s = input().strip()

    t = 0
    for i in range(n - 1):
        if s[i] != s[i + 1]:
            t += 1

    # feasibility: if no equal adjacency exists and we need first-type ops
    if a > 0 and (n - 1 - t) == 0:
        print(0)
        return

    total = a + b

    # product (n-1)(n)...(n+total-2)
    ans = 1
    for i in range(total):
        ans = ans * (n - 1 + i) % MOD

    # divide by ordering of a identical type-1 choices within total operations
    # combinatorial adjustment: C(a+b, a)
    # compute C(n, k) via factorials
    N = a + b
    fact = [1] * (N + 1)
    for i in range(1, N + 1):
        fact[i] = fact[i - 1] * i % MOD

    inv_fact = [1] * (N + 1)
    inv_fact[N] = modinv(fact[N])
    for i in range(N, 0, -1):
        inv_fact[i - 1] = inv_fact[i] * i % MOD

    comb = fact[N] * inv_fact[a] % MOD * inv_fact[b] % MOD

    ans = ans * comb % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先通过计算转换来提取字符串的混合程度。 这仅用于当不存在相等邻接但需要第一类操作时的可行性检查。 

乘法核心构建与插入位置的增长相对应的上升阶乘。 阶乘块用于计算说明两种运算类型交错的二项式系数。 

一个微妙的点是阶乘是在本地重新计算的，这在多个测试用例设置中是不必要的，但由于单一输入限制，这里是安全的。 在生产竞赛解决方案中，需要预先计算高达一百万的阶乘。

 ## 边缘情况

 一个关键的边缘情况是当字符串完全交替时，例如`010101`。 在这种情况下，不存在相等的相邻对，因此第一类操作一开始就显得不可用。 该算法通过要求如果`a > 0`，一开始必须至少存在一个相等的邻接； 否则答案为零。 这可以防止在没有有效启动触发器的情况下假设结构创建的无效推导。 

另一个边缘情况是完全统一的字符串，例如`00000`。 这里每个邻接都是相等的，因此第一类操作始终适用。 每个应用程序都会增加潜在插入点的数量，因此乘法增长是单调的并且永远不会崩溃。 

最后一个微妙的情况是`a = 0`。 然后问题纯粹简化为第二类运算，答案就变成了纯粹的上升阶乘`n - 1`选择，独立于字符串结构。
