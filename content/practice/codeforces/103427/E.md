 ## 解决方案
 ## 问题理解

 给定一个小写字符串，并要求我们计算特定模式出现的次数，即该字符串`"edgnb"`, 显示为连续子字符串。 每次出现都必须准确且不重叠或重叠，两者都是隐式允许的，因为我们只是计算模式完全匹配的所有起始位置。 

输入大小可以达到 200,000 个字符，这立即告诉我们任何具有二次行为的解决方案（例如显式检查每个子字符串）都是不可行的。 直接一个$O(n \cdot 5)$scan 很好，因为模式长度是固定的并且很小，但是任何重复构造子字符串或天真地扫描并具有额外开销的东西仍然必须仔细控制以保持线性。 

微妙的边缘情况来自重叠的出现。 例如，在像这样的假设字符串中`"edgnbedgnb"`，从位置 0 和 5 开始出现两次。正确的解决方案不得错误地合并它们或跳过重叠。 

另一种边缘情况是字符串少于 5 个字符。 在这种情况下，答案必然是零。 例如，输入`"edgn"`应该输出`0`因为它不能包含完整的模式。 

最后，重复包含部分匹配的字符串可能会欺骗天真的扫描方法，导致错误地重新启动。 例如，`"edgedgnb"`后缀仅包含一次有效出现； 早期的部分匹配，例如`"edg"`或者`"edgn"`不得被误认为是有效命中。 

## 方法

 蛮力的想法很简单。 我们扫描每个位置$i$在字符串中并检查子字符串是否从$i$比赛`"edgnb"`。 由于模式长度固定为 5，因此每次检查的时间都是恒定的，因此总体复杂度为$O(n)$时间。 一个幼稚的实现可能会使用切片来创建子字符串，对于这个问题大小，这在 Python 中仍然有效，但重复分配可能会不必要地增加常数因子。 

就渐近复杂性而言，蛮力方法已经是最佳的，因为模式是恒定大小的。 然而，概念上的改进是认识到我们不需要做任何比五个字符的局部比较更复杂的事情。 不需要前缀函数、自动机或滚动哈希，因为模式是固定且简短的。 

因此，问题简化为通过直接逐个字符比较的单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（子串切片）| O(n) | O(1) 到 O(n) 取决于切片 | 已接受但较重 |
 | 最优（直接比较）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取输入字符串$s$。 我们将从左到右扫描它，并将每个索引视为模式的可能起点。 
2. 定义目标模式`"edgnb"`一次。 这避免了重复的字符串构造并使比较明确。 
3. 迭代所有索引$i$从 0 到$n - 5$。 任何超出的索引$n - 5$无法开始完整匹配，因为剩余字符少于 5 个。 
4. 各位置$i$,比较五个字符$s[i], s[i+1], s[i+2], s[i+3], s[i+4]$与模式的相应字符。 如果全部完全匹配，则增加答案。 
5. 扫描完成后输出最终计数。 

关键思想是每个索引都是独立处理的。 我们从不尝试“扩展”匹配或重用部分计算，因为模式足够短，重新计算比维护状态更便宜。 

### 为什么它有效

 每次出现`"edgnb"`具有独特的起始位置。 该算法对每个可能的起始位置仅检查一次，并验证接下来的五个字符是否与模式匹配。 由于在不跳过起始索引的情况下不会错过任何事件，因此保证了完整性。 由于我们仅在所有字符完全匹配时才进行计数，因此可以保证检测到的每个字符的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
pattern = "edgnb"

n = len(s)
ans = 0

for i in range(n - 4):
    if s[i] == 'e' and s[i+1] == 'd' and s[i+2] == 'g' and s[i+3] == 'n' and s[i+4] == 'b':
        ans += 1

print(ans)
```该实现使用直接字符比较而不是切片。 这避免了临时字符串的创建，并以最小的恒定开销保持运行时严格线性。 循环绑定`n - 4`确保我们永远不会访问超出范围的索引。

 ## 边缘情况

 对于少于 5 个字符的字符串，例如`"abc"`或者`"edgn"`，循环`for i in range(n - 4)`永远不会执行，因为`n - 4 <= 0`。 该算法立即返回 0，这是正确的，因为不存在完全匹配。 

对于重叠图案，例如`"edgnbedgnb"`，扫描独立检查每个起始索引。 在索引 0 处它匹配，在索引 5 处它再次匹配。 不会跳过任何索引，因此两次出现都会被计数。 

对于包含类似于模式但后来有所偏离的长前缀的字符串，例如`"edgxxxxx"`，每个位置都会提前失败字符比较，从而防止误报，而无需回溯或状态跟踪。
