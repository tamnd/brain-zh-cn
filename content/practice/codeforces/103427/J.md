 ## 解决方案
 ## 问题理解

 我们有一个 4 位数字的锁。 每个测试用例提供该锁的两种状态：起始配置和目标配置。 每个状态由固定顺序的四位数字组成，就像一个长度为四的小数组。 

唯一允许的操作是修改这四个位置的连续段。 一步操作中，我们选择任意连续数字区间，然后将该区间中的所有数字加一或全部减一。 这些数字的行为类似于普通整数，没有指定环绕约束，因此我们将更改视为每个位置上独立的算术增量和减量。 

任务是计算每个测试用例将起始 4 位数组转换为目标 4 位数组所需的此类操作的最少数量。 

关键约束是测试用例的数量，最多可达 10^5。 这立即排除了任何超过恒定时间或最坏情况下以非常小的恒定因子线性的每次测试解决方案。 由于每个状态的大小固定为 4，因此每次测试的任何 O(1) 或小状态动态推理都是可接受的，但任何涉及操作序列枚举或组合搜索的情况都是不可接受的。 

天真的解释可能建议探索应用区间运算的所有方法，但即使对于长度为四，如果将其视为搜索问题，分支因子也很大。 然而，固定维数是该结构必须崩溃为直接公式或小状态贪婪结构的关键提示。 

当数字的符号变化彼此“交叉”时，就会出现微妙的边缘情况。 例如，将 1 0 0 0 转换为 0 1 0 0。一个位置需要减少，而相邻位置需要增加，这样就无法直接通过共享区间操作来覆盖它们。 这正是幼稚的每个位置差异求和失败的地方，因为操作不是独立于每个索引的。 

另一种边缘情况是当所有差值一致时，例如 1 1 1 1 到 5 5 5 5。这可以通过重复全范围操作的单个序列来完成，因此答案不是绝对差值之和。 

## 方法

 如果我们忽略位置之间的相互作用，我们可能首先会认为每个数字都可以独立调整。 对于单个位置，将 a 转换为 b 显然需要 |a - b| 单元操作。 将四个位置的总和得出基线上限。 从我们总是可以应用单位置间隔的意义上来说，这是正确的，但它不是最佳的，因为它忽略了将相邻位置分组为共享操作的可能性。 

下一个自然步骤是观察单个操作实际执行的操作。 选择一个区间 [l, r] 同时向该区间中的每个位置添加 +1 或 -1。 这意味着相邻位置之间的差异就像一个信号，并且每个操作都会统一修改该信号的连续段。 

关键的观察是将问题从绝对值转化为邻居之间的差异。 设 d[i] = 目标[i] - 开始[i]。 我们需要使用在连续段上加或减 1 的操作将所有 d[i] 归零，这对应于对 d 的子数组加或减 1。 

现在我们重新解释这个结构：每个操作正是这个差异数组的范围更新。 所需的最小操作数取决于从左到右观察时存在多少不平衡“块”。 每当差异从一个指数变为下一个指数时，我们就被迫引入或取消区间效应。

这将问题简化为计算从左到右扫描时需要多少新调整。 如果我们定义一个辅助序列来跟踪每个位置相对于前一个位置需要多少校正，那么当用零边界增强时，答案就变成相邻 d 值之间的正变化之和。 

这导致每个测试用例的 O(1) 计算量非常小：我们评估四个值的线性函数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力操作 | 指数| O(1) | O(1) | 太慢了 |
 | 每个位置的绝对和 | O(1) | O(1) | O(1) | O(1) | 错误|
 | 差异转换计数 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们根据目标和开始之间的差异向量重写每个测试用例。 令其为 d[0..3]。 

1. 计算四个差 d[i] = b[i] - a[i]。 这隔离了必须由操作“构建”的内容，而不是直接考虑数字值。 该操作现在成为递增或递减该差异数组的连续段的工具。 
2. 想象一下从左到右扫描，保持先前操作已将多少修正带入当前位置。 该携带值正是之前的间隔更新对该位置施加的值。 
3. 在位置 i，将所需的校正 d[i] 与位置 i-1 已携带的校正进行比较。 如果要求增加，我们必须开始新的区间供款。 如果减少，我们必须终止或减少正在进行的捐款。 
4. 如果我们在开始时使用 d[-1] = 0 来扩展 d，则位置 i 所需的新操作数为 max(0, d[i] - d[i-1])。 该表达式精确计算必须引入多少额外的向上力，而这些力不能从先前的位置重复使用。 
5. 将该值对从 0 到 3 的所有 i 进行求和，假设 d[-1] = 0。该总数是所需间隔增量操作的最小数量； 对称性隐式地处理负方向，因为减少可以被视为反转差异的增加。 

### 为什么它有效

 每个操作在连续段上贡献一个单位步骤。 当沿着阵列观察时，累积效应是分段恒定轮廓，只能在段边界处改变。 序列 d 代表我们必须使用这些步骤从零开始构建的目标配置文件。 

每当 d[i] 超过先前的有效水平时，某些新段必须恰好从 i 开始以提供额外的高度。 每当 d[i] 较低时，我们就不需要从那里开始额外的操作。 间隔更新的结构保证了之前创建的任何盈余都可以继续向前，但不能“向后借用”，因此向上转换是强制执行新操作的唯一点。 这使得正相邻差值的总和既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        a = list(map(int, input().split()))
        b = a[4:]
        a = a[:4]

        d = [b[i] - a[i] for i in range(4)]

        prev = 0
        ans = 0
        for x in d:
            if x > prev:
                ans += x - prev
            prev = x

        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先分离两个 4 位状态并计算它们的差异。 变量 prev 表示概念扫描中前一个索引的有效进位调整。 每当当前要求超过上一个要求时，我们都会累积间隙，因为该间隙对应于必须开始的新间隔操作。 如果它低于或等于，则在该边界处不需要新的操作。 

一个常见的实现陷阱是忘记索引 0 之前的隐式初始值 0。如果最初不将 prev 视为 0，则第一个段所需的增量将被低估。 另一个微妙之处是，相同的逻辑会自动处理正差和负差，而无需显式分离符号，因为向下转换不会在此公式中产生额外成本。

| 我| d[i] | 上一页 | 添加| 回答 |

 答案是 2。第一个位置必须减少，而第二个位置必须增加，这迫使两个单独的区间构造，因为单个连续操作无法同时支持跨边界的相反方向。

 ## 边缘情况

 一种重要的边缘情况是所有数字的变化量相同。 对于输入 1111 到 5555，差异数组是恒定的，因此扫描在第一个位置之后永远不会触发正跳转。 仅当我们将每个单位增加视为一个单独的步骤时，该算法才能在此解释中正确输出 4，但由于所有四个位置一起移动，因此正确的解释是重复的全范围操作崩溃为单个线性操作序列，并且转换计数公式通过仅从零开始为每个单位增加支付一次来处理此问题。 

另一个边缘情况是混合符号差异，例如 1 0 0 0 到 0 1 0 0。该算法强制两个单独的开始，因为要求在索引 1 处翻转方向。任何简单的绝对差异方法也会错误地返回 2，但原因错误，更重要的是，在重叠降低成本的情况下，它会失败。 

最后一个微妙的情况是，最佳策略涉及尽早开始较长的间隔以覆盖未来的多次增长。 基于转换的公式已经考虑到了这一点，因为它只计算相对于携带水平的增加，而不是独立地计算每个位置，因此任何早期间隔都会自动向前传播，而无需额外成本。
