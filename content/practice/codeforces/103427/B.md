 ## 解决方案
 ## 问题理解

 给定一个未知非负整数序列$a_1, a_2, \dots, a_n$。 我们收到的不是值本身，而是一组约束，其形式是两个位置的 XOR 是固定的：$a_u \oplus a_v = w$。 

任务不仅是确定这样的序列是否存在，而且如果存在，则构造一个满足所有约束并最小化所有元素总和的序列。 答案是这个最小可能的总和，或者$-1$如果约束相互矛盾。 

每个约束的行为就像图中两个节点之间的关系：它不固定绝对值，但固定它们之间的 XOR 差异。 这立即表明连接组件内的值是紧密耦合的，而不同的组件可以通过选择基值来独立移动。 

约束条件数量很大，最多可达$2 \times 10^5$, 超过$10^5$节点，这排除了任何二次甚至三次的$n$。 任何解决方案本质上都必须是近线性或近对数线性的，这意味着具有某种联合结构或 DFS 的图遍历是唯一现实的方向。 

一致性产生了一个微妙的问题。 考虑一个约束循环。 XOR 方程可能意味着矛盾的值。 例如，如果我们有：$a_1 \oplus a_2 = 1$,$a_2 \oplus a_3 = 1$， 和$a_1 \oplus a_3 = 0$，然后结合前两者意味着$a_1 \oplus a_3 = 0$，所以这是一致的。 但如果第三个约束是$1$，它会立即矛盾。 

另一个微妙的边缘情况是同一对节点之间的多个约束。 如果为同一对给出两个不同的 XOR 值，则答案立即不可能。 在不检查一致性的情况下覆盖边缘的简单实现可能会错过这种冲突。 

最后，目标不仅仅是可行性，而且是总和最小化。 这将问题从纯粹的约束满足任务转变为每个连接组件的优化问题。 

## 方法

 一种暴力解释是为所有节点赋值并尝试一一满足约束，当出现矛盾时回溯。 这很快就会呈指数增长，因为每个组件都可以跨周期传播约束，并且每个猜测分支。 

更结构化的蛮力会尝试修复$a_1 = x$对于每个组件并使用 BFS 或 DFS 传播约束。 这确保了检查的一致性$O(n + m)$，但仍然存在优化问题：尝试所有可能的基值是不可能的，因为值是无界整数。 

关键的观察结果是 XOR 约束定义了相对差异。 如果我们在每个连接的组件中选择一个根并定义$a_v$相对于它，则每个节点都有一个固定的表达式：$$a_v = a_{root} \oplus dist[v]$$在哪里$dist[v]$是从根开始沿路径累积的 XOR。 这将组件内的所有约束减少为单个自由变量：根值。 

现在问题变成：对于每个分量，选择一个根值$x$最小化$$\sum_v (x \oplus dist[v])$$这是一个经典的按位优化问题。 由于 XOR 每位独立运行，因此我们可以最小化每个位$x$分别地。 对于每个位，我们计算组件中有多少个节点设置了该位$dist[v]$。 如果更多的节点可以从根位上有 0 或 1 中受益，我们会选择更便宜的选项。 

因此，每个组件都独立地对答案做出贡献，而全局答案是组件的最佳成本之和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（分配+回溯/尝试根）| 指数| O(n) | 太慢了 |
 | 最优（图+异或传播+位计数）|$O((n+m)\cdot 30)$| O(n + m) | 已接受 |

 ## 算法演练

 我们单独处理每个连接的组件，同时保持与任意根的异或距离。 

1. 构建一个图表，其中每个约束$u \oplus v = w$成为带有标签的无向边$w$。 这表明知道一个端点可以决定另一个端点。 
2. 对于每个未访问的节点，启动 BFS 或 DFS，将其视为根，并将其 XOR 距离指定为 0。该根代表其组件的自由变量。 
3. 遍历时，如果从节点出发$u$到$v$有约束$w$，我们设置：$$dist[v] = dist[u] \oplus w$$如果$v$如果已经访问过，我们通过检查计算值是否与现有值匹配来验证一致性。 如果不是，方程组就会自相矛盾，答案是$-1$。 
4.完成一个组件后，我们已经修复了$dist[v]$对于其中的所有节点。 现在我们计算根值的最佳可能选择$x$一点一点。 对于每个位位置$b$，我们计算组件中有多少个节点有位$b$设置在$dist[v]$。 如果我们设置位$b$的$x$到 0 时，贡献就是该位中 1 的数量。 如果我们将其设置为 1，则贡献就是零的数量。 我们选择较小的。 
5. 将所有位的贡献相加以获得组件的最佳成本，然后将其添加到全局答案中。 

### 为什么它有效

 BFS 确保为组件中的每个节点分配一个与遍历树上的所有约束一致的值。 任何剩余的边都会根据此分配进行检查，因此循环不会默默地违反约束。 

一旦距离相对于根固定，每个有效解决方案的不同之处仅在于用相同的常数对组件中的所有节点进行异或$x$。 这是因为 XOR 约束保留了相对差异。 因此，通过选择单个根值来捕获每个可行的分配。 

成本函数按位分割，因为 XOR 不混合位。 每个位的行为就像组件中所有节点上的独立二元决策一样，最小化总和可以减少独立地最小化每个位的贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    
    for _ in range(m):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((v, w))
        g[v].append((u, w))
    
    vis = [False] * n
    dist = [0] * n
    sys.setrecursionlimit(10**7)
    
    ans = 0
    MAXB = 30
    
    for i in range(n):
        if vis[i]:
            continue
        
        stack = [i]
        vis[i] = True
        dist[i] = 0
        comp = []
        
        ok = True
        
        while stack:
            u = stack.pop()
            comp.append(u)
            for v, w in g[u]:
                if not vis[v]:
                    vis[v] = True
                    dist[v] = dist[u] ^ w
                    stack.append(v)
                else:
                    if dist[v] != (dist[u] ^ w):
                        ok = False
        
        if not ok:
            print(-1)
            return
        
        for b in range(MAXB):
            ones = 0
            for u in comp:
                if (dist[u] >> b) & 1:
                    ones += 1
            zeros = len(comp) - ones
            ans += min(ones, zeros) * (1 << b)
    
    print(ans)

if __name__ == "__main__":
    solve()
```图构造对称地编码每个 XOR 约束，确保遍历可以在两个方向上传播值。 这`dist`数组存储与每个组件中所选根的异或距离，并且当遇到先前访问过的节点时立即进行一致性检查。 

关键的实现细节是我们不尝试分配实际值，而仅尝试分配相对异或距离。 最终优化是在组件完全解析后完成的，这避免了传播和最小化之间的任何交互。 

位循环最多 30 个就足够了，因为约束条件为$2^{30}$，因此所有值都保持在该范围内。 

## 工作示例

 ### 示例 1

 输入：```
3 2
1 2 1
2 3 1
```我们构建一个组件。 从节点1开始，我们得到：$dist[1]=0$,$dist[2]=1$,$dist[3]=0$。 

| 步骤| 节点| 分布分配 | 组件|
 | --- | --- | --- | --- |
 | 1 | 1 | 0 | [1] |
 | 2 | 2 | 1 | [1,2]|
 | 3 | 3 | 0 | [1,2,3]|

 对于每一位，我们最小化贡献。 最佳根选择导致值 [0,1,0] 并且总和为 1。 

这显示了一旦根被固定，相对异或结构如何完全确定所有节点。 

### 示例 2

 输入：```
3 3
1 2 1
2 3 1
1 3 1
```遍历给出：$dist[1]=0$,$dist[2]=1$,$dist[3]=0$，但最后一个约束要求$dist[1] \oplus dist[3] = 1$，其计算结果为 0，因此出现矛盾。 

| 边缘处理| 检查 |
 | --- | --- |
 | 1-2=1 | 1-2=1 好的 |
 | 2-3=1 | 2-3=1 好的 |
 | 1-3=1 | 1-3=1 不匹配|

 该算法立即检测到不一致并返回 -1。 

这证实了循环一致性检查可以防止无效的 XOR 系统。

 ## 边缘情况

 一种边缘情况是多个边缘在同一对节点之间施加冲突的约束。 例如，如果我们有$1 \leftrightarrow 2$值为 1 和 2 时，BFS 将使用第一条边从节点 1 到达节点 2，并分配$dist[2]=1$。 当遇到第二条边时，它检查$dist[1] \oplus 2 = 2$，这不匹配$dist[2]=1$，触发立即失败并返回-1。 

另一种情况是断开连接的图。 每个组件都是独立处理的，并且由于 XOR 约束从不连接组件，因此每个组件都贡献自己的最佳根选择。 该算法自然地重置每个组件的遍历状态，因此不会发生干扰。 

最后一个微妙的情况是没有循环的树状组件。 这里永远不会触发一致性检查，并且所有节点都会收到唯一的 XOR 距离。 优化步骤单独确定最小总和，从而确认非循环结构的正确性不需要循环处理。
