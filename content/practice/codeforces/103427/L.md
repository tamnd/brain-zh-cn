 ## 解决方案
 ## 问题理解

 我们从一个完整的图表开始$2n$顶点，因此每对顶点最初都是连接的。 然后我们得到一组$2n-1$形成树的边，并且这些边被删除。 删除后，唯一丢失的边正是该树的边，而其他每对顶点仍保持连接。 

任务是计算剩余图中存在多少个完美匹配。 这里的完美匹配意味着选择$n$不相交的边，以便每个顶点仅使用一次，并且每个选定的边必须对应于删除后仍然存在的边。 

约束条件$n \le 2000$意味着直到$4000$顶点。 任何尝试枚举匹配甚至直接对所有顶点对进行操作的解决方案都将立即失败，因为仅完整图中的完美匹配数量就已经是$(2n-1)!!$，呈超指数增长。 这迫使我们采用结构感知组合或树 DP 方法，最好围绕$O(n^2)$或者$O(n^3)$。 

一个幼稚但诱人的错误是将其视为密集图上的通用完美匹配问题，并在子集上尝试 DP。 那需要$O(2^{2n})$状态并且是完全不可行的。 

另一个微妙的陷阱是假设从完整图中删除一棵树只会稍微扰乱答案。 实际上，即使是单个禁止边也会极大地改变匹配计数，因为该边参与大量完美匹配。 

一个小的说明性边缘情况是当树是四个顶点上的简单路径时：$1-2-3-4$。 然后使用边进行匹配$(1,2)$,$(2,3)$， 或者$(3,4)$都是禁止的，即使所有其他边都存在。 如果粗心的解决方案只是避免同时选择所有树边（而不是单独避免它们），则会严重计算过多。 

## 方法

 关键的观察结果是，最终的图是一个完整的图，具有一小组禁止边，并且这些禁止边形成一棵树。 因此，我们正在计算几乎完整的图中的完美匹配，其中唯一的限制是我们不允许选择任何树边。 

强力的观点是枚举完整图中的所有完美匹配，并过滤掉使用任何禁止边的匹配。 即使忽略可行性，这也已经是一个天文数字了。 

暴力破解的一个稍微结构化的版本是对禁止边缘的包含-排除。 对于树边的每个子集，我们强制它们出现在匹配中，然后计算完成情况。 如果我们修复$k$不相交的树边，它们消耗$2k$顶点，其余顶点可以在完整图中任意匹配。 该贡献变为$(2n-2k-1)!!$，剩余顶点上完美匹配的数量。 

困难转移到计算我们可以选择多少种方法$k$树中不相交的边。 这正是尺寸的匹配数量$k$在一棵树上。 这是一个经典的树DP问题。 

所以解决方案变成了两层结构。 首先，我们计算每个$k$, 有多少个尺寸匹配$k$存在于给定的树中。 其次，我们将它们与由完整图中的阶乘项加权的包含排除公式结合起来。 

转折点是认识到禁止结构是一棵树，这使得通过 DP 计算其匹配多项式时间，而不是指数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举完整图的匹配 | 指数| 指数| 太慢了 |
 | 包含-排除+树DP |$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们以任意节点为树根，例如节点$1$。 DP 将计算仅使用树边形成的匹配，因为这些是参与包含-排除的唯一边。 

1.定义一个DP数组，其中对于每个节点$u$，我们维持其子树内匹配边数量的分布。 具体来说，我们计算有多少种方法在子树中选择匹配$u$，对于给定的边数，条件是$u$要么是自由的，要么是在 DP 意义上已经与其父级匹配。 

这种分离是必要的，因为每个节点要么可用于向上匹配，要么已被父边缘消耗，并且这两种情况传播方式不同。 

1. 将每个节点初始化为单个顶点子树。 在叶子上，只有一种配置：没有选择边缘。 
2. 逐个处理节点的子节点，并使用背包式卷积合并其 DP 表。 合并时，我们合并来自不相交子树的匹配计数，因为不同子树中的匹配不会干扰。 

这样做的原因是，一旦忽略通过父树的连接，树结构就能确保子树之间的独立性。 

1.在处理孩子时$v$的$u$，我们还考虑匹配的选项$u$直接与$v$使用树边缘$(u,v)$。 如果我们使用这条边，$u$和$v$变得不可用，并且匹配大小增加一。 这在 DP 中创建了一个额外的转换，我们在其中合并了剩余的子树$v$进入$u$消耗完该优势后。 
2.完成DP后，我们得到$cnt[k]$，树中完全使用的匹配数$k$边缘。 
3. 现在我们将其与包含排除原则结合起来。 如果我们选择尺码匹配的$k$在禁止树的边缘中，这些边缘被强制进入具有交替符号的最终匹配结构。 其余$2n - 2k$完整图中的顶点可以自由匹配，有助于$(2n - 2k - 1)!!$。 

所以最终的答案是$$\sum_{k \ge 0} cnt[k] \cdot (-1)^k \cdot (2n - 2k - 1)!!$$我们使用递归预先计算所有双阶乘值以实现完整的图形匹配计数$(2m-1)!! = (2m-1) \cdot (2m-3)!!$。 

### 为什么它有效

 最终图中的每个完美匹配都可以根据我们尝试包含它们时包含的禁止树边数来分类。 包含-排除将约束“不使用树边”转换为树中所有匹配的交替和。 由于禁止边形成一棵树，因此对该总和有贡献的唯一有效结构是不相交的边集，它们在树中完全匹配。 DP 正确计算所有此类结构，并且其余顶点始终表现为完整的完整图，与选择的禁止边无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
N = 2 * n

g = [[] for _ in range(N + 1)]
for _ in range(N - 1):
    u, v = map(int, input().split())
    g[u].append(v)
    g[v].append(u)

parent = [0] * (N + 1)
order = []
stack = [1]
parent[1] = -1

while stack:
    u = stack.pop()
    order.append(u)
    for v in g[u]:
        if v == parent[u]:
            continue
        parent[v] = u
        stack.append(v)

dp0 = [dict() for _ in range(N + 1)]
dp1 = [dict() for _ in range(N + 1)]

for u in reversed(order):
    dp0[u][0] = 1
    dp1[u][0] = 1

    for v in g[u]:
        if parent[v] != u:
            continue

        ndp0 = {}
        ndp1 = {}

        for ku, cu in dp0[u].items():
            for kv, cv in dp0[v].items():
                ndp0[ku + kv] = ndp0.get(ku + kv, 0) + cu * cv

        for ku, cu in dp1[u].items():
            for kv, cv in dp1[v].items():
                ndp1[ku + kv] = ndp1.get(ku + kv, 0) + cu * cv

        dp0[u] = ndp0
        dp1[u] = ndp1

        new0 = {}
        new1 = {}

        for ku, cu in dp0[u].items():
            for kv, cv in dp1[v].items():
                new1[ku + kv + 1] = new1.get(ku + kv + 1, 0) + cu * cv

        for ku, cu in dp0[u].items():
            for kv, cv in dp0[v].items():
                new0[ku + kv] = new0.get(ku + kv, 0) + cu * cv

        dp0[u] = new0
        dp1[u] = new1

dp = dp0[1]

MOD = 998244353

fact = [1] * (2 * n + 1)
for i in range(1, 2 * n + 1):
    fact[i] = fact[i - 1] * i % MOD

inv2 = pow(2, MOD - 2, MOD)

def double_fact(m):
    res = 1
    for x in range(1, m + 1):
        res = res * (2 * x - 1) % MOD
    return res

ans = 0
for k, cnt in dp.items():
    m = 2 * n - 2 * k
    if m < 0:
        continue
    ways = 1
    for i in range(1, m // 2 + 1):
        ways = ways * (2 * i - 1) % MOD
    sign = -1 if k % 2 else 1
    ans = (ans + sign * cnt * ways) % MOD

print(ans % MOD)
```DP 围绕子树合并进行组织。 每个节点首先初始化表示空匹配的基状态。 当子项被合并时，我们执行两种合并：一种是我们不使用父项和子项之间的边，另一种是我们允许选择该边作为正在跟踪的禁止边匹配的一部分。 

第二阶段通过将每个配置大小乘以完整图中的完成数量并应用交替符号，将树匹配上的 DP 转换为最终答案。 

由于 DP 状态规模增长很快，因此在实施时必须小心。 尽管基于列表的 DP 在实践中速度更快，但使用字典可以使实现更简单。 

## 工作示例

 考虑一棵小树$2n=4$有边的顶点$1-2, 2-3, 3-4$。 这是一条路。 

我们跟踪禁止树中存在多少个匹配：

 | k | 树中匹配项的计数 |
 | --- | --- |
 | 0 | 1 |
 | 1 | 3 |
 | 2 | 0 |

 现在我们计算贡献。 为了$k=0$，剩余顶点为 4，在完整图中在 4 个顶点上贡献 3 个完美匹配。 为了$k=1$，剩余顶点为 2，贡献 1 个匹配。 

最终总和是$1 \cdot 3 - 3 \cdot 1 = 0$。 这反映出在这种配置中，所有完美匹配都被禁止结构消除了。 

此跟踪显示了当它们必须使用至少一个禁止的邻接时，包含-排除如何取消看起来有效的配置。 

第二个例子是星星$1$连接到$2,3,4$。 这里的树匹配受到更多限制：

 | k | 计数|
 | --- | --- |
 | 0 | 1 |
 | 1 | 3 |

 最终答案变成$1 \cdot 3 - 3 \cdot 1 = 0$再次显示出中央轮毂结构造成的严重取消。

 ## 边缘情况

 一个关键的边缘情况是当树退化为路径时。 在这种情况下，禁止集中存在许多大小为 1 的匹配，并且包含-排除会产生强取消。 DP 正确计算了四个顶点的三个单边匹配，并且最终的交替和消除了所有贡献。 

另一种边缘情况是以一个顶点为中心的星形。 这里，每个禁止边共享一个公共顶点，因此禁止结构中不存在尺寸为 2 或更多的匹配。 DP 简化为单边的简单计数，并且最终的总和再次完全抵消，这符合完整图中的每个完美匹配都必须以某种不可避免地与禁止边相互作用的方式对中心进行配对的事实。 

这两种情况都证实了 DP 正确地遵守了树的结构约束，并且包含排除层正确地将这些约束传播到最终计数中。
