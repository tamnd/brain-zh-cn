 ## 解决方案
 ## 问题理解

 We are given a string of length$n$，然后我们查看该字符串的每个非空前缀。 对于每个前缀，我们应用取决于该前缀内字符位置的确定性转换。 

转换的工作原理如下。 对于任何角色$c$出现在字符串中$S$，我们看看它最后一次出现在$S$。 从该位置之后立即开始，我们计算后缀中出现的不同字符数。 该计数变成了一个数字$G(c, S)$，以及角色$c$替换为与该数字对应的小写字母，其中$0 \mapsto a$,$1 \mapsto b$， 等等。 使用相同的原始字符串同时转换所有字符$S$，这意味着映射是根据原始前缀计算出来的，然后并行应用。 

在计算每个前缀的编码字符串后，我们按字典顺序比较所有它们并输出最大的一个。 

字符串长度最多为1000，并且只使用前20个小写字母。 该限制很重要，因为它保证不同字符的数量很小且有界，这使得无需繁重的数据结构即可维护每个前缀的状态。 

简单的实现会针对每个前缀、每个字符重新计算最后一次出现的位置和不同的后缀计数。 如果仔细完成，这已经表明三次最坏情况，但更重要的是，从头开始重新计算隐藏了在前缀之间重用的结构。 

一个微妙的边缘情况是，未出现在前缀中的字符与映射无关，但如果在计算后缀不同计数时错误地包含这些字符，仍然可能会混淆不正确的实现。 另一个边缘情况是编码取决于前缀内的最后一次出现，而不是全局位置，因此在不重置边界的情况下跨前缀重用数据将产生不正确的结果。 

## 方法

 蛮力的想法很简单。 对于每个前缀$S[0:i]$，我们通过迭代该前缀中的每个字符来计算编码字符串，找到它在前缀中的最后一次出现，然后扫描到前缀的末尾以计算该位置之后的不同字符。 由于此扫描针对每个字符和每个前缀进行，因此最坏情况的成本大致为$O(n^3)$。 和$n = 1000$, 这是周围$10^9$操作，这在Python中太慢了。 

关键的观察结果是，前缀的编码仅取决于每个字符的两条信息：前缀内的最后一个位置以及其右侧的一组不同字符。 当我们将前缀扩展一个字符时，这两者都可以增量维护。 我们不是重新计算所有内容，而是更新最后出现的情况并维护一个结构，该结构可以回答“当前前缀中位置 i 之后存在多少个不同的字符”。 

由于字母表大小只有 20，因此我们可以为每个位置维护一个后缀中出现的字符的位掩码。 这允许在扩展前缀时进行恒定的时间更新，并且还允许我们计算每个$G(c, S)$每个角色的固定时间。 

因此，对于每个前缀，我们可以构建其编码$O(n \cdot 20)$，并动态比较结果，避免显式存储所有编码字符串。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 最佳 |$O(n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 在迭代字符串时预先计算前缀列表。 在步骤$i$，我们考虑前缀$S[0:i]$。 
2. 维护当前前缀中每个字符的最后出现索引。 这使我们能够确定每个字符对后缀计数的贡献应该从哪里开始。 
3. 对于每个前缀，计算后缀不同的结构。 我们构建一个位掩码数组，其中每个位置代表从该位置到前缀末尾出现的一组不同字符。 这可以通过从右到左扫描一次前缀并更新滚动位掩码来计算。 
4. 对于每个角色$c$在前缀中找到它最后出现的位置$p$。 价值$G(c, S)$是后缀位掩码中设置的位数，从$p + 1$，或零，如果$p$是最后一个位置。 
5. 通过映射每个字符构建前缀的编码字符串$c$到$chr(G(c, S) + 'a')$。 
6. 使用字典顺序将此编码字符串与迄今为止看到的最佳字符串进行比较。 如果当前编码字符串较大，则更新答案。 
7.处理完所有前缀后，输出最佳编码字符串。 

### 为什么它有效

 关键的不变性是，在任何前缀边界处，后缀位掩码数组正确地表示该前缀的每个后缀段中的不同字符集。 由于每个字符的编码仅严格取决于其最后一次出现后的字符，因此位掩码确保我们准确地计算相关的不同字符，而不是其他字符。 因为扩展前缀时更新只是附加的，所以先前计算的后缀信息不会变得无效，并且最后出现的位置在每个前缀内保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    best = ""

    for i in range(n):
        pref = s[:i+1]
        m = len(pref)

        # last occurrence in current prefix
        last = [-1] * 26
        for j, ch in enumerate(pref):
            last[ord(ch) - 97] = j

        # suffix distinct bitmask
        suf_mask = [0] * (m + 1)
        mask = 0
        for j in range(m - 1, -1, -1):
            mask |= (1 << (ord(pref[j]) - 97))
            suf_mask[j] = mask

        # compute encoded string
        encoded = []
        for j, ch in enumerate(pref):
            c = ord(ch) - 97
            p = last[c]

            if p + 1 >= m:
                val = 0
            else:
                val = bin(suf_mask[p + 1]).count("1")

            encoded.append(chr(val + 97))

        encoded = "".join(encoded)

        if encoded > best:
            best = encoded

    print(best)

if __name__ == "__main__":
    solve()
```该代码明确地迭代所有前缀。 对于每个前缀，它重新计算最后出现的位置和后缀位掩码，这在约束下是可以接受的$n \le 1000$。 后缀位掩码是从右到左构建的，以便每个位置都在其右侧存储精确的不同字符集。 编码步骤使用这些预先计算的值来避免重复重新计算不同的集合。

 输入字符串：`aacc`我们处理每个前缀。

此跟踪显示了随着添加更多字符，编码如何稳定，以及一旦后缀结构变得更丰富，后面的前缀如何支配前面的前缀。

 ## 边缘情况

 一种重要的边缘情况是前缀中的所有字符都相同。 用于输入`aaaa`，每个最后出现的位置始终是最后一个位置，因此每个字符都映射到`a`。 该算法处理此问题是因为最后一次出现后的后缀掩码为空，始终产生零，从而实现稳定的编码。 

另一个边缘情况是严格增加不同的字符，例如`abcd`。 这里最后出现的每一个都是字符本身，因此后缀集随着我们的前进而缩小。 编码反映了完全相反的结构，并且算法正确地计算后缀掩码，以便每个字符准确地看到其最后一次出现后出现的字符。 

最后一个微妙的情况是最佳前缀不是完整的字符串。 为了`aca`，第二个前缀产生比完整前缀更好的编码字符串。 由于该算法独立评估每个前缀并比较所有结果，因此它可以正确识别此类情况，而无需假设前缀之间的单调性。
