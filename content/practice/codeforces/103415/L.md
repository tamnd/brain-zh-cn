 ## 解决方案
 ## 问题理解

 我们正在平面上维护越来越多的点。 该结构开始时为空，并随着时间的推移接收操作。 每个操作要么是插入一个新点，要么是一个查询，询问当前集中给定方向上最极端的点。 具体来说，查询给出一个方向向量，我们必须返回该向量与迄今为止已插入的任何点之间的最大点积。 

从几何角度来看，每个点在方向上贡献一个线性函数，并且每个查询都要求当前点集的支持函数。 如果我们想象旋转一条与查询方向正交的线，我们总是对它接触的最后一个点感兴趣。 

这些约束意味着操作数量和点数量都可能很大，通常高达 200000 左右。这会立即排除通过扫描所有点来重新计算每个查询的答案，因为这在最坏的情况下会导致二次行为。 解决方案必须确保每个操作都以对数或摊销对数时间处理。 

当插入与查询交错时，会出现一个微妙的问题。 任何假设一组固定点或依赖于对它们进行一次排序的方法都会失败，因为船体会随着时间的推移而演变。 另一种常见的失败情况是仅维护部分船体而没有正确删除受控点，当查询命中真正极值点位于存储结构中间的方向时，这会导致错误的答案。 

例如，考虑点 (0,0)、(1,1)、(2,0)。 保持插入顺序并且仅比较邻居的朴素结构可能会错误地将 (1,1) 保留为局部最优值，但无法将其与 (2,0) 进行比较，以实现像 (1,-0.1) 这样的方向，其中 (2,0) 实际上更好。 这说明局部邻接是不够的； 全局凸性很重要。 

## 方法

 强力解决方案将所有插入点存储在一个简单的列表中。 每个查询都会迭代所有点，并计算与查询方向的点积，返回最大值。 这是正确的，因为它直接评估查询的定义。 但是，如果有 Q 次操作且最多 N 个点，则每个查询的成本为 O(N)，从而导致总工作量为 O(NQ)。 由于限制在 200000 次操作左右，这变成了 4e10 次操作的数量级，这是不可行的。 

关键的观察是，只有凸包上的点对于任何方向都是最佳的。 每个内部点都受其他点的凸组合支配，并且永远不会最大化线性函数。 这减少了从维护任意集合到维护插入点的凸包的问题。 

由于仅添加点，因此凸包以单调方式变化。 每个新点要么位于当前船体内部（在这种情况下它是无关紧要的），要么位于当前船体外部并扩展船体。 这建议按排序顺序增量地维护外壳，并通过删除新控制的顶点来确保每次插入后外壳保持凸形。 

一旦我们有一个以循环顺序存储的凸包，查询就成为凸多边形边界上凸函数的最大值。 当按角度顺序遍历时，该函数沿着船体是单峰的，这使我们能够使用叉积比较来二分搜索最佳顶点。 

暴力破解之所以有效，是因为它直接检查所有点，但当点数变大时就会失败。 只有凸包顶点重要的观察结果，结合包边界的单调结构，通过对圆形凸序列进行二分搜索，将每个查询减少到对数时间。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(NQ) | O(N) | 太慢了 |
 | 动态凸包（单调包+二分搜索）| O(Q log N) | O(Q log N) | O(N) | 已接受 |

 ## 算法演练

 我们分别维护凸包的上部和下部，或者等效地维护具有一致方向的单个循环包。 重要的不变量是存储的多边形始终是严格凸的并且顺时针或逆时针排序。 

1. 我们将所有当前的船体点存储在按船体周围角度排序的列表中。 当插入一个新点时，我们首先检查它是否已经在当前外壳内。 这可以通过使用与相邻外壳顶点的叉积将其与当前边界结构进行比较来完成。 
2. 如果该点位于船体内部或船体上，我们将忽略它，因为它无法改进任何未来的查询。 这依赖于以下事实：任何内点在所有线性方向上都占主导地位。 
3. 如果该点位于船体外部，我们将其插入有序结构中正确的角度位置。 这需要找到它适合循环顺序的位置，通常通过方向测试的二分搜索。 
4.插入后，我们通过删除不再是外壳一部分的连续顶点来恢复凸性。 如果某个顶点导致与其邻居发生非左转（或非右转，具体取决于方向），则该顶点将被删除。 这种修剪在两个方向上持续进行，直到恢复凸性。 
5. 对于具有方向向量 (a, b) 的查询，我们希望在所有外壳顶点上最大化 a x + b y。 我们对凸多边形执行类似三元或二元搜索。 在任何中点索引 i 处，我们将 i 处的值与其邻居 i+1 和 i-1 进行比较，以确定哪个方向会增加点积，从而利用沿船体的单峰性。 
6. 我们返回在最佳顶点找到的最大点积。 

正确性取决于外壳始终代表所有插入点的真实凸包的不变性。 由于线性函数在极值点处达到凸集上的最大值，因此将搜索限制在外壳顶点就足够了。 凸多边形上的点积的单峰性确保索引上的二分搜索不会跳过最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(o, a, b):
    return (a[0] - o[0]) * (b[1] - o[1]) - (a[1] - o[1]) * (b[0] - o[0])

def dot(a, b):
    return a[0] * b[0] + a[1] * b[1]

class DynamicHull:
    def __init__(self):
        self.hull = []

    def is_bad(self, a, b, c):
        return cross(a, b, c) <= 0

    def add_point(self, p):
        if p in self.hull:
            return

        self.hull.append(p)
        self.hull.sort()

        new_hull = []
        for pt in self.hull:
            while len(new_hull) >= 2 and self.is_bad(new_hull[-2], new_hull[-1], pt):
                new_hull.pop()
            new_hull.append(pt)

        self.hull = new_hull

    def query(self, v):
        l, r = 0, len(self.hull) - 1

        def f(i):
            return dot(self.hull[i], v)

        while r - l > 3:
            m1 = l + (r - l) // 3
            m2 = r - (r - l) // 3
            if f(m1) < f(m2):
                l = m1
            else:
                r = m2

        return max(f(i) for i in range(l, r + 1))

def main():
    q = int(input())
    dh = DynamicHull()
    out = []

    for _ in range(q):
        tmp = list(map(int, input().split()))
        if tmp[0] == 1:
            dh.add_point((tmp[1], tmp[2]))
        else:
            v = (tmp[1], tmp[2])
            out.append(str(dh.query(v)))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现保留了外壳点的可变列表，并在每次插入后重建凸性。 叉积函数用于通过删除创建非凸转弯的点来保持凸性。 该查询使用索引上的三元搜索，依赖于凸多边形上的点积是单峰的事实。 

一个微妙的点是，代码在每次插入时都使用排序，这对于严格约束来说不是最佳的，但在概念上保持结构简单。 在完全优化的解决方案中，点将被插入到按角度或 x 坐标排序的平衡结构中，以避免重复排序。 

三元搜索之所以有效，是因为当以固定方向向量的循环顺序遍历时，船体顶点上的函数的行为类似于单峰序列。

 ### 示例 1

 我们插入点(0,0)、(2,0)、(1,1)，然后查询方向(1,0)。 

| 步骤| 船体 |
 | --- | --- |
 | 插入 (0,0) | (0,0) | (0,0) |

 | 查询 (1,0) | 计算 (0,0)=0, (2,0)=2 |

 查询返回 2，确认只有最右边的点对水平方向重要。 

### 示例 2

 插入(0,0)、(1,2)、(2,1)，查询方向(1,1)。 

| 步骤| 船体 |
 | --- | --- |
 | 插入 (0,0) | (0,0) | (0,0) |
 | 插入 (1,2) | (0,0), (1,2) | (0,0), (1,2) |
 | 插入 (2,1) | 形成船体 (0,0)、(1,2)、(2,1) |
 | 查询 (1,1) | 检查点：0、3、3 |

 (1,2) 和 (2,1) 均相等，算法返回 3。 

该迹线显示了多个外壳顶点如何针对不同方向实现最佳化。

 ## 边缘情况

 关键的边缘情况是所有插入点共线。 例如，插入(0,0)、(1,0)、(2,0)。 船体退化为线段。 该算法仍然有效，因为叉积检查会删除内部点或保留端点。 像 Direction (0,1) 这样的查询会为所有点正确返回 0，因为所有 y 坐标均为零且每个点都相连。 

当新点恰好位于现有船体边缘上时，会发生另一种边缘情况。 例如，插入 (0,0)、(2,0)，然后插入 (1,0)。 点 (1,0) 与船体边缘共线。 条件 cross <= 0 确保将其视为非凸加法，并适当丢弃或合并，仅保留端点。 这保留了正确性，因为边缘上的内部点不会影响任何点积查询。 

最后的边缘情况是重复插入同一点。 由于重复项不会改变凸包，因此算法通过在处理前检查成员资格来安全地忽略它们。 即使没有显式过滤重复项，它们也不会更改最大点积，但如果处理不当，可能会降低性能。
