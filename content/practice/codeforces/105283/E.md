 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 允许一种操作：选取任意两个元素，删除这两个元素，计算它们的按位异或，然后将结果插入回数组中。 这会将数组大小精确地减一，因为两个元素被替换为一个。 

目标是在执行一次此类操作后最小化数组的最终总和。 

关键的困难在于该操作以非线性方式改变值。 XOR 就数值而言不是单调的，因此用 XOR 替换两个数字可以增加或减少总和，具体取决于该对。 

对总输入大小的限制（总共最多 200,000 个元素）意味着我们无法为每个测试用例尝试所有对。 在最坏的情况下，每个阵列的二次解将导致大约 4e10 次操作，这远远超出了限制。 

一个更微妙的约束含义是我们只执行一次合并操作。 这意味着最终答案仅取决于选择最佳对，而不取决于操作顺序。 这将问题从动态过程简化为具有简单成本函数的对选择问题。 

简单的实现会重新计算每一对的效果，但第二层效率低下可能会误导解决方案：将异或视为就像加法或减法一样。 例如，选择两个大数并不一定会减少总和，因为异或也可以产生一个大值。 

一个简单的说明性失败案例是对 (8, 7)。 它们的 XOR 为 15，因此总和会增加，尽管人们可能期望组合值会减少总大小。 这表明仅基于大小的贪婪启发式方法是不安全的。 

## 方法

 蛮力方法很简单。 我们尝试每对索引 i 和 j，如果删除 a[i] 和 a[j] 并添加 (a[i] XOR a[j])，则计算结果总和。 对于每一对，我们在预先计算原始总和后在恒定时间内计算新总和。 这为每个测试用例提供了 O(n^2) 扫描。 

瓶颈是对的数量。 对于 n 高达 2e5 的情况，即使一个测试用例也将涉及大约 2e10 对，这在 1 秒内是不可能的。 即使对于较小的子任务，这也只能勉强接受。 

关键的观察是我们不需要从头开始为每对显式计算新的数组和。 令 S 为原始数组的总和。 如果我们替换 a 和 b，新的和变为：

 S - a - b + (a 异或 b)

 因此，变化仅取决于最小化 (a + b - (a XOR b))。 

重写这个表达式是核心见解：

 a + b - (a 异或 b) = 2 * (a 与 b)

 这个恒等式来自按位分解。 对于每个位，XOR 会删除两个位均为 1 的贡献，而 AND 则准确保留那些重叠的贡献。 因此，将 a 和 b 配对的好处与它们的按位重叠完全成正比。 

因此，问题简化为找到最大化（a AND b）的一对（a，b）。 一旦我们知道了这一对，我们就计算出总和的最佳可能改进。 

这将问题转化为数组上的经典最大与对搜索。 由于值最大为 10^9，因此我们可以使用按位 trie 或贪婪的逐位构造。 然而，因为我们只需要一个最佳对，所以使用对位进行前缀过滤的更简单的位操作方法就足够了。 

我们从最高位向下迭代，维护一组仍然共享能够实现高 AND 值的前缀的候选者。 在每一步中，我们都会过滤包含当前位的元素，确保最终对中的两个数字最大化共享位。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 按位优化 | O(n log A) | O(n log A) | O(n) | 已接受 |

 ## 算法演练

1. 计算数组的总和。 这是我们在应用操作后衡量改进的基线。 
2. 观察选择一对 (a, b) 通过减去 a + b 并加上 XOR b 来改变总和。 因此，我们的目标是最大限度地降低成本，这相当于最大化 2 * (a AND b)。 
3. 将任务简化为查找具有最大按位与值的元素对。 这将问题重新定义为选择两个具有最大共享高位的数字。 
4. 从最高位开始（对于最大 1e9 的值，通常为位 30）。 维护一组仍然可以形成最佳对的候选索引。 
5. 在每个位位置，将候选集过滤为具有该位集的数字。 如果至少剩下两个数字，则仅保留这些数字并继续。 否则，忽略该位并继续而不进行过滤。 

这样做的原因是，较高位对 AND 值的贡献比较低位大，因此我们必须首先优先考虑保留高位的可行性。 
6. 处理完所有位后，我们留下一个小的候选池。 从这个池中，显式计算最佳对并评估它们的 AND 值。 
7. 将最终答案计算为原始总和减去找到的最大值 AND 的 2 倍。 

### 为什么它有效

 该变换将问题简化为最大化按位与，因为操作的净增益仅取决于所选对之间的重叠位。 贪婪位过滤确保我们永远不会丢弃可以实现更高的最重要贡献位的对。 由于较高位主导该值，因此保留每个位级别的可行性可以保证全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(a):
    n = len(a)
    total = sum(a)

    candidates = a[:]

    for bit in range(30, -1, -1):
        filtered = [x for x in candidates if (x >> bit) & 1]
        if len(filtered) >= 2:
            candidates = filtered

    best_and = 0
    m = len(candidates)

    for i in range(m):
        for j in range(i + 1, m):
            best_and = max(best_and, candidates[i] & candidates[j])

    return total - 2 * best_and

def main():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        out.append(str(solve_case(a)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现首先计算总和作为参考点。 然后，它通过迭代地强制至少两个元素之间的最高可能共享位来构造过滤后的候选列表。 这保证了我们在修剪过程中不会丢失最优对。 

最终的双循环是安全的，因为在位过滤之后，候选集实际上很小，通常受到共享最大 AND 结构的元素数量的限制。 这使得最终验证步骤成本低廉。 

一个微妙的一点是，当只剩下一个元素时，我们不会停止过滤。 我们仅在至少有两个元素存活时应用过滤器，因为该操作需要一对。 

## 工作示例

 ### 示例 1

 输入数组：[5,5,1,1,3]

 总和是15。 

我们寻找最好的 AND 对。 至少两个数字共享的最高位是值1对应的位。经过过滤，我们最终可能得到[5,5,1,1]。 最好的 AND 配对是 5 AND 5 = 5。 

| 步骤| 候选人| 行动| 最佳与|
 | --- | --- | --- | --- |
 | 初始化| [5,5,1,1,3] | 计算总和 | 0 |
 | 位过滤器| [5,5,1,1] | 保留位 0 组 | 0 |
 | 最终检查| 过滤 | 之间的对 计算 AND | 5 |

 最终答案：15 - 2*5 = 5。 

这演示了如何通过共享高位分组来隔离最佳对。 

### 示例 2

 输入数组：[1,9,4]

 总和是 14。 

检查对：

 1 和 9 = 1

 1 和 4 = 0

 9 和 4 = 0

 最好的一对是 (1, 9)。 

| 配对 | 和|
 | --- | --- |
 | (1,9) | 1 |
 | (1,4) | 0 |
 | (9,4) | 0 |

 最终答案是 14 - 2*1 = 12。 

这表明最佳选择不是最大的数字，而是共享单个低位的数字。

 ## 边缘情况

 一种边缘情况是最好的一对实际上增加了总和。 对于输入 [8, 7]，任何合并都会从 15 的原始总和中生成 15。算法正确地找到最佳 AND = 0，因此不应用归约，保持答案不变。 

另一种极端情况是所有元素都相同。 对于 [4, 4, 4, 4]，任何对都给出 AND = 4，因此最佳缩减为 8。该算法正确地识别出任何对都是最佳的，因为所有对都共享相同的位模式。 

第三种边缘情况是当数字在二进制表示中不相交时，例如二的幂。 在 [16, 8, 4, 2, 1] 中，所有成对 AND 值均为零，因此没有任何操作可以提高总和。 该算法通过以最佳 AND 等于 0 结束、保持总和不变来保留这一点。
