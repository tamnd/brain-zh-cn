 ## 解决方案
 ## 问题理解

 给定一个路径图，其中节点编号为 1 到 n，其中每个节点 i 以概率 1/i 独立保留，否则删除。 在这个随机删除过程之后，剩余的节点形成一个简单链的子图，我们想要该导出子图中连接组件的预期数量。 

此设置中的连接组件只是保留节点的最大连续段。 由于原始结构是一条线，每次我们有一个保留的节点后跟一个删除的节点，或者一个删除的节点后跟一个保留的节点，我们就可能创建或结束一个段。 关键的困难不是模拟随机性，而是计算指数级多个子集的期望。 

输入由多个测试用例组成，每个测试用例仅给出n。 我们必须输出以 1e9+7 为模的预期连接分量数，写为模分数。 

约束 n 高达 10^6 和高达 2 × 10^5 测试用例意味着任何每次测试的线性解决方案都太慢，除非全局进行一次预计算。 因此，解决方案必须将每个测试减少到 O(1) 或 O(log n)，并且所有繁重的工作必须在 O(max n) 内预先计算。 

由于存在 2^n 个配置，因此枚举节点子集或模拟随机删除的简单方法是不可能的。 即使直接计算所有组件结构的概率也是不可行的。 

一个微妙的边缘情况是 n = 1。由于节点 1 始终保持概率 1/1 = 1，因此答案始终恰好是 1 个分量。 另一个边缘情况是 n = 2，其中节点 1 始终存在，而节点 2 以 1/2 的概率存在，因此无论节点 2 是否出现，组件的预期数量始终为 1。 

一个天真的错误是假设预期组件等于预期保留节点数减去预期相邻保留对数，这会忽略组件依赖于游程的事实，而不是在没有正确分解的情况下单独依赖于成对邻接。 

## 方法

 关键的观察来自于重新构建路径中的连接组件。 如果保留节点 i，并且不保留节点 i = 1 或节点 i − 1，则新组件恰好从节点 i 开始。 这将全局结构问题转换为局部指示变量的总和。 

令 Xi 为节点 i 启动新组件的指示符。 那么分量总数就是 Xi 的 i 之和。 根据期望的线性，我们只需要计算每个 i 的 P(Xi = 1)。 

对于 i = 1，如果保留节点 1，则组件将启动，这种情况发生的概率为 1。 

对于 i > 1，如果节点 i 被保留，则节点 i 启动一个组件，而节点 i − 1 不被保留。 由于事件是独立的，因此该概率为 (1/i) × (1 − 1/(i − 1))。 

这将整个问题简化为独立概率的前缀和。 

强力解释将考虑节点的所有子集并计算每个子集的分量，这需要 O(n · 2^n)。 即使在区间上进行动态规划也需要 O(n^2)，这对于 n 达到 10^6 来说是不可能的。 

关键的结构见解是路径中的组件是局部转换，因此期望可以干净地分解。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |
 | 前缀概率和| O(n) 预处理，O(1) 每个查询 | O(n) | 已接受 |

 ## 算法演练

 我们首先将问题重写为计算组件启动的预期数量。

1. 对于每个 i，如果保留节点 i，并且 i = 1 或节点 i − 1 不保留，则定义 Xi = 1。 答案是 Xi 对所有 i 的总和。 
2. 对于 i = 1，计算贡献为 P（保留节点 1）。 由于概率为 1/1，因此为 1。 
3. 对于每个 i > 1，将 P(Xi = 1) 计算为 P(i 被保留) × P(i − 1 不被保留)。 独立性允许乘法，因为节点状态是独立的。 
4. 替代概率：P(i 被保留) = 1/i 且 P(i − 1 不被保留) = 1 − 1/(i − 1) = (i − 2)/(i − 1)。 
5. 因此贡献变为 (1/i) × (i − 2)/(i − 1)，简化为 (i − 2) / (i(i − 1))。 
6. 预先计算 i 从 1 到 n 的前缀和 S[n] = Σ 贡献(i)。 
7. 对于每个查询 n，输出 S[n] 模 1e9+7。 

我们可以预先计算的原因是所有测试用例都是独立的，但共享相同的函数 S[n]，因此我们计算一次直到最大 n。 

### 为什么它有效

 路径中的每个连接组件都有一个左边界，它是该段中第一个保留的节点。 指标 Xi 准确捕获了这些边界位置，并且仅捕获了这些位置。 由于每个组件恰好贡献一个起始点，并且每个起始点恰好对应一个组件，因此 Xi 的总和等于每个实现中的组件数量。 取期望保持相等，因此概率之和等于期望的分量数。 不可能出现多计数或少计数的情况，因为组件开始对保留的集合进行唯一的分区。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    t = int(input())
    ns = [int(input()) for _ in range(t)]
    maxn = max(ns)

    inv = [0] * (maxn + 2)
    for i in range(1, maxn + 2):
        inv[i] = modinv(i)

    dp = [0] * (maxn + 2)
    dp[1] = 1

    for i in range(2, maxn + 1):
        # (1/i) * (1 - 1/(i-1))
        keep = inv[i]
        not_prev = (i - 2) * inv[i - 1] % MOD
        dp[i] = (dp[i - 1] + keep * not_prev) % MOD

    out = []
    for n in ns:
        out.append(str(dp[n]))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先为最大 n 之前的所有整数构建模逆。 这支持恒定时间概率评估。 然后 dp[i] 存储最多 i 的期望分量数。 

转换对从 i 开始的新分量的导出概率进行编码。 术语 keep 对应于 1/i，not_prev 对应于 (i − 2)/(i − 1)。 我们积累这些贡献是因为期望相对于职位是累加的。 

一个微妙的实现细节是正确处理 i = 2。 对于 i = 2，(i − 2) 变为零，这意味着节点 2 永远无法启动新组件，因为节点 1 始终存在，这与问题结构匹配。

 ## 边缘情况

 当 n = 1 时，算法直接设置 dp[1] = 1。 由于不存在转换，因此结果基本上是正确的。 

对于 n = 2，i = 2 的贡献变为 (2 − 2)/(2·1) = 0，因此 dp[2] 保持为 1。这与节点 1 始终锚定单个组件（无论节点 2 无关）的事实相符。 

对于较大的n，例如n = 3，只有节点3可以创建新组件，并且只有当节点2缺失时。 该公式通过 (i − 2)/(i − 1) 正确捕获了这种依赖性，确保完全遵守邻接约束，而无需显式跟踪段。
