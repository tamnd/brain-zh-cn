 ## 解决方案
 ## 问题理解

 我们被要求计算有多少个有序的整数对$(n, p)$满足涉及两个值之间的乘积关系的一小组约束。 

价值$p$首先选择并限制在范围内$0 \le p \le P$。 对于每一个这样的$p$，我们考虑整数$n$使得产品$n \cdot p$不等于$p$。 换句话说，我们正在计算相乘的对$p$经过$n$改变值。 

条件$p \neq n \cdot p$相当于说$p = 0$和$n \cdot p = 0$总是成立，或者$p \neq 0$我们需要$n \neq 1$。 这是关键的结构简化，因为它表明一对无效的唯一方式是当$p \neq 0$和$n = 1$。 

输入尺寸$P \le 10^5$意味着我们不能迭代所有可能的整数$n$以一种天真的、无限的方式。 任何循环的解决方案$n$每$p$立即是不可行的。 甚至迭代所有对$(n, p)$除非我们隐式约束，否则将是无限的$n$，因此预期的解释是只有有限多个有效对做出贡献，并且我们必须在结构上表征它们。 

出现微妙的边缘情况时$p = 0$。 然后$n \cdot p = 0$为所有人$n$，所以每对$(n, 0)$无效。 忘记这种情况的粗心实现可能会错误地计算来自$p = 0$，特别是如果它假设$p \neq 0$简化条件时。 

另一个边缘情况是$p = 1$。 在这种情况下，条件变为$1 \neq n$，所以只有$n = 1$被排除在外。 这凸显出该限制完全取决于是否$p$为零或非零。 

## 方法

 强力解释是迭代所有$p$从$0$到$P$，并且对于每个$p$，在合理的范围内迭代$n$，检查是否$n \cdot p \neq p$。 困难在于$n$在声明中不受限制，因此暴力方法是不明确的，除非我们假设有一个截止点。 即使我们人为地限制$n$到$[0, P]$，复杂度变为$O(P^2)$，其中在$P = 10^5$是$10^{10}$的运作，远远超出了可行的限度。 

关键的观察是谓词仅取决于是否$p = 0$以及是否$n = 1$。 对于每一个$p > 0$，所有值$n$均有效，除了$n = 1$。 为了$p = 0$，没有值$n$是有效的，因为$n \cdot 0 = 0 = p$总是。 

因此，我们不是枚举对，而是将问题简化为计算有多少个$n$每个值都允许$p$。 如果我们假设$n$范围与以下相同的域$p$，这与样本行为一致，那么对于每个$p > 0$我们贡献$(P + 1) - 1 = P$的有效选择$n$，并且对于$p = 0$我们贡献$0$。 

这将整个计数问题分解为直接的算术表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解对 |$O(P^2)$|$O(1)$| 太慢了|
 | 直接计数|$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 解释有效域$p$作为所有整数$0$到$P$。 这给出了$P + 1$可能的值。 
2. 根据 的值将计数分为两种情况$p$。 的行为$n \cdot p$完全取决于是否$p = 0$或者$p > 0$。 
3.为了$p = 0$，观察到$n \cdot p = 0$对于所有整数$n$。 这意味着每对$(n, 0)$违反约束$p \neq n \cdot p$，因此整个切片贡献零个有效对。 
4. 对于每个$p > 0$, 强制执行条件$n \cdot p \neq p$。 自从$p \neq 0$，我们可以将两边除以$p$，这产生$n \neq 1$。 这正好删除了一个值$n$来自允许的集合。 
5.数数有多少个$p > 0$存在，即$P$，并乘以有效的数量$n$每个这样的值$p$，这也是$P$如果$n$范围超过$[0, P]$。 这产生$P \cdot P$。 

### 为什么它有效

 转型从$n \cdot p \neq p$简单的排除条件是有效的，因为乘以非零整数是整数上的单射。 什么时候$p > 0$，两边除以保持等价。 什么时候$p = 0$，表达式折叠为常量相等，这隔离了必须单独处理的退化情况。 这将整个问题空间划分为两个不相交的区域，其中计数是直接且详尽的，无需枚举。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

P = int(input().strip())

# p = 0 contributes nothing
# for each p in 1..P, there are P valid choices of n (excluding n = 1 from P+1 choices)
print(P * P)
```解决方案读取单个整数$P$并使用导出的封闭形式直接计算最终计数。 关键的实现选择是完全避免任何循环。 无效情况的减法在推导中隐式处理，因此代码仅执行乘法。 

一个常见的陷阱是忘记$p = 0$根本不贡献有效的对。 另一种假设是错误的$n$有一个不同的界限$p$，这会改变最终的表达式。 解决方案的正确性取决于将两个变量视为在同一隐式域范围内$[0, P]$，与样本行为一致。

 ## 边缘情况

 当$P = 1$，算法只考虑$p = 0$和$p = 1$。 这$p = 0$分支没有贡献任何东西，因为每个产品都是零，而$p = 1$恰好贡献了一个有效的$n$排除后$n = 1$。 计算干净地减少到$1$，确认排除逻辑即使在最小规模下也能发挥作用。 

什么时候$P = 0$，唯一可能的$p$为零。 这种情况没有提供有效的对，因为$n \cdot 0 = 0$总是等于$p$，将总数保留为零，与公式一致。 

对于任何较大的$P$，同一分区保持：全部非零$p$与单个禁止值的行为相同$n$，并且总计数按二次方缩放，而不同之间没有相互作用$p$价值观。
