 ## 问题理解

 我们有一个矩形网格，其中每个单元格都是三种类型之一。 有些单元格是有效的高地位置，我们可以在其中放置单位，有些单元格是敌人站立的低地单元格，有些单元格被阻挡且与放置无关。 任务是计算有多少个放置单元满足特定的定向攻击条件。 

仅当我们可以选择四个基本方向之一并查看该方向上接下来的五个连续单元格时，放置单元格才有效。 这五个细胞中，至少有两个是敌方细胞。 我们不允许将起始单元格本身包含在此计数中，仅将向外延伸的五个单元格包含在内。 

输出只是至少一个方向满足这一条件的有效放置位置的数量。 

网格大小可以大到 1000 x 1000，这意味着多达 100 万个单元。 任何检查每个单元的恒定工作量的解决方案都是可以接受的，但是任何从每个单元重复扫描长段的解决方案都会变得太慢。 每个方向每个单元的简单扫描每次最多会检查五个单元，但如果不小心实施冗余检查或重复边界逻辑，它仍然会降级到大约 2000 万次操作，这很好，但任何涉及较大窗口重新计算或重复切片的操作都会变得脆弱且容易出错。 

主要的微妙之处是边界处理。 当单元靠近边缘时，某个方向上的五个位置中的一些位置不存在。 这些必须被简单地忽略，而不是被视为有效的非敌方细胞。 将丢失的单元视为非敌人的幼稚实现会错误地惩罚边缘放置。 

当高地放置在一个方向上紧邻少于五个有效单元格时，会出现另一种边缘情况。 例如，在 1 x 5 网格中，向左或向上查看可能会产生零个有效单元格。 正确的解释是只考虑现有的细胞，并且我们仍然检查其中是否至少出现两个敌人。 

## 方法

 暴力解决方案很简单。 对于每个可放置高地砖的单元格，我们尝试所有四个方向。 对于每个方向，我们向前迭代最多 5 步，计算出现了多少个低地砖，并检查计数是否至少为 2。 如果任何方向满足条件，我们就对单元进行计数。 

这是有效的，因为问题定义本身是本地的。 每个单元仅依赖于四个方向上恒定大小的邻域。 然而，即使窗口尺寸很小，暴力破解仍然对每个单元执行最多 4 次检查，每次检查最多 5 个步骤。 最坏的情况是每个单元大约进行 20 次操作，因此整个 1000 x 1000 网格大约需要 2000 万次操作。 这在 Python 中已经是可以接受的，但前提是干净地实现而没有重复字符串切片或递归等开销。 

关键的观察是，除了仔细迭代之外，不需要进一步的优化。 该结构是固定大小的定向扫描，因此我们不需要前缀和或高级预处理。 整个问题简化为检查恒定数量的固定偏移量。 

因此，最佳解决方案与暴力破解的想法相同，但要小心地实现为固定偏移量上的直接索引算术。

 ## 算法演练

 我们迭代网格中的每个单元格。 对于每个不可放置的高地瓷砖的单元格，我们立即跳过它，因为它无法对答案做出贡献。 

对于每个有效的起始单元格，我们检查四个方向：上、下、左、右。 对于每个方向，我们最多检查五个连续的单元格。

我们维护一个计数器，记录最多五个位置中出现的低地单元数量。 如果计数器达到两个或更多，我们立即将此起始单元标记为有效并停止检查它的其他方向。 

我们必须小心确保当走出网格时，我们停止扫描而不是访问无效内存。 出界位置将被简单地忽略并且不被计入任何东西。 

最后，我们将所有有效的起始单元格相加。 

### 为什么它有效

 每个单元都根据固定的本地条件进行独立评估。 该条件仅取决于每个方向上最多五步之外的一组有界位置。 由于我们明确枚举了所有可能的方向和所有可能的相关偏移，因此每个有效配置都会被测试一次。 单元格之间不存在重叠或依赖性，因此独立计算每个单元格会产生正确的全局答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]
    ans = 0

    for i in range(n):
        for j in range(m):
            if grid[i][j] != 'H':
                continue

            ok = False

            for dx, dy in dirs:
                cnt = 0
                x, y = i, j

                for _ in range(5):
                    x += dx
                    y += dy

                    if x < 0 or x >= n or y < 0 or y >= m:
                        break
                    if grid[x][y] == 'L':
                        cnt += 1
                    if cnt >= 2:
                        ok = True
                        break

                if ok:
                    break

            if ok:
                ans += 1

    print(ans)

if __name__ == "__main__":
    main()
```网格存储为字符串列表，因此索引的复杂度为 O(1)。 方向向量编码运动，无需重复的分支逻辑。 对于每个候选单元格，我们在每个方向上显式模拟最多五个步骤，这是一项持续的工作。 

当我们在一个方向上看到两个敌人时，一个微妙的细节就会提前被打破。 这可以防止沿该方向不必要地扫描剩余单元。 另一个微妙之处是当我们出界时立即停止，因为这些位置对计数没有贡献。

 ## 边缘情况

 关键的边缘情况是当候选单元格靠近边界并且一个方向上存在的单元格少于五个时。 在这样的情况下`1 5`和`H L L L H`，最左边的`H`其右侧只有四个有效单元格。 该算法仍然检查这四个并正确计算低地单元，而不假设丢失的单元有任何贡献。 

另一种边缘情况是在扫描早期正好出现两个低地砖。 例如，如果某个方向上的前两个单元格都是`L`，我们立即将方向标记为有效并停止进一步扫描。 这确保了正确性，同时避免了不必要的工作。 

最后的边缘情况是多个方向都可以满足条件。 该算法不依赖于选择哪个方向，只依赖于至少一个方向满足它。 一旦一个方向达到阈值，单元格就会计数一次，从而防止重复计数。
