 ## 解决方案
 ## 问题理解

 我们得到一个数组和三个特殊值，并且有严格的排序$Y < K < X$。 对于每个测试用例，任务是计算有多少连续子数组满足三个同时条件：子数组中的最大元素恰好是$X$，最小元素正好是$Y$，以及值$K$不会出现在子数组内的任何位置。 

因此，仅当每个元素都遵循以下规定的边界时，子数组才有效$X$和$Y$，同时还避免该范围中间的特定禁止值。 

输入数据量很大，最多可达$5 \cdot 10^5$测试用例中的总元素。 这立即排除了任何直接检查所有子数组的解决方案，因为这会导致二次或更糟糕的行为。 偶数的二次扫描$10^5$元素产生周围$10^{10}$操作，这远远超出了一秒的极限所能容忍的。 

关键结构是有效性完全由段内的局部属性决定：子数组是否位于范围内$[Y, X]$, 避免$K$，并且包含两个极端$Y$和$X$。 这表明我们不应该枚举子数组，而应该将数组转换为最大有效区域并有效地计算每个区域内的贡献。 

当天真的方法忽略禁止值时，就会出现微妙的失败情况$K$。 例如，如果$X = 7$,$Y = 4$,$K = 5$，数组是$[6, 7, 5, 4]$，暴力扫描可能有效$[6,7,5,4]$有效，因为它包含两者$7$和$4$，但它错误地包括$5$，违反条件。 另一个常见的错误是忘记了外部的价值观$[Y, X]$使任何穿过它们的子数组无效。 在$[3, 4, 7]$，任何包含的子数组$3$不能有最低限度$Y = 4$，因此不得将其视为有效区域的一部分。 

## 方法

 最直接的方法是枚举每个子数组并检查它是否满足三个条件。 对于每个子数组，我们跟踪其最小值、最大值以及是否$K$出现。 逐步维护这些仍然给我们留下了$O(N^2)$每个测试用例的子数组，并且每次检查至少$O(1)$，使得解决方案太慢$N$达到$10^5$。 

关键的观察是无效元素将数组划分为独立的段。 任何外部元素$[Y, X]$立即破坏有效性，因为它违反了包含它的任何子数组的最小或最大约束。 禁止值$K$，虽然在范围内，但也充当硬分隔符，因为它取消包含它的任何子数组的资格。 

这将问题简化为独立的部分，其中每个元素都位于$[Y, X]$并且不等于$K$。 在这样的段内，我们需要计算至少包含一次出现的子数组$Y$并且至少出现一次$X$。 

对于每个被视为右端点的位置，我们维护最近出现的$Y$和$X$。 有效起始位置的数量由最后两个出现中最早的一个确定，因为任何有效的子数组都以$r$必须从两个索引中较小的一个开始，以确保两个值都包含在内。 这种滑动窗口视角消除了显式枚举子数组的需要。

 | --- | --- | --- | --- |
 | 蛮力 |$O(N^2)$|$O(1)$| 太慢了 |
 | 段+滑动窗口|$O(N)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例并从左到右扫描数组，维护段边界并跟踪关键事件。 

### 步骤

 1. 初始化变量`start`标记当前有效段的开始。 该段仅包含不包含的元素$K$并躺在里面$[Y, X]$。 
2.维护两个指针，`lastY`和`lastX`，初始化为数组开始之前的位置。 这些存储了最近的位置$Y$和$X$。 
3. 迭代数组索引$r$从左到右。 
4.如果$A[r] = K$或者$A[r] < Y$或者$A[r] > X$，通过设置重置该段`start = r + 1`并重置`lastY`和`lastX`为无效值。 这确保没有子数组跨越无效边界。 
5. 否则，更新`lastY`如果$A[r] = Y$并更新`lastX`如果$A[r] = X$。 
6. 如果两者都`lastY`和`lastX`在当前段内有效，计算有多少个有效子数组以$r$。 This count is`max(0, min(lastY, lastX) - start + 1)`并将其添加到答案中。 

### 为什么它有效

 在任何段内，每个子数组都会自动不含无效元素和$K$。 唯一剩下的约束是有效子数组必须至少包含一个$Y$和一个$X$。 对于固定的右端点，最新位置$Y$和$X$确定保证两者同时出现的最早点。 任何较早的起始索引仍包含这两个值，而任何较晚的起始索引至少排除其中一个值。 这为在每个位置结束的所有有效子数组建立了直接计数规则，没有重复或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, X, Y, K = map(int, input().split())
        A = list(map(int, input().split()))

        start = 0
        lastY = -1
        lastX = -1
        ans = 0

        for i, v in enumerate(A):
            if v == K or v < Y or v > X:
                start = i + 1
                lastY = -1
                lastX = -1
                continue

            if v == Y:
                lastY = i
            if v == X:
                lastX = i

            if lastY != -1 and lastX != -1:
                ans += max(0, min(lastY, lastX) - start + 1)

        print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了段分解和滑动窗口逻辑。 这`start`指针确保我们永远不会考虑交叉无效元素的子数组或$K$。 变量`lastY`和`lastX`持续跟踪必要的端点以强制两个极端的存在。 最终的累加步骤将这些位置转换为每个右端点的有效起始索引的计数。 

一个常见的实现陷阱是在遇到无效元素时忘记重置最后出现的变量。 如果不进行此重置，过时索引会跨段泄漏并错误地合并独立区域。

 ## 边缘情况

 连续出现无效分隔符的情况，例如多次出现$K$确保段重置不会意外合并。 该算法通过重置来处理这个问题`start`,`lastY`， 和`lastX`在每个无效位置都是独立的，因此没有任何状态会在中断时持续存在。 

一个案例，其中$Y$和$X$仅出现在长段的边缘，检查前缀计数的正确性。 该算法仍然计算从早期有效开始扩展的所有子数组，因为`min(lastY, lastX)`正确锚定最早的可行边界。 

数组在有效值和无效值之间交替的情况测试重复的段初始化。 每个新的部分都是全新的，并且贡献仅限于本地，防止跨界过度计算。
