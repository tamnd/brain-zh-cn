 ## 解决方案
 ## 问题理解

 我们得到一个数组，每个查询都会询问段内所有有序的索引对。 对于查询段$[l, r]$，我们取每一对$(i, j)$在该范围内，计算$a_i + a_j$，然后将所有这些结果异或在一起。 

因此，查询并不是要求求和或计数，而是要求按位 XOR 下所有成对和的全局奇偶校验组合。 因为两者$i = j$和$i \neq j$成对包含在内，每个元素都与自身和所有其他元素相互作用。 

限制才是真正的困难。 数组长度最大可达$2 \cdot 10^5$，并且最多可以有$10^6$查询。 这立即排除了任何每次查询的二次处理，甚至$O((r-l+1)^2)$或任何接近于每个查询重建结构的东西。 甚至$O(\log n)$由于查询数量如此之大，每个查询都很紧张，因此解决方案必须在预处理后将每个查询的时间减少到接近恒定的时间。 

一个微妙的边缘情况来自于异或与排序和复制的相互作用有多强。 由于成对排列，$(i, j)$和$(j, i)$都包含在内并贡献相同的价值$a_i + a_j$。 这种重复很重要，因为只有当值出现偶数次时，XOR 才会取消这些值，而有序对会创建结构化的多重性，而天真的“组合计数”方法可能会错误处理这种多重性。 

另一个问题是加法引入了进位。 幼稚的按位独立假设立即失败。 例如，即使两个数字在位上没有重叠，它们的和仍然可能由于进位传播而设置更高的位，这使得 XOR 结构依赖于全局二进制交互而不是独立的位。 

## 方法

 直接的强力解决方案迭代每个查询范围中的所有对并对它们的和进行异或。 这是正确的，因为它遵循字面定义。 但是，对于单个大小的查询$k$，它执行$k^2$加法和异或运算。 和$k$可能高达$2 \cdot 10^5$和$10^6$查询，这会变得天文数字般大，大约为$10^{16}$最坏情况下的操作。 

为了改善这一点，我们尝试重构表达式。 关键的观察结果是，所有成对和的异或仅取决于段中值的分布，而不是它们的顺序。 更重要的是，所有对上的异或是双线性聚合：每个元素相对于所有其他元素对称地贡献。 

我们不再明确地考虑对，而是将视角转向比特贡献。 对于每个位位置，我们只关心该位在所有对和中是否被设置了奇数次。 这将问题简化为研究加法如何在多重集上分配进位的奇偶性。 

关键的结构见解是对于固定位位置$k$，只有较低的$k+1$数字的位影响是否$k$和的第 - 位翻转。 这将依赖性局部化：较高位不影响较低位进位行为。 这允许基于截断值对 2 的模幂进行预处理，并通过组合预先计算的频率信息来回答查询。 

然后，我们构建一个预处理方案，支持这些截断分布的快速范围提取，允许通过组合恒定数量的预先计算的位级贡献来解决每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O((r-l+1)^2)$每个查询|$O(1)$| 太慢了 |
 | 位分解预处理|$O(n \cdot B + m \cdot B)$|$O(n \cdot B)$| 已接受 |

 这里$B$是位数，通常为 30。 

## 算法演练

 我们独立处理每个位位置并计算其对最终 XOR 答案的贡献。 

1.固定位位置$k$。 我们想要确定查询范围内所有成对和的 XOR 中是否设置了该位。 这仅取决于奇数对是否产生一个总和，其$k$-第 位为 1。 
2. 观察$k$第 位$a_i + a_j$仅取决于较低的$k+1$的位$a_i$和$a_j$。 这使我们能够用每个数字的模值来替换它$2^{k+1}$不改变该位的正确性。 
3. 查询$[l, r]$，我们从概念上需要这些减少的值在段内的频率分布。 从该分布中，我们可以确定有多少个有序对产生一个总和，其值位于$k$-th 位已设置。 
4. 我们不是重新计算每个查询的频率，而是预处理前缀结构。 对于每一位$k$，我们维护一个余数模的前缀频率表$2^{k+1}$。 这让我们可以提取与模量大小成比例的时间范围内的频率分布。 
5. 对于每个查询和每个位，我们通过迭代该范围内的所有残基类来计算有效对的数量。 对于每对残基类别$(x, y)$，我们检查是否$(x + y) \bmod 2^{k+1}$落在半个区间，其中位$k$已设置。 如果是这样，我们将它们的频率乘积相加，尊重有序对。 
6. 我们使用 XOR 将这些贡献的奇偶性累积到最终答案中。 

### 为什么它有效

 该算法依赖于两个不变量。 首先，减少模数$2^{k+1}$保留的行为$k$- 加法下的第 1 位，因为较高位不能影响进位位置$k$。 其次，所有对上的 XOR 聚合仅取决于每个位位置贡献出现次数的奇偶校验，因此以 2 为模计算对频率就足够了。 这两个事实相结合，将全局成对进位问题简化为每位独立的模块化直方图。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_prefix(freqs):
    # freqs[k][i][v] is frequency of value v up to i for bit k
    return freqs

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))

    B = 30
    pref = []

    for k in range(B):
        mod = 1 << (k + 1)
        p = [[0] * mod for _ in range(n + 1)]
        for i in range(n):
            for v in range(mod):
                p[i + 1][v] = p[i][v]
            p[i + 1][a[i] & (mod - 1)] += 1
        pref.append(p)

    out = []

    for _ in range(m):
        l, r = map(int, input().split())
        l -= 1

        ans = 0

        for k in range(B):
            mod = 1 << (k + 1)
            cnt = [0] * mod

            for v in range(mod):
                cnt[v] = pref[k][r][v] - pref[k][l][v]

            bit_parity = 0

            for x in range(mod):
                if cnt[x] == 0:
                    continue
                for y in range(mod):
                    if cnt[y] == 0:
                        continue
                    if (x + y) & (1 << k):
                        bit_parity ^= (cnt[x] * cnt[y]) & 1

            ans |= (bit_parity << k)

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码为每个位位置构建截断值的前缀频率表。 每个查询都会提取范围的直方图，并检查所有残基对以确定它们是否对最终 XOR 中的位有贡献。 

重要的实现细节是，所有内容都在贡献级别以模 2 进行跟踪，因为 XOR 仅取决于奇偶校验。 这就是为什么每次积累都会减少`& 1`在合并到最后一点之前。 

残差类上的嵌套循环是检查所有模和结果的直接转换，这仅是可行的，因为每个模数相对于位宽度都很小。 

## 工作示例

 考虑一个小数组，我们可以在其中显式跟踪对和。 

对于输入：```
a = [2, 3, 4], query [1, 3]
```我们计算所有有序对：

 | 我| j | a[i] + a[j] | a[i] + a[j] |
 | --- | --- | --- |
 | 1 | 1 | 4 |
 | 1 | 2 | 5 |
 | 1 | 3 | 6 |
 | 2 | 1 | 5 |
 | 2 | 2 | 6 |
 | 2 | 3 | 7 |
 | 3 | 1 | 6 |
 | 3 | 2 | 7 |
 | 3 | 3 | 8 |

 对所有值进行异或可得出最终答案。 相反，该算法按低位残基对值进行分组，并计算每个仓中有多少有序对，从而产生相同的奇偶校验结果，而无需显式枚举对。 

第二个例子是统一数组：```
a = [1, 1, 1], query [1, 3]
```这里每对总和都是相同的，因此答案仅取决于该总和出现的次数。 该算法检测到频率分布具有单个活动残基，有序对的奇偶校验确定最终的 XOR 是否为零或重复和。

 ## 边缘情况

 单元素段，例如$[i, i]$只产生一对$(i, i)$，因此结果减少为$a_i + a_i$。 该算法正确地处理了这个问题，因为频率表产生单个非零残差，并且对计数步骤自然地包括自对。 

所有值都相同的段在 XOR 下会产生应力消除行为。 每个对和重复多次，正确性取决于总有序对的奇偶性。 基于频率的公式正确地将其减少为计数重数而不是枚举对。 

具有交替位模式（例如两个力的幂）的段会产生传播效应。 即使各个位不重叠，总和也会激活较高位。 模数的使用$2^{k+1}$残差分组确保在每个位级计算中正确捕获这些进位。
