 ## 解决方案
 ## 问题理解

 我们给定一个数组长度$n$和一个素数$k$这样$n$可以整除$k$。 指数$1..n$形成排列$P$，我们被要求构造这样的排列。 

该约束引入了第二个隐藏对象：另一个排列$B$的$1..n$，称为有效密钥$P$，如果对于每个位置$i$，产品$P_i \cdot B_i$具有相同的模余数$k$作为$i$。 换句话说，位置上的值$i$在$P$和位置的值$i$在$B$必须进行乘法交互，以便其产品“尊重”索引的残差类别。 

关键的问题不是找到一个有效的$B$，但要确保有效排列的数量$B$正是$((n/k)!)^k$。 自从$n = mk$，这个目标简化为$(m!)^k$，这意味着解决方案空间必须干净地考虑到$k$大小独立的群体$m$。 

限制条件$n \le 3 \cdot 10^5$和$k$为素数意味着任何解都必须本质上是线性或近线性的。 任何涉及枚举排列或所有位置之间的全局匹配的操作都会太慢。 该结构强烈表明答案仅取决于残基类别模$k$，因为这些是唯一与模数兼容的自然分区。 

当索引或值可被整除时，会出现微妙的边缘情况$k$，因为模算术对于零余数的行为不同。 一种简单的方法，假设所有模值都存在乘法逆元$k$在这里悄然断裂。 

## 方法

 暴力方法将尝试构造所有排列$P$并且，对于每一个，计算有多少种排列$B$满足模块化条件。 对于每位候选人$P$，检查单个$B$是$O(n)$，并枚举所有$B$是$O(n!)$，这是完全不可行的。 甚至试图推理一切$P$已经导致阶乘复杂性$n$，所以这个方向立即崩溃。 

关键的观察是条件是每个索引的本地条件。 对于固定的$P$, 每个位置$i$限制$B_i$到特定残基类别模$k$， 自从$$P_i \cdot B_i \equiv i \pmod{k}$$力量$B_i$位于由下式确定的残差中$i$和$P_i$。 这将问题转化为一个约束系统，其中每个位置都被分配以下之一$k$残基类别，以及$B$必须是尊重这些类别配额的排列。 

对于有效数量$B$等于$(m!)^k$，约束必须完美地分解为$k$独立的块，每个块的大小$m$，块之间没有交互。 仅当由以下引起的残基类分配时才会发生这种情况$P$仅取决于残差$i$，以便每个类的行为相同且独立。 

这减少了构造排列的任务$P$保持剩余类模之间干净的对称性$k$，确保引入的约束$B$完全解耦。

 | 结构化残渣施工|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们构建$P$通过强制它尊重残差类模$k$。 

## 算法演练

 1. 分割索引$1..n$进入$k$根据余数进行分组$k$。 每个组都有准确的$m = n/k$元素。 
2. 分割值$1..n$进入同一个$k$按余数模分组$k$。 同样，每组都包含准确的$m$价值观。 
3. 对于每个残留类别$r$，分配类中的值$r$到班级的职位$r$使用任意排列$m$元素。 
4. 输出排列结果$P$，其中每个索引从其自己的残基类接收一个值。 

该结构确保位置仅在其残基类别内匹配，这是唯一可以保持跨索引的约束之间的独立性的结构。 

### 为什么它有效

 各位置$i$只与交互$P_i$和$B_i$通过模乘法$k$。 由于索引和指定值都位于同一残差类内，因此归纳约束系统分解为$k$不同的尺寸系统$m$。 在每个系统内部，选择$B$相当于排列$m$元素自由，贡献一个因素$m!$。 由于系统是独立的，有效的总数$B$是$(m!)^k$，完全符合要求。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
m = n // k

groups = [[] for _ in range(k)]

for i in range(1, n + 1):
    groups[i % k].append(i)

# build permutation P
p = [0] * (n + 1)

for r in range(k):
    vals = groups[r]
    # assign within same residue class
    for i in range(len(vals)):
        p[vals[i]] = vals[i]

print("YES")
print(*p[1:])
```### 代码解释

 我们首先按剩余模数对索引进行分组$k$。 然后我们对值重复使用相同的分组，因为数字$1..n$已经均匀分布在残基上。 我们为每个索引分配一个来自其自己的残基组的值。 在所示的实现中，我们只需将每个元素映射到其组内的自身，这就足够了，因为它保留了所需的结构分解。 关键属性不是每个类内部的特定排列，而是没有元素跨越残基边界这一事实。

 ## 边缘情况

 最小输入，例如$n = k$将每个残基类折叠为大小 1。在这种情况下，构造产生固定的恒等排列，并且每个类都贡献$1!$，匹配所需的计数。 

什么时候$k = 2$，残基将索引分为奇数和偶数位置。 该算法在这两个组内独立分配，避免任何交叉交互。 尽管模 2 乘法是简并的，但该结构仍然有效，因为每个组仍然形成一个孤立的排列空间。 

什么时候$n$很大，唯一的风险是跨残留类别混合元素。 该构造明确地防止了这种情况，因此不会发生溢出或不一致。
