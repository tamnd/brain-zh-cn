 ## 解决方案
 ## 问题理解

 我们正在与一对隐藏的整数进行交互$A$和$B$，都在范围从$0$到$10^9$。 我们不能直接看到它们。 相反，我们可以发送一个查询值$X$，系统返回值$(A \oplus X) + B$， 在哪里$\oplus$是按位异或。 

每个查询给出一个数字，该数字是 XOR 的$A$使用我们选择的值，然后向上移动相同的未知偏移量$B$。 在最多五次这样的查询之后，我们必须准确地确定两个隐藏的数字。 

输出格式迫使我们陷入交互式重建问题：每个查询都会显示一个转换后的视图$A$，但总是与相同的加性常数混合$B$。 关键的难点是异或在加法下是非线性的，并且未知数$B$隐藏绝对比例。 

就交互数量而言，限制非常小，而不是输入大小。 只允许五个查询，这立即排除了任何试图通过重复探测来一点一点学习的策略。 任何隔离的方法$A$必须在不断的查询中通过仔细的代数消去来做到这一点。 

一个天真的想法是尝试多个值$X$并尝试直接求解方程。 然而，每个响应都有以下形式$A \oplus X + B$，因此减去响应就可以消除$B$但留下了异或差异，这仍然是非线性的。 如果没有仔细选择结构化查询，这将变得不确定。 

一个微妙的边缘情况是$A = 0$。 那么响应就变成了$X + B$，它看起来是线性的，可能会引发贪婪的重建$B$，但这具有误导性，因为它完全隐藏了 XOR 结构。 另一个棘手的情况是当$A$仅设置了高位，由于响应之间的减法引入了进位，使得天真的低位推理失败。 

## 方法

 蛮力策略是猜测所有可能的值$A$,计算对应的$B$来自单个查询，并针对其他查询进行验证。 对于每位候选人$A$，我们可以计算$B = \text{response} - (A \oplus X)$。 这是一致性检查，但是范围$A$取决于$10^9$，这使得这完全不可行，因为它需要最多$10^9$候选人和每个候选人的多个查询。 

关键的观察结果是 XOR 在位差异上表现出线性，并且我们可以隔离$A$通过消除$B$通过两个查询响应相减。 如果我们查询两个不同的值$X_1$和$X_2$，我们得到：$$R_1 = (A \oplus X_1) + B,\quad R_2 = (A \oplus X_2) + B$$减去得到：$$R_1 - R_2 = (A \oplus X_1) - (A \oplus X_2)$$现在$B$被彻底消除。 这减少了恢复的问题$A$仅来自 XOR 差异。 

经典的技巧是选择结构化查询，例如$0$,$2^k - 1$，以及一些以受控方式显示携带的位掩码。 一个特别干净的方法是首先恢复$A$使用三个查询，然后恢复$B$直接来自任何响应。 

我们继续发送：

 第一个查询$X = 0$, 给予$R_0 = A + B$。 

第二次查询$X = C$， 在哪里$C = 2^{30} - 1$，因此所有低 30 位均已设置。 这迫使 XOR 翻转所有低位$A$，产生互补模式。 

我们计算：$$R_C - R_0 = (A \oplus C) - A$$自从$C$都是低位，$A \oplus C$成为该范围内的按位补码，允许重构$A$逐位使用差异的已知固定结构。 

一次$A$已恢复，$B$获得为：$$B = R_0 - A$$原则上我们只需要两个查询，但由于允许五个查询，因此更安全的实现通常使用第三个查询来进行验证或避免交互式环境中的溢出边缘情况。

 ## 算法演练

 1. 查询$X = 0$， 收到$R_0 = A + B$。 这给出了组合两个未知数而没有异或失真的基线。 
2、查询$X = 2^{30} - 1$， 收到$R_1 = (A \oplus (2^{30}-1)) + B$。 这会翻转最低 30 位$A$，产生可预测的转变。 
3. 计算差异$D = R_1 - R_0 = (A \oplus C) - A$。 这删除了$B$，只留下一个纯粹的基于 XOR 的表达式，涉及$A$。 
4. 重建$A$一点一点地利用这一事实$C$反转所有低位。 对于每个位位置，比较在已知反转模式下差异如何变化，以决定原始位是否$A$是 0 或 1。 
5.一次$A$已知，计算$B = R_0 - A$。 这是有效的，因为$R_0$完全包含$A + B$。 
6. 输出$A$和$B$。 

### 为什么它有效

 不变的是每个查询响应都是纯 XOR 变换的线性移位$A$。 通过减去两个响应，我们消除了$B$完全，留下一个确定性函数$A$独自的。 所选择的面膜$2^{30}-1$确保 XOR 充当固定范围上的完整按位补码，从而使变换可以逐位可逆而不会产生歧义。 自从$A$位于内$10^9$，它适合 30 位，因此不会发生高位损坏。 这保证了独特的重建$A$，因此有一个独特的$B$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(x):
    print("?", x, flush=True)
    return int(input().strip())

def main():
    C = (1 << 30) - 1

    r0 = ask(0)
    r1 = ask(C)

    # Let:
    # r0 = A + B
    # r1 = (A xor C) + B
    # so:
    # r1 - r0 = (A xor C) - A

    diff = r1 - r0

    # Reconstruct A bit by bit using complement structure
    A = 0
    for i in range(30):
        bit = 1 << i

        # try setting bit and test consistency via XOR behavior
        # since C flips bits, A xor C = (~A) within 30 bits
        # so A + (A xor C) = C
        # => A + (~A) = C (within 30 bits)

        # derive: A = (C + diff) / 2 is NOT reliable due to signed behavior,
        # so we directly reconstruct using known identity:
        # A xor C = C - A
        #
        # thus r1 - B = C - A and r0 - B = A
        # => (r1 - B) + (r0 - B) = C
        # => r1 + r0 - 2B = C
        # but we eliminate B by:
        # A = r0 - B, B = r0 - A (circular)
        #
        # So we instead compute:
        # A = (C + r0 - r1) // 2

        pass

    # correct closed form:
    A = (C + r0 - r1) // 2
    B = r0 - A

    print("!", A, B, flush=True)

if __name__ == "__main__":
    main()
```核心实现依赖于通过选择全位掩码查询将 XOR 交互减少为线性恒等式。 关键的一步是认识到在固定的 30 位域内，使用全 1 掩码进行异或变换$A$成其按位补码，等于$C - A$。 这将系统转换为两个线性方程$A$和$B$，允许直接代数求解。 

最终公式避免了逐位重建，而是使用对称性：$A \oplus C = C - A$，因此减去响应就可以分离出可解的线性系统。 

一个常见的陷阱是试图将 XOR 差异解释为独立的每位运算； 减法进位会破坏这种方法。 线性恒等式完全避免了这个问题。 

## 工作示例

 考虑一个小的概念示例，其中$A = 3$,$B = 5$， 和$C = 7$（以 3 位进行说明）。 

第一个查询$X = 0$给出：

 | 查询 | 表达 | 结果 |
 | ---| ---| ---|
 | 0 |$3 + 5$| 8 |

 第二次查询$X = 7$:

 | 查询 | 表达 | 结果 |
 | ---| ---| ---|
 | 7 |$(3 \oplus 7) + 5 = (4) + 5$| 9 |

 现在计算：$$C + r_0 - r_1 = 7 + 8 - 9 = 6,\quad A = 6/2 = 3$$然后：$$B = 8 - 3 = 5$$这证实了线性减少的一致性。 

第二次检查$A = 0$,$B = 4$显示了相同的机制：

 | 查询 | 结果 |
 | ---| ---|
 | 0 | 4 |
 | 7 | 7 + 4 = 11 |

 然后：$$A = (7 + 4 - 11)/2 = 0,\quad B = 4$$这表明，即使 XOR 将结构完全折叠为零，线性系统仍然可以正确恢复隐藏值。

 ## 边缘情况

 当$A = 0$，XOR 运算从第一个查询中完全消失，因为$0 \oplus X = X$。 该系统的行为就像一个简单的线性偏移模型，任何依赖按位结构的解决方案都会错误地假设丢失的信息。 线性恒等式仍然有效，因为它不依赖于位之间的异或变化。 

什么时候$A$接近于$2^{30} - 1$，异或与$C$产生的数量非常小，这可能会由于大量取消而误导基于位的重建策略。 基于减法的推导完全避免了这种情况，因为它从不检查各个位。 

什么时候$A$和$B$都是大的中间值，例如$r_0 - r_1$可能会变成负值。 任何假设非负中间值的按位重构方法都会在这里失败。 最终公式可以安全地处理带符号的差异，因为它源自对称线性系统而不是逐位推断。
