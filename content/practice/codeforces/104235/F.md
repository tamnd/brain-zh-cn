 ## 解决方案
 ## 问题理解

 我们生成一个长度为随机的数组$n$，其中每个位置都是独立且统一地从整数中选择的$1$到$k$。 每一位$k^n$数组同样有可能。 

任务是计算结果数组不包含形成严格递增链的四个连续位置的概率。 也就是说，一定不能存在任何索引$i$这样$$a_i < a_{i+1} < a_{i+2} < a_{i+3}.$$只有连续的元素才重要。 非连续递增子序列是无关紧要的。 

输出是一个实数，因此该问题本质上是一个除以的计数问题$k^n$。 我们需要有效数组的数量并将其标准化。 

限制条件$n, k \le 50$意味着两个维度都足够小，可以对涉及以下值的状态进行动态规划$1$到$k$。 然而，$k^n$增长速度极快，因此无法直接枚举。 甚至$50^{50}$尺度推理迫使我们在精心选择的状态表示上进行多项式 DP。 

直接记住最后三个值的天真状态已经导致$O(n \cdot k^3 \cdot k)$转换，这是 Python 中的边界。 更糟糕的是，这样的状态是不必要的，因为条件仅取决于我们是否形成了四个连续递增比较的链，而不取决于最后三个值的确切形状。 

出现微妙的边缘情况时$n < 4$。 在这种情况下，甚至不可能容纳四个连续的元素，因此答案必须始终是$1$。 任何盲目应用 DP 公式的实现仍然有效，但如果不仔细处理标准化，可能会浪费计算或引入除法问题。 

## 方法

 暴力解决方案将生成每个长度的序列$n$，检查所有长度为四的窗口，并验证是否有窗口是严格递增的。 每张支票费用$O(n)$，并且有$k^n$序列，导致$O(n \cdot k^n)$，即使对于最小的限制，这也是远远超出可行的。 

关键的观察是禁止模式仅取决于局部连续比较。 当我们将序列扩展一个元素时，唯一的新风险是我们是否刚刚创建了一系列连续的“递增步骤”，这将对应于连续四个严格递增的值。 

这意味着我们不需要记住完整的最后三个值。 我们只需要跟踪当前严格递增的相邻比较运行的时间有多长。 如果前一个值小于当前值，则延长运行； 否则，它会重置。 游程长度永远不需要超过三，因为禁止达到四。 

这将问题简化为关于位置、最后值和当前递增游程长度的动态规划。 转变是局部的并且独立于历史的其余部分。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n \cdot k^n)$|$O(n)$| 太慢了|
 | 具有游程状态的 DP |$O(n \cdot k^2)$|$O(k)$| 已接受 |

 ## 算法演练

 我们定义一个 DP，从左到右处理数组并维护足够的信息来检测禁止模式的形成。 

1. 我们定义一个状态，表示给定前缀长度的所有有效序列，以特定值结束，并且具有当前严格递增比较后缀的已知长度。 这个后缀长度代表我们连续出现了多少次$a_{i-1} < a_i$在当前位置结束。 
2. 对于位置 1，每个值来自$1$到$k$是可能的，并且增加的游程长度始终为 1，因为没有先前的元素可供比较。 这会初始化 DP 基础层。 
3. 对于每个下一个位置，我们尝试通过选择下一个值来扩展所有先前的有效状态$x$从$1$到$k$。 这会创建从先前值的转换$v$。 
4. 如果新值$x$大于$v$，然后我们将当前增加的游程长度延长一。 否则，递增游程将重置为长度 1。这可以精确模拟严格不等式链是否继续。 
5. 如果结果游程长度变为 4，我们丢弃此转换，因为它形成四个连续的严格递增元素，这是禁止的。 
6. 我们将所有有效转换累积到下一个 DP 层。 
7. 处理完所有位置后，我们将所有结束值和游程长度的所有有效状态相加，然后除以$k^n$以获得概率。 

### 为什么它有效

 DP 状态准确地编码了检测禁止模式所需的唯一信息：最后一个值以及在当前位置结束的连续递增比较的数量。 任何对规则的违反仅取决于是否发生三个连续的增加，这由达到 4 的游程长度完全捕获。由于每次转换都保留了该本地历史的正确性，因此无法计算任何禁止序列，并且每个允许的序列都具有通过 DP 的一条对应路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())

# dp[v][l] = number of ways where last value is v and current increasing run length is l
# l in {1,2,3}
dp = [[0] * 4 for _ in range(k + 1)]

for v in range(1, k + 1):
    dp[v][1] = 1

for _ in range(2, n + 1):
    ndp = [[0] * 4 for _ in range(k + 1)]
    for v in range(1, k + 1):
        for l in range(1, 4):
            if dp[v][l] == 0:
                continue
            cur = dp[v][l]
            for x in range(1, k + 1):
                if x > v:
                    nl = l + 1
                else:
                    nl = 1
                if nl <= 3:
                    ndp[x][nl] += cur
    dp = ndp

total = 0
for v in range(1, k + 1):
    for l in range(1, 4):
        total += dp[v][l]

prob = total / (k ** n)
print(prob)
```DP 表存储有效序列的计数，这些序列按其最后一个值和当前增加的邻接比较的条纹进行分组。 转换显式地迭代所有可能的下一个值，更新增加的条纹是否继续或重置。 

一个微妙的实现细节是当前和下一个 DP 层的分离。 重复使用相同的数组会污染单个步骤内的转换。 

## 工作示例

 ### 示例 1：`3 50`自从$n = 3$，任何序列都不能包含四个连续元素。 DP 仍然运行，但永远不会触发禁止状态。 

| 职位| 状态总结|
 | ---| ---|
 | 1 | 游程长度为 1 的所有 50 个值 |
 | 2 | 所有对均有效，分为递增和非递增转换 |
 | 3 | 仍然不可能出现无效转换 |

 最终计数等于全部$50^3$序列，因此概率恰好为 1。这证实了 DP 正确处理约束为空的短序列。 

### 示例 2：`4 4`这里唯一失败的方法是选择一个严格递增的序列，例如$1,2,3,4$。 

| 步骤| 观察|
 | ---| ---|
 | 构建长度 4 个序列 | 全部的$4^4 = 256$|
 | 无效序列 | 只有一条严格递增链 |
 | 有效序列| 255 | 255

 DP 将准确计数一条达到游程长度 4 的路径并将其丢弃，与分析结果相匹配。 

这表明即使状态空间很小，DP 也能正确隔离单个禁止模式。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n \cdot k^2)$| 对于每个位置，每个状态（最后一个值、游程长度）都会在所有下一个值上转换 |

 和$n, k \le 50$，最大操作次数约为$50 \cdot 50 \cdot 50 = 125000$，完全在 Python 的限制之内。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import math

    n, k = map(int, sys.stdin.readline().split())

    dp = [[0] * 4 for _ in range(k + 1)]
    for v in range(1, k + 1):
        dp[v][1] = 1

    for _ in range(2, n + 1):
        ndp = [[0] * 4 for _ in range(k + 1)]

            for l in range(1, 4):

        dp = ndp

    total = sum(dp[v][l] for v in range(1, k + 1) for l in range(1, 4))
    return str(total / (k ** n))

 ## 边缘情况

 对于$n = 1$，该算法初始化单层状态并立即聚合它们，产生概率 1，因为没有发生任何转换。 DP 正确地避免了任何无效的状态检查。 

为了$k = 1$，每个序列都是常数。 递增游程长度永远不会超过 1，因此不会出现任何禁止模式。 DP 只保持有效状态，最终概率变为 1，无论$n$。 

为了$n = 4$，在可能发生故障的第一个时刻，DP 明确捕获了每个转换都在增加的单个无效轨迹。 当游程长度试图达到 4 时，该路径将被丢弃，以确保计数与组合期望相匹配。
