 ## 解决方案
 ## 问题理解

 我们正在生成一个长度的随机序列`n`，其中每个位置独立地统一取一个值`1`到`k`。 每个序列的可能性均等，因此序列总数为$k^n$，我们想要的概率是满足特定约束的序列的分数。 

如果一个序列不包含形成严格递增链的四个连续元素，则该序列被认为是“好的”。 换句话说，没有索引`i`这样$$a[i] < a[i+1] < a[i+2] < a[i+3].$$任务是计算均匀随机序列避免这种模式的概率。 

限制很小：$n, k \le 50$。 这立即排除了所有序列的任何指数枚举，因为即使对于$n = 50, k = 2$，状态空间为$2^{50}$，这远远超出了枚举。 解决方案必须利用序列的部分结构的动态规划。 

一个关键的微妙之处在于，禁止模式仅取决于连续元素之间的比较，而不是它们的绝对值。 这意味着 DP 不需要跟踪历史中的完整值，只需跟踪短后缀的相对顺序。 

一个幼稚的错误是尝试直接跟踪完整的最后 3 或 4 个值并转换所有选择，但这仍然留下一个$O(k^n)$分支因子在概念上除非被严重压缩。 另一个错误是忘记只有_连续位置的严格递增四倍_才重要，因此非连续递增子序列是无关紧要的。 

## 方法

 暴力方法将生成所有$k^n$序列并检查每一个是否有禁止的模式。 检查序列需要$O(n)$，所以总复杂度是$O(n \cdot k^n)$，即使对于最小的非平凡情况也是完全不可行的。 

结构观察是，条件仅取决于连续元素之间的最后三个比较。 新元素是否创建禁止四元组仅取决于最后三个值。 这表明由最后最多三个选择的值定义的动态编程状态。 

然而，存储实际值仍然太大，因为值的范围最大为 50。关键的简化是观察到，在检查四个连续值是否严格递增时，只有相对顺序才重要。 我们只需要知道长度为 4 的后缀是否严格递增，这仅取决于最后三个值加上新的候选值。 

因此，我们定义 DP 状态来显式存储最后最多三个元素。 自从$n, k \le 50$，我们可以安全地直接枚举值状态：$O(n \cdot k^3)$或更好，取决于压缩。 

更仔细的简化：我们以精确的形式跟踪最后 3 个值。 当我们添加一个新值时`x`，我们检查是否`(a, b, c, x)`是严格递增的； 如果是这样，我们就放弃该转换。 

这会在长度最多为 3 个记住的后缀的序列上产生干净的 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(k^n \cdot n)$|$O(n)$| 太慢了|
 | 最后 3 个值的 DP |$O(n \cdot k^4)$|$O(k^3)$| 已接受 |

 给定$k \le 50$,$k^4 = 6.25 \times 10^6$处于临界状态，但通过修剪和有效转换的小常数因子仍然可行； 我们还观察到大多数转换都是有效的，并且我们只进行不断的检查。 

一个更优雅的观点是：我们正在计算避免长度为 4 的固定禁止模式的序列。这是一个标准的“后缀自动机上的模式避免 DP”问题，其中状态是长度 ≤ 3 的所有可能后缀。 

## 算法演练

 我们从左到右构建序列并维护最后最多三个选择的值。 

1.定义一个DP表，其中`dp[pos][a][b][c]`是长度有效序列的数量`pos`以后缀结尾`(a, b, c)`，其中缺失的条目由特殊的空状态表示。 这使我们能够仅跟踪检测禁止增加运行所需的相关历史记录。 
2. 初始化`dp[1]`通过选择任何值`a`从`1`到`k`,设定状态`(a, empty, empty)`到 1。这反映了长度为 1 的序列不能违反条件。 
3. 对于从 1 到 n-1 的每个位置，迭代所有 DP 状态。 对于每个州`(a, b, c)`对于每个下一个值`x`在`[1, k]`，构造一个新的后缀：

 - 转变`(a, b, c)`到`(b, c, x)`当满时，
 - 或者如果存在的元素少于 3 个，则填充缺失的槽。 
4. 在接受转换之前，检查最后四个连续值是否形成严格递增的序列。 仅当我们已经拥有三个有效的先前值并且`a < b < c < x`。 如果这个条件成立，我们就放弃转换。 
5. 将所有有效转换累积到下一个 DP 层。 
6. 处理完所有位置后，将所有后缀配置中的所有 DP 状态相加，以获得有效序列的总数。 
7.除以$k^n$即可得到最终的概率。 

正确性取决于禁止条件仅取决于最后四个元素，因此维护最后三个元素对于每个转换来说都是足够的上下文。 

### 为什么它有效

 在每一步中，DP 状态都准确保留确定任何新形成的长度为 4 的窗口是否严格递增所需的后缀。 任何较旧的元素都不能参与新的禁止四元组，因为它们不再与当前位置相邻。 因此，具有相同最后三个值的两个部分序列对于未来的有效性是无法区分的，使得状态表示对于约束来说是无损的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())

    # dp[i][a][b][c] compressed via dict to avoid large allocation
    from collections import defaultdict

    dp = defaultdict(int)

    # initial states: sequences of length 1
    for x in range(1, k + 1):
        dp[(x, 0, 0)] += 1

    for _ in range(1, n):
        ndp = defaultdict(int)

        for (a, b, c), cnt in dp.items():
            for x in range(1, k + 1):
                if b == 0:
                    na, nb, nc = x, 0, 0
                elif c == 0:
                    na, nb, nc = a, x, 0
                else:
                    na, nb, nc = b, c, x
                    if a < b < c < x:
                        continue

                ndp[(na, nb, nc)] += cnt

        dp = ndp

    print(sum(dp.values()) / (k ** n))

if __name__ == "__main__":
    solve()
```DP 使用最多三个值的元组来压缩后缀状态，其中`0`充当未使用职位的哨兵。 转换逻辑仔细区分部分填充和完整后缀，确保仅当存在四个实值时才应用禁止检查。 

划分由$k^n$将计数转换为概率。 

一个微妙的实现细节是避免 DP 期间的浮点累加； 我们只在最后进行转换以保持数值稳定性。 

## 工作示例

 ### 示例 1：`n = 3, k = 50`由于禁止模式需要四个连续元素，因此任何长度为 3 的序列都自动有效。 

| 邮政 | 活跃状态| 过渡| 总计 |
 | ---| ---| ---| ---|
 | 1 | 单个元素 | k 个选择 | 50 | 50
 | 2 | 成对| 全部有效 | 2500 | 2500
 | 3 | 三倍| 全部有效 | 125000 | 125000

 所有序列都存活下来，因此概率为 1。 

这证实了 DP 永远不会触发禁止条件，因为它永远不可能形成四个连续的元素。 

### 示例 2：`n = 4, k = 4`这是存在禁止模式的最小情况。 

只有一个序列无效：`(1,2,3,4)`。 

| 步骤| 解读|
 | ---| ---|
 | 1 | 所有单一值 |
 | 2 | 允许所有配对 |
 | 3 | 允许所有三元组|
 | 4 | 删除严格增加的四倍 |

 总序列：$4^4 = 256$。 有效：255。 

DP 完全排除了最后三个值所在的状态`(1,2,3)`接下来是`4`。

 ## 边缘情况

 当`n < 4`，该算法永远不会正确触发禁止检查，因为没有状态会累积四个连续元素。 每个序列都会被计数，因此概率恰好为 1。 

当`k = 1`，每个序列都是常数，所以严格增加是不可能的。 DP仍处于单一状态`(1,0,0)`整个过程中，每个转换都是有效的。 

什么时候`n = 4`，唯一禁止的配置是长度为 4 的严格递增链。DP 仅在所有四个后缀值都存在的最终转换时检测到这一点，与问题的约束完全匹配。
