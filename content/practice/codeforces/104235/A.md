 ## 解决方案
 ## 问题理解

 我们给定一个高度为的矩形窗口`H`和宽度`W`。 进行两次切割。 

首先，我们在某个整数位置选择垂直切割`A`，将窗口分割为宽度为左的矩形`A`和一个宽度为直角的矩形`W - A`。 

其次，仅在右侧矩形内，我们选择高度水平切割`B`，将其分成顶部和底部的高度矩形`B`和`H - B`。 

这会产生三个矩形：

 左侧部分、右上部分和右下部分。 他们的面积是`S1 = H * A`,`S2 = B * (W - A)`,`S3 = (H - B) * (W - A)`。 

目标是选择整数`A`和`B`使得这三个区域中最小的一个尽可能大。 

约束很小：两个维度都最多 1000。这已经表明，即使是`O(HW)`或者`O(W)`每个测试方法都是完全安全的，而任何立方体都是不必要的。 

一个幼稚的错误是尝试所有`(A, B)`直接配对。 最多有一百万个状态，它仍然可以通过，但它隐藏了结构，如果试图错误地优化，可能会导致错误的推理。 

当试图独立对待这三个区域时，会出现一个更微妙的陷阱。 例如，人们可能会尝试分别最大化每个区域或同时平衡所有三个区域。 那失败了，因为`S2`和`S3`通过相同的因素耦合`(W - A)`并且对称于`B`。 

还有一个隐藏的边缘约束：`A`必须至少为 1，最多`W-1`， 相似地`B`必须介于`1`和`H-1`。 这可以防止出现一个区域消失的退化情况，任何解决方案都必须尊重这一点。 

## 方法

 直接的暴力策略选择每一个有效的对`(A, B)`并计算三个面积。 这是正确的，因为它评估了精确的目标函数，但它需要粗略地迭代`(W-1)(H-1)`状态，在最坏的情况下大约需要一百万次操作。 这在约束下是可以接受的，但是很浪费并且没有使用表达式的结构。 

关键的观察是对于固定的垂直切割`A`，右侧变成一个大小为的矩形`(W - A) × H`。 在里面我们选择`B`将其水平分割。 左侧区域`S1`独立于`B`， 尽管`S2`和`S3`只取决于我们如何分割高度。 

对于固定`A`，问题简化为最大化`min(B, H - B)`因为两者`S2`和`S3`乘以相同的因子`(W - A)`。 最大化两个对称线性函数最小值的最佳方法是尽可能均匀地分割，因此`B`应该尽可能接近`H/2`尽可能。 这给出了最佳可能值`floor(H/2)`对于较小的一侧。 

所以对于每一个`A`，其中最佳可实现的最小值`S2`和`S3`变成`(W - A) * floor(H/2)`。 现在第三个值`S1 = H * A`与这个数量竞争，所以我们将问题简化为单个变量优化`A`。 

此时，问题就变成了一个一维最大值`min(H*A, C*(W-A))`在哪里`C = floor(H/2)`。 该函数随着`A`在第一项中，在第二项中减少，因此最优值位于它们的交点附近。 自从`W ≤ 1000`，我们可以安全地评估所有可能的`A`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对 A、B 进行暴力破解 | O(硬件) | O(1) | O(1) | 已接受 |
 | A 上的优化减少 | O(W) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题逐步转化为单变量优化。 

1. 计算`C = floor(H / 2)`。 

这代表了最佳可能值`min(B, H - B)`对于任何有效的水平分割。 
2.修复垂直切割位置`A`从`1`到`W - 1`。 

每个选择都确定左侧矩形和右侧矩形的宽度。 
3. 对于每个`A`，计算两个竞争量：`S1 = H * A`和`S23 = C * (W - A)`。 

价值`S23`表示最佳选择后右上角和右下矩形之间的最佳最小值`B`。 
4、为此`A`，可实现的最佳答案是`min(S1, S23)`。 

这反映出所有三个区域都必须至少有这么大，因此其中最小的一个是限制因素。 
5. 跟踪该表达式在所有表达式中的最大值`A`。 

最终的答案是最佳平衡点，其中左侧区域和最佳右侧最小值尽可能接近。 

### 为什么它有效

 对于固定的`A`，除了通过因子之外，右侧与左侧无关`(W - A)`。 右侧矩形内的水平分割问题是对称且线性的，因此其最优策略始终是尽可能均匀地分割。 这减少了二维决策`(A, B)`转化为单个参数`A`而不失去最优性。 由于每个可行的配置都由一些表示`(A, B)`，并且对于每个`A`我们选择最优的`B`，除了这种减少之外，不存在更好的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    H, W = map(int, input().split())

    C = H // 2
    best = 0

    for A in range(1, W):
        s1 = H * A
        s23 = C * (W - A)
        best = max(best, min(s1, s23))

    print(best)

if __name__ == "__main__":
    solve()
```实施直接遵循简化的公式。 循环结束`A`探索所有有效的垂直切割。 对于每一个，`s1`是左侧区域并且`s23`是最佳放置水平切割后从右侧可实现的最佳瓶颈。 这`min`强制所有三个区域至少与其中最小的区域一样大。 

一个常见的实施错误是忘记最佳的`B`不需要显式迭代。 另一个正在使用`H/2`而不是`floor(H/2)`，这会破坏整数正确性`H`很奇怪。 

## 工作示例

 ### 示例 1

 输入：```
2 2
```这里`C = 1`。 

| 一个 | S1 = H*A | S23 = C*(W-A) | S23 = C*(W-A) | 分钟（S1，S23）|
 | ---| ---| ---| ---|
 | 1 | 2 | 1 | 1 |

 最好的价值是`1`。 

这显示了极端情况，即两次切割都被迫达到最小尺寸，并且在超出单个单位面积时不可能实现平衡。 

### 示例 2

 输入：```
2 3
```这里`C = 1`。 

| 一个 | S1 = H*A | S23 = C*(W-A) | S23 = C*(W-A) | 分钟（S1，S23）|
 | ---| ---| ---| ---|
 | 1 | 2 | 2 | 2 |
 | 2 | 4 | 1 | 1 |

 最大值为`2`，达到时`A = 1`。 

这证明了权衡：增加`A`改进左侧矩形，但缩小右侧矩形的速度快于增益。

 ## 边缘情况

 当出现一种边缘情况时`H = 2`。 然后`C = 1`，并且水平切割被强制分成两个高度为1的条带。 该算法可以正确处理这个问题，因为`min(B, H-B)`始终为 1，因此右侧变为`(W - A)`准确地说，优化简化为平衡`2A`和`W - A`。 

另一个边缘情况是当`W = 2`。 那么只有一个有效的垂直切割`A = 1`，因此循环仅评估一种配置。 结果是`min(H, floor(H/2))`，与右侧矩形的强制结构相匹配。 

最后一个微妙的情况是`H`很奇怪。 使用`H // 2`确保分割永远不会高估可实现的最小值。 由于无效的非整数区域假设，任何使用浮动除法的尝试都会产生不正确的比较。
