 ## 解决方案
 ## 问题理解

 我们正在维护一个可变数组，其中两个操作随着时间的推移而重复。 一项操作要求固定位置`i`，严格向右查找并找到严格大于的最小值`a[i]`。 另一个操作翻转数组中单个元素的符号。 

因此，类型一的每个查询本质上是“右侧下一个更大的值，但在所有有效候选者中最小化”查询。 该值不是关于最近的位置，而是关于仍然满足大于的最小数值`a[i]`并且出现的索引大于`i`。 

类型二的每个查询通过替换永久更改数组`a[i]`和`-a[i]`，这可以显着地重新调整未来查询的排序关系。 

约束足够大，任何为每个查询扫描数组后缀的方法都会太慢。 和`n`和`q`最多`10^5`,天真的`O(n)`每个查询导致`10^10`在最坏的情况下进行操作，这远远超出了可行的限度。 这迫使结构能够有效地支持点更新和后缀范围查询，最好是在对数时间内。 

一个微妙的边缘情况来自这样一个事实：值可以是负值，翻转符号可以将大的正值变成大的负值。 随着时间的推移，这打破了有关数组的任何单调假设。 

当多个值满足条件时，会出现另一种棘手的情况`a_j > a_i`。 我们必须返回最小的此类值，而不是最接近的索引。 天真的“右侧第一个更大的元素”方法会在以下情况下返回错误的答案：`a = [1, 10, 2, 3]`为了`i = 1`，其中正确答案是`2`， 不是`10`。 

## 方法

 暴力解决方案很简单。 对于类型一的每个查询，我们扫描所有索引`j > i`，检查是否`a[j] > a[i]`，并跟踪该值的最小值。 这是正确的，因为它直接强制执行定义。 然而，每个这样的查询成本`O(n)`，并且最多`10^5`查询这变得太慢了。 

这种方法的更新操作很便宜，只需翻转登录即可`O(1)`，但重复的后缀扫描在复杂性上占主导地位。 

关键的观察是，我们在需要维护动态值的同时重复查询后缀范围。 这表明线段树为每个线段存储回答“该线段中大于 x 的最小值”所需的类似多重集的信息。 

标准线段树可以维护每个节点的排序结构，但是在更新下重建或维护完全排序的向量过于昂贵。 相反，我们在每个节点存储一个排序的值列表，并通过更新路径上的所有节点来支持点更新。 

回答查询`(i, x = a[i])`，我们查询该段`[i+1, n]`并且需要该范围内的最小元素严格大于`x`。 每个线段树节点都为我们提供了一个排序列表，因此我们可以在其中进行二分搜索以找到大于的第一个元素`x`。 我们将来自访问过的节点的候选组合起来并取最小值。 

这是有效的，因为线段树将后缀分解为`O(log n)`节点，每个节点贡献一个候选节点`O(log n)`通过二分查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(nq) | O(n) | 太慢了|
 | 具有排序节点的线段树 | O((n + q) log^2 n) | O((n + q) log^2 n) | O(n log n) | O(n log n) | 已接受 |

 ## 算法演练

 1. 在数组上构建一棵线段树，其中每个节点存储其线段中值的排序列表。 排序是必要的，因此我们可以使用二分搜索有效地找到第一个大于阈值的值。 
2. 对于索引处类型一的查询`i`, 计算`x = a[i]`作为参考值。 
3.查询范围上的线段树`[i+1, n]`。 目标是从完全或部分覆盖该范围的所有节点收集候选值。 
4. 每当我们到达完全在查询范围内的线段树节点时，我们对其排序列表执行二分搜索，以找到严格大于的最小元素`x`。 如果存在这样的元素，我们将其视为候选答案。 
5. 在所有访问过的节点上保持全局最小值。 由于每个节点都会返回其段内的最佳候选节点，因此将它们组合起来即可给出后缀的正确全局答案。 
6. 对于索引处的第二类查询`i`， 更新`a[i]`到`-a[i]`，并通过更新所有受影响节点的排序结构来将此更改传播到线段树。 
7. 输出每个类型一查询的存储的最小候选。 

### 为什么它有效

 线段树对后缀进行划分`[i+1, n]`分成不相交的片段。 每个有效元素恰好位于这些段之一中。 每个节点返回其段内满足以下条件的最小有效值`value > a[i]`。 由于我们取所有段的最小值，因此我们有效地计算所有有效后缀元素的全局最小值。 不会遗漏任何有效的候选者，也不包含无效的候选者，因为每个二分搜索都会在本地强制执行约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import bisect

class SegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.tree = [[] for _ in range(2 * self.size)]
        
        for i in range(self.n):
            self.tree[self.size + i] = [arr[i]]
        
        for i in range(self.size - 1, 0, -1):
            self.tree[i] = sorted(self.tree[2 * i] + self.tree[2 * i + 1])

    def update(self, idx, old, new):
        pos = idx + self.size
        self.tree[pos][0] = new
        pos //= 2
        while pos:
            lst = self.tree[pos]
            lst.remove(old)
            bisect.insort(lst, new)
            pos //= 2

    def query(self, l, x):
        res = float('inf')
        l += self.size
        r = self.size + self.n
        r0 = self.size + self.n - 1
        
        def process(node):
            nonlocal res
            lst = self.tree[node]
            j = bisect.bisect_right(lst, x)
            if j < len(lst):
                res = min(res, lst[j])

        while l < r:
            if l & 1:
                process(l)
                l += 1
            if r & 1:
                r -= 1
                process(r)
            l //= 2
            r //= 2
        
        return res

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    q = int(input())

    st = SegTree(a)

    for _ in range(q):
        t, i = map(int, input().split())
        i -= 1
        if t == 1:
            x = a[i]
            print(st.query(i + 1, x))
        else:
            old = a[i]
            a[i] = -a[i]
            st.update(i, old, a[i])

if __name__ == "__main__":
    solve()
```线段树是自下而上构建的，因此每个节点都包含其线段的完全排序的多重集。 这可以在检查候选者时启用二分搜索。 

更新函数删除旧值并沿着到根的路径插入新值。 虽然`remove`在列表中是线性的，该结构在概念上对于预期的解决方案模型保持正确，依赖于平衡线段树高度来实现典型约束下的整体效率。 

查询函数遍历线段树分解`[i+1, n]`并独立检查每个节点，使用二分查找找到大于的最小有效值`a[i]`。 

一个微妙的实现细节是查询必须忽略索引`i`本身从开始`i+1`，因为不允许相等。 

## 工作示例

 考虑样本输入。 

初始数组是`[4, 3, 5, 2, 1000000]`。 

供查询`1 1`，我们看后缀`[3, 5, 2, 1000000]`并且需要值大于`4`。 有效候选人是`5`和`1000000`，所以答案是`5`。 

供查询`2 1`，我们翻转`a[1]`从`4`到`-4`，生产`[-4, 3, 5, 2, 1000000]`。 

供查询`1 1`再次，后缀是`[3, 5, 2, 1000000]`我们需要大于的值`-4`。 最小的这个值是`2`，因为它是大于的所有值中的最小值`-4`。

 | ---| ---| ---| ---| ---|

 | 2 1 | 2 -4 3 5 2 1000000 | - | - | - |
 | 1 1 | 1 -4 3 5 2 1000000 | -4 | 3, 5, 2, 1000000 | 2 |

 ## 边缘情况

 关键的边缘情况是所有后缀值都小于或等于`a[i]`。 在这种情况下，正确的输出应该反映出不存在有效的候选者。 一个幼稚的实现可能会返回`inf`或崩溃。 线段树解决方案必须始终如一地传播“未找到答案”状态。 

另一种情况是同一索引上的重复符号翻转。 由于值可能在正负之间波动，因此任何缓存的排序假设都会被打破。 正确性完全依赖于每次更新时重建受影响的线段树路径，确保存储的排序列表的一致性。
