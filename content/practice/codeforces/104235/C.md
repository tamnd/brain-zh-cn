 ## 解决方案
 ## 问题理解

 我们正在与一棵树合作$n$顶点 1 和顶点 2 以特殊方式固定的顶点：它们之间的距离恰好是$m$，并且树中没有一对顶点之间的距离大于$m$。 所以树的直径最大$m$，并且顶点 1 和 2 的距离被迫精确等于该上限。 

一个顶点$v$如果它同时位于远处，则称为“远”$m$从顶点 1 以及距离$m$来自顶点 2。任务是确定在满足约束的所有可能的树中可以存在多少这样的顶点。 

约束条件达到$n = 10^5$，这会立即排除任何试图枚举树木或评估所有结构距离的构造。 任何平方项$n$将会失败。 该解决方案必须依赖于树的结构特性和距离几何形状，而不是显式枚举。 

一个微妙的点是条件“所有距离最多$m$” 与“dist(1,2)=m”相结合会形成一个非常紧密的结构：1 和 2 的行为就像直径路径的相对两端。任何假设围绕 1 和 2 进行任意分支的幼稚尝试都将违反全局直径约束，这就是许多不正确的构造失败的地方。

 一个常见的错误是将问题视为独立于第 1 层和第 2 层的 BFS 层，然后将这些层相交。 这忽略了同一顶点必须遵循单个树结构，而不是两个独立的度量。 

边缘情况立即出现在小$n$。 例如，当$n=3, m=2$，树被迫成为一条路径，因此没有顶点可以同时距离两端为 2，从而给出输出 0 0。另一个微妙的情况是当$m$接近于$n$，树几乎被迫形成一条长链，严重限制了分支。 

## 方法

 蛮力的观点是生成所有带标签的树$n$顶点（凯莱公式表明$n^{n-2}$可能性）并测试每个是否满足直径约束以及 1 和 2 之间的固定距离。对于每个有效树，我们将计算所有对的最短路径并计算满足“远”条件的顶点。 即使对于$n=20$，因为树木的数量呈超指数增长，并且每次检查的成本至少$O(n)$。 

关键的观察是条件“所有距离最多$m$” 使树的直径精确$m$，并且 1 和 2 必须是某个直径路径的端点。 一旦这个问题被修复，每个顶点都位于一个受约束的几何区域，该区域由其相对于 1 和 2 之间的路径的位置确定。 

任意顶点$v$满足$dist(1,v)=m$必须位于以一种非常特定的方式附加到一个端点的子树上：它必须尽可能远离 1。对于 2 也是如此。同时满足这两个条件的唯一方法是附加在距离 1-2 路径的两个端点“剩余距离预算”等距的区域中。 

这将问题简化为推理在主干是长度路径的树中两个距离壳的交点处可以放置多少个顶点$m$。 最大化和最小化这个交集的结构取决于我们如何分配剩余的$n-m-1$顶点作为沿直径路径的分支。 

在最佳结构中，所有附加顶点都可以附加到 1-2 路径的单个内部节点或分布以避免在远处创建附加顶点$m$从两端。 

由此产生的行为简化为一个干净的组合事实：“远”顶点的数量仅取决于是否可以以创建至少一个交点的方式放置额外的顶点，以及是否可以将它们全部集中以最大化重叠。

 ## 算法演练

 任何有效树的结构都由顶点 1 和顶点 2 之间的唯一路径锚定，该路径的长度为$m$。 我们认为这条道路是$m+1$顶点。 

1.修复简单路径$1 \rightarrow \dots \rightarrow 2$长度$m$。 这是被迫的，因为 1 和 2 之间的距离恰好是$m$，并且树中不允许有更大的距离，所以这条路径就是直径。 
2. 剩余的全部$n-(m+1)$顶点必须作为子树附加到沿着该主干的顶点。 任何附件都会增加到两个端点的距离，具体取决于它沿路径放置的位置。 
3. 一个顶点$v$如果达到距离就很远$m$从两个端点。 这意味着它在主干上的投影必须使得两个距离同时最大化。 具体来说，如果一个顶点附加在位置$i$沿着路径，它到 1 的距离是$i + depth(v)$，并且到 2 是$(m-i) + depth(v)$。 
4. 双方平等$m$，我们需要：$$i + d = m \quad \text{and} \quad (m - i) + d = m$$减去得到$i = m-i$， 所以$i = m/2$。 然后$d = m/2$。 这立即表明远顶点只能存在于$m$是偶数，并且它们必须位于以主干的中点为根的子树中。 
5. 因此，如果$m$是奇数，没有顶点可以同时满足两个方程，所以答案是$0\ 0$。 
6. 如果$m$是均匀的，主干上有一个唯一的中间顶点。 仅深度子树中的顶点$m/2$是候选人。 
7. 要最大化远顶点的数量，请以形成以中点为根的完整树的方式附加所有剩余顶点，从而最大化精确深度处的节点数量$m/2$。 最好的情况是将所有额外的节点放置在一个完整的星状结构中，保留深度约束，允许所有$n-(m+1)$正确放置时可能做出贡献的顶点。 
8. 为了最大限度地减少数量，请分布附件，以便没有顶点达到精确的深度$m/2$，有效地将所有额外的顶点推到错误的深度或错误的骨干位置，实现零。 

### 为什么它有效

 每个顶点到 1 和 2 的距离完全由其在固定直径路径上的附着点加上其在附着子树中的深度决定。 远距离方程组崩溃为同时固定附着指数和深度的单一约束。 这种刚性消除了自由：中点要么支持这样的子树，要么不支持。 一旦确定了这种结构，所有全局树配置都会减少为节点的重新分配，而不会改变可行性条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    # If m is odd, midpoint does not exist as integer position
    if m % 2 == 1:
        print("0 0")
        return

    # number of nodes outside backbone
    extra = n - (m + 1)
    
    # if no extra nodes, only backbone exists and no far vertices
    if extra <= 0:
        print("0 0")
        return

    # In even case, at most one configuration creates far vertices
    # minimum is 0, maximum is all extra nodes + possibly midpoint itself
    print(0, extra)

if __name__ == "__main__":
    solve()
```代码首先检查奇偶性$m$。 什么时候$m$是奇数，端点之间的对称性无法对齐，因此没有顶点可以同时满足两个距离约束。 什么时候$m$是偶数，我们计算有多少顶点不属于强制直径路径。 这些顶点是唯一可以排列为可能成为远顶点的顶点。 最小值是通过避免临界深度配置的方式放置它们来实现的，而最大值是通过将结构集中在中点周围来实现的，以便每个额外的顶点都有助于远距离条件。 

必须注意骨干尺寸$m+1$，因为此处的相差一错误会导致负额外计数或不正确的可行性假设。

 ## 边缘情况

 当$m$为奇数，主干网没有整数中点，因此没有顶点可以同时满足到两个端点的剩余距离相等。 例如，使用输入$5\ 3$，直径路径有 4 个顶点，任何候选顶点总是比另一个端点更接近一个端点，因此无法满足远条件。 

什么时候$n = m+1$，这棵树被迫成为一条路径。 没有其他顶点可以放置在任何地方，因此即使存在中点，也无法在所需深度创建顶点。 例如，$5\ 4$产生一条直链并且没有远顶点。 

当额外的顶点存在但远离中点时，它们会立即失效两个距离约束之一，因为它们沿主干的距离不平衡会阻止同时相等。 这解释了为什么最大化需要集中在中心而不是分布式分支。
