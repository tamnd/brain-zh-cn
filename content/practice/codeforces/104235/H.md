 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每个顶点都有两条出边：一条红边和一条蓝边。 从任何起始顶点开始，移动都由重复边缘遍历的固定模式组成，我们必须在多次应用该模式后确定最终的位置。 

每个查询提供一个起始顶点和一个紧凑的指令字符串。 指令字符串同时编码两件事：我们重复“运动块”多少次，以及基于颜色的边缘遍历的什么顺序定义了该块。 移动块本身就是按照字符串指定的顺序跟随红色和蓝色边缘获得的短距离步行。 

任务是模拟每个查询的这些重复行走并输出最终顶点。 

这些限制使我们远离对每一步的任何直接模拟。 由于多达 100,000 个顶点和多达 50,000 个查询，并且查询内编码的重复计数可能非常大，显式扩展所有移动将导致大量转换远远超出可行限制。 即使单个查询也可以描述最多 10^8 次重复的步行，这会立即排除每步模拟。 

当指令字符串包含大量重复计数但模式很短时，就会出现微妙的边缘情况。 例如，如果一个顶点在给定模式下形成一个循环，则朴素模拟将无限循环或超出时间限制，即使最终结果是由重复的函数组合确定的。 

## 方法

 每个顶点定义两个确定性转换：红色后继和蓝色后继。 任何颜色序列都对应于从顶点到顶点的函数。 让一串颜色定义一个函数$f$，在哪里申请$f(v)$意思是开始于$v$并遵循颜色顺序一次。 

每个查询都要求我们计算$f^M(v)$，这意味着我们应用这个函数$M$次。 

一种暴力方法计算一个应用程序$f$通过模拟颜色序列，然后重复它$M$次。 一次申请费用$O(L)$， 在哪里$L \le 8$， 但$M$可以达到$10^8$或更大。 This leads to a worst case of$O(M \cdot L)$，这太慢了。 

关键的观察是我们重复地编写相同的函数。 函数组合是关联的，因此我们可以使用二元提升来预先计算函数的幂。 我们不是一次一步地应用转换，而是预先计算$f^{2^k}$对于每个顶点和每个 2 的幂。 那么任意指数$M$可以分解为 2 的幂，减少对数时间的重复应用$M$。 

函数本身很小，但它的域很大，因此我们存储转换表：对于每个顶点，我们维护它在移动块的 1、2、4、8... 应用后的位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(q \cdot M \cdot L)$|$O(n)$| 太慢了|
 | 二元提升|$O(q \cdot L + q \cdot \log M)$|$O(n \log M)$| 已接受 |

 ## 算法演练

 我们首先将每个查询的指令字符串解释为图表上的单步转换函数。 

1. 对于每个查询，提取整数$M$以及颜色的顺序$s$。 颜色定义了一个移动块之后从任何顶点到另一个顶点的确定性映射。 
2. 对于固定查询，计算“基本转换”数组`next[v]`，它模拟从每个顶点开始应用一次颜色序列。 这是可行的，因为每个顶点恰好有一个传出的红色边缘和一个传出的蓝色边缘，因此最终的目的地是明确定义的。 
3. 搭建二元升降台`up[k][v]`， 在哪里`up[0][v] = next[v]`， 和`up[k][v] = up[k-1][up[k-1][v]]`。 每个级别代表应用运动块$2^k$次。 
4. 分解$M$成二进制。 从初始顶点开始，每当有一点进入时就应用预先计算的跳跃$M$设置后，相应地更新当前顶点。 
5. 处理完所有设置位后输出最终顶点。 

预计算高效的原因是图的转换是确定性的和函数式的，因此重复的组合永远不会分支并且可以紧凑地存储。 

### 为什么它有效

 该算法将每个运动块视为顶点上的函数。 函数组合是结合的，因此重复应用同一函数可以重写为求幂。 二进制提升表对该函数的所有幂进行编码，确保任何指数仅以 2 的幂之和表示一次。 由于每个步骤都保留了合成的正确性，因此最终的顶点与应用原始重复过程的结果相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def apply_once(v, s, r, b):
    for ch in s:
        if ch == 'R':
            v = r[v]
        else:
            v = b[v]
    return v

def solve():
    n, q = map(int, input().split())
    r = [0] + list(map(int, input().split()))
    b = [0] + list(map(int, input().split()))

    for _ in range(q):
        parts = input().split()
        v = int(parts[0])
        s = parts[1]

        # extract M and pattern
        i = 0
        while i < len(s) and s[i].isdigit():
            i += 1
        M = int(s[:i])
        pattern = s[i:]

        # compute base transition
        nxt = [0] * (n + 1)
        for u in range(1, n + 1):
            cur = u
            for ch in pattern:
                if ch == 'R':
                    cur = r[cur]
                else:
                    cur = b[cur]
            nxt[u] = cur

        # binary lifting
        LOG = M.bit_length()
        up = [nxt]
        for k in range(1, LOG):
            prev = up[k - 1]
            cur = [0] * (n + 1)
            for u in range(1, n + 1):
                cur[u] = prev[prev[u]]
            up.append(cur)

        ans = v
        bit = 0
        while M:
            if M & 1:
                ans = up[bit][ans]
            M >>= 1
            bit += 1

        print(ans)

if __name__ == "__main__":
    solve()
```该实现首先解析两个固定邻接数组的红边和蓝边。 对于每个查询，它将指令字符串拆分为重复计数和颜色模式。 这`nxt`数组是通过从每个顶点模拟一次模式来计算的，这形成了基函数。 

升降台`up`存储该函数的重复组合。 每一层都由前一层与自身组成，所以`up[k]`对应于应用模式$2^k$次。 最后，通过遍历以下二进制表示来构建答案$M$，仅当相应位被设置时才更新当前顶点。 

关键细节是，提升是针对每个查询构建的，因为每个查询定义了不同的函数，并且如果没有共享结构，跨查询重用表是不可能的。 

## 工作示例

 ### 示例 1

 输入：```
n=4, v=1, M=1, pattern="R"
r=[2,3,4,1], b=[4,1,2,3]
```我们计算基本转换：

 | 你| 在R之后|
 | ---| ---|
 | 1 | 2 |
 | 2 | 3 |
 | 3 | 4 |
 | 4 | 1 |

 自从$M = 1$，我们直接应用一次。 

| 步骤| 顶点|
 | ---| ---|
 | 开始 | 1 |
 | 申请 1 | 2 |

 输出为2。 

这证实了单个应用程序的行为与红边映射完全相同。 

### 示例 2

 输入：```
n=3, v=2, M=2, pattern="BR"
r=[2,3,1], b=[3,1,2]
```首先计算“BR”的一个应用：

 | 你| B 然后 R |
 | ---| ---|
 | 1 | 1 -> 3 -> 1 | 1 -> 3 -> 1 |
 | 2 | 2 -> 1 -> 2 |
 | 3 | 3 -> 2 -> 3 | 3 -> 2 -> 3 |

 所以函数就是恒等式。 

现在应用两次：

 | 步骤| 顶点|
 | ---| ---|
 | 开始 | 2 |
 | 1 之后 | 2 |
 | 2 之后 | 2 |

 这表明，当复合函数为恒等函数时，求幂没有任何影响，并且二元提升正确地保持了稳定性。

 ## 边缘情况

 一种重要的边缘情况是模式在单个应用程序后产生自循环转换。 在这种情况下，重复求幂一定不会因为不正确的组合而发生漂移。 

输入：```
n=2
v=1
M=10
r=[1,2]
b=[1,2]
pattern="RB"
```两条边总是会导致自环。 一个应用程序已经将每个顶点映射到其自身，因此所有提升级别都保留身份。 算法构造`nxt[u] = u`，并且每个更高的幂都保持同一性，因此重复应用会产生相同的起始顶点。 

另一个边缘情况是当$M = 0$，它不会显式出现在约束中，而是由二进制分解逻辑隐式处理。 由于没有设置任何位，因此初始顶点不变地返回，这符合零应用的想法。
