 ## 解决方案
 ## 问题理解

 从四个整数的算术级数中删除一个元素后，三个数字仍按其原始顺序保留在棋盘上。 我们必须恢复丢失的数字并报告它所属的位置，以便四个结果数字形成算术级数。 

输入恰好包含三个正整数。 它们的顺序无法更改，因为它们已经以与原始进程相同的顺序出现。 输出由缺失的整数及其在四个数字中的位置组成。 位置`1`表示数字插入到所有给定值之前，位置`2`和`3`意味着它被插入到连续值和位置之间`4`意味着它被附加在最后。 

输入值最多为`10^5`，但只有三个。 由于输入大小是恒定的，因此任何仅执行固定工作量的算法都是有效的`O(1)`。 不需要复杂的数据结构或优化。 

有几种情况需要谨慎处理。 

如果缺失的数字位于级数的一端，则两个现有差异相等。 例如，```
2
4
6
```正确答案是`8`在位置`4`， 虽然`0`在位置`1`也会产生算术级数。 解决方案应该接受任一有效答案，而不是假设缺失值必须始终位于序列内。 

如果缺失的数字位于第一个值和第二个值之间，则首先出现较大的差距。```
10
16
19
```差异是`6`和`3`。 较大的差值必须分成两个相等的差值，所以答案是`13`在位置`2`。 简单地检查两个差异是否相等会错过这种情况。 

如果缺失的数字位于第二个和第三个值之间，则较大的差距出现在第二个。```
10
13
19
```差异是`3`和`6`。 答案是`16`在位置`3`。 该算法必须区分哪个间隙更大，以便将值插入到正确的位置。 

如果三个数字都相同，```
5
5
5
```进展有共同点`0`。 除以下值外的任何插入值`5`会破坏进度。 正确答案是`5`在任一端。 

## 方法

 一个简单的想法是尝试在四个可能的位置中的每一个中插入一个数字，构造结果序列，并检查它是否是算术级数。 唯一剩下的问题是插入哪个值。 由于每个有效级数完全由其公差决定，因此我们可以从现有数字中导出候选值并验证每种可能性。 这种方法是正确的，因为只有四个插入位置和恒定数量的候选者。 

关键的观察结果是，只要缺少一个元素，就一定会发生三种情况之一。 缺失的数字要么位于第一个元素之前，要么位于最后一个元素之后，或者位于两个现有间隙之一内。 如果它位于间隙内，则该间隙必须恰好是公差的两倍，而另一个间隙已经等于公差。 如果它位于一端，则观察到的两个间隙已经相等。 

这一观察结果让我们可以通过比较两个连续的差异来直接确定答案。

 ## 算法演练

 1.读出三个数字`a`,`b`， 和`c`。 
2. 计算连续差值`d1 = b - a`和`d2 = c - b`。 
3.如果`d1 == d2`，给定的数字已经是算术级数的连续元素。 在任一端再插入一个元素。 输出`c + d1`在位置`4`。 
4.如果`d1 > d2`，第一个间隙是原来的两倍。 缺失的数字位于第一个和第二个数字之间。 输出`a + d2`在位置`2`。 

第一个间隙等于`2 * d2`，所以插入`a + d2`将其分成两个大小相等的间隙`d2`。 
5. 否则，`d2 > d1`。 第二个差距是两倍。 输出`b + d1`在位置`3`。 

第二个间隙等于`2 * d1`，所以插入`b + d1`使得两个产生的间隙等于`d1`。 

### 为什么它有效

 输入保证来自四个整数的算术级数。 

如果缺失的元素位于一端，则其余三个数字是级数的连续元素，因此观察到的两个差异已经相等。 

如果缺失的元素位于级数内部，则观察到的一个间隙会恰好跳过一个已删除的元素。 该差距是公差的两倍，而另一个观察到的差距已经等于公差。 比较两个差异可以唯一地确定必须分割哪个间隙，并且在远离其左端点的地方插入一个共同差异可以重建原始进程。 

由于这些是唯一可能的配置，因此算法始终返回正确的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a = int(input())
b = int(input())
c = int(input())

d1 = b - a
d2 = c - b

if d1 == d2:
    print(c + d1)
    print(4)
elif d1 > d2:
    print(a + d2)
    print(2)
else:
    print(b + d1)
    print(3)
```该程序首先计算两个现有的间隙。 他们的关系完全决定了失踪号码的归属。 

当间隙相等时，这三个数字已经是级数的连续成员，因此在末尾扩展序列始终有效。 从一开始就延长也可以，但只需要一个有效答案。 

当一个间隙较大时，输入保证意味着它必须恰好是较小间隙的两倍。 插入的值是距较大间隙左端点的一个较小间隙距离。 

所有算术都使用整数，因此不存在舍入问题或溢出问题。

# helper: run solution on input string, return output string
import sys
import io

def solve():
    input = sys.stdin.readline

    a = int(input())
    b = int(input())
    c = int(input())

    d1 = b - a
    d2 = c - b

    if d1 == d2:
        print(c + d1)
        print(4)
    elif d1 > d2:
        print(a + d2)
        print(2)
    else:
        print(b + d1)
        print(3)

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout
    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()
    solve()
    out = sys.stdout.getvalue()
    sys.stdin = backup_stdin
    sys.stdout = backup_stdout
    return out

# provided sample

# custom cases

 ## 边缘情况

 例如，当缺失值位于所有现有数字之前时```
4
6
8
```该算法计算`d1 = d2 = 2`。 由于差异相等，因此附加`10`在最后。 这与原来的进展不同`2, 4, 6, 8`，但它仍然是一个有效的算术级数，满足问题要求。 

当缺失值位于第一个和第二个数字之间时，```
10
16
19
```差异是`6`和`3`。 第一个差异较大，因此算法插入`10 + 3 = 13`在位置`2`。 结果序列是`10, 13, 16, 19`。 

当缺失值位于第二个和第三个数字之间时，```
10
13
19
```差异是`3`和`6`。 第二个差异较大，因此算法插入`13 + 3 = 16`在位置`3`。 重建的进展是`10, 13, 16, 19`。 

当每个值都相同时，```
5
5
5
```两者差异均为零。 该算法输出另一个`5`最后，产生`5, 5, 5, 5`，这是一个具有公差的算术级数`0`。
