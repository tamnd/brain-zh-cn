 ## 解决方案
 ## 问题理解

 我们有一条街道，房屋编号为 1 到 n。 一个角色沿着街道行走两次，并根据可分性规则将海报贴在房屋上。 在第一遍中，选择索引可被 a 整除的所有房屋。 在第二遍中，会考虑可被 b 整除的每个房屋，但仅当该房屋在第一次遍历中尚未收到海报时才会添加海报。 

任务是计算在两次通过后有多少不同的房屋收到至少一张海报。 

这相当于计算 [1, n] 范围内可被 a 整除或可被 b 整除的整数，并且重复项仅计算一次。 

约束允许 n、a、b 最多为 10^9。 这立即排除了对所有房屋的迭代，因为在典型限制下不可能进行高达 10^9 的线性扫描。 该解决方案必须依赖于算术计数而不是模拟。 

一个幼稚的错误是模拟两次传球的标记房屋。 例如，如果 n = 10、a = 2、b = 3，我们将显式标记 2、4、6、8、10，然后是 3、6、9。这适用于较小的 n，但当 n 很大时完全失败。 

另一个微妙的问题是重复计算。 能被 a 和 b 整除的房子不能被计算两次。 例如，如果 n = 12、a = 3、b = 4，则房屋 12 可被两者整除。 任何正确的解决方案都必须减去此类重叠。 

## 方法

 蛮力方法很简单。 我们迭代从 1 到 n 的每个房子，并检查它是否能被 a 或 b 整除。 如果是这样，我们就增加答案。 这是正确的，因为它直接反映了所描述的过程。 然而，它的成本是 O(n)，当 n 达到 10^9 时就变得不可行，因为这需要数十亿次检查。 

关键的观察是整除性形成了规则的算术模式。 [1, n] 中能被 a 整除的整数个数正好是floor(n / a)。 类似地，能被b整除的数是floor(n / b)。 重叠部分，即能被 a 和 b 整除的数字，就是能被 lcm(a, b) 整除的数字，因此我们减去一次 Floor(n / lcm(a, b)) 以避免重复计算。 这将问题简化为常数时间算术。 

该变换之所以有效，是因为原始过程只是两个周期集的并集。 我们不是枚举元素，而是使用周期长度来计算它们。

 | --- | --- | --- | --- |
 | 蛮力 | O(n) | O(1) | O(1) | 太慢了|
 | 包容-排除 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们计算 [1, n] 中有多少个数字可以被 a 整除，有多少个数字可以被 b 整除，然后减去那些可以被两者整除的数字。 

1. 计算 ca = n // a，计算范围内 a 的倍数。 这是有效的，因为每一个数字都贡献了一个有效的房子。 
2. 以同样的理由计算 cb = n // b，计算 b 的倍数。 
3. 使用 gcd(a, b) 计算 lcm(a, b)。 重叠恰好发生在最小公倍数的倍数处，因为这些数字可以同时被 a 和 b 整除。 
4. 计算 cab = n // lcm(a, b)。 这准确地计算了前两次计算中两次包含的房屋。 
5.最终答案是ca+cb-cab。 

减法是必要的，因为每个重叠的房子在 ca 中计算一次，在 cb 中计算一次，但只应贡献一张海报。 

### 为什么它有效

 [1, n] 中的每个整数都属于三个不相交类别之一：只能被 a 整除，只能被 b 整除，或者可以被两者整除。 前两类单独计算，第三类则两次计算。 减去第三类的大小即可恢复正确的计数。 不存在其他重叠，因为整除性完全由 lcm 的倍数表征。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def lcm(x, y):
    return x // gcd(x, y) * y

n = int(input())
a = int(input())
b = int(input())

ca = n // a
cb = n // b
cab = n // lcm(a, b)

print(ca + cb - cab)
```该解决方案依赖于整数除法属性：n // k 计算 [1, n] 中有多少个 k 的倍数。 lcm 计算使用 gcd 来避免溢出问题，并确保即使 a 和 b 很大时也能保证正确性。 

一个常见的实现陷阱是直接将 lcm 计算为 a * b，这可能会溢出或在中间算术中变得不正确。 使用 // gcd(a, b) * b 可以避免这种情况。 

另一个微妙之处是确保整数除法的使用一致； 浮点除法会破坏正确性。

这证实了该公式自然可以处理两个整除集都为空的情况。

 ## 边缘情况

 当 a 和 b 相等时，会出现一种边缘情况。 例如，n = 20，a = 4，b = 4。正确的结果只是楼层（20 / 4），因为两次传递都选择相同的房屋。 该公式可以处理此问题，因为 lcm(a, b) 等于 a，因此减法仅删除一次重复计数。 

另一种情况是当一个除数超过 n 时。 对于 n = 5、a = 10、b = 3，第一遍贡献为零，第二遍表现正常。 lcm 为 30，因此没有减去重叠。 该算法正确地减少为仅计算有效倍数。 

最后一种情况是 a 和 b 都超过 n。 那么所有的楼层划分都为零，并且结果正确地为零，无需特殊处理。
