 ## 解决方案
 ## 问题理解

 我们得到一个写在黑板上的大整数。 应用一系列操作直到数字变为零。 每个操作完全取决于号码中当前存在的数字。 

如果十进制表示至少包含一位奇数，则该数字减一。 如果所有数字都是偶数，则该数字除以二。 我们被要求计算需要多少次操作才能达到零。 

关键的困难来自这样的事实：该操作不是由数字本身的奇偶性决定的，而是由其十进制形式中任何位置的任何奇数位的存在决定的。 这使得该过程对减法期间的进位和除法期间的数字级结构敏感。 

输入大小高达 10^18，因此任何在循环中天真地模拟逐位操作而无需仔细控制的方法仅当步数受值本身限制时仍然可以。 当发生多次连续递减时，最坏情况下的操作数与 n 呈线性关系。 

当数字在减法过程中由于进位而在有奇数位和没有奇数位之间交替时，会出现微妙的边缘情况。 例如，像 20 这样的数字没有奇数位，可以被除，但 21 会强制减法。 减法之后，我们可以再次以非局部方式创建或销毁奇数位。 

另一个重要的极端情况是当数字已经仅由偶数位组成但在除法后再次变得不含奇数位时。 例如，40 → 20 → 10 → 5 → 4 → 2 → 1 → 0 演示了数字奇偶性和数字奇偶性不对齐的交替行为。 

## 方法

 直接模拟是最直接的想法。 我们反复检查数字的位数，检查是否有奇数位，并应用相应的操作。 每次检查都需要 O(log n) 位扫描，并且每次操作都会将数字减少一或二倍。 

这是正确的，因为它完全遵循问题定义。 当数字很大并且有许多连续状态且奇数位持续存在时，问题是性能。 在最坏的情况下，重复减法占主导地位，导致多达 10^18 步，这是不可能直接模拟的。 

关键的观察是，除以二在减少幅度方面非常便宜，而减法仅在存在奇数位时才持续存在。 我们没有将其视为纯粹的模拟，而是将过程作为数字驱动的状态机进行跟踪。 重要的见解是，操作的数量取决于我们在安全地执行除法之前被迫进入减法阶段的频率，并且可以通过重复更新数字和检查数字奇偶性来有效地模拟这种结构。 

我们不需要在模拟之外以数学方式加速转换，因为每一步都会将数量减少 1 或至少一半。 这保证了除法阶段的对数数量和每个数字转换的减法突发的有限数量。

 ## 算法演练

 ### 1. 初始化一个操作计数器并读取数字n。 

我们维护一个简单的整数计数器，因为我们只需要计算在达到零之前发生了多少次转换。 

### 2. 重复直到 n 变为零。 

该过程本质上是迭代的，并且每一步都严格减少n，因此保证终止。 

### 3. 检查n 的数字并确定是否有数字为奇数。 

我们将 n 转换为数字并扫描它们。 该检查确定哪个操作在此状态下有效。

### 4. 如果任何数字是奇数，则将 n 减 1。 

这是规则直接规定的。 减法可能会显着改变数字结构，因此我们必须在下一次迭代中重新评估。 

### 5. 否则使用整数除法将 n 除以二。 

当所有数字都是偶数时，除法是安全的，并且会立即显着降低幅度。 

### 6. 每次转换后递增操作计数器。 

每个有效的转换都算作一个“男孩叫”，因此我们统一累积每一步。 

### 为什么它有效

 在每次迭代中，算法都准确地反映问题中描述的过程所应用的规则。 系统的状态完全由当前整数n捕获，并且数字检查条件是唯一的分支规则。 由于每次操作都严格减少n，因此该过程无法循环。 正确性来自于没有任何操作被跳过或近似的事实，并且数字检查确保在每个状态下选择正确的操作。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def has_odd_digit(x: int) -> bool:
    while x > 0:
        if (x % 10) % 2 == 1:
            return True
        x //= 10
    return False

def solve():
    n = int(input().strip())
    steps = 0

    while n > 0:
        if has_odd_digit(n):
            n -= 1
        else:
            n //= 2
        steps += 1

    print(steps)

if __name__ == "__main__":
    solve()
```该实现将数字检查分离到辅助函数中，这保持了主循环的可读性并确保逻辑与问题陈述完全匹配。 循环继续，直到 n 达到零，应用减法或除法。 

最重要的微妙之处是数字扫描。 它必须在每次迭代中检查每个数字，因为减法可能会以不可预测的方式引入或删除奇数数字。

 ## 边缘情况

 最小输入（例如 n = 1）会立即触发减法规则，因为该数字是奇数。 该算法执行减一并终止，生成 1，与预期行为匹配。 

对于像 n = 8 这样的数字，所有数字都是偶数，因此第一个操作是除法：8 → 4 → 2 → 1 → 0。每一步都会重新评估数字检查，从偶数位状态的转换可确保除法的一致性，直到出现最终的奇数位状态。
