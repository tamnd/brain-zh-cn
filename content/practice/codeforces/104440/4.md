 ## 问题理解

 我们得到了一系列数字，代表每个参与者在游戏中宣布的内容。 目标是识别在整个序列中只出现一次的数字。 在这些唯一出现的数字中，我们选择最小值并将其输出为获胜者。 如果没有数字恰好出现一次，则没有有效的获胜者，我们输出 -1。 

从根本上来说，这是一个针对最多 100000 个整数的数组的频率分析问题，每个整数可能大到 10^9。 关键操作是有效地计算出现次数，然后在过滤的子集中提取最小值。 

n 最大为 10^5 的限制意味着任何具有二次行为的解决方案（例如比较每对元素或重复扫描阵列的频率）都会太慢。 需要线性或线性算术的解决方案，并且由于在这种规模下排序是可以接受的，因此基于散列的计数和基于排序的聚合都是可行的。 

当所有数字至少重复两次时，就会出现微妙的边缘情况。 例如，如果输入是：```
2
3
9
3
9
9
2
```每个值都会出现多次，因此没有有效的候选值，正确的输出为 -1。 这里的一个天真的错误是在取最小值之前忘记按频率进行过滤，这会错误地返回原始数组中的最小元素，而不是唯一出现的元素。 

另一种边缘情况是只有一个数字恰好出现一次并且全局不是最小的，例如：```
5
10
10
10
3
3
```这里 3 只出现一次，所以即使其他值按一般顺序更小也必须返回。 

## 方法

 最简单的方法是计算每个数字出现的次数。 人们可以扫描每个元素，并且对于每个元素，再次扫描整个数组以计算出现次数。 这会产生 O(n^2) 解决方案，在最坏的情况下，当 n 为 10^5 时，这意味着大约 10^10 次操作，这远远超出了可行的限制。 

更有效的策略是将问题分为两个阶段。 首先，使用哈希图或字典计算所有数字的频率。 这将计数减少到 O(n) 预期时间。 其次，迭代所有不同的数字，并仅考虑那些频率恰好为 1 的数字，并保持其中的最小值。 

关键的结构观察是我们不关心输入中的位置或顺序，只关心多重性。 一旦频率已知，问题就简化为过滤一组并取最小值，这与不同值的数量呈线性关系。 

基于排序的替代方案也有效。 如果我们对数组进行排序，相等的值就会变得连续，从而允许我们在 O(n log n) 时间内对运行进行计数。 然后，我们扫描一次排序数组以提取游程长度为 1 的值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 哈希图计数 | O(n) 平均 | O(n) | 已接受 |
 | 排序+扫描 | O(n log n) | O(n log n) | O(1) 或 O(n) | 已接受 |

 ## 算法演练

 我们使用频率图方法。 

1. 将所有数字读入数组，同时构建频率字典。 每次我们读取一个数字时，我们都会增加它的计数。 这确保我们只遍历一次输入来进行计数。 
2. 初始化变量`best`代表尚未找到候选者的哨兵值。 
3. 迭代频率字典中的所有键值对。 对于每个数字，检查其频率是否恰好等于 1。 此条件确保该号码符合游戏规则。 
4. 如果该数字符合条件，并且尚未选择候选者或该数字小于当前候选者，则更新`best`到这个数字。 这保持了迄今为止看到的最小有效值。 
5. 处理完所有按键后，检查是否`best`已更新。 如果不是，输出-1，否则输出`best`。 

### 为什么它有效

 频率字典完全捕获了与资格相关的唯一属性，即每个数字出现的次数。 由于选择规则仅取决于过滤值中的频率和全局最小值，因此对输入的任何重新排序或部分处理都不会影响正确性。 在所有有效键上保持运行最小值可以保证一旦循环结束，存储的值恰好是频率为 1 的最小数字（如果存在这样的数字）。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    freq = {}

    for _ in range(n):
        x = int(input())
        freq[x] = freq.get(x, 0) + 1

    best = None

    for x, c in freq.items():
        if c == 1:
            if best is None or x < best:
                best = x

    print(-1 if best is None else best)

if __name__ == "__main__":
    solve()
```该解决方案首先在输入上单次构建频率图。 该字典确保每个元素的恒定时间平均更新。 之后，它仅扫描不同的值，而不是再次扫描完整数组，这避免了冗余工作。 

变量`best`初始化为`None`清楚地区分“未找到有效候选者”和有效整数值。 这避免了用大哨兵初始化可能意外与输入边界交互的陷阱。 

最终的条件输出处理没有元素满足唯一性条件的情况。

 ## 边缘情况

 一个关键的边缘情况是所有数字至少重复两次。 对于输入如：```
4
7
7
8
8
```频率图变为{7:2, 8:2}。 在迭代期间，没有条目满足频率等于 1，因此`best`保持 None 且输出为 -1。 该算法正确地避免选择 7，即使它是输入中最小的数字。 

另一种边缘情况是只有一个数字是唯一的并且它不是全局最小的：```
5
10
1
10
10
1
```这里的频率是{10:3, 1:2}。 仍然没有有效的候选者，因此输出为-1。 这证实了“只出现一次”是严格的，并且部分唯一性不符合条件。 

最后一种情况是存在多个唯一数字但分散的情况：```
6
4
2
2
7
7
1
```频率为 {4:1、2:2、7:2、1:1}。 有效候选数为 4 和 1，并且无论输入顺序如何，算法都会正确跟踪最小值 1。
