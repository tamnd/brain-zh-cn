 ## 解决方案
 ## 问题理解

 我们维护一个数组，其中每个元素都是一个小的正整数，并且我们只对其素因子结构感兴趣。 该数组通过三个操作进行修改：我们可以从单个位置删除质因数，我们可以用常量值覆盖整个范围，并且我们可以查询一个范围，要求在以最佳方式考虑所有先前的删除后，询问质因数的最小可能总和。 

关键的微妙之处在于，值运算在实践中不依赖于任何随机性。 尽管该声明提到删除“随机”素因子，但查询要求所有结果的最小可能总和，这意味着每当我们评估查询时，我们都可以假设每个位置的对抗性最佳删除。 

输入大小迫使解决方案支持多达十万次更新和查询。 任何每次操作重新计算因式分解或重新计算范围的方法都会太慢。 即使每个查询进行线性扫描，在最坏的情况下也会导致大约 10^10 次操作，这是不可行的。 

每个值最多为 10^4，这是一个强烈的提示：我们可以预先计算所有素因数分解和派生量。 这一界限还表明，每个值的信息可以以紧凑的形式存储，可能作为一小组预先计算的成本，而不是重复重新计算因子。 

一个天真的但重要的误解是将“删除素因子”操作视为确定性地将总和减少固定量。 例如，对于 12 = 2 × 2 × 3，删除随机因子可能会导致 6 或 4，并且所得的最小和取决于首先删除哪个因子。 查询必须考虑跨多个操作的此类删除的所有顺序。 

另一个微妙的问题是覆盖范围。 如果我们没有正确构建数据，我们将需要为范围内的每个元素重建因子信息，这又太慢了。 

## 方法

 蛮力方法在概念上很简单。 我们维护每个元素的准确值。 当类型 1 发生时，我们对索引 i 处的数字进行因式分解，任意删除一个素因数，并更新存储的整数。 当类型 3 发生时，我们覆盖一个范围。 当类型 2 发生时，我们对查询范围内的每个数字进行因式分解，并对所有质因数求和。 

这是正确的，因为它确实模拟了该过程。 然而，瓶颈立刻就清晰了。 类型 2 查询最多可以扫描 10^5 个元素，每次分解的成本最多为 √a_i，大约 100 次操作。 对于 10^5 次查询，这变得太大了。 

关键的观察是我们在查询期间永远不需要对数字进行完全因式分解。 我们只需要最优去除后的质因数之和。 由于每次删除恰好删除一个素数因子，因此每个元素的贡献以一种非常结构化的方式演变：每个元素始终可表示为素数的多重集，并且删除一次只会将多重集大小减少一个。 

这使我们能够将每个数字减少到一个小的动态状态。 我们不是跟踪完整的整数，而是跟踪它有多少“剩余贡献”以及它们的总数是多少。 由于值很小，我们为每个 x 预先计算 10^4 的质因数总和（具有重数）。 每次删除都会将该总和减少该数字分解中的最小可用素因数。 这一观察是关键：为了最小化未来的总和，我们总是首先删除最小的素数因子，因为删除较大的素数之后会留下较小的总和。

因此，对于每个数字，我们维护一个类似多重集的结构，但我们只需要两条信息：其当前剩余素因子之和及其最小可用素因子。 线段树可以维护范围总和，同时支持点更新和范围分配。 删除操作成为一个索引处的本地更新，用相同的值除以其最小素因子来替换当前值。 

线段树为每个线段存储当前“最佳删除后的素因子和”的总和。 通过重置存储的值并根据基数重新计算预先计算的状态，通过延迟传播来处理范围分配。

 | --- | --- | --- | --- |
 | 蛮力 | O(nq√A) | O(n) | 太慢了 |
 | 预计算的最优线段树 | O((n + q) log n) | O((n + q) log n) | O(n + A) | 已接受 |

 ## 算法演练

 我们首先预先计算 10^4 以内所有数字的质因数信息。 对于每个值 x，我们计算其具有重数的质因数列表、其总和以及通过重复删除最小质因数获得的状态序列。 每个状态代表 k 次删除后可能的最佳配置。 

然后，我们在数组上构建一棵线段树，其中每个节点存储其线段在最佳删除下的当前贡献。 

1. 预先计算最多 10^4 的所有素因子分解，并为每个数字 x 构建一个状态列表，其中每个状态是重复删除最小因子后剩余素因子的总和。 

这确保了我们可以在 O(1) 时间内回答“k 次删除后会发生什么”。 
2. 对每个数组元素使用状态 0 来初始化线段树，这意味着尚未应用任何删除操作。 

每片叶子存储其值的质因子的全部总和。 
3. 对于索引 i 处的类型 1 操作，我们将该元素的移除计数增加 1，并将其移动到下一个预计算状态。 

这是可行的，因为删除一个素因数总是对应于在预计算链中前进一个状态。 
4. 对于将值 x 分配给范围的类型 3 操作，我们将所有受影响的位置重置为 x 的状态 0。 

这会丢弃所有以前的因子历史记录，因为覆盖会替换整个因子结构。 
5. 对于 [l, r] 上的类型 2 查询，我们查询当前状态的线段树和。 

存储的值已经反映了最佳去除行为，因此不需要重新计算。 

### 为什么它有效

 正确性基于以下事实：每个数字通过连续删除最小可用素因子定义的确定性状态链独立演化。 任何旨在最小化剩余总和的删除序列必须始终在每一步删除最小的可用素因子，因为删除较大的因子将留下严格更大的剩余和。 这使得状态转换变得贪婪和单调，允许每个值表示为预先计算的归约序列的前缀。 然后线段树线性聚合这些独立的贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXV = 10000

spf = list(range(MAXV + 1))
for i in range(2, MAXV + 1):
    if spf[i] == i:
        for j in range(i * i, MAXV + 1, i):
            if spf[j] == j:
                spf[j] = i

def factor_sum(x):
    s = 0
    while x > 1:
        p = spf[x]
        s += p
        x //= p
    return s

# precompute full reduction chains
states = [[] for _ in range(MAXV + 1)]
max_steps = [0] * (MAXV + 1)

for x in range(1, MAXV + 1):
    cur = x
    seq = [factor_sum(cur)]
    while cur > 1:
        p = spf[cur]
        cur //= p
        seq.append(factor_sum(cur))
    states[x] = seq
    max_steps[x] = len(seq) - 1

class SegTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.seg = [0] * (2 * self.size)
        self.base = arr[:]  # original values
        self.step = [0] * self.n

        for i in range(self.n):
            self.seg[self.size + i] = states[arr[i]][0]
        for i in range(self.size - 1, 0, -1):
            self.seg[i] = self.seg[2 * i] + self.seg[2 * i + 1]

    def apply(self, idx):
        x = self.base[idx]
        self.step[idx] += 1
        if self.step[idx] < len(states[x]):
            val = states[x][self.step[idx]]
        else:
            val = 0
        return val

    def update_point(self, idx):
        self.seg[self.size + idx] = self.apply(idx)
        i = (self.size + idx) // 2
        while i:
            self.seg[i] = self.seg[2 * i] + self.seg[2 * i + 1]
            i //= 2

    def query(self, l, r):
        l += self.size
        r += self.size
        s = 0
        while l <= r:
            if l % 2 == 1:
                s += self.seg[l]
                l += 1
            if r % 2 == 0:
                s += self.seg[r]
                r -= 1
            l //= 2
            r //= 2
        return s

n = int(input())
a = list(map(int, input().split()))
q = int(input())

st = SegTree(a)

for _ in range(q):
    tmp = list(map(int, input().split()))
    if tmp[0] == 1:
        i = tmp[1] - 1
        st.update_point(i)
    elif tmp[0] == 2:
        l, r = tmp[1] - 1, tmp[2] - 1
        print(st.query(l, r))
    else:
        l, r, x = tmp[1] - 1, tmp[2] - 1, tmp[3]
        for i in range(l, r + 1):
            st.base[i] = x
            st.step[i] = 0
            st.seg[st.size + i] = states[x][0]
        # rebuild upwards
        for i in range(st.size - 1, 0, -1):
            st.seg[i] = st.seg[2 * i] + st.seg[2 * i + 1]
```该实现依赖于为最多 10^4 的每个值预先计算完全归约序列。 线段树的每个叶子都存储该元素在一定次数的删除后当前可能的最佳贡献。 类型 1 的更新操作只是前进该索引的状态指针并刷新其叶子。 

范围分配会重置基值和删除计数器，因为覆盖会破坏以前的因子历史记录。 然后从下到上重新计算线段树以确保正确性。 

主要的微妙之处在于每个元素的演化都是独立的，因此我们永远不需要协调跨索引的删除。

 ## 边缘情况

 关键的边缘情况是对变成 1 的数字进行重复删除。一旦数字达到 1，它就没有质因数，因此进一步的类型 1 操作不应改变它。 状态链自然会处理这个问题，因为预先计算的序列以零贡献结束。 

另一种情况是多次删除后覆盖范围。 正确的行为是丢弃所有先前的状态历史记录。 这是通过重置基值和步计数器来处理的，确保旧的因子结构不会泄漏到新的段值中。
