 ## 解决方案
 ## 问题理解

 我们得到一个整数列表，对于每个整数，我们重复应用一个转换：将数字替换为其真因数之和，这意味着所有除数都严格小于数字本身。 这种转换可以多次应用，但这里的任务不是模拟长链。 相反，对于每个数字，我们只需要查看该过程的第一步，即真除数之和一次。 

根据该单个计算值，每个数字被分为四个类别之一。 如果真因数之和等于该数，则该数称为完美数。 如果对该和应用相同的操作返回原始数字，则这两个数字形成友好的对，这里称为浪漫。 如果总和大于数量，那就是丰富的。 否则，它被归类为复杂。 

输入大小最多可达 100,000 个数字，每个数字最多 100,000 个值。 这立即排除了为每个查询通过试除法重新计算除数的情况。 为每个查询检查所有数字直到平方根的简单方法将花费大约 O(n√a)，在最坏的情况下变成大约 10^10 次操作，远远超出 1 秒的限制。 

关键的结构约束是所有查询都是独立的，但共享相同的除数和计算。 这意味着我们可以一次预先计算 100,000 以内所有值的真因数之和，然后在恒定时间内回答每个查询。 

边缘案例大多来自少数。 对于 n = 1，真因数之和为 0，这使得它既不完美也不丰富，也不属于浪漫对的一部分，除非明确检查。 幼稚的实现可能会忘记正确处理 1，并错误地将其分类为完美或不分类。 

另一个微妙的例子是浪漫的数字。 仅检查 sum(sum(a)) 是否等于 a 是不够的。 我们必须确保 sum(a) 不等于 a 本身，否则如果被误解，完美数可能会被错误地认为是浪漫的。 

## 方法

 直接方法独立计算每个数字的真因数之和。 对于数字 x，我们从 1 迭代到 √x 并成对添加除数。 这是正确的，因为每个大于 √x 的除数都与小于 √x 的除数配对。 然而，在最好的情况下，对最多 100,000 个查询执行此操作会导致大约 100,000 × 316 ≈ 3000 万次除数检查，并且 Python 中的常量明显更差，尤其是在重复模数和除法运算的情况下。 

低效率来自于重复地重新计算相同的除数信息。 每个数字都有助于多个查询，因此该结构建议进行预计算。 我们可以反转角度，而不是独立地分解每个数字：对于每个潜在的除数 d，我们将其添加到 d 的所有倍数中。 这是经典的筛式积累。 对于每个 i，i 的每个倍数 j 都会收到 i 作为真除数贡献，除非 j 等于 i 本身。 

这将问题转化为一系列谐波循环，类似于埃拉托斯特尼筛法，产生大约 N log N 的预处理操作总数。 一旦我们获得了 100,000 以内所有数字的真因数总和，每个查询的分类时间就变成了 O(1)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n√a) | O(n√a) | O(1) | O(1) | 太慢了 |
 | 筛选预计算 | O(N log N + n) | O(N log N + n) | O(N) | 已接受 |

 ## 算法演练

 我们预先计算一个表`sigma[x]`存储 x 的真因数之和。 

1.初始化一个数组`sigma`大小为 N+1，带零。 该数组将累积每个数字的除数贡献。 
2. 迭代从 1 到 N 的每个整数 i。对于每个 i，我们将 i 添加到所有倍数 j = 2i、3i、4i 等，直到 N。我们从 2i 开始，因为 i 不应该作为其自身的真约数。 
3. 在这个循环之后，`sigma[x]`包含 x 的所有真因数之和。 
4. 对于每个查询值 a，计算 b = sigma[a]。 
5. 如果 b == a，则将该数字标记为完美。 
6、如果b != a且sigma[b] == a，那么a和b在divisor-sum函数下形成二循环，所以这个数字很浪漫。 
7. 如果 b > a 并且尚未将其归类为完美或浪漫，则将其标记为丰富。 
8. 否则，将其归类为复杂。 
9. 输出数字及其分类标签。 

浪漫检查起作用的原因是我们已经预先计算了所有值的西格玛（直到极限）。 因此检查 sigma[b] 是常数时间。 

### 为什么它有效

 筛子保证 x 的每个真除数 d 都被精确地添加到 sigma[x] 一次，因为我们迭代所有 d 并将其分配给它的倍数。 这确保了 sigma[x] 恰好是除 x 本身之外的所有整数之和。 

分类逻辑仅取决于 sigma[x]、x 和 sigma[sigma[x]] 之间的比较。 由于 sigma 对于范围内的所有值都是精确的，因此浪漫条件可以正确检测二元循环，并且不会将它们与完美数或更大的循环混淆。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAX = 100000

sigma = [0] * (MAX + 1)

for i in range(1, MAX + 1):
    for j in range(i * 2, MAX + 1, i):
        sigma[j] += i

def classify(x):
    s = sigma[x]
    if s == x:
        return "perfecto"
    if s <= MAX and sigma[s] == x and s != x:
        return "romantico"
    if s > x:
        return "abundante"
    return "complicado"

n = int(input())
out = []
for _ in range(n):
    a = int(input())
    out.append(f"{a} {classify(a)}")

print("\n".join(out))
```实现的核心是类似筛子的构造`sigma`。 内循环开始于`2*i`因为数字不被视为其自身的真约数。 这可以防止意外包含自我贡献的常见错误。 

分类函数以正确的顺序评估条件。 必须首先检查完美数字，因为如果只宽松地检查相互平等，它们就会被误认为是浪漫的。 浪漫需要严格的二周期条件，所以我们保证`s != x`。 仅在排除这些情况后才检查丰富性，因为完全数在技术上满足`s >= x`但不得贴上“丰富”的标签。 

## 工作示例

 ### 示例 1

 输入数字：6、12

 我们使用西格玛值：

 西格玛[6] = 1 + 2 + 3 = 6

 西格玛[12] = 1 + 2 + 3 + 4 + 6 = 16

 | x| 西格玛[x] | 检查 sigma[sigma[x]] | 分类 |
 | --- | --- | --- | --- |
 | 6 | 6 | 不需要| 完美|
 | 12 | 12 16 | 16 西格玛[16] = 15 | 丰富|

 x = 6 的情况证实了 x 与其除数和之间的相等直接触发了完美分类。 x = 12 的情况显示出严格较大的除数和，并且不存在倒数关系，从而证实了丰度。 

### 示例 2

 输入号码：220、284

 我们使用已知值：

 西格玛[220] = 284

 西格玛[284] = 220

 | x| 西格玛[x] | 西格玛[西格玛[x]] | 分类 |
 | --- | --- | --- | --- |
 | 220 | 220 284 | 284 西格玛[284] = 220 | 浪漫|

 该迹线显示了相互映射属性。 每个数字在 sigma 下映射到另一个数字，形成固定的两个循环。 这证实了为什么使用对称检查来检测浪漫数字。

 ## 边缘情况

 对于 x = 1，sigma[1] 为 0，因为它没有真因数。 分类函数发现 s = 0，小于 x，并检查 sigma[0] 是否相关。 最终的结果很复杂。 这证实了该算法正确处理了数字没有约数的退化情况。 

对于像 6 这样的完美数，sigma[6] 计算为 2 + 3 + 1 = 6。条件 s == x 立即触发，因此在任何其他检查之前它被归类为完美数。 这可以防止错误地将其分类为丰富或浪漫。 

对于像 220 和 284 这样的友好对，两个值都在范围内，并且它们的 sigma 值指向彼此。 检查 sigma[sigma[x]] == x 对于两者来说都是对称成功的，确保两者都被标记为浪漫，而不需要任何显式的配对跟踪。 

对于像 12 这样的丰富数字，sigma[12] = 16，并且 sigma[16] = 15，不会返回到 12。由于 16 > 12，因此被归类为丰富数字。 没有两个周期可以防止它被错误地贴上浪漫的标签。
