 ## 解决方案
 ## 问题理解

 60 张卡片中的每张都对应于正整数上的固定模式。 当一个数字满足从该卡的索引导出的特定二进制条件时，该数字就会出现在卡上。 具体来说，如果我们看一下示例，卡 1 包含所有奇数，卡 2 包含第二个二进制位被设置的数字，卡 4 对应于第三位，依此类推。 该结构与每张卡代表数字的二进制表示中的单个位位置一致。 

在每个查询中，我们都会被告知卡片索引的子集。 该子集准确地代表了出现隐藏数字的卡片组。 我们的任务是重建那个隐藏的数字。 由于每张卡片对应一个位位置，因此子集直接告诉我们在数字中设置了哪些位。 

相关维度的约束非常小。 最多有 1000 个查询，每个查询最多 60 张卡片。 任何以卡片数量的线性时间处理每个查询的解决方案都足够快。 隐藏值可以大到 10^18，这意味着最多 60 位就足够了，与卡片的数量完全匹配。 这是一个强烈的暗示，表明表示是二进制的和完整的。 

天真的解释的一个微妙的失败案例是将卡片视为任意集合并尝试通过显式列表进行交叉或搜索。 例如，如果我们尝试通过扫描生成的序列来模拟成员资格，我们很快就会超出限制，因为序列是无限的。 另一个错误的假设是认为映射不是一对一的，这会导致歧义。 该结构实际​​上保证了唯一性，因为每个数字都有唯一的二进制表示形式，因此卡的组合也是唯一的。 

## 方法

 蛮力的想法是将每张卡从字面上解释为无限列表，并尝试通过测试候选者来确定隐藏的数字。 人们可以想象迭代 10^18 以内的所有数字，并检查它们是否恰好出现在给定的一组卡片中。 这在逻辑上是正确的，因为成员资格是明确定义的，但完全不可行。 即使检查单个数字也需要验证最多 60 个条件，而迭代整个范围将需要 10^18 次操作。 

关键的观察是卡系统以二进制形式对数字进行编码。 每张卡对应于 2 的幂，如果在其二进制表示中设置了相应的位，则卡上会出现一个数字。 这将问题从无限序列上的集合隶属度问题转换为从其二进制掩码重建整数。 

一旦认识到这一点，每个查询就变成一个简单的位重建任务。 如果卡 i 存在于输入子集中，我们将答案初始化为零并设置第 i 位。 这是有效的，因为每张卡对最终数字的贡献都是独立且唯一的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解数字 | O(10^18·60) | O(10^18·60) | O(1) | O(1) | 太慢了 |

 ## 算法演练

 我们依赖于这样一个事实：卡 i 对应于数字的 2^(i-1) 位。 

1. 将当前查询的结果初始化为零。 该值将逐渐累积每张所选卡牌的贡献值。 
2. 读取查询中提供的卡片索引列表。 每个索引代表最终数字中必须打开的一个位位置。 
3. 对于每个卡片索引 a，计算 1 左移 (a - 1) 对应的 2 的幂。 使用按位或或加法将该值添加到结果中。 
4. 处理完所有索引后，输出结果数。 该值是唯一的整数，其二进制表示形式与所选卡完全匹配。

正确性来自于每个数字都有唯一的二进制分解这一事实。 这些卡片实际上是该分解的分布式编码，因此选择卡片相当于选择位。 不同位之间不存在交互，因此将它们组合起来是安全且无损的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q = int(input())
    for _ in range(q):
        data = list(map(int, input().split()))
        k = data[0]
        cards = data[1:]
        res = 0
        for c in cards:
            res |= (1 << (c - 1))
        print(res)

if __name__ == "__main__":
    solve()
```该实现直接反映了位解释。 每个卡片索引都使用左移转换为 2 的幂，这是二进制位置的标准表示。 使用按位或代替加法，尽管加法也是安全的，因为每个位位置都是唯一的并且从不重叠。 

一个常见的错误是轮班时差一位。 卡 1 对应于最低有效位，因此正确的移位是 (c - 1)，而不是 c。 另一个错误是试图模拟无限序列，这是不必要且不可能的。

 ## 边缘情况

 一种边缘情况是仅存在最大的牌，例如牌 60。该算法计算 1 << 59，它在允许的 10^18 范围内并且安全地适合 Python 整数。 由于固定宽度类型的溢出问题，简单的模拟方法可能会失败。 

另一种边缘情况是仅选择卡 1 时。 结果应该正好是 1，这测试了移位的逐一正确性。 如果错误地使用 1 << c 而不是 1 << (c - 1)，则输出将错误地变为 2，这已经违反了最小有效情况。 

最后的边缘情况是在查询中选择所有卡时。 结果变为 (2^60 - 1)，即密集位掩码。 这证实了组合所有独立贡献仍然遵循二进制结构，并且不会导致信息重叠或丢失。
