 ## 解决方案
 ## 问题理解

 我们得到一个编码为字符串的一维路径`+`和`-`。 在处理字符串之前，我们从隐式高度零开始。 每个字符将当前高度精确移动一个单位：`+`高度增加一，`-`将其减一。 处理每个字符后，我们获得沿路径的一系列高度，字符串中的每个位置一个高度。 

任务是找到整个步行过程中高度达到最大值的字符串位置。 位置以 1 为索引，指的是字符串中达到该高度的字符。 

所以问题本质上是：在 ±1 数组上模拟前缀和，并报告最大前缀和出现的最早位置，并保证该最大值是唯一的。 

约束条件`|s| ≤ 10^5`意味着任何解决方案都必须是线性或近线性的。 O(n²) 模拟（例如重新计算前缀和或从每个位置扫描）会太慢，因为在最坏的情况下它会涉及多达 1010 次操作。 

O(n) 内的线性扫描就足够了，因为在 Python 中 1 秒限制下 10⁵ 操作是微不足道的。 

当立即达到最大高度时，会出现微妙的边缘情况。 例如，输入`+----`在位置 1 达到高度 1 并且再也不会超过它，所以答案是 1。另一种情况是最后达到最大值时，例如`++++`，其中答案为 4。仅跟踪最终高度的简单方法在这里会失败，因为峰值不一定位于步行的终点。 

## 方法

 最简单的想法是模拟步行并计算每一步后的高度。 我们维护一个从零开始的运行总和，并根据角色将其更新为 +1 或 -1。 一路上，我们跟踪迄今为止看到的最大值及其首次出现的位置。 

这是可行的，因为每个位置的高度仅取决于该位置之前的字符串前缀。 然而，粗心的实现可能会从头开始重新计算每个索引的高度，这将花费 O(n²) 时间，因为每个位置都会再次扫描所有先前的字符。 

关键的观察是前缀和是增量的。 每个新的高度都可以在恒定的时间内从前一个高度推导出来。 这将计算减少到对字符串的单次传递，从而使其达到最佳效果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（每次重新计算前缀）| O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳（单前缀扫描）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们扫描一次字符串，同时维护三个值：当前高度、迄今为止找到的最佳高度以及最佳高度出现的索引。 

1. 将当前高度初始化为 0，将最佳高度初始化为 0，将最佳位置初始化为 0。我们在概念上包括位置 0，以处理最大值从未改善到超出初始假设的情况。 
2. 使用从 1 开始的索引 i 从左到右迭代字符串。 
3.更新当前高度：如果角色是则增加1`+`，否则减 1。 
4. 将当前高度与最佳高度进行比较。 如果当前高度严格大于，则更新最佳高度并记录位置 i。 
5. 继续直到字符串末尾。 
6. 输出存储的最佳位置。 

每个步骤都依赖于高度累积构建的事实，因此处理后不需要重新考虑未来或过去的部分。 

### 为什么它有效

 高度的顺序完全由前缀和决定，每个前缀和仅取决于前一个。 在每一步 i，算法都会存储直到 i 的所有前缀和中的最大值。 因为每个前缀和都只计算一次并立即比较，所以不会错过更高的值。 唯一性保证确保我们永远不需要处理联系或多个候选人，因此简单的严格比较就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    
    cur = 0
    best = 0
    best_pos = 0
    
    for i, ch in enumerate(s, 1):
        if ch == '+':
            cur += 1
        else:
            cur -= 1
        
        if cur > best:
            best = cur
            best_pos = i
    
    print(best_pos)

if __name__ == "__main__":
    solve()
```代码直接实现了前缀和扫描。 变量`cur`跟踪当前高度，每个角色以恒定时间更新。 变量`best`存储迄今为止看到的最高高度，并且`best_pos`记得事情发生的地点。 

该循环使用基于 1 的索引`enumerate(s, 1)`匹配问题的位置定义。 严格比较`cur > best`确保我们只记录第一次出现的最大高度，这与唯一性保证一致。

 ## 边缘情况

 一个重要的边缘情况是在第一步立即达到最大高度。 用于输入`+----`，前缀高度是`[1, 0, -1, -2, -3]`，所以答案一定是 1。该算法处理这个问题是因为`best`从 0 开始并在 i = 1 时更新`cur = 1`。 

另一种边缘情况是启动后高度从未超过零。 用于输入`-----`，最大前缀高度在位置 0 处为 0（在任何移动之前）。 该实现自然返回 0，因为`best_pos`初始化为 0 并且从未更新。 

第三种情况是最大值出现多次但只应返回第一次出现的情况。 为了`++-++`，高度为`[1, 2, 1, 2, 3]`，并且最大值 3 在位置 5 处仅出现一次。如果存在多个相等的最大值，则严格`>`比较可确保保留第一次出现的情况。
