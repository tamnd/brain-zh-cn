 ## 解决方案
 ## 问题理解

 我们得到了一组成分，并且已知某些成分的味道相同。 这种关系不仅是成对的，而且是传递性的，所以如果成分 A 与 B 匹配，B 与 C 匹配，那么 A、B 和 C 都属于同一风味组。 

制作披萨的过程被描述为对所有原料进行随机排序并从左到右扫描它们。 当我们看到一种成分时，我们会尝试添加它。 根据等效信息，仅当它引入了已选择的成分中以前未出现过的风味时才添加。 否则它会被忽略。 

最终的输出并不是一次执行。 我们被问到，在所有可能的成分排列中，这个过程可以生产多少种不同的成分组。 如果实际选择的原料不同，而不仅仅是顺序不同，那么两个披萨就会不同。 

n 最大为 100000 且 m 最大为 100000 的约束意味着我们基本上需要线性或近线性处理。 任何尝试枚举排列或模拟排序的方法都是不可能的，因为 n 阶乘太大，甚至边缘上的二次行为也会面临超时风险。 

一个微妙的困难是，等价关系是传递的，但仅以边的形式给出。 仅考虑直接对的天真方法会错误地分类风味组。 

另一个棘手的问题是解释随机性。 问题不在于要求排列的概率或期望值。 它要求任何排列中可以出现的不同结果集的数量。 

## 方法

 强力解释是生成 n 种成分的每种排列，模拟扫描过程，并存储所选择的结果集。 模拟本身在 n 中是线性的，因此这将是 O(n · n!) 时间，即使对于非常小的 n，这也是立即不可行的。 瓶颈不是模拟，而是可能的排序爆炸。 

关键的观察结果是，等式关系图唯一重要的是它的连通分量。 在连接的组件内部，一旦选择了该组件中的任何成分，该组件中的所有其他成分都被认为具有已经见过的风味，并且在以后遇到时将被拒绝。 

现在考虑什么决定了固定排列的最终集合。 在每个连接的组件中，恰好有一个成分将是该组件之间的排列中第一个遇到的成分。 选择该成分，并跳过同一组件中的所有其他成分。 这意味着每个排列都会产生一个集合，其中包含每个连接组件的一个代表。 

关键的一点是，不同的排列可以从同一组件中选择不同的代表，并且这些选择在组件之间是独立的。 在大小为 s 的组件中，其任何 s 个节点都可以是相对于该组件出现在排列中的第一个节点，因此成为选定的代表。 由于组件不会相互干扰，因此可能的披萨总数是组件大小的乘积。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（所有排列）| O(n·n!) | O(n·n!) | O(n) | 太慢了 |
 | 连接组件+计数| O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们首先将给定的对解释为无向图中的边。 通过路径连接的两种成分属于同一风味组，因此我们需要计算连接的成分。 这可以使用不相交集联合结构或 DFS 来完成。

将节点分组为组件后，我们计算每个组件的大小。 每个组件都独立地影响最终计数，因为订购过程绝不会混合组件解析其所选代表的方式。 

最后，对于大小为 s 的每个分量，我们将答案乘以 s，因为对于哪个节点成为随机排列中该分量的第一个遇到的节点，有 s 个选择。 

我们对乘积取模 10^9 + 7。 

它起作用的原因是基于以下不变性：在任何排列中，每个连接的组件都选择一个元素，特别是排序中该组件的最早元素。 从排列到结果披萨的映射完全取决于每个组件中最早元素的选择。 每个这样的选择都可以通过构造一个排列来实现，其中所选元素出现在其组件的所有其他元素之前，并且组件不会相互约束，因此这些选择独立地相乘。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]

def solve():
    n, m = map(int, input().split())
    dsu = DSU(n)

    for _ in range(m):
        a, b = map(int, input().split())
        dsu.union(a - 1, b - 1)

    comp_size = {}
    for i in range(n):
        r = dsu.find(i)
        comp_size[r] = comp_size.get(r, 0) + 1

    ans = 1
    for sz in comp_size.values():
        ans = (ans * sz) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现使用不相交的集合并集结构来构建连接的组件。 每个联合体将风味相当的成分合并成一个具有代表性的组合。 

处理完所有边后，我们迭代所有节点一次以计算每个根组件的大小。 此步骤是必要的，因为 DSU 仅维护结构，而不维护显式组件列表。 

最后的循环在模运算下将所有分量的大小相乘。 结果直接对应于每个成分贡献其代表性成分的方式数量。

 ## 边缘情况

 当没有边缘时，每种成分都会形成自己的组成部分。 该算法将每个节点视为一个单独的根，并且乘积重复变为 1，结果为 1，这与每个排列在单个组件之间产生相同的确定性选择模式的事实相匹配。 

当所有成分通过一条链连接时，DSU 将所有内容压缩为一个组件。 最终答案变为 n，因为 n 个节点中的任何一个都可能是该组件中第一个遇到的，并且这完全确定了生成的披萨集。 

当组件高度不平衡时，例如一个大型组件和许多孤立的节点，乘法仍然可以正常工作，因为孤立的节点贡献系数为 1，只留下较大的组件大小来确定可变性。
