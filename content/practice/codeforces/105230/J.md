 ## 解决方案
 ## 问题理解

 我们有一个矩形网格$n$行和$m$列。 棋子从左上角的单元格开始，像主教一样移动，这意味着它总是沿对角线移动。 然而，与可以自由改变方向的标准国际象棋象不同，这个象在直线对角线段中移动，直到碰到边界，然后它“弹跳”并继续沿着新的反射对角线方向移动。 该运动无限期地持续下去，有效地追踪整个网格的确定性路径。 

每个测试用例的任务是确定网格中有多少个单元格从未被该移动的主教访问过。 

网格尺寸可以非常大，最大可达$10^9 \times 10^9$，而测试用例的数量可以达到$5 \cdot 10^4$。 这立即排除了任何基于模拟的方法。 即使每个测试用例在电路板上迭代一次也是不可能的，因为单个网格最多可以包含$10^{18}$细胞。 任何有效的解决方案都必须使用从运动结构导出的封闭式表达式来计算答案。 

一个微妙的点是，运动是完全确定性和周期性的：一旦主教开始在边界之间弹跳，它最终将重复由位置和方向组成的状态。 这意味着轨迹是细胞子集上的一个循环。 关键的困难是了解这个循环有多大以及它覆盖了哪些细胞。 

直接模拟运动或跟踪访问单元的简单方法将失败，不仅因为时间限制，而且因为在巨大的状态空间中检测重复是不可行的。 即使是在边界图案重复时停止的更仔细的模拟，仍然可以处理最多$O(nm)$最坏情况下的状态。 

## 方法

 一个蛮力的想法是逐步模拟主教的运动。 在每一步中，我们都会沿对角线移动，在撞到墙壁时反映方向，并在哈希集中标记访问过的单元格。 我们继续，直到检测到我们已经返回到之前看到的状态，该状态由位置和方向定义。 

这种方法原则上是正确的，因为运动是确定性的并且最终必须循环。 然而，不同状态的数量与单元数量乘以四个可能的方向成正比，因此周期长度可以是$O(nm)$。 和$n, m$最多$10^9$，这是完全不可行的。 

关键的观察结果是，具有反射的矩形网格上的对角线运动相当于无限平铺的镜像网格上的直线运动。 我们可以想象将网格展开为重复的反射，而不是考虑反弹。 在这个展开的平面中，主教沿直线对角线移动。 由于坐标的模运算，它的路径变得周期性。 

关键的结果是运动分解为沿两条对角线的独立行为。 当且仅当沿两个轴都满足同余条件时，才会访问每个单元格。 访问集形成一个结构化模式，其大小仅取决于由奇偶性和最大公约数结构引起的$n$和$m$。 

分析轨迹后，我们发现访问的单元格数量恰好是：$$n + m - \gcd(n, m)$$这对应于重复完成之前与轨迹相交的不同对角线的数量。 每个被访问的单元恰好位于特定等价类的一条对角线上，并且循环恰好覆盖了许多独特的交叉点。 

由于细胞总数为$nm$，答案就变成：$$nm - (n + m - \gcd(n, m))$$这将问题简化为每个测试用例一次 gcd 计算。

 ## 算法演练

 1. 读取整数$n$和$m$对于每个测试用例。 这些定义了网格的大小，从而定义了可用单元格的总数。 
2. 计算$g = \gcd(n, m)$。 这捕获了网格上对角线运动的基本周期结构。 
3. 计算访问过的单元格数量：$n + m - g$。 这表示在轨迹重复之前到达了多少个不同的单元格。 
4.从单元格总数中减去访问过的单元格$n \cdot m$。 这会产生无法到达的单元格的数量。 
5. 输出结果。 

gcd出现的原因是对角线移动有效地同步了行索引和列索引。 当两个坐标返回到先前看到的同余类时，该路径会重复，这按照 gcd 控制的间隔发生。 

### 为什么它有效

 轨迹可以提升到无限网格，其中反射被棋盘的周期性副本取代。 在该表示中，运动是一条带有斜率的直线$1$或者$-1$。 访问的单元对应于该线与格点模的交点$(n, m)$。 当两个坐标同时返回到其初始残基类别时，就会发生重复，这发生在由$\gcd(n, m)$。 这种结构准确地保证了$n + m - \gcd(n, m)$在路径循环之前遇到不同的位置，因此从总数中减去得到未访问单元的正确计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve():
    out = []
    for line in sys.stdin:
        if not line.strip():
            continue
        n, m = map(int, line.split())
        g = gcd(n, m)
        visited = n + m - g
        out.append(str(n * m - visited))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现独立处理每个测试用例并以对数时间计算 gcd。 减法步骤是使用 Python 的任意精度整数完成的，因此即使对于最大输入大小也不会发生溢出问题。 

一个微妙的实现细节是读取直到 EOF，因为测试用例的数量没有明确给出。 每一行对应一个独立的网格。

from math import gcd

 ## 边缘情况

 对于$n = m = 2$，网格是最小的。 gcd 为 2，因此访问过的单元为$2 + 2 - 2 = 2$。 该算法计算总计$4$, 减去$2$，和输出$2$。 模拟还将确认在路径重复之前仅到达两个对角线单元。 

对于高度倾斜的网格，例如$n = 2, m = 10$，gcd为2。访问次数变为$2 + 10 - 2 = 10$，因此 20 个单元格中有 10 个是可以到达的。 计算可以正确处理不对称性，因为 gcd 仍然捕获维度之间的周期性对齐。 

对于像这样的大方形网格$10^9 \times 10^9$，最大公约数是$10^9$。 访问次数减少到$2 \cdot 10^9 - 10^9 = 10^9$，相对于完整的单元格，几乎所有单元格都未被访问$10^{18}$网格大小。 该公式保持稳定，因为所有运算都保持在整数算术范围内，并且仅取决于 gcd，而不取决于模拟深度。
