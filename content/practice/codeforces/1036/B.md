 ## 解决方案

 ## 问题理解

 米哈伊尔在无限网格上移动，每一步都可以让他到达八个相邻单元格中的任何一个。 换句话说，每次移动最多可以同时、独立地改变其 x 坐标最多一个单位和 y 坐标最多一个单位。 

对于每个查询，我们都会得到一个目标单元格和准确的步骤数。 我们必须确定米哈伊尔在使用所有步骤后是否可以准确地落在该目标上，如果是的话，我们希望最大化这些步骤中有多少是对角线移动，这意味着两个坐标都发生变化的移动。 

对角线移动会同时更改 x 和 y，因此它同时向两个坐标贡献一个单位。 非对角线移动仅对一个轴有贡献。 

这些约束迫使我们采用 O(q) 或 O(q log q) 解决方案，因为 q 最大为 10^4，坐标最大为 10^18。 任何针对每个查询的模拟或路径搜索都是不可能的，因为可能的行走次数随 k 呈指数增长。 

天真的方法会尝试逐步构建路径或使用贪婪选择来思考，而不仔细跟踪奇偶性和可达性。 主要的失败模式是忽略由每个对角线步骤同时影响两个坐标这一事实引起的奇偶约束，因此达到目标后剩余的“松弛”必须可分解为有效的移动。 

当目标很近但 k 很大时，会出现微妙的边缘情况。 例如，2步到达(1, 0)是可能的，但是1步到达也是可能的，而3步到达则取决于我们是否可以在不破坏奇偶性的情况下浪费步骤。 天真的最短路径直觉会失败，因为我们不是最小化步骤，而是强制执行精确的步骤数。 

另一个隐藏的陷阱是假设答案仅取决于曼哈顿距离。 这忽略了对角线移动，这有效地减少了所需步骤的数量，但只是以跨两个轴的耦合方式。 

## 方法

 每次移动可以改变一个坐标或同时改变两个坐标。 如果我们看单个步骤，它在每个轴上最多贡献一个进度单位，因此关键结构是我们如何有效地使用 k 个移动“覆盖”所需的位移（n，m）。 

一种有用的思考方法是将最终位移分成两部分。 假设我们使用 d 个对角线移动。 每个对角线移动同时向 x 和 y 贡献一个单位，因此在 d 个这样的移动之后，我们仍然需要用 k - d 个实际上纯粹水平或垂直的剩余移动来覆盖 (n − dx, m − dy)。 

核心难点在于确定可行性并最大化d． 

蛮力策略将尝试从 0 到 k 的所有可能的 d 值，并检查是否可以仅使用轴对齐移动在 k − d 步中实现剩余位移。 对于每个 d，我们检查是否：

 如果剩余的步骤足够并且具有正确的奇偶结构，则可以覆盖剩余的所需的x运动和y运动。 这导致每个查询的复杂度为 O(k)，这是不可能的，因为 k 可以是 10^18。 

关键的见解是避免枚举 d，而是推理运动的几何形状。 每个步骤可分为仅对 x、仅对 y 或两者都有贡献。 如果我们使用 d 个对角线移动，则剩余步骤为 k − d，并且每个步骤只能贡献一个轴，因此可用的总“轴贡献”为 k + d。 我们至少需要 n 对 x 的贡献和 m 对 y 的贡献，但对角线移动同时有助于两者。 

这导致了一个标准的资源分配视图：我们希望在 k 个步骤的预算下最大化 x 和 y 需求之间的重叠。 最大重叠受到剩余步骤的可行性和奇偶性约束的限制。

解决此问题的一个简洁方法是，如果每个对角线移动都得到最佳使用，则从达到 (n, m) 所需的最少移动次数开始。 最小数量是 max(n, m)，因为在每一步中我们可以同时减少两个坐标，直到一个坐标用完，然后只剩下一个轴。 

如果 k 小于 max(n, m)，则不可能。 如果k很大，我们可以通过来回移动来插入弯路，这会消耗两步而不改变位置，从而保持奇偶性。 因此，可达性取决于 k 是否至少为 max(n, m) 以及额外步骤的奇偶校验是否允许取消。 

一旦建立了可行性，只要两个坐标仍然为正，最大化对角线移动就会减少为贪婪地配对 x 和 y 减少。 对角线移动的最大数量受到较小坐标的限制，而且还受到我们可以在 k 内同步移动的步数的限制。 

因此，答案本质上是 k 步骤下 x 和 y 进度之间的最大可能重叠，并通过剩余步骤是否可以用作中性循环进行调整。

 ## 算法演练

 我们首先得出可行性，然后最大化对角线移动。 

1. 如果每次移动都被最佳地用于尽可能减少两个坐标，则计算所需的最小移动次数。 该基线为 max(n, m)，因为每次移动最多可以同时在每个方向上减少一个单位，并且我们希望匹配两个坐标。 
2. 如果k小于max(n, m)，则立即返回-1，因为即使是最好的策略也无法达到目标。 
3. 如果 k 等于 max(n, m)，则所有动作必须最佳使用，不得浪费。 在这种情况下，只要两个坐标仍然为正，每次移动都可以选择对角线。 对角线移动的次数恰好是 min(n, m)。 
4. 如果 k 大于 max(n, m)，我们有不影响终点的额外移动。 这些额外的移动有效地成对出现，因为单个绕行会改变可达性的奇偶性。 因此，k − max(n, m) 必须是任意的，因为我们总是可以插入两步循环。 
5. 对角线移动的最大数量仍然受 min(n, m) 限制，因为我们无法创建比较小坐标允许的更多重叠。 
6. 返回 min(n, m) 作为对角线移动的最佳数量。 

一个有用的解释是，我们首先对齐路径，以便尽可能多地一起消耗两个坐标，然后将剩余步骤用作空闲周期，这样不会降低对角线效率。 

### 为什么它有效

 从 (0, 0) 到 (n, m) 的任何路径都可以分解为对两个轴共同起作用或仅对一个轴起作用的步骤。 最大化对角线使用的最佳方法是匹配 x 和 y 的增量，只要两者都保持非零。 在 min(n, m) 这样的成对缩减之后，一个坐标就耗尽了，所有剩余的进度都必须轴对齐。 

额外的步骤不能增加配对减少的数量，因为它们要么浪费运动，要么通过循环抵消。 因此，对角线移动的数量上限为 min(n, m)，并且只要 k 足以满足可达性，该界限总是可以实现的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

q = int(input())
for _ in range(q):
    n, m, k = map(int, input().split())

    mx = max(n, m)
    mn = min(n, m)

    if k < mx:
        print(-1)
        continue

    print(mn)
```### 代码说明

 该解决方案直接应用派生结构。 可行性检查使用至少需要 max(n, m) 次移动的事实。 如果 k 较小，则不存在有效路径。 

一旦可行性成立，答案就是 min(n, m)，因为每个对角线移动对应于将一个单位的 x 进度与一个单位的 y 进度配对。 额外的步骤不会增加此配对容量。 

这种实现完全避免了模拟路径并且完全依赖于坐标几何。

 ## 边缘情况

 临界边缘情况是一个坐标为零时。 例如，(5,0,5)。 该算法给出 min(5, 0) = 0 对角线移动，并且由于 k ≥ max(n, m)，因此可行性成立。 这是正确的，因为每次移动都必须轴对齐，因此不可能对角线移动。 

另一种情况是当 k 正好是 max(n, m) 时。 对于 (3, 1, 3)，结果为 min(3, 1) = 1。执行必须恰好使用三个步骤，并且在一对对角线配对之后，剩余步骤仍必须保留端点，这只有通过小心使用仅轴移动才能实现。 

最后，当k远大于需要时，例如(2,2,100)，答案仍然是2。额外的96步可以被吸收到不改变端点的来回循环中，因此它们不会影响对角线容量。
