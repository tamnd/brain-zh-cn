 ## 解决方案
 ## 问题理解

 该图是由 0 到 2n 整数 x 坐标处的点绘制的折线。 最初每个点都位于 x 轴上，因此每个 y 坐标均为零。 修改形状的唯一方法是选择一个奇数索引点并每次移动将其高度增加一个单位。 偶数索引点永远不会改变。 

一旦指定了高度，连续的点就会通过直线段连接起来，形成多边形链。 “绘图面积”是该链与 x 轴之间的几何面积。 “图的高度”就是所有点中的最大 y 值。 

任务是选择将每个奇数索引点增加多少次，以使总封闭面积恰好为 k，同时使最大高度尽可能小。 

约束条件允许 n 和 k 最大为 10^18。 这立即排除了任何尝试模拟增量或显式构建结构的方法。 任何解决方案都必须将问题压缩为封闭式表达式，或者最坏的情况是压缩为对数或常数时间计算。 

正确解释几何会产生微妙的边缘情况。 天真的读者可能会认为该面积以复杂的方式取决于每个线段的形状，或者相邻奇数点之间的相互作用很重要。 例如，人们可能怀疑增加相邻奇点会以一种不平凡的方式改变共享段贡献。 如果这是真的，即使很小的投入也需要仔细的几何积累。 然而，这种直觉会导致过于复杂和不正确的动态公式。 

另一个潜在的陷阱是假设不均匀分布的高度变化可以比均匀分布更有效地降低最大高度。 这通常会导致不正确的贪婪策略，例如在移动到下一个点之前完全填充一个点，这可能在局部看起来是最优的，但在全局上却失败了。 

## 方法

 蛮力的想法是将每个奇数索引点视为一个变量并模拟在它们之间分配单位增量。 每个增量都会更新几何形状，并且我们会在每次分配后重新计算总面积。 这需要探索 n 个位置上 k 个增量的所有分布。 状态的数量是组合的，本质上是由 k 部分组成的弱组合的数量，其数量级为$\binom{k+n-1}{n-1}$。 即使对于中等值，这也是完全不可行的。 

关键的简化来自于观察区域是如何形成的。 每个线段将零高度点连接到奇数点，反之亦然。 由于偶数索引点始终保持为零，因此奇数索引点创建的每个“凹凸”都会形成两个相同的三角形贡献，每个相邻线段上都有一个。 这两半组合成一个线性贡献，仅与该奇数点的高度成正比。 这消除了位置之间的所有相互作用。 

一旦面积被视为独立贡献的纯和，问题就变成纯粹的组合问题：我们将 k 个相同的单元分布在 n 个独立的存储桶中，并且我们希望最小化最大存储桶值。 这是一种经典的负载平衡结构，通过尽可能均衡来实现最优性。 

如果我们试图将最大高度保持在H，那么n个奇数位置中的每一个最多只能为总面积贡献H，因此可实现的最大面积为n·H。 这立即给出了H.的可行性条件。

 ## 算法演练

 1. 将每个奇数索引点解释为自变量$a_i$，代表增加了多少倍。 这将几何问题转化为 n 个变量的分配问题。 
2. 计算每个$a_i$对面积有贡献。 在奇数点增加的每个单位都会使面积总共增加一个单位，因为它在相邻线段上形成两个对称的半三角形。 这确定总面积等于$\sum a_i$。 
3. 将问题重新表述为在 n 个变量之间分配 k 个相同的单元$a_1, a_2, \dots, a_n$，其中目标是最小化$\max a_i$。 
4. 观察到，对于固定的最大高度 H，当所有变量等于 H 时，出现最大可能的总和，给出总 n·H。 这意味着可行性需要$n \cdot H \ge k$。 
5. 选择满足该不等式的最小整数H，因为任何较小的值都无法容纳k，并且任何较大的值都是不必要的。 
6. 得出结论$H = \lceil k / n \rceil$。 

### 为什么它有效

 关键的不变量是面积仅取决于奇数索引高度的多重集并且恰好是它们的总和。 由于变量之间不存在耦合，任何保留总和的重新分配都不会改变面积。 因此，在固定总和下最小化最大值就等于尽可能均匀地平衡分布。 任何与均匀性的偏差只会增加最大值，而不会增加超出相同总和界限的容量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
print((k + n - 1) // n)
```整个实现简化为计算上限除法。 关键的简化是几何体折叠成附加模型，因此不需要模拟或结构构建。 唯一需要注意的细节是确保正确执行整数上限除法； 使用`(k + n - 1) // n`避免浮点问题并安全处理高达 10^18 的大值。 

## 工作示例

 ### 示例 1

 输入：```
4 3
```我们有 n = 4 个独立贡献的奇数位置。 我们将 3 个单位分配给 4 个变量。 

| 步骤| 分布状态| 电流最大值| 总金额 |
 | ---| ---| ---| ---|
 | 1 | [1,0,0,0]| 1 | 1 |
 | 2 | [1,1,0,0]| 1 | 2 |
 | 3 | [1,1,1,0] | 1 | 3 |

 所需的最大高度为 1，因为每个位置可以放置的所有单元不超过 1 个。 这与公式 ceil(3/4) = 1 匹配。 

### 示例 2

 输入：```
4 12
```现在我们将 12 个单位分配到 4 个位置。 

| 步骤| 分布状态| 电流最大值| 总金额 |
 | ---| ---| ---| ---|
 | 1 | [3,3,3,3] | 3 | 12 | 12

 这里的分布是完美平衡的，因此最大高度为 3。这与 ceil(12/4) = 3 匹配，并显示所有变量必须达到相同水平的严格情况。

## 测试用例```python
import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    n, k = map(int, input().split())
    print((k + n - 1) // n)

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    old_stdout = sys.stdout
    sys.stdout = io.StringIO()
    solve()
    out = sys.stdout.getvalue()
    sys.stdin = old_stdin
    sys.stdout = old_stdout
    return out.strip()

 ## 边缘情况

 一个常见的误解是当 k 小于 n 时。 例如，使用输入`n = 10, k = 3`，最佳策略是将每个单元放置在不同的位置，产生像这样的高度`[1,1,1,0,...]`，因此最大高度为 1。公式正确返回`ceil(3/10) = 1`，符合这个直觉。 

另一种边缘情况是当 k 可以被 n 整除时，例如`n = 5, k = 10`。 最优配置是完全均匀的，每个奇数点都设置为 2。算法准确地返回 2，并且任何不均匀分布都不能在不违反总和约束的情况下进一步减小最大值。 

最后，当 k 非常大时，例如 10^18，该解决方案仍然有效，因为它避免构造任何分布并且仅依赖于整数运算，确保不会溢出或性能问题。
