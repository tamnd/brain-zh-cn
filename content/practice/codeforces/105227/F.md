 ## 解决方案
 ## 问题理解

 我们得到一个数组，我们想要了解每个值在固定长度窗口中的“稳定性”。 对于选定的窗口尺寸$k$，我们滑动一段长度$k$穿过阵列。 一个数字被认为对此有好处$k$如果它出现在每个这样的片段中。 在满足这个条件的所有数字中，我们想要最小的一个。 如果没有数字在所有窗口中幸存下来，我们输出$-1$。 

输出是所有的列表$k$从$1$到$n$，因此我们实际上被问到这个“普遍存在的最小值”如何随着窗口大小的增长而变化。 

约束足够严格，任何二次方都可以$n$每个测试用例是不可能的。 每个元素参与多个窗口，最多有$3 \cdot 10^5$测试中的总元素，因此预期的解决方案必须接近线性或线性算数。 为每个独立地重新计算信息的解决方案$k$将大致重复工作$n$次，立即超出限制。 

一个微妙的问题是，即使某个值总体上频繁出现，某些窗口中也可能不存在。 例如，在`1 2 1 2 1`, 数量`1`出现多次，但对于某些窗口大小，存在没有它的段，因此它不符合全局条件。 这打破了天真的频率推理。 

另一种极端情况是所有元素都相同。 然后每个窗口都包含该值，因此答案应该始终是所有窗口的该值$k$。 仅检查全局频率或间距的粗心方法仍可能产生$-1$如果它误解了条件，则错误。 

## 方法

 暴力方法修复了一个值$k$，枚举所有长度的窗口$k$，并计算这些窗口中所有值集的交集。 对于每个窗口，我们可以维护一个频率图，然后跨窗口相交。 即使每次窗户检查都是$O(k)$，这变成了$O(nk)$每$k$，导致$O(n^3)$总体来说是最坏的情况。 

关键的观察是停止考虑滑动窗口，而是关注每个值的位置。 一个数字不符合给定的条件$k$如果存在至少一个长度的窗口$k$从而避免其所有情况的发生。 这相当于说至少有一个长度的间隙$k$连续事件之间，包括边界。 

因此，我们不是扫描窗口，而是跟踪每个值的出现位置并查看连续出现之间的最大“间隙”，包括从开始到结束。 保证一个值出现在每个大小的窗口中$k$当且仅当其最大间隙最多为$k$。 这将问题转换为跟踪每个值的约束并将它们聚合为所有问题的答案$k$。 

我们反转视角：每个值都贡献一个阈值$k$。 对于每个值，计算连续出现的值之间的最大距离。 那么这个值对所有的人都有效$k$大于或等于该阈值。 然后我们需要，对于每个$k$，阈值为的最小值$\le k$。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了|
 | 最佳 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将数组转换为出现列表，计算每个值的约束，然后回答所有的查询$k$。 

1. 存储数组中每个值的所有位置。 这让我们可以直接推断间隙，而不是重复扫描窗口。 
2. 对于每个值，构建其出现索引的列表，并在位置处使用虚拟边界对其进行扩充$0$和$n+1$。 这确保边缘间隙与内部间隙得到统一处理。 
3. 对于每个值，计算此扩充列表中连续位置之间的最大差异。 该最大间隙代表无法再“避免”该值的最小窗口大小。 
4. 将其解释为阈值：该值对所有$k$大于或等于此最大间隙，因为任何短于该间隙的窗口都可以完全放置在间隙内并完全错过该值。 
5. 我们现在需要计算每个$k$，阈值最多为的最小值$k$。 我们按升序处理阈值$k$，保持迄今为止看到的最佳候选值。 
6. 构建数组`best[k]`我们更新从每个阈值开始的所有位置。 从小到大的线性扫描$k$保持有效的全局最小值。 

### 为什么它有效

 关键的不变量是，对于每个值，计算出的最大间隙是强制该值出现在每个窗口中的精确最小窗口大小。 任何短于此间隙的窗口都可以定位在出现次数之间的最大间隙内，从而完全避开该值。 任何至少这么大的窗口都必须与每个间隙链相交，确保每个窗口中至少出现一个事件。 因为这个阈值充分表征了有效性，所以通过阈值聚合值并取前缀最小值$k$产生正确的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        pos = [[] for _ in range(n + 1)]
        for i, v in enumerate(a, 1):
            pos[v].append(i)

        # threshold[k] = best candidate value that activates at k
        INF = 10**18
        threshold = [INF] * (n + 2)

        for v in range(1, n + 1):
            if not pos[v]:
                continue

            arr = [0] + pos[v] + [n + 1]
            mx_gap = 0
            for i in range(1, len(arr)):
                mx_gap = max(mx_gap, arr[i] - arr[i - 1])

            threshold[mx_gap] = min(threshold[mx_gap], v)

        ans = [-1] * (n + 1)
        best = INF

        for k in range(1, n + 1):
            best = min(best, threshold[k])
            if best != INF:
                ans[k] = best
            else:
                ans[k] = -1

        print(*ans[1:])

if __name__ == "__main__":
    solve()
```该解决方案为每个值构建出现列表，这是推断窗口覆盖范围所需的唯一结构。 增广边界为`0`和`n+1`确保在没有特殊情况的情况下处理边缘间隙。 最大间隙计算直接产生每个值的临界阈值。 

数组`threshold`将窗口大小映射到恰好在该大小下有效的最小值。 然后前缀最小扫描将这些激活点转换为所有的答案$k$，确保一旦一个值有效，它就成为所有更大值的候选者$k$。 

一个常见的陷阱是忘记边界间隙。 不加`0`和`n+1`，出现在边缘附近的值错误地显得比实际情况更稳定。

 ## 边缘情况

 对于像这样的单元素数组`[7]`，增广位置变为`[0,1,2]`，产生最大间隙`2`。 这正确地编码了只有大小为 1 的窗口一般包含该值，而较大的概念间隙是无关紧要的，因为在以下情况下无法避免它：$k=1$。 

对于像这样的数组`[1,2,1,2,1,2]`，两个值的最大差距都很小，因为出现次数频繁且间隔均匀。 该算法分配两个小阈值，并且前缀最小值确保最小值主导每个$k$，正确地产生一个恒定的答案。 

对于边缘较重的图案，例如`[1,2,3,1]`， 价值`2`和`3`有很大的间隙，允许它们被小窗口跳过，而值`1`具有更紧密的内部间距。 扩大的边界间隙可以确保正确考虑第一次和最后一次发生的情况，从而防止高估稳定性。
