 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的字符串。 我们可以从这个字符串中删除字符，同时保留顺序，从而产生任何子序列。 在所有这些子序列中，我们想要一个回文子序列，并且在所有回文子序列中，我们想要字典序上最大的子序列。 

字符串长度最多为 10，因此输入很小。 这立即改变了问题的本质：指数探索已经是可行的，但我们仍然应该以清晰的结构洞察为目标，而不是暴力破解所有子序列。 

这里的字典顺序遵循标准规则：从左到右比较，以字符不同的第一个位置决定，或者如果一个是另一个的前缀，则较长的字符串较大。 

一个天真的陷阱是假设我们需要从两端贪婪地“构建”一个回文。 但这会失败，因为最佳选择取决于全局结构，而不是局部匹配。 另一个错误是思考时间越长总会获胜。 这是错误的，因为较长的子序列可能以较小的字符开头，从而立即按字典顺序丢失。 

当所有字符都不同时，会出现小边缘情况。 那么每个回文子序列的长度都是1，答案就是最大字符。 另一种情况是最佳答案不是最长的回文，如示例“codeforces”中答案为“s”的情况。 

## 方法

 蛮力视图很简单：枚举字符串的所有子序列，检查每个子序列是否是回文，并保留最好的字典顺序。 最多有 2ⁿ 子序列。 对于每个，检查回文都需要 O(n)，因此总复杂度为 O(n·2ⁿ)。 当 n ≤ 10 时，最多约为 10,240 次检查，这在实践中完全没问题。 

然而，这种暴力方法在概念上是矫枉过正的。 字符串中回文子序列的结构有一个关键的简化：任何回文都有匹配的第一个和最后一个字符，并且按字典顺序排列的较大字符串优先考虑第一个字符。 所以我们首先应该了解答案的第一个字符必须是什么。 

如果我们选择任何字符 c 作为回文的第一个字符，那么最后一个字符也必须是 c，并且两者都必须来自字符串中 c 的出现。 因此，字典顺序上最大的回文必须以至少出现一次的最大可能字符开始。 一旦我们选择了这样一个字符c，我们总是可以形成一个有效的回文子序列，其中仅包含一次c，或者如果至少有两个c，则包含两次c。 

现在比较一下可能性。 如果c至少出现两次，我们就可以组成“cc”。 任何以 c 开头的较长回文也必须以 c 开头，然后在两个选定的出现之间继续。 但任何内部部分都不能在前面引入小于 c 的字符，否则字典顺序不会提高。 由于 c 已经是字符串中的最大字符，因此没有什么比以 c 开头并尽可能保留 c 的序列更好的了，它会折叠为重复 c。 

因此，问题就简化为找到字符串中的最大字符并计算它出现的次数。 如果出现一次，答案就是该字符。 如果它出现 k 次，则最好的回文是该字符重复 k 次，因为我们可以将所有出现的事件作为子序列，并且它基本上是一个回文。 

这是关键的结构崩溃：我们不需要探索回文，只需要最大字符的频率。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·2ⁿ) | O(n) | 已接受但没有必要 |
 | 最大字符频率 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

我们处理该字符串一次以确定存在的最大字符以及它出现的次数。 

1. 扫描字符串并维护目前为止看到的最大字符。 这是正确的，因为词典编排最优性首先取决于主角，所以我们必须在全局范围内最大化它。 
2. 计算这个最大字符出现了多少次。 这很重要，因为重复相同的最大字符可以保留回文结构和字典顺序值。 
3. 构造一个由该字符组成的字符串，重复该字符的次数与它在输入中出现的次数相同。 
4. 直接输出这个构造的字符串，因为它是一个有效的子序列和回文。 

不明显的部分是为什么我们可以获取所有出现的最大字符。 任何仅由相同字符组成的子序列始终是回文，因为反转它不会改变任何内容。 删除任何出现的字符只会缩短字符串，而不会改善字典顺序，因为所有字符都是相等的。 因此，在同等起始候选者中，最大值减少为最大长度。 

### 为什么它有效

 任何回文子序列都有第一个字符，并且字典顺序立即由该字符决定，除非两个候选者共享它。 因此，最佳解决方案必须从字符串中最大可能的字符开始。 一旦限制为该字符，任何其他字符都无法改善字典顺序，并且仅添加相同的字符可以保持有效性和最优性。 这迫使答案成为排列为回文的最大字符的所有出现的多重集，这只是该字符的重复。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

mx = max(s)
cnt = s.count(mx)

print(mx * cnt)
```该解决方案首先使用 Python 内置的字符比较来计算最大字符，这可以直接工作，因为 ASCII 顺序与字典顺序匹配。 然后它会计算该字符的出现次数。 输出是通过重复字符构造的。 

一个微妙的点是，我们不需要显式验证回文性，因为任何由单个重复字符组成的字符串始终是回文。 另一个微妙之处是我们不需要明确考虑子序列约束：可以按顺序选择最大字符的每次出现，因此重复始终可以作为子序列实现。 

## 工作示例

 考虑“雷达”。 

我们扫描发现最大的字符是‘r’。 它出现两次。 

| 步骤| 最大字符数 | 计数|
 | --- | --- | --- |
 | r | r | 1 |
 | 一个 | r | 1 |
 | d | r | 1 |
 | 一个 | r | 1 |
 | r | r | 2 |

 结果是“rr”。 这表明，尽管“雷达”本身是一个回文，但仅关注主导字符就可以形成字典序上更大的子序列。 

现在考虑“codeforces”。 

最大字符为“s”，仅出现一次。 

| 步骤| 最大字符数 | 计数|
 | --- | --- | --- |
 | c | c | 1 |
 | 哦| 哦| 1 |
 | d | 哦| 1 |
 | 电子| 哦| 1 |
 | f | 哦| 1 |
 | 哦| 哦| 1 |
 | r | r | 1 |
 | c | r | 1 |
 | 电子| r | 1 |
 | s | s | 1 |

 输出为“s”，因为不存在重复字符来形成更长的回文。 

这些痕迹证实该算法纯粹是由最大字符的优势驱动的，并且忽略了所有其他结构。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline
    s = input().strip()
    mx = max(s)
    cnt = s.count(mx)
    return mx * cnt

# provided samples

 ## 边缘情况

 对于像“a”这样的单字符字符串，算法将“a”识别为最大且唯一的字符，从而生成“a”。 该构造仍然有效，因为重复计数为 1，因此不需要特殊处理。 

对于最大字符出现多次的字符串，例如“zzxyzz”，扫描会正确累积所有出现的“z”。 由于全部相等，因此它们之间的顺序并不重要，并且生成的重复字符串始终是回文串，并且在有效子序列中按字典顺序最大。
