 ## 解决方案
 ## 问题理解

 我们得到一个使用水平线和垂直线绘制的网格。 网格完全定义为$n$水平线和$m$垂直线，在平面上均匀分布。 我们形成的任何矩形都必须使用这些线作为边界，这意味着矩形的每一边恰好位于其中一条网格线上。 

通过选择两条不同的水平线作为其顶部和底部边界以及两条不同的垂直线作为其左右边界来确定有效的矩形。 每一个这样的选择都唯一地定义一个矩形。 仅当两个矩形的四个角点重合时，才认为两个矩形相同，这又完全对应于选择同一对水平线和同一对垂直线。 

任务是计算存在多少个这样的非退化矩形。 

约束很小，有$n, m \le 100$，因此三次或二次方法也是可行的。 然而，问题的结构表明我们应该期待组合计数恒等式，而不是任何模拟。 任何通过检查几何形状显式迭代所有矩形的方法都已经是有效的$O(n^2 m^2)$，这在这里仍然没问题，但没有必要。 

一个常见的错误是用“网格单元”而不是“网格线”来思考。 如果有人解读$n$和$m$作为细胞而不是细胞系，他们可能会错误地使用$n \cdot m$或类似的表达方式。 另一个微妙的错误是忘记了矩形需要两条不同的水平线和两条不同的垂直线，因此情况$n < 2$或者$m < 2$必须产生零。 

例如，如果$n = 1, m = 4$，只有一条水平线，所以根本无法形成矩形。 正确答案是 0。任何假设“选择任意对”而不检查可行性的方法都会默默地产生无效的组合。 

## 方法

 强力解释首先明确选择顶部和底部水平线，然后选择左侧和右侧垂直线，并计算每个有效组合。 这已经是该问题的直接模型：每个矩形恰好对应于四个不同边界线。 

蛮力方法的工作原理是迭代所有水平线对和垂直线对。 水平对的数量是$\binom{n}{2}$，垂直对的数量是$\binom{m}{2}$。 对于每一种组合，我们计算一个矩形。 这是正确的，因为矩形是由其边界线唯一确定的。 

仅当我们以几何枚举术语思考时，例如逐个单元扫描所有矩形或显式验证几何图形，才会出现效率低下的情况。 即便如此，问题的规模仍然足够小$O(n^2 m^2)$会过去的，因为$n, m \le 100$, 最多给予$10^8$操作，边界但在优化的Python中仍然可以接受。 

关键的观察是水平和垂直选择是独立的。 一旦我们选择了两条水平线，它们就可以与任何有效的垂直线对配对。 这将问题分解为两个组合计数。 

所以矩形的总数是：$$\binom{n}{2} \cdot \binom{m}{2}$$这消除了所有迭代的需要，并将解决方案简化为常数时间算术。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2 m^2)$|$O(1)$| 已接受但没有必要 |
 | 最佳|$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 读取整数$n$和$m$。 这些表示可用水平和垂直网格线的数量。 
2. 计算有多少种方法可以选择两条不同的水平线。 这是$\frac{n(n-1)}{2}$。 原因是矩形需要顶部和底部边界，顺序并不重要。 
3. 计算有多少种方法可以选择两条不同的垂直线。 这是$\frac{m(m-1)}{2}$，出于同样的原因。 
4. 将这两个值相乘。 每个水平对可以与任何垂直对组合，并且每个组合恰好形成一个矩形。 
5. 输出产品。 

### 为什么它有效

 每个矩形都通过选择两条水平线和两条垂直线来唯一标识。 不同选择之间没有重叠，并且任何选择都会生成多个矩形。 这会在有效矩形集和两个组合集的笛卡尔积之间创建双射。 由于这种一对一的映射，计算矩形精确地减少为乘以独立的组合选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

# choose any 2 horizontal lines
h = n * (n - 1) // 2

# choose any 2 vertical lines
v = m * (m - 1) // 2

print(h * v)
```该解决方案完全依赖于组合恒等式而不是迭代。 当人们尝试模拟网格单元而不是使用行计数时，最常见的实现错误是差一索引。 另一个问题是忘记整数除法，但这里我们明确使用`//`以确保整数结果。 

由于任意精度整数，乘法步骤在 Python 中是安全的，因此不存在溢出问题。 

## 工作示例

 ### 示例 1

 输入：```
3 4
```我们一步步计算组合。 

| 步骤| 水平线| 垂直线 | 水平对| 垂直对| 结果 |
 | ---| ---| ---| ---| ---| ---|
 | 初始| 3 | 4 | - | - | - |
 | 计算对 | 3 | 4 | 3 | 6 | - |
 | 决赛| 3 | 4 | 3 | 6 | 18 | 18

 这里，$\binom{3}{2} = 3$和$\binom{4}{2} = 6$，给出 18 个矩形。 

这证实了即使是小网格也会由于边界的组合配对而产生多个矩形。

import sys, io

def run(inp: str) -> str:

 ## 边缘情况

 当$n = 1$或者$m = 1$，该公式正确地得出零，因为二项式项之一变为零。 用于输入`1 4`，计算变为$\binom{1}{2} = 0$，所以结果是 0。算法自然地处理这个问题，没有分支。 

为了`2 2`，两个维度都恰好有一对可能的直线。 计算结果$1 \cdot 1 = 1$，它对应于由外边界形成的单个矩形。 

对于较大但仍然较小的输入，例如`3 3`，算法产生$3 \cdot 3 = 9$，匹配二维交叉网格中所有可能矩形的完整枚举。
