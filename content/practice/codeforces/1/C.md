## 问题理解

 我们给出了某个正多边形的三个顶点的坐标。 多边形本身是未知的：我们不知道它有多少条边，它的中心在哪里，或者三个点对应于哪些顶点。 在所有可以包含这三个点作为顶点的正多边形中，我们必须找到面积最小的一个。 

关键的几何事实是每个正多边形都有一个外接圆。 所有顶点都位于同一个圆上，且角度间隔相等。 由于三个不共线的点唯一确定一个圆，因此三根柱子已经完全固定了外接圆。 唯一剩下的问题是该圆上存在多少个等距顶点，以便所有三个给定点都落在顶点位置上。 

就输入大小而言，约束很小，只有三个点。 挑战完全是数学上的。 对最多 100 条边的所有多边形进行强力搜索是完全可行的，但我们仍然需要可靠的几何特征来确定三个点何时可以属于同一个正多边形。 

浮点精度是这个问题的危险部分。 正确的多边形可能取决于 π/7 或 π/13 等角度，并且直接相等检查将因舍入误差而失败。 另一个微妙的问题是，这三个点不能保证是相邻的顶点。 假设连续顶点的简单解决方案会立即产生错误的答案。 

考虑这个例子：```
0 0
1 0
0 1
```这三点形成一个直角三角形。 粗心的方法可能会假设多边形是三角形，因为三个顶点是已知的。 但这些点实际上是一个正方形的顶点，其面积小于通过这些点所围成的外接等边三角形。 

当多边形有很多边并且圆心角变得非常小时，就会出现另一种棘手的情况。 例如，如果真正的多边形有 97 条边，重复角度计算中累积的浮点误差很容易使 gcd 式角度减小变得不稳定，除非我们仔细使用 epsilon。 

第三个常见错误是错误地计算近退化三角形的外心。 尽管输入保证了有效的答案，但如果不小心实施，不稳定的公式可能会产生巨大的精度损失。 

## 方法

 暴力破解的想法是从观察多边形边数最多为 100 开始的。我们可以尝试所有可能的方法`n`从 3 到 100。 

对于固定的`n`, 一个常规的`n`-gon 将圆分成多个角度的弧`2π/n`。 如果给定的三个点是该多边形的顶点，则每对点之间的圆心角必须是以下整数倍`2π/n`。 

所以暴力算法看起来像这样：

 1. 计算三点的外心和外接半径。 
2. 计算各点绕中心的极角。 
3.对于每一个`n`从 3 到 100：

 - 检查所有成对的角度差是否是以下的倍数`2π/n`。 
- 如果是，计算多边形面积。 
4.输出最小面积。 

这已经在一定范围内发挥作用了。 我们只测试 98 个多边形尺寸，并且每个测试都使用恒定时间几何。 总的工作量可以忽略不计。 

真正的挑战不是速度而是正确性。 将浮点角度与精确倍数​​进行比较是脆弱的。 如果我们检查：```
diff % step == 0
```由于精度噪声，该解决方案立即失败。 

重要的见解是多边形结构可以通过圆心角的最大公约数来描述。 

假设这三个点对应于由`a`,`b`， 和`c`多边形上的步骤。 它们的圆心角为：```
a * 2π/n
b * 2π/n
c * 2π/n
```这意味着每个角度差必须共享相同的基本单位角。 最小的有效多边形正是其单位角为所有圆心角差的gcd的多边形。 

在整数运算中，我们使用欧几里得的 gcd 算法。 对于浮点角度，我们以数值方式模拟相同的想法。 自从`n ≤ 100`，一个更简单的策略就足够了：尝试每一个`n`并验证所有角度是否是整数倍`2π/n`在epsilon内。 

一旦正确`n`众所周知，多边形面积是根据等腰三角形的标准分解得出的。 一个常规的`n`-gon 外接圆半径`R`有面积：$A = \frac{nR^2\sin\left(\frac{2\pi}{n}\right)}{2}$蛮力方法和最优方法实际上是相同的渐进方法，因为边限仅为 100。最优性来自于几何洞察力，它将多边形的验证简化为外接圆上的角度整除性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过简单的角度检查对所有多边形进行暴力破解 | O(100) | O(1) | O(1) | 精度带来风险 |
 | 几何角度与 epsilon 的整除性 | O(100) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读懂三点。 

这三个点唯一确定一个外接圆，因为它们保证不共线。 
2. 计算这些点形成的三角形的边长。 

我们需要这些长度来使用三角形面积公式计算外接圆半径。 
3. 使用叉积计算三角形面积。 

如果三角形的顶点是`A`,`B`， 和`C`， 然后：$\text{Area} = \frac{|(B-A) \times (C-A)|}{2}$4. 计算外接圆半径`R`。 

使用标准关系：$R = \frac{abc}{4\Delta}$在哪里`a`,`b`， 和`c`是边长和`Δ`是三角形面积。 
5. 计算三个圆心角。 

三角形的每条边在外心处都有一定的角度。 根据扩展的正弦定理：$\theta = 2\arcsin\left(\frac{a}{2R}\right)$我们计算所有三个三角形边。 
6. 尝试每个多边形大小`n`从 3 到 100。 

正则的基本角步长`n`-gon 是：$\frac{2\pi}{n}$7. 对于每个圆心角，检查其是否为步距角的整数倍。 

我们计算：```
ratio = angle / step
```并验证`ratio`足够接近整数。 
8. 首次有效`n`给出最小面积。 

更大的`n`表示同一外接圆上有更多边，这总是使多边形面积向圆面积增加。 
9. 使用外接圆半径公式计算正多边形的面积。 
10. 以足够的精度打印答案。 

### 为什么它有效

 这三个点位于唯一的外接圆上。 任何包含它们的正多边形都必须使用这个精确的圆作为其外接圆。 

一个常规的`n`-gon 将圆分成相等的角步长`2π/n`。 因此，两个多边形顶点之间的每条弧都必须是该步长的整数倍。 三个给定点确定三个中心角，并且这些角都必须能被相同的基本步距角整除。 

通过检查每个`n`从最小到最大，第一个有效多边形保证具有最小面积。 对于固定外接圆半径，增加边数会严格增加正多边形面积。 

## Python 解决方案```python
import sys
import math

input = sys.stdin.readline

EPS = 1e-6

def dist(x1, y1, x2, y2):
    return math.hypot(x1 - x2, y1 - y2)

def is_multiple(angle, step):
    k = round(angle / step)
    return abs(angle - k * step) < EPS

def solve():
    points = [tuple(map(float, input().split())) for _ in range(3)]

    (x1, y1), (x2, y2), (x3, y3) = points

    a = dist(x2, y2, x3, y3)
    b = dist(x1, y1, x3, y3)
    c = dist(x1, y1, x2, y2)

    cross = abs((x2 - x1) * (y3 - y1) - (y2 - y1) * (x3 - x1))
    triangle_area = cross / 2.0

    R = (a * b * c) / (4.0 * triangle_area)

    angles = []
    for side in [a, b, c]:
        value = side / (2.0 * R)
        value = max(-1.0, min(1.0, value))
        angles.append(2.0 * math.asin(value))

    answer = 0.0

    for n in range(3, 101):
        step = 2.0 * math.pi / n

        ok = True

        for angle in angles:
            if not is_multiple(angle, step):
                ok = False
                break

        if ok:
            answer = 0.5 * n * R * R * math.sin(2.0 * math.pi / n)
            break

    print(f"{answer:.10f}")

solve()
```第一部分计算三角形边长和面积。 叉积公式在数值上是稳定的，并且避免了 Heron 公式，该公式可能会丢失细三角形的精度。 

外接圆半径计算使用边长和三角形面积之间的经典关系。 由于所有坐标都很小，因此浮点溢出不是问题。 

中心角由弦长得出。 对于长度的弦`a`在一个半径的圆内`R`，对应的圆心角满足：```
a = 2R sin(theta / 2)
```我们使用反转这个关系`asin`。 夹钳至`[-1, 1]`很重要，因为浮点错误可能会产生类似的值`1.0000000002`，这会崩溃`asin`。 

验证步骤检查每个角度是否是多边形步距角的倍数。 我们将比率舍入为最接近的整数，并将重构误差与 epsilon 进行比较。 对浮点数进行直接模运算的可靠性要低得多。 

循环开始于`n = 3`，因此第一个有效多边形自动具有最小面积。

 ## 边缘情况

 常见的失败情况是假设这三个点是连续的顶点。 

输入：```
0 0
1 1
0 1
```这些点形成的三角形并不规则。 一个简单的算法可能会停止在`n = 3`并计算外接三角形面积。 相反，我们的算法检查圆心角的整除性。 三角形失败，因为角度不是角度的倍数`2π/3`。 平方成功是因为所有角度都是`π/2`。 

另一个微妙的情况是浮点舍入将值稍微超出有效的三角范围。 

输入：```
1 0
-1 0
0 1
```从数学上讲，一个弦长正好等于`2R`，所以里面的表达式`asin`应该正是`1`。 浮点运算可能会产生`1.0000000001`。 无需夹紧，`asin`抛出域错误。 该实现显式地将值限制为`[-1, 1]`。 

第三种危险情况是具有许多边的多边形，其中角步变得很小。 

输入：```
1 0
0.9980267284 0.0627905195
0.9921147013 0.1253332336
```这些是正则 100 边形的连续顶点。 对浮点角度的直接相等检查失败，因为计算出的角度包含累积的数字噪声。 基于 epsilon 的整数倍检查可以正确识别多边形结构。
