## 问题理解

 我们有一个矩形广场，即剧院广场，其尺寸为长 _n_ 米，宽 _m_ 米。 该市希望用 _a_ 面的方形石板铺砌整个区域。 问题要求完全覆盖广场所需的最少石板数量。 每块石板必须与广场的边对齐放置； 我们不能切割它们，但石板延伸到广场边缘之外是可以接受的。 

输入的数字可以非常大，最多可达 10^9。 这告诉我们，任何迭代每米正方形或每个可能的图块放置的方法都是不可行的。 我们需要一个恒定时间、基于算术的解决方案。 

非明显的边缘情况包括正方形尺寸不能被图块尺寸整除的情况。 例如，如果正方形为 6×6 米，而石板为 4×4 米，则简单的整数除法可能会建议每个维度有 1 个石板 (6 // 4 = 1)，总共为 1。这显然是错误的，因为单个 4×4 石板无法覆盖 6 米。 正确的方法必须对除法进行四舍五入以计算余数。 

另一个微妙的情况是当 _n_ 或 _m_ 可以被 _a_ 整除时，这不应该添加额外的石板。 例如，具有 4×4 块的 12×8 正方形应恰好使用 6 个块（沿长度方向 3 个，沿宽度方向 2 个），而不是更多。 

## 方法

 暴力方法会尝试模拟放置每个瓷砖，计算沿长度和宽度适合的数量，并为剩余空间添加额外的瓷砖。 从概念上讲，我们可以想象一个嵌套循环在每个图块的左上角进行迭代。 虽然正确，但对于最大输入（n = m = 10^9，a = 1），最坏情况下的操作数量可能达到 10^18，这是完全不可行的。 

关键的见解是我们不需要模拟任何东西。 每个维度都可以独立处理。 沿着一维，所需的瓷砖数量是正方形长度除以瓷砖尺寸的上限。 从数学上来说，这是`ceil(n / a)`。 对两个维度执行此操作并将结果相乘即可得出图块的总数。 向上舍入可确保即使存在少量余数，我们也会分配额外的图块来覆盖它。 

可以使用整数除法来计算上限运算，而无需浮点运算：`(n + a - 1) // a`。 这是有效的，因为如果`n`不能被整除`a`，添加`a-1`确保商向上舍入。 如果`n`可以整除`a`，相加不会改变商。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| --- | --- | --- |
 | 蛮力 | O(n * m / a^2) | O(n * m / a^2) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取输入值`n`,`m`， 和`a`。 这些代表长度、宽度和瓷砖尺寸。 
2. 使用整数算术计算沿长度方向的瓷砖数量：`tiles_along_length = (n + a - 1) // a`。 添加`a - 1`确保四舍五入如果`n`不能被整除`a`。 
3. 以同样的方式计算沿宽度方向的瓷砖数量：`tiles_along_width = (m + a - 1) // a`。 
4. 将两个结果相乘即可得到图块总数：`total_tiles = tiles_along_length * tiles_along_width`。 
5. 打印结果。 

工作原理：每个维度都是独立覆盖的，使用最少数量的图块来覆盖整个长度。 乘法确保矩形区域的二维覆盖。 即使在以下情况下，舍入步骤也保证没有未覆盖的空间：`n`或者`m`不是的倍数`a`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m, a = map(int, input().split())

tiles_along_length = (n + a - 1) // a
tiles_along_width = (m + a - 1) // a

total_tiles = tiles_along_length * tiles_along_width
print(total_tiles)
```代码直接遵循算法。 读取输入`sys.stdin.readline`避免大输入的 I/O 开销。 这`(n + a - 1) // a`技巧避免浮点除法，防止精度问题和溢出。 将两个计数相乘即可得出正确的石板总数。

 ## 边缘情况

 对于本案`n = 5, m = 5, a = 3`，计算为`(5 + 3 - 1) // 3 = 7 // 3 = 2`每个维度的瓷砖。 相乘得到 4 个方块。 尽管剩余 2 米，每个维度都被完全覆盖。 不向上舍入的简单整数除法将返回 1×1 = 1 个图块，这显然是不够的。 这证实了舍入逻辑正确处理不可整除的维度。 

对于最大值`n = m = 10^9, a = 1`，计算`(10^9 + 1 - 1) // 1 = 10^9`每个维度，给出`10^9 × 10^9 = 10^18`瓷砖。 Python 处理这个大整数时不会溢出，从而验证了极端输入的方法。
