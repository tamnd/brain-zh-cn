 ## 解决方案
 ## 问题理解

 我们得到一个有理数的集合，每个有理数表示为一对整数、一个分子和一个分母。 任务是按数值从小到大对这些分数进行排序，并按排序顺序输出它们，同时保留其原始表示形式。 

每个分数代表分子除以分母得到的实数值。 关键的困难在于我们没有给出浮点数，因此我们不能依赖近似比较。 相反，我们必须精确地比较分数。 

此处未明确提供约束，但此类问题通常涉及最多 10^5 个分数，分子和分母中可能存在较大的整数值。 这立即排除了任何转换为​​浮点的方法，因为精度错误会改变接近分数的排序，并且还表明每对的比较必须是恒定时间，否则 O(n^2) 方法将无法生存。 

当分母为负时，会出现微妙的边缘情况。 例如，如果符号未标准化，比较 1/-2 和 -1/2 很容易导致结果不一致。 另一个边缘情况是分子为零：像 0/5、-3/7 和 2/1 这样的分数仍然必须正确排序。 简单的基于字符串的排序或对的字典排序在这里完全失败，因为如果逐位比较，(1, 10) 将被错误地认为小于 (2, 3)。 

## 方法

 蛮力的想法很简单：计算每个分数的实际值并按其排序。 这意味着将每对 (a, b) 转换为浮点数 a / b 并将其用作排序键。 乍一看，正确性似乎很明显，因为实数在全序中是可比较的。 

然而，这种方法有两个问题。 首先，浮点运算会引入精度误差，尤其是当分子和分母很大时。 两个不同的分数可以折叠为相同的浮动近似值，或者它们的顺序可以由于舍入而翻转。 其次，即使我们使用精确有理算术库，与简单的整数比较相比，重复除法或标准化也会不必要地昂贵。 

关键的观察是比较两个分数 a/b 和 c/d 不需要计算它们的实际值。 相反，我们可以比较 a * d 和 c * b。 这是可行的，因为两个分母都被假定为非零，并且交叉乘法保留了排序而无需除法。 这减少了与整数乘法的每次比较，Python 使用任意精度整数安全地处理整数乘法。 

这将问题转换为使用自定义比较器或键的标准排序任务，其中每个分数通过叉积间接按值 a/b 排序。

 | 蛮力（浮点）| O(n log n) 但不可靠 | O(n) | 由于精度不正确 |
 | 最优（叉乘）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1. 将所有分数作为对 (a, b) 读取，将它们放在一起，以便在排序后我们可以将它们原封不动地输出。 必须保留原始表示形式，因为仅更改顺序，而不更改格式。 
2. 对每个分数进行标准化，以便比较表现一致。 如果分母为负，则将分子和分母都乘以 -1，使分母变为正。 这可以防止比较期间出现符号不一致。 
3. 使用代表分数值的键对分数列表进行排序。 我们依靠 Python 使用元组进行排序并定义比较代理的能力，而不是计算 a/b。 一种安全的方法是按 (a * 1.0 / b) 排序，但正确的方法是完全避免浮点，而是使用等效的比较器，或者在 Python 中，使用带有交叉乘法逻辑的 functools.cmp_to_key 进行排序。 
4. 定义两个分数 (a, b) 和 (c, d) 之间的排序规则：如果 a * d < c * b，则第一个较小。 这种比较是传递的且一致的，这保证了标准排序产生有效的全序。 
5. 使用原始存储的对按排序顺序输出分数。 

### 为什么它有效

 在整个排序过程中保持的不变性是每次比较都反映了分数表示的实数的精确顺序。 交叉乘法保留了顺序，因为将 a/b < c/d 两边乘以正 bd 会产生 a * d < c * b 而不改变不等式方向。 由于我们将分母标准化为正数，因此我们永远不会错误地翻转不等式符号。 这保证了所有比较都是一致的，因此排序算法会产生全局正确的排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from functools import cmp_to_key

def cmp(frac1, frac2):
    a, b = frac1
    c, d = frac2

    # ensure consistent sign handling
    if b < 0:
        a, b = -a, -b
    if d < 0:
        c, d = -c, -d

    left = a * d
    right = c * b

    if left < right:
        return -1
    if left > right:
        return 1
    return 0

def main():
    n = int(input())
    fracs = []
    for _ in range(n):
        a, b = map(int, input().split())
        fracs.append((a, b))

    fracs.sort(key=cmp_to_key(cmp))

    for a, b in fracs:
        print(a, b)

if __name__ == "__main__":
    main()
```该解决方案以定制比较器为中心。 最微妙的部分是确保分母符号的处理一致； 如果没有这个，像 1/-2 和 -1/2 这样的比较将表现得不一致，即使它们代表相同的值。 其余代码是标准输入解析和排序。

 ## 边缘情况

 一个关键的边缘情况是分母为负数。 例如，输入对 (1, -2) 表示 -1/2，如果没有标准化，比较可能会在某些实现中错误地将其视为正值。 该算法显式翻转符号，因此在比较之前每个分数都用正分母表示。 

另一种边缘情况是具有不同表示形式的等分数，例如 (1, 2)、(2, 4) 和 (3, 6)。 交叉乘法比较在所有成对比较中产生相等，因此排序将它们视为相等并保留它们之间的有效顺序。 

零分子也需要注意。 无论分母大小如何，像 (0, 5) 这样的分数必须正确地比较为零。 交叉乘法可以自然地处理这个问题，因为 0 * d 始终为零，确保相对于正分数和负分数的正确放置。
