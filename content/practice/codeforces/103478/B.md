 ## 解决方案
 ## 问题理解

 我们得到一个大写字符串，代表一个由字母 A 到 Z 组成的“单词”。任务是确定这个单词是否可以完全分割成化学元素符号序列，但仅使用受限集合：语句中提供的周期表的前 20 个元素。 

每个元素符号的作用就像一个图块。 有些图块只有一个字母长，例如`B`,`C`,`O`，而其他的则是两个字母长，例如`HE`,`NA`， 或者`CL`。 目标是检查是否可以通过按顺序连接这些符号来精确覆盖整个字符串，而不会跳过或重叠字符。 

这是一个经典的字符串分段问题，其中每个分段必须属于长度为 1 或 2 的字符串的固定字典。 

输入长度最多为 100。这足够小，即使是位置上的二次动态规划也能在 1 秒限制下轻松足够快，因为我们每次 DP 转换只执行恒定量的工作。 

一个微妙的问题是，某些符号是单个字母，与较长符号的前缀严重重叠。 例如，`C`,`CA`， 和`CL`都存在。 总是偏向于一个匹配的贪婪方法（例如总是先进行一个字母匹配或总是偏向于两个字母匹配）可能会失败。 

例如，考虑一个假设的字符串，例如`CA...`。 如果我们贪婪地采取`C`首先，我们稍后可能会阻止需要的有效分解`CA`在那个位置。 这表明本地选择并不安全。 

另一种边缘情况是存在多个分段但只有一个分段导致有效的完整覆盖。 例如，`NEON`可以分为`NE O N`或者`N E O N`取决于符号的可用性，并且只有某些选择是有效的。 

因此，我们需要一种能够有效探索所有有效分割的方法。 

## 方法

 暴力解决方案尝试一切可能的方法将字符串拆分为有效的元素符号。 在每个位置，我们尝试匹配一个字母或两个字母的符号并递归地继续。 这形成了一个递归树，其中每个位置最多分为两个选择。 

在最坏的情况下，每个前缀都是有效的，因此递归路径的数量会像斐波那契式分支一样增长，大约为 O(2^n)。 对于n达到100，这是完全不可行的。 

关键的观察结果是问题的状态仅取决于字符串中的当前位置。 一旦我们到达索引 i，从 i 开始的后缀就可以独立于我们如何到达那里而被求解。 这是标准的最优子结构属性。 

这表明对仓位的动态规划。 让 dp[i] 表示从位置 i 开始的子串是否可以使用允许的符号完全分段。 然后从位置 i 开始，我们最多尝试两次转换：采用一个字母符号或两个字母符号（如果有效）。 每个状态计算一次，给出线性复杂度。 

我们可以从字符串末尾向后计算 dp，或者使用记忆递归。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力递归 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | DP 后缀位置 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将允许的元素符号预处理为一组，以进行常量时间成员资格检查。 由于只有 20 个符号，因此开销可以忽略不计。 

1. 定义一个布尔型 DP 数组，其中 dp[i] 表示从索引 i 开始的子串是否可以完全分段。 我们还设置 dp[n] = True 因为空后缀始终有效。 
2. 从 n−1 到 0 迭代 i。在每个位置，我们尝试匹配从 i 开始的有效符号。 
3. 首先，我们检查单字符子串 s[i]。 如果在允许集合内，则dp[i]可以继承dp[i+1]。 这对应于消耗一个元素符号。 
4. 接下来，如果 i+1 < n，我们检查两个字符的子串 s[i:i+2]。 如果在允许集合内，则dp[i]可以继承dp[i+2]。 这对应于消耗两个字母的元素符号。 
5. 如果任一选项导致有效完成，我们设置 dp[i] = True。 
6.填表后，答案是dp[0]，它告诉我们整个字符串是否可以分段。 

我们只检查一个和两个字母切片的原因是该问题将所有符号的长度限制为最多两个。 

### 为什么它有效

 DP 维护了 dp[i] 正确表示从 i 开始的后缀是否可以被有效元素符号完全覆盖的不变量。 每个转换完全对应于选择一个与后缀的当前前缀匹配的有效符号，并且剩余的后缀通过 dp 独立检查。 由于考虑了所有可能的有效第一选择，因此不会错过任何有效分段，并且由于仅使用有效的字典匹配，因此不会引入无效分段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

symbols = {
    "H","HE","LI","BE","B","C","N","O","F","NE",
    "NA","MG","AL","SI","P","S","CL","AR","K","CA"
}

def solve():
    s = input().strip()
    n = len(s)

    dp = [False] * (n + 1)
    dp[n] = True

    for i in range(n - 1, -1, -1):
        if s[i:i+1] in symbols and dp[i + 1]:
            dp[i] = True
            continue
        if i + 1 < n and s[i:i+2] in symbols and dp[i + 2]:
            dp[i] = True

    print("YES" if dp[0] else "NO")

if __name__ == "__main__":
    solve()
```该代码直接反映了 DP 定义。 套装`symbols`对允许的元素符号的字典进行编码。 DP 数组的大小`n+1`自然地处理空后缀的基本情况。 

检查的顺序与正确性无关，但早期`continue`当单字母匹配已经成功时，会稍微减少工作量。 两个转换始终针对边界进行测试，以避免字符串末尾的无效切片。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | 每个位置最多处理一次，并进行两次常量时间字典检查 |
 | 空间| O(n) | 大小为 n+1 的 DP 数组 |

 约束允许的最大长度为 100，因此线性 DP 远低于限制。 即使更简单的 O(n^2) 方法也是安全的，但 O(n) 可以保持解决方案的干净和稳健。

 ## 边缘情况

 一种关键的边缘情况是当一个字母和两个字母符号在同一位置都有效，但只有一个可以成功完成时。 例如，在`CAK`，在位置 0 处均`C`和`CA`是有效的。 选择`C`导致`AK`，在选择时失败了`CA`导致`K`，成功了。 DP 正确评估了两种可能性，因为 dp[0] 取决于 dp[1] 或 dp[2]，而不是贪婪选择。 

另一种边缘情况是完全由单字母符号组成的字符串，例如`BCPC`。 该算法自然会处理这个问题，因为它从不需要两个字母匹配。 

最后，以有效前缀结尾但留下不匹配后缀的字符串，例如`CLX`，由于 dp 无法达到最终状态 dp[n] = True，因此在最后一步失败。
