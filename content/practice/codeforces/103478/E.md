 ## 解决方案
 ## 问题理解

 我们被赋予了一个固定的词`s`和另一个字符串`t`。 一名学生反复尝试打字`s`，但是他的打字过程并不是以干净的方式连续的。 每次尝试都是这样进行的：他开始打字`s`从头开始，如果他忘记了下一个字符，并且如果他到达了结尾，可能会提前停止`s`，他还是立即从头开始，强化记忆。 他甚至可以在完成之前随时重新开始。 

经过所有尝试后，我们将他在所有尝试中设法输入的每个字符连接起来，生成最终的字符串`t`。 任务是确定是否存在某种可以准确生成的部分或全部尝试序列`t`。 

重申这一点的一个有用方法是，我们正在尝试模拟是否`t`可以分成段，其中每个段是一个前缀`s`，最后一段也可以是任何前缀，但每个段必须从位置开始`0`的`s`。 

这些约束允许所有测试用例的总长度达到 5×10^5。 这排除了对所有可能的分割或回溯的任何二次模拟。 任何尝试所有重新启动位置或构建具有朴素转换的自动机的方法都将 TLE。 我们需要线性或近线性扫描`t`每个测试用例。 

当贪婪匹配时会出现微妙的边缘情况`t`反对`s`尝试不跟踪重置结构。 例如，如果`s = "apple"`和`t = "apaple"`，它可能看起来局部合理，但没有办法分裂`t`到有效的前缀`s`总是从头开始。 

另一个棘手的情况是重叠表明重启过早。 例如，重复的部分前缀可能会模仿进度，但如果发生不匹配，我们必须确保重新启动`s`不需要跳过已经消耗的字符`t`。 

## 方法

 暴力解释很简单：我们尝试分区`t`分成段，每个段匹配一个前缀`s`。 对于每个位置`t`，如果字符匹配，我们可以扩展当前段`s[i]`，或剪切并重新启动。 这导致在最坏的情况下探索指数级的许多分割选择，因为在每个不匹配点，我们可能选择重新启动或重新考虑早期的切割。 即使我们稍微修剪一下，最坏情况的行为也会退化为尝试所有划分点`t`，其结构为 O(2^|t|)，或者对于所有先前的切割位置进行动态规划，至少为 O(|t|^2)。 

关键的见解是我们永远不需要同时考虑多个可能的段开始。 每个段完全通过扫描确定`s`从索引 0 开始，唯一有意义的状态是进入多远`s`我们目前正在匹配`t`。 如果发生不匹配，唯一有效的操作是在位置 0 重新开始新的尝试`s`并重试当前字符`t`。 

这减少了模拟单个指针的问题`s`扫描时`t`，发生不匹配时自动复位。 的每个字符`t`被处理一次，并且指针在`s`除非重置为 0，否则绝不会向后移动。

 ## 算法演练

 我们维护一个指针`j`这代表我们在里面有多远`s`对于当前的尝试。 

1. 初始化`j = 0`。 这意味着我们正在开始新的匹配尝试`s`从一开始。 
2. 迭代每个字符`c`在`t`从左到右。 每个角色都必须属于某个尝试，因此我们尝试将其与`s[j]`。 
3.如果`c == s[j]`，我们前进`j`1. 这将继续当前的尝试。 
4. 如果`j`变得等于`len(s)`，这意味着我们已经成功完成了一次完整的打字尝试`s`。 根据流程，学生立即从头开始，所以我们重置`j = 0`。 
5. 如果在任何时候`c != s[j]`，我们模拟忘记下一个字符并重新开始尝试。 我们重置`j = 0`并尝试匹配`c`再次作为新尝试的第一个角色。 
6. 重置后，如果`c != s[0]`，那么即使是新的尝试也无法从这个角色开始，因此构建是不可能的，我们返回“否”。 
7. 如果我们设法处理所有字符`t`成功，我们返回“Yes”。 

关键的想法是每个角色`t`必须解释为某些前缀的一部分`s`，唯一的自由是我们重新启动前缀的位置。 贪婪重启确保我们永远不会提交无效的部分对齐。 

### 为什么它有效

 在任何位置`t`，算法保持不变式`j`等于前缀的长度`s`与最近的尝试后缀匹配`t`。 每当发生不匹配时，任何有效的解释都必须放弃当前的尝试，因为继续会违反前缀结构。 从索引 0 重新启动是唯一可能的恢复，因为每次尝试都必须从索引 0 的开头开始`s`。 这意味着任何有效的分割`t`准确地对应于某些重置序列，并且贪婪策略模拟最早可能的重置以保持一致性。 

因为每个字符要么匹配，要么强制重置，并且重置总是将我们移动到唯一合法的开始状态，因此不会跳过任何有效的构造，也不接受无效的构造。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    t = input().strip()
    
    n = len(s)
    j = 0
    
    for c in t:
        if j < n and c == s[j]:
            j += 1
            if j == n:
                j = 0
        else:
            j = 0
            if s[0] != c:
                return "No"
            j = 1
    
    return "Yes"

def main():
    T = int(input())
    out = []
    for _ in range(T):
        out.append(solve())
    print("\n".join(out))

if __name__ == "__main__":
    main()
```实现直接遵循模拟。 变量`j`将当前进度编码在里面`s`。 第一个分支处理单词的正常匹配和完全完成，而第二个分支处理强制重新启动。 关键的微妙之处在于，重置后，我们必须立即验证当前角色是否可以开始新的尝试； 否则我们会错误地认为我们总是可以重新启动。 

复位条件`j == n`确保已完成的尝试不会错误地延续状态。 每次完成都会立即返回到开始处`s`，匹配问题的强化规则。

 ## 边缘情况

 一种边缘情况是当`t`包含该位置不存在的字符`0`的`s`。 例如`s = "abc"`和`t = "abz"`。 该算法达到失配`z`，重置，并立即检查`s[0] != 'z'`，产生“否”。 任何正确的解决方案都必须拒绝此类字符，因为任何尝试都不能合法地以它们开始。 

另一种情况是当`t`强制在同一位置重复重置。 为了`s = "ab"`和`t = "abababx"`，算法重复匹配并干净地重置，直到最后一个字符`x`，在开始条件下失败。 这表明，局部的重复成功并不能保证全球的可行性。 

最后一个微妙的情况是立即重启循环，例如`s = "a"`和`t = "aaaaa"`。 每个角色都完成一次完整的尝试，每次都会触发重置为零。 该算法正确处理了这个问题，因为完成条件重置`j`在下一次比较之前，确保两次尝试之间不存在陈旧状态。
