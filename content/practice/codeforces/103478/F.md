 ## 解决方案
 ## 问题理解

 我们给出了长度的排列$2n$，这意味着它包含来自的每个整数$1$到$2n$正好一次。 我们可以执行相邻位置的交换，交换位置处的值$i$和$i+1$。 

目标条件不是全局排序。 相反，当每对连续的位置都满足局部顺序约束时，该数组被认为是有效的：$i$不得超过位置$i+1$对于最终结构中重要的每一对。 从样本行为来看，此约束应用于大小为 2 的固定块，因此每对$(1,2), (3,4), \dots, (2n-1,2n)$最终必须内部不减。 不需要比较不同对之间的值。 

任务是计算达到满足每对条件的任何配置所需的最小相邻交换数量。 

限制条件$n \le 5 \cdot 10^4$意味着数组大小可达$10^5$。 任何超出线性或近线性时间的解决方案都会很紧张，但仍然可行，而任何二次的解决方案都会太慢。 然而，关键的观察将问题简化为每对的恒定时间决策，从而使解决方案有效$O(n)$。 

一个微妙的陷阱来自于将条件误解为完全排序。 如果我们错误地假设全局顺序，我们将计算反转计数。 例如，对于示例输入：```
2 4 6 3 5 1
```全局反转计数远大于 2，但正确答案是 2，因为只有局部对排序很重要。 

另一个可能的错误是认为货币对之间的交换是必要的。 事实上，跨货币对互换只会带来不必要的干扰，因为最终的约束完全是局部的。 

## 方法

 天真的解释是将问题视为使用相邻交换的排序任务。 从这种观点来看，每次交换都恰好删除了一次反转，因此答案将是排列的总反转计数。 仅当目标完全排序时，这才是正确的。 可以计算为$O(n \log n)$使用 Fenwick 树或合并排序。 

这种方法的失败来自于目标条件的结构。 由于我们不需要全局顺序，因此大多数倒置都是无关紧要的。 只有每个固定对内部的违规行为才重要，并且这些违规行为可以独立解决。 

关键的见解是每对$(2k-1, 2k)$是独立的。 如果左边的元素已经较小，则不需要任何东西。 如果它更大，则该对中的一个相邻交换将修复它，并且不需要额外的交换来满足该对的约束。 这消除了阵列不同部分之间的任何相互作用。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 全局反转计数|$O(n \log n)$|$O(n)$| 型号错误 |
 | 成对局部校正|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们以两个位置的块来处理数组。 

1. 从左到右扫描数组，每次扫描一次，处理每一个$(2k-1, 2k)$作为一个整体。 
2. 对于每一对，比较两个值。 
3. 如果左值大于右值，我们算一次交换，概念上交换它们。 
4. 如果该对已经被订购，我们什么都不做。 
5. 对所有货币对的所有交换求和并输出结果。 

每个决策都是局部的，因为没有约束连接不同的对。 唯一重要的操作是一对是否是内部有序的。 

### 为什么它有效

 最终条件仅取决于固定不相交对内元素的相对顺序。 任何修复一对的交换都不需要将更改传播到其他地方，因为没有其他约束引用该位置关系。 因此，每一对都独立地影响总成本，并且每个反向对仅需要一次交换，而非反向对则不需要任何交换。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    ans = 0
    for i in range(0, 2 * n, 2):
        if a[i] > a[i + 1]:
            ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了pair scan。 循环前进 2 以与固定块对齐。 唯一的操作是对每对进行比较，确定该对是否贡献一次交换。 

一个常见的错误是尝试显式模拟交换。 这是不必要的，因为我们永远不需要中间配置； 我们只计算有多少对最初是错误的。

 ## 边缘情况

 关键的边缘情况是所有元素都已在全局正确排序。 在这种情况下，每对也是正确的，因此答案为零，并且算法仅执行比较。 

另一种情况是每对都颠倒过来。 对于这样的输入：```
2
2 1 4 3
```每一对恰好贡献一次交换，并且算法正确地计算了两次独立的更正。 

最终的结构边缘情况是本地排序交替时，例如：```
3
1 3 5 2 4 6
```只有一对是不正确的，修复它不会与其他对相互作用，从而确认对独立性完全表征了解决方案。
