 ## 解决方案
 ## 问题理解

 我们在一个无限平面上画了几个圆。 这些圆圈仅以一种非常结构化的方式相互作用：任何两个圆圈要么完全独立，要么完全包含另一个圆圈，要么只是接触。 不存在会产生复杂交叉几何形状的部分重叠。 由于这种限制，这些圆圈形成了一个干净的嵌套结构，就像一个遏制树的森林。 

平面上的每个点都被圆的某个子集覆盖。 如果一个点被奇数个圆圈覆盖，则该点被视为“受影响”，而偶数个圆圈则完全抵消。 因此，每个圆圈都会为其内部的区域提供一个切换，而最终受影响的区域纯粹由覆盖奇偶性决定。 

我们可以对圆执行一种称为反转的操作。 应用它会翻转该圆内每个点的受影响/不受影响状态。 每次反转花费一个单位，我们希望使用尽可能少的反转。 

目标是确保受影响点的总面积不超过kπ。 由于每个圆面积都是 π 的倍数，因此我们可以等效地将所有面积标准化为 π 并以半径平方的形式进行思考。 

约束 n ≤ 2000 意味着二次或 n² log n 解是合理的，但任何涉及指数子集或大值的重型背包的速度都太慢。 k 的限制为 10^17 也意味着我们不能直接依赖 DP 面积值。 

一种简单的方法是尝试直接对平面进行建模，跟踪由重叠形成的所有区域并在每次操作时切换它们。 即使有包含限制，显式维护区域仍然太大，因为每个圆可以包含许多嵌套圆，并且每次翻转后重新计算受影响的区域会太慢。 

第二个天真的想法是独立对待每个圆圈，根据它当前对总面积的贡献是正还是负来决定是否翻转它。 这会失败，因为翻转一个圆会改变其所有后代的奇偶校验结构，因此决策不是独立的。 

一个典型的隐藏失败案例来自于嵌套：

 输入：

 3 1

 0 0 10

 0 0 5

 0 0 1

 这里一切都是嵌套的。 贪婪的方法可能会首先翻转最小的圆圈来修复局部平价，但这会改变所有更高级别的贡献，并可能增加其他地方的总受影响面积，导致全局结果不理想。 

核心困难在于每个操作都会影响嵌套圆的整个子树，因此我们需要一个树DP来跟踪奇偶校验如何向下传播。 

## 方法

 因为圆要么是嵌套的，要么是不相交的，所以我们可以构建一个森林，其中每个圆都有一个唯一的父对象：包含它的最小圆。 

一旦我们将结构视为一棵树，每个圆对应一个节点，节点内部但其子节点外部的每个点都对应于一个不同的区域，该区域的面积很容易计算。 具体来说，节点的独占区域是其全部区域减去其直接子节点的区域。 

区域的初始奇偶校验取决于它在包含树中的深度。 每深入一层，覆盖范围就会翻转，因此初始奇偶校验仅取决于深度奇偶校验。 

现在考虑一下反转的作用。 翻转节点会切换其子树中每个区域的奇偶校验。 这意味着它不仅影响其自己的地区，而且影响所有后代，从而沿路径产生全球依赖性。 

这导致了标准的树 DP 结构，其中每个节点都有一个状态，表示其奇偶校验是否已被祖先翻转。 在每个节点，我们决定是否翻转它，这会影响它自己的贡献和传递给子节点的状态。

暴力解释是尝试翻转节点的所有子集。 那就是2ⁿ的可能性，完全不可行。 

相反，我们计算树上的 DP，其中每个节点返回成本（翻转次数）与其子树内产生的总受影响区域之间的最佳可实现权衡，具体取决于它是否带有来自其父级的翻转奇偶校验或未翻转奇偶校验。 

每个子树 DP 状态都成为帕累托最优对的集合，而不是单个值，因为我们必须跟踪成本和结果区域。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集的暴力破解 | O(2ⁿ·n) | O(2ⁿ·n) | O(n) | 太慢了|
 | 具有帕累托状态的树DP | O(n²) | O(n²) | 已接受 |

 ## 算法演练

 我们首先构建遏制森林。 对于每个圆，我们通过检查哪个其他圆以最小半径包含它来识别其父圆。 由于 n ≤ 2000，O(n²) 检查就足够了。 

接下来，我们计算每个节点的子列表。 这给了我们一个有根森林，其中每个节点对应一个圆，边代表包含。 

然后我们计算几何权重。 对于每个节点，其总面积为 r²。 它的专属区域面积是它自己的面积减去它的子区域的总和。 这是可行的，因为子级除了通过包含之外不会重叠，因此它们的独占区域划分了父级的内部。 

我们还计算每个节点区域的初始奇偶校验。 如果节点位于深度 d，则其区域恰好被覆盖 d 次，因此如果 d 为奇数，则最初会受到影响。 

现在我们在树上运行一个 DP。 

在每个节点u，我们定义两个DP表：

 DP[u][p]，其中 p 表示 u 的区域是否被其父奇偶校验（0 或 1）翻转。 每个 DP 状态存储一组对（成本、面积），表示子树中可实现的配置。 

对于每个节点，我们尝试两种选择：

 我们要么不翻转它，要么翻转它。 每个选择的成本为 0 或 1，并改变将奇偶性传递给子项的方式。 

当组合子树时，我们通过添加成本和添加区域来合并它们的 DP 状态，因为一旦奇偶校验固定，子树就是独立的。 

处理完所有子节点后，我们修剪主导状态。 如果 c1 ≤ c2 且 a1 ≤ a2，则状态 (c1, a1) 支配 (c2, a2)，因为它永远不会更糟。 

最后，在根部，我们查看所有 DP 状态，并在总面积 ≤ k 的状态中选择最小成本。 

## 为什么它有效

 关键的不变量是，对于每个节点，DP[u][p] 表示翻转成本和以 u 为根的子树中可实现的受影响区域之间的所有最佳权衡，假设 u 经历了来自其祖先的奇偶校验 p。 

一旦传入奇偶校验固定，每个子树都是独立的，因为翻转只影响后代。 这确保了组合子项是纯粹的加法合并，没有隐藏的交互。 

由于每种可能的翻转配置都恰好对应于在该 DP 中分配选择的一种方式，并且每个 DP 转换都保留所有可行的组合，因此不会排除任何有效的解决方案。 剪枝只能删除严格更差的状态，因此它不能消除潜在的最优全局解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n, k = map(int, input().split())
    circles = []
    for _ in range(n):
        x, y, r = map(int, input().split())
        circles.append((r, x, y))

    circles.sort(reverse=True)

    # build containment tree
    parent = [-1] * n
    children = [[] for _ in range(n)]

    def contains(i, j):
        ri, xi, yi = circles[i]
        rj, xj, yj = circles[j]
        dx = xi - xj
        dy = yi - yj
        return dx*dx + dy*dy <= (ri - rj) * (ri - rj)

    for i in range(n):
        for j in range(i):
            if parent[i] == -1 and contains(j, i):
                parent[i] = j
                children[j].append(i)

    roots = [i for i in range(n) if parent[i] == -1]

    # compute depth
    depth = [0] * n
    stack = [(r, 0) for r in roots]
    while stack:
        u, d = stack.pop()
        depth[u] = d
        for v in children[u]:
            stack.append((v, d + 1))

    area = [0] * n
    for i, (r, x, y) in enumerate(circles):
        area[i] = r * r

    # compute exclusive area
    for u in range(n - 1, -1, -1):
        for v in children[u]:
            area[u] -= circles[v][0] ** 2

    def merge(a, b):
        res = {}
        for ca, aa in a:
            for cb, ab in b:
                c = ca + cb
                w = aa + ab
                if c not in res or res[c] < w:
                    res[c] = w
        items = list(res.items())
        items.sort()
        pruned = []
        best = -1
        for c, w in items:
            if w > best:
                pruned.append((c, w))
                best = w
        return pruned

    def dfs(u, p):
        init = depth[u] & 1

        best = {}
        # x = 0 or 1
        for flip in [0, 1]:
            cost = flip
            cur_parity = p ^ flip
            val = 0
            if init ^ cur_parity:
                val = area[u]

            dp_list = [(0, val)]
            for v in children[u]:
                child = dfs(v, cur_parity)
                dp_list = merge(dp_list, child)

            for c, w in dp_list:
                nc = c + cost
                if nc not in best or best[nc] < w:
                    best[nc] = w

        return [(c, w) for c, w in best.items()]

    INF = 10**18
    ans = INF

    for r in roots:
        for c, w in dfs(r, 0):
            if w <= k:
                ans = min(ans, c)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对圆圈进行排序，以便可以通过首先检查较大的圆圈来检测遏制。 构建一个父数组，将每个圆分配给最小的包围圆，形成一个森林。 

计算深度以确定每个区域的初始奇偶性，因为覆盖范围仅取决于嵌套深度。 

DP 的实施于`dfs`。 每个节点都会尝试两种翻转选择。 对于每个选择，它都会计算节点的区域是否对最终受影响的区域有贡献，然后在相同的奇偶校验上下文下合并所有子 DP 状态。 合并功能仅保留非支配成本区域对以控制增长。 

最后，扫描所有根结果以找到实现面积约束的最小翻转次数。 

## 工作示例

 ### 示例 1

 输入：```
3 3
0 0 2
0 0 1
10 10 1
```我们构建一棵树，其中圆 1 包含圆 2，而圆 3 是独立的。 

| 节点| 深度| 初始化奇偶校验| 面积（r² 减去子项）|
 | --- | --- | --- | --- |
 | 1 | 0 | 0 | 4 - 1 = 3 | 4 - 1 = 3
 | 2 | 1 | 1 | 1 |
 | 3 | 0 | 0 | 1 |

 初始受影响面积为 3 + 1 = 4，超过 k = 3。 

DP 探索翻转节点 1，这会翻转其整个子树，从而减少 1 和 2 的贡献。这导致总受影响面积变为 2 的配置，只需一次翻转即可实现。 

所以答案是1。 

### 示例 2

 输入：```
1 0
0 0 5
```只有一个圆圈存在。 其面积为25，初步受到影响。 

| 节点| 深度| 初始化奇偶校验| 面积 |
 | --- | --- | --- | --- |
 | 1 | 0 | 0 | 25 | 25

 由于 k = 0，我们必须消除所有受影响的区域。 轻轻一按即可将整个区域切换为非活动状态。 

答案是1。 

这些例子表明，DP 不是关于局部修复，而是关于选择子树范围内的奇偶校验更改来全局重塑区域贡献。

 ## 边缘情况

 一种关键的边缘情况是完全嵌套的链，其中每个圆恰好包含另一个圆。 在这种情况下，每次翻转都会传播到所有剩余节点。 DP 正确地处理了这个问题，因为每个节点的状态都包含从所有祖先继承的奇偶校验，确保翻转高级节点被评估为全局转换而不是局部调整。 

另一个边缘情况是多个不相交的树。 由于每个根都是独立处理的，因此算法会单独处理它们，并且仅在最终选择阶段组合 DP 结果。 这确保了断开连接的组件之间不会发生意外的交互。 

最后一个微妙的情况是所有圆都相同。 在这种情况下，由于打破平局，遏制仍然形成一条链，并且 DP 仍然表现正确，因为内部边界的面积贡献变为零，只留下基于奇偶性的决策来确定最终受影响的区域。
