 ## 解决方案
 ## 问题理解

 我们有一组学生，每个学生都有一个整数“身高等级”，其中相同的值意味着相同的身高。 任务是将所有学生排成长度相等的两排。 如果有$n$学生，两行都包含$n/2$位置从左到右索引。 

每行从左到右的高度必须不降低。 此外，如果我们逐列查看两行，第二行中的学生的身高必须至少与第一行中位于他们正上方的学生一样高。 因此，每一列都形成一个弱递增的垂直对，并且每一行本身都在水平方向上弱递增。 

我们被要求计算存在多少个这样的有效安排，如果任何学生最终处于不同的位置，则将两个安排视为不同的安排。 答案必须以 998244353 为模计算。 

输入并不直接给出高度，而是按排序顺序给出排名，包括重复项。 这意味着我们正在有效地处理多重集，其中相等元素的相对顺序并不重要，但多重性却很重要。 

约束条件$n \le 5000$意味着解大约可达$O(n^2)$或者$O(n^2 \log n)$是可以接受的，但是阶乘枚举或指数状态爆炸之类的任何事情都是不可能的。 我们应该期望在前缀状态上进行动态编程或组合构造。 

一个关键的微妙之处是正确处理重复项。 如果多个学生具有相同的身高，则根据位置身份，跨行或行内交换他们可能会也可能不会产生不同的配置。 如果不小心将相同的值视为无法区分，这种幼稚的方法可能会低估或高估。 

小边缘情况是所有高度都相等。 那么任何有效的排列纯粹取决于行的结构约束，并且计数减少到计数单调 2 行填充，这在组合上仍然是不平凡的。 另一种边缘情况是当高度严格增加时，约束变得更加严格，并且基本上强制采用单一排序结构。 

## 方法

 直接的暴力解释是将每个$n$将学生分成两行之一，然后排列每一行并检查有效性。 即使我们忽略排列并只尝试赋值，也有$\binom{n}{n/2}$选择顶行的方法，对于每个选择，在行内排列会增加阶乘因子。 即使对于$n=20$。 核心问题是，有效性不仅取决于分区，还取决于每行内部引入的排序结构，因此朴素枚举浪费了探索等效排列的精力。 

结构观察是，一旦我们确定了哪些元素进入每行，两行必须独立地以非降序排序。 这完全消除了内部排列。 所以问题简化为选择大小的子集$n/2$对于第一行，在对两个子集进行排序后，按列条件成立：对于每个位置$i$,$top[i] \le bottom[i]$。 

现在的关键思想是按升序处理值并增量构建两行。 由于元素除了其值之外是无法区分的，因此唯一有意义的状态是每个值的任意值前缀处有多少元素被分配给第一行和第二行。 这导致了频率上的动态规划公式。 

让$cnt[v]$是值出现的次数$v$。 我们按升序处理值，并在每一步分配出现的次数$v$分成两组：一些进入顶行，其余的进入底行。 两行均已排序且按列有效的约束转化为一个简单的可行性条件：填充值时$v$，我们必须确保在每个前缀处，分配给顶行的元素数量永远不会超过$n/2$，对于底行也是如此。 排序约束会自动保留，因为所有较早的值都较小或相等，因此值块内的任何分配都会保留单调性。 

剩下的问题是计算所有值的分布，同时确保两行完全以$n/2$元素。 这变成了一个类似背包的 DP，我们在其中跟踪到目前为止已分配给顶行的元素数量。 

我们将 dp[i][j] 定义为处理前 i 个不同值后的路数，其中 j 个元素放置在顶行。 对于频率为 f 的每个值 v，我们尝试从 0 到 f 的所有分割 k：k 个元素到顶部，f-k 到底部。 然后我们从 j 过渡到 j+k。 

复杂度是$O(m \cdot n \cdot f)$在最坏的形式中，但由于频率的总和为 n，因此总的转换可以优化为$O(n^2)$，穿着舒适。 

蛮力之所以有效，是因为它明确地探索了分配，但由于阶乘冗余而失败。 DP 之所以有效，是因为每个值唯一有意义的决策是每行有多少个副本，并且所有排列都分解为组合二项式选择。

 ## 算法演练

 1. 按升序将输入压缩为每个不同高度值的频率。 这是必要的，因为只有多重性才重要，而不是相同值内的同一性。 
2. 初始化一个 DP 数组，其中 dp[j] 表示分配处理值的方式数，以便恰好将 j 个元素放入顶行。 最初 dp[0] = 1，因为分配的元素不是单个有效的空配置。 
3. 以频率 f 处理每个不同的值。 对于这个值，我们接下来计算一个新的 DP 数组，最初全为零。 该数组将累积在两行之间分配 f 个相同元素的方式。 
4. 对于 dp 中的每个当前状态 j，我们考虑将当前值的 k 个元素放入顶行，将 f-k 放入底行，对于从 0 到 f 的所有 k。 每个这样的选择都会贡献 dp[j] 乘以$\binom{f}{k}$到下一个[j+k]。 
5. 处理完所有值后，我们将最终答案设为 dp[n/2]，因为我们需要顶行中正好一半的元素。 

需要二项式系数，因为在每个值组中，即使值相同，选择哪些特定出现到顶行也很重要。 

### 为什么它有效

 在处理直到 v 的值的任何时刻，dp[j] 表示所有有效的部分赋值，其中处理值中的 j 个元素恰好位于顶行。 由于我们按升序处理值，因此值组内的任何分配都会保留两行中的排序顺序。 列约束会自动满足，因为两行都是由全局非递减值序列构造的，因此每行内的顺序会全局保留。 DP 转换用尽了独立分配每个值组的所有方法，并且没有两个不同的 DP 路径会产生相同的最终分配，因为每个路径都唯一对应于每个值有多少副本进入顶行的选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    a.sort()

    # compress frequencies
    freq = []
    i = 0
    while i < n:
        j = i
        while j < n and a[j] == a[i]:
            j += 1
        freq.append(j - i)
        i = j

    dp = [0] * (n + 1)
    dp[0] = 1

    used = 0

    for f in freq:
        ndp = [0] * (n + 1)
        # precompute binomial coefficients for this block
        # since f is small relative and total sum n <= 5000, direct computation is fine
        C = [[0] * (f + 1) for _ in range(f + 1)]
        for i in range(f + 1):
            C[i][0] = 1
            for j in range(1, i + 1):
                C[i][j] = (C[i-1][j-1] + C[i-1][j]) % MOD

        for j in range(used + 1):
            if dp[j] == 0:
                continue
            for k in range(f + 1):
                ndp[j + k] = (ndp[j + k] + dp[j] * C[f][k]) % MOD

        used += f
        dp = ndp

    print(dp[n // 2] % MOD)

if __name__ == "__main__":
    solve()
```该实现首先将输入排序并压缩为频率块，因为仅对每个不同的高度进行计数。 动态编程数组跟踪处理每个块后有多少元素被分配到顶行。 该转换迭代当前块中有多少元素被放置在顶行，并由表示相同元素之间的内部选择的二项式系数加权。 

关键的实现细节是仔细的索引：dp 范围只需要扩展到已处理元素的数量，因此我们使用`used`限制转换并避免不必要的计算。 

为了简单起见，二项式表按块重新计算； 自总计$n \le 5000$，总体成本仍在限制范围内。

 ## 边缘情况

 一种边缘情况是所有值都相同。 在这种情况下，排序约束是无关紧要的，答案纯粹是选择哪一半进入顶行。 DP 正确处理了这个问题，因为 f = n 的单个频率块产生 dp[n/2] =$\binom{n}{n/2}$。 

另一个边缘情况是所有值都不同。 在这里，每个值都是其自己的块，其中 f = 1，并且 DP 成为构建帕斯卡三角形的标准二项式卷积，产生$\binom{n}{n/2}$。 这符合任何子集都有效的直觉，因为排序会自动强制有效性。 

混合情况如`1 1 2 2`测试块之间的交互。 处理完第一个块后，dp 反映了 1 的所有分裂。 正确处理第二个块会乘以并移位这些计数，从而保留值组之间的独立性，同时保持总顶行大小限制。
