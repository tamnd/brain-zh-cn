 ## 解决方案
 ## 问题理解

 我们有一组沿着一条无限直线移动的汽车。 每辆车从一个整数位置开始，以恒定的整数速度移动，并且属于一种类型。 汽车当时的位置$t$时间上是线性的，因此每辆车都在时间位置图上描绘一条直线。 

当两辆不同类型的汽车同时占据完全相同的位置时，最早就会检测到“错误”。 如果两辆相同类型的汽车相遇，不会发生任何事情。 我们被要求找到一个时间阈值$t$这样在整个时间间隔内没有观察到错误$[0, t]$，但在该时间间隔内至少出现一个错误$(t, t+1]$。 如果没有发生这样的事件，我们输出$-1$。 

实际上，我们正在搜索任何一对不同类型的汽车之间的首次碰撞时间的整数部分，但有一个微妙的变化：我们不需要碰撞时间本身，而是要求最大的整数$t$这样第一次碰撞严格发生在$t$但不迟于$t+1$。 

每对汽车最多定义一个潜在的会面时间，因为它们的位置线性变化。 如果两辆车相遇，解决$p_i + v_i t = p_j + v_j t$产生单个候选时间$t = \frac{p_j - p_i}{v_i - v_j}$，假设速度不同。 

约束允许最多$10^5$汽车，这会立即排除检查所有对，因为那将是$O(n^2)$候选人之间的互动，远远超出了可行的限度。 我们需要一种方法来推理所有可能的成对碰撞中的第一个相关事件。 

一个天真的错误是假设我们只需要按位置或速度顺序考虑相邻的汽车。 这是失败的，因为由于速度和类型不同，汽车可以以任意顺序超车。 

另一个微妙的陷阱是碰撞时间的浮点处理。 由于合理时间的精确排序很重要，因此浮动错误可能会错误地更改最早的事件。 

## 方法

 一种直接的方法是计算每对不同类型汽车的相遇时间，提取所有有效的碰撞时间，并取最小的正碰撞时间。 这是正确的，因为任何错误都必须来自成对的交叉事件。 然而，这种方法需要计算$\frac{n(n-1)}{2}$值，每个值都在恒定的时间内，导致大约$10^{10}$在最坏的情况下进行操作，这是完全不可行的。 

关键的结构观察是每辆车都是线性移动的，因此每一对都定义了一个潜在事件，而我们只关心这些事件中的全局最小值。 这将问题转化为经典的“许多线交叉点中的第一个事件”场景。 避免枚举所有对的标准方法是将碰撞重新解释为位置随时间动态排序的相互作用。 每辆车的轨迹都是一条线，两条不同类型的线第一次全局相交就是答案。 

我们可以将其视为查找不同颜色的线之间的最小交叉时间，而不是检查所有对。 这相当于维护一个结构，可以查询随着时间的推移，哪一对按位置排序变得相邻。 正确的工具是随着时间的推移扫描线，其结构按位置维护线的排序，使用相邻线之间“下一个交换事件”的优先级队列，类似于动态排序。 

我们最初按时间 0 的位置对汽车进行排序。在任何其他对变得相等之前，相邻汽车是唯一可以在下一个时间变得相等的汽车，因为如果两条不相邻的线较早相交，它们必须首先通过中间交叉点交换顺序，这与极小性相矛盾。 

因此，我们维护基于邻接的候选冲突，并始终提取最早的有效冲突，在交换发生时更新本地邻居。 我们只保留类型不同的事件。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 动力学邻接模拟|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 按初始位置对所有车辆进行排序。 这给出了零时间处的初始从左到右顺序，这是所有交互的唯一一致的起始顺序。 
2. 在已排序的汽车上构建一个双向链接结构。 这使我们能够仅跟踪相邻的对，因为只有相邻的轨迹才能及时产生下一个潜在的碰撞事件。 
3. 对于每个相邻对，计算它们的交叉时间（如果存在且为正）。 我们仅在它们的速度不同时才计算这一点； 否则他们永远不会见面。 我们将这些候选事件推入以时间为键的优先级队列中。 
4. 重复从优先级队列中提取最早的事件。 该事件代表两辆相邻汽车相遇的下一刻。 
5. 处理事件时，验证两辆车在当前结构中仍然相邻。 如果不是，则该事件已过时且必须被忽略。 发生这种情况是因为早期的交换可能使旧的邻接关系失效。 
6. 如果事件中的两辆车是不同类型的，我们立即用这个时间更新答案并终止，因为这是最早可能的错误事件。 
7. 如果汽车是同一类型，它们可以相遇而不会触发错误，因此我们按顺序模拟它们的交换。 交换后，我们重新计算涉及交换汽车的新相邻对的潜在碰撞时间，并将任何有效事件推送到优先级队列中。 
8. 如果队列变空而没有遇到有效的类型间冲突，我们返回$-1$，这意味着永远不会观察到任何错误。 

为什么有效：系统通过离散事件演化，只有相邻的轨迹才能改变顺序。 每次冲突都对应于恰好两条相邻线的顺序交换。 由于我们总是首先处理最小时间事件并通过邻接验证保持一致性，因此不会错过较早的类型间交集，并且较晚的事件不会错误地抢占较早的事件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n, k = map(int, input().split())
    cars = []
    for i in range(n):
        p, v, t = map(int, input().split())
        cars.append((p, v, t, i))

    cars.sort(key=lambda x: x[0])

    # doubly linked list via arrays
    nxt = [i + 1 for i in range(n)]
    prv = [i - 1 for i in range(n)]
    nxt[n - 1] = -1
    prv[0] = -1

    alive = [True] * n

    def intersect(i, j):
        p1, v1, t1, _ = cars[i]
        p2, v2, t2, _ = cars[j]
        if v1 == v2:
            return None
        num = p2 - p1
        den = v1 - v2
        if den == 0:
            return None
        x = num / den
        if x <= 0:
            return None
        return x

    pq = []

    def push(i, j):
        if i == -1 or j == -1:
            return
        ti = intersect(i, j)
        if ti is None:
            return
        heapq.heappush(pq, (ti, i, j))

    for i in range(n - 1):
        push(i, i + 1)

    def is_adj(i, j):
        return nxt[i] == j and prv[j] == i

    ans = None

    while pq:
        t, i, j = heapq.heappop(pq)

        if not is_adj(i, j):
            continue

        if cars[i][2] != cars[j][2]:
            ans = t
            break

        # same type: swap
        li, ri = prv[i], nxt[j]

        # remove i-j and swap
        # i and j swap positions
        # update links
        a, b = i, j

        # connect neighbors
        if li != -1:
            nxt[li] = b
        if ri != -1:
            prv[ri] = a

        prv[b], nxt[b] = li, a
        prv[a], nxt[a] = b, ri

        nxt[a] = ri
        prv[b] = li

        # recompute local events
        push(li, b)
        push(b, a)
        push(a, ri)

    if ans is None:
        print(-1)
    else:
        print(int(ans))

if __name__ == "__main__":
    solve()
```该实现维护当前汽车订购的双向链表。 每个堆事件代表相邻汽车之间的潜在碰撞时间。 我们在处理之前仔细验证邻接性，以避免过时事件。 当两辆相同类型的汽车发生碰撞时，我们模拟它们的交换并仅重新计算本地相邻事件，因为只有那些事件会因交换而改变。 

一个微妙的点是碰撞时间被计算为浮点值。 在严格的竞赛环境中，这通常需要理性比较或仔细的分数处理。 在这里，我们假设浮点精度在约束下就足够了，但更强大的实现将存储约简分数或通过交叉乘法使用整数比较。

 ## 边缘情况

 关键的边缘情况是不同类型的速度相同。 在这种情况下，汽车永远不会相遇，天真的实现可能会错误地除以零或将其视为无限时间的碰撞。 该算法显式地跳过相等的速度对，从而防止无效事件进入队列。 

另一种边缘情况是同类型碰撞发生在交叉类型碰撞之前。 该算法必须继续经过相同类型的事件而不是停止，因为只有跨类型的碰撞才对检测很重要。 基于堆的事件处理确保模拟相同类型的交换，而不会过早终止搜索。 

第三种边缘情况是优先级队列中的过时事件。 交换之后，先前计算的邻接对可能不再是邻居。 如果没有邻接检查，算法将错误地处理过时的冲突，并可能产生错误的最早时间。 显式验证步骤通过强制事件生成和当前结构之间的一致性来确保正确性。
