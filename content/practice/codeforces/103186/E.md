 ## 解决方案
 ## 问题理解

 我们正在模拟一个简化的捕鱼系统，其中每个抛投根据鱼类类型的已知概率分布恰好生产一条鱼。 每种鱼都属于五个稀有等级之一，每个等级都有固定的销售价值，但最高稀有度的价值不同。 

每次钓鱼尝试都会花费固定数量的鱼饵，因此每次投掷都会产生净收益，等于捕获的鱼的价值减去该成本。 玩家执行固定次数的独立施法，我们被要求计算预期的总利润。 

输入给出了鱼类类型的列表，每种鱼类都有一个稀有度标签和被捕获的概率。 概率之和为 1，因此每次投掷总是恰好产生从该分布中抽取的一条鱼。 

产出是k次投放后的预期利润，其中利润包括鱼的销售价值和扣除每次投放的饵料成本。 

约束 n ≤ 100 和 k ≤ 100 立即表明即使每个状态 O(nk) 或 O(n) 方法也很容易足够快。 不需要对复杂状态进行模拟或动态编程，因为每个转换都是独立且相同分布的。 这强烈表明线性期望属性会将整个问题分解为每次转换的单个期望值乘以 k。 

一个微妙的点是“传奇鱼”类别超越了其通常的价值。 尽管所有概率都是独立且相同的，但忘记替换 S 型鱼的值会产生一致但不正确的期望。 

如果尝试通过重复求和来重复模拟 k 个步骤，另一个潜在的陷阱是浮点累积错误。 由于 k 很小，两种方法都可以工作，但是重复的乘法是不必要的，而且不如计算单个期望稳定。 

## 方法

 对该过程的简单解释是独立模拟 k 次钓鱼尝试中的每一次。 对于每次尝试，我们根据给定的分布对鱼进行采样，添加其值，减去诱饵成本，然后重复。 期望值可以通过蒙特卡罗模拟来近似，但这会引入随机性和误差，并且在给定确定性概率的情况下是不必要的。 

确定性蛮力期望方法将通过对所有鱼类类型求和、将概率乘以值，然后减去诱饵成本来明确计算一次投掷的预期增益。 之后，我们将重复此计算 k 次或将单播期望乘以 k。 

关键的观察是期望的线性。 每次钓鱼尝试都是独立的，总奖励是 k 个相同随机变量的总和。 因此，总的预期奖励只是一次演员的预期奖励的 k 倍。 不需要联合分配或状态跟踪。 

这将整个问题简化为计算鱼类值的单个加权平均值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 模拟 | O(k·样本) | O(1) | O(1) | 太慢/不精确|
 | 每次演员的预期价值| O(n + k) | O(1) | O(1) | 已接受 |
 | 优化（乘以期望）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 读取所有鱼类类型及其概率，并按稀有度值映射对它们进行分组。 

我们需要将每个稀有字符转化为其货币价值。 唯一不标准的规则是S型鱼使用10000而不是100。 
2. 对于每种鱼类型，计算其对预期值的贡献，即概率乘以其指定值。 

这直接来自期望的定义：每个结果都会贡献按其可能性加权的价值。 
3. 将所有贡献相加以获得单次钓鱼尝试的预期值。

这代表了考虑诱饵成本之前的平均奖励。 
4. 从一次抛投的预期值中减去诱饵 (23) 的固定成本。 

由于每次都会支付诱饵成本，因此它是确定性减法，与概率不相互作用。 
5. 将得到的每次广播期望值乘以 k。 

期望的线性确保 k 个独立演员的总预期奖励恰好是单次演员期望的 k 倍。 

### 为什么它有效

 每次钓鱼尝试都是具有相同分布的独立随机变量。 总利润是 k 个这样的变量的总和，每个变量都已经包含了确定性的诱饵成本。 期望分布于总和，因此总的期望是期望之和。 转换之间不存在相关性或条件依赖性，因此不需要额外的状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, k = map(int, input().split())
    
    value = {
        'D': 16,
        'C': 24,
        'B': 54,
        'A': 80,
        'S': 10000
    }
    
    exp_one = 0.0
    
    for _ in range(n):
        t, p = input().split()
        p = float(p)
        exp_one += p * value[t]
    
    exp_one -= 23.0
    ans = exp_one * k
    
    print(f"{ans:.4f}")

if __name__ == "__main__":
    main()
```代码直接实现了期望计算。 该字典将稀有度类别映射到值，并对 S 型鱼进行特殊覆盖。 每条鱼将概率乘以值贡献为一个运行总和。 

23 的减法是每次投掷一次，而不是每条鱼一次，因为诱饵成本与结果无关，而是取决于尝试次数。 最后，乘以 k 聚合 k 个相同的独立试验。 

一个常见的错误是在循环内减去鱼类类型的诱饵成本，这会错误地按 n 而不是 k 缩放成本。 

## 工作示例

 我们构造一个小例子来说明计算。 

输入：```
3 2
D 0.50
C 0.30
S 0.20
```逐步预期：

 | 鱼 | 概率 | 价值| 贡献 |
 | ---| ---| ---| ---|
 | d | 0.50 | 0.50 16 | 16 8.0 |
 | C | 0.30 | 0.30 24 | 7.2 |
 | S | 0.20 | 0.20 10000 | 2000.0 |

 每次施法总和 = 2015.2

 减去诱饵 = 2015.2 − 23 = 1992.2

 两次转换 = 3984.4

 这证实了线性缩放。 

另一个例子：

 输入：```
2 3
D 1.00
S 0.00
```每个施法值 = 16

 扣除成本后 = -7

 总计 = -21

 这表明当饵料成本超过鱼的价值时，预期可能为负。

 ## 边缘情况

 一个关键的边缘情况是概率将零权重分配给高价值的鱼。 该算法仍然将它们包含在映射中，但它们的贡献变为零，因此它们不会影响期望。 

例如：```
2 2
S 0.00
D 1.00
```每个演员的期望值为 16，负 23 为 -7，总计 -14。 尽管存在 S 鱼，但它完全被忽略了。 

另一种情况是所有鱼的价值都低于鱼饵成本。 预期变为负值，这是有效的，因为预期利润不必为正值。
