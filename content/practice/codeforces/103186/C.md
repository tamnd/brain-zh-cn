 ## 解决方案
 ## 问题理解

 我们有一组学生，每个学生都有一个唯一的学生 ID 和一个初始分数。 其中，有一位学生比较特殊，用输入顺序中的索引m表示，我们称这位学生为小A。 

转换根据原始分数分两个独立的阶段进行。 首先，小A调整自己的分数：如果他的初始分数低于60分，则将其提高到60分，否则保持不变。 这种改变只取决于他自己原本的价值，并不影响其他学生的评价。 

其次，小A使用原始的完整分数列表查看其他人。 阈值是根据任何修改之前的初始状态计算的平均分数。 任何其他原始分数至少为该平均分的学生的分数将减少 2 分，下限为 0。 

关键的微妙之处在于，这两个决定都是基于分数的初始快照。 小A修改后，平均值不再重新计算，减少的资格也从初始数据开始固定。 唯一动态的部分是这两个规则的最终应用。 

输出需要打印所有学生的最终成绩，并按学生 ID 升序排列。 

约束很小，n 最大为 100。这立即表明每个测试的 O(n^2) 甚至 O(n) 解决方案就足够了。 对复杂数据结构的任何尝试都是不必要的，实现的清晰度比优化更重要。 

一个常见的错误是修改小A的分数后重新计算平均值。 例如，如果初始分数为 [1, 4, 100]，则平均值为 35。小 A 的更新不会更改此阈值，但粗心的实现可能会重新计算并错误地移动截止值。 

另一个错误是就地更新分数，然后使用更新后的值来确定某人是否高于平均水平。 这打破了决策必须基于初始分数的规则。 

最后，减去 2 时忘记 0 的下限可能会产生负值，这是无效的。 

## 方法

 蛮力观点很简单。 我们首先计算所有初始分数的平均值。 然后我们迭代每个学生。 如果学生是小A，我们应用将他的分数限制在至少60的规则。否则，我们检查他们的原始分数是否至少是平均值，如果是，则减去2。 

这种方法已经在线性时间内运行。 即使我们明确地将计算分成多个遍，总成本也是 O(n)。 考虑优化的唯一原因是避免意外重新计算平均值或混合更新值和原始值。 

关键的见解是这不是带有反馈的模拟。 定义决策边界后，不会发生任何改变。 一旦我们存储原始数组并计算平均值一次，所有其他操作都是对每个元素的简单确定性转换。 这将问题简化为从初始分数到最终分数的单遍映射。 

在转换之前维护动态结构或排序没有任何好处。 排序仅在最终输出步骤中变得相关，这是根据 ID 而不是分数。

 ## 算法演练

1. 读取所有学生记录并将其存储为 (student_id, Score) 对。 我们需要完整的存储，因为输出必须按 ID 排序，而不是输入顺序。 
2. 计算所有原始分数的总和并除以 n 以获得平均值。 该值在整个计算过程中必须保持固定。 
3. 根据小A在输入序列中的位置m来识别他，并单独存储他的原始分数。 我们保持不变以供决策。 
4. 创建从学生 ID 到最终分数的映射。 我们将根据应用于原始值的规则来填充此值。 
5. 迭代每个学生。 如果学生是小A，则将他的分数设置为max(60,original_score)。 这确保了在不影响其他人的情况下强制执行最小通过调整。 
6. 对于所有其他学生，将他们的原始分数与计算出的平均值进行比较。 如果original_score≥average，则将其减少2，确保其不低于0。否则，保持不变。 
7.处理完所有学生后，按学号排序并输出最终成绩。 

### 为什么它有效

 正确性来自于所有决策都是使用固定参考状态做出的：原始分数数组。 平均值将学生静态划分为两组，小A的调整与该划分无关。 由于没有规则依赖于中间结果，因此转换是对每个元素应用的纯函数。 排序只影响输出顺序，不干扰计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

students = []
id_list = []
scores = []
a_idx = m - 1  # m is given in input order position

for _ in range(n):
    sid, sc = map(int, input().split())
    students.append((sid, sc))
    id_list.append(sid)
    scores.append(sc)

total = sum(scores)
avg = total / n

a_id, a_score = students[a_idx]

result = {}

for sid, sc in students:
    if sid == a_id:
        result[sid] = max(60, sc)
    else:
        if sc >= avg:
            sc = max(0, sc - 2)
        result[sid] = sc

students.sort(key=lambda x: x[0])

print(" ".join(str(result[sid]) for sid, _ in students))
```该实现保留了两个并行表示：学生的原始列表和最终结果的字典。 使用字典的原因是输出顺序取决于排序的 ID，这可能与输入顺序不同。 在任何修改之前都会计算一次平均值，以确保阈值逻辑的正确性。 

一个微妙的点是正确识别小A。 输入保证m指的是输入位置，而不是学生ID。 这就是为什么我们直接使用 m - 1 对存储列表进行索引。

 ## 边缘情况

 一种边缘情况是所有分数都相同。 在这种情况下，平均分等于每个分数，因此每个非小A学生都有资格减分。 对于输入：```
3 1
2 50
1 50
3 50
```平均数是 50。小 A（学生 2）变成 60。其他人都减少 2，每人产生 48。 该算法正确应用了 ≥ 条件。 

另一个极端情况是单个学生。 对于输入：```
1 1
1 59
```平均值为 59。小 A 是唯一的学生，被限制为 60。没有其他更新，因此输出为 60。算法自然地处理这个问题，因为其他人的循环是空的。 

最后一个边缘情况是小 A 已经远高于 60 且高于平均水平。 对于输入：```
2 2
1 100
2 90
```平均分是 95。学生 1 低于平均分，所以没有变化。 小A仍然是90，因为max(60, 90)是90。最终输出除了排序之外没有变化。 这证实了小A的规则不会与应用于其他规则的阈值条件相互作用。
