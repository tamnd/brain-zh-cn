 ## 解决方案
 ## 问题理解

 我们得到一个数组$n$正整数，其中每个值代表沿街道放置的植物的“生长水平”。 对于每个职位$i$，我们想象在$i$-th 植物，然后我们计算一个等于所有其他植物生长水平乘积的分数，不包括位置处的植物$i$。 

任务是独立计算每个索引的该值，并输出模数结果$998244353$。 

所以对于每个位置$i$，我们想要：$$x_i = \prod_{j \ne i} a_j \pmod{998244353}$$约束允许最多$n = 10^5$，以及每个$a_i$可以大到$10^9$。 每个索引的所有元素的直接乘法会太慢，因为这会涉及到$O(n^2)$乘法。 

这些值本身很大，因此如果不仔细使用模块化算术处理，中间产品很容易超出 64 位限制。 

当值很大并且重复时，就会出现一种微妙的情况。 例如，如果所有值都相等，则每个答案都应该相同且等于$a^{n-1} \mod 998244353$。 模除法或预计算的任何错误处理都会破坏对称性。 

另一个重要的边缘情况是当$n = 2$。 每个答案只是另一个元素，这很容易得到正确的结果，但是暴力实现有时会意外地将两个值相乘并错误地除法，而没有正确处理模逆。 

## 方法

 一个简单的解决方案计算每个$x_i$通过迭代所有其他元素并将它们相乘。 这是正确的，因为它直接遵循问题的定义。 然而，对于每个索引，我们执行$n-1$乘法，总共大约有$n(n-1)$运营。 和$n = 10^5$，这变成了$10^{10}$乘法，这远远超出了任何合理的时间限制。 

关键的观察是所有答案都共享相同的全局结构：每个$x_i$是除一个元素之外的整个数组的乘积。 如果我们预先计算所有元素的总乘积，那么每个答案都可以通过删除来获得$a_i$来自该产品。 由于模运算下不允许直接除法，因此我们使用模逆。 

我们计算：$$\text{total} = \prod_{i=1}^{n} a_i \mod 998244353$$然后：$$x_i = \text{total} \cdot a_i^{-1} \mod 998244353$$自从$998244353$是素数，每个非零元素都有一个逆元素，我们可以使用快速求幂来计算它：$$a_i^{-1} = a_i^{MOD-2} \mod MOD$$这将问题简化为计算一个全局乘积，并且$n$模幂。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最优（前缀积+逆）|$O(n \log MOD)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 1.读取数组并定义模数$MOD = 998244353$。 每次计算都将以该值为模进行减少，以防止溢出并通过模逆进行除法。 
2. 计算所有元素模的总乘积$MOD$。 这是通过在数组上迭代一次并逐步相乘来完成的。 此步骤将所有全局信息压缩为单个值。 
3. 对于每个索引$i$，计算模逆$a_i$使用指数的快速求幂$MOD - 2$。 这是可行的，因为模数是素数，所以费马小定理保证了这种反转的正确性。 
4. 将总乘积乘以以下的倒数$a_i$，然后对结果取模$MOD$。 这有效地抵消了$a_i$，留下所有其他元素的乘积。 
5. 按顺序输出所有计算值。 

### 为什么它有效

 核心不变量是，在任何时刻，变量`total`始终表示数组中所有元素的乘积模$MOD$。 当我们乘以$a_i^{-1}$，我们用代数方法精确地除掉一次出现的$a_i$在模运算下。 由于模逆在素数域中是精确的，因此该操作保留了正确性并产生除所选元素之外的所有元素的精确乘积。 索引之间的依赖关系不会被破坏，因为每个计算都从相同的共享全局产品开始。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modpow(a, e):
    res = 1
    a %= MOD
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

n = int(input())
a = list(map(int, input().split()))

total = 1
for x in a:
    total = total * x % MOD

ans = []
for x in a:
    inv = modpow(x, MOD - 2)
    ans.append(total * inv % MOD)

print(*ans)
```实现遵循算法的确切结构。 这`modpow`函数执行二进制求幂，这是有效计算模逆所需的。 总乘积在线性时间内计算一次。 然后，通过将每个元素通过其逆元素分开来独立处理。 

一个微妙的点是所有操作都是按模执行的$MOD$，包括乘方内的乘法。 这可以防止溢出并确保模运算的正确性。

import sys, io

MOD = 998244353

def modpow(a, e):
    res = 1
    a %= MOD
    while e:
        if e & 1:

        a = a * a % MOD
        e >>= 1

def solve():

 ## 边缘情况

 关键的边缘情况是所有值都相同。 对于输入：```
5
7 7 7 7 7
```总产品为$7^5 \mod MOD$。 每个答案除以一个 7，结果是$7^4$。 该算法自然地处理了这个问题，因为 7 的每个模逆都是相同的，因此所有输出保持相等。 计算从不依赖于索引排序，因此保留了对称性。 

另一个边缘情况是$n = 2$。 为了：```
2
a b
```总产品是$ab$。 第一个答案变成$ab \cdot a^{-1} = b$，第二个就变成了$a$。 这与直接解释相匹配，不需要任何特殊情况逻辑。 

最后，接近的大值$10^9$可以安全地处理，因为所有乘法都减少了模数$998244353$在每一步中，防止 Python 和中间模运算中的溢出。
