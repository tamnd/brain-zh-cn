 ## 解决方案
 ## 问题理解

 我们得到了一系列代表卡片值的整数序列，排列成一行。 两名玩家，爱丽丝和鲍勃，每回合轮流选择一张牌，直到所有牌都被拿走。 爱丽丝总是先行动。 两个玩家都是完全最优的：爱丽丝试图最大化她相对于鲍勃的最终优势，而鲍勃则试图最小化同样的优势。 

玩家的分数定义为他们所挑选的牌的值之和。 最终的利息数量是所有卡片都被拿走后，Alice 的总数与 Bob 的总数之间的差额。 

所以如果爱丽丝收集了一套$A$鲍勃收集了一套$B$，我们要计算：$$\sum A - \sum B$$不同的是，这是一个完美信息交替游戏。 每一步行动不仅会改变眼前的收益，还会改变未来牌的可用性，因此贪婪的“总是拿最好的牌”策略显然并不正确。 

约束条件$n \le 5000$建议$O(n^2)$或者$O(n^2 \log n)$解决方案是可以接受的，而任何子集或排列上的指数都是不可行的。 所有游戏状态的朴素极小极大会考虑每个可能的选择序列，其增长为$n!$，显然不可能。 

一个关键的边缘情况来自负值。 如果所有数字都是负数，“取大正数”的直觉就会完全崩溃。 例如，与$[-1, -10, -20]$，最好的移动并不明显，因为“更大”意味着更少的负面影响，而不是更多的收益。 另一个边缘情况是混合迹象，即早期获得较大的正值可能会迫使对手稍后获得更多的负值，这在战略上很重要。 

另一个微妙的情况是值相同或对称时。 例如，$[5, 5, 5, 5]$，任何最优策略都不重要，最终的差异应该为零。 任何依赖于排序工件的解决方案仍然必须保持关系下的正确性。 

## 方法

 直接的暴力方法将模拟每种可能的游戏状态。 在每一步中，玩家选择剩余的一张牌，我们在切换玩家时递归到下一个状态。 这是子集上的标准极小极大递归。 

这在概念上是有效的，因为它完全模拟了最佳游戏，但它是指数级的。 即使有记忆，状态的数量也是$2^n$，因为剩余卡牌的每个子集都是一个不同的状态，并且转换需要尝试最多$O(n)$选择。 这大致给出了$O(n \cdot 2^n)$，这远远超出了可行性$n = 5000$。 

关键的观察结果是，游戏实际上并不依赖于除了移动平价之外的顺序约束。 每张牌只拿一次，每个玩家也只拿一次$\lfloor n/2 \rfloor$或者$\lceil n/2 \rceil$牌。 鉴于鲍勃同样试图破坏她的优势，最佳策略简化为决定爱丽丝可以保护哪些牌。 

重写目标会有所帮助。 令总和为$S$。 如果爱丽丝拿总和$A$, 鲍勃得到$S - A$， 所以：$$A - (S - A) = 2A - S$$所以最大化Alice的优势就等于最大化$A$，她收集的值的总和。 

现在游戏变成：爱丽丝想要最大化她收集到的总和，鲍勃想要最小化它，但两人交替选择。 由于每次移动只是删除一个元素，因此该结构简化为没有位置约束的多重集上的经典交替选择问题。 

最优策略对排序值是贪婪的：两个玩家都有效地从值顺序的剩余极端中进行选择，但由于不存在邻接或结构约束，因此最优结果只是通过对所有值进行排序并按从最大到最小的交替顺序分配它们来确定。 

首先行动的爱丽丝将在这个最佳游戏模型中获得最大的可用值。 

因此，解决方案简化为按降序对数组进行排序，并对从索引 0 开始的每隔一个元素求和。

 ## 算法演练

 1. 按降序对卡牌值列表进行排序。 

这确保了我们总是先考虑价值较高的牌，然后再考虑价值较低的牌，这是必要的，因为两个玩家的实力是对称的，只有价值的相对顺序很重要。 
2. 通过分配索引来模拟最佳交替选择：Alice 采取位置 0、2、4 等，而 Bob 隐式采取位置 1、3、5 等。 

这是有效的，因为排序后，每个玩家总是会选择最高的剩余值，并且交替轮流会强制执行此分区。 
3. 通过将已排序数组的偶数索引处的值相加来计算 Alice 的总和。 
4. 通过从总和中减去 Alice 的总和，隐式或直接计算 Bob 的总和。 
5.输出差值$A - B$，这相当于$2A - S$。 

如果需要，这可以避免维护单独的 Bob 累加器。 

### 为什么它有效

 关键的不变量是排序后，比赛的相对顺序始终与全球排名一致。 由于两个玩家总是喜欢更高的值，并且不存在阻止访问任何剩余卡的结构约束，因此游戏崩溃为确定性的排名分配：Alice 接收按排序顺序的所有偶数排名元素，Bob 接收所有奇数排名元素。 任何局部偏差都不能改善任何一个玩家的结果，因为任何偏离最高剩余值的偏差都会立即让同一个玩家失去一个更好的选择，或者在下一步行动中将其交给对手。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort(reverse=True)
    
    alice = 0
    for i in range(0, n, 2):
        alice += a[i]
    
    total = sum(a)
    bob = total - alice
    
    print(alice - bob)

if __name__ == "__main__":
    solve()
```该实现首先读取数组并按降序对其进行排序。 该循环通过从索引 0 开始每隔一个元素来累积 Alice 的份额，反映了她在交替序列中的先手优势。 最终的差异是使用恒等式计算的$A - B = 2A - S$，这避免了鲍勃的额外传递。 

这里的一个常见陷阱是忘记 Alice 首先移动，这会改变分配索引的奇偶性。 另一种方法是尝试不排序的两指针模拟，但这种方法失败了，因为最佳玩法取决于全局排名，而不是局部邻接。

 ## 边缘情况

 对于像这样的单元素数组`[x]`，爱丽丝立即接受，所以答案很简单`x`。 该算法排序并采用索引 0，它直接匹配此行为。 

对于全相等数组，例如`[k, k, k, k]`，排序不会改变顺序，Alice 正好拿走了总和的一半。 鲍勃拿走另一半，产生零差异。 偶数指数累积恰好给出了总数的一半，在平局下保持了正确性。 

对于全负数组，例如`[-1, -2, -3, -4]`, 排序产生`[-1, -2, -3, -4]`。 爱丽丝采取`-1`和`-3`，鲍勃采取`-2`和`-4`。 该算法正确地倾向于较少的负值，这与最大化 Alice 的总和是一致的，即使所有贡献都是有害的。
