 ## 解决方案
 ## 问题理解

 我们有几个独立的游戏。 每个游戏都由一小部分由小写字母组成的字符串组成。 两名玩家轮流轮流，从爱丽丝开始，每一轮玩家选择一个字符串，并从同一字符串中删除一个字符或两个不同的字符。 如果玩家要移动但所有字符串都是空的，因为无法进行任何操作，那么玩家就输了。 

关键的观察结果是，除了轮流之外，弦不会相互作用。 一次移动仅更改一个字符串，并且没有任何操作在字符串之间移动字符。 这立即表明该游戏是独立堆的不相交和，每个堆都是一个字符串，其“状态”就是剩余的字母数量以及如何成对或单个使用它们。 

就结构而言，约束非常小，最多 10 个字符串，每个字符串长度最多 40。这排除了对所有字符串联合进行指数游戏状态搜索，但仍然允许按字符串分析，甚至在需要时对单个字符串进行每次移动 DP。 然而，由于移动规则仅取决于从单个字符串中选择 1 或 2 个字符，因此该结构强烈让人想起公正的组合游戏，其中每一堆都贡献一个 Grundy 值或一个简单的等效不变量。 

天真的思维的一个微妙的失败案例是将每个字符串视为独立的堆，只需从总和中减去 1 或 2 即可。 例如，如果将问题简化为“移动大小为 1 或 2 的字符总数”，您就会忽略“来自同一字符串的两个不同字符”约束不会全局交互的事实。 像这样的字符串`"aaaa"`与两个单独的字符串的行为不同`"aa"`和`"aa"`因为所选的两个字母必须来自同一堆。 

另一个误导性的情况是所有字符串都相同或对称。 例如，与多个`"aabb"`对于字符串，人们很容易假设对称性会导致微不足道的奇偶校验推理，但最佳发挥取决于每个字符串可以提取多少对，而不仅仅是总计数。 

## 方法

 暴力方法将尝试将整个游戏状态建模为所有字符串内容的元组，并通过记忆递归地模拟所有可能的动作。 每个状态通过选择一个字符串然后选择其中的一个或两个索引来分支。 由于每个字符串的长度高达 40，子集的数量非常巨大，即使使用记忆化，可到达配置的数量也会爆炸，因为删除两个任意字母会创建许多不同的中间状态。 分支因子也很大，因为每个长度为 k 的字符串允许 k 个单次移动和 k(k−1)/2 个配对移动。 尽管 n 最多为 10，但内部字符串的复杂性使得这个问题变得棘手。 

关键的见解是停止跟踪精确的角色身份，而是将每个字符串简化为完全确定其游戏价值的简单不变量。 每次移动都会从一个字符串中删除 1 或 2 个字符，因此在字符串中唯一相关的数量是其长度，而不是其组成。 “两个不同字母”的条件对于最佳游戏来说并不重要，因为任何长度为 k 的字符串总是允许删除任何一对不同的位置； 角色的身份不会以影响最佳计数的方式限制移动可用性。 因此，每个字符串的行为就像一堆大小为 k 的东西，每次移动您可以删除 1 或 2 个标记。 

这将每个字符串简化为一个独立的减法游戏，其中可以从大小为 k 的堆移动到 k−1 或 k−2。 这正是经典的 take-1-or-2 游戏，其 Grundy 值遵循简单的周期模式：0、1、2 以周期 3 重复。整个游戏就是所有字符串中这些 Grundy 值的异或。

 ## 算法演练

 我们将每个字符串简化为单个数字，其长度为 k，并根据 take-1-or-2 规则计算其 Grundy 贡献。 

1. 对于每个字符串，计算其长度 k。 这已经足够了，因为字符身份不会影响是否可以删除 1 个或 2 个字母。 
2. 对于每个 k，使用已知的递推式计算其 Grundy 值：g(k) = mex({g(k−1), g(k−2)})，基本情况为 g(0)=0，g(1)=1。 这会生成重复模式 g(k)=k mod 3。 
3. 对所有字符串中的所有值 g(k) 进行异或。 此步骤将独立公正的游戏合并为单一游戏结果。 
4. 如果最终的异或非零，则爱丽丝获胜，因为起始位置在最佳玩法下获胜。 否则鲍勃获胜。 

唯一重要的推理步骤是为什么字符串结构不会影响超出长度的移动的可用性。 尽管规则说的是“两个不同的字母”，但在长度为 k 的字符串中，选择任意两个位置总是对应于删除两个不同的字符，无论它们的字母是否匹配，因此可行性仅取决于 k ≥ 2。 

它起作用的原因是，每个字符串都是一个独立公正的游戏，其状态完全由其长度来表征，并且每一步都会将一堆恰好减少 1 或 2。斯普拉格-格伦迪定理适用，因此堆 Grundy 值的异或决定了获胜者。 字符串之间不存在交互作用，因此游戏状态中不会出现耦合项。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        xor_sum = 0
        for _ in range(n):
            s = input().strip()
            k = len(s)
            xor_sum ^= (k % 3)
        print("Alice" if xor_sum else "Bob")

if __name__ == "__main__":
    solve()
```该实现直接对缩减进行编码。 每个字符串被读取并立即压缩为其长度，丢弃所有字符信息。 表达式`k % 3`是 take-1-or-2 减法游戏的预先计算的 Grundy 值。 XOR 累加器结合了所有独立的组件。 

一个微妙的点是我们不模拟任何动作。 任何这样做的尝试都会不必要地超出限制。 正确性完全依赖于将每个字符串结构缩减为具有确定性 Grundy 值的堆。 

## 工作示例

 考虑单个字符串的情况`"aaa"`。 

| 步骤| 字符串| 长度 k | k 模 3 | 异或|
 | ---| ---| ---| ---| ---|
 | 1 | 啊啊| 3 | 0 | 0 |

 结果为零，所以鲍勃获胜。 这符合直觉，因为从尺寸 3 开始，第一个玩家移动到 2 或 1，并且最佳游戏会迫使玩家处于失败位置。 

现在考虑两个字符串`"aa"`和`"a"`。 

| 步骤| 字符串| 长度 k | k 模 3 | 异或|
 | ---| ---| ---| ---| ---|
 | 1 | 啊| 2 | 2 | 2 |
 | 2 | 一个 | 1 | 1 | 3 |

 最终 XOR 为 3，非零，因此 Alice 获胜。 这反映出第一步可用于在其中一个堆中留下失败的配置，同时保留控制权。 

这些痕迹表明，只有长度才重要，并且 XOR 组合可以正确捕获字符串之间的交互。

 ## 边缘情况

 对于像这样的单字符字符串`"a"`，该位置获胜，因为玩家立即将其移除，并且没有给对手留下任何一步。 

输入：```
1
1
a
```该算法计算 k=1，因此 k mod 3 = 1，给出非零 XOR，Alice 获胜。 

对于长度为 2 的字符串，假设`"aa"`，第一个玩家可以删除两个角色或一个角色。 在这两种情况下，对手在最佳发挥下都会获得较小的非获胜位置。 

输入：```
1
1
aa
```该算法计算 k=2，因此 k mod 3 = 2，仍然非零。 然而，用 XOR 术语来说，具有非零值的单堆获胜，因此 Alice 在这里获胜； 仅当多个堆中的组合 XOR 变为零时，才会出现失败位置，这符合 Grundy 解释而不是天真的奇偶校验。
