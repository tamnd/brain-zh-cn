 ## 解决方案
 ## 问题理解

 任务是对平面上相对于由四个边界点定义的固定几何区域的点进行分类：$(0, d)$,$(d, 0)$,$(n, n-d)$， 和$(n-d, n)$。 这四个点形成一个凸四边形，看起来像一个旋转 45 度的正方形，然后将其剪裁在$n \times n$网格。 

我们有多个查询点，每个查询点代表一个 Grashopper 位置。 对于每个点$(x, y)$，我们必须确定它是位于该四边形内部还是正好位于其边界上。 

限制非常小：$n \le 100$和$m \le 100$。 这立即排除了对高级几何结构或优化的任何需要。 甚至一个$O(m)$每个查询检查都很简单，因为操作总数保持在几千以下。 这里重要的是推导描述该地区的正确的不平等。 

关键的困难不在于性能，而在于正确性：将旋转的几何形状转换为简单的坐标条件。 

通常会出现一些失败案例。 

一个常见的错误是将区域视为由最小和最大坐标界定的简单矩形。 例如，假设该字段只是$d \le x \le n-d$和$d \le y \le n-d$会错的。 像这样的一点$(1, 1)$可能满足某些值的矩形边界，但仍然位于菱形字段之外。 

另一个错误是忘记了边界。 由于该问题明确地将边缘上的点计为内部点，因此严格的不等式如$<$和$>$而不是$\le$和$\ge$会错误地拒绝有效点，例如顶点。 

正确的解决方案来自于确定形状是由与旋转正方形的两条对角线相对应的两个线性约束定义的。 

## 方法

 强力几何方法将尝试使用方向测试或三角形分解来确定点是否位于多边形内部。 人们可以将四边形分成两个三角形，并使用叉积检查该点是否在任一三角形内。 这工作正常并且在概念上是通用的，但对于这种受约束的形状来说是不必要的。 

暴力破解的问题不在于正确性，而在于过度杀伤力：为每个查询计算叉积会带来比所需更多的复杂性，并且模糊了区域的结构。 在最坏的情况下，它仍然运行在$O(m)$，但常数因素较重，实施风险较大。 

关键的观察结果是，多边形是一个旋转 45 度的完美正方形，与直线对齐$x + y = d$和$x + y = n + d$，并且还受到$x - y = -d$和$x - y = n - d$。 这意味着可以仅使用从这些对角线导出的两个线性不等式来检查隶属度。 

一旦认识到这一点，每个查询就减少为检查该点是否位于两个对角方向上的两条平行线之间。

 | ---| ---| ---| ---|
 | 多边形/原始几何| O(米) | O(1) | O(1) | 已接受但没有必要 |

 ## 算法演练

 我们使用两个变换后的坐标重写位于菱形区域内的条件：$x + y$和$x - y$。 

1. 计算总和$s = x + y$。 这捕获沿一个对角线方向的位置。 该形状限制在两条等和线之间，因此这是第一个约束。 
2. 计算差异$t = x - y$。 这捕获沿另一个对角线方向的位置，与第一个约束正交。 该区域也被限制在两条恒定差线之间。 
3. 从角点导出边界。 替换顶点表明：

 - 最小总和为$d$，最大总和为$n + d$- 最小差异为$-d$，最大差值为$n - d$4. 对于每个查询点，检查两个条件是否成立：$d \le x + y \le n + d$和$-d \le x - y \le n - d$5. 如果两个不等式都满足，则该点在边界内或边界上； 否则，它就在外面。 

这样做的原因是四边形恰好是两个条带的交集：一个由边界定义$x + y$，另一个由边界定义$x - y$。 它们的交点形成旋转的正方形。 

### 为什么它有效

 转变$(x, y) \rightarrow (x + y, x - y)$将轴对齐的正方形映射为旋转的正方形。 在这个变换后的空间中，该区域变成一个轴对齐的矩形。 由于轴对齐矩形的特征在于每个坐标上的独立边界，因此隶属度减少为两个独立的区间检查。 这保证了正确性，因为原始多边形的每条边都与变换坐标中的边界线之一精确对应。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d = map(int, input().split())
    m = int(input())

    for _ in range(m):
        x, y = map(int, input().split())

        s = x + y
        t = x - y

        if d <= s <= n + d and -d <= t <= n - d:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```代码直接实现了两个派生约束。 每个查询都是独立处理的，计算总和和差一次并将它们与固定边界进行比较。 

最微妙的部分是确保包容性比较。 由于边界点有效，所有不等式都必须使用$\le$和$\ge$。 剩下的就是简单的算术。

 ## 边缘情况

 一种边缘情况是当一个点恰好位于多边形的顶点上时。 例如，$(0, d)$。 这里$x+y = d$和$x-y = -d$，两者都满足包含界限，因此算法正确返回 YES。 

另一种情况是当一个点位于一个对角线边界之外但在另一个对角线边界之内时。 例如，一个点$x+y = d-1$即使其差值有效，也会立即失败。 该算法正确地拒绝了它，因为这两个条件同时需要。 

最后一种情况是当点位于对角附近时$(n-d, n)$。 这些最大化两个变换后的坐标：$x+y = n+d$和$x-y = n-2d$，两者仍在界限之内。 该算法正确地包含它们，确认上限是严格的并且从几何中正确导出。
