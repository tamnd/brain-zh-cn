 ## 解决方案
 ## 问题理解

 我们被赋予了一个能够进化出$n$离散步骤上的维向量。 每个坐标使用相同类型的规则独立演化：以素数为模的线性变换。 

对于每个索引$i$，值$f_i^{(k)}$仅取决于其先前的值$f_i^{(k-1)}$通过仿射函数模$p_i$。 我们可以自由选择初始向量和这些仿射函数的所有系数，目的是使整个元组的序列$(f_1^{(k)}, \dots, f_n^{(k)})$在最终重复之前产生尽可能多的不同状态。 

任务是确定可以出现在这样的序列中的不同元组的最大可能数量。 

关键的限制是$n$可以达到$2 \cdot 10^5$，因此任何解都必须本质上是线性的$n$。 每个$p_i$是一个素数直到$2 \cdot 10^6$，这强烈表明我们永远不会迭代取模的值$p_i$，而是依赖于更新规则的代数结构。 

简单的模拟会尝试推理所选参数的每个坐标的周期。 这会立即失败，因为每个坐标最多有$p_i$状态，以及所有的产物$p_i$是一个天文数字。 

如果假设任意线性递归产生必须单独分析的复杂循环结构，则会出现更微妙的失败情况。 例如，人们可能会尝试计算每个仿射函数的循环长度，然后将它们组合起来，但错误地假设最大循环长度取决于两者$a_i$和$b_i$以一种复杂的方式。 实际上，素数域上的结构足够严格，我们可以强制实现最大循环。 

核心困难是认识到我们不是在分析固定的发电机。 我们选择的发生器是为了最大化轨道尺寸。 

## 方法

 每个坐标在地图下独立演化$$x \mapsto a_i x + b_i \pmod{p_i}$$在素数大小的有限域上。 所以每个坐标都是一组大小的函数$p_i$，整个系统是独立动力系统的产物。 

暴力心理模型是选择参数并模拟诱导的周期长度。 对于每个坐标，我们可以枚举所有的选择$a_i, b_i, x_i$，模拟直到重复，并计算轨道大小，然后尝试将它们组合起来。 这是完全不可行的，因为即使是一个坐标也已经有了$O(p_i)$状态，参数选择的数量为$O(p_i^2)$，所以总工作量立即爆炸。 

关键的观察是素数域上的仿射变换的最大可能轨道大小并不复杂。 该映射要么是排列，要么不是排列，当它是排列时，它的循环分解可以做成大小的单个循环$p_i$。 

在素数域上，任何形式的函数$x \mapsto x + c$和$c \neq 0$是所有元素的循环移位。 迭代它遍历所有$p_i$重复之前的值。 这已经给出了单个坐标的最大可能周期，因为没有一组尺寸的系统$p_i$周期可以长于$p_i$。 

由于坐标独立演化，不同元组的总数等于各个周期长度的乘积。 独立地最大化每个坐标可以最大化乘积。 

所以最优策略是强制每个坐标都有精确的周期$p_i$，答案就成为所有素数的乘积。

 | --- | --- | --- | --- |
 | 每个参数的强力循环推理 | 指数| O(1) | O(1) | 太慢了|
 | 素数观测的最优乘积 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 观察每个坐标在以素数为模的函数下独立演化$p_i$。 这意味着完整的元组演化是独立的一维动力系统的笛卡尔积。 
2. 对于固定坐标$i$，认识到状态空间恰好有$p_i$元素。 重复应用确定性函数产生的任何序列最终都必须重复，因此轨道长度最多为$p_i$。 这给出了每个坐标的绝对上限。 
3. 证明这个上限是可以实现的。 选择$a_i = 1$和$b_i = 1$进行转变$x \mapsto x + 1 \pmod{p_i}$。 每个应用程序都会移动到一个新值，直到访问完所有残基，然后再返回开始。 这产生了一个长度的循环$p_i$。 
4. 得出坐标系的最大可能轨道长度$i$正是$p_i$，因为它不能超过状态的数量，并且这种结构实现了它。 
5. 由于坐标是独立的，因此只有当每个坐标同时重复时，元组才会重复。 因此，完整周期是独立周期中最不常见的同步，这里它恰好是它们长度的乘积。 
6. 全部相乘$p_i$一起在模下$10^9+7$。 

### 为什么它有效

 完整状态空间的演化是独立有限循环的产物，每个坐标一个。 每个坐标周期都可以独立地最大化长度$p_i$，并且任何坐标都不能超过其状态空间大小。 独立性确保当所有坐标同时返回到其起始位置时，元组重复恰好发生，因此不同元组的总数等于循环长度的乘积。 这确定了上限及其可达到性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def main():
    n = int(input())
    p = list(map(int, input().split()))
    
    ans = 1
    for x in p:
        ans = (ans * x) % MOD
    
    print(ans)

if __name__ == "__main__":
    main()
```实现直接应用导出的结果。 唯一维护的状态是所有素数模的运行乘积$10^9+7$。 坐标之间不存在隐藏的依赖关系，因此不需要进一步处理。 

这里的一个常见陷阱是使重现过于复杂并试图分析$a_i, b_i$组合。 最佳结构通过选择保证完整周期的参数来避免所有这些。

## 测试用例```python
import sys, io

MOD = 10**9 + 7

def solve():
    input = sys.stdin.readline
    n = int(input())
    p = list(map(int, input().split()))
    ans = 1
    for x in p:
        ans = (ans * x) % MOD
    print(ans)

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    old_stdout = sys.stdout
    sys.stdout = io.StringIO()
    solve()
    out = sys.stdout.getvalue()
    sys.stdin = old_stdin
    sys.stdout = old_stdout
    return out.strip()

# provided sample
assert run("4\n2 3 5 7\n") == "210"

# minimum size
assert run("1\n2\n") == "2"

# repeated primes
assert run("3\n3 3 3\n") == str((3*3*3) % MOD)

# mixed primes
assert run("2\n2 7\n") == str((2*7) % MOD)

# larger case
assert run("5\n2 3 5 7 11\n") == str((2*3*5*7*11) % MOD)

 ## 边缘情况

 一个微妙的边缘情况是当所有$p_i$等于 2。在这种情况下，每个坐标只有两个状态，并且仿射选择之间的相互作用似乎可以减少周期。 然而，使用$x \mapsto x + 1 \bmod 2$确保每个坐标有 2 个周期，因此完整的元组空间仍然有大小$2^n$，算法正确返回乘积$2^n \bmod (10^9+7)$。 

另一种情况是当$n = 1$。 该系统退化为素数域上的单个仿射映射。 最大轨道正好是$p_1$，算法正确返回$p_1$，符合以下事实：在任何素数大小的有限域上都存在完整的循环移位。
