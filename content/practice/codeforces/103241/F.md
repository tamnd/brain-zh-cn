 ## 解决方案
 ## 问题理解

 我们得到了一份放在书架上的书籍清单。 每本书都有一个单字小写标题和出版年份。 任务是重新排序书架，以便书籍主要按标题的第一个字母按字母顺序分组。 在每个这样的组中，书籍必须按出版年份从旧到新的顺序排序。 

输出不是作为索引或对的排序本身，而只是最终排序顺序中的书名序列，每行一个。 

输入量可达一万本书左右。 这立即告诉我们，$O(n^2)$在最坏的情况下，重复扫描或插入列表的解决方案会太慢。 一个$O(n \log n)$方法是足够的并且是预期的，因为排序占主导地位，我们只需要定义正确的排序键。 

主要的微妙之处在于，排序是自定义意义上的字典顺序：它不是完整的字符串排序，只有第一个字符对于分组很重要，并且在该分组中年份很重要。 

当人们仅按年份或仅按名称排序时，就会出现天真的错误。 

例如，考虑：

 输入：```
3
banana 2000
apple 1990
apricot 1980
```正确输出：```
apricot
apple
banana
```仅按年份排序的简单方法会产生：```
apricot
apple
banana
```这在这里意外地起作用，但在字母不同的情况下失败。 

现在考虑：```
3
banana 1980
apple 2000
apricot 1990
```正确输出：```
apple
apricot
banana
```如果我们忽略第一个字母，按年份排序会得到：```
banana
apricot
apple
```这是不正确的，因为按首字母分组是强制性的。 

另一个微妙的问题是年数相等时的稳定性。 如果两本书具有相同的首字母和相同的年份，则声明中没有明确定义它们的相对顺序，因此任何一致的平局（例如按名称的字典顺序）都是可以接受的，但我们必须确保确定性。 

## 方法

 暴力的想法是通过重复扫描列表并根据规则选择下一个最小的有效书籍来模拟手动排序：第一个字母优先，然后是年份。 这类似于自定义比较器上的选择排序。 这是正确的，因为在每一步中我们都明确地按顺序选择下一个有效元素，但每次选择都会花费$O(n)$，并为所有人做这件事$n$位置导致$O(n^2)$运营。 和$n = 10^4$，这变成了大约$10^8$比较，在典型约束下，Python 中的比较处于临界状态或太慢。 

关键的观察结果是，排序关系是由一个小元组定义的严格全序：字符串的第一个字符，然后是年份，最后是可选的用于打破平局的完整标题。 一旦我们认识到这一点，问题就简化为标准的排序任务。 排序算法在内部有效地处理排序$O(n \log n)$，Python 的 Timsort 针对此类结构化键进行了优化。 

因此，我们不是模拟决策，而是将规则编码为排序键并将排序委托给标准库。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力选择 |$O(n^2)$|$O(1)$额外 | 太慢了 |
 | 用键排序 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ### 1.将所有书籍读入内存

 每个条目都存储为由标题及其出版年份组成的对。 这使我们能够计算排序键而无需重复输入访问。 

### 2.定义一个捕获排序规则的排序键

 对于每本书，我们构建一个元组`(first_letter, year, title)`。 第一个字母强制分组，年份强制每个组内的时间顺序，标题充当确定性的平局打破者。 

### 3. 使用键对列表进行排序

 我们使用定义的元组对所有书籍应用标准排序。 排序算法将按字典顺序比较元组，这与所需的排序完全匹配。 

### 4.仅输出按排序顺序的标题

 排序后，我们按顺序打印每个标题，忽略年份，因为它仅用于排序。 

### 为什么它有效

 正确性来自于这样一个事实：所需的排序是通过字典顺序比较来完整描述的。`(first_letter, year, title)`。 这定义了一个全序，意味着每一对书都是可比较的并且传递一致性成立。 排序算法保证产生与该比较器一致的序列，因此结果列表正是所需的货架排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    books = []
    
    for _ in range(n):
        name, year = input().split()
        year = int(year)
        books.append((name, year))
    
    books.sort(key=lambda x: (x[0][0], x[1], x[0]))
    
    for name, _ in books:
        print(name)

if __name__ == "__main__":
    main()
```关键的实现细节是排序键。 我们明确使用`x[0][0]`提取标题的第一个字符，确保按首字母分组。 第二个组成部分是年份，强制按时间顺序升序。 最终组件`x[0]`当前两个字段相等时稳定关系。 

一个常见的错误是完全忘记第一个字母分组或首先按完整字符串排序，这会改变排序的预期结构。

 ## 边缘情况

 当多本书共享相同的起始字母和相同的出版年份时，就会出现微妙的边缘情况。 如果没有第三个决胜局，不同的排序运行可能会产生不同的有效排列，这在确定性输出问题中是不希望的。 

例如：```
3
booka 1999
bookb 1999
bookc 1999
```该算法使用完整标题作为最终密钥，因此按照字典顺序执行，确保输出一致：```
booka
bookb
bookc
```另一个边缘情况是所有书籍都以不同的字母开头。 首字母规则完全决定顺序，年份没有任何影响。 该算法仍然有效，因为元组比较自然会优先考虑第一个字符。 

最后，当所有书籍共享相同的首字母时，问题就简化为按年份排序，并且实现会自动崩溃为该行为，无需特殊的大小写。
