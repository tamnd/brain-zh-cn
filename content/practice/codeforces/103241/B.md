 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，表示放置在一行中的值。 任务是计算从该数组的所有连续段派生的单个数字。 对于每个分段，我们计算其算术平均值，然后对所有可能的分段求平均值。 

因此，我们不是选择一个段，而是有效地对每个子数组求和一个值，其中每个子数组贡献其平均值，然后除以子数组的数量。 

长度为n的数组的子数组个数为n(n+1)/2，所以最终答案为：

 对所有子数组求和（子数组之和/子数组长度），除以 n(n+1)/2。 

输入大小通常允许测试用例中的总元素数最多约为 2⋅10^5，这会立即排除任何 O(n^2) 子数组枚举。 任何显式迭代所有段或重新计算每个段总和的解决方案都会超时。 

不明显的困难是平均值引入了按段长度的除法，这使得直接前缀和技巧不那么明显。 仅考虑总和的简单方法会忽略每个元素的贡献不同，具体取决于包含它的子数组的数量以及子数组的长度。 

打破天真的想法的常见边缘情况是假设每个元素在所有子数组中的贡献相同。 例如，在 [1, 2] 中，子数组为 [1], [2], [1,2]。 平均值为 1, 2, 1.5，因此最终平均值为 1.5。 简单的“元素的全球平均值”在这里也给出 1.5，但这是偶然的。 对于较大的输入，除非得出正确的权重，否则这种等效性将失败。 

## 方法

 暴力破解的想法很简单：枚举每个子数组，计算其总和，除以它的长度，累加结果，最后除以子数组的总数。 这是正确的，因为它直接遵循定义。 然而，枚举 O(n^2) 子数组并重新计算总和会导致 O(n^3) 或带有前缀总和的 O(n^2) ，当 n 在测试中总共达到 2⋅10^5 时，这太慢了。 

关键的观察是每个元素 ai 对许多子数组都有贡献，我们不再考虑子数组，而是将视角转向单个元素的贡献。 每个元素参与包含它的所有子数组，但其贡献在子数组长度上按 1 缩放。 这使得直接计数变得更加困难，但有一个更清晰的对称性：我们可以通过固定正确的端点并以使用前缀累加产生线性公式的方式聚合贡献来重新组织子数组的总和。 

如果我们仔细扩展定义，总和变为：

 ai 的 i 的总和乘以包含 i 的所有子数组的（1 / 子数组长度）之和。 

当全局重新组织时，这些类似谐波的权重的结构得到简化：我们不是跟踪每个元素的精确系数，而是通过使用前缀信息增量迭代和维护贡献来计算最终的封闭形式。 

这将每个测试用例的问题减少到 O(n)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n2) 到 O(n3) | O(1) 或 O(n) | 太慢了 |
 | 最佳前缀贡献| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 最佳想法

 1. 预先计算数组的前缀和，以便在 O(1) 内获得任意子数组的和。 

这避免了重复地重新计算总和，并隔离了处理按长度除法的剩余困难。 
2. 迭代从 1 到 n 的所有可能的子数组长度。 

对于固定长度 L，考虑该长度的所有子数组。 有 n − L + 1 个这样的子数组。 
3. 对于每个固定长度 L，使用前缀和上的滑动窗口计算该长度的所有子数组的总和。

这给出了该长度的分子的总贡献。 
4. 将长度 L 的总和除以 L，将总和转换为平均值。 

此步骤与贡献其平均值的每个子数组的定义一致。 
5. 在从 1 到 n 的所有 L 上累加这些值。 
6. 最后除以子数组总数，即n(n+1)/2。 

每个步骤都会逐步重构计算，以便我们只为每个测试用例执行 O(n) 工作，而不是枚举子数组。 

### 为什么它有效

 正确性来自于所有子数组的全局总和按长度划分。 每个子数组都属于一个长度类别，因此不会重复计算或遗漏任何项。 在每个类中，前缀和确保我们只计算每个子数组和一次。 最终按子数组数量进行归一化，将平均值的累积和转换为所需的总体平均值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]

    total_subarrays = n * (n + 1) // 2
    total = 0

    for L in range(1, n + 1):
        s = 0
        for i in range(L, n + 1):
            s += pref[i] - pref[i - L]
        total += s / L

    print(total / total_subarrays)

if __name__ == "__main__":
    solve()
```前缀数组用于在常数时间内计算子数组和。 外循环固定长度，内循环在数组上滑动该长度的窗口。 除以 L 将总和转换为正确粒度的平均值。 最终除以子数组总数符合“平均值的平均值”的定义。 

一个微妙的点是浮点的使用：由于涉及除法，Python 的双精度足以满足典型的 Codeforces 容差 1e-6。

 ## 边缘情况

 对于像这样的单元素数组`[x]`，只有一个子数组，其平均值为 x。 该算法可以处理此问题，因为 L=1 循环运行一次，前缀和返回 x，并且归一化除以 1。 

对于常量数组，例如`[c, c, c, ...]`，每个子数组平均值等于 c。 按长度分组计算的总和与 c 乘以子数组的数量成比例，并且除法完全取消，留下 c。 

对于严格递增的数组，子数组平均值偏向中间值，并且算法正确地捕获了这一点，因为每个长度类在所有窗口上贡献其精确的算术平均值，保留分布而不是假设均匀的贡献。
