 ## 解决方案
 ## 问题理解

 我们有一个字符串重写系统。 每条规则都描述了单个字符如何扩展为两个字符的字符串。 如果规则说`c → ab`，然后每次我们在包含的字符串中选择一个位置`c`，我们可以用该对替换该字符`ab`。 这会在每次应用时将字符串的长度增加一。 

对于每个测试用例，我们从源字符串开始`s`并想知道是否可以获得目标字符串`t`通过以任何顺序和在任何位置重复应用这些替换规则。 

一个关键的结构属性是每个字符最多有一个传出规则，并且规则图没有循环。 这意味着从任何字符开始，重复扩展最终都会终止，并且永远不会通过重写循环回到自身。 然而，扩展可以分支，因为每个字符可以扩展为两个字符，因此单个字符`s`可以产生整个字符子树`t`。 

这些限制意味着我们无法天真地模拟完全扩展。 一个字符可以重复地将字符串长度加倍，因此在最坏的情况下总的扩展大小是指数级的。 虽然`n ≤ 59`，该结构仍然允许长依赖链。 测试用例中所有字符串的长度总和最多为 3 · 10^5，因此每个测试用例的任何解决方案都必须大致线性或接近线性。 

一个微妙的困难是规则不对称并且扩展是不可逆的。 一旦角色被扩展，我们就无法“合并”回来。 这使得字符串上的贪婪本地匹配变得非常重要。 

一些边缘案例暴露了常见的陷阱。 

如果没有规则的话`s`只能等于`t`。 例如`s = "abc"`,`t = "abc"`是的，但是`t = "a"`是否。 

如果角色扩展但与目标结构不匹配，则过早扩展可能会破坏可行性。 例如，如果`a → bc`但目标开始于`a`在不可能进行反向压缩的位置，天真的贪婪替换会失败。 

另一个棘手的情况是多个扩展相互作用时。 认为`a → bc`,`b → d e`。 扩展`a`首先或扩展`b`首先导致不同的中间形状，但两者必须与相同的最终字符串一致`t`。 

核心挑战是我们将生成的树结构与平面目标字符串进行匹配。 

## 方法

 暴力方法会尝试模拟所有可能的重写`s`直到达到`t`或超过其长度。 每个字符都可以独立扩展，因此状态数量呈指数增长。 即使当长度超过时我们修剪`|t|`，单个角色可以产生许多中间组合，导致组合爆炸。 一旦字符串达到中等长度，这是不可行的。 

关键的观察是，一旦我们确定了每个字符映射到子树的方式，并且规则形成字符上的有向非循环图，该过程就是确定性的。 这建议扭转观点：而不是扩大`s`，我们可以计算出每个字符的含义`s`必须对应于`t`，就目标字符串的段而言。 

由于每个字符恰好扩展为两个子字符，因此每个字符都定义了一个二叉扩展树。 由于该图是非循环的，因此我们可以按逆拓扑顺序处理字符，并将每个字符的完整扩展签名计算为“虚拟字符串”`t`，但我们只关心这些签名是否与`t`。 

这减少了验证是否扩展的问题`s`可以分段以精确匹配`t`，其中每个字符贡献一个递归定义的段。 我们利用每个字符的扩展是固定的这一事实，在字符到间隔匹配上使用记忆或 DP。 

我们可以考虑定义一个函数`match(c, l, r)`意思是是否字符`c`可以准确生成子串`t[l:r]`。 因为每个规则分为两个子规则，所以我们尝试拆分`[l:r]`分成两部分并递归地匹配子项。 由于规则是非循环的，因此记忆超过`(c, l, r)`是安全的。 

然而，我们可以进一步简化。 由于每个字符总是恰好扩展为两个字符，因此我们可以根据叶数预先计算每个字符的完整扩展大小，但更重要的是，我们在消耗目标字符串时直接按需模拟 DFS 扩展。 

我们处理`s`从左到右，并保持一个指针`t`。 中的每个角色`s`必须匹配一个连续的块`t`。 我们递归地扩展一个字符，消耗与`t`根据其子树的要求。 如果在任何时候发生不匹配，我们就会失败。 由于该图是 DAG，因此每个字符扩展都会处理一次，从而使该过程在总扩展大小上呈线性关系，其边界为`|t|`。

 ## 算法演练

 我们将每个角色建模为图中的一个节点，具有零个或一个输出规则，产生两个子节点。 然后，我们验证扩展源字符串是否准确地生成目标字符串。 

1. 建立一个字典映射每个字符`c`到其扩张`(a, b)`如果有一条规则`c → ab`存在。 这可以在 O(1) 中直接访问子级。 
2.我们定义一个递归过程`dfs(c, pos)`试图匹配字符的扩展`c`正好从索引处开始`pos`在目标字符串中。 它返回下一个位置`t`消耗完生成的所有字符后`c`，或者如果发生不匹配则失败。 返回位置的原因是每个子树消耗一个连续的段`t`。 
3.如果`c`没有规则，它必须与其中的一个字符完全匹配`t`，所以我们检查是否`t[pos] == c`。 如果没有，我们就会失败。 否则我们返回`pos + 1`。 这是扩展树的基本情况。 
4.如果`c → ab`，我们首先递归匹配`a`开始于`pos`，获得新职位`mid`。 然后我们匹配`b`开始于`mid`。 这强制扩展顺序为左子节点后跟右子节点，从而保留生成字符串的结构。 
5. 对于初始字符串中的每个字符`s`，我们反复申请`dfs`从当前全局指针开始`t`。 如果在任何时候发生不匹配或越界，我们就得出结论，转换是不可能的。 
6. 处理完所有字符后`s`，我们检查是否已经消耗了所有字符`t`。 如果不是，则转换不完整，因此无效。 

关键思想是每个字符定义一个固定的有序扩展树，并且目标字符串必须恰好是这些树按顺序串联的。 

### 为什么它有效

 扩展规则为每个字符定义一棵有根有序树，因为系统是非循环的并且每个节点最多有一个传出边。 规则的每次应用都会按从左到右的顺序用它的两个子节点替换一个节点，因此最终的字符串正是这个扩展森林的中序遍历。 

DFS 过程强制每个子树消耗目标字符串的连续段。 因为没有循环，所以每个字符扩展都是明确定义的，并且不能无限地重新访问。 记忆或单遍消耗确保我们永远不会重新计算不一致的扩展。 如果在任何时候字符与预期的符号或分段不匹配`t`失败，无法产生有效的扩展序列`t`，因为任何有效的推导都必须遵循相同的树分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    T = int(input())
    
    for _ in range(T):
        n = int(input())
        rule = {}
        
        for _ in range(n):
            line = input().strip()
            a = line[0]
            b = line[3]
            c = line[4]
            # format: a->bc (actually a -> bc)
            rule[line[0]] = (line[3], line[4])
        
        s = input().strip()
        t = input().strip()
        
        idx = 0
        n_t = len(t)
        ok = True
        
        from functools import lru_cache
        
        @lru_cache(None)
        def dfs(ch, pos):
            if pos > n_t:
                return -1
            
            if ch not in rule:
                if pos < n_t and t[pos] == ch:
                    return pos + 1
                return -1
            
            a, b = rule[ch]
            
            mid = dfs(a, pos)
            if mid == -1:
                return -1
            return dfs(b, mid)
        
        cur = 0
        
        for ch in s:
            cur = dfs(ch, cur)
            if cur == -1:
                ok = False
                break
        
        if ok and cur == n_t:
            print("SI")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```该实现首先构建规则表，将每个字符映射到其两个扩展子级。 DFS 函数是核心逻辑：它在尊重扩展结构的同时消耗目标字符串。 

记忆化确保同一位置的同一字符的重复扩展不会被重新计算，这一点很重要，因为`s`可以扩展成相同的子树。 

指针`cur`强制执行全局串联顺序。 中的每个角色`s`必须完全消耗一个连续的段`t`，否则该结构无效。 

最后检查`cur == len(t)`保证目标中没有剩余字符保持不匹配。

 ## 边缘情况

 当一个字符没有扩展规则但出现在多个字符间接扩展到该字符的上下文中时，就会出现一种边缘情况。 例如，如果`c`是终端但出现在子树深处，DFS 正确地强制目标位置处的字符完全相等。 任何不匹配都会立即停止扩展，从而防止部分接受。 

另一个边缘情况是深链，例如`a → bc`,`b → de`,`d → fg`，其中扩展深度较大。 递归可以安全地处理这个问题，因为每次调用至少消耗一个字符`t`，并且记忆可以防止重复计算相同的`(character, position)`状态，确保线性行为。 

最后一个边缘情况是剩余的字符`t`。 即使所有扩展在本地都成功，全局指针检查也会确保中没有额外的后缀`t`仍然未使用。 这可以捕获扩展产生的字符串比所需的短的情况，例如缺少规则或终端扩展不足。
