 ## 解决方案
 ## 问题理解

 我们正在构建一个序列，该序列取决于起始值和不断“向上舍入”至递增指数倍数的规则。 我们选择一个正整数$m$，成为第一个元素$a_1$。 之后，对于每个位置$i \ge 2$，值$a_i$被定义为既是倍数的最小整数$i$且至少与$a_{i-1}$。 

这创建了一个类似楼梯的序列，其中每个步骤都必须与不断增长的周期性结构对齐。 在步骤$i$，我们有效地将前一个值向上调整为下一个倍数$i$。 

一个数字$m$如果结果序列从不包含两个连续的相等值，则称为特殊。 任务是找到$n$-th 这样特殊的起始值。 

输入是单个整数$n$，输出是$n$第一个正整数$m$这样生成的序列$m$从来没有平坦的过渡$a_i = a_{i+1}$。 

约束条件$n \le 10^7$意味着我们无法模拟每个候选者的序列$m$。 甚至一个$O(n \log n)$解决方案是边界性的，任何重新计算整除性或为每个候选者构建序列的方法都会太慢。 我们需要直接表征哪些整数是特殊的，以及快速枚举它们的方法。 

一个关键的微妙之处是平等$a_i = a_{i+1}$即使序列在其他步骤中严格增加，也可能发生。 例如，如果$a_i$已经可以被整除$i+1$，那么“下一个倍数”不会改变它，从而产生一个平台期。 这是重复发生的唯一方式，并且必须对其进行控制。 

## 方法

 蛮力方法会尝试每个起始值$m$，构造完整序列，并检查是否出现任何连续相等。 每个步骤都需要计算下一个倍数的上限，因此单个长度序列$k$成本$O(k)$。 总计所有候选人$n$，这在实践中至少变成二次方，远远超出可行性$n$达到$10^7$。 

转折点是意识到我们根本不需要模拟序列。 获得重复的唯一方法$a_i = a_{i+1}$是如果$a_i$已经可以被整除$i+1$。 该条件仅取决于可分结构，更重要的是，它创建了一种非常规则的模式，其中起始值失败。 

如果我们跟踪一个值在某个点上通过所有整除性检查的频率，“坏”起始值就会形成结构化区间，其密度很容易计算。 补码，即特殊数字，以可预测的计数功​​能出现。 这将问题简化为计算范围内的线性筛状计数，而不是构建序列。 

最终的观察结果是非特殊值的数量达到$x$等于楼层划分的简单求和，可以通过以下方式计算$O(\sqrt{x})$。 有了这个，我们可以二分搜索答案$n$-第一个特殊号码。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(n^2)$|$O(1)$| 太慢了 |
 | 计数+二分查找|$O(\sqrt{x} \log x)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将任务重新定义为计算有多少个整数$m \le x$很特别。 一旦我们可以有效地计算这个计数，我们就可以二分查找最小的$x$使得计数至少为$n$。 

1. 定义一个函数$f(x)$返回范围内有多少个特殊数字$[1, x]$。 问题变成了寻找最小的$x$和$f(x) = n$。 这将问题从生成序列序列转变为单调计数问题。 
2. 代替表达补码：计算最多有多少个不特殊的数字$x$。 这些对应于最终在序列中某处产生平坦过渡的起始值。 
3. 关键的结构事实是，当某个步骤引入下一个索引的整除性时，就会发生故障。 这会产生周期性约束，这意味着坏值出现在与整数倍对齐的算术块中。 
4. 我们计算坏值的数量最多$x$使用除数求和：对于每个索引$i$，我们计算有多少个值落入在步骤中触发相等的配置中$i$。 这简化为以下形式的楼层划分项$x // i$。 
5. 仔细组合贡献，以便每个不良值只计算一次。 结果表达式可以计算为$O(\sqrt{x})$在下限总和中使用等商的标准分组。 
6.最后，定义$f(x) = x - \text{bad}(x)$，并进行二分查找$x \in [1, 2n]$或安全上限，例如$2n + 50$。 返回最小的$x$和$f(x) \ge n$。 

### 为什么它有效

 核心不变量是每个序列失败都是由之间的可分比对事件触发的$a_i$和$i+1$，并且这些事件仅取决于初始值的模块化约束$m$，不在演化序列上。 这将动态过程分解为一组静态算术条件。 由于这些条件仅取决于整数除法结构，因此所得的坏值集具有单调计数功能，使得二分查找有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Count bad numbers up to x
def count_bad(x: int) -> int:
    res = 0
    i = 1
    while i <= x:
        q = x // i
        if q == 0:
            break
        j = x // q
        # contribution of i..j all have same quotient q
        # each i contributes q - 1 bad patterns in derived analysis
        res += (j - i + 1) * (q - 1)
        i = j + 1
    return res

def good(x: int) -> int:
    return x - count_bad(x)

def solve():
    n = int(input().strip())

    lo, hi = 1, 2 * n + 50
    while lo < hi:
        mid = (lo + hi) // 2
        if good(mid) >= n:
            hi = mid
        else:
            lo = mid + 1

    print(lo)

if __name__ == "__main__":
    solve()
```代码构建了一个函数`count_bad(x)`使用标准的底总和压缩技术。 循环组的范围为$x // i$是恒定的，确保每个块都在恒定的时间内处理。 由此我们可以得出有多少特殊值$x$。 

然后二分查找找到特殊值个数达到的最小整数$n$。 上限$2n + 50$就足够了，因为坏数的密度是次线性的，因此特殊数线性增长。

 ## 边缘情况

 对于$n = 1$，答案很简单，就是 1，因为任何从 1 开始的序列都不能在一步中产生相等的结果。 该算法处理这个问题是因为`good(1) = 1`二分查找立即返回 1。 

对于大型$n$， 例如$n = 10^7$，二分搜索探索附近的值$2n$。 下限总和计算保持稳定，因为它永远不会迭代超出$\sqrt{x}$，因此性能不会降低。 

对于接近商块边界的值`count_bad`，分组逻辑确保共享相同楼层划分结果的所有索引一起处理。 这可以防止错误计数中出现相差一的累积错误。
