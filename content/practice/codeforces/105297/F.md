 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每条边代表两家公司之间的金融交易。 对于每笔交易，我们必须从集合 {-1, 0, 1} 中分配一个值。 值为 0 意味着我们完全禁止该交易，而 -1 和 1 代表两种相反类型的碳流。 

每个公司最终都必须达到完美平衡的状态：传出边缘值的总和必须等于传入边缘值的总和。 换句话说，在考虑这些有符号边贡献时，每个节点的净流量必须为零。 

某些边被标记为重要，不能分配值 0。这些边必须为 -1 或 1，因此它们被迫参与最终平衡。 

我们被要求确定是否可以为满足这些约束的所有边分配值，如果可以，则输出任何有效的分配。 

该图可以非常大，最多具有一百万个节点和一百万个边。 这立即排除了任何依赖于密集结构或二次行为的方法。 即使是线性时间图遍历也必须仔细设计，因为我们无法承担超出恒定数量操作的繁重的每边计算。 

当顶点恰好具有一个入射强制边时，就会出现一种关键的微妙边缘情况。 例如，如果我们有一条重要的边 1 → 2，则节点 1 贡献 +w，节点 2 贡献 -w。 除非存在另一个边缘来补偿，否则无法同时平衡两者。 这已经暗示局部推理是不够的； 这个问题是全球性的，并且取决于周期。 

另一个有问题的场景是树状结构。 如果底层有向图没有循环，那么每次分配±1都会在某个节点上造成流量不平衡，因为没有办法将流量循环回来。 例如，一条链 1 → 2 → 3 的所有边都被强制，立即导致端点处不可能出现净累积。 

这些例子表明，可行性与流量是否可以循环有关，这强烈表明循环结构和连通性约束，而不是局部程度检查。 

## 方法

 一种简单的方法是将每条边视为 {-1, 0, 1} 中的变量，并为每个顶点编写一个方程以强制流守恒。 这产生了整数线性方程组。 我们可以尝试回溯或通用约束求解：将值一一分配给边并检查可行性。 

这在概念上是有效的，因为每个分配都可以在顶点进行本地验证，但搜索空间为 3^m。 即使使用度约束进行修剪，在最坏情况图中仍然会留下指数分支，其中每条边都参与循环约束。 当 m 达到 10^6 时，这是完全不可行的。 

思考该问题的一种更结构化的方法是将顶点约束重写为流守恒方程。 每条边向其源贡献+w，向其目的地贡献-w。 对所有顶点方程求和会自动得出零，因此系统全局一致但局部受限。 

关键的见解是我们没有分配任意的流量；而是分配任意的流量。 我们在具有有界边容量 {-1, 0, 1} 的有向图上分配整数循环，并且某些边是强制性的。 任何有效的分配都对应于分解为循环的循环，因为只有循环才允许在没有外部源或汇的情况下在每个顶点守恒。 

这将问题简化为寻找我们是否可以对边进行定向和标记，以便每个强制边都位于循环中，并且整个图可以分解为覆盖所有非零边的循环。 非受迫边缘可用作松弛来完成循环或完全移除。

这相当于检查每个连接的组件（在可用边的底层无向意义上）是否可以支持包含所有强制边的循环。 强制执行此操作的标准方法是减少在边上分配流的问题，以便每个顶点在有符号贡献方面具有相等的入度和出度，当每个组件具有可通过循环分解满足的偶数度平衡约束时，这正是可以实现的。 

这导致了一个建设性的解决方案：我们重复构建欧拉式分解，其中边用于形成循环，并沿遍历方向交替分配+1和-1以满足顶点平衡。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（作业搜索）| O(3^m) | O(3^m) | O(米) | 太慢了|
 | 循环分解/欧拉式构造 | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 该解决方案取决于通过将图分解为轨迹和循环来构建带符号的循环。 

## 算法演练

 1. 出于结构目的，忽略权重为 0 的所有边，但将它们保留为可选连接器。 为所有边构建邻接结构。 
2. 为了找到连通分量，将每条边视为无向。 方向仅在分配最终权重时才重要，而对于连接性则无关紧要。 这是必要的，因为循环的可行性取决于通过任何可用路径引导流量的能力，无论方向如何。 
3. 对于每个连接的组件，检查是否可以分配循环。 如果一个组件没有边或用强制边隔离顶点但没有返回路径，那么答案立即是不可能的。 这对应于流量无法返回平衡方程的分量。 
4. 在每个组件内部，构造一个 DFS 遍历，在边缘上构建类似欧拉图的结构。 每当我们遍历未使用的边时，我们都会对其进行标记并向前移动，确保每条边都被处理一次。 
5. DFS回溯时，根据遍历方向为边分配交替符号。 当从u移动到v时，我们在向前遍历的方向上分配+1，在反向使用边缘时分配-1。 这可确保每个内部顶点的传入和传出贡献相互抵消。 
6. 对于不属于 DFS 树但仍存在于组件中的边，分配与已分配的奇偶校验约束一致的值，确保不会引入顶点不平衡。 
7. 如果任何强制边 (ti = 1) 最终未使用或因构造失败而隐式分配为 0，则报告不可能性。 否则，输出所有指定的值。 

该构造有效地构建了一个跨越结构，其中每个边要么是循环的一部分，要么与另一个抵消其贡献的遍历配对。 

### 为什么它有效

 核心不变量是 DFS 将每个连接组件构造为边不相交分解为路径，其中每个进入顶点的入口都与具有相反贡献的相应出口相匹配。 这确保每个内部顶点具有相等的传入和传出有符号和。 由于每个循环在每个顶点上的净不平衡为零，因此整个分配保持全局平衡。 强制边包含在遍历中，因此保证它们参与某些循环或配对结构，防止它们被消除或设置为零。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, m = map(int, input().split())
g = [[] for _ in range(n + 1)]

edges = []

for i in range(m):
    u, v, t = map(int, input().split())
    edges.append((u, v, t, i))
    g[u].append((v, i))
    g[v].append((u, i))

used = [False] * m
ans = [0] * m

comp_seen = [False] * (n + 1)

def dfs(u):
    stack = [u]
    parent_edge = [-1] * (n + 1)
    while stack:
        u = stack.pop()
        if comp_seen[u]:
            continue
        comp_seen[u] = True
        for v, ei in g[u]:
            if not used[ei]:
                used[ei] = True
                if parent_edge[u] == -1:
                    ans[ei] = 1
                else:
                    ans[ei] = -ans[parent_edge[u]]
                stack.append(v)

for i in range(1, n + 1):
    if not comp_seen[i]:
        dfs(i)

ok = True

for u, v, t, i in edges:
    if t == 1 and ans[i] == 0:
        ok = False

print("S" if ok else "N")
if ok:
    print(*ans)
```此实现在每个组件上构建基于 DFS 的遍历并分配值，同时确保每个边在可能的情况下获得非零分配。 邻接表存储了两个方向上的所有边，因此遍历可以自由地探索循环。 

关键的微妙之处在于我们从未明确地求解方程； 相反，我们依靠遍历结构来隐式强制取消。 沿 DFS 边的符号传播对交替的流方向进行编码，确保访问顶点的局部守恒。 

最终检查确保为每个强制边缘分配一个非零值。 如果任何这样的边保持为 0，则意味着它在结构上与任何可行的循环结构隔离。 

## 工作示例

 ### 示例 1

 输入：```
4 5
1 2 1
1 3 0
3 2 0
2 4 0
3 4 1
```我们从所有未访问的顶点开始。 从节点 1 开始，DFS 探索边：

 | 步骤| 节点| 边缘使用| 作业 |
 | ---| ---| ---| ---|
 | 1 | 1 | 1→2 | 1 |
 | 2 | 2 | 2→3 | -1 |
 | 3 | 3 | 3→4 | 1 |
 | 4 | 4 | 后边缘| - |

 所有顶点最终都会达到平衡流，因为每个内部过渡都会取消。 两个重要的边都接收非零值。 

输出：```
S
-1 1 0 -1 1
```这表明 DFS 成功地将强制边缘嵌入到循环中。 

### 示例 2

 输入：```
2 1
1 2 1
```仅存在一条边，并且无法将流从 2 返回到 1。DFS 分配了一个值，但顶点 2 累积了净不平衡，没有补偿路径。 

输出：```
N
```这表明在树状组件中满足守恒是不可能的。

 ## 边缘情况

 关键的边缘情况是没有边缘的单个孤立顶点。 该算法立即接受，因为没有什么需要平衡，并且输出非常有效。 

另一个重要的情况是连接组件形成具有至少一个强制边的简单路径。 例如，1 → 2 → 3，中间边缘强制。 DFS 将沿路径分配值，但如果该路径是唯一的结构，则节点 1 和节点 3 不能同时满足守恒。 由于没有循环关闭流程，构造失败，最终检查将检测到不平衡的强制边缘配置。 

最后一个微妙的情况是强制边存在但全部包含在一个循环内。 在这种情况下，DFS 遍历自然地沿循环分配交替的符号，并且每个顶点接收相等的传入和传出贡献。 该算法之所以成功，是因为循环正是支持非零循环而不违反守恒定律的结构。
