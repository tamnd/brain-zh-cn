 ## 解决方案
 ## 问题理解

 我们有几个独立的游戏，每个游戏都由多个整数区间组成。 每个间隔代表一组当前“活动”的整数，从所有整数开始$l_i$到$r_i$。 还有一个固定的乘数$p > 1$。 两名玩家交替移动，每次移动时，玩家选择一个间隔并以特定方式缩小它。 

间隔移动的原理如下。 假设当前间隔包含当前最小值和最大值之间的所有整数。 玩家必须选择一个值$x$从区间内部这样乘以之后$p$，结果至少是该间隔的当前最大值。 一次$x$被选择，一切大于或等于$x$被删除，所以间隔变为$[l, x-1]$。 

当没有间隔允许有效的移动时，游戏结束。 不能移动的玩家就输了。 我们必须确定第一个玩家对于每个测试用例是否都有获胜策略。 

约束条件很大，最多可达$2 \cdot 10^5$总计间隔和值高达$10^9$。 这立即排除了任何每次移动模拟或间隔的状态空间探索。 甚至迭代所有有效的选择$x$每个间隔内的速度太慢，因为一个间隔最多可以包含$10^9$值，每一步都取决于全局条件，涉及$p$。 

当间隔很短或当$p$很大。 例如，如果一个区间是$[5, 5]$，根本不可能移动，因为没有$x$满意的$x \cdot p \ge 5$除非$p = 1$，这是不允许的。 另一个极端情况是当$p$足够大，甚至可以选择$x = r$不足以满足条件，使间隔立即终止。 这些情况很重要，因为它们贡献零步，但仍然影响游戏结果。 

## 方法

 直接模拟将尝试通过重复选择有效间隔并尝试所有可能的方式来玩游戏$x$选择。 原则上这是可行的，因为游戏是有限的，每一步都会严格减少间隔，但它会立即在规模上失败。 每个区间可以生成许多可能的状态，分支因子实际上是有效的数量$x$，它可以与间隔大小呈线性关系。 即使一个测试用例也可能需要处理大量转换，使得这种方法完全不可行。 

关键的观察是，每个区间上的游戏实际上并不是关于所有可能的值，而只是关于在区间变得太小而无法移动之前可以“压缩”多少次。 一旦间隔达到当前最大值$r$和最小值$l$，唯一有意义的问题是我们是否可以减少$r$重复步骤中明显较小的东西，以及每个间隔对整个游戏有多少强制减少的贡献。 

更深入地看待这一举动的方式是选择$x$相当于选择一个新的上限$x-1$，但约束$x \cdot p \ge r$力量$x$至少是$\lceil r / p \rceil$。 这意味着最小的合法选择正是$x = \lceil r / p \rceil$，选择更大的东西只会删除更多的元素，而不会增加未来的选择。 因此，最佳游戏总是选择最小的有效值$x$，将区间变为$[l, \lceil r/p \rceil - 1]$。 

这将每个间隔减少为确定性过程：重复替换$r$经过$\lfloor (r-1)/p \rfloor$（或者等效地使用阈值缩小$\lceil r/p \rceil$）直到它变得小于或等于$l$。 每次替换恰好对应一次移动。 因此，每个间隔都贡献明确定义的移动次数，并且整个游戏成为所有间隔上独立移动计数的总和。 获胜者由总步数的奇偶性决定。

 ## 算法演练

 我们独立处理每个间隔并计算它贡献了多少动作。 

1.从间歇开始$[l, r]$。 我们感兴趣的是可以执行多少次有效的收缩操作，直到没有任何移动为止。 每个操作都会严格减少当前上限，因此该过程必须终止。 
2. 当间隔仍然有效时，计算最小有效值$x$，即$x = \lceil r / p \rceil$。 该值表示我们被允许切割的最早点。 
3.如果$x \le l$，那么就不可能再进行任何移动，因为任何有效的剪切都会删除整个区间或违反约束。 在这种情况下，停止计算该间隔的移动次数。 
4. 否则，执行一步，并且间隔有效地变为$[l, x-1]$。 更新$r$到$x-1$，因为一切都高于或等于$x$被删除。 
5. 重复该过程直至终止，累计该时间间隔的移动次数。 
6. 对所有时间间隔内的移动计数求和。 若总步数为奇数，则先行者获胜； 否则，第二个玩家获胜。 

关键的简化是强制最优选择。 与最小有效值的任何偏差$x$只会更积极地缩短间隔，并不会增加未来的移动次数，因此无法提高玩家的结果。 

### 为什么它有效

 游戏分解得很干净，因为间隔不相互作用，并且每次移动仅取决于单个间隔的当前最大值。 对于固定间隔，状态由其当前上限完全描述$r$， 自从$l$永远不会改变。 过渡从$r$到$\lceil r/p \rceil - 1$在最佳游戏下是确定性的，因为所有选择都受最小有效削减支配。 这使得每个间隔的移动次数被唯一定义。 

由于玩家在整个游戏中交替移动，并且每次移动恰好减少一个间隔，因此游戏相当于一个类似 Nim 的堆，其中每个间隔贡献的堆大小等于其强制减少的数量。 获胜条件简化为计算总步数的奇偶性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def moves(l, r, p):
    cnt = 0
    while r >= l:
        x = (r + p - 1) // p  # ceil(r / p)
        if x <= l:
            break
        cnt += 1
        r = x - 1
    return cnt

t = int(input())
for _ in range(t):
    n, p = map(int, input().split())
    total = 0
    for _ in range(n):
        l, r = map(int, input().split())
        total += moves(l, r, p)
    print("First" if total % 2 == 1 else "Second")
```代码直接实现了上面描述的归约过程。 辅助函数模拟单个区间的强制收缩，总是使用上限除法跳转到最小的合法分割点。 当无效时循环停止$x$存在，当阈值低于或等于时就会发生$l$。 

重要的实现细节是计算$\lceil r/p \rceil$安全地使用整数算术。 更新内容$r = x - 1$保留间隔保持连续的不变性并正确表示剩余的有效值。

import sys, io

def solve():
    import sys
    input = sys.stdin.readline

    def moves(l, r, p):
        cnt = 0
        while r >= l:
            x = (r + p - 1) // p
            if x <= l:
                break
            cnt += 1
            r = x - 1
        return cnt

    t = int(input())
    for _ in range(t):
        n, p = map(int, input().split())
        ans = 0
        for _ in range(n):
            l, r = map(int, input().split())
            ans += moves(l, r, p)
        print("First" if ans % 2 else "Second")

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    old = sys.stdout
    sys.stdout = out
    solve()
    sys.stdout = old
    return out.getvalue().strip()

 ## 边缘情况

 一个关键的边缘情况是当$l = r$。 在这种情况下，区间只有一个元素，并且永远无法满足移动的条件，因为任何选择$x = l$需要$l \cdot p \ge l$，这是正确的，但结果操作将尝试删除所有元素$\ge x$，留下一个空的间隔，使其无法继续。 该算法正确地处理了这个问题，因为$x \le l$立即触发终止，贡献零步。 

另一个重要的情况是当$p$非常大。 例如，如果$p > r$， 然后$\lceil r/p \rceil = 1$，所以除非$l = 1$，无法移动。 该算法在第一次迭代中检测到这一点，因为$x \le l$，确保间隔被正确分类为终端。 

第三种微妙的情况发生在间隔很宽但$p$很小。 重复的减半或减少仍然很快收敛，因为每次迭代都会将上限减少至少一个因子$p$，因此循环不能循环或增长。
