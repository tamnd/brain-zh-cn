 ## 解决方案
 ## 问题理解

 我们给出的时间表是$n$天。 有些日子是固定休息日（法定节假日）。 所有其他日期最初均为工作日。 我们可以将额外的工作日转换为休息日，这些转换后的天数称为带薪休假。 法定节假日和带薪休假日与休息日的行为相同。 

目标是修改时间表，以满足连续工作日的两个结构性限制。 首先，你永远不能拥有超过$x$连续工作日。 其次，考虑将工作日分为左右两侧的任何休息日。 如果有$a$在此之前的连续工作日以及$b$紧随其后，然后是总和$a + b$不得超过$y$。 

我们希望实现有效的时间表，同时最大限度地减少转换为带薪休假的工作日。 

尽管范围很大，但按键输入结构很简单$n$：我们只关心连续的固定休息日之间的距离，包括第一个休息日之前的前缀和最后一个休息日之后的后缀。 在每个这样的间隔内，一切都开始正常工作，我们决定在哪里插入额外的休息日。 

约束条件$n \le 10^{18}$立即排除任何日常模拟。 我们甚至无法显式地表示完整的数组。 固定假期数$m \le 2 \cdot 10^5$表明任何解决方案都必须在线性时间内起作用$m$，加上每个间隙的常数。 

如果我们尝试一种幼稚的贪婪策略，只强制执行$x$- 本地限制。 该方法可能会产生违反$a+b \le y$整个休息日的状况。 

例如，如果$x = 5$,$y = 7$，并且我们贪婪地将一个长段分割成大小为 5 的块，我们可能会得到相邻的段$5$和$5$休息分开，这违反了$5+5 \le 7$。 一个简单的解决方案，只能确保“没有段超过$x$”会完全怀念这个。

 如果我们试图总是精确地切割每一个，就会发生另一个错误$x$天。 这忽略了有时我们必须缩短较早的部分以使下一个过渡在以下情况下可行$y$-约束。 

## 方法

 强力解释将模拟每一天，维持当前的工作日运行，并且每当违反限制时，插入带薪休假。 这在概念上是正确的，因为它总是立即修复违规，但它是完全不可行的，因为$n$可以是$10^{18}$。 即使明确代表国家也是不可能的。 

关键的观察结果是固定休息日之间的结构是独立的。 两个固定休息日之间的每个连续工作日间隔可以单独求解，答案是所有间隔的总和。 在一个间隔内，我们本质上是将一个长段分割成较小的工作日块，并由选定的休息日分隔开。 

每个生成的块必须满足两个本地规则。 它的长度最多为$x$，并且对于任何相邻的长度块$a$和$b$，我们必须有$a + b \le y$。 自从$x \le y \le 2x$，相邻块之间的相互作用受到严格限制，这就是使结构塌陷成重复模式的原因。 

如果我们总是尝试使每个块尽可能大，那么第一个块自然会变成$x$。 然后下一个块受到两者的约束$x$和$y - x$，所以就变成了$\min(x, y-x)$，这简化为$y-x$因为$y \le 2x$。 之后，该模式重复：一旦我们有了一个大小的块$y-x$，下一个允许的最大值变为$\min(x, x)$，即$x$。 所以大区间内的最优结构在$x$和$y-x$。 

这将每个间隔减少为可预测的交替序列。 问题就变成了计算有多少个满了$x, (y-x)$周期适合间隔长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟 |$O(n)$|$O(1)$| 太慢了 |
 | 区间+交替贪婪打包|$O(m)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理连续固定假期之间的每个航段。 令段的长度（连续工作日数）为$L$。 

1. 单独对待每个部分，因为固定休息日充当强制分隔符，无法删除或移动。 
2.如果$L = 0$，不需要任何工作，因为在此时间间隔内没有任何可安排的内容。 
3.在直接约束下取尽可能大的第一个工作日块，因此第一个块是$a_1 = \min(x, L)$。 放置后，减少剩余长度$L$。 
4. 如果没有剩余，则该时间间隔不贡献带薪休假。 
5. 否则，我们现在被迫安排休息日。 下一个块取决于前一个块$a_1$，其最大可能大小变为$\min(x, y - a_1)$。 在下面$y \le 2x$，这评估为$y-x$，所以第二个块具有固定大小$b = y-x$除非剩余间隔更短。 
6. 如果剩余长度小于$b$，我们只是将其作为最后一个块并停止。 
7. 如果不是，我们减去$b$，从这里开始，模式稳定为重复的块对：

 一块大小的$x$，然后是一个大小的块$y-x$，每人总共消耗$y$工作间隔长度天数。 
8. 计算总长度有多少对完整的$y$使用整数除法拟合剩余的段。 每个完整对贡献两个区块。 
9. 小心地处理剩余的剩余部分，贪婪地以交替模式放置下一个块，确保它永远不会超过剩余的长度。 
10. 此间隔所需的带薪休假天数是块数减一。 

### 为什么它有效

 在任何时间间隔内，我们都会构建一系列由选定的休息日分隔的最大工作块。 每个休息日仅约束其相邻的两个块，并且约束$a+b \le y$强制消除任意块大小的本地依赖性。 因为$y \le 2x$，一旦一个块达到$x$，下一个块被唯一地约束为$y-x$，之后系统返回到相同的状态。 这创建了一个两种状态的确定性自动机，因此始终采用最大有效块的贪婪选择永远不会阻碍未来的可行性，并且总是最小化分裂数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, x, y = map(int, input().split())
    if m:
        holidays = list(map(int, input().split()))
    else:
        holidays = []

    # build gaps
    prev = 0
    ans = 0

    def solve_gap(L):
        if L <= 0:
            return 0

        # first segment
        first = min(x, L)
        L -= first
        segments = 1

        if L == 0:
            return 0

        # now we are at second segment
        b = y - first
        if b > x:
            b = x
        if b <= 0:
            # degenerate, cannot place more blocks, each remaining day must be isolated
            # but constraints ensure y>=x so b>=0 always; still safe guard
            return L  # each day becomes its own segment

        # second segment
        if L <= b:
            return 1  # one extra segment

        L -= b
        segments += 1

        # now pattern cycles: x, (y-x)
        cycle = x + (y - x)

        full_pairs = L // cycle
        segments += full_pairs * 2
        L -= full_pairs * cycle

        # leftover handling
        if L > 0:
            # next expected is x
            take = min(x, L)
            segments += 1
            L -= take

            if L > 0:
                segments += 1

        return segments - 1

    for i in range(m + 1):
        left = holidays[i-1] + 1 if i > 0 else 1
        right = holidays[i] - 1 if i < m else n
        if left <= right:
            ans += solve_gap(right - left + 1)

    print(ans)

if __name__ == "__main__":
    solve()
```该实施将整个时间线压缩为固定假期之间的独立间隙。 每个间隙都会传递给一个助手，该助手计算需要多少额外的休息日才能将其分解为有效的工作段。 

里面`solve_gap`，第一个块始终最大化$x$。 如果差距立即结束，则不需要带薪休假。 否则，我们显式构造第二个块，它受到先前选择的块的约束$y - a$。 在那之后，结构稳定下来，重复完整的长度周期$x + (y-x)$，它允许使用除法在恒定时间内跳过大部分。 

最后的剩余部分由至多两个附加块处理，这已经足够了，因为一旦我们退出完整周期，该模式就是确定性的，不需要进一步的模拟。 

## 工作示例

 ### 示例 1

 考虑长度间隙$L = 17$， 和$x = 5$,$y = 8$。 

| 步骤| 行动| 剩余 L | 段 |
 | --- | --- | --- | --- |
 | 1 | 采取第一个块| 12 | 12 5 |
 | 2 | 取第二个块 (y-x = 3) | 9 | 5, 3 |
 | 3 | 一个完整周期（每个周期 5+3=8）| 1 | 5, 3, 5, 3 | 5, 3, 5, 3 |
 | 4 | 处理余数| 1 | 5, 3, 5, 3, 1 | 5, 3, 5, 3, 1 |

 这会产生 5 个段，因此在此间隔内需要 4 天带薪休假。 

该迹线显示交替结构在第二个块之后立即稳定。 

### 示例 2

 让$L = 6$,$x = 4$,$y = 7$。 

| 步骤| 行动| 剩余 L | 细分 |
 | --- | --- | --- | --- |
 | 1 | 第一块 | 2 | 4 |
 | 2 | 第二个块（y-x = 3，但有上限）| 0 | 4, 2 |

 我们只创建两个段，因此需要一次带薪休假。 

这个案例表明短间隔永远不会达到循环状态。

 ## 边缘情况

 一个关键的边缘情况是根本没有假期。 在这种情况下，整个数组是一个很大的间隙，算法仍然必须正确地将其分割成交替的块。 该解决方案可以处理此问题，因为它将整个范围视为虚拟边界之间的单个间隔。 

另一种情况是当间隙小于$x$。 在这里，算法立即返回零带薪休假，因为不需要拆分，并且自动满足两个约束。 

当出现更微妙的情况时$y = x$。 在这种情况下，相邻块不能同时为非空而不违反总和约束。 该算法自然退化，因为$y-x = 0$，迫使每个块被隔离，这意味着除了第一个间隙之外的每个工作日实际上都成为其自己的部分。
