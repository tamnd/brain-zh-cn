 ## 解决方案
 ## 问题理解

 我们得到从 1 到 n 的数字排列，然后对其运行修改后的选择排序。 对于从左到右的每个位置 i，我们扫描 i 右侧的后缀，并在发现比位置 i 处的当前值更小的元素时进行交换。 每次这样的交换都会增加该轮的计数器 ci。 

关键输出不是最终的排序数组，而是执行此过程时将产生规定的交换计数序列 t1, t2, ..., t(n−1) 的初始排列数。 

重要的观察是，一旦初始排列固定，该算法就是确定性的，因此每个排列恰好产生一个计数器序列。 任务是计算有多少种排列恰好导致了给定的序列。 

约束 n 高达 2⋅10^5 立即排除了任何模拟所有排列过程或尝试简单地重建排列的方法。 即使每个排列的 O(n^2) 也已经太大了，并且对排列的指数搜索显然是不可能的。 该解决方案必须将问题简化为线性或近线性组合结构。 

一个微妙的边缘情况是交换过程在迭代期间修改数组。 例如，从 [4,1,3,2] 这样的后缀开始，固定 i 处的交换次数取决于同一轮中早期交换后值的演变情况。 只计算原始后缀中小于 Pi 的元素的天真的解释是错误的，因为 Pi 会随着交换的发生而变化。 

正确的解释必须跟踪扫描过程中不断变化的“当前最小值”，而不仅仅是静态比较。 

## 方法

 暴力破解的想法很简单：生成每个排列，模拟选择排序变体，记录所有 ci 值，并与目标序列进行比较。 这是可行的，因为算法定义明确，并且每个排列的模拟时间为 O(n^2)。 然而，排列的数量是 n！，所以即使对于 n = 10 这也已经变得不可行，并且对于 n 高达 2⋅10^5 来说这是完全不可能的。 

关键的见解是停止从完整排列的角度思考，而是重新解释 ci 在结构上衡量的内容。 在位置 i 的扫描过程中，算法有效地跟踪后缀上的运行最小值。 每次出现新的较小元素时，它都会成为位置 i 处的新活动值，并且此事件会递增 ci。 

因此 ci 计算后缀中运行最小值减少的次数，不包括初始元素。 换句话说，从左到右读取时，ci + 1 正是序列 P[i..n] 中前缀最小记录的数量。 

这将问题转化为每个后缀必须包含多少记录最小值的全局约束。 一旦以这种方式表达，构造就变成了一个组合放置问题：每个后缀都规定了其中必须出现多少个“新的最小事件”，并且这些约束嵌套在后缀之间。 

我们可以从右到左处理位置。 当我们通过在左侧添加新元素来扩展后缀时，我们决定有多少剩余值将充当该后缀中的新记录最小值。 一旦我们跟踪有多少值仍然“可”分配为最小值，结构就会迫使这些选择在不同位置上是独立的。 这导致乘法计数过程，其中每个位置贡献一个二项式因子，选择哪些元素负责在该后缀级别引入的新最小值事件。

 ## 算法演练

我们从左到右处理约束，同时保留有多少值未使用。 

在每个位置 i，值 ti 告诉我们在后缀 i..n 内必须出现多少个新的“记录最小事件”，超出后缀 i+1..n 已经强制的内容。 由于后缀是嵌套的，因此这些新事件必须对应于从剩余值池中选择元素，这些元素将负责打破恰好在此级别的运行最小值。 

我们维护一个变量 rem，即尚未分配的元素数量，以及使用的运行值，该值跟踪已提交多少元素以充当更深后缀中的记录最小触发器。 

对于从 1 到 n−1 的每个 i，我们计算在该级别必须重新指定多少个元素，并从可用的剩余池中选择它们。 

形式上，在步骤 i 中，使用了可用候选者的数量 rem - ，并且我们必须在其中选择 ti 元素作为在此后缀级别引入的新的最小值触发元素。 每个这样的选择都确定扫描后缀 i 时运行最小值发生新的减少的位置。 

我们使用二项式系数乘以路数并更新已使用元素的数量。 

最后，处理完所有位置后，我们返回对 998244353 取模的乘积。 

### 为什么它有效

 每个后缀约束固定该后缀内运行最小值减少的次数。 这些减少事件完全取决于分配哪些元素作为每个后缀边界处的新最小值。 

由于后缀是嵌套的，因此在位置 i 处分配为新最小值的任何元素也必须属于其左侧的所有较短后缀，这会强制执行一致的层次分配。 这消除了歧义：一旦我们决定在每个级别引入多少个新的最小值，实际的身份选择是独立的，并且仅取决于剩余多少未使用的值。 

这种独立性使得最终计数能够纳入组合选择的乘积中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def build_fact(n):
    fact = [1] * (n + 1)
    invfact = [1] * (n + 1)
    for i in range(1, n + 1):
        fact[i] = fact[i - 1] * i % MOD
    invfact[n] = modinv(fact[n])
    for i in range(n, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD
    return fact, invfact

n = int(input())
t = list(map(int, input().split()))

fact, invfact = build_fact(n)

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

rem = n
used = 0
ans = 1

for i in range(n - 1):
    take = t[i]
    ways = C(rem - used, take)
    ans = ans * ways % MOD
    used += take
    rem -= 1

print(ans)
```该实现预先计算阶乘以有效地评估二项式系数。 该循环维护有多少元素仍未分配以及有多少元素已提交给早期的后缀约束。 在每一步中，我们都会乘以选择负责该后缀处的新记录最小事件的元素的方法数。 

一个常见的实现陷阱是对剩余元素的逐一处理。 后缀大小每一步缩小 1，而已提交元素的数量根据 t[i] 增长。

 ## 边缘情况

 关键的边缘情况是所有 ti 都为零时。 这意味着第一个元素之外的任何后缀中都不会出现新的记录最小值。 该算法通过在每一步始终选择零元素来强制执行这一点，因此二项式因子全部为 C(x, 0) = 1，从而准确地生成一个有效的排列结构。 

当 ti 在早期位置较大时，会出现另一种边界情况。 这会大量消耗可用元素，从而缩小后面后缀的池。 组合结构自然地处理了这个问题，因为一旦使用的元素超过可用性，二项式系数就会变为零。 该问题保证了有效性，因此输入中不会出现此类矛盾。 

最后一个微妙的情况是当 ti = n − i 对于早期 i 时，迫使每个剩余元素成为新的最小值。 这对应于后缀中严格递减的结构。 然后，每个步骤都有一种方法来选择所有剩余元素，从而保持产品稳定且定义明确。
