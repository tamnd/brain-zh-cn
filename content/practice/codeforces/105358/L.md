 ## 解决方案
 ## 问题理解

 我们正在模拟一个包含单个倒计时器的系统，其初始值是随机的。 在时间为零时，计时器被设置为 1 到 T 之间统一选择的整数。然后时间以离散秒数前进。 每过一秒，计时器就会减一。 当它达到零时，该过程结束，并将经过的时间记录为惩罚。 

不同之处在于，在每个递减步骤之后，只要计时器尚未达到零，我们就可以正常继续，或者丢弃当前值并“刷新”它，将其替换为相同范围内的新的独立统一值。 每次刷新还需要一整秒的时间，因为在做出决定之前就已经发生了递减步骤。 

任务是计算计时器到达零之前的预期总时间，假设我们遵循最小化此期望的最佳策略。 

输入包含多达一百万个独立的 T 值，每个值描述此随机过程的一个单独版本。 由于 T 可以大到 10^9，任何解决方案都必须在经过一些数学推导后在恒定时间内评估每种情况。 每个测试用例的线性甚至对数模拟是不可行的。 

一种简单的方法会尝试模拟决策过程，甚至通过定时器状态的动态编程来计算期望值。 对于大 T 来说，这会立即崩溃，因为状态空间高达 10^9，并且每个状态通过刷新操作依赖于所有其他状态。 

贪心模拟中出现了一个更微妙的问题：如果假设一种固定阈值策略而不证明最优性，那么当 T 很小时很容易得到不一致的结果。 例如，当 T = 1 时，答案很简单就是 1，因为计时器始终为 1 并且立即结束。 对于 T = 2，行为已经取决于刷新是否值得，并且幼稚的启发式方法可能会错误预测预期时间，因为它们忽略刷新会重新启动整个分布而不是改善当前轨迹。 

## 方法

 暴力公式将函数 f(x) 定义为当当前计时器值为 x 时的预期剩余时间。 从状态 x 开始，由于递减，总是会过去一秒。 之后，如果计时器达到零，我们就停止。 否则，我们面临一个二元决策：继续使用减少的值或丢弃它并从新的统一随机值重新开始。 

这导致了递归结构，其中每个状态通过均匀分布的期望值依赖于所有其他状态。 直接计算需要在整个范围 [1, T] 上重复重新计算期望，即使被记忆，它仍然形成所有状态的密集依赖图。 每个测试的转换数量实际上是 O(T)，这是不可能的。 

关键的观察结果是刷新操作并不以细粒度的方式依赖于当前值。 这仅取决于重新启动的预期值。 这将所有随机性压缩为一个常数：新开始的预期值。 一旦知道这个常数，每个状态都会将其确定性延续与相同的基准进行比较。 

这种结构的简化意味着最优政策不能在不同国家之间随意波动。 相反，随着剩余时间的减少，它必须从“刷新”切换到“保持”一次。 这就把问题变成了找到一个单一的阈值，将重新启动有益的状态与继续更好的状态区分开来。 一旦确定了这个阈值，整个期望就可以以封闭形式计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 DP 超过状态 | 每次测试 O(T) | O(T)| 太慢了 |
 | 阈值+封闭形式| 每次测试 O(1) | O(1) | O(1) | 已接受 |

## 算法演练

 我们将 f(x) 定义为当前计时器值在一秒开始时为 x 时的预期剩余惩罚。 

1. 由于强制递减步骤，每个状态都会立即消耗一秒，因此所有转换都以 1 的附加成本开始。 
2. 如果 x 等于 1，则递减后计时器变为零，因此 f(1) 恰好为 1。 
3. 当x大于1时，在支付第一秒后，系统转移到状态x−1。 此时，我们选择继续使用 f(x−1) 或使用新的统一值重新开始。 
4. 令A表示当y从1统一绘制到T时f(y)的预期值。这表示选择刷新的预期成本，因为刷新产生新的统一状态并且该过程从那里继续。 
5. 当 x > 1 时，递推变为 f(x) = 1 + min(f(x−1), A)。比较完全是在确定性延拓和常数基准 A 之间进行。 
6. 由于 f(x) 在 x 上单调递增，因此存在一个阈值 m，使得对于所有 x ≤ m 我们从不刷新，而对于所有 x > m 我们总是刷新。 
7. 在非刷新区域，该过程纯粹是确定性的，因此 f(x) = x（对于 x ≤ m）。 
8. 在刷新区域中，每个状态的行为相同并且始终选择刷新，因此 f(x) = 1 + A（对于 x > m）。 
9. 将此结构代入 A 的定义中，作为所有状态 f 的平均值，用 m 和 T 求解 A，并获得 A = (2T + m(m−1)) / (2m)。 
10. 阈值条件由两种状态之间的边界确定：在 x = m + 1 时，我们必须满足 f(m) ≤ A 且 f(m+1) > A，这导致不等式 m^2 + m > 2T。 满足这个不等式的最小整数 m 定义了正确的分割。 
11. 一旦计算出 m，预期答案就是 A 本身，因为初始状态均匀分布在 [1, T] 中的所有值上。 

### 为什么它有效

 该算法将决策过程简化为将每个可能的延续与单个全局常量 A 进行比较。这将自适应策略压缩为单调结构：如果刷新在某个状态 x 上是最优的，那么它在所有更大的状态下也必须是最优的，因为未来成本和延续成本都随着 x 的增加而增加。 这种单调性迫使存在一个单一的截止点 m，它充分表征了最优策略。 一旦以这种方式划分状态空间，所有期望就变成两个区间的线性和，从而消除任何递归依赖循环。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    out = []
    for _ in range(n):
        T = int(input())
        
        # compute m = smallest integer with m*(m+1) > 2T
        lo, hi = 1, 2 * T
        while lo < hi:
            mid = (lo + hi) // 2
            if mid * (mid + 1) > 2 * T:
                hi = mid
            else:
                lo = mid + 1
        m = lo
        
        num = 2 * T + m * (m - 1)
        den = 2 * m
        
        g = gcd(num, den)
        num //= g
        den //= g
        
        out.append(f"{num} {den}")
    
    print("\n".join(out))

def gcd(a, b):
    while b:
        a, b = b, a % b
    return a

if __name__ == "__main__":
    solve()
```该实现首先使用对不等式 m(m+1) > 2T 的二分搜索导出阈值 m。 这可以避免平方根带来的浮点精度问题，同时保持每次测试的恒定时间。 

获得 m 后，直接使用封闭式表达式 (2T + m(m−1)) / (2m) 计算期望值。 最后一步使用标准 gcd 例程减少分数以匹配所需的输出格式。 

必须注意所有乘法都必须安全地适合 Python 整数； 尽管值可以达到 1e9，但像 m^2 这样的中间乘积仍然受到大约 1e18 的限制，这在 Python 的任意精度算术中仍然是安全的。 

## 工作示例

 ### 示例 1

 考虑 T = 1。 

| 步骤| 米 | A 的公式 | 结果 |
 | --- | --- | --- | --- |
 | 计算阈值| 1 | m(m+1)=2 > 2T=2（在 m=1 之前为假） | m(m+1)=2 > 2T=2 米 = 1 |
 | 应用公式 | 1 | (2T + m(m−1)) / (2m) | (2T + m(m−1)) / (2m) | (2 + 0)/2 = 1 |

 系统始终从 1 开始，一秒后立即为零，并且任何刷新都无法改善任何情况。 期望与确定性行为相匹配。 

### 示例 2

 取T=3。 

| 步骤| 米 | 计算| 结果 |
 | --- | --- | --- | --- |
 | 门槛| 2 | 2·3 = 6 > 6 为假，因此 m=2 | 米 = 2 |
 | 分子| | 2T + m(m−1) | 2T + m(m−1) | 6 + 2 = 8 |

 | 简化 | | 8/4 | 2 |

 该结构意味着较小的计时器值始终直接完成，而较大的计时器值始终刷新，从而创建一个干净的分割来稳定期望。

 | 测量| 复杂性 | 说明|

 ## 边缘情况

 对于 T = 1，阈值计算得出 m = 1，这将所有状态置于确定性区域中。 该算法计算出 A = 1，这与计时器总是立即完成的事实相匹配。 

对于 T = 2，阈值变为 m = 2，这意味着状态 1 和 2 均在不刷新的情况下处理。 计算出的期望变为 2，这与系统几乎不会从如此小的范围内重新启动中受益的直接推理相符。 

对于非常大的 T，阈值仅增长约 sqrt(T)，但算法从不显式迭代到 T。二分搜索公式确保在不枚举状态的情况下找到截止值，并且最终公式仅取决于涉及 m 和 T 的算术，即使在 10^9 规模下也能保持计算稳定。
