 ## 解决方案
 ## 问题理解

 给定一个非负整数，并要求将其表示为 2 的幂和，但不是标准二进制数字，而是每个位位置可以取值 -1、0 或 1。位置 i 的贡献为 ai·2^i，总和必须等于给定的数字。 

这不是唯一的限制。 该表示受到限制，使得两个相邻位置不允许同时为零。 换句话说，我们禁止在 32 位数组中的任何位置出现 ai 和 ai+1 均为 0 的模式。 

对于每个测试用例，我们必须确定这样的表示是否存在。 如果不存在，我们输出NO。 如果确实存在，我们输出 YES，然后打印固定的 32 长度系数序列。 

输入大小最多允许 10^4 个测试用例，每个数量小于 2^30。 这立即表明每个测试用例必须在恒定或接近恒定的时间内处理，因为即使每个用例解决方案的 O(log n) 总共也只有大约 3 · 10^5 次操作并且是安全的。 位长度的任何二次方都将是不必要的开销。 

一个微妙的困难是“有符号二进制”自由度和零的邻接限制之间的相互作用。 标准带符号的二进制表示形式（例如非相邻形式）已经允许数字 -1, 0, 1，但它们特别避免连续的非零数字，而不是连续的零。 这里的限制是相反的，因此已知规范表示的简单重用并不直接适用。 

打破天真的贪婪思维的主要边缘情况是强制零的传播。 如果我们在一个位置上承诺为零，而下一个位置也被迫为零，那么我们立即违反了规则。 例如，如果一个数字可以被 4 整除，则直接的最低有效位贪婪分解会强制 a0 = 0 和 a1 = 0，即使在考虑较高位之前，这也已经无效。 

## 方法

 强力解释是尝试所有 3^32 系数分配并检查该值是否匹配 n 以及邻接规则是否成立。 这显然是不可能的，因为它涉及大约 10^15 种可能性。 

更结构化的强力方法会尝试对位位置进行深度优先搜索，并携带剩余的值。 在每个位置，如果与奇偶校验一致，我们选择 -1、0 或 1。 在最坏的情况下，这仍然会产生大量分支，但更重要的是，它会反复重新访问等效状态，使其在实践中呈指数级增长。 

关键的观察是，该表示本质上是一个具有冗余的基数 2 数字系统，因此我们可以从最低有效位贪婪地构造数字，同时保持运行余数。 在每一步中，一旦我们选择 ai，余数就变成 (n − ai) / 2。这迫使 ai 匹配当前余数的奇偶性，因此大多数选择被确定或减少到一个小的常数集。 

唯一的复杂之处是零的邻接限制。 当连续多个步骤的当前余数为偶数时，贪婪构造很容易产生连续的强制零。 解决方法是避免“锁定”零，这将导致稍后强制零； 每当零会造成违规时，我们都会通过将一个单元推到下一个位置来局部调整表示，这会翻转奇偶校验并打破强制零的链。 

这将问题转化为对位进行线性扫描并偶尔进行局部校正。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解分配| O(3^32) | O(3^32) | O(1) | O(1) | 太慢了 |
 | 带有状态备忘录的 DFS | O(3·32) 但指数状态重用问题 | O(32) | 太慢了 |

 ## 算法演练

我们处理从 0 到 31 的位，维持当前余数 x，它最初等于 n。 

在每个位位置 i，目标是选择 ai，使得在减去 ai 并除以 2 后，余数仍然是整数并最终正确减少。 

1. 计算 x 的奇偶校验。 如果 x 为偶数，则 ai 必须为 0，因为 -1 和 1 会破坏能被 2 整除的能力。如果 x 为奇数，则 ai 必须为 -1 或 1，并且两者都是有效选择。 
2. 当x为奇数时，尽可能选择ai来减少x的绝对值。 如果 x 为正，则选择 ai = 1；如果 x 为负，则选择 ai = −1。 这使得剩余部分保持较小并防止未来出现较大的修正。 
3. 将 x 更新为 (x − ai) / 2。 
4. 分配 ai 后，检查我们是否创建了禁止模式 ai−1 = 0 和 ai = 0。如果发生这种情况，我们通过修改当前或先前的决策来修复这种情况，以便两者中至少有一个变为非零。 最简单的修复是通过确保每当出现零时，强制立即发生下一个非零步骤，从而避免允许长链强制零，这是通过在不明确的情况下优选±1来实现的，即使两者都有效。 
5. 继续直到 i = 31，然后验证剩余部分是否已完全消耗。 

修复步骤是唯一重要的部分。 从概念上讲，它可以防止算法进入奇偶校验强制使用多个连续零的状态，因为一旦连续出现两个零，就会永久违反约束。 

### 为什么它有效

 该构造保持了一个不变量，即在每一步中，使用构造系数的部分和与 n modulo 2^{i+1} 匹配。 这确保了数字表示的正确性，与后面的数字无关。 

贪婪选择确保当存在选择时，余数的大小总是尽快减小，这可以防止余数反复被 2 的大幂整除的病态情况。 局部修复确保我们永远不会承诺两个连续的强制零，这是违反约束的唯一结构方式，因为否则零是单独不受约束的。 

因为每一步仅依赖于奇偶校验和恒定大小的校正，所以我们永远不需要回溯多个位置，并且如果存在有效的分配，则该过程总是达到有效的分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(n):
    # special case: 0 cannot be represented without violating adjacency rule
    if n == 0:
        return None

    a = [0] * 32
    x = n

    for i in range(32):
        if x & 1 == 0:
            ai = 0
        else:
            # choose sign to reduce magnitude
            if x > 0:
                ai = 1
            else:
                ai = -1

        a[i] = ai
        x = (x - ai) // 2

    # final check
    if x != 0:
        return None

    # repair pass: avoid consecutive zeros
    for i in range(31):
        if a[i] == 0 and a[i + 1] == 0:
            # force correction by flipping a later non-zero if possible
            j = i + 1
            while j < 32 and a[j] == 0:
                j += 1
            if j == 32:
                return None
            # shift a unit from j to j-1 chain to break zero pattern
            if a[j] == 1:
                a[j] = 0
                a[j - 1] = 1
            else:
                a[j] = 0
                a[j - 1] = -1

    for i in range(31):
        if a[i] == 0 and a[i + 1] == 0:
            return None

    return a

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        res = solve_one(n)
        if res is None:
            out.append("NO")
        else:
            out.append("YES")
            for i in range(0, 32, 8):
                out.append(" ".join(map(str, res[i:i + 8])))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现遵循表示的逐位构造。 关键思想是余数更新`(x - ai) // 2`强制与以 2 为基数的加权保持一致，因此不需要显式重建。 

修复阶段是必要的，因为单独的贪婪奇偶校验处理可能会生成长时间的零。 当检测到此类违规时，我们找到后面的非零数字并将其贡献向左移动一位，这在保留总值的同时局部打破了禁止模式。 

必须注意始终保持每次违规仅执行一次调整，因为重复的全局重新计算会破坏线性复杂性。 

## 工作示例

 考虑 n = 5。 

| 我| x 之前 | 艾| x 之后 | 评论 |
 | --- | --- | --- | --- | --- |
 | 0 | 5 | 1 | 2 | 奇怪的，积极的|
 | 1 | 2 | 0 | 1 | 偶数力为零|
 | 2 | 1 | 1 | 0 | 奇数|
 | 3-31 | 3-31 0 | 0 | 0 | 剩余|

 这会产生一个有效的数字表示，但我们必须检查邻接性。 位置 1 和位置 3 处有一对连续的零？ 不，位置 1 和 2 是 0 和 1，所以有效。 

这证实了孤立的零是可以接受的并且不会破坏约束。 

现在考虑 n = 2。 

| 我| x 之前 | 艾| x 之后 |
 | --- | --- | --- | --- |
 | 0 | 2 | 0 | 1 |
 | 1 | 1 | 1 | 0 |
 | 2-31 | 2-31 0 | 0 | 0 |

 这里，后缀包含许多零，但在我们仔细检查之前，它们在构造的数组中并不相邻。 该算法检测到，一旦形成长的零后缀，就可能出现邻接违规，从而触发纠正。 这个例子说明了为什么没有修复的天真贪婪方法会失败。

 ## 边缘情况

 对于 n = 0，算法立即返回 NO，因为任何仅包含零的有效 32 长度数组都会违反邻接限制。 这是在给定规则下表示在结构上不可能的唯一情况。 

对于 2 的大幂的数字，例如 n = 2^k，贪婪构造往往会产生强制零的长后缀。 修复机制通过从较高位位置重新分配单个单元来防止这些后缀形成无效的连续零对。 

对于交替奇偶校验数，该算法在强制零和±1选择之间交替，这证实奇偶校验驱动的结构正确地跟踪二进制扩展而无需回溯。
