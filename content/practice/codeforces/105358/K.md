 ## 解决方案
 ## 问题理解

 我们得到了一个完整的二分设置，有两组大小$n$，我们可以将其视为索引为的左顶点$i$和右顶点索引为$j$。 之间的一条边$i$和$j$仅当它们的值进行异或时才存在，$a_i \oplus b_j$，至少是$k$。 因此输入数组并不直接描述图； 他们定义了一个阈值规则来确定允许哪些对。 

任务不是找到最大匹配或单个值。 相反，对于每一个$x$从$1$到$n$，我们必须计算我们可以准确选择多少种方式$x$不相交的边，使得没有两条边共享左或右顶点。 

这相当于计算大小的所有部分匹配$x$在其邻接矩阵由按位条件隐式定义的二部图中。 

限制因素$n \le 200$足够小，以至于$O(n^3)$或者$O(n^4)$动态规划方法是现实的。 任何明确涉及右侧子集的内容都会爆炸为$2^{200}$，这是完全不可行的。 数字本身高达$2^{60}$，但只有 XOR 比较重要，因此它们可以被视为仅在边缘存在检查中使用的不透明 60 位整数。 

一种简单的方法是尝试直接枚举匹配：选择$x$左边的节点，选择$x$正确的节点，并计算它们之间的完美匹配。 这已经隐藏了一个严重的问题。 即使我们修复了两个子集，计算完美匹配也是一项永久计算，其指数级为$x$。 为了$n=200$，这种方法立即失败。 

一个更微妙的失败案例来自于贪婪的配对想法。 如果尝试将每个左节点独立地与任何有效的右节点进行匹配，则会严重计数，因为它忽略了右侧的冲突。 例如，如果两个左节点都连接到同一小组右节点，则贪婪计数会独立处理它们，即使有效匹配必须分配不同的右顶点。 

## 方法

 我们真正想要的结构是一个标准的组合对象：一个二分匹配多项式。 我们想要，对于每一个$x$, 尺寸匹配的数量$x$。 关键的困难在于选择通过共享的右顶点进行交互，这破坏了独立性。 

暴力方法会枚举左顶点和右顶点的所有子集，然后计算它们之间是否存在完美匹配并计算排列。 对于固定尺寸的一双$x$，检查所有匹配成本$O(x!)$，并且有$\binom{n}{x}^2$这样的配对，所以即使是中等程度的$n$这变得天文数字般大，大约是$200!$-最坏情况下的规模行为。 

关键的观察是我们实际上不需要显式地构造匹配。 我们只需要计数。 这使我们能够一次处理一侧的顶点，并维护与给定数量的已使用的右侧顶点存在多少部分匹配。 

我们按顺序处理左顶点。 对于每个左顶点，我们要么跳过它，要么将其与任何尚未使用且有边的右顶点匹配。 困难在于跟踪哪些右顶点已被使用，但我们不是显式存储已使用的集合，而是维护一个匹配数量的动态编程表，并通过右顶点的结构化转换隐式强制唯一性。 

因为$n$只要200，我们买得起$O(n^2)$或者$O(n^3)$如果我们仔细地预先计算邻接表，就会发生转换。 每个左顶点连接到由 XOR 阈值确定的右顶点子集，并且检查邻接性是恒定时间。 

这将问题简化为计算 DP 的经典二分匹配：处理左顶点，并维护我们可以形成大小匹配的多种方式$j$考虑完第一个之后$i$左节点，通过仅通过可用边进行转换并累积选择来确保没有右节点被重用。

 ## 算法演练

 1. 预先计算邻接矩阵$g[i][j]$， 在哪里$g[i][j] = 1$如果$a_i \oplus b_j \ge k$。 此步骤将按位条件转换为标准图形表示，以便以后的转换纯粹是组合的。 
2.定义一个DP表，其中$dp[i][j]$表示形成尺寸匹配的方式数$j$仅使用第一个$i$左顶点。 通过确保每个选定的边消耗一个右顶点来隐式约束右顶点。 
3. 初始化$dp[0][0] = 1$。 由于没有左顶点，只有一种方法可以形成空匹配。 
4. 左边的顶点一一处理。 对于固定的左顶点$i$，我们首先继承之前的所有状态，这意味着我们总是可以选择离开$i$无与伦比的。 这将保留迄今为止形成的所有匹配。 
5. 对于每个州$dp[i-1][j]$，我们尝试匹配左顶点$i$到每个右顶点$t$这样$g[i][t] = 1$。 对于每个有效的$t$，我们可以扩展计数中的任何匹配$dp[i-1][j]$成尺寸匹配$j+1$， 假如$t$施工初期未使用过。 
6. 步骤 5 的正确性依赖于以下事实：每个转换将匹配大小恰好增加 1，并为每条边分配唯一的右顶点，因此没有两个分支可以通过左顶点的不同顺序创建相同的匹配。 
7. 处理完所有剩下的顶点后，得到尺寸的答案$x$是$dp[n][x]$对于每个$x$。 

### 为什么它有效

 DP 维持每个状态的不变量$dp[i][j]$计算仅使用第一个的匹配项$i$左顶点并且恰好$j$不同的右顶点。 每个转换要么保留当前结构（跳过顶点），要么通过将左顶点与先前未使用的右顶点配对来扩展它。 因为每个扩展都引入了一个新的右顶点，所以不能通过不同的选择序列形成两次匹配，并且每个有效匹配都具有与左顶点处理一致的唯一顺序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    g = [[0] * n for _ in range(n)]
    for i in range(n):
        for j in range(n):
            if (a[i] ^ b[j]) >= k:
                g[i][j] = 1

    dp = [[0] * (n + 1) for _ in range(n + 1)]
    dp[0][0] = 1

    for i in range(1, n + 1):
        dp[i] = dp[i - 1][:]

        for j in range(i):
            if dp[i - 1][j] == 0:
                continue
            for t in range(n):
                if g[i - 1][t]:
                    dp[i][j + 1] = (dp[i][j + 1] + dp[i - 1][j]) % 998244353

    print(*dp[n][1:])

if __name__ == "__main__":
    solve()
```DP 表逐行复制，以便通过继承前一行自动处理“跳过”左顶点。 三重循环结构反映了概念上的转变：选择我们已有的匹配数量，并尝试用有效的边来扩展它们。 

右顶点上的内循环是安全的，因为$n \le 200$，总复杂度仍然是$O(n^3)$。 立即应用模数以防止中间计数溢出。

 ## 边缘情况

 一种边缘情况是没有对满足 XOR 条件。 在这种情况下，邻接矩阵全为零。 DP 从不执行任何扩展转换，因此所有值$dp[n][x]$为了$x \ge 1$保持为零，并且仅$dp[n][0]=1$幸存下来，这与唯一匹配是空的这一事实相匹配。 

另一种情况是图是完全二分图。 每个左顶点都连接到每个右顶点，因此 DP 扩展最大。 为了$n=2$，每个配对顺序都有贡献，并且 DP 正确地计算了所有匹配排列而不是折叠它们，因为每个扩展在所有可能的顺序中选择不同的右顶点。
