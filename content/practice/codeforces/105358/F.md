 ## 解决方案
 ## 问题理解

 我们正在跟踪一个不断变化的值，即用户的评分。 评级从固定的初始值 1500 开始，然后在即将举行的 n 场比赛中每场比赛后都会发生变化。 对于第 i 次比赛，评分变化为 ci，更新规则是附加的，这意味着新的评分变为之前的评分加上 ci。 

任务不是计算最终评分，而是检测应用比赛更改后运行评分首次达到或超过 4000 的最早时间点。 如果任何前缀的评级从未达到 4000，我们返回 -1。 

因此，该结构是带有阈值检查的前缀累积问题。 每个前缀总和代表该次数比赛后的评分，偏移初始值 1500。 

n 最大为 100000 的约束意味着任何解决方案都必须在恒定时间内处理每场比赛。 在最坏的情况下，重复重新计算每个前缀的和的二次方法将需要大约 10^10 次运算，这远远超出了 Python 中 1 秒的限制。 这立即将我们推向单遍积累策略。 

一个微妙的边缘情况来自负面更新。 由于 ci 可能为负，因此评级可能会在阈值之上和之下波动。 我们必须确保返回达到阈值的第一个索引，而不是最后一个。 

另一个边缘情况是当初始评级已经高于或等于 4000 时。在这个问题中，它是 1500，因此不会发生这种情况，但强大的推理方法仍然应该在处理之前自然地将其作为前缀检查来处理。 

## 方法

 最直接的想法就是从字面上模拟这个过程。 我们维持当前的评分，从 1500 开始，并通过添加 ci 重新计算每次比赛的评分。 每次更新后，我们都会检查评级是否至少为 4000。如果是，我们返回当前索引。 

这种方法是正确的，因为它准确地反映了该过程。 然而，它在结构上也已经是最优的。 如果我们尝试一种简单的公式，例如通过从头开始重新求和来检查每个前缀，那么唯一的低效率将来自于重复地重新计算前缀和。 该变体需要 O(n^2) 运算，因为每个前缀总和将花费 O(n)，并且有 n 个前缀。 

关键的观察是评级更新纯粹是累积的。 没有分支，没有重置，也不依赖于未来的值。 这意味着第 i 步的状态完全由第 i - 1 步的状态决定，因此我们可以维护一个运行总计，而不用重新计算任何内容。 

我们将问题简化为具有运行总和的单次扫描，并在每次更新后检查阈值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 从头开始重新计算前缀和 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 运行累计总和 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护一个代表当前评级的变量并按顺序更新它。 

1. 将评分初始化为 1500，因为这是应用任何竞赛之前的起始状态。 
2. 按顺序从 i = 1 到 n 迭代比赛，读取每个 ci 并将其添加到当前评分中。 这一步代表应用第 i 次比赛的预测变化。 
3. 更新索引 i 处的评分后，检查评分是否至少为 4000。如果是，则在达到阈值的第一时刻立即返回 i。 尽早退出至关重要，因为我们希望第一次出现。 
4. 如果循环结束时评级尚未达到 4000，则返回 -1。 

关键的设计选择是在每次更新后立即执行阈值检查，而不是推迟它。 这确保我们捕获最早的索引，而不是条件仍然成立的较晚的索引。 

### 为什么它有效

该算法保持不变式，即在处理 i 个元素后，存储的评分恰好等于 1500 加上 c1 到 ci 的总和。 这直接源自按顺序应用的更新规则。 

由于每个步骤仅添加 ci 一次并且从不重新访问过去的值，因此以后不能跳过或更改任何前缀状态。 因此，当评级在某个索引 i 处首次跨越或达到 4000 时，算法会在该步骤立即检测到它。 由于我们立即返回，因此任何后续索引都无法覆盖此答案，从而保证了最小性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    rating = 1500
    
    for i, c in enumerate(arr, 1):
        rating += c
        if rating >= 4000:
            print(i)
            return
    
    print(-1)

if __name__ == "__main__":
    solve()
```该解决方案保留一个称为评级的整数累加器。 从1500开始，就地更新。 枚举采用 1 索引，直接匹配比赛编号，避免差一错误。 

每次更新后立即执行条件检查。 立即返回可确保我们不会意外报告稍后的有效索引。 

除了输入数组之外没有任何数据结构，因此内存使用量保持在最低限度。

 ## 边缘情况

 一个关键的边缘情况是当评级恰好在某个前缀处超过阈值然后随后下降时。 例如：

 输入：```
4
1000 1000 -500 2000
```逐步：

 1:2500 之后

 2 之后：3500

 3点后：3000

 4点后：5000

 即使评分在第 3 步下降到 4000 以下，正确答案仍然是 4，因为我们只关心它第一次达到或超过 4000，这发生在第 4 步。 

该算法自然地处理这个问题，因为它只在每次更新后检查条件，并在第一次成功后立即返回。 之前的成功不会被覆盖，因此一旦找到有效的索引，以后的振荡就无关紧要了。
