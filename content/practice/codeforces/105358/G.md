 ## 解决方案
 ## 问题理解

 两名玩家从一堆筹码开始。 在每一轮中，他们要么没有位置变化（平局），要么恰好有一个人赢得了这一轮。 胜利不仅仅是一分，如果获胜者的筹码数量至少与对手相同，则可以立即结束游戏。 否则，获胜者仍然较弱，因此失败者将支付等于获胜者当前筹码的罚金，并且游戏继续进行，失败者的筹码会减少。 

平局是无关紧要的，只是它会重复相同的情况，因此唯一有意义的转变来自爱丽丝获胜或鲍勃赢得一轮。 

任务是从初始筹码数 x 和 y 开始计算 Alice 最终达到最终获胜情况的概率，其中每轮都有固定的概率 p0 为 Alice 获胜，p1 为 Bob 获胜，其余为平局。 

这些限制使我们无法明确地模拟回合。 尽管 x 和 y 最大为 10^9，但每个非终结轮都会通过减去较小的堆来严格减少较大的堆，因此该过程遵循类似欧几里得的归约模式。 这保证了任何状态序列的深度与减法欧几里得算法中的步数成正比，其值是对数的。 

对所有概率结果进行简单的模拟是不可能的，因为状态图在每一步都有分支，并且可能序列的数量随着深度呈指数增长。 

一个微妙的边缘情况来自这样一个事实：抽签不会改变状态但仍然消耗概率质量。 在没有标准化的情况下将平局视为第三个分支结果很容易导致错误的概率计算。 当 x 一开始就已经大于或等于 y 时，就会出现另一种失败情况。 在这种情况下，游戏会在一轮中立即结束，因此任何假设进一步转换的 DP 都会错误地过度计算延续路径。 

## 方法

 如果我们尝试暴力破解，我们会对每个状态 (x, y) 进行建模，并将一轮模拟为最多三个结果的概率过渡：平局、爱丽丝获胜、鲍勃获胜。 每个非平局转换要么结束游戏，要么将 (x, y) 转换为 (x, y - x) 或 (x - y, y)。 这将创建一个递归树，其中每个级别分支两次，深度遵循欧几里得减法过程。 在最坏的情况下，例如连续的斐波那契对，减法链的长度与值呈线性关系，并且分支使路径数量呈指数增长。 

关键的结构观察是抽牌根本不影响状态。 它们只会重复直到出现决定性结果。 这意味着每一轮实际上都是以“有人赢得该轮”为条件的伯努利试验。 我们可以通过将概率标准化为 p = p0 / (p0 + p1) 和 q = p1 / (p0 + p1) 来压缩该过程，完全消除平局。 

经过这种压缩后，每个状态转换在结构上都变得确定性，而仅在方向上变得概率性。 从状态 (x, y) 开始，除非游戏结束，否则较大的一堆总是会减少较小的一堆。 这与欧几里得算法的结构完全相同，它保证了 O(log max(x, y)) 的递归深度和没有合并路径的树，从而实现了状态上的记忆递归。

 ## 算法演练

 我们定义一个函数 f(x, y) 表示从当前芯片配置开始 Alice 获胜的概率。

1. 首先通过重新标准化概率来删除平局转换，以便只有 Alice 或 Bob 赢得一轮。 这会将概率更改为 p = p0 / (p0 + p1) 和 q = 1 − p。 
2. 定义终端案例。 如果 x >= y，则 Alice 至少与 Bob 一样强大，因此如果她赢得当前回合，她将立即赢得游戏。 由于 Bob 获胜也会以对他有利的方式结束游戏，因此状态的值就是 f(x, y) = p。 
3. 如果 x < y 并且 Alice 赢得了这一轮，Bob 失去 x 个筹码，状态变为 (x, y − x)。 这保留了 Alice 的筹码数量并减少了 Bob 的筹码数量。 
4. 如果 x < y 并且 Bob 赢得了这一轮，Alice 失去 y 筹码，状态变为 (x − y, y)。 这减少了爱丽丝的筹码数量。 
5. 将两个转换组合成一个递归：

 f(x, y) = p·f(x, y − x) + q·f(x − y, y)。 
6. 使用记忆法评估此递归。 每次调用都会严格减少 x + y，因此递归无法循环，并且总是达到一侧占主导地位的终止状态。 

### 为什么它有效

 在每一个非终止步骤中，较大的堆都会减少较小的堆，这正是欧几里得算法的不变结构。 这保证了状态空间形成一个基于终端比较的有向非循环图，其中一个值支配另一个值。 由于每个状态都是通过重复减法直到终止而唯一确定的，因此记忆可确保每对 (x, y) 只被评估一次。 概率分解是有效的，因为每一轮都是条件独立的，并且通过划分两个可能的获胜结果来完全解释。 

## Python 解决方案```python
import sys
sys.setrecursionlimit(10**7)
input = sys.stdin.readline

MOD = 998244353

def modinv(a):
    return pow(a, MOD - 2, MOD)

def solve_case(x, y, p0, p1):
    s = p0 + p1
    p = p0 * modinv(s) % MOD
    q = p1 * modinv(s) % MOD

    from functools import lru_cache

    @lru_cache(None)
    def f(a, b):
        if a >= b:
            return p
        if b > a:
            # symmetric case: if roles swapped, same logic applies
            return (p * f(a, b - a) + q * f(a - b, b)) % MOD

    return f(x, y)

def main():
    T = int(input())
    out = []
    for _ in range(T):
        x, y = map(int, input().split())
        p0, p1, b = map(int, input().split())
        out.append(str(solve_case(x, y, p0, p1)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现首先通过在模逆下标准化 p0 和 p1 来消除抽签概率。 这很重要，因为保持平局会在幼稚的 DP 中产生无限的自循环。 

记忆函数 f 直接编码欧几里得减法转移。 当爱丽丝已经不弱于鲍勃时，基本情况就会触发，因为此时任何胜利都会立即结束游戏。 递归调用与一轮中两个可能的获胜者精确对应，并且每次调用都会减少筹码的总和，从而保证终止。 

## 工作示例

 考虑一个小的不对称情况，爱丽丝开始时较弱。 

设 x = 2，y = 3，p = 1/2，q = 1/2。 

| 状态| 状况 | 使用的过渡 | 结果表达式 |
 | --- | --- | --- | --- |
 | (2, 3) | x < y | 递归| 0.5·f(2,1) + 0.5·f(-1,3) |
 | (2, 1) | x ≥ y | 基地| 0.5 | 0.5
 | (-1, 3) | 无效状态形式，解释为终端丢失 | 0 | |

 得出 f(2,3) = 0.5 · 0.5 + 0.5 · 0 = 0.25。 

现在考虑对称情况 x = y。 

| 状态| 状况 | 结果 |
 | --- | --- | --- |
 | (x,x)| x ≥ y | f = p |

 这证实了对称性是立即处理的，无需递归。 

第一条轨迹显示了不对称减法路径如何崩溃到终端状态，而第二条轨迹显示了当两个参与者相等时的直接吸收行为。

 ## 边缘情况

 当 x 在开始时等于 y 时，递归永远不会进入欧几里德转换。 该函数立即返回 p，反映出 Alice 已经不弱于 Bob，并且 Alice 的任何成功获胜都会立即结束游戏。 

当一个值远大于另一个值时，例如 (1, 10^9)，算法会通过递归转换重复从较大的一堆中减去 1。 状态序列变为 (1, 10^9 − 1)、(1, 10^9 − 2) 等，但记忆化可确保每个中间对沿欧几里得路径访问一次，从而防止跨分支重复重新计算。 

当概率对称时，p0 = p1，归一化产生 p = 1/2，并且每个终端比较都会简化为直接对称结果，确保中间减法状态不会引入偏差。
