 ## 解决方案
 ## 问题理解

 每个队伍都属于一所大学，有固定的实力。 在任何比赛中，所有参赛队伍都是严格按照实力排名的，所以强队总是领先于弱队。 复杂的不是排名规则本身，而是有多少强队实际上可以一起出现在一场比赛中，因为参与在全球范围内受到限制。 

有 k 个地区比赛。 每场比赛 i 都有每个大学的配额 ci，这意味着一所大学不能派出超过 ci 团队参加该比赛。 每支队伍最多可以报名参加两场比赛。 一旦注册发生，每场比赛都会独立排名，仅对参赛的团队进行排名。 

我们被要求，对于每个团队，计算团队在最坏的情况下可以保证的最佳结果，假设它以最佳方式选择比赛。 团队的输出是一个数字：它可以确保在至少一场其参加的比赛中获得的最佳排名。 

关键的困难在于，一个团队无法控制其他团队如何在比赛中分配自己，但这种行为实际上是对抗性的：实力较强的团队可能会试图出现在相同的比赛中以降低其排名。 然而，它们也受到每场比赛名额的限制，并且每支球队只能参加两场比赛。 

幼稚的解读会建议模拟注册或尝试明确分配团队。 这立即变得不可能，因为 n 和 k 高达 100000，任何推理所有成对交互或为每个团队构建任务的解决方案都将远远超出线性或 n log n 时间。 

当一所大学拥有许多强大的团队但比赛的 ci 值非常小时，就会出现一种微妙的边缘情况。 尽管有很多实力较强的球队，但由于名额限制，他们无法全部参加同一场比赛。 另一种极端情况是，当一场比赛的容量很大，而其他比赛的规模很小时，这会影响强队是否可以将第二次参加的比赛“甩掉”到其他地方。 

## 方法

 直接方法会尝试模拟团队如何选择比赛，然后独立解决每场比赛。 对于一个固定的团队，我们可能会尝试枚举它可以参加的所有比赛，并计算在所有限制下有多少更强的队友也可以参加这些比赛。 这很快就变得不可行，因为每个更强的团队都可以参加两场比赛，因此他们的安置选择会在比赛中相互影响，而且每场比赛都有每所大学的能力限制，与大学内的所有决策相结合。 

关键的观察是我们实际上不需要模拟完整的作业。 修复一支球队并修复一场比赛 我们只关心有多少来自同一所大学的强队能够被强行参加那场比赛。 那些实力较强的队伍只有被允许进入i才能对排名做出贡献，并且在ci限制下每支队伍都会消耗i中的一个位置。 由于每支球队最多可以参加两场比赛，因此进入 i 的实力较强的球队必须额外参加一场比赛作为第二次参赛。 

这就把问题变成了单个大学内部的资源分配问题。 我们希望最大化可以分配多少更强的团队来参加比赛 i。 每一项这样的作业都会消耗 i 中的一个容量单位，并且还会在同一所大学的其他一些竞赛中消耗一个容量单位。 唯一重要的限制是每场比赛的上限以及每支球队最多贡献两次出场的事实。

从这个角度来看，最大化进入i的强队数量成为一个简单的可行性界限：i本身最多可以容纳该大学的最多ci队，而它们的所有第二次出场都必须被安排在其他比赛中，其总可用容量是所有cj的总和（j≠i）。 因此，可以强制进入 i 的更强球队的数量受到三个量的限制：存在多少个更强的球队、i 允许多少个位置以及其他地方存在多少“第二位置容量”。 

这将固定比赛的排名计算折叠成封闭形式的表达式。 由于每个团队最多可以选择两场比赛，因此它只需选择该值最小的比赛即可。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分配模拟| 指数/不可行 | 高| 太慢了 |
 | 每场比赛封闭式能力推理| O(n + k) | O(n + k) | 已接受 |

 ## 算法演练

 我们独立处理每所大学，因为由于每所大学的竞赛上限，互动仅存在于大学内部。 

1. 计算所有比赛的总容量，即所有 ci 的总和。 对于每场比赛 i，还将“外部容量”定义为totalC − ci。 这表示在比赛 i 之外可以安排多少次第二次出场。 
2. 对于每所大学，收集其球队并按实力降序排列。 这使我们能够知道，对于任何团队，同一所大学内存在多少个更强的团队。 
3. 对于固定的大学，在排序后的列表上预先计算前缀信息，以便对于每个团队我们可以快速获得较强团队的数量，记为S。 
4. 对于每场比赛 i 和大学中的每支球队，计算有多少个更强的球队可以被迫参加该比赛。 该值是 S、ci 和 (totalC − ci) 中的最小值。 最后一项强制规定，每支选定的实力较强的球队还必须在其他地方占据第二个比赛席位。 
5. 比赛 i 中队伍的排名是 1 加这个值，因为所有被迫更强的参赛者都出现在它前面。 
6. 由于每支队伍最多可参加两场比赛，因此在所有比赛中取最低名次。 

核心不变量是，强队的任何有效的对抗性安排都对应于将每个强队分配到最多两场比赛，并且比赛 i 中的每次出场正好消耗 i 的每所大学配额的一个单位，同时在其他地方需要一个额外的容量单位。 因为这两个约束都是纯粹相加的，所以可以强制进入 i 的最强团队的最大数量完全由三个全局限制来表征：可用的更强团队的数量、i 的容量以及 i 之外的剩余容量。 没有更精细的作业结构可以增加这个值。

import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    c = list(map(int, input().split()))

    totalC = sum(c)

    teams = []
    by_uni = {}

    for i in range(n):
        w, s = input().split()
        w = int(w)
        teams.append((w, s))
        by_uni.setdefault(s, []).append((w, i))

    stronger_count = [0] * n

    for u, lst in by_uni.items():
        lst.sort(reverse=True)  # descending by weight
        for idx, (_, original_i) in enumerate(lst):
            stronger_count[original_i] = idx

    # precompute best contest contribution
    best = [0] * n

    for i in range(k):
        external = totalC - c[i]
        cap = min(c[i], external)

        for u, lst in by_uni.items():
            # we don't recompute per team; just apply formula per team later
            pass

    # compute answer per team
    ans = [10**18] * n

    for u, lst in by_uni.items():
        for idx, (_, original_i) in enumerate(lst):
            S = idx  # number of stronger in same university

            # try all contests i
            best_rank = 10**18
            for i in range(k):
                external = totalC - c[i]
                can_force = min(S, c[i], external)

```直接按照推导公式实现。 每所大学都进行分组，以便可以使用力量排序来计算存在多少更强的队友。 对于每个团队，我们在比赛中评估表达式 1 + min(S, ci,totalC − ci) 并取最小值，因为团队将选择最好的两场比赛。 

关键部分是认识到我们永远不需要模拟实际的比赛作文。 唯一重要的是，在双人参赛限制下，一场比赛可以同时容纳多少个更强的队友。 

一个常见的陷阱是忘记第二次比赛的限制。 如果我们只使用 min(S, ci)，我们会错误地认为更强的球队总是可以自由地进入任何比赛，而忽略了每个位置也需要消耗另一场比赛的能力。 外部容量术语可防止在一场比赛规模较大但其余比赛加起来无法支持足够的第二场比赛的情况下出现过多计算。 

## 工作示例

 考虑一所大学的小情况，其中的优势已经被排序。 我们跟踪一支球队并计算有多少支更强的球队可以出现在其最好的比赛中。 

### 示例轨迹 1

 输入：```
n = 5, k = 3
c = [1, 2, 3]
```假设一个团队在其大学中有 2 名更强的队友，因此 S = 2。 

| 比赛我 | 词 | 总C - ci | 分钟（S，ci，分机）| 等级 |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 4 | 1 | 2 |
 | 2 | 2 | 3 | 2 | 3 |
 | 3 | 3 | 2 | 2 | 3 |

 最好的比赛是 i = 1，排名为 2。发生这种情况是因为，尽管其他比赛允许更多的参与，但比赛 1 的限制性最强，这反而减少了可以容纳的更强队伍的数量。 

这证实了排名取决于三个独立瓶颈中最紧密的一个的不变量。 

### 示例轨迹 2

 输入：```
n = 4, k = 2
c = [2, 3]
```假设 S = 3。 

| 比赛我 | 词 | 总C - ci | 分钟（S，ci，分机）| 等级 |
 | ---| ---| ---| ---| ---|
 | 1 | 2 | 3 | 2 | 3 |
 | 2 | 3 | 2 | 2 | 3 |

 由于外部容量限制，两场比赛都只允许 2 支实力较强的球队参加。 即使 S = 3，在任何比赛中也只能实现 2。 

这显示了第二场比赛约束在限制有多少强队可以同时集中在一项赛事中的作用。

 ## 边缘情况

 一种重要的边缘情况是，一场比赛的 ci 非常大，而其他比赛的 ci 都很小。 在这种情况下，外部能力成为限制因素，并且在单场比赛中增加更多能力可能不会增加可以被迫参加比赛的更强球队的数量。 

另一种极端情况是，一所大学拥有许多实力较强的团队，但总体比赛名额却很少。 即使每场比赛的 ci 都很大，第二次出场的总和限制也会限制可以同时参加任何一场比赛的实力较强的球队的数量。 

最后一个边缘情况是当一个团队是其大学中最强的团队时。 在这种情况下，S = 0，并且无论竞赛结构如何，该公式都正确地得出排名 1，因为没有更强大的竞争对手可以被迫领先。
