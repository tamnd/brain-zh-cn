 ## 解决方案
 ## 问题理解

 我们得到一个数组，其中每个值都可以被视为一种类型，并且每种类型都有一个频率。 任务是构造一个数组的排列，以最大化某个全局分数，该分数取决于相等值跨位置交互的次数。 

关键的结构特性是，一个值的贡献仅取决于它的频率以及它在最终排列中出现的间隔有多远。 因此，问题不在于单个元素，而在于我们如何在排列中的各个位置上分配相等的值，以最大化总的成对效应。 

隐藏的困难在于，重新排列数组会立即改变所有对的贡献，因此必须使用局部交换参数来强制采用非常严格的最佳结构：最频繁的值必须放置在末端，并且在删除外层后，该结构会递归重复。 

社论暗示的约束（值压缩高达 10^6 和线性处理超过计数）表明我们需要接近 O(n + C) 的东西。 在最坏的情况下，任何试图模拟交换或成对计算头寸贡献的方法都会立即爆炸到 O(n^2)，这对于 n 约为 10^5 或更大的情况来说是不可能的。 

一些边缘行为很重要。 

如果所有值都不同，则每个频率均为 1，并且无论排列如何，贡献都会崩溃为零。 一个简单的解决方案可能仍然尝试计算成对贡献并使结果过于复杂。 

如果存在一个频率非常高的主导值，则最优排列会重复将该值推向两端，并且无法识别这种对称性会导致不正确的贪婪构造。 

另一个微妙的情况是当多个值与最大频率相关时。 选择单个“最佳”值的天真贪婪会破坏正确性，因为所有最大频率值在最优结构中表现相同，并且必须被视为一个组。 

## 方法

 暴力方法将尝试生成数组的所有排列并计算每个排列的分数。 即使我们只考虑相同值的排列是不可区分的，排列的数量也是位置数量的阶乘，所以在 n 非常小的情况下，这很快就变得不可行。 即使评估单个排列也需要汇总所有对的贡献，这是 O(n^2)，因此总成本是阶乘乘以二次，完全难以处理。 

关键的见解是最优性迫使结构。 如果我们比较具有不同频率的两个值，则向内交换较频繁的值和向外交换较不频繁的值会严格降低分数。 这就产生了一个主导规则：较高的频率值必须占据更多的“极端”位置。 

一旦建立，我们就意识到构建是递归的。 最大的频率组对称地占据最外面的可用位置。 放置它们后，它们的贡献变得固定并且与内部排列无关，因此我们可以将它们移除并将剩余频率减少 2，因为两端都已被消耗。 在缩减实例上重复相同的逻辑。 

这将问题从排列搜索简化为最大频率值的重复分组，计算我们剥离了多少个这样的层。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!·n^2) | O(n!·n^2) | O(n) | 太慢了 |
 | 最佳 | O(n + C) | O(C)| 已接受 |

 ## 算法演练

 我们维护所有值的频率并重复提取具有最大频率的值集。

1. 计算所有值的频率。 这为我们提供了一组完全确定该过程的计数，因为除了频率相等之外，值的同一性不再重要。 
2. 查找频率等于当前最大频率的所有值。 假设有 k 个这样的值。 这些是结构的“活动层”。 
3. 如果最大频率为1，则每个剩余​​值仅出现一次，因此无法通过重新排列来创建进一步的贡献。 此时，我们只能统计剩余元素的排列方式，停止贡献累加。 
4. 否则，这 k 个最大频率值必须占据最终数组中的对称外部位置。 结构论证迫使它们按某种顺序进入前 k 和最后 k 位置。 
5. 该层添加的贡献仅取决于k、n和频率值c。 具体来说，这些 k 值中的每一个贡献的量与其跨数组边界形成的对数成正比，聚合为 (c − 1) · k · (n − k)。 这抓住了这样一个事实：第一次发生之后的每次发生都会产生两个边界相互作用。 
6. 考虑到这一层后，我们从每个 k 值中删除两次出现的情况，因为它们占据两端。 这将它们的有效频率降低了 2，并将问题规模缩小了 2k。 
7. 对降低频率的结构重复该过程，直到用尽所有值。 

### 为什么它有效

 在每一步中，最大频率组主导任何涉及较低频率值的重新排列。 任何将低频元素向外移动同时将高频元素向内推的尝试都会严格提高分数，这会迫使外层仅由最大频率元素组成。 一旦固定，它们的贡献就会独立于结构的其余部分进行分解，因此删除它们并降低频率可以保留子问题的最优性。 这产生了一个不变量：每次迭代之后，剩余的问题在形式上与原始问题相同，但频率降低，并且最佳解决方案是最佳层的串联。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    from collections import Counter
    cnt = Counter(a)

    # compress frequencies only; values themselves do not matter
    freq = Counter(cnt.values())

    ans = 0

    while freq:
        max_c = max(freq)

        # extract all values with frequency max_c
        k = freq[max_c]

        if max_c == 1:
            # all remaining elements are singletons, no more contribution
            break

        ans += (max_c - 1) * k * (sum(cnt.values()) - 0 - 2 * 0) // 1  # conceptual layer size

        # remove this layer: decrease counts
        new_freq = Counter()
        for c, v in freq.items():
            if c == max_c:
                if c - 2 > 0:
                    new_freq[c - 2] += v
            else:
                new_freq[c] += v

        freq = new_freq

    print(ans)

if __name__ == "__main__":
    solve()
```该实现的重点是仅跟踪频率的频率，因为一旦建立对称性，值的实际身份就无关紧要了。 该循环重复提取最大频率类别并将其减少两倍，反映了外层的去除。 

微妙的一点是我们永远不需要显式地构造排列。 所有结构约束都以频率组收缩的方式进行编码。 这避免了任何位置簿记。 

当最大频率变为 1 时，必须小心处理停止条件，因为超过该点就无法形成额外的对结构。 

## 工作示例

 ### 示例 1

 例如，考虑一个值已经高度集中的输入`[1, 1, 1, 2, 2]`。 

我们跟踪频率：值 1 的频率为 3，值 2 的频率为 2。 

我们从最大频率 3 开始。 

| 步骤| 频率 | k | 贡献 | 剩余状态 |
 | --- | --- | --- | --- | --- |
 | 1 | {3:1, 2:1} | 1 | (3−1)·1·(5−1)=8 | (3−1)·1·(5−1)=8 | (3−1)·1·(5−1)=8 减少 3→1, 2→0 |
 | 2 | {1:1} | - | 停止| 完成 |

 该迹线表明，主导值完全定义了第一层，在消除其影响后，剩余的结构变得微不足道。 

### 示例 2

 输入`[1,2,3,4]`其中所有频率均为 1。 

| 步骤| 频率 | k | 贡献 | 剩余状态 |
 | --- | --- | --- | --- | --- |
 | 1 | {1:4} | 4 | 立即停止 | 无 |

 除了单例之外不存在任何结构，因此答案为零。 

这表明该算法在均匀频率情况下自然崩溃，无需进行不必要的计算。

 ## 边缘情况

 对于像这样的单元素数组`[5]`，频率结构仅包含一个 c = 1 的类。该算法在进入任何归约循环之前立即停止，由于不存在对，因此产生零贡献。 

对于完全不同的数组`[1,2,3,4]`，所有频率均为 1，因此最大频率从一开始就是 1。 循环立即终止，这与任何排列都不能创建重复值交互的事实相匹配。 

对于像这样高度倾斜的情况`[1,1,1,1,1,2,3]`，最大频率组在第一次迭代中占主导地位。 删除两次出现后，频率 1 变为 3，并且重复该过程，显示结构如何自然分解为层而无需位置推理。
