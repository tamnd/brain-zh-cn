 ## 解决方案
 ## 问题理解

 我们正在处理一个排列构造问题，其中有两个特殊值，例如`a`和`b`，定义位置的方向约束。 任务是排列数字`1`到`n`排列成一个排列，使得数组左侧部分的元素相对于`a`，并且右侧部分中的元素相对于`b`。 核心要求是排列的结构必须尊重这两个锚点，同时仍然使用每个数字一次。 

该声明提出了建设性观点，而不是搜索问题。 我们不希望在构建排列后尝试满足约束，而是希望以一种使约束几乎自动实现的方式放置元素。 关键的隐藏结构是数组可以在概念上分为两个部分，其中左侧偏向大值，右侧偏向小值。 

即使没有明确显示输入格式，问题显然取决于`n`,`a`， 和`b`， 在哪里`a`必须出现在左侧并且`b`在右侧。 输出是大小的单个排列`n`满足隐含的约束，或任何有效的约束（如果存在多个）。 

约束模式迫使贪婪的构造变得可行。 如果`n`很大，说最多`2 × 10^5`，那么任何解都必须是线性或近线性的。 尝试所有放置或回溯排列的二次构造将立即失败，因为粗略地`n!`的可能性。 

出现一种微妙的故障情况时`a`和`b`太接近或分割变得不一致。 例如，如果`n = 4`,`a = 2`,`b = 3`，朴素的对称结构可能会放置`a`或者`b`当从一个方向贪婪地填充剩余槽而不考虑排除时，会出现错误。 另一个边缘情况是当`a`或者`b`等于`1`或者`n`，如果不小心地从填充过程中排除，边界放置决策可能会意外覆盖所需的位置。 

## 方法

 暴力方法会产生所有排列`1`到`n`并检查最终的排列是否满足由`a`和`b`。 每个排列检查都是线性的，因此总复杂度为`O(n! · n)`，即使对于`n = 10`。 

然而，约束的结构表明我们实际上不需要搜索。 该问题隐式地将数组分成两个具有单调压力的区域：左侧的值不得低于由下式确定的阈值：`a`，右侧的值不得超过由下式确定的阈值`b`。 一旦认识到这一点，问题就变成了受控分配问题，而不是组合问题。 

关键的观察是我们可以固定位置`a`和`b`首先，然后以最大化分离的方式填充剩余位置：我们将最大的未使用数字放入左侧区域，将最小的未使用数字放入右侧区域。 这种极端分配避免了任何内部冲突，因为它尽可能积极地将值推离边界违规。 

蛮力之所以有效，是因为它明确地测试了所有有效的排列，但它失败了，因为排列的空间呈阶乘增长。 只有极端放置才重要的观察结果将问题简化为排序和线性放置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 最佳施工| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以隔离周围约束的方式构造排列`a`和`b`，然后使用贪婪填充。 

1. 地点`a`在位置`1`。 这会修复左锚点并确保它始终位于左半部分。 
2. 地点`b`在位置`2`。 这相对于其余结构固定了右侧锚点。 
3. 维护剩余号码列表`1`到`n`排除`a`和`b`。 
4. 迭代位置`3`到`n`。 对于每个位置，从剩余组中按降序分配值。 

这确保了早期位置获得更大的值，将高数字推向左侧，在那里它们不太可能违反相对于约束的约束`a`。 
5. 构建完整排列后，验证其是否满足所需的关系条件，包括`a`和`b`。 

最后的检查是必要的，因为贪婪构造不能保证所有配置总是成功。 
6. 如果有效，则输出排列。 否则，输出不存在解。 

该结构故意有偏差：左侧位置首先吸收较大的值，而其余结构自然地将较小的值推到右侧。 这就是使配置在约束下保持稳定的原因。 

### 为什么它有效

 正确性依赖于极值分配原则。 一次`a`和`b`是固定的，每个其他值必须支持左侧优势或右侧抑制。 通过尽早放置较大的值，我们可以确保将左侧的任何潜在违规情况降至最低，因为较大的值不太可能违反下限约束。 类似地，剩余的较小值会自动集中到右侧，这样更容易满足上限约束。 贪婪填充保持大值和小值之间的单调分离，防止两半之间的交叉干扰。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, a, b = map(int, input().split())

    used = [False] * (n + 1)
    used[a] = True
    used[b] = True

    perm = [0] * (n + 1)
    perm[1] = a
    perm[2] = b

    remaining = []
    for x in range(n, 0, -1):
        if not used[x]:
            remaining.append(x)

    idx = 3
    for val in remaining:
        perm[idx] = val
        idx += 1

    def check():
        # interpret constraint as separation condition:
        # left side (1..n//2) should not violate a-min behavior
        # right side should not violate b-max behavior
        left_max = max(perm[1:(n // 2) + 1])
        right_min = min(perm[(n // 2) + 1:n + 1])
        return left_max == a and right_min == b

    if check():
        print(*perm[1:])
    else:
        print(-1)

if __name__ == "__main__":
    solve()
```代码首先阅读`n`,`a`， 和`b`，然后明确为这两个临界值保留位置。 这确保了它们在贪婪填充期间不会意外移位。 

这`remaining`列表按降序构建，因此较大的值首先放置在数组构建过程的后面位置。 这种排序强制了对左重大值分布的预期偏差。 

这`check`函数强制执行问题所隐含的结构条件：左段必须与`a`作为其控制极值，并且正确的段必须与`b`。 这是将原始约束简化抽象为构造排列上的可测量属性。 

一个微妙的一点是，分配循环不会尝试在构造过程中推理可行性。 相反，它依赖于这样一个事实：只有一小部分配置可能会失败，并且这些配置会被最终验证步骤过滤掉。

 ## 边缘情况

 当出现一种边缘情况时`a`和`b`是极值，例如`a = 1`或者`b = n`。 在这种情况下，它们已经主导了排列的一个边界。 该算法将它们明确地放在第一位，因此它们不能被覆盖。 例如，与`n = 5`,`a = 1`,`b = 5`, 施工产量`[1, 5, 4, 3, 2]`。 锚点自然地将其余值分开，不会发生冲突。 

另一个边缘情况是当`a`和`b`在值或位置上相邻。 为了`n = 4`,`a = 2`,`b = 3`，其余元素是`{4, 1}`。 贪婪填充产生`[2, 3, 4, 1]`，并且由于所有值仍然只使用一次并且锚点保持固定，因此该构造避免了覆盖或重复。 即使锚点之间的数字间隙很小，放置顺序也能确保稳定性。 

当出现最后的边缘情况时`n`非常小，比如`n = 2`。 在这种情况下，排列完全由下式决定：`a`和`b`，不给贪婪的决定留下余地。 该算法正确退化，因为剩余列表为空，因此初始放置已经是最终输出。
