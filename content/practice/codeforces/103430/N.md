 ## 解决方案
 ## 问题理解

 我们得到了三个单独的文本行，并且必须根据目标元音计数模式检查每一行。 该模式固定为第一行5个元音、第二行7个元音、第三行5个元音。 

然而，奇怪的是这封信`y`（大写和小写）是不明确的。 它最初不被视为元音，但如果需要，可以选择将其视为元音。 其他所有字母都是固定的：它要么是元音，要么是辅音，并且不能改变其角色。 

对于每一行，我们计算两个值。 第一个是`v`，行中定元音的数量，不包括`y`和`Y`。 第二个是`y`，出现的次数`y`或者`Y`。 由于每个`y`可以独立地充当元音或辅音，该行中可实现的元音数量形成一个区间`v`到`v + y`。 

一行对于所需目标有效`V`当且仅当该目标位于该区间内时。 所有三行必须同时满足各自的目标才能接受整个输入。 

由于输入仅由三个短字符串组成，因此约束很小。 即使是直接扫描字符也足够快，因此任何与总输入大小呈线性的解决方案就足够了。 除了每行一次传递之外，不需要高级数据结构或优化。 

一个常见的错误来自于治疗`y`始终是元音或始终是辅音。 例如，考虑这一行`"bYb"`。 这里`v = 0`和`y = 1`，所以可能的元音计数是`0`或者`1`。 如果我们数错了`Y`与往常一样，我们会强迫它做出贡献，并且当需要更严格的目标时，可能会错误地得出可行性结论。 同样，始终将其视为辅音可能会错误地拒绝需要达到目标的情况。 

## 方法

 强力解释将明确尝试每个的所有分配`y`字符为元音或辅音。 对于包含以下内容的行`k`的出现次数`y`，这导致`2^k`配置，对于每个配置，我们都会计算元音并检查结果值是否与目标匹配。 跨越三行，在最坏的情况下这会变成指数，并且考虑到问题的结构是不必要的。 

关键的观察是我们实际上不需要枚举分配。 每个`y`独立地将最大可能元音计数恰好增加 1，同时不影响最小值。 这意味着每行都压缩成一个简单的区间`[v, v + y]`。 然后，问题简化为检查固定整数是否位于间隔内，该间隔是每行的恒定时间。 

这种变换是有效的，因为每个`y`独立贡献，不与其他角色互动。 一旦我们认识到这种独立性，指数选择空间就会崩溃为简单的范围检查。

 | --- | --- | --- | --- |
 | 蛮力 | 每行 O(2^k) | O(k) | 太慢了 |
 | 最佳 | 总共 O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理这三行中的每一行，并且对于每一行，我们计算它是否可以满足其所需的元音计数。 

1.读取一行并初始化两个计数器，`v = 0`对于确定元音和`y = 0`对于暧昧的`y`人物。 这种分离至关重要，因为只有`y`有助于灵活性。 
2. 迭代该行中的每个字符。 如果该角色是以下之一`a, e, i, o, u`在任何一种情况下，增量`v`。 如果角色是`y`或者`Y`, 增量`y`。 出于计数目的，所有其他字符都将被忽略。 分类是线路的整个状态。 
3. 根据该行的位置确定该行所需的元音数：第一个为 5，第二个为 7，第三个为 5。 
4. 使用区间条件检查可行性`v ≤ V ≤ v + y`。 如果任何一行失败，则整个测试用例无效，我们可以提前停止。 
5. 处理完所有三行后，如果所有条件成立，则输出“YES”，否则输出“NO”。 

### 为什么它有效

 每个`y`字符恰好贡献了一个灵活性单位：它可以保持辅音或成为元音。 没有其他角色具有此属性。 因此最小元音数固定为`v`，并且通过转换每个来实现最大值`y`变成元音，给出`v + y`。 由于所有选择都是独立的，因此该范围内的每个整数值都是可以实现的，并且不可能有超出该范围的值。 因此，可行性检查完全相当于检查每条线的目标是否位于该区间内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

VOWELS = set("aeiouAEIOU")

def check_line(s, target):
    v = 0
    y = 0
    for c in s:
        if c == 'y' or c == 'Y':
            y += 1
        elif c in VOWELS:
            v += 1
    return v <= target <= v + y

def solve():
    lines = [input().rstrip('\n') for _ in range(3)]
    targets = [5, 7, 5]
    
    for s, t in zip(lines, targets):
        if not check_line(s, t):
            return "NO"
    return "YES"

if __name__ == "__main__":
    print(solve())
```该实现直接反映了区间推理。 辅助函数`check_line`隔离计数逻辑，将确定元音与灵活元音分开`y`人物。 这避免了很容易导致差一错误的混合逻辑。 

由于模式 5-7-5 是固定的，因此目标按顺序进行硬编码。 早期的失败返回确保我们不会执行不必​​要的检查，尽管实际上输入大小非常小，这纯粹是风格上的。 

一个微妙的细节是案件处理。 转换整个字符串是不必要的，因为我们显式检查大写和小写变体，这避免了额外的开销并保持逻辑透明。

 ## 边缘情况

 一个重要的边缘情况是当一行仅包含`y`人物。 例如：```
y
```这里`v = 0`和`y = 1`，所以可能的元音计数是`[0,1]`。 如果目标是 5 或 7，则条件`v ≤ V ≤ v + y`正确地拒绝它，因为 5 超出了可实现的范围。 该算法自然地处理这个问题，因为间隔检查在没有任何特殊大小写的情况下失败。 

另一种情况是当没有`y`根本没有字符。 例如：```
aeiou
```这产生`v = 5`和`y = 0`，因此唯一可实现的值恰好是 5。如果所需的值为 7，则条件失败，因为`7 ≤ 5 + 0`是假的。 对于这种情况，算法不需要分支； 它已经在相同的间隔检查中被编码。 

最后一个微妙的情况是混合大小写，例如：```
AeIyOu
```这里通过显式检查统一处理大小写字母，所以`v`无论大小写，正确计算所有标准元音，并且`y`有助于灵活性而不影响正确性。
