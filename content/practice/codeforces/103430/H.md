 ## 解决方案
 ## 问题理解

 我们有一组学生。 每个学生都与一个特定的消息索引和一个限制值相关联，该限制值根据固定的消息总数控制他们读取固定消息的可靠性。 

我们可以选择一定数量的消息并“固定”它们。 之后，每个学生独立地阅读或不阅读自己的信息。 概率不是任意的，而仅取决于他们的消息是否被固定以及固定集有多大。 

如果学生的消息不在固定消息之列，则该学生不会做出任何贡献。 如果它被固定，贡献取决于固定消息的总数，用 t 表示。 当 t 相对于学生的阈值 ki 足够小时，学生肯定会阅读该消息。 当 t 超过 ki 时，机会被稀释并等于 ki 除以 t。 

任务是选择固定消息的数量以及要固定的消息，以使阅读其消息的预期学生数量最大化。 

这些约束意味着直接搜索所有子集是不可能的，因为有多种方法可以选择固定消息。 即使迭代所有子集大小并天真地重新计算值也会导致大约 O(n^2) 的行为，当 n 很大时，这太慢了。 

当 t 增长超过所有 ki 值时，行为会产生微妙的边缘情况。 由于每个 ki 的界限都是 20，因此描述贡献的函数在超出小阈值后就会停止以有意义的方式变化。 继续计算大 t 的幼稚实现可能会浪费计算，同时也无法实现最佳结构的稳定。 

例如，考虑所有 ki 都等于 1 的情况。如果 t 变为 50，则每个选定的学生贡献 1/50，并且显然已经由小得多的 t 确定了最佳策略。 持续评估大 t 值的蛮力方法将重复重新计算具有缩放值的基本相同的排名结构。 

另一个极端情况是所有消息都具有较大的 ki 值。 在这种情况下，对于较小的 t，每个选定的学生都贡献 1，问题就简化为任意选择最佳的 t 消息。 过早错误地应用分数规则的幼稚实现会低估该值。 

## 方法

 我们从直接的观点开始。 假设我们固定固定消息的数量 t。 如果我们知道固定了哪些消息，则通过对每个选定学生的贡献求和即可完全确定满意学生的预期数量。 

对于固定的 t，如果每个学生 i 的阈值 ki 至少为 t，则贡献 1，否则贡献 ki 除以 t。 这为特定 t 处的每个候选消息提供了确定性值。 因此，对于每个 t，最佳策略是根据消息在该 t 下的贡献对消息进行排序，并选择其中最靠前的 t。 

这立即产生了一种正确但昂贵的方法。 对于每个可能的 t（最多 n），我们为每条消息计算一个值，对它们进行排序，并取最好的 t。 这导致 O(n^2 log n) 时间，对于大输入来说太慢了。 

关键的结构观察是 ki 以 20 为界。一旦 t 超过 20，学生就不再进入“满值等于 1”状态。 相反，所有贡献都与 ki 除以 t 成正比，这只是 ki 的统一缩放。 这意味着，对于大于 20 的 t，消息的排序永远不会改变，只是它们的总和按 1/t 缩放。 增加 t 超过该点不能产生新的最佳配置； 它只会削弱贡献，同时迫使我们选择更多元素。 

这使我们能够将注意力限制在 20 以内的 t 值上。除此之外，最佳选择的行为是单调的，无法击败 1 到 20 范围内的某些 t。

因此，我们通过重新计算贡献、排序和选择最佳 t 元素来评估从 1 到 20 的每个 t 的最佳答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 t 至 n | O(n^2 log n) | O(n^2 log n) | O(n) | 太慢了 |
 | t ≤ 20 时的优化截止 | O(20 · n log n) | O(20 · n log n) | O(n) | 已接受 |

 ## 算法演练

 我们确定固定消息 t 的候选数量，并计算该 t 的最佳可实现期望。 

1. 对于固定的 t，当 ki 至少为 t 时，计算每个消息 i 的贡献为 1，否则 ki 除以 t。 这准确地编码了学生阅读是否确实受益或仅部分受益。 
2. 将所有这些贡献收集到一个列表中。 
3. 按降序对列表进行排序。 这是必要的，因为对于固定数量的固定消息，我们总是希望选择边际贡献最大的 t 条消息。 
4. 从排序列表中取出前 t 值并对它们求和。 这代表了该固定 t 的最佳预期读者数量。 
5. 对从 1 到 20 的每个 t 重复相同的过程。 
6. 最终答案是所有测试t 的最大值。 

ki 的有限性质证明 20 的截止值是合理的。 一旦 t 超过最大 ki，每个贡献都会进入分数状态，所有值都变为 ki 除以 t。 此时，最佳子集始终是同一组最高 ki 值，只是重新缩放，因此进一步增加 t 不能引入更好的配置。 

### 为什么它有效

 对于每个固定 t，该算法通过对确定性每个元素贡献的贪婪选择来精确计算最佳子集。 元素之间唯一的耦合是必须精确选择 t 的约束，而排序可以最佳地解决这个问题。 

对 t 最大为 20 的限制保留了最优性，因为贡献的每个结构变化仅发生在整数阈值 ki 处。 由于所有 ki 都位于一个小的有界范围内，因此目标的所有有意义的转变都发生在该范围内。 除此之外，目标变成了固定顺序上的缩放线性函数，因此不会出现新的最佳值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    if n == 0:
        print(0)
        return

    max_t = min(20, n)
    ans = 0.0

    for t in range(1, max_t + 1):
        vals = []
        inv_t = 1.0 / t

        for k in a:
            if k >= t:
                vals.append(1.0)
            else:
                vals.append(k * inv_t)

        vals.sort(reverse=True)
        ans = max(ans, sum(vals[:t]))

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案循环遍历 t 的可能值，最大可达 20。对于每个 t，它都会构建所有消息的贡献列表。 条件 k >= t 完全对应于确定性读取情况，而 k < t 触发分数缩放。 

排序至关重要，因为一旦贡献确定，固定哪些消息的选择是独立的。 取最大的 t 值保证了该 t 的最佳选择。 

一个常见的实现陷阱是忘记将 t 限制为 20。如果没有这种优化，解决方案将降级为 O(n^2 log n)。 另一个微妙的问题是将整数除法与浮点除法混合在一起。 使用浮点值可确保正确的比较和累加。

 ## 边缘情况

 当所有 ki 值都等于 1 时，每个大于 1 的 t 都会将所有贡献推入分数状态。 例如，当 n 等于 3 并且 k 等于 [1, 1, 1] 时，在 t 等于 2 时，每个选定元素贡献 1/2。 该算法评估 t 等于 1 和 t 等于 2，分别生成值 1 和 1.5，并正确选择 t 等于 2。 

当所有 ki 值都很大时，例如 [20,20,20]，t 等于 1 时的行为是微不足道的，因为每个值都贡献 1。在 t 等于 2 时，所有贡献仍为 1，并且顶部选择仍会产生完整值。 该算法仍然比较两种情况并一致地进行选择，而不依赖于有关分布的假设。 

当 n 很小时（例如 n 等于 1），t 上的循环仍然可以正确运行，因为 max_t 被限制为 1，从而确保不会尝试无效的选择大小。
