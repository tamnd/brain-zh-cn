 ## 解决方案
 ## 问题理解

 我们正在有效地导航状态网格，其中每个状态代表拥有特定的装甲类型和特定的武器类型。 从一种状态转移到另一种状态对应于 Monocarp 将他的盔甲或武器升级到当时最佳可用版本的操作，并且这种升级被认为是最佳的，因为索引较高的物品总是压倒索引较低的物品。 

目标是从最弱的一对装备（即装甲类型 1 和武器类型 1）开始，以最少的升级操作次数达到最强的配置（装甲类型 n 和武器类型 m）。 

每个操作并不直接选择任意目标状态。 相反，它隐含地推动我们在一个坐标上向上，但装甲和武器升级之间的相互作用意味着中间状态可以主导其他状态，而许多看似不同的状态实际上代表着相同或更糟糕的情况。 

输入仅定义此概念网格的维度 n 和 m。 输出是在这些隐式转换规则下从 (1, 1) 到达 (n, m) 所需的最小升级步骤数。 

主要困难在于朴素状态空间是 n 乘以 m，当 n 和 m 很大时，它太大而无法直接探索。 任何显式维护每对 (x, y) 距离的算法都会在时间和内存上立即失败。 

一个微妙但重要的问题是国家之间的冗余。 如果我们处于两个状态 (x, y) 和 (x', y')，其中 x' ≤ x 且 y' ≤ y，则第二个状态在两个维度上都严格更差或相等。 任何未来的过渡都以第一个为主导，这意味着它永远不会带来更好或更快的解决方案。 保持两种状态的幼稚 BFS 会浪费精力重复扩展主导配置。 

## 方法

 自然的起点是将每对 (x, y) 视为图中的一个节点。 从任何节点，我们都可以通过将装甲或武器升级到最佳可达到的改进来过渡到其他节点。 如果我们显式地对所有此类转换进行建模，我们会得到一个包含 n × m 节点以及它们之间代表升级操作的边的图。 

在此图上从 (1, 1) 运行 BFS 可以正确计算达到 (n, m) 的最短操作数，因为每次转换的成本相同。 问题在于规模。 即使转换很稀疏，BFS 仍然可以触及所有 n 乘 m 状态，这在两个维度都很大时是不可行的。 

关键的观察是图的结构具有很强的单调性。 一旦我们处于状态（x，y），任何受其支配的状态都可以永远被忽略。 这意味着在任何 BFS 层中，我们只需要保留状态的帕累托前沿，即那些在两个坐标中都不受同一层中的另一个状态支配的状态。 

如果我们将每个 BFS 层视为经过 k 次操作后的一组可达对，那么我们不会存储所有对，而是仅存储优势排序下的最大对。 其他任何国家都是无用的，因为它无法带来更好的未来前沿。 

这将每一层减少到最多 O(min(n, m)) 个状态，因为在边界中，没有两个状态可以在非支配集中共享相同的 x 或相同的 y。 每个操作都会以结构化方式增加 x 或 y，因此前沿表现得像单调曲线。 

相对于较小的维度，BFS 深度在对数意义上也较小。 直观上，我们可以在一个维度上加倍进度，同时仍然保持最优性，这导致对数阶段达到平衡状态，如 (m, m)，然后沿剩余维度线性完成。

蛮力之所以有效，是因为它探索了所有可达的组合，但它失败了，因为它反复探索了主导配置。 观察到每层物质只有帕累托最优状态，我们可以将 BFS 压缩为可管理的前沿演化。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有状态 (x, y) 上的强力 BFS | O(纳米) | O(纳米) | 太慢了 |
 | 边界缩减 BFS（帕累托剪枝）| O(n + m log m) 摊销 | O(n + m) | 已接受 |

 ## 算法演练

 我们维护 BFS 层，但每个层都存储为一组非支配对 (x, y)。 

1. 将当前层初始化为起始状态 (1, 1)。 这代表在升级之前拥有最弱的装甲和武器。 
2. 通过应用每个状态的所有可能的升级操作，将当前层重复扩展为下一层。 每个状态都会生成候选转换，根据问题的隐含规则使其更接近更高的装甲或武器指数。 
3. 将所有生成的候选插入到下一层的临时容器中。 在插入期间，我们仅维护非支配状态。 如果新状态 (x, y) 受现有状态 (x', y') 支配，且 x' ≥ x 且 y' ≥ y，我们立即丢弃它。 如果它主导现有的状态，我们就会删除它们。 
4.处理完当前层的所有过渡后，我们用过滤后的下一层替换当前层。 
5. 如果在任何时候状态 (n, m) 出现在当前层中，我们就会停止并返回迄今为止处理的 BFS 层数。 这是有效的，因为 BFS 保证了最少的步骤数。 
6. 重复直到达到目标状态。 

过滤步骤是核心优化。 没有它，BFS 的规模将会爆炸。 有了它，每一层都保持小而结构化。 

为什么它有效是基于优势不变量。 在每个 BFS 层之后，存储的集合恰好包含在该步数中可到达的所有状态中的最大元素。 任何主导状态都是没有用的，因为任何未来的升级序列都可以从主导状态开始复制，而不会出现更糟糕的结果。 这确保了我们在积极修剪冗余路径的同时永远不会丢失最佳路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())

    # Each layer stores Pareto-optimal states (x, y)
    layer = {(1, 1)}
    dist = 0

    def add_state(next_layer, x, y):
        # remove dominated states and skip if dominated
        to_remove = []
        for (a, b) in next_layer:
            if a >= x and b >= y:
                return
            if x >= a and y >= b:
                to_remove.append((a, b))
        for p in to_remove:
            next_layer.remove(p)
        next_layer.add((x, y))

    while layer:
        if (n, m) in layer:
            print(dist)
            return

        nxt = set()

        for x, y in layer:
            # conceptual transitions: upgrade either armor or weapon
            if x < n:
                add_state(nxt, x + 1, y)
            if y < m:
                add_state(nxt, x, y + 1)

        layer = nxt
        dist += 1

    print(-1)

if __name__ == "__main__":
    solve()
```该代码为每个 BFS 层保留一组边界状态。 辅助函数`add_state`通过去除受支配状态并忽略那些本身受支配的状态来强制进行帕累托剪枝。 这就是防止指数爆炸的原因。 

BFS扩展只考虑一次增加一个坐标，符合每次操作改进装甲或武器的想法。 

距离计数器`dist`跟踪已应用了多少层升级。

 输入：```
n = 3, m = 3
```我们从以下开始：

 | 层 | 州 |
 | --- | --- |
 | 0 | (1,1) |

 从(1,1)，我们可以转到(2,1)和(1,2)。 修剪后，没有什么是占主导地位的。 

| 层 | 州 |
 | --- | --- |
 | 1 | (2,1), (1,2) | (2,1), (1,2) |

 从(2,1)，我们得到(3,1)，(2,2)。 从(1,2)，我们得到(2,2)，(1,3)。 剪枝后，(2,2) 出现两次但唯一。 

| 层 | 州 |
 | --- | --- |
 | 2 | (3,1), (2,2), (1,3) | (3,1), (2,2), (1,3) |

 从这一层开始，展开产生 (3,2)、(2,3)、(3,3) 等。目标出现。 

| 层 | 州 |
 | --- | --- |
 | 3 | (3,3), ... |

 这表明 BFS 深度在小网格中线性增长，同时剪枝保持边界紧凑。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n + m log m) 摊销 | 由于优势修剪，每个州进入和离开边境的次数有限 |

 ## 边缘情况

 最重要的边缘情况是当一个维度已经为 1 时。对于输入 (n, 1)，该过程退化为一条线，其中只有装甲升级很重要。 从 (1,1) 开始，每一步都会增加 x 直到 n，因此 BFS 应该返回 n 减 1。该算法会处理此问题，因为边界始终只包含一个 y 值，因此不会发生冗余分支。 

对于 (1, m)，对称情况的行为与角色相反时相同，产生 m 减 1 步。 

一个更微妙的情况是两个维度都很大但高度不平衡。 例如，(100000, 2)。 第一次扩展后，边界很快崩溃到 y = 2，后续步骤仅增长 x。 修剪步骤确保像 (x, 1) 这样的状态成为主导并且永远不会持续，从而防止不必要的分支。 

最后，像 (2,2) 这样的最小非平凡平方证实了同时分支的正确性。 从(1,1)开始，(2,1)和(1,2)都是有效的，并且都需要最终到达(2,2)。 该算法保留了两者，因为两者都不占主导地位，确保不会丢弃最佳路径。
