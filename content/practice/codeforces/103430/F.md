 ## 解决方案
 ## 问题理解

 给定一对正整数，我们重复应用始终作用于较大值的运算。 唯一允许的移动是用较大数字与较小数字的差值来替换较大数字。 该过程一直持续到其中一个数字变为零为止。 

任务是确定目标值是否可以在此过程中的任何点（包括中间状态）作为该对的任一组成部分出现，而不仅仅是在最后。 

重要的解释是该过程不是随机分支的。 一旦该对被固定，每个状态都是通过重复从较大的值中减去较小的值来确定的，当顺序翻转时偶尔交换。 

这些约束意味着这些值可能足够大，以至于不可能及时模拟每个减法。 简单的实现会重复地从一个数字中减去另一个数字，可能会执行与初始值成比例的线性数量的操作。 在最坏的情况下，例如当数字是连续的大整数时，减法链会退化为非常长的序列，使得直接模拟不可行。 

当目标值等于原始数字之一时，会出现微妙的边缘情况。 例如，如果该对是 (10, 6) 并且目标是 10，则答案立即为“是”，即使算法稍后将状态转换为远离 (10, 6)。 仅在终止时进行检查的幼稚实现会错误地错过这一点。 

另一种边缘情况发生在目标位于两个数字之间，但如果试图错误地优化而不跟踪中间可达残差，则目标会被大的减法步骤跳过。 

## 方法

 蛮力思想严格遵循字面规则。 在每一步中，我们都会从较大的数字中减去较小的数字，直到顺序翻转，然后再次继续。 这模拟了欧几里得算法，但也隐式枚举了每个中间状态。 由于每次减法都是 O(1)，因此步骤数仍然可能很大。 在最坏的情况下，当数字很接近时，减法会一一发生，导致 O(max(a, b)) 运算。 

从概念上讲，这是可行的，因为每个可达状态都是由这种确定性归约过程产生的。 然而，瓶颈在于大多数减法并没有有意义地改变状态的结构，它们只是通过重复复制另一个坐标来减少一个坐标。 

关键的观察结果是，在我们反复从 a 中减去 b 的阶段，a 的值会演变为 a、a - b、a - 2b 等。 我们可以直接跳转到不等式条件发生变化之前的最后一个有效值，而不是逐一执行这些操作。 这与欧几里得算法中用除法代替重复减法时使用的优​​化完全相同。 

在每个阶段，我们计算可以从较大的数字中减去较小的数字多少次，同时仍然保持过程有效。 这让我们可以在对数时间内模拟整个链。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(最大(a, b)) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(log max(a, b)) | O(log max(a, b)) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们假设当前对已排序，使得 a 至少为 b。 该算法重复压缩长减法运行。

1. 确保a是较大的值，必要时交换。 这保持了当前阶段所有操作都从 a 中减去 b 的不变性。 
2. 检查目标值x是否已经与a或b匹配。 如果是这样，我们可以立即停止，因为该进程已经达到了有效状态。 
3. 如果 b 大于 a 减去 b，则我们处于减法将立即违反排序条件的状态。 在这种情况下，我们只执行有限数量的安全减法来保持结构有效。 
4. 计算在低于阈值或超出目标区域之前我们可以从 a 中减去 b 多少次。 这是代替重复减法的压缩步骤。 
5. 将 a 更新为 a 减去 cnt 乘以 b，其中 cnt 是计算的安全减法次数。 
6. 重复该过程，直到 a 或 b 变为零，或者遇到目标值。 

关键思想是，算法的每个阶段都与欧几里得算法的一部分精确对应，其中一个坐标以另一个坐标为模进行批量缩减。 

### 为什么它有效

 该过程保留了以下不变量：每个可达状态都相当于按某种顺序重复从较大值中减去较小值。 强力减法过程中出现的任何状态都必须位于由当前对定义的算术级数序列上。 通过直接跳转到每个级数的边界，我们不会跳过任何可能等于目标的值，因为每个中间值都以较小的数为模而全等，并且明确地由减法范围来解释。 因此，压缩在消除冗余步骤的同时保留了可达性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, x = map(int, input().split())
    
    while a and b:
        if a < b:
            a, b = b, a
        
        if a == x or b == x:
            print("YES")
            return
        
        if b == 0:
            break
        
        if a - b >= b:
            cnt = (a - b) // b
        else:
            cnt = 1
        
        cnt = max(1, cnt)
        
        if a - cnt * b <= 0:
            cnt = a // b
        
        a -= cnt * b
    
    if a == x or b == x:
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该代码保持了我们始终对排序对进行操作的不变性，以便减法方向是固定的。 对 x 的检查是在每次迭代的早期完成的，因为中间状态很重要，而不仅仅是最终状态。 

cnt的计算是关键的优化。 我们不是一次从 a 中减去 b，而是估计在结构改变之前可以执行多少次完全有效的减法。 这种调整可确保我们永远不会超过零或跳过相对顺序会发生变化的区域。 

最后，如果循环由于一个值​​变为零而结束，我们将执行最后一次检查，因为最终状态也是可达序列的一部分。 

## 工作示例

 考虑输入 (a, b, x) = (10, 6, 4)。

 | 1 | 10 | 10 6 | 1 | a = 10 - 6 = 4 |

 在第1步，直接出现值4，所以答案是YES。 该迹线表明中间值至关重要，因为目标不一定存在于最终状态中。 

现在考虑 (a, b, x) = (15, 7, 1)。 

| 步骤| 一个 | 乙| 碳纳米管| 行动|
 | --- | --- | --- | --- | --- |
 | 1 | 15 | 15 7 | 1 | a = 8 |

 | 3 | 1 | 7 | 交换 | (7, 1) |
 | 4 | 7 | 1 | 7 | 一个= 0 |

 这里目标值1在两次压缩后出现。 该过程显示了重复减法如何自然地生成余数的完整算术级数。

 ## 边缘情况

 当两个数字在开始时相等时，算法立即认识到每个减法步骤都保持相等，直到 1 变为零，因此任何等于初始值的目标都是可以实现的。 

当目标等于较小的数字时，它可能会出现在交换之后而不是直接减法期间。 交换步骤确保我们不会错过此配置。 

当一个数字除另一个数字时，该过程会分解为一串精确的减法。 该算法通过允许 cnt 直接跳转到商来处理此问题，确保不会跳过中间倍数，从而不会丢失 x 的潜在匹配。
