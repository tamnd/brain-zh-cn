 ## 解决方案
 ## 问题理解

 我们有两组独立的运动员，一组负责运动 A，一组负责运动 B。每个运动员都有一个数字技能值，每个运动员都必须留在自己的运动中，除非我们明确决定“交换”他们，这意味着他们参加另一项运动。 

目标是形成最终选择的恰好 k 个分配给运动 A 的运动员和恰好 k 个分配给运动 B 的运动员，从而在任何交换后最大化技能总和。 交换的成本非常特殊：如果一名运动员原本属于 A 运动，但被用于 B 运动，则其贡献会因罚金 x 而减少，同样，如果 B 运动员被用于 A 运动，则罚金为 y。 

所以结构不仅仅是独立地从每个列表中挑选最好的 k。 我们可以在不同的运动项目之间混合运动员，但每次交叉分配都会根据方向产生固定的损失。 

约束（隐含地来自 ICPC 样式设置）表明 n 可以很大，最多约为 2×10^5 或两个列表中的类似值。 这立即排除了为运动员的每一次可能的重新分配独立地重新计算部分总和的可能性。 任何重复排序、重新计算前缀或模拟每个配置选择的解决方案都会太慢。 我们需要一次排序和 O(n) 次转换。 

当每项运动独立应用贪婪选择时，就会出现微妙的边缘情况。 例如，假设将一名非常强的运动员从 B 换成 A，迫使将一名较弱的 A 运动员换成 B，并且由于 x 和 y 之间的差异，这种级联可能仍会提高总分。 天真的“从每一侧取出前 k”在这里失败了，因为它忽略了两个池之间的相互作用。 

另一种故障模式是假设对称：将交换 A→B 和 B→A 视为等效。 它们不是，因为惩罚 x 和 y 不同，并且最佳分割取决于这种不对称性。 

## 方法

 蛮力的想法是决定我们在最终 2k 个选定的角色中总共从运动 A 中选取多少名运动员 z。 一旦 z 固定，剩余的 2k − z 来自运动 B。对于每个 z，我们从每个排序列表中挑选最佳的可用运动员，然后应用交换以满足每项运动 k 的最终要求。 

如果我们固定 z，自然的贪婪结构就很清楚：从 A 中取出前 z 名运动员，从 B 中取出前 2k − z 名运动员。现在我们比较相对于每项运动所需的 k 有多少运动员处于“错误的一边”。 如果 z < k，我们缺少 A 分配，因此我们必须将 k − z 运动员从 B 转换为 A，每次交换支付惩罚 y。 如果 z > k，我们必须将 z − k 名运动员从 A 转换为 B，每次交换支付 x 罚金。 因此，总分是前缀和组合减去线性惩罚项。 

从头开始独立尝试所有 z 会重复重新计算前缀和，这会导致每个 z 的工作量为 O(k)，因此 O(k^2)。 添加排序后，这变成了 O(n log n + k^2)，当 k 很大时，这太慢了。 

关键观察结果是，随着 z 增加 1，在暂定选择中只有一名运动员从 B 移动到 A。 这逐渐改变了惩罚结构和前缀和。 如果我们维护两个排序列表的前缀和并跟踪当前的部分和，我们可以每一步更新总分 O(1)。 这会将 z 从 0 到 2k 的完整扫描转变为线性扫描。 

我们按降序对两个数组进行排序，以便前缀总和始终代表最佳可能的选择。 然后我们预先计算前缀和，以便可以立即查询任何 top-t 和。

 | --- | --- | --- | --- |
 | 蛮力 | O(k^2 + n log n) | O(k^2 + n log n) | O(n) | 太慢了 |
 | 最佳 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们按降序维护排序数组 A 和 B 以及它们的前缀和 SA 和 SB。

1. 按降序对两个数组进行排序。 这确保了任何前缀都代表每项运动中该尺寸的最佳选择。 
2. 预先计算前缀和 SA[i] 作为 A 中前 i 个运动员的总和，并类似地计算 B 中的 SB[i]。这允许对任何顶部分段总和进行恒定时间计算。 
3. 从 0 到 2k 迭代 z，将 z 解释为总共 2k 名选定运动员中从 A 中选出的运动员人数。 对于每个 z，B 中的数字为 2k − z。 
4. 将此配置的基本和计算为 SA[z] + SB[2k − z]。 这代表了任何交换之前的最佳原始技能总和。 
5. 调整最终任务的可行性。 如果 z < k，则我们没有足够的 A 标记运动员，因此我们必须将 k − z 运动员从 B 转换为 A。每次这样的转换都会使总数减少 y，因此减去 y · (k − z)。 如果 z > k，我们必须将 z − k 运动员从 A 转换为 B，因此减去 x·(z − k)。 
6. 跟踪所有 z 上的最大值。 
7.输出最好的结果。 

关键的细节是，每个配置在预处理后都会在恒定时间内进行评估，因此我们完全避免了重新计算。 

### 为什么它有效

 对于任何固定的 z，选择前 z 和前 2k − z 运动员是最优的，因为超出基数限制的选择之间不存在交互作用。 任何用较弱的运动员替换所选运动员的偏差只能减少前缀和。 两组之间唯一剩下的耦合是将每项运动的计数精确地平衡到 k，并且该耦合在不平衡中是线性的，与选择哪个特定运动员无关。 “最佳前缀选择”和“线性校正”之间的这种分离确保了对 z 的扫描探索所有结构上不同的最佳状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k, x, y = map(int, input().split())
    A = list(map(int, input().split()))
    B = list(map(int, input().split()))

    A.sort(reverse=True)
    B.sort(reverse=True)

    # prefix sums
    SA = [0]
    for v in A:
        SA.append(SA[-1] + v)

    SB = [0]
    for v in B:
        SB.append(SB[-1] + v)

    ans = 0

    # z = number taken from A among 2k total
    for z in range(0, 2 * k + 1):
        if z > len(A) or (2 * k - z) > len(B):
            continue

        base = SA[z] + SB[2 * k - z]

        if z < k:
            base -= y * (k - z)
        elif z > k:
            base -= x * (z - k)

        if base > ans:
            ans = base

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先对两个列表进行排序，以便任何前缀都对应于该大小的最佳可能选择。 前缀和允许对任何 z 选择进行恒定时间评估。 

z 上的循环检查两项运动之间所有可能的划分。 可行性检查确保我们不会访问无效的前缀索引。 调整步骤根据我们在运动 A 上的分配是否不足或过多来应用正确的惩罚。 

全局跟踪最大值并在最后打印。 该实现在很大程度上依赖于正确的前缀索引； 忘记 SB[2k − z] 必须始终在范围内是运行时错误的常见来源。

 ## 边缘情况

 一种重要的边缘情况是其中一个数组太小而无法满足 z 的极值。 例如，如果 k = 3，则 A 只有 2 个元素，而 B 有很多元素。 当z = 5时，SA[z]无效，必须被跳过。 该算法在访问前缀和之前显式检查边界，通过将不可能的配置视为非候选配置来确保正确性。 

当惩罚为零时，会出现另一种边缘情况。 在这种情况下，交换没有成本，因此最佳解决方案只是从并集中取出前 2k 个元素。 对 z 的扫描仍然有效，但最佳值出现在 z 自然地与两个数组中大值的分布对齐的位置。 

最后一种边缘情况是，一个列表严重支配另一个列表，从而使所有最佳配置偏向一侧。 例如，如果所有 B 值都远大于 A，则最佳 z 可能接近 0。对所有 z 的枚举可确保仍然评估该极端值，而不是假设 k 处的平衡。
