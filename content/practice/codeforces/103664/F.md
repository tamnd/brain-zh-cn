 ## 解决方案
 ## 问题理解

 我们有一组可以共享信息的人，以无向图表示。 每个人都是一个节点，两个节点之间有一条边就意味着他们是熟人。 

侦探按照固定的顺序叫人，从 1 到 n。 当一个人被呼叫时，他们不仅会透露他们个人所知道的信息，还会透露他们整个熟人部分所知道的一切，这意味着通过友谊边缘可到达的所有节点。 一旦通过调用其任何成员来“激活”连接的组件，整个组件的信息就会被知晓。 

任务是确定呼叫顺序中最早的位置，以便在处理这么多呼叫后，每个人的信息至少被收集一次。 

用图的术语来说，我们以递增的索引顺序处理节点。 当我们到达一个节点时，如果之前没有见过它的整个连接组件，我们将该组件标记为已发现。 我们希望排列 1 到 n 的第一个前缀能够覆盖所有连接的组件。 

约束允许最多 100,000 个节点和边。 这立即排除了任何尝试从头开始使用 BFS 或 DFS 模拟传播或重新计算每次调用的可达性的解决方案。 每个节点运行图遍历的简单方法在最坏的情况下将是二次的，并且远远超出限制。 

当图表具有孤立的节点时，就会出现微妙的边缘情况。 每个孤立的节点形成其自己的组件，并且必须通过调用它来显式地“激活”。 另一个边缘情况是完全连接的图，其中调用第一个节点立即覆盖所有内容。 

例如，考虑 n = 3，没有边。 每个节点都是隔离的，因此我们必须调用所有三个节点，给出答案 3。任何假设调用之间传播的尝试都会错误地建议更少的调用。 

另一方面，如果所有节点都已连接，例如 1-2-3，则调用节点 1 已经覆盖了所有节点，因此答案为 1。 

## 方法

 暴力解释是直接模拟该过程。 我们在节点上维护一个访问过的数组。 对于从 1 到 n 的每个调用 i，我们从节点 i 运行 BFS 或 DFS，将其连接组件中的所有节点标记为已访问。 每次调用后，我们检查是否所有节点都被访问。 

这是正确的，因为它准确地反映了信息传播规则。 然而，每个 DFS/BFS 可能在调用中总共遍历 O(n) 个节点和 O(m) 个边，在最坏的情况下导致 O(n(n + m))。 当n、m达到100,000时，这是不可行的。 

关键的观察结果是图结构不依赖于调用顺序。 每个连接的组件都表现为单个原子单元。 唯一的问题是：我们什么时候第一次按给定的顺序接触每个组件？ 

我们可以使用单个 DFS 或 DSU 将图预处理为连接的组件。 每个节点都分配有一个组件标识符。 然后我们扫描从 1 到 n 的节点并记录我们已经“激活”的组件。 当我们第一次看到来自新组件的节点时，我们会增加已发现组件的数量。 答案是第一个索引，其中该计数等于组件总数。 

这将问题从重复的图遍历减少到单个预处理步骤加线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每次调用重复 DFS | O(n(n + m)) | O(n(n + m)) | O(n + m) | 太慢了|
 | 组件+扫描| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们通过将图压缩为连接的组件，然后跟踪按给定顺序首次遇到每个组件的时间来解决该问题。

1. 根据给定的边构建图的邻接表。 这种表示方式让我们能够高效地遍历所有熟人。 
2. 在所有节点上运行 DFS 或 BFS，为每个节点分配组件标识符。 每次我们从未访问的节点开始遍历时，我们都会增加组件计数器并用该组件 ID 标记所有可到达的节点。 此步骤将可相互联系的证人分组为信息共享的单个单元。 
3. 经过预处理，我们知道总共有多少个连通分量。 
4. 在组件上创建一个布尔数组，以跟踪组件是否已被先前的调用“激活”。 
5. 按顺序迭代从 1 到 n 的节点。 对于每个节点，检查其组件 ID。 如果该组件之前未曾见过，则将其标记为已见过并增加已发现组件的计数。 
6. 一旦发现的组件数量等于组件总数，则停止并输出当前索引。 该索引表示调用序列中每个连接组至少被触及一次的最早点。 

### 为什么它有效

 每个连接的组件在信息传播下都是封闭的，这意味着调用其中的任何单个节点都会显示该组件中的所有内容，而无法显示其外部的任何内容。 因此，覆盖所有组件对于收集所有信息来说既是必要的也是充分的。 由于扫描按顺序处理节点，因此我们第一次从每个组件中选择至少一个代表正是覆盖整个图的最早的前缀。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    
    for _ in range(m):
        a, b = map(int, input().split())
        g[a].append(b)
        g[b].append(a)

    comp = [0] * (n + 1)
    comp_id = 0

    # iterative DFS to avoid recursion depth issues
    for i in range(1, n + 1):
        if comp[i] == 0:
            comp_id += 1
            stack = [i]
            comp[i] = comp_id

            while stack:
                v = stack.pop()
                for to in g[v]:
                    if comp[to] == 0:
                        comp[to] = comp_id
                        stack.append(to)

    seen = [False] * (comp_id + 1)
    got = 0

    for i in range(1, n + 1):
        c = comp[i]
        if not seen[c]:
            seen[c] = True
            got += 1
            if got == comp_id:
                print(i)
                return

    print(n)

if __name__ == "__main__":
    solve()
```邻接表构造直接捕获友谊图。 DFS 阶段将图压缩为连接的组件，确保每个节点都标有相互可达性的正确等价类。 第二遍是对固定顺序的贪婪扫描，布尔数组确保每个组件仅计算一次。 

使用迭代 DFS 可以避免 Python 中大型链可能出现的递归深度问题。 一旦看到所有组件，我们就会提前终止，这保证了正确的最小前缀长度。

 ## 边缘情况

 对于单节点图，DFS 只分配一个组件，并且扫描立即将其标记为索引 1 处所示。答案是 1，与第一个调用已经完成所有操作的直觉相匹配。 

对于没有边的图，每个节点形成其自己的组件。 扫描仅在到达最后一个节点后完成，因为每个索引都会引入一个新组件。 该算法正确地计算了 n 个不同的激活。 

对于具有多个断开连接的簇的图，该算法确保每个簇只被计数一次。 第一次出现的顺序决定了答案，并且组件压缩保证了簇内不会重复计数。
