 ## 问题理解

 约翰和保罗这两个玩家交替说一个词，从约翰开始。 玩家每次说话时，都会选择一个个人范围内的整数强度。 John 可以选择 1 到 1 之间的任何值$R_J$，和保罗从 1 到$R_P$。 所有口语优势的总和。 该总数首次达到或超过给定阈值的那一刻$h$，游戏立即停止，刚才说话的玩家输。 

一个关键的转折是，两个玩家都不想成为越过门槛的人，因此如果他们有任何合法的方法来避免，他们都会尽力避免以自己的行动结束游戏。 约翰的目标更加强烈：他要保证无论保罗怎么打，约翰都能逼迫保罗成为最终使总分至少达到的人$h$。 如果可能的话，我们还必须输出约翰应该说的第一个值。 

输入大小达到$10^{18}$，这会立即排除任何一次处理一个动作的模拟。 甚至$10^9$一秒钟内的操作会太慢，因此解决方案必须推理移动块或导出游戏的确定性结构。 

唯一微妙的情况发生在比赛接近尾声时，此时距离$h$变小。 例如，如果在某人的回合开始时剩余的所需金额为 1，则该玩家已经被迫立即失败，因为任何积极的举动都达到或超过了阈值。 另一种边缘情况发生在玩家可以一步准确地“跳”到终点时，例如，如果剩余的要求最多是他们的最大力量加一。 在这种情况下，他们可以选择迫使对手立即面对失败局面的棋步。 

一个天真的逐步模拟总是会在每轮选择一些合法值并更新总和。 失败并不是因为正确性，而是因为匝数可以是线性的$h$，太大了。 

## 方法

 蛮力的想法很简单。 我们一步步模拟游戏。 在每个回合中，当前玩家选择一些有效的强度，更新总数，并检查是否达到阈值。 如果我们为两名玩家尝试所有可能的选择，我们就可以确认约翰是否有获胜的第一步。 

这种方法原则上是正确的，因为游戏具有完美的信息和有限的分支。 然而，分支是不必要的。 在实践中，两个玩家都以一种非常有限的方式表现得最佳：如果可以避免的话，他们绝不会自愿失败。 这一观察结果极大地缩小了决策空间。 

关键的结构洞察力是，在任何时候，玩家总是会尝试最大化他们添加的金额，而不会立即损失。 如果剩余距离$h$当数值很大时，他们总是会选择最大允许强度，因为任何较小的值只会让对手以后更容易。 如果剩下的距离足够小，可以直接逼迫对方落败，他们就会立刻这么做。 

这消除了游戏中所有真正的选择。 整个过程变得确定性：在任何状态下，两个玩家的行动都由剩余距离和最大允许力量决定。 问题归结为跟踪单个不断变化的数字，而不是探索策略。

 ## 算法演练

 1. 从剩余的所需金额开始$rem = h$，约翰先行动。 当移动导致运行总和达到或超过时游戏结束$h$，相当于开车$rem$降至 0 或以下。 
2. 在玩家回合时剩余价值$rem$，看看他们是否可以立即迫使对手陷入失败的境地。 发生这种情况时$rem \le R + 1$，因为玩家可以选择$rem - 1$，准确地留给对手$rem = 1$，这轮到他们时是强制损失。 
3. 如果约翰开始并且$h \le R_J + 1$，约翰可以立即选择$W_J = h - 1$。 这保证了保罗面对$rem = 1$轮到他的时候就输了。 
4. 否则，约翰无法一招完成，将发挥最大安全值，即$R_J$。 这将剩余的要求减少到$rem = h - R_J$。 
5. 现在轮到保罗了。 对称地应用相同的逻辑：如果$rem \le R_P + 1$，保罗离开就可以立即逼胜$rem = 1$对于约翰.
 6. 如果双方都不能立即完成，则双方将重复执行最大安全棋步。 这意味着在每一整轮（约翰然后保罗）中，剩余价值减少$R_J + R_P$。 
7. 继续减去整轮，直到剩余值变得足够小，以至于其中一名玩家可以使用$rem \le R + 1$健康）状况。 此时，直接使用相同的规则解决最后一两个动作。 

### 为什么它有效

 在每种不可能立即完成任务的状态下，任何小于最大安全步数的举动只会增加对手的剩余价值，而不会改变双方都无法立即完成任务的事实。 由于双方都倾向于避免立即损失，并且都有对称的激励来尽可能减少剩余价值，因此唯一稳定的玩法是每回合最大安全减少。 这将游戏变成了确定性倒计时，当$rem$进入一名球员的完成范围。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

h, RJ, RP = map(int, input().split())

def can_finish(rem, R):
    return rem <= R + 1

# If John can finish immediately
if h <= RJ + 1:
    print("John")
    print(h - 1)
    sys.exit()

rem = h
turn = 0  # 0 = John, 1 = Paul

while True:
    if turn == 0:
        if rem <= RJ + 1:
            print("John")
            print(rem - 1)
            break
        rem -= RJ
    else:
        if rem <= RP + 1:
            print("Paul")
            break
        rem -= RP
    turn ^= 1
```代码直接实现了确定性约简过程。 第一个检查处理约翰可以一步结束游戏的唯一情况。 之后，循环交替玩家，减去他们的全部安全贡献，除非他们足够接近终点以迫使立即失败。 状态变量`rem`跟踪我们距离阈值有多远，并且`turn`编码是谁的动作。 

关键的实施细节是我们从不探索每一步的多种选择。 贪心减法是有效的，因为任何较小的减法都不能改善当前玩家的结果，只会延迟或恶化他们的位置。

 ## 边缘情况

 一种关键的边缘情况是$h \le R_J + 1$。 例如，使用输入$h = 6, R_J = 7, R_P = 8$，约翰立即选择 5，离开$rem = 1$。 该算法在任何循环之前处理此问题，确保我们正确输出 John，而无需模拟任何进一步的动作。 

另一个边缘情况是当两个玩家的限制都非常小时，例如$h = 10, R_J = R_P = 1$。 该循环通过交替减少单个单位直到一个玩家的回合中剩余值达到 1 来减少问题。 确定性减法确保结果没有歧义。 

最后一个微妙的情况是，一个玩家的范围比另一个玩家大得多。 例如，如果$R_J \gg R_P$，约翰或许可以提前直接跳入整理窗口。 条件$rem \le R_J + 1$准确地捕捉到这一点，确保我们在正确的时刻检测到这个机会，而无需模拟中间状态。
