 ## 问题理解

 我们得到了一份严格递增的硬币面额列表，其中每个面额除以下一个面额。 这意味着系统的行为就像链式乘法结构，而不是任意的硬币值。 我们还给出了目标金额`b`，我们需要确定是否可以精确地形成`b`使用这些硬币，如果是，请尽量减少使用的硬币总数。 

输出要么是硬币类型的分布，告诉每个面额使用了多少硬币，要么是不存在精确表示的声明。 

可分性条件是关键的结构约束。 由于每个`a[i]`可以整除`a[i-1]`，每个面额都可以解释为前一个面额的缩放版本。 这使得贪婪推理可能可行，但前提是我们尊重进位和余数如何跨级别交互。 

就以下方面而言，限制很小`n`，最多有 30 种面额，但是`b`可以大到 10^18。 这立即排除了任何对总和的动态规划。 即使对所有子集或数量进行 DP 也是不可能的，因为状态空间非常大。 任何正确的解决方案都必须在线性或接近线性的时间内运行`n`。 

如果我们试图贪婪地获取尽可能多的最大硬币而不考虑可分性结构，则会出现微妙的失败情况。 例如，如果面额是`[1, 3, 9]`和`b = 6`，一个天真的贪婪策略将采取`9`硬币是不可能的，所以需要`3 + 3`，这是正确的，但在具有非规范行为的更复杂的系统中，贪婪可能会失败。 在这里，正确性取决于支持基本转换风格推理的可分性链。 

另一种失败模式是忘记一个级别的剩余余额可能会或可能不会转移到下一个面额，具体取决于整除率。 如果我们没有正确地标准化，我们可能会错误地得出不可能性的结论。 

## 方法

 一个蛮力的想法是将其视为硬币找零问题：尝试每个面额的所有计数组合。 自从`b`最多可达 10^18，即使每个硬币数量的界限为`b / a[i]`，组合的数量呈指数增长`n`。 和`n = 30`，这是完全不可行的。 

第二个天真的改进是对可达总和进行动态编程，但总和的范围使得这在时间和内存上都是不可能的。 

关键的观察结果是整除条件将系统变成了混合基表示问题。 每个面额都是一个“数字位”，并且因为`a[i]`划分`a[i+1]`，我们可以定义比率`r[i] = a[i+1] / a[i]`。 那么任何有效的表示形式的行为就像在非统一基本系统中写入的数字一样，除了我们最小化数字总和而不是仅仅表示一个数字。 

这表明贪婪的自下而上的转换：从最大面额开始，决定要取多少硬币，然后向下传播剩余部分。 然而，与标准的贪婪硬币找零不同，我们必须确保所有级别的一致性，因为较高级别的选择会限制较低级别的余数。 

我们通过强制在每个级别上永远不会对下一个比率取超过必要的模来解决这个问题，并将余数向下取。

 ## 算法演练

 我们首先预处理连续面额之间的比率。 对于每个`i > 1`， 定义`r[i] = a[i] / a[i-1]`。 

然后，我们从最大面额向下进行处理，保持我们仍需要表示的剩余金额。 

1. 开始于`rem = b`。 该值代表尚未以更高面额表示的内容。 我们还初始化一个数组`cnt`尺寸的`n`为零。 
2. 对于每个面额`i = n`下降到`2`，计算有多少个完整的硬币`a[i]`我们可以从`rem`，但仅以与下一个较小比例一致的方式。 我们计算`cnt[i] = rem // a[i]`，然后减少`rem = rem % a[i]`。 

此步骤有效，因为取了一枚有价值的硬币`a[i]`仅影响更高的倍数`a[i-1]`，并且由于可整除性，较低的面额始终可以代表余数。 
3.在向下移动之前，我们必须确保`rem`与下一级缩放兼容。 具体来说，自从`a[i]`是的倍数`a[i-1]`, 余数`rem`自然已经以`a[i-1]`比例，因此除了整数除法之外不需要任何调整。 
4. 处理后降至`i = 2`，我们剩下`rem`必须使用以下方法处理`a[1]`。 我们设定`cnt[1] = rem // a[1]`，并更新`rem = rem % a[1]`。 
5. 如果最后`rem != 0`，这意味着我们不能准确地表示`b`使用可用的面额，所以我们输出`"Impossible"`。 
6. 否则，我们输出计数。 

关键思想是，由于可除性链，每个余数都在水平上`i`保证可以用较小的面额表示当且仅当它在最后一步可以整除`a[1]`。 该算法通过确保所有决策与基数结构局部一致来避免回溯。 

### 为什么它有效

 教派的结构产生了一个等级制度，其中每个教派`a[i]`是整数倍`a[i-1]`。 这意味着使用较高面额表示的每个数字始终在较低面额定义的网格上对齐。 当我们贪婪地提取每个级别的贡献时，我们实际上是在执行位置分解`b`在混合基数系统中。 不变的是处理后的级别`i`，剩余值始终严格小于`a[i]`并且完全可以使用面额来表示`1..i-1`当且仅当它仍然可以整除`a[1]`。 这保证了未来的步骤不会使先前选择的系数无效，因此构造是全局一致的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = int(input())

    cnt = [0] * n
    rem = b

    for i in range(n - 1, -1, -1):
        cnt[i] = rem // a[i]
        rem %= a[i]

    if rem != 0:
        print("Impossible")
    else:
        print(*cnt)

if __name__ == "__main__":
    solve()
```该实现直接反映了贪婪分解。 我们保持连续余数，并将捐款从最大面额到最小面额剥离。 重要的微妙之处在于，我们永远不需要显式检查相邻面值之间的整除性，因为该结构保证了对齐：每个`a[i]`是的倍数`a[i-1]`，因此减少模数`a[i]`保持较低级别的一致性。 

一个常见的错误是尝试显式地使用比率传播来调整计数。 在这里这是不必要的，因为整数除法已经在这个混合基系统中执行正确的数字提取。

 ## 边缘情况

 一个关键的边缘情况是当`b = 0`。 该算法处理所有面额并在各处产生零计数，并且余数始终保持为零。 这证实了无论面额结构如何，零总是可以表示的。 

另一种边缘情况是只有最小面额才能代表数字。 例如，`[1, 100, 10000]`和`b = 7`产生所有较高计数 0 和最终计数 7`a[1]`。 贪婪通行证永远不会错误分配，因为更高面额的货币总是需要至少 100 个单位。 

一种容易失败的情况是当人们期望在级别之间进行利差修正时。 由于可分性，不需要这样的校正，并且模块化提取已经强制了一致性。
