 ## 解决方案
 ## 问题理解

 我们有一块有边的矩形板`a`和`b`，放置在固定方向，使其底边水平。 超过`k`几天来，每天贴一张相同的方形贴纸，每个贴纸也有方向，因此其底边是水平的。 在这些之后`k`放置时，整个板区域都被覆盖，这意味着板的每个点都位于至少一个贴纸内。 

现在我们重置流程，想要购买一些整数边长的新方形贴纸`s`。 目标是确保准确地`k`天，每天再贴一张贴纸，可以完全覆盖相同的`a × b`木板。 

我们被要求计算最小的整数`s`这样`k`完全相同的`s × s`正方形可以完全覆盖矩形。 

约束条件非常大，`a, b, k`最多`10^18`， 和`a · b ≤ 10^18`。 这立即排除了任何放置或基于网格的构造的模拟。 任何解决方案都必须使用恒定数量的运算将问题简化为纯算术。 

一个微妙的点是，贴纸并不局限于网格，它们可以任意放置在平面上。 这意味着我们正在解决几何覆盖问题，但实际上只有面积和可分性约束很重要。 

一个关键的边缘考虑因素是何时`k`即使贴纸完美地平铺在板上，也太小而无法覆盖该区域。 例如，如果`a = 4`,`b = 4`,`k = 4`，那么每个贴纸必须至少覆盖面积`4`甚至达到总面积`16`。 任何更小的`s`无论放置如何都会失败。 这已经暗示面积下限是必要的，但单独存在还不够。 

另一种边缘情况是当一个维度远大于`s`。 即使总面积足够，如果贴纸太小而无法跨越可行的覆盖布置中的一个维度，则多个贴纸必须对齐以覆盖该维度，这通过天花板计数影响可行性。 

## 方法

 一种简单的方法会尝试将问题解释为几何填充任务。 对于固定的`s`，我们可以尝试模拟是否`k`正方形可以覆盖`a × b`通过最佳放置它们来形成矩形。 这很快就会变成一个连续的二维覆盖问题，具有无数的位置，因此模拟是不可能的。 即使是离散位置也会发生组合爆炸。 

第一个简化来自于注意到唯一有用的结构是轴对齐和统一的正方形大小。 任何用相等的正方形对矩形进行最佳覆盖的行为都类似于近似网格，可能会发生移动。 这将问题简化为计算需要多少个正方形来覆盖每个维度。 

如果我们修复`s`，沿宽度方向所需的贴纸数量为`ceil(a / s)`沿高度是`ceil(b / s)`，所以所需的总覆盖范围是`ceil(a / s) * ceil(b / s)`。 要求是这个值最多是`k`。 

现在问题变得单调`s`。 如果给定`s`作品，任何更大的`s`也有效，因为增加方块大小会减少上限计数或保持它们不变。 这允许二分搜索`s`。 

我们寻找最小的`s`这样`ceil(a / s) * ceil(b / s) ≤ k`。

 ## 算法演练

 1. 定义一个函数，对于给定的正方形大小`s`，计算需要多少张贴纸来覆盖电路板。 这是通过计算完成的`ceil(a / s)`和`ceil(b / s)`并将它们相乘。 这模拟了最佳的轴对齐布置。 
2. 观察`ceil(x / s)`减少或保持不变`s`增加。 这确保了所需贴纸的数量是单调的`s`，这对于二分查找至关重要。 
3. 设置二分查找范围`s`。 下界是`1`。 上限可以安全地表示为`max(a, b)`，因为比两侧都大的正方形在一张贴纸中可以轻松覆盖整个矩形。 
4. 执行二分查找`s`。 对于每个中点，计算所需的贴纸。 如果要求小于或等于`k`，这意味着`s`就足够了，所以我们尝试较小的值。 否则，`s`太小了，我们增加它。 
5. 继续直到最小的有效值`s`被发现。 

### 为什么它有效

 对于任何固定的`s`，表达式`ceil(a / s) * ceil(b / s)`准确地表示最佳轴对齐覆盖所需的最小相等正方形数。 任何覆盖物必须至少将每个维度划分为长度最多的段`s`，所以没有任何建筑可以超越这些计数。 该函数是单调的，因为增加`s`不能增加任一上限期限。 因此二分搜索收敛到最小可行`s`不遗漏任何一位候选人。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def needed(a, b, s):
    return ((a + s - 1) // s) * ((b + s - 1) // s)

def solve():
    a, b, k = map(int, input().split())

    lo, ro = 1, max(a, b)
    ans = ro

    while lo <= ro:
        mid = (lo + ro) // 2
        if needed(a, b, mid) <= k:
            ans = mid
            ro = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```实现的核心是`needed`函数，它编码基于上限的计数逻辑。 表达式`(a + s - 1) // s`是标准整数安全的计算方式`ceil(a / s)`没有浮点运算。 

二分查找保持所有大小都大于的不变量`ro`有效且所有尺寸小于`lo`均无效。 每一步都会缩小这个边界，直到最小有效`s`是孤立的。 

必须注意仅使用整数算术。 使用浮点数会引入高达以下值的精度误差`10^18`。

 ## 边缘情况

 一个重要的边缘情况是当`k = 1`。 然后我们必须用一个正方形覆盖矩形，所以答案一定是`max(a, b)`。 用于输入`7 4 1`，算法评估`s = 4`由于不够`ceil(7/4)=2`， 和`s = 7`生效，正确返回`7`。 

另一种情况是当`a`和`b`是相等的并且`k`很大。 用于输入`6 6 36`， 甚至`s = 1`之所以有效，是因为 36 个单位方格可以平铺棋盘。 二分查找正确收敛到`1`。 

最后，当尺寸非常倾斜时，例如`1 10^18 1`，算法强制`s = 10^18`因为只允许一张贴纸，而且它必须跨越两个维度。 上限计算正确产生`1 * 1 = 1`仅在该边界处。
