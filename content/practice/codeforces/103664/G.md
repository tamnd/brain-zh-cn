 ## 问题理解

 我们得到了 24 小时格式的当前时钟读数和两个限制，一个允许时钟最多向后移动一分钟，另一个允许时钟最多向前移动 b 分钟。 时钟很旧，所以我们实际上并没有改变实时时间，只是改变了显示的时间。 目标是在这些范围内选择一个偏移值（负值或正值），以便在应用它后，显示的时间变成分钟恰好为零的时刻，这意味着一整小时，并且在所有可实现的整小时时间中，我们希望是一天中最早的时间。 

输出不是移位本身，而是移位后产生的时间。 我们实际上选择了一个整数 x，使得 -a ≤ x ≤ b，将其应用于当前时间，环绕 24 小时制，并要求结果分钟字段变为 00。在所有有效选择中，我们希望按时间顺序排列最小的结果时间。 

关键限制是时间以每天 1440 分钟为模，并且与全天相比，班次限制在相对较小的窗口内。 由于 a 和 b 都小于 720，因此我们在任一方向上的覆盖时间都不会超过半天，这强烈表明直接扫描就足够了。 

微妙的边缘情况来自环绕。 忽略日换行的天真解释将在 00:00 或 23:59 附近的输入上失败。 例如，如果时间是 00:01，我们向后移动一分钟，就会到达 00:00，这是有效的。 但从 23:59 向前移动两分钟到 00:01，这也可以以一种不明显的方式与“最早时间”要求相互作用。 

另一种故障模式是尝试仅考虑转移到前一小时或下一小时边界，而不检查所有可到达的边界。 例如，如果当前时间是 11:30 并且存在较大的向后限制，则最佳答案不一定是 11:00 或 12:00； 这取决于这些时间界限是否可以在限制范围内达到。 

## 方法

 蛮力方法是尝试从 -a 到 b 的所有可能的移动 x，计算结果时间，将其标准化到 24 小时时钟的范围内，并检查其分钟分量是否为零。 如果是，我们将其与迄今为止按字典时间顺序看到的最佳候选进行比较。 这是正确的，因为它直接评估所有合法选择。 

这种方法的成本与允许的移位范围的大小成线性关系，即 a + b + 1。由于 a 和 b 都以 720 为界，因此每个测试用例的最大检查次数最多为 1441，这对于一秒的限制来说已经足够小，因此即使是暴力破解也接近足够了。 然而，我们可以通过重新构建问题来进一步简化推理。 

关键的观察是我们只关心分钟数为零的时刻。 任何有效的目标时间必须恰好为 HH:00。 我们可以迭代所有 24 个可能的小时，计算当前时间周围最接近的相应时间，并检查是否可以在允许的轮班范围内达到该小时边界，而不是迭代班次。 这将问题减少到最多检查 24 个候选人，而不是最多 1441 个班次。 

对于每个小时 h，我们以分钟为单位考虑目标时间 h:00，在 1440 分钟的循环时间线上向前和向后计算其与当前时间的距离，并检查任一距离是否在范围内。 如果是，则可以访问。 在所有可到达的时间中，我们按时间顺序选择最小的结果时间。 

这将问题从扫描一段班次转变为扫描一组固定的结构目标。

 ## 算法演练

 我们将输入的时间转换为从 00:00 开始的总分钟数，称之为 cur。 

然后，我们考虑从 0 到 23 的每个小时边界 h 并计算 target = h * 60。对于每个目标，我们计算长度为 1440 的圆形时间轴上的前向距离和后向距离。前向距离为 (target - cur) mod 1440，后向距离为 (cur - target) mod 1440。 

我们检查 [-a, b] 中是否存在达到该目标的有效移位 x。 如果前向距离 ≤ b 或后向距离 ≤ a，则该条件成立。 

我们以分钟为单位维护最佳候选时间，初始化为较大或未定义的时间，并在每当我们发现按字典顺序较小的可到达小时边界时更新它。 

扫描完所有 24 个候选者后，我们将最佳分钟值转换回 HH:MM 格式。 

它的工作原理来自于有效输出的结构。 任何可接受的最终时间都必须以 :00 结束，因此它必须恰好是圆形时钟上的 24 小时界限之一。 每个可行的解决方案都对应于选择这些边界之一并应用落在其上的偏移。 该算法在允许的前向和后向范围内检查每个边界的可达性，并选择最早可达的边界，因此不会错过任何有效的最优答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def to_minutes(s):
    h = int(s[:2])
    m = int(s[3:])
    return h * 60 + m

def fmt(x):
    h = x // 60
    m = x % 60
    return f"{h:02d}:{m:02d}"

s = input().strip()
a, b = map(int, input().split())

cur = to_minutes(s)
best = None

for h in range(24):
    target = h * 60

    forward = (target - cur) % 1440
    backward = (cur - target) % 1440

    if forward <= b or backward <= a:
        if best is None or target < best:
            best = target

print(fmt(best))
```该代码首先将时间转换为线性分钟表示，这简化了模块化推理。 24 小时内的循环枚举了所有可能的有效端点。 向前和向后距离编码两种允许的时钟调整类型。 这种比较确保我们始终保持最早可行的小时边界。 

这里的一个常见错误是忘记模运算并使用原始差异，这会在午夜左右中断。 另一种是独立处理前向和后向，而不正确检查两个方向，这可能会错误地拒绝可达时间。

 ## 边缘情况

 一个重要的边缘情况是跨越午夜。 对于像 23:59 这样向前限制为 1 的输入，目标 00:00 正好提前一分钟模 1440。该算法处理此问题是因为向前距离变为 1，即 ≤ b，因此 00:00 被正确地视为可达。 

另一种情况是不允许向后移动。 如果a = 0，则仅向前移动有效，但向后距离计算仍然存在； 该条件正确地将其过滤掉，并且仅接受可向前到达的小时边界。 

最后一个微妙的情况是当前时间已经在一个小时边界上。 对于 a = b = 0 的 10:00，算法仍然检查目标 10:00 并看到向前 = 向后 = 0，因此它立即选择它，而不需要任何特殊的大小写。
