 ## 解决方案
 ## 问题理解

 我们给出了几个独立的整数三元组。 对于每个三元组，我们需要决定是否可以构造两个非负整数$x$和$y$这样三个条件同时成立：它们的按位 AND 等于给定值$a$，它们的按位或等于给定值$b$，以及它们的算术差$x - y$等于给定值$c$。 如果存在这样的一对，我们必须输出任何有效的$x, y$; 否则我们输出$-1$。 

关键的困难在于按位约束定义了每一位的关系，而减法约束则通过进位（二进制减法中的借位）将所有位耦合在一起。 这种交互意味着我们不能独立地处理位，即使 AND 和 OR 建议每个位的结构。 

约束条件允许最多一万个三元组，每个值最多可达十亿左右。 这意味着每个数字都适合在 30 到 31 位范围内。 任何在位上以线性时间处理每个三元组的解决方案都足够了，但是任何尝试跨位以指数方式探索分配的任何解决方案都会立即失败。 

常见的失败案例来自于在不考虑减法的情况下处理按位约束。 例如，如果$a = 0$,$b = 1$， 和$c = 0$，人们可能会尝试分配$x = y = 0$或者$x = y = 1$每个比特不一致，但减法强制全局一致性。 另一个微妙的情况是当减法需要借用链时； 当借位跨多个位传播时，局部有效的位选择可能在全局范围内变得不可能。 

## 方法

 如果我们忽略减法约束，问题就变得简单了。 从$a = x \& y$和$b = x | y$，每个位的行为都是独立的。 如果有一点$b$都为零$x$和$y$该位置必须为零。 如果有一点$a$是一，两者必有一。 如果一点是一$b$和零$a$，那么该位的对必须是$(1,0)$或者$(0,1)$，给予自由。 

暴力方法会尝试对不明确的位进行所有分配，并检查结果整数是否满足$x - y = c$。 在最坏的情况下，如果许多位是不明确的，这会导致组合数量呈指数级增长，约为$2^k$，这是不可能的，当$k$接近30。 

关键的观察是唯一的全局耦合来自减法。 一旦我们解释$x - y = c$作为二进制减法，我们可以在跟踪借用状态时从最低有效位到最高有效位处理位。 每个位都贡献一个局部方程，涉及当前借位、所选对$(x_i, y_i)$，以及由此产生的位$c$。 这将问题转化为基于位和单个二进制状态的动态编程过程。 

AND/OR 约束限制每一位允许的对，而减法约束在这些对中一致地进行选择。

 | 蛮力 |$O(2^k)$每三人 |$O(k)$| 太慢了|
 | 带借位的位 DP |$O(31)$每三人 |$O(31)$| 已接受 |

 ## 算法演练

 我们独立处理每个三元组。 对于固定三元组$(a, b, c)$，我们构建所有三个数字的二进制表示，最多 31 位。 

1. 我们定义了位位置和借位的动态编程状态。 状态表示是否可以构造有效的前缀$x$和$y$直到给定位，同时保持具有特定借位的减法约束。 
2. 我们在位 0（最低有效位）处进行初始化，借位等于 0。这对应于开始减法$x - y$没有任何传入的借入。 
3. 对于每个位位置，我们确定哪些分配$(x_i, y_i)$AND/OR 约束允许。 如果 OR 位为 0，则两个位都必须为 0。如果 AND 位为 1，则两个位都必须为 1。否则，两个位混合分配$(1,0)$和$(0,1)$是可能的。 
4. 对于每个允许的赋值，我们在该位模拟二进制减法。 我们计算$x_i - y_i - \text{borrow}_{in}$。 这个结果必须匹配位$c$加上 2 的倍数，确定向外借用。 如果这种一致性失败，则转换无效。 
5. 我们向前转换 DP 状态，记录下一位的给定借用状态是否可达。 
6. 处理完所有位后，我们要求最终借位为零。 如果不是，则不存在有效整数。 
7. 如果存在有效路径，我们重建$x$和$y$通过存储过渡期间做出的决策。 

减法一致性检查是唯一发生全局交互的地方。 所有其他约束都是每比特本地的。 

### 为什么它有效

 该算法对所有有效前缀进行编码$(x, y)$尊重按位约束和部分减法一致性。 借位状态完全捕获了二进制减法中所有较低位对较高位的影响。 由于二进制减法除了借位之外没有其他隐藏状态，因此维护这一位内存足以保证正确性。 每个无效的全局分配都必须在违反允许的位模式或在借位状态下无法满足减法方程的某个位处失败。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(a, b, c):
    MAXB = 31

    # dp[bit][borrow] -> store predecessor info
    dp = [[False, False] for _ in range(MAXB + 1)]
    parent = [[None, None] for _ in range(MAXB + 1)]

    dp[0][0] = True

    def allowed_pairs(ai, bi):
        if bi == 0:
            return [(0, 0)]
        if ai == 1:
            return [(1, 1)]
        return [(0, 1), (1, 0)]

    for i in range(MAXB):
        ai = (a >> i) & 1
        bi = (b >> i) & 1
        ci = (c >> i) & 1

        for borrow in [0, 1]:
            if not dp[i][borrow]:
                continue

            for xbit, ybit in allowed_pairs(ai, bi):
                val = xbit - ybit - borrow
                out_bit = val & 1
                out_borrow = 1 if val < 0 else 0

                if out_bit == ci:
                    if not dp[i + 1][out_borrow]:
                        dp[i + 1][out_borrow] = True
                        parent[i + 1][out_borrow] = (i, borrow, xbit, ybit)

    if not dp[MAXB][0]:
        return None

    x = 0
    y = 0
    bit = MAXB
    borrow = 0

    while bit > 0:
        i, pb, xb, yb = parent[bit][borrow]
        if xb:
            x |= (1 << i)
        if yb:
            y |= (1 << i)
        borrow = pb
        bit -= 1

    return x, y

def main():
    n = int(input())
    for _ in range(n):
        a, b, c = map(int, input().split())
        res = solve_case(a, b, c)
        if res is None:
            print(-1)
        else:
            print(res[0], res[1])

if __name__ == "__main__":
    main()
```该解决方案独立处理每个三元组，并在最多 31 个位置上构建按位动态程序。 DP 表跟踪是否可以用给定的借位状态形成比特前缀。 父数组存储用于重建最终数字的转换。 

允许的对生成器在任何算术推理之前直接强制执行 AND/OR 约束。 然后，减法检查根据这些候选者是否与所需的位匹配来过滤这些候选者。$c$在当前借款下。 

重建从最高位开始向后走，恢复导致有效最终状态的准确位选择。

 ## 边缘情况

 当 OR 约束允许灵活性但减法立即强制借用链时，就会出现关键的边缘情况。 例如，当低位全部为零时$c$但唯一有效的位分配会创建非零中间差，DP 会正确拒绝下一位的所有路径。 

另一种情况是当$a = b$。 这迫使$x = y = a$，使减法等于零。 该算法通过只允许每位进行一次转换来处理此问题，并且借用状态永远不会激活。 

第三种情况是当$c$相对于可能的位分配实际上是负的。 即使输入是非负的，只有在足够的高位补偿低位结构的情况下才能实现减法。 当没有借用一致路径达到最终状态时，DP 会正确检测到不可能性。
