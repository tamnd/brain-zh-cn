 ## 解决方案
 ## 问题理解

 我们被赋予了一个$m \times n$网格，其中每个单元格包含一个非负数，表示其中生长有多少棵树。 长颈鹿从左上角的单元格开始$(1,1)$并想要到达右下角的单元格$(m,n)$。 

运动规则受到限制且不寻常。 从任何当前单元格开始，长颈鹿执行的动作包括在一个方向上的长跳和在垂直方向上的短步。 在一种类型的移动中，它会下降$k$单元格并向右移动 1 个单元格，在另一个单元格中向右移动$k$细胞减少 1 个细胞。 每次移动都准确地落在一个新单元格上，长颈鹿从该目标单元格收集叶子。 

第一个单元格不贡献任何值。 从第 1 步开始，每一步都按顺序标记。 长颈鹿在奇数次移动时吃得完全一样$t$从目标单元格离开，而在偶数次移动时它会吃掉$3t$叶子, 哪里$t$是该单元格中树木的数量。 

任务是选择一个有效的移动序列，该序列恰好结束于$(m,n)$，或者报告这是不可能的，同时在这种交替评分规则下最大化吃掉的叶子总数。 

网格大小最多为$100 \times 100$， 和$k$最多也是100。 这立即排除了路径上的任何指数搜索，因为即使是中等数量的步骤也会造成组合爆炸。 解决方案必须将其视为最多具有几百万个状态的状态空间上的结构化最短或最长路径问题。 

一个微妙的问题是，移动次数的奇偶性会改变每个单元格的权重。 这意味着访问一个单元格的值不是固定的，它取决于我们是否到达奇数步或偶数步。 任何忽略奇偶校验并将每个单元格视为具有单个值的解决方案都会产生不正确的结果。 

另一个不明显的边缘情况是可达性。 因为每次移动都会改变两个坐标，但由于步长不对称，有些$(m,n)$配置无法访问。 例如，如果$k$相对于尺寸而言较大，可能不存在精确落在目标上的合法移动序列。 在这种情况下，答案必须是$-1$，即使可以进行部分移动。 

## 方法

 自然的第一个尝试是将其视为对所有路径的强力搜索$(1,1)$到$(m,n)$。 每个单元格最多有两种可能的移动，因此我们递归地探索，保持当前位置和移动索引奇偶性。 每条路径的长度大致为$O(m+n)$除以步长效应，但指数分支仍然占主导地位。 即使在一个$100 \times 100$在网格中，可能的路径数量会超出任何可行的限制，因为在每一步我们都会分支到两个方向，直到达到边界。 

这种强力方法正确的原因很简单：它枚举每条有效路径并根据奇偶校验规则准确评估其分数。 失败点在于规模，而不是正确性。 由于分支因子 2 可能超过数十步，状态的数量变得非常大。 

关键的观察结果是，问题不在于路径，而在于由移动计数的位置和奇偶校验定义的状态。 一旦我们到达给定的单元并知道下一步的移动是奇数还是偶数，所有未来的行为仅取决于该状态，而不取决于我们如何到达那里。 这将问题转换为具有两层的分层网格上的图最短路径问题：一层用于奇数移动，一层用于偶数移动。 

各州$(i,j,p)$， 在哪里$p \in \{0,1\}$表示下一步移动的奇偶性，最多转换到两种状态，具体取决于我们是应用垂直重移动还是水平重移动。 转换成本是目标单元格的值乘以 1 或 3（具体取决于奇偶校验）。 

现在，这是有向非循环式分层图上的最长路径问题，但循环可以存在于索引空间中，因此我们将其视为状态上的标准 DP，反复松弛过渡或使用类似 BFS 的松弛直到收敛。 由于状态数量仅为$2mn \le 20000$，这是有效的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力DFS | 指数| O(mn) 递归 | 太慢了|
 | DP 状态 (i, j, 奇偶校验) | O(百万) | O(百万) | 已接受 |

 ## 算法演练

 1.定义一个DP表，其中$dp[i][j][p]$存储到达单元格时可获得的最大叶子数$(i,j)$并具有下一步奇偶性$p$， 在哪里$p=0$意味着下一步行动是奇数并且$p=1$意味着下一步的行动是偶数的。 选择这种编码是因为分数乘数仅取决于移动索引奇偶性。 
2.将除起始状态外的所有状态初始化为非常负的值$dp[1][1][0] = 0$，因为我们在进行第 1 步之前从头开始，并且不在那里收集任何东西。 
3. 从每个状态，尝试对应于两种移动类型的两次转换： 向下移动$k$并向右 1，或向右移动$k$向下 1. 对于每个有效的移动，计算目标单元格。 
4. 当转换到目标单元格时，计算增益。 如果当前移动奇偶校验为奇数（第一、第三等），则添加$t$。 如果偶数，则添加$3t$。 该权重在到达时应用，因此它与目的地状态相关。 
5. 每次移动后翻转奇偶校验，因为奇数移动变为偶数，反之亦然。 这可确保 DP 正确跟踪交替评分规则。 
6. 多次迭代所有状态，放松转换，直到没有改进发生，或者简单地运行由下式限制的固定次数的迭代$2mn$，因为每次松弛都会增加路径长度尺寸并且不能无限地改善。 
7. 答案是单元格中两个奇偶校验状态的最大值$(m,n)$。 如果两者都保持负无穷大，则输出$-1$。 

### 为什么它有效

 对于每个可达状态，DP 都保持不变$(i,j,p)$，它存储以正确奇偶性结束于该单元格的所有有效移动序列中的最佳可能得分。 每个转换都保留了正确性，因为它仅通过一次合法的移动来扩展已经有效的路径，并仅应用一次正确的奇偶校验相关权重。 由于到达状态的每条路径都可以分解为前一个状态加上一次移动，因此重复松弛最终会将最优值传播到所有可达状态。 奇偶校验分割确保不会错误地合并仅移动索引对齐方式不同的两条路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = -10**18

def solve():
    m, n = map(int, input().split())
    k = int(input())
    grid = [None] + [[0] + list(map(int, input().split())) for _ in range(m)]

    dp = [[[INF] * 2 for _ in range(n + 1)] for _ in range(m + 1)]
    dp[1][1][0] = 0

    for _ in range(m * n * 2):
        changed = False
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                for p in range(2):
                    if dp[i][j][p] == INF:
                        continue

                    cur = dp[i][j][p]
                    np = 1 - p

                    ni, nj = i + k, j + 1
                    if ni <= m and nj <= n:
                        gain = grid[ni][nj]
                        if p == 1:
                            gain *= 3
                        if dp[ni][nj][np] < cur + gain:
                            dp[ni][nj][np] = cur + gain
                            changed = True

                    ni, nj = i + 1, j + k
                    if ni <= m and nj <= n:
                        gain = grid[ni][nj]
                        if p == 1:
                            gain *= 3
                        if dp[ni][nj][np] < cur + gain:
                            dp[ni][nj][np] = cur + gain
                            changed = True

        if not changed:
            break

    ans = max(dp[m][n])
    print(-1 if ans == INF else ans)

if __name__ == "__main__":
    solve()
```DP 表是三维的，因为奇偶性对于正确评分至关重要。 每个放松步骤都会尝试两种运动类型，并在发现更好的分数时更新下一个状态。 重复的松弛是安全的，因为每次改进都对应于严格更好的路径分数，并且有限状态空间保证终止。 

一个常见的实现陷阱是忘记仅在偶数移动中乘以 3，这对应于奇偶校验$p=1$搬家之前。 另一个是错误地初始化开始：起始单元不能贡献任何值，即使它包含树。 

## 工作示例

 ### 示例 1

 输入：```
2 2
1
1 1
1 1
```我们开始于$(1,1)$得分为 0，奇偶校验为 0。 

| 步骤| 职位| 奇偶校验（下一步）| 行动| 分数 |
 | ---| ---| ---| ---| ---|
 | 0 | (1,1) | 0 | 开始 | 0 |
 | 1 | (2,2) | 1 | 移动并收集 1 | 1 |

 DP达到$(2,2)$分数为 1。只有一条有效路径，并且奇偶乘数不会改变任何内容，因为只进行了一步。 

### 示例 2

 输入：```
3 5
1
1 2 0 3 1
2 4 5 7 8
3 0 7 1 0
```我们跟踪关键的最佳过渡$(3,5)$。 

| 步骤| 职位| 平价| 移动类型 | 增益| 总计 |
 | ---| ---| ---| ---| ---| ---|
 | 0 | (1,1) | 0 | 开始 | 0 | 0 |
 | 1 | (2,2) | 1 | 向下+向右 | 4 | 4 |
 | 2 | (3,4) | 0 | 向下+向右 | 7 | 11 | 11
 | 3 | (3,5) | 1 | 右+下 | 0 或 1 个选择 | 11 或 14 |

 该表显示奇偶校验如何显着影响最后一步，因为最后一步可能会将单元格值乘以 3，具体取决于它是否为偶数。 

此示例表明，以不同的移动次数到达同一单元的不同路径可能会产生不同的最终结果，这正是 DP 必须单独存储奇偶校验的原因。

 ## 边缘情况

 一个关键的边缘情况是网格太小以至于无法移动。 例如，在一个$1 \times 1$网格中答案必须为零，因为长颈鹿从不移动也不吃东西。 

另一个边缘情况是当$k$超出第一步后的可用尺寸。 例如，如果$m=3, n=3, k=3$，大多数移动立即出界，没有留下到达目的地的有效路径。 DP正确地保持所有状态不可达，最终答案变为$-1$。 

更微妙的情况是当多个路径到达同一单元但具有不同奇偶校验时。 DP 明确存储两者，因此两个贡献都被保留。 这避免了失去最佳解决方案，其中稍长的路径会产生更好的最终乘法器结构。
