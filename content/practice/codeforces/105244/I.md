 ## 解决方案
 ## 问题理解

 输入描述一棵树，其顶点编号为 1 到 n。 除第一个顶点之外的每个顶点都恰好有一个给定的父节点，它隐式定义了每个节点 i 与其父节点 p_i 之间的无向边。 因为每个节点都指向一个索引较小的父节点，所以该结构保证是一棵以 1 为根的有效树。 

任务是考虑每对顶点 u 和 v，包括 u 等于 v 的对，计算边中它们之间的距离，并对所有这些距离求和。 当两个端点是同一顶点时，单个对贡献为零，否则贡献唯一简单路径上的边数。 

n 高达 300000 的约束迫使我们远离任何试图逐对计算距离的解决方案。 一个天真的全对最短路径风格的想法会隐式地检查 O(n^2) 对，这在最坏的情况下已经是大约 9e10 次操作，远远超出了任何可行的限制。 即使在时间和内存上存储或迭代所有对也是不可能的。 

每个节点的直接 BFS 或 DFS 都会出现一个更微妙的问题。 尽管每次遍历都是线性的，但重复 n 次会将成本乘以 O(n^2)，这在约束下再次崩溃。 

边缘情况主要是结构性的而不是数字性的。 当 n 等于 1 时，没有边，因此没有非零路径，因此答案必须为零。 另一个极端情况是高度倾斜的树，由于深度高达 300000，它强调朴素的递归实现，需要迭代遍历或仔细的递归处理。 

## 方法

 暴力方法将显式计算每对顶点之间的距离。 对于每个起始顶点 u，我们运行 BFS 或 DFS 来计算到所有其他顶点 v 的距离，然后累积这些距离。 每次遍历的成本为 O(n)，重复 n 个起点会产生 O(n^2) 总工作量。 当 n 为 300000 时，这将需要大约 10^10 次操作，这在远程是不可行的。 

关键的观察是我们实际上并不需要个体距离。 我们只需要所有路径上每条边的总贡献。 两个顶点之间的每条简单路径都与一组特定的边恰好相交一次。 这使我们能够将视角从顶点对转移到边缘。 

如果我们修复一条边，例如节点与其父节点之间的边，则删除该边会将树分成两个连接的组件。 从一个组件中的节点到另一个组件中的节点的任何路径都必须恰好穿过该边一次。 因此，最短路径使用该边的对的数量恰好是两个结果分量的大小的乘积。 

这将问题转化为计算子树大小。 一旦我们以节点 1 为树根，每条边都将一个节点连接到其父节点，并且以节点为根的子树的大小直接告诉我们该边的一侧有多少个顶点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了|
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先将父数组解释为邻接表。 2到n的每个节点i都连接到p_i，形成一棵无向树。

1. 通过在 i 和 p_i 之间为 2 到 n 中的每个 i 添加一条边，从父表示构建邻接列表。 这允许双向遍历，这对于干净地计算子树大小是必要的。 
2. 以节点 1 为树的根并使用深度优先遍历计算子树大小。 我们维护一个 size 数组，其中 size[v] 表示 v 的子树中存在多少个节点，包括 v 本身。 
3. 在 DFS 期间，我们访问一个节点并递归计算其所有子节点的大小。 处理完子 c 后，我们将 size[c] 添加到 size[v] 中。 这个聚合步骤是向上累积子树信息的。 
4. 计算完所有子树大小后，我们评估原始父列表中的每条边。 对于 i 和 p_i 之间的边，假设以 i 为根的子树是该边的“子边”。 路径穿过该边的对的数量是 size[i] 乘以 n 减去 size[i]。 
5. 对从 2 到 n 的所有 i 的这些贡献求和以获得最终答案。 

微妙的一点是我们从不明确计算距离。 每条边独立地贡献于总和，子树大小完全决定有多少条路径使用该边。 

### 为什么它有效

 树中的每对顶点都有唯一的简单路径。 该路径可以分解为边，并且每对使用它的每条边都被精确计数一次。 当删除一条边时，树分裂成两个不相交的顶点集。 当且仅当其端点位于该分割的不同分量中时，一对对总距离和有贡献。 DFS 计算确保我们知道每条边的一个分量的大小，这足以对每条边对所有此类对进行一次计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input().strip())
    if n == 1:
        print(0)
        return

    p = list(map(int, input().split()))

    g = [[] for _ in range(n + 1)]
    for i in range(2, n + 1):
        par = p[i - 2]
        g[i].append(par)
        g[par].append(i)

    size = [0] * (n + 1)

    def dfs(v, parent):
        size[v] = 1
        for to in g[v]:
            if to == parent:
                continue
            dfs(to, v)
            size[v] += size[to]

    dfs(1, -1)

    ans = 0
    for i in range(2, n + 1):
        s = size[i]
        ans += s * (n - s)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先从父表示构建邻接列表。 这是至关重要的，因为子树计算需要双向遍历，即使输入编码有向父关系。 

DFS 从节点 1 开始单遍计算子树大小。每个节点将其大小初始化为 1，并累积其子节点的大小。 父参数可防止重新访问前一个节点。 

最终的累积循环利用这样一个事实：每条边 (i, p_i) 都与作为子边的节点 i 唯一关联，因此 size[i] 正是该边分割的一个组成部分。 

一个常见的实现陷阱是使用递归而不增加递归限制，这对于链状树来说会失败。 另一个微妙之处是确保父数组索引正确，因为它从 p2 开始。 

## 工作示例

 考虑一个由四个节点组成的简单链：1 连接到 2、2 连接到 3、3 连接到 4。 

对于此输入，父数组是`1 2 3`。 从节点 1 计算出的子树大小如下所示。 

| 节点| 家长 | 子树大小 |
 | --- | --- | --- |
 | 1 | - | 4 |
 | 2 | 1 | 3 |
 | 3 | 2 | 2 |
 | 4 | 3 | 1 |

 每个边缘贡献的计算方式为 size[i] 乘以 n 减去 size[i]。 

对于 i=2，贡献为 3 × 1 = 3。对于 i=3，贡献为 2 × 2 = 4。对于 i=4，贡献为 1 × 3 = 3。总计为 10。 

这与链中所有成对距离的直接枚举相匹配，其中距离自然地跨中间边累积。 

现在考虑以节点 1 为中心的星形，节点 2、3、4 都直接连接到 1。 

| 节点| 子树大小 |
 | --- | --- |
 | 1 | 4 |
 | 2 | 1 |
 | 3 | 1 |
 | 4 | 1 |

 每片叶子贡献 1 × 3 = 3，因此总答案为 9。这对应于距离为 2 的每对叶子以及距离为 1 的每对叶子到中心，与计算的总和匹配。

 ## 边缘情况

 当 n 等于 1 时，算法在构建任何结构之前立即返回零。 这与不存在不同顶点对的事实相匹配。 

在线性链中，递归深度达到n。 DFS 是递归写入的，因此需要增加递归限制以避免堆栈溢出。 关于输入`1 2 3 ... n-1`，子树大小从根到叶严格减小，并且每个边缘贡献仍然被正确捕获为 size[i] 乘以 n 减去 size[i]。 

在星形树中，除了中心之外的每个节点的子树大小都是 1。每个这样的边都贡献 n 减 1，并且对所有叶子求和会产生正确的总数，而无需重复计算，因为每条边都由其父表示中的子节点唯一标识。
