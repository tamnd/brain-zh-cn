 ## 解决方案
 ## 问题理解

 太空探险家访问固定序列的天体。 如果研究的话，每个物体都会提供一定的科学价值，但也会消耗两种有限的资源：能量和时间。 探索者必须按照给定的顺序处理对象，并为每个对象决定是研究它还是跳过它。 一旦选择，一个物体就会完全消耗其所需的能量和时间。 

目标是选择这些对象的子序列，遵循固定顺序，使得使用的总能量不超过 K，总时间不超过 M，同时最大化总科学价值。 

输出是双重的。 首先，我们报告可实现的最大价值。 其次，我们按照访问的递增顺序输出所选对象的索引。 

这些约束足够严格，建议采用动态规划方法。 当 N 高达 100 并且两个资源限制也最多为 100 时，跟踪每个对象前缀使用了多少能量和时间的状态空间仍然很小。 朴素的子集枚举需要检查所有 2^N 个子集，这在 N=40 左右已经变得不可行，而这里 N 达到 100，因此指数搜索立即被排除。 

排序约束产生了一个微妙的问题。 这不是一个可以自由重新排序物品的标准背包，但决定仍然仅取决于是否按顺序获取或跳过每个物品，这保留了经典的 DP 结构。 

当在约束范围内无法获取任何对象时，就会出现边缘情况。 例如，如果K=5且M=5并且每个对象至少需要6个能量或6个时间，则任何选择都是有效的。 在这种情况下，正确的输出是单个零，而不是空序列或零值后跟空行。 总是打印序列头的简单重建方法很容易错误地格式化这种情况。 

另一个极端情况是存在多个最优解决方案。 由于任何有效序列都是可接受的，因此 DP 不需要强制执行字典顺序，但重建仍必须与存储的决策一致。 

## 方法

 暴力策略是尝试对象的每个子集，同时通过检查包含掩码隐式保留顺序。 对于每个子集，我们将能量、时间和价值相加，然后验证可行性。 这在概念上是有效的，因为它直接匹配问题定义，但需要评估 2^N 个子集。 当 N=100 时，这意味着大约 10^30 个候选者，这远远超出了任何计算限制。 

问题的结构揭示了决策之间的唯一交互是通过累积资源使用来实现的。 每个对象都独立地贡献价值、能量和时间，并且一旦处理，除了通过剩余容量之外，它不会影响未来的转换。 这种独立性建议对前缀和资源预算进行动态规划。 

对于每个对象，我们决定是否跳过它或接受它，如果接受，我们会减少两个剩余容量。 这自然会导致指数、所用能量和所用时间的三维 DP。 由于N、K、M都最多为100，因此状态总数约为100万个，并且每个状态的转换都在恒定的时间内完成，这是高效的。 

为了重建所选序列，我们在 DP 转换期间存储决策，记住状态是来自获取还是跳过当前项。 从具有最大值的最终状态回溯会产生所需的序列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了|
 | 最佳DP | O(N·K·M) | O(N·K·M) | O(N·K·M) | O(N·K·M) | 已接受 |

 ## 算法演练

 我们将 dp[i][j][k] 定义为使用前 i 项在花费 j 能量和 k 时间时可实现的最大值，或者将无法到达的状态视为负无穷大。

1. 将除 dp[0][0][0] 为零之外的所有 dp 状态初始化为不可到达。 这表示从没有项目且没有资源使用开始。 
2. 迭代从 1 到 N 的项目。对于每个项目，我们传播来自前一层的所有可到达状态的转换。 
3. 对于每个状态 dp[i−1][j][k]，我们首先考虑跳过当前项。 如果 dp[i][j][k] 改善了结果，我们用相同的值更新。 这在不消耗资源的情况下保留了可行性。 
4. 我们还考虑采用当前项，前提是 j + energy[i] ≤ K 且 k + time[i] ≤ M。我们用 dp[i−1][j][k] + V[i] 更新 dp[i][j + F][k + T]。 这编码了包含该项目的决定。 
5. 除了 dp 更新之外，我们还存储一个父指针，用于记录状态是否来自跳过或获取，以及它源自哪个前一个 (j, k)。 
6.处理完所有项后，我们扫描所有dp[N][j][k]以找到最大值及其对应的状态。 
7. 我们通过向后移动存储的父指针直到到达 i = 0 来重建路径，收集决策要获取该项目的所有索引。 

它之所以有效，是因为每个状态 (i, j, k) 都总结了使用前 i 项精确达到资源使用量的所有可能方法。 任何最佳解决方案都必须对应于该空间中的某些有效状态，并且转换会枚举所有合法决策，而不会遗漏或重复。 由于每次转换都保留累积值的正确性并遵守约束，因此最后一层的最大值必须是全局最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K, M = map(int, input().split())
    items = []
    for _ in range(N):
        v, f, t = map(int, input().split())
        items.append((v, f, t))

    NEG = -10**18
    dp = [[[NEG] * (M + 1) for _ in range(K + 1)] for _ in range(N + 1)]
    parent = [[[-1] * (M + 1) for _ in range(K + 1)] for _ in range(N + 1)]
    take = [[[-1] * (M + 1) for _ in range(K + 1)] for _ in range(N + 1)]

    dp[0][0][0] = 0

    for i in range(1, N + 1):
        v, f, t = items[i - 1]
        for j in range(K + 1):
            for k in range(M + 1):
                if dp[i - 1][j][k] == NEG:
                    continue

                if dp[i][j][k] < dp[i - 1][j][k]:
                    dp[i][j][k] = dp[i - 1][j][k]
                    parent[i][j][k] = (j, k)
                    take[i][j][k] = 0

                nj, nk = j + f, k + t
                if nj <= K and nk <= M:
                    if dp[i][nj][nk] < dp[i - 1][j][k] + v:
                        dp[i][nj][nk] = dp[i - 1][j][k] + v
                        parent[i][nj][nk] = (j, k)
                        take[i][nj][nk] = 1

    best_val = 0
    bj = bk = 0
    for j in range(K + 1):
        for k in range(M + 1):
            if dp[N][j][k] > best_val:
                best_val = dp[N][j][k]
                bj, bk = j, k

    if best_val == 0:
        print(0)
        return

    res = []
    i, j, k = N, bj, bk
    while i > 0:
        pj, pk = parent[i][j][k]
        if take[i][j][k] == 1:
            res.append(i)
        j, k = pj, pk
        i -= 1

    res.reverse()
    print(best_val)
    print(*res)

if __name__ == "__main__":
    solve()
```DP表是在项目上逐层构建的，确保每个状态仅依赖于前一个项目索引。 跳过转换向前复制先前的值，而获取转换则增加值并增加两个资源计数器。 

父数组和取数组存储足够的信息来重建所选的子序列。 重建从最佳最终状态向后走，直到到达空前缀，收集设置了 take 标志的索引。 

一个微妙的点是，我们只初始化 dp[0][0][0] 并将所有其他状态视为不可访问，这可以防止无效的部分状态影响转换。

我们从概念上跟踪一条有效路径的 dp 状态：

 | 步骤| 项目 | 状态（能量、时间）| 价值| 行动|
 | ---| ---| ---| ---| ---|

 ## 边缘情况

 当每个项目在能量或时间上都太昂贵时，DP 永远不会超越初始状态。 该算法正确地检测到了这一点，因为最佳值保持为零并且不存在所获取项目的父指针。 输出变为单个零，与所需的格式匹配。 

当存在多个最优路径时，例如两个不同的子集在相同的约束下产生相同的最大值，则重建遵循最后写入 DP 表中的转换。 由于该问题允许任何有效答案，因此这种非确定性是可以接受的，并且仍然会产生正确的子序列。 

当K或M等于0时，只有资源消耗为零的项目才是可选的。 由于约束保证 F ≥ 1 且 T ≥ 1，因此唯一可能的结果是零值，算法通过初始化状态自然地处理零值。
