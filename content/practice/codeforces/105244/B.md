 ## 解决方案
 ## 问题理解

 给定一棵有 n 个顶点的树，每个顶点都带有一个数值。 如果我们采用该树的任何连接组件，则其成本以一种稍微不寻常的方式定义：我们将该组件中的顶点数乘以这些顶点上存储的值的总和。 

我们可以从树中删除恰好一个顶点 u。 删除 u 会删除顶点以及与其接触的所有边，这会将树分成几个断开连接的组件。 每个结果组件都有使用上述规则计算出的自己的成本，我们将这些成本加在一起。 目标是选择u使得这个总成本尽可能大。 

输入结构是树的有根式父编码，但从概念上讲，它只是一棵无向树。 约束最多为 3×10^5 顶点，这立即排除了为每个候选删除顶点从头开始重新计算组件结构的任何解决方案。 每个节点的任何二次甚至 O(n log n) 都太慢，因此预期的解决方案必须是线性或接近线性的。 

朴素推理中的一个微妙问题是独立处理组件，而没有注意到它们对移除顶点的依赖程度。 例如，在一棵星形树中，移除中心会产生许多单节点组件，而移除叶子会产生一个大组件。 仅考虑本地节点值而不跟踪子树大小的简单方法会错误评估此类情况。 

考虑一个小例子：

 输入：

 n = 4

 值 = [1, 2, 3, 4]

 边缘：1-2、1-3、1-4

 如果我们删除顶点 1，我们会得到三个孤立的节点。 总成本变为 1·2 + 1·3 + 1·4 = 9。 

如果我们删除顶点 2，我们就会得到一个分量 {1,3,4}。 其成本为3·(1+3+4)=24。 所以最好的选择在当地并不明显； 这取决于全球结构。 

这种对组件大小和总和的依赖性使得问题变得不那么简单。 

## 方法

 直接的暴力策略很容易想象。 对于每个候选顶点 u，我们将其删除，运行 DFS 或 BFS 来查找所有结果组件，计算每个组件的大小和值的总和，并累积成本。 由于每次删除都需要遍历树的大部分内容，因此每个顶点的工作量为 O(n)，导致最坏情况下的整体操作为 O(n^2)，这远远超出了 n 高达 3×10^5 的限制。 

关键的观察是删除顶点不会创建任意结构。 在有根树中，删除 u 会将图准确地拆分为其邻居相对于 u 的子树。 每个邻居贡献一个完整的子树或 u 的子树的补集。 这意味着我们只需要子树大小和子树值总和，而不需要重新计算的组件。 

一旦我们为树建立根，每条边都定义了父子关系。 对于节点 u，子节点的所有边都对应于那些子子树的组件。 唯一剩下的组件是“树的其余部分”，它是不包括 u 的子树的整个树。 

这将问题简化为预先计算两个标准树 DP 数组：子树大小和子树总和。 之后，可以在 O(deg(u)) 时间内评估每个候选根移除，并且在所有顶点上这仍然是线性的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了|
 | 最优树DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 为了方便起见，我们固定任意根，并将该树视为远离它的方向。 

1. 使用 DFS 从根计算子树大小和子树总和。

对于每个节点 u，我们将 size[u] 计算为其子树中的节点数，将 sum[u] 计算为该子树中的值之和。 这种预处理为我们提供了有关任何“向下”组件的完整信息。 
2. 对于每个节点 u，我们根据其在根树中的邻接结构来解释删除 u 的效果。 

u 的每个子 v 都成为一个独立的组件，等于 v 的子树。u 上方的树的其余部分成为一个附加组件。 
3. 对于 u 的每个子 v，将贡献大小[v] × sum[v] 添加到 u 的答案中。 

这正是应用于该组件的成本定义，因为大小和总和都已通过预处理获知。 
4. 如果 u 不是根，则计算 u 子树外部剩余分量的贡献。 

该组件的大小为 n − size[u]，总和为 S − sum[u]，其中 S 是所有节点值的总和。 
5. 删除 u 的总成本是其子组件的所有贡献加上父组件（如果适用）的贡献之和。 
6. 取所有节点 u 的最大值。 

正确性取决于删除 u 不会扭曲任何相邻组件的内部结构，它只会分离已经明确定义的子树或子树的补集。 

预处理维护的不变量是，以任何节点为根的每个子树都准确地表示一个连接组件，该连接组件将出现在切割位于该子树之上的任何删除场景中。 因为删除后每个可能的组件要么是子子树，要么是子树的补集，因此所有必需的聚合都已可用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    parent = [0] * n
    adj = [[] for _ in range(n)]
    
    p = list(map(int, input().split()))
    for i in range(1, n):
        parent[i] = p[i-1] - 1
        adj[parent[i]].append(i)
        adj[i].append(parent[i])

    sys.setrecursionlimit(10**7)

    sz = [0] * n
    sm = [0] * n

    def dfs(u, p):
        sz[u] = 1
        sm[u] = a[u]
        for v in adj[u]:
            if v == p:
                continue
            dfs(v, u)
            sz[u] += sz[v]
            sm[u] += sm[v]

    dfs(0, -1)

    total_sum = sum(a)
    ans = -10**30

    def dfs2(u, p):
        nonlocal ans

        cur = 0

        for v in adj[u]:
            if v == p:
                continue
            cur += sz[v] * sm[v]

        if p != -1:
            rest_sz = n - sz[u]
            rest_sm = total_sum - sm[u]
            cur += rest_sz * rest_sm

        ans = max(ans, cur)

        for v in adj[u]:
            if v == p:
                continue
            dfs2(v, u)

    dfs2(0, -1)

    print(ans)

if __name__ == "__main__":
    solve()
```第一个 DFS 构建经典的子树 DP 数组。 第二个 DFS 通过对每个事件分量的贡献求和来将每个节点评估为已删除的顶点。 唯一微妙的一点是正确处理“父侧”组件，这就是为什么我们依赖总和和总大小减去子树信息，而不是尝试显式重建该组件。 

## 工作示例

 ### 示例 1

 输入：

 n = 4

 a = [1, 2, 3, 4]

 边缘：1-2、1-3、1-4

 我们计算以 1 为根的子树值。 

| 节点已删除 | 子组件贡献 | 父组件贡献 | 总计 |
 | ---| ---| ---| ---|
 | 1 | 1·2 + 1·3 + 1·4 = 9 | 无 | 9 |
 | 2 | 0 | 3·(1+3+4)=24 | 24 |
 | 3 | 0 | 3·(1+2+4)=21 | 21 | 21
 | 4 | 0 | 3·(1+2+3)=18 | 18 | 18

 最好的选择是删除节点 2，它隔离了一个大的高和组件。 

这证实了最优选择通常来自于最大化剩余结构的乘积而不是局部去除成本。 

### 示例 2

 输入：

 n = 5

 a = [1, -1, -1, -1, -1]

 链条：1-2-3-4-5

 这里子树和大量交替，因此不同位置的分裂会改变符号交互。 

| 已删除 | 组件| 总计 |
 | ---| ---| ---|
 | 3 | 左{1,2}，右{4,5} | 2·0 + 2·(-2) + 2·(-2) = -8 |
 | 1 | {2,3,4,5} | 4·(-4) = -16 | 4·(-4) = -16 |

 中间的去除效果更好，因为它将负质量平衡成更小的成分，从而减少了负产物的大小。

 ## 边缘情况

 关键的边缘情况是删除的顶点是根。 在这种情况下，没有父端组件，只有子子树起作用。 该算法干净地处理了这个问题，因为父指标是 -1 并且我们跳过补码计算。 

另一种情况是退化链。 在这样的树中，每次删除最多创建两个组件，并且正确性完全取决于正确计算补码大小和总和。 使用total_sum减去子树总和可以避免对特殊外壳内部节点的需要。 

对于星形树，子树大小为 1 或 n−1，具体取决于根位置。 该算法仍然有效，因为子树 DP 正确捕获了所有子树的贡献，并且在删除叶子时父侧项变得占主导地位。 

在所有情况下，分解为子树组件和补集可确保不会遗漏任何隐藏的组件结构。
