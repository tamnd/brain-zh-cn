 ## 解决方案
 ## 问题理解

 我们得到了一张长度为 的起始彩票$n$。 每个位置最初包含数字 1。对于每个位置$i$，有一个需要的目标值$b_i$。 如果我们能够做到$i$第 - 个位置等于$b_i$，我们赚$c_i$硬币。 

我们可以使用有限数量的操作独立地修改每个位置。 一次操作选择一个索引$i$和一个正整数$z$，并增加当前值$a_i$经过$\lfloor a_i / z \rfloor$。 所有位置从1开始，我们最多允许$k$所有指数的总操作。 目标是选择操作，以便某些仓位子集达到其相应的位置$b_i$，最大化已完成位置的硬币总数。 

约束规模适中，但运行预算较大：$n \le 1000$,$k \le 10^6$， 和$b_i \le 1000$。 这立即表明我们不会天真地模拟所有步骤的操作$k$，因为操作空间太大。 相反，我们需要了解每个职位增长速度的结构。 

一个微妙的点是，运营仅通过共享预算进行交互$k$，而每个指标独立演化。 一旦我们知道实现每个目标的成本，这种独立性通常暗示着背包式的选择问题。 

不明显的边缘情况来自以下位置$b_i = 1$。 这些需要零操作并且始终贡献$c_i$。 另一个角落是当$k = 0$，我们只能取那些已经为值 1 的值。 

关键的困难在于理解需要多少次操作才能将 1 转换为给定的值$b_i$，因为操作定义不是线性的或基于标准增量的。 

## 方法

 我们首先分析单个位置如何演变。 

从当前值$x$，一次操作可以让我们移动到$$x \rightarrow x + \left\lfloor \frac{x}{z} \right\rfloor$$对于任何$z \ge 1$。 最好的选择是最大化添加量。 自从$\lfloor x/z \rfloor$最大化时$z = 1$，最好的操作总是：$$x \rightarrow x + x = 2x$$任何$z > 1$只会减少增量，因此无法帮助减少达到目标所需的步骤数。 这将过程简化为纯加倍。 

从1开始，之后$t$操作的最大可实现值为$2^t$。 由于这在每一步也是最优的，因此达到或超过所需的最少操作数$b_i$是：$$w_i = \lceil \log_2 b_i \rceil$$现在每个位置都成为一个独立的项目：选择位置$i$成本$w_i$运营和收益价值$c_i$。 我们必须选择总成本最多的子集$k$，最大化总价值。 

这正是一个0/1背包问题。 唯一的复杂之处是$k$可以大到$10^6$，但实际重量$w_i$的边界是$\log_2(1000)$，最多为 10。因此所有物品的重量总和最多约为 10000。我们可以安全地使用背包来计算总重量总和，而不是$k$，截断容量。 

蛮力替代方案将尝试每个索引的所有子集和所有可能的操作序列，这是指数级的$n$并且也是指数级的$k$，这是完全不可行的。 最佳增长总是加倍的观察结果将整个操作系统分解为标准加权选择问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对操作和子集的暴力破解 | 指数| 指数| 太慢了|
 | 背包过衍生成本|$O(n \cdot \sum w_i)$|$O(\sum w_i)$| 已接受 |

 ## 算法演练

 ### 关键思想：将每个位置转换为成本值对

 1. 对于每个位置$i$，计算达到所需的最少操作次数$b_i$从 1 开始。 

这计算为$w_i = \lceil \log_2 b_i \rceil$。 

此步骤以简单的成本取代了复杂的操作系统。 
2.如果$b_i = 1$， 放$w_i = 0$。 

无需任何操作，因此该项目始终可以免费获利。 
3. 计算所有的总和$w_i$。 这是最大的有意义的背包容量，因为我们的花费永远不会超过所有物品的总和。 
4. 定义一个 DP 数组，其中`dp[x]`是精确使用可获得的最大硬币$x$运营。 
5. 初始化`dp[0] = 0`和所有其他状态为负无穷。 
6. 处理各个位置$i$并应用标准 0/1 背包转换：

 对于每个容量$x$从当前最大值下降到$w_i$， 更新：$$dp[x] = \max(dp[x], dp[x - w_i] + c_i)$$7. 答案是所有值中的最大值$dp[x]$为了$x \le k$，但上限为权重总和。 

反向迭代结束$x$确保每个物品最多使用一次，保持 0/1 背包的正确性。 

### 为什么它有效

 转换步骤是有效的，因为从任何状态$x$，操作总是有一个主导选择：加倍。 任何较慢的增长策略都无法减少达到阈值的步骤数$b_i$，因为与加倍相比，它在每一步都会产生严格更小的值。 因此，达到的最短时间$b_i$正是所需的加倍次数。 

一旦每个物品都有独立于其他物品的固定成本，问题就变成了具有单一约束的资源分配问题，这就是背包。 DP 不变量是在处理完第一个$i$项目，`dp`仅使用这些项目存储每个可能的运营预算的最佳可实现值。 每个转换都保留可行性并且不重复使用项目，因此不会引入无效的组合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    k = int(input())
    b = list(map(int, input().split()))
    c = list(map(int, input().split()))

    weights = []
    for bi in b:
        if bi <= 1:
            weights.append(0)
        else:
            w = 0
            x = 1
            while x < bi:
                x *= 2
                w += 1
            weights.append(w)

    max_w = sum(weights)
    max_w = min(max_w, k)

    NEG = -10**18
    dp = [NEG] * (max_w + 1)
    dp[0] = 0

    for wi, ci in zip(weights, c):
        if wi == 0:
            for j in range(max_w + 1):
                if dp[j] != NEG:
                    dp[j] += ci
            continue

        for j in range(max_w, wi - 1, -1):
            if dp[j - wi] != NEG:
                dp[j] = max(dp[j], dp[j - wi] + ci)

    print(max(dp))

if __name__ == "__main__":
    solve()
```预处理循环将每个目标值转换为达到该目标值所需的加倍次数。 DP 数组是这些派生权重的标准 0/1 背包。 零成本的项目通过将其贡献添加到所有状态来单独处理，因为它们始终包含在内。 

关键的实现细节是将 DP 大小截断为$\min(k, \sum w_i)$，即使在以下情况下也能保持快速求解$k$很大。 

## 工作示例

 考虑一个小例子：

 输入：```
n = 3, k = 3
b = [2, 3, 4]
c = [1, 1, 1]
```得出的权重为：

 - 2需要1倍
 - 3需要2倍
 - 4需要2倍

 所以我们有项目：

 (1,1), (2,1), (2,1)

 DP进化：

 | 项目 | 容量 0 | 1 | 2 | 3 |
 | ---| ---| ---| ---| ---|
 | 开始 | 0 | -∞ | -∞ | -∞ |
 | (1,1) | 0 | 1 | -∞ | -∞ |
 | (2,1) | 0 | 1 | 1 | -∞ |
 | (2,1) | 0 | 1 | 2 | 2 |

 最好的结果是 3 次操作内获得 2 个硬币。 

这表明该算法自然地更喜欢首先选择更便宜的目标，但仍然可以从最佳组合项目中受益。 

第二个例子：

 输入：```
n = 2, k = 1
b = [1, 2]
c = [5, 10]
```重量：

 - 1 → 0
 - 2 → 1

 DP 从 dp[0] = 0 开始。 

项目 (1,5) 是免费的并且到处都加 5：

 dp = [5, 5]

 项目 (1,10) 加 10：

 dp = [15, 15]

 即使只有一次操作，答案也是 15，因为第一项不需要任何操作。 

这证实了零成本项目总是自动包含在内。

 ## 边缘情况

 一个关键的边缘情况是当所有$b_i = 1$。 在这种情况下，每个权重都为零，因此 DP 退化为对所有权重求和$c_i$。 该算法可以处理这个问题，因为每个零成本项目都会立即应用于所有状态，从而有效地积累所有奖励而不消耗容量。 

另一种情况是当$k = 0$。 只有重量为零的项目才会保持活动状态。 由于DP仅在零容量下初始化，任何正权重项都是不可到达的并且没有任何贡献，这符合要求。 

对于小目标，例如$b_i = 2$，权重恰好为 1。DP 正确地将这些视为单位成本项目，确保它们在有限的预算下公平竞争。 

转换步骤还可以处理大型$b_i$正确。 即使$b_i = 1000$，计算出的权重最大为10，因此不会发生溢出或过度的DP扩展，保持性能稳定性。
