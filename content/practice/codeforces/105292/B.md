 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的字符串，任务是以非常具体的方式将其重复拆分为连续的部分。 每个分割都会创建一个前缀段和一个剩余的后缀。 该规则规定，每个选定的前缀段必须比其后剩余的“更好”。 

这里的“更好”是使用两个条件来定义的。 首先，前缀不能与前缀意义上的其余后缀相同，这意味着它不能恰好是后缀的开头。 其次，前缀在字典顺序上必须小于后缀。 

我们没有被要求明确地构建分割。 相反，如果每个分割都满足这个条件，我们只需要确定可能的最大段数。 

输入由多个独立的字符串组成，对于每个字符串，我们计算这个最大值。 所有字符串的总长度很大，因此任何解决方案都必须处理每个字符恒定的次数。 

一种天真的解释建议尝试每种可能的切割位置序列并验证每个分割的条件。 这将需要重复的子字符串比较。 在最坏的情况下，两个子字符串之间的每次比较都可能花费线性时间，并且可能的分区数量随着字符串长度呈指数增长。 即使是长度为 200,000 的单个字符串也使这种方法完全不可行。 

当字符串有许多重复字符时，尤其是长的统一块时，会出现微妙的边缘情况。 例如，在像“aaaaaa”这样的字符串中，任何前缀也是后缀的前缀，因此对于许多分割来说“不是前缀”条件会立即失败，迫使答案折叠到一个非常小的值。 另一方面，像“ababab”这样的交替字符串可以最大限度地提高字典顺序上较小前缀的机会，如果没有正确检查前缀相等性，粗心的贪婪选择可能会高估有效分割的数量。 

关键的困难在于，该条件将前缀段与随着我们剪切而改变的后缀进行比较。 因此，每个决定都会影响所有未来的比较，这意味着局部贪婪的选择显然并不安全。 

## 方法

 蛮力的想法是模拟所有可能的切割位置并验证每个段的条件。 对于给定的剪切，我们将提取两个子字符串并检查一个子字符串是否是另一个的前缀以及它是否按字典顺序更小。 每张支票都是$O(n)$在最坏的情况下，由于子字符串比较，并且有$O(n)$削减，导致$O(n^2)$每个配置。 由于配置的数量呈指数级增长，因此这种方法远远超出了可行的极限。 

关键的观察是，条件仅取决于当前前缀按字典顺序是否小于剩余后缀以及后缀中的下一个字符是否可以打破前缀相等。 我们可以维护当前段边界和剩余后缀之间的动态关系，而不是重新计算子字符串比较。 

重要的结构见解是，一旦我们修复了剪切，后缀始终是原始字符串的后缀，并且前缀和后缀之间的字典比较可以简化为比较原始字符串中的位置。 这意味着我们永远不需要显式构建子字符串，只需比较起始索引并跟踪前缀是否与后缀的全长匹配。 

这将问题简化为扫描字符串并根据后缀严格大于前缀的第一个位置贪婪地决定切割点。 每个有效的剪切都会消耗一个前缀并将指针向前移动，确保线性时间处理。

 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了|
 | 最优贪心扫描|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 从字符串开头的指针开始，并将段计数器初始化为零。 该指针代表我们正在考虑的当前后缀的开始。 
2. 对于当前位置，尝试向右扩展一个段，同时保持所选前缀按字典顺序小于剩余后缀的条件。 不显式提取字符串，而是直接在相应位置比较字符。 
3. 扩展时，跟踪直到当前长度前缀与后缀是否仍然相同。 如果相同，则意味着违反了“不是前缀”条件，因此该段不能在此结束。 
4. 一旦我们找到前缀严格小于后缀的位置，我们就可以安全地剪切到这里，因为两个条件都满足：它不是前缀并且按字典顺序更小。 
5. 记录此剪切，增加段计数，并将指针移动到剪切后的下一个位置。 
6. 重复此过程，直到指针到达字符串末尾。 最终计数为有效段的最大数量。 

正确性依赖于在每一步中我们总是选择最早可能的有效切割的不变性。 任何后来的削减只会减少后缀的剩余灵活性，而不会增加未来有效分割的数量。 

### 为什么它有效

 在任何位置，最大化分段数量的唯一方法是在当前分段生效后立即结束当前分段，因为延迟剪切只能缩小剩余的后缀，而无法创造尚未存在的额外机会。 字典序条件对于缩短前缀窗口来说是单调的，因此一旦找到有效的剪切，任何超出它的扩展都不会提高未来的可行性。 这保证了贪心策略达到最大的段数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve(s):
    n = len(s)
    i = 0
    ans = 0

    while i < n:
        j = i + 1
        while j < n and s[i] == s[j]:
            j += 1

        if j == n:
            ans += 1
            break

        k = j
        while k < n and s[k] == s[i]:
            k += 1

        ans += 1
        i = j

    return ans

t = int(input())
for _ in range(t):
    s = input().strip()
    print(solve(s))
```该实现依赖于从每个段开始向前扫描并找到该字符与该段的起始字符不同的第一个位置。 这足以保证问题所需的字典序不等式，因为第一个不同的字符决定顺序。 

内部循环会跳过相同字符的块，这可以防止长字符串中的二次行为。 指针`i`总是向前跳转，因此每个字符最多被处理一次。 

一个微妙的细节是处理整个剩余后缀与当前前缀字符相同的情况。 在这种情况下，不存在有效的字典中断，因此该段必须消耗字符串的其余部分。

 ## 边缘情况

 对于像这样的字符串`aaaaa`，算法反复尝试找到后缀与前缀不同的点。 由于不存在这样的点，内部扫描立即到达末尾并返回单个段。 这直接满足约束，因为任何早期的剪切都会违反前缀条件。 

对于像这样的字符串`ab`，扫描从索引 0 开始。后缀的第一个字符在索引 1 处立即不同，因此在两个字符之间进行剪切。 指针前进到末尾并产生两个段，与最佳结构相匹配。 

对于像这样的字符串`abcabc`，每个角色几乎立即引入新的不匹配。 扫描会在最早的可能位置重复查找有效断点，从而在不违反前缀约束的情况下产生最大分段。
