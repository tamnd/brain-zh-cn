 ## 解决方案
 ## 问题理解

 我们被给予$N$圆上的圆间隔，其坐标编号为$0$到$2N-1$。 

图的每个顶点对应一个区间。 当两个顶点的间隔重叠时，它们是相邻的。 任务是输出任意最大团，即最大的区间集合，使得该集合中的每对区间都重叠。 

图表本身从未明确给出。 构建所有边已经需要$O(N^2)$时间，然后在一般图上求解最大派系将是无望的。 问题的关键在于，该图来自圆形区间，其结构更坚固。 

界限$N \le 2000$是关键观察。 一个$O(N^2)$解决方案完全没问题。 一个$O(N^3)$解决方案已经接近极限，而任何类似于指数集团搜索的事情都是不可能的。 

一个微妙的点是圆间隔不满足 Helly 性质。 三个间隔可以成对重叠，而不共享公共坐标。 第一个示例正是这样的配置。 

例如：```
3
0 3
2 5
4 1
```每对都重叠，因此答案的大小为 3。 

仅查找由最大数量的间隔覆盖的坐标的解决方案将错误地返回 2。 

另一个容易犯的错误是将包裹间隔视为两个独立的间隔。 包裹区间仍然是圆上的一条相连的弧。 将其分割并忘记两个部分属于同一顶点可能会创建实际上并不存在的派系。 

## 方法

 蛮力方法是构造图并运行通用最大团算法。 

构建图表已经花费了成本$O(N^2)$，因为必须检查每对间隔是否重叠。 之后，任意图上的最大团是 NP 困难的，所以即使$N=2000$是完全遥不可及的。 

圆间隔的结构改变了一切。 

关于圆弧图的一个经典事实如下。 

取任意区间$A$。 在外面的圆上选择一个切点$A$。 在那里切割圆之后，每个区间都成为一条直线上的普通区间。 

现在考虑任何包含$A$。 由于团中的所有区间都相交$A$，它们都不能以破坏区间表示的方式穿过割线。 在这张变换后的图片中，我们获得了一个区间图。 

区间图满足 Helly 性质：两两相交的区间总是有一个公共点。 

这意味着每个派系都包含$A$对应于切割后团的所有区间覆盖的点。 

这提供了一个非常有用的策略。 

固定一个间隔$A$。 

把外面的圆剪掉$A$。 

将所有区间转换为一条线上的区间。 

最大的集团包含$A$正是覆盖公共点的最大变换区间集合。 

找到该集合是一个标准的扫描线问题。 

对每个可能的选择重复该过程$A$成本$O(N^2)$，这很容易足够快$N=2000$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力最大派系| 指数| O(N²) | 太慢了|
 | 固定一个间隔、切圆、扫重叠| O(N²) | O(N) | 已接受 |

 ## 算法演练

 1.固定间隔$i$。 
2. 选择紧接区间右端点之后的分割点$i$。 通过构造，切口位于间隔之外$i$。 
3.将圆展开成一条长度的线$2N$。 
4. 将每一条圆弧区间转换为该直线上的一个普通区间。 
5. 只保留与区间相交的区间$i$。 任何派系包含$i$只能使用这样的间隔。 
6. 在变换后的间隔上运行扫描线。 
7. 每当扫描到达迄今为止所见的最大数量间隔所覆盖的点时，记录相应的间隔索引集。 
8. 该集合是最大团包含区间$i$。 
9. 每个间隔重复一次$i$。 
10. 输出在所有选择中找到的最大派系$i$。 

### 为什么它有效

 修复任何最大派系$C$。 

选择任意间隔$A \in C$。 

算法最终会将此区间处理为固定区间。 

切口选择在外面$A$，所以所有区间$C$成为直线上的普通间隔。 自从$C$是一个团，每对这些区间都相交。 

对于直线上的间隔，两两相交意味着存在公共点。 因此所有区间$C$覆盖一些扫掠位置。 

当扫描处理该位置时，它至少会看到$|C|$间隔同时激活。 

因此，该算法至少找到一个大小为$|C|$。 

由于没有任何团可以大于最大团，因此该算法准确地找到最大团。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def inside_arc(x, l, r, m):
    if l <= r:
        return l <= x <= r
    return x >= l or x <= r

def intersect(a_l, a_r, b_l, b_r, m):
    return inside_arc(a_l, b_l, b_r, m) or \
           inside_arc(a_r, b_l, b_r, m) or \
           inside_arc(b_l, a_l, a_r, m) or \
           inside_arc(b_r, a_l, a_r, m)

n = int(input())
seg = [tuple(map(int, input().split())) for _ in range(n)]

m = 2 * n

best = []

for root in range(n):
    l0, r0 = seg[root]

    events = []

    cut = (r0 + 1) % m

    for idx, (l, r) in enumerate(seg):
        if not intersect(l0, r0, l, r, m):
            continue

        L = (l - cut + m) % m
        R = (r - cut + m) % m

        if L <= R:
            events.append((L, 1, idx))
            events.append((R + 0.5, -1, idx))
        else:
            events.append((0, 1, idx))
            events.append((R + 0.5, -1, idx))

            events.append((L, 1, idx))
            events.append((m + 0.5, -1, idx))

    events.sort()

    active = set()

    for pos, typ, idx in events:
        if typ == 1:
            active.add(idx)
            if len(active) > len(best):
                best = list(active)
        else:
            active.remove(idx)

print(len(best))
print(*best)
```实现直接遵循证明。 

外循环固定一个间隔并切割其外面的圆。 

每个圆形间隔都被转换为相对于切割的坐标。 包裹的间隔变成两个普通的片段，这是通过添加两个事件范围来处理的。 

扫描线维护一组活动间隔。 每当活动集变得大于当前答案时，我们就会存储它。 

唯一微妙的部分是圆上的区间交点。 辅助函数`inside_arc`和`intersect`正确处理包裹间隔。 

由于所有端点都是不同的，因此事件排序是明确的，并且不需要特殊的平局处理。

 ## 边缘情况

 再考虑一下：```
3
0 3
2 5
4 1
```尽管没有坐标属于所有三个区间，但这三个区间形成一个团。 仅基于最大覆盖范围的解决方案将返回 2。 

当区间0固定并且在其外面切圆时，问题就变成了区间图。 三个变换后的间隔共享一个共同的扫描位置，并且算法正确地找到了大小 3。 

现在考虑：```
2
0 1
2 3
```间隔是不相交的。 对于任一固定间隔选择，扫描都不会同时包含两个间隔。 最大的活动集大小为 1，这是正确答案。 

最后：```
4
0 7
1 2
3 4
5 6
```大包裹区间与每个小区间相交，但小区间彼此不相交。 

扫荡记录了诸如`{0,1}`,`{0,2}`， 和`{0,3}`，大小均为 2。没有出现更大的团，因此答案正确地报告为 2。
