 ## 解决方案
 ## 问题理解

 我们有两个矩形网格代表两个“图像”。 每个单元格都包含一些对像素进行编码的值，通常是字符或小整数。 目标是通过在平面上移动图像并计算有多少重叠位置匹配来确定一幅图像与另一幅图像的匹配程度。 

更具体地说，想象将第二个图像放置在第一个图像之上，并将其滑动到所有可能的相对位置。 对于每个班次，我们都会比较重叠的细胞并计算有多少对细胞是相同的。 输出通常是此类重叠得分的最大值，或者有时是实现该重叠得分的班次列表。 

关键结构是我们不是比较固定位置，而是比较两个 2D 图案之间的所有平移。 

从复杂度的角度来看，让网格有大小$n \times m$。 对单班成本的简单比较$O(nm)$，并且有$O(nm)$转变，导致$O(n^2 m^2)$运营。 这太大了，当$n, m$达到$2000$或类似的，这在图像匹配问题中很常见。 

困难的情况来自密集的网格，其中每次转变都会产生许多重叠的单元格。 例如，如果两个网格都填充了相同的值，则每次移位都会产生一个大的重叠区域，并且朴素计数会重复重新计算相同的贡献。 

一个微妙的边缘情况是当一个或两个图像高度稀疏时。 如果大多数单元为零或为空，则朴素的全卷积仍会不必要地迭代空结构。 另一种情况是当最佳偏移发生在重叠区域的边界时，其中部分重叠较小但仍然是最佳的。 

## 方法

 蛮力方法会尝试所有可能的翻译$(dx, dy)$。 对于每个班次，它都会迭代两个网格相交处的所有单元格并计算匹配项。 这是正确的，因为它直接评估重叠相似度的定义。 然而，对于每个班次来说，这是$O(nm)$，并与$O(nm)$轮班后，总工作变为$O(n^2 m^2)$，即使对于中等大小的网格也是不可行的。 

关键的观察结果是，每一对匹配的细胞都恰好促成一个转变：如果细胞$(i, j)$在第一张图像中匹配单元格$(x, y)$在第二张图片中，那么它们对转变做出了贡献$(i - x, j - y)$。 这将问题转化为聚合相对偏移量的贡献，而不是显式地重新计算重叠。 

这种重新表述相当于计算两个二进制或加权网格之间的二维卷积。 我们不是显式滑动，而是通过根据位移向量对所有匹配像素对进行分组来聚合它们。 如果网格值是任意整数或字符，我们将单独处理每个值，并且仅在相等值之间累积贡献。 

这减少了从枚举班次到枚举匹配单元对的问题，这可以在$O(nm)$或者$O(nm \log nm)$取决于实施策略。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力转变|$O(n^2 m^2)$|$O(1)$额外 | 太慢了|
 | 偏移聚合（卷积思想）|$O(nm)$或者$O(nm \log nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 我们将问题从“尝试所有转变”转换为“通过位移累积贡献”。 

1. 读取两个网格并分别存储第一张图像和第二张图像中每个符号的位置。 这种分离很有用，因为只有相同的符号才能促成匹配。 
2. 对于网格中出现的每个值，收集它出现在第一幅图像中的所有坐标以及它出现在第二幅图像中的所有坐标。 
3.对于每对坐标$(i, j)$从第一张图片和$(x, y)$从具有相同值的第二张图像计算位移$(i - x, j - y)$。 
4. Maintain a hash map keyed by displacement. 每次计算位移时，都会增加其计数器。 该计数器表示在该班次下对齐的匹配单元数。 
5. 处理完所有值后，答案就是所有位移中的最大计数。 

按值分组的原因是不匹配的值永远不会对任何有效的对齐分数产生影响，因此我们完全避免比较不相关的对。 

### 为什么它有效

 两个图像之间的每个有效重叠对应于唯一的位移矢量。 每对等值单元对一个位移桶恰好贡献一次。 因此，位移的哈希映射计数正是该位移下匹配对齐单元的数量。 由于每种可能的对齐方式都由一些位移表示，因此所有桶中的最大值给出了最佳匹配分数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import defaultdict

def solve():
    n, m = map(int, input().split())
    g1 = [input().strip() for _ in range(n)]
    g2 = [input().strip() for _ in range(n)]

    pos1 = defaultdict(list)
    pos2 = defaultdict(list)

    for i in range(n):
        for j in range(m):
            pos1[g1[i][j]].append((i, j))
            pos2[g2[i][j]].append((i, j))

    counter = defaultdict(int)

    for val in pos1:
        if val not in pos2:
            continue
        a = pos1[val]
        b = pos2[val]
        for i, j in a:
            for x, y in b:
                counter[(i - x, j - y)] += 1

    print(max(counter.values(), default=0))

if __name__ == "__main__":
    solve()
```实现的核心结构直接反映了算法。 我们首先按符号对坐标进行存储，以便我们只比较有意义的对。 匹配符号上的嵌套循环是唯一可能昂贵的部分，但在典型的约束下，要么字母表很小，要么分布足够稀疏，从而保持高效。 

一个常见的实施错误是忘记位移必须按相对差异而不是绝对坐标进行分组。 另一个微妙的问题是处理不存在班次的情况，在这种情况下，答案应默认为零。

 ## 边缘情况

 一种重要的边缘情况是没有位移累积任何匹配。 在这种情况下，频率图仍然是空的，因此直接取最大值会失败。 该实现通过使用来处理这个问题`default=0`在`max`，确保输出为零。 

另一种情况是所有单元格共享相同的值。 然后，每一对都会对每个位移做出贡献，并且该算法的网格大小将成为该值的二次方。 这是最坏情况下的预期行为，它强调了为什么实际约束通常会确保稀疏性或小字母表。 

最后一个微妙的情况是负位移索引。 由于位移可能为负，因此需要使用哈希图而不是数组。 任何使用固定二维数组的尝试都需要偏移移位和仔细的边界处理，这是不必要的且容易出错。
