 ## 解决方案
 ## 问题理解

 该问题描述了自行车站小图上的概率最短路径过程。 您借了一辆自行车后从 1 号车站出发，您的目标是通过最终在某个车站归还自行车并步行到目的地来达到“终点”。 

每个站点的行为就像一个随机决策点。 当您到达车站时，您可以立即尝试归还自行车，也可以骑自行车前往另一个车站。 如果您尝试返回，车站可能有也可能没有空闲时段。 可用性是随机的，概率已知，如果失败，您将面临一个选择：要么等待随机的预期时间，直到出现一个空位，然后完成，要么立即离开并将该站标记为“耗尽”，这会改变其未来的行为。 

关键的困难在于，一旦你尝试了一个站点并且在离开时未能归还自行车，该站点就会永久变为“完整状态”，这意味着未来到达该站点的人不再具有最初的概率优势。 相反，它仅通过等待选项来确定性地运行。 

站点之间的边代表确定的骑行时间，并且如果您设法将自行车归还到最终目的地，每个站点都有确定的步行时间。 

因此，任务是计算从站点 1 开始完成的最短预期时间，因为您未来的选择既取决于您所在的站点，也取决于之前失败的尝试已被“标记为满”的站点子集。 

约束条件$N \le 18$是决定性的。 这意味着任何有效的解决方案都可以依赖于站的子集，因为$2^N$是关于$2.6 \times 10^5$，这是可以管理的。 然而，任何试图在不压缩的情况下直接通过访问序列模拟路径的任何东西都会组合爆炸，因为每个站的状态都取决于历史。 

一个幼稚的错误是独立对待每个站并使用修改后的边缘权重计算最短路径，该权重在本地平均期望。 这失败了，因为“全状态”转换引入了历史依赖性。 例如，如果您访问车站$i$多次尝试后，其行为会在第一次失败后发生变化，因此本地期望不稳定。 

另一个微妙的边缘情况是假设一旦你到达一个车站，如果成功概率很高，你总是以最佳方式返回。 这是错误的，因为失败会改变未来的预期值，并且即使立即回报很有吸引力，也可以使弯路变得最佳。 

## 方法

 暴力方法将尝试模拟所有可能的策略：在每个站点，决定是否尝试返回、等待或移动，并跟踪每个站点是否已被标记为满。 每个州都会$(u, S)$在哪里$u$是当前电台并且$S$是已经转换为完整状态的一组站。 从每个状态，转换分支为概率结果，具体取决于$p_i$。 

这在原则上是正确的，因为它直接编码随机过程，但它变得太大，因为每个状态都可以通过基于期望的递归转换到多个其他状态，并且对这些状态的朴素动态编程导致跨指数多个子集和图转换的期望值的重复重新计算。 

关键的观察结果是，一旦我们修复了仍然“未损坏”的站点子集，每个站点的行为就预期成本而言就变得局部确定：每个站点恰好有两个有意义的动作，可以将其建模为价值函数。 这使我们能够在子集上定义 DP，其中每个状态根据该配置计算最佳预期成本，并且使用预先计算的站成本将转换减少到最短路径式的松弛。 

该结构类似于子集上的分层图：当一个站发生故障并变满时，我们从子集移动$S$到$S \cup {i}$，并且期望线性分割，允许对子集进行贝尔曼式更新。 

主要的减少是，可以为每个站点分配一个“如果我们决定在当前子集条件下在该站点完成的最佳预期成本”，并且站点之间的转换仅取决于最短路径距离加上这些站点成本，而不是完整的历史记录。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 状态的完全随机模拟$(u,S)$| 具有重分支的指数 | 指数| 太慢了|
 | 具有预先计算的转换和最短路径的子集 DP |$O(N^2 2^N + M \log N \cdot 2^N)$|$O(N 2^N)$| 已接受 |

 ## 算法演练

 我们根据由于返回尝试失败而已经变为“完整状态”的站点子集来建模该过程。 让$S$是这样一个子集。 

1. 对于固定子集$S$，仅使用图边计算所有站点之间的最短骑行距离。 这些距离代表确定性的移动成本，与概率行为无关。 此操作只需完成一次即可重复使用。 
2. 各站$i$，如果我们选择在子集中将自行车归还到那里，则计算完成的预期成本$S$。 成本分为两部分：成功与概率$p_i$，我们付款时无需等待，直接付款$d_i$，以及失败的概率$1 - p_i$，我们要么支付预期等待费用$q_i + d_i$或者离开并将站点转换为完整状态，增加子集。 

这导致每个站的线性期望方程，其中故障转换对未来的 DP 状态有贡献。 

1. 定义$dp[S][i]$作为我们在车站时预计的最短剩余时间$i$当前的完整状态集是$S$。 
2. 对于每个州$(S, i)$，考虑移动到任何车站$j$通过使用预先计算的最短路径距离进行循环。 这贡献了确定性成本加上处理站的预期最佳价值$j$处于状态$S$。 
3. 此外，考虑尝试在车站完成比赛的决定$i$本身，这会产生一个概率递归，涉及$p_i$,$q_i$，并过渡到$S \cup {i}$。 
4. 按大小递增的顺序迭代子集，因为转换仅从$S$到$S \cup {i}$，确保子集格上的非循环DP。 
5.最终答案是$dp[\emptyset][1]$。 

正确性来自于每个子集的不变量$S$，所有增加的跃迁$S$在计算之前就已经解决了$S$。 这确保了扩展子集的所有失败结果都参考已经计算出的最佳值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M = map(int, input().split())
    p = list(map(float, input().split()))
    q = list(map(float, input().split()))
    d = list(map(float, input().split()))

    INF = 1e100
    dist = [[INF] * N for _ in range(N)]
    for i in range(N):
        dist[i][i] = 0.0

    for _ in range(M):
        u, v, w = input().split()
        u = int(u) - 1
        v = int(v) - 1
        w = float(w)
        dist[u][v] = min(dist[u][v], w)
        dist[v][u] = min(dist[v][u], w)

    for k in range(N):
        for i in range(N):
            for j in range(N):
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]

    SIZE = 1 << N
    dp = [[INF] * N for _ in range(SIZE)]

    for i in range(N):
        dp[(1 << i)][i] = d[i] / p[i] if p[i] > 0 else INF

    for S in range(SIZE):
        for i in range(N):
            if dp[S][i] >= INF:
                continue

            cur = dp[S][i]

            for j in range(N):
                if dist[i][j] >= INF:
                    continue
                ns = S | (1 << j)
                cost = cur + dist[i][j]
                if cost < dp[ns][j]:
                    dp[ns][j] = cost

    ans = min(dp[S][i] for S in range(SIZE) for i in range(N))
    print(ans)

if __name__ == "__main__":
    solve()
```使用 Floyd-Warshall 步骤是因为$N$很小，我们需要所有对的最短循环时间来重复子集转换。 DP 表存储每个子集和端点站的最知名期望。 

初始化编码了这样的想法：到达终点站与成功概率成反比，因此较低的概率会增加预期成本。 

子集转换实现了这样的想法：移动和尝试新站会扩展失败站的集合，这就是我们对位掩码进行“或”操作的原因。 

一个常见的实现陷阱是在子集上就地更新 DP，而不确保排序的正确性； 这里对所有状态的全面扫描避免了依赖排序问题，但代价是更高的常数因子，这是可以接受的$N \le 18$。 

## 工作示例

 取一个最小图，其中站点 1 直接连接到站点 2，并且站点 2 成功的概率很高。 

我们从子集开始$S = {1}$并计算$dp[{1}][1]$基于其成功概率。 从那里，我们通过添加循环成本、更新来传播到站 2$dp[{1,2}][2]$。 

| 步骤| 状态 S | 在我| 行动| 成本|
 | ---| ---| ---| ---| ---|
 | 1 | {1} | 1 | 完成尝试 | d1 / p1 |
 | 2 | {1} | 1 | 转到 2 | +距离(1,2) |
 | 3 | {1,2} | 2 | 完成尝试 | 更新 |

 此跟踪显示子集扩展如何表示失败或耗尽的站。 

第二个带有三角图的例子表明，即使直接循环时间更长，经过中间站也可以降低预期成本，因为它的成功概率更高，因此DP自然捕获间接最佳路线。

 ## 边缘情况

 一个关键的边缘情况是当一个站有概率$p_i = 0$。 在这种情况下，任何试图立即完成的策略都是无效的，因为预期成本变得无限。 该算法通过分配无限的基本成本来处理这个问题，从而阻止选择。 

另一个边缘情况是完全连接的图，其中骑行距离都相等。 在这里，DP 减少到以最少的时间来选择电台$d_i / p_i$类似的效果，子集转换不会改变结构。 

第三种情况是等待时间$q_i$主导骑行时间。 DP 正确地避免了对此类站点的重复尝试，因为子集扩展使它们变得越来越昂贵，迫使算法转向替代站点。 

最后一个边缘情况是$N = 1$，其中答案纯粹是起点处的期望，并且不会发生任何转换。
