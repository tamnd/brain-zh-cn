 ## 解决方案
 ## 问题理解

 我们被赋予了一个$N \times N$两个玩家交替将代币放置在空单元格上的网格。 该限制是全局的：不允许两个放置的令牌正交相邻，这意味着它们不能共享一条边。 一旦一个单元格被占用，其所有四个边缘邻居在未来的放置中都会受到限制，除非它们已经被占用，因为放置在那里会违反规则。 

当玩家轮到没有合法动作时，游戏结束，该玩家输。 Ltf 率先采取行动，双方都被认为发挥得非常完美。 任务不是模拟游戏，而是确定给定的棋盘尺寸$N$，第一个玩家是否可以强行获胜。 

输入是单个整数$N$，输出是一个表示获胜者的单词。 

即使网格可以大到$10^5 \times 10^5$，规则仅取决于邻接，而不取决于游戏过程中增长的任何动态结构。 这强烈表明答案取决于网格的结构不变量而不是任何模拟过程。 

一种天真的解释会尝试将游戏状态建模为一个图表，其中每次移动都会删除一个顶点并禁止其邻居。 这立即导致网格图上具有指数分支的组合游戏。 即使对于小$N$，枚举所有移动序列变得不可行，因为每个移动最多影响四个邻居，并且博弈树生长得非常快。 

边缘情况最容易在小网格上看到。 

为了$N = 1$，只有一个单元格。 第一个玩家拿走它，第二个玩家没有动作，所以第一个玩家获胜。 

为了$N = 2$，任何移动都会阻挡棋盘的很大一部分。 事实上，在第一次放置之后，不可能进行第二次移动，因此，如果在给对手一个强制移动序列（导致获胜的答复结构）后，最佳游戏导致立即僵局，那么第一个玩家就会失败。 官方样本证实$N=2$第一个玩家输了。 

为了$N = 3$，有足够的空间来交错放置，以便第一个玩家始终可以镜像或控制剩余可用单元格的奇偶性，并且第一个玩家获胜。 

这些例子已经暗示涉及宇称或二分结构，而不是几何复杂性。 

## 方法

 暴力方法会将每个有效的放置配置建模为游戏状态，并递归地模拟所有可能的移动，使用极小极大将状态标记为获胜或失败。 单次移动包括选择一个不与任何已选择的单元格相邻的单元格，然后更新禁止区域。 在最坏的情况下，一个有效配置的数量$N \times N$独立约束下的网格呈指数分布$N^2$，所以探索这个状态空间是完全不可行的，即使对于$N=10$。 

关键的观察结果是，邻接限制将网格变成了二部图，其中每个单元格的颜色都像棋盘一样。 任何有效的放置棋子集必须在此图中形成一个独立的集。 那么游戏就相当于玩家交替选取顶点，使得没有两个选定的顶点与已经选定的顶点相邻，这相当于一步步构建一个最大独立集。 

在二部图上，最佳游戏结构崩溃为一个简单的平价论点：当且仅当可用的“强制配对机会”总数不抵消第一步优势时，第一个玩家获胜。 对于没有阻塞单元的完整网格，结果仅取决于网格大小是否导致平衡或不平衡的分区结构。 

对于一个$N \times N$网格，棋盘着色的两个颜色类别具有相同的大小$N$是偶数，并且恰好相差 1，当$N$很奇怪。 这种不平衡决定了谁在最佳比赛中拥有最后一步。 邻接限制确保每一次移动都有效地消耗这种二分结构中的一个单元格，而不允许相同颜色选择之间的交叉交互。 

这减少了整个游戏的奇偶校验$N$。

 ## 算法演练

 1. 观察网格可以像棋盘一样着色，将所有单元格分成两个不相交的集合，其中一组内没有两个单元格相邻。 这将放置规则转换为选择二分图中的独立顶点。 
2. 认识到每次移动都会从可用移动集中删除一个单元格，并可能禁用最多四个相邻单元格，但这些相邻单元格属于二分结构中相反的颜色类别。 
3. 认识到影响最佳游戏的唯一全局结构是两个颜色类别之间的不平衡。 在一个$N \times N$网格，当以下情况时计数相等$N$是偶数，并且当$N$很奇怪。 
4. 将游戏结果简化为平局决策：如果$N$是均匀的，对称性允许第二个玩家总是以保持平衡的方式做出反应； 如果$N$奇数时，第一个玩家继承额外的单元格并保持最后一步的优势。 
5. 仅根据该奇偶条件输出获胜者。 

### 为什么它有效

 关键的不变量是，在任何合法移动序列之后，剩余的可玩结构仍然可以解释为二分图，其中移动总是消耗一个顶点并仅间接约束相反分区中的顶点。 这可以防止任何玩家破坏两个分区之间的全局对称性，除非由网格尺寸引起的初始不平衡。 由于最佳游戏总是尽可能保持对称性，因此结果仅取决于该对称性是否完美（甚至$N$）或稍有偏差（奇数$N$）。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())

# For this game, parity determines the winner
# odd N -> first player wins, even N -> second player wins
if n % 2 == 1:
    print("Ltf")
else:
    print("Ian")
```该代码读取单个整数并直接应用奇偶校验规则。 不需要模拟或预处理，因为游戏结构崩溃为恒定时间决策。 

唯一的微妙之处是确保正确处理输入格式并去除空格，因为整个逻辑取决于单个整数读取。

 ## 边缘情况

 对于$N=1$，板上恰好包含一个单元格。 第一个玩家立即拿走，不给对手留下合法的棋步，因此输出为 Ltf。 

为了$N=2$，网格足够小，任何移动都会严重限制剩余空间，并且对称性完全有利于第二个玩家。 该算法正确地将其分类为偶数，生成 Ian。 

对于较大的偶数值，例如$N=10^5$，结构不随尺度变化。 奇偶校验仍然返回 Ian，并且由于仅执行单个模运算，因此不会出现溢出或性能问题。
