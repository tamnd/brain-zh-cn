 ## 解决方案
 ## 问题理解

 我们有一个由空单元格组成的矩形 Akari 板`.`和墙壁`#`。 

灯泡只能放置在空电池上。 它的光线水平和垂直传播，直到到达墙壁或棋盘的边缘。 两个灯泡不允许互相照亮，这意味着同一不间断的行段或同一不间断的列段中不能有两个灯泡。 

任务不是优化任何东西。 我们只需要输出灯泡的任何有效位置并用`L`。 

网格可以大到`2000 × 2000`，这意味着可能有多达四百万个细胞。 任何从每个灯泡位置重复扫描行和列的算法都会太慢。 我们需要细胞数量基本呈线性的东西。 

错误的常见来源是考虑单个单元而不是可见性段。 

考虑：```
...
```最左边单元格中的一个灯泡照亮了整行。 在该行中放置更多灯泡将立即违反规则，因为灯泡会看到彼此。 

另一个容易犯的错误是忘记墙壁会分割可见性。```
.#.
```左右单元格属于不同的行段。 灯泡可能存在于两侧，因为墙壁阻挡了光线。 

第三个微妙的情况是孤立的单元格：```
#
.
#
```该单元形成长度为一的行段和列段。 有效的解决方案可以在那里放置一个灯泡，并且任何基于段的表示都必须正确处理此类单例段。 

## 方法

 暴力的观点是重复选择灯泡位置，然后验证每个单元格是否都被照亮以及是否任何一对灯泡都能看到对方。 即使检查单个候选解决方案也可能需要扫描很长的行和列。 由于单元格多达 400 万个，直接探索布局是完全不可行的。 

关键的观察结果是可见性仅由行段和列段决定。 

行段是最大连续块`.`一行内的单元格。 墙壁打破了部分。 列段的定义类似。 

每个空单元格恰好属于一个行段和一个列段。 

现在考虑一个二部图。 

一侧包含所有行段。 

另一侧包含所有柱段。 

对于每个空单元格，在其行段和列段之间添加一条边。 

将灯泡放入单元格中意味着选择相应的边缘。 

没有两个灯泡互相照亮的规则变得非常简单：行段最多可以包含一个灯泡，列段最多可以包含一个灯泡。 用图的术语来说，所选的边必须形成匹配。 

照明怎么样？ 

如果行段包含灯泡，则该行段中的每个单元格都会被照亮。 对于柱段也是如此。 

如果单元的两个段端点中的至少一个与选定的匹配边缘相关，则该单元被照亮。 

这暗示了一个经典的图事实：任何最大匹配的端点形成顶点覆盖。 每条边至少有一个匹配的端点。 

因此，如果我们在线段图中构造任何最大匹配，则每个单元格的边缘至少有一个匹配的端点。 这意味着每个单元都被其行段的灯泡或列段的灯泡照亮。 

更好的是，贪婪匹配已经是最大的。 我们可以扫描所有单元格，并在两个相应的段仍未使用时放置一个灯泡。 

如果显式构建，该图将非常巨大，但网格结构使我们可以在扫描棋盘时直接处理边缘。

 ## 算法演练

 1. 计算每个空单元格的行段 ID。 

从左到右扫描每一行。 每当一个新的连续块`.`cells 开始，为其分配一个新的行段 id，并将该 id 赋予块中的每个单元格。 
2. 计算每个空单元格的列段 ID。 

从上到下扫描每一列。 每当一个新的连续块`.`cells 开始，为其分配一个新的列段 id，并将该 id 赋予块中的每个单元格。 
3. 维护两个布尔数组。 

一个数组记录行段是否已经匹配。 

另一个记录列段是否已经匹配。 
4. 扫描网格的所有单元格。 

对于每个空单元格，查看其行段 id 和列段 id。 

如果两个段都尚未匹配，则在该单元中放置一个灯泡并将两个段标记为匹配。 

这正是最大匹配的标准贪婪构造。 
5. 输出板子。 

每个选定的单元格都会变成`L`。 墙壁依然存在`#`。 所有其他空单元格均保留`.`。 

### 为什么它有效

 所选择的灯泡位置对应于匹配的边缘，因为每个行段和每个列段最多使用一次。 

贪心过程产生最大匹配。 完成后，不存在两个端点不匹配的边，否则可以添加该边。 

取出任何空单元格。 其对应的图边连接其行段和列段。 

由于匹配是最大的，因此该边的至少一个端点是匹配的。 该匹配的端点在同一段中的某处包含一个灯泡。 

如果匹配的端点是行段，则灯泡将照亮包含单元格的整个行段。 

如果匹配的端点是柱段，则灯泡将照亮包含单元格的整个柱段。 

所以每个空单元格都会被照亮。 

由于选定的边形成匹配，因此没有行段包含两个灯泡，也没有列段包含两个灯泡。 因此，没有两个灯泡可以互相照亮。 

构造始终有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
grid = [list(input().strip()) for _ in range(n)]

row_id = [[-1] * m for _ in range(n)]
row_cnt = 0

for i in range(n):
    j = 0
    while j < m:
        if grid[i][j] == '#':
            j += 1
            continue

        rid = row_cnt
        row_cnt += 1

        k = j
        while k < m and grid[i][k] == '.':
            row_id[i][k] = rid
            k += 1

        j = k

col_id = [[-1] * m for _ in range(n)]
col_cnt = 0

for j in range(m):
    i = 0
    while i < n:
        if grid[i][j] == '#':
            i += 1
            continue

        cid = col_cnt
        col_cnt += 1

        k = i
        while k < n and grid[k][j] == '.':
            col_id[k][j] = cid
            k += 1

        i = k

row_used = [False] * row_cnt
col_used = [False] * col_cnt

ans = [row[:] for row in grid]

for i in range(n):
    for j in range(m):
        if grid[i][j] != '.':
            continue

        r = row_id[i][j]
        c = col_id[i][j]

        if not row_used[r] and not col_used[c]:
            row_used[r] = True
            col_used[c] = True
            ans[i][j] = 'L'

for row in ans:
    print("".join(row))
```第一阶段标记行段。 每个最大水平空单元格接收一个标识符。 

第二阶段对柱段执行相同的操作。 

之后，每个空单元已经知道与其边对应的两个图顶点。 

贪婪匹配阶段非常小。 如果两个线段端点当前都是空闲的，我们将匹配它们并放置一个灯泡。 

输出网格被初始化为原始板的副本。 仅通过匹配选择的单元格更改为`L`。 

对于孤立的单元、长度为一的片段或包含许多壁的板，不需要特殊处理。 段结构自然可以处理所有这些问题。

 ## 边缘情况

 考虑：```
1 2
..
```有一个行段和两个列段。 

贪婪匹配将灯泡放入第一个单元格中。 行段变得匹配。 无法添加第二个单元格的边缘，因为行段已被使用。 

第二个单元格仍然亮着，因为它与灯泡属于同一行段。 

现在考虑：```
1 3
.#.
```两个空单元格属于不同的行段和不同的列段。 

贪婪扫描可能会在两个单元中放置一个灯泡。 这是合法的，因为墙壁阻挡了他们之间的可见性。 

最后，考虑：```
3 1
#
.
#
```单独的空单元格本身形成行段和列段。 

两个线段端点最初都是空闲的，因此算法在那里放置了一个灯泡。 细胞会自行发光，解决方案是有效的。
