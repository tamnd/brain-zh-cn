 ## 解决方案
 ## 问题理解

 我们有两块$300 \times 300$木板。 一步棋选择一个棋子的位置$(x,y)$并将其移动到矩形内任何严格较小的位置$[1,x] \times [1,y]$。 目的地不能是当前方格，也不能被其他棋子占据。 

游戏是公正且有限的。 每一步都会减少至少一个坐标，因此游戏最终必须结束。 不能移动的玩家就输了。 

对于每个起始配置，我们必须计算有多少第一步获胜，并假设之后的最佳玩法。 测试用例的数量为$10^5$，因此任何针对每个测试用例执行博弈论计算的方法都太慢了。 我们需要对获胜动作进行封闭式描述。 

一个微妙的一点是，这两部分并不是完全独立的。 禁止移动到另一块。 盲目计算两个独立 Grundy 数的异或的解决方案将错过这种相互作用。 

考虑：```
(1,1) and (2,1)
```较大的一块想要移动到$(1,1)$，但该方格已被占用，因此该移动不存在。 

另一个重要的情况是当两个棋子位于同一条对角线上时$x+y=\text{constant}$。 例如：```
(3,2) and (2,3)
```两个位置具有相同的单件 Grundy 编号。 粗心的基于异或的论证会说位置正在丢失，因为异或为零，这是正确的，但我们仍然需要理解为什么没有移动可以保留零状态。 

## 方法

 蛮力方法是将每对棋子位置视为一个游戏状态，并通过游戏图上的动态规划来计算其 Sprague-Grundy 值。 有$90{,}000$板单元，因此有序两件状态的数量约为$8 \times 10^9$。 甚至存储状态也是不可能的。 

关键的观察结果是单件已经具有非常简单的结构。 

让$g(x,y)$是一件的格伦迪数。 从$(x,y)$，下方和左侧矩形中的每个方格都是可到达的。 如果我们定义$$k=x+y-2,$$然后每个较小的值$0,1,\dots,k-1$出现在可达位置之中，并且没有可达位置具有价值$k$。 因此，墨西哥正是$k$。 

所以$$g(x,y)=x+y-2.$$棋盘塌陷成对角线。 同一条对角线上的每个正方形都有相同的格伦迪数。 

现在考虑两块。 

如果它们的对角线值不同，比如$a<b$，那么该位置的行为与两个大小相同的 Nim 堆完全相同$a$和$b$。 其格伦迪数为$a \oplus b$。 

如果两个棋子都在同一条对角线上，则位置很特殊。 两块棋子占据不同的方格，但具有相同的单块格伦迪数。 直接的墨西哥论证表明这样的状态有格伦迪数$0$。 

这立即给出了获胜策略结构：

 如果两个对角线值相等，则该位置处于亏损状态，并且没有获胜的先手棋。 

如果对角线值不同，则进入失败状态的唯一方法是将对角线值较大的棋子移动到对角线值较小的棋子上。 对角线上的每一个合法目的地都是一个获胜的举动。 

剩下的任务是纯粹的几何任务：计算给定对角线在允许的矩形内有多少个正方形可以到达，如果可以到达则排除已占用的正方形。

 | --- | --- | --- | --- |
 | 蛮力 |$O(\text{all states})$| 不可能| 太慢了|
 | 最佳 |$O(1)$每个测试用例|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算对角线值$$g_1=x_1+y_1-2,\qquad g_2=x_2+y_2-2.$$1.如果$g_1=g_2$，输出0。 

对角线值相等的状态是失败的位置，因此任何移动都不会导致另一个失败的位置。 
2. 假设$g_1>g_2$。 

任何获胜的举动都必须将第一个棋子移到对角线上$g_2$。 
3. 计算对角线上有多少个正方形$g_2$位于矩形内$$1 \le p \le x_1,\qquad 1 \le q \le y_1.$$对角线条件是$$p+q=g_2+2.$$让$$L=\max(1,\ g_2+2-y_1),$$

$$R=\min(x_1,\ g_2+1).$$那么该对角线上可到达的方块的数量是$$\max(0,\ R-L+1).$$1. 如果另一块是正方形$(x_2,y_2)$位于移动棋子的可到达矩形内，减一，因为移动到占用的方块上是非法的。 
2.如果$g_2>g_1$，对第二块进行对称计算。 

### 为什么它有效

 单件于$(x,y)$有格伦迪号码$x+y-2$。 每次移动都会严格减小该值，并且每个更小的值都是可以达到的。 

对于不同对角线上的两个棋子，游戏相当于两个 Nim 堆，其大小为对角线值。 达到零 Grundy 状态的唯一方法是使两个值相等。 

对于已经在同一条对角线上的两个棋子，状态本身具有格伦迪数$0$。 任何移动都会改变一个对角线值并破坏相等性，因此每次移动都会进入非零状态。 

因此，获胜的棋步正是将移动的棋子放置到另一棋子的对角线上的合法棋步。 对这些移动的计数减少为对矩形内对角线段的格点的计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_on_diagonal(x, y, k):
    s = k + 2

    left = max(1, s - y)
    right = min(x, s - 1)

    if left > right:
        return 0
    return right - left + 1

t = int(input())
ans = []

for _ in range(t):
    x1, y1, x2, y2 = map(int, input().split())

    g1 = x1 + y1 - 2
    g2 = x2 + y2 - 2

    if g1 == g2:
        ans.append("0")
        continue

    if g1 > g2:
        res = count_on_diagonal(x1, y1, g2)

        if x2 <= x1 and y2 <= y1:
            res -= 1

        ans.append(str(res))
    else:
        res = count_on_diagonal(x2, y2, g1)

        if x1 <= x2 and y1 <= y2:
            res -= 1

        ans.append(str(res))

sys.stdout.write("\n".join(ans))
```功能`count_on_diagonal`计算满足的格点$$p+q=k+2$$在一个矩形内。 它不是迭代坐标，而是计算有效范围$p$直接地。 

减法步骤是两个部分之间唯一的相互作用。 如果占据的方块位于移动棋子的可到达矩形内，则它将被视为有效目的地，因此必须将其删除。 

所有算术都可以轻松地适合标准 Python 整数。 电路板尺寸仅为 300，但解决方案决不依赖于该限制。

# helper: run solution on input string, return output string
import sys, io

def solve():
    input = sys.stdin.readline

    def count_on_diagonal(x, y, k):
        s = k + 2
        left = max(1, s - y)
        right = min(x, s - 1)
        if left > right:
            return 0
        return right - left + 1

    t = int(input())
    out = []

    for _ in range(t):
        x1, y1, x2, y2 = map(int, input().split())

        g1 = x1 + y1 - 2
        g2 = x2 + y2 - 2

        if g1 == g2:
            out.append("0")
        elif g1 > g2:
            res = count_on_diagonal(x1, y1, g2)
            if x2 <= x1 and y2 <= y1:
                res -= 1
            out.append(str(res))
        else:
            res = count_on_diagonal(x2, y2, g1)
            if x1 <= x2 and y1 <= y2:
                res -= 1
            out.append(str(res))

    sys.stdout.write("\n".join(out))

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout

    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()

    solve()

    result = sys.stdout.getvalue()

    sys.stdin = backup_stdin
    sys.stdout = backup_stdout

    return result

 ## 边缘情况

 考虑：```
1
3 2 2 3
```两块都有对角线值$3$。 局面已经是亏损状态。 Any move decreases one diagonal value and breaks the equality. 该算法检测到$g_1=g_2$并立即输出：```
0
```现在考虑：```
1
3 3 1 1
```较大的一块有价值$4$，较小的值$0$。 要创建失败状态，我们需要移动到对角线 0。该对角线仅包含单个正方形$(1,1)$，已被占用。 对角线上的计数为1，所占据的方格可达，因此减1，得到：```
0
```最后：```
1
6 6 2 2
```对角线 2 包含三个正方形：```
(1,3), (2,2), (3,1)
```被占领的广场$(2,2)$被删除，留下两个合法的获胜动作。 该算法返回：```
2
```与预期结果相符。
