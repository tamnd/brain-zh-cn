 ## 解决方案
 ## 问题理解

 我们得到一个带有标记边缘的图，以及两个同时在其上移动的实体：Waymo 和 Thomas。 系统的每个状态都由一对位置来描述，一个用于 Thomas，一个用于 Waymo。 从起始配置开始，两者都可以根据与边缘标签相关的特殊规则移动，并且该过程通过交替可达配置继续进行。 

关键任务不是模拟传统意义上随时间变化的运动，而是确定在所有有效的运动序列下，Waymo 最终可以访问多少个不同的节点，假设两个参与者都能对彼此的位置保持最佳反应。 

图结构很重要，因为每次移动都受到边标签的约束，并且转换取决于一个玩家站在端点，而另一个玩家站在所遍历的边的标签处。 这创建了一个系统，其中状态是位置对而不是单个节点，但状态之间的转换是高度结构化和稀疏的。 

尽管状态空间在概念上是节点数量的二次方，但实际可到达的部分要小得多，因为每个转换仅由特定的边标签交互触发。 这极大地限制了探索。 

从复杂性的角度来看，在这种类型的典型约束中，图最多可以有大约 2×10^5 条边。 对所有节点对的简单探索将需要 O(n^2) 或 O(n^2 log n) 行为，这远远超出了可行的限制。 除非我们能够证明可达状态的数量与 m 成线性关系，否则即使对所有状态进行完整的 BFS 也是不可能的。 

当多条边共享标签或标签与节点索引一致时，就会出现微妙的边缘情况。 幼稚的解释可能会将标签视为独立节点，而不强制执行双向状态转换，从而导致不正确的可达性。 

例如，如果边是 (1, 2，标签 5) 并且节点 5 连接到其他地方，则会出现标签 5 的行为是否类似于节点或符号连接器的混淆。 如果处理不当，求解器可能无法允许反向转换并低估可到达的节点。 

## 方法

 暴力解释是显式构造状态图，其中每个状态都是一对（Thomas_position，Waymo_position），并且根据移动规则应用转换。 从每个状态，我们尝试应用每个基于边缘的规则来生成新状态并执行 BFS 或 DFS。 

这是正确的，因为它直接遵循有效移动的定义。 然而，状态的数量可能是 O(n^2)，并且每个状态可能尝试沿多个边进行转换，从而在最坏的情况下导致 O(n^2 + m·n) 的行为。 这立即变得不可行。 

关键的见解是所有节点对之间的转换不是任意的。 仅当一个参与者恰好位于边的端点，而另一个参与者位于其标签时，才会发生转换。 这意味着每个有效状态都必须是通过某种边缘交互“创建”的。 因此，每个可达状态都由边缘事件锚定。 

这大大减少了有意义状态的数量。 我们不需要考虑任意的​​节点对，而只需要考虑直接从边产生的状态，或者通过最多一个从这种边引起的状态的转换步骤连接的状态。 这意味着可达状态的数量受 O(m) 限制，而不是 O(n^2)。 

一旦我们接受这个结构，我们就可以在由边引起的状态上构建紧凑的邻接表示。 每条边都会引起少量结构化状态之间的转换，我们可以在这个隐式状态图上执行 DFS。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2 + m·n) | O(n^2 + m·n) | O(n^2) | O(n^2) | 太慢了 |
 | 最佳 | O(m log m) | O(米) | 已接受 |

## 算法演练

 我们将问题重新解释为压缩状态空间上的图搜索。 

每个状态都是两个位置的配置，但我们不是枚举所有对，而是仅生成可通过边缘引发的转换到达的状态。 

## 步骤

 1. 将每条边 (u, v, l) 视为连接涉及 u、v 和 l 的配置的交互规则。 这是在国家之间创造运动的唯一机制。 
2. 构造一个隐式状态图，其中节点是有效配置 (u, v)。 我们从不枚举所有对，只枚举那些通过边触发的转换可到达的对。 
3. 对于每条边 (u, v, l)，添加状态 (u, l) 和 (v, l) 之间的双向转换。 这体现了这样一种想法：如果一个实体位于端点 u，另一个实体位于标签 l，它们可以移动到 v，同时保留约束。 
4. 每个状态使用哈希映射或有序映射来有效地存储传出转换，因为状态数量为 O(m) 并且邻接是稀疏的。 
5. 从初始配置开始运行 DFS 或 BFS。 每次我们到达一个状态时，我们都会探索由匹配边生成的所有转换。 
6. 跟踪访问过的状态以避免重新访问配置。 每个访问状态代表系统的可达配置。 
7. 计算 Waymo 在所有访问过的州中出现的不同节点位置的数量。 这就是最终的答案。 

我们可以安全地仅计算可达状态的原因是，每个有效配置都必须源自有效的边沿触发转换，并且系统中没有机制可以在该闭包之外生成配置。 

### 为什么它有效

 每个可达状态都是通过应用来自另一个可达状态的有效边缘转换来创建的。 由于转换仅通过边定义，并且每个转换都通过构造保留有效性，因此可达状态空间在边引起的配置上形成闭合图。 DFS 正是探索了这种闭包，确保完整性而不引入无效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    adj = {}

    def add(a, b):
        if a not in adj:
            adj[a] = []
        adj[a].append(b)

    edges = []

    for _ in range(m):
        u, v, l = map(int, input().split())
        edges.append((u, v, l))
        add((u, l), (v, l))
        add((v, l), (u, l))

    start = (edges[0][0], edges[0][2])

    stack = [start]
    vis = set([start])

    reachable_waymo = set()

    while stack:
        u, w = stack.pop()
        reachable_waymo.add(w)

        if (u, w) not in adj:
            continue

        for nxt in adj[(u, w)]:
            if nxt not in vis:
                vis.add(nxt)
                stack.append(nxt)

    print(len(reachable_waymo))

if __name__ == "__main__":
    solve()
```该实现在表示为对的压缩状态上构建邻接列表。 每条边都贡献两个定向转移，反映了可逆性：一旦进行移动，系统就可以返回或继续，而不失一般性。 

DFS 维护一个状态访问集，以确保对可达部分进行线性探索。 答案是通过收集访问状态的所有第二个组成部分得出的，这些组成部分对应于 Waymo 的可能位置。 

一个微妙的实现细节是直接将状态表示为元组。 这可以避免冲突并简化散列。 另一个是确保插入每个边缘的两个方向，因为可逆性是系统的核心属性。

 ## 边缘情况

 当只有一条边时，会出现一种极小的情况。 在这种情况下，系统只有一个有意义的转换，并且 DFS 最多探索两种状态。 该算法从第一条边缘开始初始化，并立即添加两个运动方向，确保保持对称性。 

第二种情况是边形成循环。 例如，(1,2,3)、(2,3,1)、(3,1,2)。 DFS 扩展到所有诱导状态而不会重复，因为访问集可以防止重新访问循环配置。 每个状态添加一次，达到关闭后可达性稳定。 

第三种情况涉及跨多个边缘的重复标签。 由于状态是按对索引的，因此相同的标签不会发生冲突，并且邻接结构保持正确。 每条边都贡献独立的转换，并且哈希图自然地合并共享端点而不会产生歧义。
