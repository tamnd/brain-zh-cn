 ## 解决方案
 ## 问题理解

 我们正在研究围绕三项选择游戏的重复回合构建的随机淘汰过程，其中每个参与者独立选择三个选项之一。 一组相同的实体以单一状态开始，每轮结束后，根据选择的分布方式，其中的某些子集会生存到下一轮。 这一过程一直持续到只剩下一个实体为止。 任务是计算每个初始组大小$x$，直到剩下一个幸存者所需的预期轮数。 

因此，输入代表相同玩家的初始数量，输出是在上述随机淘汰动态下只有一名玩家生存之前的预期游戏数量。 

递归隐含的约束结构是$x$可能足够大，以至于对所有选择配置的朴素状态扩展是不可行的。 即使是单回合也已经有$3^x$可能的结果，因为每个参与者独立地在三个行动中进行选择。 任何尝试直接枚举结果的解决方案都会立即呈指数增长。 唯一可行的方向是通过对称性压缩结果，并关注一轮后剩余的玩家数量，而不是确切的配置。 

当试图仅推理“至少发生了一次消除”而没有对事件空间进行适当调节时，就会出现一种微妙的故障模式。 如果错误地混合了无条件概率和条件概率，递归就会出现偏差。 例如，统一处理所有配置，同时限制消除配置会破坏规范化，并导致错误的期望，即使对于小型配置也是如此。$x$， 例如$x=2$，其中手工计算仍然易于处理。 

另一个边缘情况是$x=1$。 在这种情况下，根本不需要游戏，因此期望值必须恰好为零。 任何除以消除概率的递归都必须明确处理这个基本情况。 

## 方法

 直接的蛮力方法会枚举石头、剪刀、布的所有可能的分配方式。$x$每轮克隆，计算有多少存活下来，并对所有结果进行递归平均。 这在原则上是正确的，因为它遵循所有随机转变的期望的字面定义。 然而，每个州都分为$3^x$结果，甚至将相同的结果分组也只会将其减少到多项式计数，而多项式计数仍然呈指数增长$x$。 递归树在每个深度都会扩展，使得这种方法在非常小的深度之外完全不可行$x$， 大约$x \le 10$。 

关键的观察结果是，该过程仅取决于有多少玩家选择每个动作，而不取决于哪些特定玩家这样做。 这种对称性将状态空间从标记配置折叠成表示剩余玩家数量的单个整数状态。 一旦我们接受这种压缩，问题就变成了马尔可夫过程$x$，其中每个步骤都会过渡到更小的$k$具有一些可计算的概率。 

第二个重要的想法是将一轮分为两个阶段。 首先，有可能至少发生一次消除。 其次，以消灭发生为条件，剩余幸存者数量存在分布。 这种分离使我们能够计算第一轮“有用”的预期等待时间，然后使用标准期望线性将其与到较小状态的转换结合起来。 

一旦确定，预期值$dp[x]$可以写成所有较小状态的递归，并通过转换到每个可能的幸存者计数的概率进行加权。 计算这些概率简化为二项式计数：在仅出现两个选择的所有作业中（因为第三个意味着没有消除），我们计算有多少种方法准确产生$k$幸存者。 

这导致了动态规划解决方案$x$，其中每个状态都依赖于所有较小的状态。 转移概率可以组合地预先计算，并且预期等待时间项是从一轮实际导致淘汰的概率导出的。

 ## 算法演练

 我们定义$dp[x]$作为所需的预期轮数$x$相同的克隆体减少到一个幸存者。 

1.修复底座情况$dp[1] = 0$。 只有一个克隆体无法进行交互，因此不需要回合。 
2. 考虑配置$x$克隆。 单轮可以不产生消除或减少克隆的数量。 我们首先关注一轮实际上导致至少一次淘汰的概率。 仅当克隆中未出现所有三个选择时才会发生淘汰。 
3. 计算一轮仅使用两种选择的概率。 对于一对固定的选择，例如“石头和纸”，每个克隆以概率独立地在这两个选择中进行选择$(2/3)^x$。 然而，我们必须排除所有克隆只选择其中一个的情况，这对应于两个子情况，每个子情况都有概率$(1/3)^x$。 因此，对于固定对，有效概率为$(2/3)^x - 2(1/3)^x$。 
4. 由于石头、剪刀、布之间存在三对可能的选择，因此乘以 3。这将得出一轮“活跃”的总概率，这意味着至少发生一次淘汰：$$p_x = 3\left(\left(\frac{2}{3}\right)^x - 2\left(\frac{1}{3}\right)^x\right).$$1. 发生此类活跃回合之前的预期等待时间为$1/p_x$。 这捕获了在流程实际收缩之前我们预计花费的总轮数。 
2. 现在以一轮处于活动状态为条件。 在这样的一轮中，恰好出现两个选择，其中一个淘汰另一个。 幸存者正是那些选择非主导选项的克隆人。 我们计算准确的概率$k$克隆体得以生存。 
3. 对于固定选择对，我们选择哪个$k$克隆人选择幸存的选项，给予$\binom{x}{k}$。 其余$x-k$都必须选择失败的选项。 对所有非全等配置进行归一化会产生与以下比例成比例的条件概率$\binom{x}{k}$，导致幸存者呈二项式分布。 
4. 我们将这些转换结合起来以获得：$$dp[x] = \frac{\sum_{k=1}^{x-1} dp[k]\cdot P(x \to k)}{p_x} + \frac{1}{p_x}.$$第一项考虑了转换到较小状态后的预期未来成本，第二项是发生这种转换之前的等待时间。 

1. 我们按升序评估这个DP$x$，因为每个状态仅取决于较小的值。 
2. 预先计算阶乘和逆阶乘以有效计算二项式系数$O(1)$，使得总复杂度$O(n^2)$。 

### 为什么它有效

 该过程形成马尔可夫链，其中状态完全由存活克隆的数量来表征。 每一轮要么保持系统不变，要么严格向下移动。 通过以发生向下移动的事件为条件，我们将等待时间与状态转换分开。 期望的线性使我们能够将所有可能的下一个状态的贡献按其概率加权求和。 由于概率仅取决于分配的组合计数而不是恒等式，因此二项式结构是精确的并且保留了归一化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(a):
    return pow(a, MOD - 2, MOD)

def solve():
    n = int(input().strip())
    
    if n == 1:
        print(0)
        return

    # factorials for binomial coefficients
    fact = [1] * (n + 1)
    invfact = [1] * (n + 1)

    for i in range(1, n + 1):
        fact[i] = fact[i - 1] * i % MOD

    invfact[n] = modinv(fact[n])
    for i in range(n, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD

    def C(a, b):
        if b < 0 or b > a:
            return 0
        return fact[a] * invfact[b] % MOD * invfact[a - b] % MOD

    # probability-related constants in modular form
    inv3 = modinv(3)
    inv3x = [1] * (n + 1)
    inv2pow = [1] * (n + 1)

    for i in range(1, n + 1):
        inv3x[i] = inv3x[i - 1] * inv3 % MOD

    dp = [0] * (n + 1)

    for x in range(2, n + 1):
        px = (3 * (pow(2 * inv3, x, MOD) - 2 * inv3x[x])) % MOD
        if px < 0:
            px += MOD

        inv_px = modinv(px)

        total = 1  # waiting time contribution

        for k in range(1, x):
            ways = C(x, k)
            total += ways * dp[k] % MOD

        dp[x] = total % MOD * inv_px % MOD

    print(dp[n] % MOD)

if __name__ == "__main__":
    solve()
```该代码首先构建阶乘表以有效地计算组合。 DP 数组存储所有较小尺寸的预期值。 

期限$p_x$使用模算术实现，对应于一轮减少系统大小的概率。 它的倒数表示减少发生之前的预期等待时间。 

对于每个$x$，我们累积所有可能的幸存者计数的贡献$k$，按二项式系数加权。 这反映了任何子集的对称性$k$幸存者的可能性是相同的。 

最后除以$p_x$通过以淘汰赛为条件，匹配派生的循环来标准化期望。

 | 1 | 0 | 无 | 0 |

 对于单个克隆，不可能进行交互，因此该过程立即终止。 递归正确地避免了通过基本情况被零除。 

这证实了马尔可夫过程没有传出转移的边界条件。 

### 示例 2：x = 2

 | 状态 x | k=1 贡献 | p_x | dp[x] | dp[x] |
 | ---| ---| ---| ---|
 | 2 | dp[1] = 0 | 积极| 1/p_x |

 对于两个克隆体，任何活跃回合都必须消灭其中一个。 唯一可能的下一个状态是 1，其未来成本为零。 人们的期望完全降低到等待第一轮非平局回合。 

这验证了当仅存在一种过渡状态时，DP 正确地崩溃到几何等待时间。

 ## 边缘情况

 对于$x=1$，算法立即返回零，而不尝试概率计算。 这避免了消除概率被零除$p_x$，否则它将是未定义的，因为不需要游戏。 

为了$x=2$，DP 只计算一个有意义的转换$k=1$。 递归简化为几何等待时间，并且代码正确地处理了这个问题，因为内部循环结束了$k$仅贡献一个术语$dp[1]=0$，仅留下归一化$p_2$。
