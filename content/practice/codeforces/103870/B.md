 ## 解决方案
 ## 问题理解

 随着时间的推移，我们正在跟踪一个简单的重复过程。 每天都会生成一个运行计数器，用于测量自上次重置事件以来已经过去了多少天。 每当该计数器达到固定周期的倍数时$K$，我们执行一个操作：我们将一个称为“理智”的值增加一。 

因此，输入描述了一系列天，并且每一天都隐式增加一个天计数器。 唯一有趣的时刻是计数器可以被整除的那些日子$K$，因为那是理智增强的日子。 在这样的一天之后，计数器可以继续或重置为零，但两种解释是等效的，因为只有相对于$K$很重要。 

输出是处理序列中所有天后的最终健全值。 

尽管该声明的措辞是“Chessbot 向 Bossographer 发送电子邮件”，但其核心抽象只是定期计数：每次我们完成一个区块时，$K$天，我们给答案加一。 

约束很小，但这仍然是一个线性时间模拟问题。 如果天数$N$取决于$10^5$或者$10^6$，然后一个$O(N)$遍历是非常安全的。 任何比线性更糟糕的事情，例如重新计算周期或以嵌套方式检查整除性，都将是不必要的开销，但如果引入额外的因素，仍然不会通过。 

主要的边缘情况来自第一个周期如何开始以及如何精确倍数$K$表现。 错误重置或以错误顺序递增后进行检查的简单实现可能会将计数移动 1。 

例如，如果$K = 3$我们处理3天，理智应该只增加一次。 如果有人过早重置计数器，他们可能会完全错过增量。 相反，如果它们重置得太晚，它们可能会跨越边界计数两次。 

## 方法

 暴力解释是每天进行模拟，同时保留自上次重置以来数天的计数器。 计数器每天都会增加 1，每当计数器达到$K$，我们提高理智并将计数器重置为零。 这直接反映了问题描述并且通过构造是正确的。 

这种方法的成本与天数成线性关系，因为每天只需要持续的工作。 日期之间没有嵌套结构或依赖关系，因此不需要进一步优化。 任何“预先计算”事件的尝试都是不必要的，因为该过程已经是周期性的且简单的。 

关键的观察是计数器永远不需要存储超过其模数的值$K$。 我们可以用模算术的方式来思考，而不是用重置的方式来思考：每天贡献一个单位，并且每当累计总数可以被$K$，我们增加理智。 

这将问题重新定义为计算有多少个倍数$K$出现在范围内$1$到$N$，这就是简单的$N // K$。 模拟分解为单个算术表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(N) | O(1) | O(1) | 已接受 |
 | 直接算术| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取天数$N$和时期$K$。 这些定义了完整的时间线和理智增加的频率。 
2. 计算有多少个完整的大小块$K$适合$N$。 这是通过整数除法完成的$N // K$，它直接计算我们击中多个的次数$K$。 
3. 输出该值作为最终的健全水平。 每个完整块恰好对应一个增量，最后的部分块不贡献。 

### 为什么它有效

 关键的不变量是日计数器仅通过其余数模来起作用$K$。 每次我们完成一个完整的长度段$K$，计数器再次归零，新的周期开始。 因此，我们达到边界条件的总次数恰好是 length 前缀中的完整循环数$N$。 任何部分循环都不能触发增量，因为它永远不会达到$K$，因此只有完整的除法才能得出答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = input().strip().split()
    if not data:
        return
    n, k = map(int, data)
    
    # each full block of size k contributes +1 sanity
    print(n // k)

if __name__ == "__main__":
    solve()
```该解决方案读取两个整数并直接计算整数除法。 唯一的微妙之处是确保输入解析正确处理空格，因为某些竞赛格式在一行上提供了两个值。 

该部门$n // k$在所有情况下都是安全的，包括当$n < k$，它正确返回零。 这相当于根本没有完整的周期。

 ## 边缘情况

 对于以下情况$N < K$，比如输入`5 10`，计数器永远不会达到阈值。 该算法计算`5 // 10 = 0`，正确地表明理智没有增加。 逐步模拟将显示计数器从 1 进展到 5，而不会达到 10，从而确认没有增量。 

为了精确的边界对齐，例如`9 3`，模拟在第 3、6 和 9 天准确达到阈值。算法返回`9 // 3 = 3`，精确匹配三个重置事件。
