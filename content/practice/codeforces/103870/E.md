 ## 解决方案
 ## 问题理解

 我们得到了一个序列，表示随着时间的推移发生的消费事件，其中每个事件都与一个人标识符相关联。 同一个人可能出现多次，形成连续的活动片段。 从这个序列中，我们必须得出两件事：基于每个人的总参与程度的最终货币余额，以及取决于任何一个人的最长不间断活动连续性的特殊“税收调整”。 

第一部分本质上是簿记任务。 每当一个人出现在序列中时，我们都会将其解释为支出或贡献的单位，并累积每人的总数。 此阶段的输出是从个人到任何税收调整前的净余额的映射。 

第二部分不取决于总数，而是取决于序列的结构。 我们扫描同一个列表并跟踪相同人的连续片段。 对于同一个人的每个最大连续块，我们测量其长度。 拥有最长的区块的人决定税收幅度，该幅度是该区块的长度乘以其他参与者的数量减一。 该价值将从所选人员中减去，并以统一的方式重新分配给其他所有人。 

应用此重新分配后确定最终输出：我们识别调整后财富最大的人和调整后财富最小的人，并输出他们之间的差额。 

未明确提供约束，但此类问题通常涉及最多大约 10^5 个事件。 这立即排除了任何二次解决方案，例如为每个候选者从头开始重新计算线段长度或余额。 任何方法都必须使用哈希映射或单遍扫描，在序列长度上是线性或接近线性的。 

误解“最长连胜”会产生天真的陷阱。 例如，如果序列是`A A B B B A`，最长的连续是`B B B`，而不是 B 的总数。另一个微妙的情况是多人共享​​相同的最大条纹长度。 幼稚的实现可能会太晚或太早地覆盖候选人，具体取决于平局处理，这可能会改变纳税人。 

另一种故障模式是在边界处错误地更新条纹计数器。 考虑`A A B A A`。 如果实现在人员之间切换时忘记正确重置当前条纹，则可能会错误地合并条纹并高估长度。 

## 方法

 强力解释将为每个人重新计算他们出现的最长连续片段。这将需要扫描每个人的完整数组并跟踪片段边界。 如果有 n 个事件和可能有 n 个不同的人，则在最坏的情况下这会导致 O(n^2) 时间复杂度。 当 n 约为 10^5 时，这远远超出了可行的限制。 

关键的观察是，两个所需的计算都可以在一次传递中完成。 总余额可以使用由人键入的字典来增量累积。 最长的连续条纹也可以在扫描一次时进行跟踪，因为条纹本质上是局部属性，不需要重新访问较早的位置。 

在单次遍历期间，我们保持当前人和当前的条纹长度。 当这个人发生变化时，我们就会重置连胜。 在此过程中，我们不断更新迄今为止看到的最大连续记录。 这为我们提供了纳税人的身份以及 O(n) 中的税收乘数的值。 

一旦我们知道了纳税人和连续长度的值，我们就可以将再分配公式直接应用于所有人。 由于这是一个统一的转换，我们不需要对每个事件进行模拟，只需调整最终的聚合。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 维护字典的同时扫描一次列表`balance`存储每人的总贡献。 每出现一个人，其余额就会增加一个单位。 这给出了原始的税前总额，不需要任何额外的通行证。 
2. 在同一次扫描中，维护两个变量：`cur_len`当前连续条纹长度，以及`best_len`迄今为止所见的最大连胜纪录。 还维护`cur_person`和`best_person`。 当当前人与前一个人匹配时，递增`cur_len`，否则将其重置为 1。这确保我们只计算连续的块。 
3. 每当`cur_len`超过`best_len`， 更新`best_len`并设置`best_person`给现在的人。 此步骤确定负责最长不间断活动段的人员。 
4. 扫描后，计算税额为`best_len * (m - 1)`在哪里`m`是不同人的数量。 该值减去`best_person`并等量添加（如`best_len`) 给其他所有人。 
5. 使用以下规则构建最终调整后余额：对于纳税人，减去全部税额； 对于其他每个人，添加`best_len`。 这种转换保持了总和的一致性。 
6. 最后将答案计算为`max(final_balance) - min(final_balance)`。 

其核心思想是序列结构只对识别单个显性条纹重要； 其他一切都是线性积累。 

### 为什么它有效

 该算法依赖于这样一个事实：两个所需的统计数据在单次传递中都是可前缀分解的。 余额累加是对元素的累加，而条纹计算则由相邻等式局部确定。 任何未来因素都不能追溯影响已完成的连胜，因此仅跟踪当前运行就足够了。 一旦记录了最大运行，重新分配步骤就是确定性的，并且仅取决于该单个值，从而使余额的转换完全确定，没有歧义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    arr = input().split()

    balance = {}
    for x in arr:
        balance[x] = balance.get(x, 0) + 1

    best_person = arr[0]
    best_len = 1
    cur_person = arr[0]
    cur_len = 1

    for i in range(1, n):
        if arr[i] == cur_person:
            cur_len += 1
        else:
            cur_person = arr[i]
            cur_len = 1

        if cur_len > best_len:
            best_len = cur_len
            best_person = cur_person

    m = len(balance)

    final = {}
    for p in balance:
        final[p] = balance[p]

    tax = best_len * (m - 1)

    for p in final:
        if p == best_person:
            final[p] -= tax
        else:
            final[p] += best_len

    vals = list(final.values())
    print(max(vals) - min(vals))

if __name__ == "__main__":
    solve()
```该解决方案分为三个阶段。 第一个循环构建频率图，它代表税前财务状况。 第二个循环通过维护运行的条纹计数器来隔离最长的连续段，并仅在出现严格更大的段时更新最佳可见值。 这避免了重新计算段边界。 

最后阶段对每人应用一次重新分配规则，从而保留线性复杂性。 所选人员的减法直接使用导出的税收公式，避免了对序列的任何模拟。 

这里的一个常见错误是更新`best_person`关系上。 该实现有意仅在以下情况下更新`cur_len > best_len`，这确保了第一个最大条纹的确定性选择。

 ## 边缘情况

 一种边缘情况是所有元素都相同。 用于输入`A A A A`，整个数组形成一个单一的条纹，所以`best_len = 4`和`best_person = A`。 该算法永远不会错误地重置，因为相等条件始终成立，并且没有引入人工分段。 

当最长的条纹位于数组的末尾时，会出现另一种边缘情况，例如`B A A A`。 仅当条纹增长超过先前的最大值后，更新条件才会触发，因此即使没有边界跟随，最终的段也会被正确捕获。 

第三种情况是多人拥有相同的最长连续记录，例如`A A B B`。 由于更新仅在严格改进时发生，因此保留了第一个最大条纹。 该行为仍然是确定性的，并且重新分配仅取决于单个选定的所有者，满足问题对唯一纳税实体的隐含要求。
