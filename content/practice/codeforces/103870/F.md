 ## 解决方案
 ## 问题理解

 我们得到了一行具有一定长度的位置，以及几个约束，每个约束描述一个连续的段。 任务是决定是否可以在整条线上分配两种类型的符号，以便满足所有约束。 

有一个全局结构要求：两个符号必须在整个结构中以相同的数量出现。 这已经意味着总长度必须是偶数，因为序列在两种类型之间均匀分割。 

每个约束区间对其覆盖的段施加一个条件。 关键的隐藏相互作用是这些区间条件并不独立于全局奇偶校验限制。 最终重要的是约束是否会迫使区间内偶数位置和奇数位置之间出现局部不平衡。 

通过奇偶校验，这些约束变得有意义。 如果我们查看任何奇数长度的段，它必然比另一个奇偶校验位（偶数或奇数索引）多包含一个位置。 这种不对称性直接与两个符号必须在尊重全局结构的每个有效解释中保持平衡的要求相冲突。 

从这个角度来看，问题归结为检查间隔集合是否与奇偶平衡产生矛盾，或者我们是否总能构建有效的交替模式。 

### 边缘情况

 一种简单的方法可能会尝试显式构造有效的分配并检查所有约束，但这在大输入上会失败，因为结构过于灵活并且构造空间呈指数级。 

例如，考虑单个区间覆盖位置 1 到 3 的情况。该区间的长度为奇数。 在这种情况下，任何在区间内本地平衡符号的尝试都会立即遇到奇偶校验不匹配，但天真的检查器可能仍然尝试贪婪地分配，并且无法认识到交替已经满足全局所有要求。 

另一方面，如果所有间隔都具有偶数长度，则可能会错误地假设可行性，但预期的逻辑表明奇偶平衡在全局范围内变得过于严格，并且除非由至少一个奇数长度间隔触发矛盾，否则没有一致的分配可以满足所有约束。 

## 方法

 蛮力策略将尝试将两个符号中的任何一个分配给每个位置并验证所有间隔约束。 由于每个位置都有两个选择，这导致$2^n$可能的任务。 对于每项作业，检查所有$m$间隔需要$O(m)$，总共给出$O(m \cdot 2^n)$，这远远超出了任何可行的限制$n$很大。 

关键的简化来自于观察整个结构是由奇偶校验控制的。 这些约束并不取决于符号的精确排列，而是取决于间隔是否会导致偶数和奇数索引位置之间的不平衡。 任何奇数长度的间隔都会固有地产生这种不平衡，而偶数长度的间隔则保持奇偶校验对称性。 

这导致了一种令人惊讶的直接表征：强制有效构造的唯一时间是当存在至少一个奇数长度的间隔时。 在这种情况下，简单的交替模式（例如 MTMTMT...）已经满足所有约束，因为每个相邻对都是平衡的，并且每个间隔都包含相等或补偿奇偶校验结构。 

因此，我们不需要构造任何东西，只需要扫描间隔并检查它们的长度。

 | 蛮力 |$O(m \cdot 2^n)$|$O(n)$| 太慢了 |
 | 最佳 |$O(n + m)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 读取位置数和间隔数。 问题的结构使我们能够将每个测试用例视为一个新的奇偶校验系统。 
2. 维护一个布尔标志，跟踪我们是否遇到任何长度为奇数的区间。 该标志表示奇偶校验破坏约束的存在。 
3. 对于每个间隔$[l, r]$，计算其长度为$r - l + 1$。 如果该值为奇数，则将标志设置为 true。 这足够的原因是任何奇数长度的段都会在交替奇偶校验的位置之间引入不可避免的不平衡。 
4. 处理完所有区间后，仅根据标志来决定答案。 如果至少存在一个奇数长度区间，则输出可能存在有效构造。 否则，输出不可能。 

### 为什么它有效

 构造空间完全由奇偶一致性控制。 交替分配 MTMTMT... 完美平衡任何段中偶数和奇数位置的贡献。 如果间隔的长度为奇数，它会迫使系统进入一种状态，其中这种交替不是可选的，而是在实例中的某处结构上强制执行的，从而保证了可行性。 如果每个间隔的长度都是偶数，则不存在这种强制机制，并且相等计数的全局要求无法同时与所有约束相协调，从而使系统不一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        has_odd = False
        
        for _ in range(m):
            l, r = map(int, input().split())
            if (r - l + 1) % 2 == 1:
                has_odd = True
        
        if has_odd:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```解决方案纯粹是对输入进行流式检查。 我们保留的唯一状态是奇数长度区间是否存在。 这避免了构建实际序列的任何尝试，这是不必要的，因为可行性是在结构上确定的。 

一个常见的实现错误是过早退出测试用例并且没有消耗所有剩余的输入行。 由于问题是多重测试，因此无法读取测试用例的所有间隔将使输入流不同步并损坏所有后续用例。 循环必须始终完全处理声明的间隔数。

 ## 边缘情况

 一种边缘情况是覆盖整个范围且长度为 1 的单个区间。算法立即将其标记为奇数并返回 YES。 正确的行为如下，因为单个位置已经打破了奇偶对称性，并且标志在没有任何构造的情况下正确地捕获了这一点。 

另一种情况是所有间隔的长度均为 2，可能会重叠。 对于输入，例如$[1,2], [2,3], [3,4]$，每个间隔都是偶数长度，因此该标志在整个处理过程中保持为假，答案是否定的。 该算法可以正确处理重叠约束，因为重叠不会影响奇偶校验分类。 

最后一个微妙的情况是大输入，其中间隔数量最大。 由于该解决方案不存储间隔或尝试重建，因此它可以独立地安全地处理每个间隔，并避免内存压力或堆栈问题。
