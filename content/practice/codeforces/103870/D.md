 ## 解决方案
 ## 问题理解

 给定一个 n 行 m 列的矩形整数网格。 每个单元格贡献一个值，我们可以使用函数 f(a, b, c, d) 计算任何子矩形的总和，这意味着对 a 到 b 行和 c 到 d 列中的所有单元格求和。 

网格中只有一个特殊单元格位于 (x, y) 位置。 声明中将其描述为负数，但该单元格的关键作用是允许我们考虑通过沿行或列分割网格来消除其影响。 

任务是计算五个特定矩形和中的最大值：整个网格的总和、x行上方顶部部分的总和、x行下方底部部分的总和、y列之前左侧部分的总和以及y列之后右侧部分的总和。 

换句话说，我们不是组合区域，而是选择这五个候选矩形之一并取其中的最佳总和。 

输入大小意味着一个 n × m 的网格。 如果我们重复对子矩阵求和，从头开始直接计算每个矩形会太慢。 需要关注的自然约束是 n 和 m 可以足够大，以至于需要 O(nm) 预处理，但任何每个查询的重新计算都必须是 O(1) 或非常接近 O(1)。 

常见的失败案例来自于对允许区域的几何形状的误解。 允许的区域不是任意子矩阵，它们是水平或垂直分割的整个网格的一半。 

例如，如果错误地尝试同时删除行和列并合并区域，他们可能会错误地计算诸如左上加右下之类的内容，这不是允许的集合的一部分。 该问题仅允许单个连续矩形与全宽或全高对齐。 

## 方法

 强力解释将通过对所有包含的单元格求和来直接计算五个候选矩形中的每一个。 对于每个查询，成本为 O(nm)，因为每个矩形可以与整个网格一样大。 对于五个候选者，每次评估的时间复杂度为 O(nm)，但仍然是 O(nm)，但如果存在多个测试用例或者我们重复重新计算总和，则总体上会变得太慢。 

关键的观察结果是所有候选值都是类似前缀或后缀的全宽或全高子矩阵。 这种结构允许我们预先计算网格上的 2D 前缀和。 一旦前缀和可用，任何矩形和 f(a, b, c, d) 都可以在 O(1) 中计算。 

然后，问题简化为评估五个固定公式，每个公式描述一个大的轴对齐矩形。 前缀和将线性扫描转换为恒定时间查找，使得预处理后每个候选的解决方案实际上为 O(1)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 每个查询 O(nm) | O(1) | O(1) | 太慢了 |
 | 前缀和| O(nm) 预处理，O(1) 查询 | O(纳米) | 已接受 |

 ## 算法演练

1. 读取网格并将所有值存储在二维数组中。 问题的结构要求我们支持快速矩形和查询，因此我们准备预处理而不是直接计算。 
2. 构建一个二维前缀和数组 pref，其中 pref[i][j] 存储子矩阵中从 (1,1) 到 (i,j) 的所有值的总和。 此步骤至关重要，因为它使用包含-排除将任何矩形和转换为恒定时间公式。 
3. 定义一个辅助函数来使用前缀和计算 f(a, b, c, d)。 该函数计算 pref[b][d] 减去排除的顶部、左侧和重叠区域，使我们能够在 O(1) 中检索任何矩形总和。 
4. 使用 f(1, n, 1, m) 计算整个网格的总和。 这是五个候选者之一，代表了我们不排除任何内容的情况。 
5. 将顶部部分的总和计算为 f(1, x−1, 1, m)。 这对应于仅将行严格保留在特殊行之上。 
6. 计算底部部分的总和为 f(x+1, n, 1, m)。 这对应于仅将行严格保留在特殊行下方。 
7. 计算左侧部分的和为 f(1, n, 1, y−1)。 这对应于仅保留特殊列严格左侧的列。 
8. 计算右侧部分的和为 f(1, n, y+1, m)。 这对应于仅保留特殊列严格右侧的列。 
9. 返回这五个值中的最大值。 每个候选者都代表一个有效的允许配置，我们正在其中选择最好的。 

工作原理：每个允许的配置都是一个轴对齐的矩形，跨越网格的整个宽度或整个高度，通过沿 x 行或 y 列最多切割一次来确定。 前缀和保证每个这样的矩形和被精确计算一次，没有重叠或遗漏，并且由于我们明确地评估所有有效候选者，所以不会错过更好的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(n)]
    x, y = map(int, input().split())

    pref = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        row_sum = 0
        for j in range(1, m + 1):
            row_sum += grid[i - 1][j - 1]
            pref[i][j] = pref[i - 1][j] + row_sum

    def rect(a, b, c, d):
        if a > b or c > d:
            return 0
        return (
            pref[b][d]
            - pref[a - 1][d]
            - pref[b][c - 1]
            + pref[a - 1][c - 1]
        )

    ans = rect(1, n, 1, m)
    ans = max(ans, rect(1, x - 1, 1, m))
    ans = max(ans, rect(x + 1, n, 1, m))
    ans = max(ans, rect(1, n, 1, y - 1))
    ans = max(ans, rect(1, n, y + 1, m))

    print(ans)

if __name__ == "__main__":
    solve()
```实现以二维前缀和表为中心。 每行首先累积为运行行总和，然后使用前一行的值合并到前缀结构中。 这避免了重复求和并确保O(nm)预处理。 

rect 函数是标准的包含-排除计算。 对无效范围的边界检查是必要的，因为像 x−1 或 y+1 这样的切片可以生成空矩形，这些矩形必须贡献零而不是破坏前缀索引。 

使用此助手直接评估每个候选矩形，并增量地维护最大值。

 ## 边缘情况

 一种边缘情况是当 x 或 y 位于边界上时，导致某些候选矩形为空。 例如：

 输入：```
2 3
1 2 3
4 -5 6
1 2
```这里 x = 1，因此顶部矩形 (1, x−1, 1, m) 无效。 对于这种情况，算法返回 0，这确保它不会错误地影响最大值。 其余候选者正常计算，最终答案仅取自有效区域。 

另一种情况发生在网格非常小时，例如 1 行或 1 列。 在这种情况下，多个候选者会折叠成相同或空的矩形。 前缀和逻辑仍然适用，因为 rect 函数可以安全地处理超出范围的间隔，确保没有索引错误并保持正确性。
