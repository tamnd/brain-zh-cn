 ## 解决方案
 ## 问题理解

 我们正在使用非闰年的简化日历，其中一年有 365 天，每天可以包含一个事件或为空。 输入最终描述了哪些特定日期有事件，其他所有内容都隐式为空。 

任务是确定没有任何事件的最长连续几天。 换句话说，一旦我们标记了所有事件日，我们就需要连续几天的最大长度，其中没有任何标记的日子出现。 

思考这个问题的一个有用方法是想象一个长度为 365 的数组，其中每个位置对应于一年中的一天。 值 1 表示当天有活动发生，0 表示当天空闲。 目标是找到最长的连续零段。 

由于数组长度固定为 365，所以日历大小方面的约束非常小。即使有很多事件描述，总的模拟也会受到这个恒定大小的结构的限制。 这立即排除了对大输入的对数或线性优化的任何担忧。 即使是重复扫描全年的简单模拟也足够快。 

主要的微妙之处在于将事件描述正确地转换为每日标记。 幼稚的方法可能会尝试将事件视为直接索引，而不正确翻译日历日期或重复间隔，这可能会导致日期标记不正确。 

一些边缘情况在这里很重要。 一种是根本没有事件发生的时候。 在这种情况下，全年都是免费的，答案应该是 365。粗心的实现假设至少有一个事件可能会错误地返回 0。另一种情况是事件涵盖每天。 那么最长的空闲段是 0，差一错误通常会错误地返回 1。当事件生成使用重复步进，并且步进超出或与日期索引不对齐时，会出现第三种情况，这可能会默默地跳过标记边界日期，例如第 365 天。 

## 方法

 暴力破解的想法始于这样的观察：一旦我们知道哪些天被占用，我们只需要扫描 365 天的数组并计算连续的零。 这部分在孤立时已经是最佳的，因为它在一个非常小的常数中是线性的。 

更困难的部分是构建活动日数组。 每个事件的定义方式可以通过重复添加固定步长 C 直到经过第 365 天来生成多个受影响的日期。对于每个这样的序列，我们在日历中标记相应的位置。 

一个简单的构造是，对于每个事件源，重复增加日期并直接标记数组。 这已经是正确的并且仍然足够高效，因为每个序列最多有 365/C 次迭代，并且更新总数以事件源数量的 365 倍为界。 所有标记完成后，单次扫描会找到最长的零串。 

关键的见解是我们不需要任何高级数据结构或区间合并。 日历是固定的且很小，因此直接模拟主宰一切。 重复算术级数更新的结构与布尔标记数组完美契合。 

蛮力之所以有效，是因为每个事件只影响一组可预测的离散日期，但如果我们尝试过早优化，它在概念上就会变得混乱。 通过观察域只有 365 天，我们可以将所有事情都视为直接模拟问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（直接模拟）| O(365·N) | 奥(365) | 已接受 |
 | 最佳（相同模拟，结构化扫描）| O(365·N) | 奥(365) | 已接受 |

 ## 算法演练

 ### 1.初始化日历数组

 我们创建一个数组`v`大小为 366（基于 1 的索引很方便），初始化为零。 每个索引代表当天是否发生事件。 

这给我们提供了一种直接的方式来标记和查询每天的状态。 

### 2.解析每个事件生成器

 对于每个事件描述，我们从某个初始日开始`d`并重复添加步长`C`。 每次我们在有效日期（≤ 365）着陆时，我们都会标记`v[d] = 1`。 

我们继续这个过程，直到这一天超过 365。 

步进而不是计算封闭形式范围的原因是每个事件定义离散算术级数，而不是连续间隔。 

### 3. 构建完整的活动日历

 处理完所有事件生成器后，每天至少有一个事件将被标记为 1`v`。 没有任何事件的天数仍为 0。 

这种聚合很重要，因为多个生成器可能标记同一天，但我们只关心是否至少存在一个事件。 

### 4. 扫描最长的连续零

 我们从第 1 天到第 365 天遍历数组，维护一个正在运行的计数器`cur`对于目前连续空虚的日子。 

如果`v[i] == 0`，我们增加`cur`并更新答案，如果`cur`变得更大。 

如果`v[i] == 1`，我们重置`cur`为 0，但我们不重置全局最大值。 

当前连续记录和全局最大值之间的这种分离确保了正确性。 

### 为什么它有效

 在日历中的每个位置，我们都会对当天是否被占用进行正确的分类。 然后，扫描将问题简化为查找二进制数组中最长的连续段，这可以通过维护不变式来解决：`cur`始终等于以当前索引结尾的连续零值条目的数量。 由于每天只处理一次，并且标记步骤完全捕获所有事件发生，因此不会丢失任何片段，也不会发生片段的无效扩展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    v = [0] * 366  # 1..365
    
    for _ in range(n):
        d, c = map(int, input().split())
        
        while d <= 365:
            v[d] = 1
            d += c
    
    ans = 0
    cur = 0
    
    for i in range(1, 366):
        if v[i] == 0:
            cur += 1
            if cur > ans:
                ans = cur
        else:
            cur = 0
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案分为两个阶段：首先，我们通过标记算术级数生成的所有事件日来构建日历。 while 循环确保我们只停留在一年的范围内。 第二阶段是单次线性扫描，计算未标记天数的最长连续时间。 

一个常见的实施陷阱是在遇到事件日时忘记重置当前连续记录，这会错误地合并单独的空闲间隔。 另一个问题是对基于 1 的天数索引处理不当，如果使用基于 0 的数组而不进行调整，可能会将所有结果移一。

 ## 边缘情况

 一种重要的边缘情况是没有事件生成器。 在这种情况下，日历仍然完全由零填充。 扫描从第一天开始，不断递增`cur`，并且从不重置它，产生最终答案 365。这证实了该算法并不依赖于遇到至少一个事件来正确运行。 

当每天被标记为事件时，会出现另一种边缘情况。 例如，如果生成器从 1 开始通过步骤 1 覆盖所有天，则中的每个位置`v`变为 1。在扫描过程中，`cur`每一步都会重置，累积的值不会超过 0，因此最终答案为 0。这验证了连续重置是否已正确处理，而不会意外保留部分状态。 

最后一个微妙的情况是事件序列恰好在第 365 天结束。 while 循环条件`d <= 365`确保在到达时包含第 365 天。 例如，从 365 开始，步骤 1 仅标记第 365 天。然后，扫描会正确地将第 365 天视为阻塞时间，并且不会将任何空闲间隔延长到该天之后。
