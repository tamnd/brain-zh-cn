 ## 解决方案
 ## 问题理解

 我们在几堆上玩了一场外卖游戏。 

一次移动包括选择一堆并移除数量等于 2 的幂的石子。 只要所选堆包含足够的石子，允许的移除次数为 1、2、4、8 等。 朱莉娅先走一步，不能移动的玩家就输了。 

比赛开始前，乔瓦娜必须准确表现`X`运营。 在一次操作中，她选择了一堆并将其大小增加了一倍。 在这些强制性的加倍之后，两名球员都表现得非常完美，我们必须决定谁获胜。 

输入包含堆的数量、所需的加倍数量以及初始堆大小。 输出是获胜者的名字。 

约束条件很大：最多`10^5`桩数和桩尺寸可达`10^9`， 尽管`X`可以大到`10^9`。 任何单独模拟移动或加倍的解决方案都是不可能的。 我们需要一个`O(N)`或者`O(N log N)`解决方案。 

棘手的部分是`X`可能是巨大的。 如果我们能够理解加倍如何影响游戏的 Grundy 值，那么所有加倍后的堆的实际大小就无关紧要了。 整个问题变成了博弈论问题。 

一个常见的错误是分析加倍后的确切堆尺寸。 例如：```
N = 1
X = 1000000000
a = [1]
```试图在翻倍十亿之后追踪桩的价值是毫无意义的。 游戏结果仅取决于格伦迪数字，并且这些数字遵循一个非常简单的模式。 

另一个容易犯的错误是假设每次加倍总是改变位置。 如果堆大小可被 3 整除，则将其加倍可保持相同的 Grundy 值。 例如：```
N = 1
X = 1
a = [3]
```该堆的格伦迪数在加倍后保持不变，因此乔瓦娜可以有效地花费一招而不影响游戏状态。 

当初始 Nim xor 已经为零时，会出现最后一个微妙的情况。 乔瓦娜希望将其保持为零，但她被迫精确执行`X`加倍。 她能否“浪费”这些加倍决定了答案。 

## 方法

 蛮力视图是显式计算每个桩大小的 Grundy 数。 对于一堆大小`v`，我们考虑每个可达状态`v - 2^k`，计算他们的 Grundy 数，并取 mex。 

这适用于小值，如果我们计算我们获得的前几个状态：

 | 桩尺寸| 格伦迪 |
 | ---| ---|
 | 0 | 0 |
 | 1 | 1 |
 | 2 | 2 |
 | 3 | 0 |
 | 4 | 1 |
 | 5 | 2 |
 | 6 | 0 |

 一个模式立即出现：格伦迪数似乎是`v mod 3`。 

通过计算进行强力证明不足以满足实际约束，因为桩尺寸达到`10^9`。 我们需要数学表征。 

关键的观察结果是，2 的每个幂都等于 1 或 2 模 3：```
2^0 ≡ 1 (mod 3)
2^1 ≡ 2 (mod 3)
2^2 ≡ 1 (mod 3)
2^3 ≡ 2 (mod 3)
...
```所以从一堆残留物中搬走`k`模 3 总是到达另外两个余数之一，但永远不会到达同一个。 

假设一堆有残留物`k`。 

一举一动都到达残留`{0,1,2} \ {k}`。 

如果我们假设较小位置的 Grundy 值恰好是它们的余数模 3，则可达 Grundy 数的集合也是`{0,1,2} \ {k}`。 该集合的 mex 是`k`。 

这证明：```
Grundy(v) = v mod 3
```现在游戏变成了 Nim。 所有残基的异或决定获胜者。 

剩下的挑战是理解加倍运算的效果。 

对于一堆残留物：```
0 -> 0
1 -> 2
2 -> 1
```将 1 和 2 加倍，而 0 保持不变。 

这将整个问题简化为关于残基 1 和 2 的计数的奇偶校验参数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 Grundy 计算 | O(max(ai) log max(ai)) 或更糟 | O(最大(ai)) | 太慢了 |
 | 余数模3博弈分析| O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算余数`ai mod 3`对于每一堆。 
2. 计算 Nim 异或`S`所有残留物。 
3. 计算每个残基类别有多少堆：`freq[0]`,`freq[1]`， 和`freq[2]`。 
4. 如果`X = 0`，不进行任何预处理。 该位置是标准 Nim 位置。 

朱莉娅开始，所以她赢了 iff`S != 0`。 否则乔瓦娜获胜。 
5. 如果`X > 0`和`S = 0`，Giovana 已经喜欢这个位置了，因为第一个玩家正在输。 

她希望所有强制加倍都保持异或不变。 
6. 一堆残基为 0 的东西可以吸收任意数量的加倍，因为`0 -> 0`。 

如果存在这样的一堆，乔瓦纳可以在那里进行所有所需的操作。 
7. 如果不存在残基 0 堆，则每次加倍都会将残基 1 与残基 2 交换，反之亦然。 

在同一堆上涂抹两次可恢复其原始残留物。 

因此，只有当`X`是均匀的。 
8.如果`X > 0`和`S != 0`，Giovana 想要将位置转换为异或零。 
9. 将一堆残基 1 与残基 2 交换会改变两个残基计数的奇偶性。 

当两者都时，异或恰好为零`freq[1]`和`freq[2]`很奇怪。 
10. 一旦达到异或零，任何额外的加倍都必须被丢弃。 

这是可能的，如果`X`是奇数，或者是否存在剩余 0 堆来吸收一个额外的操作。 
11. 输出`"Giovana"`当她可以在比赛开始前强制 Julia 处于失败位置时，否则输出`"Julia"`。 

### 为什么它有效

 基本的不变量是每个桩的行为都与 Nim 桩完全相同，其大小为`ai mod 3`。 证明来自 Grundy 递归：每一步都会以 1 或 2 模 3 改变残差，使得所有其他残差类别可达，而当前残差类别不可到达。 因此，mex 是当前的残差。 

加倍也不取决于实际的堆大小。 它仅影响残数模 3，保留残数 0 固定并交换残数 1 和 2。一旦游戏简化为这三个残数类别，只有残数 1 和 2 的计数奇偶性才重要。 编辑条件正是 Giovana 可以使用她的强制加倍来获得零的 Nim xor 并将其保持在那里的情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, x = map(int, input().split())
a = list(map(int, input().split()))

freq = [0, 0, 0]
s = 0

for v in a:
    r = v % 3
    freq[r] += 1
    s ^= r

if x == 0:
    win = (s == 0)
else:
    if s == 0:
        win = (freq[0] > 0 or x % 2 == 0)
    else:
        win = (
            freq[1] % 2 == 1 and
            freq[2] % 2 == 1 and
            (x % 2 == 1 or freq[0] > 0)
        )

print("Giovana" if win else "Julia")
```第一个循环计算游戏所需的一切：Nim 异或和模 3 的残数频率。 

的`x == 0`分支是普通的 Nim。 零异或意味着起始玩家失败，因此乔瓦纳获胜，因为朱莉娅先移动。 

为了`x > 0`，逻辑直接遵循残差变换规则。 条件`freq[0] > 0`特别重要，因为残基 0 堆是在不改变位置的情况下花费奇数额外加倍的唯一方法。 

所有算术都可以轻松地适应 32 位整数，但无论如何，Python 自然可以处理更大的值。 没有堆被明确加倍，这避免了巨大界限的问题`X`。 

## 工作示例

 ### 示例 1

 输入：```
4 1
5 1 3 2
```残留物为：```
[2, 1, 0, 2]
```| 桩| 残留物| 运行异或 |
 | ---| ---| ---|
 | 5 | 2 | 2 |
 | 1 | 1 | 3 |
 | 3 | 0 | 3 |
 | 2 | 2 | 1 |

 最终值：```
S = 1
freq[0] = 1
freq[1] = 1
freq[2] = 2
```自从`S != 0`，残基 1 和残基 2 计数都必须为奇数。 这里`freq[2]`是偶数，因此 Giovana 无法使异或变为零。 

优胜者：`Julia`。 

此示例表明，仅拥有残基 0 堆是不够的。 残基 1 和 2 的奇偶校验条件也必须成立。 

### 示例 2

 输入：```
3 2
1 2 3
```残留物：```
[1, 2, 0]
```| 桩| 残留物| 运行异或 |
 | ---| ---| ---|
 | 1 | 1 | 1 |
 | 2 | 2 | 3 |
 | 3 | 0 | 3 |

 最终值：```
S = 3
freq[0] = 1
freq[1] = 1
freq[2] = 1
```两个非零残留计数都是奇数。 存在残差 0 堆，因此乔瓦纳可以调整位置并放弃任何额外的操作。 

优胜者：`Giovana`。 

此示例演示了残基 0 堆作为强制加倍缓冲区的作用。

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N) | 一次通过所有桩 |
 | 空间| O(1) | O(1) | 仅三个频率计数器和一个异或值 |

 该解决方案只处理每个堆一次，并且从不依赖于堆的大小`ai`或者`X`。 和`N ≤ 10^5`，这很容易符合限制。

 ## 边缘情况

 考虑：```
1 1
3
```堆剩余为 0。初始异或也为 0。Giovana 已经对这个位置感到满意，并且可以在该堆上花费所需的双倍，因为`0 -> 0`。 异或仍然为零，Julia 从失败的 Nim 位置开始。 该算法检测到`freq[0] > 0`和输出`Giovana`。 

现在考虑：```
2 1
1 2
```残渣是`[1,2]`。 异或已经为零。 没有残留0堆。 一次强制加倍掉期`1 -> 2`或者`2 -> 1`，将异或更改为远离零。 自从`X`奇怪的是，乔瓦娜无法取消该效果。 算法达到条件`freq[0] == 0`和`X % 2 == 1`，生产`Julia`。 

最后：```
3 2
1 2 3
```残渣是`[1,2,0]`。 异或非零，并且两个非零残差计数均为奇数。 Giovana 可以使用一次加倍来使异或为零，并使用残差 0 堆来吸收剩余的所需操作。 该算法满足所有条件`S != 0`分支和输出`Giovana`。
