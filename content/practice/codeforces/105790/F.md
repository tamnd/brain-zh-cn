 ## 解决方案
 ## 问题理解

 埃里克从能量最初为零的激光武器开始。 

对于每个级别，有两种可能的操作。 

正常路线需要杀掉`s_i`突变体。 每次射击，即使是非正能量，也会消耗一个单位的能量，因此通过关卡会改变当前的能量`e_i - s_i`。 之后，埃里克进入了一个新的水平。 

另一条路线是虫洞。 使用它会跳过`k`水平领先。 为了进入虫洞，埃里克必须杀死`X`守护突变体，所以他的能量减少了`X`。 他不与`s_i`常规突变体并且不接收能量库`e_i`。 

完成所有关卡后，埃里克到达了光之女王。 仅当武器能量严格为正时，射击才会对 Boss 造成伤害。 如果埃里克带着能量到来`E`，他可以准确处理`max(E, 0)`在能量变为负值之前损坏。 

任务是在到达boss时最大化剩余的能量。 答案是如果为正则能量最大，否则`0`。 

约束是关键的观察结果。 最多有`2 · 10^5`水平。 任何明确探索所有可能路径的解决方案都是不可能的，因为每个级别都提供两种选择，大致产生`2^N`路径。 甚至一个`O(N^2)`动态程序需要大约`4 · 10^10`最坏情况下的操作，远远超出了极限。 我们需要一些线性或接近线性的东西。 

当每条可能的路径都以非正能量结束时，就会出现微妙的边缘情况。 

例子：```
1 1 5
3 0
```正常路线提供能量`-3`，虫洞提供能量`-5`。 埃里克到达了 Boss，但无法造成任何伤害。 正确答案是：```
0
```不小心的实施可能会输出最大可达能量，`-3`，而不是将负值转换为零。 

当虫洞跳跃到最后一层时，就会出现另一种边缘情况。 

例子：```
2 2 1
10 100
10 100
```使用1级虫洞立即到达boss。 目的地必须被视为“所有级别之后”，而不是无效状态。 缺少此细节通常会导致越界错误或丢失转换。 

第三种边缘情况是当`X = 0`。 

例子：```
3 1 0
5 0
5 0
5 0
```虫洞变成了自由跳跃。 该算法仍然必须考虑它，因为避免大负值`(e_i - s_i)`值可以是最优的。 

## 方法

 蛮力的想法很简单。 在每个级别，选择正常路线或虫洞路线，递归地探索所有可能的未来。 由于每个级别都有两个选择，因此路径数量呈指数级增长。 和`N = 200000`，这是完全不可行的。 

有趣的是，当前的能量是唯一重要的数量。 未来并不取决于我们如何达到某个水平，只取决于那里可用的最大能量。 

认为`dp[i]`表示完成第一个任务后可达到的最大能量`i`水平，或同等地，当站在水平的起点时`i + 1`。 

来自州`i`，只有两个转变。 

走正常路线通过关卡`i + 1`改变能量`e_i - s_i`。 

使用虫洞改变能量`-X`并直接移动到水平`i + k`。 

由于两种转变都只是将固​​定值添加到当前能量中，因此仅保留每个状态的最佳能量就足够了。 在同一位置上任何较小的能量永远不会带来更好的未来结果。 

这将指数搜索转换为位置上的线性动态程序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^N) | O(2^N) | O(N) | 太慢了|
 | 最佳DP | O(N) | O(N) | 已接受 |

 ## 算法演练

 1. 创建数组`dp`长度`N + 1`，初始化为负无穷大。 
2. 设置`dp[0] = 0`，代表进入第一级之前的起始状态。 
3. 从左到右处理级别。 
4. 来自州`i`，尝试正常路线。 

能量变化为`(e_i - s_i)`，所以更新：`dp[i + 1] = max(dp[i + 1], dp[i] + e_i - s_i)`。 
5. 来自州`i`，尝试虫洞路线。 

目的地是`min(N, i + k)`因为跳过最后一关就意味着立即到达boss。 

更新：`dp[min(N, i + k)] = max(dp[min(N, i + k)], dp[i] - X)`。 
6. 处理完所有转换后，`dp[N]`是到达 Boss 后可获得的最大能量。 
7. 输出`max(dp[N], 0)`因为负能量或零能量无法伤害 Boss。 

### 为什么它有效

 不变的是`dp[i]`总是在所有到达状态的方式中存储最大可能的能量`i`。 

状态的每一项法律行动都对应于一次 DP 转换。 由于每次跃迁都会向当前能量添加固定量，因此同一状态下较大的能量总是至少与较小的能量一样好。 未来的任何决定都不可能让较弱的国家变得更好。 

通过处理所有状态并取所有传入转换的最大值，DP 计算每个位置的最佳可实现能量。 尤其，`dp[N]`成为到达光之女王时可获得的最大能量。 Boss只能在能量为正时受到伤害，因此最大可能伤害为`max(dp[N], 0)`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, x = map(int, input().split())

    s = [0] * n
    e = [0] * n

    for i in range(n):
        s[i], e[i] = map(int, input().split())

    NEG = -(10 ** 30)
    dp = [NEG] * (n + 1)
    dp[0] = 0

    for i in range(n):
        if dp[i] == NEG:
            continue

        dp[i + 1] = max(dp[i + 1], dp[i] + e[i] - s[i])

        nxt = min(n, i + k)
        dp[nxt] = max(dp[nxt], dp[i] - x)

    print(max(dp[n], 0))

solve()
```DP数组使用索引`i`表示处理第一个之后的状态`i`水平。 

正常的路由转换添加`e[i] - s[i]`，匹配通过战斗关卡获得的净能量并收集其能量库。 

虫洞跃迁跳转到`min(n, i + k)`。 这是最容易犯错误的地方。 一旦跳跃超出了最后一个级别，游戏就会立即到达 Boss，因此每次这样的跳跃都应该落在状态中`n`。 

取值范围大致可以达到`2 · 10^5 × 10^9`，因此需要 64 位算术。 Python 整数会自动处理这个问题。 

最终答案不是`dp[n]`本身。 负能量无法损坏 Boss，因此结果必须钳制为零。

 ## 边缘情况

 考虑所有能量都是非正的情况。```
1 1 5
3 0
```初始化给出`dp[0] = 0`。 

正常路线：```
dp[1] = -3
```虫洞：```
dp[1] = max(-3, -5) = -3
```最佳可达能量为`-3`。 由于负能量无法伤害 Boss，因此算法输出：```
0
```现在考虑一下超越终点的跳跃。```
2 2 1
10 100
10 100
```在状态`0`:```
normal -> dp[1] = 90
wormhole -> dp[2] = -1
```在状态`1`:```
normal -> dp[2] = 180
wormhole -> dp[2] = max(180, 89)
```虫洞的目的地是状态`2`, 不声明`3`，因为超越最后一关就意味着到达了boss。 最终答案变成：```
180
```最后，考虑一下自由虫洞。```
3 1 0
5 0
5 0
5 0
```每条正常路线都会损失 5 点能量。 每个虫洞损失的能量为 0。 

DP反复选择虫洞过渡并用能量到达boss`0`。 答案依然是：```
0
```这证实了该算法正确地比较了每个级别的两个选择，并且始终保持最佳的可达能量。
