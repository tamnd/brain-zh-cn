 ## 解决方案
 ## 问题理解

 我们有一系列恒星亮度值。 我们必须将数组分成连续的组。 组的美妙之处在于该组内所有值的按位或。 

对于数组的一个分区，我们计算所有组的美度之和。 在所有可能的分区中，我们想要最大可能的总和。 如果多个分区达到最大总和，我们必须输出使用最少组的分区。 

输入包含单个长度的数组`N`， 在哪里`N`可以大到`3·10^5`，并且每个值都小于`2^30`。 输出由两个数字组成：`S`= 团体美女的最大可能总和。`K`= 实现的所有分区中的最小组数`S`。 

尺寸为`N`立即排除任何检查所有子数组的动态编程。 大致有`N²/2`子数组，当`N = 300000`。 甚至一个`O(N log N)`解决方案会很舒服，同时`O(N²)`是完全不可能的。 

关键的困难是理解当多个元素合并为一组时按位或的行为。 

考虑一个简单的例子：```
2
1 2
```隔断`[1][2]`给出美丽总和`1 + 2 = 3`。 

隔断`[1,2]`赋予美丽`1|2 = 3`。 

最大金额仍为`3`，但第二个分区使用的组较少，所以答案是：```
3 1
```一个粗心的解决方案总是在可能的情况下启动一个新组，这会返回`K = 2`，这不是最小的。 

另一个有趣的案例是：```
2
1 1
```单独的组给予`1 + 1 = 2`。 

一个合并组给出`1|1 = 1`。 

合并后的版本会失去价值，因为同一位在 OR 中仅计算一次。 正确答案是：```
2 2
```任何在 OR 不减少时合并组的解决方案都会在这里失败。 

最后的边缘情况是零的存在：```
3
0 5 0
```隔断`[0,5,0]`有美丽`5`。 

隔断`[0][5][0]`还有美和`5`。 

由于两者达到相同的最大总和，因此我们必须选择组数较少的一组，即单个组。 

正确处理这些领带情况至关重要。 

## 方法

 让我们首先考虑一下当多个元素放入一组时会发生什么。 

假设一个组包含 OR 为`X`。 

组中任何位置出现的每个位都只贡献一次`X`。 

现在将其与将所有元素分开进行比较。 如果某个特定位出现在多个元素中，那么当分开时，它会对总和贡献多次，但在一个 OR 中，它只贡献一次。 

这立即给出了一个重要的观察：

 对于任何群体来说，```
OR(group) ≤ sum(elements of group)
```因为当应用 OR 时，每个位只会失去多重性。 

对所有组进行求和，总的美感永远不会超过所有数组元素的总和。 

当任何组内没有任何位被计数两次时，美丽总和恰好达到这个上限。 

这意味着在每个组内，任何位位置都不能出现在两个不同的元素中。 

同样，对于同一组中的每对元素：```
their common set bits must be empty
```或者，更实际的是，在构建组时，当前的 OR 掩码和下一个值必须满足：```
(current_mask & value) == 0
```如果此条件成立，则添加该值只会引入新的位，并且该组的 OR 值正好增加该值本身。 

蛮力的想法是尝试所有分区并检查它们的美丽值。 由于有`2^(N-1)`可能的分区，它立即变得不可行。 

上面的观察完全改变了问题。 最大可能的美丽总和始终是所有数组元素的总和。 剩下的唯一任务是找到能够达到该总和的最少组数。 

为了获得最大总和，每个组必须包含成对不相交的位。 从左到右扫描时，每当下一个元素与当前组的 OR 掩码共享一点时，将其保留在同一组中会将总体美感降低到最大值以下。 因此，一个新的组就成为强制性的。 

另一方面，如果下一个元素与当前组的蒙版没有共同位，则可以自由合并：仍然可以实现最大的美感，并且使用相同的组有助于最大限度地减少组的总数。 

这自然会导致贪婪扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解分区 | O(2^N) | O(2^N) | O(N) | 太慢了 |
 | 贪婪位掩码扫描 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 初始化`total_sum`作为所有数组值的总和。 
2. 开始第一组。 维持`mask`，当前组内所有元素的按位或。 
3. 从左到右处理数组。 
4. 对于当前值`x`，检查是否`mask & x`是非零的。 
5. 如果交集非零，则当前组中已存在某个位。 保持`x`在该组中将导致该位在 OR 内仅被计数一次，从而降低了可实现的最大美感。 开始一个新组并设置`mask = x`。 
6. 如果交集为零，则所有位`x`是当前组的新成员。 添加到当前组并更新`mask |= x`。 
7. 计算新组启动的次数。 这会产生仍然达到最大美感总和的最少组数。 
8. 输出`total_sum`和组数。 

### 为什么它有效

 所有数组元素的总和是答案的绝对上限，因为组的 OR 永远不会超过其元素的总和。 

当任何组内没有位位置出现两次时，分区恰好达到此上限。 每当`mask & x ≠ 0`, 放置`x`进入当前组将创建这样一个重复位并迫使美丽总和低于上限。 削减是强制性的。 

每当`mask & x = 0`, 合并`x`进入当前组仅引入新位，因此该组的 OR 恰好增加`x`。 美丽不会丢失，避免切割会减少组的数量。 

贪婪策略只有在不可避免的情况下才会进行削减。 它创建的每个切割都强制在每个最佳分区中。 因此，它实现了最大美丽总和，并在实现该总和的所有分区中使用尽可能少的组数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    total_sum = sum(a)

    groups = 1
    mask = 0

    for x in a:
        if mask & x:
            groups += 1
            mask = x
        else:
            mask |= x

    print(total_sum, groups)

solve()
```第一部分计算所有元素的总和。 这个值已经是最大可能的美丽总和了。 

变量`mask`存储当前组的或。 它准确地总结了已经存在的位位置。 

对于每个元素`x`，表达式`mask & x`检查某个位是否会在同一组中出现两次。 如果发生这种情况，必须立即开始一个新的小组。 

当不存在公共位时，我们通过更新来扩展当前组`mask |= x`。 

组计数器从 1 开始，因为即使单元素数组也包含一组。 

该实现仅使用常量额外内存并执行一次遍历数组。

 ## 边缘情况

 考虑：```
2
1 1
```该算法开始于`mask = 0`。 

处理完第一条后`1`,`mask = 1`。 

对于第二个`1`，我们得到：```
mask & x = 1
```这样就创建了一个新组。 结果是：```
2 2
```这是正确的，因为合并它们会产生 OR 值`1`，减少美感总和。 

现在考虑：```
3
0 5 0
```第一个零不创建任何位。 

价值`5`与当前掩码不冲突。 

最后的零也没有冲突。 

没有进行任何削减，给出：```
5 1
```美丽仍然最大化，团体数量最小化。 

最后：```
4
7 7 7 7
```每个新元素都与当前掩码冲突，因为所有位都已存在。 

该算法在每个后续元素之前开始一个新组，产生：```
28 4
```每次剪切都是强制性的，因为任何合并都会将美丽总和降低到元素总和以下。
