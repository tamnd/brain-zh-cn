 ## 解决方案
 ## 问题理解

 一艘宇宙飞船正在接近着陆跑道。 船和着陆跑道之间有$N$山。 船以每秒1公里的匀速前进，同时以每秒1公里的匀速下降。 

如果船开始从高处下降$H$，然后一秒钟后它就经过了海拔第一座山$H-1$，两秒后它经过了海拔第二座山$H-2$，并且一般来说它通过$i$第 座山的海拔高度$H-(i+1)$使用从零开始的索引时。 

每座山都有高度$h_i$。 船必须严格保持在所经过的每一座山的上方。 任务是找到最小的初始高度$H$这保证了不会发生碰撞。 

输入包括山脉的数量及其高度。 输出是一个整数，即最小安全起始高度。 

山的高度可以达到$10^9$。 这立即排除了对高度值的任何模拟或尝试所有可能的起始高度的任何方法。 山脉的数量可能足够大，以至于解决方案只需处理每座山脉一次，这表明$O(N)$扫描。 

错误的主要根源是严格的不平等。 船不能仅仅到达山顶，它必须保持在山顶之上。 

考虑：```
1
10
```如果初始高度为 11，则船在过山时达到高度 10。 这是一次碰撞，因为高度并不严格大于山的高度。 正确答案是：```
12
```另一个容易犯的错误是忘记了船在到达第一座山之前就失去了高度。 

例子：```
3
2 2 1
```在海拔 4 处，船到达海拔 3 处的第一座山，到达海拔 2 处的第二座山，并与第二座山相撞。 正确答案是：```
5
```第三个陷阱是不一致地使用从一开始的索引和从零开始的索引。 

例子：```
2
1 100
```对于第二座山，船处于海拔高度$H-2$， 不是$H-1$。 正确的条件变为$H > 102$，给出答案：```
103
```错误的索引方案会产生 102。 

## 方法

 最直接的方法是一一尝试候选起始高度。 对于每个海拔高度$H$，模拟飞行并验证是否每座山都被清除。 该模拟是正确的，因为它准确地检查了问题中描述的条件。 

问题是所需的高度可能超过$10^9$。 测试每一个可能的高度直到得到答案将需要数十亿次检查，这是完全不可行的。 

稍微聪明一点的版本对答案执行二分搜索。 对于固定高度$H$，我们可以验证安全性$O(N)$时间。 由于答案大致在一个范围内$10^9$，二分查找需要大约 31 次迭代，给出$O(N \log 10^9)$。 

关键的观察甚至消除了二分搜索。 

路过山时$i$，船舶的高度等于：$$H - (i+1)$$为避免碰撞：$$H - (i+1) > h_i$$重新排列：$$H > h_i + i + 1$$因为这对于每座山都适用，$H$必须严格大于最大值$h_i + i + 1$。 

满足该要求的最小整数是：$$H = \max(h_i + i + 1) + 1$$也可以写成：$$H = \max(h_i + i + 2)$$现在，通过对阵列进行一次扫描即可获得答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N \cdot A)$|$O(1)$| 太慢了 |
 | 二分查找 |$O(N \log A)$|$O(1)$| 已接受 |
 | 最佳|$O(N)$|$O(1)$| 已接受 |

 这里$A$表示答案的大小。 

## 算法演练

 1. 阅读$N$和山的高度。 
2. 初始化`answer = 0`。 
3. 对于索引处的每座山`i`：

 计算`h[i] + i + 2`。 

该值正是安全越过该特定山峰所需的最低起始高度。 
4. 更新`answer`是迄今为止看到的最大值。 

每一座山都对起始海拔施加了下限。 最强的下界决定了最终的答案。 
5、输出`answer`。 

### 为什么它有效

 山用$i$，船达到高度$H-(i+1)$。 安全要求：$$H-(i+1) > h_i$$这相当于：$$H \ge h_i+i+2$$因为$H$是一个整数。 

每一座山都会产生一个这样的下限。 任何小于最大界限的高度对于至少一座山来说是失败的。 最大界限本身同时满足所有不等式。 由于它既充分又必要，因此它是最低有效答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
h = list(map(int, input().split()))

ans = 0

for i in range(n):
    ans = max(ans, h[i] + i + 2)

print(ans)
```该实现直接反映了数学推导。 

表达式`h[i] + i + 2`来自将严格不等式转换为整数下界。 使用`+1`而不是`+2`这是不正确的，因为船必须严格保持在山的上方。 

该循环计算每座山所施加的要求，并仅保留最大的一座。 除了输入数组和运行最大值之外，不需要额外的存储。 

Python 整数自动处理大于$10^9$，所以溢出不是问题。

 ## 边缘情况

 考虑：```
1
10
```该算法计算：$$10 + 0 + 2 = 12$$并输出12。 

手动检查，海拔11到达海拔10的山，严格来说，海拔10并不在它之上。 严格的不平等得到了正确的处理。 

考虑：```
3
2 2 1
```计算值是：$$4,\ 5,\ 5$$所以答案是5。 

在海拔 4 处，船经过海拔 2 处的第二座山并相撞。 最大要求正确确定了最低安全高度。 

考虑：```
2
1 100
```该算法计算：$$1+0+2=3$$和$$100+1+2=103$$给出答案103。 

两秒后就到达了第二座山，而不是一秒。 该公式通过索引项自动合并正确的高度损失`i + 1`，避免相差一的错误。
