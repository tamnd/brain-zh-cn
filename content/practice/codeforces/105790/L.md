 ## 解决方案
 ## 问题理解

 我们有`N`岩石的类型。 有些类型对是不兼容的，这意味着这两种类型不能存储在同一个包中。 Dudu 恰好有两个袋子，每种岩石类型都必须分配给其中一个。 有效的分配是将每对有毒物质分装在不同的袋子中。 

输入描述了一个无向图。 每一种岩石类型都是一个顶点，每一种有毒关系都是一条边。 问题是顶点是否可以分为两组，使得没有边的两个端点都在同一组内。 如果可能的话，我们必须输出这两个组。 否则，我们报告无法进行分离。 

限制足够大，解必须是线性的。 高达`100000`顶点和`100000`边缘，一种重复检查所有对的算法将在`10^10`最坏情况下的操作，远远超出了一秒的限制。 我们需要一个接近的解决方案`O(N + M)`，其中每个顶点和边仅被处理固定次数。 

主要的边缘情况来自图结构而不是输入大小。 没有有毒关系的图表是有效的，因为每块石头都可以放入任一袋子中。 例如：```
Input:
3 0

Output:
POSSIVEL
3 0
1 2 3
```一个粗心的实现，只从顶点开始搜索`1`会错过其他组件中的孤立顶点。 

单个顶点是另一种特殊情况：```
Input:
1 0

Output:
POSSIVEL
1 0
1
```没有需要解决的冲突，因此将唯一的类型放在一个包中是有效的。 假设两个包都必须非空的代码将在此处失败。 

三角形是最小的不可能情况：```
Input:
3 3
1 2
2 3
3 1

Output:
IMPOSSIVEL
```尝试在循环中交替颜色会强制第一个和最后一个顶点接收相同的颜色，即使它们是连接的。 任何仅检查本地邻居而不检测此矛盾的方法都会错误地接受它。 

## 方法

 最直接的方法是尝试将岩石尽可能分成两袋。 由于每块岩石都有两种选择，这就产生了`2^N`可能的任务。 对于每个任务，我们可以检查每个有毒对并检查两个端点是否落在同一个袋子中。 这个方法是正确的，因为它直接测试有效分离的定义，但是当`N`达到`100000`。 最坏的情况需要检查`2^100000`作业。 

图结构提供了一种更好的方式来思考问题。 有效的双袋分离正是图的二分。 用图论术语来说，每个包都是一个独立的集合，无向图可以分为两个独立的集合当且仅当它是二分图。 

可以通过为每个顶点分配两种颜色中的一种来识别二分图，以便每条边连接不同的颜色。 从任何未访问的顶点开始，我们可以给它着色并使用图遍历来强制通过其连接的组件交替颜色。 如果我们找到一条连接两个具有相同颜色的顶点的边，则所需的分离是不可能的。 

蛮力解决方案之所以有效，是因为它探索了所有可能的颜色。 只有两种颜色很重要的观察结果让我们可以直接构造颜色，而不是搜索所有可能性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^N * M) | O(2^N * M) | O(N) | 太慢了 |
 | 最佳| O(N + M) | O(N + M) | 已接受 |

 ## 算法演练

 1. 为图构建邻接表。 每个有毒对都会产生一个无向边缘，因为不相容性在两个方向上起作用。 
2. 保留一个数组来存储每个顶点的颜色。 使用`-1`对于未访问的顶点，并使用`0`和`1`对于两个可能的袋子。 
3. 迭代每个顶点。 如果顶点尚未着色，则从它开始遍历并为其指定颜色`0`。 需要此步骤是因为图表可以具有多个断开连接的组件。 
4. 在遍历过程中，为顶点的每个邻居分配相反的颜色。 如果邻居已经与当前顶点具有相同的颜色，则该图包含矛盾并且答案是不可能的。 
5.所有组件处理完毕后，收集带有颜色的顶点`0`进入第一个袋子和带有颜色的顶点`1`放入第二个袋子。 颜色保证每对有毒物质都被分成这两组。 

为什么它有效：遍历过程中保持的不变性是每条处理过的边都连接具有不同颜色的顶点。 当我们为未着色的邻居指定相反的颜色时，我们会为该边保留此属性。 如果已经着色的邻居颜色错误，则不存在有效的双色分配，因为该边同时需要相同和不同的颜色。 如果遍历结束时没有矛盾，则每条边都在两个颜色组之间交叉，因此这两个包都是有效的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())

    graph = [[] for _ in range(n)]

    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        graph[u].append(v)
        graph[v].append(u)

    color = [-1] * n
    bag0 = []
    bag1 = []

    for start in range(n):
        if color[start] != -1:
            continue

        color[start] = 0
        stack = [start]

        while stack:
            u = stack.pop()

            for v in graph[u]:
                if color[v] == -1:
                    color[v] = color[u] ^ 1
                    stack.append(v)
                elif color[v] == color[u]:
                    print("IMPOSSIVEL")
                    return

    for i in range(n):
        if color[i] == 0:
            bag0.append(i + 1)
        else:
            bag1.append(i + 1)

    print("POSSIVEL")
    print(len(bag0), len(bag1))
    print(*bag0)
    print(*bag1)

if __name__ == "__main__":
    solve()
```邻接列表仅存储现有的有毒关系，这使内存使用量与实际图大小成比例。 遍历使用显式堆栈而不是递归，因为 Python 的默认递归限制对于链而言太小`100000`顶点。 

所有顶点上的外循环处理断开连接的图。 一个常见的错误是仅从顶点运行 DFS 或 BFS`1`，这使得其他组件处于未选中状态。 

表达式`color[u] ^ 1`之间翻转`0`和`1`。 它避免了额外的条件检查，并使两袋分配直接进行。 

最后的收集步骤将内部从零开始的索引转换回问题所使用的从一开始的编号。

 ## 边缘情况

 对于孤立顶点的情况：```
Input:
1 0
```算法访问顶点`1`, 指定颜色`0`，并立即完成，因为没有邻居。 最终集合放置顶点`1`放入第一个袋子，将第二个袋子留空。 由于不存在有毒对，因此输出保持有效。 

对于断开连接的图：```
Input:
4 1
1 2
```首先遍历颜色顶点`1`和`2`。 顶点`3`和`4`永远不会从该组件到达，因此外循环会为它们开始新的遍历。 它们独立地接收颜色，最终的分区包括每个顶点。 

对于不可能的奇循环：```
Input:
3 3
1 2
2 3
3 1
```算法给顶点着色`1`作为`0`, 强制顶点`2`和`3`成为`1`。 当它检查之间的边缘时`2`和`3`，两个端点具有相同的颜色。 由于每条边都必须连接相对的袋子，因此不存在有效的分配，并且算法会正确打印`IMPOSSIVEL`。
