 ## 解决方案
 ## 问题理解

 我们给出一个由下式定义的序列$$F(0)=1,\quad F(1)=2$$以及对于每一个$n \ge 2$,$$F(n)=F(n-1)\cdot F(n-2).$$输入包含单个整数$N$， 在哪里$N$可以大到$10^{18}$。 任务是计算$F(N)$模数$998244353$。 

递归是乘法而不是加法。 由于数字数量呈爆炸式增长，一一计算值很快就变得不可能了。 均匀收纳$F(100)$确切地说已经是不现实的了。 

关键约束是$N \le 10^{18}$。 任何从 0 迭代到$N$立即被排除。 我们需要一个对数解，大致$O(\log N)$，这建议使用隐藏在递归中的一些结构。 

一个常见的错误是应用递归模$998244353$直接并尝试迭代到$N$。 尽管模运算使数字保持较小，$10^{18}$迭代是不可能的。 

考虑一个小例子：```
N = 5
```顺序是$$1, 2, 2, 4, 8, 32$$所以答案是32。 

有趣的观察是，每个项都是 2 的幂。 缺少这种结构会导致方法变得不必要的复杂。 

另一个微妙的情况是当$N=0$。```
Input
0
```答案是```
1
```因为第一项明确定义为 1。任何涉及斐波那契指数的公式都必须正确保留此基本情况。 

对于巨大的值，会出现最后一个陷阱，例如```
N = 10^18
```斐波那契指数本身是巨大的。 精确地计算它是不可能的，因此我们必须在求幂之前使用模运算来减少指数。 

## 方法

 暴力解决方案从左到右计算序列。 

对于每个索引$i$,$$F(i)=F(i-1)\cdot F(i-2)\pmod{998244353}.$$这种递归是正确的，因为模乘保留了模乘结果$998244353$。 

问题在于迭代次数。 和$N$最多$10^{18}$，该算法需要大约$10^{18}$乘法，这是完全不可能的。 

要找到更快的解决方案，请将每一项写成 2 的幂。 

自从$$F(0)=2^0,\qquad F(1)=2^1,$$认为$$F(n)=2^{a_n}.$$代入递归式可得$$2^{a_n}=2^{a_{n-1}} \cdot 2^{a_{n-2}}
       =2^{a_{n-1}+a_{n-2}}.$$因此$$a_n=a_{n-1}+a_{n-2}.$$指数序列遵循斐波那契递推式$$a_0=0,\qquad a_1=1.$$所以$a_n$正是$n$-th 斐波那契数。 

这给出了闭合形​​式$$F(n)=2^{Fib(n)}.$$现在问题变成了计算$$2^{Fib(N)} \bmod 998244353.$$模数$998244353$是素数。 根据费马小定理，$$2^{k \bmod (998244353-1)}
\equiv
2^k
\pmod{998244353}.$$所以我们只需要$$Fib(N)\bmod 998244352.$$斐波那契数模一个值可以计算为$O(\log N)$使用矩阵求幂或快速加倍。 获得指数模后$998244352$，一次模幂给出最终答案。

4. 计算$$2^e \bmod 998244353$$使用 Python 的内置模幂运算。 
5. 打印结果。 

### 为什么它有效

 每个序列值都是 2 的幂。 写作$F(n)=2^{a_n}$将乘法递推转化为$$a_n=a_{n-1}+a_{n-2},$$这正是带有初始值的斐波那契递推$0$和$1$。 因此$a_n=Fib(n)$, 给予$$F(n)=2^{Fib(n)}.$$由于模数是素数，费马定理允许减少指数模数$998244352$。 快速加倍以对数时间计算该指数，然后模幂产生所需的值。 每个变换都保留精确的模值$998244353$，所以最终的答案是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
PHI = MOD - 1

def fib(n):
    if n == 0:
        return (0, 1)

    a, b = fib(n >> 1)

    c = (a * ((2 * b - a) % PHI)) % PHI
    d = (a * a + b * b) % PHI

    if n & 1:
        return (d, (c + d) % PHI)
    else:
        return (c, d)

n = int(input())

exp = fib(n)[0]
print(pow(2, exp, MOD))
```该解决方案首先定义素数模及其欧拉 totient 值，该值等于$MOD-1$因为模数是素数。 

这`fib`函数实现快速加倍身份：$$Fib(2k)=Fib(k)\cdot(2Fib(k+1)-Fib(k))$$和$$Fib(2k+1)=Fib(k)^2+Fib(k+1)^2.$$每次递归调用都会将参数减半，从而产生对数复杂度。 

所有斐波那契计算均以模进行$998244352$，因为只有指数模$MOD-1$是需要的。 

获得后$Fib(N)\bmod 998244352$，答案是使用 Python 的三参数计算的`pow`，它有效地评估模幂。 

一个微妙的实现细节是表达式```
(2 * b - a) % PHI
```如果没有模运算，中间值可能会变为负值。

 ## 边缘情况

 考虑尽可能最小的输入：```
0
```该算法计算$Fib(0)=0$，然后评估$$2^0 \bmod 998244353 = 1.$$输出与序列定义完全匹配。 

考虑递归生成的第一个值：```
2
```我们得到$Fib(2)=1$， 所以$$2^1=2.$$递归本身给出$$F(2)=F(1)\cdot F(0)=2\cdot1=2.$$两种观点都一致。 

对于非常大的值，例如```
1000000000000000000
```该算法永远不会迭代到$N$。 快速加倍反复使参数减半，只需要大约六十个递归层。 指数减少模数$998244352$，并且最终的模幂仍然有效。 这正是对数方法设计的目的。
