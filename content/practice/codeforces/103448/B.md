 ## 解决方案
 ## 问题理解

 我们在 2D 平面上有一个矩形建筑物，与坐标轴对齐。 每个食堂都表示为一个点。 对于每个食堂，我们想要计算其到矩形上任何点的最短欧几里德距离，包括其内部边界和角点。 计算完这些距离后，我们选择距离最小的食堂，如果多个食堂具有相同的最小值，我们选择索引最小的一个。 

从几何角度来看，这是要求从一个点到一个轴对齐的矩形的距离，根据该点相对于矩形的位置，其行为会有所不同。 如果该点与矩形的投影水平和垂直对齐，则最近的点位于边缘上。 如果它位于对角线外侧，则最近的点是角点。 如果它在内部，距离将为零，但该问题明确保证没有食堂位于内部或边界上，因此每个点都严格位于外部。 

n 最大为 100000 的约束意味着我们必须在恒定时间内处理每个食堂。 任何涉及超过 O(1) 计算的每个食堂几何的解决方案都是可以接受的，但任何涉及排序或成对比较的解决方案都是不必要的，而且会造成浪费。 仅当将每个点的时间减少到恒定时，简单的几何方法才是好的。 

微妙的边缘情况来自于正确处理最近的投影是位于矩形间隔内部还是外部。 例如，如果一个点位于矩形的正上方，则最近距离是纯垂直的； 如果是左上方对角线，则最近的点是左上角，距离是对角线。 对这些情况进行错误分类会导致距离计算不正确。 

另一个边缘情况是相等处理。 由于对称性，两个食堂可以具有完全相同的距离，在这种情况下，我们必须小心地保留最小索引，这意味着除非新距离严格更小，否则我们无法更新答案。 

## 方法

 蛮力视角很简单：对于每个食堂，我们考虑它到矩形边界上每个点的距离。 由于矩形是连续的，因此这成为几何最小化问题而不是离散枚举问题。 人们可以尝试对边缘进行采样或将矩形分成段并计算点到段的距离，但如果我们认识到从点到轴对齐矩形的距离的标准封闭式解决方案，那么这就不必要的复杂性。 

关键的观察结果是矩形引起沿 x 和 y 轴的独立行为。 对于任意点 (x, y)，通过将 x 钳入区间 [X1, X2] 并将 y 钳入 [Y1, Y2] 来获得矩形上的最近点。 这会产生欧几里得距离意义上的矩形上或矩形内的最近点。 然后计算从原始点到该投影点的距离。 

这将问题简化为简单的每点计算：构造最近的点 (px, py)，其中 px 是剪裁到矩形的 x 范围内的 x，py 是剪裁到其 y 范围内的 y，然后计算平方欧几里得距离。 平方距离就足够了，因为平方根是单调的，不需要进行比较。 

简化后，蛮力思想仍然是 O(n)，但关键的改进是认识到投影在每个轴和恒定时间上是独立的。

 ## 算法演练

 我们独立处理每个食堂并保留迄今为止看到的最佳候选者。

1. 将答案索引初始化为 1，并计算其到矩形的平方距离。 这建立了一个基线，确保我们始终有一个有效的比较目标。 使用下述投影规则计算距离。 
2. 对于从 2 到 n 的每个食堂 i，计算其在矩形上的最近点。 我们通过将其 x 坐标夹入 [X1, X2] 来实现此目的。 如果x小于X1，我们使用X1； 如果大于X2，我们使用X2； 否则我们保持 x 不变。 相同的逻辑适用于带有 [Y1, Y2] 的 y。 此步骤在矩形上构造距给定食堂几何上最接近的可行点。 
3. 计算食堂与其投影点之间的平方距离。 平方距离就足够了，因为在单调变换下保留了顺序，并且避免了浮点精度问题。 
4. 将此距离与当前最佳距离进行比较。 如果它严格小于，则更新最佳距离和最佳索引。 如果相等，则不执行任何操作，因为必须保留较早的索引。 
5、处理完所有食堂后，输出存储的最佳索引。 

为什么它有效

 凸轴对齐矩形上距外部点最近的点始终是通过独立最小化水平和垂直位移来获得的。 与夹紧坐标的任何偏差都会增加距离的至少一个平方分量。 这确保了构建的投影对于欧几里德距离来说是全局最优的。 由于每个食堂都是通过这种精确的最佳投影进行评估的，因此所有候选者中的最小值都是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def clamp(v, lo, hi):
    if v < lo:
        return lo
    if v > hi:
        return hi
    return v

def dist2(x1, y1, x2, y2):
    dx = x1 - x2
    dy = y1 - y2
    return dx * dx + dy * dy

def solve():
    n = int(input())
    X1, Y1, X2, Y2 = map(int, input().split())

    best_idx = 1
    x0, y0 = map(int, input().split())

    px = clamp(x0, X1, X2)
    py = clamp(y0, Y1, Y2)
    best_dist = dist2(x0, y0, px, py)

    for i in range(2, n + 1):
        x, y = map(int, input().split())
        px = clamp(x, X1, X2)
        py = clamp(y, Y1, Y2)
        d = dist2(x, y, px, py)

        if d < best_dist:
            best_dist = d
            best_idx = i

    print(best_idx)

if __name__ == "__main__":
    solve()
```该解决方案首先读取矩形并使用第一个食堂初始化答案。 这避免了循环内的特殊外壳，并确保比较始终具有有效的基线。 

钳位功能强制将几何投影到每个轴的矩形间隔上。 dist2 函数完全避免了浮点运算，鉴于坐标约束很小但比较必须准确，这一点很重要。 

平局决断是通过仅在找到严格较小的距离时进行更新来隐式处理的，从而保留最早的索引。

 ## 边缘情况

 当投影逻辑被到矩形边界的朴素绝对距离替换时，会发生常见的失败。 例如，将距离视为 min(|x-X1|, |x-X2|) + min(|y-Y1|, |y-Y2|) 是不正确的，因为欧几里得几何不会相加分离。 

取矩形 [0,0] 到 [4,4] 和点 (-3, -3)。 正确的投影是 (0,0)，距离的平方是 18。错误的公式可能会错误地组合轴距离，从而低估对角线成本。 

该算法可以正确处理此问题，因为两个坐标都是独立固定的，从而生成真正的最近点 (0,0)。 计算出的平方距离变为 (−3)^2 + (−3)^2 = 18，与矩形上的真实欧几里得最小值匹配。
