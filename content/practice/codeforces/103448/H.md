 ## 问题理解

 我们得到了一组战斗单位，每个单位都由攻击值和生命值来描述。 选择一个单元作为初始“活动”单元。 此后，该单位反复与其他单位一一战斗，每次战斗都是确定性的：双方同时、反复地造成等于其攻击值的伤害，直到至少一方生命值为零。 一旦一个单位的健康状况变为阴性，就被认为已死亡。 

只要活动单元仍然存在并且还存在其他单元，该过程就会继续。 在每一步中，主动单位可能会面对任何剩余的对手，并且我们可以假设对手的顺序是以最有利的方式选择的。 问题是是否存在至少一种起始单位的选择和至少一种对手的命令，使得包括起始单位在内的所有单位最终都会死亡。 

限制允许最多 100000 个单位，攻击力和生命值最多为 1e6。 这立即排除了任何模拟一对一战斗或尝试所有排列的解决方案。 即使每个候选的 O(n^2) 推理也会变得太慢，因此任何可行的解决方案都必须将问题减少到聚合每个单元的计算。 

当人们试图贪婪地模拟战斗时，就会出现一种微妙的失败案例。 例如，假设主动单位应始终首先与最弱的对手战斗可能会产生误导，因为具有高攻击力的弱对手可能比具有低攻击力的强大对手更危险。 另一种失败模式是假设生存仅取决于所受到的总伤害，而不考虑主动单位是否能够在中途死亡之前真正完成所有杀戮。 

正确的模型必须捕获“每次战斗持续多长时间”和“在此期间受到多少伤害”，同时仍然以有用的方式保持与顺序无关。 

## 方法

 直接的暴力方法将尝试每一个可能的起始单元，并针对每个单元模拟剩余对手的所有排列。 在每次模拟中，我们都会一步步运行战斗过程。 每场战斗最多可能需要 O(1e6 / atk) 轮，在最坏的情况下，每个排列有 O(n) 场战斗。 排列的数量使得这完全不可行。 

关键的观察是，一旦起始单位固定，每个对手都会为起始单位贡献完全确定的成本，与顺序无关，只要起始单位存活足够长的时间以到达战斗。 每个对手 j 需要固定数量的攻击回合，这取决于杀死它所需的起始单位的命中次数。 在每一轮中，起始单位受到的伤害等于对手的攻击力。 这意味着每个对手都会对起始单位贡献确定的总伤害成本。 

订购问题变得比最初看起来更简单。 无论顺序如何，受到的总伤害都是固定的，唯一真正的限制是起始单位是否存活足够长的时间来完成所有战斗。 由于生存仅取决于累积伤害，因此任何不超过任何前缀的生命值的排序都是有效的，并且当总伤害不超过起始单位的生命值时，这样的排序恰好存在。 

这将问题简化为评估每个可能的起始单元的封闭式表达式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n!·n·轮) | O(1) | O(1) | 太慢了|
 | 每个候选人的封闭表格 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们固定一个候选起始单位i，并分析它是否可以在死亡前消灭所有其他单位。 

### 1. 预先计算全局聚合

我们计算所有攻击值的总和以及攻击总和乘以所有单位的生命值。 这些聚合使我们能够简洁地表达每个对手的贡献。 

### 2. 零攻击案例单独处理

 如果起始单位攻击力为零，且总数超过1个，则永远无法杀死任何对手，因此立即无效。 如果它是唯一的单位，那么答案就很有效。 

原因是零攻击意味着杀死任何积极健康单位所需的无限次攻击。 

### 3. 计算固定起始单位的对手贡献

 对于攻击 A 的固定起始单位 i，每个对手 j 需要的命中次数等于 hp_j / A 的上限。在每次命中期间，j 对 i 造成 atk_j 伤害，因此 j 的总伤害为：

 ceil(hp_j / A) × atk_j。 

我们将上限项重写为 (hp_j + A − 1) // A，使其可以用整数算术计算。 

### 4. 将总伤害转化为封闭形式

 我们将总和扩展到所有对手，并使用全局聚合来表达它，以便我们可以在每个候选者的 O(1) 中计算它。 

### 5.检查生存状况

 如果受到的总伤害小于或等于其自身生命值，则起始单位有效。 平等是被允许的，因为这意味着最后的战斗以双方同时死亡而结束。 

### 6. 尝试所有候选者

 我们对每个单元评估这一条件，如果至少有一个满足它，则返回 YES。 

### 为什么它有效

 对于固定的起始单位，每个对手都会贡献确定的总伤害量，该伤害量与顺序无关。 任何有效的排序只需要确保前缀总和不超过健康状况，但只要总和在限制范围内，就可以保证前缀可行性，因为所有贡献都是非负的。 因此，将受到的总伤害与起始单位的健康状况进行比较，生存率减少到一个不等式。 由于该过程总是在起始单位死亡或所有对手死亡时结束，因此满足这个不等式恰好表征了可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    atk = []
    hp = []

    total_atk = 0
    total_ah = 0  # sum atk * hp

    for _ in range(n):
        a, h = map(int, input().split())
        atk.append(a)
        hp.append(h)
        total_atk += a
        total_ah += a * h

    if n == 1:
        print("YES")
        return

    for i in range(n):
        a_i = atk[i]
        h_i = hp[i]

        if a_i == 0:
            continue

        # compute sum_{j != i} ceil(hp_j / a_i) * atk_j
        # = sum atk_j * (hp_j + a_i - 1) // a_i

        num = total_ah - atk[i] * hp[i] + (a_i - 1) * (total_atk - atk[i])
        if num // a_i <= h_i:
            print("YES")
            return

    print("NO")

if __name__ == "__main__":
    solve()
```该代码首先构建全局总和，以便可以在恒定时间内评估每个候选者。 对于每个起始单位，它使用代数分解重建总伤害表达式，而不是迭代所有对手。 

一个关键细节是所选单位与全局总和的分离，因为它不能对自己的对手组做出贡献。 另一个微妙的点是通过跳过零攻击单位来安全地处理它们，因为否则划分将是无效的。

 ## 边缘情况

 一个关键的边缘情况是所选单位的攻击力为零。 对于输入如：```
2
0 10
5 5
```第一个单位无法杀死任何东西，因此它会立即失败。 该算法通过跳过零攻击候选者来处理这个问题。 

另一种情况是所有单位都相同时：```
3
2 2
2 2
2 2
```每个候选者都会产生相同的对称伤害结构，并且没有一个能够承受累积所需的命中，因此输出为“否”。 该公式正确地评估了平等贡献并消除了不平等。 

第三种情况是单个极强的单位：```
3
10 1
1 100
1 100
```在这里，强大的单位在很少的回合内杀死了两个对手，并且其自身的累积伤害仍然受到其生命值的限制。 该不等式仅适用于该候选者，表明该算法正确地识别了唯一的可行起点。
