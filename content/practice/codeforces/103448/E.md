 ## 问题理解

 我们得到一个无向图，其中每个节点都带有一个非负整数值。 图结构告诉我们哪些节点可以交互，并且值通过沿任何选定路径应用的操作而演变。 

单个操作在图中选择一条路径，例如一系列节点，其中连续节点通过边连接。 如果路径长度为 t，则节点沿路径对称配对：第一个与最后一个，第二个与倒数第二个，依此类推。 路径上的每个节点将其值替换为其当前值与其配对节点的值的按位与。 所有更新同时发生，因此操作基于路径上的原始值，而不是部分更新的值。 

我们可以在任何路径上执行此操作任意多次。 任务是在任意多次操作后最小化所有节点的最大值。 

这些约束允许多达五十万个节点和边，这会立即排除任何二次甚至接近二次的东西。 任何解决方案都必须在线性或接近线性的时间内有效地处理图形，通常为 O(n + m)。 这强烈表明答案取决于连接的组件和每个组件计算的一些聚合属性。 

一个微妙的点是，操作不是局部边缘更新，而是路径范围的对称对交互。 很容易错误地认为这将我们限制为仅相邻传输，但实际上路径可以连接远处的节点，并且重复选择的路径可以模拟更广泛的混合。 

一个天真的误解是认为只有所选路径的端点才重要，或者认为值传播缓慢。 例如，在链 1-2-3-4 上，人们可能认为节点 1 只能影响 2，但路径 1-2-3-4 同时耦合 (1,4) 和 (2,3)，允许信息在一次操作中跨越整个链。 

关键的边缘情况是树或链，其中重复的操作逐渐组合值。 如果我们错误地假设值仅在每个边缘本地合并，我们就会低估信息传播的速度。 

## 方法

 直接的暴力方法可以模拟操作。 我们将反复选择任意路径并应用对称的成对 AND 更新，直到无法进一步改进。 每个操作都会扫描一条路径，并且可能存在指数级数量的可能路径和序列。 即使将自己限制在简单的策略（例如在边缘或随机路径上重复应用操作）也会导致潜在的 O(nm) 或更糟糕的行为，这远远超出了限制。 

结构洞察力来自于对实际传播的信息的理解。 该运算仅使用按位 AND，即每比特单调递减。 一旦节点中的某个位变为零，它就永远不会返回。 这表明节点的最终状态完全取决于可以通过操作序列强制初始值与其交互的初始值。 

因为我们可以在连接的组件内选择任意路径，所以任何节点最终都可以直接或间接地与同一组件中的任何其他节点配对。 重复应用路径操作使我们能够合并整个组件的信息。 这意味着每个连接的组件的行为就像一个封闭的系统，其中所有值都可以组合。 

在一个组件内，我们可以强制每个节点达到的最强值是该组件中所有初始值的按位与。 即使在一个节点中丢失的任何位都无法在全局最小最大化目标中生存，因为我们总是可以设计最终迫使该位与其他节点进行“与”运算的操作，相反，任何幸存的位都必须存在于所有贡献节点中。

因此，每个组件都会折叠为等于其节点上的 AND 的统一值。 最终答案由这些按组件与值的最大值确定，因为不同的组件独立演化，并且目标是最小化全局最大值。

 | --- | --- | --- | --- |
 | 暴力模拟| 指数| O(n + m) | 太慢了|
 | 连接组件 + 按位与 | O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们将推理转化为具体的程序。 

1. 使用 DFS 或 BFS 识别图的所有连接组件。 每个组件代表一组节点，这些节点可以通过有效操作序列进行交互。 
2. 对于每个组件，计算其内部所有节点值的按位与。 这是通过将运行值初始化为所有位集（或第一个节点的值）并对组件中的每个其他节点的值进行 AND 运算来完成的。 
3. 跟踪所有组件中这些组件级 AND 结果的最大值。 
4. 输出该最大值。 

我们对组件取最大值而不是求和或更全局的东西的原因是组件独立演化。 任何操作都无法桥接两个断开连接的组件，因此它们的最终状态不会相互影响。 

为什么它有效

 在连接的组件内，重复的路径操作允许我们在任意一对节点之间传播信息。 每个操作都可以通过所选路径上的对称性来耦合两个节点，并且通过链接这些操作，每个节点最终都会受到每个其他节点的影响。 因为唯一的操作是按位与，所以任何节点上的值只能按位减少，并且所有节点中幸存的任何位都可以通过仔细配对来保留。 每个节点可以被迫达成一致的唯一稳定值是组件中所有初始值的 AND，因为任何节点中缺少的任何位都可以通过包含该节点的配对路径来消除。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    w = list(map(int, input().split()))

    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    visited = [False] * n
    ans = 0

    for i in range(n):
        if visited[i]:
            continue

        stack = [i]
        visited[i] = True
        comp_and = (1 << 30) - 1  # enough for w <= 1e9

        while stack:
            u = stack.pop()
            comp_and &= w[u]
            for v in g[u]:
                if not visited[v]:
                    visited[v] = True
                    stack.append(v)

        ans = max(ans, comp_and)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先构建图的邻接列表表示。 然后，它运行迭代 DFS 以避免大型图上的递归限制。 每次发现新组件时，它都会初始化一个全位掩码并将其与该组件中遇到的所有节点值相交。 

最终答案累加最大分量结果。 位掩码`(1 << 30) - 1`由于输入以 10^9 为界，因此可以安全地覆盖所有可能的值。 

## 工作示例

 考虑一个包含两个组件的小图。 假设组件一的值为 5 (101) 和 7 (111)，组件二的单个节点的值为 2 (010)。 该算法独立处理每个组件。 

对于第一个分量，AND 变为 101 & 111 = 101，即 5。对于第二个分量，它只是 2。最终答案是 max(5, 2) = 5。
