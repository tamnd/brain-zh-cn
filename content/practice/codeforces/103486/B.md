 ## 解决方案
 ## 问题理解

 我们给定三个整数$A$,$B$， 和$K$。 任务是计算分数的值$A / B$作为十进制数并精确输出$K$小数点后的数字。 结果必须使用四舍五入规则进行四舍五入，这意味着我们立即检查$K$- 第一个小数位：如果是 5 或更多，我们增加$K$-第一位数字加一，否则我们保持不变。 

输入尺寸很小：$A, B \le 1000$和$K \le 1000$。 这立即告诉我们，我们不需要任何渐近复杂的数值方法。 长除法的直接模拟就足够了，因为产生最多$K+1$只需要数字$O(K)$算术步骤。 

主要的微妙之处在于舍入和进位传播的正确性。 当舍入导致级联进位时，简单的实现通常会失败，例如当小数部分以 9999 结尾并且舍入将其推入整数部分时。 

例如，考虑：

 输入：$1\ 3\ 1$确切的值为$0.3333...$如果我们计算两位数字：$0.33$，舍入不执行任何操作，并且输出正确。 

但对于：

 输入：$2\ 3\ 1$确切的值为$0.6666...$我们计算$0.66$，然后看到下一个数字 6，所以我们四舍五入到$0.7$。 这表明必须在生成额外数字之后而不是在数字生成期间应用舍入。 

当舍入影响整数部分时，会出现另一种边缘情况：

 输入：$999\ 1000\ 3$我们得到$0.999...$。 通过四舍五入，这变成了$1.000$。 仅操作小数位的简单的基于字符串的方法可能无法将进位传播到整数部分。 

## 方法

 一个蛮力的想法是将精确的有理值计算为浮点数并使用以下格式对其进行格式化$K$使用内置舍入的小数。 虽然概念上很简单，但浮点表示对于高达 1000 位精度的情况并不可靠。 标准双精度数只能保证大约 15-17 位小数位的精度，因此对于更大的$K$，这种方法就变得不正确了。 

另一种简单的方法是使用重复乘以 10 和整数除法来逐一计算每个十进制数字，这本质上是长除法。 这是正确的，但必须仔细扩展以生成一位额外的数字以进行舍入。 

关键的观察是，十进制形式的整数除法可以纯粹用整数算术来模拟。 在每一步中，我们维护一个余数，将其乘以 10，然后通过除法提取下一个数字。 这完全避免了浮点错误。 为了支持舍入，我们生成$K+1$数字而不是$K$，然后在最后应用一次舍入规则。 因为$K \le 1000$，这很容易足够快。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 浮点+格式化| O(1) | O(1) | O(1) | O(1) | 对于大 K 不正确 |
 | 长除法模拟 | O(K) | O(K) | 已接受 |

 ## 算法演练

 我们模拟手动长除法$A / B$以 10 为基数，但最多可达$K+1$小数位。 

1. 将整数部分计算为$A // B$初始余数为$A \% B$。 整数部分是固定的，不依赖于精度。 
2. 重复生成小数位。 对于每一步，将余数乘以 10 并计算下一个数字：$(\text{remainder} \times 10) // B$。 相应地更新其余部分。 这反映了手动除法中十进制扩展的工作原理。 
3. 准确存储$K+1$小数部分的数字。 额外的数字仅用于舍入决策。 
4. 应用舍入。 如果$(K+1)$-第 位数字至少为 5，递增$K$第 - 位数字。 这可能会触发通过小数位向后传播进位。 
5. 如果进位传播溢出超过第一个小数位，则传播到整数部分。 
6. 构造最终的字符串$K$小数位，必要时用零填充。 

工作原理：长除法不变式是在每一步之后，余数代表按 10 的幂缩放的真实小数部分。每个提取的数字对于以 10 为底的扩展中的该位置来说都是完全正确的。 由于我们在四舍五入之前生成了一位额外的数字，因此四舍五入的决定$K$-th 位使用有关真实值是否超出截断边界的准确信息，确保半向上舍入的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    A, B, K = map(int, input().split())

    int_part = A // B
    rem = A % B

    digits = []
    for _ in range(K + 1):
        rem *= 10
        digits.append(rem // B)
        rem %= B

    # rounding from K+1 digit
    if digits[-1] >= 5:
        i = K - 1
        carry = 1
        while i >= 0 and carry:
            digits[i] += carry
            if digits[i] == 10:
                digits[i] = 0
                carry = 1
            else:
                carry = 0
            i -= 1

        if carry:
            int_part += 1

    frac = digits[:K]

    print(f"{int_part}." + "".join(map(str, frac)))

if __name__ == "__main__":
    solve()
```整数部分直接提取，因为它与小数精度无关。 循环生成$K+1$使用经典长除法和余数跟踪的数字。 舍入块显式传播向后进位，这是必要的，因为舍入可以跨多个尾随 9 级联。 

最后的格式化步骤确保准确$K$通过对数组进行切片来获取数字。 由于我们显式存储数字，因此自然会保留零填充。 

## 工作示例

 ### 示例 1

 输入：`1 2 2`我们计算$1 / 2 = 0.5$。 

| 步骤| 剩余| 数字|
 | --- | --- | --- |
 | 1 | 1 → 10 | 5 |
 | 2 | 0 → 0 | 0（四舍五入的额外数字）|

 我们有数字`[5, 0]`。 由于额外的数字为 0，因此不会发生舍入。 输出变为`0.50`。 

这证实了尾随零保留可以正确工作。 

### 示例 2

 输入：`10 99 5`我们模拟长除法：

 | 步骤| 剩余| 数字|
 | --- | --- | --- |
 | 1 | 10 → 100 | 1 |
 | 2 | 1 → 10 | 0 |
 | 3 | 10 → 100 | 1 |
 | 4 | 1 → 10 | 0 |
 | 5 | 10 → 100 | 1 |
 | 6（额外）| 1 → 10 | 0 |

 数字：`[1, 0, 1, 0, 1, 0]`由于最后一位数字为 0，因此不会发生舍入。最终输出为`0.10101`。 

这表明该算法保留了交替的十进制结构，而不受舍入逻辑的干扰。
