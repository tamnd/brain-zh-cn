 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，并要求根据其分布计算单个派生值。 “有趣值”被定义为两个量的乘积：序列的长度和序列的范围，其中范围是最大和最小元素之间的差。 

因此，任务简化为读取所有值，识别最小和最大元素，计算它们的差异，并将该差异乘以元素数量。 

约束足够小，单个线性扫描就足够了。 对于多达 10,000 个元素，任何以恒定次数检查每个元素的解决方案都足够快。 如果实现干净的话，在 Python 中 0.5 秒的限制下，即使是几百万个原始操作也是微不足道的。 

有一个微妙的陷阱可能会导致粗心的实现失败：整数范围和符号处理。 由于值可以为负，因此最小值可以为负，最大可以为正，并且范围可以变大。 例如，如果数组是`[-100000, 100000]`，范围是`200000`，并乘以`n`必须在计算出正确的极值之后完成，而不是在使用部分假设进行扫描期间完成。 

另一个边缘情况是单元素数组。 如果`n = 1`，则最大值和最小值相等，范围为零，并且无论值本身如何，答案都必须为零。 任何未正确使用第一个元素而错误地初始化 min 或 max 的实现都可能会在此处中断。 

## 方法

 强力解释可能会尝试计算每个可能的子结构的范围或重复重新计算极值，但这是不必要的，因为序列是固定的，我们只需要全局最小值和最大值一次。 

简单粗暴的想法是：对于每个元素，将其与所有其他元素进行比较以确定最小值和最大值。 这导致了 O(n²) 方法，因为 n 个元素中的每一个都参与 n 次比较。 虽然它是正确的，但它远远超出了所需的范围，并且由于重复重新计算相同的全局属性而浪费了工作量。 

关键的观察是范围仅取决于两个值：整个序列的最小值和最大值。 一旦知道了这些，其他结构就不再重要了。 这将问题从成对比较转化为单遍累积问题。 

我们可以在扫描一次数组的同时保持运行最小值和最大值。 每个新元素都会在恒定时间内更新当前最小值或最大值。 扫描后，我们计算`(max - min) * n`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 单程最小/最大 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取整数`n`，它决定了我们将处理多少个值。 这也定义了最终答案中的乘数。 
2.初始化两个变量`mn`和`mx`使用序列的第一个元素。 这可以避免错误的哨兵值，并确保即使所有数字均为负数或相同时也能保持正确性。 
3. 迭代序列的剩余元素。 对于每个元素，将其与`mn`并更新`mn`如果它更小。 做同样的事情`mx`如果它更大。 这在每一步都保持正确的全局极值。 
4. 处理完所有元素后，计算范围为`mx - mn`。 这有效是因为`mx`和`mn`现在代表整个序列的真实最大值和最小值。 
5. 将范围乘以`n`得到最终的“有趣值”并输出。 

### 为什么它有效

 在迭代的每个点上，`mn`和`mx`正是迄今为止处理的前缀的最小值和最大值。 这一不变量成立，因为每个新元素要么被忽略，要么在扩展边界时替换其中一个元素。 在最后一个元素之后，前缀是整个数组，所以`mn`和`mx`是真正的全局极值。 由于范围仅取决于这些极值并且与排序无关，因此最终计算是精确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n_and_rest = sys.stdin.read().strip().split()
n = int(n_and_rest[0])
arr = list(map(int, n_and_rest[1:]))

mn = arr[0]
mx = arr[0]

for x in arr[1:]:
    if x < mn:
        mn = x
    if x > mx:
        mx = x

answer = (mx - mn) * n
print(answer)
```该解决方案一次读取所有输入，以避免重复输入调用带来的开销。 对于 n 高达 10,000 的情况，这并不是严格要求的，但可以保持实现的简洁和稳健。 

的初始化`mn`和`mx`从第一个元素开始是至关重要的。 使用像这样的占位符`float('inf')`有效，但直接锚定到数组可以避免边缘情况错误，并保持整数类型之间的比较一致。 

最后的乘法是在计算范围后完成的，确保中间值不会影响正确性。

 ## 边缘情况

 对于像这样的单元素输入`1 100000`，算法初始化两者`mn`和`mx`到`100000`。 迭代期间不会发生更新。 范围变为`0`，并乘以`n`仍然产生`0`，匹配单个值具有零扩展的定义。 

对于完全负序列，例如`3 -5 -2 -10`，运行最小值变为`-10`最大值变为`-2`。 范围是`8`，答案是`24`。 该算法正确地避免了任何值必须为正的假设，因为所有比较都是纯粹相关的。 

对于像这样的混合极端序列`2 -100000 100000`,`mn`变成`-100000`和`mx`变成`100000`，生产一系列`200000`。 乘以`n`给出`400000`，这证实了不会发生溢出或符号处理问题，因为所有操作都保留在 Python 的任意精度整数算术范围内。
