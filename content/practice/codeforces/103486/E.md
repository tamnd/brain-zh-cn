 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个数组中都有一个正整数数组，任务是确定我们是否可以选择两个不同的位置，使得这些位置的值在二进制中恰好相差一位，具体来说，它们的 XOR 等于 1。 

话说$A_i \oplus A_j = 1$是非常有限制的。 XOR 为 1 意味着两个数字之间只有最低有效位不同，其他所有位都相同。 换句话说，一个数字必须与另一个数字完全相同，但其最后一位被翻转。 所以有效的对总是看起来像$x$和$x \oplus 1$。 

测试用例的输入大小可能很大，最多可达$5 \times 10^5$总元素。 这立即排除了任何比较所有对的方法，因为即使是单个最坏情况测试$10^5$元素已经导致大约$10^{10}$配对检查。 

当所有数字都相同时，会出现微妙的边缘情况。 例如，如果数组是$[7, 7, 7]$，没有一对可以工作，因为$7 \oplus 7 = 0$，而不是 1。另一种边缘情况是当数字相差超过一位时，例如$[2, 3]$， 在哪里$2 \oplus 3 = 1$有效，但是$[2, 4]$不，因为$2 \oplus 4 = 6$。 

核心困难在于认识到该条件不是关于任意 XOR 结构，而是关于整数之间非常具体的邻居关系。 

## 方法

 最直接的想法就是尝试所有的配对$(i, j)$并计算它们的异或。 这是正确的，因为它详尽地检查了每一个可能的候选对，并且肯定会找到有效的候选对（如果存在）。 然而，对于$n = 10^5$，这大约需要$5 \times 10^9$在一个测试用例中进行操作，这远远超出了任何可行的限制。 

关键的观察是我们实际上不需要比较任意对。 如果一个数字$x$参与有效配对，其伙伴必须完全正确$x \oplus 1$。 这将问题变成了成员资格检查问题而不是配对问题。 我们可以将所有值存储在基于哈希的结构中，并测试每个值是否通过扫描来搜索两端$x$有其对应物$x \oplus 1$存在于数组中。 

这将问题从二次比较减少为每个元素的恒定时间查找。

 ## 算法演练

 1. 对于每个测试用例，读取数组并将所有元素插入到哈希集中。 

这允许对我们可能需要检查的任何值进行常量成员资格查询。 
2. 迭代每个元素$x$在数组中。 

对于每个值，计算$x \oplus 1$，这是可以与以下形成有效对的唯一可能值$x$。 
3. 检查是否$x \oplus 1$存在于集合中。 

如果是，我们立即知道存在有效的对，并且可以停止处理当前的测试用例。 
4. 如果我们完成扫描数组而没有找到任何这样的对，则得出结论不存在有效索引。 

关键的设计选择是预先计算集合中的成员资格，而不是每次尝试在数组内动态搜索，这会重复地重新扫描结构。 

### 为什么它有效

 条件$A_i \oplus A_j = 1$唯一地确定一个值与另一个值。 对于任何固定的$x$, 恰好有一个候选伙伴$x \oplus 1$，并且没有其他值可以满足方程$x$。 这意味着每个有效对必须显示为在转换下直接互补的两个元素$x \mapsto x \oplus 1$。 通过检查所有元素是否存在补码，我们穷尽了所有可能的有效结构，没有重复或遗漏。
