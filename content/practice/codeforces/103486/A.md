 ## 解决方案
 ## 问题理解

 我们给定一个整数序列，需要判断它在奇偶性方面是否看起来“平衡”。 每个数字要么是奇数，要么是偶数，我们只需计算每个类别中有多少个数字。 

规则非常严格：计算奇数值的数量和偶数值的数量，然后取它们的绝对差。 如果该差值最多为 1，则生成器被认为是可接受的，否则则不可接受。 

输入大小高达 100,000 个数字，因此任何扫描一次数组的解决方案都足够快。 二次方法会尝试成对比较元素或重复过滤列表，这在最坏的情况下会执行大约 10^10 次操作，并且明显超出限制。 这立即表明我们只需要一次传递，每个元素都需要持续工作。 

这里的边缘情况主要是关于小输入。 当 N 等于 1 时，答案总是“好”，因为计数要么是 (1,0) 要么是 (0,1)，并且差值正好是 1。当 N 等于 2 时，像 1 个奇数和 1 个偶数这样的配置是好的，而两个相同奇偶校验的配置也是好的，因为差值是 2 减 0 等于 2，这是不允许的。 所以“1 1”或“2 2”应该输出“Not Good”，而“1 2”应该输出“Good”。 粗心的实现可能会错误地只检查两个奇偶校验是否存在，而不是正确比较计数。 

## 方法

 思考这个问题的强力方法是将数组分成两组：奇数和偶数，然后通过多次扫描或过滤来显式计算这两个组。 人们可以想象迭代所有数字，并针对每个元素检查它是奇数还是偶数并递增计数器。 这已经足够了，但如果有人在嵌套循环内重复过滤数组或重新计算计数，则复杂性将降级为 O(N²)，因为数组的每次传递都是 O(N)，并且可能对每个元素重复。 

关键的观察结果是，没有任何东西取决于元素之间的顺序或关系。 每个数字都独立地贡献于两个计数器之一。 一旦我们意识到这一点，问题就会分解为单个线性扫描，其中我们维护两个整数：一个用于奇数计数，一个用于偶数计数。 处理完所有数字后，我们只是比较它们的差异。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N²) | O(1) | O(1) | 太慢了|
 | 最佳 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 初始化两个计数器，一个用于奇数，一个用于偶数，均设置为零。 这使我们能够在单次传递中积累频率。 
2. 迭代数组中的每个数字。 对于每个值，使用条件确定其奇偶性`value % 2 == 0`。 此操作是恒定时间，不依赖于输入大小。 
3. 如果数字是偶数，则增加偶数计数器； 否则增加奇数计数器。 这可以确保每个元素对正确的类别只贡献一次。 
4. 处理完所有数字后，计算两个计数器之间的绝对差。 
5. 如果该差值小于或等于1，则输出“Good”。 否则输出“Not Good”。 

### 为什么它有效

 输入中的每个元素恰好属于两个不相交集合之一：奇数或偶数。 该算法保持两组的精确计数，没有近似或重叠。 由于每个元素都被处理一次且仅处理一次，因此最终计数器是两个集合的精确基数。 决策规则仅取决于这些基数，因此正确性直接源于以下事实：计数是准确的，并且在分类之外不会执行数据转换。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    arr = list(map(int, input().split()))
    
    odd = 0
    even = 0
    
    for x in arr:
        if x % 2 == 0:
            even += 1
        else:
            odd += 1
    
    if abs(odd - even) <= 1:
        print("Good")
    else:
        print("Not Good")

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 我们读取所有值一次，然后维护两个计数器。 奇偶校验使用模 2，这对于所有约束都是安全的，因为值最大为 10⁹。 最终比较使用绝对差，确保奇数和偶数计数之间的对称性。 

一个微妙的点是输入解析：我们在一次调用中将整个第二行读入列表中。 这是安全的，因为 N 最多为 10⁵，因此内存使用量可以忽略不计。

 ## 边缘情况

 ### 单元素输入

 输入：```
1
7
```该算法初始化 odd = 0，even = 0，然后读取奇数 7，因此 odd 变为 1。差值为 1，因此输出为“Good”。 这确认了最小输入大小仍然满足规则的边界条件。 

### 所有元素相同奇偶校验

 输入：```
6
2 4 6 8 10 12
```处理将偶数设置为 6，奇数保持为 0。绝对差为 6，超过 1，因此输出为“Not Good”。 该算法可以正确聚合计数，无需任何成对推理。 

### 接近平衡的情况

 输入：```
3
1 2 3
```奇数变为 2，偶数变为 1。差值为 1，因此输出为“Good”。 这证实了阈值条件是包含的并且正确处理`<= 1`。
