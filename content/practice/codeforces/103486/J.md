 ## 解决方案
 ## 问题理解

 我们得到了一副牌，其中包含$n \cdot m$不同的卡片标记为从 1 到$nm$。 最初，卡片从下到上按递增顺序排列，因此卡片 1 在底部，卡片$nm$位于顶部。 

然后重复应用洗牌操作。 每次洗牌时，首先将牌组切分为$n$连续大小的块$m$，保留每个块内的顺序。 然后使用这些块来构建$m$通过按顺序重复取出每个原始块的当前底牌来创建新的牌堆。 最后，这些$m$桩被堆叠起来形成新的甲板。 

任务是确定在牌组恢复到其原始配置之前必须应用此洗牌次数。 该过程保证最终循环，并且我们希望恢复初始顺序的最小正洗牌次数。 

约束条件$n \cdot m \le 10^{18}$立即排除对单个卡片甚至完整排列结构的任何模拟。 任何可行的解决方案都必须推理转换本身的结构而不是迭代它。 

微妙的一点是，shuffle 并不是像 riffle shuffle 那样简单的交错。 它以结构化网格状方式重新排列牌组，并且误解索引会导致基于模拟的方法不正确，即使对于中等大小（例如$n = m = 10^5$。 

## 方法

 直接模拟将跟踪所有$nm$卡片，将它们分成块，重建卡片堆，然后重复，直到原始顺序重新出现。 一次洗牌已经花费$O(nm)$操作，周期长度未知。 在最坏的情况下，这将需要多次重复，并且整个工作在限制下变得完全不可行。 

关键的观察结果是，洗牌并不取决于纸牌的实际值，而仅取决于它们的位置。 如果我们按照每张卡片在概念中的位置来索引$n \times m$网格中，洗牌定义了这些位置上的固定排列。 

让我们通过以下方式索引卡片$(i, j)$， 在哪里$i$是从底部开始的块索引（范围从1到$n$） 和$j$是块内的位置（范围从 1 到$m$）。 洗牌时，所有牌都具有相同的牌$j$跨越所有块并将它们分组到新的一堆中。 每堆内的顺序$i$被保留。 最后，将这些堆按升序堆叠起来$j$。 

这意味着一张牌位于位置$(i, j)$移动到仅通过交换坐标确定的位置。 仔细跟踪堆叠顺序后，它的新位置与该对完全对应$(j, i)$在新的安排中。 换句话说，shuffle 执行了转置$n \times m$甲板的网格表示。 

一旦识别出这种结构，问题就简化为研究通过交换坐标定义的排列。 应用两次随机播放会交换两次坐标，将每个元素返回到其原始位置。 因此，排列是对合，并且只要满足以下条件，其阶数恰好为 2：$n \cdot m \ge 2$。

 ## 算法演练

 我们将由一次洗牌引起的坐标变换形式化。 

### 算法演练

 1. 将牌组解释为$n \times m$网格所在位置$(i, j)$对应于$j$里面从底部开始的第一张牌$i$第-块。 该索引与牌组最初的分割方式相匹配。 
2. 跟踪单个元素的位置$(i, j)$洗牌期间移动。 它被放置到$j$-th 个新堆，并且在该堆中它占据$i$从底部算起第 - 个位置。 
3. 将此二维位置转换回新牌组中的线性索引。 新位置变为$(j - 1) \cdot n + i$，这对应于交换网格表示中的两个坐标。 
4. 得出结论，一次洗牌映射了每个位置$(i, j)$到$(j, i)$在变换后的布局中。 这是纯粹的转置操作。 
5. 在心里应用两次变换：$(i, j) \to (j, i) \to (i, j)$。 每个元素在恰好两次洗牌后返回到其原始位置。 

### 为什么它有效

 洗牌完全由不依赖于牌值的确定性位置规则决定。 因为每张牌的移动只取决于它的$(i, j)$坐标，整个操作是坐标对的排列。 该排列正是转置图，它是它自己的逆。 任何对合都具有 2 阶，除非它是恒等排列，这需要简并$1 \times 1$结构，排除约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        # For all valid inputs with nm >= 2, shuffle is a non-identity involution
        # so the order is always 2.
        print(2)

if __name__ == "__main__":
    solve()
```实现很简单，因为结构结果消除了任何每次测试的计算。 每个测试用例都会在恒定时间内得到回答。 

唯一的微妙之处是认识到不同形状的网格不需要特殊处理。 即使是高度不平衡的情况，例如$n = 1$或者$m = 1$仍然会产生非平凡的交​​换，除非乘积为 1（被约束排除）。

 ## 工作示例

 考虑$n = 2, m = 3$。 牌组被分成两个大小为 3 的块。将位置标记为$(i, j)$。 

经过一次洗牌后，映射为：

 | (i, j) | 新职位|
 | --- | --- |
 | (1,1) | (1,1) |
 | (1,2) | (2,1) |
 | (1,3) | (3,1) |
 | (2,1) | (1,2) |
 | (2,2) | (2,2) |
 | (2,3) | (3,2) |

 再次应用相同的变换后，每对都会返回到其原始位置。 这证实了对合结构。 

现在考虑$n = 1, m = 4$。 甲板是一个整体。 洗牌将元素重新组织成四堆，然后将它们堆叠回去。 映射仍然会交换坐标，因此应用两次会恢复原始顺序。 这表明即使是退化的单行或单列配置仍然具有 2 阶。

 ## 边缘情况

 一个潜在的担忧是极度不平衡的尺寸是否会产生不同的周期长度。 例如，$n = 1, m = 5$产生一个初始块。 即使在这里，shuffle 仍然会在$1 \times m$和$m \times 1$解释，并应用两次即可恢复原始顺序。 

另一种情况是$n = 2, m = 1$。 这副牌只有两张牌。 一次洗牌将它们交换，第二次洗牌将它们交换回来，循环长度为 2。 

最后一个问题是这种转变是否会意外地成为某些重要的身份$n, m$。 这需要$(i, j) = (j, i)$对于所有对，仅当每个位置在交换下固定时才成立，除非结构崩溃到一个点，否则不可能
