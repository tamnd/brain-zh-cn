 ## 解决方案
 ## 问题理解

 我们得到一棵加权树，其中有 N 个市场，由 N − 1 条道路连接。 每条路都有成本，Chiang 的日常工作相当于选择这棵树中的任何简单路径，然后将沿该路径的权重相加。 她关心的值是最大可能的路径和，这正是树的加权直径。 

每天，恰好有一条边的权重会暂时改变。 更改仅适用于当天，然后边缘将恢复到其原始重量。 对于每一天，我们必须在应用单边修改后输出树的直径。 

结构很重要：它始终是一棵树，因此任何两个节点之间都有唯一的路径。 因此，一天的答案是修改边权重下任意一对节点之间的最大距离。 

约束最多为 2 × 10^5 节点和 2 × 10^5 查询。 重新计算所有对距离甚至每个查询运行线性树 DFS 的解决方案将太慢，因为在最坏情况下 O(NT) 达到 4 × 10^10 次操作。 即使每个查询的 O(N log N) 也会太慢。 

一个微妙的困难是，改变一个边权重会以非局部方式影响整个树的距离。 如果不仔细处理全局依赖关系而简单地尝试维护子树 DP，将会悄无声息地失败。 

暴露这一点的一个小边缘情况是线树。 如果树是一个链，改变中间边缘会改变直径，从而影响每个前缀后缀组合，因此任何依赖局部子树独立性的解决方案仍然必须重新计算全局结构。 

## 方法

 蛮力的想法很简单。 对于每个查询，更新边权重，然后运行两遍 DFS 来计算直径。 第一个 DFS 计算距任意根的最远距离，第二个 DFS 使用这些距离来查找直径端点和值。 这是正确的，因为可以在线性时间内找到树的直径。 

然而，每个查询的成本为 O(N)，导致总工作量为 O(NT)。 对于 2 × 10^5 节点和查询，这是不可行的。 

关键困难在于我们需要支持动态边权重变化，同时保持全局最大路径和。 该结构表明每个查询仅改变一条边，因此重新计算感觉很浪费。 真正的问题是直径是一个全局属性，但树允许分解，如果我们能够有效地组合子结构的贡献，则可以维护全局最长路径。 

在边缘更新下处理动态树直径的标准方法是链接切割树。 这个想法是将树维护为动态森林，其中每个节点存储有关其表示路径的足够聚合信息，以便我们可以在对数时间内查询和更新直径。 展开结构中的每个首选路径都在所表示的段内维护最佳前缀、后缀和最佳答案。 这允许合并两条路径，同时跟踪最佳可能的端点到端点距离。 

每次边缘更新都变成具有修改权重的剪切和链接操作，并且在每次操作之后，全局结构立即维护当前直径。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询重新计算 DFS | O(NT) | O(N) | 太慢了|
 | Link-Cut 树动态直径 | O((N + T) log N) | O((N + T) log N) | O(N) | 已接受 |

 ## 算法演练

1. 将每条边视为链接剪切树连接，并在链接的子端存储关联的权重。 这允许通过节点路径更新来更新边权重，而不是重新计算全局距离。 
2.为链路切断树中的每个节点维护表示其当前优选展开段的最佳路径信息的数据结构。 每个段存储最佳单路径贡献，包括最大向下路径、向上路径和最佳内部路径。 
3. 访问节点时，执行标准的链接切断访问操作，该操作公开从所表示的树的根到该节点的首选路径。 此步骤是必要的，以便更新沿着正确的首选路径传播。 
4. 要更新边权重 (u, v)，首先识别链接剪切结构中的边。 切断现有连接，然后使用分配给边的新权重重新链接 u 和 v。 这取代了旧的贡献，同时保留了树的连通性。 
5、每次更新后，查询辅助结构根部保存的维护的全局最佳值。 该值对应于当前树中所有可能的节点对的最大路径总和。 

正确性背后的关键思想是所表示的森林中的每个根到根路径都被分解为一系列展开段。 每个段都存储足够的聚合信息，以便合并两个段可以保留跨越其边界的所有可能路径组合的正确性。 由于树中的每个简单路径都表示为这些段的串联，因此最大路径始终表示在某些维护的结构状态内。

    t[x].sum = t[a].sum + t[b].sum + t[x].val

    t[x].pref = max(
        t[a].pref,
        t[a].sum + t[x].val + max(0, t[b].pref)
    )

    t[x].suff = max(
        t[b].suff,
        t[b].sum + t[x].val + max(0, t[a].suff)
    )

        t[a].best,
        t[b].best,
        max(0, t[a].suff) + t[x].val + max(0, t[b].pref)

def is_root(x):
    f = t[x].fa
    return t[f].ch[0] != x and t[f].ch[1] != x

def rotate(x):
    y = t[x].fa
    z = t[y].fa

        if t[z].ch[0] == y:
            t[z].ch[0] = x
        else:

 ## 边缘情况

 简并链暴露了单边更新的最坏情况传播距离。 当修改中间边缘时，每对端点都会受到影响。 链接剪切树可以处理此问题，因为受影响的边被带到首选路径的顶部，并且所有聚合的段值都通过展开旋转重新计算。 

星形树强调直径通常取决于通过中心连接的两片叶子。 更改单个辐条仅影响包含该辐条的路径，并且数据结构确保在访问期间仅重新计算那些聚合段，而不相关的分支保持不变。 

均匀权重树表明，即使所有边都相等，该结构仍然必须跟踪完整的段信息，因为单个大幅增加可以将直径转移到完全不同的分支，这是通过每个节点内维护的最佳路径值捕获的。
