 ## 问题理解

 给定一个固定的 4×4 矩阵，其中除了一个依赖于整数变量的位置外，每个条目都是恒定的$x$。 对于每个整数$x$在范围内$[l, r]$，我们评估该矩阵的行列式，并要求找到整个范围内的最小值。 

行列式扩展为列排列上的有符号和，但对于 4×4 矩阵，它仍然是一个固定的代数表达式$x$。 由于只有一个条目依赖于$x$，行列式简化为线性函数$x$，因为行列式展开中的每一项只使用每一行一次，并且$x$恰好出现在一个位置，仅对包含该位置的那些排列产品做出线性贡献。 

对于在该范围内进行强力评估，输入大小约束非常大。 范围可达$10^5$，这立即排除了评估每个的行列式$x$如果每个评估都不平凡，则单独进行。 即使单个行列式计算是$O(1)$，对该范围进行全面扫描将是$O(r-l+1)$，这仍然可以接受，但是我们必须首先确认行列式求值本身是恒定时间的还是需要扩展。 

一个更微妙的担忧是，天真的符号推理可能会误导我们，认为行列式是非线性的。$x$，导致采样或局部最小化的错误尝试。 

常见的失败情况是假设凸性或单调性而不验证它。 例如，如果错误地假设函数是单调的并且仅检查端点，则他们可能会错过区间内的真实最小值。 由于行列式展开可以根据排列结构产生负斜率，因此除非证明线性，否则这种假设是不安全的。 

## 方法

 暴力方法计算每个整数的行列式$x \in [l, r]$。 每次计算都使用直接 4×4 行列式公式或高斯消去法，其大小恒定，因此时间恒定。 这会产生一个$O(r-l+1)$解决方案，在最坏的情况下是$10^5$。 通过严格的实现，这已经是临界点，但在 Python 中是可以接受的。 

然而，行列式的结构使得更强的事情成为可能。 由于只有一个条目包含$x$，行列式展开式中的每个排列项要么包含该条目一次，要么根本不包含该条目。 不涉及它的术语是不变的。 涉及它的项贡献系数倍$x$。 这意味着行列式精确地简化为线性函数$f(x) = ax + b$。 

一旦我们认识到线性，问题就变得微不足道：在整数区间内最小化线性函数总是发生在端点之一。 这样就无需扫描整个范围。 

关键的转变是认识到行列式的多重线性性保证了当所有其他条目都固定时对任何单个条目的线性依赖。

 | --- | --- | --- | --- |
 | 蛮力 |$O(r-l+1)$|$O(1)$| 已接受 |
 | 最佳 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 我们通过显式计算行列式作为以下函数：$x$，或更直接地在两个端点对其进行评估。 

1. 观察该矩阵与常量矩阵的区别仅在于一项，因此行列式取决于$x$以结构化的方式。 
2. 计算行列式$x = l$使用固定的 4×4 行列式公式。 这给了一个候选人价值。 
3. 计算行列式$x = r$以同样的方式产生第二个候选人。 
4. 比较两个结果，返回较小的一个。 

每次评估都是恒定时间的，因为矩阵大小永远不会改变。 

### 为什么它有效

 行列式在矩阵的行和列中是多重线性的。 当除了一个标量位置之外的所有条目都固定时，行列式就成为该标量的仿射函数。 因此函数结束$x$是线性的$x$，这意味着它在整数区间内没有内部极值。 任何最小值必须出现在边界点处$l$或者$r$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def det4(a):
    # direct expansion for 4x4 determinant (hardcoded cofactor expansion)
    # a is 4x4 list
    def det3(m):
        return (
            m[0][0]*(m[1][1]*m[2][2] - m[1][2]*m[2][1])
            - m[0][1]*(m[1][0]*m[2][2] - m[1][2]*m[2][0])
            + m[0][2]*(m[1][0]*m[2][1] - m[1][1]*m[2][0])
        )

    res = 0
    for j in range(4):
        # build 3x3 minor
        m = []
        for i in range(1, 4):
            row = []
            for k in range(4):
                if k != j:
                    row.append(a[i][k])
            m.append(row)

        sign = -1 if j % 2 else 1
        res += sign * a[0][j] * det3(m)
    return res

def build_matrix(x):
    return [
        [6, 5, -1, -3],
        [3, x, 6, 7],
        [4, -5, 7, 8],
        [4, 3, 9, 1]
    ]

l, r = map(int, input().split())

dl = det4(build_matrix(l))
dr = det4(build_matrix(r))

print(min(dl, dr))
```该实现使用直接辅因子扩展对行列式求值两次。 辅助函数构造给定的矩阵$x$，确保清晰度并避免符号代数。 

唯一的微妙之处是确保拉普拉斯展开式中正确的符号处理。 符号按列索引交替，从第 0 列开始为正。

 ## 边缘情况

 一种边缘情况是当$l = r$。 该算法仍然在同一点计算两个行列式，有效地重复工作但保留了正确性。 例如，输入`5 5`对同一矩阵求值两次并返回相同的值，因此不需要特殊的分支。 

另一种情况是负值$x$。 由于行列式使用$x$仅作为标量乘法器，负输入不会影响计算的正确性，只会影响中间结果的符号。 例如，如果$x = -3$，矩阵变为：$$\begin{bmatrix}
6 & 5 & -1 & -3 \\
3 & -3 & 6 & 7 \\
4 & -5 & 7 & 8 \\
4 & 3 & 9 & 1
\end{bmatrix}$$相同的行列式例程无需修改即可应用，并且端点评估仍然正确捕获最小值，因为线性适用于所有整数，而不仅仅是正数。
