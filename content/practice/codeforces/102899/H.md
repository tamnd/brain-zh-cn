 ## 解决方案
 ## 问题理解

 我们得到一个整数列表，代表法官分配的分数。 所有值均非零且互不相同。 我们可以删除这些数字中的一个。 删除它后，我们将所有剩余的数字相乘，该乘积即为最终分数。 任务是选择要删除的单个数字，以使所得的乘积尽可能大，并输出删除的数字本身。 

关键的困难在于乘积同时取决于所有剩余元素，因此删除一个元素会影响结果的大小和符号。 对于多达 30,000 个数字，任何为每次删除从头开始重新计算乘积的方法都必须仔细考虑算术成本，因为重复长乘积的简单乘法太慢。 

一个微妙的方面来自于符号行为。 该产品可以根据剩余的负数数量翻转符号。 删除负数可能会改变负数的奇偶性，删除正数对幅度的影响与删除负数不同。 这使得纯粹根据“删除最小”或“删除最大”进行推理是不正确的。 

打破天真的直觉的边缘情况包括：

 例如，如果恰好有一个负数`[-5, 2, 3, 4]`，去掉负数会使乘积变得正且大，但是去掉正数仍然留下负乘积。 贪婪的“删除最小”规则可能会失败，具体取决于符号处理。 

如果有很多负面因素，例如`[-4, -3, -2, -1, 10]`，去除一个负值可能会改变奇偶校验并以竞争的方式彻底改变符号和幅度。 

因为所有数字都是非零且不同的，所以我们从不处理联系或零乘积崩溃，这简化了关于单调变化的推理。 

## 方法

 暴力方法将计算除每个可能的删除的一个候选索引之外的所有数字的乘积。 对于每个 i，我们将除`a[i]`并比较结果。 这是正确的，因为它直接评估问题的定义。 然而，每次评估的成本为 O(n)，并且对所有 n 个选择执行此操作会导致 O(n²) 乘法。 当 n 达到 3·10⁴ 时，这大约变成 9·10⁸ 乘法，这在 Python 中太慢了，即使在优化的语言中也是如此。 

关键的观察结果是，我们实际上并不需要重复使用完整的产品。 如果我们将所有元素的乘积视为基值，则删除`a[i]`相当于将总产品除以`a[i]`。 挑战在于产品可能会溢出，并且必须谨慎处理符号比较。 但从逻辑上讲，每个候选人的结果都与`P / a[i]`， 在哪里`P`是所有数字的乘积。 

我们可以通过推理删除每个元素如何改变乘积来比较候选者，而不是直接使用除法和巨大的数字。 由于所有值都不为零，因此比较`P / a[i]`跨越不同的 i 相当于比较`1 / a[i]`就同​​一基础产品的乘数效应而言。 然而，由于符号和幅度相互作用，更安全的方法是使用 Python 的大整数计算一次总乘积，然后通过整数除法计算每个候选值。 

这将问题简化为 O(n) 计算：一次完整的产品传递，一次评估每个移除。

 ## 算法演练

 ### 最优策略

 1. 计算数组中所有数字的乘积。 这给出了代表任何删除之前的完整乘法的基线值。 
2. 迭代每个元素`a[i]`并通过将总产品除以计算去除它的值`a[i]`。 这给出了该移除的结果分数。 
3. 跟踪迄今为止看到的最大结果乘积并存储相应的索引。 
4.检查完所有元素后，输出移除后产生最大乘积的元素。 

关键的决策点是我们永远不会从头开始重新计算完整的产品。 我们依赖于代数关系，即从乘积中删除一个因子相当于除以该因子。 

### 为什么它有效

 每个有效结果都准确对应`P / a[i]`， 在哪里`P`是所有元素的乘积。 自从`P`在所有选择中都是恒定的，最大化`P / a[i]`over i 相当于选择相同基量的最佳变换。 不会错过任何候选者，并且每个候选者都会被精确评估一次，从而保持正确性。 唯一的微妙之处在于，整数算术在 Python 中保留了精确值，因此比较是可靠的，不会出现浮点错误。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    a = list(map(int, input().split()))
    
    total = 1
    for x in a:
        total *= x
    
    best_val = None
    best_idx = 0
    
    for i, x in enumerate(a):
        val = total // x
        if best_val is None or val > best_val:
            best_val = val
            best_idx = i
    
    print(a[best_idx])

if __name__ == "__main__":
    main()
```该解决方案首先一次性构建总产品。 这是安全的，因为 Python 整数会自动扩展到任意精度，因此溢出不是问题。 

然后，它通过将总乘积除以当前元素来评估每次移除。 整数除法是精确的，因为每个元素都是总乘积的一个因数。 比较步骤很简单：我们保留产生最大商的索引。 

一个常见的实现错误是尝试重新计算每个索引的乘积或尝试通过部分重新计算来模拟乘法，这会导致 O(n²)。 另一个微妙的错误是试图用浮点除法进行推理，当乘积变大时，这可能会引入精度错误。

 ## 边缘情况

 考虑`[-1, 2]`。 去除`-1`树叶`2`，同时去除`2`树叶`-1`。 该算法计算总乘积`-2`，然后评估移除：移除`-1`给出`2`，去除`2`给出`-1`。 最大值被正确识别为移除`-1`。 

在`[-4, -3, -2, -1, 10]`，总乘积为正。 去除不同的负值或单个正值会产生不同的量值。 该算法明确地比较精确的商，因此它自然地选择符号稳定性和幅度增加之间的最佳平衡，而不需要单独的奇偶校验逻辑。 

在`1 2`，去除`1`产量`2`并删除`2`产量`1`。 该算法正确地识别了较小元素的删除，因为它最大化了剩余的乘积。
