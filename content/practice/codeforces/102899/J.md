 ## 解决方案
 ## 问题理解

 我们得到一行文本，代表一个句子，在不同的上下文中可能包含单词“is”。 任务是扫描这个句子，并仅当小写单词“is”作为两侧被空格包围的独立标记出现时才替换它。 任何其他形式必须保持不变，包括大写字母的“Is”、行首或行尾的“is”、或附加标点符号或其他字符的“is”。 

输入大小最多可达 100,000 个字符，这立即表明，如果以二次方式简单地实现，任何涉及重复字符串连接或子字符串重复全局替换的解决方案都将存在风险。 需要线性扫描，因为任何低效地重复搜索或重建字符串的方法在最坏的情况下（当发生多次替换时）都可能降级为 O(n²)。 

从“被空间包围”的定义中自然会出现一些边缘情况。 如果字符串以“is”开头，则不应替换它，因为没有前导空格。 例如，输入“is a book”必须保持不变。 同样，如果“is”像“this is”一样出现在末尾，则仅当其右侧后面有空格时才会被替换，这意味着尾随位置也是敏感的。 另一个微妙的情况是标点符号邻接：“is”或“(is)”不得被替换，因为该单词不受空格严格限制。 

如果单词之间出现多个空格或者标点符号附加到标记上，则按空格分割并重建句子的简单方法也会失败，因为它有丢失原始格式的风险。 

## 方法

 暴力破解的想法是重复搜索字符串中的子字符串`" is "`并将其替换为`" was "`。 这在逻辑上是正确的，因为它直接编码了两边都有空格的条件。 然而，每次替换都可能会移动字符串并需要重建它，并且如果使用重复实现`find`和切片，每个操作都需要 O(n)。 在频繁发生替换的最坏情况下，这会导致 O(n²) 行为。 

更可靠的观察是，我们永远不需要多次修改字符串。 在检查本地上下文时，每个字符可以从左到右处理一次。 在任意位置，我们只需要判断当前三字符窗口是否形成`" is "`有适当的界限。 如果是的话，我们发出`" was "`而不是复制原始子字符串。 否则，我们按原样复制当前字符并继续前进。 这将问题减少到单次通过，每个位置进行恒定时间检查。 

关键的结构特性是替换“is”的决定仅取决于它的直接邻居。 不需要全局信息，因此流式构建就足够了。

 ## 算法演练

 1.将输入字符串读入变量`s`。 我们将逐步构建答案而不是修改`s`就位，因为字符串在 Python 中是不可变的。 
2.初始化一个空列表`out`有效地存储输出字符。 使用列表可以避免重复的字符串连接成本。 
3. 使用索引迭代字符串`i`从`0`到`len(s) - 1`。 
4. 各位置`i`，检查是否可以更换。 条件要求：

 子串开始于`i`正是`"is"`，并且其前后各有一个空格，意思是`s[i-1] == ' '`和`s[i+2] == ' '`。 

此检查确保我们仅替换孤立的单词，而不替换其他单词的一部分或标点符号附加的标记。 
5. 如果条件成立，则附加字符串`"was"`到输出列表，然后前进`i`2，因为我们已经消耗了两个字符。 
6. 如果条件不成立，则追加`s[i]`到输出列表并前进`i`1.
 7. 完成循环后，将列表连接成最终字符串并打印它。 

### 为什么它有效

 正确性来自于每个决定都是局部的且不可逆转的。 每当我们更换`" is "`和`" was "`，我们保证以后不会影响重叠或部分匹配，因为只有当周围的字符都是空格时才会发生替换。 这可以防止相邻匹配之间的冲突。 每个字符作为匹配或不匹配情况的一部分被检查一次，确保完全覆盖，没有重复或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().rstrip('\n')
    n = len(s)
    out = []
    i = 0

    while i < n:
        if i > 0 and i + 2 < n:
            if s[i:i+2] == "is" and s[i-1] == ' ' and s[i+2] == ' ':
                out.append("was")
                i += 2
                continue

        out.append(s[i])
        i += 1

    print("".join(out))

if __name__ == "__main__":
    solve()
```该实现依赖于每个候选位置周围的滑动窗口检查。 关键的微妙之处是确保边界安全：条件`i > 0 and i + 2 < n`检查邻居时防止超出范围的访问。 循环进度根据是否发生替换而有所不同，这避免了对匹配段内的字符进行双重处理。

 ## 边缘情况

 一个关键的边缘情况是“is”出现在最开始的时候。 例如，输入“is here”应保持不变，因为前面没有空格。 在索引 0 处迭代期间，边界检查`i > 0`立即失败，因此算法安全地复制字符而不尝试匹配。 

另一种情况是诸如“this is”之类的尾随出现。 当扫描到达最后一个有效位置时，条件`i + 2 < n`防止读取超出字符串范围的内容，因此不会发生无效访问，并且最后的“is”仅在后面跟有空格时才会被替换，但事实并非如此。 输出仍然正确。 

第三种情况涉及标点符号邻接，例如“is,”。 即使子字符串以“is”开头，但其后面的字符是逗号，而不是空格，因此替换条件失败。 该算法准确地保留了原始文本，表明严格的边界检查正确地强制执行了独立单词的定义。
