 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，都有多个“防御会话”。 在每个会话中，都会出现一个学生列表，每个学生都有一个综合分数，该分数计算为两个部分的总和。 

一名特殊学生 KK 在每节课中出现一次，并在该节课中获得分数。 任何其他学生可能会出现在一个或多个课程中，每次可能会有不同的分数。 

如果该学生每次出现在任何会话中，其在该会话中的分数严格低于 KK 在同一会话中的分数，则该学生被视为“受 KK 支配”。 如果学生在一节课中达到或超过 KK，则不计算在内。 

任务是计算有多少不同的学生满足这个条件。 

输入的结构规模较小：每个测试用例最多有10个会话，每个会话最多有10人。 这立即排除了除了简单的二次或线性扫描之外的任何事情。 即使对所有记录进行简单的嵌套扫描，每个测试用例最多也需要几千次操作，这在 1 秒的限制下是微不足道的。 

主要的微妙之处在于，学生在课程中多次出现，其有效性取决于所有出现的全局条件。 一个天真的错误是只检查学生的第一次出现，或者与单个 KK 值而不是特定于会话的 KK 值进行比较。 

具体的失败案例如下所示：

 第1节：KK评分为100，Alice评分为90

 第2节：KK得分为50，Alice得分为60

 如果我们只比较 Alice 一次，她可能看起来很好，但她违反了会话 2 中的规则，因为她在那里并不严格低于 KK。 所以爱丽丝必须被排除在外。 

另一个边缘情况是学生只出现一次。 如果他们在该会话中的分数低于 KK，则它们自动有效，但前提是确保不存在隐藏的后续违规行为（这需要跟踪所有发生的情况）。 

## 方法

 一个强力的解释是：对于每个学生的名字，收集他们出现的所有课程，并将每个课程的分数与 KK 在该课程中的分数进行比较。 如果所有比较都通过，则计算该学生。 

这是正确的，但如果我们通过扫描每个学生的整个数据集来简单地实现它，我们可能会重复遍历所有条目。 在最坏的情况下，我们可以检查每个学生多达 100 个条目，从而导致多余的重复工作。 

关键的观察结果是我们不需要针对每个学生重新计算。 相反，我们可以处理每条记录一次，维护 KK 每节课的分数，并立即使任何违反条件的学生无效。 这将问题转化为名称上的单遍聚合问题。 

我们维护一个字典，记录每个学生是否仍然有效。 当我们阅读每个会话时，我们会将每个参与者与该会话的 KK 分数进行比较，并立即更新他们的状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个学生扫描的暴力破解 | 每个测试用例 O(N²) | O(N) | 太慢（不必要的开销）|
 | 单次验证 | 每个测试用例 O(N) | O(N) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 读取会话数。 对于每个会话，我们首先确定 KK 的分数，因为 KK 始终显示为会话的第一个条目。 这为我们提供了每个会话的基准。 
2. 创建字典`valid[name] = True`用于跟踪每个学生是否仍有资格被计算在内。 
3. 对于会话中的每个参与者，将他们的分数与 KK 在该会话中的分数进行比较。 如果学生不是 KK，但其分数大于或等于 KK 的分数，我们会立即将其标记为无效。 
4. 继续处理所有课程，确保一旦学生被标记为无效，无论未来的课程如何，他们都保持无效。 
5. 处理完所有会话后，计算有多少不同名称（不包括 KK）仍然被标记为有效。 

关键的设计选择是立即失效。 这可以避免存储完整的每个会话历史记录。 

### 为什么它有效

 对于每个学生来说，我们检查的条件是对其所有表现的普遍约束：每个观察到的分数必须严格低于 KK 在相应课程中的分数。 该算法通过独立检查每个事件并在第一次失败时失效来强制执行此操作。 由于无效是单调的且永远不会逆转，因此当且仅当所有比较都通过时，学生才会在最后被标记为有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

T = int(input())
for _ in range(T):
    n = int(input())
    
    valid = {}
    kk_name = None

    for _ in range(n):
        a = int(input())
        
        session = []
        for i in range(a):
            s, b, c = input().split()
            b = int(b)
            c = int(c)
            total = b + c
            
            if i == 0:
                kk_name = s
                kk_score = total
            
            if s not in valid:
                valid[s] = True
            
            if s != kk_name:
                if total >= kk_score:
                    valid[s] = False

    ans = 0
    for name, ok in valid.items():
        if name != kk_name and ok:
            ans += 1

    print(ans)
```该代码依赖于每个会话中的第一个条目对应于 KK 的保证。 这使我们能够在处理该会话中的其他参与者之前捕获 KK 的分数。 

一个微妙的点是我们永远不需要存储每个会话的历史记录。 我们只存储每个名称的当前有效性状态，这使得内存使用量最小且逻辑简单。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    T = int(input())
    out = []
    for _ in range(T):
        n = int(input())
        valid = {}
        kk_name = None

        for _ in range(n):
            a = int(input())
            for i in range(a):
                s, b, c = input().split()
                b = int(b)
                c = int(c)
                total = b + c

                if i == 0:
                    kk_name = s
                    kk_score = total

                if s not in valid:
                    valid[s] = True

                if s != kk_name and total >= kk_score:
                    valid[s] = False

        ans = sum(1 for k,v in valid.items() if k != kk_name and v)
        out.append(str(ans))

    return "\n".join(out)

 ## 边缘情况

 一个重要的边缘情况是，学生仅出现在 KK 分数非常低的课程中。 该算法可以正确处理此问题，因为每个会话都是独立检查的，因此即使单个高分会话也会使学生无效。 

另一种情况是同一学生在一次会议中重复出现。 由于我们独立处理每一行，因此该会话内的任何违规都会立即翻转其有效性，并且后续发生的情况不会更改结果。 

最后一个微妙的情况是 KK 被​​重复用作其他学生的名字。 该代码通过名称比较显式排除 KK，确保仅使用真正的 KK 基线进行比较。
