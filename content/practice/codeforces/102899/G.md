 ## 问题理解

 我们得到了多个测试用例。 每个测试用例描述了一系列$n$不同的整数，这是一个排列$1 \ldots n$。 该序列表示舞者在观察者面前经过的顺序。 在观察的瞬间，舞者排列成一个圆圈，并顺时针或逆时针移动。 由于这种圆周运动，观察者看到的是圆的连续部分，但根据方向可以在任一方向读取该部分。 

任务是确定观察到的序列是否来自于圆形排列$1 \ldots n$并按顺时针顺序或逆时针顺序连续读取所有元素。 换句话说，我们被问到排列是恒等序的循环旋转还是逆序的循环旋转。 

约束条件$n \le 10^4$每个测试用例且最多 10 个测试用例意味着总输入大小约为$10^5$。 这排除了任何$O(n^2)$每个测试用例的比较作为边界，但仍然可能仅在小常数的情况下可接受。 然而，该结构表明更简单的线性检查就足够了。 

这里经常发生一种天真的误解：人们可能认为任何可以重新排序成循环的排列都是有效的，但条件更严格。 围绕圆圈的序列必须是单调的，没有跳跃。 

一些微妙的边缘情况说明了这一要求。 

考虑$n = 4$， 顺序$[1, 3, 2, 4]$。 这不是循环的单向遍历，因为从1到3后，我们跳过2然后返回，这打破了循环上的邻接。 

考虑$n = 5$， 顺序$[3, 4, 5, 1, 2]$。 这是有效的，因为它是排序顺序的旋转。 

考虑$n = 4$， 顺序$[4, 3, 2, 1]$。 这也是有效的，因为它对应于反向遍历。 

关键属性是邻接模$n$。 

## 方法

 蛮力的想法是尝试循环中每个可能的起点，并尝试以升序或降序模匹配序列$n$。 对于每个起始索引，我们模拟遍历并检查排列是否对齐。 每次模拟费用$O(n)$，并且有$n$起始位置，导致$O(n^2)$每个测试用例。 和$n = 10^4$，这就变成了$10^8$每个测试用例的操作，对于时间限制来说太慢了。 

问题的结构消除了尝试所有旋转的需要。 如果序列有效，则每个连续的对都必须代表循环上的邻居。 这意味着一旦我们使用第一步确定了方向，所有后续步骤都会被强制。 唯一不明确的是方向：升阶或降阶模$n$。 

因此，我们不是检查所有旋转，而是检查排列是否与 +1 步序列模一致$n$（带环绕）或 -1 步序列模$n$。 这将问题简化为两次线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 最佳 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们将数字视为长度周期上的点$n$，之后$n$1 出现且 1 出现之前$n$。 

1. 读取序列并计算取模后连续元素之间的差异$n$。 我们不是原始减法，而是将邻接性标准化为两个连续数字在循环上是否精确地向前相差 1 或向后相差 1。 这意味着对于相邻值$a$和$b$，我们检查是否$b = a + 1$,$b = a - 1$，或包装盒$(a, b) = (n, 1)$或者$(1, n)$。 
2. 从第一对确定方向。 如果第一个转换增加 1（包括从$n \to 1$），我们假设向前方向测试整个序列。 如果它减少 1（包括从$1 \to n$），我们测试向后方向。 如果两者都不成立，答案立即是否定的。 
3. 对于所选方向，从左到右扫描序列并验证每个相邻对都遵循相同的步骤规则。 所有转换的方向必须保持一致。 
4. 如果所有转换都匹配向前或向后方向，则输出 YES，否则输出 NO。 

### 为什么它有效

 有效的循环遍历具有严格的局部结构：每个元素在循环上恰好有两个邻居。 一旦我们观察到两个连续的元素，我们就可以确定正在使用哪个邻居关系。 因为循环是均匀的，所以这个局部决策决定了全局结构。 如果任何后续转换偏离此邻接模式，则该序列不能对应于任何方向上循环的连续遍历。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(a):
    n = len(a)

    def ok_forward():
        for i in range(n - 1):
            if a[i + 1] != a[i] + 1 and not (a[i] == n and a[i + 1] == 1):
                return False
        return True

    def ok_backward():
        for i in range(n - 1):
            if a[i + 1] != a[i] - 1 and not (a[i] == 1 and a[i + 1] == n):
                return False
        return True

    return ok_forward() or ok_backward()

t = int(input())
for _ in range(t):
    n = int(input())
    a = list(map(int, input().split()))
    print("YES" if check(a) else "NO")
```该实现直接对循环的两种可能的遍历进行编码。 前向检查强制每一步递增 1，除了特殊的换行情况$n$回到$1$。 向后检查对称地强制递减，并换行$1$到$n$。 

一个微妙的点是我们不会尝试“旋转”序列。 旋转是无关紧要的，因为在旋转下邻接性得以保留。 无论起始点如何，任何有效的循环遍历都将满足邻接属性。 

## 工作示例

 ### 示例 1：$[1, 2, 3, 4]$| 我| 一个[我] | 一个[i+1] | 检查 |
 | ---| ---| ---| ---|
 | 0 | 1 | 2 | +1 有效 |
 | 1 | 2 | 3 | +1 有效 |
 | 2 | 3 | 4 | +1 有效 |

 前向检查完全通过，因此输出为 YES。 

向后检查在第一次转换时立即失败，因为 2 不是 0 或从 1 换行。 

这展示了一种在一个方向上的干净的单调遍历。 

### 示例 2：$[3, 2, 1, 4]$| 我| 一个[我] | 一个[i+1] | 检查 |
 | ---| ---| ---| ---|
 | 0 | 3 | 2 | -1 有效 |
 | 1 | 2 | 1 | -1 有效 |
 | 2 | 1 | 4 | 换行有效 |

 向后方向始终成立，因此输出为 YES。 

这证实了接受相反顺序的轮换。

 ## 边缘情况

 像这样的最小情况$n = 1$总是会通过，因为单个元素很容易形成向前和向后循环。 

为了$n = 2$， 两个都$[1, 2]$和$[2, 1]$是有效的，因为每一步都是两节点循环上的有效邻居移动。 该算法接受两者，因为每个方向检查都会立即成功。 

对于重环绕序列，例如$[n, 1, 2, \ldots, n-1]$，前向检查验证从$n \to 1$并始终如一地继续。 向后检查立即失败，但只需要一个方向。 

对于混合序列，例如$[1, 3, 2, 4]$，第一个转换可能会建议一个方向，但第二个转换违反了邻接性。 扫描会在第二次比较时立即检测到这一点，从而防止任何错误接受。
