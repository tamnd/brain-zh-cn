 ## 解决方案
 ## 问题理解

 我们有一个用网格表示的矩形大厅。 Kauã 从左上角的单元格开始，想要到达右下角的单元格。 有些牢房设有警卫，每个警卫都以一个直线方向监视每个牢房，直到另一个警卫挡住视线。 卡乌阿不能踏入警卫牢房或任何有警卫看守的牢房。 任务是找到通过剩余单元格的最短路线并用`X`。 如果不存在路径，则答案是`-1`。 

输入给出网格的高度和宽度，后跟网格本身。 空单元格表示为`.`，而四个箭头字符代表警卫朝四个基本方向之一看。 输出是带有标记的最短有效路径的修改网格或`-1`。 

到2000年，网格尺寸可以达到2000，这意味着可以有多达400万个单元。 探索每条可能路径的算法是不可能的，因为路径的数量呈指数增长。 即使是具有单元数量因素的算法，例如`O(n * m)`、 是适当的，因为它们仅接触每个单元格恒定的次数。 

棘手的部分是，牢房可能是空的，但仍然无法使用，因为警卫看到了它。 仅将警卫位置视为已封锁的解决方案将错误地穿过受监视的牢房。 

例如，考虑：```
3 4
....
.>..
....
```细胞`(2,3)`和`(2,4)`被警卫监视。 正确的输出路径无法使用它们。 一个粗心的 BFS，只会避免`>`会错误地认为它们可用。 

另一种边缘情况是当一个守卫被另一个守卫阻挡时：```
3 5
.....
.>>..
.....
```第二个守卫挡住了第一个守卫的视线。 第一个守卫无法看穿第二个守卫，因此第二个守卫之后的单元仍然可用。 将整行标记为危险会删除有效路径。 

最后一种情况是开始或结束被监视的单元格包围：```
3 3
.>.
...
```该算法仍必须从一开始就正常运行BFS，并发现如果所有可能的路由都被阻塞，则目的地不可达。 

## 方法

 一种直接的方法是尝试从起始单元开始的所有可能的移动序列，并保留到达目的地的最短的移动序列。 这是正确的，因为每个探索的序列都代表一条可能的路线。 然而，可能的路线数量可能是巨大的。 在具有数百万个单元的网格中，重复探索路径会产生太多操作。 

问题的结构给了我们两个独立的任务。 首先，确定哪些单元格是禁止的。 守卫不会移动，因此在搜索路径之前可以对其可见性进行一次处理。 之后，剩下的问题是网格上的标准最短路径问题，其中每次移动都有相同的成本。 

使这种方法变得高效的观察结果是，未加权网格上的 BFS 会自动找到最短路径。 一旦被监视的小区转换为阻塞的小区，每个合法的移动都有相同的成本，因此BFS第一次到达目的地时它就找到了最佳路线。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 细胞数量呈指数增长 | O(n × m) | 太慢了 |
 | 最佳 | O(n × m) | O(n × m) | 已接受 |

 ## 算法演练

 1. 扫描每个警卫并将该警卫可见的所有单元标记为危险。 朝守卫的方向移动，直到到达网格或另一个守卫的边缘。 运动在警卫处停止，因为一名警卫挡住了另一名警卫的视线。 
2. 运行 BFS`(0, 0)`。 只移动到网格内、非守卫且未标记为危险的单元格。 存储每个访问位置的前一个单元格，以便可以重建最终路线。 
3.如果BFS永远达不到`(n - 1, m - 1)`， 打印`-1`。 访问过的数组表示通过有效移动可到达的每个单元格，因此未访问过的目的地证明不存在有效的路线。 
4. 如果到达目的地，则沿着父指针向后移动，直到到达起点。 标记该链上的每个单元格`X`。 

为什么有效：在第一阶段之后，Kauã 无法进入的每个单元都已被排除在外。 剩下的网格正是可能的运动图。 BFS 逐层探索该图，因此当第一次到达某个单元格时，会通过尽可能短的移动序列到达该单元格。 父指针存储该最短路径，并且重建它们会产生有效的最短路径。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]

    blocked = [[False] * m for _ in range(n)]

    directions = {
        '>': (0, 1),
        '<': (0, -1),
        'v': (1, 0),
        '^': (-1, 0)
    }

    for i in range(n):
        for j in range(m):
            if grid[i][j] in directions:
                blocked[i][j] = True
                di, dj = directions[grid[i][j]]
                ni, nj = i + di, j + dj
                while 0 <= ni < n and 0 <= nj < m:
                    if grid[ni][nj] in directions:
                        break
                    blocked[ni][nj] = True
                    ni += di
                    nj += dj

    parent = [[None] * m for _ in range(n)]
    visited = [[False] * m for _ in range(n)]

    q = deque([(0, 0)])
    visited[0][0] = True

    moves = [(1, 0), (-1, 0), (0, 1), (0, -1)]

    while q:
        i, j = q.popleft()
        if (i, j) == (n - 1, m - 1):
            break

        for di, dj in moves:
            ni, nj = i + di, j + dj
            if 0 <= ni < n and 0 <= nj < m:
                if not visited[ni][nj] and not blocked[ni][nj]:
                    visited[ni][nj] = True
                    parent[ni][nj] = (i, j)
                    q.append((ni, nj))

    if not visited[n - 1][m - 1]:
        print(-1)
        return

    i, j = n - 1, m - 1
    while True:
        if grid[i][j] == '.':
            grid[i][j] = 'X'
        if (i, j) == (0, 0):
            break
        i, j = parent[i][j]

    for row in grid:
        print(''.join(row))

if __name__ == "__main__":
    solve()
```第一个循环计算危险单元格。 保卫单元本身被标记为阻塞，并且扫描仅继续穿过空单元。 当发现另一个警卫时停止可以正确处理可见性规则。 

BFS 部分使用队列，因为所有移动的成本相同。 这`parent`array 存储用于到达每个单元格的先前位置，这避免了存储完整路径并保持内存使用线性。 

在重建过程中，算法从目的地开始并重复跳转到前一个单元。 这遵循 BFS 发现的最短路径。 仅更改空单元格的条件可防止保护被覆盖。

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |

 | 空间| O(n × m) | 阻塞数组、访问数组和父数组分别存储每个单元格的信息。 |

 ## 边缘情况

 对于第一个可见性情况：```
3 4
....
.>..
....
```守卫在`(1,1)`标记`(1,2)`和`(1,3)`被阻止。 BFS 仍然可以通过顶部或底部在行中移动，因此该算法不会在不检查警卫视野的情况下错误地将所有空单元视为安全。 

对于阻塞情况：```
3 5
.....
.>>..
.....
```第一个警卫只能看到第二个警卫，因为第二个警卫停止了扫描。 第二个守卫之后的单元格没有标记。 如果它们通向符合实际可见性规则的目的地，BFS 阶段就可以使用它们。 

对于完全封锁的目的地：```
3 3
.>.
...
```危险单元在 BFS 之前计算。 如果删除所有可能的路线，则目的地永远不会进入队列，并且程序会打印`-1`。 搜索阶段永远不需要推理警卫，因为所有无效单元格都已被过滤掉。
