 ## 解决方案
 ## 问题理解

 该问题涉及使用一小组原始编辑操作将一个字符串转换为另一个字符串。 想象一下用一种“外语”字母书写的源文本和用另一种“外语”字母书写的目标文本。 您可以逐步修改源，直到它与目标相同，但每次修改都有成本，并且不同类型的编辑成本可能不同。 

在每一步中，您都可以插入一个字符、删除一个字符或用另一个字符替换一个字符。 任务是计算将第一个字符串转换为第二个字符串所需的最小总成本。 

输入由两个字符串和与每个编辑操作相关的成本描述组成。 输出是一个数字，表示使用任何允许的操作序列将第一个字符串转换为第二个字符串的最小可能成本。 

从复杂性的角度来看，两个字符串都可以足够长，任何二次或接近二次的结构都必须仔细证明。 如果两个字符串的长度都达到 2000 左右，则不可能对所有编辑序列进行简单的指数探索，因为即使每个位置分支为三个操作也会增长得非常快。 这立即表明存在重叠的子问题，必须重用而不是重新计算。 

一些边缘情况对于正确性很重要。 当一个字符串为空而另一个字符串不是时，答案就是插入所有字符或删除所有字符的成本。 例如，改造`"abc"`进入`""`应该花费三个删除，同时转换`""`进入`"abc"`应该花费三次插入。 简单的递归解决方案通常会忘记一致地考虑这些基本转换，从而在递归遇到空前缀但仍尝试不匹配的字符比较时导致部分结果不正确。 

当字符已经匹配时，会出现另一种微妙的情况。 例如，改造`"abc"`进入`"abc"`成本应该为零。 总是应用替换或删除操作而不检查相等性的粗心实现可能会人为地增加成本。 

## 方法

 蛮力的想法很简单：将问题视为探索将第一个字符串转换为第二个字符串的所有可能的编辑序列。 在前缀之间的每次不匹配时，您尝试插入一个字符、删除一个字符或替换当前字符，递归地解决所产生的子问题并在所有路径中采用最小成本。 

这种方法是正确的，因为它直接反映了允许的转换的定义。 然而，可能的序列数量呈指数增长。 如果字符串有长度`n`和`m`，然后从每个状态分支到最多三个新状态，导致最坏情况的复杂性约为`O(3^(n+m))`。 即使对于中等输入，这也变得完全不可行，因为相同的前缀对通过不同的操作序列被重新计算多次。 

关键的观察结果是，转换的状态完全取决于每个字符串的处理量。 如果我们定义一个函数`dp[i][j]`作为转换前缀的最小成本`s[:i]`进入`t[:j]`，那么每个转换仅依赖于较小的前缀。 这会产生重叠的子问题，因为相同的`(i, j)`可以通过许多不同的编辑序列来达到对。 一旦认识到这一点，问题自然会简化为所有前缀对的动态规划表。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(3^(n+m)) | O(3^(n+m)) | O(n+m) | 太慢了 |
 | 最佳DP | O(n × m) | O(n × m) | 已接受 |

 ## 算法演练

 我们构造一个 DP 表，其中每个条目表示将第一个字符串的前缀转换为第二个字符串的前缀的最小成本。 

1.定义一个2D表`dp`在哪里`dp[i][j]`表示转换第一个的最低成本`i`的字符`s`进入第一个`j`的字符`t`。 此公式之所以有效，是因为任何编辑操作仅影响所考虑前缀的最后一个字符。 
2. 初始化`dp[0][0] = 0`因为转换两个空字符串不需要任何操作。 
3. 填充第一行`dp[0][j]`通过插入的所有字符`t`逐个。 每个步骤都会增加插入成本，因为从空源到非空目标的唯一方法是重复插入。 
4. 填写第一列`dp[i][0]`通过删除所有字符`s`。 每一步都会增加删除成本，因为我们必须一个接一个地删除字符才能达到空目标。 
5. 对于每对`(i, j)`如果两个前缀都非空，请考虑最后一个字符`s[i-1]`和`t[j-1]`。 如果相等，则不需要对最后一个字符进行任何操作，因此最好的解决方案来自`dp[i-1][j-1]`。 
6. 如果字符不同，请考虑三种可能的操作。 正在删除`s[i-1]`导致`dp[i-1][j]`加上删除成本。 插入`t[j-1]`导致`dp[i][j-1]`加上插入成本。 更换`s[i-1]`和`t[j-1]`导致`dp[i-1][j-1]`加上重置成本。 我们采用这些选项中的最小值，因为任何最佳转换都必须以这些操作之一结束。 
7.最终答案是`dp[n][m]`，表示将完整源字符串转换为完整目标字符串的成本。 

### 为什么它有效

 DP 依赖于每个前缀对的不变量`(i, j)`在用于计算更大的前缀之前得到最佳解决。 任何有效的编辑序列转换`s[:i]`到`t[:j]`必须以三种方式之一结束：删除最后一个字符`s`，插入最后一个字符`t`，或匹配/替换最后一个字符。 由于所有较小的子问题都已得到最优计算，因此在这些转换中选择最小值可以保证全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    t = input().strip()

    ins_cost, del_cost, rep_cost = map(int, input().split())

    n, m = len(s), len(t)
    dp = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        dp[i][0] = i * del_cost

    for j in range(1, m + 1):
        dp[0][j] = j * ins_cost

    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if s[i - 1] == t[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = min(
                    dp[i - 1][j] + del_cost,
                    dp[i][j - 1] + ins_cost,
                    dp[i - 1][j - 1] + rep_cost
                )

    print(dp[n][m])

if __name__ == "__main__":
    solve()
```该解决方案读取字符串和操作成本，然后在所有前缀对上构建动态编程表。 第一行和第一列的初始化编码了转换空前缀的唯一可能的方法。 嵌套循环以增加前缀大小的方式计算转换，以便所有依赖项在需要时都可用。 

一个常见的实施陷阱是忘记平等捷径。 如果没有它，算法仍然有效，但会变得更慢，并且当字符已经匹配时可能会错误地增加替换成本。 另一个常见问题是索引未对齐，尤其是在访问时`s[i-1]`和`t[j-1]`。

 ## 边缘情况

 一个关键的边缘情况是一个字符串为空。 例如，转换`""`到`"abcd"`强制纯插入链。 DP 正确地将第一行填充为累积插入成本，并且没有任何转换使用替换或删除。 

另一种情况是相同的字符串，例如`"foreign"`→`"foreign"`。 字符匹配副本的每个对角单元格`dp[i-1][j-1]`，在整个对角线上传播零成本。 

第三种情况是替换比删除加插入便宜。 为了`"a"`→`"b"`成本删除 = 5，插入 = 5，替换 = 3，DP 直接通过以下方式选择替换`dp[1][1] = 3`，正确避免两步替代。
