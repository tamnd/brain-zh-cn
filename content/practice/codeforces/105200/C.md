 ## 解决方案
 ## 问题理解

 我们面临一个非常小的计算任务：一对整数代表一个分数，我们需要确定该分数是否至少是二分之一。 

您可以将其视为检查某些获得的值是否“不低于最大可能值的一半”。 输入描述分子和分母，输出是二元决策，通常根据问题的约定打印为是/否样式答案或 1/0。 

核心操作是比率和固定阈值之间的比较。 由于一切都简化为单个算术条件，因此该结构是每个测试用例的恒定时间。 

尽管任务看起来微不足道，但有一些失败模式经常出现在错误的提交中。 最常见的一种是浮点除法。 例如，比较`a / b >= 0.5`使用浮点运算可能会产生不正确的结果`a`和`b`很大，因为精度损失可能会翻转边界情况。 

另一个微妙的问题是整数除法。 如果有人写`a / b >= 1/2`在整数除法截断的语言中，表达式`a / b`对于许多有效输入来说变为零，这会立即破坏正确性。 

具体的边缘情况是指值恰好位于边界上。 例如，如果`a = 5`和`b = 10`，正确答案为真，因为它恰好是一半。 浮点方法在这里可能仍然有效，但是粗心的整数除法方法会错误地计算`5 / 10`作为`0`并且检查失败。 

推理所有这些的安全方法是完全避免除法并将不等式转换为等效的整数比较。 

## 方法

 蛮力解释将直接计算分数并将其与二分之一进行比较。 这意味着评估`a / b`作为浮点数并检查它是否至少`0.5`。 这在概念上是有效的，因为它直接匹配问题的定义。 

然而，这种方法在实施中变得脆弱。 浮点运算的精度有限，当`a`和`b`变大，可能会出现舍入误差。 在最坏的情况下，极其接近阈值的值可能会被错误分类。 即使时间复杂度是恒定的，在典型的竞争性编程约束下也不能保证正确性。 

关键的观察是，我们可以通过将不等式两边相乘来完全消除除法。 条件$$\frac{a}{b} \ge \frac{1}{2}$$可以重写为$$2a \ge b$$此转换在仅使用整数算术的同时保留了正确性。 它还完全避免了被零除的陷阱，因为我们从不除。 

一旦简化为这种形式，问题就变成了每个测试用例的单一比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（浮点除法）| O(1) | O(1) | O(1) | O(1) | 有风险|
 | 最优（整数不等式）| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取两个整数`a`和`b`来自输入。 它们代表我们正在计算的分数的分子和分母。 
2. 将条件“分数至少为一半”改写为整数不等式`2 * a >= b`。 这避免了任何划分并保持所有操作的精确性。 
3. 仅使用整数算术执行比较。 如果不等式成立，则分数至少为二分之一。 
4. 根据问题要求的格式输出相应的结果。 

### 为什么它有效

 正确性来自于不等式的等价变换。 将比较的两边都乘以正数可以保留顺序，因此乘以`2b`（或者只是小心地交叉相乘）不会改变陈述的真实性。 自从`b`在任何有效分数上下文中都假定为正，不会发生符号反转。 这可确保所有有效输入的整数条件与原始小数条件完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b = map(int, input().split())
    if 2 * a >= b:
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该程序读取一对整数并直接应用变换后的不等式。 乘法`2 * a`在Python中是安全的，因为Python整数在实践中是无界的，所以溢出不是问题。 

该决定是在恒定时间内做出并立即打印的。

 ## 边缘情况

 像这样的边界情况`1 2`证明了平等条件。 该算法计算`2 * 1 = 2`并将其与`2`，产生正确的 YES 输出。 这证实了平等处理得当。 

零分子的情况，例如`0 1`产生`2 * 0 = 0`，并且自从`0 >= 1`为假，则输出为NO。 这验证了算法不会错误地将零视为通过任何正阈值。 

分数超过一的情况，例如`10 1`，产生`2 * 10 = 20`，这基本上满足了条件。 这表明该方法甚至在典型的 [0,1] 范围解释之外也能工作。
