 ## 解决方案
 ## 问题理解

 该任务将竞争简化为单一比较。 每个参与者都记录了他们解决了多少问题，并且我们还获得了每个参与者完成解决会话或达到最终记录状态的时刻。 目标是在最早的决定性时刻确定谁的表现更强，纯粹通过解决问题的数量来衡量。 

实际上，我们正在比较两个性能快照序列。 每个序列描述了参与者随时间的进展。 关键思想是，只有一个参与者严格超越其他参与者的最早点才重要。 一旦一名参与者在给定时间指数内获得更高的解决计数，该参与者就被视为“赢得”了比武。 

尽管该语句非常短，但它隐含地假定了一个按时间排序的结构。 如果输入大小为 n，这意味着我们正在有效地扫描两个长度为 n 的数组，并根据它们有决定性差异的第一个索引做出决定。 

由于约束表明线性复杂度，任何超出 O(n) 的事情都是不必要的。 如果 n 达到 200,000 或更多，则重复比较所有前缀的二次方法会太慢，因为这意味着在最坏的情况下需要进行大约 4 × 10^10 次比较。 

主要的微妙之处在于理解相等前缀并不重要。 如果两个参与者在某个位置上具有相同的已解决计数，则可以忽略整个区域。 该决定仅在第一次不匹配时出现。 

当人们假设总金额或最终价值决定获胜者时，就会出现天真的陷阱。 这是错误的，因为问题显然取决于谁先领先，而不是谁最终更强。 

例如，考虑：

 输入：```
A: 1 3 3
B: 2 2 3
```乍一看，两者都以 3 结束，但 A 永远不会领先于 B，因为在索引 1 处 B 领先。 正确答案是B，因为B立即领先。 

另一个不正确的假设是以错误的方向进行字典顺序比较或求和值，这两者都忽略了时间排序的优势条件。 

## 方法

 暴力解释将每个位置视为检查点。 在每个索引 i 处，我们通过从开始扫描到 i 并计数或比较累积进度来重新计算 A 是否领先于 B。 这会导致 O(n^2) 时间，因为对于每个 n 个位置，我们最多可以重新检查 n 个元素。 

这是合乎逻辑的，因为它忠实地模拟了在每个时刻检查谁领先的想法，但它不必要地重复工作。 当 n 变大时，效率低下变得严重。 

关键的观察是，唯一重要的是两个序列在​​累积优势上不同的第一个索引。 我们不是从头开始重新计算，而是在单次传递中维护运行总计或直接比较前缀值。 一旦出现严格的不平等，我们就可以决定胜负并停止。 

这将问题从重复的前缀评估减少到单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 阅读代表每个参与者解决问题的进展的两个序列。 我们假设两个序列在​​时间上对齐，这意味着索引 i 对应于两个序列的同一时刻。 
2. 通过从第一个索引扫描到最后一个索引来初始化运行比较。 在每个索引处，直接比较值，或者如果输入表示增量求解，则保持累积差异。 
3. 在索引 i 处，检查一个参与者的值是否严格大于另一个参与者的值。 如果是这样，请立即将该参与者作为获胜者返回。 这样做的原因是，一旦最早确立了领先优势，后来的平局或逆转就与“第一决定性优势”规则无关。 
4. 如果值相等，则继续向前扫描。 平等意味着双方都还没有形成决定性的分离。 
5. 如果扫描完成时没有任何严格的不平等，则将结果宣布为平局，因为两个参与者在任何时候都没有获得优势。 

### 为什么它有效

 该算法依赖于以下不变量：在第一个不平等指数之前，两个序列在所有有意义的竞争项中都是相同的。 在它们不同的第一个指数中，差异的符号完全决定了结果，因为没有较早的指数与其相矛盾，也没有较晚的指数可以追溯性地使第一个决定性领先优势无效。 这减少了在前缀比较中查找第一个非零差异的决策。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    n = len(a)
    
    for i in range(n):
        if a[i] > b[i]:
            print("A")
            return
        elif a[i] < b[i]:
            print("B")
            return
    
    print("TIE")

if __name__ == "__main__":
    solve()
```该代码对两个序列执行一次传递。 逐个指标进行比较，与检测第一决定性优势的算法思想完全吻合。 一旦发现严格的不平等，我们就会提前终止，避免不必要的工作。 

一个常见的错误是忘记在第一个差异处停止，而是继续跟踪后来的变化。 这将错误地允许后来的逆转推翻先前的主导地位，这与问题的“首先解决更多问题”的定义相矛盾。 

## 工作示例

 ### 示例 1

 输入：```
a = [1, 3, 3]
b = [2, 2, 3]
```| 我| 一个[我] | b[i] | 决定|
 | --- | --- | --- | --- |
 | 0 | 1 | 2 | B领先，停止|

 在索引 0 处，B 立即具有更高的值，因此 B 获胜，无需进一步检查。 

这表明，即使后来的价值观趋同，早期的主导地位也是决定性的。 

### 示例 2

 输入：```
a = [2, 2, 5]
b = [2, 2, 3]
```| 我| 一个[我] | b[i] | 决定|
 | --- | --- | --- | --- |
 | 0 | 2 | 2 | 继续 |
 | 1 | 2 | 2 | 继续 |

 平等前缀表明关系不会影响决策。 第一个严格差异出现在索引 2 处，其中 A 领先。 

这证实了该算法正确地忽略了中性前缀并且仅对第一个分歧做出反应。

 ## 边缘情况

 一个重要的边缘情况是所有职位完全平等。 例如：

 输入：```
a = [4, 4, 4]
b = [4, 4, 4]
```该算法扫描所有索引，但没有发现任何严格的不平等。 每个步骤都会触发“继续”分支。 只有在穷尽序列之后，我们才会输出 TIE，这是正确的，因为没有参与者建立过领先优势。 

另一个边缘情况是当第一个元素已经确定结果时：

 输入：```
a = [10, 0, 0]
b = [1, 100, 100]
```在索引 0 处，A 已经领先。 扫描立即停止，随后对 B 有利的逆转将被忽略。 这证实了提前终止是必要的，并且由算法的结构正确执行。
