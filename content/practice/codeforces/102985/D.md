 ## 解决方案
 ## 问题理解

 我们有一个放置在坐标平面上的矩形披萨。 左下角可以被认为是原点，披萨跨越了宽度和高度。 然后厨师进行一组水平切割和一组垂直切割。 这些切口将披萨分成轴对齐的矩形块。 

每个水平切割将高度分割为条带，每个垂直切割将宽度分割为条带。 它们一起形成一个较小的矩形网格。 每个生成的切片的面积等于两个连续垂直切口之间的宽度与两个连续水平切口之间的高度的乘积。 

任务是确定哪个生成的矩形具有最大面积。 然而，还有一个额外的打破平局的规则：如果多个矩形具有相同的最大面积，我们更喜欢顶部边缘尽可能接近披萨顶部的矩形。 如果仍然平局，我们选择左边缘尽可能靠近左侧的那个。 

输入大小建议每个方向最多进行 1000 次切割，因此候选矩形的数量最多约为 100 万个。 对所有切片进行二次扫描是可以接受的，但超出该结构的任何内容都是不必要的。 切割的坐标可以大到 10^9，因此必须以 64 位整数进行算术运算。 

一个微妙的边缘情况来自边界间隔：切割定义连续坐标之间的段，而不是坐标本身。 当多个段具有相同的宽度或高度时，会出现另一种边缘情况，这会直接触发平局打破规则，而不是执行事故。 

一个幼稚的错误是将剪切坐标视为候选矩形边界，而不在 0 和 L 或 W 处添加隐式边界。例如，如果 L = 10 并且在 5 处有一个剪切，则忘记包含 0 和 10 会导致完全丢失最顶部或最底部的段。 

当仅独立跟踪最大宽度和最大高度时，会出现另一种故障模式。 该方法会错误地假设最佳矩形来自独立的最佳尺寸，但正确的矩形需要将特定宽度段与特定高度段配对。 

## 方法

 一旦结构可见，暴力破解的想法就很简单了。 对所有水平切割和所有垂直切割进行排序后，我们计算所有连续差异。 每个差异代表一个水平带的高度或一个垂直带的宽度。 然后，通过将一个水平间隙与一个垂直间隙配对形成每个矩形，并评估所有产品。 

这是正确的，因为切割将平面完全分解为独立的区间。 每个矩形唯一对应于一对相邻的切割间隔。 然而，这种方法需要迭代所有间隙对，生成 O(HV) 矩形，在最坏情况下最多为 10^6。 尽管我们可以干净地构造它，但在 Python 中，这已经是临界值，但在 1-2 秒内仍然可以接受。 

关键的观察是我们不需要任何超出邻接性的几何推理。 网格在两个维度上都是独立的，因此最大化面积减少了检查水平和垂直段长度的所有组合。 位置之间没有依赖性，只有长度之间的依赖性。 

一个稍微优化的观点是，我们可以预先计算所有水平间隙和所有垂直间隙一次，然后直接扫描它们的乘积空间，同时通过平局打破来跟踪最佳候选者。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有矩形进行暴力破解 | O(H·V) | O(H + V) | 已接受 |
 | 优化间隙配对扫描 | O(H·V) | O(H + V) | 已接受 |

 ## 算法演练

1、对横切位置和竖切位置进行排序。 这是必要的，因为顺序决定段结构，而不是原始输入顺序。 排序确保连续差异对应于真实的几何切片。 
2. 通过取连续水平坐标（包括 0 和 L 处的边界）之间的差来构造水平段高度的数组。每个值代表一个水平条带的高度。 
3. 通过取连续垂直坐标（包括 0 和 W 处的边界）之间的差来构造垂直线段宽度的数组。每个值代表一个垂直条带的宽度。 
4. 迭代每对（高度、宽度）。 对于每一对，计算面积 = 高度 × 宽度。 这对应于由切口形成的网格中的一个矩形。 
5. 维护目前为止找到的最佳矩形。 更新时，首先按面积进行比较，然后按高度位置规则（通过跟踪索引或坐标实现）进行比较，最后按左侧位置规则进行比较。 在实践中，我们存储实际的线段坐标以干净地强制打破平局。 
6.返回最佳对应的矩形边界（高度段索引，宽度段索引）。 这些索引直接映射回前缀切割和。 

工作原理：每个有效矩形都是通过选择一个水平间隔和一个垂直间隔来唯一定义的。 网格分解是完整且不重叠的，因此最大化所有这些对可以一次性探索完整的解决方案空间。 平局打破规则是矩形的局部规则，不影响最优结构，仅在相等的最大值中进行选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    L, W, H, V = map(int, input().split())
    hs = list(map(int, input().split()))
    vs = list(map(int, input().split()))

    hs.sort()
    vs.sort()

    # add boundaries
    hcuts = [0] + hs + [L]
    vcuts = [0] + vs + [W]

    hseg = []
    for i in range(1, len(hcuts)):
        hseg.append((hcuts[i] - hcuts[i-1], hcuts[i-1], hcuts[i]))

    vseg = []
    for i in range(1, len(vcuts)):
        vseg.append((vcuts[i] - vcuts[i-1], vcuts[i-1], vcuts[i]))

    best_area = -1
    best = (0, 0, 0, 0)  # x1, y1, x2, y2

    for hlen, hy1, hy2 in hseg:
        for vlen, vx1, vx2 in vseg:
            area = hlen * vlen

            # tie-breaking: higher y first (closer to top means larger y2),
            # then smaller x1
            cand = (area, hy1, vx1, vx2, hy2)

            if cand[0] > best_area:
                best_area = cand[0]
                best = (vx1, hy1, vx2, hy2)
            elif cand[0] == best_area:
                if (hy1 > best[1]) or (hy1 == best[1] and vx1 < best[0]):
                    best = (vx1, hy1, vx2, hy2)

    print(*best)

if __name__ == "__main__":
    solve()
```该解决方案首先将切割位置转换为连续的间隔，这是该问题中唯一有意义的抽象。 网格结构变得明确，每个矩形只是一个垂直间隔和一个水平间隔的乘积。 

平局决胜是通过存储实际坐标而不是依赖索引来处理的。 这避免了间隔端点与段标识混淆的差一错误。 该比较首先通过比较较低的 y 边界来优先考虑较高的水平线段，因为靠近顶部的线段具有较大的起始 y。

我们展示了一条关注候选更新的小痕迹：

 | 水平段| 垂直段| 面积 | 迄今为止最好的 |
 | ---| ---| ---| ---|
 | (7,12) | (7,12) | 大| 更新 |

 ## 边缘情况

 当一维中没有割点时，整个维度就变成一个区间。 The algorithm still works because the segment array contains exactly one element covering the full span.

 When multiple segments share the same length, the algorithm relies on coordinate ordering rather than length alone. This ensures that tie-breaking is stable even when areas match exactly.

 当削减幅度非常接近时，差异可能很小，但仍然是积极的。 由于所有算术都是基于整数的，因此不会出现精度问题，并且区间分解仍然准确。
