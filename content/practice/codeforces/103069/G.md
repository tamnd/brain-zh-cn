 ## 解决方案
 ## 问题理解

 我们得到一个固定数组和对其连续段的许多查询。 对于每个查询间隔$[l, r]$，我们必须计算有多少个子数组$[i, j]$完全包含在该区间内的属性具有子数组内不同值的数量为奇数的属性。 

因此，任务不是对子数组本身进行计数，而是根据其不同元素计数的奇偶校验条件对每个子数组进行分类，然后聚合每个查询范围内的所有子数组。 

约束很大：数组长度和查询数量都达到$5 \times 10^5$。 任何以每个子数组的线性甚至对数时间独立处理每个查询的解决方案都会立即变得太慢。 一个天真的$O(n^2)$子数组的枚举是不可能的，因为仅此一项就大约是$10^{11}$最坏情况下的操作。 甚至一个$O(n \sqrt{n})$或者$O(n \log n)$每个查询方法都会失败$5 \times 10^5$查询。 

关键的困难在于该属性取决于不同的元素，而这些元素本质上是子数组上的全局元素。 这使得在没有某种形式的预先计算或结构重新制定的情况下很难本地化贡献。 

当段中的所有元素都相同时，就会出现微妙的边缘情况。 在这种情况下，每个子数组都只有一个不同的元素，因此每个子数组都会做出积极的贡献。 对于一段长度$k$，答案是$\frac{k(k+1)}{2}$。 任何将不同计数视为独立于每个端点的不正确方法都会在这里失败，因为所有子数组的行为都是一致的。 

例如，当值交替时会出现另一种边缘情况$[1,2,1,2,1]$。 这里，不同计数根据窗口大小而振荡，并且关于不同计数单调性的天真贪婪假设完全失败。 任何依赖于不同元素的单调增长或收缩的方法都必须显式处理删除，而不仅仅是插入。 

## 方法

 一种直接的方法是独立处理每个查询。 对于固定的$[l, r]$，我们枚举其中的所有子数组，并且对于每个子数组，我们维护一个集合或频率图来计算它包含多少个不同元素。 这给出了正确的结果，因为它直接匹配定义。 

然而，对于一段长度$k$， 有$\frac{k(k+1)}{2}$子数组，因此单个查询可能已经需要$O(k^2)$工作。 和$k$最多$5 \times 10^5$，这远远超出了可行的限度。 即使优化每个子数组的不同计数也无济于事，因为子数组的数量占主导地位。 

关键的观察结果是，条件“不同元素的数量为奇数”是奇偶校验条件，这表明将问题转化为前缀贡献上的线性问题，而不是显式计算不同值。 我们不想直接跟踪不同计数，而是希望将答案表达为扩展子数组时每个值更改不同结构的频率的函数。 

重新表述问题的一个有用方法是修复正确的端点$j$。 对于每个$j$, 考虑所有$i \le j$，并询问这些前缀中有多少在中产生奇数个不同的值$[i, j]$。 这将问题转变为每个位置根据值的最后出现次数做出贡献的结构。 

现在关键的见解：不同元素的奇偶性$[i, j]$等于扫描时出现多少个“窗口内首次出现”的奇偶校验$i$到$j$。 这可以通过维护每个值的最后一次出现来跟踪。 当我们搬家时$j$，只有最后出现的位置发生变化，并且每个值在其新出现成为最新时恰好切换贡献一次。 

这将问题转变为维护贡献的动态结构，可以通过查询的离线处理来处理，通常使用芬威克树或线段树与扫描相结合$j$，同时跟踪最后一次出现的贡献。 

我们按升序处理端点，维护每个位置的结构$i$编码子数组是否从$i$当前结束时当前有奇数或偶数不同计数$j$。 Each time we see a repeated value, we remove its previous contribution and add the new one, ensuring that each state is updated in$O(\log n)$。 

Finally, each query$[l, r]$要求总和$i \in [l, r]$端点处的有效状态$r$，可以使用维护结构上的前缀和来回答。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$每个查询|$O(1)$或者$O(n)$| 太慢了 |
 | 离线 + Fenwick 最近出现的情况 |$O((n + m)\log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过按右端点对查询进行排序来处理查询，并维护一个动态结构，该结构对每个起始索引当前是否形成一个子数组进行编码，该子数组具有以当前位置结尾的奇数个不同元素。 

1.固定右端点，从左向右扫动。 在每个位置$r$，我们从概念上激活所有以$r$。 问题简化为快速回答有多少起始位置产生奇数不同计数。 
2. 维护从每个值到其最后出现索引的映射。 当我们处理一个新的位置时$r$有价值$a[r]$，我们找到它之前出现的位置$p$。 这很重要，因为之间的贡献结构$p$和$r$变化。 
3. 对于每个值，我们维护两个切换：当它出现时，它有助于所有起始位置位于其先前出现位置和当前位置之间的子数组的奇偶校验。 这种范围效应使我们能够用间隔更新替换每个子数组的推理。 
4. 我们在起始索引上使用 Fenwick 树。 每次更新对应于翻转一系列起始点上的奇偶校验贡献。 范围翻转以两点更新或差异数组样式 Fenwick 结构的形式实现。 
5、后加工位置$r$，我们回答其右端点为的所有查询$r$。 每个查询都要求提供“有效奇奇偶校验开始”的总和$[l, r]$，我们将其检索为 Fenwick 树上的前缀和差。 
6. 继续扫描，直到处理完所有位置。 

这样做的原因是，每次一个值重新出现时，它都会取消其先前的贡献并在新的边界处重新引入它，因此每个子数组的奇偶校验状态仅当端点交叉出现重复值时才会演变。 这确保了我们维护的结构始终反映所有正确的奇偶校验分类$[i, r]$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_add(self, l, r, v):
        if l <= r:
            self.add(l, v)
            self.add(r + 1, -v)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    m = int(input())

    queries = [[] for _ in range(n + 1)]
    for idx in range(m):
        l, r = map(int, input().split())
        queries[r].append((l, idx))

    bit = Fenwick(n)
    last = {}

    ans = [0] * m

    for r in range(1, n + 1):
        x = a[r - 1]

        if x in last:
            prev = last[x]
            bit.range_add(prev, prev, -1)

        bit.range_add(r, r, 1)
        last[x] = r

        for l, idx in queries[r]:
            ans[idx] = bit.sum(r) - bit.sum(l - 1)

    print("\n".join(map(str, ans)))

if __name__ == "__main__":
    solve()
```此处使用芬威克树作为起始位置的动态计数器。 各位置$i$存储它当前是否作为以当前结尾的子数组的有效开始$r$。 当一个值重复时，它以前的贡献将被删除，并在新位置添加新的贡献。 这确保了树在每一步都反映当前的奇偶校验分类。 

查询按其右端点进行分组，以便在扫描到达其位置时准确回答每个查询$r$。 前缀和的差异给出了计数$[l, r]$。 

主要的微妙之处在于，必须在处理每个数据时应用更新$r$，在该位置回答查询之前，否则查询将看到陈旧状态。 

## 工作示例

 考虑数组$[1, 2, 1]$通过单个查询$[1, 3]$。 

在$r=1$，首先看到值 1，因此位置 1 变为活动状态。 在$r=2$，值 2 是新的，因此位置 2 变为活动状态。 在$r=3$，值 1 重复，因此索引 1 的先前贡献被删除并在索引 3 处重新添加。 

| r | 价值| 主动启动|
 | ---| ---| ---|
 | 1 | 1 | {1} |
 | 2 | 2 | {1,2} |
 | 3 | 1 | {2,3} |

 供查询$[1,3]$，我们计算活跃的开始时间$[1,3]$，即 2。 

现在考虑$[1,1,1,1]$带查询$[1,4]$。 

每一次新的发生都会消除之前的贡献并将其向前移动。 

| r | 主动启动|
 | ---| ---|
 | 1 | {1} |
 | 2 | {2} |
 | 3 | {3} |
 | 4 | {4} |

 每个子数组都只有一个不同的元素，因此所有 10 个子数组都是有效的，与活动开始时的累积相匹配。 

这些痕迹表明，该结构的行为就像每个值的贡献的滑动“所有权”，这与最后出现的逻辑相匹配。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O((n + m)\log n)$| 每次更新和查询都使用 Fenwick 操作 |
 | 空间|$O(n)$| 最后出现图和 Fenwick 树 |

 复杂度兼容$5 \times 10^5$约束，因为每个操作都是对数的，并且总操作的输入大小是线性的。

 ## 边缘情况

 对于单个元素数组$[x]$，唯一的子数组是$[1,1]$，它有一个不同的元素并且始终被计数。 该算法将位置 1 初始化为活动状态，并直接从 Fenwick 树回答查询，生成 1。 

对于像这样的数组$[1,1,1,1]$，每次重复都会向前移动主动贡献。 在处理最终位置时，所有有效的起始位置与最后一次出现的位置对齐，并且前缀和正确累加所有子数组，为整个间隔生成 10。
