 ## 解决方案
 ## 问题理解

 我们得到一个无向图，其中每条道路最初需要一秒钟才能穿过。 我们可以向单条道路投资，每一美元都会将该道路的“速度水平”提高一级。 如果道路有速度等级$a$，遍历需要$1/a$秒。 

稍后将有两人同时旅行：其中一人来自$s_1$到$t_1$，另一个来自$s_2$到$t_2$。 最多分配后$k$美元跨​​越边缘，每个旅行者使用所产生的边缘遍历时间下的最短时间路径，并且我们希望最小化这两个最短旅行时间的总和。 

关键的困难在于我们没有直接选择路径。 相反，我们正在修改边缘成本，并且两个最短路径问题通过共享预算约束相互作用。 

限制条件$n, m \le 5000$建议最短路径计算$O(m \log n)$甚至$O(nm)$很好，但是任何试图探索所有边缘预算分配的事情都是不可能的，因为$k$可以大到$10^9$。 这立即排除了任何跟踪分配给每个边缘的预算的 DP。 

一个微妙的点是，最优解并不以完全组合的方式依赖于任意图结构。 每条边上的成本函数在分配的预算中是平滑且凸的，这强烈表明解决方案应该沿着最短路径而不是任意子图集中结构。 

朴素推理的一个常见失败案例是假设我们应该贪婪地改进在当前最短路径中显得有用的边。 这种情况会被打破，因为改进一条边可以完全改变最短路径的特性，从而使之前的贪婪选择失效。 

例如，在一个简单的图中，两条不相交的路径在$s$和$t$，投资一条边可能会导致最短路径发生切换，使得之前的边际收益计算变得毫无意义。 

## 方法

 蛮力观点是考虑分布$k$所有边缘的预算单位。 对于每次分配，我们使用 Dijkstra 重新计算两对的最短路径，并计算结果总和。 这在原则上是正确的，但完全不可行，因为分配的数量是指数级的$m$，并且由于路径切换，即使连续放松也无济于事。 

关键的观察是，对于任何固定路径，沿其边缘分配预算的最佳方式不是任意的。 每条边都有$x_e$升级贡献成本$1/(1+x_e)$，并且对于路径上固定的总分配，该函数是凸的$x_e$。 凸性意味着在边缘之间不均匀地分配预算只会增加总旅行时间。 因此，对于固定选择的路径，最佳策略是在该路径的所有边缘上均匀分配预算。 

这将每条路径的优化压缩为其跳数长度的函数$L$，而不是单个边缘。 如果一条路径有$L$边缘和接收$k_i$预算，每条边有效地得到$k_i / L$，总时间变为：$$L \cdot \frac{1}{1 + k_i / L} = \frac{L^2}{L + k_i}.$$现在，图结构仅通过相关端点之间的最短跳跃距离起作用。 每个旅行者独立地偏好最短跳路径，因为在具有不同长度的所有路径中，该函数$\frac{L^2}{L+k_i}$正在增加$L$对于固定的$k_i$。 

因此，我们唯一需要的图计算是之间的未加权最短路径长度$s_1, t_1$和之间$s_2, t_2$。 

之后，剩下的问题就纯粹是连续的了：分裂$k$进入$k_1$和$k_2$，最小化：$$\frac{L_1^2}{L_1 + k_1} + \frac{L_2^2}{L_2 + k - k_1}.$$这是一个一维凸优化问题，可以通过检查导数平衡点来解决，或者通过三元搜索来解决，因为函数是单峰的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 尝试所有分配+重新计算最短路径| 指数| O(米) | 太慢了 |
 | BFS + 预算持续分割 | O(n + m + log k) | O(n + m + log k) | O(n + m) | 已接受 |

 ## 算法演练

 ### 1. 计算未加权的最短路径长度

 我们运行 BFS$s_1$到$t_1$要得到$L_1$，并从$s_2$到$t_2$要得到$L_2$。 由于所有边最初在跳数意义上具有相同的权重，因此 BFS 就足够了。 此步骤将图完全简化为两个捕获结构约束的整数。 

### 2. 减少成本函数的每条路径

 我们将每条路径解释为消耗预算$k_i$，给出成本：$$f_i(k_i) = \frac{L_i^2}{L_i + k_i}.$$该步骤通过凸性论证来证明，即固定路径上的最优分配在边缘上是均匀的。 

### 3. 重新制定全局优化

 我们现在最小化：$$f_1(k_1) + f_2(k - k_1)$$超过真实$k_1 \in [0, k]$。 这将整个问题简化为单变量凸函数。 

### 4. 找到最佳的预算分配

 我们通过搜索找到最小值$k_1$。 由于函数是凸函数，我们可以安全地使用三元搜索或通过导数等式求解：$$\frac{L_1^2}{(L_1 + k_1)^2} = \frac{L_2^2}{(L_2 + k - k_1)^2}.$$我们计算连续最优值并评估附近的整数以处理离散化。 

### 5.输出最佳值

 我们评估候选分割点的目标并打印最小值。 

### 为什么它有效

 关键的不变量是任何最佳解决方案都可以转化为每条路径在其边缘上接收均匀的预算分配而不会增加成本的解决方案。 这消除了对内部边缘结构的依赖并保留了最优性，因为凸性确保只有每条路径的总分配很重要。 一旦减少，剩下的问题是两个独立成本函数之间的凸分配，这保证了分割变量中的局部最优也是全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def bfs(n, adj, s, t):
    dist = [-1] * (n + 1)
    q = deque([s])
    dist[s] = 0
    while q:
        u = q.popleft()
        if u == t:
            return dist[u]
        for v in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                q.append(v)
    return dist[t]

def f(L, k):
    return L * L / (L + k)

def solve_one(L1, L2, k):
    if L1 == 0 and L2 == 0:
        return 0.0
    if L1 == 0:
        return f(L2, k)
    if L2 == 0:
        return f(L1, k)

    lo, hi = 0, k
    for _ in range(80):
        m1 = (2 * lo + hi) / 3
        m2 = (lo + 2 * hi) / 3
        v1 = f(L1, m1) + f(L2, k - m1)
        v2 = f(L1, m2) + f(L2, k - m2)
        if v1 < v2:
            hi = m2
        else:
            lo = m1
    return f(L1, lo) + f(L2, k - lo)

n, m, k = map(int, input().split())
adj = [[] for _ in range(n + 1)]
for _ in range(m):
    a, b = map(int, input().split())
    adj[a].append(b)
    adj[b].append(a)

s1, t1, s2, t2 = map(int, input().split())

L1 = bfs(n, adj, s1, t1)
L2 = bfs(n, adj, s2, t2)

ans = solve_one(L1, L2, k)
print(f"{ans:.12f}")
```BFS 部分仅提取跳跃距离，这已经足够了，因为边缘改进永远不会改变邻接关系，只会改变遍历权重。 三元搜索在预算分割的连续放松上进行，由于成本函数的凸性，这是有效的。 

这里的一个常见陷阱是尝试为每个边而不是每个路径分配预算。 这导致最短路径结构和优化之间的棘手耦合，而正确的视图将每条路径折叠成单个凸函数。

 ## 边缘情况

 一种重要的边缘情况是其中一条路径的长度为零，这意味着$s_i = t_i$。 在这种情况下，无论预算分配如何，该旅行者的成本函数始终为零，并且所有预算都应该流向另一条路径。 该算法通过返回零成本来直接处理这个问题$L=0$，确保不被零除或不正确的分割。 

另一个微妙的情况是两条路径的长度相等。 那么对称性迫使最优解均分预算，三元搜索自然收敛到中点。 这证实了凸松弛在对称性下表现正确，并且没有离散伪影改变结果。
