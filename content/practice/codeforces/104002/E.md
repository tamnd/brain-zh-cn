 ## 解决方案
 ## 问题理解

 我们得到一行整数，两个玩家轮流删除元素，直到什么都没有剩下。 威廉首先移动，轮到他时，他可以从数组中的任何位置选择任何仍然可用的元素。 机器人根本不进行策略性选择：它总是在每一步中删除最左边的剩余元素。 

威廉的目标是最大化他收集的价值总和。 机器人的确定性行为意味着一旦威廉的选择固定，剩余阵列的演变是完全可预测的，因此问题不在于对抗性博弈论，而在于在机器人消耗左侧结构之前控制威廉“声称”的位置。 

关键的困难在于威廉的选择会间接影响未来的可用性。 如果他延迟挑选左侧有价值的元素，机器人可能会首先消耗它。 另一方面，过早选择某些东西可能会浪费前缀中的“转弯预算”，否则机器人将被迫采取低价值元素。 

约束 n 高达 100000 意味着任何解都必须接近线性或 n log n。 任何尝试模拟所有可能的选择序列或显式考虑元素子集的方法都是不可能的，因为即使是粗略的子集探索也会呈指数增长。 这促使我们采用一种贪婪或基于数据结构的方法，该方法按顺序处理元素，并仅维护 William 有效保留的内容的紧凑表示。 

机器人最左边的规则产生了一个微妙的边缘情况。 例如，考虑像 10 1 100 1 这样的数组。一个天真的想法是 William 应该始终采用最大的可用元素，但这忽略了机器人从左侧重复消费后，如果不在前缀配额内获取，一些“大”值可能会变得无法访问。 另一个失败的情况是，当较大的值早期聚集时，因为威廉不能在早期前缀中采用太多的值，而不违反机器人已经采用这些前缀元素的一半的事实。 

因此，真正的约束不是关于原始游戏序列中的邻接，而是关于前缀容量：在前 k 个元素内，William 最终不能获取超过 k/2 个元素，因为机器人总是会从左侧同步消耗该前缀的其余部分。 

## 方法

 如果我们忽略机器人的确定性行为，蛮力的想法将是模拟游戏状态：在每一步，尝试威廉的每一个可能的选择，然后模拟机器人的被迫移动，并继续。 这会产生一个等于剩余元素数量和深度 n 的分支因子，从而导致阶乘或指数复杂度。 即使记忆化也会失败，因为状态不仅是保留哪些元素，而且还包括重复左删除后它们的顺序，这仍然留下了太多不同的配置。 

关键的观察是停止考虑“删除后剩余的数组”，而是考虑前缀约束。 处理完前 k 个位置后，总共删除了 k 个元素，并将它们分配给 William 和机器人。 由于机器人总是消耗最左边的可用元素，因此它有效地保证了在每个长度为 k 的前缀中，William 无法主导计数。 这导致了 William 只能“持有”前 k 个位置中最多 k/2 个选定元素的约束。

一旦我们接受这种解释，问题就变成了：从左到右处理数组，并维护一组 William 选择的元素，这样在每个偶数索引 k 处，我们确保最多 k/2 个选择保留在前 k 个元素中。 由于威廉想要最大总和，每当他提前选择了太多小元素时，他就应该丢弃其中最小的元素。 这自然意味着在大小限制下维持最大效益选择，但由于我们正在丢弃最无用的所选元素，因此我们实际上希望保留一个允许删除 William 当前选择中最小候选者的结构。 

最小堆正好满足这个要求：我们暂时将每个元素作为 William 集合的候选元素，每当所选元素的数量超过当前前缀允许的配额时，我们就会删除最小值。 这确保了威廉始终保留与可行性一致的最佳可能的多重选秀权。

 ## 算法演练

 我们从左到右扫描数组，将每个位置视为威廉可能会尝试占据它。 

1. 将当前值插入代表 William 暂时选择的集合的最小堆中。 这假设威廉考虑首先考虑一切，稍后我们将通过删除较弱的选择来纠正不可行性。 
2.处理完索引k后，计算William可以保留从前缀1到k的元素数量。 由于机器人总是采用最左边的可用元素，因此在任何前缀中 William 都不能在这些位置中得到超过 k/2 个元素，因此我们仅在 k 为偶数时才强制执行该约束。 
3. 如果k是偶数并且堆大小超过k/2，则重复从堆中移除最小元素，直到满足约束。 我们删除最小的元素的原因是保留较大的元素总是可以提高最终的总和，同时仍然尊重可行性。 
4. 继续直到数组末尾。 堆中剩余元素的总和是 William 可达到的最高分数。 

重要的微妙之处在于我们从未明确地模拟机器人的动作。 堆强制执行唯一重要的结构约束：William 的选择必须与消耗每个前缀总数一半的机器人保持兼容。 

### 为什么它有效

 不变量是，在处理前缀 k 后，堆包含来自前缀的元素的最大和子集，在机器人的强制最左删除规则下，可以将其扩展为有效的游戏结果。 机器人的行为确保在任何长度为 k 的前缀中，最多恰好有 k/2 个元素被有效地“保留”给 William，因为剩余的容量是从左侧消耗的。 通过在违反约束时始终丢弃最小的选定元素，我们在随 k 增长的类拟阵基数约束下维护最佳可能值子集。 这保证了未来的决策不会因为提前保留较小的元素而不是较大的元素而受益。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    heap = []
    total = 0
    
    for i, x in enumerate(a, start=1):
        heapq.heappush(heap, x)
        
        if i % 2 == 0:
            limit = i // 2
            while len(heap) > limit:
                heapq.heappop(heap)
    
    print(sum(heap))

if __name__ == "__main__":
    solve()
```该解决方案维护 William 当前选择的值的最小堆。 每个元素都会立即添加，然后在偶数位置强制执行前缀可行性约束。 堆弹出操作总是删除最小的选择值，这是唯一安全的丢弃，因为它可以最大限度地减少总和的损失。 

一个常见的实现错误是忘记约束适用于每个前缀而不是仅适用于末尾。 另一种方法是使用最大堆，这会破坏逻辑，因为我们需要丢弃最不有价值的所选元素，而不是最有价值的元素。

 ## 边缘情况

 一个关键的边缘情况是大值较早出现，例如`100 1 99 1`。 天真的贪婪策略会立即采取 100 和 99，但这可能违反前缀可行性，因为机器人将消耗干预元素并降低未来的灵活性。 该算法插入所有值，但立即强制执行前缀约束，这会在超出容量时强制删除最小的早期选择，为以后的前缀保​​留更高的值。 

另一个边缘情况是交替高值和低值，例如`1 100 2 99 3 98`。 这里堆增长很快，如果没有定期修剪，选择将超出允许的前缀计数。 该算法始终如一地修剪每个偶数索引处的最弱选择，确保威廉的选择永远不会变得不可行，同时仍然保留最大的可用值。
