 ## 解决方案
 ## 问题理解

 我们得到了一个矩形的数字网格。 对于每个单元格，我们查看与该单元格位于同一行或同一列的所有值，包括该单元格本身。 在所有这些值中，我们检查当前单元格的值是否是最小的。 任务是计算有多少细胞满足这个条件。 

另一种思考方式是，每个单元格定义一个由其行和列组成的“十字”。 我们取该十字内的最小值，并且我们想要计算有多少单元格等于该最小值。 

网格大小可以大到 1000 x 1000，最多可提供一百万个单元格。 任何通过直接扫描整个行和列来检查每个单元的解决方案都可能为每个单元执行大约 2n 或 2m 的工作，在最坏的情况下导致大约 10^12 次操作。 这远远超出了一秒的限制。 

微妙的边缘情况来自行或列内重复的最小值。 例如，如果一行是`1 2 1`，两个单元格都包含`1`从行的角度来看，它们是候选者，但在考虑其列时，其中一个可能会失败。 

另一个棘手的情况是行最小值和列最小值不一致时。 单元格可能是其行中的最小值，但不是其列中的最小值，反之亦然，因此我们必须始终正确考虑两个约束的并集。 

## 方法

 天真的想法很简单。 对于每个单元格`(i, j)`，我们扫描整行`i`找到最小值，我们还扫描整列`j`找到最小值。 然后我们将单元格值与这两个结果中的最小值进行比较。 如果它们匹配，我们就计算它。 

这是正确的，因为行和列的并集上的最小值只是行最小值和列最小值的最小值。 然而，为每个单元重新计算这些最小值是浪费的。 每行扫描的成本为 O(m)，每列扫描的成本为 O(n)，对所有 n·m 个单元执行此操作会导致 O(n·m·(n+m))，在最坏的情况下，根据常数，这大约是 2×10^9 到 2×10^12 次操作，这太慢了。 

关键的观察结果是行最小值和列最小值不依赖于查询单元格。 它们仅取决于网格结构。 所以我们可以预先计算每行每列的最小值一次。 之后，可以通过将每个单元格与行最小值和列最小值中较小的一个进行比较，在恒定时间内检查每个单元格。 

这将问题简化为预处理后通过网格的单次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·m·(n+m)) | O(1) | O(1) | 太慢了|
 | 最佳| O(n·m) | O(n+m) | 已接受 |

 ## 算法演练

 我们首先计算两个辅助数组，一个存储每行的最小值，一个存储每列的最小值。 

1.初始化一个数组`rowMin`尺寸的`n`，充满了非常大的值。 这将存储每行中看到的最小值。 
2.初始化一个数组`colMin`尺寸的`m`，也充满了非常大的值。 这将存储每列中看到的最小值。 
3. 扫描整个矩阵一次。 对于每个单元格`(i, j)`， 更新`rowMin[i] = min(rowMin[i], a[i][j])`和`colMin[j] = min(colMin[j], a[i][j])`。 此步骤在所有单元格的线性时间内为每一行和每一列构建精确的最小值。 
4. 预处理后，再次扫描矩阵。 对于每个单元格`(i, j)`, 计算`t = min(rowMin[i], colMin[j])`。 如果`a[i][j] == t`，增加答案。 
5. 输出最终计数。 

我们采取的理由`min(rowMin[i], colMin[j])`是 row 中值的并集`i`和列`j`在行或列中都有最小值。 没有其他来源可以提供更小的值，因为每个元素都已在这两个集合之一中得到考虑。 

### 为什么它有效

 对于任何细胞`(i, j)`，其交叉中考虑的每个值都属于行`i`或列`j`。 该联合的最小值必须是行中最小的元素`i`或列中的最小元素`j`，以较小者为准。 预处理步骤确保这些值在全局范围内都是正确的。 因此，比较`a[i][j]`和`min(rowMin[i], colMin[j])`准确地检查细胞是否在其交叉中达到最小值，并且没有细胞可以被错误分类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(n)]
    
    INF = 10**18
    rowMin = [INF] * n
    colMin = [INF] * m

    for i in range(n):
        for j in range(m):
            v = a[i][j]
            if v < rowMin[i]:
                rowMin[i] = v
            if v < colMin[j]:
                colMin[j] = v

    ans = 0
    for i in range(n):
        for j in range(m):
            if a[i][j] == min(rowMin[i], colMin[j]):
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将网格读入内存，这是必要的，因为我们需要对数据进行两次传递。 第一遍独立构造行和列最小值。 这可以避免以后重复扫描。 

第二遍执行实际情况检查。 表达式`min(rowMin[i], colMin[j])`计算时间为 O(1)，因此每个单元都贡献恒定的功。 这是取代昂贵的强力搜索的步骤。 

一个常见的实现错误是尝试在第二个循环内重新计算行或列最小值。 这会默默地重新引入 n 或 m 的额外因素并破坏性能。

 ## 边缘情况

 1×1 网格是最简单的边界。 行和列是相同的单个元素，因此两个最小值都等于值，并且必须对单个单元格进行计数。 算法初始化`rowMin[0]`和`colMin[0]`到该值，最终检查立即通过。 

在每个条目都相同的均匀矩阵中，每行最小值和列最小值都是相同的值。 对于任何细胞`(i, j)`,`min(rowMin[i], colMin[j])`等于单元格值，因此所有条目都会被计数。 第二遍对每个位置都正确递增，无一例外。 

如果一行包含非常小的值而列不包含非常小的值，则行最小值在大多数比较中占主导地位。 该算法仍然有效，因为它总是在评估之前重新计算正确的全局行和列最小值，确保不依赖于局部结构或遍历顺序。
