 ## 解决方案
 ## 问题理解

 我们得到一个非负整数数组，它应该表示由隐藏模数过程生成的值。 存在未知正整数$M$，并且该数组预计与通过取 2 的幂并对其模进行减少而形成的序列相匹配$M$。 换句话说，第一个元素的行为应该类似于$2^0 \bmod M$，第二个像$2^1 \bmod M$， 等等。 

任务不是找到任何有效的模数，而是确定是否存在恰好一个正整数$M$使得整个序列都符合这个规则。 如果这样一个$M$唯一存在，我们输出它。 否则，我们输出-1。 

输入大小最多允许$10^5$每个测试文件的值，在所有测试用例中聚合。 这立即排除了任何尝试候选值的方法$M$并模拟每一个的完整序列。 二次或偶数$O(n \sqrt{A_i})$样式解决方案会太慢。 

一个微妙的点是序列不是任意模运算。 它严格是模数下的确定性指数级数，这严重限制了结构。 这种结构使得重建成为可能。 

一种经常误导天真的推理的失败案例是假设如果所有值都小于某个候选值$M$，然后任何更大的$M$作品。 例如，如果序列是$[1, 2, 4, 8]$，那么任意$M > 8$保持平等，这意味着没有唯一的模数。 这直接导致“-1”情况。 

当发生环绕时，会出现另一种边缘情况。 例如，如果一个值突然变为 0 或意外地重复较早的值，则表明模量小到足以强制碰撞，并且可能存在多个候选模量或根本不存在。 

## 方法

 蛮力思想是从定义开始的。 我们尝试一位候选人$M$并验证每个术语是否匹配$2^{i-1} \bmod M$。 对于每个$M$，这需要扫描整个数组，并且可能有许多合理的值$M$。 甚至将候选人限制在这样的范围内$\max(A_i) + 1$仍然留下无限的搜索空间，因为正确性条件取决于模块化行为，而不仅仅是与观察值的比较。 

思考问题的一种更有效的方法是逆向模块化过程。 而不是测试$M$，我们询问序列对它施加了什么约束。 

从定义来看：$$A_{i+1} \equiv 2A_i \pmod{M}$$这意味着：$$2A_i - A_{i+1} \equiv 0 \pmod{M}$$所以每个相邻的对给出一个必须能被整除的数字$M$。 这意味着$M$必须除以表格的所有值$2A_i - A_{i+1}$。 

这立即将问题简化为计算这些约束的最大公约数。 如果所有这些差异都为零，则序列与加倍完全一致并且永不换行，这对应于以下情况$M$是无界的，不存在唯一解。 

否则，所有约束的 gcd 给出唯一可能的候选模数。 一旦我们计算出这个候选序列，我们仍然必须验证它是否产生给定的序列，因为基于 gcd 的重建仅强制执行成对一致性，而不是完整的模块化动态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数或$O(n \cdot M)$|$O(1)$| 太慢了|
 | 最佳|$O(n \log A_i)$|$O(1)$| 已接受 |

 ## 算法演练

 ### 1. 从连续项构建约束

 对于每一个相邻的对$(A_i, A_{i+1})$，计算：$$D_i = 2A_i - A_{i+1}$$每个$D_i$必须能被未知模数整除$M$。 这将序列约束转换为可分系统。 

### 2. 计算所有非零约束的 gcd

 保持整体运行 gcd$D_i \neq 0$。 该 gcd 表示可以同时除以每个约束的最大整数，因此是唯一可能的候选者$M$。 

如果全部$D_i$为零，我们无法得出任何界限$M$，意味着无限多个值$M$会满足结构，因此唯一性失败。 

### 3. 处理“所有差异为零”的情况

 如果每一个$D_i = 0$，序列满足$A_{i+1} = 2A_i$确切地。 这意味着该序列永远不会对任何内容进行模运算。 任何足够大的$M$有效，所以没有唯一的模数。 我们立即返回-1。 

### 4. 验证候选模数

 让$M$是计算出的 gcd。 我们模拟重现：$$x_1 = A_1, \quad x_{i+1} = (2x_i) \bmod M$$并检查它是否准确地再现给定的数组。 如果任何时候失败，则返回 -1。 

### 为什么它有效

 关键的不变量是任何有效的模数必须除以每个表达式$2A_i - A_{i+1}$。 这直接来自重写模块化递归。 因此，每个有效的$M$位于所有约束的最大公约数的除数集合中，并且最大公约数是最大可能的候选者。 

如果 gcd 候选者验证失败，那么它的除数就不可能成功，因为减少$M$只会加强模块化包装，并会引入早期的不一致。 这确保了算法不会错过任何替代的有效模数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    if n == 1:
        return -1
    
    g = 0
    for i in range(n - 1):
        d = 2 * a[i] - a[i + 1]
        if d != 0:
            if g == 0:
                g = abs(d)
            else:
                import math
                g = math.gcd(g, abs(d))
    
    if g == 0:
        return -1
    
    # verify
    m = g
    cur = a[0]
    for i in range(1, n):
        cur = (2 * cur) % m
        if cur != a[i]:
            return -1
    
    return m

def main():
    t = int(input())
    out = []
    for _ in range(t):
        out.append(str(solve()))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```核心实现的重点是将递归转换为线性约束并使用 gcd 对其进行压缩。 减法步骤`2 * a[i] - a[i+1]`是揭示整除性所需的唯一变换$M$。 

验证阶段是必要的，因为gcd聚合仅确保局部约束的一致性。 该模拟确保了从第一个学期开始模块化演化的全局正确性。 

一个微妙的实现细节是处理负差异。 在 gcd 之前取绝对值至关重要，因为整除性不受符号影响。

 ## 边缘情况

 ### 所有差异为零

 当序列恰好加倍且没有任何换行时，每个$D_i = 0$。 算法集$g = 0$并立即返回-1。 这正确地捕捉到了无限多个模数有效的事实，因此违反了唯一性。 

### 负中间差异

 如果$2A_i < A_{i+1}$，计算出的$D_i$变为负值。 取绝对值可确保 gcd 计算保持有效，因为整除性仅取决于大小。 

### 早期的不一致

 如果序列在任何点违反模加倍，验证循环会立即失败，拒绝不正确的 gcd 候选者，即使它们满足本地所有成对约束。
