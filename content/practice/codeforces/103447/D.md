 ## 解决方案
 ## 问题理解

 我们得到少量分数，每个分数由一对整数表示$p$和$q$。 允许的不寻常操作是从两个数字的十进制表示中删除数字，但只能成对删除：每当两个数字中都出现一个数字时，我们可以选择该数字的出现并将其从分子和分母中删除。 每个数字中的剩余数字在删除后形成新的整数，并且分数的值不需要在数字消除技巧的经典意义上保持数学上相等，但该问题将操作定义为唯一允许的转换。 

在删除匹配数字多重集的所有可能方法中，每个分数都会产生多个结果减少对。 目标是选择具有最小可能分子值的结果对。 如果多个操作获得相同的分子，则与其配对的任何相应的有效分母都是可接受的。 

关键结构是数字被视为多重集，而不是位置。 这意味着我们不是匹配数字位置，而是决定全局删除每个数字的多少个副本。 

约束条件$n \le 10$非常小，这表明我们需要对每个分数执行详尽或组合搜索。 每个数字最多可以有 63 位，这意味着最多大约 19 位十进制数字，因此搜索空间并不小，但仍然可以通过修剪或结构化枚举进行管理。 

一个微妙的边缘情况是删除后的前导零。 例如，如果我们形成像“007”这样的简化分子，则它必须被解释为 7。未经标准化的基于字符串的简单比较会根据字典顺序错误地将此类字符串视为更大或更小。 

另一个棘手的情况是删除数字中除零之外的所有数字。 例如，“1000”和“1000”可以通过删除所有零来减少，可能会留下空字符串。 正确的解释是空或全零字符串对应于整数 0。 

第二种边缘情况是不同的删除选择导致相同的数值但不同的数字组成。 仅跟踪数字多重集但按字典顺序比较字符串的简单方法将会失败，因为决定答案的是数值，而不是字符串顺序。 

## 方法

 强力解释是将问题视为针对 0 到 9 中的每个数字选择要删除的副本数量，以两个数字中出现的次数为界。 对于每个数字，我们选择从 0 到最小出现次数的删除计数$p$和$q$。 这定义了大小的搜索空间$\prod_{d=0}^9 (min(cnt_p[d], cnt_q[d]) + 1)$。 

在最坏的情况下，每个数字会出现多次，搜索空间会随着数字的数量呈指数增长。 即使有 10 位数字，如果每个数字出现多次，也会变得太大而无法直接枚举。 

关键的观察结果是，关于删除计划唯一重要的是分子和分母的剩余数字多重集。 我们可以逆向思考，而不是直接考虑构建删除：我们希望从原始数字形成数字子集，同时确保两者的可行性$p$和$q$。 自从$n \le 10$，我们可以独立地处理每个分数，并枚举从与两个原始计数一致的数字子集派生的所有有效结果分子。 

我们可以按如下方式对过程进行建模。 我们考虑所有方法来选择数字子集以同时保留原始分子和分母，同时考虑到我们不能使用比取消后任一数字中存在的数字更多的副本。 这自然会导致数字 DP 样式或有界多集枚举，其中我们生成所有可能的剩余数字计数，然后计算最小可能的结果分子。 

因为每个数字最多有约 19 位数字，所以我们可以紧凑地表示计数并通过修剪枚举可行的剩余数字分布。 该结构足够小，通过记忆进行数字计数的 DFS 就足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力删除 | 以数字表示的指数| O(1) | O(1) | 太慢了 |
 | 带剪枝的位数 DFS | 小指数（以数字为界） | O(10) 状态 | 已接受 |

 ## 算法演练

 我们独立处理每个部分。 

1. 两者都转换$p$和$q$转换为大小为 10 的数字频率数组。这使我们可以直接查看每个数字中存在每个数字的多少个副本。 这种表示完全消除了排序问题。 
2. 对于每个数字$d$，计算取消后我们可以将其保留在两个数字中的最大次数。 这受到两个数字之间的最小出现次数的限制。 
3. 我们定义了对数字 0 到 9 的递归搜索。在每个数字$d$，我们决定有多少个副本$d$取消后仍保留在最终的分子和分母中。 剩余副本数量不得超过原始可用数量。 
4. 在每一步中，我们维护两个部分多重集：一个用于分子数字，一个用于分母数字。 我们还确保一致性：删除的位数在双方之间是一致的，这意味着我们只构建可能因某些有效取消而产生的状态。 
5. 处理完所有数字后，我们通过按升序对数字进行排序并将它们连接起来，将两个多重集转换为整数。 
6. 我们比较结果并保留分子值最小的对。 如果平局，我们保留任何相应的分母。 

一个关键的简化是我们没有显式模拟删除。 相反，我们直接构造与两个原始数字一致的最终剩余数字多重集。 

### 为什么它有效

 不变量是在数字的每个递归深度$d$，我们已经完全决定了有多少个小于的数字副本$d$保留在分子和分母中，并且这些选择对于原始位数而言始终是可行的。 由于一旦计数固定，数字选择在各个位置上都是独立的，因此任何完整的分配都对应于有效的删除多重集。 这保证了每个可到达的最终状态仅被考虑一次，并且不会产生无效状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def digits_of(x):
    cnt = [0] * 10
    if x == 0:
        cnt[0] = 1
        return cnt
    while x > 0:
        cnt[x % 10] += 1
        x //= 10
    return cnt

def build_number(cnt):
    s = []
    for d in range(10):
        s.append(str(d) * cnt[d])
    res = "".join(s).lstrip('0')
    return 0 if not res else int(res)

def solve_one(p, q):
    cp = digits_of(p)
    cq = digits_of(q)

    best = None

    def dfs(d, up, uq, vp, vq):
        nonlocal best
        if d == 10:
            num = build_number(up)
            den = build_number(uq)
            if best is None or num < best[0]:
                best = (num, den)
            return

        max_keep = min(cp[d], cq[d])

        for keep in range(max_keep + 1):
            rem_p = cp[d] - keep
            rem_q = cq[d] - keep

            up[d] += rem_p
            uq[d] += rem_q

            dfs(d + 1, up, uq, vp, vq)

            up[d] -= rem_p
            uq[d] -= rem_q

    dfs(0, [0]*10, [0]*10, None, None)
    return best

def main():
    n = int(input())
    for _ in range(n):
        p, q = map(int, input().split())
        x, y = solve_one(p, q)
        print(x, y)

if __name__ == "__main__":
    main()
```该解决方案首先将数字转换为数字频率数组，从而完全消除排序问题。 DFS 逐位枚举所有有效的取消模式。 状态数组`up`和`uq`分别表示分子和分母消去后的剩余数字。 

一个微妙的实现细节是转换步骤。 使用显式去除前导零`lstrip('0')`，并且空字符串大小写映射为零。 这可以确保正确处理完全取消或零重结果等情况。 

## 工作示例

 ### 示例 1：1000 / 1000

 我们从计数开始：

 - 分子：{1:1, 0:3}
 - 分母：{1:1, 0:3}

 我们首先考虑数字 0。 

| 数字| 保留在两个 | 剩余分子 | 剩余分母|
 | --- | --- | --- | --- |
 | 0 | 0 | 000 | 000 000 | 000
 | 0 | 1 | 00 | 00 00 | 00
 | 0 | 2 | 0 | 0 |
 | 0 | 3 | | |

 DFS 探索所有有效的取消，当除隐式归一化之外的所有数字都被删除时，获得最小分子，产生 1 / 1。 

这表明，导致空字符串的完全取消被视为零，并且规范化正确地折叠了对称情况。 

### 示例 2：2232 / 162936

 位数计数：

 - 2232：{2:3, 3:1}
 - 162936：{1:1、2:1、3:1、6:2、9:1}

 我们只能部分取消数字2和3。 

DFS 会尝试所有有效的匹配：

 - 取消 0 个二元和 0 个三元
 - 取消 1 两个
 - 取消 1 个二和 1 个三

 最好的结果是去掉一位数 2，留下分子 232。 

该轨迹表明最佳解决方案取决于选择性取消而不是最大取消。

 ## 边缘情况

 第一个边缘情况是完全取消。 对于像 1000 / 1000 这样的输入，基于字符串的简单方法可能会在删除所有数字后生成空字符串，这可能会被误解为无效。 该算法显式地将空结果转换为 0，确保与数值解释的一致性。 

第二个边缘情况是前导零。 例如，如果取消生成“007 / 123”，则数字构造步骤 lstrip('0') 确保分子变为 7。如果没有这种标准化，比较就会错误地将同一整数的不同表示形式视为不同的。 

第三种边缘情况是不对称消除，其中由于频率不同，一侧可能比另一侧丢失更多的数字。 DFS 强制实施数字级可行性，确保数字被删除的次数不会超过其在任一数字中出现的次数。
