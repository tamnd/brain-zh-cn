 ## 解决方案
 ## 问题理解

 我们得到一个正整数数组。 目标是使用一系列操作将每个值减少为零。 在一次操作中，我们选择一个索引列表$B_1, B_2, \dots, B_m$。 这$i$此列表中的第 - 个位置贡献减去$2^{i-1}$到对应的数组元素$A_{B_i}$。 相同的数组索引可以在同一操作中出现多次，并且每次出现都会根据其在所选序列中的位置接收不同的 2 的幂。 

关键的自由度是每次操作都让我们分配权重集$1, 2, 4, 8, \dots, 2^{m-1}$以任意顺序跨越数组位置，但每个权重每次操作仅使用一次。 

任务是尽量减少需要的此类操作的数量，以便所有数组值都变为零。 

约束允许最多$10^5$每个测试套件的元素总数，其值高达$10^9$。 这立即表明任何逐步模拟操作的方法都是不可能的。 2 的幂的结构强烈暗示了按位或二进制分解的观点，因为每个操作本质上是提供每个 2 的幂的一个副本的“预算”。 

揭示天真的推理失败的一个微妙的例子是当一个数字很大而其他数字很小时。 例如，如果我们首先尝试贪婪地修复大元素，我们可能会低效地浪费小的权力，即使重新分配哪个元素接收哪个权力会减少操作数量。 正确的解决方案必须全局平衡所有元素中使用 2 的幂的频率。 

## 方法

 强力解释将直接模拟操作。 在每个操作中，我们尝试分配序列$1, 2, 4, \dots$按某种顺序索引，更新数组直到所有值达到零。 即使我们在选择分配方面很聪明，每个操作也只会删除一组结构化的值，并且我们仍然需要潜在的$O(\max A_i)$总减少量分布在各个业务部门。 由于值可以达到$10^9$，这种方法立即不可行。 

转折点是重新解释操作真正提供的内容。 每个操作恰好贡献 2 的每个幂的一个副本$2^k$对于所有有效的$k$直到选定的长度。 在所有操作中，我们有效地决定每个功率的次数$2^k$被使用，并且每次操作必须将每次使用分配给一个数组索引。 

这将问题转化为调度视图。 对于每个位位置$k$，每次我们使用$2^k$在任何分解中$A_i$，我们在操作的“位层”中消耗一个容量单位。 由于每个操作最多提供每个位层的一个单元，因此操作的数量必须至少是所有位层上的最大负载。 剩下的问题是我们是否总是可以选择所有数字的分解以使这个界限是紧的，答案是肯定的，通过使用自然二进制分解，这避免了不必要的分裂，而这种分裂只会增加较低位的拥塞。

 | --- | --- | --- | --- |
 | 暴力模拟| 指数| O(n) | 太慢了|
 | 位层计数| O(n log A) | O(n log A) | O(log A) | 已接受 |

 ## 算法演练

 我们将每个数字重新表述为二的幂的贡献。 

1. 对于每个值$A_i$，将其分解为其二进制表示形式。 每个设置位的位置$k$意味着我们需要一个实例$2^k$分配给索引$i$。 
2.维护比特位置上的频率阵列。 对于每个位位置$k$，计算有多少个索引需要该位。 
3. 答案是所有这些位频率中的最大值。 

该最大值表示阵列中最“需要”的位级别。 由于每个操作只能满足每个位位置一个单位的需求，因此该层成为瓶颈。 

### 为什么它有效

 每个操作都为 2 的每一次幂提供一个可用槽。 从列角度考虑，每个位位置的行为就像一个并行资源，其容量等于操作数。 阵列的任何分解都会在每列上产生等于该功率使用次数的负载。 任何操作都不能为同一列中的多个单元提供服务，因此操作数必须至少为最大列负载。 

使用二进制表示可确保我们永远不会创建不必要的较小幂的重复，这些重复会增加列负载而不减少另一个列负载。 因此，二元分解引起的列负载已经很小，使得最大频率既必要又充分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        cnt = [0] * 31
        
        for x in a:
            b = 0
            while x:
                if x & 1:
                    cnt[b] += 1
                x >>= 1
                b += 1
        
        print(max(cnt))

if __name__ == "__main__":
    solve()
```代码独立处理每个测试用例。 对于每个数字，它都会扫描其二进制表示形式并递增相应位位置的计数器。 最终答案是所有位位置的最大计数器值。 

一个微妙的实现细节是我们不尝试模拟操作本身。 整个解决方案依赖于只有每比特频率很重要的观察，因此跟踪计数就足够了。 循环运行最多 30 位，这足以满足高达$10^9$。

 ## 边缘情况

 一种重要的边缘情况是所有数字都相同。 例如，使用输入`A = [7, 7, 7]`，每一位位置都完全饱和。 该算法独立地计算每个位并在所有位置返回相同的值。 执行产生计数`[3, 3, 3]`，所以答案是 3，匹配贡献于每一位的元素数量。 

另一个边缘情况是数字是 2 的纯幂。 用于输入`[1, 2, 4, 8]`，每个数字激活不同的位位置，因此没有一列累积超过一个要求。 该算法产生的最大值为 1，并且每个操作都可以独立处理所有贡献而不会发生冲突。
