 ## 解决方案
 ## 问题理解

 我们得到一条由直线排列的离散位置组成的路径，每个位置都有一个我们可以自由选择的高度值。 路径必须从高度零开始并在高度零结束，并且在任何两个相邻位置之间，高度最多只能向上或向下改变一个单位。 在所有此类有效高度剖面中，任务是确定沿路径任何位置可能出现的最大可能高度。 

另一种看待这个问题的方法是，我们正在构建一个长度为 L 的“地形轮廓”。我们想要尽可能最高的山峰，但我们受到两个条件的限制：地形必须在两端接触地面，并且陡坡不能大于每步一个单位。 问题是，在这些限制下，一座山能造多高。 

约束 L 高达 100000 意味着我们需要一个在 L 中最坏是线性或对数的解决方案。尝试所有可能的高度配置的任何构造或模拟都是不可能的，因为有效序列的数量呈指数增长。 即使对线段采用二次方法也会太慢。 

当长度很小时，就会出现关键的边缘情况。 当 L 等于 1 或 2 时，地形被迫始终保持在零高度或立即返回零，因此不可能出现峰值。 对于稍大的值，例如 L 等于 3 或 4，只能形成一个小的三角形峰值，并且如果忽略两端必须为零的边界约束，很容易错误地假设可能存在更大的峰值。 

例如，当 L 等于 4 时，有效配置为 0, 0, 1, 0，最大高度为 1。天真的直觉可能表明，由于上升太快，高度 2 是可能的，但相邻差异约束阻止了这一点。 

## 方法

 一个蛮力的想法是尝试构造所有从零开始和结束的长度为 L 的有效高度序列，验证邻接约束，并计算每个序列中的最大高度。 这在原则上是正确的，因为它枚举了所有可行的地形，但序列的数量像约束随机游走一样增长。 即使对于适度的 L，可能性的数量也会变得天文数字，使得除了非常小的输入之外完全不可行。 

问题的结构是经典的整数约束游走。 每一步最多改变高度 1，并且我们从零开始和结束。 最高的可能峰值对应于在必须返回到零之前尽可能向上推动步行。 限制因素是，一旦我们每一步增加一高度，我们还需要足够的剩余步数才能在相同的坡度约束下下降到零。 

这导致了简单的几何解释。 如果我们想要达到高度 H，我们至少需要 H 步从零开始爬升，因为每次移动最多可以增加 1 步。 到达 H 后，我们还需要另一个 H 步才能下降到零。 因此，如果我们从位置角度考虑，高度为 H 的山峰至少需要 2H 步才能上下移动，再加上山峰本身的一个位置。 这直接将最大可能高度与可用长度 L 连接起来。 

因此，问题简化为找到最大整数 H，使得高度为 H 的对称山适合在 L 个位置内。 该结构是一个离散三角形。 如果我们将峰放在中间，则所需的总长度为 2H + 1。因此，我们希望最大 H 使得 2H + 1 ≤ L。 

根据这个不等式，答案变为下限((L - 1) / 2)。

 ## 算法演练

1. 读取表示路径中的位置数的整数L。 目标是确定在移动和边界约束下最高有效峰值可以有多高。 
2. 观察到，任何高度为 H 的峰值都必须通过从 0 单调增加到 H，然后单调减少回到 0 来形成。在最好的情况下，每次增加或减少步骤都会将高度恰好改变 1，因为不允许出现较大的跳跃。 
3. 计算形成这样一个峰所需的最少位置数。 从 0 到 H 需要 H 个步骤，从 H 回到 0 又需要 H 个步骤，这已经考虑了 2H 转换。 由于峰值位置只计算一次，因此所需的位置总数为 2H + 1。 
4. 通过要求 2H + 1 ≤ L 来强化可行性。这个不等式反映了这样一个事实：整座山必须适合可用的路段长度。 
5. 求解 H 的不等式，得到 H ≤ (L - 1) / 2。满足此约束的最大整数 H 是最终答案。 

### 为什么它有效

 任何达到最大值 H 的有效高度序列必须至少花费 H 步增加和 H 步减少，因为每次移动高度最多改变 1。 这对支撑高度 H 的峰值所需的长度产生了严格的下限。对称构造 0, 1, 2, ..., H, ..., 2, 1, 0 精确地实现了该界限，因此不存在更有效的构造。 由于必然性论证和匹配构造都存在，因此界限很紧并且公式是精确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

L = int(input().strip())

print((L - 1) // 2)
```整个解决方案取决于将高度约束转化为离散山体的最小长度要求。 该代码直接应用导出的公式而不进行模拟，因为对于简化为单个不等式的问题，任何迭代构造都是不必要的开销。 

唯一的微妙之处是整数除法。 使用 (L - 1) // 2 正确地对 (L - 1) / 2 的实际值进行取底，这符合 H 必须是满足约束的整数的要求。

 ## 边缘情况

 对于 L 等于 1，序列仅包含一个位置，该位置必须同时是开始和结束，强制所有位置的高度为零。 该算法返回 (1 - 1) // 2 = 0，与不可能上升的事实相匹配。 

当 L 等于 2 时，在 ±1 约束下没有上升和下降的空间，同时以零开始和结束。 该公式给出 (2 - 1) // 2 = 0，正确防止任何假峰。 

对于 L 等于 3，唯一有效的结构是高度为 1 的单峰，即 0, 1, 0。计算结果为 (3 - 1) // 2 = 1，与最佳结构完全匹配。 

对于较大的 L，该算法隐式构建最大对称山形，即使它从未显式构建。 正确性来自这样一个事实：任何超过此高度的尝试都需要违反步长约束或边界约束。
