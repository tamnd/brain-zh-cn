 ## 解决方案
 ## 问题理解

 给定一个整数数组，我们需要计算所有无序索引对的全局成对表达式。 对于每对位置$i < j$，我们将值之间的差值乘以它们的总和，将其累加到所有对上。 

展开表达式有助于揭示其结构：$$|a_i - a_j| \cdot (a_i + a_j)$$这是两个元素的对称函数，因此答案仅取决于所有对，而不取决于超出值的顺序或结构。 

输入大小可达$2 \cdot 10^5$，并且值一样大$10^8$。 迭代所有对的二次方法立即太慢，因为它需要大约$4 \cdot 10^{10}$最坏情况下的操作。 

如果不小心处理，天真的双循环也有中间和中整数溢出的风险，但主要障碍是时间复杂度而不是算术范围。 

当所有元素都相等时，会出现微妙的边缘情况。 在这种情况下，每一项都变为零，因为$|a_i - a_j| = 0$，所以答案一定是零。 任何在不考虑绝对值的情况下错误地扩展或重新排列项的方法仍可能由于符号错误而计算出非零结果。 

另一个重要的情况是当值严格增加或减少时。 例如：

 输入：```
3
1 2 3
```正确的输出是：```
8
```忽略绝对值或假设项之间取消的粗心代数简化在这里将失败，因为即使最终的和是对称的，绝对差内的排序也很重要。 

## 方法

 蛮力方法迭代所有对并直接评估公式。 这在正确性方面是简单的，因为它与定义完全匹配，但它需要$O(n^2)$评价。 和$n = 2 \cdot 10^5$，这导致大约$2 \cdot 10^{10}$的操作，这是不可行的。 

为了优化，我们首先重写表达式：$$|a_i - a_j| \cdot (a_i + a_j)$$认为$a_i \ge a_j$。 然后：$$(a_i - a_j)(a_i + a_j) = a_i^2 - a_j^2$$这通过根据排序拆分对来完全消除绝对值。 如果我们对数组进行排序，每一对都会贡献一个确定性的符号结构：$i > j$，我们总是有$a_i \ge a_j$。 这会将全局总和变为：$$\sum_{i > j} (a_i^2 - a_j^2)$$现在我们将贡献分开。 每个元素多次显示为正平方项，并多次显示为负平方项，具体取决于其在排序数组中的位置。 通过前缀计数，我们可以计算每个前缀出现了多少次$a_i^2$排序后在线性时间内正负相加。 

关键的观察是排序将绝对值结构转换为单调排序问题，剩下的任务变成使用前缀计数而不是枚举对来跟踪贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 排序+贡献计数 |$O(n \log n)$|$O(1)$额外（不包括排序）| 已接受 |

 ## 算法演练

 1. 以非降序对数组进行排序，使得对于任意对$i < j$，我们有$a_i \le a_j$。 这固定了绝对值的方向。 
2. 使用恒等式重写每对贡献：$$|a_i - a_j|(a_i + a_j) = a_j^2 - a_i^2 \quad \text{for } i < j$$这将问题转化为跟踪每个平方值出现正数和负数的次数。 
3. 从左到右遍历已排序的数组，同时维护值和平方值的运行前缀和。 
4. 对于每个位置$i$， 对待$a_i$作为具有较早索引的所有对中较大的元素。 它的贡献取决于它之前有多少元素及其累积和。 
5. 使用前缀聚合来累积结果，以便每个元素都贡献$O(1)$预处理后的时间。 
6. 对最后的和取模$10^9 + 7$处理大值。 

### 为什么它有效

 排序后，每对都有固定的顺序，这完全消除了绝对值歧义。 转换为平方差可确保每对贡献预先计算的前缀信息的线性组合。 由于在这种结构化分解中每对都只计算一次，因此不会发生取消或重复，并且最终的总和与原始定义匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

n = int(input())
a = list(map(int, input().split()))
a.sort()

pref_sum = 0
pref_sq = 0
ans = 0

for i, x in enumerate(a):
    # contribution of x with all previous elements
    # (x - y)(x + y) = x^2 - y^2
    cnt_left = i
    
    ans = (ans + cnt_left * (x * x % MOD) - pref_sq) % MOD
    
    pref_sum += x
    pref_sq = (pref_sq + x * x) % MOD

print(ans % MOD)
```排序顺序确保在处理元素时$x$，所有先前的元素都保证小于或等于，因此每对在一致的方向上只计算一次。 变量`pref_sq`维护所有早期元素的平方和，这是评估所有涉及的贡献所需的唯一历史信息$x$。 

一个常见的错误是试图不必要地维护前缀和和后缀和； 仅需要平方前缀累加，因为该表达式在扩展后消除了线性交叉项。

 ## 边缘情况

 当所有值都相同时，算法仍然执行完整的前缀累加，但每一项都会取消，因为平方贡献与前缀减法完全匹配。 用于输入`1 1 1 1`，排序后的数组保持不变，并且每一步产生的增量贡献为零。 

当数组只有一个元素时，循环会执行一次，但不会做出任何贡献，因为前缀大小为零。 这可以防止意外的负索引或无效的对计数。 

对于严格增加的序列，例如`1 2 3 4`，每个元素贡献逐渐增大的平方值乘以其索引位置，并且前缀减法确保每对以正确的方向精确计数一次，与扩展代数形式匹配。
