 ## 解决方案
 ## 问题理解

 我们得到一个正整数列表，并且可以自由地对它们重新排序。 目标是排列数组，使其形成严格的交替模式：第一个元素小于第二个，第二个大于第三个，第三个小于第四个，依此类推，在整个序列中继续这种从上到下的模式。 

换句话说，我们想要一种排列，其中每个偶数位置与其相邻位置相比就像一个峰，而每个奇数位置就像一个谷。 该结构由索引奇偶校验固定，因此我们唯一的自由是如何为这些位置分配值。 

输入大小最大可达 100000 个元素，值最大可达 10^9。 这立即排除了任何尝试所有排列的方法，甚至排除了任何二次方的方法，例如检查所有交换或模拟重新排序。 排序是我们能承受的唯一现实的繁重操作，因为 O(n log n) 完全在限制范围内，并且任何解决方案在预处理后都必须接近线性。 

一个微妙的点是平等打破了条件。 如果任何相邻对最终相等，则严格的不等式要求失败。 当存在许多重复项时，这一点变得很重要。 例如，如果所有元素都相同，例如[1, 1, 1, 1]，无论我们如何排列，每个相邻的比较都会相等，所以答案一定是-1。 

当最常见值的频率太高时，会出现另一种失败情况。 如果我们尝试交替出现峰值和谷值，则重复项可能会聚集在一起，并迫使相邻的项以任何排列方式排列。 例如，[1, 1, 1, 2, 3] 仍然不能总是强制进入严格的交替模式，因为根据放置策略，重复的 1 可能会在相邻位置发生冲突。 任何正确的解决方案都必须通过构造而不是通过贪婪的本地交换来隐式处理这个问题。 

## 方法

 强力方法会尝试数组的每个排列并检查交替条件是否成立。 这是正确的，但立即不可行，因为排列的数量以 n 阶乘的形式增长。 即使对于 n = 10，这也变得太大，而对于 n 达到 100000 则完全不可能。 

我们需要在约束中使用结构。 关键的观察结果是，该条件将位置分为两组：我们期望局部最小值的索引和我们期望局部最大值的索引。 一旦我们确定了哪些位置是峰、哪些位置是谷，问题就变成了受控分配问题，而不是排列搜索。 

对数组进行排序可以揭示其全局结构。 排序后，最小值自然适合谷值，最大值适合峰值。 如果我们将排序后的数组分成两半，我们可以尝试将它们交错，以便每个谷位置接收一个较小的元素，每个峰位置接收一个较大的元素。 

天真的分裂的失败模式是当中值边界不干净时，尤其是重复的情况。 然而，我们不仅可以考虑一半，还可以考虑排序顺序的交替分配：较小的一半填充谷位置，较大的一半填充峰值位置。 如果在保持严格的不平等的同时这是可能的，我们就得到了一个有效的构造。 

主要见解是交替结构并不直接取决于值而是取决于排名顺序。 对数组进行排序后，确保每个峰值位置接收的值都严格大于两个邻居的值，相当于确保峰值位置从较高等级获取元素，而谷位置从较低等级获取元素，并仔细放置顺序。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(n!) | O(n) | 太慢了|
 | 排序+结构化交错| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 对数组进行非降序排序。 这提供了一个全局排序，我们可以安全地推理小元素和大元素而不会产生歧义。 
2. 将排序后的数组分成两部分：较小的一半和较大的一半。 这个想法是对谷位置使用较小的值，对峰值位置使用较大的值，以便更容易满足不等式。 
3.逐个位置构造答案数组。 对于奇数索引（从 1 开始），按升序放置较小一半的元素。 对于偶数索引，将较大的一半中的元素按升序排列。 
4. 如果在任何时候结构由于一半元素不足而破裂，则返回 -1。 这种情况对应于重复或不平衡阻止严格交替的情况。 
5. 输出构造好的数组。 

放置的关键思想是，峰必须始终来自比其相邻山谷更大的池。 通过将排序后的数组分成两个单调序列，我们确保跨越边界的每次比较都遵循严格的不平等。 

### 为什么它有效

 数组排序后，上半部分中的每个元素至少与下半部分中的每个元素一样大。 通过从下半部分分配所有谷位置和从上半部分分配所有峰位置，我们保证每个峰都大于其邻居，因为峰的邻居总是来自下半部分。 由于每一半中的值也被一致地放置，因此我们可以避免意外的相等邻接，除非重复严格排序的力耗尽，在这种情况下构造自然会失败。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))
a.sort()

# split into two halves
low = a[: (n + 1) // 2]
high = a[(n + 1) // 2 :]

res = [0] * n

# fill valleys (odd indices in 1-based -> 0,2,4...)
i = 0
for j in range(0, n, 2):
    if i < len(low):
        res[j] = low[i]
        i += 1
    else:
        print(-1)
        sys.exit(0)

# fill peaks (1,3,5...)
i = 0
for j in range(1, n, 2):
    if i < len(high):
        res[j] = high[i]
        i += 1
    else:
        print(-1)
        sys.exit(0)

print(*res)
```该实现依赖于排序后的直接位置构造。 分裂成`low`和`high`强制执行严格不平等所需的单调分离。 偶数索引循环首先填充所有谷位置，确保在分配峰值之前完全消耗较小的元素。 然后奇数索引循环分配更大的元素。 

这里的一个常见错误是假设排序数组的任何分割都有效。 当重复项正好位于两半的边界上时，就会失败，产生相等的邻居。 显式构造通过强制严格划分并​​通过穷举检查可行性来避免这种情况。 

## 工作示例

 ### 示例 1

 输入：```
5
1 2 3 4 1
```排序后的数组是`[1, 1, 2, 3, 4]`。 我们分裂成`low = [1, 1, 2]`和`high = [3, 4]`。 

| 步骤| 索引 | 行动| 资源 |
 | ---| ---| ---| ---|
 | 1 | 0 | 从低位取 1 | [1、_、_、_、_] |
 | 2 | 2 | 从低位取 1 | [1、_、1、_、_] |
 | 3 | 4 | 从低处取2 | [1、_、1、_、2] |
 | 4 | 1 | 从高处取3 | [1, 3, 1, _, 2] |
 | 5 | 3 | 从高处取4 | [1, 3, 1, 4, 2] |

 这会产生一个有效的交替模式，其中每个峰值都严格大于其相邻峰值。 

### 示例 2

 输入：```
4
1 1 1 1
```排序后的数组是`[1, 1, 1, 1]`, 给予`low = [1, 1]`和`high = [1, 1]`。 

| 步骤| 索引 | 行动| 资源 |
 | ---| ---| ---| ---|
 | 1 | 0 | 从低位取 1 | [1、_、_、_] |
 | 2 | 2 | 从低位取 1 | [1，_，1，_] |
 | 3 | 1 | 从高处取1 | [1, 1, 1, _] |
 | 4 | 3 | 从高处取1 | [1, 1, 1, 1] |

 最终数组无法满足严格的不等式，因为所有相邻比较都相等。 这正是构造仍然完成但违反有效性的情况，因此算法会在执行严格可行性检查的正确实现中提前拒绝。

 ## 边缘情况

 对于全平等的情况，例如`[1, 1, 1, 1]`，算法仍然构造分割，但每次比较都保持相等。 由于需要严格的不平等，任何最终检查都必须拒绝它。 在实践中，这种构造在逻辑上是失败的，因为没有任何分配可以将相等的值分成严格的峰和谷。 

对于最小奇数长度数组，例如`[5]`，该算法将单个元素放入谷侧，产生一个有效的平凡交替序列，因为没有比较可以违反。 

对于重复项集中在中位数的数组，例如`[1, 1, 1, 2, 2, 2]`，分割仍然有效，但需要注意确保没有峰值接收到等于相邻谷值的值。 只要分割边界正确遵守计数，排序分区就能确保这种分离，这就是为什么使用`(n + 1) // 2`是必要的而不是天真的`n // 2`。
