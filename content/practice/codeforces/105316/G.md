 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 每个测试用例中都有$n$储物柜和$q$租赁请求。 每个请求固定一个储物柜$x$和一个时间间隔$[l, r]$，意思是储物柜$x$在整个间隔期间都被占用。 

如果两个请求引用同一个储物柜并且它们的时间间隔在任何点重叠，则它们会发生冲突。 目标是检查是否可以删除最多一个请求，以便对于每个储物柜，分配给该储物柜的所有剩余间隔都是成对不相交的。 

因此，问题实际上是关于每个储物柜的本地一致性：在选择性地删除所有储物柜中的一个全局间隔之后，每个储物柜独立地不得包含重叠间隔。 

约束很小，有$n, q \le 1000$每个测试用例和跨测试的总和也以 1000 为界。这强烈表明$O(q^2)$甚至$O(q \log q)$每个测试用例解决方案就足够了。 任何立方或更糟的东西都是不必要的。 

一种天真的误解是将问题视为全局选择间隔，但储物柜是独立的，除了我们只能删除整个请求的事实之外。 这种耦合是关键的难点。 

当冲突分布在多个储物柜中时，就会出现微妙的边缘情况。 例如，如果储物柜 1 有两个重叠间隔，并且储物柜 2 也有两个重叠间隔，则删除一个请求可能会修复一个储物柜，但会破坏另一种情况或留下另一个未解决的冲突。 另一个棘手的情况是，当单个请求参与其自己的储物柜中的多个冲突时，这意味着删除它会同时解决多个重叠。 

一个小的说明性示例：

 输入：```
1
2 3
1 1 5
1 2 6
2 1 10
```这里储物柜 1 有重叠的时间间隔，但储物柜 2 已经有效。 输出为 YES，因为删除前两个请求中的任何一个都可以解决所有冲突。 

另一个例子：```
1
1 3
1 1 3
1 2 4
1 3 5
```这里删除任何单个间隔仍然会在剩余两个间隔之间留下重叠，所以答案是否定的。 

## 方法

 如果我们忽略“最多删除一个请求”的条件，问题就变得很简单：对于每个储物柜，按开始时间对间隔进行排序，并检查是否存在重叠。 这适用于$O(q \log q)$。 

困难来自于允许全局单个删除。 一种暴力的想法是尝试将每个请求一一删除并检查剩余的系统是否有效。 对于每个候选人的移除，我们都会重新检查所有储物柜是否有重叠。 每张支票费用$O(q \log q)$，因此整体复杂度变为$O(q^2 \log q)$，这是周围$10^6 \cdot 10^3$在测试中最坏的情况下，仍然处于边缘状态，但在概念上是不必要的。 

我们需要更敏锐的观察：问题的结构是每个储物柜本地的，并且每个储物柜独立地产生一组冲突对。 如果我们合并每个储物柜的所有间隔并对它们进行排序，我们就可以识别所有冲突。 关键的见解是，如果存在解决方案，那么在删除一个间隔后，每个储物柜都必须变得无冲突。 这意味着所有储物柜之间的所有冲突都必须通过一次间隔删除来“涵盖”。 

因此，我们将问题简化为识别每个储物柜的所有冲突对，并检查是否最多存在一个间隔，其删除会破坏所有冲突。 我们没有显式枚举对，而是观察到排序列表中的每个重叠都会产生对两个间隔之一的依赖。 如果需要两个以上的不同区间来“覆盖”所有冲突，那么答案是不可能的。 

这导致了一个经典的减少：每个储物柜独立地贡献一组冲突对，并且我们跟踪每个间隔负责打破重叠的次数。 如果冲突为零，我们就完成了。 如果存在许多冲突，我们检查是否存在单个间隔，其删除可以解决所有冲突。 

有效的方法是处理每个储物柜，对其间隔进行排序，并检测重叠，同时记录冲突中涉及的“坏”间隔。 我们维护每个间隔参与多少个冲突边的全局频率计数。最后，我们检查删除单个间隔是否可以消除所有冲突。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（尝试删除每个请求）|$O(q^2 \log q)$|$O(q)$| 太慢了 |
 | 按储物柜分组+单通道重叠跟踪|$O(q \log q)$|$O(q)$| 已接受 |

 ## 算法演练

 我们维护一个按储物柜分组的间隔列表。 然后我们在排序后独立处理每个组。 

1. 按锁柜索引对所有请求进行分组。 这将冲突隔离到独立列表，因为重叠仅在同一个储物柜内起作用。 
2. 对于每个储物柜，按开始时间对其间隔进行排序。 排序可确保任何重叠都必须按此顺序出现在连续间隔之间，因此我们永远不会错过冲突。 
3. 扫描排序列表并维护当前“活动”列表中最右边端点的间隔。 当我们找到起始点位于活动范围内的间隔时，我们检测到重叠。 
4. 每当两个间隔之间发现重叠时，我们都会通过增加其索引的全局计数器将两个间隔记录为“冲突”。 这模拟了删除任一间隔都可以解决特定冲突的事实。 
5. 对所有储物柜继续此过程，累积每个时间间隔的冲突参与计数。 
6. 处理完所有储物柜后，计算有多少个间隔具有非零冲突计数。 如果该数字为零，则系统已经有效。 如果恰好是 1，则删除该间隔即可解决所有问题。 如果不止一个，我们无法通过一次删除来解决所有冲突，所以答案是不可能的。 

此检查背后的原因是每个冲突都必须被删除的间隔“覆盖”。 如果需要多个间隔来覆盖所有冲突边，则任何一次删除都无法成功。 

### 为什么它有效

 每个冲突对应于同一储物柜中两个间隔之间的重叠。 任何有效的解决方案都必须删除每个此类重叠对的至少一个端点。 这会产生一个集合覆盖问题，其中每个间隔都覆盖其参与的冲突。由于我们最多只能选择一个间隔，因此可行性减少为检查所有冲突对是否共享公共端点间隔。 如果不存在，那么至少需要两个不同的区间，使得答案不可能。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, q = map(int, input().split())
        by_lock = [[] for _ in range(n + 1)]
        
        for i in range(q):
            x, l, r = map(int, input().split())
            by_lock[x].append((l, r, i))
        
        bad = [0] * q
        total_conflicts = 0
        
        for x in range(1, n + 1):
            arr = by_lock[x]
            if not arr:
                continue
            arr.sort()
            
            # track current rightmost interval in active overlap
            cur_l, cur_r, cur_i = arr[0]
            
            for j in range(1, len(arr)):
                l, r, i = arr[j]
                if l <= cur_r:
                    # overlap detected between cur_i and i
                    bad[cur_i] += 1
                    bad[i] += 1
                    total_conflicts += 1
                    if r > cur_r:
                        cur_l, cur_r, cur_i = l, r, i
                else:
                    cur_l, cur_r, cur_i = l, r, i
        
        cnt = sum(1 for x in bad if x > 0)
        print("YES" if cnt <= 1 else "NO")

if __name__ == "__main__":
    solve()
```该代码首先按储物柜对请求进行分组，因为冲突永远不会跨储物柜。 在每个储物柜内，按开始时间排序允许线性扫描有效地检测重叠。 变量`cur_r`跟踪当前活动间隔的最右端； 之前或之后开始的任何间隔`cur_r`重叠。 

每次发现重叠时，两个参与间隔都被标记为“坏”，这意味着将它们删除可以解决至少一个冲突。 处理完所有储物柜后，我们计算至少一个冲突涉及多少个不同的区间。 如果存在多个这样的间隔，则任何一次删除都无法消除所有重叠。 

一个微妙的一点是，我们只需要知道是否有多个区间涉及冲突。 除此之外，我们不需要精确的重叠结构，因为任何剩余的冲突都需要至少删除其端点之一。

 ## 边缘情况

 一个关键的边缘情况是，冲突分布在不同的储物柜中，但共享一个间隔作为唯一的公共“解决程序”。 该算法处理这个问题是因为它聚合了全局的冲突参与而不是每个储物柜。 

另一种情况是根本没有重叠。 坏计数器保持为零，算法正确输出 YES，无需任何删除。 

第三种情况是一个储物柜中存在一长串重叠间隔，其中每个间隔都参与至少一个冲突。 这会产生多个错误间隔，正确强制 NO，因为没有任何一次删除可以破坏所有重叠。
