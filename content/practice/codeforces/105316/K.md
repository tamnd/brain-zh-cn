 ## 解决方案
 ## 问题理解

 每个测试用例都描述了一个课堂快照。 对于每个测试用例，我们都会分配几个学生，对于每个学生，我们都会收到代表他们在八个不同科目的分数的恰好八个整数。 任务是将八个科目分数加在一起，将每个学生的记录压缩为一个数字，然后输出每个学生的总分。 

该结构是分层的：首先我们读取有多少个测试用例，然后对于每个测试用例，我们读取有多少学生属于该用例，然后我们独立处理每个学生。 学生之间或测试用例之间不存在交互，因此一旦读取了八个数字，就可以立即计算每个总和。 

约束足够小，任何简单的处理策略就足够了。 测试用例数量最多为100个，每个测试用例最多有100名学生。 每个学生恰好贡献 8 个整数，因此整个输入处理的整数总数最多为 100 × 100 × 8 = 80000。这完全在简单线性扫描的限制范围内。 

一个常见的实施陷阱来自于错误地处理输入结构。 一个错误是在不考虑测试用例边界的情况下以平面序列读取所有数字，这仍然可以产生正确的总和，但分组错误。 另一个问题是忘记重置每个测试用例循环，导致打印跨测试用例而不是每个学生的累积结果。 例如，如果程序员在学生的单个变量中累积总和，则输出可能会错误地跨行增长，而不是按学生重置。 

## 方法

 最直接的方法是读取每个学生的八个分数并立即计算它们的总和。 这是正确的，因为每个学生的最终结果仅取决于他们自己的八个值，而不依赖于其他学生或测试用例。 蛮力解释本质上与最佳解决方案相同：显式地对每个学生的所有八个整数求和。 

如果我们将其描述为一种幼稚的策略，人们可能会想象首先将所有学生分数存储在一个结构中，然后再次迭代以计算总和。 这仍然是正确的，但由于额外的存储和第二次传递，在内存和时间复杂度方面引入了不必要的开销。 每个学生实际完成的作业始终是恒定的，因为它只是八个加法。 

关键的观察结果是该问题的元素之间没有耦合。 每个学生形成一个由八个值组成的独立块，因此可以以流方式完成处理。 这消除了对数组或预处理的任何需要。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（存储然后求和）| O(n) | O(n) | 已接受 |
 | 最佳（流媒体总和）| O(n) | O(1) | O(1) | 已接受 |

 这里n指的是处理的整数总数，或者相当于学生总数乘以8。 

## 算法演练

 我们逐个测试用例处理输入测试用例，并且在每个测试用例中我们独立处理每个学生。 

1.读取测试用例的数量。 这定义了我们将处理多少个独立的数据组。 
2. 对于每个测试用例，读取学生人数。 这控制了学生数据的行数。 
3. 对于每个学生，从输入行中准确读取八个整数。 这些代表固定宽度的特征向量。 
4. 读取这八个整数后立即计算它们的和。 这避免了存储不必要的中间数据。 
5. 在转到下一个学生之前，输出该学生的计算总和。 
6. 重复，直到处理完当前测试用例中的所有学生，然后移至下一个测试用例。

我们可以安全地立即输出每个学生的原因是没有未来的依赖性。 一旦读取了八个值，结果就完全确定，并且不能根据任何其他输入而改变。 

### 为什么它有效

 正确性取决于这样一个事实：从输入到输出的映射在学生之间是可分离的。 每个输出值都是八个输入整数的确定性函数。 由于没有转换使用学生或测试用例之间的共享状态，因此单独计算每个总和会产生全局正确的结果。 该算法有效地评估一组独立的算术表达式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        for _ in range(n):
            nums = list(map(int, input().split()))
            out.append(str(sum(nums)))
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案使用快速 I/O 读取输入，因为在最坏的情况下行数可能高达 10,000。 对于每个学生行，它将八个整数转换为列表并直接计算它们的总和。 

一个微妙的实现细节是缓冲输出。 结果不是逐行打印，而是收集到列表中并在最后打印一次。 这避免了重复的 I/O 开销，这在 Python 中即使对于很小的约束也很重要。 

另一个重要的细节是确保每个学生线路都是独立处理的。 循环结构保证学生之间或测试用例之间不携带任何状态。

 ## 边缘情况

 一个学生和八个相同值的最小案例证实该算法不依赖于求和之外的任何结构。 用于输入`1 1 1 1 1 1 1 1`，循环读取该行一次，将总和计算为 8，并立即输出，与预期行为相匹配。 

具有多个测试用例的用例可确保它们之间没有残留状态泄漏。 由于该算法永远不会跨测试用例进行累积，因此每个组都是独立处理的，并且输出在每个边界处自然重置。
