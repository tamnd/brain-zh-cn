 ## 解决方案
 ## 问题理解

 我们维护一个带标签的字符串的集合。 每个字符串都是由查询引入的，从那一刻起，它的行为就像一个对象，其标识符等于它插入的时间。 除了每个字符串之外，我们还存储一个数值。 随着时间的推移，字符串可以被删除，并且这些插入标识符的范围可以使其值被覆盖或递增。 

最困难的操作是对模式字符串 S 的查询。我们必须计算 S 中有多少个子字符串与当前活动的数据库字符串匹配，这些字符串的值位于给定的数字区间内。 子字符串每次出现都会计算一次，因此重叠和重复出现都会产生影响。 

该结构在两个独立的维度上是动态的。 一维是时间，按插入顺序索引。 另一种是值，通过范围赋值和范围添加不断修改。 该查询结合了两个维度，按时间约束选择活动字符串并按值约束进行过滤，同时还将它们作为查询字符串的子字符串进行匹配。 

这些约束迫使我们远离对所有活动字符串的任何每次查询扫描。 由于操作最多为 200000 次，并且字符串总长度也以 200000 为界限，因此任何针对每个查询重新计算子字符串匹配的解决方案都将立即超出时间限制。 类似地，由于二次爆炸，维护每个字符串的子字符串直接列表也是不可行的。 

重叠子字符串和重复插入会产生微妙的边缘情况。 

例如，如果我们插入字符串`"a"`,`"a"`， 和`"aa"`，然后查询`"aa"`具有完整的值范围，两次出现`"a"`作为子串和单个`"aa"`贡献。 对子字符串或字符串进行简单的重复删除会产生不正确的计数。 

另一个陷阱是通过插入索引进行删除。 由于删除是指第 i 个插入操作，而不是当前位置，因此删除后压缩数组的实现将导致索引错位并默默地破坏未来的范围更新。 

## 方法

 暴力解释很简单。 我们存储每个活动字符串及其插入索引和值。 对于类型 5 查询，我们迭代所有活动字符串，检查它们的值是否位于区间内，如果是，则测试它们的字符串是否作为 S 的子字符串出现。子字符串检查可以通过每个模式的简单扫描或字符串匹配算法来完成。 

这可以正常工作，但速度太慢。 即使我们使用高效的子字符串匹配（例如每个存储字符串的 KMP），每个查询仍然会触及所有活动字符串。 插入次数高达 200000 次时，在最坏的情况下会退化为大约 200000 × 200000 次操作，这是不可能的。 

关键的观察结果是，存储的字符串集在插入后内容是静态的。 只是他们的价值观和活跃状态发生了变化。 这建议将字符串匹配与动态过滤分开。 

我们不是在每个查询中迭代数据库字符串，而是反转该过程：我们考虑 S 的子字符串并询问它们是否存在于数据库中。 由于长度之和很小，我们可以枚举 S 的所有子串并将它们映射到字典条目。 

一旦子字符串标识被固定，问题就减少到为每个字符串维护其活动值在范围分配和加法下的动态多重集，并能够计算有多少值落在一个范围内。 这是一个经典的线段树，具有延迟传播，支持范围添加、范围分配和范围计数查询。 

最终的解决方案将字符串散列或基于 trie 的子字符串索引与按字符串标识分组的值的动态范围数据结构结合起来。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(Q × 总活动字符串 × | S | ) |
 | 最佳 | O((Σ | S | + Q) log Q) |

 ## 算法演练

 1. 为每个插入的字符串分配一个与其插入时间相等的唯一标识符。 存储其内容和指向其当前活动状态的指针。 
2. 通过枚举开始和结束位置，在总共 O(2 × 10^5) 时间内预先计算所有插入字符串的所有子字符串。 将每个子字符串插入到映射子字符串 → 包含该子字符串的字符串 ID 列表的哈希表中。 这会构建从子字符串到候选字符串的倒排索引。 
3. 对于每个字符串 ID，维护其当前值。 还维护一个基于插入索引的全局线段树，支持范围添加、范围分配和范围计数。 每个节点存储有关其范围内有多少活动字符串具有给定值分布的聚合信息。 
4. 对于类型 1 查询，将字符串插入活动集中，并将其值放入线段树的插入位置。 
5. 对于类型 2 查询，通过删除或标记其在线段树中的位置来停用该字符串，使其不再对查询有贡献。 
6. 对于类型 3 查询，对线段树中的索引 L 到 R 应用范围分配，使用延迟传播将该范围内的所有值设置为 x。 
7. 对于类型 4 查询，对线段树中的索引 L 到 R 应用范围加法，相应地更新存储的值。 
8. 对于模式 S 和范围 [L, R] 的类型 5 查询，枚举 S 的所有子字符串。对于每个子字符串，从哈希映射中检索候选字符串 ID，并检查每个候选字符串当前是否处于活动状态以及其值是否位于 [L, R] 中。 累积所有有效事件的计数。 

这样做的原因是模式子字符串和存储的字符串之间昂贵的匹配只能通过预先计算的结构来完成，并且值过滤被委托给支持对数更新和查询的结构。

正确性不变量是，在任何时候，线段树都会在其插入索引处存储每个活动字符串的准确当前值。 惰性操作可以保持一致性，因为范围分配会覆盖之前的添加，并且当没有待处理的分配时，范围添加可以正确组合。 因此，每个查询都会看到值的精确快照。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q = int(input())
    ops = []
    strings = []
    values = []
    alive = []

    for _ in range(q):
        parts = input().split()
        ops.append(parts)
        if parts[0] == '1':
            strings.append(parts[1])
            values.append(int(parts[2]))
            alive.append(True)

    n = len(strings)

    # Build substring index (naive but total length small)
    substr_map = {}
    for i, s in enumerate(strings):
        seen = set()
        for a in range(len(s)):
            cur = []
            for b in range(a, len(s)):
                cur.append(s[b])
                sub = ''.join(cur)
                if sub not in seen:
                    seen.add(sub)
                    if sub not in substr_map:
                        substr_map[sub] = []
                    substr_map[sub].append(i)

    import bisect

    # active set values
    active = [False] * n

    def get_active_values(ids):
        res = []
        for i in ids:
            if active[i]:
                res.append(values[i])
        return res

    for op in ops:
        if op[0] == '1':
            pass
        elif op[0] == '2':
            idx = int(op[1]) - 1
            active[idx] = False
        elif op[0] == '3':
            L, R, x = map(int, op[1:])
            for i in range(L-1, R):
                if i < n and active[i]:
                    values[i] = x
        elif op[0] == '4':
            L, R, x = map(int, op[1:])
            for i in range(L-1, R):
                if i < n and active[i]:
                    values[i] += x
        else:
            S, L, R = op[1], int(op[2]), int(op[3])
            ans = 0
            seen_sub = set()
            for a in range(len(S)):
                cur = []
                for b in range(a, len(S)):
                    cur.append(S[b])
                    sub = ''.join(cur)
                    if sub in seen_sub:
                        continue
                    seen_sub.add(sub)
                    if sub in substr_map:
                        for i in substr_map[sub]:
                            if active[i] and L <= values[i] <= R:
                                ans += 1
            print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循逆向思想。 我们构建每个插入字符串的所有子字符串，并在哈希映射中对它们进行索引。 这允许查询类型 5 避免扫描所有数据库字符串，而是直接跳转到相关候选。 

活动状态是单独跟踪的，因此删除操作只需禁用贡献而无需修改子字符串索引。 

为了清楚起见，这里简单地处理范围更新，尽管完全接受的解决方案将用线段树或平衡结构替换它以支持对数更新。 

子字符串枚举对每个字符串使用本地重复数据删除集，以避免多次插入同一源字符串中的重复子字符串。

查询`"aa"`值范围 [3, 5] 生成子字符串`"a"`,`"a"`， 和`"aa"`。

 ## 边缘情况

 关键的边缘情况是单个字符串内重复的子字符串。 例如，插入`"aaa"`不应三重计算`"a"`如果我们只关心每个子字符串出现的唯一字符串 ID，则来自同一个字符串。 该算法通过在预处理期间删除重复的子字符串来避免每个字符串的重复计算。 

另一个边缘情况是通过插入索引进行删除。 如果我们插入`"a"`,`"b"`,`"c"`然后删除第二个插入，仅`"b"`应该消失`"a"`和`"c"`保持不受影响。 这是通过为每个插入索引保留一个直接活动标志而不是压缩存储来处理的。 

多次重叠的范围更新也会导致混乱。 在加法过程中简单地覆盖值会丢失之前的增量。 预期的模型需要积累，因此必须按正确的顺序应用更新； 这就是为什么正确的解决方案会使用惰性传播或平衡结构来保持操作的可组合性。
