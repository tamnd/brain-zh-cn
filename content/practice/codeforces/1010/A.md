 ## 解决方案
 ## 问题理解

 我们有一条固定的太空旅行路线，从地球出发，依次访问几个中间行星，到达火星，然后返回地球。 每一段旅行都涉及从当前行星起飞并降落在下一个行星上，起飞和着陆时都会消耗燃料。 

关键的复杂性在于，燃料不仅会被消耗，还会增加必须提升和着陆的总质量。 在任何时候，火箭的总重量都包括固定有效载荷加上船上剩余的燃料。 当燃料燃烧时，火箭会变得更轻，这会影响未来的燃料需求。 

每个行星提供两种效率。 一个数字表示一单位燃料可以从该行星升空的总质量，另一个数字表示一单位燃料可以安全降落到该行星上的总质量。 这些效率因行星而异，因此每个起飞和着陆步骤都有自己的物理原理。 

问题是确定火箭能够完成整个循环并安全返回地球的最小初始装载地球量。 如果没有足够的燃料使旅程成为可能，那么答案是不可能的。 

约束足够小，每次模拟涉及几千步的解决方案是可以接受的。 当 n 达到 1000 时，即使 O(n log C) 或 O(n^2) 方法也足够了。 然而，尝试在没有单调结构的情况下直接猜测燃料的简单模拟将会失败。 

当其中一颗行星的着陆或起飞能力极弱时，就会出现一种微妙的边缘情况。 例如，如果某个行星的系数非常小，那么无论最初添加多少燃料，即使质量适中，也无法升起或着陆。 在这种情况下，正确的答案是 -1，除非仔细检查可行性，否则二分搜索仍然会返回不可行的值。 

如果假设燃料消耗与燃料重量无关，则会出现另一种故障模式。 这是不正确的，因为燃烧燃料会减少操作过程中的质量，因此必须使用正确的数学递归来模拟或捕获该过程。 

## 方法

 直接的方法是固定初始燃油量并逐步模拟整个行程。 在每个行星上，我们通过将当前总质量除以相应的系数来计算起飞或着陆需要多少燃料。 我们从剩余量中减去该燃料并继续。 如果在任何时候所需的燃料超过我们所拥有的，则猜测无效。 

该模拟对于固定燃料值是正确的，但它并没有立即告诉我们如何选择最小有效燃料。 尝试所有可能性是不可能的，因为燃料是一个实数，具有潜在的大范围和所需的精度。 

关键的观察是单调性。 如果一定量的初始燃料就足够了，那么任何更大的量也足够了。 这是成立的，因为更多的燃料只会增加初始质量，虽然它会稍微增加所需的燃烧，但它永远不会以打破单调行为的方式提高可行性。 该系统对于初始燃料是单调的。 

这种单调性使我们能够对答案进行二分搜索。 我们选择一个候选燃油量，模拟整个旅程，并检查它是否成功。 如果有效，我们会尝试较小的值。 如果失败，我们会增加燃料。 

可行性检查本身必须认真实施。 在每一步中，所需的燃料都是使用从一个单位燃料支持 a_i 或 b_i 吨总质量的条件导出的公式计算的。 这给出了线性关系，允许直接计算每个阶段所需的燃料。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 仅模拟 | O(C·n) | O(C·n) | O(1) | O(1) | 太慢/不切实际|
 | 二分查找+模拟| O(n log C) | O(n log C) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将答案视为连续值并对其进行搜索。 

1. 定义一个函数`check(x)`决定是否从`x`燃油充足。 我们在跟踪当前质量的同时模拟整个路线，该质量最初是有效载荷加上燃料。 该函数直接对物理约束进行编码。 
2. 对于每次起飞或着陆步骤，除以适当的系数来计算支持当前质量所需的燃油。 这是有效的，因为一单位燃料支持固定的质量，因此燃料需求与总重量成线性比例。 
3. 从当前燃料池中减去所需的燃料，并相应更新总质量。 这反映出燃烧燃料既减少了燃料又减少了总重量。 
4. 如果在任何时候所需燃料超过可用燃料或系统变得不一致（负燃料），请立即返回 false。 这种修剪是必不可少的，因为一旦一个步骤失败，后面的步骤就无法恢复可行性。 
5. 在足够大的范围内使用二分搜索，通常从 0 到安全上限（例如 1e9），正如问题所保证的那样。 
6. 返回最小值`check(x)`是真的。 

正确性取决于初始燃料量的可行性是单调的。 

### 为什么它有效

 该过程定义了一系列状态，其中每个状态仅取决于当前总质量。 增加初始燃料会均匀地增加起始状态。 由于每个转变在质量上都是线性的并且按比例消耗燃料，因此较大的起始值不能使先前可行的步骤变得不可行。 这确保了可行的初始燃料集形成一个区间，这证明了二分搜索的合理性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(m, a, b, x):
    fuel = x
    mass = m + x

    for i in range(len(a)):
        # take-off
        need = mass / a[i]
        if need > fuel:
            return False
        fuel -= need
        mass -= need

        # landing
        need = mass / b[i]
        if need > fuel:
            return False
        fuel -= need
        mass -= need

    return True

def solve():
    n = int(input())
    m = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    lo, hi = 0.0, 1e9

    for _ in range(80):
        mid = (lo + hi) / 2
        if can(m, a, b, mid):
            hi = mid
        else:
            lo = mid

    print(f"{hi:.10f}")

if __name__ == "__main__":
    solve()
```模拟功能准确地反映了物理过程。 我们维护两个量：剩余燃料和总质量。 在每个阶段，我们将所需燃料计算为当前质量的正比，因为该系数描述了一个单位燃料可以处理多少质量。 减法步骤反映了燃料消耗和质量减少。 

二分搜索以固定的迭代次数运行，因为答案保证位于有界范围内，并且精度要求最高可达 1e-6。

 ## 边缘情况

 一个关键的边缘情况是当一颗行星的系数非常小时，即使是中等质量也无法处理。 在这种情况下，可行性检查在该步骤立即失败，因为所需燃料超过可用燃料。 该算法通过二分搜索正确地将这种失败传播回来，返回一个不可能的大要求或有效地表明不可行性。 

另一个微妙的情况是燃料需求非常接近容量边界。 由于该解决方案使用浮点运算，理论上精度问题可能会累积，但单调二分搜索与足够的迭代相结合可确保在所需的 1e-6 容差范围内的稳定性。 

最后一个极端情况是有效负载最小但系数很大。 模拟仍然有效，因为每次燃烧后质量都会减少，确保燃料需求只会随着旅程的进展而减少，这与二分搜索假设的单调结构一致。
