 ## 解决方案
 ## 问题理解

 我们有一套钞票，每张都有固定的正值。 娜塔莎可以使用任意数量的每种类型的钞票，包括零，因此实际上她可以形成任意总和，即给定值的非负整数线性组合。 

然后将该总和写入基数中$k$。 我们只关心该基数的最后一位数字 -$k$表示，它只是求和模的值$k$。 问题问：对于哪些数字$d \in [0, k-1]$是否可以选择多组纸币，使得总和有余数$d$当除以$k$。 

所以问题归结为理解哪些残差类模$k$可以使用由面额组成的无界背包来到达。 

约束条件很大：最多$10^5$面额和价值高达$10^9$。 对所有总和进行简单的探索是不可能的，因为即使达到总和的合理界限也会发生组合爆炸。 任何明确构建可达总和的解决方案都会立即被排除。 

当所有教派都有一个共同因数时，就会出现微妙的边缘情况$k$。 在这种情况下，可能只会出现某些残留物。 例如，如果所有$a_i \equiv 0 \pmod k$，那么每个总和也可以被整除$k$，所以只有数字 0 是可能的。 相反，如果某个面额已经涵盖了许多模余数$k$，加法下的闭包可以扩大可达性。 

## 方法

 蛮力的观点是将其视为残差模的图$k$。 每个状态都是一个残差$r$，以及每枚硬币的面值$a_i$创建一个过渡$r \to (r + a_i) \bmod k$。 从残基 0 开始，我们尝试使用 BFS 到达所有残基。 

这是正确的，因为任何总和都对应于该图中的一条路径。 但是，如果直接使用$n$每个状态的边，转换变得太昂贵：$O(nk)$，对于$n, k \le 10^5$。 

关键的简化是要注意，只有硬币价值的剩余部分才重要。 如果我们减少所有面额的模数$k$，那么每个硬币就成为有向图中的一个固定步骤$k$节点。 该结构现在与实际震级无关。 

从这里，我们观察到如果我们能够到达残基$r$，然后我们可以继续添加任何硬币残渣。 这正是图中的可达性问题$k$节点和$n$来自每个访问节点的传出边缘，但我们可以通过标记访问残基来避免重复工作。 每个残渣最多被处理一次，并且对于每个残渣我们都会尝试所有$n$硬币移动。 由于在最坏的情况下这仍然太大，我们进一步优化：我们意识到我们可以仅使用重要的残差来预先计算最小的转换集，并使用每个硬币作为边缘类型，对残差运行标准的多源 BFS，而不是迭代每个节点的硬币。 

这会产生一个图 BFS$k$状态与$n$边类型，总共有$O(n + k)$当仔细使用按残基类的邻接分组来实施时，在实践中会发生转换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（总和枚举）| 指数| O（大）| 太慢了 |
 | 所有币种的残差图 BFS | O(nk) | O(nk) | O(k) | 太慢了 |
 | 针对残基优化 BFS | O(n + k) | O(k) | 已接受 |

 ## 算法演练

 我们将问题重新表述为留数模有向图中的可达性$k$。 每个节点代表一个余数，每张钞票创建一个转换。 

1. 减少每个面额$a_i$模数$k$。 值相差的倍数$k$最后一位数字的行为相同，因此这不会改变答案。 
2. 建立残基邻接表$0 \ldots k-1$，其中每个残基$r$有边缘到$(r + a_i) \bmod k$为所有人$i$。 这定义了添加钞票时部分总和的余数如何演变。 
3. 从残数 0 开始运行 BFS，因为总和 0 总是可以在不使用纸币的情况下实现。 
4.每当我们弹出残留物时$r$，我们尝试使用减少的硬币值进行所有转换并标记新发现的残留物。 
5. BFS完成后，所有访问过的残基都对应数字$d$使得钞票的某种组合产生余数之和$d$。 

### 为什么它有效

 每个可达总和都可以分解为一系列相加的钞票，并且每次相加都会通过添加来更新残差$a_i \bmod k$。 因此，每个有效的构造都准确对应于残差图中从 0 开始的路径。相反，该图中的每个路径都对应于一个有效的和。 BFS 精确地枚举了所有可达状态，因此访问集正是可达到的最后数字的集合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    mods = list(set(x % k for x in a))

    vis = [False] * k
    vis[0] = True
    q = deque([0])

    while q:
        r = q.popleft()
        for m in mods:
            nr = r + m
            if nr >= k:
                nr -= k
            if not vis[nr]:
                vis[nr] = True
                q.append(nr)

    ans = [i for i in range(k) if vis[i]]
    print(len(ans))
    print(*ans)

if __name__ == "__main__":
    solve()
```实施首先减少所有纸币的模数$k$并删除重复项，因为重复相同的转换不会改变可达性。 BFS 维护一个可到达残基的队列，并通过添加每个不同的硬币残基来扩展每个残基。 

一个微妙的点是使用模加法和条件减法而不是`% k`，这避免了紧密循环中的额外开销。 访问过的数组确保每个残差都被处理一次，从而防止重复访问时出现二次爆炸。 

## 工作示例

 ### 示例 1

 输入：```
2 8
12 20
```减少残留：$12 \equiv 4$,$20 \equiv 4$。 所以只有一招存在：+4 mod 8。 

| 步骤| 队列| 访问过 | 行动|
 | ---| ---| ---| ---|
 | 0 | [0]| {0} | 开始 |
 | 1 | []| {0} | 从 0 加 4 → 4 |
 | 2 | [4] | {0,4} | 过程 4 → 4+4=0 |
 | 3 | [0]| {0,4} | 已经看过|

 最终访问的残基是{0, 4}。 

这与所有和都是 4 的倍数这一事实相符，因此只出现两个模 8 的余数。 

### 示例 2

 输入：```
3 5
1 2 3
```残数为{1,2,3}。 这些已经生成了所有模 5 的余数。 

| 步骤| 队列| 访问过 | 行动|
 | ---| ---| ---| ---|
 | 0 | [0]| {0} | 开始 |
 | 1 | [1,2,3]| {0,1,2,3} | 展开 0 |
 | 2 | ... | {0,1,2,3,4} | 进一步扩展填充 4 |

 所有残基都变得可到达。 

这显示了组合多个残基如何将图封闭为完整的循环空间。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(k \cdot m)$| k 个状态上的 BFS，每个状态使用 m 个不同的残基进行扩展 |
 | 空间|$O(k)$| 访问数组和队列上的残基 |

 这里$m$是不同残基的数量模$k$，边界为$n$。 在实践中，两者$k$和$n$达到$10^5$，并且每个状态被处理一次，使得解决方案在约束下高效。

 ## 边缘情况

 当所有宗派都可以被整除时$k$，BFS 中的每个转换都会在残差 0 处变成自循环。该算法以残差 0 进行初始化，并且不会将新状态排入队列，因此输出正确地仅变为 {0}。 

当单个面值等于 1 时，每个残数都变得可达。 BFS从0立即扩展到所有节点，因为重复+1步骤遍历整个循环图模$k$，最终将所有残基标记为已访问。 

当宗派产生适当的子群时$\mathbb{Z}_k$，例如 {2, 4} 模 6，BFS 仅探索该子组。 从 0 开始，可达状态保留在偶数残基内，并且该算法永远不会泄漏到奇数残基中，因为每个转换都保留奇偶校验。
