 ## 解决方案
 ## 问题理解

 我们给出的结构是一个有根树，其中每个节点的行为都像一个逻辑组件。 叶子是固定的布尔输入，而内部节点使用标准门（例如 AND、OR、XOR 和 NOT）从其子节点计算布尔值。 根节点产生整个电路的最终输出。 

不同的是，每个输入叶子都有一个已知的初始值，但我们被告知这些输入中的一个是错误的，因为它的值可能会被翻转。 任务不是找出哪一个坏了。 相反，对于每个独立的输入叶子，我们必须确定如果该特定叶子被翻转而所有其他叶子保持不变，则根的输出会变成什么。 

因此，对于每个输入叶子，我们概念上在该叶子上应用一位反转并重新计算根输出。 天真的这样做意味着对每个叶子重新评估一棵大树一次，当节点数量达到一百万时，这太慢了。 

该约束立即意味着任何根据输入从头开始重新计算值的解决方案都是不可行的。 完整的遍历会花费线性时间，因此在最坏的情况下对所有叶子重复它会导致二次行为。 相反，我们需要一种在所有查询中重用计算并通过树传播“翻转效果”信息的方法。 

当本地思考时，会出现一个微妙的陷阱：翻转输入不仅会影响其父级，它可能会也可能不会影响根，具体取决于信号在计算路径中是否实际相关。 例如，在与门中，如果另一个输入为 0，则翻转一个子项永远不会改变输出。 任何正确的解决方案都必须捕获这种敏感性概念，而不是原始值传播。 

## 方法

 暴力方法很简单。 对于每个输入叶子，翻转其值，重新计算后序遍历中的所有节点值，并记录根。 这是正确的，因为每次评估都会完全重新计算确定性电路。 然而，每次重新计算的成本为 O(n)，并且对最多 O(n) 个叶子执行此操作会导致 O(n²)，这远远超出了 n 为 10⁶ 时的限制。 

关键的观察是电路是一棵树，每个节点的输出仅取决于其子节点。 更重要的是，我们可以将计算分成两个独立的信息：每个节点当前产生的值，以及该节点对其每个子节点的变化的敏感程度。 

这引出了函数图上树DP的一个标准思想：我们不是从头开始重新计算输出，而是计算每个节点的输出一次，然后使用第二次遍历来计算每个子树对根的贡献，回答“如果这个节点翻转，根会翻转吗？”的问题。 

我们为每个节点定义它在原始电路中产生的值。 然后我们定义第二个量：切换节点值对根输出的影响。 这本质上是通过树的反向传播。 根的敏感度固定为 1，然后对于每个父子关系，我们根据门类型和兄弟值确定影响如何流动。 

关键的简化是每个门都可以在本地进行分析。 例如，在 AND 门中，仅当另一个子级为 1 时，输出才依赖于另一个子级。如果另一个子级为 0，则输出已固定且不敏感。 类似的推理适用于 OR 和 XOR，而 NOT 则简单地通过反转传递敏感性。 

这将问题转化为对树的两次线性遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

我们以节点 1 为树根，并将边视为父子关系。 输入（叶子）是初始值的来源。 

1. 使用从叶子到根的后序遍历计算每个节点的值。 对于每个内部节点，我们将其逻辑运算应用于其子节点的值。 这给出了系统的最终输出以及敏感性推理所需的所有中间值。 
2. 对于每个内部节点，在解析期间显式记录其子节点，以便我们稍后可以向后传播影响。 这种结构至关重要，因为敏感性取决于同级值。 
3.定义布尔数组`dp[v]`含义 是否翻转节点`v`翻转根输出。 
4. 初始化`dp[1] = 1`因为翻转根显然会翻转自身。 
5. 使用 DFS 或迭代堆栈从根向下遍历树。 在每个节点`v`，我们确定如何`dp[v]`分配给它的孩子。 
6. 对于每个孩子`u`节点的`v`，计算是否改变`u`影响`v`。 这取决于门类型和所有其他子项的值`v`。 

对于 AND，只有当所有其他子项都为 1 时，子项才会影响父项。对于 OR，只有当所有其他子项都为 0 时，子项才会影响父项。对于 XOR，每个子项都会独立影响父项，而不管其他子项如何。 对于NOT，独生子女总是影响父母。 
7.如果一个孩子`u`影响其父级`v`， 然后`dp[u]`变成`dp[v]`，因为改变`u`传播到`v`，如果达到`v`，它继续向上传播，并具有与中已捕获的相同的灵敏度`dp[v]`。 
8. 传播整棵树后，输出`dp[v]`对于每个输入节点。 

### 为什么它有效

 正确性取决于根输出是叶上的布尔函数，并且每个内部节点组成局部布尔函数。 根对任意节点的依赖性完全取决于是否存在从该节点到根的“主动影响”路径。 dp 值精确跟踪这样的路径是否存在，并且本地门条件确保仅当门尚未被其他输入固定时才传递影响。 这可以防止过度计数，并保证当且仅当翻转它改变最终输出时，每个叶子都被标记为对根敏感。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n = int(input())

g = [[] for _ in range(n + 1)]
typ = [""] * (n + 1)
val = [0] * (n + 1)

children = [[] for _ in range(n + 1)]
parent = [0] * (n + 1)

for i in range(1, n + 1):
    tmp = input().split()
    typ[i] = tmp[0]
    if typ[i] == "IN":
        val[i] = int(tmp[1])
    else:
        for x in tmp[1:]:
            x = int(x)
            children[i].append(x)
            parent[x] = i
            g[i].append(x)
            g[x].append(i)

# compute values bottom-up using postorder
order = []
stack = [1]
visited = [False] * (n + 1)
visited[1] = True

while stack:
    v = stack.pop()
    order.append(v)
    for u in g[v]:
        if not visited[u]:
            visited[u] = True
            stack.append(u)

order.reverse()

for v in order:
    if typ[v] == "IN":
        continue
    if typ[v] == "NOT":
        val[v] = 1 - val[children[v][0]]
    elif typ[v] == "AND":
        val[v] = val[children[v][0]] & val[children[v][1]]
    elif typ[v] == "OR":
        val[v] = val[children[v][0]] | val[children[v][1]]
    elif typ[v] == "XOR":
        val[v] = val[children[v][0]] ^ val[children[v][1]]

dp = [0] * (n + 1)
dp[1] = 1

def dfs(v):
    for u in children[v]:
        if typ[v] == "NOT":
            dp[u] = dp[v]
            dfs(u)
        elif typ[v] == "XOR":
            dp[u] = dp[v]
            dfs(u)
        elif typ[v] == "OR":
            other = children[v][0] ^ children[v][1] ^ u
            if val[other] == 0:
                dp[u] = dp[v]
                dfs(u)
        elif typ[v] == "AND":
            other = children[v][0] ^ children[v][1] ^ u
            if val[other] == 1:
                dp[u] = dp[v]
                dfs(u)

dfs(1)

res = []
for i in range(1, n + 1):
    if typ[i] == "IN":
        res.append(str(dp[i]))
print("".join(res))
```第一阶段使用反向拓扑顺序计算实际门输出。 第二阶段从根部向下传播敏感性。 关键细节是使用 XOR 技巧有效地计算“另一个子节点”，这避免了每个节点的额外循环。 条件检查实现特定于门的条件，在该条件下子项与其父项相关。 

一个微妙的实现点是，仅当子级实际上可以影响给定当前同级值的父级时，敏感度才会传播。 这就是 OR 要求另一个输入为 0 而 AND 要求另一个输入为 1 的全部原因。

 ## 边缘情况

 当一个节点是 AND 门并且一个子节点已经为 0 时，就会出现关键的边缘情况。在这种情况下，翻转另一个子节点不会影响输出，因此敏感度必须止于此。 该算法通过在传播 dp 之前检查同级值来处理此问题。 

另一种情况是 OR 门，其中一个子级为 1。另一个子级的任何更改都是无关紧要的，并且 dp 传播被条件正确阻止`val[other] == 0`。 

最后一种情况是 XOR 和 NOT 门链，其中每次翻转总是通过路径传播。 由于 XOR 和 NOT 不具有固定输出阻塞行为，因此 dp 可以自由流动，即使在深度线性结构中也能确保正确性。
