 ## 解决方案
 ## 问题理解

 我们正在跟踪一个参与者，Eric，他从一个固定位置开始$n$人们按等级排列。 每轮比赛都会将人们配对成不相交的比赛。 在每场比赛中，两名玩家中的一名赢得了掷硬币的胜利，但只有原本排名比对手差的获胜者才能通过交换位置来真正提高自己的排名。 如果排名较高的玩家获胜，则不会发生任何变化。 

只有当埃里克遇到排名比他高的人并赢得这场比赛时，他的位置才会发生变化，在这种情况下，他会与该对手交换位置。 否则他的位置在该轮中保持不变。 重复此过程后$k$回合中，我们被要求提供 Eric 的排名模的期望值$998244353$。 

限制条件非常大：$n$和$k$可以达到$10^{18}$。 这立即排除了任何轮次模拟或任何按时间索引的状态的 DP。 如果天真地将其视为一般的马尔可夫链，那么即使存储按位置索引的状态也是不可行的。 任何可行的解决方案都必须将过程简化为封闭形式的递归或非常低维的变换。 

出现微妙的边缘情况时$n=2$。 在这种情况下，埃里克要么留在原地，要么与唯一的其他玩家交换，具体取决于每轮抛硬币的次数。 任何假设除以的解$n-1$必须小心处理这种情况，以避免在推广公式时出现模逆问题。 

另一个重要的结构观察是，Eric 的排名随着时间的推移是单调非增加的。 一旦他移动到更好的排名（更少的数字），他就永远不会被模型中的任何操作推低。 这消除了双向运动马尔可夫链的许多典型并发症。 

## 方法

 直接模拟将独立处理每一轮。 在一轮中，埃里克与一些对手匹配，在其他对手中统一匹配$n-1$人们。 如果该对手的排名高于埃里克，那么有概率$1/2$Eric向上交换； 否则什么都不会改变。 This gives a correct stochastic process description.

 然而，即使是模拟一步$n$状态是不可行的，并且迭代此$k$步骤是完全不可能的。 

The key simplification is to abandon distribution tracking and instead focus on the expectation directly. 尽管这个过程是随机的，但埃里克排名的预期变化仅取决于他当前的排名，而不取决于完整的分发历史。 

假设埃里克目前的排名$i$。 对手是从对方中统一选择的$n-1$玩家。 如果对手的排名大于$i$，无论硬币如何，都不会发生任何事情。 如果对手的排名小于$i$，那么有概率$1/2$，埃里克与该对手交换并跳转到该排名。 

这使我们能够纯粹地计算预期的下一个位置$i$，产生闭合递归：$$E[i_{t+1}] = i - \frac{i(i-1)}{4(n-1)}.$$因此，整个问题归结为迭代单个值的二次变换$i$。 现在的难度从概率转移到极大值下的函数迭代$k$。 

暴力方法将应用这种递归$k$次数、成本$O(k)$，这是不可能的，当$k \le 10^{18}$。 

为了加速，我们观察到这是有限域中单个标量的确定性递归。 此类系统通常通过将更新转换为支持组合的形式或通过找到线性化递归的不变转换来处理。 在这种情况下，递推是一个有理二次映射，它允许通过变换的二维提升表示上的函数组合来快速求幂。 

这减少了问题$k$顺序评估$O(\log k)$重复构图步骤。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 直接模拟回合 |$O(k)$|$O(1)$| 太慢了 |
 | 通过组合进行函数迭代 |$O(\log k)$|$O(1)$| 已接受 |

 ## 算法演练

 1.从Eric的初始等级开始$x$。 这是我们跟踪的唯一状态，因为重现仅取决于当前的期望值而不是完整的分布。 
2. 从当前排名导出预期的下一个排名$i$。 对手是统一选择的，所以每个人$n-1$可能的对手有概率$1/(n-1)$。 
3.根据对手等级分案。 如果对手排名比$i$，状态不变。 如果对手排名更好，那么有概率$1/2$，艾瑞克交换位置，直接移动到对手的位置。 
4. 计算所有排名较高的对手的预期贡献。 选择均匀随机的排名较高的对手时的平均改进与距对手的平均距离成正比。$i$，其评估结果为$i/2$。 结合概率得出预期减少$$\frac{i(i-1)}{4(n-1)}.$$5. 这给出了确定性递归：$$f(i) = i - \frac{i(i-1)}{4(n-1)}.$$6.我们应用这种转换$k$次使用函数组合的快速求幂。 每个合成步骤都会使应用的轮数加倍。 
7. 返回取模后的结果值$998244353$。 

### 为什么它有效

 该过程在对手之间是可交换的，并且在期望上是线性的，因此预期的下一个状态仅取决于当前的预期排名。 这将随机系统崩溃为单变量确定性递归。 函数组合保留了正确性，因为每个步骤准确地代表了一个完整回合的预期转换，并且组合转换完全对应于顺序运行多个回合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def compose(a1, b1, a2, b2):
    # (a2 x^2 + b2 x) plugged into (a1 x^2 + b1 x)
    # returns coefficients of resulting quadratic in x
    A = a1
    B = b1
    C = a2
    D = b2

    # f(g(x)) = A*(Cx^2 + Dx)^2 + B*(Cx^2 + Dx)
    # expand carefully
    return (
        A * C * C % MOD,
        (2 * A * C * D + B * C) % MOD,
        (A * D * D + B * D) % MOD
    )

def solve():
    n, x, k = map(int, input().split())
    n %= MOD
    x %= MOD

    if n == 1:
        print(x)
        return

    inv = modinv(4 * (n - 1) % MOD)

    # f(i) = i - i(i-1)/(4(n-1))
    #     = (-inv) * i^2 + (1 + inv) * i
    a = (-inv) % MOD
    b = (1 + inv) % MOD

    # identity function: x
    ra, rb = 0, 1

    e = k
    while e:
        if e & 1:
            ra, rb = compose(ra, rb, a, b)
        a, b = compose(a, b, a, b)
        e >>= 1

    # apply resulting function to x
    ans = (ra * x % MOD * x + rb * x) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先将预期的转换转换为二次多项式$i$。 然后，它对函数组合执行二元提升，将每个函数视为可以平方或相乘的变换$O(1)$。 最后一步评估初始等级的组合函数。 

关键的实现细节是将递归表示为二次多项式，并在模运算下仔细组合它。 二进制求幂结构确保我们只执行$O(\log k)$组成。

 ## 边缘情况

 当$n=1$，没有任何比赛发生，埃里克的排名保持不变。 否则递归会尝试除以$n-1=0$，所以这种情况必须直接处理。 

什么时候$n=2$，系统退化为两种状态交互，其中每一轮都是一场比赛。 在模逆下，递推仍然以代数方式起作用，但重要的是实现正确计算$4(n-1)$在本例中为 4。 

什么时候$k=0$，不应用组合，输出只是初始排名$x$。 这对应于组合框架中的恒等函数，在二进制求幂期间必须将其保留为中性元素。
