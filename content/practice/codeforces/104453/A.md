 ## 解决方案
 ## 问题理解

 给定两个复数，每个复数由整数实部和整数虚部描述。 第一个数字由一对组成$a, b$作为$a + bi$，第二个是$c + di$。 任务是计算它们的乘积并输出结果的实部和虚部。 

乘法是标准的复杂算术。 直接展开，$$(a + bi)(c + di) = ac + adi + bci + bdi^2$$自从$i^2 = -1$，这就变成了$$(ac - bd) + i(ad + bc).$$所以输出始终是两个整数：实部$ac - bd$和虚部$ad + bc$。 

约束非常小，所有值都在$-1000$和$1000$。 这意味着任何正确的算术实现就足够了。 即使是简单的公式评估也会在恒定时间内运行，因此不涉及算法优化。 

此类问题中的一个常见错误是组合涉及的术语时的符号处理$i^2$。 另一个常见问题是混淆两个交叉项$ad$和$bc$，特别是当试图在精神上扩展或在没有固定公式的情况下实施时。 

## 方法

 暴力解释实际上会对待$i$作为一个符号对象并逐步展开乘法，分别跟踪实部和虚部。 这仍然是每个操作的持续工作量，但这是不必要的开销。 复数的结构已经编码了封闭形式的乘法规则。 

关键的观察是，复数乘法总是分解为四个标量乘法和两个加法/减法。 多个输入或任何迭代过程之间不存在依赖性。 一旦写下代数恒等式，计算就立即进行。 

这将问题简化为对固定公式的直接评估。

 ## 算法演练

 1.读取四个整数$a, b, c, d$。 这些定义了两个复数$a + bi$和$c + di$。 顺序很重要，因为交叉项取决于正确配对实部和虚部。 
2. 使用恒等式计算实部$ac - bd$。 这是通过将实部的乘积与虚部的乘积相结合得出的，因为$i^2 = -1$。 
3. 使用以下方法计算虚部$ad + bc$。 这些是实部和虚部相互作用的交叉项。 
4. 按顺序输出两个值：实部在前，虚部在后。 

分成两个表达式确保在实现过程中不需要中间符号推理。 

### 为什么它有效

 正确性直接来自乘法对加法的分布性和定义属性$i^2 = -1$。 展开式中的每个乘积项都恰好属于四个类别之一：实-实、实-虚、虚-实和虚-虚。 最后一类引入了符号变化，这会产生实部的减法。 由于所有可能的配对都被计算一次，因此得到的表达式是完整且准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, c, d = map(int, input().split())
    real = a * c - b * d
    imag = a * d + b * c
    print(real, imag)

if __name__ == "__main__":
    solve()
```该解决方案读取四个整数，直接计算两个所需的表达式，然后打印它们。 由于界限很小，乘法和加法运算在 32 位整数范围内都是安全的。 

一个微妙的实现细节是维护正确的分组：$a*c - b*d$必须完全按照编写的方式进行计算，以避免优先级错误，即使 Python 在没有括号的情况下也能正确计算它。 这同样适用于虚部，它必须包括两个交叉项。

 ## 边缘情况

 一种关键的边缘情况是当一个分量为零时，例如将纯实数乘以纯虚数。 该公式仍然表现正确，因为交叉项仍然存在，但实-实和虚-虚贡献可能会消失。 

当组件之间的符号不同时，就会出现另一种边缘情况，这很容易导致心算不正确。 代数形式确保了正确性，因为实部的减法是由$-bd$术语，防止实施过程中意外的符号翻转。 

所有情况都干净地简化为相同的公式，因此不需要特殊的分支。
