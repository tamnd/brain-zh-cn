 ## 解决方案
 ## 问题理解

 我们获得了固定数量的资助博士职位，以及一份申请人名单及其成绩分数。 伊戈尔有他自己的分数，我们必须确定他相对于其他人的结果。 

录取规则是根据分数排名。 如果伊戈尔的分数严格优于足够的候选人以适合空缺职位，他将被直接录取。 如果他恰好处于截止分数上，这意味着在边界上有包括他在内的平局，那么他仍然会被录取，但必须参加入学考试，因为多个申请人共享该临界分数。 如果严格意义上超过他的人太多，他就没有机会了。 

因此，任务简化为将伊戈尔的分数与所有申请人分数的分布进行比较，包括他在其中的隐含位置。 

约束足够大，排序是自然的选择。 总分数高达 200,000，O(n log n) 解决方案很容易就足够了，而任何超过 10^5 的二次方都太慢了。 

这里的一个天真的错误是只推理有多少人的分数严格高于或相等，而没有仔细处理伊戈尔恰好落在截止分数上的边界条件。 

一个具体的边缘情况是许多人分享伊戈尔的分数。 

输入：```
n = 2, k = 50
other = [50, 50, 10]
```如果我们只计算“分数 ≥ 50 的人”，我们会得到 3 个人竞争 2 个名额，因此 Igor 可能会被错误地拒绝。 但实际上，他是在里面还是在截止点并列决定了“入学考试”。 

另一种边缘情况是没有人得分更高：

 输入：```
n = 3, k = 100
other = [10, 20, 30]
```伊戈尔显然应该不经考试就被录取。 

这些案例表明我们必须使用完整的排序进行推理，而不仅仅是计数。 

## 方法

 暴力解释是模拟完整排名。 我们将 Igor 插入到分数列表中，对所有内容进行排序，并确定他在排序顺序中的位置。 然后我们检查他的位置是否在前n名之内。 如果是，我们还需要判断他的分数是否多次出现在截止边界处。 

这是有效的，因为排序显式地构建了排名。 然而，以幼稚的方式重复重新计算或扫描列表是低效的； 但即使是简单的排序也已经足够优化了。 

更谨慎的观点是，只有伊戈尔相对于其他人的地位才重要。 排序后，我们只需要找出有多少严格超过他的分数，有多少等于它。 这决定了他是否进入前n名，以及是否会影响入学考试。 

关键的见解是我们不需要动态模拟招生。 单个排序结构一次性给出所有必要的比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（排序完整列表，检查位置）| O(m log m) | O(米) | 已接受 |
 | 最优（相同排序，直接排名计算） | O(m log m) | O(米) | 已接受 |

 在这个问题中，“强力”和“最优”解决方案崩溃为相同的基于排序的方法，因为无论如何约束已经需要排序。 

## 算法演练

 1. 读取n、k、m以及其他申请人的分数列表。 

我们将伊戈尔视为同一排名系统中的附加元素。 
2. 将 Igor 的分数插入所有分数列表中。 

这使我们能够推理单个有序序列而不是单独的情况。 
3. 将所有分数按非升序排序。 

排序为我们提供了准确的录取顺序，因为较高的分数总是压倒较低的分数。 
4. 找到 Igor 在这个已排序数组中的位置。 

这个职位告诉我们有多少申请者严格领先于他。 
5. 如果严格优于 Igor 的申请者少于 n 个，则 Igor 在承认集中。 

此时，我们仍然需要检查他是否在截止边界处绑在一起。 
6. 如果 Igor 的分数出现在多个相同分数的条目跨越索引 n 的边界的位置，则输出“入学考试”。 

当超过一名申请人共享截止分数时，就会发生这种情况。 
7. 如果少于 n 个申请人的分数严格高于 Igor，并且 Igor 严格高于任何平局边界，则输出“enter”。 
8. 否则，输出“no opportunity”。 

当太多申请人的表现严格优于伊戈尔时，就会发生这种情况。 

### 为什么它有效

 排序后，排名完全由索引顺序决定。 关于录取的任何决定仅取决于大于或等于伊戈尔分数的值的计数。 当多个相等的分数跨越截止位置 n 时，唯一的歧义就会出现，这正是触发入学考试的条件。 由于排序保留了所有相等结构，因此一次传递就足以解决排序和平局行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
m = int(input())
arr = list(map(int, input().split()))

arr.append(k)
arr.sort(reverse=True)

# position of Igor in sorted list
# (find first occurrence of k among equal elements)
pos = arr.index(k)

# number of people strictly better than Igor
better = pos

if better >= n:
    print("no chance")
else:
    # check if cutoff is a tie situation
    cutoff_score = arr[n - 1]
    # if Igor is at or beyond cutoff region with duplicates of cutoff
    if k == cutoff_score and arr.count(k) > 1:
        print("entrance exams")
    else:
        print("enter")
```该实现首先通过附加 Igor 的分数并按降序排序来构建完整排名。 这`pos`变量衡量有多少人严格领先于伊戈尔。 如果该计数已经超过或等于可用槽数，则 Igor 出局。 

第二阶段处理伊戈尔位于顶部的微妙情况`n`，但边界分数并不唯一。 我们计算索引处的截止值`n-1`并检查伊戈尔是否与其他人分享该分数。 如果是这样，这意味着一个跨越边界的平局区域，这将迫使入学考试。 

主要的微妙之处在于`.index()`和`.count()`都是线性运算； 对于严格的约束，人们通常会避免使用它们，但考虑到总体限制，它们仍然是安全的。 更生产级的版本将使用字典预先计算频率。

 ## 边缘情况

 一个重要的边缘情况是，包括伊戈尔在内的所有申请人的分数相同。 在这种情况下，排序会产生一个统一的数组，其中每个位置都相等。 截止索引落在具有相同值的块内，因此多个候选同时占据边界。 该算法正确地触发了“入学考试”，因为截止分数出现了不止一次。 

另一个边缘情况是伊戈尔严格来说是最高得分手。 排序后，他出现在索引0处，所以`better = 0`。 自从`0 < n`，他立即被录取，并且截止检查不会改变结果。 

当更强的候选者的数量恰好等于 n 时，就会出现最终的边缘情况。 在这里，伊戈尔被推到了允许的范围之外，即使他的分数与接近截止点的人持平，严格的计数条件也已经排除了他，所以答案仍然是“没有机会”。
