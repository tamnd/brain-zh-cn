 ## 问题理解

 我们得到两个二进制字符串，每个字符串的长度恰好为 N。它们可能包含前导零，因此它们的“书面形式”不一定是它们规范的二进制表示形式。 我们的任务是逐位计算它们的按位异或，然后在删除任何前导零后输出结果二进制字符串。 

从概念上讲，我们对齐两个等长的位数组并生成第三个数组，其中如果输入位不同，则每个位置为 1；如果输入位相同，则每个位置为 0。 最后一步是纯粹的格式化：我们不能打印不必要的前导零，这意味着答案应该以最短的形式表示相同的二进制值，除了结果为零的特殊情况必须输出单个`0`。 

输入大小N可以大到100,000。 这立即排除了任何重复将字符串转换为整数并使用高级构造中的朴素大整数运算执行算术的方法，因为在昂贵的循环内重复转换或逐位字符串连接会带来二次行为的风险。 对字符串的线性扫描是唯一安全的目标复杂性。 

当 XOR 结果全为零时，会出现微妙的边缘情况。 例如，如果两个输入相同，例如`0001`异或`0001`，原始结果是`0000`，但输出必须是`0`，不是空字符串。 不检查空性而删除前导零的粗心实现将错误地不打印任何内容。 

另一个边缘情况来自输入本身的前导零。 例如，`0001`和`0001`仍应产生`0`， 不是`0000`或空行。 这强调了归一化只能在计算异或之后发生，而不是之前。 

## 方法

 强力解释会将两个二进制字符串都视为数字：将它们转换为整数，应用 XOR，然后转换回二进制。 这在逻辑上是正确的，并且代码非常紧凑。 然而，大 N 的转换步骤涉及解析每个字符串多达 100,000 个字符，并且重复转换或低效的大整数操作可能会降低性能，具体取决于实现语言和运行时限制。 在 Python 中，它仍然是线性的，但在严格的竞争性编程设置中，预期的解决方案避免了不必要的解析开销和额外的分配。 

更直接可靠的方法是逐个字符地处理字符串。 由于位上的异或在各个位置上是独立的，因此每个输出位仅取决于相应的输入位。 我们可以从左到右扫描，动态计算异或，然后从结果中删除前导零。 关键的观察是我们根本不需要存储中间数字表示； 字符串已经编码了我们需要的一切。 

这将问题简化为单个线性通道，然后是简单的修剪步骤。 我们还避免构建大型临时整数或重复的字符串连接。

 ## 算法演练

 1. 读取整数N和两个二进制字符串A、B。 

除了输入验证之外，N 的值并不是严格必需的，因为两个字符串已经携带显式长度信息。 
2. 初始化一个空列表来存储结果字符。 

使用列表而不是重复的字符串连接非常重要，因为 Python 字符串是不可变的，并且重复的连接会将性能降低到 O(N²)。 
3. 迭代从 0 到 N − 1 的索引。 

在每个索引处，比较 A[i] 和 B[i]。 如果不同，请附加`'1'`到结果列表，否则追加`'0'`。 
4. 将列表转换为字符串。 

此时，我们有了完整的异或结果，但它可能包含前导零。 
5. 从字符串中去除前导零。 

这是通过查找第一次出现来完成的`'1'`。 如果不存在这样的字符，则结果完全为零。 
6. 如果剥离结果为空字符串，则输出`'0'`。 

这处理所有位抵消的特殊情况。 
7. 否则打印修剪后的字符串。 

### 为什么它有效

 XOR 在每个位位置上独立定义：输出位仅取决于同一索引处的两个输入位。 这意味着该问题不存在跨位置依赖性，因此一次传递就足够了。 唯一的全局转换是格式化：删除前导零，这不会更改位序列表示的数值。 该算法通过构造保留每比特的正确性，并且仅在最后调整表示。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    a = input().strip()
    b = input().strip()

    res = []

    for i in range(n):
        if a[i] == b[i]:
            res.append('0')
        else:
            res.append('1')

    s = ''.join(res)

    # remove leading zeros
    i = 0
    while i < len(s) and s[i] == '0':
        i += 1

    ans = s[i:]
    if ans == "":
        ans = "0"

    print(ans)

if __name__ == "__main__":
    main()
```该实现保持逻辑严格线性。 该循环构造 XOR 结果，无需任何中间数字转换。 使用列表进行累积可以避免重复的字符串重新分配。 主通道之后，第二次线性扫描将删除前导零。 当异或结果处处为零时，最终条件确保正确性。 

一个常见的错误是忘记去掉前导零会产生空字符串。 另一个微妙的问题是尝试使用`int(a, 2) ^ int(b, 2)`没有考虑到非常大的字符串仍然需要完全解析； 虽然 Python 支持大整数，但与直接按位处理相比，这会带来不必要的开销。

 ## 边缘情况

 关键的边缘情况是所有位都抵消。 用于输入`A = 0000`,`B = 0000`，计算出的 XOR 字符串变为`0000`。 修剪步骤会删除每个字符，产生一个空字符串。 该算法显式检查这一点并返回`0`，确保有效的二进制表示。 

另一种情况是结果有前导零但未完全取消，例如`A = 0010`,`B = 0001`。 异或结果为`0011`。 修剪循环仅删除前导零并保留有意义的后缀，产生`11`。 这证实了修剪逻辑只影响表示，而不影响价值。 

最后，当 N = 1 时，算法退化为单个比较。 该逻辑在没有特殊大小写的情况下仍然有效，这表明该方法在所有输入大小上都是一致的。
