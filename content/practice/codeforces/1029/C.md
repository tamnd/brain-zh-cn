[CF 1029C - 最大交集](https://codeforces.com/problemset/problem/1029/C)

 **评分：** 1600
 **标签：** 贪婪、数学、排序
 **求解时间：** 2m 50s
 **已验证：** 是的

 ## 解决方案
 ## 问题理解

 我们在数轴上得到了一组线段，如果我们恰好删除其中一个线段，我们想了解它们重叠的程度。 对于线段的任何子集，它们的公共交集也是线段（或空线段），其长度由所有选定间隔重叠的紧密程度决定。 

每个段都有一个下限和一个上限。 所有线段的交集完全由两个值控制：它们之间的最大左端点和它们之间的最小右端点。 如果最大左侧大于最小右侧，则不存在重叠并且交集为空。 

任务是选择一个线段进行丢弃，以使剩余线段的交集尽可能长。 

限制允许最多 300,000 个段。 任何尝试删除每个线段并从头开始重新计算交集的解决方案都会太慢，因为在最坏的情况下需要 O(n²) 间隔检查。 这促使我们寻求一种解决方案，在该解决方案中，我们预处理全局结构并在恒定时间内回答每次删除。 

当负责全局最大左或全局最小右的段被删除时，就会出现一个微妙的问题。 在这种情况下，限制边界会转移到第二佳候选者。 任何只跟踪单个全局最小值和最大值而不跟踪多重性的方法都会失败。 

例如，如果许多线段共享相同的最小右端点，则删除其中之一不会更改相交边界。 在这种情况下，错误地重新计算极值或忘记重复项的幼稚实现将产生错误的答案。 

## 方法

 关键的观察结果是，所有线段的交集仅取决于两个数字：最大左端点 L 和最小右端点 R。删除一个线段后，我们需要为剩余的 n−1 线段重新计算这两个值。 

暴力方法将迭代每个线段 i，重新计算所有其他线段中的最大左侧和最小右侧，并计算所得的交叉点长度。 每次删除的成本为 O(n)，总体上为 O(n²)，这对于 n 高达 300,000 来说太慢了。 

改进来自于我们意识到我们不需要每次都从头开始重新计算极值。 我们只需要知道对于左端点和右端点，最佳和次佳候选者，以及最佳值出现的次数。 如果我们预先计算：

 最大和第二大左端点，

 最小和第二小的右端点，

 然后对于每个删除的段，我们可以在 O(1) 中确定删除它是否会影响极值以及如何调整。 

如果删除的段对当前最大剩余量没有贡献，则最大剩余量保持不变。 如果它有贡献并且是唯一的，我们就会回到第二个最大值。 相同的逻辑对称地适用于最小右侧。 

这将问题简化为预处理后的简单的每个索引评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们维护端点的全局统计数据，然后评估删除每个段的效果。

1. 扫描所有段并计算 l 的最大和第二大值，以及最大值出现的次数。 同时，为了清楚起见，如果需要，请跟踪事件的索引。 这是必要的，因为删除不唯一负责最大值的段不应改变边界。 
2. 类似地计算 r 的最小和第二小值，以及最小值出现的次数。 这让我们可以确定当最小右端点消失时会发生什么。 
3. 对于每个线段 i，通过确定新的最大左端点变成什么来模拟其移除。 如果 l[i] 不是全局最大值，则最大值保持不变。 如果它是唯一的最大值，我们切换到第二个最大值。 否则，我们保持相同的最大值。 
4. 对最小右端点进行对称计算。 如果 r[i] 不是全局最小值，则它保持不变。 如果它是唯一最小的，我们切换到第二个最小值。 
5. 计算此移除的交集长度为 max(0, new_min_r − new_max_l)。 跟踪所有 i 的最大值。 

独立评估每个移除背后的原因是，交集完全由两侧的独立极值确定，因此不需要额外的结构。 

### 为什么它有效

 在任何一点，一组线段的交集仅取决于两个线段施加的极端约束：一个将左边界尽可能向右推，另一个将右边界尽可能向左推。 仅当某个段参与这些极端约束之一时，删除该段才会影响结果。 由于只有一个或两个候选者可以定义每个极值，因此跟踪最佳值和次佳值足以在删除后重建正确的交集。 这确保了每个候选删除都在其产生的状态下准确评估。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    seg = [tuple(map(int, input().split())) for _ in range(n)]
    
    INF = 10**18
    
    # track top two maxima for l
    max1_l = -1
    max2_l = -1
    cnt_max1_l = 0
    
    # track top two minima for r
    min1_r = INF
    min2_r = INF
    cnt_min1_r = 0
    
    for l, r in seg:
        if l > max1_l:
            max2_l = max1_l
            max1_l = l
            cnt_max1_l = 1
        elif l == max1_l:
            cnt_max1_l += 1
        elif l > max2_l:
            max2_l = l
        
        if r < min1_r:
            min2_r = min1_r
            min1_r = r
            cnt_min1_r = 1
        elif r == min1_r:
            cnt_min1_r += 1
        elif r < min2_r:
            min2_r = r
    
    ans = 0
    
    for l, r in seg:
        # compute new max L
        if l == max1_l and cnt_max1_l == 1:
            new_l = max2_l
        else:
            new_l = max1_l
        
        # compute new min R
        if r == min1_r and cnt_min1_r == 1:
            new_r = min2_r
        else:
            new_r = min1_r
        
        ans = max(ans, max(0, new_r - new_l))
    
    print(ans)

if __name__ == "__main__":
    solve()
```第一遍在线性时间内计算所有必要的全局极值。 第二遍通过检查删除的段是否唯一定义边界来评估每个删除。 关键的实现细节是维护极值的计数，因为仅相等不足以决定是否需要回退到第二最佳值。 

处理次佳初始化时也需要小心。 使用大标记作为最小值，使用小标记作为最大值，可以确保即使所有段都相同或许多值重合时，比较仍然有效。

 ## 边缘情况

 当多个段共享相同的极端端点时，就会出现关键的边缘情况。 例如，如果多个段具有相同的最大左端点，则删除其中一个不应更改最大值，除非它是最后一个剩余实例。 该算法通过维护出现次数来处理此问题，因此只有唯一的极端情况才会触发回退到第二最佳值。 

另一种情况是所有段都相同。 在这种情况下，删除任何一个线段都不会改变交集，交集仍然是长度为零的相同点线段。 该逻辑正确地保持两个新边界不变，因为两个端点都不是唯一负责极端的。 

当第二最佳值永远不会更新时，就会出现最后一个微妙的情况，因为所有值都是相同的。 哨兵初始化确保回退仍然产生有效值，并且由于结果被 max(0, ...) 钳位，所以空交叉点仍然得到正确处理。
