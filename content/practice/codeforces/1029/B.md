 ## 解决方案
 ## 问题理解

 我们得到了一份不同问题困难的排序列表。 从这个列表中，我们必须选择一个子序列，不一定是连续的，它将形成一个竞赛。 唯一的限制是局部的：如果我们对所选难度进行递增排序，则每对连续的难度都必须满足下一个难度最多是前一个难度的两倍。 

目标是选择一个满足该乘法约束的子序列，同时最大化其长度。 

输入大小最多可达 200,000 个元素。 这立即排除了任何尝试枚举子集甚至尝试使用嵌套探索的所有起点的解决方案，因为周围的二次行为$10^{10}$操作在一秒钟内不可行。 我们需要一些线性的东西，或者最坏的情况下是线性的。 

当最佳子序列在原始数组中不连续时，会出现微妙的边缘情况。 例如，即使两个值在索引上相距很远，如果跳过中间值，它们仍然可能属于最优链。 总是选择数组中相邻元素的天真贪婪在以下情况下会失败`1 2 4 8 9 10`，其中跳过某些元素允许更长的有效链。 

另一个问题是条件不对称。 它仅约束所选子序列中的相邻元素，而不是所有对。 这意味着该结构是一个连锁条件，而不是全局比率约束。 

## 方法

 蛮力的想法是尝试每个起始索引，并通过重复选择下一个有效元素来贪婪地向前扩展链。 对于固定的开始，扫描远期成本$O(n)$，每次开始都这样做会导致$O(n^2)$。 和$2 \cdot 10^5$，这大致变成$4 \cdot 10^{10}$比较，这太慢了。 

关键的观察是，一旦我们确定了起点，我们就不需要重新考虑之前的元素。 约束仅取决于最后选择的值。 这表明贪婪的前向构造，但贪婪地从每个索引开始是多余的。 

相反，我们维护一个滑动窗口结构：对于每个位置，我们尝试扩展从那里开始的最长有效链，但是我们重用了这样一个事实：如果一条链从$i$已知，那么最好的扩展是$i+1$可以重用部分工作。 这将问题转化为两指针样式扩展，其中我们维护一个有效的窗口并跟踪其最大尺寸。 

因为序列是排序的，所以我们只需要确保连续选择的元素之间的局部有效性。 这使我们能够在维持条件的同时延长窗口$a[j] \le 2 \cdot a[j-1]$，仅当条件破坏时才收缩。

 ## 算法演练

 我们从左到右处理数组，同时维护以当前位置结束的最长有效链。 

1. 从数组开头的指针开始，并将答案设置为 1，因为任何单个元素都是有效的竞赛。 这建立了基线链长度。 
2. 从第二个元素开始遍历数组。 对于每个元素，通过验证它是否最多是先前选择的元素的两倍来检查它是否可以扩展当前链。 
3. 如果条件成立，则将当前链长度延长一。 这反映了可以在不破坏约束的情况下附加当前元素。 
4. 如果条件不成立，则从当前元素开始将链长度重置为 1。 这是必要的，因为在给定的约束下，当前链中的较早元素无法连接到该元素。 
5. 每一步之后，用当前链长度更新全局最大值。 

该算法依赖于这样一个事实：我们总是扩展以前一个索引结尾的最长有效链。 如果扩展失败，重新开始是最佳选择，因为任何较早的起点都会违反同一边界的链条件。 

### 为什么它有效

 在任何位置，算法都会维护在该位置结束的最长有效子序列，并遵守本地条件。 由于有效性仅取决于相邻对，因此任何最优解都可以分解为条件连续成立的最大分段。 最佳竞赛对应于最佳起始重置后最长的此类段，并且算法隐式枚举所有可能的段端点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    best = 1
    cur = 1
    
    for i in range(1, n):
        if a[i] <= 2 * a[i - 1]:
            cur += 1
        else:
            cur = 1
        if cur > best:
            best = cur
    
    print(best)

if __name__ == "__main__":
    solve()
```该解决方案保留当前有效链长度的运行计数。 变量`cur`表示在数组中移动时有多少个连续选定的元素满足加倍条件。 当条件破裂时，我们会重置，因为不可能跨越该边界进行扩展。 

变量`best`跟踪数组中任何位置看到的最大链长度。 由于在这种变换意义上，每个有效子序列必须显示为连续的有效段，因此跟踪这些段就足够了。 

关键的实现细节是在何时重置`a[i] > 2 * a[i-1]`。 错过这种严格的不平等或忘记重置会导致无效链的计数过多。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))

    best = 1
    cur = 1

    for i in range(1, n):
        if a[i] <= 2 * a[i - 1]:
            cur += 1
        else:
            cur = 1
        best = max(best, cur)

    return str(best)

# provided sample

 ## 边缘情况

 对于像这样的单元素输入`1`，算法初始化`best = 1`并立即返回。 不会发生任何转换，因此不会触发错误的重置或比较。 

对于严格增加一倍以上的序列，例如`1 3 7 15`，每次比较都会失败，因为每个下一个元素都超过前一个元素的两倍。 循环重置`cur`每一步都为 1，并且`best`始终保持 1，匹配正确答案，因为在有效的竞赛中没有两个元素可以共存。 

对于完全有效的序列，例如`1 2 3 5`，每个相邻对都满足约束，所以`cur`连续增长到 4。算法永远不会重置，正确地将整个数组识别为有效的竞赛。
