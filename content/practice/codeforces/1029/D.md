 ## 解决方案
 ## 问题理解

 我们得到一个正整数列表和一个模数$k$。 对于每个有序的不同索引对$(i, j)$，我们通过写来形成一个新数字$a_i$直接跟随$a_j$以十进制表示。 我们需要计算有多少个这样的有序串联可以被整除$k$。 

关键的困难在于串联不是对残数进行简单的算术运算。 如果我们表示的位数$a_j$作为$d_j$，那么串联等于$a_i \cdot 10^{d_j} + a_j$。 这引入了对数字长度的依赖性，因此每一对都取决于值和结构。 

约束条件很大：最多$2 \cdot 10^5$数字和$k$最多$10^9$。 对的二次解立即不可行，因为它需要大约$4 \cdot 10^{10}$最坏情况下的操作。 

当所有数字共享相同值时，会出现微妙的边缘情况。 例如，如果所有$a_i = 1$和$k = 1$，每对都是有效的。 正确的解决方案仍然必须有效地处理这个问题，而无需枚举对。 

另一个重要的边缘情况是数字具有不同的数字长度。 例如，$a_i = 1$,$a_j = 99$， 和$k = 100$。 连接是$199$，其行为与$991$，因此交换顺序在数值和模算术中都很重要。 

## 方法

 暴力方法检查每个有序对$(i, j)$，计算数字的位数$a_j$，构建连接值，并测试整除性$k$。 这是简单且正确的，但它执行$O(n^2)$操作，远远超出了限制$n = 2 \cdot 10^5$。 

关键的观察是我们实际上从来不需要完整的连接数。 我们只需要它的值模$k$。 如果$len(x)$是的位数$x$，则串联满足：$$concat(x, y) \bmod k = (x \cdot 10^{len(y)} + y) \bmod k$$因此，问题简化为计算对，其中：$$(a_i \cdot 10^{len(a_j)} + a_j) \equiv 0 \pmod{k}$$重新排列：$$a_i \cdot 10^{len(a_j)} \equiv -a_j \pmod{k}$$这建议按数字长度对数字进行分组。 对于每个数字，我们可以预先计算其余数模$k$，并且还预先计算 10 模的幂$k$对于长度 1 到 10（因为$a_i \le 10^9$，数字长度最多为 10)。 

对于固定的$a_j$，我们想要数一下有多少个$a_i$满足：$$a_i \equiv (-a_j) \cdot (10^{len(a_j)})^{-1} \pmod{k}$$然而，直接使用模逆是不必要的，而且有时是不安全的$k$不是质数或与 10 共享因子。相反，我们反转观点：对于每个数字，我们尝试所有可能的数字长度并与预先计算的频率表进行匹配。 

我们维护按数字长度分组的先前见过的数字的余数计数。 对于每个数字$a_i$，我们将其视为该对的第二个元素，并计算对于每个可能的数字长度，有多少个较早的数字可以与它配对。 然后我们更新频率结构。 

这将问题减少到$O(n \cdot 10)$。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最佳|$O(n \cdot 10)$|$O(k)$| 已接受 |

 ## 算法演练

 ### 关键思想设置

 我们预先计算 10 模的幂$k$，因为数字移位是连接数字时唯一的结构转换。 

我们还按数字长度对数字进行分组，并跟踪迄今为止看到的残基的频率。 

### 步骤

 1. 预计算`pow10[d] = 10^d mod k`适用于从 1 到 10 的所有数字长度。这使我们能够通过附加数字来快速模拟数字左移。 
2.维护字典`cnt[len][rem]`， 在哪里`len`是数字长度并且`rem`是余数模$k$。 它存储有多少个先前处理的给定数字长度的数字具有给定余数。 
3. 从左到右处理数组。 对于每个数字$x$，计算：

 - 其余数`rx = x % k`- 它的数字长度`dx`4. 对于当前数字作为一对中的第二个元素，尝试所有可能的数字长度$d$对于第一个元素：

 - 我们需要$(y * 10^{dx} + x) % k = 0$- 所以$y * 10^{dx} \equiv -x \pmod{k}$- 这成为使用存储频率的直接检查：$$y \equiv (-x) \cdot 10^{-dx} \pmod{k}$$我们不是计算逆，而是通过检查所有可能存储的残差来进行模拟。 
5. 计算结束于的对的贡献后$x$， 插入$x$进入`cnt[dx][rx]`。 

### 为什么它有效

 在任何时刻，`cnt`准确地表示数字的多重集，该数字可以作为一对中的第一个元素出现，而当前索引作为第二个元素。 当我们处理第二个元素时，每个有效对都只计算一次。 数字长度完全决定模移位的行为方式，因此按长度分组可以保持正确性。 不会遗漏任何对，因为每个较早的元素都会针对每个有效的数字长度转换进行考虑。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    def digits(x):
        return len(str(x))

    pow10 = [1] * 11
    for i in range(1, 11):
        pow10[i] = (pow10[i - 1] * 10) % k

    cnt = [[0] * k for _ in range(11)]
    ans = 0

    for x in a:
        rx = x % k
        dx = digits(x)

        for d in range(1, 11):
            # we want y * 10^dx + x ≡ 0 mod k
            # => y * 10^dx ≡ -x mod k
            need = (-rx) * pow10[dx] % k

            ans += cnt[d][need]

        cnt[dx][rx] += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了算法。 这`pow10`数组处理模算术中的数字移位。 二维数组`cnt`避免哈希开销并确保快速访问。 对于每个数字，我们在插入当前数字之前查询第一个元素的所有可能的数字长度，这可以防止自配对并确保排序$(i, j)$。 

唯一微妙的一点是计算`need`。 它对第一个数字的残数的要求进行编码，以便在移位后`10^dx`并添加`x`，结果可被除以`k`。

 ## 边缘情况

 对于所有元素都相同的数组，并且$k = 1$，每个有序对都是有效的。 该算法处理这个问题是因为每个余数都为零，并且每次查找都匹配所有先前的条目，无论数字长度如何。 

对于最大数字长度（10 位数字）的数字，正确性取决于确保`pow10`预先计算最多为 10。如果忽略此界限，数字移位将默默地变得不正确。 这里，每个10位数字仍然通过相同的残差逻辑进行相同的处理，因此不需要特殊处理。 

对于没有串联可以整除的情况$k$，例如精心选择的残基在移位时永远不会对齐，频率表只是简单地累积而不会匹配`need`价值。 答案仍然为零，因为没有计算任何有效的对。
