 ## 解决方案
 ## 问题理解

 我们得到一个字符串和一个数字$k$。 从字符串中，我们查看所有具有精确长度的子序列$k$。 问题是在字符串中选择位置的任何两种不同方法是否可以产生相同的结果长度 -$k$字符序列。 

这里的子序列意味着我们选择索引$i_1 < i_2 < \dots < i_k$，按该顺序读取字符，并获得一个字符串。 不同的索引选择可以产生相同的结果字符串，但我们会被问到这种情况是否会发生。 

因此，该任务是对所有长度进行唯一性检查 -$k$子序列：我们想知道大小索引集的映射是否$k$结果字符串是单射的。 

这些约束清楚地表明，暴力破解子序列是不可能的。 字符串长度可以达到$10^5$每个测试用例最多有$10^3$测试用例总长度为$3 \cdot 10^5$。 甚至生成所有长度的子序列$k$是组合爆炸性的，因为子序列的数量是$\binom{n}{k}$，对于中等强度来说已经是天文数字了$n$。 

第二个约束是隐式的：我们只关心是否存在重复项。 这意味着我们正在寻找子序列之间的冲突，而不是完全枚举它们。 

当字符串包含相距较远的重复字母时，会出现微妙的边缘情况。 例如，在`"abca"`，子序列`"aa"`可以由索引 (1,4) 和 (2,4) 组成，这是无效的，因为字符不同，但重复的字符仍然可以为相同的子序列创建多个索引模式。 仅检查本地模式或相邻重复项的简单解决方案会错过远程碰撞。 

另一个边缘情况是当$k = 1$。 那么子序列就是单个字符。 当且仅当字符串中的所有字符都不同时，它们才是唯一的。 任何重复都会立即意味着重复的子序列。 

什么时候$k = n$，正好有一个子序列，所以答案总是肯定的。 

真正的困难是理解两个不同的索引选择何时可以产生相同的长度 -$k$子序列，以及如何在不枚举组合的情况下检测该结构。 

## 方法

 直接方法试图生成每个长度-$k$子序列并将其存储在哈希集中。 原则上这是正确的，因为我们可以比较每个生成的字符串并检测重复项。 然而，即使生成一个子序列也需要选择$k$指数，并且有$\binom{n}{k}$其中。 在最坏的情况下，这个数字会呈指数增长$n$，这使得即使对于$n = 50$。 

关键的观察结果是，仅当以保留顺序但仍产生相同结果字符串的方式为相同字符选择位置存在歧义时，才会出现重复。 我们不应该将子序列视为组合对象，而是可以考虑构建子序列的词典编排结构并跟踪不同的索引路径是否可以收敛。 

一种有用的重新表述是修复子序列字符串并询问它是否有多个嵌入到原始字符串中。 这变成了一个模式计数问题：是否存在长度-$k$至少以两种不同方式显示为子序列的字符串？ 

关键的结构简化是，如果字符串中的所有字符在特定意义上都足够全局不同，那么每个子序列都由贪婪的从左到右构造唯一确定。 当存在一个位置时，在匹配子序列期间，我们可以在两个相同的字符之间进行选择，这两个字符都可以使用而不会阻止其余字符的完成，此时就会出现歧义。$k-1$匹配。 

这导致了贪婪的可行性表征：我们模拟构建子序列并检测任何步骤是否允许多个有效的下一个选择，这些选择都会导致完成。 

我们通过跟踪每个前缀位置存在多少种扩展部分匹配的选择来避免显式子序列枚举。 当我们找到两个相同字符都可以用来完成子序列的位置时，唯一性就失效了。 

这简化为基于扫描的检查，我们维护每个前缀存在多少个有效的延续路径，但我们只需要检测它是否超过一个。 

最终的减少是，当且仅当存在一个位置，其中剩余后缀包含至少两次出现的某个字符，并且这些字符都可以用作有效长度中的下一个选定元素时，唯一性就会失败 -$k$后续延续。

 | --- | --- | --- | --- |
 | 蛮力 |$O(\binom{n}{k} \cdot k)$|$O(k)$| 太慢了 |
 | 最佳|$O(n \cdot 26)$|$O(26)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 预先计算每个字符的后缀计数。 对于每个职位$i$，我们知道每个字母出现了多少次$s[i:]$。 这允许对字符串后面可用的内容进行恒定时间推理。 
2. 我们模拟从左到右贪婪地构建一个子序列，但我们不是构建一个子序列，而是跟踪是否存在任何步骤，其中对于某个长度的下一个字符存在多个有效选择 -$k$子序列。 
3. 对于每个位置$i$，考虑使用$s[i]$作为下一个选择的角色。 我们检查是否仍然可以完成$k-1$后缀中的字符。 这种可行性是标准的贪婪子序列匹配：我们确保之后存在足够的剩余字符$i$。 
4. 当我们找到所需字符的有效位置时，我们检查字符串中后面是否存在具有相同字符的另一个位置，该位置也可以作为下一个选择，同时仍然留有足够的空间来完成子序列。 如果存在这样的第二个位置，我们立即得出结论，唯一性失败。 
5. 如果我们从未遇到过对同一子序列构造状态有两个有效选择的步骤，则每个子序列都有唯一的嵌入。 

这样做的原因是任何重复的子序列都对应于两个不同的嵌入，并且在嵌入不同的第一个索引处，两个选择都必须是本地有效的并且仍然完成剩余的后缀。 检测到这个分支点就足够了。 

### 为什么它有效

 任意长度-$k$子序列对应于关于每个字符出现的次数的决策序列。 如果两个不同的索引序列产生相同的子序列，则存在它们分歧的第一个位置，选择同一字符的两个不同出现，同时仍然允许完成剩余的$k-1$人物。 我们的扫描准确地检测到这种情况，因为它在每个阶段都会检查同一所需的下一个字符是否存在多个有效的连续位置。 如果不存在这样的分支点，则每个子序列都有唯一的嵌入，因此所有子序列都是唯一的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, k = map(int, input().split())
        s = input().strip()

        if k == 1:
            # duplicates exist iff any character repeats
            seen = set()
            ok = True
            for c in s:
                if c in seen:
                    ok = False
                    break
                seen.add(c)
            print("Yes" if ok else "No")
            continue

        if k == n:
            print("Yes")
            continue

        # suffix counts
        suf = [[0] * 26 for _ in range(n + 1)]
        for i in range(n - 1, -1, -1):
            suf[i] = suf[i + 1][:]
            suf[i][ord(s[i]) - 97] += 1

        # try to find any ambiguity
        # we scan possible start positions of subsequences
        # and check whether first pick can be ambiguous
        possible_start = [[] for _ in range(26)]
        for i, c in enumerate(s):
            possible_start[ord(c) - 97].append(i)

        bad = False

        for c in range(26):
            idxs = possible_start[c]
            if len(idxs) < 2:
                continue

            # check if at least two occurrences can both start a valid subsequence
            # we test greedily from each occurrence
            valid = []
            for i in idxs:
                need = k - 1
                j = i + 1
                cnt = 0
                # greedy count how many we can still pick
                while j < n and need > 0:
                    need -= 1
                    j += 1
                if need == 0:
                    valid.append(i)

            if len(valid) >= 2:
                bad = True
                break

        print("No" if bad else "Yes")

if __name__ == "__main__":
    solve()
```实现首先将琐碎的情况分开。 什么时候$k = 1$，答案仅取决于字符串是否有重复项。 什么时候$k = n$，恰好有一个子序列。 

主要思想是检测同一字符的两个不同位置是否都可以作为有效长度的第一个元素 -$k$子序列。 如果是这样，我们就会立即发生碰撞。 每个候选者内部的贪婪检查确保从选定的起始位置，我们仍然可以完成$k$使用右侧剩余的字符进行选择。 

关键的微妙之处在于，我们不需要完全枚举超出第一个分支点的子序列，因为任何重复都必须表现为最早不同索引处的分歧。 

## 工作示例

 考虑`s = "threes", k = 4`。 

我们检查每个角色的位置。 人物`'e'`出现两次。 我们检查这两个事件是否可以开始或参与有效的长度为 4 的子序列。 贪婪补全在两次出现时都会成功，因为后缀中保留了足够的字符。 这为同一子序列创建了两个不同的嵌入`"tres"`，所以输出是`"No"`。 

| 步骤| 人物 | 职位| 剩余需求| 可行完成 |
 | --- | --- | --- | --- | --- |
 | 1 | 电子| 3 | 3 | 是的 |
 | 2 | 电子| 4 | 3 | 是的 |

 这显示了等效子序列的两个有效起始嵌入，确认了非唯一性。 

现在考虑`s = "abcdba", k = 4`。 

在这里，虽然字母重复，但形成有效长度为 4 的子序列的任何选择都被强制采用唯一的结构，因为后面的重复项不能同时以保留两个嵌入的完整有效延续的方式使用。 从每个候选起点开始的贪婪完成最多产生一条有效路径，因此不会发生分支。

 ## 边缘情况

 对于$k = 1$，该算法崩溃为对字符的唯一性检查。 在`"aba"`，扫描检测到`'a'`两次，所以它立即返回`"No"`，匹配重复子序列的存在`"a"`。 

为了$k = n$， 例如`"abc"`和$k=3$，算法绕过所有检查并返回`"Yes"`，与恰好存在一个子序列一致。 

对于重复次数较多的字符串，例如`"aaaaaa"`和$k = 3$，每个索引选择都会产生相同的子序列`"aaa"`。 贪心可行性检查找到多个有效的起始位置`'a'`，立即确认非唯一性。
