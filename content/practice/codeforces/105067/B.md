 ## 解决方案
 ## 问题理解

 我们得到一个长度为 n 的数组和一个固定的交换序列。 每个交换交换数组中的两个位置，并且该序列始终以相同的顺序应用。 运行一次所有交换后，Jinshi 检查结果数组是否按非降序排序，然后将数组恢复到原始状态。 

每天，在 Jinshi 执行此固定交换序列之前，Maomao 对当前阵列执行一次交换。 这种修改会持续数天，因此阵列会随着时间的推移而演变。 每次修改后，我们必须决定运行固定交换序列是否会以排序数组结束。 

重要的结构是所有查询的交换顺序都是固定的，而只有初始数组每天通过交换两个位置而略有变化。 

约束允许最多 100000 个元素、100000 次固定交换和 100000 次查询。 这立即排除了任何重新模拟每个查询的完整交换序列的方法，因为这将花费 O(nm) 甚至 O(mq)，两者都太大了。 即使每个查询从头开始重新计算最终数组也会太慢。 

正确的解决方案必须在线性预处理后将每个查询减少到接近恒定的时间。 

尝试直接模拟时会出现一个微妙的问题。 例如，如果我们在 Maomao 交换之后通过再次应用所有 m 交换来重新计算最终数组，我们将传递较小的输入，但当 m 和 q 很大时会失败。 

另一个常见的陷阱是假设Maomao的交换仅影响最终排序结果中的本地位置。 除非我们明确跟踪索引如何通过固定交换序列传播，否则情况并非如此。 

## 方法

 蛮力的想法很简单。 对于每个查询，将Maomao的swap应用到数组中，然后按顺序模拟m次swap，最后检查结果数组是否已排序。 每次模拟的成本为 O(m + n)，因此总复杂度变为 O(q(m + n))。 每个维度有 100000 次操作，这是完全不可行的，在最坏的情况下会导致大约 10^10 次操作。 

关键的观察是 m 次交换的序列定义了索引的固定排列。 如果我们从索引 1 到 n 开始并将交换应用于这些索引，我们将获得一个排列 p，使得在所有交换之后，位置 i 处的元素来自原始数组的位置 p[i]。 这将整个交换过程减少为单个重新索引步骤。 

一旦知道了这种排列，交换序列之后的结果就是一个变换后的数组 c，其中 c[i] = a[p[i]]。 问题就变成了 c 是否已排序。 

现在关键的简化出现了：Maomao 只改变了 a 中的两个位置。 由于p是固定的，因此只有c中的两个位置p^{-1}(l)和p^{-1}(r)受到影响。 c 中的所有其他位置保持不变。 这意味着每个查询仅修改转换后的数组中的两个条目，并且我们只需要检查这些索引周围的排序条件是否仍然在本地成立。 

我们不是每次都重建和检查整个数组，而是维护 c 中哪些相邻对是有效的。 当且仅当每个相邻对满足 c[i] <= c[i+1] 时，数组才排序。 每次交换仅影响涉及两个更新位置及其邻居的比较，因此我们可以在每个查询的恒定时间内更新答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 重新模拟每个查询的所有交换 | O(q(m + n)) | O(n) | 太慢了 |
 | 排列+局部邻接跟踪| O(n + m + q) | O(n + m + q) | O(n) | 已接受 |

 ## 算法演练

 我们首先将整个交换序列压缩为索引上的单个排列，然后在更新下维护派生数组。

1. 从位置 1 到 n 的恒等映射开始，表示每个最终位置从原始数组中提取其值的位置。 
2. 将 m 次交换中的每一次应用到该恒等映射。 每次交换都会交换两个位置的图像，构建一个排列 p，使得最终位置 i 从原始位置 p[i] 获取值。 
3. 构造一个逆向查找数组 pos，使得 pos[x] 是唯一索引 i，其中 p[i] = x。 这使我们能够快速定位当原始数组的索引 x 发生变化时，转换后的数组中的哪个位置受到影响。 
4. 使用 c[i] = a[p[i]] 构建转换后的数组 c。 
5. 预先计算一个 bad 数组，其中如果 c[i] > c[i+1]，则 bad[i] 为 true。 维护一个计数器 bad_count 来记录存在多少违规行为。 当 bad_count 为零时，数组会精确排序。 
6. 对于每个查询，当交换 a 中的位置 l 和 r 时，使用 i = pos[l] 和 j = pos[r] 定位它们在 c 中受影响的位置。 
7. 暂时从 bad_count 中删除涉及 i 和 j 的所有邻接检查的贡献，更新 c[i] 和 c[j]，然后恢复这些位置的邻接检查。 
8.更新后，如果bad_count为零，则输出Y，否则输出N。 

这样做的原因是 c 中所有未受影响的位置保持相同，因此远离 i 和 j 的所有比较都保持有效。 只有涉及 i 和 j 的比较才会改变，因此仅跟踪局部邻接就足以保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    a = list(map(int, input().split()))

    # build permutation p on positions
    p = list(range(n))
    for _ in range(m):
        x, y = map(int, input().split())
        x -= 1
        y -= 1
        p[x], p[y] = p[y], p[x]

    # pos[value_index] = position in c
    pos = [0] * n
    for i in range(n):
        pos[p[i]] = i

    # build transformed array
    c = [a[p[i]] for i in range(n)]

    def add(i):
        nonlocal bad
        if 0 <= i < n - 1 and c[i] > c[i + 1]:
            bad += 1

    def remove(i):
        nonlocal bad
        if 0 <= i < n - 1 and c[i] > c[i + 1]:
            bad -= 1

    bad = 0
    for i in range(n - 1):
        if c[i] > c[i + 1]:
            bad += 1

    out = []

    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1

        i = pos[l]
        j = pos[r]

        # remove old affected edges
        for k in (i - 1, i, j - 1, j):
            if 0 <= k < n - 1:
                if c[k] > c[k + 1]:
                    bad -= 1

        # apply swap in original array
        a[l], a[r] = a[r], a[l]

        # update transformed values
        c[i] = a[p[i]]
        c[j] = a[p[j]]

        # add new affected edges
        for k in (i - 1, i, j - 1, j):
            if 0 <= k < n - 1:
                if c[k] > c[k + 1]:
                    bad += 1

        out.append('Y' if bad == 0 else 'N')

    print(''.join(out))

if __name__ == "__main__":
    solve()
```该实现将交换排列压缩与动态维护阶段分开。 排列 p 计算一次，并且 pos 允许对受影响的位置进行恒定时间定位。 从概念上讲，数组 c 始终表示应用固定交换序列后的当前变换状态。 

小心的部分是仅更新两个修改后的索引周围的四个边界区域。 每个排序违规完全由相邻对决定，因此我们永远不需要检查这些邻域之外的任何内容。 

## 工作示例

 ### 示例 1

 考虑一个小情况，其中 n = 5 并且交换序列引起一些固定排列。 假设预处理后我们有一个转换后的数组 c = [2, 4, 1, 3, 5]。 

我们通过检查相邻比较来维护坏对。 

| 步骤| 运营| 更新后的c | 坏对|
 | --- | --- | --- | --- |
 | 0 | 初始| [2, 4, 1, 3, 5] | (4 > 1) |
 | 1 | 掉期影响指数导致修正| [1, 4, 2, 3, 5] | 无 |

 确定相关位置后，数组已排序，因此答案为 Y。 

这表明每次查询只有一小部分数组发生变化，并且不需要全局重新计算。 

### 示例 2

 以数组在转换后已排序的情况为例。 

| 步骤| 运营| 更新后的c | 坏对|
 | --- | --- | --- | --- |
 | 0 | 初始| [1,2,3,4,5]| 无 |
 | 1 | 交换造成混乱| [2,1,3,4,5]| (2 > 1) |

 这里，单个交换恰好引入了一次反转，并且只有修改索引附近的相邻检查才能检测到它。 该算法无需扫描整个阵列即可立即识别违规行为。

 ## 边缘情况

 一个关键的边缘情况是当 Maomao 交换映射到变换后的数组 c 中相邻位置的两个位置时。 在这种情况下，两个受影响的索引在其邻接更新中重叠。 该算法可以处理此问题，因为它会在重新计算之前删除并重新添加两个索引周围所有可能受影响的边，从而确保不保留陈旧的比较。 

当 l 和 r 映射到 pos 下的同一位置时，会出现另一种边缘情况。 只有当 l == r 时才会发生这种情况，在这种情况下更新是无操作的。 该算法仍然安全地处理它，因为删除并重新添加相同的本地邻域会使 bad_count 保持不变。 

第三种情况是更新影响边界位置 i = 0 或 i = n - 1。该实现通过边界保护邻接检查，因此不会进行无效比较，并且在边缘保留正确性。
