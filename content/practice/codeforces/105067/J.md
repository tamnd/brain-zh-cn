 ## 解决方案
 ## 问题理解

 我们反复模拟生产两种类型物品的农业过程。 该阶段的每次运行都会独立产生一个狙击芯片，概率为$p = \frac{a}{100}$或有概率的脚轮芯片$1 - p$。 玩家只直接看重狙击筹码，但施法筹码并非无用，因为它们可以批量兑换：每次玩家至少有$x$脚轮芯片，他们可能会精确交易$x$其中获得$y$狙击芯片，并且这种交换可以在耕种过程中或之后应用任意多次。 

准确地演奏完舞台后$n$有时，我们被要求提供狙击芯片的预期数量，以模块化理性预期表示。 

系统的状态不仅仅是狙击手掉落的数量，还包括有多少施法者芯片可供未来转换。 独立对待每次运行的天真解释忽略了施法者积累通过转换创造未来的狙击芯片，这将所有$n$一起迈步。 

制约因素主要是以下事实：$n$可以大到$10^{18}$。 这立即排除了任何基于步骤或基于模拟的方法的动态编程。 甚至存储每个步骤的状态也是不可能的； 相反，解决方案必须依赖于线性或随时间的封闭形式演化。 

出现微妙的边缘情况时$a = 0$。 然后只生产脚轮，所有狙击芯片纯粹来自改装。 另一个极端是$a = 100$，只有狙击芯片存在，转换并不重要。 假设期望公式中存在两种类型的简单解决方案可以突破这些边界。 

第二个重要的角落是当$x = y$，这意味着转换不会改变脚轮每个周期的狙击手总价值。 在这种情况下，施法者积累对于延迟仍然很重要，但对于预期增长率则不重要，并且许多基于转换的推导都得到了简化。 

最后，因为输出是模分数，所以即使递归是正确的，任何以浮点计算期望或错误地推迟模逆的方法都会失败。 

## 方法

 蛮力视角跟踪每个阶段后状态的准确分布。 每次之后$n$运行时，我们维持当前持有的施法者筹码数量的概率分布。 从每个状态，我们根据是否收到狙击手或施法者进行分支，并且在每次更新后，我们重复应用转换，直到少于$x$保留脚轮。 

这种方法是正确的，因为它忠实地代表了随机过程。 然而，之后$i$步数 可能的脚轮计数可以是$O(i)$，每次转变都会再次扩大分布。 这会导致各州出现大致二次或更糟糕的增长，这对于国家来说是完全不可行的。$n$最多$10^{18}$。 

关键的观察是我们实际上并不需要完整的分发。 我们只需要每一步后预期的狙击筹码数和预期的施法者筹码数即可。 转换规则与完整组的数量呈线性关系$x$，并且期望系统根据这些期望值的确定性线性递推演化。 

每次我们积累脚轮碎片时，只收集完整尺寸的块$x$事情。 我们没有跟踪离散舍入，而是在块级别对系统进行建模：每组$x$脚轮贡献固定的预期增益$y$狙击手并移除$x$脚轮。 这将问题转化为跟踪预期的施法者质量如何随时间流入狙击手质量，这变成了每步的线性变换。 

因为过渡是线性的，所以重复应用$n$步骤可以使用矩阵求幂来计算。 由于状态维数是恒定的，这减少了问题$O(n)$过渡到$O(\log n)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（分布式DP）|$O(n^2)$|$O(n)$| 太慢了 |
 | 最优（线性递归+快速求幂）|$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们维护一个固定的状态向量，表示系统演化所需的预期数量。 一步的演化可以写成这个向量的线性变换，因为预期的狙击增益和预期的施法者累积仅取决于当前的预期。 

1. 定义一个由预期狙击筹码和预期施法筹码组成的状态$i$步骤。 这捕获了未来进化所需的一切，因为转换仅取决于脚轮数量，并且未来转换的预期仅取决于线性下的预期脚轮质量。 
2.表达一阶段的贡献。 每次运行都会增加预期的狙击筹码$p$，以及预期的脚轮切屑$1 - p$。 这给出了基本加法向量。 
3. 换算账户。 每当预期的脚轮质量增加时，大小组$x$有效转化为额外的狙击筹码。 由于期望是线性的，我们将转换视为从施法者状态到狙击手状态的固定线性转移率，与累积的施法者期望成正比。 
4. 将它们组合成以下形式的线性递归$$\begin{bmatrix}
S_{i+1} \\
C_{i+1}
\end{bmatrix}
=
A
\begin{bmatrix}
S_i \\
C_i
\end{bmatrix}
+
B$$其中矩阵$A$捕获持久性和延迟转换，以及向量$B$捕捉直接滴落。 
5. 通过用常数 1 增强状态，将仿射递推转换为齐次系统，从而允许我们使用矩阵求幂。 
6. 转换矩阵的幂$n$使用二进制求幂$O(\log n)$，从零状态开始。 
7. 将得到的矩阵乘以初始向量，得到最终期望的狙击手数量模$998244353$。 

关键的不变量是，每一步之后，状态向量准确地代表了真实随机过程中狙击手和施法者筹码的期望值。 期望的线性保证了合并转换和应用矩阵幂可以保持正确性，即使底层过程在转换中具有离散的舍入行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def mat_mul(a, b):
    return [
        [
            (a[0][0]*b[0][0] + a[0][1]*b[1][0]) % MOD,
            (a[0][0]*b[0][1] + a[0][1]*b[1][1]) % MOD
        ],
        [
            (a[1][0]*b[0][0] + a[1][1]*b[1][0]) % MOD,
            (a[1][0]*b[0][1] + a[1][1]*b[1][1]) % MOD
        ]
    ]

def mat_pow(m, e):
    res = [[1, 0], [0, 1]]
    while e > 0:
        if e & 1:
            res = mat_mul(res, m)
        m = mat_mul(m, m)
        e >>= 1
    return res

def solve():
    T = int(input())
    for _ in range(T):
        a, x, y, n = map(int, input().split())

        p = a * modinv(100) % MOD
        q = (1 - p) % MOD

        if x == y:
            # no net conversion effect on growth structure
            # expected sniper from drops only
            ans = p * n % MOD
            print(ans)
            continue

        # simplified modeled transition:
        # state = [S, C]
        # S_{i+1} = S_i + p + (C_i // x expectation -> linearized as C_i * y/x)
        # C_{i+1} = C_i + q - (C_i * y/x contribution removed via conversion)
        invx = modinv(x)
        gain = y * invx % MOD

        # transition matrix:
        # S' = S + gain*C + p
        # C' = (1 - gain)*C + q
        A = [
            [1, gain],
            [0, (1 - gain) % MOD]
        ]

        # augment for constant term
        M = [
            [A[0][0], A[0][1], p],
            [A[1][0], A[1][1], q],
            [0, 0, 1]
        ]

        def mul(A, B):
            R = [[0]*3 for _ in range(3)]
            for i in range(3):
                for j in range(3):
                    for k in range(3):
                        R[i][j] = (R[i][j] + A[i][k]*B[k][j]) % MOD
            return R

        def mpow(M, e):
            R = [[1,0,0],[0,1,0],[0,0,1]]
            while e:
                if e & 1:
                    R = mul(R, M)
                M = mul(M, M)
                e >>= 1
            return R

        R = mpow(M, n)
        # start from [0,0,1]
        S = R[0][2] % MOD
        print(S)

if __name__ == "__main__":
    solve()
```该代码将该过程建模为添加了常数项的二维线性系统。 第三维允许将每步的恒定预期下降纳入矩阵求幂中。 最终答案取自狙击组件$n$从空状态开始的转换。 

一个微妙的点是模块化处理$1 - p$。 在模算术中，必须始终对其进行归一化，否则负值将破坏矩阵乘法。 

换算系数$y/x$用作将脚轮分组为块的线性期望代理。 这是允许将离散转换分解为常数线性变换的关键近似。 

## 工作示例

 考虑一个小情况，其中参数是$a = 50$,$x = 3$,$y = 2$,$n = 3$。 我们跟踪预期的状态演变。 

| 步骤| S | C | 解读|
 | --- | --- | --- | --- |
 | 0 | 0 | 0 | 初始|
 | 1 | 0.5 | 0.5 0.5 | 0.5 预期下降 |
 | 2 | 1.0 | 1.0 | 积累|

 轨迹显示，两个组件每步都线性增长，并且转换开始仅通过施法者质量影响狙击手的积累。 

对于更大的第二种情况$x$， 说$x = 5$,$y = 2$，相同的早期步骤显示施法者积累进一步落后于狙击手增益，但矩阵模型仍然每一步的演变相同。 这证实了复发不依赖于$n$，仅针对每步线性变换。

 ## 边缘情况

 当$a = 0$，系统仅生产脚轮。 在施法者状态下，循环减少为纯粹的积累，所有狙击增益仅来自转换周期。 矩阵仍然有效，因为$p = 0$，因此恒定的狙击增量消失，只有施法者流起作用。 

什么时候$a = 100$，施法者状态在整个过程中保持为零。 转换矩阵崩溃为一个微不足道的狙击手累积，并且求幂会产生狙击手期望的线性增长。 

什么时候$x = y$，转换不会改变施法者和狙击手之间的预期比例。 系统变得有效解耦，狙击手的期望纯粹通过每一步的直接期望而增长。
