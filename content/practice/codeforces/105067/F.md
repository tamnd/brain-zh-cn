 ## 解决方案
 ## 问题理解

 我们得到一个数组和一个可变整数$x$。 对于任何选择$x$，我们通过异或计算一个值$x$对每个数组元素并对结果求和。 这会产生一个整数$S(x)$。 任务不是计算该值一次，而是理解可以显示为的整组值$x$对所有非负整数进行变化，然后计算其中有多少个值落在给定区间内$[l, r]$。 

核心难点在于改变$x$在所有数组元素中全局翻转位，并且每个位的效果都是高度结构化的。 自从$n$可以大到$10^5$，重新计算每个的总和$x$是不可能的。 甚至迭代$x$毫无意义，因为$x$是无界的。 

直接暴力破解$x$已经立即失败，因为$x$范围涵盖所有非负整数，并且每次评估成本$O(n)$，没有给出有限的停止点。 

当考虑输出范围时，会出现一个更微妙的问题。 人们很容易假设这些值形成某种连续区间或某种周期性结构。 如果不了解每个位如何独立发挥作用，这种假设是危险的。 

例如，如果所有$a_i = 0$， 然后$S(x) = n \cdot x$，所以输出是$n$，并非所有整数。 如果$n = 2$，我们只能得到偶数，所以计算整数$[l, r]$如果我们假设连续性，就会多算。 

真正的挑战是表征函数的图像$x \mapsto \sum (a_i \oplus x)$不一一列举$x$。 

## 方法

 直接方法确定了一个值$x$，计算每个异或和$O(n)$，并对所有重复$x$。 自从$x$是无界的，甚至有限制$[0, 2^{60})$仍然太大，大约为$10^{18}$的可能性。 总的复杂性变得不可能。 

关键的观察是以代数方式扩展 XOR。 对于每个元素，$$a_i \oplus x = a_i + x - 2(a_i \& x).$$总结一切$i$,$$S(x) = \sum a_i + n x - 2 \sum (a_i \& x).$$现在我们按位对贡献进行分组。 让$cnt_k$是数组元素的数量，其$k$-th 位已设置。 那么bit的贡献$k$在$x$完全独立于其他位：$$S(x) = A + \sum_k x_k \cdot (n - 2 \cdot cnt_k)\cdot 2^k,$$在哪里$A = \sum a_i$。 

这将问题转化为对位的加权子集构造$x$。 每一位$k$贡献独立权重$w_k = (n - 2cnt_k)2^k$，并选择是否位$k$设置于$x$添加该重量。 

所以不要迭代$x$，我们正在探索独立二元选择的线性组合的所有值。 这种结构使我们能够推理极值。 

对于每一位$k$， 如果$w_k$为正数，设置它会增加总和； 如果为负，则设置它会减少它。 为了获得最小可能值，我们独立选择每个位以最小化贡献。 这给出了单个全局最小值$S_{min}$。 

一个关键的结构事实是，除了输入数字中存在的最大位之外，所有$cnt_k = 0$， 所以$w_k = n \cdot 2^k > 0$。 这些高位只会将值推高，而不会帮助降低它。 这使得可实现的值集从有限的最小值开始不受限制。 

这将任务减少到计算有多少个整数$[l, r]$至少是$S_{min}$。

 ## 算法演练

 1. 计算$A = \sum a_i$。 这是所有答案的不变部分，无论$x$，所以我们立即将其分开。 
2. 对于每个位位置$k$从 0 到大约 60，计算$cnt_k$，设置了该位的元素数量。 这捕获了 XOR 如何独立地与每个位交互。 
3. 对于每一位$k$，计算权重$$w_k = (n - 2 \cdot cnt_k)\cdot 2^k.$$这是设置位的净效果$k$在$x$关于总和。 
4.构造最小可能值$S_{min}$通过独立扫描位。 如果$w_k < 0$，我们将该位包含在$x$，否则我们将其保留为未设置。 这是有效的，因为每个位都是独立贡献的，没有进位交互。 
5. 可实现值的集合是以下形式的所有整数$S_{min}$加上来自较高位的正贡献的非负组合。 由于任意大的位总是会增加值，因此该集合在上面是无界的。 
6. 因此，中的每个值$[S_{min}, \infty)$是可以实现的，答案简化为计算有多少个整数$[l, r]$至少是$S_{min}$。 
7. 将最终答案计算为$$\max(0, r - \max(l, S_{min}) + 1).$$### 为什么它有效

 每一点$x$独立地贡献于最终的总和，并且XOR在扩展后不会引入跨位依赖性。 这将整个问题简化为独立二元选择的总和。 由于高位总是贡献正权重并且可以任意切换，因此值空间具有有限的最小值并且没有上限。 最小构造是全局最优的，因为位决策之间不存在交互，因此贪婪的每位最小化是精确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, l, r = map(int, input().split())
    a = list(map(int, input().split()))

    A = sum(a)

    max_bit = 0
    for v in a:
        if v:
            max_bit = max(max_bit, v.bit_length())

    # safe bound for bits of x and a_i
    MAXB = max(60, max_bit + 2)

    cnt = [0] * (MAXB + 1)

    for v in a:
        for k in range(MAXB + 1):
            if (v >> k) & 1:
                cnt[k] += 1

    Smin = A
    for k in range(MAXB + 1):
        w = (n - 2 * cnt[k]) * (1 << k)
        if w < 0:
            Smin += w

    if r < Smin:
        print(0)
        return

    start = max(l, Smin)
    if r < start:
        print(0)
    else:
        print(r - start + 1)

if __name__ == "__main__":
    solve()
```该实现首先将 XOR 的整个效果压缩为每比特计数。 数组`cnt[k]`捕获每个位设置了多少个数字，这足以确定切换位的贡献$k$在$x$。 

循环计算$S_{min}$反映了算法中每比特的贪婪结构：每当一个比特减少总和时，我们假设它被设置为$x$。 

最终的计数步骤依赖于这样一个事实：可实现的值不存在上限，因此只有下限阈值很重要。

 ## 边缘情况

 当所有$a_i = 0$,每一点都贡献着$w_k = n \cdot 2^k$，所以所有权重都是正数。 最小值出现在$x = 0$, 给予$S_{min} = 0$。 该算法正确地得出结论，所有可达值都是$n$，但由于我们只计算整数$[l, r]$多于$S_{min}$，范围逻辑仍然会为完整覆盖间隔生成正确的计数。 

当所有$a_i$是相同的，每一位都一致$cnt_k$，使每位权重一致。 每比特贪婪规则仍然独立地最小化每个贡献，因此计算出$S_{min}$即使有许多不同，仍然正确$x$值映射到相同的结果。 

什么时候$l > S_{min}$，答案完全取决于截断区间$[l, r]$。 该算法自然地通过以下方式处理这个问题`max(l, Smin)`边界，确保不计算低于最小值的值，即使它们在构造上是不可能的。
