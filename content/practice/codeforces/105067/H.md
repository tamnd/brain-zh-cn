 ## 解决方案
 ## 问题理解

 我们得到一个固定的字符串，然后是许多独立的查询，每个查询选择该字符串的一个连续段。 对于每个选定的段，我们必须报告另一个相同长度的段，其字符在一个位置上不同，或者报告这是不可能的。 

当两个子字符串除在一个索引处（其中字符不同）之外的所有位置都相同时，则认为它们是兼容的。 输出段必须来自相同的原始字符串，因此我们只能在同一字符串中选择另一个相同长度的子字符串。 

字符串长度最多7000，但查询数量可以高达100万。 这种不对称性是关键约束：预处理必须在 n 中几乎是线性或二次的，而每个查询必须在常数或对数时间内得到回答。 

简单的解释是将查询子字符串与相同长度的所有其他子字符串进行比较。 这太慢了，因为对于每个查询都有 O(n) 个候选者，每次比较的成本为 O(length)，在最坏的情况下会导致 O(n^3) 的行为。 

一个更微妙的困难是，即使存在有效答案，如果我们仅通过部分散列或前缀匹配来查找“相似”子字符串，也可能很容易错过。 要求恰好是一个不匹配，而不是最多一个，因此相同的子字符串是无效的答案。 

当查询的子字符串在其结构中已经是唯一的时，就会出现一个小边缘情况，这意味着没有其他子字符串在一个位置上完全相同。 例如，像“aaaa”这样的字符串对于“aaaa”段没有有效的答案，因为每个其他段要么相同，要么在多个位置不同。 

## 方法

 暴力方法通过枚举相同长度的每个可能的候选子字符串并逐个字符检查发生了多少不匹配来处理每个查询。 这是正确的，因为它直接强制执行“恰好一个不匹配”的定义。 然而，对于每个查询，在最坏的情况下这会花费 O(n^2) 的工作量，并且对于最多 10^6 的查询，这变得完全不可行。 

关键的观察是我们不需要重复比较完整的子字符串。 相反，我们可以预先计算信息来回答这个问题：“是否有另一个长度为 L 的子字符串与除了一个位置之外的所有位置都匹配的子字符串？”

 修复查询子字符串 s[l..r]。 如果我们在其中选择一个不匹配的位置 i，那么我们正在寻找另一个子字符串 s[l'..r']，使得除了 i 之外的所有位置都是相同的。 这意味着两个子串必须在除 i 之外的每个可能对齐周围的长度为 L-1 的窗口上一致。 这将问题转化为局部模式匹配问题，其中不匹配被隔离。 

我们可以利用滚动哈希或前缀哈希来快速比较子字符串，但更重要的是，我们可以预先计算所有子字符串的哈希值。 然后，对于每个查询，我们尝试通过隐式地一次更改一个位置来构造候选者。 技巧是避免为每个查询枚举所有 O(L) 不匹配位置。 

相反，我们通过完整哈希对所有子字符串进行分组来预处理它们。 对于每个子串长度 L，我们存储所有出现的哈希集。 然后，对于查询子字符串，我们尝试通过将其分成左右部分来在一个位置 i 处“破坏”它。 对于候选匹配，我们需要另一个共享前缀 (0..i-1) 和后缀 (i+1..L-1) 的子字符串。 这相当于组合两个哈希查询：前缀哈希和后缀哈希。 

因此，对于每个查询，我们可以从左到右扫描位置 i 并构造一个由以下内容组成的键：

 长度为 i 的前缀哈希和长度为 L-i-1 的后缀哈希。 如果存在另一个匹配两个部分但在位置 i 不同的匹配项，我们可以使用预先计算的索引检索其位置。

我们为每对（i，前缀哈希，后缀哈希）维护一个与该结构匹配的子字符串的起始位置列表，使我们能够在每个查询的 O(1) 附近找到有效的不同子字符串。

 ## 算法演练

 我们使用滚动哈希预处理所有子字符串，以便任何子字符串哈希都可以在 O(1) 内计算。 然后，我们构建辅助结构，使我们能够查询“与前缀后缀分割模式匹配的所有子字符串”。 

1. 预先计算字符串的前缀哈希值和幂。 这允许稍后提取 O(1) 子字符串哈希值。 
2. 对于从 1 到 n 的每个子串长度 L，枚举所有起始位置 i。 计算其完整哈希值，并存储子字符串内每个分割位置的前缀和后缀哈希值。 这将创建以下形式的签名（L、分割位置 k、前缀哈希、后缀哈希）。 
3. 将每个子字符串索引插入到由这些签名作为键控的字典中。 每个键至少存储一个发生该模式的有效起始位置。 
4. 对于每个查询 [l, r]，计算其长度 L 和子字符串的预先计算的哈希值。 
5. 尝试从 0 到 L-1 的每个分割位置 k：

 构造（s[l..l+k-1] 的前缀哈希，s[l+k+1..r] 的后缀哈希）。 

检查预先计算的映射中是否存在该签名。 
6.如果存在这样的签名，则检索候选起始位置l'。 确保 l' 与 l 不同，因为不允许有相同的子字符串。 
7.输出(l',l'+L-1)。 如果没有分割位置产生有效的不同子字符串，则输出 0 0。 

正确性依赖于这样一个事实：任何有效答案都必须与除一个索引 k 之外的所有位置的查询子字符串一致，因此必须匹配 k 周围的前缀和后缀。 通过枚举 k，我们覆盖了所有可能的不匹配位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Hasher:
    def __init__(self, s, base=91138233, mod=10**9+7):
        self.mod = mod
        self.base = base
        self.n = len(s)
        self.h = [0] * (self.n + 1)
        self.p = [1] * (self.n + 1)

        for i, c in enumerate(s):
            self.h[i+1] = (self.h[i] * base + (ord(c) - 96)) % mod
            self.p[i+1] = (self.p[i] * base) % mod

    def get(self, l, r):
        return (self.h[r] - self.h[l] * self.p[r - l]) % self.mod

def solve():
    n, q = map(int, input().split())
    s = input().strip()

    hs = Hasher(s)

    # store signatures: (len, split, pref_hash, suf_hash) -> starting index
    mp = {}

    for l in range(n):
        for r in range(l, n):
            L = r - l + 1
            for k in range(L):
                left_hash = hs.get(l, l + k)
                right_hash = hs.get(l + k + 1, r + 1)
                key = (L, k, left_hash, right_hash)
                if key not in mp:
                    mp[key] = l

    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1
        L = r - l + 1

        ans = None

        for k in range(L):
            left_hash = hs.get(l, l + k)
            right_hash = hs.get(l + k + 1, r + 1)
            key = (L, k, left_hash, right_hash)
            if key in mp:
                cand = mp[key]
                if cand != l:
                    ans = (cand + 1, cand + L)
                    break

        if ans is None:
            print(0, 0)
        else:
            print(ans[0], ans[1])

if __name__ == "__main__":
    solve()
```预处理步骤构建一个字典，对子字符串可以拆分为“除了一个位置之外的所有内容”的所有可能方式进行编码。 每个存储的条目代表至少一个与该模式匹配的子字符串。 

每个查询都会重新计算所查询段的这些拆分哈希值，并尝试找到预先存储的匹配项。 唯一的微妙之处是避免返回相同的起始索引，这将对应于相同的子字符串而不是不同的子字符串。 

## 工作示例

 考虑一个小字符串`s = abcaab`和一个查询`[1, 3]`对应子串`abc`。 

我们计算它的候选者：

 | k | 前缀 | 后缀 | 密钥存在 | 候选人 |
 | --- | --- | --- | --- | --- |
 | 0 | “” | “BC”| 也许| 检查 |
 | 1 | “一个”| “c”| 也许| 检查|
 | 2 | “ab”| “” | 也许| 检查 |

 假设我们找到另一个子串`acc`从位置 4 开始，匹配除索引 1 之外的所有位置。然后在 k = 1 时，前缀“a”和后缀“c”匹配，我们返回`[4, 6]`。 

这表明该算法隔离了不匹配位置并准确匹配其余位置。 

现在考虑一个没有有效答案的情况，例如`aaaa`并查询`[1,4]`。 每次拆分都会为所有子字符串生成相同的前缀和后缀组合，但任何匹配的子字符串都是相同的，因此存储的索引等于查询索引。 由于自匹配被拒绝，结果为`0 0`。

 ## 边缘情况

 对于单字符子字符串，每个子字符串在零个位置上都不相同，因此不存在有效答案。 该算法自然地处理了这个问题，因为任何候选者都必须在一个位置上有所不同，这对于长度 1 来说是不可能的。 

对于像“aaaaaaa”这样完全统一的字符串，每次分割都会在所有子字符串中产生相同的前缀和后缀哈希值。 返回的唯一匹配项对应于相同的起始索引，该索引被拒绝，导致正确的输出 0 0。 

对于嵌入混合字符串中的大型统一查询，该算法仍然可以正确区分子字符串，因为前缀后缀签名取决于确切的字符结构，并且任何不匹配位置都必须同时对齐两半，以防止误报。
