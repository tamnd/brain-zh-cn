 ## 解决方案
 ## 问题理解

 输入中的每个项目代表一个配料。 顶部有一个值，并且它还带有指向另一个顶部索引的单个限制。 如果您决定在最终选择中包含顶部 i，则顶部 b_i 不再允许与其一起出现。 

任务是选择一个配料子集，在尊重所有此类限制的同时最大化价值总和。 每个限制在输入中都是单向的，但它仍然充当最终选择集的互斥规则：一旦你采用 i，你就被迫排除 b_i。 

该结构不是任意的成对冲突。 每个索引恰好产生一个禁止伙伴，这意味着每个元素最多有一个传出约束。 这个细节使得问题的表现与一般最大权重独立集不同，在一般最大权重独立集上，图结构在这种规模上会很棘手。 

当 n 达到 100000 时，任何尝试枚举子集或模拟组合选择的解决方案都会立即超出范围。 即使 O(n²) 交互也太大，并且任何涉及重新计算每个子集可行性的事情都被排除。 唯一可行的解​​决方案是那些对项目进行排序和贪婪处理或利用约束图的函数性质的解决方案。 

如果天真地对待某些情况，很容易出错。 一是假设限制是对称的。 如果 i 禁止 j，并不意味着 j 禁止 i，除非在其他地方明确说明。 例如，如果 1 禁止 2 但 2 不禁止任何内容，则单独选择 2 始终有效，选择 1 会强制排除 2，但反之则不然。 

另一个微妙的情况是链接。 如果 1 禁止 2 且 2 禁止 3，则选择 1 会删除 2，但不会自动强制删除 3，除非也选择了 2。 贪婪策略必须避免意外地将其视为传递闭包问题。 

最后一个陷阱是认为这是一个破坏循环的问题。 即使存在1禁止2、2禁止3、3禁止1等循环，约束仍然只从选定的节点向外激活，因此该结构不是标准的无向冲突图。 

## 方法

 直接的强力方法将尝试所有配料子集，检查每个选定的 i 是否不包含其禁止的 b_i，并计算总和。 这是正确的，但需要迭代 2^n 个子集，甚至检查每个子集的成本为 O(n)，导致 O(n·2^n)，一旦 n 超过 25 左右，这就变得不可能。 

关键的观察来自于约束的不对称性和稀疏性。 每个项目仅禁止另一个项目，这意味着选择一个项目只会产生一个直接后果：最多从未来的考虑中删除一个候选项目。 这使得决策在局部具有破坏性，但在全局却很简单。 

一旦物品按价值递减排序，贪婪策略就变得自然了。 处理某个项目时，如果该项目尚未被先前选择的项目删除，则选择该项目始终是安全的，并且只会使另一个项目无效。 由于我们总是首先选择更高价值的物品，因此任何后来的选择都不能通过替换所选的更高价值的物品来追溯性地提高总数。 

该结构相当于维护一组可用项目并重复选择最佳可用项目，其中每次选择最多删除一个附加节点。 因为移除不会重新引入项目，也不会级联，除非通过选择顺序，所以贪婪排序就足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |
 | 贪心按值排序 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题转化为按值排序的项目的过程。

1. 将所有配料按价格降序排列。 这确保我们始终首先考虑最有利可图的剩余选择，这一点至关重要，因为一旦选择了价值较低的商品，就永远无法弥补失去价值较高的商品的损失。 
2. 维护两个布尔数组。 记录一项是否已被选择。 另一个记录某个项目是否已被某些先前选择的项目失效。 
3. 迭代排序列表。 对于每个项目 i，如果它已经失效，则跳过它，因为包含它会违反之前提交的决策。 
4. 如果 i 仍然有效，请选择它并将其值添加到答案中。 
5. 立即使b_i无效，因为选择i会使b_i与当前解决方案不兼容。 如果已经选择了 b_i，则在处理 b_i 时会提前阻止此步骤，因此可以保持一致性。 
6. 继续，直到处理完所有项目。 

该排序可确保每当两个项目之间存在冲突时，首先考虑价值较高的项目。 根据方向，较低的值将被跳过或删除。 

### 为什么它有效

 不变的是，在处理完所有值大于某个阈值的项目后，当前选择的集合在仅限于这些高值项目的所有子集中是最优的。 处理过程中删除的任何项目都会被删除，因为更高价值的项目明确禁止它。 用删除的项目替换价值较高的项目永远不会增加总和，因为删除的项目稍后会按排序顺序严格处理。 这保证了每个决策都是局部最优且全局一致的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = [0] * (n + 1)
    b = [0] * (n + 1)

    items = []
    for i in range(1, n + 1):
        ai, bi = map(int, input().split())
        a[i] = ai
        b[i] = bi
        items.append((ai, i))

    items.sort(reverse=True)

    taken = [False] * (n + 1)
    banned = [False] * (n + 1)

    ans = 0

    for _, i in items:
        if banned[i]:
            continue
        ans += a[i]
        taken[i] = True
        bi = b[i]
        banned[bi] = True

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先读取所有项目并按值对它们进行排序，以便始终按照收益降序做出决策。 这`banned`数组是关键结构：它记录了无法再获取的项目，因为之前选择的项目已经排除了它们。 

这`taken`在这个特定的贪婪中，数组对于正确性来说并不是严格必要的，但它使逻辑变得明确，并有助于推断如果我们稍后尝试选择一个禁止的项目是否会发生冲突。 关键操作是标记`b_i`选择时禁止`i`，这确保不会形成无效配对。

 ## 边缘情况

 具有一种顶部的最小输入确认该算法正确处理琐碎的选择。 由于没有其他物品，因此不会禁止任何物品，并且始终包含其价值。 

像 1 禁止 2、2 禁止 3、3 禁止 1 这样的循环依赖不会破坏算法，因为只有在选择节点时才会触发禁止。 如果首先选择最高值的节点，则它只需删除一个邻居并且不需要全局循环推理。 

低值节点禁止高值节点的情况自然通过排序来处理。 首先处理高价值节点，如果该节点已被禁止，则稍后尝试包含低价值节点的任何尝试都将被跳过，从而防止任何不正确的交换或覆盖早期的最佳选择。
