 ## 解决方案
 ## 问题理解

 我们得到了一组大小为`2n`，我们被告知这些数字最初来自`n`平面上的点，其中每个点恰好贡献两个整数：其 x 坐标和 y 坐标。 问题是配对完全丢失了，因此我们不再知道哪两个数字属于同一点。 我们只知道存在一些数字配对`n`点，这些点被轴对齐的矩形包围。 

任务是在所有可能的有效配对上重建可以包含所有可能的矩形的最小可能区域`n`重建点。 矩形仅由所选点中的最小和最大 x 和 y 确定，因此问题简化为分配`2n`数入`n`配对和最小化`(max x - min x) * (max y - min y)`。 

约束条件允许`n`最多`100000`，这意味着我们正在处理最多`200000`整数。 任何尝试显式枚举配对甚至推理子集的解决方案都将失败，因为配对的数量呈超指数增长。 甚至`O(n^2)`在最坏的情况下已经太大了，所以我们基本上被限制为`O(n log n)`或更好。 

当许多值相同或极值可以分组到一个坐标轴时，就会出现微妙的边缘情况。 例如，如果所有数字都相同，则矩形会折叠成一个点，并且答案为零。 另一种情况是，最佳配置以最小化一个轴上的跨度的方式配对极端，这从值的原始排序中并不明显。 

## 方法

 一个天真的想法是尝试所有的分割方法`2n`将数字分为两组大小`n`，将一组解释为 x 坐标，另一组解释为 y 坐标。 对于每个分割，我们计算边界矩形并跟踪其面积。 正确性很简单，因为每个有效的分配都恰好对应于一个这样的分割。 然而，分裂的数量是`C(2n, n)`，这对于`n = 100000`。 这使得该方法完全不可行。 

关键的观察结果是，只有极值对矩形才重要。 一旦我们决定哪些数字成为 x 坐标，宽度仅取决于该子集的最小值和最大值，y 坐标也是如此。 这表明最佳解决方案必须来自排序值的非常结构化的分配。 

对数组进行排序后，我们尝试一个建设性的策略：选择`n`元素充当 x 坐标，其余元素`n`作为 y 坐标。 关键的见解是，在最佳解决方案中，这些选择对应于排序顺序中的连续分割。 直观上，如果我们任意混合 x 和 y 之间的小值和大值，我们往往会不必要地夸大这两个范围。 通过对值进行分组，我们减少了极端贡献之间的重叠。 

因此，我们对数组进行排序并尝试每个分割点`i`第一个在哪里`i`元素构成一组，其余元素构成另一组。 对于每次分割，我们评估将组分配给 x 和 y 的两种可能性。 答案是最小可达到的面积。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力配对 | O(C(2n, n) · n) | O(C(2n, n) · n) | O(n) | 太慢了 |
 | 排序分区扫描 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 阅读`2n`数字并按非降序对它们进行排序。 排序至关重要，因为它通过将极值对齐在一起来揭示最佳分组的结构。 
2.考虑将排序后的数组分成两组：前缀`[0..i]`和一个后缀`[i+1..2n-1]`。 每个分割定义了 x 和 y 坐标中数字的候选分配。 
3.对于每个分割点`i`如果两个组都非空，则计算每个组的边界范围。 组的宽度为`max - min`，候选区域是两个范围的乘积。 
4. 跟踪所有有效分割的最小面积。 这确保我们考虑将值分成两个坐标维度的所有结构上不同的方法。 
5. 返回获得的最小值。 

### 为什么它有效

 排序后，任何最佳分配都可以转换为分配给 x 坐标和 y 坐标的值形成连续线段的分配，而不会增加生成的矩形面积。 如果分配给 x 的较小值与分配给 y 的较大值配对，而较大的 x 值与较小的 y 值配对，则交换它们不会增加任何范围，而只能改进或保留边界矩形。 重复应用此类交换消除了组之间的交错，迫使最佳解决方案对应于按排序顺序的分割。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    a.sort()
    
    m = 2 * n
    ans = 10**30
    
    for i in range(n - 1, n + 1):
        x_min = a[0]
        x_max = a[i]
        y_min = a[i + 1]
        y_max = a[-1]
        ans = min(ans, (x_max - x_min) * (y_max - y_min))
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先对值进行排序，以便任何分组决策都成为在数组中选择一个切口的问题。 该循环考虑唯一有意义的分割点，其中两个组都有足够的元素来表示`n`坐标在 x 和 y 之间分割。 对于每个分割，我们直接从边界元素计算两半的范围，避免了检查内部结构的任何需要。 

乘法`(x_max - x_min) * (y_max - y_min)`直接对由该分配引起的矩形区域进行建模。 取最小值确保我们选择可实现的最紧凑的矩形。 

必须小心索引，特别是确保两个组都非空，并且在评估后缀时不会访问超出范围的位置。

 ## 边缘情况

 当所有值都相同时，每个分割都会产生零宽度和零高度，因此该算法正确返回零，因为两者`x_max - x_min`和`y_max - y_min`为零。 

什么时候`n = 1`，排序后的数组有两个元素。 唯一有效的分割在每个轴上产生一个点，导致面积为零的简并矩形，这与计算直接匹配。 

当值严格增加时，最佳分割发生在中间附近，此时两组的内部分布都最小。 该算法评估两个中心切割，确保不会遗漏任何不对称分配。
