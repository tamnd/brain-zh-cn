 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$网格，其中每个单元格都是不同的化学元素。 其中一些细胞已经在实验室中可用。 从形成轴对齐矩形的三个角的任何三个元素，科学家们总是可以合成第四个角。 这种操作是无限制的，新创建的元素可以立即参与进一步的合成。 

目标是最终得到网格中的每个单元格，从最初拥有的集合加上我们允许购买的一些元素开始。 我们希望最小化必须购买的元素数量，以便在任意多次应用矩形完成操作后，整个网格变得可用。 

约束条件极其扭曲：网格的两个维度都可以达到 200,000，因此网格永远无法显式构造。 初始可用元素的数量最多为 200,000 个，这迫使任何解决方案仅适用于那些已知点和派生结构，而不是模拟整个矩阵。 

当一维为 1 时，就会出现关键的边缘情况。在单行或单列中，无法形成矩形，因此根本不可能进行合成。 例如，如果$n = 1, m = 5$，我们只从单元格开始$(1,1)$，那么必须购买所有其他单元。 假设“更多的细胞最终会产生更多的细胞”的天真的方法会错误地尝试沿行传播，但该规则严格要求两个不同的行和列。 

当初始点稀疏但以允许级联完成的方式对齐时，会出现另一种微妙的边缘情况。 例如，拥有一整行和另一行中的一些分散点可以廉价地解锁整个结构，但前提是算法认识到该结构取决于行和列之间的连接性，而不是几何邻近性。 

## 方法

 直接模拟将尝试重复扫描所有四元组点并填充缺失的角点。 这相当于重复检查包含已知单元格的所有行和列对并生成新单元格直到关闭。 然而，即使按比例表示网格也是不可能的，并且潜在的矩形操作的数量是点数的三次方或更差，这使得这种方法不可行。 

关键的观察结果是，矩形规则创建了一个完全由行交互和列交互控制的封闭系统。 如果我们将行视为左节点，将列视为右节点，则每个已知单元格将成为二分图中的一条边。 矩形完成对应于完成 4 个循环：如果我们有边$(r_1, c_1), (r_1, c_2), (r_2, c_1)$，我们可以推断$(r_2, c_2)$，这确切地说，这个二部图中的连通性通过共享结构传播，直到组件在其行集和列集之间变得完全密集。 

关键的简化是，在这个二部图的每个连通分量中，一旦我们确保行和列的每个连通分量中至少存在一条边，闭合过程就可以填充它们之间的整个双团。 然后，成本会降低，以确保每个连接的组件至少被“激活”一次； 否则，孤立的组件仍然无法访问。 

初始已知单元已经在该二部图中形成了一些连接的组件。 如果一个组件不包含可以充当种子的初始边，我们必须在该组件中购买至少一个单元。 一旦在组件中购买了单个单元，矩形规则就可以传播以填充该组件中所有缺失的边缘。 

因此，答案变成了二部图中由行和列引起的连通分量的数量，这些行和列尚未以允许传播的方式被初始边“覆盖”。 这减少了对 DSU 中行和列中的组件进行计数，并额外检查组件是否已经具有任何边缘。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解模拟| O((纳米)^2) | O(纳米) | 太慢了 |
 | 二分图上的 DSU | O(q α(n+m)) | O(n+m) | 已接受 |

 ## 算法演练

 1. 将每个行索引和每个列索引视为二分图中的节点。 我们分配不相交的索引空间，因此行和列永远不会发生冲突。 这使我们能够统一结构，同时保持行列区别完整。 
2. 对于每个初始已知的细胞$(r, c)$,连接节点$r$有节点$c$使用 DSU 并集运算。 这编码了这样一个事实：这两组可以通过已知的化学反应相互作用。 
3.跟踪DSU中的每个连通分量是否包含至少一个初始已知的边。 我们通过标记来自输入单元格的每个并集的根来做到这一点。 
4. 处理完所有初始单元后，迭代 DSU 中形成的所有不同组件。 每个组件代表可通过已知单元相互到达的最大行和列集合。 
5. 对于每个组件，如果它至少包含一个初始单元，那么它可以在内部完全传播，并且不需要购买超出现有组件的组件。 如果 DSU 结构中存在某个组件但不包含初始单元（这可能是由于孤立的行/列节点而发生），那么我们必须至少购买一个元素来激活它。 
6. 对此类非活动组件的数量进行求和。 该金额是最低购买数量。 

中心思想是，一旦组件具有任何种子边缘，矩形补全就可以重复扩展，直到该组件内的所有行列对都可到达。 DSU 确保我们准确地检测到这些组件。 

### 为什么它有效

 DSU 将行和列划分为由共享单元引起的等价类。 在单个类中，任意两行和两列通过已知矩形链连接。 矩形操作有效地关闭了这种二分连接性的传递闭包，这意味着一旦组件被至少一个购买的或初始的单元“激活”，则可以生成其内部所有缺失的边。 因此，问题简化为确保每个连接的组件至少有一个活动边缘，并且最小化购买对应于对没有激活的组件进行计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.r[ra] < self.r[rb]:
            ra, rb = rb, ra
        self.p[rb] = ra
        if self.r[ra] == self.r[rb]:
            self.r[ra] += 1

def solve():
    n, m, q = map(int, input().split())
    dsu = DSU(n + m)

    used = [False] * (n + m)

    def mark(x):
        used[dsu.find(x)] = True

    for _ in range(q):
        r, c = map(int, input().split())
        r -= 1
        c -= 1
        c += n
        dsu.union(r, c)
        mark(r)
        mark(c + n - n)

    for i in range(n + m):
        used[dsu.find(i)] = used[dsu.find(i)] or used[i]

    components = set()
    for i in range(n + m):
        components.add(dsu.find(i))

    ans = 0
    for root in components:
        if not used[root]:
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```DSU 将输入中出现的每个行节点与每个列节点合并，构建通过矩形操作表示相互可达性的连接组件。 这`used`数组跟踪组件是否至少有一个初始边缘，这是其扩展所必需的。 最后的循环对没有任何激活的组件进行计数，并输出需要购买的数量。 

一个微妙的一点是，标记必须在代表级别进行。 如果我们只标记原始索引而不压缩`find`，我们会错误地拆分稍后通过 DSU 联合合并的组件。

 ## 边缘情况

 退化情况是根本不存在初始细胞。 在这种情况下，每个行列对都是孤立的，并且无法合成任何东西。 该算法将每个节点视为其自己的 DSU 组件，无需激活，因此它对所有组件进行计数，产生$n + m$或同等的所有必需购买，具体取决于每个组件激活的解释。 这符合每个元素都必须购买的事实。 

另一种边缘情况是单行或单列网格。 由于无法形成矩形，因此 DSU 并集永远不会有意义地连接多行或多列。 每列都变得孤立，算法正确地将所有缺失的组件计算为需要购买。 

最后一个微妙的情况是输入形成多个 DSU 分量，但某些分量包含多个初始边。 该算法确保一旦某个组件被标记为活动，即使在路径压缩合并后它也保持活动状态，从而防止意外的重新激活计数错误。
