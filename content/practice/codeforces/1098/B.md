 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$n \times m$，其中每个单元格包含四个字符之一：A、G、C 或 T。我们需要构造一个具有相同维度的新网格，以满足严格的局部规则：每个 2 x 2 子网格必须包含所有四个不同字符一次。 

这个条件极其强大。 这意味着每当我们采用任何相邻的行对和任何相邻的列对时，所得的四个单元格必须形成 A、G、C、T 的排列。目标不仅仅是构造任何有效的网格，而是选择一个尽可能接近原始网格的网格，最大限度地减少更改单元格的数量。 

约束允许最多$3 \cdot 10^5$细胞总数。 这立即排除了任何尝试显式考虑所有可能网格的方法，甚至尝试通过对多个候选网格进行昂贵的本地检查来优化每个单元的方法。 任何解决方案的细胞数量都必须接近线性。 

一个天真的但自然的错误是尝试逐行构建网格，同时在本地强制执行约束。 问题在于，约束不仅仅局限于行或列，它将每个单元格与三个邻居耦合在一起，并且早期的贪婪选择可能会迫使以后进行昂贵的修正。 

例如，在 2 x 3 网格中：```
A G C
T A G
```贪婪填充可能会确保前两列形成有效的 2x2 块，但稍后对第 3 列的调整可能会破坏早期的一致性，从而强制进行级联更改。 发生这种情况是因为约束创建了一种周期性结构，而不是局部可优化的结构。 

另一个陷阱是在任意填充网格后尝试通过独立检查所有 2 x 2 方格来强制有效性。 该方法会产生有效的检查，但不会指导构建，并且本地修复违规行为并不是独立的，因为更改一个单元会影响最多四个 2 x 2 块。 

## 方法

 关键的观察结果是有效的表是高度结构化和周期性的。 由于每个 2 x 2 块必须包含所有四个字符一次，因此任何有效的配置都必须在网格中重复一致的模式。 

如果我们查看单个有效的 2 x 2 块，它是四个字符的排列。 一旦我们选择了这样的块，网格的其余部分基本上由奇偶校验约束决定：向右或向下移动一步必须以一致的方式转换字符，以便所有相邻的 2 x 2 块保持有效。 

这意味着任何有效的网格都可以从固定的 2 x 2 模板和两个二进制维度生成。 具体来说，我们可以考虑根据$(i \bmod 2, j \bmod 2)$，但有一点不同：有多种方法可以将 A、G、C、T 分配给四个奇偶校验位置。 有$4!$四个字符的排列，但只有某些结构化布局才能在网格上产生一致的平铺。 事实上，事实证明，选择两种独立的排列，一种用于偶数行，一种用于奇数行，并将它们跨列一致地组合起来会产生一个有效的构造族。 

更稳健的思考方式是：任何有效的网格在两个方向上都必须是 2 周期的，这意味着整个网格由重复的 2 × 2 个图块组成。 因此，问题简化为从少量恒定数量的可能的 2 x 2 基数模式中选择最好的一个，并将其扩展到整个网格。 

只有 12 个本质上不同的有效 2 x 2 模式（最多行和列交换），但我们甚至不需要仔细枚举它们。 相反，我们可以修复字符的基本排序，并考虑通过在奇偶校验类之间交换和旋转分配而生成的一小组排列。 

对于每个候选模式，我们可以在 O(nm) 时间内构建完整的网格，并计算有多少单元格与原始网格匹配。 最佳候选人给出最佳答案。 

蛮力的想法是为每一行和每一列独立地尝试将所有字符分配给奇偶校验类，这将呈指数级增长，因为选择在全局范围内相互作用。 有效性迫使全局周期性的观察将搜索空间压缩为恒定数量的模板。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（局部约束求解）| 指数| O(纳米) | 太慢了 |
 | 尝试所有有效的 2x2 模式 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 1. 固定字符集S={A,G,C,T}。 我们将列举将它们排列成 2 × 2 重复模式的候选方法。 这已经足够了，因为任何有效的网格都必须在每个 2 x 2 块之间强制执行一致性，这意味着全局周期性结构。 
2. 生成四个字符的所有排列。 每个排列定义了我们如何将字符分配到 2 x 2 模板中的位置：

 左上、右上、左下、右下。 
3. 对于每个排列，通过在所有行和列上重复此 2 × 2 块来构建候选网格。 具体来说，单元格 (i, j) 通过 (i % 2, j % 2) 选择排列中的四个位置之一来确定。 
4. 将构建的网格与原始网格进行比较，并计算有多少个单元格匹配。 不匹配的数量就是需要更改的数量。 
5. 跟踪最大限度减少不匹配的排列。 这确保我们选择最接近的有效周期结构。 
6. 评估所有排列后，输出最佳构造的网格。 

工作原理：每个有效的网格都必须对行和列的奇偶校验类进行一​​致的分配，这会强制重复 2 x 2 结构。 与固定模板的任何偏差都会破坏至少一个 2 x 2 子网格约束。 因此，最佳解决方案位于这个有限的周期性平铺族中，彻底检查它们可以保证我们找到最好的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    chars = ['A', 'G', 'C', 'T']

    best_grid = None
    best_score = -1

    # try all permutations of 4 characters
    from itertools import permutations

    for perm in permutations(chars):
        # build candidate grid
        cand = []
        score = 0

        for i in range(n):
            row = []
            for j in range(m):
                val = perm[(i % 2) * 2 + (j % 2)]
                row.append(val)
                if val == grid[i][j]:
                    score += 1
            cand.append("".join(row))

        if score > best_score:
            best_score = score
            best_grid = cand

    print("\n".join(best_grid))

if __name__ == "__main__":
    solve()
```该实现迭代四个字符的所有 24 种排列。 对于每个排列，它分配一个固定的 2 x 2 模式，并使用行索引和列索引的奇偶校验将其扩展到整个网格。 

索引映射`(i % 2) * 2 + (j % 2)`按行优先顺序对 2 x 2 块位置进行编码。 这是核心简化：我们不是明确地推理约束，而是在结构上强制执行它们。 

分数计算为匹配单元格的数量，这相当于最小化编辑。

 ## 边缘情况

 关键的边缘情况是输入网格已经完全有效。 在这种情况下，一种排列将与其完全匹配，并且算法必须保留它而不进行修改。 评分机制保证了这一点，因为匹配排列达到的最大分数等于$n \times m$。 

另一个边缘情况是所有字符都相同。 对于输入：```
2 2
AA
AA
```任何有效的输出都必须以固​​定模式交替显示所有四个字母。 该算法评估所有排列并任意选择一个排列，从而生成完全有效的平铺。 正确性来自于这样一个事实：输出中的每个 2 x 2 块都包含所有四个不同的字符，与输入结构无关。 

最后一个微妙的情况是大型网格，其中局部补丁看起来一致，但存在全局奇偶校验冲突。 对所有排列的枚举可确保测试任何一致的全局奇偶校验分配，因此不会有部分配置使算法陷入不一致状态。
