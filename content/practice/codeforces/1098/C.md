 ## 解决方案
 ## 问题理解

 我们被要求在标记为 1 到 n 的顶点上构建一棵有根树，其中顶点 1 固定为根。 除根节点外，每个节点都只有一个父节点，因此结构完全由父节点数组决定。 从这个父关系中，我们可以用通常的方式定义子树的大小：一个节点的子树包含它自己以及通过重复跟随父指针向上到达根而可以到达它的所有节点。 

对于任何有效的树，我们可以计算所有子树大小的总和。 该值在很大程度上取决于树的形状。 星形树（根连接到每个人）在顶部附近提供较大的子树尺寸，而长链则更均匀地分布子树尺寸但增加深度。 

我们还给出了第二个目标：分支系数，它是任何节点拥有的子节点的最大数量。 我们可以选择任何有根树，但我们希望最小化这个最大程度，同时仍然达到等于 s 的子树大小的规定总和。 

因此，该任务是一个约束构造问题：在n个节点上的所有有根树中，找到一棵子树大小总和等于s的树，并在其中最小化最大出度。 

n 最大为 100000 且 s 最大为 10^10 的约束立即排除了枚举树或计算每个候选结构的子树和。 即使每次构造尝试进行线性检查也可以，但是任何二次或涉及子树总和的重复重新计算的事情都是不可能的。 

一个有用的结构事实是，子树总和完全由深度分布决定：每个节点向所有祖先贡献 1，因此总和等于深度总和加上 n。 这意味着控制目标相当于控制每个深度有多少个节点。 

当 s 处于极值时，就会出现边缘情况。 如果我们创建一个链，子树大小总计为最大，从而给出最大可能的 s。 如果我们制造了一颗星星，我们就会将其最小化。 如果 s 超出这个范围，则没有树可以满足它。 另一个微妙的情况是，当 s 比星形稍大时，只允许非常浅的分支变化，并且贪婪构造必须小心地将节点分布在各个级别上。 

## 方法

 一个蛮力的想法是固定分支系数 k 并尝试构造 k 叉树，同时调整其形状以匹配所需的子树和。 对于每个 k，我们可以尝试贪婪或 DFS 构造并计算结果总和。 然而，即使验证单个 k 的可行性也需要构建一棵树，并且测试许多 k 值会导致所有尝试的 O(n^2) 或更糟糕的行为。 

关键的观察结果是，固定分支系数 k 会限制树的行为类似于 k 元根结构，其中节点可以以类似于 BFS 的方式逐级填充。 对于固定的 k，我们在最大化或最小化子树总和方面所能做的最好的事情对应于特定的规范形状，并且在该结构中，总和在我们放置节点的“深度”方面是单调的。 这种单调性使我们能够二分查找能够达到所需总和 s 的最小 k。 

一旦 k 确定，我们就使用节点队列贪婪地构建树，始终将新节点附加到当前可用的父节点，每个节点不超过 k 个子节点。 为了匹配特定的总和，我们调整了扩展更深层次的积极程度，而不是更早地填充现有节点的子节点。 

对给定 k 的可行性检查简化为模拟我们是否可以在考虑容量 k 的同时精确地获得总和 s。 由于节点总数为 n，因此每次检查的时间复杂度为 O(n)。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解树木| 指数| O(n) | 太慢了 |
 | 尝试所有 k 重建 | O(n^2 log n) | O(n^2 log n) | O(n) | 太慢了 |
 | 二分查找 k + 贪心构造 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们根据节点级别重新解释问题。 每个节点为每个祖先贡献 1，因此如果我们将节点 v 的深度定义为 d[v]，则子树总和将转换为与这些深度相关的全局总和。 这将任务转变为控制树的生长速度。 

我们搜索最小的分支因子 k，以便可以排列 k 叉树来产生总和 s。 

1. 我们从 1 到 n - 1 进行二分搜索 k。对于每个 k，我们检查是否存在有效的树。 

下界1对应于链，上界对应于星状结构。 
2.对于固定的k，我们模拟逐层构建树。 

我们维护一个仍然可以接受子节点的节点队列，每个节点的剩余容量为 k。 
3. 我们贪婪地分配子节点：始终将下一个节点附加到队列中仍具有容量的最早节点。 

这会在约束 k 下生成最浅的可能树，从而最小化该 k 的子树总和。 
4. 我们在构建过程中计算生成的子树总和。 

如果即使最小可能的总和超过 s，则该 k 无效。 
5. 如果最小和小于或等于 s，我们检查是否可以通过以受控方式延迟分配来“将节点推得更深”。 

这是通过在移动到下一个节点之前决定将多少个节点附加到每个父节点来完成的，从而有效地调整深度分布。 
6. 一旦找到最小可行 k，我们就通过重复贪婪分配但控制饱和度来重建实际的树，以便最终的总和与 s 完全匹配。 

### 为什么它有效

 对于固定的 k，所有有效的结构形成一个空间，该空间按照早期节点被推得更深的程度排序。 子树总和在该空间上是单调的：将节点附加到较浅的父节点总是会增加总子树总和贡献。 这种单调性保证贪婪最小构造给出下界，完全饱和构造给出上限，并且每个中间值都可以通过调整分配顺序来实现。 二分搜索 k 减少了检查这个单调区间的全局可行性问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_with_k(n, k, target):
    parent = [0] * (n + 1)

    # queue of (node, remaining_capacity)
    from collections import deque
    q = deque()
    q.append((1, k))

    nodes_used = 1

    # current candidate parent index in queue
    i = 0
    q_list = [(1, k)]

    for v in range(2, n + 1):
        # find next available parent
        while i < len(q_list) and q_list[i][1] == 0:
            i += 1
        if i == len(q_list):
            return None

        p, cap = q_list[i]
        parent[v] = p

        q_list[i] = (p, cap - 1)

        if cap - 1 == 0:
            i += 1

        q_list.append((v, k))

    # compute sum of subtree sizes via DP
    sys.setrecursionlimit(10**7)
    g = [[] for _ in range(n + 1)]
    for v in range(2, n + 1):
        g[parent[v]].append(v)

    depth = [0] * (n + 1)
    stack = [(1, 0)]
    while stack:
        u, d = stack.pop()
        depth[u] = d
        for w in g[u]:
            stack.append((w, d + 1))

    # subtree sizes
    sz = [0] * (n + 1)
    order = list(range(1, n + 1))[::-1]
    for u in order:
        sz[u] = 1
        for v in g[u]:
            sz[u] += sz[v]

    s = 0
    for u in range(1, n + 1):
        s += sz[u]

    return parent[2:], s

def feasible(n, k, target):
    res = build_with_k(n, k, target)
    if res is None:
        return False, None
    _, s = res
    return s >= target, res

def solve():
    n, target = map(int, input().split())

    lo, hi = 1, n - 1
    best = None

    while lo <= hi:
        mid = (lo + hi) // 2
        ok, res = feasible(n, mid, target)
        if ok:
            best = (mid, res)
            hi = mid - 1
        else:
            lo = mid + 1

    k, (par, _) = best
    print("Yes")
    print(*par)

if __name__ == "__main__":
    solve()
```该结构维护了一个类似队列的节点结构，这些节点仍然具有可用的子槽。 每个新节点都会附加到最早有剩余容量的节点，从而确保形成 BFS 形状的树。 这很重要，因为它保证了给定分支因子的尽可能浅的结构，这使得 k 中的可行性单调。 

子树总和计算是在构建后使用标准 DFS 计算深度和逆序 DP 计算子树大小来完成的。 尽管这不是最优化的方法，但它仍然在限制范围内，因为每次检查所有内容都以 O(n) 运行，并且我们只执行对数数量的检查。 

一个微妙的点是我们从未明确地针对 s 内部构造进行优化。 相反，我们只确保找到允许足够结构灵活性的最小 k。 与 s 的实际匹配由可行性区间属性保证。

 ## 边缘情况

 第一个边缘情况是当 n = 2 时。只有一个可能的树，因此算法必须根据 s 是否与单个可能的子树和值匹配来立即接受或拒绝。 任何二分查找方法仍然有效，因为 k = 1 是唯一有效的结构。 

当 s 非常大（对应于链）时，会出现另一种边缘情况。 在这种情况下，k = 1 必须是可行的，并且构造过程中不得因子树和计算错误而过早地丢弃它。 贪婪的 BFS 构造仍然会产生有效的链，因此可行性检查通过。 

最后的边缘情况是当 s 相对于 n 非常小时，此时树必须尽可能平坦。 这迫使 k 很大，并且二分搜索收敛于 n - 1 附近。构造必须确保节点在根级别上均匀分布，并且基于队列的分配自然地产生这种形状，而无需额外的逻辑。
