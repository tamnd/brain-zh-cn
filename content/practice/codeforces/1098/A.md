 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中顶点 1 是根。 每个顶点最初都写有一个非负整数值，但这些值现在丢失了。 剩下的是关于沿根路径的前缀和的部分信息。 

对于任意顶点$v$， 定义$s_v$作为路径上值的总和$v$直到根，包括两个端点。 我们还隐式地给出了深度奇偶校验结构，因为深度是由树决定的，并且我们被告知所有$s_v$均匀深度处的顶点值已被删除并替换为 -1。 任务是重建顶点值的任何有效分配$a_v \ge 0$与其余一致$s_v$值，或确定不存在此类分配。 在所有有效分配中，我们必须最小化所有分配的总和$a_v$。 

关键的困难在于约束不是局部的。 一个值$a_v$影响所有$s_u$通过祖先关系在其子树中，因此决策必须沿着树一致地传播。 

约束允许最多$10^5$顶点，这立即排除了任何试图独立地为每个节点重新计算路径总和的方法。 任何重新计算每个节点总和的解决方案在倾斜树中都会变成二次方。 我们需要线性或近线性遍历，通常是具有仔细传播约束的 DFS 或 BFS。 

当约束在交替深度上相互矛盾时，就会出现微妙的失败情况。 例如，如果奇数深度的节点具有由其父约束链强制的隐含负值，则这是不可能的，因为所有$a_v \ge 0$。 另一个棘手的情况是当偶数深度节点丢失时$s_v$，迫使在其子树中可以分配多少“松弛”变得模糊，这直接影响全局最小总和。 

## 方法

 暴力方法会尝试为所有节点分配值并验证约束。 由于每个$s_v$是根路径上的总和，验证单个分配成本$O(n)$，并且尝试所有分配都是指数级的。 即使限制在小范围内也是不可能的，因为值可能高达$10^9$。 

一个更结构化的蛮力想法是独立处理每个根到叶路径并尝试推断$a_v$从连续的差异$s$价值观。 然而，这会失败，因为$s_v$一半节点缺失，因此并不总是可以直接求差。 

关键的观察是$s_v$其行为类似于树上的前缀和。 如果我们离开父母$p$给一个孩子$v$， 然后：$$s_v = s_p + a_v$$只要两个值都已知。 当父值可用时，这会沿着边缘直接重复。 

现在考虑深度奇偶校验。 全部均匀深度$s_v$未知，而所有奇数深度$s_v$是已知的。 这种交替允许我们以受控的方式传播值：每当我们知道父子对，其中父级有一个已知的$s$，我们可以计算出孩子的$a_v$如果它$s_v$是已知的，或者我们可以表达$s_v$如果未知。 

中心思想是从根向下遍历树并保持一致性约束。 对于每个节点，我们维护一个候选节点$s_v$。 如果给出，则它是固定的。 如果缺失，它就成为仅受子项一致性和非负性约束的变量$a_v = s_v - s_{parent}$。 

最小化目标简化了这个结构：我们想要最小化$\sum a_v$。 由于每个$a_v$是$s_v - s_{parent}$，这变成：$$\sum a_v = s_1 + \sum_{v \neq 1} (s_v - s_{parent(v)})$$它沿着路径进行望远镜并且仅取决于所选择的$s_v$价值观。 最小化推动每一个自由$s_v$尽可能低，同时保留约束。 

这将问题简化为 DFS，我们尝试分配尽可能小的有效值$s_v$到未知节点，确保没有负边差异并与固定值一致。 

我们比较方法：

 | 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分配| 指数| O(n) | 太慢了 |
 | 具有约束传播的 DFS | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 从父数组构建树并存储子列表。 这是必要的，因为约束从根传播到叶子。 
2. 存储给定的$s_v$值，将 -1 标记为未知。 这些固定值在遍历过程中充当硬约束。 
3. 在节点1处建立DFS根并分配$s_1$。 如果$s_1 = -1$，它必须为 0，因为任何正值只会增加所有后代并使目标恶化，而无需其他约束。 
4. 在从节点进行 DFS 期间$u$，保持当前有效$s_u$。 该值通过输入或传播来固定。 
5. 对于每个孩子$v$，确定其$s_v$：

- 如果$s_v$给定，它必须满足$s_v \ge s_u$。 否则$a_v = s_v - s_u$将为负数，这是无效的。 
- 如果$s_v$未知，则为其分配与其自身子树的约束一致的最小可能值。 最初这是$s_u$，因为任何较小的值都会迫使负贡献向下。 
6. 计算$a_v = s_v - s_u$。 将其添加到总答案中。 
7. 递归到$v$及其指定的$s_v$。 
8. 如果在任何时候出现矛盾（固定的$s_v < s_u$），构造是不可能的，我们返回-1。 

### 为什么它有效

 该算法沿每个根到节点路径保持单调不变量：$s_v$值永远不会减少到低于父前缀总和。 这确保了所有$a_v = s_v - s_{parent}$保持非负值。 对于具有固定值的节点，立即执行可行性。 对于自由节点，分配最小可行值可以保证以后的修正不会在不违反约束的情况下进一步减少总和。 由于每条边都贡献一个差分项，因此最小化每个局部分配会最小化全局总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n = int(input())
p = [0] * (n + 1)
children = [[] for _ in range(n + 1)]

parents = list(map(int, input().split()))
for i, par in enumerate(parents, start=2):
    p[i] = par
    children[par].append(i)

s = [0] + list(map(int, input().split()))

ans = 0

def dfs(u):
    global ans
    for v in children[u]:
        if s[v] != -1:
            if s[v] < s[u]:
                print(-1)
                sys.exit(0)
        else:
            s[v] = s[u]

        ans += s[v] - s[u]
        dfs(v)

# root handling
if s[1] == -1:
    s[1] = 0

dfs(1)
print(ans)
```该代码使用邻接表构建有根树，然后从根执行 DFS。 数组`s`存储固定值或未知节点的指定值。 对于每条边，对答案的贡献计算为子前缀和父前缀和之间的差，该差直接等于分配的$a_v$。 

一个关键的实现细节是处理根：当$s_1$缺失，将其设置为 0 可以最小化总和，因为没有需要满足的父约束。 另一个微妙之处是立即进行可行性检查`s[v] < s[u]`，这确保没有负数$a_v$出现。 

DFS 更新`s[v]`未知节点与父节点匹配，这是单调性下可能的最小有效分配。 

## 工作示例

 ### 示例 1

 输入：```
5
1 1 1 1
1 -1 -1 -1 -1
```我们从根开始$s_1 = 1$。 所有的孩子最初都是未知的。 

| 步骤| 节点| 家长 | 给定 s | 已分配的 | a_v 贡献 |
 | ---| ---| ---| ---| ---| ---|
 | 1 | 1 | - | 1 | 1 | - |
 | 2 | 2 | 1 | -1 | 1 | 0 |
 | 3 | 3 | 1 | -1 | 1 | 0 |
 | 4 | 4 | 1 | -1 | 1 | 0 |
 | 5 | 5 | 1 | -1 | 1 | 0 |

 所有节点的值都等于根，所以所有$a_v = 0$除了 root 隐式贡献 1 之外。总计为 1。 

这证实了当没有约束强制变化时，折叠所有值可以使总和最小化。 

### 示例 2

 输入：```
4
1 2 2
5 -1 7 -1
```| 步骤| 节点| 家长 | 给定 s | 已分配的 | a_v |
 | ---| ---| ---| ---| ---| ---|
 | 1 | 1 | - | 5 | 5 | - |
 | 2 | 2 | 5 | -1 | 5 | 0 |
 | 3 | 3 | 5 | 7 | 7 | 2 |
 | 4 | 4 | 5 | -1 | 5 | 0 |

 节点 3 强制从 5 跳到 7，贡献 2。所有其他节点保持与其父节点相同，以最小化成本。 

这显示了固定值如何创建传播到最终总和的强制增量。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n) | 每个节点在 DFS 中被访问一次并在 O(1) 时间内处理 |
 | 空间| O(n) | 邻接表和递归栈|

 该解决方案非常适合约束条件，因为内存和遍历都随着顶点数量线性扩展，最多可达$10^5$。

 ## 边缘情况

 第一个边缘情况是当所有$s_v$值缺失。 算法集$s_1 = 0$，然后将 0 传播到所有节点。 每个边缘差异为零，产生总成本 0。任何分配正根值的尝试都会严格增加总和，因此这是最优的。 

另一种情况是沿边的严格递减约束，例如父级具有$s_u = 5$和一个孩子$s_v = 3$。 DFS 立即拒绝了这一点，因为这意味着$a_v = -2$。 该算法在第一次违规时停止，确保没有部分传播破坏答案。 

第三种情况是深链，其中只有最后一个节点具有固定的大值。 该算法将相等性传播到最后一步，其中仅引入一次跳跃，表明成本是本地化的并且不会错误地分布在中间节点上。
