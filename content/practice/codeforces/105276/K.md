 ## 解决方案
 ## 问题理解

 我们有三张矩形纸，每张都有固定的宽度和高度，我们可以将它们放在一个平面上而不旋转它们。 任务是对所有三个进行排列，使它们占据的区域总面积尽可能小。 关键是不禁止重叠，但在测量最终占用面积时不应重复计算任何重叠区域。 换句话说，我们关心的是放置的矩形的并集面积，而不是它们的面积之和。 

由于只有三个矩形且尺寸很小，因此问题不在于大型结构的增量构建或优化。 相反，它是对一小组可能产生最小边界形状的几何配置进行推理。 

这些约束足够严格，任何依赖于大规模搜索或细粒度连续优化的方法都是不必要的。 该结构强烈表明，最佳排列必须来自矩形边缘对齐的一小组规范布局，因为任何最佳解决方案都可以连续转换为至少一侧对齐的解决方案，而无需增加面积。 

一个常见的失败案例来自于假设允许任意重叠总是有帮助的。 例如，如果我们采用 1×5、3×2 和 5×4 的矩形，将所有三个矩形完美地堆叠在一起将产生 20 的面积。然而，正确答案是 21。这表明预期的解释不是任意完全重叠最小化，而是矩形不重叠的放置，从而使包装结构无效； 实际上，我们正在最小化通过将矩形放置在平面上而不重叠而形成的边界排列的面积。 

另一个微妙的问题是假设只有贪婪放置（例如始终将下一个矩形放置在当前边界框附近）就足够了。 这种方法忽略了这样的配置：一个矩形充当其他两个矩形之上或旁边的“桥梁”，减少一个维度，同时增加另一个维度，从而改进产品。 

## 方法

 如果我们忽略结构，我们可以尝试将每个矩形放置在连续二维空间中的任何位置，并直接最小化联合区域。 这成为一个几何优化问题，每个矩形的坐标具有连续变量。 即使我们将自己限制在轴对齐的位置，这仍然留下无限的搜索空间。 离散位置天真地导致组合爆炸，因为每个矩形引入了两个自由度。 

关键的简化来自于这样一个事实：只有三个矩形，任何最佳排列都可以假设为对齐边缘。 直观地说，滑动一个矩形直到其一个边缘接触另一个矩形或边界边界永远不会增加占用的面积，并且通常会减少松弛空间。 这意味着我们只需要考虑通过对齐矩形边形成的布局。 

一旦我们接受边缘对齐，问题就简化为枚举一小组结构模式。 三个矩形的每个有效配置都可以简化为几种规范形状之一：全部在一条线上，全部在一列中，或者分成两行或两列，其中第三个矩形以填充或跨越剩余维度的一部分的方式放置。 

蛮力的想法是尝试在可能的 x 和 y 坐标（从宽度和高度导出）的精细网格上进行所有放置。 如果在所有边组合上离散化，该方法的增长大致类似于 O(W3H3)，但这是不必要的。 最佳布局仅依赖于矩形的排列和少数结构模式的观察结果将解决方案减少到恒定数量的检查。

我们迭代三个矩形的所有排列，并为每个排列评估所有有效的包装模式。 由于只有三个对象，因此这个常数因子枚举非常小并且很容易符合限制。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力连续放置| O(无穷) 或非常大的离散化 | O(1) | O(1) | 太慢了 |
 | 枚举排列和布局 | O(1)（3 的阶乘）| O(1) | O(1) | 已接受 |

 ## 算法演练

 选择排列后，我们将矩形标记为 A、B 和 C。 每个都有固定的宽度和高度。 

1. 考虑三个矩形的所有排列。 这很重要，因为放置哪个矩形会影响组合宽度或高度是否有益。 
2. 对于每个排列，评估所有三个排列在单个水平行中的配置。 结果宽度是所有宽度的总和，而高度是它们高度的最大值。 这反映了垂直堆叠没有任何好处的情况。 
3. 评估所有三个都放置在单个垂直列中的配置。 高度成为所有高度的总和，而宽度成为最大宽度。 这与前面的情况对称。 
4. 评估一个配置，其中两个矩形形成并排放置的底部行，而第三个矩形位于它们上方，跨越底部排列的整个宽度。 宽度变为顶部矩形宽度和底部对的组合宽度的最大值，而高度变为顶部高度和底部对中的最大高度之和。 当一个矩形足够高，可以位于紧凑的底座上方时，这种结构非常有用。 
5. 评估对称配置，其中一个矩形放置在左侧跨越整个高度，另外两个矩形垂直堆叠在右侧。 宽度变为左侧矩形宽度与右侧堆栈中的最大宽度之和，而高度变为左侧矩形的最大高度与右侧矩形高度之和。 
6. 评估剩余的对称布局，其中一个矩形位于顶部，跨越整个宽度，另外两个矩形以垂直排列方式放置在其下方。 

在评估所有排列和所有结构模式后，计算出的最小边界区域就是答案。 

正确性来自于以下事实：三个轴对齐且不重叠的矩形的任何最佳排列都可以通过滑动矩形直到它们接触边界或彼此接触来转换为这些规范形式之一。 这种变换永远不会增加边界维度，因为它只会删除空白空间。 由于三个矩形的每个不重叠结构必须要么在一个方向上形成单链，要么在正交方向上形成 2+1 的分裂，因此这些情况耗尽了所有可能性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from itertools import permutations

rects = [tuple(map(int, input().split())) for _ in range(3)]

def area(w, h):
    return w * h

ans = float('inf')

for (w1, h1), (w2, h2), (w3, h3) in permutations(rects):

    ans = min(ans, (w1 + w2 + w3) * max(h1, h2, h3))

    ans = min(ans, max(w1, w2, w3) * (h1 + h2 + h3))

    w = w1 + w2
    h = max(h1, h2)
    ans = min(ans, max(w, w3) * (h + h3))

    w = max(w1, w2)
    h = h1 + h2
    ans = min(ans, (w + w3) * max(h, h3))

    w = w1 + w3
    h = max(h1, h3)
    ans = min(ans, max(w, w2) * (h + h2))

print(ans)
```该代码系统地测试了矩形排序的每个排列，因为矩形的身份会影响哪个配对会产生最紧凑的中间形状。 对于每个排列，它计算对应于水平堆叠、垂直堆叠和主要两级分解的五种规范布局。 

一个微妙的实现细节是每个配置必须独立地重新计算宽度和高度。 跨排列重复使用中间值是不正确的，因为不同的排列会改变分组在一起的矩形。 另一个重要的一点是，我们总是将边界区域计算为宽度乘以高度，从不尝试显式跟踪联合区域，因为有效包装中不允许重叠。

该轨迹表明，简单的堆叠永远没有竞争力，因为它过度扩展了一个维度，而平衡的拆分配置则减少了宽度和高度的乘积。

 ## 边缘情况

 当所有矩形都是相同的正方形时，就会出现关键边缘情况。 在这种情况下，每种排列都会折叠成对称结构，并且行和列配置都会产生相同的边界区域。 该算法自然地处理这个问题，因为所有排列都评估相同的表达式，因此最小值是稳定的。 

当一个矩形非常平坦（例如 1×100），而其他矩形更接近正方形时，会出现另一种边缘情况。 一个幼稚的策略可能总是水平放置长矩形，但最佳配置可能会根据其他两个形状将其用作垂直边界。 基于排列的评估确保两个方向都得到隐式测试。 

最后一种边缘情况是一个矩形在宽度和高度上都占主导地位。 在这种情况下，所有最佳配置都会减少到在其周围放置另外两个配置，而不增加最大尺寸。 该算法仍然评估所有布局，并且主导矩形自动确定最小计算中的边界约束。
