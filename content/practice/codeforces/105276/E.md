 ## 解决方案
 ## 问题理解

 我们给出了几类算法，其中每一类都包含一定数量的不同算法。 在接下来的$K$几天后，鲍勃将每天只学习一个类别，并且在那一天他必须准确地学习$M$所选类别的算法。 

单个类别可以在多天内重复使用，但每一天都是独立的：Bob 只是选择$M$与当天该类别不同的​​算法。 他可以完全忽略某些算法，因此不需要穷尽一个类别。 

目标是选择尽可能大的整数$M$这样就可以为每个类别分配一个类别$K$天，并且每个所选类别至少有$M$当天未使用的可用算法。 

换种说法，对于固定的$M$，一个类别$a_i$最多可以使用算法$\left\lfloor \frac{a_i}{M} \right\rfloor$天。 我们需要检查所有类别的总可用天数是否至少为$K$。 我们想要最大的$M$满足这个条件。 

约束允许最多$10^5$类别和总算法计数高达$10^5$。 这强烈表明任何具有二次行为的解$N$或者$a_i$是不可能的。 甚至$O(N \log N)$很好，但是如果重复多次，每次检查比线性或近线性更糟糕的事情都会太慢。 

直接暴力破解$M$从 1 到$\max a_i$，每次检查可行性会太慢，因为每次检查都需要扫描所有类别，导致大约$O(N \cdot \max a_i)$，可以达到$10^{10}$最坏情况下的操作。 

当类别非常不均匀时，就会出现微妙的边缘情况。 例如，如果所有$a_i = 1$和$K = N$，那么只有$M = 1$作品。 如果假设每个类别只使用一次，那么在不考虑跨天重复使用的情况下分配类别的天真贪婪想法就会失败。 

另一种失败模式是尝试贪婪地分配日期而不进行建模重用。 例如，思考“首先选择最大的类别并减去$M$“如果不仔细计算每个类别可以支持多少个完整组，就会导致错误的耗尽建模。

 ## 方法

 蛮力的想法很简单：尝试每一个可能的值$M$，并为每个值计算我们可以支持多少天。 对于固定的$M$, 各类别$i$贡献$\lfloor a_i / M \rfloor$天。 我们对所有类别进行求和，并检查总和是否至少为$K$。 这是正确的，因为每组$M$算法在一个类别内是独立且不相交的。 

问题是性能。 对于每位候选人$M$，我们扫描所有$N$类别。 自从$M$可以达到$10^5$，总功变为$O(N \cdot \max a_i)$，太大了。 

关键的观察是可行性是单调的$M$。 如果一个值$M$是可行的，那么任何较小的值也是可行的，因为减少$M$只增加或保留$\lfloor a_i / M \rfloor$对于每个类别。 这种单调结构允许我们使用二分搜索$M$。 

每个可行性检查仍然存在$O(N)$，但我们只执行$O(\log \max a_i)$检查，使解决方案高效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N \cdot \max a_i)$|$O(1)$| 太慢了 |
 | 二分查找+贪心检查 |$O(N \log \max a_i)$|$O(1)$| 已接受 |

 ## 算法演练

 我们利用这样一个事实，即较大的值$M$减少每个类别的可用天数。 

1. 定义一个函数`can(M)`计算如果每天使用的话可以形成多少天$M$来自单一类别的算法。 对于每个类别$a_i$，我们添加$a_i // M$到一个运行总数。 这计算了有多少个完整的组大小$M$存在。 
2. 如果所有类别的组总数至少为$K$， 然后$M$是可行的。 否则，这是不可行的。 这直接符合每个的要求$K$天必须分配给一个有效的组。 
3. 执行二分查找$M$范围从 1 到$\max(a_i)$。 下限为 1，因为每天至少一种算法始终是最小有意义的选择。 
4. 对于中点$mid$, 计算`can(mid)`。 如果可行，我们通过向上移动下限来尝试更大的值。 否则，我们减小上限。 
5. 二分查找完成后，下界将指向最大可行值$M$。 

### 为什么它有效

 功能`can(M)`是单调非增的$M$。 增加$M$只能减少或保留每个类别中完整组的数量，而不能增加它。 这保证了可行性空间是整数上的连续前缀$M$，因此二分查找可以正确找到可行值和不可行值之间的边界。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(M, arr, K):
    total = 0
    for a in arr:
        total += a // M
        if total >= K:
            return True
    return False

def solve():
    N, K = map(int, input().split())
    arr = list(map(int, input().split()))
    
    lo, hi = 1, max(arr)
    ans = 1
    
    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid, arr, K):
            ans = mid
            lo = mid + 1
        else:
            hi = mid - 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```实现的核心是`can`函数，它将每个类别转换为它在给定的情况下可以支持多少个完整“天”的计数$M$。 提前退出时`total >= K`一旦可行性得到确认，就可以防止不必要的工作。 

二分查找保持以下所有值的不变性`lo`是可行的候选者，而上述所有值`hi`是不可行的。 变量`ans`跟踪迄今为止看到的最佳有效中点。

 ## 边缘情况

 单一类别案例，例如`N = 1`测试解是否正确地简化为简单除法。 用于输入`1 3`和`a = [10]`，可行性完全取决于是否`10 // M >= 3`，因此二分查找必须精确收敛到满足此约束的最大除数，从而产生`M = 3`。 

例如，当所有类别都相同时`N = 5`,`K = 5`， 和`a_i = 2`，算法正确地识别出每个类别最多贡献一天，如果`M = 2`，但贡献两天，如果`M = 1`。 二分搜索捕获此步骤更改，而不需要显式的大小写处理。 

在严重倾斜的输入中，例如`a = [100, 1, 1, 1]`，大多数容量来自单一类别。 对于大型`M`，较小的类别贡献为零，只有大类别很重要。 该算法仍然正确地对贡献进行求和，并且不会错误地假设均匀分布。
