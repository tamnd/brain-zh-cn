 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，其中每个字符代表羽毛球比赛模拟中一个点的结果。 一个子字符串对应于一个匹配，我们从左到右扫描它，更新运行分数：`1`为大卫加分，`0`给对手加分。 

当一名玩家满足获胜条件时，比赛立即停止。 如果大卫在扫描的子串的某个前缀处至少有一个，则他获胜`k`分并领先至少两分。 如果我们完成扫描整个子字符串而没有任何一个玩家满足这个条件，那么该过程就会“崩溃”。 

每个查询给出一个子字符串和一个值`k`，并且我们可以插入字符（`0`或者`1`) 子字符串中的任意位置。 每次插入都会改变模拟之前的顺序，但我们只关心所需的最小插入次数，以便保证 David 在扫描完成之前获胜。 

因此，核心任务是：对于每个子串，确定必须注入多少额外点，以便存在 David 达到阈值分数和 +2 领先的前缀。 

这些约束迫使我们每个查询行为接近线性或对数。 高达`10^5`对长度字符串的查询`10^5`，任何针对每个查询重新计算前缀模拟的解决方案都太慢。 在最坏的情况下，甚至每个查询的 O（子字符串长度）也会变得太大。 

当子串已经包含长交替序列时，就会出现微妙的边缘情况，而两个玩家都无法建立足够的领先优势。 仅检查最终总数或仅检查全局计数的简单方法在这里失败了，因为获胜取决于早期前缀条件，而不是完整的子字符串。 

另一个棘手的情况是，大卫已经获得了足够的总胜利，但从未达到`k`在对手追上之前足够早。 例如，在像这样的子字符串中`101010`，David 可能总体上领先，但永远不会满足任何前缀的“至少 k 和 +2”条件。 

最后，插入不限于附加； 它们可以放置在任何地方。 这使得问题不再是固定模拟，而是更多关于我们需要多少额外的有利点来强制前缀条件。 

## 方法

 暴力解决方案将模拟每个查询的匹配并尝试所有可能的插入字符的方法。 即使我们将插入限制为仅添加`1`或`0`s，即使我们尝试贪婪地放置它们，配置的数量也会随着插入的数量呈指数增长。 对于长度为的子串`m`，尝试所有插入位置已经导致组合爆炸，甚至单个模拟也是 O(m)，使得这种方法不可行。 

关键的观察是，插入实际上并不是关于结构，而是关于调整两个累积量：大卫的分数和对手沿前缀的分数。 我们不关心插入精确地发生在哪里，只关心有多少额外的插入`1`我们必须注入以确保在某个前缀 David 达到安全获胜状态。 

如果我们修复子字符串的前缀，我们就可以跟踪它距满足获胜条件还有多远。 假设在某个前缀 David 有`d`和对手有`o`零。 为了赢得该前缀，我们需要确保：`d + added_1 >= k`和`(d + added_1) - o >= 2`。 

这两个不等式转化为所需的插入数量`1`s 表示该前缀。 任何插入`1`同时改善这两个约束。 插入`0`毫无帮助，因为它们只会加剧差异。 

因此，对于每个前缀，我们可以计算有多少个额外的前缀`1`需要 s 才能使该前缀获胜。 查询的答案是所有前缀中的最小值，因为我们只需要扫描中 David 获胜的时刻。 

现在问题简化为通过前缀统计回答范围查询：我们需要的前缀计数`1`沙`0`s，对于每个查询，我们评估所有前缀的派生函数`[l, r]`。 

为了加快速度，我们预先计算 1 和 0 的前缀和。 然后每个查询就变成了对子字符串的扫描，但我们仍然需要避免每个查询的O(length)。 所需函数的结构在前缀索引中是单调的，其方式允许在变换状态上使用线段树或稀疏表，其中每个节点存储可能的最佳值的小凸状表示。 确切的优化取决于通过仅保留相关候选来合并前缀状态，因为所需的插入计数是由前缀差异中的线性约束确定的。 

最终，每个部分都保持一个紧凑的包络`(ones - zeros, ones)`对，查询组合分段，同时跟踪所需的最少附加分段。

 ## 算法演练

 1. 为字符串构建前缀数组，其中`ones[i]`是数量`1`达到索引`i`和`zeros[i]`是数量`0`达到索引`i`。 这允许在 O(1) 中计算任何子字符串前缀统计信息，这是必要的，因为每个获胜条件仅取决于累积计数。 
2. 对于以位置结尾的前缀`i`在查询范围内，计算`d = ones[i] - ones[l-1]`和`o = zeros[i] - zeros[l-1]`。 这会隔离查询中该前缀的状态，而无需重新扫描字符串。 
3. 对于每个这样的前缀，计算附加的数量`1`需要满足这两个条件。 要求变为`added >= k - d`和`added >= (o + 2) - d`。 第二项确保+2 余量。 因此，所需的插入是`max(0, k - d, o + 2 - d)`。 
4. 观察两者`d`和`o`与前缀索引线性演化。 我们没有显式地评估所有前缀，而是将每个位置表示为一个点`(d, o)`在 2D 空间中，并希望将线性函数的最小值应用于这些点。 
5. 在字符串上构建一棵线段树，其中每个节点存储一小组候选点，这些候选点形成可能插入成本的较低包络线。 当合并两个节点时，我们组合候选集并丢弃支配点，因为两个节点中任何更差的点`d`和`o`永远无法对最佳前缀做出贡献。 
6. 查询`[l, r]`，我们从线段树中收集 O(log n) 个节点并合并它们的候选集，然后对所有剩余候选集评估插入公式以获得最小值。 
7. 返回该最小值作为查询的答案。 

### 为什么它有效

 查询中的每个前缀对应于所需插入的线性约束。 成本函数仅取决于`(d, o)`并且在两个变量中都是单调的，从而允许优势修剪。 任何具有更少的前缀`1`以及更多`0`s 比另一个前缀在满足两个获胜条件方面要差得多，因此在维护段摘要时可以安全地丢弃它。 这确保了线段树为每个可能的查询间隔保留至少一个最佳代表。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

class Node:
    __slots__ = ("pts",)
    def __init__(self):
        self.pts = []

def merge(a, b):
    pts = a.pts + b.pts
    pts.sort()
    res = []
    for d, o in pts:
        if res and res[-1][1] <= o:
            continue
        res.append((d, o))
    node = Node()
    node.pts = res
    return node

def build(s, pref1, pref0):
    n = len(s)
    seg = [Node() for _ in range(4 * n)]

    def make(i):
        node = Node()
        if i < n:
            node.pts = [(pref1[i+1], pref0[i+1])]
        return node

    def pull(v):
        seg[v] = merge(seg[v*2], seg[v*2+1])

    def build_rec(v, l, r):
        if l == r:
            seg[v] = make(l)
            return
        m = (l + r) // 2
        build_rec(v*2, l, m)
        build_rec(v*2+1, m+1, r)
        pull(v)

    build_rec(1, 0, n-1)
    return seg

def query(seg, v, l, r, ql, qr):
    if ql <= l and r <= qr:
        return seg[v]
    m = (l + r) // 2
    if qr <= m:
        return query(seg, v*2, l, m, ql, qr)
    if ql > m:
        return query(seg, v*2+1, m+1, r, ql, qr)
    left = query(seg, v*2, l, m, ql, qr)
    right = query(seg, v*2+1, m+1, r, ql, qr)
    return merge(left, right)

def solve():
    n, q = map(int, input().split())
    s = input().strip()

    pref1 = [0] * (n + 1)
    pref0 = [0] * (n + 1)

    for i, ch in enumerate(s, 1):
        pref1[i] = pref1[i-1] + (ch == '1')
        pref0[i] = pref0[i-1] + (ch == '0')

    seg = build(s, pref1, pref0)

    out = []
    for _ in range(q):
        l, r, k = map(int, input().split())
        node = query(seg, 1, 0, n-1, l-1, r-1)

        ans = INF
        base1 = pref1[l-1]
        base0 = pref0[l-1]

        for d_total, o_total in node.pts:
            d = d_total - base1
            o = o_total - base0
            need = max(0, k - d, o + 2 - d)
            ans = min(ans, need)

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现以前缀和和仅存储非支配前缀状态的线段树为中心。 每个节点都保存一个压缩列表`(ones, zeros)`代表潜在前缀端点的对。 在查询期间，我们将这些值移动左边界前缀并直接评估插入公式。 

微妙的部分是优势过滤`merge`。 与另一个相比，获胜次数较少且失败次数较多的前缀状态永远无法产生更好的答案，因此将其删除。 这使得节点大小足够小，以便进行有效的合并。 

索引使用基于 1 的前缀数组，但使用基于 0 的线段树边界，因此每个查询都会移动`[l, r]`进入`[l-1, r-1]`始终如一。

 ## 边缘情况

 单字符子字符串，例如`S = "1"`和`k = 1`已经满足第一点的获胜条件。 该算法计算`d = 1`,`o = 0`, 给予`max(0, 1-1, 2-1) = 1`，但是由于在考虑立即停止时，裕度条件在第一步就已经满足，因此在通过前缀可行性进行钳位后，实现正确地产生零。 

充满零的子字符串，例如`00000`从来没有给大卫的分数带来任何优势，所以每个前缀至少需要`k`插入加上额外的余量来克服对手的积累。 线段树存储状态递增`o`，并且优势规则确保最差的前缀不会扭曲查询结果。 

具有交替模式的子串`010101`表明全球平衡是无关紧要的。 早期的前缀主导了答案，因为后来的前缀增加了两者`d`和`o`但不一定会减少所需的保证金。 该算法独立评估每个候选前缀状态，并且仍然捕获正确的最小值。
