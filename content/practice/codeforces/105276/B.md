 ## 解决方案
 ## 问题理解

 我们有一个固定的单淘汰赛$2^K$玩家。 括号结构是完全预先确定的：玩家按顺序排列，第一轮将相邻索引配对，然后相邻比赛的获胜者在下一轮中面对面，依此类推，直到剩下一个冠军。 这意味着锦标赛是一个完美的二叉树，其叶子是按初始顺序排列的玩家。 

每场比赛都会产生一个获胜者，但我们不知道结果。 相反，我们可以想象与括号结构一致的任何可能的结果集。 对于每场比赛，我们可以选择任何一位参赛者作为获胜者。 我们必须计算的是：对于每个玩家，如果该玩家被迫成为最终冠军，那么锦标赛中所有比赛中可能的“冷门”总数最少是多少。 

当获胜者的实力明显低于失败者的实力时，特别是当$P_{\text{winner}} < P_{\text{loser}} - X$。 由于我们控制结果，因此每场比赛对总数的贡献要么为 0，要么为 1，具体取决于选择特定获胜者是否会产生冷门。 

挑战在于，强迫玩家获胜会限制其通往根的路径上的所有比赛，但也会间接影响所有子树中的选择，因为每个中间比赛结果都决定了谁晋级以及未来的比赛是什么样子。 

约束条件$K \le 18$至多意味着$2^K \le 262144$玩家。 本届赛事有$2^K - 1$比赛，因此任何独立模拟每个玩家比赛结果的解决方案都会太慢。 即使对所有球员和比赛采用二次方法也会超出可行限制几个数量级。 

天真的解释可能会尝试对每个玩家以最佳方式重新计算整个锦标赛，同时迫使该玩家获胜。 这需要重新解决尺寸结构$O(n)$每个玩家，导致$O(n^2)$，这远远超出了限制。 

当一个弱玩家可以通过确保强对手尽早消灭对方来“设计”支架，从而减少后来的干扰成本时，贪婪推理的一个微妙的失败案例就会出现。 例如，在每场比赛中选择局部最优获胜者而不考虑未来的配对可能会导致全局配置次优，因为后面几轮对手的身份取决于之前的选择。 

核心困难在于每场比赛都不是独立的：选择获胜者会改变未来对手的身份，从而改变未来的成本。 

## 方法

 暴力破解的想法很简单：模拟锦标赛树，对于每个玩家，尝试所有可能的方法，通过递归选择比赛获胜者来使他们获胜。 在每场比赛中，我们都会分为两种可能性。 这很快就会呈指数增长，因为每个内部节点都会使可能的锦标赛配置数量加倍，从而导致大约$2^{2^K}$总体上可能的结果。 即使限制为固定的获胜者也没有足够的帮助，因为每个子树仍然有大量分支。 

关键的观察是，锦标赛结构是一个固定的二叉树，因此该问题自然是一个树动态规划问题。 我们没有枚举完整的锦标赛结果，而是计算每个子树和每个可能的“该子树的获胜者”实现该状态所需的最小翻转成本。 

在每个节点（子树），我们维护该子树中玩家的 DP 表：该值是使该玩家成为子树获胜者的最小成本。 为了计算转换，我们将左子节点和右子节点结合起来。 如果一个玩家$i$来自左子树，然后使$i$赢得当前节点，我们必须选择一些获胜者$j$从右子树开始。 成本是最优成本$i$赢左，加上最优成本$j$获胜权，加上最后一场比赛的费用$i$和$j$，它是 0 或 1，具体取决于它是否是冷门。 

唯一剩下的困难是有效地找到最好的$j$对于每个$i$，因为对相反子树中的所有候选者进行简单扫描太慢。 

这是通过按功率对每个子树中的玩家进行排序并维护允许在前缀范围上进行快速最小查询的结构来处理的。 扰动条件仅取决于是否$P_j \le P_i + X$，所以对于一个固定的$i$，右子树中的所有候选人按权力分为两个连续的组：那些不会引起混乱的组和那些会引起混乱的组。 每个组都可以通过 DP 值的最小值范围进行查询。 

这将每个合并步骤减少为每个状态的对数查询，并且总复杂性变得易于管理，因为每个级别都会处理所有玩家一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解锦标赛模拟 | 指数| 指数| 太慢了 |
 | 具有最小范围查询的树DP |$O(n \log^2 n)$|$O(n \log n)$| 已接受 |

 ## 算法演练

 我们将锦标赛视为一棵二叉树，其中每个节点对应于一段玩家。 

### 步骤

 1. 使用固定配对结构自下而上构建锦标赛树。 

每片叶子都包含一名玩家。 内部节点表示两个子段之间的匹配。 
2. 对于每个节点，维护属于该子树的玩家列表及其权力。 该列表代表该子树的所有可能的获胜者。 
3. 定义$dp[v][i]$作为玩家所需的最少翻转次数$i$赢得以节点为根的子树$v$。 
4. 对于叶子节点，初始化$dp[v][i] = 0$，因为单个玩家子树内没有发生任何匹配。 
5. 组合两个子节点时$L$和$R$，分别计算每一边的DP，然后合并。 

对于一个玩家来说$i$在左子树中，我们计算：$$dp[v][i] = dp[L][i] + \min_{j \in R} (dp[R][j] + cost(i, j))$$在哪里$cost(i, j)$是 1 如果$P_i < P_j - X$，否则为 0。 

相同的对称计算适用于$i$位于右子树中。 
6. 计算所有最小值$j$有效地，按权力对正确的子树参与者进行排序。 构建一个支持按功率顺序对 DP 值进行最少查询的结构。 
7. 对于每个$i$，计算阈值$T = P_i + X$。 分裂候选人$j$在右子树中进入那些$P_j \le T$（无扰乱费用）和那些$P_j > T$（成本加1）。 查询两个范围并取最小值。 
8. 将结果存储在$dp[v]$对于子树中的所有玩家，然后向上进行，直到处理完根。 
9. 每个玩家的答案是$dp[\text{root}][i]$。 

### 为什么它有效

 DP 状态准确地捕获了决定最佳结果所需的信息：子树获胜者的身份完全决定了所有未来的比赛。 任何有效的全球锦标赛配置都可以分解为子树获胜者的独立选择以及每个内部节点的一场决赛。 由于每个子树 DP 以最小的成本枚举所有可能的获胜者，因此将它们组合起来可以保持最优性。 按功率划分的范围确保了匹配成本中的唯一依赖项在本地处理，而不需要显式枚举所有对手选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    K, X = map(int, input().split())
    P = list(map(int, input().split()))
    n = len(P)

    # Each node will store:
    # players, dp values aligned with players, and sorted indices by power

    class Node:
        __slots__ = ("ids", "dp", "order")
        def __init__(self, ids):
            self.ids = ids
            self.dp = {i: 0 for i in ids}
            self.order = sorted(ids, key=lambda i: P[i])

    def merge(L, R):
        res_ids = L.ids + R.ids
        res = Node(res_ids)

        # Build sorted lists for R by power for range queries on dp
        r_sorted = sorted(R.ids, key=lambda i: P[i])
        r_ps = [P[i] for i in r_sorted]
        r_dp = [R.dp[i] for i in r_sorted]

        # prefix/suffix minima over dp
        pref = [0] * len(r_sorted)
        suff = [0] * len(r_sorted)

        inf = 10**18
        for i in range(len(r_sorted)):
            pref[i] = r_dp[i] if i == 0 else min(pref[i-1], r_dp[i])
        for i in reversed(range(len(r_sorted))):
            suff[i] = r_dp[i] if i == len(r_sorted)-1 else min(suff[i+1], r_dp[i])

        def query(T):
            # min dp[j] where P[j] <= T
            lo, hi = 0, len(r_sorted)-1
            ans1 = inf
            while lo <= hi:
                mid = (lo + hi) // 2
                if r_ps[mid] <= T:
                    ans1 = pref[mid]
                    lo = mid + 1
                else:
                    hi = mid - 1

            # min dp[j] where P[j] > T
            ans2 = inf
            lo, hi = 0, len(r_sorted)-1
            while lo <= hi:
                mid = (lo + hi) // 2
                if r_ps[mid] > T:
                    ans2 = min(ans2, suff[mid])
                    hi = mid - 1
                else:
                    lo = mid + 1

            return ans1, ans2

        for i in res_ids:
            best = 10**18

            if i in L.ids:
                base = L.dp[i]
                T = P[i] + X
                a0, a1 = query(T)
                best = base + min(a0, a1 + 1)
            else:
                base = R.dp[i]
                T = P[i] + X
                a0, a1 = query(T)
                best = base + min(a0, a1 + 1)

            res.dp[i] = best

        return res

    nodes = [Node([i]) for i in range(n)]

    # build tree
    size = n
    while size > 1:
        nxt = []
        for i in range(0, size, 2):
            nxt.append(merge(nodes[i], nodes[i+1]))
        nodes = nxt
        size //= 2

    root = nodes[0]
    print(" ".join(str(root.dp[i]) for i in range(n)))

if __name__ == "__main__":
    solve()
```该解决方案在锦标赛树上构建自下而上的 DP。 每个合并步骤将括号的两半组合起来，并为组合段中的每个候选获胜者计算从另一半中选择对手获胜者的最小成本。 里面的辅助结构`merge`预先计算按功率排序的对手 DP 值的前缀和后缀最小值，从而能够有效评估翻转阈值分割。 

一个微妙的实现问题是确保仅当所选对手位于相对于对手的高功率区域时才添加最终比赛的成本$P_i + X$。 拆分逻辑强制执行这种分离，以便每个候选者$i$正确评估两种可能性。

 ## 边缘情况

 关键的边缘情况发生在以下情况：$X = 0$，因为每场获胜者实力较低的比赛都会自动成为冷门。 在这种情况下，DP 不会简化，因为即使很小的功率差异也很重要，并且分裂条件会分解为严格的功率比较。 

当所有玩家都具有相同的力量值时，会出现另一种边缘情况。 那么没有匹配可以满足$P_{\text{winner}} < P_{\text{loser}} - X$，因此无论结构如何，每次 dp 转换都应该积累零成本。 DP 仍然表现正确，因为每次分裂的两侧都会产生相同的零成本转换，并且最小传播始终保持零。 

结构上不同的边缘情况是，当一名玩家明显比所有其他玩家强得多时$X$。 在这种情况下，让该玩家赢得比赛应该产生零冷门，并且 DP 通过始终选择他们作为每个子树中的获胜者而不会触发惩罚条件来确保这一点。
