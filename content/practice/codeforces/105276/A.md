 ## 解决方案
 ## 问题理解

 我们正在开发一个网格迷宫，其中每个单元格要么是一堵墙，要么是一个自由空间，并且恰好有一个单元格被标记为起点，一个被标记为出口。 移动规则并不是通常的四向步骤。 相反，玩家的状态取决于他们当前面临的位置和方向。 

对于给定的单元格，如果正前方的单元格（根据当前面向的方向）不是墙，则允许玩家执行两个操作之一。 它们可能会向前移动到该单元格中并保持方向不变，或者它们可能会向前移动并在移动后同时向右旋转 90 度。 每一个这样的动作都算作一步。 任务是计算从给定初始方向的起始单元格开始到达出口单元格所需的最小移动次数，或者确定这是不可能的。 

网格大小可达 800 x 800，这意味着最多 640,000 个单元格。 由于每个单元可以在多个方向状态下被访问，因此有效状态空间乘以四。 这立即表明，任何不小心重新访问状态或尝试指数探索路径的解决方案都将失败。 在扩展状态空间上以线性或近线性时间运行的图遍历是唯一现实的选择。 

一个微妙的困难来自于运动和方向之间的耦合。 仅在单元上的朴素最短路径是不够的，因为从不同方向到达同一单元会导致不同的未来可能性。 另一个陷阱是假设转弯与运动无关，而实际上转弯仅是向前移动的结果。 

打破天真的思维的一个简单的边缘情况是一条笔直的走廊，其中唯一有效的路径需要反复改变方向。 

输入：```
4 5
#####
#S..#
#..E#
#####
R
```单元上的朴素 BFS 可能会得出最短路径纯粹是水平距离的结论，但根据方向约束，如果不跟踪方向，某些转换可能是不可能的。 

正确的答案取决于能否同时对位置和方向进行建模。 

## 方法

 暴力方法会尝试从初始状态开始探索所有可能的移动序列，逐步扩展路径并检查是否到达出口。 由于在每个状态下最多有两种选择，并且网格尺寸很大，因此可能的序列数量随着路径长度呈指数增长。 即使中等迷宫也会发生组合爆炸，因为以不同方向重新访问同一单元会生成冗余子树，除非明确跟踪，否则这些子树无法区分。 

关键的观察结果是，这是隐式有向图上的最短路径问题。 每个状态不仅仅是一个单元，而是由位置和方向组成的一对。 从任何状态来看，转换在结构上都是确定性的：我们看着前面的单元格，要么向前移动，要么向前移动并向右旋转。 这将问题转化为在具有统一边权重的图上进行标准最短路径搜索。 

因为每次移动的成本为 1，所以一旦我们正确扩展状态空间，广度优先搜索就足够了。 唯一的挑战是在尊重墙壁和边界的同时有效地构建过渡。 

我们将网格减少到最多 4NM 个状态，每个状态代表位于面向四个方向之一的单元格 (i, j) 中。 如果前向单元有效，每个状态最多有两次传出转换。 BFS 保证当我们第一次从任何方向到达出口单元时，我们就找到了最小的移动次数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^k) | O(2^k) | O(k) | 太慢了 |
 | （单元、方向）状态上的最优 BFS | O(NM) | O(NM) | 已接受 |

 ## 算法演练

 我们将每个配置视为由行、列和面向方向组成的状态。 

1. 解析网格并找到起始单元格和结束单元格。 还要读取初始方向并将其映射为数字形式。 这建立了所有搜索开始的初始状态。 
2. 以固定循环顺序定义上、右、下、左方向向量。 这使我们能够计算前面的单元格，并通过移动方向索引来计算右转。 
3. 创建一个大小为 N × M × 4 的距离数组，并初始化为一个大值，表示到达每个状态所需的最小移动次数。 这可以防止不必要地重新访问状态并确保 BFS 的正确性。 
4. 使用起始状态和距离零来初始化队列。 BFS 将以递增的移动顺序扩展状态，保证最优性。 
5. 当队列不为空时，弹出当前状态。 如果当前位置是退出单元格，则立即返回其距离，因为 BFS 可确保极小性。 
6. 计算当前状态前面的单元格。 如果它是一堵墙，则无法从该状态进行任何转换，我们将继续。 
7. 如果向前移动有效，则生成两种可能的下一个状态：一种是向前移动并保持方向不变，另一种是向前移动然后向右旋转。 更新距离并将未见过的状态推入队列。 
8. 继续，直到队列耗尽。 如果从未到达出口，则返回 -1。 

关键思想是 BFS 在扩展状态图上运行，其中每个节点都由位置和方向唯一标识。 这确保了未来的转换不会出现歧义。 

### 为什么它有效

该算法保持不变，即每当状态出队时，存储的距离是到达该确切位置和方向所需的最小可能移动次数。 由于所有转换都具有统一的成本，因此 BFS 按距离非递减顺序处理状态。 因为每个有效的移动序列都对应于该状态图中的一条路径，并且每条这样的路径都是按成本递增顺序探索的，所以第一次到达出口单元时，我们已经在所有可能的方向中找到了最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]
    start_dir = input().strip()

    dirs = ['U', 'R', 'D', 'L']
    di = [-1, 0, 1, 0]
    dj = [0, 1, 0, -1]

    dir_map = {d:i for i, d in enumerate(dirs)}

    sr = sc = er = ec = -1

    for i in range(n):
        for j in range(m):
            if grid[i][j] == 'S':
                sr, sc = i, j
            if grid[i][j] == 'E':
                er, ec = i, j

    start_d = dir_map[start_dir]

    INF = 10**18
    dist = [[[INF]*4 for _ in range(m)] for _ in range(n)]

    dq = deque()
    dist[sr][sc][start_d] = 0
    dq.append((sr, sc, start_d))

    while dq:
        r, c, d = dq.popleft()
        if r == er and c == ec:
            print(dist[r][c][d])
            return

        nr = r + di[d]
        nc = c + dj[d]

        if 0 <= nr < n and 0 <= nc < m and grid[nr][nc] != '#':
            # move forward, keep direction
            if dist[nr][nc][d] > dist[r][c][d] + 1:
                dist[nr][nc][d] = dist[r][c][d] + 1
                dq.append((nr, nc, d))

            # move forward and turn right
            nd = (d + 1) % 4
            if dist[nr][nc][nd] > dist[r][c][d] + 1:
                dist[nr][nc][nd] = dist[r][c][d] + 1
                dq.append((nr, nc, nd))

    print(-1)

if __name__ == "__main__":
    solve()
```该实现使用 3D 距离数组显式地对状态空间进行编码。 每个 BFS 步骤最多扩展两个转换，均源自前向单元。 右转的方向更新是通过方向索引上的模运算来处理的。 

一个常见的实施陷阱是忘记必须在所有四个方向上跟踪同一个单元。 如果没有第三维，算法就会错误地合并状态，并且可能会错过有效路径或低估成本。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(NM) | 每个 4NM 状态最多处理一次，每个状态最多有两次转换 |
 | 空间| O(NM) | 距离数组存储所有位置方向状态的值 |

 包含方向后，高达 800 x 800 的网格大小会导致大约 250 万个状态。 由于每个状态都会被处理固定次数，因此该空间上的 BFS 完全在限制范围内。

 ## 边缘情况

 一种重要的情况是出口可到达，但只能从特定方向到达。 该算法可以正确处理此问题，因为它仅在达到完整状态（而不仅仅是坐标）时终止。 

例如：```
4 5
#####
#S..#
#..E#
#####
R
```如果达到 E 需要面对一个不是立即直观的方向，BFS 仍然会探索该单元的所有方向状态。 当（E，方向）第一次出队时，它的距离保证最小，因此即使存在其他方向，它们也不能产生更短的路径。 

另一种情况是走廊，始终可以向前移动，但需要反复转弯以调整未来步骤的方向。 由于只有在向前移动后才可能转弯，因此 BFS 正确地考虑了通过移动序列强制改变方向的成本，确保不会错误地假设捷径。
