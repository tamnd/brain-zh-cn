 ## 解决方案
 ## 问题理解

 我们得到一棵加权树，其中每个节点都带有一个整数值，可以是正数、负数或零。 任务是选择一些节点并将所选节点划分为多个连通的组件。 在每个选定的组件内，所有节点必须仅使用来自同一组件的节点通过边连接，并且不同的组件必须是不相交的。 

一旦选择确定，我们通过计算所有选定节点权重的总和并将其除以形成的连接组件的数量来对其进行评估。 目标是最大化这个比率。 如果多个选择达到相同的最佳比率，则通过优先选择使用更多组件的配置来解决平局。 

问题的结构具有欺骗性，因为分母取决于我们如何分割所选顶点，而不仅仅是我们选择了多少个顶点。 树结构允许通过隐式切割边缘来形成组件，但邻接不会迫使节点进入同一组件，因此我们实际上是在选择任意子集，然后计算它们产生的连接块的数量。 

高达 3·10^5 的约束 n 强制采用线性或近线性解。 任何尝试枚举子集或显式尝试所有可能的组件分解的方法都是立即不可行的，因为子集的数量是指数级的，甚至每个节点的二次转换也会超出时间限制。 

当所有权重均为非正数时，就会出现微妙的边缘情况。 一种天真的贪婪方法可能会尝试完全避免选择节点或减少组件，但问题至少需要一个组件，因此选择单个节点成为唯一有意义的操作。 当最优值为负时，会出现另一种不明显的情况。 在这种情况下，即使总和没有增加，增加分量的数量也可以提高比率，因为分母效应主导着平局决胜行为。 

## 方法

 直接方法将尝试选择节点的子集，计算它们诱导出多少个连接的组件，并评估比率。 这已经变得很困难，因为连接在节点之间不是独立的，并且子集的数量是 2^n。 即使我们固定 k，决定是否可以形成具有最大总和的 k 个分量也会减少到树上的约束划分问题，这通常仍然是指数的。 

关键的见解来自于分离节点和组件的角色。 无论其大小如何，每个分量都为分母贡献了一个单位。 因此，问题不在于直接选择边或切口，而是决定哪些节点允许“启动”组件以及哪些节点应附加到现有节点而不增加组件数量。 

这意味着一种转变：我们不再考虑组件，而是将每个节点分配为组件计数的“根贡献”，或者作为现有组件的一部分，而不增加 k。 如果我们固定平均值的目标值 x，我们可以询问是否存在一个选择，使得总和减去 x 乘以分量数为非负数。 这将问题转化为检查加权优化的可行性，该优化在树上变成线性的。 

对于固定的 x，每个节点贡献一个修改值 a_u - x。 目标是选择一组节点来最大化转换后的目标，其中连通性决定我们是否为每个组件支付 x 的额外惩罚。 树的结构允许贪婪的 DP，我们向上累积收益，在本地决定是否将节点保留在其父结构中，或者剪切它并启动一个新组件。

这导致了一个标准的“树上参数DP”的想法：我们二分搜索最优平均值x，并针对每个x计算我们是否可以通过最优组件分割实现非负调整利润。 检查本身是一个树 DP，我们向上传播剩余部分，确保我们只在有益时创建新组件。

 | ---| ---| ---| ---|
 | 对子集和分区的暴力破解 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | 参数搜索+树DP可行性检查| O(n log V) | O(n) | 已接受 |

 ## 算法演练

 我们将目标重新表述为找到每个组件的最大可能平均值。 我们对这个值 x 进行二分查找。 

对于每个候选 x，我们定义一个变换后的权重 b_u = a_u - x。 如果我们选择一组形成多个连接组件的节点，则转换后的分数将变为所有选定节点的 sum(b_u) 减去 x 乘以隐式计数的组件数量。 可行性问题变成了我们能否在最优组件形成下获得非负分数。 

我们使用植根于任何地方的 DFS 来处理树。 

1. 我们计算一个 DP 值 dp[u]，它表示如果 u 向上连接到其父组件，我们可以从 u 的子树传递到其父组件的最佳可能剩余。 
2. 对于 u 的每个子 v，我们首先计算 dp[v]。 如果 dp[v] 为正，我们保留它并将其合并到 u 中，因为附加它可以在不增加组件数量的情况下提高总剩余。 如果 dp[v] 为非正数，我们将丢弃它，这对应于将该子子树作为单独的组件进行切割或根本不选择它。 
3. 聚合子节点后，我们将 b_u 添加到当前节点。 这表示将节点 u 包含在当前的连接结构中。 
4. 如果结果 dp[u] 变为负数，我们将其重置为零并将其解释为在此处切割，这意味着我们在 u 处开始一个新组件，而不是强制它进入其父级组件。 此次削减相当于增加零部件数量，以避免拖累全球盈余。 
5. 处理完根后，我们检查 dp[root] 是否为非负数。 如果是，则候选 x 是可以实现的。 

二分搜索是在足够宽的范围内执行的，以包括所有可能的平均值，通常在最小和最大节点权重之间。 

为什么它有效源自对 DP 的删减解释。 每个节点要么为正在进行的组件做出贡献，要么当其包含会降低调整后的目标时成为新组件的根。 DP 确保每个负面贡献都被隔离到一个单独的组件中，而不是损害更大的结构，这与分母的最优控制完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
a = list(map(int, input().split()))
g = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append(v)
    g[v].append(u)

def check(x):
    b = [ai - x for ai in a]

    def dfs(u, p):
        s = b[u]
        for v in g[u]:
            if v == p:
                continue
            child = dfs(v, u)
            if child > 0:
                s += child
        return max(0.0, s)

    return dfs(0, -1) >= 0

lo = min(a)
hi = max(a)

for _ in range(60):
    mid = (lo + hi) / 2
    if check(mid):
        lo = mid
    else:
        hi = mid

print(f"{lo:.10f} 1")
```DFS 计算在按当前猜测 x 移动权重后，每个子树可以累积多少正剩余。 关键的实现细节是最大为零，它编码了在继续使用变得有害时削减组件的决定。 这就是防止负子树污染其父树的原因。 

二分搜索细化候选平均值，直到达到足够的精度。 最终答案表示为分母为 1 的分数，因为最佳值是表示为每个分量的比率的实数，并且该结构保证以这种连续形式实现最大平均值。 

## 工作示例

 考虑一棵由三个节点组成的小树，其权重分别为 1、2、3。 

我们测试候选者 x = 2。 

| 节点| b[u] = a[u] - x | b[u] = a[u] - x | 儿童贡献 | 钳位前的 dp[u] | dp[u] 之后 |
 | ---| ---| ---| ---| ---|
 | 3 | 1 | 0 | 1 | 1 |
 | 2 | 0 + 1 = 1 | 1 | 2 | 2 |
 | 1 | -1 + 2 = 1 | 2 | 3 | 3 |

 根返回正值，因此 x = 2 是可行的。 这展示了正链如何向上积累剩余。 

现在考虑以节点 2 为根的星形中的权重 1、-5、1。 

| 节点| b[u] (x = 0) | b[u] (x = 0) | 儿童总和 | dp[u] |
 | ---| ---| ---| ---|
 | 叶子| 1, 1 | 0 | 1, 1 |
 | 根 | -5 + 1 + 1 | 2 | 2 |

 这表明，如果周围结构补偿，即使是强负节点也可以被包含在内，但一般来说，二分搜索将确定包含停止有益的阈值。 

这些痕迹显示了子树贡献如何在有益时向上传播，或在消极时作为单独的组件被吸收。

import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    n = int(input())
    a = list(map(int, input().split()))
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    sys.setrecursionlimit(10**7)

    def check(x):
        b = [ai - x for ai in a]

        def dfs(u, p):
            s = b[u]
            for v in g[u]:
                if v == p:
                    continue
                val = dfs(v, u)
                if val > 0:
                    s += val
            return max(0.0, s)

        return dfs(0, -1) >= 0

    lo, hi = min(a), max(a)
    for _ in range(60):
        mid = (lo + hi) / 2
        if check(mid):
            lo = mid
        else:
            hi = mid

    print(f"{lo:.10f} 1")

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    sys.stdout = out
    solve()
    return out.getvalue().strip()

 ## 边缘情况

 单节点树隔离了可行性检查的行为：DP 简化为将节点权重与当前阈值进行比较，并且对钳位步骤的任何不正确处理都会立即破坏正确性。 

完全负树迫使算法重复丢弃子树，最终结果完全取决于最大的单个节点，如果负值没有被正确截断，就会暴露全局累加逻辑中的错误。
