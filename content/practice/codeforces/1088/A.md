 ## 解决方案
 ## 问题理解

 我们给定一个小整数$x$，我们需要构造两个整数$a$和$b$，两者都限制在 1 和$x$，使得一些算术条件同时成立。 第一的，$b$必须划分$a$， 意义$a$是的精确倍数$b$。 二、产品$a \cdot b$必须超过$x$。 三、商$a / b$仍必须严格小于$x$。 

解释这一点的一个有用方法是考虑选择除数$b$，然后选择一个倍数$a$的，并检查这对是否产生足够大的乘积和足够小的商相对于$x$。 因为$x \le 100$，搜索空间绝对很小，但约束结构使得暴力模式可预测且易于推理。 

约束立即告诉我们$O(x^2)$检查所有对是完全安全的，因为最大工作量约为$10^4$迭代。 然而，真正的目标不是性能，而是认识到构造是否始终存在，或者是否存在失败的异常小值。 

出现微妙的边缘情况时$x = 1$。 在这种情况下，两者$a$和$b$必须等于 1，并且条件崩溃：$a \cdot b = 1$不大于$x$，所以不存在解。 任何假设的幼稚构造$x \ge 2$将错误地输出一对。 

另一种失败模式来自于过于贪婪的选择，例如设置$a = x$和$b = 1$。 这总是满足整除性和商约束，但乘积条件变为$x \cdot 1 > x$，这是错误的，因此该策略系统性地失败了。 

关键是找到一对$a$略大于$x$相对于$b$，同时仍保持$a/b < x$。 这建议使用一个小的$b$和一个稍大的倍数$a$，平衡两个约束。 

## 方法

 直接的暴力策略尝试每一对$(a, b)$和$1 \le a, b \le x$，检查整除性，然后验证两个不等式。 这是简单且始终正确的，因为它详尽地测试了整个搜索空间。 候选人人数最多为$x^2$，所以与$x \le 100$，这大约是一万张支票，可以忽略不计。 

然而，这种方法没有揭示结构。 如果约束更大，蛮力将变成二次方并且立即不可行。 关键的观察是我们实际上并不需要任意对。 我们只需要一对$a$是的倍数$b$，所以我们可以重写$a = k \cdot b$。 条件变为$k \cdot b^2 > x$和$k < x$。 这种重新表述建议选择较小的$b$和小$k$，并搜索略大于的值的因子对$x$。 

自从$x$很小，最简洁的结构是测试对，其中$b$范围从 1 向上，并且$a$被选为最大的倍数$b$不超过$x$，或使用小乘数稍微向上调整。 在实践中，按升序尝试所有对并返回第一个有效的就足够了，因为约束保证了所有对的存在$x > 1$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(x^2)$|$O(1)$| 已接受 |
 | 建设性搜索 |$O(x^2)$最坏的情况|$O(1)$| 已接受 |

 ## 算法演练

 我们依赖于详尽的检查，但当发现有效的对时立即停止。 

1. 迭代所有可能的值$b$从 1 到$x$。 我们选择$b$首先因为它定义了允许的值$a$通过可分性。 
2. 对于每个$b$，迭代所有可能的值$a$从 1 到$x$。 
3. 检查是否$a$可以整除$b$。 这强制了结构性约束$a$必须是的倍数$b$。 
4. 如果整除性成立，计算是否$a \cdot b > x$。 这确保了该对的乘积空间足够大。 
5. 还要检查是否$a / b < x$。 这确保了商严格限制在以下范围内$x$。 
6. 如果两个条件都满足，则立即输出该对并终止程序。 
7. 如果穷尽所有可能性后仍未找到对，则输出-1。 

为什么它有效：显式检查问题空间中的每个有效对。 由于我们不会跳过任何组合，因此如果存在解决方案，我们就不会错过它。 提前终止只会提高效率，而不会改变正确性。 这些约束保证如果存在一个解决方案$x > 1$，在枚举时会遇到。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

x = int(input().strip())

for b in range(1, x + 1):
    for a in range(1, x + 1):
        if a % b == 0:
            if a * b > x and a // b < x:
                print(a, b)
                sys.exit()

print(-1)
```该代码直接实现了前面描述的穷举搜索。 嵌套循环按升序枚举所有候选对$b$和$a$。 可整除性检查确保仅考虑有效的倍数。 商条件使用整数除法以避免浮点错误。 程序在找到有效对后立即退出，这对于防止成功后不必要的计算很重要。

 ## 边缘情况

 结构上唯一重要的边缘情况是$x = 1$。 这里唯一可能的对是$(1, 1)$，但它不符合产品约束，因为$1 \cdot 1 \nleq 1$。 该算法明确地检查这一点：单次迭代$b = 1, a = 1$经测试，不满足不等式，循环以 -1 终止。 

另一个微妙的案例非常小$x$直觉可能表明即使存在解决方案也不存在的值。 例如，在$x = 2$，算法尝试$(a, b) = (2, 1)$并立即拒绝它，因为产品不大于$x$。 详尽的检查保证了正确性，无需案例分析。
