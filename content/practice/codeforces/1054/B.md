 ## 解决方案
 ## 问题理解

 我们得到一个序列，据称该序列是从空数组开始逐步构建的。 在每个步骤中，构建器都可以查看数组中已存在的任何元素子集，计算该子集的 mex，并将该 mex 附加为下一个元素。 

多重集的 mex 是其中未出现的最小非负整数。 因此，该过程非常灵活：在每一步中，您都可以“查询”过去的任何子集并附加缺失值。 

任务不是重建结构本身，而是检测一致性。 我们得到了最终的数组，我们必须决定是否存在可以产生它的任何子集选择序列。 如果是，我们输出-1。 否则，我们必须输出最早的前缀，其中某些步骤在任何有效的构造中都必须是不可能的。 

高达 100000 的约束 n 强制任何解都是线性或接近线性的。 任何试图在许多子集上重新计算 mex 或模拟所有选择的方法都是立即不可行的，因为子集的数量呈指数增长。 即使每一步维护动态结构也必须是 O(log n) 或更好。 

一个关键的微妙之处在于，该操作仅取决于所选子集中存在的值集，而不取决于它们的顺序或超出存在范围的多重性。 这意味着推理从根本上讲是关于每个步骤中哪些值在全球范围内已经可用。 

一些容易被忽略的失败案例：

 如果第一个元素不为 0，则构造立即不可能。 由于初始数组为空，唯一的子集为空，其 mex 为 0，因此第一个元素必须始终为 0。 

当出现一个在给定当前“可用值集”的情况下无法生成的数字时，就会出现另一种棘手的情况。 例如，如果我们看到值 x，但尚未确保所有值 0 到 x−1 都出现在前缀中较早的某个位置，则 x 不能生成为任何子集的 mex。 

最后，一旦我们看到一个数字，它就会影响未来的可行性，因为它可能在子集中变得可用。 因此，我们必须维持不断增长的“可用数量”。 

## 方法

 直接的强力解释将尝试模拟每一步：对于每个位置 m，考虑先前元素的所有子集，计算每个子集的 mex，并检查当前值是否可实现。 这在原则上是正确的，因为它完全反映了定义。 然而，即使对于单个步骤也有 2^(m−1) 个子集，使得这个指数。 当 n = 100000 时，这是完全不可行的。 

关键的观察是我们实际上不需要枚举子集。 子集的 mex 仅取决于该子集中存在从 0 向上的哪些值。 如果一个子集可以产生 mex x，那么该子集必须包含从 0 到 x−1 的每个整数，并且必须缺少 x。 因此，真正的问题变成：在步骤 i 处，是否可能存在先前见过的元素的某个子集，其内容允许 mex = a[i]？ 

这简化为全局前缀的结构条件：我们必须能够确保所有小于 a[i] 的数字都存在于较早元素中的某个位置，否则没有子集可以同时包含它们。 此外，如果 a[i] 大于当前“可构造”的值，则可能会产生矛盾。 

跟踪可行性的一种简洁方法是维护哪些值已经出现在前缀中。 然后我们可以检查当前值是否与 mex 结果一致。 当我们被要求生成一个不是给定可用集合结构的任何子集的 mex 的值时，构造就会失败，这减少了在需要时检查前缀中是否存在所有较小的值以及是否违反排序约束。

最终的解决方案变成单遍维护频率集并隐式跟踪最小缺失值模式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们从左到右扫描数组，同时维护出现的值。 

1.我们初始化一个布尔集或数组`seen`记录到目前为止出现过哪些整数，以及一个指针`mex`从 0 开始。 

该指针表示前缀中尚未出现的最小非负整数。 
2. 对于从 1 到 n 的每个位置 i，我们读取 a[i]。 
3. 如果a[i]大于mex，我们立即得出结论，该构造是不可能的。 

原因是先前元素的任何子集的 mex 都不能超过前缀的全局 mex，因为任何子集只能使用前缀中已经存在的值。 
4. 如果 a[i] 等于 mex，这总是可以实现的：我们可以采用包含所有值 0 到 mex−1（必须存在于前缀中）的子集，并省略 mex 本身，生成 mex 作为结果。 所以我们接受这一步。 
5. 如果 a[i] 小于 mex，那么它也是可以实现的，因为我们可以选择包含所有数字 0 到 a[i]−1 但排除 a[i] 的子集，这是可能的，因为 mex 更大，这意味着所有这些值都存在。 
6. 处理完a[i]后，我们将其标记为seen，并在seen[mex]为true时推进mex。 

我们跟踪第 3 步触发的最早位置。 该指数是第一个肯定会犯的错误。 

### 为什么它有效

 不变的是`mex`始终表示前缀中不存在的最小数字。 任何子集 mex 必须位于 0 到 mex 的范围内。 如果输出中出现大于 mex 的值，则意味着子集生成的 mex 大于前缀允许的值，这与 mex 的定义相矛盾。 相反，任何小于或等于 mex 的值都是可以实现的，因为前缀已经包含所有所需的较小数字，从而允许构造 mex 与目标匹配的子集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    seen = set()
    mex = 0

    for i, x in enumerate(a):
        if x > mex:
            print(i + 1)
            return

        seen.add(x)
        while mex in seen:
            mex += 1

    print(-1)

if __name__ == "__main__":
    solve()
```该代码维护一组已在数组中看到的正在运行的值，并不断更新当前的全局 mex。 键检查发生在插入当前值之前：如果该值超过当前的 mex，则违反了结构界限，即没有子集可以生成大于全局缺失值的 mex。 

更新步骤确保 mex 始终反映前缀状态，这是至关重要的，因为未来的有效性取决于迄今为止引入的值。 

## 工作示例

 ### 示例 1

 输入：```
4
0 1 2 1
```我们追踪`seen`和`mex`：

 | 我| 一个[我] | 以前见过| 墨西哥之前 | 行动| 墨西哥之后|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 0 | {} | 0 | 添加 0 | 1 |
 | 2 | 1 | {0} | 1 | 添加 1 | 2 |
 | 3 | 2 | {0,1} | 2 | 添加 2 | 3 |
 | 4 | 1 | {0,1,2} | 3 | 添加 1 | 3 |

 任何时候值都不会超过 mex，因此序列是一致的。 该算法输出-1，符合构造可行的事实。 

### 示例 2

 输入：```
3
0 2 1
```| 我| 一个[我] | 以前见过| 墨西哥之前 | 行动|
 | --- | --- | --- | --- | --- |

这证实了当结构界限被打破时，第一次违规会立即被检测到。

 ## 边缘情况

 关键的边缘情况是第一个元素非零。 用于输入`1`，第 1 步唯一的子集为空，其 mex 为 0，因此除 0 之外的任何值都立即意味着不可能。 该算法处理这个问题是因为`x > mex`当 mex 仍为 0 时，第一步为真。 

另一种情况是当值跳到当前 mex 之上时，例如`0 5 ...`。 在步骤 2 中，mex 为 1，因此看到 5 会立即触发失败。 这正确地捕捉到了这样一个事实：没有任何子集`{0}`可以产生超过 1 的 mex。 

一个微妙的情况是重复小值，例如`0 0 0 0`。 这里，在第一次插入后，mex 保持为 1，并且所有后续的零都是有效的，因为子集总是可以通过选择空子集或缺少一个 0 来产生 0。该算法正确地永远不会触发失败，因为没有值超过 mex。
