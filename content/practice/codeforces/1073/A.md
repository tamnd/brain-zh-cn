 ## 解决方案
 ## 问题理解

 给定一个由小写字母组成的单个字符串，并要求我们找到该字符串的任何连续片段，使得没有任何单个字符出现超过其长度的一半来主导该片段。 如果存在这样的段，我们可以输出其中的任何一个。 如果不存在，我们必须报告不可能。 

关键对象不是完整的字符串，而是它的子字符串。 对于每个子串，我们检查频率条件：子串是否有长度$L$，那么每个字符最多只能出现$L/2$次。 这相当于说子串不能主要由一个重复的字符组成。 

约束条件$n \le 1000$意味着一个$O(n^3)$对具有频率计数的子字符串进行暴力破解将是边界，但仍然有可能在优化的 Python 中通过，而$O(n^2)$解决方案非常安全。 这强烈表明我们应该尝试推理小型结构化子串，而不是复杂的全局结构。 

一个微妙的一点是答案不需要很长或最大。 这消除了对长度进行优化的任何需要。 任何有效的子字符串就足够了，这通常允许使用构造参数而不是搜索。 

当字符串均匀时会出现边缘情况。 例如，如果$s = "aaaa"$，每个子串也是一致的，因此总是违反多数条件。 在这种情况下，正确的输出是“NO”。 仅检查完整字符串多样性的简单方法在这里会失败，因为即使完整字符串失败，较短的子字符串可能仍然有效，但在这种特定情况下没有任何作用。 

另一个重要的边缘情况是当字符串具有混合字符但仍然在本地占主导地位时。 例如，“aaab”具有有效答案“ab”或“ba”，但“aaa”本身根本没有有效的子字符串。 这凸显出我们并不是在寻求全球多样性，而是在寻求本地平衡。 

## 方法

 一种暴力策略是枚举所有子字符串并通过计算字符频率来检查每个子字符串。 对于每个子串$s[l:r]$，我们计算 26 个计数并验证是否有任何计数超过长度的一半。 这是正确的，因为它直接匹配定义。 

问题在于复杂性。 有$O(n^2)$子串，每次检查的成本$O(n)$如果我们从头开始重新计算计数，则会导致$O(n^3)$。 即使前缀和减少了检查$O(1)$对于每个角色，我们仍然有$O(26n^2)$，这是临界点，但可以接受$n=1000$。 然而，这是不必要的，因为问题的结构允许更简单的观察。 

关键的见解是，如果存在有效的子串，那么我们总是可以找到长度最多为 2 的一个。如果我们取任意两个相邻的不同字符，则长度为 2 的子串已经满足条件，因为每个字符只出现一次并且$1 \le 2/2$仅当某个字符出现多次时才为 false。 因此任何一对不同的相邻字符都是有效的。 

这将整个问题简化为扫描任何索引$i$这样$s[i] \ne s[i+1]$。 如果存在这样的一对，我们输出它。 如果不存在这样的对，则该字符串是统一的，并且根本不存在有效的子字符串。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$|$O(1)$或者$O(n)$| 太慢了 |
 | 最佳 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 从左到右扫描字符串，检查每对相邻的字符。 
2.就位$i$， 比较$s[i]$和$s[i+1]$。 
3. 如果不同，立即输出子串$s[i:i+2]$并终止。 
4. 如果扫描完成后没有发现任何不同的相邻对，则得出结论：所有字符都相同。 
5. 在这种情况下，输出“NO”。 

步骤 3 足够的原因是任意两个不同的字符形成一个子串，其中任何字符都不能出现多次，这自动满足多样性条件。 

### 为什么它有效

 不变量是，如果字符串中的任何位置都存在有效的子字符串，则要么存在相邻不匹配，要么该字符串完全一致。 在统一字符串中，每个子字符串都有相同的字符重复，因此没有子字符串可以满足条件。 在非均匀字符串中，至少一对相邻的一对必须不同，并且该对本身形成有效的解决方案。 因此扫描相邻对既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
s = input().strip()

for i in range(n - 1):
    if s[i] != s[i + 1]:
        print("YES")
        print(s[i:i+2])
        sys.exit(0)

print("NO")
```该解决方案完全依赖于检测不同字符之间的第一次转换。 提前退出可确保我们在找到有效答案后不会进行不必要的扫描。 切片`s[i:i+2]`是安全的，因为 Python 干净地处理边界并且我们只迭代到`n-2`。

 ## 边缘情况

 单字符字符串，例如`"z"`自动无效，因为任何子字符串的长度均为 1，并且唯一字符出现的次数超过一半。 由于没有要检查的相邻对，该算法正确输出“NO”。 

完全统一的字符串，例如`"kkkkk"`行为类似。 扫描从未发现不匹配，并且输出正确地为“NO”。 每个子串都继承相同的不平衡性。 

混合字符串就像`"ba"`以最短的长度展示正例。 第一次比较已经产生不匹配，并且子字符串`"ba"`立即有效，因为两个字符都出现一次并且都不超过 2 的一半。
