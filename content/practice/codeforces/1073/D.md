 ## 解决方案
 ## 问题理解

 我们正在模拟一系列摊位价格的循环行走。 Polycarp 从位置 1 开始，并以固定周期保持顺时针移动。 在每个摊位上，他都会检查自己剩余的钱是否至少相当于该摊位上一颗糖果的价格。 如果是，他只买一颗糖果，并按该价格减少他的钱。 如果没有，他就会跳过购买。 处理完一个摊位后，他总是转移到下一个摊位。 仅当完整通行证不再产生任何购买时，该过程才会停止，实际上，当他在当前位置无法再购买任何糖果时，就会发生这种情况，并继续直到他最终用尽所有可能性。 

任务是计算在模拟结束之前他总共购买了多少糖果。 

限制很大：最多 200,000 个摊位，总金额最多 10^18。 如果每一步重新计算任何昂贵的东西，这会立即排除所有金钱扣除的天真的逐步模拟。 每次访问的直接模拟在每个周期仍然是线性的，但周期数可以大到 T 除以最低价格，在最坏的情况下，它变得太大而无法显式模拟。 

当所有价格均为 1 时，就会出现一个关键的边缘情况。在这种情况下，Polycarp 每一步都会购买一颗糖果，直到 T 达到零，从而恰好产生 T 次操作。 任何逐步模拟的方法在概念上仍然有效，但在需要许多周期的更严格的最坏情况结构中变得太慢。 

另一个有问题的情况是，只有一个摊位价格实惠，而其他摊位总是被跳过。 一个幼稚的实现可能会重复扫描所有 n 个摊位，即使大多数步骤都是浪费检查，从而导致退化解释中不必要的 O(nT) 行为。 

## 方法

 蛮力的想法很简单。 我们模拟波利卡普绕圈移动，检查每个摊位，在可能的情况下减去价格，并增加计数器。 这是正确的，因为它准确地反映了该过程。 然而，每个完整周期的成本为 O(n)，并且在最坏的情况下，Polycarp 可以进行大约 T / min(a_i) 周期。 由于 T 可以是 10^18，所以这是不可能的。 

关键的观察结果是该过程具有很强的循环结构。 所有展位的每一轮都会将预算减少固定金额，该金额等于所有价格的总和。 如果我们知道波利卡普可以承受多少轮完整的回合，我们就可以跳过它们，而不是模拟每一步。 耗尽整轮后，仅剩下部分通道，可以在最后一次线性扫描中进行模拟。 

这将问题简化为计算预算中有多少个完整周期，然后一次性处理剩余的剩余部分。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 最坏情况 | O(T × n) O(1) | O(1) | 太慢了 |
 | 循环跳转+残差扫描| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 计算所有展位一个完整周期的总成本。 这表示如果 Polycarp 完成一轮完整的 n 次访问并尽可能进行购买，则花费多少钱。 
2. 将剩余资金除以周期成本，计算可以完成多少个完整周期。 这直接给出了保证完全重复相同行为的次数。 
3. 将每个周期的购买计数乘以周期数，将整个周期内购买的糖果数量相加。 这将大量重复模拟压缩为一次算术运算。 
4. 通过减去所有完整周期的总成本来减少剩余资金。 此后，剩余预算严格小于一个完整周期成本，这保证了最多发生一次部分遍历。 
5. 模拟按顺序单次通过展位。 在每个摊位，如果当前金额至少等于价格，则减去并增加答案。 仅在完成通过后才停止。 
6. 输出累计总数。 

### 为什么它有效

 关键的不变量是，处理完整周期后，Polycarp 的状态相当于以减少的资金从 1 号展位重新启动，并且早期周期内的中间配置不会有所不同，因为每个周期在结构和成本累积上都是相同的。 由于每个完整周期都会以统一的方式影响资金和计数，因此压缩它们不会失去排序或决策的正确性。 剩余的模拟以一个周期为界，因此它准确地捕获了所有部分行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, T = map(int, input().split())
    a = list(map(int, input().split()))

    total = sum(a)

    full_cycles = T // total
    if full_cycles:
        T -= full_cycles * total

    ans = full_cycles * n

    for x in a:
        if T >= x:
            T -= x
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先使用算术压缩重复的完整循环。 所有摊位价格的总和定义了一次完整遍历的成本。 乘以这种效果可以避免模拟重复的循环。 

剩下的循环是对数组的单次传递。 每一步都会检查负担能力并更新状态。 不需要模索引，因为我们在压缩后最多只模拟一个周期。 

一个微妙的点是在修改剩余预算之前使用整数除法。 这可确保准确计算完整周期并避免浮点错误或重复的减法循环。

 ## 边缘情况

 当n = 1时，该过程变成重复单个值的减法。 用于输入`1 100`有价格`3`，该算法计算周期总和为 3，完整周期为 33，残差为 1，产生 33 次购买，这符合直接推理，因为每个周期只是一个购买机会。 

当所有价格都超过T时，比如`3 5 6 7`，整个周期计数为零，剩余通行证产生零购买量，与任何时候没有一个摊位负担得起的事实相匹配。 

当所有价格相等且相对于 T 较小时，算法简化为将 T 除以总和，然后线性消耗余数，确保即使非常大的 T 也不需要模拟每一步。
