 ## 解决方案
 ## 问题理解

 我们被要求考虑一个范围内的每个整数$[l, r]$并通过数字约束对其进行过滤：我们只保留最多使用的数字$k$采用通常的以 10 为基数表示的不同十进制数字。 在所有这些有效数字中，我们必须计算它们的总和模$998244353$。 

困难不在于评估单个数字，而在于在一个巨大的区间内进行汇总。 自从$r$可以大到$10^{18}$，该区间最多可以包含$10^{18}$值，这会立即排除任何直接迭代范围的方法。 

第二个观察结果来自数字约束。 如果一个数字的数字集大小很小，则该数字是有效的，它将全局范围内的数字的所有数字耦合在一起。 这不是像平价或整除那样的局部属性； 这取决于数字的整个结构。 这使得简单的计数或算术级数技巧无法使用。 

一个天真的方法是枚举中的每个数字$[l, r]$，提取其数字集，并检查其是否最多有大小$k$。 这在最小的非平凡输入大小下已经失败了。 甚至$10^{18}$候选人与$k = 10$导致$10^{18}$数字检查，这是不可行的。 

当数字有前导数字限制时，会出现更微妙的边缘情况。 例如，$1000$仅使用数字$\{0,1\}$且有效期为$k=2$， 尽管$999$仅使用$\{9\}$。 任何错误处理前导零的错误数字 DP 可能会错误地计算数字，例如$000123$就像在不应该引入数字 0 的情况下引入数字 0 一样。 

因此，核心挑战是有效地计算所有数字的总和，直到一个界限$x$，然后减去两个前缀答案：$$\text{answer}(l, r) = F(r) - F(l-1)$$在哪里$F(x)$是有效数字的总和$\le x$。 

## 方法

 蛮力的想法很简单：迭代所有数字直到$x$，检查数字的唯一性，并累加总和。 这是有效的，因为数字提取是$O(\log x)$，因此总复杂度变为$O(x \log x)$。 什么时候$x \approx 10^{18}$，这远远超出了任何可行的极限。 

关键的见解是数字是由数字构成的，并且对数字的约束可以使用数字动态编程自然地处理。 我们不是枚举值，而是逐位构建数字并跟踪到目前为止已使用的数字。 状态空间很小，因为只有 10 个可能的数字，因此数字子集可以用大小的位掩码表示$2^{10} = 1024$。 

这将问题转化为经典的数字 DP，其状态由位置、相对于边界的紧密度以及所使用的数字集组成。 一旦我们确定了一个前缀，所有后缀补全的贡献就可以被聚合而不是枚举。 

微妙但至关重要的改进是，我们不仅计算有效数字，而且对它们求和。 这不仅需要携带完成的计数，还需要携带按位置值加权的数字贡献。 这导致同时维护计数 DP 和总 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(x \log x)$|$O(1)$| 太慢了 |
 | 数字DP |$O(20 \cdot 1024 \cdot 10)$|$O(1024 \cdot 10)$| 已接受 |

 ## 算法演练

 我们定义一个函数$F(x)$计算所有有效数字的总和$[0, x]$。 

1. 转换$x$转换成数字数组，这样我们就可以从最高有效位开始处理它。 这使我们能够构建具有前缀约束的数字。 
2. 在数字数组中的位置上定义 DP 状态。 在每个位置，我们跟踪三件事：我们是否仍然受到前缀的约束$x$，哪些数字已经被使用（位掩码），以及我们是否已经开始形成一个数字（以正确处理前导零）。 
3. 对于每个状态，我们迭代从 0 到 9 的所有可能的下一个数字。如果我们还没有开始数字，选择 0 不会将数字引入掩码； 否则就是这样。 
4. 当我们放置一个数字时，我们更新掩码并检查不同数字的数量是否超过$k$。 如果是这样，我们就放弃这个转换。 
5. 我们传播两个量：完成后缀的方式数量（计数），以及所有这些完成所贡献的总和。 添加数字时$d$在位置$pos$，其贡献为$d \cdot 10^{remaining}$乘以完成次数，加上后缀和的移位贡献。 
6.严格的约束确保我们永远不会超越$x$。 如果我们放置的数字小于绑定数字，我们就会转换到非紧状态； 否则我们会保持紧张。 
7. 最后的DP结果给出$F(x)$，最终的答案是$F(r) - F(l-1)$。 

### 为什么它有效

 每个有效数字都作为数字选择序列构造一次。 DP 按前缀划分号​​码空间，严格约束确保无效前缀不会超出界限。 掩码强制执行至多-$k$-数字在引入数字时准确地处于状态，从而防止过早出现无效配置。 由于贡献是通过加法的线性进行聚合的，因此对 DP 状态求和可以保持正确性，而无需重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve(x, k):
    if x <= 0:
        return 0

    digits = list(map(int, str(x)))
    n = len(digits)

    # dp[pos][tight][mask] = (count, sum)
    from collections import defaultdict

    dp = [[[ (0, 0) for _ in range(1 << 10)] for _ in range(2)] for _ in range(n + 1)]
    dp[0][1][0] = (1, 0)

    pow10 = [1] * (n + 1)
    for i in range(1, n + 1):
        pow10[i] = pow10[i - 1] * 10 % MOD

    for pos in range(n):
        for tight in range(2):
            limit = digits[pos] if tight else 9
            for mask in range(1 << 10):
                cnt, sm = dp[pos][tight][mask]
                if cnt == 0:
                    continue

                for d in range(limit + 1):
                    ntight = tight and (d == limit)

                    nmask = mask
                    if mask != 0 or d != 0:
                        nmask = mask | (1 << d)

                    if bin(nmask).count("1") > k:
                        continue

                    ncnt = cnt
                    nsm = sm

                    # contribution of new digit at position
                    nsm = (nsm * 10 + cnt * d) % MOD

                    dp[pos + 1][ntight][nmask] = (
                        (dp[pos + 1][ntight][nmask][0] + ncnt) % MOD,
                        (dp[pos + 1][ntight][nmask][1] + nsm) % MOD
                    )

    res = 0
    for tight in range(2):
        for mask in range(1 << 10):
            if bin(mask).count("1") <= k:
                res = (res + dp[n][tight][mask][1]) % MOD

    return res

def pref(x, k):
    return solve(x, k)

l, r, k = map(int, input().split())
print((pref(r, k) - pref(l - 1, k)) % MOD)
```该实现遵循数字上的 DP 结构。 这`mask`跟踪出现了哪些数字，而前导零逻辑确保像这样的数字`000123`不要错误地激活数字 0。转换`nsm = nsm * 10 + cnt * d`编码如何放置一个数字来移动所有先前形成的后缀贡献。 

所有最终 DP 状态的最终总和收集了直到界限的所有有效数字。

 ## 边缘情况

 一个关键的边缘情况是在构造过程中带有前导零的数字。 对于像这样的输入$x = 100$，DP 可以构建类似“0 → 0 → 5”的状态，它应该代表数字 5，而不是使用数字 0 的数字。`started`逻辑确保在非零数字出现之前数字 0 不会进入掩码。 

另一个边缘情况是当$k = 10$。 在这种情况下，每个数字都是有效的，并且DP应该减少到标准的数字和DP。 掩码检查变得惰性，结果与间隔内的算术和相匹配，确认 DP 没有过度限制状态。 

最后，当$l = 1$，减法$F(l-1)$必须正确处理$F(0)=0$。 如果没有这个保护，负索引或无效的 DP 重用将会扭曲最终结果。
