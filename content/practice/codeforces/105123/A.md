 ## 解决方案
 ## 问题理解

 一个单元格开始于`c`染色体并分裂成两个子细胞，其中包含`a`和`b`染色体。 仅当每条染色体恰好位于两个子细胞之一时，分裂才被认为是正确的。 这意味着分裂后的染色体总数必须与分裂前完全相同。 

输入由三个染色体计数组成，一个用于每个子细胞，一个用于原始细胞。 任务只是确定两个子计数加起来是否等于原始计数。 如果他们这样做，请打印`"YES"`。 否则，打印`"NO"`。 

约束非常小，因为每个值都在 1 到 100 之间。即使是低效的算法也会立即运行。 恒定时间解决方案就足够了，因为答案取决于单个算术比较。 

错误的主要原因是检查错误的条件。 

考虑输入```
20 20 40
```正确的输出是```
YES
```粗心的实施可能会比较`a == c`或者`b == c`，这会错误地拒绝这个有效的分割。 

另一个容易犯的错误是只检查一个子细胞的染色体是否比原始细胞少。 例如，```
45 93 39
```正确的输出是```
NO
```虽然`45`接近于`39`，除法后的总和为`138`， 不是`39`，所以分割无效。 

## 方法

 思考这个问题的一种强力方法是通过单独计算每个染色体来模拟分裂。 我们可以想象分配每个`c`将染色体添加到两个子细胞之一，然后计算每个子细胞收到的染色体数量。 这正确地模拟了有丝分裂，但它所做的功与染色体的数量成正比。 在给定的限制下，这仍然是可以接受的，但它解决了一个比必要的更困难的问题。 

关键的观察结果是染色体的最终排列并不重要。 决定分裂是否正确的唯一属性是染色体总数的守恒。 如果女儿们包含`a + b`染色体完全，那么分裂在以下情况下是有效的：`a + b == c`。 

这将整个问题简化为一次加法和一次比较。

 ## 算法演练

 1.读取三个整数`a`,`b`， 和`c`。 
2. 计算总和`a + b`。 这是细胞分裂后存在的染色体总数。 
3. 将此总和与`c`。 正确的有丝分裂可以保留染色体总数。 
4. 如果`a + b == c`， 打印`"YES"`。 否则，打印`"NO"`。 

### 为什么它有效

 正确有丝分裂的定义属性是原始细胞的每条染色体恰好出现在两个子细胞之一中。 没有染色体丢失或重复。 因此，分裂后的染色体总数必须等于原始数目。 该算法准确地检查此条件，因此它接受每个有效的分割并拒绝每个无效的分割。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b, c = map(int, input().split())

if a + b == c:
    print("YES")
else:
    print("NO")
```程序首先从输入中读取三个整数。 由于只有一个测试用例，因此不需要循环。 

比较`a + b == c`直接符合正确有丝分裂的定义。 不需要特殊处理，因为所有值都是正整数，并且它们的最大值仅为 100，因此在 Python 中不可能出现整数溢出。 

输出仅包含一个单词，或者`"YES"`或者`"NO"`，匹配所需的格式。

import sys
import io

def solve():

 ## 边缘情况

 一个常见的错误是检查一个子细胞是否具有与原始细胞相同的染色体数量，而不是检查总数。 

对于输入```
20 20 40
```该算法计算`20 + 20 = 40`，将其与`40`，并打印```
YES
```这是正确的，因为染色体计数在两个子细胞中都得到保留。 

另一个错误是假设一个子细胞接近原始计数就足够了。 

对于输入```
45 93 39
```该算法计算`45 + 93 = 138`。 自从`138 != 39`，它打印```
NO
```染色体总数增加，这违反了正确有丝分裂的定义。
