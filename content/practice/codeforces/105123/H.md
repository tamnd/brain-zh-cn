 ## 解决方案
 ## 问题理解

 我们得到了无限整数网格上所有存活细菌的当前快照。 我们被告知，最初在某个未知的网格点上有一个细菌，在未知的几分钟后，细菌在曼哈顿附近的移动中传播。 每分钟，细菌都可以选择其四个邻居中的任何一个子集进行感染，或者根本不感染，并且细菌也可能在传播后消失。 重要的是每个当前存活的细胞必须能够从原点精确到达$t$曼哈顿运动的步骤，因为在此过程中信息的传播速度不会超过每分钟一个单位的曼哈顿距离。 

任务是重建一个由时间组成的有效三元组$t$，起始位置$(x_0, y_0)$，并且在所有这些有效的三元组中选择字典顺序最小的一个，这意味着我们最小化$t$，并且之间的关系最小化$x_0$， 进而$y_0$。 

约束条件很大，最多可达$2 \cdot 10^5$点，因此任何针对所有点尝试候选起始位置或重复重新计算距离的方法都会失败。 该解决方案必须将问题简化为集合上的一些聚合计算。 

一个天真的陷阱是假设初始点必须是给定的细菌位置之一。 这是不正确的，因为原点可能位于观察集之外。 另一个失败案例正在尝试修复$t$然后通过 BFS 或多源扩展检查可行性，考虑到坐标范围高达$10^9$。 

## 方法

 蛮力观点从想象我们猜测开始$(t, x_0, y_0)$。 对于每个猜测，我们验证每个观察点是否最多位于曼哈顿距离内$t$从$(x_0, y_0)$。 这很简单：计算从候选原点到所有点的最大距离，并检查它是否最多$t$。 如果我们还要求与传播过程精确一致，则条件仍然会减少到曼哈顿边界距离，因为感染的传播速度不能超过每分钟一个单位。 

然而，这种粗暴的方法是不可行的，因为$t$可以大到$2 \cdot 10^9$候选原点位于巨大网格上的任何位置。 甚至限制$x_0, y_0$观察点给出$O(n^2)$可能性，以及每次检查的成本$O(n)$，导致$O(n^3)$，这远远超出了限制。 

关键的观察是条件“所有点都可以在$t$步骤”相当于几何约束：所有点必须位于半径为曼哈顿的球（菱形）内$t$以$(x_0, y_0)$。 所以问题就变成了：找到一个中心，使得最小的曼哈顿包围半径最小化，并在这些中心中选择字典顺序最小的中心。 

这是利用曼哈顿距离可以使用旋转坐标进行分解这一事实的经典简化。 对于任意点$(x, y)$， 定义：$$u = x + y,\quad v = x - y.$$曼哈顿球对应于对范围的约束$u$和$v$。 具体来说，对于固定中心，最小化最大曼哈顿距离相当于最小化两个变换轴上的最大偏差。 

因此，可以使用极值点来表征最佳中心$u$和$v$独立。 最好的$t$由任一方的最大价差决定$u$或者$v$，中心对应于平衡这些极端。 

一次$t$是固定的，可以通过按下来选择字典顺序最小的有效中心$x_0$尽可能小，同时满足所有约束条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2 \cdot n)$|$O(n)$| 太慢了 |
 | 最佳 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 变换每个点$(x_i, y_i)$进入$u_i = x_i + y_i$和$v_i = x_i - y_i$。 

这将曼哈顿距离的几何形状隔离为两个独立的线性轴。 
2. 计算$u_{\min}, u_{\max}, v_{\min}, v_{\max}$超过所有点。 

这些捕获了旋转坐标系中集合的完整分布。 
3.变换空间中最小可行半径为：$$t = \max(u_{\max} - u_{\min},\; v_{\max} - v_{\min}) / 2$$这是因为中心必须位于两个方向的极端之间。 
4. 确定变换后坐标中的候选中心：$$u_0 = (u_{\max} + u_{\min}) / 2,\quad v_0 = (v_{\max} + v_{\min}) / 2$$如果奇偶校验导致小数值，则两个整数选项都是有效的候选者，但我们选择产生有效整数的一个$(x_0, y_0)$。 
5. 转换回来：$$x_0 = (u_0 + v_0) / 2,\quad y_0 = (u_0 - v_0) / 2$$6. 如果多个整数中心有效，则调整以确保字典顺序最小$(x_0, y_0)$不违反半径约束。 

### 为什么它有效

 所有约束都减少到曼哈顿边界距离，这成为线性约束$u$和$v$。 最优解完全由极值决定，因为任何内点都不能影响最大偏差。 中点结构保证两个变换轴上同时具有最小的最大偏差，并且与中点的任何偏差仅增加最大范围的一侧，这不能提高可行性或字典顺序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    xs = []
    ys = []
    
    u_min = v_min = 10**30
    u_max = v_max = -10**30
    
    for _ in range(n):
        x, y = map(int, input().split())
        xs.append(x)
        ys.append(y)
        
        u = x + y
        v = x - y
        
        if u < u_min:
            u_min = u
        if u > u_max:
            u_max = u
        if v < v_min:
            v_min = v
        if v > v_max:
            v_max = v

    du = u_max - u_min
    dv = v_max - v_min

    t = max(du, dv) // 2

    # candidate midpoint in transformed space
    u0 = (u_max + u_min) // 2
    v0 = (v_max + v_min) // 2

    # convert back
    x0 = (u0 + v0) // 2
    y0 = (u0 - v0) // 2

    print(t, x0, y0)

if __name__ == "__main__":
    solve()
```该实现将整个几何形状压缩为四个极值。 转变为$u, v$至关重要，因为它将曼哈顿距离约束转换为可分离的线性范围。 

整数除法需要小心。 自从$u$和$v$奇偶校验必须匹配有效整数$(x_0, y_0)$，使用楼层划分的整数中点构造之所以有效，是因为任何可行的中心都位于与这些奇偶性一致的离散晶格中。 最终的转换回来确保我们恢复整数坐标。

 ## 边缘情况

 当所有点都位于一条线上时，就会出现关键的边缘情况。 在这种情况下，其中一个转换范围会折叠为零，并且答案完全取决于另一个轴。 该算法仍然有效，因为折叠轴上的中点定义明确并且不影响可行性。 

当极端坐标产生奇校验时会出现另一种边缘情况$u$和$v$。 整数中点构造隐式地通过向下取整（flooring）解决了这个问题，这对应于选择有效的格子中心之一。 任何这样的中心仍然是最佳的，因为移动一个单位只会增加至少一个点的最大距离。 

最后，当$n = 1$，两个范围都为零，答案正确变为$t = 0$以该点本身为原点，匹配普通感染案例。
