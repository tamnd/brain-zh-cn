 ## 解决方案
 ## 问题理解

 我们给出了一个分层结构，其中每层包含一定数量的节点。 在每对连续层之间，左层中的每个节点都连接到右层中的每个节点。 这意味着如果一层有`x`节点和下一个有`y`节点之间的边数正好是`x × y`。 

任务是计算所有相邻层对的边总数并输出总和。 

输入规模较小，最多 1000 层，每层最多 1000 个节点。 这立即告诉我们，即使 O(n²) 解决方案也是安全的，但问题的结构表明我们不需要任何接近于此的解决方案。 我们只与相邻对交互，因此一次线性传递就足够了。 

这里的一个天真的错误是从构建或枚举图表的角度来思考。 如果有人尝试显式模拟边缘，例如通过迭代两层中的所有节点，他们仍然有效地进行正确的计算，但会产生不必要的概念开销。 另一种不正确的方法是误解结构并尝试全局计算边缘而不是每个相邻对，这会忽略只有连续层才重要。 

边缘情况很简单，但仍然值得进行合理性检查。 如果只有一层，则没有相邻对，因此答案必须为零。 例如，输入`1 / 5`应该产生`0`。 如果图层包含 1，例如`1 1 1 1`，每一对恰好贡献一条边，因此答案变成了转换的数量，即`n - 1`。 

## 方法

 强力解释将明确考虑每对相邻层并通过迭代节点来计算乘积。 适用于两层尺寸`x`和`y`，这意味着循环`x`次和`y`次来计算所有连接。 对于所有相邻对，这与通过嵌套迭代计算的乘积之和成正比，这是不必要的，因为乘法已经直接给出结果。 

效率低下的原因在于将每条边都视为需要单独枚举的东西。 由于两个相邻层之间的每一对可能的节点都贡献了一条边，因此我们不会从模拟连接中获得任何东西。 

关键的观察是，该结构已经为我们提供了每个转换的精确公式：对于每个`i`，贡献很简单`a[i] × a[i+1]`。 不同层对之间不存在依赖关系，因此我们可以在一次传递中累积结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（模拟节点对）| O(Σ a[i]·a[i+1]) | O(1) | O(1) | 太慢了 |
 | 最优（相邻乘法）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取层数`n`和清单`a`表示每层的节点数。 这建立了我们将要遍历的结构。 
2. 初始化累加器`ans = 0`存储边的总数。 这使计算保持增量，因此我们不需要任何额外的存储。 
3. 从索引开始迭代各层`0`到`n - 2`。 每个索引代表两个连续层之间的边界。 
4. 对于每个索引`i`, 计算`a[i] * a[i + 1]`并将其添加到`ans`。 这直接计算两层之间的所有边，因为第一层中的每个节点都连接到第二层中的每个节点。 
5.处理完所有相邻对后，输出`ans`。 

### 为什么它有效

 网络中的每条边仅存在于两个连续层之间，并且每条这样的边都由从这两层中选择的一对节点唯一标识。 对于固定的层对`i`和`i+1`，可能的节点对的数量正是笛卡尔积`a[i] × a[i+1]`。 将所有相邻层对上的这些独立贡献相加，将每条边精确计数一次，层对之间没有重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    ans = 0
    for i in range(n - 1):
        ans += a[i] * a[i + 1]
    
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先要读取网络的结构。 然后循环遍历层之间的每个边界，确保我们只相乘相邻的值。 累加器`ans`被保留为单个整数，因为最大可能值完全在 Python 的整数容量范围内。 

一个微妙的点是我们从不尝试存储中间边缘结构。 这避免了内存开销和不必要的复杂性。 循环边界`n - 1`至关重要，因为每个乘法都需要一个有效的对`(i, i+1)`。

 ## 边缘情况

 对于单层输入，例如`1 / 5`，循环`for i in range(n - 1)`永远不会执行，因为`n - 1 = 0`。 累加器仍然存在`0`，这与没有相邻层对因此没有边缘的事实相匹配。 

对于最小邻接，例如`2 / 3 4`，循环运行一次`i = 0`。 计算公式为`3 × 4 = 12`，并且立即返回。 迭代之间没有隐藏状态或遗留，因此每个边界都是完全独立的，并且可以安全地隔离处理。
