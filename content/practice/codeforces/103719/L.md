 ## 解决方案
 ## 问题理解

 我们被告知公交车队的车轮总数。 车队中的每辆车要么是 4 轮公共汽车，要么是 6 轮公共汽车，我们不知道每种类型有多少辆。 任务是重建所有可能的车队规模，以准确生产给定的车轮总数。 在所有有效配置中，我们必须报告尽可能少的总线数和尽可能大的总线数。 如果四轮和六轮公共汽车的任何组合都不能产生给定的总数，我们输出它是不可能的。 

输入是单个整数 n，可能大到 10^18，因此任何解决方案都必须在常数或对数时间内运行。 任何对可能的总线计数达到 n 的迭代都是立即不可行的。 

一个关键的微妙之处是并非每个 n 都可以表示。 由于两种总线类型都有偶数个车轮，因此根本无法形成任何奇数 n。 例如，n = 7 无解，因为 4 和 6 的和总是偶数。 另一个极端情况是非常小的值。 对于 n = 2，也不存在任何组合，因为最小的公共汽车已经有 4 个轮子。 

## 方法

 一个蛮力的想法是尝试从 1 开始的所有可能的公交车 k 数量，并检查我们是否可以分配一定数量的 4 轮和 6 轮公交车来达到总数 n。 对于每个 k，我们将检查是否存在整数 x 使得 4x + 6(k − x) = n。 这简化为检查 n − 6k 是否可被 −2 整除以及所得 x 是否在范围内。 虽然每次检查的时间都是恒定的，但在最坏的情况下迭代所有 k 直到 n/4 对于 n 高达 10^18 来说太大了。 

如果我们重写方程，结构就会简化。 设 a 为 4 轮公共汽车的数量，b 为 6 轮公共汽车的数量。 那么我们有 4a + 6b = n，可以除以 2 得到 2a + 3b = n/2。 这立即表明 n 必须是偶数，否则无解存在。 

现在修复b。 那么a被唯一确定为(n − 6b) / 4，并且我们只要求它是一个非负整数。 总线的数量为 k = a + b。 代入 a 得到 k = b + (n − 6b)/4 = n/4 − b/2。 该表达式揭示了单调关系：增加 b 会减少 k。 这意味着在有效性约束下，当我们最小化 b 时，总线数量会达到最大，而当我们最大化 b 时，总线数量会达到最小。 

唯一的限制是 a 必须保持非负且整数。 从 4a + 6b = n，我们要求 n − 6b ≥ 0 并且能被 4 整除。由于一切都是偶数，因此整除性简化为 b 上的奇偶条件。 因此，我们可以在很小的范围内确定可行的b值，并直接提取极值k值。

 ## 算法演练

 我们从公共汽车和车轮的方程开始。 每个步骤都会提取结构，直到只剩下恒定时间检查。

1. 检查n是否能被2整除。如果不能，则立即断定无解。 这是因为 4 和 6 都是偶数，所以任何和也必须是偶数。 
2. 定义m = n / 2 以简化系数。 该方程变为 2a + 3b = m，这在不改变可行性的情况下减少了算术规模。 
3. 用另一个变量表示一个变量：a = (m − 3b) / 2。这表明对于任何固定的 b，a 是唯一确定的。 
4. 施加有效性条件：a ≥ 0 且积分。 非负性给出 m − 3b ≥ 0，它从上方限制 b。 完整性要求 m − 3b 是偶数，这相当于 b 与 m 具有相同的奇偶性。 
5. 根据这些约束，确定最小可行 b 和最大可行 b。 由于 k = a + b 随着 b 的增加而减少，因此最小的 b 产生最大数量的总线，最大的 b 产生最小数量的总线。 
6. 计算 b 的两个边界值的 k 并输出它们。 

### 为什么它有效

 核心不变量是每个有效解完全对应于满足两个独立约束的 b 的单个整数值：非负界和奇偶校验条件。 一旦b被固定，a就被强制。 这将二维搜索空间折叠成单个有界算术级数。 由于k是一个随b递减的线性函数，k的极值必定出现在b的极值可行值处，因此仅扫描边界值就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    if n % 2 == 1:
        print(-1)
        return
    
    m = n // 2  # 2a + 3b = m
    
    # b must satisfy:
    # 3b <= m  => b <= m // 3
    # (m - 3b) even => b % 2 == m % 2
    
    # smallest valid b
    if m % 2 == 0:
        b_min = 0
    else:
        b_min = 1
    
    # largest valid b <= m//3 with correct parity
    b_max = m // 3
    if b_max % 2 != m % 2:
        b_max -= 1
    
    if b_max < b_min:
        print(-1)
        return
    
    def buses(b):
        a = (m - 3 * b) // 2
        return a + b
    
    x = buses(b_max)  # minimum buses
    y = buses(b_min)  # maximum buses
    
    print(x, y)

if __name__ == "__main__":
    solve()
```代码首先使用 n 的奇偶校验过滤不可能的情况。 然后将问题转化为简化方程 2a + 3b = m。 b 的约束直接源自非负性和宇称性。 我们选择满足这两个条件的 b 边界值并计算相应的总线数量。 排序是故意的：较大的 b 会减少总线总数，因此 b_max 给出最小答案。 

一个微妙的细节是整数除法和奇偶校验调整必须小心应用； 否则，候选 b 可能满足不等式但违反完整性，产生不正确的 a。

 ## 边缘情况

 对于 n = 1，算法会因奇校验而立即拒绝，因为 4 和 6 的组合无法形成奇数和。 

对于 n = 2，奇偶校验仅在除法后通过，但 m = 1 使所有候选值无效，因为 2a + 3b 不能等于 1。计算出的 b 范围变为空，触发不可能性检查。 

对于 n = 4，边界逻辑产生 b_min = 0 和 b_max = 0，从而使用一条总线生成单个有效配置。 当只有一个解决方案存在时，这证实了正确性。 

对于非常大的偶数 n，例如 10^18，所有计算都保留在整数算术中，并且仅依赖于少数除法和奇偶校验，因此无论大小如何，算法的行为都是相同的。
