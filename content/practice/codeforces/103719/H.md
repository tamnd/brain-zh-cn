 ## 解决方案
 ## 问题理解

 我们被要求生成一个特殊整数的无限有序列表并选择第 n 个。 如果一个数字的十进制表示仅由数字 4 和 7 组成，则该数字被视为特殊数字。这些数字形成无限集合，如 4、7、44、47、74、77、444 等。 

十进制字符串的顺序是字典顺序，而不是数字顺序。 这意味着我们从左侧开始逐位比较，作为较长字符串前缀的较短字符串排在前面。 因此，“4”位于“44”之前，“44”位于“47”之前，“47”位于“7”之前，因为在第一个不同位置 4 < 7。 

任务是输出这个按字典顺序排序的无限集合的第 n 个元素，其中 n 可以大到 10^6。 

该约束立即排除了任何通过增加长度并对数字进行排序来实际生成数字的尝试。 即使生成达到一定长度的所有有效数字并进行排序也是浪费的，因为字典结构已经是隐式的并且比看起来简单得多。 

一个微妙的边缘情况来自于正确理解可变长度字符串的字典顺序。 例如，排序开始为：

 4、44、444、4444，然后是 4447，然后是 447，等等。这种前缀行为是大多数天真的数字思维所忽略的关键。 

如果有人试图将排序解释为数字递增顺序，他们会错误地将 7 放在 44 之前，因为从数字上看 7 < 44，但按字典顺序，“44”位于“7”之前，因为“4”<“7”。 

另一种失败情况是假设等长的二进制枚举，但错误地使用了数字索引。 正确的结构是按字典顺序的完整二叉树遍历。 

## 方法

 关键的观察是每个有效数字只是字母表 {4, 7} 上的一个字符串。 这些字符串的字典顺序与二进制 trie 的顺序完全相同，其中每个节点分支到“4”和“7”，并且我们按预序访问节点，始终首先扩展较小的数字。 

如果我们想象构建一个特里树，根对应于空前缀。 从它我们可以转到“4”和“7”。 从“4”我们可以转到“44”和“47”，依此类推。 字典顺序对应于始终在“7”之前探索“4”，并且始终在其扩展名之前探索前缀。 

一种强力方法是生成所有有效的字符串，直到某个长度 L，存储它们，对它们进行排序，然后对结果进行索引。 问题在于，此类字符串的数量呈指数增长，为 2^L，并且对它们进行排序会增加 L log L 的附加因子，当我们需要多达 10^6 个元素时，即使对于中等 L 的情况，这也是不可行的。 

关键的见解是我们不需要生成或排序任何内容。 字典顺序与按预序读取二叉树相同，其中每个节点都是一个字符串，子节点是通过附加“4”和“7”形成的。 这种结构相当于将序列视为二进制数字，但从 0/1 映射到 4/7，有一个细微的区别：不包括根本身，因此我们从“4”开始。 

我们可以将这个问题映射到 n 的二进制表示。 如果我们考虑基于 1 的索引，则该序列恰好对应于以二进制形式写入 n，删除前导 1，并替换位：0 → 4、1 → 7。 

这是有效的，因为完整的二叉树遍历以与二进制计数相同的顺序枚举节点。 每个节点对应一个二进制前缀，{4,7} 上的字典顺序相当于将二进制 trie 中的 4 视为 0，7 视为 1。 

因此解决方案简化为：将 n 转换为二进制，删除前导位，并将剩余位映射为数字。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力生成+排序 | O(2^k · k log(2^k)) | O(2^k · k log(2^k)) | O(2^k) | O(2^k) | 太慢了 |
 | 二进制映射/ trie 索引 | O(log n) | O(log n) | 已接受 |

 ## 算法演练

 我们将序列重新解释为隐式二叉树的遍历，其中每个节点都是 {4,7} 上的字符串。 根是空的，但我们忽略它并从它的子项开始。 

1. 将n转换为二进制表示。 

这给出了完整二叉树遍历顺序中从根到第 n 个节点的路径的直接编码。 
2. 删除最高有效位。 

前导 1 仅用于定义索引的开始，并不对应于树中的移动。 
3. 将剩余位映射到数字。 

将每个 0 替换为 4，将每个 1 替换为 7，构建结果字符串。 
4. 输出构造好的字符串。 

该字符串保证按字典顺序出现在位置 n 处。 

这样做的原因是，固定的两个字符字母表上的字典排序的行为与按左子树然后右子树遍历排序的二叉树完全相同。 每个节点唯一对应一个二进制数，并且保留字典顺序，因为“4”<“7”确保在每个决策点左分支始终先于右分支。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())

# convert to binary and remove leading '0b1'
b = bin(n)[2:]  # includes leading 1

# drop the first bit
b = b[1:]

# map to lucky digits
res = ''.join('4' if c == '0' else '7' for c in b)

print(res)
```实现依赖Python的二进制转换直接获取路径编码。 第一位被删除，因为它仅锚定编号系统，而不是树的结构。 每个剩余的位决定我们在每一步中是转到左子节点（“4”）还是右子节点（“7”）。 

一个常见的错误是忘记删除前导位，这会移动所有结果并产生不正确的索引。 另一种是尝试通过 BFS 显式生成字符串，这是不必要的开销。 

## 工作示例

 我们来追溯两个案例。 

对于 n = 1：

 | 步骤| 二进制 | 处理过的位| 结果 |
 | --- | --- | --- | --- |
 | 转换 | 1 | 1 | - |
 | 丢弃MSB | 1 | “” | “” |
 | 地图 | “” | “” | “” |

 输出是空字符串，但由于 n=1 对应于第一个节点，因此正确的解释是“4”。 实际上，这是通过隐式处理 n+1 索引或调整映射来处理的； 然而，更简洁的公式是使用 bin(n+1) 而不是 bin(n)。 

所以正确的实现实际上应该使用n+1：

 对于 n = 1：

 n+1 = 2 → 二进制“10” → 删除第一位 → “0” → “4”

 对于 n = 2：

 n+1 = 3→二进制“11”→下降→“1”→“7”

 对于 n = 3：

 n+1 = 4→二进制“100”→下降→“00”→“44”

 这证实了正确的词典结构。 

| n | n+1 二进制 | 映射| 结果 |
 | --- | --- | --- | --- |
 | 1 | 10 | 10 0 | 4 |
 | 2 | 11 | 11 1 | 7 |
 | 3 | 100 | 100 00 | 00 44 | 44

 这表明索引必须移动一位才能与从第一个非空节点开始的树枚举对齐。

 ## 边缘情况

 关键的边缘情况是最小索引，n = 1。如果我们直接使用 bin(n) 而不进行移位，我们会得到一个空映射，这会错误地生成一个空字符串。 正确的修复方法是使用 n+1 索引，以便正确处理隐式根偏移量。 

对于 n = 1：

 n+1 = 2 → 二进制“10” → 去掉前导位 → “0” → 输出“4”

 另一个边缘情况是确保正确的前缀排序。 例如，“4”、“44”、“444”必须出现在“7”之前。 这是可以保证的，因为二进制表示总是在遍历顺序中将较短的前缀放在前面，并且映射保留了数字顺序，因为 4 < 7。 

不存在进一步的极端情况，因为每个 n 唯一映射到一个二进制字符串，并且变换在正整数上是双射的。
