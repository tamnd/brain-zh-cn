 ## 解决方案
 ## 问题理解

 给定一个长度为 n 的数组，我们查看通过选择任何递增的索引序列定义的子序列。 对于每个选定的子序列，我们采用值的多重集并计算其 mex，即未出现在其中的最小非负整数。 仅当每个选定元素与该 mex 值的距离最多为 1 时，子序列才被视为有效。 

任务是计算有多少个非空子序列满足这个条件，接管所有可能的索引选择，并返回对 998244353 取模的答案。 

n 最大为 100000 的约束迫使任何 O(n²) 或子序列的组合枚举都是不可能的。 即使 O(n log n) 解决方案也需要避免逐子序列推理。 该结构表明，一旦 mex 固定，值的条件就会严重压缩搜索空间，因为 mex 强烈限制哪些整数可以出现在有效集合中。 

一个微妙的点是，该条件耦合了两件事：mex 附近值的成员资格，以及 mex 本身的定义，这取决于哪些值存在和不存在。 这种反馈循环是天真的推理通常被打破的地方。 

一个常见的陷阱是假设一旦值被限制在某个候选 mex 周围的小范围内，任何子集都可以工作。 这忽略了墨西哥的要求，即所有较小的值必须至少出现一次。 

另一种失败模式是混合来自不同 mex 值的贡献，而不检查相同的子序列不能对应于多个 mex 值。 在这里，mex 唯一确定结构，因此一旦正确表征，不相交计数就是安全的。 

## 方法

 蛮力的想法是枚举所有子序列并计算每个子序列的 mex。 这已经给出了 2^n 个候选者，并且计算每个子集的 mex 成本高达 O(n)，导致不可行的指数爆炸。 

关键的观察是任何有效子序列的 mex 值都不能很大。 该约束强制子序列中的每个元素位于 {M-1, M, M+1} 内。 同时，mex M 要求从 0 到 M-1 的所有整数都存在且 M 不存在。 这两个条件都是极其严格的。 

如果M为2或更大，则必须出现0的要求与允许的值集{M-1，M+1}相冲突，其中不包括0。这立即消除了除0和1之外的所有mex值。 

这种崩溃将问题简化为两个独立的计数问题：使用 mex 0 计数子序列和使用 mex 1 计数子序列。 

对于 mex 0，0 必须不存在，并且所有元素必须根据距离条件位于 {0,1} 中，强制子序列仅由 1 组成。 

对于 mex 1，0 必须至少出现一次，1 必须不存在，并且值必须位于 {0,2} 中。 这成为两个独立类别的标准组合计数。

 ## 算法演练

 我们首先按值对数组元素进行分类，但只有值 0、1 和 2 重要。 此范围之外的任何值都不能属于有效的子序列，因为它将违反条件 |a[i] − M| 对于两个可能的 mex 值均≤ 1。 

### 1. 计算频率

 我们将 c0、c1、c2 计算为值 0、1 和 2 出现的次数。 

### 2. 计算 mex 0 的子序列

 如果不存在 0 而存在 1，则子序列的 mex 为 0（否则 mex 将为 0），但距离约束强制所有元素位于 {0,1} 中。 将这些组合起来，0 就不会出现，只剩下 1。 1 的每个非空子集都是有效的。

所以这样的子序列的数量是 2^{c1} − 1。 

### 3. 使用 mex 1 计算子序列

 对于 mex 1，0 必须至少出现一次并且 1 必须不存在。 距离约束仅允许 {0,2} 中的值。 因此，我们从所有 c0 + c2 位置中选择任何子序列，但必须排除那些不包含 0 的子序列。 

这些元素的子序列总数为 2^{c0+c2}，不包含 0 的子序列为 2^{c2}。 所以贡献是 2^{c0+c2} − 2^{c2}。 

### 4. 捐款总额

 最终的答案是两个独立案例的总和。 

### 为什么它有效

 一旦 mex 被固定，允许的值范围就变成围绕它的严格区间，并且 mex 定义强制所有较小值的存在。 这只剩下两个可行的 mex 值。 由于每个有效子序列唯一地确定其 mex，因此按 mex 分区会创建不相交的集合并避免重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

n = int(input())
a = list(map(int, input().split()))

c0 = c1 = c2 = 0
for x in a:
    if x == 0:
        c0 += 1
    elif x == 1:
        c1 += 1
    elif x == 2:
        c2 += 1

# mex = 0: non-empty subsets of ones
ans0 = (modpow(2, c1) - 1) % MOD

# mex = 1: subsets of {0,2} with at least one 0
ans1 = (modpow(2, c0 + c2) - modpow(2, c2)) % MOD

print((ans0 + ans1) % MOD)
```该代码首先将数组压缩为三个相关的频率计数器。 其他一切都是纯粹的组合。 模幂用于在所需模数下有效计算子集计数。 

减法步骤处理空或无效情况的排除，因此每个项直接匹配派生的计数表达式。 

## 工作示例

 ### 示例 1

 考虑输入：```
n = 4
a = [0, 2, 3, 2]
```我们计算 c0 = 1、c1 = 0、c2 = 2。 

对于 mex 0，我们计算仅由 1 组成的子序列，但实际上没有，所以 ans0 = 0。 

对于 mex 1，我们使用 {0,2,2}。 子序列总数：2^3 = 8。没有 0 的子序列仅使用 {2,2}，得到 2^2 = 4。所以 ans1 = 4。 

最终答案是4。 

这符合以下事实：有效子序列正是包含至少一个 0 和 2 的任何子集的序列。 

### 示例 2```
n = 3
a = [1, 1, 1]
```这里 c1 = 3，c0 = c2 = 0。 

对于 mex 0，所有非空子集都是有效的，给出 2^3 − 1 = 7。 

对于 mex 1，不存在零，因此不存在有效的子序列。 

最终答案是7。 

这表明该解决方案干净地分离了贡献，而无需案例之间的交互。

 ## 边缘情况

 如果数组仅包含值 1，则 mex 0 公式会正确计算所有非空子集，因为没有元素违反允许的范围，并且由于 0 不存在，所以 mex 自动变为 0。 

如果不存在 0，则 mex 1 贡献将变为零，因为 {0,2} 中的每个候选子集都无法满足包含 0 的要求，并且减法 2^{c0+c2} − 2^{c2} 会正确折叠为零。 

如果出现大于 2 的值，则它们会隐式地从所有有效子序列中排除，因为任何包含都会违反两个可能的 mex 值的距离约束，因此它们不会做出任何贡献并在计数减少中安全地消失。
