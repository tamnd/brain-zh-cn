 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每个顶点都带有固定的正权重。 我们首先将一枚硬币放在我们选择的任何顶点上。 每次将硬币放在一个顶点上时，我们都会在日志中记录该顶点的权重。 每次放置后，我们都可以将硬币沿着有向边移动到另一个顶点，然后重复此过程。 我们必须准确执行$k-1$移动，这意味着硬币将准确地放置在$k$顶点总数，包括起始顶点。 

目标是选择起始顶点和所有后续移动，以便日志中记录的最大值尽可能小。 如果无法制作$k-1$从任意顶点开始的有效移动，答案是$-1$。 

这些约束意味着一个大图，最多可达$2 \cdot 10^5$顶点和边，以及$k$可以大到$10^{18}$。 这立即排除了任何显式模拟长度路径的方法$k$，因为即使是线性依赖$k$是不可能的。 唯一可行的策略是将长步行压缩为图的结构属性，例如可达性模式和循环。 

当图形不包含长度路径时，就会出现关键的边缘情况$k-1$任何地方。 例如，如果图是最大路径长度为 3 的 DAG，并且$k = 10$，没有起始顶点可以产生足够的移动，所以答案一定是$-1$。 简单地尝试贪婪运动的简单模拟会错误地假设如果本地存在传出边缘，我们总是可以继续，而忽略全局路径耗尽。 

另一种微妙的情况发生在循环图中，其中存在小环，但起始顶点无法到达它。 例如，如果一个循环包含低权重节点，但与廉价的起始区域断开连接，则天真的“选择最小权重顶点”策略会失败，因为它可能不支持足够的步骤。 

## 方法

 暴力解释尝试每个起始顶点，然后探索所有可能的长度游走$k-1$，跟踪每次行走的最大顶点权重。 这是正确的，因为它直接遵循流程定义。 然而，步行的次数呈指数增长$k$，甚至对状态进行记忆化$(v, steps)$，状态空间变为$O(nk)$，这是不可能的$k$最多$10^{18}$。 

关键的观察是，约束不是关于移动的确切顺序，而是关于我们是否可以维持一段距离的行走$k$同时尽可能避免高权重顶点。 如果我们固定一个阈值$X$，我们只关心顶点$a_i \le X$。 问题是是否存在一段长度的步行$k-1$完全位于允许顶点的诱导子图内。 

这将问题转化为单调决策任务$X$: 如果步行一段距离$k-1$仅使用具有权重的顶点存在$\le X$，然后任何更大的$X$也有效。 这种单调性允许对答案进行二分搜索。 

对于固定的$X$，我们限制图并检查可行性。 如果受限图包含有向循环，那么我们可以无限期地留在该循环内，这立即保证了任意长的游走，因此任何$k$是可以实现的。 如果没有环，则该图是 DAG，最长路径长度可以通过拓扑排序或 DP 来计算。 我们检查最大路径长度是否至少为$k-1$。 

这将问题转化为对值的二分搜索，并结合循环检测和过滤引起的 DAG 上的最长路径计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力步行枚举| 指数 /$O(nk)$|$O(nk)$| 太慢了 |
 | 二分查找 + 循环/DP 检查 |$O((n+m)\log n)$|$O(n+m)$| 已接受 |

 ## 算法演练

 ### 最优算法

 1. 对顶点权重进行排序或将其视为可能的最大值的二分搜索域。 我们寻找最小的阈值$X$这样有效的步行长度$k-1$仅使用顶点存在$a_i \le X$。 这是有效的，因为允许更高的阈值只会添加更多的顶点和边，而不会降低可达性。 
2. 对于固定阈值$X$，构造一个仅包含顶点的概念子图$a_i \le X$以及它们之间的边缘。 
3. 计算该子图是否包含有向环。 如果存在循环，则立即返回 true$X$，因为我们可以在循环内循环生成任意长的移动序列，涵盖任何所需的$k-1$。 
4. 如果不存在环，则该子图是 DAG。 执行拓扑排序并根据访问的顶点数计算最长路径长度。 
5. 如果最长路径长度至少为$k$， 然后$k-1$行动是可以实现的； 否则，他们就不是。 
6. 使用二分查找$X$，根据步骤3-5的可行性更新搜索范围。 最终的答案是最小可行的$X$， 或者$-1$如果没有价值。 

### 为什么它有效

 对于任何固定阈值$X$，可行性仅取决于归纳子图是否支持足够长度的游走。 这个属性是单调的$X$，自从增加$X$仅添加顶点和边。 在固定子图中，要么存在循环，允许无限的行走长度，要么图是非循环的，并且每个行走长度都受到 DAG 中最长路径的限制。 这两种结构案例充分描述了可行性，防止了 DAG 中存在长距离行走而没有循环的任何中间行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def can(x, n, g, a, k):
    # build induced graph
    active = [i for i in range(n) if a[i] <= x]
    if not active:
        return False

    ok = [a[i] <= x for i in range(n)]
    color = [0] * n  # 0 unvisited, 1 visiting, 2 done

    has_cycle = False

    def dfs(v):
        nonlocal has_cycle
        color[v] = 1
        for to in g[v]:
            if not ok[to]:
                continue
            if color[to] == 1:
                has_cycle = True
                return
            if color[to] == 0:
                dfs(to)
            if has_cycle:
                return
        color[v] = 2

    for v in range(n):
        if ok[v] and color[v] == 0:
            dfs(v)
            if has_cycle:
                break

    if has_cycle:
        return True

    # DAG: compute longest path via DP in topo order
    indeg = [0] * n
    for v in range(n):
        if not ok[v]:
            continue
        for to in g[v]:
            if ok[to]:
                indeg[to] += 1

    from collections import deque
    q = deque([i for i in range(n) if ok[i] and indeg[i] == 0])

    dp = [1] * n  # longest path ending at v

    while q:
        v = q.popleft()
        for to in g[v]:
            if not ok[to]:
                continue
            if dp[to] < dp[v] + 1:
                dp[to] = dp[v] + 1
            indeg[to] -= 1
            if indeg[to] == 0:
                q.append(to)

    return max(dp[i] for i in range(n) if ok[i]) >= k

def solve():
    n, m, k = map(int, input().split())
    a = list(map(int, input().split()))
    g = [[] for _ in range(n)]

    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)

    if k == 1:
        print(min(a))
        return

    lo, hi = min(a), max(a)
    ans = -1

    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid, n, g, a, k):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现将可行性检查分离为评估固定阈值的函数。 循环检测使用 DFS 着色，这已经足够了，因为我们只关心过滤后的子图中是否存在任何循环。 最长路径计算仅在图是非循环时使用，其中拓扑排序上的 DP 有效。 

一个微妙的点是 DP 将每个可到达的节点初始化为值 1，因为单个顶点已经贡献了一个位置。 这符合步行长度计算访问过的顶点而不是边的要求。 

二分搜索是在权重值上执行的，而不是在路径长度上执行的，因为可行性在允许的最大顶点值中是单调的。 

## 工作示例

 ### 示例 1

 输入：```
6 7 4
1 10 2 3 4 5
1 2
1 3
3 4
4 5
5 6
6 2
2 5
```我们对$X$。 为了$X = 3$，仅顶点$\{1,3\}$仍然存在，且不存在长度为 4 的路径。 为了$X = 4$, 顶点$\{1,3,4\}$允许更长的链条，但仍然没有循环且长度不足。 为了$X = 5$, 顶点$\{1,3,4,5\}$通过边形成循环，从而实现任意长的遍历。 

| 中X | 活跃节点| 循环 | 最长路径| 可行|
 | ---| ---| ---| ---| ---|
 | 3 | 1,3 | 没有 | 2 | 没有 |
 | 4 | 1,3,4 | 没有 | 3 | 没有 |
 | 5 | 1,3,4,5 | 是的 | 无穷大| 是的 |

 答案是4。 

该轨迹显示仅当受限图中的循环可用时可行性如何跳跃。 

### 示例 2

 输入：```
2 1 5
1 1
1 2
```这里两个顶点的权重相等。 由于该图是非循环的，因此任何长距离行走都是不可能的，因为最大路径长度为 2。即使所有阈值都允许两个节点，但不存在循环。 

| 中X | 活跃节点| 循环 | 最长路径| 可行|
 | ---| ---| ---| ---| ---|
 | 1 | 1,2 | 没有 | 2 | 没有 |

 答案是-1。 

这表明当结构深度有限时，允许所有顶点是不够的。

 ## 边缘情况

 深度非常小的纯非循环图表明任何基于循环的直觉都是失败的。 即使存在许多低权重顶点，循环的缺乏也会迫使所有游走快速终止，因此算法正确地拒绝大的$k$DP计算后。 

所有顶点都是高权重的完全循环图显示相反的行为。 尽管最小阈值可能只包括顶点的子集，但一旦包含循环，答案就会变得稳定，因为无限重复是可能的，并且二分搜索收敛于最小阈值，该最小阈值暴露了从起点可达的任何循环。
