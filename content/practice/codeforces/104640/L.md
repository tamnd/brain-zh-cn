 ## 解决方案
 ## 问题理解

 我们得到一个长度为的隐藏数组$n$。 该数组是严格递增的，这意味着每个下一个值都大于前一个值。 但是，我们不可以直接看到数组。 

取而代之的是一个互动功能$f(x)$。 每个查询都询问该位置的值$x$，但该数组是通过某个未知值的隐藏循环移位来访问的$c$。 具体来说，当我们要求职位时$x$，我们实际上收到了最初位于位置的值$x+c$，在需要时环绕数组。 

所以我们通过查询观察到的数组只是排序递增数组的旋转版本。 我们的任务是恢复旋转量$c$，最多使用 42 个查询。 

约束条件意味着$n$可以大到$10^5$，所以我们买不起任何线性的东西$n$。 我们被限制在大约$O(\log n)$交互查询。 任何扫描所有位置的方法都是立即不可能的。 

关键的结构事实是，严格递增的数组在旋转时会变成循环排序的数组，并且只有一个“断点”，其中顺序从大值重置回小值。 这种结构是稳定的并且允许二分查找。 

一个天真的尝试是查询所有位置并重建整个数组，然后找到最小值并推断出移位。 这使用$n$查询并超出限制。 

另一个天真的想法是比较相邻位置以找出增量中断的位置。 虽然概念上是正确的，但它仍然需要$O(n)$查询。 

唯一有效的路径是利用旋转排序序列中的单个不连续性。 

## 方法

 如果我们忽略交互限制，最简单的方法是查询每个索引$1$通过$n$，重建数组，找到最小元素位置，并计算移位。 这是可行的，因为原始数组是严格递增的，因此最小值唯一标识了旋转点。 问题是这会消耗$n$查询次数，远远超出允许的 42 个$n$很大。 

重要的观察是观察到的数组是旋转排序数组。 这种阵列除了一个枢轴点外具有单调结构。 这意味着我们可以通过与已知边界值进行比较，使用二分搜索来定位最小元素。 

我们利用这样一个事实：在旋转递增数组中，“左段”中的所有元素都大于“右段”中的元素，并且最小元素位于右段的开头。 通过将中点与固定参考进行比较，我们可以确定哪一半包含最小值并丢弃另一半。 

一旦我们找到了查询数组中最小元素的位置，我们就可以将该索引转换回旋转移位$c$。

 | --- | --- | --- | --- |
 | 暴力扫描 |$O(n)$查询 |$O(1)$| 太慢了 |
 | 旋转数组上的二分查找 |$O(\log n)$查询 |$O(1)$| 已接受 |

 ## 算法演练

 我们将查询的序列视为旋转排序数组。 目标是找到该序列中最小元素的索引。 

1.我们首先查询一个固定的参考位置，通常$f(n)$。 该值对应于旋转数组中第一个排序段的最后一个元素，并充当分隔两个单调部分的阈值。 通过与该值的比较，我们可以检测到我们处于旋转的哪一侧。 
2. 我们对索引执行二分搜索$1$到$n$。 在每个中点$mid$，我们查询$f(mid)$。 
3.如果$f(mid) > f(n)$， 然后$mid$位于左排序段中，这意味着最小元素必须位于$mid$。 我们将搜索区间移至右半部分。 
4. 否则，$f(mid) \le f(n)$， 所以$mid$位于右侧段（包含最小值的包裹部分）。 在这种情况下，最小值为$mid$或到其左侧，因此我们将间隔缩小到左半部分，包括$mid$。 
5. 当二分查找完成后，左边界等于旋转数组中最小元素的位置。 呼叫此职位$pos$。 
6.最后，我们将这个位置转换回移位值$c$。 由于旋转是循环的，所以关系式为$c = (n + 1 - pos) \bmod n$。 

### 为什么它有效

 旋转后的数组恰好有一个不连续点，其中一个大值后面跟着一个小值。 该点一侧的每个元素都严格大于另一侧的每个元素。 与的比较$f(n)$一致地区分这两个区域，而不需要直接了解原始数组。 二分查找不变式是最小元素始终保留在当前搜索区间内，并且更新规则永远不会丢弃包含它的段。 

## Python 解决方案```python
import sys

input = sys.stdin.readline

def ask(x):
    print(f"? {x}", flush=True)
    v = int(input().strip())
    if v == -1:
        exit(0)
    return v

def main():
    n = int(input().strip())
    if n == 1:
        print("! 0", flush=True)
        return

    fn = ask(n)

    l, r = 1, n
    while l < r:
        mid = (l + r) // 2
        vm = ask(mid)

        if vm > fn:
            l = mid + 1
        else:
            r = mid

    pos = l
    c = (n + 1 - pos) % n
    print(f"! {c}", flush=True)

if __name__ == "__main__":
    main()
```该解决方案依赖于交互式查询，因此每个输出都会立即刷新。 第一个存储的值是$f(n)$，充当主元比较器。 每个二分搜索步骤都会将候选间隔减少一半，同时保留最小值的位置。 

从位置到移位的最终转换小心地处理循环索引。 即使最小值位于位置 1 或位置，模数表达式也能确保正确性$n$。

 ## 边缘情况

 当$c = 0$，数组已排序，最小值位于位置 1。二分查找总是将右边界向左推，直到收敛于 1，因为每个中点都满足$f(mid) \le f(n)$。 

什么时候$c = n-1$，最小元素出现在最后一个位置。 二分搜索最初观察左侧段上的大值，然后重复向右移动，直到隔离最终位置。 

为了$n = 1$，不需要任何查询，并且答案几乎为零，因为不可能进行旋转。
