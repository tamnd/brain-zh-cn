 ## 解决方案
 ## 问题理解

 我们得到两个独立的有向未加权加权图，它们共享同一组从 1 到 n 的顶点标签。 您可以将它们视为两个平行的城市，每个城市都有 n 座摩天大楼位于相同的坐标上，但每个城市内允许的移动路线不同。 

在第一个城市内，一些摩天大楼对通过双向边以给定的行驶时间连接。 第二个城市有自己独立的双向边缘集和自己的旅行时间。 除了这些内部动作之外，还有一个特殊的过渡：从第一个城市的任何摩天大楼 i ，你可以立即“切换世界”到第二个城市的摩天大楼 i ，支付固定成本 x 秒，反之亦然。 

你从第一个世界的摩天大楼 s 开始，并希望在最短的总时间内到达第二个世界的摩天大楼 t，结合内部移动和世界切换。 

关键结构是这不是两个独立的最短路径问题。 世界之间的移动将两个图耦合在一起，因此如果可以降低成本，最佳路线可能会在世界之间多次交替。 

约束条件直接影响算法的选择。 由于 n 最多为 100000 条，每个世界最多有 10^6 条边，因此组合图大约有几百万条边加上 100000 条交叉边。 任何尝试重新计算每个查询的最短路径或使用密集表示的解决方案都会失败。 具有对数因子的线性或近线性图遍历（例如具有堆的 Dijkstra）是唯一现实的选择。 

一个幼稚的错误是尝试分别计算每个世界中的最短路径，然后贪婪地组合答案。 但这失败了，因为切换世界可以解锁更便宜的路线。 

例如，假设在世界 1 中，从 s 到 t 的路径非常昂贵，但从 s 到某个 i 的路径很便宜，而在世界 2 中，从 i 到 t 的路径很便宜。 最佳路线必须在 i 处切换世界，而单独的计算永远无法正确考虑这一点。 

另一个微妙的失败案例是假设最多一次世界切换就足够了。 这是错误的，因为您可能会提前进入世界 2 以绕过世界 1 中的长段，然后通过另一个索引返回世界 1，并在需要时再次切换回来。 

## 方法

 暴力心理模型是将两个世界中每一个可能的移动序列视为状态空间搜索。 每个状态不仅由当前节点定义，而且由当前世界定义。 从一个状态（节点、世界），您可以遍历该世界中的所有边或以相同的索引成本 x 切换到另一个世界。 

这形成了一个有 2n 个顶点的图。 每个原始边仅贡献一层，每个索引 i 贡献 (i, world1) 和 (i, world2) 之间的交叉边。 在这个扩展图上运行最短路径给出了正确的答案。 

简单的实现已经直接引出了 Dijkstra 算法。 关键的见解是，虽然问题看起来像两个图，但它实际上是一个具有 2n 个节点的稀疏加权图。 跨世界的转变只是额外的边缘。 一旦看到这种重新表述，就不需要进一步的技巧了。 

较慢的替代方案是尝试对“开关数量”进行动态编程或反复松弛图之间的路径，但这会退化为重复的最短路径计算，并且在 10^6 边下速度太慢。 

正确的方法就是在组合图上运行一次 Dijkstra。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解路径/重复重新计算 | 指数 / O(k·(n+m) log n) | O(n+m) | 太慢了|
 | 2 层图上的 Dijkstra | O((n + m1 + m2) log n) | O((n + m1 + m2) log n) | O(n + m1 + m2) | O(n + m1 + m2) | 已接受 |

 ## 算法演练

我们构造一个具有 2n 个节点的图。 节点 i 代表世界 1 中的摩天大楼 i，节点 i + n 代表世界 2 中的摩天大楼 i。 

然后，我们在前半部分内添加来自第一个世界的所有边，在后半部分内添加来自第二世界的所有边，并在 i 和 i + n 之间添加成本 x 的交叉边。 

我们从世界 1 中的起始节点 s 运行 Dijkstra 并计算到所有节点的距离。 答案是到世界 2 中 t 的距离，如果无法到达则为无穷大。 

## 算法演练

 1. 构建一个具有 2n 个顶点的图，将问题分为两层，每个世界一层。 这种分离允许我们将世界切换编码为显式边缘而不是特殊逻辑。 
2. 对于第一个世界中的每条边 u、v、c，在第 1 层中的 u 和 v 之间添加一条权重为 c 的双向边。 这将完全按照给定的方式保留世界 1 内所有允许的移动。 
3. 对于第二世界中的每条边 u、v、c，在 u + n 和 v + n 之间添加一条权重为 c 的双向边。 这将第二世界镜像到图表的后半部分。 
4. 对于从 1 到 n 的每个索引 i，添加两条边：i 到 i + n 和 i + n 到 i，每条边的成本为 x。 该模型模拟了世界各地相同摩天大楼之间的传送。 正确性依赖于这样一个事实：切换世界不会改变索引，只会改变图层。 
5. 从世界1中的s（即节点s）开始运行Dijkstra。 这按照总时间的递增顺序探索了世界内移动和切换的所有可能组合。 
6. 提取到世界 2 中节点 t 的距离，即节点 t + n。 如果这个距离仍然是无穷大，则输出-1。 

### 为什么它有效

 任何有效的旅程都与构建的 2n 节点图中的路径完全对应，因为原始问题中的每种运动类型都表示为一条边。 相反，构造图中的每条路径都对应于原始设置中的合法移动序列。 由于所有边都有非负权重，Dijkstra 保证我们第一次最终确定一个节点时，我们已经找到了到达该节点的最小可能成本。 这确保计算出的到（世界 2 中的 t）的距离是所有可能的混合世界路线中的最佳旅行时间。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

INF = 10**30

def solve():
    n, x = map(int, input().split())
    
    m1 = int(input())
    g = [[] for _ in range(2 * n)]
    
    for _ in range(m1):
        u, v, c = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((v, c))
        g[v].append((u, c))
    
    m2 = int(input())
    for _ in range(m2):
        u, v, c = map(int, input().split())
        u -= 1
        v -= 1
        u += n
        v += n
        g[u].append((v, c))
        g[v].append((u, c))
    
    for i in range(n):
        g[i].append((i + n, x))
        g[i + n].append((i, x))
    
    s, t = map(int, input().split())
    s -= 1
    t -= 1
    
    dist = [INF] * (2 * n)
    dist[s] = 0
    pq = [(0, s)]
    
    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        if u == t + n:
            break
        for v, w in g[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    
    ans = dist[t + n]
    print(-1 if ans == INF else ans)

if __name__ == "__main__":
    solve()
```代码直接实现了2层图的构建。 邻接表`g`同时拥有两个世界，索引 [0, n-1] 代表世界 1，[n, 2n-1] 代表世界 2。 

最微妙的部分是索引处理。 所有顶点都转换为从零开始的索引，第二个世界移动+n。 为每个 i 均匀添加交叉边，确保切换方向之间的对称性。 

Dijkstra 循环使用标准优先级队列。 到达目标时的早期突破是可选的，但很安全，因为 Dijkstra 保证我们第一次以最小距离弹出目标时，这是最佳的。

 ## 边缘情况

 一种边缘情况是根本没有世界内边缘。 唯一可能的移动是世界切换，因此路径变成了简单的交替切换链。 该算法处理此问题是因为 Dijkstra 仍然正常探索交叉边，并且如果可达，答案将变为 x 的倍数。 

另一种情况是最优路径需要多个交换机。 例如，使用世界2来缩短部分旅程，然后返回世界1到达另一个区域，然后再次切换。 2n 节点图自然允许这样做，并且 Dijkstra 不受限制地探索此类序列。 

最后一种情况是 s 等于 t。 即使如此，答案也不一定为零，因为目的地具体位于世界 2 中。该算法正确地强制至少一次转换，除非 s 已经等于 t 并且世界 2 中存在零成本路径而不离开。
