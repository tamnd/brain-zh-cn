 ## 解决方案
 ## 问题理解

 我们得到一组轴对齐的矩形，每个矩形都由其高度和宽度定义，并且每个矩形还允许旋转 90 度。 这意味着对于每个矩形，我们可以将其视为具有两个可能的方向，并且我们可以自由选择对我们有帮助的方向。 

目标是选择这些矩形的子集，并将它们排列在严格的非递减链中，其中每个矩形都适合下一个矩形。 具有尺寸的矩形$(h_1, w_1)$可以放在里面$(h_2, w_2)$如果高度和宽度均不超过相应的尺寸。 我们想要尽可能长的这样的链，并且我们还需要按顺序输出形成它的矩形的索引。 

该问题本质上是要求二维偏序下的最长链，但由于旋转，每个项目都有两种可能的状态。 

约束条件$n \le 10^5$立即排除对之间的任何二次比较。 任何试图以简单的方式比较所有对或对所有先前矩形运行动态编程的方法都会导致大约$10^{10}$在最坏的情况下进行操作，这远远超出了可行的限度。 

一个天真的但诱人的错误是独立对待每个矩形，在本地选择其最佳方向，然后排序并贪婪地获取一条链。 这失败了，因为定位决策是全局性的。 可能需要将矩形“牺牲”为非最佳方向，以便稍后启用更长的链。 

当旋转后两个尺寸相等时，会出现另一个微妙的问题。 如果我们总是不小心通过排序维度来进行标准化，我们可能会无意中允许无效链或破坏平局一致性，这可能会破坏正确序列的重建。 

## 方法

 强力解决方案将尝试所有子集和所有方向选择，检查给定序列是否形成有效的嵌套链。 即使限制子序列，这也会呈指数增长$n$。 稍微结构化一点的强力方法是对子集或对进行动态编程，但即便如此，我们最终也会将每个矩形与其他每个矩形进行比较，从而导致$O(n^2)$过渡。 和$10^5$矩形，这是完全不可行的。 

关键的结构观察是，一旦矩形被放入链中，一维沿着链不会递减，我们可以通过排序来强制排序。 真正的挑战是我们正在处理二维排序，这是二维中经典的最长递增子序列（LIS）。 

旋转自由度将每个矩形简化为对其两侧的排序的选择。 最好的策略是始终调整每个矩形的方向，使其较小的边被视为高度，较大的边被视为宽度。 这种标准化确保了一致性：每个矩形都以其最小规范形式表示$(a, b)$和$a \le b$。 任何最优链都可以使用这种表示法转换为链，因为交换轴只会有助于减少约束。 

归一化后，问题简化为寻找最长的对序列$(a_i, b_i)$使得两个坐标都是非递减的。 这是一个标准的二维 LIS 问题。 按第一个坐标排序，然后在第二个坐标上找到 LIS，给出最优的链结构。 

为了重建序列，我们必须在 LIS 计算期间存储前驱序列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n)$或者$O(n^2)$|$O(n)$| 太慢了|
 | 最佳 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ## 算法演练

 1. 对于每个矩形，重新排列其尺寸，以便$a = \min(h, w)$和$b = \max(h, w)$。 这将旋转选择固定为规范表示，确保每个矩形都以最严格的嵌套方向表示。 
2. 将每个矩形存储为一个三元组$(a, b, index)$，保留原始索引，以便我们稍后可以重建答案。 
3. 对所有矩形进行排序$a$按递增顺序，并且对于相等$a$， 经过$b$按递增顺序。 这确保任何有效的嵌套链都必须遵守第一个坐标上的顺序，因此我们只需要控制前进的第二个坐标。 
4. 对序列运行最长非递减子序列过程$b$价值观。 我们维护一个 DP 结构，在其中存储每个长度的子序列的最小可能结束值。 
5. 为了允许重建，我们为每个元素存储结束于该处的最佳子序列的长度以及指向该子序列中的前一个元素的前驱指针。 这是必要的，因为问题需要实际的链，而不仅仅是其长度。 
6. 更新LIS结构时，我们使用二分查找来查找当前位置的第一个位置$b$可以放置，保持最小尾部值。 每次更新都对应于扩展或改进一个子序列。 
7. 处理完所有矩形后，我们识别具有最大子序列长度的位置，并通过向后跟随前驱指针来重建链，然后反转结果。 

### 为什么它有效

 正确性取决于将 2D 支配问题简化为 1D LIS 问题。 排序依据$a$确保任何有效链必须已经满足第一个维度约束。 之后，第二维度就成为唯一剩下的条件。 因为所有矩形都已预先排序，所以我们构建的任何子序列都会自动保留非递减$a$，因此强制执行 LIS$b$保证完全有效。 

前驱指针确保 LIS 中的每个扩展都对应于按排序顺序的矩形之间的真实有效转换，因此重构的链在两个维度上都是有效的嵌套序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    rects = []
    for i in range(n):
        h, w = map(int, input().split())
        a, b = min(h, w), max(h, w)
        rects.append((a, b, i + 1))
    
    rects.sort(key=lambda x: (x[0], x[1]))
    
    import bisect
    
    tail = []
    tail_idx = []
    parent = [-1] * n
    pos_in_tail = [-1] * n
    
    # store best ending index for each length
    best_idx = []
    
    for i, (a, b, idx) in enumerate(rects):
        j = bisect.bisect_right(tail, b)
        
        if j == len(tail):
            tail.append(b)
            best_idx.append(i)
        else:
            tail[j] = b
            best_idx[j] = i
        
        pos_in_tail[i] = j
        
        if j > 0:
            parent[i] = best_idx[j - 1]
    
    # reconstruct LIS
    length = len(tail)
    last = best_idx[length - 1]
    
    ans = []
    while last != -1:
        ans.append(rects[last][2])
        last = parent[last]
    
    ans.reverse()
    
    print(len(ans))
    print(*ans)

if __name__ == "__main__":
    solve()
```该代码首先对每个矩形进行标准化，以便以一致的方式固定方向。 按第一个维度排序可确保我们只需要管理第二个维度的可行性。 LIS 结构使用`tail`为每个长度的子序列维护最小可能的结束值，并且`best_idx`记住当前哪个矩形定义了每个子序列长度端点。 

这`parent`数组可以实现重建。 每次矩形扩展一个子序列时，它都会存储一个指向长度减一的最佳序列的指针。 这保证了当我们从最优子序列的最后一个元素回溯时，我们以正确的顺序恢复有效的链。 

一个微妙的点是使用`bisect_right`而不是`bisect_left`。 这强制执行非递减序列而不是严格递增序列，匹配条件$h_1 \le h_2$,$w_1 \le w_2$。

该跟踪显示了替换尾部如何在不破坏正确性的情况下提高未来的扩展潜力。

 ## 边缘情况

 一个微妙的边缘情况是许多矩形在标准化后变得相同。 例如，$(3,5), (5,3), (3,5)$全部标准化为$(3,5)$。 该算法允许所有这些都参与 LIS，因为我们使用非递减转换。 这是正确的，因为相同的矩形始终可以按任何顺序嵌套。 

另一种情况是轮换对于可行性很重要。 一个长方形就像$(2,10)$和另一个$(9,3)$都会标准化为$(2,10)$和$(3,9)$，并且如果没有标准化一致性，贪婪的方向选择可能会破坏链条。 规范变换在 LIS 计算之前消除了这种歧义。 

最后，两个维度上都有大量递增链的情况会强调重建的正确性。 由于每个 DP 转换都存储前趋，因此即使存在多个最优子序列，回溯步骤也始终会恢复一致的序列。
