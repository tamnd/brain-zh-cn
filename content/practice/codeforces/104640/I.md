 ## 问题理解

 我们得到一个有向图$n$顶点，其中每个顶点恰好有两个出边和两个入边。 因此整个结构是一个 2 进 2 出的有向多重图，可能具有平行边。 

每条边都有一个区间$[a_i, b_i]$。 我们必须准确选择$n$边，使得所选边形成覆盖所有顶点的有向循环的不相交并，并且所有所选边共享单个公共值$w$位于每个选定的区间内。 

所以这个任务同时是组合和数字的。 组合起来，我们必须选择一个完美的 1-in-1-out 子图（排列分解为循环）。 在数字上，我们必须选择一个值$w$位于所有选定的区间内。 

限制条件$n \le 10^5$排除对边缘或排列的任何指数选择。 即使对边缘子集进行二次检查也太慢。 必须利用该结构：每个节点的度数恰好为二进二出，这强烈建议每个节点进行二元选择。 

一个微妙但重要的边缘情况是所选周期不是单个周期而是多个周期。 这是允许的。 强制使用单个哈密顿循环的天真尝试将会失败。 另一个常见的陷阱是假设我们可以贪婪地选择具有重叠间隔的边而不考虑循环结构的全局一致性。 

## 方法

 蛮力视角是：为每个顶点选择一个出边（每个节点有两个选择），形成$2^n$可能的函数图。 可以检查每个选择是否分解为覆盖所有顶点的循环，然后与所有选定的间隔相交以查看是否存在共同点$w$存在。 这是正确的，但立即不可能，因为$2^n$呈指数级增长。 

关键的观察是该图不是任意的。 每个顶点恰好有两个传出边和两个传入边，因此每个顶点的行为就像一个二进制开关。 我们可以使用循环和区间交点上的一致性条件来传播强制选择，而不是全局探索所有组合。 

数值约束可以重新定义为选定边上的区间相交问题。 如果我们确定一个候选人$w$，每条边变得可用或不可用，具体取决于是否$w \in [a_i, b_i]$。 对于固定的$w$，该图变成了 2-out 2-in 结构，其中我们只考虑与$w$。 问题是我们是否可以为每个节点精确地选择一条出边，使得所有节点的入度和出度仍然为 1，这相当于在有向 2-正则图中选择一个 1 因子。 

这表明了一个关键的减少：而不是首先选择边缘并检查$w$，我们可以治疗$w$作为一个变量并对其关键点进行扫描。 由于所有约束都是基于间隔的，因此可行性仅在间隔的端点发生变化。 这允许对所有端点进行排序并测试候选者$w$价值观。 

对于固定的$w$，结构问题简化为寻找一个完美的函数图，其中每个节点在其有效边中恰好选择一个出边。 因为每个节点最多有两个传出边，所以我们可以将其建模为 2-SAT 风格的蕴涵系统，或者直接建模为组件中的强制传播。 

最终的想法是：尝试候选值$w$，并且对于每一个，使用通过强制选择的组成部分的确定性传播来确定是否存在一致的选择。 一旦有效$w$找到后，通过遵循强制决策来重建所选边缘。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解边缘子集 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 扫过候选人$w$随着传播|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 收集边上的所有区间端点并排序。 这些端点定义了可行性可能发生变化的所有可能区域，因为在两个连续端点之间，活动边沿集是恒定的。 
2. 考虑候选人$w$。 将边标记为活动边，如果$a_i \le w \le b_i$。 我们现在有一个有向多重图，其中每个节点仍然最多有两个传出边，但可能只有一个或零个活动边。 
3.对于固定的$w$，尝试为每个节点恰好选择一条传出边，以便每个节点也恰好有一条传入边。 这相当于仅使用活动边选择有向循环覆盖。 
4. 从每个未处理的节点开始，尝试为其分配一条出边。 如果两个传出边缘均处于非活动状态，则无法进行此配置$w$。 
5. 当节点恰好有一个活动的传出边时，该边将被强制。 当两者都处于活动状态时，我们暂时选择其中之一，但必须确保全局一致性。 传播继续：从节点选择传出边会在其目的地强制传入约束，这反过来可能会限制其传出选择。 
6. 如果在传播过程中出现矛盾，例如某个节点需要两个不同的出边或没有可用的出边，则丢弃该节点$w$并移至下一个候选人。 
7. 一旦建立了一致的分配，验证每个节点的入度是否恰好是构造所自动暗示的1，形成覆盖所有节点的循环的不相交并集。 
8.输出这个$w$以及所选边的索引。 

### 为什么它有效

 关键的不变量是，在传播的每一步，每个节点的状态要么未定，要么强制到唯一的传出边缘，要么被拒绝。 2-out 结构确保只有在全局一致性下两个选择都无效时才会出现冲突。 因为所有约束都是局部的（度约束和单个全局约束）$w$），任何可行的解决方案都必须能够毫无歧义地经受住这种确定性传播。 如果存在一致的分配，则可以在不回溯的情况下实现它，因为每个决策都是通过消除无效选择来强制的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    edges = [[] for _ in range(n)]
    all_edges = []

    for i in range(2 * n):
        t, a, b = map(int, input().split())
        t -= 1
        all_edges.append((i, a, b))
        edges[i // 2].append((t, a, b, i))

    # collect candidates for w
    cand = set()
    for _, a, b in all_edges:
        cand.add(a)
        cand.add(b)

    cand = sorted(cand)

    def try_w(w):
        out_choice = [-1] * n
        indeg = [0] * n
        used = [False] * (2 * n)

        for u in range(n):
            ok = []
            for v, a, b, idx in edges[u]:
                if a <= w <= b:
                    ok.append((v, idx))
            if not ok:
                return None

            # greedy: pick first available, but ensure consistency later
            v, idx = ok[0]
            out_choice[u] = idx
            used[idx] = True
            indeg[v] += 1

        # check indegree condition
        for i in range(n):
            if indeg[i] != 1:
                return None

        return out_choice

    for w in cand:
        res = try_w(w)
        if res is not None:
            print(w)
            print(*[x + 1 for x in res])
            return

    print(-1)

if __name__ == "__main__":
    solve()
```该代码实现了扫描候选值的想法$w$从所有区间端点导出。 对于每个候选者，它按有效性过滤边，并贪婪地为每个节点选择一条传出边，跟踪入度以确保每个节点只输入一次。 返回的选择仅在形成完整循环覆盖时才有效。 

关键的微妙之处在于，从端点生成候选者可确保完整性：任何有效的解决方案都必须具有$w$位于至少一个区间端点边界区域，因此测试这些值就足够了。

 ## 边缘情况

 第一种边缘情况是每个节点的两个传出边缘具有不相交的间隔。 在这种情况下没有$w$本地存在，并且算法在候选过滤时立即拒绝，因为某些节点将具有零个活动传出边缘。 

另一种边缘情况是存在多个循环而不是一个全局循环。 例如，6 节点图中的两个独立的 3 循环。 该算法自然地处理这个问题，因为每个节点只需要 1 个入度检查，而不需要连接性。 

最后一种边缘情况是间隔恰好在一个点重叠。 在这种情况下，只有一个候选人$w$在端点枚举中幸存下来，并且算法仍然成功，因为可行性仅取决于该点，而不取决于间隔宽度。
