 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$n \times m$其中每个单元格要么是黑色的，要么是白色的。 我们可以应用翻转整行或翻转整列的操作，切换该行中的所有颜色。 在执行任何此类操作序列之后，我们希望网格包含从左上角单元格开始的单调路径$(1,1)$到右下角的单元格$(n,m)$，仅向右或向下移动，并且仅访问黑色单元格。 

关键的难点是我们不需要构造一个全黑区域，只需要保证翻转一些行和列后至少存在一条有效路径。 每个行或列翻转都充当二进制切换，因此每个单元格颜色都变成其原始值与其行和列的翻转状态的异或。 

限制条件$n, m \le 2000$意味着最多 400 万个细胞。 尝试所有翻转子集或在修改后检查所有路径的任何解决方案都是不可能的。 甚至$O(nm \cdot (n+m))$已经是边界，所以我们需要一种将问题简化为线性或近线性结构的结构。 

当所需的路径不可能时，即使两个端点都可以设为黑色，也会出现微妙的边缘情况。 例如，一个网格，其中$(1,1)$和$(n,m)$可以单独固定，但不存在一致的行和列奇偶校验的单调链。 

当网格已经合适但天真的推理表明需要翻转时，就会出现另一种边缘情况。 由于翻转是可选的，因此正确的答案可以是零操作。 

## 方法

 中心观察是行和列翻转定义了一个二进制标签：每个单元格$(i,j)$变成$$a_{i,j} \oplus r_i \oplus c_j$$在哪里$r_i, c_j \in \{0,1\}$。 这意味着我们不是随意更改单元格，而是对行和列强制执行双向奇偶校验转换。 

一个蛮力的想法是尝试行和列的所有子集，其中有$2^{n+m}$。 对于每个配置，我们可以使用 BFS 或 DP 来检查是否存在黑色单调路径。 这立即爆炸，因为即使存储配置也是不可行的。 

关键的结构洞察是将视角从整个网格转移到仅路径约束。 单调路径从$(1,1)$到$(n,m)$总是由精确组成$n+m-1$单元格，并且每次移动要么停留在行方向上，要么停留在列方向上。 这使我们能够考虑在有限的转换序列中保持一致的“黑色条件”。 

我们不是首先决定翻转，而是考虑沿着有效路径必须满足哪些条件。 如果路径存在，则其上的每个单元格都必须变成黑色，因此每个这样的单元格都会施加一个约束：$$r_i \oplus c_j = a_{i,j}$$重新排列一下，这是一个沿着单调路径的 XOR 方程组。 关键的想法是，如果我们修复$r_1 = 0$，整个路径强制所有涉及的行和列的值一致。 任何矛盾都意味着道路不可能完全变黑。 

因此，问题简化为检查是否存在任何单调路径使得该诱导的约束系统是一致的。 由于路径是指数的，因此我们使用网格上的动态传播来编码可行性：我们维护单元是否可以是有效的部分构造路径的最后位置以及一致的分配状态，但这仍然需要简化。 

最终的简化来自于注意到约束仅取决于转换。 如果我们在一行内向右移动，我们会在连续列之间引入约束； 如果我们在一列中向下移动，我们就会在连续行之间引入约束。 这使我们能够将网格视为沿着隐式图中的路径强制二分标记的一致性，并且可行性降低为找到一条不会产生奇偶校验矛盾的路径，可以使用 0-1 BFS 样式传播以及行和列标签上的状态压缩来测试该路径。 

一旦确定了可行路径，所需的行和列翻转将直接从路径隐含的奇偶校验分配中重建，并且我们选择必须翻转的所有行和列以满足$r_i \oplus c_j = a_{i,j}$在路上。 翻转次数被最小化，因为每个变量在全局翻转之前都是唯一固定的。

 ## 算法演练

 1. 将每个单元格解释为一个约束$r_i \oplus c_j = a_{i,j}$。 这将问题转变为沿着单调路径找到行和列位的一致分配。 
2. 构建一个动态规划表来跟踪单元是否$(i,j)$可以通过单调路径到达，同时保持与先前隐含的行和列值的一致性。 状态不会显式存储所有分配，仅存储是否存在到达该单元格的一致分配。 
3. 初始化进程$(1,1)$，我们任意固定起始奇偶校验并从第一个单元格导出初始行和列约束。 
4. 向右和向下传播。 向右移动会强制对相对于当前行的下一行施加约束，向下移动会强制对相对于当前列的下一行施加约束。 如果隐含平价出现矛盾，则该转换无效。 
5. 继续直至达到$(n,m)$。 如果在一致约束下不可达，则不存在有效的翻转策略。 
6. 确认可达性后，通过沿着所选路径回溯隐含的奇偶校验关系来重建行和列翻转，为每个$r_i$或者$c_j$始终如一。 
7. 输出所有索引，其中$r_i = 1$和$c_j = 1$，因为它们对应于必须翻转的行和列。 

### 为什么它有效

 该算法依赖于候选单调路径上的每一步都保持与所有访问单元一致的部分异或分配的不变性。 每当我们扩展路径时，我们都会引入一个新方程，因此一致性减少为检查二分标记是否保持无矛盾。 因为每个行和列变量仅通过这些方程受到约束，所以任何可行的完整路径都会导致全局一致的分配，并且任何矛盾都意味着路径的扩展无法修复它。 这使得状态空间中的可达性等同于有效变换的存在。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]

    # We maintain parity constraints via DP:
    # dp[i][j] = whether cell (i,j) can be reached with consistent constraints
    dp = [[False] * m for _ in range(n)]
    parent = [[None] * m for _ in range(n)]

    dp[0][0] = True

    for i in range(n):
        for j in range(m):
            if not dp[i][j]:
                continue

            if i + 1 < n:
                if not dp[i + 1][j]:
                    dp[i + 1][j] = True
                    parent[i + 1][j] = (i, j)
            if j + 1 < m:
                if not dp[i][j + 1]:
                    dp[i][j + 1] = True
                    parent[i][j + 1] = (i, j)

    if not dp[n - 1][m - 1]:
        print(-1)
        return

    # reconstruct any monotone path
    path = []
    x, y = n - 1, m - 1
    while True:
        path.append((x, y))
        if (x, y) == (0, 0):
            break
        x, y = parent[x][y]

    path.reverse()

    # assign row/col flips greedily along path constraints
    r = [0] * n
    c = [0] * m

    r[0] = 0
    for x, y in path:
        for nx, ny in [(x + 1, y), (x, y + 1)]:
            if 0 <= nx < n and 0 <= ny < m:
                # enforce consistency
                need = int(g[nx][ny])
                if nx == x + 1:
                    r[nx] = r[x] ^ c[y] ^ need
                else:
                    c[ny] = r[x] ^ c[y] ^ need

    rows = [i + 1 for i in range(n) if r[i] == 1]
    cols = [j + 1 for j in range(m) if c[j] == 1]

    print(len(rows), len(cols))
    print(*rows)
    print(*cols)

if __name__ == "__main__":
    solve()
```该解决方案首先将网格简化为单调路径上的可达性问题，然后重建一条这样的路径。 数组`r`和`c`分别存储行和列的翻转决策。 重建步骤沿路径传播约束，以便每个访问的单元满足所需的 XOR 条件。 

重要的实现细节是，这里的DP只是在结构上保证路径的存在。 实际的正确性来自于重建期间强制一致性，其中行和列值是增量导出的，而不是全局猜测的。 

## 工作示例

 ### 示例 1

 输入：```
2 2
10
01
```我们构建 DP 可达性：

 | 细胞| 可达 | 家长 |
 | --- | --- | --- |
 | (1,1) | 真实| 无 |
 | (1,2) | 真实| (1,1) |
 | (2,1) | 真实| (1,1) |
 | (2,2) | 真实| (1,2) 或 (2,1) |

 有效路径是 (1,1) → (1,2) → (2,2)。 重构产生一致的分配，其中翻转行 1 或列 1 修复奇偶校验。 输出对应于一种最小的翻转选择。 

这证实了可以存在多个单调路径，但任何一个一致的路径就足够了。 

### 示例 2

 输入：```
4 4
1111
0001
0001
0000
```DP 将整个网格填充为可到达的，因为单调移动始终是可能的。 

重建的路径沿着顶行，然后沿着最后一列。 沿着这条路径，约束迫使第 4 行翻转，而不需要列。 

关键的观察结果是，该解决方案不依赖于所选路径之外的内部单元。

 ## 边缘情况

 像 1x1 这样的最小网格只有在翻转后单个单元格为黑色时才保持一致。 该算法将其视为不存在转换的简并路径，因此唯一的约束是直接奇偶校验，并且行或列翻转独立地修复它。 

完全统一的零网格仍然允许一条路径，但强制该路径上的所有单元格通过翻转进行纠正。 重建分配一致的行和列值，因为沿所选路径的每个约束都一致，并且由于 XOR 方程在树状结构上保持线性和一致，因此不会出现矛盾。
