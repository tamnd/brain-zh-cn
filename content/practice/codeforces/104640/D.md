 ## 解决方案
 ## 问题理解

 我们被要求构造一个长度的字符串$n$覆盖小写拉丁字母。 输出字符串后，机器会以仅取决于出现的不同字母以及每个字母出现的次数的方式对其进行评估。 

机器首先计算字符串中使用了多少个不同的字母，称之为$k$。 然后它分配给那些$k$字母不同的整数权重$1$到$k$，选择使最终得分最小的作业。 分数是通过对字符串中每个字符出现的权重求和来计算的。 

因此，机器在分配权重的方式上实际上是对抗性的，但仍然受限于使用以下排列：$1 \ldots k$。 对于任何固定的字母频率多集，机器将为更频繁的字母分配较小的权重，因为这会减少总和。 

我们需要构造一个长度为的字符串$n$最大化最终的最小分数。 

输入尺寸允许$n \le 10^5$，所以我们需要一个$O(n)$或者$O(n \log n)$建造。 任何涉及搜索所有字符串或排列的事情都是立即不可行的，因为字符串的数量是$26^n$，甚至评估一种配置也需要对频率进行排序，如果重复这样做，速度会太慢。 

当所有字符都相同时，会出现微妙的边缘情况。 例如，如果$n = 3$字符串是`"aaa"`， 然后$k = 1$，重量被迫为$1$，得分为$3$。 如果我们使用`"abc"`， 然后$k = 3$，机器分配权重$1,2,3$以尽可能最好的方式，从而获得更大的分数。 这表明增加多样性往往会增加分数，但交互不仅仅是“最大化不同的字母”，因为机器总是为我们分配最佳权重。 

## 方法

 如果我们修复一个字符串，机器的行为是确定性的：它按频率降序对字母进行排序并分配权重$1$对于最常见的字母，$2$到第二频繁的，依此类推。 这是最小化总和的最佳选择，因为频繁出现的字母的每次出现都会乘以较小的权重。 

所以该值变为：$$T = \sum_{i=1}^{k} f_i \cdot w_i$$在哪里$f_i$是按非递增顺序排序的频率，并且$w_i$是一个排列$1 \ldots k$，最优分配为$w_i = i$。 

因此：$$T = \sum_{i=1}^{k} f_i \cdot i$$现在我们要选择一个字符串，这意味着我们选择最多 26 个字母的频率分布，总和为$n$，最大化该加权和。 

关键的观察结果是，指数越高，价值越高，因此我们希望大频率与大权重保持一致。 由于权重固定为$1$通过$k$, 最大化$k$仅靠这一点是不够的。 相反，我们希望构建频率，以便较大的索引获得较大的计数。 

然而，还有更强大的简化。 对于固定的$k$，我们能做的最好的就是将尽可能多的质量集中在最大的索引上$k$，因为该字母的权重最高。 最大化$T$，我们想要最大化$k$，自从增加$k$增加项数和最大可能权重。 

但$k \le 26$，因此我们始终可以使用最多 26 个不同的字母。 一旦我们使用了所有 26 个频率，剩下的唯一选择就是如何分配频率。 对于固定$k$，最大化：$$\sum i \cdot f_i$$我们应该将最大的频率分配给最大的指数。 这意味着我们希望以反向索引顺序严格增加频率分配。 

这成为一个经典的贪婪结构：分布$n$最多 26 个字母中的字符，例如：$$f_{26} \ge f_{25} \ge \cdots \ge f_1 \ge 0$$并最大化加权和。 最佳策略是使用全部 26 个字母（或者更少，如果$n < 26$）并尽可能均匀地分布，首先为索引较高的字母提供额外的字符。 

蛮力方法会尝试所有分区$n$最多分为 26 个部分并计算分数，分数为指数$n$。 贪婪分布将其简化为线性传递。

 ## 算法演练

 1. 确定不同字母的数量$k = \min(26, n)$。 这确保了我们永远不会超出字母表限制，同时尽可能地最大化多样性。 
2.初始化一个大小为26全0的数组，代表字母出现的频率`'a'`到`'z'`。 
3. 为每个第一个分配一个出现$k$字母。 这保证了我们使用尽可能多的不同字母。 
4. 设剩余字符为$rem = n - k$。 这些是必须分发的额外事件。 
5. 重复为以下字母分配一个额外字符`'a' + k - 1`向下至`'a'`，从最高索引到最低索引循环，直到分配完所有剩余字符。 这可确保较高权重的字母获得更高的频率，从而最大化加权和。 
6. 根据每个字母出现的频率重复每个字母来构建最终的字符串。 

### 为什么它有效

 最终得分取决于高频与高权重的匹配。 由于机器为频率较低的字母分配较高的权重，因此我们希望控制频率排序，以便映射将大量的计数强制到较大的权重上。 通过按相反的字母顺序分配额外的出现次数，我们确保较高索引的字母的出现频率永远不会低于较低索引的字母。 这使得机器的排序频率顺序与我们构造的顺序保持一致，从而最大化更高权重的贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    k = min(26, n)
    freq = [0] * 26
    
    for i in range(k):
        freq[i] = 1
    
    rem = n - k
    idx = k - 1
    
    while rem > 0:
        freq[idx] += 1
        rem -= 1
        idx -= 1
        if idx < 0:
            idx = k - 1
    
    res = []
    for i in range(26):
        res.append(chr(ord('a') + i) * freq[i])
    
    print("".join(res))

if __name__ == "__main__":
    solve()
```该解决方案首先通过为每个第一个字母分配一次出现来确保最大的不同字母$k$字母。 然后，从索引最高的活动字母开始，贪婪地分配剩余的字符，这会使字典顺序较靠后的字符中的频率向上偏置，这些字符对应于排序后机器分配的较高权重。 

构造阶段很简单：我们通过根据计算的频率重复每个字符来重建字符串。 

一个微妙的实现细节是循环递减`idx`，这确保了所有选定字母之间的公平分配，而不是在以下情况下完全挨饿较低的索引$rem$很大。

 ## 边缘情况

 对于$n = 1$，算法集$k = 1$, 赋值`a:1`，和输出`"a"`。 由于没有重新分配步骤`rem = 0`，所以结果基本上是正确的。 

为了$n = 26$，我们每次都击中完整的字母表一次。 机器分配权重$1$通过$26$，并且不会发生重新分配。 该结构产生均匀的频率，与预期的极值结构相匹配。 

为了$n > 26$，循环分布确保没有字母缺乏出现次数。 由于反向分配顺序，每个额外的字符首先增加较高索引字母的权重贡献。 追踪$n = 28$显示了 26 个字母的两个分布周期，保留了预期的频率排序并使机器的排序分配与我们的结构保持一致。
