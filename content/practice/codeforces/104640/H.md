 ## 解决方案
 ## 问题理解

 我们给定一个二进制字符串长度$n$我们必须构建。 字符串的成本是通过其所有连续的长度子字符串来定义的$k$。 每一个这样的长度——$k$块有一个预先分配的惩罚，字符串的总成本只是每个大小的滑动窗口的惩罚之和$k$。 

任务是选择字符串以使总成本尽可能小。 

重新表述这一点的一个有用方法是，我们沿着一个二进制字符串行走，其中每个位置都会影响滑动窗口成本，并且每个窗口仅取决于最后一个位置$k$位。 这立即表明问题在很大程度上是局部的，因为一旦最后一次$k-1$位是固定的，下一个决定准确地确定创建哪个新窗口。 

这些限制使得该结构具有可利用性。 字符串长度最大为1000，并且$k$最多 10 个，因此可能的数量$k$-bit windows 最多$2^{10} = 1024$。 这足够小，我们可以将每个窗口模式视为一种状态，而不是尝试对原始字符串进行推理。 

天真的方法会尝试所有$2^n$字符串并计算它们的成本，即使对于$n=1000$。 另一个稍微不那么幼稚的想法是贪婪地选择下一位，但这失败了，因为成本取决于重叠窗口，因此局部最优扩展可以阻止更好的未来转换。 

微妙的边缘情况是，早期的选择看起来相同，但后来产生不同的重叠。 例如，如果两个前缀以不同的结尾$(k-1)$-bit 后缀，即使它们当前的成本相同，它们也可能会强制实现完全不同的未来窗口。 任何忽略此后缀状态的解决方案都将默默地变得不正确。 

## 方法

 暴力法枚举每个长度的二进制字符串$n$，计算所有$n-k+1$长度子串$k$，并对它们的成本求和。 每次评估费用$O(nk)$，导致$O(2^n \cdot nk)$，这远远超出了可行性。 

关键的观察结果是，新角色的成本贡献仅取决于之前的角色$k-1$位。 一旦我们知道了最后一个$k-1$当前前缀的位，附加 0 或 1 确定性地创建一个新的长度 -$k$窗口，因此成本已知。 这将问题转化为在有向图中寻找最小成本路径，其中节点代表$(k-1)$位状态和边缘表示附加一位。 

每个状态最多有两个外出转换，状态总数为$2^{k-1}$，最多为 512。然后我们需要一条最短路径$n-k+1$转换，因为每个转换对应于在第一个转换之后生成一个新窗口$k$字符是固定的。 

这自然是通过前缀长度的动态规划来处理的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | DP 超过各州 |$O(n \cdot 2^k)$|$O(2^k)$| 已接受 |

 ## 算法演练

 我们将构建过程视为从左到右构建字符串，同时跟踪最后一个字符串$k-1$位作为一种状态。 

1.我们定义一个DP表，其中状态是字符串中的当前位置和最后一个位置$k-1$前缀的位。 这种状态就足够了，因为任何未来的窗口仅取决于这些位和下一个字符。 
2.我们初始化位置处的所有状态$k-1$零成本。 这反映了这样一个事实：在形成第一个完整窗口之前，我们可以自由选择任何前缀而无需支付任何惩罚。 
3. 我们迭代位置$k$到$n$。 在每一步中，我们都会考虑通过附加 0 或 1 来扩展当前前缀。 
4. 当我们追加一点时，我们形成一个新的$k$-长度窗口由前一个组成$(k-1)$-位状态加上新位。 我们加上相应的成本$d_t$对于那个窗口。 
5. 我们更新新状态的 DP 值，该值由最后一个状态定义$k-1$移位后的位，保持达到它的所有方法中的最小成本。 
6. 我们存储父指针来重建最终的字符串，记录先前的状态和所选的位。 
7.处理完所有位置后，我们选择该位置的最佳结束状态$n$并通过回溯存储的父字符串来重建字符串。 

其工作原理是，每个有效字符串都恰好对应于该状态图中的一条路径，并且每个转换成本恰好与原始字符串中的一个窗口匹配。 因此，DP 会探索所有有效的构造，而不会重复，并且在每一步中仅保留达到给定后缀状态的最便宜的方法，这已经足够了，因为未来的成本仅取决于该后缀。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    d = list(map(int, input().split()))
    
    if k == 1:
        # each bit independently contributes
        if d[0] <= d[1]:
            print("0" * n)
        else:
            print("1" * n)
        return

    m = 1 << (k - 1)
    INF = 10**18

    dp = [ [INF] * m for _ in range(n + 1) ]
    parent = [ [None] * m for _ in range(n + 1) ]

    for state in range(m):
        dp[k - 1][state] = 0

    for i in range(k - 1, n):
        for state in range(m):
            if dp[i][state] == INF:
                continue

            for b in (0, 1):
                if i + 1 >= k:
                    full = (state << 1) | b
                    cost = d[full]
                else:
                    cost = 0

                new_state = ((state << 1) & (m - 1)) | b

                if dp[i + 1][new_state] > dp[i][state] + cost:
                    dp[i + 1][new_state] = dp[i][state] + cost
                    parent[i + 1][new_state] = (state, b)

    best_state = min(range(m), key=lambda s: dp[n][s])

    res = []
    cur_state = best_state
    i = n

    while i > k - 1:
        prev_state, bit = parent[i][cur_state]
        res.append(str(bit))
        cur_state = prev_state
        i -= 1

    prefix = []
    state = cur_state
    for i in range(k - 1):
        prefix.append(str((state >> (k - 2 - i)) & 1))

    print("".join(prefix + res[::-1]))

if __name__ == "__main__":
    solve()
```DP表`dp[i][state]`存储构建长度前缀的最小成本`i`以特定结尾$(k-1)$-位后缀。 仅当积累了足够的字符时，转换才会显式构造新窗口。 位操作确保后缀更新和窗口形成保持 O(1)。 

重建步骤使用存储的父项向后移动，恢复位选择的确切序列。 长度的初始前缀$k-1$直接从最终状态导出。

 ## 边缘情况

 当$n < k$，不会形成完整的窗口，因此每个字符串的成本为零。 DP初始化位置处的所有状态$k-1$，因此即使我们从不“支付”窗口成本，重建仍然会产生有效的任意字符串。 该算法自然会处理这个问题，因为没有任何转换会触发成本查找。 

当所有$d_t$值相等，每个字符串具有相同的成本。 DP 可以选择任何路径，但基于状态的公式确保了一致性，因为它仍然传播有效的转换。 重建只是返回许多最佳路径之一，而不依赖于任意顺序。
