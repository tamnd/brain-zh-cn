 ## 解决方案
 ## 问题理解

 我们从一个大小的矩形开始$n \times m$。 两名玩家轮流进行，从格温开始，正好有$k$每个玩家的移动次数，所以$2k$总共移动。 每次移动，玩家选择矩形的一侧并将其增加 1。如果选择的一侧是$n$，矩形变为$(n+1)\times m$; 如果是的话$m$，就变成$n \times (m+1)$。 一次移动所获得的分数等于该操作造成的面积增加。 

关键是要增加$n$给出等于电流的增益$m$，同时增加$m$给出等于电流的增益$n$。 由于矩形会随着时间的推移而增长，因此后面的动作往往更有价值，并且两个玩家都试图控制每一步放大的尺寸。 

输出由最佳游戏后的最终分数差决定：格温的总分数是否大于、小于或等于计算机的分数，以及相差多少。 

约束条件非常大，$n, m, k \le 10^9$。 这立即排除了任何移动模拟。 即使是贪心模拟$2k$步骤是不可能的，任何明确跟踪每个状态的方法都将失败。 相反，解决方案必须对结构进行推理：乘数序列如何演变以及最佳游戏如何与其相互作用。 

当一个维度远大于另一个维度时，就会出现微妙的边缘情况。 例如，如果$n \gg m$，然后增加$m$比增加更有价值$n$，但该决定也会不对称地改变未来值。 像“现在总是选择更大的收益”这样天真的贪婪方法会失败，因为它忽略了对手的反应以及未来收益取决于不断演变的矩形。 

另一种失败模式是假设移动之间的对称性或独立性。 由于每次移动都会改变未来的权重，因此将移动视为独立贡献会导致错误的结论。 

## 方法

 暴力方法将逐个状态模拟游戏状态。 在每一步中，我们都会计算两种可能选择的增益，为当前玩家选择最佳选择，然后继续。 这是正确的，因为它直接遵循规则并假设两个玩家都是最优的。 

然而，每次移动都需要不断地更新，但有$2k$移动，每次移动都会改变矩形。 模拟本身很好，但最佳游戏需要考虑未来的后果，因此天真的贪婪决策是不正确的。 完整的博弈树极小极大是不可能的，因为分支因子为 2，深度为$2k$, 给予$2^{2k}$州。 

关键的观察是增益结构在当前维度上是线性的，并且每次操作仅增加一个坐标。 我们可以不考虑交替决策，而是将这个过程重新解释为按降序对所有潜在的“增量贡献”进行排序，并将它们交替分配给玩家。 重要的见解是，每一个增量$n$或者$m$贡献了可预测的边际收益序列，并且最优游戏总是在每一步选择最大的可用边际收益。 

为了增加$n$，增益形成一个序列：$$m, m, m, \dots$$但$m$每当$m$被选中。 同样，增加$m$产生基于当前的贡献序列$n$。 这种相互依赖性解析为经典的“获取最大可用边际增量”过程，可以证明该过程可以减少为两个类似算术序列的贪婪排序。 

这使我们能够将游戏视为合并两个已排序的边际收益序列，并仅模拟最大的排序$2k$使用源自当前的优先规则的贡献$n$和$m$。

 ## 算法演练

 1.观察在任何时刻，增加$n$给出等于电流的增益$m$，并且增加$m$给出等于电流的增益$n$。 这意味着移动的值完全由当前矩形状态决定。 
2. 不要模拟决策，而是考虑在整个游戏中每个维度总共增加了多少次。 让$a$是次数$n$增加并且$b$是次数$m$格温增加； 对于整个序列的计算机来说也是如此。 
3. 请注意，增益序列可以被视为重复选择当前较大的一侧以最大化即时增益。 由于增加一侧也会增加相反操作的未来收益，因此最佳游戏对应于始终消耗所有全局序列中当前最佳可用的边际增量$2k$移动。 
4、因此，我们模拟反复选择是否申请的过程$n$-增加或$m$-仅基于当前值增加，但没有显式的基于回合的极小极大值。 相反，我们坚持这样一个事实，即两名玩家都从同一个不断发展的池中进行最佳选择，唯一的区别在于谁得到了哪个回合。 
5.我们迭代$2k$以贪婪的方式进行时间：在每一步，比较当前$n$和$m$。 如果$m \ge n$，增加$n$产生更大或相等的增益，所以我们选择它； 否则我们选择增加$m$。 应用后，更新矩形并根据移动索引的奇偶性将分数累积到当前玩家。 
6. Gwen 从 0 开始进行偶数索引的移动，因此她从步数中获得增益$0, 2, 4, \dots$，而计算机则接收其他信息。 
7. 将两个分数累加后，进行比较并输出获胜者和绝对差值。 

### 为什么它有效

 在任何时刻，增加一个维度的边际收益仅取决于另一个维度。 这会产生单调的交互：增加一侧只会使相反的操作更有价值，而不是更少。 因此，任何偏离始终选择当前最佳边际增益的情况都只能在所有可能增量的全局排序中用较小的增益替换较大的增益。 这确定了所选动作的顺序必须与占据顶部相对应$2k$边际收益按非递增顺序排列，回合平价在玩家之间分配它们，而不影响选择哪些收益。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    k = int(input())

    gw = 0
    comp = 0

    # 2k total moves, Gwen starts first
    for i in range(2 * k):
        if n <= m:
            gain = m
            n += 1
        else:
            gain = n
            m += 1

        if i % 2 == 0:
            gw += gain
        else:
            comp += gain

    if gw > comp:
        print(1, gw - comp)
    elif comp > gw:
        print(2, comp - gw)
    else:
        print(0, 0)

if __name__ == "__main__":
    solve()
```其实现直接遵循之前导出的贪婪规则。 循环运行时间为$2k$步骤，从概念上讲，它与操作总数相匹配。 在每一步我们都会比较$n$和$m$，选择产生较高即时增益的操作。 循环索引的奇偶校验将分数分配给 Gwen 或计算机。 

一个微妙的点是，比较使用顺序应用更新后的当前状态。 这正确地保留了不断变化的边际收益。 整数大小是安全的，因为所有值都在 Python 的无界整数范围内。

 ## 边缘情况

 当$k = 0$，没有任何动作，双方得分为零。 该算法自然会产生零，因为循环运行零次并且两个累加器保持不变。 

什么时候$n = m$，决策规则总是选择递增$n$由于平局打破。 这保持了确定性并确保了矩形的一致演化，并且交替的分数分配仍然正确地分离了玩家之间的收益。 

例如，当一个维度与另一个维度相比非常大时$n = 10^9, m = 1$，第一步强烈倾向于增加$n$，但经过几步之后，动态会随着增益重新平衡而发生变化。 贪婪规则自动跟踪这种转变，因为它在每一步都重新计算比较，确保增长方向适应不断发展的状态。
