 ## 解决方案
 ## 问题理解

 我们得到一个图，其中顶点代表宇宙，边代表门户。 每个门户都有最低能量要求，这意味着只有当您当前的能量水平至少达到给定阈值时，您才能穿越该门户。 能量从零开始。 

每当你第一次进入一个宇宙时，你的能量就会永久增加分配给该宇宙的值。 你从一个固定的宇宙开始，从那里你可以沿着任何可用的门户序列旅行，随着你发现新的宇宙，逐渐增加你的能量。 

任务是确定在此过程结束时可以积累的最大总能量，这相当于在这些约束下可到达的所有宇宙的值的总和。 

约束大小立即排除任何重复模拟所有可能路径的解决方案。 拥有多达十万个宇宙和二十万个门户，任何探索“当前节点加访问集”形式的状态的方法都是不可行的。 即使是扩展状态上的最短路径式算法也会发生组合爆炸，因为能量取决于节点被访问的顺序。 

一个微妙的困难来自循环依赖。 一个传送门可能在早期无法使用，因为它的要求太高，但是一旦我们从其他宇宙收集到足够的能量，它就变得可用并可能解锁更多的宇宙。 这意味着可达性不是静态的。 

一个常见的陷阱是将其视为标准可达性问题并简单地忽略权重，或者在具有边权重的节点上运行 Dijkstra。 这失败了，因为约束不是沿着边缘相加的，它是一个全局演变的阈值。 

当提供大能量的节点只能通过高阈值边缘访问时，会出现另一种故障情况。 永远不会重新访问被阻止的边的天真的贪婪遍历将会错过这些节点，即使它们对于解锁图的其余部分至关重要。 

## 方法

 蛮力的想法是模拟所有可能的访问宇宙的方式。 从初始宇宙开始，在每一步中，我们都会尝试每个当前可到达的门户，如果允许则通过它，然后递归地继续。 每个国家都需要记住访问过哪些宇宙以及当前的能量。 分支因子增长很快，因为每个新宇宙都会改变能量并改变未来的可达性。 在最坏的情况下，这会导致对节点子集的指数探索。 

关键的观察是，对于解锁边缘唯一重要的是当前的总能量，它恰好是已访问节点的值的总和。 一旦节点变得可达，访问它只会增加能量，从而只能解锁更多的边。 没有任何机制会降低灵活性。 

这种单调结构表明了一个不断增长的可达集合过程。 如果我们知道最终的可达节点集，它们的总和将决定最终的能量，而相同的能量决定哪些边应该可用。 这种循环依赖可以通过使用当前能量估计重复扩展可达到的范围来解决。 

我们可以维护已经满足要求的边集，并将它们的端点合并到连通分量中。 一旦包含起始节点的组件变得足够大，其总节点值就会增加能量，可能会解锁更多边，这可能会将更多组件合并到起始组件中。 这创建了一个自然的定点过程。

 | --- | --- | --- | --- |
 | 详尽的状态模拟 | 指数| 指数| 太慢了|
 | 使用 DSU + 迭代激活进行阈值传播 | O((n + m) log m) 或 O((n + m) α(n)) | O(n + m) | 已接受 |

 ## 算法演练

 我们在宇宙上维持一个不相交的集合联合结构，并且随着能量的增加，我们逐渐激活门户。 每次激活都可能合并组件，并且只有包含起始宇宙的组件才会对未来的能量增长做出贡献。 

1. 初始化所有宇宙上的不相交集并集。 每个 Universe 都以自己的组件开始，每个组件都存储其节点值的总和。 起始宇宙立即被认为已被访问，因此初始能量等于其值。 
2. 按所需能量升序对所有门户进行排序。 这使得我们能够在能量变得足够大时逐步激活它们。 
3. 将指针放在已排序的门户上。 重复激活所有需求最多为当前能量的传送门。 激活意味着在 DSU 中执行联合操作。 
4. 激活所有当前可用的门户后，找到包含起始宇宙的组件。 计算其总和，它代表当前解锁结构可实现的最佳能量。 
5. 如果这个新能量大于之前的值，则更新能量并重复该过程，因为更高的能量可能会解锁之前被阻止的其他门户。 
6. 当对门户列表的完全传递未激活任何新门户或未增加可到达组件大小时停止。 

关键思想是，每次能量增加时，我们都会扩展可用边缘的集合，并且每次这样的扩展只能增加起始组件的连接性，而永远不会减少它。 

### 为什么它有效

 在任何时刻，由所有激活的边构建的 DSU 准确地表示在所有使用的门户最多需要当前能量的约束下的连接性。 该结构中起始节点的组成部分代表了该能级下真正可达的所有宇宙。 由于能量被定义为访问过的宇宙的值的总和，因此从该分量重新计算它与当每个宇宙变得可达时访问该宇宙一次的过程是一致的。 由于能量和激活的边集都只会随着时间的推移而增长，因此该过程必须收敛到一个固定点，在该固定点上无法激活新的边，并且没有新的节点可以加入起始组件。 该固定点就是最大可达到的能量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n, val):
        self.parent = list(range(n))
        self.size = [1] * n
        self.sum = val[:]  # component sums

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.size[a] < self.size[b]:
            a, b = b, a
        self.parent[b] = a
        self.size[a] += self.size[b]
        self.sum[a] += self.sum[b]
        return True

def solve():
    n, m, s = map(int, input().split())
    s -= 1

    a = list(map(int, input().split()))

    edges = []
    for _ in range(m):
        u, v, w = map(int, input().split())
        edges.append((w, u - 1, v - 1))

    edges.sort()

    dsu = DSU(n, a)

    energy = a[s]
    prev_energy = -1

    i = 0
    while energy != prev_energy:
        prev_energy = energy

        while i < m and edges[i][0] <= energy:
            _, u, v = edges[i]
            dsu.union(u, v)
            i += 1

        root = dsu.find(s)
        energy = dsu.sum[root]

    print(energy)

if __name__ == "__main__":
    solve()
```DSU 在每个组件内存储连接性和值的总和，这避免了每次合并后重新计算总和。 指针`i`确保每条边仅处理一次，因为一旦满足其阈值，它就会永远保持活动状态。 

外循环至关重要，因为激活边缘可以放大起始分量，从而增加能量，从而可以解锁更多边缘。

 ## 边缘情况

 关键的边缘情况是所有边缘的阈值都大于初始能量。 在这种情况下，算法不会正确执行超出起始节点的并集，并且答案仍然是起始宇宙的值。 

当单个低阈值边缘连接到大的高价值节点链时，会发生另一种边缘情况。 该算法仅在能量充分增加后才解锁链，迭代循环确保一旦建立第一个连接，后续的扩展就会自然触发，而无需手动重新访问之前的逻辑。 

最后一个微妙的情况是组件通过稍后变得活跃的边间接合并。 DSU 干净地处理了这个问题，因为一旦激活边缘，其效果是永久性的，并且以后能量的增加不需要重新处理旧边缘或逆转决策。
