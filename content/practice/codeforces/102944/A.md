 ## 解决方案
 ## 问题理解

 我们每天都会收到一份简单的珍珠奶茶店顾客到达记录。 每天有一定数量的顾客，每次顾客总数达到固定值的倍数`k`，该顾客收到一杯免费饮料。 一天内对各个客户的计数是连续的，但是当我们只关心是否至少有一个倍数时，每天都会重置`k`那天受到打击。 

任务不是在全局范围内模拟奖励，而是独立检查每一天，并确定该天是否至少包含一个客户，其在到货订单中的位置可被`k`。 如果每天至少有一个这样的客户，我们输出`"awesome"`。 否则，我们输出不存在此类客户的第一天索引。 

输入尺寸小：两者`k`和`D`最多 1000 个，每天最多有 2000 个客户。 这立即排除了对线性扫描之外的高级数据结构或优化的任何需求。 即使是简单的每日计算也完全在限制范围内，因为总工作量最多约为 200 万次检查。 

当一天的时间少于`k`顾客。 在这种情况下，没有倍数`k`在这个范围内，所以那天的答案自动是“没有免费饮料”。 例如，如果`k = 10`和`a_i = 5`，那天没有贡献任何东西。 一个粗心的实现，只检查是否`a_i % k == 0`这是不正确的，因为重要的是恰好拥有 10、20、30 个客户，而不仅仅是总数的整除性。 

另一个极端情况是当`k = 1`。 每个客户都有资格，因此每天至少有一个客户必须成功，即使零客户的日子仍然提供零免费饮料，如果存在这样的一天，就意味着失败。 

## 方法

 一个直接的方法是模拟每一天。 对于某一天`i`和`a_i`客户，我们迭代客户位置从 1 到`a_i`并检查任何位置是否可以被整除`k`。 如果我们找到至少一个这样的位置，则该日有效； 否则无效。 

这是可行的，因为问题简化为检查整数是否存在`x`在`[1, a_i]`这样`x % k == 0`。 然而，这种强力视图有点浪费：即使我们只关心是否有多个客户，它也会扫描所有客户。`k`存在于范围内。 

关键的观察结果是`k`出现在职位`k, 2k, 3k, ...`。 所以一天包含一杯免费饮料当且仅当`a_i >= k`。 一次`a_i`至少是`k`， 这`k`当天第 - 位顾客存在并保证至少一份免费饮料。 如果`a_i < k`, 不存在的倍数`k`根本不。 

这将每天减少为一次比较，将整个问题转化为对阵列的简单扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(D * 最大 a_i) | O(1) | O(1) | 已接受 |
 | 直接观察| O(D) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们逐一处理每一天，同时跟踪每一天是否满足条件。 

1. 阅读`k`和`D`，然后读取每日客户计数数组。 这提供了独立评估每天所需的所有信息。 
2. 每天`i`，检查是否`a_i >= k`。 该条件捕获是否至少有一个倍数`k`当天的客户仓位中存在。 原因是第一个符合资格的客户始终是`k`第-次到达。 
3.如果`a_i < k`，我们立即知道这一天没有免费饮料，我们可以停止处理以后的日子。 我们输出`i`作为第一个失败的一天。 
4. 如果我们完成所有天而没有遇到失败，我们输出`"awesome"`。 

### 为什么它有效

 一天之内，免费饮料恰好发生在以下位置的倍数处：`k`。 最小的位置是`k`，因此任何合格客户的存在仅取决于该天是否至少有`k`顾客。 这创建了一个单调属性：once`a_i`十字架`k`，保证当天成功，以下`k`，成功是不可能的。 由于每一天都是独立的，因此按顺序扫描可以正确识别第一个故障。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    k, D = map(int, input().split())
    a = list(map(int, input().split()))

    for i in range(D):
        if a[i] < k:
            print(i + 1)
            return

    print("awesome")

if __name__ == "__main__":
    solve()
```该解决方案读取参数并迭代每天的客户计数一次。 核心决定是比较`a[i] < k`，它直接编码是否是任意倍数`k`存在于当天的序列中。 提前返回可确保我们在第一天违规时停止。 

一个常见的错误是尝试模拟客户指数或检查可分条件`a[i]`本身。 正确的解释是关于当天内的头寸，而不是总数的属性。

 ## 边缘情况

 当`k = 1`，每个客户都是`k`，因此任何一天至少有一名顾客。 该算法可以正确处理这个问题，因为`a_i < 1`仅当`a_i = 0`，所以只有空的日子会失败。 

当一天正好`k - 1`客户，没有倍数`k`在范围内。 例如，`k = 10`和`a_i = 9`导致失败。 支票`a_i < k`无需显式迭代即可正确捕获此边界。 

当一天正好`k`客户，`k`-第一个客户的存在并保证成功。 例如，`k = 10`,`a_i = 10`立即成功，因为位置 10 是 10 的倍数。
