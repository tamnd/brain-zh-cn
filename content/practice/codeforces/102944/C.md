 ## 解决方案
 ## 问题理解

 我们得到一个代表商店平面图的矩形网格。 每个单元格都包含一个方向字符，其作用类似于确定性指令：如果客户站在该单元格上，他们就会根据箭头向北、南、东或西移动一步。 商店在网格内没有墙壁，但离开网格就意味着退出商店。 

从每个单元格中，只有一次向外移动，因此整个网格变成一个有向图，其中每个节点的出度均为 1。从任何单元格开始，客户重复遵循这些箭头，直到它们掉出网格或在网格内永远移动。 

任务是计算有多少个起始单元不允许顾客离开网格。 用图术语来说，我们想要有向路径永远不会到达边界出口的节点数量，这意味着路径最终必须进入完全留在网格内的循环。 

网格大小可以达到 2000 到 2000，最多可容纳 400 万个节点。 任何独立模拟每个起始单元并一直走到终止的解决方案都可能会重复访问许多节点，并在最坏的情况下退化为二次行为，这太慢了。 预期的解决方案必须确保每个单元都被处理恒定的次数。 

当整个网格形成一个大循环时，会出现微妙的边缘情况。 在这种情况下，每个细胞都会被计数。 另一种边缘情况是当所有箭头在边界附近向外指向时，导致许多单元立即退出，因此大多数节点被排除。 仅检查循环而不标记出口可达性的简单方法可能会错误地计算长链后最终流出网格的节点。 

## 方法

 直接模拟方法从每个单元开始并重复遵循该方向，直到它退出网格或检测到先前看到的单元。 这在概念上很简单，因为每条路径都是确定性的。 如果我们能够独立地遍历每条路径，那么正确性就很简单：我们只需根据步行是否逃脱将每个起点分类为安全或不安全。 

问题在于路径严重重叠。 单个单元可以是许多起始路径的一部分，并且如果不缓存结果，我们就会一次又一次地重新计算长链。 在最坏的情况下，考虑一个形状像一条长蛇的网格，其中每条路径在循环之前都会合并成一条长走廊。 每个起始单元可以几乎遍历整个结构，每次起始产生 O(NM) 工作量，并导致总复杂度为 O((NM)^2)。 

关键的观察结果是，该结构是一个函数图，这意味着每个节点都有一个出边。 在此类图中，每个节点要么进入循环，要么最终流入最终结果（此处为退出网格）。 如果我们反转视角，而不是从每个节点向前模拟，我们可以从已知的丢失状态（即退出转换）向后传播信息。 然而，出口不是图中的节点，因此我们将任何超出网格的移动视为最终失败状态，并从那里向后传播。 

我们可以通过跟踪保证通向外部的节点来对此进行建模。 任何下一步走到外面的节点都会立即不安全。 那么，如果一个节点的下一个状态是不安全的，那么该节点也是不安全的。 这是函数图上的标准反向传播，可以使用带有记忆或迭代处理状态的 DFS 来完成：未访问、访问、安全退出和捕获。 

最终计数只是未标记为到达出口的节点数量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O((NM)^2) | O((NM)^2) | O(NM) | 太慢了 |
 | 具有记忆功能的 DFS | O(NM) | O(NM) | 已接受 |

 ## 算法演练

我们将每个单元视为有向图中的一个节点，具有一个出边，除非它指向外部。 

1. 对于每个单元格，我们定义一个递归函数，用于确定从该单元格开始是否通向网格之外。 

如果路径最终离开网格，则该函数返回 true。 
2. 我们维护一个具有三个值的状态数组：未访问、访问和带有布尔结果的访问。 

这可以防止重新计算并在遍历期间检测循环。 
3. 当我们进入一个单元格时，我们立即根据它的方向计算它的下一个位置。 

如果下一个位置在网格之外，我们将当前单元格标记为能够退出并返回 true。 
4. 如果下一个位置在网格内并且已经计算出来，我们将重用其存储的结果。 

这是将重复路径探索分解为恒定时间查找的关键优化。 
5. 如果当前正在访问下一个位置，则我们找到了一个循环。 

由于我们正在尝试确定是否可以退出网格，因此单独的循环并不能保证退出。 我们认为这不会导致退出这条道路。 
6. 计算节点的结果后，我们存储它并将其返回给调用者，以便所有传入路径重用它。 
7. 最后，我们计算 DFS 结果表明它们从未到达出口的所有单元格。 

### 为什么它有效

 每个单元格都只有一个出边，因此网格形成一个有向图，其中每个组件要么是进入循环的树，要么是通向外部的链。 具有记忆功能的 DFS 仅根据其后继节点为每个节点分配最终布尔值。 由于结果被缓存，一旦节点被分类，到达该节点的所有路径都会继承相同的分类。 循环的处理是一致的，因为循环中涉及的节点最终将重新访问“访问”状态，从而防止无限递归并确保它们被归类为非退出，除非循环连接到出口，根据限制在网格内的循环的定义，这是不可能的。 

## Python 解决方案```python
import sys
sys.setrecursionlimit(10**7)
input = sys.stdin.readline

N, M = map(int, input().split())
grid = [input().strip() for _ in range(N)]

# 0 = unvisited, 1 = visiting, 2 = done
# exit[i][j] = True if starting here eventually leaves grid
state = [[0] * M for _ in range(N)]
exit_reachable = [[False] * M for _ in range(N)]

dirs = {
    'N': (-1, 0),
    'S': (1, 0),
    'W': (0, -1),
    'E': (0, 1)
}

def dfs(r, c):
    if state[r][c] == 2:
        return exit_reachable[r][c]
    if state[r][c] == 1:
        return False

    state[r][c] = 1
    dr, dc = dirs[grid[r][c]]
    nr, nc = r + dr, c + dc

    if nr < 0 or nr >= N or nc < 0 or nc >= M:
        exit_reachable[r][c] = True
        state[r][c] = 2
        return True

    res = dfs(nr, nc)
    exit_reachable[r][c] = res
    state[r][c] = 2
    return res

total_escape = 0
for i in range(N):
    for j in range(M):
        if dfs(i, j):
            total_escape += 1

print(N * M - total_escape)
```该解决方案使用具有网格单元记忆功能的 DFS。 每个单元最多以有意义的方式被访问一次，因为一旦其状态变为“完成”，就永远不会重新计算。 递归遵循由网格字符定义的有向边。 边界检查在计算下一个单元后立即处理，这很重要，因为在检查边界之前尝试递归会导致索引错误或不正确的状态传播。 

周期检测依赖于“访问”标记。 如果我们重新进入当前递归中的节点，我们会停止并返回 False，这意味着我们不会通过该路径到达出口。 由于每个节点都只有一个传出边缘，因此遇到环路意味着路径被困在该环路内，除非它连接到出口，而一旦完全进入环路，这种情况就不会发生。 

## 工作示例

 ### 示例 1

 输入：```
3 3
SSW
NSE
NWS
```我们追踪了一些有代表性的细胞。 

| 细胞| 下一步| 引到外面？ | 最终结果|
 | ---| ---| ---| ---|
 | (0,0) | (0,0) | (1,0)| 取决于 | 通过 DFS | 计算
 | (1,1) | (1,2) | 里面 | 跟随链条|
 | (2,2) | (2,1) | 里面 | 进入周期|

 DFS 在右下区域发现了一个循环，这意味着有多个节点被困住。 只有链最终到达边界出口的节点才会被排除在答案之外。 传播后，算法对所有不退出的节点进行计数。 

这个例子表明，即使局部运动似乎向内发展，全球结构仍然可以形成循环。 

### 示例 2

 输入：```
3 4
SWNW
EEEN
NWWW
```这里许多箭头推向边缘，导致频繁退出。 

| 细胞| 下一页 | 可到达出口 |
 | ---| ---| ---|
 | (0,0) | (0,0) | (0,1)| 最终退出|
 | (1,1) | (1,2) | 快速退出 |
 | (2,3) | 外面| 立即 |

 DFS 快速将到达边界的路径标记为真实。 这会向后传播，因此通向边缘的整个链都被标记为可逃逸。 

这显示了记忆效应：一旦知道退出路径，大区域就会崩溃为单个计算结果。

 ## 边缘情况

 最小网格（例如指向外部的单个单元格）演示了立即终止的基本情况。 DFS 在边界检查后立即返回 true，并且该单元被视为可逃逸，因此最终答案是零捕获单元。 

像“E W”这样的两个单元循环显示了访问状态如何防止无限递归。 从任一单元开始，DFS 进入另一个单元，然后检测到对访问节点的重新访问并返回 false。 两个细胞都被标记为被捕获，从而产生正确的计数为 2。 

2x2 网格中的完整循环测试算法是否可以处理多个互连的循环。 每个单元最终都会循环回到循环检测情况，因此它们都不会被标记为退出。 结果是四个被捕获的细胞，证实了较大循环结构的一致性。
