 ## 解决方案
 ## 问题理解

 我们得到了一小部分正整数。 从这个集合中，我们考虑每个可能的非空子集，并计算该子集中数字的最大公约数。 如果该 gcd 恰好等于 1，则子集被视为“有效”。任务是计算存在多少个这样的有效子集。 

输入大小很小，最多 100 个数字，但每个数字可以大到 10^9。 大小约束是关键信号：原则上可以枚举所有子集，因为 2^100 太大，但 100 足够小，允许对从数字（尤其是除数）导出的值进行压缩或包含排除的指数方法。 

一个天真的解释会尝试迭代所有子集并直接计算 gcd。 这已经表明了核心困难：子集空间是指数的，但 gcd 运算具有可以利用的结构。 

当所有数字共享一个大于 1 的公因子时，就会出现微妙的边缘情况。例如，如果所有数字都是偶数，则每个子集的 gcd 至少为 2，因此答案必须为 0。任何未正确强制 gcd 折叠行为的方法都会错误地计算实际上从未达到 1 的子集。 

当数组包含等于 1 的单个元素时，会出现另一种极端情况。唯一的子集是 {1}，其 gcd 为 1，因此答案为 1。这是任何方法都必须满足的最低健全性检查。 

## 方法

 蛮力的想法很简单。 我们迭代数组的每个非空子集，通过折叠其元素来计算其 gcd，如果结果等于 1，则增加答案。这是正确的，因为它直接遵循问题的定义。 

然而，子集的数量是2^N。 当 N = 100 时，这大约是 10^30 个子集，这远远超出了任何计算限制。 即使 gcd 计算速度很快，子集的绝对数量也使得这种方法变得不可能。 

关键的观察结果是 gcd 结构与除数而不是子集相互作用良好。 我们不是在构造后根据子集的 gcd 值对子集进行分组，而是相反：固定一个值 d 并计算有多少个子集的所有元素均可被 d 整除。 任何这样的子集都有 gcd 可被 d 整除，如果我们仔细考虑过度计数，我们可以恢复 gcd 完全等于 d 的子集的确切数量。 这是除数的标准包含-排除思想，但在这里它变得特别干净，因为 N 很小而值很大。 

我们继续计算，对于每个整数 d，有多少个数组元素可以被 d 整除。 如果 k 个元素可被 d 整除，则存在 2^k - 1 个仅由 d 的倍数组成的非空子集。 让 f(d) 表示该计数。 然而，f(d) 计算所有 gcd 为 d 的倍数而不完全等于 d 的子集。 然后，我们使用除数上的降序筛减去 d 的倍数的贡献，产生 gcd 恰好为 d 的子集的确切数量。 

最后，我们对 d = 1 对应的值求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N·2^N) | O(N·2^N) | O(1) | O(1) | 太慢了 |
 | 除数 DP + 包含-排除 | O(N √A + D log D) | O(N √A + D log D) | O(D) | 已接受 |

 这里 D 是输入值中出现的不同除数的数量，它很小，因为我们只生成每个数字的除数。 

## 算法演练

 我们完全避免枚举子集，而是按除数类进行计数。

1. 对于数组中的每个数字，枚举其所有除数。 对于每个除数 d，将频率计数器 cnt[d] 加 1。此步骤构建从除数到可被除数整除的数组元素的映射。 这样做的原因是，任何元素均可被 d 整除的子集仅由这些 cnt[d] 元素形成。 
2. 对于 cnt 中出现的每个除数 d，计算由这些元素形成的非空子集的数量，即 f(d) = 2^{cnt[d]} - 1。这表示所有元素都是 d 的倍数的子集，这意味着它们的 gcd 至少为 d。 
3. 将所有除数按降序排列。 我们希望消除倍数中的过度计算。 如果一个子集的 gcd 恰好等于 d 的较大倍数，则它被错误地包含在 f(d) 中，因此我们减去这些贡献。 
4. 按降序处理除数。 维护一个初始设置为 f(d) 的数组 g[d]。 对于每个 d，迭代所有倍数 m = 2d、3d、...，并从 g[d] 中减去 g[m]。 这确保了处理后，g[d]代表gcd恰好为d的子集。 
5. 最终答案为g[1]。 

### 为什么它有效

 每个非空子集都有一个唯一的 gcd 值。 对于任何固定的 d，f(d) 计算 gcd 为 d 倍数的所有子集。 这些子集按其精确的 gcd 值进行划分，所有这些值都是 d 的倍数。 减法步骤实际上是除数格上的莫比乌斯求逆，确保每个子集都被准确地分配给其真实的 gcd 值一次。 因为除数关系形成偏序，所以从大到小处理可以保证当我们进行减法时，所有较大的贡献都已经确定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def get_divisors(x):
    divs = []
    i = 1
    while i * i <= x:
        if x % i == 0:
            divs.append(i)
            if i * i != x:
                divs.append(x // i)
        i += 1
    return divs

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = (res * a) % MOD
        a = (a * a) % MOD
        e >>= 1
    return res

n = int(input())
a = list(map(int, input().split()))

cnt = {}
for x in a:
    for d in get_divisors(x):
        cnt[d] = cnt.get(d, 0) + 1

vals = sorted(cnt.keys(), reverse=True)

f = {}
g = {}

for d in vals:
    c = cnt[d]
    f[d] = (modpow(2, c) - 1) % MOD
    g[d] = f[d]

for d in vals:
    for m in vals:
        if m > d and m % d == 0:
            g[d] = (g[d] - g[m]) % MOD

print(g.get(1, 0) % MOD)
```该解决方案首先构建除数频率。 对于每个数字，我们在 O(√aᵢ) 中枚举其除数，这足以满足 aᵢ 最大 10^9 的要求。 这是直接接触原始输入值的唯一步骤。 

接下来，我们为每个除数计数计算 2^k - 1。 这表示仅由可被该除数整除的数字组成的所有子集。 模幂运算是必要的，因为 k 可达 100。 

最后，我们通过减去较大 gcd 值的贡献来纠正过度计数。 除数上的嵌套循环是可以接受的，因为在这些约束下，所有数字中不同除数的数量在实践中仍然很小。 

## 工作示例

 ### 示例 1

 输入：```
3
2 3 5
```每个数字都是素数且不同，因此只有单个子集对 gcd 1 很重要。 

| 步骤| 碳纳米管| f(d) | f(d) | 克(d) |
 | ---| ---| ---| ---|
 | d = 5 | 1 | 1 | 1 |
 | d = 3 | 1 | 1 | 1 |
 | d = 2 | 1 | 1 | 1 |
 | d = 1 | 3 | 7 | 4 |

 最终值 g(1) = 4 对应于除 gcd 不为 1 之外的所有子集。有效子集是除空集和 gcd >1 之外的所有子集。 

该迹线显示了即使所有数字都是成对互质的，包含-排除如何分离 gcd 类。 

### 示例 2

 输入：```
3
2 4 8
```所有数字都是 2 的幂，因此没有一个子集的 gcd 为 1。 

| 步骤| 碳纳米管| f(d) | f(d) | 克(d) |
 | ---| ---| ---| ---|
 | d=8 | 1 | 1 | 1 |
 | d = 4 | 2 | 3 | 2 |
 | d = 2 | 3 | 7 | 4 |
 | d = 1 | 3 | 7 | 0 |

 这里，每个子集都在某个 2 次幂的 gcd 类别下进行计数，并且所有内容在 d = 1 时都抵消，从而确认没有子集达到 gcd 1。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N √A + D²) | 除数枚举加除数倍数修正|
 | 空间| O(D) | 除数计数和 DP 表的存储 |

 约束 N ≤ 100 使除数枚举变得微不足道，并且 D 仍然易于管理，因为每个数字最多贡献约 100 个除数。 这使得该解决方案能够轻松地适应限制。

 ## 边缘情况

 当所有值共享一个大于 1 的公约数时，每个子集的 gcd 至少是该除数。 用于输入`2 2 2`，除数计数产生 f(2)=7 和 f(1)=7，但减法会删除 d=1 处的所有贡献，留下零个有效子集。 该算法在达到 1 之前正确地将所有内容折叠到更高的 gcd 类别中。 

当数组包含单个元素等于 1 时，cnt[1]=1 且 f(1)=1。 不存在更大的除数可以从中减去，因此直接 g(1)=1。 该算法无需任何特殊大小写即可处理此问题，因为除数枚举自然会为每个数字包含 1。
