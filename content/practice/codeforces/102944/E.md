 ## 解决方案
 ## 问题理解

 我们得到一个大小为 100 x 100 的矩形网格。每个单元格代表一个座位，并且已被涂成绿色、蓝色或左中性。 绿色和蓝色是固定的，而中性座位可以自由分配绿色或蓝色。 

一旦分配了所有中立席位，网格就会变成二元着色。 如果两个单元格共享一条边并且最终颜色相同，则它们属于同一区域。 任务是为所有中性单元选择颜色，使得绿色单元形成的连接分量加上蓝色单元形成的连接分量的总数尽可能小。 

网格的结构比个体计数更重要。 单个重新着色决策可以合并或拆分组件，尤其是当中性单元位于多个已着色区域之间时。 

最多 10 个中立单元的限制是关键的结构限制。 如果没有它，问题将类似于对网格标签的硬优化。 有了它，搜索空间就缩减为一组可管理的配置，因为只有那几个单元会引入不确定性。 

当中性细胞充当桥梁时，就会出现一种幼稚但重要的边缘情况。 

考虑一行：```
0 ? 0
```如果中间的单元格设置为 0，则两种绿色都会成为一个组成部分。 如果设置为 1，果岭保持分离。 这表明每个中性细胞都可以对连通性产生全局影响。 

当多个中性细胞相互作用时，会出现一个更微妙的问题。 无法独立做出决定，因为更改一个分配可能会改变另一个中性单元是否连接或隔离相同颜色的区域。 

这些约束意味着任何解决方案都必须明确推理最多 10 个不确定单元的所有配置，因为任何启发式局部选择都可能在全局范围内失败。 

## 方法

 如果我们忽略对中性单元的约束，问题就变成了所有 2D 着色的全局优化，其中每个单元分配以非线性方式影响连接性。 一个强力的想法是将每个中性单元分配为绿色或蓝色，然后使用洪水填充重新计算生成的网格中连接组件的数量。 

这种强力方法是正确的，因为它评估与输入一致的每个可能的最终状态。 问题在于规模。 如果有 k 个中立单元，则配置数量将为 2^k。 在一般情况下，k可能达到10,000，这是不可能的。 

关键的观察结果是 k 的界限为 10。这改变了一切。 我们没有对整个网格进行推理，而是只枚举这几个不确定位置的分配。 每次分配都完全确定网格，之后可以直接在 O(NM) 中计算连通性。 

因此，问题简化为探索大小最多为 1024 的状态空间，并通过网格遍历独立评估每个状态。 

解决方案的结构变成了一个两级过程。 外层枚举中性单元格的所有颜色分配。 内部层计算结果网格中连接组件的数量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（所有作业 + BFS）| O(2^k·N·M) | O(2^k·N·M) | O(N·M) | 已接受 |
 | 最优（与 k ≤ 10 相同）| O(2^k·N·M) | O(2^k·N·M) | O(N·M) | 已接受 |

 区别在于，“最佳”解决方案只是认识到暴力破解由于约束而变得可行。 

## 算法演练

 我们将中性单元视为索引变量，并为它们枚举所有可能的二进制赋值。 

1. 首先，收集所有标记为中性的单元格的坐标。 这些是问题中唯一的自由度。 此步骤将问题从网格决策问题简化为最多 10 个变量的位掩码枚举问题。 
2. 迭代 0 到 2^k − 1 之间的所有整数，其中 k 是中性单元格的数量。 每个整数代表颜色的唯一分配，其中位 i 确定中性单元 i 是变为绿色还是蓝色。 
3. 对于每个作业，通过复制原始网格并根据当前位掩码替换中性单元来构造最终网格。 此步骤实现了完整的着色，以便可以毫不含糊地评估连接性。 
4. 在网格上运行标准洪水填充或 BFS，以分别计算绿色和蓝色的连接组件。 每次遇到未访问的单元格时，我们都会开始遍历标记其整个组件。 
5. 计算此分配的组件总数，即绿色和蓝色组件的总和。 跟踪所有作业的最小值。 
6. 输出找到的最佳值。 

正确性取决于每个分配都是独立的并且完全确定网格结构，因此独立评估它们不会失去任何交互效果。 

### 为什么它有效

每个有效的最终配置恰好对应于中性单元上的一个位掩码。 该算法枚举了所有此类位掩码，因此不会错过任何可行的解决方案。 对于每种配置，都会精确计算组件数量。 由于连通性是在完全构建的网格上评估的，因此不涉及近似或部分推理。 因此，所有配置的最小值与全局最优值相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import deque

def count_components(grid, n, m):
    vis = [[False] * m for _ in range(n)]
    
    def bfs(i, j):
        q = deque()
        q.append((i, j))
        vis[i][j] = True
        color = grid[i][j]
        while q:
            x, y = q.popleft()
            for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
                nx, ny = x + dx, y + dy
                if 0 <= nx < n and 0 <= ny < m:
                    if not vis[nx][ny] and grid[nx][ny] == color:
                        vis[nx][ny] = True
                        q.append((nx, ny))
    
    comps = 0
    for i in range(n):
        for j in range(m):
            if not vis[i][j]:
                bfs(i, j)
                comps += 1
    return comps

def solve():
    n, m = map(int, input().split())
    g = [list(input().strip()) for _ in range(n)]
    
    qs = []
    for i in range(n):
        for j in range(m):
            if g[i][j] == '?':
                qs.append((i, j))
    
    k = len(qs)
    ans = float('inf')
    
    for mask in range(1 << k):
        for idx, (i, j) in enumerate(qs):
            g[i][j] = '0' if (mask >> idx) & 1 == 0 else '1'
        
        ans = min(ans, count_components(g, n, m))
    
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先提取所有不确定的位置。 这些被存储，以便掩码中的每一位一致地对应于一个单元。 在迭代过程中，网格会临时就地修改以提高效率，避免重复分配。 

BFS 功能是标准的四向洪水填充。 每次遇到未访问的单元格时，它都会探索相同颜色的整个连接区域，确保每个组件都被精确计数一次。 

一个微妙的实现细节是，每个掩码都会重复覆盖网格。 由于原来的“？” 无论如何，值总是被覆盖，并且每次迭代都会重新分配所有“？” 单元格，无需在迭代之间恢复网格。 

## 工作示例

 考虑一个简单的 2 x 3 网格：```
? 0 ?
0 ? 1
```共有 3 个中性单元格，因此我们枚举了 8 个分配。 

| 面膜| 网格解读| 绿色比较| 蓝色比较| 总计 |
 | ---| ---| ---| ---| ---|
 | 000 | 000 全部 ？ = 0 | 1 | 1 | 2 |
 | 001| 最后的 ？ = 1 | 2 | 1 | 3 |
 | 010| 中间 ？ = 1 | 2 | 2 | 4 |
 | ... | ... | ... | ... | ... |

 当选择中性单元来连接现有的相同颜色区域而不是分割它们时，就会出现最佳配置。 

现在考虑一个单行情况：```
0 ? 0 1
```| 面膜| 作业 | 组件|
 | ---| ---| ---|
 | 0 | 0 0 0 1 | 0 0 0 1 绿色=1，蓝色=1 总计=2 |
 | 1 | 0 1 0 1 | 0 1 0 1 绿色=2，蓝色=1 总计=3 |

 这表明有时将中性细胞变成不同的颜色会显着增加碎片。 

该迹线表明，每个中性单元格直接影响相邻的相同颜色区域是合并还是保持分离，这证明了完整枚举的合理性。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(2^k·N·M) | O(2^k·N·M) | 最多 1024 个分配中的每一个都会触发全网格 BFS/DFS 遍历 |
 | 空间| O(N·M) | 参观阵列和网格存储 |

 当 N、M 最大为 100、k 最大为 10 时，最坏情况下的操作数约为 1024 × 10000，这完全在 Python 中使用简单 BFS 逻辑执行 1 秒的限制范围内。

## 边缘情况

 关键的边缘情况是当中性单元位于两个相同区域之间并且可以根据分配合并或分离它们时。 

输入：```
1 3
0?0
```如果中间单元格分配为 0，则整行将成为一个组件。 如果分配为 ​​1，则有两个由蓝色分隔的绿色分量。 该算法评估两个掩码，显式构建网格，BFS 正确区分这些结果，确保选择最小值。 

另一种情况涉及交替结构：```
0?1
?0?
1?0
```这里每个中性单元都参与多个潜在的合并。 该算法枚举所有 2^k 个配置，并且 BFS 每次都从头开始重新计算连接性，从而保证中立单元之间的交叉依赖关系得到充分考虑，而不是近似。
