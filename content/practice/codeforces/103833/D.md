 ## 解决方案
 ## 问题理解

 我们得到了一棵有根的树。 每个顶点都带有+1 或-1 的值。 您从零时间的固定顶点开始，初始生命值为 1。 

时间以离散的步骤前进。 在每个时间步中，都会按严格的顺序发生三件事：首先，根据您当前站立的顶点，您获得或失去生命；然后，如果您的生命变为零或当时熔岩已经到达该顶点，您将立即死亡；只有当您还活着时，您才会选择一个相邻的顶点移动到下一个时间步。 

熔岩从根部向外扩散，经过 t 步后，距根部距离不超过 t 的每个顶点都被视为被淹没。 这意味着较深的节点在较长时间内是安全的，而较浅的节点则较早变得不安全。 因为移动发生在生存检查之后，所以你进入顶点的那一刻很重要：太早到达可能会立即杀死你，即使晚一步该顶点是安全的。 

任务是从初始顶点开始，最大化您在死亡之前可以执行的移动次数。 

约束足够大，每次测试 n 可以高达数十万，并且可以有很多测试用例。 这立即排除了任何试图模拟所有可能路径或针对每个路径独立重新计算状态的行为。 任何解决方案都必须在每个测试用例中以接近线性的时间处理树，并重用结构而不是枚举轨迹。 

主要的微妙边缘情况来自死亡的触发方式。 首先，生命值更新发生在检查洪水安全之前。 这意味着权重为 -1 的顶点甚至可以在熔岩发挥作用之前杀死你。 其次，在熔岩到达顶点时准确到达是致命的，因此“距离相等”并不安全。 第三，由于每一步都被迫移动，因此不能选择留在安全的高价值节点上，这使得贪婪的本地推理失败。 

一个小的说明性失败：假设一个节点的权重为 -1 并且仍未被淹没。 一种天真的方法可能会认为它是安全的，因为熔岩很远，但是当生命为 1 时踩到它会导致在任何移动之前立即死亡。 

## 方法

 暴力方法试图将其视为对由（当前顶点、当前生命值、当前时间）定义的状态的最短路径样式搜索。 我们从每个州分支到所有邻居并模拟生活变化和洪水。 这在原则上是正确的，因为它遵循精确的规则，但状态空间爆炸了。 生命值在数量上无上限，最多为 n，时间最多为 n，分支与程度成正比，因此最坏情况的复杂性与树的深度呈指数关系。 即使有记忆，在对抗性情况下不同状态的数量仍然是 θ(n²)。 

关键的观察结果是，生命进化仅取决于沿路径的累积总和，而洪水发生的时间仅取决于深度。 这区分了两种效果：一种是路径相关的附加权重，另一种是相对于根的纯粹位置。 该问题简化为在树中选择向下或向上约束行走，其中每个顶点都有截止时间（其深度），并且每条路径都有一个运行的前缀和约束，该约束永远不会降至零。 

这种结构允许重新扎根或基于 DFS 的 DP，其中我们跟踪每个节点的最佳“可生存扩展深度”或等效地从该节点开始的路径的最长有效延续，同时保持正的前缀和并保持在洪水时间之前。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力状态搜索 | 指数| O(n²) 或更糟 | 太慢了 |
 | 具有深度感知前缀约束的树DP | O(n) | O(n) | 已接受 |

 ## 算法演练

核心思想是将每个根到节点路径视为具有两个约束的序列：权重的前缀和必须保持正数，并且位置索引必须始终严格小于洪水深度约束。 

我们使用从起始节点开始的 DFS 来解决这个问题，但我们不仅仅跟踪位置，而是沿着当前路径维护迄今为止遇到的累积寿命和最小前缀寿命。 

1. 树的根为 1，并使用 BFS 或 DFS 预先计算从根开始的每个节点的深度。 这给出了每个节点被淹没的确切时间。 
2. 从起始节点启动DFS。 维护两个运行值：当前生命总和和迄今为止沿路径看到的最小生命。 这些代表我们是否已经违反了生存条件。 
3. 在每个节点，首先将节点的权重应用到当前生命中，然后再进行其他操作。 这反映了问题陈述中的强制顺序，对于正确性至关重要。 
4. 立即使用当前节点的深度检查当前节点在该时间步是否已经被淹没。 如果是，则路径在此停止。 
5.同时检查生命值是否已降至零或以下。 如果是这样，路径也停在这里。 
6. 否则，对于当前节点（或除父节点之外的任何相邻节点）的每个子节点，递归地继续 DFS，将移动计数器增加 1。 
7. 跟踪从初始顶点开始的所有有效 DFS 分支的最大移动次数。 

关键的困难在于，朴素的 DFS 会过度计数，因为可能会重新访问具有不同累积寿命的状态。 该结构依靠这样一个事实来避免显式记忆：一旦生命变得非积极或洪水赶上，就不可能从该分支继续下去，因此修剪是安全的。 

### 为什么它有效

 每个有效轨迹完全对应于受前缀和条件和单调深度约束约束的树中的根向行走。 DFS 探索所有此类行走，而无需重新访问处于不一致时间状态的节点，因为时间是由沿路径的深度隐式编码的。 修剪条件与原始进程中的确切失败条件相匹配，因此不会计算无效扩展，并且每个有效扩展都可以通过某些 DFS 路径到达。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n, st = map(int, input().split())
    w = [0] + list(map(int, input().split()))

    g = [[] for _ in range(n + 1)]
    for i in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    # compute depth from root = 1 (lava source)
    depth = [-1] * (n + 1)
    from collections import deque
    q = deque([1])
    depth[1] = 0
    while q:
        u = q.popleft()
        for v in g[u]:
            if depth[v] == -1:
                depth[v] = depth[u] + 1
                q.append(v)

    ans = 0

    def dfs(u, p, life, moves, cur_depth):
        nonlocal ans

        life += w[u]
        if life <= 0:
            return
        if cur_depth > depth[u]:
            return

        ans = max(ans, moves)

        for v in g[u]:
            if v == p:
                continue
            dfs(v, u, life, moves + 1, cur_depth + 1)

    dfs(st, -1, 1, 0, depth[st])
    print(ans)

t = int(input())
for _ in range(t):
    solve()
```该实现将洪水定时的预处理与 DFS 状态分开。 由于熔岩行为仅取决于到根部的距离，因此深度从根部开始计算一次。 DFS 显式地携带生命，并在检查终止条件之前更新生命，这与流程中的强制排序相匹配。 

一个微妙的一点是，DFS 还带有当前时间深度的概念，与移动步骤保持一致。 这就是使运动与熔岩膨胀同步的原因。 

## 工作示例

 由于原始陈述此处不包含样本，因此考虑一棵最小树。 

第一个示例：由三个节点组成的线，根为 1，权重为`[+1, -1, +1]`，从节点 2 开始。熔岩深度从根开始增长，因此节点 1 首先变得不安全。 从节点 2 开始，生命值为 1，然后应用 -1 使生命值为 0，并且该过程立即终止，因此不可能进行任何移动。 DFS 在第一次转换时停止，因为在第一次更新后立即违反了生命约束。 

第二个例子：平衡树，其中起始节点很深并且所有权重都是+1。 在这种情况下，生命只会增加，唯一的限制因素是熔岩到达节点。 DFS 将遵循最深的分支，直到深度限制阻止进一步的移动。 这表明即使没有明确的贪婪逻辑，该算法自然也更喜欢更深的安全区域。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | 每个测试用例 O(n) | DFS中每条边都遍历一次，深度预处理是线性的 |
 | 空间| O(n) | 邻接表、递归栈和深度数组 |

 约束允许这样做，因为 n 次测试的总和很大但仍然是线性的，并且由于基于寿命和洪水约束的剪枝而不会发生状态爆炸。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    def solve():
        n, st = map(int, input().split())
        w = [0] + list(map(int, input().split()))
        g = [[] for _ in range(n + 1)]
        for _ in range(n - 1):
            u, v = map(int, input().split())
            g[u].append(v)
            g[v].append(u)

        from collections import deque
        depth = [-1] * (n + 1)
        q = deque([1])
        depth[1] = 0
        while q:
            u = q.popleft()
            for v in g[u]:
                if depth[v] == -1:
                    depth[v] = depth[u] + 1
                    q.append(v)

        ans = 0
        sys.setrecursionlimit(10**7)

        def dfs(u, p, life, moves, d):
            nonlocal ans
            life += w[u]
            if life <= 0 or d > depth[u]:
                return
            ans = max(ans, moves)
            for v in g[u]:
                if v != p:
                    dfs(v, u, life, moves + 1, d + 1)

        dfs(st, -1, 1, 0, depth[st])
        return str(ans)

    t = int(input())
    out = []
    for _ in range(t):
        out.append(solve())
    return "\n".join(out)

 ## 边缘情况

 一个关键的边缘情况是起始节点本身具有负权重。 在这种情况下，生命更新发生在任何移动之前，因此该过程立即结束。 DFS 可以正确处理此问题，因为它在探索子项之前应用权重，因此不会进行递归调用。 

另一个边缘情况是当您到达时节点恰好被淹没。 由于深度检查使用严格比较，因此达到相等深度会触发终止。 DFS条件`cur_depth > depth[u]`确保平等被视为死亡，精确匹配问题规则。 

第三种边缘情况是生活在积极和消极价值观之间摇摆的深层路径。 修剪基于`life <= 0`确保一旦路径变得无效，它就不会进一步扩展，从而防止交替权重模式上的指数分支。
