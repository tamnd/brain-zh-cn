 ## 解决方案
 ## 问题理解

 我们将点球建模为球门上的离散网格。 网格中的每个单元格对应于可能的镜头位置。 对于守门员来说，每个单元格都包含一个值，描述守门员扑救射门的可能性。 对于每个球员，每个单元格包含该球员射向该位置时得分的可能性。 

为了评估一名球员，我们将他们的射门概率与守门员的扑救概率逐个结合起来。 单元格的综合值是同一位置球员成功概率和守门员扑救失败概率的乘积。 如果该组合概率至少为 0.65，则该单元格被视为对玩家“良好”。 玩家的得分是网格中此类好单元格的数量。 

我们必须选择得分最高的五名玩家。 如果多个玩家得分相同，我们将按照他们全名的字典顺序打破平局。 

输入尺寸很小：$N, M \le 100$，并且玩家数量最多为 100。这立即表明即使是$O(KNM)$解决方案足够快，因为它最多大约$10^6$运营。 

主要的微妙之处在于浮点比较。 给出的值具有两位小数，因此必须仔细进行阈值 0.65 的比较以避免精度漂移。 安全的方法是通过乘以 100 或 10000 将所有内容缩放为整数，或者与一个小的 epsilon 进行比较。 

重要的边缘情况主要是平局处理和恰好为 0.65 的浮点相等。 幼稚的严格比较可能会对边界值进行错误分类。 另一个陷阱是全名的字典比较：空格是排序的一部分，因此必须直接使用标准字符串比较。 

## 方法

 暴力方法本质上是最终的解决方案。 对于每个玩家，我们迭代所有玩家$N \times M$单元格，与守门员网格计算乘积，计算有多少超出阈值，然后对球员进行排序。 

这是有效的，因为细胞或玩家之间不存在相互依赖。 每个玩家的分数都是独立的，因此问题可以清晰地分解为独立评估，然后进行排名。 

较慢的思维模型是考虑重新计算或成对比较玩家，但这是不必要的。 一旦计算出每个分数，选择前五名就是一个标准的排序问题。 

唯一的“优化”是认识到重新计算结构已经是最优的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个玩家进行暴力排序 | O(KNM + K log K) | O(KNM + K log K) | O(K) | 已接受 |
 | 任何过于复杂的替代方案| O(K^2 NM) | O(K^2 NM) | O(K) | 太慢了 |

 ## 算法演练

 ### 1. 阅读守门员网格

 我们首先阅读$N \times M$代表守门员的矩阵。 该网格可供所有玩家重复使用，因此我们将其存储一次。 

### 2. 解析所有玩家的名字和网格

 对于每个玩家，我们都会存储他们的姓名和$N \times M$概率矩阵。 由于约束很小，存储所有数据是安全的并且简化了计算。 

### 3.独立计算玩家得分

 对于每个玩家，我们迭代所有网格单元。 在每个单元格中，我们计算守门员价值和球员价值的乘积。 如果该乘积至少为 0.65，我们就会增加分数。 

这一步对于每个玩家来说都是独立的，这就是问题线性化的原因$KNM$。 

### 4. 将结果存储为（分数，名称）

 我们保留一个包含计算得分和玩家姓名的对列表。 该结构将用于排序。 

### 5. 按得分和字典顺序对玩家进行排序

 我们按分数降序排序。 如果分数相等，我们依赖于名字的字典顺序。 这确保了确定性的平局打破。 

### 6.输出前五个名字

 排序后我们输出前五个条目。 

### 为什么它有效

 关键的不变量是每个球员的得分仅取决于自己的网格和固定的守门员网格。 玩家之间没有交互，因此排名相当于对独立标量分数进行排序。 乘法和阈值化将每个网格单元减少为布尔贡献，使分数成为单元上的简单加法函数。 由于加法在玩家之间是关联且独立的，因此排名问题变成纯粹基于比较的排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def parse_matrix(n, m):
    mat = []
    for _ in range(n):
        row = list(map(float, input().split()))
        mat.append(row)
    return mat

def main():
    n, m = map(int, input().split())
    k = int(input())

    keeper = parse_matrix(n, m)

    players = []

    for _ in range(k):
        name = input().strip()
        mat = parse_matrix(n, m)

        score = 0
        for i in range(n):
            for j in range(m):
                if keeper[i][j] * mat[i][j] >= 0.65 - 1e-12:
                    score += 1

        players.append(( -score, name))

    players.sort()
    for i in range(5):
        print(players[i][1])

if __name__ == "__main__":
    main()
```代码读取守门员矩阵一次，然后独立处理每个球员。 分数被取反，因此按升序排序自然会给出最高分数。 比较中的 epsilon 可防止浮点乘法生成 0.6499999998 等值时出现精度问题。 

排序步骤还会自动解决字典顺序的平局问题，因为当第一个字段相等时，Python 元组比较会使用第二个字段。

 ## 边缘情况

 一种极端情况是多个玩家得分相同。 该算法自然地处理这个问题，因为排序使用字典比较作为辅助键。 例如，如果两个玩家的得分都是 10，则他们的顺序纯粹是通过名字的字符串比较来确定的。 

另一个边缘情况是浮点边界恰好为 0.65。 epsilon 防护确保由于精度误差而极其接近阈值的值不会被错误分类。 例如，像 0.64999999997 这样的计算值会在没有容差的情况下错误地满足条件。 

最后一个边缘情况是所有玩家都有相同的矩阵。 在这种情况下，所有分数都相等，并且输出只是排序规则自动生成的五个字典顺序最小的名称。
