 ## 解决方案
 ## 问题理解

 我们得到了一组剑龙，每只剑龙都与一个代表其尖刺数量的整数值相关联。 从这个集合中，我们必须选择任何两个不同的剑龙并测量它们在外观上的相似程度，其中相似性定义为它们的峰值计数之间的绝对差异。 

该任务简化为找到数值差异尽可能小的值对，并输出最小差异。 

用算法术语来说，我们得到一个数组，并且必须计算任何两个不同元素之间的最小绝对差。 

从约束中观察到的关键是$n \le 10^5$，这立即排除了任何二次成对比较方法。 一个简单的双循环会粗略地检查$\frac{n(n-1)}{2}$对，在最坏的情况下大约是$5 \times 10^9$运营。 这远远超出了竞技节目中典型的一秒或两秒的限制。 

一些边缘情况在实践中很重要。 如果许多值相同，例如输入类似`1 1 1 1`，正确答案为零，因为选择任何相等的对都会产生零差异。 如果不确保预处理后相等的值变得相邻，则仅检查未排序数组中的相邻位置的粗心方法将在此失败。 

例如，当价值观广泛分散时，就会出现另一种微妙的情况`0 100 50 49 51`。 正确答案是 1，来自 (50, 49) 或 (50, 51)。 任何在排序后不全局推理紧密度的解决方案都可能会错过这样的对。 

## 方法

 蛮力的想法很简单。 我们尝试每一对剑龙，计算它们尖峰计数的绝对差，并保持最小值。 这是有效的，因为它直接评估答案的定义而不进行任何转换。 然而，它变得太慢，因为它需要检查所有对，导致$O(n^2)$比较。 

关键的见解来自于认识到绝对差异在排序下表现良好。 如果我们对数组进行排序，那么在值方面最接近的对必须按排序顺序出现在彼此旁边。 这是因为任何非相邻对之间至少有一个元素，并且该中间元素只能减少邻居之间的差距，而永远不会将其增加到超出更好的相邻候选者。 

因此，我们不需要比较所有对，只需比较排序后的连续元素即可。 这将问题简化为扫描一次排序数组并跟踪最小相邻差异。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 排序 + 扫描 |$O(n \log n)$|$O(1)$或者$O(n)$| 已接受 |

 ## 算法演练

 1.读取整数$n$和尖峰计数数组。 这形成了我们必须分析的数据集。 
2. 按非降序对数组进行排序。 排序至关重要，因为它会重新排列值，以便值空间中邻近的数字成为数组中的邻居。 
3. 初始化变量`ans`数量非常大。 该变量跟踪迄今为止发现的最小差异。 
4. 从第一个元素到倒数第二个元素迭代数组，检查每个相邻对。 
5. 对于每个索引$i$，计算之间的差异`a[i+1]`和`a[i]`。 由于数组已排序，因此该差异保证为非负数，并且表示涉及的最接近的可能比较`a[i]`在其右侧。 
6. 更新`ans`如果该差值小于当前存储的值。 这确保我们始终保留迄今为止最好的候选人。 
7.处理完所有相邻对后，输出`ans`。 

### 为什么它有效

 正确性依赖于这样一个事实：在排序数组中，任何对$(i, j)$和$j > i + 1$不能产生比它们之间的某些相邻对更小的差异。 形式上，对于任何$i < k < j$，我们有：$$a[j] - a[i] = (a[j] - a[k]) + (a[k] - a[i])$$排序后两项均为非负数，因此总差值至少与区间内的相邻间隙之一一样大。 这保证了全局最小值必须出现在连续元素之间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

a.sort()

ans = float('inf')

for i in range(n - 1):
    diff = a[i + 1] - a[i]
    if diff < ans:
        ans = diff

print(ans)
```代码直接遵循算法。 排序步骤确保我们只需要本地比较。 相邻对上的循环是安全的，因为每个相关的候选对都在那里表示。 使用`float('inf')`是一种初始化最小跟踪器的便捷方法，无需担心输入值的范围。 

一个常见的实现错误是忘记对数组进行排序，这会导致完全丢失最佳对。 另一个微妙的问题是在未排序的数据上使用绝对差异，但仅检查邻居，这是不正确的，因为在未排序的排列中，紧密度不是局部的。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))
    a.sort()
    ans = float('inf')
    for i in range(n - 1):
        ans = min(ans, a[i + 1] - a[i])
    return str(ans)

 ## 边缘情况

 对于重复较多的输入，例如`4 4 4 4`，算法排序成一个常量数组，每个相邻的差值都为零，因此结果正确地为零，无需特殊处理。 

对于反向排序的输入，例如`5 4 3 2 1`，排序将其转化为`1 2 3 4 5`，最小相邻差值变为 1。跳过排序并仅检查原始邻居的简单方法会错误地得出更大的间隙，例如 4 或 3。 

对于大值分离情况，例如`0 1000000000 500`，排序确保最接近的对显示为`(500, 0)`或者`(1000000000, 500)`，在排序后从相邻比较中提取正确的最小值。
