 ## 解决方案
 ## 问题理解

 我们被要求构造一个从 1 到 n 的数字排列，以最大化嵌套交替绝对表达式的形式，其中在连续元素之间重复求差，并在每个减法级别应用绝对值。 从概念上讲，我们对数字进行排列，以便在评估这个“交替绝对减法链”时，最终值尽可能大。 

输入是单个整数 n，输出是最大化该表达式的长度 n 的任意排列。 由于输出是排列，因此从 1 到 n 的每个整数必须恰好出现一次。 

约束 n ≤ 10^6 立即排除了任何尝试通过排列评估目标函数的方法。 排列的数量为 n!，即使对于 n = 20 来说也太大了。即使尝试模拟固定排列的表达式也是 O(n)，因此暴力强制所有排列将是 O(n · n!)，这是完全不可行的。 

一个微妙的边缘情况是表达式结构在很大程度上取决于排序，而不仅仅是多重集。 例如，对于 n = 3，[1, 3, 2] 和 [3, 1, 2] 等排列会产生不同的中间绝对值，即使它们包含相同的元素。 贪婪的本地交换方法很容易失败，因为早期的决策会影响所有剩余的操作。 

## 方法

 关键的困难是理解交替绝对减法的行为方式。 该表达式重复获取当前值并减去下一个元素，然后将结果包装为绝对值，然后继续。 

如果我们扩展结构，我们会注意到最终结果很大程度上取决于序列早期放置的值的大小以及在绝对值破坏结构之前通过符号翻转使用小值来“放大”差异的程度。 

蛮力方法会尝试所有排列，每次在 O(n) 时间内计算表达式，并跟踪最大值。 这是正确的，但立即就崩溃了，因为它需要 O(n!) 次排列，这对于 n 最大为 10^6 来说是不可能的。 

关键的观察结果是绝对值在每一步都会破坏符号信息，这意味着我们本质上是通过选择何时发生大跳跃来控制中间结果的幅度。 为了最大化最终值，我们希望强制表达式重复创建较大的差异，而不是消除它们。 这是通过在大值和小值之间交替来实现的，以便每个减法步骤在应用绝对值之前产生较大的幅度。 

这种结构导致了一个简单的最优结构：首先放置最大的数字，然后将范围两端的剩余数字交错。 这保证了每个减法步骤都会在当前值和下一个选择的元素之间产生很大的差距，从而避免抵消效应。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·n!) | O(n·n!) | O(n) | 太慢了|
 | 交替结构| O(n) | O(n) | 已接受 |

 ## 算法演练

 最佳策略是通过交替最大剩余值和最小剩余值来构建排列。

1. 初始化两个指针，low=1，high=n。 我们认为 [low, high] 中的所有数字尚未使用。 
2. 开始构建答案列表。 我们首先取最大的剩余值 high，并将其附加。 然后我们将 high 递减，因为它现在已被使用。 从最大值开始可确保表达式中尽早出现大间隙。 
3. 接下来，我们取最小的剩余值 low，并将其附加，然后递增 low。 这与之前的大值形成了强烈的对比。 
4. 继续交替：从高位取，然后从低位取，如此反复，直到用完所有数字。 
5. 如果一个指针与另一个指针交叉，则停止。 这自然可以处理偶数和奇数 n。 

我们交替的原因是，在应用绝对值之前，连续的极端差异会最大化中间结果的大小。 如果我们按排序顺序放置数字，差异会很小，并且取消会占主导地位，从而降低最终值。 

### 为什么它有效

 在构建的排列的每一步中，下一个选择的值是剩余间隔的极值。 这保证了连续选择的元素之间的差异在给定剩余元素的情况下始终尽可能大。 由于表达式在每次减法后应用绝对值，因此中间结果的符号并不重要，重要的是每一步的差值大小。 该结构确保每一步都会产生接近最大可能的间隙，并且剩余元素的重新排列不会增加任何局部过渡而不减少另一个局部过渡。 这产生了全局最优序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    l, r = 1, n
    res = []
    
    while l <= r:
        res.append(r)
        r -= 1
        if l <= r:
            res.append(l)
            l += 1
    
    print(*res)

if __name__ == "__main__":
    solve()
```该实现维护两个代表最小和最大未使用值的指针。 每次迭代都会附加当前最大值，然后附加当前最小值，从而缩小可用范围。 这直接编码了前面描述的交替策略。 循环条件确保我们永远不会重用元素，并且最终打印输出有效的排列。 

一个微妙的点是我们总是首先附加高值。 从低开始会产生有效的排列，但会破坏最大化表达结构早期差异的预期交替模式。

 ## 边缘情况

 对于 n = 1，该算法设置 l = r = 1。它附加 1 一次并立即终止，产生有效的排列。 

对于 n = 2，我们从 l = 1、r = 2 开始。该算法附加 2，然后附加 1，生成 [2, 1]。 由于指针交叉，循环结束。 这证实了交替规则可以处理最小的有意义的输入，而无需特殊的大小写。 

对于奇数 n，例如 n = 5，最后剩余的元素是中间值。 当 l == r 时，它会在最后一步附加一次。 该构造永远不会跳过该元素，因为循环条件允许相等。
