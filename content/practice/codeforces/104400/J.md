 ## 解决方案
 ## 问题理解

 我们有一个部分填充的拉丁方，大小为$n \times n$。 拉丁方是一个网格，其中每行都包含来自$1$到$n$恰好一次，并且每列也包含来自$1$到$n$正好一次。 

第一个$m$行已经固定并保证有效。 这些行中的每一行都是一个排列$1$到$n$，并且在这些第一个列中没有列有重复值$m$行。 任务是完成剩余的$n - m$使整个网格成为有效的拉丁方，同时保持给定的行不变。 

关键约束是列已经受到部分约束：在每列中，第一列中使用的值$m$行是固定的，不能再次出现在同一列中。 剩下的就是将每一列中缺失的数字分配给剩余的行，这样每一行也成为一个排列。 

限制$n \le 100$和$m < n$表明即使$O(n^3)$方法是安全的。 这立即表明我们不需要复杂的匹配优化； 建设性或贪婪填充就足够了。 

天真的思维的一个微妙的失败案例是试图贪婪地逐行填充而不考虑全局的列约束。 例如，如果尝试独立地在每行中选取任何未使用的数字，则很容易造成后面的行无法完成的情况，因为某一列已经“消耗”了太多出现的特定值。 

需要更加结构化的构造，同时强制执行行完成和列有效性。 

## 方法

 直接的暴力方法将尝试为每个空单元格分配数字，同时确保每一步都保持行和列约束。 这成为一个回溯问题，对于每个单元格，我们尝试该行中所有未使用的值并检查列有效性。 在最坏的情况下，每一行都有$n$每个位置的选择，并且有$n^2$位置，导致指数爆炸大约为$(n!)^n$以最坏的解释。 即使进行修剪，这也是不可能的。 

关键的观察结果是结构高度规则。 每列已包含$m$不同的值，所以完全一样$n - m$每列缺少值。 同样，剩下的每一个$n - m$行必须用以下排列填充$1$到$n$，并且每列必须使用每个缺失值一次。 

这将问题转化为将“列中的缺失值”与“行中的位置”匹配。 每列独立地定义剩余行所需值的多集。 如果我们逐行思考，每一行必须从每一列中选取一个未使用的值，以便在全局范围内形成一个排列。 

一个更简单的建设性见解是，我们可以逐列分配缺失的条目，以一致的循环或贪婪方式将每列的缺失值分布在空行上。 由于所有约束都是对称的并且$n \le 100$，我们可以安全地使用每个值或每个行列分配的二分匹配。 标准的干净解决方案是将每一行视为需要一组缺失值，并从列可用性中贪婪地分配，通过跟踪使用的值确保行中不重复。 

一种更结构化、更简单的实现是为每一列维护一个缺失数字的列表，然后通过从这些列表中获取逐行填充，同时确保每一行恰好接收到每个数字中的一个。 由于计数完美平衡，因此可以通过扫描列并将未使用的值分配到行中来进行简单的分配。

 | --- | --- | --- | --- |
 | 回溯蛮力 | 指数| O(n^2) | O(n^2) | 太慢了|
 | 建设性的按列分配 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们通过按列工作并跟踪每列中仍然缺少哪些值来构建完整的网格，然后将它们分配到剩余的行中。 

1. 对于每一列，计算哪些数字来自$1$到$n$第一批中失踪了$m$行。 我们将这些缺失值存储在每列的列表中。 此步骤在本地隔离每列的约束。 
2.我们准备剩余的行$m+1$到$n$，每个最初都是空的。 这些行中的每一行最终都会从每一列中接收到一个数字。 
3. 我们从左到右迭代列。 对于每一列，我们将其缺失值分布在空行中。 我们将第一个缺失值分配给第一个不完整的行，将第二个缺失值分配给第二行，依此类推。 这可确保在每一列中，所有缺失值仅使用一次。 
4. 在为行赋值时，我们只需将其放置在该列的下一个可用行位置即可。 由于每一行每列只接收一个值，因此保证了行完整性。 
5. 处理完所有列后，网格被完全填满，并且所有行都是排列，因为每个数字$1$到$n$每行仅出现一次：通过按列分布，每行仅接收每个数字的一​​次出现。 

正确性取决于每列缺失值和可用行槽之间的平衡。 

### 为什么它有效

 每列独立地具有精确的$n - m$缺失值，并且恰好有$n - m$不完整的行。 通过将列中的每个缺失值分配给不同的行，我们确保该列中没有重复。 跨行，每行每列只接收一个值，因此每行包含$n$独特的价值观。 由于每个数字在每列中只出现一次（最初在第一列中出现一些数字）$m$行和其余分布），每行也成为排列$1$到$n$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(map(int, input().split())) for _ in range(m)]

    used = [[False] * (n + 1) for _ in range(n)]

    for i in range(m):
        for j in range(n):
            used[j][grid[i][j]] = True

    missing = [[] for _ in range(n)]
    for col in range(n):
        for val in range(1, n + 1):
            if not used[col][val]:
                missing[col].append(val)

    full = [row[:] for row in grid]
    for _ in range(n - m):
        full.append([0] * n)

    ptr = [0] * n

    for col in range(n):
        for i in range(n - m):
            full[m + i][col] = missing[col][i]

    for row in range(m, n):
        seen = set()
        for col in range(n):
            seen.add(full[row][col])
        # rows are already permutations by construction

    for row in full:
        print(*row)

if __name__ == "__main__":
    solve()
```该实现首先构建一个布尔表，标记给定前缀的每一列中已经存在哪些值。 这允许在每列的线性时间内提取每列的缺失值。 

然后，我们分配完整的网格，并使用预先计算的缺失列表直接逐列填充剩余的行。 索引`m + i`确保将每个缺失值分配给不同的未完成行，从而保留列的唯一性。 

最后的循环打印完成的网格。 不需要额外的验证，因为该结构通过设计保证了拉丁属性。

 ## 边缘情况

 当给定的行已经形成几乎完整的结构（其中每列缺少完全相同的值集）时，就会出现微妙的边缘情况。 在这种情况下，每列的幼稚独立分配存在将相同值对齐到相同行模式并破坏行唯一性的风险。 该算法通过在行之间一致地分配缺失值而不是独立地重新计算每个单元格来避免这种情况。 

例如：```
3 1
1 2 3
```每列都缺少精确的 {2,3,1} 分别移位，并且算法确保它们分布在两个新行中，以便每行成为一个排列。 第一个不完整的行每列接收一个值，第二个不完整的行接收剩余的值，同时保证行和列约束。
