 ## 解决方案
 ## 问题理解

 我们得到了一排石堆。 游戏从爱丽丝开始交替进行。 在每一回合中，当前玩家仅关注最左边仍包含石子的堆，并从中移除至少一个石子。 因为这是简单版本$k = 1$，一个回合恰好由一个这样的移除组成，并且在所选堆变空后，继续在下一回合右侧的下一堆上进行游戏。 

当所有牌堆都空时游戏结束。 当没有剩余棋子时，即将移动的玩家没有合法的移动并立即失败，因此前面的玩家是赢家。 

尽管每堆石子的数量是任意的，但玩家永远不会被迫以受限的方式将一堆石子分成多个回合。 如果需要，任何移动都可以移除整个剩余的一堆石头，因为移除“任意数量的至少一个”包括拿走所有石头。 

约束条件很大：最多$10^5$测试用例和测试的总输入大小高达$10^5$。 这立即排除了任何超出每个测试用例持续工作的模拟。 甚至一个$O(n)$如果不仔细管理，每个测试方法总体上会太慢，但在这里我们将看到该结构将问题分解为每个案例的恒定时间推理。 

堆大小的作用经常使简单的方法感到困惑，这是一个微妙的边缘情况。 人们可能会认为石子的数量很重要，但由于每堆石子都可以在一次移动中清空，因此实际值不会影响序列长度。 

例如，考虑一个测试用例：

 输入：```
n = 1
a = [1000000000]
```尽管这堆石子很大，但爱丽丝可以一举拿走所有石子，让鲍勃无能为力，立即失败。 因此，结果完全取决于堆的数量是奇数还是偶数，而不是堆的大小。 

另一个误导性的案例是：

 输入：```
n = 2
a = [1, 100]
```尽管第二堆很大，但游戏仍然由两步组成：每堆一步。 

## 方法

 暴力模拟将对每个动作进行明确的建模。 我们将维护一个指向当前堆的指针，并反复从中减去一些正数，从而可能模拟每次石头的移除。 然而，由于每堆最多可包含$10^9$石头，这种解释建议最多$10^9$最坏情况下每堆的操作数，这显然是不可行的。 

关键的观察是，最佳游戏总是立即清空当前堆。 留下棋子并没有任何战略上的好处，因为无论你如何在多个回合中分配移除，对手总是会面临相同的情况。 每堆有效地为游戏长度贡献了一步。 

这将整个游戏简化为固定序列：每一堆对应一个强制移动，玩家只需在一系列长度上交替移动$n$。 唯一剩下的问题是谁执行最后一步。 

所以这个游戏相当于对桩数进行了简单的奇偶校验。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(\sum a_i)$|$O(1)$| 太慢了|
 | 奇偶校验减少|$O(1)$每次测试|$O(1)$| 已接受 |

 ## 算法演练

 1.读取测试用例数量$T$。 每个测试用例都是独立的，因为它们之间没有任何状态。 
2. 对于每个测试用例，请阅读$n$，桩数。 桩的实际值与最终结果无关，因此可以读取并忽略它们。 
3. 仅根据是否确定获胜者$n$is odd or even. 如果$n$奇怪的是，爱丽丝走了最后一步； 如果$n$is even, Bob makes the last move.
 4. 输出“Alice”如果$n$是奇数，否则输出“Bob”。 

### 为什么它有效

 每堆棋子都为游戏贡献了一步，因为控制最左边的一堆棋子的玩家总是可以通过一个动作移除所有棋子。 一堆被清空后，它就不会再出现在游戏中，并且下一堆会在下一回合变得活跃。 因此，游戏总是精确地持续$n$移动，与桩的大小或分布无关。 由于玩家从爱丽丝开始交替移动，获胜者仅由奇偶性决定$n$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        arr = list(map(int, input().split()))
        
        if n % 2 == 1:
            print("Alice")
        else:
            print("Bob")

if __name__ == "__main__":
    solve()
```该解决方案读取每个测试用例并立即将其简化为奇偶校验。 该数组仅用于满足输入格式约束； 它的值不影响计算。 唯一关键的决定是基于$n$，因此算法在输入大小上以线性时间运行，但每个元素的工作量超出读取范围。 

这里一个常见的实施错误是试图模拟移除或推理单个石子的数量。 这会在不改变结果的情况下导致不必要的复杂性。

 ## 边缘情况

 一堆的最小输入演示了边界行为。 例如：```
1 1
10
```该算法立即分类$n = 1$作为奇数并输出 Alice。 模拟视图确认爱丽丝清空了唯一的一堆并立即获胜。 

两堆案例，例如：```
2 1
3 7
```交替移动：爱丽丝移走第一堆，鲍勃移走第二堆，让爱丽丝没有移动。 奇偶校验规则正确地预测了鲍勃。 

堆内的大值不会影响执行。 为了：```
3 1
1000000000 1000000000 1000000000
```每堆仍然对应于一次移动，因此由于奇数，结果仍然是爱丽丝$n$。
