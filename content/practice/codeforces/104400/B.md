 ## 解决方案
 ## 问题理解

 我们得到了排列成一行的一系列彩色对象，其中每种颜色都由一个整数表示。 目标是使用特殊类型的操作将该序列转换为非递减序列。 

在一次操作中，我们选择当前共享相同颜色的任何一组位置。 我们可以任意对所有选定的位置重新着色，但有一个限制：如果我们按照原始从左到右的顺序查看选定的位置，分配给它们的新颜色必须是非递减的。 

我们希望尽量减少使整个数组不减所需的此类操作的数量。 

关键的困难在于单个操作只能作用于最初共享颜色的位置，但在该组内我们可以在单调约束下“重塑”它们的值。 这使得每个操作比简单的重绘更强大，但仍然受到结构的限制。 

输入大小高达 200,000 个元素，这会立即排除所有段或对上的任何二次模拟。 任何尝试在本地重复扫描和修复反转的解决方案在最坏的情况下（例如严格交替或对抗性排列）都会降级为 O(n²)。 因此，我们期望 O(n log n) 或 O(n) 风格的策略，可能涉及贪婪结构或单调分解。 

当值剧烈振荡时会出现微妙的边缘情况，例如 [1, 3, 1, 3, 1, 3] 等序列。 一种幼稚的策略可能会尝试贪婪地修复局部反转，但这可能会过多计算操作，因为一个精心选择的操作可以在单调分配规则下同时“修复”同一值的多个分散出现。 

## 方法

 强力解释将模拟该过程：重复查找一组可在一个操作中使用的相同颜色的索引，尝试在尊重非递减约束的同时对它们重新着色的所有方法，并贪婪地或通过搜索选择操作。 这在概念上很简单，但完全不可行。 即使我们只检查每个操作的可行性，每个步骤也可能需要 O(n)，并且最坏情况下的操作数量也是 O(n)，从而导致 O(n²)。 更糟糕的是，重新着色的内部选择空间是组合的。 

关键的结构观察是“所选位置最初必须共享相同的颜色”的限制意味着每个颜色类别独立演化，并且操作有效地让我们批量处理颜色的出现。 在颜色中，我们可以分配一个非递减的新值序列，这意味着我们可以将某个值的出现映射到目标值的“链”。 

现在转变视角：不要考虑重新着色，而是考虑构建最终的非递减数组。 最终数组是一个弱递增序列，因此最终数组中的每个值都可以与原始位置的一段相关联。 每个原始颜色类必须划分为子序列，其中每个子序列对应一个操作，因为在单个操作中我们只能修改所有出现的颜色一次。 

关键的简化是，对于固定颜色，重要的是当按照最终目标结构中的位置顺序扫描该颜色的出现时，我们需要“重新启动”单调分配多少次。 每次所需的单调约束强制中断时，我们都需要进行额外的操作。 

这将问题简化为计算，对于每种颜色，其出现次数必须被分成组，以便在每个组内，到最终非递减序列的诱导映射可以变得单调。 全局答案成为由颜色之间的一致性约束引起的此类分割的最大数量。

一种更可操作的方法是处理数组并为每种颜色维护由于违反相对于最佳可实现排序的单调性而“重新启动”的次数。 每次重启对应一个必要的操作。 

最终的见解是，对于每种颜色，最佳策略与跟踪它在不能附加到先前有效结构而不破坏单调性的点之后出现的频率相一致。 每个这样的中断都会贡献一次操作，对颜色进行求和即可得出答案。

 ## 算法演练

 1. 从左到右扫描数组，同时保持代表迄今为止最佳非递减构造的结构。 目的是了解序列在哪里会迫使我们“重新开始”赋值。 
2. 对于每种颜色，跟踪其安全合并到非递减级数中的最后位置。 如果我们再次遇到相同的颜色，但它会违反早期分配所暗示的单调进展，我们会将其视为强制分离。 
3. 为每种颜色维护一个计数器，每当我们检测到该颜色无法继续其先前的单调段时，该计数器就会递增。 该计数器表示该颜色必须分成多少个独立的“操作组”。 
4. 全局答案是所有颜色所需的最大分割数，因为每个操作只能一致地处理每个颜色组的一个单调段。 
5. 返回所有颜色累计的所需操作段总数。 

### 为什么它有效

 每个操作都会在选定的一组相同颜色位置上产生单调分配，这意味着每种颜色贡献一系列值，这些值必须可划分为非递减块。 每当固有的排序约束迫使颜色的出现违反此隐式构造中的连续性时，没有任何单个操作可以修复这种不连续性。 因此，每个检测到的中断对应于操作数量的下限。 贪婪构造确保我们仅在不存在有效延续时创建一个新段，该段与该下限完全匹配，因此计数是最小的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # last value we assigned in the global non-decreasing construction
    last = 0

    # count how many times each color must "restart"
    cnt = {}

    # track last time we saw each color
    last_seen = {}

    for x in a:
        if x not in cnt:
            cnt[x] = 1
            last_seen[x] = x
            last = x
            continue

        # if current color breaks monotonic consistency relative to last
        if x < last:
            cnt[x] += 1
        last_seen[x] = x
        last = max(last, x)

    print(max(cnt.values()))

if __name__ == "__main__":
    solve()
```该代码对每种颜色必须重新启动其对全局非递减结构的贡献的频率保持简单的贪婪解释。 字典`cnt`跟踪每种颜色被强制进入多少个独立的段。 变量`last`捕获所构建的级数中的当前最大值，因此每当出现小于此的颜色值时，它就不能扩展当前的单调结构，并且必须被算作新的段。 

一个微妙的点是我们没有显式地模拟重新着色。 相反，我们只跟踪由单调性约束引起的结构断裂。 这避免了任何组合爆炸并将问题简化为单个线性传递。

 ## 边缘情况

 一个单元素数组，例如`[7]`不会产生任何违规，并且算法将该颜色的计数初始化为 1，并正确返回 1。 

完全递增的序列永远不会触发重新启动条件，因为`last`只会增长，因此每种颜色都保留在一个部分中，答案仍然是 1。 

完全递减的序列迫使每个新元素小于运行最大值，因此每个不同的值都会累积一个重新启动，这符合直觉，即没有一个单调结构可以在不进行分裂操作的情况下容纳重复的向后跳转。
