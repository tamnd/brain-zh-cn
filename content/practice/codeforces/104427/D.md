 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中除了根之外的每个顶点都只有一个父节点，因此所有边都自然地指向远离顶点 1 的方向。每个顶点都包含一定数量的人，只要存在从一个顶点到另一个顶点的有向路径并且两个顶点不同，这些人就会贡献“联系”。 

最初，该图就是这棵有根树。 我们可以将沿着路径的任何有向边链重复压缩为从链的开头到结尾的单个快捷边。 此操作删除该路径的中间结构并用直接跳转替换它。 因为这可以在任何路径上多次完成，所以我们可以有效地重构可达性如何沿着根到叶路径传播，但只能通过沿着现有有向路径跳过中间顶点。 

我们关心的数量是最终位于不同顶点的所有有序对人的总和，其中一个顶点在所有修改后可以通过有向边到达另一个顶点。 如果顶点 u 可以到达顶点 v，则 u 中的每个人都与 v 中的每个人形成联系。 

约束条件很大，顶点数高达 200,000 个，人数高达 100 万。 任何显式跟踪可达性对或模拟路径操作的解决方案都会太慢。 该结构是一棵树，因此唯一可行的解​​决方案在顶点数量上是线性或接近线性的，这强烈表明答案取决于全局结构优化而不是局部路径操作。 

天真的想法的一个微妙的失败案例来自于假设压缩路径保留可达性。 事实并非如此。 如果我们用直接边 u → v 替换链 u → a → b → v，则 u 不再一定到达 a 或 b。 这些中间顶点不再位于路径上，因此它们对接触对的贡献消失，除非它们在其他地方连接。 这意味着该操作可以减少而不是保留可达性，这是非直观行为的关键根源。 

另一个常见的陷阱是假设我们必须保留树结构。 经过足够的压缩后，结构可能会崩溃，从而使许多顶点在可达性方面成为叶子，即使它们是原始树中的内部节点。 

## 方法

 暴力解释是考虑路径压缩的每个可能序列并每次重新计算可达性关系。 即使我们在每个操作之后对图进行建模，可能的序列数量也是指数级的，因为每个根到叶路径都可以以多种不同的方式压缩，并且操作跨重叠路径进行交互。 这使得任何直接搜索都不可行。 

一个更结构化的观点是停止从边缘角度思考，而是从可达链角度思考。 在所有操作之后，每个顶点仍然位于原始树中的某个位置，但其传出可达性仅取决于压缩后它直接连接到哪个祖先。 任何顶点都可以有效地选择其根路径上的单个祖先作为其新的“父节点”，跳过中间的所有内容。 一旦这些选择被确定，可达性就成为这个新结构中的一个简单的祖先链。 

关键的观察结果是每个顶点都可以直接“提升”到根部。 如果顶点选择根作为其直接祖先，则它不再到达或穿过中间顶点。 这破坏了许多原本存在于更深树中的祖先-后代可达性。 由于接触是通过可达性创建的，因此减少深度会减少对的数量。

这导致了令人惊讶的简单极值配置。 如果除根之外的每个顶点都直接连接到根，则除根之外的任何顶点都不能到达任何其他非根顶点。 根到达每个人，但没有其他顶点传播进一步的接触。 任何更深的结构只会增加额外的中间可达性，从而增加接触的数量。

 ## 算法演练

 1. 计算树中的总人数。 这是所有顶点上所有 Ci 的总和。 
2. 单独确定根贡献，因为联系人仅计算不同顶点中的人对。 
3. 从概念上构建最优结构：除根之外的每个节点都通过压缩路径直接连接到根。 这确保了任何根到叶路径上不再有中间顶点。 
4. 在这个结构中，跨不同顶点的唯一有向可达性来自于根到每个其他顶点。 任何非根顶点都不能到达任何其他顶点，因为它们都没有超出其自身的传出结构。 
5. 计算由根引起的接触。 根中的每个人都可以到达根外的每个人，因此贡献量是 C1 乘以根外的总人数。 

### 为什么它有效

 任何将顶点 u 作为根到叶路径上的中间点的配置都会迫使 u 参与额外的可达性对：u 将到达该结构中位于其下方的所有顶点。 将 u 压缩到路径之外会严格减少多个顶点同时可达的后代的数量。 由于根到叶路径上的每个内部顶点都对后代可达性有积极贡献，因此消除所有这些内部顶点可以最大限度地减少可达对。 避免引入额外的中间可达性同时仍保持从根部的连接的唯一配置是以根部为中心的完全扁平的星形。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    p = list(map(int, input().split()))
    c = list(map(int, input().split()))

    total = sum(c)
    root = c[0]

    # Only root reaches everyone else in optimal structure
    print(root * (total - root))

if __name__ == "__main__":
    main()
```实现只需要所有人的总和以及根部的人数。 父数组是无关紧要的，因为最优策略完全忽略了原始深度结构。 

关键的实现细节是避免构建任何图形表示。 任何模拟压缩或构建邻接列表的尝试都是不必要的，只会增加开销。

import sys, io

def solve():
    input = sys.stdin.readline
    n = int(input())
    p = list(map(int, input().split())) if n > 1 else []
    c = list(map(int, input().split()))
    total = sum(c)
    print(c[0] * (total - c[0]))

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    old_stdout = sys.stdout
    sys.stdout = out
    solve()
    sys.stdout = old_stdout
    sys.stdin = old_stdin
    return out.getvalue()

 ## 边缘情况

 对于单顶点情况，输入为：```
1
1
```没有其他顶点，因此不存在有序的不同顶点对。 该算法计算总计等于根，因此结果为零，与联系人的定义匹配。 

对于所有质量都集中在根部的情况，例如：```
3
1 1
10 0 0
```总数等于根值，因此外部根质量为零。 计算正确地产生了零接触，这表明在根之外无法联系到任何人。 

对于任何更深的树，例如：```
4
1 2 2
1 2 3 2
```原始结构将创建多个祖先-后代交互，但最佳压缩会折叠根下的所有内容，只留下根发起的可达性。 这确保了中间顶点永远不会累积后代链，而这正是减少接触计数的原因。
