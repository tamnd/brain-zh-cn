 ## 解决方案
 ## 问题理解

 我们被赋予了律师之间的直接关系。 每一种关系都表示一个律师信任另一个律师，这种信任具有非常具体的操作意义：如果B律师信任A律师，那么A就为B提供辩护。 

因此，输入定义了一个有向图，其中边 A→B 意味着 A 为 B 辩护。一名律师可以为许多其他律师辩护，如果至少有一名其他律师为他们辩护，则该律师被视为无罪，这意味着在该有向图中至少有一条传入边。 

有一个全局异常改变了有效性的结构。 如果两名律师同时为对方辩护，这意味着他们之间存在长度为 2 的定向循环，那么这两名律师都会自动被宣告有罪，无论他们可能有任何其他传入辩护。 

任务是确定在所有信任关系都是固定且无法修改的情况下，根据这些规则是否有可能每位律师都被无罪释放。 

限制多达二十万名律师和二十万名亲属。 这立即意味着任何解决方案在图的大小上都必须是线性的或接近线性的。 O(NM) 或 O(N^2) 风格的方法将远远超出可行的限制，而 O(N + M) 或 O(M log N) 方法是可以接受的。 

这里的一个天真的错误是只关注“至少一个来防”条件而忽略了共同防御规则。 例如，在 1 ↔ 2 和 3 ↔ 4 这样的图中，每个节点至少有一个入边，但由于相互防御，两对都无效，因此答案为“否”。 

另一个微妙的失败情况是节点根本没有传入边。 即使图在其他方面很密集，单个这样的节点也使得普遍无罪释放变得不可能。 

## 方法

 直接的暴力解释将独立模拟每个律师的条件。 对于每个节点，我们将扫描所有传入边以验证至少存在一个。 之后，我们将扫描所有节点对，检测是否存在互防对，并使这些节点失效。 

这在概念上是有效的，因为它完全遵循定义。 然而，在最坏的情况下，扫描所有对的相互关系的成本是二次的。 对于多达 200,000 个节点，成对检查将意味着多达 4 × 10^10 次比较，这是不可行的。 

关键的观察结果是这两个条件都是边缘局部的。 节点是否具有传入边可以通过简单的入度计数器来跟踪。 可以在输入处理期间通过在哈希集或邻接集中标记有向边并验证反向边是否已经存在来检查是否存在互防。 

这将问题从所有对的全局推理减少到每个边缘的恒定时间检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N² + M) | O(N + M) | 太慢了|
 | 最佳 | O(N + M) | O(N + M) | 已接受 |

 ## 算法演练

 我们将这种情况建模为有向图，其中每个信任关系都成为有向边。 

### 步骤

 1. 读取所有边并构建有向图的表示。 

每条边 A → B 都表示 A 防守 B。 
2. 维护数组`indegree`在哪里`indegree[v]`计算有多少律师为律师诉辩护。 

每次我们读取边 A → B 时，我们都会递增`indegree[B]`。 

这直接反映了每位律师是否至少接受过一次辩护。 
3.同时，将所有有向边存储在一个集合中。 

这允许恒定时间检查是否存在反向边缘。 
4. 对于每条边 A → B，检查是否存在反向边 B → A。 

如果是，我们立即知道存在互防对，这使得配置无效。 
5. 处理完所有边后，验证每个律师的入度至少为 1。 

如果任何律师的学位为 0，则不能被无罪释放。 
6. 如果两个条件都满足，则输出YES。 否则输出NO。 

### 为什么它有效

 该算法直接对问题规则隐含的两个必要条件进行编码。 内度条件确保每位律师至少获得一次辩护。 相互边缘检查强制执行这样的规则：任何一对相互防御都会自动使两个参与者失效。 

由于这两个条件仅依赖于局部边缘信息，并且每条边缘都被检查一次，因此不存在隐藏的全局依赖性。 任何有效的配置都必须满足这些条件，任何违反这些条件的行为都直接对应于问题陈述中的规则违反。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    indeg = [0] * (n + 1)
    edges = set()
    
    bad = False
    
    for _ in range(m):
        a, b = map(int, input().split())
        # a defends b => a -> b
        indeg[b] += 1
        edges.add((a, b))
    
    # check mutual defense
    for a, b in edges:
        if (b, a) in edges:
            bad = True
            break
    
    if bad:
        print("NO")
        return
    
    for i in range(1, n + 1):
        if indeg[i] == 0:
            print("NO")
            return
    
    print("YES")

if __name__ == "__main__":
    solve()
```该解决方案将两个约束完全分开。 入度数组在输入循环中更新，因此该部分不需要第二次遍历邻接表。 边缘集纯粹用于检测相互对； 使用集合可确保 O(1) 平均查找时间。 

一个微妙的实现细节是，在读取所有输入之后而不是在插入期间检查相互边。 这两种方法都有效，但后处理使逻辑更简单，并避免丢失两个方向稍后出现在输入中的情况。

 ## 边缘情况

 最小边缘情况是 N = 1 且 M = 0 时。单个律师没有收到辩护，因此算法由于入度为零而正确输出“否”。 

在像 1 → 2 这样的两节点单边情况下，即使节点 2 得到了适当的防御，节点 1 的入度也为零。 入度扫描立即捕捉到这一点，并产生 NO。 

在像 1 ↔ 2 这样的对称对中，两个节点的入度至少为 1，因此简单的解决方案可能会错误地返回 YES。 相互边沿检测可以捕捉到这一点并正确拒绝该配置。 

在边形成长链的稀疏图中，正确性完全取决于检测链中入度为零的第一个节点。 该算法自然地处理这个问题，因为入度累积与直接邻居之外的结构无关。
