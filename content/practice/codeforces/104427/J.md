 ## 解决方案
 ## 问题理解

 我们有一排学生，每个学生都标有一个班级编号。 该过程会重复删除同一班级的两名学生，每次删除都会贡献删除前他们在队列中当前位置之间的距离。 移除一对后，线会压缩，因此剩余学生的位置会向左移动以填补空白。 

这个过程一直持续到没有班级至少有两名剩余学生为止。 每个类独立地贡献几个删除操作，等于其频率的一半，困难在于每次删除的值取决于它的执行时间，因为较早的删除通过压缩改变了后面的位置。 

任务是选择每个班级内学生的配对以及删除这些配对的顺序，以使总累积距离最大化。 

限制很大：所有测试用例的学生总数可能达到 700 万。 这立即排除了对动态过程的任何模拟。 任何重复更新列表、显式维护位置或重新计算每次操作距离的方法都会太慢。 该解决方案必须本质上将问题简化为对输入的线性或近线性扫描。 

如果我们假设每个类都可以独立处理而不考虑交互，就会出现微妙的失败情况。 例如，在像这样的混合序列中`1 2 1 2`，第 1 类和第 2 类内的配对独立地给出正确的配对，但移除的顺序会改变最终分数。 如果我们首先删除类别 1，则类别 2 的位置压缩方式与我们首先删除类别 2 的情况不同，从而导致总数不同。 因此，任何正确的解决方案都必须考虑跨类干扰，而不仅仅是每类结构。 

## 方法

 一个简单的解决方案会明确地模拟该过程。 我们维护当前列表，重复选择至少有两次剩余出现的任何类，删除一对，计算它们当前的距离，并更新结构。 即使使用平衡树或链表，每次删除都需要更新位置或维护订单统计信息。 由于具有多达 700 万个元素，并且可能有数百万个删除项，因此由于重复的索引更新和范围变化，这种方法在实践中至少会退化为二次行为。 

关键的观察是，尽管职位发生了变化，但它们变化的方式是高度结构化的。 每次删除都会删除两个元素，并且删除位置右侧的每个元素都会向左移动正好两个元素。 这意味着剩余元素的相对顺序永远不会改变，并且仅应用均匀的压缩效果。 

我们可以用不同的方式重新解释分数。 每对贡献一个基值，等于原始索引中其端点之间的距离，但只要早期删除严格发生在两个端点之间，该值就会减小。 每一次这样的早期删除都会使贡献恰好减少两倍。 因此，最终得分可以被视为固定间隔长度的总和减去由表示移除的间隔之间的交集引起的惩罚。 

这种重构将问题转化为为每个类别选择一个配对，然后选择处理间隔的顺序，以使交叉惩罚最小化。 出现的结构是，类内的最佳配对总是在对称出现之间，第一个与最后一个，第二个与倒数第二个，依此类推。 任何偏离对称性的行为都只会减少总跨度，而不会改善相互作用结构，因为较长的交叉只会增加惩罚，而不会提供补偿效益。 

一旦间隔固定，剩下的问题就变成了间隔上的排序问题。 最佳策略是按正确端点的升序处理间隔。 这确保了在处理间隔时，所有较早的间隔要么完全位于其左侧，要么以可有效计数的受控方式部分重叠。 可以使用区间端点位置上的芬威克树来跟踪惩罚。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 直接模拟| O(N²) | O(N) | 太慢了|
 | 间隔缩小+扫描+分威克| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 1. 对于每个类，收集它出现在数组中的所有索引。 这些索引自然排序，因为我们从左到右扫描。 
2. 通过将第一个与最后一个、第二个与倒数第二个等匹配来对每个类中的出现进行配对。 这会为每个类别生成一组不相交的间隔。 动机是任何最佳解决方案都可以转换为这种结构，而不会减少总碱基贡献，因为配对极值可以最大化一组固定端点内的距离。 
3. 将每一对视为一个区间$(l, r)$， 在哪里$l$和$r$是数组中的原始位置。 
4. 通过增加右端点对所有区间进行排序。 选择此顺序是为了在处理某个时间间隔时，所有较早的时间间隔的结束时间不晚于当前时间间隔。 
5. 按此顺序扫描间隔。 在位置上维护一个 Fenwick 树，我们在其中插入已处理间隔的左端点。 
6. 对于当前间隔$(l, r)$，计算有多少个先前处理的区间将端点严格保留在内部$(l, r)$。 每个这样的间隔都会产生一个单位的干扰，从而使最终分数减少固定量。 
7.添加基础贡献$r - l$到答案，然后减去从 Fenwick 查询获得的干扰计数的两倍。 
8. 插入$l$进入 Fenwick 树并继续。 

### 为什么它有效

 在我们修复端点后，每个类都会贡献独立的对，因此不同类之间的唯一耦合来自压缩效应，这表现为以定向方式重叠的间隔之间的相互作用。 对称配对确保每个类别的碱基贡献最大，并且任何替代配对仅缩​​短至少一个间隔，而不会以有益的方式减少其相互作用计数。 

通过增加右端点的扫描保证了当我们处理一个区间时，所有先前活动的区间都被完全确定，并且它们的左端点恰好形成计算它们在产生惩罚的方向上与当前区间相交的次数所需的集合。 芬威克树有效地维护了这个集合，确保每次交互都以正确的符号精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 2)

    def add(self, i, v=1):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_sum(self, l, r):
        if l > r:
            return 0
        return self.sum(r) - self.sum(l - 1)

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    pos = [[] for _ in range(n + 1)]
    for i, v in enumerate(a, 1):
        pos[v].append(i)

    intervals = []

    for v in range(1, n + 1):
        lst = pos[v]
        l, r = 0, len(lst) - 1
        while l < r:
            intervals.append((lst[l], lst[r]))
            l += 1
            r -= 1

    intervals.sort(key=lambda x: x[1])

    fw = Fenwick(n)
    ans = 0

    for l, r in intervals:
        inside = fw.range_sum(l + 1, r - 1)
        ans += (r - l) - 2 * inside
        fw.add(l, 1)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先按值对位置进行分组，然后为每个类构造对称对。 这些成为间隔。 按正确端点排序可确保一致的处理顺序。 芬威克树存储已处理间隔的左端点，对于每个新间隔，我们计算其中有多少个严格位于其中。 每个这样的交集都对应于先前的删除，该删除将当前对的有效贡献恰好减少了两倍。 

一个常见的错误是试图模拟指数的动态变化。 该解决方案通过根据原始坐标修复所有相互作用来完全避免这种情况，原始坐标始终保持稳定。 

## 工作示例

 考虑一个简单的例子：`1 2 1 2`类 1 生成区间 (1, 3)，类 2 生成区间 (2, 4)。 按右端点排序后，我们处理 (1, 3)，然后处理 (2, 4)。 

| 步骤| 间隔| 芬威克活跃| 内数 | 贡献 |
 | --- | --- | --- | --- | --- |
 | 1 | (1, 3) | {} | 0 | 2 |
 | 2 | (2, 4) | {1} | 1 | (2) - 2 = 0 |

 总计为 2。 

这表明重叠直接减少了第二个区间的贡献，因为一个较早的左端点位于其中，代表结构干扰。 

现在考虑：`1 1 2 2 3 3`间隔为 (1,2)、(3,4)、(5,6)。 根本没有重叠，因此所有贡献都只是距离 1 + 1 + 1 = 3，并且 Fenwick 查询始终返回零。 这表明非重叠结构获得了最大可能的分数。

 ## 边缘情况

 一个关键的边缘情况是某个类的所有出现都已经是连续的。 例如，`1 1 1 1`。 该配对产生区间 (1,4) 和 (2,3)。 芬威克树一开始是空的，所以这两个贡献都纯粹是它们的长度。 由于没有其他类干扰，因此顺序并不重要，并且总数变为最大。 

另一种情况是完全交替的序列，例如`1 2 1 2 1 2`。 这里每个音程都与其他音程严重重叠。 该算法正确地计算 Fenwick 结构中的每个重叠，确保当前间隔内先前插入的每个左端点都被精确计算一次，与早期删除引起的压缩损失相匹配。
