 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 在每个测试用例中，一系列点位于水平线上，从左到右排序。 每个点都有一种颜色。 我们可以在连接点对的直线上方绘制曲线，但有三个限制：曲线必须连接两种不同的颜色，曲线内部不允许交叉，曲线可以在端点处相接触但不能共享任何内部点。 

任务不仅是计算可以绘制多少条曲线，而且还要显式输出一种有效的配置，以实现最大可能的曲线数。 

几何约束转化为对索引对如何相互作用的结构限制。 每条曲线对应于两个指数之间的间隔。 当两条曲线的端点交错时，两条曲线在其内部恰好相交，这意味着我们无法选择形成像 i < j < k < l 且具有边 (i, k) 和 (j, l) 的模式的对。 但是，嵌套是可以的，因为嵌套间隔不会交叉。 

对颜色的限制将所有相同颜色对排除在考虑范围之外。 目标是选择最大可能的有效非交叉区间集。 

这些约束允许所有测试用例最多有 200,000 个点。 任何尝试检查所有对的解决方案都是二次的，并且会立即失败。 这推动我们走向线性或近线性结构，其中每个点仅参与恒定量的工作。 

一个微妙的点是曲线可以共享端点。 这意味着单个点可以在多条曲线中使用，因此我们不是在解决匹配问题。 这种区别很重要，因为它允许比典型的非交叉匹配更密集的结构。 

一个常见的失败案例来自于试图将其视为最大匹配问题。 例如，在像 1 2 1 2 这样的序列中，匹配视图可能表明只能有两对，但实际上可以在没有交叉的循环结构中绘制四条曲线。 主要区别在于顶点在使用时不会被消耗。 

## 方法

 强力解释将考虑具有不同颜色的所有对 (i, j)，并尝试贪婪地添加它们，同时检查新曲线是否与任何先前添加的曲线相交。 这需要维护间隔的动态结构并检查每个候选对的兼容性。 即使进行有效的间隔检查，在最坏的情况下也有 θ(N^2) 个可能的对，这太大了。 

关键的观察结果是，最佳图纸的结构极其严格。 一旦点被放置在一条线上并且禁止交叉，任何最大配置都会与数组的自然顺序对齐。 特别是，有用的曲线几乎总是对应于局部邻接或单个全局环绕连接。 

我们可以将注意力限制在连续点之间的边缘上，而不是选择任意对。 i 和 i+1 之间的任何曲线始终不与所有其他此类连续曲线相交，因为它们占据线上的不相交线段。 这立即给出了安全基线构造。 

在建立所有有效的连续连接后，我们观察到整个结构仍然形成单个外部边界，末端可能未使用电位。 如果第一个点和最后一个点具有不同的颜色，我们可以安全地将它们与环绕所有中间点的大外弧连接起来。 该外部弧不与任何内部连续弧相交，因为它严格位于所有内部弧之上并包围整个结构。 

这将问题简化为相邻元素之间的纯粹本地检查加上一个可选的全局连接。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过有效性检查对所有配对进行暴力破解 | O(N^2) | O(N^2) | O(N) | 太慢了|
 | 相邻+边界连接施工 | O(N) | O(N) | 已接受 |

 ## 算法演练

 1. 从左到右遍历数组并考虑每对相邻的点 i 和 i+1。 如果它们的颜色不同，请在它们之间画一条曲线。 这始终是安全的，因为每个这样的曲线在线上占据其自己的区间并且不能与另一个相邻区间的内部重叠。 
2. 将所有这些基于邻接的曲线存储为答案的基本集。 在此阶段，每条曲线都对应于直线的最小线段，并且由于间隔不相交，它们都不能交叉。 
3. 检查第一个点和最后一个点的颜色是否不同。 如果存在，请添加一条连接索引 1 和索引 N 的附加曲线。该曲线位于所有中间点之上并跨越整个区间，因此它不会与任何先前添加的相邻曲线相交。 
4. 输出所有采集到的曲线。 

### 为什么它有效

 所有邻接边都对应于线的不相交子段，因此它们形成没有内部相交的平面结构。 从第一个点跨越到最后一个点的任何附加曲线都会包围整个配置，因此不能跨越任何内部邻接段。 由于所有禁止的相互作用都是区间的交叉，并且我们的构造仅创建不相交的区间或包含所有其他区间的单个外部区间，因此不会发生违反几何约束的情况。 

最大性源于以下事实：具有不同颜色的每个相邻对都必须可用于任何最佳解决方案：跳过这样的对不能为其他地方的严格更好的非交叉结构创造空间，因为涉及这些端点的任何替代连接要么重用相同的本地段，要么创建与相邻段的交叉。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))

        res = []

        # take all adjacent differing-color edges
        for i in range(n - 1):
            if a[i] != a[i + 1]:
                res.append((i + 1, i + 2))

        # optional outer edge
        if a[0] != a[-1]:
            res.append((1, n))

        print(len(res))
        for u, v in res:
            print(u, v)

if __name__ == "__main__":
    solve()
```该解决方案对每个测试用例扫描一次数组，并用不同的颜色记录每个相邻的对。 这些边与安全的不相交的局部弧完全对应。 之后，它检查端点，并可能添加连接端点的单个全局弧。 

操作顺序很重要，因为必须在固定所有局部弧之后添加外部弧，确保它在概念上环绕它们，而不是在构造过程中交错。

 ## 边缘情况

 关键的边缘情况是所有相邻点共享相同的颜色。 在这种情况下，不会生成邻接边，也不会添加外边，因为端点也共享相同的颜色。 该算法正确输出零曲线，与每个潜在对无效的事实相匹配。 

当序列交替颜色时会出现另一种边缘情况。 这里每个相邻对都是有效的，并且结构变得最密集。 还添加了可选的外边缘，产生最大可能的配置，没有任何交叉，因为它环绕所有内部段。 

最终的结构边缘情况是仅端点颜色不同但内部不变。 在这种情况下，两个块之间的边界处恰好出现一条邻接边，并且由于端点共享相同的颜色而不会添加外边，从而防止出现任何额外的长弧。
