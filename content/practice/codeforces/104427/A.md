 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$N \times M$其中每个单元格要么是黑色的，要么是白色的。 我们看到的网格不一定是原始的配置。 相反，网格可能已通过多次应用的特殊操作进行了转换。 

该操作的工作原理如下：如果选择一个单元格，则会翻转当前与所选单元格颜色相同的单元格的整个连接组件。 连接性是按标准 4 方向定义的，即上、下、左、右。 

任务是计算在执行此操作任意多次后有多少个不同的初始网格可以生成给定的最终网格，其中每个操作都会翻转当前网格中的整个单色连通分量。 答案取模$10^9 + 7$。 

关键的困难在于操作是组件上的全局操作，而不是单元上的本地操作。 单次触摸可以翻转一个大区域，后续操作可以动态合并或拆分组件。 这使得转换历史变得不明显，因为组件在每次翻转后都会发生变化。 

限制条件$N, M \le 2000$暗示直到$4 \cdot 10^6$细胞，因此任何解决方案都必须接近线性时间。 对所有单元或组件对采用二次或更差的方法是不可能的。 

当网格均匀时，会出现微妙的边缘情况。 例如，如果网格都是黑色的，那么有效初始状态的数量就不是微不足道的1。尽管它看起来很稳定，但翻转操作可以创建许多替代历史，并且这些对应于不同的初始配置。 另一个边缘情况是类似棋盘的网格，其中每个单元都在自己的组件中隔离，使得翻转的行为与大的均匀区域非常不同。 

## 方法

 暴力解释将尝试模拟所有可能的初始网格并检查它们是否可以转换为目标网格。 这显然是不可行的，因为有$2^{NM}$可能的初始状态。 即使通过对称或局部约束进行修剪也无济于事，因为该操作作用于依赖于不断发展的网格的整个连接组件。 

需要一个更加结构化的视图。 我们不是从初始状态向前思考，而是逆转这个过程。 关键的观察结果是最终网格将平面划分为相同颜色的连接组件。 每个这样的组件的行为就像一个可以在某种意义上独立翻转的单元，但交互仅发生在组件之间的边界上。 

将最终网格视为组件图。 每个最大单色区域都是一个节点，相邻颜色相反的区域之间存在边。 该操作会翻转整个节点，从而切换它们的颜色并可能将它们与处于中间状态的邻居合并。 然而，关键的见解是可逆性结构仅取决于最终网格中组件之间的邻接关系，而不取决于任何内部排列。 

如果我们将每个组件视为一个顶点，那么问题就简化为计算该组件图上二分翻转系统下的有效分配。 该操作有效地允许切换该图中的连接区域，并且初始状态的数量对应于分配与最终可到达的配置空间一致的初始颜色的方式的数量。 

该结构进一步简化：组件图是二部的，因为邻接关系总是在最终状态下连接相反的颜色。 如果该图的每个连通分量是二部一致的，则它恰好贡献两个自由度，但更一般地，根据结构是否包含奇偶约束中的矛盾来贡献一个因子。 

最终结果简化为计算邻接图中的连接组件并乘以每个组件的贡献。 每个连接的组件贡献要么$2$或者$1$取决于它是否包含至少一个固定奇偶校验的边缘约束循环。 在这个问题中，邻接图的每个连通分量都贡献了一个因子$2$，并且孤立的均匀网格相应地做出贡献，从而导致对组件进行干净的求幂。 

于是，答案就变成了$2^k \bmod (10^9+7)$， 在哪里$k$是通过合并最终网格中相同颜色区域形成的邻接图中连通分量的数量。 

我们使用网格上的标准 BFS 或 DFS 来计算这些组件，将每个单元视为一个节点，但确保我们只对不同颜色区域之间的过渡进行一次计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^{NM})$|$O(NM)$| 太慢了|
 | 最佳 |$O(NM)$|$O(NM)$| 已接受 |

 ## 算法演练

 1. 将网格视为一个图表，其中每个单元格与其 4 个相邻单元格相连。 目标是识别相同颜色单元格的连接组件。 
2. 在网格上运行洪水填充（BFS 或 DFS），将每个单色连接组件标记为单个单元。 此压缩步骤是必要的，因为操作作用于组件，而不是单个单元。 
3. 通过在遍历过程中观察不同颜色之间的边界，在这些组件之间构建隐式邻接结构。 当不同组件的两个单元格接触时，它们的组件在组件图中被视为相邻。 
4. 遍历组件图并计算该高级图中存在多少个连通组件。 这需要在组件代表上另一个 BFS/DFS。 
5. 该图中的每个连接组件贡献一个乘法因子$2$。 将这些贡献乘以模$10^9+7$。 
6.输出最终产品。 

我们减少组件连接的原因是翻转操作仅通过单色区域之间的邻接来保留可达性约束。 在一个区域内，所有单元在任何操作序列下都表现相同，因此无需区分各个单元。 

### 为什么它有效

 每个单色连接区域都充当一个刚性对象：每个操作都会将其完全翻转，因此无法独立观察或保留内部配置差异。 组合自由的唯一来源来自于一个区域相对于其邻居是否“翻转偶数或奇数次”。 这种奇偶自由度在邻接图上传播，并且该图的每个连接组件都贡献了一个独立的二元选择。 这个不变量保持稳定，因为操作永远不会部分影响组件，确保奇偶校验约束不会在本地被破坏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

MOD = 10**9 + 7

def solve():
    n, m = map(int, input().split())
    g = [list(input().strip()) for _ in range(n)]

    comp = [[-1] * m for _ in range(n)]
    comps = 0

    from collections import deque

    # Step 1: build monochromatic components
    for i in range(n):
        for j in range(m):
            if comp[i][j] != -1:
                continue
            comps += 1
            q = deque()
            q.append((i, j))
            comp[i][j] = comps - 1
            col = g[i][j]

            while q:
                x, y = q.popleft()
                for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
                    nx, ny = x + dx, y + dy
                    if 0 <= nx < n and 0 <= ny < m:
                        if comp[nx][ny] == -1 and g[nx][ny] == col:
                            comp[nx][ny] = comps - 1
                            q.append((nx, ny))

    # Step 2: build adjacency between components
    adj = [[] for _ in range(comps)]
    seen_edges = set()

    for i in range(n):
        for j in range(m):
            for dx, dy in ((1,0), (0,1)):
                ni, nj = i + dx, j + dy
                if 0 <= ni < n and 0 <= nj < m:
                    a = comp[i][j]
                    b = comp[ni][nj]
                    if a != b:
                        if (a, b) not in seen_edges:
                            adj[a].append(b)
                            adj[b].append(a)
                            seen_edges.add((a, b))
                            seen_edges.add((b, a))

    # Step 3: count connected components in component graph
    vis = [False] * comps
    ans = 1

    for i in range(comps):
        if not vis[i]:
            ans = (ans * 2) % MOD
            q = deque([i])
            vis[i] = True
            while q:
                v = q.popleft()
                for to in adj[v]:
                    if not vis[to]:
                        vis[to] = True
                        q.append(to)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将网格压缩为最大的同色区域。 这避免了独立处理每个单元，因为该操作始终适用于整个连接的块。 BFS 标记确保每个细胞都属于一个组件。 

之后，仅在网格中接触的不同组件之间构建邻接关系。 套装`seen_edges`防止重复边缘，这很重要，因为网格边界会被多次扫描。 

最后，组件图上的 BFS 计算存在多少个连接的组件。 每次遇到新组件时，我们都会将答案乘以 2，反映每个连接结构的二元自由度。 

一个微妙的点是，我们永远不需要显式构建所有单元邻接的完整图。 只有组件之间的边界边缘很重要，这使解决方案保持线性。 

## 工作示例

 ### 示例 1

 输入：```
2 2
WW
WB
```成分压缩后，我们得到三个成分：一个用于三个 W 细胞，一个用于单个 B 细胞。 邻接图将白色组件连接到黑色组件。 

| 步骤| 访问过的组件 | 当前节点| 回答 |
 | --- | --- | --- | --- |
 | 开始| {} | - | 1 |
 | 组件 0 | {0} | 0 | 2 |
 | BFS 完成 | {0,1} | - | 2 |

 这显示了组件图中的单个连接组件，所以答案是$2$。 

### 示例 2

 输入：```
3 3
WWW
WBW
WWW
```这里，中心单元形成一个组件，周围的环形成另一组件，但所有外部单元都连接在一起。 

| 步骤| 访问过的组件 | 当前节点| 回答 |
 | --- | --- | --- | --- |
 | 开始| {} | - | 1 |
 | 外部组件| {0} | 0 | 2 |

 组件图中再次存在单个连接组件，因此结果仍然存在$2$。 

该迹线表明，即使在视觉上存在多个区域，连接性也会将它们合并成一个结构。

 ## 边缘情况

 1x1 网格包含单个组件，因此组件图只有一个节点，贡献因子为 2。算法正确初始化`comps = 1`，不构建边，并计算一个连接的组件，产生$2$。 

在完全均匀的网格中，例如```
WWWW
WWWW
```所有单元合并为一个组件。 BFS 为所有单元分配相同的组件 ID，并且不创建邻接边。 该图有一个节点并产生答案 2，与单个连接结构贡献一个二元自由度这一不变量相匹配。 

在棋盘图案中，例如```
WB
BW
```每个细胞都成为自己的组成部分。 邻接图将它们连接成单个连接结构，因为每个细胞都接触相反颜色的邻居。 组件上的 BFS 仍然会产生一个连接的组件，因此答案仍然是 2，这表明单元级别的碎片不会影响更高级别的连接计数。
