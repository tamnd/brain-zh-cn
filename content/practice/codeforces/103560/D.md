 ## 解决方案
 ## 问题理解

 我们有一盒糖果，其中每种糖果都属于某种类型。 对于每种类型，我们可以计算该类型糖果的存在数量。 我们想从这个池子中选择一些糖果来组装一份“礼物”。 

该限制是结构性的：如果我们查看最终的礼物并按类型对糖果进行分组，则每种类型的所选糖果数量必须与其他所选类型不同。 换句话说，如果类型 1 在礼物中出现 5 次，类型 2 出现 3 次，则其他选择的类型不允许恰好出现 5 次或 3 次。 有些类型可能会完全跳过，对于所选类型，我们可以自由地取任何正数，直至可用供应量。 

目标是最大化礼物中糖果的总数，即所有选定类型的选定数量的总和。 

这是一个全局分配问题：我们在容量限制下将整数“配额”分配给类型（每种类型都有最大频率），同时还强制所有选定的配额成对不同，并且我们希望最大化它们的总和。 

限制很大，跨查询的总输入大小高达大约二十万个糖果。 这排除了对所有值的任何二次或每个查询嵌套模拟。 任何比每个查询大致线性慢的操作都会失败。 

当频率非常小或非常重复时，就会出现微妙的边缘情况。 例如，如果所有类型出现多次，即使存在容量，我们也无法分配大的相等配额。 相反，如果许多类型出现一次，我们不能将所有类型都赋值为 1，因为禁止重复。 

例如，假设我们有三种类型，每种类型出现两次。 简单的方法可能会分配 2, 2, 2，但这违反了独特性规则。 正确的行为是分配 2, 1, 0，生成总和 3。另一种边缘情况是单一类型出现多次，答案就是全部出现。 

这些模式表明我们不仅仅是最大化每种类型的使用，而是仔细间隔分配的值。 

## 方法

 蛮力策略会尝试通过回溯或贪婪猜测为每种类型分配一个值。 对于每种类型，我们可以尝试从频率到 1 的所有可能计数，并确保不使用重复项。 这立即导致组合爆炸。 即使我们只考虑高达 n 的频率，尝试跨 k 个类型的所有分配也会产生类似于每种类型 n 个选择的结果，这在 k 中呈指数关系。 多达二十万个元素，这是不可能的。 

关键的观察结果是，只有最终选择的糖果才重要，而不是使用哪种糖果。 一旦我们计算出每种类型的频率，问题就变成了：为每种类型分配一个整数 x_i，使得 0 ≤ x_i ≤ c_i，所有正 x_i 都是不同的，并最大化 x_i 的总和。 

这是一个经典的贪心结构。 如果我们按照频率递减的方式对类型进行排序，我们总是希望为能够负担得起的类型提供更大的配额。 原因很简单：大的频率可以提供灵活性，因此我们应该为它们保留大的值。 较小的频率受到限制，应该吸收较小的配额或被放弃。 

然后我们从最大可能的配额向下构建解决方案。 我们维持愿意分配的最大可用配额。 对于按降序排列的每个频率，我们为其分配最大可能的有效数字，该数字严格小于我们之前使用的数字，同时也不超过其容量。 如果容量太小，我们就取该容量并继续减少。

该结构之所以有效，是因为约束只是“所有分配的值必须不同”。 这立即表明递减序列是最佳的，因为分配值的任何排列都可以在不改变可行性的情况下进行排序，并且较大的值总是对总和贡献更大。

 ## 算法演练

 我们首先将输入压缩为每种糖果类型的频率。 这将问题从处理单个项目转变为处理多组能力。 

## 算法演练

 1. 计算每种类型出现的次数，生成频率数组。 此步骤是必要的，因为仅计算每种类型的物质，而不计算单个糖果的身份。 
2. 对频率数组进行降序排序。 这确保了我们始终尝试将大配额分配给实际上可以支持它们的类型，从而防止早期在小容量类型上浪费高值。 
3. 初始化变量`cap`作为一个非常大的数字，概念上是无穷大，但实现为类似 n 的东西，因为没有任何类型可以接受超过 n 颗糖果。 
4. 按排序顺序迭代频率。 对于每个频率`f`，我们要分配一个值`x`即 ≤ f 且严格小于最后指定的值`cap`。 
5. 设置`x = min(f, cap - 1)`。 这同时强制执行两个约束：我们不会超出可用性，并且我们通过严格减少分配来保持独特性。 
6.如果`x`变为零或负数，我们停止该过程，因为不可能进一步进行正分配。 所有剩余类型都不会做出任何贡献。 
7. 否则，添加`x`到答案并更新`cap = x`。 这确保了下一个分配的值必须严格小于，从而保持唯一性。 

该过程构建了所选计数的严格递减序列，每个计数均以相应的频率为界。 

### 为什么它有效

 在每一步中，我们都会在所有先前选择的值都不同且更大的约束下确定当前类型的最大可能有效值。 因为我们按降序处理频率，所以我们永远不会后悔尽早分配一个大值：任何稍后的分配都会有更小的容量，并且交换不会增加总和。 贪心不变量是，在处理 i 个类型之后，我们使用 i 个不同的正整数构造了最大可能的总和，每个正整数都尊重其频率范围，并且具有相同前缀长度的任何替代排列都不能超过它，而不违反顺序或可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q = int(input())
    for _ in range(q):
        n = int(input())
        freq = {}
        for _ in range(n):
            a = int(input())
            freq[a] = freq.get(a, 0) + 1

        vals = sorted(freq.values(), reverse=True)

        cap = n
        ans = 0

        for f in vals:
            x = min(f, cap - 1)
            if x <= 0:
                break
            ans += x
            cap = x

        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先将每个查询压缩为频率图，因为仅对每种类型进行计数。 按降序排序是贪婪策略的支柱，确保我们始终首先考虑最灵活的类型。 

变量`cap`跟踪剩余分配的最大可用值。 它从 n 开始，因为有效的分配不能超过糖果的数量。 在每一步中，我们都会通过频率和电流上限减一来进行钳位，以保证严格的减少。 

这里一个常见的错误是忘记`-1`当计算下一个允许值时。 如果没有它，可能会出现相等的连续分配，从而违反了唯一性约束。 

早间休息的时候`x <= 0`很重要，因为一旦我们不能分配至少 1，所有剩余的贡献都为零。

 ## 边缘情况

 一种边缘情况是所有频率均为 1 时。算法分配 1，然后为下一个分配 0，并立即停止。 对于许多类型，这会正确生成 1，而不是错误地对所有类型求和。 

另一种边缘情况是频率非常高的单一类型。 该算法正确地将其减少到 n-1，因为我们必须维护严格递减的正序列。 

第三种情况是当有很多中频时，例如多种类型，每种类型的频率都是 3。贪心顺序确保我们在前几种类型中分配 3,2,1，然后停止。 任何分配相等值的尝试都会违反约束，因此递减上限机制自然会强制执行正确性。
