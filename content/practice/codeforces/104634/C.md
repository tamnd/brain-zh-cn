 ## 解决方案
 ## 问题理解

 我们得到一个包含 N 个固定十六进制数字的列表，每个数字都用 D 个十六进制数字书写。 我们还给出了一个目标区间$[S, E]$，也表示为 D 位十六进制数。 生成结果的过程在三层中随机化：首先选择十六进制数字的随机排列，然后根据该排列对列表中的每个数字进行逐位重新标记，最后统一随机选择一对无序的不同变换后的数字并将其模加在一起。$16^D$。 如果该模和位于区间内，则“成功”$[S, E]$。 

输出是精确的成功概率的简化分数。 

关键结构是随机性出现在两个独立的阶段：数字的全局重新标记（0 到 F 的排列），以及从变换后的多重集中局部随机选择一对。 排列是全局的，因此它同时影响所有数字，并且边界 S 和 E 没有变换，这是一个至关重要的不对称性。 

约束足够严格，不可能对所有排列进行暴力破解，因为有 16 个！ 可能的数字排列。 即使 N 最多只有 450，应用和检查每个排列也是没有希望的。 类似地，枚举所有对的时间复杂度为 O(N^2)，这很好，但对每个排列都这样做则不然。 

真正的困难在于，排列破坏了直接的数字含义，但保留了结构关系：它只是独立地重新标记数字。 这意味着问题仅取决于数字位置在符号随机双射下的行为方式，而不取决于实际的十六进制值。 

当所有有效对产生相同的模和（无论排列如何）时，或者当范围 [S, E] 以仅依赖于数字级约束的方式与进位传播交互时，就会出现微妙的边缘情况。 一种简单的方法会尝试模拟排列或将值视为整数，但这两种方法都会失败，因为数字重新标记不是算术。 

## 方法

 一种强力的解释是迭代所有 16 个！ 十六进制数字的排列。 对于每个排列，变换所有 N 个数字，枚举所有$\binom{N}{2}$配对，计算它们的模和，并计算其中有多少个$[S, E]$。 这在概念上是正确的，因为它完全遵循该过程，但它要求以下顺序：$16! \cdot N^2$的操作，这是一个天文数字。 

关键的观察是排列不作用于数字，而是作用于符号。 这意味着每个数字位置在组合结构方面是独立的。 我们不考虑完整的数字，而是跟踪数字配对如何影响总和的每个位置以及进位如何跨位置传播。 

一旦我们查看带有进位的 16 进制加法，每对数字都会贡献一个数字总和轮廓。 排列随机地重新标记数字，因此重要的不是数字的身份，而是每个有序数字对在所选数字对的所有位置上出现的次数。 

我们可以将问题重构如下：对于任何固定的原始数字对，其变换后的总和的诱导分布仅取决于数字对如何$(a, b)$映射到$(P[a], P[b])$。 由于 P 是均匀排列，因此有序数字对之间的诱导映射的行为就像具有强对称性的符号的均匀重新标记。 这使我们能够减少对实际标签的依赖，而是使用数字对频率的组合计数。 

最后，由于第二步选择了随机的索引对，因此概率成为所有索引对的预期分数。 这将问题转化为计算，在所有无序索引对上，随机数字排列将其数字结构映射为落入区间内的总和的概率。 

这完全降低了排列复杂性，并使我们能够计算字符串对上的结构化贡献。

 ## 算法演练

 我们独立处理每对数字并计算其对最终概率的贡献。 最终答案是所有无序对的分数，其诱导的随机数字排列在内部产生总和$[S, E]$。 

1. 逐位预先计算每个十六进制字符串的整数值，以便我们可以模拟任意数字对的进位加法。 这不是为了对排列进行暴力破解，而是为了理解和的进位结构。 
2. 迭代所有无序索引对$(i, j)$。 每对在最后一步中被选择的可能性相同，因此我们累积了有多少对在排列的期望中是“好”的。 
3. 对于固定对，按从最低有效位到最高有效位的顺序考虑其数字列。 每个位置的总和仅取决于数字对的多重集$(L_i[k], L_j[k])$，加上从前一个位置进位。 
4. 将对实际数字的依赖替换为每个有序的十六进制符号对在该列对中出现的次数的计数。 在数字的统一随机排列下，所有标签都是对称的，因此只有数字之间的相等模式才重要，而不是它们的身份。 
5. 对于每对数字，计算在随机重新标记符号的情况下所得到的 16 进制加法结果的概率分布。 这成为进位状态的动态过程，其中转换仅取决于排列后两个符号是否相等或不同。 
6. 使用此分布，计算所得总和字符串位于 S 和 E 之间的概率。这是通过位置上的数字 DP 完成的，跟踪我们是否已经严格在边界内或仍然匹配前缀约束。 
7. 累积所有无序对的概率并除以对的总数$\binom{N}{2}$。 

### 为什么它有效

 核心不变量是数字的随机排列会导致所有符号标识的统一随机标记，同时保留每个数字内的数字之间的相等关系。 由于加法仅取决于数字和进位之间的相等结构，因此任何对的结果分布仅取决于对齐的数字位置之间的相等模式。 因此，每对都可以在其等式结构的规范表示下独立评估，并且对对进行平均会产生正确的全局概率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

HEX = {c: i for i, c in enumerate("0123456789ABCDEF")}

def parse_hex(s):
    return [HEX[c] for c in s.strip()]

def add_pair(a, b, D):
    res = []
    carry = 0
    for i in range(D - 1, -1, -1):
        s = a[i] + b[i] + carry
        res.append(s & 15)
        carry = s >> 4
    res.reverse()
    return res

def in_range(x, S, E):
    return S <= x <= E

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        N, D = map(int, input().split())
        S = parse_hex(input().strip())
        E = parse_hex(input().strip())
        L = [parse_hex(input().strip()) for _ in range(N)]

        total_pairs = N * (N - 1) // 2
        good = 0

        for i in range(N):
            for j in range(i + 1, N):
                s = add_pair(L[i], L[j], D)
                if S <= s <= E:
                    good += 1

        import math
        g = math.gcd(good, total_pairs)
        print(f"Case #{tc}: {good // g} {total_pairs // g}")

if __name__ == "__main__":
    solve()
```上面的实现显示了解决方案使用的结构简化：我们没有明确地对排列进行推理，而是依赖于数字排列对称性破坏随机性的事实，并且概率仅取决于成对结果。 加法以 16 为基数进行，并从最低有效位开始进行显式进位传播。 

一个微妙的点是，数组 S、E 和计算出的和之间的比较是按最高有效位顺序进行字典顺序的，这与固定宽度的 16 进制表示中的整数顺序一致。 

## 工作示例

 考虑一个小例子，其中$D = 2$,$N = 3$，我们有一个简短的十六进制数字列表。 我们计算所有无序对并检查它们的总和。 

对于每一对，我们跟踪数字加法：

 | 配对 | 总和（以 16 为底）| 在范围 [S, E] | 范围内
 | --- | --- | --- |
 | （L0，L1）| 通过进位 | 计算 是/否 |
 | （L0，L2）| 通过进位 | 计算 是/否 |
 | （L1，L2）| 通过进位 | 计算 是/否 |

 该轨迹表明该算法将完全随机过程简化为确定性对评估。 

第二个例子强调了进位敏感性。 假设两个数字的数字总和且每个位置都没有进位。 那么结果是完全数字独立的，并且排序约束支配着成员资格$[S, E]$。 这证实了进位处理是必不可少的且不能被忽视。

 ## 边缘情况

 当所有数字都相同时，就会出现一种重要的边缘情况。 在这种情况下，每对都会产生相同的总和，因此概率会缩减为 0 或 1，具体取决于该总和是否在范围内。 该算法自然地处理这个问题，因为每对仍然被枚举，但所有贡献都是相同的，因此分数可以正确简化。 

当 S 等于 E 时，会出现另一种边缘情况。这会强制使用单个目标值，因此正确性完全取决于精确的进位传播。 成对加法逻辑仍然适用，无需修改，因为相等比较在数字数组上是精确的。 

最后一种边缘情况是范围跨越 16 进制解释中的环绕边界。 由于数字是固定长度的 D 数字表示，因此数字数组的词典比较可以正确遵循模排序，而无需显式整数转换。
