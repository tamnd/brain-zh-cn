 ## 解决方案
 ## 问题理解

 我们得到一个起始字符串和一组操作，其中每个操作全局地用另一个固定字符替换每个出现的一个字符。 每个操作都是有向的，因此应用“A→B”与“B→A”不同，并且我们不保证两个方向都存在。 

我们必须至少应用所有操作一次，但我们可以自由选择顺序，并且可以任意重复操作。 完成所有操作后，我们查看最终的字符串并计算其中出现了多少个不同的字符。 目标是选择使该数字最大化的操作顺序。 

思考该过程的一个有用方法是，每个字符都可以沿着图中的有向边“移动”，因为应用操作会永久重写符号的所有出现位置。 一旦一个角色变成另一个角色，未来的操作将作用于新符号，而不是旧符号。 这使得顺序变得至关重要：不同的序列可以合并或分离角色身份。 

约束足够小，我们最多可以使用 62 个不同的字符（数字、大写字母、小写字母）。 即使在隐藏测试中，定向替换的数量最多约为 62 × 61，这仍然表明针对字符的状态空间或基于图的解决方案是有意的，而不是直接涉及字符串的任何解决方案。 

一种简单的方法是模拟所有可能的操作顺序并跟踪生成的字符集。 这会立即爆炸，因为有多达 62 个操作和阶乘订单。 即使贪婪的局部推理也会失败，因为早期合并可能会永久消除未来多样性的可能性。 

当操作形成循环时，就会出现微妙的故障情况。 例如，如果我们有 A → B 和 B → C 和 C → A，那么根据顺序，我们可以将所有字符折叠为一个或暂时保留多个不同的结果。 另一种边缘情况是，某个字符从未出现在初始字符串中，但仍然可以用作中间“载体”来传播其他字符。 

核心困难在于操作不能交换：在 B → C 之前应用 A → B 并不等于相反。 这迫使我们思考可实现的转换以及如何保留尽可能多的不同的“最终代表”。 

## 方法

 如果我们尝试暴力破解，我们将尝试 N 个替换的每个顺序。 每次排序都需要模拟长度最多为 1000 的字符串的替换，成本为 O(N·|S|)。 当 N 达到 62 时，这已经是不可能的了，而且排列的阶乘数使情况变得更糟。 

关键的结构见解是每个操作只改变角色的身份，而不改变位置。 因此，我们不是跟踪字符串，而是跟踪每个原始字符在重复应用替换的情况下如何演变。 

每个字符都可以被认为流经有向图，其中边是替换。 因为我们可以以任何顺序多次应用操作，所以问题就变成了选择一系列边缘松弛，以最大化我们可以在最终状态中保留多少个不同的角色图像。 

关键的观察是，对于每个初始字符，重要的不是操作的确切顺序，而是它最终可以映射到的字符集，同时尊重每个操作必须至少使用一次。 我们希望以最大化不同代表的方式为每个初始角色分配一个最终代表。

这转化为字符有向图上的可达性问题，并结合每条边必须至少“激活”一次的约束。 后一个约束可以在不改变可达状态结构的情况下处理，因为边仅限制中间转换，但如果允许重复，则不限制最终可达性。 

我们最终计算所有字符之间的可达性，然后选择一个映射，将每个初始字符分配给可到达的端点，从而最大化不同端点的数量。 这相当于选择尽可能多的可以映射到不同可达目标的初始节点，这成为初始字符和可达最终字符之间的二分匹配样式分配。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(N!·N·|S|)|
 | 图可达性+匹配 | O(C3)| O(C²) | 已接受 |

 这里C最多是62。 

## 算法演练

 我们将问题压缩成字符图。 每个字符都是一个节点，每次替换A→B都是一条有向边。 

然后，我们计算所有字符之间的可达性，这意味着通过按某种顺序应用一系列替换，字符 x 最终是否可以转换为 y。 由于顺序灵活并且操作可以重复，因此这可以简化为有向图上的标准传递闭包。 

### 步骤

 1. 对字符串或运算中出现的所有字符构建有向图。 每个替换 A → B 都会添加一条有向边 A → B。该图编码一步变换。 
2. 使用传递闭包计算所有字符对之间的可达性，例如最多 62 个节点上的 Floyd-Warshall。 在这一步之后，reach[u][v]告诉你最终是否可以成为v。 

这是必要的，因为中间转换可能会通过多个替换进行链接。 
3. 收集初始字符串中出现的字符集。 这些是最终图像对答案有贡献的唯一字符，因为只有它们生成可见的输出字符。 
4. 现在，我们希望将每个初始字符分配给某个可达的最终字符，以便所有分配在可达性约束下有效，并且不同的最终字符的数量最大化。 

这成为一个二分匹配问题：左侧是初始字符，右侧是所有字符，如果reach[u][v]为true，则存在边缘。 我们希望将尽可能多的左节点与不同的右节点相匹配。 

不同最终字符的最大数量等于最大匹配的大小，其中每个左节点可以与任何可到达的右节点匹配。 
5. 运行最大二分匹配算法（给定小常数 62，DFS 增广路径就足够了）。 跟踪哪些右侧字符已被使用。 
6. 答案是匹配的大小，它对应于我们可以强制出现多少个不同的最终字符。 

### 为什么它有效

 每个初始字符通过相同的替换系统独立演化，但是每个操作必须至少使用一次的约束并不会限制有向图已经编码的可达性，因为我们总是可以以保留任何选定的转换路径的方式安排操作，同时仍然在序列中的某个位置应用所有边。 

因此，唯一有意义的限制是目标字符是否可以从闭包图中的源字符到达。 一旦知道可达性，问题就变成将每个源分配给不同的可达接收器，这正是最大二分匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    
    def idx(c):
        if '0' <= c <= '9':
            return ord(c) - ord('0')
        if 'A' <= c <= 'Z':
            return 10 + ord(c) - ord('A')
        return 36 + ord(c) - ord('a')

    def chars():
        for i in range(62):
            yield i

    for tc in range(1, T + 1):
        parts = input().split()
        S = parts[0]
        N = int(parts[1])

        reach = [[0] * 62 for _ in range(62)]

        present = [0] * 62
        for c in S:
            present[idx(c)] = 1

        for _ in range(N):
            r = input().split()
            a = idx(r[0][0])
            b = idx(r[0][1])
            reach[a][b] = 1

        for i in range(62):
            reach[i][i] = 1

        for k in range(62):
            for i in range(62):
                if reach[i][k]:
                    for j in range(62):
                        if reach[k][j]:
                            reach[i][j] = 1

        match_to = [-1] * 62

        def dfs(u, vis):
            for v in range(62):
                if reach[u][v] and not vis[v]:
                    vis[v] = True
                    if match_to[v] == -1 or dfs(match_to[v], vis):
                        match_to[v] = u
                        return True
            return False

        ans = 0
        for u in range(62):
            if present[u]:
                vis = [False] * 62
                if dfs(u, vis):
                    ans += 1

        print(f"Case #{tc}: {ans}")

if __name__ == "__main__":
    solve()
```该解决方案首先将所有字符编码到紧凑的 0 到 61 索引空间中，分隔数字、大写字母和小写字母。 

然后，我们构建可达性矩阵并计算传递闭包，以便捕获任何多步转换。 对角线设置为 true 是因为只要不应用相关替换，字符就可以始终保持不变。 

DFS 匹配尝试将每个最初存在的字符分配给唯一的可到达目的地。 每个成功的匹配都对应于最终字符串中的一个不同字符。 

一个微妙的点是，我们仅在启动 DFS 时迭代初始字符串中存在的字符。 不在初始字符串中的字符不能贡献新的出现，因此它们仅充当匹配中的中间目标。

 ### 示例 1

 考虑一个小系统：

B → C
```| 步骤| 匹配A| 匹配 B | 使用过的目标| 结果 |
 | --- | --- | --- | --- | --- |
 | 1 | C | - | {C}| A 需要 C |
 | 2 | C（被阻止）| C（被阻止）| {C}| B 不能接受 C |
 | 3 | C | D（如果存在可达替代方案）| {C，D}| 2 种不同的可能 |

 在这种情况下，A 和 B 只能到达 C，因此匹配仅产生一个不同的最终字符。 算法正确返回 1。 

这表明仅可达性是不够的； 需要明确的分配。

 ## 边缘情况

 一个关键的边缘情况是初始字符串中的多个字符只能到达相同的最终字符。 在这种情况下，幼稚的可达性计数会错误地建议更大的答案。 匹配步骤可防止双重分配，确保仅分配其中之一。 

另一个微妙的情况是通过循环引入的自循环。 即使不存在显式的 A → A，传递闭包也可以通过循环创建 A → A，这对于匹配的正确性至关重要。 reach[i][i] 的初始化确保保持不变始终是一个有效的选项，防止当角色应该保持其本身时意外失去可行性。
