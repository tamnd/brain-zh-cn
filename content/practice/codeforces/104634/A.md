 ## 解决方案
 ## 问题理解

 我们得到一个有根结构，它实际上是一棵以节点 1 为根的有向树。每个节点都可以通过一条有向路径从根到达，因此尽管边可以在输入中以任何方向列出，但底层结构的行为就像一棵具有唯一根到节点路径的树。 

每个有向边代表一个滑雪坡度。 每个斜坡都有两个关键属性：容量，限制滑雪者总共可以穿越的数量，以及每个滑雪者的成本，可以是正数（我们支付）、零或负数（我们获得赏金）。 如果多个滑雪者使用相同的坡度，则成本将独立应用于每个滑雪者，因此边缘上的总成本与通过该边缘的滑雪者数量成线性比例。 

每个滑雪者从节点 1 开始，选择与 1 不同的某个目的地节点。由于图是具有唯一路径的树，因此选择目的地相当于选择根到节点的路径。 滑雪者也可能会提前停下来或在不使用边缘的情况下走得更远，但这无关紧要，因为每个节点都已经是唯一路径的自然端点。 

目标是双重的。 首先，我们必须最大限度地增加可以派出的滑雪者数量，确保边缘的使用量不会超过其容量。 其次，在将滑雪者分配到有效目的地以实现此最大值的所有方法中，我们必须最小化总成本，即所有边的总和（通过边的流量乘以其成本）。 

这是一个树流问题，其中每个节点代表一个可能的汇，每个流单元对应于一个选择根到节点路径的滑雪者。 

这些约束允许最多 100,000 个节点，这排除了任何二次方，例如重新计算每个节点的路径或流量。 每个测试用例我们需要一个线性或近线性方法，通常为 O(N) 或 O(N log N)。 内存足够大，可以容纳邻接表和 DP 表。 

一个关键的边缘情况是由负成本引起的。 如果所有成本均为负并且容量很大，则最佳策略是推动尽可能多的流量，因为每个单元都会降低总成本。 这意味着流量最大化与成本最小化之间的相互作用并非微不足道。 两者必须同时处理。 

另一个微妙的情况是某些边的容量可能为零。 即使存在下游节点，这些边也会有效地阻止该子树之外的流量。 

最后，不保证输入方向与根到叶方向对齐。 该结构是一棵有根树，但我们必须正确解释边缘。 

## 方法

 一个蛮力的想法是独立对待每个滑雪者。 我们可以一位一位地分配滑雪者，每次选择一个目的地节点并检查其路径上的所有边是否仍有剩余容量。 在最坏的情况下，沿着路径验证和更新每个分配的时间复杂度为 O(N)，因此发送 K 个滑雪者的成本为 O(KN)。 由于最优解中 K 可以是 O(N)，因此这变成了 O(N^2)，这对于 10^5 个节点来说太慢了。 

我们可以通过观察问题不在于单个滑雪者而是在于树上的聚合流来改进。 每个滑雪者沿着根到节点的路径贡献一个单位的流量，因此我们将来自根的流量分配到每条边都有容量的树中。 这是一个经典的树流问题，其中可行性由子树需求决定。 

关键的见解是自下而上地处理树，并决定必须通过每条边的流量才能以最佳方式为其所有后代提供服务。 每个节点都可以代表一个“接收器选项”，其中流可以终止。 如果我们认为每个节点都有可能吸收一个单位的流量，那么最大化滑雪者的数量就相当于最大化我们激活为汇的节点数量，并受到边缘容量的限制。

一旦确定了最大流量，最小化成本就成为相同固定流量值上的重新分配问题。 由于每条边每单位流量的成本是线性的，因此我们希望优先通过更便宜的边推动流量，同时尊重进入每个子树的总流量由可行性约束固定。 

同时解决这两个部分的一种简洁方法是在树上进行动态规划。 对于每个节点，我们计算在考虑容量的情况下可以在其子树内分配的滑雪者的最大数量，并保持实现该流程的最小成本。 这成为对子级的类似背包的合并，但由于该结构是一棵树，并且每条边仅连接父级和子级，因此合并简化为将通过父级边的独立子树流与容量上限组合起来。 

该结构确保每个子树向上贡献单个“流量需求”，并且父边缘要么饱和要么传递所有流量。 这导致从叶子到根的贪婪积累。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个滑雪者的蛮力 | O(N^2) | O(N^2) | O(N) | 太慢了|
 | 树DP流聚合| O(N) | O(N) | 已接受 |

 ## 算法演练

 我们以节点 1 为树根，并将所有边解释为沿着连通性隐含的唯一路径结构远离根。 

我们执行后序遍历并计算每个节点可以在其子树中分配多少个滑雪者。 每个节点尝试聚合来自其子节点的贡献，并将流量通过边缘向上传递到其父节点。 

1. 为树构建邻接表并存储每个有向边的容量和成本。 
2. 以节点 1 为树的根并使用 DFS 计算父子结构。 
3. 为每个节点定义一个值`dp[u]`表示在以 为根的子树中可以完全分配的滑雪者的最大数量`u`不超过任何边缘容量。 
4. 遍历子元素`u`。 每个孩子`v`产生流量值`dp[v]`，这意味着许多滑雪者源自该子树并且必须经过边 (u, v)。 
5. 对于有容量的边(u,v)`S`，贡献是`min(dp[v], S)`。 任何超出容量的多余流量都会被丢弃，因为它无法向上输送。 
6. 应用容量上限后，累计所有子项的总流量。 这个金额是`dp[u]`。 
7、计算dp的同时，保持成本累加。 对于每个子边，如果`x = min(dp[v], S)`，然后我们添加`x * C`成本，因为通过该边缘的每个单元都会付出成本 C.
 8. 滑雪人数最多的答案是`dp[1]`，可以到达根的子边缘的总流量。 
9. 总的最小成本是DP计算期间的累计成本。 

关键的一点是，除了其连接边的容量限制之外，每个子树都是独立的，因此局部决策完全决定全局可行性。 

### 为什么它有效

 每个流量单位恰好对应于一名滑雪者和一条从根部到水槽的路径。 由于图是一棵树，因此这些路径仅在计数容量方面是边不相交的，而不是在结构方面。 DP 通过明确限制每个边的贡献来确保没有边承载的流量超过其容量。 由于每个子树都是独立处理的，并且仅通过单个父边进行交互，因此贪婪钳位是最佳的：因为没有替代路径，所以没有可以降低成本或增加流量的替代路由。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        n = int(input())
        g = [[] for _ in range(n + 1)]
        
        for _ in range(n - 1):
            u, v, s, c = map(int, input().split())
            g[u].append((v, s, c))
            g[v].append((u, s, c))
        
        visited = [False] * (n + 1)
        dp = [0] * (n + 1)
        cost = 0

        def dfs(u):
            nonlocal cost
            visited[u] = True
            total = 0
            for v, s, c in g[u]:
                if visited[v]:
                    continue
                child_flow = dfs(v)
                used = child_flow if child_flow < s else s
                total += used
                cost += used * c
            dp[u] = total
            return dp[u]

        ans_flow = dfs(1)
        print(f"Case #{tc}: {ans_flow} {cost}")

solve()
```该实现遵循直接后序 DFS。 每个节点聚合来自子节点的流量，并且每个边缘通过钳位强制其容量。 全局成本在遍历过程中累积，因为当子节点返回其流时，每个边贡献恰好确定一次。 

一个微妙的点是，我们依赖于标记访问过的节点而不是传递父指针，这避免了显式反转边。 另一个重要的细节是递归深度，由于 N 高达 100,000，因此必须增加递归深度。 

## 工作示例

 考虑第一个示例，其中节点 1 连接成一棵具有多个分支的小树。 DFS 首先处理叶子，因此如果每个叶子可以向上发送一个滑雪者，则它返回 dp 等于 1。 

| 节点| 儿童流动 | 应用边缘盖 | dp[节点] |
 | --- | --- | --- | --- |
 | 叶节点| 0 或 1 | 无 | 1 |
 | 内部节点| 子项总和 | 最小容量 | 聚合|

 从根本上来说，所有可行的流量在容量过滤后进行求和，产生最终的滑雪者数量。 成本在每个边缘上累积为流量乘以边缘成本，确保昂贵的边缘承载最小的必要流量。 

在第二个示例中，某些边具有负成本。 DFS 仍然按容量限制流量，但由于成本是按单位流量累积的，因此负成本边缘可有效降低饱和时的总成本。 这证实了流量最大化可以自动允许在可能的情况下充分利用负成本优势。

 ## 边缘情况

 一种重要的边缘情况是当一个节点有多个子节点但一条连接边的容量为零时。 在这种情况下，DFS 返回该子子树的 dp，但是`min(dp, 0)`变为零，因此没有流量向上传递，从而正确隔离了子树。 

另一种情况是当所有边都有负成本和大容量时。 该算法仍然会向上传递所有可能的流量，因为 dp 聚合了最大可行流量，并且成本累积正确地减去了奖励。 

最后一种情况是星形树，其根连接到许多叶子。 每个叶子独立贡献，并且 dp[1] 成为传出边缘上所有容量的总和，符合每个边缘独立限制流量的直觉。
