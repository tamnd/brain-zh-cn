 ## 解决方案
 ## 问题理解

 我们正在使用固定的字符串，`ECUST`，任务是生成每个可能的字符串，这些字符串可以通过独立选择每个字符的大小写而形成，同时保持底层字母不变。 每个字符都保持在其位置，但可以是大写或小写，所有此类组合都被视为有效输出。 

输出不是任意的。 每个生成的字符串都必须使用标准 ASCII 字典顺序进行排序。 此排序逐个字符地比较字符串，在它们不同的第一个位置，具有较小 ASCII 值的字符串排在前面。 由于大写字母的 ASCII 码比小写字母小，因此在同一位置进行比较时，同一字母的大写版本始终位于小写版本之前。 

尽管输入格式在技术上提供了一行，但它不包含影响答案的信息。 整个问题是确定性的，仅取决于字符串`ECUST`。 

隐式约束是字符串长度固定为5，因此生成的字符串总数也固定为$2^5 = 32$。 这立即排除了对性能或内存使用的任何担忧。 即使是简单的枚举也非常快。 

主要的微妙之处在于排序。 简单的方法可能会生成所有大小写组合，然后使用通用字符串排序对它们进行排序。 这是安全的，但它隐藏了一个结构属性，允许以正确的顺序直接生成。 

一个典型的错误是假设大小写组合的字典顺序就像二进制数一样，而不检查 ASCII 规则。 例如，将小写字母视为“较小”会颠倒正确的顺序并产生不正确的输出。 

另一个微妙的问题是忘记了每个角色都是独立变化的。 如果有人错误地全局切换大小写或交换字符，他们将不会生成有效的保留大小写的变体。 

## 方法

 暴力方法是将位置的所有子集生成为小写。 对于这 5 个字符中的每一个，我们决定是保持大写还是变为小写，从而生成 32 个字符串。 生成它们后，我们使用正常的字符串比较对列表进行排序。 

这是有效的，因为搜索空间非常小。 成本本质上是生成 32 个字符串并对它们进行排序，这是可以忽略不计的。 然而，它会执行不必​​要的工作，因为如果我们利用 ASCII 排序与大小写更改交互的方式，实际上并不需要排序。 

关键的观察是每个位置在词典比较中表现独立。 对于每个字符，同一字母的大写字母始终小于小写字母。 这意味着，如果我们将大写字母解释为 0，将小写字母解释为 1，则整个字符串的字典顺序与 5 位二进制掩码的自然递增顺序相匹配。 

所以我们可以不用生成和排序，而是枚举0到31的掩码，直接构造对应的字符串。 这会自动产生正确的顺序。

 ## 算法演练

 我们对待每个角色`ECUST`具有两种状态（大写和小写），并将选择解释为二元决策。 

1. 考虑从 0 到 31 的所有整数，其中每个数字代表大小写选择的唯一组合。 每一位对应于字符串中的一个位置。 这是可行的，因为 5 个字符正好给出 5 个二元决策。 
2. 对于每个数字，从最高有效位（位置 0）到最低有效位（位置 4）检查其二进制表示形式。 位值 0 表示我们保留大写字符，位值 1 表示我们将其转换为小写字符。 
3. 逐个字符构建结果字符串。 在每个位置，应用步骤 2 中的规则来决定是降低还是保留角色。 
4. 立即打印每个构造的字符串。 由于掩码是按升序处理的，因此输出已经按照正确的字典顺序排列。 

关键的细节是，从 0 向上迭代可以保证字典顺序的正确性，因为较早的位对应于较早的字符，并且大写字母在字典顺序上总是小于小写字母。 

### 为什么它有效

 该算法依赖于两个系统之间一致的排序对齐：掩码的二进制排序和字符串的 ASCII 字典排序。 在两个生成的字符串之间的第一个不同位置，该位置恰好对应于它们掩码中的最高有效不同位。 由于我们将 0 映射到大写字母，将 1 映射到小写字母，并且大写字母具有较小的 ASCII 值，因此掩码的顺序直接与字典顺序匹配。 这个不变量确保不需要后排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = "ECUST"
n = len(s)

for mask in range(1 << n):
    res = []
    for i in range(n):
        c = s[i]
        if mask & (1 << (n - 1 - i)):
            res.append(c.lower())
        else:
            res.append(c)
    print("".join(res))
```该程序首先修复基本字符串并迭代代表大小写选择的所有可能的位掩码。 位测试是对齐的，以便最高有效位对应于第一个字符，确保字典顺序的正确性。 

字符串的构造是在每个掩码中一次完成的，这使得实现简单并避免了对排序逻辑或辅助数据结构的任何需要。

应用同样的推理`ECUST`只需将此模式扩展到 5 个位置，产生 32 个有序输出。

 ## 边缘情况

 唯一有意义的边缘行为是边界处的排序：完全大写和完全小写的字符串。 

用于输入`ECUST`，面具`00000`产生`ECUST`，这是最小的字符串，因为每个字符都是大写的。 面具`11111`产生`ecust`，这是最大的，因为每个字符都是小写的。 

遍历第一种情况，每个位都为零，因此没有字符被转换，并且输出仍然存在`ECUST`。 由于没有其他字符串在第一个不同位置可以有更小的字符，因此它必须按字典顺序首先出现。 

对于最后一种情况，所有位都是 1，因此每个字符都是小写的。 任何其他字符串都较早或在大写字母出现的某个位置不同，并且大写字母较小，因此所有此类字符串都在它之前，使得`ecust`最终输出。
