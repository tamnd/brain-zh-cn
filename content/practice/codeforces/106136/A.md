 ## 解决方案
 ## 问题理解

 我们有一个小型 ICPC 团队以及该团队在一场比赛中赢得多少奖牌的记录。 每块金、银、铜牌均对应固定奖金池：金牌每队价值7500元，银牌每队3000元，铜牌每队1500元。 团队最多可以有三名成员，团队获得的总奖金将平均分配给所有成员。 

任务是根据团队规模和每种奖牌类型的数量，计算一名特定成员收到的金额。 

输入包含四个整数：团队规模，后跟金牌、银牌和铜牌的数量。 输出是一个整数，代表一名会员在总奖金中所占的份额。 

限制非常小。 团队规模最多为 3 人，奖牌总数最多为 7 块。这立即排除了对优化技术甚至仔细的数据结构的任何需要。 每个运算都是常数时间算术。 

因为所有值都是整数，并且除法是通过构造精确的（在预期的解释中，总奖金总是可以被团队规模整除），所以主要的陷阱只是正确地实现算术，而不会损失精度或混淆除法顺序。 

一些边缘情况仍然值得关注。 

一种极端情况是根本没有奖牌。 例如，输入`2 0 0 0`应该产生`0`，因为总奖金为零。 

另一个是尽可能最小的团队规模。 例如，`1 1 0 0`应该返回 7500 的全额金奖，因为没有任何分割。 

第三种情况是，如果由于过早的整数除法而计算不正确，则奖金无法平均分配。 例如，如果在求和之前错误地将奖牌奖金分别分配给每个人，则可能会出现浮点或截断错误。 正确的做法是先计算总奖金，然后再划分一次。 

## 方法

 这里几乎不需要暴力解释，但我们仍然可以将其描述为模拟每个奖牌事件并累积每个团队成员的奖金。 对于每块金牌，我们会将 7500 添加到运行总数中，每块银牌将添加 3000，每块铜牌将添加 1500，然后最后除以团队规模。 

即使我们错误地认为将每个奖牌奖励立即分配给成员，最坏情况下的操作计数仍然受到奖牌数量的限制，最多为 7 个。因此，即使对每个奖牌事件进行字面模拟也是微不足道的。 

关键的观察结果是该问题是纯粹的加性和线性问题。 奖牌的处理顺序并不重要，并且奖牌之间没有相互作用。 这将一切简化为计算加权和并在最后除一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟 | O(g + s + b) | O(1) | O(1) | 已接受 |
 | 直接配方 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取代表队伍规模和金、银、铜牌数量的四个整数。 这一步就建立了计算所需的所有输入。 
2. 将团队获得的总奖金计算为加权总和：每枚金牌贡献 7500，每枚银牌贡献 3000，每枚铜牌贡献 1500。此步骤将整个比赛历史记录压缩为单个标量值，该标量值是有效的，因为每块奖牌都是独立贡献的。 
3. 将总奖金除以团队规模。 由于团队平均分配奖金，这就产生了每个成员收到的金额。 
4. 将结果输出为整数。 

### 为什么它有效

 奖励系统与奖牌数呈线性关系。 每枚奖牌为总奖励贡献固定的独立金额，并且对最终总金额进行一次分割操作。 由于是先求和再除法，所以分配性保证了结果相当于将每枚奖牌奖励平均分配给成员然后再求和。 这保证了任何排序或分组效应都不会改变最终答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, g, s, b = map(int, input().split())

total = g * 7500 + s * 3000 + b * 1500
print(total // n)
```该解决方案读取一行输入并立即解压四个整数。 总奖金是通过将计数与其各自的值直接相乘来计算的，这避免了任何循环。 

除法是在最后使用整数除法完成的。 这是安全的，因为所有奖金值都是 500 的倍数，并且除法旨在根据问题的定义进行精确划分。 使用`//`确保我们继续进行整数运算，避免浮点问题。 

一个常见的错误是在求和之前将每块奖牌的贡献分开。 这在数学上仍然有效，但它在更一般的问题中引入了不必要的舍入误差风险，并且模糊了预期的结构。

这个案例证实了该公式可以自然地处理单成员团队，而不需要任何特殊的逻辑。

 ## 边缘情况

 对于输入`3 0 0 0`，该算法将总奖金计算为零并除以三，结果为零。 这证实了当分子为零时除法步骤不会引入伪影。 

为了`1 0 0 4`，除法前计算结果为 6000。 由于团队规模为 1，除法使该值保持不变，从而确认最小团队规模的正确性。 

为了`3 7 0 0`，总奖金为`7 × 7500 = 52500`，除法产量`17500`。 这表明该算法可以处理最大奖牌计数，而无需担心 Python 中的溢出问题，并在最后正确应用整数除法。
