 ## 解决方案
 ## 问题理解

 我们给出一个由起点定义的小整数范围`l`。 隐藏号码`a`保证位于区间内的某个位置`l`最多`l + 5`，因此最多有六个连续整数是可能的候选者。 

除了这个范围限制之外，`a`必须同时满足两个模条件。 除以 2 后的余数固定为`m2`，除以 3 后的余数固定为`m3`。 任务是确定允许范围内与两个余数条件匹配的任何整数，或者报告不存在这样的整数。 

限制非常小：`l`最多为 100，并且搜索间隔长度固定为 6。这立即排除了对高级数论或优化的任何需要。 即使直接扫描间隔也最多执行六次检查，这是常数时间。 

唯一的微妙之处在于自然数的定义。 由于明确允许 0，因此区间可以包含 0 或小值，其中模行为仍然定义良好，因此不存在隐藏的排除。 

此类问题中的一个典型错误是尝试使用中国剩余定理机制直接求解同余式。 这是不必要的，而且更容易出错，因为搜索空间太小，暴力已经是最佳选择。 

## 方法

 最直接的想法是尝试每个整数`a`在范围内`[l, l + 5]`并检查是否满足两个条件`a % 2 == m2`和`a % 3 == m3`。 由于最多有六个候选者，每个候选者都需要恒定的时间检查，因此速度非常快。 

这是有效的，因为约束无法扩展。 但是，如果间隔很大，例如达到 10^9，则这种直接枚举是不可能的。 在这种情况下，我们需要使用模 6 的余数周期来交叉模约束，因为模 2 和模 3 的条件组合成模 6 的结构。关键的观察结果是每个整数都由其模 6 的余数唯一确定，并且两个约束一起指定模 6 的单个余数类。 

在这个问题中，不需要更深的结构，但它解释了为什么强力已经本质上是最优的：搜索空间小于一个完整的模数周期。

 ## 算法演练

 1. 阅读`l`,`m2`， 和`m3`来自输入。 这些定义了候选范围和所需的余数。 
2. 迭代所有整数`a`从`l`到`l + 5`包容性。 这涵盖了所有可能的有效候选人。 
3. 对于每位候选人`a`, 计算`a % 2`并检查它是否等于`m2`。 如果不匹配，则丢弃`a`立即地。 
4. 如果第一个条件成立，计算`a % 3`并检查它是否等于`m3`。 如果也匹配，我们就找到了有效的解决方案并可以输出`a`。 
5. 如果循环结束时没有找到任何有效的候选者，则输出`-1`。 

检查的顺序对于正确性并不重要，但首先检查模 2 的效率稍高一些，因为它会立即过滤掉一半的候选者。 

### 为什么它有效

 每个可能的值`a`位于最多六个连续整数的集合中。 该算法显式地评估整个集合的每个元素。 由于没有跳过任何候选，并且每个候选都根据精确定义的约束进行测试，因此在迭代过程中必须遇到任何有效的解决方案。 相反，只返回有效的候选项，因为在接受值之前会检查两个模块化条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

l, m2, m3 = map(int, input().split())

for a in range(l, l + 6):
    if a % 2 == m2 and a % 3 == m3:
        print(a)
        sys.exit()

print(-1)
```实现直接遵循算法，没有任何隐藏的优化。 恰好六个值上的循环保证了有限的运行时间。 提前退出通过`sys.exit()`确保我们在找到有效候选人后立即停止，尽管即使没有它，成本也可以忽略不计。 

一个常见的实施陷阱是将范围错误地设置为`range(l, l+5)`而不是`l+6`。 由于区间两端都包含在内，因此我们必须恰好包含六个数字。 

另一个微妙之处是处理不存在有效数字的情况。 我们只打印`-1`在用尽所有候选人之后。

 第一个有效值是 5，因此我们输出 5。这表明即使多次检查提前失败，完整的范围对于找到正确的匹配是必要的。

 输入：`63 1 1`我们测试 63 到 68 之间的值。 

| 一个 | % 2 | % 3 | 有效 |

 | 63 | 63 1 | 0 | 没有|

 | 67 | 67 1 | 1 | 是的 |

 此处，67 满足两个模数约束，因此返回。 此示例表明，该解决方案不会对答案位于区间内的位置进行任何假设。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    l, m2, m3 = map(int, input().split())

assert run("10 0 1") == run("10 0 1")

 ## 边缘情况

 当出现一种边缘情况时`l`是 0。该算法仍然正确地包括 0 作为有效候选，因为循环从`l`本身。 例如，使用输入`0 0 0`，第一个检查`a = 0`立即满足这两个条件，因为`0 % 2 = 0`和`0 % 3 = 0`，算法返回 0，无需进一步迭代。 

另一种边缘情况是有效解位于上边界时`l + 5`。 例如，如果`l = 2`, 候选人`7`可能是区间内唯一有效的数字。 该循环明确包括`l + 5`， 所以`a = 7`经过测试并正确接受。 

最后的边缘情况是区间中没有数字满足两个模约束。 在这种情况下，每次迭代都至少失败一个条件，并且循环完全完成。 然后算法输出`-1`，正确地发出不可能的信号。
