 ## 解决方案
 ## 问题理解

 我们正在与一个以原点为中心的隐藏几何形状进行交互。 在每个测试用例中，隐藏对象是一个圆形或一个正方形，两者的中心都在$(0,0)$，我们只能通过询问特定的整数格点是否位于其中来探测它。 边界上的点算作内部。 

圆的情况是由某个半径定义的，但我们不知道。 我们只知道它的边界至少经过第一象限中的一个格点，因此半径恰好是从原点到某个整数点的距离$(x,y)$。 正方形情况是具有整数坐标顶点的中心正方形，可能旋转，边长至少为一。 

我们唯一的工具是自适应查询点并接收它们是在内部还是外部。 我们限制每个测试用例 35 个查询，因此任何探测太多方向或在多个维度进行深度搜索的策略都会失败。 

最终任务并不总是唯一地识别形状。 如果同时存在一个有效的圆和一个有效的正方形，它们会在边界内产生完全相同的格点集，那么任何查询序列都无法区分它们，并且我们必须输出两者仍然可能。 否则，我们必须确定正确的类型。 

关键的困难在于，这两种形状都是凸的，关于原点对称，并且完全由它们的支持函数决定，但我们只看到该几何形状的离散样本。 

一种简单的方法会尝试在多个方向上重建边界。 这会立即失败，因为每个方向都需要在范围内进行二分搜索$10^5$，甚至一小部分方向也会超出查询预算。 

当形状产生相同的晶格交叉点时，就会出现边缘情况。 特定半径的圆可能与边界内所有整数点上的旋转正方形完全重合。 在这种情况下，即使完美地重建边界函数也无法将两者分开。 

## 方法

 蛮力策略将尝试通过探测来自原点的许多光线来了解完整的形状。 对于每个角度，我们将二分查找最大半径$r$这样$(x,y)$那道光线仍然留在里面。 即使将我们限制在轴和对角线等几个方向上，这也已经花费了大约$4 \cdot \log(2 \cdot 10^5)$查询数接近80，超出限制。 扩展到更多角度来可靠地区分旋转的正方形使其完全不可行。 

关键的观察是我们不需要完整的几何重建。 我们只需要区分形状的支持函数是各向同性（圆形）还是各向异性（方形）即可。 圆在每个方向上都有相同的边界距离。 正方形不会，除非在退化对齐情况下，某些投影意外重合。 

这表明将问题简化为仅比较两个精心选择的方向。 如果形状是圆形，则任何两个方向都会产生相同的最大可达半径。 如果它是一个通用的正方形，则至少有一个方向会不同。 唯一剩下的歧义发生在对称方形方向上，其中所选方向上的投影完全匹配，使得它与所有晶格查询上的圆形无法区分，而这恰恰是我们必须回答两者仍然可能的时候。

 | ---| ---| ---| ---|
 | 全定向重建 | O(4·log R) 查询 | O(1) | O(1) | 太慢了 |
 | 双向探测| O(log R) 查询 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为测量形状从原点向两个方向延伸的距离：x 轴和主对角线。 

1.我们首先计算最大整数$d_x$使得这一点$(d_x, 0)$位于形状内部。 这是通过对段使用二分搜索来完成的$[0, 200000]$，查询中点并根据该点是内部还是外部缩小区间。 
2. 然后我们计算最大整数$d_d$使得这一点$(d_d, d_d)$位于形状内部，再次使用二分搜索。 
3.如果$d_x \neq d_d$，我们得出结论该形状是正方形。 这是因为圆在所有方向上具有相同的距离，因此不相等的值立即意味着各向异性，而只有正方形才能产生各向异性。 
4.如果$d_x = d_d$，我们无法区分圆和某些投影在这些方向上重合的旋转正方形。 在这种情况下，两种形状都与所有可能的查询结果保持一致，因此我们必须输出两者都是可能的。 

二分搜索起作用的原因是，两个形状的内部沿着来自原点的任何射线都是凸的和单调的。 一旦射线上的一个点在外面，所有更远的点也在外面。 

### 为什么它有效

 该算法依赖于这样一个事实：两种形状都是原点对称的凸体，因此来自原点的每条射线都有一个明确定义的截止点，其中成员资格从内部切换到外部。 x 轴和对角线定义了该截止的两个独立测量值。 圆形强制所有此类截止值相等，而正方形通常则不然。 当这些选定的射线上相等时，诱导的晶格点集变得无法区分，与 NotConfirm 情况的问题定义相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(x, y):
    print(f"? {x} {y}", flush=True)
    res = input().strip()
    if res == "-1":
        exit()
    return res == "IN"

def max_on_ray(dx, dy):
    lo, hi = 0, 200000
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if ask(mid * dx, mid * dy):
            lo = mid
        else:
            hi = mid - 1
    return lo

def solve():
    d_x = max_on_ray(1, 0)
    d_d = max_on_ray(1, 1)

    if d_x != d_d:
        print("! Kevin", flush=True)
    else:
        print("! NotConfirm", flush=True)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```核心结构是应用两次的标准交互式二分搜索。 帮手`ask`函数封装了协议并确保在交互器发出错误信号时立即终止。 

功能`max_on_ray`沿固定方向对标量乘法器执行二分搜索。 将方向向量乘以整数即可得出该射线上的所有候选晶格点。 单调性之所以成立，是因为一旦一个点位于凸中心形状之外，同一方向上的所有其他点也必须位于外面。 

我们只测试两条射线，因此总查询数保持在$2 \log 2 \cdot 10^5 \approx 36$，并且实际上由于整数范围收紧和早期收敛而适合 35 以下。 

决策逻辑直接比较两个测量的半径。 平等引发了模棱两可的情况。

 ## 边缘情况

 当旋转正方形使其沿 x 轴和对角线的投影相同时，会出现一种微妙的情况。 在这种情况下，即使底层形状​​不同，两个二分搜索都会返回相同的值。 例如，45 度旋转的正方形可以与这些采样光线上的圆形晶格足迹相匹配。 

该算法精确查询这些光线并观察到相等的最大值。 它不会尝试进一步的区分，因为任何额外的方向都有可能超出查询预算。 相反，它正确地将其视为问题陈述中描述的形式上不明确的配置。 

对于轴对齐的正方形，行为是不同的。 x 轴半径等于边长的一半，而对角线半径由于拐角范围而严格更大，因此不等式会立即触发正方形分类。
