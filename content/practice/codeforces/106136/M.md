 ## 解决方案
 ## 问题理解

 我们被要求构造一个长度严格递增的序列$n$，其中每个元素都是下面的非负整数$2^{30}$。 困难不在于单调性，而在于对每个相邻对的约束：对于每个连续的对$(a_i, a_{i+1})$，该对必须满足按位条件，其中两个数字的 XOR 严格小于它们的 AND。 

输入由多个测试用例组成，每个测试用例给出所需的长度$n$。 对于每种情况，我们要么输出一个有效序列，要么报告无法构建。 

约束条件总体来说很大，总共$n$跨所有测试用例直至$10^6$。 这立即表明任何解决方案都必须在每个测试用例的线性时间内构建每个序列，并且理想情况下以非常简单的确定性模式构建，无需搜索或回溯。 每个测试用例的任何二次方都是不可能的，甚至$O(n \log n)$如果常数很大，就会处于临界状态。 

该条件的一个微妙方面是它同时涉及 XOR 和 AND。 XOR 测量位不一致，而 AND 测量共享位。 不等式 XOR < AND 意味着相邻数字必须相对于它们的差异共享足够多的高权重位，这已经很可疑，因为对于大多数有序对而言，除非较高位存在强烈重叠，否则异或往往占主导地位。 

边缘情况立即出现在小$n$。 为了$n = 2$，我们需要一个有效的对，并且是否存在这样的对并不明显。 一个天真的假设可能是选择连续整数是可行的，但是$a$和$a+1$XOR 总是等于$2^{k+1}-1$在尾随位中，而 AND 为零，使得该条件不可能。 这表明并非所有递增序列都能满足约束，并且必须仔细设计结构，否则对于某些人来说答案可能是不可能的$n$。 

另一个潜在的陷阱是假设条件是对称的或随着值的增加而单调地表现。 两者都不是，因为 XOR 和 AND 完全取决于二进制重叠而不是大小。 

## 方法

 暴力方法会尝试逐步构建序列，并在每个位置尝试所有大于前一个位置的可能值，检查按位条件。 在最坏的情况下，这需要扫描最多$2^{30}$每个职位都有候选人，这使得它完全不可行。 即使限制在合理的子集内，总工作量仍将远远超出给定的限制$n$最多$10^6$。 

关键的见解来自于根据位结构重写条件。 对于两个数字$a < b$，我们要求$$a \oplus b < a \& b.$$仅当“与”贡献了一个“异或”无法抵消的高位时，这种不等式才能成立。 然而，XOR 总是只在两个数字不同的地方有位，而 AND 只在两个数字都为 1 的地方有位。这立即意味着结构张力：XOR 突出差异，AND 突出重叠，我们需要重叠来主导数值的差异。 

关键的观察结果是，如果两个数字共享一个高位，则该位对“与”有正向贡献，但对“异或”没有贡献。 同时，XOR完全由不同的位决定。 为了使 AND 大于 XOR，我们希望相邻数字仅在低位不同，同时共享至少一个高位，并且该共享位必须主导所有不同位的数值。 

这建议构建一个链，其中我们以受控方式固定最高位并仅改变较低位，确保 AND 始终包含该固定位，而 XOR 仍仅限于较低位。 然而，严格的单调性使朴素的构造变得复杂。 

一个干净的前进方法是认识到我们只需要严格递增的序列，而不是完整的组合结构。 这使我们能够在所有元素上强制使用单个显性高位，并以保持 XOR 小的方式仅改变一组受控的低位。 

最简单的工作结构是选择以下形式的数字：$$a_i = 2^{k} + i$$对于足够小的$i$，但这会失败，因为连续数字之间的 AND 变得太小。 相反，我们翻转视角：我们构建共享固定高位的对，并确保转换发生在精心选择的块内，以便每个相邻对都有一个大的共享位集，并且只有很小的差异。 

最终的建设性想法是以递增顺序构建序列，同时确保每个数字都有固定的最高位，并且在该约束内，我们确保邻接的行为就像子集中的受控增量，其中 AND 始终保留高于任何 XOR 贡献的主导位。 这导致了一种模式，我们本质上在高位掩码内编码索引，以保证 AND 优势。 

简化约束后，核心结论是所有序列都存在有效序列$n \ge 2$，并且可以使用固定位模式线性构造，该固定位模式保证整个序列共享最高位，同时保持异或受低位增量的限制。

 ## 算法演练

 该构造依赖于固定单个高位并在剩余位中编码单调递增序列，同时确保邻接始终保留该高位。 

1. 选择一个固定的位位置，例如位 29，并确保序列中的每个数字都设置了该位。 这保证了每个相邻对都有一个非零 AND 至少包含$2^{29}$，如果差异仅限于较低位，则它将主导与 XOR 的比较。 
2.通过枚举整数构造序列$0, 1, 2, \dots, n-1$并将每个值设置为$a_i = (1 << 29) | i$。 这保留了严格的单调性，因为较低位通常会增加。 
3. 隐式验证邻接结构：对于连续值，XOR 仅影响低位，因为高位被抵消，而 AND 始终包含高位。 这确保 XOR 严格受以下限制$2^{29}$，而 AND 至少是$2^{29}$，使得不等式对于所有对都成立。 
4. 直接输出序列。 

该构造是确定性的并且独立于测试用例结构。 

### 为什么它有效

 每个元素都包含一个固定的最高设置位，因此对于任何相邻对，AND 运算都会保留该位，从而保证较大的基线值。 另一方面，XOR 仅反映低 29 位的差异，这些差异始终严格小于共享高位的贡献。 这在每个相邻对的 XOR 和 AND 之间创建了严格的大小分离，确保不等式始终成立，同时通过低位计数器保持严格的排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    HIGH = 1 << 29
    
    for _ in range(T):
        n = int(input())
        if n < 2:
            print(-1)
            continue
        
        # construct sequence
        res = []
        for i in range(n):
            res.append(str(HIGH | i))
        
        print(" ".join(res))

if __name__ == "__main__":
    solve()
```该解决方案首先读取测试用例的数量，然后独立处理每个测试用例。 恒定高位$2^{29}$用于确保每个元素共享一个共同的显性位，从而稳定所有相邻对之间的 AND 运算。 

每个值都是通过将此高位与一个简单的递增计数器进行“或”运算而形成的。 这保证了严格的单调性，因为低 29 位正常增加并且不会干扰排序。 

一个常见的错误是忘记输出必须保持在$[0, 2^{30})$。 所选择的结构遵循这个界限，因为它只使用位 29 加上一个值高达$10^6$，完全在范围之内。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    T = int(input())
    HIGH = 1 << 29
    out = []

    for _ in range(T):
        n = int(input())
        if n < 2:
            out.append("-1")
            continue
        arr = [(HIGH | i) for i in range(n)]
        out.append(" ".join(map(str, arr)))

    return "\n".join(out)

# minimal size
assert run("1\n2\n") != "-1"

# multiple test cases
assert run("2\n2\n3\n") != ""

# larger sequence sanity
assert len(run("1\n5\n").split()) == 5

# all equal n=2 edge behavior
assert run("1\n2\n").count("-1") == 0

 ## 边缘情况

 对于$n = 2$，序列变为$[2^{29}, 2^{29}+1]$。 与是$2^{29}$因为两个数字共享最高位，而 XOR 为 1。不等式完全成立，并且保留了严格的排序。 

为了$n = 10^6$，该结构仍然有效，因为只有较低位发生变化。 即使计数器变大，它仍然低于$2^{20}$，远低于固定高位，因此不会发生溢出或位干扰。
