 ## 解决方案
 ## 问题理解

 我们给定一个正整数$n$，我们处理来自的数字$1$到$n$作为物品的标签。 任务是形成尽可能多的不相交对，其中一对$(a, b)$仅当以下情况下才有效$a \neq b$和最大公约数$a$和$b$大于 1。每个数字最多只能使用一次，我们必须输出此类对的最大数量以及实现该最大值的一个有效构造。 

更具体地说，我们试图匹配整数，以便每个匹配对共享至少一个素因数。 像这样的数字$1$不可用，因为它没有质因数，而合数通常可以根据共享除数以多种方式配对。 

约束条件$n \le 10^5$表明我们需要大致线性或接近线性的行为。 任何明确检查所有对的解决方案都需要$O(n^2)$的操作，这是远远超出可接受范围的。 即使像检查每对的整除性之类的事情也太慢了。 这立即排除了暴力配对的可能性。 

一个微妙的边缘情况是$n$非常小。 为了$n = 1$，没有对。 为了$n = 2$，也不存在有效的对。 为了$n = 3$，同样不存在对，因为$1,2,3$没有对共享大于 1 的公约数。另一个重要的情况是当数字是素数或接近素数时，因为它们的兼容性非常有限。 例如，任何尝试配对连续数字的贪婪都会失败$n = 5$只给出一对有效的$(4,2)$，而朴素的邻接配对会错过该结构。 

## 方法

 暴力策略会尝试每个未使用的数字，并尝试将其与 gcd 大于 1 的另一个未使用的数字配对。这在逻辑上有效，因为它直接强制执行条件，但每个选择都需要扫描最多$O(n)$候选人，导致$O(n^2)$最坏情况下的行为。 和$n = 10^5$，这变得完全不可行。 

关键的观察是在消除所有 2 的因数后，按奇偶性对数字进行分组。每个偶数都至少有一个 2 的因数，并且其中许多偶数可以使用此共享除数进行配对。 类似地，3的倍数可以通过因子3配对，依此类推。 然而，明确地跟踪所有素数是没有必要的。 

更结构化的观点是将数字配对到块中，我们从最大的数字开始贪婪地利用共享除数。 如果我们向下迭代，每次遇到一个数字$i$，我们尝试将它与它的一个小质因数的最大的未配对倍数配对。 这个问题的标准构造进一步简化：我们处理来自$n$下降到$2$，并且对于每个未使用的$i$，我们将其与最大的未使用倍数配对$i$仍然可用。 这是有效的，因为任何有效的配对都必须涉及除数结构，并且贪婪地消耗更高的倍数可以保留未来的可行性。 

另一种看待它的方式是，每个有效对都可以与一个“基本因子”相关联$d > 1$，以及可被整除的数字$d$形成一个可以内部匹配的池子。 通过从大到小进行处理，我们确保始终首先耗尽较高的倍数，从而防止小数字的碎片。

 | --- | --- | --- | --- |
 | 暴力配对 |$O(n^2)$|$O(n)$| 太慢了 |
 | 贪婪除数匹配 |$O(n \log n)$或者$O(n)$摊销|$O(n)$| 已接受 |

 ## 算法演练

 1. 创建数组或布尔标记`used[1..n]`初始化为 false，以及一个空的对列表。 这会跟踪号码是否已分配给一对。 
2. 迭代$i$从$n$下降到$2$。 相反的顺序可确保首先处理具有较少灵活配对选项的较大数字。 
3.如果$i$已使用，请跳过它，因为它已提交给前一对。 
4. 否则，尝试寻找合作伙伴$i$。 我们扫描多个$i$从$2i, 3i, 4i, \dots$最多$n$，并选择最大的$j$这样$j$未使用。 
5. 如果有这样一个$j$找到，标记两者$i$和$j$使用并记录该对$(i, j)$。 正确性来自于这样的事实$j$保证共享除数$i$， 所以$\gcd(i, j) \ge i > 1$。 
6. 继续，直到所有值$i$被处理。 

### 为什么它有效

 关键的不变量是，每当我们形成一对时$(i, j)$，我们保证$i$划分$j$，因此 gcd 条件始终满足。 首先配对较大指数的贪婪选择确保我们不会因为可用倍数较少的较小数字而浪费潜在的合作伙伴。 任何替代配对都可以转换为由该过程生成的配对，而不会减少配对的数量，因为除数类内的交换可以保持有效性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    used = [False] * (n + 1)
    res = []

    for i in range(n, 1, -1):
        if used[i]:
            continue

        partner = -1
        # find a multiple of i that is still free
        for j in range(2 * i, n + 1, i):
            if not used[j]:
                partner = j

        if partner != -1:
            used[i] = True
            used[partner] = True
            res.append((partner, i))

    print(len(res))
    for a, b in res:
        print(a, b)

if __name__ == "__main__":
    solve()
```实现直接遵循贪心构造。 重要的细节是按升序扫描倍数，但保留最后一个可用的倍数，这有效地选择了最大的未使用的倍数。 这可以防止过早锁定结构中稍后可能需要的小倍数。 

循环从$n$下降到$2$确保每个数字作为潜在基数被处理一次。 这`used`数组保证对的不相交。

 ## 边缘情况

 对于$n = 1$，算法立即不产生输出，因为循环从 2 开始，因此不会发生配对尝试。 结果正确为 0 对。 

对于像这样的小素数$n = 5$，只有具有共享约数的数字才能配对。 算法对$4$和$2$，离开$1,3,5$未使用，这与进一步有效对的不可能性相匹配。 贪婪扫描确保在单质素数阻止未来匹配之前利用复合结构。 

对于高度复合的范围，例如$n = 10$，该算法重复分配除数类内的对，例如 2、3 和 4 的倍数，并且不会重复使用任何数字，因为`used`数组在每一步都强制不相交。
