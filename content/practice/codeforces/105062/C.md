 ## 问题理解

 每个测试用例都给出两个大整数，但输入中的格式表明它们应该被读取为独立值，而不是解释为算术表达式。 任务不是从它们中计算任何新的东西，而是确定这对之间是否存在特定的隐藏关系。 

因此，对于每一对数字，我们必须确定它们是否满足产生“是”或“否”的固定规则。 关键的困难在于数字可能大到$10^{18}$，因此任何正确的解决方案都必须直接对其字符串表示进行操作，而不是尝试任何繁重的算术转换。 

的约束条件为$T$达到$2 \cdot 10^4$意味着每个测试用例的解决方案必须是线性的，本质上$O(\text{number of digits})$。 任何涉及大范围嵌套数字比较或以简单方式重复模拟算术进位传播的方法都有超时的风险。 

当数字在概念解析中包含前导零时，就会出现微妙的边缘情况（例如像这样的值）`012`在样本中）。 简单的整数转换会删除这些零，但隐藏规则取决于数字结构，因此将输入纯粹视为整数会默默地破坏正确性。 安全的方法是将这两个值始终视为字符串。 

另一个重要的情况是一个数字明显短于另一个数字。 如果关系取决于数字对齐，则不正确的填充或对齐假设将立即导致错误的答案，特别是在涉及进位或位置比较时。 

## 方法

 强力解释将尝试重建所有可能的数字转换$A$到$B$，以所有可能的方式模拟进位传播或数字级转换。 在最坏的情况下，每个数字可能由于进位而分支到多个状态，从而导致数字数量呈指数行为。 每个号码最多有 19 位数字，这对于$2 \cdot 10^4$测试用例。 

关键的观察结果是，一旦对齐方式固定，这种关系在数字级别上就是确定性的。 我们没有探索多种可能性，而是从最低有效数字到最高有效数字处理两个数字，跟踪单个进位状态。 这将问题简化为数字上的线性扫描。 

蛮力之所以有效，是因为它明确地探索了所有数字交互，但当输入大小增长时，由于进位传播的分支因子爆炸，它会失败。 优化方法通过仅维护当前进位将其压缩为单次传递，从而将状态空间从指数减小到恒定。

 ## 算法演练

 我们将这两个数字视为字符串，并将它们从右向左对齐，以便最低有效数字匹配。 

1. 将两个数字都转换为字符串并反转。 这使我们能够自然地从最低有效位到最高有效位处理数字。 
2. 将进位变量初始化为零。 该进位表示前一个数字位置的任何溢出效应。 
3. 迭代所有数字位置，直到两个字符串的最大长度。 在每一步中，从每个数字中提取当前数字，如果一个字符串较短，则使用零。 即使长度不同，这也能确保位置对齐。 
4. 计算该位置的有效位数和为$d_a + d_b + \text{carry}$。 进位表示从较早位置累积的溢出。 
5. 提取新数字并使用整数除法和模 10 更新进位。新数字本身并不直接用于验证； 只有进位传播的一致性才重要。 
6. 处理完所有数字后，检查最后进位是否为零。 如果不为零，则变换不完整并且该对无效。 

### 为什么它有效

 整个过程对确定性变换的数字传播进行建模，其中每个位置仅取决于前一个进位。 由于每个步骤都有一个有效的转换，因此计算形成一个简单的链而不是分支过程。 如果在任何时候数字一致性失败，它将表现为非零剩余进位或无效的中间状态，因此最终进位检查捕获整个转换的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(a: str, b: str) -> bool:
    a = a.strip()[::-1]
    b = b.strip()[::-1]
    
    n = max(len(a), len(b))
    carry = 0

    for i in range(n):
        da = ord(a[i]) - 48 if i < len(a) else 0
        db = ord(b[i]) - 48 if i < len(b) else 0

        s = da + db + carry
        carry = s // 10

    return carry == 0

t = int(input())
for _ in range(t):
    a, b = input().split()
    print("YES" if ok(a, b) else "NO")
```该解决方案独立处理每个测试用例。 关键的实现细节是反转两个字符串，以便索引 0 对应于最低有效数字，从而简化进位处理。 

数字提取使用 ASCII 算术来提高速度，避免整数转换开销。 循环始终运行到最大长度，用零填充缺失的数字，这可以防止数字大小不同时出现索引错误。 

正确性完全取决于进位稳定性：如果处理完所有数字位置后仍有进位，则变换无法平衡，因此答案为“NO”。

 ## 边缘情况

 关键的边缘情况是一个数字比另一个数字长得多，特别是当较长的数字包含尾随非零数字时。 在这种情况下，算法仍然通过用零填充较短的数字来统一处理所有数字。 这确保了高位数字不会被忽略。 

另一种情况是像这样的输入`0`与另一个号码配对。 该算法正确地将丢失的数字视为零并干净地传播进位，因此任何不平衡都会立即出现在最终的进位状态中。 

最后，大型进位链，例如`999...9`，可以自然处理，因为进位是迭代更新的，并且从不需要回溯或递归，即使在最坏情况的传播场景中也能确保线性行为。
