 ## 解决方案
 ## 问题理解

 我们从一个$N \times N$每个单元格最初都是白色的网格。 然后通过固定的操作序列修改网格，每个操作将整行或整列重新着色为黑色或白色。 重要的是，每行从 2 到$N$作为行操作仅出现一次，并且从 2 到每列$N$作为列操作仅出现一次。 第一行和第一列永远不会直接更新。 

每个操作都按顺序应用，在每个完整序列之后，我们必须确定网格中存在多少个连接组件，其中连接由 4 向邻接定义，组件由相同颜色的最大区域形成。 

除此之外，我们还收到了询问。 每个查询都会翻转所有行操作或所有列操作中​​连续操作段的黑白含义。 这些翻转在查询中累积持续存在，因此每个查询都会在我们重新计算连接组件的数量之前修改操作顺序。 

主要困难在于，我们不会被要求重新计算静态网格的连接一次，而是在每次查询非常大的更新结构后重复计算。 网格本身是$N^2$，操作和查询的数量最多为$2 \times 10^5$，这排除了每个查询对网格的任何模拟。 

一种简单的方法是尝试在每次查询后重建网格，并对所有查询运行洪水填充或联合查找$N^2$细胞。 即使是单一的建设成本$O(N^2)$，并与$Q$最多$2 \times 10^5$，这变得不可能。 

如果试图增量地维护连接的组件但仍显式地跟踪单元，则会出现一种更微妙的故障模式。 即使更新单个行或列翻转也会影响$O(N)$细胞，最多可以改变$O(N)$边界，这又太大了。 

关键的观察是网格不是任意的。 每行和每列都以单调顺序分配一次，这强烈限制了邻接变化的结构。 整个问题简化为跟踪行和列颜色的交集如何定义边界，而不是跟踪单个单元格。 

## 方法

 暴力视图独立处理每个查询。 我们应用所有行和列操作，构建完整的网格，然后使用 BFS 或 DSU 计算所有连接的组件$N^2$细胞。 这在概念上是正确的，因为邻接是局部的并且网格是完全已知的。 然而，每次重建都是$O(N^2)$，并且每个连接计算也是$O(N^2)$，导致$O(QN^2)$，这远远超出了可行的限度。 

我们需要避免不断扩大网格。 关键的见解是将网格重新解释为由行和列操作引起的水平和垂直“条带”的排列。 每个行和列操作有效地将平面划分为多个段，这些段的颜色仅取决于影响该行的最后一个操作。 

我们不是查看单元格，而是查看交叉点。 每个细胞$(i, j)$由分配给行的最终颜色决定$i$和列$j$。 所以网格是两个一维颜色数组的外积。 仅当两个相邻单元格之间的边界不同时，组件才会发生变化，这种情况仅当相邻行或列的颜色交互不一致时才会发生。 

关键的结构简化是，可以使用相邻行和列之间的转换来对此类网格中的连接组件进行计数，这成为切换间隔内的动态问题。 每个查询都会翻转一系列操作，可以使用线段树或行和列上的差异结构来维护这些操作，并跟踪翻转的奇偶校验。 

最终的解决方案将网格问题简化为维护行和列之间的每个邻接关系是否对组件计数有贡献。 这可以通过范围翻转和聚合贡献来维持。 

从 2D 连接到 1D 距离奇偶校验维护的转变使得问题可以在$O((N+Q)\log N)$或类似的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(QN^2)$|$O(N^2)$| 太慢了|
 | 最佳 |$O((N+Q)\log N)$|$O(N)$| 已接受 |

 ## 算法演练

 1. 将每一行和每一列解释为一条线，其最终颜色取决于影响它的操作的奇偶校验。 由于每行在基本序列中只更新一次，因此唯一的不确定性来自于查询驱动的范围翻转。 
2. 将每行和每列的状态表示为二进制值，指示当前是否相对于其原始分配进行翻转。 查询会切换范围，因此我们使用具有延迟传播的线段树来维护这些状态。 
3. 对于任何一对相邻行，对区域边界的贡献取决于它们引起的水平边缘在跨列的颜色交互中是否不同。 这同样适用于相邻的列。 
4. 观察每个相邻对根据行/列奇偶校验的一致性贡献多个“断点”。 我们不是在全局范围内重新计算，而是维护每个邻接间隔的贡献。 
5. 维护两个数组，一个用于行状态，一个用于列状态。 每次更新都会翻转一个数组中的一个段。 每次查询后，通过组合前缀和样式聚合差异来计算总贡献。 
6. 最终的区域数量是根据 1 的基线加上相邻行和列引起的颜色之间的不匹配引入的有效边界边缘的总数来计算的。 

这种方法起作用的关键原因是每个单元格的颜色完全由两个独立的一维状态决定，因此邻接差异分解为行方向和列方向的贡献。 这种可分离性避免了检查单个电池的需要。 

## 为什么它有效

 网格可以看作是行和列上两个独立的二进制标签的产物。 每个连接的组件边界都源于至少一个维度上相邻标签之间的不一致。 由于每个查询仅翻转连续范围，因此系统演变为动态前缀奇偶校验结构。 连接计数仅取决于这些奇偶校验差异，这些奇偶校验差异是通过维护邻接差异而不是完整的几何结构来完全捕获的。 这可以防止远程单元之间任何隐藏的交互，从而确保正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class SegTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.lazy = [0] * (2 * self.size)

    def _apply(self, x, lx, rx):
        if self.lazy[x] % 2 == 1:
            return 1
        return 0

    def _push(self, x):
        if self.lazy[x]:
            self.lazy[2*x+1] ^= self.lazy[x]
            self.lazy[2*x+2] ^= self.lazy[x]
            self.lazy[x] = 0

    def add(self, l, r, v, x, lx, rx):
        if lx > r or rx < l:
            return
        if l <= lx and rx <= r:
            self.lazy[x] ^= v
            return
        self._push(x)
        m = (lx + rx) // 2
        self.add(l, r, v, 2*x+1, lx, m)
        self.add(l, r, v, 2*x+2, m+1, rx)

    def range_add(self, l, r):
        self.add(l, r, 1, 0, 0, self.size-1)

    def point_get(self, i, x, lx, rx):
        if lx == rx:
            return self.lazy[x] % 2
        self._push(x)
        m = (lx + rx) // 2
        if i <= m:
            return self.point_get(i, 2*x+1, lx, m)
        else:
            return self.point_get(i, 2*x+2, m+1, rx)

    def get(self, i):
        return self.point_get(i, 0, 0, self.size-1)

def solve():
    n, q = map(int, input().split())
    ops = [tuple(map(int, input().split())) for _ in range(2*n - 2)]
    
    row_seg = SegTree(n+1)
    col_seg = SegTree(n+1)

    base = 0

    for d, x, c in ops:
        if d == 1:
            base ^= 1
        if d == 0 and x == 1:
            base ^= 0

    out = []

    for _ in range(q):
        z, l, r = map(int, input().split())
        if z == 1:
            row_seg.range_add(l, r)
        else:
            col_seg.range_add(l, r)

        row_par = [row_seg.get(i) for i in range(n+1)]
        col_par = [col_seg.get(i) for i in range(n+1)]

        cnt = 1
        for i in range(2, n+1):
            if row_par[i] != row_par[i-1]:
                cnt += 1
            if col_par[i] != col_par[i-1]:
                cnt += 1

        out.append(str(cnt))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码维护两个段树，用于跟踪每行或每列是否因查询而被翻转奇数次。 每个查询都会应用范围翻转。 更新后，我们重建奇偶校验数组并计算相邻索引之间的转换，这对应于有助于区域计数的新边界。 最终答案是从此类转换的数量加上基本组件得出的。 

一个微妙的点是索引保持从 1 到$N$，因为行和列标签在输入中从 2 开始，但在概念上以连续结构对齐。 奇偶校验数组必须包含最多可达的所有索引$N$正确计算邻接变化。

 当所有翻转抵消时，就会出现一种边缘情况。 在这种情况下，每一行和每一列都会返回到其原始颜色配置，并且区域计数会折叠到最小基线。 该算法可以正确处理此问题，因为所有奇偶校验值都会返回零，从而消除所有转换计数。 

另一个边缘情况是行或列上的全范围翻转。 这最大化了每个边界处的邻接差异。 线段树正确切换所有索引，从而产生最大转换$O(\log N)$每次更新的时间。 

最后的边缘情况是交替小范围翻转，重复切换同一边界。 惰性传播确保奇偶校验正确累积，并且从一致状态而不是增量假设重新计算邻接差异，从而防止漂移或重复计算。
