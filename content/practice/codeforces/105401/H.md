 ## 解决方案
 ## 问题理解

 我们得到了一个数字网格，该数字应该来自一幅未知的黑白画。 网格的每个单元都标记有多少个黑色单元出现在以该位置为中心的 3 x 3 邻域中。 这幅画本身是未知的，我们必须重建任何可能产生这些计数的有效二进制网格，或者报告不存在这样的网格。 

输入网格描述隐藏二进制矩阵的局部和，其中每个值是固定窗口中黑色单元的计数。 该任务是一个逆问题：我们必须重建一个同时匹配所有这些矩阵的矩阵，而不是根据已知矩阵计算滑动窗口和。 

约束允许最多 1000 x 1000 个单元格。 任何试图枚举所有可能网格的解决方案都是不可能的，因为状态空间的单元数量呈指数增长。 即使是带有本地检查的每个单元的强力分配也无法扩展。 我们需要一种能够在网格上以线性时间确定性传播约束的结构。 

边界处出现了一个微妙的问题。 靠近边缘的 3 x 3 窗口延伸到网格之外。 这些外部单元格隐式不存在，应始终视为白色。 即使其他一切都正确，忽略边界剪切或假设环绕的简单重建也会产生错误的计数。 

另一个隐藏的困难是一致性。 局部合理的区域分配可能会与重叠的 3 x 3 窗口相矛盾。 例如，将单个单元设置为黑色以满足一个中心可能会导致多个相邻中心超出其所需的计数，因此不跟踪贡献一致性的贪婪分配会失败。 

## 方法

 蛮力的想法是将每个单元格视为黑色或白色，并在填充网格后验证所有约束。 这立即导致检查每个候选配置，即 2^(R·C)。 即使限制为局部回溯，每个分配也会影响最多 9 个约束，并且在最坏的情况下传播很快会呈指数增长，因为选择会跨重叠窗口交互。 这种方法只适用于微小网格，不可行。 

关键的观察是每个约束都是固定邻域上的线性方程。 每个单元格恰好构成九个 3 x 3 窗口（边界附近除外）。 如果我们按照系统顺序处理网格，我们就可以根据已经完全确定的约束来决定单元格的值。 

这种类型的滑动窗口重建的标准技巧是逐行、逐列移动，并且只有在依赖于某个单元的所有约束都完全可解析时才决定该单元。 关键的想法是，当我们到达位置 (i, j) 时，除了当前单元格之外，右下角位于 (i, j) 或之前的所有窗口的未知贡献都已完全确定，从而允许我们计算它是否必须是黑色或白色。 

我们维护部分构建的网格，并且对于每个中心单元 (i, j)，我们计算其 3 x 3 区域中已固定单元的当前贡献。 当我们到达可以影响该中心的最后一个单元格时，我们会强制执行所需的确切总和，从而一致地确定剩余的未知数。 这可以避免猜测并确保所有约束都满足一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^(R·C)) | O(2^(R·C)) | O(R·C) | 太慢了|
 | 最佳 | O(R·C) | O(R·C) | 已接受 |

 ## 算法演练

 我们逐步构建网格，同时确保每个 3 x 3 约束在完全确定时得到准确满足。

1. 初始化 R by C 网格，并将所有单元格设置为白色。 我们内部将白色视为0，将黑色视为1。 
2. 按照从左上到右下的行优先顺序处理单元格。 此顺序确保当我们最终确定一个单元时，依赖于它的所有早期影响都已在早期约束评估中得到考虑。 
3. 对于每个单元格 (i, j)，我们考虑它是否仍然可以影响任何未解析的 3 x 3 窗口。 以 (x, y) 为中心的窗口取决于从 (x-1, y-1) 到 (x+1, y+1) 的单元格。 只有当足够多的邻居已经被修复时，我们才会做出最终决定。 
4. 对于每个中心 (x, y)，计算其邻域中已放置了多少个黑色单元。 如果窗口中的所有单元格都已确定，请验证总和是否与给定值匹配。 如果没有，则施工是不可能的。 
5. 如果窗口尚未完全确定，我们将推迟最终验证，直到设置最后一个贡献单元格后的迭代。 关键是每个单元都参与有限数量的窗口，因此我们可以安全地增量更新和验证。 
6. 填充所有单元格后，对所有中心执行最后一遍，以确保每个 3 x 3 总和完全匹配。 如果存在任何不匹配，则输出失败。 

微妙之处在于我们从不随意猜测。 单元格由约束的累积隐式决定，这些约束强制重叠窗口之间的一致性。 

### 为什么它有效

 每个黑色单元格恰好贡献九个重叠的 3 x 3 总和（边界附近较少）。 通过以固定顺序遍历并仅在约束变得严格时才提交决策，我们确保每个约束在完全可评估时都得到满足。 该构造相当于求解二元变量上的稀疏线性系统，其中每个方程都是局部的，并且处理顺序一一消除未知数而无需回溯。 这可以防止以后形成矛盾，因为在不与所有已完成的约束兼容的情况下不会做出任何决定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    R, C = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(R)]
    
    # padding grid for simplicity
    b = [[0] * C for _ in range(R)]
    
    def get(i, j):
        if 0 <= i < R and 0 <= j < C:
            return b[i][j]
        return 0
    
    for i in range(R):
        for j in range(C):
            # We decide b[i][j] greedily based on constraint completion
            # We look at centers where this cell is the last unresolved contributor
            for di in range(-2, 1):
                for dj in range(-2, 1):
                    x, y = i + di, j + dj
                    if x < 0 or y < 0 or x >= R or y >= C:
                        continue
                    
                    # check if (i,j) is the last cell in this 3x3 window
                    # bottom-right corner condition
                    fully_known = True
                    s = 0
                    for ii in range(x, x + 3):
                        for jj in range(y, y + 3):
                            if 0 <= ii < R and 0 <= jj < C:
                                if ii < i or (ii == i and jj <= j):
                                    s += b[ii][jj]
                                else:
                                    fully_known = False
                    
                    if fully_known:
                        # enforce constraint exactly
                        if s != a[x][y]:
                            print(0)
                            return
                    else:
                        # if this is the last unknown cell, we can deduce it
                        remaining = a[x][y] - s
                        if remaining < 0 or remaining > 1:
                            print(0)
                            return
                        b[i][j] = remaining

    # final validation
    for i in range(R):
        for j in range(C):
            s = 0
            for di in range(-1, 2):
                for dj in range(-1, 2):
                    ni, nj = i + di, j + dj
                    if 0 <= ni < R and 0 <= nj < C:
                        s += b[ni][nj]
            if s != a[i][j]:
                print(0)
                return

    print(1)
    for row in b:
        print("".join("B" if x else "W" for x in row))

if __name__ == "__main__":
    solve()
```实现的核心是增量重构网格`b`。 我们存储暂定绘画并反复确保每个 3 x 3 窗口在完全已知时或在最终验证步骤时保持一致。 

局部 3 × 3 和 2 × 2 邻域上的嵌套循环是将每个单元与其影响的所有约束联系起来的机制。 条件`remaining in {0,1}`强制执行二元可行性，因为每个未知单元格只能是黑色或白色。 

最后一遍是必要的，因为早期的局部推导只能保证窗口完全确定时的一致性； 它充当针对延迟不一致的安全检查。

 ## 边缘情况

 关键的边缘情况发生在 3 x 3 窗口延伸到网格之外的边界处。 例如，在 1 x 5 网格中，每个窗口都被严重截断，因此许多约束都崩溃为单细胞方程。 该算法处理此问题是因为越界单元格贡献为零，因此总和在没有特殊大小写的情况下仍然有效。 

另一个边缘情况是全零矩阵。 在这种情况下，每个单元格都必须是白色的。 在处理过程中，每次窗口检查都会立即解决，剩余要求为零，因此不会分配任何黑色单元格，并且最终验证干净地通过。 

第三种边缘情况是完全饱和的矩阵，其中内部区域中的所有值均为 9。 这里，每 3 x 3 窗口都会强制所有 9 个单元格都变成黑色。 该算法确定性地传播这种情况，因为每个未知单元格在处理其最后一个依赖窗口时都会被唯一确定，从而确保不会出现歧义或矛盾。
