 ## 解决方案
 ## 问题理解

 我们正在模拟一个概率回合制游戏，其中两个玩家反复玩剪刀石头布，但结果并不能立即决定获胜者。 相反，每一轮都会决定谁成为下一轮的“攻击者”，如果有人已经在攻击，那么只有平局才有可能结束游戏。 

对于每一轮，输入都会给出 A 和 B 的移动选择的独立概率分布。 从这些分布中，我们可以得出每一轮的三个有意义的概率：A 赢得该轮（石头剪刀布优势）、B 赢得该轮或该轮平局。 

游戏随着时间的推移以隐藏状态演变：是否没有攻击者、A 是当前攻击者，还是 B 是当前攻击者。 如果在没有进攻者的情况下出现平局，则不会发生任何特殊情况，比赛继续进行。 如果平局发生时有人已经是攻击者，则该攻击者立即获胜并且游戏结束。 如果没有平局，则该轮的获胜者成为下一轮的攻击者。 

任务是计算 A 最终成为获胜者的概率，然后该过程要么以 B 有利的方式结束，要么在没有获胜者的情况下结束。 最终答案必须作为 998244353 下的模值返回。 

约束最多为 2×10^5 轮，因此任何显式模拟所有概率路径的方法都是不可能的。 即使是对所有历史的状态空间扩展也会呈指数级增长。 针对少量状态的线性时间动态规划是唯一可行的方向。 

一个微妙的边缘情况是，在攻击者建立后，在平局发生之前没有攻击者出现。 例如，如果 A 在某个时刻成为攻击者，后来出现平局，则游戏立即结束。 仅跟踪攻击者转换但忘记关系可以终止该过程的简单模拟会高估持续概率并产生不正确的结果。 

当所有轮次都符合期望时，就会出现另一个极端情况（例如，相同的分布导致对称）。 在这种情况下，游戏结束的唯一方式是通过较早的攻击者状态，而忘记从攻击者状态传播终端转换会导致错误地假设无限延续的答案。 

## 方法

 暴力解释将明确模拟 n 轮中所有可能的结果序列。 每轮有三种结果：A 成为攻击者、B 成为攻击者或平局。 这会产生 3^n 种可能的轨迹，甚至合并相同的状态也无济于事，因为攻击者状态以路径相关的方式与终止条件交互。 This is far beyond feasible computation.

 关键的观察是，继续该过程所需的唯一信息不是完整的历史记录，而只有两部分：当前回合索引和攻击者状态。 各轮之间的其他一切都是独立的，因为选择是独立的并且只影响下一个状态。 

这将问题简化为具有三种状态的马尔可夫过程：无攻击者、A攻击者和B攻击者。 每轮都会贡献这些状态之间的固定转换概率，以及过程结束时的吸收转换。 一旦我们认识到这一点，整个问题就变成了一个超过 n 个步骤的前向动态程序。 

我们在每轮开始时维护三个状态的概率分布，并使用该轮预先计算的结果概率将其向前传播。 每当我们在攻击者状态中遇到平局时，我们就会转变为吸引人的“A 获胜”或“B 获胜”事件，并累积对最终答案的贡献。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力战胜结果 | O(3^n) | O(3^n) | O(n) | 太慢了|
 | DP 超过攻击者状态 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将每一轮压缩为三个概率：A 击败 B、B 击败 A 以及平局。 这些是直接根据给定的独立分布计算的。 

然后，我们在每轮开始时保持游戏处于给定状态的概率的 DP。 

### 步骤

 1. 计算每一轮 i 的概率 pA[i]、pB[i] 和 pT[i]，分别对应 A 获胜、B 获胜和平局。 

这些来自 R、S 和 P 选择的独立乘积概率的求和。 
2. 在第 1 轮初始化 DP，其中 dp_none = 1、dp_A = 0、dp_B = 0。 

这表明最初没有攻击者。 
3. 对于从 1 到 n 的每一轮 i，使用 pA[i]、pB[i]、pT[i] 从三个状态进行过程转换。 
4. 从状态“无攻击者”，仅传播非终端转换：

 获胜将概率质量发送至状态 A 进行下一轮，B 获胜发送至状态 B，平局则保持无攻击者状态。 
5. 从“攻击者”状态开始，谨慎处理结果：

 一场胜利让 A 成为下一轮的攻击者，

 B 获胜将攻击者切换到 B，

 平局立即得出答案，因为在这种情况下 A 获胜，并且没有概率继续前进。 
6.从状态“B攻击者”，类似：

 胜利将攻击者切换到 A，

 B 获胜，使 B 成为攻击者，

 平局以 B 获胜结束比赛，因此对 A 的答案没有贡献。 
7.处理完所有轮次后，任何剩余的概率质量都对应于从未结束的游戏； 这些对 A 的获胜概率没有任何贡献。 
8. 返回 A 取模 998244353 的累积概率。 

### 为什么它有效

 在任何时候，未来的行为仅取决于当前回合索引和攻击者状态。 所有早期历史都只能通过这种状态影响未来，因此这是一个足够的统计数据。 DP 保留了这些状态之间的精确概率质量，并且每次转换要么保留该过程，要么将其移至最终吸收事件一次，确保不会出现过度计数或概率损失。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n = int(input())
    
    A = []
    B = []
    
    for _ in range(n):
        r, s, p = map(int, input().split())
        tot = r + s + p
        A.append((r * modinv(tot)) % MOD)
        A.append((s * modinv(tot)) % MOD)
        A.append((p * modinv(tot)) % MOD)
    
    Ar = []
    As = []
    Ap = []
    for i in range(n):
        r, s, p = map(int, input().split())
        tot = r + s + p
        Ar.append((r * modinv(tot)) % MOD)
        As.append((s * modinv(tot)) % MOD)
        Ap.append((p * modinv(tot)) % MOD)
    
    dp0 = 1
    dpA = 0
    dpB = 0
    ans = 0
    
    for i in range(n):
        ar, as_, ap = A[3*i], A[3*i+1], A[3*i+2]
        br = Ar[i]
        bs = As[i]
        bp = Ap[i]
        
        pAwin = (ar * bs + as_ * bp + ap * br) % MOD
        pBwin = (br * as_ + bs * ap + bp * ar) % MOD
        pTie = (pAwin + pBwin) % MOD
        pTie = (1 - pTie) % MOD
        
        ndp0 = ndpA = ndpB = 0
        
        ndp0 = (ndp0 + dp0 * pTie) % MOD
        ndpA = (ndpA + dp0 * pAwin) % MOD
        ndpB = (ndpB + dp0 * pBwin) % MOD
        
        ndpA = (ndpA + dpA * pAwin) % MOD
        ndpB = (ndpB + dpA * pBwin) % MOD
        ans = (ans + dpA * pTie) % MOD
        
        ndpA = (ndpA + dpB * pAwin) % MOD
        ndpB = (ndpB + dpB * pBwin) % MOD
        
        dp0, dpA, dpB = ndp0, ndpA, ndpB
    
    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现在每轮开始时维护与攻击者状态相对应的三个滚动概率质量。 关键的微妙之处在于，攻击者状态中的平局结果不会向前过渡，而是直接贡献于最终答案，这就是为什么它们被累积成`ans`而不是进入下一个 DP 层。 

概率计算自始至终都使用模算术，用模逆来表示 998244353 以下的分数。每轮都在恒定时间内处理，确保线性性能。

 ## 边缘情况

 当攻击者较早建立并且下一次交互是平局时，就会出现一个重要的边缘情况。 在这种情况下，游戏立即终止，并准确贡献当前攻击者的获胜概率。 DP 明确地将所有平局概率质量从攻击者状态路由到答案累加器中，因此没有连续状态错误地生存。 

另一个边缘情况是，当没有创建攻击者并且游戏只看到平局时。 在这种情况下，dp0 只是衰减，而 dpA 和 dpB 保持为零，并且不会向答案添加任何贡献。 结果正确地变为零，因为没有触发有利于 A 的终止条件。 

当攻击者状态在 A 和 B 之间摇摆而没有联系时，就会出现最后的边缘情况。 DP 正确地保留了各轮中的两种可能性，因为非平局结果只会更新攻击者状态而不会终止，从而确保概率质量既不会丢失也不会过早吸收。
