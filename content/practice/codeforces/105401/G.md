 ## 解决方案
 ## 问题理解

 我们有一套$N$分数已经固定的俱乐部，还有一个名为 RUN 的额外俱乐部，我们可以在最后自由选择其分数。 最终排名完全取决于有多少俱乐部拥有严格更高的分数：俱乐部的排名是得分更高的俱乐部数量加一。 较低的等级意味着更好的地位。 

在选择RUN的分数之前，我们可以减少其他人的分数$N$俱乐部。 各俱乐部$i$从分数开始$a_i$，如果我们减少一些量$T$，我们支付的“怀疑成本”为$T \cdot b_i$。 对所有俱乐部的总体怀疑必须严格控制在以下水平$K$。 我们不允许增加任何分数，只能减少或保持不变，并且分数必须保持非负数。 

完成所有调整后，我们设定了RUN的分数。 我们希望 RUN 最多能获得排名$X$，在实现这一目标的所有方法中，我们希望 RUN 的初始选择分数尽可能小。 

解读排名条件，RUN最多需要$X-1$最终得分严格更高的俱乐部。 所以任务就变成了：我们可能会在全局预算约束下对已有的分数进行加权减少，然后选择尽可能小的RUN的分数，同时保证最多$X-1$俱乐部超过了它。 

这些限制迫使我们走向$O(N \log N)$或者$O(N)$解决方案。 和$N \le 10^5$，尝试不同的分数配置或子集的任何二次方都是立即不可行的。 线性成本和单调分数减少的存在强烈表明贪婪或基于前缀的结构，可能与排序相结合。 

当减少分配效率低下时，就会出现微妙的失败情况。 例如，如果我们尽早贪婪地减少错误的俱乐部，我们可能会在低影响的变化上浪费预算，然后无法抑制足够的高分俱乐部来满足排名要求。 另一个常见的陷阱是假设我们只关心减少顶部$X-1$俱乐部按初始分数，忽略其他俱乐部更便宜的折扣可能更有效。 

一个小的说明性边缘情况是当一个俱乐部得分很高但极高时$b_i$，而许多其他目标略高于目标，但降低成本较低。 天真的贪婪仅靠分数会失败，而最佳行为则取决于成本效益。 

## 方法

 如果我们修正 RUN 的最终得分$S$，条件“RUN最多有排名$X$” 的意思是最多$X-1$俱乐部仍然严格高于$S$。 同样，在所有俱乐部中，我们必须确保不超过$X-1$其中有最终分数$> S$。 

对于固定的$S$, 各俱乐部$i$属于以下三种角色之一：

 如果$a_i \le S$，我们根本不需要减少它，因为它已经不威胁 RUN 了。 

如果$a_i > S$，那么我们要么接受它保持在上面$S$，或者我们最多将其减少到$S$。 如果我们选择减少它，我们必须至少减少它$a_i - S$, 支付费用$(a_i - S)b_i$。 

所以对于一个固定的$S$, 每个俱乐部都有$a_i > S$给了我们一个二元选择：要么我们“消除”它的威胁（付出成本），要么我们将它作为竞争对手。 我们最多可以离开$X-1$竞争对手。 这立即建议选择完全减少哪些俱乐部，以便我们消除除最多之外的所有俱乐部$X-1$昂贵的。 

重构：俱乐部之间$a_i > S$，假设有$m$这样的俱乐部。 我们最多可以离开$X-1$，所以我们必须至少消除$m-(X-1)$其中。 每次淘汰都有成本$(a_i - S)b_i$。 为了最大限度地减少怀疑，我们选择最便宜的来消除。 

因此对于固定的$S$，可行性简化为对这些成本进行排序并检查所需成本的最小数量之和是否低于$K$。 

这将原始问题变成了单调决策问题$S$: 如果有分数$S$是可行的，那么任何更小的$S$也是可行的，因为更多的俱乐部变得危险，成本只会增加。 这种单调性允许对答案进行二分搜索。 

蛮力想法将尝试所有可能的 RUN 分数$0$最多$\max a_i$，并通过扫描所有俱乐部来计算每个俱乐部的淘汰成本。 那是$O(N \cdot \max A)$，这太慢了。 关键的观察是可行性是单调的并且可计算的$O(N \log N)$，启用二分搜索。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N \cdot \max A)$|$O(1)$| 太慢了|
 | 最佳 |$O(N \log N \log \max A)$|$O(N)$| 已接受 |

 ## 算法演练

 我们二分查找可能的最小 RUN 分数$S$。 

1. 定义一个函数`check(S)`决定RUN最多能否达到排名$X$如果它的分数是$S$。 
2. 内部`check(S)`，计算每个俱乐部$i$和$a_i > S$将其降低到的成本$S$，即$(a_i - S) \cdot b_i$。 俱乐部与$a_i \le S$不贡献任何成本，并且与排名压力无关。 原因是只有严格更高的分数才对排名重要。 
3. 统计目前有多少俱乐部超过$S$。 打电话给这个$m$。 
4. 如果$m \le X-1$，则无需减少，配置自动生效。 
5. 否则我们必须消除$m-(X-1)$俱乐部。 收集减少每一项的所有成本$m$俱乐部，对它们进行排序，然后将最小的相加$m-(X-1)$。 这是减少足够竞争对手的最便宜的方法。 
6. 如果该金额小于$K$，配置可行； 否则就不是。 
7. 使用二分查找$S$在范围内$[0, \max a_i]$，保持最小可行值。 

为什么这是正确的取决于决策问题的结构。 对于固定阈值$S$，如果我们决定抑制成本，每个俱乐部都会独立地增加成本，并且部分减少俱乐部而不将其降低到$S$，因为任何中间值仍然会使其高于 RUN 并且不会减少竞争对手数量。 因此，每个最优策略都相当于选择一个俱乐部的子集来完全压制，并具有附加成本，并且最优子集始终是最便宜的。 

可行性的单调性$S$因为增加$S$只会让事情变得更容易：满足要求的俱乐部越来越少$a_i > S$，以及每个成本$(a_i - S)b_i$减少或完全消失。 所以一旦值为$S$是可行的，所有较大的值仍然可行，证明二分搜索是合理的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(S, arr, K, X):
    costs = []
    m = 0

    for a, b in arr:
        if a > S:
            m += 1
            costs.append((a - S) * b)

    if m <= X - 1:
        return True

    need = m - (X - 1)
    costs.sort()

    total = sum(costs[:need])
    return total < K

def solve():
    N, K, X = map(int, input().split())
    arr = [tuple(map(int, input().split())) for _ in range(N)]

    lo, hi = 0, max(a for a, _ in arr)

    ans = hi

    while lo <= hi:
        mid = (lo + hi) // 2
        if check(mid, arr, K, X):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```实施准确地反映了决策过程。 这`check`函数重建固定阈值的成本结构，并评估我们是否有能力消除足够的俱乐部。 对成本进行排序是确保我们始终选择最便宜的抑制集的关键步骤。 

一个微妙的点是怀疑上的严格不平等：条件是“小于$K$”，因此必须拒绝相等。这在最终比较中直接处理。

 ## 工作示例

 ### 示例 1

 我们测试候选阈值$S$里面的二分查找。 检查过程的行为如下。 

| 俱乐部状态|$a_i$|$b_i$|$a_i > S?$| 成本|
 | ---| ---| ---| ---| ---|
 | 1 | 21 | 21 2 | 是的 |$(21-S)\cdot2$|
 | 2 | 100 | 100 0 | 是的 |$(100-S)\cdot0$|
 | 3 | 50 | 50 13 | 是的 |$(50-S)\cdot13$|
 | 4 | 8 | 8 | 取决于| 如果不高于则为 0 |

 计算出所有高于阈值的成本后，假设我们有$m$威胁超过$S$。 如果超过$X-1$剩下的，我们对成本进行排序并选择最便宜的消除，直到只有$X-1$保持。 

该跟踪表明只有上面的相对定位$S$重要的是，而不是低于它的确切分数值。 该算法有效地将排名约束转化为预算抑制问题。 

### 示例 2

 类似的轨迹说明了相反的情况，其中许多俱乐部已经低于所选的水平$S$。 

| 俱乐部状态|$a_i$|$b_i$|$a_i > S?$| 成本|
 | ---| ---| ---| ---| ---|
 | 1 | 3 | 15 | 15 没有| 0 |
 | 2 | 15 | 15 4 | 是的 |$(15-S)\cdot4$|
 | 3 | 24 | 11 | 11 是的 |$(24-S)\cdot11$|
 | 4 | 10 | 10 3 | 取决于| 0 或成本 |

 这里，增加$S$减少了主动威胁的数量，直接证明了单调的可行性。 一旦阈值变得可行，任何更高的阈值只会减少昂贵操作的候选集。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(N \log N \log A)$| 在分数范围内进行二分搜索，每个可行性检查排序最多$N$成本|
 | 空间|$O(N)$| 输入和临时成本清单的存储|

 约束条件允许$N = 10^5$， 和$A \le 10^6$，使双对数因子完全安全。 排序在每次检查中占主导地位，但由于常数因子较小以及排序的可行性，排序仍然在限制范围内。$N \log N$运营。

 ## 边缘情况

 当出现极端情况时$X = N+1$，意味着 RUN 可以是最后一个。 在这种情况下，任何俱乐部无论成绩如何都不需要被淘汰。 该算法自然地处理它：$m-(X-1)$变为零或负数，因此不会对成本进行求和，并且每个$S$只要是非负的，都是可行的。 二分查找正确返回 0。 

另一个微妙的情况是当所有$a_i \le S$。 然后$m = 0$，因此检查立即返回 true。 这反映了这样一个事实：无论成本考虑如何，RUN 都已经超越了所有人。 

当减少一些俱乐部的成本极其昂贵时，就会出现更微妙的情况，因此最好将它们保留在 RUN 之上，而不是压制许多更便宜的俱乐部。 基于排序的选择保证了这种行为，因为可行性仅取决于选择所需消除的最便宜的子集，而不是基于初始分数的任何启发式。
