 ## 解决方案
 ## 问题理解

 我们给定一个长度的字符串$n$超过一个字母表大小$C$，但我们不是直接处理字符串，而是通过 KMP 前缀函数（失败函数）查看其结构。 对于每个位置$i$，函数$f(i)$告诉我们字符串的最长正确前缀的长度，该前缀也匹配以结尾的后缀$i$。 这自然地定义了每个位置的有向边$i$到更小的索引$f(i)$，有节点$0$充当通用根。 

如果我们放置节点$0,1,2,\dots,n$按顺序在水平线上绘制每条边$i \to f(i)$作为线上方的弧，我们获得了这些故障链接的几何图。 该问题询问哪些弦产生可以在没有任何交叉的情况下绘制这些弧的配置。 

任务是计算有多少个字符串的长度$n$超过$C$字母生成这样一个不交叉的结构，模$998244353$。 

这些约束立即使我们远离任何依赖于位置迭代的构造。 的价值$n$可以大到$10^{18}$，这意味着我们无法模拟前缀函数构造，甚至无法显式推理字符串。 任何有效的解决方案都必须将整个结构折叠成恒定大小的递归式或可以快速求幂的直接公式。 

此类问题的一个微妙陷阱是假设嵌入约束以复杂的、依赖于结构的方式过滤掉大部分字符串。 如果这是真的，我们预计答案将取决于复杂的边界组合。 然而，限制$n$强烈建议相反：条件必须简化为仅取决于$n$通过简单的指数或线性递推。 

一个小的健全性检查已经暗示了这一点。 为了$n=3, C=3$，答案是$27$，等于$C^n$。 这表明，至少对于小情况，没有任何限制实际上是消除任何字符串。 

## 方法

 蛮力观点从定义开始：生成所有$C^n$字符串，计算它们的前缀函数，构建故障链接图，并测试在数轴上绘制时是否有任何两条弧相交。 前缀函数可以计算为$O(n)$，并且根据实施情况，也可以在线性或二次时间内完成简单的交叉测试。 这导致整体复杂性约为$O(C^n \cdot n)$，即使对于$n=30$，并且完全不相关$n \le 10^{18}$。 

关键的观察结果是，KMP 故障链接的结构比任意指针受到更多约束。 每个节点$i$连接到一个严格较小的索引$f(i)$，并且这些链接始终代表前缀的边界。 字符串的边界本质上是嵌套的：如果前缀为 length$a$也是一个后缀，那么该前缀的所有较短边界也是原始字符串的边界。 此嵌套属性可防止在线条上绘制时产生交叉弧所需的交错类型。 

当我们将其转化为几何形状时，每条边$i \to f(i)$对应一个区间$[f(i), i]$。 交叉需要两个交错的间隔，但边界结构确保这些间隔始终是嵌套的或不相交的。 失效函数永远不会产生交叉配置所需的交错模式。 

这意味着字母表上的每个字符串都会自动生成平面嵌入。 嵌入条件没有任何限制。 

一旦建立，问题就减少到计算所有可能的长度字符串$n$超过$C$符号，简单来说就是$C^n$模数$998244353$。 自从$n$可以大到$10^{18}$，我们使用快速求幂来计算它。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(C^n \cdot n)$|$O(n)$| 太慢了|
 | 最佳 |$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 1.认识到有效字符串的个数相当于统计所有长度的字符串$n$超过$C$字母，因为嵌入约束不会消除任何构造。 这来自于 KMP 故障链接的嵌套属性。 
2. 将问题重写为计算$C^n \bmod 998244353$。 
3. 使用二进制求幂来有效地计算幂。 在每一步中，当移动到更高位时，对当前基数进行平方$n$，并在当前位乘以答案$n$已设置。 
4. 返回最终累加结果。 

正确性取决于每个字符串都会产生有效的非交叉结构，因此除了计算字符串的完整组合空间之外，没有任何过滤步骤。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def mod_pow(a, e):
    res = 1
    a %= MOD
    while e > 0:
        if e & 1:
            res = (res * a) % MOD
        a = (a * a) % MOD
        e >>= 1
    return res

n, C = map(int, input().split())
print(mod_pow(C, n))
```该实现将整个问题简化为模幂运算。 唯一值得关注的细节是$C$在开始求幂循环之前对常数模取模，因为$C$可以大到$10^9$。

def mod_pow(a, e):

 ## 边缘情况

 对于$n=0$，恰好有一个空串，并且其故障结构很简单，不会产生弧线，因此满足嵌入条件。 算法返回$C^0 = 1$，符合这个解释。 

什么时候$C=1$，每个字符串都被强制为单个字符的重复。 计数变为$1^n = 1$，并且故障结构再次微不足道，仅由嵌套的自边界组成，没有任何交叉的可能性。 

什么时候$n=1$，除了$1 \to 0$，因此该结构始终是单个弧并且不能交叉任何东西。 结果$C^1 = C$正确计算所有可能的单字符字符串。
