 ## 解决方案
 ## 问题理解

 我们有一棵树$N$顶点。 我们可以删除任何边的子集。 一旦这些边被移除，树就会分裂成连接的组件。 要求是每个生成的连接组件必须具有大小$A$或者$B$。 任务是计算有多少边缘子集产生有效的分解，并且答案取模$10^9+7$。 

结构很重要：由于图是一棵树，删除边总是会产生一个森林，并且每个组件本身就是一棵树。 因此，问题不在于顶点的任意划分，而在于切割边缘，以便每个结果子树的大小受到限制。 

约束推向线性或近线性解决方案$N$。 和$N$最多$10^5$，任何在节点或边上的二次方都会立即变得太慢。 甚至$O(N \sqrt{N})$根据常量，已经是可疑的，并且任何涉及边缘子集 DP 或每个节点指数指数的状态都是不可能的。 

一个微妙的点是，答案计算的是边集，而不是顶点分区。 仅当切割结构不同时，才能通过不同的边缘去除模式产生相同的顶点划分。 这使得计数组合在树的边缘上而不仅仅是在分区上。 

一种幼稚的方法会尝试将每条边视为已切割或未切割，即$2^{N-1}$。 那是完全不可行的。 另一个天真的想法是根树并尝试自下而上地决定子树是否可以划分为有效的组件，但如果不仔细处理部分子树大小的组合方式，这会导致过度计数或丢失配置，其中多个切割跨分支相互作用。 

星形树中出现了一个具体的故障案例。 认为$A=1, B=2$，中心连接许多叶子。 每个叶子可以是孤立的，也可以与中心配对，具体取决于其他选择，并且关于切割边缘的幼稚局部决策独立失败，因为叶子是否可以形成有效组件取决于切割了多少其他边缘。 

## 方法

 暴力方法是迭代所有边子集，删除它们，使用 DFS 或并查找计算连接的组件，并检查所有组件大小是否为$A$或者$B$。 计算组件需要$O(N)$，并且有$2^{N-1}$子集，所以总复杂度是$O(N \cdot 2^N)$，这远远超出了任何限制。 

关键的观察是，由于图是一棵树，因此每个有效的分解都对应于一种将节点分组为大小为连通子树的方法$A$或者$B$，并且每个这样的分组都可以自下而上构建。 我们不是直接选择边，而是处理有根树并计算部分子树大小如何根据是否放置切割来“向上流动”。 

主要困难在于每个节点必须决定其子节点如何有助于形成有效组件。 大小的子树$A$或者$B$可以在该节点处“闭合”，这意味着到父节点的所有边都被剪切，或者它可以向上贡献一个不完整的尺寸，必须稍后完成。 

自从$A$和$B$很小（最多 500），我们可以显式地将子树大小状态处理为$B$，因为任何尺寸大于$B$无关紧要，除非无效。 这使得 DP 中的每个节点都能够维护可以生成部分组件大小的多种方式$k$，并且转换将子状态结合起来，就像背包对子状态一样。 

我们的见解是，每个节点的子树都贡献了可能的“未完成大小”的多重集，并且我们在子节点之间合并这些多重集。 当尺寸正好达到$A$或者$B$，它可以作为一个有效的组件被关闭，并且没有任何向上的贡献。 这将问题转化为具有有界背包转移的树DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N \cdot 2^N)$|$O(N)$| 太慢了|
 | 边子集上的树DP | 指数| 指数| 太慢了|
 | 优化树 DP，大小状态高达$B$|$O(N \cdot B^2)$|$O(N \cdot B)$| 已接受 |

 ## 算法演练

 我们以节点 1 为树的根，并为每个节点定义一个 DP，描述其子树可以通过多少种方式生成给定大小且尚未闭合的部分形成组件。 

1.我们定义一个DP表`dp[v]`在哪里`dp[v][k]`是节点子树的路数$v$可以分解为$v$的当前连接部分（仍附加到其父级的部分）具有大小$k$。 这一块还可以向上延伸。 
2. 初始化叶节点：唯一的可能是大小为1的组件，所以`dp[v][1] = 1`。 这反映出单个节点是大小为 1 的部分组件。 
3、对孩子进行一一处理。 假设我们正在合并孩子$u$进入节点$v$。 我们取当前的 DP$v$并将其与 DP 结合$u$。 
4. 对于每个当前尺寸$i$在$v$，以及每个尺寸$j$从孩子$u$，我们考虑两种可能性。 我们要么切断两者之间的界限$v$和$u$，在这种情况下$u$的子树必须已经完全划分为有效组件并且不向上贡献任何大小，或者我们保留边缘，在这种情况下，部分组件大小组合并且$i + j$变为新尺寸$v$。 
5. 组合时，如果$i + j$等于$A$或者$B$，我们不会向上传播它； 相反，它形成一个完整的组件，并为计数贡献一个乘法因子 1。 如果$i + j < B$且不等于$A$，我们将其存储为新的部分状态。 
6. 我们还允许“立即切割”选项，这意味着我们乘以子子树可以独立完全分区的方式数量，这对应于对已经是有效闭包的子树的所有 DP 状态求和。 
7. 处理完所有子项后，`dp[v]`表示在以下位置形成部分组件的所有方法$v$。 对于根，我们只接受最终大小恰好为的配置$A$或者$B$，因为根不能向上传递任何东西。 

### 为什么它有效

 DP 保持严格的不变量：每个状态代表子树的已处理部分的有效分解，其中所有内部组件都已经完成，并且每个不完整的组件正是附加到当前节点的连接部分。 由于树没有循环，因此在父子合并中每条边都被考虑一次，并且每个有效的分解必须恰好对应于在每条边处切割或合并的一个选择。 有界大小约束确保显式跟踪所有部分配置，因此不会省略任何有效的构造，并且不会计算无效的部分合并。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    N, A, B = map(int, input().split())
    g = [[] for _ in range(N)]
    for _ in range(N - 1):
        x, y = map(int, input().split())
        x -= 1
        y -= 1
        g[x].append(y)
        g[y].append(x)

    sys.setrecursionlimit(10**7)

    # dp[v]: dict size -> ways
    def dfs(v, p):
        dp = [0] * (B + 1)
        dp[1] = 1

        for to in g[v]:
            if to == p:
                continue
            child = dfs(to, v)

            newdp = [0] * (B + 1)

            # option 1: cut edge, child must be fully valid (sum of finished states)
            child_sum = sum(child[s] for s in range(B + 1)) % MOD

            for i in range(1, B + 1):
                if dp[i] == 0:
                    continue

                # cut child edge
                newdp[i] = (newdp[i] + dp[i] * child_sum) % MOD

                # merge child
                for j in range(1, B + 1):
                    if child[j] == 0:
                        continue
                    if i + j > B:
                        continue
                    ways = dp[i] * child[j] % MOD
                    if i + j == A or i + j == B:
                        # closed component, do not carry upward
                        newdp[i] = (newdp[i] + ways) % MOD
                    else:
                        newdp[i + j] = (newdp[i + j] + ways) % MOD

            dp = newdp

        return dp

    dp_root = dfs(0, -1)
    print((dp_root[A] + dp_root[B]) % MOD)

if __name__ == "__main__":
    solve()
```该解决方案构建了一个基于 DFS 的树 DP，其中每个节点维护一个大小可达的有界数组$B$，因为没有部分分量大于$B$永远会有用。 每个合并步骤都考虑切割和合并一条边。 切割使用所有有效子完成的预先聚合总和，而合并明确跟踪部分大小如何组合。 

关键的实现细节是将“封闭子贡献”与“开放子状态”分开，因为它们在乘法中扮演不同的角色。 另一个微妙的点是确保状态超过$B$被尽早丢弃以保持转换有限。 

## 工作示例

 考虑一个由三个节点组成的简单链$A=1, B=2$。 

我们以节点 1 为根。 

| 节点| dp 之前 | 子合并 | dp 之后 |
 | ---| ---| ---| ---|
 | 叶 3 | [0,1,0]| 无 | [0,1,0]|
 | 节点 2 | [0,1,0]| 合并叶 3 | [0,0,1]|
 | 节点 1 | [0,0,1]| 合并节点 2 | [0,0,1]|

 最终结果计算有效配置，其中完整链形成尺寸 2 组件或拆分为尺寸 1 组件。 该轨迹显示部分大小如何向上移动直到达到有效的闭合。 

现在考虑一颗中心为 1 且叶子为 2、3、4 的星形，再次$A=1, B=2$。 

每片叶子开始于`[0,1,0]`。 在中心，每片叶子都可以被剪切或合并。 切割可以使每片叶子独立地倍增配置，而合并允许一次将中心与一片叶子配对。 DP 正确累积所有有效配对，而不会重复计数，因为每个边缘决策都是在合并步骤中独立处理的。

 ## 边缘情况

 一个关键的边缘情况是当$A=1$。 在这种情况下，每个单独的节点都可以独立地形成一个有效的组件。 该算法必须确保合并不会意外地强制进行不必要的分组。 DP 处理这个问题是因为每当大小达到 1 时，它就会立即被视为有效的闭包，因此不会携带超过 1 的状态，除非它来自组合节点。 

另一个边缘情况是当$B$相对于子树大小而言较大。 在小树中，许多 DP 条目保持为零。 该实现依赖于跳过零条目，这避免了不必要的$B^2$在实践中工作。 

最后的边缘情况是倾斜的树（路径）。 在这里，每次合并都是线性的，DP 本质上就像链上的背包。 正确性依赖于确保每个中间状态都保留足够长的时间以便与后面的节点结合，这是自下而上的 DFS 所保证的。
