 ## 解决方案
 ## 问题理解

 我们有一个垂直的 Jenga 塔，由$N$水平层，每层具有三个可能的块位置。 每个位置要么存在，要么缺失。 移动包括移除单个现有块，但前提是移除后塔仍然满足严格的稳定性条件。 

稳定性是每层局部的，有一个全局限制。 每一层仍必须至少包含一个块。 如果一层只包含一个块，则该块必须位于中间位置。 此外，最顶层始终需要在所有三个块都存在的情况下保持完整，这实际上意味着不允许任何移动从顶层移除块。 

两名玩家交替移动，不能移动的玩家失败。 我们被要求在最佳发挥下确定获胜者。 

输入数据量很大，最多可达$4 \cdot 10^5$所有测试用例的总层数。 这立即排除了对博弈树的任何模拟或重复的状态探索。 任何解决方案都必须将每个测试用例的游戏缩减为线性。 

一个微妙的点是，虽然塔是垂直的，但合法性约束几乎完全是每层的。 层之间唯一的耦合是固定的顶层约束，它阻止了涉及它的任何交互。 其他所有内容的行为都类似于在删除操作下演变的独立本地配置。 

一个常见的失败案例来自于假设删除一层中的块会在结构上影响其他层。 例如，在给定的规则下，认为从下层删除块可能会使上层无效是不正确的。 

另一个棘手的情况是处理单块层。 像这样的配置`010`是有效的，但是任何从中删除中间块的尝试都会产生无效的空层，因此即使它是稳定的，从游戏意义上来说它也是终端。 

## 方法

 直接的暴力方法会将每个有效的塔配置视为游戏状态并探索所有可能的块移除。 每个州将分支到$3N$下一个状态，并且状态数量呈指数增长，因为每一层都可以随着时间的推移独立变化。 即使对于中等$N$，这变得完全不可行。 

The key observation is that layers do not interact except through move legality, and each layer evolves independently based only on its own three cells. 一旦我们修复了一个层，唯一的问题是在不破坏该层稳定性的情况下允许进行哪些删除。 这将问题变成了一系列独立公正的博弈的总和，每层一个，顶层不做出任何贡献。 

因此，游戏简化为为非顶层的每个可能的 3 位配置计算 Grundy 值。 由于每层只有 3 个单元，因此我们可以显式枚举所有状态和转换。 一旦知道这些值，整个塔就是所有层值的异或。 

顶层很特殊，因为不允许在其上移动，因此它的贡献始终为零。

 ## 算法演练

 我们独立对待每一层并计算其对游戏价值的贡献。 

### 1.修复顶层

 顶层永久无法播放，因为任何删除都会违反其保持完整的要求。 所以我们在游戏分析中完全忽略它。 

这立即将问题减少到几层$2$通过$N$。 

### 2. 将单层建模为小游戏

 每层都是一个 3 位二进制字符串。 如果结果配置在单个块必须位于中间的规则下仍然有效，则移动会删除一个现有块。 

这仅给出了有效配置的小状态图。 

### 3. 枚举有效状态

 唯一有效的配置是那些从不包含无效单块状态的配置。 所以配置就像`100`或者`001`被禁止，同时`010`,`110`,`101`,`011`， 和`111`是允许的。 

我们现在计算这些状态之间的移动转换。 

### 4. 计算终结点和转换结构

 从这个结构来看：

 国家`010`没有有效的移动，因为删除其唯一的块会创建一个空层，这是禁止的。 所以它是终端。 

国家`101`也没有有效的移动，因为任何删除都会导致无效的单块配置。 

国家`110`只有一个有效的移动：移除左边的方块会导致`010`。 

国家`011`仅有一个有效的移动：移除正确的方块会导致`010`。 

国家`111`可以移动到`110`,`101`， 或者`011`。 

### 5. 计算 Grundy 值

 从终端状态向上：

 终端状态`010`和`101`Grundy 值为 0。 

然后`110`和`011`每个都只移动到 0 状态，所以他们的 Grundy 是 1。 

对于`111`，可达值是$\{1, 0, 1\}$，所以 mex 是 2。 

因此，每一层都贡献：`010 → 0`,`101 → 0`,`110 → 1`,`011 → 1`,`111 → 2`。 

### 6. 使用 XOR 组合层

 由于层是独立的游戏，因此完整的游戏值是除顶层之外的所有层 Grundy 值的异或。 

如果此 XOR 非零，则玩家 1 获胜。 

### 为什么它有效

 每层形成一个独立公正的游戏，其行动不会影响其他层。 全局博弈是这些博弈的不相交总和，因此斯普拉格-格伦迪理论直接适用。 稳定性约束确保没有隐藏的跨层依赖关系，顶层限制消除了唯一的潜在耦合。 由于每一层的状态空间大小恒定，因此其 Grundy 值是固定的且独立于其他层，使得 XOR 聚合有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def layer_value(s):
    # s is a string of length 3
    # compute its contribution directly
    if s == "111":
        return 2
    if s == "110" or s == "011":
        return 1
    return 0

def solve():
    T = int(input())
    for _ in range(T):
        N = int(input().strip())
        x = input().strip()
        
        total = 0
        
        # skip top layer (index 0)
        for i in range(1, N):
            total ^= layer_value(input().strip())
        
        print("Yesyes" if total != 0 else "Nono")

if __name__ == "__main__":
    solve()
```该解决方案读取每一层并跳过第一层，因为它是固定的且无法修改。 对于每个剩余层，它使用预先导出的从配置到值的映射来计算其 Grundy 贡献。 最终答案是通过对所有贡献进行异或来确定的。 

一个微妙的实现细节是我们不能对待`101`或者`010`作为逻辑中的特殊情况； 它们都正确映射到零，要么是因为它们是终端，要么是因为它们不能影响 XOR 和。

 ## 边缘情况

 一个关键的边缘情况是除顶部之外的所有层都`101`。 这些层中的每一层都是单独的终端，因此它们的贡献为零。 该算法正确地产生 XOR = 0 并宣布第二个玩家获胜。 任何假设每个非完整层都做出贡献的错误解决方案都会在这里失败。 

另一种情况是一座完全由`110`顶部以下的层。 每个人贡献 1，因此结果仅取决于奇偶性。 该算法通过异或累加自然地处理这个问题。 

最后，一座塔`111`层层展示了价值的积累 2. 由于$2 \oplus 2 = 0$，偶数个这样的层会相互抵消，只有奇数奇偶校验会影响结果。
