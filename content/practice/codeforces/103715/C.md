 ## 解决方案
 ## 问题理解

 我们得到了一份糖店名单。 每个商店都有一个初始价格，并且该价格每天都会增加一枚硬币。 所以如果一家商店的起价是`a[i]`，那么在第一天的成本`a[i]`，第 2 天花费`a[i] + 1`， 等等。 

每天，您都以固定的预算去购物`x`。 在那一天，你可以访问任何商店，但每个商店每天最多只能使用一次，并且你在不超出预算的情况下购买尽可能多的包。 第二天，价格再次上涨，您重复相同的过程。 

最终，所有商店都变得太贵，以至于无法在您的预算内购买一包。 任务是计算截至该时刻您在所有日子里设法购买的包总数。 

输入由多个独立的测试用例组成。 每个测试用例都会给出商店数量和您的每日预算，然后是所有商店的初始价格。 每个测试用例的输出是一个整数，表示整个过程中购买的包的总数。 

这些限制意味着所有测试用例中总共约有二十万家商店。 这就排除了任何日复一日迭代的模拟。 简单的日常模拟会太慢，因为该过程可持续长达`max(x - min(a[i]) + 1)`天，在最坏的情况下达到十亿次迭代。 

当一些商店的起步价已经高于预算时，就会出现微妙的边缘情况。 例如，如果`x = 5`和一家商店有`a[i] = 10`，那么它就永远无法使用并且贡献为零。 另一种情况是当所有`a[i]`等于`x`，这意味着每个商店只能在第一天购买，之后立即就无法购买。 

## 方法

 暴力解释每天模拟这个过程。 每天，我们都会重新计算每家商店的价格，并计算我们能买得起的数量。 如果有`n`商店和`D`天，这花费`O(nD)`运营。 自从`D`可以大到`x`，上升到`10^9`，这种方法立即是不可能的。 

关键的观察结果是，商店在几天内不会以任何有意义的方式相互互动。 每个商店都是独立发展的，商店的决定仅取决于其当前价格是否在预算之内。 与其考虑天数，我们可以倒过来问：对于一个固定的店铺来说，能维持多少天的价格？ 

对于有初始价格的商店`a[i]`，当天还算实惠`d`恰好在什么时候`a[i] + (d - 1) <= x`。 这个不等式可以直接求解，给出连续的有效天数范围。 这意味着每个商店对总答案的贡献都是独立的，我们可以将所有商店的贡献相加，而无需模拟时间。

 | --- | --- | --- | --- |
 | 蛮力（白天模拟）| O(n·x) | O(n·x) | O(1) | O(1) | 太慢了 |

 ## 算法演练

 1. 阅读`n`和`x`，然后读取数组`a`的初始价格。 我们独立处理每个测试用例，因为没有状态延续。 
2. 各店铺`i`，确定其可购买的时间。 我们要求`a[i] + (d - 1) <= x`，重新排列为`d <= x - a[i] + 1`。 这给出了商店可以使用的确切天数。 
3.如果`a[i] > x`，那么表达式`x - a[i] + 1`变为非正数，这意味着商店永远不会贡献任何购买。 在这种情况下，我们将其贡献视为零。 
4. 将所有商店的贡献相加。 由于每个商店每天都会独立贡献，并且我们总是尽可能购买，因此该金额等于所有天购买的包总数。 
5. 输出测试用例的最终总和。 

### 为什么它有效

 关键的特性是各个商店的独立性。 在任何一天，每家商店的行为都是确定的：要么负担得起，要么负担不起，而且这个条件仅取决于当天的指数及其初始价格。 由于预算每天都会重置，并且超出预算限制的商店之间不存在竞争，因此每个商店都会贡献固定的成功天数。 将这些独立贡献相加，将每次购买准确地计算一次，没有重叠或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, x = map(int, input().split())
        a = list(map(int, input().split()))
        
        ans = 0
        for v in a:
            if v <= x:
                ans += (x - v + 1)
        print(ans)

if __name__ == "__main__":
    solve()
```该方案直接实现了每店贡献公式。 唯一的微妙之处是通过检查来防止负面贡献`v <= x`。 如果没有这个，减法就会错误地为价格过高的商店添加负值。 

每个测试用例的循环都是线性的，并且不需要排序或辅助结构，因为每个元素都是独立处理的。

 ## 边缘情况

 一种极端情况是所有价格都超出预算。 对于像这样的输入`n = 3, x = 5, a = [10, 20, 30]`，每一项都满足`a[i] > x`，因此每个人的贡献为零。 该算法正确地跳过了所有值，因为`v <= x`查看。 

另一个边缘情况是商店完全按照预算开始。 为了`a[i] = x`，公式给出`x - x + 1 = 1`，这意味着它只贡献一次。 该算法在第二天变得无法负担之前正确计算单次购买。 

最后一个边缘情况是`a[i] = 1`和`x`很大。 在这种情况下，商店会做出贡献`x`购买量，相当于每天购买一次，直至价格达到限额。 该算法将其累加为简单的算术表达式，无需模拟。
