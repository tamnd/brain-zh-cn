 ## 解决方案
 ## 问题理解

 我们给定一个整数`N`。 考虑所有三元组`(A, B, C)`其中每个值都在范围内`[1, N]`。 

比利想要验证是否`A * B = C`，但他没有直接相乘，而是比较数字根：$$d(A \cdot B) = d(C)$$使用该财产$$d(xy) = d(d(x)d(y))$$他的算法有效地检查$$d(d(A)d(B)) = d(C)$$任务是计算有多少个三元组使比利的算法接受，即使真实的方程$$A \cdot B = C$$是假的。 

数字根的行为与模数完全相同`9`，除了`9`映射到`9`而不是`0`。 更正式地说，$$d(x)= \begin{cases} 9 & x \bmod 9 = 0 \\ x \bmod 9 & \text{otherwise} \end{cases}$$所以比利的算法只验证模数相等`9`，而不是真正的平等。 

约束条件`N ≤ 10^6`立即排除迭代所有三元组。 有`N^3`的可能性，这变成了`10^{18}`在最坏的情况下。 甚至检查所有对`(A, B)`已经太大了，因为有`10^{12}`对。 

有效的解决方案必须利用数字根的微小状态空间。 只有九种可能的数字根，这建议按数字根对数字进行分组，而不是使用单个值。 

一种微妙的边缘情况来自于多个`9`。 粗心的实现可能会使用`x % 9`直接作为数字根，产生`0`而不是`9`。 

例如，当`x = 18`：$$18 \bmod 9 = 0$$但$$d(18)=9$$如果我们错误地将数字根存储为`0`，计数逻辑中断，因为问题使用值`1..9`， 不是`0..8`。 

另一个容易犯的错误是忘记了三倍`A * B = C`确切地说，不能被计算在内。 比利的算法接受许多三元组，但其中一些实际上是正确的方程，必须排除。 

例如与`N = 4`, 三重`(1,2,2)`满足数字根条件和实方程。 它一定不会对答案有所贡献。 

第三个微妙点是`A * B`可能超过`N`。 这是完全允许的。 仅有的`A`,`B`， 和`C`仅限于`[1,N]`。 

例如与`N = 4`：$$(3,4,3)$$在数学上是无效的，因为`3*4=12`， 不是`3`，但比利接受了，因为$$d(12)=3$$这个三元组有助于找到答案。 

## 方法

 暴力解决方案很简单。 迭代所有三元组`(A,B,C)`，计算数字根，检查比利是否接受三元组，并检查真正的乘法是否失败。 

这是正确的，因为它直接遵循问题的定义。 问题在于复杂性：$$O(N^3)$$和`N = 10^6`，这变得完全不可能。 

我们可以通过观察固定的`(A,B)`, Billy 只关心 的数字根`C`。 而不是迭代所有`C`，我们可以数出有多少个数字`[1,N]`有一个匹配的数字根。 这将复杂性降低到`O(N^2)`。 

即使这样也太慢了。 在`N = 10^6`，我们仍然需要`10^{12}`迭代。 

关键的观察是比利的检查仅依赖于数字根。 只有九个可能的根，因此具有相同数字根的所有数字在他的算法下表现相同。 

认为：$$cnt[r]$$是整数的个数`[1,N]`其数字根等于`r`。 

然后对于每对根`(ra, rb)`：$$rc = d(ra \cdot rb)$$每个有根的数字`ra`可以与每个带根的数字配对`rb`，比利接受每一个`C`谁的根是`rc`。 

所以比利接受的三元组数是：$$cnt[ra] \cdot cnt[rb] \cdot cnt[rc]$$总结一切`9 × 9`根对给出了比利接受的三元组总数。 

现在我们必须删除实方程实际上为真的三元组。 

对于每对`(A,B)`，只有一个可能的正确值：$$C = A \cdot B$$如果`C ≤ N`，那么这个三元组在数学上是有效的，并且必须从答案中排除。 

所以我们最多减去乘积的乘法对的数量`N`：$$\sum_{A=1}^{N} \left\lfloor \frac{N}{A} \right\rfloor$$这是经典的除数求和函数。 

整个解决方案变得接近线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N3) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算有多少个数字`[1,N]`每个数字根来自`1`到`9`。 

每个由九个连续数字组成的块恰好贡献了每个数字根的一次出现。 
2. 将这些计数存储在数组中`cnt[1..9]`。 
3. 迭代每对数字根`(a,b)`。 

只有`81`这样的对，所以这部分是常数时间。 
4. 计算得到的数字根：$$c = d(a \cdot b)$$1.添加$$cnt[a] \cdot cnt[b] \cdot cnt[c]$$至总数。 

这计算了比利算法接受的所有三元组。 

1. 计算实际正确的乘法三元组的数量。 

对于每个`A`从`1`到`N`，正好有$$\left\lfloor \frac{N}{A} \right\rfloor$$的值`B`这样$$A \cdot B \le N$$每个这样的对都对应于一个有效的`C`。 

1. 从之前的总数中减去该数量。 
2. 输出结果。 

### 为什么它有效

 当数字根满足时，比利恰好接受三元组$$d(A \cdot B)=d(C)$$数字根仅依赖于余数模`9`，因此对于比利的测试来说，共享相同数字根的所有数字都是可以互换的。 

该算法按数字根对数字进行分组，并以组合方式对所有接受的组合进行计数。 这算比利接受的每个三元组恰好一次。 

在这些接受的三元组中，唯一不应该出现在最终答案中的是真正正确的方程。 每对`(A,B)`贡献一个正确的三重当且仅当`A*B ≤ N`。 减去所有这些对就完全删除了有效方程，只留下比利的错误。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def digital_root(x):
    return 9 if x % 9 == 0 else x % 9

def solve():
    n = int(input())

    cnt = [0] * 10

    for x in range(1, n + 1):
        cnt[digital_root(x)] += 1

    ans = 0

    for a in range(1, 10):
        for b in range(1, 10):
            c = digital_root(a * b)
            ans += cnt[a] * cnt[b] * cnt[c]

    correct = 0

    for a in range(1, n + 1):
        correct += n // a

    ans -= correct

    print(ans)

solve()
```第一部分计算有多少个数字属于每个数字根类别。 自从`N`只是`10^6`，直接循环已经足够快了。 

根上的嵌套循环很小。 仅有的`81`组合存在，因此贡献计算实际上是常数时间。 

减法阶段是最重要的概念步骤。 比利的算法接受错误和正确的方程。 我们只想要错误的。 对于每一个固定的`A`，值`B`满意：$$A \cdot B \le N$$正是：$$1,2,\dots,\left\lfloor \frac{N}{A} \right\rfloor$$每个这样的对都确定一个唯一的正确的`C`。 

经常导致错误的一个实现细节是数字根函数。 使用`x % 9`直接是不正确的，因为`9`必须映射到`9`， 不是`0`。 

另一个微妙之处是整数大小。 答案可能会变得非常大，大约`N^3`，但 Python 整数会自动处理这个问题。

 ## 边缘情况

 考虑：```
1
```唯一的三元组是`(1,1,1)`。 比利接受它，但它在数学上也是正确的：$$1 \cdot 1 = 1$$该算法计算一个接受的三元组，然后减去一个正确的方程，产生`0`。 

现在考虑：```
9
```的倍数`9`在这里变得至关重要。 

例如：$$(9,1,9)$$被接受是因为：$$d(9 \cdot 1)=9$$如果我们错误地表示了数字根`9`作为`0`，这个三元组将被错误地分组，并且答案将变得错误。 

已实施的`digital_root()`函数通过显式映射多个`9`到`9`。 

另一个微妙的情况是当产品超过`N`。 

为了`N = 4`：$$(3,4,3)$$比利接受它是因为：$$d(12)=3$$但真正的方程失败了，因为`12 ≠ 3`。 

该算法正确地包含了这个三元组，因为接受的三元组纯粹是通过数字根来计数的，与实际乘积是否保持在界限内无关。
