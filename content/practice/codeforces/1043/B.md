 ## 解决方案
 ## 问题理解

 我们给出了一系列前缀值`a[0], a[1], ..., a[n]`在哪里`a[0] = 0`。 该序列最初是从隐藏数组生成的`x`长度`k`，但该数组不再可用。 

生成规则很简单：每个下一个值`a[i]`通过取前面的前缀和形成`a[i-1]`并添加一个元素`x`，以循环方式选择。 具体来说，步骤中的附加值`i`仅取决于`x[(i-1) mod k]`。 这意味着增量序列`a`是周期性的`k`。 

我们被要求找到所有可能的值`k`这样存在一些整数数组`x`长度`k`这可能会产生给定的前缀和数组。 

输入尺寸`n`最多 1000，这立即表明`O(n^2)`解决方案是可以接受的。 任何尝试重建的解决方案`x`每个候选人独立`k`并验证一致性将会顺利通过。 子集或排列上的任何指数都是不必要的，因为结构已经完全由周期性约束确定。 

一个关键的微妙之处是`x`不限于正数或有界。 这消除了许多典型的前缀和约束，只留下等式结构作为决定因素。 

一种常见的失败案例来自于将条件误解为需要特定的构造`x`。 例如，假设`x[i] = a[i] - a[i-1]`直接仅在以下情况下有效`k = n`。 对于较小的`k`， 相同`x`值必须在多个位置重用，因此索引之间的一致性才是真正的要求。 

另一个微妙的边缘情况出现时`k = 1`。 在这种情况下，所有差异必须相同。 任何忘记显式检查这一点的实现通常都会错误地接受差异不恒定的数组。 

## 方法

 第一个自然的想法是尝试每个候选长度`k`并显式重建隐藏数组`x`。 如果我们假设`x[j] = a[j+1] - a[j]`对于第一个`k`位置，然后我们可以模拟完整数组的构造并将其与给定的进行比较`a`。 这是正确的，但如果通过重复模拟来实现则效率低下，因为对于每个`k`我们可以模拟最多`O(n)`步骤，导致`O(n^2)`总体方法。 

然而，我们可以通过转变视角来简化问题。 而不是从重构的角度思考`x`，我们看看差异的顺序`d[i] = a[i] - a[i-1]`。 这个序列正是以下的重复模式`x`。 那么问题就变成了：对于哪个`k`是具有周期的周期性差值数组`k`？ 

这将任务转变为直接的周期性检查。 对于固定的`k`, 每个位置`i`必须满足`d[i] = d[i-k]`。 如果这适用于所有有效的`i`， 然后`k`是有效的。 

蛮力方法会针对每个独立地检查此条件`k`，每次都重新计算比较。 由于每张支票都是`O(n)`，完整的解决方案是`O(n^2)`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（每 k 周期检查）| O(n^2) | O(n^2) | O(n) | 已接受 |
 | 最优（同样的思路，直接实现） | O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 我们通过测试每个可能的周期长度来解决问题`k`。 

1. 计算差异数组`d`， 在哪里`d[i] = a[i] - a[i-1]`。 这隔离了与隐藏数组对应的重复模式`x`。 一旦我们提取差异，就不再需要前缀和。 
2. 迭代所有可能的值`k`从`1`到`n`。 每个`k`被视为候选周期长度。 
3.对于固定的`k`，假设它是有效的并验证差异数组的一致性。 对于每个索引`i`开始于`k+1`到`n`，检查是否`d[i] == d[i-k]`。 
4. 如果发现任何不匹配，请拒绝此`k`立即地。 这确保我们只接受真正的周期结构。 
5. 如果所有比较都成功，则添加`k`到答案列表。 
6.检查完所有候选后，按升序输出收集到的有效值。 

### 为什么它有效

 关键的不变量是差异数组必须精确地重复每个`k`位置，如果它起源于长度-`k`循环数组`x`。 因为每个`d[i]`直接对应于`x[(i-1) mod k]`, 所有元素间隔相等`k`分开既是必要的也是充分的。 如果存在任何不匹配，则不会分配`x`可以调和矛盾，因为每个位置`x`将被迫采用两个不同的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = [0] + list(map(int, input().split()))

    d = [0] * (n + 1)
    for i in range(1, n + 1):
        d[i] = a[i] - a[i - 1]

    res = []

    for k in range(1, n + 1):
        ok = True
        for i in range(k + 1, n + 1):
            if d[i] != d[i - k]:
                ok = False
                break
        if ok:
            res.append(k)

    print(len(res))
    print(*res)

if __name__ == "__main__":
    solve()
```实现首先重建差异数组，以便循环结构变得明确。 这避免了在验证期间重复处理前缀和。 

核心循环尝试一切可能的方法`k`并通过将每个元素与之前的元素进行精确比较来检查周期性`k`其背后的位置。 尽早休息很重要，因为一旦发现矛盾，它可以防止不必要的工作。 

索引时必须小心：这里的差异数组是从 1 开始的，以匹配前缀索引`a`，这简化了关系`d[i] = a[i] - a[i-1]`。 

## 工作示例

 ### 示例 1

 输入：```
5
1 2 3 4 5
```这里`a = [0, 1, 2, 3, 4, 5]`和`d = [1, 1, 1, 1, 1]`。 

| k | 检查比较| 有效 |
 | --- | --- | --- |
 | 1 | 一切平等（简单地） | 是的 |
 | 2 | 所有 d[i] == d[i-2] | 是的 |
 | 3 | 一切平等| 是的 |
 | 4 | 一切平等| 是的 |
 | 5 | 一切平等| 是的 |

 所有值都有效，因为差值数组是常数，因此任何周期都有效。 

输出：```
5
1 2 3 4 5
```这证明了存在最大对称性并且每个候选周期都是有效的情况。 

### 示例 2

 输入：```
5
1 3 5 6 8
```我们计算`a = [0,1,3,5,6,8]`和`d = [1,2,2,1,2]`。 

| k | 关键比较| 有效 |
 | --- | --- | --- |
 | 1 | 2 ≠ 1 立即 | 没有|
 | 2 | d[3]=2 与 d[1]=1 不匹配 | 没有|
 | 3 | d[4]=1 == d[1], d[5]=2 == d[2] | 是的 |
 | 4 | 位置 5 不匹配 | 没有|
 | 5 | 基本一致 | 是的 |

 输出：```
2
3 5
```此示例显示了当差异模式部分重复时多个有效周期如何共存。

 ## 边缘情况

 最小输入，其中`n = 1`很重要，因为每个`k = 1`是非常有效的。 该算法自然地处理这个问题，因为没有比较会失败，所以`1`始终包含在内。 

当所有差异都相同时，每个`k`变得有效。 该算法正确接受所有值，因为每次比较`d[i] == d[i-k]`总是成立。 

对于交替或结构化图案，较小的`k`价值观可能会失败，而更大的价值观会成功。 定期检查正确地捕获了这一点，因为不匹配恰好出现在对齐中断的位置，并且拒绝立即发生，而不需要完全重建。
