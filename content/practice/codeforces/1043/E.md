 ## 解决方案
 ## 问题理解

 我们有一组参与者，每个参与者都由两个数字描述。 这两个数字代表参与者根据他们在双人训练竞赛中解决第一个任务还是第二个任务而做出的惩罚。 当两个参与者组成一个团队时，他们会以最小化团队总惩罚的方式分配两个问题。 

一对$(i, j)$，有两种可能的分配。 如果$i$解决第一个任务并且$j$解决第二个问题，成本是$x_i + y_j$。 如果我们交换角色，成本是$x_j + y_i$。 团队总是选择这两个选项中更便宜的一个。 

然而，并非所有配对都被允许。 我们还得到了一份禁止配对的列表，这些配对永远不会组成一个团队。 对于每个参与者，我们需要计算所有有效合作伙伴的最佳团队成本总和。 

输入规模很大，高达 300,000 个参与者和 300,000 个禁止对。 这立即排除了任何显式评估所有对的二次方法，因为$O(n^2)$操作将远远超出时间限制。 即使具有隐藏二次行为的方法（例如独立地重新计算每对的最小值）也会失败。 

一个微妙的困难是，一对的最佳分配仅取决于之间的比较$x_i + y_j$和$x_j + y_i$，可以重写为比较$x_i - y_i$和$x_j - y_j$。 这种减少是问题的关键结构。 

值得注意的边缘情况包括所有配对都被禁止的情况，其中每个人的答案都为零，或者不存在禁止的配对，这意味着每个参与者都与所有其他参与者配对。 另一个重要的情况是当两个参与者具有极大的正值或负值时； 如果不小心处理，简单求和可能会溢出 64 位整数。 

## 方法

 暴力方法考虑每个有效对$(i, j)$，计算两个可能分配的最小值，并将结果添加到两个参与者的总数中。 这很简单：对于每一对，我们评估一个恒定时间表达式，因此正确性是立即的。 然而，随着$n$最多 300,000 个，对的数量约为$n^2$，这大约是$10^{10}$的行动，远远超出了可行的限度。 

关键的观察结果是成本函数在重写后得到了简化。 定义$d_i = x_i - y_i$。 然后：

 如果$d_i \le d_j$，最优分配是$i \to \text{first}, j \to \text{second}$, 给予成本$x_i + y_j$。 否则，我们分配$j \to \text{first}, i \to \text{second}$, 给予成本$x_j + y_i$。 

这意味着该决定仅取决于排序$d_i$，而不是单独的原始值。 一旦我们对参与者进行排序$d_i$，所有交互都变得结构化：对于任何对，可以使用此排序上的前缀/后缀和来表示贡献。 

禁止边引入了复杂性，因为我们不能简单地假设所有对都存在。 相反，我们计算答案，就好像所有对都存在一样，然后减去禁止对的贡献。 完整图结构密集，但排除稀疏，这是包含排除与排序和前缀和相结合的经典设置。 

对于每个参与者，我们使用排序顺序来计算其对所有其他参与者的贡献。 然后，我们通过使用相同的公式显式减去对贡献来纠正禁止对。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 排序+前缀和+修正 |$O(n \log n + m)$|$O(n)$| 已接受 |

 ## 算法演练

 我们分两个阶段进行：计算完整的交互总和，然后删除禁止的交互。 

1. 计算$d_i = x_i - y_i$对于每个参与者。 该值决定排序和交换决策。 
2. 对参与者进行排序$d_i$。 这确保了对于任何一对$i < j$，我们知道$d_i \le d_j$，固定分配规则方向。 
3. 构建前缀和$x$和$y$按排序顺序。 这些允许快速计算给定索引之前或之后所有参与者的总和。 
4. 对于每个参与者$i$，假设所有对都被允许，计算其相对于所有其他的总贡献。 

对于之前的参与者$i$,$i$总是分配最佳配对中的第二个问题； 对于那些之后的人$i$，它被分配到第一个问题。 这直接来自于以下顺序$d$-价值观。 
5. 使用前缀和来累积贡献$O(1)$每个方向。 
6. 减去禁止对的贡献。 对于每个禁止对$(u, v)$，使用以下方法计算精确的配对成本：$$\min(x_u + y_v, x_v + y_u)$$并从两个参与者的总数中减去它。 
7. 按原顺序输出最终累加值。 

关键思想是排序将成对决策转换为一致的方向规则，而前缀和将全局对聚合转换为恒定时间查询。 

### 为什么它有效

 排序后$d_i = x_i - y_i$，每对都有一个固定的最佳方向：较小$d$始终承担最佳分配中的第一个任务。 这消除了成对决策的歧义，并将问题转化为根据位置计算贡献。 

每个参与者的总数是两个线性聚合的总和：所有较小元素和所有较大元素的贡献。 这些完全由前缀和捕获$x$和$y$。 禁止对是单独处理的，因为它们是稀疏的，并且每次校正仅依赖于已经全局确定的相同的局部比较规则。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    x = [0] * n
    y = [0] * n

    for i in range(n):
        x[i], y[i] = map(int, input().split())

    order = list(range(n))
    order.sort(key=lambda i: x[i] - y[i])

    pos = [0] * n
    for i, idx in enumerate(order):
        pos[idx] = i

    xs = [x[i] for i in order]
    ys = [y[i] for i in order]

    pref_x = [0] * (n + 1)
    pref_y = [0] * (n + 1)

    for i in range(n):
        pref_x[i + 1] = pref_x[i] + xs[i]
        pref_y[i + 1] = pref_y[i] + ys[i]

    ans = [0] * n

    for i in range(n):
        p = pos[i]

        left_count = p
        right_count = n - p - 1

        left_sum_y = pref_y[p]
        right_sum_x = pref_x[n] - pref_x[p + 1]

        ans[i] += left_sum_y + right_sum_x

    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1

        cost = min(x[u] + y[v], x[v] + y[u])
        ans[u] -= cost
        ans[v] -= cost

    print(*ans)

if __name__ == "__main__":
    solve()
```该实现首先将决策规则压缩为单个排序：$x_i - y_i$。 排序后，前缀和允许我们在恒定时间内计算参与者两侧所有元素的贡献。 贡献公式划分得很清楚：对于之前的所有参与者$i$，只有他们的$y$贡献； 为所有之后$i$，只有他们的$x$做出贡献。 

禁止对上的校正循环至关重要，因为这些边缘包含在全局聚合中。 每次移除都是针对两个端点对称完成的。 

一个常见的实施陷阱是混淆了哪一方的贡献$x$以及这有助于$y$。 排序顺序确保对于任何$i < j$，最优分配总是给出$i$第一个任务和$j$第二个。

 ## 边缘情况

 一种特殊情况是每一对都被禁止。 在这种情况下，累积阶段仍然计算完整的密集贡献，但每对都被删除一次。 每个参与者最终都为零，因为每个潜在的交互都被减去。 

另一种情况是当所有参与者都非常相似时$x - y$价值观。 然后排序不会将它们分开太多，但前缀逻辑仍然有效，因为相等不会破坏排序假设，因为可以任意放置关系而不会改变分配规则的正确性。 

第三种情况涉及极值，例如$x_i = 10^9$和$y_i = -10^9$。 配对贡献可达$2 \cdot 10^9$，并且对许多参与者求和需要 64 位整数。 使用 Python 可以避免溢出，但在 C++ 中这需要`long long`自始至终。
