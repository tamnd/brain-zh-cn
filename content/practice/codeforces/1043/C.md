 ## 解决方案
 ## 问题理解

 我们得到一个仅由字符组成的二进制字符串`a`和`b`。 我们以固定的顺序从左到右处理它的前缀，这意味着我们首先决定如何处理长度为 1 的前缀，然后是长度为 2 的前缀，依此类推，直到完整的字符串。 

对于每个前缀，我们可以保留它不变或反转它。 一旦做出前缀决策，它就会永久影响字符串的当前状态，并且下一个前缀决策将作用于已修改的字符串。 目标是选择要反转的前缀，以便最终结果字符串在字典顺序上尽可能最小。 

关键的微妙之处在于每个决策都会更改未来决策所依据的前缀。 因此，我们不是选择独立的操作，而是构建一系列转换，其中每个步骤都会影响下一步。 

字符串长度最多为 1000，因此 O(n^2) 模拟是可行的。 然而，像枚举前缀反转的所有子集这样的事情是不可能的，因为这将是指数级的。 

一个常见的失败案例是由于独立处理每个前缀而引起的。 例如，尝试仅根据原始字符串的前 i 个字符来决定前缀 i 会忽略先前的反转已经更改了前缀结构。 另一个陷阱是尝试贪婪的词典改进而不跟踪方向变化，这会导致状态模拟不一致。 

## 方法

 强力解释将尝试是否反转每个前缀的所有 2^n 个选择，模拟结果字符串，并按字典顺序选取最佳选项。 这是正确的，因为它探索了每个有效的操作序列。 然而，它立即变得不可行，因为 n = 1000 使得 2^1000 种组合无法枚举。 

关键的观察是我们实际上不需要在每一步维护完整的字符串。 相反，我们只关心从左到右读取时每个前缀决策如何影响字符的相对顺序。 

关键的见解是，反转前缀会翻转前 i 个字符的顺序，这相当于以在概念上的双端队列结构的前面或后面交替附加的方式插入字符。 我们可以考虑最终构造的行为方式，而不是显式模拟反转：每个字符最终会以向前或反向的方向插入，具体取决于其前缀被翻转的次数。 

这导致了贪婪的构造：我们从左到右处理字符串，同时保持动态构建的结果。 在每一步中，我们都会决定对这个位置应用反转是否会改善最终的词典顺序。 因为字母表只是`{a, b}`，决策简化为当前有效字符是否比翻转前缀状态时得到的字符更差。 

这可以通过维护类似双端队列的结构和布尔翻转状态来实现，该状态告诉我们当前前缀是否在逻辑上反转。 每个新字符根据奇偶校验添加到左侧或右侧。 然后通过比较哪一侧在翻转或不翻转时产生按字典顺序排列更小的延续来确定最终输出。 

我们通过跟踪方向而不是实际的重新排列来避免模拟完整的字符串反转。

 ## 算法演练

 我们从左到右处理字符串，同时维护表示当前转换前缀的逻辑结构。 

1. 初始化一个双端队列来存储结果字符和一个布尔标志`rev`设置为假。 该标志表示当前前缀是否相对于原始方向在逻辑上反转。 
2. 迭代字符串中从索引 0 到 n − 1 的字符。在每一步 i，我们都决定是否对前缀 i + 1 应用反转。 
3. 确定新字符在当前方向下的插入位置。 如果`rev`为假，我们追加到右端； 如果为 true，我们将附加到左端。 这可以模拟过去反转的效果，而无需重建完整的字符串。 
4. 决定反转当前前缀是否会产生字典顺序更小的中间状态。 这相当于比较两个可能的插入中哪一个会导致当前结构中的前导字符更小。 因为只有`a`和`b`存在，这减少了检查将角色放置在另一端是否会产生更好的立即排序。 
5. 如果反转有益，则切换`rev`并记录`1`对于这个前缀。 否则，保持方向不变并记录`0`。 
6.处理完所有字符后，输出记录的决策。 

关键的不变量是，在处理前缀 i 后，双端队列准确地表示应用所有选定的反转直到 i 后的结果序列，并且`rev`正确描述当前方向是否相对于原始顺序翻转。 步骤 i 中的任何决策仅取决于该结构的当前边界行为，因此早期的正确性保证仍然有效。 

该算法之所以有效，是因为每次反转仅影响前缀内的相对顺序，并且未来的决策仅取决于所导出的顺序，而不取决于任何隐藏的历史结构。 由于我们始终保持准确的当前前缀状态，因此贪婪的局部改进与全局词典最小化保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    from collections import deque

    dq = deque()
    rev = False
    ans = []

    for i, ch in enumerate(s):
        # If not reversed, we would append to right
        # If reversed, we would append to left
        if not rev:
            left_option = ch
            right_option = ch
        else:
            left_option = ch
            right_option = ch

        # We decide whether flipping prefix helps.
        # Since alphabet is binary, we compare direct effect:
        # flipping changes insertion side effectively.

        # Simulate no flip: append at current end
        if not rev:
            dq.append(ch)
        else:
            dq.appendleft(ch)

        # Try to decide flip: compare ends
        # We only need local decision based on ends
        if len(dq) == 1:
            ans.append(0)
            continue

        # Compare front vs back lexicographically influence
        if dq[0] > dq[-1]:
            # better to flip orientation
            rev = not rev
            ans[-1] = 1 if len(ans) > 0 else 1
            ans.append(1)
        else:
            ans.append(0)

    print(*ans)

if __name__ == "__main__":
    solve()
```该代码维护一个双端队列，表示在所选操作下不断演变的前缀。 每个字符都插入在前面或后面，具体取决于当前前缀方向是否反转。 插入后，我们比较双端队列的两端，作为翻转是否会产生按字典顺序更小的结构的代理。 

这`rev`flag是中心状态变量，编码当前构造是否逻辑反转。 答案数组记录每个前缀是否被翻转。 

一个微妙的一点是，只有极端的相对顺序对于决定翻转很重要，因为内部结构总是早期决策的连续转变。 该实现依赖于这个边界属性而不是完整的字符串模拟。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import deque

    s = input().strip()
    n = len(s)

    dq = deque()
    rev = False
    ans = []

    for i, ch in enumerate(s):
        if not rev:
            dq.append(ch)
        else:
            dq.appendleft(ch)

        if len(dq) > 1 and dq[0] > dq[-1]:
            rev = not rev
            ans.append(1)
        else:
            ans.append(0)

    return " ".join(map(str, ans))

 ## 边缘情况

 对于像这样的单个字符`b`，算法处理一个步骤并立即将其附加，而不进行任何有意义的比较。 双端队列仅包含一个元素，因此不会触发反转，输出为`0`，它匹配唯一有效的操作序列。 

对于像这样的统一字符串`aaaa`，每次插入都保持双端队列对称。 在每一步中，两端都保持相等，因此永远不会触发翻转条件。 这证实了该算法在所有配置等效的退化情况下不会引入不必要的逆转。 

对于交替模式，例如`abab`，早期决策可能不明确，因为双端队列的两端在初始插入后保持相似。 该算法在每一步本地解决这个问题，产生一个有效的序列，实现字典顺序最小的最终排列，即使存在多个最优序列。
