 ## 解决方案
 ## 问题理解

 我们得到一个由大写字母组成的字符串，但仅从第一个字母开始$k$字母表中的字母。 任务不是重新排列或修改字符串，而是在保留顺序的同时选择字符子序列。 在所有这些子序列中，我们想要一个具有特殊属性的子序列：来自`'A'`到$k$-th 字母在子序列中出现的次数必须相同。 目标是最大化此类子序列的长度。 

另一种思考方式是我们试图选择一些数字$x$的每一个$k$字符，因此总长度为$k \cdot x$，并且此选择必须可以作为原始字符串的子序列。 

约束条件$n \le 10^5$立即告诉我们任何比$O(n \cdot k)$或者$O(n \log n)$会开始挣扎。 对所有子序列进行暴力破解是不可能的，因为有$2^n$其中。 甚至尝试所有可能的值$x$重复扫描仍需要仔细优化。 

一些边缘情况很重要。 

如果某个字母完全丢失，则没有有效的子序列可以包含它，因此答案必须为零。 例如，如果$k = 3$但该字符串仅包含`'A'`和`'B'`，那么我们永远无法平衡`'C'`，所以结果是$0$。 

另一个微妙的情况是频率高度不均匀时。 认为$k = 3$字符串是`"AAAAABBBBBCC"`。 即使每个字母都存在，限制因素将是三个字母中出现频率最小的一个。 

一个天真的错误是认为我们需要仔细选择头寸以最大化点差。 实际上，顺序与计数可行性无关，因为子序列保留顺序但不会将计数限制在可用性之外。 

## 方法

 一个蛮力的想法是尝试每一个可能的子序列并检查它是否平衡。 对于每个子序列，我们计算每个字母的出现次数并验证相等性。 这是正确的，但完全不可行：有$2^n$子序列，甚至检查一个$O(n)$，导致指数时间。 

更好的方向是重构问题。 我们不明确选择子序列，而是关注每个字符可以出现的次数。 假设我们决定每个$k$字母准确显示$x$次。 那么总长度是$k \cdot x$，问题就变成：是否可以选择$x$字符串中每个字符的出现次数，同时尊重顺序？ 

但顺序实际上对于这里的可行性并不重要，因为子序列只需要足够的出现次数，而不是连续的结构。 如果存在的话，我们总是可以按扫描顺序贪婪地选择事件。 

因此，真正的约束分解为一个简单的计数条件：对于每个字母，我们必须至少有$x$发生。 如果$cnt[c]$是字母的频率$c$， 然后$x \le \min cnt[c]$。 我们能做的最好的就是设定$x$到该最小频率。 

所以答案很简单：$$k \cdot \min_{c \in [A, A+k-1]} cnt[c]$$关键的见解是，子序列排序不会引入超出可用性之外的额外约束，因此问题变成了纯粹的频率最小化。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解子序列 |$O(2^n \cdot n)$|$O(n)$| 太慢了|
 | 频率最低 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 数一下第一个每项出现了多少次$k$字母出现在字符串中。 这是可行的，因为子序列只能使用现有字符，因此可用性是唯一的约束。 
2. 找出其中最小的频率$k$很重要。 该值表示我们可以在所有字母上形成的完整“平衡层”的最大数量。 
3. 将此最小值乘以$k$，因为每一层都贡献了每个字母中的一个。 
4. 输出结果。 

### 为什么它有效

 每个有效子序列必须包含相同数量的所有$k$字母，说$x$每一个。 这立即意味着任何字母的贡献都不能超过其在原始字符串中的总出现次数，因此$x \le cnt[c]$为所有人$c$。 所以$x \le \min cnt[c]$。 相反，如果我们选择$x = \min cnt[c]$，我们总是可以从字符串中按顺序选取每个字符的多次出现，因为子序列只需要选择位置，而不需要重新排列或尊重邻接性。 这确立了最优性和可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
s = input().strip()

cnt = [0] * k

for ch in s:
    cnt[ord(ch) - ord('A')] += 1

ans = min(cnt) * k
print(ans)
```实施直接遵循减少计数。 数组`cnt`跟踪第一个的频率$k$字母。 我们计算所有这些频率的最小值并乘以$k$。 

一个微妙的点是索引：因为字母保证位于第一个字母内$k$, 映射`'A'`索引 0 是安全的。 不需要条件检查。 

乘以$k$仅在取最小值后发生； 颠倒顺序是不正确的，因为我们不是对独立最大值求和，而是强制所有字母相等。 

## 工作示例

 ### 示例 1

 输入：```
9 3
ACAABCCAB
```计数演变如下：

 | 信| 计数 |
 | --- | --- |
 | 一个 | 4 |
 | 乙| 2 |
 | C | 3 |

 最小频率为$2$，所以我们可以形成 2 个完整层`A`,`B`,`C`，给出总长度$3 \cdot 2 = 6$。 

这表明尽管`'A'`和`'C'`更频繁，答案完全受到限制`'B'`。 

### 示例 2

 输入：```
5 3
ABCDE
```只有前三个字母是相关的，但是`'C'`根据输入边界的解释，可能会丢失。 如果任何必需的字母出现次数为零：

 | 信| 计数 |
 | --- | --- |
 | 一个 | 1 |
 | 乙| 1 |
 | C | 0 |

 最小值为 0，所以答案为 0。 

这证实了一个缺失的字符会导致整个结构崩溃。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(n)$| 单次计算频率并扫描 k 个字母 |
 | 空间|$O(1)$| 长度最多为 26 的固定大小数组 |

 该解决方案与字符串长度呈线性关系，这是最佳的，因为每个字符必须至少读取一次。 由于字母大小有限，内存使用量是恒定的。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n, k = map(int, input().split())
    s = input().strip()

    cnt = [0] * k
    for ch in s:
        cnt[ord(ch) - ord('A')] += 1

    return str(min(cnt) * k)

# provided sample
assert run("9 3\nACAABCCAB\n") == "6"

# k = 1, always full string
assert run("5 1\nAAAAA\n") == "5"

# missing character forces zero
assert run("5 3\nAABBB\n") == "0"

# all equal frequencies
assert run("6 2\nAABBAB\n") == "4"

# large balanced case
assert run("4 2\nBABA\n") == "4"
```| 测试输入| 预期产出 | 它验证了什么 |
 | --- | --- | --- |
 | 9 3 ACAABCCAB | 6 | 带有限制字符的正常情况 |
 | 5 1 AAAAA | 5 | k=1 边缘情况 |
 | 5 3 AABB | 0 | 失踪的信|
 | 6 2 AABBAB | 4 | 平衡频率|
 | 4 2 阿里巴巴 | 4 | 交替结构的正确性 |

 ## 边缘情况

 如果一个字母从未出现，则最小频率变为零，并且算法会正确返回零，因为没有平衡子序列可以包含该字母。 

对于具体的跟踪，请采用：```
4 3
ABAB
```计数：

 | 一个 | 乙| C |
 | --- | --- | --- |
 | 2 | 2 | 0 |

 最小值为 0，因此输出为 0。该算法不会尝试“跳过”缺失的字母，这符合所有第一个字母的要求$k$字母必须相同地出现。 

另一个边缘情况是当$k = 1$。 为了：```
5 1
ABCDE
```仅有的`'A'`考虑到，计数为 1，因此答案为 1。该算法可以正确归约，因为单个值的最小值是其本身，并且乘以 1 可以保留该值。
