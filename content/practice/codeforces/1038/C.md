 ## 解决方案

 ## 问题理解

 两个玩家各自持有一组整数。 他们轮流进行，从 A 开始。在一个回合中，玩家要么消耗自己剩余的数字之一，将其添加到自己的分数中，要么丢弃对手收集的一个数字，这不会直接影响分数，但会减少对手以后可以获得的分数。 当两个多重集合都为空时，游戏结束，并且假设两个玩家总是做出使 A 的总得分和 B 的总得分之间的最终差异最大化的决策。 

困难来自于直接获得与否定之间的相互作用。 你自己的列表中的高值很有价值，但对手列表中的高值也很有价值，因为它可以防止他们稍后得分。 

约束很大，n 高达 100000，因此任何模拟所有可能的移动序列或探索游戏状态的解决方案都是不可能的。 即使 O(n²) 推理也太慢，因为每次移动都会重复影响两个列表，导致二次行为。 

一个天真的但诱人的想法是认为每个玩家都会对自己的列表进行排序并始终获取最大的剩余值。 这会失败，因为它忽略了从对手身上移除非常大的值的可能性。 例如，如果 A 的数字很小，而 B 的数字很大，A 可能更愿意删除那个巨大的数字，而不是获取自己的小收益。 除非我们将两个列表一起建模，否则这种交互会使纯粹的局部贪婪选择变得不可靠。 

当两个玩家拥有相同的棋盘时，就会出现微妙的边缘情况。 例如，如果 A 和 B 都有 [1, 100]，对称性表明结果应该平衡，并且实际上最佳游戏会导致零差异。 一个幼稚的“总是获取最大自身价值”的策略会错误地在不考虑相互阻塞的情况下尽早优先考虑 100，并且会高估先走的优势。 

当一个玩家有许多大值而另一个玩家有许多小值时，就会出现另一种边缘情况。 即使一方“拥有”更多的总金额，积极的清除也可以通过优先拒绝而不是收集来抵消这一优势。 

## 方法

 暴力解释是将游戏模拟为对两个多重集和一个回合指示器的状态搜索。 在每种状态下，玩家都可以选择获取自己的元素之一或从对手身上移除一个元素。 这会创建一个与列表大小成比例的分支因子，并且由于每个操作都会将元素总数减少一，因此在最坏的情况下状态总数会呈阶乘增长。 即使使用记忆化，状态空间也取决于剩余元素的所有子集，这远远超出了可行的限制。 

关键的观察结果是，游戏完全由两个列表中元素的相对顺序决定。 在任何时候，只有最大的剩余值才重要，因为每个最佳操作都将涉及与任一列表中当前最大可用元素进行交互。 当存在较大的有利移动时，永远不会选择较小的元素，因为两个玩家都在最大化最终分数差异，并且任何偏差都将由涉及较大值的移动主导。 

这将问题简化为重复比较 A 的剩余元素和 B 的剩余元素的当前最大值。 无论谁行动，要么采取两者中较大的一个，因为它要么提供直接收益，要么阻止对手稍后获得更多收益。 这导致了对排序数组的两指针式贪婪过程。

我们按降序对两个数组进行排序，并使用指针模拟该过程，始终考虑每一侧当前最大的剩余候选者。 每一步要么消耗其所有者的一个元素，要么删除对手的一个元素，具体取决于哪一方当前拥有较大的值。

 ## 算法演练

 我们维护两个排序数组，一个用于 A，一个用于 B，均按降序排列。 我们跟踪每个数组的两个索引，表示当前最大的未使用元素。 

我们还模拟转弯，从 A 开始。 

1. 将两个数组按降序排序。 这确保我们始终使用指针在每步 O(1) 时间内知道每一侧的最佳候选者。 
2. 将 A 和 B 的两个指针 i 和 j 分别初始化为 0。 这些代表每个列表中最大的剩余元素。 
3. 维持分数scoreA 和scoreB，两者最初都为零。 
4. 轮到 A 时，比较 A[i] 和 B[j]（如果两者都存在）。 如果 A[i] 大于或等于，A 取 A[i]，将其添加到 ScoreA，并递增 i。 否则，A 删除 B[j] 而不会获得分数，因此 j 递增。 
5. 轮到 B 执行对称运算：如果 B[j] 大于或等于，B 将 B[j] 添加到 ScoreB，并递增 j。 否则，B 删除 A[i]，因此 i 递增。 
6. 交替转动，直到两个指针都到达数组末尾。 

决策规则始终基于最大的可用值，因为任何较小的值在立即增益和拒绝影响方面都受到严格支配。 如果玩家忽略最大可用选项，他们就会允许对手首先利用它，从而减少最终的差异。 

### 为什么它有效

 在每一步，游戏状态都可以通过两个列表中最大的剩余元素来概括。 任何最佳的移动都必须涉及这两个元素之一，因为所有较小的元素在增益和阻塞潜力方面都是严格较差的选择。 由于每次移动都会从全局池中删除一个元素，因此该过程减少为重复解析哪一方“控制”当前最大值。 这个不变量保证了没有最优策略需要超越当前的两个最大值，因此贪婪模拟与最优策略相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    a.sort(reverse=True)
    b.sort(reverse=True)
    
    i = j = 0
    scoreA = 0
    scoreB = 0
    turnA = True
    
    while i < n or j < n:
        a_top = a[i] if i < n else -1
        b_top = b[j] if j < n else -1
        
        if turnA:
            if i < n and (j == n or a_top >= b_top):
                scoreA += a_top
                i += 1
            else:
                j += 1
        else:
            if j < n and (i == n or b_top >= a_top):
                scoreB += b_top
                j += 1
            else:
                i += 1
        
        turnA = not turnA
    
    print(scoreA - scoreB)

if __name__ == "__main__":
    solve()
```该解决方案首先对两个列表进行排序，以便我们始终可以访问剩余的最佳选项。 这两个指针模拟组合多重集的收缩。 每一步都强制执行最佳局部决策：要么获取最佳的个人利益，要么消除对手的最大威胁。 

交替回合逻辑确保决策的公平性，而比较步骤则确保在同一规则中处理增益最大化和对手拒绝。 

一个常见的实现陷阱是忘记了从对手中移除并不取决于谁的回合，除了决定哪个指针移动之外。 另一个微妙的问题是不正确地处理相等性，不一致地处理相等的值可能会在元素重复的情况下破坏对称性。

 ## 边缘情况

 当两个数组包含相同的多重集时，算法会交替执行完全对称的操作。 一个列表中的每个高值元素都会镜像到另一个列表中，因此每个增益最终都会被另一侧的等效增益抵消。 基于指针的模拟确保始终以一致的顺序处理相同的最大值，从而产生零最终差异。 

当一个数组包含单个非常大的元素而另一个数组包含许多小元素时，大元素会立即被消耗以在早期回合中进行阻塞。 该算法正确地优先考虑删除而不是自我评分，确保大值不会错误地主导最终结果。 

当所有元素都相等时，A 和 B 顶部之间的每次比较都会解析为相等，因此平局打破规则始终会导致对称结果。 该算法均匀地交替去除和增益，从而保持两个玩家的平衡。
