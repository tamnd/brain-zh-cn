 ## 解决方案
 ## 问题理解

 每个块可以被视为两种颜色之间的加权无向边，其中每个端点是两种可能颜色之一，具体取决于方向。 选择方向相当于确定路径中该边的方向，因为一旦我们固定了序列中块的顺序，每个相邻的对都会强制接触的颜色完全匹配。 

任务不是排列所有块，而是选择一个子集，然后将它们排列成单个链，以便相邻块以相同的颜色连接，同时最大化所选块值的总和。 每个块最多使用一次，翻转块只是交换其端点，而不会改变其值。 

颜色空间很小，只有四种颜色，但块的数量高达 100 个，因此真正的困难是组合：决定哪个子集可以链接以及如何通过颜色图路由该链以累积最大权重。 

关键的结构约束是任何有效序列在最多具有四个顶点的多重图中形成一条轨迹。 每个块都是一条边，序列是最多使用一次边的行走。 

当局部思考时，就会出现幼稚的故障模式。 例如，总是贪婪地附加最高值的兼容块可能会提前捕获构造，即使早期选择稍差的不同顺序会在以后解锁更多高值边缘。 另一个微妙的失败是假设结构始终是一条路径：最佳解决方案可能会多次重新访问颜色，但边缘仍然不会被重用。 

因为颜色只有 1 到 4，所以连接状态空间足够小，我们可以显式跟踪部分解决方案如何连接端点。 

## 方法

 强力解释是尝试块的每个子集以及两个方向的每个排序，然后检查它是否形成有效的链。 即使没有排序，只需选择子集即可$2^n$，并且排列它们会增加阶乘的复杂性。 这在以下位置立即不可行$n=100$。 

真正的简化来自于注意到部分构造唯一重要的是哪些颜色当前在链的末端“开放”。 由于只有四种颜色，状态可以通过端点来描述，我们本质上是使用带有权重的边在 4 节点图上构建一个连接结构。 

我们不再考虑序列，而是将问题重新解释为构建一个连接的子图，其中所有选定的边都位于一条路径中。 可以通过确保最多两个顶点在所选边的多重集中具有奇数度来强制执行轨迹约束。 但是，由于我们不需要以特定颜色开始或结束，因此我们可以允许任何端点对。 

这导致对边缘引起的颜色子集进行动态编程。 解决此问题的标准方法是将图压缩为 4 个节点并考虑颜色对之间的所有边。 对于每对颜色，我们按权重对边缘进行排序，并考虑取每对颜色之间的前 k 个边缘。 然后我们在四个节点的度奇偶校验状态上运行 DP。 

更直接和标准的解决方案是将其解释为选择在多重图中形成连接的欧拉轨迹的多重边集。 由于 4 个节点的连通性微不足道，因此约束简化为奇偶校验：奇数度顶点的数量必须为 0 或 2。我们根据此约束最大化总权重。 

我们将 dp[mask] 定义为所选边的最大总权重，使得奇数度的顶点集等于 mask。 每个边都会翻转其两个端点的奇偶校验，因此转换是简单的 XOR 更新。 

我们一条一条地处理边并更新 DP，但由于我们可以选择或跳过每条边，而且 n 只有 100，所以这已经足够了。

 ## 算法演练

 我们将每个块视为两种颜色之间的无向边，忽略方向，因为翻转仅交换端点。 

1. 构建一个边列表，其中每条边连接两种颜色并带有权重。 这将问题转化为选择 4 个顶点的加权多重边集。 
2. 初始化大小为 16 的 DP 数组，其中每个索引代表 4 种颜色的奇偶校验掩码。 如果对应的颜色当前在所选边缘子集中具有奇数度，则位为 1。 
3. 设置 dp[0] = 0，这意味着没有选择的边不会给出奇数度数和零值。 所有其他状态都初始化为负无穷大，因为它们尚未到达。 
4. 迭代每条边。 对于连接 u 和 v 和权重 w 的每条边，我们考虑两种选择：跳过它或接受它。 如果我们接受它，我们通过切换位 u 和 v 来更新奇偶校验，并将 w 添加到总值中。 
5. 对于每条边，接下来计算一个新的 DP 数组，最初是从 dp 复制的，表示跳过该边的选项。 然后对于每个状态掩码，转换到掩码 XOR (1<<u) XOR (1<<v) 并添加权重。 
6. 处理完每条边后，将 dp 替换为 next。 这确保了每条边最多使用一次。 
7. 处理完所有边后，答案是所有掩码上的最大 dp[mask]，其中 mask 设置了 0 或 2 位，因为它们对应于有效的欧拉轨迹。 

### 为什么它有效

 任何有效序列都对应于选择形成轨迹的边，这意味着除了可能的两个顶点之外的所有顶点都具有偶数度。 DP 在每个选定边缘之后显式跟踪度奇偶校验，并且每个转换都保留正确性，因为翻转端点与度奇偶校验更新完全匹配。 由于边的每个子集都可以通过某种选择序列来表示，因此 DP 涵盖了所有有效的构造。 最大化有效奇偶校验掩码可确保我们包含循环和开放路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def popcount(x):
    return bin(x).count("1")

def solve():
    n = int(input())
    edges = []
    
    for _ in range(n):
        a, w, b = map(int, input().split())
        u = a - 1
        v = b - 1
        edges.append((u, v, w))
    
    INF = -10**18
    dp = [INF] * 16
    dp[0] = 0
    
    for u, v, w in edges:
        ndp = dp[:]  # skip edge
        bit = (1 << u) ^ (1 << v)
        for mask in range(16):
            if dp[mask] == INF:
                continue
            nm = mask ^ bit
            ndp[nm] = max(ndp[nm], dp[mask] + w)
        dp = ndp
    
    ans = 0
    for mask in range(16):
        if dp[mask] == INF:
            continue
        if popcount(mask) in (0, 2):
            ans = max(ans, dp[mask])
    
    print(ans)

if __name__ == "__main__":
    solve()
```DP阵列在处理边缘前缀后对所有可达奇偶校验配置进行编码。 转换步骤要么忽略边，要么包含它，并且包含会翻转其端点处的奇偶校验，这正是向图添加边的效果。 

最后的过滤步骤强制执行欧拉路径条件。 具有零个奇数顶点的掩模对应于循环，而具有两个奇数顶点的掩模对应于开放路径。 

一个微妙的实现点是我们必须在转换之前将 dp 复制到 ndp 中； 否则更新将在单次迭代中多次重复使用相同的边。

 ## 边缘情况

 一个块的最小输入被正确处理，因为初始 DP 已经允许选择无边沿或单个边沿，并且单个边沿始终形成有效序列。 

对于不相交的颜色对，例如边缘 (1,2) 和 (3,4)，DP 独立地保留两个贡献，并且最终答案正确地对它们进行求和，因为它们在奇偶校验转换中不会相互作用。 

处理循环重的输入是因为奇数度为零的掩码自然地表示闭合循环，并且 DP 允许在不需要端点的情况下形成它们，从而确保保留完整的循环权重。 

最微妙的情况是必须暂时跳过高值边缘以启用更大的链。 DP 不会贪婪地提交，因此跳过和获取都是并行探索的，并且最大值保留在最终状态中。
