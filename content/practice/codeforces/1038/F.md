 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串`s`和一个长度`n`。 我们想要构造所有二进制字符串`t`长度`n`。 每一个这样的`t`被视为循环，这意味着它的末端回绕到它的开头。 

一根圆形的绳子`t`如果我们可以在该周期中的某个位置找到，则被认为是有效的`s`作为一个连续的段。 同样，如果我们开始阅读`t`从某个位置开始并继续环绕，我们可以匹配整个字符串`s`。 

任务是统计有多少个不同的二进制字符串`t`满足这个条件。 

关键的难点是发生`s`允许环绕结束`t`，所以检查子串是不够的`t`在通常的线性意义上。 

限制很小：`n ≤ 40`和`|s| ≤ n`。 这立即表明指数或状态压缩动态规划是可行的，因为字符串的总数是`2^40`，任何解决方案都必须避免直接枚举它们。 

一种生成所有内容的天真的方法`2^n`字符串和检查每一个都已经是边界，但在概念上仍然是正确的。 然而，检查每个字符串的循环出现会花费成本`O(n * |s|)`，粗略地得出完整的解决方案`O(n * |s| * 2^n)`，这太慢了。 

一个更微妙的问题是处理环绕匹配。 粗心的实现可能只会检查`t + t`直到长度`n`，但错误处理边界处的重叠或跨越两个副本之间的分割的双重计数事件。 

正确的解决方案必须仔细模拟模式匹配在圆形结构上的行为方式，同时仍然计算线性字符串。 

## 方法

 暴力破解的想法很简单：生成每个二进制字符串`t`长度`n`，然后检查是否`s`出现在循环版本中`t`。 圆形检查可以通过形成来模拟`t + t`并寻找`s`在所有有效的起始位置。 这是正确的，因为任何环绕子字符串`t`对应于一个普通的子串`t + t`。 

问题在于复杂性。 有`2^n`字符串，每次检查都会扫描一个长度的字符串`O(n)`，可能使用长度的模式匹配`O(|s|)`。 这导致大约`O(2^n * n * |s|)`，这远远超出了可行的范围。 

关键的观察是，我们可以逐个字符地构造字符串，同时跟踪模式是否正确，而不是构造完整的字符串然后检查它们。`s`已经出现在循环结构中。 模式匹配可以表示为前缀上的有限自动机`s`。 这使我们能够将所有子串检查逻辑压缩到一个较小的状态空间中`O(|s|)`。 

剩下的困难是循环条件。 单个字符串`t`引起两次重叠扫描`s`: 一上`t`其本身以及从复制字符串中间开始的移位副本上的一个。 这迫使我们跟踪两个同时进行的自动机进程，每个进程对应循环串联的一半。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有字符串的暴力枚举 | O(2^n · n · | s | ) | O(2^n · n · | s | ) |
 | 带有双线自动机的 DP | O(n·| s | ²·2) |

 ## 算法演练

 我们使用双倍字符串重新解释循环条件`t + t`。 字符串`s`出现在循环中`t`当且仅当它作为子串出现时`t + t`。 

我们现在构建`t`从左到右，同时模拟如何`t + t`会进化。 

1. 为模式构建前缀自动机`s`。 

每个状态代表多少个字符`s`我们目前已将其作为已处理流的后缀进行匹配。 转换告诉我们匹配长度在阅读后如何变化`0`或者`1`。 
2.定义一个DP状态，处理构造`t`按索引索引`0`到`n - 1`。 

在位置`i`，我们跟踪两个自动机状态。 第一个状态对应于扫描前缀`t`本身。 第二个对应于扫描第二份副本`t`，但只有在位置之后才有效开始`n`。 
3. 初始化 DP，两个自动机状态均为零，这意味着尚未处理任何字符，并且模式尚未匹配。 
4. 对于每个位置`i`在`t`，尝试放置`0`或者`1`。 

此选择会转换第一个副本的自动机状态。 如果`i ≥ n`，同一角色也会转换第二个副本的自动机状态，因为`t + t`现在正在被揭晓。 
5. 每次转换后，检查任一自动机状态是否达到完全匹配`s`。 

如果是，则将 DP 状态标记为“匹配”。 一旦匹配，就永远保持匹配。 
6. 处理完所有位置后，对所有已达到匹配的 DP 状态求和。 

DP 有效地对所有引起至少一次出现的二进制字符串进行计数`s`在双重结构中。 

### 为什么它有效

 每个二进制字符串`t`恰好对应于通过该 DP 的一条路径，因为每个位置都是独立决定的。 自动机状态精确编码了多少`s`两个相关扫描中每个点的匹配`t + t`。 由于每一个可能发生的情况`s`循环中必须出现在这两次扫描之一中，将状态标记为成功准确地捕获是否`t`是有效的。 没有遗漏有效的字符串，因为每个构造`t`被探索，并且不计算无效字符串，因为只有在自动机中形成实际的完整匹配时才会记录成功。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_automaton(s):
    m = len(s)
    pi = [0] * m
    for i in range(1, m):
        j = pi[i - 1]
        while j > 0 and s[i] != s[j]:
            j = pi[j - 1]
        if s[i] == s[j]:
            j += 1
        pi[i] = j

    nxt = [[0, 0] for _ in range(m + 1)]
    for st in range(m + 1):
        for c in range(2):
            if st < m and int(s[st]) == c:
                nxt[st][c] = st + 1
            else:
                if st == 0:
                    nxt[st][c] = 0
                else:
                    nxt[st][c] = nxt[pi[st - 1]][c]
    return nxt

def solve():
    n = int(input().strip())
    s = input().strip()
    m = len(s)

    nxt = build_automaton(s)

    # dp[i][a][b][f]
    dp = [[[[0] * 2 for _ in range(m + 1)] for _ in range(m + 1)] for _ in range(n + 1)]
    dp[0][0][0][0] = 1

    for i in range(n):
        for a in range(m + 1):
            for b in range(m + 1):
                for f in range(2):
                    cur = dp[i][a][b][f]
                    if not cur:
                        continue

                    for c in (0, 1):
                        na = nxt[a][c]
                        nb = b
                        if i >= n:
                            nb = nxt[b][c]

                        nf = f or (na == m) or (nb == m)
                        dp[i + 1][na][nb][nf] += cur

    ans = 0
    for a in range(m + 1):
        for b in range(m + 1):
            ans += dp[n][a][b][1]

    print(ans)

if __name__ == "__main__":
    solve()
```代码首先为模式构建一个前缀自动机`s`，允许匹配状态之间的恒定时间转换。 然后 DP 迭代目标字符串的位置`t`。 在每个位置，它都会分支两个可能的二元选择并更新两个自动机状态：一个跟踪第一个副本`t`，一旦第二个副本在索引后变为活动状态，则跟踪第二个副本`n`。 

布尔维度`f`记录模式是否已在模拟循环扫描中的任何点匹配。 当稍后的转换使自动机远离完全匹配状态时，这可以避免丢失有关早期匹配的信息。 

最终答案汇总了至少发生一场比赛的所有州。 

## 工作示例

 ### 示例 1

 输入：```
2
0
```我们跟踪长度为 2 的字符串的 DP 状态。 

| 步骤| 我| 选择| 状态 A | 状态 B | 匹配|
 | --- | --- | --- | --- | --- | --- |
 | 开始 | 0 | - | 0 | 0 | 0 |
 | 0 之后 | 1 | 0 | 1 | 0 | 1 |
 | 1 之后 | 1 | 1 | 0 | 0 | 0 |

 在位置 0 处，选择`0`立即匹配`s`，因此所有延续仍然有效。 这会产生字符串`00`,`01`,`10`，给出 3 个有效的循环字符串。 

### 示例 2

 输入：```
3
11
```我们需要长度为 3 的字符串，其中包含`11`循环地。 

DP 探索所有 8 个字符串，并且仅排除`000`,`001`， 和`100`（其中不包含相邻的`1`甚至在环绕下）。 其余 5 个字符串有效。 

这演示了环绕案例如何`101`仍然有资格，因为循环`101`包含`11`越过边界。

 ## 边缘情况

 像这样的最小模式`s = "0"`在第一个字符处公开立即匹配。 一旦出现匹配，DP 必须正确地将所有扩展标记为有效； 否则它会低估字符串，例如`01`和`10`。 

仅跨越边界匹配的模式，例如`s = "101"`在`t = "110"`，测试第二个自动机流是否正确表示`t`。 DP 通过仅在索引后激活第二个状态来确保这一点`n`，再现圆形匹配所需的移位扫描。
