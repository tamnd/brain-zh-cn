 ## 解决方案
 ## 问题理解

 给定一个整数数组，我们感兴趣的是该数组中满足按位条件的连续段：该段中所有元素的按位 AND 恰好等于同一段中所有元素的按位 OR。 

任务是计算存在多少个这样的段。 

输入可能很大，通常最多大约 10^5 个元素，这会立即排除任何显式检查每个段的方法。 对所有子数组进行简单的 O(n^2) 枚举，每次重新计算 AND 和 OR，在最坏的情况下将涉及 10^10 次运算，这远远超出了典型时间限制下的任何实际限制。 这迫使我们寻找允许聚合而无需重新计算的结构。 

当所有元素都相同时，就会出现关键的边缘情况。 例如，如果数组是`[5, 5, 5]`，那么每个子数组都满足条件，因为 AND 和 OR 都等于 5。仅检查端点或假设成对相等的粗心方法可能仍然会错过较长的段也是有效的。 另一种微妙的情况是元素仅相差一位。 例如`[1, 3]`二进制是`[01, 11]`，其中 AND 是`01`或是`11`，它们不相等，因此即使很小的差异也会立即破坏有效性。 

## 方法

 蛮力的想法很简单。 我们考虑每个子数组`[l, r]`，在该间隔内计算按位 AND 和 OR，并检查它们是否相等。 这是正确的，因为它直接匹配定义。 然而，重新计算每个子数组的 AND 和 OR 会导致每个段的 O(n) 工作量，除非我们预先计算诸如稀疏表之类的东西。 即使进行预处理，检查所有 O(n^2) 子数组仍然太慢，需要大约 O(n^2) 子数组进行验证。 

关键的观察是理解一组数字的 AND 和 OR 何时可以相等。 对于固定位位置，如果至少一个元素在该位中具有 1，则 OR 为 1，而仅当每个元素在该位中都具有 1 时，AND 才为 1。 为了使这两个结果在每个位上都匹配，每个位在两种操作下的行为必须相同。 发生这种情况的唯一方法是，对于每一位，要么所有元素都为 0，要么所有元素都为 1。该条件意味着子数组中的所有数字都相同。 

因此，问题归结为计算所有元素都相等的子数组。 我们现在只需要跟踪相等值的连续运行，而不是考虑按位运算。 每个最大的等值段贡献一个简单的子数组组合计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) 或 O(n² log n) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 我们从左到右扫描数组，同时将连续的相等元素分组为最大块。 这是有效的，因为任何有效的子数组必须完全位于相同值的块内。 
2. 对于每个块，我们确定其长度，例如`k`。 一旦我们知道该段中的所有元素都相同，其中的每个子数组就会自动满足 AND 等于 OR 条件。 
3. 我们计算该块的贡献为`k * (k + 1) / 2`。 该公式计算一段长度内所有可能的连续子数组`k`。 
4. 我们累积所有区块的这些贡献以形成最终答案。 

为什么它有效

 在任何包含两个不同值的子数组中，至少存在一位值不同的位。 在该位，OR 将为 1，而 AND 将为 0，从而打破相等性。 这意味着有效子数组不能跨越相邻值不同的边界，并且不能包含异构元素。 因此，有效子数组正是那些完全包含在相等值的最大游程中的子数组，并且对它们进行计数减少为对每次游程内的所有子数组进行计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    ans = 0
    i = 0
    
    while i < n:
        j = i
        while j < n and a[j] == a[i]:
            j += 1
        
        length = j - i
        ans += length * (length + 1) // 2
        
        i = j
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现是将数组拆分为最大等值段的思想的直接转换。 两指针循环标识每个块`[i, j)`所有值都匹配的地方。 一旦找到一个块，就会使用子数组计数的算术公式计算其贡献。 

一个微妙的点是确保`j`即使数组有许多重复值，也能正确前进，因为未能正确前进将导致无限循环或重复计数。 整数除法是安全的，因为`k*(k+1)`总是均匀的。

 ## 边缘情况

 对于所有元素都相等的数组，该算法将整个数组折叠成一段，并通过三角数公式正确计算所有子数组。 对于像这样的输入`[4, 4, 4]`，循环形成长度为 3 的单个块并输出 6，与所有可能的子数组的数量相匹配。 

对于没有相邻元素匹配的数组，例如`[1, 2, 3, 4]`，每个元素形成一个长度为 1 的块。每个元素恰好贡献一个有效子数组，因此总数为 4。该算法很自然地处理这个问题，因为每次迭代都会在继续前进之前立即关闭一个大小为 1 的块。 

对于混合图案，例如`[1, 1, 2, 1, 1]`，中间元素将结构分成三个块。 该算法独立处理每个数据，确保不会计算任何无效的跨边界子数组。
