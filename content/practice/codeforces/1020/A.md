 ## 解决方案
 ## 问题理解

 这座建筑可以被认为是一个由点组成的网格$n$垂直柱（塔）和$h$水平层（楼层）。 从任何一点，你都可以在同一塔内垂直移动，一次上下一层，每次移动只需要一分钟。 

相邻塔楼之间的水平移动受到更多限制。 你只能在塔之间移动$i$和$i+1$，并且仅适用于编号在该区间内的楼层$[a, b]$。 如果您位于此范围之外的楼层，则无法从那里直接穿越到另一座塔，因此您需要先垂直走到可用的楼层，然后穿过，并可能再次垂直行走。 

每个查询给出该结构中的两个位置，并询问使用这些运动在它们之间移动所需的最短时间。 

关键的难点在于$n$和$h$可以大到$10^8$，因此我们无法模拟构建或针对每个查询运行任何图形搜索。 相反，我们必须推导出最短路径的直接公式。 

当两个位置位于同一塔中时，会出现微妙的边缘情况。 在这种情况下，根本不需要水平移动，答案只是垂直距离。 总是强制“移动到走廊范围并返回”的幼稚解决方案会错误地高估。 

当两个位置位于不同的塔楼但已经位于走廊范围内时，会出现另一个重要情况$[a, b]$。 在这种情况下，我们可以水平移动，无需任何垂直调整，最佳路径就是水平距离加上直接层差对齐。 

## 方法

 如果我们忽略优化约束，自然的想法是将每个查询建模为网格图上的最短路径问题。 每个细胞$(t, f)$垂直连接到$(t, f \pm 1)$，并水平地到$(t \pm 1, f)$仅当$f \in [a, b]$。 每个查询运行 BFS 或 Dijkstra 将正确找到最短路径。 

然而，这个图是巨大的，多达$10^8 \cdot 10^8$可能的节点，所以即使是单个 BFS 也是不可能的。 即使我们只限制相关节点，每个 BFS 仍然可以遍历$O(hn)$最坏情况下的状态，这远远超出了任何限制$k \le 10^4$。 

问题的结构拯救了我们。 水平移动是自由选择的：每当我们想要在塔楼之间移动时，我们只关心距离可用走廊间隔有多远$[a, b]$。 最优策略总是归结为决定是留在当前楼层，还是垂直移动到最近的走廊楼层，然后水平移动，最后再次垂直调整。 

这将每个查询减少为少量恒定数量的算术运算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力 BFS |$O(k \cdot n \cdot h)$|$O(nh)$| 太慢了|
 | 最优配方|$O(k)$|$O(1)$| 已接受 |

 ## 算法演练

 对于每个查询，我们比较两个位置$(t_a, f_a)$和$(t_b, f_b)$。 

1. 首先检查两个塔是否相同。 如果$t_a = t_b$，答案很简单$|f_a - f_b|$，因为我们永远不需要离开塔，水平运动是无关紧要的。 
2. 计算塔之间的水平距离：$|t_a - t_b|$。 如果塔不同，这始终是答案的一部分。 
3. 如果起始楼层和结束楼层均位于走廊区间内$[a, b]$，那么我们就可以水平移动而无需任何垂直绕行。 总成本为：$$|t_a - t_b| + |f_a - f_b|$$因为我们可以独立地垂直和水平对齐。 
4.否则，至少有一个位置在走廊范围之外。 在这种情况下，我们必须垂直移动才能在某个点进入走廊。 
5、最优策略是选择走廊楼层$x \in [a, b]$从而最大限度地降低总成本：$$|f_a - x| + |f_b - x|$$结合水平距离$|t_a - t_b|$。 该函数在整数上是凸的，因此在最接近的点处达到最小值$[a, b]$到两端。 
6. 因此我们将每一层固定在区间内：$$f'_a = \min(\max(f_a, a), b), \quad f'_b = \min(\max(f_b, a), b)$$并计算：$$|f_a - f'_a| + |f_b - f'_b| + |t_a - t_b|$$### 为什么它有效

 关键的不变量是任何穿过塔楼之间的有效路径必须完全在走廊带内$[a, b]$。 在此区间之外的任何绕行只会增加垂直成本，而不会启用额外的水平选项。 因此，最佳路径在走廊内总是有一个“水平阶段”，而其他一切都是朝着该阶段的垂直运动。 由于垂直运动是线性的并且每个塔都是独立的，因此交叉楼层的最佳选择始终是最接近的可行楼层$[a, b]$，这正是钳位计算的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, h, a, b, k = map(int, input().split())
    
    def clamp(x):
        if x < a:
            return a
        if x > b:
            return b
        return x

    for _ in range(k):
        ta, fa, tb, fb = map(int, input().split())
        
        if ta == tb:
            print(abs(fa - fb))
            continue
        
        # move both floors into corridor range
        fa_c = clamp(fa)
        fb_c = clamp(fb)
        
        # horizontal distance + vertical adjustments
        dist = abs(ta - tb) + abs(fa - fa_c) + abs(fb - fb_c)
        print(dist)

if __name__ == "__main__":
    solve()
```该解决方案是围绕将每个查询直接转换为恒定时间计算而构建的。 这`clamp`函数编码将任何楼层投影到有效走廊带上的想法。 这避免了在遍历期间明确地推理中间楼层。 

特殊情况`ta == tb`首先处理，因为水平移动会引入不必要的术语。 一旦塔楼不同，我们总是只支付一次水平距离，其余的计算则减少为两侧独立的垂直校正。 

该结构确保不需要浮点推理或路径枚举，只需要整数算术和比较。

 ## 边缘情况

 当两个点位于同一塔中时，算法立即返回垂直距离。 例如，在单个塔中从第 9 层移动到第 1 层会产生成本 8，并且不使用走廊推理。 该实现正确地避免了不必要的水平项。 

当两层楼都已在里面时$[a, b]$， 例如$a=2, b=5$并指向第 3 层和第 4 层，钳位操作使两者保持不变。 结果成为网格上的纯曼哈顿距离，这与我们可以直接在这些楼层水平移动的事实相匹配。 

当两个端点都在相对两侧的走廊之外时，例如 1 和 10 有走廊$[4, 6]$，夹紧力分别为 4 和 6。 该算法有效地选择了最接近的有效穿越结构，其成本反映了两次垂直爬升加上水平行驶，与唯一可行的最佳路线结构相匹配。
