 ## 解决方案
 ## 问题理解

 我们有一个针对学生的定向结构，其中每个学生都指向另一个学生。 这形成了一个函数图：每个节点的出度为1，因此从任何节点开始并重复跟随指针最终迫使我们进入一个循环。 

老师从某个学生开始`a`。 该学生立即被标记一次。 然后老师继续从当前的学生移开`i`到`p[i]`，标记每个被访问的学生。 当老师到达已经被评分的学生时，该过程就会停止，并且该学生将获得第二个分数。 

任务是确定对于每个可能的起始节点`a`，哪个学生将是第一个被访问两次的学生。 

约束条件`n ≤ 1000`足够小，O(n²) 甚至 O(n³) 模拟是安全的。 任何从每个起始节点运行新遍历的解决方案都已经在限制范围内，因为每次遍历最多可以访问`n`重复之前的节点。 

一个微妙的地方是，这个过程并不是静态意义上简单的“寻找循环条目”，因为访问节点的集合取决于起始点。 第一个重复节点是按遍历顺序出现两次的第一个节点，不一定是通常分解意义上的循环的开始。 

重要的边缘情况包括：

 自循环例如`p[i] = i`。 从这样一个节点开始，第一步就会立即重新访问它，所以答案本身就是微不足道的。 

另一种情况是所有节点最终流入一个循环。 即使这样，尾部中的不同起点也可能导致不同的首次重复节点，因为循环的入口点取决于路径的遍历方式。 

## 方法

 蛮力的想法很简单：为每个开始的学生模拟老师的动作。 我们维护一个访问过的数组，沿着`p[current]`，并在到达模拟中已看到的节点后立即停止。 该节点被记录为开始的答案。 

这是有效的，因为每个模拟都会独立地重建语句中描述的确切过程。 正确性是立竿见影的，因为我们实际上是在一步一步地遵循规则。 

低效率来自于从头开始重复相同的行走。 每个模拟可能会遍历长度为 O(n) 的长链，并且我们对所有 n 个起点执行此操作，总共给出 O(n²) 个步骤。 当 n 达到 1000 时，这仍然没问题，但它暗示了更深层次的结构：每次遍历都是确定性的，并且仅取决于函数图中的后续指针。 

关键的观察结果是，起始节点的答案仅取决于其路径首次与循环相交的位置或重新访问其路径上已有的节点。 由于该图是函数图，因此每个节点要么属于通向循环的尾部，要么属于循环本身。 任何开始的第一个重复节点正是路径上的第一个节点，其下一次访问将违反该遍历中的“第一次看到”属性。 

由于n很小，我们不需要大量的预处理或循环分解； 使用新访问的数组进行直接模拟就足够了。

 | --- | --- | --- | --- |

 | 最佳 | O(n²) | O(n) | 已接受 |

 ## 算法演练

 1. 对于每个开始的学生`a`，创建一个新的布尔数组`visited`尺寸的`n + 1`。 这确保了每个模拟都是独立的，并且不会在启动时泄漏状态。 
2. 初始化`current = a`。 
3、反复检查是否`current`已被访问过。 如果有，这意味着我们已经返回到在此模拟中已经看到的节点，所以`current`就是这个起点的答案。 
4.如果没有访问过，标记`current`正如访问过的那样。 
5. 移动`current = p[current]`并重复。 
6. 存储触发“已访问”条件的第一个节点作为答案`a`。 

此步骤顺序很重要的原因是检查必须在前进之前进行，因为重复节点被定义为我们刚刚进入的节点，而不是我们即将离开的节点。 

### 为什么它有效

 模拟期间从固定开始`a`，算法保持不变式`visited[x]`当且仅当节点为真`x`在此确切的遍历中较早已访问过。 由于该图的出度为 1，因此访问节点的序列形成单次行走。 当我们第一次遇到一个已经被标记的节点时，根据我们如何维护的定义，该节点必须是序列中最早的重复节点`visited`。 较早的节点不能重复，否则循环会提前终止。 这保证了返回的节点正是该过程中访问两次的第一个节点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    p = [0] + list(map(int, input().split()))
    
    ans = [0] * (n + 1)
    
    for start in range(1, n + 1):
        visited = [False] * (n + 1)
        cur = start
        
        while True:
            if visited[cur]:
                ans[start] = cur
                break
            visited[cur] = True
            cur = p[cur]
    
    print(*ans[1:])

if __name__ == "__main__":
    solve()
```实现完全按照描述进行遍历。 关键的细节是我们检查`visited[cur]`在标记或移动之前，因为在我们降落在已经看到的节点上时检测到重复。 每次启动都会重新计算自己访问过的数组，以避免模拟之间的干扰。 

1 索引数组简化了从学生标签到索引的直接映射，避免读取时出现差一错误`p`。 

## 工作示例

 ### 示例 1

 输入：```
3
2 3 2
```我们分别计算每个开始。 

为了`start = 1`，我们跟踪遍历：

 | 步骤| 当前| 之前访问过 | 行动|
 | --- | --- | --- | --- |
 | 1 | 1 | {} | 标记 1 |
 | 2 | 2 | {1} | 标记 2 |
 | 3 | 3 | {1,2} | 标记 3 |
 | 4 | 2 | {1,2,3} | 重复发现|

 答案是2。 

对于`start = 2`:

 | 步骤| 当前| 之前访问过 | 行动|
 | --- | --- | --- | --- |
 | 1 | 2 | {} | 标记 2 |
 | 2 | 3 | {2} | 标记 3 |
 | 3 | 2 | {2,3} | 重复发现|

 答案是2。 

对于`start = 3`:

 | 步骤| 当前| 之前访问过 | 行动|
 | --- | --- | --- | --- |
 | 1 | 3 | {} | 标记 3 |
 | 2 | 2 | {3} | 标记 2 |
 | 3 | 3 | {2,3} | 重复发现|

 答案是3。 

这证实了第一次重复取决于遍历进入循环的位置，而不仅仅是循环本身。 

### 示例 2

 考虑：```
4
1 2 3 4
```每个节点都指向下一个节点，形成一个循环。 

为了`start = 1`，我们访问`1 → 1`，所以答案是1。 

对于`start = 2`，我们访问`2 → 2`，所以答案是2。 

对于其他人来说也是如此。 在这种遍历意义上，每个节点都是自返回，因为我们在循环遍历该运行中已经看到的所有节点后立即重新访问。

 ## 边缘情况

 自循环就像`p[i] = i`是最直接的案例。 开始于`i`，我们标记它并在一次转换后立即再次看到它，因此算法返回`i`正确地因为`visited[i]`当我们重新审视它时，这已经是事实了。 

进入循环的链的行为类似，但重复延迟。 例如`1 → 2 → 3 → 3`。 从 1 开始，我们访问 1、2、3，然后尝试再次移动到 3。 由于 3 已被标记，因此它被正确识别为第一个重复节点。 模拟可确保在第一次重新访问时准确检测到周期条目，而不是更早或更晚。
