 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 每个测试用例都有一个整数数组，我们可以任意重新排序。 选择排列后，我们查看排列数组的所有分割点。 

对于索引 i 和 i+1 之间的每个分割位置，我们计算 i 之前的前缀的按位 OR 以及从 i+1 开始的后缀的按位 OR。 要求是对于每个可能的分割点，这两个值始终相等。 我们需要统计有多少个数组的排列满足这个性质，并输出对 998244353 取模的结果。 

这些约束允许所有测试用例最多包含 200,000 个元素，这会立即排除任何尝试所有排列或显式检查每个排列的解决方案。 即使 n = 2e5 的单个测试也已经使每个排列的 O(n^2) 或 O(n log n) 变得不可能，因此解决方案必须将问题简化为每个测试用例的简单计数公式，最好是线性或带有预计算的线性算式。 

当考虑小数组时，会出现一种微妙的边缘情况。 例如，如果 n = 2 并且数组为 [1, 2]，则唯一的排列是其本身或交换，但是否任何排序都能满足条件并不明显。 幼稚的解释可能会提出许多有效的排列，但实际上，条件极其严格，并且通常会强制采用严格的结构，特别是对于某些值的少量计数。 

## 方法

 暴力破解的想法很简单：生成数组的每个排列，计算所有前缀 OR 和后缀 OR，并检查它们在每次分割时是否匹配。 这是正确的，但不可行。 生成 n! 当 n = 12 时，排列已经变得难以管理，并且每次检查的成本为 O(n)，导致无法优化的阶乘增长。 

关键的观察结果来自于重写条件。 如果对于每个分割位置 i，前缀 OR 等于后缀 OR，则所有这些值在所有 i 中必须相同。 该公共值还必须等于整个数组的 OR，因为在有限的意义上，整个数组可以被视为前缀或后缀。 

将此全局 OR 值称为 S。该条件意味着对于每个 i，前缀 a1 | ... | ai 和后缀 ai+1 | ... | an 必须已经等于 S。 

这创建了一个强大的结构约束：每个前缀必须已经包含 S 的所有位，这意味着不允许任何前缀“错过”出现在数组中任何位置的位。 对称地，每个后缀也必须包含 S 的所有位。 

现在考虑一下这对于各个元素意味着什么。 如果第一个元素不等于 S，则一个元素后面的前缀将丢失一些位，这与它必须已经等于 S 的要求相矛盾。因此第一个元素必须恰好是 S。从后缀方面的相同推理强制最后一个元素也为 S。 

一旦两端都固定为S元素，所有剩余元素都可以是S的任意子集，因为它们不需要引入新的位，并且它们不能破坏条件，因为S已经完全存在于两端。 

这将问题简化为纯粹的组合任务：如果有 k 个元素等于 S，我们必须选择这些 S 元素占据的两个不同位置（第一个和最后一个），然后自由排列剩余的 n−2 个元素。 

如果 k < 2，则 n ≥ 2 时不存在有效排列。否则，有序端点的路数为 k × (k − 1) 个选择，乘以 (n − 2)！ 以便安排其余的事情。

 | 蛮力 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 最佳| 预处理后每个测试的 O(n) | O(n) | 已接受 |

 ## 算法演练

1. 计算整个数组的按位 OR，将其称为 S。该值表示所有前缀和后缀 OR 必须匹配的唯一可能值，因为它们被限制为在各处都相等。 
2. 计算数组中有多少个元素等于S，称此计数为k。 这些是唯一可以占据强制完全覆盖所有位的位置的候选者。 
3. 如果k小于2，则立即返回0。如果全或元素少于两个，我们就无法满足第一个位置和最后一个位置必须已经包含S的所有位的要求。 
4. 在所有测试用例中以 998244353 为模，预先计算阶乘直至最大可能的 n。这允许快速计算剩余元素的排列。 
5. 对于每个测试用例，将答案计算为 k × (k − 1) × (n − 2)！ 以 998244353 为模。 
6. 输出结果。 

仅修复端点背后的原因是，一旦两端都是 S，每个前缀都会自动包含 S，因为第一个元素已经贡献了所有丢失的位。 类似地，每个后缀都包含 S，因为最后一个元素已经贡献了所有位。 内部排序不再影响 OR 属性。 

### 为什么它有效

 不变量是每个有效的排列必须保持每个前缀 OR 和每个后缀 OR 等于全局 OR S 的属性。这迫使第一个元素已经完全实现 S，否则第一个前缀将小于 S。相同的逻辑对称地应用于最后一个元素。 

一旦两个端点都固定为等于 S 的元素，所有剩余元素就与 OR 约束无关，因为它们不能引入超出 S 的新位，也不能删除现有的位。 因此，问题完全简化为计算具有受限端点的排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    t = int(input())
    tests = []
    maxn = 0

    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        tests.append((n, arr))
        maxn = max(maxn, n)

    fact = [1] * (maxn + 1)
    for i in range(1, maxn + 1):
        fact[i] = fact[i - 1] * i % MOD

    out = []

    for n, arr in tests:
        S = 0
        for x in arr:
            S |= x

        k = 0
        for x in arr:
            if x == S:
                k += 1

        if k < 2:
            out.append("0")
            continue

        ans = k * (k - 1) % MOD
        if n >= 2:
            ans = ans * fact[n - 2] % MOD

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现首先聚合所有测试用例以构建阶乘一次，这避免了每个测试重复的 O(n) 预处理。 然后，每个测试都会计算全局 OR 并计算有多少元素实现它。 直接应用最终公式。 

一个常见的错误是忘记端点是有序的，而不仅仅是作为一组选择的。 这就是为什么出现 k × (k − 1) 而不是组合项。 

## 工作示例

 考虑一个数组 [1, 2, 3]。 全局 OR 为 3，因此只有等于 3 的元素对端点有意义。 存在一个这样的元素，因此 k = 1，这立即使答案为 0。 

| 步骤| 价值|
 | --- | --- |
 | S | 3 |
 | k | 1 |
 | 有效排列 | 0 |

 由此可见，即使数组中可以出现3，但单个出现并不能满足控制两端的要求。 

现在考虑[3,3,1]。 全局 OR 为 3，k = 2。 

| 步骤| 价值|
 | --- | --- |
 | S | 3 |
 | k | 2 |
 | 结束选择| 2 × 1 | 2 × 1
 | 中间排列 | 1！ |
 | 最终答案| 2 |

 这演示了一旦端点被固定，剩余的元素可以自由放置而不影响有效性。

import sys, io

MOD = 998244353

def solve_io(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    tests = []
    maxn = 0
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        tests.append((n, arr))
        maxn = max(maxn, n)

    fact = [1] * (maxn + 1)
    for i in range(1, maxn + 1):
        fact[i] = fact[i - 1] * i % MOD

    res = []
    for n, arr in tests:
        S = 0
        for x in arr:
            S |= x
        k = sum(1 for x in arr if x == S)

        if k < 2:
            res.append("0")
        else:
            ans = k * (k - 1) % MOD
            ans = ans * fact[n - 2] % MOD
            res.append(str(ans))

    return "\n".join(res)

# provided-style sanity checks
assert solve_io("1\n2\n1 2\n") == "0"
assert solve_io("1\n2\n3 3\n") == "2"

# custom cases
assert solve_io("1\n3\n1 1 1\n") == str(3 * 2 % MOD * 1 % MOD), "all equal"
assert solve_io("1\n3\n1 2 4\n") == "0", "no full-or duplicates"
assert solve_io("1\n4\n7 1 7 7\n") == str(3 * 2 % MOD * 2 % MOD), "multiple S elements"
assert solve_io("2\n2\n1 1\n2\n1 2\n") == "2\n0", "multi-test mix"

 ## 边缘情况

 当所有元素相同时，每个元素都等于全局 OR S，因此 k = n。 公式变为 n × (n − 1) × (n − 2)!，简化为 n!，符合每个排列在每次分割时都会保留相同 OR 值的事实。 

当恰好有一个元素等于全局 OR 时，算法会正确返回 0。尽管内部元素可能看起来很灵活，但无法满足端点要求，并且公式会立即正确捕获此失败。 

当多个元素等于 S 但 n 较大时，阶乘项占主导地位。 该算法仍然表现正确，因为只有端点选择会影响可行性，而内部排列仍然完全不受限制。
