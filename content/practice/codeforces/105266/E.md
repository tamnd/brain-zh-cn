 ## 解决方案
 ## 问题理解

 我们有两个长度相等的数组。 我们从数组开始`a`，并且我们可以选择一个连续的段并将其反转。 最多执行一次此操作后，我们将结果数组与固定数组进行比较`b`相同长度的。 

配置的成本定义为两个数组的对齐元素之间按位异或的所有索引的总和。 反转一个段`a`改变哪些元素`a`与哪些元素配对`b`，因此该操作不是本地的：一个反转同时以结构化方式重新排列许多配对。 

任务是在选择最佳可能的段进行反转或选择不反转任何内容后，找到最小可能的总异或和。 

约束条件`n ≤ 5000`立即排除任何尝试所有部分并从头开始重新计算全部成本的解决方案，因为这将导致粗略`O(n^3)`operations in the worst case. 甚至`O(n^2)`解决方案需要仔细的增量更新才能在 2 秒内轻松通过。 

当贪婪地思考时，就会出现一个微妙的问题。 通过反转一个段来局部改善一些位置可能会以一种不明显的方式使许多其他位置恶化，因为异或不保序。 例如，反转修复其边界附近的大不匹配的段可能会破坏该段内的几个已经良好的匹配。 这种相互依赖正是防止贪婪选择的原因。 

## 方法

 如果我们忽略反转操作，问题就很简单：我们计算`sum(a[i] XOR b[i])`。 困难在于理解当我们反转段时这个总和如何变化`[l, r]`。 

蛮力方法会尝试每一个可能的部分。 对于每个`(l, r)`，我们模拟倒车`a[l..r]`并重新计算完整的异或和。 从头开始重新计算成本`O(n)`每个段，并且有`O(n^2)`细分，导致`O(n^3)`总操作。 和`n = 5000`，这是远远不可能的。 

我们可以通过观察反转如何影响配对来减少重新计算成本。 在反转段内，索引`i`交换其伙伴`b[i]`到`b[l + r - i]`从配对结构来看。 所以只有里面的位置`[l, r]`改变贡献，而外部一切保持不变。 这将每个段的重新计算减少到`O(n)`仍然如此，但建议我们应该重点关注对部门贡献的快速评估。 

关键的见解是停止从完整阵列的角度思考，而是从配对结构的角度思考。 每次反转都定义了索引之间的新匹配`a`和`b`。 对于段外的指数，配对不变。 在该段内，我们将对称位置配对`(i, l + r - i)`。 这将问题转化为选择细分市场并评估结构化配对的成本。 

现在关键的观察是，我们只需要计算反向段中对称位置形成的对的贡献，加上不变的外部成本。 这使我们能够预先计算基线成本，然后评估每个部分`O(r - l)`或摊销`O(1)`如果我们仔细扩展该段，则每次转换都会更新。 

我们迭代所有中心并向外扩展，保持配对的效果`(l, r)`随着我们的成长。 每次扩展都会通过用两个新配对替换两个原始配对来更新成本，这可以在恒定时间内计算。 

这将解决方案转变为`O(n^2)`扩展过程而不是`O(n^3)`重新计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（重新计算每个段的总和）| O(n^3) | O(n^3) | O(1) | O(1) | 太慢了 |
 | 通过增量 XOR 更新扩大间隔 | O(n^2) | O(n^2) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从没有应用逆转的基线开始。 初始成本是以下总和`a[i] XOR b[i]`。 

1. 通过将每个配对来计算基线成本`a[i]`和`b[i]`。 这代表没有执行任何操作的情况。 
2. 考虑每一个可能的反转中心。 我们将每个索引视为段的潜在中间，并以两种方式向外扩展：一种用于奇数长度段，一种用于偶数长度段。 
3. 对于每个扩展步骤，我们维护一个当前段`[l, r]`。 最初`l = r = i`对于奇数长度的情况，或者`l = i`,`r = i + 1`对于偶数长度的情况。 
4. 当我们将该段向外扩展一位时，我们包括两个新索引：`l - 1`和`r + 1`。 逆转交换了配对结构，所以代替`(l - 1 with b[l - 1])`和`(r + 1 with b[r + 1])`，这些位置现在配对为`(l - 1 with b[r + 1])`和`(r + 1 with b[l - 1])`。 
5. 我们通过删除这两个位置的旧贡献并添加新的交换贡献来更新当前成本。 此更新是恒定时间，因为它仅取决于四个 XOR 评估。 
6. 我们跟踪所有扩展和所有中心的最低成本。 

### 为什么它有效

 每一个可能的片段`[l, r]`is uniquely generated either as an odd-length or even-length expansion around some center. 对于每个片段，该算法通过使用本地更新在恒定时间内从前一个片段进行转换来计算其确切成本，这些本地更新准确地反映了反转如何改变配对。 由于每次反转仅影响端点配对的方式，并且扩展以归纳方式保留正确性，因此不会遗漏任何片段，也不会重复计算任何贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    base = 0
    for i in range(n):
        base += a[i] ^ b[i]

    ans = base

    # odd length segments
    for c in range(n):
        cur = base
        l = c - 1
        r = c + 1
        while l >= 0 and r < n:
            cur -= (a[l] ^ b[l]) + (a[r] ^ b[r])
            cur += (a[l] ^ b[r]) + (a[r] ^ b[l])
            ans = min(ans, cur)
            l -= 1
            r += 1

    # even length segments
    for c in range(n):
        l = c
        r = c + 1
        if r >= n:
            continue
        cur = base
        cur -= (a[l] ^ b[l]) + (a[r] ^ b[r])
        cur += (a[l] ^ b[r]) + (a[r] ^ b[l])
        ans = min(ans, cur)

        l -= 1
        r += 1
        while l >= 0 and r < n:
            cur -= (a[l] ^ b[l]) + (a[r] ^ b[r])
            cur += (a[l] ^ b[r]) + (a[r] ^ b[l])
            ans = min(ans, cur)
            l -= 1
            r += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先计算无反转的基线 XOR 成本。 该值已作为候选答案，因为该操作是可选的。 

实现的核心是围绕每个中心的对称扩展。 对于奇数长度的段，我们从单个中心元素开始并向外扩展。 对于偶数长度的线段，我们从二元中心开始。 每次扩展段时，我们都会删除新包含的端点的贡献，就好像它们没有改变一样，并用反转后的贡献替换它，其中端点交换它们的值`b`partners.

 减法和加法模式是关键的实现细节。 它强制只更新两个新包含的位置，而其他所有内容都与之前的状态保持不变。 这种增量过渡使得复杂性保持二次方。 

## 工作示例

 考虑第一个示例，其中段反转可以完全对齐数组。 我们追踪一个中心的流程。 

| 步骤| l | r | current cost |
 | --- | --- | --- | --- |
 | base | - | - | initial sum |
 | expand 1 | 1 | 3 | 交换端点后更新|
 | expand 2 | 0 | 4 | further refined |

 每次扩展都严格替换端点配对，而不触及内部已处理的结构。 

这表明，一旦建立了一个段，进一步的扩展就会一致地进行，因为只有边界配对发生了变化。 

现在考虑一个没有逆转可以改善答案的情况。 在这种情况下，每次扩张要么增加成本，要么保持不变，最低成本保持在基线。 

| 步骤| l | r | current cost |
 | --- | --- | --- | --- |
 | base | - | - | initial sum |
 | any expansion | - | - | ≥ base |

 这表明，当不存在有益片段时，算法自然地保留原始答案。

 ## 边缘情况

 单元素数组是最简单的情况，反转不会改变任何内容。 该算法会处理它，因为两个扩展循环都会立即终止，而不会进入内部 while 循环，从而使基线保持不变。 

什么时候`n = 2`，仅存在一个偶数长度的线段。 算法正确初始化`l = 0, r = 1`and performs exactly one update, which corresponds to the only possible reversal.

 如果值之间相同`a`和`b`，每个 XOR 都是零，并且所有更新都保持零成本。 每次扩展都会减去和添加相同的值，因此在整个执行过程中成本保持不变的不变量成立。
