 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 在每种情况下，都有多种类型的权重，并且每种类型都有无限的供应。 然而，我们如何使用它们有一个限制：所有选择的重量必须放在天平的同一侧。 使用这些权重的一些选择，我们希望能够表示从 1 到 m 的每个整数权重，这意味着对于该范围内的每个目标值 x，我们必须能够选择一组总和恰好为 x 的可用权重。 

目标不是最大化可表示范围，而是在使用尽可能少的单独权重块的同时实现区间 [1, m] 的完全覆盖，其中我们可以重复使用类型，但每个选择的副本都计入所使用的块总数。 

一个关键的微妙之处是输入给出的是权重类型，而不是单个的块。 我们有效地选择了每种重量类型的副本数量，然后这些副本就成为我们可用的集合。 问题变成：要使从 1 到 m 的所有总和都可以使用这些所选副本的无限重复来实现，总副本的最小数量是多少。 

约束总体上很大：最多 50 个测试用例，每个用例最多 10,000 个权重类型，并且 m 可以大到 1e9。 任何试图直接模拟子集总和达到 m 的解决方案都是不可能的，因为即使每个测试用例的 O(m) 也已经超出限制，并且 O(n·m) 是完全不可行的。 

一个天真的错误是假设我们可以贪婪地选择权重而不跟踪可表示的区间。 例如，如果我们在 m = 4 时选择权重 2 和 3，我们可能会认为我们可以以某种方式覆盖所有内容，但 1 立即变得无法到达。 这表明可表示性在很大程度上取决于我们是否可以构建从 1 开始的连续范围。 

另一个常见的失败案例是忽略差距。 假设我们选择权重 {1, 3}。 我们可以形成 1 和 3，但是缺少 2，之后系统完全崩溃，因为我们无法仅使用 1 和 3 的加法来填补空白。任何正确的策略都必须确保可表示范围保持连续。 

## 方法

 蛮力的想法是考虑我们可能采用的权重副本的每个子集，然后使用这些所选权重的无限制使用来模拟最多 m 的所有可能总和。 即使我们限制自己最多选择 k 个副本，选择它们的方式数量也是 n 级的组合，那么每个选择都需要最多 m 的背包式 DP。 这很快就会爆炸：即使单个 DP 达到 1e9 也是不可能的，而枚举子集的情况则呈指数级恶化。 

关键的观察是，我们实际上并不关心超出我们已经形成的当前连续前缀的各个总和。 如果我们已经知道我们可以形成 [1, R] 中的每个值，那么新的权重 w 将以一种非常结构化的方式表现：使用它，当且仅当 w 至多为 R + 1 时，我们可以将覆盖范围扩展到 [1, R + w]。否则，在 R + 1 处会出现间隙，我们以后永远无法修复它。 

这导致了一种贪婪的构造，让人想起用最少的元素构建可达到的总和。 我们按递增顺序处理权重并维护最大的连续可表示前缀 R。最初 R = 0，因为没有任何东西是可表示的。 每次我们决定采用权重 w 时，我们都会使用它的一个副本，这允许我们将范围扩展到 R + w，前提是 w 在当前约束下可用。 如果最小的剩余重量大于R + 1，我们被迫得出结论1..m无法完成。 

为了最大限度地减少所选件的数量，我们应该始终选择有助于扩大范围的最小可用重量，因为较大的重量会消耗每件物品更多的“能量”，并且无助于填补小间隙。

 | --- | --- | --- | --- |
 | 暴力破解（子集 + DP）| 指数 / O(2^n · m) | O(米) | 太慢了 |
 | 最优贪婪前缀扩展| O(n log n) | O(n log n) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们对权重进行排序，以便我们始终首先考虑较小的值。 

1.初始化一个变量R = 0，代表我们当前可以形成的最大连续和。 同时初始化索引 i = 0 和计数器 ans = 0。 
2. 当R<m时，我们尝试扩大范围。 我们查看最小的未使用权重 w = a[i]。 
3. 如果不存在这样的权重或者 w > R + 1，我们立即停止并返回 -1。 这是因为我们无法形成 R + 1，因此范围永久断开。 
4. 否则，我们将权重取一次，将 ans 加 1，并更新 R = R + w。 然后我们继续前进。 
5. 重复直到 R >= m，此时 ans 是所需的最小权重副本数。 

重要的设计选择始终是消耗尽可能最小的重量，从而延长可到达的间隔。 在早期阶段，较大的权重永远不会更好，因为它们无助于弥补小的缺失值，并且只有在前缀已经增长之后才变得有用。 

### 为什么它有效

 在任何时刻，假设我们可以表示 [1, R] 中的每个整数。 任何新选择的权重 w 只有在不超过 R + 1 的情况下才能有意义，因为否则无论我们如何将现有权重与 w 组合，R + 1 仍然无法达到。 如果 w ≤ R + 1，则添加 w 的单个副本会将可达区间扩展为 [1, R + w]，因为我们可以形成直到 R 的所有值，然后移位 w 以连续填充下一段。 这个不变量确保 R 始终代表所选多重集可实现的最大连续前缀，并且贪婪选择确保我们以后不会在缩小间隙至关重要时浪费一点权重。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []
    for _ in range(T):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort()

        R = 0
        i = 0
        ans = 0

        while R < m:
            if i == n or a[i] > R + 1:
                out.append("-1")
                break
            R += a[i]
            ans += 1
            i += 1
        else:
            out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码直接反映了贪婪构造。 排序确保我们总是首先看到最小的候选者。 变量`R`跟踪可达前缀，并且循环不变性保证中的所有值`[1, R]`使用当前选定的权重可以实现。 条件`a[i] > R + 1`是间隙处的精确失效点`R + 1`变得不可避免。 

一个微妙的细节是`while-else`结构，它将成功完成（R 达到 m）与提前终止明确分开。

 ## 边缘情况

 临界边缘情况是最小可用权重大于 1 时。例如，如果输入以 2 开头，我们会立即失败，因为我们无法形成 1，因此无法开始连续间隔。 该算法在初始化时捕获了这一点：由于 R = 0，我们要求权重 ≤ 1，而 2 违反了这一点。 

另一种情况是权重增长太快，例如 [1, 100, 101]。 取1后，R = 1，但下一个可用权重必须≤2。由于100太大，我们立即终止。 该算法正确地尽早拒绝，而不是稍后尝试补偿。 

最后一个微妙的情况是，非常大的权重出现在排序列表的早期，但较小的权重出现在后面。 排序可以防止错误的决定，确保在大跳跃之前始终考虑小桥接权重。
