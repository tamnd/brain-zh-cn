 ## 解决方案
 ## 问题理解

 给定一个正整数序列，并要求计算有多少个连续子数组满足涉及两个经典范围函数的看似简单的不等式：子数组中所有元素的最大公约数以及同一子数组中所有元素的总和。 

对于任何间隔$[l, r]$，我们计算该区间内所有元素的最大公约数，并计算它们的总和。 如果 gcd 值小于或等于总和值，则认为子数组有效。 任务是计算存在多少个这样的子数组。 

乍一看，这看起来像是一个标准的“计算具有属性的所有子数组”问题。 约束条件$n \le 10^5$立即排除枚举所有$O(n^2)$子数组并为每个子数组独立计算 gcd 和 sum，因为这会导致大致$10^{10}$最坏情况下的操作，远远超出2秒的限制。 

条件中还隐藏着结构观察。 gcd 和 sum 在扩展区间时以不同的方式都是单调的，但不是以允许直接两指针解而无需额外结构的方式。 当我们扩展一个段时，gcd只能保持不变或减少，而总和总是增加。 这种不平衡使得问题变得微妙。 

一个关键的边缘情况来自以下事实：对于正整数，gcd 始终至少为 1。 如果所有元素均为 1，则每个区间的 gcd 均为 1，而 sum 只是区间的长度。 条件始终成立，所以答案应该是$n(n+1)/2$。 任何错误地假设 gcd 随区间长度增长的方法都会在这里失败。 

当数组包含大值且小 gcd 快速崩溃为 1 时，会出现另一个极端情况。在这种情况下，许多间隔变得微不足道，因为总和很快就主导了 gcd。 简单的每个子数组 gcd 计算会重复重新计算相同的 gcd 值，从而导致冗余。 

## 方法

 蛮力的想法很简单。 我们枚举每个区间$[l, r]$，通过迭代计算其 gcd$l$到$r$，类似地计算其总和，并检查条件。 这是正确的，因为它直接遵循问题的定义。 

然而，成本却令人望而却步。 有$O(n^2)$间隔，以及每个间隔的计算成本$O(n)$在最坏的情况下，如果天真地这样做，会导致$O(n^3)$。 即使我们通过携带前缀和进行求和优化，gcd仍然需要$O(n)$每个区间展开，给出$O(n^2 \log A)$或者更糟，具体取决于实施情况。 为了$n = 10^5$，这还是不可能的。 

关键的观察是，虽然 sum 很容易增量维护，但当我们扩展固定的右端点时，gcd 的行为方式会大大减少不同状态的数量。 对于固定的$r$，如果我们考虑所有可能的情况$l$, 结尾的后缀的 gcd 值$r$形成一个小集合，因为 gcd 仅当除以数组中存在的某个素因数时才会改变。 事实上，所有以 结尾的后缀的不同 gcd 值的数量$r$值是对数的。 

这导致了一个标准技巧：对于每个右端点，我们维护一个表示不同 gcd 值及其延伸距离的压缩对列表。 每次添加新元素时，我们都会将其合并到现有的 gcd 段中。 同时，我们维护前缀和，以便可以查询任何区间和$O(1)$。 

对于每个右端点，一旦我们知道以结尾的子数组的所有可能的 gcd 值$r$，我们可以检查每个候选左边界范围并计算有多少满足gcd$\le$使用前缀和进行求和。 由于每个位置的 gcd 状态数量很少，因此总复杂度变为$O(n \log A)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了 |
 | 最佳|$O(n \log A)$|$O(n \log A)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组，将每个位置视为子数组的端点。 

1. 维护一个配对列表$(g, l)$， 在哪里$g$是 gcd 值并且$l$表示对于以当前位置结束的子数组，此 gcd 持续多久。 该列表始​​终被压缩，以便 gcd 值不同并在相等时合并。 我们存储段的原因是许多不同的起始位置共享相同的 gcd 结果。 
2. 对于每个新元素$a[r]$，我们从以下开始创建一个新列表$(a[r], r)$，因为单元素子数组的 gcd 等于其自身。 
3. 我们通过使用 gcd 来扩展之前的 gcd 段$a[r]$。 对于之前的每一对$(g, l)$，我们计算$\gcd(g, a[r])$。 如果此 gcd 等于最后存储的 gcd，我们将合并该段而不是添加新条目。 这种压缩至关重要，因为它确保我们只保留不同的 gcd 值。 
4.同时，我们维护前缀和$pref[i]$这样任何区间的总和$[l, r]$是$pref[r] - pref[l-1]$。 
5.对于每个gcd段$(g, l)$，所有以$r$并从相应的范围开始贡献有效的候选者，如果$g \le sum(l, r)$。 由于总和增加为$l$减少，我们可以有效地检查每个段边界的有效性并累积计数。 
6. 添加结束于的有效子数组的数量$r$到全球的答案。 

为什么这是有效的，因为每个位置只贡献少量的 gcd 段，所以我们永远不会迭代所有的$l$明确地。 

### 为什么它有效

 对于固定端点$r$，考虑所有以$r$。 如果我们根据生成的 gcd 值对起始索引进行分组，则每组都会形成起始位置的连续间隔。 这是扩展下 gcd 的一个已知属性：一旦 gcd 在一定范围内开始稳定，进一步向左扩展就不能重新引入先前较大的 gcd 值。 

这种分段保证算法仅跟踪每个位置的对数 gcd 转换。 由于每个有效子数组都通过其端点精确计数一次$r$及其独特的gcd段，不存在重复计算或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]
    
    # list of (gcd_value, leftmost_start_index)
    cur = []
    ans = 0
    
    for r in range(n):
        x = a[r]
        new = [(x, r)]
        
        for g, l in cur:
            ng = gcd(g, x)
            if ng == new[-1][0]:
                new[-1] = (ng, l)
            else:
                new.append((ng, l))
        
        cur = new
        
        # count valid subarrays ending at r
        for g, l in cur:
            total = pref[r + 1] - pref[l]
            if g <= total:
                ans += (l - (cur[cur.index((g, l)) - 1][1] + 1 if cur.index((g, l)) > 0 else -1))
    
    print(ans)

if __name__ == "__main__":
    solve()
```该代码维护前缀和以允许恒定时间范围和查询。 主循环为每个右端点构建压缩的 gcd 列表。 每次扩展时，我们都会合并相等的 gcd 值，以便列表保持最小。 

最后的计数步骤迭代 gcd 段并检查 gcd 约束是否适用于相应的区间总和。 这个想法是每个段代表一个连续的起始位置块。 

微妙的部分是压缩步骤。 如果不合并相等的 gcd 值，列表将不必要地增长并降低性能。 合并确保只存储有意义的转换。 

## 工作示例

 考虑数组$[1, 2]$。 

为了$r = 0$，我们只有子数组$[1]$。 gcd 为 1，sum 为 1，因此有效。 

为了$r = 1$，我们有子数组$[2]$和$[1,2]$。 gcd值分别为2和1，和为2和3。两者都满足gcd ≤ sum。 

| r | 当前 GCD 段 | 子数组计数 |
 | --- | --- | --- |
 | 0 | (1,0)| [1] |
 | 1 | (2,1), (1,0) | (2,1), (1,0) | [2], [1,2] |

 这显示了单个端点的多个 gcd 状态如何共存。 

现在考虑$[3, 6, 9]$。 

在$r=2$, 以 2 结尾的子数组有 gcd 链$(9), (3), (3)$压缩后，意味着只有两个不同的 gcd 状态很重要。 子数组结构很快崩溃，因为大多数区间的 gcd 稳定为 3。 

## 复杂度分析

 | 测量| 复杂性 | 说明|

 该算法非常适合约束条件，因为每个索引的 gcd 转换数量在实践中很小，并且在理论上是对数有界的。

def run(inp: str) -> str:

    from math import gcd

    n = int(input())
    a = list(map(int, input().split()))
    
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]
    
    cur = []
    ans = 0
    
    for r in range(n):
        x = a[r]
        new = [(x, r)]
        
        for g, l in cur:
            ng = gcd(g, x)
            if ng == new[-1][0]:
                new[-1] = (ng, l)
            else:
                new.append((ng, l))
        
        cur = new
        
        for g, l in cur:
            s = pref[r + 1] - pref[l]
            if g <= s:
                ans += 1
    
    return str(ans)

 ## 边缘情况

 对于像这样的单元素数组$[7]$，该算法初始化一个gcd段$(7,0)$并立即数数。 总和也是 7，因此条件成立，答案为 1。 

对于全一数组$[1,1,1]$，每个扩展都使 gcd 等于 1，而总和严格增加。 段结构会分解为每个端点的单个 gcd 值，并且每个间隔都计算一次。 该算法自然产生$n(n+1)/2$无需特殊处理。 

对于像这样的数组$[8,4,2,1]$，当我们向左延伸时，gcd值逐渐减小。 分段确保每次减少都被准确记录一次，并且不会因重叠间隔而发生冗余重新计算。
