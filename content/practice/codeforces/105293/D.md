 ## 解决方案
 ## 问题理解

 我们从包含从 1 到 n 的数字的多重集开始。 每个操作选取两个当前存在的值 x 和 y，删除这两个值，然后插入它们的差值 x − y。 经过 n − 1 次这样的操作后，只剩下一个数字。 任务是决定我们是否可以选择操作序列以使最终剩余值等于给定的目标 m，如果可以，则输出任何有效的对序列。 

每个操作都会将多重集的大小减少一，因此该过程是一个减法二叉树：每个内部节点将两个值组合成它们的差值，最终的根是以某种顺序重复应用 x − y 的结果。 

约束 n 最大为 2 × 10^5，总和也为 2 × 10^5，这意味着我们必须在每个测试用例的线性时间内构造序列。 任何尝试所有对或回溯的模拟都是不可能的，因为即使 O(n^2) 也已经太大了，甚至 O(n log n) 也必须小心使用，但也是可以接受的。 

一个微妙的边缘情况是减法不是对称的。 选择 x − y 与 y − x 会改变结果符号，因此选取的顺序非常重要。 例如，对于 {1, 2, 3}，不同的配对可以根据结构产生 0、2、-2 或 6。 即使存在解决方案，天真的“贪婪地组合”方法也很容易错过目标 m。 

另一个重要的边缘情况是 n = 2。我们只执行一次操作，根据顺序直接得到 1 − 2 = −1 或 2 − 1 = 1。 所以只有 m = ±1 是可能的； 任何其他 m 必定是不可能立即出现的。 

## 方法

 暴力视图将此视为在 n 个叶子上构建完整二叉树，其中叶子是数字 1 到 n，每个内部节点是两个子树的减法。 可能的树的数量呈超指数增长，甚至忽略树的形状，每个配对决策都会分支成许多可能性。 尝试枚举超出非常小的 n 的所有序列是完全不可行的。 

关键的观察结果是减法是线性的并且允许取消。 每个操作都保留了这样一个事实：最终结果是原始数字与系数在 {−1, 0, 1} 中的整数线性组合，重要的是，我们可以通过选择顺序来自由分配符号。 这表明我们不需要探索结构，只需确保我们可以控制贡献，以便我们达到可实现范围内的任何值。 

建设性的想法是维护一个“主动累加器”，并以受控的方向重复将剩余的数字合并到其中。 通过仔细选择是从累加器中减去还是从累加器中减去，我们可以控制最终结果。 经典的技巧是首先安排过程，以便我们可以产生正累加和负累加，然后调整最终值以准确命中 m。 

具体来说，我们维护一个运行值 cur，最初为 1，对于每个下一个数字 i，我们将其与 cur 组合为 cur − i 或 i − cur，具体取决于我们还需要向 m 移动多远。 因为我们总是可以自由选择顺序，所以我们可以确保 cur 可以在连续区间内的所有整数值上逐步移动。 这将问题转化为使用以可用元素为界的增量构建从 1 到 m 的步行。 

一种更结构化的方式是，我们可以通过适当的符号分配实现区间 [−S, S] 中的任何整数，其中 S 是总和 1 + 2 + … + n。 由于每个数字都可以通过减法顺序进行翻转，因此我们可以有效地选择符号，而操作序列只是实现符号分配的一种方式。

因此，构造简化为构建一系列合并，将 + 或 − 分配给每个数字，以便总和等于 m。 我们贪婪地从最大到最小分配符号以达到 m，然后使用成对减法运算来实现该分配。

 ## 算法演练

 我们将最终值构造为 1 到 n 的有符号和。 

1. 计算总和 S = n(n + 1) / 2 并检查可行性：m 必须位于 [−S, S] 中。 如果没有，我们立即输出NO。 这是必要的，因为 ± 分配的序列不能超出此范围。 
2. 维持当前剩余目标 t = m。 我们将分配从 n 到 1 的符号。 
3. 对于从 n 到 1 的每个值 i，确定其贡献。 如果|t| 很大且 t 为正数，我们尝试通过将 i 设为负数来减去 i，否则我们将其相加。 具体来说，如果 t > 0，我们分配 -i； 如果 t < 0，我们分配 +i； 如果 t = 0，我们可以任意分配，通常是 +i。 然后我们相应地更新 t。 
4. 分配符号后，我们现在将数字分为两组：正面贡献者和负面贡献者。 我们现在模拟多重集运算，以便将所有正数组合成一个值 P，将所有负数组合成一个值 N，并保留其符号含义。 
5. 我们首先通过将其中两个重复组合为 x − y 来合并所有正数，同时确保结果保持正数，然后类似地合并所有负数。 该步骤是通过维护工作列表并重复应用操作来实现的。 
6. 最后，我们将两个累加值结合起来，通过按正确顺序进行最后一次减法来产生最终目标 m。 

关键思想是减法允许我们模拟有符号加法，并且这种构造确保我们永远不会失去表示中间部分和的能力。 

### 为什么它有效

 不变的是，在处理每个数字 i 后，我们维护部分符号和的多重集表示，该表示等于已处理元素及其分配符号的总和。 每个合并操作都保留多重集表示相同总符号值的不变式，因为用 x − y 替换 x 和 y 相当于以线性组合重新分配符号。 由于可行区间中的每个整数都可以表示为 1 到 n 的有符号和，并且每个步骤都保留可表示性，因此我们最终精确地达到 m。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n, m = map(int, input().split())
        S = n * (n + 1) // 2
        
        if m < -S or m > S:
            out.append("NO")
            continue
        
        # We construct a simple greedy sign assignment
        # target t is what we still need to achieve
        tval = m
        sign = [0] * (n + 1)  # +1 or -1
        
        for i in range(n, 0, -1):
            if tval > 0:
                sign[i] = -1
                tval -= -i
            else:
                sign[i] = 1
                tval -= i
        
        # Now tval should be 0
        # Build operations
        pos = []
        neg = []
        for i in range(1, n + 1):
            if sign[i] == 1:
                pos.append(i)
            else:
                neg.append(i)
        
        ops = []
        
        # merge positives
        if pos:
            cur = pos[0]
            for x in pos[1:]:
                ops.append((cur, x))
                cur = cur - x
        
        # merge negatives
        if neg:
            cur2 = neg[0]
            for x in neg[1:]:
                ops.append((cur2, x))
                cur2 = cur2 - x
        
        # combine
        if pos and neg:
            ops.append((cur, cur2))
        
        out.append("YES")
        for a, b in ops:
            out.append(f"{a} {b}")
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先使用总和界限检查可行性，因为任何构造都不能超出可表示的区间。 然后，它贪婪地为每个数字分配一个符号以匹配目标 m，从大到小工作，以便保留剩余的灵活性，同时剩余的间隙可预测地缩小。 

符号分配后，我们显式构造一个有效的减法运算序列，以实现相同的代数组合。 正组和负组独立合并，这样我们就不会过早混合符号冲突，然后最终合并将两个累积值合并到目标表达式中。 

一个常见的陷阱是假设任何配对顺序在选择标志后都有效。 这是错误的，因为过早混合会破坏预期的结构。 正聚合和负聚合的分离可以保持正确性。

 ## 边缘情况

 一种微妙的情况是当 n = 2 时。算法的符号分配仍然会产生有效的分割，但合并步骤必须直接输出单个操作。 例如，如果 n = 2 且 m = 1，我们分配符号，使 2 为负，1 为正，然后合并恰好产生一个减法，从而产生目标。 

另一种边缘情况是 m 等于最大或最小可能总和。 在这些情况下，所有符号都变得一致，并且算法退化为以固定顺序重复减去。 该构造仍然有效，因为合并相同符号组可以保留正确性，但重要的是我们不要尝试过早地混合组，因为这会改变可达的结果集。
