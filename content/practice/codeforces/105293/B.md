 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 目标是重复应用操作，以便最终数组中的每个元素都是非正数。 

一项操作的工作原理如下。 我们选择一个索引$i$。 该位置的元素减少了$a$，而数组中的每个其他元素都会减少$b$， 在哪里$a > b > 0$。 所以每个操作都有一个减法的“全局效应”$b$从一切中减去，再加上额外的减法$a-b$恰好在一个选定的位置上。 

任务是找到所需的此类操作的最少数量，以便没有元素保持严格为正。 

一个关键限制是所有测试用例的元素总数最多为$2 \cdot 10^5$，并且值的大小可以很大，高达$10^9$。 任何尝试直接模拟操作的解决方案都会太慢，因为每个操作都会影响整个数组。 

天真的解释会建议重复应用操作，直到所有值都变为非正值。 这会立即失败，因为值可能很大并且需要许多操作，使得模拟不可行。 

当所有数字都已经为非正数时，就会出现微妙的边缘情况。 例如，如果数组是`[-1, -2, 0]`，答案为零，因为不需要任何操作。 任何盲目尝试“修复”元素（无论符号如何）的方法都会错误地计算此处的操作。 

另一个重要的角点是只有一个元素为正时。 由于选择其指数使其受到较大的跌幅$a$，而其他人只得到$b$，最优操作的结构变得不对称。 单独处理所有元素将无法捕捉到共同的减少效应。 

## 方法

 蛮力的想法是模拟操作。 在每一步中，我们都会在尝试每种可能的索引选择后重新计算数组$i$，并递归地继续，直到所有值都是非正数，取所有序列中的最小值。 这在原则上是正确的，因为它探索了完整的决策树，但每个操作都是$O(n)$，并且操作数量也可能很大，导致每个测试用例出现指数或至少二次行为。 和$n$最多$2 \cdot 10^5$，这是不可能的。 

关键的观察是每个操作都有两个组成部分：均匀减少$b$整个阵列，并额外减少$a-b$应用于一个选定的位置。 这意味着之后$k$操作中，每个元素都减少了$k \cdot b$，此外，一些元素根据被选为特殊索引的次数而受到额外处罚。 

所以我们不用考虑一项一项的操作，而是可以考虑每个索引被“选择”了多少次。 如果一个元素$c_i$被选中$x_i$次，则其最终值变为：$$c_i - k \cdot b - x_i \cdot (a-b)$$在哪里$k = \sum x_i$。 

我们希望所有最终值都是$\le 0$。 重新排列给出了一个约束：$$x_i \cdot (a-b) \ge c_i - k \cdot b$$这通过以下方式链接所有变量$k$，但是如果我们修复，结构就会简化$k$。 对于固定数量的操作，我们可以计算每个元素需要多少“额外提升”，然后检查所需的总提升是否可以分布在$k$运营。 

这导致了贪婪的可行性检查：对于给定的$k$，计算每个元素必须被选为特殊索引多少次才能将其推为非正数，对这些要求求和，并检查它是否符合$k$。 由于可行性是单调的$k$，我们可以二分查找答案。

 ## 算法演练

 1. 修复候选操作数$k$。 每个元素都会减少$k \cdot b$自动地。 

这隔离了唯一剩下的自由：每个索引被选为“特殊”元素的频率。 
2. 对于每个元素$c_i$，计算全局归约后剩余的正盈余：$$r_i = c_i - k \cdot b$$如果$r_i \le 0$，这个元素已经安全了。 
3.如果$r_i > 0$，确定需要多少额外选择。 每个选择都贡献$a-b$， 所以：$$x_i = \left\lceil \frac{r_i}{a-b} \right\rceil$$这是索引的最小次数$i$必须选择。 
4. 总和$x_i$。 这代表整个阵列中所需的“特殊选择”的总数。 
5. 将这个总数与$k$。 如果总和是$\le k$， 然后$k$操作就足够了，因为我们可以在操作之间分配所有必需的特殊选择。 否则，$k$太小了。 
6.二分查找最小$k$通过可行性检查。 

正确性取决于这样一个事实：每个操作恰好贡献一个单位的“特殊选择预算”，并且一旦全局减少确定，每个元素独立地需要一定数量的这样的单位。 

### 为什么它有效

 修复后$k$，每项操作都恰好贡献了一个全局减少$b$，并且恰好是一个额外的局部减少$a-b$。 这使得问题等同于分布$k$跨索引的相同标记，其中每个索引$i$需要最少数量的令牌才能使其变为非正数。 可行性检查准确地捕获了这种分布是否可能。 自从增加$k$只增加可用代币，同时减少所需代币$r_i$，可行性是单调的，使得二分查找有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(k, a, b, arr):
    need = 0
    diff = a - b
    for x in arr:
        r = x - k * b
        if r > 0:
            need += (r + diff - 1) // diff
    return need <= k

def solve():
    t = int(input())
    for _ in range(t):
        n, a, b = map(int, input().split())
        arr = list(map(int, input().split()))

        lo, hi = 0, 2 * 10**9

        while lo < hi:
            mid = (lo + hi) // 2
            if can(mid, a, b, arr):
                hi = mid
            else:
                lo = mid + 1

        print(lo)

if __name__ == "__main__":
    solve()
```功能`can`对固定数量的操作的可行性条件进行编码。 减法`x - k * b`捕获所有操作的统一效果。 只有正残差很重要，因为非正值已经得到满足。 

吊顶划分`(r + diff - 1) // diff`计算一个元素必须被选择多少次才能完全补偿其剩余的积极性。 将所有要素相加即可衡量对特殊选择的总需求。 

二分搜索范围设置得很大，因为答案最多是$\max(c_i) / b$，并且在约束下使用大的固定上限是安全的。 

## 工作示例

 考虑一个数组`[5, 1, 4]`和`a = 5`,`b = 2`。 

我们检查不同值的可行性$k$。 

| k | 全球减少$k \cdot b$| 残差$r_i$| 所需提升| 总需求| 可行|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 2 | [3, -1, 2] | [1, 0, 1] | 2 | 没有 |
 | 2 | 4 | [1, -3, 0] | [1, 0, 0] | 1 | 是的 |

 为了$k = 1$，两个元素仍然需要至少一个额外的选择，但只有一个操作存在，所以它失败了。 为了$k = 2$，总体只需要多选一项，就合适了。 

现在考虑`[3, 6, -1]`具有相同的参数。 

| k | 全球减少| 残差| 所需提升| 总需求| 可行|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 2 | [1, 4, -3] | [1, 2, 0] | 3 | 没有 |
 | 2 | 4 | [-1, 2, -5] | [0, 1, 0] | 1 | 是的 |

 第二个元素主导需求，并且增加$k$减少了残差和所需的提升，显示出单调结构。

 ## 边缘情况

 当所有元素都已为非正数时，可行性检查立即返回 true$k = 0$因为每一个残差$r_i \le 0$，因此不需要累积所需的提升。 因此二分搜索收敛于零运算。 

当只有一个元素为正时，比如说`[10, -5, -2]`，该算法将问题简化为查找必须选择单个索引的次数。 每个操作都有助于它$a$相对于$b$，但公式仍然正确地将所需的提升计算为其残差的上限除法，并且二分搜索收敛到将其推至零以下所需的确切最小操作数。 

当所有元素都相等且为正值时，可行性检查会均匀分配所需的提升。 为了`[x, x, x]`，每个元素独立计算相同的要求，并且总和约束捕获了操作共享的事实，从而防止在独立求解每个元素时发生低估情况。
