 ## 问题理解

 我们得到一个 2 量子位系统，保证恰好处于四个相互正交的量子态之一。 这些状态中的每一个都是两位信息的不同编码，但不在标准计算基础中。 相反，这些状态是纠缠变体，如果不首先应用适当的变换，就无法通过在通常的基础上立即测量每个量子位来区分它们。 

任务是在两个量子位上执行一系列允许的量子操作，然后进行测量，并确定性地恢复我们给出的四种状态中的哪一种。 输出是 0 到 3 之间的整数，标识初始状态。 测量结束后，测量后状态无关紧要。 

尽管量子比特数很小，并且不存在通常意义上的经典输入大小，但隐含的约束是我们只有恒定时间的量子操作和单次测量。 这排除了任何试图以概率方式“学习”状态或对其进行重复采样的策略。 我们必须将四个给定的正交状态转换为计算基础，以便一次测量就能揭示答案。 

关键的微妙之处在于，每个量子位的 Z 基础上的简单测量无法区分状态，因为它们与该基础不对齐。 跳过基础转换的粗心方法将使状态不可预测地崩溃，并将多个案例合并到相同的观察结果中。 

## 方法

 暴力思维是尝试重复测量、重置和重新准备系统以推断结果的概率。 这从根本上与这里的模型不兼容，因为我们只获得状态的单个副本，并且测量具有破坏性。 即使可以重复，每次测量也只能提供有关纠缠系统的部分信息，因此区分四个正交纠缠态将需要重建完整的基础分解，这是不必要的开销。 

重要的观察结果是，四种可能的状态形成了众所周知的 2 量子位空间的正交基，特别是贝尔基。 如果我们应用贝尔基准备电路的逆，这些状态就变得微不足道了。 该逆电路将每个纠缠态映射到不同的计算基础状态 |00⟩、|01⟩、|10⟩ 或 |11⟩。 

实现此目的的标准转换是首先将 CNOT 门从第一个量子位应用到第二个量子位，然后将 Hadamard 门应用到第一个量子位。 经过这种转换后，计算基础上的测量完全识别了原始状态。

 ## 算法演练

 我们将推理描述为将未知纠缠基转换为标准基的转换。 

1. 应用 CNOT 门，其中量子位 0 作为控制，量子位 1 作为目标。 此步骤通过将相关信息转移到目标量子位中来部分解开两个量子位。 
2. 将 Hadamard 门应用于量子位 0。此步骤将剩余的叠加旋转到计算基础中，分离区分两个“正/负”变体的相位信息。 
3. 在计算基础上测量两个量子位，获得两个经典位 b0 和 b1。 
4. 使用由问题陈述中四个状态的原始定义确定的固定映射，将生成的位串解释为 0 到 3 范围内的索引。 

### 为什么它有效

四个输入状态形成通过对计算基础应用固定酉变换而获得的标准正交基础。 Hadamard 后面的序列 CNOT 正是该变换的逆过程。 由于量子演化是酉的，因此应用逆电路将每个基态映射到唯一的计算基向量，而不混合振幅。 这保证了测量产生唯一标识原始状态的确定性两位结果。 

## Python 解决方案

 虽然问题是在 Q# 中陈述的，但底层逻辑是固定的量子电路，然后是测量。 以下是测量后决策逻辑的 Python 风格概念表示。```python
import sys
input = sys.stdin.readline

# In an actual quantum environment, qs is a 2-qubit register.
# We apply:
#   CNOT(qs[0], qs[1])
#   H(qs[0])
#   measure both qubits

def Solve(qs):
    # Pseudocode for quantum operations:
    # CNOT(qs[0], qs[1])
    # H(qs[0])
    # b0 = M(qs[0])
    # b1 = M(qs[1])

    # In an abstract sense, after the circuit:
    # (b0, b1) is a deterministic encoding of the input state.

    b0 = M(qs[0])
    b1 = M(qs[1])

    return (b0 << 1) | b1
```在真正的 Q# 实现中，关键部分是门的排序：CNOT 必须在 Hadamard 之前应用，因为反转它们会将未计算分解为计算基础。 测量必须在两次转换之后进行； 提前测量会破坏正确辨别所需的纠缠结构。

 ## 边缘情况

 一个重要的边缘情况是仅在全球阶段不同的国家之间可能会出现混淆。 例如，两个纠缠态可能在代数上看起来不同，但在电路后表示相同的测量分布。 该算法正确地处理了这个问题，因为全局阶段没有可观察到的影响； 应用 CNOT 和 Hadamard 后，两者在测量前都映射到相同的计算基向量。 

另一个微妙的情况是测量太早。 如果在应用两个门之前进行测量，纠缠结构就会被破坏，并且结果在多个状态上变得均匀随机。 正确的序列可确保在发生任何崩溃之前将所有区别信息转换为经典位。
