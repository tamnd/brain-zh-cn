 ## 解决方案
 ## 问题理解

 我们得到一个保证处于两种可能的纯状态之一的量子位。 一种状态是经典基础状态$|0\rangle$，另一个是叠加$|+\rangle = (|0\rangle + |1\rangle)/\sqrt{2}$。 任务不是确定性地区分它们（这是不可能的，因为这些状态不是正交的），而是设计一种测量程序，有时会产生有保证的正确答案，否则会报告不确定性。 

操作的输出必须是三个整数之一。 仅当算法在逻辑上确定输入状态为时才允许值 0 或 1$|0\rangle$或者$|+\rangle$分别。 值 -1 表示过程拒绝猜测的不确定测量。 

重要的约束是许多独立运行的行为。 大多数时候我们可以不确定，但我们必须确保我们永远不会输出错误的确定答案。 此外，我们不能过于保守：不确定的结果必须最多占运行的 80%，并且在相同输入概率下，必须以至少 10% 的频率正确识别两种状态中的每一种。 

时间限制和内存限制在经典意义上是无关的，因为核心工作是单量子位操作，然后是测量，但逻辑结构必须尊重量子测量规则。 

一种简单的方法会尝试直接在计算基础上进行测量并根据概率进行猜测。 这会立即失败，因为两种状态都可以以非零概率产生两种测量结果。 例如，测量$|+\rangle$在计算基础上以 1/2 的概率产生 0 和 1，因此从结果到状态的任何确定性映射偶尔都会出错。 

如果我们总是尝试根据给定测量结果的最可能状态进行猜测，就会出现第二个微妙的失败情况。 即使最佳贝叶斯猜测也无法满足“永远不会错误”的要求，因为状态之间的重叠保证了每个固定测量基础的模糊性。 

前进的唯一途径是使用有时会产生某个州不可能产生的结果的测量方法。 这些结果使我们能够绝对确定地证明另一个状态。 

## 方法

 蛮力思维是尝试所有可能的量子测量并针对结果设计决策规则。 从概念上讲，这是搜索将一个量子位映射到三个输出的所有 POVM。 虽然这是正确的抽象，但它在计算上没有意义，更重要的是，它掩盖了关键的结构观察：我们不需要在全局范围内最大化成功概率，我们只需要找到足够频繁地产生明确证书的任何测量。 

关键的观察是，尽管$|0\rangle$和$|+\rangle$不正交，每个状态都有一个与另一个状态正交的方向。 向量正交于$|0\rangle$是$|1\rangle$，和正交的向量$|+\rangle$是$|-\rangle = (|0\rangle - |1\rangle)/\sqrt{2}$。 如果测量产生$|1\rangle$，那么输入不可能是$|0\rangle$，所以一定是$|+\rangle$。 类似地，如果测量产生$|-\rangle$，输入必须是$|0\rangle$。 

困难在于单一的固定基础无法以足够的概率同时产生这两种证明结果。 计算基础上的测量可以证明$|+\rangle$当结果 1 发生时，但没有给出证明$|0\rangle$。 Hadamard 基础上的测量可以证明$|0\rangle$当结果$|-\rangle$发生，但没有给出任何证书$|+\rangle$。 

解决方案是在这两个互补测量之间进行随机化。 以二分之一的概率，我们在计算基础上进行测量，并且只接受结果 1 作为$|+\rangle$。 以二分之一的概率，我们在 Hadamard 基中进行测量，并且仅接受该旋转基中的结果 1，这对应于$|-\rangle$，作为证书$|0\rangle$。 所有其他结果都被视为不确定。 

这种结构保证了正确性，因为每个结论性输出都对应于与错误状态正交的测量结果。 它还保证了足够的频率，因为每个状态都会在随机选择的至少一个分支中以恒定概率触发验证事件。

 ## 算法演练

 我们将该过程描述为应用于单个量子位的操作。 

1. 掷一枚公平的经典硬币来决定使用哪种测量基础。 这引入了状态空间的两个互补证明视图之间的受控权衡。 
2. 如果硬币选择计算基础，则直接在计算基础中测量量子位$|0\rangle, |1\rangle$基础。 如果结果为 1，则输出 1，因为$|0\rangle$在考虑了通过互补分支中的正交性进行认证的逻辑之后，永远不可能产生这种结果。 否则输出-1。 
3. 如果硬币选择 Hadamard 基，则应用 Hadamard 变换来旋转基，使得$|+\rangle$和$|-\rangle$成为计算基础状态。 
4. 旋转后在计算基础上进行测量。 如果在此旋转坐标系中结果为 1，则输出 0，因为这对应于$|-\rangle$方向，正交于$|+\rangle$。 否则输出-1。 
5. 切勿在任一分支中输出相反的标签。 仅当观察到的结果在逻辑上与其他可能的输入状态不兼容时才输出状态。 

正确性取决于这样一个事实：每次我们输出标签时，测量结果都位于与候选状态之一正交的子空间中。 这种正交性使得推论在逻辑上是确定的。 

## 为什么它有效

 不变的是，每个结论性结果都对应于一个投影仪，该投影仪的支持完全排除了两种可能的输入状态之一。 计算基础结果 1 不包括$|0\rangle$，Hadamard 基础结果 1 排除$|+\rangle$。 由于量子测量概率由振幅平方给出，因此与某个状态正交的任何结果在该状态下的概率为零，因此报告的标签永远不会不正确。 

基数之间的随机选择确保每个状态都有恒定的概率达到其验证正交结果，从而防止算法经常默认为不确定。 

## Python 解决方案

 实际的 Codeforces 接口是量子感知的，但逻辑结构可以表示为允许的量子操作的经典包装器。```python
import sys
input = sys.stdin.readline

import random

def Solve(q):
    # random choice between two USD measurements
    if random.getrandbits(1) == 0:
        # computational basis measurement
        # measure(q) returns 0 or 1 in Z basis
        res = measure_in_z(q)
        if res == 1:
            return 1
        return -1
    else:
        # Hadamard basis measurement
        apply_h(q)  # rotate basis
        res = measure_in_z(q)
        if res == 1:
            return 0
        return -1
```该结构依赖于两个基元：直接 Z 轴测量和哈达玛旋转测量。 Hadamard 分支将$|-\rangle$方向为可检测的计算基础结果。 

一个微妙的实现细节是每个分支中只有一个结果被视为结论性的。 这种不对称是故意的。 尝试解释这两个结果会重新引入错误的猜测，因为其中一个结果总是与两个候选状态重叠。 

## 工作示例

 考虑状态中的输入量子位$|0\rangle$。 该算法有时选择 Z 基，有时选择 H 基。 

在Z分支中，测量总是返回0，这是不确定的。 在 H 分支中，应用 Hadamard 映射$|0\rangle$到$|+\rangle$，并且测量以相同的概率返回 0 或 1。 当结果为1时，我们输出0。 

| 步骤| 基础| 测量前状态| 测量结果| 输出|
 | ---| ---| ---| ---| ---|
 | 1 | Z| ( | 0\角度) | 0 |
 | 2 | 哈 | ( | +\rangle) | 1 |

 这表明该算法永远不会错误地为此状态输出 1。 

现在考虑输入$|+\rangle$。 

| 步骤| 基础| 测量前状态| 测量结果| 输出|
 | ---| ---| ---| ---| ---|
 | 1 | Z| ( | +\rangle) | 0 或 1 |
 | 2 | 哈 | ( | 0\角度) | 总是 0 |

 这里，只有 Z 基础结果 1 是决定性的，产生输出 1，而所有 H 基础结果都是不确定的。 

这些痕迹显示了每个州如何拥有独特的认证事件。

 ## 边缘情况

 一种边缘情况是量子位总是在不对其进行认证的分支中进行测量。 例如，如果输入是$|0\rangle$并且算法反复选择Z基分支，所有结果都是不确定的。 这是预期的行为，并且不违反约束，因为随机性确保哈达玛分支仍然以恒定的频率发生，偶尔产生结论性结果。 

另一个边缘情况是当输入是$|+\rangle$并选择 Hadamard 分支。 在这种情况下，旋转基础上的测量结果始终为 0，这是不确定的。 这是安全的，因为 Z 基础分支仍然以 1/2 的概率产生结果 1，这提供了有效的识别事件。 

最后一个微妙的例子是误解了哪些结果正在证明。 该算法绝不能将高概率结果解释为证据。 只有正交结果才具有确定性。 将它们混合起来会导致错误的答案，并立即违反“永不错误”的要求。
