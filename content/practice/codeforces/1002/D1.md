 ## 解决方案
 ## 问题理解

 我们得到一个非常小的量子系统，由两部分组成：一个 N 个量子位的输入寄存器和一个输出量子位。 除此之外，我们还得到一个长度为 N 的经典二进制向量 b。任务是实现一个可逆量子变换，将简单的线性布尔函数编码为输出量子位上的相位或位翻转。 

在操作上，预言机应该采取基础状态$|x\rangle|y\rangle$，其中x是由量子位表示的N位串，y是单个量子位，并将其变换为$|x\rangle|y \oplus f(x)\rangle$，其中函数定义为点积模 2：$f(x) = (b \cdot x) \bmod 2$。 具体来说，这意味着我们查看 b[i] = 1 的所有位置 i 并对相应位 x[i] 进行异或。 如果这些选定位的奇偶校验为 1，则翻转 y，否则保持不变。 

重要的限制是 N 最多为 8。这意味着整个状态空间很小，但在量子编程中我们并不是直接模拟状态。 我们必须使用基本量子门构建转换。 这立即排除了任何试图明确检查振幅或枚举基态的方法，因为量子预言必须保持统一并一致地作用于叠加。 

一个微妙的边缘情况来自这样一个事实：输出量子位必须以可逆的方式就地修改。 如果不正确地转换为量子门，计算奇偶校验并覆盖 y 而不取消计算中间结构的朴素经典风格实现将违反可逆性。 另一个潜在的错误是忘记 XOR 累加必须通过 CNOT 结构完成，而不是算术加法，因为量子运算必须保持幺正性。 

## 方法

 强力解释是根据完整的状态向量来思考。 人们可以想象迭代所有 2^(N+1) 个基本状态，为每个 x 经典地计算 f(x)，并相应地翻转幅度。 这在概念上是正确的，因为预言只是基本状态的排列。 然而，状态空间增长为 2^(N+1)，即使这里 N 很小，这种方法从根本上与量子编程模型不兼容，在量子编程模型中，我们必须将变换表示为独立于状态的电路。 

关键的观察结果是函数 f(x) 在 XOR 上是线性的。 每一项 b[i] * x[i] 要么是 x[i]（如果 b[i] = 1），要么是 0（如果 b[i] = 0），最终结果只是对所选位进行异或。 在量子电路中，XOR 自然是使用 CNOT 门来实现的。 从 x[i] 到 y 的 CNOT 将 x[i] 添加到 y 模 2 中，而不破坏可逆性。 因此，b[i] = 1 的每个索引恰好对应于一个针对 y 的 CNOT 门。 

这将问题从“计算一个函数”简化为“应用一组受控 X 运算”，这是线性布尔函数的量子预言的标准形式。

 | 蛮力状态推理 | O(2^N) 概念 | O(2^N) | O(2^N) | 不适用 |
 | 最佳 CNOT 构造 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们直接将预言构建为一系列受控的 NOT 操作。 

1. 初始化系统而不修改任何量子位。 我们纯粹通过附加门来操作，因为量子操作是转换，而不是返回值的计算。 
2. 迭代从 0 到 N−1 的所有索引 i。 在每个索引处，检查 b[i] 是否等于 1。这确定 x[i] 是否对 XOR 和有贡献。 
3. 如果 b[i] 为 1，则应用具有控制量子位 x[i] 和目标量子位 y 的 CNOT 门。 这确保了当 x[i] 为 1 时 y 被精确翻转。 
4. 如果 b[i] 为 0，则不执行任何操作。 这相当于在布尔表达式中将 x[i] 乘以零，这对奇偶校验没有任何贡献。 
5. 处理完所有索引后，输出量子位 y 包含所有选定 x[i] 的异或，即 f(x)。 输入寄存器保持不变，保持可逆性。 

### 为什么它有效

 函数 f(x) 是位子集上的奇偶校验函数。 每个 CNOT 门将单个位的异或运算贡献给目标量子位。 由于 XOR 具有关联性和交换性，因此应用这些 CNOT 的顺序并不重要，并且重复应用可以正确累积奇偶校验。 不变量是，在处理前 k 个索引后，目标量子位 y 等于其原始值 XOR 所有 i < k 的贡献，其中 b[i] = 1。此不变量一直延伸到 k = N，此时 y 包含模 2 的完整点积。 

## Python 解决方案

尽管原始签名是 Q#，但我们用相同门序列的 Python 风格伪代码表示来表达电路逻辑。```python
import sys
input = sys.stdin.readline

# In actual Q# this would be CNOT operations; here we just show structure.

def solve(x, y, b):
    # x: list of control qubits
    # y: target qubit
    # b: binary vector

    for i in range(len(b)):
        if b[i] == 1:
            CNOT(x[i], y)
```在真正的 Q# 实现中，CNOT 是 Microsoft.Quantum.Primitive 提供的标准受控非运算。 逻辑是相同的：b 中的每个有效位恰好触发 y 上的一次受控翻转。 不需要临时量子位，也不需要取消计算，因为每个操作都是自逆的。 

主要的微妙之处在于确保目标量子位始终是 y 而不是 x[i] 之一，因为翻转输入量子位会破坏预期的映射。 

## 工作示例

 考虑一个简单的情况，其中 N = 3、b = [1, 0, 1] 和 x = (x0, x1, x2)。 

最初，y 是状态|y⟩ 中的某个量子位。 

| 步骤| 我| b[i] | 运营| y 状态效应 |
 | ---| ---| ---| ---| ---|
 | 1 | 0 | 1 | CNOT(x0, y) | CNOT(x0, y) | y ← y ⊕ x0 |
 | 2 | 1 | 0 | 无 | y 不变 |
 | 3 | 2 | 1 | CNOT(x2, y) | CNOT(x2, y) | y ← y ⊕ x2 |

 最终结果是 y ⊕ x0 ⊕ x2，与 f(x) 匹配。 

现在考虑 b = [0, 0, 0]。 不应用任何操作，因此 y 对于所有输入保持不变。 这与点积始终为零的事实相匹配。 

这两个示例演示了预言机的激活和无效行为，具体取决于 b 的结构。

 ## 边缘情况

 一种边缘情况是 b 的所有条目均为零。 在这种情况下，循环运行但永远不会触发 CNOT，因此 y 对于所有基础状态保持不变。 这正确地表示了常数零函数。 

另一种边缘情况是所有条目都是一个。 该算法将 CNOT 从每个 x[i] 应用于 y。 对于任何输入，y 成为所有位的奇偶校验，这与完整的点积定义相匹配。 

最后一个微妙的情况是 N = 1。该电路简化为单个条件翻转，即单个 CNOT 门。 这证实了结构可以干净地缩小，而不需要特殊处理。
