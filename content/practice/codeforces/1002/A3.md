 ## 解决方案
 ## 问题理解

 我们正在量子环境中工作，其中初始寄存器$N \le 8$量子位从全零计算基础状态开始。 除了该寄存器之外，我们还获得了两个长度相等的经典位串，每个位串都描述了这些量子位的有效基础状态。 保证两个位串至少在一个位置上不同，这意味着它们在计算基础上表示两个不同的基础向量。 

任务是将全零量子态转换为这两个基本态的等量量子叠加。 换句话说，在应用该操作之后，系统应该处于测量它以概率产生第一个比特串的状态$1/2$和第二个位串的概率$1/2$，两个幅度之间具有适当的相位一致性。 

这里的重要限制是我们不允许计算振幅或模拟量子态。 我们必须使用基本量子门（例如 X、H 和受控非运算）构建目标变换。 

自从$N$最多为 8，任何线性或二次解$N$速度非常快。 真正的挑战不是性能，而是构建一个正确的电路，该电路能够准确地产生两个叠加的计算基础状态，而不会引入意外的额外状态。 

当两个位串在多个位置不同时，就会出现微妙的边缘情况。 一种幼稚的方法可能会尝试“独立地将位翻转为叠加”，但这会创建所有位的叠加$2^k$不同位置的组合，这是不正确的。 例如，如果我们尝试哈达玛所有不同的量子位，我们将生成四个或更多状态而不是两个。 正确的转换必须关联量子位，以便仅存在两种全局配置。 

如果我们忽略初始基础对齐，则会出现另一种故障模式。 如果我们尝试直接构建两个给定位串的叠加$|0^N\rangle$，除非我们首先将问题标准化为规范形式，否则我们将面临构建错误阶段结构的风险。 

## 方法

 蛮力思维是明确构建一个映射的电路$|0^N\rangle$到每个目标基础状态，然后以某种方式“添加”它们。 在经典模拟中，这对应于构建大小为的完整状态向量$2^N$，将两个条目设置为$1/\sqrt{2}$，并标准化。 该方法在概念上可行，但不是有效的量子构建路径，因为它假设直接幅度操纵。 即使我们将其解释为电路综合，枚举所有基本状态并强制执行幅度也需要指数推理$2^N$状态，考虑到目标的结构，这是不必要的。 

关键的结构观察是两个目标状态仅通过固定位掩码不同。 如果我们将两个字符串的按位异或表示为$d$，那么可以通过精确翻转其中的位来从另一种状态获得$d$有的。 这将问题简化为准备叠加$|0^N\rangle$和$|d\rangle$，首先将其中一个基础状态映射为零之后。 

这是核心简化：我们不考虑两个任意的位串，而是移动坐标系，使其中之一成为原点。 那么问题就变成了构造一个双状态叠加，其中一个分量是全零状态，另一个是已知的位掩码配置。 该结构可以使用单个 Hadamard 和 X 门的受控传播来实现。

 ## 算法演练

 我们通过将其还原为规范叠加态然后构建受控纠缠模式来构建所需的量子态。 

1. 识别两个位串不同的所有位置。 这定义了一个位掩码$d$。 该掩码精确编码了哪些量子位必须翻转才能将一种基本状态转换为另一种基本状态。 
2. 将 X 门应用于第一个位串为 1 的所有量子位。这映射了对应于`bits0`进入全零状态。 这一步至关重要，因为它将问题简化为标准起点。 
3. 应用相同的 X 门`bits1`隐式地通过转换：在步骤 2 之后，第二个状态变为位掩码$d = bits0 \oplus bits1$。 
4. 选择任意索引$i$这样$d[i] = 1$。 这样的索引必须存在，因为两个位串不同。 
5. 将 Hadamard 门应用于量子位$i$。 这会产生局部叠加$(|0\rangle + |1\rangle)/\sqrt{2}$，它将充当生成两个全局状态的“控制支柱”。 
6. 对于所有其他索引$j \ne i$在哪里$d[j] = 1$，应用由量子位控制的 CNOT 门$i$靶向量子位$j$。 这传播了来自量子位的激发$i$到所有需要的位置，确保所有这些位保持为零或全部翻转在一起。 
7.最后系统处于状态$(|0^N\rangle + |d\rangle)/\sqrt{2}$在改造后的基础上。 应用步骤 2 的逆操作（再次使用相同的 X 门）可恢复原始坐标系，从而产生原始两个基本状态所需的叠加。 

### 为什么它有效

 该构造强制执行严格的二元相关结构：量子位$i$是叠加的唯一来源，所有其他不同的量子位都通过 CNOT 确定性地与其关联。 这保证了状态的计算基础支持恰好包含两种配置。 初始 X 层确保这些配置恰好是原始位串，而不是移位后的版本。 由于所有操作都是单一且可逆的，因此不会在预期的二维子空间之外引入无关的幅度。 

## Python 解决方案

 尽管最初的问题是在 Q# 中提出的，但逻辑结构可以用类似 Python 的量子 API 风格来表达，其中假定原始门可用。```python
import sys
input = sys.stdin.readline

# We assume quantum primitives exist:
# X(q), H(q), CNOT(control, target)

def Solve(qs, bits0, bits1):
    n = len(qs)

    # Step 1: compute difference mask
    diff = [bits0[i] ^ bits1[i] for i in range(n)]

    # Step 2: align bits0 to |0...0>
    for i in range(n):
        if bits0[i]:
            X(qs[i])

    # Step 3: find pivot qubit
    pivot = -1
    for i in range(n):
        if diff[i]:
            pivot = i
            break

    # Step 4: create superposition on pivot
    H(qs[pivot])

    # Step 5: propagate correlations
    for j in range(n):
        if j != pivot and diff[j]:
            CNOT(qs[pivot], qs[j])

    # implicit end of operation
```第一个循环是很容易被忽视的基础对齐步骤。 如果没有它，该构造将产生锚定在错误的计算基点上的叠加。 

主元选择是安全的，因为该问题保证至少有一个不同的位。 任何有效的索引都有效，但选择第一个索引可以保持电路的确定性。 

受控传播步骤可以防止状态爆炸。 如果没有 CNOT 将所有不同的位绑定到单个控制，每个 Hadamard 将独立地将基本状态的数量加倍。 

## 工作示例

 考虑一个具有三个量子位的简单情况。 

输入位串是`bits0 = 000`和`bits1 = 110`。 

计算差异掩模后，我们得到`d = 110`。 

然后我们不应用 X 门，因为`bits0`已经为零。 枢轴被选为索引 0。

 这证实了只出现了两个计算基础状态，并且第二个匹配`bits1`。 

现在考虑一个情况`bits0 = 101`和`bits1 = 011`。 

异或后，`d = 110`。 我们首先将 X 门应用于量子位，其中`bits0`有1s，映射$|101\rangle$到$|000\rangle$。 然后系统的处理方式与前面的示例相同，产生$(|000\rangle + |110\rangle)/\sqrt{2}$。 再次应用 X 门恢复原始坐标系，产生$(|101\rangle + |011\rangle)/\sqrt{2}$。 

第二条轨迹表明，变换在基移下是不变的，并且仅取决于 XOR 结构。

 ## 边缘情况

 一种关键的边缘情况是两个位串仅在一个位置不同。 在这种情况下，XOR 掩码只有一个设置位，因此 CNOT 传播循环不执行任何操作。 该算法简化为应用 X 门，然后使用单个 Hadamard，直接产生正确的单量子位叠加，而不会出现意外纠缠。 

另一种边缘情况是第一个位串不全为零。 最初的 X 层保证了这里的正确性。 如果没有它，基于枢轴的构造将在错误的基础原点周围生成叠加。 通过显式映射`bits0`为零，我们确保后续的 Hadamard 和 CNOT 结构在一致的坐标系中运行，并且最终的反转保留了预期的状态对。
