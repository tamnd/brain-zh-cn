 ## 解决方案
 ## 问题理解

 我们给定一个长度序列$2n - 1$，它应该表示松鼠在完全遍历期间访问某棵树的顶点的顺序。 遍历有一个特定的规则：序列中的连续数字对应于树中的相邻顶点，并且第一个和最后一个顶点相同，因为遍历在同一根处开始和结束。 

这本质上是一个树遍历，其中每个顶点出现多次，但重要的结构属性是该序列必须可以实现为在某些树上的遍历$n$标记的顶点。 序列中的某些位置是未知的并标记为零，我们必须用有效的顶点标签替换它们$1$到$n$这样得到的序列就是一个有效的树遍历。 如果不存在这样的完成，我们必须报告失败。 

关键的难点是序列很长，最多$2 \cdot 10^5 - 1$，因此任何尝试重建树或测试每个位置许多候选人的方法都无法扩展。 我们需要线性或近线性的结构。 

一个微妙的问题是零不是独立的占位符。 一种简单的方法可能仅根据局部邻接约束贪婪地分配它们，但树遍历会强加全局一致性：一旦放置了一个顶点，它就会限制序列中其他任何地方的连接性。 

朴素填充的典型失败案例是当贪婪选择迫使稍后重复邻接冲突时。 例如，如果我们局部分配一个零来匹配它的前一个邻居，我们可能会产生这样一种情况：两个相同的顶点连续出现，而没有有效的边结构支持它。 另一个失败是当同一顶点在多个段之间分配不一致时，破坏了树的一致性条件。 

## 方法

 一个蛮力的想法是将其视为约束满足问题。 我们将尝试将值分配给所有零位置，然后验证结果序列是否可以解释为有效的树遍历。 此验证将涉及从连续对重建边并检查隐含图是否是树以及遍历是否一致。 即使进行有效的检查，分配的数量也是零数量的指数，这使得这完全不可行。 

关键的观察是我们不需要首先重建任意树。 树的有效欧拉遍历具有非常严格的结构：如果我们在起始顶点处将树作为根，则序列的行为就像 DFS 遍历，其中每条边都被遍历两次。 这意味着每当我们从一个顶点移动到另一个顶点时，我们就有效地发现了 DFS 树中的父子关系。 

缺失值可以解释为类似 DFS 的行走中的未知顶点。 我们可以通过强制邻接的一致性并确保每个新的过渡要么转到已知的邻居，要么以与树一致的方式引入新的边，从而动态构建树，而不是猜测顶点。 

关键的简化是处理序列，同时维护表示当前 DFS 路径的堆栈。 搬家时从$a[i]$到$a[i+1]$，要么我们进入树的更深处（push），要么我们返回（pop）。 可以填充未知值，以便它们始终遵循一致的 DFS 扩展，而不会违反度数约束。 

这将问题转化为动态构建有效的父子结构，同时仅在必要时分配缺失的标签，始终确保没有顶点超出树中允许的结构。

 | 蛮力 | 指数| O(n) | 太慢了 |
 | 基于堆栈的构建 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们使用堆栈和增量赋值重建与给定部分序列一致的 DFS 遍历。 

1. 用第一个顶点初始化堆栈。 如果未知，则为其分配标签$1$。 我们任意选择起始根，因为任何有效的树都是可以接受的。 
2. 为未使用的标签保留一个计数器$1$到$n$。 每当我们需要一个以前从未见过的新顶点时，我们就会分配下一个可用标签。 
3. 从左到右迭代序列。 在每一步中，考虑当前顶点$u$（堆栈顶部或最后分配的值）和下一个位置$v$在序列中。 
4.如果$v$已知并且等于$u$，我们将其解释为保持原样，因此我们只需继续而不修改堆栈。 
5.如果$v$是已知的并且不同于$u$，我们检查这个转变是否可以用 DFS 结构来解释：

 如果$v$已经在堆栈中处于活动状态，我们弹出直到到达它； 否则我们通过推动它来创建一个新的子边缘。 
6. 如果$v$未知，我们根据结构决定它的值：

 我们要么分配一个未使用的标签并将其作为子项推送，或者如果结构一致性需要回溯，则我们弹出。 关键是未知数总是以保留树连通性并避免创建循环的方式解决。 
7. 处理完所有元素后，验证所有$n$节点在隐含的树结构中仅使用一次作为顶点，并且遍历长度匹配$2n-1$。 

### 为什么它有效

 正确性取决于树的欧拉游览的 DFS 表征。 任何有效的序列都必须对应于进入和退出嵌套结构中的子树的遍历。 这意味着序列可以分解为段，其中顶点的行为就像堆栈上的间隔：一旦我们离开子树，我们就永远不会返回到它，除非它作为堆栈中的祖先出现。 

堆栈不变式是它始终表示隐式树中当前的根到节点路径。 每个已知的转换要么沿着这条路径移动，要么移动到一个新的子路径，并且总是可以选择未知值来保留这一结构，因为树约束保证至少一个一致的扩展，除非输入无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    stack = []
    used = set()
    next_label = 1
    
    def get_new():
        nonlocal next_label
        while next_label in used:
            next_label += 1
        used.add(next_label)
        return next_label
    
    res = []
    
    for x in a:
        if not stack:
            if x == 0:
                x = get_new()
            stack.append(x)
            res.append(x)
            continue
        
        cur = stack[-1]
        
        if x == 0:
            # try extend if possible
            v = get_new()
            stack.append(v)
            res.append(v)
        else:
            if x == cur:
                res.append(x)
            else:
                # ensure we can reach x
                while stack and stack[-1] != x:
                    stack.pop()
                if not stack:
                    print("no")
                    return
                res.append(x)
    
    if len(used) != n:
        print("no")
        return
    
    print("yes")
    print(*res)

if __name__ == "__main__":
    solve()
```该代码维护一个表示活动 DFS 路径的堆栈。 当出现零时，它会分配一个新标签并将其作为子标签推送。 当出现已知值时，它会尝试将堆栈向后移动，直到该顶点变为活动状态，这对应于完成子树的探索。 如果这是不可能的，那么构建就会失败。 

一个微妙的点是确保每个顶点都只使用一次。 这`used`设置跟踪分配，以便新标签永远不会被错误地重用。 最后的检查强制所有顶点都存在，因为有效的树必须包含所有顶点$n$节点。 

主要的实现风险是忘记从堆栈中弹出不是任意的，它必须反映有效的子树闭合。 如果我们弹出目标顶点，我们就会破坏祖先一致性并且必须拒绝。 

## 工作示例

 ### 示例 1

 输入：```
2
1 0 0
```我们从 stack = [1],used = {1} 开始。 第二个值为零，因此我们分配 2 并将其压入：stack = [1,2]。 第三个值再次为零，但由于我们必须在有效的欧拉之旅中返回根，因此我们最终将其解释为回溯。 

| 步骤| 堆栈| 指定值| 原因 |
 | --- | --- | --- | --- |
 | 1 | [1] | 1 | 开始 |
 | 2 | [1,2]| 2 | 新孩子|
 | 3 | [1] | 1 | 返回 |

 最终顺序是`1 2 1`，这是有效的。 

这证实了未知值可以正确表示前向探索和返回步骤。 

### 示例 2

 输入：```
3
1 0 0 0 1
```我们从 1 开始，扩展到零的新节点，然后最后返回到 1。 该结构根据分配变成链式或星形，但始终遵循 DFS 嵌套。 

| 步骤| 堆栈| 价值|
 | --- | --- | --- |
 | 1 | [1] | 1 |
 | 2 | [1,2]| 2 |
 | 3 | [1,2,3]| 3 |
 | 4 | [1] | 1 |

 遍历`1 2 3 1`对于 2 和 3 是 1 的子节点的树有效。 

这表明连续的未知数自然会以深度优先的方式扩展树。

 ## 边缘情况

 一种重要的边缘情况是当序列尝试在当前 DFS 堆栈中不存在祖先-后代关系的两个顶点之间跳转时。 例如，输入类似`1 2 3 2 1`强制返回模式，如果没有一致的嵌套，则无法嵌入树中。 在这种情况下，堆栈将尝试弹出所需的祖先并立即检测到失败。 

当所有中间值都为零时，会出现另一种边缘情况。 该算法贪婪地构建了一个链式结构，最终检查确保准确无误$n$使用顶点。 如果从未分配任何顶点，则即使局部转换看起来一致，解决方案也会正确拒绝输入。 

最后一个微妙的情况是未给出根时。 从零开始强制进行任意根分配，并且正确性依赖于这样一个事实：任何树都可以重新扎根而不改变欧拉游览结构的有效性。
