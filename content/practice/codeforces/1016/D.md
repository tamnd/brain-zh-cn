 ## 解决方案
 ## 问题理解

 我们被要求构建一个大小为 n × m 的网格，其中每个单元格包含一个非负整数。 我们预先知道的不是网格本身，而是由它衍生出的两组约束：每行的异或是固定的，每列的异或是固定的。 

换句话说，每一行 i 都有一个目标值 ai，它必须等于该行中所有元素的异或，每列 j 都有一个目标值 bj，它必须等于该列中所有元素的异或。 任务是决定是否至少有一个网格可以同时满足所有这些行和列异或条件，如果可能，则输出任何这样的网格。 

约束很小：n 和 m 最多为 100。这个大小很重要，因为它允许 O(nm) 构造和验证，而无需考虑性能。 任何最多几百万次的操作在这里都是微不足道的，因此问题不在于效率，而在于 XOR 约束的结构一致性。 

关键的微妙之处在于行异或和列异或不是独立的。 贪婪地填充行的天真尝试通常会破坏列异或，反之亦然。 当所有行目标的全局 XOR 与所有列目标的全局 XOR 不匹配时，会出现常见的失败情况。 例如，如果 n = m = 2、a = [1, 0]、b = [1, 1]，则行的 XOR 为 1，列的 XOR 为 0，并且任何赋值都无法协调这种不匹配。 

## 方法

 暴力的想法会尝试为所有 nm 单元分配值并验证约束。 每个单元格都有许多可能的值（最多 10^9），因此搜索空间非常大。 即使单独限制位也没有足够的帮助，因为约束是跨行和列耦合的。 彻底的搜索很快就变得不可能。 

关键的结构观察是 XOR 在 GF(2) 上表现出线性。 每个单元格独立地贡献恰好一个行约束和一个列约束。 这建议增量构建矩阵，同时确保行和列 XOR 之间的一致性。 

我们可以将除最后一行和最后一列之外的所有单元格视为自由变量。 一旦这些被修复，最后的列条目将由行异或约束强制，最后的行条目将由列异或约束强制。 唯一剩下的问题是右下单元格在两种定义下是否一致。 这将整个可行性条件简化为涉及行和列目标的全局总和的单个 XOR 等式。 

更具体地说，我们首先将任意值分配给左上角的 (n-1) x (m-1) 子矩阵，通常全为零。 然后我们计算最后一列，以便每行异或匹配 ai，并计算最后一行，以便每列异或匹配 bj。 最后，我们验证根据行约束计算出的最后一个单元格是否等于列约束所隐含的单元格。 如果它们匹配，我们就有一个有效的矩阵。

 ## 算法演练

 我们构造矩阵的方式为除最后一行和最后一列之外的所有行和列保留自由，然后使用 XOR 约束来强制一致性。

1. 初始化一个用零填充的 n × m 矩阵。 这提供了一个干净的基线，其中所有行和列的异或都从零开始。 
2. 对于从 0 到 n-2 的每一行 i，计算必须进入单元格 (i, m-1) 的值，以便行 i 的异或变为 ai。 由于该行中的所有其他元素均为零，因此我们只需将该行的最后一个列条目设置为 ai。 
3. 对于从 0 到 m-2 的每一列 j，计算必须进入单元格 (n-1, j) 的值，以便列 j 的异或变为 bj。 同样，由于该列中的所有其他条目均为零，因此我们相应地设置底行条目。 
4. 此时，除了最后一行和最后一列可能同时满足之外，所有约束均已满足。 我们根据最后一行 XOR 条件计算单元格 (n-1, m-1) 的隐含值。 
5. 根据最后一列 XOR 条件独立计算同一单元格的隐含值。 
6. 如果这两个值不同，则不存在一致的矩阵，因为该单个单元以两种不兼容的方式被强制。 
7. 如果它们匹配，则将该值分配给单元格并输出完整矩阵。 

这种结构起作用的原因是，除了最后一行之外的每一行和每一列都使用每个单元格独立校正，因此不会发生干扰。 所有剩余的一致性都被压缩到单个交叉单元中，作为最终的兼容性检查。 

### 为什么它有效

 每个行约束在其最后一个列条目中精确地固定一个自由度，并且每个列约束在其最后一个行条目中精确地固定一个自由度。 通过将所有调整推至边界，我们确保约束除了在一个共享单元之外不会相互作用。 当且仅当确定共享单元的两种方法一致时，该算法才有效，这正是网格上异或系统的全局一致性条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    # build matrix
    c = [[0] * m for _ in range(n)]

    # satisfy all rows except last column
    for i in range(n - 1):
        c[i][m - 1] = a[i]

    # satisfy all columns except last row
    for j in range(m - 1):
        c[n - 1][j] = b[j]

    # compute bottom-right from row condition
    last_row_xor = 0
    for j in range(m - 1):
        last_row_xor ^= c[n - 1][j]
    c[n - 1][m - 1] = last_row_xor ^ a[n - 1]

    # verify column condition
    last_col_xor = 0
    for i in range(n - 1):
        last_col_xor ^= c[i][m - 1]
    col_value = last_col_xor ^ b[m - 1]

    if col_value != c[n - 1][m - 1]:
        print("NO")
        return

    print("YES")
    for row in c:
        print(*row)

def main():
    solve()

if __name__ == "__main__":
    main()
```该构造首先固定一个由零组成的基本矩阵，然后使用每个非最终行的最后一列来强制其行异或。 这避免了调整内部单元的任何需要。 

然后对称地使用最后一行来强制执行列异或。 由于每列中的所有其他条目都已固定，因此每个列约束恰好确定底行中的一个值。 

唯一微妙的点是 (n-1, m-1) 处的交叉单元。 它隐式计算两次：一次根据最后一行异或要求，一次根据最后一列异或要求。 最终的相等性检查确保两个系统兼容。

 ## 边缘情况

 关键的边缘情况是当 n = 2 或 m = 2 时，整个系统崩溃为单个行和列交叉点之间的直接耦合。 该算法仍然表现正确，因为所有调整都强制进入边界单元，并且只有交集保持自由。 

另一种边缘情况是当所有 ai 和 bj 都为零时。 该构造用零填充矩阵，并且最终的一致性检查很容易通过，因为右下单元格的两个计算结果都为零。 

朴素方法的一个容易失败的场景是独立构造行来满足 ai 而不考虑 bj。 例如，用单个非零元素填充每行来修复其 XOR 通常会立即破坏列 XOR。 建设性方法通过确保每列仅在受控位置校正一次来避免这种情况，从而防止级联冲突。
