 ## 解决方案
 ## 问题理解

 我们得到一个基本字符串$s$和一个模式字符串$t$，均由小写字母组成。 任务是回答许多独立的查询，其中每个查询询问：在给定的子字符串内$s$，有多少次$t$显示为连续子串？ 

思考这个问题的一个有用的方法是，每个位置$s$可能是某个事件发生的开始$t$。 但是，对于查询$[l, r]$，我们只计算星星$i$使得完整的匹配$s[i..i+m-1]$完全位于查询范围内。 

约束以非常具体的方式塑造解决方案空间。 字符串长度$n, m \le 1000$足够小，可以检查之间所有可能的对齐情况$s$和$t$是可行的。 另一方面，查询的数量$q \le 10^5$足够大，以至于重新计算每个查询的匹配将立即 TLE。 这创建了一个经典的分离：预处理$s$很便宜，但每个查询的工作必须是$O(1)$或对数。 

每个查询的暴力破解会尝试子字符串中的每个位置并进行比较$t$, 成本核算$O(n \cdot m)$最坏情况下的每个查询。 和$10^5$查询，这变得太慢了。 

当匹配边界穿过查询边缘时，会出现常见的微妙失败情况。 例如，如果$s = \texttt{aaaaa}$和$t = \texttt{aaa}$，出现次数从位置 1、2 和 3 开始。对于像这样的查询$[2,4]$，仅从 2 和 3 开始是有效的，但是在不验证边界条件的情况下检查完整匹配的简单子字符串扫描可能会错误地计算从 1 开始的匹配，即使它超出了查询范围。 

另一个边缘情况是当$m = 1$。 那么每个字符都等于$t$是一个有效的事件，问题就简化为计算范围内的字符。 假设的解决方案$m > 1$由于索引计算，例如$r - m + 1$。 

## 方法

 直接方法很简单：对于每个查询，扫描所有位置$i$从$l$到$r$，并检查是否$s[i..i+m-1] = t$。 这是正确的，因为它直接遵循发生的定义。 然而，每张支票的费用$O(m)$，并且有$O(n)$每个查询可能的起始位置，导致$O(nm)$每个查询和$O(qnm)$总体来说，这是不可行的。 

关键的观察是有效匹配的结构不依赖于查询。 无论$t$匹配位置$i$独立于任何查询； 这仅取决于$s$和$t$。 这意味着我们可以预先计算一个二进制数组$ok[i]$， 在哪里$ok[i] = 1$如果$t$发生在位置开始$i$，否则为 0。 

完成此操作后，每个查询都会简化为对此数组的范围总和查询。 通过前缀和，我们可以回答每个查询$O(1)$。 从字符串匹配到前缀求和的转变消除了对$q$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(q \cdot n \cdot m)$|$O(1)$| 太慢了|
 | 预计算 + 前缀和 |$O(n \cdot m + q)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题转化为静态标记+前缀和查询问题。 

1. 对于每个索引$i$从$1$到$n - m + 1$,检查子串是否$s[i..i+m-1]$等于$t$。 

此步骤识别独立于查询的模式的所有有效起始位置。 
2. 构建数组$ok$长度$n$， 在哪里$ok[i] = 1$如果比赛开始于$i$，否则为 0。 

我们只定义有效的起始位置$n-m+1$，因为除此之外，完整的匹配是不可能的。 
3.构造前缀和数组$pref$， 在哪里$pref[i] = pref[i-1] + ok[i]$。 

这使我们能够计算在恒定时间内任何间隔中有多少场比赛。 
4. 对于每个查询$[l, r]$，将答案计算为$pref[r-m+1] - pref[l-1]$，但前提是$r-m+1 \ge l$。 否则答案为 0。 

该条件确保我们只计算全长符合查询范围的出现次数。 

关键的细节是正确的边界$r - m + 1$。 任何超出此范围的起始位置都会产生一个超过$r$，所以必须将其排除。 

### 为什么它有效

 正确性依赖于以下事实：每次出现$t$由其起始位置唯一确定。 一旦我们预先计算了所有有效的起始索引，每个查询就变成了固定二进制数组上的计数问题。 前缀总和保留了间隔内的精确计数，并且约束$i + m - 1 \le r$通过将右端点限制为来强制执行$r - m + 1$。 这保证了不会发生部分计数或遗漏的事件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m, q = map(int, input().split())
    s = input().strip()
    t = input().strip()

    if m > n:
        # pattern longer than text, no matches possible
        for _ in range(q):
            input()
            print(0)
        return

    ok = [0] * n

    for i in range(n - m + 1):
        if s[i:i + m] == t:
            ok[i] = 1

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + ok[i]

    out = []
    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1

        right = r - m + 1
        if right < l:
            out.append("0")
        else:
            out.append(str(pref[right + 1] - pref[l]))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```预处理循环显式检查所有可能的对齐方式$t$在$s$。 这是安全的，因为两个字符串都足够小，可以$O(nm)$检查是可以接受的。 

前缀数组以标准 1 索引形式构建，以避免计算范围总和时出现差一问题。 每个查询都转换为从零开始的索引，然后将有效的结束位置钳位到$r - m + 1$。 

一个微妙的实现细节是处理以下情况$r - m + 1 < l$，这意味着查询间隔内不能容纳完整的出现。 如果没有此检查，前缀减法将访问无效范围和溢出。

 ## 边缘情况

 当模式长度超过查询段长度时，算法会正确生成零，因为计算的范围$r - m + 1$变得小于$l$，触发早期拒绝条件。 例如，如果$s = \texttt{abcde}$,$t = \texttt{abc}$，查询是$[4,5]$， 然后$r - m + 1 = 3$，小于$l = 4$，所以答案正确为 0。 

对于$m = 1$，每个匹配完全对应于中的单个字符$s$。 预处理步骤标记每个索引，其中$s[i] = t[0]$，并且前缀和可以正确计算任何范围内的出现次数，无需修改。
