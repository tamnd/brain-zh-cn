 ## 解决方案
 ## 问题理解

 我们得到了几组独立的棍子长度。 对于每一组，我们需要选择四根棍子，使它们形成一个矩形，这意味着我们需要两对相等的长度：一对用于高度，一对用于宽度。 每根棍子只能使用一次，而且我们不能拆分棍子。 

在我们可以形成的所有有效矩形中，我们并不是简单地试图找到任何矩形。 我们通过取决于其几何形状的特定表达式来评估每个候选矩形：如果边是$a$和$b$，周长是$2(a+b)$面积是$ab$，所以该值变为$$\frac{P^2}{S} = \frac{4(a+b)^2}{ab}.$$这个表达式严重惩罚了两边之间的不平衡，因为当一侧变得比另一侧大得多时，分母相对于平方和会缩小。 当矩形最接近正方形时，表达式最小化。 

每个测试用例总共最多可以包含一百万根棍子，因此任何解决方案对于每个测试用例都必须基本上是线性的或接近线性的。 任何尝试考虑所有四元组甚至所有对的方法都会太慢。 

一个天真的想法是尝试每对至少出现两次的长度并计算分数。 失败的情况是当一个长度出现多次时，因为从频率中贪婪地选择对而不考虑排序顺序的邻接性可能会错过由稍微不同的边长形成的更好的矩形，这些矩形在数值上更接近。 

例如，如果我们有这样的长度$1,1,100,100,2,2,99,99$，基于频率的贪婪选择可能会跳转到极端对，例如$1 \times 100$，但最佳矩形来自$2 \times 99$或另一个更紧密的配对，具体取决于完整的结构。 关键问题是，最优性取决于价值的接近程度，而不仅仅是丰富性。 

## 方法

 矩形是通过选择两个不同的长度来完全确定的，每个长度至少出现两次。 如果我们知道所有可用的相等棒的候选对，问题就简化为选择两个这样的对来最小化表达式。 

暴力解决方案将首先收集频率至少为 2 的所有长度，然后尝试所有此类长度对。 如果有$k$如此不同的长度，这是$O(k^2)$。 在所有值出现多次的最坏情况下，$k$可能很大，使得这种二次方法太慢。 

关键的观察是成本函数平滑地依赖于货币对$(a,b)$并且最小化时$a$和$b$很接近。 这表明我们不需要所有对，只需要按长度排序的相邻候选。 一旦我们对候选长度进行排序（每个长度作为可用对重复两次），最佳的一对必须来自一个小邻域：要么相同的长度重复四次（一个正方形），要么两个相邻的不同长度。 

因此，我们不是测试所有组合，而是将频率压缩成可用的对并进行本地扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有配对组合进行暴力破解 |$O(k^2)$|$O(k)$| 太慢了 |
 | 排序+检查相邻对候选者 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

### 1. 计算所有棒长度的频率

 我们构建一个频率数组，其值最多为 10000。这已经足够了，因为约束限制了棒的长度。 

我们需要频率，因为矩形需要相等长度的对。 

### 2. 提取可用对

 对于每个长度$x$，我们计算它贡献了多少个不相交对：`freq[x] // 2`。 每对代表一个矩形的一条边。 

我们将每个可用对存储为候选方。 

这里的原因是，矩形每边恰好使用两根棍子，因此我们只关心每个长度可以贡献多少个完整的对。 

### 3. 构建候选方的排序列表

 我们扩展每个长度$x$到其对贡献中，但为了优化，我们最多只需要按候选值排序顺序的每条边长度的两个副本。 这是因为最佳矩形仅取决于排序空间中的相邻值。 

因此，我们构建一个列表，其中每个有效长度出现多次，具体取决于它贡献的对数量。 

### 4. 扫描相邻对以形成矩形

 我们迭代排序的候选列表并获取每个连续的对$(a, b)$。 每对都代表一个可能的矩形。 

我们计算分数：$$(a + b)^2 / (a \cdot b)$$（在缩放常数范围内，比较是等效的）。 

我们追踪迄今为止看到的最好的一对。 

邻接之所以有效，是因为远离亲密关系会增加不平衡，从而使目标更加恶化。 

### 5. 正方形的特殊处理

 如果任意长度至少出现4次，我们就可以直接组成一个正方形。 正方形最小化了具有固定周长到面积行为的所有矩形之间的表达式，因此我们显式检查这种情况。 

### 为什么它有效

 目标函数仅取决于边之间的比率。 对于固定乘积，当边相等时，总和最小。 因此，在所有有效矩形中，最好的矩形必须来自重复的相同值（正方形）或来自按排序顺序最接近的两个可用值。 任何不相邻的选择都可以通过将一侧转向另一侧来改善，减少不平衡并降低目标。 

这创建了局部最优条件：按照可行边长的排序顺序在相邻候选中找到全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))

        freq = [0] * 10001
        for x in arr:
            freq[x] += 1

        sides = []
        square_candidate = None

        for x in range(1, 10001):
            c = freq[x]
            if c >= 2:
                pairs = c // 2
                if pairs >= 2:
                    if square_candidate is None:
                        square_candidate = (x, x)
                if pairs >= 1:
                    # each pair contributes one side
                    sides.append(x)

        sides.sort()

        best_val = float('inf')
        best = None

        def value(a, b):
            return (a + b) * (a + b) / (a * b)

        for i in range(len(sides) - 1):
            a, b = sides[i], sides[i + 1]
            v = value(a, b)
            if v < best_val:
                best_val = v
                best = (a, b)

        if square_candidate is not None:
            print(square_candidate[0], square_candidate[0], square_candidate[1], square_candidate[1])
        else:
            a, b = best
            print(a, a, b, b)

if __name__ == "__main__":
    solve()
```频率阵列将输入压缩为可用的几何构建块。 这`sides`列表表示由配对棒形成的所有可能的矩形边。 

对相邻元素的扫描是核心优化：我们不是测试所有对，而是依靠排序来确保最佳平衡必须位于相邻元素之间。 

方形检查是分开的，因为它代表一种退化但通常是最佳的配置，可能不会出现在相邻对的比较中。

 ## 边缘情况

 一种重要的边缘情况是单个值至少出现四次。 在这种情况下，最佳矩形始终是正方形。 该算法在建立频率时明确地检测到这一点。 例如，输入`5 5 5 5`立即产生两对 5，形成一个正方形并避免与其他值进行比较。 

当多个值恰好形成一对时，会出现另一种边缘情况。 例如`1 1 100 100 2 2 99 99`。 该算法将其压缩为候选边`[1,2,99,100]`。 排序确保邻接比较测试`(1,2)`,`(2,99)`,`(99,100)`，最好的矩形从最接近的一对中出现。 这可以防止算法错误地配对遥远的值，例如`(1,100)`这会产生更严重的不平衡。 

第三种边缘情况是只有一个有效的矩形配置。 邻接扫描仍然会产生至少一个候选对，因为由于保证矩形存在，边列表必须包含至少两个元素。 这确保算法永远不会访问无效索引或产生空输出。
