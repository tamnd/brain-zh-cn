 ## 解决方案
 ## 问题理解

 我们可以将宿舍视为一个有向图，其中每个房间都有一个出边。 从每个房间`i`，鼠标确定性地移动到`a[i]`一秒钟后。 因为每个节点都有一个出边，所以图会分解成链，最终进入有向循环，然后永远留在这些循环内。 

放置在房间中的陷阱会将该房间移至安全状态。 如果老鼠落在一个被困的房间里，这个过程就会停止。 困难在于起始房间是未知的，因此我们必须保证每个可能的起始位置最终都会沿着其确定性行走到达一个被困顶点。 

任务是选择最小化总成本的节点子集，以便此函数图中的每次无限行走最终都会到达至少一个选定的节点。 

约束条件`n ≤ 2 * 10^5`排除任何模拟从每个节点独立开始并向前行走直至检测的解决方案，因为这将是`O(n^2)`在最坏的情况下，当图形形成长链或循环结构时。 我们需要一种线性或近线性图分解方法，通常`O(n)`或者`O(n log n)`。 

一个简单但重要的边缘情况是纯循环。 如果所有节点形成一个循环，并且我们没有在其中放置陷阱，则从该循环开始的鼠标将永远不会被捕获。 例如：

 输入：```
3
5 1 5
2 3 1
```这里`1 → 2 → 3 → 1`。 任何正确的解决方案都必须在这个循环中至少选择一个节点。 只考虑树状结构的粗心贪婪在这里会失败，因为它忽略了循环是封闭递归类的事实。 

另一个微妙的情况是自循环`i → i`。 该节点单独形成一个长度为 1 的循环，如果没有选择它，则从那里开始的鼠标永远不会被捕获。 这迫使每个周期都被至少一个选定的节点“覆盖”。 

## 方法

 关键结构是每个节点都属于一个有向循环，可能有树流入其中。 老鼠一旦进入一个循环，就永远不会离开，因此保证捕获的唯一方法是在每个循环内至少放置一个陷阱。 循环之外的节点（在通向循环的树中）是可选的，因为从它们出发的任何路径最终都会到达它们的循环。 

因此，问题简化为：识别函数图中的所有有向循环，并为每个循环选择其中的最小成本节点。 

暴力策略将从每个节点开始，模拟其路径，并尝试决定必须选择哪些节点，以便拦截所有路径。 这很快就会变得昂贵，因为每次模拟都可以遍历`O(n)`步骤，并对所有起点执行此操作，得到`O(n^2)`。 

改进来自于认识到该图是具有传入树的不相交循环的集合。 每个节点的出度为 1，因此使用 DFS 或入度剥离可以直接进行循环检测。 一旦循环被隔离，每个循环就变得独立：在一个循环中选择节点不会影响任何其他循环。 

因此，我们不是推理路径，而是推理组件。 对于每个周期，我们计算其最小成本节点并将其添加到答案中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n²) | O(n) | 太慢了 |
 | 循环分解| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们在函数图中使用循环检测。 

1. 将所有节点标记为未访问。 我们将每个节点恰好遍历一次作为 DFS 起点。 
2.对于每个未访问过的节点`i`，开始向前走`a[i]`，将节点标记为“在当前递归堆栈中”。 

这个堆栈标记至关重要，因为它允许我们检测何时返回到当前路径上已有的节点，这表示一个循环。 
3. 如果在遍历过程中我们到达的节点已经在当前堆栈中，则我们找到了有向循环。 然后我们从该循环中提取所有节点。 

提取是通过从重复节点开始直到返回到它来完成的。 
4. 对于每个检测到的周期，计算最小成本`min(c[v])`在所有节点上`v`在该循环中，并将其添加到答案中。 

这是正确的，因为我们每个周期只需要一个陷阱，并且选择最便宜的节点可以最大限度地降低成本。 
5. 继续，直到处理完所有节点。 

关键的微妙之处在于，不属于循环的节点最终总是会到达循环，但它们永远不会成为堆栈重新访问冲突的一部分。 只需简单地遍历和标记它们即可确保线性覆盖。 

### 为什么它有效

 每个节点要么位于一个循环中，要么位于恰好通向一个循环的树中。 任何起始位置最终都会达到其循环并且永远不会离开它。 因此，如果我们在每个循环中至少放置一个陷阱，则每次无限行走都保证最终到达被捕获的节点。 相反，如果任何循环没有陷阱，则从该循环开始将永远避免所有陷阱。 因此，该解决方案既是必要的也是充分的，并且最小化每个周期的成本是最佳的，因为周期是独立的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    c = [0] + list(map(int, input().split()))
    a = [0] + list(map(int, input().split()))

    visited = [0] * (n + 1)
    in_stack = [0] * (n + 1)
    ans = 0

    for i in range(1, n + 1):
        if visited[i]:
            continue

        stack = []
        v = i

        while not visited[v]:
            visited[v] = 1
            in_stack[v] = 1
            stack.append(v)
            v = a[v]

            if in_stack[v]:
                cycle = []
                idx = len(stack) - 1
                while stack[idx] != v:
                    cycle.append(stack[idx])
                    idx -= 1
                cycle.append(v)

                ans += min(c[x] for x in cycle)
                break

        for node in stack:
            in_stack[node] = 0

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案维持两种状态：`visited`确保每个节点在全局范围内被处理一次，同时`in_stack`跟踪当前的遍历路径，以便当我们重新访问仍在活动链中的节点时可以立即检测到循环。 

堆栈存储精确的遍历顺序，这允许在发现重复时在线性时间内重建循环。 当我们检测到一个循环时，我们只隔离形成它的节点并计算它们之间的最小成本。 清算`in_stack`完成每个类似 DFS 的步行后，可以防止不同组件之间的错误循环检测。 

## 工作示例

 考虑示例：```
5
1 2 3 2 10
1 3 4 3 3
```我们跟踪循环检测。 

| 开始| 遍历路径| 检测到循环 | 循环节点| 成本选择|
 | ---| ---| ---| ---| ---|
 | 1 | 1 → 1 | 自循环| {1} | 1 |
 | 2 | 2 → 3 → 4 → 3 | 3周期| {3,4} | 3 |
 | 5 | 5 → 3 | 已处理 | - | - |

 这表明节点 1 和循环 {3,4} 定义了所有必要的捕获点。 

另一个例子：```
4
4 1 2 3
```这就形成了一个单循环`1 → 4 → 3 → 2 → 1`。 

| 开始| 遍历路径| 检测到循环 | 循环节点| 成本选择|
 | ---| ---| ---| ---| ---|
 | 1 | 1 → 4 → 3 → 2 → 1 | 全周期| {1,2,3,4} | 分钟（c）|

 该算法正确地将整个图识别为一个循环。

 ## 边缘情况

 自循环节点演示了最简单的循环。 该算法从该节点开始，将其标记在堆栈中，然后立即重新访问自身。 循环提取仅返回该节点，并添加其成本。 没有其他节点干扰，因为没有。 

通向循环的长链显示了为什么非循环节点不做出贡献。 遍历将所有链节点标记为已访问，但从未将它们检测为循环条目，因为它们不会在当前堆栈中重新访问。 只有当达到循环时才会发生重复，并且只有该循环才能得出答案。
