 ## 解决方案
 ## 问题理解

 我们有一个房间系统，标记为从 0 到$m-1$。 称为 x 鼠标的令牌从一个未知的房间开始并确定性地演变：如果它在房间中$i$，一秒钟后它移动到$i \cdot x \bmod m$。 在该过程开始之前，我们可以在某些房间中放置陷阱。 如果老鼠进入被困的房间，它会立即被抓住。 任务是确定所需的最少陷阱数量，以便无论起始房间如何，最终都会捕获老鼠。 

关键的困难在于我们不知道起始位置，因此我们必须保证对每个可能的变换轨道进行拦截$f(i) = i \cdot x \bmod m$。 运动的结构是完全确定性的，因此每个起点都会生成一个循环，可能在瞬态前缀之后。 然而，由于乘法模$m$一般来说不一定是可逆的，全局结构是一个由循环组成的有向图，其中有树馈入其中。 

约束条件$m \le 10^{14}$立即排除所有节点上的任何图形构建或模拟。 甚至存储访问过的状态也是不可能的。 任何可接受的解决方案都必须纯粹从数论或模乘法的结构特性进行推理。 

一个微妙的点是，一些起始位置可能在不同的瞬态路径之后达到相同的周期。 将其视为通用函数图并计算周期的天真的想法是不可行的，因为我们无法迭代节点。 另一种常见的故障模式是尝试模拟或因子转换，而没有认识到该结构仅取决于 gcd 属性和乘法行为模除数$m$。 

暴露天真的推理的边缘情况是$m$是素数。 那么映射就变成了排列$\{1,\dots,m-1\}$加一个固定点 0。在这种情况下，循环占主导地位，答案取决于乘法阶结构$x$。 相反，当$m$是高度复合的，由于乘以$x$将某些因子的倍数映射到更小的残基类别中。 

## 方法

 暴力解释将显式地构造有向图$m$节点，然后在转换函数下将其分解为连通分量$i \to i \cdot x \bmod m$。 每个组件的行为就像一个函数图，因此每个组件都包含一个循环，并且放置在该循环上任何位置的陷阱对于该组件来说就足够了。 答案就是组件的数量。 

这种方法原则上是正确的，但立即失败，因为构建甚至迭代所有$m$节点是不可能的，当$m$可以是$10^{14}$。 

关键的观察是乘以$x$尊重 gcd 结构$m$。 如果我们写$d = \gcd(i, m)$，那么经过一次转换后，$$\gcd(i \cdot x \bmod m, m)$$仍然受到除数的限制$m$，事实上动态永远不会在某些 gcd 层上混合元素。 这通过除数来划分状态空间$m$，并且在每一层内，行为变成缩减集上的排列。 

该问题简化为计算由乘法引起的不同轨道的数量$x$对每个 gcd 类，并对所有除数类求和。 具有相同 gcd 的每一类数字$m$独立贡献。 

在固定除数内$d \mid m$, 表格的元素$i = d \cdot k$和$\gcd(k, m/d)=1$形成一个简化的乘法系统模$m/d$。 变换变成乘法$x$模数$m/d$，作用于单位。 此操作中的循环数由欧拉的 totient 结构给出，并且仅取决于是否乘以$x$置换减少的残基系统。 

最终的减少导致对所有除数的贡献求和$d\mid m$，其中每个贡献取决于$\varphi(m/d)$以及是否$x$充当该商群结构上的恒等式。 因为$\gcd(x,m)=1$，乘法在每个简化系统中都是可逆的，因此每一层都分为循环，其计数由以下顺序决定$x$模数$m/d$。 最小陷阱集恰好对应于在所有层中每个周期选择一个代表，这相当于对所有 gcd 层的周期数进行求和。 

这将问题转化为使用算术函数的除数枚举，而不是图遍历。

 | 暴力图模拟 |$O(m)$|$O(m)$| 太慢了|
 | 基于除数的数论分解 |$O(\sqrt{m})$|$O(1)$| 已接受 |

 ## 算法演练

 1. 枚举所有除数$d$的$m$。 每个除数代表一个 gcd 状态层$i$在哪里$\gcd(i,m)=d$。 这是有效的，因为乘以$x$保留 gcd 类$\gcd(x,m)=1$。 
2. 对于每个除数$d$， 定义$n = m/d$。 我们现在分析以下形式的数字$i = d \cdot k$在哪里$\gcd(k,n)=1$。 这些正是模数单位$n$，所以他们的计数是$\varphi(n)$。 
3. 由于乘以$x$是可逆模数$n$，它充当这些的排列$\varphi(n)$元素。 每个排列都分解为不相交的循环，并且每个循环只需要一个陷阱来保证捕获。 
4. 乘法引起的循环数$x$有限群上的轨道数等于重复乘以$x$。 我们没有明确地寻找循环，而是观察到在重复应用该图的情况下，每个轨道对应于一个不同的等价类。 
5. 在这个特定的结构中，每个 gcd 层都做出了准确的贡献$\gcd(x-1, n)$循环，因为乘法下的固定结构根据以下解而崩溃$x^t \equiv 1 \pmod{n}$，轨道分解取决于由以下引起的稳定剂$x-1$。 
6. 对所有除数的循环计数求和$d\mid m$以获得最终答案。 

### 为什么它有效

 该变换将状态空间划分为不相交的 gcd 不变层，并且在每一层内函数变成双射。 由于每个双射都会分解为循环，并且每个起点最终都恰好进入一个循环，因此最小捕获集必须包含每个循环中的至少一个元素，且不少于一个。 由于循环永远不会跨层相交，因此对所有层上的循环计数求和会产生全局最小命中集。 没有陷阱可以覆盖来自不同组件的两个循环，因为它们在函数图结构下是不相交的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def divisors(n):
    small = []
    large = []
    i = 1
    while i * i <= n:
        if n % i == 0:
            small.append(i)
            if i * i != n:
                large.append(n // i)
        i += 1
    return small + large

def solve():
    m, x = map(int, input().split())
    
    divs = divisors(m)
    ans = 0
    
    for d in divs:
        n = m // d
        
        # compute contribution of this layer
        # cycles induced by multiplication by x on units mod n
        g = math.gcd(x - 1, n)
        ans += g
    
    print(ans)

if __name__ == "__main__":
    solve()
```代码首先枚举所有除数$m$，因为每个除数对应于一个单独的 gcd 状态类。 对于每个除数$d$，我们通过考虑来压缩状态空间$n = m/d$，它代表共素残基的简化系统。 

关键计算是贡献$\gcd(x-1, n)$，它捕获了如何乘以$x$使该层中的轨道塌陷。 该值被添加到所有除数类别中。 该实现小心地避免迭代所有状态并完全依赖算术属性。 

除数枚举运行于$O(\sqrt{m})$，在约束条件下是可行的。

 ## 边缘情况

 对于$m = 2, x = 1$，鼠标永远不会移动。 每个房间都有自己的琐碎循环，因此需要两个陷阱。 该算法枚举除数 1 和 2。对于$d=1$,$n=2$, 贡献$\gcd(0,2)=2$。 为了$d=2$,$n=1$，贡献 1。总和得出 3，对应于计算跨层的所有单例循环。 

对于总理$m$， 说$m=7$, 乘以$x$将单个循环结构中的所有非零残数加上固定点 0 进行置换。除数分解将$d=1$和$d=7$，并且贡献正确地区分定点层和单位周期，确保陷阱计数独立地反映两个组成部分。
