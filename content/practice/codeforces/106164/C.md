 ## 解决方案
 ## 问题理解

 我们被要求构造一个整数的表示$X$使用从 1 开始的自然数的固定前缀。对于某些选定的长度$N$，我们取序列$1, 2, 3, \dots, N$并为每个数字分配一个加号或减号。 所得的签名总和必须等于$X$。 在所有有效的结构中，我们想要尽可能最小的$N$，并且我们还必须输出一个有效的符号分配。 

表达式的结构非常严格。 我们不会选择数字的子集，也不允许重新排序或跳过元素。 从 1 到的每个整数$N$必须恰好出现一次，只有符号是灵活的。 

约束允许最多 1000 个测试用例，总共$|X|$在所有情况下以$2 \cdot 10^5$。 这个界限很重要，因为它意味着任何解都与$X$每个测试用例都是可以接受的，而任何二次方$X$已经没有必要了，但如果仔细实施，仍然可能处于临界点。 一个大致运行的结构$O(\sqrt{|X|})$每个测试用例很容易就足够了。 

一个微妙的一点是，这个问题并不是简单地对称的。 当改变符号翻转贡献时，前缀结构使得可达和形成固定范围的连续范围。$N$，但前提是了解奇偶约束。 

一些边缘情况值得隔离。 

如果$X = 0$, 最小有效值$N$是 1 因为$1$单独不能形成零。 第一个重要的取消发生在$1 - 2 = -1$，所以零至少需要$1 - 2 + 3$，等于 2，所以$N = 3$作品。 

如果$X$量级非常小但为负，总是尝试先添加正数的天真贪婪方法可能会超出范围并需要纠正，必须系统地处理而不是修补。 

如果$X$量级很大，关键难点在于确定最小$N$，之后不再构建标志。 

## 方法

 蛮力观点从修复开始$N$并询问我们是否可以分配标志来实现$X$。 对于给定的$N$， 有$2^N$签署作业。 每个赋值产生一个总和，因此直接枚举即使对于$N = 30$。 

我们可以通过重新构造表达式来减少这种情况。 令总和为$S = 1 + 2 + \dots + N = \frac{N(N+1)}{2}$。 选择符号相当于选择一个数字子集减去其值的两倍$S$，因为每个负号都会翻转一个项$+i$到$-i$，改变贡献$2i$。 因此表达式变为$$X = S - 2 \cdot (\text{sum of chosen subset}).$$这将问题转化为寻找总和的子集$\frac{S - X}{2}$，前提是奇偶校验匹配并且该值是非负的并且最多$S$。 

关键的见解是为了增加$N$，可实现的子集和的范围变得足够密集，我们只需要找到最小的$N$这样$S \ge |X|$和奇偶校验匹配。 一旦满足该条件，建设性的贪婪调整总是可能的。 

一种更直接、更清晰的方法是从添加的所有数字开始，然后有选择地将符号从大到小翻转，直到我们匹配为止，而不是明确地考虑子集和术语$X$。 这是有效的，因为翻转$i$准确地改变总和$2i$，给出了减少差异的确定性方法。 

我们选择最小的$N$这样$S \ge |X|$。 然后我们计算当前的总和$S$并想把它改造成$X$，所以我们需要减少$D = S - X$。 如果$D$是奇数，我们增加$N$直到奇偶校验对齐。 之后我们贪心地减去$2i$尽可能通过翻转标志来消除差异。

 | --- | --- | --- | --- |
 | 暴力枚举|$O(2^N)$|$O(1)$| 太慢了|
 | 前缀 + 贪婪符号翻转 |$O(N)$每次测试 |$O(N)$| 已接受 |

 ## 算法演练

 我们围绕从所有正数开始并校正总和的想法构建解决方案。 

1. 计算绝对值$A = |X|$因为该结构相对于符号是对称的，我们可以稍后调整方向。 
2. 找到最小的$N$使得总和$S = \frac{N(N+1)}{2}$至少是$A$。 这确保了我们有足够的“权重”来达到$X$。 这样做的原因是每个数字最多贡献其全部值，因此总的可达到的调整量呈二次方增长$N$。 
3. 检查奇偶性$S$和$X$。 由于每次翻转都会改变总和$2i$，最终总和的奇偶性由下式确定$S$。 如果$S \bmod 2 \neq X \bmod 2$, 增量$N$直到奇偶校验匹配。 这是必要的，因为否则差异不能表示为偶数增量之和。 
4. 初始化一组符号，假设所有数字都有正向贡献。 
5. 计算所需的差异$D = S - X$。 这是我们需要通过翻转标志来减少的总量。 
6. 迭代自$N$下降到 1。在每一步，如果$2i \le D$, 翻转符号$i$并减去$2i$从$D$。 这种贪婪的步骤是正确的，因为较大的数字可以更有效地减少差异，并且首先使用它们可以最大限度地减少所需的翻转次数。 
7. 处理完所有数字后，$D$变为零，构造的符号模式恰好产生$X$。 

关键的不变量是在处理完所有大于的整数之后$i$, 剩余差值$D$总是可以仅使用数字来表示$1 \dots i$，因为我们只减去与可用翻转相对应的有效偶数增量。 由于我们总是首先使用最大的可行贡献，因此我们永远不会将自己陷入一个较小的数字无法完成余数的角落。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build(x):
    sign = {}
    x_abs = abs(x)

    n = 1
    s = 1
    while s < x_abs:
        n += 1
        s += n

    while s % 2 != x % 2:
        n += 1
        s += n

    # start all positive
    for i in range(1, n + 1):
        sign[i] = 1

    diff = s - x

    for i in range(n, 0, -1):
        if diff >= 2 * i:
            sign[i] = -1
            diff -= 2 * i

    expr = []
    for i in range(1, n + 1):
        if i == 1:
            expr.append("1")
        else:
            if sign[i] == 1:
                expr.append("+" + str(i))
            else:
                expr.append("-" + str(i))

    return n, "".join(expr)

def main():
    t = int(input())
    for _ in range(t):
        x = int(input())
        n, expr = build(x)
        print(n)
        print(expr)

if __name__ == "__main__":
    main()
```该实现直接反映了建设性推理。 第一个循环找到足够大的前缀和，第二个循环修复奇偶校验，因为奇偶校验在符号翻转下不变。 贪婪向后扫描是核心正确性步骤：从大到小迭代可确保每个缩减步骤都具有最大效率，并且不会阻碍未来的可行性。 

重要的一个细节是计算`diff = s - x`使用有符号值`x`， 不是`|x|`，因为目标方向会影响需要进行哪些翻转。 另一个微妙之处是确保我们始终打印从 1 到$N$，即使有些人仍然积极。

 ## 边缘情况

 对于$X = 0$，算法增加$N$直到和与奇偶条件都一致。 开始于$N=1$，总和为 1，不够。 在$N=2$，总和为 3，与 0 的差仍然是奇数，所以我们转向$N=3$，总和变为 6，并且$6 - 0 = 6$是均匀的。 然后贪婪翻转会产生一个有效的配置，例如$1 + 2 - 3 = 0$。 

对于小的负值，例如$X = -1$，该构造避免了尝试的天真失败$N=1$。 相反，它会成长为$N=2$或者$N=3$取决于奇偶校验，确保最终的差异可分解为有效的翻转。 

对于接近最大约束的大值，算法保持稳定，因为$N$只长到大约$\sqrt{2|X|}$，永远不会接近线性尺度$X$，并且贪婪传递总是终止，因为每次翻转都会严格减少剩余的差异。
