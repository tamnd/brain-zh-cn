 ## 解决方案
 ## 问题理解

 我们正在处理一个单行队列$N$人，从前面（位置 1）到后面（位置$N$）。 这一行中有两个具体的人是 Alice 和 Bob，但他们的具体位置没有直接给出。 相反，我们被告知两个相对的观察结果。 

艾丽丝知道有多少人站在她的面前。 如果她看到$A$前面有人，这意味着她在队列中的位置是$A + 1$。 

鲍勃知道有多少人支持他。 如果他看到$B$后面的人，那么他的位置一定是$N - B$，因为他之后的每个人都算在其中$B$人们。 

任务是确定这条线上有多少人严格位于 Alice 和 Bob 之间。 

限制很小：$N \le 1000$。 这个值足够低，即使是直接算术或强力重建位置也可以是瞬时的。 在这里，任何达到二次时间的事情仍然是微不足道的，但问题的结构表明，一旦确定了位置，就可以进行恒定时间计算。 

情况中出现微妙的边缘情况$A + B \ne N - 1$。 如果平等成立，爱丽丝和鲍勃将以某种方式相邻，从而消除了排序中的歧义。 该问题避免了退化边界，但实现仍然必须处理两种可能的排序，因为根据值，Alice 可能位于 Bob 之前或之后。 

一个常见的错误是假设爱丽丝总是在鲍勃之前。 这是不能保证的。 例如，如果$N = 10$,$A = 7$,$B = 1$，Alice 在位置 8，Bob 在位置 9，所以 Alice 在 Bob 之前。 但如果$A = 1$,$B = 7$，Alice 位于位置 2，Bob 位于位置 3，顺序仍然一致，而不同的配置可能会翻转角色。 该公式必须考虑两个方向。 

## 方法

 强力解释将尝试通过迭代队列中满足其约束的所有可能位置来重建 Alice 和 Bob 的有效位置。 我们可以把爱丽丝放在位置$i = A + 1$，鲍勃在位置$j = N - B$，然后显式地严格计算它们之间的索引数量。 这已经足够了，因为两个位置都是根据输入唯一确定的，因此一旦我们正确解释约束，实际上就不存在组合歧义了。 

较慢的心态可能会模拟队列，将 Alice 和 Bob 放入一个大小为$N$并计算偏移量。 那仍然会运行在$O(N)$，这是不必要的，但在约束下可以接受。 然而，这种方法在索引和逐一处理方面引入了可避免的复杂性。 

关键的见解是这两个位置都是可以直接计算的。 一旦我们将 Alice 映射到$A+1$和鲍勃$N-B$，答案纯粹是这些索引之间的绝对距离减一。 这避免了任何模拟，并将整个问题简化为常数时间算术。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 模拟排队|$O(N)$|$O(N)$| 已接受但没有必要 |
 | 直接位置计算 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 将 Alice 的位置计算为$pos_A = A + 1$。 这直接源自“前面的人”的定义。 
2. 将 Bob 的位置计算为$pos_B = N - B$。 这是通过计算鲍勃后面的人数并将其转换为前面的位置得出的。 
3. 计算它们之间的绝对距离：$d = |pos_A - pos_B|$。 这给出了线上它们的位置之间的步数。 
4. 将此距离减一以排除 Alice 和 Bob 本身，从而产生严格位于他们之间的人数。 

减法步骤至关重要，因为索引中的距离间接计算端点，而我们只需要内部元素。 

### 为什么它有效

 队列是一种线性结构，每个人恰好占据一个整数坐标。 一旦爱丽丝和鲍勃的坐标被独立的约束固定，他们的放置就没有剩余的自由度了。 一行中两个位置之间的人数恰好是它们之间严格的整数索引的计数，这始终是$|i - j| - 1$。 由于两个位置都是唯一确定的，因此计算值在队列的任何解释中都是不变的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

N, A, B = map(int, input().split())

posA = A + 1
posB = N - B

print(abs(posA - posB) - 1)
```该解决方案直接将推理转化为算术。 关键的实现细节是对位置一致使用基于 1 的索引。 Alice 的位置从前面的计数中移动 +1，而 Bob 的位置则从后缀计数中得出。 

末尾减一很容易搞错。 如果没有它，结果将包括 Alice 或 Bob 在计数中，这与问题要求相矛盾。

这些迹线证实了将计数转换为位置并使用标准间隙公式的正确性。

 ## 边缘情况

 当$A = 0$，Alice 位于队列的最前面，位置 1。例如，$N = 5, A = 0, B = 1$。 那么爱丽丝在 1，鲍勃在$5 - 1 = 4$。 计算出的间隙为$|1 - 4| - 1 = 2$。 该算法干净地处理了这个问题，因为它从不假设爱丽丝在队列中间。 

什么时候$B = 0$, Bob 处于最后一个位置$N$。 例如，$N = 6, A = 2, B = 0$。 Alice 是 3，Bob 是 6，产生$|3 - 6| - 1 = 2$。 相同的公式适用，无需特殊外壳。 

当Alice和Bob相邻时，比如$N = 7, A = 1, B = 4$，它们映射到位置 2 和 3。距离为 1，减一得到 0。该算法自然会产生零，而不需要显式的邻接检查，这避免了脆弱的分支逻辑。
