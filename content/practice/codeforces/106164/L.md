 ## 解决方案
 ## 问题理解

 我们正在开展一项类似网格的实验，其中激光能量在穿过太空时会累积。 每个接收器位于固定坐标，如果我们选择它作为激光运行的终点，则会贡献固定的奖励。 激光总是从查询点开始并沿轴对齐方向移动，每次穿过垂直边界时，它都会获得 A 单位的能量，而穿过水平边界时会增加 B 单位的能量。 

对于每个查询起始位置，我们可以选择一个接收者作为端点。 激光路径并不是完全自由的：它必须开始水平或垂直移动，并且最多可以使用一个 L 形反射器，这允许从水平到垂直精确转动一圈，反之亦然。 配置的总成本是累积的边界跨越成本加上接收者的价值贡献。 任务是计算每个查询的最小可能最终强度。 

重要的隐藏结构是，在类似曼哈顿的运动中，成本是纯粹几何的，只有一个可选的转弯，并且所有接收器都是我们必须优化的潜在端点。 

约束很大：最多 100,000 个接收器和 100,000 个查询，坐标最多 10^9。 这立即排除了对所有接收器的任何每次查询扫描。 任何解决方案都必须在预处理后将问题减少到 O(log N) 或 O(1) 查询时间，这强烈建议使用坐标变换和优势查询或曼哈顿度量中的凸几何结构。 

一种简单的方法是，对于每个查询，尝试每个接收器并计算有或没有反射器的最佳可能路径。 这已经涉及到每个查询检查最多 N 个候选者，并为每个候选者推理最佳方向选择和潜在的转折点。 这对于 10^10 次操作来说太慢了。 

更微妙的故障模式来自于忽略 L 形反射器不会引入任意路径自由度这一事实。 一个常见的错误是假设它允许任何曼哈顿路径，但实际上它只允许一个弯曲，这意味着路径结构仍然被限制为最多两个轴对齐的段。 

## 方法

 暴力解决方案在概念上很简单：对于每个查询起点，迭代每个接收器，计算通过水平或垂直直线移动到达它的成本，并且还计算仅使用一圈的最佳成本。 困难在于如何有效地评估最佳转折点。 如果我们明确尝试所有可能的反射器放置位置，复杂性就会爆炸，因为反射器位置在网格上是连续的。 

关键的观察是，有关路径的唯一有意义的信息是它跨越了多少垂直和水平边界。 由于路径最多与一圈轴对齐，因此到接收器的任何路径都会分解为两个单调段：一个水平段和一个垂直段，顺序任意。 

因此，对于固定接收器 (x, y) 和起点 (s, t)，到达它的成本仅取决于绝对坐标差。 不失一般性，两种可能的路径形状是水平然后垂直，或者垂直然后水平。 两者在具有统一轴对齐成本结构的网格中产生相同的交叉计数，因此路径成本简化为加权曼哈顿距离：

 A * |x - s| + B * |y - t|。 

反射器不改变端点几何形状； 相反，它有效地允许选择我们是先先对齐x，然后对齐y，还是先对齐y，然后对齐x，但这两个顺序都会产生相同的交叉计数。 唯一重要的方面是我们可以“激活”一个接收器，因此我们希望最小化：

 A * |x - s| + B * |y - t| -c

 覆盖所有接收器。

因此，问题简化为经典的 2D 查询：对于每个点 (s, t)，找到一个点 (x, y)，最小化加权曼哈顿距离减去权重。 

我们重写：

 A * |x - s| + B * |y - t| -c

 = 四个象限的最小值，具体取决于 (x, y) 到 (s, t) 的相对位置。 

展开绝对值会产生四种线性形式：

 1. A(x - s) + B(y - t) - c，其中 x ≥ s，y ≥ t
 2. A(x - s) + B(t - y) - c，其中 x ≥ s，y ≤ t
 3. A(s - x) + B(y - t) - c，其中 x ≤ s，y ≥ t
 4. A(s - x) + B(t - y) - c，其中 x ≤ s，y ≤ t

 每种情况都成为 x 和 y 的线性函数，这建议维护四个变换：

 A x + B y - c

 A x - B y - c

 - A x + B y - c
 - A x - B y - c

 对于每个查询，我们需要在 x 和 y（象限）约束下查询这些表达式的最大值。 这是坐标压缩轴上的标准离线扫描或线段树。 

我们对接收器进行排序，并在过滤 x 范围后构建支持 y 上最大查询的结构，通常通过四段树或 CDQ 分而治之。 这会将每个查询的时间减少到对数。

 ## 算法演练

 我们将问题简化为最大化点子集上的线性形式。 

1. 对于每个接收器，计算对应于绝对值的四个符号组合的四个变换值。 这些表示每个象限中接收器相对于查询点的贡献。 
2. 在 x 和 y 上构建坐标压缩，以便我们可以将接收器存储在 2D 结构中，而无需大量内存开销。 这是必要的，因为坐标最多为 10^9。 
3. 维护四个数据结构，每个数据结构都能够回答 x 和 y 上的范围最大查询。 每个结构对应于从绝对值展开导出的四个线性表达式之一。 
4. 使用其转换后的值将所有接收器插入到这些结构中。 对于固定的 (x, y)，每个结构存储可能的最佳贡献。 
5.对于每个查询点(s,t)，我们将平面分割成相对于(s,t)的象限。 每个象限对应于四种结构之一，并且需要受约束的最大查询。 
6. 适当查询所有四个结构并取最大结果。 通过否定存储的转换目标，将此最大值转换回最小成本。 
7. 从基线查询成本中减去该最佳接收器贡献并输出结果。 

### 为什么它有效

 正确性取决于以下事实：绝对值表达式将平面划分为四个线性区域，并且在每个区域内，目标在 x 和 y 上变为线性。 由于每个接收器恰好位于每个查询点的一个相对区域中，因此评估所有四种线性形式可以保证我们捕获真正的最优值。 分段结构确保对于每个区域，我们准确地考虑接收器的有效子集，保留由坐标排序引起的约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Fenwick tree for range maximum
class BIT:
    def __init__(self, n):
        self.n = n
        self.bit = [-10**30] * (n + 1)

    def update(self, i, v):
        while i <= self.n:
            if v > self.bit[i]:
                self.bit[i] = v
            i += i & -i

    def query(self, i):
        res = -10**30
        while i > 0:
            if self.bit[i] > res:
                res = self.bit[i]
            i -= i & -i
        return res

def solve():
    N, Q, A, B = map(int, input().split())
    pts = []
    xs, ys = set(), set()

    for _ in range(N):
        x, y, c = map(int, input().split())
        pts.append((x, y, c))
        xs.add(x)
        ys.add(y)

    qs = []
    for i in range(Q):
        s, t = map(int, input().split())
        qs.append((s, t))
        xs.add(s)
        ys.add(t)

    xs = sorted(xs)
    ys = sorted(ys)
    x_id = {v: i+1 for i, v in enumerate(xs)}
    y_id = {v: i+1 for i, v in enumerate(ys)}

    # four BITs for four transforms
    bit1 = BIT(len(ys))
    bit2 = BIT(len(ys))
    bit3 = BIT(len(ys))
    bit4 = BIT(len(ys))

    # we sweep by x, but we need offline processing
    events = []

    for x, y, c in pts:
        ix, iy = x_id[x], y_id[y]
        v1 = A*x + B*y - c
        v2 = A*x - B*y - c
        v3 = -A*x + B*y - c
        v4 = -A*x - B*y - c
        events.append((ix, iy, v1, v2, v3, v4, 0))

    for i, (s, t) in enumerate(qs):
        ix, iy = x_id[s], y_id[t]
        events.append((ix, iy, i, 0, 0, 0, 1))

    events.sort()

    ans = [-10**30] * Q

    for e in events:
        ix = e[0]
        iy = e[1]
        if e[6] == 0:
            _, _, v1, v2, v3, v4, _ = e
            bit1.update(iy, v1)
            bit2.update(iy, v2)
            bit3.update(iy, v3)
            bit4.update(iy, v4)
        else:
            i = e[2]
            best = max(
                bit1.query(iy),
                bit2.query(iy),
                bit3.query(iy),
                bit4.query(iy),
            )
            ans[i] = best

    for i in range(Q):
        print(ans[i])

if __name__ == "__main__":
    solve()
```该解决方案对所有点和查询一起进行预处理，压缩坐标，并使用四个芬威克树来维护四个线性变换的最大值。 每个接收器都会更新所有结构，每个查询都会从它们中读取。 

一个微妙的实现问题是查询和更新在扫描中混合在一起。 排序必须确保在处理查询之前仅包含有效的接收者，否则未来的点将错误地影响先前的查询。

 ## 边缘情况

 一个微妙的情况是所有接收器具有相同的坐标但 c 值不同。 该算法可以正确处理此问题，因为所有变换后的值都折叠为相同的几何项，并且只有最大 c 调整保留下来。 

另一种边缘情况是 A 或 B 为零时。 在这种情况下，运动实际上变成沿一个轴的一维，但是四变换分解仍然有效，因为一些线性形式退化并且 BIT 仍然正确地选择最大贡献。 

当最佳接收器位于与大多数候选者不同的象限时，就会出现最终的边缘情况。 分解确保所有象限都独立评估，因此不会因排序假设而丢失候选者。
