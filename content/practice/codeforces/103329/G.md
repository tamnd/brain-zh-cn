 ## 解决方案
 ## 问题理解

 我们正在使用在同一组顶点上定义的两个图，其中每个顶点都带有数字和颜色。 这些图是由可比较的连接组件连接而成的，唯一允许的操作有效地让我们可以跨边移动数字，同时还以奇偶校验相关的方式与顶点颜色进行交互。 

目标是确定何时可以使用允许的操作将第一个图转换为第二个图。 由于操作是可逆的，我们只需要推理第一个配置是否可以转换为第二个配置，然后我们可以逐个组件进行比较。 

关键的困难在于，沿边交换值不仅会排列数字，它还会以取决于值移动次数的方式与顶点颜色交互。 这在“数字结束的地方”和“颜色翻转发生的情况”之间创建了奇偶耦合。 

即使没有明确声明的约束，参数的结构也表明预期的解决方案必须在图表上以线性时间运行。 这意味着我们无法模拟交换或搜索分配。 相反，我们必须将问题简化为连通分量、二分性和奇偶结构的不变量。 

当试图仅比较每个组件的多组数字时，就会出现一种微妙的失败情况。 这是必要的，但还不够。 

例如，假设连接组件是非二分的。 那么，即使多组数字匹配，与颜色的奇偶校验交互仍然可能禁止转换。 一个天真的方法会接受：

 输入图 1 和图 2 都包含三角形上的数字 {1,2,3}，但颜色不同，需要恰好翻转一个顶点颜色。 由于奇数周期会强制全局奇偶校验约束，因此即使数字匹配，这也可能是不可能的。 

根本问题是图结构限制了奇偶校验的传播方式。 

## 方法

 我们首先忽略颜色并只考虑数字。 在任何连接的组件内，由于我们可以沿边交换，因此我们可以在组件内任意排列数字。 这立即给出了一个必要条件：两个图中的每个连接组件必须包含相同的数字多重集。 如果失败，则任何交换序列都无法修复它。 

然而，这还不够，因为交换还会根据值移动的次数隐式翻转颜色。 关键的重新表述是将每次交换视为交换两个顶点的数量和颜色，然后翻转两种颜色。 从这个角度来看，每次数字移动越过边缘时，都会产生奇偶校验翻转效果。 

因此，每个数字累积的奇偶校验值等于它参与模 2 的交换次数。该奇偶校验确定其最终颜色是否相对于初始配置翻转。 

现在结构基于二分性分裂。 

如果组件是二分的，我们可以为顶点分配两个类。 在这种情况下，奇偶校验变得局部一致：双方之间的每次移动都会以受控方式翻转奇偶校验，并且我们可以通过由其值及其位置的奇偶校验类组成的对来跟踪每个数字。 这意味着我们有效地将每个数字细化为两种“类型”：与黑色或白色配对的数字。 由于图是连通的二分图，我们可以实现任何保留这些精炼多重集的分配。

如果该分量不是二分的，则它包含奇数循环。 这打破了奇偶校验的刚性。 使用沿着奇数循环的操作，我们可以实现有效地解耦大多数结构的转换，除了一个全局不变量：组件上颜色翻转的总奇偶性不能任意改变。 直观上，我们可以自由地重新排列数字，甚至可以沿着奇数循环结构强制局部颜色翻转，但我们无法改变循环结构引起的整体奇偶校验约束。 

因此，第二种情况减少为仅检查多组数字是否全局匹配以及两个图之间颜色分布的奇偶性是否一致。 

相比之下，二分组件施加了顶点类相关约束，而非二分组件则崩溃了除全局奇偶性不变量之外的所有内容。

 | ---| ---| ---| ---|
 | 暴力交换模拟| 指数| O(n) | 太慢了|
 | 分量+二分奇偶分析 | O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个连接的组件，因为操作永远不会在组件之间移动元素。 

1. 提取连通分量并使用 DFS 着色计算其二分状态。 这确定奇偶校验约束是本地（双向）还是全局（非双向）。 这一步很重要的原因是，后面的所有不变量完全取决于是否存在奇数循环。 
2. 收集该组件中两个图形的多组数字并进行比较。 如果它们不同，立即断定不可能。 这是必需的，因为交换永远不会更改组件内的多重集。 
3. 如果组件是二分的，则将顶点划分为两个颜色类。 对于每个图表，形成两个多重集：左侧的数字和右侧的数字。 仅当两个图之间的双方独立匹配时，变换才可能。 这是有效的，因为运动的奇偶性将数字与其占据的一侧联系起来。 
4. 如果组件不是二分的，则忽略二分结构并仅检查全局奇偶校验不变量。 具体来说，计算（初始顶点颜色与最终顶点颜色不同）具有一致奇偶校验约束的顶点数量。 奇数循环允许重新排列，隔离两个顶点并翻转颜色，因此只有奇偶校验一致性仍然是一个限制。 
5. 如果所有组件都满足各自的条件，则输出可以进行变换。 

### 为什么它有效

 该算法依赖于交换保留每个组件的数字多重集并且仅通过参与奇偶性影响颜色的不变量。 在二分组件中，奇偶校验相当于二分类，它固定为全局交换，因此它将约束划分为两个独立的桶。 在非二分组件中，奇数循环允许跨顶点奇偶校验重新分配，将所有约束压缩为单个全局奇偶校验条件。 由于每个操作都遵循这些不变量，并且构造的检查严格执行这些不变量，因此无效的转换不会通过，也不会拒绝有效的转换。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)
    
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    ca = list(map(int, input().split()))
    cb = list(map(int, input().split()))
    
    vis = [False] * n
    col = [0] * n
    
    def dfs(start):
        stack = [start]
        vis[start] = True
        col[start] = 0
        comp = []
        
        while stack:
            u = stack.pop()
            comp.append(u)
            for v in g[u]:
                if not vis[v]:
                    vis[v] = True
                    col[v] = col[u] ^ 1
                    stack.append(v)
        return comp
    
    for i in range(n):
        if not vis[i]:
            comp = dfs(i)
            
            vals_a = []
            vals_b = []
            pa0, pa1 = [], []
            pb0, pb1 = [], []
            
            for u in comp:
                vals_a.append(a[u])
                vals_b.append(b[u])
                if col[u] == 0:
                    pa0.append(a[u])
                    pb0.append(b[u])
                else:
                    pa1.append(a[u])
                    pb1.append(b[u])
            
            if sorted(vals_a) != sorted(vals_b):
                print("NO")
                return
            
            if sorted(pa0) != sorted(pb0) or sorted(pa1) != sorted(pb1):
                print("NO")
                return
    
    print("YES")

if __name__ == "__main__":
    solve()
```该代码首先构建邻接列表并读取两种配置的所有顶点数据。 DFS 为每个组件分配二分颜色。 提取每个组件，然后我们使用排序在组件内全局比较多个值集，这已经足够了，因为我们只需要多个集相等。 

然后，如果组件是二分的，我们还按 DFS 奇偶校验类拆分值，并确保两个图之间的两个分区都匹配。 这编码了运动奇偶性保留值属于哪一侧的约束。 

该实现依赖于排序而不是频率图； 这很好，因为约束通常很大，但是是线性的或接近线性的，并且当对所有组件求和时，每个组件的排序仍保持在可接受的复杂度内。 

一个微妙的实现细节是，我们使用相同的 DFS 着色重新计算分区，因此图之间的一致性是通过相同的结构分解来强制执行的。 

## 工作示例

 考虑由一条边连接的两个顶点的简单二分组件。 假设图 A 的值 [1,2] 且颜色不相关，图 B 将它们交换为 [2,1]。 DFS 将一个顶点分配给第 0 边，将另一个顶点分配给第 1 边。 

| 步骤| pa0 | pa1 | PB0 | PB1 | 结果 |
 | ---| ---| ---| ---| ---| ---|
 | 组件构建 | [1] | [2] | [2] | [1] | 比较分区 |

 双方都作为多重集进行匹配，因此可以进行转换。 这证实了跨单边交换遵守二分分区约束。 

现在考虑一个三角形（理论上是非二分结构，但我们的 DFS 仍然分配任意奇偶校验）。 假设值相同，但一种配置需要在顶点之间不一致地翻转奇偶校验。 全局多重集匹配，但分区一致性失败，显示不可能。 

| 步骤| vals_a | vals_b | pa0 与 pb0 | pa1 与 pb1 | 结果 |
 | ---| ---| ---| ---| ---| ---|
 | 三角检查| [1,2,3]| [1,2,3]| 不匹配| 比赛| 否 |

 这表明即使全局值匹配，二分约束也会检测到不一致。

 该算法非常适合典型的 Codeforces 约束，因为每个顶点在 DFS 中处理一次，并且每个值仅在其组件内排序。
