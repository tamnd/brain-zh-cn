 ## 解决方案
 ## 问题理解

 我们被要求回答关于整数值的多个独立查询$x$。 对于每个$x$，我们需要构造一个短整数区间$[l, r]$使得该区间内的所有整数之和为素数。 在所有有效区间中，我们可以输出任何一个满足条件的区间，但构造必须是确定性和高效的。 

关键操作是区间和。 对于任何$[l, r]$，总和是算术级数：$$S(l, r) = \frac{(l + r)(r - l + 1)}{2}$$因此，问题简化为选择整数端点，以便该表达式计算结果为素数。 

约束条件（最多约$10^7$必须检查素数的值的大小）意味着我们不能在每个查询中天真地测试素数。 相反，我们必须将素数预处理到大约$2 \cdot 10^7$，可轻松放入筛子中。 然后，每个查询必须减少到恒定数量的候选结构。 

当出现一个微妙的问题时$l$是非正数。 在这种情况下，区间结构会改变和表达式的奇偶校验行为，并且朴素的对称性论证可能会失败。 另一个常见的陷阱是假设长间隔仍然可能产生素数。 这是错误的，因为总和因式分解为两个整数，并且对于长度至少为 3 的情况，其中一个因式会变成合数或产生不可避免的因式分解结构，从而使素性成为不可能。 

朴素推理的失败案例是尝试周围的所有间隔$x$。 例如，检查所有$[x - k, x + k]$当有很多查询时，每个查询达到一定的界限会太慢，因为每次检查都需要计算总和并测试素数。 

## 方法

 暴力破解的想法很简单：对于每个查询$x$, 枚举可能的对$(l, r)$大约$x$，计算总和，并使用筛查找或直接素性测试来测试它是否为素数。 正确性是显而易见的，因为我们正在详尽地搜索所有候选者。 

问题在于间隔的数量。 如果我们允许$l, r$在一定尺寸范围内变化$O(N)$，每个查询变成$O(N^2)$在最坏的情况下，这是完全不可行的。 

从结构上看，有效区间是极其有限的。 总和表达式施加强因式分解约束。 对于长度至少为 3 的区间，算术级数和总是在至少一个乘法分量中引入复合结构，因此素数只能从非常短的区间中产生。 

这将整个问题简化为仅检查每个周围的恒定数量的间隔形状$x$，加上偶尔寻找下一个素数$y$或一个数字$z$这样$2z - 1$是素数。 这些搜索可以通过预先计算的素数表和简单的向上扫描来完成。 

问题变成了本地案例处理和最近素数查询的混合，所有这些都由筛子支持。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 间隔暴力破解 |$O(T \cdot N^2)$|$O(N)$| 太慢了 |
 | 素筛+常数候选|$O(N \log \log N + T \cdot \sqrt{N})$最差扫描|$O(N)$| 已接受 |

 ## 算法演练

 我们将素性预先计算到略高于安全范围$2 \cdot 10^7$，因为像这样的表达式$2z - 1$或者$y$可以达到这个量级。 

### 步骤

 1. 建立一个埃拉托色尼筛直至$2.001 \cdot 10^7$，存储一个布尔数组`is_prime`。 这允许稍后进行 O(1) 素性检查。 
2.对于每个查询值$x$，根据是否存在分为两种情况$x \le 0$或者$x > 0$，因为有效区间的结构在零附近变化。 
3.如果$x > 0$，首先尝试最简单的有效结构。 间隔$[x, x]$有效，如果$x$本身是素数，因为总和是$x$。 这是最直接的候选人。 
4. 接下来考虑$[x, x+1]$。 其总和为$2x+1$，这可能是素数并且检查起来很便宜。 
5. 还要考虑$[x-1, x]$，其总和为$2x-1$。 这通常捕获附近奇数是素数的情况。 这三个检查涵盖了以$x$。 
6. 如果这些都不起作用，我们就转向不对称结构，其中区间进入非正区域。 我们寻找最小的素数$y \ge x$和输出$[-y+1, y]$。 该区间的总和等于$y$，因为对称抵消正好留下端点贡献。 
7. 如果仍然不适用，我们将搜索最小的$z \ge x$这样$2z - 1$是素数，输出$[-z+2, z]$。 这种结构的设计是为了使总和折叠成以下形式$2z - 1$。 
8.如果$x \le 0$，我们跳过琐碎的以正为中心的情况，直接使用两种不对称模式：一种针对主要端点$y$，一个针对形式的质数$2z - 1$，调整下界确保区间结构的正确性。 

### 为什么它有效

 正确性取决于结构约束，即在正状态下，任何产生素数和的有效区间的长度必须最多为 2。 较长的间隔会在算术和中引入因式分解，从而防止素数。 因此，所有有效的解决方案都简化为单点间隔或相邻对，或特殊构造的对称间隔，迫使总和成为受控代数形式$y$或者$2z - 1$。 

我们测试的每个候选者都完全对应于唯一仍然可以在求和公式下产生素数的代数有效形式之一。 由于我们在每种形式中搜索最接近的合适素数，因此只要存在一个有效区间，我们就能保证找到一个有效区间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXN = 2_000_100

is_prime = [True] * (MAXN + 1)
is_prime[0] = is_prime[1] = False

for i in range(2, int(MAXN ** 0.5) + 1):
    if is_prime[i]:
        step = i
        start = i * i
        for j in range(start, MAXN + 1, step):
            is_prime[j] = False

def next_prime(x):
    while x <= MAXN and not is_prime[x]:
        x += 1
    return x

def solve_case(x):
    if x > 0:
        if is_prime[x]:
            return x, x
        if 2 * x + 1 <= MAXN and is_prime[2 * x + 1]:
            return x, x + 1
        if 2 * x - 1 >= 0 and is_prime[2 * x - 1]:
            return x - 1, x

        y = next_prime(x)
        if y <= MAXN:
            return -y + 1, y

        z = x
        while z <= MAXN and (2 * z - 1 > MAXN or not is_prime[2 * z - 1]):
            z += 1
        return -z + 2, z

    else:
        y = next_prime(1 - x)
        return -y + 1, y

def main():
    t = int(input())
    for _ in range(t):
        x = int(input())
        l, r = solve_case(x)
        print(l, r)

if __name__ == "__main__":
    main()
```筛子构建一次素性表，这很重要，因为每个候选区间都需要恒定时间的素性检查。 这`next_prime`函数执行线性扫描，但在所有查询中它仍然保持快速，因为值向前移动并且搜索空间是有限的。 

对于每个查询，我们首先尝试周围的三个局部间隔模式$x$。 这些完全对应于锚定于的所有长度至多 2 的间隔$x$。 如果都不起作用，我们就会转向全局结构的构造，这取决于找到下一个合适的素数或半素数模式。 

负面的处理$x$直接避免了不必要的局部检查，因为代数结构已经迫使解变成对称形式。

 考虑一个带有两个查询的小型说明性输入。 

输入：$x = 3$和$x = -2$为了$x = 3$:

 所以答案是$[3,3]$，因为总和是 3。 

对于$x = -2$:

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    MAXN = 2_000_100
    is_prime = [True] * (MAXN + 1)
    is_prime[0] = is_prime[1] = False
    for i in range(2, int(MAXN ** 0.5) + 1):
        if is_prime[i]:
            for j in range(i * i, MAXN + 1, i):
                is_prime[j] = False

    def next_prime(x):
        while x <= MAXN and not is_prime[x]:
            x += 1
        return x

    def solve(x):
        if x > 0:
            if is_prime[x]:
                return (x, x)
            if 2 * x + 1 <= MAXN and is_prime[2 * x + 1]:
                return (x, x + 1)
            if 2 * x - 1 >= 0 and is_prime[2 * x - 1]:
                return (x - 1, x)
            y = next_prime(x)
            return (-y + 1, y)

            y = next_prime(1 - x)
            return (-y + 1, y)

    t = int(input())
    for _ in range(t):
        x = int(input())
        l, r = solve(x)
        out.append(f"{l} {r}")
    return "\n".join(out)

 ## 边缘情况

 对于$x = 1$，算法立即检查$[1,1]$。 总和为 1，不是质数，因此移动到$[1,2]$给出总和 3，它是素数。 该代码正确地捕获了这一点，因为它显式检查$2x+1$。 

为了$x = 0$，触发否定式构造。 我们计算最小素数$y \ge 1$，即 2，然后返回$[-1, 2]$。 总和是$2$，正确素数，这避免了围绕零的无效单元素推理。 

对于负值，例如$x = -5$，该算法不会尝试局部区间枚举。 相反，它直接计算下一个素数$y \ge 6$，即 7，并返回$[-6, 7]$。 即使周围有邻居，这也能确保正确性$x$不包含有用的局部结构，因为该结构与局部密度无关。
