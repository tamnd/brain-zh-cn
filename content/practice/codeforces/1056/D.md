 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，节点 1 固定为根。 只有叶节点最初接收颜色，所有其他节点都是未着色的。 在这个问题中，叶子被定义为子树仅由其自身组成的节点，这在有根树中意味着没有子节点的节点。 

一旦将颜色分配给叶子，我们就会查看每个节点 t 并检查其子树。 如果在 t 的子树内，所有有色叶子都具有成对不同的颜色，则节点 t 被称为“快乐”，这意味着 t 以下的叶子之间没有颜色重复。 未着色的节点并不直接重要，只是它们定义了哪些叶子位于哪个子树中。 

对于从 1 到 n 的每个 k，我们想要确定所需的最小颜色数，以便树中至少有 k 个快乐节点。 

关键的困难在于，快乐节点的数量取决于叶子颜色如何在树中向上传播冲突，并且我们必须进行全局推理而不是局部推理。 

n 高达 100000 的约束迫使我们远离任何重复重新计算子树属性的构造。 任何二次或什至 O(n log n) 且每 k 需要大量重新计算的操作都太慢。 我们需要一个单一的全局结构，让我们了解在给定数量的颜色下可以使多少个节点感到高兴，然后反转该关系。 

如果假设每个子树独立地贡献快乐节点，就会出现一种幼稚的故障模式。 例如，在一棵星形树中，对所有叶子进行独特的着色会让每个节点都感到高兴，但是重复使用颜色会以一种不属于任何单个子树的方式破坏幸福感。 任何忽略祖先重叠的贪婪分配都会错误计数。 

另一个微妙的问题是，“快乐”在添加颜色方面是单调的。 增加颜色数量只能增加或保留快乐节点的数量，而不能减少它。 任何不依赖这种单调性的方法在尝试回答所有 k 值时都会失败。 

## 方法

 蛮力的想法是固定一些颜色 C 并尝试以最大化快乐节点数量的方式为叶子分配颜色。 对于固定的 C，我们可以模拟分配或尝试对所有着色进行推理，但叶子分配的数量会随着叶子的数量呈指数增长。 即使将我们限制在结构化作业中，仍然会导致组合爆炸。 

关键的观察是，节点 t 快乐的条件仅取决于其子树中不同叶子的数量是否超过 C。如果子树包含多于 C 个叶子，那么无论我们如何仅使用 C 颜色对它们进行着色，都会强制重复，并且 t 会变得不快乐。 如果它最多包含 C 个叶子，我们总是可以在该子树中分配不同的颜色，以便 t 保持快乐。 

这彻底改变了问题。 我们不需要考虑任意的​​颜色分配，我们只需要知道每个节点的子树中有多少叶子。 这个数字完全决定了该节点是否可以在 C 颜色下感到高兴。 

因此对于固定的 C：

 当且仅当一个节点的子树最多包含 C 个叶子时，该节点才是快乐的。 

现在问题变成：对于每个 C，计算有多少个节点的 subtree-leaf-count ≤ C。然后我们反转这个函数来回答 k 上的查询。 

我们通过 DFS 计算叶子计数。 然后，我们根据这些计数对节点进行存储，并根据可能的值构建频率数组。 对于任何 C，该数组的前缀和给出了有多少个节点是满意的。 

最后，我们通过从 1 到 n 扫描 C 并维护叶子计数 ≤ C 的节点的累积数量来进行反转。对于每个 k，我们找到至少达到 k 的最小 C。

 | ---| ---| ---| ---|
 | 暴力着色 | 指数| O(n) | 太慢了 |

 ## 算法演练

 ## 1. 树根并计算子树叶子计数

 我们从父表示构建邻接列表，并从节点 1 执行 DFS。对于每个节点，我们计算其子树中存在多少个叶子。 如果一个节点没有子节点，那么它就是叶子。 

该值准确地捕获了有多少彩色对象可以影响该子树。 

## 2. 存储每个节点的叶子计数

 对于每个节点 v，我们计算 leaf_count[v]。 如果 v 是叶子，则该值为 1。否则，它是其子节点的叶子计数之和。 这种聚合确保每个祖先的每个叶子都被精确计数一次。 

## 3. 叶子计数的计数频率

 我们构建一个数组 freq[x]，计算有多少个节点在其子树中恰好有 x 个叶子。 

这将树信息压缩成子树大小的分布。 

## 4. 构建频率上的前缀和

 我们计算 pref[C] = 子树最多有 C 个叶子的节点数。 

这是通过从小到大的值累积频率来完成的。 此时，pref[C] 等于如果我们有 C 种颜色就会满意的节点数。 

## 5. 反转函数以获得答案

 现在，我们将 pref[C] 解释为 C 中的单调函数。对于每个 k，我们找到最小的 C，使得 pref[C] ≥ k。 

我们通过扫描 C 一次并按顺序填写所有 k 的答案来做到这一点。 

## 为什么它有效

 关键的不变量是，当且仅当其子树最多包含 C 个叶子时，节点在 C 种颜色下感到满意。 这种等价性之所以成立，是因为每个叶子都必须接收一种颜色，并且一旦叶子数量超过可用颜色，子树中的独特性就不可能实现。 相反，如果存在足够的颜色，我们可以独立于其他子树在每个子树内分配唯一的颜色，因为叶集仅通过祖先重叠，而不是在不相交的子树内重叠。 

这将整个着色问题简化为树的确定性结构属性，从而消除了实际着色构造的任何需要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    parents = list(map(int, input().split())) if n > 1 else []

    g = [[] for _ in range(n + 1)]
    for i, p in enumerate(parents, start=2):
        g[p].append(i)

    leaf_cnt = [0] * (n + 1)

    def dfs(v):
        if not g[v]:
            leaf_cnt[v] = 1
            return 1
        s = 0
        for u in g[v]:
            s += dfs(u)
        leaf_cnt[v] = s
        return s

    dfs(1)

    freq = [0] * (n + 1)
    for v in range(1, n + 1):
        freq[leaf_cnt[v]] += 1

    pref = [0] * (n + 1)
    for i in range(1, n + 1):
        pref[i] = pref[i - 1] + freq[i]

    ans = [0] * (n + 1)

    c = 1
    for k in range(1, n + 1):
        while c <= n and pref[c] < k:
            c += 1
        ans[k] = c

    print(*ans[1:])

if __name__ == "__main__":
    solve()
```DFS是结构核心，自底向上计算子树叶子计数。 频率数组将树压缩为直方图，前缀和将其转换为单调函数。 最终的两指针扫描避免了每 k 的重新计算并确保线性总工作。 

一个常见的错误是将子树大小与子树叶子计数混淆。 只有叶子很重要，因为只有它们携带颜色，并且内部节点不会直接导致颜色冲突。 

## 工作示例

 ### 示例 1

 输入：```
3
1 1
```树：

 1 有孩子 2 和 3，都离开了。 

叶子数：

 节点 2 → 1

 节点 3 → 1

 节点 1 → 2

 | 节点| 叶子数|
 | ---| ---|
 | 1 | 2 |
 | 2 | 1 |
 | 3 | 1 |

 频率：

 频率[1]=2，频率[2]=1

 前缀：

 C=1 → 2 个节点

 C=2 → 3 个节点

 我们反转：

 k=1 → C=1

 k=2 → C=1

 k=3 → C=2

 输出：```
1 1 2
```这表明小 C 已经让除根之外的大多数节点都满意了。

 1 → {2,3}, 2 → {4,5}

 叶子数：

 节点 3、4、5 是叶子 → 各 1 个

 节点 2 → 2

 节点 1 → 3

 | 节点| 叶子数|
 | ---| ---|
 | 1 | 3 |
 | 2 | 2 |
 | 3 | 1 |
 | 4 | 1 |

 前缀：

 C=1 → 3 个节点

 ## 边缘情况

 单节点树是最简单的配置。 唯一的节点既是根又是叶，因此它的叶数为 1。该算法设置 freq[1]=1，并为所有 k=1 生成答案 1。 这证实了基本情况是自然处理的。 

在链中，除了最后一个节点之外的每个节点在其子树中都只有一个叶子。 这会产生平坦的频率分布，并确保所有答案都保持为 1。DFS 正确累积叶计数，而不会过度计数内部节点。 

在星形中，根下面有许多叶子，而叶子本身的叶子数为 1。这在叶子和内部节点之间形成了强烈的分离，并且前缀逻辑正确地反映了增加 C 如何首先满足叶子，然后满足内部节点。
