 ## 解决方案
 ## 问题理解

 城市是一个图，其中十字路口是顶点，道路是无向边。 在该图中，每次骑行都是一条简单的路径，这意味着驾驶员永远不会重新访问该骑行中的顶点。 在所有游乐设施中，我们观察到许多这样简单的路径。 

问题不在于图表是否一致或游乐设施是否有效。 相反，它与路由的唯一性有关。 对于每个有序的顶点对$a, b$，考虑司机可能离开的所有可能方式$a$到$b$作为任何记录的骑行的一部分。 如果同一有序对之间存在两条不同的简单路径，则该驾驶员被宣布为人类。 如果对于每个有序对，路径都是有效强制且唯一的，那么驾驶员可能是一个机器人。 

一个关键的微妙之处在于，记录的行程不会直接询问所有对之间的查询。 它们只给我们提供了很长的简单路径，我们必须从中推断出全局一致性约束。 

约束足够大，任何关于顶点或游乐设施的二次方都是不可能的。 所有路径长度的总和受下式限制$3 \cdot 10^5$，因此任何解决方案在输入大小上都必须基本上是线性的或接近线性的。 这立即排除了任何试图比较所有路径对或显式计算所有路径对连接结构的方法。 

当多个游乐设施部分重叠但随后又分开时，就会出现幼稚的失败案例。 

例如，如果一次行程包含$1 \to 2 \to 3 \to 4$另一个包含$1 \to 2 \to 5 \to 4$，那么之间有两条不同的简单路径$2$和$4$，强制人类回答。 仅检查每个骑行是否单独有效的简单方法将完全错过这一点，因为每个骑行单独都是一条有效的简单路径。 

另一个微妙的情况是循环。 如果游乐设施集体允许在一个循环中以多个方向或多个弦结构遍历，则路径的唯一性就会失败。 挑战在于检测所有游乐设施的联合何时意味着某些对之间有不止一条简单路线，而无需明确枚举所有对。 

## 方法

 暴力解释将尝试重建所有路径，然后比较每对顶点$a, b$跨越所有游乐设施，检查是否存在多个简单路径。 即使我们将每个骑行表示为一系列边，每个骑行的子路径数量也是二次方，导致$O(\sum k^2)$，对于$3 \cdot 10^5$总长度。 

更好的视角来自于翻转问题。 我们不考虑顶点对，而是考虑游乐设施中连续边施加的局部结构约束。 每次骑行都会贡献“顶点”形式的约束$v$必须以一致的方式连接其前驱和后继”。如果一个顶点被迫连接到两个以上独立的延续，而这些延续在单个链状结构中未对齐，那么系统中的某个地方必须存在多个简单路径。

 这就引出了核心观察：保证任意两个顶点之间的简单路径唯一性的结构本质上是顶点不相交的简单路径的集合，可能形成单线状结构，其中每个内部顶点在隐含约束图中的度数最多为 2。 如果任何顶点被迫以与单一路径排序不一致的方式充当分支点，则会出现歧义。 

我们处理游乐设施并构建由连续顶点引起的邻接约束。 然后，我们检测是否有任何顶点被迫陷入冲突的排序约束，这意味着分支或循环与唯一路径结构不兼容。 该问题简化为检查生成的约束图的行为是否类似于一组具有一致顺序的不相交链。 

关键技巧是将每次骑行视为诱导邻接约束，并验证诱导结构是否可以定向到一组没有矛盾的路径。 这可以使用度约束和邻居排序的一致性来完成。

 | ---| ---| ---| ---|
 | 蛮力 |$O(\sum k^2)$|$O(n^2)$| 太慢了 |
 | 最佳|$O(\sum k)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将所有行程转换为连续顶点之间的无向邻接信息。 

1. 对于每次骑行，迭代连续的对$(c_i, c_{i+1})$，并记录这两个顶点在约束意义上是直接连接的。 这捕获了观察到的路径中的所有局部强制转换。 
2. 为每个顶点维护一组（或散列结构）它被约束连接的邻居。 我们不是构建原始图，而是所有强制转换的并集。 
3. 对于每个顶点，检查它在此约束结构中有多少个不同的邻居。 如果任何顶点有两个以上不同的邻居，我们立即得出结论，驾驶员是人类。 原因是，在每对顶点都有唯一简单路径的任何结构中，导出的结构必须表现为路径的集合，并且路径顶点的度数最多为 2。 
4. 此外，我们必须确保游乐设施引起的订购的一致性。 我们通过确保如果一个顶点出现在多次骑行中，其邻接关系不会暗示分支歧义来模拟遍历约束。 实际上，一旦处理了所有游乐设施，约束图必须是最大度为 2 的伪森林。 
5. 如果没有顶点违反度数约束，我们就得出结论，该结构与确定性路径状配置和输出机器人一致。 

基本思想是，每次骑行都会强制执行局部连续性，任何违反“每个顶点最多两个方向”的行为都会创建多种可能的方式来在某些顶点对之间进行遍历。 

### 为什么它有效

 如果每个顶点在导出约束结构中最多有两个邻居，则每个连通分量要么是简单路径，要么是循环。 然而，唯一性条件不允许循环，因为任何循环都会立即在其上的任意两个顶点之间创建两条不同的简单路径。 因此，对于作为机器人的驾驶员来说，结构实际上必须是没有分支和循环的简单路径的集合。 该构造确保如果隐含转换中存在循环或分支，则某些顶点在约束图中的度数至少为三，这正是我们检测到的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        q = int(input())
        
        deg = [0] * (n + 1)
        seen_edges = set()

        bad = False

        for _ in range(q):
            tmp = list(map(int, input().split()))
            k = tmp[0]
            path = tmp[1:]
            
            for i in range(k - 1):
                a, b = path[i], path[i + 1]
                if a > b:
                    a, b = b, a
                if (a, b) not in seen_edges:
                    seen_edges.add((a, b))
                    deg[a] += 1
                    deg[b] += 1
                    if deg[a] > 2 or deg[b] > 2:
                        bad = True

        print("Human" if bad else "Robot")

if __name__ == "__main__":
    solve()
```该代码使用边缘集构建邻接约束的紧凑表示，以避免重复计算游乐设施之间的重复转换。 度数数组跟踪每个顶点被迫连接到多少个不同的邻居。 一旦任何顶点超过两个不同的邻居，就会触发早期退出条件，因为这已经保证了破坏唯一性的结构分支。 

排序规范化`(a, b) = sorted pair`确保无论游乐设施中的遍历方向如何，无向边都不会被重复计算。 这很重要，因为游乐设施可以在两个方向上穿越边缘，但我们只关心结构连接性，而不关心方向。

 | 空间|$O(n + \sum k)$| 度数组加上存储每个不同邻接的边集 |

 转换总数的界限为$3 \cdot 10^5$，因此该解决方案可以在限制范围内顺利运行。 基于散列的边缘重复数据删除确保我们不会因同一段的重复出现而增加复杂性。

 ## 边缘情况

 一个关键的边缘情况是在多个游乐设施中重复遍历同一路段。 该算法通过以下方式处理这个问题`seen_edges`设置，确保重新访问边缘不会人为地增加度数。 如果没有这个，单个有效链可能会错误地超出程度界限。 

另一个边缘情况是一个顶点在不同的游乐设施中多次出现，但只有两个一致的邻居。 基于程度的条件正确地保持其有效，因为唯一性取决于不同的邻居，而不是出现的频率。 

最后一种边缘情况是大型星状结构，其中一个顶点连接到许多路径。 一旦出现第三个不同的邻居，算法就会将其标记为人类，从而正确捕获第一个点，使端点之间的多个简单路由成为可能。
