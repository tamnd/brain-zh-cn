 ## 解决方案
 ## 问题理解

 中的每个单元格$n \times n$网格定义了糖果的数量等于$i^2 + j^2$， 在哪里$i$和$j$是行索引和列索引。 对于每个细胞，我们想象取出许多相同的糖果并尝试将它们平均分配$m$朋友们。 仅当单元格的糖果数可被除以时才算作计数$m$，这意味着糖果可以分为$m$相等的整数部分，没有剩余。 

任务是数出有多少对$(i, j)$和$1 \le i, j \le n$满足$$i^2 + j^2 \equiv 0 \pmod m.$$输入大小立即排除任何直接枚举。 自从$n$可以达到$10^9$，迭代所有$n^2$成对是不可能的。 甚至迭代所有$i, j$对远远超出了时间限制，因此解决方案必须仅依赖于模块化结构而不是实际值。 

一个天真的尝试是尝试检查每个单元格并计算模数之和$m$，但这需要$O(n^2)$运营。 和$n = 10^9$，这是一个天文数字。 

第二个微妙的陷阱是假设对称性本身就足以减少工作量。 尽管$i^2 + j^2$是对称的，对称性仍然存在$O(n^2)$对，所以它不会改变复杂性。 

另一个边缘情况出现时$m = 1$。 每个数字都可以被 1 整除，所以答案应该是$n^2$。 任何模块化推理都必须保留这种琐碎的情况，而没有意外的除法逻辑。 

## 方法

 蛮力方法很简单：迭代所有$i$和$j$, 计算$i^2 + j^2$，并检查整除性$m$。 这是正确的，因为它直接遵循条件的定义。 然而，它执行$n^2$检查，当$n$很大。 

关键的观察是整除性仅取决于余数模$m$。 自从$$i^2 + j^2 \equiv (i \bmod m)^2 + (j \bmod m)^2 \pmod m,$$我们只需要考虑$i$和$j$直至其完整周期长度$m$。 每个完整的大小块$m$重复相同的残留模式。 

因此，我们不再考虑坐标，而是转向残基类。 让$f[r]$有多少个整数$[1, n]$有余数$r$模数$m$。 然后我们计算残基对$(a, b)$这样：$$a^2 + b^2 \equiv 0 \pmod m,$$并乘以每个残基对的形成方式：$$f[a] \cdot f[b].$$这将问题减少到$O(m^2)$，这是可以接受的，因为$m \le 1000$。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最优（模块化计数）|$O(m^2)$|$O(m)$| 已接受 |

 ## 算法演练

 1. 将问题简化为计算数字的剩余频率$1$到$n$模数$m$。 每个残基类别代表整个范围内的重复模式。 
2. 构建数组`cnt`尺寸的$m$， 在哪里`cnt[r]`是整数的个数$[1, n]$这样$i \bmod m = r$。 这是有效的，因为数字在长度周期内均匀分布$m$，带有余数段。 
3. 迭代所有残基对$(a, b)$从$0$到$m-1$。 
4. 对于每一对，计算$(a^2 + b^2) \bmod m$。 如果它等于零，则该残基对贡献有效单元。 
5.添加贡献`cnt[a] * cnt[b]`到答案。 这将计算行残差为的所有网格单元$a$柱残余物为$b$。 
6. 返回累计金额。 

### 为什么它有效

 中的每个整数$[1, n]$恰好属于一个余数类模$m$，并且同一类中的所有整数在求模平方方面表现相同$m$。 因此，每个网格单元$(i, j)$完全由两人决定$(i \bmod m, j \bmod m)$。 该算法以正确的重数一次性枚举所有此类残基对，因此不会遗漏任何有效对，也不包含无效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())

    cnt = [0] * m

    # count how many numbers in [1..n] fall into each residue class mod m
    for r in range(m):
        # first number with residue r is r (or r+m if r==0 and starting from 1)
        # easier: shift range [1..n] -> [0..n-1] then adjust
        # compute count of x in [0..n-1] with x % m == r
        cnt[r] = (n - r + m - 1) // m

    ans = 0

    for a in range(m):
        for b in range(m):
            if (a * a + b * b) % m == 0:
                ans += cnt[a] * cnt[b]

    print(ans)

if __name__ == "__main__":
    solve()
```这`cnt[r]`计算计算有多少个整数$1$到$n$有余数$r$模数$m$，通过有效地移动范围并使用算术级数计数。 每个残基形成近乎均匀的分布，最多有一个元素不同。 

残基上的嵌套循环检查所有$m^2$仅当模块化条件成立时才进行组合并累积贡献。 这直接镜像网格，但是在压缩的剩余空间中。 

这里的一个常见错误是错误地将残留物处理为零。 将范围视为$0 \ldots n$而不是$1 \ldots n$导致差一错误`cnt[0]`，然后传播到不正确的对计数中。

 ## 边缘情况

 当$m = 1$，每个数字都可以被 1 整除，所以所有$n^2$必须对网格单元进行计数。 该算法自然地处理这个问题，因为唯一的残差类别是 0，并且$0^2 + 0^2 \equiv 0$。 

什么时候$n < m$，残留物计数稀疏且不均匀。 例如，$n = 3, m = 5$产生`cnt = [1,1,1,0,0]`。 该算法仍然有效，因为它计算精确的出现次数而不是假设完整的周期。 

当出现一个微妙的情况时$n$正好可以整除$m$。 那么所有的残基都有相同的频率$n/m$，并且计算变得完全一致。 该公式仍然成立，因为它从不明确假设均匀性，只有算术级数才有效。
