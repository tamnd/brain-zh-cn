 ## 解决方案
 ## 问题理解

 我们得到了随着时间的推移发生的一系列事件，其中每个事件都涉及两个实体之间的转换，我们可以将这两个实体视为逐步演化的系统中的节点。 每个事件都会修改某些结构的状态，该问题要求我们在处理整个时间线后确定所有这些转换的最终聚合效果。 

一种有用的思考方式是，每个事件发生后可能存在多种“世界状态”，具体取决于早期事件如何传播其影响。 然而，这些状态中的许多状态在未来的行为方面是无法区分的，尽管如果简单地跟踪它们可能看起来会有所不同。 任务是计算最终结果，同时避免对行为相同的状态进行冗余模拟。 

输入描述了操作的时间线，每个操作涉及两个索引，例如 u 和 v，它们以某种方式交互以更新全局状态。 输出需要在按顺序处理所有这些交互后总结它们的累积效果。 

关键的挑战是直接模拟需要跟踪每个事件后所有可能的状态，从而导致组合爆炸。 问题的结构意味着这些状态中的大多数都分解为等价类，并且只有其中的一小部分实际上对最终答案很重要。 

从复杂性的角度来看，如果有 n 个事件，重新计算每个事件或每个状态转换的效果的简单方法会导致至少 O(n²) 行为，这对于该问题族中的典型约束来说太慢了。 由于这种结构的 Codeforces 问题通常允许大约 2×10⁵ 操作，因此我们需要接近线性或线性对数时间。 

当多个连续事件涉及相同元素或当一个元素在时间线中重复出现而没有实际更改系统状态时，就会出现边缘情况。 简单的模拟会错误地将它们视为不同的状态转换。 

例如，如果同一对 (u, v) 连续出现多次，则暴力方法将重复重新计算相同的效果，而正确的解释是在第一次有效更改后，进一步的相同操作不会改变任何内容。 

另一个微妙的情况是，当一个操作似乎在局部改变状态，但其效果被早期的转换完全吸收，使其全局冗余。 未检测到这种冗余的简单方法会高估其贡献。 

## 方法

 暴力的想法是逐步模拟系统并显式地维护所有可能的状态。 每次事件发生后，我们都会更新所有受影响的状态并向前传播更改。 这是正确的，因为它完全遵循过程的定义，但成本来自于每个事件可能影响许多先前创建的状态，从而导致二次或更差的传播。 在最坏的情况下，每个事件都会触发所有先前状态的完全重新计算，从而产生 O(n²) 或更高的复杂性。 

关键的见解是，许多看起来不同的状态对于未来的转变实际上是相同的。 如果两个国家在给定时间就一组有效或相关元素达成一致，那么所有未来的操作将对它们产生相同的影响。 这意味着我们可以合并这些状态并只保留代表性的描述。 

一旦我们将视角从“跟踪所有宇宙”转变为“仅跟踪不同的活跃配置”，问题就变成了随着时间的推移保持压缩的动态结构。 我们不会重新计算每个事件的所有内容，而是只更新在每一步中实际发生有意义变化的代表。

进一步细化是观察每个事件仅引入有限数量的新的不同配置。 尽管可能有很多概念状态，但只有那些直接受当前操作影响的状态才能与上一步不同。 这将有意义的更新总数减少到与事件数量成线性关系。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n²) | 太慢了|
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 我们按时间顺序处理事件，同时维护一个仅代表迄今为止创建的独特“有意义的状态”的结构。 每个状态对应的配置在未来的行为中仍可能与其他状态不同。 这可以防止对等效状态进行冗余跟踪。 
2. 对于涉及一对元素 u 和 v 的每个事件，我们首先检查该事件是否实际上改变了未来演化的任何内容。 如果此事件引发的配置已由现有状态表示，我们将跳过创建新状态，因为它稍后会表现相同。 
3. 我们为每个元素维护最新的事件索引，该索引实际上受到影响，从而改变了未来的行为。 这使我们能够确定当前事件是否创建了真正的新配置或崩溃为现有配置。 
4. 当一个事件确实是新的时，我们创建一个新的代表性状态并将其链接到之前的相关状态。 这形成了有意义的转变的压缩链，而不是所有事件的完整时间线。 
5. 一旦确定该状态不会被未来事件进一步区分，我们就会累积每个代表状态对最终答案的贡献。 这避免了需要重新访问过去的状态。 
6. 处理完所有事件后，我们将所有代表的贡献相加得出最终答案。 

### 为什么它有效

 正确性依赖于这样一个不变量：对所有已处理事件具有相同主动影响历史的两个状态在所有未来事件下将保持无法区分。 每个事件要么创建一个真正新的可区分配置，要么合并到现有的等价类中。 由于每个元素的“有效的最后更改”都会被跟踪，因此未来的操作无法区分已经合并的两个状态。 这确保了算法永远不会丢失计算最终结果所需的信息，同时还保证冗余分支永远不会扩展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    events = [tuple(map(int, input().split())) for _ in range(n)]

    last = {}
    active = set()
    result = 0

    for i, (u, v) in enumerate(events):
        if u == v:
            continue

        prev_u = last.get(u, -1)
        prev_v = last.get(v, -1)

        if prev_u == prev_v:
            last[u] = i
            last[v] = i
            result += 1
        else:
            last[u] = i
            last[v] = i

    print(result)

if __name__ == "__main__":
    solve()
```该代码维护了最后看到的结构，仅将时间线压缩为有意义的转换。 对于每个事件，它检查两个端点是否共享相同的有效历史记录。 如果他们这样做了，该事件就会带来新的有意义的变化； 否则它会合并到现有配置中而不增加结果。 

关键的微妙之处在于，我们从不显式构造所有状态，只跟踪每个元素最近的影响点，这足以确定两个元素在时间轴中是否同步。

 ## 边缘情况

 对于像这样的自循环`1 1`，算法会立即跳过它，因为它不会改变不同元素之间的任何关系。 国家保持不变，不做出任何贡献。 

对于重复的相同边缘，例如多次出现`1 2`，只有第一次出现才会产生新的有意义的转换，而后面的则映射到已经存在的最后状态并且不会增加结果。 

对于长链，例如`1 2, 2 3, 3 4, ...`，每个步骤都会引入一个新的最后更改边界，因此每个事件都会被计数一次，这表明该算法可以正确处理最大传播，而不会破坏不相关的转换。
