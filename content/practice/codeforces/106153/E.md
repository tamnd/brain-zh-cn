 ## 解决方案
 ## 问题理解

 我们得到一个数字数组，并要求选择一个最大化其元素方差的子序列。 在这种情况下，方差衡量所选值在其平均值周围的分布情况，因此我们需要一个值尽可能远离的子序列。 

该问题的关键观察结果是，我们不需要保留所有元素或保留超出子序列排序的结构。 我们可以自由选择任何索引子集。 这种自由将问题从子序列的组合问题简化为仅依赖于值选择的问题。 

如果数组的大小达到典型的 Codeforces 规模（例如 2⋅10^5），则任何尝试所有子序列的方法都立即不可能，因为子序列的数量呈指数增长。 即使迭代所有对或三元组也已经太慢了。 这迫使解决方案仅依赖于数组中的聚合信息而不是枚举。 

当人们假设方差不仅仅取决于极端值时，就会出现一种微妙的失败情况。 例如，如果数组是`[1, 2, 3]`，天真的直觉可能表明混合值比仅采用极端值能提供更高的方差。 但是，当值最分散时，方差会最大化，因此最佳子序列始终集中在最小值和最大值上。 任何包含中间值的尝试只会将平均值拉向中心并减少离散度。 

另一个边缘情况是所有值都相等。 为了`[5, 5, 5]`，任何子序列的方差为零。 涉及最小值和最大值之间差异的简单公式仍然可以正确生成零，但如果处理不仔细，浮点实现可能会引入噪声。 

最后一个重要问题是数值稳定性。 该问题需要计算一个自然分数的值。 直接浮点计算可能会引入精度误差，因此需要有理数或高精度表示。 

## 方法

 我们从方差的定义开始。 对于选定的值序列$x_1, x_2, \dots, x_k$，方差为$$\mathrm{Var} = \frac{1}{k}\sum x_i^2 - \left(\frac{1}{k}\sum x_i\right)^2.$$蛮力将枚举每个子序列，计算其均值和方差，并跟踪最大值。 这是正确的，但完全不可行。 和$n$元素，有$2^n$子序列，甚至$n=40$已经太大了。 

关键的结构洞察是通过最大化价差来最大化方差。 由于方差取决于平均值的平方偏差，因此包括最小值和最大值之间的任何中间值都会减少平方距离贡献，而不会增加散布。 这意味着最优子序列将仅使用数组的极值。 

更准确地说，一旦我们意识到最佳子序列将仅包含最小和最大元素，问题就简化为计算仅包含这两个值的多重集的方差。 它们之间任何额外的不同值只会减少方差，因此它不会出现在最佳解决方案中。 

因此，我们只需要考虑由出现的子序列`mn`和`mx`。 最佳配置仅取决于每个出现的次数，但封闭形式进一步简化为直接表达式：`mn`和`mx`，如所提供的解决方案中导出的。 

这种减少将问题从子序列优化转化为扫描阵列后的恒定时间计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 最佳（最小/最大减少）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 读取数组并计算其最小值和最大值。 

此步骤捕获与方差最大化相关的所有信息，因为只有极值才决定分布。 
2. 扫描后，我们得到两个标量：`mn`和`mx`。 

这些定义了对构建最佳子序列重要的唯一候选者。 
3. 计算导出表达式$$ans = 2(mn^2 + mx^2) - (mn + mx)^2.$$该公式来自于扩展二值分布的方差表达式并简化为问题实现中使用的整数可缩放形式。 
4. 通过除以 4 将结果分为整数部分和小数部分，以固定小数格式输出结果。 

除法的产生是因为代数展开式自然引入了分母 4。 

### 为什么它有效

 核心不变量是任何达到最大方差的子序列都可以转换为仅包含全局最小值和全局最大值的出现而不减少方差的子序列。 任何中间值都会严格减少与均值的平方偏差，而不增加极差，因此无法改善目标。 一旦问题空间折叠为两个值，方差仅取决于它们的计数和值，并且最佳计数由语句推导中提供的封闭式简化隐式处理。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    mn = 10**30
    mx = -10**30

    for _ in range(n):
        x = int(input())
        if x < mn:
            mn = x
        if x > mx:
            mx = x

    ans = (mx * mx + mn * mn) * 2 - (mx + mn) * (mx + mn)

    print(ans // 4, end=".")
    frac = (ans % 4) * 25
    print(f"{frac:02d}")

t = int(input())
for _ in range(t):
    solve()
```该实现首先执行单个线性扫描以提取最小值和最大值。 除了两个变量之外不需要额外的存储。 

表达式为`ans`的计算与编辑推导中完全相同。 整个过程中都使用整数运算以避免精度问题。 

最终输出格式取决于分母固定为 4，因此余数始终为 0、1、2 或 3。乘以 25 可将这些情况转换为正确的小数百分位。

 ## 边缘情况

 当所有值都相同时，算法设置`mn = mx`，并且计算的表达式折叠为零。 例如，输入`[7, 7, 7]`产量`ans = 0`，所以输出正好是`0.00`无需任何浮点计算。 

当值包含负数时，例如`[-5, 10]`，该算法仍然正确地识别极值并纯粹根据它们的代数关系计算方差。 该公式仍然有效，因为它仅取决于算术运算`mn`和`mx`，而不是基于符号假设。 

当数组只有一个元素时，`mn`等于`mx`，并且表达式的计算结果再次为零。 扫描自然地处理这个问题，不需要特殊的分支，因为 min 和 max 都从相同的初始化开始并一致更新。
