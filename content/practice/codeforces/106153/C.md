 ## 解决方案
 ## 问题理解

 我们有一块尺寸为的长方形板$n \times m$。 我们还有尺寸为矩形的块$a \times b$。 任务是确定在棋盘上可以放置多少个这样的块，如果它们与网格对齐并且不能重叠，而且最重要的是，它们不能旋转。 

解释很简单：我们从左上角开始平铺棋盘，将每个块放置在固定的宽度方向上$a$和身高$b$。 问题归结为沿垂直维度适合多少个完整块以及沿水平维度适合多少个完整块，然后将这两个计数结合起来。 

输入由多个测试用例组成，每个测试用例给出四个代表板尺寸和块尺寸的整数。 对于每个测试用例，我们输出一个整数，表示适合的块的最大数量。 

约束足够小，每个测​​试用例必须在恒定时间内处理。 即使有多达$10^5$测试用例中，任何每个案例执行多个算术运算的解决方案都会太慢。 这立即排除了对网格的任何模拟或任何逐一放置块的尝试。 

人们有时在此类问题中犯的一个微妙错误是假设允许轮换。 如果允许轮换，我们需要考虑两者$(a, b)$和$(b, a)$并取得更好的结果。 另一个错误是试图考虑部分重叠或贪婪的打包策略，这在这里是不必要的，因为网格结构使解决方案纯粹是算术。 

值得强调的一个边缘情况是$a > n$或者$b > m$。 在这种情况下，根本没有任何块适合，答案应该是零。 例如，如果$n = 3, m = 3, a = 5, b = 1$，那么不可能垂直放置，因此输出必须是$0$，即使存在水平空间。 

## 方法

 思考问题最直接的方法就是模拟放置方块。 我们可以迭代网格中所有可能的左上角位置并检查块是否适合。 这将涉及检查$n \cdot m$每个测试用例的位置，并且每次检查都将涉及恒定时间验证。 这导致$O(nm)$每个测试用例，这已经是不必要的，但在概念上仍然是可以管理的。 

然而，这种模拟忽略了问题的结构。 一旦我们观察到块是与网格对齐的固定矩形，我们就会发现沿行和列的放置是独立的。 沿着垂直方向，我们只能拟合$\lfloor n / a \rfloor$完整的块堆叠。 沿着水平方向，我们只能拟合$\lfloor m / b \rfloor$块并排。 一个方向上的每个有效展示位置都与另一方向上的展示位置独立组合，因此总计数就是它们的乘积。 

这将问题从几何填充任务简化为两个整数除法和乘法。 关键的结构见解是，因为没有旋转，也不允许部分放置，所以网格分解成独立的条带。

 ## 算法演练

 1.读取测试用例的数量。 每个测试用例都是独立的，因此我们可以单独处理它们而不存储任何状态。 
2. 对于每个测试用例，读取四个整数$n, m, a, b$，代表板尺寸和块尺寸。 目标是计算有多少个完整的$a \times b$矩形适合$n \times m$长方形。 
3. 使用整数除法计算垂直方向有多少个块$n // a$。 这代表有多少整行的高度$a$可以堆叠，不超过板的高度。 
4. 使用整数除法计算水平放置的块数$m // b$。 这表示有多少个完整的列宽度$b$可以全盘放置。 
5. 将这两个值相乘以获得块的总数。 这是有效的，因为每个垂直槽可以独立地与每个水平槽组合以形成有效的放置区域。 
6. 输出测试用例的结果。 

### 为什么它有效

 关键的性质是维度的可分性。 由于每个块必须占据连续的$a \times b$区域并且不能旋转，每个有效位置唯一对应于在其中选择一个垂直索引$\lfloor n / a \rfloor$可能性和水平索引$\lfloor m / b \rfloor$的可能性。 这些选择之间没有相互作用，并且没有剩余区域可以对另一个块做出部分贡献。 这将创建有效位置的笛卡尔积结构，从而保证两个整数商相乘的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, a, b = map(int, input().split())
    print((n // a) * (m // b))

t = int(input())
for _ in range(t):
    solve()
```该解决方案读取每个测试用例并直接应用整数除法来计算每个维度适合多少个完整段。 乘法步骤是安全的，因为涉及的值足够小，足以适合 Python 整数。 

一个常见的陷阱是忘记整数除法并使用浮点除法，这会引入精度问题。 另一个错误是试图向上舍入而不是向下舍入； 只有完整的街区才算在内，因此楼层划分至关重要。 

运算顺序在这里并不重要，因为乘法是可交换的，但将结构保持为两个单独的除法可以提高清晰度并降低混合维度的风险。

 ## 边缘情况

 当$a > n$，垂直商立即变为零，并且无论宽度如何，算法都输出零。 例如，输入`n=4, m=10, a=5, b=2`产生`(4//5)*(10//2) = 0*5 = 0`，符合无法放置方块的事实。 

什么时候$b > m$，同样的对称性适用于水平方向。 即使垂直堆叠是可能的，缺乏水平拟合也会迫使结果为零。 

当两个维度均分时，例如`n=8, m=6, a=2, b=3`，计算变为`(8//2)*(6//3) = 4*2 = 8`，它完全对应于没有剩余空间的完整网格分区。
