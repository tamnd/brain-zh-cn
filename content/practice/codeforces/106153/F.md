 ## 问题理解

 我们为每个测试用例提供一个整数数组，并要求我们选择两个不同的元素并评估由它们形成的表达式。 表达式不固定：对于任何不同索引的有序对，我们可以计算三个可能的值，即它们的和、它们的差和它们的乘积。 任务是确定两个不同元素和所有三个操作的所有选择可获得的最大值。 

输入由多个测试用例组成。 每个测试用例都提供一个可以包含正数、负数和零的数组。 每个测试用例的输出都是一个整数，这是通过选择两个不同位置并应用允许的操作之一可实现的最大值。 

简单的解读表明，所有对的枚举都是 O(n²) 的，如果测试用例中的 n 很大，那么这已经是边界了。 然而，运营结构严重限制了重要的事情。 

关键的微妙之处在于，并非数组中的所有元素都同样相关。 极值主导所有三种操作。 对于乘法，最佳候选通常是最大的正数或最大的负数。 对于减法，最好的结果来自于将最大的数与最小的数配对。 对于加法，再次以最大值为主。 这种“有趣”的候选者集中在极端情况下使得问题变得可压缩。 

在以下情况下，粗心的解决方案通常会失败：

 输入：```
3
-100 1 2
```天真地只关注大正数会尝试 2 * 1 或 2 + 1 并错过 2 - (-100) = 102 是最佳值。 

另一个陷阱是忽略两个负数的乘积可以超过任何正数组合：

 输入：```
3
-10 -9 1
```最好的答案是 (-10) * (-9) = 90，它主导所有的和与差。 

这些例子说明了为什么必须对称地考虑正极端和负极端。 

## 方法

 暴力方法迭代每个有序索引对并计算所有三个表达式。 这是直接且正确的，因为它直接评估定义。 然而，它每个测试用例执行 O(n²) 对，当 n 很大时，这变得不可行。 

关键的观察是只有恒定数量的元素才能成为最佳解决方案的一部分。 对于乘法和减法，只有极值才重要：数组中最大的两个和最小的两个值已经涵盖了每个最佳配对情况。 正值和负值表现对称，因此当零可以与负值相互作用时，我们还需要考虑零。 

我们不是评估所有对，而是将数组缩减为一个由两个最大值、两个最小值和零（如果存在）组成的小候选集。 每个最佳答案必须至少涉及这些候选者之一，因为任何非极端元素都可以替换为更极端的元素，以改进或保留所有三种操作的结果。 

蛮力之所以有效，是因为它探索了一切，但当 n 增大时，它就会失败。 只有极值才重要的观察结果让我们将问题压缩到最多恒定数量的候选者，然后检查这个缩减集中的所有对。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳（仅限极端）| O(n log n) 或 O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 将数组分成三个概念组：正数、负数和零。 这种分离很有用，因为不同符号的乘法表现不同。 
2. 对于正数，仅保留两个最小值和两个最大值。 最小的积极因素对于减法可能很重要，而最大的积极因素对于所有涉及增长的业务都很重要。 
3. 对于负数，也仅保留两个最小（最负）和两个最大（最接近零）值。 最负的值对于乘法很重要，而最接近零的值可以与减法中的正值相互作用。 
4. 如果至少有一个零，则在候选集中包含一个零。 零仅与乘法和减法边缘行为相关，它可以主导或消除值。 
5. 将所有选定的候选者组合成一个小列表，通常最多 8 或 9 个。 
6. 枚举所有有序的不同候选对。 对于每对 (i, j)，计算 a[i] + a[j]、a[i] - a[j] 和 a[i] * a[j]，并跟踪最大值。 
7. 输出找到的最佳值。 

正确性取决于这样一个事实：每个操作对于在适当的方向上用更极端的元素替换元素而言都是单调的。 如果候选值不属于极值，则用更极值替换它不会使三个表达式中的任何一个恶化，并且可以严格改进至少一个。 

## 为什么它有效

 任何最佳对都必须包含在各自符号域中处于极值的元素。 对于加法，用较大的值替换任一操作数只会增加结果。 对于减法，最大化第一项并最小化第二项会产生最佳值，因此只有全局最大值和最小值很重要。 对于乘法，符号结构将候选者减少到两个最大的正数或两个最小的负数。 因此，任何最优解都可以映射到缩减候选集中的一对，而不会减少其值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    pos = []
    neg = []
    zero = False
    
    for x in a:
        if x == 0:
            zero = True
        elif x > 0:
            pos.append(x)
        else:
            neg.append(x)
    
    def add_candidates(arr, cand):
        arr.sort()
        m = len(arr)
        for i in range(min(2, m)):
            cand.append(arr[i])
        for i in range(max(0, m - 2), m):
            cand.append(arr[i])
    
    cand = []
    add_candidates(pos, cand)
    add_candidates(neg, cand)
    
    if zero:
        cand.append(0)
    
    ans = -10**18
    m = len(cand)
    
    for i in range(m):
        for j in range(m):
            if i == j:
                continue
            x, y = cand[i], cand[j]
            ans = max(ans, x + y, x - y, x * y)
    
    print(ans)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```实现直接遵循归约思想。 我们通过符号显式地分隔值，这样我们就可以安全地仅提取边界元素。 辅助函数在排序后从每组中选择两个最小和两个最大元素。 这确保我们捕获价值范围的两端。 

候选集足够小，因此完整的成对检查是安全的。 双循环是每个测试用例的持续工作。 请注意通过 Python 整数隐式使用 64 位整数行为，这避免了原始 C++ 上下文中存在的溢出问题。

 ## 边缘情况

 当最佳值来自于将大的正值与大的负值混合时，就会出现关键的边缘情况。 例如，输入`[-100, 1, 2]`产生 202 来自`2 - (-100)`或乘法 200，具体取决于存在的值。 该算法保留两个极端，因此在最终枚举中直接评估两个候选者。 

另一个边缘情况是零与负数一起出现。 如果没有明确包含零，则类似的情况`[-5, 0, 1]`会错过正确答案`5`从`0 - (-5)`。 该算法显式地将零插入候选集中，确保考虑涉及零的减法和乘法。 

最后，当所有值都相同时，例如`[7, 7, 7]`，候选约简仍然保持正确性，因为最大值和最小值相同，并且配对评估很容易产生正确的重复乘积、总和和差。
