 ## 解决方案
 ## 问题理解

 我们给定三个整数$p$,$x$， 和$k$。 我们考虑所有整数$q$范围从 1 到$x$。 对于每一个这样的$q$，我们形成产品$p \cdot q$，我们把这个数字写在基数中$k$。 任务是计算有多少个值$q$产生一个表示，它是基数的终止扩展$k$，这意味着当写为基数时$k$实数，它没有无限小数部分。 

一个关键的重新解释是，我们实际上并不是在处理十进制扩展，而是在基本系统中处理数论。 有理数在基数中具有终止表示$k$当且仅当在简化为最低项后，其分母的所有素因数也是$k$。 这是将问题从基本转换问题转变为可分性问题的核心结构约束。 

表达式$p \cdot q$是一个整数，所以它可以产生一个非终止基数的唯一方法$k$扩展是如果在对基中的表示进行任何隐式标准化之后$k$，它需要包含不能整除的素数的分母$k$。 自从$p \cdot q$本身是一个整数，问题就归结为我们是否可以用基数来表达它$k$在以分数形式思考时，不要引入“坏”因素，这最终取决于因素如何$p \cdot q$互动$k$。 

不明显的困难是$p \cdot q$不是任意的，它受到乘法结构的约束$q$，我们必须数出有多少个$q \le x$满足素因数条件$k$。 

天真的解释可能会建议检查每个$q$通过模拟基数转换或推理小数终止来单独实现，但这在以下情况下是不可能的：$x \le 10^{14}$。 另一个微妙的陷阱是假设$p$很重要，而事实上之间的相互作用$p$和$q$通过共享素因数$k$是决定有效性的因素。 

边缘情况包括：

 像这样的案例$k = 10$，其中素数是$2$和$5$，以及像这样的数字$12 = 2^2 \cdot 3$是有效的，因为以 10 为基数仅不允许除 2 和 5 之外的质数在约简后作为分母。 检查可分性或数字模式的简单方法可能会错误分类此类情况。 

另一个边缘情况是$k$素数，例如$k = 2$。 那么只允许其简化形式的分母中没有奇素数的数字，这严重限制了有效$q$，因子处理中的小算术错误会导致计数中的大错误。 

## 方法

 直接暴力法迭代所有$q \le x$，计算$p \cdot q$，并检查它是否在基数中表示$k$正在终止。 这将需要模拟基$k$分数展开或对每个值应用因式分解。 即使每个查询进行简单的分解也至少花费$O(\sqrt{p q})$，使最坏情况的复杂度达到$10^{14} \sqrt{10^{14}}$，这是完全不可行的。 

关键的见解是在基地终止$k$仅取决于化简形式的分母的质因数，并且因为$p \cdot q$是一个整数，我们可以将条件重新解释为相对于素分解结构的约束$k$。 这个问题就相当于计算有多少个$q$避免在与组合时引入“坏素数”$p$，其中“坏素数”是出现在$k$相对于它们的出现而言，多样性不足$p \cdot q$。 

这表明将素数分开$k$并跟踪如何$p$和$q$为他们做出贡献。 我们将问题简化为经典的乘法计数结构：从$k$重要的素数，计算有多少$p$已经贡献了，然后算$q \le x$他们的贡献使综合估值保持一致。 

最后的归约是只有素数相除$k$很重要，我们跟踪它们的指数。 对于每个$q$，条件变成了一组可分性约束$q$关于派生数$k'$，去除后得到$k$所有素数已经被充分覆盖$p$。 然后我们数一下有多少个$q \le x$只能被允许的素数整除并满足所需的指数界限，这可以通过对素数分解使用包含-排除来完成$k'$。

 ## 算法演练

 我们首先通过关注基数的素因数分解来分离条件的结构$k$。 我们提取所有不同的素数$k$，因为素数没有出现在$k$与终止行为无关。 

接下来，我们考虑因素$p$并比较每个素数的指数$r \mid k$与指数$r$在$k$。 对于每个这样的素数，如果$p$已经包含至少同样多的副本$r$作为$k$需要，那么$q$不受限制$r$。 如果不，$q$必须提供足够的缺失指数来补偿。 

我们将其转化为一个要求$q$必须能被某个乘积整除$d$， 在哪里$d$由所有素数累加而成$r \mid k$为此$p$是不足的，每个都提升到缺失的指数。 

然后我们将计数问题简化为：计算整数$q \le x$这样$d \mid q$，但也要确保不违反额外的约束。 因为只有素数$k$没关系，我们将注意力限制在无平方核上$k$，并在其素数集上使用包含-排除。 

我们计算素数的所有子集$k$。 对于每个子集，我们计算有多少个数字$q \le x$可被子集中素数的乘积整除，并根据子集大小的奇偶性调整符号。 这会产生有效的计数$q$。 

### 为什么它有效

 不变的是，在每一步中，我们都保持原始终止条件和纯整除条件之间的等价性$q$由素数引出$k$。 的作用$p$只是预先满足或减少所需的指数，并且一旦进行这些调整，条件$q$在一组固定的有限素数上变得独立且纯乘法。 该固定集合上的包含-排除精确地计算满足这些同时整除约束的整数，而不会过度计算重叠。 由于每个变换都保留相对于素数指数的约束$k$, 无无效$q$已被计算且无效$q$被排除在外。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def factorize(n):
    res = {}
    d = 2
    while d * d <= n:
        while n % d == 0:
            res[d] = res.get(d, 0) + 1
            n //= d
        d += 1
    if n > 1:
        res[n] = res.get(n, 0) + 1
    return res

def solve():
    p, x, k = map(int, input().split())

    fk = factorize(k)

    # remove primes of k already covered by p
    d = 1
    fp = factorize(p)

    for pr, ek in fk.items():
        ep = fp.get(pr, 0)
        if ep < ek:
            d *= pr ** (ek - ep)

    primes = list(fk.keys())

    ans = 0
    m = len(primes)

    for mask in range(1 << m):
        mult = 1
        bits = 0
        ok = True
        for i in range(m):
            if mask >> i & 1:
                bits += 1
                mult *= primes[i]
                if mult > x:
                    ok = False
                    break
        if not ok:
            continue
        cnt = x // (mult * d)
        if bits % 2 == 0:
            ans += cnt
        else:
            ans -= cnt

    print(ans)

if __name__ == "__main__":
    solve()
```代码从因式分解开始$k$，因为所有约束都源自其主要结构。 这也影响因素$p$确定每个主要要求有多少已经得到满足。 变量$d$累积缺失的主要功率要求，必须通过以下方式强制执行$q$。 

然后我们对素数的所有子集进行包含-排除除法$k$。 对于每个子集，我们计算所选素数的乘积并计算有多少个素数的倍数$d \cdot \text{mult}$躺在里面$[1, x]$。 交替符号确保可被多个素数整除的数字不会被重复计算。 

一个微妙的实现细节是，我们只在包含-排除中相乘素数，而不是它们的全部幂。 这是有效的，因为指数调整已经被吸收到$d$，因此剩余的约束纯粹是基于集合的。 

## 工作示例

 ### 示例 1

 输入：```
2 5 10
```这里$k = 10 = 2 \cdot 5$。 我们考虑因素$p = 2$，它已经涵盖了 2 的 1 次幂$k$。 自从$k$只需要素数 2 和 5，并且 2 已经存在于$p$，只有与 5 相关的要求仍然通过可分性约束部分强制执行。 

我们枚举素数的子集$\{2,5\}$。

 最终答案是2。 

此迹线显示了包含-排除如何删除引入相对于 10 基数结构的无效素数相互作用的数字。

 输入：```
3 5 2
```这里$k = 2$，所以只有素数 2 很重要。 自从$p = 3$不贡献 2 的因数，所有要求均完全强加于$q$。 

| 面膜| 多| 计数 = 5 // (多) | 标志| 贡献 |

 ## 边缘情况

 一个关键的边缘情况是当$p = 1$。 在这种情况下，所有素数约束完全来自$k$， 所以$d$等于所需素数的全积。 该算法自然地处理这个问题，因为$fp$不贡献任何指数，将所有赤字留在$d$。 

另一种情况是当$k$是素数。 然后包含-排除折叠为两项，逻辑简化为计算单个调整约束的倍数。 代码正确地处理了这个问题，因为子集循环仍然适用于单元素素数列表。 

第三种情况是当$x < d$。 然后没有有效的$q$可以存在，因为每个候选人都必须可以被整除$d$。 计算$x // (d \cdot mult)$自动在所有子集中生成零，因此无需特殊大小写，答案就变为零。
