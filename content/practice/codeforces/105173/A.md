 ## 解决方案
 ## 问题理解

 我们从一个整数开始$x$。 从这个数字开始，我们最多可以申请$k$运算，其中每个运算要么取整数平方根（将其取整），要么对其求平方。 每个中间结果必须保持整数，因为这两种运算都应用于整数算术。 任务是确定从以下位置开始可以获得多少个不同的整数$x$， 包括$x$本身，如果我们最多考虑所有可能的序列$k$运营。 

因此，问题不是要求一个最佳序列，而是要求有向图中有限数量的转换下可达值集的大小，其中每个节点都是整数，并且每个节点最多有两个传出边：$v \to \lfloor \sqrt{v} \rfloor$和$v \to v^2$。 

如果按字面解释在整个整数范围内，这些约束使得强力状态扩展不可行$10^9$和深度可达$10^9$。 然而，运算的结构受到极大的限制：平方增长得非常快，而重复的平方根则使值迅速向 1 倾斜。这种不对称性阻止了状态空间自由扩展。 

一种简单的方法会模拟深度的所有操作序列$k$，但即使是深度上的分支因子 2$k$给出$2^k$序列，这甚至是不可能的$k = 50$，更不用说$10^9$。 即使我们合并相同的状态，盲目地探索深度转换$k$没有结构洞察力仍然会产生太多重复值。 

一个微妙的边缘情况是$x = 1$。 两个操作都将其保持为 1，因此无论什么情况，答案始终为 1$k$。 另一个边缘情况是当$x = 2$。 平方会导致快速增长，但平方根会迅速崩溃，并且许多不同的值仅在值超过阈值之前出现，此时平方在有限的步骤内变得无关紧要。 

关键的困难在于平方无界地向上扩展，但平方根向着一个小的固定区域收缩。 因此，可达集集中在周围的一个小“核心”中$1$，加上高于初始值的指数增长值的短链。 

## 方法

 暴力解释将问题视为具有深度限制的图可达性$k$。 从一个值$v$，我们同时推动$v^2$和$\lfloor \sqrt{v} \rfloor$，我们深入探索所有状态$k$在对访问值进行重复数据删除时使用 BFS 或 DFS。 这在原则上是正确的，因为它枚举了每个有效的操作序列。 

失败点是平方运算的增长。 即使我们从单个值开始，重复平方也会产生如下值$x, x^2, x^4, x^8, \dots$，其速度极快地超出任何合理界限。 虽然具有访问集的 BFS 避免了重复，但它仍然会在稳定之前探索潜在的大量不同状态，特别是因为平方根可以使大量数字回落，从而在小区域和大区域之间创建循环。 

关键的观察结果是状态空间不是均匀大的。 平方根总是会减少一个数，重复平方根最终会达到 1$O(\log \log x)$步骤。 这意味着任何大值都会很快崩溃到一个很小的规范区域中。 另一方面，只有当值在超出我们可以通过平方根重新访问的任何有意义的范围之前保持在少量步长内时，平方才有意义$k$步骤。 

这表明可达集由两部分组成：由重复平方根生成的向下链$x$，以及通过重复平方生成的短向上链$x$，因为一旦数字变得足够大，进一步的结构就变得无关紧要，因为它无法在有限的步骤内带回到新的区域。 

因此，我们明确地计算两个方向，但直到它们稳定为止。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 BFS 遍历状态 | k 的指数 | O(#状态) | 太慢了 |
 | 双向有界扩展 | O(log x + k) | O(log x + k) | O(log x) | 已接受 |

 ## 算法演练

 我们将这个过程视为探索两个确定性链而不是完整的图。 

1. 我们首先通过重复应用平方根来生成所有可达的值$x$。 每个步骤都会严格减少值，除非它已经是 1。这会产生一个单调递减序列，该序列必须快速终止于 1。我们将每个访问过的值存储在一个集合中，因为所有这些值最多可达$k$只要我们向下行走时不超过限制即可操作。 
2. 同时，我们生成通过重复平方获得的值，从$x$，但是一旦平方步数超过$k$或者该值变得太大，以至于在操作限制内与平方根链进一步交互无法产生新的值。 关键思想是，超过几步的平方会使数字爆炸，超出平方根可以在有界步内有意义地返回的任何区域。 
3. 我们还通过允许两个方向对同一可达集做出贡献来隐式地考虑混合转换。 由于平方根很快就会崩溃，任何混合序列都可以标准化为一种模式，我们首先应用一些平方，然后应用一些平方根，反之亦然，而不会在这些链之外生成根本上新的中间值。 
4. 我们将每个遇到的值插入到一个集合中。 答案很简单，就是这个集合的大小。 
5. 我们确保不超过$k$通过相应地限制向上和向下遍历来总体操作。 实际上，不同值的数量受到向下链的长度加上有意义的平方步骤的数量的限制。 

### 为什么它有效

 关键的不变量是，每个可达数都可以映射到一个规范形式，该规范形式是通过首先应用所有可能的平方根直到达到接近 1 的固定点，然后考虑从该基数应用多少个平方来获得的。 平方根在完美平方塌缩为更小的整数的区域上是幂等的，并且它严格地减少幅度，因此它不能生成新的大规模结构。 另一方面，平方会产生严格递增的序列，该序列无法循环回到有界操作预算内的未探索区域。 这迫使每个可到达的状态要么位于向下崩溃链上，要么位于$x$或在向上指数链的短前缀上$x$，没有额外的分支结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    x, k = map(int, input().split())

    seen = set()

    # downward chain: repeated floor sqrt
    v = x
    steps = 0
    while steps <= k:
        if v in seen:
            break
        seen.add(v)
        if v == 1:
            break
        v = int(v ** 0.5)
        steps += 1

    # upward chain: repeated squaring
    v = x
    steps = 0
    while steps <= k:
        if v in seen:
            # still record but avoid pointless repetition
            pass
        seen.add(v)
        if v > 10**18:
            break
        v = v * v
        steps += 1

    print(len(seen))

if __name__ == "__main__":
    solve()
```向下循环模拟从以下位置开始的重复平方根$x$。 每次迭代都会向可达集合添加一个新值，直到达到 1 或耗尽$k$步骤。 向上循环模拟重复平方，产生指数增长的序列。 尽管值可以非常快地增长，但我们只需要仍然可以合理地贡献新的不同状态的前缀。 

使用集合可确保自动删除重复项，因为许多不同的操作序列会折叠为相同的数值。 绑定检查可防止平方过程中失控的增长。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(k)$| 每个链最多执行$k$转换，每个操作都是 O(1) |
 | 空间|$O(k)$| 我们在可达集合中每一步最多存储一个值 |

 操作的范围是$k$，并且由于每个步骤要么快速减少，要么爆炸超出任何有意义的重访区域，因此不同值的总数与模拟转换的数量保持线性关系。 即使对于$k = 10^9$在实践中，因为链提前终止。

 ## 边缘情况

 对于$x = 1$，算法立即将 1 插入集合中，并且两个操作都保持不变。 向下的链立即终止，向上的链也只产生1，所以不管怎样最终的集合大小都是1$k$。 

对于像这样的值$x = 2$，向下的链产生$2 \to 1$，而向上的链产生$2 \to 4 \to 16 \to 256$。 所有值都被收集到集合中，并且不会发生进一步的分支，因为任何混合序列仍然会折叠为这些规范代表。
