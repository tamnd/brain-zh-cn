 ## 解决方案
 ## 问题理解

 我们得到一个代表线性化学条的长字符串。 条带上的每个位置都是一个小写字母。 有些字母被标记为“有风险”。 对于该条带的任何子字符串，我们可以计算它包含多少个危险字符。 任务是计算有多少个子串恰好包含 K 个危险字符。 

子字符串是通过选择两个索引 l 和 r（其中 l ≤ r）并获取它们之间的所有字符来定义的。 由于字符串长度可达一百万，我们显然无法枚举所有 O(N²) 子字符串。 

输出是一个整数：风险字母数量恰好为 K 的子串总数。 

主要的困难是我们要计算所有子串，这些子串的数量是二次方的。 任何显式检查子字符串的方法都会超时。 这立即迫使采用线性或近线性方法。 

当 K 为零时，会出现微妙的边缘情况。 在这种情况下，我们正在计算根本不包含危险字符的子字符串，因此答案完全取决于安全字符的最大块。 另一个边缘情况是当 L = 0 时，这意味着根本不存在风险字母。 那么只有当 K = 0 时每个子串才有效，否则答案为零。 

## 方法

 蛮力的想法很简单。 对于每个子字符串，我们计算它包含多少个危险字符，并检查它是否等于 K。这在概念上是有效的，因为它直接匹配问题的定义。 然而，如果简单地完成，每个子字符串需要 O(N) 来评估，并且有 O(N²) 个子字符串，导致总复杂度为 O(N³)。 即使我们使用前缀和来优化计数，每个子字符串仍然需要 O(1)，剩下 O(N²)，这对于 N 高达 10⁶ 来说太大了。 

关键的观察是“恰好 K 个危险字符”的条件可以转化为标准的前缀和计数问题。 我们将字符串转换为二进制数组，其中每个位置如果是危险字符则为 1，否则为 0。 现在问题变成了计算总和恰好为 K 的子数组。 

对于此类问题，前缀和是自然的工具。 令 prefix[i] 为前 i 个位置中的危险字符数。 那么子串 (l, r) 的总和为 K 当且仅当 prefix[r] - prefix[l - 1] = K，重新排列为 prefix[l - 1] = prefix[r] - K。这意味着对于每个 r，我们需要知道有多少个先前的前缀和等于 prefix[r] - K。 

当我们扫描一次字符串时，我们维护一个前缀和的频率图。 这会产生 O(N) 解。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N²) 或更差 | O(1) | O(1) | 太慢了|
 | 前缀和+哈希| O(N) | O(N) | 已接受 |

 ## 算法演练

 我们首先将危险字符集转换为快速查找结构，通常是布尔数组或集合。 然后，我们从左到右扫描字符串，同时维护风险字符的运行前缀计数。 

1. 我们初始化一个频率图，它存储每个前缀和出现的次数。 我们首先插入频率为 1 的前缀和 0，因为空前缀的风险字符为零。 
2. 我们迭代字符串，更新运行计数器 curr。 如果当前角色有风险，我们将 curr 加 1。否则，保持不变。 这记录了迄今为止我们见过的危险角色的数量。 
3. 在每个位置，我们想知道有多少个较早的前缀会形成一个以 K 个危险字符结尾的子串。 该条件转化为查找 curr - K 之前出现过多少次。 
4. 我们将该频率添加到答案中。 
5. 然后，我们通过增加 curr 的计数来更新频率图。

每个步骤都建立在前面的前缀结构的基础上，确保以当前位置结尾的每个子字符串满足条件时都被精确计数一次。 

关键不变量是在索引 i 处，频率图准确包含从位置 0 到 i 的所有前缀和的计数。 这确保使用唯一的较早前缀对以 i 结尾的每个有效子字符串进行计数，并且不包含无效子字符串，因为前缀总和的差异强制执行精确的 K 约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, l = map(int, input().split())
    s = input().strip()
    risky = set(input().strip())

    # prefix sum frequency map
    freq = {0: 1}
    curr = 0
    ans = 0

    for ch in s:
        if ch in risky:
            curr += 1

        need = curr - k
        ans += freq.get(need, 0)

        freq[curr] = freq.get(curr, 0) + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先读取输入并将有风险的字母存储在一组中以进行 O(1) 成员资格检查。 我们维护 curr 作为风险字符的运行前缀计数。 字典 freq 存储到目前为止每个前缀和已被观察到的次数。 对于每个字符，我们计算有多少个较早的前缀将形成以当前索引结尾的有效子字符串，将其累积到 ans 中，然后更新前缀频率图。 

一个常见的错误是在查询之前更新频率图，这会错误地允许空子串或错误计数自对。 正确的顺序是先查询，再更新。

 ## 边缘情况

 当 L = 0 时，风险集为空，因此 curr 永远不会增加。 当 K = 0 时，该算法仅正确计算子字符串，因为只有零匹配的前缀差异。 

当 K 大于任何可能的危险字符数时，查找频率[curr - K] 总是安全失败并贡献零。 

当字符串包含所有危险字符时，curr 会增加每一步和前缀频率，确保所有子数组都被正确计数为前缀差异的组合。
