 ## 解决方案
 ## 问题理解

 我们正在跟踪当需求每月翻一番时后端系统如何随着时间的推移而增长。 一开始，有几个独立的微服务，每个微服务已经处理一定数量的请求。 总流量只是这些初始负载的总和。 

每个月，总需求都会翻倍。 当需求增长时，我们必须决定需要多少物理主机来为其提供服务。 每个主机都有固定的容量限制。 如果需求超出单个主机的处理能力，我们会将流量分散到多个相同的主机上，以便任何主机都不会超出容量。 

问题不是针对每个查询手动逐月模拟系统，而是回答，对于请求的几个月，重复翻倍后需要多少主机。 

就微服务和月份而言，限制很小，但关键的困难在于，单独对每个月进行简单模拟会重复重新计算指数增长，这是不必要的。 由于增长纯粹是翻倍，因此第 m 个月的总负载确定性地缩放为 2^m 倍，因此一旦知道初始总数，就可以独立回答每个查询。 

当考虑溢出或零初始负载时，边缘情况会立即出现。 如果所有服务都从零开始，则每个月都保持为零并且需要零个主机。 如果初始总和能被容量整除，则余数不需要额外的主机，如果四舍五入不正确，则很容易出现错误处理。 

另一个微妙的情况是非常大的增长：经过 54 次加倍后，值可以超过 10^16 或更多，因此任何依赖一步一步的朴素整数模拟而不关心缩放的解决方案将在 Python 中保持正确，但在固定宽度语言中会溢出。 此外，在没有缓存的情况下重复重新计算每个查询的幂是不必要的，但考虑到 M 较小，仍然可以接受。 

## 方法

 蛮力的想法很简单。 我们通过对所有微服务求和来计算总初始流量。 然后，对于每个月，我们重复将该值加倍 m 次，并通过除以容量并向上舍入来计算需要多少主机。 这是有效的，因为该过程纯粹是指数缩放。 

暴力模拟的问题不在于正确性，而在于效率低下和冗余。 如果我们通过循环 m 次为每个查询单独重新计算 2^m，即使 m 至多为 54，我们也会重复工作。这仍然很小，但更简洁的方法是预先计算一次 2 的幂并重用它们。 

关键的观察结果是所有查询都是独立的并且仅依赖于表达式 Total * 2^m。 这将问题转化为简单的预处理+每个查询的恒定时间评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 使用循环模拟每个查询 | O(N + M·m) | O(N + M·m) | O(1) | O(1) | 已接受但多余 |
 | 预计算 2 的幂 | O(N + M) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将所有内容减少到一个基值并重复使用指数缩放。 

1. 通过对所有微服务求和来计算初始请求总数。 这表示第 0 个月的系统负载。 
2. 预先计算 2 的幂直至所请求的最大可能月份。 每个条目代表几个月后负载增长了多少。 
3. 对于每个查询月份 m，计算总负载：initial_total 乘以 2^m。 
4.将此负载除以容量C，转换为主机数量。如果有余数，则需要额外一台主机，因为部分使用仍然需要完整的主机。 
5. 独立输出每个查询的结果。 

重要的推理步骤是认识到主机分配仅取决于缩放后的总负载，而不取决于单个服务行为。 

### 为什么它有效

每个月，每个微服务独立地加倍，因此它们的总和也加倍。 这意味着整个系统的行为就像每月乘以固定因子的单个聚合数量。 由于主机分配仅取决于总需求，而不取决于分配，因此将系统减少到一个数量可以保持正确性。 上限划分可确保为任何主机分配的负载不会超过允许的负载。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    C, N, M = map(int, input().split())

    total = 0
    for _ in range(N):
        total += int(input())

    months = []
    max_m = 0
    for _ in range(M):
        m = int(input())
        months.append(m)
        max_m = max(max_m, m)

    pow2 = [1] * (max_m + 1)
    for i in range(1, max_m + 1):
        pow2[i] = pow2[i - 1] * 2

    for m in months:
        load = total * pow2[m]
        hosts = load // C
        if load % C != 0:
            hosts += 1
        print(hosts)

if __name__ == "__main__":
    main()
```该代码首先将所有微服务负载聚合为一个值。 然后，它构建一个 2 的幂表，以便可以在恒定时间内回答每个查询。 关键步骤是手动使用整数算术进行上限除法，因为浮点舍入会在大值时引入错误。

 ## 边缘情况

 如果所有微服务均为零，则无论月份如何，总数仍为零。 该算法正确地输出零主机，因为负载保持为零并且除法产生零。 

如果初始总数可以被 C 整除，则不需要舍入，并且主机计数是准确的。 整数除法分支正确地避免了添加额外的主机。 

如果月份包含零，则算法使用 2^0 = 1，保留原始负载，这确保月份 0 的处理一致，无需特殊大小写。
