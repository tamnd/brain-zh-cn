 ## 解决方案
 ## 问题理解

 我们正在模拟长度直线段上的受约束运动过程$N$。 Amy 从位置 0 开始，最终必须到达位置$N$。 她的动作不是任意的：第一个动作固定为长度1，最后一个动作也必须是长度1，并且每个中间动作都依赖于前一个动作。 如果她最后的跳跃有长度$k$，下一次跳跃必须有长度$k-1$,$k$， 或者$k+1$，并且跳跃长度必须始终保持正值。 

我们关心的数量是准确着陆所需的最小跳跃次数$N$根据这些规则。 

约束条件$N \le 10^{12}$排除任何关于位置的动态规划或关于距离的直接状态模拟。 任何探索所有跳跃序列的解决方案都将呈指数级爆炸，因为有效序列的数量随着时间的推移而迅速增长$N$。 

一个微妙的边缘条件是两端的强制结构。 第一次和最后一次跳跃固定为 1。这会创建一个“山状”结构：我们必须从 1 增加到某个峰值，然后再减少回 1，而所有跳跃长度的总和正好是$N$。 即使是很小的例子也表明这是多么严格：

 对于$N = 2$， 仅有的$1 + 1$有效，提供 2 次跳跃。 

为了$N = 3$，我们必须使用$1 + 1 + 1$，因为任何立即增加的尝试都会超出限制。 

为了$N = 4$，最优的是$1 + 2 + 1$，产生 3 次跳跃。 

关键的困难在于，允许的步长变化限制了我们增长和收缩的速度，因此我们正在有效地优化形状受到严格控制的约束序列。 

## 方法

 一种直接的方法是使用递归或 BFS 状态来尝试所有有效的跳转序列$(position, last\_jump)$。 从每个状态，我们分支到最多三个下一个跳跃尺寸。 当我们到达确切位置时我们停下来$N$最终跳跃为 1。 

这是正确的，但完全不可行。 状态数量随着位置和最后跳转大小的增加而增长，并且分支因子是恒定的，但应用于可以达到大约的路径长度$N$。 即使对于中等$N$，这变成指数。 

关键的观察结果是，最佳序列具有非常结构化的形状。 由于跳跃最多只能改变 1，因此任何最大速度策略都必须从 1 增加到某个峰值$h$，可能保持在该值附近，然后对称地减小回 1。因此，最小化跳跃次数的序列与我们可以适应多大的“三角累积”密切相关$N$。 

如果我们固定最大高度$h$，从 1 上升到的最短可能序列$h$然后返回 1 具有固定长度和固定总和。 增加斜坡的总和是$1 + 2 + \dots + h = \frac{h(h+1)}{2}$，并且除了峰值重叠之外，完整的上下结构使该值加倍。 从这个结构中，我们可以得出需要多少次跳转来表示给定的$N$，然后进行最小调整$N$并不完全是一个完美的对称结构。 

我们不是模拟序列，而是计算峰值可以有多大，同时保持在$N$，然后根据剩余距离调整最终答案。

 ## 算法演练

 1. 我们将该过程解释为构建一个序列，该序列首先从 1 每步最多增加 1 到某个峰值，然后减少回 1。该结构受到 ±1 约束以及开始和结束跳跃为 1 的要求的强制。 
2. 我们计算到达峰值高度需要多少步$h$。 向上的部分贡献一个三角和$1 + 2 + \dots + h$，向下的部分反映了它，但排除了峰值重复。 这给出了高度对称的“山”的可预测总距离$h$。 
3.我们选择最大的$h$使得完全对称结构的总距离不超过$N$。 这确保我们使用尽可能大的“快速增长”区域，从而最大限度地减少跳跃次数。 
4.一次$h$是固定的，我们确定需要多少次额外的跳跃来弥补剩余的差距$N - \text{base}(h)$。 由于我们被限制以最多 1 的跳跃长度增量移动，因此每个额外的距离单位都会有效地迫使高原区域增加额外的平坦或调整的步骤。 
5. 我们将基础结构长度​​和剩余校正结合到最终答案中，确保两个端点约束（第一个和最后一个跳跃等于 1）仍然满足。 

### 为什么它有效

 任何有效的序列都受到跳跃长度最多为 1 的局部斜率变化的限制，这意味着累积距离的最快方法是尽可能接近允许的最大跳跃大小。 该结构不可避免地形成一个单峰。 如果有多个峰值，我们会引入不必要的下降和上升步骤，增加总跳跃次数，但不会提高到达范围。 因此，最佳序列始终等效于单峰分布，并且最大化该峰值可以最小化总跳跃计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N = int(input().strip())

    # We binary search the maximum height h such that
    # we can form a valid symmetric structure within N.
    #
    # For a peak h:
    # sum up = h(h+1)/2
    # sum down = (h-1)h/2
    # total distance = h^2

    lo, hi = 1, 10**6
    best = 1

    while lo <= hi:
        mid = (lo + hi) // 2
        if mid * mid <= N:
            best = mid
            lo = mid + 1
        else:
            hi = mid - 1

    h = best

    used = h * h
    remaining = N - used

    # base number of jumps in full peak structure:
    # up: h steps, down: h-1 steps => 2h-1 jumps
    ans = 2 * h - 1

    # Each extra unit beyond perfect square requires extending flat region,
    # effectively increasing jump count by 1 per unit adjustment in this model.
    ans += remaining

    print(ans)

if __name__ == "__main__":
    solve()
```实施的第一部分找到最大的可行峰值高度$h$从而形成完美对称的“先上后下”结构$N$。 所使用的关键身份是这样的结构准确地使用$h^2$总距离，这就是二分查找条件检查的原因$mid^2 \le N$。 

确定后$h$，我们计算还有多少距离未被覆盖。 基地建设贡献确切$h^2$，并且必须通过扩展结构来吸收剩余部分，而不违反跳跃长度的 ±1 限制。 这被建模为对总跳跃计数的额外单位贡献。 

最后，答案从完美峰值的最小跳跃计数开始，$2h - 1$，然后计算剩余距离。 

## 工作示例

 我们在结构可见的小值上追踪逻辑。

 我们挑选$h = 1$， 自从$1^2 \le 2$。 基础结构有效地使用了 1 次跳跃，但我们需要一个额外的单位来达到 2 次，这会强制进行第二次跳跃。 这匹配唯一有效的序列$1 + 1$。 

### 示例 2：$N = 4$| 步骤| 小时 | h² | 剩余| 定点跳伞 | 答案|
 | --- | --- | --- | --- | --- | --- |
 | 开始 | 2 | 4 | 0 | 3 | 3 |
 | 计算后| 2 | 4 | 0 | 3 | 3 |

 这里$h = 2$给出了准确的覆盖范围，因为$2^2 = 4$。 顺序是$1,2,1$，使用 3 次跳跃并准确到达终点。 

这些痕迹表明，基于正方形的结构正确地捕获了最佳解决方案的主要形状。

 ## 边缘情况

 对于$N = 1$，算法仍然选择$h = 1$，并产生一次跳跃，这与以 1 长度跳跃开始和结束的强制结构相匹配。 

对于刚好高于完美平方的值，例如$N = 5$，峰值保持不变，但余数变为非零。 该算法随该余数线性增加跳跃计数，反映了在不改变峰结构的情况下需要额外的校正步骤。 这可以避免过早地错误地增加峰值，从而引入不必要的额外跳跃。 

对于非常大的$N$，二分搜索确保我们永远不会尝试显式构造序列。 计算仅取决于整数平方根行为，因此即使在$10^{12}$。
