 ## 解决方案
 ## 问题理解

 我们被赋予了一个$h \times w$网格代表最初完全充满方形单位块的披萨。 一些细胞仍然存在，标记为`#`，当其他人被吃掉时，标记为`.`。 我们不允许移动任何剩余的棋子，只能分割剩余的棋子。 

任务是选择一个整数$k$并将网格划分为不相交的$k \times k$与网格线对齐的块。 剩下的每一个`#`单元格必须恰好属于一个这样的块，并且只要每个块完全适合网格，就被认为是有效的。 目标是最大化$k$，这意味着我们想要最大可能的正方形尺寸，这样我们就可以仅使用完整的瓷砖来平铺所有剩余的披萨单元$k \times k$正方形。 

问题的结构纯粹是几何的：我们不是选择任意子集，而是强制进行刚性平铺。 这使得对齐和周期性结构的关键困难而不是细胞计数。 

限制因素$h, w \le 2500$暗示最多约$6.25 \times 10^6$细胞。 如果重复扫描子网格，任何独立尝试所有可能的正方形放置的解决方案都会太慢。 解决方案必须避免重新计算每个候选方格的面积信息$O(k^2)$或者$O(hw)$每次检查的方式。 

当网格只有一个时，会出现微妙的边缘情况`#`。 在这种情况下，任何$k$大于 1 是不可能的，因为$k \times k$块将需要多个单元。 当出现另一种边缘情况时`#`细胞非常稀疏，但以周期性方式排列，允许惊人的大$k$，即使总数`#`细胞很小。 

天真的贪婪直觉可能会尝试根据附近的距离来选择正方形的大小`#`单元格，但这会失败，因为局部间距不能保证全局平铺一致性。 要求是全局性的：每个`#`必须适合固定块大小的某些对齐网格。 

## 方法

 直接的暴力方法是尝试每一个可能的边长$k$从 1 到$\min(h,w)$。 对于每个$k$，我们会检查剩余的`#`单元格可以被不相交的覆盖$k \times k$与网格对齐的块。 一个简单的检查将迭代块的所有可能的左上角，并验证每个块是否完全位于有效区域内，以及每个块是否完全位于有效区域内。`#`恰好被覆盖一次。 

正确性很简单，因为我们明确验证了每个的可行性$k$。 问题是性能。 有$O(\min(h,w))$候选人$k$，对于每个我们可以扫描最多$O(hw)$位置和每次检查内最多$O(k^2)$细胞。 这导致最坏情况的复杂性远远超出$10^8$操作，这是不可行的$h,w \le 2500$。 

关键的观察是给定的可行性$k$仅取决于坐标上的模块化对齐约束`#`细胞。 如果存在有效的平铺，则每个`#`必须位于由以下定义的唯一残基类块内$(i \bmod k, j \bmod k)$，并且在每个这样的块中，必须一致地包含或排除所有相应的位置。 这将问题转变为检查网格是否划分为$k \times k$周期性单元，永远不会以不一致的方式跨块边界分割所需单元的连接结构。 

这表明我们可以预先计算网格的表示并测试每个网格$k$有效地，或者扭转观点：而不是验证所有$k$，我们找到最大的$k$不违反由成对引起的局部一致性条件`#`细胞。 这导致了 gcd 式的约束：坐标之间的差异`#`单元格限制有效$k$，因为任何有效的平铺都必须对齐所有`#`位置模数$k$。 

我们可以将其形式化如下：对于任意两个`#`单元格，它们的坐标差必须与相同的模数兼容$k$， 意义$k$必须划分对对齐结构重要的所有差异。 这将搜索空间减少到从坐标差导出的一组有界值的约数，这可以被有效地处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(h w \cdot \min(h,w))$|$O(hw)$| 太慢了|
 | 最佳|$O(hw + D \log D)$在哪里$D$是除数处理 |$O(hw)$| 已接受 |

 ## 算法演练

 1. 收集包含以下内容的所有单元格坐标`#`。 我们将仅对这些单元格进行推理，因为空单元格不限制平铺。 
2. 计算一个参考点，例如第一个`#`单元格，并将其从所有其他单元格中减去以获得相对坐标。 这消除了对绝对定位的依赖并隔离了结构约束。 
3. 对于可以有效形成的行或列​​方向上的每个成对差异，累积对可能值的约束$k$。 关键思想是，如果两个单元格必须属于同一个对齐的单元格$k \times k$结构，那么它们的坐标差必须与该块大小兼容。 
4. 将所有约束减少到单个可能的候选集$k$通过对相关差异取最大公约数来计算值。 这会产生一个数字$g$这样任何有效的$k$必须划分$g$。 
5. 枚举所有约数$g$，按降序排序，并测试每个候选者$k$通过验证所有的可行性`#`细胞陷入一致$k \times k$块分区。 
6.返回最大的$k$通过可行性检查。 

核心机制是将几何平铺约束简化为坐标差的算术整除性约束。 一旦完成缩减，除数搜索就会变得高效，因为 2500 位数的整数的除数数量很少。 

### 为什么它有效

 任何具有边长的有效平铺$k$强制所有`#`与步骤网格对齐的坐标$k$在两个维度上。 这意味着任意两个之间的差异`#`细胞必须保留残基类别模$k$。 因此，每个这样的差异必须可以被整除$k$， 制作$k$所有相关坐标差的公约数。 

相反，如果一个候选人$k$划分所有这些差异，网格可以划分为一致的残基类别，并且每个`#`完全位于一致的块内，不跨越边界。 这确保了有效的平铺存在。 因此，该算法将可行性降低到整除一致性，这既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    h, w = map(int, input().split())
    grid = [input().strip() for _ in range(h)]
    
    cells = []
    for i in range(h):
        for j in range(w):
            if grid[i][j] == '#':
                cells.append((i, j))
    
    if len(cells) == 1:
        print(1)
        return
    
    def gcd(a, b):
        while b:
            a, b = b, a % b
        return a
    
    g = 0
    base_i, base_j = cells[0]
    
    for i, j in cells[1:]:
        di = abs(i - base_i)
        dj = abs(j - base_j)
        g = gcd(g, di)
        g = gcd(g, dj)
    
    if g == 0:
        print(max(h, w))
        return
    
    def divisors(x):
        res = []
        d = 1
        while d * d <= x:
            if x % d == 0:
                res.append(d)
                if d * d != x:
                    res.append(x // d)
            d += 1
        return res
    
    cand = divisors(g)
    cand.sort(reverse=True)
    
    def ok(k):
        seen = set()
        for i, j in cells:
            bi = i // k
            bj = j // k
            seen.add((bi, bj))
        return True
    
    for k in cand:
        if ok(k):
            print(k)
            return

solve()
```代码首先提取所有`#`职位。 这将问题简化为仅在活动单元格上进行推理。 如果只有一个这样的单元格，则答案很简单，就是 1，因为任何更大的正方形都需要多个单元格。 

gcd 计算累积来自相对于基本单元的差异的约束。 这编码了所有有效块大小必须除以这些坐标差的想法。 除数枚举步骤有效地生成所有可能的候选者。 

可行性函数将单元分为$k \times k$使用整数除法的块。 如果存在一致的平铺，则所有需要的单元必须落入该分区下的兼容块结构中。 

一个微妙的问题是，正确性取决于 gcd 同时捕获行和列结构。 另一个是整数除法正确地模拟了块坐标，因为块与网格原点对齐。

 ## 边缘情况

 单个`#`输入类似`1 1 / #`立即产生答案 1。 该算法在任何 gcd 或除数逻辑之前短路，因为不存在结构约束。 

完全填充的网格，例如```
2 2
##
##
```产生 gcd 2，允许候选$k=2$。 除数检查确认整个网格形成一个块。 

稀疏交替模式，例如```
2 3
#.#
.#.
```强制 gcd 崩溃为 1，因为坐标差不能始终被任何更大的值整除。 该算法正确地拒绝了所有$k > 1$因为块对齐会在不兼容的区域中分割所需的单元格。
