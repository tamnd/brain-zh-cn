 ## 解决方案
 ## 问题理解

 我们被要求构建从 1 到 n 的数字的单个排列，解释为沿行的建筑物高度。 两个整数a和b分别描述从左端和右端看时可以看到多少个建筑物。 如果建筑物在该观察方向上严格高于其前面的所有建筑物，则从侧面可见。 

从左到右，我们扫描并计算遇到新最大值的频率。 从右开始，我们从右向左扫描并计算相同类型的历史新高。 任务是确定是否存在一个排列，该排列可以从左侧精确生成 a 个可见建筑物，从右侧精确生成 b 个可见建筑物，如果存在，则输出任何此类排列。 

约束 n ≤ 1000 足够小，O(n^2) 构造甚至简单的贪婪构造策略都是可以接受的。 然而，问题的结构表明直接组合构造而不是任何搜索。 由于输出只是一种有效的排列，因此我们不需要枚举可能性。 

当 a 和 b 都很大时，就会出现关键边缘情况。 例如，如果a = n且b = n，那么双方都要求每个元素都是新的最大值，这是不可能的，因为全局最大值只能在一个位置。 类似地，如果当 n > 1 时 a = 1 且 b = 1，则双方都要求最大值紧邻边缘，而这不能同时发生。 这些失败暗示存在连接 a、b 和 n 的结构约束。 

## 方法

 蛮力的想法是生成 1 到 n 的所有排列并检查两侧的可见性。 这在概念上是正确的，因为我们可以直接计算每次排列的 O(n) 可见建筑物的数量。 然而，有n！ 排列，即使对于 n = 10，这也变得完全不可行，因为 10！ 已经是 360 万，而 n = 1000 远远超出了任何枚举方法。 

关键的观察结果是，可见性完全取决于扫描中出现的“历史新高”数量。 从左侧开始，我们正好需要一个历史高点，从右侧开始，我们正好需要 b 个历史高点。 这些记录高点的表现是独立的，除了全局最大值 n 始终从两侧可见并充当中心锚这一事实之外。 

这建议首先固定 n 的位置，将数组分成左段和右段，然后仔细构建每一侧，使其独立产生所需数量的记录高点。 一旦 n 固定，就可以安排剩余的数字，以便每一侧都准确地贡献增加最大值所需的数量。 

关键的结构约束来自于计算可见性“消耗”了多少个位置：左侧在 n 之前贡献 a−1 个额外可见元素，右侧在 n 之后贡献 b−1 个元素，这意味着我们至少需要 a+b−1 个位置，且不得超过 n。

 | ---| ---| ---| ---|
 | 蛮力 | O(n!) | O(n) | 太慢了 |
 | 建设性安置 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们直接构造排列。

1. 首先检查 a + b − 1 ≤ n 是否成立。 如果没有，就无法将所有必需的可见“记录高点”加上全局最大值放入数组中。 在这种情况下，我们立即返回“否”。 
2.固定最大元素n的位置。 我们将其放置在索引 a（1 索引）处。 这是因为我们想要从左侧恰好有一个可见的建筑物，并且将 n 作为第 a 个可见建筑物可确保它成为对该计数有贡献的最终记录高点。 
3. 用最小的数字按升序填充 n 左边的段，即位置 1 到 a−1：1, 2, ..., a−1。 这保证了从左侧扫描时，该前缀中的每个元素都是新的最大值，在到达 n 之前恰好生成 a−1 个可见建筑物。 
4. 使用剩余的最大数字按降序填充 n 右侧的线段：n−1, n−2, ..., n−(b−1)。 这确保了从右侧扫描时，每个元素依次成为新的最大值，在 n 之后恰好生成 b−1 个可见建筑物。 
5. 按照不影响可见性的任何顺序，用剩余的数字填充任何剩余的未使用位置。 实际上，一旦将两个结构化段放置在 n 周围，如果 a + b − 1 = n；则所有数字都已被使用； 否则，中间的间隙将由剩余的最小可用数字填充。 

### 为什么它有效

 该结构将排列分为三个区域：严格递增的前缀、最大元素 n 和严格递减的后缀。 该前缀保证从左侧恰好有 a−1 个记录高点，因为每个新元素都超过了所有先前的元素。 后缀保证从右侧恰好出现 b−1 记录高点，因为从右侧扫描时遇到的每个新元素都大于从该方向到目前为止看到的所有元素。 中心元素 n 是唯一的全局最大值，并且计入两个可见性计数中，充当协调两个约束而不受干扰的桥梁。 由于每个数字仅使用一次并且可见性贡献在两侧独立控制，因此只要满足长度约束 a + b − 1 ≤ n ，所得排列就有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, a, b = map(int, input().split())
    
    if a + b - 1 > n:
        print("no")
        return

    ans = [0] * n

    # place n
    pos = a - 1
    ans[pos] = n

    # left side: 1..a-1 increasing
    cur = 1
    for i in range(a - 1):
        ans[i] = cur
        cur += 1

    # right side: n-1 .. n-b+1 decreasing
    cur = n - 1
    for i in range(n - 1, n - b, -1):
        ans[i] = cur
        cur -= 1

    # fill remaining positions
    used = set(ans)
    cur = 1
    for i in range(n):
        if ans[i] == 0:
            while cur in used:
                cur += 1
            ans[i] = cur
            used.add(cur)

    print("yes")
    print(*ans)

if __name__ == "__main__":
    solve()
```代码首先强制执行可行性条件 a + b − 1 ≤ n。 然后，它通过将 n 放置在从零开始的索引中的位置 a − 1 处来锚定排列。 左前缀从 1 向上按顺序填充，保证严格递增的行为。 右后缀从 n − 1 向下填充，使得值从左到右严格减小，这相当于从右到左严格增加可见性。 任何剩余的间隙都由未使用的值填充，这不会影响可见性，因为它们严格位于受控的增加和减少区域之间，并且永远不会大到足以破坏所构建的创纪录高的结构。 

## 工作示例

 ### 示例 1：n = 5，a = 2，b = 2

 我们将 5 放在位置 2 处。 

| 步骤| 数组| 推理|
 | ---| ---| ---|
 | 地点 n | _ 5 _ _ _ | 修复位置 a | 处的全局最大值
 | 左填充| 1 5 _ _ _ | 前缀 1 元素增加 |
 | 右填充| 1 5 _ _ 4 | 开始后缀构造 |
 | 继续 | 1 5 _ 3 4 | 1 5 _ 3 4 减少后缀 |

 从左边开始，我们看到 1，然后是 5，所以 2 可见。 从右侧，我们看到 4，然后是 5，因此 2 可见。 这确认了前缀和后缀独立创建所需的记录计数。 

### 示例 2：n = 5，a = 3，b = 4

 我们将 5 放在位置 3 处。 

| 步骤| 数组| 推理|
 | ---| ---| ---|
 | 地点 n | _ _ 5 _ _ | 锚|
 | 左填充| 1 2 5 _ _ | 两项不断增加的历史新高|
 | 右填充| 1 2 5 _ 4 | 开始后缀 |
 | 继续 | 1 2 5 3 4 | 1 2 5 3 4 完整后缀 |

 从左边开始，我们看到 1、2、5，其中 3 可见。 从右边，我们看到 4、5 给出 2 加 5 给出 3 可见，但由于 b=4，这种情况实际上不可行，除非调整； 它演示了后缀长度如何强制约束 a+b−1 ≤ n 以及为什么正确的构造必须遵守它。

 ## 边缘情况

 对于 a = 1 和 b = 1，构造迫使 n 成为从两侧唯一可见的建筑物，只有当所有其他元素在两个方向上都隐藏在其后面时才起作用。 该算法将 n 放置在位置 0 处，使两侧都为空或最少填充。 由于不需要增加前缀或后缀，因此该结构完全退化。 

对于 a + b − 1 = n，排列完全确定：每个位置都用于递增前缀、中心最大值或递减后缀。 没有留下任何灵活的位置，并且直到精确的分区为止，构造变得唯一，算法自然地通过填充连续的段来强制执行。 

对于较大的 n 和较小的 a 和 b，大多数元素位于中间无约束区域。 这些值在结构化部分之后任意填充，并且它们不会影响可见性，因为它们永远不足以在受控前缀和后缀之外创建新的记录高点。
