 ## 解决方案
 ## 问题理解

 我们需要构造一个密码集合，密码数量为$n$， 在哪里$n \le 1000$。 每个密码都是由可打印 ASCII 字符组成的字符串（从代码 33 到 126），并且每个字符串的长度必须在 8 到 12 之间（含 8 和 12）。 

每个密码都必须满足固定的组成约束：它必须至少包含一个小写字母、一个大写字母、一个数字和一个非字母数字字符，即标点符号或括号等任何可打印符号。 

除了个人有效性之外，密码必须在编辑距离下相互良好分开。 对于任何两个不同的密码，必须至少执行两次单字符操作（插入、删除或替换）才能将一个密码转换为另一个密码。 换句话说，没有一对可以完全相同，也没有一对可以仅因一次编辑而不同。 

输出很简单$n$以任何顺序输入此类密码。 

这些约束足够小，我们不会处理传统意义上的性能限制。 真正的困难是组合：我们必须系统地生成许多有效的字符串，同时保证组合约束和全局编辑距离约束。 

如果我们尝试生成仅在小后缀或前缀上有所不同的密码，则会出现微妙的失败情况。 例如，如果我们修复一个基本字符串，例如`Aa0!aaaa`并且仅更改最后一个字符，我们可能会意外地通过替换在编辑距离 1 处生成字符串。 如果两个字符串仅长度不同，则会出现另一种失败情况，例如`Aa0!abcd`和`Aa0!abcde`，通过插入位于编辑距离 1 处。 这些模式必须通过构造来避免，而不是事后检查，因为天真的成对检查仍然会错过较大构造中微妙的近碰撞。 

## 方法

 暴力方法将尝试生成候选字符串并针对所有先前生成的密码验证所有约束。 自从$n \le 1000$每个字符串的长度最多为 12，我们可以检查编辑距离$O(12^2)$每对，这很便宜，但真正的问题是搜索空间爆炸。 没有结构的随机或增量生成经常会产生近碰撞，迫使重复生成。 

关键的观察是字母表很大并且所需的结构非常灵活。 我们不将其视为搜索问题，而是将其视为确定性构造问题。 我们设计了一小组“模板”，其中每个密码都共享一个严格的结构，自动保证距离至少为 2。 

强制编辑距离至少为 2 的最简单方法是确保每个密码至少有两个位置不同，或者具有无法通过一次编辑解决的结构差异。 一个可靠的技巧是修复编码索引的前缀$i$，同时保留强制执行所有必需字符类的常量后缀。 如果索引编码以改变的方式完成$i$改变至少两个字符或更改长度类对齐，我们会自动获得所需的分隔。 

我们可以将每个密码编码为：

 一个固定的 4 字符核心，包含来自每个所需类别的一个有效字符，后跟一个表示$i$以 94 为基数（或任何 ASCII 安全范围）填充到固定长度。 这保证：

 1. 每个字符串具有相同的结构和长度。 
2.固定核心保证所有字符类别的有效性。 
3. 不同索引在编码后缀中至少有一个位置不同。 
4. 由于长度相同，插入/删除无法一步将一个后缀转换为另一个后缀； 任何转换都需要至少一次替换加上至少一次附加操作，确保编辑距离≥2。 

这种结构消除了所有成对检查的需要。

 ## 算法演练

 我们使用确定性模板独立构建每个密码。 

1. 修复满足类别要求的四个字符：1 个小写字母、1 个大写字母、1 个数字和 1 个符号。 每个密码都保持相同，无需额外工作即可保证有效性。 
2. 表示索引$i$（从 0 到$n-1$) 作为可打印 ASCII 符号上高基编码的固定长度字符串。 我们通过填充确保所有编码具有相同的长度。 
3. 连接固定核心和编码索引以形成最终密码。 这保证了长度保持在范围内，因为两个组件都很小且受控。 
4. 输出每个构造的字符串。 

这样做的原因是每个密码在编码后缀中至少有一个位置与其他密码不同。 由于所有密码都具有相同的长度，因此一次编辑操作只能更改一个字符或通过一次插入/删除进行移位，但移位无法协调两种不同的固定长度编码。 因此，将一个密码转换为另一个密码始终需要至少两次编辑。 

保持的不变性是所有密码共享固定前缀和固定长度，并且后缀将唯一标识符单射编码到固定长度字母表中。 这确保没有对位于编辑距离 1 内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def to_base(x, base, length, alphabet):
    s = []
    for _ in range(length):
        s.append(alphabet[x % base])
        x //= base
    return ''.join(reversed(s))

def solve():
    n = int(input().strip())

    lower = "abcdefghijklmnopqrstuvwxyz"
    upper = "ABCDEFGHIJKLMNOPQRSTUVWXYZ"
    digits = "0123456789"
    symbols = "!@#$%^&*()_+"

    core = lower[0] + upper[0] + digits[0] + symbols[0]

    alphabet = lower + upper + digits + symbols
    base = len(alphabet)

    suffix_len = 4

    res = []
    for i in range(n):
        suffix = to_base(i, base, suffix_len, alphabet)
        pw = core + suffix
        res.append(pw)

    sys.stdout.write("\n".join(res))

if __name__ == "__main__":
    solve()
```该实现使用每个所需类别中的一个字符构建固定的有效前缀。 后缀是索引的定长基数转换，保证唯一性。 

固定长度编码的选择至关重要：如果没有它，像 9 和 10 这样的两个数字可能会因插入而不是替换而不同，从而破坏编辑距离要求。 填充可以防止结构模糊。

 ## 边缘情况

 一种边缘情况是$n = 1$。 该构造仍然有效，因为 0 的后缀编码会产生有效的固定长度块，并且核心保证所有字符类。 由于只有一个字符串，因此不会出现比较问题。 

另一个边缘情况是$n = 1000$，我们依赖于基本编码不会溢出固定后缀长度。 由于后缀长度是恒定的并且基数很大（94个可打印字符），因此我们至少可以表示$94^4$不同的值，远远超过 1000。因此该结构保持无碰撞。 

最后一个微妙的情况是确保编辑距离不会由于在核心和后缀之间的边界附近插入或删除而意外地为 1。 由于所有密码都具有相同的总长度，因此插入或删除总是会更改长度，因此无法立即在两个固定长度的有效输出之间进行映射。 这迫使任何转换仅使用替换，并且不同的编码后缀确保整个字符串空间中至少有两个独立的字符差异。
