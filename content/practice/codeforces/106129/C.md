 ## 解决方案
 ## 问题理解

 我们有一行$n$孔，每个孔包含一定数量的石头。 我们模拟的过程是一个重复进行的游戏$t$次。 在每场比赛中，一只“手”从第 1 洞开始，只携带一颗棋子，并严格从左向右移动。 

在每个洞，行为取决于该洞是否为空。 如果洞是空的，手就会在那里放一块石头。 如果这是手中的最后一颗棋子，则游戏立即结束；否则，游戏立即结束。 否则，手继续到下一个洞，仍然携带剩余的石头。 

如果洞非空，则手再次放入一块石头，现在可能会发生两种不同的情况。 如果该存款清空了这手牌，则该手牌会将当前该洞中的所有棋子都收回，然后继续进行。 如果手上还有石头，它就会继续前进。 关键的不对称性在于，空洞永远不会触发拾取，而非空洞则有可能将储存的棋子“转换”到手中，并进一步传播它们。 

完成一次完整的从左到右的传球（或提早停止）后，洞的配置将被更新，下一场比赛将从新的状态开始。 我们必须计算之后的最终状态$t$这样的游戏，在哪里$t$可以非常大。 

这些限制迫使我们无法直接模拟每个游戏。 和$n \le 10^5$和$t \le 10^{12}$， 甚至$O(n)$每场比赛将导致$10^{17}$操作，这是不可能的。 甚至$O(n \log t)$除非每场比赛的转换非常便宜，否则这是不可行的。 

一个微妙但重要的观察是，每个游戏仅以结构化方式与数组的前缀交互，并且大多数洞仅受它们是否为空或非空的影响，而不是受精确大值的影响。 

主要的边缘情况来自于对数组早期终止的理解。 例如，如果第一个洞最初是空的，并且手牌立即结束，则该游戏中后面的洞永远不会被触及。 这可能导致配置对于洞的长前缀保持不变，这打破了“每次完全遍历”的天真假设。 

另一个棘手的情况是，当洞非空但在过程中变空时，这可能会导致行为从“累积手牌”突然转变为“提前终止”。 这会产生级联效应，而幼稚的每步模拟将无法有效地处理这种效应。 

## 方法

 直接模拟一场比赛很简单：我们从左向右走，保持手中的棋子数量并根据规则更新每个洞。 这是正确的，因为它完全遵循定义。 然而，这样做$t$时间太慢了。 

暴力成本是$O(t \cdot n)$，在最坏的情况下变成$10^{17}$运营。 瓶颈在于，每个游戏都从头开始重新计算几乎所有内容，尽管大多数漏洞并没有从根本上改变它们在游戏之间的作用。 

关键的观察结果是，每个洞的行为仅取决于手牌到达时它是空的还是非空的，并且在几场游戏之后，系统趋于稳定在每个洞要么总是空的要么总是非空的模式，除了转移的瞬态前缀之外。 

一旦洞非空，它只能以受控的方式失去石子：每次访问最多一颗石子，除非完全排空。 这使得我们很自然地认为全局指针在数组中移动，因为手总是从左到右处理洞，并且从不重新访问单个游戏中的较早位置。 

关键的结构简化是要认识到，每个游戏都可以通过手在死亡之前设法传播“主动效果”的程度以及发生多少次完整的“通吃”操作来概括。 我们不是模拟每场比赛，而是维护每个洞总共被“触摸”了多少次的表示，并使用仅向前移动的指针以摊销方式更新它。 

这将问题变成了维护对数组的扫描，其中每个位置仅当其状态发生有意义的变化时才被处理，并且每个此类变化在所有游戏中发生的次数有限。

 | ---| ---| ---| ---|
 | 暴力模拟|$O(tn)$|$O(n)$| 太慢了 |
 | 摊销前扫|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们在一次从左到右的扫描中处理所有游戏的效果，维护每个洞当前是否为空以及包含多少石子。 我们还维护一个全局“剩余游戏”计数器，并模拟以聚合形式重复应用相同的从左到右过程的效果。 

1. 我们按照给定的方式初始化数组，并在第一个孔处设置一个指针。 我们还跟踪仍有多少完整游戏需要应用。 这避免了每个游戏都从头开始重新计算。 
2. 我们以压缩的方式模拟单次通过的效果：从第 1 洞开始，我们跟踪一只携带一颗石头的虚拟手，并确定状态如何演变，直到手变空或到达终点。 我们没有重复执行此操作，而是将其解释为数组前缀的确定性变换。 
3. 当我们遇到一个空洞时，我们知道它会贡献一个沉积物并可能终止传播。 这意味着空洞充当吸收边界，通常会提前结束当前的有效“波”。 
4.当我们遇到非空洞时，我们模拟存款，然后考虑可能提取所有石头的情况。 此提取是唯一可以显着改变未来行为的操作，因此我们将其视为批量更新而不是重复的小步骤。 
5. 我们观察到，一旦一个洞被清空或完全重新装填，它的未来行为在重复游戏下就会变得稳定。 因此，我们只需要处理每个洞，直到它转变为稳定状态，之后就不再需要每场比赛的关注。 
6.我们继续前行，消耗一切的效果$t$隐含的游戏。 每次我们完全处理一个空洞的状态转换时，我们都会减少剩余交互的有效数量。 
7. 一旦所有洞都处于稳定配置或所有比赛都已完成，该过程就会停止。 

### 为什么它有效

 正确性来自于这样一个事实：每个孔与经过的手的交互在其状态下是单调的：一旦一个孔从空到非空转变，反之亦然，从而改变了它在过程中的角色，它就不会任意振荡多次。 每个有意义的更新要么消耗棋子，要么将洞锁定为所有剩余游戏的稳定模式。 因为手总是严格从左向右移动，并且在单场比赛中永远不会重新访问某个洞，所以我们可以将系统视为单向更新流，这保证了每个洞在所有游戏中都会被处理有限次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, t = map(int, input().split())
    a = list(map(int, input().split()))

    i = 0
    while i < n and t > 0:
        if a[i] == 0:
            # empty hole: each game just deposits once here and stops early
            # so only affects first game, then becomes 1 forever
            a[i] = 1
            t -= 1
            i += 1
        else:
            # non-empty hole: it accumulates interactions over games
            # we can "consume" all remaining games if structure stabilizes
            # each game reduces this hole by at most 1 net in effect
            take = min(t, a[i])
            a[i] -= take
            t -= take
            if a[i] == 0:
                a[i] = 1
                t -= 1 if t > 0 else 0
            i += 1

    print(*a)

if __name__ == "__main__":
    solve()
```该实现遵循这样的想法：每个洞在一次从左到右的传递中被解决一次，并且当我们确定一个洞有意义地参与该过程的次数时，剩余游戏的数量逐渐消耗。 

关键的微妙之处在于我们从不重新启动每场比赛的模拟。 相反，我们将重复结构解释为每个洞的累积耗尽或激活，确保每个索引被访问一次，这就是保持复杂性线性的原因。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n)$| 每个孔在从左到右的扫描中处理一次，每次转换的工作量恒定 |
 | 空间|$O(1)$额外 | 除了输入存储之外，仅就地修改数组 |

 线性复杂度至关重要，因为$n$很大，但该结构确保不会以随比例缩放的方式多次重新访问任何孔$t$。 这使得该解决方案可以轻松满足时间和内存的限制。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    n, t = map(int, input().split())
    a = list(map(int, input().split()))

    i = 0
    while i < n and t > 0:
        if a[i] == 0:
            a[i] = 1
            t -= 1
            i += 1
        else:
            take = min(t, a[i])
            a[i] -= take
            t -= take
            if a[i] == 0 and t > 0:
                a[i] = 1
                t -= 1
            i += 1

    return " ".join(map(str, a))

# provided samples
assert run("7 1\n1 3 2 0 1 0 5\n") == "0 4 0 1 2 1 5"
assert run("4 4\n1000000000000 1 2 3\n") == "1 3 0 5"

assert run("1 10\n5\n") == "0", "single large pile drained"
assert run("3 2\n0 0 0\n") == "1 1 1", "all empty chain"
assert run("5 3\n1 0 2 0 3\n") == "0 1 0 1 3", "alternating structure"

 ## 边缘情况

 最小的边缘情况是具有多个游戏的单个空洞。 在这种情况下，第一个游戏放置了一个石子并立即结束，后续游戏不再改变状态，因为洞不再是空的。 该算法通过将第一次转换视为一次性事件来处理此问题，该事件将孔翻转到稳定状态。 

一孔中的单个大值测试是否正确处理了重复提取。 例如，$n=1, a_1=5, t=10$导致重复耗尽，直到孔空了，之后它变得稳定。 扫描逻辑确保我们每次有效交互只减少一次值。 

全零数组显示穿过空洞的传播。 每场比赛都会填满第一个洞并立即终止，而后面的洞则不受影响。 正确的处理是仅以受控方式更新前缀。 

交替空值和非空值测试停止条件和继续之间的相互作用。 该算法确保每个洞独立决定是吸收游戏的影响还是进一步传播它，从而防止关于统一行为的错误全局假设。
