 ## 解决方案
 ## 问题理解

 我们得到了一系列法术值，每个值都是非负整数，以及具有非常特定使用模式的盾牌。 当护盾被激活时，它会精确地阻挡一个连续的方块`d`咒语，每次激活后它必须休息以进行下一次咒语`d`在此期间不能使用它的法术。 第一次激活是在第一个咒语之前允许的，因此每个决定都是关于选择一些起始位置，所选择的起始位置至少是`2d`分开。 

所有未被阻挡的法术都会“击中”我们，我们会收集它们的值。 最终伤害不是总和或最大值，而是所有命中值的混合值，即未出现在其中的最小非负整数。 我们的目标是安排护盾激活，使这个 mex 尽可能小。 

输出是一个数字，即未受阻法术值的多组可实现的最小 mex。 

限制条件表明`n`最多`10^5`，因此任何比线性或近线性更差的解决方案都会失败。 特别是，尝试所有激活子集或重复模拟 mex 重新计算的任何操作都会立即变得太慢。 我们应该期待`O(n log n)`或者`O(n)`解决方案。 

一个关键的微妙之处是 mex 仅取决于存在，而不取决于频率。 一个值要么出现在命中法术中，要么不出现，因此护盾有效地决定哪些值可以从命中集中完全消除。 

一个常见的失败案例是假设我们可以独立“删除所有出现的小值”。 例如，如果一个值出现在太多间隔比`d`，我们可能被迫让某些事件通过，使得该值在命中集中不可避免。 

另一个微妙的边缘情况是当`d = 1`。 然后每个咒语都可以交替被阻止，因此我们可以删除所有偶数或奇数位置，这使得 mex 有可能`0`取决于安排。 相反，当`d = n`，我们可以阻止一切，也可以什么都不阻止，使墨西哥纯粹由完全存在或不存在决定。 

## 方法

 蛮力的想法是将每个位置视为被阻止或未被阻止，尊重任何两个激活必须至少是的约束`d`分开。 对于每个有效的激活模式，我们计算保持畅通的一组值并评估其 mex。 这是正确的，因为它直接遵循定义，但有效模式的数量呈指数增长`n/d`。 即使进行剪枝，最坏的情况仍然接近指数复杂度，使其不可行。 

关键的观察是扭转视角。 我们不是选择要阻挡哪些法术，而是要求目标墨西哥人`m`。 为了实现墨西哥`m`，我们需要每个值`0, 1, ..., m-1`在命中法术中至少出现一次，并且价值`m`不在命中法术中。 所以对于每个候选人`m`，问题就变成检查我们是否可以阻止所有出现的值`m`同时仍然保留每个较小值的至少一次出现未被阻止。 

现在结构变成了几何结构。 每个值都会引发一组禁止块决策：消​​除值`x`，每次出现`x`必须位于某个阻塞区间内。 由于阻塞间隔的长度是固定的并且不能重叠得太近，因此我们选择的间隔起点“覆盖”某些值的所有出现，同时保留其他值的至少一个出现。 

关键的简化是认识到对于任何固定的`m`，我们只关心是否可以覆盖包含以下内容的所有位置`m`使用可用的块间隔，不会意外覆盖任何出现的情况`0..m-1`。 这变成了每个值位置上的贪婪调度问题。 

我们可以按升序处理值并维护每个值出现的最早位置。 对于固定的`m`，我们尝试贪婪地放置块以消除所有出现的`m`，同时确保对于所有较小的值，至少保留一个未覆盖的事件。 可行性检查可以在发生次数的线性时间内完成。 

而不是从头开始重新计算每个`m`，我们对出现的情况进行排序并模拟一次块放置，跟踪哪些值无法完全消除，因为`m`成长。 答案是最小的`m`这在热门集中是无法避免的。

 ## 算法演练

 我们将问题转化为在阻塞约束下是否可以将某个值从命中集中完全删除的推理。 

### 第 1 步：按价值对头寸进行分组

 我们为数组中出现的每个值构建一个索引列表。 这使我们能够独立地推理每个值的覆盖范围，同时仍然尊重全局块间距。 

### 第 2 步：了解删除值的含义

 为了保证一个值`x`没有出现在最终命中集中，每次出现`x`必须位于某个阻塞段内。 由于每个块都有固定的长度`d`，每个块从位置开始`i`封面`[i, i+d-1]`。 所以删除`x`意味着选择块开始共同覆盖所有索引，其中`x`出现。 

限制是块开始必须至少相差`d`，因此块沿着时间线形成稀疏集。 

### 步骤 3：固定值的贪婪可行性

 对于单个值`x`，我们尝试使用最少数量的块来覆盖所有出现的情况。 我们按升序扫描事件，每当我们在位置发现未覆盖的事件时`p`，我们放置一个块，从`p`，覆盖尽可能远的位置。 

这种贪婪的选择是最优的，因为尽可能晚地放置块，同时仍然覆盖第一个未覆盖的事件，可以最大限度地覆盖未来的事件，并且不会损害可行性。 

### 步骤4：全局约束交互

 我们放置的每个区块都是有价值的`x`也会影响所有其他值，也可能会删除它们。 这就是 mex 进入的地方：如果在消除的过程中`x`，我们还消除了所有出现的较小值`y`， 然后`y`不能再成为最终热门集的一部分，这将使实现墨西哥`> y`不可能的。 

因此，在模拟过程中，我们跟踪每个值是否在所有选定块之外至少出现过一次。 

### 步骤 5：确定 mex

 我们按升序处理值。 对于每个候选墨西哥人`m`，我们测试是否值`0..m-1`都可以保留至少一个未覆盖的事件，而值`m`可以完全覆盖。 第一个故障点给出了答案。 

### 为什么它有效

 该算法依赖于单调覆盖属性：一旦放置一个块来覆盖某个事件，它就会删除固定的位置间隔，并且从左到右贪婪地放置块可确保对未来必要的未覆盖事件的干扰最小。 因为 mex 仅取决于存在性，而不取决于多重性，所以为每个值保留单个未覆盖的出现就足够了，并且贪婪策略保证我们在尝试删除候选 mex 值时不会意外地消除不必要的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d = map(int, input().split())
    p = list(map(int, input().split()))

    pos = [[] for _ in range(n + 1)]
    for i, v in enumerate(p):
        pos[v].append(i)

    # try mex from 0 upward
    for mex in range(n + 1):
        # check if we can fully eliminate mex
        if not pos[mex]:
            print(mex)
            return

        # simulate blocking all occurrences of mex
        last_block_end = -1
        ok = True

        for i in pos[mex]:
            if i > last_block_end:
                # place block starting at i
                last_block_end = i + d - 1

        # now check if all smaller values still have at least one uncovered occurrence
        blocked = [0] * n
        last_block_end = -1

        for i in pos[mex]:
            if i > last_block_end:
                start = i
                end = i + d - 1
                last_block_end = end
                for j in range(start, min(end + 1, n)):
                    blocked[j] = 1

        for v in range(mex):
            if not pos[v]:
                ok = False
                break
            if all(blocked[i] == 1 for i in pos[v]):
                ok = False
                break

        if ok:
            print(mex)
            return

    print(n)

if __name__ == "__main__":
    solve()
```该实现遵循迭代候选 mex 值和测试可行性的思想。 关键细节是在尝试消除某个值时贪婪地放置块：每次遇到未覆盖的事件时，我们都会在那里启动一个块，这会最大化前向覆盖并确保我们使用尽可能少的块。 

第二阶段检查小于候选 mex 的值是否在所有阻塞间隔之外仍然至少出现一次。 这是至关重要的，因为即使我们可以消除候选值，我们也不允许销毁所有出现的较小值。 

一个微妙的点是保持`last_block_end`正确。 它通过确保没有块在前一个块的禁止区域内开始来强制再生约束。

 ## 边缘情况

 一个重要的边缘情况是某个值根本不出现。 在这种情况下，它的 mex 候选者可以立即实现，因为我们不需要做任何事情来消除它，它已经不在命中集中了。 该算法通过检查来处理这个问题`if not pos[mex]`并立即返回`mex`。 

另一种边缘情况发生在`d = n`。 这里只能应用一个块。 贪婪的构造会退化为要么阻塞整个数组，要么什么都不阻塞。 该算法可以正确处理此问题，因为任何块都会覆盖完整的连续段，因此要么覆盖所有出现的值，要么不覆盖任何值，从而使 mex 完全依赖于全局存在。 

最后一个微妙的情况是密集重复，例如用小值交替值`d`。 在这种情况下，贪婪的块放置很快会重叠许多值，并且检查`all(blocked[i] == 1 for i in pos[v])`确保当某个值的所有出现都在单个强制块内时，我们不会错误地假设该值仍然存在。
