 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中顶点代表火车站，边代表单向火车连接。 两个人从两个不同的车站出发，每个人都可以沿着有向边行驶任意多次。 问题是是否存在至少一个从两个起点均可到达的车站。 

因此该任务简化为有向图中的可达性问题：从节点`a`和节点`b`，我们想要找出它们的可达集是否相交，如果相交，则输出相交处的任意节点。 

约束最多为 100,000 个节点和 100,000 条边，这会立即排除任何二次方法，例如检查每对节点之间的可达性或重新计算每个候选节点的最短路径。 任何解决方案在图的大小上都必须基本上是线性的，通常`O(n + m)`。 

如果试图从结构中贪婪地推理，就会出现一个微妙的问题。 例如，根据本地连接或 BFS 树中最早相遇来选择“会议站”并不安全，因为可达性不对称，并且路径在再次收敛之前可能会严重分歧。 

另一种边缘情况是当其中一个源没有传出路径时。 例如：

 输入：```
3 1
1 2
3 1
```从`3`你可以到达`1`进而`2`，但是从`1`你只能到达`2`。 唯一可能的集合地点是`2`。 一种简单的方法，只检查直接邻居`a`或者`b`会想念这个的。 

最后，如果图是断开连接的或包含长有向链，则很容易基于无向直觉错误地假设重叠。 方向在这里非常重要。 

## 方法

 一个蛮力的想法是计算可到达节点的完整集合`a`使用 BFS 或 DFS，并分别计算可达集`b`，然后将这两个集合相交。 这是正确的并且概念上简单。 

每个 BFS/DFS 运行在`O(n + m)`，并且设置交集为`O(n)`如果用布尔数组实现。 因此，即使是简单的方法也已经是每个源的线性，因此`O(n + m)`全部的。 这对于约束来说已经足够有效了。 

然而，对于每个节点，可能会尝试更概念化的暴力破解`v`，检查是否`v`两者均可到达`a`和`b`通过在每个节点运行 BFS 两次或在每个节点运行反向可达性检查。 这会导致`O(n(n + m))`，太大了。 

关键的观察结果是，来自单个源的可达性将图划分为可达区域。 我们只需要知道两个可达区域是否重叠，如果重叠，找到任何公共节点就可以立即停止。 这使我们能够通过交错或提前停止来避免不必要的工作，但即使没有优化技巧，两次 BFS 运行也足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个节点的暴力破解 | O(n(n + m)) | O(n(n + m)) | O(n + m) | 太慢了 |
 | 两个BFS/DFS+交集| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们使用图遍历独立地计算从每个起始站可到达的所有节点。 

1. 从有向边构建邻接表。 这种结构允许有效地遍历来自任何站的传出连接。 
2.从站开始运行BFS或DFS`a`，在布尔数组中标记每个访问过的节点`reach_a`。 这准确地捕获了 A 乘坐任意次数的火车可以到达的车站集合。 
3.从站开始运行另一个BFS或DFS`b`，标记访问过的节点`reach_b`。 这同样捕获了 Hannah 可到达的所有车站。 
4. 扫描所有电台`1`到`n`。 对于每个车站`v`，检查两者是否`reach_a[v]`和`reach_b[v]`是真的。 第一个这样的节点是有效的交汇点并且可以立即输出。 
5、扫描所有站点后，如果不存在该节点，则输出`"no"`。 

我们可以安全地返回任何交叉点节点的原因是该问题不需要诸如最短旅行时间或最早车站之类的最优性； 任何可到达的重叠都是有效的。 

### 为什么它有效

 该算法显式计算两个起始节点的前向可达性集。 根据定义，当且仅当两个参与者都可以沿着有向路径独立到达该节点时，该节点才位于这些集合的交集中。 BFS/DFS 探索保证了这些集合的完整性，因为每条边都从每个源恰好遍历一次，因此不会遗漏任何可达节点。 因此，在两个访问集中找到的任何节点都是正确的交汇点，如果不存在这样的节点，则交集为空并且不存在解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def bfs(start, adj, n):
    vis = [False] * (n + 1)
    stack = [start]
    vis[start] = True

    while stack:
        v = stack.pop()
        for nxt in adj[v]:
            if not vis[nxt]:
                vis[nxt] = True
                stack.append(nxt)
    return vis

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n + 1)]

    for _ in range(m):
        x, y = map(int, input().split())
        adj[x].append(y)

    a, b = map(int, input().split())

    reach_a = bfs(a, adj, n)
    reach_b = bfs(b, adj, n)

    for i in range(1, n + 1):
        if reach_a[i] and reach_b[i]:
            print("yes")
            print(i)
            return

    print("no")

if __name__ == "__main__":
    solve()
```该解决方案构建一个有向邻接列表，然后使用堆栈执行两个独立的深度优先搜索。 访问过的数组`reach_a`和`reach_b`对每个起始节点的可达性进行编码。 

一个重要的小实现细节是在压入堆栈时立即将节点标记为已访问。 这可以防止重复插入并保持遍历线性。 最终扫描是必要的，因为我们可以输出任何有效的会面站，而不必是遍历过程中发现的第一个会面站。 

## 工作示例

 ### 示例 1

 输入：```
3 2
1 3
2 3
1 2
```我们计算可达性`1`和`2`。 

| 步骤| 节点 A | 访问过 A | 来自 B 的节点 | 访问 B | 找到交叉路口 |
 | ---| ---| ---| ---| ---| ---|
 | 开始| 1 | {1} | 2 | {2} | 无 |
 | 展开A | 3 | {1,3} | 2 | {2} | 无 |
 | 展开 B | 已看过 3 个 | {1,3} | 3 | {2,3} | 是的 |

 均到达节点`3`，所以输出为：```
yes
3
```这表明即使源不同且路径不对称，收敛也可能发生。 

### 示例 2

 输入：```
3 2
2 1
2 3

```可达性：

 | 步骤| 节点 A | 访问过 A | 来自 B 的节点 | 访问 B | 找到交叉路口 |
 | ---| ---| ---| ---| ---| ---|
 | 开始| 1 | {1,3} | 3 | {3} | 无 |
 | 展开 B | - | {1,3} | 完成 | {3} | 无 |

 在这里，从`1`我们可以到达`3`，并且从`3`我们只能到达自身，所以交集是`{3}`输出是：```

```这表明该算法正确处理了不对称可达性，其中起点已经接近交汇点。

 ## 边缘情况

 一种关键的边缘情况是一个起始节点已经可以从另一个起始节点到达。 例如，如果`b`可以达到`a`， 然后`a`可能已经是有效的集合点。 

输入：```
2 1
1 2
1 2
```从`1`，可达集是`{1,2}`。 从`2`，可达集是`{2}`。 交点是`{2}`，所以输出是`yes 2`。 BFS 来自`2`只正确标记自身，最终扫描找到节点`2`一样常见。 

另一种情况是断开连接的图：```
4 2
1 2
3 4
1 3
```从`1`我们到达`{1,2}`， 从`3`我们到达`{3,4}`，交集为空。 算法正确打印`no`扫描完所有节点后，因为两个访问过的数组中都没有标记节点。 

最后一个微妙的情况是存在多个有效答案。 由于我们按升序扫描节点，因此我们返回最小的标记会议站，但根据问题，任何顺序都是正确的。
