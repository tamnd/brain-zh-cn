 ## 解决方案
 ## 问题理解

 我们有一个矩形网格，其中每个单元格都包含一个强度值。 一个特殊的单元格是玩家的起始位置，另一个是包含 Unobtanium 矿的目标单元格。 其他每个单元格都包含一个敌人，其力量等于该单元格中存储的值。 

玩家从起始单元的强度开始。 允许在四个方向上移动。 当玩家进入一个新的牢房时，严格要求自己的实力必须超过该牢房内的敌人，否则就会立即死亡。 如果他们赢了，他们就会吸收敌人的力量，而自己的力量也会增加一定量。 

还有一个额外的约束使得这个问题变得不那么简单。 每次成功战斗后，都会检查所有相邻的单元格。 如果任何邻居包含的敌人的强度严格大于玩家更新的强度，则该邻居立即攻击并杀死玩家。 因此，在战斗中生存下来不仅取决于当前的牢房，还取决于周围的边界。 

任务是确定是否存在从起点到目标的任何路径，以便在这些不断变化的约束下每一步都是安全的。 

每个测试的网格总共可以有 10^6 个单元格，因此任何重复访问状态或显式模拟所有路径的解决方案都会失败。 简单的最短路径或状态上的 DFS 是不可行的，因为每个状态都取决于电流强度，电流强度沿着路径变化，从而使状态空间有效地呈指数分布。 

“报复检查”产生了一个微妙的边缘情况。 即使你能打败一个细胞，你也可能会因为战斗后遇到更强大的邻居而立即死亡。 例如，如果您更新的强度变为 10，而邻居的强度为 12，则即使该邻居不在您的路径上，该移动也是无效的。 仅检查目标单元的幼稚 BFS 会错误地接受此类转换。 

另一种棘手的情况发生在当一条路径在当地看起来安全但需要在重新访问受限区域之前在不同区域尽早积累力量时。 这意味着贪婪的最短路径思维失败了。 

## 方法

 一个蛮力的想法是将其视为图形搜索，其中每个状态由位置和电流强度定义。 从一个状态开始，我们尝试所有四个动作，更新强度，并验证目标单元格和复仇约束。 这是正确的，但会立即爆炸，因为强度可以增长到所有单元的总和，并且可以用许多不同的强度访问同一个单元。 在最坏的情况下，这会创建指数级的多个状态。 

关键的观察结果是，复仇约束仅取决于_当前强度_，而不取决于路径历史。 一旦你进入一个牢房，你唯一的问题是你当前的力量是否足以在牢房及其邻居中生存。 这表明我们应该始终尝试按照所需强度的递增顺序到达细胞，类似于 Dijkstra 算法。 

我们可以将问题重新解释为一个可达性过程，当我们有足够的力量安全占据每个单元格时，每个单元格就变得可用。 每次我们进入一个细胞，我们的力量就会增加，这可能会解锁更多的细胞。 这正是一个单调扩展过程，按当前强度排序的优先级队列自然捕获了它。 

我们总是从当前最强的可达状态开始扩展，因为这样可以最大化解锁新单元的机会。 任何较弱的国家都无法提前进入新领域。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | Brute Force（用强度说明 BFS/DFS）| 指数| O(r·c·S) | 太慢了 |
 | 使用优先级队列的最佳优先搜索 | O(rc log(rc)) | O(rc log(rc)) | O(rc) | 已接受 |

 ## 算法演练

1. 将每个网格单元解释为一个节点，其权重等于其强度要求，但动态奖励等于其收集时的值。 起始节点以等于其像元值的初始强度开始。 
2. 维护可到达小区的优先级队列，按到达时的当前强度排序。 我们总是首先扩张实力最强的国家。 这确保我们永远不会忽视可以更早解锁更困难区域的路径。 
3. 跟踪访问过的单元格，以确保我们不会多次处理同一个单元格，因为一旦单元格被获取，它对强度的贡献就会永久增加。 
4. 从当前单元格开始，尝试向四个方向移动。 对于每个邻居，检查当前强度是否严格大于邻居的值。 如果没有，请跳过它。 
5. 如果允许移动，通过增加力量来模拟进入牢房。 更新强度后，验证报复条件：新小区的所有四个邻居的强度必须小于或等于更新的强度。 如果任何邻居违反了这一点，则放弃该举动。 
6. 如果移动有效并且邻居尚未被访问，则将其以更新的强度推入优先级队列。 
7. 当目标单元从队列中弹出时立即停止。 如果队列清空而未到达，则目标无法到达。 

关键思想是，每次我们处理一个细胞时，我们都会以目前已知的所有方法中最大可能的强度来处理它，因此未来的任何改进都必须来自以前未探索过的区域。 

### 为什么它有效

 在任何时候，该算法都会维护具有相关强度值的可到达单元格的边界，并且始终首先扩展最大强度边界。 由于强度仅沿着路径增加而不会减少，因此一旦以给定强度处理单元，则稍后到达同一单元的任何替代路径在不经过已处理的节点或先前扩展的更高强度状态的情况下都无法产生更强的状态。 这表明，当我们第一次确定一个单元格时，我们已经考虑了以最大强度到达它的所有方法，因此不会错过到达目的地的更好路径。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        r, c = map(int, input().split())
        grid = [list(map(int, input().split())) for _ in range(r)]
        isr, jsc, itr, jtc = map(int, input().split())
        isr -= 1
        jsc -= 1
        itr -= 1
        jtc -= 1

        start_val = grid[isr][jsc]
        target = (itr, jtc)

        visited = [[False] * c for _ in range(r)]
        pq = []
        heapq.heappush(pq, (-start_val, isr, jsc))

        dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

        while pq:
            neg_str, x, y = heapq.heappop(pq)
            strength = -neg_str

            if visited[x][y]:
                continue
            visited[x][y] = True

            if (x, y) == target:
                print("UNOBTANIUM")
                break

            for dx, dy in dirs:
                nx, ny = x + dx, y + dy
                if nx < 0 or nx >= r or ny < 0 or ny >= c:
                    continue
                if visited[nx][ny]:
                    continue

                if strength <= grid[nx][ny]:
                    continue

                new_strength = strength + grid[nx][ny]

                ok = True
                for ddx, ddy in dirs:
                    ax, ay = nx + ddx, ny + ddy
                    if 0 <= ax < r and 0 <= ay < c:
                        if grid[ax][ay] > new_strength:
                            ok = False
                            break

                if ok:
                    heapq.heappush(pq, (-new_strength, nx, ny))
            else:
                continue
        else:
            print("UNOBTAINABLE")

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现通过否定强度来使用最大堆，以便始终首先扩展最强的可达状态。 被访问的数组确保每个单元以其最知名的强度被处理一次。 

关键细节是，在模拟移动之后、推动状态之前立即检查复仇条件。 这避免了将无效状态传播到搜索空间中。 

## 工作示例

 ### 示例 1

 考虑一个小网格，其起点已经足够强大，可以沿着单调递增路径移动。 

| 步骤| 细胞| 实力前| 移动有效| 强度后| 复仇检查|
 | --- | --- | --- | --- | --- | --- |
 | 1 | (1,1) 开始 | 3 | - | 3 | 好的 |
 | 2 | (1,2) | 3 | 是 (3 > 1) | 4 | 好的 |
 | 3 | (2,2) | 4 | 是（4 > 4？否，所以无效）| - | - |

 这表明，即使一条路径在几何上看起来是可达的，平等也会阻止移动，从而防止错误的扩展。 

由于不存在有效的延续，该算法正确地提前停止。 

### 示例 2

 需要绕道的情况：

 | 步骤| 细胞| 实力前| 移动有效| 强度后| 复仇检查|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 开始 | 5 | - | 5 | 好的 |
 | 2 | 高价值安全节点| 5 | 是的 | 12 | 12 好的 |
 | 3 | 目标| 12 | 12 是的 | 15 | 15 好的 |

 在这里，优先级队列排序确保首先探索更强的中间状态，从而解锁目标，而天真的 BFS 可能首先尝试较弱的路由并被阻止。 

这证实了扩展顺序对于正确性至关重要。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(rc log(rc)) | O(rc log(rc)) | 每个单元最多在堆中插入和删除一次，每次操作花费对数时间 |
 | 空间| O(rc) | 网格、访问数组和堆每个单元最多存储一个条目 |

 这些约束保证测试用例中的单元总数最多为 10^5，因此即使在 Python 中，这种复杂性也完全符合限制。
