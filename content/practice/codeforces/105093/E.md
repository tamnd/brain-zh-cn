 ## 解决方案
 ## 问题理解

 我们给出了几个独立的场景。 每个场景中都有$n$国家，每个国家都有一个代表其技术水平的整数值。 

当且仅当两国技术水平的绝对差异不超过选定的阈值时，两个国家才可以通过通信电缆直接连接$a$。 如果存在这样的电缆，则总是会建造它。 如果存在一系列电缆在两国之间形成路径，则两国之间的通信是可能的。 

目标是选择尽可能小的值$a$这样，在建造所有允许的电缆后，每个国家都可以通过某种连接顺序到达其他国家。 

输入大小建议最多$10^5$所有测试用例的总值，这排除了任何直接比较所有对的二次方法。 任何检查所有国家对的解决方案都需要$10^{10}$最坏情况下的比较，远远超出了可行的限度。 因此，我们需要一种结构来避免显式地对所有边进行推理。 

当值未排序时，幼稚思维的微妙失败案例就会出现。 例如，如果值为：

 输入：```
1
4
1 100 2 3
```如果人们在没有排序的情况下错误地推理任意对连接，很容易忽略阻止连接的“间隙”实际上是在 3 到 100 之间，而不是在任意远距离元素之间。 正确答案仅取决于排序中最严格的瓶颈，而不取决于任意对距离。 

另一种边缘情况是所有值都相等，例如：```
1
5
7 7 7 7 7
```这里任何$a = 0$已经连接了一切，因为所有差异为零。 

这些观察结果表明，最佳连接的结构取决于值按排序顺序排列时的排列方式。 

## 方法

 蛮力的想法是将每对国家视为潜在的边缘，并模拟哪些边缘存在于固定值$a$。 对于给定的$a$，我们可以构建一个图并运行 BFS 或 DSU 来检查连接性。 然后我们可以二分查找$a$，每次都检查连接性。 

这是正确的，因为连接是单调的$a$，但每次检查都需要检查所有$O(n^2)$最坏情况下成对。 即使经过优化，构建边缘仍然需要花费二次方的时间，这在以下情况下会失败$n = 10^5$。 

关键的观察结果是，对技术值进行排序后，对连接性重要的唯一边是按排序顺序的相邻元素之间的边。 如果两个连续排序值之间存在较大差距，则两者之间不存在更小的值来“弥合”该差距。 任何试图绕过它的路径仍然必须穿过相同的数字跳跃，这是不可能的，除非$a$至少是这个差距。 

因此，整个问题简化为寻找排序数组中连续元素之间的最大差异。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（每图$a$) |$O(n^2)$每张支票 |$O(n^2)$| 太慢了 |
 | 最优（排序+扫描）|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 按非降序对技术值数组进行排序。 排序揭示了值之间最小“跳跃”的自然结构，这是连接可能失败的地方。 
2. 计算已排序数组中每对连续元素之间的差异。 每个差异代表直接弥合局部差距所需的最小阈值。 
3. 跟踪这些连续差异的最大值。 这种最大差距是价值链中最薄弱的环节。 
4. 输出这个最大间隙作为测试用例的答案，因为任何较小的值都会导致至少一个不可避免的连接中断。 

### 为什么它有效

 一旦对值进行排序，从最小元素到最大元素的任何路径都必须按升序穿过中间值。 即使非相邻节点之间存在边，任何此类连接仍然无法绕过连续排序值之间的最大间隔，因为不存在中间值来减少跳跃。 当每个相邻间隙都是“可桥接的”时，该图就会精确连接，并且最差的间隙决定了所需的最小阈值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []
    for _ in range(T):
        n = int(input())
        arr = list(map(int, input().split()))
        arr.sort()
        
        ans = 0
        for i in range(1, n):
            ans = max(ans, arr[i] - arr[i - 1])
        
        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案独立处理每个测试用例。 排序是关键的步骤，因为它将密集连接问题转化为对相邻差异的线性扫描。 答案是通过跟踪最大间隙来累积的，该最大间隙直接对应于完全连接所需的最小阈值。 

一个常见的错误是试图推理所有成对差异。 这超出了不相关的限制。 只有排序顺序中的连续差异才重要，因为它们代表值行中的最小分隔符。

 ## 边缘情况

 一种边缘情况是当$n = 2$。 该算法简化为排序后直接相减，最大连续差值就是两个值的绝对差值。 对于输入：```
1
2
8 3
```排序给出$[3, 8]$，唯一的间隙是 5，正确返回。 

另一种边缘情况是重复值与单个异常值混合。 为了：```
1
5
4 4 4 4 100
```分选率$[4, 4, 4, 4, 100]$。 该算法扫描连续差异并找到 96 的单个大间隙。这正确捕获了离群值与集群断开连接，除非阈值至少为 96。
