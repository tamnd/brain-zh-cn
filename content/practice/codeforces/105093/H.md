 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个晶体中都有一系列晶体，每个晶体都有颜色和辐射值。 我们最多只能选择$m$水晶，但在使用它们之前，我们可以合并任意数量的相同颜色的水晶，方法是反复将其中两个水晶替换为辐射是两个原始水晶之和的一个。 合并不会改变颜色，只会聚合辐射。 

全部合并后，剩余的每种颜色最多贡献一颗水晶，因为如果剩下两颗相同颜色的水晶，那么对于最终的装置来说是无效的。 最终分数是所有选定的不同颜色晶体的辐射值的乘积。 任务是接$m$原始晶体，将它们最佳地融合在颜色中，并最大化该产品。 

关键的结构约束是$n \le 15$，所以即使是指数子集也是可行的。 然而，辐射值很小，并且乘法行为表明，在不枚举配置的情况下，贪婪的选择并不明显安全。 这个小$n$立即排除任何尝试在大型状态空间上进行优化或对总和进行连续 DP 的解决方案，但强烈要求使用子集枚举或位掩码动态编程。 

一个微妙的陷阱是误解“背包大小”$m$“约束。你不挑选颜色；你挑选原始晶体。但是合并意味着选择一种颜色的多个晶体可以有效地将它们转化为一个更强的物品。例如，选择三个颜色 2 的晶体会产生具有总辐射的单个晶体，并且在最终产品中只贡献一个因素。

 另一个微妙的情况是，获取更多颜色的晶体并不总是有益的，除非它在合并后增加了产品。 例如，辐射为 1 和 1 的两个晶体变成 2，这可能会也可能不会改善产品，具体取决于其他选择的值。 

## 方法

 蛮力方法是尝试最多大小的晶体的每个子集$m$。 对于每个子集，我们通过按颜色分组并对每组的辐射求和来模拟合并。 然后我们计算得到的每种颜色总和的乘积。 这是正确的，因为它直接对过程进行建模。 子集的数量是$2^n$，并与$n \le 15$，这最多有 32768 个子集，已经足够小了。 然而，我们还需要强制子集大小约束并计算每个子集的分组，从而提供额外的$O(n)$因素，孤立来看仍然很好。 

微妙的问题是，如果我们不小心的话，我们实际上会重复计算结构：合并顺序并不重要，只有每种颜色的总数才重要。 因此，每个子集都唯一地映射到具有聚合权重的多组颜色。 

关键的观察是，自从$n$很小，我们不需要巧妙地使用 DP 压缩或贪婪策略。 我们直接枚举子集，计算每种颜色的聚合总和，并评估产品。 “大小背包”$m$” 约束的处理只需跳过大于的子集$m$。 

这将问题简化为对每个子集进行线性评估的子集枚举。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集的暴力破解 |$O(2^n \cdot n)$|$O(n)$| 已接受 |
 | 位掩码评估（最佳）|$O(2^n \cdot n)$|$O(n)$| 已接受 |

 实际上，两者是相同的； “优化”是认识到不需要更高级的结构。 

## 算法演练

 1. 对于每个测试用例，读取所有晶体并将它们存储为颜色和辐射对。 
2. 迭代所有位掩码$0$到$2^n - 1$，其中每个位掩码代表晶体的一个子集。 这编码了我们在合并之前选择哪些晶体的决定。 
3. 对于每个子集，计算选择了多少个晶体。 如果这超过$m$，立即丢弃该子集，因为它违反了背包约束。 
4. 构建按颜色索引的数组或字典，并累积该颜色的所有选定晶体的辐射值。 这会模拟所有允许的合并，因为颜色内的合并相当于求和。 
5. 计算所有非零颜色和的乘积。 
6. 跟踪所有有效子集的最大乘积。 

正确性取决于这样一个事实：一旦子集固定，合并就是确定性的：给定颜色的所有晶体都会崩溃为等于其总和的单个值。 没有其他配置可以改变结果。 

### 为什么它有效

 任何有效的最终配置都对应于选择原始晶体的某些子集，然后任意合并到每种颜色中。 合并不会改变每种颜色的总辐射，只会改变每种颜色的项目数量。 因此，每个子集唯一地决定了最终产品，并且每个可行解都恰好对应于一个子集。 穷尽所有子集可以保证我们检查每一个可能的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        c = []
        r = []
        for _ in range(n):
            ci, ri = map(int, input().split())
            c.append(ci)
            r.append(ri)

        ans = 0

        for mask in range(1 << n):
            if mask.bit_count() > m:
                continue

            color_sum = {}
            for i in range(n):
                if mask & (1 << i):
                    color_sum[c[i]] = color_sum.get(c[i], 0) + r[i]

            prod = 1
            for val in color_sum.values():
                prod *= val

            if prod > ans:
                ans = prod

        print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循子集枚举策略。 这`bit_count()`check 尽早强制执行大小约束，修剪无效子集。 字典`color_sum`聚合每种颜色的辐射，对所有允许的隐式合并进行建模。 

产品计算将所有聚合值相乘，跳过空颜色，因为它们不出现在字典中。 答案在所有子集中贪婪地更新。 

一个微妙的点是，单个辐射值很小，但产品可以快速增长。 Python 的任意精度整数可以防止溢出问题，因此不需要额外的处理。
