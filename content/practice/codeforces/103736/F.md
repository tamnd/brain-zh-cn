 ## 解决方案
 ## 问题理解

 给定一个整数序列，并要求计算有多少个连续子数组的总和可被给定整数整除$k$。 这里的子数组意味着我们选择一个起始位置和一个结束位置，并无间隙地获取其间的所有内容，然后计算这些元素的总和。 我们想知道这些总和中有多少是以下的倍数$k$，包括零。 

数组长度可达$10^5$，每个值可以大到$10^9$。 这立即排除了任何尝试从头开始重新计算每个子数组之和的解决方案，因为子数组本身的数量约为$O(n^2)$，太大了。 即使是单个二次环也已经存在$10^{10}$最坏情况下的操作，1秒限制内无法通过。 

一个微妙的点是值可以为零，因此在以下情况下必须对由单个零组成的子数组进行计数：$k$除零。 另一个极端情况出现时$k = 1$，其中每个子数组都是有效的，因为每个整数都可以被 1 整除，这意味着答案应该是$n(n+1)/2$。 任何正确的解决方案都必须自然地处理这个问题，而无需特殊的外壳。 

简单的前缀和检查可能看起来很安全，但由于固定宽度整数的语言中的溢出问题仍然会失败，因为前缀和可以达到$10^{14}$。 在 Python 中这不是问题，但在其他上下文中它在概念上很重要。 

## 方法

 蛮力的想法很简单。 对于每对$(l, r)$，计算子数组的总和$a_l \dots a_r$并检查它是否能被整除$k$。 即使我们预先计算前缀和，我们仍然会迭代所有索引对，从而导致$O(n^2)$检查。 

失败的原因不仅在于子数组的数量，还在于每对子数组的成本$O(1)$带有前缀和或$O(n)$没有他们。 无论哪种方式，二次尺度都是不可避免的。 

关键的观察是，当通过前缀和表示时，子数组和变得简单。 让$S_i = a_1 + \dots + a_i$。 然后对子数组求和$[l, r]$是$S_r - S_{l-1}$。 我们希望它可以被整除$k$， 意义：$$S_r \equiv S_{l-1} \pmod{k}$$所以问题不再是关于子数组，而是关于计算具有相等余数模的前缀和对$k$。 每个有效子数组完全对应于一对索引，其前缀和具有相同的模类。 

这将问题转化为对组内的组合进行计数：对于每个余数值$r$，如果出现$c_r$次前缀和中，它贡献$c_r(c_r - 1)/2$有效的子数组。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) 或 O(n) | 太慢了 |
 | 最优（前缀 mod 计数）| O(n) | O(min(n, k)) | O(min(n, k)) | 已接受 |

 ## 算法演练

 1. 迭代数组时计算前缀和，保持和模数$k$而不是全额。 这避免了大整数增长并仅保留相关的等价类。 
2. 维护一个频率图，记录到目前为止每个余数出现的次数。 用余数 0 出现一次来初始化它，表示在获取任何元素之前的空前缀。 
3. 对于每个元素，更新运行前缀和并计算其余数模$k$。 
4.每次看到余数时$r$，我们知道具有相同余数的所有先前前缀和可以形成以当前位置结尾的有效子数组。 所以我们添加当前频率$r$到答案。 
5. 增加余数的频率$r$。 

关键的决定是第 4 步。我们不是尝试显式地形成子数组，而是将每个前缀和解释为一个“状态”，并且状态的每次重复都会创建以当前索引结尾的新的有效子数组。 

### 为什么它有效

 正确性来自于在任意位置的不变量$i$，频率图准确地存储了索引中每个余数有多少个前缀和$0 \dots i$。 一个子数组$[l, r]$总和可除以$k$当且仅当前缀余数为$r$和$l-1$是平等的。 因此每次我们遇到余数时$r$，我们实际上选择任何先前出现的相同余数作为以当前索引结束的有效子数组的起始边界。 没有遗漏有效对，也没有计算无效对，因为不同的余数不能满足整除条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
arr = list(map(int, input().split()))

freq = {0: 1}
prefix = 0
ans = 0

for x in arr:
    prefix = (prefix + x) % k
    ans += freq.get(prefix, 0)
    freq[prefix] = freq.get(prefix, 0) + 1

print(ans)
```该实现跟踪前缀和模$k$直接，这避免了溢出和不必要的全和计算。 字典`freq`存储每个余数的计数。 初始条目`{0: 1}`是必不可少的，因为它考虑从索引 1 开始的子数组，其中前缀本身已经可以被整除$k$。 

一个常见的陷阱是在添加答案之前更新频率。 这样做会错误地计算当前前缀及其自身，它不对应于任何有效的子数组。

 ## 边缘情况

 像单个元素这样的最小情况测试初始前缀余数是否被正确处理。 有输入`1 3`和数组`[0]`，前缀余数立即为零，并且由于频率图以`{0:1}`，我们精确地计算出一个有效子数组。 该算法将此作为一个步骤处理，其中`prefix = 0`，添加`freq[0] = 1`，并返回 1。 

案例与`k = 1`更极端，因为每个前缀都有余数零。 为了`n = 4`和`[1,2,3,4]`，每一步都会增加余数零的频率，产生所有子数组的总数，即 10。频率机制自然地累积三角形计数，无需任何修改。 

全零数组的行为类似，但对于任何`k`。 由于每个前缀和的模数相同`k`，每对索引形成一个有效的子数组。 频率图重复计算增加的​​组合，产生$n(n+1)/2$，匹配预期的组合结构。
