 ## 解决方案
 ## 问题理解

 我们维护一个以空开头并通过一系列操作进行修改的字符串。 每个操作要么在末尾附加一个小写字符，要么删除最后一个字符（如果存在），要么执行全局替换，将当前字符串中出现的每个给定字符替换为另一个字符。 

任务是按顺序模拟所有操作并输出最终结果字符串。 如果没有留下任何内容，我们会打印一条特殊消息来指示空。 

关键难点是第三次操作。 扫描整个字符串并替换每个类型 3 查询的字符的直接模拟太慢。 由于多达 100000 次操作、一串线性大小以及可能的许多全局替换，这种简单的方法会退化为二次或更糟糕的行为。 

边缘情况主要来自删除和替换之间的相互作用。 替换不依赖于位置，只依赖于角色身份，但删除意味着角色在经过逻辑转换后可以消失。 例如，如果我们附加“ab”，然后用 c 替换 a，然后删除，最终的结构取决于我们是急切还是惰性地应用转换。 幼稚的实现也可能会忘记重复的替换可以形成链，例如从 a 到 b，然后从 b 到 c。 

另一个微妙的情况是重复的更换周期。 如果将 a 替换为 b，然后将 b 替换为 a，则重复重写整个字符串的简单实现会带来重复完全重新扫描的风险。 

## 方法

 暴力解决方案直接存储字符串并按字面应用操作。 末尾的追加和删除都是 O(1)，但替换操作会扫描整个字符串并将所有出现的 x 替换为 y。 在最坏的情况下，我们可能有 O(q) 次操作，每次替换的成本可能为 O(n)，其中 n 增长到 q。 这会产生 O(q²)，这对于 100000 次操作来说远远超出了可接受的范围。 

关键的观察是，当发生替换时，字符串本身不需要重写。 每个字符的重要性并不在于它的字面值，而在于它当前在动态字符转换系统下的映射方式。 我们不修改字符串，而是维护从每个字母到其当前代表的映射。 当我们附加一个字符时，我们存储它当前的映射形式。 当我们查询替换时，我们会更新映射。 当我们删除时，我们会删除最后存储的映射字符。 

重要的想法是替换是对字符的函数组合，而不是对存储文本的修改。 字符串中的每个字符都会记住其原始身份，并且映射告诉我们它当前代表什么。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(q²) | O(q) | 太慢了 |
 | 映射+堆栈| O(q) | O(q) | 已接受 |

 ## 算法演练

 我们维护两种结构。 一种是表示附加字符序列的堆栈。 第二个是数组`mp`在哪里`mp[c]`是当前的代表人物`c`。 

我们还隐式地维护了相反的想法：我们从不重写堆栈，我们只通过以下方式解释它的值：`mp`。 

1.初始化一个数组`mp`这样每个字符都映射到它自己。 初始化一个空堆栈。 
2. 对于类型 1 的字符操作`x`， 推`mp[x]`到堆栈上。 我们立即存储映射的版本，以便将来的更改不会影响历史意图。 
3、对于类型2的操作，如果栈不为空则出栈。 这直接模拟删除当前字符串的最后一个字符。 
4. 对于类型3的字符操作`x`和`y`，更新映射，以便当前映射到的每个字符`x`应该映射到`y`。 具体来说，我们更新所有条目`mp[c]`在哪里`mp[c] == x`，将它们设置为`y`。 由于字母表大小仅为 26，因此这是常数时间。 
5. 处理完所有操作后，通过使用最终映射转换堆栈中的每个存储字符来重建答案。 

为什么这种方法有效，是因为将身份与表征分开。 堆栈存储原始追加操作的序列，而`mp`累积之后应用的所有转换。 堆栈中的任何字符始终通过以下方式反映其当前含义`mp`。 由于替换是基于角色身份的全局而不是位置相关的，因此这种分离是精确且无损的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q = int(input())
    mp = [chr(ord('a') + i) for i in range(26)]
    st = []

    for _ in range(q):
        parts = input().split()
        if parts[0] == '1':
            x = parts[1]
            st.append(mp[ord(x) - 97])
        elif parts[0] == '2':
            if st:
                st.pop()
        else:
            x = parts[1]
            y = parts[2]
            x = mp[ord(x) - 97]
            y = mp[ord(y) - 97]
            for i in range(26):
                if mp[i] == x:
                    mp[i] = y

    if not st:
        print("The final string is empty")
    else:
        print("".join(st))

if __name__ == "__main__":
    solve()
```该实现将堆栈作为结构的真实来源，而映射数组则不断发展以反映全局替换。 关键的微妙之处在于应用`mp[x]`在插入期间，这确保旧字符被冻结为其当前的语义形式。 在更换过程中，我们将两者标准化`x`和`y`在应用更新之前通过当前映射，防止存在替换链时出现不一致。 

最终的重建是微不足道的，因为堆栈中的所有字符都已经反映了它们的最终映射状态。

 ## 边缘情况

 一种边缘情况是超出空值的重复删除。 对于像这样的输入`1 a`随后是多个`2`操作时，堆栈必须保持为空且不会出现错误。 该实现在弹出之前检查堆栈长度，以确保安全。 

另一种情况是重复更换形成循环。 例如，`a→b`,`b→c`,`c→a`。 映射方法可以处理这个问题，因为每个操作都会根据当前代表进行重写，从而保持一致性而无需历史回滚。 

最后一个微妙的情况是删除后的替换。 由于删除仅影响堆栈，而不影响映射，因此删除元素后应用替换不会破坏先前的状态。 映射继续仅适用于剩余字符，弹出的元素将被永久丢弃。
