 ## 解决方案
 ## 问题理解

 我们得到一个小写字符串并要求计算固定模式出现了多少次`"hznu"`在其内部显示为一个连续的块。 每个有效的出现必须按顺序匹配所有四个字符，并且必须占据连续的位置，因此只要它们从不同的索引开始，就允许重叠。 

输入长度最多可达 100000 个字符。 该大小立即排除了任何重复构建子字符串或以二次行为执行繁重的每个位置处理的方法。 检查每个可能的起始索引并仅检查恒定数量的字符的解决方案已经足够了，但是只有仔细实施时，涉及重复字符串切片或对整个字符串进行嵌套扫描的任何操作仍然是安全的。 

一些边缘情况自然会出现。 该字符串可以短于四个字符，例如`"hzn"`显然应该返回 0，因为不存在完全匹配。 完全由重复字母组成的字符串，例如`"hhhhhhhh"`也返回 0，因为该模式需要特定的序列。 另一个微妙的情况是重叠模式，例如`"hzhznu"`恰好包含从位置 2 开始的一个有效出现，如果实施不正确，粗心的子串提取会错误地移位，可能会错过边界或重复计算边界。 

## 方法

 最直接的想法是检查每个可能的起始索引并检查长度为 4 的子串是否等于`"hznu"`。 对于每个位置 i，我们比较字符 s[i]、s[i+1]、s[i+2] 和 s[i+3]。 如果全部匹配，我们就增加答案。 

这种强力方法是正确的，因为模式的每次出现都必须从某个索引 i 开始，并且检查所有此类索引可确保不会跳过任何有效匹配。 成本来自于扫描最多 n 个位置并在每个位置执行恒定的工作量，从而提供线性时间。 尽管它已经很高效，但在 Python 中重复构建像 s[i:i+4] 这样的子字符串的简单变体对于此约束仍然是安全的，但它引入了不必要的开销。 

不需要像 KMP 这样的高级字符串算法，因为模式非常小并且固定。 关键的观察是模式匹配减少到每个索引处的局部比较。

 ## 算法演练

 1. 读取输入字符串 s 并定义目标模式为`"hznu"`。 模式是固定的，永远不会改变，因此我们可以将其视为四个常量字符比较。 
2. 将计数器变量初始化为零。 这将累积在字符串中找到的有效匹配项的数量。 
3. 迭代从 0 到 len(s) - 4（含）的所有索引 i。 每个索引代表完整 4 字符匹配的可能起始位置。 我们在 len(s) - 4 处停止，因为超出该值的任何索引都无法容纳四个字符。 
4. 对于每个索引 i，将子串 s[i:i+4] 与`"hznu"`。 如果它们相等，则将计数器加一。 这种直接比较确保我们只计算完整的匹配项。 
5. 扫描完成后，输出计数器作为最终结果。 

### 为什么它有效

 每个有效出现的`"hznu"`恰好对应于字符串中的一个起始索引 i。 该算法对每个此类索引精确检查一次，并针对所需模式执行精确的相等测试。 由于没有位置被跳过并且没有无效位置可以满足相等性检查，因此计数既完整又正确。 重叠是自然处理的，因为每个起始索引都是独立评估的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    s = input().strip()
    target = "hznu"
    n = len(s)
    ans = 0

    for i in range(n - 3):
        if s[i:i+4] == target:
            ans += 1

    print(ans)

if __name__ == "__main__":
    main()
```该代码读取字符串并仅迭代到`n - 3`，这保证我们永远不会访问越界索引。 切片`s[i:i+4]`即使在 Python 中的边界附近也是安全的，但循环边界确保我们只考虑有效的起始位置。 

直接针对固定字符串进行比较`"hznu"`，这避免了任何额外的预处理。 该解决方案保持线性，因为每次迭代都执行恒定的工作。

 ## 边缘情况

 短字符串，例如`"hzn"`由循环条件处理`range(n - 3)`，这变成`range(-1)`并导致零次迭代，因此输出保持为 0。 

边界对齐匹配，例如`"hhhhhznu"`被正确处理，因为最终的有效索引包含在循环和切片中`s[i:i+4]`准确地捕捉到了最后的模式。 

重叠的结构，例如`"hznhznu"`自然会被计算在内，因为每个指标都是独立测试的。 第一次出现从位置 0 或 1 开始，具体取决于对齐方式，第二次出现在稍后的索引处，检查之间没有干扰，因为迭代之间没有共享状态。
