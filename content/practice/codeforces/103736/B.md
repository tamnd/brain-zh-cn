 ## 解决方案
 ## 问题理解

 我们得到了英文小写字母表的自定义排序，其中 26 个字母以定义严格总顺序的特定顺序出现。 如果某个字母在此排序中出现较早，则它被视为较小。 这会在由这些字母构建的字符串上产生字典顺序。 

定义此自定义字母顺序后，我们将获得一个字符串集合。 任务是确定如果我们使用此自定义词典顺序对所有给定字符串进行排序，然后使用长度扩展的标准词典比较作为最终的决胜局，则确定哪个字符串将出现在位置 K。 

关键是“字典顺序最小”并不是基于正常的`'a' < 'b' < ... < 'z'`，而是根据提供的字母表排列。 

从计算的角度来看，n 最多为 1000，每个字符串长度最多为 1000。这使得 O(n log n) 的字符串比较排序可行，因为每次比较最坏的时间为 O(1000)。 对所有排列进行强力枚举或构建完整的 trie 排名结构将是不必要的矫枉过正。 

字母顺序不标准这一事实导致了一种微妙的边缘情况。 例如，如果订单开头为`zabcdefghijklmnopqrstuvwxy`， 然后`'z'`是最小的并且`'a'`是第二小的。 忘记重新映射字符的幼稚实现会产生完全错误的排序，同时仍然通过许多意外测试。 

另一个潜在的陷阱是误解词典比较规则。 比较是前缀敏感的：如果一个字符串是另一个字符串的前缀，则较短的字符串较小。 例如，如果`"a"`和`"ab"`进行比较，`"a"`无论字母顺序如何，都必须排在第一位。 

## 方法

 强力方法是定义一个比较器，使用给定的字母顺序直接逐个字符地比较两个字符串，然后对整个数组进行排序。 这是正确的，因为字典顺序完全由成对比较确定。 然而，这种方法仍然依赖于字符串的重复比较，并且每次比较最多可以扫描 O(L) 个字符，其中 L 是最大字符串长度。 当 n 最大为 1000，L 最大为 1000 时，排序的结果大约为 O(n log n · L)，在最坏的情况下大约是 10^7 个字符检查，这在 Python 中仍然可以接受。 

更结构化的观察是，唯一的困难在于有效地比较字符。 一旦我们将每个字符转换为自定义字母表中的排名，每个字符串就变成整数序列，并且标准字典顺序直接应用。 这消除了比较中重复字典查找的开销，并使排序在实践中更干净、更快。 

因此，最佳解决方案就是使用转换后的键构建从字符到排名和排序的映射。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力比较器| O(n log n·L) | O(n log n · L) | O(1) 额外 | 已接受 |
 | 排名映射 + 键排序 | O(n·L log n) | O(n·L log n) | O(26 + nL) | O(26 + nL) | 已接受 |

 ## 算法演练

 1.读取自定义字母顺序字符串并构造数组`rank`这样`rank[c]`给出角色的位置`c`按这个顺序。 这允许任意两个字符之间进行 O(1) 比较。 这是必要的原因是直接字符串比较依赖于对非标准顺序的重复查找。 
2. 将所有输入字符串读入列表中。 每个字符串稍后将根据自定义顺序转换为可比较的表示形式。 
3. 对于每个字符串，构造一个转换版本，其中每个字符都被其排名值替换。 这会将自定义字母表下的词典比较转换为整数上的正常词典比较。 从概念上讲，此步骤是问题的标准化。 
4. 使用这些转换后的表示形式作为排序键对字符串列表进行排序。 然后，Python 的内置排序将正确地对元组或整数列表应用字典顺序。 
5、输出原列表中排序后的第K个字符串。 我们将原始字符串与其转换后的键一起保留，这样我们就不会丢失实际的输出格式。 

### 为什么它有效

 该转换保留了顺序，因为字典比较规则仅取决于字符的相对顺序，而不取决于它们的实际身份。 通过将每个字符映射到其在自定义字母表中的排名，我们将自定义顺序嵌入到标准整数顺序中。 由于Python按字典顺序比较序列，因此比较转换后的数组相当于比较给定字母表下的原始字符串。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    order = input().strip()
    rank = [0] * 26
    for i, ch in enumerate(order):
        rank[ord(ch) - 97] = i

    n = int(input())
    arr = []
    for _ in range(n):
        s = input().strip()
        key = tuple(rank[ord(c) - 97] for c in s)
        arr.append((key, s))

    k = int(input())
    arr.sort(key=lambda x: x[0])
    print(arr[k - 1][1])

if __name__ == "__main__":
    solve()
```该实现首先构建从字符到其自定义顺序索引的直接映射。 这避免了在比较期间重复扫描字母串。 

每个字符串都按照新的排序与其字符的元组表示形式配对。 使用元组是因为 Python 在 O(元组长度) 中按字典顺序比较元组，这与预期的字符串比较行为完全一致。 

排序是使用这些键完成的，排序后我们提取第 K 个元素。 

一个常见的微妙错误是忘记将字符转换为等级，而是依赖 ASCII 排序，只要自定义字母表与标准顺序不同，这就会完全使结果无效。

 ## 边缘情况

 关键的边缘情况是字符串相同，但在自定义字母表中顺序不重要的字符除外。 例如，通过订购`acb...`, 比较`"ab"`和`"ac"`完全取决于相对顺序`b`和`c`。 该算法可以正确处理此问题，因为两个字符在比较之前都会转换为排名。 

另一个边缘情况是前缀关系。 考虑`"ab"`和`"a"`。 即使`b`比字母表中的大多数字符都小，`"a"`仍然必须排在第一位，因为比较在较短字符串的末尾终止。 元组表示自动保留这一点，因为`(rank('a'),)`短于`(rank('a'), rank('b'))`，Python 的字典元组比较将较短的前缀视为较小的。 

最后，可以安全地处理具有最大长度字符串的情况，因为每个字符串的转换都是线性的，并且不会修改转换之外的结构，从而确保不会出现隐藏的递归或深度复制开销。
