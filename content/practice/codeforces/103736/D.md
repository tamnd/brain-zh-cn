 ## 解决方案
 ## 问题理解

 我们正在使用一棵树，这意味着一个连接的非循环图，其中每对顶点都由一条简单路径连接。 对于每个查询顶点 x，我们需要计算树中有多少条不同的简单路径经过 x。 路径被认为是无向的，因此从 a 到 b 的行进与从 b 到 a 的行进相同。 

一个关键细节是，有效路径可以在树中的任何位置开始和结束，只要它沿路线的某个位置经过 x 即可。 路径必须很简单，因此没有重复的顶点。 此外，长度至少为 1 的路径也会被计算在内，因此不包括单个顶点路径。 

输入大小高达 100000 个顶点和 100000 个查询，因此任何在线性时间内重新计算每个查询信息的解决方案都会太慢。 直接枚举所有路径是不可能的，因为具有 n 个节点的树已经具有 θ(n²) 条简单路径，并且根据每个查询检查每个路径将远远超出任何可行的限制。 

当树是以 x 为中心的星形时，会出现微妙的边缘情况。 在这种情况下，几乎每条路径都经过 x，并且朴素的枚举方法往往会因错误地处理方向或端点而导致计数过多。 例如，如果x连接到a，b，c，则路径[a，x，b]与[b，x，a]相同，并且只能计算一次。 

## 方法

 一个蛮力的想法是尝试枚举树中的所有简单路径，并且对于每个查询节点 x，检查 x 是否位于该路径上。 这立即遇到两个问题。 首先，枚举所有简单路径已经是树中的 θ(n²) 。 其次，每个查询执行此操作都会导致 θ(n²q)，这在给定的约束下是完全不可行的。 

我们需要一种方法来计算通过 x 的路径而不列出它们。 关键的结构观察是，删除节点 x 会将树拆分为多个连接的组件，每个组件对应 x 的每个邻居。 任何经过 x 的简单路径都必须来自一个组件，经过 x，然后继续进入另一个组件（或结束于 x）。 这将问题简化为计算来自不同邻居子树的节点对以及以 x 开始或结束的路径。 

如果我们任意对树求根并计算子树大小，我们就可以了解每个分支中有多少个节点与 x 相邻。 假设 x 的度数为 k，移除 x 会将树分割为大小为 s1、s2、...、sk 的 k 个分量。 通过选择不同组件中的两个端点或选择组件中的一个端点和 x 本身来确定穿过 x 的任何路径。 

因此，答案就变成了不同组件中节点对的数量加上与 x 相关的所有单边路径。 跨组件贡献是标准组合恒等式：总对减去每个组件内的对。 

我们使用以任意节点为根的 DFS 预先计算子树大小。 对于每个节点x，我们可以确定每个子端组件的大小。 唯一缺少的情况是 x 的“父端”，它对应于当有根时 x 子树之外的树的其余部分。 

这给出了 O(n) 预处理解决方案和 O(1) 查询应答。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（枚举路径）| 每次查询 O(n²) | O(n²) | 太慢了 |
 | 最优（子树分解）| O(n) 预处理 + O(1) 每个查询 | O(n) | 已接受 |

 ## 算法演练

 我们以节点 1 为树根并使用 DFS 计算子树大小。 对于每个节点 x，我们还跟踪每个子子树的大小。 

对于固定的查询节点 x，我们将其关联边解释为将图拆分为多个组件：

1. 计算 x 周围每个分量的大小。 x 的每个子 v 贡献一个大小等于 v 子树大小的组件。剩余组件大小是 n 减去 x 的子树大小。 
2. 将每个组件大小视为一个节点桶。 通过 x 的任何有效路径都是通过从这些存储桶中选择端点或使用 x 本身来确定的。 
3. 统计所有以 x 为端点的路径。 恰好有 (n − 1) 条这样的路径，因为 x 可以通过唯一的简单路径连接到每个其他节点。 
4. 计算 x 严格为内部的所有路径。 这些路径的端点位于两个不同的组件中。 如果组件大小为 s1、s2、...、sk，则此类路径的数量为 si * sj 的 i < j 之和。 
5. 将两个贡献结合起来得到答案。 

计算跨分量和的一种便捷方法是使用恒等式：

 sum_{i < j} si * sj = ( (sum si)² − sum si² ) / 2。 

这里总和 si = n − 1。 

### 为什么它有效

 每条经过 x 的简单路径都必须从一个邻居进入 x 并离开到另一个邻居，或者终止于 x。 删除 x 会将树划分为独立的组件，因此路径的端点唯一地确定它们位于哪个组件中。这会在有效路径与与 x 配对的单个端点或不同组件中的两个端点之间创建双射。 没有路径会被计算两次，因为每个简单路径都有一对唯一的端点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n + 1)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    g[u].append(v)
    g[v].append(u)

parent = [0] * (n + 1)
sz = [0] * (n + 1)

def dfs(u, p):
    parent[u] = p
    sz[u] = 1
    for v in g[u]:
        if v == p:
            continue
        dfs(v, u)
        sz[u] += sz[v]

dfs(1, 0)

q = int(input())

for _ in range(q):
    x = int(input())

    comp_sizes = []

    for v in g[x]:
        if v == parent[x]:
            comp_sizes.append(n - sz[x])
        else:
            comp_sizes.append(sz[v])

    total = 0
    sum_s = 0
    sum_sq = 0

    for s in comp_sizes:
        sum_s += s
        sum_sq += s * s

    total_pairs = (sum_s * sum_s - sum_sq) // 2

    # paths where x is endpoint
    total = total_pairs + (n - 1)

    print(total)
```DFS 计算子树大小，以便每个节点知道有根树中有多少个顶点位于其下方。 对于查询节点 x，我们检查其邻接列表并将每个邻居转换为组件大小。 一种特殊情况是父边，它代表 x 子树之外的树的其余部分。 

然后，我们使用标准代数恒等式计算来自不同组件的节点对的数量。 最后，我们添加以 x 为端点的所有路径，即 n − 1，因为每个其他节点都定义了一条到 x 的简单路径。 

必须小心正确识别父组件。 如果不存储父级信息，我们将错误地将所有邻居视为子树子级，并丢失外部组件。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n = int(input())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    sys.setrecursionlimit(10**7)
    parent = [0] * (n + 1)
    sz = [0] * (n + 1)

    def dfs(u, p):
        parent[u] = p
        sz[u] = 1
        for v in g[u]:
            if v == p:
                continue
            dfs(v, u)
            sz[u] += sz[v]

    dfs(1, 0)

    q = int(input())
    out = []
    for _ in range(q):
        x = int(input())
        comp = []
        for v in g[x]:
            if v == parent[x]:
                comp.append(n - sz[x])
            else:
                comp.append(sz[v])

        s = sum(comp)
        ss = sum(c * c for c in comp)
        ans = (s * s - ss) // 2 + (n - 1)
        out.append(str(ans))

    return "\n".join(out)

 ## 边缘情况

 二节点树是最简单的边界。 对于输入：```
2
1 2
1
1
```节点 1 具有大小为 1 的单个组件。跨组件对为零，端点路径贡献恰好为 1，因此答案为 1，与单个路径 [1,2] 匹配。 

星形树强调组件分裂逻辑。 对于具有四个叶子的中心 x，分量为 [1,1,1,1]。 交叉对给出 6 条路径，端点贡献加 4，总计 10。在删除 x 后，该算法正确地将所有叶子视为单独的组件。 

链条检查方向偏差。 对于路径 1-2-3-4-5 中的节点 3，分量为 [2,2]。 交叉对给出 4，端点贡献给出 4，总共 8？ 实际上端点路径是 4，所以总数是 8，匹配包含 3 的所有路径一次。 计算保持一致，因为子树大小正确反映了链的两侧。
