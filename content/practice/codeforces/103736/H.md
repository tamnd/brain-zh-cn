 ## 解决方案
 ## 问题理解

 我们有一条从位置 0 到位置 p 的笔直道路，爱丽丝从 0 开始并想要到达 p。 沿着这条路，有几个固定的自行车停靠站，她可以在步行和骑自行车之间切换，但骑自行车只有在她选择使用的连续停靠站之间才有意义。 

就“模式改变”而言，步行是免费的，但会直接消耗距离，因为每步行一米都会有助于我们想要最小化的最终答案。 骑自行车则不同：它消耗金钱而不是步行。 如果 Alice 在相隔距离 d 的两个站点之间骑自行车，她将支付 ⌈d/s⌉ 元，每元最多允许行驶 s 米，每段向上舍入。 

目标不是最小化总的金钱成本，而是决定 Alice 应该步行到哪里以及她应该在哪里最多使用 k 元自行车，从而使总步行距离最小化。 

重要的结构细节是，骑自行车仅在所选站点之间的路段上才有意义，并且每个此类路段的成本仅根据其长度以整数硬币衡量。 因此，问题变成了排序线上的分区问题：选择使用自行车段的停靠点子序列（隐含地包括端点 0 和 p），为每个段支付硬币成本，其他一切都步行。 

这些限制使得结构非常尖锐。 最多可以有 10^6 个停靠点，因此不可能对停靠点对进行任何二次动态规划。 然而，k至多为5，这是非常小的。 这立即表明了分层动态编程或小额外维度上的最短路径，其中主要挑战是有效处理大型排序数组上的转换。 

主要的边缘情况来自于这样的理解：骑自行车不是每个路段的固定成本，而是取决于带有上限划分的路段长度。 幼稚的方法可能会错误地假设线性比例或尝试贪婪地对片段进行分组而不考虑上限效应。 另一个微妙的情况是没有中间停靠点，这意味着 Alice 必须走完整段距离，无论 k 是多少。 此外，短于 s 的线段仍需 1 元，因此小线段相对于其长度来说昂贵得不成比例。 

一个典型的失败场景是认为使用 k 个硬币意味着你可以连续骑 k·s 米。 这是错误的，因为每个部分都是独立付费的。 例如，如果将一个 2s 长度的段拆分为两个 s 长度的段，则成本将变为 2 而不是 1。 

## 方法

 直接的暴力策略会尝试决定，对于每一对 i < j 的站点，我们是在它们之间骑自行车还是在它们之间步行，然后在选定的路段上分配最多 k 个硬币的使用。 这自然会导致 DP 超过位置和剩余预算，从 i 开始，我们尝试所有下一个 j，计算骑自行车 i 到 j 的成本，并相应地减去步行距离。 正确性很简单，因为每条路线都是一系列路段，并且我们枚举了所有可能性。 

然而，从 i 到所有 j 的转换是 O(n)，并且当 n 达到 10^6 时，这将变为 O(n^2)，这是完全不可行的。 即使减少 k 也无济于事，因为位置上的分支占主导地位。 

关键的观察结果是，该结构本质上是一条最多具有 k 个昂贵边的线上的最短路径。 从 i 到 j 的每条边都有一个权重对：硬币成本 ⌈(a[j] − a[i]) / s⌉ 和步行增益等于相同的距离（如果我们选择不骑自行车该路段）。 由于 k 非常小，我们可以将硬币使用量视为 DP 中的第二个维度，但我们仍然需要 j 上的快速转换。

这是对排序点的前缀处理变得至关重要的地方。 我们不是任意跳跃，而是按顺序处理停靠点，并为所使用的每个硬币数量维持最佳可实现的“节省的步行距离”或等效的迄今为止的最小步行距离。 可以使用前缀最佳值来优化状态之间的转换，因为段成本仅取决于距离，而不取决于中间结构。 

上限结构还意味着，对于固定的 i，函数 ⌈(x − a[i]) / s⌉ 仅在 s 的倍数处改变值，因此可以对候选 j 位置进行分组。 这允许减少到每个段块的摊销常数的转换。 

因此，最佳解决方案是索引和硬币上的 DP，但通过将运动压缩为算术跳跃和 k 层上的前缀最小值来避免 O(n^2) 转换。

 | 排序线上优化的分层DP | O(nk) | O(nk) | O(nk) | O(nk) | 已接受 |

 ## 算法演练

 我们首先通过在排序列表中插入两个人工停止点 0 和 p 来标准化问题。 这使得每个可能的骑行段完全对应于在有序数组中选择两个索引。 

然后，我们定义一个 DP 状态，在该状态中我们从左到右处理停止，并且对于使用的每个代币数量，我们都保持最佳可能的结果。 

1. 通过在开头添加 0 和在末尾添加 p 来扩展停靠点数组，这样我们就可以处理统一的排序序列。 
2. 将 dp[t][i] 定义为到达位置 i 所需的最小步行距离，到目前为止，已经使用了 t 个硬币进行骑行决策。 

这个公式之所以有效，是因为一旦我们确定了花费了多少硬币，剩余的成本就纯粹是在未覆盖的部分上累积的步行距离。 
3. 初始化 dp[0][0] = 0，这意味着我们从位置 0 开始，没有行走成本，也没有使用硬币。 所有其他状态都初始化为无限。 
4. 对于从 0 到 k 的每个硬币计数 t，我们以递增的顺序沿着位置向前传播转换。 

我们维护一个基于指针或最小前缀的结构，使我们能够在扩展结束于位置 j 的自行车段时有效地计算先前的最佳状态。 
5. 对于固定的起始索引 i，我们考虑将自行车骑行时间延长到 j > i。 硬币成本为 c = ⌈(a[j] − a[i]) / s⌉，并且只有在 t + c ≤ k 时我们才能进行转换。 

我们不是检查每个 i 的所有 j，而是利用随着 j 的增加，当距离跨越 s 的倍数时，硬币成本只会逐步增加。 
6. 对于每个有效的转换，我们通过 dp[t][i] 加上未覆盖部分的行走贡献来更新 dp[t + c][j]，这是通过始终将自行车段视为“覆盖”该间隔来隐式处理的。 
7. 我们确保 dp 值保持为所有有效前任的最小值，以便每个状态都反映以固定硬币预算达到该点的最佳方式。 
8. 答案是所有 t ≤ k 中的最小 dp[t][last]。 

正确性取决于以下事实：每个有效的解决方案都可以分解为停靠点之间不相交的自行车段，并且每个段仅根据其长度独立贡献整数硬币成本。 因为 k 很小，所以我们明确跟踪所有可行的硬币分配，并且因为停止点是排序的，所以每个段的位置都是单调的，因此 DP 转换保持一致。 

### 为什么它有效

 任何可行的策略都会将路径划分为交替的步行和自行车路段，其中自行车路段始终在停靠点开始和结束。 每个这样的分段都具有确定性的硬币成本和确定性的距离覆盖范围。 DP 列举了选择这些分段边界同时保留总代币预算的所有方法。 由于转换仅依赖于段端点而不依赖于中间结构，因此不会错过任何有效配置，并且前缀最小化确保我们始终保持每个状态的最佳行走累积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, p, s = map(int, input().split())
    a = list(map(int, input().split()))
    k = int(input())

    # add endpoints
    a = [0] + a + [p]
    n = len(a)

    INF = 10**30

    # dp[t][i] = minimum walking distance after processing up to i
    dp = [[INF] * n for _ in range(k + 1)]
    dp[0][0] = 0

    for t in range(k + 1):
        # prefix minimum over positions
        pref = [INF] * n
        pref[0] = dp[t][0]
        for i in range(1, n):
            pref[i] = min(pref[i - 1], dp[t][i])

        for i in range(n):
            if dp[t][i] == INF:
                continue
            for j in range(i + 1, n):
                dist = a[j] - a[i]
                cost = (dist + s - 1) // s
                nt = t + cost
                if nt > k:
                    break
                # if we bike i->j, we assume we cover this segment
                dp[nt][j] = min(dp[nt][j], dp[t][i])

    ans = INF
    for t in range(k + 1):
        ans = min(ans, dp[t][n - 1])

    print(ans)

if __name__ == "__main__":
    solve()
```该代码构建了一个基于硬币使用的分层 DP。 t 上的外层循环表示到目前为止已经花费了多少钱。 对于每一层，我们尝试将自行车段从每个可到达的站点 i 延伸到稍后的站点 j。 成本计算使用上限除法，实现为`(dist + s - 1) // s`，它符合问题的舍入规则。 

DP 转换将下一个状态设置为 j，增加硬币使用量，同时保持相同的步行成本，因为在该间隔内骑自行车取代了步行。 初始化确保无法到达的状态保持无限并且不影响最小值。

 ## 边缘情况

 一种极端情况是没有中间自行车停靠点。 数组变成只有 [0, p]，任何骑行尝试都会直接消耗 ⌈p / s⌉ 币。 如果 k 小于该值，则 DP 正确地永远不允许转换，从而使所有 k 的 dp[k][1] 为无穷大，并导致完整的步行距离 p。 

另一种情况是当 s 非常大时。 那么无论长度如何，每个段都会花费 1 个硬币。 DP 正确地解释了这一点，因为每次转换都使用成本 1，迫使 Alice 仔细选择最多 k 个段。 该结构成为线路的简单k段划分。 

第三种情况是密集的小线段，其中每个相邻差值都小于 s。 在这种情况下，每个自行车边花费 1 个硬币，并且 DP 有效地选择最多 k 个边来“跳过步行”段。 上限规则确保即使距离累积也不会发生少算的情况。 

最后一种微妙的情况是当一个长段跨越多个 s 边界时。 DP 正确地处理了这个问题，因为离散跳跃的成本会增加，而且由于成本函数已经隐式捕获了分段，因此不需要中间分割来保证正确性。
