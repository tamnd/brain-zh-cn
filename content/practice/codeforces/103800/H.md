 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$网格，其中每个单元格包含一个不同的整数。 两个代理同时在这个网格上移动。 Ginger 从左上角的单元格开始，最初面朝下。 他的克隆体从右下角的单元格开始，最初面朝上。 

两个代理遵循相同的移动规则。 每一步，他们都试图让一个细胞朝着当前的方向前进。 如果下一个单元格超出网格，它们会向左旋转 90 度，并继续使用新方向从当前位置移动。 这会为每个代理产生确定性行走，该行走持续到所有可到达的单元最终以螺旋状模式被访问为止。 

我们被要求输出 Ginger 访问的值序列和克隆访问的序列。 还有一个额外的同步规则：如果两个智能体同时到达同一个单元，则该单元的值仅记录在 Ginger 序列中。 

网格大小最多为$200 \times 200$，所以最多有 40,000 个单元格。 任何一步步模拟运动的解决方案都是可行的，因为每一步都是$O(1)$总步骤数受每个代理的细胞数量限制。 

一个经常导致错误解决方案的天真的误解是将运动视为独立的螺旋顺序而不同步时间。 例如，在 2×2 网格中：```
1 2
3 4
```在相同的翻转规则下，姜访问 1 → 3 → 4 → 2，而克隆访问 4 → 2 → 1 → 3。 如果我们在没有时间对齐的情况下错误地单独计算螺旋，我们可能会错误地处理冲突，例如在较大的网格中以相同的步骤到达相同的单元格。 

关键的微妙之处在于，“同一时间步长”条件迫使我们以同步方式模拟两次行走，而不是预先计算两次独立的遍历。 

## 方法

 暴力方法是直接模拟两个代理。 我们维护每个代理的位置、方向和访问的单元格集。 在每个时间步，两者都尝试移动。 如果下一个单元格出界，我们向左旋转方向并重新计算移动。 我们将像元值附加到各自的序列中，当两者同时落在同一像元上时应用平局规则。 

每个代理最多访问每个单元一次。 由于有$n^2$细胞和两个智能体，总步数为$O(n^2)$。 每一步都是持续不断的工作，因此完整的模拟已经足够高效$n \le 200$。 不需要更先进的结构，因为除了边界转向之外，没有回溯或重新审视复杂性。 

唯一有意义的优化是仔细实施方向变化，并确保我们在完成所有单元格后不会意外地模拟额外的移动。 该结构本质上是从对角开始的同步螺旋行走。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(n^2)$|$O(n^2)$| 已接受 |
 | 最佳模拟 |$O(n^2)$|$O(n^2)$| 已接受 |

 实际上，两者是相同的，因为问题结构不允许除了模拟行走本身之外进行渐近改进。 

## 算法演练

 我们一步步模拟两个步行者，保持他们当前的位置和方向。 方向被编码为向量，“左转”对应于逆时针旋转方向向量。 

1. 初始化 Ginger$(0,0)$方向为“向下”，并克隆于$(n-1,n-1)$方向为“向上”。 我们还准备了两个访问矩阵或计数器，以确保每个代理的每个单元格被准确记录一次。 
2. 维护两个输出列表，一份用于 Ginger，一份用于克隆。 
3. 在每个时间步，通过尝试沿当前方向移动一步来计算 Ginger 的下一个位置。 如果该位置在网格之外，则向左旋转方向并重新计算步骤。 重复直到找到有效的移动。 
4. 在同一时间步中对克隆独立应用相同的逻辑。 这确保了两个代理同步发展。 
5. 确定两次移动后，比较目标单元格。 如果两个代理都落在同一个单元格上，则仅将其值附加到 Ginger 的序列中。 否则将每个值附加到其各自的序列中。 
6. 将两个访问过的单元标记为已为其各自的代理处理，并继续，直到两个序列都包含准确的内容$n^2$总覆盖意义上的元素，这意味着每个遍历规则所有网格单元都被分配了一次。 

正确性取决于每个代理的移动规则定义了网格上的确定性行走，并且同步仅影响输出标记，而不影响移动。 

### 为什么它有效

 每个智能体的移动规则定义了任何状态（位置、方向）的唯一后继函数。 由于网格是有限的，并且转弯仅发生在边界命中时，因此路径最终在螺旋状遍历中恰好覆盖所有单元一次。 由于两个代理独立地遵循相同的确定性规则，因此它们在每个时间步的位置都是明确定义的。 当两个智能体在同一时间步到达同一单元时，唯一的歧义就会出现，并且通过将单元分配给 Ginger 可以保持一致性，而不影响未来的移动，因为移动仅取决于位置和方向，而不取决于所有权。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    grid = [list(map(int, input().split())) for _ in range(n)]

    # directions: down, left, up, right (clockwise order of turns is left rotation)
    dirs = [(1,0),(0,-1),(-1,0),(0,1)]

    # Ginger: start (0,0), direction down (0)
    gx, gy, gd = 0, 0, 0

    # Clone: start (n-1,n-1), direction up (2)
    cx, cy, cd = n-1, n-1, 2

    g_seen = [[False]*n for _ in range(n)]
    c_seen = [[False]*n for _ in range(n)]

    g_res = []
    c_res = []

    def move(x, y, d):
        for _ in range(4):
            nx = x + dirs[d][0]
            ny = y + dirs[d][1]
            if 0 <= nx < n and 0 <= ny < n:
                return nx, ny, d
            d = (d + 1) % 4
        return x, y, d

    total = n * n

    g_seen[gx][gy] = True
    c_seen[cx][cy] = True
    g_res.append(grid[gx][gy])
    c_res.append(grid[cx][cy])

    for _ in range(total - 1):
        gx, gy, gd = move(gx, gy, gd)
        cx, cy, cd = move(cx, cy, cd)

        if gx == cx and gy == cy:
            g_res.append(grid[gx][gy])
        else:
            g_res.append(grid[gx][gy])
            c_res.append(grid[cx][cy])

        g_seen[gx][gy] = True
        c_seen[cx][cy] = True

    print(*g_res)
    print(*c_res)

if __name__ == "__main__":
    solve()
```实现的核心是`move`函数，它尝试沿当前方向前进并向左旋转，直到找到有效的网格单元。 这直接编码了“向前走，如果被阻挡，向左转”的规则。 

我们明确地模拟$n^2$步骤，因为每个步骤对应于每个智能体根据问题的遍历规则输入的一个新单元格。 起始位置在循环之前单独初始化。 

平局处理逻辑放置在计算同一时间步长的两次移动之后，以确保保持同步。 这种顺序很重要，因为在两次移动完成之前进行检查会错误地偏向一个代理。

 ## 边缘情况

 一个关键的边缘情况是两个代理同时到达边界并且必须以相同的步长旋转。 对于 2×2 网格，两个智能体都经常在早期步骤中碰壁。 移动功能确保每个智能体独立应用方向校正，因此即使两者在同一步骤中旋转，它们的下一个位置也保持一致。 

另一个微妙的情况是奇数尺寸网格中心的立即碰撞。 在 3×3 网格中，两个步行器都可以汇聚到中心单元上。 同步规则将中心单元分配给 Ginger 的序列，但两个代理仍然按照自己的方向从共享位置继续移动，因此在后续步骤中不会发生分歧。
