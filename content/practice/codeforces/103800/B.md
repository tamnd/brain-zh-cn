 ## 解决方案
 ## 问题理解

 我们得到了一系列怪物的健康值。 在任何事情开始之前，我们可以独立地减少每个值，但我们永远不能将其增加到超过其原始值。 准备工作完成后，我们选择一个起始位置$i$。 从该位置开始，我们一直向左移动到索引 1，并且该段上的值必须形成一个非递增序列。 我们获得的奖励是该所选分段的最终调整值的总和。 

所以这个过程有两个决定。 首先，我们将数组向下整形，尊重上限。 然后我们选择一个以某些结尾的前缀$i$，但该前缀必须以受约束的方式重新排列：当我们向左移动时，值不能上升。 

的约束条件为$n$最多$2 \cdot 10^5$立即排除任何为每个起始位置独立重新计算最佳调整的二次尝试。 任何重建或模拟前缀进程的解决方案$i$会太慢。 

当值已经减少时，会出现微妙的边缘情况。 在这种情况下，不需要减少，答案只是最佳前缀和。 在另一个极端，当价值观像这样振荡时$1, 100, 2, 99, 3$，朴素的局部调整可能看起来局部最优，但向左扩展时会打破全局单调性要求。 

## 方法

 如果我们固定一个起始位置$i$，我们可以问什么是最好的调整序列$[1, i]$看起来像。 由于我们只能减少值，并且需要从右到左的非递增序列，因此最佳策略是从右到左贪婪：将最后一个值设置为尽可能高，然后限制每个先前的值不超过其右侧的值。 

具体来说，如果我们修复$i$，我们将最终值设置为$i$到$a_i$。 然后向左移动，每个位置$j$变成$\min(a_j, b_{j+1})$。 这会产生该端点的最大可行序列，因为每个约束都是局部的并且仅取决于下一个位置。 

蛮力的想法是对每个重复这个构造$i$。 每次我们通过从右到左传播最小值来重新计算整个前缀。 这需要花费$O(n)$每$i$，导致$O(n^2)$，这远远超出了极限$2 \cdot 10^5$。 

关键的观察是重新解释这种贪婪的构造实际上正在计算什么。 对于固定的$i$, 位置处的值$j$成为区间上的最小值$[j, i]$。 所以终点的总分$i$成为以 结尾的所有子数组的最小值之和$i$。 

一旦问题以这种形式出现，它就变成了一个经典的“以每个索引结束的子数组最小值之和”问题。 我们可以使用单调堆栈来跟踪先前较小元素所在的位置，从而增量地维护贡献，而不是从头开始重新计算。 这使我们能够更新每个索引的摊余常数时间的总贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$到$O(n)$| 太慢了 |
 | 基于堆栈的DP |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时维护单调递增的索引堆栈，其中值随着我们深入堆栈而增加。 除此之外，我们计算一个动态值$dp[i]$，定义为恰好在位置处结束的所有子数组的总贡献$i$，其中每个子数组贡献其最小元素。 

1. 维护一个索引栈，使得对应的值从下到上严格递增。 这种结构确保当我们处于位置时$i$，堆栈对先前较小元素所在的位置进行编码。 
2. 对于每个索引$i$，从堆栈中弹出，直到顶部有一个严格小于的值$a[i]$。 经过这个过程，栈顶就变成了之前的索引$p[i]$在哪里$a[p[i]] < a[i]$，如果不存在则为零。 
3.关键的转变是计算$dp[i]$利用所有子数组结尾的事实$i$可以按相对于它们的起始位置进行分组$p[i]$。 之后开始的子数组$p[i]$有$a[i]$作为他们的最低限度，因为他们之间没有任何东西$i$小于$a[i]$。 子数组开始于或之前$p[i]$行为与那些结尾的完全一样$p[i]$。 
4. 这将计算分为两部分：继承自的所有内容$dp[p[i]]$，加上一个由以下人员贡献的新贡献的矩形块$a[i]$。 达到这个新最小值的启动次数是$i - p[i]$，所以附加的贡献是$a[i] \cdot (i - p[i])$。 
5. 跟踪最大值$dp[i]$在所有职位上$i$，因为任何端点都是有效的选择。 

### 为什么它有效

 堆栈保证$p[i]$是最靠近左侧的位置，该位置的值严格小于$a[i]$。 这对所有以以下结尾的子数组进行分区$i$进入那些地方$a[i]$是最小值以及不是最小值的那些。 第二组完全等同于以以下结尾的子数组$p[i]$，因为之前的任何事情$p[i]$已经具有更小或相等的极限值。 这个不变量确保每个子数组都以$i$只计算一次并归因于其正确的最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    stack = []
    dp = [0] * n

    for i in range(n):
        while stack and a[stack[-1]] >= a[i]:
            stack.pop()

        p = stack[-1] if stack else -1
        left_count = i - p

        prev = dp[p] if p != -1 else 0
        dp[i] = prev + a[i] * left_count

        stack.append(i)

    print(max(dp))

if __name__ == "__main__":
    solve()
```该解决方案构建了一个单调堆栈，以便每个元素都知道其先前严格较小的值。 变量`p`正是子数组不再具有的边界`a[i]`作为他们的最低限度。 乘法`a[i] * (i - p)`考虑所有以以下结尾的新子数组`i`新采用的`a[i]`作为他们的最低限度。 

复发为`dp[i]`是使解呈线性的原因。 我们不重新计算所有子数组最小值，而是重用`dp[p]`，它已经包含了早期结构的所有贡献，并用由创建的新块来扩展它`a[i]`。

 ## 边缘情况

 单个元素输入演示了没有先前结构的基本情况。 栈是空的，所以$p = -1$，答案就是元素本身，符合这样的想法：只有一个子数组以该元素结尾。 

当所有元素都相等时，每个新索引都会成为所有先前索引的边界，因为堆栈会不断弹出直到为空。 每个$dp[i]$变成三角累加，公式仍然正确地对每个子数组计数一次。 

在严格递增的数组中，每个元素都成为新的最大值并扩展所有先前的子数组。 堆栈永远不会弹出，所以每个$p[i]$始终是前一个索引，贡献在不断增长的细分中线性累积，无需重置。
