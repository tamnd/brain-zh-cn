 ## 解决方案
 ## 问题理解

 我们有一个两盘平衡，每一方最多可以持有`n`一次相同的物品。 有`m`总共有一件物品，其中只有一件是“坏”的，这意味着它的重量与所有其他物品不同，尽管我们不知道它是更重还是更轻。 其余`m - 1`物品无法区分并且重量相同。 

每次称重都包括将一些物品放在左盘上，一些物品放在右盘上，并限制两侧均不超过容量`n`。 天平告诉我们左侧是更重、更轻还是相等。 任务是确定在最坏情况下所需的最少称重次数，以保证我们能够识别出不良品。`m`候选人。 

关键结构是每次称重都会给出三元结果，但容量限制`n`限制了我们一次可以有意义地比较的项目数量。 这在每次称重的信息增益和我们可以包含的物品数量之间造成了紧张。 

约束条件`m ≤ 10^6`立即排除对所有配置或交互式搜索的任何模拟。 甚至`O(m log m)`如果每个步骤都很昂贵，那么显式模拟子集的结构就会太大。 该问题显然需要对可以区分多少项进行数学表征`k`称重。 

当出现微妙的边缘情况时`m = 1`。 在这种情况下，不需要称重，因为不良品已经被识别出来。 另一个极端情况是当`n = 1`，其中每次权重只能将一项与另一项进行比较，从而有效地将问题简化为分支因子非常有限的线性搜索。 在这种情况下，每次权重平均分割搜索空间的天真假设将失败，因为容量约束阻止了可区分情况下的指数增长。 

## 方法

 思考这个问题的一种强力方法是模拟所有可能的策略，将物品放置在天平上并跟踪结果。 每次权重都有三个结果，因此人们可以想象在所有可能的权重序列上探索三元决策树。 然而，分支因子不能随意使用，因为每次称量都受到天平上可放置的物品数量的限制。 

如果我们尝试模拟每次称重时向左、向右或未使用的所有可能的项目分配，则状态空间在两者中都将变为组合`m`以及称重次数。 即使对于中等`m`，这远远超出了任何可行的计算。 

观察的关键是倒转观点。 我们不是构建策略，而是提出一个能力问题：给定`k`称重时，我们总能区分出多少物品？ 每件物品实际上都有两种可能性，它可能是坏的，也可能是更重或更轻的。 这意味着每个项目对应于两种状态，因此我们要区分`2m`的可能性。 

每次称重最多可放置`n`每一侧都有物品，所以最多`2n`项目参与每次称重。 每次称重都会产生三个结果，因此`k`称重最多能区分`3^k`结果序列。 因此，约束条件是可区分假设的数量不得超过可能结果模式的数量，但我们还必须尊重每个项目每次称重都会消耗容量。 

这种类型的平衡问题的标准简化导致了对数关系：我们需要最小的`k`使得总的可区分容量`k`重量足以覆盖`m`项目，其中每次称量都会贡献一个乘法因子，该因子由我们可以同时测试的项目数量决定。 每次称量的有效生长量为`2n + 1`每个项目组结构的可区分状态，导致经典边界：

 我们寻求最低限度`k`这样`(2n + 1)^k ≥ m`。 

因此答案只是一个对数底`(2n + 1)`的`m`, 四舍五入。

 ## 算法演练

 1. 计算有效分支因子`b = 2n + 1`。 这表示当天平两侧都得到充分利用时，每次称重可以解决多少个不同的结果或项目状态。 期限`2n`来自放置`n`每面的项目，以及`+1`对应于称量设计中的“未使用或中性”贡献。 
2.如果`m = 1`， 返回`0`立即，因为不需要比较。 这避免了对微不足道的情况取对数。 
3. 初始化计数器`k = 0`和一个容量蓄电池`cap = 1`，表示之后可区分的状态数`k`称重。 
4. 虽然`cap < m`, 增量`k`并乘以`cap`经过`b`。 每一次乘法对应于执行一次额外的加权并乘法扩展可区分空间。 
5.一次`cap ≥ m`， 返回`k`作为所需称量的最少次数。 

这种迭代乘法有效的原因是，每次权重在完全饱和两个范围的最佳策略下通过固定的乘法因子细化搜索空间。 

### 为什么它有效

 核心不变量是之后`k`权衡，最优策略最多可以区分`(2n + 1)^k`关于哪个项目不好的不同假设。 每次权重将可能性集最多划分为`2n + 1`一致的结果类别，因为每个项目最多可以在平衡结果中发挥三个作用，但受到泛能力的限制。 因此，任何策略都无法超越指数增长。`k`带底座`2n + 1`，并且存在一种结构，可以通过权重之间平衡分配项目来渐近地实现此界限。 最小的`k`因此是该容量满足或超过的最小整数`m`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

if m == 1:
    print(0)
    sys.exit()

b = 2 * n + 1

cap = 1
k = 0

while cap < m:
    cap *= b
    k += 1

print(k)
```该实施直接反映了指数容量增长的论点。 变量`b`对每个称重扩展因子进行编码。 该循环重复模拟每次额外称重后可以区分多少物品。 停止条件确保我们达到最小`k`使得积累的辨别能力足以满足所有`m`项目。 

这里的一个常见陷阱是忘记`m = 1`情况下，这会错误地返回`1`而不是`0`如果通过对数或循环处理。 另一个微妙的问题是整数大小固定的语言中的整数溢出，但 Python 会自动避免这种情况。

 ## 边缘情况

 当`m = 1`，算法立即返回`0`，因为不需要称重。 任何从容量开始的基于循环的解决方案`1`并且增量会错误地返回`1`，因为它将执行至少一个乘法步骤。 

为了`n = 1`和大`m`，生长因子为`3`，所以循环完全运行`ceil(log_3(m))`次。 例如，与`m = 10`，我们得到容量`1 → 3 → 9 → 27`，所以答案是`3`。 此案例验证了该算法正确处理了可能的最慢增长率，而没有溢出或精度问题。
