 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 n 的数字排列。 您可以将其视为一个有向图，其中每个节点都指向另一个节点，并且由于它是一种排列，因此每个节点也恰好有一个传入边。 该结构分成不相交的有向循环。 

从 1 到 n 的每个数字必须为黑色或白色。 该限制将每个位置 i 与值 p[i] 联系起来：i 的颜色必须与 p[i] 的颜色不同。 换句话说，每条有向边 i → p[i] 都禁止两个端点共享相同的颜色。 

任务是计算在此约束下存在多少个有效着色，以 998244353 为模。 

约束 n ≤ 100000 迫使我们远离任何以 n 为指数的东西。 尝试所有 2^n 种颜色的简单方法是立即不可能的，因为 2^100000 远远超出了任何可行的计算。 即使任何在没有强力修剪的情况下按节点进行分支的方法也会失败。 排列的结构表明我们应该在线性时间内利用循环分解。 

当节点指向自身时，会出现微妙的边缘情况。 如果 p[i] = i，则条件强制 color(i) ≠ color(i)，这是不可能的。 所以任何固定点都会立即使答案为零。 更一般地说，任何奇怪的循环都是不可能的。 

例如，如果n = 1且p = [1]，则唯一的节点必须与其自身不同，这是不可能的，因此答案为0。 

如果 n = 2 且 p = [2, 1]，我们得到长度为 2 的单个循环，这是有效的并产生 2 种着色。 

如果 n = 3 且 p = [2, 3, 1]，则循环长度为 3，当我们返回到起点时，一致性就会中断，使得答案为 0。 

## 方法

 蛮力的想法是将 n 个元素中的每一个分配为黑色或白色，然后检查所有约束。 这枚举了 2^n 次分配，并且每次检查都会扫描所有位置以验证 color[i] 与 color[p[i]] 不同。 成本为 O(n·2^n)，即使对于 n = 30 来说也太大了，因为它已经超过了十亿次操作。 

关键的观察来自于排列的结构：每个元素恰好属于一个循环。 在循环 v1 → v2 → ... → vk → v1 内，约束强制该循环中的相邻元素必须交替颜色。 这将每个周期变成奇偶校验一致性问题。 

如果我们尝试为 v1 分配颜色，则循环的其余部分将被强制执行：v2 必须与 v1 不同，v3 必须与 v2 不同，依此类推。 当我们返回 vk 时，我们必须确保它也不同于 v1。 仅当循环长度为偶数时，此功能才有效，因为当步数为偶数时，交替恰好返回到起始颜色。 对于奇数周期，我们会遇到矛盾。 

对于每个偶数长度的循环，恰好有两个有效的分配：我们可以选择起始节点为黑色或白色，其余的都是强制的。 不同的循环是独立的，因此如果根本没有奇数循环，则着色总数为 2 的偶数循环数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了 |
 | 循环分解| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过将排列分解为循环并独立分析每个循环来解决问题。 

### 步骤

 1. 将所有节点标记为未访问，并准备一个循环计数器和一个指示答案是否已无效的标志。 
2. 从 1 到 n 迭代每个节点 i。 如果 i 已经被访问过，则跳过它，因为它属于先前发现的循环。 
3. 如果 i 未被访问，则开始沿着排列边 i → p[i] 重复行走，直到返回到访问过的节点。 标记本次行走期间访问过的每个节点，并计算本次循环中有多少个节点。

这样做的原因是，排列保证每个节点恰好属于一个循环，因此这种遍历恰好恢复一个连接的组件。 
4. 提取一个循环后，检查其长度。 如果长度是奇数，则立即将答案标记为零并停止进一步推理其他循环，因为单个矛盾会使整个着色无效。 
5. 如果循环长度为偶数，则增加有效循环数。 
6. 处理完所有节点后，将最终答案计算为 2^（偶数周期数）模 998244353。 

### 为什么它有效

 在每个循环内，约束 c[i] ≠ c[p[i]] 强制沿循环边缘严格交替颜色。 这意味着一旦选择了任何单个节点的颜色，循环中的所有其他节点都将被唯一确定。 返回起始节点提出了仅取决于周期长度奇偶性的一致性要求。 偶数循环保持一致性，奇数循环与之相反。 由于循环是不相交的，因此它们的选择独立相乘，每个偶数循环给出 2 个选择的乘积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

n = int(input())
p = [0] + list(map(int, input().split()))

visited = [False] * (n + 1)
even_cycles = 0

for i in range(1, n + 1):
    if visited[i]:
        continue

    cur = i
    cycle_len = 0

    while not visited[cur]:
        visited[cur] = True
        cycle_len += 1
        cur = p[cur]

    if cycle_len % 2 == 1:
        print(0)
        sys.exit(0)

    even_cycles += 1

print(pow(2, even_cycles, MOD))
```遍历在由排列引起的函数图上使用标准循环行走。 每个节点在第一次遇到时就被标记为已访问，因此每个循环只处理一次。 

对周期长度的奇偶校验是关键步骤：任何奇数周期都会强制立即终止并输出为零。 否则，每个偶数周期都会贡献一个乘法因子 2。 

最终的求幂使用Python内置的模函数，它以对数时间运行，并且在模数下是安全的。

## 测试用例```python
import sys, io

MOD = 998244353

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(input())
    p = [0] + list(map(int, input().split()))

    visited = [False] * (n + 1)
    even_cycles = 0

    for i in range(1, n + 1):
        if visited[i]:
            continue
        cur = i
        cycle_len = 0
        while not visited[cur]:
            visited[cur] = True
            cycle_len += 1
            cur = p[cur]
        if cycle_len % 2 == 1:
            return "0"
        even_cycles += 1

    return str(pow(2, even_cycles, MOD))

# sample-like tests
assert solve("2\n2 1\n") == "2"
assert solve("3\n2 3 1\n") == "0"

# minimum size
assert solve("1\n1\n") == "0"

# all fixed points
assert solve("4\n1 2 3 4\n") == "0"

# two disjoint even cycles
assert solve("4\n2 1 4 3\n") == "4"
```| 测试输入| 预期产出 | 它验证了什么 |
 | ---| ---| ---|
 | 2 周期交换 | 2 | 单偶循环|
 | 3周期| 0 | 奇数周期无效|
 | n=1 自环 | 0 | 最小边缘情况|
 | 身份排列| 0 | 多个奇数周期|
 | 两个 2 周期 | 4 | 周期的独立性|

 ## 边缘情况

 诸如 p[i] = i 之类的定点立即创建长度为 1 的循环。在遍历过程中，算法检测到循环长度为 1 并以零终止。 例如，输入`1 1`产生周期长度 1 并被拒绝。 

完全由自映射或奇数循环组成的排列会提前崩溃，因为第一个检测到的奇数循环会立即触发终止。 这可以防止对图的其余部分进行不必要的遍历，同时保持正确性，因为任何单个无效循环都会使整个配置变得不可能。 

具有多个偶数循环的情况，例如`2 1 4 3`，产生两个长度为 2 的独立循环。每个循环贡献一个因子 2，算法正确地将它们相乘以获得 4，反映每个循环的独立二进制选择。
