 ## 解决方案
 ## 问题理解

 我们在平面上得到一组直线，每条直线都由以下形式的方程描述$y = a_i x + b_i$。 任务是计算有多少无序的不同直线对恰好相交于一个点。 

几何上，如果两条线不平行，则它们相交。 对于斜截形式的直线，当它们的斜率相等时，即发生平行。$a_i = a_j$。 该问题明确添加了一个细节：如果两条线相同，意味着斜率和截距都匹配，则不应将它们视为相交对。 

所以输出本质上是对的数量$(i, j)$和$i < j$使得这些线不平行且不相同。 

约束条件是$n \le 5 \cdot 10^3$。 这足够小，以至于$O(n^2)$方法是可以接受的。 任何比二次更糟糕的尝试，例如通过大量计算检查每一对或在嵌套循环内使用排序，都有超时的风险。 线性或$n \log n$方法并不是绝对必要的，但期望有一个干净的组合计数解决方案。 

当存在多条相同的线路时，就会出现一种微妙的情况。 例如，如果输入包含：```
1 1
1 1
1 1
```所有三条线完全重叠。 天真地计算“非平行对”会计算所有 3 对，但它们都不应该被视为相交。 正确答案是 0，因为排除了相同的行。 

另一种边缘情况是许多直线具有相同的斜率但截距不同：```
2 0
2 1
2 2
```这些都是平行但不同的，因此没有一对相交，答案为 0。 

## 方法

 解决问题最直接的方法是考虑每对直线并检查它们是否相交。 两条线$y = a_i x + b_i$和$y = a_j x + b_j$相交当且仅当$a_i \ne a_j$。 如果斜率相等，除非它们相同，否则它们永远不会相交，并且无论如何都会明确排除相同的线。 

这导致了一个简单的强力解决方案：迭代所有对$(i, j)$，检查是否$a_i \ne a_j$，并计算它们。 这是正确的，因为平面中的每对非平行线都恰好相交一次。 

然而，这种方法执行$\frac{n(n-1)}{2}$比较一下，最坏情况下大约有 1250 万次操作$n = 5000$。 这已经是临界点，但如果做得最少的话，在 Python 中仍然可以接受。 尽管如此，我们可以通过从成对检查转向计算排除了多少对来进一步简化。 

我们不是直接计算相交对，而是计算所有对并减去那些不相交的对。 不相交的线对恰好是具有相等斜率的线对。 如果出现斜率值$k$次，那么它就贡献了$\frac{k(k-1)}{2}$不相交的对。 唯一剩下的问题是相同的线，但相同的线已经包含在斜率组内，并且正确减去所有等斜率对可以消除此问题定义所需的平行和重复贡献，因为重复永远不会贡献交叉点。 

因此，问题简化为按斜率对线进行分组并对每组内的组合求和。 

## 算法演练

 1.读取所有线并按斜率分组$a_i$，存储每个斜率出现的次数。 我们只需要计数，因为截距不会影响线是否相交。 
2. 使用以下命令计算所有行中无序对的总数$\frac{n(n-1)}{2}$。 这表示如果没有平行线，则交叉点的最大可能数量。 
3.对于每个具有尺寸的斜坡组$k$，计算不能相交的对的数量，即$\frac{k(k-1)}{2}$。 从总数中减去这个。 
4. 输出结果值。 

步骤 3 有效的原因是任何具有相等斜率的直线永远不会相交，因此我们精确地从完整的线对集中删除所有无效的线对。 

### 为什么它有效

 每对直线都属于两类之一：要么它们的斜率不同，要么它们的斜率相等。 如果斜率不同，则两条线恰好相交一次。 如果斜率相等，则无论截距如何，线都是平行的并且永远不会相交，包括相同线的简并情况。 因此，计算所有对并减去相同斜率对可以精确地隔离相交对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def solve():
    n = int(input())
    cnt = defaultdict(int)

    for _ in range(n):
        a, b = map(int, input().split())
        cnt[a] += 1

    total = n * (n - 1) // 2
    bad = 0

    for k in cnt.values():
        bad += k * (k - 1) // 2

    print(total - bad)

if __name__ == "__main__":
    solve()
```该代码仅按斜率对线进行分组，这已经足够了，因为截距不会影响交叉点的存在。 对的总数计算一次，然后每个斜率组贡献一个校正项，删除所有平行的对。 

一个常见的错误是尝试显式检查所有对，但这是不必要的。 另一个错误是将相同的线与平行线分开处理，但两者自然是在同一斜率分组内处理的。

 ## 边缘情况

 对于单行，例如：```
1
10 20
```斜率图仅包含一个计数为 1 的条目。对的总数为 0，并且没有发生减法，因此结果正确为 0。 

对于完全相同的线路：```
3
1 1
1 1
1 1
```斜率组为{1:3}。 总对数为 3，组内的坏对也为 3，给出最终答案 0。这正确地避免了将重叠线计算为相交。 

对于许多平行但不同的线：```
4
2 0
2 1
2 2
2 3
```斜率组为{2:4}。 总对数为 6，坏对数为 6，最终答案为 0，正确反映平行线之间不存在交点。
