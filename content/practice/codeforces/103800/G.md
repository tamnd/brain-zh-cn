 ## 解决方案
 ## 问题理解

 我们被要求重建一个完整的密码长度$k$覆盖小写英文字母。 密码不是任意的：它必须按字典顺序不递减，这意味着每个字符至少与字母顺序中的前一个字符一样大。 

我们还得到了一个记住的子序列$s$，它必须按顺序出现在最终密码中，但不一定是连续的。 换句话说，我们必须嵌入$s$成一个长度$k$非递减字符串。 

还有一个全局约束：每个字母$a \ldots z$它在最终密码中出现的次数有上限。 这些限制适用于完整构造的字符串，而不仅仅是子序列。 

任务是计算存在多少个满足所有约束并包含的有效完整字符串$s$作为一个子序列，或者报告不可能。 

关键的张力是我们正在计算具有全局容量限制和强制子序列位置的单调字符串。 

限制条件$n, k \le 10^3$26 个字母强烈建议对位置和字母进行动态规划解决方案。 即使在强制执行子序列和上限之前，对所有非递减字符串的暴力破解也已经是天文数字了。 

如果处理不当，一些失败案例会立即出现。 

首先，如果子序列本身违反非递减顺序，则即使在考虑计数之前也不存在解决方案。 例如，如果$s = "ba"$，它永远不能出现在非递减字符串中。 

其次，如果我们贪婪地尝试放置$s$然后在不跟踪转换的情况下独立地填充每个字母的剩余插槽，我们将过量计数，因为填充字母的位置受到最后使用的字符的限制。 

第三，如果仅在最后而不是在构建过程中全局检查字母限制，则 DP 状态将爆炸或计算无效路径。 

## 方法

 蛮力视图是生成每个长度不递减的字符串$k$然后检查是否包含$s$作为子序列并遵守频率限制。 长度不减的字符串的数量$k$超过 26 个字母是$\binom{k+25}{25}$，已经在附近$10^{25}$规模为$k=1000$，所以枚举是不可能的。 即使使用子序列匹配进行修剪也没有任何意义，因为结构仍然是指数的。 

关键的观察结果是，非递减字符串完全由每个字母出现的次数来描述，因为一旦计数固定，就会强制排序：所有 a，然后是所有 b，依此类推。 这将问题转化为分布问题$k$26 个具有上限的桶中的位置，同时还确保子序列$s$可以按照顺序嵌入。 

然而，子序列嵌入在这个固定的全局排序中引入了位置约束。 关键的想法是模拟匹配$s$在按升序构建多组字母时，跟踪有多少个字符$s$当我们“分配”字母时已经匹配了。 

这导致 DP 超过字母以及有多少个字符$s$到目前为止已匹配，并结合剩余仓位的有界背包式分配。 

该构造处理从“a”到“z”的字母。 对于每个字母，我们决定在最终字符串中使用它多少次，同时考虑其限制和剩余容量。 这样做的同时，我们更新了有多少个字符$s$这组相同的字母可以按顺序使用。 

这融合了两个经典的想法：有界背包计数和自动机式子序列匹配。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| 指数| O(k) | 太慢了 |
 | 字母和后续进展的最佳 DP |$O(26 \cdot k \cdot n)$|$O(k \cdot n)$| 已接受 |

 ## 算法演练

 我们根据已处理的字母以及已匹配的子序列的字符数定义 DP 状态。 

让$dp[i][j]$表示处理完第一个之后的路数$i$字母（“a”到$i$-th 个字母），已匹配$j$的字符$s$，同时通过剩余容量隐式地遵守总长度约束。 

由于总长度固定为$k$，在每个阶段，我们都在剩余的字母之间有效地分配剩余的槽位，但 DP 强制执行精确的分配。 

我们还预先计算了一个转换函数，描述使用单个字母块如何影响子序列匹配。 

### 1. 尽早验证子序列的可行性

 我们首先检查是否$s$是非减的。 如果不是，我们立即知道不存在解决方案。 

这是必要的，因为任何有效的最终字符串都是非递减的，因此每个子序列也必须是非递减的。 

### 2. 在字母上构建子序列自动机

 我们对待匹配$s$作为一个指针，每当我们放置正确的下一个字符时就会前进。 

如果我们当前处于位置$j$在$s$，然后我们附加一个字符$c$，然后我们前进$j$如果$s[j] = c$。 

这允许我们在构造最终字符串时更新子序列进度。 

### 3. 字母和匹配前缀上的 DP

 我们处理从“a”到“z”的字母。 在步骤$i$，我们考虑所有的分配方式$x$字母的出现次数$i$， 在哪里$0 \le x \le a_i$，并且总剩余容量允许。 

对于每个可能的当前 DP 状态$dp[i][j]$，我们尝试添加$x$信件副本$i$，模拟有多少个字符$s$在此块期间进行匹配。 

因为所有这些$x$字母相同，匹配行为具有确定性：我们反复尝试匹配$s[j]$反对这封信，并在可能的情况下提前。 

### 4. 容量限制实施

 我们确保所有字母中的字符总数总计为$k$。 任何超出剩余容量的转换都会被丢弃。 

### 5.最终答案提取

 处理完所有 26 个字母后，我们查看具体的州$k$使用了字符并且满足所有子序列约束。 所有有效最终子序列位置的总和就是答案。 

### 为什么它有效

 不变量是在处理完第一个之后$i$字母，每个 DP 状态完全对应于仅使用字母的部分构造$a \ldots i$，通过构造尊重单调性。 由于字母是按排序顺序处理的，因此将来不可能违反非递减顺序。 

由于 DP 在附加字母时显式地模拟按顺序匹配，因此保留了子序列的正确性。 没有状态会合并两个在后续进度或字母使用方面不同的结构，因此计数保持准确。 

每个字母的有限选择确保所有有效的字符分布都被考虑一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, k = map(int, input().split())
    limits = list(map(int, input().split()))
    s = input().strip()

    # check subsequence validity in non-decreasing sense
    for i in range(len(s) - 1):
        if s[i] > s[i + 1]:
            print("NO SOLUTION!")
            return

    # dp[pos in s][used length]
    dp = [[0] * (k + 1) for _ in range(len(s) + 1)]
    dp[0][0] = 1

    for c in range(26):
        ndp = [[0] * (k + 1) for _ in range(len(s) + 1)]
        lim = limits[c]

        for j in range(len(s) + 1):
            for used in range(k + 1):
                if dp[j][used] == 0:
                    continue

                cur = dp[j][used]

                # try using x copies of this letter
                # but we can compress by iterating x
                for x in range(lim + 1):
                    if used + x > k:
                        break

                    nj = j
                    for _ in range(x):
                        if nj < len(s) and s[nj] == chr(ord('a') + c):
                            nj += 1

                    ndp[nj][used + x] = (ndp[nj][used + x] + cur) % MOD

        dp = ndp

    ans = 0
    for j in range(len(s) + 1):
        ans = (ans + dp[j][k]) % MOD

    if ans == 0:
        print("NO SOLUTION!")
    else:
        print(ans)

if __name__ == "__main__":
    solve()
```DP 表跟踪已匹配的子序列的字符数以及到目前为止已放置的总共字符数。 字母的外层循环自动确保单调性，因为我们永远不会重新访问较早的字符。 每个字母的可能计数的内部循环强制执行频率限制。 

该转换模拟添加相同字母的块可能会或可能不会推进子序列指针。 由于字符是按升序处理的，因此子序列匹配始终是一致的。 

最终的聚合仅接受完全使用的状态$k$人物。

该跟踪表明 DP 强制执行严格的顺序：一旦“a”被消耗，只有“b”可以延长匹配，对于“c”也是如此。

 ## 边缘情况

 临界边缘情况是当子序列已经按顺序无效时，例如输入`ba`。 该算法在 DP 开始之前立即检测到这一点。 由于任何非递减结构都不能嵌入递减子序列，因此提前终止是正确的。 

当限制强制精确放置时，会出现另一种边缘情况，例如，当只有一个字母具有非零限制但子序列需要多个不同的字母时。 DP 将在字母转换中传播零状态，从而得出最终的零答案。 

第三种边缘情况是当$k$大于所有极限的总和。 DP 自然地避免无效状态，因为它仅在每个字母的可用容量内转换，因此最终状态无法达到总长度$k$，产生零。 

这些案例证实，每次 DP 转型时都会一致地执行结构和容量限制。
