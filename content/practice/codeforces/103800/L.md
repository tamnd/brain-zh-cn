 ## 解决方案
 ## 问题理解

 我们得到一个由独立因素组成的多项式。 每个因素都贡献一小组可能的力量$x$，当我们将所有因子相乘时，我们得到最终的展开多项式。 任务是回答许多询问特定幂系数的问题$x^a$在这个完全扩展的产品中。 

每个因子都具有贡献三项的形式：常数项、指数项取决于除以二的值的下限，以及指数项等于原始值。 具体来说，其结构是每个因子的行为就像一个带有一些非零单项式的小多项式，最终的表达式是它们的乘积$n$因素。 

输入大小约束$n \le 11$是中心提示。 尽管指数$f_i$可以大到$10^6$，因子的数量很少。 查询的数量可能很大，最多可达$10^4$，因此我们必须预先计算一次所有相关系数，然后在恒定时间内回答每个查询。 

一种天真的解释会尝试直接展开乘积，但这很快就会产生巨大的中间多项式，无论是次数还是项数。 即使在不进行剪枝的情况下显式表示多项式也是不可能的，因为指数可以达到所有多项式的总和$f_i$，并且系数可能由许多组合产生。 

当多种组合产生相同的指数时，会出现微妙的边缘情况。 粗心地枚举项而不组合相同幂的方法将产生重复的条目而不是聚合系数。 另一个陷阱是假设指数很小，因为$n$很小，这是错误的，因为每个$f_i$可以很大并且总度可以达到大约$11 \cdot 10^6$。 

真正的困难是子集的组合爆炸：每个因素都贡献多种选择，我们必须有效地考虑所有组合。 

## 方法

 暴力破解的思想是逐步模拟多项式乘法。 我们从等于 1 的多项式开始，对于每个因子，我们将当前多项式乘以三项多项式。 每次乘法都会采用每个现有项并将其扩展为三个新项，移动指数并添加系数。 

如果当前多项式有$M$项，经过一次乘法后，它大约增长到$3M$，以及之后$n$步骤它变成$3^n$合并重复项之前最坏情况下的术语。 和$n = 11$，这大约是 177,000 个原始术语，这是临界值，但只有在合并非常高效的情况下仍然可以管理。 然而，在实践中，指数冲突严重，如果不仔细分组，实现就会变得缓慢或不正确。 

关键的观察是$n$对于索引上的中间相遇式子集动态规划而不是迭代多项式卷积来说足够小。 每个因素独立地贡献三个选择之一，因此每个最终单项式对应于每个索引选择一个选项。 这意味着每个系数都是通过枚举所有系数来确定的$3^n$选择的组合，但是$3^{11}$大约只有177,000个，计算一次是可行的。 

我们可以预先计算所有子集组合，累积它们生成的指数和系数，并将计数存储在按指数索引的字典或数组中。 经过此预处理后，回答查询就变成了直接查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力卷积 |$O(3^n \cdot n)$|$O(3^n)$| 勉强可行但有风险|
 | 枚举所有选择 |$O(3^n + q)$|$O(3^n)$| 已接受 |

 ## 算法演练

 我们将每个因素解释为在构造过程中贡献三种可能的“移动”：选择常数项、选择中间指数或选择全指数。 所有因素的每次完整选择都会在最终展开式中产生一个单项式。 

然后，我们使用索引的递归或迭代遍历来显式枚举所有此类选择。 

1. 从表示空选择的单个状态开始：当前指数为 0，系数为 1。这表示尚未从任何因子中选择常数项。 
2.工艺因素一一列举。 对于每个因素$i$，我们采用每个现有的部分状态，并以与因子中的三项相对应的三种方式对其进行扩展。 这会创建指数适当增加的新状态。 
3. 扩展状态时，我们通过添加 0 来更新指数，$f_i // 2$， 或者$f_i$。 该系数乘以相应项的系数。 在这个问题中，系数实际上是$-1$对于非常量项，因此每个选择都可能根据做出的非常量选择的数量来翻转符号。 
4. 处理完所有因素后，我们获得将指数值映射到最终系数的贡献集合。 由于许多不同的路径可能产生相同的指数，因此我们将贡献累积到字典中。 
5. 预处理完成后，我们通过返回存储的指数系数来回答每个查询$a$，如果不存在则默认为0。 

这样做的原因是最终多项式中的每个单项式都唯一对应于从每个因子中选择的一项。 该构造将所有此类选择恰好枚举一次，并且累加步骤合并相同的指数，保留正确的系数和。 这在选择状态和扩展多项式中的项之间建立了一对一的映射，确保了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def solve():
    n, q = map(int, input().split())
    f = list(map(int, input().split()))

    # dp will store (exponent -> coefficient)
    dp = defaultdict(int)
    dp[0] = 1

    for fi in f:
        ndp = defaultdict(int)
        a = fi // 2
        b = fi

        for exp, coeff in dp.items():
            # choose constant term (1)
            ndp[exp] += coeff

            # choose -x^(fi//2)
            ndp[exp + a] += -coeff

            # choose -x^(fi)
            ndp[exp + b] += -coeff

        dp = ndp

    for _ in range(q):
        a = int(input())
        print(dp.get(a, 0))

if __name__ == "__main__":
    solve()
```该解决方案维护指数到系数映射的字典。 每次迭代都会用新构建的多项式状态替换当前的多项式状态。 关键的实现细节是我们不能就地更新字典，否则来自同一因素的贡献会相互干扰。 相反，我们为每个因素构建一个新的地图。 

另一个微妙的点是在查询期间处理丢失的指数。 由于并非所有指数都出现在最终展开式中，因此对于不存在的键我们返回 0。 

## 工作示例

 考虑一个具有两个因素的小实例。 

输入：```
2 3
3 4
queries: 0, 1, 5
```我们跟踪 dp 状态。 

### 第一个因子之后 (f1 = 3)

 | 所选术语 | 指数变化| 系数 |
 | ---| ---| ---|
 | 1 | 0 | +1 |
 | -x^1 | 1 | -1 |
 | -x^3 | 3 | -1 |

 所以 dp 变为：

 | 指数 | 系数 |
 | ---| ---|
 | 0 | 1 |
 | 1 | -1 |
 | 3 | -1 |

 ### 第二个因素之后 (f2 = 4)

 我们扩展每个状态：

 | 以前的经验 | 系数| +0 | +2 | +4 |
 | ---| ---| ---| ---| ---|
 | 0 | 1 | 0：+1 | 2：-1 | 4：-1 |
 | 1 | -1 | 1：-1 | 3：+1 | 5：+1 |
 | 3 | -1 | 3：-1 | 5：+1 | 7：+1 |

 合并后：

 | 指数 | 系数 |
 | ---| ---|
 | 0 | 1 |
 | 1 | -1 |
 | 2 | -1 |
 | 3 | 0 |
 | 4 | -1 |
 | 5 | 2 |
 | 7 | 1 |

 这符合多种组合可以取消或加强贡献的想法。 例如，指数 3 由于两个相反的贡献而抵消。 

该迹线显示了为什么累加是必要的：相同的指数来自不同的路径，并且必须求和。

 ## 边缘情况

 一种重要的边缘情况是，多种组合产生具有相反符号的相同指数，从而导致抵消。 例如，经过精心挑选$f_i$，不同的路径可以生成相同的指数，但贡献+1和-1，结果为零。 该算法可以正确处理此问题，因为每次更新都使用字典中的累加累积而不是覆盖。 

另一种情况是所有因素仅贡献常数项。 在这种情况下，多项式仍为 1，并且仅存在指数 0。 该算法初始化 dp[0] = 1 并且永远不会丢失此状态，因为每个因素都保留恒定的选择路径。 

最后一个微妙的情况是大指数。 尽管值可以达到数百万，但该算法永远不会在指数范围内迭代，而只会在可达状态上迭代。 每个状态转换显式地构造新的指数，因此稀疏性自然保留，并且没有大小数组$10^6$是必须的。
