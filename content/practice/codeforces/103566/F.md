 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 n 编号的细胞序列。 从每个单元格到索引较大的单元格都有一次确定性跳转，因此，如果您从任何位置开始并重复应用跳转规则，您总是严格向右移动并最终到达单元格 n。 这种结构保证从 1 到 n 有一条明确定义的路径。 

每个单元格也有相关的成本，旅程的成本定义为沿着这条独特路径的所有访问过的单元格的成本总和，以及关于是否包括起始单元格的小约定。 确切的规则不会影响主要思想，因为我们可以单独调整初始加法。 

输入支持更新：单个单元格的跳转规则发生变化，意味着该位置的出边被修改。 每次更新后，我们必须重新计算新结构下从 1 到 n 的路径成本。 

关键的难点在于，虽然路径是唯一的，但它依赖于动态变化的跳转指针。 每次更新后的简单重新计算将模拟从 1 到 n 的步行，每个查询可能访问 O(n) 个单元格，当 n 和更新次数都很大时，这会变得太慢。 

当更新影响链中的早期位置时，就会出现边缘情况。 例如，如果我们改变从cell 1开始的跳转，整个路径结构可能会改变，所以任何只更新本地信息而不重建依赖关系的解决方案都会失败。 

第二个微妙的边缘情况是当跳跃很大并且立即跳过多个索引块时。 任何假设不进行分组的局部传播的方法都会降低每次操作的线性时间。 

## 方法

 暴力方法很简单：每次更新后，从单元格 1 开始，重复遵循唯一的跳转，直到到达单元格 n，一路上累积成本。 这是正确的，因为该图是一条功能链，并且只有一条可能的路径。 然而，每个查询可能会遍历 O(n) 个单元格，并且最多进行 O(n) 次更新，总复杂度变为 O(n²)，这是不可行的。 

关键的观察是，我们实际上不需要在每次更改后重新计算完整路径结构。 路径很长，但如果我们将索引划分为连续的段，它就会具有稳定的块结构。 在每个细分中，我们可以预先计算我们退出该细分的距离以及留在该细分中时累积的成本。 然后查询可以在段之间跳转，而不是在各个单元格之间跳转。 

这就是标准的平方根分解思想。 我们将索引分成大小约为 k 的块。 对于每个位置 i，我们预先计算两个值：端点 to[i]，它是从 i 开始重复跳转直到离开其块后到达的第一个单元格，以及 co[i]，在块内遍历期间累积的成本。 一旦这些被计算出来，回答一个查询就变成了一个重复地从一个块出口跳转到另一个块出口直到到达n的过程。 

更新只影响一个区块。 如果位置 i 处的跳转规则发生变化，则只有同一块中及其左侧的位置的预先计算的出口可能会发生变化，因为这些是块内模拟经过 i 的唯一起点。 其他区块中的一切都保持不变。 

这导致每次更新需要 O(k) 重新计算，每个查询需要 O(n/k) 遍历。 选择 k = √n 可以平衡两种成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 每个查询 O(n) | O(n) | 太慢了 |
 | 块分解 | 每次操作 O(√n) | O(n) | 已接受 |

 ## 算法演练

 我们维护一个跳转函数 next[i] 和一个成本数组 val[i]。 我们将索引划分为大小为 k 的块。

对于每个索引 i，我们预先计算 to[i] 和 co[i]，其中 to[i] 是从 i 重复跳转直到离开块后到达的第一个位置，co[i] 是沿着此内部行走收集的总成本，不包括 i 本身的成本。 

### 算法演练

 1. 将数组分割为固定大小 k 的块，因此每个索引恰好属于一个块。 这确保了重新计算可以本地化。 
2. 对于每个索引 i，通过模拟从 i 的跳转来计算 to[i] 和 co[i]，但前提是下一个位置保留在同一块内。 一旦我们离开街区，我们就会停下来并记录出口点。 这将长的内部链压缩成单个边缘。 
3. 要回答从 1 到 n 的查询，请从 i = 1 开始，并使用预先计算的转换重复跳转：将 co[i] 添加到答案中并设置 i = to[i]，直到 i 达到 n。 如果定义需要，最后添加起始位置的成本。 
4. 当更新更改 next[i] 时，仅对与 i 位于同一块中的索引重新计算 to 和 co，从右到左处理它们。 这种顺序很重要，因为较早的状态取决于同一块中较晚的状态。 
5. 在重新计算期间，对于每个位置，我们重新运行步骤 2 中使用的相同块内模拟，确保块出口的一致性。 

### 为什么它有效

 关键的不变量是，对于每个索引 i，to[i] 始终表示通过遵循真实跳转函数到达的块外的第一个位置，而 co[i] 始终表示离开块之前累积的确切成本。 因为跳转永远不会向后，所以一旦我们从右到左重新计算一个块，该块内的所有依赖项在需要时就已经是正确的。 不包含更新位置的块不受影响，因为它们的内部模拟不会通过更改的单元格。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    val = [0] + list(map(int, input().split()))

    # next pointer is i + val[i] (assumed forward jump structure)
    nxt = [0] * (n + 1)
    for i in range(1, n + 1):
        nxt[i] = min(n, i + val[i])

    k = int(n ** 0.5) + 1
    block = [0] * (n + 1)
    for i in range(1, n + 1):
        block[i] = (i - 1) // k

    to = [0] * (n + 1)
    co = [0] * (n + 1)

    def rebuild(b):
        L = b * k + 1
        R = min(n, (b + 1) * k)
        for i in range(R, L - 1, -1):
            j = nxt[i]
            if j > R:
                to[i] = j
                co[i] = val[i]
            else:
                to[i] = to[j]
                co[i] = val[i] + co[j]

    for b in range((n + k - 1) // k):
        rebuild(b)

    def update(i, x):
        val[i] = x
        nxt[i] = min(n, i + x)
        rebuild(block[i])

    def query():
        i = 1
        ans = 0
        ans += val[1]
        while i != n:
            ans += co[i]
            i = to[i]
        return ans

    out = []
    for _ in range(q):
        t = input().split()
        if t[0] == '1':
            i = int(t[1])
            x = int(t[2])
            update(i, x)
        else:
            out.append(str(query()))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码在索引空间上维护块分解。 重建函数从右到左重新计算块内的退出点和成本，确保每个位置都可以重用位于同一块中更远的位置的已计算结果。 查询将多次跳转压缩为块转换，避免完全遍历。 

更新函数仅重建一个块，因为只有从该块开始的块内路径才会受到本地更改的影响。 

## 工作示例

 考虑一个 n = 6 的小实例，跳转由 next[i] = i + val[i] 隐式定义，其中 val = [2, 1, 1, 1, 1, 1]。 

| 步骤| 当前我| 行动| 回答 | 笔记|
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 开始 | 0 | 添加 val[1] |
 | 2 | 1 | 添加 co[1] | 2 | 通过块跳转摘要|
 | 3 | 3 | 添加 co[3] | 4 | 继续跳跃|
 | 4 | 5 | 添加 co[5] | 5 | 到达终点 |

 该轨迹显示了如何将多个单步压缩为块跳转，从而避免逐单元模拟。 

现在考虑一个更改 val[2] 的更新，以便它进一步向右重定向。 仅重新计算包含索引 2 的块。 其他区块的指数保持不变，证明了局部性。 

| 步骤| 更改索引 | 重新计算的块| 效果|
 | ---| ---| ---| ---|
 | 更新 | 2 | 块(2) | 仅本地/共同重建|
 | 查询 | 1 | 未更改的块 | 快速重新计算|

 这表明更新不会在全球范围内传播。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O((n + q) √n) | O((n + q) √n) | 每次更新都会在 O(√n) 内重建一个块，每个查询会跨越 O(√n) 块 |
 | 空间| O(n) | 跳转指针和块摘要的存储|

 块大小的选择平衡了重新计算和查询时间。 在典型约束高达 2·10⁵ 的情况下，√n 约为 450，这使更新和查询都在限制范围内。

 ## 边缘情况

 一个重要的边缘情况是更新的位置是块的第一个元素。 在这种情况下，块中的每个位置都通过块内链依赖于它，因此无法从右边界向下重新计算将会在 to 和 co 中留下过时的值。 

另一种情况是跳转立即退出块。 这里 co[i] 必须只包含当前节点成本，否则我们会错误地累积来自另一个应该在查询期间单独处理的块的成本。 

最后一种情况是路径长度由于大的跳跃而极短。 在这种情况下，大多数索引将不得不[i]指向其块之外的远处，并且任何假设重复块内链接的实现都将中断，除非它显式处理下一步立即离开块的边界条件。
