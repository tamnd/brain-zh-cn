 ## 问题理解

 我们有两个整数，代表数量$a$和$b$，我们需要根据大小将它们的比例分为三种咖啡类型之一$a$与$b$。 

该决策不是使用浮点比率，而是使用不等式做出。 如果$a$小于三倍$b$，输出应该是第一杯饮料。 如果$a$大于五倍$b$，输出应该是第二杯饮料。 否则，它属于中间类别。 

因此，输入是一对整数，输出恰好是三个固定字符串之一。 

约束没有明确显示，但问题的结构清楚地表明解决方案必须是每个测试用例的恒定时间。 任何涉及模拟或重复减法的解决方案都是不必要的，并且在典型的竞争性编程限制下会失败，其中$a$和$b$可以很大。 

主要的微妙边缘情况来自边界相等。 例如，当$a = 3b$，第一个条件不满足，因此答案必须移至下一个检查。 同样，当$a = 5b$，第二个条件也不满足，这会强制采用默认情况。 

粗心的实现通常会因使用带有舍入错误的严格浮点除法或混合严格和非严格不等式而失败。 

例如，如果$a = 3$和$b = 1$，正确的输出是中间类别，因为$a$不小于$3b$并且不大于$5b$。 任何使用浮点比较的实现，例如$a / b < 3$由于其他设置中的精度问题，可能会错误地分类。 

## 方法

 最直接的方法是按字面计算比率$a / b$作为浮点数并将其与 3 和 5 进行比较。理论上这在逻辑上是正确的，但它引入了不必要的浮点运算，并且在边界情况下存在精度错误的风险。 它还需要除法，这比整数乘法慢，并且在整数较多的问题中不太干净。 

更稳健的方法通过重写不等式来完全避免除法。 条件$a / b < 3$相当于$a < 3b$，并且类似地$a / b > 5$变成$a > 5b$。 这种转换保留了正确性，同时确保所有计算都保持在整数算术中，这是准确且安全的。 

关键的见解是，我们实际上不需要该比率，只需要它相对于两个阈值的位置。 一旦认识到这一点，问题就会减少到两次比较和一个默认情况。 

蛮力解释之所以有效，是因为决策仅依赖于一次大小比较，但一旦我们观察到将两边都缩放$b$完全消除分裂。

 ## 算法演练

 1. 读取整数$a$和$b$从输入。 它们代表两个数量，其比率决定输出分类。 
2. 检查是否$a < 3b$。 如果成立，立即输出第一杯饮料。 此检查首先放置，因为它捕获值的最小区域。 
3. 如果第一个条件为假，检查是否$a > 5b$。 如果成立，则输出第二杯饮料。 这确定了大比率制度。 
4. 如果两个条件都不满足，则输出中间的饮料。 这恰好对应于间隔$3b \le a \le 5b$。 

### 为什么它有效

 该逻辑将数轴划分为$a$相对于三个不相交的区域$b$：值严格低于$3b$，值严格高于$5b$，以及介于两者之间的值。 每个整数对$(a, b)$恰好属于这些区域之一，因为不平等是互补的并且涵盖了所有可能性而没有重叠。 由于每个区域都映射到唯一的输出，因此该算法无法分配冲突的结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b = map(int, input().split())

if a < 3 * b:
    print("macchiato")
elif a > 5 * b:
    print("latte")
else:
    print("cappuccino")
```该实现直接反映了派生的不平等。 乘法取代了除法，这确保了精确的整数比较而不会出现精度问题。 条件的顺序很重要：首先检查小阈值，然后检查大阈值，其他所有内容都属于中间类别。 

一个微妙的点是，这两种比较都必须严格使用派生的不等式。 将它们更改为$\le$或者$\ge$会将边界值转移到错误的类别中。

 输入：```

```| 步骤| 一个 | 乙| 条件 a < 3b | 条件 a > 5b | 输出|

这一痕迹凸显了严格不平等的重要性。 平等案例正好位于中间区间。

 ## 边缘情况

 当$a = 3b$，第一个不等式失败，因为它是严格的。 例如，输入$a=3, b=1$既不产生第一个条件，也不产生第二个条件，因此输出变成卡布奇诺。 该算法可以正确处理这个问题，因为相等性自然落入 else 分支。 

什么时候$a = 5b$，第二个条件也失败。 例如，$a=10, b=2$产生平等并被正确分类为中间类别。 这证实了上限的严格性是至关重要的。 

什么时候$b$很大，比如$a=0, b=10^9$, 乘法$3b$和$5b$仍然适合 64 位整数范围，因此 Python 中不会出现溢出问题。 第一个条件立即触发，按预期生产玛奇朵。 

这些案例证实，该算法在边界和极端输入上表现一致，无需特殊处理。
