 ## 解决方案
 ## 问题理解

 我们得到 N 个参与者的集合，每个参与者由三条信息描述：潜在的“朋友参考”Fi、就绪标志 Pi 和时间戳 Ti。 该过程定义了一个派生数组 cnt，最初全为零，用于计算有多少其他参与者成功“验证”每个参与者为合格的朋友。 

对于每个参与者 i，我们尝试确定 i 是否可以充当其他人的朋友 Fi 的有效推荐者。 如果 Fi 非零，则参与者 i 被视为可能对增加 Fi 计数做出贡献的人。 但是，只有当 i 满足两个独立条件时才允许这样做：Pi 必须为正，这意味着我已经通过了一定的资格检查，而 Ti 必须严格大于 TFi，这意味着我比他们所指的朋友注册得晚。 

如果所有条件成立，我们将 cnt[Fi] 加一。 处理完所有参与者后，我们再次检查每个参与者 i，并输出 cnt[i] 至少为 2 的那些索引。输出必须按索引升序排列。 

输入大小 N 是影响解决方案的关键因素。 由于我们在直接扫描中仅对每个参与者执行恒定量的工作，因此需要 O(N) 或 O(N log N) 方法。 任何涉及对参与者对的嵌套检查都会导致 O(N²)，当 N 达到典型的 Codeforces 约束（如 2⋅10⁵）时，这将变得不可行。 

当 Fi 为零时，会出现微妙的边缘情况。 在这种情况下，参与者不会对任何计数做出贡献，如果不仔细处理，尝试索引 cnt[0] 将是无效的。 当时间戳相等时，会出现另一种边缘情况：由于条件严格为 Ti > TFi，相等必定失败，并且使用非严格比较的幼稚实现将错误地增加计数。 

最后一个陷阱是忘记多个参与者可以独立为同一个 Fi 做出贡献。 这意味着 cnt 不是布尔数组，而是频率累加器，并且预计会有重复的贡献。 

## 方法

 暴力解释将直接模拟定义：对于每个参与者 i，我们扫描所有其他参与者 j 并检查 j 是否可以对 cnt[i] 做出贡献。 这导致检查每对涉及 Fi、Pi 和 Ti 的条件，从而有效地使过程二次化。 虽然逻辑上是正确的，但在最坏的情况下这会导致大约 N2 的比较，当 N 很大时，比较速度太慢。 

关键的观察是每个参与者最多向单个目标索引 Fi 贡献一个信息单元，并且该贡献是独立于所有其他参与者确定的。 对 cnt 的不同更新之间不存在依赖性，这意味着我们可以在单个线性传递中计算所有贡献。 

我们没有为每个参与者搜索有效的贡献者，而是直接迭代所有参与者一次，并且对于每个 i，我们检查它是否有资格作为贡献者。 如果是，我们增加 cnt[Fi]。 这将问题从“查询传入的有效关系”转变为“处理传出的有效边”，从而消除了任何嵌套迭代的需要。 

构建 cnt 后，第二次线性扫描提取 cnt[i] ≥ 2 的所有索引。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N²) | O(N) | 太慢了 |
 | 最佳 | O(N) | O(N) | 已接受 |

 ## 算法演练

1. 读取所有数组 F、P 和 T。这些数组定义了每个参与者对另一个参与者的定向潜在贡献。 该结构是静态的，因此不需要动态更新。 
2. 将 cnt 初始化为大小为 N、全为零的数组。 这将累积每个参与者收到的有效贡献者数量。 
3. 从 1 到 N 迭代每个参与者 i。对于每个 i，我们决定它是否有助于其他人的计数。 
4. 如果 Fi 等于 0，则立即跳过 i。 这避免了无效索引并反映出不存在贡献目标。 
5. 检查Pi 是否为正。 如果 Pi 为零，我就无法贡献，所以我们跳过它。 此条件会过滤掉未通过资格步骤的参与者。 
6. 检查Ti 是否严格大于TFi。 如果没有，请跳过 i。 这强制了排序约束，即只有后来的参与者才能验证前面的参与者。 
7. 如果所有检查均通过，则将 cnt[Fi] 加 1。 这记录了Fi已经获得了一名有效的支持参与者。 
8. 处理完所有参与者后，再次从 1 到 N 迭代 i，并收集 cnt[i] 至少为 2 的所有索引。这些是收到足够有效确认的参与者。 
9. 按升序输出收集到的索引，这是自动保证的，因为我们按升序遍历 i。 

正确性取决于步骤 7 之后的不变量，cnt[x] 等于参与者 i 的数量，使得 i 在所有约束下只对 x 贡献一次。 由于每个参与者都是独立处理的，并且最多贡献一个增量，因此不会出现重复计算或遗漏的情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    F = [0] + list(map(int, input().split()))
    P = [0] + list(map(int, input().split()))
    T = [0] + list(map(int, input().split()))

    cnt = [0] * (n + 1)

    for i in range(1, n + 1):
        f = F[i]
        if f == 0:
            continue
        if P[i] <= 0:
            continue
        if T[i] <= T[f]:
            continue
        cnt[f] += 1

    res = []
    for i in range(1, n + 1):
        if cnt[i] >= 2:
            res.append(i)

    print(*res)

if __name__ == "__main__":
    solve()
```该实现完全反映了算法：第一遍构建频率计数，第二遍提取有效索引。 关键细节是严格的不平等检查`T[i] <= T[f]`，它的编写方式可以正确拒绝相等和较小的时间戳。 

索引是通过将数组转换为从 1 开始的形式来处理的，这避免了有效参与者索引和用于 Fi 的零标记之间的混淆。

 ## 边缘情况

 一种边缘情况是所有参与者的 Fi 都为零。 在这种情况下，循环永远不会触发任何 cnt 增量，并且最终输出为空。 该算法自然地处理这个问题，因为第一个条件`if f == 0`防止无效索引并跳过所有更新。 

当看起来有效的对的时间戳相等时，会出现另一种边缘情况。 例如，如果 Fi = j 且 Ti = Tj，则由于严格不等式，该条件失败。 代码明确使用`T[i] <= T[f]`拒绝平等，确保不会发生意外的增量。
