 ## 解决方案
 ## 问题理解

 我们得到了一组火柴棍，其中每根火柴棍都有一个整数长度。 相同的长度可以出现多次，重要的只是每个长度出现多少次。 我们想要用这些火柴棒建造两种可能类型的塔，并且对于每种类型，我们想要计算在建造规则下可实现的最佳塔高度。 

在第一种类型中，塔仅由正方形组成。 边长为 i 的正方形是由相同长度 i 的火柴棒组成的，并且垂直堆叠正方形会以固定模式消耗额外的火柴棒。 如果我们固定长度 i，我们只使用该长度的火柴棍，并且我们想知道可以构建多少个完整的正方形级别。 在第一个关卡之后，每个关卡都会消耗三根长度为 i 的火柴棒，因此如果构建 Fi 个关卡，则所需的火柴棒总数为 1 + 3Fi。 可用供应量为 cnt[i]，因此 Fi 是满足 1 + 3Fi ≤ cnt[i] 的最大整数。 对答案的贡献是 Fi·i，因为每个级别贡献高度 i。 

在第二种类型中，塔由矩形层组成。 每一层需要两个长度为 h 的垂直边和一个长度为 w 的水平顶部。 关键是垂直和水平分量使用不同的长度。 如果我们固定垂直长度h，则垂直棒的数量为cnt[h]，因此我们最多可以支持floor(cnt[h] / 2)层。 对于长度为 w 的水平顶部，第一层之后的每一层都需要一个顶部，因此我们最多可以构建 cnt[w] − 1 层。 因此，完整层数是这两个量中的最小值，并且高度贡献取决于 h 和 w。 

输入大小最多允许 2 · 10^5 个不同的长度，因此任何直接尝试所有长度对的解决方案都会太慢。 对所有 (h, w) 对进行二次检查将需要 4 · 10^10 次运算，这在典型限制下是不可能的。 我们需要利用结构来了解固定 h 下 w 的最佳行为。 

当垂直和水平方向使用相同的长度时，会出现微妙的边缘情况。 在这种情况下，这两个约束相互作用，我们必须避免重复计算或错误地假设独立供应。 另一个问题是当 cnt[i] 非常小时，尤其是 1 或 2 时，如果不小心处理，涉及除法和减法的公式可能会产生零或负值。 

## 方法

 方形案例的直接方法很简单。 对于每个长度 i，我们从 cnt[i] 计算 Fi，因为所有决策都是 i 的局部决策。 这是最佳的，因为正方形不会混合长度。 

对于矩形，一种简单的方法是尝试每一对 (h, w)，计算 min(cnt[h] / 2, cnt[w] − 1)，并采用高度加权的最佳值。 这可以正确地模拟所有结构，但由于 O(A^2) 对而速度太慢。 

关键的观察是，对于固定的 h，限制因素通常是垂直容量 cnt[h] / 2。 w 唯一重要的时间是当 cnt[w] − 1 小于该值时，这意味着水平棒成为瓶颈。 在这种情况下，我们希望 w 能够最大化 cnt[w]，因为增加 cnt[w] 只能改善或维持最小值。 

这大大减少了对 w 的搜索。 我们不考虑所有 w，而是主要关心最常见的长度，因为它最大化 cnt[w] 并且最不可能成为瓶颈。 我们还需要考虑第二频繁的长度，因为如果将最频繁的长度用作 h，我们就无法在不违反独立性的情况下将其重新用作 w，因此我们需要一个替代候选者。 

因此，对于每个 h，我们仅评估恒定数量的候选 w 值：全局最频繁的长度，并且在 h 等于该值的特殊情况下，是第二频繁的长度。 这将问题简化为 A 上的线性时间。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有对 | O(A^2) | O(A^2) | O(A) | 太慢了|
 | 基于频率的优化 | O(A) | O(A) | 已接受 |

 ## 算法演练

 我们首先构建一个频率数组 cnt，其中 cnt[x] 存储有多少根长度为 x 的木棒。 这使得我们可以恒定的时间访问塔中的任何角色可以使用多少根棍子。 

接下来，我们通过迭代所有长度 i 来计算最佳方塔贡献。 对于每个 i，我们将 Fi 计算为 cnt[i] // 3，因为在第一根棍子之后，每个附加级别都会花费三根棍子。 我们将高度计算为 Fi * i 并跟踪最大值。 

然后我们识别 cnt 中最常见的两个长度。 最频繁的长度是水平顶部的最佳候选，因为它最大化 cnt[w]，从而最大化 cnt[w] − 1。当最频繁的长度用作垂直顶部时，需要第二频繁的长度作为后备，因为我们无法以最佳方式重用同一个池。 

之后，我们迭代所有可能的垂直长度 h。 对于每个 h，我们将垂直容量计算为 cnt[h] // 2。对于水平候选 w，我们仅考虑最佳频率选择。 对于每个候选 w，我们将水平容量计算为 cnt[w] − 1。级别数是这两个值中的最小值，然后我们计算所得的高度贡献。 

我们更新所有有效 (h, w) 选项的答案。 

### 为什么它有效

 对于固定的垂直长度 h，塔高度由层数决定，层数受垂直和水平供应的限制。 垂直项仅取决于cnt[h]，而水平项仅取决于cnt[w]。 由于目标随着级别数量的增加而增加，因此我们希望最大化这两个值的最小值。 

如果 cnt[h] / 2 很小，那么任何满足 cnt[w] − 1 ≥ cnt[h] / 2 的 w 都同样好，所以只有可行性很重要，而不是精确的选择。 如果 cnt[h] / 2 很大，那么我们就会受到 cnt[w] 的瓶颈，而最大化 cnt[w] 是最优的。 因此，只有最大和第二大频率才重要，具体取决于它们是否与 h 冲突。 

这减少了 w 的有效搜索空间，而不会丢失任何最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))

    MAXA = 200000
    cnt = [0] * (MAXA + 1)

    for x in arr:
        cnt[x] += 1

    ans = 0

    for i in range(MAXA + 1):
        f = cnt[i] // 3
        ans = max(ans, f * i)

    first = 0
    second = 0

    for i in range(MAXA + 1):
        if cnt[i] > cnt[first]:
            second = first
            first = i
        elif cnt[i] > cnt[second]:
            second = i

    best_ws = {first, second}

    for h in range(MAXA + 1):
        if cnt[h] == 0:
            continue
        vert = cnt[h] // 2

        for w in best_ws:
            if cnt[w] == 0:
                continue
            horiz = cnt[w] - 1
            levels = min(vert, horiz)
            if levels > 0:
                ans = max(ans, levels * h)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先构建频率数组，这很重要，因为两种构建仅依赖于单位长度的计数。 平方计算是直接的：每组三根额外的木棍贡献一个级别，因此整数除以 3 给出完整级别的数量。 

选择两个最常见的长度是一种优化，它取代了对所有可能宽度的二次搜索。 我们只需要最大化 cnt[w] 的候选者，因为在 cnt[h] 很大的情况下，任何次优的 w 都无法改善最小值。 

最后的循环将每个 h 评估为垂直结构，并将其与仅两个可能的 w 候选者组合。 垂直和水平容量的最小值决定了层数，乘以 h 得出高度。

 ## 边缘情况

 一种重要的边缘情况是 cnt[h] 为 1 时。在这种情况下，cnt[h] // 2 为零，因此无论 w 如何，垂直容量都会阻止任何矩形构造。 该算法自然地处理这个问题，因为级别变为零并且没有任何贡献。 

另一种情况是最常见的长度也用作 h。 那么我们就不能以不受约束的方式依赖 cnt[w] = cnt[h] 作为水平顶部。 第二个最常见的候选者确保我们仍然考虑有效的替代方案。 例如，如果所有棍子都是一种长度，则第二个候选者实际上未被使用，并且矩形正确地缩小为两个角色共享同一池的约束形式。 

最后的边缘情况是当 cnt[w] = 1 时。然后 cnt[w] − 1 = 0，这意味着 w 甚至不能支持单个级别。 这些候选者会被安全地忽略，因为级别变为零并且不会影响最大值。
