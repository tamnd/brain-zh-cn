 ## 解决方案
 ## 问题理解

 我们提供了几种可用的解决方案，每种解决方案都以盐浓度为特征。 我们可以从每个溶液中取出一些整数克，每个溶液的上限为 10⁴ 克。 混合后，总质量为所选质量之和，总盐量为按浓度加权求和。 

目标是构建至少 100 克浓度恰好为 Q% 的新混合物。 换句话说，如果我们用 xi 表示从浓度为 Ti 的溶液 i 中取出的质量，则最终比例必须满足

 （总和 xi * Ti）/（总和 xi）= Q。 

这可以通过两边乘以分母以更代数的形式重写：

 总和 xi * (Ti − Q) = 0。 

因此，每个解贡献一个系数 ai = Ti − Q，并且我们需要一个非负整数向量 x，使得 ai 的加权和为零，同时还满足 0 ≤ xi ≤ 10⁴ 且总质量至少为 100。 

这些约束最多允许 10⁵ 个解，因此任何针对解对的二次策略都会立即受到怀疑。 需要线性或近线性结构。 

一个微妙的限制是，即使存在有效的线性组合，它也必须产生至少 100 克的总质量。 像等幅系数的单个抵消这样的简并解可能满足方程但不满足质量要求。 

当根本没有解决方案时，就会出现边缘情况。 例如，如果所有 Ti 都严格大于 Q，则每个 ai 都是正数，并且任何非零非负组合的总和不能为零。 类似地，如果所有 Ti 都严格小于 Q，也会发生同样的不可能。 另一个特殊情况是当某个 Ti 等于 Q 时； 那么仅使用该解决方案就存在一个简单的构造。 

## 方法

 蛮力的想法是选择解的子集并将整数质量分配给它们，检查所得的加权平均值是否等于 Q。这很快就变成了求解 n 个变量中有界整数线性方程的问题。 即使忽略边界，尝试子集组合也已经导致指数行为，并且添加系数上限使得直接搜索不可行。 

将条件重写为 sum xi(Ti − Q) = 0 将问题转化为平衡正贡献和负贡献。 每个 Ti > Q 都充当正权重，每个 Ti < Q 充当负权重。 有效的解决方案必须混合两边，除非某个 Ti 已经等于 Q。 

关键的简化在于，只有两个值足以构造零和：一个来自正侧，一个来自负侧。 如果我们选择索引 i 和 j 且 ai > 0 且 aj < 0，我们需要

 xi * ai = xj * (−aj)。 

令 g 为 gcd(ai, −aj)。 最小整数解是

 xi = (−aj / g), xj = (ai / g),

 并且该对的任何缩放版本仍然有效。 

xi ≤ 10⁴ 和 xj ≤ 10⁴ 的约束限制了我们可以缩放该对的程度。 在所有可能的对中，我们希望有一个允许足够大的比例因子，以便最终的总质量至少为 100。 

尝试所有对的时间复杂度为 O(n²)，因此我们转而利用结构：只有 ai 的大小才影响可行性。 极值选择往往是最好的，因为小系数允许在达到 10⁴ 上限之前进行更大的缩放。 

因此，我们将搜索减少为选择一个正侧索引和一个负侧索引，以最小化 |Ti − Q| 在每一侧。 这通常会最大化可用的缩放因子，同时保持比率简单。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有任务| 指数| O(n) | 太慢了 |
 | 结对构造与贪婪选择 | O(n) | O(n) | 已接受 |

 ## 算法演练

我们将每个浓度转换为 ai = Ti − Q 并将指数分为三组：正、负和零。 

如果存在任何 ai = 0 的指数，我们可以通过取 100 克该溶液并从所有其他溶液中取 0 来立即构造一个有效答案，因为它已经与目标浓度相匹配。 

如果所有 ai 均非零且符号相同，则任何非负组合都不能产生零，答案是不可能的。 

否则，我们必须将一个正值和一个负值结合起来。 

1.我们选择一个候选正索引和一个候选负索引。 一个自然的选择是选择最小的正 ai 和最小的负 ai，因为它们给出了最简单的比率，并且在达到 10⁴ 限制之前允许更大的缩放。 
2. 对于所选的对，我们使用 gcd 计算减少的比率。 令 ai = p > 0 且 −aj = q > 0。我们计算 g = gcd(p, q)，然后定义基本量 x0 = q / g 和 y0 = p / g。 
3. 我们确定在尊重边界的情况下可以缩放该对多少次。 缩放因子 k 受到 xi ≤ 10⁴ 和 yi ≤ 10⁴ 两个约束的限制，因此

 kmax = min(10⁴ / x0, 10⁴ / y0)。 

1. 我们检查使用最大可能的缩放比例是否已经产生足够的总质量，即 kmax * (x0 + y0) ≥ 100。如果不是，则该对不能满足要求。 
2. 如果足够，我们输出 xi = kmax * x0 和 yi = kmax * y0，所有其他值为零。 

如果没有一对有效，答案是不可能的。 

正确性依赖于这样一个事实：任何有效的解决方案都必须平衡正面和负面的贡献。 任何这样的解决方案都可以分解为成对的抵消，并且至少一个抵消对本身必须具有足够的可扩展性，以在约束下达到所需的总质量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve():
    n, Q = map(int, input().split())
    T = list(map(int, input().split()))

    pos = []
    neg = []
    zero = []

    for i, t in enumerate(T):
        a = t - Q
        if a > 0:
            pos.append((a, i))
        elif a < 0:
            neg.append((a, i))
        else:
            zero.append(i)

    # if exact match exists
    if zero:
        res = [0] * n
        res[zero[0]] = 100
        print(*res)
        return

    if not pos or not neg:
        print(-1)
        return

    # pick minimal absolute representatives
    pos.sort()
    neg.sort(key=lambda x: x[0], reverse=True)  # closest to zero negative

    candidates = []
    candidates.append((pos[0][0], neg[0][0]))

    # try a few more extreme combinations (safety)
    for i in range(min(3, len(pos))):
        for j in range(min(3, len(neg))):
            candidates.append((pos[i][0], neg[j][0]))

    best = None

    for p, nval in candidates:
        a = p
        b = -nval
        g = gcd(a, b)
        x0 = b // g
        y0 = a // g

        kmax = min(10000 // x0, 10000 // y0)
        if kmax <= 0:
            continue

        total = kmax * (x0 + y0)
        if total >= 100:
            best = (x0, y0, kmax, p, nval)
            break

    if best is None:
        print(-1)
        return

    x0, y0, kmax, p, nval = best

    res = [0] * n
    # find indices
    for i, t in enumerate(T):
        if t - Q == p and x0 > 0:
            res[i] = kmax * x0
            x0 = 0
        elif t - Q == nval and y0 > 0:
            res[i] = kmax * y0
            y0 = 0

    print(*res)

if __name__ == "__main__":
    solve()
```解决方案首先将浓度条件转换为线性方程。 然后，它将索引分为正、负和与 Q 的零偏差。零情况会立即处理，因为它直接满足 100 克的要求。 

对于非零情况，该算法构造候选平衡对。 对于每一对，它使用 gcd 计算减少的整数比率，然后检查在达到每种成分 10⁴ 限制之前可以缩放的程度。 仅接受总重量至少为 100 克的配对。 

最终分配将计算出的金额分配回所选索引。

 ## 边缘情况

 当解决方案包含完全等于 Q 的 Ti 时，算法会立即返回仅使用该成分的有效构造。 这样就避免了不必要的平衡逻辑，保证了可行性。 

当所有偏差具有相同的符号时，每个组合都保留在 Q 的一侧，因此线性方程不能达到零； 该算法正确返回-1。 

当平衡对存在但 gcd 约化系数很大时，缩放会受到 10⁴ 上限的阻碍。 该算法通过 kmax 显式检查这一点，确保即使方程可无界解，也不会构造无效的赋值。
