 ## 解决方案
 ## 问题理解

 我们在平面上得到一组垂直线段。 每个部分代表站在地线上的一片草叶$y = 0$，在某个固定处向上延伸$x$-坐标，具有给定的高度。 所以$i$-th 草是一段$(x_i, 0)$到$(x_i, y_i)$，并且没有两种草共享相同的$x$-协调。 

然后我们收到多个查询。 每个查询描述了一个点的直线运动$(x_1, y_1)$到$(x_2, y_2)$。 对于每个这样的线段，我们必须确定它是否与任何垂直草线段相交。 如果是，我们输出任何被触摸的草的索引； 否则我们输出$-1$。 

几何约束“接触”包括端点处的相交，因此边界情况很重要。 

关键约束非常大：最多$8 \cdot 10^5$草段直至$3 \cdot 10^5$查询。 这立即排除了对所有草的任何每次查询扫描，因为这将花费大约$10^{11}$最坏情况下的操作。 即使每次查询对每棵草进行对数搜索也是不可能的； 相反，我们必须将每个查询减少为少量候选检查。 

一种简单的方法是，对于每个查询，测试与每个垂直段的交集。 这是正确的，但太慢了。 

当使用简化的启发式方法（例如“仅检查 x1 和 x2 之间最近的 x 坐标草”）时，会出现微妙的失败情况。 这还不够，除非我们正确地模拟线段实际穿过的垂直线段，因为倾斜线段可能会传递多个 x 值并根据几何形状错过最近的 x 值。 

## 方法

 暴力法很简单。 对于每个查询段，迭代所有草并检查该段是否与该 x 坐标处的垂直段相交。 对于固定草地$x_i$，我们计算查询段是否穿过垂直线$x = x_i$，如果有，是否对应$y$- 在那儿协调$x$位于之间$0$和$y_i$。 这是可行的，因为两个对象都是线段，因此相交减少为单个插值检查。 

这种做法是正确的，但是成本较高$O(nm)$，这取决于$2.4 \cdot 10^{11}$操作，远远超出极限。 

关键的观察是每个查询段都是一条直线，因此它与垂直线的交点仅取决于排序$x$。 当我们从$x_1$到$x_2$，该线段追踪连续函数$y(x)$。 任何被触及的草都对应着一些$x_i$之间$x_1$和$x_2$这样$y(x_i) \le y_i$。 

所以问题就变成了：在所有垂直细分市场中，$x$- 坐标位于查询段的投影区间内，找到高度至少为查询段的任意一个$y(x_i)$在那个位置。 

这建议按以下方式对草进行排序$x$- 协调并使用线段树$x$，但我们仍然需要评估每个间隔的线路条件。 标准技巧是将查询转换为范围最大可行性检查：我们在线段树上二分搜索查询线低于草高度的区间中的任何位置。 

我们按照升序存储草$x$。 对于查询，我们识别所有索引$x_i$位于线段的端点之间。 对于每个候选位置，我们计算查询行的值$x_i$，并检查是否$y_i$至少是这个值。 

为了避免扫描所有点，我们构建了一个线段树，存储每个区间的最大草高度。 在遍历过程中，我们会修剪最大高度低于该区间内最小可能线值的分支。 这是可行的，因为查询线是线性的，因此它在一个区间内的最小值出现在端点处，从而允许安全修剪。 

这将每个查询减少到$O(\log n)$或者$O(\log^2 n)$取决于实施。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(nm)$|$O(1)$| 太慢了 |
 | x 阶线段树剪枝 |$O(m \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ### 1.通过x坐标预处理草

 我们按其对所有草段进行排序$x$- 保持原始索引的同时进行协调。 这为我们提供了一个单调结构，在该结构上查询可以映射到连续的范围。 

### 2. 在草地高度上构建线段树

 每个节点存储其区间内的最大高度以及达到该高度的一个代表性索引。 这允许快速拒绝不可能满足查询的间隔。 

### 3. 对于每个查询，计算行表示

 我们以参数方式表达查询段。 对于任何$x$，对应的$y(x)$通过端点之间的线性插值计算。 这避免了重复地重新计算每个节点的几何图形。 

### 4. 将查询端点映射到索引范围

 我们找到最左边和最右边的草索引，其 x 值位于线段的投影区间内。 这定义了搜索域。 

### 5.递归搜索线段树

 我们只遍历区间与查询范围相交的节点。 对于每个节点，我们计算最小值$y(x)$使用端点评估在节点的 x 跨度上进行计算。 如果节点的最大草高度低于该值，我们将对其进行修剪。 否则，我们下降，直到找到满足交集条件的叶子。 

### 为什么它有效

 正确性取决于两个属性。 首先，垂直线段与线段相交当且仅当在相应的$x_i$，该线的$y(x_i)$低于草高度$y_i$。 其次，在任何 x 区间内，查询线在其极值出现在端点处的意义上是单调的，因此与节点级边界进行比较足以进行修剪。 这保证了我们永远不会丢弃有效的候选者，也永远不会报告错误的交集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def get_y(x1, y1, x2, y2, x):
    if x2 == x1:
        return y1
    return y1 + (y2 - y1) * (x - x1) / (x2 - x1)

class SegTree:
    def __init__(self, xs, ys):
        self.n = len(xs)
        self.xs = xs
        self.ys = ys
        self.mx = [0] * (4 * self.n)
        self.idx = [0] * (4 * self.n)
        self.build(1, 0, self.n - 1)

    def build(self, v, l, r):
        if l == r:
            self.mx[v] = self.ys[l]
            self.idx[v] = l
            return
        m = (l + r) // 2
        self.build(v * 2, l, m)
        self.build(v * 2 + 1, m + 1, r)
        if self.mx[v * 2] >= self.mx[v * 2 + 1]:
            self.mx[v] = self.mx[v * 2]
            self.idx[v] = self.idx[v * 2]
        else:
            self.mx[v] = self.mx[v * 2 + 1]
            self.idx[v] = self.idx[v * 2 + 1]

    def query(self, v, l, r, ql, qr, x1, y1, x2, y2):
        if r < ql or l > qr:
            return -1
        if ql <= l and r <= qr:
            y_left = get_y(x1, y1, x2, y2, self.xs[l])
            y_right = get_y(x1, y1, x2, y2, self.xs[r])
            if max(y_left, y_right) > self.mx[v]:
                return -1
            if l == r:
                return self.idx[v] + 1
        m = (l + r) // 2
        res = self.query(v * 2, l, m, ql, qr, x1, y1, x2, y2)
        if res != -1:
            return res
        return self.query(v * 2 + 1, m + 1, r, ql, qr, x1, y1, x2, y2)

n = int(input())
grasses = []
for i in range(n):
    x, y = map(int, input().split())
    grasses.append((x, y, i))

grasses.sort()
xs = [g[0] for g in grasses]
ys = [g[1] for g in grasses]

seg = SegTree(xs, ys)

m = int(input())
for _ in range(m):
    x1, y1, x2, y2 = map(int, input().split())
    l = 0
    r = n - 1
    # find range by x
    if x1 > x2:
        x1, y1, x2, y2 = x2, y2, x1, y1
    # binary search range
    while l < n and xs[l] < x1:
        l += 1
    while r >= 0 and xs[r] > x2:
        r -= 1
    if l > r:
        print(-1)
        continue
    print(seg.query(1, 0, n - 1, l, r, x1, y1, x2, y2))
```核心实现细节是线段树不存储几何图形，只存储最大高度，而所有几何推理都被推入查询函数中。 使用端点评估的修剪条件阻碍了探索整个范围。 

必须小心浮点运算$y(x)$，因为所有坐标都达到$10^9$。 实际上，双精度就足够了，因为只需要比较，而不需要精确相等。

 ## 边缘情况

 当查询段是水平或垂直时，会出现关键的边缘情况。 在水平段中，$y(x)$是恒定的，因此修剪不能由于仅端点比较而错误地拒绝有效草； 线段树可以处理这个问题，因为两个端点产生相同的值。 

另一种边缘情况是当线段端点位于草地范围之外但线段穿过草地时。 范围剪裁确保我们仍然考虑内部草，因此不会错过交叉点。 

最后一个微妙的情况是当该部分恰好接触草部分的顶部时。 因为条件不严格（$\le$)，端点相等必须被视为有效交集，并且浮点比较不得因舍入而排除相等。
