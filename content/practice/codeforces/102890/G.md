 ## 解决方案
 ## 问题理解

 我们得到一系列位置，每个位置代表一个具有立即奖励的“状态”。 从各个位置$i$，我们可以向前跳跃两个边界之间的距离$a_i$和$b_i$，降落在某个位置$i + j$。 目标是计算对于每个位置，如果我们从该位置开始并反复向前跳跃直到我们无法再移动，则可获得的最大总奖励。 

位置处的值$i$，称之为$f(i)$，被定义为奖励$i$加上其跳跃范围内所有有效下一个位置中的最佳可达值。 依赖关系总是向前指向的，因此计算本质上是一个向后动态规划问题。 

预期问题中的输入大小足够大，以至于$O(n^2)$解决办法太慢了。 一个天真的方法是，对于每个位置$i$, 扫描所有位置$[i + a_i, i + b_i]$，取最大值。 当范围很宽时，这会导致二次行为。 

需要更快的解决方案，因为每个位置可能会影响许多早期的位置，并且反复重新计算最大值会成为多余的工作。 

当跳转范围超出数组边界时，会出现微妙的边缘情况。 在这种情况下，只应考虑有效的指数。 另一个极端情况发生时$a_i = b_i = 0$，这意味着一个位置可能会在不小心的解释中转变为自身。 正确的解释仍然假设仅向前移动，因此自循环不能被视为无限递归； 相反，这些状态在添加奖励后立即终止。 

## 方法

 暴力动态规划方法计算每个$f(i)$通过扫描所有可到达的下一个状态来独立地进行。 这是正确的，因为重复率被明确定义为前向间隔内的最大值。 然而，对于每个$n$职位，我们最多可以扫描$O(n)$未来的职位，生产$O(n^2)$最坏情况下的操作。 什么时候$n$达到$10^5$，这变得完全不可行。 

关键的结构观察是，一旦我们从右到左计算值，所有未来状态$f(i+1), f(i+2), \dots$处理时已经知道$i$。 递归成为对向后填充的动态数组的范围最大查询。 

这正是线段树（或任何范围最大数据结构）消除重复扫描的设置。 我们维护一个支持查询范围内的最大值并有效更新单个位置的结构，而不是重新计算每个索引的最大值。 

我们处理索引$n$下降到$1$。 当在位置时$i$，线段树已经包含所有可到达的前向位置的正确值。 我们查询范围内的最大值$[i + a_i, i + b_i]$，将奖励添加到$i$，并将结果存储回结构中。

 ## 算法演练

 我们将问题重新解释为在有向非循环结构上计算 DP，其中边仅向前移动。 这保证了逆序处理将始终看到已经计算的状态。 

1. 在大小数组上初始化线段树$n + 2$，最初填充中性值，例如零或负无穷大，具体取决于奖励是否可以为负。 该树代表了当前最著名的$f(i)$已处理位置的值。 
2. 迭代自$i = n$下降到$1$。 反向迭代的原因是所有的转换都从$i$到严格更大的指数，所以当我们达到$i$，所有可达状态都已被计算并插入到结构中。 
3. 对于每个$i$,计算有效查询范围$[L, R]$， 在哪里$L = i + a_i$和$R = i + b_i$。 将此范围限制在以内$[1, n]$。 这确保我们永远不会查询问题域之外的无效内存。 
4. 查询线段树中的最大值$[L, R]$。 这给出了从位置开始的最佳可能延续$i$。 如果范围为空，则将结果视为零，这意味着当前位置之外没有进一步的增益。 
5. 计算$f(i) = g_i + \text{bestNext}$，结合本地奖励和最佳可达到的未来价值。 
6.更新位置的线段树$i$和$f(i)$，使其可用于较早的职位。 
7. 处理完所有指标后，答案通常是所有指标中的最大值$f(i)$，因为允许从任何位置开始。 

### 为什么它有效

 目前我们计算$f(i)$，其依赖范围内的所有位置都已被固定并插入到线段树中。 这建立了一个不变量：处理索引之前$i$，树包含所有索引的正确最终值$j > i$。 由于每个转换都严格增加索引，因此没有循环，因此未来的步骤不会修改这些值。 因此，每个查询都会读取最终值，而不是中间近似值，从而使递归精确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

NEG = -10**30

class SegTree:
    def __init__(self, n):
        self.n = 1
        while self.n < n:
            self.n <<= 1
        self.t = [NEG] * (2 * self.n)

    def update(self, i, v):
        i += self.n
        self.t[i] = v
        i >>= 1
        while i:
            self.t[i] = max(self.t[2*i], self.t[2*i+1])
            i >>= 1

    def query(self, l, r):
        if l > r:
            return NEG
        l += self.n
        r += self.n
        res = NEG
        while l <= r:
            if l & 1:
                res = max(res, self.t[l])
                l += 1
            if not (r & 1):
                res = max(res, self.t[r])
                r -= 1
            l >>= 1
            r >>= 1
        return res

def solve():
    n = int(input())
    g = [0] + list(map(int, input().split()))
    a = [0] + list(map(int, input().split()))
    b = [0] + list(map(int, input().split()))

    st = SegTree(n + 2)
    dp = [0] * (n + 2)

    for i in range(n, 0, -1):
        l = i + a[i]
        r = i + b[i]
        if l > n:
            best = 0
        else:
            r = min(r, n)
            best = st.query(l, r)
            if best == NEG:
                best = 0
        dp[i] = g[i] + best
        st.update(i, dp[i])

    print(max(dp[1:n+1]))

if __name__ == "__main__":
    solve()
```线段树仅用作动态范围最大结构。 自底向上迭代树的实现选择避免了递归开销并严格保持更新和查询$O(\log n)$。 使用大的负哨兵可确保无效或未初始化的段不会意外地支配最大值。 

DP 数组在技术上对于最终答案是可选的，但它使推理和调试更清晰，因为每个状态都是显式存储的。 

周围的边界处理$l > n$至关重要。 如果没有它，查询将错误地换行或访问无效段。 同样，夹紧$r$确保我们只考虑可达索引。

认为：$a = [0, 1, 1, 1, 0]$,$b = [1, 2, 2, 1, 0]$我们从右向左处理。

 ## 边缘情况

 第一个边缘情况是位置无法跳转到任何地方，因为$i + a_i > n$。 在这种情况下，正确的行为是将连续值视为零。 该算法通过在查询线段树之前检查范围来显式处理此问题。 如果没有此检查，查询将返回标记或陈旧值，从而错误地影响早期状态。 

当有效范围仅包含一个元素时，会出现另一种边缘情况。 例如，如果$i = 2$,$a_2 = 1$,$b_2 = 1$，那么我们只考虑位置 3。线段树查询减少到单点最大值，它仍然表现正确，因为更新是逐点的和确定性的。 

最后一个边缘情况是所有奖励为零或负数。 使用大的负标记可确保空范围不会主导有效值。 当不存在有效的未来时，我们明确地回落到零，而不是通过 DP 传播无效的最小值。
