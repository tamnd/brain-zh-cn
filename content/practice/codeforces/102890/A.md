 ## 解决方案
 ## 问题理解

 我们正在处理三个不相交的学生组的选择问题，我们可以将其视为标记为 A、B 和 C 的三个项目。每个组都有固定数量的学生，并且我们希望组建一个由恰好 K 个学生组成的团队。 额外的约束是，这 K 个学生中必须有 c 个来自 C 组。 

一旦我们承诺从 C 中选拔 c 个学生，剩下的 K − c 个学生必须完全从 A 组和 B 组的并集中选出。在剩余部分中，A 和 B 之间的分配是灵活的，只要总数为 K − c 即可。 

所以真正的组合结构是：首先选择来自C的c个元素，然后从A和B组合中选择K−c个元素，其中A和B之间的分割是任意的。 

输出是满足这些约束的有效团队的总数。 

尽管该陈述以某种代数和重求和的方式呈现，但关键的隐藏结构是我们正在计算不相交集合上的组合，唯一重要的部分是计算我们可以在 A 和 B 之间分配 K − c 选择的方式有多少种。 

如果我们将组的大小表示为 a、b 和 d（分别表示 A、B 和 C），则约束迫使我们从 C 中选择 c，从 A ∪ B 中选择 K − c。 

一种简单的方法会尝试分别从 A 和 B 中选择多少个元素，并对所有有效的分割进行求和。 这导致二项式系数的卷积。 

此问题的典型版本中的约束意味着组大小可能足够大，以至于不可能对每个查询进行阶乘重新计算，因此任何解决方案都必须在预处理后将计算量减少到 O(1) 或 O(log n)。 

在重复重新计算大二项式系数的情况下，特别是当 K 很大时，简单的实现将会失败。 例如，如果 a = b = 10^5 且 K = 10^5，则迭代从 0 到 K − c 的所有分割 i 已经导致每个测试执行 10^5 次操作，如果重复，则速度太慢。 

另一个微妙的问题是重复计算或忘记导致相同总数的 A 和 B 的不同分割必须恰好相加一次。 粗心的组合实现可能会乘法而不是求和，并产生不正确的过度计数。 

## 方法

 蛮力思想直接源自约束分解。 从 C 中选择 c 个元素后，我们仍然需要从 A 和 B 中选择 K − c 个元素。考虑这个问题的一种方法是迭代所有可能的值 i，其中 i 是从 B 中选择的元素数量。对于每个这样的分割，我们从 B 中选择 i 个元素，并从 A 中选择 K − c − i 个元素，对所有有效 i 求和。 这会产生表达式

 C(b, i) 乘以 C(a, K − c − i) 的 i 的总和，再乘以 C(d, c)。 

这是正确的，因为它枚举了在 A 和 B 之间划分选择的所有可能方法。 

然而，求和在 K 中是线性的，并且在最坏的情况下 K 可以与 A 和 B 的总大小一样大。这使得该方法在重复或约束很大时太慢。 

关键的观察结果是 A 和 B 之间的分割之和正是二项式卷积恒等式，称为范德蒙恒等式。 我们可以将 A 和 B 视为大小为 a + b 的单个组合池，而不是显式地对所有分布求和。 然后可以直接使用单个二项式系数 C(a + b, K − c) 从 A ∪ B 中选择 K − c 元素。 其余部分，从 C 中选择 c 个元素，是独立的，并贡献一个因子 C(d, c)。 

因此，整个问题归结为计算两个二项式系数并将它们相乘。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（分割枚举）| O(K) | O(1) | O(1) | 太慢了|
 | 最优（Vandermonde + 组合）| 预处理后 O(1) | O(n) | 已接受 |

## 算法演练

 我们使用组合分解和预先计算的阶乘来计算答案。 

1. 读取三组尺寸及参数K、c。 这些定义了必须总体选择多少元素以及必须专门从 C 组中选择多少元素。 
2. 计算从 C 组中准确选择 c 个学生的方法数。这是标准二项式系数 C(|C|, c)，因为我们可以自由选择该大小的任何子集。 
3. 观察到，一旦从 C 中固定了 c 个元素，剩下的任务就是从 A 和 B 的并集中选择 K − c 个元素。我们没有将选择分开，而是将 A 和 B 一起视为一组大小 |A| + |B|。 
4. 计算从 A ∪ B 中选择 K − c 元素的方法数为 C(|A| + |B|, K − c)。 这取代了 A 和 B 之间分割的整个求和。 
5. 将两个独立的选择相乘以获得最终答案，因为来自 C 和 A ∪ B 的选择不相互作用。 

### 为什么它有效

 关键的属性是，每个有效的团队都可以通过两个独立的决策来唯一地描述：哪些 c 个元素来自 C，哪些 K − c 元素来自 A ∪ B。一旦我们折叠集合，A 和 B 之间的那些 K − c 元素的内部分布不会改变总计数，因为每个这样的分裂都恰好对应于联合中大小 K − c 的一个子集。 这正是组合恒等式，将分区上的总和与组合集合上的单个二项式系数等同起来。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

MAXN = 200000  # adjust if needed

fact = [1] * (MAXN + 1)
invfact = [1] * (MAXN + 1)

for i in range(1, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN] = pow(fact[MAXN], MOD - 2, MOD)
for i in range(MAXN, 0, -1):
    invfact[i - 1] = invfact[i] * i % MOD

def ncr(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

def solve():
    a, b, c, K, c_needed = map(int, input().split())
    ans = ncr(c, c_needed) * ncr(a + b, K - c_needed) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```阶乘预计算允许在恒定时间内计算每个二项式系数。 功能`ncr`通过返回零来安全地处理无效范围，这自然涵盖了不可能的选择，例如请求比现有元素更多的元素或负计数。 

最后一行将独立的组合选择相乘，反映了分解为“从 C 中选择”和“从 A ∪ B 中选择”。 

## 工作示例

 考虑一个示例，其中 A 有 3 个元素，B 有 2 个元素，C 有 4 个元素，K 为 4，并且我们要求 C 中的 c = 2。 

| 步骤| 来自 C 的价值 | 剩下 A∪B | 计算|
 | ---| ---| ---| ---|
 | 从 C | 选择 C(4,2) = 6 | | |
 | 从 A∪B | 选择 | C(5,2) = 10 | C(5,2) = 10 | |
 | 决赛| | | 60|

 这表明，一旦我们固定了 C 中的元素数量，其余的元素就变成了合并集中的纯子集选择。 

现在考虑 K 等于 c 的边界情况，这意味着我们从 C 中获取所有内容。 

| 步骤| 来自 C 的价值 | 剩下 A∪B | 计算|
 | ---| ---| ---| ---|
 | 从 C | 选择 C(4,4) = 1 | | |
 | 从 A∪B | 选择 | C(5,0) = 1 | |
 | 决赛| | | 1 |

 这证实了该公式正确处理没有从 A 或 B 中获取任何元素的退化情况。

 ## 边缘情况

 当 C 中所需的数字为零时，就会出现一种边缘情况。 在这种情况下，算法简化为从 A 和 B 中选择所有内容。对于像 a = 3、b = 4、c = 5、K = 3、c_needed = 0 这样的输入，计算变为 C(5,0) 乘以 C(7,3)，计算结果为 35。算法自然地处理这个问题，因为 C(n,0) 在阶乘公式中被定义为 1。 

当 K − c 超过 a + b 时，会发生另一种边缘情况。 例如，如果 a = 2、b = 2、c = 10、K = 6、c_needed = 1，则 C(a + b, K − c_needed) 变为 C(4,5)，其正确计算结果为 0。这可以防止无效的过度选择对答案产生影响，并确保组合公式自动丢弃不可能的配置。
