 ## 解决方案
 ## 问题理解

 该任务本质上是在简单的标记化规则下计算给定文本中出现的“单词”数量。 给定一个表示一行文本的字符串，我们必须根据问题隐含的定义确定其中存在多少个不同的单词标记。 单词由连续的字母字符组成，其他任何内容（例如空格、标点符号或数字）都充当分隔符。 输出是一个整数，表示分割文本后出现的此类词段的数量。 

查看输入的另一种方法是我们扫描原始字符流并将其分组为最大连续字母子串。 每次从非字母转换为字母时，我们都会开始一个新单词，每次从字母转换为非字母时，我们都会结束当前单词。 

此类问题中的约束通常足够大，字符串长度最多可达 10^5 或 10^6 个字符。 这立即意味着任何二次方法，例如重复切片或在循环内使用昂贵的字符串操作，都将失败。 唯一可行的方法是对输入进行单次线性扫描，对每个字符处理一次。 

一个幼稚的错误来自于仅在空格上进行分割。 例如，给定输入`hello,world`,天真的`split()`on space 会将其视为一个单词，而正确答案是两个单词：`hello`和`world`。 当多个分隔符连续出现时，会发生另一个微妙的故障，例如`a---b`。 如果不小心，简单的计数器可能会错误地计算空标记，但正确的解释会忽略空段，只计算字母序列的转换。 

## 方法

 暴力方法是将字符串视为由每个可能的非字母字符分隔的子字符串序列。 人们可以迭代每个可能的起始索引，展开直到找到分隔符，并记录子字符串。 虽然这在概念上很简单，但它存在重复扫描重叠区域的风险。 在最坏的情况下，例如所有字母组成的字符串，每个子字符串扩展变为 O(n)，导致 O(n^2) 行为。 

关键的观察是我们永远不需要具体化子串。 我们只需要检测“单词内部”和“单词外部”之间的边界。 这减少了从左到右扫描时跟踪单个布尔状态的问题。 每个字符只处理一次，并且仅当我们从非字母状态输入字母段时才会增加字数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 |---|---|---|---|
 | 暴力子串提取 | O(n^2) | O(n^2) | O(n) | 太慢了|
 | 单通道状态跟踪 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护两条信息：一个单词计数器和一个指示我们当前是否在单词内的标志。 

1.初始化一个计数器`words = 0`和一个布尔值`in_word = False`。 这种状态表示最初我们在任何词段之外。 

2. 从左到右迭代字符串中的每个字符。 每个字符都被分类为字母或分隔符。 

3. 如果当前字符是字母且`in_word`为 False，这意味着我们正在输入一个新单词。 我们递增`words`减 1 并设置`in_word = True`。 这一步至关重要，因为它确保我们只计算每个连续字母块的第一个字符。 

4. 如果当前字符是字母且`in_word`是真的，我们继续在同一个单词内，什么也不做。 这可以防止多字符单词的计数过多。 

5.如果当前字符不是字母，我们设置`in_word = False`，标记任何后续字母都将开始一个新单词。 此转换可确保正确处理一行中的多个分隔符。 

6.处理完所有字符后，得到的值`words`就是答案。 

正确性来自于不变量`in_word`当且仅当前一个字符是属于当前活动词段的字母时才为 True。 每个单词在其第一个字符处只计算一次，并且不再计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().rstrip("\n")
    
    words = 0
    in_word = False
    
    for ch in s:
        if ch.isalpha():
            if not in_word:
                words += 1
                in_word = True
        else:
            in_word = False
    
    print(words)

if __name__ == "__main__":
    solve()
```该解决方案读取输入行并逐个字符地处理它。 这`in_word`flag 是避免重复计算同一单词内字母的核心机制。 唯一微妙的一点是使用`isalpha()`而不仅仅是检查空格，因为分隔符可以包含标点符号或其他非字母符号，具体取决于问题的定义。

 ## 边缘情况

 一种重要的边缘情况是输入根本不包含字母，例如`"12345!!!"`。 算法初始化`in_word = False`并且从未遇到过字母，因此计数器始终保持为零。 输出正确`0`。 

另一种边缘情况是直接以分隔符开头，后跟单词的字符串，例如`"--abc"`。 第一个字符重置`in_word`但不影响计数器。 什么时候`'a'`达到时，它会正确触发新的字数统计，从而导致`1`。 

最后一种情况是交替分隔符和单个字母，例如`"a-b-c"`。 每个字母前面都有一个非字母，因此每个字母都开始一个新单词。 该算法每个字母恰好递增一次，产生`3`，它符合预期的分割规则。
