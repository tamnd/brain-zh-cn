 ## 解决方案
 ## 问题理解

 我们得到了一个字符串列表，所有字符串的长度都相同，并且要求我们通过旋转定义的等价关系对它们进行分组。 如果两个字符串可以通过循环移位字符从另一个字符串获得，则被认为是同一组，这意味着我们在某个位置切割字符串并交换两个结果部分。 

任务是计算所有给定字符串中存在多少个不同的底层“旋转类”。 每个字符串都属于一个类，并且如果不同的字符串相互旋转，它们仍然可以代表同一类。 

解释该问题的一个有用方法是想象每个字符串都写在一个圆圈上。 旋转圆圈不会改变我们所代表的对象。 我们想要计算所有输入字符串中存在多少个不同的圆圈。 

此类问题隐含的约束通常足够大，直接比较每对字符串的速度太慢。 如果有 N 个字符串，并且每个字符串的长度为 L，则所有对之间的所有旋转的简单比较会导致最坏情况下的 O(N²·L²) 行为，当 N 很大时，这变得不可行。 

一个关键的微妙之处在于，相等性不是标准字符串相等性，而是循环相等性。 例如，“abc”、“bca”和“cab”都应被视为同一对象。 一个幼稚的错误是只直接比较字符串而不考虑旋转，这会错误地将它们视为不同的。 

当字符串包含重复模式时，会出现另一种边缘情况。 例如，“aaaa”等于其所有旋转，如果不小心的旋转检查依赖于不完整的匹配逻辑，则可能会过度计数。 

## 方法

 暴力破解的想法很简单：对于每个字符串，将其与所有先前处理过的字符串进行比较，并检查它们是否是彼此的循环旋转。 要测试两个字符串是否旋转，可以尝试一个字符串的每一种可能的移位，并将其与另一个字符串逐个字符进行比较。 这是有效的，因为旋转等效性是通过对齐所有字符的移位位置的存在来定义的。 

然而，这很快就会变得昂贵。 对于每一对，我们可以尝试最多 L 次旋转，每次比较的成本为 O(L)，每对的成本为 O(L²)。 对于 N 个字符串，这变为 O(N²·L²)，这远远超出了典型约束的可用范围。 

关键的观察是每个轮换类都有一个唯一的规范代表。 我们可以将每个字符串转换为单个确定性形式，以便所有旋转都映射到相同的代表，而不是比较每一对。 一旦我们计算出这个代表，问题就减少到计算不同值。 

执行此操作的标准方法是计算每个字符串的字典顺序最小旋转。 使用 Booth 算法可以在每个字符串的 O(L) 时间内有效地完成此操作。 将每个字符串转换为其最小旋转后，我们将这些规范形式插入到哈希集中并计算出现了多少个唯一值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N²·L²) | O(1) 额外 | 太慢了|
 | 规范旋转（展位）| O(N·L) | O(N·L) | 已接受 |

 ## 算法演练

 ### 步骤 1：将每个字符串解释为旋转类问题

 我们将每个字符串视为代表一个圆形排列。 我们的目标是为每个圆分配一个唯一的标识符，该标识符对于同一圆的所有旋转都是相同的。 这种重构至关重要，因为它将成对等价问题转换为归一化问题。 

### 步骤 2：计算每个字符串按字典顺序最小的旋转

对于每个字符串，我们找到其所有循环移位中字典顺序最小的旋转。 这是有效的，因为旋转等价保留了所有循环移位的集合，并且该集合中的最小元素对于每个等价类都是唯一的。 

此步骤是通常使用布斯算法的地方，因为它在线性时间内计算最小旋转，而无需显式生成所有旋转。 

### 步骤 3：将规范形式插入集合中

 一旦每个字符串被转换为其最小旋转，我们将其插入到哈希集中。 由于相同的旋转类会产生相同的规范字符串，因此重复项会自动崩溃。 

### 第四步：输出集合的大小

 集合中不同元素的数量直接对应于不同旋转类的数量。 

### 为什么它有效

 每个字符串都恰好有一个按字典顺序最小的旋转。 字符串的所有旋转都会生成相同的循环集，因此它们共享相同的最小元素。 相反，彼此旋转的两个字符串必须具有相同的旋转组，因此具有相同的最小值。 这在轮换类和规范代表之间建立了一对一的映射，确保了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def booth(s: str) -> str:
    s = s + s
    n = len(s)
    f = [-1] * n
    k = 0

    for j in range(1, n):
        i = f[j - k - 1]
        while i != -1 and s[j] != s[k + i + 1]:
            if s[j] < s[k + i + 1]:
                k = j - i - 1
            i = f[i]
        if s[j] != s[k + i + 1]:
            if s[j] < s[k]:
                k = j
            f[j - k] = -1
        else:
            f[j - k] = i + 1

    start = k
    return s[start:start + len(s) // 2]

def solve():
    n = int(input())
    seen = set()

    for _ in range(n):
        s = input().strip()
        seen.add(booth(s))

    print(len(seen))

if __name__ == "__main__":
    solve()
```实现的核心思想是，每个字符串在 Booth 过程中概念上都被加倍，以便所有旋转都成为连续的子字符串。 返回的切片对应于最小旋转的起始索引。 

套装`seen`仅存储规范形式，因此跨轮换的重复项自然会崩溃。 唯一微妙的部分是确保返回的子字符串具有原始长度，因为我们操作的是双倍字符串。

 ## 边缘情况

 对于像“aaaa”这样的单字符重复字符串，每次旋转都是相同的。 该算法将它们全部映射到同一个规范字符串，因此该集合自然只包含一个条目。 简单的轮换比较可能仍会迭代轮班，但只要相等检查正确，就不会错误分类任何内容。 

对于长度为 1 的字符串，布斯算法会彻底退化，因为双倍字符串的长度为 2，并且只存在一次有效的旋转。 返回的规范形式是字符串本身，因此每个不同的字符恰好贡献一个类。 

对于像“ababab”这样的重复模式，多次轮换可能在几个班次中看起来是相同的。 最小旋转仍然选择确定性起点，防止部分比较方法可能出现的过度计数。
