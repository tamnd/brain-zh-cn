 ## 解决方案
 ## 问题理解

 我们有两个字符串，一个较长的文本字符串和一个较短的模式字符串。 任务是确定模式作为连续子字符串在文本中出现的次数。 每次出现的情况都会被计算在内，包括重叠，因此如果模式可以从文本中的多个相邻位置开始，那么所有这些都会对答案产生影响。 

输入只是两行，一行包含文本，一行包含模式。 输出是一个整数，表示文本中模式完全匹配的起始位置的数量。 

驱动该解决方案的关键约束是字符串可能很大，在典型的竞争性编程设置中可能达到大约 10^5 个字符。 在每个可能的起始位置将模式与文本进行比较的简单方法将在最坏的情况下导致大约 O(nm) 的字符比较，当两个字符串都很大并且长度相似时，这会变得太慢。 

一些边缘情况往往会破坏粗心的实现。 一种是当模式与文本相同时，正确答案恰好是 1，但错误处理边界或索引的实现可能会过度计算或完全错过它。 另一种情况是当模式具有重复结构时，例如“aaaaaaaa”内的“aaaa”，其中重叠匹配很频繁，暴力解决方案可能会退化为二次行为。 第三种情况是当模式长度为一时，文本中的每个字符匹配都是有效的，并且循环边界中的相差一错误很常见。 

## 方法

 解决问题最直接的方法是尝试在文本中每个可能的起始位置对齐模式并逐个字符进行比较。 对于文本中的每个索引 i，我们检查从 i 开始的子字符串是否与模式完全匹配。 这是正确的，因为它详尽地测试了比赛可能开始的所有候选位置。 

然而，这种方法对每个起始位置执行 m 次比较，其中 m 是模式长度。 由于有 n 个可能的起始位置，因此总工作量与 n 乘以 m 成正比。 在最坏情况的输入（例如重复字符的文本和重复字符的模式）中，每个比较序列在失败之前几乎完全进行，从而产生二次行为。 

这种改进来自于认识到模式的重叠移位之间的重复比较是浪费的。 当我们在匹配模式的前缀后因不匹配而失败时，可以重用该前缀信息。 准确捕获字符串的前缀与其后缀如何重叠的结构是 Knuth-Morris-Pratt 算法中使用的前缀函数。 一旦为模式预先计算了这个前缀结构，我们就可以一次性扫描文本，同时维护模式已经匹配的数量。 文本中的每个字符都会处理一次，并且模式指针仅使用预先计算的回退链接向前移动或向后移动。 

这将匹配的重复重新计算变成了文本上的线性时间状态机遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(纳米) | O(1) | O(1) | 太慢了|
 | KMP（前缀函数）| O(n + m) | O(米) | 已接受 |

 ## 算法演练

 我们使用 KMP 字符串匹配框架来解决该问题。

1. 计算模式字符串的前缀函数。 对于模式中的每个位置，该值表示最长的正确前缀的长度，该前缀也是以该位置结尾的后缀。 这个结构告诉我们，如果发生不匹配，我们可以在哪里恢复匹配，而无需从头开始重新检查字符。 
2. 初始化一个指针j = 0，表示当前有多少个模式字符与文本匹配。 
3. 从左到右迭代文本中的每个字符。 对于每个字符，尝试扩展当前匹配。 
4. 如果当前文本字符与位置 j 处的模式字符匹配，则将 j 加 1。 这扩展了当前的部分匹配，因为保留了前缀条件。 
5. 如果发生不匹配且 j 不为零，则使用前缀函数回退：将 j 设置为 j - 1 处的前缀函数值，然后再次尝试匹配。 此步骤重用仍可与已处理文本的当前后缀对齐的最长有效前缀。 
6. 重复回退过程，直到 j 变为零或可以匹配。 如果 j 为零且当前字符仍然不匹配，则继续处理下一个文本字符。 
7. 每当 j 达到模式的完整长度时，我们就发现了一次出现。 增加答案，并将 j 重置为模式最后位置的前缀函数值，以便自然地处理重叠匹配。 

其工作原理是，在扫描中的每个点，j 表示与以当前位置结尾的文本后缀匹配的模式的最长前缀的长度。 前缀函数保证当发生不匹配时，我们转换到下一个可能的最佳前缀，而不会丢失正确性或丢失任何有效的对齐。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def prefix_function(p):
    n = len(p)
    pi = [0] * n
    j = 0
    for i in range(1, n):
        while j > 0 and p[i] != p[j]:
            j = pi[j - 1]
        if p[i] == p[j]:
            j += 1
        pi[i] = j
    return pi

def count_occurrences(s, p):
    if not p or not s:
        return 0

    pi = prefix_function(p)
    j = 0
    ans = 0

    for i in range(len(s)):
        while j > 0 and s[i] != p[j]:
            j = pi[j - 1]
        if s[i] == p[j]:
            j += 1

        if j == len(p):
            ans += 1
            j = pi[j - 1]

    return ans

def main():
    s = input().strip()
    p = input().strip()
    print(count_occurrences(s, p))

if __name__ == "__main__":
    main()
```该解决方案将预处理和扫描分开。 前缀函数计算为模式构建后备结构。 在扫描期间，变量 j 跟踪当前匹配的模式数量。 扫描内的 while 循环是 KMP 转换逻辑的核心，确保不匹配触发受控回退，而不是从头开始重新启动比较。 

一个微妙的实现细节是完整比赛后的重置。 我们没有将 j 设置为零，而是在最后一个匹配位置重用前缀函数值。 这使得重叠匹配能够正确计数，而无需重新扫描字符。

 ## 边缘情况

 一种重要的边缘情况是模式恰好是一个字符长。 在这种情况下，文本中的每个匹配字符都应算作一次出现。 该算法自然地处理这个问题，因为每次匹配都会立即触发完全匹配条件，并使用前缀数组正确重置，该数组始终为零。 

另一种边缘情况是文本中任何地方都没有出现的模式。 指针 j 将反复回落到零，并且不会触发任何匹配事件，因此答案仍然为零。 

第三种情况是最大重叠，例如重复相同字符的文本和较短但也重复的模式。 在这种情况下，每次匹配后，前缀函数都会保持 j 非零，从而允许扫描有效地继续而无需重新启动，并确保每个有效的起始位置都被精确计数一次。
