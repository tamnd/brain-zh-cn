 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$N \times M$。 每个单元格代表一个具有初始“亮度”值的窗口。 这种亮度不仅仅代表局部效应，它还充当将蚊子发送到建筑物内每个窗户（包括其自身）的源头。 

位置有一个窗口$(i, j)$将蚊子贡献给目标窗口$(a, b)$根据基于切比雪夫距离的衰减规则：$$\left\lfloor \frac{F_{i,j}}{2^{\max(|a-i|, |b-j|)}} \right\rfloor$$每个源窗口对每个目标窗口都有独立的贡献，到达窗口的最终蚊子数量是所有源贡献的总和。 

每个窗口最多可承载$S$蚊子。 任何超出的部分$S$那个窗户就被认为掉到了地上。 任务是计算两个值：成功落在所有窗口上的蚊子总数（在应用每个窗口的容量限制后），以及掉落的总数。 

限制因素$N, M \le 500$意味着最多 250,000 个单元。 源和目标之间的天真的全对交互已经暗示了最坏的情况$6.25 \times 10^{10}$交互作用，对于一秒钟来说太大了。 即使每一项贡献$O(1)$，这是不可能的。 

关键的结构特性是贡献仅取决于切比雪夫距离，这意味着它在以每个源为中心的正方形上是恒定的。 这种对称性允许聚合而不是显式枚举所有对。 

当亮度为零时，会出现微妙的边缘情况。 这样的单元在任何地方都没有任何贡献，但简单的实现可能仍然会浪费时间来传播它。 

另一个问题是每个单元格的中间和溢出。 即使最终夹紧发生在$S \le 10^9$，如果管理不当，中间金额可能会大大超过这个数字。 

最后，衰减随距离呈指数增长。 一旦距离超过大约 9 到 10，对于最大亮度 500，大多数贡献将变为零，因为重复除以 2 很快会将值驱动为零。 忽略这个截止点是一个主要的性能差异。 

## 方法

 蛮力方法很简单：对于每个源单元格，迭代每个目标单元格，计算切比雪夫距离，应用整数除以$2^d$，并将结果添加到目标中。 这是正确的，因为它直接遵循定义。 然而，它执行完整的$N^2 M^2$计算，大约是$2.5 \times 10^{10}$在最坏的情况下运作，远远超出了可行性。 

关键的观察结果是，每个源以仅取决于距离层的模式分配值。 对于固定源，切比雪夫距离处的所有单元$d$形成一个方形环。 我们可以考虑逐层累积贡献，而不是单独更新每个单元。 每层$d$对应于以源为中心的正方形，并且整个层的贡献值是恒定的。 

这将问题转化为在网格上应用许多方形范围添加，其中每个源生成$O(\log F_{i,j})$有意义的层，因为在足够除以 2 后值会变为零。可以使用 2D 差分数组有效地处理此类平方更新，其中每个平方更新都在$O(1)$，最终值用前缀和重建。 

在构建了总传入蚊子的完整网格后，我们在最后一遍中应用每个单元的容量约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^2 M^2)$|$O(NM)$| 太慢了 |
 | 图层 + 2D 差异 |$O(NM \log F)$|$O(NM)$| 已接受 |

 ## 算法演练

 我们将每个单元格作为源进行处理，并将其影响分解为距离层。 

1. 对于每个单元格$(i, j)$， 对待$F_{i,j}$作为距离 0 处的初始贡献。该值仅影响第一层中的自身。 
2. 对于每个距离$d \ge 1$，计算值$v_d = \left\lfloor \frac{F_{i,j}}{2^d} \right\rfloor$。 如果$v_d = 0$，停止处理距此源更远的距离，因为所有更深的层也将为零。 
3.对于每个固定距离$d$，所有具有切比雪夫距离的单元格$d$形成边长为正方形的边界$2d+1$。 我们不只显式更新边界，而是利用贡献从嵌套方块逐层累积的事实。 
4.我们将每一层的效果转化为正方形上的加法$[i-d, i+d] \times [j-d, j+d]$。 这种方形表示是有效的，因为层贡献是累积的：距离内的每个单元格$d$至少收到$v_d$，更深的层将细化内部区域。 
5. 我们使用二维差分数组应用这些平方更新。 每次更新都是通过添加$v_d$到左上角，在边界处适当减去它，然后通过前缀和重建。 
6. 处理完所有源和所有层后，我们使用差异数组上的 2D 前缀和构建总传入蚊子的最终网格。 
7. 最后，对于每个单元格，我们将定居的蚊子计算为$\min(\text{total}[i][j], S)$并累计已结算金额和溢出金额。 

### 为什么它有效

 每个源独立贡献，并且衰减仅取决于切比雪夫距离，这创建了恒定贡献水平的嵌套方形区域。 通过将每个源分解为单调递减的平方层，每个单元准确地接收其平方覆盖其的所有有效层贡献的总和。 二维差分结构保留了正确性，因为它精确地表示了轴对齐矩形区域的加法叠加，并且每个正方形层都表示为这些矩形的组合。 不会重复计算或遗漏任何贡献，因为每一层都是独立应用并线性重建的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def add(diff, n, m, x1, y1, x2, y2, val):
    if x1 > x2 or y1 > y2:
        return
    x1 = max(0, x1)
    y1 = max(0, y1)
    x2 = min(n - 1, x2)
    y2 = min(m - 1, y2)
    diff[x1][y1] += val
    if x2 + 1 < n:
        diff[x2 + 1][y1] -= val
    if y2 + 1 < m:
        diff[x1][y2 + 1] -= val
    if x2 + 1 < n and y2 + 1 < m:
        diff[x2 + 1][y2 + 1] += val

def solve():
    n, m, s = map(int, input().split())
    g = [list(map(int, input().split())) for _ in range(n)]

    diff = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(n):
        for j in range(m):
            f = g[i][j]
            if f == 0:
                continue
            d = 0
            while f > 0:
                val = f >> d
                if val == 0:
                    break
                x1, x2 = i - d, i + d
                y1, y2 = j - d, j + d
                add(diff, n, m, x1, y1, x2, y2, val)
                d += 1

    pref = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(n):
        for j in range(m):
            pref[i][j] = diff[i][j]
            if i > 0:
                pref[i][j] += pref[i - 1][j]
            if j > 0:
                pref[i][j] += pref[i][j - 1]
            if i > 0 and j > 0:
                pref[i][j] -= pref[i - 1][j - 1]

    total_ok = 0
    total_fall = 0

    for i in range(n):
        for j in range(m):
            v = pref[i][j]
            if v <= s:
                total_ok += v
            else:
                total_ok += s
                total_fall += v - s

    print(total_ok, total_fall)

if __name__ == "__main__":
    solve()
```该实现构建了一个二维差异数组，其中每个源提供一系列具有指数递减值的扩展平方更新。 位移位`f >> d`直接编码除以二的幂。 前缀重建步骤将差异表示转换为每个单元的实际累积值。 

最后一个循环应用每个单元的容量约束，将定居的蚊子与溢出的蚊子分开。 

一个微妙的细节是差异数组中的有界网格。 它的大小为$n+1 \times m+1$因此边界减法不需要在主逻辑内部重复进行条件处理。 另一个重要的细节是尽早跳过零亮度单元，这可以避免不必要的层循环。

 ## 边缘情况

 一个关键的边缘情况是所有亮度值都为零时。 该算法立即跳过所有更新，并且前缀网格保持为零。 每个单元格的蚊子数量为零，因此定居和掉落的总数都为零。 

另一种情况是每个单元的最大亮度。 每个单元贡献约 10 层，但值很快降至零。 差异数组可以处理重叠的大方块更新，而不会导致每个单元格爆炸。 

最后，案例是$S = 1$迫使几乎所有多余的东西都下降。 该算法仍然首先计算全部总和，仅在最后应用裁剪，从而确保正确性，而不受容量限制。
