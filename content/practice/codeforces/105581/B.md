 ## 解决方案
 ## 问题理解

 我们有一名警察，他以单位速度沿着两个固定点 A 和 B 之间的直线段连续来回巡逻。 该运动是完全周期性的：从时间 0 处的 A 开始，前往 B，然后立即转身返回 A，无限期地重复此过程，没有停顿。 

在某个时间 T，外部事件中断了该运动。 在那一刻，无论警官当前位于路段上的哪个位置，他都会停止巡逻，并再次以单位速度直接走向同一条线上的第三个点 C。 任务是计算从时间 T 到到达 C 需要多长时间。 

输入给出 A 和 B 作为巡逻段的端点，然后是 T 和 C。关键的隐藏任务是确定警员在这个来回运动下在时间 T 的确切位置，然后计算到 C 的剩余直线旅行时间。 

约束允许最多 10^5 的坐标和最多 10^9 的时间。 逐秒步进的模拟是不可能的，因为 T 可能很大，因此任何解决方案都必须使用周期运动的算术结构来计算位置。 

一个天真的陷阱是假设警员总是只从 A 移动到 B，而忽略方向反转。 另一个是忘记了方向取决于发生了多少次完全遍历段长度。 

具体的边缘情况：

 输入：

 A = 3、B = 8、T = 10、C = 1

 在 t = 0 时，他位于 3 处，在 t = 5 处，他位于 8 处，在 t = 10 处，他回到 3 处。假设单调运动的错误方法会错误地将他置于 13 处，该位置位于段之外且无效。 

正确的输出是根据实际振荡计算出来的，再次给出 T = 3 处的位置，因此到达 C 的时间为 |3 − 1| = 2。 

当 T 正好落在转折点时，会发生另一种边缘情况，其中方向立即翻转，但位置仍然是端点。 

## 方法

 暴力模拟将使军官一次前进一秒，保持当前位置和方向，每当到达终点时就翻转方向。 每一秒的成本为 O(1)，因此总复杂度为 O(T)。 这是正确的，但当 T 达到 10^9 时就完全不可行了。 

关键的观察结果是，运动在长度 L = |A − B| 的线段上是周期性的。 一个完整的循环包括从 A 移动到 B 并返回，在 2L 时间内移动 2L 的距离。 我们可以减少时间 T 以周期长度 2L 为模，并确定官员在该周期内的位置，而不是逐步进行模拟。 

一旦知道一个周期内的剩余时间，位置就变成一个简单的线性函数：首先从 A 移动到 B，然后从 B 返回到 A。这消除了所有模拟，并将问题简化为常数时间算术。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(T)| O(1) | O(1) | 太慢了 |
 | 减少周期| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们假设 WLOG A 和 B 定义了一个段，并定义 L = |A − B|。 我们将 A 视为循环的起始端点。 

1. 计算 L = |A − B|。 这是巡逻队的一条腿的距离，因此完整的往返长度为 2L。 
2. 确定该官员在当前周期内的有效时间：t = T mod (2L)。 在此之前的任何完整循环都不会影响最终位置，因为运动会精确重复。 
3. 如果 t ≤ L，则军官位于从 A 到 B 的前腿上，因此他的位置为 A + 方向 * t，其中方向取决于 A < B 是否。 
4. 如果 t > L，则他在回程航段。 我们将其转换为距 B 的剩余前进距离：t' = t − L，则位置为 B − 方向 * t'。 
5. 一旦知道时间 T 的位置 P，就将答案计算为 |P − C|，因为他直接以单位速度移动到 C。

微妙之处在于始终如一地处理方向。 我们没有将 A 视为总是较小的，而是显式地编码方向，以便相同的公式适用于这两种情况。 

### 为什么它有效

 该运动是在具有反射边界的线段上的确定性行走。 每 2L 时间单位，状态（位置和方向）精确地返回到 A 处向 B 移动的起始构型。这创建了一个完美的周期轨道。 以周期为模减少时间可以保留该轨道内的精确状态，因此计算出的位置与完整模拟产生的位置相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

A, B = map(int, input().split())
T, C = map(int, input().split())

L = abs(A - B)
period = 2 * L

# If A == B (degenerate, though constraints say A != B)
# motion still handled normally since L > 0 guaranteed

t = T % period

# normalize direction: start moving from A toward B
if A < B:
    start = A
    end = B
    forward = 1
else:
    start = A
    end = B
    forward = -1

if t <= L:
    pos = start + forward * t
else:
    t2 = t - L
    pos = end - forward * t2

ans = abs(pos - C)
print(ans)
```该解决方案首先计算段长度并使用模运算减少时间。 方向被标准化，因此“向前”始终意味着从 A 移动到 B，即使 A > B。 

第一阶段处理从 A 到 B 的运动。如果减少的时间仍在该阶段内，则通过线性运动计算位置。 否则我们切换到从B开始的返回阶段并对称地向后移动。 

最后，答案是到C的直接距离，因为T之后的运动是不受限制的直线运动。 

## 工作示例

 ### 示例 1

 输入：

 A = 3、B = 8、T = 10、C = 1

 这里 L = 5，周期 = 10，因此 t = 10 mod 10 = 0。 

| 步骤| t | 相| 职位|
 | --- | --- | --- | --- |
 | 开始| 0 | 在 A | 3 |

 在 t = 0 时，军官在一个完整周期后再次恰好位于 A 处。 从那里到 C 的距离为 |3 − 1| = 2。 

这表明完整周期通过模正确崩溃。 

### 示例 2

 输入：

 A = 2、B = 6、T = 7、C = 5

 L = 4，周期 = 8，因此 t = 7。 

| 步骤| t | 相| 职位|
 | --- | --- | --- | --- |
 | 开始 | 7 | 前进 2，后退 3 | 计算|

 由于 t ≤ L 为假，我们处于返回阶段。 我们计算 t2 = 7 − 4 = 3，因此位置为 6 − 3 = 3。 

到 C 的距离为 |3 − 5| = 2。 

这表明向后运动段的正确处理。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    A, B = map(int, input().split())
    T, C = map(int, input().split())

    L = abs(A - B)
    period = 2 * L
    t = T % period

    if A < B:
        start, end, f = A, B, 1
    else:
        start, end, f = A, B, -1

    if t <= L:
        pos = start + f * t
    else:
        t2 = t - L
        pos = end - f * t2

    return str(abs(pos - C))

# sample-like cases
assert run("3 8\n10 1\n") == "2"
assert run("6 1\n7 9\n") == "2"

# minimum movement
assert run("0 1\n0 0\n") == "0"

# immediate stop
assert run("2 5\n0 4\n") == "2"

# mid forward phase
assert run("1 5\n2 10\n") == "8"

# mid backward phase
assert run("1 5\n6 2\n") == "3"

 ## 边缘情况

 一种重要的边缘情况是 T 是整个周期长度 2L 的精确倍数。 在这种情况下，t 在模约减后变为 0，这意味着军官正好在 A 处再次向 B 移动。 该算法自然地处理这个问题，因为 t ≤ L 将我们送入前进阶段并产生位置 A。 

另一种边缘情况发生在军官在时间 T 恰好位于 B 处时，即 t = L。条件 t ≤ L 确保将其视为前进阶段的端点，从而产生没有歧义的位置 B，这与瞬时方向反转一致。 

第三种情况是当 A > B 时。由于方向是显式编码的而不是依赖于排序，因此适用相同的公式。 例如，当 A = 8，B = 3 时，前向步骤向递减坐标移动，后向步骤正确地反转它，确保对称性而无需特殊情况。
