 ## 解决方案
 ## 问题理解

 我们得到一个 n × n 的网格，代表一个方形战场。 一些牢房最初着火，其余的则空无一人。 骑士从任何空的格子开始，并试图通过最终移出网格来逃脱。 

时间以离散的步骤进展。 在每一步中，都会按顺序发生两件事：首先，骑士进行标准的国际象棋骑士移动，然后火焰从任何当前燃烧的单元格蔓延到马移动可到达的所有单元格。 如果骑士在任何时候降落在已经燃烧或在扩散后变得燃烧的细胞上，则被视为被捕获。 只有当骑士最终能够在与火共享一个单元格之前走出网格时，他才能成功。 

任务是计算在这个耦合的动态过程下有多少个起始单元可以让骑士安全地到达网格外。 

网格大小可达 500 x 500，最多可容纳 250,000 个单元格。 从每个单元直接模拟所有可能的骑士路径是不可能的，因为每个单元分支为最多 8 个动作，并且火焰同时演化。 即使每个单元只有一个 BFS 也太大了，因为在最坏的情况下，这会将 250,000 个状态乘以另一个 250,000 倍。 

一个幼稚的错误是将骑士的移动和火势蔓延分开对待。 例如，假设我们可以预先计算火力到达时间，然后在没有仔细同步时间的情况下为骑士运行最短路径，如果等时间冲突处理不当，则会导致错误的结果。 

另一个微妙的失败案例是忽略类似奇偶校验的时序约束。 由于骑士和火力都以离散的步骤移动，因此骑士是否同时在火力之前或之后到达决定了生存，并且差一处理变得至关重要。 

## 方法

 直接模拟视角将从每个空单元格开始，尝试模拟所有可能的骑士动作，同时逐步模拟火力扩展。 这立即导致状态空间爆炸。 每个位置最多可分为 8 个下一个位置，并且火势也会类似地扩展，因此组合状态实际上是位置和时间的乘积。 在最坏的情况下，探索所有路径直到逃脱或死亡可能会在不同的时间多次重新访问相同的状态，从而产生指数行为。 

关键的观察是我们实际上不需要独立模拟所有骑士路径。 火势蔓延与骑士无关，因此我们可以预先计算每个单元格因火而变得不安全的最早时间。 一旦我们知道了这一点，问题就简化为受约束的最短路径：骑士能否在火到达每个访问的单元格之前到达边界。 

这将问题转化为用于射击计时的多源 BFS，然后是具有时间约束的骑士可达性的 BFS。 火灾 BFS 为每个单元给出了一个“危险时间”。 仅当严格早于该时间到达时，来自起始单元的骑士 BFS 才有效。 我们不是从每个单元运行这个 BFS，而是相反：我们计算哪些单元可以安全到达边界，然后在时间限制的情况下通过反向骑士移动向后传播该信息。 

这种逆转之所以有效，是因为逃逸纯粹是通过到达网格外部来定义的，而不是通过到达特定的目标单元格来定义的，因此我们可以将边界逃逸视为全局接收器。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每次启动模拟 | 指数| O(n²) | 太慢了 |
 | Fire BFS + 约束反向可达性 | O(n²) | O(n²) | 已接受 |

 ## 算法演练

 我们通过首先计算每个单元何时因火灾而变得危险来解决这个问题，然后确定哪些单元在那一刻之前仍然可以到达边界。 

1. 计算矩阵`fire_time`每个单元存储着火到达它的最早的步骤。 我们用时间 0 初始化所有火单元并将它们推入 BFS 队列。 所有其他单元格均以无穷大开始。 我们扩展了骑士的动作，因为火焰的传播就像骑士一样。 
2. 同时从所有初始火单元运行多源 BFS。 当在时间 t 从单元格扩展时，如果尚未分配，每个可到达的骑士移动邻居都会被分配时间 t + 1。 这给出了火焰到达每个单元的最短时间，因为 BFS 按递增的时间顺序进行探索。 
3. 确定哪些细胞可以在一次骑士移动中立即逃离网格。 在这些单元格中，至少有一个马的动作会离开棋盘。 这些边界相邻的单元格被视为最初的“获胜”状态，但前提是骑士在火灾到来之前存活了足够长的时间并站在那里。 
4. 建立骑士走法的反向图。 对于每个单元格，我们知道可以通过一次骑士移动到达它的所有前驱位置。 
5. 从所有可立即逃逸的单元开始运行 BFS（或队列传播）。 如果骑士可以在时间 t 到达某个单元格，并且 t 严格小于`fire_time[cell]`，然后就可以达到获胜状态。 
6. 向后传播时，如果较早到达 v 仍然保持安全，则仅接受从单元 u 到前驱 v 的转换，这意味着传播时间不超过`fire_time[v] - 1`。 
7. 计算最后标记为获胜的所有单元格。 

关键思想是，我们正在有效地计算所有状态的集合，从这些状态中存在一条到达边界的安全的、尊重时间的路径，其中火灾充当每个节点的最后期限约束。 

### 为什么它有效

 火灾 BFS 为每个单元计算一个固定的最早到达时间，该时间不依赖于骑士。 这会导致每个单元都有一个严格的截止日期。 任何有效的骑士道路都必须在每一步都遵守这些最后期限。 

马边上的反向 BFS 探索所有最终可以导致逃脱的状态，同时尊重单调时间可行性。 由于该反向图中的每条边都精确对应于有效的前向骑士移动，并且我们仅在满足时间约束时进行传播，因此每个标记状态都对应于真正可行的逃脱策略。 相反，任何有效的逃逸路径都必须终止于边界，并且可以通过有效的前驱状态向后追踪，这意味着它将被这种传播发现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n = int(input())
    grid = [input().strip() for _ in range(n)]

    INF = 10**18
    fire_time = [[INF] * n for _ in range(n)]

    q = deque()

    for i in range(n):
        for j in range(n):
            if grid[i][j] == '#':
                fire_time[i][j] = 0
                q.append((i, j))

    moves = [(2,1),(2,-1),(-2,1),(-2,-1),(1,2),(1,-2),(-1,2),(-1,-2)]

    while q:
        x, y = q.popleft()
        for dx, dy in moves:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < n:
                if fire_time[nx][ny] == INF:
                    fire_time[nx][ny] = fire_time[x][y] + 1
                    q.append((nx, ny))

    rev = [[[] for _ in range(n)] for _ in range(n)]
    for i in range(n):
        for j in range(n):
            for dx, dy in moves:
                ni, nj = i + dx, j + dy
                if 0 <= ni < n and 0 <= nj < n:
                    rev[ni][nj].append((i, j))

    dist = [[INF] * n for _ in range(n)]
    dq = deque()

    def can_escape_cell(i, j):
        for dx, dy in moves:
            ni, nj = i + dx, j + dy
            if not (0 <= ni < n and 0 <= nj < n):
                return True
        return False

    for i in range(n):
        for j in range(n):
            if can_escape_cell(i, j) and fire_time[i][j] > 0:
                dist[i][j] = 0
                dq.append((i, j))

    while dq:
        x, y = dq.popleft()
        for px, py in rev[x][y]:
            nd = dist[x][y] + 1
            if nd < fire_time[px][py] and nd < dist[px][py]:
                dist[px][py] = nd
                dq.append((px, py))

    ans = 0
    for i in range(n):
        for j in range(n):
            if dist[i][j] < INF:
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```第一个 BFS 使用多源扩展计算火灾到达时间。 第二阶段构建反向骑士邻接，以便我们可以从边界相邻单元向后传播逃逸可行性。 这`dist`数组以逆时单位追踪最早的安全逃生距离。 关键条件`nd < fire_time[px][py]`强制我们绝不允许出现骑士到达火场或火场后的状态。 

边界检查功能至关重要，因为只有当单个骑士移动退出网格时才有可能逃脱。 这些细胞充当反向传播的初始接收器。

 | 测量 | 复杂性 | 说明|

 ## 边缘情况

 一种边缘情况是，一个单元可以立即逃离棋盘，但也与在时间 1 到达的火相邻。仅当满足以下条件时，该算法才会通过初始化边界逃逸单元来处理此问题：`fire_time[i][j] > 0`，确保骑士存活足够长的时间以进行逃跑。 

例如：```
#..
...
...
```单元格 (0,1) 可以向上逃逸，但 (0,0) 处的火会一步向它蔓延。 BFS 仅在严格发生在火灾到达之前发生逃生的情况下才接受它，因此只有在安全时才正确包含它。 

另一个边缘情况是孤立的安全岛，火灾永远不会到达某些区域。 在那些细胞中，`fire_time`保持无穷大，因此它们总是有资格传播。 反向 BFS 自然地通过它们进行扩展，无需额外处理，并且它们完全有助于最终计数。
