 ## 解决方案
 ## 问题理解

 有两条平行的人行道 A 和 B，每条都包含从 1 到 n 的位置。 您从人行道 A 上的位置 1 开始，想要到达人行道 B 上的位置 n。 

沿着人行道向前移动一个位置需要一秒钟。 在某些位置有人行横道连接两条人行道。 通过人行横道过马路也需要一秒钟，但只有在交通灯为绿色时才允许。 

所有交通灯都有相同的时间表。 它们在时间 0 处开始呈绿色。每 k 秒颜色就会翻转一次。 这意味着有时灯是绿的`[0, k-1]`，红色期间`[k, 2k-1]`，绿色期间`[2k, 3k-1]`， 等等。 每一秒你都可以向前走、在允许的情况下穿过，或者等待。 

您必须在某个位置恰好使用一条人行横道 c. 一旦穿过，您继续在人行道 B 上行走，直到到达位置 n。 

输入给出人行道长度n、人行横道数量m、切换间隔k以及所有人行横道的位置。 对于每个测试用例，我们必须输出最短到达时间。 

约束是关键的观察结果。 我们最多有$10^5$每个测试用例的人行横道，最多 10 个测试用例。 任何在每个人行横道上执行超过持续工作的解决方案都有可能变得太慢。 一个$O(m)$解决方案很容易足够快，而任何二次方都是不可能的。 

唯一的决定是使用哪个人行横道。 一旦人行横道确定，路径就完全确定了。 

一个微妙的问题是，在红色阶段到达人行横道可能需要等到下一个绿色阶段。 

考虑：```
n = 6, k = 1
crosswalk = 2
```您在时间 1 到达位置 2。由于 k = 1，时间 1、3、5... 为红色。 您不能立即过马路，必须等到时间 2。仅检查到达时信号灯是否为绿灯而忽略等待的解决方案会产生错误的答案。 

当您恰好在绿色阶段开始时到达时，会出现另一种边缘情况。```
n = 10
k = 5
crosswalk = 6
```您在时间 5 到达。时间 5 属于区间`[5, 9]`，它是红色的，而不是绿色的。 一个常见的差一错误是错误地处理间隔端点并允许立即交叉。 

第三种情况是唯一的人行横道接近尽头时。```
n = 100
crosswalk = 99
```实际上，在过马路之前步行几乎整个距离可能是最佳选择。 任何总是选择第一个可用人行横道的贪婪策略都会失败。 

## 方法

 蛮力的想法很简单。 对于每个人行横道位置 c，如果我们选择该人行横道，请计算总行驶时间。 

从位置 1 步行到位置 c 需要`c - 1`秒。 令到达时间为`t`。 

如果在时间 t 信号灯是绿灯，我们可以立即过马路。 否则我们会等到下一个绿色阶段。 过马路后，我们在路口停留一秒钟，然后步行`n - c`在人行道上停留更多秒 B.

 由于只有 m 种可能的人行横道选择，因此独立评估所有这些选择已经给出了$O(m)$算法。 

有趣的部分是有效地确定等待时间。 

指示灯每 k 秒交替一次。 如果我们计算```
block = t // k
```那么块 0 是绿色的，块 1 是红色的，块 2 是绿色的，依此类推。 

什么时候`block`均匀后，即可立即开始穿越。 

什么时候`block`很奇怪，我们处于红色区间内。 下一个绿色间隔开始时间```
(block + 1) * k
```所以所需的等待时间是```
(block + 1) * k - t
```一旦进行了这种观察，就可以在恒定的时间内评估每个人行横道。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过直接模拟等待，对所有人行横道进行暴力破解 | O(米) | O(1) | O(1) | 已接受 |
 | 最佳 | O(米) | O(1) | O(1) | 已接受 |

 在这个问题中，暴力选择枚举已经是最优的，因为唯一的决定是使用哪个人行横道。 

## 算法演练

 1. 将答案初始化为一个非常大的数字。 
2. 对于每个人行横道位置`c`，计算到达该人行横道的时间：```
arrive = c - 1
```每秒行走一个位置意味着恰好在`c - 1`秒。 
3.判断此时灯是绿灯还是红灯`arrive`:```
block = arrive // k
```偶数块为绿色，奇数块为红色。 
4.如果`block`为偶数时，将等待时间设置为零。 
5.如果`block`是奇数，计算直到下一个绿色阶段的等待时间：```
wait = (block + 1) * k - arrive
```6. 计算该人行横道的总行驶时间：```
total = arrive + wait + 1 + (n - c)
```这些术语代表步行到人行横道、等待、过马路和在人行道 B 上行走。 
7. 更新最低答案。 
8. 处理完所有人行横道后，输出找到的最小值。 

### 为什么它有效

 对于固定的人行横道位置c，只有一条可能的路线。 到达人行横道的时间总是`c - 1`，独立于任何未来的决定。 除了步行之外唯一的额外费用是交通灯时间表造成的等待时间。 

时间 t 时的交通灯状态仅取决于`t // k`。 如果相应的区块是绿色的，则立即穿越是最佳选择。 如果是红色，则在下一个绿色区块开始之前不可能穿越，并且恰好等到那一刻是最佳选择，因为任何额外的等待只会延迟到达。 

由于每种可行的策略都必须选择给定的人行横道之一，因此评估所有人行横道并采取最小值检查每种可能的解决方案。 计算出的最小旅行时间正是最佳答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())

    for _ in range(t):
        input()  # consume possible blank line

        line = input().strip()
        while not line:
            line = input().strip()

        n, m, k = map(int, line.split())
        crosswalks = list(map(int, input().split()))

        ans = 10**30

        for c in crosswalks:
            arrive = c - 1
            block = arrive // k

            if block % 2 == 0:
                wait = 0
            else:
                wait = (block + 1) * k - arrive

            total = arrive + wait + 1 + (n - c)
            ans = min(ans, total)

        print(ans)

solve()
```实现直接遵循数学公式。 

最重要的细节是灯光时间表的解释。 当时的状态`arrive`是由`arrive // k`。 偶数块对应于绿色，奇数块对应于红色。 

另一个容易犯的错误是忘记穿越会消耗一秒钟。 即使绿灯立即变绿，过马路动作本身仍然需要时间。 

Gym 语句中的输入在各部分之间包含空行。 该代码在读取每个测试用例之前安全地跳过空行，使其对格式化输入和紧凑输入都具有鲁棒性。

 ## 边缘情况

 考虑：```
1
6 1 1
2
```到达时间是`1`。 自从`1 // 1 = 1`，灯为红色。 该算法计算：```
wait = (1 + 1) * 1 - 1 = 1
```总时间变为：```
1 + 1 + 1 + 4 = 7
```输出是正确的，因为在时间 1 交叉是不可能的。 

现在考虑一个精确的相边界：```
1
10 1 5
6
```到达时间是`5`。```
block = 5 // 5 = 1
```块 1 是红色的，所以：```
wait = 10 - 5 = 5
```该算法正确地识别出时间 5 已经在红色区间内。 结果是15。 

最后，考虑一下晚点的人行横道：```
1
100 2 10
2 99
```对于人行横道 2，到达时间为 1，总数为 100。 

对于人行横道 99，到达时间为 98。`98 // 10 = 9`，灯为红色，等待 2 秒。 总计为：```
98 + 2 + 1 + 1 = 102
```该算法会比较两种可能性并选择 100。这证实了每个人行横道都是独立评估的，并选择了最好的一个。
