 ## 解决方案
 ## 问题理解

 我们在 8 × 8 的棋盘上进行一系列类似国际象棋的动作。 初始设置是固定的：白色棋子占据第 1 行和第 2 行的所有方格，而黑色棋子占据第 7 行和第 8 行。每个棋子永久属于一方； 没有促销或创作新作品。 

每一步都是从给定的起始方格中取出一个现有的棋子并将其放置在目标方格上。 关键的简化是，无论几何形状如何，棋子都可以移动到任何方格。 唯一的限制是棋子不能放置在当前被相同颜色棋子占据的方格上。 如果目标方格包含对手棋子，则该对手棋子将被移除。 

动作顺序按时间顺序给出，但记录可能包含错误。 如果在应该发生的时刻，正在移动的棋子不存在于起始方格上，或者如果起始方格不包含先前未移除的正确颜色的棋子，则移动被视为无效。 一旦动作无效，之后的一切都无关紧要。 我们必须找到最小的索引 i，使得第一个 i 移动已经产生矛盾。 如果没有出现矛盾，我们输出-1。 

限制很小：最多 100 步，棋盘只有 64 个方格。 这立即表明我们可以直接模拟该过程。 任何明确跟踪棋盘状态并在每次移动时更新它的解决方案都足够快。 

主要的微妙之处在于，棋子在被捕获时会消失，因此移动可能会变得无效，不是因为最初不可能，而是因为该棋子之前已经被移除。 一个幼稚的错误是只检查一个片段是否曾经存在过，而不是检查它当时是否仍然存在。 

当具有相同颜色的多块在概念上可能从输入中的同一个方块开始时，就会出现第二种微妙的情况。 这在有效状态下是不可能的，因此遇到它就已经表示不一致。 

## 方法

 最直接的做法就是维持8乘8的棋盘，一一模拟走法。 每个单元格要么不包含棋子，要么包含白色棋子，要么包含黑色棋子。 对于每一步，我们都会检查起始方块当前是否包含正确颜色的棋子。 如果不是，我们立即知道这一步的记录不一致。 

如果起始方格有效，我们就从中取出棋子，然后将其放在目标方格上。 如果目标方格包含对手棋子，我们只需覆盖它即可。 如果它包含相同颜色的棋子，则该移动无效，因为规则禁止将棋子放在友方棋子上。 

由于 n 最多为 100，因此每次操作都是 O(1)，因此这种方法已经是最优的。 

不需要更高级的数据结构，因为状态空间很小并且更新时间恒定。 任何进一步优化的尝试只会使正确性变得复杂，而不会提高性能。

 ## 算法演练

 我们将棋盘表示为二维数组。 每个单元格存储 0 表示空、1 表示白色、2 表示黑色。 

我们通过用白色棋子填充第 1 行和第 2 行，用黑色棋子填充第 7 行和第 8 行来初始化棋盘。 

我们按顺序处理动作。

1. 对于每一步 i，读取 (x1, y1, x2, y2)。 我们将坐标直接解​​释为棋盘索引。 这一步设置了我们此时正在推理的确切平方。 
2. 检查起始单元格 (x1, y1)。 如果为空，则不存在可移动的棋子，因此第 i 次移动立即无效。 我们可以停下来返回i，因为之前的动作没有产生矛盾。 
3. 检查 (x1, y1) 处的棋子颜色是否与预期玩家匹配。 移动顺序交替：白色在奇数索引上移动，黑色在偶数索引上移动。 如果颜色不匹配，则这是结构不一致，因为玩家无法移动对手的棋子。 
4. 检查目标单元格 (x2, y2)。 如果它包含与移动者颜色相同的棋子，则该移动违反了棋子不能放置在友方棋子上的规则。 在更新电路板之前必须检测到这一点。 
5. 如果目的地包含对手棋子，我们将其视为已捕获并覆盖它。 
6. 最后，移动棋子：清除(x1,y1)并将(x2,y2)设置为移动颜色。 

如果所有的移动都被处理且没有矛盾，我们返回-1。 

### 为什么它有效

 棋盘数组始终表示处理有效移动前缀后的确切状态。 每一个无效的举动都恰好对应于违反规则中的存在或占用限制。 由于移动是按顺序应用的，并且每个移动仅取决于当前的棋盘状态，因此任何矛盾都必须在状态变得不可能的最早点出现。 因此，第一个检测到的无效移动保证是出现不一致的最小索引。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    board = [[0] * 9 for _ in range(9)]
    
    for i in range(1, 3):
        for j in range(1, 9):
            board[i][j] = 1
    
    for i in range(7, 9):
        for j in range(1, 9):
            board[i][j] = 2
    
    for i in range(1, n + 1):
        x1, y1, x2, y2 = map(int, input().split())
        
        piece = board[x1][y1]
        
        if piece == 0:
            print(i)
            return
        
        expected = 1 if i % 2 == 1 else 2
        if piece != expected:
            print(i)
            return
        
        if board[x2][y2] == expected:
            print(i)
            return
        
        board[x1][y1] = 0
        board[x2][y2] = expected
    
    print(-1)

if __name__ == "__main__":
    solve()
```该解决方案首先按照描述构建初始棋盘，其中白色棋子占据第 1 行和第 2 行，黑色棋子占据第 7 行和第 8 行。然后模拟逐行进行，在应用任何状态更改之前检查有效性。 

关键的顺序是我们在修改棋盘之前验证移动。 这可以避免意外覆盖正确性检查所需的信息。 对空起始方块的检查是防止使用已经捕获的棋子的主要措施，而颜色交替检查则确保移动顺序尊重回合顺序。

 考虑一个由三个动作组成的简化序列，其中早期的捕获会使后面的动作无效。

这表明正确性取决于跟踪删除，而不仅仅是初始放置。

 ## 边缘情况

 一个重要的边缘情况是当一个片段被捕获并随后被引用为起始片段时。 在这种情况下，棋盘单元会变空，任何从中移动的尝试都必须立即失败。 模拟在执行其他操作之前会显式检查占用情况，因此这种情况可以自然处理。 

另一种情况是，当移动试图将棋子放置到由相同颜色占据的方块上时。 因为我们总是将当前状态存储在棋盘数组中，所以在覆盖之前会在恒定时间内检查它。 如果我们错误地先更新然后检查，我们将丢失检测违规所需的信息。 

最后一个微妙的情况是重复无效的动作。 一旦找到第一个无效的着法，答案一定是它的索引，我们不能继续扫描。 该算法立即返回，保留找到最小此类索引的要求。
