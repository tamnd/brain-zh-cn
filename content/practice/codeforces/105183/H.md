 ## 解决方案
 ## 问题理解

 我们从一系列严格增加的塔高度开始。 然后游戏以离散的步骤发展。 在步数处`j`，我们查看每对相邻的塔。 每当塔之间的差异`i`和塔`i-1`正是`j`，我们执行一个操作：我们从位置增加每个塔`i`到最后`1`。 

这个过程会随着时间的推移而改变高度，因此高度的差异和总高度都会随着步骤的进展而变化。 重要的是，对于每个怪物查询，该过程都会从初始数组重新启动，并且我们会被问到一个问题：最早的步骤`j`所有塔高度的总和是否足够大以杀死该怪物。 

怪物的答案仅取决于总和随时间的演变，而不取决于找到最后一步后各个塔的状态。 

限制很大：最多一百万个塔和一百万个查询。 任何逐一模拟步骤的解决方案都会立即变得太慢，因为步骤索引`j`它本身可能很大，并且该过程不能天真地迭代。 

动态中出现了一个微妙的问题：操作改变了数组，这反过来又改变了未来的差异。 一种天真的解释是，每一对只贡献一次，这会导致过程模型不正确并低估增长。 正确的模型必须考虑到随着差异的变化，在多个步骤中重复触发同一位置。 

打破天真的推理的边缘情况包括答案为零的情况，这意味着初始总和已经杀死了怪物，以及非常大的情况`h_i`，其中答案是由长期线性增长而不是早期跳跃决定的。 

## 方法

 直接模拟将维护数组并在每一步重新计算所有相邻差异。 每一步`j`，我们扫描所有`n`位置、应用更新并重新计算总和。 这已经花费了`O(n)`每一步，并且在最坏的情况下`j`本身可以增长到数组中最大值的数量级，最多可达`1e9`。 即使该过程尽早稳定下来，重复扫描也会使其变得不可行。 

关键的观察是我们实际上在任何步骤都不需要完整的数组。 我们只需要一步后所有高度的总和`j`。 如果我们了解每个位置对步骤增量的贡献次数，则可以独立跟踪每个位置的效果`j`。 

让我们关注单个相邻对。 定义`d_i = a_i - a_{i-1}`。 在步骤`d_i`，条件触发，我们从以下位置开始增加后缀`i`。 经过这次操作后，`a_i`增加同时`a_{i-1}`不，这会增加差异`d_i`经过`1`。 这意味着在步骤`d_i + 1`，条件再次成立。 相同的逻辑无限重复：一旦某个位置开始触发，它就会在后续的每个步骤中继续触发。 

这将这个过程变成了累积效应：对于每个索引`i`，它对每一步的后缀和都有贡献`j`这样`j >= d_i`，其贡献随时间线性增加。 

由此，我们可以表示步骤后的总和`j`纯粹就初始数据而言。 每个指数贡献一个算术级数`j`，仅取决于其初始差异及其权重，即其后缀中的元素数量。 

这将问题简化为评估函数`S(j)`有效地然后为每个查询找到最小的`j`这样`S(j) >= h_i`。 自从`S(j)`是单调的`j`，我们可以二分查找`j`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 步进模拟 | O(n·最大步长) | O(n) | 太慢了 |
 | 前缀贡献+二分查找 | O((n + q) log n) | O((n + q) log n) | O(n) | 已接受 |

 ## 算法演练

 ### 1.构建差异结构

 计算`d_i = a_i - a_{i-1}`对于每一个`i >= 2`。 每个`d_i`确定何时位置`i`开始影响总金额。 

我们还定义了一个权重`w_i = n - i + 1`，这正是位置增加时增加的塔数`i`触发器。 

### 2. 按激活时间排序贡献

 我们对所有对进行排序`(d_i, w_i)`经过`d_i`。 这使我们能够有效地推断出在给定步骤中有多少位置处于活动状态`j`。 

### 3. 构建前缀聚合

 我们维护排序位置的前缀和：`W(k) = sum of w_i for d_i <= k`

`WD(k) = sum of w_i * d_i for d_i <= k`这些让我们可以计算任何步骤的总贡献，而无需迭代所有索引。 

### 4. 评估给定步骤的总和

 对于固定的`j`，每个索引仅在以下情况下贡献`d_i <= j`，其贡献与它​​活跃的步数成正比。 

总和变为：`S(j) = base + sum w_i * (j - d_i + 1) for all d_i <= j`使用前缀值扩展它：`S(j) = base + j * W(j) - (WD(j) - W(j))`这个公式允许计算`S(j)`在对数时间内使用二分搜索排序`d_i`。 

### 5. 使用二分搜索回答每个查询

 对于每个怪物的生命值`h`，我们二分查找最小的`j`这样`S(j) >= h`。 

我们包括`j = 0`作为没有发生任何操作的初始状态。 

### 为什么它有效

 各位置`i`在其激活时间后独立地贡献于总和`d_i`。 一旦激活，它的贡献就会随着步数线性增长。 由于激活时间仅取决于初始差异并且永远不会减少，因此激活索引集是单调的`j`。 这保证了`S(j)`是单调递增的，使得二分查找有效。 前缀公式准确地捕获了所有主动贡献，而无需重复计算，因为每个贡献都被分解为线性函数`j`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    h = list(map(int, input().split()))

    base = sum(a)

    events = []
    for i in range(1, n):
        d = a[i] - a[i - 1]
        w = n - i
        events.append((d, w))

    events.sort()

    ds = [0]
    ws = [0]
    wds = [0]

    for d, w in events:
        ds.append(d)
        ws.append(ws[-1] + w)
        wds.append(wds[-1] + w * d)

    def S(j):
        # find last d_i <= j
        lo, hi = 0, len(events)
        while lo < hi:
            mid = (lo + hi + 1) // 2
            if ds[mid] <= j:
                lo = mid
            else:
                hi = mid - 1
        k = lo

        W = ws[k]
        WD = wds[k]

        return base + j * W - (WD - W)

    def can(j, target):
        return S(j) >= target

    max_d = max([d for d, _ in events], default=0)
    ans = []

    for target in h:
        lo, hi = 0, max(max_d, target) + 1

        while lo < hi:
            mid = (lo + hi) // 2
            if can(mid, target):
                hi = mid
            else:
                lo = mid + 1

        ans.append(str(lo))

    print(" ".join(ans))

if __name__ == "__main__":
    solve()
```该代码首先使用初始差异来压缩所有转换点。 然后，它构建前缀和来评估任何步骤中所有活跃指数的总贡献。 功能`S(j)`计算步骤后的总伤害`j`在对数时间内使用二分搜索超过激活阈值。 

外部二分搜索找到计算出的伤害达到每个怪物所需阈值的最早步骤。 

一个微妙的点是处理`j = 0`，它对应于任何更新之前的初始数组。 通过返回，这自然包含在计算中`base`当没有活动活动的事件时。

在步骤`j = 0`，没有活跃的贡献，所以`S(0) = 10`。 

在步骤`j = 2`， 仅有的`d = 2`是活跃的，因此贡献从第一个索引开始增长。

 ## 边缘情况

 一个关键的边缘情况是怪物可以立即被杀死。 这对应于`h <= sum(a)`。 在这种情况下，正确答案是`0`，并且二分查找必须正确允许`j = 0`作为有效候选人。 

另一种情况是所有差异都很大，这意味着不会发生早期激活。 然后函数增长非常缓慢，直到第一个激活点，并且解决方案必须避免跳过`j = 0`到`j = min(d_i)`错误地。 

大的`h`值测试所有激活开始贡献后线性增长阶段的正确性，其中每个索引都处于活动状态并且函数变为密集线性表达式`j`。
