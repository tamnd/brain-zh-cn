 ## 解决方案
 ## 问题理解

 我们有固定的培训天数和目标总培训时数。 每天贡献正常的训练时间或增加的训练时间。 正常的一天总是提供 24 小时的训练，而特殊的“超级日”则提供 72 小时的训练。 任务是选择应将多少可用天数转变为超级日，以便所有天的累计训练时间总和至少达到所需的目标，同时保持超级日的数量尽可能少。 

输入由两个数字组成：可用天数和所需的总培训时数。 输出是一个整数，表示必须升级到超级天才能满足或超过目标总数的最小天数。 

所需时间不得超过天数 72 倍的约束保证了可行性。 即使每天都是超级日，目标也总能达到。 

从复杂性的角度来看，这两个值都可以高达 100,000。 这排除了任何尝试使用递归或穷举搜索来模拟每天超级日的所有可能选择的方法，因为这会随着天数呈指数增长。 即使是数天的二次循环在孤立的情况下也是安全的，但没有必要。 问题的结构表明我们正在优化单个整数参数，因此预计会出现封闭形式或贪婪推理。 

当目标无需任何超级日即可实现时，就会出现打破天真的推理的微妙情况。 例如，如果有 5 天，目标是 100 小时，则正常时间表已经给出 120 小时。 在这种情况下，正确答案为零，任何盲目计算正要求的公式都是错误的。 

当目标恰好等于正常总数时，会出现另一种边缘情况。 例如，如果时间为 3 天，目标为 72，则不需要超级日。 这些边界条件很重要，因为解决方案自然涉及可能为负的减法。 

## 方法

 蛮力的想法是尝试从 0 到 d 的所有可能的超级日数，计算每个选择的总训练时间，并选择最小的有效值。 对于每个候选 x，总时间计算为 24(d − x) + 72x。 检查所有可能性需要 O(d) 次评估，每次评估都是 O(1)，因此总体复杂度是 O(d)。 当 d 达到 100,000 时，这仍然是可以接受的，但它是不必要的，并且在概念上比需要的更重。 

关键的观察结果是，用超级日替换正常日并不会改变日程安排的结构，它只会增加固定的额外训练量。 正常一天贡献 24 小时，而超级日贡献 72 小时，因此与正常一天相比，升级一天的净收益是额外 48 小时。 这将问题转化为一个简单的增量增益场景：我们从正常的日子开始，计算基线总计，然后询问需要多少 48 小时增量才能达到目标。 

这将问题简化为找到最小整数 x，使得 24d + 48x ≥ h。 解决这个不等式直接给出了使用上限除法的封闭形式表达式，从而消除了几天内进行任何模拟的需要。

 ## 算法演练

1. 如果所有天数都正常，则计算总训练时间，即 24 × d。 这代表没有任何升级的基准容量。 
2. 如果该基线已达到或超过目标 h，则返回 0，因为不需要超级日。 
3. 否则计算剩余赤字，定义为 h − 24 × d。 这是升级某些天必须提供的额外小时数。 
4. 与正常一天相比，每个超级日会额外贡献 48 小时，因此请确定需要多少这样的增量来弥补不足。 计算结果为满足 48 × x ≥ 赤字的最小整数 x。 
5. 返回 x 作为答案。 

正确性来自于这样一个事实：每个有效的时间表都可以转换为所有非超级日都是正常日而所有超级日都是相同升级的时间表。 由于每次升级都有相同的边际效益，并且各天之间没有交互作用，因此问题变成了单个变量的简单线性覆盖问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    d, h = map(int, input().split())
    
    base = 24 * d
    if base >= h:
        print(0)
        return
    
    deficit = h - base
    # each superday adds 48 extra hours compared to normal day
    x = (deficit + 48 - 1) // 48
    print(x)

if __name__ == "__main__":
    solve()
```实现直接反映了导出的公式。 第一步计算基线总计。 提前返回处理不需要升级的重要边界情况。 剩余的计算执行上限除法，使用整数算术实现以避免浮点不准确。 表达式`(deficit + 47) // 48`确保当赤字不能被 48 整除时正确舍入。

 ## 边缘情况

 一个重要的边界是在无需升级的情况下已满足要求的情况。 用于输入`d = 2, h = 40`，基线为 48 小时。 该算法在比较步骤后立即返回零，因为没有计算赤字。 这避免了负算术传播到上限公式中的任何风险。 

另一种情况是赤字小于单个超级日捐款。 例如`d = 1, h = 50`，基线为 24，赤字为 26。由于一个超级日增加了 48 小时，上限除法产生 1，正确选择了单个升级。 

当每天都必须升级时，就会出现最终的结构情况。 为了`d = 3, h = 216`，基线为 72，赤字为 144。每个超级日贡献 48，因此需要 3 个超级日。 公式自然饱和`d`当目标接近最大可能的训练总量时。
