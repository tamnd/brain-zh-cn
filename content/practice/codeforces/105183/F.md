 ## 解决方案
 ## 问题理解

 我们得到一个索引从 1 到 n 的数组，其中每个位置代表一条街道，每条街道都有一个正值 a[i] 描述其“强度”或“大小”。 

我们需要选择索引 i1 < i2 < ... < ik 的子序列。 所选索引必须遵守每个连续对之间的间距规则：两个所选位置之间的距离必须至少是两个相应值中较小的一个。 换句话说，只有当指数差距足够大以“容纳”两条街道中较弱的街道时，才允许从一条选定的街道转移到另一条街道。 

任务是在尊重这一规则的同时最大化我们可以选择的指数数量。 

约束 n ≤ 10^6 意味着对索引对的任何二次方法都是立即不可能的。 即使 O(n log n) 解决方案也需要仔细构建，因为我们无法显式存储或处理对之间的所有边。 任何解决方案都必须在线性或近线性时间内有效地本地决定是否包含每个位置或跳过它，而无需重新访问许多早期状态。 

一个微妙的困难来自于约束取决于值 a[i] 和 a[j]。 这排除了标准的“纯”间隔调度公式，其中约束仅取决于一个端点。 

一种简单的方法可能会尝试对所有先前选择的元素进行动态编程，但这需要检查与许多早期状态的兼容性，并且无法扩展。 

经常打破天真的贪婪直觉的一种边缘情况是当大值聚集在一起时。 例如，如果一个段的值为 [5, 6, 7, 8]，连续选择就变得不可能，因为相邻索引仅相差 1，该值小于 min(5,6)、min(6,7) 等。任何假设“高值总是好的”的错误贪心方法在这里都会失败，因为获取一个大元素可能会阻止许多未来的选择。 

另一个棘手的情况是小值和大值交替。 例如，[1, 100, 1, 100, 1] 允许选择几乎每个索引，但过早跳过小值的幼稚策略可能会不必要地降低未来的灵活性。 

## 方法

 暴力解决方案会尝试每个有效的子序列。 对于每个当前子序列，我们尝试使用满足 j - i ≥ min(a[i], a[j]) 的每个可能的下一个索引 j > i 来扩展它。 这立即变成指数，因为每个元素分支成潜在的许多有效延续，甚至检查转换也是 O(n^2)。 

一种自然的改进是动态规划，其中 dp[i] 是以 i 结尾的最大子序列长度。 为了计算 dp[i]，我们将扫描所有 j < i 并检查 j 是否可以连接到 i。 这是正确的，但成本为 O(n^2)，这对于 n 高达 10^6 来说太慢了。 

关键的结构观察是过渡的可行性仅取决于最后选择的指数。 一旦我们确定了最后选择的位置 j，每个候选 i 都会针对 j 进行独立评估。 不需要记住 j 之前的任何内容，因为扩展子序列的条件纯粹是最后一个元素的本地条件。 

这表明我们实际上并不是在解决所有先前状态的一般DP，而是在有向无环图中构建单个最长链，其中边对局部兼容性条件进行编码。 在这种情况下，通常可以通过始终采用最早的有效扩展来贪婪地找到最佳链，因为延迟选择永远不会提高未来的可行性。 

这减少了为最后选择的元素维护单个指针并向前扫描一次，决定每个位置是否可以扩展当前链的问题。

 ## 算法演练

 我们从左到右贪婪地构建子序列，始终维护最后选择的索引。 

1. 从不选择任何元素开始，并将答案长度初始化为零。 
2. 按升序迭代从 1 到 n 的索引。 
3. 如果尚未选择索引，则选择第一个索引作为子序列的开始。 这是安全的，因为没有事先的限制。 
4. 对于每个下一个索引 i，测试它是否可以遵循最后选择的索引 j。 条件是 i - j 必须至少为 min(a[j], a[i])。 如果成立，我们将 i 附加到子序列并将 j 更新为 i。 
5. 如果条件不成立，我们跳过 i 并继续。 跳过是必要的，因为采用 i 会违反最后选择的元素的间距规则。 
6. 继续直到数组末尾，计算我们成功延长链的次数。 

关键的想法是该算法永远不会重新考虑之前的决策。 一旦跳过索引，就永远不会重新访问它，并且一旦采用索引，它就成为所有未来决策的唯一参考点。 

### 为什么它有效

 子序列约束仅取决于相邻的选定元素，因此整个结构形成一条链，其中每个链接都针对前一个链接进行独立验证。 当我们处于固定的最后选择的位置 j 时，任何有效的扩展 i 仅取决于 j 和 i，而不取决于早期历史。 在所有可行的扩展中，选择最早的扩展并不能减少未来选项的数量，因为未来的可行性仅取决于最后选择的索引，与推迟选择具有可能更大的 a[i] 的较晚索引相比，采用较小的索引总是会为后续步骤产生较小或相等的间隙要求。 这可以防止跳过有效扩展而选择后续扩展的任何优势。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    last = -1
    ans = 0

    for i in range(n):
        if last == -1:
            last = i
            ans += 1
        else:
            if i - last >= min(a[i], a[last]):
                last = i
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```代码维护一个指针`last`，它跟踪子序列中最近选择的索引。 对于每个新索引，我们使用与问题陈述完全相同的条件来检查扩展链的有效性。 如果有效，我们立即提交，这对应于贪婪地构建最大链。 

一个常见的实现陷阱是忘记问题陈述中的索引是从 1 开始的，而 Python 使用从 0 开始的索引。 由于条件仅取决于差异，因此只要双方一致，移动索引就不会影响正确性。 

另一个微妙的点是，我们永远不需要存储子序列本身，只需要存储它最后选择的位置和长度。

 ## 边缘情况

 一种重要的边缘情况是所有值都相等且很大。 例如，`n = 4, a = [10, 10, 10, 10]`。 该算法仅选择第一个元素，因为每个连续对都需要至少 10 的距离，但索引仅相距 1。 贪婪过程正确地拒绝了自条件以来的每个后续索引`i - last >= 10`每一步都失败。 

另一种情况是非常小和非常大的值的混合。 为了`a = [1, 100, 1, 100, 1]`，该算法会交替选择，因为每个具有较小值的相邻对都会强制最小距离为 1，而这始终是满足的。 这表明大值本身不会减少链长度，除非它们相邻或紧密堆积。 

最后一种边缘情况是，在长时间运行小值之后，大值出现较晚。 如果间隙条件与最后选择的索引一致，则算法将接受它，这表明该决策始终是纯粹本地的，并且不依赖于先前跳过的机会。
