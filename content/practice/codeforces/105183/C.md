 ## 问题理解

 我们得到一个长度的数组$n$。 对于任何连续的段，我们查看每个不同值在该段内出现的次数。 这会产生多组频率。 当这些频率本身形成排列时，该段被称为“特殊”，这意味着如果列出所有非零频率，它们包含来自$1$最多恰好一次的不同值的数量。 

对于每一个可能的值$i$，我们必须计算到底有多少个段$i$不同的值并满足此频率排列条件。 

输出是一个序列，其中$i$-th 数字告诉我们有多少个段$i$不同的元素，其频率多重集是以下的排列$\{1, 2, \dots, i\}$。 

约束条件$n \le 2 \cdot 10^5$排除任何枚举所有的解决方案$O(n^2)$从头开始分段并计算频率。 甚至$O(n^2 \log n)$是不可能的。 我们需要一种方法，其中每个部分都不是独立处理的，而是汇总或间接计算贡献。 

一个微妙的问题是值范围可达$10^9$，因此坐标压缩对于有效维护频率数组或图是必要的。 

一个常见的陷阱是误解条件：频率不是任意的，它们必须恰好是以下的排列：$1 \dots k$。 例如，像这样的频率$[1, 1, 2]$失败，因为存在重复项和缺失值$3$。 另一种失败情况是假设“所有频率都是不同的”就足够了，但这是不正确的，因为$[1, 3, 5]$对于尺寸 3 仍然会失败。 

## 方法

 蛮力的想法很简单：枚举每个段$[l, r]$，计算其中所有值的频率，提取非零频率列表，并检查它是否是排列。 这是正确的，因为它直接遵循定义。 然而，维护每个细分市场的频率图会产生成本$O(n)$，导致$O(n^3)$最坏情况下的总计，或$O(n^2)$增量更新，对于$2 \cdot 10^5$。 

关键的观察结果是“频率形成排列”这一约束极其严格。 如果一个段有$k$不同的值，那么频率之和必须等于段长度，并且必须恰好是以下的重新排列$1, 2, \dots, k$。 这意味着段长度一旦固定$k$被选择： 必须是$$1 + 2 + \dots + k = \frac{k(k+1)}{2}.$$所以任何有效的段$k$不同值的长度必须精确$T_k = k(k+1)/2$。 这立即限制了$k$经过$O(\sqrt{n})$，因为较大$k$使$T_k > n$。 

这将问题从任意长度的滑动窗口转换为目标窗口长度的有界族。 对于每一个可能的$k$，我们只需要计算段的长度$T_k$其频率多重集满足排列条件。 

现在的任务变成：对于固定的窗口大小，计算有多少窗口满足非常结构化的频率约束。 这可以使用滑动窗口进行检查，同时维护频率表并跟踪每个频率有多少个值。 排列条件可以增量地维持而不是重新计算。 

这将问题减少到$O(\sqrt{n} \cdot n)$，这是可行的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 每 k 的滑动窗口 |$O(n \sqrt{n})$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过坐标压缩对数组进行预处理，以便频率可以存储在由紧凑值索引的数组中。 

对于每个$k \ge 1$，我们计算目标窗口大小$T_k = k(k+1)/2$。 如果$T_k > n$，我们停下来。 

然后我们滑动一个长度的窗口$T_k$遍历数组，维护频率信息并检查当前窗口是否满足排列条件。 

### 步骤

 1. 将数组值压缩到一个范围内$[0, m-1]$。 这允许直接索引频率数组而无需哈希映射，从而保持更新$O(1)$。 
2. 对于每个$k \ge 1$, 计算$T_k = k(k+1)/2$。 如果$T_k > n$，中断，因为这种类型的有效段不能存在。 
3.初始化频率数组`cnt[val] = 0`以及辅助结构`freqCount[f]`跟踪当前有多少个值具有频率$f$。 这种结构使我们能够知道频率是否准确形成$1..k$无需扫描所有值。 
4. 构建第一个大小的窗口$T_k$。 对于每个元素，更新其频率：减少其旧频率类别`freqCount`，增加其新的频率类别，并更新`distinct`计数频率从 0 到 1 的时间。 
5. 通过验证以下内容来检查当前窗口是否有效：

 不同元素的数量恰好是$k$,

 以及对于每一个$j \in [1, k]$, 恰好存在一个具有频率的值$j$。 

第二个条件使用以下方法验证`freqCount[j] == 1`。 
6. 通过删除左侧元素并添加下一个右侧元素来滑动窗口。 每次更新都会调整`cnt`和`freqCount`在恒定的时间内。 
7. 对于每个窗口位置，如果满足条件，则增加答案$k$。 

### 为什么它有效

 关键的不变量是在每一步中，`cnt`正确表示当前窗口中的频率，并且`freqCount[f]`准确跟踪当前有多少不同值的频率$f$。 由于更新是按元素插入和删除自动完成的，因此不会保留过时的计数。 

当窗口的频率分布与排列相匹配时，窗口就被准确计数$1..k$。 该条件通过以下方式强制执行`freqCount`确保完整性（所有频率 1..k 都出现）和唯一性（每个频率只出现一次），它们一起定义所需的结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # coordinate compression
    vals = {v: i for i, v in enumerate(sorted(set(a)))}
    a = [vals[v] for v in a]
    m = len(vals)

    ans = [0] * (n + 1)

    for k in range(1, n + 1):
        need_len = k * (k + 1) // 2
        if need_len > n:
            break

        cnt = [0] * m
        freq_cnt = [0] * (need_len + 2)

        distinct = 0

        def add(x):
            nonlocal distinct
            old = cnt[x]
            if old > 0:
                freq_cnt[old] -= 1
            cnt[x] += 1
            new = cnt[x]
            freq_cnt[new] += 1
            if old == 0:
                distinct += 1

        def remove(x):
            nonlocal distinct
            old = cnt[x]
            freq_cnt[old] -= 1
            cnt[x] -= 1
            new = cnt[x]
            if new > 0:
                freq_cnt[new] += 1
            if new == 0:
                distinct -= 1

        for i in range(need_len):
            add(a[i])

        def ok():
            if distinct != k:
                return False
            for f in range(1, k + 1):
                if freq_cnt[f] != 1:
                    return False
            return True

        if ok():
            ans[k] += 1

        for i in range(need_len, n):
            add(a[i])
            remove(a[i - need_len])
            if ok():
                ans[k] += 1

    print(*ans[1:])

if __name__ == "__main__":
    solve()
```该实现遵循每个固定的滑动窗口$k$。 频率阵列`cnt`是每个本地的$k$，避免了不同尺寸窗户之间的交叉污染。 这`freq_cnt`数组跟踪当前有多少个值具有每个频率，这是避免重新计算分布的核心优化。 

功能`ok()`故意简单但被称为$O(n)$每次$k$，这是可以接受的，因为$k$只上升到$O(\sqrt{n})$。

 | 测量| 复杂性 | 说明|

 | 时间 |$O(n \sqrt{n})$| 对于每个$k$，我们在数组上滑动一个窗口，并且$k$只上升到$\sqrt{n}$由于窗口尺寸呈三角形增长|

 ## 边缘情况

 关键的边缘情况是所有值都相同。 用于输入`[5,5,5,5]`， 仅有的$k=1$可以工作，因为任何更大的$k$需要多个不同的值，这是无法满足的。 该算法可以正确处理这个问题，因为`distinct != k`所有人立即失败$k > 1$。 

另一个边缘情况是值完美交替时，例如`[1,2,1,2,1,2]`。 这里多个大小为 3 的窗口满足$k=2$，频率跟踪可确保重叠窗口全部独立计数，不会重复计数或丢失转换。 

最后一个边缘情况是$T_k$接近于$n$，其中仅存在一个或零个窗口。 滑动窗口仍然表现正确，因为初始化构建了第一个完整段，并且循环根本不执行或执行一次，不需要特殊的外壳。
