 ## 解决方案
 ## 问题理解

 我们得到一个固定的单词词典，然后是一系列查询字符串。 对于每个查询，我们想知道字典中有多少单词以该查询作为前缀开头。 如果单词以查询字符串作为起始段，则该单词会被计数，即使它与查询完全相等或比查询更长。 

输入大小清楚地表明，字典和查询列表最多可以包含十万个字符串，并且所有字符串的字符总数以一百万为界。 这排除了任何每次查询重复扫描完整字典的方法。 将每个查询与每个单词逐个字符进行比较的简单检查将需要大约$10^{10}$最坏情况下的比较，远远超出了Python中一秒的运行速度。 

一个更微妙的约束来自字符串长度分布。 即使单个字符串很短，比较次数仍然会随着$N \cdot Q$，这是主要瓶颈。 任何可接受的解决方案都必须在预处理后将每个查询最多减少到对数或恒定时间工作。 

一些边缘情况对于正确性很重要。 如果查询字符串比所有字典单词长，则答案必须为零。 例如，如果字典包含“a”、“ab”、“abc”并且查询是“abcd”，则答案为零，因为没有单词可以匹配该前缀。 另一种情况是字典中存在多个相同的单词。 每次出现都应该单独计算，因为问题要求的是单词数，而不是不同的单词。 最后，输入中不存在空前缀，因此我们不需要显式处理这种情况。 

## 方法

 最直接的想法是通过扫描每个字典单词并检查它是否以查询开头来处理每个查询。 这是正确的，因为它直接测试前缀的定义。 问题是成本。 每次检查可能需要比较查询的完整长度，因此一次查询会花费$O(N \cdot L)$，并且在所有查询中这都变得太慢了。 

关键的观察是，如果字典按字典顺序排序，前缀查询就会变得容易。 按排序顺序，共享前缀的所有单词形成一个连续的块。 发生这种情况是因为字典顺序按最早的不同字符对字符串进行分组。 排序后，回答查询就只剩下查找具有该前缀的单词可能出现的第一个位置以及单词停止匹配该前缀的第一个位置。 

这将问题简化为两次二分搜索。 一次搜索找到左边界，即不小于前缀的第一个单词。 第二次搜索找到正确的边界，可以通过搜索严格大于任何以前缀开头的字符串的最小字符串来模拟。 标准技巧是将大于“z”的字符附加到前缀，例如“{”，因为 ASCII 排序保证所有小写字符串都位于它之前。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N \cdot Q \cdot L)$|$O(1)$额外 | 太慢了|
 | 排序 + 二分查找 |$O((N+Q)\log N)$|$O(1)$超出存储范围的额外 | 已接受 |

 ## 算法演练

 我们依赖于按字典顺序排列的字符串的排序和二分搜索边界。 

1. 阅读所有字典单词并按字典顺序排序。 这会将具有相同前缀的所有单词分组为连续的段，这使得范围计数成为可能。 
2. 对于每个查询字符串，使用二分搜索计算左边界。 这会找到排序数组中单词不小于查询字符串的第一个索引。 
3. 通过将严格大于任何小写字母的字符（例如“{”）附加到查询来构造辅助字符串。 这定义了共享前缀的所有单词的上限。 
4. 在此修改后的字符串上使用二分搜索计算右边界。 这会找到以前缀开头的所有单词之后的第一个索引，其中单词不小于下一个词典存储桶。 
5. 查询的答案是右边界和左边界之间的差。 此差异精确计算前缀与查询匹配的字典单词的数量。 

### 为什么它有效

 排序保证所有具有共同前缀的字符串按字典顺序占据连续的区间。 左边界隔离了第一个可以匹配前缀的候选者。 人工上限字符串确保以前缀开头的每个字符串都严格小于它，因此右边界立即在最后一个匹配字符串之后停止。 因此，减法会精确计算该连续块的大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    words = [input().strip() for _ in range(n)]
    words.sort()

    import bisect

    out = []
    for _ in range(q):
        s = input().strip()
        left = bisect.bisect_left(words, s)
        right = bisect.bisect_left(words, s + '{')
        out.append(str(right - left))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```核心实现取决于Python的字典字符串比较。 对列表进行一次排序可确保二分搜索在字符串顺序上的行为一致。 功能`bisect_left(words, s)`找到第一个位置`s`可以在不破坏顺序的情况下插入，它恰好对应于不小于的第一个单词`s`。 

第二个边界使用`s + '{'`，它严格大于以任何开头的小写字符串`s`。 这避免了计算复杂的下一个前缀函数。 一个常见的错误是尝试`s + 'z'`，当前缀已经以“z”结尾时，该操作会失败。 使用“{”可以干净地避免所有此类边缘情况。

 ## 边缘情况

 一种边缘情况是所有字典单词都相同。 对于所有单词均为“a”并且查询为“a”的输入，两个边界都会折叠到整个数组范围，因此结果正确地等于出现次数。 

另一个边缘情况是当查询按字典顺序大于任何字典单词时。 例如，字典 [“a”，“b”] 和查询“z”。 两个二分搜索边界都返回数组的末尾，无需特殊处理即可生成零。 

最后一种情况是查询本身是字典中存在的完整单词。 由于左边界包含相等性，因此可以正确计算匹配单词，而右边界仅排除不匹配的扩展名，从而确保结果中包含精确匹配。
