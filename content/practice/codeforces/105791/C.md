 ## 解决方案
 ## 问题理解

 海滩大道上有一排售货亭，每个售货亭只提供一种饮料。 旅行者从满意度值 1 开始，选择一个连续的自助服务终端段进行步行。 在浏览该部分时，他将当前的满意度乘以按顺序每杯饮料的价值。 

每个饮料值要么为零，要么是 2 的有符号幂，这意味着对于某个整数 k，它可以写为 0 或 ±2^k。 零值代表一种特殊饮料，它会立即将满意度重置为零并有效地打破乘法链。 负值代表“坏”饮料，会以通常的乘法方式翻转当前满意度的符号，但如果当前满意度已经是负值，则问题陈述强调喝另一个负值会恢复正值，这与正常的乘法规则一致。 

目标是选择单个连续段的开始和结束位置（或选择根本不进入），以使最终满意度最大化。 答案必须以 1e9 + 7 为模打印，但结果的比较是在取模之前对实际整数值进行的。 

约束 n 最大可达 200,000，这会立即排除所有段的任何 O(n^2) 枚举。 任何解决方案都必须是线性或接近线性的，因为二次扫描在最坏的情况下需要 4e10 次运算，而这在一秒钟内是不可行的。 

一个微妙的点是零完全分割了这个过程。 任何跨越零的段都会在该点将其乘积重置为零，因此最佳段永远不需要跨越零。 

另一个重要的边缘情况是，旅行者根本不可以乘坐任何航段，保持满意度为 1。这条基线很重要，因为任何负乘积在实际整数排序中总是比 1 差，即使在取模后它可能看起来很大。 

一个天真的错误是将最终结果解释为纯粹对 1e9 + 7 求模，并意外地选择模余数较大的负积。 这是不正确的，因为最大化发生在取模之前。 

## 方法

 一种直接的方法是尝试每一个可能的起始位置并将其扩展到每一个可能的结束位置，从而维持运行的产品。 这可以正确计算每个子数组的乘积。 然而，每次扩展都是 O(1)，并且有 O(n^2) 个子数组，导致最坏情况下大约需要 2e10 次操作，这太慢了。 

问题的结构表明我们只需要跟踪在每个位置结束的最佳产品。 这是经典的“最大乘积子数组”想法，但有两个复杂之处：零重置所有内容，并且值是 2 的有符号幂，这使得跟踪幅度相当于跟踪指数和。 

我们不跟踪实际产品，而是跟踪 2 的指数。 值 ±2^k 将 k 贡献给指数，而符号则单独跟踪。 乘法变成指数加法和符号翻转。 

这将问题简化为以指数和的形式维持每个位置的最佳正乘积和最佳负乘积。 零重置这两种状态并允许从头开始。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有子数组进行暴力破解 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | DP 正/负指数态 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理数组，维持两种运行状态：以当前位置结束的最佳正积，以及以当前位置结束的最佳负积。 我们还为任何有效细分维持全球最佳指数。

我们用 2 的指数来表示乘积，因为所有值都是 2 的幂。 

### 步骤

 1.将最佳答案指数初始化为0，代表值为1的空选择。同时将当前的正负状态初始化为空。 
2. 对于数组中的每个元素，根据其值进行处理。 
3. 如果该值为零，则重置这两个状态。 这是因为任何经过零的段作为乘法链都变得无效。 重置后，将全局答案更新为 0，因为我们总是可以在零之后重新开始。 
4. 如果值为正 2^k，则通过从该元素重新开始或通过将 k 添加到其指数来扩展先前的正状态来更新正状态。 负状态（如果存在）也会通过添加 k 来扩展，因为乘以正数会保留符号。 
5. 如果值为负 -2^k，则通过扩展先前的负状态（因为负乘以负变为正）来计算新的正候选，并通过扩展先前的正状态或从新开始作为单个负元素来计算新的负候选。 
6. 处理完每个元素后，用当前正指数更新全局最佳值，因为只有正结果才能在实际值排序中击败基线 1。 

### 为什么它有效

 在任何位置，以该位置结束的每个有效子数组都必须来自两个历史记录之一：要么从前一个索引开始，要么从最近的零开始。 民主党国家恰恰抓住了这些可能性。 分离正态和负态就足够了，因为乘以 ±2^k 只会影响符号并向指数添加固定量，因此不需要其他信息。 零充当硬重置，防止跨段状态的交叉污染。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    best_exp = 0
    
    pos = None
    neg = None
    
    for x in a:
        if x == 0:
            best_exp = max(best_exp, 0)
            pos = None
            neg = None
            continue
        
        sign = 1 if x > 0 else -1
        k = x.bit_length() - 1 if x != 0 else 0
        
        new_pos = None
        new_neg = None
        
        if sign > 0:
            if pos is not None:
                new_pos = max(new_pos or 0, pos + k)
            else:
                new_pos = k
            
            if neg is not None:
                new_neg = neg + k
        
        else:
            if neg is not None:
                new_pos = max(new_pos or -10**30, neg + k)
            else:
                new_pos = k
            
            if pos is not None:
                new_neg = pos + k
            else:
                new_neg = k
        
        pos = new_pos
        neg = new_neg
        
        if pos is not None:
            best_exp = max(best_exp, pos)
    
    print(pow(2, best_exp, MOD))

if __name__ == "__main__":
    solve()
```该实现将每个值压缩为符号和指数。 使用位长度提取指数，因为每个非零值恰好是 ±2^k。 DP 维护两个运行状态，并为每个元素就地更新。 全局最佳值仅跟踪正指数值，因为只有那些对应于可以超过基线 1 的有效候选者。 

一个常见的实现陷阱是忘记零必须完全重置两个状态。 另一个微妙的问题是错误地允许负结果状态影响最终答案，这会错误地优先考虑大量的负结果。

 ## 边缘情况

 一个关键的边缘情况是整个数组由零组成。 在这种情况下，每个段都是无效的，正确的答案是基线 1。算法会处理这个问题，因为每个零都会强制重置，并明确允许用 0 更新最佳值，而最终答案至少保持 0 指数，对应于 1。 

另一种边缘情况是一系列不带零的交替符号，例如 [-2, 2, -2]。 DP 正确跟踪正态和负态，确保当第二个负态出现时，偶数长度的负态链被转换为正候选链。 

最后一个微妙的情况是单个元素数组。 如果为正，答案就是该元素的值。 如果是负数，最好的仍然是1，除非可以配对，这是不可能的，因此DP正确地避免了选择有害的单个负数段。
