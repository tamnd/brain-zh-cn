 ## 解决方案
 ## 问题理解

 两名玩家轮流爬椰子树。 在第一次攀爬时，Samuel 采取了固定的量$x$，在第二次攀爬时，Lleumas 采取了$y$。 从第三次攀登开始，每个玩家都会改变他们的策略：在每次新的攀登中，一名玩家准确地采取他们在上一次攀登中采取的行动以及其他玩家在上一次攀登中采取的行动。 

这定义了两个不断演变的值序列。 一个序列对应于 Samuell 的产量，另一个对应于 Lleumas 的产量。 任务不是直接模拟这些值，因为爬升次数$n$可以非常大，最多$10^{18}$。 相反，我们必须确定由谁执行$n$- 第次攀登以及他们在该攀登中采摘了多少椰子，取模$10^9 + 7$。 

交替结构立即给出了登山者的身份：奇数索引对应于 Samuell，偶数索引对应于 Lleumas。 真正的困难在于计算多次转换后的值而不是逐步迭代。 

一个简单的模拟将更新每个步骤的两个序列，直到$n$。 这是不可能的$n = 10^{18}$，因为即使$10^7$行动已经处于严格限制的边缘。 递归必须折叠成封闭形式。 

边缘情况出现时$n = 1$或者当一个或两个初始值为零时。 例如，如果$x = 0$和$y = 5$，那么第二步立即变成$5$，之后一切都会确定性地增长。 一种粗心的尝试，假设所有的东西都有一个统一的公式$n$也会错误地对第一步应用求幂。 

另一个微妙的情况是$n = 2$，其中该值仍处于初始过渡阶段，尚未进入指数倍增状态。 

## 方法

 一种直接的方法是逐步模拟该过程。 我们维护两个变量，一个用于 Samuell，一个用于 Lleumas，并重复应用递归。 每个新值仅取决于两个玩家之前的值。 这是正确的，因为它完全遵循定义。 

然而，这需要$O(n)$过渡。 自从$n$可以大到$10^{18}$，即使是单个线性通道也是不可能的。 状态空间的复杂性并没有增加，但步骤的数量使得模拟变得不可行。 

关键的观察结果是，在第二步之后，两个序列变得相同。 在第 2 步我们得到$x + y$对于两名球员来说。 从那时起，每个新值只是前两个相同值的总和，这意味着序列每一步都会加倍。 

这将问题转化为常数前缀后的简单指数增长。 我们不是模拟转换，而是使用快速求幂来计算 2 的幂。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(n)$|$O(1)$| 太慢了 |
 | 封闭形式+快速动力|$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们分别处理前两个步骤，然后处理指数状态。 

1.如果$n = 1$，答案直接是Samuell with value$x$，因为尚未发生复发。 这纯粹是一个基本定义案例。 
2.如果$n = 2$，答案是具有值的 Lleumas$x + y$，因为第二步是直接根据第一对值计算的。 
3. 对于所有人$n \ge 3$，我们观察到从第 2 步开始两个序列变得相同。 我们定义一个统一的序列$T$这样$T_2 = x + y$。 
4. 对于任何步骤$i \ge 3$，递归变为$T_i = T_{i-1} + T_{i-1}$，因为两个玩家现在贡献了相同的先前值。 这简化为$T_i = 2 \cdot T_{i-1}$。 
5. 这意味着$T_n = (x + y) \cdot 2^{n-2}$为所有人$n \ge 2$。 
6.我们计算$2^{n-2} \bmod (10^9 + 7)$使用快速求幂，然后乘以$(x + y) \bmod (10^9 + 7)$。 
7. 最后，我们根据奇偶性决定输出名称$n$： 奇怪的$n$甚至给塞缪尔$n$给了卢马斯。 

### 为什么它有效

 关键的不变量是从步骤 2 开始，两个序列在每个索引处都相等。 一旦建立了相等性，递归就会折叠成单个标量序列，其中每一项恰好是前一项的两倍。 由于递归不会再次引入不对称性，因此为所有后续步骤保留相等属性，从而迫使单个共享值以基数 2 呈指数增长。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def mod_pow(a, e):
    res = 1
    while e > 0:
        if e & 1:
            res = (res * a) % MOD
        a = (a * a) % MOD
        e >>= 1
    return res

n = int(input().strip())
x, y = map(int, input().split())

if n == 1:
    print("Samuell")
    print(x % MOD)
elif n == 2:
    print("Lleumas")
    print((x + y) % MOD)
else:
    total = (x + y) % MOD
    val = total * mod_pow(2, n - 2) % MOD

    if n % 2 == 1:
        print("Samuell")
    else:
        print("Lleumas")
    print(val)
```实现隔离了特殊情况$n = 1$和$n = 2$，因为指数公式仅从步骤 2 开始有效。 快速求幂函数用于计算$2^{n-2}$有效地进行模运算，确保非常大的正确性$n$。 

最后的奇偶校验直接对交替匝结构进行编码，避免了任何匝的模拟。

 ## 边缘情况

 当$n = 1$，根本没有应用递推式，因此任何涉及的公式$x + y$会错误地多算。 该算法明确绕过递归和输出$x$。 

什么时候$n = 2$，系统只看到了两个玩家之间的一次互动。 这是值尚未受指数结构控制的唯一步骤。 该算法直接计算$x + y$，避免过早倍增。 

什么时候$x = 0$和$y = 0$，无论幂如何，每个后续值都保持为零。 该算法自然地处理了这个问题，因为即使在模运算下，乘以零也会保留零。 

什么时候$n$是非常大的，比如$10^{18}$，奇偶校验决策和快速求幂保持稳定，因为它们仅依赖于指数上的模算术和位运算，而不依赖于迭代$n$。
