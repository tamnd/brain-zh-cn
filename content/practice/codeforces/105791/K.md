 ## 解决方案
 ## 问题理解

 墙被指定为 n × m 的单元网格。 每个单元格要么是空的，要么被钉子堵住。 海报只能挂在墙上的矩形区域，并且该矩形内的每个单元格都必须没有钉子。 海报不能以改变其轴对齐的方式任意旋转，因此它始终对应于网格中的轴对齐矩形。 

任务是找到仅由无指甲细胞组成的最大可能矩形。 

网格大小最多为 250 x 250，该大小足够小，使得行或列上的二次解可行，但对于直接尝试所有 O(n²m²) 矩形的任何方法来说太大。 在最坏的情况下，对所有子矩形进行暴力破解将涉及大约 10^10 种可能性，这远远超出了 1 秒所允许的范围。 

一个微妙的点是输入编码与直觉相反：`.`代表钉子（被阻塞），而`#`代表一个空的壁细胞。 所以我们想要的矩形是最大的`#`子矩阵。 

当网格完全被阻塞或完全空闲时，就会出现典型的边缘情况。 如果一切都是`.`，答案为零，因为不存在有效的矩形。 如果一切都是`#`，答案是n·m，因为整面墙都可以使用。 

## 方法

 一种直接的方法是考虑每一个可能的左上角和右下角坐标对，并验证它们之间的矩形是否只包含`#`。 这需要使用前缀和或扫描检查每个矩形的 O(1) 个单元格，并且由于存在 O(n²m²) 个矩形，因此总工作量变为 O(n²m²)，在 250 x 250 网格中约为 4×10^10 检查。 即使进行了优化，这也是不可行的。 

关键的观察结果是，这个问题与寻找二进制矩阵中最大的矩形相同，其中`#`被视为 1 且`.`为 0。我们可以将每一行重新解释为直方图的基础，而不是直接推理 2D 矩形。 对于每一行，我们维护一个高度数组，其中 height[j] 计算有多少个连续的`#`单元格结束于第 j 列的第 i 行。 然后每一行将问题转化为寻找直方图中最大的矩形。 众所周知，直方图问题可以使用单调堆栈在每行的线性时间内解决。 

这种缩减是有效的，因为每个矩形都有一个底部边界。 固定矩形的底行，在那里结束的最佳矩形仅取决于每列可以向上延伸多远而不碰到`.`，这正是高度数组编码的内容。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有矩形 | O(n²m²) | O(1) | O(1) | 太慢了 |
 | 每行直方图 + 堆栈 | O(纳米) | O(米) | 已接受 |

 ## 算法演练

 1. 将网格转换为二进制解释，其中`#`是可用空间并且`.`被阻止。 这为所有后续计算建立了基础表示。 
2. 维护数组`height`大小为 m，最初全为零。 该数组表示逐行构建的直方图。 
3. 从上到下迭代每一行 i。 对于每一列 j，更新`height[j]`。 如果细胞是`#`, 增量`height[j]`减 1，否则将其重置为 0。这可确保`height[j]`始终反映以当前行结束的连续空闲单元的数量。 
4. 更新第 i 行的直方图后，计算该直方图中的最大矩形面积。 这是使用单调递增的索引堆栈来完成的。 堆栈确保按递增顺序处理高度，以便每个条形成为跨越其有效范围的最大矩形的限制高度。 
5. 处理直方图时，每当当前高度小于栈顶高度时，从栈中弹出，并以弹出的高度作为限制因素计算矩形面积。 宽度由当前索引和新的堆栈顶部确定。 
6. 继续，直到处理完所有条，并确保剩余的堆栈元素也通过刷新哨兵边界来解析。 
7. 跟踪所有行的最大面积。 最终答案是在任何直方图中遇到的最大矩形。 

核心思想是网格中的每个有效矩形都必须有一个底行，并且对于每个底行，我们通过直方图处理详尽地考虑以该处结尾的所有矩形。 

### 为什么它有效

 每个`height[j]`精确编码以行 i 和列 j 结束的矩形的最大可能垂直范围。 任何在第 i 行结束的矩形都对应于直方图中的一个连续段，并且其高度以该段中的最小高度为界。 单调堆栈确保每个这样的段以其正确的限制高度被恰好考虑一次。 这保证了不会遗漏任何矩形并且不会计算无效矩形。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def largest_histogram(heights):
    stack = []
    best = 0
    n = len(heights)

    for i in range(n + 1):
        cur = 0 if i == n else heights[i]
        while stack and cur < heights[stack[-1]]:
            h = heights[stack.pop()]
            left = stack[-1] + 1 if stack else 0
            width = i - left
            best = max(best, h * width)
        stack.append(i)

    return best

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    height = [0] * m
    ans = 0

    for i in range(n):
        for j in range(m):
            if grid[i][j] == '#':
                height[j] += 1
            else:
                height[j] = 0

        ans = max(ans, largest_histogram(height))

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案维护一个运行直方图`height`，逐行更新。 每行都会触发完整的直方图评估。 单调堆栈例程计算线性时间内最大的矩形，确保每个索引最多被压入和弹出一次。 

一个常见的陷阱是忘记直方图循环中的哨兵迭代。 额外的迭代在`i == n`强制处理剩余的堆栈元素，否则将错过延伸到右边界的矩形。 

另一个微妙的细节是遇到时将高度重置为零`.`。 自从`.`代表一个被阻挡的单元格，任何穿过它的矩形都是无效的，所以垂直累加必须重新开始。

 ## 边缘情况

 一个完全阻塞的网格，其中每个单元格都被`.`，生成全零的高度数组。 每个直方图调用都会立即产生零，因为不存在正高度，因此答案始终保持零。 

一个完全开放的网格，其中每个单元格都`#`，每行生成一个稳定增加的直方图。 除非刷新，否则堆栈永远不会触发早期弹出，并且最大矩形扩展到最后一行的完整矩阵。 

在大量被阻塞的单元中，单个孤立的自由单元会产生高度峰值 1，并立即重置。 直方图步骤正确地将所有候选矩形限制为宽度或高度 1，确保不会发生过度计数，因为任何扩展都将包含零高度列。
