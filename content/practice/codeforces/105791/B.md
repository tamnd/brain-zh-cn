 ## 解决方案
 ## 问题理解

 我们在平面上得到一条折线，该折线由按 x 坐标递增排序的 n 个点定义。 该路径从第一个点开始，并在连续点之间以直线段行进，因此步行器沿着分段线性曲线移动。 沿着这些路段行驶的总距离以欧几里得距离来测量。 

一个人从第一个点开始，沿着这条虚线行走。 沿着曲线走了 K 个单位的距离后，他们停下来。 任务是确定它们沿路径到达停止位置的任何位置的最大 y 坐标，包括线段内的点。 

重要的微妙之处在于，除非 K 恰好落在某个顶点，否则步行器不会停在该顶点。 停止点可以严格位于线段内部，如果线段向上倾斜，最高点也可能位于线段内部。 

这些限制使得对每一个小步骤进行简单的模拟都是不可能的。 当 n 达到 200,000，K 达到 10^18 时，我们无法模拟小增量的运动。 即使逐段迭代也需要小心，但如果每个段都处理一次，仍然是可行的。 然而，真正的困难是在部分遍历之后计算线段内的确切点并沿着部分遍历的线段跟踪最大高度。 

一个天真的错误是只考虑顶点高度。 当最高点位于线段内时，该方法会失败。 例如，如果一段从 (0, 0) 到 (10, 10) 并且 K 中途停止，则最大高度为 5，而不是顶点值。 

另一个微妙的边缘情况是 K 恰好在一个顶点处结束。 那么答案必须包括该顶点的 y 坐标，但不包括任何未来的线段。 

最后，浮点精度很重要，因为距离是欧几里德距离，我们可能会在线段上输出非整数点。 

## 方法

 暴力方法将显式地沿着路径逐段行走，从 K 中减去段长度，直到 K 耗尽。 对于每个线段，我们使用欧几里得距离公式计算其长度，然后确定剩余的 K 是否位于该线段内或之外。 如果位于范围内，我们使用线性插值计算准确位置并评估该点的高度。 

这可以正常工作，但是每个段执行大量浮点操作仍然没问题，但关键问题不是性能本身，而是沿部分遍历的段处理最大高度的正确性。 简单的版本通常只跟踪顶点最大值，而在增加的段上错过内部最大值。 

关键的观察结果是，在每个线段内，y 值随线段的弧长线性变化。 因此，完全遍历的线段上的最大值就是其端点的最大值。 对于部分遍历的线段，最高点是遍历的端点或线段的端点，具体取决于坡度方向。 由于运动是线性的，因此段的任何前缀上的最大值都在该前缀的端点处实现。 

因此，问题简化为逐段行走，保持沿该段的当前位置，并更新在顶点或最终部分点处看到的最大 y。

 ## 算法演练

1. 从第一个点开始，初始化剩余距离 K，并将当前最大高度设置为起始 y 坐标。 原因是路径从已知的有效点开始，因此必须立即包含它。 
2. 迭代由连续点形成的每个线段。 对于每个线段，计算其欧几里德长度。 这是必要的，因为运动是沿着曲线测量的，而不是独立地沿着 x 或 y 测量的。 
3. 如果 K 大于或等于线段长度，则步行器完全穿过该线段。 使用两个端点的最大 y 更新答案，然后从 K 中减去线段长度并移动到下一个线段。 原因是完全遍历的线段内的任何点都包含在步行中。 
4. 如果K小于线段长度，则步行器停在该线段内。 计算分数 t = K/segment_length，然后在 x 和 y 坐标上使用线性插值找到停止点。 这给出了步行终点的精确几何位置。 
5. 知道停止点后，用其 y 坐标更新答案。 此外，由于线段内部的路径在 y 方向上相对于弧长是线性的，因此无法到达超出该点的内部点，因此该过程立即结束。 
6. 输出在所有完全遍历的线段和最后的部分线段中遇到的最大 y 值。 

正确性依赖于以下事实：在每个段内，y 相对于段参数单调变化。 因此，段前缀上的任何最大值候选必须出现在该前缀的任一端点处。 

## 为什么它有效

 每个线段均以严格的线性运动穿过，因此沿线段的位置是端点之间的线性插值。 这意味着沿线段的 y 坐标在遍历参数中也是线性的。 线性函数在端点之一的闭区间上达到最大值，因此对于任何完全遍历的线段，最大值位于顶点处，而对于部分遍历的线段，遍历前缀上的最大值位于起点或停止点。 由于已经考虑了起点并且明确计算了停止点，因此所有可能的最大值都被覆盖一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dist(a, b):
    dx = a[0] - b[0]
    dy = a[1] - b[1]
    return (dx * dx + dy * dy) ** 0.5

n, K = map(int, input().split())
pts = [tuple(map(int, input().split())) for _ in range(n)]

ans = pts[0][1]
remaining = K

for i in range(n - 1):
    x1, y1 = pts[i]
    x2, y2 = pts[i + 1]

    dx = x2 - x1
    dy = y2 - y1
    seg_len = (dx * dx + dy * dy) ** 0.5

    if remaining >= seg_len:
        ans = max(ans, y1, y2)
        remaining -= seg_len
    else:
        t = remaining / seg_len
        y = y1 + dy * t
        ans = max(ans, y)
        print(f"{ans:.10f}")
        sys.exit()

ans = max(ans, pts[-1][1])
print(f"{ans:.10f}")
```该解决方案维护跑步剩余距离并逐段消耗它。 对于每个完整段，它使用端点比较安全地更新答案，这由于线性而有效。 当到达最后一个部分段时，插值会给出准确的停止位置，并将其 y 值与当前最大值进行比较。 

一个微妙的实现点是避免除了段长度计算之外不必要的重复平方根计算。 另一个是通过一致地使用双精度算术并以足够的小数精度打印来确保浮点精度。

 ## 边缘情况

 关键的边缘情况是 K 等于 0 时。 在这种情况下，步行者永远不会离开起点，因此答案只是第一个 y 值。 该算法可以处理此问题，因为剩余距离永远不会被消耗，并且 ans 已正确初始化。 

另一种情况是当整个路径短于 K 时。循环正常结束并包含最终顶点。 即使步行者超越所有段，这也确保了正确性。 

一种微妙的情况是一个斜率为负的线段，后面跟着一个正斜率的线段，其中最高点位于中间线段链。 由于每个段都完全更新端点最大值，并且检查部分插值，因此即使它不是顶点，也不会错过任何峰值。
