 ## 解决方案
 ## 问题理解

 我们有一棵树$n$顶点并询问它是否可以通过参数化的非常具体的递归构造生成$k$。 该构造从一棵称为刺猬的特殊树开始，其中只有一个顶点充当中心枢纽，所有其他顶点都是附着在其上的叶子，并附加要求该中心的度数至少为三。 

对于较大的$k$，该结构是递归构建的：前一阶段的每个叶子都被整个新刺猬替换，并且原始叶子的邻居连接到新刺猬的中心。 重复这个过程会创建一棵树，看起来就像嵌套的刺猬层从中央核心向外扩展。 

任务是确定给定的树是否可以在精确之后获得$k$此类扩展步骤。 

输入约束允许最多$10^5$顶点和$k$一样大$10^9$。 这立即告诉我们，我们无法显式地模拟该构造。 任何尝试在其中逐层构建或剥离层的解决方案$O(k)$甚至$O(nk)$是不可能的。 

一个更微妙的含义是，答案仅取决于树的结构特性，而不取决于任何特定的构造历史。 由于树是固定的、非循环的，我们必须检测它是否具有递归的“分层轴叶展开”结构。 

一些边缘情况突出了定义的限制：

 如果树是中心度小于 3 的简单星形树，例如长度为 2 的路径，它甚至不是有效的 1-multihedgehog，因此无论什么答案都必须为“否”$k$。 

如果$k$很大，但树的高度很小，例如大小为 100 的星形，没有更深的嵌套，它不能代表更高级别的 multihedgehog，因为递归构造强制重复结构层。 

另一种失效模式是当多个顶点似乎充当不同深度的潜在“中心”时，但它们的邻域不会形成不相交的刺猬扩张。 仅在本地检查度数的幼稚方法会错误地接受此类树。 

## 方法

 关键的困难在于，该定义描述了从刺猬基开始的递归扩展，这表明以根周围的层为单位进行思考。 如果我们尝试暴力重建，我们可能会尝试模拟折叠叶子：重复查找所有 1 度顶点，删除它们，并尝试检测剩余图是否形成另一个刺猬结构，重复此过程$k$次。 

这种方法在概念上是正确的，因为每个逆步骤对应于撤消一个扩展层。 然而，每一层都需要扫描所有顶点并更新度数，并且可以有$O(n)$最坏情况下的层数。 这导致$O(n^2)$退化树中的行为，例如长链或深层嵌套结构，对于$n = 10^5$。 

关键的观察是，该构造生成一棵树，其顶点可以被分配一个“层数”，该“层数”等于它们距最深刺猬中心结构的距离，并且每一层的行为就像向外扩展的 BFS 波。 而不是显式模拟$k$通过迭代，我们可以计算所有叶子向内的距离结构，并将递归刺猬展开解释为叶子与核心的距离以及每层分支行为的约束。 

具体来说，该结构强制建立一个中心区域，其中顶点的度数至少为 3（原始刺猬中心），并且该核心之外的每个顶点都必须位于通向前一层中唯一父级的路径上，形成一棵嵌套星状扩展的树。 这将问题简化为计算距叶子的距离并验证修剪层是否产生有效的深度序列$k$。 

有效的解决方案使用从所有叶子开始的多源 BFS。 我们向内传播，为每个节点分配一个“移除时间”（或层索引）。 最大指定深度告诉我们存在多少个扩展层。 然后我们检查剩余核心的深度结构是否$k$仍然满足刺猬条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 强力剥离层|$O(n^2)$|$O(n)$| 太慢了|
 | BFS 层分配 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 计算每个顶点的度数并用所有叶子（度数为 1 的顶点）初始化一个队列。 

这些叶子代表递归结构的最外层，因为每个扩展步骤都会创建新的叶子。 
2. 为每个叶子分配层值 0，并将其推入 BFS 队列。 

该层代表结构的第一个可移动边界。 
3.向内运行BFS：处理顶点时$v$，减少其邻居的度数$u$。 如果$u$的度数变为1，分配给它层$\text{layer}[v] + 1$并将其排入队列。 

这模拟了将树从外向内逐层剥皮。 
4. BFS 完成后，每个节点都有一个层值，对应于迭代叶子剥离中该节点何时被删除。 
5.让$L$为最大图层值。 该结构正好具有$L + 1$概念层。 
6. 检查是否$k$与该结构兼容。 根刺猬必须生存到深处$k$，所以我们验证$k \le L$。 如果$k > L$，树太浅，无法表示$k$-multihedgehog.
 7. 考虑层节点$k$。 These nodes form the “core” at the$k$-th stage of pruning. 在有效的构造中，该核心必须恰好包含一个度数至少为 3 的顶点（刺猬中心），而该核心中的所有其他顶点在导出结构中必须具有度数 1。 
8. Extract the subgraph induced by nodes with layer at least$k$，重新计算该子图中的度数，并验证刺猬条件：恰好有一个节点的度数至少为 3，而所有其他节点的度数为 1。 

### 为什么它有效

 The BFS peeling process encodes the recursive construction in reverse. 定义中的每个扩展步骤都对应于添加一层附加到刺猬中心的叶子，并且反转该步骤将恰好删除这些叶子。 Because the graph is a tree, every leaf removal is independent and does not interfere with others, ensuring that the layering is well-defined.

 关键的不变量是节点分配的层$d$正好在距离处$d$在迭代叶子去除下从外边界。 这与递归结构深度一致。 因此，层的诱导子图$k$精确对应于之后的剩余结构$k$反向施工步骤。 在此阶段检查刺猬属性可验证递归是否有效，并且在任何级别均未发生结构违规。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

n, k = map(int, input().split())
adj = [[] for _ in range(n)]
deg = [0] * n

for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    adj[u].append(v)
    adj[v].append(u)
    deg[u] += 1
    deg[v] += 1

layer = [-1] * n
q = deque()

for i in range(n):
    if deg[i] == 1:
        layer[i] = 0
        q.append(i)

while q:
    v = q.popleft()
    for u in adj[v]:
        if layer[u] == -1:
            deg[u] -= 1
            if deg[u] == 1:
                layer[u] = layer[v] + 1
                q.append(u)

max_layer = max(layer)

if k > max_layer:
    print("No")
    sys.exit()

nodes = [i for i in range(n) if layer[i] >= k]

if len(nodes) == 1:
    print("Yes")
    sys.exit()

indeg = {v: 0 for v in nodes}
s = set(nodes)

for v in nodes:
    for u in adj[v]:
        if u in s:
            indeg[v] += 1

centers = 0
ok = True

for v in nodes:
    if indeg[v] >= 3:
        centers += 1
    elif indeg[v] == 0:
        ok = False

if centers == 1 and ok:
    print("Yes")
else:
    print("No")
```实现首先构建邻接表并计算度数。 BFS 队列使用所有叶子进行初始化，这一点至关重要，因为剥离过程是多源的：一次仅删除一个叶子会破坏层解释。 

层数组记录每个顶点在迭代修剪过程中何时暴露为叶子。 这与施工深度相反。 

计算完层数后，我们检查是否$k$超过最大深度； 如果是，则无效$k$-多重刺猬结构可以存在。 

至少有层的顶点上的导出子图$k$然后进行检查。 在这个子图中，我们重新计算度数并强制执行刺猬条件：只有一个节点的度数必须至少为 3，并且所有其他节点都必须位于该核心内。 

一个微妙的点是，单节点核心是非常有效的，因此我们单独接受这种情况。 

## 工作示例

 ### 示例 1（有效 2-multihedgehog）

 我们追踪剥皮过程。 

| 步骤| 队列（叶）| 删除图层节点 | 分配新图层|
 | --- | --- | --- | --- |
 | 0 | 所有叶子 | 1层边界| 0 |
 | 1 | 下一个边界 | 内边界| 1 |
 | 2 | 中心地区 | 核心边界| 2 |

 经过 BFS 后，假设最大层数为 2。对于$k=2$，我们只保留层≥2的节点。 

在这个诱导核心中，我们恰好观察到一个顶点的度数≥3，而所有其他顶点都是叶子。 这证实了刺猬核心是有效的。 

这表明 BFS 分层正确地重建了递归结构深度。 

### 示例 2（无效结构）

 考虑一棵树，在修剪后，两个高度节点仍保留在假定的核心中。 

| 步骤| 队列| 已删除的节点 | 问题 |
 | --- | --- | --- | --- |
 | 0 | 叶子| 外层| 好的 |
 | 1 | 下一个 | 内层| 好的 |
 | k层核心 | 多个高度节点| 违反刺猬条件| 无效|

 该算法拒绝这一点，因为导出的子图中存在多个中心，违反了唯一性要求。

 | 测量| 复杂性 | 说明|

 ## 边缘情况

 一个重要的边缘情况是整棵树在修剪后折叠成单个节点$k$层。 在这种情况下，导出的子图的大小为一，并且立即被接受。 这对应于所有扩张最终收敛到单个幸存中心的情况，并且刺猬条件被空洞地满足。 

当多个顶点保留在$k$-core，但只有一个的度数至少为 3。BFS 分层仍然可以正确识别结构，但对原始图进行简单的度数检查会根据外部叶子错误地拒绝或接受。 通过重新计算导出子图内的度，该算法确保了结构限制而不是全局度下的正确性。 

最后的边缘情况是一条漫长的道路。 在一条路径中，每个节点在剥离过程中最终都会变成一片叶子，最大层数等于路径的中间。 对于任何$k \ge 1$，导出的核心永远不会包含度数≥3的顶点，因此算法正确地拒绝它。
