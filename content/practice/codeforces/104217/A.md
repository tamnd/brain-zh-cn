 ## 解决方案
 ## 问题理解

 我们得到两个大写字符串，一个代表板上显示的当前单词，另一个代表我们想要替换它的单词。 这项任务不是将字符一一转换到位，而是使用可用字母以物理方式重建最终单词。 

我们关心的成本是必须新放置到板上才能得到目标单词的字母数量。 已经正确定位并在两个单词之间匹配的字母可以在概念上重复使用，因为它们已经存在并且不需要再次添加。 

重新构建问题的一个有用方法是想象对齐两个字符串并检查哪些位置已经匹配。 每次不匹配都意味着我们必须在该位置放置一个新字母，因为现有字符是错误的并且必须被替换。 超出较短字符串的位置的行为就好像丢失的字符必须从头开始完全构造一样。 

约束允许字符串的长度最大为 100,000。 任何在单遍中直接比较字符的解决方案都是可以接受的，但是任何涉及子字符串嵌套循环的解决方案都会太慢。 在最坏的情况下，二次方法最多可以执行 10^10 次比较，这远远超出了时间限制。 

一些边缘情况很重要：

 当两个字符串相同时，不需要进行任何更改，因此答案必须为零。 如果没有显式检查相等性，则假设至少一个替换的简单方法可能会错误地输出正值。 

当一个字符串为空时，答案应该是另一个字符串的长度，因为所有字符都必须重新放置。 仅迭代到最小长度的粗心实现可能会忘记考虑剩余的后缀。 

当字符串仅在末尾或开头不同时，我们必须确保在每个位置正确计算不匹配，而不仅仅是在前缀比较中。 

## 方法

 暴力破解的想法很简单：尝试通过模拟替换并计算修复每个不匹配所需的操作次数来从源字符串构造目标字符串。 然而，任何涉及重复移动或修改字符串的模拟都会降级为 O(n^2)，因为每次修改都可能涉及重建部分字符串。 

关键的观察是我们永远不需要模拟中间状态。 我们只关心每个位置是否已经匹配。 每个不匹配都恰好贡献了一个所需的字母位置，因为我们假设我们可以直接覆盖或将正确的字符放置在该位置而不影响其他字符。 

这将整个问题简化为比较同一索引处两个字符串的字符的单个线性扫描。 对于超出较短字符串的索引，较长字符串中的每个额外字符都必须计为所需的添加。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 直接位置比较 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取两个字符串`s`和`t`。 我们只需要它们的长度和字符对齐即可决定答案。 
2. 计算两个字符串的最小长度。 直到这个索引，两个字符串都可以直接逐个字符进行比较。 
3. 初始化计数器`k = 0`。 这将存储有多少位置需要新的字母放置。 
4. 从索引迭代`0`到`min(len(s), len(t)) - 1`。 对于每个指标，比较`s[i]`和`t[i]`。 如果不同，则增加`k`因为这个位置必须被重写。 
5. 完成重叠区域后，处理剩余的后缀。 如果`t`长于`s`， 添加`len(t) - len(s)`到`k`。 如果`s`更长，这些额外的字符不会有助于形成`t`，因此它们在成本中被忽略。 
6. 输出`k`。 

将过程分为重叠和后缀背后的逻辑来自于这样的事实：不匹配和丢失字符都需要显式的字母放置，但它们发生在字符串的不相交区域中。 

### 为什么它有效

 在两个字符串都存在的每个索引处，字符正确的唯一方法是它是否与目标完全匹配。 没有任何操作允许部分重用不匹配的字符，因此每个不匹配都会独立贡献一个所需的位置。 对于重叠之外的位置，没有什么可重用的，因此重叠之外的目标中的每个字符都必须重新放置。 这确保了计数是必要且充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
t = input().strip()

n, m = len(s), len(t)
k = 0

common = min(n, m)
for i in range(common):
    if s[i] != t[i]:
        k += 1

if m > n:
    k += (m - n)

print(k)
```实现直接遵循算法。 我们首先剥离输入字符串以避免换行问题，然后比较重叠长度。 后缀调整可确保我们解决目标字符串中丢失的字符。 

一个微妙的点是我们永远不需要考虑删除`s`。 中的额外字符`s`是无关紧要的，因为输出仅取决于形成`t`，不修改`s`就位。 这使逻辑严格保持单向。
