 ## 解决方案
 ## 问题理解

 我们正在处理一个圆形轨道`n`均匀分布的位置，标记为`0`到`n-1`。 每只狗都以独特的位置开始`i`，每只狗都以恒定的速度顺时针移动`v_i`，意味着之后`t`时间步长，狗`i`将位于位置`(i + t · v_i) mod n`。 

目的是确定是否存在时间`t`（从`0`最多`1000`）使得所有的狗在圆圈上占据相同的位置。 如果存在这样的时刻，我们必须报告最小的这样的时刻`t`，以及共同立场`p`所有狗相遇的地方。 如果在允许的时间窗口内不存在这样的时间，则答案是`-1`。 

限制很小：`n ≤ 1000`和`v_i ≤ 100`，并且时间限制隐含地限制我们只能考虑最多`t = 1000`。 这立即表明对所有时间步进行模拟是可行的，因为我们要检查的状态总数最多约为一百万个位置。 

微妙的一点是，见面位置并不是提前固定的。 由当时的动态决定`t`。 一个天真的错误是假设我们可以猜测目标位置或将问题简化为零时间处的成对对齐。 但这会失败，因为同步取决于时间演化，而不是初始结构。 

另一个潜在的陷阱是忘记模行为。 即使所有的狗看起来都在线性移动，它们的位置也环绕着圆圈，并且必须在模算术下检查对齐情况，而不是原始整数。 

一个小边缘情况是`n = 1`，答案很简单`t = 0, p = 0`。 另一种是没有时间满足条件时，在这种情况下我们必须显式输出`-1`而不是部分配置。 

## 方法

 一种直接的方法是模拟系统。 对于每一次`t`，我们使用公式计算每只狗的位置`(i + t · v_i) mod n`，然后检查所有计算的位置是否相同。 如果是，我们就返回`t`和那个位置。 

这是可行的，因为系统在任何固定时间的状态都是完全确定的并且可以独立计算。 从定义可以看出其正确性。 

这种强力方法的成本很容易分析。 对于每个`t`最多`1000`，我们扫描所有`n`狗，导致大约`1000 · 1000 = 10^6`最坏情况下的评价。 每次评估都是恒定时间，因此这完全在限制范围内。 

这里不需要更深层次的代数简化，因为时间限制足够小，直接模拟已经很适合。 任何导出封闭式解的尝试都会引入不必要的复杂性，而不会提高渐近性能。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n·T) | O(n·T) | O(1) | O(1) | 已接受 |
 | 最佳（同样的想法）| O(n·T) | O(n·T) | O(1) | O(1) | 已接受 |

 这里`T = 1000`。 

## 算法演练

 我们一步步模拟时间并检查对齐情况。 

### 步骤

 1. 随着时间的推移进行迭代`t`从`0`到`1000`。 

我们从`0`因为需要最早的解决方案，并且时间零是有效配置。 
2. 对于每个`t`，将第一只狗的位置计算为`p0 = (0 + t · v_0) mod n`。 

这成为候选人会议位置。 
3. 对于其他每只狗`i`, 计算`pi = (i + t · v_i) mod n`。 

如果有的话`pi`不同于`p0`， 这次`t`这不可能是一个解决方案，因为并非所有的狗都一致。 
4. 如果所有狗都匹配`p0`，立即返回`(t, p0)`。 

The first such`t`遇到的自动是最小的。 
5. 如果范围内没有时间产生完全对齐，则输出`-1`。 

### 为什么它有效

 任何固定时间`t`，系统状态完全由从索引到位置的确定性映射来描述。 该算法准确检查该映射是否在所有索引中保持不变。 由于我们按升序检查时间并接受第一个有效配置，因此构造返回的时间是最少的。 中间状态不能被跳过或部分有效，因为条件要求所有狗同时一致，而不是增量一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    v = list(map(int, input().split()))
    
    for t in range(1001):
        p0 = (0 + t * v[0]) % n
        ok = True
        
        for i in range(1, n):
            if (i + t * v[i]) % n != p0:
                ok = False
                break
        
        if ok:
            print(t, p0)
            return
    
    print(-1)

if __name__ == "__main__":
    solve()
```实现直接反映了算法。 外循环枚举时间。 每次，我们计算狗的参考位置`0`并将所有其他狗与它进行比较。 The early exit on mismatch prevents unnecessary work once inconsistency is found.

 A common implementation mistake is forgetting to take modulo`n`在每个位置计算。 没有它，价值观就会快速增长，比较就变得毫无意义。 另一个微妙的问题是没有独立地重新启动每个时间步的正确性检查，这将错误地跨迭代携带状态。

 ## 边缘情况

 对于`n = 1`，循环立即发现唯一的狗已经与自身对齐`t = 0`，算法返回`0 0`。 计算`(0 + 0 · v_0) mod 1`是`0`，所以一致性是微不足道的。 

对于速度相同的情况，每只狗都保持相对间距模数`n`。 如果初始位置不相等，则以后的时间无法修复偏移量，因此算法正确地耗尽所有时间并返回`-1`除非`n = 1`。 

对于在时间范围内较晚发生对齐的情况，逐步扫描可以保证正确性，因为不会跳过任何时间。 最早有效`t`总是返回，因为迭代是严格增加的。
