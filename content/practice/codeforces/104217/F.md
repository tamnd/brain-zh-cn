 ## 解决方案
 ## 问题理解

 我们得到一组分散在 2D 平面上的检查点。 每个检查点仅在特定时间出现，如果我们恰好在该时间恰好位于其坐标处，我们就可以获得一定数量的奖励。 

我们从时间零的原点开始，可以在任何方向上以每单位时间一个单位距离的速度连续移动。 这意味着从A点移动到B点之间的距离正好是欧几里德距离，如果我们想等待，就必须不早于预定时间到达，但迟到也无济于事，因为宝藏只能在准确的时间获得。 

目标是从原点开始选择一系列要访问的检查点，使得每个转变在时间上在物理上都是可行的，并且收集的值的总和最大化。 

输入大小最多为 5000 个检查点。 检查所有可能的访问序列的天真的尝试将需要探索具有 5000 个节点的图中的排列或路径，每个状态转换步骤都会呈阶乘或至少呈二次方增长，从而使 O(N^2 log N) 或更糟的潜在边界取决于常数，而 O(N^3) 或指数显然是不可能的。 

关键的微妙之处在于时间约束：即使一个点在几何上很接近，也可能无法到达，因为它的时间戳太早了。 相反，如果经过足够的时间，远点仍然可以到达。 这使得该问题成为由时空可达性定义的有向图中的约束最长路径问题。 

一些边缘案例打破了天真的贪婪思维。 一个常见的失败是假设我们总是更喜欢最近的下一个点。 例如，由于时间限制，早期附近的低值点可能会阻止到达稍后的高值点，即使跳过它可以获得更大的增益。 另一个微妙的情况是相等的时间戳：除非位置相同，否则不能同时获取两个点，这通常是不可能的，因此必须严格遵守顺序约束。 

## 方法

 思考问题的一种直接方法是将每个检查点视为一个节点，如果可以及时从 i 旅行到 j，则将节点 i 连接到节点 j。 从时间 Ti 的节点 i 出发，如果 Ti + dist(i, j) ≤ Tj，我们可以在时间 Tj 到达节点 j。 如果我们还包括位于 (0,0,0) 的虚拟起始节点，我们可以计算在每个检查点结束的最佳奖励路径。 

这立即表明了一个动态规划公式，其中 dp[i] 是在检查点 i 结束的最大奖励。 对于每个 i，我们尝试从 Tj ≤ Ti 的所有早期可行检查点 j 进行转换，并检查 j 是否可以及时到达 i。 这给出了一个简单的 O(N^2) 解决方案。 

蛮力结构之所以有效，是因为每个检查点仅依赖于先前可到达的检查点。 然而，我们必须考虑所有排列或任意序列的天真的解释太大了。 改进来自于认识到时间严格排序可行性：只有从较小或相等的时间到较大的时间的过渡才有效。 这会将问题转换为 DAG 最短或最长路径问题，其中边仅在时间上前进。 

与典型的 DAG 最短路径问题不同，边没有明确给出； 它们必须使用几何距离来计算。 这在 O(N^2) 中仍然可以接受，因为检查一对是常数时间。 

不需要更复杂的几何或数据结构，因为 N = 5000 将二次转换保持在范围内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 排列上的强力路径 | O(N!) | O(N) | 太慢了 |
 | 所有有效转换的 DP | O(N^2) | O(N^2) | O(N) | 已接受 |

 ## 算法演练

我们按时间对检查点进行排序，以便所有有效的转换都从较早的点到较晚的点。 然后我们对该订单运行动态规划。 

### 步骤

 1. 读取所有检查点并将每个检查点与其值、坐标和时间相关联。 在 (0, 0, 0) 处添加值为 0 的虚拟起始检查点。 
2. 按时间递增对所有点进行排序。 这确保了任何有效的移动方向始终按数组顺序向前移动。 这种排序至关重要，因为它可以防止在计算 dp 时重新访问未来状态。 
3. 初始化一个 dp 数组，其中 dp[i] 表示在检查点 i 处精确结束时可达到的最大总值。 将除起始节点 0 之外的所有值设置为负无穷大。 
4. 对于按排序顺序的每个检查点 i，迭代所有先前的检查点 j < i。 
5. 对于每一对 (j, i)，计算它们之间的欧氏距离并检查 dp 转移是否可行：dp[j] 有效且 Tj + dist(j, i) ≤ Ti。 
6. 如果可行，更新 dp[i] = max(dp[i], dp[j] + Vi)。 这意味着我们通过检查点 i 来扩展以 j 结束的最著名路线。 
7. 处理完所有节点后，答案是所有检查点上的最大 dp[i]。 

### 为什么它有效

 该算法保持不变式，即 dp[i] 存储在检查点 i 精确结束的所有有效路径中的最佳可能奖励，仅考虑按时间顺序不晚于 i 发生的检查点。 因为每次转换都尊重时间顺序和行进可行性，所以每次 dp 更新都对应于物理上有效的移动。 由于检查了所有可能的有效前驱，因此不会错过通往 i 的更好路径。 时间排序保证了转换图中的非循环性，因此未来的状态不会影响过去的状态，从而防止循环或不一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def solve():
    n = int(input())
    pts = []
    
    for _ in range(n):
        x, y, t, v = map(int, input().split())
        pts.append((t, x, y, v))
    
    # sort by time
    pts.sort()
    
    # dp[i] = best value ending at i
    dp = [0] * n
    
    ans = 0
    
    for i in range(n):
        t_i, x_i, y_i, v_i = pts[i]
        dp[i] = v_i
        
        for j in range(i):
            t_j, x_j, y_j, v_j = pts[j]
            
            dx = x_i - x_j
            dy = y_i - y_j
            
            if dp[j] == 0 and j != 0:
                # still valid, dp[j] might be unreachable; skip only if desired
                pass
            
            dist = math.hypot(dx, dy)
            
            if t_j + dist <= t_i:
                dp[i] = max(dp[i], dp[j] + v_i)
        
        ans = max(ans, dp[i])
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现依赖于按时间排序，以确保当我们计算 dp[i] 时，所有潜在的前驱都已被处理。 内部循环检查每个较早的检查点并使用欧几里德距离验证可达性。 

一个微妙的点是距离比较中的浮点精度。 由于坐标和时间是 1e9 以内的整数，因此使用`math.hypot`是标准的，但严格的问题有时需要 epsilon 容差。 在这样的竞争环境中，通常是安全的，但更可靠的方法是仔细比较平方距离和平方时间差，尽管这里的时间差包括旅行等待，因此直接比较更简单。

 ## 边缘情况

 一种边缘情况是多个检查点共享相同的时间戳。 排序后，它们变得连续，但不能相互转换，除非距离为零。 该算法自然会处理这个问题，因为条件`t_j + dist <= t_i`除非 dist 为零，否则失败，因此 dp 状态保持独立。 

另一个边缘情况是当一个非常高价值的检查点在空间中很接近但具有较早的时间戳时。 任何按值或距离进行选择的天真贪婪者都会尝试首先获取它，但如果它阻碍了以后积累的可行性，则 dp 会正确地避免它。 

最后一种情况是只能通过一长串中间点才能到达某个点。 DP 确保一旦计算出每个中间 dp 状态，它就可以向前传播。 例如，每跳勉强满足时间约束的链仍然可以正确累积，因为每个中间状态都被视为 O(N²) 扫描中的有效前驱状态。
