 ## 解决方案
 ## 问题理解

 我们正在构建长度有序的序列$N$，其中每个位置都填充了两种无法区分的类型之一：斑点或棕色。 因为同一类型的奶牛是相同的，所以有效的配置由长度为二进制的字符串完整描述$N$。 

限制是我们必须避免$K$或序列中任意位置的更多连续斑点奶牛。 棕色奶牛充当打破条纹的分隔符，因此限制完全与连续斑点条目的游程长度有关。 

任务是统计有多少个这样长度的二进制字符串$N$满足此限制，并输出结果模$10^9 + 7$。 

约束条件$N \le 10^6$立即排除所有指数枚举或朴素递归$2^N$字符串。 甚至$O(N^2)$在最坏的情况下解决方案太慢，因为$N$可以达到一百万。 解必须是线性或接近线性的。 

仅跟踪位置和最后游程长度的简单动态编程接近正确，但如果嵌套转换或重新计算实现得不好，它就不会通过。 

当发生微妙的边缘情况时$K = 1$。 在这种情况下，任何有斑点的牛都是被禁止的，因为即使是单个有斑点的牛也会形成长度为 1 的游程，因此唯一有效的序列是全棕色序列。 另一个边缘情况是$K > N$，其中约束永远不会触发并且所有$2^N$序列有效。 

## 方法

 暴力解决方案将生成每个长度的二进制字符串$N$并检查它是否包含至少长度为 1 的斑点奶牛$K$。 检查每个字符串需要$O(N)$，并且有$2^N$字符串，所以总工作是$O(N 2^N)$，即使对于小公司来说，这也是不可能的$N$30左右。 

关键的结构观察是有效性仅取决于斑点奶牛当前的连续运行长度。 一旦出现棕色牛，条纹就会完全重置。 这意味着字符串的整个历史可以压缩为单个状态变量：当前有多少连续的斑点奶牛结束前缀。 

这导致了一个动态编程公式，我们从左到右构建序列，并为每个长度维护有多少有效序列以精确的条纹结尾$i$斑点奶牛$0 \le i < K$。 过渡是局部的，仅取决于我们是否附加棕色或斑点。 

添加棕色会将条纹重置为零。 附加斑点会使条纹长度增加一，除非它达到$K$，在这种情况下不允许进行转换。 

这减少了从指数枚举到线性时间状态转换的问题$K$状态，产生$O(NK)$DP。 然而，自从$N$和$K$两者都可以很大，但在最坏的情况下这仍然太慢。 

最后的改进是观察 DP 递归具有滑动窗口结构：涉及斑点奶牛的转换在过去的总和中$K-1$州。 这允许维护前缀和，因此可以在恒定时间内计算每个步骤，从而将复杂性降低到$O(N)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N2^N)$|$O(N)$| 太慢了|
 | Naive DP（连续状态）|$O(NK)$|$O(K)$| 太慢了|
 | 优化 DP（前缀和）|$O(N)$|$O(K)$| 已接受 |

 ## 算法演练

 我们定义一个 DP，在其中从左到右处理序列长度。 让$dp[i]$表示当前处理长度的、以恰好为 的条纹结尾的有效序列的数量$i$连续的斑点牛。 我们还追踪$dp[0]$作为以棕色结尾的序列，因为棕色会重置条纹。 

我们维护所有有效状态的运行总计，以避免重复重新计算总和。 

### 步骤

 1.初始化一个数组$dp$尺寸的$K$全为零，并设置$dp[0] = 1$，表示以没有斑点条纹结尾的空序列。 这是唯一有效的启动配置。 
2. 对于每个位置$1$到$N$，计算一个新数组$ndp$最初全为零。 这代表新长度的所有序列。 
3. 首先处理放置一头棕色牛。 先前长度的任何有效序列都可以通过棕色进行扩展，这会将条纹重置为零。 所以我们设置$$ndp[0] = \sum_{i=0}^{K-1} dp[i]$$此步骤是正确的，因为棕色不依赖于先前的条纹长度。 
4. 接下来处理放置一头斑点牛。 如果我们将一头斑点牛附加到当前以$i$发现奶牛，我们搬到州$i+1$。 这仅在以下情况下有效$i+1 < K$。 所以对于所有人来说$i$,$$ndp[i+1] += dp[i]$$这确保我们只延长有效的条纹，而永远不会达到长度$K$。 
5. 更换$dp$和$ndp$并重复直到所有$N$位置已处理。 
6. 最终答案是所有状态的总和$dp$，因为只要从未达到，任何结束连续都是允许的$K$内部。 

### 为什么它有效

 不变的是处理后$t$职位，$dp[i]$精确计算长度有效序列的数量$t$谁的最后一头斑点牛有长度$i$，并且所有序列都计入$dp$从未违反任何前缀的约束。 

每个过渡都保留有效性：添加棕色不能创建禁止的条纹，而添加斑点只会扩展有效的条纹（如果其保持在以下水平）$K$。 由于每个有效的长度序列$t+1$必须恰好来自一个有效的长度序列$t$通过这两个操作之一，DP涵盖了所有可能性，没有重复或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, k = map(int, input().split())
    
    if k == 1:
        print(1)
        return
    
    if k > n:
        print(pow(2, n, MOD))
        return
    
    dp = [0] * k
    dp[0] = 1
    
    for _ in range(n):
        ndp = [0] * k
        
        total = sum(dp) % MOD
        
        ndp[0] = total
        
        for i in range(k - 1):
            ndp[i + 1] = dp[i] % MOD
        
        dp = ndp
    
    print(sum(dp) % MOD)

if __name__ == "__main__":
    solve()
```实现直接反映状态定义。 特殊情况$k = 1$首先处理，因为不允许有斑点的牛，这使得除了全棕色的序列之外的所有序列都无效。 

棕色的过渡使用先前状态的全部总和，因为任何条纹都与重置兼容。 斑点转变将状态右移一个，确保我们永远不会构建一条长度的条纹$K$。 

循环运行$N$次，每次迭代执行$O(K)$由于数组移位和求和而起作用。 考虑到限制，这已经足够了。

 ## 边缘情况

 当$K = 1$，DP 公式会退化，因为任何有斑点的牛都会立即产生禁止奔跑。 该算法短路并返回 1，对应于全棕色序列。 例如，输入`3 1`只生产`BBB`。 

什么时候$K > N$, 无游程$K$可以物理地发生。 算法正确返回$2^N$，因为所有二进制序列都是有效的。 用于输入`4 10`，所有 16 个序列均被接受。 

什么时候$K = 2$，系统在任何发现的牛之后强制执行严格的轮换。 DP 在 streak-0 和 streak-1 状态之间振荡，但任何扩展 streak-1 的尝试都会通过状态转换隐式消除。 用于输入`3 2`，有效序列正是那些没有`SS`，DP 正确地将它们枚举为 5。
