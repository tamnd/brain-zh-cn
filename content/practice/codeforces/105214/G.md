 ## 解决方案
 ## 问题理解

 我们有一棵树$n$节点，我们可以自由地为每个节点分配一个小写字母。 标记后，树中的每个简单路径都变成字母序列，沿着其端点之间的唯一路径读取。 

我们还得到了一个简短的模式词$w$长度最多为三。 任务是选择节点标签，使得连接标签完全匹配的有向简单路径的数量$w$尽可能大。 有向路径意味着我们选择一对有序的节点$(u, v)$并沿着独特的简单路径阅读信件$u$到$v$。 

因为底层图是一棵树，所以每一对节点都定义了一条简单的路径，因此唯一的自由来自于我们如何为节点分配字母。 

约束条件$n \le 3 \cdot 10^5$立即排除任何尝试明确评估每个作业或每个路径贡献的解决方案。 从简单的意义上来说，即使计算所有路径也已经是二次的，因此任何有效的解决方案都必须将问题简化为树的局部结构属性，例如度或边。 

一些边缘情况很重要。 

如果$w$长度为一，每个节点独立地贡献一条有效路径，因为单个节点是一条平凡路径。 

如果$w$长度为二，我们正在计算与有序字母对匹配的有向边。 

如果$w$长度为三，我们正在计算长度为二的路径$u \to v \to x$，所以只有以中间节点为中心的路径才重要。 

一种天真的方法会尝试分配字母，然后通过检查所有路径来重新计算贡献，这会失败，因为即使评估单个分配也需要$\Theta(n^2)$树中的路径。 

## 方法

 关键的困难在于我们不是计算固定标签的路径，而是选择使计数最大化的标签。 由于字长最多为三，有效路径的结构变得极其局部，并且每种情况都分解为节点或边缘上的简单组合优化。 

暴力方法会尝试将字母分配给所有节点，然后计算匹配路径。 即使我们将自己限制在相关字母中$w$，分配的数量呈指数增长$n$，所以这是没有希望的。 

结构观察是，每条长度最多为 3 的有效路径完全由恒定大小的邻域决定：

 对于长度一，只有节点本身很重要。 

对于长度二，只有边缘很重要。 

对于长度三，只有中心节点及其关联边很重要。 

这会将树折叠成独立节点、独立边或节点周围独立的局部星形结构。 

这种独立性使得$O(n)$解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数 /$O(n^2)$每次作业计数|$O(n)$| 太慢了 |
 | 最佳 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们根据单词的长度来处理问题$w$。 

### 案例 1：$|w| = 1$1. 每个节点都可以用单个字符标记$w[0]$。 
2. 每个节点单独形成一条长度为1的有效路径。 

答案很简单$n$，因为每个节点恰好贡献一条有效的有向路径。 

### 案例 2：$|w| = 2$1. 这个词是两个字符，比如说$w = ab$。 
2. 有效路径与有向相邻对完全对应$(u, v)$这样$u$被标记为$a$和$v$被标记为$b$。 
3. 每个无向边最多贡献一条有效的有向路径，具体取决于我们如何分配其端点。 

现在观察到每条边最多只能贡献一个有效方向，并且如果可能的话，我们希望每条边都贡献恰好一个这样的定向匹配。 由于树是二分树，我们可以将节点着色为两组并分配一侧$a$，另一个$b$。 然后，每条边都连接相对的字母，因此每条边都贡献了一条有效的有向路径。 

因此最大值为$n - 1$，通过字母的任何二分分配来实现$a$和$b$。 

### 案例 3：$|w| = 3$1. 词有形式$w = abc$。 
2. 有效路径是三个节点$u \to v \to x$， 在哪里$v$是中间节点。 
3. 这迫使$v$携带信件$b$， 尽管$u$和$x$必须是邻居$v$贴上标签$a$和$c$为了。 

修复一个节点$v$指定的字母$b$。 让它有一些邻居被标记$a$和一些标记的$c$。 每条有效路径都以$v$是通过选择一个而形成的$a$- 邻居和一个$c$-邻居，所以贡献$v$是$$(\#a\text{-neighbors of }v) \cdot (\#c\text{-neighbors of }v).$$我们现在想要选择每个节点有多少个邻居$a$,$b$， 或者$c$最大化这个总和。 

关键的简化是只有标记的节点$b$做出贡献，因此我们可以有效地决定哪些节点充当路径中心。 如果我们选择多个$b$-node，它们的邻居分配会相互干扰，因为每个节点在全局范围内都有一个固定的标签。 与将所有结构集中在单个最佳中心周围相比，这种耦合使得多中心解决方案不是最佳的。 

所以我们尝试选择单个节点$v$作为独特的$b$- 生成所有贡献的标记节点。 然后所有其他节点都被分割$a$和$c$，并且只有与$v$事情。 

让$d = \deg(v)$。 我们将它的邻居分为$x$贴上标签$a$和$d-x$贴上标签$c$。 贡献变为$$x(d-x),$$当分割尽可能平衡时最大化，给出$$\left\lfloor \frac{d^2}{4} \right\rfloor.$$我们在所有节点上取最好的值。 

## 为什么它有效

 每个有效的长度为三的路径都有一个唯一的中心节点，因此计算路径相当于对中心上的独立贡献求和。 任何未被选为中心的节点都不会做出贡献，并且在多个中心之间划分责任会导致全局标签冲突，从而减少可实现的邻居分区。 将所有贡献集中在最佳度节点上可以将其邻接最佳地划分为两组，从而最大化有序邻居对的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    w = input().strip()

    if len(w) == 1:
        print(n)
        return

    edges = []
    deg = [0] * (n + 1)

    for _ in range(n - 1):
        u, v = map(int, input().split())
        deg[u] += 1
        deg[v] += 1

    if len(w) == 2:
        print(n - 1)
        return

    ans = 0
    for i in range(1, n + 1):
        d = deg[i]
        ans = max(ans, (d * d) // 4)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了三种结构情况。 边列表仅用于计算度数，因为最终公式仅取决于局部度数信息。 

唯一微妙的一点是长度为三的情况，其中表达式$(d^2)//4$正确实现最优分割$x(d-x)$无需显式尝试所有分区。

 ## 边缘情况

 对于$|w| = 1$，算法为所有节点分配相同的字母。 单节点输入如`1`正确地产生一条有效路径，因为唯一的节点形成有效的长度为一的路径。 

为了$|w| = 2$，诸如长链之类的路径仍然精确地产生$n-1$因为每条边都可以通过二分分配来交叉。 该算法隐式地利用了树是二分树的事实，因此不会出现边冲突。 

为了$|w| = 3$，考虑度节点$d$。 该算法将其指定为唯一中心并以最佳方式分割邻居。 在星图上输入类似`5 abc`边以 1 为中心，计算明确地评估$d=4$, 给予$4$，匹配可以形成的有序邻居对的数量。 

这些案例证实，所有贡献都被本地结构捕获，并且没有遗漏跨节点交互。
