 ## 解决方案
 ## 问题理解

 我们在平面上得到一组加权点。 每个点代表一个潜在的石油开采地点，位于整数坐标处，每个点都有一个利润值。 我们必须选择一个轴对齐的矩形（可能退化为一段或一个点），并将其放置在平面上的任何位置。 位于该矩形内部或边界上的每个点都贡献其利润，而我们支付的成本等于矩形的周长。 目标是最大化总利润减去周边成本。 

矩形完全可以自由定位，但一旦选择，它就会准确地收集它所包围的点。 因此，该任务从根本上来说是点子集上的几何优化问题，其中每个子集对应于一个最小边界矩形以及基于其宽度和高度的惩罚。 

约束 n ≤ 400 是关键信号。 成对的三次或二次结构是可以接受的，但任何隐式依赖于所有矩形的结构都是不可接受的。 具有强大内循环或预计算的 O(n^3) 解决方案是可行的，但任何接近 O(n^4) 的解决方案都有可能在 8 秒内超时。 

一个天真的错误是假设我们可以独立地贪婪地考虑 x 和 y 范围或在一维中排序和扫描。 矩形耦合了两个坐标，因此任何忽略这种耦合的分解都会失去最优性。 

另一个微妙的陷阱来自退化矩形。 允许使用零宽度或零高度的矩形，这意味着我们可以选择垂直或水平线段。 在选择单一坐标线可提供高利润点同时避免周边成本的情况下，这一点很重要。 例如，如果两个点位于相同的 x 坐标上且权重较大，则即使任何完整的矩形扩展增加了不必要的周长，退化垂直线也可能是最佳的。 

最后一个边缘情况是所有利润都很小，甚至由周边成本主导。 那么最佳答案可能是零或负数，并且算法必须正确比较空矩形或单点矩形，而不是假设我们必须选择多个点。 

## 方法

 蛮力的想法是考虑点的每个子集，计算其最小边界矩形，并评估利润减去周长。 这是正确的，因为任何矩形都精确对应于封闭点的子集，并且最佳矩形由它捕获的子集确定。 

然而，子集的数量是指数级的，如果我们考虑定义边界的点对，即使限制为几何子集，仍然会留下 O(n^2) 个候选矩形。 对于每个候选矩形，计算包含的点并求和权重已经花费了 O(n)，从而导致 O(n^3)。 当 n = 400 时，这是临界值，但如果天真地完成仍然太慢。 

关键的见解是首先停止考虑矩形，而是考虑选择边界约束。 矩形是通过选择其左、右、下、上边界来完全确定的。 如果我们固定左右边界，则最佳矩形将成为基于该条带内的点选择最佳底部和顶部的矩形。 

因此，我们将问题简化为扫描 x 区间。 对于按排序顺序定义左右边界的每对 x 索引，我们将点沿 y 投影到一维数组中，累积权重。 那么问题就变成了一个类似最大子数组的问题，其惩罚取决于所选的最小值和最大值 y。 

对于固定的 x 间隔，每个点贡献权重，周长成本变为 2（宽度 + 高度）。 宽度由所选的 x 端点固定，而高度取决于所选的 y 范围。 因此，在条带内部，我们要选择一个连续的 y 区间，使 sum(权重) 减 2*(y_max - y_min) 最大化。

这是一个经典的变换：按 y 排序并在可能的间隔内维护动态结构使我们能够有效地计算最佳垂直跨度。 

因此，完整的解决方案变为：固定 x-l、x-r，按 y 压缩该条带中的点，并使用扫描或 DP 对具有前缀和的排序 y 计算最佳 y 间隔。 

根据实现细节，这会产生 O(n^2 log n) 或 O(n^3) 解决方案，这对于 n ≤ 400 来说已经足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 显式修复矩形 | O(n^3) | O(n^3) | O(n) | 边界|
 | 扫描 x 对 + 优化 y | O(n^2 log n) | O(n^2 log n) | O(n) | 已接受 |

 ## 算法演练

 我们首先按 x 坐标对所有点进行排序，以便任何矩形的左边界和右边界都对应于此顺序的索引。 

1. 迭代排序列表中所有可能的左边界 l。 
2. 维护数组`yvals`当我们扩展 r 时，它将存储 l 和 r 之间的点。 
3. 对于从 l 到 n − 1 的每个 r，我们将点 r 插入活动集中并更新允许我们计算最佳垂直线段的结构。 
4. 每个活动集对应于 x[r] − x[l] 定义的固定宽度，因此 x 方向的周长贡献是固定的并等于 2 * (x[r] − x[l])。 
5. 现在我们必须计算 y 的最佳贡献。 我们按 y 对活动点进行排序，并考虑在这个排序列表中选择一个子段 [i, j]。 对于每个这样的片段，我们计算权重总和减去 2*(y[j] − y[i])。 
6. 我们维护排序 y 值的前缀和，以便可以在 O(1) 内计算段和。 然后，我们使用对调整值的单调优化来扫描所有可能的 i，同时保持最佳的 j 转换。 
7. 对于每个（l，r），我们计算最佳垂直选择并将其与水平成本相结合以获得总利润。 
8. 跟踪所有对 (l, r) 的最大值，包括选择单个点的可能性。 

关键思想是，在固定的水平条带内，问题分解为加权点上的一维优化，其成本与跨度成正比，这可以通过排序和前缀技术来解决。 

### 为什么它有效

 固定 x 边界会删除矩形的一个自由度。 任何最佳矩形都可以通过排序后按 x 顺序排列的最左边和最右边选择的点来唯一表示。 在该条带内，最佳 y 范围独立于其他条带，因为周长贡献干净地分为水平和垂直分量。 这种可分离性确保为每个 x 对独立优化 y 不会错过全局最优值，因为每个矩形在其极端 x 边界下只出现一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    pts.sort()

    ans = -10**30

    for l in range(n):
        active = []
        for r in range(l, n):
            x2 = pts[r][0]
            active.append((pts[r][1], pts[r][2]))

            # sort by y
            active.sort()

            m = len(active)
            prefix = [0] * (m + 1)
            for i in range(m):
                prefix[i+1] = prefix[i] + active[i][1]

            # try all y-intervals
            for i in range(m):
                for j in range(i, m):
                    total = prefix[j+1] - prefix[i]
                    y_cost = 2 * (active[j][0] - active[i][0])
                    width_cost = 2 * (x2 - pts[l][0])
                    ans = max(ans, total - y_cost - width_cost)

    print(f"{ans:.10f}")

if __name__ == "__main__":
    solve()
```代码直接实现了三层结构：左x边界、右x边界，然后对活动条带内的y间隔进行全扫描。 对于每个条带，我们重新计算排序的 y 顺序和前缀和，以有效地评估区间和。 周长分为 x 跨度的水平贡献和 y 跨度的垂直贡献。 

一个微妙的点是，我们明确允许 i = j，这对应于选择单个 y 级别，确保正确包含退化矩形。 

的初始化`ans`非常重要的负数很重要，因为如果周长占主导地位，最佳解决方案可能是负数。

 ## 边缘情况

 关键的边缘情况是所有点都位于同一条垂直线上。 在这种情况下，对于 l 和 r 的任何选择，x 成本都为零，并且解决方案完全简化为选择最佳 y 间隔。 该算法可以正确处理此问题，因为宽度成本变为 0，并且只有 y 跨度优化才能驱动结果。 

另一个边缘情况是单点输入。 该算法仍然有效，因为 l = r 生成大小为 1 的活动集，并且唯一考虑的区间是点本身，产生利润减去零周长。 

最后一个微妙的情况是，包含更多点会增加总利润，但也会显着增加垂直跨度。 该算法显式枚举每个 x 条带内的所有 y 间隔，因此它自然会比较“获取所有点”与“仅获取密集子集”，确保找到正确的权衡。
