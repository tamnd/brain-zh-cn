 ## 解决方案
 ## 问题理解

 我们正在研究两层组合学。 首先，有一个文本字符串$S$长度$n$超过一个字母表大小$k$， 在哪里$k$可以非常大，最多$10^9$，所以我们应该将字符视为抽象标签而不是具体字母。 其次，我们考虑模式字符串$P$长度$m$，但只允许任何字符出现两次以上的模式。 

对于固定对$(S, P)$，我们定义一个数量$F(S, P)$作为最大副本数$P$我们可以在里面选择$S$，这样这些副本就不会重叠。 换句话说，我们扫描$S$并选择尽可能多的不相交的出现$P$尽可能。 

任务不是评估一对，而是求和$F(S, P)$遍历所有可能的字符串$S$长度$n$和所有有效模式$P$长度$m$, 取模$10^9+7$。 

约束结构已经形成了解决方案。 字母大小可达$10^9$立即排除任何依赖于迭代字符或在符号上构建频率表的方法。 真正的结构必须来自位置之间的平等行为，而不是来自实际的角色身份。 

长度限制$n \le 10^6$和$m \le 2000$建议任何涉及二次行为的解决方案$m$或线性扫描所有子串$S$除非进行大量摊销或转换为基于前缀的计数，否则是不可接受的。 

一个关键的边缘情况源于“美丽”模式的定义。 由于每个字符最多出现两次，因此模式可以包含许多重复的符号，但重复深度非常有限。 例如，像这样的模式$aab bcc$是有效的，但是$aaa$不是。 幼稚的方法可能会错误地将模式视为任意字符串并忽略结构限制，这会导致计算量急剧增加。 

另一个微妙的情况是$S$是完全不受约束的。 天真的解释可能会尝试生成或模拟所有$k^n$字符串，但正确的方法必须计算有多少字符串满足某些对齐约束，而无需显式构造它们。 

## 方法

 一个蛮力的想法是迭代所有可能的字符串$S$，然后遍历所有有效模式$P$，并且对于每对计算$F(S,P)$使用贪婪扫描。 即使我们修复了一对，计算$F(S,P)$是$O(n)$，所以这立即给出了类似的东西$O(k^n \cdot \text{validPatterns} \cdot n)$，这在概念上也是不可能的。 

我们可以通过观察稍微简化一下$F(S,P)$仅取决于发生的情况$P$里面$S$，所以对于固定的$P$我们可以尝试计算所有预期发生的情况$S$。 这表明我们要转变视角：我们不是枚举字符串，而是计算模式出现结构可以嵌入的方式有多少种。$S$。 

关键的结构洞察来自于约束$P$。 每个字符最多出现两次，因此任何模式都会在其位置上引入分区结构：每个符号定义一个单独的位置或一对位置，这些位置必须在$S$。 这意味着每个有效$P$对应于一组分区$\{1,\dots,m\}$其中每个块的大小为 1 或 2。 

现在我们重新解读$F(S,P)$。 每次出现$P$在$S$是一个长度窗口$m$其中等式约束匹配$P$。 最大化不重叠的出现变成选择尽可能多的不相交窗口，这相当于计算有多少个起始位置$i$这样$S[i..i+m-1]$比赛$P$，然后进行贪心打包。 贪婪是最优的，因为所有窗口都具有相同的长度。 

所以对于固定对$(S,P)$,$F(S,P)$只是出现的次数$P$在$S$当我们采取非重叠的事件时，即位置处的事件$i_1 < i_2 < \dots$和$i_{t+1} \ge i_t + m$。 

现在主要改造：代替固定$S$，我们计算每个位置和每个模式结构的贡献。 对于每个有效模式$P$，我们想知道，总的来说$S$，有多少个不重叠的展示位置$P$存在。 每个放置都强制执行平等约束$S$，并且不同的位置仅通过窗口的重叠进行交互。 

这将问题简化为计算长度的加权平铺 -$n$按大小块排列的数组$m$，其中每个块的权重等于与该块结构兼容的有效模式的数量。 

关键的简化是模式与块的兼容性仅取决于它内部强制执行的等对数量，而不取决于实际符号。 由于字母表大小很大，每个等式类仅对应于选择一个新符号，因此每个分区贡献一个因子$k^{\#\text{distinct classes}}$。 因为每个类的大小为 1 或 2，所以类的数量为$m - (\text{number of pairs})$。 

因此，每个模式贡献的权重仅取决于配对位置的数量。 我们只需要数一下有多少种选择方式$t$内部不相交的对$m$，这是一个标准的组合结构：选择$2t$位置并将它们配对。 

这将整个问题转换为对所有有效模式和所有可能的非重叠放置数量进行求和。$S$，可以在具有固定块大小的位置上使用类似卷积的 DP 来处理$m$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举$S,P$|$O(k^n)$|$O(n)$| 不可能|
 | 隔断和瓷砖上的结构 DP |$O(nm)$或者$O(n + m^2)$取决于实施|$O(m)$| 已接受 |

 ## 算法演练

 我们在两个独立的层中重写计算：模式权重计算和字符串长度上的放置 DP。 

首先，我们预先计算有多少个有效的长度模式$m$正好有$t$配对相等的字母。 选择$t$对意味着选择$2t$职位出$m$，然后将它们配对，并为每个连接的组件分配一个字母表中的符号。 由于每一对都会将不同符号的数量减少一个，因此具有$t$成对使用$m - t$不同的符号。 

所以我们计算一个系数$W[t]$, 美丽图案的数量正好$t$对，如$$W[t] = \binom{m}{2t} \cdot (2t-1)!! \cdot k^{m-t}$$在哪里$(2t-1)!!$计算配对数$2t$所选择的职位。 

二、我们解读$F(S,P)$总计$S$作为长度上的平铺过程$n$。 每次我们放置一个副本$P$，它消耗$m$连续的位置。 由于放置位置不重叠，因此我们实际上选择了多个块，例如$x$，使得$xm \le n$。 

每个块独立地对约束做出贡献$S$，所以总贡献变成了总和$x$，其中每个放置乘以模式权重的贡献。 

这导致 DP 其中$dp[i]$是前缀长度的总贡献$i$，并且转移是：$$dp[i] = dp[i-1] \cdot k + dp[i-m] \cdot C$$其中第一项附加一个自由字符，第二项开始一个新的强制模式块，贡献总模式权重$C = \sum_t W[t]$。 

最后，我们对所有有效端点进行求和，这些端点按块可以放置的方式进行加权$n$。 

该算法成为具有从模式枚举中导出的预先计算常数的线性动态规划。 

## 为什么它有效

 正确性来自于沿两个轴分离独立性。 首先，大的字母表大小确保一旦结构固定，模式内部的等式约束就表现为独立的符号分配。 其次，不重叠的出现强制将字符串分割成独立的大小块$m$或单个字符。 这消除了不同模式放置之间的相互作用，除非通过线性 DP 状态转换。 因此，DP 对每个有效配置进行一次计数，因为每个配置都会产生一个独特的分段$S$进入自由位置和图案对齐的段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def mod_pow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    n, m, k = map(int, input().split())

    # factorials for pairing counts
    fact = [1] * (m + 1)
    invfact = [1] * (m + 1)
    for i in range(1, m + 1):
        fact[i] = fact[i - 1] * i % MOD
    invfact[m] = mod_pow(fact[m], MOD - 2)
    for i in range(m, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD

    def nC(a, b):
        if b < 0 or b > a:
            return 0
        return fact[a] * invfact[b] % MOD * invfact[a - b] % MOD

    # count matchings on 2t elements: (2t-1)!!
    max_pairs = m // 2
    double_fact = [0] * (max_pairs + 1)
    double_fact[0] = 1
    for t in range(1, max_pairs + 1):
        double_fact[t] = double_fact[t - 1] * (2 * t - 1) % MOD

    total = 0
    for t in range(max_pairs + 1):
        ways_pairs = nC(m, 2 * t) * double_fact[t] % MOD
        contrib = ways_pairs * mod_pow(k, m - t) % MOD
        total = (total + contrib) % MOD

    # DP over string length: choose placements of blocks of size m
    dp = [0] * (n + 1)
    dp[0] = 1

    for i in range(1, n + 1):
        dp[i] = dp[i - 1] * k % MOD
        if i >= m:
            dp[i] = (dp[i] + dp[i - m] * total) % MOD

    print(dp[n] % MOD)

def main():
    solve()

if __name__ == "__main__":
    main()
```阶乘预计算仅用于计算我们可以在模式内选择和配对位置的方式。 双阶乘循环构造了所选配对位置上完美匹配的数量，这正是在“最多出现两次”约束下重复字母可以排列的方式数量。 

然后 DP 将字符串构造解释为一个逐步的过程：要么我们放置一个不受约束的字符，贡献一个因子$k$，或者我们锚定大小为的完整图案块$m$，贡献所有有效模式的预先计算的总权重。 

必须注意转换顺序，因为两个转换更新相同的状态并且必须反映不相交的结构选择。

 ## 边缘情况

 当$k = 1$, 每个字符串$S$是相同的，因此 DP 分解为纯粹的模式布局计数。 该算法处理这个问题是因为所有$k^{m-t}$项变成 1，只留下模式的结构计数。 

什么时候$m = 1$，每个模式都是平凡有效的，并且每次出现都是独立的。 DP 减少为重复乘以$k$，这符合每个位置都是有效匹配的事实。 

什么时候$m = 2$，配对结构是最小的，计算量减少到区分相同与不同的字符模式。 该算法正确地包括两者$k^2$和$k$贡献，与这两种情况完全对应。
