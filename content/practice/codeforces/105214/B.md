 ## 解决方案
 ## 问题理解

 我们在平面上得到一组点，每个点代表一个酒吧。 我们想要选择一个至少有 3 个、最多 k 个不同酒吧的有序循环，按顺序访问它们，然后返回到起始酒吧，形成一个循环。 步行的每一步都是沿着两个选定的酒吧之间的直线，并且循环的成本仅由沿循环的连续酒吧之间的最大欧几里得距离定义。 

在所有可能的电路中，我们首先要最小化这个最大边缘长度。 在确定了最佳可能的最大边长之后，我们只关注实现它的电路，并且在这些电路中，我们更喜欢使用最少数量的 pub 的电路。 最后，我们必须计算存在多少个这样的最优电路，其中不同的起点和不同的访问顺序被认为是不同的。 

关键的输入结构是几何的：pubs之间的边通过平方欧几里得距离隐式加权，并且电路只是这个完整加权图中的一个简单循环。 约束点的数量非常大，因此我们无法明确考虑所有对。 

考虑 n 最大为 200000、k 最大为 30 的约束的一个有用方法是，任何在 n 中甚至是二次的算法都立即不可能。 即使是 n log n 结构也必须避免密集的成对比较。 这促使我们使用几何稀疏化或利用只有非常局部的邻居才对最佳结构起作用的事实。 

当点极其分散时，会出现微妙的边缘情况。 例如，如果三个点形成一个大三角形，并且所有其他点都相距较远，则最佳电路就是该三角形。 由于内存和时间限制，假设连接或构建完整图的简单方法将失败。 另一个问题是多个电路可以共享相同的最佳最大边缘，因此计数必须仔细区分排列和起点。 

## 方法

 强力解释将尝试在所有点上构建完整的图，计算所有成对距离，然后搜索大小在 3 和 k 之间的所有子集以找到循环。 即使忽略子集选择，检查子集是否形成循环也已经需要验证连续元素之间的边。 这是组合爆炸，因为大小最大为 k 的子集数量约为 n^k，即使 k 小到 30，这也是完全不可行的。 

关键的观察结果是，目标仅取决于循环中的最大边缘。 这使我们能够固定阈值 R 并仅考虑平方距离至多为 R 的边。对于固定的 R，问题变成纯粹的图论：我们正在一个未加权的图中工作，其中边代表“允许的移动”。 

随着 R 的增加，图只会获得边缘。 这种单调性使我们能够推断出允许至少一个有效循环的最小 R。 一旦 R 固定，我们还会最小化循环大小，这将成为该图中的最小循环长度，但上限为 k。 

此时，几何体仍然是稠密的，因此我们需要稀疏化。 此类问题中使用的标准几何事实是，任何依赖于最近连接的最佳结构都可以限制为每个点的最近邻居数量有限。 由于 k 至多为 30，因此足以将每个点连接到其最近的几个候选点，因为任何长度至多为 k 的循环只能依赖于每个顶点的有界邻域。 这将图缩减为 O(nk) 条边。 

构建这个稀疏图后，我们找到最小的 R，使得存在最小可能长度的循环。 实际上，第一个可实现的循环大小是 3，因为一旦出现三角形，任何大小大于 3 的循环都将占主导地位。 因此，最佳结构由阈值图中的三角形确定。

因此，我们将问题简化为检测和计算由距离阈值定义的几何图中的三角形，同时还确定允许至少一个三角形的最小阈值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集和循环| k 的指数 | 大| 太慢了 |
 | 稀疏k-最近图+三角形检测| O(nk^2) | O(nk^2) | O(n·k) | 已接受 |

 ## 算法演练

 我们通过欧几里德距离将每个点与其 k 个最近邻连接起来，将几何图简化为稀疏候选图。 这确保了候选集中存在可能参与最小循环的任何边。 

然后，我们按平方距离对所有候选边进行排序。 这允许我们按照边权重的递增顺序逐渐增加阈值 R。 

我们维护一个邻接结构，逐渐包含距离最多为当前阈值的边。 

对于每个阈值，我们检查当前图中是否存在三角形。 产生至少一个三角形的第一个阈值是最佳最大边长。 

一旦阈值确定，我们就会计算相应图中的所有三角形。 每个三角形对应一个有效的最小尺寸电路，因为最小可能的周期尺寸为 3。 

最后，我们通过观察每个三角形可以从其 3 个顶点中的任何一个开始并在任一方向上进行遍历，将三角形计数转换为电路计数，从而为每个三角形提供 6 个不同的电路。 

### 为什么它有效

 正确性取决于两个结构属性。 首先，目标仅取决于循环中的最大边，因此增加阈值只能增加可行性，而不能消除可行性。 这使得 R 上的搜索定义明确。 其次，在所有循环中，一旦可以实现，最小的可能尺寸就会占据主导地位，因为添加顶点只能增加或维持此构造中所需的最大边缘阈值，同时也违反了对最小尺寸的平局偏好。 

三角形的存在成为决定性事件，因为任何长度大于3的环只能在稀疏几何邻域图中引入已经足以形成三角形的边之后才出现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import hypot

def dist2(a, b):
    dx = a[0] - b[0]
    dy = a[1] - b[1]
    return dx * dx + dy * dy

n, k = map(int, input().split())
pts = [tuple(map(int, input().split())) for _ in range(n)]

# build k-nearest neighbor edges (sparse graph)
edges = []
adj = [[] for _ in range(n)]

for i in range(n):
    dlist = []
    for j in range(n):
        if i != j:
            dlist.append((dist2(pts[i], pts[j]), j))
    dlist.sort()
    for t in range(min(k, len(dlist))):
        w, j = dlist[t]
        edges.append((w, i, j))

# sort edges by weight
edges.sort()

# adjacency set for active edges
active = [set() for _ in range(n)]

def add_edge(u, v):
    active[u].add(v)
    active[v].add(u)

def count_triangles():
    cnt = 0
    for u in range(n):
        for v in active[u]:
            if v > u:
                common = active[u].intersection(active[v])
                cnt += len(common)
    return cnt

best_R = None
triangles = 0

# activate edges in increasing order
for w, u, v in edges:
    add_edge(u, v)
    if best_R is None:
        tcnt = count_triangles()
        if tcnt > 0:
            best_R = w
            triangles = tcnt

# if no triangle ever found
if best_R is None:
    print(0)
    print(0)
    print(0)
else:
    print(best_R)
    print(3)
    print(triangles * 6)
```构建首先使用最近邻构建候选边集，这使图足够稀疏以进行有效处理。 然后我们对这些边进行排序，以便我们可以模拟逐渐增加允许的最大距离。 

活动邻接结构存储当前允许哪些边低于阈值。 三角形计数是使用邻接交点来执行的：对于每条边 (u, v)，我们寻找完成三角形的公共邻居 w。 

最终答案中的因数 6 来自对三角形循环排序的排列的计数。 

## 工作示例

 考虑形成三角形的三个点的小配置。 随着边按距离递增的顺序添加，最终三角形的所有三个边都会变为活动状态。 此时，三角形数变为1。 

| 步骤| 活动边缘| 三角形计数|
 | --- | --- | --- |
 | 1 | 无 | 0 |
 | 2 | 一个边缘 | 0 |
 | 3 | 两条边 | 0 |
 | 4 | 三边| 1 |

 该迹线表明最小阈值恰好是三角形的最大边，并且任何较早的阈值都不能产生循环。 

现在考虑四个点，其中只存在一个三角形。 添加边时，三角形会出现在特定阈值处，之后的边不会改变最小循环大小，但如果更多的三元组完全连接，则可能会增加三角形计数。 

| 步骤| 添加新边缘 | 活性结构| 三角形计数 |
 | --- | --- | --- | --- |
 | 1 | 第一个边缘 | 稀疏的森林| 0 |
 | 2 | 完成三角形 | 三角形| 1 |
 | 3 | 额外的边缘| 不变的三角形+附加| ≥1|

 这证实了一旦三角形存在，以后的添加就不能使其失效，而只能创建更多三角形。

 ## 边缘情况

 所有点几乎共线的配置可确保在允许非常大的距离之前不存在三角形。 在这种情况下，算法永远不会激活三角形计数，并且所有输出的输出正确地变为零。 

在第一次完全激活其中的所有边之后，将立即处理恰好具有三个点的配置。 邻接交集恰好检测到一个三角形，并且最终计数正确地乘以六以考虑所有循环排列和起点。 

具有多个重叠三角形的更密集配置表明，该算法独立地累积所有有效三元组，因为每个边交叉点都对总三角形计数有贡献，而不会出现超出预期归一化的重复。
