 ## 解决方案
 ## 问题理解

 我们从代表城市和双向道路的连接无向图开始。 政府想要准确选择`k`并为每条选定的道路分配一个方向，以便在执行此操作后，每个城市仍然可以仅使用有向道路到达其他城市。 

关键的限制是我们不允许定向所有边，只能定向`k`其中。 其余边缘保持无向且无法用于定向行进。 所以最终的结构是通过pick形成的有向图`k`原始图中的边并为每个边指定一个方向。 

要求很严格：生成的有向图必须保持强连通。 这意味着对于每对顶点`u`和`v`，必须有一条有向路径`u`到`v`。 

输入约束意味着我们需要每个测试用例的近线性解决方案。 高达`2 * 10^5`整个测试中每个案例的边缘，任何比`O(n + m)`或者`O(m log n)`存在超时风险。 这将我们推向基于 DFS 或 BFS 的结构分解，而不是组合搜索。 

一种简单的方法会尝试选择边缘子集并测试方向，但即使在每次尝试后检查强连通性也是线性的，并且子集的数量是指数级的。 这立即排除了暴力。 

一些边缘情况值得尽早隔离。 

如果`k = 0`，我们被问到空有向图是否是强连通的。 这只有在以下情况下才可能实现：`n = 1`，否则这是不可能的，因为根本不存在可达性。 

如果图是一棵树（`m = n - 1`），那么即使我们对所有边进行定向，也无法形成有向环。 树没有环，因此不可能使其强连接`n > 1`。 

最重要的结构要求是有向图中的强连通性意味着存在覆盖所有顶点的环。 这表明我们需要依赖 DFS 树结构来创建后沿和循环。 

## 方法

 暴力方法会选择每个子集`k`边缘并尝试所有可能的方向。 对于每个配置，我们都会运行 BFS/DFS 检查是否具有强连接性。 即使对于中等`m`，子集的数量是`C(m, k)`，这是一个天文数字。 每张支票费用`O(n + m)`，所以这种方法立即崩溃。 

关键的观察结果是可以使用 DFS 生成树来强制实现强连接。 在任何连接的无向图中，DFS 都会生成树边和后边。 树的边缘定义了有根结构，后边缘提供了允许向上和向下遍历的循环。 

这个想法是沿着 DFS 定向边缘，以保持相邻 DFS 级别之间双向可达性。 但是，我们不需要定向所有边，只需`k`其中。 这意味着我们可以选择边的子集，形成足够丰富的结构以保证强连接性。 

一个关键事实是，只有当图包含至少一个环时，它才能具有强连通方向，更一般地，我们可以提取一种生成结构，其中每个树边都由至少一个形成环的后边支持。 DFS 自然地暴露了这样的边缘。 

我们以任意节点为 DFS 的根并考虑 DFS 树。 每一个树边`(u, v)`可以从父母到孩子进行引导。 对于每个后边缘`(u, v)`将节点连接到祖先，我们可以将其从后代定向到祖先。 这将创建沿 DFS 路径的定向循环。 

为了确保强大的连接性，我们不需要使用所有边。 我们只需要选取足够多的边，使得每个树边至少位于一个由后边形成的有向环上。 标准构造是采用所有树边缘和足够的后边缘来闭合每个子树边缘的循环。 

如果图是连通的并且至少有一个环，我们总是可以选择一组`k`包括所有 DFS 树边的边，除了可能的一些调整，具体取决于`k`，然后用后边缘填充剩余的选区。 可行性条件简化为图不是树：`k > n - 1`，并确保存在足够的后边缘以达到`k`。 

因此，构造策略是运行 DFS，收集树边和后边，然后按特定顺序选取边：首先获取所有树边，然后添加后边，直到达到`k`。 最后，将树边缘向下，后边缘向上。 

核心见解是，DFS 分类已经将边划分为维持连接结构的边（树边）和创建循环的边（后边），而强连接性需要至少有足够的循环边来闭合方向。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(米) | 太慢了|
 | 基于DFS的构建| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们假设该图是连通的。 

1. 从节点运行 DFS`0`，记录DFS树。 对于每次遍历`u`到一个未被访问过的`v`, 存储边缘`(u, v)`作为树的边缘。 这将构建一个组织所有顶点的有根生成树。 
2. 在探索邻接表时，每当我们遇到一个已访问过的非父节点时，将其记录为后边。 这些边表示图结构中的循环。 
3. DFS 完成后，我们有两个不相交的边集：定义可达性结构的树边和闭环的后边。 如果记录的边总数小于`k`，我们无法选择足够的边，所以答案是不可能的。 
4. 如果图是一棵树（不存在后边）并且`k > 0`，立即返回 NO，因为在定向设置中没有循环支持双向可达性。 
5. 按此顺序选择边：首先获取所有树边，然后获取后边，直到我们完全收集到为止`k`边缘。 如果我们在到达之前用完后边缘`k`，输出NO。 
6. 按 DFS 顺序将选定的树边从父级定向到子级。 这确保了从根向下的可达性。 
7. 将选定的后边缘从后代定向到祖先。 这确保了沿 DFS 血统的向上连接。 

### 为什么它有效

 DFS 树确保每个节点都可以使用定向树边从根到达。 后边缘创建有向循环，允许从任何子树遍历回其祖先。 因为每个节点都有一条通过后边缘链到根的路径，并且通过树边缘从根向下，所以我们获得了所有节点对之间的相互可达性。 不变量是每个子树至少有一个到祖先的传出后边缘，防止有向结构退化为单向层次结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n, m, k = map(int, input().split())
        adj = [[] for _ in range(n)]
        edges = []

        for i in range(m):
            u, v = map(int, input().split())
            adj[u].append((v, i))
            adj[v].append((u, i))
            edges.append((u, v))

        parent = [-1] * n
        used = [False] * m
        depth = [0] * n

        tree_edges = []
        back_edges = []

        def dfs(u):
            for v, eid in adj[u]:
                if used[eid]:
                    continue
                used[eid] = True
                if parent[u] == v:
                    continue
                if parent[v] == -1:
                    parent[v] = u
                    depth[v] = depth[u] + 1
                    tree_edges.append((u, v))
                    dfs(v)
                else:
                    back_edges.append((u, v))

        parent[0] = 0
        dfs(0)

        if len(tree_edges) + len(back_edges) < k:
            out.append("NO")
            continue

        if len(tree_edges) == n - 1 and k > len(tree_edges):
            out.append("NO")
            continue

        res = []
        for u, v in tree_edges:
            if len(res) < k:
                res.append((u, v))

        for u, v in back_edges:
            if len(res) < k:
                res.append((u, v))

        if len(res) < k:
            out.append("NO")
        else:
            out.append("SI")
            for u, v in res:
                out.append(f"{u} {v}")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现依赖于 DFS 边缘分类。 这`used`array 确保每个无向边被处理一次，避免重复分类。 当我们第一次发现一个新顶点时，会记录树的边；当我们遇到一个已访问过的非父顶点时，会记录后边。 

一个微妙的点是确保我们不会重复计算无向邻接列表中的边。 这`used[eid]`Guard 保证每条边都被准确分类一次。 

构建阶段只需选择第一个`k`来自树和后边缘的并集的边缘。 方向规则隐含在存储中`(u, v)`排序：树边遵循 DFS 方向，遇到后边时从后代到祖先存储。

| 步骤| 树边| 后边缘| 已选择 |

 我们总共选择 4 条边。 后边缘 1→4 创建一个循环，允许移动回祖先节点，确保所选结构的强连接性。

 ## 边缘情况

 树输入直接暴露了关键限制。 例如，`4 3 2`边形成一条线在 DFS 中没有后边，因此遍历后仅存在树边。 如果`k`超过`n-1`，选择立即失败，算法正确返回 NO。 

诸如三角形之类的全循环图可确保 DFS 生成至少一个后沿。 什么时候`k = m`，选择所有边，方向遵循 DFS 发现。 后边缘保证可以向上移动，防止定向结构倒塌成有根的树。 

最小图与`n = 1`和`k = 0`被接受是因为没有可违反的可达性约束。 DFS运行一般，输出为空，仍然满足空洞的强连接性。
