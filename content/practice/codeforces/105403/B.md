 ## 解决方案
 ## 问题理解

 每个测试用例都会给出两个分数数组，一组属于 Arturo，一组属于 Benito。 对于每个索引，我们都有一对值，但配对是完全未知的：我们可以按照我们想要的任何顺序将 Arturo 的数字和 Benito 的数字分配给不同的竞赛问题。 

对于每个问题，分配值较大的人将赢得该问题。 因为阿图罗的分数总是偶数，而贝尼托的分数总是奇数，因此永远不会出现相等的情况，因此每次配对都会产生严格的获胜者。 

任务不是重建单个分配，而是推理如何匹配两个多重集的所有可能排列。 在所有配对值的方法中，我们希望最大化 Arturo 获胜的比较次数。 

约束条件$N \le 5 \cdot 10^4$迫使任何解决方案最多$O(N \log N)$每个测试用例。 任何平方项$N$速度会太慢，因为分类或扫描 40 个大箱子就已经超出限制。 这立即表明我们需要在排序数组上使用贪婪匹配策略，而不是任何组合赋值搜索。 

当值重复时，就会出现一个微妙的问题。 如果 Arturo 有许多小偶数，而 Benito 有小奇数和大奇数的混合，那么我们匹配它们的顺序可能会显着改变结果。 一种天真的贪婪方法，如果没有仔细对齐最小到最小的比较，可能会在已经获胜的比赛上浪费大量的 Arturo 值。 

另一个边缘情况是所有 Arturo 值都大于所有 Benito 值。 在这种情况下，答案很简单$N$，但是如果实施不正确，粗心的贪婪算法会在早期消耗大量的 Benito 值，但仍然可能会浪费匹配。 

## 方法

 暴力解决方案将尝试将阿图罗的数字与贝尼托的数字配对的每种排列。 对于每个配对，我们计算有多少个位置满足$a_i > b_{\pi(i)}$。 这探讨了$N!$匹配，甚至对于$N = 10$这已经变得不可行了，因为$10! \approx 3.6 \cdot 10^6$，对于更大的情况，它会完全爆炸。 

关键的观察是，这是具有单调比较规则的完整二分结构上的最大匹配问题。 一旦两个数组都排序完毕，最佳策略仅取决于相对顺序，而不取决于同一性。 我们希望以保留尽可能多的“胜利”的方式将每个 Arturo 值分配给 Benito 值。 

如果我们对两个数组进行排序，我们可以将这个过程解释为尽可能将最小的与最小的配对，但在不可能获胜时跳过。 该结构与经典的“成对比较中获胜最大化”贪心问题相同：如果可能的话，我们要么使用弱 Arturo 值来击败弱 Benito 值，要么在必要时牺牲它来对抗更强的 Benito 值，以保留更强的 Arturo 值以供以后获胜。 

这自然会导致两指针贪婪策略。 我们对两个数组进行递增排序。 我们通过两个数组维护指针，总是尝试匹配仍然可以战胜最小剩余 Benito 值的最小 Arturo 值。 如果阿图罗当前的最小的比贝尼托的最小的大，我们就获胜。 否则，在这种贪婪结构中，Arturo 的当前值无法战胜任何更大的 Benito 值，因此我们推进 Benito 的指针并将该比较视为损失。 

这避免了在本来可以由较小的比赛获胜的比赛中浪费大的阿图罗价值，同时确保我们永远不会错过可能的胜利。

 | 蛮力 |$O(N!)$|$O(N)$| 太慢了|
 | 最优贪心 |$O(N \log N)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 将Arturo的数组和Benito的数组按非降序排序。 排序很重要，因为它公开了结构：一旦排序，我们就可以在本地推理出最佳匹配。 
2. 初始化两个指针，一个用于 Arturo，一个用于 Benito，两者都从排序数组的开头开始。 还为阿图罗的胜利初始化一个计数器。 
3. 将当前 Arturo 值与当前 Benito 值进行比较。 

如果 Arturo 的值更大，我们将这两个任务分配给同一个问题，并算 Arturo 获胜。 我们将两个指针都向前推进，因为两个值都被消耗了。 

如果Arturo的值不大于，那么这个Arturo值无法战胜剩余的最小的Benito值，也无法通过改进贪婪结构的方式战胜任何更大的Benito值。 因此，我们通过推进贝尼托的指针来放弃这个贝尼托值，有效地在阿图罗输掉的比赛中将其配对。 Arturo 指针保持在原位，因为该值仍可能战胜稍后的、较小的剩余 Benito 值（如果存在）。 
4. 继续此过程，直到其中一个指针到达其数组的末尾。 
5. 记录的成功比较次数是Arturo 可能获胜的最大次数。 

### 为什么它有效

 关键的不变量是，在任何时刻，当前指针之前的所有 Benito 值都已被视为与未来的获胜“无关”，因为它们已以无法提高获胜总数的方式配对或跳过。 同样，每个 Arturo 值要么已经与剩余候选者中的最佳对手匹配，要么保留以供将来比较。 

贪婪的决定是安全的，因为每当 Arturo 无法击败最小的可用 Benito 值时，在其他地方使用该 Benito 值不会比现在丢弃它创造更多的胜利。 相反，每当 Arturo 可以击败最小的 Benito 值时，将它们配对就是最佳选择，因为延迟 Arturo 只会冒稍后将其与更强的 Benito 值匹配的风险，而这不会增加获胜次数。 

每个指针步骤的局部最优性保证了整个序列的全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))

        a.sort()
        b.sort()

        i = j = 0
        ans = 0

        while i < n and j < n:
            if a[i] > b[j]:
                ans += 1
                i += 1
                j += 1
            else:
                j += 1

        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先读取多个测试用例。 对于每种情况，两个数组都已排序，以便我们可以应用单调贪婪扫描。 两个索引遍历数组。 当 Arturo 的当前价值超过 Benito 的当前价值时，我们承诺匹配并推进两个指数。 否则，我们会丢弃当前的 Benito 值，因为它无法被当前最小的 Arturo 候选者击败，并且我们在 Benito 的列表中向前移动，寻找更弱的对手。 

重要的实现细节是，在失败情况下，只有 Benito 的指针前进。 在那里推进阿图罗会错误地放弃稍后可能获胜的潜在有用价值。

 ## 边缘情况

 一个关键的边缘情况是 Arturo 有许多小值，而 Benito 有一些非常大的值与较小的值混合在一起。 如果我们错误地按顺序配对而没有策略性地跳过，则较小的 Arturo 值会浪费在较大的 Benito 值上，从而减少最终计数。 

贪婪指针策略通过始终丢弃当前最小的 Arturo 候选者无法击败的 Benito 值来避免这种情况，从而为未来的机会保留 Arturo 更强的值。 

另一种边缘情况是一侧的值一致。 如果所有 Arturo 值都相等，则正确性完全取决于它们是否超过足够的 Benito 值。 该算法自然地处理这个问题，因为比较严格是局部的，并且不假设值的多样性。 

最后，当所有值都对 Arturo 有利时，指针会同步前进，产生$N$立即执行，无需跳过。
