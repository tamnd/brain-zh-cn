 ## 问题理解

 我们得到一个写为紧凑字符串的算术表达式。 该表达式仅包含十进制数字、加号和乘号。 每个数字都是一个独立的数字，因此没有多位整数，也没有括号。 

任务是在标准运算符规则下计算此表达式，其中乘法比加法更强，并且在相同的运算符类型中，计算从左到右进行。 最终输出是完全计算表达式的整数结果，它可能会变得非常大，因此需要任意精度的算术。 

约束长度较小，最多 1000 个字符。 这立即排除了对二次或三次解析技术太慢的任何担忧。 即使 O(n²) 方法也可能存在，但没有必要。 如果我们正确处理优先级，线性扫描显然就足够了。 

一个微妙的陷阱来自于假设从左到右评估而不尊重优先级。 例如，口译`2+3*4`严格从左到右会给`(2+3)*4 = 20`，根据规则这是不正确的。 另一个错误是试图单独评估数字而没有正确分组乘法链。 

当乘法链中涉及零或表达式以长乘法序列开始或结束时，就会出现边缘情况。 例如，`0*1*2+3`在添加最后一项之前，必须将整个乘积正确地折叠为零。 

## 方法

 强力解释将重复扫描字符串并评估最高优先级操作或模拟完整括号。 一种方法是将表达式转换为所有可能的括号，或者通过查找乘法段并将其替换为计算值来重复减少表达式。 这很快就会变得低效，因为每次缩减都会移动字符串并需要重新扫描，从而导致 O(n²) 或更糟糕的行为。 

一种更结构化的强力方法是显式地将表达式转换为标记，并通过首先重复解决乘法，然后再解决加法来模拟求值。 即使这样，通常也需要多次遍历令牌列表。 在最坏的情况下，每次传递仅消除一个运算符，从而导致 O(n²) 行为。 

关键的观察结果是乘法仅在局部相互作用：它形成由加号分隔的连续段。 由于乘法具有更高的优先级并且是左结合的，因此该形式的每个最大段`a * b * c * ...`可以完全独立折叠。 一旦每个这样的段都减少到一个数字，就只剩下加法，并且可以在一次传递中累积。 

这允许单次线性扫描，其中我们维护当前乘法链的运行乘积，并在遇到加号时将其添加到运行总计中。

 ## 算法演练

 我们从左到右处理表达式，同时维护两个状态：当前的乘法累加器和最终的和。 

1. 从运行总和等于 0 且当前乘积等于 1 开始。 这些分别代表对迄今为止处理的所有内容和活动乘法链的评估。 
2. 逐个字符地读出表达式。 当遇到数字时，将其转换为整数并乘以当前的乘积。 这是有效的，因为数字属于乘法链，直到加号运算符打破它。 
3. 当遇到加号时，我们通过将当前乘积添加到运行和中来最终确定当前乘法链，然后将乘积重置为 1。 这反映了加法将独立的乘法块分开。 
4. 当遇到乘号时，我们不立即执行任何操作，因为乘法只会影响即将到来的数字如何分组为当前乘积。 
5. 处理整个字符串后，我们将最终的乘积添加到和中，因为最后一个乘法链不以加号结尾。 

关键思想是，乘法是通过连续将数字折叠成乘积来延迟处理的，而加法则将完成的乘积提交到最终结果中。 

### 为什么它有效

 在扫描中的任何点，当前乘积准确地表示自最后一个加号以来表达式的连续段的值。 这是成立的，因为乘法是关联的，并且链的左关联求值相当于增量乘法。 每当我们看到一个加号时，我们都会将表达式切割成独立的块，这些块的值不会进一步相互作用。 因此，对这些块积求和可以再现原始表达式的精确优先级正确的计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

total = 0
cur = 1

for ch in s:
    if ch.isdigit():
        cur *= int(ch)
    elif ch == '*':
        continue
    else:  # '+'
        total += cur
        cur = 1

total += cur
print(total)
```该实现直接反映了前面描述的不变结构。 变量`cur`累加完整的乘法段，同时`total`存储所有已完成段的总和。 

一个微妙但重要的细节是最后添加的`cur`循环结束后。 如果没有这一步，如果表达式不以加号结尾（根据定义总是如此），最后一个乘法链将永远不会被提交。 

另一个细节是乘法符号被显式忽略。 这是安全的，因为乘法不需要在读取运算符时执行操作，只需要在其连接的数字上执行操作。

 ## 边缘情况

 一种边缘情况是表达式包含以字符串边界结束的长乘法链，例如`2*3*4`。 该算法将数字处理成`cur`直到最后，并且由于没有出现加号，因此最后的添加步骤正确地提交了产品`24`。 

例如，另一种边缘情况是在链深处乘以零`9*0*5+7`。 乘法累加在第一个零位处立即崩溃为零，并且该零在链的其余部分中持续存在，直到它被添加到总数中，产生`0 + 7 = 7`。 

最后一个微妙的情况是混合两个运算符的连续操作，例如`8+2*3*4+1`。 该算法确保每次遇到加号时，当前的乘法段都会被冻结并相加，从而防止跨加法边界的任何乘法泄漏。
