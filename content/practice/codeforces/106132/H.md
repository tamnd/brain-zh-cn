 ## 解决方案
 ## 问题理解

 患者的体重以千克为单位，处方总药物剂量以毫克为单位。 该药物有一个以单位体重表示的安全指南：每公斤体重允许的安全剂量在 10 至 20 毫克（含）之间。 任务是通过将给定剂量与按患者体重缩放的允许间隔进行比较，对处方进行分类。 

具体来说，如果患者体重`w`，则最小安全剂量为`10w`最大安全剂量为`20w`。 如果按照规定的剂量`d`小于`10w`，处方不足。 如果`d`大于`20w`，超过了安全限度。 否则就在可接受的范围之内。 

约束很小：权重最多为 200，剂量最多为 10000。这立即意味着所有相关计算都可以轻松地适合 32 位整数，甚至在每个案例的恒定时间内跨多个测试用例进行重复评估也很简单。 除了直接算术比较之外的任何算法都是不必要的。 

边缘情况主要与边界相等有关。 一个常见的错误是将界限视为严格的不平等而不是包容性的不平等。 例如，如果`w = 10`，那么安全范围是`[100, 200]`。 剂量正好`100`必须被分类为安全，同样`200`也必须是安全的。 另一个潜在的错误是忘记正确重新计算两个端点，导致比较`d`反对`10`和`20`而不是按重量缩放。 

## 方法

 对问题的强力解释将模拟检查每公斤每一个可能的安全剂量，然后对其进行缩放并与处方进行比较。 对于给定的重量`w`，我们可以想象迭代所有值`10w`到`20w`并检查是否`d`符合任何有效剂量或超出范围。 这是可行的，因为有效区域是一个连续的区间，因此可以通过枚举来确定成员资格。 

然而，这是不必要的矫枉过正。 虽然`w`很小，真正的问题是概念性的：枚举整个范围与区间的大小呈线性关系，区间的大小随着`w`。 在最坏的情况下`w = 200`，范围的长度为 2000。虽然仍然很小，但这种方法在结构上变得比所需的更糟糕，并且如果扩展约束则无法扩展。 更重要的是，它隐藏了关键结构：安全区域完全由两个线性边界定义。 

观察结果是，有效集合是由乘法确定的单个区间。 一旦我们计算出端点`10w`和`20w`，问题简化为简单的三向比较。 这完全消除了迭代并减少了常数时间算术的决策。

 | --- | --- | --- | --- |
 | 暴力破解（扫描范围）| O(w) | O(1) | O(1) | 已接受但没有必要 |
 | 最优（直接边界检查）| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取患者体重`w`和规定的剂量`d`。 这两个值完全确定了安全范围，因此不需要处理额外的输入结构。 
2. 计算安全下限为`low = 10 * w`。 这代表了医学指南规定的最小有效剂量。 
3. 计算安全上限为`high = 20 * w`。 这代表副作用成为风险之前的最大安全剂量。 
4.比较处方剂量`d`违反这些界限。 如果`d < low`，将其归类为“太低”，因为它未能达到最低治疗阈值。 
5. 如果`d > high`，将其归类为“过高”，因为它超出了每公斤定义的安全限值。 
6. 否则，将其分类为“安全”，因为它位于闭区间内`[low, high]`。 

### 为什么它有效

 安全剂量定义与重量呈明确线性关系，这意味着每个有效处方必须满足`10w ≤ d ≤ 20w`。 该算法计算这些精确的边界并执行直接的隶属度测试。 由于两个不等式都被彻底检查并且间隔是连续的，因此不会错过任何有效值，也不会错误地接受无效值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    w = int(input().strip())
    d = int(input().strip())

    low = 10 * w
    high = 20 * w

    if d < low:
        print("Too Low")
    elif d > high:
        print("Too High")
    else:
        print("Safe")

if __name__ == "__main__":
    main()
```该解决方案读取两个整数并立即将医疗规则转换为数字范围。 乘法步骤至关重要，因为忘记缩放`w`会将问题简化为与常数 10 和 20 进行比较，这是不正确的。 

决策逻辑被构造为严格的案例排序。 “太低”检查必须首先进行，因为它隔离了低于该间隔的所有值。 “太高”检查排在第二位，涵盖其上方的所有值。 只有剩余区域是安全的，这避免了冗余比较。

 ## 边缘情况

 对于最小重量的情况`w = 1`，安全区间变为`[10, 20]`。 如果`d = 10`，算法计算`low = 10`和`high = 20`，以及条件`d < low`是假的，而`d > high`是假的，产生“安全”。 这证实了包含下边界的正确处理。 

对于像下面这样的边界情况`w = 1`,`d = 9`，我们计算相同的区间`[10, 20]`。 这里`9 < 10`立即触发，因此输出为“Too Low”，显示低于间隔的正确严格性。 

对于最大重量`w = 200`，区间变为`[2000, 4000]`。 如果`d = 4000`，两次比较都失败，结果是“安全”，确认正确包含上边界。 如果`d = 3999`，条件`d < 2000`是假的但是`d > 4000`假也还不够； 反而`3999 < 2000`是假的并且`3999 > 4000`是假的，等等，实际上我们评估正确：因为`3999 < 2000`是假的并且`3999 > 4000`是假的，如果逻辑错误，它会错误地进入安全状态，但这里我们使用正确的边界，所以`3999 > 4000`是假的，但我们还需要正确的排序，并且由于 3999 实际上小于 4000 但大于 2000，因此仅当在界限内时它才正确地落入安全状态，并且因为它在界限内`[2000,4000]`，这是安全的。 这证实了区间逻辑在整个范围的极值范围内是一致的。
