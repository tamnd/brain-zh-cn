 ## 解决方案
 ## 问题理解

 我们从恒等排列开始，这意味着数组包含按升序排列的从 1 到 n 的数字。 我们可以重复应用递归构造，其行为类似于二进制分裂过程。 在任何时候，我们都会获取一个连续的段，将其分成两个非空的连续部分，可以选择交换这两个部分，然后在每个部分内递归地应用相同的过程。 最后，将处理后的左右部分连接起来。 

关键的观察是这个过程不会任意排列元素。 它只允许通过将数组递归地划分为连续的块并可以选择在二叉递归树的每个内部节点处交换两半来构建排列。 

任务是确定给定的排列是否可以通过从排序数组开始的某些序列的分割和交换来生成。 

约束很大，测试用例的总 n 高达 2×10^5。 任何尝试模拟所有可能的分割点或所有递归结构的解决方案都会组合爆炸，因为每个段都有 O(n) 分割选择，并且每个分割都会创建两个递归分支。 即使单个测试用例也会导致指数行为。 

一个微妙的陷阱是假设这相当于“通过段反转可获得的任何排列”。 那是错误的。 例如，反转任意子数组比这种递归更强大，因为这里每个分割都必须在所有递归级别上保持连续性。 

另一个常见的错误是认为这只是检查排列是否是“分治可排序”而不需要交换，这会减少到更严格的条件。 每个节点的交换自由度至关重要，并且会显着改变结构。 

## 方法

 暴力方法试图重建递归树。 我们尝试当前段的每个可能的分割点，并且在每个分割处我们分支到交换或不交换。 选择分割后，我们递归地验证双方。 这形成了对n个标记叶子上的所有二叉树的递归，其结构是加泰罗尼亚语，但每个节点也有O(n)的分裂选择。 状态的数量呈超指数增长，即使 n 约为 20，也变得完全不可行。 

关键的结构洞察力是反转该过程。 我们不问如何构造数组，而是问每个递归步骤必须保留什么属性。 

在任何段，该过程都会采用一个连续的间隔并将其分成两个连续的部分。 由于我们从 [1..n] 开始，递归中的每个段都对应于某个值区间，而不一定是位置。 关键的事实是，在恒等排列中，任何有效段中的值必须形成连续的整数范围。 

现在考虑有效拆分时会发生什么。 假设我们正在处理与值范围 [l, r] 相对应的段。 任何有效的分割都会将该范围划分为两个不相交的连续值范围 [l, m] 和 [m+1, r]。 最终数组必须保留属于每一侧的所有元素在输出中显示为连续块，可能会交换。 

这将问题简化为检查排列是否可以分解为递归树，其中每个节点对应于一个连续的值区间，根区间为 [1, n]。 在每个节点，属于值区间的元素必须在排列中显示为连续段。 

关键的操作技巧是使用由位置范围和值范围引导的类似堆栈的贪婪分解来模拟该结构，验证每个段的行为是否类似于有效的递归区间。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力递归分割| 指数| O(n) | 太慢了 |
 | 通过贪婪递归结构进行区间验证 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过检查每个段是否可以解释为表示连续值范围的有效递归块来处理排列。 

1.将整个排列视为代表值区间[1,n]的候选段。 我们使用由数组中的位置范围及其预期值范围定义的间隔堆栈来跟踪段。 
2. 重复获取当前段并找到其中的最小值和最大值。 这两个极端决定了该段是否可以对应于有效的递归块。 如果该段有效，则其值必须完全形成一个连续的区间，没有缺失的数字。 
3. 如果段内的值不是连续范围（即 max - min + 1 不等于段长度），则该结构不能来自递归拆分，因此我们立即拒绝。 
4. 一旦某个段在值范围方面有效，我们就会根据值划分为两个连续组的位置，在概念上将其分为左右部分。 我们确定哪一侧包含较小的一半值，哪一侧包含较大的一半。 
5. 我们将生成的子段推回以进行进一步验证，递归地重复相同的推理，直到所有段都减少为单个元素。 
6. 如果所有段都可以一致地分解直到到达单元素块，则排列有效。 

重要的机制是每个递归块纯粹通过检查位置和值空间中的连续性来验证。 递归结构是通过在值边界上重复分割来隐式强制执行的。 

### 为什么它有效

 原始过程中的每个有效构建步骤都维护两个不变量。 首先，每个递归节点对应于数组中的一个连续段。 其次，该段内的值形成一个连续的整数区间，因为我们从完美排序的序列开始，并且仅通过交换整个子段来重新排列。 由于交换永远不会交错来自不同子树的值，因此递归不能混合来自不相交间隔的值。 

因此，任何有效的段必须始终满足其最小值和最大值精确定义其成员资格的属性。 这使得递归被唯一确定：一旦识别了一个段，其内部结构就被值分区所强制。 这消除了分割选择中的歧义，并将指数结构折叠为线性验证过程。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        stack = [(0, n - 1)]

        ok = True

        while stack:
            l, r = stack.pop()
            if l >= r:
                continue

            mn = min(a[l:r+1])
            mx = max(a[l:r+1])

            if mx - mn + 1 != r - l + 1:
                ok = False
                break

            mid = (mn + mx) // 2

            left_part = []
            right_part = []

            for i in range(l, r + 1):
                if a[i] <= mid:
                    left_part.append(i)
                else:
                    right_part.append(i)

            if not left_part or not right_part:
                ok = False
                break

            if left_part:
                stack.append((min(left_part), max(left_part)))
            if right_part:
                stack.append((min(right_part), max(right_part)))

        print("Yes" if ok else "No")

if __name__ == "__main__":
    solve()
```该实现使用堆栈来避免递归深度问题。 每个段由其索引间隔表示。 对于每个段，我们计算最小值和最大值以检查它是否对应于连续的值区间。 如果未通过此测试，我们立即拒绝该排列。 

然后，我们根据元素是否属于值范围的下半部分或上半部分将它们分为两组。 此步骤重建与连续值区间的二进制递归划分一致的唯一可能的划分。 生成的索引范围形成下一个要验证的段。 

计算子段时必须小心，因为索引必须按递归顺序保持连续。 使用最小和最大索引可确保我们在分区后正确恢复段边界。

 ## 边缘情况

 关键的边缘情况是一个段，其中值形成连续的间隔，但位置严重交错。 例如，`[3, 1, 4, 2]`仍然有 min=1 和 max=4，但该结构需要检查是否可以分成两个值区间，这两个值区间也显示为连续的位置段。 该算法通过强制在值的中点进行拆分来处理此问题，确保违反递归结构的交错会在分区期间导致拒绝。 

另一种边缘情况是尝试分裂后一侧变空。 当所有值都落入值范围的一半时，就会发生这种情况，而该值范围不能对应于有效的二进制递归节点。 该算法显式检查左右分区是否为非空，从而正确拒绝退化情况。
