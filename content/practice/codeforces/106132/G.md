 ## 解决方案
 ## 问题理解

 我们得到了一个由轴连接的有向齿轮网络。 每个轴将扭矩从一个齿轮传递到另一个齿轮，但只有一小部分输入扭矩得以保留，具体取决于该轴的效率百分比。 如果轴来自齿轮`u`齿轮`v`有效率`e`，那么任意扭矩`x`到达`u`贡献`x * (e / 100)`在`v`。 

对于每个查询，我们首先在源齿轮上应用固定扭矩`s`。 然后，该扭矩沿着定向边缘传播，并乘以沿途的效率。 如果有多个可能的路径到达目标齿轮`t`，除非我们考虑最大可能的传递扭矩，否则扭矩贡献不会以线性方式在不同路径上相加。 由于我们想要最好的可实现结果，因此我们正在有效地寻找路径`s`到`t`最大化边缘效率的乘积，然后将初始扭矩乘以该最大乘积。 

因此，每个查询都简化为在有向加权图中查找最大乘法路径值，其中边权重是`[0, 1]`。 

约束足够小，我们可以负担每个查询的最短路径样式计算。 和`n ≤ 1000`和`m ≤ 5000`，每个查询运行类似 Dijkstra 的算法是可行的，因为`q ≤ 100`。 像 Floyd-Warshall 这样的简单的全对预处理方法将涉及大约`n^3 ≈ 10^9`操作，这在 Python 中是边界但太慢了。 每个查询重新运行基于堆的最佳路径搜索完全在限制范围内。 

当考虑多条路径时，会出现一个微妙的问题。 贪婪松弛必须确保我们始终传播最已知的乘积，因为如果通过高效边缘扩展，次优中间值稍后可以变为最优。 这排除了任何 BFS 式的“先到者胜”的方法。 

重要的边缘情况：

 第一，目标遥不可及。 如果没有路径从`s`到`t`，答案一定是`0`，不是一个很小的浮点值。 例如，如果`s = 1`,`t = 2`，并且不存在路径，正确的输出是`0`，即使内部距离可能保持在`0`或者`-inf`取决于代表性。 

其次，效率超过 100% 的循环是不可能的，因为`e ≤ 100`，因此不存在放大。 这保证了路径积永远不会通过循环而增加，因此 Dijkstra 式的单调性成立。 

第三，浮点精度很重要，因为我们要乘以许多百分比。 使用双精度就足够了，因为所需的误差容限是`1e-6`。 

## 方法

 蛮力的想法是枚举所有可能的路径`s`到`t`，计算沿每条路径的效率乘积，并取最大值。 这是正确的，因为它直接符合问题的定义。 然而，图中带有循环的路径数量可能呈指数增长。 即使在中等连接的图中，不同的简单路径的数量也可能会爆炸超出任何可行的界限，使得这种方法在微小实例之外无法使用。 

更结构化的观点是注意到每条路径都贡献一个乘法分数。 如果我们通过取对数来变换边权重，则最大化乘积就相当于最大化对数值的总和。 这会将问题转换为对数空间中具有正边权重的图中的最长路径问题。 但是，因为值的日志`(0,1]`是非正的，我们实际上是在最大化负权重的总和，这相当于在求反后在图中找到一条具有非负边成本的最短路径。 与显式转换符号不同，直接对乘法权重运行 Dijkstra 风格的算法更简单，将松弛视为最大化。 

关键的观察是，节点的最佳值在松弛下是单调的：一旦我们为节点提供了更大的乘积，扩展它只能为邻居产生更好或相同的候选节点。 这允许优先级队列方法，我们总是扩展当前最知名的状态。 

由于我们有多达 100 个查询，因此每个查询运行这种单源最大乘积搜索非常高效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（所有路径）| 指数| O(n) 递归堆栈 | 太慢了 |
 | 最优（每个查询的 Dijkstra）| O(q·m log n) | O(q·m log n) | O(n + m) | 已接受 |

 ## 算法演练

 我们独立处理每个查询。 

### 1. 将效率转换为乘法权重

 每条边`u → v`有效率`e`被视为一个因素`e / 100`。 这是我们沿着路径相乘的值。 

### 2. 初始化最佳扭矩数组

 我们维护一个数组`dist[]`在哪里`dist[i]`存储可以到达节点的初始扭矩的最大分数`i`。 我们将所有值设置为`0`， 除了`dist[s] = 1`。 

这表明我们在发生任何传动损失之前保留了全部扭矩。 

### 3. 使用最大优先级队列

 我们存储状态`(current_best_factor, node)`在优先级队列中，按最高因子首先排序。 我们从`(1, s)`。 

我们使用最大堆行为（通过负值或反向比较实现），以便我们始终首先扩展最有希望的部分路径。 

### 4. 放松传出边缘

 处理节点时`u`与当前因素`f`，我们尝试改善所有邻居`v`通过`f * (e / 100)`。 如果该值大于`dist[v]`，我们更新`dist[v]`并将其推入堆中。 

此步骤确保对节点的任何改进立即向前传播，类似于 Dijkstra 松弛。 

### 5. 提前停止是可选的

 一旦弹出目标节点，我们就可以停止`t`从优先级队列中取出，因为此时我们已经找到了它的最优值。 

### 6.输出结果

 如果`dist[t]`遗迹`0`，该节点不可达，我们输出`0`。 否则我们乘以`dist[t]`由初始扭矩`T`。 

### 为什么它有效

 核心不变量是，每当从优先级队列中弹出一个节点时，关联的值就是该节点可实现的最大乘法因子。 这是成立的，因为所有边权重都在`[0, 1]`，因此延长路径永远不会增加其价值。 因此，任何后来发现的到已处理节点的路径都不能超过已经确定的最佳值。 这使得该过程在正确性上与 Dijkstra 算法相同，只是我们最大化而不是最小化。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    g = [[] for _ in range(n + 1)]

    for _ in range(m):
        u, v, e = map(int, input().split())
        w = e / 100.0
        g[u].append((v, w))

    for _ in range(q):
        s, t, T = map(int, input().split())

        dist = [0.0] * (n + 1)
        dist[s] = 1.0

        pq = [(-1.0, s)]

        while pq:
            neg_f, u = heapq.heappop(pq)
            f = -neg_f

            if f < dist[u]:
                continue

            if u == t:
                break

            for v, w in g[u]:
                nf = f * w
                if nf > dist[v]:
                    dist[v] = nf
                    heapq.heappush(pq, (-nf, v))

        print(dist[t] * T if dist[t] > 0 else 0.0)

if __name__ == "__main__":
    solve()
```该图存储为具有预先计算的概率的邻接列表。 对于每个查询，我们重置`dist`数组，因为每个查询都有独立的源。 

自Python以来，优先级队列使用负值来模拟最大堆`heapq`是一个最小堆。 我们还包括陈旧状态检查`if f < dist[u]`跳过过时的条目。 

到达后提前退出`t`在实践中减少了运行时间，但对于正确性来说并不是必需的。

 ## 边缘情况

 一种重要的边缘情况是目标无法到达。 假设我们有：```
3 1 1
1 2 50
1 3 100
```没有路径从`3`到`1`，所以算法初始化`dist[3]=1`但从未达到`1`。 优先级队列清空而没有访问目标，留下`dist[1]=0`。 输出正确变为`0`。 

另一种情况是存在具有不同长度和效率的多条路径时：```
1 → 2 → 3 (0.5 * 0.5 = 0.25)
1 → 3 (0.2)
```该算法首先发现`3`和`0.2`，然后改进为`0.25`。 松弛步骤确保改进不会被忽略，因为我们仅在从堆中提取节点的最佳值时才最终确定节点。 

第三种情况是自包含循环。 因为所有的权重最多`1`，反复遍历一个循环永远无法增加乘积。 即使存在循环，也不能创造无限的改进循环。 优先级队列自然会忽略重新访问具有较差值的节点，从而防止无限处理。
