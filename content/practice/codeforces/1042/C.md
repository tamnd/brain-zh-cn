 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，我们不断地减少它，直到只剩下一个值。 每个归约步骤要么通过将两个位置的值相乘并将结果存储到其中一个位置来合并两个位置，要么完全删除单个元素，但该删​​除操作最多可以使用一次。 

头寸的索引永远不会改变，这意味着即使某些头寸已被逻辑删除，我们也始终引用原始索引。 职位一旦删除，将永久无法使用。 恰好之后$n-1$操作时，只有一个数组单元格仍然包含一个值，我们希望最终值尽可能大。 

关键的约束是我们不被要求输出最终值本身，而是输出一个有效的操作序列，以实现最大可能的结果。 

约束条件允许$n$最多$2 \cdot 10^5$，这会立即排除任何尝试所有可能的合并订单的策略。 任何模拟不同消除树或尝试回溯贪婪决策的方法都会太慢，因为仅二进制合并结构的数量就会呈指数增长。 我们需要线性或近线性的结构。 

问题的一个微妙方面是单个可选删除操作。 这是丢弃某个值而不将其乘以最终结果的唯一方法。 这立即表明某些元素，通常是最有害的元素，应该被删除而不是用于乘法。 

重要的边缘情况：

 总是将所有东西相乘的天真贪婪者在出现负数时将会失败。 例如，奇数个负值会减少乘积。 如果我们有一个零和多个​​负值，选择是否删除零个或一个负值会完全改变结果。 例如，与$[-5, -4, 0]$，盲目地相乘所有结果为零，但删除零可以得到正积$20$。 

另一种失败情况是除了一个非零元素之外所有数字都为零。 任何合并顺序都可以，但是删除操作可以用来简化或避免不必要的乘法，粗心的逻辑可能会浪费它或放错地方。 

最后，同时具有正值和负值的数组需要决定是否要消除最小绝对值负数或可能为零，具体取决于负数的奇偶校验。 

## 方法

 强力解释将考虑所有可能的方法来选择要删除的元素（或不删除），然后考虑合并剩余元素的所有可能的顺序。 每次合并对应于在数组上构建一棵二叉树。 此类树的数量呈加泰罗尼亚式增长，大致呈指数增长$n$，甚至使得不可能$n = 40$，更不用说$2 \cdot 10^5$。 

一旦我们意识到最终值只是可能删除一个元素后所有剩余数字的乘积，问题的结构就会简化。 每次合并操作不会改变多重集乘积； 它仅跨步骤重新分配乘法。 因此，唯一真正的决定是删除哪个元素，因为其他所有元素都必须乘以最终结果。 

这将任务简化为最大化除最多一个被移除元素之外的所有数字的乘积。 最佳选择始终是删除最能改善总乘积的符号和大小的单个元素。 具体来说，我们的目标是确保负值的数量是偶数。 如果它已经是偶数，则删除零（如果存在）是有益的，因为它可以防止将产品折叠到零。 如果负数的数量是奇数，我们会删除绝对值最小的负值，因为它对翻转符号的贡献最小。 

一旦确定了删除选择，我们就可以通过将当前累加值重复乘以下一个剩余元素来构建操作序列。

 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 扫描数组并将索引分为正数、负数和零，同时跟踪绝对值最小的负数。 
2. 计算存在多少个负数。 如果该计数为奇数，则标记一个负数（绝对值最小的那个）进行删除。 这种选择使最终产品最大化，因为它在固定奇偶校验的同时最大限度地减少了幅度损失。 
3. 如果负数为偶数，则检查是否存在零。 如果是，请准确标记一个零以进行删除。 这可以防止最终产品不必要地变为零。 
4. 如果两个条件都不适用，则不执行删除操作。 该阵列已经具有最佳结构。 
5. 建立删除后剩余索引的列表。 
6. 选择第一个剩余索引作为累加器。 该索引将通过乘法运算逐渐吸收所有其他索引。 
7. 对于每个其他剩余索引，输出一个操作，将其值乘以累加器并删除源索引。 
8. 如果选择删除，请在任何乘法运算之前输出它，因为它必须在某个时刻执行，并且尽早执行可以避免干扰以后的合并。 

该结构确保除累加器之外的每个元素都被恰好消耗一次，并且累加器成长为最终产品。 

### 为什么它有效

 所有操作后的最终值始终是所有未显式删除的元素的乘积。 由于结合性，乘法的顺序不会改变结果。 因此，唯一的自由度是选择删除哪个单个元素（如果有）。 负奇偶性和零存在的贪心规则直接最大化乘积的符号和幅度，因此构造的序列必须达到最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

neg_indices = []
zero_indices = []
pos_indices = []

min_neg_idx = -1
min_neg_abs = 10**30

for i, v in enumerate(a):
    if v < 0:
        neg_indices.append(i)
        if abs(v) < min_neg_abs:
            min_neg_abs = abs(v)
            min_neg_idx = i
    elif v == 0:
        zero_indices.append(i)
    else:
        pos_indices.append(i)

delete_idx = -1

# if odd negatives, delete the smallest abs negative
if len(neg_indices) % 2 == 1:
    delete_idx = min_neg_idx
else:
    # if even negatives, prefer deleting a zero
    if zero_indices:
        delete_idx = zero_indices[0]

ops = []

# perform deletion first if chosen
if delete_idx != -1:
    ops.append((2, delete_idx + 1))

# build remaining list
alive = []
for i in range(n):
    if i != delete_idx:
        alive.append(i)

# if only one element remains
if len(alive) == 1:
    for op in ops:
        print(op[0], op[1])
    exit()

root = alive[0]

for i in alive[1:]:
    ops.append((1, root + 1, i + 1))

for op in ops:
    print(*op)
```该代码将索引分为三组，以快速决定要删除哪个元素。 在构建合并链之前计算删除决策，确保最终产品结构的正确性。 

然后，剩余的元素连接在一个简单的链中，其中第一个活动索引吸收所有其他元素。 这避免了对复杂树构造的任何需要，因为该问题不限制合并的形状。 

一个常见的微妙之处是正确处理索引，因为操作是从 1 开始的，而内部存储是从 0 开始的。 另一个微妙的点是确保删除最多发生一次，这是通过使用单个`delete_idx`。 

## 工作示例

 ### 示例 1

 输入：```
5
5 -2 0 1 -3
```我们对元素进行分类并决定删除。 

| 步骤| 负面| 零| 删除选择 | 活着集 |
 | --- | --- | --- | --- | --- |
 | 初始| 2 | 1 | 还没有 | 全部 |
 | 决定| 甚至负面| 零存在 | 删除一个零 | 删除索引 0 |

 删除后，我们连接剩余元素：

 | 运营| 行动|
 | --- | --- |
 | 2 3 | 删除零|

 | 1 2 4 | 1 2 4 中间产物吸收 1 |
 | 1 4 5 | 1 4 5 最终吸收|

 这会产生最大正乘积 30。 

跟踪显示，删除零可以避免产品崩溃，并允许所有非零值做出贡献。 

### 示例 2

 输入：```
3
-1 -2 4
```我们有两个负数，所以奇偶性已经是偶数了。 无需删除。 

| 步骤| 负面| 删除选择 | 活着集 |
 | --- | --- | --- | --- |
 | 初始| 2 | 无 | 全部 |

 然后我们按顺序合并：

 | 运营| 状态变化|
 | --- | --- |
 | 1 1 2 | 1 1 2 -1吸收-2变成2 |
 | 1 1 3 | 1 1 3 2 吸收 4 变成 8 |

 最终结果是 8，这是最佳结果，因为使用了所有值。 

这证实了当负平价已经平衡时，保留所有元素会产生最佳产品。

 ## 边缘情况

 一种重要的边缘情况是数组恰好包含一个非零元素和多个零。 该算法选择删除零，保留非零元素作为累加器。 用于输入`[5, 0, 0]`，删除会删除零，而合并只是将 5 作为最终值传播，而不与零相乘。 

另一种情况是没有零且只有一个负数。 为了`[3, -2, 4]`，算法删除`-2`因为它是唯一的负面因素，去除它会使产品变得积极。 然后剩下的乘法链产生`12`。 

最后一个微妙的情况是所有数字都为零。 在`[0, 0, 0, 0]`，没有有意义的负平价决定。 该算法删除一个零，然后将其余的链接起来，确保只有一个值保留下来并保持为零。
