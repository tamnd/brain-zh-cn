 ## 解决方案
 ## 问题理解

 我们得到了几张长凳，每张都已经坐满了一些人。 然后一群新人到来，每个人都必须选择一张长凳坐在那里。 当所有人都入座后，一张长凳将是所有长凳中人数最多的，这个值称为$k$。 

任务是计算最终最大负载的两个极值。 首先，我们想要最小的可能值$k$，假设我们尽可能均匀地分配新人，以避免任何一个替补席超载。 其次，我们想要最大可能的值$k$，假设我们故意集中人员，使一张长凳尽可能拥挤。 

约束足够小，任何解决方案都可以粗略地$O(n \log(\max a_i + m))$甚至$O(n \cdot m)$可能会过去，因为$n \le 100$和$m \le 10000$。 这意味着我们可以在答案空间上进行贪婪模拟或二分搜索，而不必担心性能问题。 

当试图贪婪地分配人员而不考虑未来的后果时，就会出现一个微妙的失败案例。 例如，总是将新人放在当前负载最少的替补席上看起来很合理，但并不能直接保证最优性，除非我们将其形式化为容量问题。 另一个边缘情况是当所有工作台都相同时，最小和最大答案都会崩溃到相同的值，并且如果不仔细处理，天真的不均匀分布思维仍然会产生错误的推理。 

## 方法

 的最大可能值$k$很简单。 由于我们希望一张长凳变得尽可能大，因此我们只需选择已经拥有最多人员的长凳并分配所有人员$m$那里的新人。 没有其他分配可以进一步增加最大值，因为将人员分散到其他地方只会减少添加到目标替补席的人数。 因此，最大情况完全由初始最大值决定。 

的最小可能值$k$更加结构化。 我们正在尝试分发$m$人们不让长凳变得太大。 这相当于要求最小的数$k$这样我们就可以“限制”所有长凳的高度$k$通过适当地安置新人。 

如果我们固定一个候选值$k$, 每个长凳$i$最多可以接受$k - a_i$在到达之前有更多的人$k$。 如果我们将所有长凳的容量相加，我们就得到了我们可以容纳的总人数，但不超过$k$。 如果这个总数至少是$m$，那么就可以分配每个人，同时保持最多的最大值$k$。 否则的话，那是不可能的。 

这将问题转化为单调条件$k$，这自然会导致二分查找。

 ## 算法演练

 我们将计算分为两个独立的部分：最大值和最小值。 

＃＃＃ 最大限度$k$1. 计算所有长凳的最大初始占用率。 
2.添加全部$m$同一张板凳上的新人。 
3. 返回结果值。 

这是可行的，因为将所有添加物集中在一个长凳上只能增加全局最大值，并且没有其他安排可以超过此构造。 

＃＃＃ 最低限度$k$1. 设置搜索范围$k$从$\max(a_i)$到$\max(a_i) + m$。 下限是当前最大值，因为我们无法减少现有负载。 上限是所有新人都坐在一张长凳上。 
2. 对于固定候选人$k$，计算每个长凳在不超过的情况下仍然可以容纳多少额外的人$k$，即$\max(0, k - a_i)$。 
3. 将所有工作台的这些能力相加。 
4. 如果总容量至少为$m$，这意味着我们可以分配所有新人而不超过$k$， 所以$k$是可行的。 
5. 否则，$k$太小了。 
6. 使用二分查找找到最小的可行值$k$。 

### 为什么它有效

 关键的不变量是可行性是单调的$k$。 如果某个$k$允许所有$m$人员被安置，然后任何更大的价值$k$还允许放置，因为每个长凳只会获得更多的容量。 这种单调性保证二分搜索收敛到最小的有效阈值，而不会丢失任何边缘情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(k, a, m):
    total = 0
    for x in a:
        if k > x:
            total += k - x
    return total >= m

n = int(input())
m = int(input())
a = [int(input()) for _ in range(n)]

mx = max(a)

# maximum k
max_k = mx + m

# minimum k via binary search
lo, hi = mx, mx + m
while lo < hi:
    mid = (lo + hi) // 2
    if can(mid, a, m):
        hi = mid
    else:
        lo = mid + 1

min_k = lo

print(min_k, max_k)
```该代码将问题干净地分成两个独立的计算。 辅助函数`can(k, a, m)`对之前导出的可行性条件进行编码。 二分搜索在答案空间上运行，而不是在索引上运行，这是安全的，因为条件在$k$。 

一个常见的错误是忘记包括`max(0, k - a_i)`逻辑，而不是对原始差异求和，这将错误地允许已经超满的替补席产生负面贡献。

 ## 边缘情况

 一个关键的边缘情况是只有一张长凳的情况。 在这种情况下，每个新人都必须去那里，因此最小和最大结果都会崩溃到相同的值。 该算法可以自然地处理这个问题，因为容量函数简化为单个项，并且二分搜索立即收敛到$a_1 + m$。 

另一个边缘情况是当所有工作台都已经有很大的值并且$m$很小。 二分搜索仍然有效，因为可行性仅取决于即使稍微增加阈值是否可以吸收所有新来者，并且搜索范围从当前最大值开始，确保不考虑无效的较低猜测。 

最后，当所有工作台都相等时，分布对称意味着分布始终是最小化最大值的最佳选择。 可行性检查通过将每个工作台视为贡献相同的容量来自动捕获这一点，因此搜索完全收敛于平衡阈值。
