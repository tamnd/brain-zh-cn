 ## 解决方案

 ## 问题理解

 我们有一棵树，其中只有一些顶点重要：那些度数恰好为 1 的顶点，我们称之为叶子。 该任务不是对所有顶点进行分组，而只是对这些叶子进行分组。 

每个组必须满足树度量内的几何约束。 如果我们取同一组内的任意两片叶子，它们沿树的距离不得超过$k$。 我们需要将所有叶子划分为不相交的组，以便每个组在此约束下都有效，并且我们希望最小化使用的组数量。 

输入大小可达$10^6$顶点，这会立即强制线性或近线性解决方案。 任何试图计算叶子之间的成对距离的方法，即使是间接的，都会失败，因为叶子的数量可以是线性的$n$，并且成对推理会导致二次行为。 同样，任何重复重新计算距离或从每个叶子执行 BFS 的解决方案都是不可行的。 

一个微妙的点是，树结构本身很大，但只有叶子与分组相关。 然而，内部顶点仍然很重要，因为它们决定叶子之间的距离。 

如果我们尝试在本地贪婪地对叶子进行分组而不考虑全局结构，就会出现幼稚的失败模式。 例如，在所有叶子都连接到中心的星形树中，每对叶子的距离为 2。如果$k = 1$，没有两个叶子可以组合在一起，但是按邻接对叶子进行分组的贪婪方法会错误地通过中心将它们合并。 相反，如果$k$如果很大，当来自不同分支的叶子违反直径约束时，按子树邻近度进行分组可能仍然会失败。 

另一个失败案例出现在长链中。 叶子仅位于端点，基于子树边界的朴素聚类很容易低估沿链累积的距离。 

## 方法

 一个直接的蛮力想法是计算叶子之间的所有成对距离，然后尝试将它们划分为最小数量的集合，其中每个集合的直径最多$k$。 这成为完整叶子图上的图划分问题，其中边权重是树中的距离。 甚至计算距离矩阵成本$O(L^2)$在哪里$L$是叶子的数量，可以是$O(n)$，使得这个二次方程变得不可能。 

即使距离可用，划分步骤也相当于用最小数量的直径有界簇覆盖度量空间中的点，这在一般度量中是 NP 困难的。 因此，暴力破解不仅太慢，而且在结构上与树度量不一致。 

关键的观察结果是，树度量沿着任何路径结构都是一维的，并且叶到叶的距离由它们的最低共同祖先决定。 我们可以将问题转换为沿 DFS 顺序在叶子上进行配对的过程，而不是直接推理所有对。 

如果我们给树扎根，那么每片叶子都有明确定义的深度。 考虑从叶子向上行走并尝试将叶子匹配到组中，以便每个组在树中的“跨度”保持在$k$。 我们不是明确地维护所有成对距离，而是维护叶子可以“向上到达”多远，同时仍然与群体兼容。 

一种更具结构性的重新表述是在 DFS 遍历中贪婪地配对叶子：当我们处理完子树时，我们向上携带有关不匹配叶子的信息。 每次我们组合两个这样的部分结构时，我们都会决定它们是否可以合并成一个有效的组，或者必须保持独立，具体取决于它们的距离是否超过$k$。 

这将问题转化为树上的局部组合问题，其中每条边都有助于距离累积。 关键的想法是，在任何最佳解决方案中，同一组内的叶子根据其成对路径形成一个连接区域，我们可以自下而上地处理这个问题，同时只跟踪每个子树有限数量的“活动叶子”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（所有叶对+聚类）|$O(n^2)$|$O(n^2)$| 太慢了|
 | 树DP/DFS贪婪合并|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将树的根设在任意顶点（通常为 1），并执行 DFS。 

每个子树返回有关仍“未解析”的叶子的信息，因为它们尚未完全分配到完全在子树内的完整组中。 

1. 我们从根开始运行 DFS，并将每个节点视为其子节点的组合结果。 这是很自然的，因为任何叶组都是由位于树的某个连接区域的叶子形成的。 
2. 对于每个节点，我们收集来自其子节点的所有候选“活动叶路径”。 每个这样的值代表该子树中的一个叶子以及从该叶子到当前节点的距离。 这种抽象将下面的所有内部结构压缩为单个距离值。 
3. 我们合并来自不同孩子的候选列表。 在合并时，我们总是尝试以贪婪的方式将相距最远的叶子配对，因为距离大的叶子首先有违反约束的风险。 这相当于按深度对候选进行排序并尝试匹配极端值。 
4. 每当我们考虑两个有距离的候选叶子时$a$和$b$从当前节点开始，如果通过该节点配对，它们在树中的距离为$a + b$。 如果这个值最多是$k$，它们可以属于同一组，我们将它们作为一对删除。 
5. 如果总和超过$k$，我们无法将它们合并，并且更深的一个必须保留为未解决的候选者才能向上传递。 如果另一个仍然有可能在其他地方合并，那么它也会被保留，但在实践中，贪婪的结构确保我们只向上携带最小的代表集。 
6. 处理完节点的所有子节点后，我们返回剩余的不匹配候选节点，将其深度增加 1 以考虑向上移动。 
7. 最后，在根处，所有剩余的候选者代表不能在约束内配对的组的端点。 组的数量是根据剩余的此类端点的数量得出的，并根据 DFS 期间成功形成的对的数量进行调整。 

### 为什么它有效

 关键的不变量是，从子树返回的每个候选者都代表尚未完全分配给该子树内部的完整有效组的叶子，并且其存储的值是到其当前部分配对结构中最深点的精确距离。 

因为树中的任何有效组都必须通过最低公共祖先连接，所以每个可行的配对都必须在其 LCA 中表示为合并候选。 贪心策略总是首先尝试解决最具限制性的对，以确保如果在任何最佳解决方案中都可能出现配对，则不会被早期决策所阻止。 树结构保证一旦两个叶子在给定节点处相距太远，后面的祖先就无法缩短它们的距离，因此无法在本地将它们配对是安全的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n, k = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    INF = 10**18
    ans = 0

    def dfs(u, p):
        nonlocal ans
        cur = []

        for v in g[u]:
            if v == p:
                continue
            child = dfs(v, u)
            for d in child:
                nd = d + 1
                if nd <= k:
                    cur.append(nd)

        cur.sort()

        i, j = 0, len(cur) - 1
        while i < j:
            if cur[i] + cur[j] <= k:
                i += 1
                j -= 1
                ans += 1
            else:
                j -= 1

        res = []
        while i <= j:
            res.append(cur[i])
            i += 1

        return res

    dfs(1, -1)
    print(ans)

if __name__ == "__main__":
    solve()
```DFS 返回每个子树的候选叶深度列表。 每个深度表示叶子距当前节点的距离。 当向上移动时，我们将子深度增加一，这保留了真实的树距离。 

排序允许我们贪婪地匹配最小和最大的剩余候选者，这是最优的，因为如果最小的不能与最大的配对，它也不能与任何更大的配对。 

每个成功的配对都会增加答案，因为它形成一个完整的组，其直径约束在以当前节点为根的子树内得到满足。 

返回的列表仅保留未配对的候选者，它们是将在树中更高层解析的组的潜在端点。 

递归限制增加，因为树可以是长度链$10^6$，否则会溢出 Python 的默认递归堆栈。

 ## 边缘情况

 在纯路径图中，所有叶子都是长链的端点。 该算法自下而上进行处理，携带单个候选距离。 在每一步中，直到到达根部才可能进行配对，在根部做出最终配对决策，产生正确的最小分组。 

在星形图中，所有叶子都在根处相交，因此所有距离恰好为 2。如果$k \ge 2$，根立即以贪婪的方式配对叶子。 如果$k < 2$，不会发生配对，每片叶子都成为自己的群体。 DFS 确保两个结果得到一致的处理，因为配对始终在 LCA 最小的节点进行检查。
