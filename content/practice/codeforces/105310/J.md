 ## 解决方案
 ## 问题理解

 我们有一个大小为的正方形网格$n \times n$和一组恰好$k$那些和$n^2 - k$零。 任务是将所有值放入网格中。 

网格构建完成后，我们将每一行读取为二进制字符串，将每一列读取为二进制字符串。 我们想要最小化的“分数”是所有行加上所有列中不同字符串的数量，将行和列一起视为单个集合。 

因此，如果许多行看起来相同，或者许多列重复相同的模式，则分数会变小。 如果每一行和每一列都不同，则分数会变大。 

约束条件$n \le 1000$意味着网格有多达一百万个单元。 任何尝试搜索布局或模拟许多候选网格的解决方案都会立即变得太慢。 我们必须直接构造一个模式，使用线性或近线性构造$n^2$。 

一个微妙的边缘情况是$k$非常小或者非常大。 例如，当$k = 0$，网格被迫全为零，并且所有行和所有列都相同。 答案很简单，就是 1 个不同的行和 1 个不同的列，因此构造必须彻底退化，没有特殊情况的错误。 同样，当$k = n^2$，一切都是一。 

另一个重要的情况是当$k$接近于$n$或者$n^2 - n$，其中天真的贪婪行填充往往会创建许多不同的行模式，同时意外地增加列多样性，这正是目标所惩罚的。 

## 方法

 一个蛮力的想法是尝试所有的分发方式$k$跨越的$n^2$单元格并计算不同行和列的结果数量。 这在组合上是不可能的，因为网格的数量是$\binom{n^2}{k}$，这对于$n = 1000$是一个天文数字。 

即使我们限制自己贪婪地逐行构建，我们仍然面临硬耦合：更改单个单元格会影响其行和列，这意味着本地决策会在全球范围内传播。 任何试图在不控制列的情况下“最小化不同行”的策略都会很快失败，反之亦然。 

关键的观察结果是，不同的行和列不是由单独的位置驱动的，而是由网格的结构决定的。 如果行重复少量模式并且列重复少量模式，则分数很小。 在两个方向上同时强制重复的最稳定方法是构建一个网格，其中结构仅取决于索引之和，即反对角线。 

如果我们按照从左上角到右下角的对角线顺序填充网格，并沿着这些对角线连续放置网格，我们可以确保任何行或列都是可预测模式的前缀。 这迫使行和列之间存在高度重复，因为每一行都移动相同的对角线模式，并且每一列都会执行相同的操作。 

该结构将问题简化为分布$k$以固定顺序沿对角线排列，而不是选择任意单元格。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数在$n^2$|$O(n^2)$| 太慢了|
 | 对角线结构|$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们使用对角线顺序确定性地构建网格。 

1. 初始化一个$n \times n$网格充满零。 这确保了所有行和列最初都相同的有效基线。 
2. 迭代索引为的对角线$s = 0$到$2n - 2$。 每条对角线包含所有单元格$(i, j)$这样$i + j = s$。 我们按升序处理对角线，以便填充的单元格按此顺序形成连续区域。 
3. 对于每条对角线，迭代所有有效单元格$(i, j)$在那条对角线上。 如果仍有剩余配额，则为单元格分配 1$k > 0$，并递减$k$。 否则将其保留为 0。 
4. 如果出现以下情况，请尽早停止$k = 0$，因为所有必需的都已放置。 
5. 输出结果网格。 

我们使用对角顺序而不是行优先顺序的原因是行优先顺序创建了长的水平前缀，这使得前面的行密集而后面的行保持空，产生许多不同的行模式。 对角顺序可以在两个维度上更均匀地分布。 

### 为什么它有效

 通过按对角线顺序填充创建的网格可确保任意两行仅在对角线滑动窗口的位置上有所不同，并且列也具有相同的对称性。 由于所有行都是由相同的全局对角线扫描形成的，因此不同行模式的数量受到严格控制，这同样适用于列。 该构造强制执行共享的全局结构，而不是独立的行式或列式结构，这使得组合多样性最小化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    grid = [[0] * n for _ in range(n)]
    
    for s in range(2 * n - 1):
        for i in range(max(0, s - (n - 1)), min(n, s + 1)):
            j = s - i
            if k > 0:
                grid[i][j] = 1
                k -= 1
    
    for row in grid:
        print("".join(map(str, row)))

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个完整的零网格，确保每个位置都可以有效分配。 然后，它迭代反对角线，仔细计算有效的索引范围，以便仅访问边界内的单元格。 

停止条件隐含在检查中`if k > 0`，这可以防止过度填充。 这很重要，因为耗尽后继续填充$k$会违反输入约束。 

对角线索引逻辑是最微妙的部分：对于固定的总和$s$， 有效的$i$范围从$\max(0, s - (n - 1))$到$\min(n - 1, s)$。 这里任何相差一的错误都会导致索引崩溃或丢失单元格。

 ## 边缘情况

 当$k = 0$，对角线循环运行但没有发生分配，使网格完全为零。 每一行都是相同的，每一列都是相同的，从而产生尽可能小的多样性。 

什么时候$k = n^2$，在对角线遍历期间每个单元格都被填充。 提前终止条件永远不会被触发，并且网格变为全1，再次产生单行类型和单列类型。 

什么时候$k$非常小，只有前几条对角线被部分填充。 这避免了在单行或单列内创建长的结构化模式，因为对角线本质上将早期布局分布在多行和列上。
