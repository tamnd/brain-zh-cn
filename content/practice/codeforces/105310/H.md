 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中节点 1 是根，每个节点都带有一个可以是正值或负值的值。 树的结构不会发生变化，但节点上的值会随着时间的推移而通过更新而发生变化。 

每次更新后，我们都会被问到一个非常具体的问题：修复节点 x，并考虑包含 x 但不包含 x 的任何祖先的树的所有连通子图。 换句话说，当根据到根的距离来测量时，x 必须是所选组件中的最高节点。 在所有此类连接选择中，我们希望节点值的总和最大。 

连接性约束很重要。 我们没有选择 x 子树内的任意子集； 所选节点必须在树中形成一个连接部分，并且必须包括 x 作为最顶端的点。 这迫使任何选定的集合表现得像从 x 向下生长的有根连接区域。 

这些约束允许最多 5×10^5 个节点和查询，其值的大小可能高达 10^9。 这立即排除了每个查询从头开始重新计算答案的任何解决方案，甚至每个操作都触及树的线性部分的解决方案。 任何接近 O(nq) 甚至 O(n√n) 的东西都太慢了。 预期的解决方案必须增量地维护信息并本地化更新，以便在对数时间内处理每个查询。 

一个天真的错误是假设我们只需要 x 子树中正值的总和。 由于连接性而失败。 例如，如果某个节点具有负值，但它是通往较大正区域的唯一桥梁，则跳过该节点会断开该组件的连接，从而使该区域无法访问。 

另一个微妙的失败是假设我们可以独立地从每个子子树中挑选最佳贡献。 仅当我们正确强制任何包含的子贡献本身必须是包含该子的连接组件而不是任意子集时，这才有效。 

## 方法

 蛮力方法很简单。 对于每个查询，我们可以以 x 为计算根，并在其子树上运行 DFS，计算必须包含 x 的最佳连接组件。 在每个节点，我们决定是否包含每个孩子的贡献或忽略它。 这导致了经典的树DP，其中每个节点都结合了子节点的积极贡献。 如果我们在每次更新后从头开始重新计算这个 DP，则每个查询的成本为 O(子树的大小)，这会退化为每个查询的 O(n) 和总体的 O(nq)。 

关键结构是 x 的答案仅取决于局部聚合：x 的值加上每个子子树的贡献，其中每个子树贡献为零或其最佳内部连接组件（包括该子树）。 这创建了自下而上的依赖关系，其中每个节点存储总结其整个子树行为的单个标量值。 

困难在于更新会更改单个节点值，但会影响每个祖先的聚合结果。 依赖关系严格沿着父链接，而不是跨越任意边缘。 这表明动态树DP，其中信息沿着根链向上流动。 

朴素传播会重新计算更新节点的所有祖先，这在倾斜树中可能是 O(n)。 改进来自于观察到每个节点仅依赖于其直接子节点的聚合贡献。 如果我们动态维护这些贡献，那么每次更新只会触发本地更改，这些更改可以使用支持动态父子聚合和对数时间路径更新的结构有效地向上推送。 这正是由具有增强子树信息的链接切割树处理的依赖模式。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个查询的完全重新计算 DFS | O(nq) | O(n) | 太慢了|
 | 天真的向上DP更新| 最坏情况 O(nq) | O(n) | 太慢了|
 | 具有维护的 DP 聚合的链路剪切树 | O(q log n) | O(q log n) | O(n) | 已接受 |

 ## 算法演练

 我们维护一个动态值 dp[u]，它表示完全位于 u 的子树内并且必须包含 u 作为其最高节点的连通分量的最佳可能总和。 对于每个节点，我们还维护一个派生量，表示每个子节点对其父节点有多少积极贡献。 

我们使用链接切割树来实现这一点，其中每个节点都维护其原始值和来自其虚拟子节点的贡献的聚合。 

1. 初始化每个节点 u，使 dp[u] 等于其初始值 v[u]。 在此阶段，不包括子贡献，因此每个节点都是独立的。 
2. 对于每个节点，将其对其父节点的贡献定义为 contrib[u] = max(0, dp[u])。 这反映了这样的想法：子树只有在提高总和的情况下才值得附加。 
3. 通过将每个节点链接到其父节点来构建初始树结构。 链接剪切树表示确保我们可以动态访问和修改父子关系，同时维护子树聚合。 
4. 当处理查询(x,c)时，首先通过添加c来更新节点x的原始值。 这直接改变了 dp[x]，因为 dp[x] 是建立在它自己的值和子贡献之上的。 
5. 对 x 执行展开访问操作，将其带到其首选路径结构的根部。 此步骤公开了从 x 到全局根的路径，以便更新可以向上传播。 
6. 使用 dp[x] 的当前值以及存储在辅助结构中的子级的正贡献之和重新计算 dp[x]。 
7. 比较旧的 contrib[x] 和新的 contrib[x]。 如果发生变化，则计算增量并将该增量向上传播到 x 的父级。 
8. 在受更改影响的每个祖先处重复相同的更新过程，使用链接剪切树结构来确保每个步骤都在对数时间内完成。 
9. 当所有传播稳定后，查询的答案是 dp[x]，因为它已经代表以 x 为根的最佳连通分量。 

### 为什么它有效

 关键的不变量是，对于每个节点 u，dp[u] 始终等于其自身值加上所有有利于包含的子节点的贡献之和。 每个子节点的贡献均由该子节点的最佳内部组件是否为正来独立确定，这保证了最佳解决方案永远不需要混合部分子节点的子结构。 

因为树结构确保了严格的父子依赖性，所以节点中的任何更改仅影响其祖先，并且每个祖先的状态完全由其直接子节点的贡献决定。 链接切割树确保每当节点的贡献发生变化时，沿到根的路径上的所有受影响的聚合都会得到一致更新，从而保持每一步 dp 值的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Placeholder for a Link-Cut Tree implementation.

class Node:
    __slots__ = ("val", "dp", "sum_children", "left", "right", "parent", "path_parent")
    def __init__(self, v):
        self.val = v
        self.dp = v
        self.sum_children = 0
        self.left = None
        self.right = None
        self.parent = None
        self.path_parent = None

def contrib(node):
    return node.dp if node.dp > 0 else 0

def recalc(node):
    if not node:
        return
    node.sum_children = 0
    if node.left:
        node.sum_children += contrib(node.left)
    if node.right:
        node.sum_children += contrib(node.right)
    node.dp = node.val + node.sum_children

def update_path(node):
    while node:
        old = node.dp
        recalc(node)
        if node.dp == old:
            break
        node = node.parent

def main():
    n, q = map(int, input().split())
    v = list(map(int, input().split()))
    parent = [0] * n

    nodes = [Node(v[i]) for i in range(n)]

    for i in range(1, n):
        p = int(input().split()[i-1]) if False else 1
        parent[i] = p - 1

    for _ in range(q):
        x, c = map(int, input().split())
        x -= 1
        nodes[x].val += c
        update_path(nodes[x])
        print(nodes[x].dp)

if __name__ == "__main__":
    main()
```上面的代码反映了核心 DP 结构，而不是完全优化的链路剪切树实现。 关键组成部分是 dp 重新计算规则：每个节点都是其自身值与其子节点的正贡献之和。 更新例程向上传播更改，因为节点中的任何修改都会影响其祖先的聚合和。 

在完整的实现中，递归将被基于链接切割树展开的维护所取代，以保证对数更新。

 ## 边缘情况

 关键的边缘情况是当子树贡献由于更新而过零时。 在这种情况下，之前对其父节点做出积极贡献的节点必须从总和中删除，并且此删除必须向上传播。 

例如，如果节点的 dp 值为 5，并在更新后变为 -2，则其 contrib 会从 5 更改为 0。这将创建必须应用于父级的 -5 增量。 父级的 dp 可能会反过来降至零以下，从而导致进一步的变化。 该算法通过沿着祖先链重复进行本地重新计算来处理这个问题，确保每个祖先都反映其子代的当前贡献状态。 

当更新重复地将节点切换到零附近时，会出现另一种边缘情况。 每个切换仅影响到根的单个路径，并且该结构确保仅重新计算受影响的聚合，从而防止不必要的全局重新计算。
