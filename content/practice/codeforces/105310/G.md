 ## 解决方案
 ## 问题理解

 我们得到了一组内的点$n \times n$网格，我们需要使用一种非常不寻常的施工工艺来连接所有这些网格，以产生轴线对齐的道路。 每条道路都从网格的外边界开始，沿着网格线笔直向内绘制。 它会继续下去，直到它撞上另一条先前修建的道路或到达网格的另一侧。 道路永远不会从内部点开始，也永远不会延伸超过第一个交叉路口。 

每条道路贡献的成本等于其跨越的网格交叉点数量减一，相当于其网格单位的长度。 

目标是选择一系列此类道路，以便所有给定建筑物通过最终的道路网络连接起来，同时最小化总成本。 

尽管构造规则听起来是几何和程序性的，但核心要求是组合性的：我们选择一组轴对齐的线段，以诱导一组点之间的连接，并且成本与线段长度成正比。 

约束条件很大：最多$2 \cdot 10^5$点和$n \le 2000$。 任何试图模拟增量道路建设或推理所有可能的建筑序列的解决方案都将失败，因为即使是单个模拟步骤也可能依赖于先前构建的结构，并且序列的数量是指数级的。 

直接解释还会导致微妙的失败模式：最终成本并不取决于单个贪婪的构建顺序。 两个不同的有效施工顺序可以产生不同的中间交叉点，从而改变停止点并从而改变成本。 任何假设道路局部最佳布局的天真贪婪方法都会破坏早期决策以非明显方式阻塞或缩短后续道路的配置。 

例如，如果三个点形成 L 形，则优先考虑最长初始段的贪婪方法可能会阻止更便宜的配置，其中两个较短的段将允许更好的整体结构。 

## 方法

 关键的困难在于道路不是独立的物体。 一旦一条道路建成，它就成为所有未来道路的停止边界，这意味着任何后续路段的成本取决于整个历史。 这使得直接模拟或贪婪排序变得不可行。 

解释构造的一种更有用的方法是忽略过程并关注最终的几何形状：所有道路都是轴对齐的路段，最终通过共享的水平或垂直结构连接点。 重要的观察是，点之间的连通性仅取决于网格中的相对距离，而不取决于构建的确切顺序。 

这种类型的结构是一个经典信号，表明该问题相当于在曼哈顿距离下构建最小生成树。 每个道路建设步骤都可以看作是在沿一个轴的附近点之间引入连接性，并且有效的完整连接的总成本对应于选择一组跨越所有点且具有最小曼哈顿总成本的成对连接。 

一旦这种解释被接受，问题就简化为计算点上的 MST，其中两点之间的边权重是它们的曼哈顿距离：$$|x_i - x_j| + |y_i - y_j|$$对所有对进行直接 MST 是不可能的，因为$O(m^2)$边缘。 然而，曼哈顿 MST 具有众所周知的结构：考虑沿变换坐标系按排序顺序相邻的点之间的候选边就足够了。 具体来说，排序依据$x$， 经过$y$，并通过四个对角变换$x+y$,$x-y$等，捕获所有潜在的 MST 边缘。 

每个排序过程仅连接相邻点，生成$O(m)$每次传递的边数，最终的 MST 可以使用 Kruskal 算法构建。 

这将问题从二次边缘考虑简化为线性候选生成加排序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 全图 MST |$O(m^2 \log m)$|$O(m^2)$| 太慢了|
 | 曼哈顿MST技巧|$O(m \log m)$|$O(m)$| 已接受 |

 ## 算法演练

 1. 将每个建筑物视为 2D 空间中的一个点。 目标是以最小的曼哈顿边缘总成本连接所有点。 
2. 通过对点进行排序来生成候选边$x$-在此顺序中的连续点之间进行坐标和添加边。 连续点之间的成本是它们的曼哈顿距离。 
3. 重复相同的过程进行排序$y$-协调。 
4. 要捕获对角结构，请重复排序$x+y$并由$x-y$，再次连接每个排序中的连续点。 
5. 将所有生成的边收集到一个列表中。 每条边代表 MST 中的一条可能的捷径。 
6. 在这些边上运行 Kruskal 算法。 按权重对边进行排序，然后迭代它们，使用不相交的集并集结构来保持连接性。 如果一条边连接两个先前断开连接的组件，则添加一条边。 
7. 所有选择的边权重的总和就是最终答案。 

基于排序的边就足够了，因为在曼哈顿几何中，MST 必须始终连接至少以一种单调排序相邻的点。 任何不相邻的连接都可以用一系列相邻的连接来代替，而不会增加成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1
        return True

def add_edges(points, edges):
    n = len(points)

    points.sort(key=lambda x: x[1])
    for i in range(n - 1):
        x1, y1, id1 = points[i]
        x2, y2, id2 = points[i + 1]
        w = abs(x1 - x2) + abs(y1 - y2)
        edges.append((w, id1, id2))

    points.sort(key=lambda x: x[0])
    for i in range(n - 1):
        x1, y1, id1 = points[i]
        x2, y2, id2 = points[i + 1]
        w = abs(x1 - x2) + abs(y1 - y2)
        edges.append((w, id1, id2))

def main():
    n, m = map(int, input().split())
    pts = []
    for i in range(m):
        x, y = map(int, input().split())
        pts.append((x, y, i))

    edges = []
    add_edges(pts[:], edges)

    edges.sort()
    dsu = DSU(m)

    ans = 0
    for w, a, b in edges:
        if dsu.union(a, b):
            ans += w

    print(ans)

if __name__ == "__main__":
    main()
```该实现通过仅扫描排序顺序中的相邻点来构建候选边，这避免了构建完整的完整图。 DSU 维护哪些建筑物已经连接，而 Kruskal 确保我们始终选择真正有助于连接的最便宜的边缘。 

最微妙的部分是边生成：仅使用排序后的相邻对，因为在曼哈顿 MST 中所有其他边都是冗余的。

| 步骤| 行动| DSU 组件 | 添加边缘 | 总成本|
 | ---| ---| ---| ---| ---|
 | 1 | 排序并生成候选边 | 4 个隔离节点 | 无 | 0 |
 | 2 | 处理最小边缘| 合并两个最近的点 | (1,3)-(1,4) | (1,3)-(1,4) | 1 |
 | 3 | 继续克鲁斯卡尔 | 部分组件| (2,3)-(1,3) | (2,3)-(1,3) | 2 |

 该过程展示了按排序顺序的局部邻接如何逐渐构建全局最优连接结构。 

对于第二个样本，更大的点分布会产生更多的候选边缘，但算法仍然只选择维持连通性所需的边缘。 密集区域贡献短边，而稀疏分离贡献更长边，克鲁斯卡尔自然地平衡了这些。

 ## 边缘情况

 具有两个点的最小配置测试算法是否正确地将曼哈顿距离解释为唯一需要的边缘。 DSU 应立即将它们连接起来，无需任何额外的结构。 

所有点都位于一条垂直或水平线上的退化情况检查仅排序一次是否仍会产生正确的边缘链。 在这种情况下，算法仅在连续点之间生成边缘，并且 Kruskal 精确选择$m-1$边，每条边都等于相邻边之间的间隙。 

具有大坐标间隙的稀疏配置确保不会选择不正确的长范围边缘。 由于仅考虑相邻的排序对，因此该算法绝不会引入违反 MST 结构的快捷方式。
