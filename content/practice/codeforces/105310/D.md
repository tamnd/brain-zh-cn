 ## 解决方案
 ## 问题理解

 给定两个长度相等的字符串，我们希望使用范围操作将第一个字符串转换为第二个字符串。 单个操作选择一个连续的段，并将两个固定转换之一应用于该段中的每个字符：每个字母都映射到字母表周围的“镜像”（a 变为 z，b 变为 y 等），或者每个字母在字母表中旋转 13 个位置（a 变为 n，b 变为 o 等，环绕）。 

关键是每个操作在一个区间上统一应用，我们希望这样的区间操作的次数最少，才能使整个字符串与目标完全匹配，否则确定无法完成。 

n 最大为 10^6 的约束立即排除了任何尝试模拟所有区间或搜索状态的解决方案。 n 的任何二次方都是不可能的。 即使 O(n log n) 只有在单遍或使用简单数据结构时才是可接受的。 该结构强烈建议我们应该对每个位置进行本地推理，然后跨部门合并决策。 

当 s 中的字符无法使用两个允许的转换的任意组合到达 t 中对应的字符时，就会出现微妙的边缘情况。 例如，如果 s =“a”且 t =“b”，则反转或平移 13 都不会产生单步匹配，并且在任何间隔中应用操作都无济于事，因为每个位置在这两个对合的组合下独立演化。 在这种情况下，这使得答案立即为-1。 

另一个棘手的情况是相邻位置需要不同的变换。 例如，如果一个索引需要反向操作，而下一个索引需要反向操作，则朴素分组可能会根据间隔的合并方式对操作进行过量计数或不足计数。 正确的解决方案必须认识到，我们本质上是在形成一致的“操作需求差异”的片段。 

## 方法

 如果可能的话，每个位置 i 定义将 s[i] 映射到 t[i] 所需的转换。 职位只有三种有意义的状态：不需要更改、应用反向或应用相反。 第一个观察结果是，这两种变换都是对合变换，并且它们的组合结构很小且固定。 如果我们将反向表示为 R，将相反表示为 O，那么在某个段上应用其中任何一个都意味着我们在该间隔上切换状态。 

蛮力会尝试为每个区间决定是应用 R 还是 O，有效地搜索每个区间 2 个以上的选择以及指数级的多种组合。 即使尝试贪婪地选择段也会失败，因为应用操作会更改段中的所有字符，因此本地选择可能会不可预测地传播。 

关键的见解是停止用字母来思考，而是用 s 和 t 之间的差异来思考。 对于每个位置，我们确定它是否需要 R 翻转、O 翻转、两者都需要，或者两者都不需要。 这将问题简化为跟踪这些需求如何随字符串变化。 

需要特定转换的位置可以被视为标签。 由于操作适用于连续的段，因此问题就变成了计算实现目标标记所需的段翻转的最小数量，其中每次翻转都会切换两个独立的二进制层之一。 

这简化为一个经典的观察：答案是在验证一致性之后，所需操作状态发生变化的连续块的数量，分别针对每种操作类型。

 | ---| ---| ---| ---|
 | 间隔蛮力 | 指数| O(n) | 太慢了|
 | 最佳差异扫描| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先将每个字符映射翻译成所需的操作状态。

1. 对于每个索引 i，计算将 s[i] 映射到 t[i] 的变换。 我们测试它是否是同一、反向或相反。 如果没有匹配，我们立即返回-1。 此步骤是必要的，因为操作仅生成一小部分封闭的排列，因此以后无法修复不匹配。 
2. 我们将每个位置表示为一对二进制标志，指示是否需要反向以及是否需要反向。 这是有效的，因为应用反向两次会抵消，并且同样适用于反向。 
3. 我们扫描字符串并将每个标志独立地视为位置上的 0/1 序列。 
4. 对于每个序列，计算相邻索引之间的值发生变化的次数。 每次更改都表示新段的开始，我们必须在其中应用该操作。 这是最佳的，因为单个间隔翻转可以覆盖相同要求的任何最大连续块。 
5. 分别将反向和反向所需的段数相加。 这个总和就是最小移动次数。 

### 为什么它有效

 每个操作对应于翻转一个二进制数组中的连续区间。 任何解决方案都可以分解为需要翻转的最大连续区域。 每个区域内，一次操作就足够了，分割区域只会增加移动次数。 由于反向和相反在单独的二元约束上独立作用，因此总答案是每个约束的最佳分段覆盖之和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def transform_type(a, b):
    x = ord(a) - ord('a')
    y = ord(b) - ord('a')

    if x == y:
        return (0, 0)

    # reverse: x + y == 25
    if x + y == 25:
        return (1, 0)

    # opposite: shift by 13
    if (x + 13) % 26 == y:
        return (0, 1)

    return None

def solve():
    n = int(input())
    s = input().strip()
    t = input().strip()

    rev = []
    opp = []

    for i in range(n):
        res = transform_type(s[i], t[i])
        if res is None:
            print(-1)
            return
        r, o = res
        rev.append(r)
        opp.append(o)

    def count_segments(arr):
        if n == 0:
            return 0
        cnt = arr[0]
        for i in range(1, n):
            if arr[i] != arr[i - 1]:
                cnt += arr[i]
        return cnt

    print(count_segments(rev) + count_segments(opp))

if __name__ == "__main__":
    solve()
```该解决方案首先将每个字符转换减少为一对布尔要求。 辅助函数`transform_type`编码位置是否需要反向或相反，并拒绝不可能的映射。 

然后我们维护两个二进制数组，一个用于相反的要求，一个用于相反的要求。 功能`count_segments`计算出现了多少个连续的 1 组，因为每一组恰好对应一个间隔运算。 

一个微妙的细节是，我们仅在遇到从 0 到 1 的转换时才递增； 我们依赖这样一个事实：开始一个段与新的所需操作间隔完全对应。 我们不计算 1 到 0 的转换，因为它们只是标记结束，而不是新的移动。 

## 工作示例

 ### 示例 1

 输入：

 s =“abcde”，t =“abcde”

 两个数组相同，因此不需要任何操作。 

| 我| s[i] | t[i] | t[i] | 转 | 欧普 |
 | ---| ---| ---| ---| ---|
 | 0 | 一个 | 一个 | 0 | 0 |
 | 1 | 乙| 乙| 0 | 0 |
 | 2 | c | c | 0 | 0 |
 | 3 | d | d | 0 | 0 |
 | 4 | 电子| 电子| 0 | 0 |

 两个数组都是常量零，因此段计数为 0。答案为 0。 

这证实了不需要转换的基本情况。 

### 示例 2

 输入：

 s =“aaaaa”，t =“znmnm”

 我们计算每个字符所需的操作。 

| 我| s[i] | t[i] | t[i] | 转 | 欧普 |
 | ---| ---| ---| ---| ---|
 | 0 | 一个 | z | 1 | 0 |
 | 1 | 一个 | n | 0 | 1 |
 | 2 | 一个 | 米 | 0 | 1 |
 | 3 | 一个 | n | 0 | 1 |
 | 4 | 一个 | 米 | 0 | 1 |

 反向数组：[1,0,0,0,0] 给出 1 个段。 

相反数组：[0,1,1,1,1] 给出 1 个段。 

总答案是2。 

这显示了需求的独立运行如何直接转化为间隔操作。

 ## 边缘情况

 一种边缘情况是所有字符都不需要转换。 该算法生成两个用零填充的数组，因此不会计算任何段起始位置，并且输出为零。 这正确地避免了在开始时错误地计算转换。 

另一种情况是完全统一的转换，例如反转每个字符。 反向数组变成全一，并且段计数器恰好看到一个连续的块。 这证实了全范围操作是最佳的，而不是分成更小的间隔。 

第三种情况是交替需求，例如 101010。这里每次更改都会引入一个新的段开始，因此每个独立的需求都成为其自己的操作。 该算法自然会捕获这一点，因为每次 0 到 1 的转换都会增加计数。
