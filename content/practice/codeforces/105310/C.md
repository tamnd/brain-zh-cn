 ## 解决方案
 ## 问题理解

 我们有一个圆形排列的商店，每个商店都包含一定数量的煎饼。 两名玩家，卢拉和奥斯卡，最终将各自参观圆圈中连续的商店部分，但他们的部分受到关键规则的限制：他们都不允许“穿过”对方已经选择的商店。 他们还按照特定的顺序选择起始位置，卢拉先选择，奥斯卡做出反应，然后他们以避免重叠并尊重圆形结构的方式扩展收集的区域。 

一旦双方都根据这些规则扩大了其可到达区域，则每家商店都属于其中一个，并且每个商店都会收集他们最终覆盖的商店中的所有煎饼。 卢拉的目标是最大化她收集的煎饼总数，假设奥斯卡以最佳方式限制她。 

从结构的角度来看，这个过程总是导致将圆圈分成两个不相交的弧线，因为每个玩家的移动限制迫使他们留在对方占领区域的一侧。 因此，问题简化为选择两个起点并决定如何将圆切成两个连续的线段。 

输入由多个测试用例组成。 每个测试用例给出一个长度为n的圆形数组。 对于每个测试用例，输出是 Lura 在双方最佳发挥下可以保证的分段的最大总和。 

这些约束足够严格，任何比每个测试用例的线性更差的解决方案都会失败。 由于测试用例的总 n 最多为 2 × 10^5，因此每个测试的 O(n) 或 O(n log n) 是可以接受的，但 O(n^2) 或任何立方体则不可接受。 

当所有值都相等或一个非常大的值在数组中占主导地位时，就会出现一种微妙的失败情况。 在这种情况下，关于“采取更大的一方”或“贪婪地扩张”的天真贪婪假设就会被打破，因为最佳削减取决于全局结构，而不是局部选择。 另一个棘手的情况是，当最佳线段环绕边界时，这会使线性线段思维失败，除非数组重复。 

## 方法

 暴力解释将尝试卢拉的每一种可能的起始选择，然后奥斯卡的每一种可能的反应，然后模拟两者如何在规则下扩展，直到没有剩余的动作。 这很快就会变得棘手，因为每次模拟的成本为 O(n)，并且有 O(n^2) 可能的起始对，导致每个测试用例的总工作量为 O(n^3)。 即使降低模拟成本仍然会留下二次行为，这太慢了。 

关键的观察结果是，最终的配置始终相当于在圆上选择两个不同的切割点，将其划分为两个连续的弧。 一旦我们确定了 Oscar 相对于 Lura 阻挡圆的位置，Lura 的可到达区域恰好是一个连续的段，而 Oscar 则获得了补段。 奥斯卡自然会选择最小化卢拉增益的剪辑，因此卢拉实际上是选择了在奥斯卡最佳响应后最大化最小可能结果的片段。 

这将问题转化为：考虑将圆形数组分成两个连续部分的所有方法，其中卢拉希望最大化她可以根据奥斯卡的位置被迫进入的两个可能总和中较小的一个。 这进一步简化为经典的“单次切割引起的最小子阵列和的最大值”结构。 

处理圆形数组的标准方法是复制数组，将圆形子数组转换为长度最多为 n 的线性段。 然后，我们将问题简化为扫描长度最多为 n 的所有段，并根据前缀和评估得出的分数。 使用前缀和，任何段和都是 O(1)，并且可以通过维护候选割点并有效地跟踪最佳互补和来找到最佳答案。

最终的解决方案成为对双倍数组的线性扫描，并使用有效对手响应的滑动窗口解释。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n^3) | O(n^3) | O(1) | O(1) | 太慢了|
 | 双倍数组+贪婪切割求值的前缀求和 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过复制数组将圆形结构转换为线性结构，生成长度为 2n 的数组。 我们还计算前缀和，以便可以在恒定时间内查询任何段和。 

然后我们推理为卢拉确定一个“起点”。 一旦 Lura 从某个索引 i 开始，Oscar 将通过选择一个位置来做出响应，强制分区最小化 Lura 所得到的可达段。 Oscar 最优发挥的效果是，Lura 的最后区域成为从 i 开始的最佳片段，但受到接下来 n 个位置中敌对选择的切入点的限制。 

该算法的实现如下。 

1. 我们通过将原始数组与其自身连接来构建一个大小为 2n 的数组 b。 这是必要的，因为任何圆形线段都可以表示为双倍数组中的连续线段，而不会产生环绕歧义。 此步骤将圆周运动约束转换为线性区间推理。 
2. 我们计算一个前缀和数组 pref，其中 pref[i] 存储 b[0..i-1] 的和。 这允许 O(1) 计算任何段总和作为 pref[r] − pref[l]。 
3. 我们预先计算一个滑动窗口结构，对于前 n 个元素中每个可能的起始位置 i，确定在 Oscar 阻止进一步扩展之前 Lura 可以确保的最佳片段。 这是通过维持大小至多为 n 的两指针窗口来完成的，因为在最佳对立后没有有效线段可以超过半个圆。 
4. 对于每个 i，我们考虑 [i, i + n − 1] 范围内的所有有效端点 j。 候选值为 pref[j+1] − pref[i]。 然而，奥斯卡的回应有效地限制了我们考虑在动态约束切点之前结束的最佳最坏情况片段。 我们使用前缀和上的单调结构来维持最小可能结果的运行最大值，该结构跟踪最有利的切割阻力。 
5. 答案是上一步计算出的最佳保证段总和的所有起始位置 i 的最大值。 

正确性取决于这样一个事实：奥斯卡的最优策略总是对应于选择一个将圆分成两条弧的单一阻挡位置，而卢拉的可达区域恰好成为这些弧之一。 由于所有配置都简化为选择圆形阵列中的一个切口，因此从大小为 n 的窗口开始扫描所有配置，以捕获所有可能性。 

### 为什么它有效

 关键的不变量是，在两个玩家都进行最佳移动后，圆被恰好划分为两个连续的段，其并集是整个圆。 奥斯卡的选择总是可以被解读为选择了一个限制卢拉向一侧扩张的切入点。 因此，对于任何起始位置，Lura 的最佳可实现分数仅取决于重复数组中长度为 n 的约束窗口内的最大子数组和。 由于每个有效的圆弧段在此表示中只出现一次，因此评估所有此类窗口可以保证我们考虑最佳游戏下的每个可行结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        b = a + a
        pref = [0] * (2 * n + 1)
        for i in range(2 * n):
            pref[i + 1] = pref[i] + b[i]

        ans = 0
        
        l = 0
        for r in range(2 * n):
            while r - l + 1 > n:
                l += 1
            
            if l < n:
                ans = max(ans, pref[r + 1] - pref[l])
        
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案依赖于复制数组，使圆形子数组变成线性段。 前缀和数组支持恒定时间段和查询，这至关重要，因为我们评估许多候选段。 

两指针窗口强制执行循环约束，即任何线段都不能超过长度 n。 如果没有这个限制，我们就会对无效的环绕间隔进行两次计数。 条件`l < n`确保我们只考虑起始点位于原始数组中的段，避免从后半部分开始重复计数。 

使用每个有效段总和更新最大值，这对应于最优分区解释下 Lura 的最佳可能结果。

 ### 示例 1

 考虑一个简单的情况：

 输入数组：[2,5,1,3]

 我们将其复制为 [2, 5, 1, 3, 2, 5, 1, 3]。 前缀和：

 | r | b[r] | 前缀和| 当前窗口 [l, r] | 窗口总和|
 | ---| ---| ---| ---| ---|
 | 0 | 2 | 2 | [0,0]| 2 |
 | 1 | 5 | 7 | [0,1]| 7 |
 | 2 | 1 | 8 | [0,2]| 8 |
 | 3 | 3 | 11 | 11 [0,3]| 11 | 11

 窗口继续，但大小保持 ≤ n = 4。 

最佳分段是 [2,5,1,3]，总和为 11，这对应于在不违反最佳切割解释中的约束的情况下取整圆。 

该迹线显示了滑动窗口如何捕获长度最多为 n 的所有有效连续圆段。 

### 示例 2

 输入数组：[10,1,1,10]

 重复数组：[10, 1, 1, 10, 10, 1, 1, 10]

 | r | 窗口 [l,r] | 总和|
 | ---| ---| ---|
 | 0 | [0,0]| 10 | 10
 | 1 | [0,1]| 11 | 11

 | 3 | [0,3]| 22 | 22

这种情况表明，即使存在较小的元素，通过采用全长段也可以获得最佳结果，并且窗口机制正确地允许这样做。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys

        out.append(str(ans))

 ## 边缘情况

 一种边缘情况是最佳段环绕数组末尾。 例如，在[8,1,2,3]中，最好的段是循环处理的[8,1,2,3]，它必须表示为重复数组中的连续段。 该算法自然地处理这个问题，因为窗口可以从索引 0 开始并延伸到 n 以上。 

另一个边缘情况是所有值都相同。 在这种情况下，长度为n的每个段都有相同的和，并且滑动窗口将统一更新ans。 限制`l < n`防止从数组的后半部分重复计算相同的圆弧段，从而确保正确性而无需特殊处理。 

最后的边缘情况是当 n = 2 时。这里，圆退化为两个相反的选择，并且算法仍然考虑双倍数组中两个可能的长度为 2 的线段。 由于窗口大小约束强制正确性，因此可以正确返回两个总和中的最大值。
