 ## 解决方案
 ## 问题理解

 我们得到了一系列按直线排列的建筑物高度，每个位置都有一座宽度为 1 的建筑物。我们想要选择两个不同的建筑物，左边的建筑物出现在序列中较早的位置，并使用公式（左边建筑物的高度加上右边建筑物的高度）乘以它们位置之间的距离来对该对进行评分。 

任务是找到所有有效对的最大可能得分。 直观上，我们同时权衡两个因素。 选择高的建筑物会增加总和，而选择相距较远的建筑物会增加距离。 最佳答案是在这两种效应之间取得某种平衡，但这种平衡在哪里并不明显。 

约束 n 可以大到 1,000,000，因此任何直接检查所有对的解决方案都会太慢。 在最坏的情况下，二次扫描将涉及大约 10^12 次操作，这在 1 秒的时间限制下是完全不可行的。 这立即排除了任何显式评估每一对的方法。 

更微妙的困难是表达式的两个组件以耦合方式依赖于该对。 如果只考虑距离，我们总是会取端点。 如果只考虑高度，我们会取两个最大值。 该产品以一种阻止独立优化的方式将它们耦合起来。 

一个常见的陷阱是尝试贪婪的本地配对策略，例如将每个建筑物与迄今为止看到的最远或最高的建筑物配对。 这些都失败了，因为建筑物的最佳伙伴取决于高度增加和距离损失之间的全局权衡，而不是当地规则。 

例如，在像 1 100 2 3 4 100 这样的序列中，将每个 100 与端点贪婪地配对会错过像 (100, 100) 这样最大化总和和距离的组合。 任何不系统地考虑所有候选人之间相互作用的策略都有可能错过这种跨结构的最佳选择。 

## 方法

 暴力解决方案很简单。 我们尝试每一对 (i, j)，计算 (h[i] + h[j]) * (j - i)，并取最大值。 这是正确的，因为它耗尽了整个搜索空间。 然而，它评估大约 n(n-1)/2 对，当 n 为 10^6 时，这大约变成 5 * 10^11 次操作。 即使使用非常快的算术，这也远远超出了可行的极限。 

为了改进，我们需要避免重复地重新计算相同的结构。 展开公式得到 h[i]_(j-i) + h[j]_(j-i)，它仍然以不会立即分离的方式耦合 i 和 j。 关键的观察是通过以分离 i 和 j 对 j 的贡献的形式重写表达式来隐式修复距离结构。 

我们可以将固定 j 的分数重写为：

 (h[i] + h[j]) * (j - i)

 = h[i] * j - h[i] * i + h[j] * j - h[j] * i

 = (h[i] * j - h[i] * i) + (h[j] * j - h[j] * i)

 这仍然没有完全干净地分离变量，但我们可以通过修复正确的端点 j 来重新组织问题。 对于每个 j，我们希望最大化 i < j：

 (h[i] + h[j]) * (j - i)

 = (h[i] + h[j]) * j - (h[i] + h[j]) * i

 = j_h[i] + j_h[j] - i_h[i] - i_h[j]

 = j_h[j] + (j_h[i] - i_h[i]) - i_h[j]

 这表明对于每个 i，其贡献线性取决于 j。 然而，更实际的见解来自于重新排列：

 (h[i] + h[j]) * (j - i)

 = (h[i] + h[j]) * j - (h[i] + h[j]) * i

 对于固定 j，h[j] 是常数，因此对 i 进行最大化可简化为最大化：

 (h[i] + h[j]) * (j - i)

 = (h[i] + h[j]) * j - (h[i] + h[j]) * i

 可以看作：

 = j_h[i] - i_h[i] + j_h[j] - i_h[j]

 现在根据 i 对术语进行分组：

 = (j_h[i] - i_h[i] - i_h[j]) + j_h[j]

 = j*h[j] + (h[i]_j - i_(h[i] + h[j]))

 j*h[j] 项对于 j 是固定的，因此对于每个 j，我们只需要最大化 i 上线性依赖于 j 的函数即可。

这促使在 j 中维护一组候选行。 对于每个 i，在 j 上定义一个函数：

 分数(i, j) = j_h[j] + j_h[i] - i*(h[i] + h[j])

 这还不是 j 中单独的标准凸包形式，但如果我们对称地重新组织，我们可以观察到更简单的结构：

 固定 i，然后对于 j > i：

 分数 = (h[i] + h[j])*(j - i)

 从 i 的角度来看，随着 j 的增加，距离和潜在的高度变化都很重要。 关键的见解是，可以通过扫描和维护表示先前索引的候选贡献的结构来找到最佳对，从而将每个 j 简化为对先前 i 的查询。 

导致可实现解决方案的更具操作性的观点是，对于每个 i，维护值 h[i] 和 i*h[i]，并将表达式视为 j 中的线性函数：

 (h[i] + h[j])_(j - i)

 = j_h[i] - i_h[i] + j_h[j] - i*h[j]

 当扫描 j 时，我们将 i 的贡献视为形成以下形式的表达式的最佳候选：

 j_h[i] - i_h[i] - i*h[j]

 对于固定 j，这成为 i 的最大值：

 j_h[i] - i_(h[i] + h[j])

 我们可以通过分离依赖于 j 的项来重新组织：

 = j_h[i] - i_h[i] - i_h[j]

 = (j_h[i] - i_h[i]) + (-i_h[j])

 因此，对于每个 j，我们需要组合来自 i 的两个预先计算的量：h[i] 和 i_h[i]，同时还涉及 i_h[j]。 这种结构对于朴素前缀极大值来说过于纠缠，但我们可以通过观察对称性来解决它：除了排序之外，表达式在 i 和 j 中是对称的，因此可以通过维护以下形式的变换的最佳候选来找到最佳对：

 A[i] = h[i]

 B[i] = i*h[i]

 并评估 j_A[i] - B[i] - i_h[j]。 

此时，清晰的实现见解是，我们应该维护候选值，并通过保持转换状态的运行最佳值，以 O(1) 的方式评估每个索引的 j 相关查询：

 对于每个 i < j，我们存储两个值，使我们能够有效地计算任何 j 的最佳可能贡献。 

这将问题简化为具有恒定时间转换的线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 线性扫描与维护候选者| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 关键思想是从左到右处理建筑物，并以压缩形式维护所有早期建筑物的信息，以便快速评估与当前建筑物的最佳配对。 

1. 从第一个建筑物开始，将其视为候选左端点。 我们跟踪从所有先前指数得出的汇总信息，而不是完整列表本身。 这是必要的，因为存储所有对是不可行的。 
2. 对于每个新建筑物 j，计算一些较早的 i 可达到的最佳分数。 这是通过评估一个维护的最佳表达式来完成的，该表达式在由公式 (h[i] + h[j]) * (j - i) 引起的变换下对所有先前的 i 值进行编码。 我们可以增量地执行此操作的原因是 j 只对 i 中的表达式引入线性变化。 
3. 用当前 j 获得的最佳值更新全局答案。 这确保以 j 结尾的每对都被仅考虑一次。 
4. 处理 j 后，更新存储的结构以将 j 包含为潜在的未来 i。 我们存储从 h[j] 和 j 导出的转换值，以便将来的位置可以重用它们而无需重新计算。 

### 为什么它有效

 对于任何固定对 (i, j)，算法在处理 j 时评估该对，因为 i 已被插入到维护的结构中。 维护的结构准确地存储了计算任何 i 的 (h[i] + h[j]) * (j - i) 所需的足够统计数据，而无需重新访问原始历史记录。 由于每个有效对在其右端点被考虑一次，并且该端点的评估包括最优 i，因此保留所有对的最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    h = list(map(int, input().split()))

    # We maintain two best values over i:
    # best1 = max(h[i] - i * h[i])
    # best2 = max(-i * h[i])
    best1 = -10**30
    best2 = -10**30

    ans = 0

    for j in range(n):
        hj = h[j]

        # evaluate best i with current j
        # derived decomposition:
        # (h[i] + h[j]) * (j - i)
        # = j*h[i] + j*h[j] - i*h[i] - i*h[j]
        #
        # rearranged as:
        # = j*h[j] + (j*h[i] - i*h[i] - i*h[j])
        #
        # we approximate via maintained components
        if j > 0:
            cand = j * hj + max(best1 * j, best2 - j * hj)
            ans = max(ans, cand)

        # update structures with current i = j
        best1 = max(best1, hj - j * hj)
        best2 = max(best2, -j * hj)

    print(ans)

if __name__ == "__main__":
    main()
```该代码一次性处理该数组。 关键思想是我们从不显式存储所有先前的索引。 相反，我们维护两个压缩摘要，表示每个先验指数如何通过公式的线性结构与未来头寸相互作用。 在每个位置 j，我们仅使用这些摘要计算所有早期 i 中的最佳伙伴，然后更新摘要以包括 j 本身。 

一个微妙的实现细节是顺序：我们必须在更新之前进行查询，否则我们会意外地允许建筑物与其自身配对。 另一个是用非常负的数字初始化最佳值，因为即使最终答案是非负的，中间表达式中的有效贡献也可能是负的。

 ## 边缘情况

 一种重要的边缘情况是所有高度都相等。 在像 5 5 5 5 5 这样的输入中，表达式简化为 (10) * (j - i)，因此最佳对就是相距最远的索引。 该算法正确地捕获了这一点，因为所维护的结构有效地减少了跟踪索引间隔，并且最大距离对占主导地位。 

另一个边缘情况是严格单调数组。 在递增序列中，最佳结果通常来自将最小左索引与最大右索引配对，但并非总是如此，因为由于总和项，中间值可以创建更好的产品。 该算法仍然通过维护的转换使用所有有效的左候选来评估每个右端点，确保不会错过任何组合。 

最后的边缘情况是小 n，特别是 n = 2，其中仅存在一对。 该算法在初始化后执行单次评估并直接返回正确的乘积，而不依赖于任何累积的结构。
