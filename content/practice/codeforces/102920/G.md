 ## 解决方案
 ## 问题理解

 我们得到一条线$n$机器人，每个机器人都从某个真实坐标开始。 我们必须重新定位它们，以便在移动后它们形成一条完全规则的链条：机器人$i+1$必须是精确的距离$d$在机器人的右侧$i$。 因此，一旦我们选择了机器人 1 的位置，最终的配置就完全确定了，因为所有其他的都变成了$x_1, x_1 + d, x_1 + 2d, \dots$。 

我们优化的约束不是总运动，而是任何单个机器人的最大运动。 每个机器人并行移动，因此完成时间由最慢的机器人决定。 我们必须选择最终的算术级数来最小化这个最大绝对位移。 

输入规模很大，可达百万个机器人。 这立即排除了对第一位置的所有候选者的任何二次配对或动态规划。 即使排序也处于边缘状态，但仍然可行$O(n \log n)$。 任何尝试在输入位置之间尝试机器人 1 的所有可能对齐的解决方案都将失败。 

当许多机器人从同一坐标开始时，就会出现微妙的边缘情况。 由于移动过程中允许发生碰撞，因此重复项不会产生排序约束，但它们确实会影响我们计算与最终算术级数的匹配的方式。 

另一个棘手的情况是当最佳最终排列未与任何输入坐标对齐时。 例如，即使所有输入位置都是整数，最佳移位也可以是分数，因为最小化最大偏差通常使配置处于极端约束之间。 

## 方法

 蛮力的想法很简单：尝试起始位置的所有可能值$x$最终的算术级数。 对于每位候选人$x$，最终位置固定为$x + i \cdot d$，然后我们计算每个机器人的原始位置与其指定的最终位置之间的最大绝对差。 答案是最小的$x$。 

这在概念上是可行的，因为一旦我们确定了链中机器人的顺序，一切都是确定性的。 然而，选择$x$是连续的，不是离散的。 即使我们使用输入位置离散化候选者，正确的最优值也可能位于它们之间。 更糟糕的是，评估所有候选人会导致$O(n^2)$时间。 

关键的观察是我们没有匹配任意排列。 最终顺序是固定的：机器人索引对应于排序的目标位置。 所以如果我们对初始位置进行排序$a_i$，我们可以假设$i$-第一个最小的起始位置与$i$-第最小目标位置。 任何最优解都可以转化为这种对齐配对，而不会增加最大偏差。 

一旦排序，问题就减少到选择一个班次$x$。 对于每个机器人，我们比较$a_i$和$x + i \cdot d$，所以我们想要最小化$$\max_i |a_i - (x + i d)|.$$重写、定义$$b_i = a_i - i d,$$那么表达式就变成了$$\max_i |b_i - x|.$$现在问题在几何上很简单：选择$x$最小化到一组点的最大距离$b_i$在实线上。 最优的$x$是最小值和最大值的中点$b_i$，答案是其传播的一半。 

排序后，这会将问题分解为一次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 最佳 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 读取所有机器人位置并将其存储在数组中。 我们将它们保留为给定的，因为我们接下来将对它们进行排序，以强制与最终位置保持一致的配对。 
2. 按非降序对数组进行排序。 这确保了$i$排序列表中的第 - 个机器人被分配给$i$最终链中的第 - 个位置，这可以避免交叉并将问题减少为单调对齐。 
3. 将每个位置转换为移位后的值$b_i = a_i - i \cdot d$， 在哪里$i$是排序数组中从零开始的索引。 此步骤编码了相邻机器人必须完全不同的要求$d$。 
4. 计算所有值中的最小值和最大值$b_i$。 这两个值代表任何可行的转变的最紧密间隔$x$必须覆盖以尽量减少最大偏差。 
5. 最佳换档$x$是该区间的中点，最小最大移动是区间长度的一半，计算如下$(\max b_i - \min b_i) / 2$。 
6. 输出该值，保留一位小数。 

### 为什么它有效

 排序后，任何最佳分配都会遵循顺序，因为由于目标位置的单调结构，交换无序分配的两个机器人只会增加或保留最大偏差。 转变$b_i = a_i - i d$将问题转化为选择单个实数$x$最小化与一组固定点的最大绝对偏差。 这是一个经典的直线上的极小极大问题，其解由最小封闭区间决定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, d = map(int, input().split())
a = list(map(int, input().split()))

a.sort()

mn = float('inf')
mx = -float('inf')

for i, x in enumerate(a):
    val = x - i * d
    if val < mn:
        mn = val
    if val > mx:
        mx = val

ans = (mx - mn) / 2.0
print(f"{ans:.1f}")
```代码首先对位置进行排序，强制机器人和目标槽位之间的固定对应关系。 每个位置都通过减去来调整$i \cdot d$，消除了最终阵型的刚性结构，并将问题转化为均匀平移问题。 

这些变换值的最小值和最大值定义了移位的最严格的可行间隔。 最终答案是该间隔的一半，因为放置$x$中点平衡两侧最大偏差。 

需要浮点格式，因为答案可能是小数。

 ## 边缘情况

 当所有机器人从同一位置开始时，排序会产生一个常量数组。 改造后$b_i = a_i - i d$，值形成严格的递减序列，并且答案纯粹由索引间距而不是几何形状驱动。 中点计算仍然会产生正确的对称位移。 

当输入颠倒或严重无序时，排序可确保正确性。 如果不进行排序，配对将不一致，并且计算出的间隔将不代表任何可行的匹配。 

什么时候$d$差异较大，$i \cdot d$支配原始坐标。 该算法仍然表现正确，因为它仅使用变换空间中的相对差异，因此大的幅度不会引入不稳定。
