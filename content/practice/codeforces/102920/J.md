 ## 解决方案
 ## 问题理解

 我们给出了一个具有相同数量的开关和灯的系统，以及描述开关如何影响灯的二进制连接矩阵。 当一个开关打开时，它会同时影响多个灯，每个灯的最终状态仅取决于其连接的开关当前打开的数量。 当活动连接开关的数量为奇数时，灯亮起；当该数量为偶数时，灯熄灭。 

任务不是模拟切换。 相反，对于每个灯，我们必须独立地确定是否存在某些开关子集，使得该特定灯打开而其他所有灯关闭。 如果这对于所有灯都是可能的，那么我们必须为每个灯输出一个有效的开关子集来实现这一条件。 如果甚至一盏灯都无法以这种方式隔离，我们输出-1。 

输入矩阵自然地被解释为域 GF(2) 上的线性变换。 每个开关都是 0-1 灯光空间中的一个向量，打开开关相当于将向量模 2 相加。每个灯光对应一个我们想要隔离为 1 的坐标，而所有其他灯光则变为 0。 

约束 N ≤ 500 表明 O(N3) 高斯消去法是可以接受的，但任何涉及枚举开关子集的操作都是不可能的，因为那将是 O(2^N)。 该结构强烈建议在 GF(2) 上求解线性方程组，而不是强力搜索。 

一个微妙的点是，我们不会被问到固定系统中每个灯是否可以独立于其他灯而独立地到达。 方程组必须能够同时求解，从而产生独立输出的完整基础。 这比独立地一一检查可达性要强。 

一个常见的失败案例来自于独立处理每个灯并尝试贪婪地选择开关。 例如，选择仅切换该灯的开关可能会失败，因为它们的组合可能会无意中影响之前固定的灯。 

## 方法

 直接的强力方法将尝试每个开关子集并计算所得的灯光配置。 对于每个子集，我们检查它是否隔离了特定的光。 这是正确的，因为它详尽地探索了所有可能的开关配置，但成本是指数级的：有 2^N 个子集，每次评估成本为 O(N)，导致 O(N·2^N)，这对于 N 高达 500 来说是完全不可行的。 

关键的观察结果是开关行为在 GF(2) 上是线性的。 每个开关对应一个二进制向量，打开开关对应于对这些向量模 2 求和。系统变成大小为 N×N 的矩阵 A，任何开关配置都是向量 x，在 GF(2) 上产生输出 Ax。 问题简化为询问矩阵是否可用于生成所有标准基向量 e1、e2、...、eₙ，如果可以，则为每个基向量构造原像。 

这相当于询问 A 是否在 GF(2) 上可逆。 如果它是可逆的，则对于每个光 i，存在唯一解 x 使得 Ax = eᵢ。 如果 A 不可逆，则至少有一个基向量不可到达，答案为 -1。 

因此，问题简化为对 GF(2) 执行高斯消元法，检查可逆性，然后通过单位矩阵进行增广来隐式计算 A⁻1。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N·2^N) | O(N·2^N) | O(N) | 太慢了|
 | GF(2) 上的高斯消元法 | O(N3) | O(N²) | 已接受 |

 ## 算法演练

 我们将输入矩阵视为 N×N 二进制矩阵 A。我们构建一个增广矩阵 [A | I]，其中 I 是单位矩阵。 然后我们使用 XOR 而不是减法对 GF(2) 执行高斯消除。

1. 对于从左到右的每一列，我们尝试在尚未使用的行中选择该列中值为 1 的数据透视行。 如果不存在这样的行，则矩阵是奇异的，我们立即返回 -1。 这确保了每一列都贡献一个主元，这是可逆性所必需的。 
2. 一旦找到枢轴行，我们将其交换到当前枢轴位置。 这稳定了淘汰顺序并确保我们构建结构化的行梯队形式。 
3. 我们通过将主元行与这些行进行异或来消除当前列中的所有其他 1。 此步骤保持系统的等价性，同时将矩阵推向恒等形式。 
4. 处理完所有列后，如果左半部分变为单位矩阵，则右半部分变为 A⁻1。 右半部分的每一行直接给出激活单个灯所需的开关组。 
5. 对于每个光 i，我们输出逆矩阵第 i 行中 1 的位置。 

其工作原理归结为将行解释为基础转换。 每行操作都保留 GF(2) 上线性系统的解空间。 如果消除成功将 A 转化为恒等式，则意味着 A 具有满秩并定义了从开关配置到光配置的双射。 增强的恒等式跟踪基向量如何用原始开关来表达，因此变换后的恒等式的每一行精确地编码必须打开哪些开关以产生单个活动光。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = [list(map(int, input().split())) for _ in range(n)]

    # augmented matrix [A | I]
    for i in range(n):
        a[i].extend([1 if i == j else 0 for j in range(n)])

    row = 0
    for col in range(n):
        pivot = -1
        for r in range(row, n):
            if a[r][col]:
                pivot = r
                break
        if pivot == -1:
            print(-1)
            return

        a[row], a[pivot] = a[pivot], a[row]

        for r in range(n):
            if r != row and a[r][col]:
                for c in range(2 * n):
                    a[r][c] ^= a[row][c]

        row += 1

    # extract inverse
    for i in range(n):
        res = []
        for j in range(n):
            if a[i][n + j]:
                res.append(j + 1)
        print(*res)

if __name__ == "__main__":
    solve()
```该实现显式构造增广矩阵，将每行宽度加倍以存储 A 和身份。 消除循环从上到下贪婪地选择枢轴，确保每列都被处理一次。 XOR 步骤取代了 GF(2) 中的减法，并且迭代所有行可以保持正确性，因为我们需要简化的行梯形形式，而不仅仅是上三角形式。 

最终提取将右半部分读取为逆矩阵。 每行对应一盏灯所需的开关组。 

一个微妙的实现细节是，我们必须消除上方和下方的主元行以保证干净的单位矩阵。 将消除限制为仅较低的行将产生三角形形式，但不会直接产生相反的形式。

 ## 边缘情况

 一个重要的边缘情况是多个开关影响相同的灯组。 在这种情况下，矩阵的两列是相同的，立即使矩阵成为奇异的。 该算法在主元选择期间检测到这一点，因为每列只能分配一个主元，最终某一列无法找到有效的主元行，从而触发 -1。 

另一种情况是矩阵是可逆的，但消除顺序对于构造干净的逆矩阵很重要。 该算法始终自上而下选择主元，确保行操作的一致性。 由于每次消除都对称地应用于左半部分和右半部分，因此变换保留了等价性，因此只要主元有效，最终的逆元就与主元顺序无关。 

最后一个微妙的情况是矩阵是满秩的，但在稀疏性方面条件较差。 即使这样，GF(2) 消除也能确保正确性，因为运算完全基于 XOR，并且不依赖于数值稳定性。
