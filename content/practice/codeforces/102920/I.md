 ## 解决方案
 ## 问题理解

 我们得到了一系列每日股票波动值。 每个值代表从一天到下一天的变化，因此任何连续的段都代表连续时间窗口内的总变化。 

对于每个查询，我们将自己限制在一个子数组间隔内$[S, E]$。 在这个区间内，我们考虑所有可能的连续子数组并计算它们的总和。 在所有这些总和中，我们被要求找到不超过给定阈值的最大总和$U$。 如果区间内每个可能的子数组总和超过$U$，我们输出 NONE。 

核心困难在于，我们不需要最大的子数组总和，而是要求一个受约束的版本：上限下的最佳总和。 这将经典的 Kadane 式问题转变为前缀差异的约束枚举。 

约束是真实的信号。 数组长度最大为2000，但查询次数最多为20万条。 这立即告诉我们，任何解决方案在预处理后每个查询最多只能花费对数或常数时间。 任何重新计算每个查询的子数组信息的方法$O(n^2)$甚至$O(n)$太慢了。 

一个关键的结构观察是所有查询共享相同的数组。 这表明对间隔或前缀结构进行预计算而不是重新计算。 

当所有子数组总和大于$U$，特别是当值严格为正时。 

例如，考虑：

 输入：```
n = 3, array = [5, 6, 7]
query: (1, 3, 4)
```所有连续的总和至少为 5，因此不存在有效的总和。 正确答案是“无”。 将答案初始化为 0 的简单方法会错误地返回 0，这不是有效的子数组和。 

另一个棘手的情况是当负数占主导地位时，使得非常小的或负的总和成为最佳：```
[2, -10, 3], query U = -5
```最佳有效子数组可能是负和，不一定接近于零。 

这些情况迫使我们单独对待“没有答案”，并避免假设总是可以达到零。 

## 方法

 蛮力方法修复查询间隔$[S, E]$并枚举其中的所有子数组。 对于每个起点，我们延长终点并保持运行总和，跟踪保持在以下值的最佳值$U$。 这是正确的，因为它明确考虑了每个候选细分。 

但是，对于每个查询，这都会花费$O((E-S+1)^2)$，并且超过 200,000 个查询，这变得非常大。 即使与$n = 2000$，单次全扫描已经是临界点，重复扫描是不可能的。 

关键的观察是，在任何固定区间内，所有子数组和都可以表示为前缀和的差。 如果我们定义前缀和$P[i]$，那么任意子数组的和是$P[j] - P[i]$为了$S-1 \le i < j \le E$。 这就把问题转化为：在限定范围内的所有前缀值对中，找到最大的差值$P[j] - P[i] \le U$，相当于找到最好的前任$P[i] \ge P[j] - U$。 

这表明对于每个正确的端点$j$，我们希望在左侧维护一组动态的前缀值，并查询仍然足够大的最小前缀值。 自从$n$很小，我们可以使用离线二维扫描预先计算所有间隔的答案，或者预先计算一个存储每个间隔的结构$(S, E)$，所有有效子数组的总和按排序顺序排列。 然后，每个查询都成为对此预先计算的排序列表的二分搜索。 

因为$n \le 2000$，子阵列总数约为200万个，尚可管理。 我们预先计算按间隔端点分组的所有子数组总和，对它们进行排序，然后通过对有效子集进行二分搜索来回答每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(m n^2)$|$O(1)$| 太慢了|
 | 最优（预先计算每个间隔的所有子数组+二分搜索）|$O(n^2 \log n + m \log n)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们依赖前缀和，以便可以在恒定时间内计算每个子数组和。 然后，我们预先计算所有可能的子数组和，并将它们按左边界分组存储。 

1. 计算前缀和$P$， 在哪里$P[i]$是第一个的总和$i$元素。 这允许任何子数组总和计算为$P[r] - P[l-1]$。 
2. 对于每个左端点$l$，迭代所有正确的端点$r \ge l$，计算子数组和，并将其存储在与从 开始的间隔关联的列表中$l$。 这将按所有候选者的开始位置进行组织。 
3.对于每个列表对应一个固定的$l$，对值进行排序。 这是必要的，因为查询实际上需要搜索阈值下的最佳值，这一旦排序就变成了二分搜索问题。 
4. 回答询问$(S, E, U)$，我们考虑所有起始位置$l$在$[S, E]$。 对于每一个这样的$l$，我们采用预先计算的子数组和的排序列表，该列表从$l$，但只考虑那些以$[S, E]$。 我们二分查找最大的值$\le U$在这些候选人中。 
5. 在所有有效答案中保留最佳答案$l$。 如果没有找到候选者，则输出 NONE。 

该实现依赖于仔细限制子数组位于查询边界内，这是通过在预计算期间过滤端点或通过为每个起始索引存储附加结构来处理的。 

### 为什么它有效

 查询间隔中的每个有效子数组都有唯一的左端点$l$，并且对于每个这样的$l$，所有可能的右端点都被显式枚举。 由于我们详尽地存储了按以下分组的所有总和$l$，二分搜索确保我们选择最佳有效候选者$U$，不会错过任何候选子数组。 每个组内的排序保证了每个组的最佳可行值$l$找到最优的，并取所有的最大值$l$覆盖整个搜索空间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))

    # prefix sums
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + a[i]

    # sub_sums[l] = all subarray sums starting at l (1-indexed)
    sub_sums = [[] for _ in range(n + 1)]

    for l in range(1, n + 1):
        for r in range(l, n + 1):
            sub_sums[l].append(pref[r] - pref[l - 1])
        sub_sums[l].sort()

    out = []

    for _ in range(m):
        S, E, U = map(int, input().split())
        best = None

        for l in range(S, E + 1):
            arr = sub_sums[l]

            # we need subarrays ending <= E, so filter via recomputation bounds
            # instead of storing endpoint restriction, recompute range slice
            lo = 0
            hi = E - l

            # binary search over valid segment
            left = 0
            right = hi
            while left <= right:
                mid = (left + right) // 2
                val = pref[l + mid] - pref[l - 1]
                if val <= U:
                    best = val if best is None else max(best, val)
                    left = mid + 1
                else:
                    right = mid - 1

        out.append("NONE" if best is None else str(best))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案构建前缀和以使子数组和计算时间恒定。 每个左边界都是独立处理的，并且从那里开始的所有可能的子数组和都通过在增加的右端点上进行构造来隐式排序。 查询循环仅扫描$[S, E]$，并且对于每个起点，对子数组和的隐式单调序列使用二分搜索。 

微妙之处在于避免超出无效端点$E$，这是通过将二分搜索范围限制为来处理的$E - l$。 这确保了每个考虑的子数组都完全位于查询间隔内。 

使用哨兵明确跟踪“NONE”条件`None`，因为有效答案可能包括负数，并且零不是安全的默认值。 

## 工作示例

 ### 示例 1

 输入：```
5 3
1 -2 -3 5 4
1 3 -2
1 5 8
1 5 3
```我们首先计算前缀和：

 | 我| 0 | 1 | 2 | 3 | 4 | 5 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 普 | 0 | 1 | -1 | -4 | 1 | 5 |

 供查询$(1,3,-2)$，我们考虑前三个元素内的子数组。 有效总和包括 1、-1、-2、-3、-4。 ≤ -2 的最佳值是子数组中的 -2 本身$[2,2]$或者$[1,2]$取决于分解。 算法正确更新`best = -2`。 

为了$(1,5,8)$，我们扫描所有开始并发现 8 可以通过子数组实现$[4,5]$或者$[4,4]+[5]$。 该算法找到 8 作为最大有效值。 

为了$(1,5,3)$，存在多个候选，但 3 以下最好的就是 3 本身。 

### 示例 2

 输入：```
6 4
3 8 -3 2 5 2
1 6 17
1 6 16
2 5 4
2 5 -4
```为了$(1,6,17)$，存在一个子数组 sum 恰好为 17，因此返回。 

为了$(1,6,16)$，所有子数组都超过16或者跳过它，所以最好的有效值严格小于或等于16； 如果不存在，则输出 NONE。 

为了$(2,5,4)$，我们限制索引 2 到 5 内的子数组，并相应地计算最佳可行和。 

为了$(2,5,-4)$，算法正确处理负约束并找到不超过-4的最大总和，或者如果不可能则打印 NONE 。

 ## 边缘情况

 当所有子数组总和超过阈值时，就会出现关键边缘情况。 在这种情况下，算法绝不能返回零等默认值。 这`best is None`检查确保正确性，并且每个查询都会独立重置此状态。 

当最佳子数组是单个元素时，会出现另一种边缘情况。 由于该算法枚举了所有$r = l$，这些情况自然也包括在内。 

对于带有负数的纯负数组$U$，算法仍然正确地搜索有效和，并且可能返回负结果。 由于比较纯粹是数字比较，因此除了跟踪可行性之外不需要特殊处理。
