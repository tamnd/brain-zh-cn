 ## 解决方案
 ## 问题理解

 我们有两个立方体，每个立方体都有六个面。 每个面都包含一个正整数，对于每个立方体，六个值的总和为 21。滚动立方体时，每个面出现的可能性相同，因此每个立方体在其六个面值上定义了均匀的概率分布。 

任务是计算第一个立方体的随机滚动产生的值严格大于第二个立方体的随机滚动的值的概率。 每个面值结果的可能性相同，因此这变成了对 36 个同样可能的有序面值对的比较，一个从第一个立方体中选择，一个从第二个立方体中选择。 

输出必须是不可约分数。 这意味着我们计算 36 对中有多少对满足“第一个值大于第二个值”的条件，然后除以 36，通过将分子和分母除以它们的最大公约数来简化分数。 

约束很小而且是固定的。 每个输入行恰好包含六个整数，因此任何甚至是 6 的二次方的算法实际上都是恒定时间。 唯一有意义的考虑是概率计算的正确性和仔细处理值中的重复项，因为重复的数字会影响比较中的多重性。 

当许多面具有相同的值时，就会出现一种微妙的情况。 例如，如果两个骰子的所有面都等于 3，则每个结果都相等，因此答案必须是 0/1。 仅比较唯一值或假设排列的简单方法在这里会失败，因为多重性至关重要。 

另一种边缘情况是当值严重倾斜时，例如一个骰子大部分是小值，而其他骰子大部分是大值。 在这种情况下，仅计算不同的比较而不进行频率扩展会低估或高估概率。 

## 方法

 思考问题最直接的方法就是模拟所有可能的滚动。 每个骰子有六个面，因此我们可以迭代所有 36 个有序对。 对于每一对，我们检查第一个值是否大于第二个值，并计算这种情况发生的次数。 这已经非常小了，因为 36 次比较是持续性的工作。 

这种强力方法是正确的，因为骰子是均匀且独立的，因此每对面出现的可能性相同，概率为 1/36。 我们想要的概率正是有利对在所有对中所占的比例。 

除此之外，没有真正需要进行优化，但概念结构概括了：如果骰子有很多面或被加权，我们仍然会依赖于计算成对比较，可能使用频率图而不是原始枚举。 

关键的观察结果是，问题简化为笛卡尔积的离散概率。 一旦认识到这一点，解决方案就变成了一个简单的计数任务，而不是任何组合推理。

 ## 算法演练

1. 读取第一个骰子的六个值和第二个骰子的六个值。 这些代表了他们脸上的均匀概率分布。 
2. 初始化成功结果的计数器，这意味着第一个骰子的面严格大于第二个骰子的面的情况。 
3. 迭代第一个骰子中的每个值。 
4. 对于每个这样的值，迭代第二个骰子中的每个值。 
5. 比较两个值。 如果第一个值较大，则增加成功计数器。 
6. 处理完所有对后，结果总数固定为 36，因为每个骰子有 6 个选择。 
7. 形成分数 success / 36。 
8. 通过将分子和分母除以它们的最大公约数来减少分数。 

### 为什么它有效

 每个骰子上的每个面都有相同的可能性，并且骰子是独立的。 这意味着每对有序的面孔以相同的概率 1/36 出现。 该算法显式地枚举所有此类同样可能的结果，并准确计算有利的子集。 由于没有对被遗漏或加权不正确，因此计算出的比率与真实概率相匹配。 通过 gcd 减少可以保留分数的等价性，而不改变其值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

a = list(map(int, input().split()))
b = list(map(int, input().split()))

win = 0
for x in a:
    for y in b:
        if x > y:
            win += 1

total = 36
g = gcd(win, total)

print(f"{win // g}/{total // g}")
```该代码直接实现了前面描述的pair枚举。 嵌套循环反映了面的笛卡尔积。 由于两个骰子始终具有 6 个值，因此我们可以安全地将结果总数硬编码为 36。 

gcd 约简可确保输出按要求为不可约形式。 不需要浮点运算，这完全避免了精度问题。

## 测试用例```python
import sys, io
from math import gcd

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    win = 0
    for x in a:
        for y in b:
            if x > y:
                win += 1
    g = gcd(win, 36)
    return f"{win//g}/{36//g}"

 ## 边缘情况

 完全统一的情况，例如`1 1 1 1 1 1`与自身对抗产生零获胜对。 该算法仍然迭代所有 36 对，但每次比较都会失败，留下`win = 0`。 减少后，`0/36`正确地简化为`0/1`。 

支配地位的情况，例如`6 6 6 6 6 6`相对`1 1 1 1 1 1`产生`win = 36`。 36 的 gcd 为 36，结果简化为`1/1`，反映确定性。 

混合分布如`1 1 1 1 1 6`反对`6 6 6 6 6 1`确保获胜和失败的结果都会出现。 该算法通过原始枚举正确计算频率加权比较，无需任何符号推理即可产生准确的概率。
