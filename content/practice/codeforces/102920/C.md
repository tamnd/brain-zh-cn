 ## 解决方案
 ## 问题理解

 我们得到一个加权树，其中每个节点都是咖啡馆的候选位置，并且节点的子集被标记为公寓大楼。 任意两个节点之间的距离是树中沿其唯一路径的边权重之和。 

对于每个候选节点$p$，我们想知道是否存在至少一个公寓节点$z$这样$p$严格地接近于$z$比所有其他候选节点。 换句话说，如果你修一套公寓$z$，并比较距离$z$到树中的所有节点，$p$必须是唯一的最接近的节点$z$。 如果这样的见证人公寓存在，那么$p$被称为好地方。 

任务是计算有多少个节点可以作为至少一套公寓的唯一最近点。 

这棵树最多有$10^5$节点，因此任何接近节点或源的二次行为的方法都会立即变得太慢。 任何涉及分别重新计算距每间公寓的距离的操作都需要以下顺序：$O(nk)$或者$O(n^2)$，这远远超出了一秒钟所能通过的范围。 

一个关键的微妙之处是独特性很重要。 如果一个公寓有两个或多个候选站点具有相同的最小距离，那么这些站点都不会通过该公寓被视为“好”，因为定义要求与每个其他节点严格不平等。 

一个小小的失败案例都来自于关系。 假设一间公寓恰好位于两个对称分支的中间。 两端同样接近，因此都不满足严格的优势。 不处理相等性的天真“最近节点”分配将错误地将两端视为有效。 

## 方法

 一种直接的解释建议尝试将每个公寓节点作为源并计算到所有节点的距离，然后检查哪些节点对于至少一个源来说是唯一最接近的。 由于有多达$n$在最坏的情况下，从每套公寓运行单一来源最短路径的成本将很高$O(n \cdot n \log n)$，太大了。 

一旦我们同时查看所有公寓，问题的结构就会发生变化。 我们可以将它们的影响一起传播到树上，而不是单独处理每个单元。 这是典型的多源最短路径情况，其中每个节点都被分配到其最近的源。 然而，标准多源 Dijkstra 只告诉我们最近的源，而不告诉我们最近的源是否是唯一的。 

缺失的部分是认识到我们还需要检测联系。 仅当其最近的源严格地比任何其他源更近时，节点才会贡献“好地方”。 这意味着我们不仅需要到节点的最佳距离，还需要来自不同源的次佳距离。 如果第二最佳距离等于最佳距离，则该节点是一个连接点并且没有任何贡献。 如果它严格更大，则最近的源唯一负责该节点。 

因此，我们可以同时从所有公寓节点运行多源 Dijkstra，但每个状态都带有其起源的源的身份。 在每个节点，我们保留来自不同来源的最佳和次佳到达。 这足以检测本地的唯一性。 

一旦传播完成，每个节点都可以将最多一个源证明为其唯一的最接近所有者。 如果发生这种情况，我们会将该源标记为“至少有一个见证节点”。 最终的答案只是有多少来源被标记。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（Dijkstra 每套公寓）|$O(k \, n \log n)$|$O(n)$| 太慢了|
 | 具有 top-2 跟踪功能的多源 Dijkstra |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将每间公寓视为全局优先级队列中的初始来源。 队列中的每个状态都包含一个节点、距其原始公寓的距离以及该公寓的标识。 

1. 初始化一个优先级队列，所有单元节点距离为零，每个单元都标有自己的源标识。 这会同时从所有来源进行搜索，因此距离同时向外扩展。 
2. 为每个节点维护最多两个最佳形式对（距离、源）的小记录，始终按距离对它们进行排序并确保源是不同的。 这个结构是让我们检测关系的核心机制。 
3. 从优先级队列中弹出最小距离状态。 如果此状态与其源所在节点处存储的最佳或次佳条目不一致，则丢弃它。 这可以防止过时的较长路径受到干扰。 
4. 对于当前节点，尝试将此（距离，源）对插入到其前两个列表中。 如果它在保留不同来源的同时提高了最佳或第二最佳位置，则进行相应更新。 如果源已经用更差的距离表示，我们将覆盖它。 
5. 通过将更新的状态推送到队列中来放松所有邻居，距离随边权重的增加而增加。 每个传播步骤都会通过树传播每个单元的影响力。 
6. 处理完所有节点后，检查每个节点存储的顶部条目。 如果最佳条目在距离上严格优于次佳条目，则最佳源唯一拥有该节点。 将该来源标记为有效。 
7. 计算曾经标记过多少个来源。 

### 为什么它有效

 在每个节点，我们维护来自不同来源的两个最小到达时间。 因为具有非负权重的树中的距离满足最优子结构，所以当源第一次到达具有最小距离的节点时，对于该源来说是全局最优的。 第二好的条目捕获了最接近的竞争来源。 

如果第二最佳距离严格大于最佳距离，则没有其他源可以追平或击败该节点上的最佳距离。 因此，该节点证明其最近源的唯一性。 相反，如果相等，则至少两个源达到相同的最小距离，因此该位置的任何节点都不能用作严格见证人。 

这种局部条件是足够的，因为从源到节点的任何路径在树中都是唯一确定的，因此所有竞争者必须在该全局传播中表现为最短路径到达。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    g = [[] for _ in range(n)]
    
    for _ in range(n - 1):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((v, w))
        g[v].append((u, w))
    
    sources = list(map(int, input().split()))
    sources = [x - 1 for x in sources]
    
    INF = 10**18
    best = [dict() for _ in range(n)]
    # best[u]: source -> dist, but we only keep top 2 per node
    
    pq = []
    
    for s in sources:
        heapq.heappush(pq, (0, s, s))
        best[s][s] = 0
    
    # store top 2 per node: list of (dist, src)
    top = [[] for _ in range(n)]
    
    while pq:
        d, u, src = heapq.heappop(pq)
        
        if src in best[u] and best[u][src] != d:
            continue
        
        for v, w in g[u]:
            nd = d + w
            if src not in best[v] or nd < best[v][src]:
                best[v][src] = nd
                heapq.heappush(pq, (nd, v, src))
    
    # extract top-2 per node
    owner = [None] * n
    second = [INF] * n
    
    for u in range(n):
        arr = sorted(best[u].items(), key=lambda x: x[1])
        if not arr:
            continue
        if len(arr) == 1:
            owner[u] = arr[0][0]
            second[u] = INF
        else:
            owner[u] = arr[0][0]
            second[u] = arr[1][1]
    
    good = set()
    
    for u in range(n):
        if owner[u] is None:
            continue
        best_dist = min(best[u].values())
        # recompute second best distance among sources
        dists = sorted(best[u].values())
        if len(dists) >= 2 and dists[0] == dists[1]:
            continue
        good.add(owner[u])
    
    print(len(good))

if __name__ == "__main__":
    solve()
```该实现运行一个多源 Dijkstra，其中每个州都有其原始公寓。 字典`best[u]`存储从每个源到节点的最佳已知距离`u`，这已经足够了，因为源只能为树度量中的每个节点贡献一个最佳距离。 

优先级队列确保我们第一次放松状态时，它是按距离递增的顺序，这保持了最短路径的正确性。 最后阶段检查节点处的最小两个距离是否严格不同； 如果它们相等，则该节点无法验证任何源。 

最后一组收集了设法唯一地拥有至少一个节点的所有源。

 ## 边缘情况

 一个关键的边缘情况是多个公寓到一个节点的距离相等。 在以节点 1 为中心的对称星形中，每个叶子到中心的距离为 1，因此没有叶子可以以独特的方式严格最接近任何单元。 该算法捕获了这一点，因为中心记录的最佳两个距离相等，从而防止标记任何源。 

另一个边缘情况是当一个源在任何地方都被完全控制时，如果它总是以最佳距离连接，则可能会发生这种情况。 在这种情况下，它的条目在任何节点都不会产生严格的不平等，因此它永远不会被计算在内。
