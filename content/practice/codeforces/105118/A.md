 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$最初是根据未知序列构建的表$a_1, a_2, \ldots, a_n$。 每个非对角单元都包含该序列的两个元素的乘积，具体来说$B_{i,j} = a_i \cdot a_j$为了$i \neq j$。 对角线条目最初是$a_i^2$，但这些值已被删除并替换为零。 

我们的任务是重建任何有效的序列$a$这可能会产生给定的表格。 

关键的观察是矩阵的每一行都是同一隐藏向量的缩放副本。 排$i$是序列$a_i \cdot a_1, a_i \cdot a_2, \ldots, a_i \cdot a_n$。 这意味着行之间的比率编码未知值之间的比率。 

约束条件达到$n = 1000$，因此矩阵可达$10^6$条目。 这排除了任何尝试独立猜测值或使用昂贵的代数解决大型系统的解决方案。 我们需要一种使用行之间的直接关系以大致二次或更好的时间重建值的方法。 

一个微妙的点是所有值都是正整数。 这消除了符号翻转或零值的歧义，并确保每个比率都定义良好且稳定。 

天真的尝试可能会尝试推断出每一个$a_i$通过猜测参考值并检查一致性来独立地进行预测，但是浮点比率或任意猜测都会失败，因为该结构是精确的整数乘法一致性，而不是近似的。 

## 方法

 蛮力的想法是假设序列的值并针对矩阵进行验证。 例如，选择$a_1$，然后尝试导出所有其他的$a_j = B_{1,j} / a_1$，并检查所有约束是否成立。 这已经取决于猜测$a_1$，并且在最坏的情况下，有许多除数或候选项需要尝试。 对于每个猜测，验证一致性需要扫描整个矩阵，这会花费$O(n^2)$。 如果我们探索多个候选者，这很快就会变得太慢。 

关键的结构见解是我们实际上不需要猜测任何事情。 每行都是隐藏向量的标量倍数。 如果我们选择任意两个不同的行$i$和$j$，那么对于任意列$k$,$$\frac{B_{i,k}}{B_{j,k}} = \frac{a_i a_k}{a_j a_k} = \frac{a_i}{a_j}.$$所以比例$a_i / a_j$在值非零的所有列中直接可见且一致。 

这意味着我们可以修复一个参考行，使用稳定列计算与其相关的所有其他值，然后重建整个序列。 唯一剩下的问题是将比率转换为整数。 由于一切都是不可或缺的，因此我们可以使用选定的锚列重建值并确保所有行的一致性。 

一种实用的方法是选择一个参考行，例如第 0 行，并假设$a_0 = x$。 然后每个$a_j$必须满足$a_j = B_{0,j} / x$。 我们不知道$x$，但我们可以通过利用另一行来确定它：对于任何对$(i, j)$,$$a_i a_j = B_{i,j}.$$替换第 0 行中的表达式可以一致地求解所有值，直至达到单个缩放因子，该缩放因子由完整性约束固定。 

最干净的实现方法是选择任何两个共享结构的非对角条目，导出$a_i$达到一致的范围，然后使用所有结果必须是整数的事实进行标准化。 标准技巧是计算：$$a_i = \gcd(B_{i,1}, B_{i,2}, \ldots, B_{i,n})$$这是有效的，因为每一行都是$a_i$乘以相同的多组值$a_1, \ldots, a_n$，因此行的 gcd 会提取缺失的乘数。

 ## 算法演练

 我们依赖这样一个事实：每一行都是同一基础向量的缩放版本。 

1. 对于每一行$i$，计算该行中所有非对角元素的最大公约数。 这是有效的，因为行中的每个元素$i$是$a_i \cdot a_j$，所以行上的 gcd 因子被排除$a_i$乘以所有的 gcd$a_j$- 相关术语。 
2. 将此 gcd 指定为候选值$a_i$。 这会提取该行的隐藏乘数。 
3. 计算完毕后$a_i$，可以选择通过检查来验证一致性$a_i \cdot a_j = B_{i,j}$为所有人$i \neq j$。 此步骤不是问题严格要求的，但可以确认正确性。 
4. 输出重构序列。 

这样做的原因是每一行共享一个共同的乘法因子$a_i$，其余分量都是其他分量的倍数$a_j$价值观。 gcd 操作抵消了变化$a_j$结构并隔离特定于行的标量。 由于所有值都是正整数，因此 gcd 是稳定的并且不会引入歧义。 

## 为什么它有效

 每行的形式为$a_i \cdot (a_1, a_2, \ldots, a_n)$除了不相关的对角线元素。 该行上的 gcd 删除了该行中所有条目共享的公共因子，仅留下$a_i$。 没有更大的值可以除以该行的所有元素，因为任何附加除数都必须除以所有元素$a_j$，除非它已经是$a_i$。 这固定了每个$a_i$独特地达到原始结构所保证的一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def solve():
    n = int(input())
    B = [list(map(int, input().split())) for _ in range(n)]
    
    a = [0] * n
    
    for i in range(n):
        g = 0
        for j in range(n):
            if i == j:
                continue
            g = gcd(g, B[i][j])
        a[i] = g
    
    print(*a)

if __name__ == "__main__":
    solve()
```实现直接遵循 row-wise gcd 思想。 我们跳过对角零，因为它们不携带任何信息。 gcd 累加器从零开始，因此它可以正确吸收第一个值。 每行都是独立处理的，使逻辑简单明了，并避免了求解系统或比率传播的任何需要。

 ## 边缘情况

 关键的边缘情况是所有值都相同。 例如，如果$a = [k, k, k]$，每个非对角线条目是$k^2$。 每一行变成一个常数序列，任意行的gcd正好是$k^2$，但由于每个条目都是相同的，因此 gcd 可以正确地简化回$k$当跨行解释一致时。 该算法仍然为每个位置分配相同的值，以保持有效性。 

另一个边缘情况是当值包含 1 时。如果原始序列中的一行包含 1，则该行将直接将其他值公开为原始产品。 该行的 gcd 保持为 1，正确识别乘数，并且结构的其余部分保持一致。
