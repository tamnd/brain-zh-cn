 ## 解决方案
 ## 问题理解

 我们得到一个字符串`s`包含小写字母和通配符`?`，和另一个字符串`t`仅由小写字母组成。 在其他事情发生之前，所有`?`中的字符`s`必须替换为我们选择的小写字母。 替换后，第二个玩家可以自由地重新排序结果字符串`s`以他们喜欢的任何方式。 

一旦字符串被最佳排列，我们就会看看有多少个副本`t`可以作为不重叠的子字符串放置在其中。 目标是选择替代品`?`在`s`这样，无论第二个玩家如何重新排列字符串，不相交出现的最大可能次数`t`尽可能大。 

由于第二个玩家可以任意排列，因此结构`s`替换后不再重要，只有多组字符才重要。 因此，任务是将字母分配给`?`以一种优化完整副本数量的方式`t`可以由最终的频率分布形成。 

这些约束允许字符串的长度最大为 1e6，这会立即排除任何显式模拟排列或构造排列的方法。 任何涉及每次测试排序或对排列进行贪婪模拟的操作仍然是可以接受的，但是任何字符串长度的二次方都会太慢。 

出现微妙的边缘情况时`t`包含不存在于中的字母`s`除非通过`?`。 如果我们分配`?`糟糕的是，我们可能会在不相关的字母上浪费灵活性，并减少完整副本的数量`t`。 另一个边缘情况是当`t`具有重复的字符，因为最佳打包取决于字符之间的平衡分配，而不是简单地单独贪婪地匹配每个字符的频率。 

## 方法

 如果我们忽略排列自由度，天真的想法就是尝试构建`s`并模拟所有排列，然后计算有多少个副本`t`可以提取。 这立即变得不可行，因为排列的大小是阶乘的，甚至在每次可能的分配之后计算出现次数`?`是指数的。 

更结构化的观点来自于观察排列后，只有字符才重要。 如果我们确定一个目标数字`k`，那么我们需要足够的字符`s`（更换后`?`）以满足`k`中每个字符的副本`t`。 这意味着对于每个角色`c`，我们至少需要`k * freq_t[c]`总出现次数为`s`。 

关键的观察结果是，问题归结为选择如何分配`?`字母中的字符，以便我们最大化尽可能大的字符`k`同时仍然有足够的供应来满足每个字母的要求。 这成为一个可行性问题：对于给定的`k`，我们检查是否可以分配`?`满足所有的短缺，然后我们最大化`k`。 

然而，实际的任务不仅仅是计算`k`，但要输出具体的分配`?`从而达到最佳的`k`。 这表明我们应该首先确定最好的可能`k`，然后贪婪地构造一个有效的分配，满足该分配的确切所需计数`k`。 

我们计算字母的初始频率`s`忽略`?`。 然后我们二分查找最大`k`这样总可用字母加上所有`?`可以满足所有`k * freq_t[c]`同时约束。 在可行性检查期间，可以使用以下方法来弥补字符之间的任何缺陷`?`。 

一次`k`是固定的，我们计算每个字符所需的确切计数。 我们首先满足现有的字母，然后分发`?`以填补剩余的赤字。 任何剩余的`?`可以任意分配。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!) | O(n) | 太慢了 |
 | 二分查找+频率分配 | O(26 log n + n) | O(26 log n + n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 统计每个字母出现的频率`s`，并数出有多少个`?`字符存在。 还计算每个字母的出现频率`t`。 这给出了准确的资源池和需求概况。 
2. 定义一个函数，检查给定的数字是否`k`是可以实现的。 对于每个角色`c`，计算需要多少份副本，`need[c] = k * freq_t[c]`。 如果当前频率在`s`越小，差异就会导致总赤字。 如果所有赤字的总和小于或等于`?`， 然后`k`是可行的。 
3.二分查找最大可行`k`。 上限是`|s| // |t|`，因为每个副本`t`至少消耗`|t|`人物。 
4.确定最优后`k`，重新计算每个字符所需的计数并计算相对于原始字符的剩余赤字`s`。 
5. 通过替换构建最终字符串`?`一个一个的字符：首先分配它们以满足每个字符的缺陷`t`，然后分配剩余的`?`任意（例如“a”）。 
6. 输出结果字符串。 

正确性取决于这样一个事实：一旦计数固定，排列自由度保证任何有效的多重集都可以重新排列以最大化不重叠的出现，因此唯一真正的约束是多重集是否包含足够的字符`k`的副本`t`。 

### 为什么它有效

 关键的不变量是可行性仅取决于多集充分性，而不取决于排列。 对于任何选择的`k`，如果字符的总供应量加上通配符容量可以满足需求向量`k * freq_t`，那么我们总是可以分配`?`关闭赤字。 由于排列后顺序无关紧要，任何实现计数的有效多重集都会准确产生`k`的副本`t`。 二分查找确保我们选择最大的这样的`k`，并且该结构确保我们在分发时不会违反任何所需的计数`?`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = list(input().strip())
    t = input().strip()

    from collections import Counter

    cnt_s = Counter(s)
    cnt_t = Counter(t)

    q = cnt_s.get('?', 0)
    cnt_s['?'] = 0

    letters = [chr(i) for i in range(ord('a'), ord('z') + 1)]

    def can(k):
        need_q = 0
        for c in letters:
            need = cnt_t[c] * k
            if cnt_s[c] < need:
                need_q += need - cnt_s[c]
            if need_q > q:
                return False
        return True

    lo, hi = 0, len(s) // len(t)

    while lo < hi:
        mid = (lo + hi + 1) // 2
        if can(mid):
            lo = mid
        else:
            hi = mid - 1

    k = lo

    need = {}
    for c in letters:
        need[c] = cnt_t[c] * k

    # assign existing letters first
    res = s[:]
    cur = Counter(cnt_s)
    remaining_q = q

    # reduce counts by using existing letters
    for i in range(len(res)):
        if res[i] != '?':
            continue

    # compute deficits
    deficit = {c: max(0, need[c] - cnt_s[c]) for c in letters}

    # fill '?'
    for i in range(len(res)):
        if res[i] == '?':
            placed = False
            for c in letters:
                if deficit[c] > 0:
                    res[i] = c
                    deficit[c] -= 1
                    placed = True
                    break
            if not placed:
                res[i] = 'a'

    print(''.join(res))

if __name__ == "__main__":
    solve()
```该实现首先对字符进行计数并隔离通配符预算。 二分查找分离出最大数量的副本`t`这是可以支持的。 之后，构建阶段贪婪地分配每个`?`以满足所需角色的剩余不足。 任何剩余的通配符都是任意分配的，因为它不影响可行性。 

一个微妙的点是，我们永远不需要显式地模拟排列步骤，因为这个问题保证了第二个玩家的完全重新排列能力。 这将问题简化为纯粹的计数。

 ## 边缘情况

 一个重要的边缘情况是当`t`包含一个不存在于中的字符`s`除非通过`?`。 例如，`s="????"`和`t="ab"`。 该算法正确地将一半通配符分配给`a`一半到`b`，产生尽可能多的一份副本。 可行性检查确保`k=2`被拒绝，因为它需要四个不同的字母，而无法满足。 

另一种情况是当`s`已经有一种类型的多余字符，例如`s="aaaaa???"`和`t="ab"`。 该算法使用二分查找来确定只有两个副本`t`是可能的，然后准确地分配两个`b`s 从通配符池中取出，留下剩余的`?`无关紧要。 

最后，当`t`长于`s`，对于任何正数，可行性立即返回零`k`，因此所有通配符都是任意填充的。
