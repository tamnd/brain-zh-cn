 ## 解决方案
 ## 问题理解

 我们正在使用一个非常小的方形网格，最多八乘八，其中每个单元格要么包含一条鱼，要么是空的。 在这个网格旁边有一个特殊的标记，即海胆，它只占据一个单元格，并且每天都会移动到共享边缘的相邻单元格。 

鱼以离散的步骤进化。 每天分为两个阶段。 首先，海胆移动到四个相邻单元格之一。 然后，使用细胞自动机规则同时更新整个网格，该规则本质上是康威的生命游戏，并强制覆盖海胆的位置。 不管其他什么，任何含有海胆的细胞在下一代都会变空。 每个其他细胞根据其周围八个邻居中当前包含鱼的数量进行更新：生存和出生都取决于该计数是恰好是二还是三，当计数恰好是三时发生出生。 

目标是确定从鱼的初始配置和海胆给定的起始位置开始，在尊重运动约束和确定性进化规则的情况下，是否有可能在最多 d 天内达到鱼的目标配置。 

约束非常严格：网格最多有 64 个单元，步数最多为 8。这立即排除了任何尝试将网格视为大网格或在一般输入大小中运行多项式或指数算法的方法。 然而，时间跨度短是关键的结构性优势。 任何有效的过程最多都会经历八次转变，而海胆每一步最多有四种选择，这非常严格地限制了可能轨迹的分支因子。 

一些微妙的边缘情况很重要。 首先，海胆在移动后会覆盖鱼的状态，因此即使生命游戏规则会在那里创建一条鱼，它也总是会被删除。 在删除海胆位置处的鱼之前应用自动机的粗心实施将产生不正确的过渡。 

其次，这些规则很容易被误读，因为一个条款重复了另一条款的一部分。 正确的解释是标准生命规则，即在三个邻居中出生并在两个或三个邻居中生存，统一应用于所有非海胆细胞。 

第三，海胆在进化步骤之前移动，这对于时间对齐很重要。 如果先模拟进化，然后再模拟运动，则得到的状态空间会根本不同，并导致不正确的可达性。 

## 方法

 蛮力的想法是将每个时刻视为由鱼的配置和海胆位置组成的完整状态，然后探索直到深度 d 的所有可能的移动序列。 在每个状态中，海胆最多有四个有效的下一个位置，一旦选择了下一个位置，鱼网格就会确定性地演化。 这自然形成了深度最多为 d、分支因子最多为 4 的搜索树。 

关键的困难是有效地表示和更新鱼网格。 对于 64 个单元，我们可以将网格编码为 64 位掩码。 对于每个状态转换，我们需要计算所有细胞的邻居计数并应用生命游戏规则，同时在更新后强制海胆细胞为零。 

如果没有优化，计算邻居计数每个状态的成本为 O(n²)，并且最多可以有 4ᵈ 状态，最多 65536。这已经很合适了，但我们仍然需要一个干净的表示来避免常数因子爆炸。 

关键的观察是网格尺寸是固定的并且很小，因此位掩码模拟是理想的。 每个单元格的邻居可以预先计算为位掩码。 然后，对邻居进行计数减少为按位交集，并且每个单元最多对八个预先计算的掩码进行人口计数。

这将每次转换减少到 O(64) 或更好，使得状态图上的完整 BFS 在限制内可行。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解 DFS 状态 | O(4^d · n²) | O(4^d · n²) | O(4^d) | O(4^d) | 常数太慢，有风险 |
 | 位掩码 BFS 结束（网格、海胆位置）| O(4^d·64) | O(4^d·64) | O(4^d) | O(4^d) | 已接受 |

 ## 算法演练

 我们将每个配置视为由鱼位掩码和海胆位置组成的一对。 时间按天离散化，我们将所有可达状态逐级扩展至深度 d。 

1. 将初始网格编码为 64 位整数，其中每一位代表是否有鱼。 还将海胆位置存储为从 0 到 n² − 1 的单个整数索引。 
2. 为每个单元预先计算代表其八个邻居的位掩码。 这允许使用按位运算快速计算邻近鱼的数量，而不是重复扫描网格。 
3. 从初始状态开始进行广度优先搜索。 每个 BFS 层对应一天。 在每一层，我们处理所有当前可到达的状态。 
4. 对于一个状态，考虑海胆向共享边缘的相邻单元格的所有有效移动。 每一步都定义了强制空单元格在下一代中的位置。 
5. 对于每个候选移动，计算下一个鱼网格。 对于每个单元格，使用预先计算的邻居掩码确定当前有多少个邻居包含鱼。 然后应用更新规则：如果一个单元恰好有三个邻居，或者如果它当前处于活动状态并且有两个或三个邻居，则该单元变为活动状态。 计算完整的下一个网格后，将海胆的新位置强制为零。 
6. 如果在下一个 BFS 层中没有看到结果状态，则将其插入到该层中。 如果在任何时候鱼网格与目标配置匹配，则返回成功。 
7. 继续直至达到深度 d。 如果没有状态与目标网格匹配，则返回失败。 

正确性背后的关键思想是每个有效的演化序列都精确对应于该状态图中的一条路径。 BFS 探索了海胆移动到长度 d 的所有可能序列，并且对于每次移动，鱼的进化都是唯一确定的。 由于自动机的模拟在给定当前网格和所选海胆位置的情况下是确定性的，因此不会错过有效的转换，也不会引入无效的转换。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d = map(int, input().split())
    r, c = map(int, input().split())
    
    def read_grid():
        grid = 0
        for i in range(n):
            line = input().strip()
            for j, ch in enumerate(line):
                if ch == '1':
                    grid |= 1 << (i * n + j)
        return grid

    start = read_grid()
    target = read_grid()

    # precompute neighbor masks
    neigh = [0] * (n * n)
    for i in range(n):
        for j in range(n):
            idx = i * n + j
            mask = 0
            for di in (-1, 0, 1):
                for dj in (-1, 0, 1):
                    if di == 0 and dj == 0:
                        continue
                    ni, nj = i + di, j + dj
                    if 0 <= ni < n and 0 <= nj < n:
                        mask |= 1 << (ni * n + nj)
            neigh[idx] = mask

    moves = []
    for i in range(n):
        for j in range(n):
            idx = i * n + j
            nxt = []
            if i > 0: nxt.append((i - 1) * n + j)
            if i + 1 < n: nxt.append((i + 1) * n + j)
            if j > 0: nxt.append(i * n + (j - 1))
            if j + 1 < n: nxt.append(i * n + (j + 1))
            moves.append(nxt)

    from collections import deque

    start_pos = r * n + c
    q = deque()
    q.append((start, start_pos))
    visited = set()
    visited.add((start, start_pos))

    if start == target:
        print("Yes")
        return

    for _ in range(d):
        nq = deque()
        visited.clear()
        while q:
            grid, pos = q.popleft()

            for np in moves[pos]:
                # compute next grid
                new_grid = 0
                for i in range(n * n):
                    cnt = (grid & neigh[i]).bit_count()
                    alive = (grid >> i) & 1

                    if i == np:
                        continue  # forced empty

                    if cnt == 3 or (alive and cnt == 2):
                        new_grid |= 1 << i

                state = (new_grid, np)
                if state not in visited:
                    visited.add(state)
                    nq.append(state)

        q = nq
        if any(grid == target for grid, _ in q):
            print("Yes")
            return

    print("No")

if __name__ == "__main__":
    solve()
```实现直接遵循分层BFS结构。 网格被压缩为位掩码，以便状态复制很便宜。 邻居计数是使用预先计算的位掩码计算的，并且生命转换规则逐个单元地应用。 海胆的作用是通过明确阻止该细胞在下一代中存活来强化的。 BFS 逐层重启，以便我们只探索在允许的天数内可达的状态。 

一个微妙的点是我们清除每个深度的访问集。 这是有意为之的，因为由于移动和网格更新之间的相互作用，在不同时间达到相同的配置可能会导致不同的未来演化路径。 将所有时间折叠成单个访问集会错误地修剪有效轨迹。

 ## 边缘情况

 一种边缘情况是海胆在一个单元上启动，否则该单元会因邻居配置而变得活跃。 强制覆盖可确保即使生命游戏规则在该位置产生 1，最终状态仍为 0。 该算法在计算下一个网格时明确跳过设置该位，因此保证了覆盖。 

另一种情况是，即使没有移动，网格也会在状态之间振荡。 由于 BFS 探索深度为 d 的所有路径，因此振荡自然会被捕获为跨不同层的重复配置，并且算法不会由于每层访问而过早地丢弃它们。 

最后一个微妙的情况是，多个运动序列导致相同的网格配置但不同的海胆位置。 这些被视为不同的状态，这是必要的，因为未来的进化取决于海胆的位置。 状态表示包括这两个组件，因此这些分支保持独立并正确探索。
