 ## 解决方案
 ## 问题理解

 我们被要求分发总共`k`鱼入`n`鱼缸，每个鱼缸必须包含正整数数量的鱼。 修复此分布后，我们查看水箱的每个非空子集，并提出一个有关它的结构问题：该子集本身是否可以分为两个不相交的组，其总鱼数相等。 

如果存在这样的分割，则子集被认为是“坏的”。 要求极其严格：非空罐子集不允许有坏的。 换句话说，每个子集本质上必定是“不平衡的”，因为不可能将其元素分为具有相同总和的两部分。 

输出要么是鱼数量的有效分配，要么是不存在此类分配的声明。 

约束很小，两者都有`n`和`k`最多 200，这表明我们没有处理子集上的大量动态规划或值上的任何指数搜索。 相反，条件的结构是主要的困难。 

一个微妙的边缘情况是全套本身。 如果我们选择一种适用于较小子集但允许将整个数组划分为相等和的两半的排列，则已经违反了条件。 例如，像这样的对称配置`[1, 1, 2]`失败，因为全套可以分为`{1,1}`和`{2}`。 

另一个重要的边缘情况是，大小为 1 的子集始终自动有效，因为单个正整数不能分为两个非空组。 任何正确的构造只需要担心大小至少为 2 的子集。 

## 方法

 暴力解释是尝试每一个可能的分配`n`正整数总和为`k`，并为每个作业枚举所有`2^n - 1`子集，然后对于每个子集尝试将所有分区分成两组，或者等效地检查其中的子集和相等。 即使对于中等程度的人来说，这也很快变得不可行`n`，因为整数组合的配置数量已经非常巨大，并且每个配置都需要对子集进行指数检查。 

关键的结构简化来自于条件的重写。 对于固定子集`S`，说它可以分成两个等和组就相当于说存在一个子子集`T ⊂ S`这样`sum(T) = sum(S) / 2`。 因此，我们实际上并不是直接处理分区，而是处理每个子集中的子集和结构。 

这立即表明我们应该设计数字，以便子集和以非常受控的方式运行。 控制子集总和的最强已知方法是使用超级递增序列，其中每个元素严格大于所有先前元素的总和。 

如果我们设法确保`a[i] > a[1] + ... + a[i-1]`，然后一些强大的事情发生了。 取任意子集`S`看看它最大的元素`x`。 该元素严格大于整个前缀中所有其他元素的总和，因此也大于子集中所有其他元素的总和。 这可以防止内部出现任何等和分区`S`，因为无论哪一边包含`x`立刻就占据了对方的优势。 

这将整个问题简化为构造一个长度递增的序列`n`这正好等于`k`。 

然而，超级递增序列在其最小形式下至少呈指数增长：最小的此类序列是`1, 2, 4, ..., 2^{n-1}`，其总和为`2^n - 1`。 这立即限制了可行性：因为`k ≤ 200`，我们最多只能支持`n = 7`。 

一旦可行性确定，施工就变得简单：采取第一个`n-1`2 的幂并分配最后一个值来吸收剩余的总和。

 ## 算法演练

 1. 检查是否`n`太大，无法支持总和限制内的任何超级递增序列。 由于长度的最小总和`n`是`2^n - 1`， 如果`n ≥ 8`， 输出`No`。 这个界限直接来自于任何有效结构的指数增长。 
2. 计算第一个`n-1`值为 2 的幂：`a[0] = 1`,`a[1] = 2`，依此类推，直到`2^{n-2}`。 这保证了前缀已经超级增加，并严格控制子集总和。 
3. 首先计算这些的总和`n-1`价值观。 叫它`S`。 
4. 将最后一个元素设置为`k - S`。 这迫使总和完全符合要求。 
5. 输出构造好的数组。 

唯一不明显的部分是为什么像这样分配最后一个值不会破坏属性。 原因是只要构造有效，前缀和就已经严格小于最后一个值，从而在整个数组中保持递增条件。 

### 为什么它有效

 对于任意子集，令`x`为其最大元素。 在超增序列中，`x`严格大于整个前缀中所有其他元素的总和，因此也大于该子集中所有其他元素的总和。 如果我们假设将子集划分为总和相等的两组，则无论哪一组包含`x`总和严格大于其他组，这是不可能的。 所以没有子集允许相等的划分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    k, n = map(int, input().split())

    # minimal sum for superincreasing sequence of length n is 2^n - 1
    if n >= 8:
        print("No")
        return

    # build powers of two
    a = []
    for i in range(n - 1):
        a.append(1 << i)

    s = sum(a)
    last = k - s

    # must keep positivity and superincreasing property
    if last <= s:
        print("No")
        return

    a.append(last)

    if sum(a) != k:
        print("No")
        return

    print("Yes")
    print(*a)

if __name__ == "__main__":
    solve()
```该构造首先固定一个严格的前缀，以保证对子集总和的控制，然后使用最终元素作为平衡项来达到精确的总和`k`。 围绕可行性的检查确保我们不会意外破坏超增结构。 

一个微妙的实现细节是`last <= s`查看。 如果没有它，最后一个元素可能会违反正确性论证的最后一步所需的优势属性。 尽管理论上的界限`n ≥ 8`已经消除了大多数不可能的情况，该防护装置可以确保建筑安全达到临界值。

 输入：`k = 9, n = 3`我们为前两个元素构造二的幂：

关键属性是每个子集都有一个主导元素，阻止平等划分。 

### 示例 2

 输入：`k = 6, n = 3`我们尝试同样的想法：

 | 步骤| 数组| 前缀和 | 最后计算 |
 | ---| ---| ---| ---|
 | 构建前缀 | [1, 2] | 3 | |

 我们得到`[1, 2, 3]`，但这失败了，因为`3`并不严格大于前缀和`3`，打破超增结构。 因此，该构造正确地拒绝了该实例。 

这说明了为什么不平等检查至关重要。

 ## 边缘情况

 对于`n = 1`，该算法立即输出一个等于`k`。 由于不存在大小大于 1 的非空子集，因此不存在要违反的分区条件，因此任何正值都是有效的。 

为了`n ≥ 8`，构造是不可能的，因为即使是最小的超增序列也已经超过了允许的总和界限。 该算法预先拒绝这些情况，这与任何有效结构的指数下界一致。 

对于边界情况，其中`k`恰好是最小总和`2^n - 1`，该构造产生规范的二次幂序列。 每个子集仍然是安全的，因为超增属性在每个级别都严格成立，确保没有子集可以被均匀分割。
