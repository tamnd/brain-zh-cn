 ## 解决方案
 ## 问题理解

 我们有两个隐藏的排列，都包含从 1 到 n 的数字恰好一次。 一种排列是另一种排列的循环旋转，但我们不知道它们中的任何一个，也不知道旋转量 k。 形式上，存在一个未知的移位 k，使得每个位置 i 满足 a[i] = b[(i + k) mod n]。 

获取信息的唯一方法是通过查询操作，该操作采用两个索引 x 和 y 并返回 max(a[x], b[y])。 每个查询都会给我们一个整数，我们必须使用最多 2n 个这样的查询来推导出移位 k。 

约束 n ≤ 200 足够小，我们可以承受线性数量的查询，并且仍然显式地推理每个位置。 理论上，尝试比较所有对的任何解决方案都很好，但会超出查询预算，因为这将是 O(n²)。 这立即表明必须利用排列的结构和最大值的特殊作用。 

当 k = 0 时，会出现微妙的边缘情况。在这种情况下，两个数组都是相同的，因此任何位置 i 都满足 a[i] = b[i]，并且我们应该正确输出 0。另一个重要的边缘情况是当最大值 n 由于零移位而出现在两个排列中的相同索引处时。 在这种情况下，关于“两个不同的最大值位置”的天真推理就会失效，必须谨慎处理。 

## 方法

 蛮力的想法是完全重建这两种排列。 如果我们可以确定每个 a[i] 和 b[i]，那么我们可以通过查找序列对齐的位置来简单地计算移位。 然而，每个查询只给出 max(a[x], b[y])，它不会直接显示任何一个值，除非我们已经知道其中一个值足够大以占主导地位。 尝试恢复所有值将需要 O(n²) 推理和可能的许多查询，这大大超出了 2n 限制。 

关键的观察是我们不需要完整的排列。 我们只需要以受控方式识别两个结构中都存在的单个锚元素。 最大值 n 是理想的，因为它的行为就像哨兵一样。 如果查询返回 n，我们就知道 a[x] = n 或 b[y] = n。 由于 n 在每个排列中只出现一次，因此它允许我们仅使用 O(n) 查询来定位两个数组中 n 的位置。 

一旦我们确定了 n 出现在 a 中的索引以及 n 出现在 b 中的索引，旋转就变成了这些位置之间的直接算术差异。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力重建| O(n²) 查询 | O(n) | 太慢了 |
 | 最大价值本地化| O(n) 次查询 | O(1) 额外 | 已接受 |

 ## 算法演练

1. 对于从 0 到 n − 1 的每个索引 i，查询 (i, i) 对并存储结果。 目标是检测两个数组中至少一个的值为 n 的位置。 由于 n 是唯一可以作为最大结果出现的值，因此任何返回 n 的查询都必须触及 a 或 b 中 n 的位置。 
2. 收集所有索引 i，使得 query(i, i) 返回 n。 该集合将包含一个或两个索引。 如果它只包含一个索引，则两个排列的 n 位于同一位置，这意味着旋转为 0，我们可以立即输出 k = 0。 
3. 如果有两个索引 i 和 j，我们知道其中一个是 n 在 a 中的位置，另一个是 n 在 b 中的位置，但我们还不知道哪个是哪个。 
4. 解析方向，执行一次交叉查询query(i,j)。 如果结果是n，那么a[i]一定是n，因为否则a[i]和b[j]都不能产生n。 这意味着 i 是 n 在 a 中的位置，j 是 n 在 b 中的位置。 否则，如果结果小于n，则角色互换，即i是n在b中的位置，j是n在a中的位置。 
5. 一旦我们知道了 posA 和 posB，就可以将偏移计算为 k = (posB − posA) mod n。 

### 为什么它有效

 正确性取决于以下事实：n 在两种排列中都是唯一的，并且在最大查询中主导所有其他值。 任何返回 n 的查询都会对查询的索引给出直接的逻辑约束。 对角线查询精确地隔离了任一结构中 n 出现的位置。 交叉查询解决了歧义，因为只有正确的对齐才允许n出现在最大值中。 循环移位定义确保一旦单个值的位置对齐，整个旋转偏移就固定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(x, y):
    print(f"? {x} {y}")
    sys.stdout.flush()
    return int(input())

def main():
    n = int(input())

    candidates = []

    for i in range(n):
        res = ask(i, i)
        if res == n:
            candidates.append(i)

    if len(candidates) == 1:
        print(f"! 0")
        sys.stdout.flush()
        return

    i, j = candidates[0], candidates[1]
    res = ask(i, j)

    if res == n:
        posA = i
        posB = j
    else:
        posA = j
        posB = i

    k = (posB - posA) % n
    print(f"! {k}")
    sys.stdout.flush()

if __name__ == "__main__":
    main()
```该实现围绕单个辅助函数进行，该函数执行交互式查询并立即刷新输出，因为忘记刷新会破坏与判断的同步。 

所有索引的循环恰好使用 n 个查询，每个查询检查对角线位置是否隐藏最大值。 之后，最多需要一次额外的查询来消除方向歧义。 模计算确保旋转正确归一化到范围 [0, n − 1] 内。 

一个常见的陷阱是假设两个检测到的索引始终对应于不同的角色； 尽早处理 k = 0 的情况可以防止不必要的查询并避免对相同位置的误解。 

## 工作示例

 考虑一个小的概念示例，其中 n = 4，隐藏数组为 a = [2, 3, 1, 4] 和 b = [4, 2, 3, 1]。 正确的移位是 k = 1。 

我们模拟对角查询：

 | 我| 查询（我，我）| 解读|
 | ---| ---| ---|
 | 0 | 最大 (2, 4) = 4 | 候选人 |
 | 1 | 最大 (3, 2) = 3 | 没有|
 | 2 | 最大 (1, 3) = 3 | 没有|
 | 3 | 最大(4, 1) = 4 | 候选人 |

 我们获得候选者 {0, 3}。 现在我们查询 (0, 3)。 结果是 max(a[0], b[3]) = max(2, 1) = 2，不是 4，所以角色颠倒了：索引 3 是 n 在 a 中的位置，索引 0 是 n 在 b 中的位置。 

因此 posA = 3，posB = 0，得出 k = (0 − 3) mod 4 = 1。 

该迹线显示了单个值（最大值）如何足以完全确定循环移位，而无需重建任一排列。

 ## 边缘情况

 当 k = 0 时，两个数组相同，因此每个对角线查询都会返回相同的值。 在这种情况下，只会收集一个索引作为候选索引，算法会立即输出 0，而不执行任何其他查询。 

当 n 的两个位置在索引空间中重合时，候选列表的大小为 1，这正确地反映了最大值出现在两个数组中的同一位置。 在这种情况下，算法不会尝试交叉查询，以避免歧义。 

当n出现在不同的位置时，交叉查询清楚地区分了方向，因为只有一次角色分配使得最大值有可能出现在查询对中。 这可以防止移位方向的错误反转。
