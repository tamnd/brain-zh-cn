 ## 解决方案
 ## 问题理解

 我们得到一个图表，其中每个节点代表一颗炸弹，并且每个炸弹都有一种颜色。 边代表炸弹之间的连接。 一旦选定的炸弹开始爆炸，它就会扩散，但只有在非常特定的条件下：只有当相邻炸弹已经爆炸并且两个炸弹沿该边缘具有相同颜色时，炸弹才能触发相邻炸弹。 

还有一个额外的转折。 在选择从哪里开始爆炸之前，我们最多可以将一枚炸弹重新着色为我们想要的任何颜色。 之后，我们选择一枚起始炸弹来点燃。 然后爆炸根据上述规则确定性地传播，并且通过有效传播到达的每个炸弹都被视为爆炸。 任务是最大化最终爆炸的炸弹数量。 

从结构的角度来看，图的边仅在两个端点共享相同颜色时才对传播有用。 这立即表明每种颜色内部的行为几乎是独立的，除了我们可以策略性地重新着色单个节点以合并以前单独的区域这一事实。 

约束多达三十万个节点和边，这排除了任何试图为每个起始节点或每个重新着色选择单独模拟爆炸过程的解决方案。 任何接近二次行为的行为都会立即失败，因此解决方案必须依赖于图的预先计算结构。 

当贪婪地思考传播时，会出现一个微妙的失败案例。 例如，人们可能会假设选择相同颜色节点的最大连通分量始终是最佳的。 这是错误的，因为重新着色单个节点可以合并多个组件。 

考虑一个简单的链 1-2-3，其中颜色为 A、B、A。如果不重新着色，就不会发生大的传播。 但是，如果我们将节点 2 重新着色为 A，突然间所有三个节点都在颜色 A 传播规则下表现为一个连通结构。 天真的“最大组件”方法将完全错过这种交互。 

另一个常见的错误是认为起始炸弹的重要性很复杂。 实际上，一旦我们确定了最终的可达结构，就始终可以在其中选择起始节点，因此除了要求我们选择一个非空可达集之外，它不会限制最终的大小。 

## 方法

 暴力方法会尝试每一种可能的起始炸弹和每一种可能的重新着色选择。 对于每种配置，我们将使用仅限于相同颜色边缘的 BFS 或 DFS 来模拟传播过程。 每个模拟都是 n + m 线性的，并且有 n 个起始炸弹的选择和 n 个重新着色的选择（包括不重新着色的选项）。 这导致大约 O(n(n + m))，这对于 3 × 10^5 来说远远超出了可行范围。 

关键的观察结果是，每种颜色内部的传播由该颜色引起的子图的连通分量控制。 如果我们暂时忽略重新着色，每个颜色类都会分解为连接的组件，并且每个组件都表现为一个原子单元：一旦其中的任何节点被激活，整个组件最终都会被激活。 

这将问题简化为推理组件大小而不是单个节点。 

现在我们介绍一下重着色操作。 如果我们将节点 v 重新着色为某种颜色 c，则 v 充当与 v 相邻的颜色 c 的所有分量之间的桥梁。我们不再处理原始节点，而是再次将所有内容简化为分量：我们只关心哪些颜色分量与 v 相邻。 

对于每个节点 v 和出现在其邻居中的每种颜色 c，将 v 重新着色为 c 允许我们合并与 v 接触的颜色 c 的所有不同分量以及 v 本身。 如果 v 还不是这些组件的一部分，则结果大小是这些组件大小的总和加一。

这提出了一种策略：预先计算每种颜色的所有连接组件，存储它们的大小，并为每个节点聚合按邻居组件标识符分组的贡献。 由于度数之和与 m 成线性关系，因此邻接表的迭代足够高效。 

我们还将基线答案保留为所有颜色的最大组件尺寸，而无需重新着色。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n(n + m)) | O(n(n + m)) | O(n + m) | 太慢了|
 | 组件压缩 + 一次重新着色合并 | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 1. 通过运行仅限于两个端点具有相同颜色的边的图形遍历（DFS 或 BFS），为每种颜色单独构建连接组件。 这将每个颜色引起的子图划分为多个组件。 我们这样做的原因是，在单一颜色内部，爆炸行为将每个连接区域视为一个单元。 
2.为每个节点分配一个组件标识符并存储每个组件的大小。 这允许我们稍后将任何节点组视为单个加权对象，而不是重复重新计算可达性。 
3. 计算基准答案作为所有组件的最大组件大小。 这对应于我们不使用重新着色或重新着色没有任何改进的情况。 
4. 对于每个节点 v，将其视为潜在的重新着色节点。 我们想要评估如果 v 重新着色为某种颜色 c 会发生什么。 为此，我们检查 v 的所有邻居并按其组件标识符对它们进行分组，但仅限于每个固定颜色 c 内。 
5. 对于 v 的邻居中出现的每种颜色 c，收集与 v 相邻的颜色 c 的不同分量 id 的集合。将它们的分量大小相加，并为 v 本身加 1。 这表示将 v 重新着色为 c 时创建的合并结构。 
6. 维持所有此类配置的全局最大值。 最终答案是基线组件和所有基于重新着色的合并之间的最大值。 

关键的效率思想是，在构建基于邻接的聚合时，每条边都会被处理恒定的次数，因此即使我们将每个节点视为潜在的桥梁，我们也不会以二次方式重复每种颜色的繁重工作。 

### 为什么它有效

 该算法依赖于固定颜色内的不变量，爆炸可达性完全等同于该颜色的诱导子图中的连通性。 对单个节点重新着色只会通过该节点潜在地合并先前单独的相同颜色组件来影响连接性，并且不能创建任何其他形式的远程交互。 因此，每个可达爆炸集完全对应于单个组件或通过一个重新着色的桥节点连接的组件的并集，并且该算法详尽地评估所有此类并集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    c = list(map(int, input().split()))

    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    comp_id = [-1] * n
    comp_size = []

    sys.setrecursionlimit(10**7)

    def dfs(start, cid):
        stack = [start]
        comp_id[start] = cid
        cnt = 0
        col = c[start]
        while stack:
            u = stack.pop()
            cnt += 1
            for v in g[u]:
                if comp_id[v] == -1 and c[v] == col:
                    comp_id[v] = cid
                    stack.append(v)
        return cnt

    cid = 0
    for i in range(n):
        if comp_id[i] == -1:
            comp_size.append(dfs(i, cid))
            cid += 1

    best = max(comp_size) if comp_size else 1

    for v in range(n):
        by_comp = {}
        for u in g[v]:
            cid_u = comp_id[u]
            if cid_u == -1:
                continue
            by_comp[cid_u] = comp_size[cid_u]

        for val in by_comp.values():
            best = max(best, val + 1)

    print(best)

if __name__ == "__main__":
    solve()
```该代码首先将每个颜色连接区域压缩为一个组件并存储其大小。 DFS 仅沿着颜色匹配的边缘扩展，确保我们不会将不同的颜色混合到同一组件中。 

之后，我们不再明确尝试所有具有颜色定位的重新着色选择，而是使用观察结果，即任何有益的重新着色都可以通过一个节点有效地连接多个相邻组件。 对于每个节点，我们查看其相邻组件并计算最佳合并大小。 这`+1`说明重新着色的节点充当桥梁。 

一个微妙的实现点是我们实际上不需要在第二阶段跟踪颜色。 组件 ID 已经对分色进行了编码，因为组件仅在相同的颜色中构建。 

## 工作示例

 ### 示例 1

 考虑一个小图：

 输入：```
5 4
1 1 2 2 2
1 2
2 3
3 4
4 5
```这形成了一条颜色分割相互作用的链。 

| 步骤| 节点| 组件构建 | 元件尺寸|
 | ---| ---| ---| ---|
 | 构建 | 1-2 | 1-2 比较 A（颜色 1）| 2 |
 | 构建| 3 | 比较 B（颜色 2）| 1 |
 | 构建 | 4-5 | 比较 C（颜色 2）| 2 |

 基线答案是 2。 

现在将节点 2 视为重新着色候选者。 如果我们将其重新着色为颜色 2，它将通过邻接连接组件 B 和 C。 

| 节点 v | 邻居比较（颜色 2）| 合并结果 |
 | ---| ---| ---|
 | 2 | 乙、丙 | 1 + 1 + 2 = 4 |

 所以答案就变成了4。 

该跟踪表明，改进来自于通过单个桥接节点合并多个相同颜色的组件。 

### 示例 2

 输入：```
4 3
1 2 3 4
1 2
2 3
3 4
```每个节点都有不同的颜色，因此每个组件的大小均为 1。 

| 节点| 相邻组件| 最佳合并|
 | --- | --- | --- |
 | 任何| 所有尺寸 1 | 至多 2 |

 即使重新着色，也没有任何节点可以有效连接两个以上不同的邻居，因此答案仍然很小。 

这表明，只有当多个相同颜色的组件已经与单个节点相邻时，重新着色才有帮助。

 ## 边缘情况

 一种重要的边缘情况是重新着色根本没有好处。 例如，如果每个节点都是孤立的或者每个组件都已在其颜色范围内最大，则算法会正确回退到基线最大组件大小。 

另一种边缘情况是当一个节点连接许多相同颜色的组件时。 该算法可以自然地处理这个问题，因为它会聚合所有相邻的组件 ID，并对每个组件的大小进行一次求和，从而避免重复计算。 

最后的边缘情况是这样的图，其中重新着色不会创建新的连接，因为邻居属于不同的颜色。 在这种情况下，每个节点的组件字典不会产生大于现有组件的有意义的合并，并且基线答案仍然是最佳的。
