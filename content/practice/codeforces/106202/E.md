 ## 问题理解

 我们得到一棵有 n 个顶点的未知树上的顶点对集合。 每对代表从一个顶点到另一个顶点的行程，这意味着一旦树被固定，它就对应于这两个顶点之间的唯一简单路径。 

我们可以选择这些对的子集，但树本身并不是固定的。 我们可以自由地在 n 个顶点上构建任何树。 所选子集必须满足对该构造树的约束：如果我们采用与所选对相对应的所有路径，则不允许单个树边出现在多个路径中。 

任务是在假设我们也可以以对我们有利的方式设计敌对树的情况下，最大化我们可以保留的对数。 

输入只是 m 个顶点对的列表。 输出是这些对的子集的最大大小，可以在某些树中同时实现，同时遵守边缘使用约束。 

约束最多为 2 · 10^5 个顶点和对，这会立即排除任何指数子集搜索或任何尝试显式推理所有可能树的方法。 每次操作超过线性或接近线性时间的任何操作都会失败。 

当成对形成循环时，会出现微妙的边缘情况。 例如，考虑三对 (1,2)、(2,3)、(1,3)。 如果我们天真地将它们视为独立的联系，我们可能会认为这三个都可以选择。 然而，在任何一棵树中，至少有一条对应路径必须与另一条路径在边上重叠，因此在该约束下三者不可能共存。 

这已经暗示真正的限制不是树结构本身，而是所选对如何组合交互。 

## 方法

 关键的难点是树不是固定的。 这使我们能够从结构的角度重新解释问题：我们不是推理未知树内的路径，而是询问这种树的存在对所选对施加了哪些约束。 

直接的强力方法是尝试对的所有子集，并且对于每个子集，尝试构建一棵树并路由所有相应的路径，以便不共享边。 即使我们忽略树的构造，简单地检查所有子集也已经花费了 O(2^m)，这对于 m 达到 2·10^5 来说是不可能的。 即使反复尝试测试可行性的贪婪子集选择仍然需要过于昂贵的全局一致性检查。 

关键的观察是我们可以简化路径的作用。 由于我们被允许设计树，因此我们不必将路径嵌入为长且受约束的路径。 如果两个顶点在所选结构中连接，我们总是可以使它们在树中相邻，除非这会与其他所选对产生结构冲突。 

这意味着我们可以用更强大的抽象来思考：每个选定的对的行为就像我们想要包含在树中的一条边。 然后，树的要求强加了一个全局条件：所选的集合必须是非循环的，因为所选连接中的任何循环都会与最终结构是树的要求相矛盾。 

一旦达到这个观点，问题就变成了一个经典问题：我们从给定的集合中选择最大可能的边子集，以便它们形成森林。 这正是最大非循环子集，可以使用不相交集并集结构找到它。 

我们一对一地处理对，并且仅当每对连接两个先前断开连接的组件时才保留它。 这保证了我们永远不会引入循环，并确保我们获取尽可能多的对。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集和树结构的暴力破解 | O(2^m) | O(2^m) | O(n) | 太慢了|
 | 基于DSU的最大森林构建| O(m α(n)) | O(m α(n)) | O(n) | 已接受 |

 ## 算法演练

 我们将问题转化为构建给定对的最大尺寸非循环子集。 

1. 初始化一个不相交的并集结构，每个顶点都在其自己的组件中。 这表示最初不存在强制连接。 
2. 迭代所有给定的对 (a_i, b_i)。 对于每一对，检查 a_i 和 b_i 是否已在同一组件中。 
3. 如果它们位于不同的组件中，请将这一对包含在我们的答案中并合并这两个组件。 这确保了我们在不创建循环的情况下扩展森林。 
4. 如果它们已经在同一组件中，请跳过该对，因为添加它会创建一个循环，该循环不能成为任何有效选择的一部分。 
5. 最终接受的对数就是答案。 

这种贪婪选择起作用的原因是，每次我们拒绝一条边时，都是因为它关闭了一个已经由先前接受的对强制执行的循环。 任何包含此边的解决方案都必须排除该循环中的至少一个边，因此拒绝它不会降低最佳答案。 

### 为什么它有效

 每走一步，被接受的对就形成一片森林。 任何有效的解决方案也必须是森林，因为所选择的对的循环将迫使至少一个树边缘被两条不同的路径重用，从而违反了约束。 DSU过程构建一个最大森林，任何最大森林的大小n减去连通分量的数量，这是不形成循环的最大可能的边数。 由于每个有效选择都对应于一些森林，因此这种结构是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.size[a] < self.size[b]:
            a, b = b, a
        self.parent[b] = a
        self.size[a] += self.size[b]
        return True

def main():
    n, m = map(int, input().split())
    dsu = DSU(n)

    ans = 0
    for _ in range(m):
        a, b = map(int, input().split())
        a -= 1
        b -= 1
        if dsu.union(a, b):
            ans += 1

    print(ans)

if __name__ == "__main__":
    main()
```DSU 结构维护由已选择的对引起的顶点的连接组件。 每个成功的联合对应于安全地添加一对而不形成循环。 答案很简单，就是成功工会的数量。 

一个常见的陷阱是忘记我们不需要显式地构造树。 DSU 已经隐式保证所选对可以嵌入到某些树中，因为任何森林都可以扩展到生成树。

| 步骤| 配对 | 相同的组件？ | 行动| 已接受计数 |

 ## 边缘情况

 一个关键的边缘情况是所有对形成一个循环。 例如，在三角形中，每条边单独有效，但所有三条边一起则无效。 DSU 的处理过程如下：前两条边合并组件，第三条发现两个端点都已连接，因此被拒绝。 这确保最终答案尊重循环约束，而无需显式检测循环。 

另一种情况是当图已经是一棵树时。 在这种情况下，每条边都会被接受，因为没有并集在同一组件中找到两个顶点。 该算法自然会返回 m，这与树恰好具有 m = n - 1 条边并且完全无环的事实相匹配。 

当没有给出对时，就会发生退化情况。 DSU 从不执行并集，结果为零，这正确反映了无法选择任何路径。
