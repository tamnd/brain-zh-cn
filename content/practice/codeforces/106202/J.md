 ## 解决方案
 ## 问题理解

 我们得到了一个由以下组成的几何结构$n$“层”。 对于每个整数$i$，有一个特殊点$(i,i)$。 从每个这样的点开始，有两条直线段延伸：一条垂直向下延伸到$(i,0)$，另一个水平向左移动到$(0,i)$。 这些线段与坐标轴一起创建了一个共享原点的方形框架的嵌套系统。 

旅行者开始于$(0,0)$并且可以沿着轴和任何给定的线段自由移动。 任务是访问每个对角点$(i,i)$为了$i = 1 \ldots n$，同时最大限度地减少方向变化的次数。 每当运动在水平方向和垂直方向之间切换（包括 180 度反转）时，都会计算方向变化。 

输入由单个整数组成$n$，可能大到$10^{12}$，并且我们必须输出完成遍历所需的最小圈数。 

尺寸为$n$立即排除对点或几何体的任何模拟。 任何解决方案都必须是常数时间，这意味着答案是从结构推理中得出的封闭式表达式。 

一个微妙的边缘情况是$n = 1$。 旅行者只需到达$(1,1)$，并且最优路线恰好涉及一个方向改变。 任何公式都必须干净地处理这个边界，因为模式源自更大的$n$经常多算或少算初始步骤。 

另一个潜在的陷阱是错误地假设每一层独立且均匀地贡献总匝数。 共享轴允许部分重用运动，但对角点是孤立的约束迫使重复返回到轴，这是大多数转弯发生的地方。 

## 方法

 暴力策略将显式地对由轴和所有线段形成的图进行建模，然后在跟踪方向状态的同时搜索访问所有对角线节点的最短路径。 这成为状态空间最短路径问题，其中每个状态包括位置和传入方向。 即使进行积极的修剪，可达到的几何状态的数量也会随着$n$，并且层之间的转换仍然需要探索大小与$n$。 这很快就变得不可行了$n$达到$10^{12}$。 

关键的观察结果是几何形状是高度重复的。 每个新层都会引入一个新的方框，该方框嵌套在先前的方框内。 达到$(i,i)$，路径必须不可避免地通过一个轴进入该层，访问该点，然后返回到一个轴，然后继续向外或向内。 这迫使每层方向变化的固定模式。 

一旦我们将运动视为沿着轴和“辐条”到对角点的水平和垂直遍历之间的交替，就会清楚地看到该过程的行为就像受控的之字形。 每个附加层轮流贡献可预测的增量，与早期结构无关。 

仔细跟踪最佳运动表明，第一层需要转一圈才能到达其对角点，并且每个后续层都贡献恰好两圈的额外转弯，以将新的访问整合到现有的遍历中，同时保持连续性。 

这直接导出线性公式$n$。

 ## 算法演练

 1. 观察所有运动都发生在轴对齐的段上，因此每个运动要么是水平的，要么是垂直的。 成本的唯一来源是在这些方向之间切换。 
2. 从$(0,0)$。 到达第一个对角点$(1,1)$，任何有效路径必须恰好执行一次方向改变。 这确定了基本成本为 1。 
3.考虑添加一个新层$i$。 参观$(i,i)$，路径必须首先位于 x 轴或 y 轴坐标处$i$，然后沿着垂直线段移动到对角点。 
4.参观后$(i,i)$，路径不能停在那里，因为剩余的层仍然需要遍历。 该结构迫使返回轴以继续到达更远的点。 
5. 每次“绕道”进入新的对角点都会引入两个不可避免的方向变化：一是进入该点，二是返回到轴对齐的遍历。 
6、贡献求和，第一层贡献1回合，其余各层贡献1回合$n-1$层贡献 2 圈，总共$2n - 1$。 

### 为什么它有效

 不变的是路径总是在轴遍历（纯水平或垂直运动）和垂直偏移到对角点之间交替。 每一次旅行都进入一个新的领域$(i,i)$需要进入和退出轴对齐的段，并且这些过渡是由几何形状而不是选择强制的。 由于该结构是严格嵌套的并且不允许绕过或合并访问，所以没有两层可以共享入口或出口转弯。 这将总圈数锁定为固定的线性累加。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    if n <= 1:
        print(1 if n == 1 else 0)
        return
    print(2 * n - 1)

if __name__ == "__main__":
    main()
```该解决方案直接应用导出的封闭式公式。 唯一值得关注的实现细节是边界情况$n = 1$，它必须显式返回 1 以避免将表达式折叠成仅渐近地适用于较大值的模式。 

所有计算均在恒定时间内完成，不需要几何模拟。 

## 工作示例

 ### 示例 1：$n = 1$我们只有重点$(1,1)$。 最佳路径是单轴移动，然后垂直移动到该点。 

| 步骤| 职位| 方向 | 转|
 | ---| ---| ---| ---|
 | 1 | (0,0) → (1,0) | 水平| 0 |
 | 2 | (1,0) → (1,1) | 垂直| 1 |

 总共1圈，符合公式$2 \cdot 1 - 1 = 1$。 

这证实了仅需要一个方向开关的基本情况行为。 

### 示例 2：$n = 3$我们期望$2 \cdot 3 - 1 = 5$轮流。 

概念上的最佳遍历通过重复使用轴和绕道按顺序访问每个对角点来进行。 

| 步骤| 行动| 方向 | 转|
 | ---| ---| ---| ---|
 | 1 | 沿轴向第 1 层移动 | 水平| 0 |
 | 2 | 输入 (1,1) | 垂直| 1 |
 | 3 | 返回轴 | 水平| 2 |
 | 4 | 输入 (2,2) | 垂直| 3 |
 | 5 | 返回并继续结构 | 水平| 4 |
 | 6 | 进入最后一层 (3,3) | 垂直| 5 |

 该迹线表明，每个新层恰好添加两圈，除了初始条目贡献一圈之外。

 ## 边缘情况

 对于$n = 1$，结构塌陷到单个目标点。 该算法直接返回 1，这与从轴到对角线段所需的单个方向变化匹配。 

为了$n = 2$，公式给出$2 \cdot 2 - 1 = 3$。 遍历从水平移动开始，切换到垂直到达$(1,1)$，然后再次交替以在完成之前整合第二层，从而产生恰好三个方向的变化。 

对于大型$n$，计算在形式上保持相同。 没有迭代确保不需要中间几何推理，并且公式在最大约束下保持稳定。
