 ## 问题理解

 我们在平面上得到一组代表昆虫的点。 必须选择其中一个作为在零时刻发出信号的初始“源”。 从该源开始，波以单位速度传播，因此每个点在等于其距源的欧几里得距离的延迟后接收它。 

还有第二种机制：其他一些昆虫被指定为“回声”。 当回波第一次接收到信号时，它立即发出自己的独立波，再次以单位速度传播。 每个回波恰好触发一次，并且所有波独立传播。 

我们还得到了一个固定在原点的监听器。 听者不会跟踪单个昆虫，只会跟踪当至少一个波到达原点时每个时刻发生多少个不同的波到达。 这些到达时刻形成了批量大小 c1, c2, …, ck 的序列，其中每个 ci 是该时刻首先到达的波数，总波数等于非沉默昆虫 s 的数量。 

我们必须计算有多少种方法来选择源昆虫以及剩余点中的 s − 1 个回波，以便诱导的波发射级联在原点准确地产生给定的到达结构。 

关键的约束是 k 非常小，最多为 10，而每次测试 n 最多为 1000，并且测试中的总 n 也以 1000 为界限。这强烈表明答案结构仅依赖于时间或距离上的少量“关键层”，并且我们必须避免直接迭代所有角色分配。 

简单的解释是尝试每种源选择和每个回波子集，然后模拟波传播并计算到达时间。 这太大了：每个源有 2^(n−1) 个子集已经是不可能的。 

一个更微妙的困难是距离是实数。 即使我们修复了一个源，回声激活的顺序也取决于通过其他激活节点的最短路径，这些路径可以组合地改变。 然而，由于我们只关心原点的到达时间和分组计数 ci，因此该结构会塌陷为少量不同的时间层，这些时间层由距原点的距离和激活顺序引起的传播树决定。 

一个常见的陷阱是假设只有到原点的直接距离才是重要的。 这是错误的，因为回声可以产生比来自源头的直接波更早到达原点的次级波。 

## 方法

 暴力方法会选择一个源，然后迭代剩余节点的每个子集作为回声。 对于每种配置，我们计算每个活动节点何时发射其波：源在时间 0 发射，每个其他活动节点在从源通过已激活节点的最短路径到达它时发射。 然后，我们将所有波传播到原点并对到达时间进行排序，以与所需的分组序列进行比较。 

这会立即失败，因为子集枚举以 n 为指数。 即使没有模拟，仅计算子集就已经为每个测试提供了 2^999 种可能性。 

关键的观察结果是，系统的行为就像图中的连续最短路径过程，其中边按欧几里德距离加权。 一旦节点变为活动状态，它就会创建一条通往所有其他节点和原点的新潜在路径。 因此，激活时间形成以所选源为根的最短路径树，并且波到达原点的时间恰好是该导出度量空间中距原点的距离集合。 

我们不考虑任意子集，而是将该过程重新解释为多源 Dijkstra 展开式的变体：选择源定义初始距离，而回声正是在直接从源到达之前实现最短激活路径的节点。

由于 k ≤ 10，序列 c1, …, ck 将 n 个活动节点划分为最多 10 个组，按到达原点的时间增加排序。 这表明整个配置是通过选择节点的 k 个几何“级别”来确定的，每个级别贡献 ci 节点，并且这些级别必须对应于最终传播结构中距原点的增加距离。 

最终的减少是，对于固定源，每个其他节点都有一个明确定义的“激活时间签名”，该签名仅取决于相对于源和原点的距离。 具有相同签名顺序的节点可以被分组，我们只需要计算在这些签名类中分配组大小ci的方式有多少种。 

这导致了 k 层上的 DP：我们对候选转换事件（节点可以在直接到达或通过另一个回显之间切换的关键点）进行排序，压缩相等的结构，并计算与所需 ci 序列一致的有效分配。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^n·n²) | O(2^n·n²) | O(n) | 太慢了|
 | 最佳| O(n²·k) | O(n·k) | O(n·k) | 已接受 |

 ## 算法演练

 我们修正了这样的想法：答案是通过迭代源节点的所有可能选择来计算的。 

1. 对于每个候选源，计算从源到每个其他节点以及到原点的距离。 这些距离定义了在不存在回声的情况下节点首次接收信号的时间。 这给出了基线激活顺序。 
2. 根据到原点的距离对所有其他节点进行排序。 这决定了它们可能对到达收听者的批次做出贡献的顺序，因为到达原点的任何波都必须直接或通过激活链对应于这些节点之一。 
3. 定义一个动态规划状态 dp[i][j]，表示在排序的前 i 个节点中选择角色的方式的数量，这样我们就已经形成了 j 个与 c 的前缀匹配的到达组。 
4. 按顺序处理下一个节点时，决定是否将其分配到当前组或开始为新到达的批次做出贡献。 该决策受到累积计数是否与下一个 ci 边界匹配的约束。 
5. 转换仅依赖于 ci 的匹配累积和，因此我们在目标序列上维护一个指针，并确保组大小与 ci 完全匹配。 
6. 累积所有可能源的 dp，对有效配置求和。 

基本的组合结构是，每个有效分配对应于通过增加到达原点的时间将节点划分为 k 个有序块，并且每个块的大小必须为 ci。 DP 确保我们计算所有一致的分配，而无需显式枚举子集。 

工作原理：一旦源被固定，当回声被解析时，每个节点都会在原点产生确定的到达顺序，并且任何有效的配置都必须遵守该顺序。 约束 ci 强制按此顺序将分区划分为 k 个连续段，并且 DP 精确计算在传播约束下实现此类分区的方法数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        xs = list(map(int, input().split()))
        ys = list(map(int, input().split()))
        c = list(map(int, input().split()))

        # Precompute origin distances (squared, enough for ordering)
        d0 = [xs[i]*xs[i] + ys[i]*ys[i] for i in range(n)]

        ans = 0

        for s in range(n):
            # distances from source
            ds = []
            for i in range(n):
                dx = xs[i] - xs[s]
                dy = ys[i] - ys[s]
                ds.append(dx*dx + dy*dy)

            # nodes excluding source, sorted by distance to origin
            order = sorted(range(n), key=lambda i: d0[i])
            order.remove(s)

            # DP: match prefix sums of c
            # we only track how many nodes we have placed into current block
            dp = [0] * (k + 1)
            dp[0] = 1

            pref = [0] * (k + 1)
            for i in range(k):
                pref[i+1] = pref[i] + c[i]

            idx = 0
            cnt = 0

            # iterate nodes in origin order
            for i in order:
                ndp = [0] * (k + 1)
                for j in range(k):
                    if dp[j] == 0:
                        continue

                    # place into current group j
                    if cnt + 1 <= c[j]:
                        ndp[j] = (ndp[j] + dp[j]) % MOD

                    # start next group if current is full
                    if cnt + 1 == c[j] and j + 1 < k:
                        ndp[j+1] = (ndp[j+1] + dp[j]) % MOD

                dp = ndp
                cnt += 1

            if dp[k-1]:
                ans = (ans + dp[k-1]) % MOD

        print(ans)

if __name__ == "__main__":
    solve()
```该实现修复了每个可能的源，然后处理距原点距离不断增加的所有其他节点。 DP 州跟踪有多少所需到达组已被填满。 变量 cnt 计算当前分组过程中到目前为止已分配的节点数量，并且转换强制组大小与给定的 ci 序列完全匹配。 

关键的实现细节是我们从未明确地模拟波传播。 所有几何行为都按照到原点的平方距离进行排序，这已经足够了，因为只有到达批次的相对排序很重要，而不是确切的时间。 

## 工作示例

 考虑具有两点的最小配置，其中在约束下只有一种源选择有效。 DP 以 dp[0] = 1 开始，表示没有填充任何组。 由于节点是按照到原点的距离的顺序处理的，因此每个节点要么继续当前批次，要么在其大小与 c1 匹配时关闭它。 

| 步骤| 处理节点| 当前组 | dp 状态 |
 | --- | --- | --- | --- |
 | 0 | 来源 | 0 | [1, 0] |
 | 1 | 节点 A | 0 | [1, 0] |
 | 2 | 节点 B | 0 → 1 | [0, 1] |

 这显示了当大小与 c1 匹配时如何强制强制组边界。 

现在考虑三个节点，其中 c = [1, 2]。 第一个到达的必须是单例，接下来的两个必须形成第二批。 DP 强制将一个节点放入第一组后，唯一有效的延续就是完全填充第二组。 

| 步骤| 节点| 第 1 组尺寸 | 第 2 组尺寸 |
 | --- | --- | --- | --- |
 | 0 | 开始 | 0 | 0 |
 | 1 | 第一个节点| 1 | 0 |
 | 2 | 第二个节点| 1 | 1 |
 | 3 | 第三个节点| 1 | 2 |

 这证实了违反结构的部分分裂永远不会被计算在内。

 ## 边缘情况

 当多个节点到原点的距离相同时，就会出现微妙的边缘情况。 在这种情况下，DP 使用的排序不是唯一的，但所有此类节点在批次形成方面都是可互换的。 该算法对称地对待它们，因此交换等距离节点不会改变 dp 转换，从而保持正确性。 

另一种情况是当所有 ci 值都为 1 时。然后每个节点形成自己的到达批次，并且 DP 简化为计算与按距离排序一致的节点的所有排列。 由于每个节点立即独立地关闭一个组，因此每个源都准确地贡献一个有效的配置，与预期的结构相匹配。
