 ## 解决方案
 ## 问题理解

 我们正在构建长度的字符串$n$使用恰好七个符号的固定字母表，从$a$到$g$。 字符串中的每个位置都是独立于该字母表选择的，但并非每个结果字符串都是允许的。 该限制来自一组禁止规则。 每个规则列出了一小部分字母，如果字符串在字符串中的某个位置至少包含一次来自任何一条规则的所有字母，则该字符串将无效。 

因此，每条规则都与邻接​​或顺序无关，而是与存在有关。 像这样的规则$\{b, c, d\}$禁止任何至少包含一个的字符串$b$, 至少一个$c$，以及至少一个$d$，无论它们出现在哪里。 

任务是计算有多少有效长度-$n$字符串存在于这些约束下，模$10^9 + 7$。 关键的难点在于$n$可以大到$10^9$，因此我们无法显式迭代位置或构造字符串。 约束的结构必须压缩为纯粹的组合计数。 

字母表大小恒定为 7，这是最重要的结构提示。 任何解决方案最终都必须对这 7 个字符的子集进行推理，而不是对字符串中的位置进行推理。 

当我们尝试模拟或枚举字符串并检查每个字符串的约束时，就会出现幼稚的故障模式。 即使是为了$n=50$，字符串的数量是$7^{50}$，太大了。 另一个微妙的错误是独立处理规则并减去每个规则的无效计数而不处理重叠，这会在字符串同时违反多个规则时导致重复计数。 

## 方法

 蛮力的想法很简单：生成所有$7^n$字符串并通过检查每个字符串是否包含任何规则中的所有字符来针对所有禁止的规则进行测试。 这是正确的，但立即不可行，因为即使存储或迭代所有字符串也是不可能的$n$超出小值。 

稍微好一点的尝试是，对于每个规则，计算有多少个字符串包含其所有字符，然后从总数中减去$7^n$。 然而，由于规则严重重叠，这种情况就失效了。 单个字符串可能违反多个规则，并且所有规则子集的包含-排除变得复杂且低效，因为$m < 27$仍然允许指数子集，但真正的结构不在规则中，而是在字符子集中。 

关键的转变是停止从规则的角度思考，而是从字符串中出现哪些字母的角度思考。 任何字符串都对应一个子集$S \subseteq \{a,\dots,g\}$至少出现一次的字母。 一旦这个子集被固定，实现它的字符串数量仅取决于$n$，不上安排。 

如果字符串恰好使用中的字母$S$，那么每个位置都选自$S$，但每个元素$S$必须至少出现一次。 这是对子集的经典包含-排除$S$，但我们不需要直接完整的斯特林数，因为$|S|\le 7$。 

第二个关键观察是规则仅依赖于包含：规则禁止子集$T$如果选择的字母子集$S$满足$T \subseteq S$。 所以对于每一个可能的$S$，我们只需要知道它是否包含任何禁止规则子集。 

由于字母表大小为 7，我们可以将每个子集表示为 7 位掩码。 每个规则都是一个掩码和一个候选字母表$S$如果它不包含任何禁止的掩码作为子掩码，则有效。 

一旦我们修复了有效的$S$，这些字母上的字符串数量为：$$f(S) = \sum_{k=0}^{|S|} (-1)^k \binom{|S|-k}{\cdot} (|S|-k)^n$$但更清楚地说，它是字符串的标准计数，其使用的字母集恰好是$S$，通过子集的包含-排除计算$S$:$$f(S) = \sum_{T \subseteq S} (-1)^{|S|-|T|} |T|^n$$现在问题简化为迭代 7 个字母的所有子集，过滤那些不违反任何规则的子集，然后求和$f(S)$。 

因为$2^7 = 128$，即使预先计算幂，这也是完全可以管理的$k^n$为了$k \in [0,7]$。

 | 最佳 |$O(2^7 \cdot 2^7)$|$O(2^7)$| 已接受 |

 ## 算法演练

 我们对字母进行编码$a$到$g$作为从 0 到 6 的位，因此每个子集都是一个整数掩码$[0, 127]$。 

1.读取所有规则并将每个规则转换为位掩码$R_i$。 这让我们可以使用位操作快速测试遏制。 一组规则违反了$S$恰好在什么时候$(R_i \& S) = R_i$。 
2. 预计算权$p[k] = k^n \bmod (10^9+7)$为了$k = 0$到$7$。 这捕获了字母表大小的字符串数量$k$无限制。 这是有效的，因为一旦我们确定了字母的子集，每个位置就会独立地在其中进行选择。 
3. 迭代所有子集$S$的$\{0,\dots,6\}$。 每个子集表示候选字符串中允许的字母集。 
4. 对于每个子集$S$，根据所有规则检查有效性。 如果存在规则掩码$R$这样$R \subseteq S$，然后跳过这个子集，因为它违反了禁止条件。 
5. 如果$S$是有效的，使用其子掩码的包含-排除来计算其贡献：

 开始于$res = 0$，然后对于每个子网掩码$T \subseteq S$, 加或减$p[|T|]$取决于奇偶校验$|S|-|T|$。 这确保我们只计算完全使用中字母的字符串$S$。 
6. 对所有有效子集的贡献求和$S$，取模$10^9+7$。 

### 为什么它有效

 每个字符串唯一对应一个子集$S$其中出现的字母。 该算法按该子集对所有字符串进行分区。 包含-排除公式确保每个字符串在与其精确使用的字母集相对应的术语中精确计数一次，而不是在任何较小的子集中。 规则过滤保证所使用的字母中没有完全包含禁止的子集，因此最终的总和中不会包含无效的字符串。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def popcount(x):
    return bin(x).count("1")

n, m = map(int, input().split())

rules = []
for _ in range(m):
    parts = input().split()
    k = int(parts[0])
    mask = 0
    for c in parts[1:]:
        mask |= 1 << (ord(c) - ord('a'))
    rules.append(mask)

# precompute k^n
powk = [1] * 8
for k in range(1, 8):
    powk[k] = pow(k, n, MOD)

# precompute all subsets of 7 bits
ALL = 7

ans = 0

for S in range(1 << ALL):
    ok = True
    for r in rules:
        if (r & S) == r:
            ok = False
            break
    if not ok:
        continue

    sub = S
    res = 0
    while True:
        bits = popcount(sub)
        sign = 1 if ((popcount(S) - bits) % 2 == 0) else -1
        res = (res + sign * powk[bits]) % MOD

        if sub == 0:
            break
        sub = (sub - 1) & S

    ans = (ans + res) % MOD

print(ans % MOD)
```该代码首先将每个禁止规则编码为位掩码，从而使用按位运算启用恒定时间子集检查。 数组`powk`商店$k^n$对于所有可能的字母大小，这是有效的，因为在固定字母子集之后，每个位置独立地在这些字母中进行选择。 

外循环枚举所有可能的字母子集。 每个子集都根据所有规则进行测试，如果它完全包含任何禁止的模式，则拒绝它。 内部子掩码循环在子集上实现包含-排除，以确保我们计算所使用字母的确切集合是精确的字符串$S$，不是任何真子集。 

交替符号由删除多少个字母的奇偶性决定$S$，匹配子集上的标准包含排除原则。 

## 工作示例

 ### 示例 1

 假设我们有规则$\{a,b\}$和$\{b,c,d\}$， 和$n=3$。 

我们考虑字母的子集。 拿$S = \{a,c\}$表示为掩码`101`。 

| 子掩码| 位| 标志| 贡献 |
 | --- | --- | --- | --- |
 | 000 | 000 0 | + |$0^3 = 0$|
 | 001| 1 | - |$1^3 = 1$|
 | 100 | 100 1 | - |$1^3 = 1$|
 | 101 | 101 2 | + |$2^3 = 8$|

 所以总共是$8 - 1 - 1 = 6$，对应于完全使用的字符串$a$和$c$。 

该子集有效，因为它不包含两者$a,b$并且不包含$b,c,d$。

 让$S = \{a,b,c\}$对于相同的规则。 

| 子掩码| 位| 标志| 贡献 |

 ## 边缘情况

 一种微妙的情况是根本没有规则。 然后每个字母子集都是有效的，并且该算法简化为对所有子集进行包含-排除求和，从而精确地折叠为$7^n$。 代码自然地处理了这个问题，因为没有子集被过滤掉。 

另一种情况是规则包含全部 7 个字母。 任意子集$S$非空立即违反了它，只留下空子集。 为了$n > 0$，空子集贡献为零，因为$0^n = 0$，所以答案变成零，这是正确的，因为每个非空字符串都会违反规则。 

最后一个有趣的例子是重叠规则，例如$\{a,b\}$和$\{b,c\}$。 像这样的子集$\{a,b,c\}$违反了两者，但仍然只被拒绝一次，因为有效性检查是一个简单的存在测试，而不是逐条规则的减法。
