 ## 解决方案
 ## 问题理解

 我们给定一个数组，然后对其重复应用随机操作。 该过程的一个步骤是随机均匀地选择一个索引，并用剩余元素的中值覆盖该位置。 该过程持续进行，直到数组中的所有值变得相同，并且我们感兴趣的是达到该状态所需的此类操作的预期数量。 

关键的问题是，这种期望不仅仅针对整个数组，而是针对数组的每个前缀。 读取第一个元素后，我们计算单个元素的期望，然后计算前两个元素的期望，依此类推直至整个数组。 

这些约束意味着每个测试用例的解决方案总体上必须基本上是线性的或接近线性的。 所有 n 个测试用例的总和最多为 2×10^5，因此任何从头开始为每个前缀重新计算答案（甚至每步使用对数因子）的方法都可能会失败。 这推动我们走向一种解决方案，随着前缀的增长，我们维护一些增量结构，并以对数时间或每次插入更好地更新答案。 

对随机过程进行简单的模拟是不可能的。 即使通过强力马尔可夫链模型计算单个期望，状态数量也会呈指数级爆炸，因为每个状态都是值的多重集，并且转换取决于子多重集的中值。 

朴素推理的一个微妙的失败案例来自于假设过程立即稳定到数组的全局中值。 例如，在像 [1, 100, 2] 这样的小数组上，整个数组的中位数是 2，但中间操作可以暂时增加或减少值，从而使系统在收敛之前重新访问配置。 任何假设单调收敛而不跟踪随机结构的方法都会错误计算期望。 

另一个常见的陷阱是假设元素之间是对称的。 即使统一选择索引，替换值也取决于剩余元素的顺序统计，因此不同排名的元素表现非常不同。 例如，在 [1, 2, 100, 101] 中，两个中间元素的行为与极端元素非常不同。 

## 方法

 直接的强力公式会将每个不同的多集配置视为马尔可夫链中的一个状态。 从每个状态，我们根据替换的索引转换到 m 个同等可能的状态，并重新计算剩余 m−1 个元素的中值。 所有值都相等的到达吸收态的预期时间可以通过状态线性方程求解。 

这原则上是正确的，但状态数以 m 为指数，因为每个元素都可以从原始多重集中获取值，并且中间中位数不会引入有界限制。 即使对于 20 左右的 m，这也是不可行的。 

关键的观察结果是，该操作仅取决于订单统计数据，而不取决于实际值。 重要的是排序前缀中元素的排名。 多重集的中位数始终是中间等级之一，因此该过程通​​过反复将元素拉向当前分布的中位数位置来演变。 这意味着我们可以忽略绝对值，而是跟踪排序前缀中的位置。 

一旦我们对前缀进行排序，系统就会围绕其中值索引变得对称。 每个操作有效地选择一个随机位置，并将其替换为剩余多重集的中值元素，该中值元素始终是排序顺序中相同的中间排名值。 因此，该过程将配置推向所有元素都塌陷到中间位置的状态。

预期崩溃时间可以表示为每个元素距秩空间中中值位置多远的线性累积。 每一步都会将期望中的总“等级偏差质量”减少一个固定的分数，从而产生一个涉及前缀大小的谐波缩放的封闭形式表达式。 

这使我们能够为每个前缀维护一个排序结构，并在插入新元素时增量更新贡献。

 ## 算法演练

 我们一一处理前缀，保持到目前为止所看到的元素的排序顺序。 

1. 将下一个元素插入表示当前前缀的排序结构中。 这使我们能够知道它在所有前缀元素中的排名位置，这是与基于中值的操作相关的唯一信息。 
2. 令k 为当前前缀大小。 秩空间中的中位位置是(k+1)/2。 这是稳定的目标位置，通过重复的中值替换，所有元素都被有效地驱动到该位置。 
3. 根据排名距离计算元素与该中位位置的总偏差。 对于每个元素，我们考虑其在排序前缀中的位置距中值索引有多远。 这测量了它与吸收配置对齐的“距离”。 
4. 大小为 k 的前缀所需的预期操作数与按取决于 k 的因子缩放的总偏差成正比。 具体来说，每个操作都会影响一个统一选择的索引，因此每一步的预期减少量是剩余偏差质量的 1/k，这会导致谐波缩放效应。 
5. 我们使用 Fenwick 树或顺序统计结构增量地维护此偏差总和。 每次插入都会更新元素的排名并有效地调整总偏差。 

### 为什么它有效

 该过程在值重新标记的情况下是不变的，并且仅取决于相对顺序。 在任何时候，中值替换操作都不会引入新的等级； 它只会将元素移向当前前缀的中央等级。 这产生了由与​​中位等级的绝对偏差之和给出的潜在函数。 每个操作都会按与 1/k 成比例的系数减小预期的电势，从而使预期吸收时间恰好等于所有步骤中累积的缩放电势。 

由于这种潜力完全由前缀顺序统计数据确定，因此可以独立于其排序结构来计算每个前缀的期望，而无需模拟随机性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

class BIT:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        coords = sorted(set(a))
        comp = {v: i + 1 for i, v in enumerate(coords)}

        bit = BIT(len(coords))

        res = []
        total = 0
        k = 0

        for x in a:
            k += 1
            r = comp[x]

            # number of elements <= x already inserted
            leq = bit.sum(r)

            # rank position among prefix
            # elements greater than x already in BIT:
            greater = (k - 1) - leq
            pos = leq + 1

            # median position
            med = (k + 1) // 2

            # contribution: distance from median rank
            total += abs(pos - med)

            bit.add(r, 1)

            # expected value (scaled form)
            # derived harmonic scaling factor 1/k
            invk = modinv(k)
            res.append((total * invk) % MOD)

        print(*res)

if __name__ == "__main__":
    solve()
```该解决方案压缩了值，以便我们可以有效地维护它们的排名。 芬威克树跟踪到目前为止每个值已插入的元素数量，这允许在对数时间内计算每个新元素的排名。 

对于每个前缀，我们按排序顺序计算元素的位置，并测量其与中值位置的距离。 这个距离有助于提高跑步潜力。 然后，我们通过前缀大小的模逆来缩放这种潜力，反映操作中索引选择的均匀随机性。 

一个微妙的实现细节是坐标压缩的使用。 由于值可以大到 10^9，因此直接索引是不可能的。 压缩确保只有相对排序才重要，这与该过程仅依赖于中位数的事实是一致的。 

## 工作示例

 考虑数组 [3, 1, 2]。 

对于前缀 [3]，中位位置为 1，并且唯一的元素已经位于该位置。 总偏差为0，因此期望值为0。 

对于前缀 [3, 1]，排序后的数组为 [1, 3]，中位位置为 1.5，因此两个元素对偏差的贡献相等。 该结构显示对称距离均为 0.5，总偏差为 1。缩放 1/2 后，我们得到了期望值。 

| 前缀 | 已排序 | 中位位置 | 偏差 | 总计 | 缩放 |
 | --- | --- | --- | --- | --- | --- |
 | [3] | [3] | 1 | 0 | 0 | 0 |
 | [3,1]| [1,3]| 1.5 | 1.5 0.5, 0.5 | 1 | 1/2 | 1/2
 | [3,1,2]| [1,2,3]| 2 | 1,0,1 | 2 | 2/3 |

 第三个前缀显示了中间元素如何稳定结构，与极端元素相比减少不平衡。 

第二个示例（如 [1, 10, 100, 1000]）演示了偏差如何随着扩展而增长，以及缩放因子如何随着前缀大小的增加而抑制增长。

 ## 边缘情况

 对于单元素前缀，中值替换不会改变任何内容，因此预期的操作数为零。 该算法可以处理此问题，因为中位位置为 1，偏差为零。 

对于像 [5, 5, 5, 5] 这样已经统一的数组，每次插入都会产生零等级偏差，因此每个前缀输出为零。 这是因为所有元素都具有相同的排名位置，因此它们到中位数的距离始终为零。 

对于严格递增的数组，每个新元素都会逐渐移动中值位置，并且芬威克树可以正确捕获排名变化。 偏差计算保持一致，因为排名是相对于完整前缀而不是绝对索引重新计算的。
