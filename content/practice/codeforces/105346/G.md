 ## 解决方案
 ## 问题理解

 我们有一个网格，其中每个单元格的行为就像小迷宫中的地形类型。 旅行者从恰好标记的一个单元格开始`S`并且必须到达标记为的唯一出口单元`E`。 允许在四个基本方向上移动，并且每次移动花费一个单位时间。 

大多数单元格要么是空的，要么是阻塞的，要么是特殊的。 空单元格可以自由遍历。 细胞标记`P`是永久性的墙。 细胞标记`C`是收藏品，踩到每一个都会使可用的玉米糖数量增加 1 个。 细胞标记`J`就像上锁的门一样：只有当前持有至少一颗玉米糖才能进入，踩在它们上会消耗一颗玉米糖。 

关键的复杂性是通过的能力`J`细胞数量取决于多少`C`沿路径收集了单元格，并且由于总共最多有 8 个糖果，因此状态空间仍然有界。 

任务是计算从`S`到`E`，或报告不存在有效路径。 

网格大小最多为 100 x 100，因此最多有 10,000 个单元格。 忽略糖果状态的天真的最短路径搜索将会失败，因为它会处理`J`细胞不正确。 同时，任何解决方案都必须处理因收集糖果而引起的状态变化，因此仅靠位置是不够的。 

正确的状态必须包括位置和当前糖果数量。 由于最多有 8 颗糖果，因此完整状态空间最多为 100 × 100 × 9，对于 BFS 来说已经足够小了。 

当重新访问具有不同糖果数量的同一单元格时，会出现微妙的边缘情况。 例如，到达一个`J`拥有 0 颗糖果的单元格是不可能的，但稍后到达并拥有 1 颗糖果可能是有效的。 仅按位置键控的天真的访问数组会错误地丢弃有效路径。 

当糖果需要通过一个`J`细胞，但最佳路径需要先绕道收集。 忽略未来约束的贪婪最短路径在这里将会失败。 

## 方法

 暴力方法将尝试枚举所有可能的路径`S`到`E`，沿着每条路径跟踪糖果收集。 每个步骤最多分为四个方向，因此可能的路径数量随着路径长度呈指数增长。 即使对于 100 x 100 的网格，这也是完全不可行的，因为仅简单路径的数量就已经是天文数字了。 

这是不必要的，因为该问题具有最优的子结构：无论我们如何到达那里，用相同数量的糖果到达相同的单元格都是等效的。 一旦我们认识到状态是通过位置和糖果数量来完全描述的，问题就变成了未加权图中的最短路径。 

这直接表明在扩展的状态空间上进行广度优先搜索。 每个节点都是一个三元组`(i, j, k)`在哪里`k`是当前持有的糖果数量。 转换对应于移动到相邻单元格并更新`k`取决于细胞类型。 由于每次移动只花费一个单位，因此 BFS 保证最短路径。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数| 指数| 太慢了 |
 | 扩展状态图上的 BFS | O(n·m·9) | O(n·m·9) | 已接受 |

 ## 算法演练

 我们将每个可达配置视为图中的一个状态，并从起始状态运行 BFS。 

1. 找到起始单元格`S`并用状态初始化 BFS 队列`(S_row, S_col, 0, 0)`，其中最后一个值是所采取的步数。 糖果计数从零开始，因为尚未收集任何糖果。 
2. 创建一个已访问的结构，索引为`(row, col, candy_count)`。 这确保了我们不会错误地丢弃重新访问具有不同数量糖果的单元格的状态。 
3. 当队列不为空时，提取最前面的状态`(r, c, k, dist)`。 
4.如果当前单元格是出口`E`，立即返回`dist`，因为 BFS 保证这是最短的可能路径。 
5. 对于四个可能的移动方向中的每一个，计算下一个单元格`(nr, nc)`。 如果出界或南瓜，则跳过它`P`。 
6.根据目标单元格判断移动是否有效：

 如果是的话`J`， 确保`k > 0`，然后递减`k`因为吃了一颗糖果而增加了一个。 

如果是的话`C`, 增量`k`踏入其中后增加一分。 

如果是空的话，`S`， 或者`E`，除非已经处理，否则糖果数量保持不变。 
7. 如果结果状态`(nr, nc, nk)`没有被访问过，标记为已访问过，并按距离推入队列`dist + 1`。 
8. 如果队列已耗尽而未到达`E`， 返回`"SPOOKED!"`。 

### 为什么它有效

 该算法以距离递增的顺序探索状态，因为 BFS 在深度上处理所有状态`d`在任何深度之前`d+1`。 每个状态都对做出未来决策所需的所有信息进行完全编码：位置和糖果库存。 这可以防止错误的修剪，因为两次访问具有不同糖果数量的同一单元格确实是不同的情况。 因此，当第一次遇到出口时，不能存在更短的有效路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]
    
    sr = sc = er = ec = -1
    
    for i in range(n):
        for j in range(m):
            if grid[i][j] == 'S':
                sr, sc = i, j
            elif grid[i][j] == 'E':
                er, ec = i, j
    
    # visited[r][c][k]
    visited = [[[False] * 9 for _ in range(m)] for _ in range(n)]
    
    q = deque()
    q.append((sr, sc, 0, 0))
    visited[sr][sc][0] = True
    
    directions = [(1,0), (-1,0), (0,1), (0,-1)]
    
    while q:
        r, c, k, dist = q.popleft()
        
        if grid[r][c] == 'E':
            print(dist)
            return
        
        for dr, dc in directions:
            nr, nc = r + dr, c + dc
            if not (0 <= nr < n and 0 <= nc < m):
                continue
            if grid[nr][nc] == 'P':
                continue
            
            nk = k
            cell = grid[nr][nc]
            
            if cell == 'C':
                nk += 1
            elif cell == 'J':
                if nk == 0:
                    continue
                nk -= 1
            
            if not visited[nr][nc][nk]:
                visited[nr][nc][nk] = True
                q.append((nr, nc, nk, dist + 1))
    
    print("SPOOKED!")

if __name__ == "__main__":
    solve()
```该实现直接反映了状态定义。 BFS 队列存储位置和糖果数量，这对于正确性至关重要。 访问的数组是三维的，以防止重新访问相同的状态。 处理单元格效果的顺序很重要：我们根据输入的单元格而不是剩下的单元格来计算下一个糖果数量。 

一个常见的错误是在不考虑糖果数量的情况下将单元格标记为已访问，这会破坏正确性，因为用更多糖果到达相同位置可以解锁未来`J`细胞。 

另一个微妙的点是检查`E`当从队列中弹出而不是推送时，这确保返回的距离对应于最短的有效路径。

 ## 边缘情况

 一种重要的边缘情况是当使用不同的糖果数量重新访问单元格时。 考虑这样一种情况：到 a 的最短几何路径`J`牢房无法提供足够的糖果，但稍微绕一段路就可以了。 该算法通过处理来处理这个问题`(r, c, k1)`和`(r, c, k2)`作为不同的状态，因此两条路径都错误地消除了另一条路径。 

退出时会出现另一种边缘情况`E`只有在耗尽所有糖果后才能到达。 BFS 正确地允许沿路径消耗糖果，并达到`E`和`k = 0`只要消费顺序合法，就有效。 

最后一个微妙的情况是，在遇到任何糖果之前收集了多个糖果。`J`。 BFS 确保糖果数量较高的状态与糖果数量较低的状态一起探索，因此算法自然会找到最大化可行性的配置，而不是强制执行任何贪婪的收集策略。
