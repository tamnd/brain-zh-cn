 ## 解决方案
 ## 问题理解

 我们得到一条从 0 到 n 的整数位置的线段。 几个学生站在 1 到 n−1 之间的整数点上，并且多个学生可以共享相同的位置。 每个学生在零时刻以相同的概率独立选择一个方向，要么朝向 0，要么朝向 n。 

一旦过程开始，每个学生每秒移动一步。 当两个学生相遇时，他们会反转方向，但由于两者都以相同的速度移动，如果我们只关心时间上的位置，这种相互作用相当于他们互相穿过。 对于退出来说，重要的是每个学生的有效行为就像他们继续沿着最初的方向前进，直到他们离开该部分。 

我们被要求，对于每个查询时间 t，计算在时间 t 之前没有学生离开线段端点的概率。 换句话说，每个学生必须在 t 秒后仍在间隔内。 答案保证为零或二分之一的幂，因此我们输出指数 m，使得概率等于 2^{-m}，如果事件不可能，则输出 -1。 

约束n和q最大可达100000，t可以大到1亿。 这立即排除了随着时间的推移或每个学生每个查询的重新计算的任何模拟，因为即使 O(nq) 也已经太慢了。 我们需要一个解决方案，在预处理后，每个查询都能在对数或恒定时间内得到答复。 

当学生如此接近出口以至于两个方向都会导致他们在时间 t 之前或在时间 t 离开时，就会出现微妙的边缘情况。 例如，如果学生位于位置 2 并且 t 为 10，那么无论方向如何，他们都会在时间 t 之前很久就退出，从而使概率为零。 一种仅检查一个方向或假设对称性的简单方法会错过这一点并错误地返回正概率。 

另一种失败模式来自于忽视多个学生可以占据同一位置。 由于他们的贡献独立成倍增加，因此按位置汇总至关重要； 渐进地迭代单个学生仍然可以，但在概念上是多余的。 

## 方法

 如果我们尝试直接模拟，我们将为每个学生分配一个随机方向，然后在处理碰撞时模拟 t 步的移动。 即使我们利用冲突的等价性来交换身份，我们仍然需要跟踪每个学生的退出时间并对每个查询重复此推理。 如果显式模拟碰撞，这会导致 O(nq) 检查或更糟，这远远超出了限制。 

关键的简化是碰撞根本不影响退出时间。 每个学生的退出时间仅取决于他们的初始位置和选择的方向。 位置 x 的学生如果面向左，则在 x 秒后从左退出；如果面向右，则在 n−x 秒后从右退出。 因此，对于固定的查询时间 t，每个学生独立地贡献一个约束：两个方向中至少有一个必须将它们保持在段内，直到时间 t。 

这种独立性将概率转化为头寸的乘积。 对于每个学生来说，都有三种情况。 如果两个方向都将它们保持在内部，则学生贡献的概率为 1。如果只有一个方向是安全的，则它们贡献的概率为 1/2。 如果没有安全的方向，整个配置就变得不可能。 

由于许多学生共享职位，我们按职位汇总计数。 对于固定 t，每个位置 x 根据 t、x 和 n−x 之间的比较分为四个类别之一。 这允许我们使用位置频率上的前缀和来计算最终指数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 直接模拟运动 | O(nq) 或更糟 | O(n) | 太慢了 |
 | 预计算频率+范围计数| O(n + q) | O(n) | 已接受 |

## 算法演练

 1. 计算从 1 到 n−1 的每个位置有多少学生。 这将重复的位置压缩为频率，因为同一位置的所有学生表现相同。 
2. 对这些频率建立一个前缀和数组，以便我们可以快速查询有多少学生位于任意位置区间。 
3.对于给定的查询时间t，根据x是否大于t以及n−x是否大于t对位置x进行分类。 这两个比较决定了向左或向右移动是否安全。 
4. 计算有多少个位置属于以下区域：两个方向都是安全的、只有一个方向安全的位置以及两个方向都不安全的位置。 这是使用频率数组上的区间和来完成的。 
5. 如果任何位置属于两个方向都不安全的区域，那么答案立即是不可能的，因为至少有一个学生必须退出。 
6. 否则，每个位置要么对指数没有贡献，要么在只有一个方向有效时贡献 1。 将所有位置的这些贡献相加形成 m。 
7. 输出查询的m。 

它之所以有效，是因为每个学生的方向选择都是独立的，并且独立于其他学生决定他们的退出时间。 由于碰撞不会改变退出时间，因此全局事件分解为独立的每个学生约束。 每个学生要么有 0、1 要么 2 个有效方向选择，并且只有只有一个有效选择的情况才对总概率贡献 1/2。 按位置聚合可以保留这种独立性，因此对每个类别中的学生进行计数可以完全确定指数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    arr = list(map(int, input().split()))

    freq = [0] * (n + 1)
    for x in arr:
        freq[x] += 1

    pref = [0] * (n + 1)
    for i in range(1, n + 1):
        pref[i] = pref[i - 1] + freq[i]

    def range_sum(l, r):
        if l > r:
            return 0
        l = max(l, 1)
        r = min(r, n - 1)
        if l > r:
            return 0
        return pref[r] - pref[l - 1]

    for _ in range(q):
        t = int(input())

        # both safe: x > t and x < n - t
        both_l = t + 1
        both_r = n - t - 1
        both = range_sum(both_l, both_r)

        # left only safe: x > t and x >= n - t
        lo_l = max(t + 1, n - t)
        lo_r = n - 1
        left_only = range_sum(lo_l, lo_r)

        # right only safe: x <= t and x <= n - t - 1
        ro_l = 1
        ro_r = min(t, n - t - 1)
        right_only = range_sum(ro_l, ro_r)

        # none safe: x <= t and x >= n - t
        none_l = max(1, n - t)
        none_r = min(t, n - 1)
        none = range_sum(none_l, none_r)

        if none > 0:
            print(-1)
        else:
            print(left_only + right_only)

if __name__ == "__main__":
    solve()
```该解决方案首先将学生位置压缩到频率数组中，以便集中处理重复的位置。 该数组上的前缀和可实现恒定时间范围查询。 

对于每个查询时间 t，我们推导出对应于四个行为类别的间隔。 区间边界直接来自 x 与 t 和 n−x 的比较，将条件“向左移动则生存”或“向右移动则生存”转化为 x 上的不等式。 

关键部分是“非安全”区域的检测。 如果任何位置位于该区间内，则无论方向如何，该位置的每个学生都不可避免地退出，将概率强制为零。 否则，只有具有一个有效方向的位置才会对指数产生影响。 

## 工作示例

 ### 跟踪示例

 我们展示了一个 n = 7 且学生位于位置 [2, 2, 3] 的小型配置。 

我们认为 t = 1。 

| 位置 x | 计数| x > t | n−x > t | n−x > t | 类别 |
 | --- | --- | --- | --- | --- |
 | 2 | 2 | 是的 | 是的 | 都安全|
 | 3 | 1 | 是的 | 是的 | 都安全|

 这里两个位置都位于“均安全”区域，因此没有位置对指数有贡献。 

结果是 m = 0。 

这表明，当所有学生相对于 t 都距离两个出口足够远时，随机性不会降低概率，因为每个方向都是安全的。 

### 第二个示例跟踪

 学生处于位置 [1, 5]，取 n = 7。 

令 t = 2。 

| 位置 x | 计数| x ≤ t | n−x ≤ t | 类别 |
 | --- | --- | --- | --- | --- |
 | 1 | 1 | 是的 | 没有| 只对 |
 | 5 | 1 | 没有| 是的 | 只剩下 |

 两个位置恰好贡献一个有效方向，因此 m = 2。 

这证实了独立约束在指数中累加。

 ## 边缘情况

 当t很大时，每个位置都可能落入“非安全”区域。 例如，如果 t ≥ n−1，则位置 1 的任何学生如果面向左都无法避免退出，而任何靠近 n−1 的学生如果面向右都无法避免退出。 在这种情况下，算法将所有位置放入非安全区间并立即输出 -1。 

当所有学生共享一个位置时，分类仍然有效，因为频率聚合不假设独特性。 该算法对该位置进行一次评估，并将其计数正确地乘以指数。 

当 t 很小时，许多头寸都会落入“均安全”区域。 这些对指数没有任何贡献，并且前缀和查询自然返回零贡献，无需特殊处理即可保持正确性。
