 ## 解决方案
 ## 问题理解

 我们得到一个字符序列，仅包含`0`,`1`， 和`2`。 我们想要选择这个序列的一个连续的段，使得在该段内，`2`字符数不超过给定限制`k`。 在所有这些有效段中，我们想要最大可能的长度。 

换句话说，我们在字符串上滑动一个窗口，并查找最长的长度，其中数字的计数`2`保持在允许的预算范围内。 

输入大小可达$10^5$，这会立即排除任何显式尝试所有子字符串的解决方案。 暴力方法会考虑$O(n^2)$子串，甚至计算子串的数量`2`每个里面的 s 都会将其推向$O(n^3)$或者$O(n^2)$，对于 1 秒的限制来说太慢了。 因此，我们需要一种线性或接近线性的方法，通常是一次性处理字符串的方法。 

朴素推理的主要微妙失败案例是忘记最佳片段可能包括有效窗口的多个延伸，并且从错误的一侧收缩可能会失去最佳答案。 例如，如果`k = 1`字符串是`220202`，一个幼稚的方法可能会贪婪地跳过太多位置并错过在早期删除后开始的较长有效窗口`2`s。 

## 方法

 蛮力策略很简单：枚举每个可能的子字符串，计算有多少个`2`s 它包含，并跟踪计数最多的那些中的最大长度`k`。 对于每个起始索引，我们扩展结束指针并维护一个计数器。 这导致大约$n$起始位置，对于每个我们可以扫描到$n$字符，导致$O(n^2)$时间。 这已经太大了$n = 10^5$，因为它需要的顺序是$10^{10}$运营。 

问题的结构提出了一种更有效的方法，因为我们强制执行的条件是单调的：一旦某个段因超过`k`的出现次数`2`，进一步扩展只会让情况变得更糟。 这正是滑动窗口发挥作用的设置。 我们维护一个窗口，而不是重新计算每个子字符串的计数`[l, r]`并跟踪有多少`2`s 在里面。 我们扩展`r`一步一步，每当窗口违反约束时，我们就移动`l`转发直至其再次有效。 

这将问题转化为单个线性扫描，其中每个字符最多进入和离开窗口一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 滑动窗 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们维护一个窗口`[l, r]`在绳子和柜台上`cnt2`存储多少个`2`s 当前位于窗口内。 

1. 初始化`l = 0`,`cnt2 = 0`， 和`best = 0`。 我们从一个空窗口开始，逐渐构建有效的段。 
2. 迭代`r`从`0`到`n - 1`。 每次我们包括`s[r]`进入当前窗口。 
3.如果`s[r] == '2'`, 增量`cnt2`。 这跟踪我们是否消耗了有限预算的一部分。 
4. 如果`cnt2 > k`，窗口无效。 我们必须从左侧缩小它，直到它再次有效。 我们通过移动来做到这一点`l`向前并减去`cnt2`每当我们删除一个`2`。 
5. 恢复有效性后，当前窗口`[l, r]`满足约束条件，所以我们更新`best = max(best, r - l + 1)`。 
6. 继续直到`r`到达字符串末尾。 答案是`best`。 

关键的想法是`l`只向前移动，从不向后移动，这确保了线性行为。 

### 为什么它有效

 在每个位置`r`，该算法保持了窗口的不变性`[l, r]`最多包含`k`的出现次数`2`，还有那个`l`是使当前情况成立的最小索引`r`。 因为任何较大的窗口结束于`r`只会增加或维持数量`2`s，任何无效的扩展必须通过移动来纠正`l`正确的。 这保证了没有有效的候选窗口结束于`r`被错过，因为所有较小的有效前缀都被隐式地视为`l`进步。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = input().strip()

    l = 0
    cnt2 = 0
    best = 0

    for r in range(n):
        if s[r] == '2':
            cnt2 += 1

        while cnt2 > k:
            if s[l] == '2':
                cnt2 -= 1
            l += 1

        best = max(best, r - l + 1)

    print(best)

if __name__ == "__main__":
    solve()
```该实现直接反映了滑动窗口逻辑。 唯一重要的细节是收缩循环必须在更新答案之前完全恢复有效性，否则我们可能会计算无效段。 指针`l`永远不会重置，这保留了线性复杂性。

 ## 边缘情况

 一个关键的边缘情况是当`k = 0`，意思是没有`2`是完全允许的。 对于像这样的输入`10202`，每当`2`遇到。 在`r = 2`，我们击中了第一个`2`, 强迫`l`搬到`2`，然后窗口重置。 这表明该算法不会累积无效状态，因为`cnt2`立即得到纠正。 

另一种情况是当字符串包含小于或等于`k`的出现次数`2`总共，例如`s = 1112011`和`k = 2`。 在这种情况下，`cnt2`永远不会超过`k`， 所以`l`从不移动。 该算法有效地返回`n`，因为整个字符串都是有效的。
