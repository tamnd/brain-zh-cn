 ## 解决方案
 ## 问题理解

 我们得到一排从左到右索引的建筑物，其中每个建筑物包含一定数量的学生。 弗雷迪总是从第一栋建筑开始，只能按顺序向前走。 对于每个查询值，我们想知道学生总数至少为该查询值的建筑物的最小前缀。 如果即使在访问了所有建筑物之后，总数仍然小于查询，则答案是不可能的。 

这从根本上来说是一个前缀积累问题。 每个查询都要求前缀和达到或超过阈值的第一个位置。 

这些约束允许最多 100,000 座建筑物和 100,000 条查询。 每个查询的直接模拟每次都会从头开始扫描，在最坏的情况下会导致 10^10 次操作，这在 Python 中太慢了。 这立即排除了任何重复重新计算总和的解决方案。 

当构建值包含零时，就会出现边缘情况。 如果许多连续的建筑物没有做出任何贡献，假设每个建筑物总是取得进展的天真的指针方法可能会失败。 例如，如果所有 a_i 均为零且 q_j 为正，则正确答案为 -1，并且任何依赖增量进度的方法都必须根据总和显式检查可达性。 

另一个微妙的情况是 q_j 为零时。 正确答案始终为 1，因为零个学生不需要参观任何有意义的建筑物，但第一座建筑物仍然是最小前缀。 

## 方法

 暴力方法独立处理每个查询。 对于给定的 q，我们从第一栋楼开始并累积学生人数，直到达到或超过 q。 这是正确的，因为它直接遵循问题定义。 然而，在最坏的情况下，每个查询可能需要扫描所有n个建筑物。 对于 m 个查询，这将变为 O(nm)，即最多 10^10 次操作。 

关键的观察是所有查询都在相同的静态前缀结构上运行。 一旦我们计算了数组的前缀和，每个查询就简化为一个搜索问题：找到最小的索引 i，使得 prefix[i] ≥ q。 由于前缀和是非递减的，因此我们可以对每个查询使用二分搜索。 这将每个查询减少到 O(log n)，使总数达到 O((n + m) log n)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(纳米) | O(1) | O(1) | 太慢了|
 | 前缀+二分查找| O((n + m) log n) | O((n + m) log n) | O(n) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 计算建筑物数组上的前缀总和，其中 prefix[i] 存储从建筑物 1 到 i 的学生总数。 这将问题转化为范围可达性问题，而不是重复求和。 
2. 对于每个查询q，检查q是否大于总和prefix[n]。 如果是，则立即输出 -1，因为即使访问所有建筑物也是不够的。 
3. 如果 q 为零，则输出 1，因为最小的有效前缀是第一个建筑物。 这处理了不需要累积的退化情况。 
4. 否则，对前缀数组执行二分查找以找到最小索引 i，使得 prefix[i] ≥ q。 这是可行的，因为前缀和是单调非递减的。 
5. 输出每个查询的 i。 

### 为什么它有效

 前缀和数组对累积可达性进行编码：前缀 [i] 准确地表示通过停止构建 i 可以实现的目标。 由于当我们向右移动时，值永远不会减少，因此条件 prefix[i] ≥ q 定义了有效答案的连续区域。 二分搜索利用这种单调结构来定位第一个有效索引。 该算法不会错过有效答案，因为一旦 prefix[i] 超过 q，所有后面的索引也满足它，确保第一个找到的边界的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    q = list(map(int, input().split()))

    prefix = [0] * n
    prefix[0] = a[0]
    for i in range(1, n):
        prefix[i] = prefix[i - 1] + a[i]

    total = prefix[-1]
    out = []

    for x in q:
        if x == 0:
            out.append("1")
            continue
        if x > total:
            out.append("-1")
            continue

        lo, hi = 0, n - 1
        ans = n - 1

        while lo <= hi:
            mid = (lo + hi) // 2
            if prefix[mid] >= x:
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1

        out.append(str(ans + 1))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个前缀和数组，以便每个查询都成为对累积总数的搜索，而不是重复求和。 二分搜索是手动实现的，以避免开销并显式控制满足前缀约束的第一个位置。 转换`ans + 1`是必需的，因为数组内部是零索引的，但问题使用一索引的构建位置。 

特殊处理为`x == 0`防止不必要的二分搜索并确保第一个前缀始终有效的边界条件的正确性。

 ## 边缘情况

 一种重要的边缘情况是所有建筑物价值为零。 在这种情况下，前缀数组永远不会增加，因此任何正查询必须立即返回 -1。 该算法处理此问题是因为二分搜索条件 prefix[mid] ≥ x 永远不会满足，并且总和检查会提前拒绝查询。 

另一个边缘情况是当第一座建筑物已经满足查询时。 例如，a = [100, ...], q = 50。前缀数组从一个大值开始，因此二分查找正确返回索引 0，它转换为输出 1。正确性取决于搜索最左边的有效索引，而不仅仅是任何有效索引。
