 ## 解决方案
 ## 方法

 暴力方法正是问题所暗示的：对于每个联系人，将电话号码转换为数字，计算它们的总和，并检查奇偶性。 这已经是最佳选择，因为每个数字都必须至少检查一次才能知道其对总和的贡献。 成本与所有联系人的总位数成正比，最多约为 9000，因此它会立即运行。 

不需要更深层次的组合结构或预处理技巧。 唯一的“优化”是避免将字符串转换为整数，而是直接从字符代码累加总和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力数字扫描| O(总位数) | O(1) 额外 | 已接受 |
 | 任何高级预处理 | O(总位数) | O(1) | O(1) | 矫枉过正|

 ## 算法演练

 1. 读取联系人数量，然后逐行处理。 流式传输特性避免了存储不必要的数据。 
2. 将每一行拆分为姓名和电话号码字符串。 每行进行一次分割，因此解析保持线性。 
3. 对于电话号码中的每个字符，将其转换为数值并累加总和。 此步骤是必要的，因为决策取决于全数字和的奇偶校验，而不是部分结构。 
4. 计算总和后，检查是否为偶数。 如果是偶数，则立即输出关联名称； 否则丢弃它。 
5. 继续，直到处理完所有接触，自动保留输入顺序，因为输出是按相同的遍历顺序生成的。 

### 为什么它有效

 该算法依赖于以下不变量：在处理第 i 个联系人后，我们已正确计算出其电话号码的精确数字和。 由于每个数字对总和都有独立贡献，并且奇偶校验仅取决于总和模 2，因此不能跳过或近似任何数字。 因此，“是否打印姓名”的决定对于每个联系人来说都是本地正确的，并且不依赖于其他条目。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    for _ in range(n):
        line = input().strip().split()
        name = line[0]
        number = line[1]

        s = 0
        for ch in number:
            s += ord(ch) - ord('0')

        if s % 2 == 0:
            print(name)

if __name__ == "__main__":
    solve()
```该解决方案一次性处理输入。 每行都被恰好分成两个标记，因此即使名称是字母数字，解析时也不会出现歧义。 

数字和是使用直接字符算术计算的，这避免了整数转换开销并保留前导零的正确性。 立即执行奇偶校验，并以流式方式输出，从而保持内存使用量恒定。

 ## 边缘情况

 一种重要的边缘情况是电话号码仅由零组成，例如：```
1
Zero 000000
```该算法计算出的总和为 0，为偶数，因此打印出名称。 这里的一个错误是首先将字符串转换为整数，这仍然可以在数字上工作，但存在不必要的解析复杂性和其他变体中潜在溢出的风险。 

另一个边缘情况是个位数，例如：```
1
Odd 7
```总和是 7，是奇数，所以没有打印任何内容。 该算法可以正确处理此问题，因为它不假设任何最小长度。 

最后，考虑混合前导零：```
1
Lead 00012
```总和为 1+2 = 3，为奇数，因此被排除。 前导零没有任何贡献，但仍然必须对其进行迭代； 跳过它们会改变正确性。
