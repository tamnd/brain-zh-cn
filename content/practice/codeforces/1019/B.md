 ## 解决方案
 ## 问题理解

 我们给定一个圆$n$学生，哪里$n$是均匀的。 每个学生都持有一个未知的整数，关于这些值的唯一结构保证是局部的：圆上的任何两个邻居的值正好相差一。 这迫使整个配置的行为就像在整数线上“行走”一样，每一步向上或向下移动 1，并最终关闭循环。 

这种互动使我们能够通过查询职位来揭示个人价值观。 我们的任务不是重建整个数组，而是确定圈中是否存在至少一对坐在正对面且值相等的学生。 如果存在这样的对，我们必须输出属于这样的对的任何索引； 否则我们输出-1。 

因为我们只能查询值而看不到数组，所以困难在于在最多使用 60 个查询的情况下决定如何定位相对位置之间的碰撞，即使$n$可以大到$10^5$。 

约束条件$n \le 10^5$立即排除任何探测每个位置甚至大部分位置的方法。 完整扫描需要$O(n)$查询，这远远超出了 60 的限制。因此，解决方案必须战略性地对位置进行采样，并利用相邻差异总是存在的结构约束$\pm 1$。 

如果尝试随机抽样或简单的配对检查，就会出现一个微妙的问题。 例如，如果值像这样振荡$1,2,1,2,1,2,\dots$，相反的对可能全部不同，并且随机采样很容易完全错过结构。 另一个陷阱是假设相同的值必须频繁出现。 事实上，该结构可以有很少的重复，可能只有我们正在寻找的相反的一对。 

缺少的关键见解是，由于数组的行为就像受约束的行走，因此了解精心选择的对称位置处的值使我们能够以受控方式“传播”比较，从而缩小必须发生矛盾或匹配的范围。 

## 方法

 暴力策略是查询每个学生，存储所有值，然后检查所有值$n/2$相反的对。 这是正确的，但需要$n$查询，这立即违反了限制。 即使部分扫描也无济于事，因为没有单调结构保证重复发生的位置。 

关键的观察是圆和约束$|a_i - a_{i+1}| = 1$意味着一种非常严格的结构：一旦我们知道一个值，所有其他值都将取决于方向选择的全局转变。 这意味着沿着圆的差异可以预见地累积，相反的点通过一条长度的路径连接起来$n/2$完全由±1步组成。 

如果我们比较索引处的值$i$和$i + n/2$，它们的差异仅取决于 ±1 跃迁段的净和。 这个总和的行为类似于随机游走，其大小范围为$n/2$，我们想要检测它是否曾经达到零。 

我们不是重建所有值，而是对对数数量的锚点进行采样，并使用二分搜索风格推理在两个方向上局部扩展：如果我们检测到某个位置的值高于其相反的候选值，我们就知道步行必须如何漂移穿过圆，这让我们在每一步消除一半的搜索空间。 

核心简化是，可以通过在圆周围的单调分段差分函数中搜索零来检查是否存在相等的相反对。 这允许对仅具有以下位置的位置进行二分搜索$O(\log n)$每次检查的查询次数，最多是恒定数量的检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n)$查询 |$O(n)$| 太慢了|
 | 最优（对圆进行二分搜索）|$O(\log n)$查询 |$O(1)$| 已接受 |

 ## 算法演练

 我们利用相反位置的对称性。 对于任意索引$i$，定义其相反数为$i + n/2$（模数$n$）。 我们试图找到一个索引，其中$a_i = a_{i+n/2}$。 

1. 选择任意起始索引，通常$1$，并查询。 这为我们提供了锚定所有比较的参考值。 
2. 对于候选索引$i$，我们查询两者$i$和$i + n/2$。 这给了我们相反点之间的差异。 
3. 我们定义索引的搜索范围$i \in [1, n/2]$，因为每对相反的位置在此范围内唯一表示。 
4. 我们在此范围内执行二分搜索。 在中点$m$，我们查询两者$m$和$m + n/2$。 如果它们相等，我们立即返回$m$，因为我们找到了一个有效的对。 
5. 如果它们不相等，我们使用差值的符号来决定保留哪一半搜索空间。 ±1 邻接约束保证相反对之间的差异以受控方式变化，因此一旦我们越过过渡点，不平衡方向保持一致。 
6. 继续缩小范围，直到找到有效索引或用尽区间。 
7. 如果所有查询后都没有发现相等的对，则输出-1。 

关键思想是相反值之间的差异表现为循环索引上的分段单调函数，允许二分搜索隔离零（如果存在）。 

### 为什么它有效

 邻接约束迫使整个序列成为整数上的离散游走。 当投影到相反的对上时，函数$f(i) = a_i - a_{i+n/2}$因为转移而逐渐改变$i$by 1 以相关方式将两项最多更改 1。 这确保了$f(i)$不能任意摆动； 它以有限的增量发展。 因此，如果存在零，它必须出现在二分搜索可以隔离而不会丢失的连续区域中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(i):
    print("?", i)
    sys.stdout.flush()
    return int(input().strip())

def solve():
    n = int(input().strip())
    
    def get(i):
        return ask(i)

    l, r = 1, n // 2
    a_l = get(l)
    a_l_op = get(l + n // 2)
    if a_l == a_l_op:
        print("!", l)
        sys.stdout.flush()
        return

    a_r = get(r)
    a_r_op = get(r + n // 2)
    if a_r == a_r_op:
        print("!", r)
        sys.stdout.flush()
        return

    while l + 1 < r:
        m = (l + r) // 2
        a_m = get(m)
        a_m_op = get(m + n // 2)

        if a_m == a_m_op:
            print("!", m)
            sys.stdout.flush()
            return

        # decide direction using consistency with left endpoint
        if (a_m - a_m_op) * (a_l - a_l_op) > 0:
            l, a_l, a_l_op = m, a_m, a_m_op
        else:
            r = m

    print("! -1")
    sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该实现在搜索空间中维护两个边界点，每个边界点都存储一个位置及其相反值。 在每一步中，我们都会查询中点并将其相反的差异与左边界进行比较，以确定哪一侧仍然包含潜在的零交叉。 

一个微妙的细节是，我们从不假设值本身的单调性​​，而只假设相反差分函数的符号的单调性。 这就是二分搜索在这里有效的原因。 

所有查询都会立即刷新，因为每次输出后交互都需要严格同步。 

## 工作示例

 考虑一个小的概念示例，其中$n = 8$值为：$$[1, 2, 1, 2, 3, 4, 3, 2]$$相反对是 (1,5)、(2,6)、(3,7)、(4,8)。 该算法查询位置 1 和 4。

 | 中| 2 | 4 | 2 | 2 | 4 | 向右移动|
 | 中| 3 | 4 | 3 | 1 | 3 | 向右移动|

 这演示了如何缩小搜索范围，直到发现相等的相反对。 

现在考虑一个无解的情况：$$[1,2,3,4,5,6,7,8]$$相反对是 (1,5)、(2,6)、(3,7)、(4,8)，均不相等。 二分搜索仍然评估差异，但从未遇到零，最终耗尽搜索空间并返回 -1。 

这表明根的缺失也得到了正确处理，没有误报。

 ## 边缘情况

 当$n = 2$，圆退化为一对相反的圆。 该算法立即查询索引 1 及其相反的 2，因此正确性并不重要。 

如果值像这样振荡$1,2,1,2,\dots$，相反的对可能都不同，但基于符号的搜索仍然一致地划分空间并返回 -1，而不需要完全探索。 

当正确的对位于搜索两半之间的边界附近时，二分搜索仍然保留它，因为在修剪之前会在每个中点检查相等性，以确保不会跳过任何有效的解决方案。 

在紧密变化的序列中，例如$1,2,3,2,1,0$，相反差值的函数恰好过零一次，并且算法的减半策略可靠地隔离了该交叉点。
