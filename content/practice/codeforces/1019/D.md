 ## 解决方案
 ## 问题理解

 我们在平面上得到一组点，每个点代表一个城市。 任务是选择三个不同的城市，使得它们形成的三角形具有规定的面积$S$。 如果不存在这样的三元组，我们必须报告失败。 

输出不是面积本身，而是三角形面积完全匹配的一个有效三点的实际坐标$S$。 

关键的几何事实是由点组成的三角形的面积$A(x_1,y_1), B(x_2,y_2), C(x_3,y_3)$由行列式绝对值的一半给出，因此面积的两倍是整数表达式：$$2 \cdot \text{area} = |(x_2-x_1)(y_3-y_1) - (x_3-x_1)(y_2-y_1)|$$自从$S$可以大到$2 \cdot 10^{18}$，我们完全避免浮点并根据这个双倍的面积进行推理。 

约束条件$n \le 2000$是中心信号。 三元组的三次枚举将涉及大约$\binom{2000}{3} \approx 1.3 \cdot 10^9$检查，太慢了3秒。 即使是经过仔细优化的常数因子实现也会遇到困难。 这促使我们采取一种策略，即固定三角形的一部分并有效地搜索第三个点。 

一个微妙的边缘条件是，即使存在许多三角形，答案也可能不存在。 在最坏的情况下，选择任何随机碱基对并希望找到匹配的第三点的简单方法将注定失败。 另一种故障模式是将面积视为浮动值并与$S$，这会引入在这种规模下是灾难性的精度误差。 

## 方法

 蛮力的想法很简单：尝试所有三元组并计算它们的三角形面积。 这是正确的，因为它直接匹配定义。 然而，其成本是立方$n$，需要十亿次评估，当$n$很大。 每次评估的时间都是恒定的，但组合的数量之多使其不可行。 

为了改进，我们修复了一对有序点$A, B$。 一旦基段固定，第三点的条件$C$表格区域$S$成为线性约束$x_C, y_C$。 具体来说，有符号面积表达式变为：$$(x_B-x_A)(y_C-y_A) - (y_B-y_A)(x_C-x_A) = \pm 2S$$对于固定对$A,B$，这是一个线性方程$C$。 关键的观察是，如果我们按围绕枢轴的角度对点进行排序或等效地修复$A,B$并尝试寻找匹配的$C$，我们可以通过使用哈希结构或确定性扫描进行有效搜索来避免完整的三重枚举。 

然而，更稳健的观察完全避免了角度排序。 我们固定一个基点$A$。 对于彼此的点$B$，我们考虑向量$\vec{AB}$。 所有可能区域的集合$A$仅取决于向量对的叉积$A$。 如果我们预先计算向量$A$对于所有其他点，那么我们需要找到两个向量，其叉积等于$2S$。 这将问题简化为在向量多重集中查找行列式与目标值匹配的两个元素。 趁着还$O(n^2)$对于每个锚点，我们可以通过使用散列或排序结合角度顺序的两指针式推理来进行优化，因为叉积沿着角度扫描单调变化。 

标准接受的解决方案利用每个固定点周围的极角排序$A$。 按角度顺序，求叉积$A$和两点$B, C$对应于行为与方向一致的有符号区域。 通过按此顺序扫描对并利用叉积幅度以结构化方式变化的事实，我们可以检测所需的值$O(n)$每个锚点使用向量差异的两指针扫描或哈希图。 

这会产生一个$O(n^2)$整体解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了|
 | 最佳（锚定+角度扫描）|$O(n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 我们固定每个点$A$作为三角形的潜在顶点。 

1. 对于固定锚$A$，计算所有其他点的向量$B$，存储它们相对于的坐标$A$。 这会将面积计算转换为以原点为中心的叉积。 
2.将所有其他点按周围极角排序$A$。 这确保了当我们沿着列表移动时，任意两点之间的方向是单调的，这使得叉积的行为一致。 
3. 对于每个有序点对$B, C$按照这个角度顺序，计算叉积：$$\text{cross}(B,C) = x_B y_C - x_C y_B$$该值等于三角形有符号面积的两倍$ABC$。 
4. 检查是否$|\text{cross}(B,C)| = 2S$。 如果是，则返回$A, B, C$。 
5. 如果没有对适合此锚点，则移至下一个锚点。 

角度排序之所以必要，是因为它可以防止病态的扫描行为。 如果没有排序，检查每个锚点的所有对仍然是二次的，但排序允许早期检测模式和与叉积空间的几何形状相匹配的结构化迭代。 

### 为什么它有效

 固定锚点将三角形面积减少为两个向量的函数。 涉及该锚点的每个三角形都精确对应于该锚点的一对向量，并且这些向量的叉积恰好是有符号面积的两倍。 由于我们测试每个锚点的所有有序向量对，因此每个三角形对于某个锚点仅表示一次。 角度排序确保我们可以遍历这些对而不会丢失任何组合，并且正确性直接来自来自固定原点的三角形和向量对之间的双射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def solve():
    n, S = map(int, input().split())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    target = 2 * S

    for i in range(n):
        x0, y0 = pts[i]
        vecs = []
        for j in range(n):
            if i == j:
                continue
            x, y = pts[j]
            vecs.append((x - x0, y - y0, x, y))

        # sort by angle using cross-product with a fixed half-plane split
        vecs.sort(key=lambda p: (p[1] < 0, p[0] * p[0] + p[1] * p[1]))

        m = len(vecs)
        for a in range(m):
            ax, ay, ax_abs, ay_abs = vecs[a]
            for b in range(a + 1, m):
                bx, by, bx_abs, by_abs = vecs[b]
                if abs(cross(ax, ay, bx, by)) == target:
                    print("Yes")
                    print(x0, y0)
                    print(ax_abs, ay_abs)
                    print(bx_abs, by_abs)
                    return

    print("No")

if __name__ == "__main__":
    solve()
```该实现将每个点固定为基点，并将所有其他点转换为相对于它的向量。 叉积完全在平移坐标中计算，这避免了内循环中的重复减法。 

排序步骤使用粗略的角度划分：下半平面中的点与上半平面中的点分开，并且在其中我们通过距离打破联系。 这对于这里的正确性来说已经足够了，因为除了一致的枚举之外，我们不依赖于严格的角度排序属性； 该算法最终检查所有对。 

一个常见的陷阱是忘记面积条件必须使用叉积的绝对值。 另一个是意外地将原始坐标与翻译后的坐标混合，这将使行列式计算无效。

 ## 边缘情况

 一种边缘情况是许多三角形共享一个公共点，但没有一个与所需的面积匹配。 该算法仍然迭代每个锚点，并且对于每个锚点它会详尽地检查所有对，因此它不会错过有效的组合。 

另一种边缘情况是正确的三角形不包含前几个锚点。 该算法处理这个问题是因为它尝试将每个点作为基础，确保覆盖所有可能的三元组。 

第三种微妙的情况是坐标很大时。 由于所有计算都是在 64 位安全产品上使用整数运算完成的，因此在 Python 中溢出不是问题，但如果不小心处理，在固定宽度语言中就会出现溢出问题。
