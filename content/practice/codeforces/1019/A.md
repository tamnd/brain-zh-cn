 ## 解决方案
 ## 问题理解

 选举有一组固定的选民，每个选民最初都致力于几个政党中的一个。 联合党是第一党，其目标不仅仅是获得大量选票，而是在最终计票中严格超越其他所有政党。 

每个选民都会收到两条信息：他们目前支持哪个政党，以及说服他们将投票转向第一党需要多少钱。一旦被贿赂，选民就会停止支持原来的政党，转而投票给联合党。 

任务是确定所需的最低总贿赂成本，以便在说服部分选民改变立场后，第一方最终获得比任何其他政党更多的选票。 

最多 3000 个选民和 3000 个政党的输入大小限制表明二次解是可以接受的，而任何三次解或涉及对大州进行重复完全重新计算的速度都太慢。 试图模拟所有选民子集的解决方案是立即不可能的，因为状态空间是指数的。 

一个天真的但有启发性的方法是尝试对所有可能的选民子集进行贿赂并检查第一方是否获胜。 这会失败，因为有 2^n 个子集，即使 n = 3000，这也远远超出了任何可行的计算。 

第二个天真的想法是尝试为第一方尝试所有可能的最终投票计数并贪婪地调整其他方。 这更接近解决方案，但仍然需要小心：在不尊重政党平衡的情况下简单地贿赂全球最便宜的选民可能会破坏正确性。 

当对手方已经非常庞大时，就会出现微妙的失败案例。 例如，如果第 2 党有 1000 票，而所有其他党有 1 票，盲目贿赂全球最便宜的选民可能会减少小党而不是大党，使第 2 党仍然占主导地位，即使相同的预算可以首先抵消它。 

关键的困难在于，贿赂选民会同时产生两种效应：减少他们原来的政党并增加第一政党。任何正确的策略都必须同时考虑这两种效应。 

## 方法

 暴力基线是考虑要贿赂的每个选民子集，计算所有各方的最终投票计数，并检查第一方是否严格最大。 这是正确的，因为它枚举了所有可能的结果，但它需要迭代大小最大为 n 的所有子集，这会导致大约 O(2^n · n) 次操作。 即使 n = 40 这也是不可行的，更不用说 3000 了。 

一种更结构化的暴力破解方法是猜测第一方最终获得的最终票数 x。 对于每个 x，我们尝试强制所有其他方最多拥有 x − 1 票。 这已经引入了结构：我们现在只在最终计数的约束下决定贿赂哪些选民，而不是任意子集。 

关键的观察是，我们从另一党贿赂的每个选民都会减少该党的人数并增加第一党的人数。 因此，每一个受贿的选民都可以同时实现两个目标：降低竞争对手和提高我们的分数。 这意味着我们应该始终考虑选择一组选民将其从原来的政党中剔除。 

对于固定目标 x，如果 cj ≥ x，则当前计数 cj 的各方 j > 1 必须减少。 在这种情况下，我们被迫贿赂至少 cj − (x − 1) 个该政党的选民。 为了最大限度地降低成本，我们总是首先选择各党内最便宜的选民。 

在执行这些强制减少之后，第一方已经获得了一定数量的选票，数量等于我们贿赂的选民数量。 如果这仍然不足以达到 x，我们还可以从任何剩余的池中贿赂额外的选民，总是选择最便宜的可用选民。 

这将每个候选 x 转化为具有排序和前缀和的贪婪选择问题。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 使用贪婪选择尝试所有目标 x | O(n^2 log n) | O(n^2 log n) | O(n) | 已接受 |

 ## 算法演练

 1. 统计各政党的初始选票，并收集每个非 1 政党的所有选民，存储其贿赂成本。 这给出了可以减少哪些地方的结构化视图。 
2. 对于 j > 1 的每个政党，按成本升序排列其选民。 这确保了如果我们被迫减少该政党，我们总是首先选择最便宜的选民。 
3. 计算第 1 方的初始票数。这是我们尝试改进到某个目标 x 的基线。 
4. 迭代从初始党 1 投票到 n 的所有可能的目标值 x。 对于每个 x，我们评估是否有可能使政党 1 达到 x 票数，同时保持所有其他政党低于 x。 
5. 对于固定的 x，确定每个政党 j > 1 必须贿赂多少选民，使其剩余票数最多变为 x − 1。所需金额为 max(0, cj − (x − 1))。 
6. 对于每个政党，选取那么多最便宜的选民并将他们标记为已选中。 这些是强制性的移动，因为没有它们，目标配置是不可能的。 
7. 追踪到目前为止有多少选民被贿赂； 这直接增加了第一方的投票数。 
8. 如果强制选择后党 1 的票数仍然少于 x 票，则从所有剩余的未选择选民中获取额外的选民，始终首先选择最便宜的选民，直到达到 x 或耗尽候选人。 
9. 计算该 x 的所有选定投票者的总成本并更新全局最小值。 

### 为什么它有效

 对于固定目标x，任何有效的解决方案都必须将每个过大的对手减少到x以下。 在每一方内部，选择更昂贵的选民而不是更便宜的选民只会增加总成本，而不会提高可行性，因此每一方的贪婪选择是被迫的。 

在满足所有强制约束后，所有剩余选民在可行性方面都是可互换的，唯一的目标是增加第一方的数量，通过选择最便宜的剩余选民来最大化每成本。 这种结构保证了算法对每个可行目标x探索最优成本，其中最好的是全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    party = [[] for _ in range(m + 1)]
    cnt = [0] * (m + 1)
    
    voters = []
    
    for _ in range(n):
        p, c = map(int, input().split())
        cnt[p] += 1
        voters.append((p, c))
        if p != 1:
            party[p].append(c)
    
    for j in range(1, m + 1):
        party[j].sort()
    
    base = cnt[1]
    
    ans = float('inf')
    
    for target in range(base, n + 1):
        need = [0] * (m + 1)
        cur = base
        
        chosen = []
        used = [0] * (m + 1)
        
        for j in range(2, m + 1):
            if cnt[j] >= target:
                need[j] = cnt[j] - (target - 1)
                for i in range(need[j]):
                    chosen.append(party[j][i])
                    cur += 1
                    used[j] += 1
        
        if cur < target:
            rem = []
            
            for j in range(2, m + 1):
                for c in party[j][used[j]:]:
                    rem.append(c)
            
            rem.sort()
            
            cur_cost = sum(chosen)
            for c in rem:
                if cur >= target:
                    break
                cur += 1
                cur_cost += c
            
            ans = min(ans, cur_cost)
        else:
            ans = min(ans, sum(chosen))
    
    print(ans)

if __name__ == "__main__":
    solve()
```实施过程首先按当前政党对选民进行分组，并按贿赂成本对每个组进行排序。 这使得当一个政党必须减少时，总能​​以最便宜的价格吸引所需的选民。 

主循环尝试为政党 1 尝试所有可能的最终投票计数。对于每个目标，它首先通过从代表性过高的政党中选择最便宜的必要选民来强制执行约束，即没有其他政党超过阈值。 这体现在`need`计算和`chosen`列表。 

在强制选择之后，代码检查第一方是否已经达到目标。 如果没有，它会建立一个包含所有剩余未选定选民的列表，并按成本贪婪地使用它们，直到达到目标。 强制缩减和可选扩展之间的这种分离可以保持正确性。 

最终的答案是所有可行目标中成本最小的。

 ## 边缘情况

 一个关键的边缘情况是，第一方已经拥有足够的选票，可以在没有任何贿赂的情况下严格领先。 例如，如果输入是`n = 3`, 有票数`[1, 1, 2]`，一方已经获胜，算法正确地认为目标等于初始计数并返回零成本。 

另一种情况是当一个对手与其他对手相比非常大时。 如果第 2 方拥有几乎所有选票，则算法将被迫在强制阶段从该方中挑选许多选民以实现任何合理的目标。 这正确地优先考虑减少主导政党，而不是将精力浪费在较小的政党上，因为按政党排序可确保我们始终从实际违反约束的唯一政党中删除最便宜的选民。 

当强制删除已经超过第一方的目标时，会出现最后一个微妙的情况。在这种情况下，不需要额外的贿赂，并且算法正确地避免从剩余池中进行选择。
