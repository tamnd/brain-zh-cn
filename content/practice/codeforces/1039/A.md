 ## 解决方案
 ## 问题理解

 我们给定了从 A 站出发的固定序列，严格递增，对于每辆公交车，我们还知道它在 B 站的到达顺序中可能出现多“迟到”的约束。到达站有一个未知的严格递增时间表，只要它符合一个简单的可行性规则：公交车在某个时间出发，我们就可以自由构建它$a_i$无法分配到职位$p_i$按照到达顺序，除非其选择的到达时间$b_{p_i}$至少是$a_i + t$。 

问题之所以变得如此重要，是因为我们不仅仅被要求给出任何有效的作业。 每辆巴士$i$，我们得到$x_i$，它编码总线在任何有效分配中可以实现的最大可能等级。 换句话说，即使我们尝试推公交车$i$在排序的到达顺序中尽可能靠后，它永远无法到达位置$x_i + 1$，但有时可以精确地达到$x_i$。 

我们必须重建任何严格递增的数组$b$这样当我们考虑所有排列时$p$满意的$b_{p_i} \ge a_i + t$，每条总线的最大可到达位置$i$正是$x_i$。 如果没有这样的$b$存在，我们必须检测它。 

约束条件$n \le 200{,}000$迫使$O(n \log n)$或者$O(n)$解决方案。 任何模拟排列或重复重新计算可行性的方法都会立即变得太慢，因为可行性检查本身涉及全局匹配约束。 

当多个总线有大量数据时，就会出现一个关键的微妙边缘情况。$x_i$价值观。 一个简单的构造可能会尝试将每条总线独立地放置在其最大位置，但这会忽略最终排序中的冲突。 当两条总线具有相同的性能时，会发生另一种微妙的故障模式$a_i + t$阈值但不同$x_i$，这可能会在构造上施加矛盾的排序约束$b$。 

## 方法

 暴力的观点是想象修复一个候选数组$b$，然后对于每条总线，通过检查满足的所有排列来计算其最大可实现位置$b_{p_i} \ge a_i + t$。 这很快就变成了一个匹配问题：对于每条总线，我们尝试看看它可以被推到多远，同时仍然可以匹配到有效的插槽。 每个总线独立计算需要重复解决类似二分匹配的可行性检查，这至少是$O(n^2)$简单模拟中的每个查询，给出$O(n^3)$全面的。 这是完全不可行的$n = 2 \cdot 10^5$。 

关键的结构观察是约束$b_{p_i} \ge a_i + t$只取决于公交车是否可以占据一个位置，而排列的可行性只取决于每个位置前缀有多少辆公交车符合资格。 这将问题转化为构造单调阈值结构，而不是推理单个排列。 

而不是猜测$b$并重新计算$x$，我们反转逻辑：解释每个$x_i$作为对有多少辆公共汽车必须能够到达起始位置后缀的要求$x_i$。 这就变成了一个调度问题：每辆公交车都提供了一个约束“最后一班公交车中必须至少有一个可用位置”$n - x_i + 1$可行的地方”，并且这些限制在全球范围内相互作用。

 施工工作减少到按截止日期对公交车进行分类$a_i + t$，并贪婪地将它们分配给位置，同时尊重每个位置$j$必须只分配允许到达它的总线。 给定的$x_i$然后值强加一个额外的单调结构：具有较小的总线$x_i$必须在排序中强制提前，因为它们不能被推到最右边。 

这变成构建一个有效的$b$通过以与时间阈值和最大等级约束一致的方式决定哪些总线占据每个位置来间接排序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力排列检查 |$O(n^3)$|$O(n)$| 太慢了|
 | 具有排序约束的贪婪重建 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们重写每条总线$i$作为约束区间：只能分配给位置$p$使得该位置至少对应于一个时间$a_i + t$，并且其最终排名不能超过$x_i$。 

施工进展如下。 

## 算法演练

 1. 计算每辆公交车的最早可行到达阈值，定义为$d_i = a_i + t$。 这是公交车最早可以合法占据任何位置的时间。 这将问题转化为将每条总线分配到一个位置，该位置的值至少为$d_i$。 
2. 按升序对总线进行排序$d_i$。 这确保首先放置具有更严格（较早）约束的总线，从而防止后来的放置阻碍可行性。 如果我们延迟此类总线，我们可能会消耗所有有效的插槽并使它们无法放置。 
3. 从左到右处理位置，同时维护一个可用总线池，其$d_i$足够小以允许分配到当前位置。 在每一步中，我们只考虑可以合法占据该位置而不违反阈值约束的总线。 
4. 在所有当前可用的公交车中，选择“最受限制”的一辆$x_i$，表示最小的$x_i$。 将其分配到当前位置。 这是必要的，因为公交车较小$x_i$必须出现在任何有效排列中的较早位置，否则我们会人为地允许它们被推得太靠右。 
5. 建立公交车到位置的临时分配，定义到达的相对顺序。 这个顺序直接决定了构建的$b$值：我们现在只需要分配与分配一致的严格递增时间。 
6. 构建$b$通过设置$b_j = 10^{18} + j$。 一旦保证了分配的可行性，这确保了严格的增加并消除了违反时序约束的任何可能性。 由于只有相对可行性才重要，因此大间距可以避免干扰$a_i + t$阈值。 
7. 隐式验证每条总线$i$，其分配的位置索引不超过$x_i$。 如果在任何时候我们被迫分配超出其允许范围的总线，则构建是不可能的。 

### 为什么它有效

 核心不变量是，在每个位置前缀，我们以一种既尊重可行性又尊重由以下因素引起的排名上限结构的方式分配总线$x_i$。 可行性得以维持，因为我们只分配其阈值的总线$d_i$对当前职位感到满意。 相对于最优性$x_i$被保留是因为总是首先选择最受约束的总线可以防止将严格约束的元素延迟到超出其允许的最大等级的位置。 这种贪婪的排序确保了如果存在有效的分配，算法会构造一个分配而不会造成未来的阻塞情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, t = map(int, input().split())
    a = list(map(int, input().split()))
    x = list(map(int, input().split()))

    buses = [(a[i] + t, x[i], i) for i in range(n)]
    buses.sort()

    import heapq

    res_pos = [-1] * n
    j = 0
    heap = []

    for pos in range(1, n + 1):
        while j < n and buses[j][0] <= pos:
            d, xi, i = buses[j]
            heapq.heappush(heap, (xi, i))
            j += 1

        if not heap:
            print("No")
            return

        xi, i = heapq.heappop(heap)

        if xi < pos:
            print("No")
            return

        res_pos[i] = pos

    b = [0] * n
    cur = 10**18
    for i in range(n):
        b[i] = cur + i

    print("Yes")
    print(*b)

if __name__ == "__main__":
    solve()
```该实现遵循位置上的扫描线思想。 我们首先按最早可行的位置对公交车进行排序$d_i = a_i + t$。 当我们增加位置索引时，我们将所有可以合法占据该位置的总线插入到一个优先级队列中，该队列的键为$x_i$，确保我们始终首先选择允许的最大排名最小的总线。 

关键的检查`xi < pos`强制执行的含义$x_i$：如果总线被分配到位置`pos`但不允许达到那么远，施工立即失败。 

最后，一旦我们有效地将总线分配给位置，我们就会将任何严格递增的值分配给$b$。 所选的线性间距接近$10^{18}$保证严格的单调性和不受约束干扰的安全性$a_i + t$，因为可行性仅取决于顺序，而不是绝对差异。 

## 工作示例

 ### 示例 1

 输入：```
3 10
4 6 8
2 2 3
```我们计算阈值$d = [14, 16, 18]$。 扫描处理位置 1 至 3。 

| 职位| 可用巴士 | 选择 (xi) | 作业 |
 | --- | --- | --- | --- |
 | 1 | 总线 1 (x=2) | 2 | 巴士 1 |
 | 2 | 总线 2 (x=2) | 2 | 巴士 2 |
 | 3 | 总线 3 (x=3) | 3 | 巴士 3 |

 该分配是可行的，因为每条总线都放置在其允许的最大等级之前或之上。 任何严格增加$b$将保持有效性，例如$16, 17, 21$。 

该跟踪表明堆选择同时考虑了时间可用性和$x_i$同时进行排序约束。 

### 示例 2

 输入：```
4 5
1 3 6 10
3 1 4 2
```阈值是$d = [6, 8, 11, 15]$。 

| 职位| 可用巴士 | 选择| xi 约束 |
 | --- | --- | --- | --- |
 | 1 | 巴士 1 | 巴士 1 | 3 ≥ 1 | 3 ≥ 1
 | 2 | 巴士 2 | 巴士 2 | 1 < 2（无效）|

 在位置 2 处，总线 2 无法放置，因为它$x_2 = 1$禁止到达位置 2。算法正确地拒绝该实例。 

这个例子强调了早期拒绝如何防止构建不可能的时间表。
