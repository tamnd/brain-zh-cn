 ## 解决方案
 ## 问题理解

 我们得到一个整数序列。 将它们视为从零开始的某些运行值的每日“变化”。 当我们从左到右时，我们保持一个连续的总和。 每个元素都会增加或减少该运行总和。 

任务是确定从左到右处理数组时此运行总和达到的最高值。 

因此，如果该数组被解释为分数的递增和递减，我们就会被问到：在任何前缀边界处获得的最大分数是多少。 

输出是一个整数：整个数组的最大前缀和。 

这个问题的约束还不够大，不需要任何超出单个线性扫描的东西。 这已经排除了任何尝试重复重新计算前缀和或使用嵌套循环的解决方案，因为这些会变成二次行为并且是不必要的。 

当所有值都为负时，会出现微妙的边缘情况。 在这种情况下，运行总和从零开始并且永远不会提高，因此答案应保持为零。 任何假设至少增加一个正前缀的解决方案都将在这里失败。 例如，输入`[-3, -2, -5]`必须产生`0`， 不是`-3`。 

另一种边缘情况是最大前缀和出现在第一个元素处。 例如`[5, -10, 1]`应该返回`5`。 仅在完全累积后进行检查或忘记考虑中间前缀的幼稚实现可能会错过这一点。 

## 方法

 暴力方法将分别计算每个前缀的总和并取最大值。 对于每个索引 i，我们从 0 到 i 求和。 这是正确的，因为它直接遵循前缀和的定义。 然而，重新计算每个前缀的总和会导致总共大约 n + (n-1) + ... + 1 次操作，即 O(n²)。 对于高达 10^5 的典型约束，这变得太慢了。 

关键的观察是前缀和是增量的。 我们可以在扫描一次数组时维护一个运行总和，而不是从头开始重新计算。 每个新元素都会更新当前的总和，我们立即将其与迄今为止看到的最佳值进行比较。 这将问题从重复重新计算减少到单遍累积。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解前缀重新计算 | O(n²) | O(1) | O(1) | 太慢了|
 | 单程运行总和 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从两个变量开始：`current_sum = 0`和`best = 0`。 初始最佳值为零，因为我们可以在应用任何元素之前考虑起始状态。 
2. 从左到右迭代数组。 在每个位置，将当前元素添加到`current_sum`。 这模拟应用下一个“情绪变化”或“增量”。 
3.更新后`current_sum`，将其与`best`。 如果`current_sum`较大，更换`best`与它。 此步骤确保我们始终记住迄今为止遇到的最高前缀值。 
4. 继续直到数组末尾。 最终值`best`就是答案。 

### 为什么它有效

 位置 i 处的运行和恰好是前 i 个元素的前缀和。 每个可能的前缀和在扫描期间只计算一次。 由于在增量比较下保留了集合的最大值，因此跟踪迄今为止看到的最大值可以保证不会遗漏任何候选前缀。 除了累加和之外，非相邻前缀之间不存在依赖关系，因此不需要额外的结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

current_sum = 0
best = 0

for x in a:
    current_sum += x
    if current_sum > best:
        best = current_sum

print(best)
```该实现是上述扫描的直接转换。 唯一微妙的决定是初始化`best`为零而不是负无穷。 这反映出运行总和在获取任何元素之前开始，因此空前缀有效。 

## 工作示例

 ### 示例 1

 输入：`[1, -2, 3]`| 步骤| 元素| 当前总和| 最佳|
 | --- | --- | --- | --- |
 | 0 | 开始 | 0 | 0 |
 | 1 | 1 | 1 | 1 |
 | 2 | -2 | -1 | 1 |
 | 3 | 3 | 2 | 2 |

 这表明最大值如何在随后的积极反弹之后出现，而不一定是在开始时出现。 

### 示例 2

 输入：`[-1, -2, -3]`| 步骤| 元素| 当前总和| 最佳|
 | --- | --- | --- | --- |
 | 0 | 开始 | 0 | 0 |
 | 1 | -1 | -1 | 0 |
 | 2 | -2 | -3 | 0 |
 | 3 | -3 | -6 | 0 |

 这证实了当所有前缀减少时该算法正确地保留零。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | 每个元素都被处理一次并不断更新 |
 | 空间| O(1) | O(1) | 仅维护两个运行变量 |

 线性扫描是最佳的，因为每个元素必须至少读取一次，并且不需要额外的结构。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(input())
    a = list(map(int, input().split()))
    current_sum = 0
    best = 0
    for x in a:
        current_sum += x
        best = max(best, current_sum)
    return str(best)

 ## 边缘情况

 对于完全由负数组成的输入，运行总和不断减小。 由于该算法始终与零初始化的最佳值进行比较，因此它会正确返回零。 例如`[-4, -2, -7]`产生`0`因为没有前缀超过空前缀。 

对于单元素数组，如果为正，则第一次更新会立即将 current 和 best 设置为该值；如果为负，则将 best 保持为零。 这符合我们在处理任何元素之前始终与起始状态进行比较的想法。
