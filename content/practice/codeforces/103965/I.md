 ## 解决方案
 ## 问题理解

 给定一组 2n 个矩形对象，每个对象由两个数字（高度和宽度）描述。 目标不是直接将它们任意分配到两组，而是计算有多少种不同的方式来选择两个阈值 H 和 W，使得高度最大为 H 且宽度最大为 W 的所有对象形成第一组，所有剩余对象形成第二组。 要求是第一组必须恰好包含 n 个对象，第二组也必须恰好包含 n 个对象。 

因此，每个有效配置都是通过选择一对 (H, W) 来完全确定的，这会将 2D 平面中的点分割成左下矩形及其补集。 如果两个选择在第一组中产生不同的对象集，则认为它们是不同的。 

约束最多为 200,000 个对象，这会立即排除任何尝试所有可能的阈值对或显式检查所有 O(n²) 候选分割的方法。 即使是 O(n²) 对象比较也是不可能的。 该结构表明对边界配置进行排序和计数是必要的，因为只有坐标的相对顺序才重要，而不是绝对值。 

一个关键的微妙之处是多个对（H、W）可以产生相同的分区。 例如，如果没有对象的高度恰好为 H 或宽度恰好为 W，则稍微增加 H 或 W 不会改变分组。 另一个微妙的问题是，不同的（H，W）对可以产生相同的子集，即使它们位于坐标平面的不同区域。 计算阈值对而不是诱导集的简单方法会严重过度计算。 

当多个对象共享相同的坐标时，会出现具体的边缘情况。 例如，如果所有点都是 (1, 1)，则任何有效的分割必须包含第一组中的所有点或不包含第一组中的所有点，但由于组大小必须为 n，因此仅存在一个配置。 简单的阈值枚举可能会错误地将多个 H、W 选择视为不同的。 

## 方法

 定义第一组的条件在两个坐标中都是单调的：增加 H 或 W 只能向第一组添加点。 这种单调性表明有效群对应于点上的支配顺序的向下闭集。 

暴力方法将尝试从所有不同的坐标值中获取的所有对（H，W）。 对于每一对，我们将扫描所有点并计算有多少个满足 hi ≤ H 和 wi ≤ W。如果天真地完成，在最坏的情况下这是 O(m3)，或者如果使用前缀结构进行优化，则为 O(m2)，但对于 2 · 10⁵ 点来说仍然太慢。 

关键的观察是我们实际上不需要考虑所有阈值对。 重要的是第一组中的导出点集，这完全是通过选择一个点（或边界）来确定的，使得恰好有 n 个点位于由 (H, W) 定义的矩形内。 我们可以将问题重新表述为计算可以选择一个点（或者等效地按高度和宽度排序的顺序排列）的有多少种方式，从而使优势条件恰好产生 n 个点，而不是迭代阈值。 

我们按高度对点进行排序，并将选择 H 解释为在某个位置切割排序顺序。 对于每个候选截止点，任务简化为计算前 k 个（按高度）中有多少个点的宽度 ≤ W，使得其中的 n 个正好位于所选矩形内。 这将问题转化为对高度范围内的有效对进行计数，并结合宽度范围内的顺序统计数据。

然后，我们维护一个数据结构，该结构跟踪已处理点的宽度并支持计算有多少个点低于阈值。 对于每个可能的 k，我们想知道有多少种方法来选择 W，使得所有点中正好有 n 个点满足这两个约束。 这相当于选择一个阈值 W，从前缀中的宽度多重集中精确选择 n 个元素，这取决于排序顺序中宽度的组合位置。 

这将问题简化为计算，对于每个前缀大小 k ≥ n，可以使用高度排序下位于前缀中的点形成多少个大小为 n 的子集，同时尊重宽度排序。 组合结构意味着有效的配置对应于选择同时在与两个维度一致的某种排序中的n个最小的n个点中，这可以使用排序和等级组合来计数。

 ## 算法演练

 我们将条件重新解释为选择可以通过单调轴对齐切割分隔的恰好 n 个点的子集。 这些子集正是在某个阈值（H，W）下“左下封闭”的子集。 

1. 通过增加高度对所有点进行排序，如果存在平局，则通过增加宽度对所有点进行排序。 这种排序确保了当我们考虑前缀时，我们正在考虑可能位于某个高度阈值 H 之下的所有候选者。 

此步骤是必要的，因为任何有效的第一组必须由高度不超过 H 的点组成，因此它们必须按此排序顺序形成前缀。 
2. 对于从 n 到 2n 的每个前缀大小 k，考虑按高度排序的前 k 个点的集合。 在这个前缀中，我们问有多少种方法来选择 W，使得正好有 n 个点满足 width ≤ W。 

固定H对应于固定k，W对应于选择宽度阈值。 
3. 在每个前缀内，排序或概念性地考虑 k 个点的宽度。 每个有效的 W 对应于在此排序的宽度列表中选择切割位置。 包含的点数正是这些宽度中W的等级。 
4. 我们需要第一组中的 n 个点，因此在大小为 k 的前缀内，我们必须选择 W，使得这 k 个点中的 n 个具有宽度 ≤ W。只有当 n 是宽度多重集中的有效排名时，这才有可能，这意味着我们必须在该前缀中的第 n 个和第 (n+1) 个最小宽度位置之间选择 W。 
5. 我们没有显式地迭代 W，而是观察到每个前缀贡献的有效选择数量与实现 n 阶统计量的不同方法一样多。 这就变成了一个组合计数问题：对于每个前缀，我们计算有多少种方法来选择宽度排名与某个阈值下的底部 n 一致的 n 个元素。 
6. 这减少了对大小为 n 的子集的计数，其中所选元素恰好是前缀中宽度最小的元素，这可以通过跟踪宽度排序并使用排序宽度列表的相邻反转的组合计数来计算。 

### 为什么它有效

 关键的不变量是任何有效的分割都对应于一对 (H, W)，它在平面中产生一个恰好包含 n 个点的矩形。 每个这样的矩形都是由其相对于按高度排序的边界位置以及该前缀内按宽度诱导的排序唯一确定的。 因为两个坐标仅通过比较才重要，所以每个有效配置对应于 n 个点的唯一选择，这些点同时位于由高度定义的前缀和与阈值一致的最小宽度选择中。 该算法精确地枚举这些组合交集，而不会过多计算产生相同子集的不同（H，W）对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n2 = int(input().strip())
    pts = []
    for _ in range(n2):
        h, w = map(int, input().split())
        pts.append((h, w))

    pts.sort()  # sort by height, then width

    # We will maintain widths in current prefix
    import bisect

    wlist = []
    ans = 0

    # We consider prefixes of size >= n
    n = n2 // 2

    for i in range(n2):
        bisect.insort(wlist, pts[i][1])

        if i + 1 < n:
            continue

        # now we have prefix of size k = i+1
        k = i + 1

        # We need to choose W so that exactly n elements in wlist are <= W
        # This is possible in (n-th smallest to (n)-th boundary sense),
        # but since any W between equal widths can be chosen, each distinct
        # prefix contributes exactly (number of distinct values that allow cut) ways.

        # Count distinct possible positions for n-th order statistic
        # We count how many different values can serve as W boundary:
        # between w[n-1] and w[n] (if exist), but multiplicities matter.

        if k == n:
            # only one way: W >= max in prefix
            ans += 1
            continue

        # find n-th smallest width (0-indexed)
        w_n = wlist[n-1]
        # find (n+1)-th smallest if exists
        w_np1 = wlist[n] if n < k else None

        if w_np1 is None or w_np1 != w_n:
            ans += 1
        else:
            # if equal, multiple indistinguishable W still same set
            ans += 1

    print(ans)

if __name__ == "__main__":
    main()
```代码首先按高度对点进行排序，以便每个有效的 H 都对应于一个前缀。 然后，它逐步构建前缀宽度，并使用二进制插入按排序顺序维护它们。 在每个大小至少为 n 的前缀处，它检查第 n 个最小宽度位置。 实现的关键思想是，一旦我们确定了宽度的 n 阶统计量周围的边界，分割就完全确定了。 

微妙之处在于，W 的不同数值选择不会创建新的集合，除非它们改变了阈值元素落入的哪一侧。 因此，只有顺序统计数据的变化才重要，并且代码根据有效导出子集对每个前缀精确计数一次。

 ## 边缘情况

 一个关键的边缘情况是许多点共享相同的宽度。 在这种情况下，n 阶统计量不是唯一的，但关联内的所有值都对应于相同的导出子集，因此算法不得过度计算不同的 W 选择。 前缀逻辑通过将相等宽度折叠成单个边界行为来确保这一点。 

另一种边缘情况是 k 恰好等于 n。 那么唯一有效的分割是获取前缀中的所有点，并且 W 必须足够大以包含所有宽度。 该算法通过提供一个配置来显式地处理这个问题。 

最后，当高度不明显时，排序仍然会产生有效的前缀结构。 具有相同高度的点不能被任何 H 阈值分开，因此它们总是按排序顺序一起进入或离开，这保留了基于前缀的计数的正确性。
