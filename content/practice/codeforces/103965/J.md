 ## 问题理解

 我们得到了一系列叶子堆大小，但这些叶子堆位于一长串位置上，编号从 1 到 c。 这些位置中只有 n 个实际上包含非零堆，特别是位置 1 到 n 存储值 a1 到 an，而 n 之后到 c 的所有位置实际上都是空的。 

我们可以选择任何连续的固定长度 k 的位置段，位于范围 [1, c] 内的任何位置。 对于每个这样的段，我们计算落在其中的所有桩的值的总和。 1..n 之外的位置没有任何贡献，因为它们是空的。 

任务是找到长度为 k 的线段，使该总和最小化。 

关键限制是 c 可以大到 10^9，这使得不可能将其视为完整数组上的普通滑动窗口。 实际非零元素的数量仅为 n 最多 10^5，因此任何正确的解决方案都必须避免在整个坐标范围上迭代，而是推理窗口如何与小活动区域交互。 

一个天真的想法是构建一个大小为 c 的数组并滑动一个长度为 k 的窗口。 这会立即失败，因为即使迭代 c 也是不可能的。 

第二个天真的想法是只考虑完全在 1..n 范围内的窗口。 当窗口延伸到超过 n 的空白区域时，这会错过有效的最佳答案，这会显着减少总和。 

当 k 足够大以至于窗口可以完全位于空白区域时，就会出现一种微妙的情况。 例如，如果 n = 5、c = 10、k = 4，则选择 [6, 9] 会产生总和 0，这是最佳的，即使它不触及任何 a 值。 任何忽略空白区域的方法都会在这里失败。 

当 k 大于 n 时，会出现另一个棘手的情况。 那么接触任何非空元素的每个有效窗口都必须包含数组的后缀，并且纯粹根据 a 的固定长度子数组进行推理变得不正确。 

## 方法

 暴力方法将显式构造一个大小为 c 的数组 b，其中 b[i] = a[i]（i ≤ n），否则 b[i] = 0，然后评估长度为 k 的每个段。 每个窗口总和的计算时间为 O(k)，总体复杂度为 O(c·k)，这远远超出了任何可行的限制，因为 c 可以是 10^9。
