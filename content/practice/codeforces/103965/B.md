 ## 解决方案
 ## 问题理解

 我们得到了一组歌曲，每首歌曲都有一个基本的欣赏价值。 我们通过反复选择下一首要播放的歌曲来构建长度为 k 的播放列表。 不同之处在于，重复连续播放同一首歌曲会降低其乐趣：如果连续播放一首基值为 a 的歌曲，第一次给出 a，第二次给出 a−1，第三次给出 a−2，依此类推，但永远不会低于零。 一旦我们切换到另一首歌曲，前一首歌曲的衰减就会重置。 

每一步都会贪婪地构建播放列表。 我们查看每首歌曲当前的“下一个播放价值”，选择其中最大值，然后选择该歌曲。 如果出现平局，我们可以选择任何与前一首不同的歌曲。 只有当不存在这样的替代方案时，我们才会继续上一首歌曲。 

目标是计算在该规则下玩完 k 次游戏后获得的总乐趣。 

这些限制使得直接模拟变得不可能。 当 k 达到 10^9 时，即使 O(k) 工作量也太大了。 歌曲数量 n 最多为 2×10^5，这表明任何涉及排序或堆的事情都可以，但前提是我们不单独迭代 k 个步骤。 关键的困难在于贪婪过程会产生重复歌曲的长条纹，因此我们需要将这些条纹压缩成块。 

简单的模拟会重复扫描所有歌曲以找到下一个最佳选择。 仅此一项就花费 O(nk)，这立刻就太慢了。 

第二种简单的方法是维护当前值的优先级队列，但每次选择同一首歌曲后值都会发生变化。 即便如此，我们仍然面临 k 次更新，这个更新仍然太大。 

当最好的歌曲也是上一首歌曲，但另一首歌曲具有相同的当前值时，就会出现微妙的边缘情况。 在这种情况下，即使继续上一首歌曲也是最佳选择，打破平局规则也会强制切换。 这意味着我们不能将其视为纯粹的“总是取最大值”的过程； 之前的选择限制了我们。 

## 方法

 暴力模拟很简单。 在每 k 个步骤中，我们计算每首歌曲的当前有效值，根据规则选择最好的一首，更新其连续计数器，并在需要时重置其他计数器。 这是有效的，因为状态很简单：只有最后选择的歌曲具有递减的值，而所有其他歌曲都处于其基本值。 然而，这仍然每步成本为 O(n)，导致 O(nk)，当 k 达到 10^9 时，这远远超出了可行的限制。 

关键的观察结果是，除了一首“活跃”歌曲（当前正在重复的歌曲）之外，该系统几乎没有进化结构。 所有其他歌曲均保持其基本值不变。 这意味着竞争始终在两个量之间：当前歌曲的递减值和所有其他歌曲中的最佳固定值。 

这将问题简化为推理，在当前歌曲停止严格优于最佳选择之前，我们可以继续连续多长时间。 一旦发生这种情况，我们就必须切换，然后重复这个过程。 每条条纹都可以使用算术级数和一次性处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(nk) | O(nk) | O(n) | 太慢了|
 | 条纹压缩| O(n log n) 或 O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先对基本享受值列表进行预处理，并确定最大值和第二大值。 我们还跟踪有多少歌曲具有最大值，因为当前一首歌曲是唯一最大值时，这会影响平局决胜局。 

然后，我们模拟播放列表，但我们不是一次步进一首歌曲，而是处理同一首歌曲的整个连续块。

1. 我们将第一首选择的歌曲初始化为基值最大的歌曲。 它的第一次发挥就给出了全部价值，我们将其连续连胜设置为 1。 
2. 在任何时刻，我们都知道当前的歌曲、它的基值以及它已经连续播放了多少次。 这决定了它当前的贡献：基数减去条纹偏移量。 
3. 我们计算最佳替代值，如果当前歌曲是唯一最大值，则该替代值是全局最大值或第二最大值。 
4. 如果当前歌曲的下一个值严格大于最佳替代值，我们将继续连续。 我们计算在其递减的值降至或低于替代方案之前我们可以安全地继续多少次。 这给出了我们可以使用算术和一次性添加的完整段。 
5. 如果当前歌曲无法继续，我们切换到最佳替代歌曲，将其连胜重置为 1，然后重复该过程。 

连胜中的关键计算是确定我们可以采取多少步。 如果段开头的当前值是 cur 并且最佳替代值是 best_other，则我们继续，同时 cur、cur−1、cur−2 等保持严格大于 best_other。 这会在段长度上产生一个干净的界限。 

### 为什么它有效

 在任何时刻，只有两种类型的候选者很重要：当前重复的歌曲，其值线性下降，以及所有其他歌曲中最好的歌曲，其值保持不变。 贪心规则确保我们总是选择这两者中的最大值。 由于其中只有一个会随着时间的推移而变化，因此每次连续只能跨越一次决策边界。 这保证了每首歌曲仅在块中变得活跃，并且每个块完全由一个简单的不等式决定，使得该过程精确且无损。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    a.sort()
    mx = a[-1]
    smx = a[-2] if n > 1 else 0
    
    # frequency of maximum value
    cnt_mx = a.count(mx)
    
    def best_other(prev_is_unique_max):
        if prev_is_unique_max:
            return smx
        return mx
    
    # start with max element
    cur_val = mx
    streak = 1
    k -= 1
    ans = mx
    
    prev_is_unique_max = (cnt_mx == 1)
    
    while k > 0:
        bo = smx if (cur_val == mx and prev_is_unique_max) else mx
        
        # try to continue current song
        if cur_val > bo:
            # max steps we can take
            t = cur_val - bo
            t = min(t, k)
            
            # sum of decreasing sequence: cur_val + (cur_val-1) + ...
            ans += t * cur_val - t * (t - 1) // 2
            
            cur_val -= t
            k -= t
            streak += t
            
            if k == 0:
                break
        
        # switch song
        # choose best alternative value
        if cur_val == mx and prev_is_unique_max:
            new_val = smx
        else:
            new_val = mx
        
        if new_val == 0:
            break
        
        ans += new_val
        cur_val = new_val
        streak = 1
        k -= 1
        
        # after switching, previous is no longer unique max situation
        prev_is_unique_max = False
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先识别两个最大的基值，因为整个决策过程仅取决于它们。 在主循环内，它要么使用封闭式求和来扩展当前连续，要么切换到最佳可用替代方案。 

算术级数公式至关重要：我们不是将逐一递减的值相加，而是在恒定时间内计算块的全部贡献。 使用整数除法时必须小心，并确保仅当条纹长度为正数时才应用该公式。 

一个微妙的点是处理前一首歌曲是唯一最大值的情况。 在这种情况下，替代方案是第二个最大值，否则它是全局最大值。

 ## 边缘情况

 一个关键的边缘情况是最大值仅出现一次。 在这种情况下，当另一首歌曲在衰减后具有相同的当前值时，可能不允许停留在最大歌曲上。 在这种情况下，算法显式切换到第二个最大值，确保遵守平局规则。 

当 k 非常大时，会出现另一种边缘情况。 简单的模拟永远不会完成，但基于块的方法会消耗大块的 k，从而防止单个步骤的迭代。 

最后，当所有值都相等时，每次切换都受到平局规则的强制，并且该过程退化为重复交替。 该算法仍然可以正确处理这个问题，因为最佳替代方案和当前衰减立即满足。
