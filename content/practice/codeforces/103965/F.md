 ## 解决方案
 ## 问题理解

 我们正在模拟一辆火车车厢，一开始就挤满了普通乘客。 随后，特权乘客一一到达并试图尽快入座，但只有当普通乘客决定站起来时才能有座位。 

每个普通乘客都有周期性的行为。 乘客 i 每 ai 分钟检查一次情况，有时是 ai 的倍数。 当他们检查并发现至少有一名特权乘客正在站立等待座位时，他们会立即永久让出座位。 一旦普通乘客站起来，该座位就永远免费，并且永远不会被收回。 

特权乘客随着时间的推移以固定的非递减顺序到达时刻 bi 到达。 当特权乘客到达时，他们首先尝试占据任何已经空闲的座位。 如果没有空位，他们就会等待。 重要的是，稍后腾出的座位可以立即被较早等候的特权乘客占据。 

时间以离散的分钟为单位演变，每一分钟内的行动顺序都是固定的：特权乘客首先到达，然后等待的乘客尝试使用当前空闲的座位，只有在此之后“检查时间”的普通乘客才有可能站起来并释放座位。 

输出要求每位特权乘客在他们设法坐下的确切时刻。 

约束 n、m 高达 100000 意味着任何逐分钟模拟的解决方案都是不可能的。 即使迭代所有时间直至 100000 次并每次检查所有乘客也会太慢。 该结构强烈表明，每个普通乘客最多贡献一次有意义的事件，因为一旦他们站起来，他们就不会再互动。 这促使我们计算每个普通乘客的单次激活时间。 

一个微妙的极端情况来自在一分钟内的订购。 如果特权乘客恰好在普通乘客起身的同时到达，则在新座位释放之前，特权乘客可以首先使用任何已经空闲的座位。 这可能会改变某人是立即坐下还是必须等待稍后发布。 

## 方法

 直接模拟将尝试处理每一分钟并更新所有 n 个普通乘客的状态，检查他们是否应该站起来。 在最坏的情况下，这需要每次检查所有乘客数量达到 100000，导致大约 1010 次操作，这太慢了。 

关键的观察结果是普通乘客只会改变一次状态。 当系统变得“活跃”时，意味着至少有一名特权乘客正在等待或没有座位，每个普通乘客最终都会在第一次注意到这种情况时站起来，这种情况发生在第一个特权到达时或之后的第一次检查时间。 

因此，整个过程减少到等于 b1 的单个全局阈值时间 t0，即第一个特权到达。 从那一刻起，直到所有特权乘客就座为止，每个普通乘客的行为仅取决于他们的下一个ai倍数是否发生在t0之后。 每个乘客在其第一次检查时间（至少为 t0）准确地贡献了一个潜在的座位释放。 

这样我们就可以为每个普通乘客i预先计算出一个时间ti，它是ai的第一个不早于t0的倍数。 每个 ti 代表一个可用的座位。

一旦我们有了所有这些座位释放时间，剩下的问题就变成了调度过程。 我们合并两个排序序列：座位释放事件和特权到达事件。 每次到达时，如果有空位，就会立即被占用。 否则，乘客将等待直到下一个座位被释放。 每当发生座位释放时，它要么立即被最早等待的乘客使用，要么添加到空闲座位池中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每分钟蛮力模拟| O(T·n) | O(T·n) | O(n) | 太慢了|
 | 基于事件的到达和座位释放合并 | O((n + m) log n) | O((n + m) log n) | O(n + m) | 已接受 |

 ## 算法演练

 我们将这个过程转化为事件：每个普通乘客在时间ti产生一个“座位释放事件”，每个特权乘客在时间bi产生一个“到达事件”。 

1. 将 t0 计算为 b1，即最早的特权到达时间。 从这一刻起，系统开始启动，普通乘客可能会开始永久离开。 
2.对于每个普通旅客i，计算他们在t0或之后的第一次登机时间。 这是 ti = ceil(t0 / ai) * ai。 这代表他们的座位可用的那一刻。 
3. 收集所有 ti 值并按升序对它们进行排序。 这些代表未来的座位可用时刻。 
4. 按到达顺序处理特权乘客，同时按时间顺序处理座位释放。 在已排序的 ti 列表上维护一个指针，并在计数器上显示当前有多少空闲座位，以及到达但无法立即坐下的等待特权乘客的队列。 
5. 当处理时间 t 的到达时，首先将时间严格小于 t 的所有座位释放插入到空闲座位池中。 如果有空闲座位，请立即尝试为到达的乘客安排座位。 如果没有，将它们添加到等待队列中。 
6. 处理完时间 t 的所有到达后，处理恰好在时间 t 发生的所有座位释放。 每次释放座位时，要么立即将其分配给最早等待的特权乘客，要么在没有人等待的情况下增加空闲座位数。 
7. 每当特权乘客被分配座位时，记录当前时间作为他们的答案。 

每个时间步内的顺序至关重要。 到达必须始终在同时释放之前进行处理，因为在时间 t 空闲的座位不能被在时间 t 到达的乘客使用，除非该座位在此时刻之前已经存在。 

### 为什么它有效

 该过程保证一旦系统在时间 b1 激活，总是有至少一名特权乘客等待，直到所有人就座。 这确保了每个普通乘客最终到达他们的第一次检查时间，其中“有人站立”的条件成立，因此每个人都贡献了一个确定的释放时间 ti。 在这种减少之后，系统变成一个标准的双流调度问题，其中时间顺序的贪婪分配是最优的，因为较早到达的总是优先于较晚到达的，并且座位是不可区分的。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    t0 = b[0]

    releases = []
    for ai in a:
        # first multiple of ai >= t0
        t = ((t0 + ai - 1) // ai) * ai
        releases.append(t)

    releases.sort()

    ans = [-1] * m
    free_seats = 0
    wait = deque()

    i = 0
    j = 0  # pointer over releases

    # We process in time order of events from both lists
    # We'll iterate over arrivals and also inject releases as needed
    for i in range(m):
        t = b[i]

        # process all releases strictly before t
        while j < n and releases[j] < t:
            if wait:
                idx = wait.popleft()
                ans[idx] = releases[j]
            else:
                free_seats += 1
            j += 1

        # arrival at time t
        if free_seats > 0:
            free_seats -= 1
            ans[i] = t
        else:
            wait.append(i)

        # process releases at exactly time t
        while j < n and releases[j] == t:
            if wait:
                idx = wait.popleft()
                ans[idx] = t
            else:
                free_seats += 1
            j += 1

    # remaining releases after last arrival
    while j < n:
        if wait:
            idx = wait.popleft()
            ans[idx] = releases[j]
        else:
            free_seats += 1
        j += 1

    print(*ans)

if __name__ == "__main__":
    solve()
```该实现首先将每个普通乘客折叠成单个发布时间戳。 然后，它使用两个指针将这些时间戳与到达序列合并。 排队是必要的，因为无法立即就座的到达者必须保持秩序。 空闲座位柜台处理在任何等待的乘客出现之前释放的情况。 

一个常见的陷阱是错误地处理相等的时间戳。 该代码同时显式处理释放之前的到达，确保在时间 t 释放的座位不能被在时间 t 到达的人使用，除非它之前已经存在。

 ## 边缘情况

 一个关键的边缘情况是，特权乘客恰好在座位释放时到达。 例如，如果座位在时间 10 变得空闲，并且乘客也在时间 10 到达，则到达者必须在处理同时释放之前看到预先存在的空闲座位。 这可以确保他们不会错误地跳过本来应该有的座位。 

另一个微妙的情况是多个普通乘客的释放时间相同。 在这种情况下，多个座位同时可用，并且必须按到达顺序将它们分配给等待的乘客。 即使同时出现多个座位，队列也会确保先分配较早等候的乘客。 

第三种情况是当座位空出时没有特权乘客在等待。 座位应该累积在空闲池中并在下一次到达时消耗而不是丢失。
