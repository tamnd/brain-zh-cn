 ## 解决方案
 ## 问题理解

 我们得到一个或多个整数序列，对于每个序列，我们需要确定哪些值是“唯一”的，因为它们在该序列中只出现一次。 输出完全来自频率信息：除了识别出现次数为 1 的元素之外，我们不会转换数组或构造任何新的东西。 

从计算的角度来看，输入大小可以达到典型的竞争性编程限制，其中所有测试用例的元素总数可能很大，通常高达约\(10^5\)或更多。 这立即排除了任何将每个元素与所有其他元素进行比较的二次方法。 对数组执行嵌套扫描的解决方案将降级为 \(O(n^2)\)，这在超过几千个元素时变得不可行。 

当重复项分散而不是聚集时，就会出现微妙的失败情况。 例如，如果数组是\([1, 2, 1, 3, 2]\)，唯一元素只有\(3\)。 如果不维护整个序列的全局计数，天真的“标记已访问”或“动态删除重复项”方法很容易对元素进行错误分类。 

单元素数组会出现另一种边缘情况。 用于输入\([7]\)，正确的输出是\(7\)，因为它只出现一次。 任何假设“唯一性需要与邻居进行比较”的逻辑都会由于缺少边界上下文而错误地丢弃它。 

## 方法

 暴力破解的想法很简单：对于每个元素，扫描整个数组并计算它出现的次数。 如果计数等于 1，我们将其包含在输出中。 这是正确的，因为它直接遵循唯一性的定义。 

但是，此方法对每个元素执行完整扫描。 对于大小的数组\(n\)，这导致\(n\)扫描尺寸\(n\)，导致 \(O(n^2)\) 次操作。 在\(n = 10^5\)，这需要\(10^{10}\)的操作，远远超出任何合理的时间限制。 

关键的观察结果是重复计数是浪费的。 每个元素的频率与我们检查它的顺序无关，如果我们存储中间结果，则可以一次性计算频率。 这建议首先使用哈希映射或字典来累积计数，然后根据这些计数过滤元素。 

这将问题转换为两个线性传递：一个用于构建频率信息，另一个用于提取计数等于 1 的元素。 问题的结构使这一点变得自然，因为唯一性仅取决于全局频率，而不取决于位置关系。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 |---|---|---|---|
 | 蛮力 | \(O(n^2)\) | \(O(1)\) | 太慢了 |
 | 频率图 | \(O(n)\) | \(O(n)\) | 已接受 |

 ## 算法演练

 1. 读取输入序列并将其存储在数组中。 这是必要的，这样我们既可以计算频率，又可以在生成输出时保留原始顺序。 

2. 初始化一个空字典来存储每个整数的计数。 每个键将代表数组中的一个值，关联的值将跟踪它出现的次数。 

3. 遍历数组一次，通过增加每个元素的计数来更新字典。 此步骤将所有重复信息压缩为单个结构。 

4. 以相同的顺序再次遍历数组。 对于每个元素，检查其在字典中的频率。 如果频率等于 1，则将其附加到输出列表中。 

5. 按出现顺序打印唯一元素的结果列表。 

第二次遍历而不是迭代字典键的原因是输出必须保留原始顺序。 单独的字典并不能保证顺序要求。 

### 为什么它有效

 该算法依赖于以下不变量：在第一次传递后，字典包含数组中每个值的准确全局频率。 由于唯一性纯粹由等于 1 的频率定义，因此第二遍中选择的任何元素都必须满足问题条件。 不会遗漏任何元素，因为每个数组元素都按顺序检查一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    freq = {}
    for x in arr:
        freq[x] = freq.get(x, 0) + 1
    
    res = []
    for x in arr:
        if freq[x] == 1:
            res.append(x)
    
    print(*res)

if __name__ == "__main__":
    solve()
```第一个循环使用字典构建频率表，确保每个元素的恒定时间平均更新。 第二个循环根据计算的频率过滤元素，同时保留输出格式所需的输入顺序。 

实现中的一个常见错误是迭代`freq.keys()`而不是原来的数组。 即使频率逻辑正确，这也会破坏排序信息并产生不正确的输出。 另一个微妙的问题是忘记处理没有唯一元素的情况，这应该输出空行而不是错误或占位符。

 ## 边缘情况

 对于像这样的输入`1\n42\n`，第一遍之后的字典变成`{42: 1}`。 在第二遍中，唯一的元素频率为1，因此直接输出。 这证实了该算法不依赖邻接或多次出现来正确运行。 

对于像这样的输入`4\n5 5 6 6\n`，频率图变为`{5:2, 6:2}`。 在过滤过程中，每个元素都会被检查并拒绝，从而产生空输出。 该实现通过打印空行而不是跳过输出生成逻辑来正确处理此问题。
