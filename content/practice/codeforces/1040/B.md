 ## 解决方案

 ## 问题理解

 我们得到一行$n$串从左到右排列。 每个串都有一个二元状态：要么仍处于初始状态，要么已被翻转。 

通过选择位置进行单次操作$i$。 该操作翻转串$i$并且还翻转距离内的每一个串$k$只要这些职位存在，就可以从中受益。 因此，一个操作会影响以$i$，特别是来自的所有索引$i-k$到$i+k$。 

多次操作的效果会以奇偶校验方式累积：如果串被翻转奇数次，它最终会翻转，如果翻转偶数次，它会返回到其原始状态。 目标是确保每个串在最终意义上至少翻转一次，这意味着每个位置必须由至少一个选定的操作覆盖。 

输出不仅是最少的操作次数，而且还包括我们执行这些翻转的确切位置。 

限制因素$n \le 1000$和$k \le 1000$立即建议$O(n^2)$甚至$O(nk)$方法就足够了，因为操作和检查的最大数量很小。 这是一条直线上的经典贪婪覆盖问题，因此我们期望最优解是线性的。 

当出现一个微妙的问题时$k = 0$。 在这种情况下，每个操作仅影响一个串，这意味着必须单独选择每个串。 任何贪婪间距逻辑都必须显式处理这种情况。 

另一个边缘情况是当$2k + 1 \ge n$。 然后，在精心选择的中心进行一次操作即可覆盖整个阵列，因此答案就只剩下一步了。 

一个天真的错误是认为必须避免或动态跟踪翻转重叠。 这会导致模拟翻转并重新计算未覆盖的头寸，这是不必要的，并且存在错误平价推理的风险。 该问题不需要明确管理最终状态奇偶性，只需要确保完全覆盖。 

## 方法

 蛮力的想法是模拟所有可能的操作序列并跟踪覆盖范围。 对于每一步，我们可以尝试每个可能的位置作为下一次翻转，模拟生成的覆盖范围，并递归直到覆盖所有位置。 这很快就会呈指数增长，因为每个状态都会分支到$n$选择，甚至通过覆盖范围进行修剪在最坏的情况下也无法挽救它。 

关键的观察是每个操作覆盖固定长度的连续间隔$2k+1$。 一旦我们将问题重新解释为覆盖该细分市场$[1, n]$当间隔数量最少时，结构就变得纯粹贪婪。 除了覆盖范围之外，间隔之间没有依赖性。 最好的策略始终是使用从该点开始尽可能向右延伸的间隔来覆盖最左侧的未覆盖位置，这是通过最佳地集中操作来实现的。 

因此，我们不是去探索选择，而是反复“跳过”$2k+1$位置，将每个中心放置在尽可能靠左的位置，同时仍然覆盖当前未覆盖的点。

 | --- | --- | --- | --- |
 | 暴力模拟| 指数| O(n) | 太慢了|
 | 贪心区间覆盖| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从最左边的串杆位置 1 开始，这是第一个必须覆盖的位置。 

我们将此视为第一个未发现的点。 
2.如果$k = 0$，输出从1到的每个位置$n$。 

这是必要的，因为每个操作仅翻转一个串，因此无法一次覆盖多个位置。 
3. 否则，对于当前最左边未覆盖的位置$l$, 将操作放置在位置$l + k$。 

这个选择是最优的，因为它最大化了右侧的覆盖范围，同时仍然覆盖$l$, 产生覆盖范围$[l, l + 2k]$。 
4. 下完操作后，将下一个未覆盖的位置更新为$l + 2k + 1$。 

这是因为一切直到$l + 2k$现在已被当前操作覆盖。 
5. 重复直到$l > n$。 

到那时，每个位置都被覆盖了。 

### 为什么它有效

 该算法保持所有位置严格小于的不变量$l$已经被覆盖了。 在每一步中，我们都会选择一个涵盖以下内容的操作$l$并尽可能扩大覆盖范围。 任何替代选择涵盖$l$必须将其中心置于$[l, l+2k]$，并且这些选择都不能将覆盖范围扩大到$l+2k$。 因此，选择$l+k$与任何有效的解决方案相比，永远不会增加操作数量，并且贪婪级数会产生整个段的最小尺寸覆盖。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())

    if k == 0:
        print(n)
        print(*range(1, n + 1))
        return

    res = []
    i = 1

    while i <= n:
        pos = i + k
        if pos > n:
            pos = n
        res.append(pos)
        i += 2 * k + 1

    print(len(res))
    print(*res)

if __name__ == "__main__":
    solve()
```实现直接遵循贪婪结构。 特殊情况$k = 0$首先处理，因为一般公式会错误地尝试逐步$1$同时仍然放置冗余中心。 

变量`i`代表最左边未覆盖的位置。 每一个选择的`pos`是该段的最佳中心。 每次选择后，我们都会准确跳转$2k + 1$步骤，因为这是操作的完整覆盖宽度。 

夹紧`pos`到`n`对于所有公式的正确性来说并不是严格必要的，但它确保我们在以下情况下永远不会引用有效范围之外的位置：$l + k > n$。

 ## 边缘情况

 当$k = 0$，每次操作只影响一个位置。 该算法显式绕过贪婪区间逻辑并直接输出所有索引，避免了错误的步进，否则会跳过位置。 

什么时候$2k + 1 \ge n$，第一个选择的中心$l + k$到达或超过数组末尾。 在这种情况下，第一个操作已经覆盖了每个位置，并且循环在一次迭代后立即终止。 

什么时候$n = 1$，循环运行一次并选择位置 1，无论什么，它都可以轻松满足完全覆盖$k$。
