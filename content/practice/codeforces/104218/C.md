 ## 解决方案
 ## 问题理解

 我们有 n 只狗，它们被放置在围成一圈的 n 个等距点上。 狗 i 在时间 0 从位置 i 开始，每条狗每单位时间以固定步长 v_i 顺时针向前移动。 因为围绕圆的运动是模数的，所以位置总是以 n 为模。 

在任何时间 t，每只狗在圆圈上都有一个确定的位置。 我们被要求找到所有狗同时占据相同位置的最早时间 t。 如果这样的时间从未出现在允许的范围内（实际上以 1000 为界），我们输出 -1。 如果存在多个解决方案，我们返回最小的 t 以及该 t 的共同位置。 

关键的观察结果是，每只狗都在大小为 n 的循环组上遵循线性运动，因此问题在于同步以 n 为模的算术级数。 

约束 n ≤ 1000 和 v_i ≤ 100 表明 O(n^2) 或 O(n^2 log n) 方法是可以接受的。 任何 n 的三次方或涉及在一段时间内重复进行最多 1000 个步骤的完整模拟也属于边界，但如果仔细实施，仍然可能是可以接受的。 

当人们认为仅检查成对碰撞或仅跟踪一只参考狗就足够时，就会出现天真的陷阱。 例如，两只狗可能会在较早的某个时间点相遇，但这并不能保证所有狗同时相遇。 另一个常见的错误是模拟直到所有狗在时间 t=1000 匹配时没有仔细检查中间状态，这可能会错过早期的有效解决方案。 

具体的边缘情况是，由于模块化环绕，狗仅在非零时间对齐。 例如，在小周期中，同步通常会在几次旋转后发生，而不是立即发生，因此即使稍后存在解决方案，仅检查 t=0 或 t=1 也会失败。 

## 方法

 直接强力模拟考虑从 0 到 1000 的每个时间步长 t 并计算所有 n 个位置。 对于每个 t，我们检查所有位置是否相同。 每个时间步计算位置的成本为 O(n)，因此总复杂度为 O(1000·n)。 当 n 达到 1000 时，这大约需要 10^6 次操作，这在 Python 中实际上没问题，但前提是实现得干净利落。 然而，这仍然会进行冗余计算，因为每个位置都可以增量更新。 

更深层次的结构是，每只狗的位置是一个模线性函数：

 pos_i(t) = (i + t·v_i) mod n。 

我们正在寻找一个时间 t 使得所有这些表达式变得相等。 我们可以重新表述相对于所选参考狗（例如狗 0）的条件，而不是每次都检查所有位置。如果所有狗都一致，则对于每个 i：

 (i + t·v_i) == (0 + t·v_0) (mod n)

 这成为一个模线性同余系统：

 t·(v_i − v_0) ≡ −i (mod n)

 每个 i 给出对 t 的约束。 解决方案是所有这些同余的交集。 我们可以使用模块化线性方程求解器迭代地维护 t 的单个同余式（本质上是通过扩展 gcd 逻辑合并约束）。 因为 n ≤ 1000，模数很小，并且对所有 i 进行重复合并是可行的。 

关键的好处是，我们不是随时间搜索，而是直接以代数方式构造所有有效时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n·T) 其中 T ≤ 1000 | O(1) | O(1) | 已被接受但处于边缘|
 | 同余合并 | O(n log n + n log n) | O(n log n + n log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们固定狗0作为参考点。 我们导出对 t 的约束，以便每只狗同时匹配狗 0 的位置。 

1. 我们将狗 i 与狗 0 重合的条件表示为模方程：

 t·(v_i − v_0) == −i (mod n)。

这满足了两个位置都匹配模 n 的要求。 
2. 我们从 t 的一个简单同余开始：所有整数最初都是有效的。 
3. 我们将每只狗 i 从 1 迭代到 n−1，并将当前约束与新同余合并。 
4. 对于每个 i，我们求解线性同余 a·t ≡ b (mod n)，其中 a = (v_i − v_0) mod n 且 b = (−i) mod n。 

如果 a 为 0 模 n，则方程简化为检查 b 是否也为 0。如果不为 0，则不存在解。 
5. 如果 a 不为零，我们计算 gcd(a, n) 并检查一致性：b 必须能被 gcd(a, n) 整除。 如果不是，则不存在解决方案。 
6. 我们通过除以 gcd 来简化方程，然后使用扩展欧几里得计算 a/g 模 n/g 的模逆。 这产生了 t 模 n/g 的基本溶液。 
7. 我们使用两个线性同余的标准 CRT 样式组合将此解决方案与当前全局同余合并。 
8. 处理完所有狗后，我们要么获得单个同余 t ≡ x (mod M)，要么确定不存在解。 
9. 最终答案是允许限度（≤ 1000）内最小的非负 t。 我们还使用任何狗的公式计算相应的位置。 

### 为什么它有效

 所有狗必须在时间 t 时在某个位置 p 重合，因此每只狗对 t 模 n 施加线性约束。 这些约束定义了算术级数的交集。 合并过程保持当前同余恰好代表满足迄今为止所有处理过的狗的所有时间的不变量。 因为每次合并都保留与先前系统的等效性并强制执行新的约束，所以最终的同余准确地描述了有效时间集。 如果交集为空，则不存在全局同步。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def egcd(a, b):
    if b == 0:
        return a, 1, 0
    g, x1, y1 = egcd(b, a % b)
    return g, y1, x1 - (a // b) * y1

def mod_inv(a, m):
    g, x, _ = egcd(a, m)
    if g != 1:
        return None
    return x % m

def merge_congruence(r1, m1, r2, m2):
    # solve: x ≡ r1 (mod m1), x ≡ r2 (mod m2)
    # returns (r, m) or (None, None)
    g, p, q = egcd(m1, m2)
    diff = r2 - r1
    if diff % g != 0:
        return None, None

    lcm = m1 // g * m2

    # adjust solution
    t = (diff // g) * p % (m2 // g)
    x = (r1 + m1 * t) % lcm
    return x, lcm

def solve():
    n = int(input())
    v = list(map(int, input().split()))

    r, m = 0, 1  # t ≡ r (mod m)

    for i in range(n):
        a = (v[i] - v[0]) % n
        b = (-i) % n

        if a == 0:
            if b != 0:
                print(-1)
                return
            continue

        g, _, _ = egcd(a, n)
        if b % g != 0:
            print(-1)
            return

        n_ = n // g
        a_ = a // g
        b_ = b // g

        inv = mod_inv(a_ % n_, n_)
        if inv is None:
            print(-1)
            return

        x = (b_ * inv) % n_

        # merge t ≡ x (mod n_) with current
        r, m = merge_congruence(r, m, x, n_)
        if r is None:
            print(-1)
            return

    # smallest valid t
    ans_t = r
    if ans_t > 1000:
        print(-1)
        return

    pos = (0 + ans_t * v[0]) % n
    print(ans_t, pos)

if __name__ == "__main__":
    solve()
```该代码为有效时间构建了全局一致性。 每只狗贡献一个模线性方程，该方程使用扩展 gcd 求解，然后使用广义 CRT 合并进行合并。 最后一步检查时间限制并使用第一只狗的轨迹计算共享位置。 

一个微妙的点是处理系数以 n 为模为零的情况。 在这种情况下，我们必须验证右侧也为零； 否则系统会不一致并且没有时间。 

## 工作示例

 ### 示例 1

 输入：```
3
1 2 3
```我们使用狗 0 作为参考来跟踪约束。 

| 我| a = v_i - v_0 | b = -i 模 3 | 约束结果|
 | ---| ---| ---| ---|
 | 0 | 0 | 0 | 基地|
 | 1 | 1 | 2 | t = 2 (mod 3) | t ≠ 2 (mod 3) |
 | 2 | 2 | 1 | 一致合并|

 处理i=1后，我们得到t ≡ 2 mod 3。检查i=2，我们验证一致性并保持相同的同余。 

最终答案是t=2，位置是(0 + 2·1) mod 3 = 2。 

该跟踪显示系统如何简化为单个模块化条件而不是显式模拟。 

### 示例 2

```所有狗的动作都相同，因此同步是立即的。 

| 我| 一个 | 乙| 约束|
 | ---| ---| ---| ---|
 | 1 | 0 | 3 | 不可能|
 | 2 | 0 | 2 | 不可能|

 由于 a=0 但 b≠0，因此不存在解。 

这演示了简并方程的临界一致性检查。

 ## 边缘情况

 当所有 v_i 相等时，会出现一种重要的边缘情况。 在这种情况下，位置永远保持均匀分布。 用于输入`n=4, v=[2,2,2,2]`，每个方程都变为 0·t ≡ -i mod 4，这对于任何 i≠0 都是不可能的。 该算法正确地检测到了这一点，因为对于每个 i，a 为零且 b 不为零，从而导致立即拒绝。 

另一种边缘情况是同步发生在 t=0 时。 为了`n=3, v=[0,0,0]`，所有的狗都保持固定在它们的起始位置，因此只有当所有狗的起始位置相等时它们才重合，除非 n=1，否则这是错误的。 当 i≥1 时，同余系统立即显示出不一致。 

最后一个微妙的情况是，有效解存在但超过 1000。算法仍然可以正确计算它，但在最终边界检查时拒绝它，确保符合问题的时间限制。
