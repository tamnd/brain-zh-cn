 ## 解决方案
 ## 问题理解

 我们在飞机上有一组检查点。 每个检查点都有固定的位置、宝藏出现的特定时间以及该宝藏的附加值。 参赛者从零时刻的原点开始，向任意方向连续移动，但其速度受到限制，因此单位时间内最多只能走一个单位的距离。 只有在宝藏出现的确切时间他们正好到达检查站的坐标，他们才能收集宝藏。 

任务是选择一系列要收集的宝藏，以便尊重移动限制并使总收集价值最大化。 

这本质上是一个具有几何旅行约束的调度问题。 每件宝藏都成为一项“工作”，只有我们能够及时从之前选择的工作到达它的位置，才能接受它。 

最大 5000 的约束 N 是此处的关键信号。 朴素的 O(N3) 甚至一些 O(N2 log N) 方法是边界性的，但只有当常数因子很小时才可能可接受。 然而，任何尝试显式模拟路径或在没有结构的情况下重复重新计算可达性的解决方案都不会通过。 

一些边缘情况值得关注。 

一种是多个宝藏共享同一时间但相距较远。 例如，如果在时间 10 的位置 (0,0) 和 (100,100) 处出现两个检查点，则只能选择一个，而天真的方法可能会错误地假设这两个检查点从一开始就可以独立到达。 

另一种情况是宝藏在零时间或很早的时间出现。 如果检查点位于 (0,0,0)，则它始终可以立即收集，但某些转换取决于对零时间差的仔细处理。 

第三种微妙的情况是，后来的宝藏在几何上接近，但在时间上太早。 例如，从 (0,0,10) 到 (1,1,11) 是不可能的，因为即使空间间隙很小，所需的行进距离也超过了可用时间。 

这些示例强调，除非我们明确强制可达性，否则仅按时间排序是不够的。 

## 方法

 思考该问题的一种直接方法是对检查点子集进行动态规划。 我们将 dp[S] 定义为通过按有效顺序选择子集 S 可获得的最佳值。 对于每次转换，我们都会检查是否可以从一个选定的检查点移动到另一个检查点，同时遵守速度限制。 这会立即导致指数复杂度，因为子集的数量为 2^N，这对于 N = 5000 来说是完全不可行的。 

我们可以显着简化状态。 我们观察到任何有效路线都是检查点序列，而不是跟踪子集，一旦我们修复了序列中的最后一个检查点，唯一相关的历史记录就是紧接其之前的哪个检查点。 这表明端点上的 DP：dp[i] 是在检查点 i 结束的有效路由的最大值。 

关键的难点是确定检查点j是否可以跟随检查点i。 我们需要检查点之间的旅行时间是否足以覆盖欧几里德距离。 即|Ti - Tj| 必须至少为 sqrt((Xi - Xj)² + (Yi - Yj)²)，并且时间必须向前移动，因此 Ti < Tj。 

一旦看到这一点，问题就变成了有向无环图中的最长路径，但没有明确给出边。 我们通过检查所有对来计算转换。 由于 N 为 5000，因此 O(N²) DP 是可以接受的。 

强力对检查之所以有效，是因为每个有效转换都是独立的并且不需要中间状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 DP | O(2^N) | O(2^N) | O(2^N) | O(2^N) | 太慢了|
 | 端点上的成对 DP | O(N²) | O(N) | 已接受 |

 ## 算法演练

我们以时间递增的顺序处理检查点，因为任何有效的转换都必须尊重时间单调性。 

1. 通过增加 T 对所有检查点进行排序。这确保了当我们计算 dp[i] 时，所有可能的前驱点都已被处理。 
2. 对所有 i 初始化 dp[i] = V[i]。 如果我们从检查点 i 本身开始一条路径，这表示最佳值。 
3. 对于从左到右的每个检查点 i，我们尝试扩展所有先前的检查点 j < i。 
4. 对于每一对 (j, i)，我们通过比较欧氏距离平方和时间差平方来检查从 j 移动到 i 是否可行：

 (Xi - Xj)² + (Yi - Yj)² ≤ (Ti - Tj)²。 
5. 如果条件成立，我们更新 dp[i] = max(dp[i], dp[j] + V[i])。 

每次更新都代表选择 j 作为 i 之前最后收集的宝藏。 这个不等式确保单位速度的旅行者可以在物理上及时从 j 到达 i。 

处理完所有i后，答案就是dp中的最大值。 

### 为什么它有效

 在任何检查点 i，dp[i] 表示在 i 处精确结束的所有有效序列中可实现的最佳值。 按时间排序可保证所考虑的每个前驱 j 的时间都严格小于或相等，因此不存在无效的未来依赖关系。 转换条件强制物理可达性，因此 DP 中的每个边都对应于有效的移动。 由于每条有效路径都有最后一个检查点，并且 dp 考虑每个检查点的所有可能的前驱路径，因此不会错过最佳路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    pts = []
    for _ in range(n):
        x, y, t, v = map(int, input().split())
        pts.append((t, x, y, v))

    pts.sort()
    
    dp = [0] * n
    ans = 0

    for i in range(n):
        ti, xi, yi, vi = pts[i]
        dp[i] = vi
        best = vi

        for j in range(i):
            tj, xj, yj, vj = pts[j]
            dt = ti - tj
            dx = xi - xj
            dy = yi - yj
            if dx * dx + dy * dy <= dt * dt:
                if dp[j] + vi > dp[i]:
                    dp[i] = dp[j] + vi

        ans = max(ans, dp[i])

    print(ans)

if __name__ == "__main__":
    main()
```代码首先按时间对检查点进行排序，以便 DP 转换始终及时向前移动。 dp 数组存储在每个检查点结束时可获得的最佳奖励。 对于每对检查点，它使用平方距离检查可达性，以避免浮点精度问题。 

一个微妙的点是使用 dt * dt 而不是 sqrt。 这可以避免精度错误并将所有内容保留为整数，这对于坐标值高达 10^9 的情况是必要的。 

答案被取为最大 dp 值，因为最佳路径可能在任何检查点结束。

 输入：```
3

2 2 40 8

```按时间排序后，顺序为：

 (20,20,25,1000),(2,2,40,8),(1,1,100,10)

 我们一步步计算dp。 

| 我| 点| dp[i] 初始化 | 有效前任 | dp[i] 最终 |
 | --- | --- | --- | --- | --- |
 | 0 | (20,20,25) | 1000 | 1000 无 | 1000 | 1000
 | 1 | (2,2,40) | 8 | 0 无法访问 | 8 |

 最佳路径是仅按时间顺序获取可到达的第一个事件，然后选择最佳独立值。 理论上，最佳结果是 1000 + 8 + 10，但可达性会阻止链接，因为相对于距离而言，时间间隙不足。 

这表明了几何约束支配贪婪直觉的重要性。

 ## 边缘情况

 一个关键的边缘情况是两个检查点具有相同的时间戳。 该算法永远不允许它们之间的转换，因为 dt 变为零，并且除非坐标相同，否则距离检查将失败。 例如，(0,0,5,10) 和 (1,1,5,20) 不能链接。 DP 独立处理两者，产生与物理约束相匹配的 max(10,20)。 

另一种情况是检查点在空间上非常接近，但稍早。 假设 (0,0,0) 到 (100,0,1)。 距离为 100，但只有 1 个时间单位可用，因此不等式失败，并且 dp 不会错误传播。 

第三种情况是存在长链但需要中间节点时。 DP 确保正确性，因为一旦中间检查点成为到达后面节点的最佳选择，它就已经存储在 dp 中，并将在以后的转换期间重用。
