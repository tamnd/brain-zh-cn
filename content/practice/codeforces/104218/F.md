 ## 解决方案
 ## 问题理解

 我们得到一组事件，每个事件都位于 2D 平面上的一个点并在特定时间发生。 每个事件也都有其价值。 玩家从零时刻的原点开始，可以在平面上以单位速度连续移动，即移动一段距离$d$准确地要求$d$时间。 

任务是选择事件的子集，以便玩家可以从原点开始，按照某种顺序在所需的时间和地点准确访问每个选定的事件，并最大化其值的总和。 

换句话说，每个事件都是一个带有时间戳和加权奖励的节点，我们希望获得最佳的可行序列，其中连续节点之间的行程时间约束得到尊重。 

关键约束是$N \le 5000$，这会立即排除任何立方或更差的解决方案。 具有中间检查的天真的全对动态编程需要考虑事件对之间的所有转换，这表明$O(N^2)$是自然目标上限。 然而，即使$O(N^2)$必须谨慎实施，因为每次转换都涉及计算欧几里德距离和时间可行性检查。 

一个微妙的问题是事件不能保证按时间排序。 如果我们忘记排序，我们可能会尝试从较晚的事件过渡到较早的事件，这在物理上是不可能的。 

当两个事件具有相同的时间戳但空间上相距很远时，就会出现另一个极端情况。 一个不严格执行可行性的天真的 DP 可能会错误地链接它们。 例如，如果事件 A 和 B 都在时间 10 发生，但相距 100 个单位，则它们不能同时被顺序访问，因为在零时间内不可能进行移动。 

最后一个微妙的情况是浮点精度。 由于距离涉及平方根，粗心的实现可能会直接比较浮点值并引入精度误差。 稳健的解决方案通过比较平方距离来完全避免平方根。 

## 方法

 蛮力的想法是将每个事件视为一个决策点，并尝试访问它们的所有可能的子集和排列。 对于每个序列，我们将检查连续事件之间是否满足旅行时间约束并计算总值。 这是正确的，但完全不可行。 即使将我们限制在排列中，排序的数量也是$N!$，甚至每个订单的一次可行性检查都是天文数字。 

更结构化的暴力是对子集的动态编程，其中我们使用子集掩码将 DP[mask][i] 定义为在事件 i 处结束的最佳值。 这仍然是指数级的$N$，因为单独的状态空间是$O(2^N N)$，这远远超出了任何限制。 

关键的观察结果是，转换的可行性仅取决于事件之间的成对兼容性。 如果我们以递增的时间固定事件的顺序，那么任何有效的路径都必须遵循这个顺序。 来自较早的事件$i$到稍后的活动$j$，我们只需要检查玩家是否可以从$i$到$j$及时$T_j - T_i$。 这将问题简化为有向无环图中的最长路径，因为边仅在时间上前进。 

一旦我们按时间排序，该结构就成为节点上的 DP，其中每个节点都尝试扩展所有先前的可到达节点。 这给出了一个干净的$O(N^2)$过渡DP。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| 指数| 太慢了|
 | 排序事件上的最优 DP |$O(N^2)$|$O(N)$| 已接受 |

 ## 算法演练

 1. 按时间对所有事件进行排序$T_i$。 这确保任何有效的转换仅按照 DP 顺序前进，这是必要的，因为时间沿着任何可行路径严格增加。 
2. 对于每个事件$i$，如果我们在以下位置结束路径，则计算其最佳可实现值$i$。 我们将其存储在 DP 数组中，其中 DP[i] 表示在事件 i 处结束的最大奖励。 
3. 用事件 i 本身的值初始化 DP[i]。 这对应于开始一条从原点直接到达 i 的新路线。 
4. 对于每对事件$i < j$，检查是否可以从i移动到j。 这需要验证从 i 到 j 的行程时间最多为$T_j - T_i$。 我们计算平方距离以避免浮点错误并将其与$(T_j - T_i)^2$。 
5. 如果转换有效，则将 DP[j] 更新为 DP[j] = max(DP[j], DP[i] + V_j)。 这体现了这样的想法：如果可达，任何以 i 结束的最佳路径都可以扩展到 j。 
6. 处理完所有对后，答案就是 DP 数组中的最大值。 

为什么隐式处理原点：我们将每个事件从一开始就视为可达，因为在时间 0 从 (0,0) 到达事件 i 是可行的，如果$X_i^2 + Y_i^2 \le T_i^2$。 这可以在初始化 DP[i] 时显式检查，或者通过允许 DP[i] 以 V_i 启动来统一处理，因为在 DP 排序中永远不会正确扩展不可达状态。 

### 为什么它有效

 一旦事件按时间排序，任何可行的路线都必须遵循递增的索引。 DP 保持不变式，即 DP[i] 是恰好在事件 i 处结束的任何有效路由的最佳可能值。 每次 i 到 j 的转换都会考虑在时间 T_i 之前到达 i 的所有有效方式，并检查之后是否可以在不违反速度限制的情况下到达 j。 由于每条有效路径都有一个唯一的最后一个事件，并且我们按时间顺序枚举了所有可能的前趋，因此不会错过任何最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    events = []
    for _ in range(n):
        x, y, t, v = map(int, input().split())
        events.append((t, x, y, v))
    
    events.sort()
    
    dp = [0] * n
    
    for i in range(n):
        t_i, x_i, y_i, v_i = events[i]
        dp[i] = v_i
        
        # try to reach i from origin implicitly
        if x_i * x_i + y_i * y_i <= t_i * t_i:
            dp[i] = max(dp[i], v_i)
        
        for j in range(i):
            t_j, x_j, y_j, v_j = events[j]
            
            dt = t_i - t_j
            dx = x_i - x_j
            dy = y_i - y_j
            
            if dx * dx + dy * dy <= dt * dt:
                dp[i] = max(dp[i], dp[j] + v_i)
    
    print(max(dp))

if __name__ == "__main__":
    solve()
```核心结构是经典的时间间隔动态规划。 排序确保我们永远不会尝试无效的向后时间转换。 

DP 初始化将每个事件设置为独立路径。 这很重要，因为即使一个事件无法从原点到达，它仍然可以从另一个事件间接到达，所以我们不会提前丢弃它。 

嵌套循环检查所有先前事件并使用平方欧几里德距离执行可行性检查。 与平方时间差的比较至关重要，因为它完全避免了浮点精度问题。 

最终的 DP 最大值反映了这样一个事实：最佳路线可能在任何情况下结束，而不一定是按时间顺序的最后一条。 

## 工作示例

 ### 示例 1

 输入：```
3
1 1 100 10
2 2 40 8
20 20 25 1000
```按时间排序后，顺序变为：

 (20,20,25,1000),(2,2,40,8),(1,1,100,10)

 我们一步步计算DP。 

| 我| 活动 | 最佳前任| DP[i] |
 | ---| ---| ---| ---|
 | 0 | (20,20,25,1000) | 起源 | 1000 | 1000
 | 1 | (2,2,40,8) | 起源 | 8 |
 | 2 | (1,1,100,10) | 从 i=0 或 1 | 18 | 18

 最佳路径是采取高价值的早期事件，然后转向稍后可到达的事件。 

这表明按时间排序允许仅链接物理上有效的动作并避免无效的反向推理。 

### 示例 2

 输入：```
2
15 20 25 100
7 24 25 50
```两个事件同时发生。 他们的距离是$\sqrt{(15-7)^2 + (20-24)^2} = \sqrt{80}$，它是正数，而时间差为零，因此它们之间不可能有过渡。 

| 我| 活动 | DP[i] |
 | ---| ---| ---|
 | 0 | (15,20,25,100) | 100 | 100
 | 1 | (7,24,25,50) | 50 | 50

 答案是100。 

这表明同等时间事件无法正确链接。

 ## 边缘情况

 一种边缘情况是事件同时发生。 由于行程时间为零，因此只有相同的坐标才能允许链接。 DP 正确处理这个问题，因为条件$dx^2 + dy^2 \le dt^2$变成$dx^2 + dy^2 \le 0$，迫使职位平等。 

另一个边缘情况是可以从源单独访问但不能相互访问的事件。 DP 仍然正确地选择最佳的单个事件或组合，因为每个事件都以自己的值开始，并且只有有效的转换才能改进它。 

最后的边缘情况是大坐标值。 使用平方整数可以安全地适合 Python 中的 64 位算术，并且完全避免浮点问题，即使在最大约束下也能确保正确性。
