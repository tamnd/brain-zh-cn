 ## 解决方案
 ## 问题理解

 我们得到了一个有向结构$N$村庄。 每个村庄都有两条出路：一条标记为左，一条标记为右，每条都指向某个（可能相同或不同）村庄。 

训练课程由起始村庄和整数定义$K$。 从这个起点开始，巴尔托按照固定的长度模式移动$4K$：他占据了左边的边缘$K$次，然后是右边缘$K$次，然后又离开$K$次，终于又对了$K$次。 每次移动都会应用于当前村庄，因此路径完全是通过重复跟踪传出边缘来确定的。 

任务是回答最多$10^5$这样的查询有效，其中每个查询可能有$K$一样大$10^9$。 

这些约束立即排除了逐步模拟每个查询的可能性。 单个查询最多可能需要$4K$过渡，在最坏的情况下是$4 \cdot 10^9$步骤。 乘以$10^5$查询使得任何直接模拟都变得不可能。 

该结构不是任意的图遍历； 每个节点恰好有两个出边，因此每个查询只是重复应用两个确定性函数（左和右）。 

一些微妙的情况很重要：

 一个问题是左右过渡何时形成短周期。 例如，节点可能在两条边上循环到自身。 在这种情况下，简单的模拟会浪费时间，但仍然会产生正确的结果。 

另一个问题是交替模式是固定且对称的。 当左右路径以不同方式与循环相交时，假设分段之间独立的粗心优化可能会被破坏。 例如，如果左和右都映射到同一周期但具有不同的入口点，则在不同步的情况下单独处理它们可能会导致状态不一致。 

最后，因为$K$很大，任何解决方案都必须避免迭代$K$直接次。 任何线性依赖于的方法$K$每个查询立即不可行。 

## 方法

 直接蛮力方法遵循字面定义。 对于每个查询，我们模拟$K$向左移动，然后$K$正确的动作，然后再一次$K$左移，最后$K$正确的动作。 每次移动都是使用存储的邻接的单个指针跳转。 

这是正确的，因为它完全反映了问题中描述的过程。 然而，其成本却令人望而却步。 每个查询执行$4K$过渡，并且与$K$最多$10^9$，即使是单个查询也可能超出时间限制几个数量级。 

关键的观察结果是顺序不是任意的； 它是左和右两个确定性函数的重复。 重要的不是一步步的轨迹，而是这些函数多次组合的结果。 

我们不考虑单个动作，而是将左和右视为功能图。 我们需要快速重复应用这些功能。 这是二进制提升的经典设置。 

我们预先计算每个节点的结束位置$2^j$left 的应用，并分别在其结束后$2^j$权利的应用。 一旦构建完成，任何大量重复的向左或向右移动都可以在$O(\log K)$时间。 

每个查询由四个块组成$K$左右交替移动。 我们可以使用二进制提升来计算每个块的结果，并将它们按顺序链接起来。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(QK)$|$O(1)$| 太慢了|
 | 最佳（二元提升）|$O(N \log K + Q \log K)$|$O(N \log K)$| 已接受 |

 ## 算法演练

 我们将左右移动视为同一图表上的两个独立的跳跃函数。 

1. 预先计算左右过渡的二进制提升表。 

对于每个节点$v$， 店铺$upL[j][v]$= 之后到达的节点$2^j$向左移动自$v$，并且类似地$upR[j][v]$为了正确的行动。 
2. 初始化基准面$j = 0$使用给定的边。 

这一步对直接转换进行编码。 
3. 使用组合构建更高级别。 

对于每个$j > 0$， 定义$upL[j][v] = upL[j-1][ upL[j-1][v] ]$，对于右边也同样如此。 

这是有效的，因为应用$2^j$移动等于两个连续的块$2^{j-1}$移动。 
4. 对于每个查询，从给定的村庄开始。 
5. 申请$K$左移动使用二元升降台进行左过渡。 

分解$K$成二的幂并相应地跳跃。 
6. 申请$K$right 使用右表从结果节点移动。 
7. 再次重复步骤 5 和 6，因为完整的图案是左、右、左、右。 

每个块都是独立的，因为一个段之后的状态将成为下一个段的输入。 

### 为什么它有效

 每种运动类型都定义了从节点到节点的确定性函数。 二元提升通过将大指数分解为 2 的幂来有效地计算重复函数组合。 组合属性确保正确性：应用$2^a + 2^b$步骤相当于应用$2^a$接下来的步骤$2^b$步骤，因此预先计算的跳跃正确地代表了所有可能的步数。 由于查询只是按顺序应用的四个函数幂的序列，因此链接这些结果可以保留与原始过程的精确等价性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

LOG = 31  # enough for K up to 1e9

def build_up(n, nxt):
    up = [[0] * (n + 1) for _ in range(LOG)]
    for v in range(1, n + 1):
        up[0][v] = nxt[v]
    for j in range(1, LOG):
        for v in range(1, n + 1):
            up[j][v] = up[j - 1][up[j - 1][v]]
    return up

def lift(up, v, k):
    bit = 0
    while k:
        if k & 1:
            v = up[bit][v]
        k >>= 1
        bit += 1
    return v

def main():
    n = int(input())
    left = [0] * (n + 1)
    right = [0] * (n + 1)

    for i in range(1, n + 1):
        l, r = map(int, input().split())
        left[i] = l
        right[i] = r

    upL = build_up(n, left)
    upR = build_up(n, right)

    q = int(input())
    out = []

    for _ in range(q):
        v, k = map(int, input().split())
        v = lift(upL, v, k)
        v = lift(upR, v, k)
        v = lift(upL, v, k)
        v = lift(upR, v, k)
        out.append(str(v))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```该实现将预处理和查询处理完全分开。 这`build_up`函数为给定的转换数组构造二进制提升表。 它被重复用于左边缘和右边缘，从而保持逻辑紧凑并避免重复。 

这`lift`函数对预先计算的表应用二次方分解。 它迭代一些位$k$，仅当相应位被设置时才应用跳转。 这避免了显式构造二进制表示。 

每个查询按顺序应用四个提升的转换。 顺序很重要，因为每个段都从前一个段的结果开始。 

一个常见的陷阱是试图将左右合并为一个结构。 这是因为段之间的转换函数发生了变化； 它们不可互换。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(N \log K + Q \log K)$| 构建二进制提升表并使用 log-K 跳转回答每个查询 |
 | 空间|$O(N \log K)$| 存放两个升降台用于左右过渡|

 预处理成本按线性比例缩放$N$乘以表示所需的位数$K$，这足以满足$N \le 10^5$。 每个查询都是对数$K$， 所以$10^5$查询在时间限制内保持高效。

 ## 边缘情况

 无处不在的自循环，其中每个节点的左右都指向自身，产生了一个简单的系统。 从任何地方开始，任意次数的移动都会保持状态不变。 该算法可以处理此问题，因为二进制提升表在每个级别都会折叠为标识映射，因此每个提升的查询都会返回原始节点。 

双节点交换循环，其中左右两个节点交换，强调重复组合。 尽管每个函数都很简单，但重复求幂必须正确保留奇偶校验行为。 提升表捕获了这一点，因为重复平方自然地编码循环奇偶校验，并且每个查询减少到检查$K$。 

从同一节点开始左右转换形成不同循环的情况测试是否保留顺序应用。 由于我们在每个分段之后重新计算当前节点，因此状态可以通过不同的功能图正确地演化，而无需混合转换。
