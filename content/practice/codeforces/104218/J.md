 ## 解决方案
 ## 问题理解

 我们得到了一个有向图$n \le 20$标记为城市，其中已经存在一些单向道路。 预期目标是确保有一条有效的“三月路线”，按从 1 到 1 的顺序访问城市$n$，这意味着对于每个连续的对$i \to i+1$，我们必须能够沿着有向边移动$i$到$i+1$。 

我们可以进行两种修改。 首先，我们可以以成本在任意有序城市对之间建造一条新的有向道路$c$。 其次，我们可以重新标记城市，但重新标记是有限制的：一个城市$i$可以带标签$j < i$自由，但如果使用更差的标签，意味着更大的数量，我们必须支付成本$d$。 这会在通过边修改图形结构和通过标签修改排序之间产生张力。 

任务是选择城市的重新标记（符合规则）并决定依赖或构建哪些边，以便最终的标记图支持从 1 到 的哈密顿路径$n$，最小化总成本。 

就以下方面而言，限制非常小$n$，这立即表明指数状态探索是可以接受的。 和$n \le 20$，任何枚举城市子集或排列的解决方案都是可行的，只要它能避免最坏情况下的阶乘爆炸。 这强烈指向位掩码动态编程公式或子集 DP。 

当现有图已经支持完整链但重新标记比构建缺失边更便宜时，就会出现微妙的边缘情况。 另一种情况是，由于成本不对称，在实践中不允许重新标记，从而使解决方案退化为纯粹的图增强。 

天真的思维的一个更具体的失败模式是假设我们必须保留标签的原始顺序。 例如，如果存在边$1 \to 3$,$3 \to 2$， 和$2 \to 4$，一种幼稚的方法可能会尝试修复局部边缘，但最佳解决方案可能会重新标记城市以使路径变得简单，从而避免昂贵的边缘建设。 

## 方法

 暴力解释是考虑所有可能的城市最终标签，然后检查是否可以使用现有边加上可选添加的边将结果顺序制成有效路径。 对于每个标记，我们将计算重新标记的成本加上丢失边缘的成本。 检查可行性需要验证沿固定长度链的连接性$n$，即$O(n)$，但枚举所有重新标记是$O(n!)$，即使对于$n = 20$。 

关键的观察是我们实际上不需要明确地推理排列。 重要的是哪些城市子集已经被赋予了“好”标签而没有惩罚结构冲突，以及我们如何在它们之上建立一条路径。 由于目标结构是单链，因此我们可以考虑从左到右逐步构建链，并在每一步决定哪个城市占据下一个位置。 

在位置$i$，我们选择一个未使用的城市$v$。 如果之前放置的城市已经有优势$u$到$v$，我们无需支付任何费用。 如果没有，我们必须付款$c$来构建它。 另外，分配违反原始排序约束的标签会根据有多少城市被迫陷入更糟糕的位置而受到惩罚。 

这自然会导致位掩码 DP，我们在其中跟踪到目前为止已放置的城市子集以及最后放置的城市。 过渡成本包括边缘建设成本加上因将城市置于与其原始指数不一致的位置而引起的重新标记惩罚。 

因为$n$很小，一个$O(n^2 2^n)$DP就够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 |$O(n!)$|$O(n)$| 太慢了|
 | 位掩码 DP 超过订购 |$O(n^2 2^n)$|$O(n 2^n)$| 已接受 |

 ## 算法演练

 我们将问题重新解释为构建所有城市的有序序列，其中将城市放置在其索引之前的位置是免费的，但将其放置在其索引之后的成本$d$每单位班次。 

1. 定义DP状态为$dp[mask][i]$， 在哪里`mask`是已按最终顺序排列的城市集合，$i$是最后放置的城市。 此状态表示构建结束于的有效前缀的最低成本$i$。 
2. 通过设置初始化基本情况$dp[1 << i][i] = 0$对于所有城市$i$。 这相当于从任何单个城市开始序列而无需任何成本。 
3. 对于每个州$(mask, i)$，尝试通过选择城市来扩展序列$j$不在`mask`。 这将创建一个过渡到$(mask \cup \{j\}, j)$。 
4. 分两部分计算转换成本。 首先检查是否有向边$i \to j$已经存在。 如果没有，增加成本$c$来建造它。 其次，计算重新标记惩罚：放置城市$j$在位置$|mask| + 1$可能会违反其原始索引顺序，因此我们添加$d \cdot \max(0, (|mask| + 1) - j)$。 
5.用每个新状态的最小成本更新DP。 
6.填写DP后，答案是所有结束城市中的最小值$i$的$dp[(1<<n)-1][i]$。 

关键思想是掩码隐式编码位置顺序。 每次扩展序列时，我们都会固定一个城市的最终排名，并且我们可以直接计算该排名是否比其原始标签差。 

### 为什么它有效

 DP 保持了这样一个不变量：每个州都对应于具有明确定义成本的不同城市的有效部分排序。 每个转换都恰好附加一个未使用的城市，因此不会出现无效的排列。 因为每个可能的完全排序都可以表示为通过这些状态的路径，并且每个转换都考虑了本地的边缘构建和重新标记惩罚，所以全局最小值是通过所有完整状态上的 DP 最小化来实现的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    c, d = map(int, input().split())

    adj = [[False] * n for _ in range(n)]
    for _ in range(m):
        a, b = map(int, input().split())
        adj[a - 1][b - 1] = True

    INF = 10**18
    size = 1 << n
    dp = [[INF] * n for _ in range(size)]

    for i in range(n):
        dp[1 << i][i] = 0

    for mask in range(size):
        k = mask.bit_count()
        for i in range(n):
            if dp[mask][i] == INF:
                continue
            if not (mask & (1 << i)):
                continue

            for j in range(n):
                if mask & (1 << j):
                    continue

                nmask = mask | (1 << j)

                cost = dp[mask][i]

                if i != j:
                    if not adj[i][j]:
                        cost += c

                pos = k + 1
                if j + 1 < pos:
                    cost += d * (pos - (j + 1))

                if cost < dp[nmask][j]:
                    dp[nmask][j] = cost

    full = size - 1
    ans = min(dp[full])

    print(ans)

if __name__ == "__main__":
    solve()
```DP 表通过位掩码和最后一个节点进行索引，因此该实现确保每个前缀状态仅被考虑一次。 该位置是通过掩码的当前弹出计数加一得出的，这是安全的，因为每个转换都会将掩码大小恰好增加一。 

一个常见的陷阱是忘记重新标记成本取决于最终位置，而不是图结构。 另一个是错误地收取边缘成本，即使边缘已经存在。 邻接矩阵避免重复扫描并保持过渡$O(1)$。

 ## 边缘情况

 一个关键的边缘情况是当图表已经包含完美的链但重新标记会带来不必要的惩罚时。 例如，如果边缘已经形成$1 \to 2 \to 3 \to \dots \to n$，DP 将始终选择边缘成本为零且重新标记成本为零的路径，因为每个城市都以其自然位置出现。 

另一种情况是原始索引较小的节点被放置在后面。 DP明确补充道$d \cdot (pos - index)$，因此将节点 1 放在末尾会产生很大的惩罚。 转换仍然有效，但自动变为次优。 

第三种情况是边以相反的顺序存在。 DP 可以干净利落地处理这个问题，因为缺失的边缘总是需要花费一定的成本在本地进行修复$c$，因此反向链成为边缘构造的统一序列而不是无效状态。
