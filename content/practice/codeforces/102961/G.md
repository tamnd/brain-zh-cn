 ## 解决方案
 ## 问题陈述

 我们得到一个整数列表和一个目标值。 任务是确定列表中是否存在两个不同的元素，其总和等于目标。 如果存在这样的一对，我们必须输出它们的位置（通常是 1 索引）。 如果存在多个答案，则任何有效对都是可接受的。 如果无法形成对，我们输出这是不可能的。 

## 问题理解

 输入描述了排列在一行中的一系列值，以及充当目标总和的单个数字。 我们被要求检查是否可以在此序列中选择两个不同的位置，以便这些位置的值加起来恰好等于目标。 

输出不是值本身，而是它们在原始序列中的索引，这意味着我们在搜索时必须保留位置信息。 

从复杂性的角度来看，数组可能足够大，以至于显式检查每一对变得不可行。 简单的二次扫描会检查每对位置，这会导致在最坏的情况下进行大约 n²/2 次检查。 当 n 达到 100,000 左右时，这将变成 10^10 次操作，这远远超出了典型的时间限制。 

需要线性或近线性方法，这表明我们必须避免重复重新计算配对关系，而是重用以前看到的信息。 

在实践中，一些边缘情况很重要。 如果所有数字都相同，例如目标为 10 的 [5, 5, 5, 5]，则正确答案取决于是否至少存在两次。 仅跟踪每个值出现一次的简单方法可能会错误地失败。 另一个极端情况是根本不存在有效对，例如目标为 100 的 [1, 2, 3]，其中正确的输出是失败指示符，即使各个值是有效整数。 最后，重复的值需要小心处理，因为相同的值可以在多个位置使用，但相同的索引不能使用两次。 

## 方法

 暴力方法很简单：迭代每对索引 (i, j)，其中 i < j 并检查 a[i] + a[j] 是否等于目标。 这是有效的，因为它直接验证所有可能的组合，不会遗漏有效的对。 问题在于规模。 对于 n 个元素，这需要大约 n(n−1)/2 个加法和比较，它呈二次方增长，并且当 n 很大时变得太慢。 

关键的观察是，当我们固定一个元素 a[i] 时，可以完成该对的唯一值是 target − a[i]。 我们不必每次都搜索所有可能的伙伴，而是可以记住在扫描数组时已经看到的值。 这将问题转化为单遍，其中每个元素都会触发恒定时间查找。 

这种结构自然建议使用从值到索引的哈希映射。 当我们迭代时，对于每个元素，我们计算其补集并检查它是否已经遇到。 如果有，我们立即返回存储的索引和当前索引。 如果没有，我们会存储当前值以供将来匹配。 这避免了冗余比较并确保每个元素都被处理一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 哈希映射查找 | O(n) | O(n) | 已接受 |

 ## 算法演练

1. 初始化一个空字典，将每个数字映射到第一次出现的索引。 该结构允许恒定时间查找先前处理的值。 
2. 从左到右迭代数组，同时跟踪值及其索引。 这确保我们只考虑第二个元素在第一个元素之后的对，避免重复使用相同的索引。 
3. 对于每个元素 a[i]，计算达到目标所需的值，即 target − a[i]。 这表示与当前元素组成有效对的确切数字。 
4. 检查字典中是否已存在该所需值。 如果是，我们就找到了一个有效的对：存储的索引对应于较早的元素，当前索引完成总和。 
5. 如果未找到补集，则将当前值及其索引存储在字典中，然后继续。 这确保了未来的元素可以在需要时与其配对。 
6. 如果循环结束时没有找到任何有效对，则得出结论不存在解。 

### Why it works

At any point in the iteration, the dictionary contains exactly the set of elements that have appeared earlier in the array, each mapped to a valid index. 当我们处理一个新元素 a[i] 时，任何涉及 i 的有效解决方案都必须将其与某些 j < i 配对。 如果这样的 j 存在，那么它的值一定已经存储在字典中。 Since we check the complement before inserting the current element, we guarantee that we never pair an element with itself and that every valid pair is detected at the moment its second element is encountered.

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    a = list(map(int, input().split()))

    pos = {}

    for i, v in enumerate(a):
        need = x - v
        if need in pos:
            print(pos[need] + 1, i + 1)
            return
        if v not in pos:
            pos[v] = i

    print(-1)

if __name__ == "__main__":
    solve()
```The solution is built around a single pass over the array. 字典`pos`stores the first occurrence of each value, which is important because we want stable indices and we avoid overwriting earlier positions that might be needed for valid pairs.

 补码检查发生在插入当前值之前。 这种排序可以防止在以下情况下意外地将元素与其自身匹配`x = 2 * v`并确保我们只使用以前见过的元素。 

打印时索引会转换为从 1 开始，因为这种类型的竞争性编程问题通常需要它。

 ## 边缘情况

 常见的故障模式是覆盖较早出现的值。 考虑相同数字出现多次且有效对使用第一次出现的输入。 该算法通过仅存储字典中尚不存在的值来避免这种情况。 这可以确保保留最早的索引，这对于可能进行多个配对时的正确性至关重要。 

当目标正好是某个值的两倍时，会出现另一种微妙的情况，例如目标为 8 的 [4, 1, 4]。在处理第二个 4 时，补码也是 4，但由于前一个 4 已经存储，算法可以正确识别该对。 插入之前的查找顺序可确保同一元素永远不会与其自身配对。 

最后的边缘情况是当解决方案位于数组的最末尾时。 由于该算法在插入当前元素之前的每一步都会检查补集，因此即使是在最后一次迭代中完成的对也会立即检测到，无需任何后处理扫描。
