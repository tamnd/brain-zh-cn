 ## 解决方案
 ## 问题理解

 我们在数轴上得到一组区间，每个区间代表一个具有左端点和右端点的线段。 对于每个间隔，我们需要了解其相对于所有其他间隔在嵌套方面的位置。 

当一个音程完全位于另一个音程内部时，两个音程可以在严格的包含意义上关联起来。 对于每个区间，我们必须计算有多少个其他区间完全包含它，以及有多少个其他区间完全包含它。 

所以对于一个区间$[l_i, r_i]$，我们实际上正在计算两个数量。 一是间隔数$[l_j, r_j]$这样$l_j \le l_i$和$r_i \le r_j$。 另一个是间隔的数量，使得$l_i \le l_j$和$r_j \le r_i$。 

输入由此类间隔的列表组成。 输出是两个长度的数组$n$，与输入顺序对齐，其中每个位置报告这两个计数。 

简单的阅读建议直接比较每对间隔。 最多约$10^5$间隔，这立即意味着大致$10^{10}$最坏情况下的比较，这远远超出了两秒限制可以处理的范围。 任何显式检查所有对的解决方案都会失败。 

当间隔共享端点时会出现一个微妙的问题。 例如，两个相同的区间$[1, 5]$和$[1, 5]$根据非严格不等式定义，应该算作在两个方向上相互包含。 在这些情况下，粗心的严格比较会低估。 当仅由一个端点完成排序而不进行仔细的平局处理时，会出现另一种失败情况，这可能会导致具有相等左端点的间隔按破坏前缀信息的顺序进行处理。 

## 方法

 蛮力方法检查每对间隔并直接测试其中一个是否包含另一个。 这是正确的，因为定义纯粹是成对的并且不需要全局结构。 对于每个间隔，我们扫描所有其他间隔并根据端点执行两次比较。 这导致大约$n^2$间隔比较，每个常数时间。 

什么时候$n$长到$10^5$，比较次数达到$10^{10}$，这对于通常允许的时间限制来说太大了$10^8$简单的操作。 瓶颈不是每次比较的计算量，而是比较的绝对数量。 

关键的观察是，区间之间的包含可以简化为二维优势关系中的计数点。 每个区间都成为一个点$(l, r)$，我们需要计算有多少点位于由两个坐标上的不等式定义的区域中。 这正是一种可以通过一个维度上的排序和另一个维度上的前缀聚合来处理的结构。 

一旦间隔按一个端点排序，我们就可以维护一个数据结构来跟踪到目前为止我们已经看到的正确端点的数量。 压缩右端点上的 Fenwick 树允许我们计算有多少个区间满足以下条件：$r$在对数时间内。 通过仔细选择排序顺序，我们可以将两个包含方向转换为对此结构的前缀或后缀查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 最优（排序+芬威克树）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们使用相同的底层机制分别解决这两个计数。 

1. 首先，将每个区间及其原始索引存储在一起，以便可以按输入顺序写回结果。 这是必要的，因为我们将在处理过程中重新排序间隔。 
2. 将所有右端点压缩为密集范围。 这允许我们使用按等级索引的 Fenwick 树而不是原始值，从而保持内存和更新的效率。 
3. 要计算有多少个间隔包含给定间隔，请通过增加左端点对间隔进行排序，对于相等的左端点则通过减少右端点进行排序。 这种排序确保当我们处理一个区间时，所有可能包含该区间并且具有更小或相等左边界的区间都已经以与重复项一致的方式被考虑。 
4. 扫描排序列表，维护一个 Fenwick 树，记录每个右端点已看到的间隔数。 对于当前间隔$(l, r)$，我们需要计算至少有多少个先前看到的区间具有正确的端点$r$。 这对应于 Fenwick 树上的后缀和查询。 
5. 处理完一个区间后，将其右端点插入 Fenwick 树，以便将来查询可用。 
6. 要计算给定间隔包含多少个间隔，请重复对称过程。 通过减少左端点对间隔进行排序，对于相等的左端点通过增加右端点进行排序。 
7. 按此顺序进行扫描，再次使用 Fenwick 树。 现在对于每个间隔$(l, r)$，我们计算之前看到的区间最多有多少个具有正确的端点$r$，这是一个前缀和查询。 
8. 分别存储两个结果，最后使用保存的索引按原始顺序输出它们。 

它之所以有效，是因为将遏制解释为二维平面中的主导关系。 按一个坐标排序可确保处理顺序自动强制执行不等式的一半。 芬威克树维护第二个坐标，因此每个查询都成为前缀或后缀上的一维计数问题。 排序平局可以保证具有相同端点的间隔不会被错误地重复计算或跳过，即使在退化情况下也能保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

def solve():
    n = int(input())
    intervals = []
    rvals = []

    for i in range(n):
        l, r = map(int, input().split())
        intervals.append((l, r, i))
        rvals.append(r)

    rvals = sorted(set(rvals))
    comp = {v: i + 1 for i, v in enumerate(rvals)}

    arr = [(l, r, i, comp[r]) for (l, r, i) in intervals]

    contains = [0] * n
    contained_by = [0] * n

    arr1 = sorted(arr, key=lambda x: (x[0], -x[1]))
    bit = Fenwick(len(rvals))

    for l, r, idx, cr in arr1:
        contained_by[idx] = bit.sum(len(rvals)) - bit.sum(cr - 1)
        bit.add(cr, 1)

    arr2 = sorted(arr, key=lambda x: (-x[0], x[1]))
    bit = Fenwick(len(rvals))

    for l, r, idx, cr in arr2:
        contains[idx] = bit.sum(cr)
        bit.add(cr, 1)

    print(*contains)
    print(*contained_by)

if __name__ == "__main__":
    solve()
```该解决方案分为两次扫描，每次处理一个方向的遏制。 芬威克树用于维护已处理的右端点的计数。 后缀查询是使用总减前缀实现的，而前缀查询则直接使用。 

排序顺序是正确性的关键部分。 第一次扫描保证当我们查询“有多少个间隔至少结束”时，具有有效左端点的所有候选都已经在结构中。 第二次扫描反转了逻辑，使得“有多少个不晚于我结束”在处理后的集合中变得有效。 

坐标压缩时必须小心。 如果没有压缩，芬威克树将需要处理潜在的大坐标值，并且在内存中变得不可行。

 ## 边缘情况

 对于相同的间隔，例如$[1, 2]$重复多次后，算法在排序时依赖平局规则来确保它们以稳定的顺序进行处理。 在第一次扫描期间，每个间隔将所有先前处理的相同间隔视为有效容器，从而产生与早期重复项数量相匹配的递增计数。 

对于完全不相交的区间，例如$[1,2], [3,4], [5,6]$，芬威克树永远不会以满足支配条件的方式积累重叠的右端点。 每个查询都会正确返回零，因为没有间隔同时满足任一方向上的两个坐标约束。 

对于完全嵌套的链，例如$[1,10], [2,9], [3,8], [4,7]$，第一次扫描建立一个结构，其中每个内部间隔将所有外部间隔视为有效容器，第二次扫描反转这种关系，产生与理论嵌套深度匹配的对称递减模式。
