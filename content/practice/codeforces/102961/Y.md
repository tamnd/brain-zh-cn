 ## 解决方案
 ## 问题理解

 我们得到一个数字序列和一个目标值，任务是确定序列中是否存在四个不同的位置，以便这些位置上的值加起来恰好等于目标值。 如果存在这样的四元组，我们必须输出它们的索引； 否则我们报告不可能。 

输入可以理解为放置在一条线上的权重的集合。 我们被问到是否可以选择四个不同的权重，其总和与所需的总和相匹配，如果可以，我们还必须确定这些权重的来源。 不同索引的要求很重要，因为即使同一元素的值有助于达到总和，也无法重复使用该元素。 

从复杂性的角度来看，这个问题的自然输入大小足够大，以至于任何直接检查所有四元组的方法都变得不可行。 对四个指数的简单列举已经导致了$O(n^4)$，即使对于中等值，它也会变得天文数字般大$n$。 这立即排除了对所有四元组的暴力搜索。 问题的结构表明我们必须通过预先计算元素对之间的关​​系来以某种方式压缩搜索空间。 

当多个元素共享相同的值时，会出现微妙的边缘情况。 例如，如果数组是$[1, 1, 1, 1]$目标是$4$，使用所有四个不同的位置都存在正确的答案。 仅跟踪没有索引的值的粗心解决方案可能会错误地多次重复使用同一事件。 当对组合重用索引时，例如形成两个重叠的对，就会出现另一个问题； 必须明确防止这种情况。 

## 方法

 暴力方法会尝试四个不同索引的每种组合，并检查它们的总和是否等于目标。 这是正确的，因为它详尽地枚举了整个解决方案空间。 然而，对于每个四元组，我们执行恒定的工作，并且四元组的数量随着$\binom{n}{4}$，与$n^4$。 即使当$n$只有几千个，这在时间限制内执行起来太慢了。 

关键的观察是四元组可以分成两对。 我们不是一次搜索四个元素，而是预先计算所有对和。 每对都贡献一个总和及其索引，然后我们尝试匹配两个不相交的对，其总和等于目标。 这将问题从四维搜索减少到二维搜索。 

我们必须强制执行的主要约束是这两对不共享指数。 这是通过将索引与每对总和一起存储并检查具有互补和的所有对组合来处理的，同时确保不相交。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^4)$|$O(1)$| 太慢了|
 | 对和哈希 |$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们围绕对和和允许快速补查询的查找结构构建解决方案。 

1. 生成所有索引对$(i, j)$和$i < j$，并计算它们的总和$a[i] + a[j]$。 将每对存储在从总和到索引对列表的映射中。 这是必要的，因为多个对可以共享相同的总和，并且我们必须保留所有可能性。 
2. 迭代每对总和$s$存在于结构中。 对于每个$s$，计算所需的补集$t - s$， 在哪里$t$是目标总和。 
3. 对于每对$(i, j)$存储在总和下$s$，尝试将其与每对配对$(k, l)$存储在总和下$t - s$。 这检查两个不相交的对是否可以形成所需的总数。 
4. 在接受候选人之前，请验证所有四个指数是否不同。 这确保了我们不会在多个角色中重复使用相同的元素。 
5. 一旦找到有效的配置，输出四个索引并立即终止。 

我们可以安全地在第一场比赛中停止的原因是，该问题只需要任何有效的解决方案，而不是所有的解决方案。 

### 为什么它有效

 每个有效的解决方案都包含四个索引，这些索引可以分为两个不相交的对。 通过枚举所有对并按总和对它们进行分组，我们保证任何有效的四元组都对应于我们结构中的两个条目。 互补和之间的详尽配对确保不会遗漏任何有效组合，并且显式索引不相交检查可防止元素的无效重用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, target = map(int, input().split())
    arr = list(map(int, input().split()))

    pairs = {}

    for i in range(n):
        for j in range(i + 1, n):
            s = arr[i] + arr[j]
            if s not in pairs:
                pairs[s] = []
            pairs[s].append((i, j))

    # search for complement pairs
    for s in list(pairs.keys()):
        t = target - s
        if t not in pairs:
            continue

        list1 = pairs[s]
        list2 = pairs[t]

        for i1, j1 in list1:
            for i2, j2 in list2:
                if i1 != i2 and i1 != j2 and j1 != i2 and j1 != j2:
                    print(i1 + 1, j1 + 1, i2 + 1, j2 + 1)
                    return

    print("IMPOSSIBLE")

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个对和的字典，其中每个键存储产生该和的所有索引对。 这很重要，因为不同的索引对可能会产生相同的总和，而失去多重性将导致错过解决方案。 

第二阶段迭代这些和并搜索互补的和。 存储对上的嵌套循环确保正确性，而显式不等式检查强制索引不同。 到基于 1 的索引的转换仅在输出时发生，从而保持干净的内部逻辑。

 ## 边缘情况

 一种重要的边缘情况是当多个对共享相同的总和时，可能会导致索引的重用。 例如，在像这样的数组中$[3, 3, 3, 3, 3]$有目标$12$，许多对组合产生总和$6$。 该算法通过在接受匹配之前显式检查索引的独特性来确保正确性，从而防止无效重用。 

另一种情况是存在有效解但仅在重叠总和组内。 例如，如果相同的值出现在多个位置，则对和映射可能包含许多相同的和。 该算法仍然探索所有组合，因为它存储所有索引对，而不仅仅是每个和的一个代表性对，确保在聚合期间不会丢失有效的四元组。
