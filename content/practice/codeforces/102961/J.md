 ## 解决方案
 ## 问题理解

 我们得到了一组可以解释为硬币值的正整数。 每个值最多可以使用一次，通过选择这些硬币的一些子集，我们可以形成不同的总和。 任务是确定使用给定硬币的任何子集都无法形成的最小正整数金额。 

输入只是一个硬币值的列表。 输出是一个整数，表示从零开始的可实现总和中的第一个“间隙”，忽略零本身，因为空子集总是形成它。 

关键的困难在于子集的数量随着硬币的数量呈指数增长，因此一旦数组增长超过小尺寸，直接推理总和是不可行的。 如果硬币数量达到 10^5 左右，则任何枚举子集或对总和执行动态编程的方法都将超出时间限制，因为状态空间将爆炸超出实际限制。 

一种简单的方法会尝试生成所有子集和，然后搜索最小的缺失整数。 即使对于中等大小的输入，这也会失败。 例如，如果硬币是`[1, 2, 3, 10]`，子集和涵盖许多值，但显式构造它们已经需要跟踪最多 2^n 个组合。 

当最小的硬币大于 1 时，会出现更微妙的失败情况。例如，输入`[5, 7]`无法形成 1，因此答案立即为 1。任何在不首先检查此边界的情况下开始求和的方法都会浪费计算探索不相关的组合。 

另一个重要的边缘情况是硬币连续或几乎连续的情况。 为了`[1, 2, 3]`，每个最多 6 的数字都是可达的，缺失的值变成 7。正确的解决方案必须认识到中间结构，而不仅仅是单个硬币的贡献。 

## 方法

 蛮力视角从定义开始：计算所有子集和，然后从 1 向上扫描，直到缺少一个值。 这是正确的，因为它详尽地代表了每个可实现的配置。 然而，每添加一枚硬币，子集的数量就会加倍，从而导致大约 2^n 个状态。 一旦 n 超过 25 或 30，甚至存储这些总和也变得不可能。 

突破来自于认识到我们实际上并不需要知道每一个可实现的金额。 我们只关心从 1 到某个边界的所有值是否可以连续实现。 如果我们已经知道中的每个值`[1, x]`是可达的，然后是一个有价值的新硬币`v`只有在不产生间隙的情况下才能扩展此范围。 具体来说，如果`v`至多是`x + 1`，那么它可以填充下一个缺失的段并将可达性扩展到`x + v`。 否则，会出现间隙`x + 1`立即地。 

这将问题从子集枚举转变为对排序硬币的贪婪扫描。 每个硬币要么扩展一个连续的可达前缀，要么揭示第一个不可达的整数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力子集和| O(2^n·n) | O(2^n·n) | O(2^n) | O(2^n) | 太慢了|
 | 贪婪前缀扩展 | O(n log n) | O(n log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 将硬币按升序排列。 排序确保我们始终首先考虑最小的可用扩展，这对于维持连续的可达范围至关重要。 
2. 初始化变量`reach = 0`，表示范围内的所有总和`[1, reach]`目前是可以实现的。 最初，只能实现总和 0，因此除非扩展，否则第一个缺失的正整数为 1。 
3. 逐个迭代排序后的硬币。 对于每枚硬币的面值`v`，将其与`reach + 1`。 
4. 如果`v`大于`reach + 1`，立即停止。 这意味着存在一个间隙`reach + 1`不能用任何剩余硬币的组合来填充，因为所有剩余的硬币都更大。 
5. 否则，如果`v`至多是`reach + 1`， 更新`reach`到`reach + v`。 这是有效的，因为每个值达到`reach`以前是可以实现的，并且添加`v`允许所有值达到`reach + v`通过包含或排除该硬币而形成。 
6.处理完所有硬币或提前停止后，答案是`reach + 1`。 

### 为什么它有效

 该算法保持了以下不变式：`[1, reach]`可以使用加工硬币的子集形成。 排序保证我们总是考虑可以延长这个间隔的最小硬币。 如果一枚硬币超过`reach + 1`，后来的硬币的任何组合都无法弥补差距，因为所有未来的值都更大，因此第一个无法到达的整数是固定的。 否则，可达区间会不间断地扩展，从而保持连续性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    arr = list(map(int, input().split()))
    
    arr.sort()
    
    reach = 0
    
    for v in arr:
        if v > reach + 1:
            break
        reach += v
    
    print(reach + 1)

if __name__ == "__main__":
    solve()
```解决方案首先读取列表并对其进行排序，以便我们始终按升序处理硬币。 变量`reach`跟踪从零开始可实现的最大连续总和。 每个硬币要么被吸收到这个范围内，要么导致不连续。 

关键的实施细节是条件`v > reach + 1`。 相差一错误通常发生在这里：使用`>=`当一枚硬币恰好填补下一个缺口时，就会错误地实现收支平衡。

 ## 边缘情况

 当最小的硬币大于 1 时，就会出现关键的边缘情况。对于输入`[5, 7, 9]`，排序不会改变顺序，并且第一个硬币已经违反了`reach + 1`健康）状况。 算法立即停止并返回 1，正确识别无法形成正和。 

另一种情况是硬币开始时很密集，但后来变得稀疏。 为了`[1, 2, 3, 4, 10, 11]`，可达前缀平滑增长到 10，然后 11 处的间隙变得无关紧要，因为如果缺失，第一个中断已经发生在 5 处； 这里没有提前发生中断，因此最终答案由完全密集前缀之后的第一个大间隙决定。
