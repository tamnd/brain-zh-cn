 ## 解决方案
 ## 问题理解

 我们得到一个数字序列和一个目标值。 任务是判断序列中是否存在三个不同的元素，其总和等于目标值，如果存在，则返回它们在原数组中的位置。 

关键细节是我们不是孤立地选择值，而是选择索引，因此即使相同的数字出现多次，我们也必须确保在形成三元组时引用不同的出现次数。 输出通常是任何有效的索引三元组，不一定是所有可能性或超越存在的最佳组合。 

这种经典设置中的约束通常允许最多大约 2×10^5 元素。 这立即排除了对所有三元组进行三次和二次扫描。 三次 O(n^3) 枚举显式检查所有三元组，并且在最坏的情况下将执行大约 10^15 次操作，这远远超出了可行的限制。 即使 O(n^2) 也可能很紧，但在排序后通过仔细的两指针结构变得可行。 

当存在重复值并且简单的解决方案处理值而不是索引时，就会出现微妙的失败情况。 例如，如果数组为 [3, 3, 3, 4]，目标为 9，则正确答案是值 3 的三个不同索引。基于值的折叠重复项的方法可能会错误地认为只有一个 3 存在，并且无法找到解决方案，即使它在索引意义上是有效的。 

另一种边缘情况是当数组很小时，例如 n = 3。在这种情况下，任何解决方案都必须直接验证唯一可能的三元组，并且假设排序或两指针移动而不仔细处理边界的算法可以轻松跳过有效答案。 

## 方法

 蛮力的想法很简单：尝试索引 i、j、k 的每个三元组并检查它们的值总和是否等于目标。 这是有效的，因为它详尽地枚举了所有可能性并且绝不会错过有效的组合。 然而，对于 n 个元素，这需要大约 n 选择 3 个检查，其增长为 O(n^3)。 当 n 约为 10^5 时，这在计算上变得不可能。 

我们可以通过首先修复一个索引，然后搜索另外两个值来完成所需的总和来减少冗余。 一旦修复了一个元素，问题就简化为在剩余数组中查找两个数字，它们的总和为已知值。 该子问题是经典的二和问题，可以使用排序数组和两指针扫描有效地解决。 

关键的观察是，如果我们保留原始位置，排序不会破坏恢复索引的能力。 排序后，我们可以根据当前和是否太小或太大来向内移动两个指针。 这将内部搜索从每个固定元素的线性扫描减少到线性两指针扫描。 

暴力破解之所以有效，是因为它枚举了所有三元组，但当 n 增长时，它会失败，因为它会多次重复相同的部分和。 排序后可以在线性时间内求解二和的观察结果将整体复杂性从三次降低到二次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了|
 | 固定+两个指针| O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 我们首先对数组进行排序，但我们会跟踪原始索引，因为输出需要原始排序中的位置。

1. 将每个值与其原始索引配对并按值排序。 这使我们能够应用排序逻辑，同时仍然能够稍后报告正确的索引。 
2. 迭代数组，将每个位置 i 视为三元组的固定第一个元素。 目标是在后缀中找到两个元素，其总和等于目标减去固定值。 
3. 对于每个固定的 i，初始化两个指针，一个在 i+1 处，一个在数组末尾。 它们代表候选的第二个和第三个元素。 
4. 计算 i、左指针和右指针处的值之和。 如果这个总和与目标匹配，我们就找到了一个有效的三元组，并且可以输出它们的原始索引。 
5. 如果总和太小，向右移动左指针可增加总和。 如果总和太大，请将右指针向左移动以减小总和。 
6. 重复直到指针交叉或找到有效的三元组。 

为什么有效：排序后，数组具有单调结构。 对于固定元素，任何对和对于指针移动的行为都是可预测的。 移动左指针会增加剩余范围内的总和，移动右指针会减少剩余范围内的总和。 这保证了不会跳过任何有效对，因为每次调整都会严格消除不能包含有效解决方案的范围，同时保留所有可行的候选方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    arr = list(map(int, input().split()))
    
    a = [(arr[i], i + 1) for i in range(n)]
    a.sort()
    
    for i in range(n):
        target = x - a[i][0]
        l, r = i + 1, n - 1
        
        while l < r:
            s = a[l][0] + a[r][0]
            if s == target:
                print(a[i][1], a[l][1], a[r][1])
                return
            if s < target:
                l += 1
            else:
                r -= 1
    
    print("IMPOSSIBLE")

if __name__ == "__main__":
    solve()
```核心结构清楚地分离了关注点。 排序步骤会附加索引，这样我们就不会丢失原始位置。 外部循环固定一个元素，内部两指针循环执行对互补对的约束搜索。 

一个常见的实现陷阱是忘记排序后，索引不再对应于原始位置。 另一个微妙的问题是指针初始化：两个指针必须严格在固定索引之后开始，以避免重用相同的元素。 最后，当找到有效的三元组时，必须立即终止，因为可能存在多个解决方案，但只需要一个。 

## 工作示例

 考虑 n = 5、x = 10 且数组为 [2, 3, 7, 5, 1] 的输入。 与索引配对并排序后，我们得到一个按值排序的结构。 

我们跟踪执行情况：

 | 我（固定）| 我| r | 考虑的价值观| 总和| 行动|
 | ---| ---| ---| ---| ---| ---|
 | 0 | 1 | 4 | 1 + 2 + 3（概念上）| 太小| 移动我|
 | 0 | 2 | 4 | 下一对 | 比赛| 返回 |

 该迹线显示了当总和不足时，算法如何通过扩展最小边来快速收敛。 

现在考虑一个有重复项的情况：n = 4，x = 9，数组 [3, 3, 3, 4]。 排序后，算法固定 1 个 3，然后在剩余元素中进行搜索。 两指针扫描选择另外两个 3，生成不同索引的有效三元组。 

| 我（固定）| 我| r | 价值观 | 总和| 行动|
 | ---| ---| ---| ---| ---| ---|
 | 0 | 1 | 3 | 3 + 3 + 3 | 3 + 3 + 3 9 | 发现 |

 即使解决方案需要重复项，这也可以确认正确性。

 ## 边缘情况

 关键的边缘情况是有效解决方案使用重复值。 对于输入 n = 4、x = 9、数组 [3, 3, 3, 4]，算法排序为 [(3,1), (3,2), (3,3), (4,4)]。 固定第一个元素 (3,1)，两指针搜索对后缀进行操作。 指针最终选择 (3,2) 和 (3,3)，产生正确的输出。 这里的不变性是通过排序保留不同的索引，因为每个元素都带有其原始位置。 

当不存在解决方案时，会出现另一种边缘情况。 对于 n = 4、x = 100、数组 [1, 2, 3, 4]，指针将穷尽所有组合，而不会匹配目标总和。 外循环完成而不会提前终止，并且算法正确地打印“IMPOSSIBLE”。
