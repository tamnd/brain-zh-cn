 ## 解决方案
 ## 问题理解

 给定一个整数序列，我们需要找到该序列的连续段的最大可能和。 连续的段意味着我们选择一个起始位置和一个结束位置，并获取其间的所有元素而不跳过任何元素。 任务是计算所有这些段的最大总和，包括将单个元素作为段的可能性。 

输入表示按顺序排列的数字列表，我们可以选择其中的任何连续块。 输出是一个数字：所有这些块中可实现的最佳总和。 

Codeforces 上此类问题的约束级别几乎总是意味着数组大小高达 200,000 左右或更大，值可以是正数或负数，而且数量级很大。 这立即排除了子数组的任何二次或三次探索。 检查每对端点并对它们之间的段求和的解决方案需要 O(n²) 时间，这在 n = 2 × 10⁵ 时太慢，因为在最坏的情况下会导致大约 4 × 10^10 次操作。 

有一些边缘情况值得明确说明。 

如果所有数字都是负数，则正确答案是最大的单个元素。 例如，对于输入`-5 -2 -8`，答案是`-2`。 假设我们可以从零“重新开始”的天真的方法会错误地返回`0`如果它允许空子数组，这是不允许的。 

如果数组中混合有较大的负数和正数，则采取所有内容并不总是最佳的。 例如，在`4 -10 6`，最好的子数组是`[6]`或者`[4]`或者`[6]`取决于细分； 正确答案是`6`， 不是`0`或者`-10`。 

如果存在多个被大负值分隔的不相交的正值区域，则最佳片段会在正确的点重置，而不是通过负值间隙进行累积。 

## 方法

 蛮力策略很简单：枚举每个可能的子数组，计算其总和，并跟踪最大值。 对于每个起始索引 i，我们将结束索引 j 从 i 扩展到 n − 1，保持运行总和，这样我们就不会从头开始重新计算。 这给出了 O(n²) 个子数组，并且每次扩展都是 O(1)，导致总时间为 O(n²)。 

这是正确的，因为它明确地考虑了每个可能的连续段。 故障点纯粹是计算性的：当 n 很大时，段数呈二次方增长，程序无法及时完成。 

关键的结构观察是，当从左到右扫描时，在任何位置，我们只需要知道继续前一段是否有帮助，或者从当前元素重新开始是否会产生更好的总和。 如果运行总和变为负数，则将其结转只会损害未来的总和，因为添加负前缀会减少后续的任何段。 这将问题简化为在迭代时维持最佳“结束于此”值，并更新全局最佳值。 

这会将所有子数组的全局优化转换为仅依赖于先前状态的局部递归。

 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最优（Kadane 算法）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 最优思想：维护在每个位置结束的最佳子数组

 1.初始化两个变量：`current`跟踪以当前索引结尾的最佳子数组总和，以及`best`跟踪迄今为止看到的最佳答案。 两者都从数组的第一个元素开始。 这是必要的，因为每个有效的子数组必须包含至少一个元素，因此第一个元素定义了有效的基线。 
2. 从第二个元素开始迭代数组。 
3. 各位置`x`，决定是否扩展前一个子数组或开始一个新子数组`x`。 这是通过比较来完成的`current + x`和`x`。 如果`current`是负数，延长它只会减少该值，因此重新启动会更好。 否则，扩展会保留更大的总和。 
4. 更新`current`到所选值。 这表示必须以当前索引结尾的子数组的最佳可能总和。 
5. 更新`best`作为`max(best, current)`。 这确保我们记住在任何地方看到的最高子数组总和，因为最佳子数组可能在任何位置结束。 
6.处理完所有元素后，返回`best`。 

### 为什么它有效

 在每个索引处，算法都保持不变式：`current`is the maximum sum of any subarray that ends exactly at that index. 任何以索引 i 结尾的最佳子数组必须仅由 i 处的元素组成，或者扩展以 i − 1 结尾的子数组。如果以 i − 1 结尾的最佳子数组为负，则会减少总数，因此丢弃它并重新启动始终是最佳选择。 这个局部决策完全捕获了所有全局可能性，因为每个子数组都有一个唯一的端点，并且每个端点仅被评估一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    current = a[0]
    best = a[0]

    for i in range(1, n):
        x = a[i]
        current = max(x, current + x)
        best = max(best, current)

    print(best)

if __name__ == "__main__":
    solve()
```该代码直接实现了算法演练中描述的递归。 这`current`变量对应于以当前索引结尾的最佳子数组，并且`best`变量跟踪所有此类子数组的全局最大值。 

关键的实现细节是更新的顺序。`current`必须在更新之前计算`best`， 自从`best`取决于新形成的在每个位置结束的子数组。 这`max(x, current + x)`线是做出重新开始或延长决定的精确时刻。 

## 工作示例

 ### 示例 1：`a = [4, -1, 2, 1]`| 我| x| 当前| 最好的|

 | 0 | 4 | 4 | 4 |
 | 1 | -1 | 3 | 4 |
 | 2 | 2 | 5 | 5 |
 | 3 | 1 | 6 | 6 |

 尽管存在负值，但运行总和仍会增长，因为早期的正值部分足以吸收它。 最终的最佳值对应于整个数组。 

### 示例 2：`a = [-2, 3, -1, 2, -5]`| 我| x| 当前| 最好的|
 | ---| ---| ---| ---|
 | 0 | -2 | -2 | -2 |
 | 1 | 3 | 3 | 3 |
 | 2 | -1 | 2 | 3 |
 | 3 | 2 | 4 | 4 |
 | 4 | -5 | -1 | 4 |

 该轨迹显示了重新启动行为：在强正开始后，算法仍然继续延伸通过一个小的负数，因为它不会破坏总数，但是当最后出现一个大的负数时，最好放弃延续。

 ## 边缘情况

 全负数组是最重要的边缘情况。 考虑输入`n = 4`,`[-3, -1, -7, -4]`。 算法初始化`current = -3`,`best = -3`。 每一步，`current = max(x, current + x)`始终选择单个元素，因为添加任何先前的总和都会使其变得更负。 留下最好的`-1`，这是正确的最大子数组和。 

另一个微妙的情况是当一个大的否定出现在一个强的肯定前缀之后时。 为了`[-2, 10, -3, -2]`，迹线是`current = -2`， 然后`10`， 然后`7`， 然后`5`。 该算法绝不会不必要地重新启动，因为前缀仍然是有益的，直到它被下一个元素完全超过。
