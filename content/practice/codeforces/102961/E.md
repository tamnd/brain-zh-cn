 ## 解决方案
 ## 问题理解

 我们得到了顾客访问餐厅的时间表，每个顾客都会在某个时刻出现并在稍后的某个时刻离开。 每个顾客都贡献了他们在餐厅内的连续时间间隔。 

任务是确定任意时刻同时在场的最大顾客数量。 

重新构造输入的一个有用方法是将每个客户变成数轴上的一个段。 然后，该问题要求所有段之间的最大重叠深度。 

约束（在这个问题族中通常最多大约 2×10^5 间隔）意味着 O(n²) 方法立即不可行。 在最坏的情况下，任何将每个间隔与其他间隔进行比较的解决方案都将执行大约 10^10 次操作，这远远超出了典型的时间限制。 这将我们推向 O(n log n) 或 O(n) 方法，这是扫描线或基于排序的计数问题的标准。 

一个微妙的边缘情况来自于如何处理同时发生的事件。 考虑两个客户：一个在时间 5 离开，另一个在时间 5 到达。如果我们错误地将两者视为重叠，我们可能会多算。 例如：

 输入：```
1
1 5
5 10
```根据解释，正确答案通常是 1，因为当第二个顾客到达时，第一个顾客已经不在里面了。 在处理相同时间戳的出发之前进行递增的简单方法会错误地输出 2。 

这种模糊性迫使我们仔细定义相等时间的排序，而不是简单地仅按时间排序。 

## 方法

 蛮力方法会模拟发生变化的每个时刻。 对于每个间隔，可以迭代所有其他间隔并计算与其重叠的数量。 这是可行的，因为重叠检查很简单，但它会多次重复相同的比较。 

对于 n 个间隔，这会导致每个间隔大约进行 n 个检查，从而产生 O(n²) 总操作。 当 n 约为 200,000 时，这将完全无法使用。 

关键的观察是我们实际上并不关心任意间隔对之间的关​​系。 我们只关心活动间隔的数量如何随时间变化。 我们不是问“有多少个间隔与这个间隔重叠”，而是问“随着时间的推移，活跃计数如何演变”。 

这将问题转化为事件处理任务。 每个间隔恰好贡献两个事件：进入事件和退出事件。 如果我们按时间对所有事件进行排序并扫描它们，维护一个运行计数器，我们就可以跟踪每时每刻的活跃客户数量。 答案就是该计数器的最大值。 

唯一剩下的困难是处理同时发生的事件。 我们通过确保在相同时间戳的到达之前处理出发航班来解决此问题，以便在时间 t 离开的客户不会与在时间 t 到达的客户重叠。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 扫线（事件排序）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 将每个顾客间隔转换为两个事件：一个代表在时间 l 到达，一个代表在时间 r 离开。 这种重新表述很有用，因为它用离散状态变化代替了范围重叠推理。 
2. 将到达编码为活跃客户数量的 +1 变化，将出发编码为 -1 变化。 这使我们能够维护单个运行变量而不是重新计算重叠。 
3. 按时间对所有事件进行排序。 当两个事件共享相同时间戳时，请将出发事件放在到达事件之前。 这种排序可以防止精确边界点处的人为重叠。 
4. 从左到右扫描已排序的事件，维护活跃客户的运行计数器。 
5. 在每个事件中，使用其增量值更新计数器并跟踪迄今为止观察到的最大值。 
6. 处理完所有事件后返回最大计数器值。 

工作原理：扫描线不变式是在处理完时间 t 之前的所有事件后，运行计数器恰好等于所选边界约定下包含 t 的间隔数。 因为每个间隔都贡献了 1 +1 和 1 -1，并且排序确保了相等时间戳的正确处理，所以不会重复计算或错过任何间隔。 因此，所有扫描状态的最大值必须等于最大同时重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    events = []

    for _ in range(n):
        l, r = map(int, input().split())
        events.append((l, 1))
        events.append((r, -1))

    events.sort(key=lambda x: (x[0], x[1]))

    cur = 0
    best = 0

    for t, delta in events:
        cur += delta
        if cur > best:
            best = cur

    print(best)

if __name__ == "__main__":
    solve()
```实现的核心是事件转换。 每个间隔变成两个条目，并且我们再也不会在该点之后显式存储间隔。 排序步骤强制执行正确的时间顺序，并且嵌入的打破平局规则`(time, delta)`确保 -1 事件在同一时间先于 +1 事件处理。 

扫描本身是一次扫描。 变量`cur`代表当前有多少顾客在里面，而`best`跟踪看到的最大值。 更新顺序很重要：我们首先应用事件，然后考虑新状态是否是答案的候选状态。

 ## 边缘情况

 一种重要的边缘情况是许多间隔共享相同的端点。 如果所有顾客在时间 1 到达并在时间 10 离开，则算法必须在任何离开之前累积所有到达的顾客，达到等于 n 的峰值。 事件排序保证了这一点，因为仅当时间不同时，到达才被视为 +1 并在 -1 之后处理，而相等时间排序保持预期结构一致。 

另一个边缘情况是单个区间。 该算法正确地产生 1，因为它恰好引入了 1 个 +1 和 1 个 -1，并且扫描从未超过计数 1。 

最后一个微妙的情况是交替端点，例如 (1,2)、(2,3)、(3,4)。 正确答案是 1，并且扫描会保持这一点，因为时间 t 的每个 -1 都会在 +1 之前或旁边同时处理，从而防止在边界处人为堆叠。
