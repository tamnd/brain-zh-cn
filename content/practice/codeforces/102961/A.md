 ## 解决方案
 ## 问题理解

 该任务是获取一个整数序列并确定其中出现了多少个不同的值。 给定一个数字列表，输出是一个整数，表示这些数字形成的集合的大小，这意味着重复项将被忽略，并且仅计算唯一值。 

从计算的角度来看，输入大小是驱动解决方案选择的因素。 如果序列长度达到大约 10^5 或更长，任何将每个元素与其他元素进行比较的方法都会变得太慢，因为它需要 n² 运算的顺序，这远远超出了典型的时间限制所允许的范围。 这促使我们寻求在恒定或接近恒定的时间内处理每个元素的解决方案。 

当重复频繁或所有值都相同时，就会出现一个微妙的问题。 例如，如果输入是`5 5 5 5 5`，正确答案是`1`。 在没有正确跟踪的情况下错误地计算比较或重置每个元素的计数器的幼稚方法可能会错误地返回`5`。 另一种边缘情况是当所有元素都已经不同时，例如`1 2 3 4 5`，其中答案等于输入大小。 任何解决方案都必须一致地处理这两个极端，而无需特殊外壳。 

## 方法

 解决问题最直接的方法就是将每个元素与其他每个元素进行比较，并标记它是否以前出现过。 这种暴力策略在概念上很简单：对于每个数字，扫描数组的其余部分以查看它是否较早发生过。 它是正确的，因为它显式检查所有对之间的相等性，确保检测到重复项。 然而，对于长度为 n 的序列，这需要对每个元素进行大约 n 次检查，从而导致总共进行 n² 次比较。 当 n 很大时，这在计算上变得不可行。 

关键的观察是我们实际上不需要成对比较元素。 我们只需要一个能够记住我们已经看到的内容的结构。 一旦我们认识到问题纯粹与成员跟踪有关，我们就可以用基于哈希的结构代替重复扫描。 集合自然地维护唯一元素并支持平均恒定时间内的插入和成员资格检查。 我们不需要将每个元素与所有先前的元素进行比较，而是只需将其插入集合中并依靠结构来丢弃重复项。 

这将问题从重复比较减少到对每个元素进行恒定时间操作的数组的单次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 使用集 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们处理该序列一次，同时维护我们已经遇到的值的集合。 

1. 初始化一个空集，用于存储迄今为止看到的所有不同值。 该集合一开始是空的，因为我们还没有处理任何元素。 
2. 从左到右迭代输入序列中的每个数字。 每个元素仅被考虑一次，这确保了线性处理。 
3. 对于每个数字，将其插入集合中。 如果该数字已经存在，则该集合保持不变。 此行为至关重要，因为它会自动处理重复项，而无需额外的逻辑。 
4. 处理完所有元素后，计算集合的大小。 该大小表示遍历期间遇到的唯一值的数量。 

### 为什么它有效

 在迭代过程中的任何时刻，集合都准确地包含出现在迄今为止处理的数组的前缀中的元素。 当读取新元素时，将其添加到集合中会保留此不变式：集合始终反映已处理前缀的唯一元素。 由于每个元素都只处理一次，因此在循环结束时，该集合包含完整数组中的所有不同值，而没有其他值。 因此，最终大小等于输入中唯一值的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = input().strip().split()
    if not data:
        return
    
    # If format is: n followed by array
    # we try to detect and handle both common variants safely
    try:
        n = int(data[0])
        arr = list(map(int, data[1:1+n]))
        if len(arr) != n:
            arr = list(map(int, data))
    except:
        arr = list(map(int, data))
    
    seen = set()
    for x in arr:
        seen.add(x)
    
    print(len(seen))

if __name__ == "__main__":
    solve()
```该实现以灵活的方式读取输入，因为简单问题的竞赛输入格式有时会在给出明确的 n 或仅给出原始列表之间变化。 解析之后，核心逻辑很简单：一个集合累积所有遇到的值。 

唯一重要的细节是确保我们不会意外地错误地处理输入结构。 集合插入会自动处理重复项，因此无需进行条件检查或手动记账。

 ## 边缘情况

 对于像这样的单元素输入`7`，集合开始为空，插入`7`，并以大小 1 结束。该算法不需要特殊分支，因为循环自然处理单次迭代。 

对于完全统一的输入，例如`9 9 9 9`，第一个插入添加`9`到该集合，并且所有后续插入都没有效果。 该集合包含已处理前缀的唯一元素的不变量在每一步之后都保持不变，并且最终大小正确地保持为 1。 

对于完全不同的输入，例如`1 2 3 4`，每次插入都会将集合大小增加一，并且没有重复项会干扰该过程。 最终的集合大小与输入长度完全匹配，确认算法没有错误地抑制新值。
