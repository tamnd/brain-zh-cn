 ## 解决方案
 ## 问题理解

 我们有一排人，每个人都有一个重量，还有一个摩天轮，每个舱最多可以容纳两个人，只要他们的总重量不超过固定的限制。 每个人都必须被安置在一个小屋中，目标是尽量减少使用的小屋数量。 

输入描述了人数、每个舱室的重量限制以及个人重量列表。 输出是一个数字：在每个客舱容纳一到两个总重量不超过限制的约束下，容纳所有人所需的最小客舱数量。 

推动该解决方案的关键约束是人数可能很大，通常可达 200,000 人左右。 这立即排除了我们尝试所有人员组合的任何二次配对策略。 检查每一对的解决方案将需要 n² 操作的顺序，而这些操作不会在时间限制内完成。 即使是 O(n log n) 的解决方案也必须仔细设计，以便排序不会被重复扫描所支配。 

当在没有排序的情况下应用贪婪策略时，会出现常见的失败情况。 例如，按输入顺序将每个人与下一个可用的人配对可能会严重失败。 

考虑如下输入：```
n = 4, x = 10
weights = [9, 1, 8, 2]
```如果我们贪婪地将相邻元素配对，我们会得到 (9,1) 和 (8,2)，它使用 2 个小屋，在这里是最优的。 但顺序略有不同：```
weights = [9, 8, 1, 2]
```相邻配对使 (9,8) 无效，因此简单的修复可能会隔离 9，然后将 8 与 1 配对，并保留 2，从而形成 3 个小屋。 最优解是(9,1)和(8,2)，只有2个小屋。 这表明，如果没有排序或全局策略，局部决策可能会阻碍更好的配对。 

另一个微妙的问题是忘记配对是可选的。 即使两个小重物安装在一起，强制它们配对也可以防止稍后配对较大的重物，从而增加舱室总数。 

## 方法

 蛮力的想法是尝试以所有可能的方式将人们分配到小屋中，要么将每个人单独放置，要么将他们与总重量有效的其他人配对。 这对应于探索兼容性图中的所有匹配，其中如果两个权重之和至多达到极限，则存在边。 虽然原则上是正确的，但状态的数量呈指数级增长，因为每个人都可以在多种配置中配对或取消配对，并且选择在全球范围内相互作用。 在最坏的情况下，这会变成大约 2ⁿ 状态的组合爆炸。 

关键的结构观察是，一旦我们对权重进行排序，我们就获得了单调排序，使我们能够做出安全的贪婪决策。 剩下的最重的人是最难安置的。 如果这个人可以与任何人配对，那么最好的候选人是剩下的最轻的人。 如果连这个都不合适，那么最重的就必须单独去。 这消除了探索多个配对的需要，因为任何其他最重的配对都只会使用较重的伴侣，因此永远不会更好。 

这将问题简化为排序数组上的两指针过程，其中我们反复尝试匹配最小和最大的剩余元素。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 最优贪心（排序+两个指针） | O(n log n) | O(n log n) | O(1) 额外 / O(n) 总计 | 已接受 |

 ## 算法演练

 ### 最优策略

 1. 按非降序对所有权重进行排序。 这样就形成了一种结构，其中最轻和最重的剩余元素总是很容易识别。 
2. 初始化两个指针，一个位于数组开头，一个位于数组末尾。 这些代表最轻和最重的未分配人员。 
3. 维护一个计数器来记录所使用的舱室数量。 
4. 在左指针未经过右指针的同时，考虑右指针当前最重的人。 
5. 如果最轻和最重合计未超过限制，则将它们分配到同一舱室，并将两个指针向内移动。 这是合理的，因为将最重的伙伴与最小的伙伴配对可以最大限度地提高未来配对的机会。 
6. 如果超过限制，则单独分配最重的人，并仅向内移动右指针。 这是必要的，因为在当前约束下，对于这个最重的人来说，不存在有效的配对。 
7. 为每个任务增加客舱计数器。 
8. 继续，直到分配完所有人员。 

### 为什么它有效

 该算法保持了不变式，即当前指针之外的所有人员都已在相同的贪婪规则下最优地分配到舱室。 在每一步中，决策都集中在剩余最重的人身上。 该人的任何可行配对都必须涉及不比当前最轻的候选者重的人，因此检查最小的剩余重量足以确定配对是否可能。 由于选择尽可能小的合作伙伴永远不会恶化剩余元素的未来可行性，因此贪婪的选择在整个过程中保持了最优​​性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    a = list(map(int, input().split()))
    
    a.sort()
    
    l, r = 0, n - 1
    ans = 0
    
    while l <= r:
        ans += 1
        if l == r:
            break
        if a[l] + a[r] <= x:
            l += 1
            r -= 1
        else:
            r -= 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先对数组进行排序，以便我们可以推断极端情况。 两指针循环每次迭代总是准确地分配一个小屋，要么分配给一对，要么分配给单个重个体。 特殊情况`l == r`干净地处理最后剩下的人，防止重复计数或无效的配对尝试。 

一个常见的实现陷阱是忘记在配对和未配对的情况下增加答案。 另一种是终止条件处理不当，尤其是当只剩下一个元素时。

 ## 边缘情况

 一种极端情况是只有一个人存在时。 排序后，两个指针从相同的索引开始。 该算法立即将舱室计数增加一次，并在以下情况下退出：`l == r`，正确返回1。 

另一种情况是当所有重量超过限制的一半时，所有配对都变得不可能。 该算法总是无法进行总和检查，并重复递减右指针，单独分配每个人。 结果正好是 n 个小屋，这是正确的，因为不存在有效的对。 

最后一种情况是所有权重都非常小，允许完全配对。 该算法总是成功地将左指针和右指针配对，直到它们相遇或交叉，从而精确地产生 ⌈n/2⌉ 个小屋。
