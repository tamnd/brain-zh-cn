 ## 解决方案
 ## 问题理解

 我们得到一行彩色染料，每种染料都有一个正整数值，表示它有多“美丽”。 我们希望选择这些染料的子集来最大化总的美感总和，但有一个限制：我们不允许选择原始系列中彼此相邻的两种染料。 

更具体地说，这是一个数字序列，我们必须选择索引的子集，使得没有两个选定的索引是连续的。 目标是最大化所选值的总和。 

输入大小最多可达 100,000 个元素。 尝试所有索引子集的解决方案是不可能的，因为子集的数量是指数级的。 在这种规模下，即使是检查所有对的二次方法也会太慢。 这些约束强烈建议采用线性或近线性动态规划解决方案，其中每个元素都被处理一次，并且我们重用之前计算的结果。 

一些边缘情况值得关注。 如果只有一种染料，答案就是该染料的价值。 

例如，输入：```
1
7
```正确的输出是：```
7
```假设至少有两个元素并错误地初始化 DP 的简单方法可能会返回 0 或崩溃。 

如果所有值都相等，请说：```
4
5 5 5 5
```最佳策略是选择交替元素，给出 5 + 5 = 10。如果不能正确执行全局间距，任何总是选择局部最优相邻选择的贪婪方法都可能会失败。 

当一个非常大的值被两个中等值包围时，会出现另一种微妙的情况。 例如：```
3
1 100 1
```正确答案是 100，而不是 2 或 101，这取决于有缺陷的贪婪逻辑。 这说明了为什么像“总是取相邻元素中较大的一个”这样的局部决策是不正确的。 

## 方法

 暴力解决方案将尝试索引的每个子集并检查它是否有效（没有两个连续的选择），然后计算其总和。 从概念上讲，这是可行的，因为它直接强制执行约束，但需要迭代大小为 2^N 的所有子集。 即使我们通过回溯和修剪相邻选择来生成子集而略有改进，最坏的情况仍然会探索指数数量的状态，这在 N 达到 40 之前就变得不可行。 

问题的结构表明，位置 i 处的决策仅取决于我们是采用 i 还是跳过它。 如果我们取 i，那么 i-1 就不能取。 如果我们跳过 i，那么我们将继承最好的结果直到 i-1。 这在数组的前缀上创建了一个简单的最佳子结构。 

让 dp[i] 代表我们仅使用前 i 个染料即可实现的最大美感总和。 对于每个位置 i，我们有两个选择：要么不采用 b[i]，在这种情况下我们保留 dp[i-1]，或者我们采用 b[i]，在这种情况下我们必须将其添加到 dp[i-2]。 这给出了直接重现并将问题简化为线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^N) | O(2^N) | O(N) | 太慢了 |
 | 动态规划| O(N) | O(N)（或 O(1)）| 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时保持每个前缀的最佳答案。 

1. 定义 dp[0] = 0，因为没有元素我们就无法选择任何东西。 
2. 定义 dp[1] = b[0]，因为对于一个元素，我们要么采用它，要么没有更好的选择。 
3. 对于从 2 到 N - 1 的每个索引 i，按如下方式计算 dp[i]：

 我们比较跳过当前元素与获取当前元素。 跳过得到 dp[i-1]。 得到 dp[i-2] + b[i]。 我们选择这两个中的最大值。 
4. 处理完所有元素后，dp[N-1] 包含答案。 

内存效率更高的版本仅保留最后两个 DP 状态，而不是整个数组。 

### 为什么它有效

 在每个索引 i 处，前缀 [0..i] 上的任何有效选择都必须恰好属于两个类别之一：它要么包含 i，要么不包含 i。 如果它包含 i，则 i-1 被禁止，剩余的贡献必须来自 [0..i-2] 上的最优解。 如果它排除 i，那么最佳可能值正是 [0..i-1] 上的最优解。 这两种情况涵盖了所有可能性，没有重叠，因此取最大值可以保留每个前缀的最优性。 因为每个前缀解决方案都是从较小的最佳前缀构建的，所以后来的决策不能使早期的最佳选择无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    b = list(map(int, input().split()))
    
    if n == 1:
        print(b[0])
        return
    
    prev2 = 0
    prev1 = b[0]
    
    for i in range(1, n):
        cur = max(prev1, prev2 + b[i])
        prev2 = prev1
        prev1 = cur
    
    print(prev1)

if __name__ == "__main__":
    solve()
```该实现将 DP 数组压缩为两个变量。`prev1`代表 dp[i-1]，并且`prev2`代表 dp[i-2]。 在每一步中，我们计算是否将当前值加上`prev2`胜过跳过它并保留`prev1`。 

显式处理单元素情况以避免引用未初始化的 dp 状态。 没有这个，`prev2 + b[i]`当 i = 1 时无效。

 ## 边缘情况

 对于单元素数组，例如：```
1
7
```算法集`prev1 = b[0]`并且永远不会进入循环。 输出直接是 7，这与正确答案匹配，因为唯一有效的子集包含该元素。 

对于具有交替高值的情况，例如：```
5
10 1 10 1 10
```DP 的演变如下：从 10 开始，然后是 10，然后是 20，然后是 20，然后是 30。在每一步中，都会全局评估跳过或获取，因此算法会正确选择所有三个 10，同时避免相邻的选择。
