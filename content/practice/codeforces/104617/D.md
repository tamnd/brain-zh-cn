 ## 解决方案
 ## 问题理解

 我们得到一个序列$n$层，每层包含一个由字符组成的字符串$R$和$G$。 每个字符代表一种特定颜色的糖果，字符串代表糖果在该层中从上到下出现的顺序。 

关键的微妙之处在于锣谷如何“观察”甜点。 他并没有立即将所有层物理合并为一个序列。 相反，他按顺序一层一层地检查，但是当考虑他连续吃什么时，我们实际上可以将整个甜点视为通过按顺序堆叠所有字符串而形成的单个串联序列。 

任务是找到整个序列中所有糖果具有相同颜色的连续段的最大长度。 

所以问题简化为：给定$n$串过$\{R, G\}$，按顺序连接它们并计算最长的相同字符。 

这些限制以一种直接的方式发挥作用。 所有字符串的总长度最多为$2 \cdot 10^5$，因此任何处理每个字符恒定次数的解决方案都是可以接受的。 如果重复地重新访问字符，则层上的二次方法或重复扫描子字符串已经太慢了。 

常见的失败情况是独立处理每一层并在单个字符串内获取最大游程。 这错过了跨越边界的跑步。 

例如：```
3
RR
R
GG
```正确答案是 3，因为前两层合并为`RRR`。 每层最大值将错误地返回 2。 

另一种失败情况是无条件重置每层边界的计数。 连续性取决于上一层的最后一个字符和下一层的第一个字符。 

## 方法

 暴力方法是显式构建完整的连接字符串，然后扫描它以计算最长的相等字符。 这是正确的，因为它直接模拟了糖果的最终序列。 构建字符串后，单次传递会维护当前的条纹长度并更新最大值。 

成本来自内存和重复串联开销。 如果我们重复连接字符串，则每次追加都会在当前总长度中花费线性时间，从而导致$O(n^2)$最坏情况下的行为。 即使我们预分配并加入一次，我们仍然会执行完整扫描$2 \cdot 10^5$，这很好，但预期解决方案的精神是避免构建不必要的中间结构。 

关键的观察是我们不需要完整的字符串。 我们只需要跟踪当前的角色、当前的条纹长度以及迄今为止看到的最大条纹。 当我们按顺序处理每一层时，如果该层的第一个字符与当前字符匹配，我们将继续连续，否则我们将重置。 在一层内，我们只需按顺序扫描，边扫描边更新条纹。 

这将问题转化为跨所有层的所有字符的单个线性传递，而无需构建全局字符串。

 ## 算法演练

 我们按输入顺序处理字符，同时保持连续运行。 

1. 初始化变量`best`存储迄今为止找到的最大条纹，一个变量`cur_char`对于当前的连胜字符，以及`cur_len`对于它的长度。 

一开始，`cur_char`是未定义的并且`cur_len`为 0，因为没有处理任何糖果。 
2. 对于每个图层字符串，从左到右迭代其字符。 

这保留了每层内的真实进食顺序。 
3. 对于每个角色$ch$，将其与`cur_char`。 

如果匹配，则递增`cur_len`因为连胜继续不间断。 
4. 如果$ch \neq cur_char$，重置连胜：设置`cur_char = ch`和`cur_len = 1`。 

这是必要的，因为即使更改发生在层内或跨层，不同的颜色也会破坏连续性。 
5.更新后`cur_len`， 更新`best = max(best, cur_len)`。 
6.继续直到所有层都处理完毕，然后输出`best`。 

关键的设计选择是我们从不特殊对待层边界。 扫描自然会处理它们，因为邻接纯粹是由输入顺序中的连续字符定义的。 

### 为什么它有效

 在处理过程中的任何时刻，`cur_len`等于以当前位置结尾且由相同字符组成的最长后缀的长度。 该后缀是唯一的，因为它仅取决于最后看到的字符以及它连续重复的次数。 由于每个可能的连续段必须在某个位置结束，因此跟踪所有后缀条纹可确保考虑每个候选段，并且`best`捕获其中的最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    best = 0
    cur_char = None
    cur_len = 0
    
    for _ in range(n):
        s = input().strip()
        for ch in s:
            if ch == cur_char:
                cur_len += 1
            else:
                cur_char = ch
                cur_len = 1
            if cur_len > best:
                best = cur_len
    
    print(best)

if __name__ == "__main__":
    solve()
```该解决方案依赖于一次遍历所有字符。 状态变量`cur_char`和`cur_len`实现当前运行的不变量，同时`best`累积全局最大值。 

一个微妙的点是初始化：`cur_char`必须开始为`None`以便第一个字符始终触发重置，确保正确处理初始条纹。 

层边界不需要特殊处理，因为迭代顺序已经强制执行邻接。

 ## 边缘情况

 一种边缘情况是从一层开始并无缝地继续到下一层的运行。 例如：```
2
RRR
RR
```该算法开始于`R`，延伸穿过第一层，并继续进入第二层而不重置。`cur_len`变为 5，并且`best`正确跟踪它。 简单的每层方法会错误地输出 3。 

另一个边缘情况是每个字符都完全重置：```
3
R
G
R
```在这里，每一个转变都会打破连续性。 算法重置`cur_len`每一步，保持`best = 1`。 这证实了最大碎片下的正确性。 

第三种边缘情况是非常长的单层。 该算法只需扫描一次，线性更新条纹，并且不依赖于任何外部结构，这避免了内存开销和边界混乱。
