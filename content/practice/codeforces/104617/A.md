 ## 解决方案
 ## 问题理解

 我们得到了一组冰块，每个冰块都标有风味名称和代表该冰块融化所需时间的数字。 工厂需要决定以什么顺序将这些块发送到切割机，以便首先处理首先消失的块。 换句话说，我们希望通过增加融化时间来重新排序口味。 

输入只是一个对的列表。 每对包含一个不同的整数和一个唯一的字符串。 整数是排序键，字符串是我们必须输出的标签。 任务是输出按关联整数排序的所有字符串。 

约束足够大，任何比线性时间更差的方法都会陷入困境。 对于多达 100000 个项目，O(N log N) 左右的算法是实际的上限。 任何二次方的事情，例如从未排序的列表中重复选择最小值，在最坏的情况下都面临着 100 亿次比较的风险，这在 2 秒内是不可行的。 

一个微妙的问题是这里不需要稳定性，因为所有熔化时间都是不同的。 这消除了排序中的歧义。 

边缘情况主要是结构性的而不是逻辑性的：

 单个元素输入是微不足道的。 输出只是一种味道。 

已排序或反向排序的输入不会改变正确性，但可能会给简单的实现带来压力。 

大字符串不会影响正确性，但会强化排序键是整数，而不是风味的词汇顺序。 

一个潜在的错误是意外地按字符串而不是整数排序。 例如，如果我们按字典顺序排序，“19 Strawberry”可能会错误地出现在“3 Pineapple”之前，因为字符串比较会忽略数字含义。 

## 方法

 蛮力的想法很简单：重复扫描列表以找到剩余项目中的最短融化时间，输出其风味，并将其从考虑中删除。 这是正确的，因为在每一步中我们都明确选择最小的剩余时间，因此输出是全局排序的。 

然而，每次选择都需要扫描所有剩余元素。 第一步检查 N 个项目，第二步检查 N−1 个项目，依此类推。 总的比较次数约为 N(N−1)/2，当 N 为 100000 时，会增加到大约 5×10^9 次操作。这远远超出了可行的范围。 

改进来自于认识到我们不断地问同一个问题：“其余元素中下一个最小的元素是什么？” 这正是排序问题。 我们不是重复重新计算最小值，而是使用排序算法一次计算整个顺序。 

通过存储对（融化时间、风味）并按整数键排序，我们将问题转换为标准排序。 这将重复扫描开销减少到单个 O(N log N) 排序。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N^2) | O(N^2) | O(N) | 太慢了 |
 | 最优（排序）| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 1. 读取所有对（融化时间、风味）。 我们先存储时间，以便Python中默认的排序顺序可以直接将其用作主键。 
2. 将每对插入到列表中。 除了存储之外不需要任何处理，因为所有决策仅取决于排序。 
3. 对列表进行排序。 排序主要通过熔化时间完成，因为在 Python 中元组是按字典顺序比较的。 
4. 遍历排序列表并按顺序收集风味名称。 这会产生正确的输出序列。 
5. 打印结果，用空格分隔。 

关键的设计选择是在字符串之前存储时间。 这确保元组比较自然地优先考虑正确的字段，而不需要自定义比较器。 

### 为什么它有效

在每个点，正确的下一个输出是剩余融化时间最短的口味。 排序将所有元素按照该键的全局顺序放置，因此排序数组的每个前缀都已经满足以下属性：后一个元素的键不小于前一个元素的键。 这保证了按排序顺序输出与所需的贪婪选择序列相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    arr = []

    for _ in range(n):
        a, s = input().split()
        a = int(a)
        arr.append((a, s))

    arr.sort()

    print(" ".join(s for _, s in arr))

if __name__ == "__main__":
    main()
```由于 N 可能很大，因此该解决方案使用快速 I/O 有效地读取输入。 每一行都分为一个整数和一个字符串。 该对首先与整数一起存储，因此 Python 的默认元组排序行为正确。 

排序是核心操作。 Python 的排序使用 Timsort，它在一般情况下运行时间为 O(N log N)，并且针对部分有序输入进行了优化。 

最后的循环仅提取风味名称。 这可以避免打印元组或额外的格式化开销。 

一个常见的错误是忘记将时间转换为整数。 如果保留为字符串，排序将变为字典顺序并破坏数字顺序。

 ## 边缘情况

 单个冰块案例虽然微不足道，但很重要。 对于输入：```
1
42 Mango
```该算法存储一对，对 size-1 列表进行排序，然后立即输出。 没有发生超出初始化的迭代，确认不需要特殊的外壳。 

另一种情况是逆序：```
3
9 X
2 Y
5 Z
```该列表变为 [(9,X),(2,Y),(5,Z)]，排序得到 [(2,Y),(5,Z),(9,X)]。 该算法不依赖于输入顺序，因此即使最坏情况的排列也会在 O(N log N) 中产生正确的输出。 

最后，考虑熔化时间的大差距。 即使值类似于 1、10^9、500，排序也仅取决于相对比较，而不取决于大小。 排序不受影响，因为对整数的比较是恒定时间的。
