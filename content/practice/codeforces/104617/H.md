 ## 解决方案
 ## 问题理解

 我们有几个锥形模具放置在一条线上的不同整数位置。 我们可以精确地选择一个起始位置，从该位置开始，面糊垂直流开始向下流动。 

工厂还包含水平吊具部分。 每个线段覆盖 x 轴上的一个区间并位于一定的高度。 如果垂直流到达路段的任何点，则流立即扩散到整个路段，将单个垂直线变成活动流的整个水平间隔。 从该区间内的每个活动 x 位置开始，流动独立地继续向下。 

这种传播可以重复：一旦某个区间变得活跃，它可能会激活其下方与该区间相交的其他线段，从而进一步扩展活跃 x 位置集。 任何 x 坐标曾被此传播过程覆盖的模具都被视为已填充。 

任务是选择起始 x 位置，以使最终填充模具的数量最大化。 

这些限制意味着模具和分段的数量最多可达 100,000 个，因此任何模拟每个起始位置传播的解决方案都太慢。 简单的模拟需要在每个起始点的重叠间隔中重复传播，从而在最坏的情况下导致二次行为。 

当多个段间接重叠时，就会出现幼稚推理的微妙失败情况。 例如，如果段 A 与 B 重叠，B 与 C 重叠，即使 A 和 C 不直接重叠，A 内的单个起点仍然可以通过 B 激活 C。任何仅检查直接包含而不考虑传递合并的解决方案都会低估可达性。 

## 方法

 关键的困难是理解对于固定的起始 x 位置传播实际上会产生什么。 一旦我们重新解释这个过程，动态就变成了纯粹的几何间隔。 

强力方法会尝试每个可能的起始模具位置或每个整数 x 坐标。 对于每次启动，我们模拟向下流动并重复合并所有可达段。 每次模拟都可能触及所有段，给出最坏的情况 O(nm)，因为每次启动都可能触发再次扫描所有段。 对于 100,000 个元素来说这太慢了。 

关键的观察是，一旦我们考虑连通性，高度排序不会以有意义的方式改变最终的可达结构。 当一个段被激活时，它会激活整个间隔，并且与该间隔相交的任何段最终也将被激活。 这意味着可达性仅取决于间隔是否通过重叠连接，而不取决于激活的确切顺序。 

因此，我们可以将每个分布器视为一条线上的一个区间，如果两个区间重叠，则将它们连接起来。 这在区间图中形成了不相交的连通分量。 每个连接的组件都会折叠成一个合并的段，这只是该组件中所有间隔的并集。 

一旦建立起来，这个过程就变得简单了。 选择组件内的起始位置会激活整个组件间隔。 选择所有段之外的位置只会激活该单个点。 

因此，答案是位于任何合并区间内的模具的最大数量，并且还可以选择恰好产生一个模具的位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(纳米) | O(n + m) | 太慢了 |
 | 合并间隔+计数| O((n + m) log m) | O((n + m) log m) | O(n + m) | 已接受 |

 ## 算法演练

 我们首先将每个分布器视为数轴上的一个区间。

1. 按左端点对所有吊具进行排序。 这确保我们按照从左到右的扫描顺序处理间隔，其中合并决策是本地的。 
2. 扫描间隔并合并任何重叠或接触的间隔。 如果当前间隔在活动合并间隔之前或结束时开始，我们将扩展合并间隔的右边界。 否则，我们关闭当前组件并启动一个新组件。 这构造了区间重叠图的所有连通分量。 
3. 对所有模具位置进行排序。 这使我们能够使用二分搜索或两个指针有效地计算有多少模具位于任何区间内。 
4. 对于每个合并的区间，计算其中有多少个模具位置。 跟踪所有此类间隔的最大值。 
5. 还要考虑我们从没有被任何区间覆盖的位置开始的情况。 在这种情况下，如果我们选择起始位置与模具位置一致，则可以恰好填充一个模具。 所以答案至少是1。 

关键的决策点是纯粹基于重叠的合并间隔，因为任何重叠都保证激活可以在它们之间传播，而不管高度排序如何。 

### 为什么它有效

 每当到达其内部的任何点时，每个扩展器间隔都会完全激活。 一旦激活，它就会充当其范围内所有点的激活源。 因此，当且仅当两个区间之间存在重叠链时，它们才属于同一可达结构。 这正是区间重叠图中连通分量的定义。 

在连接的组件内，从与组件相交的任何位置开始都会导致该组件中每个间隔的完全激活，从而实现其覆盖范围的完全联合。 在所有组件之外，不会发生任何传播。 这意味着每个起始位置都映射到单个组件联合或单个点，并且最大化模具减少了对这些情况的评估。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    p = list(map(int, input().split()))
    p.sort()

    segs = []
    for _ in range(m):
        l, r, h = map(int, input().split())
        segs.append((l, r))

    if m == 0:
        print(1)
        return

    segs.sort()

    merged = []
    cur_l, cur_r = segs[0]

    for l, r in segs[1:]:
        if l <= cur_r:
            if r > cur_r:
                cur_r = r
        else:
            merged.append((cur_l, cur_r))
            cur_l, cur_r = l, r
    merged.append((cur_l, cur_r))

    def count_range(L, R):
        import bisect
        left = bisect.bisect_left(p, L)
        right = bisect.bisect_right(p, R)
        return right - left

    ans = 1
    for L, R in merged:
        ans = max(ans, count_range(L, R))

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先对模具进行排序，以便能够在间隔内快速计数。 然后，它对吊具间隔进行排序和合并，以构建连接的组件。 合并步骤是一个标准扫描，它保持当前的活动间隔，并在检测到重叠时扩展它。 

计数步骤使用二分搜索来计算有多少模具位置落入每个合并区间。 最终答案是所有组件中最好的，对于我们避免所有传播者的情况，基线为 1。 

一个微妙的点是高度值从未被使用。 这是因为传播能力仅取决于是否可以通过重叠链到达间隔，而不取决于一旦存在连接的垂直排序。 

## 工作示例

 ### 示例 1

 输入：```
5 2
1 2 3 4 5
1 2 1
3 5 2
```对区间进行排序后，我们有 [1,2] 和 [3,5]。 它们不重叠，因此我们得到两个合并的组件。 

| 步骤| 活动间隔| 模具覆盖|
 | ---| ---| ---|
 | 1 | [1,2]| {1,2} |
 | 2 | [3,5]| {3,4,5} |

 最好的组件是[3,5]，覆盖3个模具。 

输出：```
3
```这表明断开间隔的行为是独立的，我们只选择最好的组件。 

### 示例 2

 输入：```
2 2
1 1000000
1 500000 1
500000 1000000 2
```两个间隔相交为 500000，因此它们合并为一个组件 [1,1000000]。 

| 步骤| 活动间隔| 模具覆盖|
 | ---| ---| ---|
 | 1 | [1,1000000] | {1,1000000} |

 由于两个模具都位于合并区间内，因此答案为 2。 

这说明了为什么传递重叠很重要：即使是单个接触点也会合并整个可达性。

 ## 边缘情况

 一个关键的边缘情况是不存在吊具时。 在这种情况下，每个起始位置都会独立运行，并且只能填充其正下方的模具。 正确答案是 1，算法通过答案的基线初始化来处理这个问题。 

另一种情况是所有区间都不相交。 合并步骤会产生多个分量，只有最大的间隔才重要。 计数逻辑正确地独立地评估每个。 

第三种情况是当间隔通过接触端点形成一条链时。 即使没有单个间隔跨越整个范围，合并过程也会将它们组合成一个组件，并且最终的计数步骤正确地将其视为单个可到达区域。
