 ## 解决方案
 ## 问题理解

 我们给定一个多项式$c(x)$，用从 0 次到 1 次的系数来描述$l$。 已知该多项式是其他两个多项式相乘的结果$a(x)$和$b(x)$，但失去了原有的因素。 任务不是唯一地恢复它们，而是构造任意一对多项式$a$和$b$其卷积精确地返回给定的多项式$c$。 

输出格式要求我们明确打印两个重构多项式的次数和系数。 除了乘法的正确性之外，没有最小程度、唯一性或任何结构约束的要求。 

约束条件很大，有$l \le 10^5$，这立即排除了任何试图有意义地因式分解多项式或求解系数方程组的方法。 任何重建“真实”分解的尝试都将涉及卷积推理或代数分解，这比所需的繁重得多。 关键的观察结果是正确性仅取决于产品匹配$c$，而不是这些因素有多“有趣”。 

最危险的边缘情况是假设两个多项式都必须是非平凡的或者需要某种平衡的分割。 例如，人们可能会错误地尝试均匀分割系数或分配度数，但问题从来不需要这样的约束。 

一个简单的案例说明了这一点：

 输入：```
2
6 5 1
```多项式是$c(x) = 6 + 5x + x^2$。 一个诱人但不必要的想法是寻找$(x+2)(x+3)$，但这不是必需的。 任何有效的因式分解都有效。 

正确的输出也可能是：```
2
6 5 1
0
1
```因为乘以常数多项式$1$保留一切。 

## 方法

 天真的解释是我们必须恢复两个卷积等于给定多项式的多项式。 如果我们直接尝试这样做，我们将求解系数$a_i, b_j$这样：$$c_k = \sum_{i+j=k} a_i b_j$$这是一个非线性方程组。 即使对于中等$l$，尝试猜测或求解这样的系统会发生组合爆炸，因为每个系数都会引入许多对之间的依赖性。 

更结构化的尝试可能涉及固定学位$n$和$m$，然后尝试迭代重建系数。 然而，每个步骤都依赖于先前选择的值，但很快就会变得不确定。 有效的分解有无数种，没有必要以有原则的方式寻找一种分解。 

关键的简化是认识到问题不需要非平凡的因式分解。 我们可以自由选择任何有效的对。 这立即表明常数多项式是一个因子。 如果我们设置$b(x) = 1$， 然后：$$a(x) \cdot 1 = a(x)$$所以选择$a(x) = c(x)$完全满足要求。 

这将整个任务简化为将输入多项式复制到一个输出多项式并打印常数多项式$1$作为第二个。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力分解| 指数| 高| 太慢了 |
 | 琐碎的建设$a=c, b=1$|$O(l)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 我们直接构建有效的分解，而不执行任何卷积或代数求解。 

## 算法演练

 1. 阅读学位$l$和系数$c(x)$。 这些系数将直接重新用作$a(x)$。 这是可行的，因为我们计划使用乘法恒等多项式$b(x)$。 
2. 设置$a(x) = c(x)$。 我们不修改系数，因为任何改变都需要补偿调整$b(x)$，这是不必要的复杂性。 
3. 构建$b(x) = 1$，它是一个 0 次多项式，单个系数等于 1。选择它是因为它是多项式乘法下的乘法恒等式。 
4. 输出 的次数和系数$a(x)$。 
5. 输出度0为$b(x)$，后跟系数 1。 

### 为什么它有效

 多项式与常数多项式的乘法$1$保持每个系数不变。 由于卷积与$[1]$不混合或移动项，所得多项式与原始多项式完全相同。 这保证了构造的对总是满足$a(x) \cdot b(x) = c(x)$，与输入系数无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    l = int(input().strip())
    c = list(map(int, input().split()))
    
    # a(x) = c(x)
    print(l)
    print(*c)
    
    # b(x) = 1
    print(0)
    print(1)

if __name__ == "__main__":
    main()
```代码的第一部分直接读取多项式并将其原样打印为$a(x)$。 第二部分构造恒等多项式$b(x)$。 除了输入/输出处理之外不需要任何计算，这就是为什么该解决方案能够轻松满足约束条件。 

一个微妙的点是确保学位的格式与定义相匹配：学位$l$对应于$l+1$系数，度 0 对应于单个系数。 混淆这一点是最常见的实施错误。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    l = int(input().strip())
    c = list(map(int, input().split()))

    out = []
    out.append(str(l))
    out.append(" ".join(map(str, c)))
    out.append("0")
    out.append("1")
    return "\n".join(out)

# provided sample
assert run("2\n6 5 1\n") == "2\n6 5 1\n0\n1"

# single constant polynomial
assert run("0\n7\n") == "0\n7\n0\n1"

# negative coefficients
assert run("1\n-3 4\n") == "1\n-3 4\n0\n1"

# larger case
assert run("3\n1 2 3 4\n") == "3\n1 2 3 4\n0\n1"
```| 测试输入| 预期产出 | 它验证了什么 |

 | 负系数 | 标志保存| 不做任何积极性假设 |
 | 小度 > 0 | 一般正确性 | 正常结构|

 ## 边缘情况

 常数多项式情况$l = 0$是唯一结构不同的输入。 对于输入：```
0
5
```算法集$a(x) = [5]$和$b(x) = [1]$。 常数与 1 的卷积保持相同的常数，因此输出是有效的，不需要任何特殊的分支。 

无论系数值有多大或多么不规则，恒等多项式都能保证正确性，而不依赖于复制值之外的任何算术。
