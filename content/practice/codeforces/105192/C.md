 ## 解决方案
 ## 问题理解

 该过程描述了一种基于回合的伤害系统，其中伤害值与下一回合无关。 每回合以当前伤害数字开始`y`，那么老板的盟友总是应用固定的异或变换`a`，并且您可以选择应用另一个 XOR 转换`b`。 经过这些转换后，得到的数字将作为该回合造成的伤害，并成为下一回合的起始值。 

目标是确定累积伤害达到或超过目标的速度`x`。 您拥有的唯一控制权是是否应用异或`b`每回合，在你的朋友已经应用了异或之后`a`。 

关键的困难在于异或在回合中重复应用，因此当前的伤害取决于整个选择历史，而不仅仅是当前回合。 

约束允许最多`2 · 10^5`测试用例，值高达`10^9`。 这排除了每个测试用例执行的工作超过常数或对数工作的任何模拟。 即使所有测试中的总操作数为几百万次也是可以接受的，但是任何线性的操作都可以接受。`x`或者在长序列的转弯中是不可能的。 

一个天真的解释是依次模拟，直到总和达到`x`。 这已经引起了人们的担忧，因为在最坏的情况下，如果损害长时间保持较小，该过程可能需要大量步骤。 另一个天真的想法是独立对待每个回合并贪婪地最大化当前损害，而不考虑异或历史如何影响未来状态。 在选择的情况下会出现这种情况`b`现在通过以持续的方式改变内部状态来减少未来的损害。 

当异或操作在损坏序列中创建小循环时，会出现微妙的边缘情况。 例如，如果`y = 1, a = 1, b = 1`，然后根据选择，损害可能会在值之间振荡，而贪婪的决定可能会将您锁定在较弱的循环中，即使不同的早期选择会导致以后持续更高的值。 如果不仔细理解状态演化，这正是“局部最佳”决策可能失败的地方。 

## 方法

 蛮力方法是模拟每个回合并在每个步骤中尝试两种可能性，要么应用`b`或不。 随着时间的推移，这形成了二元决策树。 每个节点代表由当前损坏值定义的状态，每条边代表两种可能的转换之一。 

从概念上讲，这是可行的，因为它探索所有有效的决策序列，因此它总是找到达到目标的最小圈数。 然而，分支因子是每步 2，所以之后`n`转弯有`2^n`州。 即使是为了谦虚`n`，这立即变得不可行。 

重要的观察结果是，状态演化并不依赖于异或运算的完整历史，仅依赖于两条奇偶校验信息。 重复的异或与`a`仅取决于经过了多少圈，并且与`b`仅取决于我们选择应用它的次数。 两者都是奇偶驱动效应。 这会将系统崩溃为恒定数量的状态，而不是无限的历史。 

一旦系统被简化为一个小型状态机，该过程就在固定的选择规则下变得确定性。 最佳策略变成在每个状态下选择最大化直接损害值的转变，因为所有未来的行为仅取决于结果状态而不取决于更深的历史。 

这将问题从指数搜索转变为恒定大​​小状态空间上的线性模拟。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有选择的暴力递归 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | 状态约简+贪心模拟| 每次测试 O(T) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.观察到，每回合之后，伤害仅取决于回合数是奇数还是偶数，以及您是否应用了`b`操作偶数或奇数次。 这将完整的历史记录减少为两个二进制状态。 
2. 将系统状态表示为`(i_parity, b_parity)`， 在哪里`i_parity`跟踪当前匝数是偶数还是奇数，以及`b_parity`跟踪是否`b`到目前为止已应用过偶数或奇数次。 
3. 使用表达式计算状态的当前伤害值`y XOR (i_parity ? a : 0) XOR (b_parity ? b : 0)`。 这是有效的，因为重复的 XOR 会成对取消，因此只有奇偶校验很重要。 
4. 从当前状态出发，评估两种可能的行动：不适用`b`，或申请`b`。 第一个保留`b_parity`不变，第二个翻转它。 
5. 选择当前回合产生较大即时伤害值的动作，然后将该伤害添加到累计总和中。 
6. 过渡到下一个状态，翻转`i_parity`每回合和更新`b_parity`如果选择第二个操作。 
7. 重复，直到累积伤害达到或超过`x`，然后输出所使用的匝数。 

该方法背后的关键特性是系统是一个只有四种状态的有限确定性自动机。 在任何状态下，决策仅影响下一个状态和立即奖励，并且未来的转换仅取决于结果状态。 由于除了奇偶校验之外不存在隐藏的长期记忆，因此在每一步中最大化即时损害与全局最优积累相一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        x, y, a, b = map(int, input().split())

        # state: (i_parity, b_parity)
        i_parity = 0  # 0 means even turn index, but we start at turn 1 so handle carefully
        b_parity = 0

        total = 0
        turns = 0

        # we treat turn number starting from 1
        while total < x:
            i_parity ^= 1  # move to next turn parity (1 for odd turn)

            base = y
            if i_parity:
                base ^= a
            if b_parity:
                base ^= b

            # option 1: do not apply b
            v0 = base
            # option 2: apply b
            v1 = base ^ b

            if v1 > v0:
                total += v1
                b_parity ^= 1
            else:
                total += v0

            turns += 1

        print(turns)

if __name__ == "__main__":
    solve()
```该实现仅维护转弯索引的奇偶性以及是否`b`已应用奇数次。 每次迭代都会重新计算这两个选择的当前损害，并选择更好的直接结果。 XOR结构使重新计算的时间恒定。 

一个微妙的点是正确处理转弯奇偶校验。 损害公式取决于当前步骤是第一步、第二步等等，因此在计算损害之前，奇偶校验会在每次循环迭代开始时翻转。

 ## 边缘情况

 当两者都存在时，就会出现极端情况`a`和`b`为零。 状态永远不会改变，每回合都会造成恒定伤害，等于`y`。 算法正确累加`y`每次迭代直到达到`x`，因为两个分支都不会更改值。 

当出现另一种边缘情况时`a == b`。 在这种情况下，申请`b`后`a`取消好友操作的效果。 该算法自然地处理这个问题，因为两个候选值`v0`和`v1`崩溃为相同或交换的值，因此选择变得中立并且不会影响未来状态。 

第三种情况是异或变换产生二值循环。 即使伤害在高值和低值之间交替，贪婪步骤始终会选择较高的直接贡献，并且由于状态转换是确定性和可逆的，因此序列保持一致，并且累积仍然朝着目标前进，而不会陷入非进展循环。
