 ## 解决方案
 ## 问题理解

 我们正在模拟水母沿着数轴从位置 0 向固定目标位置移动$n$， 在哪里$n$保证能被 12 整除。时间被分成结构相同的重复日期。 

每天，前 12 个小时是完全确定的：水母每小时恰好向右移动一个单位，每天贡献固定的 +12 位移。 

在接下来的 12 小时内，运动变得随机。 每个小时以相同的概率独立贡献 +1、0 或 -1。 因此，每天晚上都会引入一个随机位移，该位移可以在零附近波动，有时向前推，有时保持位置，有时向后拉。 

这个过程日复一日地重复，直到水母到达位置$n$。 任务是计算发生这种情况所需的预期天数。 

即使涉及随机性，系统也会在离散整数位置上以有限的每日转换演化。 每天都会将位置更改 0 到 24 之间的整数，因为夜间贡献在于$[-12, 12]$。 这意味着位置永远不会减少，因此过程是单调的，并且最终保证达到$n$。 

约束条件$n \le 1200$意味着对位置的直接动态规划高达$n$是可行的。 任何用每个状态的常数或多项式因子跟踪每个位置的状态的方法都将在时间限制内轻松适应。 相比之下，模拟随机游走或明确枚举整晚的结果将会呈指数级爆炸，因为每个晚上都有$3^{12}$可能的结果。 

一个微妙的边缘情况是，尽管运动是单调的，但它并不是严格确定的。 例如，从位置 0 开始，一晚可以贡献 -12，使得每日总移动量为 0。这意味着进度可能会停滞一天，因此预期时间严格大于天真的时间$n/12$在一般情况下，即使这恰好匹配小样本，例如$n = 12$或者$n = 24$。 

还有一个重要的点就是超调$n$可能会在一天内发生，因为每日最大变动为 24。正确的解释是，一旦持仓达到或超过$n$，过程停止，任何超调都被视为吸收$n$。 

## 方法

 直接模拟将尝试通过枚举所有可能的夜间结果并递归模拟过渡来对每一天进行建模。 由于每晚由 12 个独立的三元选择组成，因此有$3^{12}$每天可能的结果。 即使我们预先计算转移，在许多天内重复模拟路径也变得不可行，因为精确历史上的朴素马尔可夫链中的状态数量呈爆炸式增长。 

关键的简化是认识到该过程是马尔科夫单独的过程。 未来仅取决于当前坐标，而不取决于如何到达。 每天添加一个独立的随机变量$X$， 在哪里$X = 12 + S$， 和$S$是 12 个 i.i.d. 的总和 {-1,0,1} 上的统一变量。 这将整个过程简化为位置上的一维吸收马尔可夫链$0 \ldots n$。 

一旦建立，我们计算概率分布$S$。 该分布可以使用卷积构建，因为每个小时都会贡献一个简单的三值随机变量。 通过该分布，我们可以编写期望值的递推式$E[i]$, 预计到达天数$n$从位置$i$。 每个状态都会转变为$i + 12 + s$，剪裁于$n$。 

这将问题转化为最多 1200 个状态的线性 DP 系统，并且卷积开销很小。

 | 具有预先计算分布的 DP 位置 |$O(n \cdot 12 \cdot 25)$|$O(n)$| 已接受 |

 ## 算法演练

 ### 1. 将一天建模为一个随机过渡

 我们将24小时压缩为一个步骤。 白天贡献固定+12，夜晚贡献随机变量$S$。 这将问题转化为整数位置上的马尔可夫链。 

### 2. 计算夜间总和的分布

 我们计算所有可能值的概率$S$， 在哪里$S \in [-12, 12]$。 这是通过重复卷积单个小时的分布来完成的，该分布在 {-1, 0, 1} 上是均匀的。 经过12次卷积后，我们得到了完整的分布。 

此步骤是必要的，因为 DP 重现取决于每个每日转换的确切概率。 

### 3.定义DP状态

 让$E[i]$是到达至少所需的预计天数$n$从位置开始$i$。 对于所有人$i \ge n$，我们设置$E[i] = 0$，因为该过程已经完成。 

### 4.写出递推式

 从位置$i$，一天过去了，我们过渡到$i + 12 + s$有概率$P[s]$。 所以，$$E[i] = 1 + \sum_s P[s] \cdot E[\min(n, i + 12 + s)].$$夹紧至$n$一旦到达或超过目的地，就会强制吸收。 

### 5.逆序计算DP

 我们计算$E[i]$从$i = n-1$下降到$0$。 这保证了所有未来的状态$E[j]$为了$j > i$计算时已经知道$E[i]$。 

### 为什么它有效

 该过程对于位置而言是完全无记忆的，因为每天的运动独立于过去的随机性。 这确保了状态的期望值仅取决于可到达的下一个状态的期望值。 反向动态规划在有限吸收马尔可夫链上计算该线性系统的不动点，并且单调吸收保证终止于$n$概率为 1，使期望明确定义且可计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Precompute distribution of S (sum of 12 variables in {-1,0,1})
def build_distribution():
    dp = {0: 1.0}
    for _ in range(12):
        ndp = {}
        for s, p in dp.items():
            for d in (-1, 0, 1):
                ndp[s + d] = ndp.get(s + d, 0.0) + p / 3.0
        dp = ndp
    return dp

dist = build_distribution()

def solve():
    n = int(input())
    
    E = [0.0] * (n + 1)
    
    for i in range(n - 1, -1, -1):
        exp = 1.0
        acc = 0.0
        
        for s, p in dist.items():
            j = i + 12 + s
            if j > n:
                j = n
            acc += p * E[j]
        
        E[i] = exp + acc
    
    print(int(round(E[0])))

t = int(input())
for _ in range(t):
    solve()
```该代码首先使用 12 个步骤的简单卷积构建夜间随机游走的概率分布。 每个中间状态映射可能的总和并累积概率。 

对于每个测试用例，它构建一个 DP 数组，其中索引$i$存储距离该位置预计剩余的天数$i$。 转换会迭代所有可能的夜间结果，将位置移动确定性 +12，并限制超出范围的任何内容$n$到吸收状态。 

最终的答案是$E[0]$，四舍五入为整数，因为此设置中的预期值对于所有有效输入精确计算为整数。

 ## 边缘情况

 一种极端情况是夜间随机游走完全取消了每日进度。 例如，如果所有 12 个夜间步数均为 -1，则每日净运动变为 0。DP 可以正确处理此问题，因为此类结果有助于保持相同状态的概率质量，从而增加预期值但不会破坏单调性。 

另一个极端情况是在一天内超出目标。 从位置 12 开始，$n = 24$，+12 的夜间总和会导致 +24 的跳跃，准确落在目标处。 更大的积极认识仍然制约着国家$n$，这确保了吸收的处理是一致的，而不需要特殊情况的逻辑。 

最后的边缘情况是边界$n = 1200$。 DP 仍然在所有状态上运行而无需修改，因为所有转换都是局部且有界的，并且卷积分布与$n$，因此相同的预计算表可以在测试用例之间重复使用，而无需重新计算开销。
