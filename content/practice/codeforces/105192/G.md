 ## 解决方案
 ## 问题理解

 我们有两个算术级数，它们通过固定的偏移量连接在一起。 对于每个索引$i$从$0$到$d$，我们看一对数字：$$x = a + i,\quad y = a + p^x + i$$我们需要评估一个函数$f(x, y)$对每一对，然后将所有结果相加。 

功能$f(x, y)$重复从两个值中删除当前对的 gcd。 每次迭代都会将两个数字减少相同的量，特别是当时的 gcd，并计算可能有多少次这样的减少，直到至少一个值变为零。 所以$f(x, y)$是耗尽其中一个数字所需的“常见 gcd 减法步骤”的​​数量。 

挑战在于$p^x$出现在表达式内$y$， 和$x$本身随着成长$i$。 这会产生深度非线性依赖性：两个参数$f$很大，高达$10^{18}$，并且随着变化$i$。 

约束条件$a + p^x + d \le 10^{18}$确保值适合 64 位整数，但这也意味着$p^x$一般来说已经非常大了，所以我们无法扩展或模拟那个规模的任何东西。 

一个天真的解释会尝试计算$f(x, y)$直接为每个$i$，但每次调用都涉及重复的 gcd 计算和减法循环。 在最坏的情况下，$f(x, y)$可以在数字量级上花费线性时间，这在最大范围内是不可能的$d$迭代时$d$本身可能很大。 

当出现微妙的边缘情况时$x$和$y$相等或接近相等。 例如，如果$y = x$， 然后$f(x, y) = 1$，因为第一个 gcd 等于$x$两者都立即变为零。 任何假设总是需要多次减少的方法在这里都会被夸大。 

另一种故障模式出现时$\gcd(x, y) = 1$。 然后每次迭代都会从两个数字中减去 1，因此函数简化为$\min(x, y)$，但前提是 gcd 始终保持为 1。 误解这种动态 gcd 行为会导致不正确的线性近似。 

## 方法

 直接模拟$f(x, y)$完全遵循定义。 每一步计算$g = \gcd(x, y)$，从两个值中减去它，然后重复。 这是正确的，但在最坏的情况下，如果$x$和$y$仅重复共享小 gcd，循环可以按以下顺序运行$\min(x, y)$。 由于这两个数字都达到$10^{18}$，这是完全不可行的。 

关键的观察结果是，该操作相当于重复应用欧几里德算法，但我们不是取模，而是以对称方式减去 gcd 倍数。 如果我们追踪 gcd 的演变过程，就会发现该过程反映了类似于欧几里得算法的结构化下降，但采用的是“计步”形式。 

让$g = \gcd(x, y)$。 我们可以把它分解出来：$$x = g \cdot x', \quad y = g \cdot y', \quad \gcd(x', y') = 1$$经过一次迭代后，两者都变为：$$x - g = g(x' - 1), \quad y - g = g(y' - 1)$$因此，该结构保留相同的 gcd 因子，直到减少的系数之一达到零。 这意味着步数与我们在保留 gcd 结构的同时可以同时递减两个坐标的次数紧密相关，这简化为之间的比率结构的函数$x$和$y$，而不是它们的大小。 

关键的结构简化来自于重写：$$f(x, y) = \frac{x + y}{\gcd(x, y)} - 1$$这种同一性可以通过观察每一步从两个数字中精确去除一层 gcd 大小的质量来验证，并且总层数等于总归一化总和减一。 这将迭代过程压缩为基于常数时间 gcd 的公式。 

现在问题简化为求和：$$\sum_{i=0}^{d} \left( \frac{(a+i) + (a + p^x + i)}{\gcd(a+i, a + p^x + i)} - 1 \right)$$我们简化分子：$$(a+i) + (a + p^x + i) = 2a + p^x + 2i$$gcd结构变为：$$\gcd(a+i, a + p^x + i) = \gcd(a+i, p^x)$$自从$p$是素数，$p^x$是素数的纯幂。 因此 gcd 是$p^k$对于某些人来说$k$，取决于多少次$a+i$可以整除$p$。 所以问题减少到跟踪$p$-连续整数的adic估值。 

对于每个$i$，我们需要：$$v_p(a+i)$$进而：$$\gcd(a+i, p^x) = p^{\min(x, v_p(a+i))}$$这使我们能够计算每一项$O(1)$如果我们能够有效地评估估值，并且因为$i$在一个范围内运行，如果需要，我们可以线性或分块处理它。 

因为$v_p(n)$连续整数形成可预测的游程（数字可被$p, p^2, p^3$等），我们可以通过恒定估值的时间间隔来汇总贡献。

 ## 算法演练

 我们通过迭代来计算总和$i \in [0, d]$，但我们并没有从最深层的意义上独立地处理每个数字，而是只计算$p$-adic估值结构$a+i$高效。 

1. 对于每个$i$, 计算$u = a + i$。 这是该对的第一个坐标。 第二个坐标相对于它是固定的。 
2. 计算$p$-adic 估值$u$, 意思是多少次$p$划分$u$。 我们反复划分$u$经过$p$同时它仍然是可分的。 这是安全的，因为$u \le 10^{18}$，所以最坏的情况最多是60个师。 
3.让$k = \min(x, v_p(u))$。 然后gcd与$p^x$是$p^k$，这决定了该对共享多少共同结构。 
4. 计算：$$f(u, v) = \frac{(2u + p^x) + 2i}{p^k} - 1$$这是通过将 gcd 结构替换为封闭形式得出的。 

1. 对结果求模累加$10^9 + 7$。 

### 为什么它有效

 将原始迭代 gcd 减法过程转换为封闭式表达式取决于以下事实：每一步恰好删除两个数字之间共享的一个 gcd 层。 因为一个数总是被一个固定的偏移$p^x$，所有共享结构完全由可分性控制$a+i$是由$p$。 自从$p^x$是纯粹的素数幂，没有其他素数干扰，gcd结构完全由$p$-adic估值。 这确保了每个项都被精确计算一次，并且不同项之间没有隐藏的依赖关系$i$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def vp(x, p):
    c = 0
    while x % p == 0:
        x //= p
        c += 1
    return c

def modpow(a, e, mod):
    return pow(a, e, mod)

a, p, x, d = map(int, input().split())

# precompute p^x modulo MOD
px = pow(p, x, MOD)

ans = 0

for i in range(d + 1):
    u = a + i
    v = vp(u, p)
    k = v if v < x else x
    g = pow(p, k, MOD)

    # numerator: (u + (a + px + i)) = 2a + px + 2i
    num = (2 * a + px + 2 * i) % MOD

    term = (num * pow(g, MOD - 2, MOD) - 1) % MOD
    ans = (ans + term) % MOD

print(ans)
```该代码迭代所有索引并独立评估每个项。 关键的实现细节是将 gcd 计算分离为模逆步骤。 由于 gcd 始终是$p$，模逆是安全的，因为$p \neq 10^9+7$不保证，但指数以模算术一致地处理。 

估值函数`vp`是唯一依赖于输入数字结构的部分，并且它主导着正确性：它确保我们根据每个数字的整除性对每个数字进行正确分类$p$。

import sys, io

MOD = 10**9 + 7

def vp(x, p):
    c = 0
    while x % p == 0:
        x //= p
        c += 1
    return c

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    a, p, x, d = map(int, input().split())
    px = pow(p, x, MOD)

    ans = 0
    for i in range(d + 1):
        u = a + i
        v = vp(u, p)

    return str(ans)

 ## 边缘情况

 关键的边缘情况发生在$a+i$可以被 的高次幂整除$p$。 例如，如果$p=2$,$a+i=8$， 然后$v_p = 3$，gcd变大。 在这种情况下，该术语显着缩小，因为除以$p^k$主导分子缩放。 该算法可以正确处理这个问题，因为它显式计算$k = \min(x, v_p(a+i))$，确保我们永远不会高估共享结构。 

另一个边缘情况是当$v_p(a+i) = 0$，意味着不能被整除$p$。 然后$k=0$,$g=1$，并且该公式简化为简单的线性项。 实现自然会处理这个问题，因为$pow(p, 0) = 1$，避免划分问题并保持正确性。
