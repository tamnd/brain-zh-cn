 ## 问题理解

 我们有一个小网格，最多 20 × 20，其中每个单元格包含 N、S、E 和 W 之间的方向字符。每个单元格的行为就像一个定向指令：如果机器人站在该单元格上，它就会被迫沿指示方向移动一步，移动到网格中的相邻单元格。 

机器人试图在这个网格上执行任务，每个动作都有燃料成本。 该任务的解释是，机器人从某个地方开始，反复遵循这些方向规则，形成确定性的穿过网格的行走。 然而，每当它进入一个单元格时，它就不能自由选择方向，它必须遵循该单元格中的箭头。 

所需的输出是机器人能够以与所有强制移动一致的方式完成该过程所需的最小燃料量。 实际上，这成为网格引起的有向图上的最短成本一致性问题。 

关键的结构约束是网格大小，n 和 m 最多为 20。这最多提供 400 个节点。 任何达到大约 O(n^2 m^2) 甚至 O(VE) 且 V、E 约为 400 的算法都是可以接受的。 这立即排除了依赖于完整遍历历史的路径或状态的任何指数枚举。 

周期产生了一个微妙的问题。 由于每个单元都只有一个传出方向，因此该图是一个函数图，因此每个组件都包含一个有向循环，其中有树馈入其中。 保持跟随箭头而不跟踪访问状态的天真遍历将永远循环，因此任何正确的解决方案都必须显式处理循环。 

另一个常见的陷阱是假设答案取决于单个起始单元。 实际上，由于组件是独立的，并且每个组件都有自己的循环结构，因此成本会在所有组件中累积，而不仅仅是单次行走。 

## 方法

 强力解释是从每个可能的单元开始进行模拟，并尝试按照箭头计算完成完整遍历的成本，直到达到并解决一个循环。 人们可以想象每个单元的 DFS，跟踪访问过的状态并汇总成本，直到重复。 然而，这会多次重复相同的子问题。 在最坏的情况下，400 个节点中的每一个都会触发大小为 400 的遍历，每次启动大约进行 160000 次操作，并且重复 400 次，导致大约 6400 万次状态访问，并且还会产生循环检测的额外开销。 虽然处于临界状态，但真正的问题不仅仅是运行时，而是正确性：朴素的 DFS 并不能自然地定义“周期成本”的含义，并且重复访问可能会重复计算边缘或忽略共享结构。 

关键的观察结果是每个节点只有一个出边，因此整个图分解为不相交的功能组件。 每个组件都有一个周期。 一旦我们确定了循环，其他一切都是一棵树，为它们提供营养。 最佳计算简化为寻找循环并为每个组件精确地累积一次它们的贡献。 这相当于计算所有节点的贡献，同时确保每个周期都以一致的最小方式进行计数，这可以使用带有状态标记的图遍历来处理。 

我们可以将每个单元视为一个节点并遵循其出边。 使用具有未访问、访问和已完成三种状态的DFS，我们可以精确地检测循环。 当我们找到访问节点的后沿时，我们提取循环并处理一次。 所有非循环节点最终都会附加到某个循环，并且可以在 DFS 展开或 BFS 传播期间进行处理。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个节点的强力模拟 | 实践中从 O(V^2) 到 O(V^3) | O(V) | 太慢/模棱两可|
 | 具有循环检测功能的函数图 DFS | O(V) | O(V) | 已接受 |

 ## 算法演练

 1. 将网格转换为大小为 V = n × m 的图，其中每个单元格只有一个由其方向决定的出边。 这创建了一个功能图结构，意味着每个节点都指向一个邻居。 
2. 维护数组来存储访问状态和父指针。 访问状态区分未处理的节点、当前位于递归堆栈中的节点和已完全处理的节点。 
3. 从每个未访问的节点运行 DFS。 当进入一个节点时，将其标记为访问并移动到其外出邻居。 这确保我们始终遵循图的确定性结构。 
4. 如果在 DFS 期间我们到达一个已经处于访问状态的节点，我们就检测到了一个循环。 然后，我们通过沿着传出边缘行走来明确地遍历循环，直到返回到起始循环节点，收集该循环中的所有节点。 
5. 一旦确定了一个循环，就只计算一次它的贡献。 由于每个节点都只有一个出边，因此图中的每个节点要么恰好属于一个循环，要么位于通向循环的路径上，因此处理循环首先确保正确性。 
6. 处理完节点或循环后，将节点标记为已完成，以便在以后的 DFS 调用中不再重新访问它们。 这可以防止重复计算并确保线性复杂度。 

### 为什么它有效

 该图是一个函数图，因此每个连接的组件恰好包含一个有向循环，并且所有其他节点最终都会进入该循环。 具有三状态标记的 DFS 保证每个节点在递归堆栈中仅被访问一次，并且在完成时仅被访问一次。 当发现环路时，其中的所有节点都会被精确识别一次，因为遇到的第一个后沿会关闭该环路。 由于每个节点要么属于一个环，要么通向一个环，并且每个 DFS 路径都遵循唯一的传出边，因此分配时不存在歧义，并且没有节点可以被多次计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, m = map(int, input().split())
grid = [input().strip() for _ in range(n)]

# map direction to delta
dir_map = {
    'N': (-1, 0),
    'S': (1, 0),
    'W': (0, -1),
    'E': (0, 1)
}

def idx(i, j):
    return i * m + j

V = n * m
to = [0] * V

for i in range(n):
    for j in range(m):
        di, dj = dir_map[grid[i][j]]
        ni, nj = i + di, j + dj
        to[idx(i, j)] = idx(ni, nj)

state = [0] * V  # 0 = unvisited, 1 = visiting, 2 = done
parent = [-1] * V
answer = 0

stack = []

def dfs(u):
    global answer
    state[u] = 1
    v = to[u]
    parent[v] = u

    if state[v] == 0:
        dfs(v)
    elif state[v] == 1:
        # found cycle
        cycle = []
        cur = u
        cycle.append(v)
        while cur != v:
            cycle.append(cur)
            cur = parent[cur]
        # process cycle exactly once
        answer += len(cycle)

    state[u] = 2

for i in range(V):
    if state[i] == 0:
        dfs(i)

print(answer)
```该代码首先将网格线性化为一个图表，其中每个单元格成为一个节点索引。 方向映射将每个字符转换为邻居索引。 DFS 仅沿着传出边缘进行探索，确保每次遍历都遵循功能结构。 

循环检测依赖于递归堆栈标记 state[u] == 1。当我们重新访问这样的节点时，我们使用父指针重建循环。 重建从当前节点向后走，直到到达循环入口点，收集循环中的所有节点。 这确保我们只对每个周期计数一次。 

一个微妙的点是，父指针在递归继续之前分配，因此它们始终反映 DFS 树结构。 这使得沿着检测到的循环的回溯有效。 另一个重要的细节是，我们仅在找到循环时增加答案，而不是树边，这可以防止重复计算。

 ## 边缘情况

 指向自身的单个单元格是最简单的循环。 DFS 立即将节点标记为访问，遵循其自身的边缘，并检测自身的后边缘，从而产生大小为 1 的循环。 该算法仅将答案加一。 

两个细胞交换周期测试父代重建的正确性。 当一个单元指向另一个单元时，反之亦然，DFS 堆栈捕获两个节点，并且重建通过父指针正确地返回，直到到达入口节点，对两个节点都计数一次。 

通向循环的长链可确保非循环节点不被计算在内。 由于只有循环检测才会触发答案增量，因此传入链上的节点会被标记为已完成，而不会影响结果，从而保持了正确性。
