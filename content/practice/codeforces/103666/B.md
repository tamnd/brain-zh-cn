 ## 问题理解

 我们得到一个类似斐波那契的序列，其中前两项固定为$F_1 = 1$和$F_2 = 2$，后面的每一项都是前两项的总和。 这会产生一个确定性的无限整数序列。 任务不是直接计算大值，而是关注该序列中索引的一段$L$到$R$，并计算有多少个索引斐波那契数可以被 3 整除。 

输入给出两个整数$L$和$R$，描述斐波那契序列中的连续索引块。 输出是一个整数，表示该索引范围内有多少个斐波那契数除以 3 时余数为零。 

约束条件允许$L, R \le 10^5$，这立即表明任何重新计算斐波那契值到每个查询端点的解决方案在原始迭代计数方面都已经很好了，因为生成$10^5$斐波那契数列是微不足道的。 然而，真正的问题不是计算斐波那契数本身，而是有效地检查可除性并识别该属性的结构。 

当尝试直接计算大的斐波那契值时，会出现一个微妙的陷阱。 虽然$n \le 10^5$，斐波那契数呈指数增长并很快超出标准整数范围。 在 Python 中使用大整数的简单实现仍然是可能的，但没有必要。 更重要的是，重新计算完整值是浪费精力，因为我们只关心能被 3 整除，这仅取决于序列模 3。 

输入格式或范围不会产生棘手的边缘情况，但有一个概念性的情况：如果有人直接计算斐波那契值，然后检查整除性，他们可能会认为 Python 可以安全地处理它。 确实如此，但它隐藏了这样一个事实：问题的结构允许在预处理后使用恒定时间的每个索引解决方案。 

## 方法

 一种直接的方法是逐一生成斐波那契数$F_1$最多$F_R$，检查每个值，并计算有多少个在范围内$[L, R]$能被 3 整除。这是正确的，因为它直接遵循定义。 成本是线性的$R$，每个步骤执行加法和模检查。 在Python中这仍然可行$10^5$术语，但它正在做不必要的工作。 

关键的观察是我们永远不需要完整的斐波那契值。 我们只需要它们模 3。一旦我们减少了模 3 的递归，我们就得到了一个小的状态机：每一项仅依赖于模 3 的前两个残数。因为每个残数都在$\{0,1,2\}$，整个过程最终必须重复。 事实上，该序列具有非常短的周期结构，因此被 3 整除的模式在索引中变成周期性的。 

一旦我们知道了这种模式，问题就简化为计算有多少个索引$[L, R]$落入斐波那契余数为零的位置。 这可以通过预先计算第一个周期并使用算术计数来回答，或者通过在周期模式上构建前缀和来解决$10^5$。 

一种更简单且同样安全的方法是预先计算残基序列$R$，构建位置的前缀和，其中$F_i \bmod 3 = 0$，并在常数时间内回答查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力斐波那契 + 检查 |$O(R)$|$O(1)$| 已接受 |
 | 模 + 前缀和 |$O(R)$预处理,$O(1)$查询 |$O(R)$| 已接受 |

 ## 算法演练

 我们仅以模 3 构造斐波那契数列，因为整除性仅取决于余数。 

1. 计算数组`f[i]`储存$F_i \bmod 3$为所有人$i \le R$。 每个值都是使用递归计算的$f[i] = (f[i-1] + f[i-2]) \bmod 3$。 这可以使数字保持较小并完全避免溢出。 
2.构建前缀和数组`pref[i]`在哪里`pref[i]`计算有多少个索引$1 \le j \le i$满足$f[j] = 0$。 这将问题转化为范围计数，这就是前缀和在这里有用的原因。 
3. 回答询问$[L, R]$通过计算`pref[R] - pref[L-1]`。 这是有效的，因为前缀和编码了累积频率，因此减法精确地隔离了我们关心的范围。 

工作原理：斐波那契递推模 3 定义了一个确定性的残差序列，因此每个索引都预先有一个固定值。 前缀数组存储满足任意索引的整除条件的位置的精确计数。 由于序列是完全预先计算的，没有近似，因此减去前缀值可以精确地隔离任何间隔中的有效元素的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

L = int(input().strip())
R = int(input().strip())

n = R

f = [0] * (n + 1)
pref = [0] * (n + 1)

if n >= 1:
    f[1] = 1 % 3
    pref[1] = 1 if f[1] == 0 else 0

if n >= 2:
    f[2] = 2 % 3
    pref[2] = pref[1] + (1 if f[2] == 0 else 0)

for i in range(3, n + 1):
    f[i] = (f[i - 1] + f[i - 2]) % 3
    pref[i] = pref[i - 1] + (1 if f[i] == 0 else 0)

ans = pref[R] - (pref[L - 1] if L > 1 else 0)
print(ans)
```该实现直接镜像递归，但立即减少每个斐波那契值模 3，防止不必要的增长。 前缀数组在同一循环中更新以避免第二次传递。 

一个微妙的细节是处理$L = 1$， 自从`pref[0]`数组中不存在。 这是通过将其视为零来显式处理的。 

## 工作示例

 考虑以下示例：$L = 3$和$R = 7$。 

我们计算序列：

 | 我| F_i | F_i 模 3 | 前缀计数 |
 | --- | --- | --- | --- |
 | 1 | 1 | 1 | 0 |
 | 2 | 2 | 2 | 0 |
 | 3 | 3 | 0 | 1 |
 | 4 | 5 | 2 | 1 |
 | 5 | 8 | 2 | 1 |
 | 6 | 13 | 1 | 1 |
 | 7 | 21 | 21 0 | 2 |

 查询结果为$pref[7] - pref[2] = 2 - 0 = 2$。 

该跟踪表明，一旦跟踪了残留物，问题就简化为简单的计数。 

现在考虑$L = 1, R = 5$：

 | 我| F_i 模 3 | 前缀 |
 | --- | --- | --- |
 | 1 | 1 | 0 |
 | 2 | 2 | 0 |
 | 3 | 0 | 1 |
 | 4 | 2 | 1 |
 | 5 | 2 | 1 |

 答案是$pref[5] - pref[0] = 1$，确认该范围内只有一个斐波那契数可以被 3 整除。

 ## 边缘情况

 案例$L = 1$是唯一的结构边缘条件，因为它强制处理丢失的前缀索引。 例如，输入：```
1
1
```我们只计算$F_1 = 1$，它不能被 3 整除，所以正确答案是 0。该算法通过处理来处理这个问题`pref[0]`明确为零。 

对于第二种情况：```
2
3
```我们有$F_2 = 2$,$F_3 = 3$。 只有索引 3 起作用，所以答案是 1。前缀减法`pref[3] - pref[1]`正确隔离此单个有效索引，无需超出边界处理的特殊情况逻辑。
