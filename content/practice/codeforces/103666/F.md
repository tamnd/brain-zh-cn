 ## 解决方案
 ## 问题理解

 我们收到了一系列俄罗斯套娃，每个娃娃都有一个数字尺寸。 只有当一个娃娃的尺寸严格小于另一个娃娃时，才可以将其放入另一个娃娃内。 每个娃娃最多可以直接包含另一个娃娃，因此我们构建的结构是链式结构而不是分支结构。 

任务是丢弃一些玩偶，以便将剩余的玩偶排列成一个嵌套序列，其中每个玩偶都适合下一个玩偶。 我们希望最大化这样的序列中剩余的玩偶数量。 

换句话说，这是要求最长的严格递增的数字序列，我们可以任意重新排序所选元素，因为只要大小沿着链严格递增，我们就可以选择任何嵌套顺序。 

输入大小最多为 1000 个娃娃，大小最多为 10000 个。这立即表明 O(n^2) 解决方案是可以接受的，而任何立方体也仍然可以通过，但没有必要。 O(n log n) 的解决方案也是可能的，但不是必需的。 

一个微妙的点是我们不限于保留原始顺序。 如果有人错误地将输入视为序列并试图找到最长的递增子序列，他们可能会低估。 在这里我们可以自由地重新排序，因此只有值的多重集才重要。 

第二个边缘情况是由相同的大小引起的。 由于嵌套要求严格不相等，因此相等的元素不能连续放置。 例如，输入`3 3 3 3`应该产生`1`， 不是`4`，因为在任何严格递增链中只能选择其中之一。 

## 方法

 一种天真的解释是通过尝试所选玩偶的每种可能的排序并检查它是否形成有效的严格递增序列来考虑构建一条链。 这很快就会变得复杂，因为我们将排列子集并验证约束，这在 n 非常小的情况下是不可能的。 

另一个蛮力的想法是尝试每个子集，然后检查它是否可以排序为严格递增的序列。 这将问题简化为检查子集中的所有值是否不同，因为当且仅当不存在重复项时，对任何子集进行排序都会给出有效的嵌套。 这一观察已经大大简化了问题。 

关键的见解是顺序无关紧要。 一旦我们对所有大小进行排序，任何有效的嵌套都对应于选择严格递增值的子序列。 由于排序消除了排列问题，我们只需要计算可以链接多少个不同的值，但我们还必须在更一般的 DP 框架中正确考虑多重性。 

一种简洁的查看方法是对数组进行排序并计算最长的严格递增子序列。 因为排序破坏了原始结构，所以 LIS 变得微不足道：相等的值不能扩展序列，而递增的值总是可以扩展。 在排序数组中，我们能做的最好的事情就是从每个不同的值中挑选一个代表，但如果我们错误地解释 LIS，这并不总是足够的。 正确的解释仍然是原始数组上的 LIS，但由于我们可以自由地重新排序，因此我们可以先排序并将问题简化为对相等的值进行分组。 

因此，解决方案减少为计算我们可以选择值的次数，使得每个下一个值都严格更大，这相当于多重集中不同值的数量。 

然而，还有一个更仔细的视角：如果我们对重复项进行排序和压缩，最长的链正是唯一值的数量。 

蛮力通过探索所有子集和排列来发挥作用，但会失败，因为它以指数方式重复相同的可行性检查。 只有按值排序才重要的观察结果将问题简化为排序后的重复数据删除。

 ## 算法演练

 1. 阅读娃娃尺寸列表。 
2. 按非降序对列表进行排序。 排序可确保任何有效的嵌套序列必须按此顺序显示为严格递增的子序列。 
3. 初始化答案计数器。 首先计算第一个元素（如果存在）。 
4. 从左到右扫描排序后的数组。 
5. 每次我们看到一个值严格大于前一个值时，就增加计数器。 此步骤确保我们仅在有更大的娃娃可用时才扩展嵌套链。 
6. 最终计数器值是可以嵌套的最大玩偶数量。 

### 为什么它有效

 排序后，任何有效的嵌套链都必须对应于选择具有严格递增值的索引序列。 由于相等的值在这样的链中不能相邻，因此每个不同的值最多只能贡献一个元素。 相反，按升序从每个不同值中挑选一个代表总是形成有效的嵌套序列。 这表明最佳解决方案正是数组中不同值的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

a.sort()

if n == 0:
    print(0)
    sys.exit()

cnt = 1
for i in range(1, n):
    if a[i] > a[i - 1]:
        cnt += 1

print(cnt)
```该解决方案首先对数组进行排序，使所有相等的值变得相邻。 我们将答案初始化为 1，因为非空数组始终允许链中至少有一个玩偶。 然后我们线性扫描，只有当我们遇到比前一个值严格更大的值时才增加答案。 这保证了重复项被忽略，因为它们不能扩展严格递增的链。 

这里一个常见的错误是使用`>=`而不是`>`，这会错误地允许相同大小的链延伸。 另一个微妙的问题是忘记空输入情况，尽管约束保证了至少一个元素。

 ## 边缘情况

 关键的边缘情况是所有值都相等。 对于输入：```
4
7 7 7 7
```排序产生相同的序列。 该算法从 cnt = 1 开始，并且后面的元素不严格大于前一个元素，因此结果保持为 1。这与只能选择一个娃娃的事实相符。 

另一个边缘情况是严格减少输入：```
5
5 4 3 2 1
```排序后：```
1 2 3 4 5
```每一步都会增加 cnt，产生 5。这证实了完全允许重新排序，并且原始顺序无关。 

最后一个微妙的情况是混合重复：```
6
1 2 2 3 3 4
```排序不会改变它。 该算法对转换 1→2、2→3、3→4 进行计数，得出 4。这表明重复项永远不会贡献额外的嵌套深度，而只会严格增加问题。
