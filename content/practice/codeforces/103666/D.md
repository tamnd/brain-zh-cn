 ## 问题理解

 我们给出了一个长度为 2 的二进制表$n$，其中每个位置代表阿森尼计划在特定时间内做什么：训练或饮食。 时间表固定为两个字符的字符串，其中一个字母代表训练，另一个字母代表饮食。 

我们必须执行的约束是局部的、非常具体的：最终的时间表中没有任何地方允许饭后立即进行训练。 换句话说，训练时间紧随进食时间的模式不得出现在最终字符串中。 

我们可以改变日程中的任何位置，每次改变都会将角色从训练切换到进食，反之亦然。 目标是进行尽可能少的更改，以便生成的字符串满足约束。 

约束条件$n \le 10^5$立即排除任何二次策略。 任何尝试所有可能的修改甚至使用嵌套扫描模拟校正的方法都会太慢。 解必须是线性或近线性的$n$，因为我们只有大约时间$10^7$运营。 

一个微妙的一点是，如果不小心地贪婪地修复一个不良邻接，可能会影响之前或之后决策的有效性。 例如，仅在本地纠正违规的简单从左到右修复如果在不考虑下一个位置的情况下更改字符，则可能会引入新的违规。 

当字符串频繁交替时会出现另一种边缘情况，例如`ete...`。 在这种情况下，天真的贪婪修正可能会反复翻转字符，导致不必要的变化，而最佳解决方案往往会将字符串重塑为稳定的形式，在进食和训练之间具有单一边界。 

## 方法

 暴力方法会尝试所有可能的最终长度字符串$n$超过两个字符，并选择与原始字符编辑距离最小且满足约束的字符。 这样的字符串的数量是$2^n$，并且检查每个的有效性和计算距离将花费$O(n)$，导致$O(n 2^n)$，这是完全不可行的。 

更结构化的蛮力会观察到有效字符串具有非常严格的形式。 一旦我们看到训练后跟着吃东西，它是被禁止的，这意味着字符串不能包含该模式`te`。 这意味着所有`e`必须出现在所有人面前`t`，因此任何有效的最终字符串都必须采用以下形式`eeee...tttt...`有一些分割点。 

这是关键的结构见解。 我们不用随意修改，只需要选择一个边界索引即可$k$，其中之前的位置$k$都是`e`和职位来自$k$往后都是`t`。 对于每个分割，我们可以计算与原始字符串有多少不匹配，并选择最小值。 

这减少了从所有字符串的指数搜索到线性扫描的问题$n+1$分割点，每个点使用前缀计数在恒定时间内进行评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有字符串进行暴力破解 |$O(n \cdot 2^n)$|$O(1)$| 太慢了 |
 | 尝试所有具有前缀计数的分割点 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将任何有效的最终时间表解释为从饮食到训练的单一过渡。 任务变成选择这种转变发生的位置。 

1. 预先计算前缀计数有多少`t`出现到每个位置。 这使我们能够快速知道如果我们强制将前缀全部替换，则需要进行多少更改`e`。 
2. 同样，计算后缀数有多少`e`从每个位置开始出现。 这让我们可以评估如果我们强制后缀全部为后缀，则需要进行多少更改`t`。 
3.对于每个可能的分割点$k$，将其解释为：位置$[0, k-1]$变得`e`，和职位$[k, n-1]$变得`t`。 
4. 计算本次拆分的成本：`t`在前缀加上数量`e`在后缀中。 这是使字符串与分割形式匹配所需的翻转次数。 
5. 以最小的成本跟踪分割情况。 如果多次分割的成本相同，则任何一种都是可以接受的。 
6. 找到最佳分割后，直接从该分割重建最终字符串。 

这样做的原因是每个有效字符串之间都有一个边界`e`和`t`。 任何违反约束的行为`te`当我们强制执行这样的边界时，它就会完全消失，因此我们不会错过任何有效的候选者。 

### 为什么它有效

 任何有效的最终配置都必须避免子字符串`te`。 如果这样的子字符串存在于任何地方，则表示违规。 在全球范围内消除所有此类违法行为的唯一方法是确保一旦`t`出现，没有`e`稍后出现。 这迫使字符串的结构变成单调形式：一块`e`接下来是一个块`t`。 由于每个有效的解决方案都属于这个系列，并且我们测试了该系列中所有可能的边界，因此保证最佳解决方案位于其中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    # prefix_t[i]: number of 't' in s[0:i]
    prefix_t = [0] * (n + 1)
    for i in range(n):
        prefix_t[i + 1] = prefix_t[i] + (1 if s[i] == 't' else 0)

    # suffix_e[i]: number of 'e' in s[i:n]
    suffix_e = [0] * (n + 1)
    for i in range(n - 1, -1, -1):
        suffix_e[i] = suffix_e[i + 1] + (1 if s[i] == 'e' else 0)

    best_cost = n + 1
    best_k = 0

    for k in range(n + 1):
        cost = prefix_t[k] + suffix_e[k]
        if cost < best_cost:
            best_cost = cost
            best_k = k

    res = []
    for i in range(n):
        if i < best_k:
            res.append('e')
        else:
            res.append('t')

    print(best_cost)
    print("".join(res))

if __name__ == "__main__":
    solve()
```该代码首先构建前缀和后缀累加器，以便可以在恒定时间内评估每个分割。 分割位置上的主循环选择最佳边界。 重建是所选结构的直接应用，避免了增量编辑或模拟的任何需要。 

一个常见的错误是通过扫描每个分割的字符串来重新计算成本，这会导致$O(n^2)$。 另一个是忘记分割点可以是$0$或者$n$，对应于所有`t`或全部`e`。

 ## 边缘情况

 无论值如何，单字符字符串始终满足约束，算法自然会考虑两个边界位置$k = 0$和$k = 1$，产生零成本。 

完全交替的字符串，例如`tetete`很有趣，因为局部纠正会失败：翻转一个违规行为可能会在附近产生另一个违规行为。 基于拆分的公式通过采用全局结构完全避免了这种情况，并且前缀后缀成本正确地捕获了一次评估中的所有必要更改。 

完全统一的字符串不需要任何更改，并且成本函数可以正确评估所有拆分，至少一次拆分产生零成本，从而保留原始字符串。
