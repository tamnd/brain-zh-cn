 ## 解决方案
 ## 问题理解

 我们正在使用带有基数的位置数字系统$k$，其中数字是使用以下数字编写的$0$到$k-1$。 一个数字被称为“足够圆”，如果，当用基数书写时$k$，其表示至少以$n$零数字。 换句话说，这个数字可以被$k^n$。 

任务是按升序列出具有此属性的所有正整数并返回$i$- 普通十进制表示中的第一个。 

从约束条件来看，$k$可以大到$10^9$,$n$可以达到$100$， 和$i$可以达到$10^9$。 答案保证符合$10^{18}$，这意味着我们可以安全地使用 64 位算术逻辑并避免超出该范围的溢出问题。 

一个天真的解释是迭代自然数，将每个数字转换为基数$k$，计算尾随零，并选择那些符合条件的零。 这会立即失败，因为检查一个号码需要花费$O(\log_k x)$，我们可能需要最多$10^9$有效数字，这意味着我们将模拟最多$10^9$的候选人，这远远超出了可行的限度。 

出现微妙的边缘情况时$n = 0$。 那么每个数字都是有效的，答案很简单$i$。 任何错误地强制整除性的解决方案$k^n$作为一个严格的条件，即使$n=0$必须小心避免强制除以$1$-类似可能扭曲索引的逻辑。 

另一个边缘情况是当$k = 2$和$n$很大。 虽然$k^n$增长很快，问题不需要我们枚举大量数字； 相反，我们必须从结构上推理尾随零的数字在基数中的表现$k$。 

## 方法

 关键的观察是一个数字至少有$n$基数尾随零$k$当且仅当它能被$k^n$。 这将问题转化为一个简单的算术级数问题：我们被要求枚举所有正倍数$k^n$。 

所以足够整数的序列是：$$k^n,\; 2k^n,\; 3k^n,\; \dots$$这$i$因此，第 -th 个数字恰好是：$$i \cdot k^n$$暴力方法将显式测试每个整数，将其转换为基数$k$，并检查尾随零。 这在概念上是可行的，因为基数转换很简单，但在计算上却失败了，因为有效数字的密度是$1/k^n$，所以我们仍然会扫描到$i \cdot k^n$最坏情况下的候选人。 

优化的方法完全绕过表示并依赖于基数中尾随零之间的等价性$k$和整除性$k^n$。 一旦建立了这种等价性，问题就简化为计算幂和乘法。 

唯一的技术难点是计算$k^n$安全。 自从$n \le 100$，通过重复平方求幂在 64 位算术中是足够且稳定的，因为可以保证最终答案不超过$10^{18}$。

 ## 算法演练

 ## 1. 解释base中的条件$k$以以下结尾的数字$n$基数为零$k$意味着它可以被整除$k^n$。 这会将基于数字的条件转换为纯算术整除条件。 

## 2. 计算$k^n$我们计算$p = k^n$使用快速求幂。 这一步是必要的，因为直接乘法$n$时间可能会溢出或效率低下，如果$n$更大。 

## 3. 构造序列

 所有有效数字形成一个算术级数：$$p, 2p, 3p, \dots$$所以$i$- 第一个有效数字就是$i \cdot p$。 

## 4.输出结果

 返回$i \cdot k^n$作为十进制形式的答案。 

### 为什么它有效

 关键的不变量是每个数字至少$n$基数尾随零$k$恰好是$k^n$，以及每个倍数$k^n$至少有$n$尾随零。 这是位置表示的直接结果：每个尾随零对应于一个因子$k$。 因此，通过增加值对数字进行排序可以保留倍数的顺序，并且$i$第 - 个有效数字正是$i$的倍数$k^n$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def pow_fast(a, e):
    res = 1
    while e > 0:
        if e & 1:
            res *= a
        a *= a
        e >>= 1
    return res

def solve():
    k = int(input())
    n = int(input())
    i = int(input())

    p = pow_fast(k, n)
    print(i * p)

if __name__ == "__main__":
    solve()
```实现首先读取$k$,$n$， 和$i$。 它计算$k^n$使用二进制求幂，当设置当前指数位时，它会重复对基数进行平方并乘以结果。 最后，将结果乘以$i$，直接产生$i$的倍数$k^n$。 

一个微妙的点是我们从不显式检查尾随零或执行基数转换。 整个基于数字的条件被吸收到求幂中。

 ## 边缘情况

 当$n = 0$，每个数字都是有效的，因为每个整数在任何基数中都至少有零个尾随零。 该公式仍然有效，因为$k^0 = 1$，所以答案就变成了$i \cdot 1 = i$，匹配预期序列。 

什么时候$k = 2$和$n$很大，说$n = 60$，我们计算$2^{60}$，仍在 64 位范围内。 该序列变成了的倍数$2^{60}$，以及$i$第一项只是一个按比例缩放的整数。 该算法从不直接依赖于二进制表示结构，因此不存在误解尾随零的风险。 

什么时候$i = 1$，答案正是$k^n$。 这是最小的有效数字，算法会正确返回它，而无需任何逐一调整，因为索引从第一个倍数开始。
