 ## 解决方案
 ## 问题理解

 我们得到了两个兄弟的两个不同生日的快照，他们总是在一年中的同一天庆祝，这意味着他们的年龄总是每年同步增加一岁。 

在过去的某个生日，维蒂亚`n`岁，他的兄弟`m`岁了。 若干年后，今天又是一个生日，现在维蒂亚的年龄和他哥哥的年龄都增加了同样的时间。 今天我们被告知他们的年龄之间的关系：Vitya 比他的兄弟年轻 1 倍`k`，意味着弟弟现在的年龄正好是`k`乘以 Vitya 目前的年龄。 

任务是确定这样的时间线是否可能，如果可能，则计算 Vitya 当前的年龄。 如果约束不一致，我们必须输出`-1`。 

关键约束结构非常严格：所有年龄最多10000，并且`k`也最多是10000。 这立即表明需要 O(1) 或 O(log n) 代数解，因为在问题的更一般版本中，多年来的任何模拟都是不必要的并且太慢。 

当派生年龄不是整数或产生负时间级数时，会出现微妙的失败情况。 例如，如果`n = 1, m = 10, k = 2`，那么方程意味着今天的分数年龄，这在这个离散年度模型中是不可能的。 当计算的“时间平移”需要时间倒退时，就会发生另一种失败情况，这意味着得出的当前年龄小于记录的过去年龄。 

## 方法

 如果我们尝试从过去的状态向前模拟`(n, m)`年复一年，我们将增加两个年龄，直到达到比率条件`brother = k * Vitya`已满足。 在最坏的情况下，有效配置之间的差距可能很大，长达 10,000 年，而在问题的广义版本中，范围可能会大得多。 虽然在这里仍然可行，但这种方法是不必要的，并且隐藏了问题的结构。 

关键的观察结果是，两个年龄都随着时间线性增长。 如果我们让`t`是 Vitya 目前的年龄并且`x`是自记录状态以来经过的年数，则系统变为两个线性方程：

 维蒂亚：`t = n + x`兄弟：`k * t = m + x`这将问题从动态过程简化为求解简单的线性系统。 消除`x`给出一个直接方程`t`，充分判断配置是否有效。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 多年来的模拟| O(最大年龄差) | O(1) | O(1) | 可以接受但没有必要 |
 | 代数解 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取数值`n`,`m`， 和`k`。 这些代表了过去的快照和今天的固定乘数关系。 
2. 使用时移表达未知数`x`，使得当前年龄变为`t = n + x`和`k * t = m + x`。 这反映了两个年龄随着时间的推移而同等增长的事实。 
3.消除`x`通过减去方程，得出`k * t - t = m - n`。 此步骤消除了未知的时间偏移并隔离了当前年龄。 
4. 对左边进行因式分解，得到`t * (k - 1) = m - n`。 这将问题简化为检查是否存在单个整数解。 
5. 计算`t = (m - n) / (k - 1)`仅当`(m - n)`可以整除`(k - 1)`。 如果它不可整除，则该场景无法对应整数年增长，因此答案无效。 
6. 验证计算出的`t`至少是`n`，因为时间仅从记录的状态向前移动。 如果`t < n`，隐含的偏移将为负，这与设置相矛盾。 
7. 如果两个条件都成立，则输出`t`。 否则输出`-1`。 

### 为什么它有效

 核心不变量是，随着时间的推移，兄弟俩的衰老速度相同，这意味着他们的年龄差异保持不变。 这将系统变成两个具有相同斜率的仿射线性函数。 任何有效的解决方案都必须位于这些线的交点处。 如果交点不是整数或暗示负的经过时间，则没有有效的年度增量序列可以产生所需的比率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
m = int(input())
k = int(input())

# t * (k - 1) = m - n
num = m - n
den = k - 1

if num % den != 0:
    print(-1)
else:
    t = num // den
    if t < n:
        print(-1)
    else:
        print(t)
```该实现直接对导出的方程进行编码。 唯一微妙的部分是在执行除法之前确保整数整除性，因为即使界限很小，浮点运算也会引入精度问题。 第二个检查强制执行时间一致性，确保计算出的当前年龄不早于记录的年龄。

 ## 边缘情况

 一种极端情况是比率强制得出非整数结果。 例如，`n = 3, m = 10, k = 2`导致`t = 7/1 = 7`，这看起来有效，但稍作修改，例如`n = 3, m = 11, k = 2`给出`t = 8/1`它在数字上是有效的，但不对应于从起始状态开始的整数年对齐，因为导出的时间偏移在原始系统中变成了分数。 

另一种情况是计算的当前年龄早于记录的年龄。 例如，`n = 5, m = 20, k = 2`产量`t = 15`，这是有效的，但是如果输入是`n = 10, m = 20, k = 2`，我们得到`t = 10`，意味着已经过去了零时间，这是有效的，而任何较小的结果都表明时间向后移动，必须被拒绝。 

这些情况由可分性检查和单调性检查直接处理`t >= n`，确保每个接受的解决方案都对应于物理上一致的生日进程。
