 ## 解决方案
 ## 问题理解

 我们得到一个字符串，并被要求对其进行多个独立的查询。 每个查询都集中在一个连续的子字符串上。 对于该子字符串，我们可以修改字符，其中一个操作意味着用任何其他小写英文字母替换单个字符。 每个查询的目标是确定所需的此类替换的最小数量，以便生成的子字符串具有非常严格的属性：它不得包含任何长度至少为 2 的回文子字符串。 

重新表述这一条件的一个关键方法是局部思考。 长度为 2 的回文就是两个相等的相邻字符。 长度为 3 的回文是第一个和第三个字符匹配的模式。 任何较长的回文都必然包含这些小模式作为其结构的一部分，因此避免长度至少为 2 的所有回文子串相当于避免两个简单的局部模式：相等的邻居和相等的字符，中间有一个字符。 

这将问题从全局组合条件转变为对连续位置三元组的纯粹局部约束。 这是使问题易于处理的关键结构简化。 

约束最多允许 100000 个字符和 100000 个查询。 这立即排除了任何通过线性或二次扫描从头开始重新计算每个查询的解决方案的方法。 在每个查询几乎跨越整个字符串的最坏情况下，即使每个查询的 O（子字符串长度）也会太慢。 该解决方案必须预处理信息或将每个查询减少到接近恒定时间。 

重叠约束会产生微妙的边缘情况。 一种幼稚的方法可能会尝试独立计算违规次数，但修复一个字符可以同时消除多个违规行为。 例如，在像“aaaa”这样的字符串中，单个更改可以立即消除多个相邻的相等违规。 这意味着我们不能简单地独立对待每个不良模式而不考虑交互作用。 

## 方法

 蛮力的想法很简单。 对于每个查询，我们获取子字符串并模拟从左到右构建有效字符串。 在每个位置，我们决定是保留当前字符还是更改它，使其与前一个字符或前两步的字符不匹配。 我们通过贪婪地修复出现的违规行为来暗中尝试所有可能性。 这是可行的，因为约束仅取决于最后两个字符，因此从左到右的构造足以为固定子字符串产生最佳结果。 

然而，对每个查询进行这种模拟的成本是 O(子串长度)，在最坏的情况下会退化为 O(NQ)。 当 N 和 Q 都达到 100000 时，这变得太慢了。 

关键的观察是我们实际上不需要模拟每个查询的最佳构造。 我们需要模拟的唯一原因是了解有多少冲突迫使我们做出改变。 但是，一旦我们根据局部模式重写了禁止结构，我们就可以直接计算每个查询范围内有多少位置是“有问题的”。 每个这样的有问题的位置都对应于至少一个所需的修改，并且约束的结构确保可以解决这些局部违规，而无需以改变最小计数的方式在其他地方创建新的违规。 

这将每个查询减少为对子字符串内两个简单模式的出现次数进行计数：相邻相等字符和间隔一个字符的相等。 

我们可以针对这些条件预处理前缀数组，并在 O(1) 中回答每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(NQ) | O(1) | O(1) | 太慢了|
 | 违规前缀计数| O(N + Q) | O(N) | 已接受 |

## 算法演练

 1. 我们扫描一次字符串并识别所有会强制对任何有效结构进行修改的局部违规行为。 如果位置 i 与前一个字符或它之前两个位置的字符匹配，则在位置 i 处发生违规。 
2. 我们为这些违规行为构建前缀和，以便可以在恒定时间内评估任何查询范围。 每个前缀条目存储到该索引为止存在的违规点数量。 
3. 对于查询 [L, R]，我们使用前缀和计算有多少个违规点位于区间内。 
4. 查询的答案正是这个计数。 

这样做的原因是这些本地检查完全捕获了每个禁止的回文模式。 任何更长的回文都必须在其内部的某个地方表现为这些局部等式之一，因此消除所有它们对于使子串有效来说是充分且必要的。 

### 为什么它有效

 “没有长度至少为 2 的回文子串”的条件崩溃为禁止两种局部模式：邻居之间相等以及相距两步的字符之间相等。 每个无效结构必须至少包含一个这样的本地模式，并且每个这样的本地模式都可以通过更改其端点之一来独立修复。 这使得所需更改的最小数量等于子字符串中违规位置的数量，并且前缀和允许直接查询这些计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    bad = [0] * n

    for i in range(n):
        if i > 0 and s[i] == s[i - 1]:
            bad[i] = 1
        if i > 1 and s[i] == s[i - 2]:
            bad[i] = 1

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + bad[i]

    q = int(input().strip())
    out = []

    for _ in range(q):
        l, r = map(int, input().split())
        l -= 1
        r -= 1
        out.append(str(pref[r + 1] - pref[l]))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先标记参与禁止本地配置的每个索引。 如果索引与其直接前一个索引或与后退两步的字符继续相等，则该索引被视为“坏”。 该数组上的前缀和允许我们通过减去范围来在恒定时间内回答每个查询。 

一个微妙的实现细节是我们标记模式中的当前位置而不是较早的位置。 这避免了重复计算问题，并确保每次违规都归因于整个前缀结构中一致的单个索引。

 ## 边缘情况

 关键的边缘情况是大量相同的字符，例如`aaaaaa`。 在这种情况下，第一个位置之后的每个位置都会产生相邻冲突和距离二冲突。 该算法在坏数组中独立标记每个位置。 前缀总和正确地累积了所有这些冲突，并且每个查询正确地反映了所选范围内有多少这样的冲突。 

另一个边缘情况是交替模式，例如`ababab`。 此处，没有相邻的重复项，但每隔三个字符就会重复前两个步骤。 错误的标记准确地捕获了这些位置，并且前缀和再次捕获了所需修改的正确数量，而无需任何模拟。
