 ## 解决方案
 ## 问题理解

 我们得到了一堆石头。 两名玩家轮流轮流，每轮一名玩家移除一定数量的石子。 限制是移除的石子数量必须严格小于当前堆大小的最高有效位值。 

最高有效位值是不超过当前数字的 2 的最高幂。 如果当前堆大小是$x$，其二进制表示中的最高设置位是$2^k$，那么玩家可以从其中删除任何数字$1$最多$2^k - 1$。 移除石子后，石子堆会减少，回合将转移给另一位玩家。 当牌堆为零时，无法做出有效动作的玩家就输了。 

任务是确定许多独立的起始值$N$，第一个玩家是否在最优玩法下强制获胜。 

约束条件达到$N \le 10^{18}$，这立即排除了任何游戏模拟。 即使是一场比赛也能持续下去$O(N)$在最坏的情况下移动，并且最多可以有$10^5$测试用例。 因此，任何解决方案都必须在每个测试用例的最多对数或恒定时间内计算答案，可能使用二进制表示的属性。 

一种简单的方法会尝试模拟所有可能的动作或动态计算游戏状态。 这不仅因为时间而失败，还因为状态空间巨大并且转换非线性地依赖于当前值的 MSB。 

当出现微妙的边缘情况时$N$略低于或略高于 2 的幂。 例如，在$N = 7$，最高有效位是$4$，但在$N = 8$，最高有效位是$8$。 这极大地改变了移动范围，并使关于单调性的天真的直觉变得不正确。 任何假设相邻整数具有类似行为的解决方案都将在这里失败。 

## 方法

 如果我们尝试直接解决游戏，如果存在向失败状态的移动，我们可以将状态定义为获胜。 从一个位置$x$，玩家可以移动到任意状态$x - k$在哪里$1 \le k < \text{MSB}(x)$。 暴力递归将计算每个数字的获胜状态$1$到$N$，检查所有有效的动作。 

这在原则上是正确的，因为它完全遵循游戏规则。 然而，对于每个州$x$，我们可能有多达$O(x)$最坏情况下的转换（当 MSB 相对于$x$），并为所有人计算这个$x$最多$10^{18}$是不可能的。 即使我们限制$N$，单个测试用例已经超出了任何可行的计算。 

关键的观察结果是，影响游戏的唯一结构是当前两个的最高功率。 对于任何间隔$[2^k, 2^{k+1}-1]$，MSB固定为$2^k$，因此该范围内的所有位置都具有相同的移动约束。 这将游戏划分为独立的块，并且在每个块内，行为以可以精确分析的方式变得周期性。 

在 MSB 等于的块内$2^k$，允许的移动减去任何$1$到$2^k - 1$。 这意味着从一个位置$x$，你可以达到区间内的每个值$[x-(2^k-1), x-1]$。 这是一个经典的可达性结构：只有当所有可达状态都获胜时，位置才会失败。 

关键的简化是，在每个这样的块内，失败的位置形成一个简单的模式：每个位置的余数模$2^k$为零的行为与其他不同。 这导致了这样一个事实：游戏的结果仅取决于是否$N$是否是其 MSB 的倍数。 

具体来说，如果$N = 2^k$，那么第一个玩家就处于失败的位置，因为任何举动都会将游戏带入对手获得严格优势的前一个区块。 否则，如果$N > 2^k$，第一个玩家总是可以将堆减少到恰好$2^k$，迫使对手处于失败位置。 

这将整个问题简化为对二进制结构的简单检查$N$。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 计算两个的最大幂$p$这样$p \le N$。 这是最高设置位$N$。 我们可以使用位运算来计算它。 
2. 检查是否$N$正好等于$p$。 这决定了该数字本身是否是二的幂。 
3.如果$N = p$，宣布第一个玩家的位置失败。 
4. 否则，宣布第一个玩家获胜。 

步骤 2 足够的原因是 MSB 定义了整个移动范围。 一次$N$超过2的幂，总有足够的灵活性迫使对手进入边界状态$p$，正在丢失。 

### 为什么它有效

 游戏自然按照 MSB 间隔进行划分。 每个区间内$[2^k, 2^{k+1}-1]$，除了左端点之外的每个位置都可以直接移动到$2^k$。 该端点很特殊，因为从$2^k$，每一步棋都严格低于该区间，对手实际上获得了更有利的位置。 这会在每个时间间隔内创建一个失败状态，并且它恰好是 2 的幂。 

由于每个非二的幂位置都可以直接进入失败状态，因此它是获胜状态。 由于二次幂位置无法保持在同一区间内，因此它们正在失败。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        p = 1 << (n.bit_length() - 1)
        if n == p:
            print("Second")
        else:
            print("First")

if __name__ == "__main__":
    solve()
```该解决方案计算两个不超过的最高幂$n$使用`bit_length`，直接给出最高设置位的位置。 移位重建了二的力量。 

关键的决定是平等检查。 如果$n$正好是2的幂，位置就输了。 否则，它就会获胜，因为玩家总是可以一步移动到两种状态的幂。 

该实现避免了任何模拟并纯粹依赖于二进制结构，考虑到上限，这是至关重要的$10^{18}$。

 ## 边缘情况

 对于$N = 1$，MSB为1，因此不存在小于1的正整数，使得第一个玩家无法移动。 该算法计算$p = 1$并正确输出丢失。 

为了$N = 2^k$， 例如$N = 16$，最高有效位等于$N$本身。 条件$N == p$触发，将其标记为失败。 任何举动都会落入前一个区间，对手可以立即做出反应以恢复结构性优势。 

为了$N = 2^k + 1$， 例如$N = 9$，MSB 为 8。玩家可以移走 1 个石头，达到 8，这对于对手来说是一个失败的位置。 该算法将其归类为获胜，因为$9 \ne 8$，匹配建设性策略。
