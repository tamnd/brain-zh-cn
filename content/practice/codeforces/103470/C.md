 ## 解决方案
 ## 问题理解

 我们得到一个表示一行上的值的整数数组，并且允许我们选择一个连续的段并将该段内的每个值增加一个固定常数`k`。 

最多执行一次此操作后，我们查看整个数组并计算其模式，即出现最频繁的值。 任务不是输出众数本身，而是在最佳选择段后最大化某个值可以作为众数出现的次数。 

一个关键的微妙之处在于该操作不仅仅均匀地增加计数。 所选段内的每个元素都会更改其值，这既可以破坏旧值的出现，也可以创建新值的出现。 最终的答案取决于在失去一些频率和获得其他频率之间的仔细权衡。 

从约束的角度来看，数组大小足够大，以至于`O(n^2)`甚至`O(n sqrt n)`每个值的模拟太慢。 该解决方案必须依赖于构建问题，以便在所有推理中仅处理每个元素恒定的次数。 这强烈暗示使用位置列表和段推理的线性或近线性方法，而不是对整个数组进行逐值模拟。 

主要的边缘情况来自于操作如何与相同值交互以及之间的冲突`x`和`x + k`。 例如，如果所有元素都相等并且`k = 0`，那么任何段都不执行任何操作，但仍算作有效操作。 一个简单的解决方案可能会重复计算收益。 当一个值出现时，会发生另一个棘手的情况`v`和`v - k`两者都密集出现，因为最佳段可能有选择地仅包含数组的一部分，以最大限度地提高转换率，同时避免删除。 

## 方法

 蛮力的想法很简单。 我们尝试每一个可能的环节`[l, r]`，应用增量，重建数组，并计算所有值的频率。 这需要计算频率`O(n)`每个段，并且有`O(n^2)`段，给予`O(n^3)`总的复杂性。 即使我们使用前缀结构优化频率重新计算，我们仍然面临`O(n^2)`分段评价，远远超出了极限。 

关键的观察是我们永远不需要显式重建数组。 重要的是所选段如何转换特定值的计数`v`。 在该段内，每次出现`v`丢失了，并且每次出现`v - k`变成`v`。 该部分之外的所有内容都保持不变。 

So for a fixed target value`v`，最终频率变为原始总计数`v`，加上该部门的贡献：我们获得`+1`对于每个`v - k`在段内并丢失`-1`对于每个`v`段内。 其他一切都是中性的。 这将问题转化为找到一个使权重总和最大化的分段，其中每个位置都有贡献`+1`,`-1`， 或者`0`取决于其相对于的值`v`。 

然而，为每个独立地执行此操作`v`还是会太慢。 第二个关键见解是有意义的交互仅发生在值对之间`(x, x + k)`。 每个元素仅作为一个元素做出贡献`+1`为了`v = x + k`或作为`-1`为了`v = x`。 这意味着我们可以按值对对计算进行分组，而不是迭代所有可能的`v`。 

对于每个值`x`，我们计算出最佳分段效果`x + k`更频繁，仅使用位置`x`和职位`x + k`。 由于每个索引恰好属于一个值，因此所有对的总处理是线性的`n`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n3) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 构建从每个不同值到其在数组中的位置的排序列表的映射。 这使我们能够推断事件的发生，而无需重复扫描整个数组。 
2. 计算数组中每个值的基线频率。 如果我们选择不执行任何操作，这已经代表了答案。 
3. 对于每个不同的值`x`，将其视为对提升价值的贡献`v = x + k`。 收集两个有序列表：所有位​​置`x`以及所有位置`x + k`。 
4. 按递增索引顺序合并这两个位置列表。 合并时，分配权重`+1`对于出现的`x`和`-1`对于出现的`x + k`。 这对通过选择片段引起的增益和损失结构进行编码。 
5. 对该合并序列运行 Kadane 式扫描，将其视为沿数组的贡献序列。 最佳子阵列和对应于最大化转换净增益的最佳片段`x`进入`x + k`。 
6. 将此最佳增益添加到基线频率`x + k`，因为这是正在改进的值。 用此结果更新全局答案。 
7. 处理完所有值后`x`，将所有获得的结果与基线最大频率进行比较，并输出最佳值。 

### 为什么它有效

 固定目标值`v`。 任何操作都会影响以下情况的发生`v`仅通过两种机制：删除现有的`v`在选定的段内并创建新的`v`从`v - k`。 这种依赖性纯粹是局部的位置`v`和`v - k`，并且独立于所有其他值。 因此，每一个固定的最优解`v`完全对应于选择一个子数组来最大化总和`+1`和`-1`对这两个职位集的贡献。 由于片段的每个有效变换都在此合并序列中表示，并且每个合并序列对应于真实片段，因此 Kadane 最大值正确捕获了最佳可实现增益。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    pos = {}
    freq = {}

    for i, v in enumerate(a):
        freq[v] = freq.get(v, 0) + 1
        if v not in pos:
            pos[v] = []
        pos[v].append(i)

    ans = max(freq.values())

    for x in pos:
        y = x + k
        if y not in pos:
            continue

        A = pos[x]
        B = pos[y]

        i = j = 0
        cur = 0
        best = 0

        while i < len(A) or j < len(B):
            if j == len(B) or (i < len(A) and A[i] < B[j]):
                cur += 1
                i += 1
            else:
                cur -= 1
                j += 1

            if cur < 0:
                cur = 0

            if cur > best:
                best = cur

        ans = max(ans, freq[y] + best)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先构建频率和位置列表，以便我们可以推理值转换，而无需重复扫描数组。 基线答案在任何操作之前被初始化为最大频率。 

对于每个值`x`，我们尝试改进`x + k`。 我们合并出现的`x`和`x + k`按索引顺序，处理`x`作为增益和`x + k`作为损失。 每当运行总和变为负数时运行 Kadane 式重置可确保我们始终考虑数组的最佳连续段。 

微妙的一点是我们从不显式计算段边界`[l, r]`。 相反，合并遍历隐式模拟它们，因为合并列表的每个前缀对应于原始数组排序中的有效间隔。 

## 工作示例

 ### 示例 1

 考虑`a = [1, 2, 1, 2]`,`k = 1`。 

我们有`pos(1) = [0, 2]`,`pos(2) = [1, 3]`。 基线频率均为`2`。 

我们评估转型`1 -> 2`。 

| 步骤| 活动 | 当前总和| 最佳|
 | --- | --- | --- | --- |
 | 0 | +1（1 为 0）| 1 | 1 |
 | 1 | -1（2 于 1）| 0 | 1 |
 | 2 | +1（1 于 2）| 1 | 1 |
 | 3 | -1（2 于 3）| 0 | 1 |

 最佳增益是`1`，所以最终的价值答案`2`是`2 + 1 = 3`。 

这显示了如何选择段`[0,2]`转换一`1`进入`2`同时最大限度地减少损失。 

### 示例 2

 考虑`a = [3, 3, 3, 3]`,`k = 0`。 

所有值都是相同的，因此每个转换都完美重叠。 

| 步骤| 活动 | 当前总和| 最佳|
 | --- | --- | --- | --- |
 | 0 | +1 | 1 | 1 |
 | 1 | -1 | 0 | 1 |
 | 2 | +1 | 1 | 1 |
 | 3 | -1 | 0 | 1 |

 这里`best = 1`，但自从`k = 0`，变换不会改变值，所以基线`freq[3] = 4`主导，给出答案`4`。 

这表明，当操作没有提供真正的好处时，算法可以安全地回退到未更改的配置。

 ## 边缘情况

 一个重要的边缘情况是当`k = 0`。 在这种情况下，每个转换都会将一个值映射到自身，这意味着任何选定的段仅同时减去和添加相同的值。 除了平凡的子阵列之外，合并的贡献序列总是相互抵消，因此最佳结果始终是原始最大频率。 该算法自然地处理这个问题，因为计算出的增益永远不会超过基线的零改进。 

当一个值出现时，会出现另一种边缘情况`x + k`数组中不存在。 在这种情况下，任何段仅删除出现的`x + k`不产生新的，所以最好的策略是永远不要考虑这种转变。 该算法完全跳过这些对，保持基线答案不变，这与正确的行为相匹配。 

当出现以下情况时，会发生最后一个微妙的情况`x`和`x + k`是严重交错的。 合并的 Kadane 扫描确保我们始终选择一个连续区域，以最大化净转化，同时避免删除占主导地位的区域。 每当运行总和变为负数时重置为零，保证我们永远不会将有害的前缀决策带入未来的段。
