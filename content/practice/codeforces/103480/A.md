 ## 问题理解

 我们得到了几个测试用例。 在每种情况下，我们都会收到多个单个字符集。 每个字符最多可以使用一次，并且我们可以以任意顺序重新排列所选字符。 目标是选择这些字符的一些子集并将它们排列成回文字符串，从而最大化该回文的长度。 

回文完全取决于有多少个字符可以围绕中心对称配对，再加上可能有一个未配对的字符放置在中间。 这种结构施加了核心约束：字符只能通过成对的方式构成回文，除了最多一个可以坐在中心的剩余字符。 

输入大小最多允许 10 个测试用例，每个测试用例最多 100,000 个字符。 这立即排除了任何尝试构造排列或执行组合搜索的解决方案。 每个测试用例的线性扫描是可以接受的，但是每个测试用例的字符数的二次方都会超出限制。 

当所有字符都不同时，就会出现微妙的边缘情况。 例如，输入类似`A B C D`回文中只允许有一个字符，因为不存在对。 另一种边缘情况是，例如，一个字符出现多次而其他字符出现一次`a a a b c`。 在这里，最好的回文使用所有偶数对`a`并忽略除可能的一个中心字符之外的单例。 贪婪地交替或试图“向外构建”而不计算频率的幼稚方法会错误地计算这些贡献。 

## 方法

 蛮力的想法是考虑字符的所有子集和每个子集的所有排列，检查每个排列是否形成回文并跟踪最大长度。 这在概念上很简单，因为它探索了有效结构的整个搜索空间。 然而，即使单独选择子集也已经给出了 2^n 种可能性，并且排列将其乘以阶乘增长。 当 n 达到 100,000 时，这种方法是不可行的。 

回文结构大大简化了问题。 回文完全由字符频率决定，而不是顺序。 每个角色贡献尽可能多的镜像对。 每对贡献两个字符来得出最终答案。 在形成所有可能的对后，如果至少有一个字符未使用，我们可以将一个这样的字符恰好放置在中心，将长度增加一。 

这减少了从组合排列到频率计数的问题。 关键的观察结果是，最佳结构从来不取决于选择了哪些特定位置，而只取决于所有字母可以形成多少对。

 ## 算法演练

 1. 计算每个字符在输入中出现的次数。 这捕获了所有相关的结构，因为一旦频率已知，排列就无关紧要了。 
2. 对于每个字符频率，计算通过整数除以 2 可以形成多少个完整对。每对恰好为回文贡献两个字符。 
3. 将所有这些对的贡献乘以 2 求和。这给出了回文对称边贡献的总长度。 
4. 追踪配对后是否存在至少一个剩余计数为奇数的字符。 这表明至少有一个角色可以作为中心。 
5. 如果存在奇数余数，则在答案中加一以放置中心字符。 
6. 输出最终计算出的长度。 

为什么有效：回文的构造强制对称性，因此左侧使用的每个字符都必须在右侧有一个匹配的对应字符。 这将使用划分为不相交的对。 除了成为中心的一个字符外，任何剩余的字符都不能对称放置。 由于配对对于每个字符都是独立的，并且每个频率都贪婪地最大化，因此最终的结构是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        freq = {}
        
        arr = input().split()
        for c in arr:
            freq[c] = freq.get(c, 0) + 1
        
        length = 0
        has_odd = False
        
        for v in freq.values():
            length += (v // 2) * 2
            if v % 2 == 1:
                has_odd = True
        
        if has_odd:
            length += 1
        
        print(length)

if __name__ == "__main__":
    solve()
```该解决方案是围绕单通道频率累积构建的。 字典存储每个字符的计数，这足够了，因为字符是独立的。 

关键计算是`(v // 2) * 2`，它将频率转换为可用的对称贡献。 乘以 2 可以明确避免失去左右配对的清晰度。 

布尔值`has_odd`至关重要，因为它编码我们是否可以放置中心字符。 所有字符中出现任何一个奇数就足够了，因为只存在一个中心位置。

 ## 边缘情况

 一种重要的边缘情况是没有重复的字符。 对于像这样的输入`a b c d`，频率图产生所有的。 该算法根据对贡献计算零并检测到至少一个奇数值，因此它正确地输出 1。 

另一种情况是只有一个字符出现频率较大，而其他字符都是单例，例如`a a a a b c d`。 该算法形成两对`a`，贡献 4，并使用其中一个单例作为中心，给出 5。剩余的单例不会影响结果，因为它们无法形成额外的对。 

最后的边缘情况是单元素输入。 有输入`A`，对和为零且存在奇数，因此答案变为 1，这与唯一可能的回文匹配。
