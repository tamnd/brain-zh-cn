 ## 解决方案
 ## 问题理解

 我们有两个相同大小的方形网格，每个单元格包含一个整数颜色。 第一个网格上允许的唯一操作是绕其中心顺时针或逆时针旋转 90 度，并且我们可以多次应用此操作。 任务是确定使每个单元位置中的第一个网格与第二个网格完全相同所需的最小旋转次数。 如果没有旋转序列可以将第一个网格转换为第二个网格，我们必须输出 -1。 

关键的观察结果是变换空间非常小。 正方形上的任何 90 度旋转序列最多会循环经过四种不同的状态。 这立即表明答案只能是四种可能性之一：顺时针旋转 0、1、2 或 3 次（或者等效地，顺时针和逆时针步骤的任何组合都会减少到这四种净旋转之一）。 这将看起来像优化问题的问题转化为有限比较问题。 

约束非常小，N 最大为 20，T 最大为 100。这意味着即使重复恒定次数的 O(N²) 比较也非常快。 任何显式构造旋转网格并比较它们的解决方案都会轻松通过。 不需要散列、大型状态空间模拟或高级数据结构。 

主要的微妙边缘情况是对旋转等价性的误解。 一个幼稚的错误是将顺时针和逆时针步骤视为独立的序列，从而导致不必要的探索或错误的计数。 另一种假设是，一些旋转后的部分匹配意味着可扩展性，这是错误的，因为每次旋转都必须保留全局结构。 

例如，考虑：

 输入：```
1
2
1 2
3 4
3 1
4 2
```这里，第二个网格是第一个网格的旋转版本，但没有旋转就没有对齐。 即使存在有效的旋转，从左上角进行的天真的逐元素贪婪检查也会提前失败。 正确答案取决于全局对齐，而不是局部匹配。 

另一个边缘情况是网格相同时。 答案必须是 0，尽管应用 4 次旋转也会返回恒等式。 我们必须确保选择最小非负旋转计数。 

## 方法

 蛮力视角首先模拟所有可能的旋转序列。 从第一个网格开始，我们可以应用 0、1、2、3 或更多次旋转，但每 4 次旋转后网格就会重复。 因此任何更长的序列都相当于前四个状态之一。 对于每个候选旋转状态，我们检查结果网格是否与目标网格匹配。 每次检查的成本为 O(N²)，并且只有四种状态，因此暴力破解已经非常小：O(4N²)。 

关键的结构见解是旋转操作形成了一个 4 阶循环群。这意味着我们不是在搜索可能性树，而是在旋转下评估初始网格的微小固定轨道。 该问题简化为计算最多四个确定性变换并比较它们。 

探索序列或单独混合顺时针和逆时针移动没有任何好处，因为每个序列都会减少到模 4 的净旋转。

 | 最佳（检查 4 次旋转）| O(4·N²) | O(N²) | 已接受 |

 ## 算法演练

 我们显式生成第一个网格的所有四个可能的方向，并将每个方向与第二个网格进行比较。 

1.读取大小为N×N的两个网格A和B。目标是测试B是否与A的任何旋转匹配。 
2. 定义一个将网格顺时针旋转 90 度的函数。 对于每个单元 (i, j)，它移动到 (j, N − 1 − i)。 这种映射保留了结构并确保矩阵的正确刚性旋转。 
3. 从等于A的工作网格开始，并将其视为旋转状态0。 
4. 将当前网格与 B 进行比较。如果完全匹配，则记录当前旋转计数作为候选答案。 
5. 旋转一次 90 度以获得下一个状态。 重复此过程总共 3 次，产生网格的所有不同方向。 
6. 生成所有四个状态后，在与 B 匹配的状态中选择最小旋转计数。如果没有状态匹配，则输出 -1。 

我们显式生成所有状态而不是尝试将 B 转换回 A 的原因是，正向旋转更容易正确实现，并避免逆映射错误。 

### 为什么它有效

 每个允许的操作序列都对应于以某个整数 k 模 4 进行的旋转。从 A 出发的可达构型集合正是 A 在该循环群下的轨道。 由于轨道最多包含四个不同的元素，因此详尽地枚举它们可以保证如果 B 可达，则它将在这些状态中恰好出现一次。 该算法是完整的，因为它检查了旋转下所有可能的等价类，并且它是健全的，因为每个构造的状态都是原始网格的有效旋转。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def rotate(mat):
    n = len(mat)
    res = [[0] * n for _ in range(n)]
    for i in range(n):
        for j in range(n):
            res[j][n - 1 - i] = mat[i][j]
    return res

def same(a, b):
    n = len(a)
    for i in range(n):
        for j in range(n):
            if a[i][j] != b[i][j]:
                return False
    return True

t = int(input())
for _ in range(t):
    n = int(input())
    a = [list(map(int, input().split())) for _ in range(n)]
    b = [list(map(int, input().split())) for _ in range(n)]

    cur = a
    ans = None

    for k in range(4):
        if same(cur, b):
            ans = k if ans is None else min(ans, k)
        cur = rotate(cur)

    print(-1 if ans is None else ans)
```该解决方案维护当前矩阵并重复应用确定性旋转函数。 比较函数检查 O(N²) 中的相等性，在给定约束的情况下，这是有效的。 k 上的循环确保测试所有四种可能的方向，包括身份情况。 

一个常见的错误是不一致地旋转 B 而不是 A，或者忘记在 4 次旋转后矩阵返回到原始状态。 该实现通过严格迭代四次来避免这种情况。

 ## 边缘情况

 一个关键的边缘情况是网格已经相同。 在这种情况下，正确答案是零，并且算法会在 k = 0 时立即捕获该答案，而不会执行不必​​要的旋转。 

另一种边缘情况是只有经过多次旋转才能到达目标，而不仅仅是一次。 例如，3×3 网格通常需要检查所有中间状态； 在一次不匹配后提前停止是不正确的，因为旋转是全局转换。 

最后的边缘情况是不可到达的网格，它们可能共享局部模式但全局不同。 该算法正确地拒绝它们，因为在每个旋转状态的整个矩阵上检查相等性，确保没有部分结构可以错误地触发匹配。
