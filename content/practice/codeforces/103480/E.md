 ## 解决方案
 ## 问题理解

 我们有几个数组，我们必须从每个数组中准确选择一个值。 选择之后，我们在概念上就有了大小为 n 的多重集。 然后，该问题定义了一个合并过程，该过程重复选择两个值，将它们的绝对差值添加到累积成本中，丢弃其中一个值，并继续下去，直到只剩下一个值。 

尽管该过程是动态描述的，但关键的观察结果是总成本仅取决于从阵列中选择的数字，而不取决于任意的模拟选择。 一旦我们确定了每个数组的一个选定值，我们就可以自由地以任何顺序执行合并。 

目标是从每个数组中选择一个元素，以便存在一系列合并，产生尽可能小的绝对差总和。 

从约束来看，n 最多为 10，而所有数组的元素总数最多为 10^6。 这立即表明我们应该避免对选择进行任何指数搜索，但我们可以负担元素总数呈线性或接近线性的算法，可能包含涉及 n 的因子。 

关键的结构困难在于每个数组都有一个约束：我们必须从每个列表中精确挑选一个元素，然后针对所有选定的值优化全局目标。 

当所有数组都包含广泛分离的值时，会出现微妙的边缘情况。 例如，如果一个数组仅包含较小的值，而另一个数组仅包含较大的值，则任何选择都会强制使用较大的最终范围，并且答案将由跨列表极端值主导。 独立优化每个列表的天真贪婪在这里会失败，因为列表之间的交互很重要。 

另一个特殊情况是多个数组包含重复或相同的值。 在这种情况下，最佳解决方案可能来自于将所有选定的值非常紧密地对齐，而独立选择局部中位数的简单方法很容易错过全局紧密配置。 

## 方法

 第一步是重新解释合并过程。 一旦我们确定了选定的值，我们就会重复选择两个数字，支付它们的绝对差值，并丢弃一个数字，同时保留另一个数字。 这相当于通过合并数轴上的一组点来逐渐减少它们。 

推理这个问题的一个有用方法是询问固定选择的集合 S 的最小可能成本是多少。假设我们忽略操作顺序，而是考虑将所有数字连接到一个结构中，其中每个合并对应于两个值之间的一条边，成本等于它们的绝对差。 这会在 S 上生成一棵树，其总权重恰好是所有合并成本的总和。 

现在关键的观察是，在一维中，一组点的最小生成树只需通过对它们进行排序并连接相邻点即可获得。 这棵树的总权重是连续差异的总和，它缩小到 S 中最大和最小元素之间的差异。 

因此，对于每个数组一个元素的任何固定选择，最佳可能成本等于所选值的范围。 

这将问题简化为一种更清晰的形式：从每个数组中选择一个值，以最小化所选值中最大值和最小值之间的差异。 

现在问题变成了一个经典的“覆盖 k 列表的最小范围”问题。 每个数组都是一个排序列表，我们希望从每个列表中选取一个元素，以使所选值尽可能靠近。 

强力解决方案会尝试每个数组一个元素的每一种组合。 如果每个数组有 m 个元素，这会导致 m^n 种可能性，即使 n = 10 也是完全不可行的。

最佳的见解是我们永远不需要探索任意组合。 相反，我们在列表的并集上维护一个滑动窗口，在任何时刻我们都从每个数组中精确挑选一个元素并跟踪当前的最小值和最大值。 然后，我们尝试通过在当前贡献最小值的数组中向前移动来缩小范围。 

这是可行的，因为增加最小值是可能减小范围的唯一方法，同时保留每个数组仅贡献一个活动元素的约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（所有选择）| O(m^n) | O(m^n) | O(n) | 太慢了 |
 | 最优（堆+指针）| O(m log n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个数组视为一个排序列表，并维护一个指向每个列表的指针，表示该数组中当前选择的元素。 

我们还维护一个数据结构，可以快速告诉我们当前选择的值是最小值以及它来自哪个数组。 

1. 为每个数组在索引 0 处初始化一个指针，这意味着我们最初从每个数组中选择最小的元素。 
2. 将所有选定的值插入到一个结构中，该结构允许我们有效地检索当前最小值，同时还跟踪选定值中的当前最大值。 当前范围计算为最大值减去最小值。 
3.反复考虑当前的配置并尝试改进。 在每一步中，确定当前所选元素是所有所选元素中最小的数组。 
4. 将该数组的指针向前移动一个位置，用该数组中的下一个元素替换其所选值。 这个方向正确的原因是增加最小值是潜在缩小总体范围的唯一方法； 移动任何其他指针都无法消除当前最小侧的瓶颈。 
5. 更新指针后，如果新引入的值大于之前的最大值，则更新当前最大值，并重新计算范围。 
6. 继续此过程，直到任何指针到达其数组的末尾，因为此后我们无法再从该数组中选取有效元素。 

### 为什么它有效

 在任何时刻，我们都从每个数组中保留一个有效的选择。 电流范围由这些选定值的最小值和最大值确定。 如果我们想减小范围，则必须减小最大值或增大最小值。 但是，如果不更改当前不是最大值的值，则不可能减小最大值，并且这样做只会冒增加其他地方范围的风险。 唯一局部有效的改进步骤是将当前保持最小值的指针向上移动，因为这是唯一可以在保持有效性的同时增加范围下限的操作。 

这种贪婪的运动确保了随着指针的前进，每个可能的候选范围都被隐式地考虑，并且观察到的最佳范围是全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n, m = map(int, input().split())
    a = []
    for _ in range(n):
        arr = list(map(int, input().split()))
        arr.sort()
        a.append(arr)

    ptr = [0] * n

    heap = []
    current_max = -10**18

    for i in range(n):
        val = a[i][0]
        heapq.heappush(heap, (val, i))
        current_max = max(current_max, val)

    ans = current_max - heap[0][0]

    while True:
        current_min, i = heapq.heappop(heap)

        if ptr[i] + 1 >= len(a[i]):
            break

        ptr[i] += 1
        new_val = a[i][ptr[i]]

        heapq.heappush(heap, (new_val, i))

        if new_val > current_max:
            current_max = new_val

        current_min_new = heap[0][0]
        ans = min(ans, current_max - current_min_new)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现对每个数组进行排序，以便前进的指针始终移动到严格更大的候选者。 堆存储每个数组当前选择的元素，允许有效提取全局最小值。 变量`current_max`跟踪当前选择中的最大值，以便每个步骤都可以在恒定时间内计算范围。 

一个常见的陷阱是忘记仅向上更新最大值。 从头开始重新计算每个步骤仍然有效，但会不必要地缓慢。 另一个微妙的问题是确保在任何列表耗尽时准确终止，因为超出该点我们无法维持有效的选择。

 ## 边缘情况

 当所有数组包含相同的值时，每个选择都会产生一个零范围。 该算法从索引零处的所有指针开始，并立即看到零范围，并且任何指针移动都无法改善它，因此它正确地输出零。 

当一个数组包含极大的值而另一个数组包含极小的值时，初始范围很大，算法会尝试将最小值向上移动。 每次移动都会减少对极小值的依赖，直到找到可实现的最佳重叠，并且最终答案对应于所有数组中最紧密的交叉区域。 

当数组具有不重叠的间隔时，例如一个完全低于另一个，该算法有效地将较低的数组向上推，直到它们遇到最接近的可能区域，然后耗尽一个列表。 耗尽之前的最后一个有效配置给出了最佳边界范围，这正是堆进程捕获的范围。
