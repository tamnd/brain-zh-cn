 ## 问题理解

 我们有两个以数字七段样式编写的时间显示，其中每个数字呈现为固定的 3 x 3 字符块。 每个时间由六位数字组成：两位代表小时，两位代表分钟，两位代表秒。 输入提供两个完整时间，第一个是正确的参考时间，第二个是可能不正确的电子时钟显示的时间。 

每个时间都由三行编码。 每行上并排写有 6 个数字，因此每行的宽度为 18 个字符，因为每个数字正好占据三列。 我们的任务是对两个时间进行解码，比较它们，并确定电子钟是否比实际时间提前、落后或相同。 如果它们不同，我们还会输出以相同的七段表示形式再次格式化的绝对差值。 

限制很小，因为我们只读取固定的 6 行并执行恒定的工作。 真正的计算工作是正确解释数字模式。 任何在恒定时间内扫描网格并解码每个数字的解决方案都将足够快。 除了仔细解析之外无需进行优化。 

常见的失败案例来自于错误地切片数字或不匹配的段模式。 

一种边缘情况是两个时间相同。 例如，如果两个输入都表示`12:34:56`，输出必须恰好是：```
gang gang hao
```接下来是零时差`00:00:00`以数字形式。 幼稚的方法可能仍然会计算差异，并由于符号处理错误而将其标记为早或晚。 

另一个微妙的情况是当差异跨越小时或分钟界限时。 例如，如果实时是`00:00:10`并且电子钟显示`23:59:59`，正确的行为是将它们视为绝对秒差，而不是天真的分量减法。 

关键的挑战不是算术，而是强大的解码和格式一致性。 

## 方法

 暴力破解的想法是将每个数字视为未知模式，并尝试通过扫描所有可能的数字模板来匹配它。 由于只有 10 个数字，并且每个数字占据 3 x 3 的块，因此我们可以将每个提取的块与每个候选数字模式进行比较。 在实践中这已经是常数时间，因为比较的总数是固定的并且很小。 

一种稍微不太结构化的粗暴方法会尝试动态解释段或使用段计数的启发式重建数字，但这变得脆弱且不必要。 

更可靠的方法是预先定义数字 0 到 9 的精确 3 x 3 表示。一旦此映射被固定，解码就变成直接查找：提取每个 3 x 3 块，将其转换为字符串密钥，然后将其映射到数字。 将两次解码为表示总秒数的整数后，我们计算它们的差值，确定符号，并将绝对差值格式化回数字。 

关键的观察结果是该结构是完全刚性的。 每个数字都具有相同的宽度和高度，间距是隐含的，一旦模板已知，就不存在歧义。 这将问题简化为纯粹的解析和格式化。

 ## 算法演练

 1. 定义数字 0 到 9 的标准七段 3 x 3 表示形式。每个数字都存储为通过连接其三行形成的 9 个字符签名。 这给了我们一个稳定的解码密钥。 
2. 阅读前三行和后三行。 每组代表一个编码数字形式的时间值。 
3. 将每条线分成宽度为 3 的六块。 对于每个数字位置，将其三个垂直切片组合成一个 3 x 3 块。 
4. 将每个 3 x 3 块转换为字符串键，并使用预先计算的字典将其映射到相应的数字。 这每次都会重建一个六位数字的字符串。 
5. 使用以下命令将两个解码时间转换为总秒数`hh * 3600 + mm * 60 + ss`。 这给出了用于比较的线性数字尺度。 
6. 比较两个值。 如果相等，则输出相等字符串。 否则通过检查哪个值较大来确定电子时间是提前还是落后。 
7. 计算绝对差（以秒为单位）。 将其转换回`hh mm ss`使用除法和模运算。 
8. 使用从数字到 3 x 3 网格的反向映射，将此差异渲染回七段形式并打印三行。 

它之所以有效，是因为解码是双射的。 每个 3 x 3 模式唯一标识一个数字，因此在解析过程中不会引入歧义。 将时间转换为秒可以准确地保留顺序，因此比较是一致的。 格式化回使用逆映射，确保输入和输出之间的结构对称。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

digit_map = {
    " _ | ||_|": 0,
    "     |  |": 1,
    " _  _||_ ": 2,
    " _  _| _|": 3,
    "   |_|  |": 4,
    " _ |_  _|": 5,
    " _ |_ |_|": 6,
    " _   |  |": 7,
    " _ |_||_|": 8,
    " _ |_| _|": 9
}

rev = {
    0: [" _ ", "| |", "|_|"],
    1: ["   ", "  |", "  |"],
    2: [" _ ", " _|", "|_ "],
    3: [" _ ", " _|", " _|"],
    4: ["   ", "|_|", "  |"],
    5: [" _ ", "|_ ", " _|"],
    6: [" _ ", "|_ ", "|_|"],
    7: [" _ ", "  |", "  |"],
    8: [" _ ", "|_|", "|_|"],
    9: [" _ ", "|_|", " _|"]
}

def decode(lines):
    digits = []
    for i in range(6):
        block = []
        for r in range(3):
            block.append(lines[r][i*3:(i+1)*3])
        key = "".join(block)
        digits.append(str(digit_map[key]))
    return "".join(digits)

def encode(num):
    s = f"{num:06d}"
    out = ["", "", ""]
    for ch in s:
        d = int(ch)
        for i in range(3):
            out[i] += rev[d][i]
    return out

lines1 = [input().rstrip("\n") for _ in range(3)]
lines2 = [input().rstrip("\n") for _ in range(3)]

t1 = decode(lines1)
t2 = decode(lines2)

h1, m1, s1 = int(t1[:2]), int(t1[2:4]), int(t1[4:])
h2, m2, s2 = int(t2[:2]), int(t2[2:4]), int(t2[4:])

sec1 = h1*3600 + m1*60 + s1
sec2 = h2*3600 + m2*60 + s2

if sec1 == sec2:
    print("gang gang hao")
    diff = encode(0)
else:
    if sec2 > sec1:
        print("late")
        diff = encode(sec2 - sec1)
    else:
        print("early")
        diff = encode(sec1 - sec2)

for line in diff:
    print(line)
```解码函数的工作原理是在行中分割三个字符的固定宽度块。 每个数字都是通过堆叠三行来重建的。 编码函数执行相反的操作，通过连接数字模板构建三个输出行。 

一个微妙的实现细节是将差异填充到六位数。 如果没有这个，前导零将会消失并破坏与所需显示格式的对齐。

 ## 边缘情况

 当两个时间相同时，算法将两者解码为相同的六位数字字符串，从而产生相同的秒数。 等式分支在任何减法之前触发，因此不会计算负差。 输出差异被明确强制为零，确保格式化路径仍然产生有效的显示。 

当电子时间比实际时间早很多时，比较仍然有效，因为两个值都被归一化为总秒数。 即使小时和分钟分别表明不同的关系，标量比较也能正确解决排序问题。 绝对差异避免了任何负格式问题。 

当时间差恰好为零秒时，仍然以零调用编码函数，产生`000000`它干净地映射到`00:00:00`呈七段形式。
