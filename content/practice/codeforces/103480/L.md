 ## 解决方案
 ## 问题理解

 我们有一个圆形轨道$n$等距位置标记为$0$到$n-1$。 青蛙从指定位置开始$0$。 每个动作都会迫使青蛙准确地跳跃$x$沿着圆圈向前迈出一步，意思是从位置开始$i$它总是降落在$(i + x) \bmod n$。 

问题是青蛙能否回到起始位置$0$经过一定次数的跳跃后。 由于运动是确定性的，这相当于询问是否重复添加$x$模数$n$最终产生$0$再次。 

输入包含多个测试用例，每个测试用例都是独立的，因此我们必须回答每对的可达性问题$(n, x)$。 

约束允许最多$10^3$测试用例和值$n, x$最多$10^6$。 这立即排除了执行最多的任何模拟$n$在最坏的情况下，每次测试都会跳跃，因为这可能会降级为$10^9$总操作。 

当发生微妙的边缘情况时$x = 0$。 在这种情况下，青蛙永远不会动。 如果它开始于$0$，已经是立刻“回归”了。 所以正确的答案总是“是”$x = 0$， 不管$n$。 

另一个边缘情况是当$x \ge n$。 由于运动是模数$n$，这相当于使用$x \bmod n$。 如果尝试模拟原始跳转，忘记这种减少的简单实现可能会表现得不一致。 

## 方法

 直接模拟从位置开始$0$并重复应用过渡$i \to (i + x) \bmod n$，标记访问过的位置，直到返回到$0$或重复一个状态。 因为只有$n$状态，这总是最多终止于$n$步骤。 

这是正确的，因为它完全遵循运动规则。 然而，在最坏的情况下，当$x = 1$或者$x$与$n$，周期长度变为$n$，以及以上$10^3$这导致的测试用例$O(n \cdot t)$行为，当$n$很大。 

关键的观察结果是该运动是一个模块化算术循环。 可到达位置的集合正是由下式生成的加法子群：$x$模数$n$。 数论的一个经典事实是，该子群由所有的倍数组成$\gcd(n, x)$，循环回到$0$恰好在之后$n / \gcd(n, x)$步骤。 由于我们只关心回归是否发生，所以答案总是肯定的，除了运动不会以有意义的方式改变状态结构的微不足道的情况。 事实上，从$0$，重复加法始终保持在有限循环内，并且必须返回$0$最终对于任何$x$， 包括$x = 0$。 

因此，可达性条件简化为恒定时间检查：对所有情况都返回“是”，除非退化解释禁止移动，但在模算术下甚至$x = 0$让青蛙固定在$0$，仍然满足要求。 

因此，在正确解释模块化循环的情况下，问题就归结为一个微不足道的总是肯定的答案。

 ## 算法演练

 1. 阅读$n$和$x$对于每个测试用例。 这些定义了有限状态空间上的确定性模块化转换系统。 
2. 观察到每个状态都恰好有一个外出边缘，因此该过程最终必须进入一个循环。 
3. 由于系统是有限且确定性的，因此从$0$，路径最终必须重复先前看到的状态。 
4.一旦出现重复，序列就形成一个循环，并且由于该系统是模加法，$0$始终是开始于的循环的一部分$0$。 
5.得出结论，无论选择什么，青蛙总是会回到起始位置$x$。 

看待同一事实的一种更代数的方式是，之后$k$跳跃的位置是$kx \bmod n$。 我们要问是否存在$k > 0$这样$kx \equiv 0 \pmod n$。 选择$k = n$总是有效，因为$nx \equiv 0 \pmod n$，所以回报是有保证的。 

### 为什么它有效

 不变的是之后$k$移动青蛙始终处于该位置$kx \bmod n$。 该表达式永远不会离开循环群$\mathbb{Z}_n$，并乘以$x$地图$\mathbb{Z}_n$进入其自身。 自从$\mathbb{Z}_n$是有限的，状态序列是周期性的，并且周期划分$n$。 因此国家$0$必须重新出现在从$0$，确保答案始终是肯定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n, x = map(int, input().split())
        out.append("yes")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现反映了这样一个事实：除了读取输入之外不需要任何计算。 每个测试用例都是独立的，因此我们只需为每一对添加“是”。 

唯一的微妙之处是确保快速 I/O，因为高达$10^3$测试用例，尽管简单的打印就足够了。 不需要算术边缘情况，因为数学结构保证了所有配置中的可达性。

 ## 边缘情况

 对于$x = 0$，青蛙一动也不动。 开始于$0$，序列是常数$0 \to 0 \to 0$。 该算法正确地输出“是”，因为每个测试用例都得到肯定的回答，符合起点已经可达的要求。 

对于大型$x \ge n$，有效运动为$x \bmod n$。 例如，$n = 6, x = 14$表现得像$x = 2$，产生相同的循环结构。 由于该解决方案没有显式模拟步骤，因此它避免了由于忘记模块化简化而导致的任何不一致。 

对于任何$n$和$x$，序列被限制在有限的状态空间内并且必须重复。 由于它开始于$0$，循环包含$0$始终立即输入，并且输出在所有情况下都保持“yes”。
