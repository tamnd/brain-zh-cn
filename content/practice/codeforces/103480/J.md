 ## 解决方案
 ## 问题理解

 我们的手牌很小，最多有十张牌，还有一个具有固定战斗统计数据的头目怪物。 我们的目标是确定是否可以将这个 Boss 从游戏中移除，并防止其再次出现。 

老板与我们的卡牌互动分两个阶段。 首先，我们可以尝试使用手中的怪物卡来摧毁它。 怪物卡能否摧毁它取决于Boss的战位。 如果处于攻击位置，我们需要一个攻击值至少为2500的怪物。如果处于防御位置，我们需要一个攻击严格大于2100的怪物，因为它必须超过防御值2100。 

如果Boss在战斗中被消灭，它就会进入墓地。 从那里，可以使用一张特殊的咒语卡将其驱逐，从而防止未来的复活。 或者，有一张类似陷阱的卡可以直接将 Boss 驱逐出场，但使用它需要从我们的手牌中丢弃另一张卡。 

任务是确定是否存在使用给定手牌的任何游戏序列，以确保 Boss 最终被驱逐。 

约束非常小，最多 10 张卡，这立即排除了对大型结构的任何繁重的组合搜索。 即使对所有子集进行指数推理也是可行的，因为最坏的情况只有 1024 个状态。 这强烈表明问题在于检查一些关键的结构条件，而不是模拟复杂的游戏交互。 

一个微妙的一点是，动作的顺序仅以有限的方式起作用。 直接驱逐牌需要弃牌，但弃牌可以是任何其他牌，因此唯一的要求是总共至少有两张牌。 另一个微妙之处是，只有当我们还有一张从坟墓场驱逐的卡时，摧毁老板才有用，否则它仍然可以在以后复活。 

不存在涉及多个回合或部分效果的隐藏边缘情况，因为所有事情都发生在一个回合内，并且所有交互都是立即的。 

## 方法

 暴力方法会尝试所有可能的卡牌子集以及所有可能的使用它们的顺序。 对于每一个命令，我们都会模拟是否可以摧毁boss然后驱逐它或者直接驱逐它。 由于最多有 10 张卡，因此子集的数量为 2^10 = 1024，并且每个子集内的排列可以进一步增加复杂性。 尽管这在技术上是可行的，但它不必要地复杂并且隐藏了问题的结构。 

关键的观察是我们实际上并不关心排序或详细的排序。 游戏简化为检查是否可以满足两个独立获胜条件中的至少一个。 

第一个获胜条件是使用需要丢弃另一张牌的特殊牌直接驱逐。 当且仅当我们至少有一张这样的牌并且手中至少有一张额外的牌可以丢弃时，这是可能的。 其他限制都不重要。 

第二个获胜条件是一个两步过程。 我们首先必须有至少一个能够在当前位置规则下消灭Boss的怪物。 那么我们还必须至少拥有一张墓地驱逐卡。 如果两者都存在，我们可以先摧毁，然后将其从墓地中驱逐出去。 

这两个条件就足够了并且涵盖了所有有效的策略。 不存在一种策略以有意义的方式阻止另一种策略的交互，因为所有卡牌都可以在一个回合内以任何顺序使用。

 ## 算法演练

1.扫描所有卡牌，记录在给定的阵位规则下，我们是否有至少一个可以消灭boss的怪物。 这一步很重要，因为如果没有有效的攻击者，墓地路线是不可能的。 
2. 记录我们是否拥有至少一张类型1的牌，该牌允许将一张牌从坟墓场中驱逐。 这是“摧毁然后驱逐”策略所必需的。 
3.记录我们是否至少有一张类型2的牌，可以直接将boss逐出场上，但前提是我们可以丢弃另一张牌。 
4. 首先检查直接排除条件。 如果存在一张类型2的牌并且牌的总数至少为2，那么我们总是可以丢弃任何其他牌并立即驱逐老板。 如果这个条件成立，答案立即是肯定的。 
5. 如果无法直接驱逐，请检查两步路线。 如果存在至少一张有效怪物卡和至少一张类型1卡，那么我们可以消灭该Boss，然后将其从坟墓场中放逐。 如果两者都存在，则答案是肯定的。 
6. 如果两个条件都不成立，则没有有效的游戏顺序可以阻止 Boss 复活，因此答案是否定的。 

这一推理背后的关键不变量是，每一个成功的策略都必须以 Boss 被放逐而告终，而实现这一目标的唯一两种方法是直接从战场上放逐或在破坏后放逐。 规则中没有其他转换可以将 Boss 更改为永久移除状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

has_monster = False
has_banish_grave = False
has_black_core = False

for _ in range(n):
    parts = input().split()
    t = int(parts[0])
    
    if t == 0:
        atk = int(parts[1])
        if m == 0:
            if atk >= 2500:
                has_monster = True
        else:
            if atk > 2100:
                has_monster = True
    elif t == 1:
        has_banish_grave = True
    else:
        has_black_core = True

# direct banish via black core
if has_black_core and n >= 2:
    print("haoye")
    sys.exit()

# destroy then banish
if has_monster and has_banish_grave:
    print("haoye")
else:
    print("QAQ")
```该代码将手牌分为三个逻辑标志。 怪物的有效性取决于老板的位置，因此我们在扫描输入时相应地检查攻击阈值。 直接驱逐检定仅使用类型2卡的存在加上丢弃任何其他卡的能力，这相当于总共至少有两张卡。 

如果该路线失败，我们将测试唯一剩余的可行策略，即具有消灭能力的怪物和坟墓场驱逐卡的组合。 如果两者都存在，我们就成功了。 

检测到直接驱逐后的提前退出并不是严格必要的，但反映了该策略是独立且严格充分的事实。

## 测试用例```python
import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n, m = map(int, input().split())

    has_monster = False
    has_banish_grave = False
    has_black_core = False

    for _ in range(n):
        parts = input().split()
        t = int(parts[0])

        if t == 0:
            atk = int(parts[1])
            if m == 0 and atk >= 2500:
                has_monster = True
            if m == 1 and atk > 2100:
                has_monster = True
        elif t == 1:
            has_banish_grave = True
        else:
            has_black_core = True

    if has_black_core and n >= 2:
        return "haoye"
    if has_monster and has_banish_grave:
        return "haoye"
    return "QAQ"

# provided sample
assert solve("2 0\n0 2500\n1 0\n") == "haoye"

# minimal fail: single black core only
assert solve("1 0\n2\n") == "QAQ"

# direct banish works
assert solve("2 1\n2\n0 100\n") == "haoye"

# destroy + grave banish
assert solve("3 0\n0 2500\n1 0\n0 10\n") == "haoye"

# no valid monster
assert solve("2 0\n0 1000\n1 0\n") == "QAQ"

 ## 边缘情况

 一个重要的边缘情况是手牌仅包含一张 2 类牌。 在这种情况下，这张牌本身是没有用的，因为它需要丢弃另一张牌。 该算法通过在允许直接消除之前显式检查卡牌总数至少为两张来处理此问题。 

另一种边缘情况是存在怪物但不存在坟墓场驱逐卡。 即使我们可以摧毁 Boss，它仍然会复活，并且算法通过同时要求这两个条件来正确地拒绝这种情况。 

第三种边缘情况是存在多个单独攻击值不足的弱怪物时。 由于我们只需要至少一个有效的怪物，因此该算法正确地忽略了所有无效的怪物，并且仅跟踪合格卡牌的存在而不是聚合值。
