 ## 解决方案
 ## 问题理解

 给定一个整数 n，并要求我们准确打印固定消息 n 次，每次出现在其自己的行上。 消息本身总是相同的，并且不依赖于除 n 之外的任何输入。 

从计算的角度来看，输入仅决定我们重复常量字符串的次数。 没有转换，没有条件逻辑，也没有除了重复计数之外的状态累积。 这将问题牢牢地归入线性输出生成类别，其中输出大小为 θ(n)，算法必须至少花费 θ(n) 时间来简单地写入结果。 

约束 1 ≤ n ≤ 100 非常小。 即使是最简单的重复打印方法也很容易就足够了，任何对性能或内存的担忧在这里都是无关紧要的。 

在结构方面没有有意义的边缘情况，但在类似的任务中仍然出现一些小的正确性陷阱。 如果忘记在输出之间包含换行符，所有消息可能会连接成一行。 另一个错误是在每行末尾打印额外的尾随空白行或额外的空格，这会导致严格的输出比较失败，即使可见输出看起来相似。 例如，如果 n = 3，则正确的输出是三行，而有问题的解决方案可能会生成一行，例如“Welcome to HZNUWelcome to HZNUWelcome to HZNU”。 

## 方法

 解决这个问题最直接的方法就是从字面上模拟需求。 我们读取 n，然后在运行 n 次的循环中打印所需的字符串。 每次迭代都会输出相同的常数线。 

在这里，强力解释和最佳实现实际上是相同的。 蛮力视图只是“对于从 1 到 n 的每个计数，打印字符串”。 这是可行的，因为每个输出线都是独立的，不需要先前迭代的内存或派生值的计算。 

这种模式在其他问题中变得有趣的唯一原因是当 n 很大时，例如 10^7 或 10^9，其中原始打印在运行时占主导地位并且需要仔细的 I/O 优化。 这里，由于 n 最多为 100，因此即使 Python 的标准打印也足够快。 

没有可供利用的替代算法结构。 没有数学公式可以减少打印行数，因为输出本身明确需要 n 行。

 | --- | --- | --- | --- |
 | n 次打印的暴力迭代 | O(n) | O(1) | O(1) | 已接受 |
 | 最优（同上）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们现在描述用于构建输出的确切步骤。 

1. 从标准输入读取整数n。 该值决定必须生成输出行的次数。 
2. 精确地重复循环 n 次。 每次迭代对应一个所需的输出行。 
3. 在每次迭代中，打印字符串“Welcome to HZNU”，后跟换行符。 换行符至关重要，因为每次重复都必须出现在单独的行上。 

该算法不维护任何附加状态。 循环计数器本身足以控制重复。 

### 为什么它有效

 正确性来自循环迭代和所需输出线之间的直接一一对应关系。 每次迭代都会生成固定字符串的一份副本，并且迭代之间没有交互。 由于输出规范要求完全相同的 n 行，因此在 n 次迭代中每次迭代打印一次字符串的任何序列都必须与规范完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    out = "Welcome to HZNU"
    for _ in range(n):
        print(out)

if __name__ == "__main__":
    main()
```该解决方案使用快速输入读取整数，并将输出字符串存储在变量中，以避免在循环内重复重建它。 对于如此小的限制来说，这并不是绝对必要的，但反映了标准的竞争性编程卫生。 

该循环使用简单的范围迭代，确保恰好有 n 个输出。 每次调用 print 都会自动附加一个换行符，该换行符与所需的格式匹配。 

类似问题中常见的一个微妙错误是手动连接字符串并打印一次，这可能会导致丢失换行符或当 n 很大时超出内存。 在这里，我们通过逐行流式输出来避免这种情况。 

## 工作示例

 考虑输入 n = 3。

 每次迭代独立地产生相同的输出线。 第三次迭代后，程序终止。 

这证实了该算法不依赖于先前的状态，并且每次迭代始终发出所需的常量字符串。 

现在考虑 n = 1。 

| 迭代 | 打印字符串|
 | --- | --- |
 | 1 | 欢迎来到湖南师范大学 |

 通过单次迭代，输出仅由一行组成，满足要求，没有任何额外的间距或丢失换行符问题。

 ## 边缘情况

 主要的边缘情况是下界 n = 1。在这种情况下，循环只执行一次，并且输出仍必须包含换行符。 该实现自然地处理了这个问题，因为循环不会处理特殊情况的小值。 

例如，输入：```
1
```执行执行一次迭代并打印：```
Welcome to HZNU
```没有多余的行，也没有丢失的换行符。 由于循环边界是包含的并且直接由 n 控制，因此不存在偏差一的风险。 

类似问题中的另一个微妙情况是不带分隔符的输出的意外串联。 在这里，每个 print 调用独立地发出一个换行符，因此即使 n = 100，该结构作为不同行的序列仍然保持正确。
