 ## 解决方案
 ## 问题理解

 我们得到了一组歌曲，其中每首歌曲都有独特的流行度值和独特的名称。 流行度值就像一个严格的排名键：没有两首歌曲具有相同的分数，较高的值意味着歌曲更受欢迎。 

任务是想象所有歌曲按受欢迎程度降序排列。 一些最受欢迎的歌曲已经被其他用户获取。 我们被准确地告知有多少最佳选择（例如其中 k 个）已经消失。 我们的工作是确定下一首最佳可用歌曲，即按排序顺序出现在位置 k + 1 的歌曲。 

输入大小最多可达 100,000 首歌曲。 这立即排除了任何以线性方式重复扫描列表以进行每次删除或在循环内使用重复排序的方法。 O(n log n) 的解决方案是可以接受的，但任何二次方在 1 秒限制下都会失败。 

当 k 为零时，会出现微妙的边缘情况。 在本例中，没有提取任何歌曲，因此答案就是全球最受欢迎的歌曲。 另一个极端情况是当 k 为 n − 1 时，只有一首歌曲保持有效，并且无论位置如何都必须返回它。 

一个天真的错误是按升序排序并意外地选择第 k 个最小的而不是第 k 个最大的。 例如，如果我们错误地解释了“最想要的”，我们就会颠倒排名并始终返回错误的歌曲，即使排序是正确的。 

## 方法

 最直接的想法是按照流行度值降序对所有歌曲进行显式排序。 排序后，我们直接索引第 k 个位置。 

这是有效的，因为排序为我们提供了根据定义偏好的完全相同的比较规则对所有项目进行全局排序。 排序后，删除前 k 项在概念上相当于忽略它们并获取下一项。 

一种强力替代方法是重复扫描列表以找到当前最大值，将其删除，然后重复此 k + 1 次。 每次扫描的成本为 O(n)，执行 k 次会导致 O(nk)，在最坏的情况下变为 O(n²)。 当 n 达到 10⁵ 时，这远远超出了可行的极限。 

关键的观察是我们不需要动态删除。 我们只需要一次最终排名。 这将问题转化为静态排序问题，这正是排序可以有效解决的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 重复最大萃取 | O(n²) | O(1) | O(1) | 太慢了 |
 | 按值排序 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 将所有歌曲作为（流行度、名称）对读取到列表中。 这保留了我们必须输出的排名键和标识符。 
2. 按受欢迎程度降序对列表进行排序。 这创建了一个全球排名，其中第一个元素是最想要的歌曲，而接下来的每个元素都严格低于前一个元素。 流行度的唯一性保证了没有联系，因此顺序是确定的。 
3. 读取整数k，它代表已经获取了多少首排名靠前的歌曲。 
4. 直接访问排序列表中索引k处的元素。 这对应于基于 1 的排名中第 (k + 1) 位最受欢迎的歌曲。 
5. 输出与该元素关联的名称。 

### 为什么它有效

 排序后，列表是歌曲按照热度由高到低的总顺序。 由于所有受欢迎度值都是不同的，因此这种排序是严格且稳定的。 删除前 k 个元素不会改变剩余元素的相对顺序，因此第一个剩余元素恰好是从零开始索引中的第 k 个索引。 该算法从不依赖于中间更新或删除，因此不会发生状态损坏或重新排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    songs = []
    
    for _ in range(n):
        w, s = input().split()
        w = int(w)
        songs.append((w, s))
    
    k = int(input())
    
    songs.sort(reverse=True)
    
    print(songs[k][1])

if __name__ == "__main__":
    solve()
```实现的核心是排序调用，它按第一个元素按降序对元组进行排序，因为`reverse=True`。 由于Python按字典顺序比较元组，因此名称仅用作辅助键，但这并不重要，因为权重保证是唯一的。 

直接索引`songs[k]`是安全的，因为 k 保证满足 0 ≤ k < n。 除了信任输入约束之外，不需要边界检查。

 ## 边缘情况

 当k为零时，算法直接返回排序后的第一个元素。 例如，输入：```
3
5 A
2 B
9 C
0
```排序后得到(9,C),(5,A),(2,B)。 索引 0 是 C，这是正确的，因为没有歌曲被视为已被占用。 

当 k 等于 n − 1 时，仅保留最小元素。 对于输入：```
3
5 A
2 B
9 C
2
```排序列表为 (9, C)、(5, A)、(2, B)。 索引2是(2,B)，这是除去最上面的两首歌曲后剩下的最后一个选择。 

由于算法在排序后不会修改列表，因此这些边缘情况不需要特殊处理。 相同的索引规则始终适用于 k 的所有有效值。
