 ## 解决方案
 ## 问题理解

 我们得到的初步阵容是$n$孩子们，每个孩子$i$有固定的幸福值$h_i$。 这些玩具最初按排列顺序进行洗牌：儿童$i$收到一个贴有标签的玩具$w_i$，所以数组$w$是一个排列$1 \dots n$。 

然后这个过程会循环发展。 在每一轮中，每个孩子拿起他们当前拿着的玩具，并将其交给按该玩具上的标签索引的孩子。 具体来说，如果孩子$i$拿着一个玩具（假设它通过前几轮到达那里），他们将其传递给孩子$w_i$。 每轮结束后，每个孩子都会检查他们当前持有的玩具的标签。 如果在任何时候它等于他们的初始标签$w_i$，他们永久停止参与。 

对于每个孩子，我们计算他们在停止之前存活了多少轮。 如果这个数字是$c_i$，固定排列的总分是$\sum h_i \cdot c_i$。 任务是将所有大小排列的分数相加$n$, 取模$998244353$。 

约束条件$n \le 2000$排除任何迭代排列或模拟每个排列动态的方法。 由于有$n!$排列，甚至$n=10$已经变大了。 任何有效的解决方案都必须将贡献压缩为封闭形式的组合表达式。 

当考虑立即停止时，会出现一种微妙的边缘情况。 由于孩子一开始拿着分配给他们的玩具，人们可能会错误地认为$c_i = 0$。 然而，仅在完成一轮后才检查停止，因此每个孩子至少参与一轮，除非结构强制在一步后立即返回。 

## 方法

 蛮力解释很简单。 对于每个排列，逐轮模拟该过程，跟踪哪个孩子拿着哪个玩具，直到每个孩子都稳定下来。 每次模拟费用$O(n \cdot \text{cycle length})$，并且有$n!$排列，使得这完全不可行。 

关键的观察是该过程完全由排列图控制$w$。 每个玩具都沿着由下式定义的功能图确定性地移动$i \to w_i$。 自从$w$是一个排列，图分解为不相交的循环，并且每个玩具在其组件内循环。 

一个孩子$i$玩具开始时正好停止$i$返回到$i$。 即包含节点的环的长度$i$排列中$w$。 所以$c_i$就是周期长度$i$在$w$。 

这将分数变成了干净的组合形式。 对于固定排列，$$H(w) = \sum_{i=1}^n h_i \cdot \text{cycleLen}_w(i)$$线性允许我们按索引分离贡献：$$\sum_{w} H(w) = \sum_{i=1}^n h_i \cdot \sum_{w} \text{cycleLen}_w(i)$$根据对称性，每个的内部和都是相同的$i$，因此问题简化为计算固定元素对所有排列的总循环长度贡献。 

最终的结构见解是随机排列中固定元素的循环长度分布在所有大小上是均匀的$1 \dots n$，它将整个问题折叠成一个封闭的形式。

 ## 算法演练

 我们现在计算所需的总和而不枚举排列。 

1. 观察总答案与值呈线性关系$h_i$，因此我们独立分离每个指标的贡献。 这是有效的，因为排列动态不会耦合不同的$h_i$价值观。 
2.固定位置$i$。 研究孩子走了多少步$i$在排列中生存$w$。 这正是包含节点的循环的长度$i$。 
3. 将任务重新表述为计算固定元素（例如元素）的循环长度在所有排列上的总和$1$。 
4. 计算元素出现的频率$1$属于一个长度循环$k$。 我们选择另一个$k-1$循环中的元素$\binom{n-1}{k-1}$方式，将它们排列成一个循环$(k-1)!$方式，并排列剩余的$n-k$任意元素$(n-k)!$方式。 相乘得出$(n-1)!$，独立于$k$。 
5. 由于每$k$从$1$到$n$发生的次数相同，固定元素的预期周期长度是以下的平均值$1 \dots n$，即$(n+1)/2$。 
6. 将期望乘以排列数$n!$获得所有排列的循环长度总和：$$S = n! \cdot \frac{n+1}{2}$$1、最终答案是：$$\left(\sum h_i\right) \cdot S$$### 为什么它有效

 每个排列对总和都有独立的贡献，并且每个索引在排列对称性下表现相同。 关键的不变量是固定元素的循环长度分布在所有可能的循环大小上是均匀的，这使得它的总贡献可以干净地分解。 一旦认识到这种对称性，指数之间就不再存在相互作用，并且总和会完全分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

n = int(input())
h = list(map(int, input().split()))

s = sum(h) % MOD

fact = [1] * (n + 1)
for i in range(1, n + 1):
    fact[i] = fact[i - 1] * i % MOD

ans = s * fact[n] % MOD
ans = ans * (n + 1) % MOD
ans = ans * modinv(2) % MOD

print(ans)
```该实现依赖于预先计算阶乘$n$，因为该公式取决于$n!$。 除以 2 是使用模逆来处理的$998244353$。 

代码中的关键点是我们从不模拟排列或循环。 一切都归结为计算从组合对称性导出的单个标量表达式。

 ## 边缘情况

 一个关键的边缘情况是当$n = 1$。 置换集仅包含一个元素，循环长度一般为 1。公式给出$1! \cdot (1+1)/2 = 1$，匹配直接解释。 

另一个微妙的情况是当所有$h_i$是平等的。 在这种情况下，问题纯粹简化为对所有排列的循环长度进行平均。 均匀性参数确保索引之间不存在隐藏偏差，因此答案成为总循环质量的干净标量倍数，公式准确地捕获了该值。
