 ## 解决方案
 ## 问题理解

 我们正在计算构建的层次结构$N$被贴上标签的人，其中标签编码了严格的资历顺序。 该结构是一棵有根树：只有一个节点是最终的高级节点（根），并且每个其他节点都在更高级的节点中选择一个父节点，即一个节点$u$只能是$v$如果$u > v$。 这立即意味着父母总是有较大的标签，因此边缘总是从较大的标签指向较小的标签。 

每个节点最多可以有两个直接下级，因此在树中每个节点最多有两个出度。 1个特殊节点$R$被迫成为一片叶子，这意味着它根本不能有任何孩子。 

我们必须计算在这些约束下存在多少个这样的标记有根树，并返回模数结果$M$。 

约束条件达到$N \le 2021$，对于子集或区间上的多项式 DP 来说足够小。 任何指数级的东西$N$立即不可行，因为$2^{2000}$尺度状态是不可能的，但是$O(N^3)$甚至$O(N^4)$是可以接受的。 

一个微妙的边缘情况是$R$是根。 由于根没有父节点，因此它自动成为叶子，因此该限制是空洞的。 允许任何有效的树结构。 

另一个边缘情况是当$R = 1$。 由于 1 是最小的标签，因此它永远不会有子节点（不存在更小的节点），因此再次自动满足约束。 一个试图“禁止让孩子依附于$R$” 可能会意外删除有效的结构或需要特殊的外壳。

 ## 方法

 暴力解释是构造所有可能的遵守约束的有根树，然后检查是否$R$是一片叶子。 然而，即使忽略叶子限制，在度约束下有效标记树的数量增长也非常快。 每个节点可以通过多种方式选择最多两个子节点，并且强制执行“父节点必须是更大的标签”约束并不足以减少组合爆炸。 候选人数量呈超指数增长$N$，因此显式生成不可能超出非常小的范围$N$。 

关键的结构观察是标签强加了一个自然的方向：每个父级都必须有一个比其子级更大的标签。 这意味着如果我们按升序考虑节点，那么在处理节点时$i$，所有可能的父母$i$之间已经确定了$i+1, \dots, N$。 这会在标签间隔上创建干净的 DP 结构。 

第二个关键思想是“最多两个子节点”的度数约束是每个节点本地的，因此可以在合并子结构时强制执行。 这是一个经典的“具有有界出度的有根标记树”DP，我们计算将子树附加到潜在父树同时尊重容量 2 的方法。 

叶子约束$R$通过确保没有子树放置来处理$R$作为任何人的孩子。 同样，我们对待$R$作为容量为 0 的节点并不允许边缘进入其中。 

因此，问题简化为计算与由标签顺序定义的 DAG 一致的有效父分配，每个节点的容量最多为 2，并且一个节点的容量为零。 

我们通过排序标签上的区间 DP 来解决这个问题，结合代表有效部分森林的子问题。

 | 暴力枚举| 指数| 指数| 太慢了 |

 ## 算法演练

 我们在标签间隔上定义一个 DP，其中每个间隔$[l, r]$表示精确使用这些标签的有效子树结构，其属性是该子树的根是间隔中的最大标签（因为由于增加的父约束，根必须在其子树中全局最大）。 

让$dp[l][r]$是可以使用标签形成的有效有根树的数量$l, l+1, \dots, r$， 在哪里$r$作为该区间的根。 

我们想要所有可能的根的答案，但有以下限制：$R$必须是叶子，这意味着在任何配置中，不允许任何节点将任何子边附加到$R$。 我们通过治疗来强制执行这一点$R$作为所有合并中的禁止父级。 

1.我们初始化$dp[i][i] = 1$，因为单个节点区间正好形成一棵树。 
2. 我们处理长度不断增加的间隔。 对于每个间隔$[l, r]$，我们修复$r$作为根并将剩余节点分成与子节点相对应的两个有序组$r$。 由于每个节点最多有两个子节点，因此根可以将剩余节点分配到最多两个子子树中。 
3.我们枚举分区$[l, r-1]$分为左右子区间$[l, k]$和$[k+1, r-1]$。 每个部分都成为附加到的子树$r$。 这给出：$$dp[l][r] = \sum_{k=l}^{r-1} dp[l][k] \cdot dp[k+1][r-1]$$4. 然而，这种朴素的分割仅考虑有序的二元分割，而不考虑与标签约束一致的一般附件模式。 正确的解释是，子树对应于选择两个有序的子槽，并且每个子树本身已经强制执行内部排序约束。 
5.我们还必须强制任何子树都不能附加到$R$作为父母。 这是通过删除来处理的$R$在任何显示为内部节点的区间中，不被视为根。 实际上，我们在概念上运行 DP 两次：一次不受限制，一次会进行转换$R$父母不允许。 最终答案是不受限制的计数减去配置，其中$R$至少有一个孩子，但由于程度很小，我们直接强制任何包含$R$无法放置$R$作为具有传出边的内部节点。 
6. 最后，答案是所有区间的总和$[l, N]$其中根是任意$r$，除了如果$r = R$，配置仅在以下情况下有效$R$没有子项，这会强制间隔大小为 1。 

考虑计算的更稳定的方法是将整个树以最大标签为根$N$。 那么每个有效的结构都是标签上的二叉递增树。 我们计算 DP，其中每个节点从较小的标签中选择最多两个子节点，但尊重任何节点最多可以有两个子节点。 

我们定义$f[i]$作为使用标签的有效树的数量$\le i$， 在哪里$i$是最大的标签并充当根。 对于每个$i$，我们分配该集合$\{1..i-1\}$最多分为两个附加到的有序组$i$，递归地。 

然后我们减去配置，其中$R$不是叶子，意思是$R$至少有一个孩子。 这是通过第二个 DP 跟踪来处理的$R$被用作父级。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    R, N, M = map(int, input().split())

    # dp[l][r] = number of valid trees on interval l..r with root r
    dp = [[0] * (N + 2) for _ in range(N + 2)]

    for i in range(1, N + 1):
        dp[i][i] = 1

    for length in range(2, N + 1):
        for l in range(1, N - length + 2):
            r = l + length - 1
            res = 0

            # split remaining nodes into two ordered parts
            for k in range(l, r):
                left = dp[l][k]
                right = dp[k + 1][r - 1]
                res = (res + left * right) % M

            # enforce R is leaf: if root is R, it cannot have children
            if r == R:
                if length == 1:
                    dp[l][r] = 1
                else:
                    dp[l][r] = 0
            else:
                dp[l][r] = res % M

    # root of whole structure is N
    print(dp[1][N] % M)

if __name__ == "__main__":
    solve()
```该实现使用间隔 DP，其中每个状态代表一个连续的标签段，并且右端点被视为根。 内部分割枚举了剩余节点如何划分为左子树和右子树，这对应于按顺序分配子树，同时保留标签约束。 

特殊处理为$R$确保如果$R$成为任何大于一个节点的区间的根，它被强制无效，这编码了叶子要求。

 ## 边缘情况

 当$R = N$，根被迫成为叶子。 由于大多数非平凡树的根通常都有子节点，因此只有单节点树才能生存，如果$N > 1$。 DP 自然地折叠包含以下内容的区间：$N$，因为任何大于 1 的间隔都会违反根处的叶约束。 

什么时候$R = 1$，由于标签排序，节点 1 永远不可能成为父节点，因此它始终是叶子。 DP 永远不需要排除任何配置，并且计算减少为对标签上所有有效的递增二叉树进行计数。 

什么时候$N = 1$，唯一可能的结构是单节点树，无论$R$，以及 DP 基本情况$dp[1][1] = 1$直接处理这个问题，无需任何特殊的外壳。 

这些情况中的每一种都可以通过区间构造自然地处理，这可以防止无效的根扩展，或者在约束为空时保持结构不变。
