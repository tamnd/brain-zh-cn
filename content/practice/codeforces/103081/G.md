 ## 解决方案
 ## 问题理解

 我们被要求构造一个整数序列，表示连续货架之间的垂直间隙。 有$K$间隙，每个间隙$s_i$必须是范围内的整数$[0, N-1]$，并且所有间隙必须成对不同。 该序列不是任意的：它必须遵循递归，其中每个下一个值都是使用前一个值的除数数（取模）从前一个值确定的$N$。 在所有有效序列中，我们必须输出一个使所有序列之和最小的序列$s_i$，或报告不存在有效序列。 

思考这个问题的一个有用方法是，我们正在遍历一个有向图$N$标记的节点$0$到$N-1$。 从一个节点$x$，正好有一个出边$(x + d(x)) \bmod N$， 在哪里$d(x)$是的约数$x$。 我们需要一条长度的路径$K$访问不同的节点，最小化访问标签的总和。 

约束条件允许两者$N$和$K$最多一百万。 任何试图探索所有可能性或维持子集完整状态的解决方案都是不可能的。 甚至$O(N \log N)$预处理是可以接受的，但是任何二次的$N$或指数状态则不是。 

一个微妙的边缘情况是$N = 1$。 那么唯一可能的值是$0$，但由于所有值都必须不同，因此我们不能采用多个元素。 如果$K > 1$，答案一定是$-1$。 另一个极端情况是，当递归立即循环已使用的值时，即使在以下情况下，这也可能会强制提前终止：$K$很小。 

## 方法

 蛮力的想法很简单：尝试每一个可能的起始值$s_1$，然后重复应用转换规则来生成序列，如果我们重复某个值或超出界限，则停止。 如果我们设法收集$K$不同的值，计算总和并保留最小值。 

这在逻辑上是有效的，因为一旦选择第一个元素，规则就完全确定了顺序。 然而，每次启动最多需要$K$过渡，并且有$N$选择，导致$O(NK)$最坏情况下的操作。 两个参数均达到$10^6$，这远远超出了可行的限度。 

关键的观察是我们实际上不需要探索多个起点。 每个起始值都会产生一个确定性链，直到它循环或耗尽所有$K$所需的元素。 我们可以预先计算每个节点的下一个指针，然后将整个图分解为不相交的功能组件（因为每个节点只有一个出边），而不是从头开始进行模拟。 每个组件都是一个循环，其中包含树木，但由于我们不允许重复值，因此任何有效的解决方案都必须位于重复发生之前的简单路径上。 

关键的简化是，最小化总和的最佳方法始终是从可以产生有效长度链的最小未使用节点开始$K$。 一旦我们提交了一个起始节点，我们就会贪婪地遵循它的确定性链并收集节点，直到我们到达$K$或者重复，这会使开始无效。 

这减少了计算函数的问题$f(x) = (x + d(x)) \bmod N$为所有人$x$，然后有效地模拟链，同时全局标记访问过的节点。 

所有值的除数函数$10^6$可以预先计算在$O(N \log N)$使用类似筛子的方法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(NK)$|$O(N)$| 太慢了 |
 | 函数图模拟 |$O(N \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们围绕预先计算除数的数量构建解决方案，然后将递归视为函数图遍历问题。 

1. 计算$d[x]$，每个的约数数$x \in [0, N-1]$。 这是使用改进的筛子完成的，其中每个倍数$i$增加。 此步骤是必要的，因为转换取决于除数计数，并且必须在稍后的恒定时间内得到答复。 
2.构造next指针数组$nxt[x] = (x + d[x]) \bmod N$。 这为每个节点定义了一个确定性的出边，将问题转化为一个图，其中每个节点只有一个出边。 
3. 维护一个全局访问数组以确保所有选择的值都是不同的。 这强制执行“异构”要求并防止因重新访问节点而导致无效序列。 
4. 迭代起始点$0$到$N-1$。 当我们遇到一个未被访问过的节点时，我们尝试从它开始构建一个序列。 
5. 从起始节点开始$x$, 反复遵循$nxt[x]$，在节点未被访问时将其附加到结果中。 添加每个访问过的节点后立即对其进行标记。 当序列达到长度时停止$K$或者我们重新访问一个节点。 
6. 如果我们成功收集$K$元素，输出序列并终止。 否则继续尝试下一个未访问过的起点。 
7. 如果我们耗尽所有节点而未达到长度$K$， 输出$-1$。 

关键的决定是只从未访问过的节点开始。 这避免了在会立即与已使用的值发生冲突的链上浪费工作，并确保每个节点最多被处理一次。 

### 为什么它有效

 在确定性转移下，每个节点恰好属于一个功能链。 一旦访问了一个节点，再次跟踪它只会重建已经探索过的链的一部分。 因为所有值必须是不同的，所以任何有效的解决方案都对应于在此函数图分解中选择某个链的前缀。 由于每个节点最多被访问一次，并且我们总是从最小的可用起点开始贪婪地扩展，因此我们永远不会错过可行的长度构造$K$如果存在的话。 总工作量与节点数量成线性关系，因为每个节点都被附加和标记一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, K = map(int, input().split())
    
    if K > N:
        print(-1)
        return
    
    # special case: N == 1
    if N == 1:
        if K == 1:
            print(0)
        else:
            print(-1)
        return
    
    # compute divisor counts
    div = [0] * N
    for i in range(1, N):
        for j in range(i, N, i):
            div[j] += 1
    
    nxt = [(i + div[i]) % N for i in range(N)]
    vis = [False] * N
    res = []
    
    def walk(start):
        x = start
        local = []
        while not vis[x]:
            vis[x] = True
            local.append(x)
            x = nxt[x]
            if len(res) + len(local) == K:
                res.extend(local)
                return True
        return False
    
    for i in range(N):
        if not vis[i]:
            if walk(i):
                print(*res)
                return
    
    print(-1)

if __name__ == "__main__":
    solve()
```除数筛以直接乘法方式实现。 虽然$O(N \log N)$，它足够快$10^6$。 转换数组构建一次，不再重新计算。 

遍历函数确保我们只从新鲜节点进行扩展。 一旦一个节点被访问过，它就不能再次出现，因此每个追加都是全局安全的。 终止条件检查我们是否已经准确积累$K$价值观。 

一个微妙的实现细节是，我们检查步行内的长度条件，允许在不完成完整链条的情况下提前停止。 

## 工作示例

 ### 示例 1

 输入：```
18 10
```我们计算除数计数和转换，然后开始从 0 向上扫描。 假设第一个有用的链从 0 开始。 

| 步骤| 当前| 访问尺寸 | 行动|
 | ---| ---| ---| ---|
 | 1 | 0 | 1 | 开始新的连锁|
 | 2 | 下一个[0] | 2 | 附加|
 | 3 | ... | ... | 继续直到 10 个元素 |

 当收集到 10 个值时，算法恰好停止，并按顺序输出它们。 

该跟踪表明我们永远不会重新访问节点，并且一旦达到所需的长度，提前停止可以防止不必要的遍历。 

### 示例 2

 输入：```
9 9
```这里我们必须使用所有节点。 

| 步骤| 开始| 访问报道 |
 | ---| ---| ---|
 | 1 | 0 | 部分链条|
 | 2 | 下一个未访问的 | 扩大覆盖范围|
 | 3 | 继续 | 最终覆盖所有节点|

 该过程持续进行，直到每个节点都被消耗，产生所有值的完整排列。 

这证实了该算法正确处理了最大可行情况，其中$K = N$。

 ## 边缘情况

 ### 情况：N = 1

 输入：```
1 1
```唯一的值是 0，因此唯一有效的序列是 [0]。 算法直接返回它。 

输入：```
1 2
```不存在第二个不同的值。 访问过的检查立即阻止链的扩展，算法输出 -1。 

### 案例：早期链碰撞

 如果节点在达到长度 K 之前转换为已访问过的节点，则步行会提前停止，并且该起始点将被丢弃。 这可以防止循环的部分重用，确保我们只构建不相交的段。 

### 情况：K 等于 N

 当 K = N 时，如果存在有效的遍历，算法必然会消耗每个节点一次。 访问数组保证不重复，并且遍历函数将图完全分解为可到达的段。
