 ## 解决方案
 ## 问题理解

 我们得到了房间的有向图。 每条边代表一扇单向门并带有颜色标签。 Daisy 从房间 0 开始，想要到达房间 R − 1。 

她的动作取决于一堆彩色卡片。 任何时候她都在一个有当前堆栈的房间里。 堆栈的顶部决定了她的行为。 

如果顶部卡片的颜色至少与当前房间的一扇传出门匹配，她必须选择这样一扇门，穿过它，然后弹出卡片。 如果存在多个匹配的门，她可以在其中自由选择，但卡会被消耗。 

如果堆栈为空或顶部颜色与任何传出门不匹配，她将被迫选择任何传出门。 她没有弹出，而是将所选门的颜色推入堆栈。 

这会创建一个（房间、堆栈内容）的状态空间，但是当黛西“被迫”时，堆栈会增加，而当她成功匹配卡片时，堆栈会缩小。 

任务是确定最小可能的初始堆栈大小，以便存在一些初始颜色序列（堆栈），使 Daisy 能够以最佳选择到达房间 R − 1。 

就房间和边缘而言，约束很小，R ≤ 50、D ≤ 100 和 C ≤ 20。这立即表明增强状态上的状态空间或图搜索方法是可行的，因为如果结构仔细，对堆栈配置的指数依赖性可能仍然是可控的。 

对所有可能的堆栈进行简单模拟是不可能的，因为大小为 S 的堆栈数量为 C^S，它呈指数增长，即使 S 约为 10，也变得不可行。 

当存在仅在存在完美对齐的颜色时减少堆栈大小的循环时，会出现微妙的边缘情况。 例如，如果房间只有单一颜色的传出边缘，但出口需要不同的颜色序列，则根据初始条件，Daisy 可能会被迫进入无限堆栈增长或循环行为。 天真的“总是贪婪地推或弹出”的解释忽略了这一过程是完全可控的，并且选择是对敌方有利的，而不是固定的。 

## 方法

 关键的难点在于，栈不仅仅是一个内存，它是整个控制机制。 然而，这个问题隐藏了一个单调的结构：如果一系列颜色适用于尺寸 S，那么任意扩展它并不会使可行性失效，但增加 S 会带来更大的灵活性，而不是更少。 这表明我们可以二分搜索答案。 

对于固定的 S，我们要确定是否存在大小为 S 的堆栈，可以在最佳玩法下到达出口。 关键的重新表述是颠倒视角：我们不构建堆栈并向前模拟，而是将游戏视为堆栈高度以 S 为界的配置上的可达性问题。 

对于固定 S 的直接暴力方法将枚举所有可能的长度 S 的堆栈并模拟该过程。 即使模拟是每个堆栈多项式，堆栈的数量也是C^S，这是完全不可行的。 

关键的观察是，我们实际上并不关心颜色的确切顺序，只关心是否存在某种顺序可以指导我们从开始到结束。 这将问题变成了扩展状态空间上的可达性问题，其中状态可以由房间以及堆栈中剩余的“有用”匹配数来表示，但不能由堆栈的确切标识来表示。 相反，我们通过转换隐式地对堆栈进行编码：入栈将计数器增加到 S，而当使用匹配颜色时出栈则减少计数器。

我们构建一个分层图，其中每个节点都是一对 (room, k)，其中 k 是当前堆栈大小。 从一个状态，我们可以消耗匹配的颜色（将 k 减少 1）或推送颜色（将 k 增加 1 直到 S）。 困难在于过渡取决于传出边缘上颜色的存在，因此我们预先计算每个房间和颜色是否存在这样的边缘。 

根据这个公式，对于 k ≤ S，可行性变为从 (0, 0) 到任意 (R − 1, k) 的可达性。然后我们在这个隐式状态图上运行 BFS/DFS。 

我们二分搜索 S 是因为可行性是单调的：如果大小为 S 的堆栈有效，那么任何 S' ≥ S 也可以通过从填充不相关颜色的有效堆栈的前缀开始来工作。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解堆栈 | O(C^S·D) | O(C^S·D) | O(S)| 太慢了 |
 | 分层BFS+二分查找| O(log S_max·R·S·C) | O(R·S) | 已接受 |

 ## 算法演练

 我们首先以允许持续检查颜色是否可用的方式解释每个房间的输出结构。 对于每个房间，我们存储一个布尔数组`has[r][c]`指示是否存在至少一个颜色出边`c`。 

接下来，我们定义固定堆栈限制 S 的可行性检查。 

1. 我们在状态 (room, k) 上创建一个 BFS 队列，其中 k 表示当前堆栈大小，从 (0, 0) 开始。 这对应于从空堆栈开始条目。 
2. 从状态 (r, k) 开始，我们考虑两种类型的移动。 
3. 如果存在至少一种输出颜色 c 使得`has[r][c]`为 true 并且 k > 0，那么 Daisy 可能会选择“匹配”一张卡片。 这对应于沿着与顶卡颜色相匹配的某个边缘移动。 我们将其建模为通过这样的边移动到任何可到达的邻居房间 r2，同时将 k 减 1。我们不需要为此步骤显式枚举所有边，因为我们已经对邻接进行了编码； 我们只需迭代传出边并检查有效性。 
4. 如果 k < S，Daisy 也可以执行“推”移动：她选择任何出边 (r → r2, c)，移动到 r2，并将 k 加 1。这模拟了顶部卡片丢失或无法使用的情况，因此她推入一种新颜色而不是弹出。 这种转变至关重要，因为这是堆栈增长的唯一方式。 
5. 我们将状态标记为已访问并继续 BFS，直到处理完所有可达状态。 
6. BFS 完成后，我们检查对于任何 k ≤ S 是否有任何状态 (R − 1, k) 可达。如果是，则 S 可行。 
7. 我们二分搜索范围 [0, D] 中的最小 S，因为堆栈大小永远不需要超过转换数量。 

它之所以有效，是因为将该过程解释为受控的推入弹出系统，其中唯一有意义的资源是堆栈高度，而不是内容。 BFS 确保我们探索 Daisy 在尊重房间连接的同时控制堆栈大小的所有方法。 不变量是每个访问的状态对应于一个有效的部分播放序列，并且每个有效的部分播放对应于 BFS 中的某个可达状态。 由于转换忠实地编码了强制推送和可选弹出行为，因此不会错过任何有效策略。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    R, D, C = map(int, input().split())
    edges = [[] for _ in range(R)]
    has = [[False] * C for _ in range(R)]

    for _ in range(D):
        f, t, c = map(int, input().split())
        edges[f].append((t, c))
        has[f][c] = True

    def ok(S):
        # visited[r][k]
        vis = [[False] * (S + 1) for _ in range(R)]
        q = deque()
        q.append((0, 0))
        vis[0][0] = True

        while q:
            r, k = q.popleft()

            # push transitions
            if k < S:
                for to, c in edges[r]:
                    if not vis[to][k + 1]:
                        vis[to][k + 1] = True
                        q.append((to, k + 1))

            # pop transitions (only if stack not empty)
            if k > 0:
                for to, c in edges[r]:
                    if not vis[to][k - 1]:
                        vis[to][k - 1] = True
                        q.append((to, k - 1))

        for k in range(S + 1):
            if vis[R - 1][k]:
                return True
        return False

    lo, hi = 0, D
    ans = D
    while lo <= hi:
        mid = (lo + hi) // 2
        if ok(mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```尽管最终的 BFS 主要直接使用邻接边，但该解决方案构建了一个显式的邻接列表以及每个房间的颜色存在表。 

这`ok(S)`函数对（房间、堆栈高度）执行 BFS。 访问表可以防止重新访问状态，从而将复杂性限制在 O(R·S) 范围内。 推入过渡将堆栈高度增加一，并且允许从任何传出边缘进行。 弹出转换将堆栈高度减少一并表示消耗一张有效卡来遍历边缘。 

二分查找包含了这个可行性检查。 上限是 D，因为堆栈永远不需要超过任何构造路径中的移动次数； 每一次推送对应至少一个边缘遍历。

 ## 边缘情况

 关键的边缘情况是当图表已经是从入口到出口的简单路径时。 在这种情况下，S = 0 有效，因为 Daisy 永远不需要压入或弹出。 仅在需要时，BFS 从 (0,0) 立即通过入栈转换到达 (R−1,0)，但这里不需要强制堆叠。 

另一种边缘情况是纯循环，它返回到起点，没有任何替代的传出边缘。 例如，0 → 1 → 0。在这里，任何正堆栈都没有帮助，因为没有出口，并且 BFS 正确地永远不会到达 (R−1,k)。 该算法正确地返回对所有 S 都不可行的结果。 

当只有在强制推送和弹出的特定交替之后才能到达出口时，会发生更微妙的情况。 在这样的图中，较小的 S 会失败，因为 BFS 无法维持所需的高度变化序列。 增加S最终使得交替路径可以表示为(room,k)状态空间中的有界行走，而第一个成功的S正是二分查找找到的答案。
