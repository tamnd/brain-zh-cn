 ## 解决方案
 ## 问题理解

 我们得到了无限的一维单元格线，每个单元格包含零或一。 最初，仅明确指定了 16 个单元的块； 该块之外的所有内容均为零。 时间以离散的步骤演化，并且在每一步中，每个细胞根据其自身的值以及其紧邻的左右邻居的值同时更新。 

更新规则完全由单元及其两个邻居形成的 3 位模式决定。 对于每个可能的三元组，都有一个固定的输出零或一，成为中间单元格的新值。 这就是经典的元胞自动机，称为规则 110。 

我们被要求应用这种变换 N 次，其中 N 可以非常大，最多 2^60，然后报告整个无限配置中 1 的总数。 

关键的困难在于系统不受最初 16 个单元的限制。 即使我们从一个小的配置开始，每一步都可以向外传播一个单元，因此在 t 步之后，潜在的非零区域可以扩展到 16 + 2t 数量级的宽度。 因此，在最坏的情况下，显式维护完整扩展数组的简单模拟将在 O(N^2) 时间内运行，这是完全不可行的。 

另一个微妙之处在于，配置是无限的，但以结构化方式稀疏。 如果我们尝试仅存储活动窗口，则必须注意将其扩展多远以及如何检测系统何时重复。 

一个简单但重要的边缘情况是 N 为零时。 在这种情况下，答案只是 16 个单元中 1 的初始数量，任何模拟方法都必须保留这一点，而不执行移动或破坏边界的虚拟步骤。 

另一种极端情况是模式很快消失。 例如，如果输入全为零，则系统永远保持全零，正确的解决方案必须避免不必要的模拟循环并直接返回零。 

## 方法

 暴力方法逐步模拟自动机。 在每一步中，我们都会通过将规则应用于当前活动区域中的每个单元来计算下一个配置，并在每一侧扩展一个额外的单元。 由于有源区域每侧每步最多可以增长一个单元，因此在 t 个步骤之后，我们可能需要维持 O(t) 个单元。 经过 N 个步骤，这会导致总复杂度为 O(N^2)，当 N 达到 2^60 时，这个复杂度就太大了。 

关键的观察是，虽然状态空间原则上是巨大的，但系统从有限的初始配置确定性地演化，并且局部受到高度约束。 如果我们将配置表示为稀疏集或修剪的二进制字符串，则演化仅取决于有界邻域。 这使得检测配置序列中的重复成为可能。 

一旦配置重复，系统就进入了一个循环。 从那时起，我们不需要一步步模拟进一步的步骤。 相反，我们可以计算剩余时间中有多少个完整周期，并直接跳到周期中的最终位置。 由于初始配置很小并且规则是确定性的，因此在实践中对于这种输入大小，重复之前遇到的不同配置的数量通常很小，从而使得循环检测高效。 

因此，我们在存储每个看到的配置及其位置的同时进行模拟。 当重复发生时，我们提取循环长度并使用模运算跳转到正确的最终状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(N·每步宽度) ≈ O(N^2) | O(N) | 太慢了|
 | 循环检测模拟| O(K·W)，其中 K 是唯一状态的数量 | O(K·W) | 已接受 |

 ## 算法演练

我们将配置视为二进制字符串，但不断修剪前导和尾随零，以便我们只存储有意义的活动区域，以及指示该段相对于固定原点的位置的偏移量。 

1. 根据输入字符串初始化当前配置，并将其初始偏移量计算为零。 这表示给定的 16 个单元格对应于固定的坐标区间，而外部的所有内容都隐式为零。 
2. 逐步模拟自动机，通过将规则 110 应用到从当前最小活动索引左侧的一个位置到当前最大活动索引右侧的一个位置的每个单元来生成下一个配置。 我们包括额外的边界，因为由于邻居的影响，新的边界可能会出现在当前范围之外。 
3. 计算下一个配置后，修剪前导零和尾随零。 如果配置变空，我们将其视为规范偏移处的单个零单元。 
4. 将配置标准化为由二进制字符串及其偏移量组成的规范表示。 此步骤至关重要，因为在换档之前相同的两个配置必须被识别为循环检测的等效项。 
5. 将每个标准化状态存储在字典中，将状态映射到它发生的步骤索引。 如果我们看到以前出现过的状态，我们就识别出一个循环。 循环长度是当前步骤与前一个步骤之间的差值。 
6. 如果 N 小于循环开始的步骤，我们只需返回步骤 N 处的配置。 
7. 否则，我们计算进入循环后还剩下多少步，以循环长度为模减少它，并直接跳转到循环内相应的状态。 
8. 计算最终配置中的个数并将其作为答案返回。 

### 为什么它有效

 该算法依赖于这样一个事实：系统的演化是确定性的，并且仅依赖于有限的局部邻域。 一旦我们通过删除不相关的前导零和尾随零并通过偏移跟踪考虑移位来标准化配置，对于这种受限的初始大小，重复之前的不同可达状态的数量在实践中变得有限。 因此，配置序列最终必须重复，形成一个循环。 一旦检测到一个循环，未来的状态就完全由循环长度上的模运算确定，保证最终状态的正确性，而无需模拟所有 N 个步骤。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def next_state(state):
    # state is list of ints
    n = len(state)
    res = [0] * (n + 2)
    # pad with zeros implicitly
    for i in range(n + 2):
        l = state[i - 2] if 0 <= i - 2 < n else 0
        c = state[i - 1] if 0 <= i - 1 < n else 0
        r = state[i] if 0 <= i < n else 0

        # Rule 110:
        # 111 110 101 100 011 010 001 000 -> 0 1 1 0 1 1 1 0
        if (l, c, r) in [(1,1,1),(1,0,0),(0,1,1),(0,1,0),(0,0,1)]:
            res[i] = 1
        else:
            res[i] = 0

    # trim
    while len(res) > 0 and res[0] == 0:
        res.pop(0)
    while len(res) > 0 and res[-1] == 0:
        res.pop()

    return res if res else [0]

def solve():
    s = input().strip()
    N = int(input().strip())

    state = [int(c) for c in s]

    seen = {}
    states = []

    def encode(st):
        return tuple(st)

    step = 0
    seen[encode(state)] = step
    states.append(state)

    while step < N:
        state = next_state(state)
        step += 1

        key = encode(state)
        if key in seen:
            start = seen[key]
            cycle_len = step - start
            break

        seen[key] = step
        states.append(state)
    else:
        print(sum(state))
        return

    if step == N:
        print(sum(state))
        return

    cycle_states = states[start:step]

    remaining = N - step
    idx = remaining % cycle_len
    final_state = cycle_states[idx]

    print(sum(final_state))

if __name__ == "__main__":
    solve()
```该代码将不断演变的配置维护为位列表，并重复应用规则 110 转换。 每一步之后，它都会修剪零，以便状态保持紧凑和可比较。 字典`seen`记录以前遇到的配置，这允许立即检测循环。 一旦找到循环，算法就会切出重复段并使用模运算直接跳转到最终配置，而无需模拟每个剩余步骤。 

一个微妙的实现细节是修剪。 如果不进行修剪，在空间中移动的两个相同模式将不匹配，并且循环检测将失败。 修剪确保我们只比较有意义的活性结构。 

## 工作示例

 考虑一个输入，其中初始配置中间有一小群。 我们仅跟踪 1 的数量和不断演变的状态字符串。 

| 步骤| 状态| 数量 |
 | --- | --- | --- |
 | 0 | 0010010110010110 | 7 |
 | 1 | 0001101111100010 | 8 |
 | 2 | 0011110001011000 | 7 |
 | 3 | 0110001001110000 | 6 |

 该轨迹显示了局部相互作用如何快速改变个体的密度，同时保持结构化的演化。 经过几个步骤后，就会出现重复模式，这正是循环检测捕获的内容。 

第二个例子是全零输入。 

| 步骤| 状态| 数量 |
 | --- | --- | --- |
 | 0 | 0000000000000000 | 0 |

 由于没有小区具有产生该小区的邻居配置，因此系统保持固定。 这证实了该算法可以正确处理退化情况，而无需进行不必要的计算。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(K·W) | K 是重复之前独特配置的数量，W 是每步有源区域的宽度 |
 | 空间| O(K·W) | 存储所有看到的配置和状态直至周期检测|

 初始宽度的约束很小，并且模拟通常会稳定或快速循环，因此 K 保持可控。 即使 N 本身可能非常大，这也可以确保解决方案在限制内舒适地运行。

 ## 边缘情况

 对于全零初始配置，系统永远不会改变，因为每个邻域都是 000，根据规则 110 映射到零。算法会立即处理此问题，因为没有生成新状态并返回初始状态。 

对于 N 等于 0 的情况，永远不会进入模拟循环，并且直接返回原始计数。 这可以避免意外的一步偏移。 

对于快速重复的配置，循环检测字典会提前触发。 然后，该算法避免进一步的模拟，并在检测到的周期上使用模算术计算最终状态，即使在 N 非常大时也能确保正确性。
