 ## 解决方案
 ## 问题理解

 我们得到一个由大写英文字母组成的长字符串。 将其视为活动的时间表。 每个角色都是一项活动，并且该时间线的任何连续部分都被视为我们可以检查的“几天的子序列”。 

任务是找到整个字符串中只出现一次的最短连续子字符串。 在所有具有此最小可能长度且仅出现一次的子串中，我们必须输出最早出现的子串。 

字符串长度可以大到 300,000，因此任何尝试显式计算所有子字符串甚至固定长度的所有子字符串而不经过仔细优化的方法都会太慢。 子串的数量是二次方的，大约是 N²/2，即使天真地检查它们也已经太大了。 任何解决方案都必须避免重复扫描字符串以进行相等性检查。 

一个常见的陷阱是假设使用哈希集或直接比较来检查长度增加的子字符串就可以了。 例如，即使我们固定长度 L 并滑动一个窗口，每个长度仍然有 O(N) 个子串，并且如果在多个值上尝试 L，那么在最坏的情况下复杂度会变成三次方。 另一种微妙的失败情况来自哈希冲突，如果尝试在没有仔细的双哈希或后缀结构的情况下滚动哈希，这可能会产生错误的唯一性决策。 

关键的困难不仅在于找到唯一的子串，而且还在于高效地确定每个子串出现的次数。 

## 方法

 暴力破解的想法很简单：枚举每个子字符串，通过将其与其他每个子字符串进行比较来计算其出现次数，并跟踪最好的一个。 这在概念上是正确的，因为它直接匹配“发生一次”的定义，但如果通过朴素比较完成，它会立即导致 O(N³) 解决方案，因为存在 O(N²) 子字符串，并且每次比较可能需要 O(N) 时间。 

即使我们使用哈希优化计数，我们仍然面临 O(N²) 子串和沉重的内存压力。 真正的问题是子串比较和计数在重叠子串中是多余的。 

关键的观察是我们实际上不需要独立地知道每个子串的确切频率。 相反，我们只需要知道一个子串是否是唯一的，并且在所有唯一的子串中，我们想要最短的一个。 这表明了一种可以隐式聚合子字符串出现次数的结构。 

带有 LCP（最长公共前缀）的后缀数组是很自然的选择。 后缀按字典顺序排序将相同的前缀分组在一起。 如果两个子串相同，则它们对应于共享至少该长度的公共前缀的后缀的前缀。 LCP 数组允许我们识别每个后缀按排序顺序为其邻域贡献了多少个相同的子字符串。 

一旦后缀被排序，我们就可以为每个后缀确定至少出现两次的前缀的最大长度。 从该后缀开始，任何长于该长度的前缀都是唯一的。 通过跟踪每个后缀唯一的最小长度前缀，我们可以找到全局最短的唯一子字符串。 

这减少了从显式计算子字符串到分析排序后缀空间中的邻接性的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N3) | O(1)-O(N²) | 太慢了 |
 | 后缀数组+LCP| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们使用后缀数组和 LCP 来解决这个问题。

1.构造字符串的后缀数组。 每个后缀对应于字符串中的起始索引，并按字典顺序对它们进行排序，将相似的前缀分组在一起。 
2. 构建LCP数组，其中LCP[i]存储排序后后缀数组中位置i和i−1处的后缀之间的最长公共前缀的长度。 这捕获了相邻后缀一致的时间长度。 
3. 对于按排序顺序的每个后缀，确定至少出现两次的最长前缀。 这只是涉及排序数组中其邻域中该后缀的最大 LCP。 具体来说，对于位置 i 处的后缀，我们考虑 LCP[i] 和 LCP[i+1]，取最大相关重叠。 
4. 如果后缀具有长度为 k 的最长重复前缀，则从该后缀开始的任何长度大于 k 的前缀在整个字符串中仅出现一次。 因此，最短的前缀的长度为 k+1。 
5. 对于每个后缀，我们计算长度为 k+1 的候选唯一子串。 我们追踪全球最小的长度。 如果多个候选者共享相同的长度，我们会选择起始索引最小的一个，这自然是通过按排序顺序迭代后缀并比较位置来强制执行的。 

### 为什么它有效

 当且仅当两个子字符串对应的后缀共享该长度的公共前缀时，两个子字符串才相等。 对后缀进行排序使所有相等的前缀出现在连续的块中。 LCP 数组精确测量按此排序顺序的相邻后缀之间的重叠，这意味着每个重复的子字符串必须表示为后缀间隔，其中相邻 LCP 值至少是其长度。 

对于任何后缀，与其邻居的最大 LCP 告诉我们从那里开始并在其他地方重复的最长子串。 任何超出该边界的东西都必须是唯一的。 这保证了我们永远不会将重复的子字符串错误地分类为唯一，因为任何重复都会在某些相邻后缀比较中强制匹配前缀。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_suffix_array(s):
    n = len(s)
    k = 1
    sa = list(range(n))
    rank = [ord(c) for c in s]
    tmp = [0] * n

    while True:
        sa.sort(key=lambda i: (rank[i], rank[i + k] if i + k < n else -1))

        tmp[sa[0]] = 0
        for i in range(1, n):
            prev = sa[i - 1]
            cur = sa[i]
            tmp[cur] = tmp[prev] + (
                (rank[cur], rank[cur + k] if cur + k < n else -1)
                != (rank[prev], rank[prev + k] if prev + k < n else -1)
            )

        rank = tmp[:]
        if rank[sa[-1]] == n - 1:
            break
        k <<= 1

    return sa

def build_lcp(s, sa):
    n = len(s)
    rank = [0] * n
    for i, v in enumerate(sa):
        rank[v] = i

    h = 0
    lcp = [0] * n

    for i in range(n):
        if rank[i] == 0:
            continue
        j = sa[rank[i] - 1]
        while i + h < n and j + h < n and s[i + h] == s[j + h]:
            h += 1
        lcp[rank[i]] = h
        if h:
            h -= 1

    return lcp

def solve():
    s = input().strip()
    n = len(s)
    if n == 0:
        return ""

    sa = build_suffix_array(s)
    lcp = build_lcp(s, sa)

    best_len = 10**18
    best_pos = 0

    for i in range(n):
        start = sa[i]

        left_lcp = lcp[i] if i > 0 else 0
        right_lcp = lcp[i + 1] if i + 1 < n else 0
        mx = max(left_lcp, right_lcp)

        cand_len = mx + 1

        if start + cand_len <= n:
            if cand_len < best_len or (cand_len == best_len and start < best_pos):
                best_len = cand_len
                best_pos = start

    return s[best_pos:best_pos + best_len]

if __name__ == "__main__":
    print(solve())
```后缀数组构造使用标准加倍技术，其中我们使用先前计算的排名按后缀的前 2^k 个字符重复对后缀进行排序。 关键思想是，一旦排名稳定，我们就有了完整的字典顺序。 

LCP 计算使用 Kasai 算法，该算法通过重用先前的比较并仅在迭代之间减少匹配的前缀长度，以线性时间运行。 

在主循环中，我们检查每个后缀，并通过检查相邻的 LCP 值来计算保证其前缀不唯一的程度。 候选子串是唯一性开始的第一个位置。 

一个微妙的点是我们必须考虑后缀数组中的左邻居和右邻居。 子字符串可以在任意一侧重复，并且两个方向都对正确性很重要。 

## 工作示例

 ### 示例 1

 输入：```
AABAABB
```后缀数组（概念上已排序）：```
0: AABAABB
1: AABB
2: ABAABB
3: B
4: BAABB
5: BBAABB
6: BB
```我们关注邻居之间的 LCP 关系。 重要的观察是，重复的前缀如“A”、“B”、“AA”、“BA”等出现在相邻的后缀中。 

| 后缀开始 | 后缀| 与邻居的最大 LCP | 候选长度 | 候选人 |
 | ---| ---| ---| ---| ---|
 | 0 | 啊啊啊啊| 2 | 3 | AA |
 | 1 | AABB | 2 | 3 | AA |
 | 2 | 阿巴布 | 0 | 1 | 一个 |
 | 3 | 乙| 2 | 3 | 工商管理学士 |

 | 5 | BBAABB | 1 | 2 | BB |

该迹线表明，唯一性是按后缀顺序在本地确定的，并且选择了最小的有效候选者。 

### 示例 2

 输入：```
AAAAA
```所有后缀都共享长重叠，因此长度为 1 到 4 的每个前缀都会重复。 只有全长 5 的子串是唯一的。 

| 后缀开始 | 最大 LCP | 候选长度 | 候选人 |
 | ---| ---| ---| ---|
 | 0 | 4 | 5 | 啊啊啊|
 | 1 | 3 | 4 | AAAA |
 | 2 | 2 | 3 | AAA |
 | 3 | 1 | 2 | AA |
 | 4 | 0 | 1 | 一个 |

 只有完整的字符串是唯一的，所以答案是“AAAAA”。

 ## 边缘情况

 对于像这样的字符串`AAAAA`，每个后缀与其后缀数组中的邻居共享长公共前缀。 该算法计算从左到右后缀递减的 LCP 值，并且对于每个位置，候选者成为重复停止影响前缀的第一个索引。 由于没有比全长短的子字符串是唯一的，因此除了完整字符串之外的所有候选字符串都是无效的，并且算法正确地返回完整字符串。 

对于严格递增的字符串，例如`ABCDEF`，没有后缀与邻居共享任何非零 LCP。 每个后缀立即有`mx = 0`，所以每个候选者都是一个字符。 平局决胜规则选择最早的位置，产生`A`，这是正确的，因为每个字符都出现一次并且是最短的可能子字符串。
