 ## 解决方案
 ## 问题理解

 该任务是经典的算术构建块：我们给出整数对，对于每一对，我们必须输出它们的和。 每条输入行代表两个应直接组合的数字，无需图形或数组等附加结构。 输出只是每对的结果整数，按顺序写入。 

尽管这个问题看起来微不足道，但在实践中限制仍然很重要。 此类问题通常允许非常大的整数，绝对值通常高达 10^18。 这立即排除了在存在溢出风险的语言中依赖固定宽度整数假设的任何方法。 在 Python 中，这是自然处理的，但在其他语言中，它强制谨慎使用 64 位类型。 

输入大小也很重要。 由于每条线都是独立的，因此线对总数可能很大，可能高达 10^5 或更多。 这意味着超出每行恒定时间的任何每次测试开销（例如重复的解析逻辑或不必要的数据结构）仍然会通过，但在概念上应该避免。 

有一些微妙的边缘情况可能会破坏简单的实现。 一种是假设仅存在一个测试用例。 例如，如果输入是：```
1 2
3 4
```正确的输出是：```
3
7
```只读取一行的错误实现将错误地仅输出`3`。 

另一个边缘情况是负数，例如：```
-5 2
```正确的输出是`-3`。 无法正确处理减号的脆弱字符串分割或解析方法将在这里失败。 

最后，非常大的值，例如：```
1000000000000000000 1000000000000000000
```在固定大小的整数语言中必须处理无溢出。 

## 方法

 暴力方法已经是预期的解决方案：读取每对整数，计算它们的总和，然后打印它。 其背后不存在隐藏的组合结构或优化问题。 正确性直接来自加法的定义。 

如果我们描述一个“次优”版本，它只会在输入处理方式上有所不同。 例如，将整个文件读入令牌列表并重复将子字符串转换为整数仍然是正确的，但它会引入不必要的开销。 另一种低效的变体可能会重建字符串或对每个字符执行冗余解析，从而在不改变渐近复杂性的情况下增加常数因子。 

关键的观察是每对都是独立的。 任何结果都不依赖于先前的计算，因此我们永远不需要当前行之外的存储。 这将问题简化为对输入的单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 逐行解析与求和 | O(n) | O(1) | O(1) | 已接受 |
 | 具有重复解析开销的令牌缓冲 | O(n) | O(n) | 已接受但没有必要 |

 这里，n是输入行的数量。 

## 算法演练

 1. 读取输入直到文件末尾，将每一行视为一对单独的整数。 这确保我们可以处理任意数量的测试用例，而不需要预定义的计数。 
2. 对于每一行，将其分成两个整数标记。 结构是固定的，因此我们总是期望每行恰好有两个值。 
3. 将两个标记转换为整数。 此步骤对于确保算术加法而不是字符串连接是必要的。 
4. 解析后立即计算两个整数的和。 由于每条线都是独立的，因此不需要中间存储。 
5. 在移至下一行之前输出结果。 无论输入大小如何，这都可以保持内存使用量恒定。 

### 为什么它有效

 正确性取决于每个输入行定义一个独立的算术表达式。 行之间不存在隐藏的依赖关系，因此单独计算每个总和会产生全局解决方案。 该算法保持了这样的不变性：处理 k 行后，恰好输出 k 个正确的和，每个对应于第 k 个输入对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    for line in sys.stdin:
        line = line.strip()
        if not line:
            continue
        a, b = map(int, line.split())
        print(a + b)

if __name__ == "__main__":
    main()
```该解决方案使用流式处理方式处理输入`sys.stdin`，这避免了将整个数据集存储在内存中。 每条线都被剥离和分割一次，确保每个测试用例的持续工作。 

一个微妙的实现细节是处理空行。 竞争性编程输入有时可能包含尾随换行符，因此跳过空字符串可以防止意外的解析错误。 另一个重要的细节是使用`map(int, ...)`直接，避免了中间的列表构造，保持代码简洁。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | n 行中的每一行都被解析并处理一次 |
 | 空间| O(1) | O(1) | 任何时候只存储两个整数 |

 该算法与输入行数呈线性关系，这是最佳的，因为每行必须至少读取一次。 由于不需要累积结果，因此内存使用量保持不变。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    from io import StringIO as _StringIO

    out = _StringIO()
    backup = sys.stdout
    sys.stdout = out

    try:
        for line in sys.stdin:
            line = line.strip()
            if not line:
                continue
            a, b = map(int, line.split())
            print(a + b)
    finally:
        sys.stdout = backup

    return out.getvalue().strip()

 ## 边缘情况

 对于多行输入，算法独立地正确处理每一对。 例如：

 输入：```
1 2
3 4
```执行处理第一行，输出`3`，然后进入下一行并输出`7`。 循环不变量被保留，因为每次迭代之后恰好消耗一行并产生一个结果。 

对于负数：

 输入：```
-5 2
```解析步骤正确地将标记转换为有符号整数，总和计算如下`-3`。 表示中不存在歧义，因为`int()`正确解释减号。 

对于非常大的值：

 输入：```
1000000000000000000 1000000000000000000
```Python 的整数类型会自动扩展精度，因此可以精确计算总和而不会溢出。 该算法不引入任何中间缩小转换，因此保留了正确性。
