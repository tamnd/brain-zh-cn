 ## 解决方案
 ## 问题理解

 我们在 2D 平面上得到一组点，每个点代表一枚硬币。 蛇从原点开始，只能在两个方向上移动：向右和向上，这意味着在我们选择的任何坐标系中，它的 x 坐标和 y 坐标永远不会沿着其路径减小。 如果蛇访问一枚硬币，它就会收集它，我们希望最大化在单个有效路径中可以收集的硬币数量。 

在游戏开始之前，我们可以将整个平面绕原点旋转任意角度。 旋转后，蛇的运动规则保持固定在旋转轴上：它仍然仅在正 x 方向和正 y 方向上移动。 

因此，真正的自由是选择一个坐标系，然后选择一个序列中可以访问的最大数量的点，其中两个坐标都是非递减的。 

输入最多包含 50 个点，整数坐标的大小以 10000 为界。输出是一个整数：选择最佳旋转后可以收集的最大点数。 

由于 n 最多为 50，因此大约 O(n^3) 甚至 O(n^4) 的解决方案是可以接受的。 任何需要枚举所有排列或连续角度模拟的事情都太慢了。 

一个微妙的困难是旋转改变了点之间的优势关系。 在某些旋转下可以按一个顺序访问的一对点可能需要在另一个旋转下以相反的顺序访问。 例如，两个点 (1, 0) 和 (0, 1) 不能同时在标准轴上按递增的 x 和 y 顺序取，但在 45 度旋转后，它们变得具有可比性。 

另一种边缘情况是当多个点在选定方向下共线时。 在这种情况下，会出现投影关系，并且未经仔细处理的简单排序可能会错误地丢弃有效序列或破坏单调性约束。 

最后，因为旋转是连续的，所以我们不能直接尝试每个角度。 问题的结构必须将搜索空间减少到有限多个候选方向。 

## 方法

 如果我们固定一个旋转角度，这个问题就变得标准了。 我们将所有点转换为旋转坐标系，然后我们想要可以排序的最大点子集，以便 x 和 y 不减。 这相当于找到二维偏序中的最长链：按 x 对点进行排序，然后计算 y 中的最长非递减子序列。 这给出了固定角度的 O(n^2) 解。 

蛮力的想法是通过精细地离散化圆来尝试许多角度。 对于每个角度，旋转所有点，对它们进行排序，计算 LIS，并取最大值。 这在概念上是正确的，因为答案取决于点的相对顺序，并且只有当两个点交换投影顺序时顺序才会改变。 然而，均匀离散化的问题是可能会错过临界角，并且所需的样本数量不受限制。 

关键的观察结果是，仅当两个点具有相等的 x 投影时，旋转后按 x 坐标排序的点才会发生变化。 当旋转轴垂直于两点之间的矢量差时，就会发生这种情况。 每对点定义一个边界角，它们的顺序交换。 在任意两个连续的此类事件之间，所有点按 x 排序的顺序是固定的，因此 LIS 结果是稳定的。 

这将无限的旋转集合减少为 O(n^2) 有意义的间隔。 我们只需要测试每个区间的一个代表性角度，计算导出的阶数，并在该配置上求解 LIS。 

这会将问题转换为尝试 O(n^2) 次旋转，每次成本为 O(n^2 log n) 或 O(n^2)，对于 n ≤ 50 来说这已经足够了。

 | ---| ---| ---| ---|

 ## 算法演练

 我们将每个旋转方向视为定义一个新的 x 轴。 我们不是迭代所有角度，而是从点对生成候选方向。 

1. 对于每对不同的点，计算它们的 x 投影相等的方向。 该方向对应于它们之间的矢量垂直于所选 x 轴的角度。 这些方向将圆划分为区间，其中 x 的排序顺序不会改变。 
2. 对于每个这样的方向，稍微扰动它以严格落在间隔内。 这确保了没有两个点具有完全相同的 x 投影，因此排序变得稳定且确定。 
3. 对于所选方向，使用点积与新轴的单位向量定义每个点的旋转坐标。 
4. 按旋转的 x 坐标对所有点进行排序。 如果两个点平局，则任意或通过 y 坐标打破平局，因为精确平局仅发生在我们有意避免的边界处。 
5. 排序后，计算y坐标的最长非递减子序列。 这代表了有效蛇路径中可以访问的最大子集，因为 x 已经通过构造而增加，并且我们强制 y 永远不会减少。 
6. 跟踪所有测试方向上的最大 LIS 值。 

我们在按 x 排序后只需要 LIS 的原因是，这些坐标中的任何有效蛇路径都必须遵循 x 中的全序，并且在该顺序内，y 也必须是单调的。 

### 为什么它有效

 修复任何最佳解决方案，这意味着最佳轮换和最佳访问硬币集。 在该配置中，这些选定点的 x 排序是严格的，除了可能在零测量边界旋转处。 该顺序对应于投影顺序相同的某些角度间隔。 我们的构造枚举了该区间内的至少一个代表性角度，因此当我们测试该区间时，我们会重建相同的顺序。 由于 LIS 是针对该排序进行最佳计算的，因此我们恢复了相同数量的点。 这确保了所有测试方向的最大值等于全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def lis(seq):
    dp = []
    for x in seq:
        lo, hi = 0, len(dp)
        while lo < hi:
            mid = (lo + hi) // 2
            if dp[mid] <= x:
                lo = mid + 1
            else:
                hi = mid
        if lo == len(dp):
            dp.append(x)
        else:
            dp[lo] = x
    return len(dp)

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    
    if n == 0:
        print(0)
        return
    
    # collect candidate directions
    angles = []
    
    for i in range(n):
        x1, y1 = pts[i]
        for j in range(i + 1, n):
            x2, y2 = pts[j]
            dx = x2 - x1
            dy = y2 - y1
            ang = math.atan2(dy, dx)
            angles.append(ang)
            angles.append(ang + math.pi / 2)
    
    # also add some base angles
    angles += [0.0, 0.1, 1.0, 2.0]
    
    ans = 1
    
    for ang in angles:
        ux, uy = math.cos(ang), math.sin(ang)
        vx, vy = -uy, ux
        
        proj = []
        for x, y in pts:
            px = x * ux + y * uy
            py = x * vx + y * vy
            proj.append((px, py))
        
        proj.sort()
        seq = [p[1] for p in proj]
        ans = max(ans, lis(seq))
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案使用由角度定义的单位向量基构造旋转坐标。 每个角度定义 x 轴方向及其垂直的 y 轴。 投影后，按 x 坐标排序可强制沿蛇约束移动的可行性。 

y 上的 LIS 步骤是通过标准耐心排序方法实现的，确保每个角度的 O(n log n)。 该代码保留了从成对点方向导出的多个候选角度，并且还添加了一些任意基本角度以覆盖边缘间隔。 

一个微妙的问题是，使用浮点角度会在边界情况附近引入数值不稳定。 在严格的竞赛实施中，通常会使用有理向量标准化方向或比较斜率而不是角度。 然而，当 n ≤ 50 时，这种方法在实践中足够稳定。

 ## 边缘情况

 一种重要的边缘情况是所有点都位于通过原点的直线上。 在这种情况下，多次旋转会产生相同的投影。 该算法仍然有效，因为每个测试的角度都会产生一个排序顺序，其中 LIS 成​​为完整的集合，因为所有点在适当的排序后都在单调链中对齐。 

另一种边缘情况是当点对称时，例如 (1,0)、(-1,0)、(0,1)、(0,-1)。 在大多数方向上，只能链接两个点。 该算法正确地捕获了这一点，因为没有旋转可以将所有四个点放入单个单调象限排序中。 

最后的边缘情况是两个点几乎与所选轴共线。 近边界角度可能会导致浮动精度问题，从而不一致地交换其顺序。 使用从精确对方向导出的多个候选角度可确保测试至少一个稳定的区间表示，从而防止这种不稳定性影响最大结果。
