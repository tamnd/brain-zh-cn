 ## 问题理解

 我们得到一棵树，它来自应用于某些有向图的转换过程。 该过程运行深度优先搜索，分配发现时间，维护顶点堆栈，每当满足特定的低链接条件时，它就会创建一个新的辅助节点并向其附加一批顶点。 结果总是一棵树，我们称之为$T$。 重要的一点是，这棵树不是任意的，它编码了原始图的顶点和“结构组件”在 DFS 期间如何粘合在一起。 

现在我们没有被要求重建原始图。 相反，我们可以删除该树中的任何边子集$T$，产生一片森林。 然后，该森林的每个连接组件都被解释为一棵独立的树，并且我们被告知假设它对应于某个有效的原始图，当且仅当该组件在相同的 DFS 构造规则下“有效”。 为了解决这个问题，该声明保证有效性仅取决于组件作为树的形状，而不取决于节点标签。 

对于每个查询值$x$，我们必须计算有多少种方法来选择一组要删除的边，以便两个条件同时成立。 首先，每个结果组件都必须是上述意义上的有效树。 其次，如果每个组件被解释为来自某个图，那么我们计算该图中的铰接点的数量，并且所有组件的这些计数的总和必须等于$x$。 答案取模$10^9+7$。 

这些约束意味着所有测试用例的节点总数最多为 5000，而查询数量最多为$5 \cdot 10^5$。 这立即迫使解决方案将繁重的预处理与查询应答分开。 任何按查询的遍历都是不可能的，因此所有相关信息必须压缩为全局组合结构，通常是可以在以下位置查询的多项式或 DP 表：$O(1)$或者$O(\log n)$。 

一个微妙的困难是组件的“分数”并不直接附加到边缘。 这取决于节点是否成为其组件内部的连接点，而这又取决于删除后其程度。 这意味着切割一个边缘可能会以非线性方式影响两个端点的贡献，因此天真的独立假设会失败。 

一个常见的失败案例是假设每条边独立地贡献答案。 例如，在星形树中，删除单个边会改变中心的度数是否至少为二，这可以完全翻转其贡献。 所以该结构是全局耦合的。 

## 方法

 强力方法将枚举所有边子集，相应地分割树，检查每个组件的有效性，计算关节计数并累积频率。 这是正确的，但立即不可行，因为一棵树$n-1$边缘已经有$2^{n-1}$子集，甚至远远超出任何限制$n=20$。 

关键的观察结果是，删除后的结构仍然是森林，因此组件仅通过我们切割的边缘进行交互。 如果我们以树为根并自下而上地处理它，则每个关于切割边缘的决定只会将子树与其父树分开。 这表明树DP中每个节点聚合来自其子节点的信息，并且每条边代表一个二元选择：将其保留在组件内或将其剪切。 

主要困难在于分数取决于每个最终组件内的节点度。 然而，组件内部的程度取决于保留的入射边的数量。 这使我们能够将问题从“成分有效性”转移到“局部学位计数”。 

我们最终构建了一个 DP，对于每个子树，计算它可以通过多少种方式贡献给定数量的保留边以及对清晰度分数的给定贡献。 由于树的大小总共只有 5000 棵，因此对子树进行仔细的背包式合并会产生$O(n^2)$解决方案。

 ## 算法演练

 我们以任意节点为树根。 这个想法是处理每个子树并根据其父级的边缘是保留还是剪切来计算它的行​​为方式。 

### 1.DP定义

 对于每个节点$u$，我们根据边缘是否来自$u$其父级是否保留（状态 1）或不保留（状态 0）。 对于每个状态，我们存储子树内保留的边数和该子树内的总清晰度贡献的分布。 

DP条目可以被认为是：$$dp_u[a][k][s]$$在哪里$a \in \{0,1\}$指示是否保留父边，$k$是子树内保留的子边的数量，并且$s$是子树中节点的累积清晰度分数，不包括$u$自己的最终贡献。 

### 2.初始化

 叶节点没有子节点。 它唯一可能的度数来自父边。 因此，如果保留父边，则其度数为 1，否则为 0。在这两种情况下，它对分数的贡献均为 0，因为它无法达到至少 2 的度数。 

### 3. 合并子项

 对于一个节点$u$，我们一一迭代它的孩子。 处理孩子时$v$，我们有两个选择：保持优势$(u,v)$或剪掉它。 保留它会增加程度$u$该组件中的值减 1，而切割则将子组件完全分离。 

这成为 DP 状态上的一个卷积步骤：我们将保留边缘的分布和子项的累积分数结合起来。 

关键点是，在处理完所有子项之后，我们确切地知道有多少个关联边$u$保存在其组件内部。 

### 4. 添加贡献$u$一旦保留的入射边数为$u$确定后，我们可以计算是否$u$成为其组件内的一个接合点。 当且仅当组件内部的总度数至少为 2 时，才会发生这种情况。该度数为：$$deg(u) = (\text{kept child edges}) + (\text{parent edge if kept})$$所以我们添加：$$+1 \quad \text{if } deg(u) \ge 2$$### 5.最终聚合

 计算根的 DP 后，我们对两个根状态求和并收集每个可能的总分的计数$x$。 这给出了直接回答所有查询的全局频率数组。 

### 为什么它有效

 正确性来自于以下事实：除了通过节点度之外，每个边决策都是独立的，并且节点度完全由事件边的局部选择决定。 DP 确保对于每个子树，我们考虑所有一致的配置，并且在合并子树时，我们保留精确的度数计数。 由于衔接贡献仅取决于最终程度，因此不存在隐藏的全局依赖性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    sys.setrecursionlimit(1000000)

    parent = [-1] * n
    order = []

    def dfs(u, p):
        parent[u] = p
        for v in g[u]:
            if v == p:
                continue
            dfs(v, u)
        order.append(u)

    dfs(0, -1)

    dp = [None] * n

    def merge(a, b):
        na = len(a)
        nb = len(b)
        res = [0] * (na + nb)
        for i in range(na):
            if a[i] == 0:
                continue
            for j in range(nb):
                if b[j] == 0:
                    continue
                res[i + j] += a[i] * b[j]
                res[i + j] %= 1000000007
        return res

    def dfs2(u, p):
        dp0 = [1]
        dp1 = [0]

        for v in g[u]:
            if v == p:
                continue
            child = dfs2(v, u)
            new0 = merge(dp0, child)
            new1 = merge(dp1, child)
            dp0, dp1 = new0, new1

        size = len(dp0)
        add0 = [0] * size
        add1 = [0] * (size + 1)

        for k in range(size):
            ways0 = dp0[k]
            ways1 = dp1[k]
            if k >= 2:
                add0[k] = (add0[k] + ways0) % 1000000007
            if k + 1 >= 2:
                add1[k + 1] = (add1[k + 1] + ways1) % 1000000007

        dp0 = [(dp0[i] + add0[i]) % 1000000007 for i in range(size)]
        dp1 = [(dp1[i] + add1[i]) % 1000000007 for i in range(size + 1)]

        dp[u] = (dp0, dp1)
        return dp[u]

    dp_root = dfs2(0, -1)
    dp0, dp1 = dp_root

    q = int(input())
    freq = {}

    def add(arr, shift):
        for i, v in enumerate(arr):
            if v:
                freq[i + shift] = (freq.get(i + shift, 0) + v) % 1000000007

    add(dp0, 0)
    add(dp1, 0)

    for _ in range(q):
        x = int(input())
        print(freq.get(x, 0) % 1000000007)

if __name__ == "__main__":
    solve()
```该实现是围绕后序 DFS 构建的。 每个节点通过一次合并一个子节点来构建其 DP，将每个子边缘视为二元包含选择。 每个节点的最终调整根据其最终度数阈值考虑该节点是否成为铰接点。 

一个微妙的实现细节是，我们必须为是否包含父边缘保留单独的状态，因为这会影响程度，从而影响清晰度条件。 只有在根 DP 完成后才会聚合最终答案。 

## 工作示例

 考虑一个由三个节点组成的小链$1-2-3$。 我们以 1 为根。 

在节点 3 处，只有一种可能的配置：它没有任何贡献，因为在所有情况下其度数最多为 1。 在节点 2 处，根据是否保留到 1 和 3 的边，其度数范围可以为 0 到 2，并且只有保留两条边的情况才会产生 1 的清晰度贡献。 

| 节点| 在子树 | 中保留边 学位条件| 贡献|
 | ---| ---| ---| ---|
 | 3 | 0 或 1 | 绝不≥ 2 | 0 |
 | 2 | 0,1,2 | 取决于| 0 或 1 |
 | 1 | 取决于| 取决于| 0 或 1 |

 此示例演示了贡献如何不是边缘局部的而是取决于组合子树结构。 

现在考虑一颗以 1 为中心、叶子为 2、3、4 的星形。 仅当至少保留两条边时，中心才成为铰接点。 这迫使 DP 跟踪所选边的计数，而不是独立的边贡献。

 ## 边缘情况

 单节点树很重要，因为它迫使 DP 正确处理不存在边缘决策且无法出现连接的组件。
