 ## 解决方案
 ## 问题理解

 我们得到了一组信号。 每个信号都有两个数字属性：能量值和频率值。 任务是考虑每对有序信号并累积由两个独立部分的乘积定义的成本。 

第一部分仅取决于能量，并且只是两个能量值之间的绝对差。 第二部分仅取决于频率，并且是两个频率值的 gcd 的平方。 每对都将这两个因素相乘，然后我们对所有对进行求和，包括订单对和自对（如果按字面解释的话）。 

因此，输入的直接解释是由信号索引的两个数组。 一个阵列控制能量对的能量差异程度，另一个阵列控制它们的频率通过公约数相互增强的程度。 

这些约束立即排除了二次方法。 对于多达二十万个信号，迭代所有对已经给出了 4 × 10^10 次运算，并且每个运算都包含一个 gcd 和一个绝对值。 即使进行了优化，这也远远超出了几秒钟内可以完成的范围。 

一个更微妙的困难是该函数不能以简单的方式分离。 gcd 项以非线性方式耦合两个索引，而绝对差取决于完全不同的数组。 这可以防止对整个对集进行简单的卷积或排序技巧。 

还有一些边缘情况暴露了幼稚推理的陷阱。 例如，如果所有频率都相等，则 gcd 项是常数，并且问题会分解为经典的绝对差之和。 在这种情况下，任何过度设计 gcd 处理的解决方案仍然必须正确减少。 相反，如果所有能量相等，则无论频率结构如何，每一项都变为零，这是对正确性的有用的健全性检查。 

另一个极端情况是频率成对互质。 在这种情况下，gcd 始终为 1，因此问题再次简化为绝对差的纯和，并且任何超重频率结构的解决方案都会错误地引入不存在的复杂性。 

## 方法

 暴力法很简单。 对于每对指数，计算其频率的 gcd，对其进行平方，计算其能量的绝对差，并将乘积添加到答案中。 这是正确的，因为它直接遵循定义，但它执行的是二次方数量的 gcd 计算。 当n达到200000时，这会导致数百亿次操作，这是不可行的。 

主要障碍是 gcd 平方项。 它建议按公约数结构对对进行分组，而不是独立处理每对。 如果两个频率共享 gcd g，则两者都必须是 g 的倍数。 这一观察结果使我们能够通过可分性而不是直接配对来对信号进行分组。 

我们扭转观点。 我们不是先配对并计算 gcd，而是固定候选 gcd 值 g 并查看频率均可被 g 整除的所有对。 这些对贡献 g² 乘以它们的能量差，但只有其中一些的 gcd 恰好等于 g。 这是除数的经典包含-排除结构。 

对于固定的 g，定义其频率可被 g 整除的所有索引。 在这些指标中，我们可以通过排序有效地计算能量的绝对差之和。 然而，这计算了所有 gcd 是 g 倍数的对，而不是精确的 g。 为了隔离精确的 gcd 贡献，我们在除数格上应用莫比乌斯反演。

我们首先计算，对于每个 g，所有频率可被 g 整除的对的总贡献，无论它们的精确 gcd 是多少。 然后我们使用莫比乌斯函数减去来自较大 gcd 倍数的贡献。 在获得每个 gcd 值的精确对贡献后，我们乘以 g²并对所有 g 求和。 

唯一剩下的技术任务是计算每个 g 的频率可被 g 整除的指数之间所有能量绝对差对的总和。 这可以通过收集该组中的能量，对它们进行排序，并使用前缀和来计算排序后线性时间内的总成对绝对差来完成。 

一个关键的效率观察是，每个索引都出现在仅包含其频率的约数的可整除列表中，并且最大 2 × 10^5 的所有数字的约数总数平均而言很小。 这使得整个工作易于管理。

 | 蛮力 | O(n² log max b) | O(n² log max b) | O(1) | O(1) | 太慢了|

 ## 算法演练

 1. 预先计算莫比乌斯函数直至最大可能的频率值。 这允许稍后对除数倍数进行包含-排除。 
2. 对于每个可能的除数 g，构建索引 i 的列表，使得 b[i] 可被 g 整除。 这重新组织了问题，以便每个组对应于一个候选共享频率因子。 
3. 对于每个 g，计算值 G(g)，定义为绝对差之和 |a[i] - a[j]| 覆盖可被 g 整除的索引组内的所有对。 这是通过对该组中的能量进行排序并使用前缀和来有效地累积成对差异来完成的。 
4. 将 G(g)（计算所有可被 g 整除的对进行计数）转换为 F(g)（仅计算频率 gcd 恰好为 g 的对）。 这是通过使用莫比乌斯反演从 g 的所有倍数中减去贡献，并对 mu(k) * G(gk) 求和来实现的。 
5. 每个精确的 gcd 类 g 都会对最终答案贡献 g² * F(g)。 对所有 g 进行累加并返回对 998244353 取模的结果。 

正确性来自两层分解。 第一层用除数约束替换 gcd，将成对的 gcd 条件转变为除数组中的成员资格。 第二层使用莫比乌斯反演消除共享倍数的过度计数，确保每对在正确的 gcd 类中精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def build_mobius(n):
    mu = [1] * (n + 1)
    prime = []
    vis = [False] * (n + 1)
    for i in range(2, n + 1):
        if not vis[i]:
            prime.append(i)
            mu[i] = -1
        for p in prime:
            if i * p > n:
                break
            vis[i * p] = True
            if i % p == 0:
                mu[i * p] = 0
                break
            else:
                mu[i * p] = -mu[i]
    return mu

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    maxv = max(b)
    mu = build_mobius(maxv)

    divs = [[] for _ in range(maxv + 1)]
    for i, bi in enumerate(b):
        j = 1
        while j * j <= bi:
            if bi % j == 0:
                divs[j].append(i)
                if j * j != bi:
                    divs[bi // j].append(i)
            j += 1

    def pair_abs_sum(vals):
        vals.sort()
        res = 0
        pref = 0
        for i, x in enumerate(vals):
            res += x * i - pref
            pref += x
        return res

    G = [0] * (maxv + 1)

    for g in range(1, maxv + 1):
        idxs = divs[g]
        if not idxs:
            continue
        vals = [a[i] for i in idxs]
        G[g] = pair_abs_sum(vals)

    F = [0] * (maxv + 1)

    for g in range(maxv, 0, -1):
        if G[g] == 0:
            continue
        s = 0
        k = 1
        while g * k <= maxv:
            s = (s + mu[k] * G[g * k]) % MOD
            k += 1
        F[g] = s % MOD

    ans = 0
    for g in range(1, maxv + 1):
        if F[g]:
            ans = (ans + F[g] * (g * g % MOD)) % MOD

    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现首先构造莫比乌斯值以实现除数的反转。 然后，它根据频率的可分性对索引进行分组，这是远离成对处理的关键结构转变。 对于每个组，在线性时间内对能量进行排序和处理，以计算总绝对差贡献。 最后，在应用平方 gcd 权重之前，对倍数进行反向包含-排除会隔离精确的 gcd 类别。 

一个微妙的实现点是，按除数分组会创建重叠的集合，因此原始总和 G(g) 被故意多算。 莫比乌斯步骤可以恢复正确性，省略它会根据它们共享的除数数多次对对进行计数。 

## 工作示例

 考虑一个小型配置，其中频率共享结构：a = [1, 4, 2]，b = [2, 4, 6]。 

我们首先建立除数组。 对于 g = 1，所有索引都包括在内；对于 g = 2，我们包括所有索引，因为所有频率都是偶数；对于 g = 3，仅通过 6 间接包括最后两个，依此类推。 

| 克| 指数（b 能被 g 整除）| 能量| 重力（克）|
 | ---| ---| ---| ---|
 | 1 | [0,1,2]| [1,4,2]| 从所有对计算 |
 | 2 | [0,1,2]| [1,4,2]| 结构相同 |
 | 3 | [2] | [2] | 0 |

 计算 G(g) 后，莫比乌斯求逆将贡献分开，以便将 gcd 恰好为 2 的对与 gcd 为 1 或 4 的对区分开。然后用 g² 对每个类别进行加权，产生最终的总和。 

该迹线显示了同一组索引如何影响多个除数级别，并且只有反转才能正确隔离真正的 gcd 分类。 

现在考虑一个退化情况 a = [5, 5, 5], b 任意。 每个能量差为零。 

| 克| 重力（克）| F（克）|
 | ---| ---| ---|
 | 任何| 0 | 0 |

 该算法无需推理频率结构即可正确产生零，从而确认能量分量完全主导正确性。

 ## 边缘情况

 所有频率都相同的情况会将所有除数组折叠成一个完整的集合。 该算法通过将每个索引放入该频率的每个除数桶中来处理此问题，莫比乌斯反转会取消除正确 gcd 级别之外的所有贡献，确保不会出现过度计数。 

当所有频率成对互质时，每个非平凡除数桶最多包含一个元素。 在这种情况下，对于所有大于 1 的 g，G(g) 为零，并且只有 g = 1 起作用。 反演步骤变得微不足道，算法正确地简化为加权 1² 的纯绝对差和。 

如果所有能量都相同，则每个计算出的 G(g) 为零，因为排序的前缀差异完全抵消。 这通过反演传播，因此所有 F(g) 都为零，并且无论频率结构如何，最终答案都为零。
