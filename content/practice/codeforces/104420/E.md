 ## 解决方案
 ## 问题理解

 我们正在开发一个网格，其中每个单元格都包含一个值。 玩家在这个网格中移动并从他们访问的单元格中“收集”值。 移动受到限制：在任何单元格中，您可以向右、向左或向下移动，但不能向上移动。 每次移动都会受到固定的惩罚，因此较长的徘徊路径会降低最终得分。 

分数的计算方式为所有不同访问过的单元格的值之和减去移动次数乘以每次移动的成本。 重新访问单元格不会再次增加总和，但它仍然会消耗移动次数，因此会增加惩罚。 任务是选择一个起始单元、一个结束单元以及一条使该分数最大化的有效路径。 

关键的困难在于路径不是简单的定向行走。 由于允许向左和向右移动，因此您可以以之字形方式遍历行，但只能向下移动或保持在同一行级别。 该结构成为水平探索与受控垂直进展的结合。 

这些限制使得暴力破解路径变得不可能。 每个测试的网格可以大到 2000 x 2000，并且最多有 1000 个测试用例，整个测试的总网格大小上限为 2000 x 2000。这强烈表明每个测试预计采用 O(nm) 或 O(nm log nm) 方法。 

一个天真的想法是尝试所有路径或将其视为具有状态“访问集”的图上的最短路径/最长路径。 这会立即失败，因为重新审视结构和路径组合呈指数爆炸。 

一种更微妙的简单方法是修复起始单元并尝试对以某种方式压缩的访问集的位置进行动态编程。 这也会失败，因为重新访问破坏了标准 DP 单调性。 

一个关键的不明显的陷阱是假设重新访问单元格总是无用的。 它不是。 有时需要重访，才能少走有利可图的弯路。 例如，考虑一行：```
5  -100  5
```如果向右移动的成本为 2，如果您可以重复使用正值并在多个高价值单元上分摊成本，那么从左 5 到右 5 并返回可能仍然是最佳选择。 贪婪的“一次取正邻居”的方法在这里被打破。 

另一个微妙的失败案例是假设最佳路径是单调向下和单调向右的。 由于允许向左移动，因此您可以在下降之前多次蜿蜒穿过一行，这通常是最佳结构。 

## 方法

 暴力解释将每个状态视为位于具有一组已访问节点的单元中。 转换对应于向左、向右或向下移动，更新访问集并支付每次移动的成本。 这是正确的，但不可行，因为状态空间以 nm 为指数。 

下一步是删除“访问集”依赖项。 关键的观察结果是，重新访问一个单元格并不重要，除了它是否已经被包含过一次之外。 因此，路径的增益仅取决于哪些单元至少被包含一次，而不取决于它们被访问的次数。 这将问题转换为在运动约束下选择连接结构。 

现在考虑有效路径可以形成什么结构。 由于您可以在一行内自由左右移动，但只能在行之间向下移动，因此每行可以被视为您可能多次遍历的段，但垂直过渡仅发生在相邻行之间。 这强烈建议采用逐行动态编程，我们决定如何遍历每一行，同时考虑转换成本。 

关键的转变是将移动成本解释为连续访问之间的边缘成本。 我们不考虑路径，而是考虑每行条目的贡献和行之间的转换。 在一行中，最佳行为减少为选择连续的段或将段与回溯成本组合起来。 

这导致了经典的优化：对于每一行，计算从上方进入的最佳方式，在考虑成本的同时遍历左右段，然后向下退出。 问题简化为维持每列每行的最佳可实现值，向下传播，同时允许水平松弛。 

我们维护 DP，其中 dp[i][j] 是处理第 i 行后以单元格 (i, j) 结尾的最佳分数。 我们每行计算两次：从左到右和从右到左，积累最佳增益，同时减去移动成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（具有已访问集的路径）| 指数| 指数| 太慢了|
 | 具有水平松弛的行式 DP | O(纳米) | O(米) | 已接受 |

 ## 算法演练

 我们逐行处理网格，保持当前行中每个单元格结束时可获得的最佳分数。

1. 初始化第一行的 DP 数组，其中每个单元格以其自己的值减去无移动成本开始，因为开始不需要移动。 这表示从顶行中的任何单元格开始路径。 
2. 对于每一行，计算从左到右的遍历。 我们维持一个运行最佳值，该值代表水平延伸路径。 当从 j-1 列移动到 j 列时，我们减去成本 c 并添加当前单元格值。 该模型在同一行中继续一条路径。 
3. 将从左到右传递的最佳值存储到临时数组中。 
4. 在同一行上执行从右到左的传递。 这反映了上一步，确保也考虑从右侧进入的路径。 再次，我们保持最佳跑步状态并应用每步的移动成本。 
5. 通过从任一方向获取每个单元可实现的最大值来组合两次传递。 这可确保捕获行内的最佳之字形遍历。 
6. 行水平优化后，向下传播值。 对于下一行中的每个单元格，考虑从当前行单元格直接向下转换，减去移动成本 c 并添加下一行的单元格值。 相应地更新 DP。 
7. 重复此操作，直到处理完最后一行。 答案是所有 DP 状态的最大值。 

### 为什么它有效

 DP 状态将所有路径历史压缩为以每个单元结束的最佳可实现分数。 水平传递正确地模拟一行内所有可能的左右行走，因为任何此类行走都会分解为相邻移动的序列，并且每个移动具有统一的成本。 垂直转换是独立的，仅取决于前一行的结束位置，因此不需要额外的历史记录。 由于重新访问不会增加价值，因此将多次访问压缩为每个单元格的单个最佳前缀可以保留最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m, c = map(int, input().split())
        a = [list(map(int, input().split())) for _ in range(n)]

        dp = [-10**30] * m

        for i in range(n):
            left = [-10**30] * m
            cur = [-10**30] * m

            for j in range(m):
                if i == 0:
                    best = 0
                else:
                    best = dp[j]
                if j > 0:
                    best = max(best, left[j - 1] - c)
                left[j] = best + a[i][j]
                cur[j] = left[j]

            right = [-10**30] * m
            for j in range(m - 1, -1, -1):
                if i == 0:
                    best = 0
                else:
                    best = dp[j]
                if j + 1 < m:
                    best = max(best, right[j + 1] - c)
                right[j] = best + a[i][j]
                cur[j] = max(cur[j], right[j])

            if i < n - 1:
                new_dp = [-10**30] * m
                for j in range(m):
                    new_dp[j] = cur[j] - c + a[i + 1][j]
                dp = new_dp
            else:
                dp = cur

        print(max(dp))

if __name__ == "__main__":
    solve()
```该代码为每行维护一个滚动 DP。 左右数组模拟了所有可能的具有成本累积的水平遍历。 关键的实现细节是，在应用水平扩展之前，我们始终将前一行的最佳贡献带入当前行。 行之间的转换减去移动成本并立即添加下一行的单元格值，这符合每次移动支付成本且每个访问的单元格贡献一次的规则。 

第一行的初始化使用零作为基数，因为开始不需要从上面输入。 这避免了错误地惩罚第一个访问的小区。 

## 工作示例

 ### 示例 1

 考虑一个小网格：```
1  -2
3   4
```成本c = 1

 我们逐行跟踪 dp。 

第 0 行：

 | j | dp 从上面 | 左/右放松| 决赛|
 | --- | --- | --- | --- |
 | 0 | 0 | 0 + 1 = 1 | 1 |
 | 1 | 0 | 最大(-1, 1-1)=0 → 0 + (-2) = -2 | -2 |

 第 1 行：

 | j | 上面的 dp | 水平| 决赛|
 | --- | --- | --- | --- |
 | 0 | 1 | 1 + 3 = 4 | 4 |
 | 1 | -2 | 最大值(4-1, -2) = 3 → 3 + 4 = 7 | 7 |

 答案是7。 

该迹线显示了水平松弛如何允许跨行携带值并在有益时将多个单元格组合成一行。 

### 示例 2

 网格：```
-1  9  1
 2 -5  3
```成本c = 2

 第 0 行：

 最佳路径向右移动收集 9 和 1：

 | j | 价值|
 | --- | --- |
 | 0 | -1 |
 | 1 | 9 |
 | 2 | 10 - 2 = 8 | 10 - 2 = 8

 第 1 行：

 我们向下传播并组合：

 | j | 上面的 dp | 最佳水平| 决赛|
 | --- | --- | --- | --- |
 | 0 | -1 | -1 + 2 = 1 | 1 |
 | 1 | 9 | 最大(1-2, 9)=9 → 9 + (-5)=4 | 4 |
 | 2 | 8 | 最大(4-2, 8)=8 → 8 + 3=11 | 11 | 11

 最终答案是 11，这是通过优先考虑第二行中最右边的路径来实现的，同时仍然受益于早期的高值单元格。 

这些示例强调，最佳路径通常会在向下移动之前“收集”一行中的高价值单元格。

 ## 边缘情况

 一个关键的边缘情况是，留在单行内比向下移动更好。 例如：```
1  100  1
```由于移动成本较高，该算法将 dp 保持在同一行，并使用水平松弛来捕获两个正细胞，而无需不必要的下降。 左右通道确保两端均可到达，而不会错误地重复计算移动成本。 

另一个边缘情况是单列网格。 在这种情况下，水平过渡完全消失，算法简化为沿列向下的直线 DP。 该实现仍然有效，因为左右通道没有任何作用，只有垂直过渡保持活动状态。 

最后一个边缘情况是负网格，其中每一步的成本都很高。 该算法正确地更喜欢单个单元格，因为 dp 初始化允许选择任何开始而无需强制移动，因此最佳答案是最大单个单元格值。
