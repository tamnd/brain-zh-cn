 ## 解决方案
 ## 问题理解

 我们正在研究一个无限的方形网格，其中每个单元格都以白色开始。 然后我们准确选择`n`单元格并将它们重新涂成红色。 网格被视为一个图，其中每个单元通过单位边连接到其四个邻居。 

着色后，我们查看连接红细胞和白细胞的所有边。 这些被称为蓝边。 任务是安排`n`红色单元格，使此类边界边的数量尽可能少，然后计算该最小值。 

关键的困难在于相同数量的红细胞的不同形状会产生不同的边界尺寸。 细长的形状具有较大的周长，而紧凑的形状则使其周长最小化。 问题本质上是要求任何连接或断开连接的最小周长`n`网格单元。 

约束条件`n ≤ 10^18`立即排除任何建设性模拟或 DP 形状。 任何显式构建形状或迭代网格单元的方法都是不可能的。 我们需要一个封闭的形式或直接的数学表征。 

出现微妙的边缘情况时`n`很小。 例如，当`n = 1`，红细胞有 4 个蓝色边缘。 什么时候`n = 2`，根据两个单元是否相邻，边界会发生变化。 线性放置单元的天真的贪婪方法不一定会最小化周长，因为它忽略了共享边降低边界成本的事实。 

## 方法

 思考这个问题的强力方法是考虑所有可能的配置`n`红细胞并计算它们的边界。 从概念上讲，这是通过将每个配置视为网格点的子集，然后对接触白色单元格的所有边缘进行计数来实现的。 然而，即使是谦虚的`n`，配置的数量以组合方式增长，因为我们选择`n`来自无限网格的细胞。 即使限制在边界框内仍然会导致形状枚举呈指数增长，这使得这完全不可行。 

关键的见解是将问题重新构建为网格上的几何优化问题。 一组网格单元的边界就像一个周长，最佳形状总是尽可能紧凑。 在网格中，紧凑性对应于形成接近正方形的区域。 这类似于最小化连续几何中固定区域的周长，其中正方形在 L1 邻接约束下是最佳的。 

一旦我们接受最佳形状是矩形，我们就可以将问题简化为选择尺寸`a × b`这样`a * b ≥ n`和周长`2(a + b)`被最小化。 最佳配置是面积至少为矩形`n`其周长最小。 

因为我们可以自由地在矩形中保留一些未使用的单元格（只要我们至少有`n`槽），我们只需要尝试候选维度`sqrt(n)`。

 | --- | --- | --- | --- |
 | 暴力枚举| 指数| 指数| 太慢了 |
 | 最佳矩形搜索| 每次测试 O(sqrt(n)) | O(1) | O(1) | 已接受 |

 ## 算法演练

 现在，我们将几何见解转化为具体的过程。 

1.观察到任何最佳配置都可以被视为尺寸的矩形`a × b`和`a * b ≥ n`。 这是因为任何不规则的形状都可以局部重新排列以减小周长而不减少单元的数量。 
2.对于固定宽度`a`，确定最小高度`b`需要适合所有`n`细胞，即`b = ceil(n / a)`。 这可确保矩形包含足够的单元格。 
3. 计算该矩形的周长贡献为`2(a + b)`。 这表示所有四个侧面暴露于白细胞的边缘数量。 
4. 迭代所有可能的值`a`从`1`到`floor(sqrt(n))`。 我们只需要求平方根，因为除此之外，配对维度`b`变得更小并且已经在另一次迭代中被考虑。 
5. 跟踪所有有效区域的最小周长`(a, b)`对。 遇到的最小值就是答案。 

### 为什么它有效

 关键的不变量是任何最佳配置都可以转化为单调的凸形状而不增加其边界。 在网格上，对于固定数量的单元格，周长最有效的凸形状是尽可能接近正方形的矩形。 所有因子对的迭代确保我们检查这种紧凑排列可能产生的每个候选纵横比，因此最小周长矩形必然出现在搜索空间中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        
        ans = 10**30
        
        a = 1
        while a * a <= n:
            b = (n + a - 1) // a
            ans = min(ans, 2 * (a + b))
            a += 1
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码独立处理每个测试用例，并对可能的矩形宽度应用直接搜索。 变量`a`表示候选行数，而`b`计算为覆盖所有所需的最小列数`n`细胞。 

表达式`2 * (a + b)`直接编码矩形的周长。 整数上限除法是使用以下方法实现的`(n + a - 1) // a`，这避免了浮点运算并确保高达以下大值的正确性`10^18`。 

循环绑定`a * a <= n`很关键。 它确保我们只迭代到平方根，防止对对称情况进行不必要的重新计算，在这种情况下交换维度不会产生更好的周长。

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |

 ## 边缘情况

 对于`n = 1`，算法检查`a = 1`, 给予`b = 1`和周长`2(1+1) = 4`。 这与单个细胞接触四个白色邻居的事实相符。 

为了`n = 2`，候选人是`(1,2)`和`(2,1)`，都产生周长`2(3) = 6`。 这证实了该算法可以正确处理不对称的最小形状。 

对于完美的正方形，例如`n = 100`，循环到达`a = 10`，生产`b = 10`和周长`40`。 没有其他因素对可以改善这一点，这表明该算法正确地将平衡配置识别为最佳配置。
