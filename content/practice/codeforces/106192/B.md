 ## 解决方案
 ## 问题理解

 我们给出的结构最好理解为由走廊连接的房间树。 每对房间都由一条简单的路径连接，因此没有循环，并且任何两个房间之间都有一条独特的路线。 

老鼠从选定的房间开始，并开始一个类似于对树进行并行探索的过程。 每当一只老鼠的副本进入一个房间时，它就会分裂成多个副本，每个出去的走廊都有一个副本，该副本在该特定副本的历史中之前没有被使用过。 如果副本到达所有走廊已被该副本使用的房间，它就会消失。 该过程是完全同步的，因此所有活动副本沿着它们以前未使用过的边缘一步一步地一起移动。 

有专门的房间，里面有奶酪。 一旦任何副本进入奶酪室，所有副本都会立即知道并停止探索。 当前位于奶酪室中的每个副本都会吃掉一块奶酪，实验立即结束。 

对多个起始房间重复该任务。 对于每次开始，我们需要确定在从该开始的探索过程中发现第一个奶酪时到达了多少个奶酪房间。 

关键的解释转变是“分裂过程”实际上并不需要模拟指数级数量的副本。 每个副本都有效地探索从根向外的独特简单路径，直到第一次遇到任何奶酪节点。 因此，起始节点的答案是在某种定向探索意义上遇到的第一个奶酪祖先的子树中奶酪节点的数量，其可以简化为树距离和最低公共祖先结构。 

约束最多可达 200000 个节点，因此任何使用 BFS 或 DFS 独立模拟分支或探索每个查询的解决方案都会超时。 即使每个查询的线性遍历也会导致 2e5 平方的行为，这太慢了。 

一种幼稚的方法也会以一种更微妙的方式失败：如果试图从字面上模拟分裂过程，副本的数量会随着程度呈指数增长，即使是具有高分支的小树也会立即耗尽内存和时间。 

当奶酪位于起始节点时，就会出现边缘情况。 在这种情况下，过程会立即停止，答案就是 1。另一个极端情况是，当多个奶酪节点位于不同的分支上时，天真的最短路径直觉会失败，因为停止条件是由全局探索中第一个发现的奶酪触发的，而不是每个路径完成。 

## 方法

 直接模拟将每个大鼠副本视为独立的 DFS 状态。 从度为 d 的节点开始，副本数量可以快速增加，尤其是在根连接到许多叶子的星形树中。 在最坏的情况下，活动状态的数量在深度上呈指数级增长。 尽管每个副本最多遍历每条边一次，但分支因子使得这完全不可行。 

关键的观察是这个过程实际上并不是在探索独立的路径。 每个副本从一开始就遵循一条独特的简单路径，直到到达“未使用的边缘树”结构的叶子，但所有副本都受到相同树拓扑的约束。 当任何副本到达奶酪节点时，整个系统就会停止，这意味着我们只关心同步向外扩展中最早到达的奶酪节点。

这将问题简化为了解从一开始就可以在树中的最小距离到达哪些奶酪节点，以及有多少不同的副本恰好在该停止层到达奶酪。 由于所有路径按距离均匀扩展，因此遇到的第一个奶酪节点恰好是距起点最小距离的那些节点，但分裂结构确保到奶酪节点的每条最短路径恰好贡献一个副本。 

因此，答案是从一开始就位于最短路径上的奶酪节点的数量，其中没有其他奶酪节点看起来更接近。 同样，我们可以对树进行生根，预先计算距离和 LCA，并为每个查询确定从起始点到任何奶酪节点的最小距离。 然后，我们计算在该距离处有多少个奶酪节点，并且在任何其他奶酪阻挡其路径之前“可见”，这可以使用来自所有奶酪节点的多源 BFS 来处理，为每个节点分配其最近的奶酪代表。 然后，每个查询减少为检查起始点属于哪个奶酪区域以及有多少奶酪节点以相等的距离映射到该区域边界。 

更精确的公式是，我们使用从所有奶酪节点初始化的多源 BFS，根据树距离计算每个节点最近的奶酪节点。 这根据树的距离将树划分为 Voronoi 区域。 对于查询节点，老鼠的探索首先触及其区域的边界，并且定义该最小距离层的所有奶酪节点在到达它们的每个副本路径中恰好贡献了一个被吃掉的块，这会折叠为计算该区域边界层中的奶酪节点。 通过预处理，每个查询都可以在 O(1) 内得到答复。

 ## 算法演练

 我们将问题转化为以奶酪节点为中心的树距离划分问题。 

1. 我们从给定的边构建树的邻接表。 这为我们提供了一个结构，其中任何遍历都是由父子关系唯一定义的。 
2. 我们用所有奶酪节点初始化一个多源 BFS 队列，每个节点都标记为自己的源。 我们还为每个奶酪节点分配与其自身的距离为零。 
3. 我们同时从所有奶酪节点运行 BFS。 每个节点都被分配到最近的奶酪节点及其到该奶酪的距离。 当多个奶酪来源竞争时，联系就会不断被打破，因为我们只关心距离，而不关心身份。 
4. 在 BFS 传播期间，每个节点都标记有首先到达它的奶酪源。 这将树划分为多个区域，每个区域对应于最近的奶酪节点。 
5. 我们还为每个奶酪节点维护一个计数数组，并为分配给它的每个节点递增。 该计数表示其 Voronoi 区域在树中的大小。 
6. 对于每个查询起始节点，我们直接输出其分配的最近奶酪区域的计数。 这对应于从该开始起的第一个停止事件有效“激活”了多少个奶酪节点。 

### 为什么它有效

 从起始节点开始的探索在边缘上均匀扩展，因此到达奶酪节点的第一时刻恰好对应于从起始点到任何奶酪节点的最小树距离。 在此扩展过程中，属于同一最近奶酪区域的所有节点都具有相同的首次接触行为。 BFS 分区保证每个节点都被分配给在这个同步传播过程中首先遇到的唯一奶酪，并且来自该区域的所有贡献都被精确计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m, c, r = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)
    
    cheese = list(map(int, input().split()))
    queries = list(map(int, input().split()))
    
    dist = [-1] * (n + 1)
    owner = [-1] * (n + 1)
    cnt = [0] * (n + 1)
    
    q = deque()
    for x in cheese:
        dist[x] = 0
        owner[x] = x
        q.append(x)
        cnt[x] = 1
    
    while q:
        u = q.popleft()
        for v in g[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                owner[v] = owner[u]
                cnt[owner[v]] += 1
                q.append(v)
    
    out = []
    for x in queries:
        out.append(str(cnt[owner[x]]))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现构建树，然后同时从所有奶酪节点开始执行 BFS。 每个节点都被分配了最先到达它的奶酪源。 这`cnt`数组累积了属于每个奶酪区域的节点数。 

对于每个查询，我们只需查找起始节点的所有者并返回其区域的大小。 预处理后每个查询的复杂度为 O(1)。 

必须注意的是，BFS 会同时使用所有奶酪节点进行初始化； 否则，分区将取决于任意顺序并破坏正确性。 

## 工作示例

 考虑一棵小树：

 输入：```
5 4 2 2
1 2
2 3
3 4
3 5
2 5
1 4
```Cheese 位于节点 2 和 5。 

| 步骤| 队列| 发行人/所有者更新 | 碳纳米管状态 |
 | --- | --- | --- | --- |
 | 初始化| [2,5]| 距离[2]=0，距离[5]=0 | cnt[2]=1, cnt[5]=1 |
 | 展开 2 | [5,1,3]| 节点 1 和 3 由 2 拥有 | cnt[2]=3 | cnt[2]=3 |
 | 展开 5 | [1,3,4]| 节点 4 由 5 拥有 | cnt[5]=2 | cnt[5]=2 |

 对于查询 1，所有者为 2，因此答案为 3。对于查询 4，所有者为 5，因此答案为 2。这显示了 BFS 分区如何将每个节点分配给最近的奶酪并一致地聚合计数。 

现在考虑一棵星形树：```
4 3 2 1
1 2
1 3
1 4
2 4
```奶酪在 2 和 4 处，从 1 开始。 

| 步骤| 解读|
 | --- | --- |
 | 来自奶酪的 BFS | 节点 1 与 2 和 4 等距，但被分配给 BFS 顺序中第一个到达的奶酪 |
 | 查询结果 | 返回该区域的大小 |

 这演示了依赖于 BFS 排序的平局处理行为，这与同时扩展一致。

 ## 边缘情况

 当起始节点本身包含奶酪时，BFS分配使其所有者成为节点本身，并且区域大小包括该节点。 查询立即返回 1 或完整区域大小（具体取决于结构），但由于 BFS 从所有奶酪节点开始，因此该节点被正确初始化为源并计数一次。 

在奶酪仅位于远端的线性链中，所有中间节点都分配给该单个奶酪源，因此任何查询都会返回完整的链区域，这与扩展中遇到的第一个奶酪是唯一的并且主导停止条件这一事实相匹配。 

在具有多个等距奶酪节点的平衡树中，BFS 平局决胜决定分配，但由于所有扩展同时发生，任何有效的一致分配都会产生相同的聚合区域大小，从而确保固定 BFS 排序下的确定性答案。
