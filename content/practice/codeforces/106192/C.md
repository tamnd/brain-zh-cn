 ## 解决方案
 ## 问题理解

 王国中的每个城市都选择一个目的地城市，并在它们之间修建一条道路。 结果是每个城市的有向选择，但道路本身是无向的并且具有整数长度。 因为每个城市都只有一个选择，所以每个城市都有一条外出道路，因此整个王国形成了一个结构，其中每个连接的组件都包含恰好一个循环，并且树木依附于该循环。 

现在每条路不再只是一条边。 它被细分为单位段：从一个端点城市开始，我们在沿路的每个整数距离处放置一个柱子，直到到达另一个城市。 所以路还很长`l`成为一条路径`l-1`新的中间职位。 与两个端点城市一起，这意味着每条边都被长度为节点的链所取代`l`。 

所有这些帖子，无论是在城市还是在道路上，都必须使用最多颜色`K`颜色。 唯一的限制是任何路段上的两个相邻的柱子必须具有不同的颜色。 我们被要求计算有多少个有效的颜色取模`1e9 + 9`。 

关键的困难在于该图不是简单的树或简单的循环。 它是一个函数图（每个节点只有一个出边），但是将边扩展为路径后，我们有效地得到了一个更大的图，其中插入了许多二阶节点，并且其长度取决于边权重的环。 

从约束条件来看，`N`可以达到`1.5e5`，边长可达`1e5`。 显式扩展边是不可能的，因为扩展后的节点总数可以达到大约`sum l_i`，它太大而无法直接构建。 这立即迫使解决方案适用于原始功能结构并且仅聚合路径长度。 

一种简单的方法是尝试扩展图并运行 DP 或 BFS 着色计数。 这会失败，因为扩展的图形大小可能会`10^10`在最坏的解释中，或者至少远远超出了记忆和时间的限制。 

重要的边缘情况来自循环处理。 例如，总长度为 3 的单个循环的行为与树或长循环不同，并且对循环中奇偶校验的错误处理将给出错误的答案。 另一个陷阱是自循环：一个城市可能会连接到自身，形成一个长度为 1 的循环，必须按照循环公式进行一致处理。 

## 方法

 强力解释是显式构建扩展图并使用每个连接组件上的 DP 计算正确的颜色。 这在原则上是正确的，因为图表只是一组附加了树的循环，并且分别为树和循环着色是标准的。 然而，扩展步骤使其不可行：节点数量与所有边长度的总和成正比，并且每个 DP 转换在该尺寸上都是线性的。 

关键的观察是扩张是不必要的。 一条有长度的路`l`两个节点之间提供与链完全相同的约束`l`边缘。 我们不需要插入中间节点，只需要知道约束如何沿着路径传播。 对于树木来说，贡献永远是`(K-1)`每条边，无论边是原始边还是扩展边。 对于循环来说，唯一重要的是扩展后循环的总长度。 

这将问题简化为函数图，其中每条边都有一个权重，并且我们只关心沿循环的权重总和以及跨树边的总和。 

每个连通分量都恰好有一个有向循环。 如果我们去掉环的边缘，一切就变成植根于环的森林。 树部分提供了一个简单的乘法因子，而循环则提供了一个已知的闭合形式表达式，用于长度循环的着色`L`。 

对于一个周期长度`L`，有效着色的数量`K`颜色是：$$(K-1)^L + (-1)^L (K-1)$$这来自循环的标准色多项式。 

循环之外的所有内容都像植根于循环节点的树一样，并做出贡献`(K-1)`除循环节点本身之外的每个节点，循环节点本身在循环公式内处理。 

因此，完整的答案成为树贡献和循环贡献的乘积，计算时无需扩展边缘。

 ## 算法演练

 我们直接处理函数图，其中每个节点只有一个出边，但每条边都带有权重。 

1. 我们建立每个城市的邻近信息`u_i`到`v_i`有重量`l_i`。 这定义了带有加权边的函数图。 
2.我们遍历节点来识别环。 对于每个未访问的节点，我们跟踪传出边，直到到达当前遍历中的已访问节点或进入之前处理过的组件。 当我们检测到当前路径内的重访时，我们提取循环并通过求和沿循环的边权重来计算其总长度。 
3. 一旦识别出一个环，我们将其中的所有节点标记为属于该环，并记录其总加权长度`L`。 
4. 从每个循环节点，我们将所有不属于循环的传入边视为树的根。 我们使用 DFS 或通过累积访问过的节点来计算这些树的大小，但我们从不扩展边； 我们只是统计可达的节点，而无需再次进入循环。 
5. 不在循环中的每个节点都属于附加到循环的某个树。 每个这样的节点贡献一个因子`(K-1)`独立地，因为一旦循环节点颜色固定，每个树边都会强制选择不同的颜色，从而给出`(K-1)`每个节点。 
6. 对于每个周期长度`L`，我们将答案乘以循环着色值：$$(K-1)^L + (-1)^L (K-1)$$7. 我们将所有周期贡献相乘，然后乘以`(K-1)`提高到整个图中非循环节点的数量。 

### 为什么它有效

 关键的不变量是，去除循环边缘后，剩余的结构是森林，并且树的着色因素独立地给出根上的固定颜色。 每个非循环节点仅受其父约束的强制，恰好贡献了一个乘法因子`(K-1)`与全局结构无关。 

循环是唯一存在约束的地方。 循环的色多项式仅取决于其总长度，而不取决于该长度的组成方式。 因为每个扩展边都是一条简单路径，所以它的唯一作用是增加总循环长度，保留循环公式保持有效所需的结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 9

def solve():
    n, k = map(int, input().split())
    to = [0] * n
    w = [0] * n
    for i in range(n):
        u, v, l = map(int, input().split())
        u -= 1
        v -= 1
        to[u] = v
        w[u] = l

    visited = [0] * n
    in_stack = [0] * n

    parent = [-1] * n
    pw = [0] * n

    def dfs(start):
        stack = []
        cur = start
        while True:
            visited[cur] = 1
            in_stack[cur] = 1
            stack.append(cur)

            nxt = to[cur]
            if not visited[nxt]:
                parent[nxt] = cur
                pw[nxt] = w[cur]
                cur = nxt
                continue

            if in_stack[nxt]:
                cycle_nodes = []
                cycle_len = 0

                idx = len(stack) - 1
                while True:
                    node = stack[idx]
                    cycle_nodes.append(node)
                    if node == nxt:
                        break
                    idx -= 1

                cycle_nodes.reverse()

                m = len(cycle_nodes)
                for i in range(m):
                    u = cycle_nodes[i]
                    v = cycle_nodes[(i + 1) % m]
                    cycle_len += w[u]

                return cycle_nodes, cycle_len

            break

        for v in stack:
            in_stack[v] = 0
        return None, 0

    total_nodes = n
    cycle_lengths = []
    in_cycle = [False] * n

    for i in range(n):
        if not visited[i]:
            cycle_nodes, clen = dfs(i)
            if cycle_nodes:
                cycle_lengths.append(clen)
                for x in cycle_nodes:
                    in_cycle[x] = True

    # count non-cycle nodes via reachability pruning is complex;
    # here we rely on functional graph property: nodes not in cycles = n - cycle nodes
    cycle_node_count = sum(in_cycle)
    tree_nodes = total_nodes - cycle_node_count

    def modpow(a, e):
        res = 1
        while e:
            if e & 1:
                res = res * a % MOD
            a = a * a % MOD
            e >>= 1
        return res

    ans = 1

    # tree contribution
    ans = ans * modpow((k - 1) % MOD, tree_nodes) % MOD

    # cycle contributions
    for L in cycle_lengths:
        term = (modpow(k - 1, L) + ( -1 if (L % 2) else 1) * (k - 1)) % MOD
        ans = ans * term % MOD

    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现首先根据每个城市的单一外出道路构建功能图。 然后，它执行遍历，使用递归堆栈样式标记来检测循环。 一旦找到一个循环，它的总长度就通过沿循环边缘的权重求和来计算。 

识别出所有环后，不属于任何环的节点将被视为树节点并贡献统一因子`(K-1)`每个。 每个循环根据其总加权长度贡献自己的色多项式值。 所有贡献均在模算术下相乘。 

一个微妙的点是在函数图中正确处理循环检测而无需递归。 基于堆栈的方法确保我们可以重建循环及其精确的加权长度。

 ## 边缘情况

 自循环是最小的可能循环，当城市与自身连接时就会发生。 在这种情况下，循环长度就是该边的权重。 该算法在遍历过程中重新访问自身时立即将该节点检测为循环的一部分，并将循环公式应用为`L = l_i`。 这确保了正确处理长度为 1 的循环，而幼稚的实现通常会忘记特殊的结构。 

另一个边缘情况是所有节点形成一个循环。 遍历永远不会遇到树节点，因此整个答案简化为循环多项式的单个应用。 该算法正确地累积了循环周围的所有权重，并且不会错误地乘以树因子。 

最后的边缘情况是多个独立的周期。 由于每个分量都是单独处理的，因此每个周期对最终结果的贡献是独立的，并且由于模运算，乘法顺序不会影响正确性。
