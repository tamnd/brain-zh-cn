 ## 问题理解

 我们给定一个目标值$n$，我们希望将其表示为所选构建块的总和。 每个构建块都有一个“幂”，并且我们有无限量的块，其幂恰好是素数，并且特殊规则是 1 也被视为素数。 

所以可用的组件是$1, 2, 3, 5, 7, 11, \dots$，我们可以选择这些数字的任意多重集。 表示的成本是所选组件的数量，它们的总和必须恰好等于$n$。 任务是尽量减少所需的组件数量。 

输入是单个整数$n$, 直至$10^9$，所以任何尝试枚举所有素数的解决方案$n$，或在整个范围内执行 DP，直至$n$，将会失败。 线性或偶数$O(n)$立即接近是不可能的，甚至$O(n \log n)$远远超出了限制。 

唯一有意义的运算必须取决于数字的结构而不是它们的值范围。 

出现微妙的边缘情况时$n = 1$。 由于 1 本身是一个有效组件，因此答案是 1。另一个有趣的情况是$n$是一个大素数。 在这种情况下，最佳答案仍然是 1，因为我们可以将数字本身视为单个分量。 

一种幼稚的方法可能假设我们总是想将数字分解为更小的素数，但这并不总是有益的，因为使用$n$如果它本身是素数或 1，则始终允许它本身。 

## 方法

 一个直接的蛮力想法是将其视为最短路径或类似背包的问题。 我们尝试素数的所有组合并反复从中减去它们$n$，跟踪达到零所需的最小元素数量。 这本质上是一个无界硬币找零问题，其中硬币值都是素数（包括 1）。 标准 DP 将定义$dp[x]$作为组件的最小数量总和$x$，以及所有素数的过渡$p \le x$, 更新$dp[x] = \min(dp[x], dp[x-p] + 1)$。 

这是正确的，但立即不可行。 素数个数最多为$10^9$足够大，即使生成它们也是昂贵的，并且对每个状态迭代它们大致给出$O(n \pi(n))$，这远远超出了任何限制。 

关键的观察结果是可用数字的结构使问题崩溃。 由于 1 可用，因此始终可以形成任何整数，并且由于 2 是质数，因此每个整数都可以使用大部分 2 和可能的 1 来分解。这已经表明奇偶校验在结构中占主导地位：我们只需要担心是否可以使用最少数量的大组件来求和。 

更准确的了解方法是询问到底是什么减少了组件数量。 使用大素数$p$取代$p$具有单个元素的成本单位 1 或 2。 所以每次我们使用素数$p$，与使用较小的单元相比，我们减少了组件数量。 每个元素的最佳可能减少来自使用最大可能的单个块，即$n$本身当$n$是素数。 

因此，最优策略在最强意义上总是贪婪的：要么整个数已经是质数或 1，要么我们将其分成 2，如果需要的话可能还分成一个 1。 这导致了一个简单的表征：任何偶数都可以形成为$n/2$2 的副本，而任何奇数合数都受益于用 3 替换一个 2（或等效地使用一个 1 和其余 2），给出$(n-3)/2 + 1$。 这简化为$\lfloor n/2 \rfloor + 1$为所有人$n > 1$，但特殊情况$n = 2$和素数必须小心处理。 

最终的简化是，答案仅取决于是否$n$是否为 1。

 ## 算法演练

 1.读取整数$n$。 我们立即检查是否$n = 1$，因为 1 明确是一个有效分量，无法进一步分解，所以答案固定为 1。 
2.对于任意$n > 1$，观察到$n$本身总是可以使用素数来表示，因为它要么是素数（因此一个分量就足够了），要么可以使用 2 和 3 来表示。 
3. 我们比较两种结构：使用单个组件$n$如果它是素数，否则仅使用小素数。 由于 1 可用，我们总是可以回退到仅使用 1，但这绝不是最佳选择，除了$n = 1$。 
4. 在此设置中，最佳构造永远不需要超过 2 个分量，因为任何合数都可以从可用集合中拆分为最多两个素数（使用 2 和 3 个组合）。 
5. 因此，对于所有$n > 1$，如果组件的最小数量为 1$n$是质数，否则为 2。 

### 为什么它有效

 关键属性是 1 是通用填充物，2 是最小的非单元构建块。 任何合数要么本身就是素数，要么可以使用最多两个允许的分量来表示，因为我们总是可以使用 2 来减少奇偶校验问题，并使用 3 或 1 来调整奇数。因为没有任何构造可以使用少于 1 个分量，并且任何非素数$n > 1$不能由单个组件形成，答案被迫为 1 或 2，具体取决于素数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_prime(x: int) -> bool:
    if x <= 1:
        return False
    if x <= 3:
        return True
    if x % 2 == 0:
        return x == 2
    i = 3
    while i * i <= x:
        if x % i == 0:
            return False
        i += 2
    return True

n = int(input())

if n == 1:
    print(1)
elif is_prime(n):
    print(1)
else:
    print(2)
```该解决方案直接对结构观察进行编码。 素性检查是必要的，因为只有素数可以使用单个分量（本身）来表示，而所有合数都需要至少两个分量，因为没有一个允许的元素等于它们。 特殊情况$n = 1$单独处理，因为 1 明确是一个有效组件。 

素性测试已经足够优化$10^9$使用试除法最多$\sqrt{n}$，速度快于 1 秒。

 ## 边缘情况

 对于$n = 1$，该算法直接返回 1，因为 1 被明确允许作为组件。 永远不需要素数逻辑，从而防止将 1 错误地分类为合数。 

为了$n = 2$，算法返回 1，因为 2 是质数。 尽管它也是最小的偶数，但不需要分解，检查正确地识别了它。 

对于像这样的大素数$10^9 + 7$，循环仅运行到$\sqrt{n}$，大约有 31623 次迭代，完全在限制范围内，并且正确返回 1。 

对于像 100 这样的合数，素性测试会提前失败，结果是 2。一个有效的构造，例如$97 + 3$或多个较小的素数证实了恰好两个分量的可行性。
