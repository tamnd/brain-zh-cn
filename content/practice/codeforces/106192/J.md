 ## 问题理解

 我们正在使用一个 3D 坐标系，该坐标系不是标准的笛卡尔坐标系，而是三角晶格嵌入，其中点由整数三元组表示。 射线由两个不同的点定义，因此其方向由从第一个点到第二个点的向量确定。 

对于每个测试用例，我们都会得到一条这样的射线。 任务是构造另一条从同一初始点开始并垂直于原始射线的射线。 我们不需要描述整条射线，只需要输出一个定义其方向的整数点。 在所有有效的选择中，我们必须选择距离起点最近的一个，如果有多个相同距离的，则任何一个都可以接受。 此外，构造点的坐标必须保持在一个大范围内，因此我们不能依赖极大的缩放技巧。 

关键的几何困难来自于坐标系本身。 尽管这些点以三个坐标给出，但它们处于受约束的结构中，其中有效方向实际上是嵌入在三维空间中的二维对象。 这意味着将天真的解释为自由 3D 几何会产生误导，并且解决方案必须尊重隐藏的平面结构。 

测试用例数量的限制很大，可达两万个，而坐标值则可达千万个数量级。 这迫使每个测试用例的构建时间恒定。 任何涉及搜索候选点或用无界迭代求解系统的方法都会太慢。 

如果我们假设我们可以仅使用一个坐标轴来选择任意垂直向量，则幼稚推理的微妙失败情况就会出现。 例如，如果我们尝试通过简单地交换或取反坐标来构造向量，我们可能会意外地留下有效的三角形平面或在该几何中失败正交性。 

当尝试随机构造或强力搜索垂直整数方向时，会出现另一种故障模式。 尽管总有解决方案存在，但搜索空间原则上是无界的，如果没有直接的公式，在 20,000 次查询下就会立即超时。 

## 方法

 一个蛮力的想法是尝试从某个有界立方体内的当前点构造所有整数向量，并测试所得方向是否垂直于原始射线。 对于每个候选点，我们将计算方向向量并在适当的点积下验证正交性。 即使我们将坐标限制在一个合理的范围内，比如开始时的±10^6，在最坏的情况下仍然会产生大约 10^18 个候选，这是完全不可行的。 

问题的结构表明正在发生更严重的事情。 我们不是在任意 3D 空间中工作，而是在受约束的三角坐标系中工作。 在此类系统中，所有有效方向都位于一个平面内，这意味着所有有效向量共享一个隐式法线方向。 一旦我们认识到这一点，问题就简化为构造一个与两个独立方向正交的向量：给定的射线方向和平面的固定法线。 

这一观察结果将任务从搜索问题转变为直接线性代数构造。 嵌入 3D 的平面中的垂直方向可以通过与平面法线的叉积获得。 由于法线是固定且简单的，我们可以在常数时间内构造出所需的方向。

 ## 算法演练

 ### ## 算法演练

1. 计算给定射线的方向向量作为第二点和第一点之间的差。 该向量表示我们需要正交的几何方向。 
2. 识别坐标系的结构约束：所有有效的运动方向都位于嵌入 3D 空间的固定平面内。 这意味着该平面有一个固定的法向矢量，可以将其视为独立于输入的恒定方向。 
3. 使用叉积构造一个垂直于给定方向和平面法线的向量。 这会产生一个向量，该向量与有效方向位于同一平面内，并且也与原始射线正交。 
4. 使用这个构造的向量作为新射线的方向，并将其与起点相加，得到所需的第二个点。 
5. 如果结果向量是零向量，则构造无效，但在有效射线方向的约束下，这种简并实际上不会发生。 
6. 输出端点坐标。 如果向量及其负数都有效，则任一方向都是可接受的，因为它们距起点的距离相同。 

### 为什么它有效

 三角坐标系中所有有效的射线方向都位于 ℝ³ 的二维子空间中。 原始方向向量位于该子空间中。 与固定法向量的叉积产生一个与原始方向正交的向量，同时仍保留在同一子空间内。 这保证了坐标系的有效性和限制在平面的欧几里得几何下的垂直性。 由于构造是确定性的并且仅使用整数算术，因此它总是在边界内产生有效的整数点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    # fixed normal of triangular coordinate plane
    nx, ny, nz = 1, 1, 1

    for _ in range(t):
        x0, y0, z0 = map(int, input().split())
        x1, y1, z1 = map(int, input().split())

        dx, dy, dz = x1 - x0, y1 - y0, z1 - z0

        # cross product d × n
        vx = dy * nz - dz * ny
        vy = dz * nx - dx * nz
        vz = dx * ny - dy * nx

        # move one step in this direction
        print("DA")
        print(x0 + vx, y0 + vy, z0 + vz)

if __name__ == "__main__":
    solve()
```该代码首先计算原始射线的方向，然后使用叉积与表示三角形嵌入法线的固定向量构造垂直方向。 将此方向添加到起点将为新射线生成有效的第二个点。 每个测试用例的构造时间是恒定的，考虑到输入大小，这是必要的。 

一个微妙的实现细节是我们从不标准化或缩放方向。 这是故意的，因为问题只关心方向和整数有效性，而不关心单位长度。 任何非零标量倍数都会产生等效射线，并且保留原始整数值可以避免精度问题。

 ## 边缘情况

 当原始射线与坐标轴之一对齐时，就会出现极端情况。 在这种情况下，旋转坐标或交换分量的简单方法可能会意外地产生离开三角形平面的向量。 叉积构造完全避免了这种情况，因为它总是产生一个与射线方向和固定平面法线正交的向量，保证它保持在有效的几何结构内。 

另一种边缘情况是坐标值很大时。 由于所有操作都是输入值的线性组合，因此结果坐标保持在允许的 ±10^18 范围内，无需额外的缩放逻辑。
