 ## 解决方案
 ## 问题理解

 我们正在建模一个确定性转移过程，该过程从一堆苹果开始，然后穿过一排 m 个孩子。 第一个孩子收到所有 n 个苹果。 每个孩子都执行相同的操作：他们只吃掉一个苹果，然后尝试将剩余的苹果分成 x 个相等的堆。 其中一堆被保留，而其他 x − 1 堆则作为下一个孩子的输入向前传递。 

一个关键的约束是，在每一步中，吃完一个苹果后剩余的苹果数量必须能被 x 整除，并且最后一个孩子剩余的一堆苹果必须是正整数。 任务是确定使整个过程对所有 m 个孩子都有效的最小可能的苹果初始数量 n，并输出该最小 n 对 998244353 求模。 

该结构本质上是顺序的：每个子级通过线性算术条件（包括减一并除以 x）将输入值转换为较小的输出值。 一个步骤的输出成为下一步的输入，因此整个系统是一个递归，必须在 m 个转换中保持有效。 

约束允许 m 最大为 10^9，x 最大为 10^9，而测试用例的数量可以达到 10^3。 这立即排除了所有子级的任何模拟，因为即使每个测试用例的 O(m) 也太大了。 该解决方案必须将重复相同变换 m 次的效果压缩为封闭式表达式。 

当 x = 2 或 m 很大时，会出现微妙的边缘情况：递归增长非常快，中间值超出标准范围，因此任何依赖迭代乘法而没有模运算或没有封闭形式的方法都会失败。 另一个微妙之处是整除性必须在每一步都成立，因此即使最终表达式看起来正确，也不是每个算术序列都是有效的。 

## 方法

 如果我们尝试直接模拟，我们会从某个未知的 n 开始并重复应用变换：减一，除以 x，然后向前传播。 向前运行是不可能的，因为除非我们已经知道 n，否则我们不知道中间值。 

相反，我们逆转这个过程。 假设第 i 个孩子收到某个值 a。 吃完一个苹果后，剩下的 a − 1 必须等于传递给下一个孩子的值的 x 倍。 这意味着 a − 1 总是能被 x 整除，下一个状态是 (a − 1) / x。 这定义了反向递归：如果我们知道最后一个有效值，我们就可以重建之前的值。 

让最后一个孩子的剩余堆为 1，因为它必须是正整数并且我们想要最小的起始值。 向后工作 m 次可根据几何构造扩展数字：每个反向步骤乘以 x 并加 1。重复此 m 次可产生闭合形式的几何级数。 

这导致表达式 n = 1 + x + x^2 + ... + x^{m}。 这是一个标准的几何和，当 x ≠ 1 时，可以重写为 (x^{m+1} − 1) / (x − 1)。由于 x ≥ 2，除法在模算术中始终有效，因为 x − 1 可逆模 998244353。 

因此，问题简化为快速求幂和模逆计算。

 ## 算法演练

 ### 1. 逆向重新表述该过程

 我们在每个孩子之后定义状态。 如果子节点的值为 a，则对于某个整数 b，前一个状态必须满足 a = x * b + 1。 这种倒置是由这样的要求所强制的：减一后，余数必须均匀地分成 x 堆。 

### 2. 确定最小有效最终状态

最后一个孩子必须以正整数堆结束。 为了最小化初始值，我们将此最终值设置为 1。任何更大的选择都会严格增加所有早期重建的值。 

### 3. 反复向后展开

 我们重复应用逆变换：

 a_{k-1} = x * a_k + 1

 从 a_m = 1 开始，我们生成：

 a_{m-1} = x + 1

 a_{m-2} = x^2 + x + 1

 等等。 

该模式表明，经过 k 个反向步骤后，该值变为 x 的幂之和。 

### 4.认识几何结构

 m步后，初始值为：

 n = sum_{i=0 到 m} x^i

 这会将顺序依赖关系转换为封闭形式表达式。 

### 5. 模数下的高效计算

 我们使用快速求幂模 998244353 来计算 x^{m+1}。然后使用费马定理计算 x − 1 的模逆，因为模数是素数。 最后将它们组合成：

 n = (x^{m+1} − 1) * inv(x − 1)

 ### 为什么它有效

 一旦最终状态固定，反向递归唯一地确定所有先前的状态。 由于每个步骤都是仿射的（线性函数加常数），因此重复组合会产生几何级数。 每个有效的前向配置恰好对应于一次后向重建，因此选择最小最终值会强制执行最小初始值。 模运算保留了相等性，因为所有变换都是线性的并且模数是素数，确保 x − 1 存在倒数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    T = int(input())
    for _ in range(T):
        m, x = map(int, input().split())

        if x == 1:
            # degenerate case: sum of 1 repeated (m+1) times
            print((m + 1) % MOD)
            continue

        # geometric sum: (x^(m+1) - 1) / (x - 1)
        num = modpow(x, m + 1) - 1
        num %= MOD

        den_inv = modpow(x - 1, MOD - 2)
        ans = num * den_inv % MOD
        print(ans)

if __name__ == "__main__":
    solve()
```代码直接实现了封闭式推导。 快速幂函数以对数时间计算幂，这是必要的，因为 m 可达 10^9。 模逆使用费马定理，因为 998244353 是素数。 

特殊情况 x = 1 由于几何公式退化而单独处理； 总和变为 m + 1，因为每一项都是 1。

MOD = 998244353

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1

def solve():
    input = sys.stdin.readline

    for _ in range(T):

        solve()
    sys.stdin = old_stdin
    return out.getvalue().strip()

 ## 边缘情况

 当 x = 1 时，每一步变为 − 1 = 1 * next_value，因此每一步只需减一。 从最后一个值 1 开始，反转 m 次，得到 n = m + 1。该实现显式处理这种情况，因为几何公式会尝试除以 0 模 MOD。 

当 m = 0 时，实际上没有变换。 唯一有效的值为 n = 1，因为最后一个子级也是第一个子级。 

对于 x = 2 且 m 较大，值呈指数增长，但模幂可防止溢出。 递推在模数下是稳定的，因为所有运算都是线性的，并且通过求幂而不是迭代来处理。
