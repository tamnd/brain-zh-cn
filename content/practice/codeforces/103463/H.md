 ## 解决方案
 ## 问题理解

 我们得到一个初始文本缓冲区，其唯一相关属性是其长度，称之为 x。 从这个起始状态开始，我们希望使用一组键盘操作来转换缓冲区，使其长度恰好变为 n。 每次操作花费一个单位时间，目标是最小化总成本。 

可用操作的行为类似于典型的文本编辑基元，但在选择的工作方式上有不寻常的变化。 键入一个字符会附加一个符号并将长度增加一。 退格键从末尾删除一个字符，或者如果选择了某些内容，则删除整个选择。 Ctrl+A 选择缓冲区中当前的所有内容，Ctrl+C 将所选内容复制到剪贴板中，Ctrl+V 将剪贴板内容追加到末尾而不影响现有缓冲区。 一个关键的细节是，选择不会替换或删除内容，它只会标记内容，因此选择后的操作永远不会覆盖任何内容。 

问题归结为控制缓冲区长度。 实际的字符是无关紧要的，因为一旦我们决定选择或复制什么，每个操作都会确定性地改变长度。 

这些约束允许 x 和 n 最多为一百万，这排除了任何尝试显式模拟操作序列的解决方案。 所有长度上的线性或近线性动态规划方法都是可行的，而任何关于 n 的二次方则不可行。 

当初始长度大于目标长度时，会出现微妙的边缘情况。 人们可能会认为只需要退格操作，但也可以选择使用 Ctrl+A 后按退格键一次性删除所有内容。 例如，如果x为10，n为3，则一一删除需要7次操作，而全选和删除则需要3次操作，但是我们仍然需要从零开始重建。 这种交互使得有必要同时考虑部分删除和完全重置策略。 

另一个极端情况来自于对 Ctrl+A + 键入或粘贴的误解。 由于选择不会替换内容，因此这些操作总是附加，这意味着它们永远不会减少长度，并且不能用于“就地编辑”。 任何假设覆盖行为的解决方案都会计算出不正确的转换。 

## 方法

 思考该问题的一种强力方法是将每个可能的缓冲区长度建模为一个状态并模拟所有有效操作。 从长度 i 开始，我们可以通过键入移动到 i+1，通过退格键移动到 i-1，通过全选和删除移动到 0，通过复制和粘贴移动到 i 的倍数。 在这些状态上运行像 Dijkstra 这样的最短路径算法是正确的，但图又大又密。 每个节点可以生成到 n 以内的所有倍数的转换，这导致大约 n log n 条边，并且优先级队列操作的恒定开销使这种方法处于边缘状态，但在理论上仍然可行。 

简化一切的关键观察是，最佳行为在有用的意义上是单调的。 一旦我们决定使用某个长度 i 的复制操作，最好的策略就是重复粘贴，直到我们不再从该块大小中受益，因为每次粘贴都会为单位成本提供固定的 i 长度增益。 这将复制过程压缩为从 i 到 i 的任意倍数的单个转换。 

这将问题转化为 0 到 n 整数上的结构化动态规划系统，其中转换是简单的增量和乘法跳转。 我们避免对操作序列进行显式模拟，而是计算构建每个长度的最便宜的方法。 

在增长阶段内可以忽略退格转换，因为减少长度和重建永远不会比直接选择更好的构造点或在需要时在开始时使用一次重置操作更好。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有操作的完整最短路径 | O(n log n) | O(n log n) | O(n) | 已接受但沉重|
 | 具有乘法转换的动态规划 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们首先分开起始长度的问题。 如果初始长度大于目标长度，减少长度的唯一有用方法是逐个删除字符或使用 Ctrl+A 和 Backspace 重置整个缓冲区。 我们独立计算该选择，并将问题简化为从不大于 n 的起始长度进行构建。 

之后，我们将该过程视为构建从初始位置 x 到 n 的长度。 

1. 我们初始化一个 dp 数组，其中 dp[i] 表示从起始长度 x 达到长度 i 所需的最少操作次数。 
2. 我们将 dp[x] 设置为零，因为我们已经从该长度开始而无需花费任何操作。 
3. 我们按升序从 x 到 n 迭代 i，将每个可达长度视为潜在的构造基础。 
4. 从每个 i 开始，我们总是考虑通过键入一个字符来扩展字符串，这会产生 i+1，成本为 dp[i] + 1。这表示后备增量构造。 
5.从相同的状态i，我们考虑使用复制和粘贴。 复制需要全选并复制，需要2次操作，然后每次粘贴追加i个字符一次操作。 如果我们粘贴 k 次，我们就会达到长度 (k+1)·i，成本为 dp[i] + 2 + k。 这意味着可以通过成本 dp[i] + j/i + 1 达到 i 的任意倍数 j。 
6. 对于每个有效倍数 j = 2i、3i，依此类推直至 n，如果它提高了当前最佳值，我们就用计算出的成本更新 dp[j]。 

核心思想是，每次我们决定使用复制粘贴时，我们都会承诺块大小 i，并且该块的最佳使用始终是连续粘贴，直到达到所需的倍数。 

为什么它有效是基于最佳结构的结构。 任何使用某个长度 i 的复制粘贴，然后在充分利用该副本之前中断其他操作的序列，严格来说比立即完成乘法或延迟复制操作直到更有利的长度更糟糕。 这确保了每个有用的复制粘贴操作完全对应于从 i 到 i 的倍数的跳转，并且所有其他转换都通过键入或选择不同的基点来控制。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

def solve():
    x, n = map(int, input().split())

    if x == n:
        print(0)
        return

    if x > n:
        # option 1: delete one by one
        option1 = x - n

        # option 2: reset then build from 0
        dp = [INF] * (n + 1)
        dp[0] = 0

        for i in range(n + 1):
            if i + 1 <= n:
                dp[i + 1] = min(dp[i + 1], dp[i] + 1)

            if i > 0:
                dp[0] = min(dp[0], dp[i] + 3)

            for j in range(2 * i, n + 1, i):
                dp[j] = min(dp[j], dp[i] + (j // i) + 1)

        option2 = 3 + dp[n]
        print(min(option1, option2))
        return

    dp = [INF] * (n + 1)
    dp[x] = 0

    for i in range(x, n + 1):
        if i + 1 <= n:
            dp[i + 1] = min(dp[i + 1], dp[i] + 1)

        for j in range(2 * i, n + 1, i):
            dp[j] = min(dp[j], dp[i] + (j // i) + 1)

    print(dp[n])

solve()
```该解决方案是围绕单个长度动态编程数组构建的。 增量转换处理始终可用的类型操作，即使不使用乘法跳转，也保证所有值的可达性。 

乘法循环将复制、粘贴和重复粘贴编码为一种结构化松弛。 表达式 j // i 给出了结果块的数量，从该数量中减去一个粘贴即可产生正确的成本项。 

x > n 的情况单独处理，因为主 DP 只允许增加长度。 在这种情况下，我们将直接删除与重置策略进行比较，重置策略移动到零并以最佳方式重建字符串。

 考虑 x 较小且 n 稍大的输入，例如 x = 1 且 n = 4。

 ## 边缘情况

 当初始长度已经与目标匹配时，算法立即返回零，因为没有操作改善状态。 

当初始长度超过目标时，将明确比较两种竞争策略。 第一个逐步减少长度，第二个完全重置缓冲区并从零开始重建。 DP 确保从零开始的重建成本最小，并且比较捕获了擦除所有内容比部分删除更便宜的情况。 

当目标是可达中间长度的倍数时，乘法转换正确地捕获了复制粘贴链的优势。 DP 保证一旦达到最佳长度，其所有倍数都会通过一致松弛继承该最佳结构。
