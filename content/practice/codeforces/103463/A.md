 ## 解决方案
 ## 问题理解

 给定一组从 0 到 n 的数字，并要求我们考虑这些数字的所有可能排列。 通过按顺序连接数字，每个排列都被解释为一个数字。 但是，任何以零开头的排列都会被丢弃，因为它会产生一个带有前导零的数字。 

在所有有效排列中，我们计算有多少个结果数字可以被给定整数 m 整除。 

因此，核心任务不是直接构造数字，而是在两个约束下计算一小组数字的排列：无前导零和形成的数字可被 m 整除。 

约束很小：n 至多 15，m 至多 100。这立即表明阶乘规模枚举高达 16 阶乘是不可能显式执行的，因为 (15 + 1)! 已经是天文数字了。 任何解决方案都必须避免直接生成排列。 

前导零规则产生了微妙的边缘情况。 例如，如果 n = 2，则数字为 {0,1,2}。 像 012 或 021 这样的排列是无效的，即使它们是所有数字的排列。 仅计算第一个数字非零的排列。 简单的排列生成器可能会先对所有排列进行计数，然后进行过滤，但这会浪费无效状态上的计算，并且仍然无法扩展。 

另一个重要的边缘情况是当 n = 0 时。在这种情况下，唯一的数字就是 0 本身，整除性取决于是否 0 % m = 0，这总是正确的。 任何正确的解决方案都必须确保它不会意外地丢弃这个微不足道的案例。 

## 方法

 强力解决方案将生成从 0 到 n 的每个数字排列，构造相应的整数，并测试可被 m 整除。 这在概念上很简单：我们尝试所有可能的排序，跳过从零开始的排序，并计算有效的排序。 

问题是排列的数量随着位数的阶乘而增长。 n 到 15 为止，总共有 16 位数字，结果是 16！ 排列，这远远超出了任何可行的计算。 即使每次检查都是恒定时间，枚举本身也是不可能的。 

关键的观察是我们不需要显式地构造完整的数字。 当我们逐位构建数字时，我们只需要跟踪模 m 的余数。 如果我们知道当前的余数以及已使用哪些数字，我们可以通过添加任何未使用的数字来扩展数字，并使用模运算有效地更新余数。 

这将问题转化为对数字子集和余数值的状态空间搜索。 每个状态代表一个部分排列，由所用数字的位掩码和当前余数模 m 编码。 由于最多有 2^(n+1) 个子集和 m 个可能的余数，因此状态总数最多约为 2^16 × 100，这很容易管理。 

然后，我们对子集使用动态编程，增量构建排列并累积有效完成的计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O((n+1)!) | O(n) | 太慢了 |
 | 位掩码 DP | O(2^n·n·m) | O(2^n·n·m) | O(2^n·m) | O(2^n·m) | 已接受 |

 ## 算法演练

 我们将 0 到 n 的每个数字视为可以放置在排列中的元素。 我们使用子集上的动态规划来构建所有有效的排列。

1. 用由位掩码和模 m 余数组成的对来表示每个状态。 位掩码指示哪些数字已被使用，余数跟踪迄今为止形成的模 m 的数字的值。 
2. 用 dp[0][0] = 1 初始化 DP 表，这意味着有一种方法可以形成余数为 0 的空数。 
3. 对于每个状态（mask、rem），尝试附加任何未使用的数字 d。 如果数字 d 已在掩码中，则跳过它。 否则计算新的掩码和新的余数。 余数更新遵循 rem_new = (rem * 10 + d) % m。 此步骤至关重要，因为它避免显式构造大整数。 
4.我们必须确保没有数字以零开头。 通过防止第一个选择的数字为零且掩码为空的转换来强制执行此约束。 
5. 处理完所有状态后，对所有 dp[full_mask][r] 求和，其中 r = 0。这些表示使用所有数字且可被 m 整除的完整排列。 

正确性来自于这样一个不变式：dp[mask][rem] 始终存储使用 mask 中的数字构建部分排列的方法数，该部分排列产生一个余数 rem 模 m 的数字。 每次转换都保留了这种不变性，因为附加数字会更新掩码和余数，与十进制数字的工作方式保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    digits = list(range(n + 1))
    N = n + 1
    
    if N == 1:
        return 1 if 0 % m == 0 else 0

    dp = [[0] * m for _ in range(1 << N)]
    dp[0][0] = 1

    for mask in range(1 << N):
        for rem in range(m):
            cur = dp[mask][rem]
            if not cur:
                continue
            for i in range(N):
                if mask & (1 << i):
                    continue
                if mask == 0 and digits[i] == 0:
                    continue
                nmask = mask | (1 << i)
                nrem = (rem * 10 + digits[i]) % m
                dp[nmask][nrem] += cur

    full = (1 << N) - 1
    return dp[full][0]

if __name__ == "__main__":
    print(solve())
```该实现直接遵循前面描述的基于子集的 DP。 dp 表通过掩码和余数进行索引，并存储到达每个状态的方式计数。 转换循环遍历所有未使用的数字并更新掩码和余数。 

前导零限制在第一个转换级别显式处理：当掩码为零时，我们禁止选择数字 0。这确保我们永远不会生成以零开头的无效排列。 

模更新是使用增量执行的`(rem * 10 + digit) % m`，这是在模运算下维护数值而不构造大整数的标准方法。 

## 工作示例

 考虑示例输入：

 输入：```
2 3
```数字为 {0, 1, 2}。 完整排列为 102、120、201、210。我们计算那些能被 3 整除的排列。 

我们跟踪小子集的简化 DP 进展：

 | 面膜| 选择的数字| 其余状态（部分摘要）|
 | --- | --- | --- |
 | 000 | 000 {} | {0:1}|
 | 001| {0} 跳过了无效的开头 | |
 | 010| {1} | {1：1} |
 | 100 | 100 {2} | {2：1} |

 从这些部分构建中，完整排列产生余数：

 102%3=0

 120%3=0

 201%3=0

 210%3=0

 因此，所有 4 个排列都是有效的并且可以被 3 整除，所以答案是 4。 

该跟踪表明 DP 正确累积了所有有效排序，并且余数传播与全数整除性一致。 

第二个例子：

 输入：```
1 2
```数字是{0,1}。 有效排列仅为“10”（因为“01”无效）。 10 % 2 = 0，所以答案是 1。 

这会检查最小非平凡情况下的前导零约束行为。

 ## 边缘情况

 对于 n = 0 且 m = 1，输入为“0 1”。 DP 减少为一位数字集 {0}。 唯一有效的排列是数字 0 本身。 该算法初始化 dp[0][0] = 1 并直接将其计为完整掩码状态，产生余数 0，与预期输出 1 匹配。 

对于 n = 1 且 m 较大的输入（例如“1 100”），数字为 {0,1}。 DP正确拒绝从0开始的排列，只考虑“10”。 余数计算为 (1 * 10 + 0) % 100 = 10，它不为零，因此结果为 0。转换规则确保了正确性，因为每个有效排列仅通过允许的第一位数字选择和一致的模块化更新来构建。
