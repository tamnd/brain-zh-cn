 ## 解决方案
 ## 问题理解

 我们正在模拟苹果穿过生产线的确定性转移过程$m$孩子们。 该过程从初始数字开始$n$，它被给予第一个孩子。 每个孩子执行相同的操作：他们恰好吃掉一个苹果，然后剩余的苹果必须均匀地分成恰好$x$相等的桩。 形成这些堆后，孩子保留其中一堆并传递其余的$x-1$堆给下一个孩子。 最后一个孩子也遵循同样的规则，只不过拿完一堆后，过程就结束了。 

关键的限制是每次孩子收到一定数量的苹果$A$，值$A-1$必须能整除$x$。 如果我们写$A-1 = x \cdot k$，然后孩子采取$k$苹果和通行证$(x-1)\cdot k$到下一个孩子。 这使得状态转换完全由乘数决定$k$，但整除性约束限制了哪些值$A$每一步都有效。 

我们被要求构建任何有效的初始$n$以便该过程可以准确地进行$m$子级，并且所有约束都得到满足，附加要求是所有中间值保持有效整数并且最终采用的堆大小为正。 答案保证存在于$10^{18}$，所以我们只需要一个建设性的解决方案。 

不明显的困难是有效性不是局部的。 为一个子项选择有效值不会自动允许前一个子项使用有效值，因为可除性条件向后传播并约束整个链。 试图“猜测”的天真前向模拟$n$将会失败，因为大多数选择很快就会违反 mod 条件。 

边缘情况立即出现$x=2$。 在这种情况下，每个步骤在减一后将剩余结构减半，这打破了许多关于可逆模的假设$x-1$。 另一个微妙的情况是当$x\ge 3$，其中结构是可逆的，但前提是我们在所有步骤中小心地保持一致的模块残基。 

## 方法

 一个蛮力的想法会尝试枚举$n$并模拟所有人的过程$m$孩子们。 每个模拟步骤都会检查是否$A-1$可以整除$x$，计算下一个状态，并在最后验证积极性。 这是正确的，但完全不可行，因为$n$可以大到$10^{18}$，并且有效值极其稀疏。 即使限制在有限的范围内，分支结构也不存在，因此蛮力会退化为对天文大空间的线性扫描。 

关键的观察结果是该过程在中间量上是线性的$k = (A-1)/x$。 每一步都有转变$k_i$进入$k_{i+1}$通过简单的仿射变换，但前提是满足特定的模条件。 我们不是向前构造，而是转换视角：我们向后构造一个有效的序列，选择最后一个孩子的状态并反向传播约束。 

这个相反的过程将问题简化为维护整数序列$k_i$这样每个转换都满足模块化约束并保持完整性。 该结构变得易于管理，因为在每一步我们都可以通过选择适当的倍数来强制执行所需的一致性，并且增长保持受控，因为$m \le 15$。

 ## 算法演练

 我们使用中间变量重写该过程$k_i = (A_i - 1)/x$。 然后：$$A_i = xk_i + 1, \quad A_{i+1} = (x-1)k_i$$由此，我们可以推导出逆关系：$$k_{i+1} = \frac{(x-1)k_i - 1}{x}$$仅当以下情况下才有效$(x-1)k_i \equiv 1 \pmod x$。 

我们现在构建$k_m$首先并向后传播。 

1. 选择最后一个孩子的$k_m$使最后一次操作有效。 这需要$k_m \ge 1$还有那个$(x-1)k_{m-1} - 1$可以整除$x$当相反时，最容易通过强制来满足$k_m \equiv x-1 \pmod x$。 最小的这个值是$k_m = x-1$。 
2. 计算$A_m = xk_m + 1 = x(x-1) + 1$。 这可确保最后一个孩子收到有效的号码并可以执行所需的拆分。 
3.从child开始向后移动$i+1$给孩子$i$通过强制执行：$$k_{i+1} = x - 2 + (x-1)t$$在哪里$t$选择这样的方式使得下一步的模块化约束保持有效。 我们总是选择最小的$t$保留不变式$k_i \equiv x-1 \pmod x$。 
4. 重复这个向后扩展$m-1$次。 每一步都会产生一个有效的$k_i$，因此有效的$A_i$，不破坏整除性。 
5.到达后$k_1$， 输出$n = A_1 = xk_1 + 1$。 

### 为什么它有效

 整个构造基于维护一个不变量：每个$k_i$被选择使得$(x-1)k_i \equiv 1 \pmod x$，这保证了下一个反向步骤产生一个整数值。 通过在每个阶段强制执行这种一致性，我们确保每个部门$x$前向过程是精确的，每个孩子都会收到有效的整数个苹果。 由于每个后向步骤显式构造一个满足相同残差类别的值，因此任何步骤都不能违反所需的整除条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    m, x = map(int, input().split())

    if x == 2:
        # special linear chain
        # k_m must be odd, minimal is 1
        A = 3  # works for all m
        # propagate backward: A_i = 2*A_{i+1} + 1
        for _ in range(m - 1):
            A = 2 * A + 1
        print(A)
        return

    # general case x >= 3
    # work with k values
    k = x - 1  # minimal valid choice for last step

    # build backwards
    for _ in range(m - 1):
        # enforce next valid k in reverse construction
        # k_new = x - 2 + (x - 1) * t, choose minimal t = x - 1
        k = (x - 2) + (x - 1) * (x - 1) + (x - 1) * k

    n = x * k + 1
    print(n)

if __name__ == "__main__":
    solve()
```该解决方案干净利落地分为特殊情况$x=2$，其中变换变成简单的加倍递归，并且一般情况$x \ge 3$，其中反向构造依赖于维护稳定的模块化类$k$。 循环内的表达式是重复选择保留所需同余性的最小有效参数的扩展形式。 

一个常见的错误是尝试从$n$。 可分性约束看起来是局部的，但实际上强制了一个全局一致性条件，只有在向后工作时才变得容易$k$。 

## 工作示例

 ### 示例 1：$m = 2, x = 3$我们从$k_2 = x - 1 = 2$， 所以：$$A_2 = 3 \cdot 2 + 1 = 7$$现在我们向后移动一次：

 | 步骤|$k_i$|$A_i = xk_i + 1$|
 | --- | --- | --- |
 | 2 | 2 | 7 |
 | 1 | 4 | 13 |

 因此$n = 13$。 

这表明逆向构造如何自然地产生有效的起点而无需猜测$n$直接地。 

### 示例 2：$m = 3, x = 4$从以下开始：$$k_3 = 3$$计算：$$A_3 = 13$$后退一步：

 | 步骤|$k_i$|$A_i$|
 | --- | --- | --- |
 | 3 | 3 | 13 |
 | 2 | 15 | 15 61 | 61
 | 1 | 63 | 63 253 | 253

 所以$n = 253$。 

每个步骤都保留了所需的模块化条件，我们看到受控的增长仍然远低于$10^{18}$。

 ## 边缘情况

 ###案例$x = 2$什么时候$x=2$，一般模反演会失败，因为$x-1 = 1$，消除了一般构造中使用的自由度。 该过程简化为严格的线性递归：$$A_{i+1} = A_i + (A_i - 1) = 2A_i - 1$$所以向前或向后工作都是安全的。 实现通过直接迭代递归来明确处理这个问题。 

＃＃＃ 小的$m = 1$只有一个孩子，不需要繁殖。 任何$n$这样$n-1$可以整除$x$和$(n-1)/x \ge 1$是有效的。 建筑自然满足这一点，因为我们始终确保$k_1 \ge 1$。 

### 大型增长案例

 尽管中间值增长很快，但边界$10^{18}$由于深度极小，因此不会超出$m \le 15$。 每一步都是仿射变换，因此增长是指数级的$m$，但仍在限制之内。
