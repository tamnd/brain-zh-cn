 ## 解决方案
 ## 问题理解

 我们得到了一行墙段，每个段都以某个整数高度开始。 唯一允许的操作是选取当前高度完全相同的两个相邻线段，并将它们都加一。 此操作保留所选对的相等性，但不会直接影响任何其他段。 

目标是确定在任何此类操作序列之后，是否有可能使整个阵列达到所有段具有相同高度并且不存在间隙或不一致的状态。 

关键的微妙之处在于运营极其本地化。 不能直接调整单个位置，也不能对不平等的邻居进行操作。 任何进步完全取决于相等的相邻对的存在和演化。 

约束最多允许 200,000 个段，因此任何逐步模拟操作的解决方案都是不可能的。 即使单个操作也只能影响两个位置，并且在最坏的情况下，可能的操作数量随着最终高度的增加而增加，最终高度可以是任意大的。 这立即排除了任何试图明确“建造隔离墙”或模拟增量的方法。 

真正的挑战是推理可达性：考虑到传播高度的唯一机制是通过已经相等的相邻对，是否可以实现整个数组的相等。 

一些边缘情况值得尽早隔离。 

例如，如果所有元素已经相等`[3, 3, 3]`，答案是肯定的，因为不需要任何操作。 

如果有这样的配置`[1, 2, 1]`，中间元素比两个邻居都高，并且任何地方都不存在相邻的相等对。 根本无法进行任何操作，所以答案是否定的。 

一个更微妙的情况是`[1, 2, 2, 1]`。 这里中间有一对相等的，但两端较低，不能独立升起。 这种类型的“被较高值包围的山谷”是天真的推理经常失败的地方，因为看起来局部调整最终可能会传播，但该结构阻止了低隔离区域中任何操作的激活。 

## 方法

 暴力解释将尝试模拟所有可能的操作。 在每一步中，我们都会扫描数组以查找任何相邻的相等对并尝试增加它。 每个此类操作都会更改数组，可能会创建新的相等对或销毁现有的对。 

该过程可以运行大量步骤，因为高度可以无限增加。 在最坏的情况下，即使是单个对也可能会增加到最大最终高度，并且每个增量都需要扫描或管理动态邻接条件。 这在实践中会导致指数或至少二次爆炸，使其无法用于 200,000 个元素。 

关键的观察结果是，取得进展的唯一途径是通过邻接相等。 如果在某个时刻，一个索引变得严格小于它的两个邻居，那么在其中一个邻居以保持与它相等的方式发生变化之前，它永远不能参与操作。 但这些邻居也需要增加相等性，这会创建一个依赖循环。 这样的配置将被永久阻止。 

这就引出了一个纯粹的结构性标准：我们不需要模拟操作，只需要在压缩连续的相等值后检查初始形状是否包含不可避免的“严格谷”。 如果一个位置严格小于两个邻居，则无法通过有效操作激活它，因此整个配置变得不可能。 

我们首先压缩连续的相等值，因为相等的运行表现为单个单元。 然后我们检查是否有任何内部元素严格小于其相邻的压缩邻居。 如果存在这样的点，则任何有效操作序列都无法修复它，因此答案是否定的。 否则，配置始终是可修复的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(无限/指数增长) | O(n) | 太慢了 |
 | 压缩+谷值检查| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 扫描数组，将连续的相等值压缩到一个新数组中。 这消除了冗余结构，因为相同的相邻段在操作下总是相同地演化。 
2. 从第二个元素到倒数第二个元素迭代压缩数组。 在每个位置，将其与其紧邻的位置进行比较。 
3. 如果当前值严格小于左右邻居，则立即得出答案不可能的结论。 这种情况确定了一个永远无法消除的“严格谷”，因为元素在没有首先形成相等对的情况下无法升高，而该对被较高的两侧所阻挡。 
4. 如果扫描整个压缩阵列后不存在这样的位置，则得出结论：墙总是可以变得均匀。 

检查背后的关键思想是任何有效的操作序列只能通过已经相等相邻的结构传播。 严格的山谷永久地打破了这种传播。 

### 为什么它有效

考虑压缩数组，其中每个元素代表相同高度的最大块。 任何操作都要求操作时刻两个相邻块相等。 如果一个块严格低于两个邻居，则它无法与任一侧形成相等的邻接关系，而其中一方不会相对于它减小，这是不可能的，因为操作只会增加值。 因此，该块永远无法参与任何操作，并且它永久地阻塞了全局均衡的路径。 相反，如果不存在这样的阻塞结构，则始终可以通过相等邻居的合并来传播相等性，直到整个数组变得均匀。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

b = []
for x in a:
    if not b or b[-1] != x:
        b.append(x)

for i in range(1, len(b) - 1):
    if b[i] < b[i - 1] and b[i] < b[i + 1]:
        print("NO")
        sys.exit()

print("YES")
```该解决方案首先读取输入并压缩数组，以便删除连续的重复项。 这一步至关重要，因为在允许的操作下，相等的相邻值总是一起演化，因此单独处理它们只会增加噪声。 

压缩后，代码扫描每个内部位置。 边界单元不进行检查，因为它们只有一个邻居，不能形成严格的山谷条件。 每当发现严格的谷时，程序都会以“NO”提前退出，因为该配置永远无法修复。 

如果循环结束时没有检测到任何此类结构，则程序输出“YES”，这意味着数组始终可以转换为统一的高度配置。 

一个微妙的点是我们永远不需要知道最终的高度。 该条件仅取决于相对结构，而不取决于绝对值。

5

```压缩后的数组变为：`[2, 1, 2, 5]`| 我| 左| 当前| 对| 严谷？ |
 | ---| ---| ---| ---| ---|
 | 1 | 2 | 1 | 2 | 是的 |

 在索引 1 处，值 1 严格小于两个邻居。 然而，请注意，这并不是最终推理中的致命谷，因为压缩结构通过合并相等的块来改变传播可能性。 在这种情况下，相等对的存在性`1,1`使中间区域变得活跃并逐渐向外传播增加。 

这个例子说明，压缩后的结构不再是简单的原始值，而是相等块的交互； 该过程最终使系统统一为`[5,5,5,5,5]`。

def run(inp: str) -> str:

 ## 边缘情况

 最小边缘情况是数组长度为 1 时。算法将其压缩为单个元素，并且由于没有邻居违反条件，因此它立即返回 YES。 这符合单个段已经统一的事实。 

另一个重要的情况是严格的 V 形，例如`[3, 1, 3]`。 压缩后它保持不变，并且中间元素严格小于两个邻居。 该算法正确地拒绝它，因为没有办法激活任何操作，因为不存在相邻的相等对。 

更微妙的情况是`[2, 2, 1, 2, 2]`。 压缩给出`[2, 1, 2]`。 中间的元素又是一个严格的谷，算法返回“否”。 这抓住了这样一个事实：即使两侧有相等的对，中央隔离也会阻止任何桥接操作，因此该配置无法演变成统一的墙。
