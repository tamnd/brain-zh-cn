 ## 解决方案
 ## 问题理解

 我们得到一个代表学生技能水平的整数数组。 学生必须准确配对$\frac{n}{2}$不相交的对，因此每个学生都只使用一次。 仅当其中的两个学生最终具有完全相同的技能水平时，一对才“有效”。 由于技能只能增加（不能减少），因此我们可以对个别学生进行任意多次的增加，每次增加支付一个单位成本。 

任务是选择配对结构和增量，以便每对都包含相等的值，同时最小化所有学生执行的增量总数。 

从约束中观察到的关键是$n \le 100$，因此即使是对或排序的二次或三次推理也是完全安全的。 我们不担心超越的渐近优化$O(n^2)$，这表明我们应该关注贪婪或配对策略，而不是任何复杂的数据结构或大状态上的 DP。 

天真的思维的一个微妙的失败案例是首先尝试匹配相等的值，然后贪婪地修复剩余的值。 例如，如果价值严重不平衡，尽早修复本地匹配可能会迫使未来进行昂贵的调整。 另一个陷阱是没有排序或结构的配对，这很容易错过全局最佳配对。 

例如，如果我们采取`1 100 2 3`，像这样的天真的配对`(1,100)`和`(2,3)`结构看似合理，但成本巨大。 更好的全球配对是`(1,2)`和`(3,100)`这可以更有效地传播大的价值调整。 

这表明排序至关重要。 

## 方法

 暴力方法将尝试所有可能的方法将学生配对，并为每个配对计算使每个配对相等的成本。 由于有$(n-1)!!$配对方式$n$项目，增长速度非常快。 即使是为了$n = 20$，这变得不可行。 每个配对仍然需要计算均衡对的成本，因此即使成本计算很快，枚举仍占主导地位。 

关键的见解是对数组进行排序。 一旦排序，最佳配对就变得结构化：我们将相邻元素配对。 这是有效的，因为将两个接近的值配对可以最大限度地减少提高较小值以匹配较大值的成本，并且排序可确保我们避免在存在更好的本地替代方案时将较小的值与非常大的值配对。 

排序后，我们只需要考虑pair$(a_0, a_1), (a_2, a_3), \dots$。 对于每一对，成本很简单$a_{i+1} - a_i$，因为我们只增加较小的元素。 

这将问题从组合配对问题简化为对排序数组的线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力配对枚举|$O(n!)$|$O(n)$| 太慢了 |
 | 排序+贪婪相邻配对 |$O(n \log n)$|$O(1)$或者$O(n)$| 已接受 |

 ## 算法演练

 1. 将所有学生技能值读入数组。 目标是最小化形成相等对所需的总增量。 
2. 按非降序对数组进行排序。 此步骤可确保数值上接近的值成为相邻值，这对于最小化调整成本至关重要。 
3. 初始化变量`answer = 0`累积所需增量的总数。 
4. 考虑连续对，以两步迭代已排序的数组。 
5. 对于每对`(a[i], a[i+1])`，将成本计算为`a[i+1] - a[i]`。 这表示增加较小的值以匹配较大的值。 
6. 将此成本添加到运行总计中。 
7. 输出最终累计成本。 

### 为什么它有效

 排序后，任何最优解都可以转化为相邻元素之间进行配对的解，而不会增加成本。 如果两对交叉，例如配对`(a_i, a_k)`和`(a_j, a_l)`和$i < j < k < l$，将它们交换成`(a_i, a_j)`和`(a_k, a_l)`永远不会增加总成本，因为它减少了对之间的巨大差距。 重复应用此参数可以消除所有交叉，从而使相邻配对达到最佳状态。 

这表明贪婪的相邻配对不仅方便，而且在结构上是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

a.sort()

ans = 0
for i in range(0, n, 2):
    ans += a[i+1] - a[i]

print(ans)
```该解决方案首先读取输入并对技能数组进行排序。 排序至关重要，因为它创建了一种结构，使最佳配对成为局部的。 

该循环以两个块为一组处理数组。 每次迭代都会将两个相邻的排序值配对，这保证了在全局最优结构下该对的成本最小。 差值是直接相加的，因为只允许增量，所以我们总是提高较小的值来匹配较大的值。 

不存在边界问题，因为$n$甚至可以保证，所以`a[i+1]`始终有效。

import sys, io

 ## 边缘情况

 一种边缘情况是所有值都已经相等。 该算法对它们进行排序，但每个相邻的差异都为零，因此结果仍然为零。 用于输入`4 / 5 5 5 5`，排序后的数组不变，所有计算出的差异消失。 

另一种边缘情况是当值高度倾斜时，例如`1 1 1 100`。 排序后，成对变成`(1,1)`和`(1,100)`。 第一个贡献零成本，第二个贡献 99，这是不可避免的，因为只存在一个大值。 

最终的边缘情况是可能的最小输入`n = 2`。 该算法只是返回排序后两个值之间的差异，直接匹配唯一有效的配对。
