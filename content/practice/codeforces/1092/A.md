 ## 解决方案
 ## 问题理解

 我们被要求为每个查询构造一个字符串，使得该字符串具有固定的长度`n`并且仅使用第一个`k`小写拉丁字母，来自`'a'`直到`k`第-个字母。 这些中的每一项`k`字母必须在最终字符串中至少出现一次。 在所有有效的结构中，我们希望最大化字符串中使用的任何字母的最小频率。 

换句话说，我们正在分发`n`之间的位置`k`不同的符号。 每个符号必须至少出现一次，我们希望尽可能实现最平衡的分布，其中最小的桶尽可能大。 

限制很小：`n ≤ 100`,`k ≤ 26`，直到`t ≤ 100`查询。 这立即排除了对高级数据结构或优化技术的任何需要。 每个查询以线性甚至二次时间运行的任何解决方案都足够了。 该结构提出了一种建设性的解决方案，而不是搜索或动态规划。 

一个关键的微妙点是理解“最大化最小频率”意味着什么。 如果我们分配计数`c1, c2, ..., ck`，我们正在最大化`min(ci)`在大家的约束下`ci ≥ 1`和总和为`n`。 这是一个纯粹的平衡问题。 

边缘情况出现时`n`仅略大于`k`。 例如，如果`n = k`，那么每个字符必须恰好出现一次，因为我们已经需要所有字母并且没有多余的空间。 任何“进一步平衡”的尝试都是不可能的。 

另一个边缘情况是当`n`远大于`k`。 一种总是循环附加下一个字母的天真的贪婪方法是有效的，但如果不考虑频率平衡，人们可能会错误地认为需要更复杂的重新排列。 

## 方法

 暴力方法会尝试分发`n`之间的位置`k`以所有可能的方式字母，使得每个字母至少出现一次。 对于每个分布，我们将计算最小频率并跟踪最佳频率。 这种分布的数量相当于`n`进入`k`正整数，呈指数增长`n`。 即使是为了`n = 100`，这变得完全不可行。 

目标函数的结构使问题变得更加简单。 由于我们只关心最大化最小计数，因此最好的解决方案会尝试尽可能均衡所有频率。 如果每个字母都准确无误`n / k`发生，这将是理想的，但整数除法会妨碍完美的相等。 因此，最佳最小频率取决于有多少个完整的块大小`k`适合`n`。 

一旦我们将目标解释为平等分配，构建就变得简单了。 我们在第一个循环中以循环方式重复分配字母`k`人物。 这确保了每个前缀`k`字符是完美平衡的，任何剩余部分都从头开始分配，保持最佳的最小频率。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | n 中的指数 | O(k) | 太慢了 |
 | 最佳施工| 每个查询 O(n) | O(n) | 已接受 |

 ## 算法演练

 我们为每个查询独立构建字符串。 

1. 初始化一个空结果字符串。 
2. 迭代自`0`到`n - 1`。 
3. 对于每个位置`i`, 追加对应的字符`(i mod k)`。 

这确保我们循环通过第一个`k`反复写信。 
4. 循环结束后，输出构造好的字符串。 

步骤 3 背后的原因是循环保证了统一使用。 每一块`k`连续位置恰好包含每个允许字符的一次出现。 这可以防止任何角色在频率上落后于其他角色。 

### 为什么它有效

 每个完整周期的长度`k`每个字符只出现一次`'a'`到`'a' + k - 1`。 后`⌊n / k⌋`完整周期，所有字符具有相同的基频。 任何剩余的`n mod k`位置被分配给前几个字符，最多将其频率增加一个。 这保证了任意两个字符频率之间的差异最多为1，这是可能的最佳平衡。 因此，最小频率被最大化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n, k = map(int, input().split())
    
    res = []
    for i in range(n):
        res.append(chr(ord('a') + (i % k)))
    
    print("".join(res))
```实现直接遵循循环构造。 名单`res`为了提高效率，使用而不是字符串连接，尽管考虑到约束，任何一个都会通过。 

关键细节是模运算`(i % k)`，这确保我们始终保持在允许的字母范围内。 映射`chr(ord('a') + x)`将整数偏移量转换为小写字母。 

对于像这样的边缘情况不需要特殊处理`n = k`或者`k = 1`，因为同一循环在两种情况下自然会产生有效的输出。

## 测试用例```python
import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n, k = map(int, input().split())
        res = []
        for i in range(n):
            res.append(chr(ord('a') + (i % k)))
        out.append("".join(res))
    return "\n".join(out)

def run(inp: str) -> str:
    return solve(inp)

# provided samples
assert run("3\n7 3\n4 4\n6 2\n") == "abcabca\nabcd\nababab"

# custom cases
assert run("1\n1 1\n") == "a", "single character"
assert run("1\n5 1\n") == "aaaaa", "single letter repeated"
assert run("1\n5 5\n") == "abcde", "each letter once"
assert run("1\n10 3\n") == "abcabcabca", "repeating cycle"

 ## 边缘情况

 当`n = k`，循环只产生每个字符一次，因为`(i % k)`涵盖了所有值`0`到`k - 1`正好一次。 例如，`n = 4, k = 4`产生`abcd`，它已经满足每个字母至少出现一次的要求，除此之外不可能进行平衡。 

什么时候`k = 1`，每个位置映射到`'a'`。 模运算始终为零，因此输出是一串`n` `'a'`人物。 这是最佳选择，因为只允许使用一个字母。 

什么时候`n`不能被整除`k`，其余字符只是增加前几个字母的频率。 例如，`n = 7, k = 3`产生`a, b, c, a, b, c, a`，给出最多相差一的计数。 这证实了无法实现更好的最小频率，因为任何重新分配都会迫使至少一个角色下降到这一层以下。
