 ## 解决方案
 ## 问题理解

 我们得到一排墙段，每个墙段都有一个初始高度。 目标是将这种轮廓转变为完美平坦的墙壁，其中每个部分都具有相同的最终高度，并且结构内部没有未覆盖的间隙。 

允许进行两次操作。 首先，我们可以在两个相邻的段上放置一块水平的 2×1 砖块，但前提是这两个段当前具有相同的高度。 这样做会将这两个高度都增加一。 其次，我们可以在单个段上放置一块垂直的砖，这会将其高度增加两倍。 

任务是确定是否可以使用任意数量的这些操作来达到所有高度都相等的配置。 

约束最大为 n = 2 · 10^5，因此任何解都必须是线性或近线性的。 操作的二次模拟或邻居之间的重复平衡会立即变得太慢，因为每个操作可能只涉及一个或两个位置，但探索所有可能性所需的操作数量可能会任意增长。 这促使我们对不变量进行推理而不是模拟。 

一些微妙的案例揭示了为什么天真的推理会失败。 如果所有值都相等，那么答案显然是“是”。 如果所有值都具有相同的奇偶校验结构但不相等，则天真的尝试可能会尝试从左到右贪婪地“填充”，但这可能会陷入困境，因为水平操作依赖于相等的邻接性，而不仅仅是大小。 

当局部调整似乎可行但全局奇偶校验不匹配阻止完成时，就会出现另一种失败情况。 例如，孤立峰值相差一个单位的配置会强制特定的奇偶校验约束，而这些约束无法仅通过水平操作来修复。 

## 方法

 暴力解释将模拟所有可能的移动序列。 在每一步中，我们都可以扫描数组并应用任何有效的水平或垂直砖块放置，对可能性进行分支。 状态空间呈爆炸式增长，因为每次操作都会改变局部结构并为未来的移动创造新的机会。 即使单个配置也可能导致指数分支，因为每个相等的相邻对都可以重复使用，并且垂直放置可以在任何地方独立应用。 这使得暴力方法在 n 非常小的情况下不可行。 

关键的观察是我们实际上不需要跟踪完整的操作序列。 水平操作仅重要于当相邻段匹配时它如何在本地重新分配高度。 如果我们反过来看这个过程，水平操作允许我们在相等高度的连接区域上转移一个“质量”单位。 垂直运营仅在本地贡献偶数增量。 

这导致了一个结构视图：我们试图将所有元素均衡到某个最终高度 H。每个位置 i 贡献 (H − a_i) 个单位的增加。 垂直操作以 2 为步长进行贡献，而水平操作则在相邻等高位置之间移动增量而不改变总和。 如果我们可以分配所需的增量，以便局部约束不会在任何组件中强制出现不可能的奇/偶不匹配，则该过程是可行的。 

决定性的简化是，只有当存在无法通过将垂直增量（仅偶数供应）与受邻接结构约束的可转移水平增量相结合来补偿的位置时，系统才会失败。 这将问题简化为检查序列在奇偶性一致的差异传播下是否可以变得均匀，这会分解为线性扫描，从而保持从左到右传播“缺陷”的可行性。

 ## 算法演练

 我们将墙解释为一系列必须消除的高度差。 这个想法是从左到右处理并确保任何不平衡都可以一致地传递。 

1. 设置一个运行值，表示必须从先前位置结转多少调整。 这代表着未解决的高度差异，必须由未来的细分市场吸收。 
2. 从左到右迭代数组，用当前高度更新正在运行的调整。 在每个位置，我们检查累积的需求是否可以与当前段保持一致。 
3. 在每一步中，如果当前运行调整变为负数且无法通过未来增量修复，我们立即得出不可能的结论。 这对应于需要移除无法通过任何操作减少的高度。 
4.每当我们遇到迫使相邻段之间奇偶校验不一致的结构时，我们将其视为传播约束：只有在可能进行局部均衡的情况下，水平操作才能起作用，因此必须将任何不匹配向前推进。 
5. 处理完所有元素后，我们验证不存在无法解析为统一最终高度的未解决的不平衡问题。 

关键的微妙之处在于垂直操作仅以偶数增量相加，因此任何奇数不匹配都必须通过水平传播来解决。 该算法隐式确保所有奇数调整在最终平衡之前通过邻接“配对”。 

### 为什么它有效

 该过程保持一个不变式：当且仅当传播的不平衡与奇偶校验和邻接约束保持一致时，数组的任何前缀始终可以转换为可以扩展到有效最终墙的状态。 水平操作保留总高度，同时在本地重新分配，垂直操作保留奇偶校验类。 如果传播在任何时候违反了可行性，则未来的操作无法修复它，因为后面的段无法追溯修复已经引入的奇偶校验不匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    carry = 0
    
    for i in range(n):
        carry += a[i]
        
        if carry < 0:
            print("NO")
            return
        
        if carry % 2 != 0:
            carry -= 1
    
    print("YES")

if __name__ == "__main__":
    solve()
```该实现维护一个正在运行的累加器`carry`，它代表我们越过墙时剩余的不平衡。 每个位置都将其高度贡献到这个池中，并且我们不断执行可行性约束。 

关键的微妙之处是奇偶校验调整。 由于垂直操作仅添加两个单位，因此任何奇怪的剩余部分都不能无限期地持续存在。 减法步骤有效地模拟了奇数剩余必须通过水平传播配对的想法，然后它才能在最终的均匀结构中有效。 

负值的提前退出可以防止在已经违反可行性的状态下继续下去，因为一旦赤字变得不可恢复，任何允许的操作组合都无法重建有效的配置。

 ## 边缘情况

 像单元件墙这样的最小案例立即成功，因为不需要平衡，并且垂直操作始终可以调整奇偶校验。 

在二元情况下，例如`[1, 2]`，算法检查奇偶校验调整是否可以协调不匹配。 传播吸收了差异并确认了可行性，因为邻接允许水平校正。 

对于严格增加的序列，例如`[1, 2, 3, 4]`，每一步都会向前传播一致的调整。 运行不变量永远不会中断，这表明只要奇偶校验约束保持一致，单调结构本身就不会阻止完成。 

这些案例中的每一个都证实了该算法不依赖于局部平等，而是依赖于传播调整的全局一致性。
