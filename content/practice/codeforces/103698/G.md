 ## 解决方案
 ## 问题理解

 我们被给予$n$多项式，每个多项式都由其系数按升序描述。 然后我们回答$q$查询，每个查询给出一个间隔$[l, r]$。 对于每个区间，我们在概念上将其内部的所有多项式相乘，并检查所得多项式是否具有镜像系数。 

直接读取表明我们正在处理多个范围内的多项式乘法，这通常会很昂贵，因为次数可以增长并且系数可以变大。 然而，声明中给出的特殊代数性质改变了问题的整个性质：唯一重要的是每个多项式是否单独回文。 

因此，真正的隐藏任务是预处理哪些多项式是对称的，然后回答范围查询，检查区间中的所有条目是否都是对称的。 

约束条件意味着最多$10^5$多项式和$10^5$查询，总度数总计为$5 \cdot 10^5$。 任何根据查询重新计算对称性或模拟乘法的方法都是不可能的。 在最坏的情况下，即使每个查询检查多项式也会太慢，因此我们必须在线性时间内预先计算并在常数或对数时间内回答查询。 

一个微妙的边缘情况是多项式的次数为零。 根据定义，此类多项式始终是回文多项式，因为只有一个系数。 

另一个重要的边缘情况是对称性必须尊重实际程度，而不是声明的大小。 例如，如果前导系数非零但内部结构对称，则有效； 但如果尾随系数破坏对称性，则事实并非如此。 

考虑一个天真的错误：假设多项式表示为$[1, 2, 1, 0]$。 如果错误地将声明的度数视为 3 并比较包括尾随零在内的所有位置而不进行修剪，则可能会错误地对其进行分类。 正确的解释是度数是由输入决定的$k_i$，所以后面的零超出$k_i$不存在。 

## 方法

 强力解释是通过将区间内的多项式相乘来直接模拟每个查询，然后检查结果的对称性。 如果我们将两个次数最多的多项式相乘$K$，卷积成本$O(K^2)$。 在一段长度上重复此操作$n$使其在最坏情况输入的实践中有效呈指数增长，因为中间度数增长并且每个查询重新计算大型卷积。 即使使用快速乘法，这样做$10^5$查询是不可行的。 

该语句的关键观察结果完全改变了问题：当且仅当范围内的每个多项式都是回文时，乘积才是回文的。 这将复杂的代数条件分解为二进制数组上的简单逻辑“与”。 

一旦每个多项式被简化为布尔值，问题就变成了经典的范围查询：检查是否所有值都在$[l, r]$是真的。 这相当于检查范围内的最小值是否为1，或者范围内是否存在0。 前缀和或线段树就足够了。 

我们预先计算一个数组$a[i]$在哪里$a[i] = 1$如果多项式$i$是回文，否则为 0。然后构建前缀和，以便范围和告诉我们该段中是否存在任何非回文多项式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的强力多项式乘法 |$O(q \cdot n \cdot K^2)$|$O(K)$| 太慢了 |
 | 预计算回文+前缀和|$O(nK + q)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 读取每个多项式，通过两端对称比较系数来检查是否为回文。 此步骤直接对定义进行编码并避免任何代数操作。 
2. 对于每个多项式，迭代$j = 0$到$k_i$并比较$a_j$和$a_{k_i - j}$。 如果发现任何不匹配，则将多项式标记为非回文。 
3. 将结果存储在数组中`good[i]`，其中 1 表示回文，0 表示不是。 
4.构建前缀和数组`pref`， 在哪里`pref[i] = pref[i-1] + good[i]`。 这允许快速范围聚合。 
5. 对于每个查询$[l, r]$, 计算`pref[r] - pref[l-1]`。 如果结果等于$r-l+1$，每个多项式都是回文的，否则至少有一个不是。 

前缀和在这里起作用的原因是我们将问题转化为检查段是否包含任何无效元素。 前缀和是通过计数检测存在的直接方法。 

### 为什么它有效

 乘法结构保证回文性在乘法下得以保留并被任何单个非回文因子破坏。 这就产生了单调性：一旦产品中出现“坏”元素，就无法通过与其他多项式相乘来修复它。 因此，当且仅当每个因子都是回文时，一定范围内的乘积才是回文，这将问题简化为二进制数组上的范围均匀性检查。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_pal(coef):
    i, j = 0, len(coef) - 1
    while i < j:
        if coef[i] != coef[j]:
            return False
        i += 1
        j -= 1
    return True

n, q = map(int, input().split())
good = [0] * (n + 1)

for i in range(1, n + 1):
    arr = list(map(int, input().split()))
    k = arr[0]
    coef = arr[1:]
    good[i] = 1 if is_pal(coef) else 0

pref = [0] * (n + 1)
for i in range(1, n + 1):
    pref[i] = pref[i - 1] + good[i]

out = []
for _ in range(q):
    l, r = map(int, input().split())
    total = pref[r] - pref[l - 1]
    out.append("1" if total == (r - l + 1) else "0")

print("\n".join(out))
```核心实现选择是将对称检查与查询处理分开。 多项式解析读取声明的次数并忽略任何卷积或归一化的需要。 前缀数组是 1 索引的，以简化范围减法并避免差一错误。 

这里的一个常见错误是忘记每个多项式行中的第一个数字是次数，而不是系数。 另一个是在前缀和中意外使用从零开始的索引，而查询是从一开始的，这会错误地移动每个答案。 

## 工作示例

 考虑三个多项式的小序列：

 输入：```
3 2
2 1 2 1
1 3 4
2 5 0 5
1 3
2 3
```回文性检查产生：

 | 我| 多项式| 检查结果 | 好[我] |
 | --- | --- | --- | --- |
 | 1 | 1 2 1 | 1 2 1 对称| 1 |
 | 2 | 3 4 | 不对称| 0 |
 | 3 | 5 0 5 | 5 0 5 对称| 1 |

 前缀和变为`[0,1,1,2]`。 

询问$[1,3]$给出`pref[3] - pref[0] = 2`，但范围大小为 3，因此输出为 0。这表明即使单个非回文多项式也会破坏乘积。 

询问$[2,3]$给出`pref[3] - pref[1] = 1`，范围大小为 2，因此输出再次为 0。 这证实了任何包含坏元素的段都会失败。 

现在考虑一个完全对称的集合：```
2 1
2 2 3 2
1 5 5
1 2
```两个多项式都是回文，因此前缀和为`[0,1,2]`。 查询返回 1，因为整个范围都是干净的，说明了单调保存属性。

 ## 边缘情况

 一个关键的边缘情况是常数多项式。 例如，输入`0 7`表示一个 0 次多项式。该算法将其视为单元素数组，因此`is_pal`立即返回 true，与定义匹配。 

另一种边缘情况是多项式的表示形式带有尾随零，这可能会在视觉上暗示不对称性。 例如，`3 1 0 1`实际上是对称且有效的。 该检查比较精确的位置，因此可以正确识别对称性。 

第三种边缘情况是单个多项式查询。 即使当$l = r$，答案仅取决于该多项式是否是回文。 前缀和公式自然可以处理这一问题，因为它减少了检查一个元素间隔。
