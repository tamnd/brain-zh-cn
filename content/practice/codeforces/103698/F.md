 ## 解决方案
 ## 问题理解

 该任务可以理解为无向图上的经典线性代数计数问题。 我们不是直接对生成树进行组合推理，而是通过根据图的结构构建的矩阵重新解释图，并计算对答案进行编码的行列式。 

具体来说，输入描述了一个简单的无向图$n$顶点和一组边。 目标是计算该图中存在多少个生成树。 生成树是边的子集，它连接所有顶点但不包含环，并且精确地使用$n-1$边缘。 

输出是表示该计数的单个整数。 在典型的竞争性编程设置中，该值以某个大素数为模，尽管底层数学也适用于整数。 

从复杂性的角度来看，关键的约束是图足够大，以至于不可能对生成树或边子集进行任何枚举。 即使是中等大小的图也会使生成树的数量呈指数增长，因此任何可接受的解决方案都必须将问题简化为多项式时间线性代数。 唯一可行的路径是对$n \times n$矩阵，它立即表明$O(n^\omega)$或者$O(n^3)$- 类型方法。 

一些边缘情况在实践中很重要。 具有单个顶点的图始终具有一棵生成树，即使没有边。 断开连接的图具有零生成树，因为不可能在不添加边的情况下连接所有顶点。 在同一对顶点之间具有多条边的图会以一种不平凡的方式增加生成树的数量，因为每条边分别对拉普拉斯做出贡献。 在这种情况下，粗心地执行重复数据删除操作会默默地产生不正确的结果。 

另一个微妙的情况是当图已经是一棵树时。 在这种情况下，答案正是一个，并且任何基于行列式的实现都必须保持完整性，尽管在模块化算术或随机化下工作。 

## 方法

 蛮力观点从生成树的定义开始。 人们可以尝试枚举所有子集$n-1$边并测试每个子集是否形成有效的树。 可以使用 union-find 或 DFS 检查连通性，如果子集恰好具有非循环性，则自动遵循非循环性$n-1$边并且相连。 这种方法是正确的，但是边缘子集的数量随着$\binom{m}{n-1}$，即使对于小图，这也是不可行的。 瓶颈不在于验证步骤，而在于候选者的数量。 

更结构化的方法使用基本转换：我们不直接计算组合对象，而是将它们编码为行列式中的代数项。 图的拉普拉斯矩阵捕获对角线上的顶点度数和非对角线的邻接结构。 基尔霍夫矩阵树定理指出，该拉普拉斯算子的任何辅因子都等于生成树的数量。 这将问题转换为计算一个行列式$(n-1) \times (n-1)$矩阵。 

这有帮助的原因是行列式计算允许有效的代数算法。 高斯消除或更先进的矩阵乘法技术降低了从指数计数到多项式算术的复杂性。 组合爆炸被吸收到行列式展开内的抵消中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| 指数| O(n + m) | 太慢了 |
 | 矩阵树（行列式）| O(n3) 或 O(n^ω) | O(n²) | 已接受 |

 ## 算法演练

 该解决方案是围绕构建拉普拉斯矩阵并计算其辅因子之一而构建的。 

1. 建立一个$n \times n$矩阵$L$，最初用零填充。 该矩阵将以代数形式表示图结构。 
2. 对于每一个边缘$u - v$， 增加$L[u][u]$和$L[v][v]$减一，并将两者都减少$L[u][v]$和$L[v][u]$一个。 这同时对顶点度数和邻接度进行编码。 每条边都贡献一个保持对称性的结构化更新。 
3. 删除任意行和列，通常是最后一个顶点，产生一个$(n-1) \times (n-1)$矩阵。 此步骤是必需的，因为由于行总和为零，完整拉普拉斯算子的行列式始终为零，但任何辅因子都带有生成树计数。 
4. 使用所需模数下的高斯消去法或使用无分数消去法的整数来计算结果矩阵的行列式。 行列式聚集了矩阵中编码的所有生成树贡献。 
5. 输出行列式作为最终答案。 

关键的非显而易见的步骤是为什么删除行和列不会丢失信息。 拉普拉斯算子有等级$n-1$对于连通图，所有辅因子都相等。 这使我们能够将问题简化为满秩行列式计算。 

### 为什么它有效

 每棵生成树恰好对应于拉普拉斯变换后的行列式展开式中的一项。 行列式中的代数取消消除了所有包含循环的结构，仅留下非循环连接的边缘选择。 剩余的幸存项与生成树一一对应，这确保行列式等于它们的计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def det_mod(mat, mod):
    n = len(mat)
    det = 1
    sign = 1

    for i in range(n):
        pivot = i
        for j in range(i, n):
            if mat[j][i] != 0:
                pivot = j
                break
        if mat[pivot][i] == 0:
            return 0

        if pivot != i:
            mat[i], mat[pivot] = mat[pivot], mat[i]
            sign *= -1

        inv = pow(mat[i][i], mod - 2, mod)
        det = det * mat[i][i] % mod

        for j in range(i + 1, n):
            factor = mat[j][i] * inv % mod
            for k in range(i, n):
                mat[j][k] = (mat[j][k] - factor * mat[i][k]) % mod

    return det * sign % mod

n, m = map(int, input().split())
mod = 10**9 + 7

L = [[0] * n for _ in range(n)]

for _ in range(m):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    L[u][u] += 1
    L[v][v] += 1
    L[u][v] -= 1
    L[v][u] -= 1

for i in range(n):
    for j in range(n):
        L[i][j] %= mod

mat = [row[:-1] for row in L[:-1]]

print(det_mod(mat, mod))
```该实现首先直接根据边缘贡献构建拉普拉斯算子。 最后一行和最后一列的删除是通过切片完成的，这在连通性假设下保持矩阵平方和满秩。 

行列式例程使用以素数为模的高斯消去法。 主元选择通过尽可能避免零主元来确保模算术中的数值稳定性。 行操作就地执行，行列式通过主元缩放累积。 符号跟踪处理行交换，这很重要，因为每次交换都会翻转行列式符号。 

一个微妙的点是模逆假设模数是素数。 这是竞争性编程的标准，其中$10^9 + 7$或使用类似的素数。 

## 工作示例

 考虑具有三个顶点和三个边的三角形图。 

拉普拉斯算子是$$\begin{bmatrix}
2 & -1 & -1 \\
-1 & 2 & -1 \\
-1 & -1 & 2
\end{bmatrix}$$删除最后一行和最后一列给出$$\begin{bmatrix}
2 & -1 \\
-1 & 2
\end{bmatrix}$$行列式计算进行如下。 

| 步骤| 矩阵状态| 运营|
 | --- | --- | --- |
 | 1 | [[2, -1], [-1, 2]] | 初始|
 | 2 | [[2, -1], [0, 3]] | 消除第二行 |
 | 3 | 6 | 枢轴的乘积 |

 结果 3 与三角形中已知的生成树数量相匹配。 

现在考虑一棵在一条线上有三个顶点的树。 拉普拉斯辅因子成为行列式为 1 的矩阵。这证实一棵树恰好贡献了一个生成树。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import math

    n, m = map(int, sys.stdin.readline().split())
    mod = 10**9 + 7

    L = [[0]*n for _ in range(n)]
    for _ in range(m):
        u, v = map(int, sys.stdin.readline().split())
        u -= 1
        v -= 1
        L[u][u] += 1
        L[v][v] += 1
        L[u][v] -= 1
        L[v][u] -= 1

    mat = [row[:-1] for row in L[:-1]]

    def det(a):
        n = len(a)
        detv = 1
        sign = 1
        for i in range(n):
            p = i
            for j in range(i, n):
                if a[j][i]:
                    p = j
                    break
            if a[p][i] == 0:
                return 0
            if p != i:
                a[i], a[p] = a[p], a[i]
                sign *= -1
            inv = pow(a[i][i], mod-2, mod)
            detv = detv * a[i][i] % mod
            for j in range(i+1, n):
                f = a[j][i] * inv % mod
                for k in range(i, n):
                    a[j][k] = (a[j][k] - f*a[i][k]) % mod
        return detv * sign % mod

    return str(det(mat))

 ## 边缘情况

 由于拉普拉斯余因子变为奇异，因此可以正确处理断开连接的图。 在消除过程中，最终出现零主元，迫使行列式为零，这正确地反映了生成树的不存在。 

删除行和列后，单个顶点图简化为空矩阵。 按照惯例，0×0 矩阵的行列式是 1，与只有一棵生成树的事实相匹配。 

树输入在消除后导致拉普拉斯辅因子中的三角形结构。 每个主元恰好对应一个边缘约束，产生行列式而不取消，从而确认最小连接情况下的正确性。
