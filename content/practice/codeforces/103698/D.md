 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$n \times m$。 网格的每个单元格要么是 0，要么是 1。网格不是任意的：它必须满足全局一致性规则，将每个单元格与其行和列的奇偶校验结构联系起来。 

对于任何职位$(i, j)$，如果您采用行中的所有值$i$并将它们异或在一起，并取列中的所有值$j$并将它们异或在一起，然后异或这两个结果必须重现存储在单元格中的值$(i, j)$。 一个微妙之处在于，细胞$(i, j)$包含在行和列 XOR 中，因此它实际上被计数两次并在 XOR 算术中取消自身。 

任务不是构造一个这样的矩阵，而是计算有多少个大小相同的二元矩阵$n \times m$对于许多测试用例都满足这个条件。 答案必须以模计算$998244353$。 两个都$n$和$m$可以非常大，最多$10^{18}$，因此任何依赖于迭代行或列的解决方案都是立即不可能的。 

约束大小完全排除了任何$O(nm)$甚至$O(n + m)$每个测试方法。 唯一可行的策略是将问题简化为封闭形式的组合结构或每个查询的恒定时间公式。 

常见的故障模式是假设行和列是独立的。 例如，将每一行视为自由选择，然后调整列将会严重计算过多，因为 XOR 约束同时耦合所有行和列。 另一个微妙的边缘情况出现时$n = 1$或者$m = 1$。 在这种情况下，约束会退化，每个行或列的选择都会变成一个更简单的条件。 

例如，如果$n = 1$，该条件变得微不足道，因为每个单元格都是其行和列聚合器。 粗心的解释可能仍然试图应用一般公式，但最终会多算或少算。 

另一个微妙的情况是最小的非平凡网格$2 \times 2$。 如果我们尝试暴力枚举，我们可能会怀疑独立性，但这些约束实际上在所有四个单元之间强加了全局奇偶耦合，因此并非所有单元都具有独立性。$2^4$配置有效。 

## 方法

 暴力破解的想法很简单：尝试每个二进制矩阵并检查它是否满足所有单元格的异或条件。 对于每个单元格，计算其行 XOR 和列 XOR 并验证相等性。 这是可行的，因为条件是可以直接测试的。 然而，成本与电池数量成指数关系。 即使是为了$2 \times 2$，需要检查$16$矩阵，并且对于$10 \times 10$，已经变成了$2^{100}$，这是不可行的。 

关键的观察结果是约束在整个场上是线性的$\mathbb{F}_2$。 XOR 的行为类似于模 2 加法，因此每个条件实际上是二进制变量的线性方程组。 一旦认识到这一点，问题就从组合数学转向线性代数。 

我们不再从细胞的角度思考，而是重新解释结构。 每行都有一个关联的 XOR 值，每列都有一个关联的 XOR 值。 方程为$(i, j)$链接行$i$， 柱子$j$，以及细胞本身以对称方式。 当扩展到整个网格时，大多数约束都不是独立的。 系统崩溃到一个低维空间，其中仅保留一小部分自由度。 

查看此情况的一个有用方法是修复第一行和第一列中的所有值。 一旦选择了这些，所有其他单元都受到约束结构的强制。 这意味着整个矩阵由大小边界确定$n + m - 1$，但即使该边界也不是完全独立的，因为一个全局一致性条件将其联系在一起。 

在完成依赖结构之后，有效矩阵的数量减少到 2 的幂，其指数等于自由变量的数量，最终为$(n - 1)(m - 1)$。 一旦行和列的相互作用得到解决，这些位置中的每一个都对应于一个独立的选择。 

因此，我们不是枚举矩阵，而是计算线性系统中的独立自由度$\mathbb{F}_2$。

 ## 算法演练

 1. 观察到每个单元约束都是 XOR 上的线性方程，因此整个系统可以视为二元线性系统。 这种重新表述至关重要，因为它允许根据等级和自由度而不是显式配置进行推理。 
2. 确定行和列异或约束不是独立的； 对所有行方程和所有列方程求和会产生冗余的全局约束。 这种冗余减少了独立方程的有效数量。 
3. 通过固定第一行和第一列中除交集之外的所有单元格来选择解空间的基础。 一旦这些边界值固定，每个内部单元就通过传播 XOR 约束来唯一确定。 
4. 计算冗余后真正自由变量的数量。 系统准确离开$(n - 1)(m - 1)$独立的选择，每个选择对总数的贡献为 2。 
5. 将结果转换为模形式，计算$2^{(n-1)(m-1)} \bmod 998244353$使用快速求幂，因为指数可以大到$10^{18}$。 
6. 独立地为每个测试用例返回该值。 

### 为什么它有效

 XOR 约束形成一致的线性系统$\mathbb{F}_2$。 任何有效矩阵都对应该系统的一个解，并且任何解唯一确定一个矩阵。 系统的等级决定了有多少约束是真正独立的。 消除行列对称性产生的相关方程后，解空间具有维度$(n-1)(m-1)$，因此这些自由变量的每次赋值都恰好产生一个有效矩阵，并且不会遗漏或重复计算任何有效矩阵。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modpow(a, e):
    res = 1
    while e > 0:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

t = int(input())
for _ in range(t):
    n, m = map(int, input().split())
    exp = (n - 1) * (m - 1)
    print(modpow(2, exp))
```该实现将每个测试用例的所有内容减少为一次求幂。 唯一重要的部分是认识到指数是$(n-1)(m-1)$，它来自计算 XOR 约束网格中的独立自由度。 

模幂运算是必要的，因为指数和结果数对于直接计算来说都太大了。 二进制求幂循环确保指数中的对数时间。 

一个常见的实现错误是忘记指数必须以 64 位算术计算。 自从$n$和$m$可以达到$10^{18}$，他们的乘积必须存储在一种类型中，在减少模数之前避免溢出$MOD-1$由于直接求幂，这里被认为是无关紧要的。 

## 工作示例

 考虑一个小案例$n = 2, m = 2$。 指数是$(2-1)(2-1) = 1$，所以答案是$2$。 这意味着恰好有两个矩阵满足约束。 如果我们一一列举，我们就会发现，一旦一个细胞被自由选择，其他所有细胞都被强制选择。 

| 步骤| 解读|
 | --- | --- |
 | 计算指数 |$(2-1)(2-1) = 1$|
 | 结果 |$2^1 = 2$|

 这表明该系统并不是完全自由的，但仍然允许一个二元自由度。 

现在考虑$n = 3, m = 3$。 指数变为$(3-1)(3-1) = 4$，所以有$2^4 = 16$有效的矩阵。 

| 步骤| 解读|
 | --- | --- |
 | 计算指数 |$(3-1)(3-1) = 4$|
 | 结果 |$2^4 = 16$|

 此案例演示了有效配置的数量如何随着内部自由度而不是总单元数呈指数增长。

 ## 边缘情况

 当$n = 1$，网格折叠成一行。 每个单元格同时是其行和列的一部分，因此 XOR 约束强制每个单元格自身相等，没有交叉依赖性。 指数公式给出$(1-1)(m-1) = 0$，准确地生成一个有效的配置，这与一旦强制执行一致性，每个分配都是唯一确定的这一事实相匹配。 

什么时候$m = 1$，情况是对称的。 体系再次崩溃，内部自由不复存在。 指数变为零，结果为$1$，对应于单个一致配置。 

对于最小的非平凡正方形$2 \times 2$，约束耦合所有四个单元。 如果我们尝试独立改变一个单元格，异或条件会立即强制其余三个单元格。 这通过指数反映$1$，这意味着在应用约束后，只有一个自由的二元选择仍然存在。
