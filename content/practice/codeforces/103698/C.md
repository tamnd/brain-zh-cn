 ## 解决方案
 ## 问题理解

 我们拥有一系列银行账户，每个账户都持有一定数量的资金。 我们的任务不是对通常意义上的子集进行优化，而是要了解当我们将人们分为已排序总体的前缀与其补集时，分布会变得多么“不均匀”。 

正式地，想象一下按财富对人们进行分类，然后选择一个阈值将人口分为两组。 对于任何这样的划分，我们计算两个百分比：第一组中的人所占的比例，以及他们共同拥有的总财富的比例。 这给出了一对值 A 和 B。在所有可能的分割中，我们想要最大化 B − A 的对。如果多个分割达到相同的最大值，我们更喜欢 A 较大的分割。 

输入给出账户数量及其余额。 输出是一对最佳百分比。 

这些限制允许最多 100000 个帐户，这会立即排除任何以朴素的二次方式为每个可能的分割重复重新计算总和的方法。 任何解决方案都必须预处理数据并在恒定或对数时间内评估候选分割。 

当所有余额都相同时，就会出现微妙的边缘情况。 在这种情况下，每次分裂都会产生 B 等于 A，因此目标 B − A 始终为零。 然后，平局规则迫使我们选择具有最大 A 的分割，这对应于获取所有元素。 例如，使用输入`10 10`，两个人相同，每个分区的财富比例相等，正确答案变为`100.00 100.00`。 

当有许多重复值但并非全部相等时，就会出现另一种棘手的情况。 假设唯一性并在局部密度变化时贪婪地切割的简单方法可能会失败，因为最佳分割仅取决于排序后的前缀和，而不取决于单个位置或任意分组。 

## 方法

 蛮力的想法很简单：尝试一切可能的方法来选择一个人的子集，计算他们总共拥有多少财富，计算他们代表总人口的比例，并评估 B – A。这在概念上是正确的，但完全不可行。 当 n 达到 100000 时，子集的数量为 2^n，甚至限制为类似前缀的结构仍然留下 O(n) 个候选，每个候选都需要重新计算总和，除非我们进行预处理。 

即使我们限制自己只考虑排序的前缀，我们仍然需要计算财富的前缀总和并评估每个分割。 这显着减少了问题，但主要的见解是答案必须对应于选择排序数组（最大元素）的某些后缀，因为最大化给定人口比例的财富份额总是倾向于首先采用较高的值。 

一旦数组按降序排序，我们就维护财富的前缀和。 对于每个 k（代表前 k 个帐户），我们计算 A = k / n 和 B = sum(top k) / 总和。 目标是在所有 k 上最大化 B − A。 

这将问题在排序后转化为简单的线性扫描。 关键的结构特性是排序引起的单调性：任何最佳子集都可以重新排列以获取最大元素，而不会减少财富份额，同时保持计数不变。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 | O(2^n·n) | O(2^n·n) | O(1) | O(1) | 太慢了 |
 | 排序后的前缀评估 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 读取所有账户余额并计算总财富。 需要这个总数来标准化所有未来的百分比计算。 
2. 按降序对余额进行排序，以便前缀选择始终代表其规模最丰富的可能组。 这确保了我们认为的任何 k 人小组都是该规模的最佳小组。 
3. 在已排序的数组上构建前缀和。 每个前缀总和代表前k个人的财富总和。 
4. 迭代从 1 到 n 的所有 k。 对于每个 k，将 A 计算为 100 × k / n，将 B 计算为 100 × prefix_sum[k] /total_sum。 
5. 跟踪 B − A 的最大值。如果多个 k 值产生相同的差异，则优先选择 k 较大的值，因为它对应于较大的 A。 
6. 输出最好的（A，B）对，四舍五入到小数点后两位。 

我们只需要测试前缀的原因是，对于任何固定的 k，选择除前 k 个最富有之外的任何群体都不能增加总财富。 由于 B 仅取决于所选集合的总财富，因此用较大的元素替换任何元素都会严格改进或保留 B，同时保持 A 固定。 

### 为什么它有效

 关键的不变量是，按降序排序后，任何固定基数 k 的最佳可能财富总是通过取前 k 个元素来实现。 这将搜索空间从所有子集减少到单个嵌套前缀链。 由于 A 仅取决于 k，而 B 通过前缀和最大化，因此对所有子集的优化会崩溃为仅对 k 进行优化。 目标 B − A 成为 k 上的一维函数，扫描所有 k 可以保证全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    total = sum(a)
    a.sort(reverse=True)
    
    pref = 0
    
    best_diff = -10**18
    best_a = 0.0
    best_b = 0.0
    
    for i in range(n):
        pref += a[i]
        k = i + 1
        
        A = 100.0 * k / n
        B = 100.0 * pref / total
        
        diff = B - A
        
        if diff > best_diff or (abs(diff - best_diff) < 1e-12 and A > best_a):
            best_diff = diff
            best_a = A
            best_b = B
    
    print(f"{best_a:.2f} {best_b:.2f}")

if __name__ == "__main__":
    solve()
```其实现直接遵循前缀扫描的思想。 按降序排序可确保每个前缀都适合其大小。 前缀和可以有效地积累财富，因此每个候选分割都会在恒定的时间内进行评估。 浮点格式仅在输出时处理，而所有比较都使用原始计算值，且相等性容差较小。 

一个常见的错误是忘记打破平局规则。 代码显式检查差异相等性，然后选择较大的 A。

 ## 边缘情况

 当所有值都相等时，每个前缀都会产生相同的 B − A，因此算法必须完全依赖平局打破规则。 排序仍然可以正确工作，但最终答案必须选择 k = n，而不是 k = 1。实现会处理此问题，因为当差异相等时它会显式比较 A。 

当一个值支配其余值时，最佳 k 可能很小，通常 k = 1 或 2。前缀扫描正确地捕获了这一点，因为早期前缀在从较小值稀释之前会产生较大的 B − A。 

当值高度倾斜但不严格递减时，排序可确保局部不规则性不会误导选择过程。 任何未排序的贪婪方法都会在这里失败，因为稍后在不排序的情况下获取高值会低估可能的前缀和并错过最佳分割。
