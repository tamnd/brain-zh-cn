 ## 解决方案
 ## 问题理解

 我们给定一个系统，其中有相同数量的狗和笼子，都标记为 0 到 N − 1。每只狗独立、均匀地随机选择一个笼子，每个笼子最多可以容纳一只狗，这意味着最终的配置是笼子上狗的随机排列。 

我们没有被问到结果的完整分布，只问到最终没有被关在笼子里的狗的预期数量与它们自己的指数相同。 换句话说，对于每只狗 i，我们关心它是否避免被放入笼子 i，并且我们想要这样的狗的预期数量。 

输入由多个测试用例组成，每个测试用例给出一个值 N，对于每个测试用例，我们输出“不匹配”狗的预期数量。 

这些约束允许最多 10^5 个测试用例，N 个最多 10^5，因此任何每个测试用例的模拟或基于阶乘的推理都是不可能的。 解决方案必须在最少的预处理或直接公式计算后将每个查询的时间减少到 O(1)。 

一个微妙的边缘情况是 N = 1。对于一只狗和笼子，狗总是在正确的笼子里，因此答案必须恰好为 0。任何错误地假设多个元素对称但忘记了基本情况的推导仍然可以为较大的 N 生成看起来正确的公式，但在这里会失败。 

另一个概念上的陷阱是将这个过程视为每只狗的独立选择。 这表明每只狗犯错的概率为 1 − 1/N，但不强制笼子的唯一性。 幸运的是，即使全局分配是排列而不是独立抽签，期望线性也允许这种简化保持有效。 

## 方法

 暴力视角首先想象将狗放入笼子的所有可能的有效分配。 每个赋值都是大小为 N 的排列，对于每个排列，我们计算有多少个索引 i 满足 permutation[i] ≠ i。 然后我们对所有 N 个值进行平均！ 排列。 

这是正确的，但完全不可行。 枚举N！ 即使 N = 10，配置也是不可能的，因为 10！ 已经是 360 万，对于 N = 20 来说，它就变得天文数字了。 

关键的观察是我们实际上并不需要排列的联合结构。 我们只需要每个单独指数的预期贡献。 定义一个指示变量 Xi，如果狗 i 不在笼子 i 中，则该变量为 1，否则为 0。 答案是所有 Xi 总和的期望值。 

期望的线性使我们能够对各个指标的期望进行求和，而无需考虑位置之间的依赖性。 对于固定的狗 i 来说，对称性意味着在均匀随机排列中，狗 i 出现在任何笼子中的可能性相同，因此它落在自己的笼子中的概率恰好是 1/N。 因此，不匹配的概率为 1 − 1/N = (N − 1)/N。 

对所有 N 只狗求和，得到 N·(N − 1)/N = N − 1。这将整个问题分解为常数时间公式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(N!) | O(1) | O(1) | 太慢了 |
 | 每个职位的预期价值| 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 读取当前测试用例的 N。 
2. 处理N=1的特殊情况，立即输出0，因为唯一的狗一定是正确的。 
3. 对于 N ≥ 2，使用导出公式 N − 1 计算预期的不匹配数。 
4. 以足够精度的浮点格式打印结果。 

计算背后的推理步骤是每个位置在预期中独立贡献，即使排列将它们全局耦合。 指标分解确保我们从不依赖实际的独立性。 

### 为什么它有效

设 Xi 为狗 i 是否不在笼子 i 中的指标。 在随机排列中，每只狗被固定在正确位置的概率为 1/N，因此 E[Xi] = (N − 1)/N。 总期望为 E[ΣXi] = ΣE[Xi] = N · (N − 1)/N。 这保证了正确性，而无需枚举或模拟排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for i in range(1, t + 1):
        n = int(input())
        if n == 1:
            out.append(f"Case #{i}: 0.0000000000")
        else:
            ans = n - 1
            out.append(f"Case #{i}: {ans:.10f}")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现直接应用导出的封闭式表达式。 唯一的分支是 N = 1，以在格式化要求下保持正确性，并避免在琐碎情况下进行不必要的浮点运算。 具有 10 位小数的格式与预期值问题的典型 Codeforces 输出容差相匹配。

```我们计算：

 | 步骤| 尼 | 每只狗的失配概率| 预计总数 |
 | --- | --- | --- | --- |
 | 计算| 2 | 1/2 | 1/2 2 × 1/2 = 1 |

 两种排列的可能性相同：恒等给出 0 个不匹配，交换给出 2 个不匹配，平均为 1。这与公式 N − 1 匹配。

 ## 边缘情况

 对于 N = 1，算法显式返回 0。公式 N − 1 也会给出 0，但该分支避免了格式化逻辑中的任何歧义。 

例如输入 N = 1：

 | 步骤| 价值|
 | --- | --- |
 | 读 N | 1 |
 | 检查 N == 1 | 真实|
 | 输出| 0.0000000000 |

 这证实了最小可能配置的行为与期望参数一致。 

对于较大的 N，例如 N = 2 或 N = 3，计算直接减少到 N − 1，而不依赖于排列结构。 该算法不需要模拟分配，因为期望已经抽象了狗之间的所有交互。
