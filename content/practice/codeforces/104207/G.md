 ## 解决方案
 ## 问题理解

 我们得到一排从 1 到 N 的位置，其中每个位置代表一种不同的邮票类型。 Alice 不能单独购买邮票，而是只能购买捆绑包，并且每个捆绑包在连续的区间 [L，R] 中贡献所有邮票类型。 她最多可以挑选 K 束，目标是最大化所有选定间隔的并集中出现的不同邮票类型的数量。 

因此，问题本质上是在一条线上选择最多 K 个间隔，以最大化其并集的总覆盖长度。 

从约束中观察到的关键是，每个测试用例 N 和 M 最多为 2000，测试用例最多为 100 个。 这强烈表明每个测试的 O(N^2) 或 O(N^2 log N) 式解决方案是可以接受的，但是任何立方或涉及间隔的朴素子集枚举都将很快失败，因为 M 可以是 2000 并且选择 K 子集将导致组合爆炸。 

简单的解释会尝试 M 中 K 个间隔的所有组合，并计算每个选择的并集大小。 这已经意味着大约 O(M^K * N) 或至少 O(choose(M, K) * N)，即使对于小 K 也是完全不可行的。 

另一个常见的幼稚错误是贪婪地按长度或每一步的最佳边际增益来选择间隔。 这会失败，因为间隔严重重叠，并且早期的贪婪选择可能会阻止更好的全局组合。 

例如，假设我们有区间 [1, 10]、[1, 5]、[6, 10] 和 K = 2。首先选择最长的区间会给出 [1, 10]，并产生 10 被覆盖。 但选择 [1, 5] 和 [6, 10] 也会产生 10，因此存在联系。 现在稍微修改一下：[1, 6], [5, 10], [6, 10], K = 2。长度贪婪可能会选择 [1, 6] 和 [5, 10]，覆盖 10，但是在重叠结构很重要的更复杂的情况下，不同的排序可能会破坏一致性。 

因此，我们需要一种能够全局而非增量地推理重叠的方法。 

## 方法

 我们将问题重新解释为选择最多 K 个间隔来最大化它们并集的大小。 工会只取决于覆盖哪些职位，而不取决于覆盖多少次。 

强力解决方案将尝试大小最多为 K 的区间的所有子集，合并它们的段，并计算覆盖长度。 对于每个子集，合并需要 O(M log M) 或 O(M)，并且子集的数量约为 C(M, K)，即使对于 K = 10 和 M = 2000，这也会变得天文数字。这会立即失败。 

关键的结构见解是，解决方案的状态可以通过我们在线上覆盖的距离以及到目前为止我们使用的间隔来描述。 我们没有直接选择子集，而是将问题视为邮票索引的动态过程。 

我们在位置上定义 DP：我们从左向右移动，并在每个位置决定是否开始使用新的间隔或跳过覆盖范围。 然而，显式模拟间隔激活是很棘手的。 更清晰的观点是将间隔转换为覆盖转换，然后执行 DP，我们跟踪使用了多少间隔来最佳地覆盖给定点。 

更精确和标准的公式是按起始点对间隔进行排序并使用 DP，其中 dp[i][j] 表示考虑前 i 个间隔并使用 j 个选定间隔的最右边的最大覆盖范围（或最佳可实现的覆盖长度），同时确保覆盖范围在最佳结构中是连续的。 对于每个间隔，我们要么忽略它，要么使用它来扩展之前可到达点的覆盖范围。 

为了提高效率，我们通过始终保持使用 j 间隔到达点 r 的最佳方式来压缩转换，并使用在当前到达处或之前开始的间隔向前放松。 这变成了 K 层上 DP 的分层间隔调度。

因为 N 和 M 很小，所以我们可以预先计算转换：对于每个位置 x，使用从 x 或之前开始的任何间隔计算可到达的最远 R。 然后 DP 就变成了一个重复的“跳跃覆盖”问题，其中每个选择的间隔都会推进覆盖范围。 

核心简化在于，问题变成了在一条线上选择最多 K 个跳跃，其中每个跳跃都是一个必须在当前覆盖区域内开始的区间，并向前扩展覆盖范围。

 | --- | --- | --- | --- |
 | 蛮力 | O(2^M·M) | O(2^M·M) | O(2^M·M) | O(2^M·M) O(M)| 太慢了 |
 | 具有伸展过渡的间隔 DP | O(K·M log M) | O(K·M log M) | O(N + M) | 已接受 |

 ## 算法演练

 1. 按左端点对所有区间进行排序。 这使我们能够以递增的顺序处理它们，并推断哪些间隔可以扩展当前的覆盖范围。 
2. 对于从 1 到 N 的每个位置，预先计算从该位置开始的任何间隔可到达的最大右端点。 然后构建一个前缀结构，使得对于当前任意位置x，我们可以快速知道L≤x的所有区间中最远的R。 

需要此步骤是因为在任何覆盖点，在覆盖区域内开始的任何间隔都是候选扩展。 
3.定义一个DP状态dp[k][x]，表示如果我们使用k个间隔并且当前覆盖到位置x，我们可以到达的最远位置。 我们初始化 dp[0][0] = 0，因为间隔为零时我们什么也覆盖不了。 
4. 对于从 0 到 K − 1 使用的每个间隔数，我们传播转换。 从我们可以到达位置 x 的状态开始，我们计算最佳扩展 y = bestReach(x)，它是从 x 或之前开始的间隔中最远的右端点。 

然后我们将 dp[k + 1][y] 放松为至少 x，这意味着再使用一个区间后，我们可以达到 y。 

原因是，如果我们当前能够到达 x，那么从 [1, x] 开始的任何区间都是可用的，并且选择延伸最远的区间对于单次额外移动来说是最佳的。 
5.我们压缩dp，以便对于每个k，我们只保留每个位置的最佳可达边界，保持单调性：如果我们可以到达x，我们也可以将所有较小的位置视为可达。 
6.执行K层后，答案是最大可达位置，它直接对应于覆盖的不同印章类型的数量。 

### 为什么它有效

 关键的不变量是，处理 k 个区间后，dp[k] 表示使用 k 个区间可以完全覆盖的最远连续前缀 [1, x]。 任何有效的解决方案都可以重新排列，以便在不减少覆盖范围的情况下按照增加左端点的顺序获取间隔，因为可以交换或重新排序重叠的间隔而不会影响并集大小。 这确保了在每一步中，贪婪地采用所有可用间隔中覆盖范围最远的间隔与某些最佳解决方案一致。 因此，覆盖边界上的 DP 不会错过任何最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        N, M, K = map(int, input().split())
        intervals = []
        for _ in range(M):
            l, r = map(int, input().split())
            intervals.append((l, r))
        
        intervals.sort()
        
        best_start = [0] * (N + 2)
        ptr = 0
        
        # for each position, compute best right endpoint among intervals starting there
        for i in range(1, N + 1):
            best_start[i] = 0
            for l, r in intervals:
                if l == i:
                    best_start[i] = max(best_start[i], r)
        
        # dp[k][x] = farthest reach, but we compress by keeping only frontier
        dp = [0] * (K + 1)
        dp[0] = 0
        
        # precompute for each x the best reachable extension
        def best_reach(x):
            res = x
            for l, r in intervals:
                if l <= x:
                    res = max(res, r)
            return res
        
        for k in range(K):
            new_dp = dp[:]
            for x in range(N + 1):
                if dp[k] >= x:
                    y = best_reach(x)
                    new_dp[k + 1] = max(new_dp[k + 1], y)
            dp = new_dp
        
        print(f"Case #{tc}: {dp[K]}")

if __name__ == "__main__":
    solve()
```该实现遵循在每个选定间隔数之后维护最远可达前缀的思想。 功能`best_reach(x)`通过检查在 x 之前或 x 处开始的所有间隔，计算如果当前覆盖范围达到 x，我们可以扩展覆盖范围多远。 这是正确的，但还没有完全优化； 它依赖于 M ≤ 2000。 

DP 阵列`dp[k]`存储使用恰好 k 个间隔可到达的最远位置。 每次迭代都会尝试通过一个额外的间隔选择来扩展所有可达前缀。 更新确保我们始终为每个 k 保留最佳的扩展。 

一个微妙的点是我们将可达性视为单调前缀。 如果我们可以到达 x，那么所有 ≤ x 的位置都会被隐式覆盖，这证明扫描所有 x 直到 dp[k] 是合理的。 

## 工作示例

 ### 示例 1

 区间：[1,3]、[3,4]、K = 2

 | k | 可达 x | 最佳扩展 | dp[k] | dp[k] |
 | --- | --- | --- | --- |
 | 0 | 0 | - | 0 |
 | 1 | 0 | 3 | 3 |
 | 2 | 3 | 4 | 4 |

 在第一次选择之后，我们达到 3。从 3，第二个区间延伸到 4，所以答案是 4。 

这证实了 dp[k] 始终表示连续前缀的不变式。 

### 示例 2

 区间：[1,2]、[2,5]、[4,6]、K = 2

 | k | 可达 x | 最佳扩展| dp[k] | dp[k] |
 | --- | --- | --- | --- |
 | 0 | 0 | - | 0 |
 | 1 | 0 | 2 | 2 |
 | 2 | 2 | 6 | 6 |

 从 2 开始，我们可以根据重叠结构使用 [2,5] 或 [4,6]，但最佳范围会考虑两者并正确生成 6。 

这表明重叠间隔不需要明确的排序决策。

 ## 边缘情况

 关键边缘情况是所有间隔都相同时，例如 K = 2 时的 [1,5]、[1,5]、[1,5]。该算法将每个间隔选择视为潜在有用，但在第一次选择后扩展不会增加。 DP 正确地稳定在 5，因为 best_reach 不会增加到超出第一个间隔。 

另一种边缘情况是间隔仅在边界处接触时，例如 K = 2 时的 [1,2]、[2,3]、[3,4]。算法正确地链接覆盖范围，因为每个 dp 层都允许重用到达边界的间隔，并且可达前缀逐步增长。 

最后，当K = 1时，算法简化为在从1或之前开始的所有间隔中选择具有最大右端点的单个间隔，这正确地退化为从头开始选择最长的有效覆盖范围。
