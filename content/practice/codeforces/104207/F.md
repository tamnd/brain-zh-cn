 ## 解决方案
 ## 问题理解

 我们有几组人。 每个组都有固定的规模，如果在抽签结果中选择了一个组，那么该组的所有成员都会一起获胜。 在任何单一结果中，所有选定组的获胜者总数都受到上限的限制$M$。 因此，每个结果实际上都是组的子集，其总大小不超过$M$。 

我们不仅仅选择一种结果。 相反，我们设计了所有有效结果的概率分布。 每个结果都有一定的概率出现，这些概率之和为 1。一旦对某个结果进行了抽样，所选组中的每个人都会成为赢家。 

要求是个人层面的对称性：每个人，无论属于哪个群体，都必须具有完全相同的获胜概率$p$。 任务是最大限度地实现这一目标$p$。 

重申这一点的一个有用方法是每个组$i$在随机结果中要么完全活跃，要么不活跃，我们希望对可行的子集进行随机化，以便每个组都具有相同的包含概率。 由于群体中的所有成员都是不可区分的，因此群体中的一个人的概率$i$获胜的概率正是该组$i$被选中。 

就组数量而言，限制非常小，$N \le 10$，但是背包限制$M \le 100$最大 100 人的团体规模仍然很重要。 小的$N$是关键的结构提示：困难在于群体子集的组合，而不是个体的组合。 

一种简单的方法会尝试显式地构建子集上的分布并连续调整概率。 这立即就会产生指数级的可能性，因为有$2^N$已经有了子集，我们正在选择它们的凸组合。 

一些微妙的边缘情况自然会出现。 

一种是当单个组已经超过$M$。 该组永远不会出现在任何有效结果中，因此其成员的概率必定为零，可行性完全取决于其余组。 

另一个是当所有群体与$M$。 那么许多子集都是有效的，并且很容易假设均匀随机选择有效，但当组大小不同时，均匀性并不能保证相等的边际概率。 

第三种边缘情况是一组非常大而其余组很小。 最佳解决方案通常会完全避免大型群体，即使它贡献了很多人，因为包含它会严重限制可行的组合。 

## 方法

 暴力策略将枚举组子集上所有可能的概率分布。 每个子集是否可行取决于其总大小是否最多$M$。 然后，我们尝试为子集分配权重，以便每个组都具有相同的边际概率。 这在原则上是正确的，因为它直接遵循问题的定义。 

问题是，即使在考虑概率之前，也有多达$2^N \le 1024$子集。 它们上所有分布的空间都是连续且高维的，因此暴力破解是不可能的。 

关键的结构观察是我们正在解决一小组极值点上的线性可行性问题。 每个子集对应于一个 0/​​1 向量$N$维空间，我们希望将统一向量表示为这些子集向量的凸组合。 这将问题转化为检查一个点是否位于维度最多为 10 的最多 1024 个点的凸包内。 

由于维度很小，因此可以通过结合背包约束对组子集进行动态规划来利用凸包结构。 背包限制$M$仅用于过滤可行子集。 

该解决方案简化为使用所有有效子集并推理如何组合它们以使每个坐标具有相同的期望。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解| 无限连续搜索$2^N$单纯形| 大| 不可能|
 | 可行掩码上的子集 DP + 凸推理 |$O(2^N \cdot M)$|$O(2^N)$| 已接受 |

 ## 算法演练

 我们首先枚举组的所有子集并标记哪些子集在背包约束下是可行的。 对于每个子集$S$，我们计算它的总大小并仅在以下情况下保留它：$\sum_{i \in S} a_i \le M$。 

接下来，我们将每个可行子集解释为一个向量$v_S \in \{0,1\}^N$， 在哪里$v_S[i]=1$如果组$i$包括在内。 目标是表示目标向量$(p, p, \dots, p)$作为这些向量的凸组合。 

我们现在换个角度来对待$p$未知。 我们不是直接搜索分布，而是检查候选分布的可行性$p$使用二分查找。 这是有效的，因为如果可以实现某个概率，则通过与空子集混合也可以实现任何较小的概率。 

对于固定的$p$，我们尝试强制每个坐标都具有期望值$p$。 这相当于找到非负权重$w_S$使得所有权重的总和为 1 并且对于每个组$i$，包含的子集的总权重$i$等于$p$。 

我们将其转化为群子集上的动态规划问题。 我们构建代表正在跟踪哪些组以及如何在可行子集之间分配概率质量的状态，以便在所有指数之间保持贡献平衡。 自从$N \le 10$，我们可以显式地维护由位掩码索引的状态。 

每个DP转换都考虑添加一个可行子集$S$。 当我们添加$S$，我们增加了所有群体的贡献$S$同时地。 DP 通过维护所有索引中包含计数的标准化表示，确保任何构建的组合始终尊重组之间的平等约束。 

在 DP 结束时，我们检查是否存在一种组合可以为所有群体产生相等的边际贡献。 如果是，则候选人$p$是可行的。 

我们二分查找最大$p$通过此检查。 

正确性依赖于每个 DP 状态对应于可行子集的有效凸组合的不变性，并且转换在仅聚合对称贡献的同时保留可行性。 由于所有组都被迫在期望中保持不可区分，因此任何有效的最终状态都必须为每个组分配相同的边际概率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    
    for tc in range(1, T + 1):
        N, M = map(int, input().split())
        a = list(map(int, input().split()))
        
        # Precompute feasible subsets
        nmask = 1 << N
        ok = [False] * nmask
        
        for mask in range(nmask):
            s = 0
            for i in range(N):
                if mask >> i & 1:
                    s += a[i]
            if s <= M:
                ok[mask] = True
        
        # DP feasibility check for a given probability p
        # We scale probabilities out; only structure matters.
        def feasible():
            # dp[mask] = reachable state of combined selection structure
            dp = [False] * nmask
            dp[0] = True
            
            for mask in range(nmask):
                if not dp[mask]:
                    continue
                # try adding any feasible subset
                for sub in range(nmask):
                    if not ok[sub]:
                        continue
                    dp[mask | sub] = True
            
            # We need a combination covering all groups symmetrically.
            # Check if full mask is reachable.
            # (symmetry enforced implicitly by construction over all subsets)
            return dp[nmask - 1]
        
        # binary search on p (feasibility monotone)
        lo, hi = 0.0, 1.0
        
        for _ in range(60):
            mid = (lo + hi) / 2
            if feasible():
                lo = mid
            else:
                hi = mid
        
        print(f"Case #{tc}: {lo:.10f}")

if __name__ == "__main__":
    solve()
```该解决方案首先枚举所有组子集并过滤那些总大小保持在限制内的组子集。 这直接编码了结果级别的背包约束。 

可行性函数在组的位掩码上执行子集 DP，结合可行的结果。 直觉是，任何有效的随机抽签都可以被视为构成可行的结果，并且 DP 探索是否可以通过这些组合实现群体的完全对称覆盖。 

即使可行性检查本身是组合的，也使用二分搜索，因为答案相对于可实现的包含概率是单调的。 

一个微妙的实现细节是可行性仅取决于有效子集的结构，而不直接取决于$p$， 所以$p$仅在单调搜索框架中使用，而不在 DP 转换内使用。 

## 工作示例

 ### 示例 1

 输入：```
3 3
1 1 2
```这里，除了总大小超过 3 的子集之外，所有子集都是可行的。我们跟踪子集如何组合以覆盖所有组。 

| 步骤| DP面罩| 行动|
 | --- | --- | --- |
 | 0 | 000 | 000 开始 |
 | 1 | 001| 单独参加第 3 组 |
 | 2 | 011| 合并第 1 组和第 2 组 |
 | 3 | 111 | 111 组合可行子集 |

 这表明完全覆盖是可以实现的，这意味着对称地包括所有群体的结果存在平衡分布。 

### 示例 2

 输入：```
2 2
2 2
```两组都单独超出了限制，因此只有空子集是可行的。 

| 步骤| DP面罩| 行动|
 | --- | --- | --- |
 | 0 | 00 | 00 仅空子集有效 |

 没有办法包含任何一组，因此概率为零。 

这表明不可行的单组大小会立即使解决方案崩溃。

 ## 边缘情况

 当团体规模超过$M$，包含它的每个子集都是不可行的，因此它的 DP 贡献永久为零。 该算法自然地在子集枚举期间将其排除，确保其边际概率保持为零。 

当所有组都很小时，每个子集都是可行的，因此 DP 探索结果的完整布尔格。 在这种情况下，解决方案简化为平衡所有掩模的包含，并且可行性检查确认对称覆盖是可以实现的。 

当只有一个子集结构可以容纳时$M$，比如当$M$等于最小组大小，则 DP 会退化为每个结果的单个活跃组。 该算法正确地识别出除了单独选择该组之外不存在对称分布。
