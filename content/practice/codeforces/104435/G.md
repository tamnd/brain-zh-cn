 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中顶点代表事件，有向边代表允许的时间转换。 从事件A，如果存在从A到B的有向路径，我们就可以到达事件B。问题仅集中在“不可逆可达性”，即A可以到达B但B不能到达A的情况。在强连接区域中，一切都是相互可达的，因此在这样的区域内没有任何东西相对于自身是不可逆的。 

当我们查看所有以某个固定事件 X 结束但从可达性意义上严格“之前”X 的节点开始的有向路径时，就会出现有趣的结构。 如果每对这样的路径共享至少一个公共有向边，则事件 X 被认为是安全的。 如果我们可以找到两种不同的方式到达 X，并且两条路径不共享任何边，那么 X 就是有问题的。 

任务是删除尽可能少的有向边，以便每个节点在这个意义上都变得安全。 

输入图可能很大，测试用例中最多有 3 × 10^5 个节点和 4 × 10^5 条边。 这立即排除了任何尝试枚举路径、比较路径对或对每个节点执行流计算的解决方案。 每个测试用例的任何二次甚至接近二次都会失败。 

一些边缘情况值得牢记。 

如果图已经是一棵指向某个根的树，则每个节点都只有一种从上面到达的方式，因此不需要删除。 

如果一个节点有多个早期分叉的独立传入路由，例如两条不相交的路径在 X 处合并，那么 X 就会变得无效，除非我们删除足够的边以强制所有路由通过单个强制边进行漏斗。 

循环不会创建不可逆结构，因为在强连接组件内，每个节点都可以到达其他节点，因此这些节点永远不会处于严格的单向关系。 如果我们不压缩 SCC，我们就有可能过度计算冗余的内部结构。 

## 方法

 直接尝试是检查每个节点 X，枚举到 X 的所有传入路径，并检查其中两个是否可以边不相交。 这很快就会变成路径爆炸问题。 即使计算所有简单路径在最坏的情况下也是指数级的，所以这是不可行的。 

关键的结构观察是“到 X 的所有路径共享公共边”这一条件相当于说到 X 的传入路由集具有单个不可避免的瓶颈边。 如果我们从流的角度来看该图，这相当于说 X 的传入结构的行为类似于所有路径都强制通过单个边链的结构，因此在边不相交的意义上不允许分支。 

现在考虑是什么创建了两条到 X 的边不相交路径。当 X 至少有两条在到达 X 之前分叉的“独立”传入路由时，就会发生这种情况。一旦不相交的上游结构支持进入 X 的两个不同的传入边，我们立即得到两条边不相交的路径。 

这表明了一种简化：对于每个节点，我们希望消除除一个有效传入边之外的所有传入边，因为拥有两个保留的传入边已经可以实现路径的分歧。 如果最终图中每个节点至多有一个传入边，则到 X 的任何路径都被迫遵循单个传入边链，因此所有此类路径共享该链上的每个边。 

因此，问题简化为选择边的子集，使得每个节点最多保留一条传入边，同时删除尽可能少的边。 

这是每个节点的本地优化。 如果一个节点在原始图中的入度为 k，我们最多可以保留其中一条边，并且必须删除剩余的 k − 1。对所有节点求和即可得到最小删除次数。

强连接组件不会改变答案，因为在 SCC 内部，每个节点都是相互可达的，因此“不可逆路径”条件永远不会在内部适用。 收缩 SCC 仅删除不相关的内部循环，并在生成的 DAG 上留下相同的入度逻辑。

 | --- | --- | --- | --- |
 | 暴力破解路径比较 | 指数| O(n + m) | 太慢了|

 ## 算法演练

 1. 将图分解为强连通分量。 每个 SCC 被视为单个节点，因为循环内的内部可达性不会影响不同组件之间的不可逆路径。 
2. 构建压缩图，其中每个 SCC 都是一个节点，不同 SCC 之间的每条边都成为有向边。 
3. 对于压缩图中的每个节点，计算其入度，计算有多少条不同的传入边从其他组件到达。 
4. 对于入度为 k 的节点，如果 k ≥ 1，则仅保留一条入边，并删除所有其他入边。 这保证了没有节点接收到多个独立的入口点，这些入口点可能会创建边不相交的传入路径。 
5. 对所有节点求和删除的边数，即 max(0, in Degree − 1)。 

### 为什么它有效

 关键的不变量是，处理每个节点后，结果图中的每个节点最多有一个传入边。 这迫使通向任何节点的所有路径合并成单个传入边链。 因此，终止于同一节点的任何两条路径必须共享进入该节点的最后一个传入边，并且还递归地共享所有上游强制边。 由于在边缘级别消除了分支，因此不可能构造到任何节点的两条边缘不相交的路径。 

相反，如果节点有两个或多个剩余的传入边，那么这些边已经提供了两个不同的入口路由，并且因为 SCC 收缩消除了内部循环，所以这些入口路由对应于真正不同的上游结构，这些结构可以扩展到边不相交的路径。 因此，保留多个传入边正是造成发散的原因，并且删除除一条以外的所有边既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def kosaraju(n, g, gr):
    visited = [False] * n
    order = []

    def dfs(v):
        visited[v] = True
        for to in g[v]:
            if not visited[to]:
                dfs(to)
        order.append(v)

    def rdfs(v, comp_id):
        comp[v] = comp_id
        for to in gr[v]:
            if comp[to] == -1:
                rdfs(to, comp_id)

    for i in range(n):
        if not visited[i]:
            dfs(i)

    comp = [-1] * n
    cid = 0
    for v in reversed(order):
        if comp[v] == -1:
            rdfs(v, cid)
            cid += 1

    return comp, cid

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        g = [[] for _ in range(n)]
        gr = [[] for _ in range(n)]
        edges = []

        for _ in range(m):
            a, b = map(int, input().split())
            a -= 1
            b -= 1
            g[a].append(b)
            gr[b].append(a)
            edges.append((a, b))

        comp, c = kosaraju(n, g, gr)

        indeg = [0] * c

        for a, b in edges:
            ca, cb = comp[a], comp[b]
            if ca != cb:
                indeg[cb] += 1

        ans = 0
        for x in indeg:
            if x > 1:
                ans += x - 1

        print(ans)

if __name__ == "__main__":
    solve()
```该实现首先使用 Kosaraju 算法压缩强连通分量，这确保循环不会干扰入度结构。 压缩后，每条边仅连接不同的组件。 

然后我们计算每个组件的传入边缘。 由于我们允许每个组件最多保留一个传入边，因此必须删除第一个之外的每个额外传入边，从而为答案提供精确的入度 − 1 。 

一个常见的错误是跳过 SCC 收缩。 这会错误地将内部循环边缘计算为多个传入替代方案，从而夸大答案。 另一个微妙之处是，同一对组件之间的多个边必须独立计数，因为每个边都代表一个单独的过渡，可能有助于形成独立的路径。 

## 工作示例

 ### 示例 1

 输入：```
3 2
1 2
2 3
```没有循环，因此每个节点都是其自己的组件。 入度为：

 | 组件| 入度 |
 | --- | --- |
 | 1 | 0 |
 | 2 | 1 |
 | 3 | 1 |

 入度不超过 1，因此不会删除任何边。 

输出：```
0
```这符合图形已经是一个简单链的直觉，因此到任何节点的所有路径都被迫通过相同的边。 

### 示例 2

 输入：```
4 3
1 2
2 3
1 3
```SCC 收缩后（不存在循环），我们计算入度：

 | 节点| 传入边缘 |
 | --- | --- |
 | 1 | 0 |
 | 2 | 1 |
 | 3 | 2 |

 节点 3 有两个传入边，这意味着有两种独立的方式可以到达它。 我们必须移除其中之一。 

输出：```
1
```这演示了关键的故障模式：直接边 1 → 3 创建了一条绕过 2 → 3 的替代路线，从而产生发散。

 ## 边缘情况

 包含纯有向循环的图在 SCC 收缩后应产生零移除。 每个节点都合并为一个组件，不留下组件间的边进行比较。 

具有来自同一前驱结构的多个并行传入边的节点在压缩后仍算作多个传入边，因为每个边代表一个不同的转换，可以形成进入该节点的单独路径。 

许多节点指向单个节点的星形图会在该节点处产生较大的入度，并且该算法正确地删除除其中一条边之外的所有边，从而确保进入中心的所有剩余路径都强制通过单个入口路由。
