 ## 解决方案
 ## 问题理解

 几何形状从由两个生成向量描述的固定平行四边形开始。 如果我们将原点表示为点 A，则形状内的任何点都可以使用沿这些向量的两个参数唯一地写入。 一个向量从 A 到 B，另一个向量从 A 到 D，因此每个点都是这两个方向的线性组合，系数在 0 到 1 之间。 

区域内的运动有两种模式。 第一个是平面内的普通欧几里得运动，在映射回坐标后以通常的方式测量。 第二种是沿着平行四边形相对边的特殊标识进行瞬时传送。 一对相对的侧面以直接方式粘合，保持沿一个方向对齐。 另一对以翻转的方式粘合，反映形状中心的位置。 

任务是当允许您将正常的欧几里得运动与这些零成本边界跳跃混合时，计算该区域内两点之间的最短可能旅行时间。 

多达十万个测试用例的约束迫使每个用例在预处理几何图形后必须在恒定的时间内求解。 任何试图模拟运动或构建许多可能的中间状态的精细几何图的方法都会太慢，因为即使每次测试几千个状态也已经超出了可接受的限制。 

当点位于识别的边缘附近或跨过识别的边缘时，就会出现一个微妙的问题。 仅考虑端点之间的直接欧几里得距离的天真的最短路径尝试将失败，因为远距传态可能会创建更短的缠绕路径。 另一种失败模式是假设仅存在一个环绕方向，而实际上一个标识保留方向，另一个标识翻转它，从而生成目的地的多个有效图像。 

## 方法

 直接方法会尝试将区域建模为连续图，其中每个边界点根据虫洞规则连接到另一个边界点，然后在离散化上运行最短路径算法。 即使我们精细地离散化，每个测试也需要许多状态和边，并且复杂性会爆炸，因为在如此密集的几何图中的最短路径计算远远超出了我们对 10^5 个独立查询的计算能力。 

关键的观察是平行四边形结构将整个系统转变为线性坐标空间。 如果我们在向量 AB 和 AD 形成的基上表示点，则每个点都有坐标 (a, b)。 在此坐标系中，欧几里德距离不是轴对齐的，但它仍然是从基向量导出的固定二次形式。 

虫洞变成了这个坐标系中的简单变换。 第一个识别使得 b 坐标具有周期 1 的周期，因为 AB 与 DC 没有反转地连接。 第二个标识连接 a = 0 和 a = 1，但围绕中心翻转 b 坐标，因此 (0, b) 映射到 (1, 1 − b)。 这两种对称性生成任意点的一组有限的等效“图像”。 

我们可以枚举由这些变换引起的目标的所有有效图像，并计算从源到原始平面中每个图像的欧几里得距离，而不是搜索路径。 最短的行进时间是这些距离中的最小值，因为任何有效路径都可以展开到通用封面中的这些图像位置之一，并且欧几里得运动在每个展开的副本中都是最佳的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力几何图| 每个查询的 O(N log N) 或更差 | O(N) | 太慢了|
 | 坐标基础上的图像枚举| 每个查询 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

我们首先使用平行四边形基础重写所有点。 令 A 中的向量为 u = (x1, y1) 和 v = (x2, y2)。 任意点P表示为A+a·u+b·v。 我们通过反转 u 和 v 定义的 2×2 系统来求解 (a, b)。 

一旦源和目标都被表示为（a，b）坐标，我们就构建了这两个识别产生的目标的所有候选图像。 

AB 和 DC 之间的直接识别使 b 具有周期性，因此一个候选者是 (at, bt + 1)，另一个候选者是 (at, bt − 1)，对应于向上或向下缠绕该接缝。 

AD 和 BC 之间的翻转标识将 (a, b) 映射到 (1 − a, 1 − b)，并将其与周期性 b 移位相结合产生额外的有效图像。 

然后，我们使用基向量将每个候选转换回笛卡尔坐标，并计算平方欧几里德距离，以避免中间步骤中的浮点开销。 

最后，我们取所有候选值中的最小值并输出其平方根。 

正确性来自于这样一个事实：每个允许的运动要么停留在基本单元内，要么通过两个标识之一跨越边界。 每个交叉点都精确对应于进入平面的平移或反射副本。 因此，任何最佳路径都对应于这些展开副本之一中的直线段，并且我们只需要测试其中的有限多个。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        x1, y1, x2, y2, xs, ys, xt, yt = map(float, input().split())

        # basis vectors
        ux, uy = x1, y1
        vx, vy = x2, y2

        # solve for (a, b) in basis: P = a*u + b*v
        det = ux * vy - uy * vx

        def to_ab(x, y):
            a = (x * vy - y * vx) / det
            b = (ux * y - uy * x) / det
            return a, b

        def to_xy(a, b):
            return a * ux + b * vx, a * uy + b * vy

        sa, sb = to_ab(xs, ys)
        ta, tb = to_ab(xt, yt)

        # candidate transforms of target in (a,b)
        cands = [
            (ta, tb),
            (ta, tb + 1),
            (ta, tb - 1),
            (1 - ta, 1 - tb),
            (1 - ta, 2 - tb),
            (1 - ta, -tb),
        ]

        sx, sy = xs, ys

        ans = float('inf')

        for a, b in cands:
            tx, ty = to_xy(a, b)
            dx = sx - tx
            dy = sy - ty
            ans = min(ans, dx * dx + dy * dy)

        print(ans ** 0.5)

if __name__ == "__main__":
    solve()
```该实现首先将所有内容转换为由平行四边形边定义的坐标系，这使得隐形传态规则变成简单的代数变换而不是几何交集。 行列式用于反转基，这是唯一的数字敏感步骤，因此所有进一步的逻辑都是稳定的。 

每个候选对应于将目的地提升到基本区域的相邻副本的不同方式。 我们测试所有这些，因为最短路径可能在每个方向上穿过接缝零次或一次，具体取决于点的相对位置。

反射很重要的第二个例子：a = 0 附近的源和 a = 1 附近的目标通常更喜欢翻转图像，因为它将在反射副本中将长水平遍历变成短水平遍历。

 ## 边缘情况

 当平行四边形几乎退化时，就会发生一种微妙的情况，这意味着基向量的行列式非常小。 在这种情况下，朴素浮点求逆变得不稳定，但从数学上讲，映射仍然有效，因为该问题保证了有效的非共线平行四边形。 

另一个重要的情况是当两个点恰好位于接缝上时。 在这种情况下，多个候选图像重合，并且该算法仍然有效，因为候选集中的重复图像不会影响最小计算。 

第三种情况是最优路径恰好穿过边界一次。 对于此类输入，只有一幅变换后的目标图像会产生正确的最小距离，而所有其他图像都会产生严格更大的值。 这确保了限制到一组恒定的变换不会错过真正的最佳值。
