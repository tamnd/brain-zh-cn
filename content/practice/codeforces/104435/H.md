 ## 解决方案
 ## 问题理解

 我们有几根棍子，每根棍子的长度都是整数，我们必须将它们组织成几何结构，最终产生一个三角形的庇护所。 这些木棍首先被分成两组，每组通过将木棍首尾相连而形成一根梁，因此每根梁的行为就像一个单独的段，其长度是其分配的木棍的总和。 

形成两根梁后，我们将每根梁的一个端点连接到地面上方的公共铰接点。 两根光束的另一端点位于地线上。 这保证了最终的形状是一个三角形，其两条边恰好是梁的长度，第三条边是两个地面接触点之间的距离。 我们可以自由选择顶部铰链的角度，因此唯一真正的自由是我们如何将棍子划分为两个梁长度。 

一个关键的几何简化来自这样一个事实：对于两个固定边长，当它们之间的角度为 90 度时，三角形面积最大化。 在这种情况下，面积变为梁长度乘积的一半。 由于所有棒长度的总和是固定的，因此最大化光束和的乘积相当于使一根光束尽可能接近总和的一半。 

不同的是，在分割之前，我们必须选择一根棍子并将其分割成两个正整数部分。 这不会改变总和，但会改变我们调整子集和的精细程度。 

就棒的数量而言，约束很小，每个测​​试用例最多 35 个，但值很大，最多 1e8。 这立即排除了任何对总和的动态规划。 唯一可行的方向是对大约 2^17 个状态进行中间相遇子集枚举。 

一个天真的错误是认为劈开棍子是无关紧要的，因为它保留了总和。 这是错误的，因为拆分增加了组合灵活性：我们获得了 0 到 xi 之间连续范围的可实现贡献，而不是对一项进行二元选择。 这使得可实现的接近度显着降低了一半。 

另一个微妙的失败案例是假设贪婪分区有效，例如总是将最大的木棍放入较小的一侧。 这在近乎完美的平衡需要组合许多中等元素而不是单个大修正的情况下会被打破。 

## 方法

 强力策略将尝试将一根棍子的每一种可能的分割以及将所有棍子的每一种可能的划分分成两组，计算光束总和并评估所得面积。 对于每个分割索引 i，我们将尝试所有整数对 a、b，其中 a + b = xi，并且对于每个结果多重集，枚举所有 2^n 分配。 如果每个配置枚举所有 a、b 和 2^35 分区，这会导致 i 大约有 35 个选择，最坏的解释是最多 10^8 个分区。 即使忽略拆分枚举，2^35 也已经远远超出了可行的限制。 

关键的观察结果是，面积仅取决于一束光束的总和，而不取决于其组成。 如果总和为 S，且一束光束的总和为 x，则面积与 x(S − x) 成正比，当 x 最接近 S/2 时面积最大化。 该问题变成了具有单一受控修改的子集和紧密度问题：一个元素可以被两个灵活部分替换，这两个灵活部分的总和是固定的，但可以在两侧之间任意分割。 

如果没有分割，这是一个经典的中间相遇子集和：枚举一半数组的所有子集和并选择最接近 S/2 的子集和。 拆分将一个元素从严格的选择更改为范围贡献者，有效地允许该元素在连续间隔内将子集总和调整为任意整数。 这意味着我们只需要尝试每个可能的分割元素并有效地解决修改的子集和问题。

因此，我们固定要分割的元素，将其删除，使用中间相遇计算剩余元素的子集和，然后确定当允许将所选子集和移动 [0, xi] 中的任何整数时，我们可以接近 S/2。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对分区和分割进行全面的暴力破解 | O(n·2^n) | O(n·2^n) | O(2^n) | O(2^n) | 太慢了|
 | 中间会面并尝试每次拆分 | O(n·2^(n/2)) | O(n·2^(n/2)) | O(2^(n/2)) | O(2^(n/2)) | 已接受 |

 ## 算法演练

 我们固定了一个候选棒 i，我们决定将其分开。 所有其他棒保持不变，我们将它们的总和表示为R。 

然后我们考虑在没有这根棍子的情况下子集和的表现。 使用中间相遇，我们将剩余元素分成两半，枚举每一半的所有子集和，并将它们合并到所选元素所有可能和 S 的排序列表中。 

对于每个这样的 S，我们观察到分裂棒 xi 可以分布在两个光束之间。 如果我们将其中的 t 放入第一束，则其余的 xi − t 将进入第二束。 这意味着该棒的有效贡献允许我们将子集和 S 向第一束移动区间 [0, xi] 中的任何整数值。 

这将每个子集和 S 转换为第一波束和的可达区间 [S, S + xi]。 我们的目标是尽可能接近总和 T/2 的一半，因此我们希望间隔能够最小化到该目标的距离。 

我们扫描所有子集和 S 并计算区间 [S, S + xi] 与 T/2 的接近程度。 我们跟踪实现最小偏差的最佳对 (i, S, t)。 

选择最佳配置后，我们从中间相遇结构重建子集并相应地分配元素。 对于分裂棒，我们将 t 分配给第一根光束，将 xi − t 分配给第二根光束。 

### 为什么它有效

 几何形状将目标简化为最大化 S1(T − S1)，这仅取决于 S1 与 T/2 的接近程度。 分裂棒不会改变总和，但它将一个离散决策转变为子集和的连续区间调整。 每个有效的构造都对应于选择一个子集 S 和一个分割 t，并且每个这样的选择恰好映射到一个波束和。 因此，搜索所有子集和以及来自分裂棒的所有可行移位可以覆盖整个解空间，而不会丢失任何配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def meet_in_middle(arr):
    n = len(arr)
    half = n // 2
    left = arr[:half]
    right = arr[half:]

    def gen(a):
        res = []
        m = len(a)
        for mask in range(1 << m):
            s = 0
            for i in range(m):
                if mask & (1 << i):
                    s += a[i]
            res.append((s, mask))
        return res

    L = gen(left)
    R = gen(right)

    sums = {}
    for s1, m1 in L:
        for s2, m2 in R:
            s = s1 + s2
            if s not in sums:
                sums[s] = (m1, m2)
    return sums

def solve_case(n, arr):
    total = sum(arr)
    best_diff = float('inf')
    best = None  # (i, S, subset_mask, split_t, side_choice)

    full_indices = list(range(n))

    for i in range(n):
        xi = arr[i]
        others = arr[:i] + arr[i+1:]

        sums = meet_in_middle(others)

        # target for first beam
        target = total / 2

        for S, (m1, m2) in sums.items():
            # interval [S, S+xi]
            if S <= target <= S + xi:
                diff = 0
                t = int(target - S)
            else:
                d1 = abs(S - target)
                d2 = abs(S + xi - target)
                if d1 <= d2:
                    diff = d1
                    t = 0
                else:
                    diff = d2
                    t = xi

            if diff < best_diff:
                best_diff = diff
                best = (i, S, m1, m2, t, xi)

    i, S, m1, m2, t, xi = best
    target_mask = (m1, m2)

    # reconstruct beams
    beam1 = []
    beam2 = []

    idx = 0
    for j in range(n):
        if j == i:
            continue
        if idx < len(arr[:i]):
            bit = (m1 >> idx) & 1
        else:
            bit = (m2 >> (idx - len(arr[:i]))) & 1

        if bit:
            beam1.append(j + 1)
        else:
            beam2.append(j + 1)

        idx += 1

    # split stick i
    a = t
    b = xi - t

    # assign split parts
    if a > 0:
        beam1.append(n + 1)
    if b > 0:
        beam2.append(n + 2)

    S1 = S + a
    S2 = total - S1
    area = 0.5 * S1 * S2

    return i + 1, a, b, beam1, beam2, area

def main():
    T = int(input())
    for _ in range(T):
        n = int(input())
        arr = list(map(int, input().split()))
        i, a, b, b1, b2, area = solve_case(n, arr)

        print(i, a, b)
        print(*b1)
        print(*b2)
        print(f"{area:.12f}")

if __name__ == "__main__":
    main()
```该解决方案的结构是评估分裂棒的每种可能选择，然后使用中间相遇解决剩余元素的约束子集和问题。 重建步骤跟踪子集掩码，以便我们可以明确输出哪些棒属于每个光束。 通过将所选的移位 t 转换为两个新块的实际分配来单独处理分割棒。 

实现中的一个微妙之处是，我们必须将分割棒视为引入连续调整，但在重构时，我们将其转换回索引 n+1 和 n+2 的离散分配。 

## 工作示例

 考虑一个带有棍子的小输入[6,7,6]。 最佳策略是将第一根棍子分成 3 和 3，总共 16 个。最好的分区平衡 8 和 8 处的梁，产生最大的产品。 

| 步骤| 选择拆分 | 子集和 S | 调整t | 光束 1 总和 | 光束 2 总和 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 将 6 分成 3,3 | 7 | 1 | 8 | 8 |

 该轨迹显示了分割如何实现精确的平衡，而如果只允许整根木棍，这是不可能实现的。 

现在考虑[5,8,2,6]。 总数为 21，目标为 10.5。 如果不进行分割，最接近的子集总和可能会达到 10 或 11，但假设我们选择一个配置，给出 S = 9 且 xi = 2。则区间为 [9, 11]，其中包含 10.5，因此我们可以在分割后实现完美平衡。 

| 步骤| S | 习 | 间隔 | 目标| 结果 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 9 | 2 | [9, 11] | 10.5 | 10.5 准确命中|

 这说明了分割的重要性：它将离散子集总和转换为重叠间隔，从而允许与中点精确或接近精确对齐。

 ## 边缘情况

 当最佳解决方案不需要以“对称”方式分割任何棒，而是纯粹使用分割来弥补子集和中的差距时，就会出现关键的边缘情况。 在这种情况下，最佳 t 变为 0 或 xi，有效地将分裂棒恢复为正常项目。 该算法自然地处理这个问题，因为区间评估总是考虑端点 S 和 S + xi，因此它从不假设必须以平衡的方式使用分割。 

另一种边缘情况是当多个子集和达到相同的最小距离一半时。 重建仍然有效，因为任何这样的子集都会产生相同的目标值，并且波束分配仅取决于所选的掩模，而不取决于解决方案的唯一性。
