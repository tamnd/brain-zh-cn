 ## 解决方案
 ## 问题理解

 每个测试用例包含两个小写单词的集合。 

第一个集合是 Wah-List，其中包含加密消息解码后必须出现的每个单词。 

第二个集合是加密消息本身。 每个单词都使用相同的凯撒密码移位进行加密。 在加密过程中，每个字母都向前移动了`k`字母表中的位置，环绕在后面`z`。 为了解码，我们必须将每个字母向后移动相同的量。 

我们的任务是找到最小的**正**偏移值`k`这样，在解码整个消息之后，哇音列表中的每个单词都会出现在解码消息中的某个位置。 如果没有正移满足这个条件，我们输出`-1`。 

字母表仅包含 26 个字母，因此只有 26 个不同的凯撒移位。 由于该问题只允许正值`k`，我们只需要测试从`1`通过`25`。 的转变`26`将再次产生原始消息，相当于`0`，所以它永远不可能是最小的正答案。 

消息长度最多为 1500 个字符，Wah-List 最多包含 20 个单词，长度最多为 50 个。即使对每个可能的移位解码整个消息也只需要大约`25 × 1500 = 37500`每个测试用例的字符操作，非常小。 

一种微妙的情况是，多次转变会产生有效的解码消息。 例如，```
Wah-List:
aaa

Message:
bbb ccc ddd
```轮班`1`,`2`， 和`3`全部产生一条消息，其中包含`aaa`。 正确答案是`1`因为我们必须返回最小的正偏移，而不仅仅是任何有效的偏移。 

另一个容易犯的错误是忘记了移位是围绕字母表的。```
Wah-List:
xyz

Message:
abc
```解码用`k = 3`轮流`abc`进入`xyz`。 简单地从每个字符中减去三个而不换行会产生无效字母。 

第三个陷阱是错误地处理重复的消息词。 考虑```
Wah-List:
cat dog

Message:
fdw fdw grj
```使用正确的移位解码后，消息变为```
cat cat dog
```重复项并不重要。 我们只需要知道每个所需的单词是否至少出现一次，因此将解码后的单词存储在一个集合中就足够了。 

## 方法

 最直接的解决办法是尝试一切可能的积极转变。 对于每个班次，解码消息中的每个单词，将解码的单词收集到一个集合中，并检查每个 Wah-List 单词是否属于该集合。 由于只有 25 个候选班次，因此这种方法已经非常高效。 

假设我们忽略了字母表只有 26 个字母的事实，而是搜索任意大的值`k`。 这显然是不可能的，因为存在无限多个转变。 关键的观察结果是凯撒密码移位每 26 个位置重复一次。 移位 27 与移位 1 完全相同，移位 52 与移位 0 完全相同，依此类推。 这种周期性将搜索空间减少到只有 25 个有意义的候选者。 

一旦我们认识到这一点，就不需要更复杂的算法，例如散列、尝试或字符串匹配。 解码每条候选消息已经完全在限制之内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解任意班次 | 无界| O(1) | O(1) | 不可能|
 | 尝试全部 25 种凯撒班次 | O(25 × L) | O(M)| 已接受 |

 这里，`L`是加密消息中的字符总数，`M`是消息中的字数。 

## 算法演练

 1. 读取哇音表并将其单词存储在一组中。 使用集合使得以后的成员资格检查时间恒定。 
2. 以单词列表的形式读取加密消息。 
3. 对于每个班次`k`从`1`到`25`，通过将每个字符向后移动来解码每个消息字`k`模 26 算术的位置。 
4. 将每个解码后的单词插入一个集合中。 重复的解码单词是无关紧要的，因为我们只关心所需的单词是否至少出现一次。 
5. 检查每个 Wah-List 单词是否属于解码单词集合。 
6. 一旦一个班次满足条件，则输出该班次并停止搜索。 由于班次是按升序测试的，因此第一个有效的班次自动是最小的肯定答案。 
7、如果25次移位都没有成功，则输出`-1`。 

### 为什么它有效

 每个可能的凯撒密码都是由其模 26 的移位唯一确定的。测试移位来自`1`通过`25`准确地覆盖每个不同的正密码一次。 对于固定移位，解码会准确地重建在使用该移位加密之前存在的消息。 当解码后的消息中出现每个所需的单词时，算法会准确地接受。 由于班次是按升序检查的，因此第一个成功的班次保证是最小的有效答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def decode_word(word, shift):
    res = []
    for c in word:
        x = (ord(c) - ord('a') - shift) % 26
        res.append(chr(ord('a') + x))
    return "".join(res)

def solve():
    t = int(input())

    for _ in range(t):
        n = int(input())
        wah = set(input().split())

        m = int(input())
        message = input().split()

        answer = -1

        for shift in range(1, 26):
            decoded = set()

            for word in message:
                decoded.add(decode_word(word, shift))

            if wah.issubset(decoded):
                answer = shift
                break

        print(answer)

if __name__ == "__main__":
    solve()
```该解决方案首先将哇音列表存储在一个集合中，因为成员资格测试是我们唯一需要的操作。 每个候选轮班都是独立处理的。 

辅助函数执行凯撒解码。 每个字母都转换为一个值`0`到`25`，减去移位，模算术自动处理环绕，并将结果转换回字符。 

对于每个班次，每个消息字都被解码一次并插入到一个集合中。 使用集合自然会忽略重复项，同时提供恒定时间的查找。 子集测试直接回答是否每个所需的单词都存在。 

找到第一个有效班次后，循环立即停止。 由于班次是按升序检查的，这保证了最小的肯定答案。

import sys
import io

def run(inp: str) -> str:
    from contextlib import redirect_stdout

    sys.stdin = io.StringIO(inp)
    out = io.StringIO()

    def decode_word(word, shift):
        return "".join(chr((ord(c) - 97 - shift) % 26 + 97) for c in word)

    def solve():
        input = sys.stdin.readline
        t = int(input())
        for _ in range(t):
            n = int(input())
            wah = set(input().split())
            m = int(input())
            msg = input().split()

            ans = -1
            for k in range(1, 26):
                dec = {decode_word(w, k) for w in msg}
                if wah.issubset(dec):
                    ans = k
                    break
            print(ans)

    with redirect_stdout(out):
        solve()

    return out.getvalue()

 ## 边缘情况

 考虑```

```唯一使消息保持不变的转变是`0`，但只允许正向偏移。 该算法检查班次`1`通过`25`，发现没有有效的解码，并正确输出`-1`。 

现在考虑```
1
1
xyz
1
abc
```当算法达到移位时`3`，每个字符均使用模 26 算术进行解码。 这个词`abc`变成`xyz`，满足哇音表。 这证实了环绕已正确实现。 

最后，考虑```
1
2

3
fdw fdw grj
```带班次`3`，解码后的消息是`cat cat dog`。 解码后的字集变为`{cat, dog}`尽管重复出现`cat`。 子集测试成功，表明消息中的重复项不会影响正确性。
