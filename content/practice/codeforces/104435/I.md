 ## 解决方案
 ## 问题理解

 我们正在使用精确地在无限网格上形成的形状$k$单位平方。 每个形状必须通过边邻接连接，这意味着只有当它们共享一个完整的边时，我们才能在方块之间移动。 触摸角落无助于连接，移动时会被忽略。 

如果两个形状可以旋转或平移以匹配另一个，则认为两个形状是相同的。 镜面反射被视为潜在的不同，因此翻转形状可以产生仍然算作新对象的独特配置。 

这里关键的隐藏对象不是我们输出的网格，而是所有不同大小的多联骨牌的集合$k$。 任务是决定我们是否可以构建一个矩形网格，其中每个这样的多联骨牌都作为相等字母的连通分量出现在某处，并且每个连通分量恰好对应于一个有效的大小形状$k$。 网格本身只是一个容器； 真正的要求是每一个可能的$k$- 细胞连接形状必须在某处作为单色区域存在。 

输入是单个整数$k$，最多 15 个。输出要么是任务不可能完成的声明，要么是对所有形状进行编码的构造性网格。 

约束条件$k \le 15$是最重要的信号。 自由多骨牌的数量迅速增长$k$，但仍然足够小$k \le 15$我们可以明确地组合推理它们或依赖已知的分类事实。 这是一个经典的暗示，表明多骨牌存在的暴力枚举或结构表征是预期的，而不是几何堆积。 

微妙的边缘情况出现在非常小的值$k$。 为了$k = 1$，唯一的形状是单个单元格，因此任何单字母网格都可以。 为了$k = 2$，只有一种形状，即多米诺骨牌，因此任何有效的构造都是微不足道的。 有趣的行为开始于$k = 3$，其中存在多个不同的多联骨牌。 假设“总是可能”的天真的方法会失败，因为问题不在于构造一个形状，而是同时将所有形状嵌入到单个网格中，而没有破坏连接定义的重叠约束。 

第二种失败模式是假设我们可以独立放置形状。 由于每个区域必须准确无误$k$单元格并且不能干扰其他单元格，粗心的平铺想法可能会意外地合并组件或创建无效的多联骨牌形状，因为通过相同的字母进行了意外的连接。 

## 方法

 暴力解释将尝试显式生成所有大小的多联骨牌$k$，然后尝试将它们作为不相交的连接组件打包到网格中，每个组件都标有唯一的字母。 这立即遇到两个问题。 首先，生成所有多联骨牌是指数级的$k$，甚至对于$k = 15$这个数字已经足够大，以至于枚举变得昂贵。 其次，即使我们拥有所有形状，解决每个形状必须作为连接组件恰好出现一次的填充问题也是一个受约束的平铺问题，类似于几何设置中的精确覆盖，这通常在计算上很难处理。 

关键的观察是，输出网格不需要避免不同组件之间的邻接，它只要求由相同字母形成的每个连接组件具有精确的大小$k$并对应于有效的多联骨牌。 这意味着我们并没有真正以几何方式嵌入任意形状； 我们通过字母模式对它们进行组合编码。 

这将问题简化为一个更简单的结构问题：对于哪个$k$是否存在足够多的不同连接标记模式可以实现所有多联骨牌类别？ 决定性的事实是，对于$k \ge 3$，在给定的约束下，该要求变得不可能，因为任何有限字母网格划分为连接的组件，这些组件太规则而无法同时表示所有自由多骨牌等价类，而不会在邻接结构中发生冲突。 预期的结论是只有非常小的$k$值是可行的，除此之外，多骨牌的多样性不能在单个有限网格中同时实施。 

因此，解决方案分解为简单的可行性检查$k$，然后在允许的情况下进行简单的构造。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 强力多骨牌枚举 + 打包 | 指数为$k$| 指数| 太慢了 |
 | 结构可行性分类| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取整数$k$。 所有行为仅取决于该值，因此不需要进一步的输入处理。 
2. 检查是否$k$属于可行尺寸的集合。 对于这个问题，可行性崩溃到最小的琐碎情况，其中多联骨牌集合是退化的并且不需要复杂的嵌入。 具体来说，我们对待$k \le 2$尽可能可构造，而所有更大的值都是不可能的。 
3.如果$k > 2$，输出字符串“Ignominious”并立即停止。 不可能的原因在于存在多个非全等多联骨牌，并且不能在不违反唯一性要求的情况下同时表示为有界网格中不相交的均匀连通分量。 
4. 如果$k \le 2$，构造一个最小网格。 为了$k = 1$，输出单个单元格。 为了$k = 2$，输出具有相同字母的 1 x 2 网格，形成独特的多米诺骨牌形状。 
5. 打印“Ominous”，然后打印尺寸和网格本身。 

由于不需要多个形状之间的相互作用，因此有意将结构最小化。 每个有效配置在对称性方面都是唯一的，因此不存在需要解决的组合冲突。 

### 为什么它有效

 不变量是每个相同字母的连通分量恰好对应于一个有效的多联骨牌大小$k$，并且网格恰好包含一个这样的组件，当$k \le 2$。 由于在这些情况下，多骨牌集合的大小为一，因此可以轻松满足所有形状都出现的要求。 为了$k \ge 3$，不变量会失败，因为不同的多骨牌类的数量超过了可以嵌入的数量，而不会引入意外的等价或合并结构，从而使通用嵌入变得不可能。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

k = int(input().strip())

if k > 2:
    print("Ignominious")
else:
    print("Ominous")
    if k == 1:
        print(1, 1)
        print("A")
    else:
        print(1, 2)
        print("AA")
```代码首先读取$k$并立即就可行性进行分支。 首先处理不可能的情况，以避免构建任何不必要的东西。 

为了$k = 1$，网格是一个单元格，它简单地形成了唯一可能的多联骨牌。 为了$k = 2$，我们输出一个两单元水平条，这是旋转之前唯一连接的两单元形状。 

只要正确表示连通性，字母选择就无关紧要； 单个重复字符可确保整个组件是一个整体。

 ## 边缘情况

 对于$k = 1$，算法输出单个单元网格。 唯一的多骨牌是一个正方形，因此立即满足所有形状都出现的要求。 由于没有邻居，因此该施工不会引入额外的邻接问题。 

为了$k = 2$，网格变成 1 x 2 线。 两个正方形共享一条边，形成大小为 2 的有效连通分量。 不存在替代形状，因此不存在丢失配置的风险。 

为了$k \ge 3$，算法立即拒绝。 这可以避免任何构建网格的尝试，其中组件的意外合并可能会意外地形成无效或重复的形状表示。
