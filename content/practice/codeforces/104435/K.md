 ## 解决方案
 ## 问题理解

 我们得到了袜子类型的集合，其中每种类型都包含许多成对分组的相同袜子。 类型 i 恰好贡献了 2·k[i] 只袜子，在该类型中所有袜子都无法区分。 

在所有类型中，只有 m 种类型的子集被认为“适合”某个事件。 布兰克正在盲目地从一堆混在一起的袜子中抽奖，我们希望她必须拿走的袜子数量最少，这样无论抽奖多么不幸，她都保证至少能从合适的类型中抽到一双袜子。 

这里的“一双”只是指她所拿的袜子中，至少有两只属于同一类型的袜子。 

关键的困难在于，从最坏的情况来看，平局是对抗性的。 我们并不是按照预期随机抽样；而是随机抽样。 相反，我们必须假设堆是以最不方便的方式排列的，并且无论顺序如何，我们都必须保证成功。 

输入大小意味着每个测试用例最多 10^3 个类型和最多 10^2 个测试用例。 因此，任何解决方案都应该在每个测试用例的线性时间内运行。 对袜子的所有子集进行二次或组合模拟会太慢。 

当考虑从不合适的类型中取出许多袜子是有益还是有害时，会出现一个微妙的边缘情况。 由于不合适的类型对目标没有贡献，它们仅充当“安全填充”，无助于形成所需的对，但会延迟到达合适的对。 

另一个陷阱是假设我们需要全球任何类型的两只袜子。 该要求严格仅涉及合适的类型，因此来自不合适类型的配对是无关紧要的。 

## 方法

 思考这个问题的一种蛮力方法是模拟一只一只地画袜子，并跟踪我们收集到的所有可能的状态。 在每一步中，我们都会考虑堆的所有可能排列方式，并检查糟糕的排列是否仍然无法形成合适的对。 这很快就会变成组合爆炸，因为每个袜子都会分支成多种类型选择，并且我们需要推理所有序列上的对抗性安排。 即使对于中等 n 且袜子总数达到 10^8 的情况，这也是不可行的。 

关键的观察是，我们没有被问到概率或特定序列，而是被问到最坏情况的保证。 这意味着我们应该构建尽可能最长的抽签序列，但仍能避免成功。 一旦我们知道了最大失效长度，答案就是比它多一。 

为了避免形成合适的一双，对于每种合适的类型，我们最多只能从中取出一只袜子。 如果我们从相同的合适类型中选取两个，我们就已经成功了。 对于不适合的类型，即使拿走所有袜子也无法帮助形成所需的一双，因此它们可能会完全耗尽而不会触发成功。 

这将问题简化为计算在遵守这些约束的情况下可以拿走多少只袜子，然后添加一只。

 | 暴力模拟| 指数| 指数| 太慢了 |
 | 计算最坏情况下的容量 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们现在构建最糟糕的抽签，以尽可能长时间地避免成功。

1. 确定适合该活动的袜子类型。 我们明确给出了它们的索引。 
2. 计算属于所有不合适类型的袜子的总数。 这些袜子都可以在不帮助形成有效一双的情况下被拿走，所以在最坏的情况下，在我们接触任何有用的结构之前它们就被完全收集了。 
3. 对于每种合适的类型，请注意我们最多可以安全地从中取出一只袜子，而无需形成一双该类型的袜子。 从这些类型中的任何一种中取出第二只袜子都可以立即满足要求，因此对手在强制成功之前确保最多从每种合适的类型中遇到一只袜子。 
4. 总结这些贡献：所有不合适类型的袜子加上每种合适类型的一只袜子。 
5. 在仍可能避免成功的情况下，我们可以拿走的袜子的最大数量是这个总和。 因此，保证成功的最小数字就是这个值加一。 

### 为什么它有效

 该论点基于严格的对抗性结构。 最坏情况的安排总是会通过隔离合适的袜子来推迟成功，以便在抽签的初始部分中最多出现一份袜子，同时免费提供所有不合适的袜子。 这种结构饱和了避免合适一双的所有方法，这意味着任何额外的袜子都必须在合适的类型中创建一个副本，从而强制满足所需的条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        k = list(map(int, input().split()))
        types = list(map(int, input().split()))

        suitable = set(types)

        total_unsuitable = 0

        for i in range(1, n + 1):
            if i not in suitable:
                total_unsuitable += 2 * k[i - 1]

        # one sock per suitable type in worst case
        max_safe = total_unsuitable + m

        print(max_safe + 1)

if __name__ == "__main__":
    solve()
```解决方案直接遵循论证的结构。 我们首先使用一组进行 O(1) 成员资格检查来标记合适的类型。 然后，我们通过对不在集合中的索引求和 2·k[i] 来累积来自不合适类型的所有袜子。 

期限`m`表示从合适类型中安全单抽的最大数量，因为每个合适类型最多可以贡献一只袜子而不形成一对。 最后，我们添加一个来强制在合适的类型之间进行第一个不可避免的重复。 

一个常见的实现错误是仅迭代合适的类型，而忘记了不合适的类型将其所有袜子贡献给最坏情况的前缀。 另一种是错误地计算双袜子而不是单只袜子； 该堆由单独的袜子组成，因此我们始终以 2·k[i] 为单位进行操作。 

## 工作示例

 考虑一个小案例，有五种类型，但只有两种是合适的。 

我们会追踪在不强制挑选一双合适的情况下可以拿走多少只袜子。 

| 步骤| 不适合采取| 适合单打 | 总计 |
 | --- | --- | --- | --- |
 | 初始| 0 | 0 | 0 |
 | 拿走所有不合适的袜子| 6 | 0 | 6 |
 | 从每种合适的类型中取出一只袜子 | 6 | 2 | 8 |

 这表明最多 8 只袜子仍可能无法形成合适的一双。 下一只袜子会强制复制其中一种合适的类型。 

现在考虑一种所有类型都适用的情况。 

| 步骤| 不适合采取| 适合单打 | 总计 |
 | --- | --- | --- | --- |
 | 初始| 0 | 0 | 0 |
 | 每种类型带一只袜子 | 0 | 3 | 3 |

 我们最多可以从每种类型中选取一个。 第四只袜子保证一双。 

这些痕迹证实，界限仅取决于计算在重复不可避免之前可以“延迟”的类型数量。

 ## 边缘情况

 当没有合适的类型时，公式会减少为所有袜子加一只。 这反映出无论多少绘图都无法满足该条件，因此答案退化为一个微不足道的不可能守卫。 

当每种类型都合适时，该算法简化为经典的鸽巢原理：每种类型拿一只袜子可以避免成功，而下一只袜子则强制在某种类型中重复。 

当合适的类型稀疏时，所有复杂性都会崩溃，以正确解释不合适类型的全部贡献。 该算法通过对整个计数求和来直接处理此问题，确保它们不会干扰合适计数之间的“每种类型一个”约束。
