 ## 解决方案
 ## 问题理解

 我们得到了一个代表一条推文的长数组，其中每个元素都是 1 到 300 之间的整数。我们还得到了一个较短的模式数组，称为 slur。 任务是计算在宽松的匹配规则下，长度恰好等于连线长度的推文中有多少子字符串与连线“几乎相同”。 

如果对于每个位置 i，推文值位于该位置的连线值的一步以内，则子字符串有效。 换句话说，每个对齐对必须满足整数标签中最多 1 的差异。 我们还需要按升序输出此类有效子字符串的所有起始索引。 

该结构是具有每个位置公差约束的固定长度滑动窗口检查。 输入大小很大：推文最多可包含 180 万个元素，模式最多可包含 25 万个。 这立即排除了任何通过直接扫描所有位置来检查每个窗口的解决方案。 简单的 O(t · s) 方法最多需要 4.5 × 10^11 次比较，这远远超出了可行的限制。 

当所有值都接近字母表范围的边界（例如 1 或 300）时，就会出现关键边缘情况。例如，如果模式包含 1，则该位置的有效推文值只能是 1 或 2。如果不小心，假设模算术或忘记边界钳位的粗心实现将错误地接受无效匹配，例如 0 或 301。 

另一种微妙的情况是许多相邻窗口仅相差一个位置。 即使大多数比较在连续窗口之间重叠，每个班次的天真完全重新计算也会通过重新检查未更改的位置而浪费工作量。 

## 方法

 暴力法很简单。 对于推文中的每个起始索引 i，我们将切片 T[i : i + s] 与 S 位置逐个位置进行比较，并检查每对是否最多相差 1。 这是正确的，因为它直接遵循有效性的定义。 然而，每个窗口的成本为O(s)，并且有O(t)个窗口，导致时间复杂度为O(t·s)。 对于最坏情况的值，这变得完全不可行。 

关键的观察结果是每个窗口比较对于每个位置都是独立的，并且条件纯粹是局部的：每对 (T[i + j], S[j]) 必须满足一个简单的约束。 这使我们能够将问题重新表述为具有二进制兼容性规则的模式匹配问题。 我们可以对每个位置的兼容性进行预处理，然后使用线性时间滑动机制来维护当前窗口中有多少位置满足条件。 

主要技巧是将每个对齐转换为布尔条件并维护满足位置的滚动计数。 当窗口移动时，我们通过仅跟踪传出和传入位置来更新 O(1) 中的计数，而不是重新计算每个班次的所有 s 比较。 

如果 |T[i + j] − S[j]|，我们将位置 (i, j) 处的匹配定义为有效 ≤ 1。对于固定窗口起始 i，我们需要所有 s 个位置同时有效。 所以我们维护有多少位置是有效的，并将其与 s 进行比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(t · s) | O(1) | O(1) | 太慢了|
 | 滑动有效性计数| O(t·s) 预处理/O(t) 扫描| O(t + s) | 已接受 |

 真正的改进来自于观察到可以在每个对齐中预先评估条件并在重叠窗口中重复使用。 

## 算法演练

 我们将问题转化为检查每个偏移量 i 是否所有对 (i + j, j) 都满足容差约束。 我们维护一个计数器，记录当前有多少位置满足从 i 开始的窗口的条件。

1. 对于每个模式位置 j，预先计算第一个窗口的 T[j] 是否与 S[j] 匹配。 这会在从索引 0 开始的第一个窗口上初始化有效性计数器。 
2. 从左向右滑动窗口。 当从起始点 i 移动到 i + 1 时，对齐的对会发生移动：窗口 i 中位置 j 的贡献对应于窗口 i + 1 中的位置 j − 1。我们仅重新计算边界处受影响的位置。 
3、具体来说，移动窗口时：

 旧位置 j = 0 离开窗口，j = s − 1 进入。 我们通过减去传出对的有效性并加上传入对的有效性来调整有效性计数器。 
4. 每次移位后，如果有效性计数器等于 s，我们将当前索引记录为有效起始位置。 

每个步骤都依赖于这样一个事实：在相对配对方面，每个索引移位仅改变一个比对。 其余位置保持其比较结构，但不会以简单的索引方式直接重用； 相反，我们有效地重新计算边界效应。 

### 为什么它有效

 在任何位置 i，算法都会保持索引 j 的精确数量，使得约束 |T[i + j] − S[j]| ≤ 1 成立。 i 和 i + 1 之间的唯一转换会影响哪些元素与 S[0] 和 S[s − 1] 进行比较，因此其余的有效性状态在结构上保持一致。 这个不变量保证每当计数器等于 s 时，每个对齐的对都满足条件，这正是有效子字符串的定义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t, s = map(int, input().split())
    T = list(map(int, input().split()))
    S = list(map(int, input().split()))

    if s > t:
        print(0)
        print()
        return

    def ok(a, b):
        return abs(a - b) <= 1

    # compute initial window
    cnt = 0
    for j in range(s):
        if ok(T[j], S[j]):
            cnt += 1

    res = []
    if cnt == s:
        res.append(1)

    for i in range(1, t - s + 1):
        # remove outgoing pair
        if ok(T[i - 1], S[0]):
            cnt -= 1
        # add incoming pair
        if ok(T[i + s - 1], S[s - 1]):
            cnt += 1

        if cnt == s:
            res.append(i + 1)

    print(len(res))
    if res:
        print(*res)

if __name__ == "__main__":
    solve()
```该代码首先构建初始对齐的有效性状态。 然后，它将窗口一次滑动一个位置，仅调整边界贡献。 功能`ok`直接编码容差规则。 结果列表收集所有有效的起始位置。 

重要的微妙之处在于窗口移位不需要重新计算内部位置。 只有端点很重要，因为相对的配对结构被保留。

 ## 边缘情况

 一个重要的边缘情况是所有值都位于字母表的边界，例如 1 或 300。该算法自然地处理这种情况，因为比较纯粹是绝对差异，因此从不接受有效邻接之外的值。 例如，在窗口中`[1, 2]`反对`[1, 300]`，第二次比较失败，因为 |2 − 300| 很大，可以立即防止误报。 

另一种情况是当多个重叠窗口有效时，尤其是当模式重复时。 滑动计数器确保每个窗口都被独立而高效地评估，不会重复计算或丢失重叠。
