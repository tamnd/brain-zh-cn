 ## 问题理解

 我们得到了几个测试用例。 每个测试用例都描述了一组放置在数轴上的怪物和一组武器。 每个怪物都坐在一个整数坐标上，多个怪物可以共享相同的位置。 每种武器都对应于数轴上的一个闭区间，如果我们使用该武器一次，它就会消灭坐标位于该区间内的所有怪物。 

任务是确定我们必须发射的最少武器数量，以便每个怪物至少被摧毁一次。 每个武器最多可以使用一次，并且我们可以跳过武器。 如果即使使用所有武器也不能覆盖所有怪物坐标，那么答案是不可能的。 

这些约束建议每个测试用例都有一个线性或近线性解决方案。 所有测试用例中的怪物和武器总数以 3×10^5 为界，因此任何比 O(n log n) 整体更差的方法都有超时的风险。 这强烈暗示了排序加上贪婪选择或扫线结构与有效的区间处理相结合。 

当某些怪物位于所有间隔之外时，就会出现微妙的边缘情况。 例如，如果怪物位于位置 [1, 10] 并且所有枪支仅覆盖 [2, 5] 内的范围，那么即使我们使用每种武器，也没有任何选择可以覆盖 1 和 10。 正确答案是 -1，仅关注局部覆盖的贪婪算法可能会错误地认为部分进展就足够了。 

当间隔严重重叠但未共同覆盖间隙时，会出现另一种失败情况。 例如，位于 [1, 2, 3, 10] 的怪物和枪 [1, 3]、[2, 2]、[3, 3] 看起来很有希望出现在第一个集群中，但剩下 10 个集群未被覆盖。 该算法必须明确确保覆盖整组怪物位置，而不仅仅是贪婪地最大化覆盖范围。 

## 方法

 蛮力的想法是将每种武器视为二元选择：接受或不接受。 我们将模拟所有武器子集并检查哪些子集覆盖了所有怪物位置。 即使我们删除明显无用的选择，武器数量的结构仍然呈指数级增长。 对于多达 3×10^5 的武器，这是不可能的。 

一种稍微结构化的暴力方法是对怪物进行排序，并尝试在每个未发现的怪物处贪婪地挑选覆盖它的任何区间，并递归地尝试不同的选择。 分支因子仍然太大，因为多个间隔可能覆盖同一个怪物，并且后来的决定会影响未来的覆盖范围。 

关键的观察是，一旦怪物从左到右被最佳覆盖，我们就不需要重新考虑它。 如果我们按位置对怪物进行排序，我们可以想到从最小坐标到最大坐标进行扫描。 在每一步中，我们只关心可以覆盖当前未覆盖位置的间隔，并且在这些间隔中，选择将覆盖范围尽可能向右扩展的间隔始终是最佳的。 这就是区间覆盖的经典贪婪结构。 

这个问题的关键在于，我们并不被迫使用所有间隔；而是需要使用所有间隔。 我们选择一个子集。 这与一组点的最小区间覆盖相同。 我们对间隔进行预处理，以便我们可以针对任何位置快速检索从该位置之后开始的最佳间隔。 

我们对怪物和间隔进行排序。 然后，我们维护一个间隔指针和一个数据结构，用于保存当前可用的候选间隔。 我们贪婪地穿过怪物，反复选择最大范围的间隔。

 ## 算法演练

1.按照升序对怪物位置进行排序。 这确保我们始终以一致的从左到右的顺序处理覆盖范围，而不会遗漏任何间隙。 
2. 按左端点对所有枪进行排序。 这使我们能够在间隔与当前未覆盖位置相关时立即激活间隔。 
3. 将指针保持在枪支上方，并以右端点为键控的最大堆。 堆代表左端点至多是当前怪物位置的所有区间。 
4. 初始化指针`i = 0`超过怪物，以及用过的枪的柜台。 
5.对于当前怪物位置`x = monsters[i]`, 推动所有枪`l <= x`到堆中。 每次推送都包含间隔的右端点。 
6. 从堆中，重复丢弃右端点小于的任何区间`x`，因为它不能再覆盖当前的怪物了。 
7. 如果堆变空，则意味着没有剩余间隔可以覆盖该怪物位置，所以答案是不可能的。 
8. 否则，取右端点最大的区间，将其作为一次射击，然后移动`i`前进到第一个大于该右端点的怪物位置。 

每个选定的间隔都会有效地让我们尽可能向右跳跃，有可能一步覆盖多个怪物。 

### 为什么它有效

 贪心选择依赖于这样一个事实：在任何未覆盖的怪物位置，在可以覆盖它的所有区间中，选择具有最大右端点的位置永远不会减少未来的可能性。 任何较早结束的替代间隔只能覆盖所选间隔已覆盖的怪物的子集，同时为未来留下至少同样多或更多的工作。 由于该过程总是推进最左边未覆盖的怪物，因此这种局部最优扩展会产生全局最优的最小射击次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        monsters = list(map(int, input().split()))
        guns = [tuple(map(int, input().split())) for _ in range(m)]

        monsters.sort()
        guns.sort()

        heap = []
        j = 0
        ans = 0
        i = 0

        while i < n:
            x = monsters[i]

            while j < m and guns[j][0] <= x:
                heapq.heappush(heap, -guns[j][1])
                j += 1

            while heap and -heap[0] < x:
                heapq.heappop(heap)

            if not heap:
                ans = -1
                break

            r = -heapq.heappop(heap)
            ans += 1

            while i < n and monsters[i] <= r:
                i += 1

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案对怪物和间隔进行排序，以便可以在单个前向传递中处理覆盖范围。 指针`j`确保每支枪在相关时插入一次。 堆存储候选右端点，我们总是提取最大的可达端点。 

删除无效间隔的内部清理循环是必要的，因为右端点已经位于当前怪物左侧的间隔无法再做出贡献。 如果没有这个，堆可能会错误地建议可用的覆盖范围。 

前进`i`使用 while 循环可确保一步跳过所选时间间隔内的所有怪物，从而防止不必要的重复处理。 

## 工作示例

 考虑一个小案例，怪物位于位置 [1, 2, 4]，枪支位于 [1, 3]、[2, 3]、[2, 5]。 

我们对两个列表进行排序，然后模拟：

 | 步骤| 当前怪物 x | 候选堆 (r) | 选择的间隔 | 搬家后我|
 | --- | --- | --- | --- | --- |
 | 1 | 1 | [3] | [1,3]| 2 |
 | 2 | 4 | [3,5] → 清理为 [5] | [2,5]| 3 |

 第一次射击后，1和2处的怪物被覆盖。 第二个镜头涵盖了 4 个镜头，因此我们以 2 个镜头结束。 

现在考虑一个无法覆盖的失败案例：怪物 [1, 10]，枪支 [1, 3]，[4, 6]。 

在怪物 1 处，我们只能选择 [1,3]，这使我们前进到位置 4。在 4 处，只有 [4,6] 可用，前进到 7。在 7 处，没有区间覆盖它，因此算法停止并正确输出 -1。 

这表明该算法不假设存在全球覆盖范围，并且可以正确检测覆盖范围中的间隙。

 ## 边缘情况

 当怪物严格增加但间隔仅局部重叠时，就会出现关键的边缘情况。 对于输入怪物 [1, 2, 100] 和枪 [1, 3], [2, 3], [3, 4]，算法将在一次射击中正确覆盖前两个怪物，但后来在 100 处失败，因为没有间隔达到它。 当 x = 100 时堆变空，产生 -1。 

另一个微妙的情况是，许多间隔在第一个怪物之前开始，但也在它之前结束。 对于怪物 [10] 和枪 [1,5]、[2,6]，堆已满，但在清理过程中所有间隔都被丢弃，因为它们的右端点小于 10。算法正确地得出了不可能的结论。 

最后一个边缘情况是严重重叠：怪物 [1,2,3,4]，枪支 [1,4]，[1,3]，[2,4]。 贪婪的选择总是先选择[1,4]，立即清除所有怪物。 任何替代的第一选择仍然需要第二步，因此算法的最大到达策略避免了不必要的射击。
