 ## 解决方案
 ## 问题理解

 我们得到了研究人员论文的引用计数列表，已经按非递增顺序排序。 每个数字代表特定论文被引用的次数。 任务是计算 H 指数，它是最大的整数 H，使得至少 H 篇论文的引用计数至少为 H。 

解释该定义的一个有用方法是想象从被引用次数最多的论文开始扫描排序列表。 在位置 i（1 索引）处，我们询问这篇论文是否仍然符合大小为 i 的组的一部分，这意味着它的引用计数至少为 i。 答案是满足该条件的最大位置。 

约束 n 可以大到 10^6，因此解决方案必须在线性时间内运行。 任何二次方，例如检查每个可能的 H 并每次扫描数组，在最坏的情况下将涉及多达 10^12 次操作，并且不会通过。 

当所有引用计数为零时，就会出现微妙的边缘情况。 例如，如果 n = 5 并且数组为 [0, 0, 0, 0, 0]，则没有位置满足 i ≤ ci，因此答案必须为 0。假设至少一篇有效论文的简单实现可能会错误地返回 1。 

当所有引用都很大时，就会出现另一种边缘情况。 例如，[10, 10, 10] 应返回 3，因为所有三篇论文都满足至少 3 次引用，即使它们各自也满足较高的原始值​​。 约束与位置有关，而不仅仅是引用程度。 

## 方法

 蛮力的想法是尝试从 0 到 n 的每个可能的候选值 H，并检查是否至少有 H 篇论文至少有 H 次引用。 由于数组是按降序排序的，因此可以通过扫描直到找到第一个 ci < H 的索引并计算有多少元素有效来检查固定的 H。 此检查的成本为 O(n)，并且对所有 H 值执行此操作会导致总工作量为 O(n^2)，对于 n 高达 100 万而言，这变得太慢。 

关键的观察来自排序结构。 由于引用是按降序排列的，一旦我们到达 ci < i 的索引，后面的每个元素也将失败，因为数组只会减少。 这意味着答案由条件中断的第一个位置决定。 我们可以从左到右扫描一次，当 ci < i 时立即停止，返回 i - 1 作为 H 索引。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 单次扫描 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 最优策略

 我们利用数组已经按非递增顺序排序的事实。 

1. 从索引 i = 1 的第一篇论文开始扫描数组。我们将引用计数 ci 与 i 进行比较，因为 i 代表我们尝试验证作为 H 索引组一部分的论文数量。 这直接符合H指数的定义。 
2. 对于每个位置i，检查是否ci ≥ i。 如果这一点成立，则意味着我们仍然有可能形成至少为 i 的 H 指数，因为我们至少有 i 篇论文，至少有 i 次引用。 
3. 只要条件仍然成立，就继续前进。 每个有效位置都扩展了可能的 H 指数。 
4. 当我们第一次遇到 ci < i 的位置时，我们停止。 此时，不可能有任何 H 指数大于 i - 1，因为即使是第 i 篇论文也不满足要求。 
5. 返回 i - 1 作为最终答案。 如果我们永远不破坏条件，那么所有论文都满足 ci ≥ i，所以答案是 n。 

### 为什么它有效

正确性来自于单调性。 由于数组按降序排序，因此当我们向右移动时，引用值永远不会增加。 同时，每一步所需的阈值 i 恰好增加 1。 一旦 ci < i，我们知道对于所有 j > i，cj ≤ ci 且 j > i，因此 cj < j 也必须成立。 这保证了第一个故障点正是最大可行 H 指数结束的位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    if n == 0:
        print(0)
        return
    
    c = list(map(int, input().split()))
    
    for i, val in enumerate(c, start=1):
        if val < i:
            print(i - 1)
            return
    
    print(n)

if __name__ == "__main__":
    solve()
```解决方案读取 n 和排序的引文列表。 然后，它执行一次传递，使用枚举跟踪从 1 开始的索引。 当引文值第一次低于其索引时，我们将之前的索引作为 H 索引返回。 如果没有发生此类中断，则 H 指数等于 n。 

一个常见的差一陷阱是忘记条件中索引是从 1 开始的，因此枚举必须从 1 开始。另一个微妙之处是处理 n = 0，其中不存在输入列表并且答案直接为零。

 ## 边缘情况

 对于空出版物列表，输入为 n = 0。算法立即返回 0，而不尝试访问引文数组，这避免了无效读取并匹配没有论文意味着 H 索引为零的定义。 

对于像 [0, 0, 0] 这样的数组，扫描在 i = 1 时失败，因为 0 < 1。算法返回 0，正确反映没有论文满足单个引用阈值。 

对于像 [100, 100, 100, 100] 这样的完全强大的数据集，扫描期间不会发生中断。 该算法到达末尾并返回 n = 4，因为所有四篇论文都满足所有 i 的条件 ci ≥ i。
