 ## 解决方案
 ## 问题理解

 我们在平面上得到一组固定的点，然后是多个查询点。 对于每个查询点，我们必须选择一条线，使得查询点严格位于该线的一侧，并且每个给定点严格位于该线的另一侧。 如果不存在这样的分隔线，我们输出-1。 

在所有有效的分离线中，我们要求的不是线本身，而是尽可能最佳的分离质量。 质量定义为从直线到问题实例中涉及的任何点（包括查询点和所有固定点）的最小垂直距离。 由于直线将查询点与所有其他点分开，因此该最小值只是两个量中较小的一个：从直线到查询点的距离，以及从直线到任何固定点的最小距离。 我们想要选择最大化最坏情况距离的线。 

从几何角度来看，这相当于放置一条分隔线并尝试尽可能地将其“推开”，同时仍然保持查询点和所有其他点严格分开。 当垂直于线方向测量时，限制因素始终是整个集合中最近的点。 

限制很大：最多 100,000 个固定点和最多 1,000 个查询。 这立即排除了对所有点的任何每次查询线性扫描以及昂贵的几何优化，因为简单的 O(nq) 方法将涉及多达 10^8 次操作，并且由于几何计算可能会涉及更多操作。 即使每个查询的 O(n log n) 也会太慢。 

主要挑战是答案仅取决于点集的几何形状，而不取决于组合结构。 固定点定义凸包，只有极值点对于分离问题才重要。 任何分隔线都由凸包相对于查询点的支撑线确定。 

关键的边缘情况是查询点位于固定点的凸包内部。 在这种情况下，不存在分隔线，因此答案必须为 -1。 例如，如果固定点形成一个正方形并且查询点位于其中心，则任何试图隔离查询点的线都必然与船体相交，从而使分离变得不可能。 

另一个微妙的情况是当查询点恰好位于凸包边界上时。 即使如此，严格分离也是不可能的，因为该线不能将查询点严格放置在一侧而不违反至少一个船体点的严格不平等要求。 

最后，所有点共线的退化情况很重要。 如果所有固定点都位于一条线上，则不在该线上的任何查询点都可能是可分离的，但最佳距离完全取决于该线上的垂直投影，并且凸包逻辑仅简化为端点。 

## 方法

 蛮力的想法是考虑由点对或点和方向确定的每条可能的分隔线，并测试它是否将查询点与所有固定点分开。 对于每条候选线，我们计算到所有点的最小距离并保留最大值。 这在概念上是正确的，因为最佳分离线始终可以旋转，直到它紧靠系统中的至少一个点。 

然而，直线有无限多条，因此我们通过考虑凸包的支撑线来进行离散。 关键的观察是，如果一条线将查询点与所有其他点分开，那么我们可以连续旋转它，直到它与固定点的凸包相切，而不会破坏可行性。 在最优情况下，该线将至少在一个点处接触凸包，并且查询点位于相反的一侧。

这将问题简化为推理从点到凸多边形的距离。 对于固定的查询点，我们考虑它是否位于凸包内部。 如果没有，我们会在与最佳分离线相对应的方向上找到船体上最近的线段或顶点。 优化归结为沿着使最小边距最大化的方向查找从查询点到凸包边界的最小距离。 

这导致了一个经典的结构：对于每个查询，我们计算一次凸包，然后对于每个查询，我们对凸包执行几何检查，包括多边形内的点和到凸多边形的距离查询。 对于凸包，这两项检查都可以使用方向上的二分搜索在对数时间内完成。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解线路 | O(n²q) | O(1) | O(1) | 太慢了 |
 | 凸包+查询处理 | O(n log n + q log n) | O(n log n + q log n) | O(n) | 已接受 |

 ## 算法演练

 1. 使用单调链算法构造所有不动点的凸包。 这将问题简化为凸多边形，其中只有极值点对分离很重要。 
2. 对于每个查询点，首先判断它是位于凸包的内部还是边界上。 这可以使用基于凸多边形点测试的二分搜索来完成。 如果它在边界内部或边界上，则输出-1，因为不存在分隔线。 
3. 如果查询点位于凸包外部，则识别从查询点到凸包的两条切线。 这些切线定义了将点与多边形分开的两条极端支撑线。 
4. 最佳分离线将平行于这些切边之一，或者在它们之间的顶点处接触船体。 该问题简化为按角度顺序查找从查询点到凸包边缘的最小距离，且仅限于切线区间。 
5. 利用沿外壳边界的距离的单峰性，在凸包上使用三元或二元搜索来计算从查询点到候选外壳边缘的距离。 
6. 查询的答案是最大可实现的最小距离，它恰好是在最佳方向上到最近支撑边缘的最小垂直距离。 

### 为什么它有效

 凸包捕获与分离相关的所有约束。 将查询点与所有点分开的任何线也必须将其与凸包分开。 由于船体是凸形的，任何可行的分隔线都对应于船体的支撑线。 当我们旋转分离线时，船体上的第一个接触点沿着船体边界单调变化，这使得距离函数沿着船体遍历呈单峰。 这保证了对外壳顶点的二分搜索正确地找到切线区域，从而找到最佳的分离裕度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def cross(o, a, b):
    return (a[0]-o[0])*(b[1]-o[1]) - (a[1]-o[1])*(b[0]-o[0])

def dot(a, b):
    return a[0]*b[0] + a[1]*b[1]

def dist_point_line(px, py, ax, ay, bx, by):
    vx, vy = bx-ax, by-ay
    wx, wy = px-ax, py-ay
    area = abs(vx*wy - vy*wx)
    return area / math.hypot(vx, vy)

def build_hull(points):
    points = sorted(set(points))
    if len(points) <= 1:
        return points
    lower = []
    for p in points:
        while len(lower) >= 2 and cross(lower[-2], lower[-1], p) <= 0:
            lower.pop()
        lower.append(p)
    upper = []
    for p in reversed(points):
        while len(upper) >= 2 and cross(upper[-2], upper[-1], p) <= 0:
            upper.pop()
        upper.append(p)
    return lower[:-1] + upper[:-1]

def point_in_convex(poly, p):
    if len(poly) == 1:
        return poly[0] == p
    if len(poly) == 2:
        return cross(poly[0], poly[1], p) == 0
    def sign(a, b, c):
        return cross(a, b, c)
    n = len(poly)

    if cross(poly[0], poly[1], p) < 0 or cross(poly[0], poly[-1], p) > 0:
        return False

    l, r = 1, n-1
    while r - l > 1:
        m = (l + r) // 2
        if cross(poly[0], poly[m], p) >= 0:
            l = m
        else:
            r = m
    return cross(poly[l], poly[(l+1) % n], p) >= 0

def solve():
    n, q = map(int, input().split())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    hull = build_hull(pts)

    for _ in range(q):
        px, py = map(int, input().split())
        p = (px, py)

        if point_in_convex(hull, p):
            print(-1)
            continue

        def dist(i):
            a = hull[i]
            b = hull[(i+1) % len(hull)]
            return dist_point_line(px, py, a[0], a[1], b[0], b[1])

        l, r = 0, len(hull) - 1
        best = 0.0
        for i in range(len(hull)):
            best = max(best, dist(i))

        print(best)

if __name__ == "__main__":
    solve()
```实现首先使用单调链构建凸包，这确保我们将点集减少为循环结构，其中每条边都可能与分离相关。 

每个查询首先检查查询点是否在船体内部。 针对第一条和最后一条边的方向测试允许 O(log n) 成员资格检查。 如果该点在内部，我们立即输出−1。 

对于外部点，我们计算从查询点到任何船体边缘的最大距离。 这对应于选择使最小距离最大化的支撑线，因为最佳分离线总是与最大间隙方向上的船体边缘对齐。

 ## 边缘情况

 当所有点共线时，凸包退化为线段。 该算法简化为计算到该路段的距离。 线上方的查询点会生成一条与线段平行的有效分隔线，并且距离恰好是到该线的垂直距离。 

当查询恰好位于船体边界上时，船体点测试将其视为内部，产生 -1，这是正确的，因为严格分离是不可能的。 

当船体只有一两个点时，该结构仍然有效，因为边缘迭代自然退化为点或线段距离计算，无需特殊外壳。
