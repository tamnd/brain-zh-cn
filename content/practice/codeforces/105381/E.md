 ## 解决方案
 ## 问题理解

 我们从标记为 1 到 n 的卵石开始，其中卵石 i 的重量为 i。 在每次移动中，选择两个当前可用的卵石并通过两个设备之一。 一个设备始终返回两个输入中较轻的一个，另一个设备始终返回较重的一个。 应用该装置后，只有返回的卵石才能继续进入下一轮。 这个过程精确地重复 n − 1 次，因此最终只剩下一颗卵石。 

不同的是，这两种设备不能随意使用。 最小化器最多可以使用 a 次，而最大化器最多可以使用 b 次，其中 a + b = n − 1。问题是是否存在一系列配对和设备选择，使得最终剩余的卵石恰好是 x。 

虽然这个过程看起来像是重复的比较，但关键的难点在于，每次操作不仅仅是比较，而是强制选择规则，使哪个元素存活下来。 整个过程就是一系列的淘汰，塑造出能够生存到最后的价值观。 

这些限制使得对所有消除树进行暴力破解是不可能的。 尽管n最多为1000，但可能的二元锦标赛结构的数量已经以n为指数，并且每个节点都有两种可能的操作类型，因此直接枚举的增长远远超出了可行的极限。 

当 x 为 1 或 n 时，会出现微妙的边缘情况。 在这些情况下，我们可能会错误地假设答案总是可能的，因为最小或最大的元素似乎“容易”保留，但可行性仍然取决于我们是否有足够的最小化或最大化操作来强制极值的生存。 

## 方法

 直接模拟将尝试构建二叉消除树，在每一步选择是应用最小值还是最大值。 每个配置对应于 n 个项目的完整配对结构，并且每个内部节点都标有操作类型。 这导致了巨大的状态空间：即使固定一个树形状，也有 2^(n−1) 种分配操作的方法，并且有加泰罗尼亚语数量的形状。 这太大了。 

关键的观察是停止考虑比较的结构，而是跟踪 x 的每一侧必须消除多少个值。 

想想最后的幸存者 x。 所有其他元素都必须在某个时刻被消除。 每个元素 y < x 只有在以丢弃它的方式被选择时才会消失。 类似地，每个元素 y > x 也必须被消除。 关键的约束是只存在两种类型的操作：最小化器始终保持较小的值，最大化器始终保持较大的值。 

这意味着：

 - 最小化操作最多可以消除一个大元素（保留较小的元素）。 
- 最大化操作最多可以消除一个小元素（它保留较大的元素）。 

因此，我们将操作解释为删除 x 错误一侧的元素的“工具”。 

为了确保 x 生存：

 - 所有小于 x 的值都必须是可移除的，而不会杀死 x。 
- 所有大于 x 的值也必须可以在不杀死 x 的情况下被删除。 

现在考虑需要什么。 为了消除元素 y < x，我们最终必须将其放入针对更大或等于的元素的最大化操作中，以便将其丢弃。 每个小元素消除都会消耗一次最大化器使用，除非只要 x 受到保护，消除就可以与其他元素仔细配对。 

类似地，必须使用最小化运算消除每个大于 x 的元素。 

因此，可行性简化为检查我们是否有足够的最小化操作来消除所有大于 x 的元素，以及是否有足够的最大化操作来消除所有小于 x 的元素。 

计数很简单：

 - 大于 x 的元素：n − x
 - 小于 x 的元素：x − 1

 所以我们需要：

 - b ≥ n − x
 - a ≥ x − 1

由于 a + b = n − 1，这两个条件也是充分的。 

这将整个问题简化为简单的计数可行性检查，独立于任何结构。

 ## 算法演练

 1. 对于每个测试用例，读取 n、a、b 和 x。 x 值是我们希望存活到最后的石子。 
2. 计算有多少个元素小于x。 这是 x − 1。这些元素必须以永远不会删除 x 的方式全部消除。 
3. 计算有多少个元素大于x。 这是 n − x。 这些元素也必须被消除，同时保留 x。 
4. 检查最大化操作是否足以消除所有小于 x 的元素。 由于最大化器保留较大的值，因此每次使用都可以被视为能够删除一个较小的元素，因此我们需要 a ≥ x − 1。 
5. 检查最小化操作是否足以消除所有大于 x 的元素。 由于最小化器保留较小的值，因此每次使用都允许删除一个较大的元素，因此我们要求 b ≥ n − x。 
6. 如果两个条件都成立，则输出“Yes”，否则输出“No”。 

### 为什么它有效

 在此过程中，除 x 之外的每个元素都必须恰好被消除一次。 在不影响 x 的情况下消除较小元素的唯一方法是通过保留较大伙伴的最大化交互，并且对称地，在不影响 x 的情况下消除较大元素需要最小化交互。 由于每个操作最多可以安全地从 x 的另一侧消除一个元素，因此所需消除的总数直接与每种操作类型的可用计数相匹配。 这就创造了一个严格的充要条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n, a, b, x = map(int, input().split())
    
    need_maximizer = x - 1
    need_minimizer = n - x
    
    if a >= need_maximizer and b >= need_minimizer:
        print("Yes")
    else:
        print("No")
```实施是对导出的可行性条件的直接翻译。 每个测试用例在恒定时间内独立处理。 关键细节是保持映射一致：最大化器使用对应于消除小于 x 的元素，而最小化器使用对应于消除大于 x 的元素。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline
    
    t = int(input())
    out = []
    for _ in range(t):
        n, a, b, x = map(int, input().split())
        if a >= x - 1 and b >= n - x:
            out.append("Yes")
        else:
            out.append("No")
    return "\n".join(out)

 ## 边缘情况

 当 n = 1 时，没有任何操作，因此唯一可能的幸存者是 1。条件给出 a = 0 ≥ 0 且 b = 0 ≥ 0，因此算法正确返回 Yes。 

当 x = 1 时，我们有 x − 1 = 0，因此不需要最大化操作。 所有其他元素必须使用最小化器消除，它匹配 b ≥ n − 1。 

当 x = n 时，我们有 n − x = 0，因此不需要最小化操作。 所有较小的元素必须使用最大化器删除，匹配 a ≥ n − 1。 

在每种情况下，计算出的不等式都正确地简化为仅约束必要的操作类型的直观要求。
