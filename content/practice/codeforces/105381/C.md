 ## 解决方案
 ## 问题理解

 我们得到一个最多有 300 个顶点的简单无向图，其中每个输入边都保证存在并且不会出现重复项。 该图表示国家之间的旅行路线。 “旅行”是在同一个国家开始和结束的封闭式步行，但沿途不允许重新访问任何其他国家。 用图术语来说，这是一个简单的循环，只是我们将不同的旋转和方向区分为不同的行程，因为每当任何位置不同时，定义都会将序列视为不同的。 

我们被要求计算有多少个这样的简单循环恰好有 5 条边，这意味着它们按顺序由 5 个不同的顶点组成，最后一个顶点连接回第一个顶点。 

就顶点而言，约束很小，但图不一定足够稀疏以允许对长度为 5 的所有路径进行简单枚举。当 n 达到 300 时，任何接近 O(n^5) 甚至每条边的密集三重嵌套探索的方法都会变得有问题，而 O(m * n^2) 或 O(n^3) 是可行的。 

一个微妙的问题是过度计算。 由于可以从任何顶点开始并在两个方向上遍历循环，因此返回到起点的路径的简单枚举将多次对同一循环进行计数。 例如，如果我们考虑所有起点和方向，循环 1-2-3-4-5-1 会生成 10 个不同的序列。 任何正确的解决方案都必须确保根据问题的序列相等性定义，每个不同的序列都被精确计数一次。 

另一个常见的陷阱是将简单的自行车与一般的封闭式步行混淆。 像 1-2-3-2-1-... 这样的步行是无效的，因为它会重新访问顶点 2。即使它在 5 步内正确闭合，它也必须使用 5 个不同的顶点。 

## 方法

 暴力方法会尝试枚举 6 个顶点的所有序列（因为长度为 5 的行程有 6 个位置，包括重复的起始位置）。 我们修复起始节点并执行深度为 5 的深度优先搜索，确保我们永远不会重新访问顶点，除非在最后一步可能返回到起始位置。 这保证了正确性，因为它直接强制执行定义。 

然而，这在每个步骤中探索了大约 n 个选择，为密集图提供了 O(n^5) 最坏情况行为。 当 n = 300 时，这远远超出了可行的限制。 

关键的观察是，有效的解决方案是通过选择不同顶点（v1、v2、v3、v4、v5）的有序 5 元组来完全确定的，这样所有必需的边都存在，并且 v5 连接回 v1。 该结构是刚性的：它是一个长度为 5 的简单循环。我们可以修复部分结构并使用邻接交集来计算完成情况，而不是动态探索路径。 

我们可以将其视为对无向图中的 5 个循环进行计数，其中每个循环都被视为从其第一个顶点开始的不同有向序列。 这建议固定起始顶点，然后使用邻接表逐步构建循环，同时通过逐步限制选择来确保明确性。 

干净的优化来自于 m 很小（≤ 500）这一事实。 我们可以使用邻接集来表示图，并迭代边而不是所有对。 然后我们构建部分路径并扩展它们，同时检查邻接性，时间复杂度为 O(1)。 这将问题简化为迭代长度为 4 的可能路径并关闭它们。 

一种更结构化的方法是：对于每个有向边 v1 → v2，我们尝试扩展到 v3 → v4 → v5 → 返回 v1，确保所有顶点都是不同的。 这足以减少对称性以避免过度计数，并将复杂性保持在 O(m * d^2) 左右，在 m ≤ 500 的情况下这是安全的。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 长度为 5 的路径上的强力 DFS | O(n^5) | O(n^5) | O(n) | 太慢了 |
 | 带有邻接检查的以边为中心的枚举 | O(m·d^2) | O(m·d^2) | O(n + m) | 已接受 |

 ## 算法演练

 我们通过将循环锚定在有向边缘上并向外构建来重组计数。 

1. 为所有顶点构建邻接集，以便我们可以在 O(1) 内测试边是否存在。 这是必要的，因为我们在构造过程中会反复检查两个顶点是否相连。 
2. 迭代每个有向边 (a, b)。 我们将此视为潜在循环的第一步，这意味着我们假设循环从 a 开始并移动到 b。 固定方向可避免从不同的起始方向对同一周期进行多次计数。 
3. 对于 b 的每个邻居 c，选择 c 作为第三个顶点，但如果它等于 a 或 b，则跳过 c。 这会在构造时立即强制执行简单循环约束。 
4. 对于 c 的每个邻居 d，选择 d 作为第四个顶点，确保 d 与 a、b 和 c 不同。 我们还要求 d 不直接等于 a，因为最后一步必须通过第五个不同的顶点干净地关闭循环。 
5. 现在我们需要一个将 d 连接回 a 的顶点 e。 我们迭代 d 的邻居 e，并仅在关闭时对 e 等于 a 的邻居进行计数。 然而，由于循环必须恰好有 5 个不同的顶点，因此我们强制 e 在末尾恰好是 a，因此只有在确保路径有 4 个不同的顶点后，我们才直接检查边 (d​​, a) 是否存在。 
6. 每个有效构造都会贡献一个从 a 开始的有效有向循环序列，因此我们直接累加计数。 

更有效的解释是，我们正在计算从 a 到简单邻居结构的长度为 4 的路径，其中通过邻接而不是最终步骤的枚举来检查闭合情况。 

### 为什么它有效

 每个有效的长度为 5 的简单循环都可以通过选择其第一个有向边 (v1, v2) 来唯一表示。 一旦这个问题解决了，剩下的三个顶点就由路径顺序（v2，v3，v4，v5）唯一确定，并且最终的边必须返回到v1。 因为我们在构造过程中始终强制执行明确性，所以我们永远不会生成无效循环。 因为我们在第一个边缘固定方向，所以每个有效序列只计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    adj = [set() for _ in range(n + 1)]
    
    edges = []
    for _ in range(m):
        u, v = map(int, input().split())
        adj[u].add(v)
        adj[v].add(u)
        edges.append((u, v))
    
    ans = 0

    for a, b in edges:
        for c in adj[b]:
            if c == a:
                continue
            for d in adj[c]:
                if d == a or d == b:
                    continue
                if a in adj[d]:
                    ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该代码构建邻接集以启用恒定时间边缘检查。 我们迭代每条边作为起始有向步骤，然后扩展到另外两个顶点，同时保持所有顶点都是不同的约束。 最终条件`a in adj[d]`确保循环关闭回到起点。 

关键的微妙之处在于我们从未显式地迭代第五个顶点。 相反，闭合是通过邻接来验证的，这会折叠最后一步并避免额外的循环。 这就是使解决方案保持在界限之内的原因。

 ## 边缘情况

 关键的边缘情况是图几乎完整但缺少关闭循环所需的单个边缘。 例如，如果除 (1, 3) 之外的五个节点之间存在所有边，则任何需要通过该缺失边进行闭合的尝试构造都将在最终邻接检查时失败`a in adj[d]`。 该算法仍然探索部分路径，但它们在关闭时被过滤掉，确保正确性而无需特殊的大小写。 

另一种情况是多个 5 周期共享边沿。 该算法正确地计算每个不同的有序序列，因为每个起始有向边锚定一个唯一的枚举路径。 即使两个循环严重重叠，它们不同的顶点顺序也确保它们被单独计数。
