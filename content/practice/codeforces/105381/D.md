 ## 解决方案
 ## 问题理解

 我们有一个矩形网格$n$行和$m$列。 每个单元格都包含一个整数，唯一允许的操作是在每列内独立地排列值。 在这些重新排列之后，我们查看每一行，并通过对相邻列之间的绝对差求和来衡量它的“平滑”程度。 目标是排列每一列，使所有这些行方向相邻差异的总和尽可能小。 

考虑最终表达式的一个有用方法是每对相邻列独立贡献：对于每一行$i$，我们支付$|a_{i,j} - a_{i,j+1}|$，然后我们对所有行和所有相邻列对求和。 

关键的约束是每一列都是值的多重集，我们可以任意重新排序，但我们必须保持列内容完整。 

界限很小：$n, m \le 100$。 这表明$O(n^2 m \log n)$甚至$O(n^3)$可以舒服地过去。 主要提示是，重新排列是每列独立的，但目标是耦合相邻的列，因此我们不能完全独立地对待列。 

一个天真的误解是对每一列进行独立排序。 但这会失败，因为重要的是两列之间的行之间的配对，而不是内部排序本身。 

当值在行之间高度交错时，会出现微妙的边缘情况：

 示例：```
2 2
1 100
100 1
```如果我们对两列进行排序，我们会得到：```
1 100
1 100
```成本变成$0 + 0 = 0$，这里是最优的。 但在其他配置中，当跨列存在更复杂的分布时，贪婪独立排序并不能保证最小的跨列匹配成本。 

当最佳配对需要在所有相邻列对中一致地匹配不同行中的最小与最大时，会出现另一种失败情况。 每列对的纯粹局部贪婪策略可能会在多个列之间造成不一致。 

这表明我们需要对每列进行全局排序，以一致的方式同时对齐所有列。 

## 方法

 暴力方法将独立尝试每列的每个排列。 每列都有$n!$排列，因此总复杂度变为$(n!)^m$，即使对于$n = 10$。 

我们可以稍微简化一下这个想法：我们可以考虑相邻列之间的匹配行，而不是枚举所有排列。 对于列的固定顺序$j$，我们想要找到列的最佳排序$j+1$最大限度地减少配对条目之间的总绝对差异。 

这是两个多重集之间的经典分配式问题。 如果我们修复两列的排序，则最佳配对是按排序顺序匹配元素，因为一维绝对差异的重排不等式。 

关键的见解是，问题分解为相邻列之间的独立匹配问题，并且通过对两列进行排序并按顺序配对来最佳地解决每个此类问题。 但是，我们只能对每列进行一次排列，并且该排列必须在所有相邻对之间保持一致。 

这导致了全局一致性观察：我们应该修复第一列的排列，然后向前传播诱导排序，确保下一列相对于前一列以最佳方式重新排列。 该结构变成了最佳匹配链，每个匹配都通过对两侧进行排序来解决，并且我们在行之间保持一致的排序。 

这将问题转化为重复对列进行排序并逐行累积成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O((n!)^m)$|$O(nm)$| 太慢了 |
 | 列排序+贪心匹配|$O(m \cdot n \log n)$|$O(nm)$| 已接受 |

 ## 算法演练

 我们逐列处理网格，保持每一列都以相对于前一列最佳的一致行顺序存储。 

1.将每一列视为一个长度的数组$n$。 我们首先对所有列进行独立排序。 这确保了在每列中，值按递增的幅度排序，这是匹配的规范代表。 
2. 对于每对相邻的列$j$和$j+1$，我们通过配对来计算绝对差的最小可能和$k$- 列的最小元素$j$与$k$- 列的最小元素$j+1$。 这是根据排序匹配的最优性得出的$L_1$成本。 
3. 累积所有相邻列对的成本。 
4. 最终答案是所有结果的总和$j$列之间的匹配成本$j$和列$j+1$。 

为什么此步骤有效是因为每个列排列完全通过排序确定，并且一旦排序，列内的任何进一步重新排列都将破坏至少一个相邻对的最优性。 因此，当所有列独立排序并按索引对齐时，即可实现全局最优配置。 

### 为什么它有效

 对于任意两列，考虑其元素之间的任意配对。 如果两条边按值顺序交叉，交换它们永远不会增加总的绝对差值。 重复删除此类反转可将任何配对转换为排序列上的恒等配对，而不会增加成本。 这正是应用于绝对差异的重排不等式，确保对两列进行排序并按索引配对对于每个相邻对都是最佳的。 由于每列恰好出现在两个相邻比较中（边界除外），因此每对的独立最佳配对保持全局一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    cols = [[] for _ in range(m)]
    
    for _ in range(n):
        row = list(map(int, input().split()))
        for j in range(m):
            cols[j].append(row[j])
    
    for j in range(m):
        cols[j].sort()
    
    ans = 0
    for j in range(m - 1):
        a = cols[j]
        b = cols[j + 1]
        for i in range(n):
            ans += abs(a[i] - b[i])
    
    print(ans)

if __name__ == "__main__":
    solve()
```我们首先将网格转置为列列表，以便列操作变成直接数组操作。 对每一列进行排序可以为任何基于匹配的成本建立最佳的内部排列。 

关键的实现细节是排序后，我们不再对行重新排序。 我们完全依赖索引对齐，即行对齐$i$在列中$j$与 row 配对$i$在列中$j+1$。 这就是最佳匹配的编码。

 ## 边缘情况

 极端情况是列中的所有值都相同。 在这种情况下，排序不会执行任何操作，并且该列对的贡献仅取决于相邻列的分布。 该算法可以处理这个问题，因为排序配对仍然可以任意对齐相同的值，而不影响成本。 

另一种情况是列相对于彼此反向排序。 即使一列是升序，下一列是降序，排序也可以消除反转并确保最小的配对成本。 该算法将两者转换为升序，然后一致地计算成本。 

什么时候$n = 1$，排序没有灵活性，答案简化为单行的绝对差之和。 该算法可以正确处理此问题，因为对单元素列进行排序是无操作，并且配对循环仍然计算正确的相邻差异。
