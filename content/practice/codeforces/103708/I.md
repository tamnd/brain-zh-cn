 ## 问题理解

 该任务围绕写为数字序列的单个整数展开。 从这个数字中，我们检查每个数字并检查该数字是否可以作为整个数字的约数。 我们忽略任何为零的数字，因为除以零是未定义的。 对于每个非零数字，我们测试原始数字是否可以被它整除而没有余数。 输出只是有多少位数字满足这个条件。 

输入只是一个整数，以最多八位数字的字符串形式给出。 将其视为数字和数字序列至关重要：算术使用完整的整数值，而迭代则对单个字符进行运算。 

最多八位数字的限制意味着数值完全符合标准 32 位整数限制。 这消除了对大整数处理的任何需要，并允许直接转换为整数类型和每个数字的简单模数运算。 

最常见的故障情况来自零数字和重复数字。 例如，在`1010`, 仅数字`1`应考虑，同时`0`必须被跳过。 另一个微妙的情况是重复：`111`，每次出现`1`必须独立计算，不能只计算一次。 

一个小例子阐明了边缘行为：

 输入：`120`只有数字是`1`,`2`， 和`0`。 我们忽略`0`。`1`除以120，`2`也能整除 120，所以答案是`2`。 

一个幼稚的错误是将数字视为唯一值而不是位置，这会错误地将重复的数字折叠成单个检查。 

## 方法

 蛮力方法已经非常接近最优了。 我们将数字转换为字符串，然后对于每个数字，我们将其转换回整数并测试它是否能整除整个数字。 这需要扫描最多八位数字并对每个数字执行恒定时间模数检查。 

操作计数受一个小常数限制，每个测试用例最多八次模数操作。 即使跨大型输入流，这也非常快。 

关键的观察结果是不需要预处理或高级结构。 问题的结构本质上是局部的：每个数字独立地做出是或否的决定。 数字之间没有交互，因此直接迭代之外的任何优化只会增加不必要的复杂性。 

唯一的微妙之处是安全地处理零数字并确保字符串字符和整数之间的转换正确。

 ## 算法演练

 1. 将输入数字作为字符串读取，以便可以单独访问数字而无需进行算术提取。 
2. 将整个字符串转换为整数值`N`，因为所有整除性检查都与这个固定数字进行比较。 
3. 初始化计数器`ans = 0`存储除法的位数`N`。 
4. 迭代每个字符`c`在字符串表示中`N`。 
5. 转换`c`转换为整数`d`。 
6. 如果`d`为零，跳过它，因为除以零是未定义的。 
7. 否则，检查是否`N % d == 0`。 如果为真，则增加`ans`一个。 
8.处理完所有数字后，输出`ans`。 

### 为什么它有效

 每个数字都针对相同的固定整数进行独立测试。 由于整除性是每个数字的二进制属性，因此最终答案只是独立有效性检查的总和。 跳过零可以防止无效算术，同时保持正确性，因为零不能提供有效的除法。 不存在数字交互，因此有效数字的个数恰好是满足模数条件的位置的个数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = int(s)
    ans = 0

    for ch in s:
        d = ord(ch) - 48
        if d == 0:
            continue
        if n % d == 0:
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将数字读取为字符串以保留数字结构，同时还将其一次转换为整数以进行快速整除检查。 使用`ord(ch) - 48`避免了开销`int()`在一个紧密的循环中，尽管考虑到很小的输入大小，任何一个都是可以接受的。 

正确处理的主要细节是模数之前的零检查。 如果没有它，程序将因被零除而崩溃。

 ## 边缘情况

 对于包含零的输入，例如`1002003`，算法在尝试除法之前正确地跳过每个零，确保不会发生无效的模数运算。 该循环只是忽略这些位置并继续累积有效数字。 

对于重复的数字，例如`11111111`，每次出现都是独立评估的。 计数器增加八倍，如果`n % 1 == 0`，反映位置计数而不是基于集合的计数。 

对于没有有效数字的数字，例如`39`仅在哪里`3`对数字进行除法时，计数器自然会保持低电平，因为每个非除法数字都不满足模数条件，而不会影响其他数字。
