 ## 解决方案
 ## 问题理解

 这个问题给出了一组富有的个人和一组公司。 每个人都有一份愿意购买的公司清单，每个公司最多可以分配给一个人。 有些人可能根本没有允许的公司，有些公司可能出现在多个列表中。 

理事会必须将公司分配给人员，以便每项分配都遵循列表，这意味着一个人只能接收他们明确列出的公司。 所有分配完成后，有些公司可能仍处于未分配状态。 目标是以最小化未分配公司数量的方式选择分配。 

从图的角度来看，这是人和公司之间的二分匹配问题，我们希望最大化匹配公司的数量，因为每个匹配的公司都会从“未分配”计数中删除一个。 

约束足够大，人员和公司的数量最多可达 10,000 个，所有人的偏好总数最多可达 100,000 个。 这排除了任何试图枚举所有匹配或以幼稚的方式重复模拟贪婪重新分配的方法。 任何解决方案都需要在边数上表现出接近线性或近线性的特性。 

当有些人的列表为空时，就会出现微妙的边缘情况。 例如，如果某人没有允许的公司，则它们不会对匹配做出任何贡献，但仍然存在于输入中。 当多个人想要同一家公司时，就会出现另一种边缘情况，例如两个人都只列出公司 1。按输入顺序处理人员的天真贪婪分配可以轻松地将公司分配给次优人员，并阻止更好的整体匹配。 

例如，考虑两个人都对同一两家公司感兴趣，但结构不对称：

 输入：```
2 2
2 1 2
1 1
```如果我们立即将人员1分配给公司1，人员2就无法满足，但更好的分配方式是人员1分配给公司2，人员2分配给公司1，实现零未分配公司。 这表明局部贪婪决策可能会失败。 

## 方法

 蛮力策略将尝试探索公司对人们的所有可能分配，同时尊重偏好。 每个人可以在多个公司中进行选择，我们会递归地分配或跳过公司，当出现冲突时回溯。 这很快就会呈指数增长，因为多人共享的每个公司都会创建分支决策，并且在每个人都想要许多公司的最坏情况下，状态数量会随着匹配的阶乘类型搜索而增长。 

关键的观察结果是，我们实际上并不是试图根据人员的利益来最佳地分配人员，而是最大限度地增加匹配公司的数量。 这将问题转化为寻找人和公司之间的最大双向匹配。 每一次成功的匹配都会使未分配的公司数量减少一个，因此最大化匹配相当于最小化剩余公司。 

这种结构允许我们使用经典的匹配技术。 由于边未加权且图是二分图，因此我们可以应用标准的基于 DFS 的增广路径方法（库恩算法）。 这个想法是迭代人员并尝试将每个人分配给一家公司，如果可以实现更好的整体匹配，可能会重新安排以前的分配。 当一家公司已经被占用时，我们会尝试递归地将其当前所有者“推”到另一个可接受的公司，从而将其释放给当前人。 这种局部重新分配机制正是修复了贪婪分配失败的情况。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力搜索分配| 指数| O(N + M) | 太慢了 |
 | Kuhn 的 DFS 增强路径匹配 | O(N·E) 最坏情况，~O(E√V) 典型 | O(N + M + E) | 已接受 |

 ## 算法演练

 我们在二分图的左侧对人员进行建模，在右侧对公司进行建模。 一条边将一个人与其偏好列表中的每个公司连接起来。 

1. 建立邻接列表，每个人都存储他们愿意购买的所有公司。 这种表示允许有效地遍历一个人的所有可能的匹配。 
2. 维护数组`match_to_company`记录当前拥有每家公司的人。 最初，所有公司都未分配，因此该数组为空。 
3. 定义一个深度优先搜索函数，尝试将公司分配给给定的人。 该函数会探索该人列表中的所有公司。 
4. 对于每个公司，检查当前是否未分配。 如果有空闲，立即分配给当前人并返回成功，因为我们已经增大了匹配大小，不存在冲突。 
5. 如果公司已分配给另一个人，请尝试通过递归调用该人员的 DFS 将该现有人员重新分配给其他公司。 如果递归调用成功，则意味着我们已经找到了替代位置，因此我们可以安全地将公司替换为当前人员。 
6. 对所有人重复此过程。 每次进行新的分配时，都可能会触发一系列重新分配，从而在增加匹配总数的同时保留可行性。 
7. 处理完所有人员后，统计还有多少家公司尚未分配。 答案是公司总数减去匹配的公司数。 

正确性背后的关键思想是，每次重新分配公司时，我们都会保留有效的匹配，同时可能为另一次分配释放空间。 DFS 保证如果存在可通过沿着交替路径的本地交换到达的更好的配置，它将被发现。 

## 为什么它有效

 在算法中的任何一点，分配的公司集都会形成有效的匹配。 DFS 搜索探索自由公司和被占领公司之间的交替路径。 每次成功的递归重新分配都对应于遵循一条以自由公司结束的交替路径，这允许翻转整个路径。 这保持了一个不变量，即没有公司被分配给超过一个人，同时尽可能单调增加分配的公司数量。 由于每次成功的 DFS 要么分配一个先前未分配的公司，要么重新配置一条链以释放一个公司，因此当不存在进一步的增广路径时，匹配总数达到最大。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    
    for i in range(n):
        data = list(map(int, input().split()))
        k = data[0]
        if k > 0:
            g[i] = data[1:]

    match = [-1] * (m + 1)
    vis = None

    def dfs(u):
        for v in g[u]:
            if vis[v]:
                continue
            vis[v] = True
            if match[v] == -1 or dfs(match[v]):
                match[v] = u
                return True
        return False

    for i in range(n):
        vis = [False] * (m + 1)
        dfs(i)

    matched = sum(1 for x in match if x != -1)
    print(m - matched)

if __name__ == "__main__":
    solve()
```邻接列表直接存储每个人允许的公司，因此 DFS 只能枚举有效的转移。 这`match`数组按公司索引并存储当前分配的人员。 

每次 DFS 尝试都会重新创建访问过的数组，以防止在单个增强搜索中发生循环，这是必要的，因为图在探索方面是动态的，但在结构上是静态的。 重置`vis`每次外部迭代可确保每次尝试都探索新的搜索树。 

最后的减法`m - matched`将最大匹配规模转换为未分配的公司数量，这就是所需的输出。

 ## 边缘情况

 一个关键的边缘情况是多人列出同一家公司的情况。 在这种情况下，天真的贪婪方法会分配第一个人并阻止其他人，但基于 DFS 的匹配可确保仅保留一个最终分配，并将其余分配正确地计为未分配。 

另一个边缘情况是偏好列表为空的人。 这些节点不参与任何匹配尝试，DFS自然会跳过它们，而不会影响正确性。 该算法仍然正确计算未分配的公司，因为匹配规模仅取决于成功的分配。 

当偏好列表很大并且高度重叠时，就会出现最后一种边缘情况，从而创建很长的增强路径。 递归 DFS 通过沿交替路径重复重新分配直到到达空闲公司来正确处理这些链，确保不保留局部最优但全局次优的配置。
