 ## 解决方案
 ## 问题理解

 该测试包含几个必须按顺序回答的问题。 每个问题都有多个答案选择，其中只有一个是正确的。 瓦甘内奇事先并不知道正确答案，但在犯错后他会记住哪些选项是错误的。 

每当他点击错误答案时，整个测试就会重新开始。 要继续，他必须再次单击所有已知的正确答案，直到到达失败的问题。 

对于每个问题，我们只知道答案变体的数量。 我们必须计算最坏情况下所需的点击次数，假设在找到正确答案之前每个问题都已尽可能糟糕地解决。 

输入只是一个数组，其中`a[i]`是问题可能答案的数量`i`。 输出是一个整数，即最坏情况下的点击总数。 

限制非常小。 最多有 100 个问题，因此任何线性或二次算法都是完全安全的。 答案本身可能会变得很大，因为每个`a[i]`可能达到`10^9`，所以我们必须使用 64 位整数。 Python 会自动处理这个问题。 

棘手的部分是了解什么算作点击。 每次测试重置时，之前解决的问题都必须再次回答。 粗心的实现通常只计算对当前问题的尝试，而忘记了重置导致的重复工作。 

考虑这个例子：```
2
2 2
```正确答案是`5`。 

对于第一个问题，最坏的情况是尝试一个错误的选项，然后尝试正确的选项。 那成本`2`点击次数。 

对于第二个问题，最坏的情况是：

 1.到达问题2，
 2.点击一个错误答案，
 3.从问题1重新开始，
 4.再次正确回答问题1，
 5.最终正确回答第2题。 

这增加了`3`更多点击次数，总计`5`。 

另一个容易犯错误的地方是只有一个选项的问题。```
3
1 1 1
```答案是`3`， 不是`0`。 即使只有一个选择，瓦甘内奇仍然必须点击它。 

当一个问题有很多答案时，就会出现第三种微妙的情况。```
1
5
```答案是`5`。 

没有重置惩罚，因为没有之前的问题需要重新审视。 瓦甘内奇只是尝试了四种错误的选项，然后尝试了正确的选项。 

## 方法

 思考该过程的最直接方法是完全按照故事描述来模拟每次点击。 对于每一个问题，我们都会反复尝试答案，直到找到正确的答案。 每一次错误的尝试都会迫使我们回到问题 1，因此我们会再次重温所有已经解决的问题。 

这种暴力模拟在这里实际上是可行的，因为`n ≤ 100`。 即使每个问题都有很多答案，模拟操作的总数仍然可以在给定的限制范围内进行管理。 该逻辑也很容易验证，因为它直接反映了该语句。 

一旦我们专注于单个问题会发生什么，问题就会变得简单得多。 

假设我们当前正在解决问题`i`，使用基于 0 的索引。 之前的问题`0...i-1`是已知的。 在最坏的情况下，瓦甘内奇会尝试`a[i] - 1`在最终选择正确的答案之前，先选择错误的答案。 

每次错误尝试的成本：

 1.一键点击错误答案，
 2.然后稍后重播所有`i`之前解决过的问题要返回。 

所以每一次错误的尝试都会造成`i + 1`点击次数。 

经过所有错误的尝试后，最终的正确答案需要再点击一次。 

这给出：$$(a_i - 1)(i + 1) + 1$$如果我们扩展这个：$$(a_i - 1)i + a_i$$将所有问题相加即可得出总答案。 

暴力模拟之所以有效，是因为一旦我们假设最坏情况的猜测，该过程就是确定性的。 观察到问题的每个错误答案`i`强制精确重播`i`前面的问题让我们用直接公式代替模拟。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(总和(aᵢ)) | O(1) | O(1) | 对于巨大来说太慢了`aᵢ`|
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 阅读题数`n`和数组`a`。 
2. 初始化`answer = 0`。 
3. 迭代每个问题索引`i`。 
4. 对于问题`i`，计算最坏情况下需要多少次点击。 

问题有`a[i] - 1`在找到正确答案之前就出现错误的答案。 
5.每一次错误的尝试都会付出代价`i + 1`点击次数。 

一次点击花费在错误答案本身上，并且`i`重置后，稍后需要更多点击才能重播之前的正确问题。 
6. 添加`(a[i] - 1) * (i + 1)`到答案。 
7. 再点击一次即可获得该问题的最终正确答案。 
8. 处理完所有问题后，打印总数。 

### 为什么它有效

 解答问题时`i`，第一个`i`问题是已知的，并且必须在每次失败的尝试后重播。 最坏的情况总是意味着在正确的选择之前尝试每一个错误的选择。 

每个错误的选项都恰好贡献了一次失败的点击加上`i`稍后重播点击。 这些成本在尝试之间是独立的，因此我们可以直接将它们相加。 

最终的正确答案不会触发重置，因此只贡献一次点击。 

由于一旦解决了早期问题，每个问题都会独立运行，因此将每个问题的贡献相加即可得出准确的总点击次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

answer = 0

for i in range(n):
    answer += (a[i] - 1) * (i + 1)
    answer += 1

print(answer)
```该实现遵循演练中推导的公式。 

循环索引`i`代表失败后必须重播多少个早期问题。 由于Python使用从0开始的索引，所以有`i`问题之前的较早问题`i`。 

表达式：```
(a[i] - 1) * (i + 1)
```计算当前问题的所有失败尝试。 这`+1`里面的乘法很容易出错。 它包括点击错误答案本身。 

全部失败后，还需要再点击一次才能得到最终的正确答案：```
answer += 1
```Python 整数自动支持任意大的值，因此即使答案变大也不存在溢出风险。

 ## 边缘情况

 考虑尽可能最小的输入：```
1
1
```该算法计算：$$(1 - 1)(0 + 1) + 1 = 1$$没有错误的尝试，但仍然需要一键选择唯一的答案。 输出正确`1`。 

现在考虑一个重置非常重要的情况：```
2
2 2
```对于第一个问题：

 - 一次错误的点击，
 - 一次正确的点击。 

到目前为止总计：`2`。 

对于第二个问题：

 - 第 2 题点击错误，
 - 重置，
 - 重播问题 1，
 - 正确点击问题 2。 

这有助于`3`。 

该算法计算：$$(2-1)(1)+1 = 2$$对于问题 1，以及$$(2-1)(2)+1 = 3$$对于问题 2，给出`5`。 

最后，考虑一个更大的重播链：```
3
1 2 2
```问题1贡献`1`。 

问题2贡献：$$(2-1)(2)+1 = 3$$问题 3 贡献：$$(2-1)(3)+1 = 4$$全部的：$$1 + 3 + 4 = 8$$关键细节是，问题 3 的错误答案会迫使您重温之前的两个问题。 该公式通过因子自动处理这个问题`(i + 1)`。
