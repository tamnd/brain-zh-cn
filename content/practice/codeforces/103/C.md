 ## 解决方案
 ## 问题理解

 气缸有`n`槽排列成一圈，并且恰好`k`其中包含子弹。 在游戏开始之前，圆柱体随机均匀旋转，因此每次循环移位的可能性是相等的。 

轮换后，萨沙先投篮。 如果当前槽位为空，则圆柱体前进一个位置，罗马射门。 第一个面对子弹的玩家会立即死亡。 

任务不是模拟游戏。 相反，我们必须构建子弹的排列，以最大限度地减少萨沙的死亡概率。 在所有最优排列中，我们必须输出字典顺序最小的一个，其中`.`小于`X`。 

输入并不要求我们直接打印整个字符串。 自从`n`可以大到`10^18`，打印所有位置是不可能的。 相反，我们收到`p`询问特定位置的查询，对于每个查询的索引，我们打印该槽是否包含项目符号。 

巨大的界限`n`彻底改变了问题的本质。 任何算法正比于`n`是不可能的。 我们无法构建整个字符串、模拟游戏，甚至无法一次遍历所有位置。 该解决方案必须仅使用算术公式和本地决策来工作。 

微妙的部分是了解决定萨沙失败概率的因素。 

假设游戏在随机轮换后的某个时段开始。 

如果第一颗子弹出现在偶数个空槽之后，那么萨莎最终会面对它并死亡。 

如果第一颗子弹出现在奇数个空位之后，罗马就会死亡。 

这意味着连续空槽的每个最大块都会根据其奇偶性贡献获胜或失败的起始位置。 

天真的直觉是，均匀分布子弹总是最佳的。 这还不够。 我们还需要字典顺序最小的最优排列。 

例如：

 输入：```
5 2
```一个平衡的安排可能是：```
X..X.
```但：```
..X.X
```给出相同的概率，但按字典顺序更小，因为它尽可能地延迟子弹。 

另一个容易犯的错误是忘记圆柱体是循环的。 

例如：```
n = 6
k = 2
```安排：```
X...X.
```包含一个长度的间隙`3`和一个长度间隙`1`，不是两个单独的线性间隙。 第一个和最后一个位置在圆中相邻。 

案例`k = 0`也很重要。 没有人会死亡，所以萨沙的损失概率是`0`。 唯一有效的排列是全点。 

案例`k = n`是相反的极端。 每个槽位都有一颗子弹，萨莎有概率立即死亡`1`，并且安排是全部`X`。 

## 方法

 蛮力方法将枚举每个可能的位置`k`之间的子弹`n`插槽。 对于每种安排，我们可以通过检查所有内容来计算 Sasha 失败的概率`n`循环起始位置。 

有`C(n, k)`安排，并且每个人都采取`O(n)`来评估。 即使对于中等值，例如`n = 50`，这变得天文数字般大。 

关键的观察结果是游戏结果仅取决于子弹之间空段的长度。 

假设两颗子弹之间的间隙有长度`g`。 

如果游戏在这个间隙内开始，玩家会轮流穿过它。 到达子弹的玩家仅取决于距离的平价。 

长度间隙内`g`:

 -`ceil(g / 2)`首发位置让萨沙输了。 
-`floor(g / 2)`首发位置让罗马输球。 

因此，每一个奇数长度的差距都会为萨莎带来一个额外的失败位置。 

这立即将问题转变为组合优化问题：

 我们想要拆分`n - k`空插槽进入`k`循环间隙，使得奇数间隙的数量最小化。 

如果`n - k`是均匀的，我们可以使所有间隙均匀，达到完美的平衡。 

如果`n - k`为奇数，则至少有一个间隙必须为奇数。 

因此奇数间隙的最小可能数量是：

 -`0`如果`n - k`是偶数
 -`1`否则

 在最小化奇数间隙的数量之后，我们仍然需要字典顺序最小的排列。 

字典序越小意味着尽可能将项目符号延迟到右侧。 由于间隙决定了子弹的位置，因此我们希望早期的间隙尽可能大。 

所有最优解中：

 - 除可能有一个奇数间隙外，所有间隙均应为偶数，
 - 早期的差距应该贪婪地最大化。 

这导致了一个建设性的算术解决方案，可以独立地回答每个查询的位置，而无需构建完整的字符串。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数| 指数| 太慢了 |
 | 最佳| O(k + p) | O(k) | 已接受 |

 ## 算法演练

 1. 让`m = n - k`，空槽数。 
2.我们必须分发这些`m`空插槽进入`k`子弹之间的循环间隙。 
3. 为了最小化 Sasha 失败的概率，最小化奇数间隙的数量。 

如果`m`是均匀的，每个间隙都可以是均匀的。 

如果`m`是奇数，则恰好有一个间隙必须是奇数。 
4. 要获得字典顺序最小的排列，请最大化较早的间隙。 

较早的点会推迟较早的项目符号，这使得字符串按字典顺序更小。 
5. 从左到右贪婪地构造间隙。 

每个间隙必须保持全局奇偶条件：

 - 大多数间隙应该是均匀的，
 - 最多允许一个奇数间隙，当`m`很奇怪。 
6. 将尽可能多的空槽放入当前间隙，同时为剩余间隙保留足够的槽。 
7. 确定所有间隙后，从概念上重建子弹位置。 

从位置开始`1`， 地方：

-`gap[0]`点，
 - 一颗子弹，
 -`gap[1]`点，
 - 一颗子弹，
 - 如此循环往复。 
8. 自从`n`可能很大，永远不要构建完整的字符串。 

相反，将项目符号位置存储在集合中并仅回答查询的索引。 

### 为什么它有效

 概率仅取决于每个空间隙的奇偶性。 均匀的差距对获胜和失败的开局贡献相同。 奇怪的差距为萨沙带来了额外的失败。 

因为空时隙的总数具有固定的奇偶校验，所以奇数间隙的最小可能数量仅由奇偶校验强制。 

确定最佳奇偶校验结构后，字典顺序由最早的不同位置确定。 最大化较早的间隙会尽可能向右延迟项目符号，从而使字符串按字典顺序最小化。 

贪婪的构造之所以有效，是因为只要保留剩余的奇偶约束，选择较大的早期间隙就永远不会损害未来的可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, p = map(int, input().split())

    queries = [int(input()) for _ in range(p)]

    if k == 0:
        ans = ['.'] * p
        print(''.join(ans))
        return

    if k == n:
        ans = ['X'] * p
        print(''.join(ans))
        return

    m = n - k

    gaps = []

    odd_needed = m % 2
    remain = m

    for i in range(k):
        left = k - i - 1

        # largest feasible gap
        g = remain - left

        # adjust parity
        if odd_needed == 0:
            if g % 2 == 1:
                g -= 1
        else:
            if g % 2 == 0:
                g -= 1

        gaps.append(g)

        remain -= g

        if g % 2 == 1:
            odd_needed = 0

    bullets = set()

    pos = 1

    for g in gaps:
        pos += g
        bullets.add(pos)
        pos += 1

    out = []

    for x in queries:
        if x in bullets:
            out.append('X')
        else:
            out.append('.')

    print(''.join(out))

solve()
```第一个特殊情况处理`k = 0`和`k = n`。 这些避免了构造逻辑中的极端情况，也避免了无效的循环解释。 

变量`m`存储空槽的总数。 我们将这些分发到`k`差距。 

贪心步骤选择最大的可行当前间隙。 条件：```
g = remain - left
```意味着我们为每个剩余的间隙至少留出一个位置。 

然后我们调整奇偶校验。 如果我们仍然需要奇数间隙，则当前间隙应该是奇数。 否则应该是均匀的。 

一个微妙的点是，减少 1 总是保留可行性，因为奇偶校验恰好改变 1，并且我们已经为以后的间隙保留了足够的空间。 

重建阶段永远不会创建完整的字符串。 相反，它仅计算包含项目符号的位置。 由于只有`k`子弹，这个效率很高。 

位置是 1 索引的，因为查询是 1 索引的。

 ## 边缘情况

 考虑：```
1 0 1
1
```没有子弹。 游戏永远不会结束，所以萨莎失败的概率是`0`。 该算法立即返回所有点，而不进入构建阶段。 

现在考虑：```
4 4 4
1
2
3
4
```每个插槽都包含一颗子弹。 萨莎总是立刻死去。 该算法再次直接处理这个问题并输出所有`X`。 

更微妙的循环情况：```
6 2
```粗心的线性解释可能会认为：```
...X.X
```有间隙`3`和`1`。 

但最终循环往复`X`连接回起点，所以间隙实际上是`0`和`4`。 

该算法以循环顺序显式构造间隙，因此它永远不会犯此错误。 

最后：```
7 2
```我们有`m = 5`，这很奇怪。 至少有一个间隙必须是奇数。 

施工选择：```
5, 0
```给予：```
.....XX
```恰好有一个奇数间隙，这是最佳的，并且最早的项目符号被尽可能地推到最右边，这保证了词典编排的最小性。
