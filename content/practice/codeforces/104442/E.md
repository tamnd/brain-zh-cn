 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例描述了单向街道上的一系列建筑物高度。 这座城市想要改变这个顺序，以便当我们从左向右移动时，高度永远不会降低。 

唯一允许的操作是选择一个连续的段并强制该段中的所有建筑物具有完全相同的高度。 这种操作的成本是应用于该段中每座建筑物的总绝对变化。 

目标不是最小化操作数量，而是最小化将原始数组转换为通过这些段重写操作可到达的任何非递减序列所需的总成本。 

一个关键的观察是，我们不限制执行多少操作，也不限制最终形状，除了它必须是非递减的。 这意味着最终数组的结构是完全灵活的，只要它尊重单调性，并且成本仅取决于我们如何“重塑”原始值。 

约束足够小，每个测​​试用例的 O(n^2) 或 O(n^3) 动态编程方法都是可行的，但我们必须小心，因为所有测试用例的 n 总和仅为 5000。这强烈表明每个测试用例的二次解是可以接受的，但必须优化掉任何三次解。 

一种天真的解释可能会建议尝试模拟段操作或贪婪地在本地修复违规行为，但这会失败，因为更改一个段可能会以非本地方式与未来的选择交互。 

一个更微妙的问题是，最优转换可能会以从局部贪婪角度来看并不明显的方式重复覆盖重叠段。 例如，通过局部扁平化贪婪地修复每个下降可能会增加以后的成本，因为它破坏了重用已经对齐的值的机会。 

## 方法

 一个蛮力的想法是直接从操作的角度思考：选择产生非递减数组的任何段分配序列。 然而，枚举所有分段是指数级的，因为每个步骤都会选择一个分段和目标高度，并且此类操作的序列严重重叠。 

相反，我们重新解释问题：最终的数组是一些非递减序列，每个段操作只是支付成本以强制块达到选定的常量值的一种方式。 如果我们修复最终数组，则成本在各个位置之间变得独立：每个索引贡献其原始值与其最终分配值之间的绝对差，并对所有索引求和。 

所以问题归结为选择一个非递减的目标数组`b`最小化：

 总和 |p[i] - b[i]|

 现在的问题纯粹是一个约束优化：我们想要一个在 L1 距离上最接近原始序列的非递减序列。 

这是一个经典的动态规划结构。 让`dp[i][v]`是第一个的最低成本`i`元素，如果第 i 个最终值恰好是`v`。 转型需要`b[i] >= b[i-1]`，它引入了前缀最小约束`v`。 

由于值最多为 5000，因此我们可以使用候选值上的前缀最小值来优化转换。 

关键的见解是，对于固定 i，选择 b[i] = v 会贡献 |p[i] - v|，并且我们只关心最后一个值 ≤ v 的最佳先前状态。这将每个转换变成 dp[i-1] 上的前缀最小查询。 

因此，我们逐行构建 dp，扫描可能的高度并维护前缀最小值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力操作 | 指数| 高| 太慢了 |
 | 具有前缀最小值的 DP over 值域 | O(n * 最大A) | O(最大A) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

## 步骤 1：将问题解释为选择非递减目标数组

 我们用更简单的等价物替换操作的概念：每个位置以选定的高度结束，并且最终序列必须是非递减的。 成本是原始值和所选值之间的绝对差之和。 

这是有效的，因为任何最终的单调序列都可以通过段操作构造，并且任何操作序列最终都定义了这样的目标数组。 

## 步骤 2：定义动态规划状态

 我们定义`dp[v]`作为处理数组当前前缀的最小成本，以最终值精确结束`v`。 

在初始化时，对于第一个元素，我们直接计算 dp[v] = |p[0] - v|。 

## 步骤 3：每个新职位的过渡

 对于每个新元素 p[i]，我们计算一个新数组`new_dp`。 

对于位置 i 处的候选最终值 v，我们必须确保先前值 ≤ v。因此，我们需要：

 new_dp[v] = |p[i] - v| + min(dp[u]) 对于所有 u ≤ v

 内部最小值是 dp 上的前缀最小值。 

我们在将 v 从 1 迭代到 MAXA 时保持运行前缀最小值。 

## 步骤 4：更新 dp

 计算完 new_dp 后，我们用它替换 dp 并继续下一个索引。 

## 步骤5：答案提取

 处理完所有元素后，答案是所有 v 的 min(dp[v])。 

### 为什么它有效

 DP 精确编码最终高度的所有单调分配。 前缀最小值确保允许任何有效的先前高度，同时强制执行非递减顺序。 每个状态代表在该高度结束的所有序列中的最佳成本，因此不会错过任何可行的配置，并且不允许无效的递减转换。 

由于每次转换都会通过前缀最小值考虑所有可能的有效先前值，因此递归会探索整个解空间，而无需显式枚举序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    MAXV = 5000

    for _ in range(t):
        n = int(input())
        p = list(map(int, input().split()))

        dp = [0] * (MAXV + 1)

        for v in range(1, MAXV + 1):
            dp[v] = abs(p[0] - v)

        for i in range(1, n):
            new_dp = [0] * (MAXV + 1)

            best = float('inf')
            for v in range(1, MAXV + 1):
                if dp[v] < best:
                    best = dp[v]
                new_dp[v] = best + abs(p[i] - v)

            dp = new_dp

        print(min(dp[1:]))

if __name__ == "__main__":
    solve()
```初始化步骤对强制第一个元素达到每个可能高度的成本进行编码。 转换循环小心地维护前缀最小值`best`，它表示 v 之前最便宜的有效高度。这正是单调性约束。 

最终答案采用所有可能的结束高度中的最小值，因为最后一个值不受超出可行性的约束。

 ## 边缘情况

 一个关键的边缘情况是完全递减的数组，例如`[5,4,3,2,1]`。 贪婪的修复会反复压平相邻对并支付过高的费用，但 DP 相反会选择一个平滑的单调目标，例如`[3,3,3,3,3]`或类似的取决于最佳平衡。 前缀最小值确保每个位置都可以继承最佳的早期高度，而无需强制进行局部修正。 

对于单元素测试，DP 会正确初始化，因为无论选择什么值，答案都为零，并且所有最终高度的最小值正确反映了不需要任何操作。 

对于交替序列，例如`[1,100,1,100]`，天真的直觉可能会建议交替修正，但 DP 正确地积累了全局一致的单调目标，避免了违反排序约束的振荡决策。
