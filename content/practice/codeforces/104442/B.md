 ## 解决方案
 ## 问题理解

 我们有一个在无限整数网格上移动的机器人。 它从给定的坐标开始，必须到达目标坐标，同时避开一组无法踩到的被阻挡的网格点。 机器人不占据区域，它被视为单个点，因此仅在精确坐标处检查碰撞。 

移动规则不是通常的最短路径规则。 机器人有一个方向，始终与四个轴方向之一对齐。 从网格点开始，它可以沿当前方向向前移动一个单位，也可以向左或向右旋转 90 度。 前进需要花费一个单位时间。 旋转需要四个时间单位。 旋转后，如果机器人再向前移动，该移动仍会额外花费一个单位时间。 

一个微妙但重要的规则是，在起始位置，机器人可以免费自由选择其初始方向。 目标是计算到达目的地所需的最短总时间，无论最终面向的方向如何。 

尽管运动是基于网格的，但成本结构使其不同于标准 BFS 最短路径。 旋转比移动要昂贵得多，因此最佳路径不一定是步数最少的路径。 

从约束的角度来看，网格可能很大或无界，但阻塞单元的数量是有限的，并且通常小到足以存储在哈希集中。 这强烈表明我们不应该尝试构建显式网格。 相反，我们只生成根据移动规则从一开始就实际可达的状态。 由于每个状态都有位置和方向，因此自然状态空间乘以四。 

这立即排除了仅针对头寸的任何未加权 BFS。 我们也无法承受所有路径的简单递归或强力枚举，因为分支因子增长很快并且每个操作的成本都不同。 需要具有权重的最短路径算法。 

一个常见的失败案例来自于忽视作为国家一部分的方向。 例如，朝北到达某个单元格并不等于朝东到达该单元格，因为由于旋转惩罚较高，未来的成本在很大程度上取决于方向。 

## 方法

 暴力方法会尝试枚举从起点到目标的所有可能路径，探索每个移动和旋转序列。 这在理论上是正确的，因为考虑了每条有效路径，但可能的动作序列的数量随着路径长度呈指数增长。 即使是适度数量的步骤也会产生大量难以处理的组合，因为在每个单元中机器人都可以旋转或移动，并且可以在不改变位置的情况下重复旋转。 

关键的观察结果是，这是隐式加权图上的最短路径问题。 每个状态不仅由位置定义，还由方向定义。 每个状态最多有 3 个转换：以成本 1 向前移动，以成本 4 向左旋转，以成本 4 向右旋转。这将问题转换为具有非负权重的图，这正是 Dijkstra 算法应用的设置。 

重要的结构简化是，我们永远不需要考虑以更高成本重新访问相同状态的路径。 一旦我们治疗`(x, y, direction)`作为一个节点，我们可以安全地使用优先级队列以成本递增的顺序扩展状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举路径| 指数| 指数| 太慢了 |
 | Dijkstra 在 (x, y, 方向) 状态 | O(E log V) | O(V) | 已接受 |

 ## 算法演练

 我们将机器人的每种配置建模为由其位置和当前面向方向组成的状态。 然后我们在这些状态上运行最短路径算法。 

1. 我们初始化一个优先级队列，其起始位置在所有四个可能的方向上，每个方向的成本为零。 该模型模拟了可以自由选择初始方向而不会受到惩罚的规则。 将所有方向视为有效的起始状态可以防止我们错过需要初始方向改变的解决方案。 
2. 我们维护一个距离字典，键为`(x, y, direction)`并存储到达每个州的最已知成本。 任何以较高成本重新访问的状态都会被忽略。 
3. 在每一步中，我们从优先级队列中提取累积成本最小的状态。 这保证了当我们处理一个状态时，我们已经有了它的最优成本。 
4、从目前的状态来看，我们考虑向左和向右旋转。 每次旋转都会保持位置不变，但会改变方向，并增加 4 的成本。如果我们找到更便宜的方法来到达这些相邻状态，我们就会放宽它们。 
5.我们也考虑按照目前的方向前进。 这会产生一个具有相同方向的新位置状态，并增加成本 1。但是，只有在下一个单元未被阻塞的情况下，我们才允许这种转换。 
6. 我们继续扩展状态，直到优先级队列为空或到达目标位置。 答案是目标坐标所有方向中成本最小的。 

正确性背后的关键思想是，每个合法的动作序列都对应于该状态图中的一条路径，并且每个路径成本都由边权重之和精确表示。 由于 Dijkstra 总是按照成本递增的顺序进行探索，因此当我们第一次确定一个状态时，我们就已经找到了达到该状态的最佳方法。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

# direction encoding: 0=N, 1=E, 2=S, 3=W
dx = [-1, 0, 1, 0]
dy = [0, 1, 0, -1]

def solve():
    sx, sy = map(int, input().split())
    tx, ty = map(int, input().split())
    n = int(input().strip())

    blocked = set()
    for _ in range(n):
        x, y = map(int, input().split())
        blocked.add((x, y))

    INF = 10**18
    dist = {}

    pq = []

    # free initial orientation
    for d in range(4):
        dist[(sx, sy, d)] = 0
        heapq.heappush(pq, (0, sx, sy, d))

    while pq:
        cost, x, y, d = heapq.heappop(pq)

        if dist.get((x, y, d), INF) != cost:
            continue

        if x == tx and y == ty:
            print(cost)
            return

        # rotate left
        nd = (d + 3) % 4
        nc = cost + 4
        if dist.get((x, y, nd), INF) > nc:
            dist[(x, y, nd)] = nc
            heapq.heappush(pq, (nc, x, y, nd))

        # rotate right
        nd = (d + 1) % 4
        nc = cost + 4
        if dist.get((x, y, nd), INF) > nc:
            dist[(x, y, nd)] = nc
            heapq.heappush(pq, (nc, x, y, nd))

        # move forward
        nx = x + dx[d]
        ny = y + dy[d]
        if (nx, ny) not in blocked:
            nc = cost + 1
            if dist.get((nx, ny, d), INF) > nc:
                dist[(nx, ny, d)] = nc
                heapq.heappush(pq, (nc, nx, ny, d))

    print(-1)

if __name__ == "__main__":
    solve()
```该解决方案按需构建隐式状态图，而不是预先计算任何网格。 优先级队列确保按成本递增顺序处理状态，这是必要的，因为旋转边缘比移动边缘具有更高的权重。 

一个微妙的实现细节是开始时所有四个方向的初始化。 如果没有这个，算法将错误地假设固定的初始方向，并且可能会错过需要从不同方向开始的最佳解决方案。 

另一个重要的细节是我们存储每个距离`(x, y, direction)`而不是每个坐标。 方向崩溃将合并未来成本根本不同的状态并破坏最优性。

| 步骤| 状态| 成本| 行动|

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O((V + E) log V) | 迪杰斯特拉 (Dijkstra) 的状态`(x,y,dir)`每个位置最多有 4 个方向 |
 | 空间| O(V) | 存储每个访问过的状态的最佳距离 |

 访问状态的数量取决于可到达的网格单元而不是完整的坐标范围，这使得算法在实践中保持高效。 每个状态最多生成三个转换，因此常数因子仍然很小。 这完全符合带有障碍物的网格最短路径问题的典型约束。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import heapq

    dx = [-1, 0, 1, 0]
    dy = [0, 1, 0, -1]

    sx, sy, tx, ty, *rest = map(int, inp.split())
    n = rest[0]
    blocked = set()
    idx = 1
    for _ in range(n):
        blocked.add((rest[idx], rest[idx+1]))
        idx += 2

    INF = 10**18
    dist = {}
    pq = []

    for d in range(4):
        dist[(sx, sy, d)] = 0
        heapq.heappush(pq, (0, sx, sy, d))

    while pq:
        cost, x, y, d = heapq.heappop(pq)
        if dist.get((x, y, d), INF) != cost:
            continue
        if x == tx and y == ty:
            return str(cost)

        for nd in [(d+3)%4, (d+1)%4]:
            nc = cost + 4
            if dist.get((x, y, nd), INF) > nc:
                dist[(x, y, nd)] = nc
                heapq.heappush(pq, (nc, x, y, nd))

        nx, ny = x + dx[d], y + dy[d]
        if (nx, ny) not in blocked:
            nc = cost + 1
            if dist.get((nx, ny, d), INF) > nc:
                dist[(nx, ny, d)] = nc
                heapq.heappush(pq, (nc, nx, ny, d))

    return "-1"

 ## 边缘情况

 常见的边缘情况是目标与起始位置相同。 该算法可以正确处理此问题，因为所有四个初始状态都已从目标坐标开始，因此第一个弹出状态立即触发终止，成本为零。 

另一个微妙的情况是当障碍物迫使机器人从特定方向接近细胞时。 由于方向是在状态中编码的，因此该算法可以区分到达的坐标是否面向有用的方向和不可用的方向，从而确保正确评估强制绕行。 

进一步的边缘情况是，在同一单元中多次旋转比稍后移动和纠正方向更便宜。 Dijkstra 框架自然地探索了这两种可能性，因为旋转边缘是显式建模的，并且如果可以降低总成本，则可以重复应用。
