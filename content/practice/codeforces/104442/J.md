 ## 问题理解

 该网格描述了一个黑白图像，其中大多数单元格是背景，其余单元格属于飞机。 每个飞机单元格都标有从 0 到 9 的数字，所有共享相同数字的单元格形成一个“区域类型”。 任务是决定应加强哪种区域类型。 

除了图像之外，我们还获得了撞击点列表。 每次撞击都会落在有效的飞机单元上，这意味着它总是落在数字标记的像素之一上。 对于每个数字，我们计算有多少次影响击中了该数字的像素。 我们还知道每个数字的区域有多大，这是通过有多少个网格单元携带该数字来衡量的。 

决策规则是分层的。 首先，我们选择命中次数最少的数字。 如果多个数字相等，我们更喜欢区域在图像中占据较少像素的数字。 如果仍然存在平局，我们选择具有最小数字标识符的数字。 

约束很小：网格最多 100 x 100，最多有 2000 个影响。 这意味着直接扫描整个图像并对每次影响进行简单计数很容易足够快，因为所有操作都与网格大小加上影响数量呈线性关系。 

一个常见的错误是假设数字总是出现在连续或单个连接的组件中，但这在这里无关紧要。 另一个微妙的陷阱是忘记某些数字可能根本不会出现在网格中； 这些必须排除在考虑之外，因为它们不代表任何区域。 

## 方法

 直接解决方案为从 0 到 9 的每个数字计算两条信息：网格中有多少个单元格包含该数字，以及有多少影响落在该数字上。 两者都可以通过网格的单次传递和影响列表的单次传递来累积。 

回答每个影响的强力方法是扫描整个网格并检查它落在哪里，但这将使每个查询的时间复杂度为 O(MN)，从而导致总工作量为 O(DMN)，这是不必要的，因为我们可以直接索引到网格中以查找影响位置处的数字。 

关键的观察是网格已经编码了我们分类所需的所有内容。 每个影响只是对预先计算的标签的查找，因此我们可以在每个事件的恒定时间内累积计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（每次冲击扫描网格）| O(D·M·N) | O(M·N) | 太慢了 |
 | 最优（预计算+计数）| O(M·N + D) | O(M·N + D) | O(M·N) | 已接受 |

 ## 算法演练

 1. 读取网格尺寸并将网格存储为字符串列表。 每个单元格都包含“.” 或数字字符。 我们只关心数字单元格，因为它们定义了有效区域。 
2. 初始化一个大小为 10 的数组，统计每个数字属于多少个单元格。 我们还初始化另一个大小为 10 的数组来计算每个数字有多少次撞击。 
3. 扫描网格中的每个单元格。 每当我们看到数字字符时，将其转换为整数并增加其大小计数器。 这会在线性时间内构建每个区域类型的总面积。 
4. 读取冲击次数，然后处理每个冲击坐标。 对于每个坐标，直接访问网格单元并将其转换为数字。 增加相应的命中计数器。 这是有效的，因为每次撞击都保证落在数字单元上。 
5. 处理完所有输入后，迭代从 0 到 9 的数字，并使用比较规则选择最佳候选：首先是最小的命中数，然后是最小的区域大小，然后是最小的数字值。 
6. 输出选择的数字。 

### 为什么它有效

每个影响都独立地贡献于一位数字类别，因此计数只是固定标签上事件的划分。 网格扫描提供每个标签的精确权重，冲击扫描提供精确的频率。 由于这两个数量都是按数字完全聚合的，因此比较数字可以简化为预先计算的统计数据的简单字典序最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    M, N = map(int, input().split())
    grid = [input().strip() for _ in range(N)]

    size = [0] * 10
    hits = [0] * 10

    for y in range(N):
        row = grid[y]
        for x in range(M):
            c = row[x]
            if c != '.':
                size[int(c)] += 1

    D = int(input())
    for _ in range(D):
        y, x = map(int, input().split())
        d = grid[y][x]
        hits[int(d)] += 1

    best = None
    for d in range(10):
        if size[d] == 0:
            continue
        cand = (hits[d], size[d], d)
        if best is None or cand < best:
            best = cand

    print(best[2])

if __name__ == "__main__":
    solve()
```该解决方案首先构建区域大小和影响计数的频率表。 网格遍历是对所有单元格的直接嵌套循环，确保每个数字的总面积一次就知道。 第二阶段通过直接索引到网格中来在恒定时间内处理每个影响。 

最终的选择步骤依赖于Python的元组比较，它自然地实现了所需的优先级顺序：最小化命中，然后最小化大小，然后最小化数字。

 ## 边缘情况

 一个重要的情况是当一个数字出现在网格中但收到零命中时。 在这种情况下，它成为一个强有力的候选者，因为零是最小可能的命中数。 该算法自然地处理这个问题，因为所有数字都以零命中进行初始化，并且只考虑出现在网格中的数字。 

另一种情况是多个数字的命中为零。 例如，如果数字 2 和 8 都存在但没有受到影响，则决策会落到最小区域大小。 选择循环显式地比较命中计数后的大小，因此选择正确的较小区域。 

最后一个微妙的情况是，从 0 到 9 的某些数字从未出现在网格中。 这些必须被排除； 否则它们会错误地显示为零大小区域。 支票`if size[d] == 0: continue`确保仅考虑有效区域。
