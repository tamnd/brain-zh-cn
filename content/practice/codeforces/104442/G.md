 ## 解决方案
 ## 问题理解

 我们得到一个固定长度的二元元胞自动机。 每个配置都是一行 C 个单元，其中每个单元都是 0 或 1。规则 R（从 0 到 255）定义如何生成下一行：每个位置查看当前行中三个连续单元的块，将该三元组作为 3 位数字，并使用 R 来决定下一行中的结果位。 

任务不是模拟前进。 相反，我们给定一个目标配置 A，并且必须确定是否存在任何先前的配置 B 可以在给定规则下演变成 A。 如果不存在这样的B，则A称为伊甸园配置，我们输出SI，否则我们输出NO。 

网格的宽度很小，C 最多为 30，但测试用例的数量很大，最多 5000 个。这种组合强烈表明每个测试必须在大约 O(C) 或 O(C log C) 时间内解决，因为每个测试的任何二次方都已经太慢了。 

一个微妙的点是数组外部的边界单元被固定为0并且永远不会改变。 这意味着第一个和最后一个转换受到虚拟零邻居的约束，并且任何有效的前驱都必须遵守这一点。 

一个天真的误解是认为我们可以贪婪地从左到右重建前任。 但这会失败，因为每个单元都依赖于三个连续的先前位，因此选择重叠并产生全局约束。 

第二个常见错误是尝试暴力破解所有可能的前辈。 当 C 达到 30 时，有 2^30 个候选者，每个测试已经有大约 10 亿种可能性，即使在考虑 5000 个测试用例之前也是完全不可行的。 

## 方法

 蛮力方法尝试每一种可能的长度为 C 的先前配置 B，并模拟自动机的一个步骤，看看它是否与 A 匹配。每次模拟的成本为 O(C)，因此每次测试的总成本为 O(C · 2^C)。 当 C = 30 时，这会立即爆炸，其中状态空间约为 10^9 配置。 

关键结构是有效性是局部的：每个位置 i 仅取决于 (B[i−1], B[i], B[i+1])。 这意味着我们不需要独立探索完整的字符串。 相反，我们可以一点一点地构建前驱，同时保留足够的上下文来强制执行局部约束。 

解锁解决方案的观察结果是，当从左到右扫描时，决定是否可以扩展部分分配所需的唯一信息是最后两个选定的位。 一旦我们固定了 B[i−1] 和 B[i]，下一位 B[i+1] 就仅受其是否产生规则下所需输出 A[i] 的约束。 

这将问题转化为由相邻位对定义的状态上的路径存在问题，可以通过位置动态规划来解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(C·2^C) | O(C·2^C) | O(C)| 太慢了 |
 | 配对 DP | O(C·4) | O(4) | 已接受 |

 ## 算法演练

 我们将未知前驱视为长度为 C 的二进制字符串 B，具有两个固定虚拟边界位 B[0] = 0 和 B[C+1] = 0。 

然后，我们尝试从左到右构造 B，同时确保每个位置 i 与其相邻位置组合时都会产生所需的 A[i]。

1. 通过将位置 1 的假想左邻居设置为 0 来修复边界条件。这可确保第一个转换明确定义并与问题陈述一致。 
2. 将位置 i 处的动态定义为对 (B[i−1], B[i])。 这是足够的，因为位置 i 处的约束恰好涉及这两位加上下一位 B[i+1]。 
3. 在处理位置 1 之前，用唯一可能的起始对 (0, 0) 初始化 DP。这编码 B[0] 固定为 0 并且 B[1] 尚未选择。 
4. 对于从 1 到 C 的每个位置 i，尝试所有可能的当前状态 (x_{i−1}, x_i)。 对于每个状态，尝试通过在 {0, 1} 中选择 x_{i+1} 来扩展它。 仅当将规则应用于 (x_{i−1}, x_i, x_{i+1}) 产生 A[i] 时，扩展才有效。 这会立即强制执行本地一致性，而不是推迟它。 
5. 只要存在有效扩展，就转换到下一个状态 (x_i, x_{i+1})。 这会将窗口向右移动一步，同时保留未来约束所需的所有信息。 
6. 处理位置 C 后，通过要求最后计算的转移与 B[C+1] = 0 一致来强制执行最终边界条件，这意味着只有 x_C 和 0 的状态必须满足位置 C 处的规则。 
7. 如果末尾存在任何有效的 DP 状态，则输出 NO，因为至少存在一个前驱状态。 否则输出SI。 

正确性依赖于这样一个事实，即在引入 x_{i+1} 时，会精确检查涉及位置 i 的每个约束，因此没有无效的部分赋值会在其违规点之外继续存在。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def rule_output(R, a, b, c):
    idx = (a << 2) | (b << 1) | c
    return (R >> idx) & 1

def solve_case(R, C, A):
    # dp over states (prev, cur)
    dp = [[False, False], [False, False]]
    dp[0][0] = True  # B[0]=0, B[1]=0 initially

    for i in range(1, C + 1):
        ndp = [[False, False], [False, False]]
        ai = (A >> (C - i)) & 1

        for p in range(2):
            for q in range(2):
                if not dp[p][q]:
                    continue
                for nxt in range(2):
                    if rule_output(R, p, q, nxt) == ai:
                        ndp[q][nxt] = True

        dp = ndp

    # enforce boundary: B[C+1] = 0
    return any(dp[p][0] for p in range(2))

def main():
    N = int(input())
    out = []
    for _ in range(N):
        R, C, A = map(int, input().split())
        out.append("NO" if solve_case(R, C, A) else "SI")
    print("\n".join(out))

if __name__ == "__main__":
    main()
```DP表存储直到位置i的部分前驱是否可以以给定的位对结束。 每次转换都会通过提取正确的 3 位邻域并将其与位置 i 处的目标配置位进行比较来检查自动机规则。 从 A 中提取位是从最高有效位到最低有效位进行的，以便位置索引与从左到右的遍历相匹配。 

最后的检查通过仅接受最后构造的位可以一致地跟随 0 的状态来强制右边界为零。

 ## 边缘情况

 关键的边缘情况是规则允许多个延续，但边界条件删除了所有延续。 例如，对于像 R = 255 这样的宽松规则，每个三元组都会产生 1，因此系统倾向于在内部强制所有的值，但最终的要求 B[C+1] = 0 会破坏一致性，除非 C 非常小。 

另一个边缘情况是 A = 0，对于像 R = 0 这样的限制性规则，每个三元组都生成 0。在这种情况下，任何前驱都是有效的，并且 DP 应该正确报告“否”，因为存在解决方案，即使结构可能表明存在歧义。 

DP 统一处理这两种情况，因为每个约束都是在每个步骤中本地强制执行的，并且最终的边界检查是唯一可以消除其他有效构造的全局限制。
