 ## 解决方案
 ## 问题理解

 我们正在使用一个机器模型，其中所有算术都是使用 32 位无符号整数执行的。 这意味着每个结果都会模数减少$2^{32}$当它超出了可表示的范围时。 换句话说，乘法不会产生无界整数，它会环绕模数$2^{32}$。 

对于每个测试用例，我们都会给出一个值$N$。 我们从概念上考虑所有可能的 32 位无符号值$P$。 对于每一个这样的$P$，我们计算产品$N \cdot P$使用32位溢出算法，检查结果是否等于$P$再次按照相同的算术规则。 任务是计算有多少个值$P$满足这个定点条件。 

所以核心问题就变成了：在整数环中模$2^{32}$, 有多少个元素$P$满足$$N \cdot P \equiv P \pmod{2^{32}}.$$约束条件$t \le 10^5$意味着我们必须在常数或对数时间内回答每个查询。 任何迭代所有方法$2^{32}$的可能值$P$完全不可行，因为每个测试用例需要大约 40 亿次检查。 

出现微妙的边缘情况时$N = 1$。 在这种情况下，每个$P$满足方程，因为乘以 1 没有任何作用，所以答案应该是$2^{32}$。 另一个重要案例是$N = 0$，其中方程变为$0 \equiv P \pmod{2^{32}}$，所以只有$P = 0$有效，给出了确切的一种解决方案。 

幼稚实现的主要风险是尝试模拟每个的溢出算法$P$。 即使在 C++ 或 Python 中进行优化，迭代整个域也是不可能的。 

## 方法

 暴力解释直接测试所有可能的$P$, 计算$N \cdot P \bmod 2^{32}$并检查是否相等$P$。 根据定义这是正确的，但需要$2^{32}$每个测试用例的操作，远远超出任何时间限制。 

关键的简化是停止将其视为机器整数的算术，而是将其视为模方程：$$N \cdot P \equiv P \pmod{2^{32}}.$$重新排列给出：$$(N - 1)\cdot P \equiv 0 \pmod{2^{32}}.$$现在问题变成了一个经典的线性同余：我们正在计算有多少个残基$P$使产品可除以$2^{32}$。 让$M = 2^{32}$和$a = N - 1$。 我们希望解决方案的数量能够：$$aP \equiv 0 \pmod{M}.$$标准数论结果告诉我们，解的数量模$M$到$aP \equiv 0$正是$\gcd(a, M)$。 直觉是只有$M / \gcd(a, M)$生存下来，并且恰好有$\gcd(a, M)$这样的残留物在整个范围内。 

自从$M = 2^{32}$，答案特别简单：就是二除法的最大幂$N - 1$，除了当$N = 1$，我们对待$N - 1 = 0$并得到所有$2^{32}$解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有 P |$O(2^{32})$每次测试 |$O(1)$| 太慢了|
 | 基于 GCD 的还原 |$O(\log M)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 我们将问题简化为计算每个测试用例的最大公约数。 

1. 阅读$N$对于测试用例并解释算术模$M = 2^{32}$。 这确保了所有溢出行为都按照要求精确建模。 
2. 计算$a = N - 1$在整数算术中。 我们在概念上将该值视为模$M$，但使用 Python 整数可以避免溢出问题。 
3. 计算$g = \gcd(a, M)$。 该值表示乘数之间的结构重叠$a$和模数。 
4. 输出$g$。 这直接统计有多少残基$P$满足$aP \equiv 0 \pmod{M}$。 

正确性取决于将同余解释为整除条件。 我们正在计算解决方案$M \mid aP$，gcd 准确地描述了$M$的主要权力结构是与$a$，它决定了如何自由$P$可能会有所不同。 

### 为什么它有效

 条件$aP \equiv 0 \pmod{M}$相当于$M \mid aP$。 写作$g = \gcd(a, M)$，我们分解$a = g a'$和$M = g M'$在哪里$\gcd(a', M') = 1$。 可分性条件变为$M' \mid a'P$。 自从$a'$是可逆模数$M'$，这迫使$M' \mid P$。 因此$P$必须是的倍数$M'$，并且正好有$g$这些值取模$M$。 这确保了计数准确且详尽。 

## Python 解决方案```python
import sys
import math

input = sys.stdin.readline

M = 1 << 32

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        N = int(input().strip())
        a = N - 1
        g = math.gcd(a, M)
        out.append(str(g))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现依赖于 Python 的任意精度整数，因此减去 1$N$永远不会面临溢出问题。 模数$2^{32}$精确地表示为 2 的幂。 gcd 计算非常高效，因为一个操作数是 2 的固定幂，因此在实践中，它减少为提取最低设置位$N-1$，虽然使用`math.gcd`保持溶液清洁和安全。 

一个常见的实现错误是尝试在循环内显式模拟 32 位溢出乘法。 这是不必要的，因为代数变换完全消除了乘法。 

## 工作示例

 考虑两个有代表性的案例。 

第一的，$N = 1$。 然后$a = 0$和$g = \gcd(0, 2^{32}) = 2^{32}$。 每一个$P$满足方程。 

| 步骤| 价值|
 | --- | --- |
 | 尼 | 1 |
 | a = N - 1 | 0 |
 | gcd(a, 2^32) | gcd(a, 2^32) | 2^32 | 2^32
 | 答案| 2^32 | 2^32

 这证实了恒等乘法产生所有可能的不动点。 

第二，$N = 5$。 然后$a = 4$，并且自从$2^{32}$是2的幂，gcd是2除4的最大幂，就是4。 

| 步骤| 价值|
 | --- | --- |
 | 尼 | 5 |
 | a = N - 1 | 4 |
 | gcd(a, 2^32) | gcd(a, 2^32) | 4 |
 | 答案| 4 |

 这表明仅$P$可除以$2^{30}$在模结构内进行适当缩放后幸存下来，恰好产生四个有效残基。 

这些例子强调了解决方案如何仅依赖于二进制结构$N - 1$，而不是其大小。

 ## 边缘情况

 当$N = 1$，表达式变为$1 \cdot P = P$为所有人$P$，所以算法计算$a = 0$并正确返回$\gcd(0, 2^{32}) = 2^{32}$。 这避免了代码中的任何特殊分支。 

什么时候$N = 0$，我们有效地计算$a = -1$。 在模算术中取模$2^{32}$，这表现得像$2^{32} - 1$，它与$2^{32}$，因此 gcd 变为 1。算法自然地处理这个问题，而不需要无符号转换。 

什么时候$N$是 2 的大幂，$N - 1$变成一系列设置位，gcd 提取最低设置位，确保答案始终是与模数结构一致的 2 的幂。
