 ## 解决方案
 ## 问题理解

 我们得到一个无向加权图。 根据该图，我们被要求构造一个最小生成树，然后将该树视为无根结构，其中距离是沿树边缘的最短路径距离。 最后的任务是确定 MST 内任意两个顶点之间的最大可能距离，这正是生成的树的直径。 

所以管道在概念上是两层。 首先，将图压缩成生成树，以最小的总边权连接所有顶点。 然后，在该树内，我们测量沿唯一路径的所有对距离并提取其中的最大值。 

输入大小约束（如预期解决方案复杂度 O(m log n) 所暗示的那样）表明该图最多可以有 10^5 个顶点和边。 这立即排除了任何在构建树后尝试计算所有节点对之间距离的解决方案，因为即使是单个全对遍历也将是 O(n^2)。 我们被迫采用具有对数开销的线性或近线性图算法，例如用于 MST 构造的 Kruskal 或 Prim，然后是直径的线性树遍历。 

如果尝试在原始图上而不是 MST 上计算直径，则会出现一种微妙的故障模式。 MST 不仅仅是任何生成树，它还专门更改了存在的边。 例如，考虑一个具有边 (1-2，权重 1)、(2-3，权重 1) 和 (1-3，权重 100) 的三角形。 如果我们错误地允许重边，则原始图直径为 100，但 MST 将其删除并产生长度为 2 的路径。正确答案完全取决于 MST 结构，而不是原始图。 

另一种边缘情况是多个边缘具有相同的权重。 在这种情况下，可能有多个有效的 MST，但它们仍然都会生成有效的树。 直径可能会因平局决胜而有所不同，但问题假设采用标准确定性 MST 结构。 在实践中，具有稳定排序的 Kruskal 或 Prim 将产生一个一致的 MST。 

最后，一个幼稚的错误是使用 BFS 或 DFS 重新计算 MST 中每个节点的距离。 这在逻辑上是正确的，但太慢了。 在具有 10^5 个节点的树上，进行 10^5 次遍历，每次成本为 O(n)，会导致 10^10 次操作。 

## 方法

 蛮力思想是从最直接的解释开始的。 我们首先构建 MST，然后对于每对节点，我们使用 DFS 或 BFS 计算沿树的距离，跟踪看到的最大值。 由于树距离是唯一的路径，因此这给出了正确的答案。 

这种方法的问题不在于正确性，而在于规模。 在 O(m log n) 中构建 MST 后，每个源节点仍需要 O(n) 工作，总共为 O(n^2)。 当n达到10^5时，这是完全不可行的。 

关键的观察是树的直径不需要全对计算。 树具有一种结构特性，只需两次遍历即可找到其最长路径。 如果我们从任何节点开始，我们到达的最远节点保证是直径的一个端点。 从该端点再次开始给出真实的直径长度。 

这将第二阶段从二次行为减少为对树的线性遍历。 结合 MST 构建，完整的解决方案变为 O(m log n)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（MST 上的所有对）| O(n^2) | O(n^2) | O(n) | 太慢了|
 | 最优（MST + 两个 BFS/DFS）| O(m log n) | O(n + m) | 已接受 |

 ## 算法演练

 我们将解决方案分为构建 MST 和计算树直径。

1. 按权重对所有边进行排序，并使用 Kruskal 算法按升序处理它们，维护不相交的集并集结构。 每次我们遇到连接两个不同组件的边时，我们都会将其包含在 MST 中。 这确保我们构建一个没有循环的最小总权生成树。 
2. 处理完所有边后，我们得到一棵恰好有 n − 1 条边的树。 我们将其表示为邻接列表，因为我们需要快速遍历来计算距离。 
3. 选择任意节点（通常为节点 1），并运行 BFS 或 DFS 来计算到 MST 中所有可到达节点的距离。 我们记录离起点最远的节点。 该节点是直径的候选端点，因为在任何树中，最远的第一个遍历都会落在直径端点上。 
4. 从这个最远的节点开始第二个 BFS 或 DFS。 再次计算到树中所有节点的距离并跟踪遇到的最大距离。 该最大值是 MST 的直径。 
5. 输出这个最大距离。 

需要第二次遍历的原因是第一次遍历仅保证直径的一个端点，而不是其全长。 第二遍将计算锚定在真正的端点，确保我们捕获最长的路径。 

### 为什么它有效

 在一棵树中，任何两个节点都通过一条简单路径连接。 如果我们取任意一个节点并找到距它最远的节点，那么该最远的节点必须位于至少一条直径路径上； 否则，可以通过延长距离的矛盾来构建更长的路径。 一旦我们到达直径的一个端点，距它最远的节点必定是相反的端点，因为任何偏差都会缩短路径。 这个不变量确保两次 BFS 遍历足以恢复全局最大距离。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.rank[a] < self.rank[b]:
            a, b = b, a
        self.parent[b] = a
        if self.rank[a] == self.rank[b]:
            self.rank[a] += 1
        return True

def bfs(start, adj):
    from collections import deque
    dist = [-1] * len(adj)
    q = deque([start])
    dist[start] = 0
    far = start

    while q:
        u = q.popleft()
        for v, w in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + w
                q.append(v)
                if dist[v] > dist[far]:
                    far = v
    return far, dist[far]

def solve():
    n, m = map(int, input().split())
    edges = []
    for _ in range(m):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        edges.append((w, u, v))

    edges.sort()
    dsu = DSU(n)

    adj = [[] for _ in range(n)]

    for w, u, v in edges:
        if dsu.union(u, v):
            adj[u].append((v, w))
            adj[v].append((u, w))

    start = 0
    far, _ = bfs(start, adj)
    _, diameter = bfs(far, adj)

    print(diameter)

if __name__ == "__main__":
    solve()
```DSU 类处理 Kruskal 算法期间的循环检测。 路径压缩可确保每次操作的平摊时间接近恒定，这是有效处理多达 10^5 条边所必需的。 

邻接列表仅针对在 MST 选择中幸存下来的边构建。 每条边都在两个方向上存储，因为生成的结构是无向树。 

BFS 函数同时计算距离并跟踪最远的节点。 使用双端队列可确保对树进行线性时间遍历。 关键的实现细节是仅当找到严格更大的距离时才更新最远的节点。 

求解函数首先构造 MST，然后运行两阶段 BFS 来计算直径。

 ## 边缘情况

 一种边缘情况是图已经是一棵树。 在这种情况下，Kruskal 简单地接受所有边，并且 MST 与输入相同。 该算法仍然有效，因为 BFS 直接应用于原始结构而没有修改。 两遍 BFS 仍然正确地找到直径。 

另一种情况是星形图，其中所有边权重都相等。 任何 MST 都是有效的，Kruskal 可以按任意顺序选择边，但生成的树仍然是星形树。 直径始终为 2，并且无论先选取哪条边，BFS 都会正确识别两片叶子。 

最后一种情况是 MST 构建期间断开的中间状态，其中 DSU 阻止循环。 即使多个边连接已经连接的组件，它们也会被安全地跳过。 最终的邻接表始终保持为一棵树，从而保持直径计算的正确性。
