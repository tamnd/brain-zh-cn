 ## 解决方案
 ## 问题理解

 根据语句的解释，我们会得到一个有向或无向加权图，其中每条边不会对路径成本产生加法贡献，而是乘法贡献。 路径的总成本是沿其所有边权重的乘积。 任务是计算从固定源节点到所有其他节点的最小可能产品路径成本，并以对数形式返回答案。 

我们不返回原始产品，而是使用值的对数。 这会将沿路径的乘法转换为加法，因为 log(ab) 变为 log a 加 log b。 经过这种转换，问题就变成了具有非负权重的加权图上的标准单源最短路径问题。 

输入结构是通常的图形描述：一组节点和加权边，后面是查询或单个源要求，具体取决于版本。 每条边都贡献一个正的乘法因子，因此取对数后，每条边的权重都变成非负的加性成本。 

这些约束意味着我们必须以典型的 Codeforces 方式处理最多大约 10^5 条边。 这立即排除了任何二次甚至 O(nm) 松弛策略。 我们需要接近 O(m log n) 的东西，这是具有二叉堆的 Dijkstra 的特征。 由于所有变换后的权重都是非负的（正数的对数），因此 Dijkstra 是有效的。 

在这种情况下，一些边缘情况很重要。 首先，如果我们尝试直接计算乘积，具有较大乘法增长的路径可能会溢出浮点。 例如，如果边是 10^9 并且我们采用长度为 10^5 的路径，则产品无法存储，而日志保持稳定和可加性。 其次，断开连接的节点必须正确报告不可达状态，这在日志空间中对应于无穷大。 第三，小于1的非常小的边权重会产生负对数，因此即使Dijkstra仍然有效，我们也不能假设变换后的权重为正，因为非负条件实际上变成“所有边对数都是非负的”，这要求原始权重至少为1或者问题保证对数域一致性。 在这个问题中，意图是日志空间中的标准最短路径，因此假设权重是兼容的。 

一种简单的方法是枚举所有路径、计算产品并取最小值。 这会立即失败，因为路径数量呈指数增长。 在密集图中，即使是路径长度上的动态规划也需要 O(n^2) 或更糟。 

关键的观察结果是，沿路径的乘法在结构上与对数变换后的加法相同，因此问题简化为经典的最短路径计算。 

## 方法

 蛮力的想法是从源头探索每条可能的路径并计算其乘积。 这原则上是正确的，因为它评估所有候选人。 然而，图中的路径数量随着分支呈指数增长，因此即使对于适度的 n 和 m，这也变得不可行。 在分支因子为 2、深度为 20 的图中，我们已经拥有超过一百万条路径，而真正的约束远不止于此。 

我们需要一种方法来避免重新计算重叠的子问题。 重要的结构是，通过一条边扩展路径仅取决于中间节点的已知成本，而不取决于我们如何到达它。 一旦我们取对数，每条边都会贡献附加成本，因此我们正在寻找具有非负权重的图中的最短路径。 

这立刻让人想到了 Dijkstra 算法。 优先级队列总是扩展当前已知的最便宜的节点，保证一旦节点被弹出，其最短距离就被确定。 这是有效的，因为所有边权重在对数空间中都是非负的，从而保留了贪婪的正确性条件。

因此，我们不是探索所有路径，而是维护一个最著名的距离数组并使用最小堆放松边缘。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（所有路径）| 指数| O(n) 递归堆栈 | 太慢了|
 | Dijkstra 论对数权重 | O(m log n) | O(n + m) | 已接受 |

 ## 算法演练

 1. 读取图形并将每条边权重 w 转换为 log(w)。 这将沿路径的乘法转换为加法。 立即应用变换，以便所有进一步的计算都在加性空间中进行。 
2. 为除源之外的所有节点初始化一个无穷大的距离数组，将其设置为 0。这表示 log(1)，因为空路径具有乘法恒等式 1。 
3. 将源节点推入距离为0的优先级队列。优先级队列始终存储下一个最有希望扩展的节点。 
4. 反复从优先级队列中提取暂定距离最小的节点。 该节点代表当前已知的来自源的最短日志成本。 
5. 对于提取节点的每个邻居，通过将边对数权重添加到当前节点的距离来计算候选距离。 如果该候选者小于先前记录的距离，则更新它并将邻居推入优先级队列。 
6. 继续，直到优先级队列为空。 此时，每个节点都已确定其最小日志路径成本。 
7. 输出距离数组，通常根据问题要求用“inf”等标记替换无法到达的值。 

### 为什么它有效

 正确性来自标准 Dijkstra 不变量：当从优先级队列中提取节点时，其当前距离是所有未探索路径中可能的最小距离。 由于对数空间中的所有边权重都是非负的，因此到该节点的任何替代路径都必须经过具有等于或更大暂定距离的节点，并且添加非负边不能减少它。 这保证了一旦最终确定，距离以后就无法改进，因此算法收敛到真正的最短对数距离。 

## Python 解决方案```python
import sys
import math
import heapq

input = sys.stdin.readline

def solve():
    n, m, s = map(int, input().split())
    g = [[] for _ in range(n + 1)]

    for _ in range(m):
        u, v, w = map(int, input().split())
        cost = math.log(w)
        g[u].append((v, cost))
        g[v].append((u, cost))

    INF = float('inf')
    dist = [INF] * (n + 1)
    dist[s] = 0.0

    pq = [(0.0, s)]

    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue

        for v, w in g[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    print(*dist[1:])

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个邻接列表，其中每个边权重都由其对数替换。 这确保了路径成本是累加的。 

距离数组存储最著名的对数成本。 优先级队列强制贪婪选择最小的暂定距离，并进行陈旧状态检查`if d != dist[u]`防止过时的堆条目导致冗余工作。 

每个松弛步骤直接对应于原始问题中乘以边缘权重，但在对数空间中作为加法执行。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(m log n) | 每个边缘松弛可能会推入一个堆，并且堆操作的成本为 log n |
 | 空间| O(n + m) | 邻接表加上距离和堆存储|

 复杂性与标准 Dijkstra 相匹配，并且足以在典型限制内绘制最多 10^5 条边的图。

import sys, io
import math

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import heapq

    n, m, s = map(int, sys.stdin.readline().split())
    g = [[] for _ in range(n + 1)]

    for _ in range(m):
        u, v, w = map(int, sys.stdin.readline().split())
        g[u].append((v, math.log(w)))
        g[v].append((u, math.log(w)))

    INF = float('inf')
    dist = [INF] * (n + 1)
    dist[s] = 0.0
    pq = [(0.0, s)]

    while pq:

        if d != dist[u]:
            continue
        for v, w in g[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    return " ".join(f"{x:.6f}" if x < INF else "inf" for x in dist[1:])

 ## 边缘情况

 断开连接的节点情况，例如`3 1 1 / 1 2 2`离开节点 3，距离无穷大。 该算法永远不会放松它，因为没有堆条目到达它，因此它的距离保持不变，正确反映了到达它的不可能。 

单节点图初始化距离 0 并且从不进入松弛循环。 优先级队列弹出一次并立即终止，确认最小输入大小的正确性。 

多个路径竞争的图（例如直接重边与较小边的间接乘积）说明了为什么需要 Dijkstra。 该算法确保通过松弛来发现和比较间接路径，并保留最小对数和。
