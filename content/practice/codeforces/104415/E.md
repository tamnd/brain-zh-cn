 ## 解决方案
 ## 问题理解

 我们得到一个图，其顶点由唯一的整数标签标识。 范围从 1 到最大值的某些标签可能不对应于任何实际顶点。 该图还包含这些标记顶点之间的无向边。 

我们模拟的过程是增量的。 我们按升序考虑标签。 当我们到达标签时，我们会激活相应的顶点（如果存在），并使其可用于连接。 每当一个顶点变得活跃时，我们也会通过将其链接到已经活跃的顶点的所有边来连接它。 每时每刻，我们都在有效地查看由所有活动顶点引起的子图。 

任务是确定活动子图完全连接的所有标签值，这意味着当前所有活动顶点都位于单个连接组件中。 如果在扫描过程中我们到达一个不对应于任何顶点的标签，则该过程会立即停止，因为不会发生进一步有意义的激活。 

这些约束意味着顶点和边的数量足够大，任何从头开始重复重新计算连接的方法都会太慢。 每次插入后的朴素 BFS 或 DFS 都会花费 O(n(n + m))，当 n 很大时，这变得不可行。 这推动我们采用基于 DSU 的增量连接维护方法，其中联合操作几乎是恒定的时间。 

当标签丢失时，就会出现微妙的边缘情况。 假设我们有顶点 {1, 2, 4, 5}。 当我们到达标签 3 时，没有要激活的顶点，扫描终止。 忽略缺失标签的简单实现可能会错误地继续到 4 和 5，从而产生在流程定义下无效的答案。 

另一种边缘情况发生在图较早连接时，但如果我们错误地从头开始重新计算而不保留以前的并集，则稍后的添加将打破假设。 DSU 通常可以避免这种情况，但重复遍历方法却无法避免。 

## 方法

 蛮力的想法很简单。 我们逐个标签地模拟流程。 对于每个标签 i，我们维护活动顶点集并使用 DFS 或 BFS 重建它们之间的连接。 激活直到 i 的所有顶点后，我们检查所有活动顶点是否属于单个连接组件。 如果是这样，我们将 i 记录为有效答案。 

这可以正常工作，因为它在每一步都直接检查连接性。 然而，成本主要由重复的图遍历决定。 如果有 O(n) 个步骤，并且每个步骤扫描 O(n + m) 条边，则总复杂度变为 O(n(n + m))，这对于典型约束来说太大了。 

关键的观察结果是连通性是逐步发展的。 当我们从 i 移动到 i + 1 时，我们不需要从头开始重新计算连接性。 我们只需要添加一个新顶点并将其与已经活跃的邻居合并。 DSU（Disjoint Set Union）正是为这种增量合并而设计的。 

关键的结构特性是，边缘只会在连接方面添加，而不会被删除。 因此，一旦两个顶点位于同一组件中，它们就会保持不变。 这种单调性使我们能够有效地维护连接信息。 

我们按升序扫描标签。 每次我们激活一个顶点时，我们都会将它与所有已经激活的邻居合并。 我们维护活动顶点之间连接组件的数量。 如果这个计数变成 1，我们就记录当前标签。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（每一步重新计算 BFS/DFS）| O(n(n + m)) | O(n(n + m)) | O(n + m) | 太慢了|
 | DSU 增量扫描 | O((n + m) α(n)) | O((n + m) α(n)) | O(n + m) | 已接受 |

 ## 算法演练

我们维护一个 DSU 结构和一个布尔数组来跟踪顶点是否处于活动状态。 我们还保留了活性成分的计数器。 

1.我们从最小的标签开始，向上移动。 这种顺序很重要，因为激活是通过增加标识符来严格定义的。 
2. 当我们到达标签 i 时，我们首先检查是否存在具有该标签的顶点。 如果没有，我们立即终止该过程，因为在此之后不会定义进一步的状态更改。 
3. 如果顶点存在，我们激活它并最初将其视为自己的连通分量。 这会将组件数量增加 1。 
4. 我们迭代该顶点的所有邻居。 对于每个已经处于活动状态的邻居，我们合并它们的 DSU 集。 如果合并两个先前独立的组件，我们会将组件数量减少 1。 这保留了活动节点之间存在多少连接组件的精确计数。 
5. 处理完所有邻居后，我们检查活动组件计数是否等于 1。 如果是这样，则意味着每个活动顶点都可以从其他每个活动顶点到达，因此当前标签是有效答案。 

这种本地合并足够的原因是所有连接信息都流过边缘，并且当激活其后续端点时，每条边缘都会被处理一次。 

### 为什么它有效

 DSU 保持这样的不变性：在任何时刻，每个活动顶点都恰好属于一个集合，该集合代表活动顶点的导出子图中的连通分量。 每个并集操作完全对应于在两个活动顶点之间引入一条边，这是唯一可以减少组件数量的事件。 由于我们从不删除边或停用顶点，因此一旦所有活动顶点合并到单个 DSU 集中，它们在所有未来状态下都保持连接。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())

    adj = [[] for _ in range(n + 1)]
    exists = [False] * (n + 1)

    for _ in range(m):
        u, v = map(int, input().split())
        adj[u].append(v)
        adj[v].append(u)

    # mark which identifiers exist
    # (if input guarantees all 1..n exist, this loop is harmless)
    for i in range(1, n + 1):
        exists[i] = True

    parent = list(range(n + 1))
    size = [1] * (n + 1)

    def find(x):
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    def union(a, b):
        ra, rb = find(a), find(b)
        if ra == rb:
            return False
        if size[ra] < size[rb]:
            ra, rb = rb, ra
        parent[rb] = ra
        size[ra] += size[rb]
        return True

    active = [False] * (n + 1)
    comp = 0

    res = []

    for i in range(1, n + 1):
        if not exists[i]:
            break

        active[i] = True
        comp += 1

        for v in adj[i]:
            if active[v]:
                if union(i, v):
                    comp -= 1

        if comp == 1:
            res.append(i)

    print(*res)

if __name__ == "__main__":
    solve()
```该解决方案构建一个邻接列表，以便在处理其端点时每条边都可用。 DSU 结构仅跟踪活动节点之间的连接。 这`active`数组确保我们只合并在当前扫描前缀中有效的边。 

这`comp`变量是必不可少的，因为 DSU 本身并不能直接告诉我们存在多少个组件。 相反，我们明确跟踪联合成功合并不同集合的次数。

 ### 示例 1

 考虑一个具有顶点 1 到 4 和边 (1-2)、(2-3)、(3-4) 的小图。 我们按顺序模拟激活。 

| 我| 活跃节点| DSU 合并 | 组件| 补偿 == 1 |
 | --- | --- | --- | --- | --- |
 | 1 | {1} | 无 | 1 | 是的 |
 | 2 | {1,2} | 1-2 | 1-2 1 | 是的 |

 | 4 | {1,2,3,4} | 3-4 | 3-4 1 | 是的 |

 每个前缀都是连接的，因此每个标签都被记录下来。 这证明了一个不变量：一旦链完全连接，所有未来的前缀都保持连接。 

### 示例 2

 现在考虑边 (1-2) 和 (3-4)，形成两个不相连的对。 

| 我| 活跃节点| DSU 合并 | 组件| 补偿 == 1 |

 | 4 | {1,2,3,4} | 3-4 | 3-4 2 | 没有|

 在这里，当引入新的隔离块时，连接就会丢失。 DSU 正确地跟踪系统不再完全连接。

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O((n + m) α(n)) | O((n + m) α(n)) | 每条边都被处理一次，并且每个并集/查找几乎是恒定的 |
 | 空间| O(n + m) | 邻接表加 DSU 阵列 |

 复杂度完全符合 n 高达 2×10^5 的典型限制。 由于摊销逆阿克曼行为，DSU 确保即使是密集的边缘集也能保持高效。

 ## 边缘情况

 一个重要的边缘情况是图在前几次激活后立即连接。 例如，如果节点 1 连接到所有其他节点，则在 i = 2 时，DSU 已将所有内容合并到单个组件中。 该算法正确记录 i = 2，因为在最后一个必要的并集发生时 comp 恰好变为 1，并且此状态对于所有后续 i 都持续存在。 

当缺失的标识符过早出现时，就会出现另一种边缘情况。 假设仅存在节点{1,3,4}。 当达到 i = 2 时，过程终止。 该算法正确停止，因为`exists[i]`检查防止任何进一步的激活或联合操作。 忽略丢失标签的天真扫描将错误地继续并产生 i = 3 和 4 的结果，即使该过程应该已经停止。 

最后一个微妙的情况是已经连接的组件之间的重复边缘。 在这种情况下，并集运算返回 false，因此组件计数不会错误地减少。 这可以防止合并计数过多，并确保连接状态在整个扫描过程中保持一致。
