 ## 解决方案
 ## 问题理解

 我们有一系列课程的出勤点序列，必须严格从第一堂课开始参加，不能跳过或从中间开始。 每个班级都会贡献固定数量的分数，这些分数会随着我们在序列中的移动而累积。 

除了这个数组之外，我们还得到了多个查询。 实际上，每个查询都会询问卡约从学期开始起必须走多远才能达到所需的累积出勤分数水平。 由于出勤从一开始就是连续的，因此每个有效答案都对应于数组的某个前缀。 

因此，对于每个查询值，我们需要确定数组中总和达到或超过该值的最小前缀，并报告有关该前缀的信息，例如其位置和累计到该前缀的总点数。 

关键的结构性限制是上课总是从头等舱开始。 这消除了对任意子数组处理的任何需要，并将问题完全简化为有关前缀和的推理。 

从复杂性的角度来看，数组可能足够大，以至于独立地重新计算每个查询的总和会太慢。 如果有多达 100000 个类和类似数量的查询，则每个查询的简单 O(n) 扫描在最坏的情况下将导致大约 10^10 次操作，这远远超出了典型的限制。 这立即表明我们必须预处理数组并在对数或常数时间内回答每个查询。 

当查询要求的值大于所有类的总和时，就会出现微妙的边缘情况。 在这种情况下，没有达到所需阈值的前缀，因此必须仔细实施来决定返回什么。 类似地，当阈值为零或负数时，正确答案立即是具有总和零前缀行为的第一个位置，通常是索引 1 和总和 a[0] 或索引 0，具体取决于索引约定。 如果不明确处理，这些情况通常会打破朴素的二分搜索边界。 

## 方法

 直接方法通过从第一类迭代并累积点直到达到所需的阈值来独立处理每个查询。 这很简单：对于给定的查询值，我们不断添加类别点，直到运行总数足够大。 正确性是立即的，因为它模拟了问题中描述的确切过程。 

然而，这种方法对每个查询重复相同的前缀求和工作。 在最坏的情况下，每个查询可能需要扫描几乎整个数组，总复杂度为 O(n·q)。 如果约束很大，这会变得太慢，因为它会有效地重复计算相同的部分和。 

关键的观察是所有查询都在相同的前缀结构上运行。 一旦我们计算了一次前缀和，任何前缀的和就变成了直接数组查找。 每个查询的剩余任务是找到前缀和跨越阈值的第一个位置。 因为前缀和数组是非递减的，所以这成为一个经典的二分搜索问题。 

因此解决方案减少为两个阶段。 首先，将数组预处理为前缀和数组。 其次，通过二分查找第一个索引来回答每个查询，其中前缀和至少是查询值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n·q) | O(n·q) | O(1) | O(1) | 太慢了|
 | 前缀和+二分查找 | O(n + q log n) | O(n + q log n) | O(n) | 已接受 |

 ## 算法演练

 我们分两个阶段构建解决方案：预处理和查询处理。

1. 计算一个前缀和数组，其中每个位置存储从第一类到该位置累积的总分。 这将范围累积转换为恒定时间查找。 
2. 对于每个查询值x，我们需要定位总和至少为x的最早的前缀。 由于前缀和是单调递增的，因此我们可以在索引范围内应用二分搜索。 
3. 二分查找时，我们反复检查中点。 如果该位置的前缀和小于 x，则答案必定位于右侧。 否则，可能是这个位置或更早的位置，所以我们向左移动。 
4. 二分查找完成后，得到的索引就是满足要求的最小前缀。 然后我们输出索引（如果需要，转换为基于 1 的索引）和相应的前缀和。 
5. 如果查询值超过整个数组的总和，我们直接返回数组的全长及其总和，因为没有更早的前缀可以满足它。 

它工作的原因是基于单调性属性：当我们在数组中前进时，前缀和永​​远不会减少。 这保证了条件“前缀总和> = x”定义了有效索引的连续后缀。 一旦索引有效，其右侧的所有索引也有效，而其左侧的所有索引均无效。 二分搜索之所以有效，正是因为它利用了这个单一的转换边界。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def lower_bound_prefix(prefix, x):
    lo, hi = 0, len(prefix) - 1
    ans = len(prefix) - 1
    
    while lo <= hi:
        mid = (lo + hi) // 2
        if prefix[mid] >= x:
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1
    
    return ans

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    
    prefix = [0] * n
    prefix[0] = a[0]
    
    for i in range(1, n):
        prefix[i] = prefix[i - 1] + a[i]
    
    total = prefix[-1]
    
    out = []
    for _ in range(q):
        x = int(input())
        
        if x <= prefix[0]:
            idx = 0
        elif x > total:
            idx = n - 1
        else:
            idx = lower_bound_prefix(prefix, x)
        
        out.append(f"{idx + 1} {prefix[idx]}")
    
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先构建前缀总和数组，以便每个条目代表该班级的总出勤率。 这避免了查询期间的重新计算。 

辅助函数对达到所需阈值的第一个前缀执行二分搜索。 返回的索引是从零开始的，我们在打印时调整它。 

使用此搜索独立处理每个查询，并且我们仔细保护查询小于第一个前缀或大于总和的边界情况。

 ## 边缘情况

 当数组只有一个元素时，前缀计算和二分搜索都会崩溃为微不足道的行为。 该算法立即返回索引 0，因为任何正查询都必须映射到该单个前缀。 

当查询恰好等于前缀和之一时，二分搜索仍必须返回最早的此类索引。 即使遇到相等，该实现也会通过向左移动右边界来确保这一点，从而防止选择稍后的有效位置。 

当查询超过总和时，算法将结果限制到最后一个索引。 这可以避免二分搜索超出有效范围，并保证即使没有前缀满足条件也能得到定义的输出。
