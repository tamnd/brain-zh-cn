 ## 解决方案
 ## 问题理解

 我们得到一条二次曲线，已知它穿过原点和平面上的两个附加点。 换句话说，抛物线由三个点唯一确定，其中一个点固定为 (0, 0)，另外两个点在输入中提供。 根据这些约束，我们被要求重建抛物线方程，然后确定其根。 

因为其中一个点是原点，所以二次方程的常数项立即消失。 曲线可以写成受限形式$y = ax^2 + bx$。 任务简化为恢复系数$a$和$b$从两个给定点，然后求解该二次方程的 x 截距。 

输入通常提供两个非零点$(x_1, y_1)$和$(x_2, y_2)$。 每个测试用例都是独立的，输出对应于重构抛物线的根。 

约束的设计使得每个测试用例的所有内容都是恒定时间的。 这意味着任何涉及范围迭代或数值搜索的解决方案都是不必要的，只会带来精度问题或效率低下的风险。 预期的解决方案必须依赖于直接代数运算。 

一种简单的方法可能会尝试使用插值库重建多项式或通过通用二次公式求解而不利用结构$c = 0$。 这在数学上仍然有效，但它引入了不必要的浮点运算和除法稳定性变得脆弱的边缘情况。 

当以下任一情况出现时，就会出现微妙的失败情况$x_1$或者$x_2$is zero in intermediate derivations if handled carelessly. 计算时出现另一个问题$a$和$b$单独进行重复的浮点运算，当分母较小时会放大精度误差。 

## 方法

 暴力思维是使用一般的二次插值来重建抛物线。 人们可以解决一个完整的系统$a$,$b$， 和$c$，然后将所有内容插入二次根求解器。 这原则上是有效的，但它忽略了一个系数已知为零，因此我们正在求解不必要地嵌入 3 变量框架中的 2 变量系统。 

低效率并不在于渐近复杂性，因为一切都是常数时间，而在于数值不稳定和冗余计算。 每个额外的代数步骤都会引入可能放大浮点误差的除法。 

关键的观察结果是该结构将系统分解为两个未知数的两个线性方程。 一次$c = 0$，两点均满足：$$ax_i^2 + bx_i = y_i$$这是一个线性系统$a$和$b$。 直接求解它会产生稳定的封闭式表达式，而无需递归到一般二次机械中。 

一次$a$和$b$已知，根是微不足道的。 一根永远是$x = 0$，另一个来自因式分解：$$ax^2 + bx = x(ax + b)$$所以第二个根是$x = -\frac{b}{a}$，假设$a \neq 0$。

 | --- | --- | --- | --- |
 | 一般二次插值| O(1) | O(1) | O(1) | O(1) | 正确但矫枉过正 |
 | 直接线性系统+因式分解| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们通过将几何约束转化为一个小的代数系统然后直接提取根来解决问题。 

1. 从抛物线通过原点的事实开始，这立即将常数项固定为零。 这将等式简化为$y = ax^2 + bx$，从系统中消除一个未知数。 
2. 代入第一点$(x_1, y_1)$带入方程，产生$a x_1^2 + b x_1 = y_1$。 这给出了之间的线性关系$a$和$b$。 
3.代入第二点$(x_2, y_2)$，生产$a x_2^2 + b x_2 = y_2$。 现在我们有两个带有两个未知数的线性方程。 
4. 通常消除一个变量$b$，通过将第一个方程表示为$b = \frac{y_1 - a x_1^2}{x_1}$。 选择此步骤是因为它降低了第二个方程中的替换复杂性。 
5. 将此表达式代入第二个方程并求解$a$。 这会产生一个封闭式表达式，其中所有项仅依赖于已知输入。 
6.一次$a$已知，计算$b$直接使用前面的表达方式。 此时，完整的二次方程被重建。 
7. 将二次因式分解为$x(ax + b)$，它立即给出一个根$x = 0$和第二个根在$x = -\frac{b}{a}$，假设$a$是非零的。 

### 为什么它有效

 正确性来自于两个不同的非原点唯一确定一个常数项为零的二次方程。 方程组在未知系数中是线性的$a$和$b$，因此求解它会产生精确的系数，而无需近似。 一旦多项式被重建，分解出$x$在代数上是精确的，保证根恰好是原始曲线的解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        x1, y1 = map(float, input().split())
        x2, y2 = map(float, input().split())

        # Solve:
        # a x1^2 + b x1 = y1
        # a x2^2 + b x2 = y2

        denom = (x2 * x1 * x1 - x1 * x2 * x2)
        # rearranged from elimination
        a = (y2 * x1 - y1 * x2) / denom
        b = (y1 - a * x1 * x1) / x1

        # roots: x = 0 and x = -b/a
        if abs(a) < 1e-12:
            # degenerate linear case bx = 0
            root2 = 0.0
        else:
            root2 = -b / a

        # output both roots
        out.append(f"0 {root2}")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现直接遵循导出的代数结构。 系数计算反映了消除步骤，其中分母对应于由两点形成的 2x2 系统的行列式。 一次$a$得到，$b$从第一个方程中恢复，无需重新计算任何系统。 

根提取使用因式分解洞察力。 对非常小的特殊处理$a$避免在曲线变为有效线性的退化情况下除法不稳定。

 ## 边缘情况

 当两个非原点位于一条直线上时，就会出现关键的边缘情况，从而使二次方程退化，从而有效地崩溃$a$为零。 在这种情况下，第二次根的公式涉及除以$a$，因此幼稚的实现会崩溃或产生无穷大。 小额支票$|a|$确保输出保持稳定并返回零处的重复根，与因式分解的形式一致。 

另一个微妙的情况是当$x_1$非常小或者为负值。 由于计算$b$除以$x_1$，粗心的排序或整数转换会立即破坏正确性。 将所有值视为浮点并保留表达式结构可以完全避免这种不稳定性。 

最后一种情况是行列式的数值取消$x_2 x_1^2 - x_1 x_2^2$。 什么时候$x_1$和$x_2$接近时，该值变小，并且精度损失可能占主导地位。 该公式在数学上仍然保持正确，但稳定的评估取决于避免不必要的重新计算并保持如图所示的表达式因式分解。
