 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，我们从左到右处理它。 在扫描时，我们维护每个值已经出现的次数。 在每个位置，在将当前值插入到我们的运行频率结构中之前，我们要计算有多少较早的元素与其形成特定的乘法关系。 具体来说，对于当前元素$a_i$，我们寻找之前的元素$a_j$使得$a_j$划分$a_i$或者$a_i$划分$a_j$，取决于如何通过比率解释条件$x / a_i$声明中描述。 预期的计算简化为计数对，其中一个值是固定目标除以当前值的商，并且该商必须已经出现。 

关键的隐藏结构是值很小，因此我们可以维护一个按值索引的直接频率数组，而不是将输入视为需要散列或映射的任意整数。 这将每个关于“存在多少次等于某个计算商的值的先前出现”的查询转变为恒定时间查找。 

约束条件意味着$n$可以足够大，以至于$O(n^2)$扫描先前的元素是不可能的。 甚至$O(n \log n)$如果涉及重复的乘法或映射操作，每个元素会太慢。 唯一可接受的方法是接近线性时间，其中每个元素贡献恒定量的工作。 

当除法不精确时，就会出现微妙的边缘情况。 如果我们尝试在不检查整除性的情况下计算商，我们可能会错误地将小数结果计为有效索引。 当当前元素为零时会出现另一个问题，因为除法逻辑变得未定义或需要单独处理。 盲目计算的幼稚实现$x / a_i$未经验证将崩溃或计数错误的对。 

例如，如果数组是$[2, 3, 4]$我们正在检查乘积等于 6 的对，然后在$4$, 计算$6 / 4$给出非整数结果，不得使用该结果。 正确答案应该只考虑整数商。 

另一个边缘情况是重复值。 如果数组包含许多相同的元素，则简单的成对检查可能会重复计数或错过排序约束，而当我们从左到右进行时，基于频率的方法自然会累积正确的计数。 

## 方法

 蛮力的想法很简单。 对于每个位置$i$，我们回顾之前的每一个位置$j < i$并测试涉及乘法或除法的条件是否成立。 这保证了正确性，因为每对都被显式检查一次。 然而，这导致$O(n^2)$最坏情况下的比较，当$n$很大。 

低效率来自于重复扫描所有先前的元素，即使其中大多数与当前值无关。 条件的结构揭示了我们不需要先前元素的个体身份，只需要每个值出现了多少次。 一旦我们认识到每次检查都简化为“较早元素中是否存在特定值”，我们就可以用频率查找替换内部循环。 

这一观察将问题转化为维护计数数组。 在迭代时，我们计算每个元素所需的补值，并直接查询它之前出现过多少次。 这将每个步骤减少到恒定时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 频率阵列|$O(n)$|$O(V)$| 已接受 |

 ## 算法演练

 我们维护一个频率数组`cnt`在哪里`cnt[v]`存储多少倍的值`v`到目前为止已经出现在前缀中。 

1. 初始化`cnt`带零并设置`answer = 0`。 这准备了一个前缀计数结构，以便每个查询仅依赖于较早的元素。 
2. 从左到右迭代数组。 在每个索引处$i$,读取当前值$a_i$。 我们将此值视为“新到达”元素，其对未来对的贡献必须仅基于较早的值来计算。 
3.更新前的频率$a_i$，计算从问题的比率条件导出的所需互补值。 这通常是$x / a_i$， 在哪里$x$是固定目标值或导出参数。 关键要求是该商必须是整数，否则它无法对应于有效的前一个元素。 
4.如果$a_i$划分$x$确切地说，添加`cnt[x // a_i]`到答案。 这会计算有多少个先前元素可以与当前元素配对以满足乘法条件。 除法检查可防止无效的分数索引。 
5. 处理完早期元素的贡献后，增量`cnt[a_i]`记录该值现在可用于未来的位置。 

### 为什么它有效

 在每个索引处，`cnt`准确地表示到目前为止在前缀中看到的值的多重集。 涉及当前元素和前一个元素的任何有效对必须仅使用此前缀信息。 因为我们只查询`cnt`在插入当前元素之前，我们确保在处理右端点时，每个有效对都被精确计数一次。 可除性检查保证只考虑结构上有效的补集，从而保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    a = list(map(int, input().split()))

    # assuming values are small enough for direct indexing
    MAXV = max(max(a), x) + 1
    cnt = [0] * (MAXV + 1)

    ans = 0

    for v in a:
        if v != 0 and x % v == 0:
            ans += cnt[x // v]
        cnt[v] += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先构建一个直接频率数组，这避免了哈希结构的开销。 迭代顺序至关重要，因为我们只想计算第二个元素稍后出现在数组中的对。 

条件`x % v == 0`确保我们永远不会使用非整数商对频率数组进行索引，否则会导致不正确的内存访问或逻辑错误。 处理`v == 0`单独避免被零除的问题。 

更新内容`cnt[v] += 1`在计算贡献之后发生，以便当前元素不与自身配对。

 ## 边缘情况

 对于重复值，例如$[2,2,2,2]$和$x = 4$，每个新元素与所有先前出现的元素配对。 频率数组随着我们扫描而增长：在前 2 个之后，`cnt[2]=1`; 第二个之后就变成2，以此类推。 每个步骤都会添加当前有效补集的计数，从而生成与有效对的数量相匹配的三角形累加。 

对于不可分割的情况，例如$[1, 3, 5]$和$x = 4$，每个元素都未通过整除性检查，因此算法从不执行查找。 频率数组仍然正确更新，但没有添加任何贡献，根据需要产生零。 

对于最小的输入$[a_1]$，循环执行一次，不执行任何查找，因为没有先前状态，并且只是将值插入到频率数组中。 答案仍然为零，与配对的缺失相匹配。
