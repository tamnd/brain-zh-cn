 ## 问题理解

 该任务描述了一个简单的物理情况，其中一个飞盘从原点抛出并落在一个点$(x, y)$。 一个名叫阿福拉（Asfora）的玩家只有在球落地前跑得足够远才能接住它。 几何形状很简单：到达飞盘所需的距离是从原点到飞盘的欧几里得距离$(x, y)$，而可用的运行能力是通过类似速度的参数来表示的$v$和一个时间$t$。 

对于每个测试用例，我们都会给出定义着陆位置和移动能力的值。 该决策是二元的：确定 Asfora 是否能够及时到达飞盘。 

关键的观察是，一切都归结为将几何距离与可达半径进行比较。 欧几里得距离是$\sqrt{x^2 + y^2}$。 可到达距离与$v \cdot t$。 因此，比较的是该点是否位于以原点为圆心的圆内部或圆上。 

每个测试用例的输入大小都很小，并且每个查询都是独立的。 这立即表明任何超出每个测试用例恒定时间的解决方案都是不必要的。 即使有很多测试用例，每个用例的计算也只涉及很少的算术运算。 

一个微妙但重要的实现问题是浮点精度。 使用平方根的直接计算可能会引入舍入误差，特别是当值很大或接近边界时。 另一个实际问题是性能：即使算术本身很简单，在慢速 I/O 环境中重复解析浮点数也可能成为瓶颈。 

边缘情况来自边界相等和数值精度。 例如，当$x^2 + y^2$非常接近$v^2 t^2$，浮点比较可能会错误地翻转结果。 另一个边缘情况是当所有值都为零时，答案必然是肯定的。 

## 方法

 最直接的方法是显式计算欧几里德距离。 对于每个测试用例，我们评估$\sqrt{x^2 + y^2}$，然后将其与$v \cdot t$。 这在数学上是正确的，因为它直接符合距离的几何定义。 然而，它带来了两个低效率的问题。 首先，计算平方根比乘法和加法慢。 其次，浮点运算可能会引入精度误差，尤其是在处理大整数或严格不等式时。 

更好的方法通过完全消除平方根来避免这两个问题。 而不是比较$\sqrt{x^2 + y^2} \le v \cdot t$，我们对两边进行平方，产生$x^2 + y^2 \le v^2 t^2$。 这种转换保留了正确性，因为涉及的所有数量都是非负的。 如果输入是整数，则比较将变成纯粹基于整数，或者至少避免中间浮点不稳定。 

蛮力之所以有效，是因为它直接镜像几何体，但是当扩展到许多测试用例时，它变得不太可靠并且稍微慢一些。 平方公式消除了超越运算，并将整个问题简化为一些乘法和加法。

 | --- | --- | --- | --- |
 | 与 sqrt 的直接距离 | 每次测试 O(1) | O(1) | O(1) | 正确但脆弱|
 | 平方比较 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取测试用例的数量。 每个测试用例都是独立的，因此它们之间不传递任何状态。 
2. 对于每个测试用例，读取值$x$,$y$,$v$， 和$t$。 这些定义了着陆位置和可用的行驶能力。 
3. 计算距原点的平方距离：$x^2 + y^2$。 这代表了不引入平方根的精确几何要求。 
4. 计算可达距离的平方为$v^2 \cdot t^2$。 这将可用的移动半径扩展到与距离相同的平方空间。 
5. 比较两个值。 如果$x^2 + y^2 \le v^2 t^2$，输出飞盘可达； 否则，输出它不是。 
6. 对所有测试用例重复此操作，立即打印每个结果。 

### 为什么它有效

 该算法依赖于这样一个事实：平方是对非负数的严格递增变换。 欧几里德距离和可达距离始终为非负，因此对两侧应用相同的单调变换可以保持排序。 这确保了平方形式的不等式相当于原始几何条件，消除了数值不稳定而不改变正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = input().strip().split()
    if not data:
        return
    t = int(data[0])
    idx = 1
    out = []

    for _ in range(t):
        x = int(data[idx]); y = int(data[idx + 1])
        v = int(data[idx + 2]); t_ = int(data[idx + 3])
        idx += 4

        lhs = x * x + y * y
        rhs = (v * t_) * (v * t_)

        if lhs <= rhs:
            out.append("YES")
        else:
            out.append("NO")

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现将所有内容保留在整数算术中。 产品$v \cdot t$首先计算，然后平方，这在 Python 中是安全的，因为它支持大整数。 在 C++ 设置中，需要小心避免溢出，但 Python 可以自然地处理任意精度。 

将输入读取为平面列表可以避免重复的函数调用并最大限度地减少开销。 这在大量测试用例被打包到单个输入流的环境中很重要。

 ## 边缘情况

 当点正好位于边界圆上时，就会出现一种边缘情况。 用于输入$x=3, y=4, v=5, t=1$，计算得出$x^2 + y^2 = 25$和$(vt)^2 = 25$。 该算法可以正确比较相等性，因为它使用`<=`，确保边界包含。 

另一个边缘情况是时间为零时。 用于输入$x=0, y=0, v=10, t=0$，双方的计算结果都为零，并且条件成立。 平方公式避免了任何除法或浮点歧义。 

第三种边缘情况涉及大坐标。 为了$x=y=10^5, v=2\cdot10^5, t=1$，两边都能安全地适合 Python 整数，并且比较仍然准确。 在具有固定宽度整数的语言中，必须小心处理中间乘法以避免溢出，这正是计算后对乘积进行平方的原因$v \cdot t$加宽类型是必要的。
