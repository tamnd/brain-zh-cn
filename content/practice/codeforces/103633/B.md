 ## 解决方案
 ## 问题理解

 我们得到一个整数数组和三个参数：目标值和模数。 任务是计算有多少个有序四元组索引$(i, j, k, l)$我们可以这样形成，使得由两个独立比率形成的表达式与目标匹配。 每个比率都使用下限除法，因此每对指数都会贡献一个以下形式的值$\lfloor A_i / A_j \rfloor$，我们希望两个这样的值的总和等于$T$。 

关键点是索引允许重复，因此四个位置中的每一个都是独立选择的$[1, N]$。 这立即告诉我们该结构不是不同元素的组合，而是完全基于数组中值的频率。 

从约束条件来看，$N$可以很大（最多大约$10^5$），所以任意$O(N^2)$对的枚举太慢。 甚至建造所有$N^2$成对商直接是边界，因为它会创建$10^{10}$最坏情况下成对。 因此，预期的解决方案必须按值压缩对，而不是按索引。 

微妙的边缘情况来自重复的值和小数字。 例如，如果所有元素都是$1$，那么每个楼层划分为$1$，所以每个四元组都有贡献$2$。 答案就变成了$N^4$，必须通过计算重数来处理，而不是尝试任何成对逻辑。 

另一种边缘情况是除法频繁产生零时。 例如，如果$A_i < A_j$， 然后$\lfloor A_i / A_j \rfloor = 0$，它可以主导分布并创建许多贡献为零的对。 假设正比率或忽略指数之间的排序的幼稚实现会严重误算。 

## 方法

 蛮力方法是迭代所有指数的四元组并直接评估两个底线划分。 这是正确的，因为它准确地反映了定义。 然而，它需要$N^4$操作，其中在$N = 10^5$是完全不可行的，按顺序$10^{20}$评价。 

我们可以通过将问题分成两个独立的部分来减少这种情况。 该表达式是两个独立对函数的和：$f(i, j) = \lfloor A_i / A_j \rfloor$。 我们不使用四元组，而是首先计算每个可能值的出现频率$f(i, j)$发生在所有有序对中$(i, j)$。 一旦我们有了这个频率表，原来的问题就变成了计算总和为$T$，这是一个标准的卷积式计数问题。 

关键的见解是该结构是成对乘法的，但两项是加法的。 一旦我们压缩所有$N^2$将贡献配对到频率数组中$cnt[x]$，最终答案变为$\sum_x cnt[x] \cdot cnt[T - x]$，它可以在楼层划分的可能值范围内有效计算。 

计算$cnt[x]$有效地，我们利用了这样一个事实：$\lfloor A_i / A_j \rfloor = x$适用于连续范围$A_i$值一次$A_j$是固定的。 这允许对值进行分组$A_i$使用频率数组并迭代块中的除数而不是单个对。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^4)$|$O(1)$| 太慢了|
 | 对频率+卷积|$O(N \sqrt{V} + V)$|$O(V)$| 已接受 |

 这里$V$是数组中的最大值。 

## 算法演练

 我们将问题转化为首先计算成对的地板划分结果，然后将它们组合起来。 

1.构建频率数组$freq[v]$计算每个值在输入数组中出现的次数。 这使我们能够使用值而不是索引。 
2. 对于每个可能的分母值$b$，迭代所有可能的值$a$在分组范围内，使得$\lfloor a / b \rfloor$保持不变。 而不是检查每一个$a$，我们跳过区间$[k \cdot b, (k+1)\cdot b - 1]$，因为该区间内的所有值产生相同的商$k$。 
3. 对于每个这样的间隔，累计有多少对$(a, b)$产生给定的商$k$在频率数组上使用前缀和。 这构建了一个数组$cnt[k]$，表示楼层划分等于的有序对的数量$k$。 
4.一次全部$cnt[k]$计算，通过迭代所有可能的结果来计算最终答案$x$，并添加$cnt[x] \cdot cnt[T - x]$。 每个这样的产品都有数种方法来挑选一对生产$x$和另一个生产$T-x$。 
5. 对所有内容取模$MOD$。 

关键的设计选择是预先计算所有配对结果一次。 这避免了在四重循环内重复重新计算楼层划分。 

### 为什么它有效

 正确性依赖于将四元组完全分离成两个独立的事件对。 每个有效的四元组都唯一对应于一对有序的结果对：一个产生$x$，另一个生产$T-x$。 由于索引是独立的并且允许重复，因此可以独立选择两对，因此计数的乘法是有效的。 按商间隔分组确保每个有序对$(i, j)$只计算一次$cnt$，保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, T, MOD = map(int, input().split())
    A = list(map(int, input().split()))

    maxA = max(A)
    freq = [0] * (maxA + 1)

    for x in A:
        freq[x] += 1

    # compute cnt[q] = number of ordered pairs (i, j) with floor(A[i]/A[j]) = q
    cnt = {}

    for b in range(1, maxA + 1):
        if freq[b] == 0:
            continue
        fb = freq[b]

        # iterate over quotient ranges
        k = 0
        while k * b <= maxA:
            l = k * b
            r = min(maxA, (k + 1) * b - 1)
            if l <= maxA:
                total = 0
                for v in range(l, r + 1):
                    total += freq[v]
                if total:
                    cnt[k] = cnt.get(k, 0) + fb * total
            k += 1

    keys = list(cnt.keys())
    ans = 0
    for x in keys:
        y = T - x
        if y in cnt:
            ans = (ans + cnt[x] * cnt[y]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```频率数组将输入压缩为值计数，因此我们再也不会迭代索引。 嵌套循环结束$b$商范围是地板除法聚合的标准技巧：对于固定除数，商在连续段上是恒定的，我们通过范围跳跃而不是每个元素检查来利用它。 

最终基于字典的卷积避免了在大的固定商范围上迭代，因为仅存储可实现的值。 

## 工作示例

 ### 示例 1

 考虑输入：```
3 2 1000
1 1 1
```所有部门均满足$\lfloor 1/1 \rfloor = 1$。 所以每对贡献 1。 

| 步骤| 碳纳米管[1] | 其他碳纳米管| 推理|
 | --- | --- | --- | --- |
 | 建立对| 9 | - | i 有 3 个选择，j 有 3 个选择 |
 | 卷积| ans = cnt[1] * cnt[1] | - | 只有 x=1 有贡献 |

 最终答案是$9 \cdot 9 = 81$。 

这证实了重复值通过独立配对自然会产生乘法爆炸。 

### 示例 2

 取一个小的混合数组：```
3 1 1000
1 2 3
```我们计算楼层划分对：

 | 对类型 | 价值|
 | --- | --- |
 | (1,2),(1,3) | (1,2),(1,3) | 0 |
 | (2,1) | 2 |
 | (2,3) | 0 |
 | (3,1) | 3 |
 | (3,2) | 1 |
 | (3,3) | 1 |

 所以计数变成：$cnt[0]=3, cnt[1]=2, cnt[2]=1, cnt[3]=1$现在与$T=1$，我们检查所有分解$x + y = 1$：

 | x | y | 贡献 |

 总计 = 12。 

这表明小比率的零重贡献如何显着主导答案。

 ## 边缘情况

 当所有元素都相同时，每对都会产生相同的商，因此频率图会折叠为单个键。 该算法正确地聚合了所有$N^2$配对成一个条目，然后在卷积过程中将其平方，匹配所需的$N^4$四倍计数。 

当数组严格递增时，大多数除法的结果为零。 该算法可以处理此问题，因为商块中的所有值都已正确分组，因此无需特殊的大小写即可自然捕获零贡献的主导地位。 

什么时候$T$大于任何可能的两个商之和，卷积循环不会产生匹配对，并且答案正确地保持为零。
