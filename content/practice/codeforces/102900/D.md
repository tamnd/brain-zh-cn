 ## 解决方案
 ## 问题理解

 我们得到一个从 0 到 n 的一维段。 两个步行者从该路段的某处开始。 每个步行者都有自己的起始位置和自己的恒定速度，并且都可以沿着路段来回行走，但永远不能走出区间。 

一旦至少有一个步行者在任何时刻穿过该线段上的一点，该点就被视为“被覆盖”。 由于它们可以自由改变方向，因此每个步行者都可以在任一方向扫过间隔，从而有效地选择遵守速度限制和边界的任何路径。 

任务是确定最小时间 T，使得 [0, n] 中的每个点在时间 T 之前已被两个步行器中的至少一个访问过。 

关键的困难在于覆盖范围是在真实的段上连续的，而不是离散的点。 这意味着我们正在有效地尝试确保随着时间的推移，两个扩展的到达区域所覆盖的间隔的并集成为整个段。 

这些约束意味着最多 10^4 个测试用例，每个用例 n 最多 10^4 个。 对时间步长或连续运动进行简单的模拟是不可能的。 任何方法都必须在恒定或对数时间内计算每个测试用例的答案。 

当一个步行者从远离另一个步行者开始并且两者都非常慢时，就会出现一种微妙的边缘情况。 例如，如果 n 很大并且两个速度都很小，则答案将由任一方最慢覆盖所需的时间决定。 另一个极端情况是，当两个步行器在同一端附近开始时，除非另一个步行器做出贡献，否则到达路段一侧的成本会很高。 

说明结构的最小示例是两个步行器从相反的两端开始：p1 = 0，p2 = n。 那么答案就是 max(n / v1, n / v2)，因为每个步行者独立向内扫掠，并且除了平行覆盖之外没有重叠好处。 

## 方法

 蛮力视角是模拟步行者随时间的运动并跟踪该部分的哪些部分被覆盖。 人们可以将路段离散成小点，并以小时间步长模拟运动，更新每个步行者在每一步的可到达位置。 如果将段离散化为 k 个点，并以与分辨率成比例的步长来模拟时间，则每一步都会更新两个步行器，并且我们会重复检查覆盖范围。 这很快就变得不可行，因为实现 10^-6 精度将需要极其精细的离散化，导致每个测试用例需要数十亿次操作。 

关键的观察结果是，每个步行者独立贡献一个随时间线性增长的“覆盖半径”，但受到段边界的限制。 在时间 t 之后，步行者 i 可以精确地达到区间 [max(0, p_i − v_i t), min(n, p_i + v_i t)]，因为它们可以自由转动并在两个方向上全速向外扫掠。 

因此，在时间 t 时，我们得到两个间隔，每个步行者各有一个，问题就简化为找到最小的 t，使得这两个间隔的并集完全覆盖 [0, n]。 这是纯区间覆盖条件。 

当且仅当两个步行器的最左边可到达点和最右边可到达点之间不存在未覆盖的间隙时，并集才覆盖整个段。 该条件简化为检查并集后最左边的边界是否为 0，最右边的边界是否为 n，以及两个区间是否重叠或接触，因此中间没有间隙。 

形式上，我们计算可达间隔并检查它们是否重叠或覆盖边界间隙。 这导致t中的条件是单调的，因此我们可以二分查找最小有效时间。 每次检查都是 O(1)，使得解决方案高效。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 离散模拟| 每次测试 O(n / ε) | O(n) | 太慢了|
 | 区间+二分查找| 每次测试 O(log precision) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 在固定时间 t 内，计算每个步行者以速度 v 移动并自由转弯所能到达的最大范围，即其走过的区间。 这给出了 [l1, r1] 和 [l2, r2]。 
2. 将这些间隔限制在段边界 [0, n] 上，因为两个步行者都不能离开该段。 
3. 对区间进行排序或比较，以便我们可以推理它们的并集。 不失一般性，假设 l1 ≤ l2。 
4. 检查区间的并集是否覆盖整个段。 如果 l1 为 0 并且并集的右端点至少为 n，并且间隔之间没有间隙，则情况成立。 具体而言，如果按正确顺序 r1 ≥ l2 或 r2 ≥ l1，则覆盖范围成立，并且极值端点覆盖两端。 
5. 在连续范围内对 t 使用二分查找。 下限为 0，上限可以安全地为 2n 除以最小速度，因为在最坏的情况下，一个步行者可以单独穿越该路段两次。 
6. 对于每个中点，运行覆盖率检查。 缩小搜索范围，直到间隔小于所需的精度。 

应用二分搜索的原因是增加时间只会单调扩展可达间隔，因此一旦完全覆盖成为可能，它就永远是可能的。 

### 为什么它有效

 核心不变量是，在任何时间 t，每个步行者的可达集恰好是距离其起点 v_i t 内的点的间隔，并被边界裁剪。 这是因为与直线向外运动的任何偏差只会减少一个方向的范围，而转弯则允许在两个方向上独立扩展，直至达到速度限制。 

因此，问题简化为两个连续扩展区间的并集。 由于这些区间随t单调扩展，谓词“全覆盖[0, n]”是单调的，保证二分查找收敛到最小有效时间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(n, p1, v1, p2, v2, t):
    l1 = max(0.0, p1 - v1 * t)
    r1 = min(n, p1 + v1 * t)
    l2 = max(0.0, p2 - v2 * t)
    r2 = min(n, p2 + v2 * t)

    if l1 > l2:
        l1, l2 = l2, l1
        r1, r2 = r2, r1

    if r1 < l2:
        return False

    return min(l1, l2) <= 0 and max(r1, r2) >= n

def solve():
    it = sys.stdin
    t = int(it.readline())
    for _ in range(t):
        n, p1, v1, p2, v2 = map(float, it.readline().split())

        lo, hi = 0.0, 1e10

        for _ in range(80):
            mid = (lo + hi) / 2
            if ok(n, p1, v1, p2, v2, mid):
                hi = mid
            else:
                lo = mid

        print(f"{hi:.10f}")

if __name__ == "__main__":
    solve()
```功能`ok`将时间猜测转换为两个可达间隔。 关键的实现细节是限制为 [0, n]，因为忽略边界会高估覆盖范围。 

间隔交换确保我们在检查间隙之前正确推理排序。 二分搜索循环运行固定次数的迭代以保证浮点精度。 

最终打印值使用高精度格式，因为该问题允许较小的绝对或相对误差。 

## 工作示例

 考虑线段为 [0, 10] 的情况，步行者从 2 和 8 开始，速度均为 1。 

在时间 t = 0 时：

 | t | l1 | r1 | l2 | r2 | 覆盖范围|
 | --- | --- | --- | --- | --- | --- |
 | 0 | 2 | 2 | 8 | 8 | 没有 |

 在 t = 2 时：

 | t | l1 | r1 | l2 | r2 | 覆盖范围|
 | --- | --- | --- | --- | --- | --- |
 | 2 | 0 | 4 | 6 | 10 | 10 是的 |

 此时步行者一覆盖了[0,4]，步行者二覆盖了[6,10]，他们在4和6之间仍然留有间隙，所以还没有实现全覆盖。 这表明仅仅到达两端是不够的； 重叠必须消除内部间隙。 

现在考虑第二种情况，n = 10，p1 = 0，p2 = 10，v1 = v2 = 1。 

| t | l1 | r1 | l2 | r2 | 覆盖范围|
 | --- | --- | --- | --- | --- | --- |
 | 5 | 0 | 5 | 5 | 10 | 10 是的 |

 这证实了当两个端点都到达中点时，从端点的对称扩展可以实现完美的覆盖。 

第一个示例演示了重叠的重要性，而第二个示例则证实端点到端点对称是最简单的最佳配置。

 ## 边缘情况

 当两个步行者从同一位置开始时，它们的间隔总是相同地扩展。 该算法处理这一问题是因为并集间隔只是一个扩展间隔，并且覆盖检查正确地简化为该间隔是否同时达到 0 和 n。 

当一个步行器非常慢而另一个很快时，二分搜索正确地收敛到完全由快步行器到达最远边界主导的时间，因为慢步行器仅在本地做出贡献。 一旦快速间隔单独覆盖该段，间隔检查自然会忽略慢速贡献。 

当两个步行者都靠近一个边界时，关键的困难是到达另一端。 该算法处理此问题是因为每个间隔都被限制，因此不会计算超出 0 或 n 的人为扩展，并且二分搜索会增加时间，直到至少一个间隔跨越整个段。
