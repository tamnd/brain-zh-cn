 ## 解决方案
 ## 问题理解

 我们有两个整数$X$和$Y$，并且我们考虑所有对$(i, j)$在哪里$0 \le i \le X$和$0 \le j \le Y$。 对于每一对，我们只关心两个数字的按位与为零的数字，这意味着这两个数字永远不会共享公共设置位。 

对于每个有效对，我们计算值$\lfloor \log_2(i + j) \rfloor + 1$，这本质上是表示所需的位数$i + j$以二进制形式。 我们对所有有效对求和该值，并输出结果模$10^9 + 7$。 

因此，问题是整数对网格上的加权计数任务，其中有效性取决于按位约束，而贡献仅取决于总和的二进制长度。 

限制条件非常大：$X, Y \le 10^9$直至$10^5$测试用例。 这立即排除了任何对每个测试用例迭代甚至在一个维度上迭代的解决方案。 甚至$O(X)$每个测试都是不可能的，并且任何明确枚举范围内贡献的内容都必须替换为基于数字或按位组合的方法，通常涉及位的动态编程。 

打破朴素推理的主要边缘情况是忽略或部分强制执行 AND 约束。 例如，对于$X = 3, Y = 3$，这对$(1, 2)$有效期自$1 \& 2 = 0$， 但$(1, 3)$无效，因为它们共享了一点。 一个朴素的前缀和$i+j$如果不过滤位冲突，将会严重计数。 

另一个微妙的问题是该函数仅取决于最高位$i+j$，如此多的对归入同一贡献类别。 如果我们将贡献视为每对独立的，我们就会错过允许聚合的组合结构。 

## 方法

 暴力方法将枚举所有对$(i, j)$，检查是否$i \& j = 0$, 计算$i+j$，然后添加二进制长度。 这是正确的，但要付出代价$O(XY)$每个测试用例的操作，其顺序为$10^{18}$在最坏的情况下并且立即不可行。 

甚至减少迭代$i$并采用快速计数方法进行有效$j$仍然留给我们$O(X)$，乘以时太大$10^5$测试用例。 

关键的结构观察是约束$i \& j = 0$各个位之间是按位独立的。 在每个位位置，我们不能同时在两个数字中放置 1。 这表明数字 DP 超过二进制表示$i$和$j$，跟踪我们是否仍然受$X$和$Y$，并确保没有重叠位。 

一旦我们可以计算出有多少有效对产生给定范围的总和，我们就可以按最高设置位对对进行分组$i+j$。 我们不是单独计算每一对，而是计算有多少对属于范围$[2^k, 2^{k+1}-1]$。 每个范围贡献$(k+1)$到答案，因此问题简化为对位长度桶进行求和。 

这将问题转化为多维数字 DP，其中状态跟踪二进制位置、边界紧密度以及进位传播$i+j$。 AND 限制简化了转换，因为在每一位我们只允许$(0,0), (0,1), (1,0)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举|$O(XY)$|$O(1)$| 太慢了|
 | 具有范围聚合的基于位的数字 DP |$O(60^3)$每次测试（或类似）|$O(60)$| 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 转换$X$和$Y$转换为固定长度的二进制数组（最多 30-31 位，但为了进位安全，我们将其填充到 60）。 这使我们能够对所有位位置进行统一推理。 
2. 定义位位置上的数字 DP 状态$p$，我们决定的位$i$和$j$从最重要到最不重要。 在每个位置，我们还跟踪是否有前缀$i$和$j$仍然等于前缀$X$和$Y$。 这种“严格”的跟踪是必要的，因为我们必须保持在界限之内。 
3. 对于每个状态，我们尝试所有有效的位分配$(b_i, b_j)$这样它们就不会违反 AND 约束。 这限制了我们每个位只能进行三个转换：$(0,0), (0,1), (1,0)$。 
4.我们维护一个计算进位感知机制$i+j$。 我们不是显式地形成数字，而是跟踪下一位的进位以及当前前缀是否已经保证最终的总和将超过某个阈值。 
5. 处理完所有位后，我们按最高位对结果对进行分类$i+j$。 这是通过跟踪总和中最高有效进位或最高有效生成 1 的位置来完成的。 
6. 我们通过将属于每个位长度类别的有效对的数量乘以该类别值来累积贡献，并对所有内容求模求和$10^9+7$。 

关键的设计选择是我们从不明确计算$i$,$j$， 或者$i+j$作为整数。 一切都是从 DP 状态转换推断出来的。 

### 为什么它有效

 DP 维护有效前缀的完整特征$(i, j)$在两个约束下：有界性和不相交位。 每个完整的位分配都对应于 DP 中的一条有效路径，并且每个有效对都只表示一次。 因为贡献仅取决于最高设置位$i+j$，按 DP 派生的位长度分组可以保留正确性，而无需重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

MAXB = 60

def solve_case(x, y):
    # dp[pos][tight_x][tight_y][carry]
    dp = [[[[0] * 2 for _ in range(2)] for _ in range(2)] for _ in range(MAXB + 1)]
    dp[0][1][1][0] = 1

    for pos in range(MAXB):
        xb = (x >> pos) & 1
        yb = (y >> pos) & 1

        for tx in range(2):
            for ty in range(2):
                for carry in range(2):
                    cur = dp[pos][tx][ty][carry]
                    if not cur:
                        continue

                    for a in (0, 1):
                        for b in (0, 1):
                            if a & b:
                                continue

                            if tx and a > xb:
                                continue
                            if ty and b > yb:
                                continue

                            ntx = tx and (a == xb)
                            nty = ty and (b == yb)

                            s = a + b + carry
                            nc = s >> 1

                            dp[pos + 1][ntx][nty][nc] = (dp[pos + 1][ntx][nty][nc] + cur) % MOD

    ans = 0
    for tx in range(2):
        for ty in range(2):
            for carry in range(2):
                cnt = dp[MAXB][tx][ty][carry]
                if not cnt:
                    continue

                msb = MAXB + (1 if carry else 0)
                ans = (ans + cnt * msb) % MOD

    return ans

t = int(input())
for _ in range(t):
    x, y = map(int, input().split())
    print(solve_case(x, y))
```DP 按位位置组织，每个转换直接在循环中强制执行不相交位条件$(a, b)$。 严格的旗帜确保我们永远不会超过$X$或者$Y$在它们的二进制前缀中。 进位是向上传播加法所需的唯一信息。 

最终的聚合利用了这样一个事实：DP 之后唯一缺失的信息是有效位长度$i+j$，这取决于进位是否在最高处理位之后幸存下来。 

一个微妙的点是 DP 将所有数字视为固定宽度的二进制文件。 如果不填充到足够大的 MAXB，进位传播将错误地截断接近 2 的幂的和的贡献。

 ## 边缘情况

 当$X = 0$或者$Y = 0$，仅存在一对有效的：$(0,0)$。 DP 从位置 0 处的单个状态开始，并且由于所有位都为零，因此它永远不会分支。 进位始终保持为零，因此最终的位长度贡献为 1，与正确的二进制表示长度 0+0 匹配。 

什么时候$X$和$Y$都是 2 减 1 的幂，除了同时出现的 1 之外，所有低位组合都是允许的。 DP探索完整的深度三元树$B$，但通过严格约束进行修剪可确保不会发生无效溢出。 进位仅在两个位均为 1 时出现，这已经被禁止，因此在这种情况下进位保持为零，并且最高位长度在所有路径上都是稳定的。
