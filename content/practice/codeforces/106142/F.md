 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，对于每个位置，我们必须回答一个关于该位置值的单独优化问题。 索引 i 处的元素被视为固定参考元素。 我们可以从数组中删除任何其他元素，删除后我们查看剩余的序列。 索引 i 的目标是使其值成为所有剩余元素中的最大值，并允许存在平局。 我们希望实现这一目标所需的删除次数最少。 

查看任务的另一种方式是，对于每个元素 a[i]，我们希望保留仍包含 a[i] 的数组的子集，并且在该子集中不能有任何元素严格大于 a[i]。 所有大于 a[i] 的内容都必须被删除。 在小于或等于a[i]的元素中，我们可以自由保留或删除，但删除越少越好，因为删除的成本很高。 

约束条件高达 n = 200000，因此任何在所有 i 上具有二次行为的解都是不可能的。 通过扫描数组并计算删除次数来重新计算每个 i 的答案的简单方法将导致 O(n^2)，这太大了。 这迫使我们采用一种重用数组全局结构的解决方案，而不是从头开始重新计算。 

当值重复时会出现一个微妙的情况。 如果一个值出现多次，选择一次出现作为“最大代表”意味着我们可以保留其他相等的元素。 例如，在[5, 1, 5]这样的数组中，对于中间元素1，必须删除所有5，所以答案是2。对于5，我们可以保留所有5，所以不需要删除。 

另一个重要的边缘情况是当元素已经是全局最大值时。 在这种情况下，不需要删除，因为它已经满足条件。 任何错误地将相等元素计为删除的方法都会在这里失败。 

## 方法

 蛮力的想法很简单。 对于每个索引 i，固定值 x = a[i]。 然后我们计算有多少元素严格大于 x。 所有这些元素都必须被删除。 删除它们后，剩余的数组自动满足 x 为最大值。 所以 i 的答案就是大于 a[i] 的元素的数量。 这是正确的，因为任何有效的子集都不能包含更大的元素。 

一个简单的实现是，对于每个 i，扫描整个数组并计算大于 a[i] 的值。 每个查询的成本为 O(n)，导致总操作量为 O(n^2)。 当n达到200000时，这远远超出了可行的极限。 

关键的观察结果是，答案仅取决于值 a[i]，而不取决于其位置。 如果两个位置具有相同的值，则它们的答案是相同的。 因此，我们不需要重新计算每个索引，而只需知道对于每个不同的值 x，数组中有多少元素大于 x。 

这立即表明排序或频率积累。 如果我们对数组值进行排序，我们可以使用频率后缀和来计算每个值 x 严格大于 x 的元素数量。 一旦我们知道了每个可能的值，我们就会将预先计算的结果分配回所有位置。 

问题的结构简化为全局排名问题：每个元素的答案取决于其在值排序中的排名，而不是其在原始数组中的位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 值频+排序| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 ## 算法演练

1. 计算每个不同值在数组中出现的次数。 这将问题从位置压缩为值，因为相同的值在最终答案中表现相同。 
2. 按升序对不同值进行排序。 这建立了从最小到最大的清晰排序，这使我们能够推断出有多少元素大于给定值。 
3. 从最大到最小遍历排序值，同时维护到目前为止已看到的元素数量的运行总数。 此运行总计表示有多少元素严格大于当前值。 
4. 对于按降序排列的每个值 x，在添加 x 的频率之前，将 answer[x] 指定为等于当前运行总计。 这是可行的，因为此时我们已经考虑了所有严格大于 x 的值。 
5. 将 x 的频率添加到运行总计中，以便较小的值能够正确地看到其计数中包含的所有较大值。 
6. 最后，使用频率表或字典查找将这些预先计算的答案映射回每个原始索引 i。 

为什么它有效：对于任何固定值 x，必须在任何有效子集中删除每个大于 x 的元素。 出于正确性原因，小于或等于 x 的元素永远不需要被删除，因为 x 仍然是其中的最大值。 因此，最小删除恰好是严格大于 x 的元素计数，并且排序值上的后缀累积针对每个不同值精确地计算该数量一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    freq = {}
    for v in a:
        freq[v] = freq.get(v, 0) + 1

    vals = sorted(freq.keys())
    
    greater_count = 0
    ans_value = {}

    for v in reversed(vals):
        ans_value[v] = greater_count
        greater_count += freq[v]

    out = []
    for v in a:
        out.append(str(ans_value[v]))

    print(" ".join(out))

if __name__ == "__main__":
    solve()
```实现首先构建所有值的频率图。 这是至关重要的，因为该算法对值类别而不是位置进行操作。 键的排序列表定义了我们累积较大元素计数的顺序。 

反向遍历确保当我们分配ans_value[v]时，变量greater_count已经包含严格大于v的所有元素。更新步骤发生在分配之后，这对于避免意外计算当前值本身至关重要。 

最后，我们使用直接字典查找将答案映射回每个位置，这使最终的重建保持线性。

 ## 边缘情况

 当所有元素都相等时，算法将为每个值分配零，因为反向遍历不会看到严格更大的元素。 用于输入`[10, 10, 10]`，频率映射有一个键，并且 Greater_count 从零开始，因此每个索引都接收零。 这符合所选元素已经是最大值的要求。 

当数组严格递增时，除最后一个元素之外的每个元素都必须删除所有较大的后缀元素。 为了`[1, 2, 3, 4]`，后缀累积分配正确的递减答案，因为每个值都按值顺序准确地看到其右侧的元素，这对应于所有严格更大的值。
