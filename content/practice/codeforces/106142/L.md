 ## 解决方案
 ## 问题理解

 给定三角形的两条固定边长，$a$和$b$，我们想了解第三边的整数值是什么$c$可以使这三个线段形成一个面积为正的有效三角形。 

三角形在非退化时具有正面积，这意味着所有三角形不等式都是严格的。 因此，第三条边必须同时满足两个条件：它必须足够短以适合其他两条边，并且它也必须足够长以使三角形不会塌陷成一条直线。 

具体来说，对于双方$a$,$b$， 和$c$，有效性要求：$a + b > c$,$a + c > b$， 和$b + c > a$。 

任务是找到最小和最大的整数$c$满足这些条件的。 

约束条件达到$10^6$，这意味着任何解决方案都必须在每个测试用例的恒定时间内运行。 迭代所有可能值的解决方案$c$在最坏的情况下，达到边的总和会太慢。 约束的结构表明是直接的代数表征而不是枚举。 

一种常见的微妙故障情况是一侧与另一侧相比非常大。 例如，如果$a = 1$和$b = 10$，那么许多幼稚的实现错误地假设任何$c$之间$1$和$11$有效，但他们忘记了不等式的严格性和三角形条件施加的下界$c > |a - b|$。 

## 方法

 暴力方法会尝试每个整数$c$从 1 到$a + b - 1$，检查是否$a, b, c$形成一个有效的三角形。 每次检查都是恒定时间，因此复杂度变为$O(a + b)$，在最坏的情况下大约是$2 \cdot 10^6$。 虽然这在某些环境中可能会通过，但它是不必要的且脆弱的，特别是因为循环内的逻辑只是验证可以简化的不等式。 

关键的观察是三角形有效性约束崩溃成一个紧密的区间$c$。 由三角不等式得：$c < a + b$,$c > a - b$,$c > b - a$。 

最后两者合并为$c > |a - b|$。 自从$c$是一个整数，这变成$c \ge |a - b| + 1$。 类似地，严格上限变为$c \le a + b - 1$。 

所以有效范围是一个连续的整数段：$$|a - b| + 1 \le c \le a + b - 1$$这将问题变成了简单地计算区间的端点而不是搜索。

 | --- | --- | --- | --- |
 | 蛮力 |$O(a + b)$|$O(1)$| 太慢了 |
 | 最佳 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 我们直接从三角形不等式计算第三条边的有效范围。 

1. 计算下界$c$作为$|a - b| + 1$。 这确保了三角形不会退化并且满足$a + c > b$和$b + c > a$。 绝对差异捕获了两个较低约束中较强的一个。 
2. 计算上界$c$作为$a + b - 1$。 这通过确保区域的严格积极性$c < a + b$。 
3. 输出两个值作为答案。 

### 为什么它有效

 三角不等式系统定义了一组凸可行值$c$。 因为所有约束都是线性的$c$，它们的交集是一个连续区间。 下界来自确保$c$大于两个差值$a - b$和$b - a$，而上限来自总和约束。 由于需要整数值，因此连续间隔会转换为整数段，由于严格的不等式，端点调整为±1。 不存在间隙或不相交的区域，因此端点完整地描述了解决方案集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a = int(input())
b = int(input())

low = abs(a - b) + 1
high = a + b - 1

print(low, high)
```该解决方案读取两个整数并直接计算有效区间。 唯一微妙的一点是遵守严格的不平等：上限减一，下限增加到超过绝对差值。 这两项调整都是必要的，因为相等会产生退化三角形。 

不需要循环，也不需要额外的数据结构。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    a = int(input())
    b = int(input())

    low = abs(a - b) + 1
    high = a + b - 1

    return f"{low} {high}"

# provided sample
assert run("6\n14\n") == "9 19"

# equal sides

# minimal case

 ## 边缘情况

 一种重要的边缘情况是两个给定边相等。 例如，$a = b = 5$。 该算法计算$low = |5 - 5| + 1 = 1$和$high = 9$。 这正确地允许从 1 到 9 的所有有效三角形第三边。关键点是下限不会塌陷到零； 严格的不等式至少迫使 1。 

另一种边缘情况是一侧比另一侧大得多，例如$a = 1$,$b = 10$。 计算结果$low = 10$,$high = 10$，意味着仅存在一个有效的三角形。 追踪不平等证实了这一点：$c$必须大于$9$并且小于$11$，仅留下$10$。 

最后，当两个值都很大并且很接近时，间隔就会变宽。 为了$a = 10^6$,$b = 10^6$，我们得到从 1 到几乎全范围$2 \cdot 10^6$，表明约束由和条件而非差值决定。
