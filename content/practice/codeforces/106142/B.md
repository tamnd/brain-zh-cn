 ## 解决方案
 ## 问题理解

 我们得到了一个隐藏的长度排列$n$。 只有两个特殊元素很重要：最小值所在的位置$1$所在的位置，以及最大值所在的位置$n$坐着。 对于每个职位$i$，我们测量它距位置有多远$n$以及距离 的位置有多远$1$，然后我们将所有位置上的所有距离相加。 总和就是价值$m$。 

任务是相反的：我们不是从排列开始，而是给出$n$和$m$，并且我们必须构造任何可以精确产生该和的排列，或者确定不存在这样的排列。 

尽管该声明提到了完整的排列，但影响该值的唯一元素是$1$和$n$。 一旦它们的位置固定，所有其他数字都可以任意排列，而不影响计算值。 

约束足够小$n$所有测试用例总共可以达到 6000 个。 这意味着对所有位置对进行二次扫描是可接受的。 任何类似的东西$O(n^3)$或者由于全局限制，每个测试用例的重复重新计算仍然会通过，但没有必要。 

当一个人试图“贪婪地”选择职位时，就会出现一种微妙的边缘情况。$1$和$n$基于当地贡献。 例如，选择极端值来最大化或最小化距离总和会失败，因为位置对与总和之间的关系在不检查完整公式的情况下支持贪婪构造的方式不是单调的。 另一个问题是假设围绕中心对称总是有效的； 例如与$n=5$, 放置$1$和$5$在对称位置并不能唯一确定$m$，因此在不计算精确总和的情况下进行猜测会导致错误的值。 

## 方法

 天真的想法是重建整个排列并直接计算距离。 人们可以尝试放置$1$和$n$在所有可能的位置，然后任意填充其余位置，计算结果$m$每次。 这需要尝试$O(n^2)$放置并重新计算完整的$O(n)$每次求和，给出$O(n^3)$每个测试用例。 和$n$测试中总共最多 6000 个，这变得太慢了。 

关键的观察是，所有对排列的依赖都崩溃为只有两个索引：$1$，称之为$L$，以及位置$n$，称之为$R$。 一旦这些被固定，所有其他值仅通过到这两点的距离做出贡献。 

我们可以预先计算一个函数$g(i)$，距位置的距离总和$i$对于所有索引：$$g(i) = \sum_{j=1}^n |i - j|$$那么所需的值就变成：$$m = g(L) + g(R)$$这将问题简化为找到两个不同的索引$L$和$R$使得它们的预先计算值总和为$m$。 这是一个大小数组的经典二和$n$，可解为$O(n^2)$每个测试用例甚至哈希。 

一次$L$和$R$已知，我们通过放置来构建任何有效的排列$1$在$L$,$n$在$R$，并用任意未使用的数字填充剩余位置。

 ## 算法演练

 我们完全用位置对来重写问题。 

1. 预先计算一个数组$g$，其中对于每个位置$i$，我们计算总距离$i$到所有职位$1$通过$n$。 这可以在$O(n)$每个索引使用前缀推理而不是直接求和。 公式为：$$g(i) = \frac{i(i-1)}{2} + \frac{(n-i)(n-i+1)}{2}$$这将左侧和右侧的贡献分开$i$。 
2.我们现在需要找到两个不同的索引$L$和$R$这样：$$g(L) + g(R) = m$$我们迭代所有$L$，并为每个计算所需的补集$m - g(L)$。 
3. 我们存储的值$g(i)$在字典中将值映射到索引。 当我们找到补集的匹配时，我们确保索引是不同的。 
4. 如果不存在这样的对，则答案是不可能的，我们输出 0。 
5. 一旦我们有了$L$和$R$，我们通过放置来构建排列$1$在位置$L$,$n$在位置$R$，并用 2 到 的数字填充剩余位置$n-1$以任何顺序。 

构建步骤与总和的正确性无关； 所有的正确性都是通过选择来保证的$L$和$R$。 

### 为什么它有效

 关键的不变量是每个有效的排列都是唯一确定的，关于值$m$，通过其最小和最大元素的位置。 所有其他元素对两个距离和都有对称贡献，但它们的同一性根本不影响表达式。 这将状态空间从排列折叠为有序的索引对。 由于每个可行解完全对应于一对$(L, R)$，搜索所有对既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n, m = map(int, input().split())

        g = [0] * (n + 1)
        for i in range(1, n + 1):
            left = i * (i - 1) // 2
            right = (n - i) * (n - i + 1) // 2
            g[i] = left + right

        pos = {}
        L = R = -1

        for i in range(1, n + 1):
            need = m - g[i]
            if need in pos:
                L = pos[need]
                R = i
                break
            if g[i] not in pos:
                pos[g[i]] = i

        if L == -1:
            out.append("0")
            continue

        perm = [0] * n
        perm[L - 1] = 1
        perm[R - 1] = n

        cur = 2
        for i in range(n):
            if perm[i] == 0:
                perm[i] = cur
                cur += 1

        out.append(" ".join(map(str, perm)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现反映了对二和的减少。 数组$g$直接从闭合形式表达式计算，避免任何每个索引求和循环。 该字典存储以前见过的值$g(i)$，每一步都会检查补集是否存在。 

一个常见的陷阱是忘记了相同的值$g(i)$在其他问题中原则上可以出现在多个索引处，但是这里$g(i)$是严格凸的并且唯一标识位置，因此仅存储第一次出现的位置是安全的。 

排列结构是任意的，无法固定$1$和$n$，所以我们只需按升序填充剩余的槽即可。

 ## 边缘情况

 对于非常小的$n=3$，函数$g$只有两个不同的值，并且唯一有效的配对是强制的。 该算法可以正确处理此问题，因为哈希映射会立即检测或拒绝补码对。 

例如输入$n=3, m=2$，我们计算$g = [3,2,3]$。 在$i=2$，补集是$0$，它在映射中永远不存在，因此算法根据配置正确返回 0 或有效对。 

对于对称分布，例如$n=5, m=20$，存在多个有效对。 该算法选择它遇到的第一个有效匹配。 由于任何有效对都对应于正确的位置$1$和$n$，即使解不唯一，输出仍然有效。
