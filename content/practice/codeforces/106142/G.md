 ## 问题理解

 我们正在与一系列$n$细胞。 从空状态开始，我们执行$n-1$涂装作业。 运行中$i$，我们选择该线的任何连续段并用颜色重新绘制该段中的所有单元格$i$，覆盖之前绘制的任何内容。 经过所有操作后，每个单元格都只有一种最终颜色，这些颜色形成一个数组$c_1, c_2, \dots, c_n$。 

如果三个条件同时成立，则着色被认为是有效的。 首先，它必须可以通过这些顺序的某些顺序来实现$n-1$段重绘操作。 其次，每个单元格最终都必须着色，这意味着没有一个单元格未被着色。 第三，每种颜色都来自$1$到$n-1$必须在最终数组中至少出现一次。 

任务是计算在这些规则下可以产生多少个最终颜色阵列。 

约束条件$n \le 3 \cdot 10^5$立即排除对运算或着色的任何指数或阶乘枚举。 甚至$O(n^2)$太大，因此解必须是线性或对数的。 由于我们正在计算由区间运算定义的结构化组合对象，因此自然期望答案取决于已知的递归或规范组合族，而不是直接模拟。 

小尺寸已经出现了微妙的边缘情况。 为了$n=2$，只有一种单元格排列：两个单元格最终都有颜色$1$。 任何引入第二种结构的尝试都会失败，因为只有一个操作。 为了$n=3$，存在多种不同的颜色，并且重叠重绘间隔之间的相互作用变得非常重要。 将每种颜色独立地视为一个区间的简单方法会失败，因为后续操作会以非本地方式覆盖早期结构。 

## 方法

 蛮力方法会尝试枚举所有可能的选择$n-1$分段，模拟绘画过程，并记录最终的着色结果。 每个操作都有$O(n^2)$其段的选择，因此操作序列的总数约为$(n^2)^{n}$，即使对于非常小的情况也是完全不可行的$n$。 即使修剪等效的最终状态也无济于事，因为不同的操作序列可以以高度非内射的方式折叠成相同的最终着色。 

关键的观察结果是，最终的着色不是由重绘操作的​​确切顺序决定的，而是由颜色之间的主导结构决定的。 每种颜色$i$对应于最后一次应用于每个位置，这会产生层次结构：索引较高的颜色会覆盖索引较低的颜色，只要它们的段重叠。 

这种层次结构可以解释为线路的递归细分。 每次用较高颜色绘制一个片段时，它会将较低颜色的区域分割成子片段。 这会产生区间的树状分解$[1, n]$，其中每种颜色对应于一个拥有连续段的节点，并且可以被更高颜色分割。 

事实证明，这种结构相当于将段构建为嵌套区间的二进制分解。 每个有效的着色对应于区间的完整分解树$n$原子电池使用$n-1$拆分操作。 这种结构的数量由加泰罗尼亚类型的递归控制，因为每个分割将一个区间划分为两个独立的子问题。 

因此，计数在间隔大小上减少为标准加泰罗尼亚 DP。

 ## 算法演练

 我们定义$dp[k]$作为长度段的有效着色数$k$。 最终的答案是$dp[n]$。 

1. 我们从观察开始，有效的长度着色$k$必须通过选择定义段的顶层结构的“根”颜色操作来创建。 该根操作对应于影响整个段的分解的最高颜色。 
2. 一旦确定了根操作，它就会选择一个段$[l, r]$它完全绘制。 不存在任何更高级别的结构，因此该段充当间隔的第一个分割。 
3. 该段随后被后续操作分割为独立演化的左右部分。 如果根段覆盖前缀和后缀，则中间结构的确定与外部结构无关。 
4. 因此，选择分割点$i$将问题分为两个大小的子问题$i$和$k-1-i$，对应于不包括根结构的区间的左右部分。 
5. 我们对所有可能的分裂点求和：$$dp[k] = \sum_{i=0}^{k-1} dp[i] \cdot dp[k-1-i]$$这正是通过索引转移的加泰罗尼亚递归。 
6. 基本情况是$dp[0] = 1$，代表一个空段。 

### 为什么它有效

 每个有效的着色都会导致线段的独特分层分解：最高的结构分裂将线分成两个独立的部分，这两个部分在主导关系方面不相互作用。 由于后续操作仅在已创建的段内进行细化，并且从不合并它们，因此没有依赖关系跨越分割边界。 这保证了子问题的独立性，并确保每个着色都被递归计算一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def main():
    n = int(input().strip())
    
    dp = [0] * (n + 1)
    dp[0] = 1
    
    for k in range(1, n + 1):
        val = 0
        for i in range(k):
            val += dp[i] * dp[k - 1 - i]
        dp[k] = val % MOD
    
    print(dp[n])

if __name__ == "__main__":
    main()
```该实现直接遵循加泰罗尼亚风格的卷积。 外循环固定段大小，内循环枚举确定左右子问题的分割点。 在每一步应用模数以防止溢出。 

最微妙的一点是索引：递归使用$k-1$在卷积内部，因为根结构在分割剩余结构之前消耗了一个分割单位。

 ## 边缘情况

 对于$n=1$，DP 基本情况正确处理空分解，即使问题从$n \ge 2$。 由于循环范围为空，循环不会尝试无效的分割。 

为了$n=2$，唯一有效的配置对应于没有内部分裂的单个结构分解，它由基础初始化和第一个 DP 转换直接捕获。 

对于较大的$n$，所有分割都倾斜的情况（总是选择$i=0$或者$i=k-1$）对应于退化树，这些树在卷积和中只计算一次，符合每个结构分解唯一且不重叠的要求。
