 ## 问题理解

 我们正在观察一个按固定顺序循环经过四个阶段的交通灯。 灯光从红色开始，然后切换到黄色，然后绿色，然后再次黄色，最后返回红色，永远重复这个循环。 每种颜色都有固定的持续时间：红色每次出现持续r秒，黄色每次出现持续y秒，绿色持续g秒。 

在时间零时，灯刚刚变成红色。 我们被要求确定交通灯在时间 n 的状态，即开始后正好 n 秒。 

微妙之处在于我们必须区分两种情况。 如果时间 n 严格位于其中一个颜色区间内，我们就会输出该颜色。 如果时间 n 恰好与过渡时刻重合，则我们必须在该时刻稳定时输出单一颜色，或者输出一对描述过渡之前的颜色和过渡之后的颜色。 

每个测试用例的约束对于直接算术推理来说足够小，因为 r、y 和 g 最多为 1000，并且最多有 1000 个测试用例。 然而，n可以大到10^9，所以我们不能逐秒地模拟。 我们必须在周期长度上使用模运算来压缩时间。 

处理边界时间时会出现一个幼稚的错误。 例如，如果 r = 3、y = 4、g = 2 和 n = 3，则红色阶段恰好在时间 3 结束。简单的实现可能仍会显示“红色”，而正确的输出必须反映从红色到黄色的过渡，产生“红黄”。 类似的错误发生在每个相界，特别是在绿色到黄色的转变处，这很容易被忽视，因为黄色每个周期出现两次。 

另一个边缘情况来自整个周期长度的精确倍数。 在这些时刻，系统返回到红色开始，但我们仍然必须正确识别该时刻是边界点还是内部点。 

## 方法

 强力方法实际上是逐秒模拟交通信号灯，在跟踪当前阶段和该阶段的剩余时间的同时递减 n 直到达到零。 每次转换都会以固定循环红→黄→绿→黄→红的方式切换到下一种颜色。 这是正确的，因为它准确地反映了该过程，但当 n 很大时，它太慢了。 如果 n 达到 10^9，则每秒模拟每个测试用例将需要多达 10 亿步，这是不可行的。 

关键的观察结果是系统是周期性的。 一个完整的周期由红色表示 r 秒、黄色表示 y 秒、绿色表示 g 秒、黄色表示 y 秒组成，因此总周期长度为 r + g + 2y。 我们不是直接模拟时间，而是以该周期长度为模减少 n。 之后，我们只需要确定减少的时间在周期内的哪个位置即可。 

一旦我们将问题简化为单个周期，我们就将时间线分成连续的间隔，并将时间 n 的位置与间隔边界进行比较。 唯一需要注意的是正确检测 n 是否正好落在边界上或严格落在区间内，因为这决定了我们是输出一种颜色还是输出过渡对。

 ## 算法演练

 我们及时明确地模拟一个完整的周期。

1. 计算总周期长度为 T = r + y + g + y。 这代表系统再次返回红色的时刻。 
2. 减少 n modulo T，将 n 替换为 n mod T。如果 n 变为 0，我们将其解释为正好处于循环的开始处，这对应于边界点处的红色。 
3. 按顺序遍历时间线段来解释减少的时间：红色间隔从 0 到 r，第一个黄色间隔从 r 到 r + y，绿色间隔从 r + y 到 r + y + g，第二个黄色间隔从 r + y + g 到 T。 
4. 检查哪个段包含n。 如果 n 严格位于段内，则直接输出该颜色。 
5. 如果 n 恰好位于段边界上，则识别以 n 结束的段以及在 n 之后开始的下一个段。 将它们作为一对输出。 

关键的实现细节是，在决定我们是否在段“内部”之前，必须检查边界情况。 否则，像 n = r 或 n = r + y 这样的时间将被错误分类。 

### 为什么它有效

 交通灯是完全周期性的，周期为 T，任意时刻的状态仅取决于其在周期内的位置。 通过以 T 为模减少 n，我们将每个时间点映射到第一个周期中的等效状态。 由于循环边界是颜色之间的排他过渡，因此每个点要么严格位于一个区间内，要么恰好位于两个相邻区间共享的边界处。 这种结构保证了单色和过渡对之间的决策是明确且完整的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        r, y, g, n = map(int, input().split())

        T = r + y + g + y
        n %= T

        # boundaries
        red_end = r
        yellow1_end = r + y
        green_end = r + y + g
        yellow2_end = T

        if n == 0:
            print("red")
            continue

        if n < red_end:
            if n == red_end:
                print("red yellow")
            else:
                print("red")
        elif n < yellow1_end:
            if n == yellow1_end:
                print("yellow green")
            else:
                print("yellow")
        elif n < green_end:
            if n == green_end:
                print("green yellow")
            else:
                print("green")
        else:
            if n == yellow2_end:
                print("yellow red")
            else:
                print("yellow")

if __name__ == "__main__":
    solve()
```该代码首先使用模算术将时间压缩为单个周期。 然后它明确定义了四个阶段之间的所有转换边界。 通过在这些间隔内定位 n 来回答每个查询。 

最微妙的部分是处理边界上的平等。 该结构使用有序比较，以便一致地检查每个间隔，并通过精确端点处的内部检查来处理相等性。 

## 工作示例

 考虑 r = 3、y = 4、g = 2、n = 4。 

| 步骤| 价值|
 | --- | --- |
 | 周期 T | 13 |
 | n 模 T | 4 |
 | 段检查 | 3 < n < 7 | 3 < n < 7 |
 | 结果 | 黄色|

 这表明时间 4 在第一个黄色间隔内，而不是在边界处，因此输出只是黄色。 

现在考虑 r = 3、y = 4、g = 2、n = 3。 

| 步骤| 价值|
 | --- | --- |
 | 周期 T | 13 |
 | n 模 T | 3 |
 | 段检查 | n == red_end | n == red_end |
 | 结果 | 红黄|

 在这里，时间 3 恰好落在从红色到黄色的过渡处，因此必须报告两种颜色。 

这两条迹线说明了内部点和边界点之间的关键区别，这是问题的全部难点。

 ## 边缘情况

 一种关键的边缘情况是模数归约后 n 恰好为 0。 在这种情况下，我们正处于一个周期的开始，这意味着灯是红色的并且没有过渡。 该算法在任何间隔逻辑之前显式检查这一点并立即返回红色。 

另一种边缘情况是 n 恰好落在 r、r + y 或 r + y + g 上。 这些是转变时刻，算法确保在将值视为区间内之前检测到它们。 例如，如果 r = 3 且 n = 3，则检查 n == red_end 会触发并输出红黄色，而不是错误地报告红色。 

最后，当 n 是整个周期长度的倍数时，模得到 0，我们将其视为红色。 如果没有这种调整，边界逻辑会将其错误分类为位于所有段之外。
