 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中边描述城市之间的单向运动。 除了沿着这些边缘旅行之外，我们还可以在选定的顶点放置特殊的“门户”。 如果多个顶点包含门户，则它们可以直接相互访问，这意味着您可以立即从任何门户顶点跳转到任何其他门户顶点。 

目标是选择尽可能少的顶点来放置门户，以便在添加这些传送连接后，每个顶点都可以使用原始有向边和门户跳跃的组合到达每个其他顶点。 

换句话说，我们希望生成的有向图，通过所选门户顶点上的完整有向集团进行增强，变得强连接。 

约束允许最多$2 \cdot 10^5$顶点和边，因此任何解本质上都必须是线性的或线性算数的。 顶点或边上的二次方法将无法生存。 这立即表明我们需要对图进行结构压缩，而不是对单个顶点进行推理。 

当图已经强连接时，就会出现关键的边缘情况。 在这种情况下，根本不需要门户，因为每个顶点都已经通过有向路径到达每个其他顶点。 另一个重要的情况是图在可达性方面完全断开连接，例如具有许多组件的 DAG。 那么门户就是连接它们的唯一途径，而答案完全取决于凝结结构而不是原始图。 

## 方法

 如果我们尝试直接在原始图上进行推理，我们很快就会遇到一个问题：可达性是可传递的并且在有向边下是混乱的，并且添加门户会创建依赖于所选顶点子集的附加“快捷边”。 尝试所有子集是指数级的。 

更直接的暴力想法是选择一组顶点，在增强图上使用 BFS/DFS 模拟可达性，并检查图是否变为强连接。 即使我们修复了候选集，验证连接成本$O(n + m)$，子集的数量是$2^n$，这是不可能的。 

有向图的结构表明了一种标准压缩：强连通分量。 在强连接组件内，每个顶点都可以到达其他顶点，因此同一组件内的门户是多余的。 真正的问题发生在组件之间。 

如果我们将每个强连接组件收缩为单个节点，则生成的图是有向无环图。 在这个 DAG 内部，唯一的移动方式是沿着组件之间的边缘移动。 门户操作现在意味着，如果我们选择多个组件，我们可以在它们之间双向自由跳转，有效地将它们合并为一个“超级组件”。 

因此，任务变为：在凝聚 DAG 中选择最少数量的节点，使它们相互可达，使整个 DAG 变得强连接。 

在 DAG 中，强连接的根本障碍是源无法从其他节点到达，汇点也无法到达其他节点。 如果一个组件没有传入边，那么它就是一个源。 如果它没有出边，则它是一个汇点。 

一个重要的观察结果是，每个源组件必须包含一个门户或能够到达门户组件，并且对称地，每个接收器都必须可以从门户组件访问。 最佳策略简化为覆盖这些边界组件。 

事实证明，答案只是凝结图中源组件数量和汇组件数量的最大值，除了当图已经强连接并且两个计数都为零时的简并情况。 

直觉是，每个门户可以服务于凝结 DAG 中的一个“入口点”和一个“出口点”，但源和汇有独立的要求。 最佳匹配它们会导致两个计数中的最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对门户子集的暴力破解 |$O(2^n (n+m))$|$O(n+m)$| 太慢了|
 | SCC 冷凝 + 源/汇计数 |$O(n+m)$|$O(n+m)$| 已接受 |

 ## 算法演练

 我们首先使用 Kosaraju 算法或 Tarjan 算法将图压缩为强连通分量。 每个顶点都分配有一个组件 ID，不同组件之间的边定义一个压缩图。 

接下来，我们计算压缩图中每个组件的入度和出度。 

1. 在图上运行强连通分量分解。 这将顶点分组为相互可达性成立的最大集合。 在每个这样的集合中，内部连接不需要门户。 
2. 通过迭代原始图的所有边并在端点属于不同组件时在组件 ID 之间添加边来构建凝聚图。 重复的边并不重要。 
3. 对于每个组件，计算它是否具有来自另一个组件的传入边。 如果不是，则它是源组件。 类似地计算它是否有到另一个组件的传出边缘； 如果不是，则它是一个接收器组件。 
4. 计算凝结图中源组件和汇组件的数量。 
5. 如果总体上只有一个强连通分量，则返回 0，因为该图已经完全相互可达。 
6. 否则，返回源和汇的最大数量。 

这样做的原因是，在 DAG 中，每个组件在添加门户后都必须能够到达其他组件。 放置在组件中的门户可以有效地允许所有选定组件之间的双向旅行，因此它可以充当入口和出口中心。 然而，每个源都需要至少一种进入方式，每个接收器都需要至少一种出去方式，并且单个门户组件只能满足每个配对结构的这些角色之一。 因此，紧界由较大的一侧决定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def kosaraju(n, adj, radj):
    visited = [False] * n
    order = []

    def dfs1(v):
        visited[v] = True
        for to in adj[v]:
            if not visited[to]:
                dfs1(to)
        order.append(v)

    for i in range(n):
        if not visited[i]:
            dfs1(i)

    comp = [-1] * n

    def dfs2(v, c):
        comp[v] = c
        for to in radj[v]:
            if comp[to] == -1:
                dfs2(to, c)

    cid = 0
    for v in reversed(order):
        if comp[v] == -1:
            dfs2(v, cid)
            cid += 1

    return comp, cid

def solve():
    n, m = map(int, input().split())
    adj = [[] for _ in range(n)]
    radj = [[] for _ in range(n)]

    edges = []
    for _ in range(m):
        a, b = map(int, input().split())
        a -= 1
        b -= 1
        adj[a].append(b)
        radj[b].append(a)
        edges.append((a, b))

    comp, k = kosaraju(n, adj, radj)

    if k == 1:
        print(0)
        return

    indeg = [0] * k
    outdeg = [0] * k

    for a, b in edges:
        ca, cb = comp[a], comp[b]
        if ca != cb:
            outdeg[ca] += 1
            indeg[cb] += 1

    sources = sum(1 for i in range(k) if indeg[i] == 0)
    sinks = sum(1 for i in range(k) if outdeg[i] == 0)

    print(max(sources, sinks))

if __name__ == "__main__":
    solve()
```该解决方案首先读取图表并构建 Kosaraju 的双通道 SCC 分解所需的两个邻接表。 第一个 DFS 产生整理顺序，而反转图上的第二个 DFS 分配组件标识符。 

SCC 压缩后，我们明确检查是否只有一个组件。 这与已经强连接的图完全对应，因此不需要门户。 

然后，我们迭代原始边以隐式构建凝结图并计算每个组件的入度和出度。 这足以对源和汇进行分类，而无需显式存储所有压缩边缘。 

最后，我们计算有多少个分量的入度为零，有多少个分量的出度为零，并返回它们的最大值。 

一个微妙的点是多重边缘和自循环是无害的。 自循环永远不会影响 SCC 结构，并且重复边不会改变入度/出度分类，因为我们只关心组件是否至少有一个传入或传出边缘。

 ## 边缘情况

 关键的边缘情况是已经强连接的图。 该算法将其压缩为单个 SCC 并立即返回 0。这可以避免错误地计算组件级别不存在的人工源或汇。 

另一种情况是每个顶点都是孤立的图。 每个顶点都成为自己的 SCC，并且每个组件既是源又是汇，因为它没有边。 该算法计算源和汇均等于 n，并返回 n，这符合每个顶点至少选择一个门户来连接它们的需要。 

第三种情况是链式DAG。 压缩后，只有第一个节点的入度为零，只有最后一个节点的出度为零，因此答案变为 1。算法正确地识别出，一旦引入双向门户旅行，放置在链中任何位置的一个门户都可以充当连接所有组件的枢纽。
