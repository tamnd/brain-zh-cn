 ## 解决方案
 ## 问题理解

 我们得到从 1 到 n 的所有整数，每个整数都写在一张卡片上，并且使用固定的构造规则将这些卡片排列成单个序列。 该构造首先获取所有偶数并将它们按升序排列，然后附加所有奇数但按降序排列。 任务是确定哪个数字出现在最终序列中的位置 k 处。 

所以顺序不是任意的，也不是动态查询的。 它是通过将范围 [1, n] 分为两组（偶数和奇数）、对每组进行不同排序并将它们连接起来来完全确定的。 

n 高达一百万的约束意味着，如果我们可以直接计算位置，我们就无法显式构建或存储不必要的大型中间结构。 然而，即使是 n 上的线性构造在 Python 中在一秒钟内仍然可行，但预期的解决方案避免了完全构造，而是以 O(1) 或 O(log n) 逻辑计算答案。 

偶数和奇数之间的边界如何确定会产生微妙的边缘情况。 偶数的个数是floor(n/2)，该位置之后的所有内容都是相反顺序的奇数。 一个天真的错误是假设奇数也按递增顺序出现，或者错误计算存在多少个偶数，这会移动分割点。 

例如，如果 n = 7，则偶数为 [2, 4, 6]，奇数为 [7, 5, 3, 1]。 边界位于位置 3。如果 k = 4，则我们已经处于奇数段中，并且必须在该段内反转索引。 计算偶数时的任何差一错误都会立即产生错误的答案。 

## 方法

 暴力解决方案通过从 1 迭代到 n、将偶数和奇数分成两个数组、反转奇数数组并将它们连接起来，直接构造序列。 这正确地符合构造规则。 成本是 O(n) 时间和 O(n) 内存。 

虽然在严格的语言中，对于 n 高达 10^6 的情况，O(n) 是可以接受的，但 Python 开销和重复的列表操作使其不太优雅，并且根据环境限制可能会处于边缘状态。 更重要的是，序列的结构使得直接索引成为可能，而无需构建完整的数组。 

关键的观察结果是序列的第一部分是完全确定的：所有偶数都按升序排列。 这样的数字正好有 n // 2 个，它们的第 k 个位置就是第 k 个偶数，即 2k。 一旦 k 超过这个计数，我们就处于奇数段。 奇数段由所有奇数按降序排列组成，因此我们可以将 k 映射到奇数块内的位置并直接计算相应的奇数值。 

这消除了构建任何数组的需要，并将问题简化为简单的算术。

 ## 算法演练

 我们通过将位置 k 分为位于偶数块还是奇数块来计算答案。

1. 首先计算 1 和 n 之间有多少个偶数。 这是 n // 2。这给出了最终序列的第一段的长度。 
2. 如果 k 小于或等于 n // 2，则答案位于偶数段。 偶数列为 2、4、6 等，因此第 k 个元素就是 2 * k。 这是有效的，因为序列正是事件的排序列表。 
3. 否则，答案位于奇数段。 我们通过计算 k - n // 2 将 k 转换为奇数块内的索引。这给出了我们距离奇数段有多远。 
4. 奇数按升序排列为 1, 3, 5, ...，但序列以相反的顺序使用它们。 所以我们必须将奇数块中的位置t映射到第t大的奇数。 
5. 奇数元素总数为 n - n // 2. 按降序排列的第 t 个元素对应于按升序排列的第 (total_odds - t + 1) 个奇数。 将其转换为算术可以直接产生正确的值。 

为什么它有效

 该序列是两个独立排序的子序列的串联，偶数升序，赔率降序。 每个子序列都有一个针对其第 k 个元素的封闭式公式。 它们之间的边界由奇偶校验计数确定，与排列细节无关。 由于没有发生交错，每个索引都唯一地映射到一个算术级数，从而保持了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
k = int(input())

even_count = n // 2

if k <= even_count:
    print(2 * k)
else:
    t = k - even_count
    odd_count = n - even_count
    # t-th largest odd number
    # largest odd is (2*odd_count - 1)
    print((2 * odd_count - 1) - 2 * (t - 1))
```该代码首先计算存在多少个偶数，这确定了序列中的分割点。 如果 k 属于偶数前缀，我们直接返回 2k，因为偶数是从 2 开始按递增顺序排列的。 

如果 k 位于奇数后缀中，我们将 k 转换为奇数块内的索引。 最大的奇数是 2 * odd_count - 1，向后每一步减 2，因此我们减去 2 * (t - 1) 即可得到正确的元素。 

一个常见的陷阱是忘记奇数是按相反顺序放置的，这会导致错误地使用 2t - 1，而不是从最大值反转。 

## 工作示例

 考虑 n = 7，k = 4。 

Even_count = 3，因此偶数占据位置 1 到 3：[2, 4, 6]。 由于 k = 4，我们处于奇数段。 

| 步骤| k | 偶数计数 | t | 奇数计数 | 计算|
 | --- | --- | --- | --- | --- | --- |
 | 初始化| 4 | 3 | - | 4 | k > 偶数计数 |
 | 地图 | - | - | 1 | 4 | t = 4 - 3 | t = 4 - 3 |
 | 结果| - | - | 1 | 4 | 最大奇数 = 7，答案 = 7 |

 所以输出是7。 

现在考虑 n = 12，k = 8。 

Even_count = 6，偶数为 [2, 4, 6, 8, 10, 12]。 k = 8 在于赔率。 

| 步骤| k | 偶数计数 | t | 奇数计数 | 计算|
 | --- | --- | --- | --- | --- | --- |
 | 初始化| 8 | 6 | - | 6 | k > 偶数计数 |
 | 地图 | - | - | 2 | 6 | t = 8 - 6 | t = 8 - 6 |
 | 结果| - | - | 2 | 6 | 最大奇数 = 11，答案 = 11 - 2 = 9 |

 这证实了正确映射到反向奇数排序。

 ## 边缘情况

 对于 n = 1，序列仅包含 [1]。 偶数计数为零，因此每个 k 都落入奇数分支。 计算得出 odd_count = 1 且最大奇数 = 1，因此答案为 1，与序列匹配。 

对于 n = 2，序列为 [2, 1]。 Even_count = 1。对于 k = 1，我们返回 2。对于 k = 2，我们进入奇数分支，t = 1，odd_count = 1，最大奇数 = 1，给出 1。这证实了对最小非平凡分割的正确处理。 

对于 n = 7，k = 3（偶数段的边界），even_count = 3，因此我们返回 2 * 3 = 6，它与构造的序列 [2, 4, 6, 7, 5, 3, 1] 匹配。
