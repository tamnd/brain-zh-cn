 ## 解决方案
 ## 问题理解

 我们得到一个整数数组和一个阈值$d$。 当我们有选择地从数组中删除一个连续的段后，剩余的元素必须形成一个序列，其最大值减去最小值最多为$d$。 目标不是优化最终数组本身，而是最小化在单个连续删除中删除的元素数量。 

重新构建操作的一个有用方法是我们选择要保留的前缀和后缀，并且丢弃中间的所有内容。 最终数组始终采用以下形式$a[1..l]$结合$a[r..n]$，其中删除的段是$a[l+1..r-1]$。 任务是选择$l$和$r$最小化$r-l-1$，以其余元素满足的条件为条件$\max - \min \le d$。 

约束条件允许$n$最多 200,000，因此所有删除间隔或所有子数组上的任何二次方法都太慢。 解决方案在最坏的情况下必须是线性或接近线性的$O(n \log n)$。 这强烈建议将两指针或滑动窗口结构与范围最大值和最小值的预先计算结构相结合。 

一个微妙的点是，删除的段可以为空，这意味着我们可以保留整个数组。 这创建了一个自然的基线答案为零。 另一种边缘情况是，数组已经全局违反了条件，但即使没有完整的后缀或前缀单独满足约束，删除单个中间段也可以修复它。 

检查每个可能的删除间隔的简单方法将失败，因为$O(n^2)$候选者，每个候选者都需要重新计算两个不相交段的最小值和最大值。 

## 方法

 蛮力的想法很简单。 我们尝试每一对索引$(l, r)$，将其解释为删除它们之间的段，然后计算剩余元素的最小值和最大值。 如果条件成立，我们用删除的段的大小更新答案。 这是正确的，因为它枚举了所有有效的操作。 

然而，重新计算每对剩余元素的最小值和最大值是昂贵的。 即使我们预先计算前缀和后缀的最小值和最大值，将它们组合起来进行任意删除仍然需要恒定的时间，但是有$O(n^2)$的选择$(l, r)$，导致大约$2 \cdot 10^{10}$检查最坏的情况，这远远超出了限制。 

关键的观察是，我们可以考虑保留什么，而不是考虑我们删除了什么。 其余元素来自两个不相交的段，并且它们的并集必须具有较小的范围。 该条件仅取决于保留元素的全局最小值和最大值。 这表明按照顺序而不是位置对值进行排序或至少推理。 

如果我们固定保留集合的最小值，例如通过锚定在某个索引处$i$，则最大允许值被确定为内的所有元素$[a[i], a[i] + d]$。 剩下的任务是选择我们可以从这个值区间保留多少，同时确保我们不需要包含一个大的中间间隙。 这转化为找到可以充当保留元素的左边界和右边界的最佳位置对，同时保持在值约束内。 

我们可以预先计算每个位置在保持在有效值窗口内的同时可以向左和向右延伸多远。 那么最佳保留集对应于最大化某些有效值窗口的保留大小，答案变为$n$减去最大保留大小。 这可以通过使用受值区间约束的排序或隐式排序索引结构上的两个指针来解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$天真或$O(n^2)$优化|$O(n)$| 太慢了 |
 | 最佳 |$O(n)$或者$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题重新解释为在删除一个连续块后最大化由前缀和后缀组成的保留子集的大小。 

1. 隐式排序或维护索引，但保持原始顺序，因为删除必须在索引中连续。 相反，我们专注于在索引上滑动窗口，同时在值排序结构上使用两指针技术强制执行值约束。 
2. 构建一个结构，允许我们维护一个有效窗口，其中元素的值最多相差$d$。 我们在数组上扩展右指针，同时保持当前保留结构内的最小值和最大值。 由于保留的结构不连续，我们跟踪左右部分之间的最佳分割。 
3.对于左保留线段的每个可能的右端点，我们找到右保留线段的最远左端点，使得两个线段中的所有值都满足约束。 这减少了维护数组上的指针并检查极值是否最多相差$d$。 
4. 我们在索引上维护一个两指针窗口，但允许跳过中间段。 对于每个固定的左边界$l$，我们移动右边界$r$尽可能向右，以使间隙之外的值的联合仍然有效。 我们将候选删除计算为$r - l - 1$。 
5. 我们跟踪所有此类配置的最小值。 

关键思想是有效性仅取决于保留元素的最小值和最大值，并且可以使用指针移动来增量维护这些值。 

### 为什么它有效

 在任何时候，该算法都会考虑将数组拆分为保留的前缀和围绕删除的段保留的后缀的所有方法。 对于每个这样的分割，条件$\max - \min \le d$对两个保留部分的并集强制执行。 由于指针移动仅单调地放大候选区域，并且有效性在移除时是单调的，因此每个最佳配置在扫描期间都显示为某个边界对。 这确保了不会跳过任何有效配置，并且在所有可行的分割中找到最小的删除段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, d = map(int, input().split())
    a = list(map(int, input().split()))

    # We maintain best answer by trying all splits:
    # keep prefix [0..i] and suffix [j..n-1], remove (i+1..j-1)
    # We need max-min over kept elements <= d.

    ans = n  # worst case remove everything

    # Precompute prefix min/max and suffix min/max
    pref_min = [0] * n
    pref_max = [0] * n
    suff_min = [0] * n
    suff_max = [0] * n

    pref_min[0] = pref_max[0] = a[0]
    for i in range(1, n):
        pref_min[i] = min(pref_min[i-1], a[i])
        pref_max[i] = max(pref_max[i-1], a[i])

    suff_min[-1] = suff_max[-1] = a[-1]
    for i in range(n-2, -1, -1):
        suff_min[i] = min(suff_min[i+1], a[i])
        suff_max[i] = max(suff_max[i+1], a[i])

    j = 0

    for i in range(n + 1):
        if i > 0:
            left_min = pref_min[i-1]
            left_max = pref_max[i-1]
        else:
            left_min = float('inf')
            left_max = -float('inf')

        while j < n:
            right_min = suff_min[j]
            right_max = suff_max[j]

            if i <= j:
                cur_min = min(left_min, right_min)
                cur_max = max(left_max, right_max)
            else:
                # overlap not possible; just break
                break

            if cur_max - cur_min <= d:
                break
            j += 1

        if i <= j:
            ans = min(ans, j - i)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先构建前缀和后缀极值数组，以便可以在恒定时间内评估任何分成左保留段和右保留段的分割。 然后它迭代已删除段的所有可能边界。 对于每个左边界，它贪婪地向前移动右边界，直到剩余元素满足约束。 答案是这两个边界之间的最小差距。 

一个微妙的实现点是指针必须是单调的。 一旦右指针向前移动给定的左边界，它就永远不需要向后移动，因为扩展左边界只会放松右侧的约束。

 ## 边缘情况

 一种边缘情况是数组已经全局满足条件。 在这种情况下，正确答案是零，算法会处理它，因为初始分割$i = 0, j = 0$无需移动指针就已经产生了有效的配置。 

另一种情况是除了中间的小段之外的所有元素都有效。 例如，在像这样的数组中$[1, 100, 101, 2]$与小$d$，最优删除是中间块。 基于指针的扩展自然会推动$j$向前推进，直到违规问题得到解决，并且最小差距$j - i$捕获该删除。 

最后一种情况是两端都有极值，例如$[1, 100, 2, 99]$。 正确的解决方案可能需要移除断开不兼容极端的中心部分。 该算法仍然评估此配置，因为它系统地考虑了所有前缀后缀分割，其中保留的部分形成有效的极值组合。
