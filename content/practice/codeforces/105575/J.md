 ## 解决方案
 ## 问题理解

 我们得到一个可能包含小写字母和通配符的字符串。 每个通配符都可以替换为任何小写字母。 选择替换后，字符串被分割成连续的段，并且每个段必须是最终填充字符串中的回文。 

在将字母分配给通配符并分割字符串的所有可能方法中，我们同时需要两件事：回文段的最小数量，以及实现此最小值的不同方法的数量，其中如果填充的字符串或分段不同，则两种方法是不同的。 答案以 998244353 为模计算。 

关键的困难在于回文结构和通配符灵活性相互作用。 子串不是简单的“是否是回文”，而是“是否可以构成回文，以及有多少种方式可以补全”。 

弦的长度足够大，任何三次甚至二次重常数方法都会变得紧密。 任何重新计算回文有效性或对每个分割点独立计数而不重用的操作都会超时。 这促使我们预处理所有回文区间和每个区间的有效填充数，然后运行标准区间动态规划进行分区。 

当忽略通配符多重性时，会出现微妙的失败情况。 例如，考虑子字符串“??”。 它在结构上始终是回文，但它提供 26 个选择，而不是 1 个。仅跟踪可行性的朴素 DP 会计算出正确的最小切割，但计数完全错误。 

另一个边缘情况是在对称位置混合固定字母和通配符。 例如，“a?b”在任何赋值下都不能是回文，即使两端看起来都是灵活的。 当围绕中心扩张时，这种类型的不匹配会迫使提前终止。 

## 方法

 直接方法尝试枚举字符串的每个分区，并针对每个段检查它是否可以转换为回文，然后计算有多少赋值使其有效。 即使用两个指针优化回文检查，分区的数量是指数级的，子串的数量是二次级的，所以这种方法立即变得不可行。 

一个更结构化的想法是将问题分成两个独立的层。 第一层回答每个子串是否可以是回文以及可以有多少种方式补全。 第二层将每个子串视为候选块，并在索引上运行最小分段动态程序。 

关键的观察结果是，回文有效性和赋值计数对于子串来说是局部的，并且可以使用中心扩展来预先计算。 一旦我们确定了一个中心，我们就可以向外扩展并保持一致性约束。 每次扩展匹配对时，我们要么传播现有计数，要么乘以 26（如果两端都是通配符）。 这将子串评估转变为每个中心的摊销线性工作，这对于所有 O(n^2) 子串来说足够了。 

经过这样的预处理，问题就变成了一个标准的“带权重的最小回文划分”DP。 对于每个端点 i，我们认为所有 j < i 使得子串 (j+1, i) 是有效的回文段。 我们更新最佳切割计数并累积路数，乘以该段的填充数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力分区 | 指数| O(n) | 太慢了 |
 | 预计算回文 + DP | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们将解决方案分为三个阶段：预处理回文有效性和计数、用于最佳分割的动态编程以及重建一个有效的填充解决方案。

1. 对于每个索引 i，将其视为回文中心并向外扩展为奇数长度回文。 在从 (j, k) 扩展时，我们检查字符的兼容性。 如果出现冲突，扩展就会停止。 如果字符匹配或涉及通配符，则子字符串仍然可能是回文。 我们将其标记为有效的回文间隔。 

在此扩展过程中，我们还计算了可以完成此间隔的方式。 如果两端都是“？”，则填充数量会增加 26 倍，因为我们可以自由选择匹配的字母对。 如果只有一侧是“？”，则该值将从内部子字符串继承，因为通配符是强制使用的。 这会为每个间隔构建一个乘法计数。 

1. 从相邻中心 (i, i+1) 开始，对偶数长度回文重复相同的展开。 适用相同的有效性和计数规则。 
2. 存储两张表：一张布尔表指示子串是否可以是回文，一张表存储该子串的有效完成次数。 
3. 定义前缀上的 DP。 让 dp[i] 存储一对，其中包含覆盖以 i 结尾的前缀所需的最小回文段数，以及实现该最佳值的方法数。 
4. 用一种方式将 dp[0] 初始化为零段。 
5. 对于每个端点 i，迭代所有可能的先前切割位置 j。 如果子串 (j+1, i) 是有效的回文，我们尝试从 dp[j] 转换到 dp[i]。 
6. 更新 dp[i] 时，我们首先最小化段数。 如果找到严格更好的最小值，我们会覆盖最小值和计数。 如果达到相同的最小值，我们就会增加方式的数量。 每个转换将 dp[j].ways 乘以子串 (j+1, i) 的回文填充数。 
7.维护一个前驱数组来重建一个最优分区。 
8. DP之后，通过从n向后走来重建分区。 对于每个段，通过镜像字符填充通配符。 如果两端都是“?”，则指定一个默认字母，例如“a”。 如果只有一侧是“？”，则从其对称位置复制。 

### 为什么它有效

 每个回文段完全由其端点和一致的镜像约束决定。 扩展阶段将所有可能的有效回文间隔恰好枚举一次，并且附加到每个间隔的计数反映了通配符替换的独立选择，而不受其他段的干扰。 然后，DP 将每个间隔视为一个原子块，并且最佳子结构成立，因为前缀的任何最佳分段都必须以一些有效的回文后缀结尾，并且所有后缀选择都被枚举。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input().strip())
    s = input().strip()
    s = " " + s

    isp = [[0] * (n + 1) for _ in range(n + 1)]
    cnt = [[1] * (n + 1) for _ in range(n + 1)]

    dp = [(10**9, 0) for _ in range(n + 1)]
    pre = [0] * (n + 1)
    dp[0] = (0, 1)

    def relax(i, j):
        nonlocal dp, pre
        cost = dp[j][0] + 1
        ways = dp[j][1] * cnt[j + 1][i] % MOD

        if cost < dp[i][0]:
            dp[i] = (cost, ways)
            pre[i] = j
        elif cost == dp[i][0]:
            dp[i] = (cost, (dp[i][1] + ways) % MOD)

    for i in range(1, n + 1):
        isp[i][i] = 1
        if s[i] == '?':
            cnt[i][i] = 26

        j, k = i - 1, i + 1
        while j >= 1 and k <= n:
            if s[j] == '?' and s[k] == '?':
                cnt[j][k] = cnt[j + 1][k - 1] * 26 % MOD
            elif s[j] == s[k] or s[j] == '?' or s[k] == '?':
                cnt[j][k] = cnt[j + 1][k - 1]
            else:
                break
            isp[j][k] = 1
            j -= 1
            k += 1

    for i in range(1, n):
        if s[i] == s[i + 1] or s[i] == '?' or s[i + 1] == '?':
            if s[i] == '?' and s[i + 1] == '?':
                cnt[i][i + 1] = 26
            isp[i][i + 1] = 1

            j, k = i - 1, i + 2
            while j >= 1 and k <= n:
                if s[j] == '?' and s[k] == '?':
                    cnt[j][k] = cnt[j + 1][k - 1] * 26 % MOD
                elif s[j] == s[k] or s[j] == '?' or s[k] == '?':
                    cnt[j][k] = cnt[j + 1][k - 1]
                else:
                    break
                isp[j][k] = 1
                j -= 1
                k += 1

    for i in range(1, n + 1):
        for j in range(i):
            if isp[j + 1][i]:
                cost = dp[j][0] + 1
                ways = dp[j][1] * cnt[j + 1][i] % MOD
                if cost < dp[i][0]:
                    dp[i] = (cost, ways)
                    pre[i] = j
                elif cost == dp[i][0]:
                    dp[i] = (cost, (dp[i][1] + ways) % MOD)

    p = n
    ans = []
    while p:
        l = pre[p] + 1
        t = list(s[l:p + 1])
        x, y = 0, len(t) - 1
        while x <= y:
            if t[x] == '?' and t[y] == '?':
                t[x] = t[y] = 'a'
            elif t[x] == '?':
                t[x] = t[y]
            elif t[y] == '?':
                t[y] = t[x]
            x += 1
            y -= 1
        ans.append("".join(t))
        p = pre[p]

    ans.reverse()
    print(dp[n][0], dp[n][1])
    print(" ".join(ans))

t = int(input().strip())
for _ in range(t):
    solve()
```代码的第一阶段围绕中心进行扩展，以填充回文有效性和重数表。 第二阶段执行经典的前缀 DP，其中每个有效的回文后缀都被视为转换。 重建步骤使用前驱数组向后移动，并通过强制对称性确定性地将字母分配给通配符。 

一个常见的微妙之处是转换期间计数的乘法。 每个部分独立贡献，因此全局构造的数量在各个部分之间成倍增加。 这就是 dp 转换乘以 cnt[j+1][i] 的原因。 

## 工作示例

 考虑像“a??a”这样的短字符串。 扩展阶段将整个区间标记为有效回文，中间的通配符对引入了乘法选择。 然后，DP 更喜欢单个片段，产生最少 1 次切割和大量填充。 

| 我| 最佳剪辑| 方式| 最后一剪|
 | --- | --- | --- | --- |
 | 0 | 0 | 1 | - |
 | 4 | 1 | 从完整段计算| 0 |

 该轨迹表明，即使存在多个内部填充，算法也能正确地优先考虑较少的段。 

对于第二个例子，考虑“a？b？”。 由于不匹配，子串无法在整个范围内形成回文，因此 DP 被迫分成更小的段。 扩展阶段无法正确标记无效的大间隔，这确保 DP 不会考虑它们。 

| 前缀 | 有效段 | 决定|
 | --- | --- | --- |
 | 1 | “一个”| 开始 |
 | 2 | “一个？” | 有效 |
 | 3 | “b”| 力量削减|

 这表明无效的回文间隔被尽早修剪，从而防止了错误的长程转换。

 ## 边缘情况

 单字符字符串的行为就像一个简单的回文。 DP 正确初始化 dp[0]，因此如果字符为“？”，则 dp[1] 立即成为具有 26 路的一段，否则为 1 路。 重建步骤只是返回角色本身。 

完全通配符字符串，例如“???” 完全是回文的，并且扩展阶段在每个对称配对上乘法传播计数。 DP 应该认识到，最优解是一段，填充数变为 26，提高到对称选择数，通过区间乘法隐式累积。 

强制不匹配的字符串（例如“ab”）不能是回文，因此 isp[1][2] 仍为 false。 DP 被迫分成两个单字符段，这确保了正确性，即使每个单独的字符仍可能具有与通配符相关的多重性。
