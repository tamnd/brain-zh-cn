 ## 解决方案
 ## 问题理解

 我们面临一个日程规划问题。 有几个人，每个人都提供一个或多个他们有空的时间间隔。 目标是确定同时可用人数最多的单个时刻。 找到最佳时刻后，我们还会报告当时有多少人没空。 

更具体地说，时间被离散化为从 1 到 m 的整数位置。 每个人贡献几个闭合间隔，每个间隔表示该人在其内部的所有时间点都可用。 如果一个人至少有一个间隔覆盖某个时间点，则认为该人在某个时间点可用，但在这个问题表述中，每个间隔都被直接视为在其范围内贡献可用性，因此我们有效地计算每次有多少个间隔覆盖重叠。 

输出需要两个东西：可用人数最大化的时间点，以及当时不可用的人数。 

这些约束意味着 n 可能很大，并且每个人可能有多个间隔，因此快速检查每个时间点与每个间隔的简单方法变得太慢。 如果我们假设多达 10^5 个时间点和许多间隔，则直接模拟在最坏的情况下会导致大约 O(n · m) 的行为，这在典型限制内是不可行的。 

边界处理出现了一个更微妙的问题。 间隔是包容性的，因此如果一个间隔以 r 结束，我们必须确保它对时间 r 有贡献，但对 r + 1 没有贡献。忘记这个差一细节的幼稚前缀实现将错误地向前移动计数。 

一个简单的边缘案例清楚地表明了这一点。 假设 m = 5 并且单个区间为 [2, 3]。 在时间 3 时，仍应对该人进行计数。 如果我们错误地在位置 3 而不是位置 4 处递减，我们将在时间 3 处失去覆盖范围并计数不足。 

## 方法

 蛮力的想法很简单：对于从 1 到 m 的每个时间点，检查每个人的每个间隔并计算有多少个间隔包含该时间。 这是正确的，因为它直接模拟了可用性的定义。 但是，如果最多有 n 个人，每个人都有 p 个间隔和 m 个时间点，则这将变为 O(m·total_intervals)，在最坏的情况下可能会降低到大约 10^10 次操作。 

关键的观察结果是，我们实际上不需要为每个时间点独立地重新计算覆盖范围。 每个间隔贡献一个+1覆盖范围的连续块。 我们可以使用差异数组对其效果进行编码，而不是在间隔内的每次都重复应用此贡献：在 l 处递增，在 r + 1 处递减。处理完所有间隔后，前缀和会重建每次的活动间隔数量。 

这将问题从重复添加范围转变为单个线性扫描。 一旦我们获得了每个时间点的覆盖范围，我们就可以扫描一次以找到最大位置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(m·总间隔) | O(1) | O(1) | 太慢了 |
 | 差值数组 + 前缀和 | O(m + 总间隔) | O(米) | 已接受 |

 ## 算法演练

 我们在时间轴上使用差异数组。 

1. 创建数组`diff`大小 m + 2 初始化为零。 该数组将存储间隔贡献如何开始和停止。 我们使用 m + 2 来安全地处理 r + 1 索引，而不会出现边界问题。 
2. 对于每个人，读取他们的时间间隔。 对于每个区间 [l, r]，增量`diff[l]`加 1 并减`diff[r + 1]`1。这表示间隔从 l 开始贡献 +1，并在 r 之后停止贡献。 
3. 处理完所有区间后，计算前缀和`diff`获得`cnt[i]`，覆盖时间 i 的间隔数。 每个位置都会累积包含该位置的所有活跃间隔贡献。 
4、遍历从1到m的所有时间点，跟踪其中的索引`cnt[i]`被最大化。 这表示间隔重叠最多的时间。 
5. 输出最佳时间和当时不可用的人数，计算公式为 n 减去最大覆盖值。 

### 为什么它有效

 每个区间 [l, r] 可以被视为向连续段中的每个位置添加 +1。 差异数组在每个时间间隔对该段更新进行一次编码，并且前缀和重建每个点的精确累积值。 由于加法是线性的并且跨区间独立，因此最终的前缀和相当于显式应用所有区间贡献。 因此，最大重建值是真正的全局最大重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    diff = [0] * (m + 2)

    for _ in range(n):
        p = int(input())
        for _ in range(p):
            l, r = map(int, input().split())
            diff[l] += 1
            if r + 1 <= m:
                diff[r + 1] -= 1

    best_time = 1
    best_val = -1
    cur = 0

    for i in range(1, m + 1):
        cur += diff[i]
        if cur > best_val:
            best_val = cur
            best_time = i

    print(best_time, n - best_val)

if __name__ == "__main__":
    solve()
```代码直接反映了差异数组的思想。 这`diff`数组仅记录区间的边界效应。 变量`cur`维护运行前缀总和，即每次活动间隔的实际数量。 每当发现更高的重叠时，比较步骤就会更新最佳时间。 最后的减法`n - best_val`将最大参与人数转换为缺席人数。 

一个微妙的实现细节是条件`if r + 1 <= m`。 由于我们只关心 m 以内的时间，因此任何超出该范围的递减都是无关紧要的，但保持数组安全可以避免意外的越界写入。

 ## 边缘情况

 一种重要的边缘情况是所有间隔恰好在 m 处结束。 在这种情况下，r + 1 处的减量不得影响有效索引。 当 r + 1 超过 m 时，代码会安全地忽略它，因此最终的前缀仍然反映完全覆盖，直到最后。 

另一种情况是间隔仅在端点处接触。 例如，[1,2] 和 [2,3] 必须在时间 2 都做出贡献。差异数组在 i = 2 时正确保持两者处于活动状态，因为第一个间隔的递减发生在 3，而不是 2。这完全按照要求保留了边界处的重叠。 

最后一种边缘情况是每个区间都不相交。 该算法仍将计算正确的计数，因为每个段独立贡献并且从不干扰其他段，并且在任何覆盖点处最大值自然变为 1。
