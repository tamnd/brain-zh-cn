 ## 问题理解

 我们得到一个图结构，在同一组顶点上具有有向约束和无向关系。 有向边代表严格的排序规则，不能违反，而无向边则比较灵活，必须在最终结果中指定一个方向。 

任务是决定是否可以定向所有无向边，以便整个图变成非循环，同时保留给定的有向边。 如果可能的话，我们必须为每个无向边输出一致的方向； 否则我们输出不存在有效的构造。 

关键的困难在于有向边已经施加了偏序约束。 如果这些约束包含循环，则向无向边分配方向无法解决矛盾，因为有向边是不可变的。 如果有向部分是非循环的，则无向边的方向始终可以与从该非循环结构导出的全局排序一致。 

这些约束意味着图最多有 200000 个顶点和边。 这迫使任何解决方案都以线性或接近线性的时间运行。 边或顶点上的任何二次行为都会立即失败。 这排除了尝试通过重复模拟边缘方向或尝试回溯分配来测试一致性的方法。 

当有向边形成循环时，即使存在看似能够直观地“破坏”循环的无向边，也会出现微妙的失败情况。 例如，如果 1 → 2、2 → 3、3 → 1 都是有向边，那么即使有很多无向边，答案也是不可能的。 尝试首先定向无向边的天真的方法可能会错误地认为它可以避免循环，但有向循环已经是致命的。 

另一种边缘情况是根本没有有向边。 在这种情况下，任何顶点排序都有效，因此我们始终可以为无向边生成有效的方向。 

## 方法

 蛮力的想法是将每个无向边视为二元选择并尝试所有可能的方向。 分配方向后，我们将使用拓扑排序或 DFS 循环检测来检查生成的有向图是否是非循环的。 由于在最坏的情况下可能存在多达 O(m2) 个无向边，因此这种方法会爆炸为 2^m 个配置，甚至单个有效性检查的成本也为 O(n + m)，使其完全不可行。 

关键的观察是有向边定义偏序。 如果这个偏序是一致的，意味着它没有循环，那么它可以扩展到所有顶点的完整线性顺序。 一旦我们有了这样的线性顺序，每个无向边就可以简单地按照该顺序从早到晚定向。 这会自动保证不会引入循环，因为每条边都遵循单一的全局排序。 

因此，整个问题简化为检查有向子图是否是 DAG，如果是，则构建拓扑排序。 一旦存在这种顺序，所有无向边的定向就变得微不足道了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^m · (n + m)) | O(2^m · (n + m)) | O(n + m) | 太慢了 |
 | 最优（拓扑排序）| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

1. 仅使用有向边构建有向图。 计算所有顶点的入度。 这隔离了强制性的约束，并且无论如何处理无向边都必须满足这些约束。 
2. 使用队列运行标准卡恩算法进行拓扑排序。 从入度为零的所有顶点开始。 这些顶点没有先决条件，并且可以在任何有效排序中最早出现。 
3. 反复从队列中删除一个顶点，将其分配到全局排序中的下一个位置，并减少其邻居的入度。 每当邻居的入度变为零时，将其推入队列。 
4.如果最后处理的顶点少于n个，则有向图中存在环。 在这种情况下，不存在有效的方向，因为定向约束本质上是矛盾的。 
5. 如果获得完整排序，则使用它来为所有无向边指定方向。 对于每条无向边 (u, v)，比较它们在拓扑顺序中的位置，并将边从较早的顶点指向较晚的顶点。 

### 为什么它有效

 拓扑顺序定义了与所有有向边一致的严格总排序。 每个有向边 u → v 都保证 u 按此顺序出现在 v 之前。 根据此顺序定向的任何边永远不能创建有向循环，因为后续边总是严格增加顺序中的位置。 由于每个无向边都被迫遵循这个单调规则，因此最终的图直接从线性顺序继承了非循环性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m1, m2 = map(int, input().split())

    undirected = []
    adj = [[] for _ in range(n + 1)]
    indeg = [0] * (n + 1)

    # undirected edges
    for _ in range(m1):
        u, v = map(int, input().split())
        undirected.append((u, v))

    # directed edges
    for _ in range(m2):
        u, v = map(int, input().split())
        adj[u].append(v)
        indeg[v] += 1

    from collections import deque
    q = deque([i for i in range(1, n + 1) if indeg[i] == 0])

    topo = []
    while q:
        u = q.popleft()
        topo.append(u)
        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)

    if len(topo) != n:
        print(-1)
        return

    pos = [0] * (n + 1)
    for i, v in enumerate(topo):
        pos[v] = i

    for u, v in undirected:
        if pos[u] < pos[v]:
            print(u, v)
        else:
            print(v, u)

t = 1
for _ in range(t):
    solve()
```该解决方案立即分离无向边和有向边，因为只有有向边会影响可行性。 邻接表和入度数组纯粹用于卡恩算法。 队列最初包含没有传入有向边的所有顶点，确保我们始终扩展有效的偏序。 

关键的一步是验证拓扑排序是否覆盖所有顶点。 如果不是，则定向约束已经相互矛盾，因此进一步的处理是没有意义的。 

计算出顺序后，我们将每个顶点映射到其位置。 这允许对每个无向边进行恒定时间比较。 每个无向边都严格按照全局顺序定向，保证一致性。 

## 工作示例

 考虑一个具有小型有向结构的情况：

 输入：

 n = 4

 有向边：1 → 2, 2 → 3

 无向边： (1, 3), (4, 2)

 拓扑过程：

 | 步骤| 队列| 弹出| 拓扑| 入度变化 |
 | --- | --- | --- | --- | --- |
 | 1 | [1,4]| 1 | [1] | 2→0 |
 | 2 | [4,2]| 4 | [1,4]| 无 |
 | 3 | [2] | 2 | [1,4,2]| 3→0 |
 | 4 | [3] | 3 | [1,4,2,3] | 完成 |

 最终顺序是[1,4,2,3]。 现在定向无向边：

 边(1,3)：1在3之前，变成1→3

 边(4,2)：4在2之前，变成4→2

 这表明，一旦存在拓扑顺序，所有无向边都会与其一致。 

现在考虑一个失败案例：

 输入：

 n = 3

 有向边：1 → 2、2 → 3、3 → 1

 无向边：无

 队列立即变空或处理的节点数少于 3 个。 该算法检测到并非所有节点都包含在拓扑中，因此输出为-1。 这表明有向循环无法修复。

 ## 边缘情况

 关键的边缘情况是根本没有有向边缘。 在这种情况下，每个顶点最初的入度为零，因此队列从所有节点开始。 任何拓扑排序都是有效的，并且算法会立即生成完整的排列。 然后，所有无向边根据该排列任意定向，该排列始终有效。 

另一种边缘情况是有向边形成一条链加上孤立的节点。 拓扑顺序会将孤立节点放置在相对于链的任意位置，但这不会影响正确性，因为孤立节点没有施加任何约束。 

故障边缘情况是隐藏在许多节点之间的有向循环。 例如，如果在较大的图中只有三个节点形成一个循环，那么卡恩的算法将提前停止。 最终的拓扑长度检查可以检测到这一点，而无需显式搜索循环。
