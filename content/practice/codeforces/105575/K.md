 ## 问题理解

 我们给出了几个二维整数向量。 每个向量代表一个允许您使用的步骤，并且您可以采用这些向量的整数线性组合，这意味着您可以将它们添加或减去任意次数，并按整数缩放它们。 问题是，使用这些操作是否可以构建与两个坐标中的单位基础状态等效的目标配置，这实际上意味着生成的整数晶格在强意义上跨越了整个网格。 

另一种表达目标的方式是，我们想知道给定向量的集合是否可以生成一对向量，其行为类似于整数网格的基础，以便两个坐标可以通过允许的整数组合独立地减少到最小非零形式。 

关键的困难是我们不是在一个领域工作。 对于实数或有理数，高斯消元法可以直接解决这个问题，因为除法总是可能的。 对于整数，我们不能自由除法，因此我们必须依赖 gcd 式的约简来保留整数结构。 

这些约束表明最多有几千个向量，因为预期的解决方案在对之间重复执行类似欧几里德的归约。 这排除了任何尝试枚举所有线性组合或显式探索格的方法。 任何指数级的或涉及大型状态搜索的事情都是立即不可能的。 

当一个坐标过早变为零时，就会出现一种微妙的边缘情况。 例如，如果所有向量的第一个坐标为零，我们永远无法构建具有非零第一个坐标的任何东西，因此答案必须立即是否定的。 在坐标解耦且无法组合成共享晶格基础的情况下，仅独立检查每个坐标的全局 gcd 的简单实现会错误地回答“是”。 

当在每个坐标上独立执行归约而不考虑通过相同的交换序列和欧几里得归约耦合操作时，会出现另一种故障模式。 这种耦合可以保持正确性。 

## 方法

 强力解释将尝试模拟向量的所有可能的整数线性组合。 每个操作都允许添加或减去向量，因此状态空间是二维整数格。 即使我们限制系数，可达状态的数量也会呈爆炸性增长。 对于系数在中等范围内的 m 个向量，例如 [-K, K]，组合的数量为 (2K+1)^m，即使对于较小的 m，这也是不可行的。 

关键的见解是我们实际上并不需要构建所有组合。 相反，我们只需要知道这些向量生成的格子是否包含两个坐标绝对值都等于1的向量，这相当于检查整数跨度是否形成完整的Z^2格子。 

这将问题简化为计算系统的类似 gcd 的不变量。 在一维中，经典结果是所有数字的整数组合都恰好生成其 gcd 倍数的集合。 在二维中，结构更加微妙，但我们仍然可以通过在向量分量之间重复应用欧几里得式消除来减少问题。 

该算法有效地模拟了二维扩展 gcd 过程：它减少一个向量相对于另一个向量，同时保留它们生成的整数格，逐渐将系统折叠成仅保留 gcd 信息的规范形式。 最后，我们检查在此变换下两个坐标方向是否独立地使 gcd 等于 1。

 ## 算法演练

 我们维护一对工作值，代表每个坐标方向上晶格的演化基础。 我们使用欧几里德归约将每个输入向量重复合并到该结构中。 

1. 首先选择第一个向量作为当前参考对 (x, y)。 这就像我们试图针对所有其他生成器进行简化的电流生成器。 
2. 对于每个剩余向量 (a[i][0], a[i][1])，确保其第一个坐标不为零。 如果它为零，而参考仍然具有非零的第一坐标，我们将其与参考交换。 这确保我们总是减少一个实际上可以影响第一维当前 gcd 过程的向量。 
3. 对第一个坐标应用欧几里得步骤。 我们计算当前向量的第一个坐标适合 x 的次数，从两个向量中减去该倍数，然后交换角色。 这正是余数除法的整数版本，但扩展到成对值，以便第二个坐标跟踪相同的线性变换。 
4. 重复此减少过程，直到工作矢量的第一个坐标变为零。 此时，x的剩余值是迄今为止处理的所有第一坐标的gcd，y是第二坐标空间中对应的累加线性组合。 
5. 在已远离主第一坐标链减少的向量的第二坐标上独立维护 gcd 累加器 fy 。 这捕获了一旦第一维度固定后第二维度中剩余多少自由度。 
6、处理完所有向量后，检查最终减少的第一坐标值和累加的第二坐标gcd的绝对值是否都等于1。 如果两者都是 1，我们可以在两个方向上生成完整的单位步长，这意味着晶格跨越完整的整数网格。 

正确性来自于每个归约步骤都保留由向量集生成的整数格的不变量。 每个欧几里得减法都是整数的可逆变换，这意味着我们只改变基数，而不改变跨度。 该过程本质上是计算每个坐标方向上晶格的规范形式。 如果任一坐标折叠为大于 1 的 gcd，则该坐标从根本上受到限制，无法达到 1，因此答案必定是否定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def gcd(x, y):
    return x if y == 0 else gcd(y, x % y)

def solve():
    T = int(input())
    for _ in range(T):
        m = int(input())
        a = [None] + [list(map(int, input().split())) for _ in range(m)]

        for i in range(2, m + 1):
            if a[i][0] != 0:
                a[1], a[i] = a[i], a[1]

            if a[1][0] == 0:
                print("NO")
                break

            x, y = a[1][0], a[1][1]
            fy = 0

            for i in range(2, m + 1):
                if a[i][0] == 0:
                    fy = gcd(fy, a[i][1])
                    continue

                while a[i][0] != 0:
                    length = x // a[i][0]
                    nx = a[i][0]
                    ny = a[i][1]

                    a[i][0] = x - a[i][0] * length
                    a[i][1] = y - a[i][1] * length

                    x, y = nx, ny

            else:
                if abs(fy) == 1 and abs(x) == 1:
                    print("YES")
                else:
                    print("NO")

solve()
```该代码反映了上述欧几里德归约。 主要结构保留一个代表向量 (x, y) 并针对它重复减少其他向量，直到第一个坐标稳定为 gcd 形式。 变量 fy 累积不再参与第一坐标缩减的向量的 gcd 信息。 

关键的微妙之处在于每次减法同时更新两个坐标。 这就是保留晶格结构的原因。 如果我们只减少第一个坐标，我们将丢失有关第二个坐标如何依赖于相同整数运算的信息，并且最终条件将毫无意义。 

另一个微妙的点是每次迭代开始时的交换，这确保我们在第一个坐标中始终有一个非零主元。 没有它，欧几里得步骤将会失败，因为除以零会发生，更重要的是，我们将失去继续 gcd 约简的能力。 

## 工作示例

 考虑一个具有向量 (6, 10)、(4, 6)、(2, 3) 的小实例。 

| 步骤| 枢轴 (x, y) | 当前矢量| 运营|
 | --- | --- | --- | --- |
 | 开始| (6, 10) | (4, 6) | 初始枢轴|
 | 减少| (6, 10) | (4, 6) | 6 mod 4 给出了新的支点 |
 | 交换 | (4, 6) | (2, 3) | 继续减持|

 经过反复归约后，第一个坐标崩溃到 gcd 2，这意味着我们无法达到 1。 

这表明，即使所有向量看起来都“丰富”，它们生成的晶格仍然可能受到一个坐标中的公约数的约束。 

现在考虑向量 (3, 5), (2, 7)。 

| 步骤| 枢轴 (x, y) | 矢量| 结果 |
 | --- | --- | --- | --- |
 | 开始| (3, 5) | (2, 7) | 枢轴|
 | 减少| (2, 7) | (1, -2) | (1, -2) | 欧几里得步骤|
 | 决赛| (1, -2) | (1, -2) | - | gcd 变为 1 |

 这里两个坐标都可以减少到1，所以格子是满秩的，答案是YES。 

这些痕迹显示了算法如何有效地执行整数高斯消除、折叠结构直到仅保留 gcd 信息。

 ## 边缘情况

 一个重要的边缘情况是所有向量的第一个坐标为零。 在这种情况下，算法立即检测到无法形成枢轴。 对于像 (0, 3), (0, 7) 这样的输入，无法影响第一维，因此晶格是一维的，无法在两个坐标中产生单位向量。 

另一种情况是一个向量在大小上支配所有其他向量。 欧几里得约简将很快将所有内容折叠到该向量的 gcd 结构中。 例如，对于 (100, 1), (50, 3)，第一个坐标 gcd 变为 50，并且由于它不是 1，因此算法正确地拒绝了形成单位步的可能性。 

最后一个微妙的情况是当第一坐标小和大的向量之间交替进行归约时。 基于交换的欧几里得循环确保即使在这种振荡情况下，该过程仍然收敛，因为每次迭代都会严格减少至少一个活动坐标对的绝对值。
