 ## 解决方案
 ## 问题理解

 我们有一行$n$孔标记为从 1 到$n$。 每个洞都包含一个鼹鼠，并且每个鼹鼠在一场持续的游戏中只出现一次$n$秒。 出现顺序是随机的：每一秒，都会随机选择一个剩余的看不见的鼹鼠，并且仅在那一秒内变得活跃。 

比赛开始前，两个机器人被放置在这条线上。 一个从第 1 洞开始，另一个从第 1 洞开始$n$。 在每一秒内，两个机器人可以同时移动，并且每个机器人最多可以移动$v$在那一秒内沿线的位置。 移动后，如果机器人站在鼹鼠洞上，它可以立即击打鼹鼠。 每颗鼹鼠必须在它出现的那一秒被准确地击打，并且每颗鼹鼠必须由两个机器人中的一个恰好处理。 

任务是计算在鼹鼠出现的随机顺序下，机器人在给定最佳移动和鼹鼠到机器人的最佳分配的情况下成功捕获每只鼹鼠的概率。 

关键点是随机性取决于$n$空洞，因为“从剩余摩尔中均匀地生成”相当于生成随机排列。 

约束条件$n \le 8$立即发出信号，任何解决方案呈指数增长$n$是可行的。 大小的状态空间$2^n$甚至$2^n \cdot n^2$是可以接受的，特别是对于多项式的过渡$n$。 被排除的是显式迭代所有排列的方法，因为这已经是$n!$， 虽然$n!$是小$n=8$，但我们仍然需要对所有结果进行概率聚合，而不是枚举。 

如果假设将鼹鼠贪婪地分配给最近的机器人，就会出现微妙的失败情况。 考虑一条短线，两个机器人都可以到达鼹鼠，但选择“局部最近”的分配会阻碍未来所需的可达性。 另一个失败案例是假设步骤之间是独立的，因为机器人位置的演变并以耦合方式限制未来的可达性。 

## 方法

 暴力视图从定义开始：枚举鼹鼠出现的所有可能的排列，模拟游戏，并检查机器人是否可以按顺序捕获所有鼹鼠。 对于每个排列，我们一步一步地进行模拟，并在每一步决定机器人 1 还是机器人 2 应该采用当前的鼹鼠，检查可达性约束。 这是正确的，但它需要迭代所有$n!$排列和做$O(n)$模拟每个排列，这导致$O(n \cdot n!)$工作。 虽然$n=8$使这个边界变得可行，但它是不必要的并且在概念上是浪费的。 

关键的观察是随机性不需要显式的排列。 在每一步中，下一个痣都是在剩余痣中统一选择的，因此我们可以按照已出现痣子集的概率 DP 顺序处理该过程。 我们不是枚举阶数，而是累积所有部分状态的概率质量。 

重要的结构是，任何时刻的系统状态都由三件事来完整描述：哪些鼹鼠已经出现，以及两个机器人的当前位置。 从该状态开始，下一个事件是在剩余的鼹鼠中进行统一选择，并且如果移动约束允许，则将鼹鼠分配给两个机器人之一，并且转换是确定性的。 

这将问题转换为子集上的分层动态程序，其中每一层对应于处理的摩尔数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n \cdot n!)$|$O(n)$| 太慢了|
 | 状态上的 DP 子集 |$O(2^n \cdot n^2)$|$O(2^n \cdot n^2)$| 已接受 |

 ## 算法演练

 我们定义了一个动态编程表，其中状态编码了我们在显示的鼹鼠序列中的距离、哪个子集已经出现以及两个机器人当前站立的位置。 

我们将“时间步长”解释为迄今为止发现的痣的数量。 时$t$， 确切地$t$痣出现了。 

### 步骤

 1. 用位掩码表示状态`mask`指示哪些鼹鼠已经出现，以及机器人的位置`(a, b)`。 
2. 初始化 DP`dp[0][1][n] = 1`，表示还没有出现鼹鼠，机器人1在位置1，机器人2在位置$n$，概率质量为 1。 
3. 对于每个州`(mask, a, b)`和$k = \text{popcount(mask)}$，考虑所有不在`mask`。 每一个这样的痣都有可能成为下一颗痣，概率$1/(n-k)$。 
4. 对于每个候选者的下一个鼹鼠位置$x$，如果机器人 1 可以在一秒内到达，我们尝试将其分配给机器人 1，这意味着$|a - x| \le v$，或机器人 2，如果$|b - x| \le v$。 
5. 如果机器人 1 拿走了鼹鼠，我们会过渡到机器人 1 移动到的新状态$x$机器人2停留在$b$。 如果机器人 2 拿走，机器人 2 移动到$x$。 
6. 我们将当前状态的概率均匀地分布到所有有效的下一个摩尔上，并将其累积到后续状态中。 
7.全部处理完毕后$n$步骤，我们对所有状态的概率求和`mask`包括所有痣。 

### 为什么它有效

 不变的是`dp[mask][a][b]`存储恰好产生该集合的所有有效随机历史的总概率`mask`作为一组显示的鼹鼠并将机器人留在位置`(a, b)`服务完所有的人后。 每个转换都保留了正确性，因为每个下一个摩尔都是从剩余的集合中统一选择的，并且该摩尔的每个可行分配都被恰好探索一次。 由于机器人的决策不会影响未来鼹鼠选择的随机性，DP 干净地将概率分布与可行性检查分开。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, v = map(int, input().split())

    size = 1 << n
    dp = [[0.0] * (n + 1) for _ in range(size)]
    dp[0][0] = [0.0] * (n + 1)

    dp = {}
    dp[(0, 1, n)] = 1.0

    for mask in range(size):
        k = bin(mask).count("1")
        rem = n - k
        if rem == 0:
            continue

        nxt = {}
        for (m, a, b), prob in dp.items():
            if m != mask:
                continue

            if prob == 0:
                continue

            base = prob / rem

            for i in range(n):
                if mask >> i & 1:
                    continue
                x = i + 1

                if abs(a - x) <= v:
                    key = (mask | (1 << i), x, b)
                    nxt[key] = nxt.get(key, 0.0) + base

                if abs(b - x) <= v:
                    key = (mask | (1 << i), a, x)
                    nxt[key] = nxt.get(key, 0.0) + base

        dp = nxt

    full = (1 << n) - 1
    ans = 0.0
    for (m, a, b), prob in dp.items():
        if m == full:
            ans += prob

    print(f"{ans:.15f}")

if __name__ == "__main__":
    solve()
```该实现维护活动 DP 状态的字典，而不是完整的 3D 数组，因为许多状态无法访问。 每次迭代对应于固定的子集大小，概率质量均匀地分布在剩余的摩尔上。 将鼹鼠分配给机器人时，会在本地强制执行运动约束。 

一个常见的微妙之处是除以每个状态下剩余摩尔的数量。 这是至关重要的，因为排列是通过重复均匀采样而无需放回构建的，并且未能在每个步骤进行归一化会破坏概率模型。 

另一个重要的细节是，两个作业选择都是独立考虑的； 两个机器人都可以到达鼹鼠，并且必须添加两个分支，因为它们对应于不同的有效历史。 

## 工作示例

 ### 示例 1：n = 4，v = 1

 我们跟踪一小部分进展。 初始状态是`(mask=0000, a=1, b=4)`概率为 1。 

第一步，4颗痣中的任意一颗都有概率出现$1/4$。 只能分配距离机器人位置 1 以内的位置。 

| 步骤| 面膜| （一，二）| 剩余| 过渡说明|
 | ---| ---| ---| ---| ---|
 | 0 | 0000 | 0000 (1,4) | 4 | 开始 |
 | 1 | 0001| (1,4) 或 (2,4) | 3 | 仅摩尔 1 或 2 适合机器人 1 侧 |
 | 2 | ... | ... | 2 | 国家分裂进一步|
 | 4 | 1111 | 1111 有效的最终状态 | 0 | 概率总和 |

 该迹线显示了可行性如何限制转换，而概率在每一层均匀分裂。 

### 示例 2：n = 5，v = 1

 这是一个更严格的情况，其中可达性约束迫使更多分支。 

| 步骤| 关键状态| 观察|
 | ---| ---| ---|
 | 0 | (00000, 1,5) | 开始 |
 | 1 | 多个面具 | 仅可到达附近的端点 |
 | 3 | 混合状态| 机器人位置出现分歧|
 | 5 | 全面罩| 只有有效的历史才能做出贡献|

 这证实了 DP 正确地只累积了全局有效的调度，而不是局部贪婪的调度。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(2^n \cdot n^2)$| 每个子集最多处理$n$下一个选择和商店$O(n^2)$机器人位置|
 | 空间|$O(2^n \cdot n^2)$| DP 状态关于掩码和两个位置 |

 界限$n \le 8$使$2^n \cdot n^2 \approx 256 \cdot 64$，即使在 Python 中，它也足够小，可以达到 5 秒的限制。 常数因子很低，因为转换是简单的算术和字典更新。

 ## 边缘情况

 当$n = 1$，只有一颗痣，而且立刻就出现了。 DP从完全成功状态开始，因为位置1的机器人总能抓住它，所以答案是1。 

当$v$相对于$n$，每一个鼹鼠都可以通过一次移动从任一机器人位置到达，因此 DP 转换永远不会阻塞，并且所有排列都提供有效路径。 该算法自然会累积总概率 1，因为每个分支仍然可行。 

什么时候$v = 0$，机器人永远不会移动。 只有已经位于位置 1 的痣和$n$可以正确分配。 DP 立即过滤掉所有其他分配，因为可达性在转换步骤中失败，仅留下所需摩尔在正确时间准确出现在端点的排列的概率，这是极其受限的。
