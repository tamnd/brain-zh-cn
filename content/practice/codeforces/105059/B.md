 ## 解决方案
 ## 问题理解

 我们有一条街道，其停靠点标记为 1 到 n，我们想要从停靠点 1 移动到停靠点 n。 仅当通过公交车路线进行时，移动才是免费的，否则将按照沿线步行的距离付费。 

每条公交线路都是一个区间[L,R]。 当您在公交车路线上时，您可以在该区间内的任何站点进出，更重要的是，您可以以零步行成本在该区间内的任何站点之间自由移动。 关键是，一旦您进入巴士路线，在其覆盖的路段内行驶就可以沿着该区间“自由移动”。 

在公交车外，您只需沿着线路步行，从 x 步行到 y 的成本为 |x − y|。 目标是尽量减少从 1 到 n 所需的步行时间，使用公交车间隔作为自由出行区域。 

这里的结构本质上是一条具有零成本路段（公交车覆盖区域）的线路，只有当您走出这些区域时，才会产生正成本移动。 

约束很大，每个测试用例最多有 2 · 10^5 个停靠点和 2 · 10^5 条路线，以及最多 10^4 个测试用例。 这立即排除了任何尝试沿街道逐步模拟运动或明确在停靠点上构建图表的解决方案。 任何方法都必须将问题简化为每个测试用例的排序和线性处理或类似的问题。 

一个微妙的失败案例来自于独立处理每条总线。 例如，如果一辆公交车覆盖 [1, 2]，另一辆公交车覆盖 [2, 3]，那么即使两者都没有单独跨越整个范围，它们一起也允许从 1 到 3 自由行驶。不合并重叠或链式连接间隔的简单方法将错误地收取 2 和 2 之间的步行成本，或者错过这些路线形成单个连接的零成本区域。 

另一种边缘情况是 1 或 n 不被任何区间覆盖。 在这种情况下，我们必须明确步行到最近的可到达的公交车区域，或者如果根本没有公交车帮助，则直接前往目的地。 任何假设我们总是在总线区域内开始的方法都会在这里失败。 

## 方法

 思考这个问题的强力方法是将每个停靠点建模为图中的一个节点。 我们在同一公交车间隔内的所有对站点之间添加权重 0 的边，并在代表步行的连续站点 i 和 i+1 之间添加权重 1 的边。 然后我们运行从 1 到 n 的最短路径算法。 

这是正确的，但完全不可行。 如果完全展开，每个长度 L 的区间都会在最坏的解释中添加 O(L^2) 条零权重边，并且即使经过优化，我们最终仍然会得到一个超过 2·10^5 个节点的图，并且可能会重叠密集连接。 由于区间连通性的隐式结构，像 Dijkstra 这样的最短路径算法会太慢。 

关键的观察是总线间隔不会创建任意连接，它们仅合并一条线上的连续区域。 如果两个区间重叠或接触，那么它们联合中的任何停靠点都可以到达任何其他停靠点，而无需步行。 这意味着我们根本不需要单个停靠点的图表。 我们只需要了解区间如何合并成数轴上的最大连通线段。 

一旦我们将所有重叠的间隔压缩成不相交的合并段，问题就变成纯粹是在这些段之间行走。 在每个细分市场内，成本为零。 在各段之间，前进的唯一方法是走过一个段的末尾和下一个段的开始之间的间隙。 

然后，我们按照从左到右的顺序处理路段，仅当我们需要从一个路段移动到下一个路段且不重叠时才累积行走成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力图+最短路径| O(n + k log n + 重边扩展) | O(n + k) | 太慢了|
 | 合并间隔+线性扫描| O(k log k) | O(k log k) | O(k) | 已接受 |

## 算法演练

 我们将问题转化为区间合并，然后沿数轴进行贪心遍历。 

1. 按起点对所有公交车间隔进行排序。 这确保我们按照街道从左到右的自然顺序处理它们。 
2. 将重叠或接触的间隔合并为最大分段。 我们维护当前段 [curL, curR]。 对于每个区间 [L, R]，如果 L ≤ curR，我们扩展 curR = max(curR, R)，否则我们完成当前段并开始一个新段。 这一步骤是合理的，因为任何重叠都会在联盟内部创建完全的零成本连接。 
3. 合并后，我们获得代表所有可以自由移动的位置的不相交线段。 
4. 现在模拟从位置 1 移动到位置 n。 我们保持一个从 1 开始的指针 pos 并按升序扫描段。 
5. 对于每个段[L, R]，如果 pos 已经在段内部（L ≤ pos ≤ R），我们可以免费跳转到 R 并继续。 
6. 如果 pos 位于线段的左侧（pos < L），我们必须从 pos 走到 L，因此我们将 L − pos 添加到答案中，然后“进入”线段，之后我们可以自由移动到 R。 
7. 如果 pos 超出了一个段（pos > R），我们忽略它，因为它不能帮助我们前进。 
8. 继续，直到到达或经过 n，此时停止。 

关键思想是，我们只在穿越互不相连的自由区域之间的间隙时才支付步行成本。 

### 为什么它有效

 合并的段代表每个点通过重叠总线间隔连接的最大区域。 在这样的区域内，任何运动都可以通过链接公交车来模拟，因此成本为零。 

在两个连续的合并路段之间，不存在弥合间隙的间隔重叠，这意味着任何公交车行程都不能在不超出公交车覆盖范围的情况下跨越该间隙。 因此，线段之间的任何过渡都必然需要步行穿过线路的未覆盖部分，并且恰好步行到该间隙的贪婪策略是最佳的，因为任何绕行只会增加线路度量上的距离。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        segs = []
        
        for _ in range(k):
            l, r = map(int, input().split())
            segs.append((l, r))
        
        if k == 0:
            print(n - 1)
            continue
        
        segs.sort()
        
        merged = []
        cur_l, cur_r = segs[0]
        
        for l, r in segs[1:]:
            if l <= cur_r:
                cur_r = max(cur_r, r)
            else:
                merged.append((cur_l, cur_r))
                cur_l, cur_r = l, r
        merged.append((cur_l, cur_r))
        
        pos = 1
        ans = 0
        
        for l, r in merged:
            if pos > n:
                break
            if r < pos:
                continue
            if l <= pos <= r:
                pos = r
                continue
            if pos < l:
                ans += l - pos
                pos = r
        
        if pos < n:
            ans += n - pos
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对所有间隔进行排序，以便可以在一次传递中合并重叠区域。 合并步骤构建线路的最大零成本组件。 

之后，遍历模拟从左到右的移动。 变量`pos`始终代表我们免费或支付累积步行费用后可以到达的最远点。 什么时候`pos`在合并段内，我们将其扩展到段的右端点，无需任何成本。 什么时候`pos`在一个段之前，我们支付恰好的间隙来进入它。 

一个微妙的点是我们总是跳跃`pos`直接到`r`当在一个路段内时，因为在一个连接的区间内，所有中间站点都可以免费到达，所以没有理由提前停止。 

## 工作示例

 ### 示例 1

 输入：```
n = 10, k = 2
[2, 4]
[7, 9]
```合并段：

 | 步骤| 当前部分 | 行动| 邮政 | 成本|
 | ---| ---| ---| ---| ---|
 | 初始化| - | 开始 | 1 | 0 |
 | 1 | [2,4]| 步行 1→2 | 2 | 1 |
 | 1 | [2,4]| 扩展到 4 | 4 | 1 |
 | 2 | [7,9]| 步行 4→7 | 7 | 4 |
 | 2 | [7,9]| 延长至 9 | 9 | 4 |
 | 决赛| - | 步行 9→10 | 10 | 10 5 |

 输出为5。 

该轨迹表明，只有在跨越未覆盖的间隙时，成本才会累积，而段内的移动是自由的。 

### 示例 2

 输入：```
n = 8, k = 3
[1,3]
[2,5]
[6,8]
```合并后的段变为：

 [1,5], [6,8]

 | 步骤| 细分 | 之前的位置 | 行动| 后 | 成本|
 | ---| ---| ---| ---| ---| ---|
 | 1 | [1,5]| 1 | 在里面，跳到5 | 5 | 0 |
 | 2 | [6,8]| 5 | 步行 5→6 | 8 | 1 |

 最终成本为1。 

这说明了为什么合并是必要的。 如果不合并 [1,3] 和 [2,5]，人们可能会错误地认为重叠区域内需要多个转换。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(k log k) | O(k log k) | 排序间隔占主导地位，合并和扫描是线性的 |
 | 空间| O(k) | 存储间隔和合并段|

 这些约束允许每个测试用例最多有 2 · 10^5 间隔，因此 O(k log k) 解决方案很容易在 3 秒内足够快，并且线性扫描使每次测试的开销保持最小。

 ## 边缘情况

 一个关键的边缘情况是根本没有公交车路线。 在这种情况下，合并的段列表为空，答案必须退回到从 1 到 n 的纯粹行走。 该算法明确地处理这个问题`k == 0`检查，产生 n − 1，它与沿线所需的距离相匹配。 

当总线间隔恰好从位置 1 开始时，会发生另一种微妙的情况。算法会正确处理此问题，因为如果 1 位于第一个合并段内，则初始位置被视为位于第一个合并段内，从而允许立即自由扩展至段的右端点。 

最后一个重要的情况是紧密链接的区间，例如 [1,2]、[2,3]、[3,4]。 即使除了端点之外没有任何对完全重叠，合并步骤会正确地将它们统一为单个段，从而确保算法不会错误地在中间停靠点之间行走。
