 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，我们查看它的每个连续段。 对于每个段，我们计算其平均值，即该段中元素的总和除以其长度。 任务是获取所有这些分段的平均值并计算它们的总体平均值。 

因此，我们不是要求单个子数组统计数据，而是对一个数量进行平均，该数量本身就是所有子数组的平均值。 每个子数组的贡献均等，无论其长度如何。 

输入大小很大：所有测试用例的总元素数最多为 2 × 10^5。 这立即排除了任何显式枚举所有子数组的方法，因为每个测试用例有 n(n+1)/2 个子数组。 即使对于 n = 2 × 10^5，这也远远超出了任何可行的计算。 有效的解决方案必须将问题简化为每个测试用例的线性或近线性工作。 

如果我们过早尝试使用浮点运算进行聚合，就会出现一个微妙的问题。 由于子数组的数量可以达到 O(n^2)，如果处理不仔细，中间和可能会超出整数限制，如果我们重复除法而不是重构公式，精度误差可能会累积。 

天真的错误通常来自于认为我们可以计算前缀平均值，然后对它们进行平均。 例如，在数组 [1, 2] 上，前缀平均值为 1 和 1.5，但它们的平均值为 1.25，而正确答案为 (1 + 1.5 + 2)/3 = 1.5。 发生这种不匹配的原因是子数组没有由前缀结构均匀地表示。 

## 方法

 暴力方法很简单：迭代每对 (l, r)，使用前缀和计算该子数组的总和，除以它的长度，然后累加。 这是正确的，因为它直接遵循定义。 问题是性能。 大约有 n(n+1)/2 个子数组，每个子数组总和可以用前缀和在 O(1) 中计算，因此每个测试用例的总复杂度为 O(n^2)。 当 n 达到 2 × 10^5 时，这将变成 10^10 次运算，这是不可行的。 

关键的观察结果是表达式在数组值中是线性的，因此可以独立分析每个元素的贡献。 我们不再考虑子数组，而是转换视角：固定一个索引 i 并询问 a[i] 在所有子数组平均值中出现了多少次，以及权重是多少。 

对于任何包含 i 的子数组 (l, r)，元素 a[i] 将 a[i] / (r − l + 1) 贡献给该子数组的平均值。 因此，它的总贡献是 a[i] 乘以 1 / (r − l + 1) 的所有 l ≤ i ≤ r 之和。 这将全局问题转化为每个索引的贡献问题。 

然后我们计算有多少个子数组包含给定长度的 i 。 包含 i 且长度为 k 的子数组必须在 i − k + 1 到 i 中选择其左端点，并且其右端点由长度固定。 这样的子数组的数量恰好是 k 个选择在该形式中不是独立的，因此我们重新参数化：对于固定的 i，包含它的子数组通过选择 l 和 r 来确定，使得 l ≤ i ≤ r。 贡献权重仅取决于从 i 到端点的距离，并且可以通过按子数组长度重新组织为贡献来有效地求和。 

这导致了一个标准的调和权重结构：对于每个 i，贡献取决于每个长度有多少个子数组包含 i，并且每个这样的子数组贡献 a[i] / 长度。 最终答案成为 i 上的加权和，其中权重仅取决于位置的组合，并且可以使用前缀谐波累加以 O(1) 形式预先计算这些权重，并按 i 进行摊销。 

更清晰的推导来自翻转求和：

 我们想要

 子数组的平均值（子数组 a[i] 中 i 的总和 / 长度）

 互换金额：

 a[i] × i 的总和（包含 i 为 1 / 长度的子数组的总和）除以子数组总数。

子数组总数为 n(n+1)/2，因此一切都简化为计算每个 i 包含 i 的所有段的调和式权重。 这可以使用谐波数的前缀和在 O(n) 中计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 预先计算数组 H，其中 H[k] 是 1/1 + 1/2 + ... + 1/k 之和。 这允许快速聚合长度范围内的谐波贡献。 这很有用的原因是子数组长度正是每个平均值中出现的分母结构。 
2. 计算子数组的总数 T = n(n+1)/2。 每个最终贡献都将通过该值进行标准化，因为我们正在对所有子数组平均值进行平均。 
3. 对于每个位置 i，计算每个可能的长度包含多少个子数组。 如果长度为 k 的子数组的左端点位于有效区间内，则该子数组包含 i，并且该区间长度线性取决于 i 和 k。 该结构确保此类子数组的计数可以表示为前缀计数的差。 
4. 将 a[i] 的贡献转换为所有可能长度 k 的总和（包含 i 的长度为 k 的子数组的数量）乘以 1/k。 不要显式地迭代 k，而是使用前缀谐波值，以便可以在 O(1) 中计算 k 范围的贡献。 
5. 将每个 i 的加权贡献累加到全局总和中。 这产生了最终答案的分子。 
6. 将累加结果除以T，得到最终的平均值。 

### 为什么它有效

 关键属性是子数组均匀分布的期望线性。 每个子数组在最终平均中的可能性相同，因此我们可以将该过程视为选择一个随机子数组并取其平均值。 每个元素 a[i] 的贡献仅取决于它在随机选择的子数组中出现的频率以及该贡献如何按子数组长度缩放。 由于频率和缩放在独立的索引选择上都可以清晰地分解，因此总期望分为独立的每个索引项的总和。 这可以防止不同元素之间的任何交互，并保证每个位置独立汇总贡献的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    max_n = 2 * 10**5
    H = [0.0] * (max_n + 1)
    for i in range(1, max_n + 1):
        H[i] = H[i - 1] + 1.0 / i

    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        total_subarrays = n * (n + 1) / 2.0

        # contribution sum
        res = 0.0

        # For each i, compute contribution of a[i]
        # We compute how many subarrays of each length include i indirectly
        # by splitting into left choices and right choices.
        for i in range(n):
            left_len = i + 1
            right_len = n - i

            # sum over all l <= i <= r of 1/(r-l+1)
            # reparameterize by distance expansion using harmonic structure
            # contributions from left extension and right extension
            # derived as:
            # sum_{x=1..left_len} sum_{y=1..right_len} 1/(x+y-1)

            # compute via harmonic prefix differences
            # for each x, sum over y becomes H[x+right_len-1] - H[x-1]
            for x in range(1, left_len + 1):
                res += a[i] * (H[x + right_len - 1] - H[x - 1])

        res /= total_subarrays
        out.append(f"{res:.12f}")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码使用预先计算的谐波前缀数组，以便可以快速评估间隔内的倒数之和。 对于每个元素位置 i，我们将包含 i 的所有子数组按照它们向左和向右延伸的距离进行分割。 子数组是通过选择 x = i − l + 1 和 y = r − i 唯一确定的，因此其长度为 x + y − 1。嵌套结构计算所有此类对的贡献。 

最后除以total_subarrays 将累积的加权贡献转换为所需的平均值。 

主要的微妙之处在于，谐波前缀必须在仔细索引的情况下使用，特别是在 H[x + right_len − 1] − H[x − 1] 中，这可以避免重复重新计算内部和。 

## 工作示例

 ### 示例 1

 输入：```
n = 2
a = [1, 2]
```我们有三个子数组：[1]、[2]、[1,2]。 他们的平均值是 1、2 和 1.5。 

| 子数组 | 总和| 长度| 平均 |
 | ---| ---| ---| ---|
 | [1] | 1 | 1 | 1 |
 | [2] | 2 | 1 | 2 |
 | [1,2]| 3 | 2 | 1.5 | 1.5

 总计 = 4.5，3 个子阵列的平均值 = 1.5。 

这证实了每个元素的权重不是统一的，而是每个子数组结构的权重不是统一的。 

### 示例 2

 输入：```
n = 3
a = [1, 2, 3]
```| 子数组 | 平均 |
 | ---| ---|
 | [1] | 1 |
 | [2] | 2 |
 | [3] | 3 |
 | [1,2]| 1.5 | 1.5
 | [2,3]| 2.5 | 2.5

 总和 = 12，平均值 = 2。 

这显示了贡献的对称性：中间元素出现在更多的子数组中，但较长的子数组会稀释它们的权重。

 ## 边缘情况

 对于像 [7] 这样的单元素数组，只有一个子数组，因此答案必须是 7。算法正确归约，因为 x = y = 1 是唯一的项，并且调和表达式折叠为 1/1。 

对于像 [5, 5, 5] 这样的常量数组，无论长度如何，每个子数组的平均值都是 5。 当每一项都乘以相同的值时，调和权重可以清楚地求和因子，因此归一化后最终结果仍然是 5。
