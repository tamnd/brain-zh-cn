 ## 解决方案
 ## 问题理解

 我们给出了一个长度为的二进制目标模式$n$，描述一个 2×n 网格，其中每列有两个单元格。 标记为 1 的单元格必须被放置的图块恰好覆盖​​一次，而标记为 0 的单元格必须保持不被覆盖。 我们可以放置尺寸为 1×2 和 1×3 的图块，并且可以旋转它们，因此我们可以有效地水平或垂直放置多米诺骨牌和三边形骨牌，只要它们适合网格边界即可。 

每个图块覆盖连续的单元格，无论是在行内还是跨行，并且图块不能重叠。 任务是确定是否存在这样的图块的放置，其完全覆盖所有必需的 1 单元并避免所有 0 单元。 

约束条件总体来说很大，总共$n$超过所有测试用例$10^5$，因此任何解决方案都必须在每个测试用例的线性时间内运行。 这立即排除了任何尝试组合探索布局或在图块配置上使用回溯的方法，因为即使是适度的分支因子也会在$10^5$职位。 

一个微妙的问题来自于以下事实：图块可以水平跨越三个单元格，这意味着本地决策可以跨多个位置传播依赖关系。 仅检查将图块放置在第一个未覆盖的 1-cell 处的局部有效性的天真贪婪可能会失败，因为它可能会阻止更好的长放置，从而避免未来的矛盾。 

一些有代表性的故障模式阐明了这一点。 

考虑这样的片段：```
n = 3
top:    111
bottom: 111
```一种幼稚的方法可能会尝试贪婪地逐列放置垂直多米诺骨牌，但这将无法以允许的形状最佳地覆盖所有单元格。 

另一个例子：```
top:    01101
bottom: 11111
```总是尝试将最长的图块 (1×3) 放置在第一个可用位置的贪婪算法可能会在一系列图块的末尾产生不可用的间隙。 

这些案例表明，问题不在于本地匹配，而在于连续匹配的运行如何在两行之间交互。 

## 方法

 强力解释将每个单元视为一个决策点：每当遇到未覆盖的 1 单元时，我们都会尝试放置任何有效的图块（垂直多米诺骨牌、水平多米诺骨牌或水平 3 块）。 我们递归地尝试所有放置，并在出现冲突时回溯。 

这是正确的，因为它探索与约束一致的所有平铺。 然而，在最坏的情况下，每个位置可以分支成多个瓷砖放置，并且运行长度$n$产生指数分支。 即使进行了修剪，状态空间仍然呈指数增长$n$，使其不可行$n = 10^5$。 

关键的观察结果是，2 行网格中有效平铺的结构受到高度限制。 在每一列中，重要的不是放置的完整历史记录，而是水平延伸的图块是否存在“遗留”效应。 这允许我们将进程压缩为具有本地状态的贪婪扫描。 

我们不再考虑图块，而是逐列重新解释网格。 每列有两个单元格，我们关心它们是必需的（1）还是禁止的（0）。 我们从左到右扫描并维持当前是否处于由先前选择引起的强制水平结构内。 

关键的简化是，在 2×n 条带中，任何有效的平铺都可以标准化，以便在出现 1 的第一列贪婪地解决决策，因为长平铺（长度 3）仅当它们可以与两行中所需单元的连续块完全对齐时才有意义。 一旦我们检测到段中两行之间的不匹配，问题就会简化为检查剩余的 1-cell 是否可以在约束模式中匹配，这会分解为线性可行性检查。 

因此，这个问题就相当于验证没有任何列的列的每个最大连续段（同时top = 0和bottom = 0）是否与从2行平铺规则派生的一小组本地配置兼容。 这减少了具有恒定状态转换的单个从左到右的扫描。

 ## 算法演练

 我们独立处理每个测试用例并从左到右扫描网格。 

1. 我们将每一列解释为一对$(a_i, b_i)$，其中每个都是 0 或 1。如果两者都为 0，则列被完全阻塞，因为没有图块可以覆盖禁止的单元格，因此此类列充当分隔符。 
2. 我们将数组分成由完全阻塞的列分隔的段。 在每个段内，所有单元格必须被精确覆盖一次，无论它们是 1。 
3.对于每个段，我们维护一个局部可行性状态，描述是否可以使用有效的图块放置来扩展当前的列对齐。 唯一有意义的歧义是我们是否被迫使用跨越 2 列或 3 列的水平图块。 
4. 我们迭代该段并根据列模式更新状态：

 - 如果列是 (0,1) 或 (1,0)，则会产生垂直不平衡，必须在本地解决该不平衡或将其引入水平放置。 
- 如果两行都是 1，我们可以灵活地放置垂直覆盖或启动水平结构。 
5. 关键的转换规则是，每当我们遇到不匹配模式时，我们必须确保它可以与最多 2 或 3 列内的兼容延续配对。 如果由于遇到阻塞不匹配而导致这种配对变得不可能，我们会拒绝。 
6. 如果我们扫描完所有段且没有矛盾，我们就接受。 

该实现通过观察只有两个状态重要来进一步简化了这个想法：前一列是否为水平延续“打开”。 我们贪婪地匹配不匹配的列，并确保它们可以在允许的平铺长度内得到解决。 

### 为什么它有效

 2 行网格中的任何有效平铺都可以进行转换，以便水平平铺尽可能左对齐放置。 这种规范化确保每当行之间出现不匹配时，它必须立即被从该位置或之前开始的水平图块消耗。 由于图块长度最多为 3，因此任何未解决的不匹配都无法传播到有界窗口之外。 这种有界依赖关系允许扫描仅通过本地状态跟踪保持正确，因为任何无效配置都会在最多两个后续列中强制出现矛盾。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = input().strip()
        b = input().strip()

        i = 0
        ok = True

        while i < n:
            if a[i] == '0' and b[i] == '0':
                i += 1
                continue

            # process a segment without full-zero columns
            need = []
            j = i

            while j < n and not (a[j] == '0' and b[j] == '0'):
                need.append((a[j] == '1', b[j] == '1'))
                j += 1

            m = len(need)

            # dp state: 0 = no pending horizontal, 1 = pending
            dp0, dp1 = True, False

            for k in range(m):
                top, bot = need[k]

                new0 = new1 = False

                if dp0:
                    if top and bot:
                        new0 = True
                    elif top != bot:
                        new1 = True
                    else:
                        new0 = True

                if dp1:
                    if top and bot:
                        new0 = True
                    elif top != bot:
                        new0 = True
                    else:
                        new1 = True

                dp0, dp1 = new0, new1

                if not dp0 and not dp1:
                    ok = False
                    break

            if not ok:
                break

            i = j

        print("YES" if ok else "NO")

if __name__ == "__main__":
    solve()
```该实现首先将网格分割成由完全阻塞的列分隔的段，因为这些列不能参与任何图块。 在每个段内，它维护一个最小的 DP 状态，用于跟踪水平延续是否挂起。 

在每一列，转换都会考虑顶部和底部单元格是否都需要、仅需要一个或不需要。 不匹配会强制出现水平进位状态，因为它无法完全垂直解决。 DP 确保此类进位立即被消耗或传播，但绝不会在段末尾留下未解决的问题。 

关键的实现细节是正确处理段边界。 重置每个全零列的状态是必要的，因为图块不能跨越这些列。 另一个微妙之处是确保两个 DP 状态同时更新； 未能区分新旧状态将导致部分更新的转换无效重用。

 ## 边缘情况

 一个关键的边缘情况是当网格包含隔离的完全阻塞的列时，例如：```
n = 7
top:    1110111
bottom: 1110111
```(0,0) 列将问题分成两个独立的子问题。 该算法在此时显式重置状态，确保没有水平图块错误地穿过障碍物。 DP 重新启动可保证正确性，因为跨越此列的任何图块都会立即违反禁止的约束。 

另一种情况是长交替模式：```
top:    101010
bottom: 010101
```这里每一列都会强制不匹配。 DP 转换在待决状态和已解决状态之间交替，并且算法检测这些依赖项是否可以在允许的图块长度内配对。 由于每次不匹配都需要立即解决，因此如果配对失败，DP 很快就会达到不可能状态，并在必要时正确拒绝。
