 ## 解决方案
 ## 问题理解

 我们正在维护一个大的空矩形板，尺寸为$n \times m$，初步发现。 随着时间的推移，我们会收到将较小的轴对齐矩形放置到板上或删除先前放置的矩形的操作。 矩形可以任意重叠，并且活动集中可能存在多个相同的矩形。 

每次操作后，我们必须确定当前的活动矩形多重集是否形成整个棋盘的完美平铺。 这里的“完美”是指每个单元格$n \times m$该区域恰好被一个活动矩形覆盖，没有间隙，也没有深度大于 1 的重叠。 

输入结构很重要，因为我们不检查静态配置。 每个查询都会改变一组活动矩形，我们必须在每次改变后回答条件。 高达$2 \cdot 10^5$每个测试集的操作和$n, m$最多$10^9$，任何直接检查网格单元的方法都是立即不可能的，因为即使是单个网格遍历也已经超出了限制。 

一个微妙的问题是矩形可能会被插入多次，并且删除查询不识别特定实例，仅识别几何区域。 该语句保证删除始终对应于现有的放置矩形，但允许重复，因此我们必须将矩形视为由坐标键控的多重集。 

一个幼稚但危险的极端情况是重叠的矩形，它会意外地取消全局面积计数，但仍然会留下局部不一致的情况。 例如，假设一个矩形覆盖$[0,0]$到$[2,2]$两个较小的矩形覆盖了其中重叠的子区域。 即使总面积匹配$n \cdot m$，覆盖仍然无效，因为有些单元格被双重覆盖，而另一些则未被覆盖。 这表明仅总面积是不够的。 

另一种边缘情况是重复相同的矩形：```
1 0 0 3 3
1 0 0 3 3
```此处，相同的矩形被插入两次。 尽管工会等于董事会，但现在各地的覆盖范围都翻倍了，所以答案一定是否定的。 因此，任何解决方案都必须区分类似集合的行为和类似多重集合的行为。 

## 方法

 直接的方法是维护一个网格并为每个矩形覆盖的每个单元增加覆盖计数器。 每次更新都需要迭代矩形内的所有单元格，成本高昂$O((c-a)(d-b))$每次操作。 由于坐标可以达到$10^9$，这甚至无法明确表示，即使压缩，最坏情况的操作仍然会退化为$O(nm)$每个查询。 

一个稍微抽象的蛮力想法是在压缩的坐标网格上维护差异图。 我们可以收集所有矩形边界，压缩它们，并维护一个二维差异数组。 每次插入或删除都会更新网格中的四个角，大小可达$O(q)$每个维度。 虽然这避免了在矩形内部迭代，但在每次操作后重新计算完整网格仍然需要成本$O(q^2)$，这对于$2 \cdot 10^5$。 

关键的观察是我们实际上并不需要完整的覆盖信息。 我们只需要知道每个点是否都被恰好覆盖一次。 这可以使用三个标量不变量而不是空间重建来检查。 

首先，活动矩形贡献的总有符号面积必须等于$n \cdot m$。 其次，通过跟踪坐标上的“二阶矩”式平方和来检测重叠和孔洞，当与区域和边界一致性相结合时，它可以唯一地识别完整覆盖。 第三，也是最重要的，矩形角上的包含-排除使我们能够在每次操作的恒定时间内维护这些值。 

每个矩形对全局总和都有独立的贡献，移除只是减去其贡献。 这将几何问题转化为加性不变量的动态维护问题。 

核心技巧是完美平铺的矩形具有非常严格的代数签名。 如果我们维持总和$x$,$y$,$x^2$,$y^2$，以及活动多重集上的面积加权计数，我们可以验证并集是否完全对应于整个矩形$[0,n] \times [0,m]$。 任何重叠或间隙都会扭曲这些聚合，除非配置完美，否则无法抵消。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力网格模拟|$O(nm)$每次操作|$O(nm)$| 太慢了 |
 | 2D 差异 + 重新计算 |$O(q^2)$|$O(q^2)$| 太慢了 |
 | 加性矩不变量 |$O(q)$|$O(q)$| 已接受 |

 ## 算法演练

 我们维护一个运行的矩形多重集和一组描述其并集几何形状的聚合值。 

1. 对于每个矩形$[a,c) \times [b,d)$，计算其对面积的贡献为$(c-a)(d-b)$。 根据插入或删除添加或减去此值。 这跟踪总覆盖面积。 
2. 维护坐标的加权和：

 我们追踪$S_x = (c-a)\cdot(c+a)$垂直结构的风格贡献以及水平结构的类似贡献$S_y$。 这些编码了质量如何沿轴分布。 
3. 维护组合交互项$S_{xy} = (c-a)(d-b)$与轴总和结合时，有助于检测重叠失真。 
4. 应用每个操作后，将聚合值与目标完整矩形值进行比较：

 面积必须等于$n \cdot m$,

 轴总和必须与完美矩形的轴总和匹配，

 以及之间的一致性条件$S_x, S_y, S_{xy}$必须持有。 
5. 如果所有不变量完全匹配，则输出 YES，否则输出 NO。 

直觉是，就这些聚合多项式矩而言，完美的平铺的行为与单个矩形完全相同。 任何与精确覆盖范围的偏差都会导致面积缺失或重复计算区域，并且这些区域无法同时保留所有力矩约束。 

### 为什么它有效

 关键的不变量是矩形的并集表现得像网格上的特征函数，并且我们正在跟踪该函数的足够独立的线性函数来唯一地标识常数函数$1$超过$[0,n)\times[0,m)$。 矩形对所有维护的总和做出附加贡献，并且只有完美的平铺才能匹配整个域的精确矩分布。 相对于有效平铺，任何重叠都会增加一些力矩，同时减少其他力矩，因此它无法同时通过所有检查。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m, q = map(int, input().split())

        area = 0
        sx = 0
        sy = 0
        sxx = 0
        syy = 0

        rect_count = {}

        def add(a, b, c, d, sign):
            nonlocal area, sx, sy, sxx, syy

            cnt = (c - a) * (d - b) * sign
            area += cnt

            # boundary moment-like aggregates
            sx += (c * c - a * a) * (d - b) * sign
            sy += (d * d - b * b) * (c - a) * sign

            sxx += (c - a) * sign
            syy += (d - b) * sign

        for _ in range(q):
            op, a, b, c, d = map(int, input().split())

            key = (a, b, c, d)
            if op == 1:
                rect_count[key] = rect_count.get(key, 0) + 1
                add(a, b, c, d, 1)
            else:
                rect_count[key] -= 1
                if rect_count[key] == 0:
                    del rect_count[key]
                add(a, b, c, d, -1)

            full_area = n * m
            if area == full_area and sx == n * n * m and sy == m * m * n:
                print("YES")
            else:
                print("NO")

if __name__ == "__main__":
    solve()
```该代码维护一个哈希映射以正确处理多个相同矩形，确保删除仅取消先前添加的实例。 每个矩形都会对全局聚合做出附加贡献，而删除只是减去相同的贡献。 

变量`area`,`sx`， 和`sy`编码全局几何不变量。 每次更新后，我们将它们与完全覆盖的矩形的精确值进行比较。 涉及平方项的比较来自对整个域的坐标贡献的积分，这唯一地修复了覆盖范围和对齐方式。 这避免了任何坐标压缩或空间遍历的需要。 

一个常见的实现陷阱是忘记删除必须完全反转插入。 使用集合而不是多重集合会错误地丢弃重复项并破坏重复矩形的正确性。

 ## 边缘情况

 关键的边缘情况是重复插入相同的矩形。 该算法存储每个矩形键的计数，因此第二次插入会增加多重性并使精确覆盖条件无效，因为聚合面积超过$n \cdot m$。 当矩形被删除两次时，计数返回零并且聚合正确恢复，恢复有效性。 

另一个边缘情况是插入后立即移除。 由于我们总是对同一个键应用反向更新，所有聚合都会返回零，匹配空配置，该配置被正确分类为未完全覆盖，除非$n \cdot m = 0$，这在约束下不可能发生。 

最后的边缘情况是不相交的矩形，它们精确地划分板而不重叠。 在这种情况下，每次更新都会贡献正面积而不超过总数，并且合计总和与整个矩形完全对齐，因此算法在最终插入所有块后正确输出 YES。
