 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每个测试用例中，都有一个整数列表。 我们可以从这个列表中删除一个元素。 这样做之后，我们查看剩余的数字并尝试选择一个值$d$仍然出现在列表中并且严格小于删除的值$c$。 我们的目标是选择移除和结果值$d$以便$d$尽可能大。 

简单来说，我们选择一个元素作为“阈值”$c$，删除它的一个出现，然后在剩余数组中查找仍然小于的最大数字$c$。 我们希望最大限度地发挥最好的作用$d$的所有选择$c$。 

约束很小：每个数组最多有 100 个元素，值最多 100 个。这立即表明，即使是三次或二次解也是可以接受的，但我们仍然应该以一种使逻辑透明的结构为目标，而不是显式地模拟每个删除。 

一个微妙的点是，删除元素只会影响该确切值的可用性。 如果我们选择$d \neq c$，然后删除$c$永远不会删除所有出现的$d$。 这意味着大多数“删除并重新计算”的想法都是不必要的开销。 

值得考虑的边缘情况来自最大值出现多次或有许多重复项的数组：

 如果所有值都相等，例如`3 3`, 没有有效的对$d < c$，因为无论你去掉什么，都不存在严格意义上更小的值。 该问题保证这种情况不会以退化的方式发生，因为它确保至少存在两个不同的值。 

如果数组是`1 2 2`，去掉一个`2`仍然离开`1 2`，以及最好的$d$仍然是`1`。 幼稚的模拟可能会错误地认为删除唯一的最大值会改变答案，但事实并非如此。 

## 方法

 蛮力的想法很简单。 对于每个索引$i$， 对待$a[i]$作为删除的值$c$。 删除后，扫描剩余的数组并检查每个值$a[j]$小于$c$，取其中的最大值。 这需要$O(n)$每次移除的工作量，并且由于有$O(n)$选择，总复杂度为$O(n^2)$每个测试用例。 和$n \le 100$，这已经是安全的了。 

然而，这种方法重复地重新计算相同的结构。 关键的观察结果是删除操作几乎无关紧要：删除一个值的出现并不会改变数组中是否存在另一个值。 因此，唯一重要的问题是是否存在任何严格大于的值$d$。 如果存在这样的值，我们总是可以将其选择为$c$，并且删除它的一个出现不会影响$d$。 

所以问题就简化为寻找最大值$d$使得数组中至少存在一个严格更大的值。 这正是这组数字中第二大的不同值。 

因此，我们可以完全忽略多样性，只使用独特的价值观。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(n)$| 已接受 |
 | 最佳（独特扫描）|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 读取数组并提取所有不同的值。 

这会删除重复项，因为重复项不会改变值之间的顺序，只会改变频率。 
2. 按降序对不同值进行排序。 

排序为我们提供了从最大到最小的清晰排名，它直接对条件进行编码$d < c$。 
3. 获取此已排序的不同列表中的第二个元素。 

该值是最大的数字，并且至少还有一个严格大于它的值。 
4. Output this value for the test case.

 步骤 3 背后的推理是解决方案的核心。 任何候选人$d$必须有一些$c > d$。 这立即意味着$d$不能是最大不同值。 在所有剩余的候选者中，最好的可能选择是仍低于最大值的最大选择，即第二大的不同值。 

### 为什么它有效

 让$M$是数组中的最大值。 任何有效的$d$必须满足$d < M$，因为如果$d = M$，没有严格更大的值可以作为$c$。 对于任何值$d < M$，我们总是可以选择$c = M$（或任何发生$M$）并将其删除。 由于删除一个元素并不能消除所有出现的元素$d$,$d$保留在数组中。 因此，除了最大值之外的每个值都是可行的，我们通过选择其中最大的值来最大化，这是第二大的不同值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        vals = sorted(set(a), reverse=True)
        print(vals[1])

if __name__ == "__main__":
    solve()
```该解决方案独立读取每个测试用例，将数组转换为一个集合以删除重复项，按降序排序，然后输出第二个元素。 唯一微妙的要求是确保数组至少有两个不同的值，这是由问题陈述保证的。

 ## 边缘情况

 当最大值出现多次时，删除一个出现的值仍然会保留在数组中。 这意味着选择最大值始终有效$c$，确保每个较小的不同值作为候选值仍然可达$d$。 该算法自然地处理这个问题，因为重复项在排序之前被删除，因此频率永远不会影响排序。 

当值紧密排列时（例如连续整数），第二大不同值仍然是明确定义的。 例如，在`[1, 2]`，答案是`1`，并且删除任一元素都不会改变只有一个值可以作为有效值的事实$d$。 

当中间值有多个重复时，例如`[1, 2, 2, 3]`, 额外的存在`2`s 不会改变不同值的顺序，算法仍然正确返回`2`。
