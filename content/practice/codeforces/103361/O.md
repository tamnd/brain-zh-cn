 ## 解决方案
 ## 问题理解

 我们得到一个表示一条单元格的二进制字符串。 有些单元格已经包含一个十字，标记为`X`，其余为空，标记为`.`。 关键的限制是不允许两个十字相邻，这意味着我们永远不可能有`XX`在邻近的位置。 

Misha 已经放置了一组有效的十字架。 然后另一个人，Lёsha，添加了更多的十字架，仍然遵守任何两个十字架都不相邻的规则。 最后，我们被告知 Katya 确认配置是最大的，这意味着 Misha 和 Lёsha 放置交叉后，没有空单元格，我们仍然可以在不违反邻接规则的情况下放置额外的交叉。 

任务是确定在给定初始配置的情况下，Lёsha 最多可以添加多少个十字架。 

输入大小很小，n 最大为 1000。这立即表明 O(n²) 方法原则上是可以接受的，但由于结构非常局部，我们应该期望线性贪婪解决方案。 

一个微妙但重要的限制是 Misha 的配置已经有效，因此没有相邻的`X`最初的字符。 这消除了对验证或修复逻辑的需要，使我们能够专注于增强。 

主要的边缘情况来自现有交叉口和边界之间的短间隙：

 如果字符串是`"..."`，那么所有位置都是空闲的，但我们仍然不能将十字放置在彼此相邻的位置，因此最大放置位置是每隔一个单元格。 

如果字符串是`"X.X"`，则不能添加额外的交叉。 

如果字符串是`"..X.."`，则只能在左侧或右侧放置一个额外的十字，具体取决于奇偶校验。 

一个幼稚的错误是尝试贪婪地放置而不标记新占据的邻居，或者忘记放置的十字会阻挡两个相邻的单元格。 

## 方法

 暴力策略会尝试空位置的每个子集，并检查在那里放置十字是否违反邻接约束。 对于每个候选子集，我们将在 O(n) 中验证整个字符串，从而导致 O(2ⁿ·n) 复杂度，即使对于 n = 1000，这也是完全不可行的。 

稍微更合理的暴力方法是迭代位置并递归地决定是否放置十字。 即使进行了修剪，分支因子仍然很高，因为每个空单元仅与邻居局部相互作用，并且我们最终仍然探索指数配置。 

关键的观察结果是，约束纯粹是局部的：放置十字仅禁止其直接邻居。 这意味着我们永远不需要回溯。 相反，我们可以从左到右扫描，并在两个邻居都为空时贪婪地放置一个十字。 

我们还必须尊重已经存在的`X`位置，充当强制占用的单元格。 一旦我们看到某个单元格已被邻居占用或阻塞，我们只需跳过它即可。 否则，立即放置十字总是安全且局部最优的，因为它保留了在剩余后缀中放置尽可能多的十字的能力。 

这将问题简化为单次线性传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2ⁿ·n) | O(2ⁿ·n) | O(n) | 太慢了 |
 | 最优贪婪扫描| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.初始化一个计数器`added = 0`追踪 Lёsha 有多少个十字架。 我们只计算新的展示位置，而不计算现有的展示位置。 
2. 将字符串转换为可变数组，以便我们可以标记新放置的十字。 这很重要，因为新放置的十字也会阻挡相邻的单元格。 
3.使用索引从左到右遍历字符串`i`。 
4. 对于每个位置`i`，检查是否为空。 如果已经是`X`，我们跳过它，因为它已经被占用了。 
5.如果位置`i`是空的，我们检查是否可以在这里放置一个十字。 这要求左邻居（如果存在）不是`X`，并且右邻居（如果存在）不是`X`。 我们也以同样的方式将新放置的传中视为阻挡。 
6. 如果两个邻居都安全，我们在位置画一个十字`i`, 增量`added`，并标记`i`作为`X`在数组中，因此它会影响未来的决策。 
7. 继续扫描直到字符串末尾。 

扫描完成后，`added`是 Lёsha 可以添加的最大十字架数。 

### 为什么它有效

 核心不变量是在扫描的每一步，前缀`[0..i-1]`已经处于这样的状态：对早期位置的进一步修改无法在不违反邻接约束的情况下改进答案。 任何有效的解决方案都可以转换为与前缀上的贪婪放置一致的解决方案，而无需减少交叉数量，因为位置处的决策`i`只取决于`i-1`和`i+1`。 一旦我们通过位置`i`，我们永远不会重新访问它，任何替代选择都会阻止右侧的相同或更多位置。 因此，尽可能放置交叉是局部最优且全局安全的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = list(input().strip())

    added = 0

    for i in range(n):
        if s[i] == 'X':
            continue

        left_ok = (i == 0 or s[i - 1] == '.')
        right_ok = (i == n - 1 or s[i + 1] == '.')

        if left_ok and right_ok:
            s[i] = 'X'
            added += 1

    print(added)

if __name__ == "__main__":
    solve()
```该解决方案将字符串读取到列表中，因此更新高效并立即反映新放置的交叉。 对左右邻居的检查包括边界处理，其中边缘在外部被视为自动安全。 

关键的实现细节是我们检查数组的当前状态，而不是原始字符串，以便新添加的交叉正确地阻止后续放置。

 ## 边缘情况

 最小输入，例如`"."`处理正确，因为两个边界检查都将缺失的邻居视为安全，因此算法放置一个十字并返回 1。 

完全交替的模式，例如`"X.X.X"`导致零加法，因为每个空位置至少有一个相邻的`X`。 扫描看到每个`.`但每次邻居检查都失败，所以没有进行放置。 

一个长的空段，例如`"........"`证明贪婪放置自然会产生每隔一个位置。 不变量成立，因为一旦十字放置在位置`i`， 位置`i+1`会被自动阻止并且不会重新考虑。
