 ## 解决方案
 ## 问题理解

 我们有一块边长为整数的矩形板。 任务是沿着网格线切割这个矩形，将其分割成更小的正方形。 每一次切割都会将一个现有的矩形分割成两个较小的矩形，最终的目标是将原始矩形完全分解为正方形。 目标是最小化最终分解中的方格数量。 

您可以将此过程视为重复选择当前矩形并水平或垂直切割它，生成两个较小的矩形，直到每个部分都是正方形。 输出是最终可能得到的最小方块数。 

此类问题的约束通常允许边长达到大约几百或几千。 这立即排除了所有切割序列的任何指数枚举，因为递归分割矩形的方法数量增长得非常快。 尝试所有可能的切割顺序的简单蛮力会组合爆炸，因为每个矩形最多可以以 O(n + m) 种方式分割，并且这种分支在每个子矩形上重复。 

当矩形已经是正方形时，会出现微妙的边缘情况。 例如，如果输入为 5 x 5，则正确答案为 1。任何始终执行至少一次剪切的策略都会错误地返回更大的值。 另一个重要的情况是高度倾斜的矩形，例如 1 × n。 在这种情况下，最佳答案恰好是 n，因为唯一有效的分解是分解为 1 x 1 平方，并且天真的贪婪水平或垂直分割策略可能会高估或低估，具体取决于其分割方式。 

## 方法

 蛮力方法直接对问题进行建模。 对于尺寸为 a × b 的矩形，我们尝试在位置 i 处尝试所有可能的垂直切割，产生矩形 i × b 和 (a − i) × b，并尝试在位置 j 处的每个水平切割，产生 a × j 和 a × (b − j)。 对于每个结果对，我们递归计算最小平方数并将结果组合起来。 那么矩形的答案就是所有选择中的最小值。 

这种方法是正确的，因为它探索了所有可能的剪切序列，但它失败了，因为每个矩形都会产生 O(a + b) 转换，并且每个转换都会导致进一步的递归。 即使使用记忆化，状态空间也是 O(a·b)，但每个状态仍然需要扫描 O(a + b) 次转换，导致复杂度约为 O(a·b·(a + b))。 对于 a = b = 500，这已经是 500 次操作的量级，这对于典型限制来说太慢了。 

关键的观察是该问题具有最佳子结构：一旦我们修复了第一次切割，得到的两个矩形就是独立的子问题。 这意味着动态规划就足够了，我们只需要存储每个矩形大小的最佳答案。 我们将 dp[a][b] 定义为平铺 a x b 矩形所需的最小正方形数，而不是重新计算。 每个过渡仅考虑沿一个剪切分割并组合子结果。 

我们还通过对称性减少状态空间，因为 dp[a][b] 等于 dp[b][a]，允许我们仅计算 a ≤ b。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力递归 | O(n·m·(n + m)) | O(n·m·(n + m)) | O(n·m) | 太慢了 |
 | 矩形上的 DP | O(n·m·(n + m)) 具有修剪/对称性 | O(n·m) | 已接受 |

 真正的增益不是转换的渐近改进，而是通过对称性消除重复的重新计算和修剪，这使得状态空间变得易于管理。 

## 算法演练

 我们定义一个表 dp，其中 dp[a][b] 存储平铺 a x b 矩形所需的最小正方形数。

1. 对于所有a、b，将dp[a][b]初始化为一个较大的值。 这表示我们尚未计算出该矩形大小的解决方案。 
2. 对于每个矩形，首先检查它是否已经是正方形。 如果 a 等于 b，则将 dp[a][b] 设置为 1。这是基本情况，因为不需要切割。 
3. 对于每个不是正方形的矩形，尝试所有可能的垂直切割。 对于位置 i 处的切割，矩形将由 b 分割为 i 和由 b 分割为 (a − i)。 这种分割的成本是 dp[i][b] + dp[a − i][b]。 我们用所有此类分割中的最小值更新 dp[a][b]。 
4. 同样，尝试在位置 j 处进行所有水平切割。 这分为 a by j 和 a by (b − j)，并贡献 dp[a][j] + dp[a][b − j]。 我们再次取最小值。 
5. 我们确保在访问 dp 值时，我们始终依赖于先前计算的或对称的状态，因此较小的矩形已经在较大的矩形之前被解析，通常通过按升序迭代 a 从 1 到 n 和 b 从 1 到 m 迭代。 

我们可以按升序填充表格的原因是，任何切割都会严格减少至少一个维度，因此子问题总是指已经计算过的较小或相等的维度。 

### 为什么它有效

 正确性取决于以下事实：矩形的任何最佳平铺都必须从第一次切割开始（除非它已经是正方形）。 第一个切割将问题划分为两个独立的矩形。 最优解必须选择尽可能最好的第一次切割，然后最优地求解每个得到的矩形。 由于 dp 枚举了所有可能的首次切割并使用子矩形的最佳子结构，因此它不会错过更好的分解。 该表保证每个子问题在用于更大的计算之前都得到最佳解决，从而防止次优部分解决方案的传播。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    dp = [[0] * (m + 1) for _ in range(n + 1)]
    
    for a in range(1, n + 1):
        for b in range(1, m + 1):
            if a == b:
                dp[a][b] = 1
                continue
            
            best = float('inf')
            
            for i in range(1, a // 2 + 1):
                best = min(best, dp[i][b] + dp[a - i][b])
            
            for j in range(1, b // 2 + 1):
                best = min(best, dp[a][j] + dp[a][b - j])
            
            dp[a][b] = best
    
    print(dp[n][m])

if __name__ == "__main__":
    solve()
```该实现从较小的矩形到较大的矩形构建了一个二维 DP 表。 循环只达到一半维度，因为 i 和 a − i 处的分割与 a − i 和 i 处的分割对称，因此我们避免了冗余计算。 方形单元的初始化确保了正确的基础传播。 最终答案是dp[n][m]。 

这里的一个常见错误是在整个范围而不是一半上迭代剪切，这会在不改变正确性的情况下使工作量加倍。 另一个微妙之处是确保仅在所有较小维度可用后才计算 dp[a][b]，这是通过增加循环阶数来保证的。 

## 工作示例

 考虑一个 2 x 3 的矩形。 我们按升序计算 dp。 

| 矩形 (a, b) | 类型 | 考虑削减| dp值|
 | --- | --- | --- | --- |
 | (1,1) | 方形| 无 | 1 |
 | (1,2) | 非正方形| 仅垂直 | 2 |
 | (2,2) | 方形| 无 | 1 |
 | (1,3) | 非正方形| 仅垂直 | 3 |
 | (2,3) | 非正方形| 垂直+水平| 3 |

 对于(2,3)，垂直切割得到(1,3)+(1,3)=6，而在1处水平切割得到(2,1)+(2,2)=2+1=3，所以dp[2][3]=3。 

该轨迹显示了 DP 如何避免朴素分裂并正确地选择与最佳子结构对齐的分解而不是相等的划分。 

现在考虑一个 3 x 3 的矩形。 由于它已经是一个正方形，因此 dp[3][3] = 1 立即，无论任何切割考虑。 这表明基本情况覆盖了不必要的递归。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n·m·(n + m)) | O(n·m·(n + m)) | 对于每个矩形，我们尝试最多 n 个水平分割和 m 个垂直分割，每个 O(1) 转换 |
 | 空间| O(n·m) | DP 表为每个矩形大小存储一个值 |

 对于 n 和 m 高达几百的中等约束，这种复杂性是可以接受的。 对称性约简有效地将常数因子减半，并且迭代结构确保没有递归开销。

 ## 边缘情况

 对于 1 x n 矩形，该算法仅反复考虑垂直分割。 每个 dp[1][k] 解析为 k，因为形成正方形的唯一方法是分成 1 × 1 块。 DP 表正确地累积了这一点，因为每个分割都会减少到更小的 1 x x 状态，最终达到 1 x 1 基本情况。 

对于 7 x 7 等平方输入，算法在考虑任何切割之前立即分配 dp[7][7] = 1。 尽管循环在技术上可以评估切割，但保护条件可以防止覆盖基本情况，确保不会引入不正确的分解。
