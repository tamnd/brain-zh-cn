 ## 解决方案
 ## 问题理解

 我们给定一个大小为的矩形$m \times n$，我们想使用固定大小的较小矩形完全平铺它$1 \times k$。 每个瓷砖可以放置在任一方向，这意味着它可以用作$1 \times k$或者$k \times 1$，并且我们可以将大矩形切成这样的块，没有重叠或间隙。 任务是确定这种完美的平铺是否存在。 

输入由三个整数组成$m$,$n$， 和$k$, 每个最多$10^9$。 由于只有一个测试用例，并且所有操作都是恒定时间算术检查，因此任何解决方案都必须在恒定时间内运行。 

关键的结构限制是每块瓷砖总是准确地覆盖$k$单位平方。 这立即意味着该区域$m \cdot n$必须能整除$k$。 然而，仅可整除性是不够的，因为将细长矩形拟合到网格中的几何形状对沿行和列的对齐施加了额外的约束。 

出现微妙的边缘情况时$k = 1$。 在这种情况下，每个单元格都已经是有效的图块，因此任何矩形都可以简单地图块化。 当一个维度已经为 1 时，会出现另一种边缘情况，因为那时我们正在有效地处理一维段。 

一个天真的错误是只检查区域的可分性。 例如，$m = 2, n = 3, k = 4$给出面积 6，它不能被 4 整除，所以它是不可能的。 但考虑一下$m = 2, n = 3, k = 2$：面积是6，能被2整除，确实有效。 真正的挑战是区分可分性成立但不可能放置的情况，例如$m = 6, n = 6, k = 4$，其中面积为 36 可被 4 整除，但由于可实现的条带配置不匹配，无法平铺。 

## 方法

 暴力方法会尝试模拟放置$1 \times k$或者$k \times 1$网格内的图块，回溯所有位置。 每个放置都会减少可用空间，我们将递归地尝试填充剩余区域。 状态的数量增长得非常快，因为每一步都有许多可能的位置，即使对于适度的网格，这也会很快呈指数增长。$m \cdot n$。 这仅作为正确性参考有用。 

关键的观察结果是矩形是均匀的，并且图块也是均匀的，因此任何有效的图块都必须遵守全局算术约束而不是局部搜索结构。 每个图块都贡献准确$k$单元格，因此总面积必须能除以$k$。 除此之外，唯一的结构限制来自于我们是否可以对齐长度的条带$k$沿着任一维度。 

如果我们尝试使用水平平铺$1 \times k$条带，我们需要$n$可以被整除$k$。 同样，如果我们使用垂直平铺$k \times 1$条带，我们需要$m$可以被整除$k$。 然而，我们可以混合方向，因此我们不限于单一的全球方向。 剩下的不平凡的情况是当两个维度都不能被单独整除时$k$。 在这种情况下，两个方向必须以尊重整数网格边界的方式组合，只有当两个方向都$m$和$n$甚至是与以下内容兼容的倍数$k$，唯一的障碍出现在$k$无法在两个维度上干净地分解。 

查看结构的更精确方法是将矩形划分为不同大小的块$k$。 任何这样的块必须完全位于以倍数对齐的行或列内$k$除非它跨越两个方向，这会迫使一致性条件$m \bmod k$和$n \bmod k$。 最后的简化是，当且仅当至少一个维度可以完全划分为长度段时，平铺才是可能的$k$， IE。，$m \bmod k = 0$或者$n \bmod k = 0$。 如果两个维度都不能被$k$，任何混合方向的尝试都会不可避免地产生无法完成的剩余碎片。 

因此，决定简化为检查两个简单的条件：面积可分性和至少一维可被$k$。 一旦一维可整除，面积条件实际上就隐含了，因此最终检查变得简单。

 ## 算法演练

 1. 读取数值$m$,$n$， 和$k$。 它们定义了网格和图块的形状。 
2.如果$k = 1$，立即输出“YES”。 每个单元格都已经是有效的$1 \times 1$平铺，因此矩形始终是可平铺的。 
3. 检查是否$m \cdot n$可以整除$k$。 如果不是，则输出“NO”。 这是必要的，因为每个瓷砖完全覆盖$k$单元格，因此总面积必须是平铺面积的整数倍。 
4. 检查是否至少有一项$m$或者$n$可以整除$k$。 如果是，则输出“YES”。 在这种情况下，我们可以将矩形划分为与该尺寸对齐的条带，每个条带使用旋转或未旋转的图块形成有效的图块。 
5. 如果两个维度都不能被整除$k$，输出“否”。 在这种情况下，任何尝试放置$k$-length 矩形最终将留下无法完成的剩余条带。 

### 为什么它有效

 每一块瓷砖总是精确消耗$k$晶胞，因此晶胞总数必须能被$k$。 这给出了全局必要条件。 第二个约束来自对齐：至少没有一个维度可被$k$，任意分解为长度-$k$段必须以留下不兼容的残留条带的方式跨越边界。 如果一个维度可以被$k$，我们可以首先沿着该维度进行划分，将问题简化为独立问题$k$逐一或逐一$k$条状，每条都可以平铺。 这两个条件一起充分表征了完美平铺的存在。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

m, n, k = map(int, input().split())

if k == 1:
    print("YES")
elif (m * n) % k != 0:
    print("NO")
elif m % k == 0 or n % k == 0:
    print("YES")
else:
    print("NO")
```代码直接遵循决策过程。 第一个条件隔离了瓦片是单个单元格的简并情况。 第二个强制执行全局区域约束。 最后一项检查捕获至少一个方向上是否存在干净的条带分解，这是避免剩余碎片的结构要求。 

一个微妙的点是顺序：检查$k = 1$首先避免了在具有固定整数大小的语言中不必要的乘法和潜在的溢出问题，即使 Python 可以安全地处理大整数。

 ## 边缘情况

 案例$k = 1$其行为与所有其他输入不同，因为它完全消除了几何约束。 用于输入$1\ 1\ 1$，算法立即返回 YES，与矩形已平铺的事实相匹配。 

什么时候$m \cdot n$不能被整除$k$， 例如$3\ 3\ 2$，算法在任何结构推理之前正确返回 NO。 执行在区域检查处停止，防止任何错误接受。 

当两个维度都不能被整除时$k$， 例如$6\ 6\ 4$，算法达到最终条件并正确返回 NO。 尽管 36 可以被 4 整除，但由于缺乏干净的条带分解，导致无法覆盖剩余区域。 

当一维完全等于$k$， 例如$4\ 7\ 4$，算法返回 YES 因为$m \bmod k = 0$，并且网格可以被分割成大小为垂直的条带$4 \times 1$，每个可独立平铺。
