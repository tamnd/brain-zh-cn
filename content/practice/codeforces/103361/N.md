 ## 解决方案
 ## 问题理解

 我们得到了一小部分圣诞树，每棵圣诞树的高度都是正整数。 任务是挑选三棵不同的树，使三棵树的高度完全相同。 输出是这三棵树的索引。 如果不存在这样的三元组，我们报告这是不可能的。 

输入大小非常小，最多 100 棵树，每个高度也以 100 为界。这立即告诉我们，即使相对较慢的二次或三次方法也是可以接受的，因为最坏情况下的操作总数约为 10^6 或 10^7，这对于 Python 中的 2 秒限制来说是微不足道的。 

当人们试图选择“任何重复的高度”时，就会出现一种微妙的失败模式。 至少有两个相同的高度是不够的。 例如，如果高度是`[5, 5, 7]`，有重复但没有足够的副本来形成三元组，所以答案仍然必须是`-1`。 严格的条件是某个值至少出现三次。 

另一种边缘情况是存在多个有效答案时。 任何有效的不同索引三元组都是可接受的，因此算法不应过度考虑选择或尝试优化字典顺序。 

## 方法

 一个直接的蛮力想法是检查索引的每个三元组`(i, j, k)`并检查是否`h[i] == h[j] == h[k]`。 这是简单且正确的，因为它明确地验证了所有可能的三元组。 然而，它的成本与三元组的数量成正比，大致为`n^3 / 6`。 和`n = 100`，这大约是 160,000 次迭代，这在 Python 中实际上仍然很好，但考虑到结构的简单性，它不必要地繁重。 

关键的观察是，除了价值平等之外，我们并不关心我们选择哪个三元组。 我们只需要知道某个值是否至少出现三次，如果是这样，我们需要它出现的任意三个索引。 这将问题从搜索索引组合转变为频率和簿记任务。 

我们不生成三元组，而是跟踪每个高度出现的位置。 一旦我们找到具有三个记录位置的高度，我们就可以立即输出它们。 这将问题简化为对数组的单次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力三倍 | O(n^3) | O(n^3) | O(1) | O(1) | 已接受但没有必要 |
 | 频率与指数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们维护从每个高度到它出现的索引列表的映射。 

1. 从左到右遍历树列表，同时记录每个高度的索引。 这确保了我们保留发生的自然顺序，从而立即给出有效的索引。 
2. 每当我们向特定高度的列表追加新索引时，检查列表大小是否已达到 3。当达到 3 时，我们可以停止进一步处理，因为我们已经有了有效的答案。 
3. 以任意顺序输出这三个存储的索引并终止程序。 
4. 如果扫描完所有树，没有任何高度达到频率 3，则输出`-1`。 

关键的设计选择是在步骤 2 中提前退出。一旦找到有效的三元组，就无需完全构建频率表。 

### 为什么它有效

 对于每个高度值，我们精确地维护它出现的索引集。 如果任何高度至少出现三次，则根据问题的定义，这三次出现形成有效的解决方案。 由于我们只存储输入中的实际索引，因此我们从不伪造或重复仓位。 因此，该算法无法输出无效索引或不匹配的高度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    h = list(map(int, input().split()))
    
    pos = {}
    
    for i, val in enumerate(h, start=1):
        if val not in pos:
            pos[val] = []
        pos[val].append(i)
        if len(pos[val]) == 3:
            print(pos[val][0], pos[val][1], pos[val][2])
            return
    
    print(-1)

if __name__ == "__main__":
    solve()
```该解决方案使用以高度为键的字典来存储索引。 每个索引都是从 1 开始的，以匹配典型的竞争性编程约定。 提前返回可以确保我们在找到有效的三元组后立即停止。 唯一微妙的一点是确保我们在检查长度条件之前追加，以便当前索引包含在候选三元组中。

 ## 边缘情况

 当所有树具有相同高度时，前三个索引立即形成有效答案。 一旦到达第三次出现，算法就会处理这个问题，而不扫描数组的其余部分。 

当每个高度恰好出现两次时，没有列表达到大小三，因此扫描完全正确地完成并输出`-1`。 

当有效三元组出现在数组中较晚时，算法仍然会检测到它，因为它不断累积索引并且不依赖于顺序处理之外的任何排序假设。
