 ## 解决方案
 ## 问题理解

 我们从一位数开始`x`。 从这个数字中，我们通过将当前数字乘以 3 并取结果的最后一位十进制数字，然后将其附加到板上所写数字的右侧来重复生成新数字。 后`n`步骤，我们有一个`n`-以这种确定性方式形成的数字。 

任务不是显式构造这个数字，而是确定最终是否`n`- 数字能被3整除。 

这些约束立即排除了任何直接模拟该过程的尝试。 自从`n`可以大到`10^9`，甚至生成`n`数字是不可能的。 任何解决方案都必须在读取输入后在恒定时间内起作用。 

一种天真的但自然的方法是实际模拟数字生成并计算模 3 的数字总和。这适用于小数字`n`，但对于大的情况来说变得不可行`n`。 

该过程本身不存在棘手的结构分支，因此唯一的潜在陷阱来自于对实际上决定被 3 整除的因素的误解。关键的微妙之处在于，由于重复的乘法和截断，数字以非线性方式演化，因此数字和是否具有任何简单的结构并不是立即显而易见的。 

## 方法

 一个简单的暴力解决方案模拟了这个过程。 我们从`a1 = x`，然后重复计算`ai = (3 * a(i-1)) % 10`并附加每个数字。 全部建成后`n`数字，我们计算它们的总和并检查它是否能被 3 整除。 

这是正确的，因为能被 3 整除仅取决于数字之和。 然而，该方法执行`n`迭代次数，最多可达`10^9`。 即使是 10 亿个简单的算术运算也远远超出了 Python 或 C++ 中 2 秒的限制。 

关键的观察结果来自于分析以 3 为模而不是以 10 为模的数字生成。递推式为`ai = (3 * a(i-1)) % 10`。 如果我们减少这个模 3，我们得到`ai ≡ (3 * a(i-1) mod 10) mod 3`。 自从`10 ≡ 1 (mod 3)`，取模 10 不会影响模 3 的值。这使我们能够像从未被截断一样进行推理。 

因此`ai mod 3 ≡ (3 * a(i-1)) mod 3 ≡ 0`为所有人`i ≥ 2`。 这意味着第一个数字之后的每个数字对模 3 的数字和没有任何贡献。整个整除条件崩溃为仅第一个数字`x`。 

因此，尽管该过程看起来是动态的，但它具有完全静态的模块化结构。

 ## 算法演练

 该解决方案将生成的数字减少为模块化属性，而不是构造它。 

1. 读取数值`x`和`n`。 价值`n`只控制生成多少个数字，但不会影响我们后面使用的模块化行为。 
2. 观察到能被 3 整除取决于模 3 的数字之和。这让我们可以忽略完整的数字结构，只关注数字贡献。 
3.注意第一个数字正好是`x`，因此它对总和的贡献是固定的。 
4. 对于每一个下一个数字，观察重复性`ai = (3 * a(i-1)) % 10`。 当考虑模 3 时，每个这样的数字都变成 0。发生这种情况是因为乘以 3 会使该值在截断之前可被 3 整除，仅影响 10 的倍数，从而保留模 3。 
5. 得出结论，除了第一位之外的所有数字都不影响模 3 的和。 
6. 检查是否`x % 3 == 0`。 如果是，则整数能被 3 整除，否则不能被 3 整除。 

### 为什么它有效

 该算法依赖于以下不变式：第一个之后生成的每个数字都与 0 模 3 全等。由于 3 的整除性仅取决于模 3 的数字之和，并且该过程在第一步之后不会引入任何非零贡献，因此最终答案完全由初始数字决定。 无论多大`n`变为，递推结构可防止后面的数字影响模 3 和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    x, n = map(int, input().split())
    if x % 3 == 0:
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该代码直接应用了只有第一位数字重要的观察结果。 的价值`n`被读取但从未在输入解析之外使用，因为它不影响整除性。 

关键的实施决策是完全避免任何模拟循环。 这就是即使在以下情况下也能保持解决方案恒定时间的原因：`n`非常大。

尽管第一个数字可以被 3 整除，但这显示了一个微妙的点：虽然后面的数字不贡献模 3，但它们仍然可以改变原始总和。 然而，从模的角度来看，它们的贡献始终是 3 的倍数，因此最终的整除性仅取决于初始残基类别。

def run(inp: str) -> str:

 ## 边缘情况

 对于`n = 1`，该数字只是起始数字。 该算法立即返回基于`x % 3`，它与定义匹配，因为没有发生任何转换。 

对于非常大的`n`， 例如`n = 10^9`，任何模拟都会失败，但解决方案忽略`n`完全读完之后。 计算保持恒定时间。 

对于像这样的数字`x = 9`，结果始终为 YES，无论`n`，因为第一个数字已经保证能被 3 整除，后面的数字永远不会改变模贡献。
