 ## 解决方案
 ## 问题理解

 我们有两个字符串集合，一个称为数组`a`另一个称为数组`b`。 任务是选择一个非空字符串`s`即同时满足两个条件。 

第一的，`s`必须作为子字符串出现在每个字符串中`a`。 这意味着如果我们从`a`，我们可以在其中的某个地方找到`s`作为一个连续的段。 

其次，在所有满足第一个条件的字符串中，我们想要`s`以子串的形式出现在尽可能少的字符串中`b`尽可能。 最小化此计数后，如果有多个候选者达到相同的最小值，我们选择长度最小的一个。 

输出只有两个数字：中的最小字符串数`b`包含所选子字符串以及该子字符串的长度。 如果根本不存在有效的子字符串，我们输出`-1 -1`。 

就字符串数量而言，约束看起来很大，但关键的隐藏结构是每个测试中所有字符串的总长度仅为 50,000。 这完全改变了复杂性的格局。 尽管可能有多达 50,000 个字符串，但它们总的来说很短，因此任何每个字符串都是二次的解决方案仍然可行。 

一个天真的解释会建议迭代所有字符串的所有子字符串`a`并检查每个字符串中的所有字符串`b`。 这会立即失败，因为每个字符串的子字符串数量呈二次方增长，并且重复检查成员资格会使该成本成倍增加。 

如果我们尝试对候选人进行一一测试，就会出现一个更微妙的问题。 例如，从第一个字符串中选取子字符串`a`并且针对所有其他字符串验证它们仍然可能导致崩溃，因为每个字符串候选子字符串的数量可能达到大约 125,000 个，并且针对许多字符串检查每个子字符串变得太慢。 

通常打破朴素方法的边缘情况包括以下情况：

 如果所有字符串都在`a`仅共享一个单字符子字符串，那么任何解决方案都必须正确检测该最小共享子字符串。 例如：```
a = ["abc", "bca", "cab"]
```只有单个字母是公共子串，并且未能在所有字符串中正确地交叉子串会导致不正确的候选者。 

另一个边缘情况是当没有子字符串在所有字符串之间共享时`a`， 例如：```
a = ["ab", "cd"]
```正确答案是`-1 -1`，并且假定至少一个公共字符的算法可能会错误地输出某些内容。 

## 方法

 暴力方法首先生成中每个字符串的所有子字符串`a`，然后检查每个子字符串是否出现在每个其他字符串中`a`。 如果是，我们然后计算有多少个字符串`b`包含它。 这在概念上是正确的，但在操作上却是灾难性的。 长度为 500 的单个字符串已经贡献了大约 125,000 个子字符串，而对多达 50,000 个字符串进行成员资格检查会大大超出可接受的限制。 

关键的观察是我们实际上不需要跟踪每个候选者的子字符串。 相反，我们可以颠倒视角。 我们想要出现在所有字符串中的子字符串`a`，因此我们可以独立处理每个字符串，提取其所有不同的子字符串，并计算包含每个子字符串的字符串数量。 这将“全局交集”问题转变为集合上的频率计数问题。 

一旦我们得到了每个子串的频率`a`，有效候选者恰好是那些频率等于中字符串数量的候选者`a`。 我们可以同时计算相同类型的频率`b`字符串，使我们能够有效地评估第二个优化标准。 

这是可行的，因为对总输入长度的约束确保了只要我们删除每个字符串中的重复子字符串，枚举所有字符串中的所有子字符串都是可行的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N·L3) | O(N·L3) | O(N·L²) | O(N·L²) | 太慢了 |
 | 使用散列计算子字符串的频率 | O(总 L²) | O(总 L²) | 已接受 |

 ## 算法演练

 ### 1. 以可比较的形式表示子字符串

 我们计算子字符串的滚动哈希，以便每个子字符串都可以存储为紧凑键。 这避免了重复存储原始字符串并保持比较高效。 

### 2. 处理每个字符串`a`对于中的每个字符串`a`，我们枚举所有子字符串，但将它们存储在一个集合中，以便同一字符串内的重复项不会增加计数。 处理完一个字符串后，我们更新一个全局映射`cntA[sub]`跟踪该子字符串出现了多少个不同的字符串。 

这一步很重要，因为条件是“在每个字符串中出现”，而不是“整体出现多次”。 

### 3. 重复相同的过程`b`我们再次枚举每个字符串的子字符串`b`，在每个字符串内去重，并维护`cntB[sub]`，计算有多少个字符串`b`出现子字符串。 

### 4. 筛选有效候选人

 我们迭代中看到的所有子字符串`cntA`。 子串仅在以下情况下有效`cntA[sub] == n`，这意味着它出现在所有字符串中`a`。 

### 5.选择最佳有效子串

 在有效子串中，我们选择最小化的一个`cntB[sub]`。 如果存在平局，我们选择最小的子串长度。 

### 为什么它有效

 该算法依赖于直接等价：子串出现在每个字符串中`a`当且仅当在每个字符串的子字符串集中对每个字符串计数一次。 每个字符串内的重复数据删除可确保频率解释的正确性。 由于每个候选在聚合后都会被评估一次，因此我们可以避免重复昂贵的检查，同时保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_counts(strings):
    MOD1 = 10**9 + 7
    MOD2 = 10**9 + 9
    BASE = 91138233

    cnt = {}

    for s in strings:
        seen = set()
        n = len(s)

        for i in range(n):
            h1 = 0
            h2 = 0
            for j in range(i, n):
                c = ord(s[j])
                h1 = (h1 * BASE + c) % MOD1
                h2 = (h2 * BASE + c) % MOD2
                seen.add((h1, h2, j - i + 1))

        for key in seen:
            cnt[key] = cnt.get(key, 0) + 1

    return cnt

def build_b_counts(strings):
    MOD1 = 10**9 + 7
    MOD2 = 10**9 + 9
    BASE = 91138233

    cnt = {}

    for s in strings:
        seen = set()
        n = len(s)

        for i in range(n):
            h1 = 0
            h2 = 0
            for j in range(i, n):
                c = ord(s[j])
                h1 = (h1 * BASE + c) % MOD1
                h2 = (h2 * BASE + c) % MOD2
                seen.add((h1, h2, j - i + 1))

        for key in seen:
            cnt[key] = cnt.get(key, 0) + 1

    return cnt

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = [input().strip() for _ in range(n)]
        m = int(input())
        b = [input().strip() for _ in range(m)]

        cntA = build_counts(a)
        cntB = build_b_counts(b)

        best_b = None
        best_len = None

        for (h1, h2, ln), c in cntA.items():
            if c == n:
                cb = cntB.get((h1, h2, ln), 0)
                if best_b is None or cb < best_b or (cb == best_b and ln < best_len):
                    best_b = cb
                    best_len = ln

        if best_b is None:
            print("-1 -1")
        else:
            print(best_b, best_len)

if __name__ == "__main__":
    solve()
```该实现是围绕两个对称预处理函数构建的，其中一个用于`a`和一个用于`b`。 每个函数都会枚举每个字符串的子字符串，但关键是使用每个字符串`seen`设置以确保同一字符串内的重复出现不会扭曲频率计数。 

每个子字符串都由双滚动散列加上其长度表示。 包含长度是为了避免在退化情况下可能共享哈希对的不同子字符串之间发生意外冲突。 

预处理后，求解器仅扫描所有字符串中有效的子字符串`a`，并使用预先计算的计数来评估它们`b`。 

## 工作示例

 考虑第一个样本：```
a = ["abc", "cab", "aba"]
b = ["abb", "acc", "aaa"]
```我们只关心出现在所有字符串中的子串`a`。 处理后，假设`"a"`是所有三个字符串共享的唯一有效的单字符子字符串。 

| 子串| cntA | cntB | 长度|
 | --- | --- | --- | --- |
 | “一个”| 3 | 3 | 1 |
 | “b”| 2 | 2 | 1 |

 最好的有效子串是`"a"`，因为它满足`cntA == 3`。 它出现在 3 个字符串中`b`，其长度为1，产生输出`3 1`对于这个部分。 

现在考虑只有长子串能够存活的情况：```
a = ["abacaba", "abacab"]
b = ["bacaba"]
```所有子字符串都必须出现在两个字符串中`a`，这强烈限制候选者使用更长的共享核心，例如`"abaca"`或者`"bacab"`取决于重叠结构。 

| 子串| cntA | cntB | 长度|
 | --- | --- | --- | --- |
 | 马尼拉麻 2 | 0 | 5 |
 | “巴卡布”| 2 | 1 | 5 |

 这里我们选择`"abaca"`因为它最大限度地减少了出现`b`，即使长度关系可以通过最小化解决`cntB`。 

这些示例说明了如何过滤`cntA`减少优化前的搜索空间`b`。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(Σ L²) | 每个字符串通过哈希一次生成所有子字符串 |
 | 空间| O(Σ L²) | 每个字符串的每个不同子字符串在映射中存储一次 |

 所有字符串的总长度以 50,000 为界，因此即使每个字符串的二次功也保持在限制范围内。 该算法依赖于这样一个事实：子字符串枚举不会全局爆炸，只会在每个字符串本地爆炸。

 ## 边缘情况

 一种关键的边缘情况是当没有子字符串在所有字符串之间共享时`a`。 该算法自然地处理这个问题，因为没有输入密钥`cntA`达到频率`n`，离开`best_b`取消设置并正确返回`-1 -1`。 

另一种情况是只有单字符子字符串有效。 由于每个字符串都贡献自己的去重复子字符串集，因此该算法仍然可以正确计算每个字符串的出现次数，确保频率相等精确捕获共享字符。 

最后，当多个子串达到相同的最佳计数时`b`，按长度进行的平局可以确保最小子串的确定性选择，因为最终的比较明确地跟踪长度和频率。
