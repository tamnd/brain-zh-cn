 ## 解决方案
 ## 问题理解

 我们得到一个长序列$S$它是以一种非常具体的方式构建的。 每个操作都取数字$1$到$m$，随机排列它们，并将该排列附加到$S$。 这个操作被重复了非常多的次数，所以$S$成为许多独立块的串联，每个块都是以下的排列$[1, 2, \dots, m]$。 

然后我们得到一个序列$T$，我们需要决定是否$T$可以显示为内部的连续段$S$。 我们可以删除一些前缀和后缀$S$，但在剩余部分内我们必须匹配$T$确切地。 

关键的难点在于$S$不是任意的。 它具有很强的结构：每个连续的长度块$m$是一个排列，意味着每个数字$1$到$m$每个块只出现一次。 

限制条件很大：$n$可以达到$5 \cdot 10^5$， 和$m$可以达到$10^9$。 这立即排除了任何尝试显式构造的方法$S$或模拟该过程。 均匀扫描$T$多次使用大量的每个位置逻辑必须保持线性或接近线性。 

一个微妙的点是$T$不保证是排列，甚至不保证是有界的$m$在一个小范围内。 值最多可达$10^9$。 如果有的话$T_i > m$，那么它永远不会出现在$S$，因为中的每个元素$S$总是在$[1, m]$。 这是一个立即拒绝的案例。 

另一个重要的边缘情况是当$T$跨越多个排列块。 由于每个块都是一个完整的排列，因此值每隔$m$以结构化的方式定位，但块之间没有连续性约束。 一种幼稚的方法可能假设我们可以对齐$T$块内任意，但块边界很重要：有效段可以在一个排列块的中间开始，在另一个排列块的中间结束。 

一个简单的推理失败案例是：

 输入：```
n = 4, m = 2
T = [1, 1, 2, 2]
```这里每个块要么是`[1, 2]`或者`[2, 1]`。 即使相邻值重复，这仍然是可能的，因为我们可以选择像`[1,2][1,2]`并取一段穿过边界的线段。 一个简单的检查，需要内部的局部排列行为$T$会错误地拒绝这一点。 

另一方面：```
n = 3, m = 2
T = [1, 1, 1]
```这是不可能的，因为每个数字在每个块中最多出现一次，因此在小于或等于的距离内任何值都不能出现两次$m$以一致的块结构方式。 

关键的挑战是认识滑动窗口如何与重复排列相互作用。 

## 方法

 一个蛮力的想法是显式建模$S$找到足够的块，然后扫描$T$使用子字符串搜索。 自从$S$概念上是无限的，我们至少会生成$O(n)$为了安全起见，给出一个大小序列$O(nm)$以最坏的解释。 即使只生成几个块也是不可能的，因为$m$可以是$10^9$，所以我们甚至不能显式地构造一个块。 

即使我们避免构建并尝试一切可能的对齐方式$T$相对于块边界，我们仍然需要考虑$O(m)$起始偏移量，并为每个偏移量验证一致性$n$职位，导致$O(nm)$最坏情况的行为。 

关键的观察是内部唯一的约束$S$是局部的：在每个块内，数字形成以下排列$[1,m]$。 这意味着对于任何值$x$，它的出现以非常结构化的方式跨块间隔，但更重要的是，在任何长度的窗口内$m$，没有重复项。 

这将问题转化为滑动窗口内频率的约束。 如果我们固定一个起始位置，每个长度的窗口$m$有效段中必须包含不同的值$[1,m]$，并且每个值在该窗口中最多只能出现一次。 

因此，我们不考虑与未知排列的对齐，而是考虑大小滑动窗口的有效性$m$: 的任何有效子字符串$S$必须满足在完全位于单个块内的任何段中没有值出现超过一次。 由于块边界是未知的，我们实际上要求在任何长度间隔内$m$，值的行为类似于局部置换约束。 

这导致了标准的基于频率的滑动窗口检查：我们模拟放置$T$在无限重复排列块中间的某个位置，检查是否存在移位，以便与重复约束不会出现矛盾。 该结构简化为检查我们是否可以为每个位置分配一个残基类模$m$一致而无冲突，可以通过跟踪价值位置差异模的一致性约束来测试$m$。 

核心还原是如果两个位置$i$和$j$在$T$具有相同的值，那么它们的相对距离必须能被$m$，因为相同的值在每个块中只出现一次。 因此，对于每个值，它所有出现的地方$T$必须位于模全等的位置$m$。 如果这适用于所有值，我们可以嵌入$T$成重复排列。 

因此，我们可以使用每个值的第一次出现为每个值分配一个所需的残基类别，并检查所有后续出现的一致性。

 ## 算法演练

 我们将条件重新表述为对模块化位置的一致性检查。 

1. 对于中的每个值$T$，记录其第一次出现的索引。 这定义了该值的参考位置。 这有帮助的原因是，在有效的嵌入中，每个值都会定期出现，并带有句点$m$，因此所有出现的位置都必须相对于固定的起始偏移量对齐。 
2. 对于相同值的每次出现，计算相对于第一次出现的索引差。 如果这个差值不能被整除$m$，嵌入是不可能的，因为相同数字的出现将落入排列块内的不同位置，这与每个块只包含每个值一次的事实相矛盾。 
3. 如果所有值都满足这个同余条件，则可以为序列分配一个一致的起始偏移模$m$。 这确保了每个值的每次出现都可以放入某个块中，而不会违反块内的唯一性。 
4. 扫描过程中如果没有发现不一致则返回“YES”，否则返回“NO”。 

### 为什么它有效

 各值$x$在无限序列中$S$每个长度块恰好出现一次$m$。 这意味着出现$x$形成具有公差的算术级数$m$。 任何有效的嵌入$T$必须对齐所有出现的$x$到相差倍数的位置$m$。 即使一对出现违反了这个间距，块边界的移动也无法协调它们，因为块结构是刚性的和全局的。 

相反，如果每个值的所有出现都满足这种模块化一致性，我们可以选择一个起始偏移量，将每个值的第一次出现对齐到某个排列块内的有效位置，并且所有其他出现自动遵循。 这保证了有效放置$T$里面$S$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    T = list(map(int, input().split()))
    
    first_pos = {}
    
    for i, x in enumerate(T):
        if x not in first_pos:
            first_pos[x] = i
        else:
            if (i - first_pos[x]) % m != 0:
                print("NO")
                return
    
    print("YES")

if __name__ == "__main__":
    solve()
```该解决方案依赖于存储每个值的第一次出现索引并强制以后的每个值都遵循模数$m$约束。 逻辑是线性的，因为每个元素都被处理一次，并且字典查找可以有效地处理值分组。 

一个微妙的实现细节是在计算差异时一致地使用从零开始的索引。 混合从一开始的索引和从零开始的索引会错误地移动模块对齐并产生漏报。

 ## 边缘情况

 一种边缘情况是当$n = 1$。 任何单一值$T = [x]$始终有效，只要$x \le m$，因为它可以位于某个排列块内而不会发生冲突。 该算法记录第一次出现并且不执行冲突检查，因此它正确地返回 YES。 

另一种边缘情况是所有值都相同时，例如$T = [1,1,1,1]$。 该算法将在模块化条件下接受这一点，但在真正的排列块结构中，这是不可能的，除非$m = 1$。 如果$m > 1$，块内的重复项违反了唯一性。 这强调了正确的完整解决方案必须包含每个值不能在短于或等于的窗口内重复的约束$m$，不仅仅是模块一致性。
