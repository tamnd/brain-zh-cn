 ## 解决方案
 ## 问题理解

 我们有一组节点，每个节点代表一个恶魔。 每个节点都有所需的强度阈值、奖励强度增加和一组先决“密钥”。 只有同时满足两个条件，节点才能尝试：已从先前击败的节点获得其所有先决密钥，并且当前强度至少达到其要求。 

当一个节点被击败时，它将被从考虑中删除，它贡献其索引作为可以解锁其他节点的密钥，并通过其奖励值增加当前强度。 该过程从初始强度开始并贪婪地继续，直到没有其他节点可以被击败。 

关键的结构约束是键依赖图是有向无环图。 这消除了循环锁定的可能性，因此一旦节点在密钥方面变得可用，它就永远不会再次失效。 

约束达到多达二十万个节点和二十万个总依赖边。 任何重复扫描所有节点或每一步从头开始重新计算可行性的解决方案都会失败，因为这在最坏的情况下会导致二次行为。 这立即将解决方案推向维护增量结构，通常是优先级队列或通过 DAG 进行类似 BFS 的传播。 

当多个节点同时可用但由于强度而目前只有一些节点可以承受时，就会出现微妙的故障情况。 以任意顺序选择它们可能会导致死胡同。 

考虑这种情况：当前强度为 10，并且有两个节点可用，一个需要 10 强度并提供 +1，另一个需要 1 强度并提供 +100。 如果我们错误地首先选择了 +1 节点，即使存在更好的排序，我们也可能无法到达稍后需要稍高阈值的节点。 这说明仅有可行性是不够的，我们必须仔细选择下一步处理哪个可用节点。 

## 方法

 直接模拟很容易描述。 我们重复扫描所有节点，每当一个节点收集到所有密钥并且当前强度满足其要求时，我们就会击败它，更新强度，然后重复。 这是正确的，因为它完全遵循规则。 

这种方法的问题是性能。 每次失败可能需要扫描所有 n 个节点以找到有效的候选者。 如果节点数高达 200,000 个，这会导致每步大约有 n 个操作，而最多 n 个步骤，导致二次复杂度远远超出极限。 

关键的观察结果是关键依赖项形成 DAG，因此“变得可用”是由入度减少驱动的单调事件。 一旦满足节点的先决条件，就可以将其插入到候选池中一次。 从那时起，唯一剩下的限制就是是否满足其所需的强度。 

这将问题分成两个独立的过滤器。 第一个过滤器是结构性的，通过拓扑解锁来处理。 第二个过滤器是数字过滤器，由强度阈值处理。 我们可以维护一个未锁定的节点池，并在当前满足要求的节点中动态选择。 

为了提高选择效率，我们维护两个优先级结构。 One 存储按要求订购的解锁节点，因此我们可以快速将新的负担得起的节点移至活动池中。 第二个商店目前可负担的节点按奖励强度增益排序，因此我们总是在目前可行的情况下选择最有利的下一步。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n²) | O(n + m) | 太慢了|
 | 双优先级队列优化| O((n + m) log n) | O((n + m) log n) | O(n + m) | 已接受 |

 ## 算法演练

我们处理依赖图，同时不断维护当前可到达且可负担的节点集。 

1. 根据每个节点的关键要求计算入度。 构建从每个键到它解锁的节点的邻接列表。 入度为零的节点最初被解锁。 
2. 将所有最初解锁的节点插入按其所需强度排序的结构中。 这些节点在结构上是可用的，但不一定是负担得起的。 
3. 从初始值开始，保持当前强度为变量。 
4. 重复将所需强度至多为当前强度的节点从解锁结构移动到按奖励增益排序的第二个结构中。 这确保我们只考虑此时实际可以击败的节点。 
5. 如果可负担结构为空，则过程终止，因为没有剩余节点可以以当前强度被击败。 
6. 否则，选择奖励增益最高的节点，击败它，记录在答案中，并增加当前强度。 
7. 对于依赖于它的每个节点，减小度数。 如果任何依赖节点达到零入度，则将其插入到未锁定的结构中。 
8. 从步骤 4 开始重复，直到无法再继续。 

正确性取决于这样一个事实：一旦节点解锁（收集所有密钥），它就永远保持解锁状态，并且一旦变得可以承受，它就仍然可以承受，因为强度只会增加。 

### 为什么它有效

 任意时刻，算法维护两个集合：满足结构先决条件的节点和其中满足强度要求的节点。 原始问题中任何有效的下一步都必须来自它们的交集。 

该过程永远不会忽略有效的候选者：每个节点在结构上变得可能时准确地进入解锁集，并在数字上变得可能时准确地进入负担得起的集。 在负担得起的节点之间的选择总是选择一个能够最大化即时强度增益的节点，并且由于强度是单调的，一旦它变得负担得起，延迟具有更高奖励的节点就永远不会有好处。 

这确保了如果给定状态存在任何有效失败序列，算法不会阻止自身随后达到相同或更强的状态。 

## Python 解决方案```python
import sys
import heapq
input = sys.stdin.readline

n, p = map(int, input().split())
a = list(map(int, input().split()))
b = list(map(int, input().split()))

g = [[] for _ in range(n)]
indeg = [0] * n

for i in range(n):
    parts = list(map(int, input().split()))
    k = parts[0]
    for x in parts[1:]:
        x -= 1
        g[x].append(i)
        indeg[i] += 1

ready = []
for i in range(n):
    if indeg[i] == 0:
        heapq.heappush(ready, (a[i], i))

affordable = []
visited = [False] * n
ans = []

while True:
    while ready and ready[0][0] <= p:
        ai, i = heapq.heappop(ready)
        heapq.heappush(affordable, (-b[i], i))

    if not affordable:
        break

    negb, i = heapq.heappop(affordable)
    if visited[i]:
        continue

    visited[i] = True
    ans.append(i + 1)
    p += b[i]

    for to in g[i]:
        indeg[to] -= 1
        if indeg[to] == 0:
            heapq.heappush(ready, (a[to], to))

print(len(ans))
print(*sorted(ans))
```实现首先从键关系构建依赖图，将每个键视为边先决关系。 入度跟踪每个节点仍然缺少多少个键。 

这`ready`堆存储满足关键条件的节点，按所需强度排序，以便我们可以有效地提取随着强度增加而可能变得可用的候选节点。 

这`affordable`堆存储结构上已解锁且当前可击败的节点，按负奖励排序，使其表现为最大堆。 这确保我们始终选择最有利的立即升级。 

主循环首先推广新的可负担节点，然后选择最佳候选节点进行击败。 每次失败后，我们都会传播密钥解锁并可能将新节点引入系统。 

对最终答案进行排序是必要的，因为不保证选择顺序是数字。

 ## 边缘情况

 当节点在结构上较早解锁但直到很晚之后才可负担时，就会出现关键的边缘情况。 该算法确保此类节点保留在`ready`堆并且仅移入`affordable`一旦达到强度阈值就会堆起来，因此它们永远不会丢失或过早丢弃。 

另一种情况是当多个节点同时解锁时，只有一个节点会导致足够的强度增长来解锁其余节点。 由于选择是由负担得起的节点之间的奖励最大化驱动的，因此算法始终优先考虑增长路径，确保延迟但必要的解锁链最终被激活。 

最后的边缘情况是所有剩余节点都已解锁，但没有一个节点是负担得起的。 在这种情况下，就绪堆可能仍包含节点，但没有一个通过强度过滤器，因此算法正确终止，而不会尝试无效转换。
