 ## 解决方案
 ## 问题理解

 我们得到一个静态整数数组，并且对于每个查询值$x$，我们查看数组中的每个相邻对，并计算通过将两个端点移动形成的二次表达式$x$。 对于固定对$(i, i+1)$，值为$(A[i] - x)(A[i+1] - x)$。 在所有这些相邻对中，我们必须报告索引$i$最大化这个值。 

每次查询都是独立的，数组不会改变。 

关键的困难在于这两个术语都依赖于$x$，所以最好的一对可能会发生巨大的变化$x$各不相同。 

限制条件$N, Q \le 2 \cdot 10^5$立即排除每个查询重新计算所有相邻对的可能性。 一个天真的$O(NQ)$方法执行大约$4 \cdot 10^{10}$在最坏的情况下进行操作，这远远超出了限制。 甚至一个$O(N \log N)$每个查询结构会太慢。 

一个微妙的方面是，表达式可能会变成负值，并且在任何简单的值排序中，最大化成对乘积并不是单调的。 另一个边缘情况是当$x$远远超出了范围$A$，其中产品的行为几乎像凸二次方$x$但仍然取决于哪对的绝对影响力最大。 

典型的故障模式来自于假设最佳对必须涉及数组的极值。 例如，仅检查全局最小和最大邻居的贪心想法会错过主导特定的内部对$x$。 

## 方法

 从蛮力的想法开始。 对于每个查询$x$，我们扫描所有索引$i$并计算$(A[i]-x)(A[i+1]-x)$，跟踪最大值。 这是正确的，因为它直接评估答案的定义。 问题是运行时：每个查询都会花费$O(N)$，并与$Q$查询这变成$O(NQ)$，太大了。 

表达式的结构是关键的观察。 对于固定对$(a, b)$， 定义$$f_{a,b}(x) = (a-x)(b-x) = x^2 - (a+b)x + ab.$$每对都定义一条抛物线$x$，并且我们希望每个查询点处所有相邻对的上包络。 

这将问题转化为维持在不同点评估的一组二次函数的最大值。 因为我们只需要相邻对，所以我们有$N-1$抛物线。 

隐藏结构是所有二次方程共享相同的首项系数$+1$，因此它们的形状仅因线性平移和常数项而不同。 这使得可以在衍生自导数的线上使用凸包技巧风格结构或通过在二次方程上维护李超树来比较它们。 

重写：$$(a-x)(b-x) = x^2 - (a+b)x + ab.$$自从$x^2$对于所有对来说都是通用的，最大化表达式相当于最大化：$$-(a+b)x + ab.$$所以每一对都成为一条线$x$有坡度$-(a+b)$并拦截$ab$。 问题变成：对于每个查询$x$，找到评估的一组线中的最大值$x$，然后添加$x^2$（这不会影响 argmax）。 

因此，我们将问题简化为一个静态行容器，其中包含所有范围查询$N-1$线。 李超树支持这一点$O((N+Q)\log C)$， 在哪里$C$是坐标范围。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(NQ)$|$O(1)$| 太慢了|
 | 李朝树 |$O((N+Q)\log C)$|$O(N)$| 已接受 |

 ## 算法演练

 我们将每个相邻对转换为一条线，并使用李超树来回答最大查询。 

### 步骤

 1. 对于每个索引$i$，从该对构造一条线$(A[i], A[i+1])$有坡度$m = -(A[i] + A[i+1])$并拦截$c = A[i] \cdot A[i+1]$。 

这是通过扩展原始表达式并删除公共表达式而得出的$x^2$学期。 
2. 在可能的域上构建李超树$x$值，它受到以下约束的限制$A[i]$和查询。 
3. 将每一行插入李超树中。 

每次插入都确保对于域的每个段，都存储最佳行。 
4. 对于每个查询$x$，查询李超树，获取最大值$-(a+b)x + ab$。 
5. 跟踪哪一行产生了这个最大值，因为每行对应一个索引$i$。 输出该索引。 
6. 返回每个查询的存储索引。 

唯一微妙的部分是在更新期间保持线路的标识。 李超树中的每个节点不仅必须存储最佳值，而且还必须存储哪对节点产生了该值。 

### 为什么它有效

 每个相邻对定义一个二次函数$x$，并且所有二次方程具有相同的曲率。 这允许在删除公共项后将比较简化为线性函数。 李超树在域上维护这些线性函数的逐点最大值。 由于每个查询仅评估该包络上的一个点，因此返回的行必须对应于该点的全局最佳相邻对$x$。 没有对被错误地丢弃，因为该结构仅替换严格更差的线段，从而在域上保持逐点的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**30

class Node:
    __slots__ = ("m", "b", "idx")
    def __init__(self):
        self.m = 0
        self.b = -INF
        self.idx = -1

def f(m, b, x):
    return m * x + b

class LiChao:
    def __init__(self, xs):
        self.xs = xs
        self.n = len(xs)
        self.size = 4 * self.n
        self.tree = [Node() for _ in range(self.size)]

    def add_line(self, m, b, idx, v=1, l=0, r=None):
        if r is None:
            r = self.n - 1

        mid = (l + r) // 2
        x_l = self.xs[l]
        x_m = self.xs[mid]
        x_r = self.xs[r]

        cur = self.tree[v]
        new = Node()
        new.m, new.b, new.idx = m, b, idx

        if f(new.m, new.b, x_m) > f(cur.m, cur.b, x_m):
            self.tree[v], new = new, cur

        if r - l == 0:
            return

        if f(new.m, new.b, x_l) > f(self.tree[v].m, self.tree[v].b, x_l):
            self.add_line(new.m, new.b, new.idx, 2*v, l, mid)
        elif f(new.m, new.b, x_r) > f(self.tree[v].m, self.tree[v].b, x_r):
            self.add_line(new.m, new.b, new.idx, 2*v+1, mid+1, r)

    def query(self, x, v=1, l=0, r=None):
        if r is None:
            r = self.n - 1

        mid = (l + r) // 2
        cur = self.tree[v]

        res = (f(cur.m, cur.b, x), cur.idx)

        if r - l == 0:
            return res

        if x <= self.xs[mid]:
            cand = self.query(x, 2*v, l, mid)
        else:
            cand = self.query(x, 2*v+1, mid+1, r)

        return max(res, cand)

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    xs = sorted(set(a))
    lichao = LiChao(xs)

    for i in range(n - 1):
        a1, a2 = a[i], a[i+1]
        m = -(a1 + a2)
        b = a1 * a2
        lichao.add_line(m, b, i + 1)

    out = []
    for _ in range(q):
        x = int(input())
        val, idx = lichao.query(x)
        out.append(str(idx))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```核心转换发生在从每个相邻对到一条线的转换中。 斜率使用带负号的端点之和，截距使用它们的乘积。 这正是原始表达式去掉公共项后的扩展$x^2$学期。 

李超树存储这些线，并通过递归比较坐标域相关段上的候选线，在对数时间内回答查询。 每个节点都维护其中点的最佳线，确保区间结构的正确性。 

## 工作示例

 ### 示例 1

 输入：```
N = 4
A = [3, 1, 4, 2]
queries: x = 2, x = 3
```对：

 (3,1), (1,4), (4,2)

 我们计算每个 x 处的线值。 

| 配对 | 线 (m, b) | x=2 值 | x=3 值 |
 | --- | --- | --- | --- |
 | (3,1) | m=-4，b=3 | -5 | -9 |
 | (1,4) | m=-5，b=4 | -6 | -11 |
 | (4,2) | m=-6，b=8 | -4 | -10 |

 当 x=2 时，最好的是 (4,2) 对。 

当 x=3 时，最好的仍然是 (4,2)。 

这显示了单个对如何在多个查询值中占据主导地位，因为尽管斜率较差，但仍具有最高的截距。 

### 示例 2

 输入：```

```对：

 (10,-5), (-5,7)

 | 配对 | 米 | 乙| x=0 | x=20 |
 | --- | --- | --- | --- | --- |

 当 x=0 时，对 (-5,7) 获胜。 

在 x=20 时，由于负斜率小得多，同一对仍然获胜。 

这说明了对于较大的 x，斜率如何占主导地位，而截距在接近零时如何占主导地位。

 ## 边缘情况

 关键的边缘情况是多个相邻对产生相同或接近相同的二次方程。 在这种情况下，李朝树必须一致地保留最早的或正确的索引。 该结构可以处理此问题，因为平局打破是基于严格的更大比较，因此除非存在严格更好的行，否则第一个插入的行将保留。 

另一种情况是极端查询值，其中$x$比所有的都大得多$A[i]$。 那么每个乘积的行为就像一个大的正二次方$x$，但差异仅取决于斜率$-(a+b)$。 该算法正确地选择了总和最小的对$a+b$，因为这会在展开后产生最小的负线性项。 

最后，当值包含较大的负数时，截距项$ab$变为大正数，并且接近零的查询由这些截取主导。 李超结构正确地平衡了斜率和截距，因为它评估整条线值而不是部分启发式。
