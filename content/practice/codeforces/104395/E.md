 ## 解决方案
 ## 问题理解

 我们得到了一组活动，每项活动都由持续时间（以小时为单位）和幸福值来描述。 Trotles 正好有三天假期，每天他有 16 个小时的可用时间。 他可以选择任何活动子集，但每个选定的活动必须在一天内完全完成。 他不能将一项活动分成几天进行，并且每项活动最多只能使用一次。 我们的目标是将活动分配到三个 16 小时的日子中，独立尊重每一天的能力，从而使所有选定活动的总幸福感最大化。 

这不仅仅是一个背包问题，因为容量约束重复了三次。 每天就像一个容量为 16 的独立背包，每项活动最多只能放入其中一个背包中。 

限制很小：最多 1000 个活动，每个活动的大小最多为 16。所有天的总容量实际上是 48，但关键的限制是跨天的分配很重要，而不仅仅是总容量。 这推动我们采用动态编程解决方案，跟踪每天的填充量。 

一种天真的解释是尝试将每项活动分配给四个状态之一，要么未使用，要么放置在第 1 天、第 2 天或第 3 天，然后检查可行性。 这导致每个项目的指数分支因子为 4，这在 N = 1000 时很快变得不可行。 

当所有活动都很大（接近 16 小时）时，就会出现微妙的边缘情况。 先填满一天的贪婪策略可以在以后阻止更好的分配。 例如，考虑活动 (16, 10)、(8, 9)、(8, 9)、(8, 9)。 贪婪填充可能会将 16 小时的活动放在第一位，消耗一整天的时间并迫使进行次优打包，而最佳解决方案将 8 小时的活动分成几天，以实现更高的总体幸福感。 

当存在许多小型活动时，会出现另一种边缘情况。 尽管每个都很小，但它们在多天内的组合会产生贪婪排序无法捕获的组合放置选择。 

## 方法

 蛮力方法尝试将每项活动分配给四个选择之一：跳过它，或者如果适合剩余容量，则将其放入三天之一。 这形成了 N 个项目的递归分支过程。 即使进行剪枝，在最坏的情况下状态空间也会大致增长 4^N，这远远超出了可行的计算范围。 

关键的结构观察是，除了每个项目只能使用一次的共享约束之外，每天的容量相同且独立。 这表明我们正在将物品装入多个相同的背包中。 我们可以将流程视为首先构建第 1 天，然后是第 2 天，然后是第 3 天，而不是跟踪完整的分配，从而确保每个项目在全局范围内仅使用一次。 

这导致了一个三层动态编程结构，我们每天按顺序解决背包问题，更新哪些物品可用。 然而，明确跟踪每天的“使用过的物品”是昂贵的。 关键的简化是，如果我们以固定顺序处理项目并通过仔细传播状态来确保每个项目最多使用一次，则不需要跟踪精确的项目身份转换。 

我们根据天数和容量定义 DP，并且对于每个项目，我们决定将其分配到多少天（0 或 1，因为它不能重复使用）。 我们逐一处理项目，并根据三天中每一天的充实程度维护 3D DP。 

这将问题简化为有界多维背包，其中每个状态跟踪三个独立箱的剩余容量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(4^N) | O(4^N) | O(N) | 太慢了|
 | 最优DP（3D背包）| O(N × 16³) | O(16立方) | 已接受 |

 ## 算法演练

我们将问题建模为填充三个背包，每个背包容量为 16，每个物品最多使用一次。 

1. 定义一个 DP 数组，其中 dp[a][b][c] 表示第 1 天使用 a 小时、第 2 天使用 b 小时、第 3 天使用 c 小时时可实现的最大幸福感。 此状态完全捕获每天剩余的容量。 
2. 初始化 dp，将所有值设置为非常负的数字，除了 dp[0][0][0] 为 0。这表示从没有分配任何活动且幸福感为零开始。 
3. 逐项处理每项活动。 对于时间为 t 且幸福度为 h 的活动，我们尝试将其放入三天中的任何一天。 
4. 对于每个状态 (a, b, c)，我们考虑转换：

 如果a + t ≤ 16，我们可以更新dp[a + t][b][c]。 

如果 b + t ≤ 16，我们可以更新 dp[a][b + t][c]。 

如果c + t ≤ 16，我们可以更新dp[a][b][c + t]。 

我们小心地使用临时 DP 数组进行转换，以便每个项目最多使用一次。 
5. 处理完所有项目后，答案是所有 dp[a][b][c] 状态的最大值。 

关键细节是我们必须反向迭代状态或使用 DP 数组的副本进行转换。 否则，一个项目可能会在同一迭代中多次重复使用。 

为什么有效：每个 DP 状态都将唯一有效的部分项目分配编码到三天中。 每个转换对应于将当前项目恰好分配给某一天或跳过它。 因为我们在处理某个项目时仅从先前的状态进行转换，所以不会对任何项目进行多次计数。 DP 将所有有效的物品划分探索到三个有界背包中，确保最佳组合包含在状态中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

CAP = 16
NEG = -10**18

N = int(input())
items = [tuple(map(int, input().split())) for _ in range(N)]

dp = [[[NEG] * (CAP + 1) for _ in range(CAP + 1)] for _ in range(CAP + 1)]
dp[0][0][0] = 0

for t, h in items:
    new_dp = [[[NEG] * (CAP + 1) for _ in range(CAP + 1)] for _ in range(CAP + 1)]
    
    for a in range(CAP + 1):
        for b in range(CAP + 1):
            for c in range(CAP + 1):
                if dp[a][b][c] == NEG:
                    continue

                val = dp[a][b][c]

                # skip
                if val > new_dp[a][b][c]:
                    new_dp[a][b][c] = val

                # place in day 1
                if a + t <= CAP:
                    if val + h > new_dp[a + t][b][c]:
                        new_dp[a + t][b][c] = val + h

                # place in day 2
                if b + t <= CAP:
                    if val + h > new_dp[a][b + t][c]:
                        new_dp[a][b + t][c] = val + h

                # place in day 3
                if c + t <= CAP:
                    if val + h > new_dp[a][b][c + t]:
                        new_dp[a][b][c + t] = val + h

    dp = new_dp

ans = 0
for a in range(CAP + 1):
    for b in range(CAP + 1):
        for c in range(CAP + 1):
            ans = max(ans, dp[a][b][c])

print(ans)
```该实现显式地复制每个项目的 DP 表，这保证每个转换阶段每个活动都被恰好考虑一次。 三重嵌套循环反映了三个独立的日容量，并且转换编码了将当前活动分配给任何一天或跳过它的决定。 

使用大负值进行初始化可确保无效状态永远不会对转换产生影响。 所有状态的最终最大值是必需的，因为我们不需要完全填满所有的日子，只需要最大化幸福感。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    
    CAP = 16
    NEG = -10**18

    N = int(sys.stdin.readline())
    items = [tuple(map(int, sys.stdin.readline().split())) for _ in range(N)]

    dp = [[[NEG] * (CAP + 1) for _ in range(CAP + 1)] for _ in range(CAP + 1)]
    dp[0][0][0] = 0

    for t, h in items:
        new_dp = [[[NEG] * (CAP + 1) for _ in range(CAP + 1)] for _ in range(CAP + 1)]
        for a in range(CAP + 1):
            for b in range(CAP + 1):
                for c in range(CAP + 1):
                    if dp[a][b][c] == NEG:
                        continue
                    val = dp[a][b][c]
                    new_dp[a][b][c] = max(new_dp[a][b][c], val)
                    if a + t <= CAP:
                        new_dp[a + t][b][c] = max(new_dp[a + t][b][c], val + h)
                    if b + t <= CAP:
                        new_dp[a][b + t][c] = max(new_dp[a][b + t][c], val + h)
                    if c + t <= CAP:
                        new_dp[a][b][c + t] = max(new_dp[a][b][c + t], val + h)
        dp = new_dp

    ans = max(max(max(row) for row in plane) for plane in dp)
    return str(ans)

 ## 边缘情况

 一种极端情况是当一项活动恰好占据一天时。 用于输入`1\n16 50\n`，DP 从 (0,0,0) 转换到 (16,0,0)、(0,16,0) 或 (0,0,16)。 所有这些都代表有效的等效放置，并且算法在所有对称状态中正确保留最大值 50。 

另一个极端情况是所有活动太大而无法在一天内合并。 例如，多个 16 小时的活动迫使总共最多选择三个项目。 DP 自然会限制放置，因为任何额外的分配都会超出容量，并且永远不会创建这些转换。 

第三种情况是小件物品很多，但一天就可以装满。 DP 仍然考虑将它们分散到几天内，但由于各州是独立的，它还捕获了最佳包装，其中所有物品都在一天或多天内不受干扰。
