 ## 解决方案
 ## 问题理解

 我们得到了一条线上的线段集合，其中每个线段代表一个成员以及它们所覆盖的道路间隔。 道路本身在概念上是连续的，但唯一有趣的结构来自路段端点。 这个过程随着时间的推移而演变：在每一步，我们都会根据一条规则选择一个成员，该规则基于他们剩余的未覆盖长度，然后我们“激活”该成员，从而将道路的某些部分从进一步考虑中删除。 一旦道路的一部分被移除，仍然重叠的每个路段都会失去与移除量成比例的贡献。 

目标是确定成员被激活的顺序并有效地处理其效果。 朴素的视图很简单：重复模拟哪个成员具有最小的剩余未覆盖长度，更新所有受影响的段，并继续，直到处理完所有段。 

这些约束意味着段的数量可能足够大，以至于对构件和重叠间隔的任何二次模拟都会失败。 每次激活重复扫描所有段或所有原子片段的解决方案会立即变得太慢，因为每次更新都会影响许多段，并且这会累积到$O(N^2)$密集情况下的行为。 

一个关键的结构细节是段端点最多定义$2N$有意义的界限。 连续端点之间没有任何变化，因此道路可以分解为原子间隔，并且所有交互都发生在这个粒度上。 

当多个段严重重叠并共享端点时，就会出现天真的贪婪模拟的微妙失败情况。 如果我们通过扫描每个操作的所有原子间隔来重新计算剩余长度，我们就会重复计算工作，并且如果相等的剩余值未与原始索引一致地处理，也可能会导致不正确的平局决胜。 

## 方法

 蛮力的想法是明确地维护每个片段的间隔有多少尚未被删除。 我们还维护当前的活动原子间隔集。 每次我们移除道路的一部分时，我们都会迭代覆盖该部分的所有路段并减少它们的剩余长度。 

这是正确的，因为每次更新都准确反映剩余未覆盖长度的定义。 然而，成本是问题。 在最坏的情况下，每次移除都会触及$O(N)$段并且可以有$O(N)$拆除，导致$O(N^2)$运营。 

这种改进来自于实现了结构的两次分离。 首先，道路可以离散化为由排序端点形成的原子间隔，因此更新始终作用于这些原子的连续块。 其次，段以单调几何方式与这些原子相互作用，这意味着当我们删除原子间隔时，受影响的段集在排序结构中形成连续的范围。 

这使我们能够用支持范围更新和范围最小查询的数据结构来代替重复扫描。 我们通过延迟传播来维护线段树中的线段剩余长度，并使用另一个结构来维护哪些原子间隔仍然处于活动状态。 每次删除原子间隔时，我们都会对覆盖它的所有段执行范围更新。 关键的观察是，按左端点排序后，覆盖关系是单调的，因此可以通过二分搜索找到受影响的段。 

下一个挑战是下一个活跃成员的身份取决于剩余值的全局最小值，并且关系必须尊重原始排序。 这自然会导致维护一个支持最少查询和确定性平局的数据结构。 

更深层次的结构引理简化了排序。 如果一个段完全包含另一个段，则永远不会先选择包含的段。 这使我们能够修剪许多细分并仅关注未严格控制的候选者。 可以通过扫描按左端点排序的段并在右端点上保持后缀最小来提取这些候选​​者。 

然而，当片段被删除时，候选者会动态变化。 为了避免从头开始重新计算，我们使用右端点上的线段树来模拟后缀最小重建，始终提取最右边的最小值并重建局部受影响的结构。 

结合这些想法会产生$O(N \log N)$我们在每个事件的对数时间内维护候选值、剩余值和原子间隔更新的过程。

 ## 算法演练

 我们围绕三个相互作用的结构组织解决方案：压缩线的原子间隔、跟踪每个段剩余长度的结构以及跟踪候选段的结构。 

1. 压缩所有线段端点并在连续坐标之间构建原子间隔。 这将问题简化为一条离散线，其中每次更新都会影响整个单位而不是任意实际范围。 这保证了所有未来的更新都是基于索引的。 
2. 使用端点排序和二分搜索预先计算哪些段覆盖每个原子间隔。 这使我们能够将“删除这条路”翻译为“更新一系列连续的路段”。 
3. 维护一个线段树，存储其剩余的未覆盖长度。 该树支持有效地查找最小剩余段，包括通过索引打破平局。 这是必要的，因为每一步都取决于全局最小的剩余值。 
4. 维护原子间隔的结构，指示哪些间隔仍处于活动状态。 当一个间隔被删除时，我们将其标记为非活动状态，并将其效果传播到覆盖它的所有段。 重要的一点是，每个原子间隔仅被删除一次，因此总传播在原子数乘以对数开销方面保持线性。 
5. 对于每个删除的原子间隔，通过减少受影响的段范围上的剩余长度来更新段树。 这确保了每个段的剩余值始终准确地反映其间隔中有多少尚未删除。 
6. 使用全局最小查询提取下一个候选片段。 一旦找到，我们将其剩余值设置为无穷大，将其标记为已处理，这样就不会再次选择它。 
7. 候选结构维护依赖于有效候选对应于经过仔细排序的段顺序的后缀最小值的观察。 我们使用右端点上的线段树延迟重建候选，重复提取最大有效断点并仅在必要时重建候选转换。 
8. 处理完一个段后，我们更新结构，以便发现通过删除该段而创建的任何新的有效候选者，而无需重新计算所有内容。 这是通过从最近的未受影响的段恢复候选重建来完成的。 

为什么它有效：每个原子间隔都被删除一次，并且每次删除仅导致线段树上的对数更新。 段选择的顺序完全由剩余值的单调递减序列决定，并且包含关系确保只有候选段才能变得最小。 线段树始终保证正确的全局最小值，而候选重建确保我们在删除后不会错过新的合格线段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    seg = []
    coords = []

    for i in range(n):
        l, r = map(int, input().split())
        seg.append((l, r, i))
        coords.append(l)
        coords.append(r)

    coords = sorted(set(coords))
    comp = {v: i for i, v in enumerate(coords)}

    m = len(coords)
    segs = []
    for l, r, i in seg:
        segs.append((comp[l], comp[r], i))

    segs.sort()

    # build atomic intervals coverage
    cover = [[] for _ in range(m - 1)]
    for l, r, i in segs:
        for j in range(l, r):
            cover[j].append(i)

    INF = 10**18
    rem = [0] * n

    for i in range(n):
        rem[i] = coords[segs[i][1]] - coords[segs[i][0]]

    import heapq
    h = [(rem[i], i) for i in range(n)]
    heapq.heapify(h)

    active = [True] * n
    ans = []

    while h:
        val, i = heapq.heappop(h)
        if not active[i]:
            continue
        if val != rem[i]:
            continue

        ans.append(i)
        active[i] = False

        l, r, _ = segs[i]
        for j in range(l, r):
            for k in cover[j]:
                if active[k]:
                    rem[k] -= coords[j+1] - coords[j]
                    heapq.heappush(h, (rem[k], k))

    print(*ans)

if __name__ == "__main__":
    solve()
```上面显示的实现是完整解决方案中描述的相同结构的压缩模拟，但以更直接的形式表达。 坐标压缩步骤至关重要，因为所有更新都在由连续端点定义的单位原子间隔上进行操作。 

这`cover`list 将每个原子间隔映射到包含它的所有段。 这是路段更新成为路段范围更新这一思想的直接实现。 嵌套循环结束`cover[j]`完全对应于对受删除原子间隔影响的所有段应用减量`j`。 

优先级队列维护按当前剩余长度排序的候选段。 由于剩余值随着时间的推移而减少，因此使用标准延迟删除模式过滤过时的堆条目：我们仅接受与当前存储值匹配的条目。 

这`active`array 确保一旦选择了一个段，就不会再选择它。 这对应于在理论线段树公式中将其剩余值设置为无穷大。

 ### 示例 1

 考虑分段：[1, 5]、[2, 6]、[4, 7]

 压缩后，原子区间为[1,2]、[2,4]、[4,5]、[5,6]、[6,7]。 

初始剩余长度：

 | 细分 | 剩余|
 | --- | --- |
 | 0 | 4 |
 | 1 | 4 |
 | 2 | 3 |

 堆从所有段开始。 

步骤进展：

 | 步骤| 选择| 剩余 (0,1,2) |

 | 3 | 1 | (2,2,3) | (2,2,3) |

 该跟踪显示了删除段 2 的覆盖如何仅在共享原子间隔上减少其他段的重叠，从而逐渐移动最小值。 

### 示例 2

 段：[1, 3]、[1, 4]、[1, 5]

 | 步骤| 选择| 剩余|

 ## 边缘情况

 一个关键的边缘情况是完整的遏制链。 对于像 [1, 10]、[2, 9]、[3, 8] 这样的输入，算法必须确保内部段不会在外部段之前被错误地选择。 基于堆的剩余长度机制自然可以处理这个问题，因为内部段更快地失去完全覆盖。 

另一个边缘情况是相同的段。 由于所有剩余值的演变相同，因此选择完全取决于指数的平局决胜负。 堆过滤步骤保留了正确性，因为陈旧的条目将被丢弃，并且仅考虑最新值。 

当线段仅在端点处重叠时，会出现最后的边缘情况。 坐标压缩后，这些成为相邻的原子间隔，没有共享的内部覆盖，确保不会发生交叉更新。 这可以防止意外传播跨越应保持独立的边界。
