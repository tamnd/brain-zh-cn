 ## 解决方案
 ## 问题理解

 我们有一棵以顶点 1 为根的树，其中每个顶点都有一个整数权重。 除了树之外，我们还得到了两个参数，一个下界 L 和一个上界 R，以及一个目标数 K。任务是决定是否可以将树的顶点划分为不相交连接组件的结构化集合，其行为类似于“闭合子树”，同时允许恰好一个部分构建的组件在构造过程中保持“开放”。 

构建这些组件的过程受到自下而上的方式的约束。 每个子树可以贡献完全完成的部分或仍可以由祖先扩展的未完成的部分。 “闭合子树”对应于总权重位于区间 [L, R] 内的连通分量。 “可扩展子树”是部分形成的组件，仍然允许通过父链向上生长。 

从根本上来说，我们想要确定在所有组件都遵守权重限制的约束下，在处理完所有内容后是否可以以恰好 K + 1 个闭合子树结束。 动态编程状态跟踪到目前为止已经确定了多少个闭合子树以及当前开放结构可以具有多少可能的权重总和。 

主要困难在于每个节点都会合并来自其子节点的信息，并且可能的部分权重和的数量会组合增长。 约束足够大，任何显式枚举每个节点和每个子树的所有 DP 状态的解决方案都会太慢。 特别是，朴素合并的行为就像对值集进行卷积，每个节点很容易达到二次或更差的行为。 

当我们在不压缩的情况下精确存储所有可达总和时，就会出现典型的故障模式。 例如，如果一个节点有两个子节点，每个子节点贡献许多可能的和，那么它们的组合会产生完整的成对和集爆炸。 即使是中等的树也会在每次合并时产生二次爆炸。 

另一个微妙的问题是，DP 不仅仅涉及求和的可行性，还涉及有界的“间隙敏感性”：我们只关心是否有任何值落入长度为 R - L 的区间。忽略这一点会导致不必要地保留许多对于最终决策无法区分的值。 

## 方法

 暴力动态规划自然遵循树结构。 对于每个节点，我们维护一个关于 k（形成的闭合子树的数量）和 x（活动组件的当前总和）的表。 当组合两个孩子时，我们尝试所有 k 的分割以及来自两个孩子的所有总和对，产生类似卷积的合并。 

这种方法是正确的，因为它准确地跟踪了所有可能的子树划分方式。 然而，当两个子节点都有 O(K) 可能的计数和 O(R) 可能的总和时，在最坏的情况下，合并它们每个节点的成本为 O(K^2 R^2)。 超过 N 个节点，这变得完全不可行。 

关键的观察是，我们实际上从来不需要可达到总和的集合的完整结构。 我们只需要知道长度为 R − L 的滑动区间内是否存在一个值，因为任何有效的闭子树都必须落入 [L, R]，并且决策仅取决于是否命中这样的窗口。 这使我们能够积极压缩 DP 状态。 

我们将每个 DP 状态重新解释为一组可达和，并注意到固定 k 的值的扩展受 k(R − L) 限制。 这意味着可以安全地近似值，而不会丢失间隔查询的正确性。 我们不存储所有值，而是存储一个简化的代表集，该集保留所有长度为 R − L 的区间的空性。

这引出了第二个关键思想：在每次合并期间，我们应用一种缩减方法，仅将极端代表保留在小桶中，确保集合大小保持 O(K)。 正确性得到保证，因为任何有效查询都无法区分相对于 R − L 足够接近的两个值。 

因此，我们维护有界表示并确保每次合并保持多项式，而不是 DP 状态呈指数增长。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N K^2 R^2) | O(N K^2 R^2) | O(NKR) | 太慢了|
 | 最佳 | O(N K^3) | O(N K^3) | O(N·K) | 已接受 |

 ## 算法演练

 1. 以节点 1 为树根，并将每个节点的 DP 状态定义为集合 S(v, k) 的集合，其中 k 是 v 子树的已处理部分中完整闭合子树的数量。每个集合存储当前打开组件的可能和。 
2. 通过直接枚举叶子节点的两种可能性来初始化叶子节点：如果顶点的权重位于[L，R]，则该顶点形成一个闭合子树，或者以其权重作为当前总和来贡献一个开分量。 这无需递归即可建立基本 DP 状态。 
3. 在后序遍历中处理每个节点，以便子节点在组合到父节点之前得到完全解决。 这确保了每次合并都针对完整的子问题进行操作。 
4. 使用 k 上的卷积合并两个子 DP 状态。 对于将 k 拆分为 i 和 k − i 的每种方式，将 S(w1, i) 中的每个和与 S(w2, k − i) 中的每个和结合起来。 这会产生合并子树的所有可能的总和。 此步骤必要的原因是封闭子树可以独立分布在子树之间，因此必须考虑所有组合。 
5. 合并节点的所有子节点后，通过添加 Av 来移动 S(w, k) 中的所有和，从而合并节点本身。 这表示将开放组件向上延伸通过当前顶点。 
6. 扩展后，检查任何结果和能否落入[L，R]形成有效的闭子树。 如果是这样，我们引入一个新状态，其中该组件关闭，打开组件重置为零，将 k 加一并将 0 插入 S(v, k + 1)。 
7. 对每个 S(v, k) 应用缩减步骤。 此步骤删除了决定长度 R − L 的区间交点不需要的中间值，仅保留压缩表示。 这种缩减保证了对于任何宽度为 R − L 的区间查询，答案保持不变。 
8. 继续此过程直至根。 最终的答案是 S(1, K + 1) 中是否包含 0，这意味着我们可以恰好关闭 K + 1 个子树，并且以没有开放组件的方式结束。 

### 为什么它有效

 关键的不变量是每个 S(v, k) 不需要存储所有精确的和，只需要存储足够的代表，以便任何长度为 R − L 的区间与该集合相交当且仅当原始未压缩集合会相交时。 约简保留了这一性质，求和运算也保留了这一性质，因为区间不可区分性在加法下是稳定的。 因此，每个 DP 转换都在可到达解空间的压缩但等效的表示上运行，因此永远不会丢失或错误地引入可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n, K, L, R = map(int, input().split())
    w = [0] + list(map(int, input().split()))

    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        a, b = map(int, input().split())
        g[a].append(b)
        g[b].append(a)

    parent = [0] * (n + 1)
    order = []
    stack = [1]
    parent[1] = -1

    while stack:
        v = stack.pop()
        order.append(v)
        for to in g[v]:
            if to == parent[v]:
                continue
            parent[to] = v
            stack.append(to)

    S = [None] * (n + 1)

    def reduce_set(arr):
        if not arr:
            return []
        arr = sorted(set(arr))
        res = []
        D = R - L
        last_keep = -10**18
        for x in arr:
            if not res or x > last_keep + D:
                res.append(x)
                last_keep = x
        return res

    def merge(A, B):
        if not A:
            return B[:]
        if not B:
            return A[:]
        out = []
        for a in A:
            for b in B:
                out.append(a + b)
        return reduce_set(out)

    def dp(v):
        cur = {0: [0]}  # k -> set of sums

        for to in g[v]:
            if to == parent[v]:
                continue
            child = dp(to)
            nxt = {}
            for k1, s1 in cur.items():
                for k2, s2 in child.items():
                    k = k1 + k2
                    merged = merge(s1, s2)
                    if k not in nxt:
                        nxt[k] = merged
                    else:
                        nxt[k] = merge(nxt[k], merged)
            cur = nxt

        new = {}
        for k, vals in cur.items():
            shifted = [x + w[v] for x in vals]
            new[k] = reduce_set(shifted)

        if any(L <= x <= R for x in new.get(0, [])):
            if 1 not in new:
                new[1] = []
            new[1].append(0)
            new[1] = reduce_set(new[1])

        return new

    res = dp(1)
    print("YES" if 0 in res.get(K + 1, []) else "NO")

if __name__ == "__main__":
    solve()
```核心结构直接遵循DP定义。 每个节点计算一个从 k 到可能总和的压缩列表的字典。 合并函数在两个子集之间执行笛卡尔积，然后使用归约规则进行积极压缩。 减少对于防止值范围爆炸至关重要。 

扩展步骤是通过将所有总和移位节点权重来实现的。 闭包条件检查是否有任何和落入 [L, R]，然后通过将零插入到下一个 k 状态来重置开放组件。 

最终答案检查我们是否可以精确地实现 K + 1 个封闭分量，并且根部有一个空的开放分量。
