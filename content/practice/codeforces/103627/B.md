 ## 解决方案
 ## 问题理解

 我们正在与一家$n \times n$网格，其中每个单元格可以填充或留空。 目标是构建一种空单元格图案，避免形成任何完整的“宾果线”，其中宾果线对应于完全空的行、完全空的列或完全空的主对角线或反对角线（取决于此处使用的构造中的解释）。 

与其从游戏规则的角度思考，不如将任务重新表述为组合构造问题更有用。 我们希望在一个单元格中放置尽可能多的空单元格$n \times n$网格，同时确保没有行、列或对角线完全为空。 每行和每列必须至少包含一个填充单元格。 

一个简单的计数参数给出了我们可以放置多少个空单元格的上限。 由于每个$n$行必须至少包含一个填充单元格，我们至少需要$n$填充细胞总数。 这立即意味着空单元格的数量最多为$n^2 - n$。 关键问题是这个界限是否真的能够实现。 

限制是最小的，因为问题纯粹是建设性的并且仅取决于$n$。 这意味着我们不关心通常意义上的渐近性能，而是关心我们是否可以为所有值生成有效的配置$n$以及如何明确地构建它。 

唯一微妙的边缘情况是$n = 2$。 在这种情况下，行、列和对角线的结构太密集，不可能达到上限。 快速手动检查显示，任何放置两个空单元格的尝试都会强制整行或整列空单元格。 

例如，当$n = 2$，网格有 4 个单元格。 如果我们尝试留下 2 个空单元格，我们将被迫进行如下配置：```
..
##
```或者```
.#
.#
```在这两种情况下，至少一行或一列会完全变空或违反预期的约束结构。 所以$n = 2$必须单独对待。 

对于所有其他值$n$，任务简化为找到一种系统模式，在每行和每列放置一个填充单元格，同时保持结构一致。 

## 方法

 暴力方法会尝试所有单元格子集，并检查是否有任何行、列或对角线完全为空。 这涉及到选择$k$之间的空单元格$n^2$职位，与$k$可能接近$n^2$。 配置数量呈指数增长$\binom{n^2}{k}$，即使对于小规模来说也是不可行的$n$。 正确性很简单，因为它直接测试所有可能性，但运行时间会立即超出非常小的网格。 

关键的结构观察是“每行必须包含至少一个填充单元格”的约束已经规定了一种非常严格的形式。 我们可以构建一种模式，其中每行只分配一个填充单元格，其余单元格为空，而不是事后考虑避免错误的配置。 这会自动保证上限$n^2 - n$。 

剩下的困难是确保列和对角线不会意外地完全变空或形成意外的图案。 简单的对角线结构的工作原理是沿着移动的对角线放置填充单元格，以便每一行和每一列都恰好有一个填充单元格。 这对应于填充位置$(i, i)$或它的移动版本。 

然而，当更严格地考虑对角线约束时，这种朴素的对角线模式会失败，特别是对于偶数$n$，其中一条主对角线可能会完全变空，具体取决于结构的对齐方式。 解决方法是通过交换第一行和最后一行的位置来稍微打破对称性，这确保两个对角线在空时都不会完全均匀。 

最后的例外是$n = 2$，如果没有这样的重新排列就无法避免整行，因此必须单独处理。

 ## 算法演练

 我们构造一个$n \times n$网格并明确决定哪些单元格被填充，哪些单元格为空，以便每一行中恰好出现一个填充的单元格。 

1. 从概念上充满空单元格的空网格开始。 目标是确定填充单元格的位置，以便每一行和每一列都有一个填充单元格。 这保证了空单元格的最大数量。 
2. 对于$n = 2$，直接输出唯一有效的配置，避免形成完整的空行或列，因为没有构造可以达到上限。 
3.为了$n \ge 3$，沿对角线图案放置填充的单元格，但移动以便该行$i$在列中获取一个已填充的单元格$i$对于大多数行。 这会自动确保列的唯一性。 
4. 通过交换第一行和最后一行的填充位置来修改第一行和最后一行，从而破坏主对角线结构。 这可以防止任何对角线由于对称性而意外变空。 
5. 放置填充的单元格后，将所有剩余的单元格标记为空。 由于每一行都包含一个填充单元格，因此每一行都贡献$n - 1$空单元格，给出总计$n(n - 1) = n^2 - n$。 

### 为什么它有效

 该构造强制执行比所需更强的不变量：每一行恰好包含一个填充单元格，每一列也恰好包含一个填充单元格。 这意味着没有行或列可以完全为空，因为每一行都有一个保证填充的位置。 对角线交换步骤消除了唯一的结构对称性，该结构对称性可以沿单个对角线对齐所有填充的单元格，并无意中创建一个禁止的全空对角线。 一旦这种对称性被打破，每个线约束都得到满足，同时保留空单元的最佳数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())

    if n == 2:
        print(-1)
        return

    grid = [['.' for _ in range(n)] for _ in range(n)]

    # place one filled cell per row in a shifted diagonal pattern
    for i in range(n):
        j = (i + 1) % n
        grid[i][j] = '#'

    # swap first and last row placements to break diagonal symmetry
    if n > 2:
        for j in range(n):
            grid[0][j], grid[n - 1][j] = grid[n - 1][j], grid[0][j]

    for row in grid:
        print(''.join(row))

if __name__ == "__main__":
    solve()
```网格完全用空单元格初始化。 循环放置`#`使用循环移位确保每行恰好有一个填充单元格，以便列也只使用一次。 

第一行和最后一行之间的交换是关键的修正，可以避免对称情况下的对角线对齐问题。 如果没有它，图案将退化为完美对齐的对角线，从而无法达到预期的对角线约束。 

最后的打印步骤只是逐行输出构建的网格。

这显示了结构如何线性扩展并保持结构，即使$n$，其中对角对称问题最危险。

 ## 边缘情况

 对于$n = 2$，算法直接输出`-1`。 这与在不违反对角线约束的情况下每行恰好放置一个填充单元格的可能性相匹配。 网格太小，无法破坏对称性，因此任何尝试都会陷入禁线。 

为了$n = 1$，该构造放置一个单个填充单元，生成一个有效的普通网格。 该算法自然地处理这个问题，因为循环放置仍然有效并且不需要交换。 

对于甚至$n$，第一行和最后一行之间的交换至关重要。 如果没有它，该模式将成为完美的循环对角线，并且主对角线的对称性可能会产生意想不到的完全空的对角线。 交换可确保与该结构至少有一个偏差，从而打破对齐方式，同时保留行和列的唯一性。
