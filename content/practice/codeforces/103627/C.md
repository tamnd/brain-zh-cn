 ## 解决方案
 ## 问题理解

 我们得到一个从 0 到 N − 1 索引的元素宇宙。这个宇宙的每个子集都通过函数 a(S) 与一个值相关联。 该问题要求我们找到三个成对不相交的子集，通常称为 x、y 和 z，使得这些集合的组合之间存在一定的不等式。 

该表达式是通过对这些子集的并集上的函数 a 求值而构建的。 直观上，我们正在比较当我们在组之间移动元素时值如何变化。 一侧测量将 y 添加到 x 的增益，而另一侧测量将 z 额外移动到系统后该增益如何变化。 任务是检测一种配置，其中这种不平衡在所需方向上变得严格为正。 

尽管该语句是用抽象集合运算来表述的，但真正的困难是组合：我们正在将地面集的分区搜索为最多三个部分，并且函数 a(S) 的作用就像分配给每个子集的黑箱权重。 

从复杂性的角度来看，输入大小 N 意味着大小为 2^N 个子集的宇宙。 任何显式评估所有子集或所有子集三元组的方法都是立即指数的。 如果 N 约为 20，则 2^N 约为一百万，这在重剪枝的情况下是可行的。 如果 N 较大，甚至 O(2^N · N) 也会变得太慢，因此任何解决方案都必须以限制有效配置的方式利用结构。 

一个微妙的边缘情况是，许多简单的解决方案假设所有三个子集都必须非空。 当其中一个是单例时，甚至当其中一个部分在证明所​​暗示的变换后崩溃时，这种不等式仍然成立。 另一个问题是假设所有元素都必须是分布式的； 实际上，缩减表明大多数元素可以分组为单个“背景”集 x，并且只有一两个特殊元素很重要。 

例如，假设 N = 3 并且函数 a(S) 是任意的。 一个简单的求解器可能会尝试将元素详尽地分配到所有三个组中。 然而，不等式的结构意味着，如果存在一个解，它总是可以转化为只有一个或两个元素是特殊的，而其他所有元素都位于 x 中的解。 缺少这一点会导致不必要地探索 θ(3^N) 分区。 

## 方法

 这个问题的暴力观点很简单：我们迭代所有可能的不相交子集 (x, y, z) 的三元组，使用 a(S) 计算所需的表达式，并检查不等式。 对于每个候选分区，评估表达式需要多次调用 a(S)，并且每次调用本身可能需要迭代子集。 即使我们预先计算 a(S) 的所有值，分区的数量仍然是 3^N，这远远超出了 N ≈ 15。 

关键的结构洞察来自于分析当我们在集合之间逐渐移动元素时不平等如何变化。 如果我们固定x和y并考虑将元素一一添加到z中，则差分函数会逐渐变化。 如果存在任何 z 使不等式成立，那么当我们逐个元素构建 z 时，必须存在第一个元素，其加法翻转不等式。 这意味着我们永远不需要大 z，因为在插入单个元素时违规已经可见。 

当交换 y 和 z 的角色时，应用对称参数。 第二个转换表明我们还可以将 y 限制为单个元素。 应用这两种归约后，任何有效的配置都可以转换为其中 z 是单个元素，y 是单个元素，而 x 包含见证人中未涉及的所有剩余元素的配置。

这极大地减少了搜索空间。 我们不需要迭代任意分区，只需选择一个子集 x 和两个不在 x 中的不同元素 e 和 f，分别表示 y = {e} 和 z = {f}。 对于每个这样的选择，我们验证是否存在满足不等式的有效 x 分配。 由于 x 可以隐式表示为“我们决定包含的除 e 和 f 之外的所有元素”，因此我们有效地将 x 与一对 (e, f) 一起枚举。 

这种简化结构的蛮力变为 O(2^N · N^2)，这比 3^N 有了显着改进，并且根据常数因子，N 达到 20 到 22 左右是可行的。

 ## 算法演练

 1. 预先计算或假设对任何子集 S 的函数 a(S) 进行快速评估。这是必要的，因为每个候选检查都会重复查询子集值。 
2. 枚举地面集的所有子集x。 每个 x 代表分区的“背景”部分，包含在不等式中不发挥特殊作用的所有元素。 
3. 对于每个子集 x，迭代不同元素 (e, f) 的所有有序对，使得 e ∉ x 和 f ∉ x 且 e ≠ f。 这些代表单例集合 y = {e} 和 z = {f}。 
4. 对于每个三元组 (x, e, f)，使用集合表达式 a(x ∪ {e})、a(x ∪ {f})、a(x ∪ {e, f}) 和 a(x) 的直接计算来评估不等式。 比较完全按照转换后的条件中所述进行。 
5. 如果不等式成立，则输出对应的子集x、y、z并立即终止。 

这个枚举足够的原因是每个有效的解决方案都可以转换为只有两个元素位于 x 之外的解决方案。 枚举保证所有此类配置都被访问。 

### 为什么它有效

 正确性依赖于 z 构造上的“第一次改变”论证。 如果我们采用任何具有可能很大的 z 的有效解并将其元素逐个相加，则必须存在第一个元素，其相加使得不等式成立。 此时，所有先前添加的元素都可以被吸收到 x 中，而不会影响见证结构。 这会将 z 折叠为单例。 对称论证也会使 y 崩溃。 因此，任何解都可以用算法枚举的受限形式表示，从而确保搜索空间的完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Placeholder for subset value function.
# In an actual implementation, this would be provided or precomputed.
def solve():
    n = int(input().strip())

    # Assume we can read or compute a(S) somehow; details depend on full problem statement.
    # For editorial purposes, we focus on search structure.

    # This is a conceptual representation:
    # val[mask] = a(mask)
    val = [0] * (1 << n)

    # Enumerate x
    for x in range(1 << n):
        # try all ordered pairs (e, f)
        for e in range(n):
            if (x >> e) & 1:
                continue
            for f in range(n):
                if e == f:
                    continue
                if (x >> f) & 1:
                    continue

                xe = x | (1 << e)
                xf = x | (1 << f)
                xef = x | (1 << e) | (1 << f)

                # inequality check (from transformed condition)
                # a(x ∪ y) - a(x) < a(x ∪ y ∪ z) - a(x ∪ z)
                if val[xe] - val[x] < val[xef] - val[xf]:
                    # output representation
                    # exact format depends on original problem specification
                    print("YES")
                    print(x, e, f)
                    return

    print("NO")

if __name__ == "__main__":
    solve()
```该代码反映了搜索空间的减少。 x、e 和 f 上的三重循环直接实现了结构定理。 关键的实现细节是将子集表示为位掩码，以便联合运算变成按位或。 这使得每次转换的时间复杂度为 O(1)。 

一个常见的错误是忘记明确强制执行不相交条件。 支票`(x >> e) & 1`和`(x >> f) & 1`确保 e 和 f 不在 x 中。 另一个微妙的问题是错误地处理有序对。 (e, f) 和 (f, e) 可能代表不等式中的不同角色，因此必须考虑两者。

x = ∅ 的第二个例子强调了所有结构仅由两个元素决定的极端情况。 在这种情况下，该算法可以有效地直接测试所有有序对 (e, f)，从而确认即使是空背景也被枚举覆盖。

 ## 边缘情况

 一种边缘情况是 N 太小以至于不存在有效对 (e, f)。 在这种情况下，e 和 f 上的循环永远不会触发，并且算法会正确输出失败，而不会尝试无效的子集评估。 

当 x 已包含除一个元素之外的所有元素时，会出现另一种边缘情况。 那么内部循环没有有效的 f，因为 x 之外没有足够的元素来形成一对。 该算法自然会跳过这些配置，从而避免错误的自配对。 

第三种边缘情况是存在多个有效配置时。 由于该算法在第一个成功的见证人处停止，因此它不依赖于唯一性。 正确性论证只需要存在一种简化形式的解，因此提前终止不会影响有效性。
