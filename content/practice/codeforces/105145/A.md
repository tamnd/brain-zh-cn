 ## 解决方案
 ## 问题理解

 我们给出从 L 到 R 的连续整数范围，其中每个整数代表一种材料。 从这个范围中，我们必须选择两种材料（它们可能相同）并根据它们的十进制表示形式的差异来计算分数。 

为了计算分数，我们首先将两个数字写为长度相等的数字字符串，并用前导零填充较短的数字。 然后我们逐位比较，并将相应数字的绝对差值相加。 这实际上是数字向量之间的 L1 距离。 

任务是最大化区间 [L, R] 中所有数字对的距离。 

限制非常大：L 和 R 的长度最多可达 10 位数字，即最多约为 10^10。 这立即排除了尝试所有对的任何解决方案，因为间隔最多可以包含 10^10 个数字。 即使在范围内迭代也是不可能的。 

关键的结构困难在于范围不是任意的：它是一个连续的区间，但数字差异很大程度上取决于进位结构和位置约束，这使得局部贪婪推理变得不平凡。 

当 L 和 R 具有不同的数字长度时，会出现微妙的边缘情况。 例如，L = 90 且 R = 100。在这里，比较数字需要仔细处理前导零。 比较原始整数或没有填充的字符串的简单实现将产生不正确的结果。 

另一种边缘情况是当区间仅包含一个数字时，例如 L = R = 7。那么唯一可能的对是 (7, 7)，得分为 0。任何解决方案都必须显式处理此问题。 

最后，跨越数字边界的转换很重要。 例如，L = 190 和 R = 209 允许混合来自不同“形状”的数字的数字，并且最佳对通常位于端点附近而不是区间内。 

## 方法

 强力解决方案将迭代 [L, R] 中的所有对 (x, y)，计算它们的逐位差异，并取最大值。 每次比较的成本为 O(d)，其中 d 是位数，因此总复杂度为 O((R−L+1)^2 · d)。 当 R 达到 10^10 时，这是完全不可行的。 

关键的观察是，一旦我们确定了要比较的数字，每个数字位置的贡献都是独立的。 分数是职位的总和，因此最大化总分意味着同时最大化每个职位的贡献。 然而，数字在数字之间并不独立，因为我们必须选择一致的整数，而不是任意的数字向量。 

关键的结构见解是，在区间施加的可行性约束下，最佳对总是由每个数字位置的极端选择决定。 我们不是枚举数字，而是同时考虑数字对上的数字 DP，跟踪每个数字是否仍然受到 L 或 R 的限制。 

我们从左到右逐位构建两个数字，确保两者都在 [L, R] 范围内。 在每个位置，我们尝试最大化所选数字之间的绝对差，同时尊重前缀约束。 

这将问题转化为数字 DP，其状态表示两个构造数字的前缀如何与 L 和 R 边界进行比较。

 | ---| ---| ---| ---|
 | 蛮力 | O(n^2·d) | O(n^2·d) | O(1) | O(1) | 太慢了 |
 | 成对数字 DP | O(d·10^2·4) | O(d·4) | 已接受 |

 ## 算法演练

 我们将数字处理为带有前导零的固定长度数字字符串，使用的长度等于 |L| 的最大值 和|R|。 

我们定义一个 DP，同时构造两个数字。 

1. 如有必要，通过用前导零填充 L，将 L 和 R 标准化为相同长度。

这确保了数字索引的一致性并使边界检查统一。 
2. 定义一个递归函数 dp(pos,ightL1,tightR1,tightL2,tightR2)，其中 pos 是当前数字索引，tight 标志表示每个构造数字的前缀在边界处是否仍然精确等于 L 或 R。 需要这些标志来确保我们永远不会在 [L, R] 之外构造无效数字。 
3. 在每个位置，迭代 [0, 9] 中所有可能的数字对 (a, b)。 对于每一对，检查在当前严格约束下放置这些数字是否将两个数字保持在允许的范围内。 
4. 如果数字分配有效，则计算贡献 |a − b| 并将其添加到下一个位置 dp(pos + 1, Updated flags) 的最佳可实现值。 
5. 取所有有效数字对中的最大值。 
6. 从位置 0 开始，所有紧密标志设置为 true，这意味着两个数字最初都受到 L 和 R 的约束。 
7. 返回 dp(0, true, true, true, true) 的结果。 

关键的微妙之处在于标志更新的紧密程度。 如果我们放置的数字等于相应的边界数字，我们就保持紧； 否则，我们放松该方向的约束。 

### 为什么它有效

 DP 保持了这样的不变性：在每个位置，两个部分构造的数字都是 [L, R] 中某些数字的有效前缀。 每个转换都通过根据边界约束检查数字可行性来保留这种不变性。 由于所有有效数字对恰好对应于该状态空间中的一条路径，并且对每条路径进行评估，因此所有 DP 状态的最大值等于最大可能得分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    L = input().strip()
    R = input().strip()

    n = max(len(L), len(R))
    L = L.zfill(n)
    R = R.zfill(n)

    from functools import lru_cache

    @lru_cache(None)
    def dp(pos, tL1, tR1, tL2, tR2):
        if pos == n:
            return 0

        lo1 = int(L[pos]) if tL1 else 0
        hi1 = int(R[pos]) if tR1 else 9
        lo2 = int(L[pos]) if tL2 else 0
        hi2 = int(R[pos]) if tR2 else 9

        best = 0

        for a in range(lo1, hi1 + 1):
            ntL1 = tL1 and (a == int(L[pos]))
            ntR1 = tR1 and (a == int(R[pos]))
            for b in range(lo2, hi2 + 1):
                ntL2 = tL2 and (b == int(L[pos]))
                ntR2 = tR2 and (b == int(R[pos]))
                best = max(best, abs(a - b) + dp(pos + 1, ntL1, ntR1, ntL2, ntR2))

        return best

    print(dp(0, True, True, True, True))

if __name__ == "__main__":
    solve()
```其实现直接遵循数字DP结构。 填充确保数字对齐。 DP 状态显式编码每个数字是否仍受 L 或 R 约束。转换枚举数字对并累积每个数字的绝对差。 

一个微妙的实现点是数字的界限独立于每个数字的紧密度。 另一个是我们必须仔细更新所有四个标志； 即使缺少一个也会导致无效的数字生成。

 ## 边缘情况

 当L等于R时，DP立即达到所有数字都固定的状态，并且每个贡献都为零。 递归仍然运行，但仅遵循单个强制路径，正确生成 0。 

当 L 和 R 长度不同时，填充可确保正确对齐。 例如，L = 90 和 R = 100 变为 090 和 100。DP 正确地将前导零与 R 中的数字进行比较，从而允许较高位置的较大贡献。 

当数字在整个范围内相同时，例如 L = 111 和 R = 111，所有转换都是强制的，并且没有数字对贡献任何积极的东西。 

当范围很大时，DP 仍然只探索数字状态，而不是数值，因此它仍然不受幅度的影响。
