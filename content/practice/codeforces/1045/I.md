 ## 解决方案

 ## 问题理解

 我们给定了一组字符串，我们需要计算有多少无序索引对形成了“好对”。 如果在连接两个字符串之后，可以将结果的字符重新排列成回文，那么一对就是好的。 

一个关键的观察是，我们不需要直接从串联形成回文。 我们可以自由地排列字符，因此原始串联的结构根本不重要。 只有两个字符串中的多组字符在一起才重要。 

当且仅当至多一个字符在组合字符串中出现奇数频率时，字符串才能排列成回文。 这将问题转化为关于字符计数奇偶校验而不是字符串顺序的推理。 

因此每个字符串都可以用一个 26 位掩码来表示，其中第 i 位表示该字符的频率是否为奇数。 对于两个字符串，它们的串联奇偶校验掩码等于其掩码的异或。 条件变为：XOR 掩码必须最多设置一位。 

我们正在计算无序对 (i, j)，i ≠ j，其中该条件成立。 

约束很大，最多有 100000 个字符串，总长度最多 1e6。 任何 O(n^2) 配对都是不可能的，甚至每对字母表中的任何线性都太慢。 我们需要接近 O(n · 26) 或 O(n · 26 log n) 的东西，或者更好，O(n · 26) 散列或频率聚合。 

一种天真的失败模式是尝试直接检查每对字符串。 即使检查一对的时间复杂度为 O(26)，在最坏的情况下也会导致大约 5e9 次操作，这是不可行的。 

另一个微妙的陷阱是误解这个条件：有些人可能错误地认为我们需要连接本身是回文而不需要排列。 这将变成一个不同的、顺序敏感的问题并产生错误的答案。 

边缘情况包括：

 具有所有偶数字符数的单个字符串，可以与自身或其他偶数奇偶校验字符串配对。 

字符串有多个奇数位不同，无法形成有效的对。 

高度重复的字符串，例如“aaaaa”，其中掩码为零。 

## 方法

 暴力方法会检查每对字符串，并计算那些连接可以排列成回文的字符串。 对于每一对，我们计算两个字符串的字符频率，将它们组合起来，并计算有多少个字符具有奇数频率。 如果该数字最多为 1，则该对有效。 即使我们预先计算每个字符串的频率向量，每对检查的成本仍然是 O(26)，导致 O(n^2 · 26)，这远远超出了 n = 100000 的限制。 

关键的见解是用奇偶校验掩码替换全频率矢量。 每个字符串变成一个 26 位整数。 如果掩码的 XOR 汉明权重最多为 1，则一对是有效的。这意味着掩码要么相等（XOR 为零），要么它们相差恰好一位翻转。 

这将问题简化为计算每个掩码有多少个先前掩码相同或相差一位。 我们维护迄今为止所见掩模的频率字典。 对于每个新掩码，我们查询其计数加上恰好翻转一位的所有掩码的计数。 

这将复杂性从二次配对降低到每个字符串 26 个查询，使其在实践中呈线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²·26) | O(n) | 太慢了|
 | 最佳（位掩码+散列）| O(n·26) | 最坏情况为 O(2²⁶)，实际情况为 O(n) | 已接受 |

 ## 算法演练

 我们一一处理字符串，同时维护先前见过的奇偶校验掩码的频率表。

1. 对于每个字符串，计算一个 26 位掩码，其中如果字符在该字符串中出现奇数次，则位 i 为 1。 这简洁地表示了回文重排所需的所有奇偶校验信息。 
2. 在将当前掩码插入频率表之前，我们计算有多少个先前掩码与其形成有效对。 有两种情况。 首先，相同的掩码，因为 XOR 变为零并且所有奇偶校验都取消。 其次，掩码仅相差一位，因为这会在联合中产生恰好一个奇数字符。 
3. 我们直接从频率表中添加相同的先前掩码的计数。 
4. 然后我们迭代所有 26 个字母，翻转当前掩码的每一位，并添加这些掩码的频率。 每次翻转对应于在最终串联中恰好允许一个字符具有奇数奇偶校验。 
5. 计算贡献后，我们将当前掩码插入到频率表中。 

每个字符串都是独立处理的，并且所有有效对都只计算一次，因为我们只计算第二个索引是当前字符串且第一个索引较早的对。 

### 为什么它有效

 正确性来自于将回文排列条件简化为奇偶校验约束。 当且仅当至多一个字符的计数为奇数时，多组字符可以重新排列为回文。 两个掩码的异或表示组合多重集的奇偶校验。 因此，有效对与汉明权重 ≤ 1 的 XOR 掩码完全对应。频率表确保在处理该对的较晚端点时，每个较早的兼容掩码都会被计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def mask_of(s: str) -> int:
    m = 0
    for c in s.strip():
        m ^= 1 << (ord(c) - 97)
    return m

n = int(input())
freq = {}

ans = 0

for _ in range(n):
    s = input().strip()
    m = mask_of(s)

    ans += freq.get(m, 0)

    for i in range(26):
        ans += freq.get(m ^ (1 << i), 0)

    freq[m] = freq.get(m, 0) + 1

print(ans)
```核心实现围绕为每个字符串构建奇偶校验掩码。 XOR 累加是跟踪奇偶校验的正确方法，因为每次出现字符时切换一位可确保准确表示奇数计数，而无需存储完整的频率数组。 

计数步骤仔细区分精确匹配和一位差异。 我们不在查询阶段包含当前字符串，因此每对都只计算一次。

import sys, io

def solve():
    import sys
    input = sys.stdin.readline

    def mask_of(s: str) -> int:
        m = 0
        for c in s.strip():
            m ^= 1 << (ord(c) - 97)
        return m

    n = int(input())
    freq = {}
    ans = 0

    for _ in range(n):
        s = input().strip()
        m = mask_of(s)

        ans += freq.get(m, 0)
        for i in range(26):
            ans += freq.get(m ^ (1 << i), 0)

        freq[m] = freq.get(m, 0) + 1

    print(ans)

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    try:
        solve()
        return ""
    finally:
        sys.stdin = old_stdin

# provided sample
assert run("3\naa\nbb\ncd\n") == "", "sample 1"

# all identical strings

assert run("3\nab\nac\nad\n") == "", "mixed small case"

# maximum-like simple case

 ## 边缘情况

 一个棘手的情况是所有字符串都具有相同的奇偶校验掩码，例如所有字符都重复偶数次。 例如：```
3
aa
bb
cc
```每个字符串都映射到掩码 0。该算法会正确计算所有对，因为每个新字符串都通过相同的掩码查找来匹配所有先前的字符串。 

另一种情况是字符串相差多个奇数字符时，例如：```
2
abc
def
```两个掩码都设置了多个位，并且它们的 XOR 具有多个位。 该算法不会对它们进行计数，因为相同的掩码和单位翻转都不匹配，这会正确排除无效对。 

最后，单字符字符串测试最小掩码。 每个字符串都提供一位掩码，并且只有最多一个字符差异的对才符合资格，算法通过翻转检查正确评估这一点。
