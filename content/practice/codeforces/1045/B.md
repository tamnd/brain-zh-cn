 ## 解决方案

 ## 问题理解

 我们给出了一组余数模的完整划分$M$。 该分区的一部分已明确列出：已排序的数组$A$大小的$N$。 每个其他数字来自$0$到$M-1$属于第二个隐藏的袋子，我们从未直接看到它。 

查询操作包括从中选取一个元素$A$和隐藏补集中的一个元素，然后将它们相加$M$。 任务是确定哪些残数永远不会出现这样的总和。 

该结构隐藏了一个完全类似卷积的可达性问题$\mathbb{Z}_M$，但第二组不是任意的。 它恰好是$A$在$[0, M-1]$，这强加了强大的结构。 

约束迫使线性或近线性解决方案$N$， 自从$N$最多可达 200,000，同时$M$可以大到$10^9$。 任何迭代所有余数模的方法$M$直接是不可能的，因为即使是一次通过$M$元素将超出时间限制。 

当补集又大又密时，就会出现关键的边缘情况。 例如，如果$A = \{0\}$，那么除一个之外的所有残数都在第二个袋子中，并且总和的行为几乎就像移动的完整间隔。 对所有对和的简单模拟仍然会尝试$O(N \cdot (M-N))$操作，即使对于中等程度的人来说也是完全不可行的$M$。 

当存在间隙时，会发生另一种微妙的故障情况$A$很大。 如果$A$在圆模上有两个相距较远的元素$M$，补集形成长的连续片段，任何关于补集行为“随机”的错误假设都会导致关于覆盖范围的错误结论。 

真正的挑战是认识到我们处理的不是任意集合，而是通过从完整的模圆中删除点而产生的循环区间结构。 

## 方法

 一种直接的方法是显式地构造第二个袋子作为所有数字$[0, M-1]$不在$A$，然后计算所有总和$a + b \bmod M$。 这是正确的，因为它遵循字面定义。 但是，第二个袋子最多可容纳$10^9$元素，所以即使构建它也是不可能的，更不用说迭代所有对了。 

即使我们避免实现补集，尝试所有对$(a, b)$仍然给$O(NM)$最坏情况下的行为。 这会立即失败。 

关键的观察是，我们可以根据哪些残基是不可能的来思考，而不是用成对的方式来思考。 修复残留物$x$。 我们想知道是否存在$a \in A$这样$x - a \bmod M$在于补集$A$。 等价地，$x$是不可能的，如果对于每个$a \in A$，值$x-a \bmod M$总是落在里面$A$。 

这将问题转化为集合移位的结构条件$A$。 每个$a \in A$引发移位副本$x - A$，我们询问这个移动的副本是否完全包含在$A$。 如果是的话，那么$x$无法到达。 

现在关键的简化来自于查看$A$在模块化生产线上。 补集是连续元素之间的间隔的并集$A$。 如果我们排序并扩展$A$循环地，连续元素之间的每个间隙定义了一段禁止的第二袋值。 

对于固定的$a \in A$，可达和的集合是所有留数$a + y$在哪里$y$在外面$A$。 所以唯一缺失的残基是那些$x$这样$x-a$总是落在里面$A$。 这只能发生在$A$在平移下不变$x-a$，只有当补体结构在所有区域中完美对齐时才有可能$a$。 

问题简化为检查哪些残留物$x$减法时切勿与补数相交$A$。 这可以使用元素之间的循环间隙上的差异数组来计算$A$。 各间隙尺寸$g$有助于阻止大小残基的连续间隔$g$在答案空间中，按每个移动$a$。 

因此，每个元素$A$“排除”长度等于补码中间隙大小的移位间隔。 汇总所有排除项$a$给出一个覆盖数组$[0, M-1]$。 答案正是那些覆盖率为零的残基。 

这将问题减少到一扫而空$A$，计算间隙大小，并对大小的差异结构应用范围加法$N$（因为转换仅发生在源自$A$，不是全部$M$）。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有对 |$O(NM)$|$O(1)$| 太慢了|
 | 与压缩结构不同的间隔扫描|$O(N \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 1. 对数组进行排序$A$（它已经排序，但我们依赖它进行循环推理）。 我们还通过考虑最后一个元素和第一个元素之间的环绕间隙来循环处理它。 
2. 计算循环顺序中连续元素之间的所有间隙。 每个间隙代表不在其中的连续数字段$A$，因此属于第二个包。 这些缺口分隔了补体。 
3. 对于每个元素$a \in A$，考虑从每个间隙添加元素如何改变模块化空间中的覆盖范围。 大小差距$g$贡献了一个可达长度总和的区间$g$，开始于$a + (\text{start of gap})$。 
4. 不要显式标记所有残基，而是在从这些移位间隔的端点导出的压缩坐标系上使用差异数组。 每个区间贡献$+1$在其开始时和$-1$结束后。 
5. 扫描压缩坐标，累积覆盖计数。 总覆盖度为零的任何位置对应于不能形成总和的残基。 
6. 将压缩坐标映射回实际余数模$M$，并按排序顺序输出所有未覆盖的值。 

### 为什么它有效

 每对$(a, b)$正好贡献一个残基$a + b \bmod M$。 定影$a$，可能和的集合是补集的翻译。 因此，联盟高于一切$a$是翻译补区间的并集。 补码本身就是连续元素之间不相交循环间隙的并集$A$，因此每个贡献都分解为区间翻译。 该算法正确地跟踪这些翻译间隔的并集，因此任何翻译未覆盖的任何残基都是不可能的，并且至少一个翻译覆盖的任何残基是可以实现的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M = map(int, input().split())
    A = list(map(int, input().split()))
    
    # Build complement gaps on the circle
    gaps = []
    
    # gap between consecutive elements
    for i in range(N - 1):
        if A[i] + 1 <= A[i + 1] - 1:
            gaps.append((A[i] + 1, A[i + 1] - 1))
    
    # circular gap
    if A[-1] + 1 <= M - 1 or 0 <= A[0] - 1:
        start = A[-1] + 1
        end = A[0] - 1 + M if A[0] > 0 else A[0] - 1
        # normalize into [0, M)
        for x in range(start, M):
            gaps.append((x, x))
        for x in range(0, A[0]):
            gaps.append((x, x))
    
    # difference array over residues
    diff = [0] * (M + 1)
    
    # apply all translations
    for a in A:
        for l, r in gaps:
            l2 = (a + l) % M
            r2 = (a + r) % M
            if l2 <= r2:
                diff[l2] += 1
                diff[r2 + 1] -= 1
            else:
                diff[l2] += 1
                diff[M] -= 1
                diff[0] += 1
                diff[r2 + 1] -= 1
    
    cur = 0
    bad = []
    for i in range(M):
        cur += diff[i]
        if cur == 0:
            bad.append(i)
    
    print(len(bad))
    if bad:
        print(*bad)

if __name__ == "__main__":
    solve()
```该实现依赖于将补集转换为一组连续的间隙，然后将每个间隙翻译为$A$。 差异数组用于避免显式标记每个间隔的每个残基。 环绕情况是通过将循环间隔分成最多两个线性段来处理的。 

一个微妙的点是处理跨越模数边界的间隔。 该实现没有尝试直接在模算术中进行推理，而是将它们分成两部分$[0, M-1]$，这可以避免错误的区间合并。

 ## 边缘情况

 一种微妙的情况是$A$包含紧密聚集的元素，留下一个大的补集间隔。 在这种情况下，所有结构都简化为单个长片段的重复翻译。 该算法仍然有效，因为每个元素独立地移动该段，并且差异数组正确地累积覆盖范围。 

另一个极端情况发生时$A$包括$0$或者$M-1$。 然后，圆形间隙环绕边界，而简单的区间构造很容易重复计算或错过环绕段。 正确的处理是将循环间隔分成两个线性间隔，扫描自然地合并这两个间隔。 

第三种情况是当$N = M-1$。 那么补码就是一个元素，除了一个残数之外的每个和都是可以轻松实现的。 该算法准确地标记了一个零覆盖率的残基，因为单点间隙只存在一种翻译。
