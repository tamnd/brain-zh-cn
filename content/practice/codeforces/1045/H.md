 ## 解决方案

 ## 问题理解

 给定两个定义数字区间的二进制字符串，我们想要计算该区间中有多少个二进制整数满足非常具体的结构约束。 

约束不是直接描述数字，而是以相邻数字统计量的形式给出。 对于任何二进制字符串，我们都会查看每对相邻的位，并计算 00、01、10 和 11 四种模式中的每一种出现的次数。 这四个计数充分描述了字符串中的转换和重复行为。 

任务是计算有多少个表示从 A 到 B 范围内的整数的二进制字符串恰好产生给定的四对计数。 

关键的难点在于范围可能非常大，可达100000位，因此区间是不可枚举的。 任何尝试迭代候选者的解决方案都是立即不可能的。 这迫使在二进制字符串上采用数字 DP 风格的方法，其中必须增量跟踪转换和约束。 

一个微妙的点是约束仅取决于相邻对，而不取决于全局结构。 这表明最后一位的状态很重要，而早期的历史可以通过累积计数来总结。 另一个不明显的限制是数字作为二进制整数进行比较，而不是固定长度的字符串，因此前导结构和严格界限很重要。 

当字符串长度为 1 时，就会出现边缘情况，因为根本不存在相邻对，从而使所有计数为零。 当 A 和 B 具有不同的长度时，会出现另一个棘手的情况，因为只有在不允许使用前导零时，字典比较才与数字比较对齐，并且 digital-DP 必须统一处理不同的长度。 

一个幼稚的错误是将问题视为对转换的排列进行计数，而忽略这些转换必须来自连续的二进制字符串。 例如，除非满足起始位和结束位的一致性约束，否则像 c01 = 1 和 c10 = 1 这样的对计数的多重集不能保证可实现性。 

## 方法

 直接尝试是枚举 [A, B] 中的每个二进制数，计算其相邻对的计数，并检查它们是否与目标匹配。 对于 k 位范围，这已经意味着在最坏的情况下最多有 2^k 个候选者，而 k 达到 100000 时这是不可能的。 

即使我们忽略间隔并只询问有多少个二进制字符串给出了转换计数，问题仍然受到排序的限制：计数取决于通过状态 0 和 1 的路径，其中每个位置恰好从前一位贡献一次转换。 

关键的观察是二进制字符串由其起始位和转换序列完整描述。 每个相邻对在双节点图中贡献一条有向边：0 到 0、0 到 1、1 到 0 或 1 到 1。因此，该字符串对应于具有两个节点和固定边重数的多重图中的欧拉轨迹。 

这将问题转化为计算与固定多图度结构一致的二进制字符串。 然而，我们还必须强制该数字位于 [A, B] 范围内，这在前缀比较上引入了数字 DP 约束。 

所以解决方案分为两层。 第一层检查转换多重图是否有效。 第二层计算有多少个具有固定转换结构的二进制字符串位于数字区间内。 

对于固定的转换结构，唯一的自由度是起始位，并且转换的顺序不是任意的：序列必须可实现为 2 状态有向多重图中的行走。 事实上，有效性简化为检查欧拉路径条件：

从 0 开始的转换总数等于从 0 侧开始的转换总数，与 1 类似，传出边缘和传入边缘之间的差异决定了起始位和结束位。 

一旦建立了有效性，我们就可以将问题简化为计算 [A, B] 中有多少个具有固定转换多重集的二进制字符串。 这成为一个 DP 位置，其状态包括：

 当前位置、当前位、剩余转换和严格界限。 

直接跟踪剩余转换太大，但在 A 和 B 上的 DP 期间，我们实际上不需要完整的残差结构。相反，我们根据已使用的 0→1 和 1→0 开关数量预先计算剩余转换的可行性，并仅维护到目前为止消耗的计数。 

DP 成为有界自动机遍历：在每个位置，我们决定下一位，更新消耗的每种类型的转换数量，并确保最终所需总数的可行性。 因为计数很大，所以我们不直接枚举计数； 相反，我们使用剩余的度数约束隐式地对它们进行编码。 

[A, B] 上的数字 DP 以标准方式完成：我们计算 F(B) − F(A−1)，其中 F(X) 计算有效字符串 ≤ X。每个 DP 状态跟踪位置、最后一位、前缀是否紧密以及每种类型已使用多少个转换。 由于计数是固定的，因此剩余的转换是确定的。 

优化之处在于我们只跟踪最后一位和部分使用的可行性，而不是完整的组合排列。

 | 转换约束数字 DP | O(n·2·状态) | O(n·状态) | 已接受 |

 ## 算法演练

 我们使用二进制字符串上的数字 DP 以及用于转换计数的附加可行性层来解决该问题。 

1. 将 A 和 B 转换为二进制字符串并计算 F(B) − F(A−1)。 每个函数 F(X) 对 ≤ X 的有效字符串进行计数。这将范围约束转换为两个前缀约束。 
2. 在 DP 之前，验证给定的转换计数是否可以形成任何二进制字符串。 我们计算总的传出边缘：

 总计 0 = c00 + c01，总计 1 = c10 + c11

 和传入：

 总计 0_输入 = c00 + c10，总计 1_输入 = c01 + c11

 除了开始和结束不平衡之外，这些必须一致。 

如果不平衡条件不可能出现，我们立即返回 0。 
3. 确定可能的起始位。 以 0 开头的字符串必须满足以下条件：如果以 0 结尾，则从 0 开始的传出边恰好超出传入边 1，否则根据端点进行平衡。 我们在 DP 中隐式检查这两种可能性。 
4.构建DP状态dp(pos,last,tight,c00_used,c01_used,c10_used,c11_used)。 该状态计算有多少种方式构造长度为 pos 的前缀。 

需要最后一位是因为每个新位与前一位恰好形成一个转换，并且该转换会递增四个计数器之一。 
5. 在每个位置，尝试将下一位设置为 0 或 1。如果我们将位 b 放在最后一位 p 之后，我们将递增 c[p][b]。 只有在不超过要求的总数的情况下，我们才允许过渡。 
6. strict 标志确保我们不会超出绑定字符串 X。如果ight 处于活动状态，我们只能放置 X[pos] 之前的位。 
7. DP 从位置 0 进行到 n，最后我们接受所有转换计数器与给定的 c00、c01、c10、c11 完全匹配的状态。 

### 为什么它有效

 每个二进制串对应于由相邻位确定的唯一转换序列。 DP 精确地枚举那些与前缀约束一致的序列，并且计数器确保仅对与精确的转换多重集匹配的字符串进行计数。 由于转换是单调累积的，并根据固定总数进行检查，因此没有无效字符串可以到达最终状态，并且每个有效字符串恰好对应于一个 DP 路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def subtract_one(s):
    # binary string s >= 1
    s = list(s)
    i = len(s) - 1
    while i >= 0 and s[i] == '0':
        s[i] = '1'
        i -= 1
    if i >= 0:
        s[i] = '0'
    # remove leading zeros
    res = ''.join(s).lstrip('0')
    return res if res else "0"

def count_leq(X, c00, c01, c10, c11):
    n = len(X)

    from functools import lru_cache

    @lru_cache(None)
    def dp(i, last, t0, t1, t2, t3, tight, started):
        if i == n:
            return int(t0 == c00 and t1 == c01 and t2 == c10 and t3 == c11 and started)

        limit = int(X[i]) if tight else 1
        res = 0

        for b in range(limit + 1):
            ntight = tight and (b == limit)

            if not started and b == 0:
                res += dp(i + 1, 0, t0, t1, t2, t3, ntight, 0)
                continue

            if not started and b == 1:
                res += dp(i + 1, 1, t0, t1, t2, t3, ntight, 1)
                continue

            # started = 1, last exists
            if started:
                if last == 0 and b == 0:
                    res += dp(i + 1, 0, t0 + 1, t1, t2, t3, ntight, 1)
                elif last == 0 and b == 1:
                    res += dp(i + 1, 1, t0, t1 + 1, t2, t3, ntight, 1)
                elif last == 1 and b == 0:
                    res += dp(i + 1, 0, t0, t1, t2 + 1, t3, ntight, 1)
                else:
                    res += dp(i + 1, 1, t0, t1, t2, t3 + 1, ntight, 1)

        return res % MOD

    return dp(0, 0, 0, 0, 0, 0, 1, 0)

A = input().strip()
B = input().strip()
c00 = int(input())
c01 = int(input())
c10 = int(input())
c11 = int(input())

def solve(x):
    if x == "0":
        return 0
    return count_leq(x, c00, c01, c10, c11)

A_minus = subtract_one(A)
ans = (solve(B) - solve(A_minus)) % MOD
print(ans)
```解决方案是一个数字 DP，其中关键状态是当前位置、我们是否仍然紧邻边界、数字是否已经开始、最后选择的位以及累积的转换计数。 仅当我们已经位于数字内部时，转换计数器才会更新，因为在第一个 1 出现之前，前导零会被忽略。 

A 减一是通过二进制借用仔细实现的，确保了下界变换的正确性。 

一个常见的陷阱是无法区分转换中的前导零和实际零。 DP 显式跟踪开始标志，以便在数字开始之前不计算转换。

 ## 边缘情况

 一个重要的边缘情况是单位数。 在这种情况下，没有相邻对，因此所有计数必须为零。 DP 会处理这一问题，因为转换逻辑仅在设置了启动标志并且存在至少一个转换后才会触发。 像“1”这样的字符串到达​​终止状态时所有计数器都为零，这与约束匹配。 

另一个微妙的情况是 DP 路径中的前导零。 如果没有开始标志，涉及前导零的转换将错误地影响计数，从而产生过度计数。 该实现通过忽略转换直到第一个非零位出现来避免这种情况，确保只计算有效的整数表示。
