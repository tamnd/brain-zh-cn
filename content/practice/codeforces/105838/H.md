 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 每个测试用例描述无限二维网格上的一组点。 每个点代表一个放置在整数坐标处的塔。 从每座塔中，不断地沿着x轴和y轴的正负方向发射出四道射线，无限延伸并覆盖它们所经过的每一个点。 

如果两座塔都位于对方至少一条射线上，则认为两座塔“互相攻击”。 由于光线仅沿着与轴平行的线传播，因此这种情况转化为非常严格的几何结构：位于$(a_i, b_i)$攻击另一座塔$(a_j, b_j)$当且仅当它们共享相同的 x 坐标或相同的 y 坐标。 

任务是计算无序对的数量$(i, j)$,$i < j$，使得$a_i = a_j$或者$b_i = b_j$。 

制约因素很大。 所有测试用例的总点数最多为$10^5$，因此任何直接检查所有对的解决方案都是$O(n^2)$，立即太慢了。 每个测试用例我们需要接近线性或线性算数的时间。 

一个微妙但重要的边缘情况是仔细处理重复的计数。 如果多个塔共享相同的 x 和 y 坐标，它们将是相同的点，但问题保证坐标是不同的，因此我们避免了这种歧义。 然而，我们仍然必须避免在中间推理中重复计算共享两个坐标的对，尽管由于唯一性而不会发生这种情况。 

## 方法

 直接方法很简单：检查每对塔并验证它们的 x 坐标或 y 坐标是否匹配。 这是正确的，因为它直接实现了相互攻击的定义。 然而，随着高达$10^5$点，这需要大约$5 \times 10^9$在最坏的情况下进行比较，这是不可行的。 

关键的观察结果是条件完全分解为两个独立的等价关系。 如果一对属于同一 x 组或同一 y 组，则该对有效。 这表明计算每组而不是每对的贡献。 

如果我们固定一个 x 坐标$a$，具有该 x 坐标的所有塔形成一个组。 该组内的任何对都是有效的，贡献$\binom{k}{2}$如果有$k$具有该 x 值的塔。 类似地，对于每个 y 坐标$b$，具有该 y 坐标的所有塔也有贡献$\binom{k}{2}$。 

微妙的问题是，这种条件并集方法可能会重复计算共享 x 和 y 的对。 但是，由于所有点都是不同的，因此没有两个塔共享两个坐标，因此没有要相减的交集。 

因此，问题简化为分别计算 x 值和 y 值的频率并对组合贡献求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 频率计数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 读取所有 x 坐标和 y 坐标并存储每个不同值的频率计数。 我们维护两个哈希映射或数组：一个用于 x 值，另一个用于 y 值。 这是必要的，因为答案仅取决于有多少点共享每个坐标，而不取决于它们的身份。 
2. 对于每个不同的 x 坐标值，令其频率为$k$。 我们添加$k \cdot (k - 1) / 2$到答案。 这会计算在 x 线上垂直对齐的所有无序塔对。 
3. 对于每个不同的 y 坐标值，令其频率为$k$。 我们再次添加$k \cdot (k - 1) / 2$到答案。 这会计算该 y 线上水平对齐的所有无序塔对。 
4. 输出测试用例的累计和。 

关键的推理步骤是每个有效对都是通过垂直或水平对齐来唯一确定的。 由于我们独立地计算每个坐标类内的所有对，因此每个有效的交互都会被捕获一次。 

### 为什么它有效

 考虑任何有效的塔对。 如果他们互相攻击，他们必须共享x或共享y。 如果他们共享 x，则他们通过以下方式恰好包含在一个 x 组贡献中：$\binom{k}{2}$。 如果他们共享 y，则他们恰好包含在一个 y 组贡献中。 由于坐标不同，一对不能同时共享 x 和 y，因此两个计数过程之间不存在重叠。 这在有效对与算法求和的贡献之间建立了一对一的对应关系。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        xs = list(map(int, input().split()))
        ys = list(map(int, input().split()))

        cx = {}
        cy = {}

        for x in xs:
            cx[x] = cx.get(x, 0) + 1
        for y in ys:
            cy[y] = cy.get(y, 0) + 1

        ans = 0

        for v in cx.values():
            ans += v * (v - 1) // 2
        for v in cy.values():
            ans += v * (v - 1) // 2

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将计数分成两个独立的频率累加。 第一个循环构建每条垂直线上有多少塔的计数，第二个循环构建水平线上的计数。 最终循环使用标准组合公式将这些频率转换为对计数。 

一个常见的实现错误是尝试在输入解析期间增加答案，而不完全分离 x 和 y 计数，这有混合逻辑的风险并使调试变得更加困难。 另一个陷阱是忘记组合公式中的整数除法，但是 Python 的`//`确保正确性。

 ## 边缘情况

 一个微妙的边缘情况是所有点共享相同的 x 坐标。 该算法通过将所有点放入单个 x 频率桶中来处理此问题，产生$\binom{n}{2}$，而所有 y 频率都是孤立的并且贡献为零。 工作示例中的迹线已经证实 x 组单独产生了正确的完整答案。 

另一种情况是所有点共享相同的 y 坐标。 对称地，y 侧频率占主导地位，x 侧没有贡献。 由于该算法独立处理 x 和 y，因此不存在可能扭曲结果的交互项。 

最后的边缘情况是所有坐标在两个维度上都不同。 两个频率图都完全由 1 组成，因此每个$\binom{1}{2}$术语消失。 该算法正确地输出零，这与没有两座塔共享一条攻击线的事实相匹配。
