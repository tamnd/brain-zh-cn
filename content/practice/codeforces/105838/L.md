 ## 解决方案
 ## 问题理解

 我们从一组精确的$k$数字。 其中，$k-1$副本等于某个值$x$，单个元素等于$y$。 因此，最初除了一个“干扰”之外，结构非常均匀。 

每个操作选择任意两个元素$a$和$b$，删除它们，计算它们的平均值，向下舍入，然后插入该舍入值的两个副本。 实际上，一对被两个相同的值替换，等于$\left\lfloor \frac{a+b}{2} \right\rfloor$。 多重集的大小永远不会改变，但值会逐渐相互漂移，同时丢失小数部分。 

该过程任意重复多次。 最终，总是有可能达到所有元素变得相等的状态。 任务是确定这个最终共同值的最小可能值$S$。 

限制条件非常大：$k$可以达到$10^9$，值可以达到$10^{18}$。 这立即排除了对该过程的任何模拟。 即使存储多重集也是不可能的，因此解决方案必须将整个动态压缩为从操作不变量导出的封闭形式表达式。 

一个微妙的边缘情况来自于平均中的底线。 如果$a+b$为奇数，两个结果元素之和严格小于$a+b$，意味着不保留总和。 这创造了一种潜在的“损失”，而天真的基于平均的论点可能会忽略这种损失。 例如，使用值$1$和$2$，该操作产生两个$1$s，将总和减少$1$。 任何假设总和保存的推理都会立即失败。 

## 方法

 暴力模拟会重复选择对，应用变换，并希望系统收敛。 甚至代表国家成本$O(k)$，每个操作都是$O(1)$，但稳定之前的操作次数原则上是无界的，并且可以轻松地随着值的大小而缩放。 和$k$最多$10^9$，这种做法是完全不可行的。 

关键的结构观察是，除了受控的舍入损失之外，该操作的行为呈线性。 如果我们定义多重集的总和，则每个操作都会变换$(a,b)$分成两份$\left\lfloor \frac{a+b}{2} \right\rfloor$，因此总和正好减少$(a+b) \bmod 2$。 这意味着总和改变的唯一方法是当所选对具有奇数和时丢失一个单位。 

这将视角从跟踪个人价值转变为跟踪总和减少。 由于最终状态的所有元素都等于$S$，最终的总和是$kS$。 因此，问题归结为理解我们可以使最终总和有多小，同时遵守每次操作最多只能丢失一个单位的约束，并且只有在奇偶校验允许的情况下。 

从初始配置的结构来看，基线上的总“多余质量”$x$集中在单个元素中$y$。 该过程允许将多余的部分重新分配给所有$k$头寸，每次以造成奇偶校验不匹配的方式重新分配时，我们都可能会损失一个单位。 最优策略有效地尽可能均匀地分配盈余，不可避免的下限来自于对总和进行平均$k$元素。 

这导致了一个封闭的形式：可实现的最佳最终值由平均总和的下限确定，并且在这个特定的二值配置中，它被彻底简化。

 ## 算法演练

 我们直接从全局不变量中得出答案，而不是模拟操作。 

1. 计算所有元素的初始总和。 该数组包含$k-1$的副本$x$和一份$y$，所以总和是$(k-1)x + y$。 
2. 观察每个操作都会保留元素的数量，因此最终的配置必须恰好包含$k$相同的值$S$，给出总和$kS$。 
3. 跟踪操作中总和的变化情况。 更换$a, b$有两份$\left\lfloor \frac{a+b}{2} \right\rfloor$将总和减少$(a+b) \bmod 2$，为 0 或 1。 
4. 由于我们可以重复选择对，因此我们总是可以实现尽可能均匀地分配值的操作序列。 唯一不可逆转的影响是奇偶校验不匹配造成的累积损失，它永远不会增加总和。 
5. 因此，最小可能的最终总和是最大的倍数$k$这不会超过初始总和，因为我们只能以单位步长减少总和，而不能低于均匀分配剩余质量的情况。 
6. 将其转换为最终答案$S = \left\lfloor \frac{(k-1)x + y}{k} \right\rfloor$。 

自从$(k-1)x + y = kx + (y-x)$，这简化为$S = x + \left\lfloor \frac{y-x}{k} \right\rfloor$。 

### 为什么它有效

 不变量是固定基数和总和的受控单调递减的组合。 每个操作要么保留总和，要么将其减一，并且任何操作都不能增加总和。 由于最终状态完全由重复的单个值决定$k$有时，总和完全表征了终止时的系统。 唯一的自由是在达到统一配置之前可以通过奇偶校验损失“花费”多少初始金额。 通过取与总和界限一致的最大可能统一值来精确地捕获该自由度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        k, x, y = map(int, input().split())
        s = (k - 1) * x + y
        print(s // k)

if __name__ == "__main__":
    solve()
```该实现直接对不变参数进行编码。 它不是模拟运算，而是计算初始总和并除以$k$，依赖于这样一个事实：该过程只能以整数步减少总和，直到达到统一的配置。 

关键细节是整数除法：由于最终值必须是整数并且总和只能减少，因此答案恰好是平均和的下限。 

## 工作示例

 考虑类似示例的配置$k=3, x=3, y=5$。 

| 步骤| 总和| 表达 |
 | ---| ---| ---|
 | 初始| 11 | 11$2\cdot 3 + 5$|
 | 每个元素的最终结果| 3 |$11 // 3 = 3$|

 这表明，即使我们从一个较大的值开始，我们能做的最好的是将总质量分布在三个元素上并取整数平均值。 

现在考虑$k=4, x=2, y=10$。 

| 步骤| 总和| 表达 |
 | ---| ---| ---|
 | 初始| 16 | 16$3\cdot 2 + 10$|
 | 每个元素的最终结果| 4 |$16 // 4 = 4$|

 这里多余的质量足够大，可以将最终的均匀值提高到上面$x$，并且平均过程将其充分展开。 

这些痕迹证实，该过程的行为类似于总质量的受控重新分配，仅具有向下舍入效应。

 ## 边缘情况

 最重要的边缘情况是当所有值都已经相等时，例如$k=5, x=y=7$。 该操作仍然适用，但不能更改任何值，因为对相同的数字进行平均可以保留它们。 该算法直接处理这个问题，因为总和是$5 \cdot 7$，除法返回$7$，匹配不变量。 

另一个边缘情况是当$y$仅略大于$x$， 例如$k=4, x=10, y=11$。 这里额外的质量只有 1，但它分布在 4 个位置上，因此在最终均匀状态下不可能将任何元素增加 1。 公式正确返回$10$，并且任何“分散”额外单位的尝试都会失败，因为它无法在所有元素上进行平均。 

最终的边缘情况发生在接近约束极限的大值处，其中$y-x$接近于$2k-1$。 即使在这种极端情况下，计算仍然保持稳定，因为一切都简化为单个整数除法，避免了溢出或迭代不稳定。
