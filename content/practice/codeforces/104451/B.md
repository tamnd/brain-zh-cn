 ## 问题理解

 我们得到了一组不同的整数，并且允许我们精确地选择一个连续的段并将其反转。 任务是判断是否存在某个段，其反转将整个数组变成严格递增序列，如果存在，则输出一个这样的段。 

输入只是数组的长度，后跟数组本身。 输出要么是对可以与段边界一起进行的确认，要么是单个反转无法对数组进行排序的声明。 

关键的结构限制是数组大小最多可达 100,000，因此任何尝试所有可能段的解决方案都是立即不可行的。 简单的 O(n3) 甚至 O(n2) 方法会超时，而且如果重复执行，即使 O(n2) 的反转模拟也太慢。 这迫使我们基于近排序数组的结构特性进行 O(n) 或 O(n log n) 推理。 

当数组已经排序时，会出现微妙的边缘情况。 在这种情况下，反转长度为 1 的段是有效的并且必须被接受。 另一个重要的极端情况是当未排序的部分非常小时，例如像这样的单个反转`[2, 1, 3, 4]`，其中仅存在一个交换等效段。 最后，在某些情况下，数组在本地看起来“几乎已排序”，但无法通过单次反转来修复，例如`[3, 1, 2, 4]`，任何逆转都会扰乱秩序的其他部分。 

## 方法

 蛮力策略是尝试每对可能的索引`l, r`，反转该段，并检查数组是否已排序。 每次检查需要 O(n)，并且有 O(n²) 段，总体上为 O(n³)。 即使通过预计算优化检查也不会消除候选数的二次，因此对于 n = 100,000 来说这太慢了。 

关键的见解是从“尝试所有逆转”转变为“描述有效逆转必须是什么样子”。 如果反转段对数组进行排序，则在该段之外，数组必须已按正确的顺序排序。 在段内部，顺序必须与排序数组完全相反，这意味着段在原始数组中必须严格递减。 

这导致了一种结构观察：如果存在解决方案，则该数组与其排序版本恰好在一个连续块中不同，并且该块是严格递减的。 在该块之外，数组必须已经与排序顺序匹配。 

因此，我们不是尝试操作，而是将数组与其排序版本进行比较并识别不匹配边界。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（尝试所有部分）| O(n3) | O(1) | O(1) | 太慢了|
 | 与排序+单段验证比较| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 创建数组的排序副本。 这代表成功反转后的目标最终配置。 
2.找到第一个索引`l`其中原始数组与排序数组不同。 这是必须包含在反转段中的最早位置，因为它之前的所有内容都已经正确。 
3.查找最后一个索引`r`其中原始数组与排序数组不同。 这是必须反转的段的最远边界，因为它之后的所有内容都已经正确。 
4. 反转子数组`l`到`r`在原始数组的副本中。 
5. 检查修改后的数组是否与排序后的数组完全匹配。 如果是这样，则该段`[l, r]`是有效的。 否则，任何一次逆转都无法修复阵列。 

我们可以安全地关注这个单一候选段的原因是，任何有效的解决方案都必须与数组及其排序版本之间的不匹配间隔完全对齐。 如果存在有效反转但与此间隔不同，则意味着不匹配之外的部分正在被更改，这将立即破坏其他地方的排序顺序。 

### 为什么它有效

 正确性取决于这样一个事实：通过一次反转进行排序不能在反转段之外引入新的正确排序的元素。 所选段之外的每个位置都保持固定。 因此，所有已经与排序数组匹配的位置都必须位于该段之外，并且所有不匹配的位置都必须位于该段之内。 这迫使反转段恰好是覆盖所有不匹配的连续区间。 一旦确定了这个区间，就只有一个候选反转可供测试，并且它要么有效，要么无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    b = sorted(a)

    l = 0
    while l < n and a[l] == b[l]:
        l += 1

    if l == n:
        print("yes")
        print(1, 1)
        return

    r = n - 1
    while r >= 0 and a[r] == b[r]:
        r -= 1

    segment = a[:]
    segment[l:r+1] = reversed(segment[l:r+1])

    if segment == b:
        print("yes")
        print(l + 1, r + 1)
    else:
        print("no")

if __name__ == "__main__":
    solve()
```该代码首先构建排序的引用数组，然后隔离第一个和最后一个不匹配的位置。 这些索引定义了可以修复数组的唯一合理的段。 反转副本中的该段就足够了，因为任何有效的解决方案都必须与此不匹配窗口一致。 

一个常见的错误是在不验证全局一致性的情况下假设任何递减段都是有效的。 针对排序数组的最终相等性检查至关重要，因为局部单调性不能保证段外的正确性。 

## 工作示例

 ### 示例 1

 输入：```
3
3 2 1
```排序后的数组是`[1, 2, 3]`。 第一个不匹配位于索引 0 处，最后一个不匹配位于索引 2 处。 

| 步骤| 我| r | 反转后的数组 |
 | --- | --- | --- | --- |
 | 初始| 0 | 2 | 3 2 1 | 3 2 1
 | 反向| 0 | 2 | 1 2 3 | 1 2 3

 反转后，数组与排序后的版本匹配，所以答案是`yes 1 3`。 

这证实了整个数组在严格递减时可以反转。 

### 示例 2

 输入：```
4
3 1 2 4
```排序后的数组是`[1 2 3 4]`。 不匹配区间是从索引 0 到 2。 

| 步骤| 我| r | 反转后的数组 |
 | --- | --- | --- | --- |
 | 初始| 0 | 2 | 3 1 2 4 | 3 1 2 4
 | 反向| 0 | 2 | 2 1 3 4 | 2 1 3 4

 结果未排序，因此单个反转不起作用。 

这表明，即使所有错配都是连续的，内部结构在反转后仍然必须完美对齐。

 ## 边缘情况

 当数组已经排序时，不匹配区间为空，算法将其视为简单的有效情况。 返回`[1, 1]`是正确的，因为反转长度为 1 的段会使数组保持不变。 

当数组完全递减时，失配区间跨越整个数组。 反转它会产生一个完全递增的序列，并且对排序数组进行检查以确认正确性。 

当不匹配形成一个连续的块但内部结构没有严格反转时，例如`[3, 1, 2, 4]`，算法仍然选择正确的区间，但最终验证失败。 这可以防止错误接受无法对数组进行全局排序的部分反转。
