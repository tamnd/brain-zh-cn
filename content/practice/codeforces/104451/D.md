 ## 解决方案
 ## 问题理解

 我们得到一个以字符串形式编写的数字，并被告知它已经满足严格的结构规则。 根据位置奇偶性，数字在两个固定值之间交替：位置 1、3、5 等中的所有数字都相同，位置 2、4、6 等中的所有数字都相同。 这两个数字可能重合，因此像 1111 这样的常量字符串也是有效的。 

任务是构造严格大于给定数字的最小数字，同时保持完全相同的交替结构和相同的长度。 

关键限制是长度可以达到 100000，这会立即排除任何尝试枚举候选者或以简单方式执行重复字符串构造的解决方案。 任何有效的方法都必须在线性时间内工作，因为即使 O(n log n) 具有大量常数也是可以接受的，但任何二次方法都会太慢。 

一个天真的解释是将数字视为标准整数并递增它，然后检查结果是否仍然满足交替约束。 这种方法立即在两个方面被打破。 首先，重复递增基于字符串的大数字对于每次操作都是线性的，并且可能会重复多次。 其次，大多数增量都会产生违反交替结构的数字，因此我们会浪费精力检查无效候选者。 

如果我们尝试按字典顺序生成长度为 n 的所有有效数字，则会出现更微妙的故障模式。 尽管每个有效数字仅由两位数字确定，但有 100 个可能的数字对，将所有数字与原始数字对进行比较仍然可以降级为 O(100·n)，这很好，但前提是要小心操作。 真正的问题是，如果没有结构化的转换规则，我们就有重复重建大型字符串的风险。 

## 方法

 数字的结构将整个问题简化为选择两位数字，一位用于奇数位置，一位用于偶数位置。 设它们为A和B。然后通过在奇数索引中重复A和在偶数索引中重复B来完全确定该数量。 

暴力方法是将给定的数字解释为（A，B），然后按字典顺序递增尝试所有可能的对（A'，B'），构造相应的交替数字，并返回第一个严格大于原始数字的数字。 这是正确的，因为每个有效数字都恰好对应一对这样的数字。 

然而，每个候选者的构造成本是 O(n)，并且最多有 100 对，所以这是 O(100n)。 虽然这可能会过去，但它是不必要的浪费，并且模糊了对之间的排序结构。 

关键的观察是两个交替数字之间的比较仅取决于它们不同的第一个位置。 由于奇偶校验是固定的，我们可以按字典顺序比较对，即重复放置在位置 1 和 2 的数字。 这意味着我们可以将问题视为在原始字符串定义的特定归纳顺序下查找下一个字典顺序更大的对（A，B）。 

我们可以直接模拟隐式对表示上的“下一个排列”思想，而不是枚举所有对。 我们从字符串的末尾开始扫描，寻找可以增加数字的位置，同时保持与奇偶校验约束的一致性，然后最小限度地重建后缀。 

这将问题减少到检查每个位置的少量恒定数量的候选修改。

 ## 算法演练

 我们将数字视为从 0 开始索引的交替位置。

1. 提取数字模式：所有偶数索引必须共享一位数字，所有奇数索引必须共享另一位数字。 我们从输入中将它们读为 A 和 B。 这给了我们当前的配置。 
2. 我们试图找到一种在保持结构的同时增加数量的方法。 由于增加较早的位置会比以后的任何更改产生更大的数字，因此我们从右向左扫描位置以找到可以安全地增加数字的最右边的位置。 
3.对于每个候选位置i，我们根据奇偶性来确定它是由A还是B控制。 我们尝试将该数字增加到尽可能小的高位数字。 
4. 一旦我们在位置 i 增加一位数字，所有之前的位置都保持不变。 对于i之后的所有位置，我们必须构建与交替结构一致的尽可能最小的后缀。 这意味着我们将每个剩余位置设置为与奇偶校验一致的最小允许 A 或 B，但尊重已经固定的所选数字。 
5. 我们构造结果字符串并立即返回它，因为从右到左的第一次成功修改保证了最小性。 

### 为什么它有效

 该算法依赖于字典顺序的位置优势属性。 较重要的索引的任何更改产生的数字严格大于较不重要的索引的任何更改。 通过从末尾扫描，我们保证只更改允许增加的最不重要的可能位置。 一旦该位置增加最少，贪婪地填充后缀可确保不会发生进一步不必要的增加。 这反映了下一个词典排列的逻辑，但仅限于二值周期结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    # Try all positions from right to left
    for i in range(n - 1, -1, -1):
        cur = ord(s[i]) - ord('0')

        # try increasing this digit
        for nd in range(cur + 1, 10):
            # build candidate
            res = list(s[:i] + chr(ord('0') + nd))

            # fill suffix respecting alternating structure
            for j in range(i + 1, n):
                if j % 2 == 0:
                    res.append(res[0])  # even index uses first position's digit
                else:
                    res.append(res[1])  # odd index uses second position's digit

            cand = "".join(res)

            if cand > s:
                print(cand)
                return

    # fallback (theoretically unnecessary for valid inputs)
    print(s)

if __name__ == "__main__":
    solve()
```该代码尝试增加最右边的可能数字并以最小一致的方式重建后缀。 后缀构造直接使用交替规则，依赖于这样的事实：一旦前缀固定，结构的其余部分就是确定性的。 

一个微妙的点是后缀是使用已经选择的前缀数字填充的。 这确保了与交替约束的一致性，而无需单独跟踪原始 A 和 B。

 ## 边缘情况

 一个关键的边缘情况是，整个字符串由 9 组成，而在不影响早期结构的情况下不可能增加这些 9。 例如，像 99999 这样的模式不能在不改变多个位置的情况下增加。 该算法通过从右向左扫描并最终无法找到有效增量来处理此问题，在这种情况下，它将返回原始字符串作为后备。 

另一种边缘情况是最佳更改发生在字符串开头附近。 在这种情况下，后缀重建可确保最小程度的完成，通过始终从最早的有效增加点进行重建来防止错误的大于必要的结果。
