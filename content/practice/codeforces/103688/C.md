 ## 解决方案
 ## 问题理解

 我们有一棵树`n`节点，每个节点携带一个整数值。 我们修复节点`1`作为特殊的根候选，我们需要决定是否可以将所有节点划分为两个不相交的组`A`和`B`这样单调约束沿着从节点开始的每条简单路径成立`1`。 

更准确地说，考虑任何节点`v`在树上。 查看来自节点的唯一路径`1`到`v`。 要求是在组内`A`，当远离时，值必须沿着该路径顺序严格增加`1`，并且在组内`B`，值必须严格沿着该路径顺序递减。 分区是全局的，这意味着每个节点都被分配到两个组之一，并且此分配必须使所有根到节点的路径与所选的排序规则一致。 

输出只是 root 是否存在这样的分区`1`。 

约束条件`n ≤ 100000`立即排除任何尝试显式测试分区的行为。 甚至检查所有`2^n`分配是不可能的，甚至子集上的本地 DP 也会失败。 任何有效的解决方案都必须与树的大小呈线性或接近线性，通常`O(n)`或者`O(n log n)`。 

由于认为每条路径都可以独立处理，因此出现了一个微妙的边缘情况。 例如，在一棵星形树中，其中节点`1`连接到许多节点，很容易根据每个子节点相对于其的值贪婪地分配它`a[1]`。 当子树通过祖先约束间接交互时，这种情况就会中断，因为条件不仅限于边缘，还取决于路径上所有祖先的顺序。 

另一种失败情况是假设我们可以通过仅将每个节点与其父节点进行比较来独立决定每个节点的成员资格。 这忽略了更深层次的约束：节点可能满足父子排序，但违反了相对于同一路径的祖父母的排序。 

## 方法

 暴力方法会尝试所有可能的节点分配`A`和`B`。 对于每个分配，我们将通过检查每个节点来验证条件`v`，步行路径从`1`到`v`，并在组内验证`A`该序列是严格递增的并且在`B`严格递减。 即使我们预先计算父指针，每次检查在深度上仍然是线性的，因此每次分配的总验证是`O(n^2)`在一棵链状的树上。 和`2^n`任务，这将变得天文数字。 

我们需要用对这些单调约束真正执行的结构观察来取代全局分配搜索。 关键思想是，条件只关心沿根路径的值的相对顺序，并将节点分成两个单调链：一个表现为从根开始的递增序列，另一个表现为从根开始的递减序列。 

如果我们修复一个节点`t = 1`，这个条件相当于说，对于每个节点，我们正在决定它是属于“从根递增”结构还是“从根递减”结构，并且这个决定必须沿着祖先链保持一致。 一旦我们将一个节点放入两个组之一，它就会限制如何放置其所有后代，因为任何后代路径都包含该节点。 

这导致了一个经典的转换：我们不考虑路径，而是以有向方式沿树边缘强制实施约束，其中每个节点必须与其父节点和其自己分配的单调类型兼容。 问题简化为检查我们是否可以为每个节点分配两个标签之一，以便每条边都遵循从值比较得出的本地可检查规则。 

重要的简化是，只有在将树扎根后，才需要比较节点与其父节点`1`。 如果存在有效的全局分区，则可以通过确保每个节点的选择与沿唯一根路径的排序约束一致来构造它，并且任何违规都必须已经出现在某些父子关系中。 这将全局路径约束转变为边缘上的局部一致性检查。

 ## 算法演练

 我们在节点处建立树根`1`并向下传播约束。 

1. 在节点处建立树的根`1`并使用 BFS 或 DFS 计算父子结构。 

此步骤是必要的，因为所有约束都是沿着从节点开始的路径定义的`1`，而生根将路径转换为祖先链。 
2. 对于每个节点，我们尝试为其分配两种状态之一，对应于它的行为是否像沿根路径递增组或递减组的一部分。 

根最初不受约束，但它定义了比较的起始参考。 
3. 从根开始按 BFS 或 DFS 顺序遍历树。 对于每条边`(u, v)`，我们确定哪些分配`v`与已经选择的分配兼容`u`。 

这是通过检查相对值来完成的`a[u]`和`a[v]`，因为单调条件迫使任何根到节点段的方向保持一致。 
4. 如果一个节点可以无矛盾地接受这两个分配，我们推迟决策。 如果只能需要一个，我们会修复它。 如果不能接受任何内容，我们立即得出结论：根无效。 

此步骤强制执行本地可行性传播，确保我们永远不会提交破坏路径约束的配置。 
5. 继续传播，直到处理完所有节点。 如果没有发现矛盾，则分区存在。 

中心思想是，一旦树有根并且约束被解释为单调方向选择，每个禁止的配置都会在某些父子边缘上出现立即不一致。 

### 为什么它有效

 不变量是处理完一个节点后`u`，其子树中的所有节点都具有与涉及的每个根到节点路径约束一致的分配`u`。 由于从节点开始的每条路径`1`对于任何通过唯一的边序列的节点，任何违反严格单调性的行为都必须首先发生在沿该路径的某些相邻对上。 因此，如果所有边都满足导出的局部约束，则路径不再可以引入局部尚未检测到的矛盾。 这将全局路径条件简化为一组边缘局部可行性检查。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        x, y = map(int, input().split())
        x -= 1
        y -= 1
        g[x].append(y)
        g[y].append(x)

    parent = [-1] * n
    parent[0] = 0
    order = [0]
    stack = [0]

    while stack:
        u = stack.pop()
        for v in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            stack.append(v)
            order.append(v)

    # dp[u][0/1]: whether u can be in group A (0) or B (1)
    dp = [[True, True] for _ in range(n)]

    for u in order[::-1]:
        for v in g[u]:
            if v == parent[u]:
                continue

            # propagate constraints from u to v
            # if we pick same group, check monotonic consistency
            new0 = new1 = False

            # try u in group 0 or 1 with v in group 0 or 1
            for gu in [0, 1]:
                for gv in [0, 1]:
                    if not dp[v][gv]:
                        continue
                    if not dp[u][gu]:
                        continue

                    ok = True
                    if gu == gv:
                        if gu == 0 and not (a[u] < a[v]):
                            ok = False
                        if gu == 1 and not (a[u] > a[v]):
                            ok = False

                    if ok:
                        if gu == 0:
                            new0 = True
                        else:
                            new1 = True

            dp[u][0] = dp[u][0] and new0
            dp[u][1] = dp[u][1] and new1

    # check root has at least one valid state
    print("YES" if (dp[0][0] or dp[0][1]) else "NO")

if __name__ == "__main__":
    solve()
```该实现使用基于堆栈的 DFS 构建有根树，然后尝试从叶子向上传播可行性。 这`dp[u][t]`数组表示是否有节点`u`可以分配到组`t`同时与其子树约束保持一致。 嵌套转换使用所需的严格不平等规则检查父分配和子分配之间的兼容性。 

关键的微妙之处在于，只有当两个相邻节点放置在同一组中时，比较才有意义，因为只有这样，严格的排序约束才会直接沿根路径的该段应用。

 ## 边缘情况

 一个重要的边缘情况是严格递增或严格递减的链。 例如，在路径中`1 - 2 - 3 - 4`有价值观`[4,3,2,1]`，每条边都满足一致的递减模式，因此所有节点可以放置在同一组中`B`。 该算法将所有边缘转移标记为在同一组规则下兼容，因此可行性干净地传播到根并返回`YES`。 

另一个边缘情况是以节点为中心的星形`1`其中子值是无序的。 由于每个子项仅相对于根受到约束，因此不存在子项到子项的约束。 DP 允许对每个子边进行独立分配，并且由于没有形成约束循环，因此根仍然有效。 

第三种边缘情况是当单个边缘违反强制组分配内的严格不平等时。 例如`1 - 2`和`a[1] = 5`和`a[2] = 5`立即使同组分配无效，不为节点留下有效状态`1`。 传播在叶级别捕获此问题，并且根立即变得无效，这表明违规行为是在本地检测到的，并且无法隐藏在树的更深处。
