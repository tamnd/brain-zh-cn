 ## 解决方案
 ## 问题理解

 我们在平面上得到三个固定点，每个点代表一个单位圆的中心（每个球的半径为 1）。 一球开始于$O_1$并且我们可以任意选择它的初始速度矢量。 该运动是理想化的：它沿直线行进，直到碰到以$O_2$，然后立即将运动转移到$O_2$根据简化的弹性碰撞规则，最后我们想要$O_2$移动并最终击中以$O_3$。 

当球从$O_1$首先触及周围的圆圈$O_2$。 到那时，一个特定的点$P$圆上的轨迹由轨迹决定。 碰撞发生后，$O_2$沿垂直于圆切线的直线运动$P$。 由于圆上一点的切线垂直于半径，这意味着$O_2$沿半径移动$O_2P$，即直接从中心$O_2$朝向接触点$P$。 

这个问题纯粹是存在主义的：是否存在任何初始方向？$O_1$这样引发的碰撞链$O_1 \rightarrow O_2 \rightarrow O_3$发生？ 

坐标大小方面的约束很小，但测试用例的数量很大。 这强烈表明$O(1)$或每个测试用例非常简单的几何检查。 任何涉及搜索方向或模拟轨迹的方法都会立即被排除，因为可能的方向集是连续的，并且强力离散化将会失败。 

一个微妙的边缘情况是声明中描述的“正好相切”的情况：如果$O_1$与圆相切$O_2$，那么根本就没有碰撞。 这对应于轨迹接触但不进入圆的简并配置。 任何正确的推理都必须明确避免将此类情况视为有效的首次碰撞。 

另一个重要的边缘情况是所有三个点共线。 在这种配置中，选择方向的几何自由度受到严格限制，并且每个候选轨迹都有可能命中$O_2$强迫$O_2$向着无法到达的方向移动$O_3$。 

## 方法

 考虑这个问题的简单方法是考虑从以下位置开始的可能初始方向的完整连续集$O_1$。 对于每个方向，我们模拟射线是否与以$O_2$，计算接触点$P$，得出碰撞后方向$O_2$作为半径方向$O_2P$，最后检查这条射线是否与圆相交于$O_3$。 每个模拟都涉及求解射线-圆相交并检查相切条件。 

即使每次模拟都是$O(1)$，真正的困难在于方向空间是连续的。 离散化方向需要极其精细的角分辨率，因为撞击圆的方向的有效间隔是连续的，并且禁止的切线方向是单个测量零约束。 这使得暴力破解不仅不正确而且在计算上毫无意义。 

关键的观察是第二阶段，$O_2$，仅取决于点$P$，位于周围的圆上$O_2$。 然而，可达方向的集合$O_2$正是所有光线来自$O_2$对应于其圆上可以被某条线击中的点$O_1$。 自从$O_1$可以任意瞄准周围几乎每个方向$O_2$除了单个简并切线配置之外，都是可以实现的。 

这将问题简化为检查是否存在来自$O_2$到其圆上的某个点，使得该方向上的射线与周围的圆相交$O_3$。 唯一的障碍是当几何体退化时，唯一可用的方向被由$O_1$。 

这将整个三步过程分解为一个几何一致性检查，涉及相对放置$O_1$,$O_2$， 和$O_3$。 最终条件仅取决于配置是否避免完全对齐的退化情况，其中所有有效碰撞方向都被迫进入单个禁止射线。

 ## 算法演练

 我们根据几何形状重新表述该过程$O_2$，因为第二次碰撞是真正的约束。 

1. 计算相对位置$O_1$,$O_2$， 和$O_3$。 我们移动坐标系，使得$O_2$是在原点。 这简化了所有推理，因为每个碰撞后运动都从$O_2$。 
2.观察任何有效的第一次碰撞都对应于选择一个点$P$在单位圆上$O_2$，相当于选择一个方向向量$d$从$O_2$与单位长度。 
3. 然后完全确定第二个运动：碰撞后，$O_2$沿着射线开始移动$O_2$方向$d$。 成功的条件是这条射线与以$O_3$。 
4. 确定方向的角度间隔$O_2$与周围的圆相交$O_3$。 因为两个圆的半径都是 1 并且不相交，所以当$O_2O_3 \ge 2$，形成连续的可行方向范围。 
5. 排除第一次碰撞变为相切的单个简并方向。 该禁止方向对应于进线的唯一方向$O_1$与圆相切于$P$，转化为所选方向上的单个几何约束$d$。 
6. 检查是否达到可行的角度间隔$O_3$包含至少一个方向不是由 引起的禁止切线方向$O_1$。 如果是，则回答是； 否则不行。 

关键思想是我们将有效方向的连续区间与单个排除方向相交。 除非区间完全塌陷到那个方向，否则我们总是可以自由地稍微调整轨迹。 

### 为什么它有效

 第一次碰撞后系统的状态仅取决于所选点$P$在 的 圆上$O_2$，相当于选择了一个方向$d$。 所有可能有效的集合$d$是一个整圆减去至多一个简并切线约束$O_1$。 成功的要求是至少一个这样的方向也允许光线从$O_2$与周围的圆相交$O_3$。 由于有效方向形成连续区间，并且禁止条件最多删除一个方向，因此存在解，除非几何形状强制可行区间精确地塌陷到禁止方向上，而这种情况仅发生在完全对齐的简并配置中。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dot(ax, ay, bx, by):
    return ax * bx + ay * by

def solve():
    t = int(input())
    for _ in range(t):
        x1, y1, x2, y2, x3, y3 = map(int, input().split())

        # shift so O2 is origin
        a1x, a1y = x1 - x2, y1 - y2
        a3x, a3y = x3 - x2, y3 - y2

        # collinearity check between O1, O2, O3
        # if all three lie on a line through O2, configuration is degenerate
        cross = a1x * a3y - a1y * a3x

        if cross == 0:
            print("no")
        else:
            print("yes")

if __name__ == "__main__":
    solve()
```该实现会改变坐标，以便所有推理都发生在$O_2$。 这使得几何体居中并消除了对绝对位置的不必要的依赖。 

唯一真正的结构故障情况是$O_1$,$O_2$， 和$O_3$共线。 在这种情况下，任何成功触发第一次碰撞的方向都会受到约束$O_2$沿着无法调整到达的线移动$O_3$在不违反相切约束的情况下，破坏可用的角度自由度。 叉积准确地检测到了这种简并性。

 ## 边缘情况

 当三点在同一条直线上时，移动坐标系使两个向量$O_1 - O_2$和$O_3 - O_2$共线。 在这种情况下，每一个可行的方向$O_2$被迫进入单轴对齐的射线，任何引入轻微偏差的尝试都会破坏碰撞链结构。 在这种情况下，叉积恰好为零，并且算法正确地输出“否”。 

对于任何非共线配置，矢量跨越具有至少一个角度自由度的平面方向空间。 这允许从以下位置选择有效的碰撞方向$O_2$避免了单切线简并性并且仍然与周围的目标圆相交$O_3$，确保存在有效的链并且算法输出“是”。
