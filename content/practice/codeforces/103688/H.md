 ## 解决方案
 ## 问题理解

 我们得到一个从 1 到 n 索引的单词序列，旁边还有一个相同长度的字符串，该字符串由三个可能的字符组成：左括号、右括号和破折号。 

括号形成正确匹配的结构。 每个左括号都有一个唯一匹配的右括号，并且在每个前缀处，左括号的数量永远不会少于右括号的数量。 

该字符串不直接描述计算。 相反，它定义了索引 1 到 n 的遍历规则。 我们从位置 1 开始并生成所有索引的读取顺序。 该规则是递归的：在每个位置 i，根据字符，我们要么立即输出 i，要么使用匹配的括号间隔以结构化方式延迟它。 

如果字符是破折号或右括号，我们只需输出当前索引并继续前进。 如果它是左括号，我们在位置 j 处找到其匹配的右括号。 在这种情况下，我们首先递归处理 i+1 和 j−1 之间的段，然后输出 i 本身，最后从 j+1 继续。 这将创建一个嵌套遍历，类似于以树形式计算表达式的方式，只不过结构是由有效的括号序列隐式编码的。 

输出是从 1 到 n 的索引排列，表示在此递归规则下访问位置的确切顺序。 

高达 100000 的约束 n 强制任何解都是线性或接近线性的。 任何重复搜索匹配括号或在没有记忆的情况下递归地重新处理段的模拟都会面临二次行为的风险。 按需扫描匹配括号或重复切片子字符串的简单递归会由于重叠间隔的重复工作而超出限制。 

当存在深层嵌套的括号时，例如像“((((-))))...”这样的字符串，就会出现朴素方法的微妙失败情况。 如果通过向前扫描来搜索每个匹配项，则每个级别的成本为 O(n)，从而导致 O(n^2)。 

另一个失败案例来自于递归顺序的错误处理。 例如，统一处理所有字符并简单地将索引推入堆栈将忽略“(”的特殊“process inside before self”行为，这会反转相对于线性扫描的本地排序。

 ## 方法

 模拟该过程的直接方法是按照字面意思进行定义。 我们编写一个递归函数，给定一个范围，从左到右迭代它。 当遇到破折号或右括号时，我们会附加索引。 当遇到左括号时，我们首先找到与其匹配的右括号，递归处理内部段，然后追加左索引并在右括号之后继续。 

这种做法是正确的，但效率低下。 瓶颈是重复查找匹配的括号。 即使我们使用 O(n) 的堆栈预先计算匹配，递归本身仍然对所有段执行结构化遍历。 如果不小心实现切片或重复扫描，隐藏常量就会变大，并且 Python 递归开销可能会出现问题。 

关键的观察结果是括号引发的结构是一棵树。 每个左括号对应一个节点，该节点的子节点是其匹配区间内的元素。 遍历规则有效：首先访问子节点，然后访问节点本身，然后继续访问下一个兄弟段。 这是对树的类似后序的遍历，其中节点是括号位置，叶子是破折号或结束符。

一旦我们认识到这一点，我们就可以预先计算匹配的括号，然后使用堆栈迭代地模拟遍历。 每个堆栈帧代表我们仍然需要处理的一个段，并且我们显式控制扩展嵌套段的顺序。

 | --- | --- | --- | --- |
 | 具有重复匹配搜索的朴素递归 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 基于堆栈的区间模拟 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先使用堆栈预先计算每个开头位置的匹配括号。 这使我们可以直接 O(1) 访问任何嵌套段的端点。 

然后，我们使用显式的间隔堆栈来模拟遍历，始终控制我们是处于正常的从左到右扫描模式还是处于特殊的括号扩展内。 

1. 我们计算一个数组匹配，其中 match[i] 给出与 i 对应的右括号的索引（如果 i 是 '('）。
 2. 我们用代表从 1 到 n 的完整范围的单个区间初始化堆栈，并将其标记为应该从左到右处理的正常段。 
3. 当堆栈不为空时，我们弹出一个区间。 我们从左到右扫描它。 
4. 当我们遇到'-'或')'时，我们立即将其索引附加到答案序列中，因为规则指定直接读取。 
5. 当我们遇到'('时，我们首先递归地处理它的内部区间[i+1, match[i]−1]。这必须在输出i之前发生，所以我们将当前的延续状态压入堆栈，然后将内部区间压入堆栈。
 6. 完成内部段后，我们将 i 本身追加到输出中，然后在 match[i] 之后继续扫描。 

推送的顺序确保内部段始终在开括号位置本身之前得到处理。 

### 为什么它有效

 括号结构定义了一个有根有序树，其中每个左括号都是一个节点，其子节点是其中未在同一级别进一步嵌套的最大连续段。 遍历规则正是对这棵隐式树的后序遍历：在输出父节点之前完全处理子节点，并按从左到右的顺序处理兄弟节点。 

堆栈模拟通过确保每当我们输入 '(' 段时，我们在输出 '(' 索引之前完全处理其内部间隔，然后才恢复外部扫描，从而强制执行这种排序。由于每个索引恰好属于每个嵌套级别的一个段，因此不会跳过或重复任何元素。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    match = [-1] * n
    st = []

    for i, ch in enumerate(s):
        if ch == '(':
            st.append(i)
        elif ch == ')':
            j = st.pop()
            match[i] = j
            match[j] = i

    res = []
    sys.setrecursionlimit(10**7)

    def dfs(l, r):
        i = l
        while i <= r:
            if s[i] == '(':
                j = match[i]
                dfs(i + 1, j - 1)
                res.append(i + 1)
                i = j + 1
            else:
                res.append(i + 1)
                i += 1

    dfs(0, n - 1)
    print(*res)

if __name__ == "__main__":
    solve()
```该解决方案首先使用线性时间内的堆栈配对括号。 每个右括号检索最近不匹配的左括号，由于输入字符串的有效性而保证正确匹配。 

递归函数`dfs(l, r)`实现对段的遍历。 它线性扫描整个区间。 当它看到非括号分支字符时，它立即输出。 当它看到左括号时，它会在附加左索引本身之前递归地处理内部间隔。 由于问题中索引是从 1 开始的，因此需要将输出移位 +1。 

关键的实现细节是指针`i`根据是否处理了括号，前进的方式有所不同。 处理完括号块后，我们直接跳转到其匹配的结束边界，以避免重新访问内部结构。

 ## 边缘情况

 完全嵌套的结构，例如`(((())))`每个级别内都有交错的破折号，强调匹配和递归顺序。 该算法将每个左括号推入堆栈一次，并在其段完成时精确解析一次。 尽管递归下降了多层，但每个索引在每个嵌套层中只参与一次进入和一次退出操作，因此不会发生重复扫描。 

扁平结构如`-----`是相反的极端。 这里根本没有括号，因此算法退化为简单的线性输出。 扫描循环处理每个字符一次并按顺序附加索引，确认括号逻辑不会干扰非括号段。 

混合交替图案，例如`(-)-(-)-(-)`验证递归段和线性段之间的正确切换。 每个括号内的块都由匹配边界隔离，因此在完成一个块后，指针会在下一个段正确恢复，而无需重新处理内部字符。
