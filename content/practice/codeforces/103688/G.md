 ## 解决方案
 ## 问题理解

 我们得到了 n 颗珍珠的圆形排列。 每颗珍珠 i 都有一个非负整数值 ci。 该过程是交互式的，因为我们反复选择起始珍珠 i，但前提是 ci 至少为 1 并且当前仍然存在足够的珍珠。 

当我们选择珍珠i时，我们按顺时针顺序删除从i开始的连续的ci珍珠块，并且该块中的第一颗珍珠此时必须仍然存在。 移除后，剩余的珍珠闭合成一个新的圆圈，保持相对顺序。 

我们继续下去，直到不存在有效的移动。 仅当所选的起始珍珠仍然存在并且其所需的块大小不超过当前剩余珍珠的数量时，移动才是合法的。 

任务是最大化移除珍珠的数量。 在实现此最大值的所有方法中，我们计算存在多少个不同的解决方案，其中解决方案仅由每个步骤使用的起始索引集定义，忽略顺序。 

约束 n ≤ 2000 和 ci ≤ 2000 表明 O(n3) 或 O(n2 log n) 动态编程方法在 C++ 中可能勉强通过，但在 Python 中却不行。 需要接近 O(n²) 或 O(n² log n) 的解决方案，并且任何涉及直接枚举删除序列的操作都是不可能的，因为序列的数量随 n 呈指数增长。 

一个微妙的困难是移除会动态改变圆形结构。 试图逐步模拟该过程的天真解释很快就会变得模棱两可，因为“顺时针下一颗珍珠”的身份在每次删除后都会发生变化。 

对于大多数珍珠来说，一个关键的边缘情况是 ci = 0。 例如，如果所有 ci = 0，则不可能进行任何移动，答案为 0 1，因为空序列是唯一有效的解。 另一种边缘情况是当一个 ci 很大时，例如 n = 5，c1 = 5。然后在 1 处删除会立即删除所有内容，但根据剩余大小，从其他位置开始可能是非法的，因此解决方案的集合定义非常重要。 

## 方法

 暴力方法将模拟所有可能的有效删除序列。 在每个状态，我们选择任何有效的起始珍珠并递归地删除一个块，更新循环结构。 

这在原则上是正确的，因为它完全遵循流程。 然而，国家空间是巨大的。 即使表示圆圈也需要跟踪剩余的珍珠，并且有 2ⁿ 可能的子集。 每个转换还取决于循环邻接，这使得在不编码完整循环结构的情况下简单记忆变得困难。 

即使我们在子集上尝试 DP，每个状态都需要 O(n) 次转换，导致 O(n 2ⁿ)，这是完全不可行的。 

关键的观察结果是，圆形结构仅与仍然存活的珍珠的间隔有关。 一旦我们确定了删除的顺序，每个操作都会删除循环序列的一个连续段，这表明间隔 DP。 然而，真正的简化来自于颠倒这个过程。 

我们不考虑删除线段，而是考虑通过在固定的初始圆中选择不相交的间隔来构造有效的删除序列。 每个操作都会消耗从 i 开始的长度为 ci 的线段，这意味着在原始循环中，每个选择的操作对应于选择一个区间 [i, i+ci−1] (mod n)，并且这些区间在最终的序列顺序中不得重叠。 

这将问题转化为在圆上选择最大一组加权间隔，其中每个间隔都有权重 ci ，并且还强制该间隔仅在 ci ≤ 当前可用线段大小时才有效。 关键的见解是，最佳策略对应于在固定断点后在圆的线性化版本中选择一组不重叠的区间。

我们在每个可能的起始位置打断圆，将其转换为线性数组，并计算最佳间隔调度DP。 每个间隔 i 定义一个以 i+ci−1 结束的段。 然后，我们求解加权间隔调度变体，其中权重为 ci，但这里所有权重也是对目标的贡献。 

确定移除珍珠的最大数量后，计数解决方案就成为最佳过渡上的标准 DP 计数问题。

 ## 算法演练

 1. 固定一个索引作为圆的起始切割点，并复制数组以处理环绕。 这将圆形结构变成长度为 2n 的线性阵列。 
2. 对于每一个可能的间隔起始 i，定义一个以 j = i + ci − 1 结束的移除间隔。如果 j 超过 i + n − 1，则丢弃它，因为它会包裹多个完整周期，这在线性化后无效。 
3. 现在，我们对每个固定旋转的段 [0, n−1] 执行 DP。 DP 状态 dp[i] 表示从线性阵列中的位置 i 开始的最佳结果，意味着从 i 开始可实现的最大移除珍珠数。 
4. 转换是跳过位置 i 或使用从 i 开始的间隔。 如果我们使用它，我们会跳转到 i + ci 并将 ci 添加到分数中。 
5.为了保证正确性，必须保证区间不重叠。 这是自然强制的，因为在选择一个间隔后，我们会向前移动到其终点之外。 
6. 与 dp 一起，维护 cnt[i]，即实现 dp[i] 的方法数量。 当多个转换产生相同的 dp 值时，我们将它们的计数以 998244353 为模求和。 
7. 计算完所有旋转的 dp 后，取所有旋转中的最大 dp[0]。 答案就是这个最大值和相应的计数。 

关键的不变量是 dp[i] 始终表示所选线性化中从 i 开始的后缀的最优解。 每个有效的移除序列恰好对应于一个区间选择链，并且每个链都对应于原始循环中的有效移除序列。 因为我们考虑了所有旋转，所以不会错过任何有效的循环解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input().strip())
    c = list(map(int, input().split()))
    
    a = c * 2
    
    best_ans = 0
    best_cnt = 0

    for start in range(n):
        dp = [0] * (2 * n + 1)
        cnt = [0] * (2 * n + 1)
        dp[2*n] = 0
        cnt[2*n] = 1

        for i in range(2 * n - 1, start - 1, -1):
            if i >= start + n:
                dp[i] = 0
                cnt[i] = 1
                continue

            # option 1: skip
            best = dp[i + 1]
            ways = cnt[i + 1]

            # option 2: take interval
            if a[i] > 0:
                j = i + a[i]
                if j <= start + n:
                    val = a[i] + dp[j]
                    if val > best:
                        best = val
                        ways = cnt[j]
                    elif val == best:
                        ways = (ways + cnt[j]) % MOD

            dp[i] = best
            cnt[i] = ways % MOD

        if dp[start] > best_ans:
            best_ans = dp[start]
            best_cnt = cnt[start]
        elif dp[start] == best_ans:
            best_cnt = (best_cnt + cnt[start]) % MOD

    print(best_ans, best_cnt)

if __name__ == "__main__":
    solve()
```该代码显式地实现了旋转技巧。 该数组加倍，以便任何循环间隔都成为线性段。 对于每个起始位置，我们运行一个后缀 DP，它要么跳过一个位置，要么从该位置开始获取移除间隔。 

dp 数组存储给定索引的最大可移动总和，cnt 存储有多少种方法可以实现该值。 向后迭代确保在计算状态 i 时，所有未来状态都已计算完毕。 

当确保间隔不超出所选的 n 长度窗口时，边界处理至关重要。 这是通过检查 j ≤ start + n 来强制执行的。 

最终答案汇总了所有旋转，因为圆的初始切割是任意的，并且每个有效的解决方案在一次旋转对齐中都有唯一的表示。

 ## 边缘情况

 当所有 ci 都为零时，DP 不会占用任何间隔。 该算法正确初始化 dp 转换，以便仅存在跳跃转换，产生 dp[0] = 0 和 cnt = 1。 

当单个 ci 等于 n 时，该间隔会消耗整个选定的旋转窗口。 在DP中，采取这个区间直接跳到结束边界，得到n的满分。 由于不可能存在重叠间隔，因此该轮换的计数恰好保持为 1。 

当多个等长间隔重叠时，skip-vs-take DP 确保只有非重叠链才有助于有效序列。 任何尝试获取间隔都会强制跳转以防止重叠，因此在状态转换期间永远不会形成无效组合。
