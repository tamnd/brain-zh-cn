 ## 解决方案
 ## 问题理解

 我们有两种类型的书籍，它们在直立放置时宽度相同：每本书恰好占据书架宽度的一个单位。 区别在于高度。 A型书有高度`a`，而B型书则随着高度变高`b`， 在哪里`a < b`。 我们还有一个最大允许高度的书架`h`，我们被告知`h`至少是`b`，因此两种类型都可以直立而不会违反高度限制。 

最初，所有书籍都垂直放置在一行中：所有`n`首先是 A 类书籍，然后是所有书籍`m`B类书籍。 在这种配置中，总宽度很简单`n + m`。 

不同的是，我们可以从右侧拿一些 B 型书，特别是`k`书籍与`0 ≤ k ≤ m - 1`，然后水平旋转它们。 这些旋转的书籍放置在现有垂直排列的顶部，这改变了高度的消耗方式，但消除了它们作为独立垂直书籍对水平宽度的贡献。 

目标是选择旋转多少本 B 类图书，以便最终所需的书架宽度尽可能小，同时仍遵守高度限制`h`。 

约束条件很大，最多可达`10^3`测试用例和值`n`和`m`最多`10^9`，因此任何解决方案都必须在每个测试用例的恒定时间内运行。 这立即排除了对书籍的任何模拟或任何迭代可能值的方法`k`。 

当推理高度和旋转书籍数量之间的相互作用时，出现了一个微妙的失败案例。 例如，如果`h`仅略大于`b`，那么即使有很多可用的 B 书，也只有极少数 B 书可以轮换。 相反，如果`h`很大，我们仅受约束的限制`k ≤ m - 1`。 

一个天真的错误是假设我们总是可以旋转除一本之外的所有 B 书，将宽度减小到`n + 1`。 当堆叠太多水平书籍超过高度限制时，此操作会失败，而高度限制是控制可行性的真正约束。 

## 方法

 蛮力策略会尝试一切可能的方法`k`从`0`到`m - 1`，计算是否旋转`k`books 在高度约束下有效，然后计算得到的宽度`n + m - k`。 问题是检查每个项目的可行性`k`需要对水平堆叠的书籍如何增加高度进行建模。 即使每次检查都是 O(1)，迭代至`10^9`的值`k`是不可能的。 

关键的见解是这种安排是完全单调的。 每增加一本旋转的 B 型书都会在书架的同一区域增加完全相同的高度压力。 这意味着一旦一定数量的B书可以在不超过高度的情况下旋转`h`，任何较小的数字也有效，任何较大的数字无效。 

这将问题简化为寻找最大可行值`k`。 该结构进一步简化，因为唯一的高度交互来自于将水平书籍堆叠在现有最高垂直书籍（即 B 类书籍）之上。 每本书在已有高度的区域上贡献一个额外的高度单位`b`，所以堆叠`k`此类书籍增加了该地区的有效高度要求`b + k`。 约束条件`b + k ≤ h`直接限制了答案。 

我们还必须尊重不能轮换所有 B 书的条件，所以`k ≤ m - 1`。 

一旦达到最大有效`k`确定后，最终宽度只需减少`k`，因为每本旋转的书都会删除一个垂直宽度的贡献。

 ## 算法演练

 1. 请注意，旋转 B 型书籍会将其从垂直布局中移除，因此每次有效旋转都会使总宽度恰好减少一个单位。 这就把问题变成了求B书可以轮换的最大数量。 
2. 确定旋转 B 本书时高度如何变化。 每个水平 B 都会在现有 B 高度堆叠的顶部贡献一个额外的高度单位，因此堆叠`k`旋转 B 书强制高度为`b + k`在受影响的地区。 
3. 通过要求强制执行高度限制`b + k ≤ h`。 这给出了上限`k ≤ h - b`。 
4. 强制执行问题约束，即我们不能轮换所有 B 书，给出`k ≤ m - 1`。 
5. 结合两个约束以获得最大可行旋转数：`k = min(m - 1, h - b)`。 
6. 通过从原始宽度中减去旋转书籍的数量来计算最终宽度：`w = n + m - k`。 

### 为什么它有效

 问题的结构确保所有旋转的书籍以相同且累积的方式与高度相互作用。 以不同方式分配旋转没有任何好处，因为每增加一个水平 B 书都会为限制高度区域贡献一个统一的 +1。 这使得可行性仅取决于旋转书籍的数量，而不是它们的位置。 结果，有效集合`k`形成一个从零开始的前缀区间，并最大化`k`足以最小化宽度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        a, b, n, m, h = map(int, input().split())
        
        # maximum number of B books we can rotate
        k = min(m - 1, h - b)
        if k < 0:
            k = 0
        
        # final width after removing k books from vertical layout
        print(n + m - k)

if __name__ == "__main__":
    solve()
```该解决方案读取每个测试用例并在恒定时间内计算答案。 唯一重要的步骤是正确限制`k`。 表达式`h - b`捕获在超过货架高度之前可以堆叠多少额外的水平层。 我们也夹`k`至少为零，因为如果`h == b`，不可能水平堆叠。 

一个常见的错误是忘记`m - 1`约束，防止旋转所有 B 书。 另一个错误是将水平放置误解为对宽度的乘法影响。 实际上，它只影响高度并将这些书从垂直宽度计数中删除。

 ## 边缘情况

 当`h`等于`b`，堆叠水平书籍的高度预算变为零。 在这种情况下，公式得出`k = min(m - 1, 0) = 0`，所以宽度保持不变`n + m`。 这符合物理解释，即即使是一本水平书也会立即超过堆叠区域中允许的高度。 

什么时候`m = 1`，约束条件`k ≤ m - 1`力量`k = 0`与身高无关。 该实现自然地处理了这个问题，因为`min(0, h - b)`始终为零或负数，我们将其钳位为零。 

什么时候`h`非常大，限制因素变为`m - 1`，这意味着我们可以轮换几乎所有 B 书。 最终宽度变为`n + 1`，这与在垂直排列中仅保留一本未旋转的 B 书是一致的。
