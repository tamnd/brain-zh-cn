 ## 解决方案
 ## 问题理解

 我们得到一个网格，其中每个单元格都包含一个符号，描述它如何参与由 L 形三联骨牌组成的平铺。 每个 L 形由一个中心单元组成，标记为`C`和三个相邻的臂在四个基本方向上延伸，标记为`U`,`D`,`L`， 和`R`。 有效的配置必须由不相交的完整 L 形组成，覆盖除可能的一个特殊空单元之外的所有单元。 仅当该空单元格位于网格的右上角时才允许使用，否则每个单元格必须恰好属于一个 L 形状，并且每个 L 形状必须完全一致。 

每个非中心单元通过定向指向其中心来编码它属于哪个中心。 一个`U`细胞声称其中心位于其正上方，`D`单元格声称其中心位于其正下方，左侧和右侧也是如此。 有效的配置必须满足一致性和完整性：每个臂必须指向真实的`C`， 每一个`C`必须形成一个有效的 L 形状，并且恰好有 3 个臂围绕它，并且任何单元格都不能被多个 L 形状所占据。 

输出是每个测试用例的简单有效性检查，打印`Yes`如果整个网格在这些规则下形成完美的平铺并且`No`否则。 

总网格面积的约束很大，所有测试用例最多有 10^6 个单元格。 这立即排除了任何对每个单元执行重复探索或任何回溯的解决方案。 该结构提出了一种线性扫描解决方案，其中每个单元都被处理恒定的次数。 

一些失败案例很容易被忽视。 

一种情况是不完整的L形。 例如，一个`C`在位置 (2,2) 处可能在一个方向上没有有效的邻居，或者可能缺少一只手臂。 

另一种情况是相互矛盾的主张。 两个不同的中心可能声称相同的臂细胞是其 L 形的一部分，必须检测到这一点。 

第三种情况是无效的方向指针。 单元格可能指向网格外部或指向非中心单元格。 

最后，特殊的空单元格规则很微妙。 一个网格可能恰好包含一个`.`但只有出现在 (1, m) 时才有效。 即使其他一切都正确，任何其他位置也是无效的。 

## 方法

 验证网格的直接方法是显式地重建每个 L 形状，从每个`C`。 对于每个中心，我们尝试扩展其四个臂，检查相应的单元格是否存在并正确指向回，并将它们标记为已访问。 处理完所有中心后，我们验证每个非空单元格都被访问过一次，并且没有单元格被共享。 

这种强力扫描已经接近最佳状态，因为每个细胞最多参与一个 L 形，因此每个臂都被验证了恒定的次数。 检查总数与网格大小成正比。 然而，粗心的实现可能会意外地重复扫描邻居或对每个单元执行递归验证，从而导致不必要的开销。 

关键的观察结果是，每个有效配置都定义了确定性映射：每个非中心单元通过其方向唯一地标识其中心，并且每个中心确定性地定义其四个邻居。 这使我们能够在本地进行验证，而无需全局搜索。 我们不探索结构，只验证每个单元的约束并确保相互指针的一致性。 

因此，问题简化为一次性验证三个条件。 首先，每`C`必须在正确的位置形成正好四个有效的臂。 其次，每个定向单元必须正确指向`C`还有那个`C`必须同意相反的关系。 第三，每个单元格必须只计算一次。 

这减少了从图形构建到网格上的本地一致性检查的问题。

 | ---| ---| ---| ---|
 | 每个中心的强力 L 形重建 | O(纳米) | O(纳米) | 已接受 |
 | 本地一致性验证 | O(纳米) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将网格视为局部约束系统，并根据每个单元的角色对其进行验证。 

1. 我们首先判断是否存在单个空单元格。 如果恰好有一个`.`我们检查它是否位于第 1 行第 m 列。 如果有零个点或点错位，我们会立即拒绝或根据确切的规则解释进行相应处理。 此步骤将特殊例外与 L 形的结构约束隔离开来。 
2. 我们迭代网格中的每个单元格。 每当我们遇到一个`C`，我们尝试将其验证为 L 形的中心。 四个预期臂在四个方向上直接相邻的单元。 
3. 对于围绕 a 的四个方向中的每一个`C`，我们同时检查两个条件。 邻居必须位于网格内部，并且它必须包含指向中心的正确方向字符。 例如，上面的单元格必须包含`D`，下面的单元格必须包含`U`，左边必须包含`R`，并且右边必须包含`L`。 这确保了中心和手臂之间的双向一致性。 
4.如果所需的四个邻居中的任何一个无效或不匹配，我们立即拒绝，因为中心没有形成完整的L形。 
5. 我们维护一个访问数组，标记属于经过验证的 L 形的所有单元格。 每次我们成功验证一个中心及其四个臂时，我们都会将所有五个相关单元标记为已使用。 
6. 处理完所有中心后，我们验证网格中的每个单元格要么是有效 L 形状的一部分，要么是允许的空单元格。 任何未访问或多次使用的小区都意味着无效重叠或丢失覆盖。 

### 为什么它有效

 核心不变量是每个有效的配置都强制中心及其四个臂之间的严格双射。 每个`C`必须被恰好四个唯一确定的邻居包围，并且每个臂单元必须恰好指向一个中心。 因为这些关系是确定性的和局部的，所以独立验证每个中心不能错过全局的不一致。 如果两个中心重叠，某些单元格将被标记两次或无法达到方向一致性。 如果中心不完整，则至少有一项邻接检查失败。 因此，局部检查足以保证全局正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

DIRS = {
    'U': (-1, 0, 'D'),
    'D': (1, 0, 'U'),
    'L': (0, -1, 'R'),
    'R': (0, 1, 'L')
}

def solve():
    T = int(input())
    for _ in range(T):
        n, m = map(int, input().split())
        g = [input().strip() for _ in range(n)]

        visited = [[False] * m for _ in range(n)]
        dot_count = 0
        dot_pos = (-1, -1)

        for i in range(n):
            for j in range(m):
                if g[i][j] == '.':
                    dot_count += 1
                    dot_pos = (i, j)

        if dot_count not in (0, 1):
            print("No")
            continue

        if dot_count == 1:
            if dot_pos != (0, m - 1):
                print("No")
                continue

        ok = True

        for i in range(n):
            for j in range(m):
                if g[i][j] != 'C':
                    continue

                cells = [(i, j)]
                for c, (di, dj, need) in DIRS.items():
                    ni, nj = i + di, j + dj
                    if ni < 0 or ni >= n or nj < 0 or nj >= m:
                        ok = False
                        break
                    if g[ni][nj] != need:
                        ok = False
                        break
                    cells.append((ni, nj))

                if not ok:
                    break

                for x, y in cells:
                    if visited[x][y]:
                        ok = False
                        break
                    visited[x][y] = True

            if not ok:
                break

        if not ok:
            print("No")
            continue

        for i in range(n):
            for j in range(m):
                if g[i][j] == '.':
                    continue
                if not visited[i][j]:
                    ok = False
                    break
            if not ok:
                break

        print("Yes" if ok else "No")

if __name__ == "__main__":
    solve()
```该解决方案首先对可选的点单元进行计数和验证，因为它是网格中唯一的非 L 形例外。 尽早隔离此检查以避免干扰结构验证。 

主循环处理每个`C`作为一个潜在的L型中心。 对于每个中心，它使用固定偏移量直接检查其四个邻居。 方向字母和所需相反符号之间的映射确保一致性：如果邻居声称连接到给定方向的中心，则该关系必须是对称的。 

被访问的数组对于强制不相交至关重要。 如果没有它，重叠的 L 形将无法可靠地检测到，因为单独的本地检查并不能阻止单元的共享使用。 

最后，全扫描可确保完整性：每个非点单元必须恰好属于一个有效的 L 形。 

## 工作示例

 考虑一个小的有效网格：```
C L
D R
```这是一个以 (0,0) 为中心的 L 形。 

| 步骤| 单元 (0,0) | 右 (0,1) | 向下 (1,0) | 访问更新 |
 | ---| ---| ---| ---| ---|
 | 验证 C | 好的 | 预期 L | 预期 D | 标记所有 4 个单元格 |

 处理后，所有非点单元都被恰好覆盖一次。 

现在考虑无效重叠：```
C L C
D R D
```两个中心试图共享相同的手臂结构。 

| 步骤| 第一个 C 有效 | 第二个 C 有效 | 冲突|
 | ---| ---| ---| ---|
 | 首先处理 C | 标记 (0,0),(0,1),(1,0) | - | - |
 | 处理第二个C | - | 重叠于 (0,1) 或 (1,0) | 访问冲突|

 第二个中心失败，因为它所需的至少一个单元已被标记为已访问。

 ## 边缘情况

 一种边缘情况是当`C`被放置在边界上。 例如，位于 (0,0) 的中心需要上方和左侧的邻居，这些邻居超出了范围。 该算法在访问邻居之前显式检查网格边界，因此这会立即触发拒绝。 

另一种边缘情况是网格完全充满非中心定向单元且没有`C`。 在这种情况下，不会验证任何 L 形，因此所有单元格都未被访问。 最终扫描检测到这一点并拒绝配置。 

第三种边缘情况是允许位置 (1, m) 处的单个点。 由于点单元在访问检查中被忽略，因此它们不会干扰验证，并且网格的其余部分仍然必须形成完整的平铺。
