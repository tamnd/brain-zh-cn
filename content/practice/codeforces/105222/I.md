 ## 解决方案
 ## 问题理解

 我们有一个矩形甲板，充当固定的 2D 容器，其左下角位于原点，右上角位于$(l, h)$。 在这个空间中，我们必须放置一系列轴对齐的矩形框。 每个盒子都有固定的方向，这意味着它不能旋转，并且它必须完全位于甲板内，不能与任何先前放置的盒子重叠。 

箱子按照严格的顺序到达。 对于每个盒子，如果当时不存在有效的放置，我们必须放置它或立即丢弃它。 放置是贪婪且确定性的：在框适合且不与先前框重叠且保持在边界内的所有有效位置中，我们选择左下角 x 坐标最小的位置。 如果存在多个这样的位置，我们通过选择最小的 y 坐标来打破联系。 

关键的困难在于，有效位置集在每次放置后都会发生变化，因为每个放置的矩形都会阻挡未来的空间。 该过程是在线的，因此决定是不可逆转的。 

约束允许最多 50 个框，坐标最多$10^9$。 小的$n$强烈建议我们能够负担得起$O(n^2)$甚至$O(n^3)$几何模拟。 重要的不是数字大小，而是候选人安置的组合结构。 

一个天真的陷阱是假设我们可以贪婪地放置一个盒子$(0,0)$如果可能的话或仅沿一个轴扫描。 但这会失败，因为最佳放置通常会“卡入”先前放置的矩形形成的角中，并且有效位置同时取决于 x 和 y 约束。 

当多个矩形创建不相交的候选走廊时，会发生微妙的失败情况。 例如，大矩形可能会遮挡左下角，迫使下一个矩形从较高的 y 开始，即使更右侧存在较低的 y 空间。 仅从左到右检查而不考虑垂直约束的贪婪扫描将错误地错过有效的位置。 

## 方法

 蛮力的想法是显式测试可能放置新矩形的每个可能的左下角。 由于矩形是轴对齐且不旋转的，因此有效放置的 x 坐标必须等于 0 或某些现有矩形的右边缘，类似地，其 y 坐标等于 0 或某些现有矩形的上边缘。 这是因为任何有效的“紧密”放置都必须接触边界或其他矩形； 否则我们可以将其向左或向下移动，这与极简性相矛盾。 

因此，简单的模拟将从现有矩形中收集所有候选 x 位置和 y 位置，形成所有对，并测试矩形是否适合且没有重叠。 对于每个传入的矩形，我们扫描所有候选者并选择字典顺序最小的有效矩形。 

这是有效的，因为它直接编码了可行性的定义。 然而，每次插入后，矩形的数量都会增加，候选位置的数量也会增加。 在最坏的情况下，每个$n$矩形贡献$O(n)$x 和 y 中的候选边，导致$O(n^2)$每个查询的位置和$O(n^3)$总重叠检查。 每个重叠检查本身可能会扫描所有现有的矩形，将其推向$O(n^4)$在一个天真的实现中。 

关键的观察是我们实际上不需要枚举所有候选坐标。 问题的结构确保最佳放置始终发生在紧邻某个现有矩形的右侧或紧邻其上方的坐标处，或者位于原点处。 这是一个经典的“矩形天际线”结构：所有可行的布局都位于由先前布局定义的一组有限的边界点上。 

我们可以维护一组不断增长的候选位置，并且对于每个新矩形，按照 x 然后 y 的排序顺序测试它们。 一旦放置了一个矩形，它就会引入新的候选角：$(x + w, y)$和$(x, y + h)$。 我们还丢弃由于重叠或越界限制而不再有效的任何候选者。 

自从$n \le 50$，我们可以维护所有放置的矩形，并且对于每个候选位置，通过检查所有先前的矩形来测试重叠。 这使得实现简单，同时仍然足够快。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力候选枚举+完全重叠检查|$O(n^4)$|$O(n)$| 太慢了|
 | 全面检查的前沿候选人模拟 |$O(n^3)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 维护已放置矩形的列表。 每个矩形存储为$(x, y, w, h)$。 这是用于所有重叠验证的基本事实。 
2. 维护一组可能开始新矩形的候选位置。 最初，这仅包含$(0, 0)$。 这反映了任何放置存在之前唯一有效的起始角球。 
3.对于每个传入的size的矩形$(w, h)$，迭代按 x 然后 y 排序的所有候选位置。 这种顺序强制要求问题选择最左边的可行位置，以及其中最低的位置。 
4. 对于每个候选人职位$(cx, cy)$，首先检查矩形是否位于甲板边界内，即$cx + w \le l$和$cy + h \le h$。 如果它违反了界限，请立即跳过它，因为它无效。 
5. 如果它在边界内，请检查所有先前放置的矩形的重叠情况。 如果两个矩形的投影在 x 和 y 维度上严格相交，则两个矩形重叠。 如果发现任何重叠，则拒绝该候选人。 
6. 选择第一个通过两项检查的候选者作为该矩形的位置。 将其记录为答案的一部分。 
7. 如果没有候选者成功，则对该矩形输出-1。 
8. 如果放置成功，则将新矩形插入放置列表中，并通过添加来更新候选者$(cx + w, cy)$和$(cx, cy + h)$，因为这些代表通过触摸新放置的矩形的右边缘或上边缘而形成的新的潜在左下角。 

### 为什么它有效

 该算法依赖于这样一个不变量：矩形的每个有效位置都可以向左或向下“推”，直到它碰到边界或接触另一个矩形。 这意味着任何最小有效放置都必须与现有结构定义的角对齐。 通过将所有此类角点保留为候选点，我们保证字典顺序中的第一个可行候选点正是所需的位置。 重叠检查确保我们永远不会接受违反先前约束的位置，并且候选扩展确保不会错过未来的有效角点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def overlaps(a, b):
    x1, y1, w1, h1 = a
    x2, y2, w2, h2 = b
    if x1 >= x2 + w2 or x2 >= x1 + w1:
        return False
    if y1 >= y2 + h2 or y2 >= y1 + h1:
        return False
    return True

def can_place(x, y, w, h, L, H, rects):
    if x + w > L or y + h > H:
        return False
    for rx in rects:
        if overlaps((x, y, w, h), rx):
            return False
    return True

def solve():
    n, L, H = map(int, input().split())
    rects = []
    candidates = {(0, 0)}
    
    for _ in range(n):
        w, h = map(int, input().split())
        cand_list = sorted(candidates)
        placed = None
        
        for x, y in cand_list:
            if can_place(x, y, w, h, L, H, rects):
                placed = (x, y)
                break
        
        if placed is None:
            print(-1)
            continue
        
        x, y = placed
        print(x, y)
        rects.append((x, y, w, h))
        
        candidates.add((x + w, y))
        candidates.add((x, y + h))

if __name__ == "__main__":
    solve()
```该代码直接遵循候选集模拟。 重叠检查是使用轴分离逻辑显式编写的，这完全避免了浮点几何问题。 

候选集存储为 Python 集以进行重复数据删除，并根据每个查询进行排序以强制字典选择。 给定$n \le 50$，重复排序可以忽略不计。 

一个关键的实现细节是严格的分离条件`overlaps`。 使用`>=`确保接触的边缘不被视为重叠，这一点很重要，因为矩形允许接触但不能相交。

 ## 边缘情况

 关键的边缘情况是一个矩形与另一个矩形完全接触而不重叠。 例如，放置一个$5 \times 5$矩形位于 (0,0) 处，另一个矩形位于 (5,0) 处。 第二个放置是有效的，因为允许边缘接触。 该算法可以正确处理这个问题，因为重叠检测使用严格的分离`>=`，允许边界接触配置。 

另一种情况是，当一个大矩形阻挡所有低 x 位置时，即使更右侧存在较小的 y 区域，也会强制放置在较高的 y 位置。 候选扩展步骤确保一旦放置了矩形，其顶部和右侧边缘就成为新的有效起点。 这可以防止算法丢失与原点不相邻的有效配置。 

最后，考虑由于宽度和高度限制而没有候选适合的情况。 该算法耗尽所有候选并正确输出 -1，因为每个潜在角点都针对边界和重叠条件进行了显式测试，不会留下任何未检查的隐藏放置可能性。
