 ## 解决方案
 ## 问题理解

 我们有一堆石头和两个交替轮流的玩家，GG 首先移动。 每次移动，玩家都会从堆中取出一颗或两颗石子。 无法采取行动的玩家就输了。 除了简单地试图获胜之外，每个玩家还希望在整个游戏过程中最大化他们个人消除的石子数量。 在双方的最佳发挥下，我们必须为每个测试用例确定两件事：谁获胜，以及获胜者收集了多少石子。 

输入由多个独立游戏组成。 每个游戏仅由初始石子数量定义$n$，可以大到$10^{12}$。 这立即排除了所有位置上的任何模拟或基于状态的动态编程$n$，因为即使$O(n)$每个测试用例都会太慢。 甚至$O(\sqrt{n})$每个查询将紧于$10^4$测试用例。 该结构强烈建议基于模运算的恒定时间或周期解决方案。 

问题的一个微妙方面是次要目标：玩家不仅关心获胜，还关心最大化自己收集的石头。 这意味着，当玩家有多个不会改变最终获胜者的棋步时，他们会更喜欢导致个人总分更大的棋步。 即使输赢结果已经确定，这也会影响棋子的准确分配。 

边缘情况出现在非常小的值$n$，其中博弈时间太短，渐近模式无法稳定。 例如，当$n = 1$，GG只需拿走唯一的石头，立即以1石头获胜。 什么时候$n = 2$，GG又拿走两颗棋子获胜。 然而，在$n = 3$，行为发生变化：GG 无法强行获胜，因此获胜者会发生变化，石子的分配取决于双方玩家在最佳决胜局下的表现。 任何假设统一公式而不检查模 3 的小余数的错误方法都会在这里失败。 

## 方法

 暴力解决方案会将每个游戏状态建模为“当前的棋子数量以及轮到谁”，并递归地尝试移除 1 或 2 个棋子。 每个状态最多分支为两个下一个状态，记忆化减少了重复工作，但状态空间仍然与$n$。 这给出了一个$O(n)$每个测试用例的解决方案，当$n \le 10^{12}$。 

关键的观察结果是，这是一个具有固定移动集 {1, 2} 的减法游戏。 众所周知，此类游戏具有周期性获胜结构。 如果我们只关心输赢，我们会立即认识到经典模式：持仓$n \bmod 3 = 0$第一个玩家输了，所有其他玩家都赢了。 

困难来自第二个目标：最大化收集的石头。 然而，这并没有破坏周期性。 相反，一旦赢家/输家结构固定，等效赢家或输家分支中的最佳选择也会在周期 3 中重复。这使我们能够将所有状态压缩为三个剩余类别，并在每个测试用例的恒定时间内计算确切的结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 DP 超过状态 |$O(n)$每次测试 |$O(n)$| 太慢了|
 | 周期/模 3 分析 |$O(1)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 该解决方案依赖于通过残差来分析游戏$n$模 3 并跟踪最佳游戏如何在这些周期内分配棋子。 

1、首先按岗位分类$n \bmod 3$。 这决定了起始玩家（GG）能否强行获胜。 职位所在$n \bmod 3 = 0$第一个玩家会输，因为每一步都会为对手留下一个获胜的位置。 
2. 如果GG处于获胜位置（$n \bmod 3 \neq 0$），假设GG下了一手棋（1或2子），导致GG仍保持强胜状态。 在所有这些动作中，GG 选择使他最终的总棋子数最大化的那一招。 这在获胜策略中创建了二次优化。 
3. 如果GG处于亏损状态（$n \bmod 3 = 0$），如果双方都发挥最佳，GG 无法阻止 YY 获胜。 在这种情况下，GG仍然会在最小化YY的最终收益和最大化他自己收集的石头之间做出选择，但获胜者的结果已经确定。 
4. 将每个位置减少到其相应的剩余类别，并通过在三状态循环内传播最优选择来计算结果。 因为每一个动作都会减少$n$通过 1 或 2，转变仅取决于这些还原如何在残基之间移动，从而产生稳定的重复模式。 
5. 根据残数输出获胜者，并根据派生模式计算出该获胜者收集的棋子总数。 

### 为什么它有效

 游戏图形成一个有向无环结构，其中每一步都严格递减$n$。 由于步数只有 1 或 2，因此状态转换模式在获胜结构方面每 3 步重复一次。 次要目标（最大化自己的棋子）不会引入新的状态维度，超出通过跟踪哪个玩家当前在每个剩余类别中最佳而捕获的状态维度。 这使得系统在模 3 转换下保持关闭，确保一致性并防止发散为更复杂的状态依赖。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(n: int):
    r = n % 3

    # GG winning positions
    if r != 0:
        # GG wins
        # derive winner stones
        if r == 1:
            # pattern: 1, 2, 4, 5, 7, 8...
            # k = n//3
            if n == 1:
                v = 1
            else:
                v = 2 * (n // 3)
        else:  # r == 2
            v = 2 * (n // 3) + 1
        return 0, v

    # GG loses -> YY wins
    k = n // 3
    v = 2 * k - 1
    return 1, v

def main():
    t = int(input())
    for _ in range(t):
        n = int(input())
        w, v = solve_case(n)
        print(w, v)

if __name__ == "__main__":
    main()
```代码直接实现了残差分类。 第一个分支处理 GG 获胜的所有情况，即当$n \% 3 \neq 0$。 其中，值$v$使用每个残基类别的稳定模式计算。 第二个分支处理 GG 的失败位置，其中 YY 是获胜者，并且该值是从观察到的每三个石头块的线性行为得出的。 

唯一微妙的实现细节是处理$n = 1$，这是最小的获胜位置，尚未拟合稳定$3k$的模式$r = 1$班级。 

## 工作示例

 考虑输入$n = 4$。 残渣为$1$，所以 GG 获胜。 计算将其放入$3k+1$与 一起上课$k = 1$，获胜者得分为 2。GG 首先拿走 1 或 2 个石子，但只有拿走 1 的棋子才能保留获胜结构，因此 GG 被迫进入特定的最佳分支，从而总共获得 2 个石子。 

现在考虑$n = 5$。 残渣为$2$，所以GG又赢了。 这里$k = 1$，公式给出$v = 2k + 1 = 3$。 GG可以选择第一手棋，迫使YY处于失败位置，同时仍最大化自己积累的棋子，总共有3个棋子。 

| n | 残留物 | 获胜者 | GG搬家| 结果|
 | ---| ---| ---| ---| ---|
 | 4 | 1 | GG | 最佳强制获胜路径| GG = 2 |
 | 5 | 2 | GG | 选择导致三态优势的举动| GG = 3 |

 这些例子展示了残差类别如何修复游戏结构，而次要目标如何在同等的获胜延续中进行选择。

 ## 边缘情况

 对于$n = 1$，GG立即获胜并拿走唯一的石头。 这种情况绕过所有模块化模式，必须明确处理以避免错误的从零开始的公式。 

为了$n = 3$，GG处于劣势。 尽管GG可能会尝试不同的第一步，但这两种选择都让YY最终获胜。 然而，GG仍然在失败的分支中选择最大化自己收集的棋子的举动，这唯一地确定了最终的分配。 

对于非常大的$n$， 例如$10^{12}$，该解完全依赖于模运算。 该算法从不构造中间游戏状态，因此不存在溢出或性能下降的风险，并且每个测试用例的计算保持恒定时间。
