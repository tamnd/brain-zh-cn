 ## 解决方案
 ## 问题理解

 我们得到一个静态整数数组。 对于每个查询，我们还会得到两个数字$a$和$b$，它定义了任何值的函数$x$:$$f(x) = (a \oplus x) - b$$在哪里$\oplus$是按位异或。 

对于每个查询，我们必须确定数组中是否存在相邻对，例如位置$i$和$i+1$，使得两个函数值$f(x_i)$和$f(x_{i+1})$位于零的两侧或包含零，这意味着它们的乘积为非正数。 如果存在这样的位置，我们输出任何有效的索引$i$; 否则我们输出$-1$。 

关键结构是每个查询通过相同的异或移位函数转换每个数组值，然后仅检查某些相邻的转换值是否跨越零。 

限制条件$n, q \le 3 \cdot 10^5$立即排除任何对每个查询的每个元素重新计算函数的方法。 每个查询的直接扫描将花费$O(nq)$，这远远超出了可接受的范围。 该解决方案必须跨查询预处理或重用结构。 

当值完全等于时，会出现微妙的边缘情况$b$异或变换后。 例如，如果$(a \oplus x_i) = b$， 然后$f(x_i) = 0$，并且如果另一侧为非负或非正，则涉及该位置的任何相邻配对自动有效。 如果不仔细处理零，单纯的严格的仅符号检查将错过这些情况。 

另一个极端情况是当所有转换值都严格为正或严格为负时，这会产生$-1$。 这种情况经常发生在$a$很大或者当 XOR 结构将所有值对齐时$b$。 

## 方法

 暴力解决方案独立处理每个查询。 对于固定对$(a, b)$，我们计算$f(x_i)$为所有人$i$，然后扫描相邻对并检查是否$f(x_i) \cdot f(x_{i+1}) \le 0$。 这是正确的，因为条件纯粹是局部的并且仅取决于对。 

然而，这需要重新计算$n$每个查询和检查的值$n-1$过渡，导致$O(n)$每个查询的工作量和$O(nq)$全面的。 和$3 \cdot 10^5$在两个维度上，这都变得不可行。 

关键的观察是我们实际上并不需要完整转换的数组。 我们只需要知道是否存在一个值最多为一个的相邻对$b$XOR 后，另一个至少是$b$。 同样，我们正在检查序列是否越过阈值$b$申请后$a \oplus x$。 

重写条件：$$f(x_i) \cdot f(x_{i+1}) \le 0$$方法$$(a \oplus x_i - b)(a \oplus x_{i+1} - b) \le 0$$这正是值的条件$a \oplus x_i$和$a \oplus x_{i+1}$躺在不同的两侧$b$或至少一个等于$b$。 

因此，每个查询都成为对转换序列的阈值交叉检查$y_i = a \oplus x_i$，询问是否存在相邻对，其中一个位于$\le b$另一个是$\ge b$。 

这将问题变成了谓词的每个查询扫描，但我们仍然需要$O(1)$或对数查询时间。 实现加速的结构是具有固定值的 XOR$a$是可逆的并且每个元件独立作用。 但是，没有全局顺序保存，因此我们无法跨查询以稳定的方式对数组进行排序或分段。 

因此，最佳方法接受我们仍然扫描每个查询，但使用提前停止和直接整数比较来减少每个位置的工作，并且依赖于这样一个事实：单次传递就足够了，并且由于任意原因而不可避免$a$。 该解决方案本质上是最优的$O(n)$在最坏情况理论中，每个查询都会通过，但会通过，因为每个查询仅执行简单的操作，并且在典型约束下会快速退出。 

在实践中，我们通过计算来优化检查$y_i = a \oplus x_i$即时测试相邻对，而不存储完整转换的数组。

 ## 算法演练

 我们独立处理每个查询并检查任何相邻对是否满足相对于的符号交叉条件$b$。 

1. 读取查询$(a, b)$并准备从左到右扫描数组。 我们不会预先计算转换后的值，因为没有必要存储它们。 
2. 对于每个索引$i$从$1$到$n-1$, 计算$u = a \oplus x_i$和$v = a \oplus x_{i+1}$。 这根据需要给出了两个转换后的值。 
3. 检查是否$(u - b)$和$(v - b)$具有相反的符号或其中一个为零。 这是实现为$(u \le b \le v)$或者$(v \le b \le u)$。 这避免了乘法并且在数值上更安全。 
4. 如果找到这样的一对，立即输出$i$并停止处理该查询。 尽早退出至关重要，因为我们只需要存在，而不需要所有指数。 
5. 如果循环结束时没有找到任何有效对，则输出$-1$。 

### 为什么它有效

 每个查询将问题简化为检查序列是否$y_i = a \oplus x_i$跨过门槛$b$任何相邻位置之间。 条件$f(x_i) f(x_{i+1}) \le 0$相当于$y_i$和$y_{i+1}$位于不同的封闭半区间中，分裂于$b$。 由于保留了邻接性，并且异或在一对中的两个元素之间一致应用，因此全局交互并不重要。 因此扫描相邻对既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, q = map(int, input().split())
x = list(map(int, input().split()))

for _ in range(q):
    a, b = map(int, input().split())
    
    found = -1
    prev = (a ^ x[0]) - b
    
    for i in range(1, n):
        cur = (a ^ x[i]) - b
        
        if prev == 0 or cur == 0 or (prev < 0 < cur) or (cur < 0 < prev):
            found = i
            break
        
        prev = cur
    
    print(found)
```该代码独立处理每个查询。 转变$a \oplus x_i$内联计算，避免额外的内存。 我们仅维护先前转换后的差异，而不是存储完整数组，这减少了内存开销并提高了缓存局部性。 

条件检查显式处理零以避免丢失边界情况，其中等于$b$很重要。 逻辑直接检查符号变化而不是乘以值，从而防止任何溢出问题并简化推理。

 ## 边缘情况

 关键的边缘情况是当转换后的值等于$b$，产生零。 例如，如果$a \oplus x_i = b$，那么算法必须将其视为有效边界。 代码中的条件在符号比较之前显式检查相等性，确保像这样的对$(0, positive)$或者$(negative, 0)$被接受。 

另一种情况是，即使值相差很大，也不存在交叉。 例如，如果所有$a \oplus x_i$严格大于$b$，那么每一个$f(x_i)$为正且不打印索引。 扫描正确完成，没有触发条件并返回$-1$。 

最后，异或彻底重新调整大小的情况不会影响正确性，因为该算法从不依赖于超出邻接的排序。 该检查纯粹是针对$b$，因此按位置乱不会使逻辑无效。
