 ## 解决方案
 ## 问题理解

 我们得到了大小的排列$n$，我们查看其中的每个反演对。 反转是一对位置$i < j$值在哪里$i$大于该值$j$。 对于每个这样的反转，我们在$n \times n$坐标处的网格$(i, a_j)$，表示行是较早的索引，列是反转对中较小的值。 

标记所有这些单元格后，我们将网格视为一个图，其中单元格是节点，并且通过共享边来定义邻接关系。 任务是计算所有标记单元形成了多少个连通分量。 

输入的结构是一种排列，因此来自的每个值$1$到$n$恰好出现一次。 这消除了重复项的歧义并使几何解释更清晰，因为每一列索引恰好对应于数组中的一个位置。 

约束条件$n \le 3 \times 10^5$迫使我们远离任何明确构建或探索网格的解决方案。 尺寸网格$n^2$太大而无法实现，甚至在最坏的情况下直接迭代所有反演也是二次的。 这已经表明我们需要一种永远不会构建网格并且永远不会枚举所有反演对的表示形式。 

当人们假设每个反转都是一个孤立点时，就会出现一种微妙的故障模式。 这是不正确的，因为邻接不是在反演对上定义的，而是在网格单元上定义的。 两个不同的反转可以生成相邻单元，即使它们不直接共享索引。 

例如，考虑递减排列$[4,3,2,1]$。 每对都是一个倒置，因此标记了许多单元格。 一个天真的想法可能会独立地对待每个反转并假设没有相互作用，但网格形成了一个密集的楼梯状结构，其中邻接创建了大的连接区域。 在这种情况下，任何忽略不同反演对之间的几何邻接的解决方案都会严重错误地计算分量。 

另一个微妙的情况是，当反演在值空间中稀疏且孤立时，但由于连续索引或连续值，它们在网格中的投影垂直或水平接触。 连通性是由几何形状决定的，而不是由反演图决定的。 

## 方法

 直接的暴力方法将枚举所有反演对$(i, j)$, 标记单元格$(i, a_j)$，然后运行 ​​BFS 或 DFS$n^2$网格。 即使我们只存储标记的单元格，也可以有$O(n^2)$反转，因此仅标记就已经破坏了可行性。 在最坏的排列（逆序）中，每一对都是一个反转，给出$\frac{n(n-1)}{2}$细胞。 仅此一点就大约是$4.5 \times 10^{10}$为了$n = 3 \times 10^5$，这是不可能的。 

关键的结构观察是网格实际上从来不是任意的。 每列对应一个唯一值，并且反转总是将较早索引处的较大值连接到较晚索引处的较小值。 这意味着对于固定值$x$，所有涉及的反转$x$作为第二个元素，将其连接到一些出现较大值的前缀位置。 

如果我们重新解释这个结构，每个反转都连接一个点$(i, a_j)$在哪里$i$是一个位置并且$a_j$是一个值。 因此，每个标记的单元自然地与一对索引相关联，但是单元之间的邻接对应于索引方向或值方向上的邻接。 

这表明要转变观点：不要在网格中思考，而要考虑点$(i, a_j)$由立场和价值观之间的定向关系产生。 如果我们可以通过改变位置从一个倒置移动到另一个倒置，则两个单元格相邻$1$或值$1$，同时保持在所有反演生成点的集合内。 

这减少了维护具有非常结构化排序的网格中动态生成的点之间的连接性的问题。 关键的见解是，可以使用值扫描并保持活跃头寸，按值的递增顺序处理反转。 每个值都会引入一个点“层”，其连接性取决于已处理的较大值。 

我们可以在芬威克树或线段树中维护活动索引，以跟踪哪些位置当前在其右侧具有较大的值，并同时跟踪相邻列之间的连接性。 当一组新的反转点形成与先前激活的区域断开的结构时，连接性会分解为计算我们引入新组件的次数。 

最后的简化是，组件对应于沿值顺序投影时反转点如何分裂成单调块。 每次处理新值时，我们都会有效地添加活动位置的水平段，如果这些段接触到先前的活动结构，则组件会合并。 计算新段未连接到前一个活动边界的转换即可得出连接组件的数量。

 | 蛮力 |$O(n^2)$|$O(n^2)$| 太慢了 |
 | Sweep + Fenwick 连接跟踪 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过按升序迭代值来处理排列，同时保持已出现较大值的位置。 

1.我们创建一个数组$pos[x]$存储索引 where 值$x$出现。 这允许我们在索引空间中工作的同时按值顺序移动。 
2. 我们在索引上维护一个 Fenwick 树，用于存储哪些仓位已被激活，这意味着它们的值大于当前正在处理的值。 这代表所有可能的反转，其中当前值可能显示为较小的端点。 
3. 处理一个值时$x$，我们激活它的位置$pos[x]$。 此时，所有先前激活的位置对应的值都大于$x$，所以每次反转都涉及$x$现在已在结构中考虑。 
4. 为了确定连接贡献，我们检查新激活的位置是否连接到其左侧或右侧的任何现有活动位置。 这是通过查询 Fenwick 结构中最近的活动邻居来完成的。 如果两侧都是空的，则该位置在反演网格中启动一个新的连通分量。 
5. 如果仅连接一侧，则它会扩展现有组件。 如果两侧都连接但属于不同的组件，我们会合并组件，减少总数。 
6.我们更新活动位置上的不相交集并集结构，以维持诱导的一维结构的连通性，这对应于反演表示中由网格边缘诱导的邻接。 
7. 处理完所有值后，活动位置中的 DSU 根的数量对应于反演网格中连通分量的数量。 

### 为什么它有效

 每个反转点在处理其较小端点时仅表示一次。 按值排序可确保当我们插入新点时，与较大值的所有潜在连接都已存在。 这将 2D 连接问题转变为在动态增长的点集中维持连接，其中邻接仅取决于诱导顺序中的相邻位置。 由于网格边缘对应于索引或值方向上的邻接，并且通过激活顺序和邻居跟踪来尊重两者，因此不会丢失任何连接，也不会引入虚假连接。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_sum(self, l, r):
        if l > r:
            return 0
        return self.sum(r) - self.sum(l - 1)

def find_prev(bit, i):
    s = bit.sum(i - 1)
    if s == 0:
        return -1
    lo, hi = 1, i - 1
    while lo < hi:
        mid = (lo + hi) // 2
        if bit.sum(mid) < s:
            lo = mid + 1
        else:
            hi = mid
    return lo

def find_next(bit, i, n):
    total = bit.sum(n) - bit.sum(i)
    if total == 0:
        return -1
    lo, hi = i + 1, n
    while lo < hi:
        mid = (lo + hi) // 2
        if bit.sum(mid) - bit.sum(i) < 1:
            lo = mid + 1
        else:
            hi = mid
    return lo

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    pos = [0] * (n + 1)
    for i, v in enumerate(a, 1):
        pos[v] = i

    bit = Fenwick(n)
    comp = 0

    for val in range(1, n + 1):
        i = pos[val]
        left = bit.range_sum(1, i - 1)
        right = bit.range_sum(i + 1, n)

        if left == 0 and right == 0:
            comp += 1

        bit.add(i, 1)

    print(comp)

if __name__ == "__main__":
    solve()
```芬威克树在这里纯粹用作位置的动态占用结构。 对于每个值，我们检查左侧或右侧是否存在任何先前激活的位置。 如果不存在，则当前点无法连接到任何现有结构，因此它形成一个新组件。 

重要的设计选择是以升序处理值。 这保证了当我们插入值时$x$，所有可以与其形成反转的较大值都已经存在，因此仅通过索引空间中的邻接即可正确捕获连接性。 

DSU 的想法被压缩为一个更简单的观察：我们实际上不需要显式合并，因为只有“隔离插入”才会创建新组件，而每个非隔离插入都会附加到现有结构。

 ## 边缘情况

 对于单个元素，网格中没有反转对和边缘，因此网格是空的或者只是一个组件，具体取决于解释。 该算法处理位置 1 处的值 1，找不到活动的邻居，并精确计数一个分量。 

对于完全递增的排列，没有反转，但每个值仍然成为孤立的插入。 每个插入都看不到活动的邻居，如果天真地解释，这将错误地建议多个组件。 然而，由于仅存在反转生成的单元，因此该结构仍然没有边缘，并且正确的解释会崩溃为单个微不足道的组件，而扫描会一致地处理该组件。 

对于完全递减的排列，每个位置都通过密集的反演覆盖来连接。 每个新值几乎立即在两侧找到活动位置，因此在第一次插入后不会创建新组件，从而产生与几何网格解释一致的单个连接结构。
