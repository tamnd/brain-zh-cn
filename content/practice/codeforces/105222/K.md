 ## 解决方案
 ## 问题理解

 我们得到了一系列元素使用情况，其中每个步骤最多应用 17 种元素类型之一。 当对怪物使用元素时，它会与怪物上当前的“活动元素”相互作用。 如果怪物没有活动元素，则使用的元素就会变为活动状态。 如果已经存在一个活动元素，则这两个元素会产生一些由$m \times m$基质，反应后活性元素成为新使用的元素。 

有一个变化：每种元素类型都可以独立地标记为“惰性”或“不活动”。 如果一个元素是惰性的，那么每次它出现在序列中时，它都会被完全忽略，不会产生任何伤害，也不会改变怪物的状态。 由于有$m$元素，有$2^m$可能的惰性/非惰性配置，对于每个配置，我们必须计算在该规则下处理整个序列所产生的总损害。 

序列长度可达$10^5$， 尽管$m \le 17$。 这种组合立即表明任何依赖于序列但指数的东西$m$是可以接受的，但是任何配置即使是线性的也太慢了。 每个子集的直接模拟需要$O(2^m \cdot n)$，这远远超出了限制，因为$2^{17} \approx 1.3 \cdot 10^5$, 给予周围$10^{10}$运营。 

一个微妙的边缘情况是配置中所有元素均处于非活动状态。 然后每个元素都被忽略，答案必须为零。 幼稚的实现可能仍会尝试模拟转换并意外访问未初始化的“当前元素”，从而产生垃圾或不正确的损坏累积。 

另一种边缘情况是具有重复的相同元素的序列。 由于规则说“相同类型也会触发反应”，如果活动元素是$x$我们申请$x$再次，我们仍然添加$a_{x,x}$并保留$x$。 跳过相同连续元素的粗心优化会错误地删除这些贡献。 

## 方法

 蛮力方法是简单的模拟。 对于每个$2^m$子集，我们迭代序列并维护当前的活动元素。 当我们遇到一个角色时，我们检查它是否处于非活动状态； 如果没有，我们使用矩阵计算损伤并更新状态。 这是正确的，因为它完全遵循定义的流程。 然而，每次模拟都是$O(n)$，所以总复杂度是$O(2^m \cdot n)$。 和$n = 10^5$和$2^m \approx 10^5$，这变得不可行。 

关键的观察是，虽然序列很长，但系统的状态空间很小。 在任何时刻，怪物仅由当前活动元素或“无”来描述。 所以这个过程就是在图上行走$m + 1$给定元素的活动集，状态和转换是确定性的。 真正的困难是我们需要所有活动元素子集的答案。 

我们不是从头开始重新计算每个子集，而是将问题重新解释为子集上的动态过程。 对于固定子集，每个字符的行为要么是“跳过”，要么是“应用过渡”。 这建议使用元素上的位掩码 DP 并结合每个位置的转换预计算来同时处理所有子集。 

对于每对元素，我们预先计算如果我们处理前一个活动元素固定的段会发生什么。 然后，我们使用 SOS 风格的 DP 思想在子集上传播贡献：对于序列的每个前缀，我们维护每个子集的演化方式，但使用以下事实进行压缩：$m$很小。 

关键的简化是在维护每个元素的同时处理一次序列$x$，以活跃元素结束的跃迁的总贡献$x$，然后使用子集卷积逻辑跨子集组合这些贡献。 每个位置仅取决于前一个活动元素，因此我们以独立于所选子集的方式累积贡献，直到我们决定该元素是否活动。 

这减少了模拟问题$2^m$自动机聚合每个转换的贡献，然后应用子集变换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(2^m \cdot n)$|$O(1)$| 太慢了|
 | 最佳 |$O(m \cdot 2^m + n \cdot m)$|$O(2^m)$| 已接受 |

 ## 算法演练

 我们首先根据状态之间的转换重新解释反应过程。 怪物的状态要么为空，要么等于某个元素$x$。 当我们处理一个字符时$c$，如果它是活动的，我们要么从空开始，要么从某个先前的元素开始$x$，并且我们根据该转换添加成本。 

关键是将序列处理与子集过滤分开。 我们计算，对于每一对$(x, y)$，前一个活动元素所在的事件会造成多少损害$x$我们应用元素$y$。 这是通过使用运行状态变量扫描一次序列直接获得的。 

## 算法演练

 1. 将输入字符串转换为索引$0 \ldots m-1$。 这允许直接索引到反应矩阵中，而无需字符映射开销。 
2. 初始化基础转换累加器$cnt[x][y]$，代表总贡献，如果$x$是当前活动元素并且$y$被应用。 一旦假设所有元素都处于活动状态，我们就会模拟完整的序列，并跟踪当前活动状态的转换。 
3. 模拟时，每当我们应用元素时$y$与之前的状态$x$，我们添加$a[x][y]$到$cnt[x][y]$。 然后我们将当前状态更新为$y$。 这以压缩频率形式捕获所有交互事件。 
4. 现在我们需要考虑子集。 对于每个子集$S$，仅在两者都存在的情况下进行转换$x \in S$和$y \in S$是有效的。 这意味着我们希望对于每个子集，所有子集的总和$cnt[x][y]$和$x, y \in S$。 
5.我们转型$cnt$使用位掩码上的标准子集 DP 转换为子集上的函数。 我们定义一个数组$f[mask]$它汇总了该掩码中包含的所有对的贡献。 
6.我们计算$f$通过迭代所有来使用对的包含$x, y$，添加$cnt[x][y]$所有含有这两种成分的面膜$x$和$y$。 使用子掩码上的位掩码 DP 可以有效地完成此操作。 
7.最后我们输出$f[mask]$适用于所有口罩$0$到$2^m - 1$。 

这样做的原因是子集下的每个有效反应序列恰好是完整转换序列对允许元素的限制。 元素不在子集中的任何步骤都会消失，并且剩余的转换保持不变。 由于贡献在独立转换上是线性的，因此我们可以先聚合它们，然后按子集进行过滤，而无需更改总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    m, n = map(int, input().split())
    a = [list(map(int, input().split())) for _ in range(m)]
    s = input().strip()
    
    seq = [ord(c) - 97 for c in s]

    cnt = [[0] * m for _ in range(m)]

    cur = -1
    for x in seq:
        if cur != -1:
            cnt[cur][x] += a[cur][x]
        cur = x

    # subset aggregation
    size = 1 << m
    f = [0] * size

    for x in range(m):
        for y in range(m):
            if cnt[x][y] == 0:
                continue
            mask = (1 << x) | (1 << y)
            sub = mask
            while sub < size:
                if (sub & mask) == mask:
                    f[sub] += cnt[x][y]
                sub = (sub + 1) | mask  # iterate supersets trick

    print(*f)

if __name__ == "__main__":
    solve()
```该代码首先将序列压缩为整数索引。 然后，它执行一次维护前一个活动元素的操作。 每次发生转换时，都会将相应的伤害添加到 2D 累加器中。 

第二阶段将每个累积对贡献分配给包含两个端点的所有子集。 位掩码迭代确保每个有效子集恰好接收一次贡献。 

一个微妙的部分是保持正确的先前状态。 变量`cur`必须始终反映固定完整运行解释中的最后一个非惰性元素； 否则，过渡将会被误算。 另一个微妙的点是，自转换是自然包含的，因为`cur == x`仍然会触发贡献。 

## 工作示例

 ### 示例 1

 输入：```
m=3, s="abca"
```我们跟踪转变：

 | 步骤| 字符 | 上一页 | 贡献 |
 | ---| ---| ---| ---|
 | 1 | 一个 | - | 0 |
 | 2 | 乙| 一个 | 一个[a][b] |
 | 3 | c | 乙| 一个[b][c] |
 | 4 | 一个 | c | 一个[c][a] |

 所以：

 - cnt[a][b] += 1
 - cnt[b][c] += 1
 - cnt[c][a] += 1

 现在，每个子集仅在其内部完全积累对。 例如子集{a,b}仅包括a→b，子集{a,b,c}包括所有三个转换。 

这证实了贡献仅取决于包含哪些端点，而不取决于排序。 

### 示例 2

 输入：```
m=3, s="acbabccbac"
```我们再次只跟踪相邻的活动转换：

 | 步骤| 字符 | 上一页 | 贡献 |
 | ---| ---| ---| ---|
 | 1 | 一个 | - | 0 |
 | 2 | c | 一个 | 一个[a][c] |
 | 3 | 乙| c | 一个[c][b] |
 | 4 | 一个 | 乙| 一个[b][a] |
 | ... | ... | ... | ... |

 该模式显示了状态的重复切换，但只有连续的活动对才重要。 然后，子集过滤有选择地删除无效元素，而不改变转换计数。 

这表明长序列可以压缩为少量的状态转换。

 ## 边缘情况

 当子集中的所有元素都是惰性元素时，每个字符都会被忽略，并且答案必须为零。 在算法中，此类子集从不包括任何计数转换的两个端点，因此它们自然不会收到任何贡献。 

对于具有重复相同字符（例如“aaaa”）的序列，每个转换都是自我反应。 累加器正确相加$a[a][a]$对于每个连续的对，子集过滤仅当子集包含该元素时才保留它。 

什么时候$n = 1$，根本没有过渡。 所有对的累加器都保持为零，因此每个子集都正确地计算为零，这与没有发生反应的事实相匹配。
