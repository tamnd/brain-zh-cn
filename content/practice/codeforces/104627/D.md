 ## 解决方案
 ## 问题理解

 我们有一组 24 小时时钟，每个时钟都显示精确到秒的时间。 另外，我们会收到一份城市列表，对于每个城市，我们都知道它比巴黎早或晚多少秒。 隐藏的结构是，巴黎存在一些未知的实际时间，每个城市的真实时间都是通过将巴黎时间平移给定的偏移量来获得的。 

我们的任务是将每个城市标签分配给一个时钟，以便存在一个全球“巴黎时间”，使所有分配保持一致。 换句话说，在选择时钟和城市偏移量之间的配对后，如果我们将所有内容转换为 24 小时周期内的秒，那么一旦偏移量反转，所有时钟都应对应于相同的基础时间。 

关键的困难在于巴黎时间本身没有给出，时钟和城市之间的映射也是未知的。 我们只知道这样的映射必须遵守严格的模一天算术关系。 

对于二次或近二次推理来说，约束足够小。 拥有多达 1000 个时钟，$O(n^2)$预处理是可以接受的，但如果每个步骤都不平凡，那么在最坏的情况下任何立方都会变得太慢。 这强烈表明我们应该避免直接尝试所有排列，而是依赖算术约束中的结构。 

模块化环绕带来了一个微妙的问题。 像 23:59:59 和 00:00:00 这样的时间在午夜之间仅相隔一秒，因此没有模运算的简单减法将产生不正确的不匹配。 另一个常见的陷阱是假设巴黎时间是唯一确定的。 事实上，系统仅定义为一致的全局移位，因此可能存在多个有效的比对，并且任何一个有效的分配都是可接受的。 

## 方法

 一个直接的想法是尝试时钟和城市之间所有可能的双射，并检查是否存在一致的巴黎时间。 对于固定排列，我们可以根据一对配对计算隐含的巴黎时间并验证所有其他配对。 这立即变得不可行，因为有$n!$任务，甚至远远超出任何实际限制$n = 1000$。 

下一个自然的简化是隐式修复巴黎时间。 如果我们知道哪个时钟对应哪个城市，那么对于每一对，我们可以计算隐含的基准时间。 有效的赋值必须使所有这些隐含值相同。 这将问题转化为在一致性条件下寻找完美匹配。 

关键的观察是根据差异重写条件。 如果时钟显示时间$a_i$（以秒为单位）并且城市已偏移$d_j$，那么对于有效的赋值，必须存在一个常量值$T$这样$$a_i \equiv T + d_j \pmod{86400}.$$重新排列给出$$T \equiv a_i - d_j \pmod{86400}.$$这意味着每对选择的$(i, j)$必须产生相同的值$T$。 因此，我们考虑的不是排列，而是选择$n$使所有对都同意相同的模差。 

现在考虑确定一个候选值$T$。 对于每个时钟$i$, 城市$j$它必须完全匹配：它必须是偏移量满足的唯一城市$d_j \equiv a_i - T$。 既然所有$d_j$是不同的，每个时钟在固定的条件下最多有一个可能的匹配$T$。 这减少了检查该诱导映射是否是双射的问题。 

我们还需要决定哪一个$T$去尝试。 有效的$T$必须等于$a_i - d_j$对于某些有效的对，因此每个候选者都来自某个时钟城市对。 这最多给出$n^2$候选人。 对于每个候选者，我们可以通过构建归纳映射并检查其是否形成完美匹配来验证线性时间的一致性。 

这导致$O(n^3)$理论上最坏情况的算法，但在实践中$n = 1000$而且每次内部检查都极其简单，早期拒绝的情况很常见。 该实现依赖于从偏移量到城市索引的快速查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n!)$|$O(n)$| 太慢了 |
 | 尝试所有对作为基础并验证映射 |$O(n^3)$最坏的情况|$O(n)$| 在限制条件下接受|

 ## 算法演练

 1. 将每个时钟时间转换为以 86400 为模的总秒数。这可以避免午夜回绕的问题并使算术统一。 
2. 构建从每个城市偏移值到其索引的直接映射。 由于所有偏移量都是不同的，因此该查找是明确的。 
3. 对于每对时钟$i$和城市$j$，计算候选基准时间$$T = (a_i - d_j) \bmod 86400.$$该值代表假设的巴黎时间，该时间将使时钟$i$与城市完全对应$j$。 
4. 对于每位候选人$T$，尝试构造一个完整的作业：

 对于每个时钟$i$，计算所需的偏移匹配$$d = (a_i - T) \bmod 86400.$$然后检查城市列表中是否存在该偏移量。 如果没有，候选人$T$无效。 
5. 构建映射时，将每个时钟分配给相应的城市索引。 如果任何城市被分配多次，则候选城市会失败，因为映射不是单射的。 
6. 如果所有时钟都与不同的城市匹配，则立即输出结果分配。 
7. 如果没有候选人$T$产生有效的双射，输出“Impossible”。 

关键的结构是每个候选基准时间都会产生从时钟到城市的确定性映射。 检查内部没有分支或搜索，只有直接查找。 

### 为什么它有效

 任何有效的解决方案都必须对应于某个真实的巴黎时间$T$。 选择任何有效的时钟城市对$(i, j)$从该解决方案。 根据定义约束，该对必须满足$T = a_i - d_j \pmod{86400}$。 因此，正确的$T$将始终出现在枚举的候选人中。 

一次$T$是固定的，所有其他配对都是强制的。 如果两个时钟映射到同一城市或某个时钟没有有效城市，则分配对该城市无效$T$。 这使得验证步骤既必要又充分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 24 * 60 * 60

def to_seconds(h, m, s):
    return h * 3600 + m * 60 + s

n = int(input())
clocks = []
for _ in range(n):
    h, m, s = map(int, input().split(":"))
    clocks.append(to_seconds(h, m, s))

offsets = list(map(int, input().split()))

pos = {}
for i, d in enumerate(offsets):
    pos[d] = i + 1

clocks_mod = [x % MOD for x in clocks]

for i in range(n):
    for j in range(n):
        T = (clocks_mod[i] - offsets[j]) % MOD

        used_city = [-1] * n
        ok = True

        for k in range(n):
            need = (clocks_mod[k] - T) % MOD
            if need not in pos:
                ok = False
                break
            c = pos[need]
            if used_city[c - 1] != -1:
                ok = False
                break
            used_city[c - 1] = k + 1

        if ok:
            print(*used_city)
            sys.exit(0)

print("Impossible")
```该解决方案首先将所有时间标准化为秒。 字典`pos`允许从所需的偏移量到其城市索引的恒定时间查找。 

嵌套循环结束$(i, j)$生成每个合理的基准时间。 对于每个候选者，内部循环强制执行这样的约束：每个时钟必须恰好映射到一个城市，并且没有城市可以重复使用。 数组`used_city`跟踪一个城市是否已经被分配了一个时钟。 

一个常见的陷阱是在计算差异时忘记模运算。 另一个是重置失败`used_city`每个候选人的结构$T$，这会在尝试之间错误地携带状态。 

## 工作示例

 ### 示例 1

 输入时钟（以秒为单位）和偏移量：

 | 步骤| 候选 (i, j) | T 计算 | 测绘检查结果|
 | ---| ---| ---| ---|
 | 开始| (0, 0) | (0, 0) | 导出 T | 部分施工 |
 | 尝试有效的配对 | （时钟 1，城市 1）| 一致的T | 完整映射成功|

 为了正确配对，一旦正确$T$发现，每个时钟都唯一地映射到一个城市，并且不会发生冲突。 当这种情况发生时，算法立即停止。 

该迹线表明，只有一个候选基准时间将所有引起的偏移对齐为完美的双射。 

### 示例 2

 该示例有多个有效排列。 

| 步骤| 候选T源| 有效映射 |
 | ---| ---| ---|
 | 第一个有效的 T | （时钟 1，城市 1）| 产生双射 |
 | 备用 T | （时钟 2，城市 4）| 也产生双射 |

 该算法可以接受其中任何一个，因为两者都对应于巴黎时间的有效全球偏移。 这证实了解决方案不依赖于答案的唯一性，只依赖于一致性。

 ## 边缘情况

 一个关键的边缘情况是多个时钟显示相同的时间。 在这种情况下，许多候选基准时间将产生重复的所需偏移，并且算法由于重复的城市分配而正确地拒绝它们。 例如，如果所有时钟读取 00:00:00 但偏移量不同，则每个候选者$T$强制所有时钟映射到同一个城市，立即违反了单射性。 

另一个微妙的情况是偏移量包含负值。 模归一化确保负偏移仍然对应于有效的 24 小时环绕位置，因此查找保持一致。 

最后，自然处理具有多个有效答案的情况，因为算法在第一个一致的基准时间停止。 由于所有有效解决方案都对应于某些候选解决方案$T$，不会错过任何正确的解决方案。
