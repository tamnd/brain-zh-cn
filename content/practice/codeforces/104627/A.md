 ## 解决方案
 ## 问题理解

 我们得到一个代表一张纸的字符网格，其中每个单元格要么为空，要么被标记。 任务是确定标记单元的图案是否可以由单个矩形印章产生，该印章被按压到最初空的网格上一次或多次。 每个图章应用程序都会在某个轴对齐的矩形内绘制每个单元格，并且多个应用程序可能会重叠。 

关键的约束是最终网格中的所有标记单元必须可以解释为单个固定图章形状的矩形的并集。 然而，印章形状本身并没有给出，因此问题简化为检查标记单元组是否可以通过重复应用某个矩形来“一致地覆盖”，而无需删除或部分雕刻绘制的单元。 

网格尺寸足够小，O(nm) 或 O((nm)^2) 解决方案就可以了。 更重要的是几何结构而不是原始优化。 这立即排除了任何显式尝试所有可能矩形的方法，因为在最坏的情况下有 O(n^2 m^2) 个候选矩形，并且每个候选矩形都需要扫描网格。 

当标记的单元形成几乎矩形的形状但具有内部孔或不相交区域时，会出现微妙的失败情况，而这些内部孔或不相交的区域无法通过重复冲压单个矩形来获得。 

例如，考虑：```
###
#..
###
```如果我们假设一个有效的冲压过程，如果顶部和底部行在任何一致的矩形冲压下完全填充，则中间行不能部分为空。 仅检查本地连接或计算填充单元总数的简单方法将错误地接受此类模式。 

另一个重要的边缘情况是，当标记的单元形成多个断开的组件时，每个组件看起来都是矩​​形的，但没有以可以通过重复冲压同一矩形来解释的方式对齐。 

核心挑战是确定是否存在一致的矩形，使得所有标记的单元都可以通过放置在任意位置的矩形的并集生成。 

## 方法

 一个蛮力的想法是尝试网格中每个可能的矩形作为候选邮票形状。 对于每个候选矩形，我们模拟将其标记在所有可能的位置上，并检查是否可以准确地重建目标网格。 

这是可行的，因为问题简化为找到一个矩形，其翻译后的副本可以覆盖所有标记的单元格，而不会产生任何不匹配。 然而，矩形的数量是 O(n^2 m^2)，对于每个矩形，我们可能需要 O(nm) 次模拟，导致最坏情况的复杂度为 O(n^3 m^3)，即使对于中等网格，这也远远超出了限制。 

关键的观察是，如果存在有效的冲压过程，则所有标记单元的最小边界矩形本身必须是冲压形状的有效候选者。 这是因为任何冲压操作都无法在边界矩形之外创建标记单元，并且重复冲压无法以排除最终形状的凸矩形外壳的部分的方式向内“弯曲”边界。 

因此，我们不是搜索任意矩形，而是关注包围所有标记单元格的最小矩形。 然后我们验证是否标记了该边界矩形内的每个单元格。 如果内部的任何单元格是空的，则无法通过冲压完整的矩形来生成图案，因为覆盖对角的任何冲压都必然会填充该单元格。 

这将问题简化为简单的结构检查：计算所有标记单元的边界框并验证其是否完全填充。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有矩形进行暴力破解 | O(n^3 m^3) | O(n^3 m^3) | O(1) | O(1) | 太慢了 |
 | 边界矩形检查 | O(纳米) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们继续直接分析网格结构。 

1. 扫描整个网格以找到所有标记的单元格，同时跟踪包含标记的最小和最大行索引和列索引。 这给出了包含所有标记单元格的最小轴对齐矩形。 
2.如果不存在标记的单元格，我们立即返回网格有效，因为不需要标记并且空网格很容易实现。 
3. 迭代计算出的边界矩形内的每个单元格。 
4. 如果我们发现该矩形内有任何未标记的单元格，则我们得出结论，无法通过冲压完整矩形来形成该形状，因为任何有效的冲压序列都会连续填充整个边界框。 
5. 如果边界矩形内的所有单元格都被标记，我们返回配置有效。 

这样做的原因是，冲压矩形总是会产生一个区域，该区域是完整矩形平移的并集。 因此，任何有效的最终形状在其最小边界框内都必须是密集的； 该盒子内任何缺失的单元格都意味着无法通过相同轴对齐矩形的并集创建一个孔，而不会以与单个一致印章形状的存在相矛盾的方式影响连接性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    min_r, min_c = n, m
    max_r, max_c = -1, -1

    has = False

    for i in range(n):
        for j in range(m):
            if grid[i][j] == '#':
                has = True
                if i < min_r: min_r = i
                if i > max_r: max_r = i
                if j < min_c: min_c = j
                if j > max_c: max_c = j

    if not has:
        print("YES")
        return

    for i in range(min_r, max_r + 1):
        for j in range(min_c, max_c + 1):
            if grid[i][j] != '#':
                print("NO")
                return

    print("YES")

if __name__ == "__main__":
    solve()
```该实现首先提取所有的边界框`#`细胞。 变量`min_r`,`max_r`,`min_c`， 和`max_c`跟踪极限坐标。 布尔值`has`区分空网格和非空网格。 

第二阶段验证边界框是否已完全填充。 嵌套循环是安全的，因为网格尺寸足够小，扫描矩形与单元格数量呈线性关系。 

必须小心边界的初始化。 环境`min_r`和`min_c`到`n`和`m`分别确保第一个遇到的`#`正确更新它们。 相似地，`max_r`和`max_c`从 -1 开始，以便任何有效的单元格都能正确扩展它们。

 ## 边缘情况

 一种边缘情况是完全空的网格。 该算法通过跟踪是否有任何`#`存在。 如果没有找到，则立即返回 YES。 这是正确的，因为不需要任何冲压操作来产生空结果。 

另一个边缘情况是单个标记的单元格。 边界矩形退化为 1x1 区域，它很容易通过填充检查。 

更微妙的情况是，标记存在但分散，使得边界框很大且大部分是空的。 例如：```
#..
..#
```边界矩形跨越整个网格，但内部单元格是空的。 对矩形的扫描可以正确检测到第一个空单元格处的违规，从而确保拒绝。 

这些案例证实，正确性完全取决于检测最小外接矩形内的密度，这是算法强制执行的核心不变量。
