 ## 解决方案
 ## 问题理解

 我们正在使用一棵有根树，其中顶点 1 被视为根。 对于每个顶点$x$，我们从概念上查看其子树中的所有节点，并根据它们与$x$就树中向下的边缘而言。 

更具体地说，对于固定顶点$x$， 定义$d_{x,i}$作为顶点数$y$这样$y$位于子树中$x$以及来自的路径$x$到$y$准确地下降$i$边缘。 所以$d_{x,0}$总是 1，因为它很重要$x$本身。 数值较大$i$计算深度存在多少个节点$i$以下$x$。 

对于每个顶点，我们被要求找到该分布的“主导索引”。 该指数$j$选择这样的值$d_{x,j}$严格大于其左边的所有内容，并且永远不会被其右边的任何内容超过。 换句话说，我们正在寻找序列中的一个峰值，其中它之前的所有内容都严格较小，而它之后的所有内容都不大。 

该树最多可以包含一百万个顶点，这立即排除了任何为每个节点独立重新计算子树深度分布的解决方案。 单独探索每个子树的简单方法会重复遍历树的大部分，在最坏的情况下导致二次行为。 

链状树中出现了微妙的边缘情况。 如果树是单路径，则每个子树也是一条路径。 在这种情况下，每个深度的所有计数都恰好为 1，因此每个索引都满足条件，但由于左侧严格不等，定义强制使用最小的有效索引。 这导致到处都回答 0。 

另一个极端情况出现在星形树中。 根有很多子节点，所有深度分布都集中在深度 1。对于根，索引 1 占主导地位，而对于叶子，仅存在索引 0。 简单的基于 DFS 的重新计算经常会错误地处理不同根之间的子树贡献。 

## 方法

 暴力方法将为每个节点计算$x$，一个仅限于其子树的 DFS，并计算每个深度出现的节点数量。 这需要花费$O(n)$在最坏的情况下，每个节点产生$O(n^2)$总的复杂性对于$n = 10^6$。 

关键的观察结果是序列$d_{x,i}$正好是$i$子树的第-层$x$。 我们正在寻找每个子树中最“填充”的深度级别，并具有特定的平局规则：较早的深度必须严格小于它，较晚的深度不得超过它。 

这是一个经典的设置，其中“从小到大”在树上合并变得很有用。 如果我们自下而上地处理树，我们可以为每个节点维护其子树中深度的频率数组。 合并子项时，我们总是将较小的结构合并为较大的结构，以保证整体近线性的复杂性。 

在合并时，我们还跟踪当前最佳候选深度索引。 每次我们添加移动 1 个深度的子项的计数时，我们都会更新频率值并通过比较受影响的级别来调整最佳索引。 约束结构保证每个节点的贡献最多通过合并向上移动$O(\log n)$次摊销。 

这减少了维护每个子树的深度频率图并在向上构建时提取主导索引的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(n)$| 太慢了 |
 | 树上从小到大的 DSU |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将树的根设为 1，并运行 DFS 在处理父级之前处理子级。 

1. 从根开始执行 DFS 遍历，确保处理当前节点之前的所有子节点。 这建立了一个后序结构，其中子树信息在处理节点之前可用。 
2.对于每个节点$x$，初始化频率结构$freq_x$和$freq_x[0] = 1$。 这表示节点本身在深度 0 处贡献了一个顶点。 
3. 对于每个孩子$c$的$x$，首先计算$freq_c$。 然后将其所有深度计数移+1，因为中的每个节点$c$的子树相对于更深的一条边$x$。 
4. 合并$freq_c$进入$freq_x$。 为了控制复杂性，请始终将较小的频率图合并到较大的频率图中。 如果$freq_x$较小，合并前交换它们。 这确保每个元素仅移动对数次数。 
5. 合并所有子项后，计算主导索引$x$。 我们按升序扫描深度值，同时跟踪迄今为止看到的最佳值。 我们选择最小的索引$j$这样所有早期的值都严格小于它，并且没有后来的值超过它。 实际上，这变成了识别最大频率出现的深度，并朝着满足前缀约束的最小有效索引进行平局决胜。 
6. 将此索引存储为节点的答案$x$。 
7. 返回$freq_x$向上到父调用。 

### 为什么它有效

 在每个节点，该算法都维护子树深度的正确多重集。 从小到大的合并确保子树贡献的处理次数不会超过对数次数，但更重要的是，它保留了每个深度的精确频率计数。 

主导索引条件仅取决于单个子树内跨深度的频率的相对比较。 由于合并为每个子树生成精确的计数，并且不会丢失或重复计数任何计数，因此每个节点处的计算索引直接与定义匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n + 1)]
for _ in range(n - 1):
    a, b = map(int, input().split())
    g[a].append(b)
    g[b].append(a)

ans = [0] * (n + 1)

def dfs(u, p):
    mp = {0: 1}
    best_idx = 0

    for v in g[u]:
        if v == p:
            continue
        child_mp = dfs(v, u)

        # shift depths by +1
        shifted = {}
        for d, cnt in child_mp.items():
            shifted[d + 1] = shifted.get(d + 1, 0) + cnt

        # small-to-large merge
        if len(shifted) > len(mp):
            mp, shifted = shifted, mp

        for d, cnt in shifted.items():
            mp[d] = mp.get(d, 0) + cnt

    # compute dominant index
    max_val = -1
    best_idx = 0
    for d in sorted(mp.keys()):
        if mp[d] > max_val:
            max_val = mp[d]
            best_idx = d

    ans[u] = best_idx
    return mp

dfs(1, -1)
print("\n".join(str(ans[i]) for i in range(1, n + 1)))
```DFS 自下而上构建子树深度分布。 每个节点从深度 0 处的单个计数开始，然后在平移一级后吸收每个子节点的分布。 合并步骤经过精心编写，以确保较小的字典始终合并到较大的字典中，这可以防止二次爆炸。 

主要索引的计算方法是简单地扫描所有存储的深度计数并选择具有最大频率的深度，通过第一次出现最大值满足“严格更大前缀”条件，自动打破与较小索引的联系。 

增加了递归限制，因为否则链形树将超过 Python 的默认递归深度。

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n \log n)$| 每个节点的频率贡献按照从小到大的方式进行合并，确保每个元素的摊销对数参与 |

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n = int(input())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        a, b = map(int, input().split())
        g[a].append(b)
        g[b].append(a)

    ans = [0] * (n + 1)

    sys.setrecursionlimit(10**7)

    def dfs(u, p):
        mp = {0: 1}
        for v in g[u]:
            if v == p:
                continue
            child = dfs(v, u)
            shifted = {d + 1: c for d, c in child.items()}
            if len(shifted) > len(mp):
                mp, shifted = shifted, mp
            for k, v in shifted.items():
                mp[k] = mp.get(k, 0) + v
        best = 0
        bestv = -1
        for k, v in mp.items():
            if v > bestv:
                bestv = v
                best = k
        ans[u] = best
        return mp

    dfs(1, -1)
    return "\n".join(str(ans[i]) for i in range(1, n + 1))

 ## 边缘情况

 在单节点树中，DFS 将频率图初始化为`{0:1}`并立即分配主导索引 0。不会发生合并，因此结构仍然简单且正确。 

在根为 1 的星形树中，每片叶子仅贡献`{0:1}`，当根积累`{0:1,1:k}`在哪里$k$是孩子的数量。 最大值出现在深度 1 处，因此根返回 1，而叶仍为 0。合并过程可以正确处理此问题，因为每个子项都是独立的并且只移动一次。 

在深度链中，每个节点都继承严格递增的深度结构。 由于每个深度都具有相同的频率，因此严格来说，后面的深度不会超过前面的深度，因此主导索引始终为 0。该算法正确地反映了这一点，因为频率值在各个深度上保持一致。
