 ## 解决方案
 ## 问题理解

 我们有一条路分成$n$单位段，每个段都有一个“基础疲劳成本”$a_i$这适用于 Leha 开始新的驾驶课程时。 关键在于，疲劳程度不仅取决于公里指数，还取决于自上次休息以来行驶了多少公里。 

如果 Leha 开车了$i$自上次休息以来的公里数，则下一公里的费用$a_{i+1}$。 由于阵列是非递减的，长时间的连续驾驶只会变得更加困难。 

在任何一个$n-1$公里之间的位置，可能有也可能没有独立的休息站。 这意味着休息站的每种配置都对应于将道路分成几段，并且在每个段内，成本模式从$a_1$。 

我们没有被要求直接计算期望值的分数。 相反，我们计算期望乘以$2^{n-1}$，相当于对所有休息站子集的总成本求和。 

限制条件很大：$n \le 10^6$。 任何迭代子集甚至每个子集的所有段的解决方案都是不可能的。 我们至少需要线性时间，并且理想情况下需要每个位置具有恒定工作量的单次通过。 

一个天真的方法会枚举所有$2^{n-1}$休息站的子集并模拟每个旅程$O(n)$，导致$O(n 2^n)$，这远远超出了可行的限度。 

第二个天真的想法是修复分段分解并尝试组合地计算每个分段的贡献，但如果没有仔细构建，这仍然会导致指数枚举。 

主要微妙的边缘情况是误解了正在平均的内容。 我们不是对分段进行平均；而是对分段进行平均。 我们正在对所有子集的贡献求和，这允许在每公里位置的水平上使用期望的线性度。 

## 方法

 蛮力的观点很简单：对于休息站的每个子集，模拟行程，计算其总疲劳度，并对结果求和。 这是正确的，因为它直接匹配定义。 然而，它需要迭代指数级的许多配置，并且每次模拟都是线性的$n$，因此它在约束下立即崩溃。 

关键的结构转变是停止考虑完整路径，而是专注于单公里位置$i$。 总的答案是所有配置中每公里的所有贡献的总和。 如果我们可以计算每次有多少次$a_k$如果出现在全局总和中，问题就变成了组合问题而不是程序问题。 

固定位置$i$。 在任何配置中，成本$i$第 1 公里取决于紧接其之前出现的连续非休息位置的数量。 这相当于距最近休息站（或起点）的距离。 

关键的观察是，对于固定的$i$，其“段长度位置”的概率分布仅取决于其左侧发生了多少个连续的非休息决策，以及其左侧的休息位置$i$是独立的。 这使我们能够通过考虑以结尾的不间断段的所有可能长度来计算贡献$i$。 

现在翻转视角。 我们不是分配每个配置的成本，而是计算每个配置的次数$a_j$用于位置的所有配置$i$。 为了$a_j$用于位置$i$, 之前或之后的最后一次休息$i$必须完全是$i-j$一步之遥，并且之间的所有位置都必须没有休息。 此类配置的数量完全可以分解为 2 的幂。 

这产生了一个类似卷积的结构：每个$a_j$为许多职位做出贡献$i \ge j$，按在约束块外选择休息站的方式数量进行加权。 最终的表达式简化为前缀和 DP，其中贡献在线性时间内向前累积。 

我们维护一个不断发展的贡献乘数，捕获有多少配置扩展了给定的活跃段，并累积了加权贡献$a_i$当我们扫地时。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n 2^n)$|$O(n)$| 太慢了 |

 ## 算法演练

 我们将答案重新解释为所有休息位置子集的总和，但在整个数组中增量计算。 

1. 我们从左到右处理位置，同时维护以当前位置结束的段存在多少个有效的剩余位置配置。 休息站子集的总数始终为$2^{i-1}$对于前缀$i$，因此我们不断扩大在这个领域的贡献。 
2.就位$i$，我们计算有多少配置使当前公里成为路段中的第一公里。 这恰好发生在之前休息的时候$i$，或者我们正处于开始阶段。 这个结构边界决定了多久$a_1$用于位置$i$。 
3.我们保持运行价值$cur$，代表当前段结束的累计贡献权重$i$。 这捕获了先前的休息放置允许段在不重置的情况下延伸的方式。 
4. 对于每个$i$，我们更新$cur$将其加倍（表示扩展配置时是否在之前的位置放置休息），然后添加基本贡献$a_1$，对应于在以下位置开始一个新段$i$。 
5. 对位置答案的贡献$i$正是$cur$，因为它聚合了确定公里成本的所有有效路段状态$i$跨所有配置。 
6. 我们将这些贡献累加到最终答案模中$998244353$。 

### 为什么它有效

 休息站的每种配置都会产生独特的阵列分段，每公里的成本仅取决于其在分段内的位置。 当对所有配置求和时，每个部分段结束于$i$在该州只计算一次$cur$。 加倍步骤枚举先前的位置是否引入新的段边界，确保所有子集都被表示。 贡献的线性确保求和$cur$全面的$i$重建所有配置的总和，而不会过度计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    cur = 0
    ans = 0

    for i in range(n):
        if i == 0:
            cur = a[0]
        else:
            cur = (cur * 2 + a[i]) % MOD
        ans = (ans + cur) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```实施保持滚动贡献`cur`表示以当前位置结束的所有段状态的总权重。 加倍步骤反映了每个先前的配置可以在没有休息的情况下延伸或被新的休息位置分开，这使得兼容的段结构的数量加倍。 添加`a[i]`引入从位置开始的新段$i$，对应于疲劳模式的新重置。 

最终答案累积了所有职位的所有此类细分贡献。

 ## 边缘情况

 单个元素输入测试算法是否正确初始化而不依赖于先前的状态。 更新规则绕过加倍，直接将答案设置为$a_1$，这与行驶一公里只有一种方法这一事实相符。 

当所有值相等时，每个段配置的贡献是对称的，因此加倍结构必须正确考虑段选择的指数增长。 运行循环确保每个新位置使所有先前配置加倍，同时添加新的开始，这与休息放置引起的组合扩展相匹配。
