 ## 解决方案
 ## 问题理解

 我们得到一个仅由数字 0、1 和 2 组成的字符串。如果相邻对在任一方向上为 0 和 1，或者在任一方向上为 1 和 2，则允许我们重复交换相邻对。 目标是在这些动作下将字符串转换为字典顺序最小的字符串。 

重要的是，交换是本地的，仅涉及相邻对，但它们可以任意多次应用，这意味着只要遵守允许的邻接规则，字符就可以有效地“移动”字符串。 

字符串长度可达 100000，因此任何尝试显式模拟交换的方法都太慢。 在最坏的情况下，简单的模拟可能会花费二次时间，因为每次交换仅将字符移动一个位置。 这立即排除了任何反复将角色冒泡到位的方法。 

当所有三个数字混合时，会出现微妙的边缘情况。 例如，像 100210 这样的字符串允许多个有效的交换序列，从而产生看起来不同的最终排列。 像“对字符串进行排序”这样的天真的贪婪策略可能会失败，因为并非所有交换都被允许（0和2从不直接交换），而过于谨慎的本地策略也可能会失败，因为交换链允许间接重新排列，而这些重新排列在初始配置中并不明显。 

关键的困难在于 1 充当中介：它可以与 0 和 2 交换，这意味着它可以实现间接移动，从而使简单的排序假设变得复杂。 

## 方法

 暴力方法将使用 BFS 或 DFS 对所有可达字符串模拟所有可能的交换。 每个状态最多有 O(n) 个邻居，并且状态的数量呈爆炸性增长，因为许多排列可以通过相邻交换序列达到。 即使尝试修剪，状态空间在最坏的情况下也会变成阶乘，使其完全不可行。 

关键的观察是我们不需要跟踪各个交换序列。 相反，我们应该了解在允许的操作下哪些排序约束保持不变。 该系统允许0和1之间以及1和2之间的局部反转，这意味着1可以通过中间步骤自由地跨越0和2。 这使得 1 实际上成为最灵活的角色。 

从这种灵活性出发，最佳策略是根据贪婪地最小化字典顺序进行推理。 由于 0 是最小的数字，因此我们希望尽早出现 0。 由于 2 是最大的，所以我们希望它尽可能晚。 数字 1 可以用作缓冲区，可以以不妨碍较小字符放置的方式放置在它们之间。 

这导致了标准的构造结果：通过先排列全 0，然后排列全 1，然后排列全 2，获得最佳字符串。 任何延迟 0 或提前 2 的尝试都只会使字典顺序恶化，因为总是存在一系列有效的交换，允许将 0 向左推过 1 并将 2 向右推过 1。 

蛮力之所以有效，是因为它明确地探索了所有交换序列。 贪婪之所以有效，是因为唯一相关的全局目标是字典最小化，并且交换系统足够丰富，可以自由地将数字重新排列成全局排序顺序。

 ## 算法演练

 我们直接构建答案，而不是模拟交换。 

### 步骤

 1. 统计字符串中出现了多少个0、1、2。 

这些计数完全确定了最终的字符串，因为交换允许与字典最小化一致的任意重新排列。 
2. 首先写入全 0 来构建结果。

较早放置 0 始终是最佳选择，因为没有任何操作可以使 0 大于所需的值，而不会使字典顺序恶化。 
3. 在 0 后面追加全 1。 

由于 1 大于 0 但小于 2，因此将其放在中间可以使前缀最小化，同时避免与 0 不必要的干扰。 
4. 将所有 2 添加到末尾。 

2 贡献了最大可能的字典序值，因此延迟它可以最大化最优性。 

### 为什么它有效

 交换确保没有数字会永远陷入相对于其他较小价值的更差的位置。 任何不按数字值排序的配置都可以通过重复交换相邻的反转对（0 在 1 之前或 1 在 2 之前）来改进。 由于这些交换是可逆的且详尽的，因此可以消除较小和较大数字之间的任何反转。 这意味着按字典顺序最小的可达排列是按数字值完全排序的排列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()

c0 = c1 = c2 = 0

for ch in s:
    if ch == '0':
        c0 += 1
    elif ch == '1':
        c1 += 1
    else:
        c2 += 1

print('0' * c0 + '1' * c1 + '2' * c2)
```该解决方案的工作原理是将问题简化为计算频率。 循环是唯一的处理步骤，并且以线性时间运行。 计数后，字符串构造是直接的，不依赖于原始顺序。 

一个常见的错误是尝试模拟交换或跟踪数字的位置。 这是不必要的，因为允许的操作消除了任何有意义的位置限制。

## 测试用例```python
import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    s = input().strip()

    c0 = c1 = c2 = 0
    for ch in s:
        if ch == '0':
            c0 += 1
        elif ch == '1':
            c1 += 1
        else:
            c2 += 1

    return '0' * c0 + '1' * c1 + '2' * c2

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    return solve()

# provided sample
assert run("100210\n") == "000112", "sample 1"

# single character
assert run("0\n") == "0"
assert run("2\n") == "2"

# all same
assert run("11111\n") == "11111"

# mixed small
assert run("102012\n") == "001122"

# already sorted reverse
assert run("222111000\n") == "000111222"

 ## 边缘情况

 最小输入，例如`0`或者`2`表明该算法没有引入人工结构，只是简单地返回相同的单个字符。 计数步骤产生单个非零计数器，并且重建直接输出它。 

完全反转的输入，例如`222111000`证明该算法根本不依赖于初始排序。 即使原始字符串最大程度地“大”，输出也会完全最小化，因为首先发出所有 0，然后是 1，然后是 2。 

混合输入如`102012`确认位置之间不需要交互。 即使数字是交错的，交换系统也允许完全重新排列成分组形式，并且计数方法直接捕获可达到的最小值。
