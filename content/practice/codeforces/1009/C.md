 ## 解决方案
 ## 问题理解

 我们从一个长度的数组开始$n$最初只包含零。 鲍勃表演$m$运营。 每个操作由一对定义$(x, d)$，并且鲍勃可以选择一个中心位置$i$在数组中。 一旦他这样做了，每个位置$j$接收仅取决于其距离的附加贡献$i$：该位置的增加值$j$是$x + d \cdot |i - j|$。 

以任意顺序应用所有操作并为每个操作选择中心后，数组就变成了某个最终的整数序列。 目标是最大化数组的平均值，这相当于最大化所有元素的总和，因为$n$是固定的。 

关键的困难在于，每个操作都以结构化的“V 形”模式将值分布在整个数组中，并且我们可以自由地将其中心放置在任何地方。 

限制条件$n, m \le 10^5$这意味着任何试图模拟所有位置上的每个操作的解决方案都是不可能的。 直接模拟需要$O(nm)$更新，其顺序为$10^{10}$，远远超出了极限。 即使存储每个操作的完整每个位置效果也会太大。 

选择中心的自由产生了一个微妙的问题$i$。 一种幼稚的方法可能假设我们可以在不考虑全局结构的情况下贪婪地选择每个操作的最佳位置，但这会忽略同一操作如何根据放置以及其线性结构与整个数组的交互方式做出不同的贡献。 

幼稚推理的一个小失败例子是$d < 0$。 然后，该操作会给出离中心更远的更高值，这会颠覆关于放置的直觉$i$。 另一个边缘情况是$d = 0$，其中操作成为独立于位置的常数加法，使得放置无关紧要。 

## 方法

 蛮力解释很简单。 对于每个操作$(x, d)$，我们尝试所有可能的中心$i$，计算对数组总和的总贡献，并选择最佳的。 对于每一个选择$i$，我们将评估总和$x + d|i-j|$全面的$j$，这已经花费了$O(n)$。 为所有人做这件事$i$进行单一操作$O(n^2)$，并跨越$m$这变成了操作$O(n^2 m)$，这是完全不可行的。 

关键的观察是我们实际上从来不需要完整的数组。 我们只关心所有操作后的总和。 对于固定操作，总贡献仅取决于所选中心左侧和右侧有多少元素。 如果我们定义$i$为中心，则可以使用距离的前缀和来表示贡献。 的结构$|i - j|$干净地分成左右两部分，并且两部分都是算术级数。 

这减少了每个操作的选择$i$最大化涉及到左端点和右端点的距离的简单表达式。 最佳位置总是位于边界之一，或者可以用单调函数来表征$i$，使我们能够计算出最佳位置$O(1)$每次操作。 

因此，我们不是模拟阵列，而是独立计算每个操作的最佳可能总增益并进行累加。

 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2 m)$|$O(n)$| 太慢了 |
 | 最佳|$O(m)$|$O(1)$| 已接受 |

 ## 算法演练

 我们重写单个操作的贡献，并通过中心的选择对其进行优化。 

1.修复操作$(x, d)$假设我们选择中心$i$。 对职位的贡献$j$是$x + d|i-j|$。 我们首先将常数部分和距离相关部分分开。 常数部分贡献$n \cdot x$不管$i$，因此优化仅取决于距离项。 
2. 我们将总距离和重写为：$$\sum_{j=1}^n |i - j|$$这分为两个算术和：左侧贡献$\frac{i(i-1)}{2}$, 右侧贡献$\frac{(n-i)(n-i+1)}{2}$。 
3. 一项操作的总贡献变为：$$n x + d \left( \frac{i(i-1)}{2} + \frac{(n-i)(n-i+1)}{2} \right)$$4. 自从$n x$是常数$i$，我们只最大化：$$f(i) = d \cdot \left( \frac{i(i-1)}{2} + \frac{(n-i)(n-i+1)}{2} \right)$$5. 内函数是凸函数$i$，这意味着它的整数最大值必须出现在边界之一$i=1$或者$i=n$。 这来自对称性：距离随着远离中心而增加，距离之和在中间最小，在边缘最大。 
6. 因此，对于每个操作，我们评估两个端点$i=1$和$i=n$，计算他们的贡献，并取最大值。 
7. 将所有操作中所有选定的最佳贡献相加，然后除以$n$以获得最终答案。 

### 为什么它有效

 每个操作都是独立的，因为所有更新都是附加的和线性的。 一项操作的贡献不会影响另一项操作的分配方式。 对于任何固定操作，目标简化为最大化对称二次函数$i$。 该函数在整数上没有内部最大值，因此评估边界点就足够了。 这确保了我们不会错过更好的布局，并通过操作期望的线性来保证全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def total_dist(i, n):
    left = i * (i - 1) // 2
    right = (n - i) * (n - i + 1) // 2
    return left + right

n, m = map(int, input().split())
ans = 0

for _ in range(m):
    x, d = map(int, input().split())

    # evaluate both endpoints
    left_val = n * x + d * total_dist(1, n)
    right_val = n * x + d * total_dist(n, n)

    ans += max(left_val, right_val)

print(ans / n)
```该实现直接对导出的公式进行编码。 辅助函数使用封闭式算术级数计算距所选中心的距离总和，而不是迭代所有位置。 

我们明确评估仅$i=1$和$i=n$，这避免了对所有中心的任何搜索。 最终划分为$n$将总和转换为算术平均值。 

必须注意在最后一步之前保持整数计算以避免浮点漂移。 Python 的任意精度整数即使对于大型整数也能确保安全$n$。

 ## 边缘情况

 当$d = 0$，贡献变得与位置无关，因此两个端点产生相同的结果。 该算法评估两者并正确选择其中一个，产生$n x$正如预期的那样。 

什么时候$d < 0$，距离会受到奖励而不是惩罚。 该公式仍然可以正确评估端点，因为对称性确保在边界处实现最大距离总和。 例如，与$n=4, x=1, d=-1$， 两个都$i=1$和$i=4$产生相同的最大距离和，算法直接捕获它。 

为了$n=1$，距离项完全消失。 该公式简化为$x$，并且端点评估会返回正确的值，因为两个端点重合。 

这些案例证实，没有遗漏任何隐藏的内部极值，并且边界评估完全捕获了所有最佳配置。
