 ## 解决方案
 ## 问题理解

 我们有一个矩形网格，其中每个单元的行为就像一个有向系统。 大多数细胞都是正常的，这意味着踩到它们每次只需花费一小时。 相反，一些细胞包含虫洞，它迫使细胞沿一个固定方向瞬时移动：向上、向右、向下或向左。 

任务是从左上角移动到右下角。 移动有两种不同的成本相互叠加。 到相邻单元的正常每一步都需要花费一小时。 使用虫洞时，不会消耗时间，而是立即将飞船运送到其方向的下一个单元。 然而，虫洞是危险的：它们可能会以每个单元一次操作的代价被移除，将该单元变成正常单元并禁用强制跳跃。 

目标是按字典顺序结构的。 首先，尽量减少被移除的虫洞数量。 在实现最少移除次数的所有解决方案中，尽量减少行驶所花费的总时间。 

这些约束意味着图最多有 100 万个顶点和最多 200 万条隐式边（四方向邻接加上每个单元最多一个虫洞边）。 仅当在接近线性的时间内运行时，状态上的直接最短路径算法才是可行的。 任何针对每个状态进行大量重新计算或多个图构造的方法都将通过 10^6 尺度，但任何网格大小的二次方都是不可能的。 

一个微妙的问题是虫洞可以形成零成本链。 具有虫洞的细胞可能会进入另一个虫洞，从而产生长时间的瞬时传播。 另一个边缘情况是指向网格外部的虫洞，它们是无效的，除非被删除，否则必须被视为不可用。 

在以下情况下，幼稚的方法通常会失败：

 输入：```
1 3
R.R
```将所有移动视为成本 1 的天真的 BFS 忽略了使用虫洞可以传送，因此它会错误地计算距离 2，而不是认识到第一个单元强制进行正确的移动。 

另一个失败案例：```
2 2
R.
..
```这里虫洞指向界限之外。 任何未经验证而盲目遵循虫洞转换的算法都会崩溃或错误地扩展图。 

## 方法

 暴力方法将明确地模拟是否删除每个虫洞单元或保留它的决策。 对于虫洞的每个子集，我们构建结果图并通过 BFS 计算最短路径。 这立即爆炸到 O(2^k·nm)，即使对于小网格也是不可行的。 

一种更结构化的暴力思想是将每个具有虫洞的单元视为具有两种状态（移除或未移除），并在扩展的状态空间上运行最短路径。 这创建了一个大小为 O(nm·2) 的图，但转换仍然需要在删除决策上进行分支，从而在最坏的情况下导致指数路径。 

关键的观察是，我们不是独立决定每个路径步骤的删除，而是全局选择一组要禁用的虫洞，以便最终路径中使用的所有强制转换都是有效且有益的。 这表明我们需要翻转视角：我们不是首先选择移除，而是允许虫洞存在，并且只有在选择忽略它们时才会付出惩罚。 

这自然会导致具有两个成本的最短路径问题：第一个优先级是我们选择绕过虫洞的次数（即，将其视为已删除），第二个是旅行时间。 这是一个字典最短路径问题，可以用修改后的 BFS 或 Dijkstra 来解决，其中状态带有一对成本。 

每个单元通过自由有向边（虫洞）或通过 cost-1 边（手动移动）进行转换，无效虫洞被视为需要在移动前进行 cost-1“移除”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 | O(2^{nm} · nm) | O(2^{nm} · nm) | O(纳米) | 太慢了 |
 | 0-1 BFS / 字典最短路径 | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们将每个单元建模为图中的一个节点。 从每个节点，如果有效，我们可以以零移除成本跟踪其虫洞转换，或者我们可以“删除”虫洞，而是以时间成本 1 沿四个方向之一移动。 

该结构最好使用双端 BFS 变体来处理，其中边具有成本（0 或 1），但具有二维的字典顺序扭曲。 

1. 隐式构建一个图，其中每个单元连接到最多五个选项：其虫洞目标（如果有效），其成本（0 个移除，0 时间），以及其四个邻居的成本（0 或 1 移除，具体取决于我们是否覆盖虫洞，时间 +1）。 
2. 定义一个距离数组，其中每个状态存储一对（移除、时间）。 我们按字典顺序比较对。 
3、使用双端队列实现0-1 BFS扩展到字典序比较。 当转换成本为 (0, 0) 时，推到前面。 当增加时间或移除时，推到后面。 
4. 使用（0 次移除，0 次）初始化起始单元。 
5. 在松弛过程中，如果虫洞指向网格之外，则将该方向视为不可用，除非我们支付移除成本并考虑正常移动。 
6. 对于每个邻居转换，仅当新的一对在字典顺序上小于存储的一对时才更新状态。 
7. 答案是存储在目标单元中的对。 

关键的实现细节是我们从未明确地在全局范围内删除虫洞。 相反，每当虫洞导致无效或次优移动时，我们都会在本地将其视为成本增量。 这将组合选择问题转换为最短路径问题。 

### 为什么它有效

从开始到结束的每条路径都对应于一系列决策：要么我们尊重虫洞，要么我们超越它。 每个覆盖对移除计数的贡献正好为 1。 任何有效的解决方案都可以以相同的成本映射到该图中的一条路径，并且该图中的任何路径都对应于有效的修改策略。 由于所有边都遵循相同的成本结构，因此字典排序下的最短路径会产生最佳对。 BFS 松弛确保一旦以最小对成本最终确定状态，以后的更新就无法在不与边缘单调性相矛盾的情况下改进它。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

INF = (10**18, 10**18)

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]

    # distance: (removals, time)
    dist = [[INF] * m for _ in range(n)]

    # directions
    dirs = {
        'U': (-1, 0),
        'D': (1, 0),
        'L': (0, -1),
        'R': (0, 1)
    }

    dq = deque()
    dist[0][0] = (0, 0)
    dq.append((0, 0))

    def relax(x, y, nd):
        if nd < dist[x][y]:
            dist[x][y] = nd
            dq.append((x, y))

    while dq:
        x, y = dq.popleft()
        cd = dist[x][y]

        # 1. wormhole move (cost 0,0 if valid)
        c = g[x][y]
        if c in dirs:
            dx, dy = dirs[c]
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < m:
                relax(nx, ny, cd)

        # 2. normal moves (cost (0 or 1), +1 time)
        for dx, dy in dirs.values():
            nx, ny = x + dx, y + dy
            if not (0 <= nx < n and 0 <= ny < m):
                continue

            if g[x][y] in dirs:
                # we remove wormhole: +1 removal +1 time
                nd = (cd[0] + 1, cd[1] + 1)
            else:
                # normal move: +1 time only
                nd = (cd[0], cd[1] + 1)

            relax(nx, ny, nd)

    print(dist[n-1][m-1][0], dist[n-1][m-1][1])

if __name__ == "__main__":
    solve()
```该实现保留了一个 2D 距离表，该表存储按字典顺序排列的最小对。 松弛函数强制单调改进。 首先处理虫洞转换，因为它们不消耗时间或移除。 然后我们探索标准邻接，其成本取决于当前单元的虫洞是否被忽略。 

一个微妙的点是，当我们决定不使用其虫洞行为时，移除是按每个单元收费的，而不是按边缘收费。 这就是为什么通过正常运动离开虫洞单元会产生成本增量。

    from collections import deque

    INF = (10**18, 10**18)

    def solve():
        n, m = map(int, input().split())
        g = [input().strip() for _ in range(n)]

        dist = [[INF] * m for _ in range(n)]
        dirs = {'U':(-1,0),'D':(1,0),'L':(0,-1),'R':(0,1)}

        dq = deque()
        dist[0][0] = (0,0)
        dq.append((0,0))

        def relax(x,y,nd):
            if nd < dist[x][y]:
                dist[x][y] = nd
                dq.append((x,y))

        while dq:
            x,y = dq.popleft()
            cd = dist[x][y]

            c = g[x][y]
            if c in dirs:
                dx,dy = dirs[c]
                nx,ny = x+dx,y+dy
                if 0<=nx<n and 0<=ny<m:
                    relax(nx,ny,cd)

            for dx,dy in dirs.values():
                nx,ny = x+dx,y+dy
                if not (0<=nx<n and 0<=ny<m):

                relax(nx,ny,nd)

        return dist[n-1][m-1]

    return str(solve())

 ## 边缘情况

 指向网格之外的虫洞是最脆弱的情况。 该算法将其视为不可用，强制通过正常边缘处理移动。 例如：```
1 2
R.
```从 (1,1) 开始，虫洞建议移动到 (1,2)，这是有效的，但在：```
1 1
.
```根本没有任何移动，算法正确地将距离保持为零。 

另一个边缘情况是虫洞链。 由于虫洞跃迁会立即应用，而不消耗时间或移除成本，因此重复弛豫自然会通过单个 BFS 波中的长链传播。 

最后，没有虫洞的网格简化为网格图中的标准最短路径。 该算法以单位成本干净地退化为曼哈顿边缘上的 BFS，确认了与经典行为的一致性。
