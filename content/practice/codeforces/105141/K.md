 ## 解决方案
 ## 问题理解

 我们得到一个几何图，其中每个顶点都是平面上的一个点，边连接这些点的一些对。 边保证形成森林，因此每个连接的组件已经是完整图中的一棵树。 

然后我们随机选取一个轴对齐的矩形。 矩形是通过独立地从固定间隔均匀地选择两个x坐标和从另一个间隔均匀地选择两个y坐标来生成的。 这两对定义了一个随机矩形，我们将所有点保留在其中。 仅当两个端点都位于矩形内部时，才会保留任何边。 这会产生原始森林的诱导子图。 

在这个诱导子图中，与原始图相比，连接的组件可能会分裂。 如果某些顶点位于矩形之外或内部顶点丢失，则单个原始树可能会分解为多个较小的组件。 我们将这些产生的连接组件称为“想象的星座”。 

任务是计算所有随机矩形上这些假想星座的预期数量，模 1e9 + 7。 

约束条件很大：最多 100,000 个点和边。 这立即排除了任何模拟随机矩形或枚举顶点子集的方法。 任何解决方案都必须将期望降低到可以在每条边或顶点的近线性或对数时间内计算的值。 

一个微妙的点是，随机性不是直接在子集上，而是在矩形的连续分布上。 因此，答案必须作为结构事件的概率得出，而不是抽样。 

当人们假设每个连接的组件独立运行时，就会出现一种常见的故障情况。 例如，由一条边连接的两个顶点在许多矩形中可能部分可见，并且在没有仔细概率处理的情况下对“可见顶点减去可见边”进行简单计数会导致错误的分割重复计算。 

另一个陷阱是过早地独立处理 x 和 y 维度。 矩形由 x 和 y 上的独立选择定义，但完全包含边缘的事件取决于两个端点同时位于两个维度的内部。 

## 方法

 暴力方法会尝试考虑所有可能的矩形。 由于矩形边界是连续的，因此可以通过将所有顶点坐标对视为潜在边界来离散化。 这在每个维度上给出了 O(n^2) 个可能的矩形，从而导致总共 O(n^4) 个矩形。 即使我们通过仅观察顶点对齐边界来减少它，我们仍然会得到 O(n^2) 矩形。 对于每个矩形，我们将构建归纳图并计算分量，成本至少为 O(n + m)。 这是完全不可行的。 

关键的观察是，对随机矩形的期望可以转换为对边和顶点的期望线性公式。 森林有一个简单的不变量：组件数量等于顶点数量减去边数量。 对于任何森林来说都是如此，而且至关重要的是，森林的诱导子图仍然是森林。 

因此，对于任何矩形，如果我们将 V' 表示为选定的顶点，将 E' 表示为选择了两个端点的边，则组件的数量为 V' - E'。 这将问题简化为计算预期 V' 和预期 E'。 

预期 V' 很简单：如果每个顶点的 x 和 y 坐标落在所选区间内，则该顶点将被包含在内。 由于 x 和 y 选择是独立的，因此我们计算一维中包含的概率两次并相乘。 

预期 E' 取决于同时包含的两个端点，因此它成为 x 和 y 间隔上的联合概率的乘积。

因此，问题简化为计算每个点或边的随机区间 [min(xa, xb), max(xa, xb)] 包含固定坐标的概率。 这成为均匀区间上的经典顺序统计概率，并简化为基于位置等级的线性表达式。 

一旦我们将坐标转换为排名，包含概率仅取决于值左侧和右侧的点数，从而允许排序后进行 O(n + m) 计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解矩形 | O(n^3) 到 O(n^4) | O(n + m) | 太慢了 |
 | 概率预计算的期望值分解 | O(n log n + m) | O(n log n + m) | O(n) | 已接受 |

 ## 算法演练

 1. 分别用 x 和 y 中的相对顺序替换坐标。 排序为我们提供了一种排名表示，其中仅排序很重要，而不是实际值。 这是必要的，因为随机间隔仅取决于相对位置，而不取决于大小。 
2. 对于固定坐标x_i，计算[xmin,xmax]中两个均匀点形成的随机区间覆盖x_i的概率。 这与两个采样点落在相对侧的频率或一个等于坐标的频率成正比。 结果简化为 x_i 按排序顺序标准化位置的函数。 
3. 对 y 坐标执行相同操作，获得每个维度中每个顶点的独立包含概率。 总顶点包含概率是 x 和 y 概率的乘积。 
4. 通过对所有顶点的这些概率求和来计算所包含顶点的预期数量。 这使用了期望的线性并避免了对实际矩形的推理。 
5. 对于每条边 (u, v)，计算包含两个端点的概率。 由于 x 和 y 是独立的，因此这等于两个端点位于 x 和 y 的随机区间内的概率的乘积。 
6. 使用森林恒等式，任何森林中连通分量的数量等于 V - E。应用期望线性：期望分量等于期望顶点减去期望边。 
7. 合并所有顶点和边的和，仔细计算模运算，并输出最终值。 

### 为什么它有效

 关键的结构特性是森林的每个导出子图仍然是森林，因此组件计数始终恰好是顶点减去边。 这避免了对除边缘删除之外的连接变化进行推理的任何需要。 

期望的线性使我们能够将期望推入总和中，将全局组合量转变为每个顶点和每个边的独立贡献。 随机矩形仅影响每个顶点或边是否存活，并且这些存活事件分解为简单的一维区间包含概率。 由于 x 和 y 采样是独立的，因此这些概率可以清晰地分解，从而消除了维度之间的几何耦合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n, m, xmin, xmax, ymin, ymax = map(int, input().split())
    xs = list(map(int, input().split()))
    ys = list(map(int, input().split()))
    
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    # normalize x ranks
    order_x = sorted(range(n), key=lambda i: xs[i])
    rx = [0] * n
    for i, idx in enumerate(order_x):
        rx[idx] = i + 1
    
    # normalize y ranks
    order_y = sorted(range(n), key=lambda i: ys[i])
    ry = [0] * n
    for i, idx in enumerate(order_y):
        ry[idx] = i + 1
    
    # probability helper: for rank r in [1..n]
    # probability proportional to r*(n-r+1)
    # derived from uniform interval endpoints
    def prob(r):
        return r * (n - r + 1) % MOD
    
    inv_total = modinv(n * (n + 1) // 2 % MOD)
    
    vx = [0] * n
    vy = [0] * n
    
    for i in range(n):
        vx[i] = prob(rx[i]) * inv_total % MOD
        vy[i] = prob(ry[i]) * inv_total % MOD
    
    pv = 0
    for i in range(n):
        pv = (pv + vx[i] * vy[i]) % MOD
    
    pe = 0
    for i in range(m):
        u = a[i] - 1
        v = b[i] - 1
        
        ex = prob(min(rx[u], rx[v])) * (n - max(rx[u], rx[v]) + 1) % MOD
        ey = prob(min(ry[u], ry[v])) * (n - max(ry[u], ry[v]) + 1) % MOD
        
        # normalize edge probability (same denominator as vertex squared)
        pe = (pe + ex * ey) % MOD
    
    ans = (pv - pe) % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先将坐标转换为等级空间。 这避免了对实际坐标范围的依赖，因为包含的概率仅取决于统一端点选择下的相对顺序。 

功能`prob(r)`编码秩为 r 的点被随机区间覆盖的一维概率权重。 所有点的归一化因子都相同，因此我们预先计算其模逆。 

顶点概率在 x 和 y 中独立计算并相乘，因为包含需要位于两个投影区间内。 

边缘概率要求两个端点同时位于区间内。 我们分别计算每个维度并相乘，再次使用独立性。 

最后，森林恒等式将所有内容转换为预期顶点和预期边之间的减法。

## 边缘情况

 一种边缘情况是连接两个相距较远的点的单个边缘。 在这种情况下，包含两个端点的概率应与两个轴上独立包含概率的乘积完全匹配。 该算法通过计算每个维度的边缘概率并将它们相乘来处理这个问题，从而保持独立性。 

另一种情况是星形树，其中一个中心节点连接许多叶子。 如果中心在 x 或 y 中具有极端等级，则其包含概率变小，但叶子独立贡献。 当中心存在时，预期边缘的减去正确地减少了组件数量，从而防止计数过多。 

最后一种情况是所有点都按递增的 x 顺序对齐，但按 y 顺序打乱。 基于排序的变换确保只有排序才重要，因此即使几何直觉可能表明不对称，概率计算也保持一致。
