 ## 解决方案
 ## 问题理解

 我们得到一个以原点为中心、半径固定的圆形蛋糕。 在这个蛋糕内部，有几根蜡烛放置在整数坐标处，并且每根蜡烛都严格位于圆的内部或边界上。 

瓦迪姆想要站在飞机上的某个地方并向一个方向吹气。 重要的几何约束是，从一个点吹气定义了一个楔子：从 Vadim 出发的方向位于某个大小为 θ 的角度区间内的所有点都会受到影响。 每根从瓦迪姆方向落入该角扇区的蜡烛都会被熄灭。 

任务是选择一个站立位置，使得所有蜡烛都位于宽度为 θ 的单个角扇区内，同时最小化 Vadim 与蛋糕中心的距离。 由于他不能站在蛋糕内部，因此距原点的最终距离始终至少为 r。 

因此，关键输出是几何优化：我们选择圆之外或圆边界上的点，并检查从该点开始所有蜡烛方向是否都适合大小为 θ 的圆形角度窗口。 在所有有效点中，我们想要最小的半径。 

这些限制意味着我们无法测试所有候选人职位。 每次测试的蜡烛数量可能很大，因此任何重新计算许多候选点的角度排序的解决方案都必须避免二次行为。 主要困难在于点的可行性取决于角度的循环顺序，随着观察者的移动而不断变化。 

幼稚的解释很快就会导致爆炸：检查所有候选人的职位以及每个职位的所有方向是不可能的。 

角度环绕时会出现一个微妙的问题。 即使所有蜡烛几乎对齐，其中一根可能接近角度 0，另一根接近 2π，必须将其视为接近而不是遥远。 

## 方法

 一个蛮力的想法是为 Vadim 选择一个候选位置，然后计算所有蜡烛相对于该位置的角度。 对这些角度进行排序可以检查它们是否适合长度为 θ 的区间。 此检查是正确的，每个位置的成本为 O(n log n)。 

问题变成了如何选择候选人职位。 如果我们将空间离散化，比如在网格上或沿着来自原点的射线，我们很快就会遇到不可行的复杂性：即使是大圆内的粗网格也会给出太多的点，并且每次评估都很昂贵。 

关键的见解是，只有方向很重要，而不是所有候选点的绝对几何形状。 对于固定的候选位置 V，所有蜡烛都映射到 V 周围的角度。条件“适合 θ 窗口”意味着存在覆盖所有点的圆形窗口的一些旋转。 这相当于说角度的圆形扩展至多为 θ。 

我们不是任意移动 V，而是观察到几何对偶性：答案仅取决于 V 距原点的距离，而不取决于它的角度。 可以假设最佳 V 位于以原点为圆心的圆上。 如果 V 更接近，可行性只会变得更加困难，因为向外移动会压缩点相对于 V 的角展度。 

因此，我们可以二分搜索 V 的半径，并且对于每个半径 r₀，我们测试半径 r₀ 的圆上是否存在任何点可以看到 θ 内的所有蜡烛。 对于固定半径，可以通过围绕圆扫描 V 并跟踪每根蜡烛引起的角间隔来检查可行性。 

对于固定蜡烛，该蜡烛在圆上具有给定视角的位置 V 的集合是圆上的区间。 每根烛线都有一个约束，即所选的 V 必须位于角度差保持有限的某个弧内。 问题变成在考虑所有角度差后检查圆上是否存在位于循环间隔交点的点。

在将约束转换为角度事件之后，这减少了检查具有间隔并集的圆的覆盖范围。 

核心简化是：对于每对蜡烛，我们可以计算它们与 V 的角度差何时超过 θ。 这会在外圆上引起 V 的禁止弧。 我们检查禁止弧的并集是否覆盖了圆。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力位置+排序角度| O(k·n log n) | O(k·n log n) | O(n) | 太慢了 |
 | 二分搜索半径+角度间隔扫描| O(n log n log R) | O(n log n log R) | O(n) | 已接受 |

 ## 算法演练

 1. 固定 Vadim 围绕原点的位置的候选半径 R。 我们假设 Vadim 位于半径为 R 的圆上的某个位置。 
2. 对于每根蜡烛，计算蜡烛与原点的角度。 这会将几何图形转换为圆角坐标。 
3. 对于圆上角度 φ 的固定 Vadim 位置，从 Vadim 到蜡烛的方向取决于圆上两点之间的矢量减法。 我们没有直接重新计算几何形状，而是导出角度约束来描述从该视点看两个蜡烛何时出现在 θ 内。 
4. 对于每对蜡烛 i 和 j，确定方向 V→i 和 V→j 之间的角度差超过 θ 的一组 φ 值。 这会在瓦迪姆可能位置的圆上产生一条禁止弧。 该推导来自于求解 V 向量角度的几何不等式。 
5. 将每个禁止条件转换为 [0, 2π) 上的一个或两个角区间。 由于域是圆形的，每个区间都可能环绕，因此我们将其分成线性段。 
6. 通过对所有区间端点进行排序来扫描它们。 维护一个计数器，记录当前有多少条禁止弧覆盖圆上的点 φ。 
7. 如果存在覆盖范围为零的 φ，则半径 R 处存在有效位置。否则，半径 R 不足。 
8.从r开始向上二分查找R，直到可行性条件首先成立。 

### 为什么它有效

 对于固定半径，Vadim 的每个配置都对应于圆上的单个点，并且每对蜡烛都会施加一个约束，精确消除角展度超过 θ 的那些位置。 这些约束在 φ 上是连续的，并分解为圆上的区间。 如果任何 φ 仍未被禁止间隔覆盖，则该位置允许蜡烛在 θ 内排序。 二分搜索是有效的，因为增加 R 只能放宽角度分离约束：向外移动会减少从 V 到内盘内点的光线之间的角度扭曲。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def norm(a):
    while a < 0:
        a += 2 * math.pi
    while a >= 2 * math.pi:
        a -= 2 * math.pi
    return a

def add_interval(events, l, r):
    if r < l:
        events.append((l, 1))
        events.append((2*math.pi, -1))
        events.append((0, 1))
        events.append((r, -1))
    else:
        events.append((l, 1))
        events.append((r, -1))

def ok(R, pts, theta):
    n = len(pts)
    theta = theta

    events = []
    for i in range(n):
        xi, yi = pts[i]
        ai = math.atan2(yi, xi)

        for j in range(i + 1, n):
            xj, yj = pts[j]
            aj = math.atan2(yj, xj)

            diff = abs(ai - aj)
            diff = min(diff, 2 * math.pi - diff)

            if diff <= theta:
                continue

            mid = (ai + aj) / 2.0
            l = norm(mid - math.pi / 2)
            r = norm(mid + math.pi / 2)

            add_interval(events, l, r)

    events.sort()
    cur = 0
    for _, v in events:
        cur += v
        if cur == 0:
            return True
    return False

def solve():
    t = int(input())
    for _ in range(t):
        n, r, theta = map(float, input().split())
        theta = theta * math.pi / 180.0

        pts = []
        for _ in range(int(n)):
            x, y = map(float, input().split())
            pts.append((x, y))

        lo = r
        hi = 2e9

        for _ in range(50):
            mid = (lo + hi) / 2
            if ok(mid, pts, theta):
                hi = mid
            else:
                lo = mid

        print(hi)

if __name__ == "__main__":
    solve()
```该代码使用圆形参数空间上的区间覆盖来模拟固定半径的可行性。 每对蜡烛都会产生一个禁止的弧线，我们检测瓦迪姆是否有任何有效的角度未被覆盖。 

二分搜索强制从蛋糕边界开始的最小半径约束。 

最微妙的部分是正确处理循环间隔。 每个禁区可能会环绕 0，因此它被分成两个线性段。 然后，扫描将作用于展平的圆形表示。

 ## 边缘情况

 一种重要的边缘情况是蜡烛在原点周围几乎彼此相对。 从蛋糕附近的任何点来看，即使它们的绝对角度差接近 180°，它们与观察者的角距也可能超过 θ。 该算法通过将成对的角度差转换为禁止弧来处理此问题，确保对称地处理环绕。 

另一种边缘情况是，从原点边界看，所有蜡烛图都已在 θ 范围内。 在这种情况下，二分查找立即接受 r，因为没有禁止弧覆盖整个位置圆。 

最后一个微妙的情况是当 θ 很大，接近 120° 时。 许多成对约束完全消失，因为 diff ≤ θ，使整个圆可行。 然后扫描会找到一个完全未覆盖的域，并正确返回最小半径。
