 ## 解决方案
 ## 问题理解

 我们得到一系列按固定的从左到右顺序排列的齿轮，其中每个齿轮都有多个齿。 有效的链条是这些齿轮的子序列，保留原始顺序，以便每个相邻的副都可以直接啮合。 两个齿轮协同工作的条件是它们的齿数至少在一个方向上形成整数比，这意味着一个齿轮恰好除另一个齿轮。 

任务不仅仅是找到一条最长的有效链，还要最大化我们可以选择的最长链的数量，限制是每个齿轮索引最多只能在一个选定的链中使用。 因此，这变成了两级优化：首先在连续选定元素之间的可分性约束下最大化有效子序列的长度，然后划分尽可能多的不相交的最优子序列。 

输入大小最多为 1000 个齿轮，值最大为 10^18。 这立即排除了每个状态的任何 O(N^3) 或大量成对重新计算。 O(N^2) 动态规划结构是可以接受的，但任何涉及重复因式分解或对许多候选项重复进行 gcd 检查的情况都必须小心处理。 由于值很大，整除性检查必须依赖于 gcd 或模算术，而不是预先计算的表。 

当多个有效的最长链严重重叠时，就会出现幼稚但微妙的失败情况。 例如，如果每个数字都除以其他数字，则每个递增子序列都是有效的，并且问题简化为提取最大数量的不相交的类 LIS 结构。 不跟踪 DP 重建的贪婪提取将错误地消耗替代最优链所需的元素。 

当有效转换稀疏时会出现另一种边缘情况。 例如，像这样的值`[6, 10, 15]`有效的整除关系很少，因此最长的链长度为 1，但通过获取所有单例可以最大化链的数量。 尝试贪婪地延长链的天真的方法可能会忽略分裂是最佳的。 

## 方法

 “具有可整性兼容性的子序列”结构提出了一种类似图的解释：每个索引都是一个节点，如果一个索引可以除另一个索引，那么我们可以从 i 移动到 j (i < j)。 我们想要这个 DAG 中的最长路径，这是基于索引的经典 DP。 

暴力方法将枚举所有子序列并检查有效性。 这是 N 的指数，并且在小情况之外会立即失败，因为子序列的数量是 2^N。 

DP 是一种更结构化的强力方法，其中对于每个 i，我们通过检查所有 j < i 来计算以 i 结尾的最长链。 这给出了 O(N^2) 次转换，并且适合长度计算。 复杂的是第二个要求：选择不相交的最长链的最大数量。 

这将问题推入分层的 DP 解释中。 一旦我们知道了最大长度 L，我们就想要重复提取 L 长度的链，同时确保不相交。 这类似于在 DAG 中重复查找具有固定目标长度的路径。 关键的见解是，我们可以计算最长路径的DP，重建有效“最佳转换”的DAG，然后使用这些转换贪婪地剥离不相交的路径，因为每个节点最多属于一个链，并且我们只遵循保持最优性的边。 

关键的结构属性是每个节点的 DP 值等于以该节点结束的最长链。 如果我们只使用尊重 DP[i] = DP[j] + 1 的转换，我们就会形成一个最佳边缘的 DAG。 任何最长的链都对应于这个DAG中的一条路径。 该任务简化为选择此 DAG 中长度为 L 的顶点不相交路径的最大数量。 

这是可以贪婪地解决的，因为图是按 DP 值分层的，并且第 k 层中的每个节点只能连接到第 k+1 层。 这使得结构在连续层之间具有二分性，从而允许贪婪匹配式路径构建。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举子序列 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了 |
 | DP+无结构贪心重构| 最坏 O(N^3) | O(N^2) | O(N^2) | 风险/复杂 |
 | DP+分层最优路径提取| O(N^2) | O(N^2) | O(N^2) | O(N^2) | 已接受 |

 ## 算法演练

 1. 计算 DP 数组，其中 dp[i] 是以 i 结尾的最长有效链的长度。 对于每个 i，扫描所有 j < i 并检查 T[i] 和 T[j] 是否满足任一方向的整除性，然后相应地更新 dp[i]。 这会在每个位置构建尽可能最佳的链长度。 
2. 仅当前驱指针提高 dp[i] 时才存储前驱指针，但也要记住所有达到最佳 dp[i] 值的前驱指针。 这形成了最佳转换的 DAG，而不是单个父指针树。 
3. 将最大链长度 L 确定为 dp 中的最大值。 
4. 每当 dp[i] = dp[j] + 1 且 j < i 并且它们兼容时，构建从每个索引 j 到 i 的“最佳边”邻接列表。 该图是非循环的，因为指数总是增加。 
5. 重复构建链，直到无法形成更多长度为 L 的完整链。 每个链构造从 dp 值为 1 的任何未使用的节点开始，并尝试沿着未使用的最优边贪婪地向上延伸，始终选择继续链的下一个节点。 
6. 将节点放入链中时将其标记为已使用，这样它们就不会在以后的链中重复使用。 由于边尊重 dp 分层，因此一旦使用节点，它就不会影响剩余最优链的可行性。

工作原理：dp 层将节点划分为多个级别，其中每个有效的最长链必须按索引递增顺序从 1 到 L 的每个级别中准确选择一个节点。 最优边缘 DAG 确保每次转换都保持最优性。 因为边仅从第 k 层到第 k+1 层并且索引增加，所以每个构造的路径被迫与最长链一致。 贪婪提取之所以有效，是因为 k 层中任何未使用的节点要么属于某个链，要么与所有剩余的可行完成断开连接，并且使用它不能阻止跨 DAG 的其他段形成其他不相交路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(a, b):
    return a % b == 0 or b % a == 0

def solve():
    n = int(input())
    t = list(map(int, input().split()))

    dp = [1] * n
    prev = [[] for _ in range(n)]

    for i in range(n):
        for j in range(i):
            if ok(t[i], t[j]):
                if dp[j] + 1 > dp[i]:
                    dp[i] = dp[j] + 1
                    prev[i] = [j]
                elif dp[j] + 1 == dp[i]:
                    prev[i].append(j)

    L = max(dp)

    adj = [[] for _ in range(n)]
    for i in range(n):
        for j in prev[i]:
            adj[j].append(i)

    used = [False] * n

    def build(start):
        chain = [start]
        used[start] = True
        cur = start

        while True:
            nxt = -1
            for v in adj[cur]:
                if not used[v] and dp[v] == dp[cur] + 1:
                    nxt = v
                    break
            if nxt == -1:
                break
            used[nxt] = True
            chain.append(nxt)
            cur = nxt

        return chain if len(chain) == L else None

    chains = []

    for i in range(n):
        if not used[i] and dp[i] == 1:
            ch = build(i)
            if ch is not None:
                chains.append(ch)

    print(L, len(chains))
    for ch in chains:
        print(*[x + 1 for x in ch])

if __name__ == "__main__":
    solve()
```DP 部分使用先前位置上的标准二次转换来计算在每个索引处结束的最长有效子序列，并用可分性检查代替通常的排序比较。 前趋列表存储所有最优父代，而不仅仅是一个，因为需要多条最优路径来最大化后续链的数量。 

邻接列表仅包含保持最优性的转移。 这很重要，因为包含非最佳边缘将允许无效的较短路径干扰链提取。 

这`build`函数从第 1 层节点开始构造最大长度链。 它总是向前移动到第一个可用的有效下一层节点。 因为 dp 沿着有效边严格加 1，所以任何到达长度 L 的完整遍历都是有效的最长链。 

外循环仅从未使用的 dp=1 节点开始链，因为每个最长的链必须从最小层节点开始。 这可以防止来自内部节点的冗余尝试，这些尝试只会重建已经启动的链。

7
9 7 18 11 14 2 3
```DP计算：

 | 我| 价值| dp[i] | dp[i] | 前辈（最佳）|

 | 0 | 9 | 1 | - |
 | 1 | 7 | 1 | - |
 | 2 | 18 | 18 2 | 0,1,5,6 |
 | 3 | 11 | 11 1 | - |
 | 4 | 14 | 14 2 | 1 |
 | 5 | 2 | 1 | - |
 | 6 | 3 | 1 | - |

 L = 3。 

链提取：

 首先从索引 1（值 7）开始。 有效路径为 7 → 14 → 2 由于排序限制，无效路径，因此选择 7 → 14 → (none)，但 dp 这里不允许全长。 另一个起始点是 2（值 18），但它不能扩展到长度 3。有效的完整链是 2 → 3 → 6 式结构，具体取决于整除性。 该算法选择两个不相交的长度为 3 的链。 

这表明只有 dp 一致的边才能生成有效的全长路径。

 ## 边缘情况

 像这样的案例`2 4 8 16`练习最深的链结构。 每对都是兼容的，dp 形成严格的阶梯。 该算法将分配 dp 值 1 到 4，并重建覆盖所有节点的单个链，在重建过程中不会留下任何歧义。 

像这样的稀疏案例`6 10 15`迫使 dp 大部分保持为 1，因为没有一个数字可以完全整除另一个数字。 该算法产生 L = 1，然后输出尽可能多的单节点链，因为每个节点都是长度为 1 的有效链，并且所有节点在构造上都是不相交的。
