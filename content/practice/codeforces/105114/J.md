 ## 解决方案
 ## 问题理解

 我们得到一个有向网格，其中每个单元格都包含一条指令：向北、向东、向南或向西移动一步。 网格不是一个有限的移动板，而是在各个方向上无限重复，就像壁纸一样。 所以平面上的每个单元都属于同一单元的某个副本$n \times m$图案。 

Alice 从原始网格的左上角单元格开始，并沿着当前单元格中的箭头不断移动。 因为网格是无限的，所以路径永远不会终止。 有趣的部分是在重新访问无限平面中的绝对位置时路径的行为方式。 

关键区别在于两种行为之间。 在一种情况下，尽管爱丽丝永远移动，但无限平面中的单个绝对单元不会被无限次访问。 轨迹不断漂移，并且永远不会重复返回相同的位置。 在另一种情况下，在爱丽丝无限次访问的无限平铺中至少存在一个绝对单元，只有当运动最终在空间中形成一个不会漂移的循环时，这种情况才会发生。 

我们可以修改原来的一些单元格$n \times m$网格。 修改会更改平铺的每个副本中的单元格。 目标是使最终的无限行走变得重复，即至少一个绝对位置被无限多次访问，同时最小化修改单元的数量。 

限制因素$n, m \le 3000$意味着多达 900 万个单元，因此任何解决方案的网格大小都必须接近线性。 任何平方项$nm$在最坏的情况下已经太慢了。 

朴素推理的一个微妙的失败案例来自于假设网格图中的任何循环都是足够的。 例如，右、下、左、上移动的 4 个循环在网格图中形成一个循环，但在无限平铺中，如果循环跨越每个完整循环的净位移不等于零的平铺边界，则它仍然可能会漂移。 这意味着我们不能纯粹依赖于图循环$n \times m$不跟踪空间位移的状态空间。 

## 方法

 直接模拟视图将每个单元视为具有单个出边的图中的节点。 因为网格是无限的，跟随边缘会导致走进$\mathbb{Z}^2$，而不仅仅是有限的网格状态集。 尽管当地政府重复了每一次$n \times m$，绝对位置发生变化，因此有限图中的循环不会自动意味着重新访问相同的绝对坐标。 

关键的观察结果是，当存在平面内总位移为零的闭合行走时，配置就会变得重复。 如果运动图中的循环具有非零净位移，则每次遍历都会将 Alice 移动到无限平面的新区域，从而防止任何绝对单元被无限频繁地重新访问。 

创建这种零位移循环的问题可以大大简化。 可以具有零位移的最小可能循环是两个相邻单元之间的2个循环。 任何更长的循环都可以分解为更简单的段，但每个附加边只会增加所需修改的数量，而不会提高最优性。 

因此任务减少为强制一对相邻单元格$u$和$v$互相指向对方，形成$u \to v \to u$。 这保证了长度为 2 的循环的净位移恰好为零。 

每个单元已经有一个传出方向，因此要构建这样一个相互对，我们可能需要修改一个或两个端点。 对于每个相邻的对，我们计算需要多少改变来强制执行两个所需的方向并取最小值。 

强力方法将尝试所有单元子集来修改并测试是否出现零位移循环。 这是指数级的$nm$，因为每个单元有四种可能的状态，使得搜索空间$4^{nm}$。 即使尝试使用位移跟踪在完整状态图上搜索周期也太昂贵。 

通过仅关注邻接对和最小循环的结构，问题就分解为网格上的线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解搜索| 指数| O(1) | O(1) | 太慢了 |
 | 检查所有相邻对| O(纳米) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 迭代网格中的每个单元并考虑其右邻居和下邻居。 这些是唯一需要的候选者，因为每个有效的 2 周期都必须使用原始平铺结构中的相邻单元。 这避免了冗余检查并涵盖了所有可能的局部循环。 
2. 对于每对相邻单元格$u$和$v$，计算在它们之间强制执行 2 个周期的成本。 这需要检查是否$u$的当前箭头已经指向$v$，以及是否$v$的箭头已经指向$u$。 每个不匹配都会造成 1 的成本，因为我们必须修改该单元格。 
3. 跟踪具有最小修改成本的对。 最好的一对代表在无限平面中创建零位移循环的最便宜的方法。 
4. 扫描所有对后，通过对所选对应用必要的修改来重建解决方案，将它们的方向设置为指向彼此。 

我们只考虑 2 个周期的原因是，该系统中的任何重复行为最终都必须对应于净位移为零的闭合行走，并且该网格运动模型中最小的此类结构是邻居之间的双向边缘。 任何更大的周期将至少需要相同数量的修改，同时不会降低成本。 

### 为什么它有效

 每个单元定义了一个确定性的转换，因此系统是网格位置上的有向图，空间位移附加到边缘。 重复路径需要封闭行走，其位移矢量之和为零。 任何这样的行走都包含一个简单的循环，并且任何简单的零位移循环必须包含至少一对相邻单元，这些单元可以转变为相互转换，而不会增加超出最佳成本的成本。 因此，将注意力限制在执行单个 2 周期上不会错过最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dir_to_delta(c):
    if c == 'N':
        return -1, 0
    if c == 'S':
        return 1, 0
    if c == 'W':
        return 0, -1
    return 0, 1

def opposite_delta(dx, dy):
    if (dx, dy) == (-1, 0):
        return 'S'
    if (dx, dy) == (1, 0):
        return 'N'
    if (dx, dy) == (0, -1):
        return 'E'
    return 'W'

n, m = map(int, input().split())
g = [list(input().strip()) for _ in range(n)]

best_cost = 10**9
best = None

def cost_and_fix(i1, j1, i2, j2):
    c1 = g[i1][j1]
    c2 = g[i2][j2]

    dx = i2 - i1
    dy = j2 - j1

    need1 = opposite_delta(dx, dy)
    need2 = opposite_delta(-dx, -dy)

    cost = (c1 != need1) + (c2 != need2)
    return cost, (i1, j1, need1, i2, j2, need2)

for i in range(n):
    for j in range(m):
        if j + 1 < m:
            c, info = cost_and_fix(i, j, i, j + 1)
            if c < best_cost:
                best_cost = c
                best = info
        if i + 1 < n:
            c, info = cost_and_fix(i, j, i + 1, j)
            if c < best_cost:
                best_cost = c
                best = info

u_i, u_j, u_c, v_i, v_j, v_c = best

g[u_i][u_j] = u_c
g[v_i][v_j] = v_c

print(best_cost)
if best_cost > 0:
    print(u_i + 1, u_j + 1, u_c)
if best_cost > 0 and (v_i, v_j) != (u_i, u_j):
    print(v_i + 1, v_j + 1, v_c)
```网格被扫描一次，并且仅考虑右邻居和下邻居以避免重复的对评估。 对于每个候选对，我们计算必须更改多少个端点才能强制执行相互连接。 

重建步骤直接应用所选方向。 根据需要，索引在输出中转换为基于 1 的格式。 

一个微妙的细节是，即使只需要一个更改，我们也总是覆盖两个端点。 无论初始配置如何，这都可确保输出的一致性。

 ## 边缘情况

 一个关键的边缘情况是多个相邻对提供相同的最低成本。 该算法自然地处理这个问题，因为它只跟踪最小成本并存储一对代表性的。 这些对中的任何一个都是有效的，因为该问题允许任何最佳解决方案。 

另一种边缘情况是网格已经包含相互对时。 在这种情况下，两个端点已经指向彼此，因此计算的成本为零并且不会发出任何修改。 该算法仍然输出有效的空修改列表。 

最后的边缘情况是最佳解决方案需要更改一对的两个端点。 重建步骤显式应用这两个更改，确保即使两个方向最初都不正确，所得配置仍然形成零位移的有效 2 周期。
