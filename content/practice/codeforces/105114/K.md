 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并要求考虑它的每个连续切片。 对于每个切片，我们计算其元素的总和并检查该总和是否为偶数。 在所有这些切片中，我们想要计算有多少个不同的切片满足这个奇偶校验条件。 

这里的“不同”指的是实际的子数组内容和位置，因此只有当两个切片来自原始数组中相同的索引时，它们才是相同的。 换句话说，每对索引$(l, r)$恰好定义了一个候选子数组，如果它的总和为偶数，我们就对它进行计数。 

数组大小可以达到$10^6$，这立即排除了任何检查所有的解决方案$O(N^2)$子数组。 即使是适度的常数因子也会使二次方法变得不可行，因为它需要大约$10^{12}$最坏情况下的操作。 

关键的边缘情况围绕奇偶校验行为。 如果所有元素都是偶数，则每个子数组的总和也是偶数，因此答案变为子数组的总数。 如果所有元素都是奇数，则奇偶校验完全取决于子数组长度，并且只有偶数长度的子数组才会起作用。 另一个微妙的情况是混合符号或大值； 由于只有奇偶校验很重要，因此大小无关紧要，但粗心的实现仍然可能会显式地重新计算总和并导致溢出或超时。 

## 方法

 解决该问题的直接方法是枚举每个可能的子数组并计算其总和。 对于每个起始索引$l$，我们扩展$r$从$l$到$N-1$，保持运行总和。 每次扩展时，我们都会检查总和是否为偶数，如果是则进行计数。 这种方法是正确的，因为它明确地对每个候选子数组进行一次评估。 

然而，这会产生大约$N(N+1)/2$子数组，并且每次扩展都需要常数时间，因此总工作量仍然是二次的。 和$N = 10^6$，这变得天文数字般大。 

关键的观察是我们实际上并不关心全额；我们关心的是金额。 我们只关心总和是否为偶数。 奇偶校验在加法下表现良好：如果元素是奇数，则添加元素会翻转奇偶校验；如果元素是偶数，则保留奇偶校验。 这建议用其奇偶性替换每个值$A_i \bmod 2$。 

现在，当两端的前缀奇偶校验匹配时，子数组的和恰好为偶数。 如果我们定义前缀奇偶校验$P[i] = (A_1 + \dots + A_i) \bmod 2$，然后求和$l$到$r$即使且仅当$P[l-1] = P[r]$。 这将问题转化为对相等前缀奇偶校验值对进行计数。 

我们维护每个前缀奇偶校验（0 或 1）出现的次数。 每次我们遇到前缀奇偶校验时，它都会贡献与之前出现相同奇偶校验的次数一样多的在此结尾的有效子数组。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N²) | O(1) | O(1) | 太慢了 |
 | 前缀奇偶校验计数| O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们根据前缀奇偶校验重写问题并增量计数匹配对。 

1.初始化频率数组`cnt`大小为 2，代表我们看到前缀奇偶校验 0 和 1 的次数。从`cnt[0] = 1`因为在处理任何元素之前，空前缀的和奇偶校验为 0。 
2. 维护一个正在运行的奇偶变量`p = 0`，表示前缀和到当前索引的奇偶校验。 
3. 从左到右迭代数组。 对于每个元素，将奇偶校验更新为`p ^= (A[i] & 1)`。 当且仅当当前元素为奇数时，才会翻转奇偶校验。 
4. 在每个位置，每个先前的前缀具有相同的奇偶校验`p`形成一个以当前索引结尾的有效偶数和子数组。 添加`cnt[p]`到答案。 
5. 计数完毕后，递增`cnt[p]`记录该前缀奇偶校验现在又发生了一次。 

每个步骤都确保我们在子数组的端点具有匹配的前缀奇偶校验时准确地对子数组进行计数，这相当于偶数和。 

### 为什么它有效

 子数组和来自$l$到$r$可以使用前缀和表示为$P[r] - P[l-1]$。 对于整数，均匀性仅取决于该差是否能被 2 整除。在模 2 算术中，减法相当于加法，因此条件变为$P[r] \equiv P[l-1]$。 该算法精确计算所有具有相同前缀奇偶校验的索引对，因此每个有效子数组都被计算一次且仅一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    cnt = [0, 0]
    cnt[0] = 1
    
    p = 0
    ans = 0
    
    for x in arr:
        p ^= (x & 1)
        ans += cnt[p]
        cnt[p] += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案将每个数字压缩为单个奇偶校验位，这避免了任何大型算术并确保每个元素的恒定时间更新。 前缀奇偶校验变量`p`使用XOR进行更新，直接对应于遇到奇数时切换奇偶校验。 

这`cnt`数组跟踪有多少个前缀位置产生了每个奇偶校验。 最初的`cnt[0] = 1`很重要，因为它考虑从索引 0 开始的子数组； 如果没有它，我们将错过从第一个元素开始的所有有效前缀。

 ## 边缘情况

 对于像这样的单元素数组`[2]`，前缀奇偶校验从 0 开始，再次翻转到 0，并且恰好贡献一个有效子数组。 算法算数`cnt[0]`正确处理前后，生成 1。 

对于所有奇数元素的数组，例如`[1, 1, 1]`，奇偶校验交替 1, 0, 1。仅当奇偶校验重复时，频率跟踪才会产生贡献，这与只有偶数长度子阵列具有偶数和的事实相匹配。 计算结果2对应子数组`[1,1]`和`[1,1]`在不同的位置，每一个都有不同的计数。 

对于像这样的全偶数组`[2,2,2,2]`，奇偶校验永远不会改变，因此每个前缀都与其他每个前缀匹配。 算法累计$4 \times 5 / 2 = 10$子数组，并且每个子数组的和都是偶数，与前缀奇偶校验 0 的连续重复一致。
