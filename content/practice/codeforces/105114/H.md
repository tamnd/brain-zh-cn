 ## 解决方案
 ## 问题理解

 给定两个相同长度的整数序列。 从任何连续的索引段，我们可以计算两个值：第一个数组中所选段的总和以及第二个数组中同一段的总和。 因此，每个段都会生成一对数字，每个数组中都有一个数字。 

任务是选择一个连续的线段，使得通过平方和相加所形成的数量尽可能小。 从几何角度来看，每个线段对应于二维平面中的一个点，我们正在寻找以平方欧几里德距离为单位最接近原点的点。 

约束允许最多五十万个元素。 所有子阵列上的任何二次方法都会尝试 N 平方段的顺序，这远远超出了可行的限制。 即使是具有轻常数因子的 O(N^2) 算法，在最坏的情况下也会尝试大约 10^11 次操作，并且无法在两秒内运行。 这立即迫使我们以避免显式枚举所有段的方式解释问题。 

当推理天真的贪婪想法时，会出现一个微妙的问题。 在一个数组中总和较小但在另一个数组中具有较大量值的线段仍然比仅最小化一个坐标的线段更好。 例如，选择单独最小化 A-sum 的段会完全忽略 B 的贡献，并且可能会错过真正的最优值。 另一个失败的情况是尝试贪婪地缩小或扩大窗口，因为目标在段长度上不是单调的。 

关键的困难在于，每个段都依赖于两个独立的累积和，因此我们实际上是在对由数组引起的二维结构进行优化。 

## 方法

 暴力法很简单。 我们迭代所有端点对 L 和 R，计算该区间上 A 和 B 的总和，并评估目标。 使用前缀和，这将每个查询减少到 O(1)，但 L 和 R 仍然有 O(N^2) 个选择，导致在最坏的情况下进行大约 10^11 次评估。 这太慢了。 

关键的观察是子数组和可以使用前缀和重写。 令 SA[i] 和 SB[i] 为前缀和。 然后线段 [L, R] 上的总和变为 SA[R] 减去 SA[L−1]，对于 B 也是如此。因此，每个线段对应于 2D 平面中两个前缀点之间的差。 

如果我们定义一个点 P[i] = (SA[i], SB[i])，包括 P[0] = (0, 0)，那么每一段都对应于 P[R] − P[L−1]。 目标成为该前缀和点集中两点之间的平方距离。 因此，问题简化为在二维的 N+1 个点中找到最近的一对点。 

这将问题转化为经典的计算几何任务：平面上最接近的点对。 分而治之的方法通过递归地求解按 x 坐标排序的两半，然后仅检查一个狭窄的垂直条带，其中来自两半的候选者可能形成更接近的对，从而实现 O(N log N)。 在该条带内，按 y 坐标排序可以进行有效的比较。 

这正是我们需要的结构，因为前缀点除了任意坐标之外没有特殊约束，因此不适用更简单的一维技巧。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N²) | O(1) 额外（或 O(N) 带前缀和）| 太慢了 |
 | 最佳（最接近的对）| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 1. 构造前缀和数组 SA 和 SB，并为 i 从 0 到 N 构建点 P[i] = (SA[i], SB[i])。 

这会将每个子数组转换为两点之间的差异，从而消除了直接的区间处理。 
2. 按 x 坐标对所有点进行排序。

这种排序需要将问题分成具有平衡几何形状的左半部分和右半部分。 
3. 定义一个递归函数，该函数采用按 x 排序的点段，并返回该段内的最小平方距离以及按 y 坐标排序的相同点的列表。 

y 排序列表对于有效构建跨界候选集至关重要。 
4. 如果线段尺寸较小，则直接计算所有成对距离并返回。 

小基本情况避免了递归开销，并且比进一步分割更快。 
5. 按中值 x 坐标将点分成左半部分和右半部分，并递归求解两半。 

每半部分独立地找到其最佳的内部对。 
6. 合并步骤：收集与分割线的 x 距离小于迄今为止找到的最佳距离的点。 

只有这些点才能形成跨越边界的更好的对，因为在 x 上相距较远的任何对都必须已经超过当前的最佳平方距离。 
7. 按 y 坐标对该条带进行排序，并根据 y 顺序中的接下来的几个点检查每个点。 

几何填充参数确保每个点只需要恒定数量的比较。 
8. 返回左半边、右半边和跨边界检查中找到的最佳距离，以及更高递归级别的 y 排序列表。 

最终答案是在所有递归级别中找到的最小平方距离。 

### 为什么它有效

 每个子数组唯一对应于一对前缀点，因此最小化子数组平方和与查找所有前缀点中最接近的一对相同。 分而治之的步骤保证任何候选对要么完全在一半之内，要么跨越划分。 对垂直条带的跨边界限制是有效的，因为任何具有大水平间隔的对必须已经具有大于当前最佳距离的平方距离。 在条带内，y 顺序的限制比较依赖于几何填充：太多的接近点会迫使其中两个点比当前最佳点更接近，从而与最优性相矛盾。 这保持了正确性，同时避免了二次比较。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    A = list(map(int, input().split()))
    B = list(map(int, input().split()))

    px = 0
    py = 0
    pts = [(0, 0)]

    for i in range(n):
        px += A[i]
        py += B[i]
        pts.append((px, py))

    pts.sort(key=lambda x: x[0])

    def dist2(p1, p2):
        dx = p1[0] - p2[0]
        dy = p1[1] - p2[1]
        return dx * dx + dy * dy

    def closest(arr):
        m = len(arr)
        if m <= 3:
            best = float('inf')
            for i in range(m):
                for j in range(i + 1, m):
                    best = min(best, dist2(arr[i], arr[j]))
            return best, sorted(arr, key=lambda x: x[1])

        mid = m // 2
        midx = arr[mid][0]

        left = arr[:mid]
        right = arr[mid:]

        d1, ly = closest(left)
        d2, ry = closest(right)
        d = min(d1, d2)

        merged_y = []
        i = j = 0
        while i < len(ly) and j < len(ry):
            if ly[i][1] < ry[j][1]:
                merged_y.append(ly[i])
                i += 1
            else:
                merged_y.append(ry[j])
                j += 1
        merged_y.extend(ly[i:])
        merged_y.extend(ry[j:])

        strip = []
        for p in merged_y:
            if (p[0] - midx) * (p[0] - midx) <= d:
                strip.append(p)

        for i in range(len(strip)):
            for j in range(i + 1, len(strip)):
                if (strip[j][1] - strip[i][1]) * (strip[j][1] - strip[i][1]) > d:
                    break
                d = min(d, dist2(strip[i], strip[j]))

        return d, merged_y

    ans, _ = closest(pts)
    print(ans)

if __name__ == "__main__":
    solve()
```代码首先将数组转换为前缀和点，这是使问题几何化的结构转变。 递归函数保持了始终返回按 y 坐标排序的点的不变性，这对于高效的条带处理至关重要。 

带状结构使用平方距离与当前最佳值进行比较，完全避免浮点运算。 内循环中的早期中断依赖于 y 排序顺序，确保我们只比较附近的候选者。 

一个微妙的点是递归返回最佳距离和 y 排序列表。 如果不保留跨递归级别的 y 顺序，合并步骤将变得昂贵并破坏 O(N log N) 保证。

 ## 边缘情况

 单元素数组是最简单的情况。 该算法将前缀点 (0,0) 和 (A1,B1) 视为唯一候选点，并正确评估它们的平方距离。 

当所有值都为零时，所有前缀点都会折叠到原点。 在整个递归过程中，最近的对距离保持为零，并且条带永远不会产生无效比较，因为所有距离都为零。 

在交替符号数组中，多个前缀点重合或变得非常接近。 条带逻辑仍然可以正确运行，因为比较是纯粹的几何比较，并且不假设坐标的唯一性。
