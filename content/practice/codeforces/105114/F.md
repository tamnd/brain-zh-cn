 ## 解决方案
 ## 问题理解

 我们得到一个字符串$S$长度$N$，我们想从中提取一个特殊的“生成器”字符串。 该生成器称为键，被定义为如果我们重复放置它的副本，则可以重现整个原始字符串的最短子字符串，从而允许连续副本之间的重叠。 

思考该过程的一个有用方法是，我们从一个短模式开始，然后将其标记到一条线上，但只要字符与重叠的位置匹配，我们就可以向左或向右移动每个新标记。 目标是找到仍然具有足够内部结构的最小模式，以通过这种重叠的重复过程重建完整的字符串。 

约束条件$N \le 10^7$立即排除任何二次或偶数$O(N \log N)$重复尝试候选子串并通过模拟验证它们的方法。 任何解决方案在时间和内存方面都必须在字符串长度上是线性或接近线性的，并且还必须避免沉重的每个字符开销，例如嵌套比较或重复的子字符串提取。 

当弦具有强周期性结构但有位移时，会出现微妙的边缘情况。 例如，像这样的字符串`aaaaa`是微不足道的，因为任何单个角色都可以工作，但是类似`ababab`尽管重叠可能暗示多种解释，但其行为仍然像一个简单的重复结构。 另一个重要的情况是当字符串有一个不是完整句点的小边框结构时，例如`ababa`，其中仅使用前缀后缀相等的朴素周期性推理会误判最小生成器。 

核心困难在于重叠比纯粹的周期性平铺具有更大的灵活性，因此如果没有仔细的论证，标准的“弦的最小周期”是不够的。 

## 方法

 暴力策略会尝试所有可能的子串长度$L$， 提炼$S[0:L]$，并尝试通过贪婪地放置该子字符串的副本并检查所有字符是否可以通过重叠来匹配来重建整个字符串。 对于每位候选人$L$，这个模拟可以降级为$O(N)$，并且由于有$O(N)$候选人，总复杂度变成$O(N^2)$，这对于$10^7$。 

关键的见解是基于重叠的重建实际上并没有引入周期性结构之外的新表达能力。 如果一个子串可以在重叠的情况下生成完整的串，那么该串必须与该子串的重复对齐一致，这意味着对完整串的周期性约束。 这将问题简化为查找行为类似于字符串有效周期的最小前缀。 

这直接与 Knuth-Morris-Pratt 算法的前缀函数风格推理相关。 最长的固有前缀也是后缀，编码最大的内部重复结构，并从中我们得出最小的周期候选。 一旦我们计算了前缀函数，我们就可以确定最小周期并检查字符串是否完全由该周期的重复组成。 这个周期的长度就是答案。 

除了这种周期性分解之外，重叠解释不需要额外的机制，因为任何有效的重叠构造都意味着在每个位置以密钥长度为模的一致字符对齐。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^2)$|$O(N)$| 太慢了 |
 | 前缀函数 (KMP) |$O(N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们使用前缀函数$\pi[i]$，它存储最长的专有前缀的长度$S[0..i]$这也是该子字符串的后缀。 

1. 从左到右计算整个字符串的前缀函数。 在每个位置，我们使用先前计算的值维护最长边界，确保我们重用结构而不是重新计算比较。 
2. 计算后$\pi[N-1]$，我们知道整个字符串的最长的正确前缀，它也是一个后缀。 设该值为$k$。 
3. 最小重复单元候选为$p = N - k$。 这来自标准事实，即如果字符串的边框长度为$k$，那么它的周期就会减少重叠量。 
4、检查字符串是否与这个周期长度完全一致$p$。 实际上，这已经由前缀函数结构保证了，所以$p$对于重叠构造下的分解有效。 
5. 返回$p$作为答案。 

步骤 4 不需要显式验证的原因是前缀函数对一致性所需的所有前缀后缀约束进行了编码。 周期结构中的任何矛盾都会提前破坏边界计算。 

### 为什么它有效

 前缀函数将字符串划分为最大自重叠结构。 如果存在比以下更短的密钥$N - \pi[N-1]$，那么它会产生比$\pi[N-1]$，与极大值相矛盾。 反之，长度边界的存在$\pi[N-1]$确保重复长度的块$N - \pi[N-1]$在重叠允许的移位下将字符串与其自身对齐。 这使得派生周期最小且足够。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    s = input().strip()

    pi = [0] * n
    j = 0

    for i in range(1, n):
        while j > 0 and s[i] != s[j]:
            j = pi[j - 1]
        if s[i] == s[j]:
            j += 1
            pi[i] = j

    period = n - pi[-1]
    sys.stdout.write(str(period))

if __name__ == "__main__":
    solve()
```解决方案完全由前缀函数数组驱动`pi`。 循环维护一个指针`j`当我们向前扫描时，它会跟踪当前最长的匹配前缀。 当发生不匹配时，我们会使用先前计算的前缀长度，这避免了从头开始重新检查字符。 

最终值`pi[-1]`捕获整个字符串的最长边界，并将其从中减去`n`产生最小重复单元大小。 该实现小心地避免了切片或子字符串操作，这对于$N = 10^7$。

 ## 边缘情况

 对于像这样的单字符字符串`a`，前缀函数全为零，所以`pi[-1] = 0`期间变为`1`。 该算法正确返回 1，与唯一可能的键是字符串本身的事实相匹配。 

对于完全统一的字符串，例如`aaaaaa`，每个位置都会延伸边界，产生`pi[-1] = 5`长度为 6，导致周期 1。即使在基于重叠的重建下，算法也发现单个字符就足够了。 

对于没有内部结构的字符串，例如`abcde`，前缀函数永远不会扩展，所以`pi[-1] = 0`答案就变成了`N`。 这符合这样的直觉：没有更短的重叠生成器可以无矛盾地重建不同的字符。
