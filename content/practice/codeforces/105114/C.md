 ## 解决方案
 ## 问题理解

 我们得到一个以原点为中心的圆形蛋糕，每个学生都精确地进行一次直线切割。 每个切口都是一条线段，其端点位于圆的圆周上，因此每个切口的行为就像一个弦。 绘制完所有弦后，圆被划分为多个相连的区域，任务是计算存在多少个这样的区域。 

这里的关键对象不是线段本身，而是它们在圆内如何相互作用。 单个和弦将一个现有片段一分为二。 然而，当添加多个和弦时，圆圈内的和弦之间的交叉点会产生超出简单的“每次切割+1”效果的额外分割。 输出是绘制所有弦后圆盘内形成的平面区域的最终数量。 

对切​​割次数的限制（N 最多为 30）立即表明成对交互方法是可行的。 任何检查所有线段对或显式构建所有交叉点的算法都足够快。 这也暗示该结构从根本上来说是在相交方面而不是几何面积计算方面的组合。 

一种天真的但常见的失败模式是假设每个新和弦总是将乐曲数量恰好增加一个。 一旦和弦相交，这种情况就会中断。 

例如，考虑两个交叉的直径：

 输入：```
2
10000 0 -10000 0
0 10000 0 -10000
```如果我们错误地假设“每次切割添加一个区域”，我们将输出 3。正确答案是 4，因为第二个和弦与第一个和弦相交并创建四个区域。 

另一个微妙的问题是重复计算交叉点或错误分类端点。 端点位于边界圆上，不应被视为有助于区域分裂的交点。 只有内部交叉点才重要。 

## 方法

 如果我们纯粹局部思考，我们可以模拟一一添加和弦。 当插入一个新的和弦时，它会分割它所经过的每个区域。 这很难直接跟踪，因为区域是动态的并且取决于之前的交叉点。 

强力几何模拟将尝试明确地维持平面细分。 我们将把每个线段插入平面图中，在交叉点处分割边，并维护面。 虽然概念上是正确的，但考虑到 N 最多为 30，这很快就会变得不必要的矫枉过正。 

关键的简化是反转视角。 我们不跟踪区域，而是跟踪和弦如何彼此相交。 圆内的每个交点就像平面图中的一个顶点。 整个配置成为嵌入磁盘中的平面图：

 弦是边，交点是顶点，圆边界构成一个外表面。 

一旦我们这样看待问题，我们就可以使用平面图的欧拉公式。 如果我们知道嵌入图的顶点数 V、边数 E 和连通分量 C，则面数为：

 F = E − V + C + 1

 这里：

 - 每个弦最初都是一条边。 
- 每个交点都会分割边缘，因此我们必须在交点处细分弦。 
- 每个交点都会增加 V。 
- 连接性取决于和弦如何通过交叉点链接。 

我们可以增量地构造排列：检测弦对之间的所有交点，计算交点，并将每个弦视为在交点之间被分成多个段。 然后我们可以构建一个图表并计算组件数量。 

由于 N 很小，因此可以使用更简单的等效方法：计算所有交点，对它们进行计数，然后使用已知的组合恒等式来进行圆中的线段排列。 每个弦最初贡献 1 个面，每个交点都会使面数增加 1。这是有效的，因为每个交点恰好分割一个现有区域。 

因此问题就简化为：

 从 1 个片段开始，然后为每个和弦添加 1 个片段，并为在圆内相交的每对和弦添加 1 个额外片段。 

所以：

 F = 1 + N +（内部交叉点的数量）

 我们只需要计算每对弦是否严格在圆内相交。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力平面细分| O(N² log N) 或更多 | O(N²) | 太复杂|
 | 成对交集计数 | O(N²) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将每次切割建模为圆上两点之间的一段。 

1. 读取由其端点定义的所有段。 

每个片段代表一个和弦。 
2. 将答案初始化为1。 

这对应于初始未切割的磁盘。 
3. 在答案中添加 N。 

每个和弦单独将一个现有区域精确地分成两部分，因为和弦总是分割一个简单连接的区域。 
4. 对于每对弦，检查它们是否严格相交在圆内。 

端点位于边界上，因此端点处的交点不得计算在内。 
5. 对于每个有效的交叉点，将答案加 1。 

每次内部交叉都会使区域数量恰好增加一个。 
6. 输出最终计数。 

为了检测两个线段之间的交叉点，我们使用方向测试。 两条线段 (a, b) 和 (c, d) 相交当且仅当：

 orient(a, b, c) 和 orient(a, b, d) 具有相反的符号，并且

 orient(c, d, a) 和 orient(c, d, b) 具有相反的符号。

我们还明确排除在端点处发生交叉的情况。 

### 为什么它有效

 不变量是，处理完k个弦后，圆内的平面被这些弦完全分割，每个面对应于当前排列中的一个区域。 添加新弦会使面数增加 1 加上它在其内部相交的现有弦数。 每个这样的交点恰好将一个现有面一分为二。 由于相交是独立事件，而弦是直线段，因此除了成对相交之外，不会发生更高阶的相互作用。 这确保了总面数仅取决于 N 和成对交叉点的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def orient(ax, ay, bx, by, cx, cy):
    return (bx - ax) * (cy - ay) - (by - ay) * (cx - ax)

def on_segment(ax, ay, bx, by, cx, cy):
    return min(ax, bx) <= cx <= max(ax, bx) and min(ay, by) <= cy <= max(ay, by)

def segments_intersect(a, b, c, d):
    ax, ay = a
    bx, by = b
    cx, cy = c
    dx, dy = d

    o1 = orient(ax, ay, bx, by, cx, cy)
    o2 = orient(ax, ay, bx, by, dx, dy)
    o3 = orient(cx, cy, dx, dy, ax, ay)
    o4 = orient(cx, cy, dx, dy, bx, by)

    if o1 == 0 and on_segment(ax, ay, bx, by, cx, cy):
        return False
    if o2 == 0 and on_segment(ax, ay, bx, by, dx, dy):
        return False
    if o3 == 0 and on_segment(cx, cy, dx, dy, ax, ay):
        return False
    if o4 == 0 and on_segment(cx, cy, dx, dy, bx, by):
        return False

    return (o1 > 0) != (o2 > 0) and (o3 > 0) != (o4 > 0)

def solve():
    n = int(input())
    seg = []
    for _ in range(n):
        x1, y1, x2, y2 = map(int, input().split())
        seg.append(((x1, y1), (x2, y2)))

    ans = 1 + n

    for i in range(n):
        for j in range(i + 1, n):
            if segments_intersect(seg[i][0], seg[i][1], seg[j][0], seg[j][1]):
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将每个和弦编码为一对端点。 然后，它使用磁盘的基线结构加上独立和弦来初始化区域计数。 嵌套循环检查所有和弦对，这是可行的，因为 N 最多为 30。 

交叉口检查使用方向来确定正确的交叉口。 需要特别注意的是排除端点相交，因为它们不会在磁盘内创建额外的面。

 ## 边缘情况

 一种微妙的情况是两个和弦共享一个端点。 例如，如果一个和弦从 A 到 B，另一个和弦从 A 到 C，它们会在边界上的 A 处相交。 这不应算作内部交叉路口。 该算法通过排除段端点重叠来明确拒绝此类情况。 

另一种情况是三个或更多弦相交于同一内点。 即使多个弦在一个点处交叉，这仍然代表排列中的一个顶点，在基线之外恰好贡献了一个额外区域。 成对计数仍然有效，因为每一对都有正确的贡献，并且所有此类交叉点在几何上重合，从而保持面计数公式的一致性。
