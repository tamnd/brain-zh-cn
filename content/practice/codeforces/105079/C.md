 ## 解决方案
 ## 问题理解

 我们正在使用一个以原点为中心的圆形“基础”区域，以及放置在其顶部的几个附加圆形区域。 任务是计算有多少个格点（即具有两个坐标整数的点）位于基圆内并且也被至少一个磨砂圆覆盖。 

每个糖霜圆都由其半径和中心来描述。 仅当格点位于纸杯蛋糕边界内且同时位于至少一个糖霜圆内时，该格点才对答案有贡献。 输出是一个整数：此类整数坐标点的总数。 

约束很小：圆的数量和所有坐标大小都最多为 50。这立即排除了对复杂几何数据结构的任何需要。 对所有相关整数点进行直接枚举是可行的。 唯一有意义的性能考虑因素是半径为 50 的圆内存在多少个整数点，大约为几千个。 

这里的一个天真的错误是迭代大小为 101 × 101 的方形边界框，而忘记首先按基圆进行过滤。 这仍然是正确的，但一个更微妙的错误是检查浮点距离的精度问题。 由于所有坐标都是整数，并且平方距离保持在安全整数范围内，因此一切都应该使用整数算术来完成。 

边缘情况主要是几何边界条件。 

一种极端情况是磨砂圆完全位于基圆之外。 例如，如果磨砂中心位于 (100, 100)，则它没有任何贡献，并且算法不得由于不正确的边界框处理而意外地包含点。 

另一种情况是完全覆盖：完全包含基圆的磨砂圆。 在这种情况下，答案应该是基圆本身的整数点数，并且即使多个圆重叠，正确的解决方案也必须避免重复计算。 

最后一个微妙的情况是恰好位于边界上的点。 对于任何磨砂圆，如果 x² + y² ≤ R² 或 x² + y² ≤ r_i²，则应包含点 (x, y)。 平等条件很重要。 

## 方法

 蛮力的想法很简单：迭代每个可能位于基圆内的整数点，检查它是否在基圆内，然后检查它是否位于至少一个磨砂圆内，并计算它。 

由于基圆的半径最大为 50，因此边界正方形最多为 101 x 101，因此大约有 10,000 个候选点。 对于每个点，检查所有 N 个圆圈的成本为 O(N)。 这会产生大约 10,000 × 50 = 500,000 次距离检查，这在 Python 中已经可以接受。 

然而，我们可以更简洁地构造它：我们不扫描正方形，而是只扫描基圆内的点。 这进一步减少了迭代并简化了逻辑。 

不需要更高级的几何形状，因为约束故意很小。 关键的观察结果是答案完全由整数网格成员资格测试定义，因此暴力枚举是最佳的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力网格扫描| O(R²·N) | O(1) | O(1) | 已接受 |
 | 优化圆形扫描 | O(R² + N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们直接枚举基圆内的所有整数点并测试磨砂覆盖率。

1. 将 x 从 -R 迭代到 R。我们只考虑这些值，因为该范围之外的任何点都不能满足 x² + y² ≤ R²。 
2. 对于每个 x，计算使该点保持在基圆内的有效 y 值的最大垂直范围。 这里我们并不严格需要优化，但我们仍然直接检查条件。 
3. 对于 [-R, R] 中的每个 y，使用 x² + y² ≤ R² 测试 (x, y) 是否位于基圆内。 
4. 如果没有，请立即跳过，因为它无法做出贡献。 
5. 如果在基圆内，请检查是否被至少一个磨砂圈覆盖。 
6. 对于每个磨砂圆 i，检查是否 (x - x_i)² + (y - y_i)² ≤ r_i²。 
7. 如果任何磨砂圆圈覆盖了该点，则增加答案。 

### 为什么它有效

 每个可以贡献的整数点必须位于基圆定义的有限集中。 该算法精确地枚举了该集合。 对于每个这样的点，它检查它是否满足磨砂圆的存在条件。 由于直接检查圆并集中的成员资格，因此不会遗漏任何点，也不会计算无效点。 逻辑结构是离散点上集合交集和并集定义的精确实现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    n = int(data[0])
    R = int(data[1])
    arr = list(map(int, data[2:]))

    circles = []
    idx = 0
    for _ in range(n):
        r = arr[idx]
        x = arr[idx + 1]
        y = arr[idx + 2]
        idx += 3
        circles.append((r, x, y))

    ans = 0

    for x in range(-R, R + 1):
        for y in range(-R, R + 1):
            if x * x + y * y > R * R:
                continue

            covered = False
            for r, cx, cy in circles:
                dx = x - cx
                dy = y - cy
                if dx * dx + dy * dy <= r * r:
                    covered = True
                    break

            if covered:
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先将所有圆数据读入元组列表。 每个元组都存储半径和中心坐标，以便在检查期间快速访问。 

嵌套循环遍历基圆的边界正方形。 条件`x * x + y * y > R * R`使用整数运算强制基圆约束，完全避免浮点运算。 

对于每个有效的格点，我们扫描所有磨砂圆。 早期的休息很重要，因为一旦一个圆圈覆盖了该点，就不需要进一步的检查。

 ## 边缘情况

 一个关键的边缘情况是没有磨砂圆覆盖基圆中的任何点。 该算法仍然枚举所有点，但从不增加计数器，因为每次覆盖检查都会失败，根据需要生成零。 

另一种情况是糖霜圈严重重叠。 该算法在第一个覆盖圆上很早就中断，因此重叠区域不会增加计数。 

当点正好位于圆边界上时，就会出现边界情况。 由于条件使用`<=`，这些点已正确包含在内。 例如，如果 x² + y² 等于 R²，则该点仍然是底座的一部分，并且如果任何磨砂圆也包含该点，则仍可能被计数。 

最后，远离基圆的孤立磨砂圆将被安全地忽略，因为基圆检查在任何磨砂检查发生之前过滤掉所有不相关的点。
