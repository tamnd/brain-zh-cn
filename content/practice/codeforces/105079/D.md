 ## 问题理解

 我们得到了一系列纸杯蛋糕类型，每种类型都有固定的辣度值。 法官按某种顺序吃完所有纸杯蛋糕，每种类型都吃一个。 纸杯蛋糕的贡献不仅仅取决于它本身的辣度，还取决于之前已经吃掉了多少纸杯蛋糕。 

如果是一个带有辣味的纸杯蛋糕`s`被吃到位`j + 1`按顺序，其贡献为`(j + 1) * (A * s + B)`。 总分是所有纸杯蛋糕的这些贡献的总和。 任务是对纸杯蛋糕进行排列以使总数最大化。 

关键的难点在于仓位乘数`(j + 1)`线性增长，因此提前或推迟放置会改变每个纸杯蛋糕的重量。 该决定是关于对权重进行排序的项目，该权重取决于位置和转换后的值`A * s + B`。 

约束允许最多`N = 100000`，因此任何尝试所有排列的解决方案都是不可能的。 排列的阶乘数量远远超出了计算极限。 即使是二次方`O(N^2)`方法有超时的风险，具体取决于常数，因此我们正在寻找一种`O(N log N)`或者`O(N)`贪婪结构。 

一个微妙的点是`A`和`B`可以是负数。 这意味着转换后的贡献`(A * s + B)`可以是正数、负数或零。 与经典的“按值排序”调度问题相比，这显着改变了排序行为。 

打破天真的直觉的一种边缘情况是所有转换后的值都相同。 例如，如果`A = 0`，那么每个纸杯蛋糕都有贡献`(j + 1) * B`，并且顺序根本不重要。 另一个棘手的情况是当`A < 0`，颠倒了“大辣好”的意思。 

如果假设排序依据，就会出现一个小的说明性失败案例`s`总是有效。 如果`A = -1`和`B = 0`, 较大`s`值变得更加负，并且尽早放置它们可以减少它们的权重损失，因此与正值情况相比，最佳顺序是相反的。 

## 方法

 暴力解决方案将尝试纸杯蛋糕的每种排列，计算每个排序的分数，并取最大值。 这是正确的，因为它直接评估目标函数，但它需要评估`N!`排列。 即使是为了`N = 10`，这已经成为边界，并且对于`N = 100000`这是完全不可行的。 

为了找到结构，重写固定顺序的贡献`p`:

 我们对头寸求和`i`:`i * (A * s_{p[i]} + B)`分配总和：`A * sum(i * s_{p[i]}) + B * sum(i)`第二项仅取决于`N`， 自从`sum(i)`固定为`N(N+1)/2`。 这意味着`B`完全不影响订购； 它只会改变最终值。 

所以整个优化简化为最大化：`A * sum(i * s_{p[i]})`现在问题变成了一个经典的加权排序任务：根据 的符号最大化或最小化位置和值之间的点积`A`。 

如果`A > 0`，我们想要大的`s`出现在大位置，因为两者`i`和`s`积极贡献。 这符合重排不等式：以相同顺序对两个序列进行排序可以使总和最大化。 

如果`A < 0`，我们实际上希望最小化`sum(i * s_{p[i]})`，所以我们颠倒顺序：大`s`应该搭配小`i`。 

如果`A = 0`，整个第一项消失，只有常数`B * N(N+1)/2`仍然存在，因此任何排列都是最优的。 

这将问题转化为排序一次并计算单个线性通道。

 ## 算法演练

 1. 根据 的符号计算变换后的比较规则`A`。 如果`A`是正数，我们想要放置更小的`s`较早； 如果`A`是负数，我们想要更大`s`早些时候。 如果`A`为零，排序无关紧要。 
2. 根据此规则对辣度值数组进行排序。 这确保了位置和值之间的配对在重​​排不等式下是最佳的。 
3. 计算恒定贡献`B`。 由于每个位置`i`恰好出现一次，所有位置权重之和为`N(N+1)/2`，所以总贡献来自`B`是固定的。 
4. 通过迭代排序数组来计算主项。 维持运行位置指数`i + 1`并积累`(i + 1) * A * s[i]`。 
5. 添加常数`B * N(N+1)/2`累加结果并输出。 

### 为什么它有效

 目标干净地分解为应用于排列的位置相关线性函数。 扩展后，唯一受排序影响的术语是`sum(i * s_{p[i]})`。 重排不等式通过在系数为正时以与另一个序列相同的顺序对一个序列进行排序、在为负时以相反的顺序对一个序列进行排序来保证该总和最大化。 自从`B`贡献一个独立于排列的固定常数，它不影响最优排列。 这种不变性确保任何与排序结构的偏差只会使加权和恶化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, A, B = map(int, input().split())
    s = list(map(int, input().split()))
    
    if A == 0:
        # all permutations equal
        return B * N * (N + 1) // 2

    s.sort()

    if A < 0:
        s.reverse()

    res = 0
    for i, val in enumerate(s, 1):
        res += i * (A * val)

    res += B * N * (N + 1) // 2
    return res

if __name__ == "__main__":
    print(solve())
```该代码首先读取输入并处理退化情况，其中`A = 0`，从那时起排序就不起作用了，答案就变成了一个常数算术和。 

列表默认按升序排序，当`A`为负数以确保位置索引和转换值之间的正确配对。 这直接实现了由重排不等式决定的最佳对齐。 

该循环计算的贡献`A`仅项，将每个值乘以其从 1 开始的索引。 这`B`使用第一个的封闭形式总和在末尾添加一次术语`N`整数，避免对排序的任何依赖。
