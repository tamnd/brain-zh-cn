 ## 解决方案
 ## 问题理解

 我们有一个大小为的正方形网格$N \times N$，其中每个单元格要么被阻止，要么可用。 被阻止的单元格标记为$-1$，且无法输入。 每个其他单元格都包含一个非负数，表示该单元格中可用的纸杯蛋糕。 爱丽丝从左上角开始$(1,1)$并且必须到达右下角$(N,N)$，每一步仅向右或向下移动。 每当她进入牢房时，她就会收集里面所有的纸杯蛋糕。 

任务是计算爱丽丝从开始到结束沿着任何有效路径可以收集的最大纸杯蛋糕总数，或者确定不存在这样的路径。 

网格尺寸约束$N \leq 1000$意味着最多一百万个细胞。 任何试图枚举所有路径的解决方案都立即不可行，因为单调路径的数量呈指数增长，大致约为$\binom{2N}{N}$。 即使对于中等水平的人来说，这也会很快变得天文数字般大。$N$，因此暴力路径枚举不是一个选项。 

相反，我们需要一种方法来处理每个单元恒定的次数，理想情况下$O(N^2)$。 

当由于单元格阻塞而无法到达起点或终点时，就会出现微妙的边缘情况。 另一种情况是，某个单元格在几何形状上是可到达的，但通向该单元格的所有路径都被阻塞。 例如：

 输入：```
3
0 1 1
-1 -1 1
1 1 1
```尽管目的地存在并且包含纸杯蛋糕，但中间的障碍物会断开网格，使其无法到达右下角的单元格。 正确答案是$-1$，任何忽略可达性的幼稚 DP 都会错误地将值累积到不可达状态。 

## 方法

 暴力策略会尝试探索从左上角到右下角的每一条有效路径，并沿途累积纸杯蛋糕的数量。 由于移动仅限于向右和向下，因此每条路径都是精确的序列$2N-2$移动，并且此类序列的数量呈指数级增长$N$。 即使我们修剪路径$-1$细胞，最坏情况下没有障碍的网格迫使探索所有单调路径，这是太大了。 

关键的观察结果是，在任何单元格收集的值仅取决于到达该单元格的最佳方式。 一旦我们到达牢房$(i,j)$，除了可能的最佳累积值外，截至该点的最佳得分与我们如何到达该点无关。 这是一个经典的最优子结构情况。 

因此，我们不是探索路径，而是计算一个动态规划表，其中$dp[i][j]$代表到达细胞时可获得的最大纸杯蛋糕$(i,j)$。 每个单元仅依赖于其顶部和左侧的邻居，因为这是进入它的唯一方法。 如果某个单元被阻塞，它就不会做出任何贡献，并且被标记为无法访问。 

这将问题从指数路径枚举简化为每个单元的工作量恒定的简单网格遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^{2N})$|$O(N)$递归栈| 太慢了 |
 | 动态规划|$O(N^2)$|$O(N^2)$| 已接受 |

 ## 算法演练

 我们定义了一个 DP 表，其中每个状态都存储到达该单元格时可实现的最佳纸杯蛋糕总数。 我们还需要一种方法来表示无法到达的状态。 

1. 初始化一个2D数组$dp$尺寸的$N \times N$，填充代表不可达的哨兵值，如$-\infty$。 这确保我们永远不会意外地将无效路径视为贡献最大。 
2.设置起始单元格$dp[0][0]$。 如果它被阻挡，整个问题就立即不可能解决，因为爱丽丝根本无法移动。 否则将其初始化为起始单元格的值，该值保证为零。 
3. 逐行迭代网格。 对于每个单元格$(i,j)$，如果被阻止，则离开$dp[i][j]$一样无法到达。 这保留了正确性，因为没有路径可以合法地进入它。 
4. 如果单元格未被阻塞，计算到达该单元格的最佳可能方式。 爱丽丝只能从上面来$(i-1,j)$或从左边$(i,j-1)$，因此我们取这两个 dp 值中的最大值，前提是它们是可达的。 当前单元格值将添加到该最大值。 如果两个邻居都无法访问，则该单元仍然无法访问。 
5.填表后答案为$dp[N-1][N-1]$。 如果仍然无法访问，则输出$-1$，否则输出存储的值。 

### 为什么它有效

 在每个细胞$(i,j)$，DP值表示在所有有效路径中可实现的最大纸杯蛋糕$(0,0)$到$(i,j)$。 任何这样的路径必须以来自以下任意一个的位置结束：$(i-1,j)$或者$(i,j-1)$，并且这两个状态都已经存储了各自前缀的最佳值。 因为转换考虑了两者并取最大值，所以不会错过更好的前缀路径。 被阻塞的单元格可以正确地中断转换，因为它们消除了经过它们的所有路径，从而确保无法到达的区域仍被排除在未来的计算之外。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    grid = [list(map(int, input().split())) for _ in range(n)]
    
    NEG = -10**18
    dp = [[NEG] * n for _ in range(n)]
    
    if grid[0][0] == -1 or grid[n-1][n-1] == -1:
        print(-1)
        return
    
    dp[0][0] = 0
    
    for i in range(n):
        for j in range(n):
            if grid[i][j] == -1:
                continue
            
            if i == 0 and j == 0:
                continue
            
            best = NEG
            if i > 0:
                best = max(best, dp[i-1][j])
            if j > 0:
                best = max(best, dp[i][j-1])
            
            if best == NEG:
                continue
            
            dp[i][j] = best + grid[i][j]
    
    print(dp[n-1][n-1] if dp[n-1][n-1] != NEG else -1)

if __name__ == "__main__":
    solve()
```该实现直接镜像 DP 定义。 哨兵值$NEG$防止将无法到达的状态与有效的总和混合在一起。 起始单元单独初始化，以避免错误地将其值添加两次或依赖于不存在的邻居。 每个转换都会仔细检查边界并完全跳过阻塞的单元格。 

一个常见的错误是将无法到达的单元格初始化为零，这错误地允许路径“穿过墙壁”。 另一个问题是未能检查父母双方，这将错过仅从一个方向接近的有效路径。

 ## 边缘情况

 一个关键的边缘情况是开始或结束被阻止。 该算法在 DP 开始之前明确检查这些。 如果其中一个是$-1$，函数立即返回$-1$，因为无论中间结构如何，都不可能存在路径。 例如：```
2
-1 0
0 0
```在这里，即使目的地在网格方面是可达的，但开始是无效的，因此 DP 初始化永远不会发生，正确的输出是$-1$。 

另一种边缘情况是孤立的单元格，它们在几何上可到达，但在动态上不可到达。 考虑：```
3
0 1 1
-1 -1 1
1 1 1
```DP 正确地将整个右下区域标记为不可到达，因为关键细胞的两个潜在父代都被阻止。 这表明可达性不仅仅与坐标有关，还与有效前驱链的存在有关，DP 不变量会自动强制执行该链。
