 ## 解决方案
 ## 问题理解

 我们有一个最低门槛$n$，我们要选择一个数字$x$代表纸杯蛋糕上圆点的数量。 这个数字必须同时满足两个约束。 第一的，$x \ge n$。 第二，$x$一定不能是质数。 在所有这些有效值中，我们必须返回尽可能小的值。 

因此，该任务本质上是一个“下一个有效号码”查询，从$n$，其中有效性表示“合数或 1”，因为 1 不是素数，并且根据“非素数”规则也可以接受。 

约束条件$2 \le n \le 1000$立即表明我们处于一个很小的搜索空间中。 即使是天真的向上扫描$n$最坏的情况是 1000 个检查，可以忽略不计。 每个检查中唯一真正的计算是素性测试。 

唯一微妙的边缘情况来自如何围绕小整数定义素数。 数字 2 是素数，所以如果$n = 2$，我们必须跳过它并继续前进。 数字3也是素数，所以可能会发生连续跳数。 另一个微妙的情况是答案可以等于$n$本身如果$n$已经是非素数了，例如$n = 1000$。 一种粗心的方法，假设我们必须始终增加$n$是不正确的。 

## 方法

 直接的方法是从$x = n$并反复检查是否$x$是素数。 如果它是素数，我们就递增$x$然后再试一次。 返回第一个未通过素性测试的值。 

这是有效的，因为搜索空间是单调的：一旦一个值无效（素数），我们只需要向前移动，并且遇到的第一个有效值必然是最佳的，因为我们按升序扫描。 

瓶颈是素性测试。 一个朴素的素性检查运行$O(\sqrt{x})$，在最坏的情况下我们可能会执行大约1000次。 这大致给出了$1000 \cdot 31$操作，这是微不足道的。 即使实施速度稍慢，它仍然在限制范围内。 

稍微更结构化的优化是使用筛子预先计算最多 1000 个素数，然后简单地向上扫描并检查布尔数组。 这将每次检查减少到$O(1)$，使解决方案在最坏的情况下呈纯线性。 

两种方法都是可以接受的； 如果我们期望多个查询，则筛子会更干净，而直接素性检查对于单个输入来说是最简单的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（每一步检查素数）|$O((m-n)\sqrt{m})$|$O(1)$| 已接受 |
 | 筛分+扫描|$O(M \log \log M + (m-n))$|$O(M)$| 已接受 |

 这里$M = 1000$。 

## 算法演练

 我们描述基于筛子的方法，因为它使最终扫描变得微不足道。 

### 步骤

 1. 构建布尔数组`is_prime`对于从 2 到 1000 的所有索引，大小为 1001 的值初始化为 true。 

我们立即将 0 和 1 视为非素数，因为它们不满足素数定义。 
2. 从 2 到 1000 进行埃拉托斯特尼筛。 

对于每个数字$i$仍标记为素数，标记所有倍数$2i, 3i, \dots$作为非素数。 

这一步是正确的，因为任何合数都必须有一个最小的质因数，并且它将通过该因数的迭代而被消除。 
3. 从$x = n$并向上移动。 
4. 对于每个$x$，检查是否`is_prime[x]`是假的。 

如果为假，立即输出$x$。 

否则递增$x$并继续。 

遇到的第一个非质数值是最小的有效答案，因为我们按升序扫描而不跳过任何候选值。 

### 为什么它有效

 筛子正确地将数字划分为素数和非素数，最多 1000 个。预处理后，素数查询变成恒定时间查找。 扫描自$n$向上保留顺序，因此通过构造遇到的第一个非质数是最小的。 不可能错过更小的有效答案，因为小于输出的每个候选都已被检查并被拒绝为素数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

N = 1000

is_prime = [True] * (N + 1)
is_prime[0] = is_prime[1] = False

p = 2
while p * p <= N:
    if is_prime[p]:
        for m in range(p * p, N + 1, p):
            is_prime[m] = False
    p += 1

n = int(input().strip())

x = n
while True:
    if not is_prime[x]:
        print(x)
        break
    x += 1
```筛子在最大 1000 的固定范围内构建一次。内循环从$p^2$因为较小的倍数已经被较小的素数标记，从而避免了多余的工作。 

扫描循环故意简单：它只向上前进，直到找到合数或 1。条件`not is_prime[x]`正确地将 1 视为非素数，并且还处理所有合数。 

## 工作示例

 ### 示例 1：输入 5

 我们预先计算素数至少为 5。 

| x| is_prime[x] | is_prime[x] | 行动|
 | ---| ---| ---|
 | 5 | 真实| 跳过|
 | 6 | 假 | 停止并输出|

 该算法首先检查 5，发现它是素数，然后移动到 6（它是合数）并返回它。 这表明答案不需要严格大于$n$，仅非素数并且至少$n$。 

### 示例 2：输入 2

 | x| is_prime[x] | is_prime[x] | 行动|
 | ---| ---| ---|
 | 2 | 真实| 跳过|
 | 3 | 真实| 跳过|
 | 4 | 假 | 输出|

 扫描正确地跳过连续的素数。 输出变为4，这是大于或等于2的最小合数。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    N = 1000
    is_prime = [True] * (N + 1)
    is_prime[0] = is_prime[1] = False

    p = 2
    while p * p <= N:
        if is_prime[p]:
            for m in range(p * p, N + 1, p):
                is_prime[m] = False
        p += 1

    n = int(sys.stdin.readline().strip())
    x = n
    while True:
        if not is_prime[x]:
            return str(x)
        x += 1

 ## 边缘情况

 对于像 2 和 3 这样的小素数，扫描必须正确跳过多个连续值。 对于输入 2，算法检查 2 和 3 是否为素数，然后在 4 处停止。这证实了重复增量不会错过有效候选者。 

对于已经复合的输入，例如 4 或 1000，第一次检查会立即成功，因为`is_prime[x]`是假的。 当答案等于输入本身时，循环不会进一步增加，从而确认正确性。
