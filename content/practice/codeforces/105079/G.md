 ## 解决方案
 ## 问题理解

 我们得到一排纸杯蛋糕，每个纸杯蛋糕都撒有一些糖粉。 随着时间的推移，Alice 执行一系列操作。 在每次操作中，她都会将每个纸杯蛋糕的糖粉增加固定数量，并按固定顺序从数组中取出。 每次这样的全局增加之后，鲍勃都会立即进行干预：他查看当前糖粉最多的纸杯蛋糕，去掉一半的糖粉（向下舍入），其余的保持不变。 

每个添加的层都会重复该过程一次。 在应用所有层并且鲍勃的所有干预都发生后，我们被要求提供所有纸杯蛋糕上剩余的糖粉总数。 

关键的困难在于，这两种操作都是全球性的，但方式不同。 Alice 的更新统一涉及所有元素，而 Bob 的操作取决于随时间以非平凡方式变化的全局最大值。 

这些约束足够大，任何在大型结构上以简单的方式模拟每个步骤的方法都可能会失败。 对于多达 50,000 个纸杯蛋糕和 50,000 次操作，我们重复扫描所有纸杯蛋糕以在每一步后找到最大值的直接模拟将花费大约 25 亿次比较，这在 Python 中太慢了。 

第二个微妙的问题是排序。 鲍勃总是在爱丽丝完全添加该步骤之后采取行动。 如果我们错误地交错加法和减半或忘记顺序，即使输入很小，我们也会得到不正确的结果。 

当多个纸杯蛋糕共享最大值时，就会出现最终的边缘情况。 Bob 只影响其中之一，但选择任意最大值都可以。 然而，如果我们无法维持一致的数据结构，我们可能会意外地选择过时的最大值或错过之前减半后的更新。 

## 方法

 直接模拟将当前喷洒计数保存在数组中。 对于每一层，我们首先将层值添加到每个纸杯蛋糕上，然后扫描整个数组以找到最大值，然后将其减半。 

这是正确的，因为它准确地反映了该过程。 问题是成本。 每个步骤需要 O(N) 进行更新，加上 O(N) 来查找最大值，重复 N 次。 这给出了 O(N^2)，上限达到 25 亿次操作，不可行。 

关键的观察结果是，我们实际上并不需要重复地完全了解所有纸杯蛋糕。 每次全局添加后，每个纸杯蛋糕都会增加相同的常数。 这意味着它们按大小的相对排序不会因为 Alice 的操作而单独改变。 只有 Bob 的减半操作改变了相对顺序，并且只修改了单个元素。 

这种结构建议将纸杯蛋糕维护在一个数据结构中，该数据结构始终可以有效地访问当前的最大值，同时支持对单个元素的更新。 如果我们显式存储当前值并延迟更新或直接调整受影响的元素并重新插入它，则最大堆就足够了。 

我们还利用了这样一个事实：Alice 的全局加法可以被视为运行偏移。 我们维护一个全局增量变量，而不是每次更新所有值。 每个纸杯蛋糕的实际值是其存储的基值加上此偏移量。 这将全局加法转换为 O(1)。 

现在每个步骤变成：应用偏移增量，提取最大值（考虑偏移），将其减少一半，然后重新插入更新的值。 这将每个操作减少到 O(log N)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N^2) | O(N^2) | O(N) | 太慢了 |
 | 堆+惰性偏移| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们维护纸杯蛋糕的最大堆和表示爱丽丝添加的总数量的全局偏移量。 

每个堆条目存储“基值”，即实际值减去当前偏移量。

1. 使用所有初始纸杯蛋糕值减去零偏移量来初始化堆。 

这种表示确保我们可以通过添加偏移量随时重建真实值。 
2. 初始化变量`offset = 0`。 
3.对于每层值`b[i]`，将其添加到`offset`。 

这代表爱丽丝将每个纸杯蛋糕增加了`b[i]`无需触及单个元素。 
4. 从堆中提取最大元素。 

由于堆存储基值，因此这对应于具有最高实际值的纸杯蛋糕。 
5. 通过添加将该基值转换为实际值`offset`。 
6.应用Bob的操作：将该值替换为`floor(value / 2)`。 
7. 通过减去将更新后的值转换回基本形式`offset`。 
8. 将修改后的基值推回到堆中。 

处理完所有层后，每个纸杯蛋糕的实际值是其存储的基础值加上最终的偏移量。 对所有堆元素求和并添加`N * offset`得出最终答案。 

### 为什么它有效

 该算法依赖于维护的不变量：在任何时候，每个堆元素都表示真实的纸杯蛋糕值减去相同的全局偏移量，并且堆属性反映了真实值的排序，因为所有元素的偏移量都是相同的。 由于 Alice 的操作仅更改此共享偏移量，因此它保留了相对顺序。 Bob 的操作只影响一个元素，我们立即重新插入其更新后的形式，恢复堆结构的正确性。 这确保了每一步都在当时的真实最大值上运行。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    # max heap via negatives
    heap = []
    offset = 0

    for x in a:
        heapq.heappush(heap, -x)

    for add in b:
        offset += add

        # extract max (real value = -heap[0] + offset)
        x = -heapq.heappop(heap)
        real = x + offset

        # Bob halves it
        real //= 2

        # store back as base value
        heapq.heappush(heap, -(real - offset))

    # final sum
    total = 0
    for x in heap:
        total += (-x + offset)

    print(total)

if __name__ == "__main__":
    solve()
```该解决方案依赖于使用负数将值存储在最大堆中，这避免了实现自定义堆。 偏移量永远不会直接应用于堆，这可以防止重复的 O(N) 更新。 

当 Bob 删除最大值的一半时，我们使用偏移量临时重建实际值，应用操作，然后将调整后的基数存储回来。 

最终求和将偏移量添加回每个元素一次，避免重复重新计算。

 ## 边缘情况

 一个关键的边缘情况是在爱丽丝添加后所有纸杯蛋糕都相等。 例如：```
3
5 5 5
1 1 1
```第一次相加后，全部变成 6。Bob 选择一个并将其减半为 3。堆仍然包含一致的表示，因为只有一个元素发生变化，并且偏移量继续正确地表示共享增长。 即使存在多个有效最大值，任何一个选择都会导致有效状态，因为该问题不需要确定性选择。 

另一种边缘情况是当减半将最大值降低到低于其他元素时。 例如：```
2
1 100
0 0
```第一步之后，100 减半为 50，这可能仍然是最大值，但经过几个步骤后，它可能会低于另一个纸杯蛋糕。 堆可以正确处理此问题，因为重新插入后，排序会自动恢复，并且下一个最大查询会反映更新后的状态。
