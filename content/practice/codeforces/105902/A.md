 ## 解决方案
 ## 问题理解

 我们获得完成同一任务的三种候选路线或计划，每条路线或计划都与时间成本相关。 目标只是选择三个选项中最快的并输出其时间。

 每个输入恰好由三个整数组成。 您可以将它们视为解决同一子问题的三种竞争策略，其中每种策略都有已知的固定成本。 该任务简化为选择这三个值中最小的一个并打印它。 

约束非常小，每个值都在 1 到 60 之间。这立即告诉我们，任何解决方案，即使是不必要的繁重的解决方案，仍然会在恒定时间内运行。 无需预处理、对大型结构进行排序或重复计算。 任何甚至检查每个值一次的算法都已经是最佳的。 

不存在涉及序列或范围的棘手结构边缘情况，但此类问题中经常出现一个微妙的正确性问题：确保相等不会影响正确性。 如果两个或多个值相等且最小，则它们中的任何一个都是有效的，因此实现不得由于严格比较或假定唯一性的提前退出而错误地丢弃有效的最小候选值。 

例如，如果输入是`1 1 45`，前两个选项都是最优的。 正确的解决方案仍然必须输出`1`。 有缺陷的方法可能会错误地假设唯一的最小值并跳过相等的值，但在这里相等是正确性的一部分。 

## 方法

 直接的方法是手动比较这三个值。 我们可以通过一系列比较来检查三者中哪一个最小。 例如，我们可以首先假设第一个值是答案，然后将其与第二个值进行比较，如果需要则更新，然后再次与第三个值进行比较。 这已经足够了，因为输入大小是固定的。 

这是可行的，因为问题本质上是计算一组三个元素的最小值。 值之间没有依赖关系，不需要转换，只需选择。 

人们可能会考虑更通用的方法，例如对三个值进行排序并获取第一个元素。 这也是可行的，但是对于如此小的固定大小输入，在推理和实现方面都会带来不必要的开销。 三个元素上的循环也是有效的，但对于常量大小的结构来说又是不必要的复杂性。 

关键的观察结果是该结构是固定的且微小的。 蛮力解决方案和最优解决方案的复杂度实际上是相同的； 唯一的区别是清晰和直接。

 ## 算法演练

 1. 从输入中读取三个整数。 这些代表三个候选计划的成本。 
2. 初始化变量`ans`与第一个值。 这为我们提供了默认有效的基线候选。 
3.比较`ans`与第二个值。 如果第二个值较小，则更新`ans`到第二个值。 这确保我们始终保留迄今为止最好的选择。 
4. 比较`ans`与第三个值。 如果第三个值较小，则更新`ans`再次。 这保证了在处理完所有三个值之后，`ans`持有所有候选人中的最小值。 
5、输出`ans`。 

### 为什么它有效

 每一步，变量`ans`存储迄今为止处理的所有元素中的最小值。 第一步之后，它是第一个元素的最小值。 第二次比较后，它成为前两个元素中的最小值。 第三次比较后，它成为所有三个元素中的最小值。 这个不变量确保不会错过任何更小的值，因为每个候选值都会与当前最佳值进行显式比较。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b, c = map(int, input().split())

ans = a
if b < ans:
    ans = b
if c < ans:
    ans = c

print(ans)
```该解决方案读取一行中的三个整数并将它们存储在变量中。 变量`ans`使用第一个值进行初始化以建立有效的起点。 后续每次比较都会更新`ans`仅当找到严格较小的值时，即使值相等也能保持正确性。 

对于相等的值不需要特殊处理，因为条件使用`<`而不是`<=`，这自然会保留第一个遇到的最小值，而不影响正确性。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    a, b, c = map(int, input().split())
    ans = a
    if b < ans:
        ans = b
    if c < ans:
        ans = c
    return str(ans)

# provided samples
assert run("14 12 18") == "12", "sample 1"
assert run("1 1 45") == "1", "sample 2"

# custom cases
assert run("1 2 3") == "1", "increasing order"
assert run("3 2 1") == "1", "decreasing order"
assert run("5 5 5") == "5", "all equal"
assert run("60 1 60") == "1", "minimum in middle"
```| 测试输入| 预期产出 | 它验证了什么 |

 ## 边缘情况

 ### 所有值都相等

 输入：`7 7 7`该算法开始于`ans = 7`，然后与第二个和第三个值进行比较。 两个更新都不会触发，因为两者都不是严格意义上的较小。 最终输出仍为`7`，这是正确的，因为所有选项都是等效的。 

### 不同位置的最小值

 输入：`9 2 8`算法首先设定`ans = 9`。 然后它更新为`2`与第二个值进行比较后。 第三次比较并没有改变它。 输出`2`无论位置如何，都能正确反映全局最小值。 

### 严格降序

 输入：`10 6 1`算法逐步更新`ans`在每一步中，都证明重复的改进得到了正确的处理。 最终结果`1`确认任何早期决定都不会阻止以后的纠正。
