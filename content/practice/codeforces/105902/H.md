 ## 解决方案
 ## 问题理解

 我们得到了一系列树高。 我们必须准确执行$s$操作，每一次操作都会将一棵树的高度减少一个单位。 在所有操作之后，一些树可能仍然具有正高度，而另一些树可能会变为零或保持正高度，具体取决于我们如何分配切割。 

最终评估忽略所有高度为零的树。 对于剩余的正高度，我们计算标准统计意义上的方差：与平均值的平方偏差的平均值。 

目标是选择每一步要砍伐的树木，以便在准确之后$s$总的减量，剩余正高度的方差尽可能小。 

约束条件很大：最多$2 \cdot 10^5$跨所有测试用例的树，最多$10^9$运营。 这立即排除了任何一一模拟操作的方法。 甚至$O(s)$或者$O(s \log n)$方法是不可能的。 我们必须推理全局结构的变化，而不是逐步模拟。 

该声明的一个微妙但至关重要的结果是，每个$a_i > s$。 这保证了没有树可以减少到零。 无论我们如何分配切割，每棵树都保持正值，这意味着剩余树的集合始终是完整的集合$n$树。 这消除了更改的复杂性$k$，影响方差的元素数量。 方差始终是在所有数据上计算的$n$元素。 

一个常见的错误是认为积极砍伐可能会消除小树并减少方差。 这在这里是不可能的，因为我们没有足够的总削减来完全删除任何一棵树。 

## 方法

 第一直觉是动态地思考问题：每次操作都会减少一个元素，并且我们尝试保持数组“平衡”。 一种自然的贪婪想法是始终减少当前最大的树，因为减少大值往往比减少小树更有效地减少传播。 这是正确的，但是如果直接实现的话太慢了。 

简单的模拟将使用优先级队列：重复提取最大值，将其减一，然后将其推回。 这是正确的，因为当值尽可能相等时方差会最小化，并且当应用于最大元素时，每次减量都会最有效地减少不平衡。 然而，这需要$s$堆操作，这对于$s = 10^9$。 

关键的观察是我们永远不需要模拟单独的减量。 该过程仅取决于每个不同高度在遇到另一个高度之前减少了多少次。 我们不考虑操作，而是考虑高度的“级别”以及在与下一个级别合并之前最大级别可以降低多远。 

这将问题转化为重复压平最高的值块，直到它到达下一个不同的值，从而大量消耗操作。 当剩余预算不足以完全压平某个级别时，我们将剩余的减量尽可能均匀地分布在当前最大块上。 

一旦应用了所有归约，最终数组的平均值就固定了，因为总和正好减少了$s$，独立于分布。 这意味着最小化方差相当于最小化最终数组的平方和。

 | --- | --- | --- | --- |
 | 暴力堆模拟|$O(s \log n)$|$O(n)$| 太慢了|
 | 级别压缩（排序/堆合并）|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们想要模拟重复减少最大元素的效果，但以聚合形式。 

### 1. 对相同的值进行排序和分组

 我们首先将相同高度分组在一起。 这允许我们将数组视为一系列值块，每个值块都有一个频率。 

### 2. 维护值块的最大结构

 我们从最高值向下处理块。 在任何时刻，我们都会考虑价值最高的区块$v$并数数$c$。 

我们将其与下一个较低的值进行比较$u$。 全部携带的费用$c$元素来自$v$下降到$u$是$(v - u) \cdot c$。 

如果$s$足够大，我们将这个块完全压平为$u$，合并计数，然后继续。 

### 3. 处理部分展平

 如果$s$还不足以完全减少$v$到$u$，我们将剩余的操作分布在$c$要素：

 我们计算$s // c$，这是顶部块中每个元素可以均匀减少的数量。 然后我们计算$s \% c$，表示有多少元素获得额外的减量。 

这会产生最终的分割，其中一些元素位于高度$v - t$其余的在$v - t - 1$。 

在这一步之后，不需要进一步的全局合并，因为所有操作都已耗尽。 

### 4. 计算最终的平方和

 一旦以分组形式确定最终值，我们就计算$\sum b_i^2$。 则方差为：$$\frac{1}{n} \sum b_i^2 - \left(\frac{\sum b_i}{n}\right)^2$$总和$\sum b_i$简直就是$\sum a_i - s$，所以均值是固定的。 

### 为什么它有效

 在每一步中，我们总是对当前最大值组应用减少，因为减少较大的值比减少较小的值更能减少平方和。 减少值的边际变化$x$除 1 是$(x-1)^2 - x^2 = -2x + 1$，对于较大的情况更负$x$。 这保证了任何最优策略必须始终优先考虑最大元素，从而使贪婪的“首先压平最高层”过程达到最优。 

由于我们仅在组变得相等时才对其进行压缩，因此我们保留了数组始终表示为排序值块的不变性，并且不会跳过有益的重新分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n, s = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort(reverse=True)

        # build value blocks: (value, count)
        blocks = []
        i = 0
        while i < n:
            j = i
            while j < n and a[j] == a[i]:
                j += 1
            blocks.append([a[i], j - i])
            i = j

        # merge downward
        k = 0
        while k < len(blocks) - 1 and s > 0:
            v, c = blocks[k]
            u, c2 = blocks[k + 1]

            gap = (v - u) * c

            if s >= gap:
                blocks[k + 1][1] += c
                s -= gap
                k += 1
            else:
                dec = s // c
                rem = s % c

                new_v = v - dec
                # split c into rem and c-rem
                blocks[k] = [new_v, c - rem]
                blocks.insert(k + 1, [new_v - 1, rem])
                s = 0
                break

        # compute variance
        total_sum = sum(v * c for v, c in blocks)
        mean = total_sum / n

        var = 0.0
        for v, c in blocks:
            var += c * (v - mean) * (v - mean)
        var /= n

        out.append(f"{var:.10f}")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先将相同高度压缩为块，以便我们可以对频率而不是单个元素进行操作。 主循环反复尝试将最高的块合并到下一个块中，从而大量消耗操作。 

当操作不足以完成完全合并时，我们将剩余预算均匀地分布在当前块上，最多产生一个分裂为两个相邻值。 

最后，使用标准公式直接根据分组频率计算方差，从而避免了重建整个阵列的需要。 

一个微妙的实现细节是在部分归约期间正确处理整数除法和余数，因为这是分布变得不均匀的唯一点。 

## 工作示例

 ### 示例 1

 输入：```
5 3
1 2 3 4 5
```降序排序给出`[5,4,3,2,1]`。 

我们尝试将 5 扁平化为 4、3 个可用单位，因此我们减少了 5→2，仅部分影响顶部元素。 

| 步骤| 积木| 剩余 s | 行动|
 | --- | --- | --- | --- |
 | 开始| (5,1)(4,1)(3,1)(2,1)(1,1) | (5,1)(4,1)(3,1)(2,1)(1,1) | 3 | 从最大 | 开始
 | 部分| (2,1)(4,1)(3,1)(2,1)(1,1) | (2,1)(4,1)(3,1)(2,1)(1,1) | 0 | 5 减少 3 |

 最终数组是`{2,4,3,2,1}`。 

均值固定为$(15 - 3)/5 = 12/5$。 根据最终值计算的方差得出报告结果。 

该跟踪表明我们只影响最高的元素，因为预算太小而无法与更深层次的交互。

 ## 边缘情况

 一种边缘情况是所有值都相同。 该算法立即发现级别之间的间隙为零，因此不会发生合并，并且所有操作都均匀分布在单个块内。 由于所有元素在任何均匀减少后都保持相等，因此方差保持为零。 

当预算太小而无法达到下一个水平时，就会出现另一种边缘情况。 在这种情况下，只有顶部块受到影响，并且它最多分裂成两个相邻值。 该实现通过整数除法和余数显式处理这个问题，确保没有值减少到其逻辑下限以下。 

最后的边缘情况是在操作耗尽之前重复合并产生单个块。 在这种情况下，所有剩余操作均匀分布在所有元素上，保持接近均匀性并完全按照要求最小化方差。
