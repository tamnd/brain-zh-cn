 ## 问题理解

 我们得到了一个由左括号和右括号组成的字符串。 我们可以一个一个地改变字符，每次改变都会将括号变成相反的类型。 执行一定数量的此类更改后，我们查看结果字符串的每个可能的非空子序列。 要求是这些子序列都不应该是“平衡的”，其中平衡意味着插入后可以被解释为有效的括号表达式`+`和`1`适当地，这相当于正确括号序列的通常概念。 

任务是计算所需的单位置更改的最小数量，以便每个非空子序列作为平衡括号序列变得无效。 

跨测试用例的输入大小总共高达二十万个字符，因此任何解决方案都必须在每个测试用例的线性时间内运行。 任何二次方的事情，例如检查所有子序列或模拟每个子集的变换，都是立即不可行的，因为子序列的数量呈指数增长。 

一个微妙的问题是子序列平衡意味着什么。 最小的平衡子序列是`"()"`，并且只需要一个`'('`和一个`')'`与`'('`出现在之前`')'`在原始字符串中。 这创建了一个重要的结构约束：如果最终字符串中存在两种类型的括号，则长度为 2 的有效子序列始终是可能的。 

经常混淆朴素推理的边缘情况来自以下观察：

 如果字符串是`"()"`，那么它已经包含一个平衡子序列，因此零操作是不可接受的，因为完整的子序列集包含一个有效的子序列。 正确答案需要修改至少一个字符。 

如果字符串是`"((("`，根本无法形成平衡子序列，因为没有右括号。 在这种情况下，所有子序列都会自动不平衡。 

如果字符串是`")))"`，同样的推理对称地适用。 

这些例子表明，唯一危险的情况是同时存在两种类型的括号。 

## 方法

 暴力策略会尝试一切可能的方法来修改字符串，并且对于每个候选字符串，枚举所有子序列并检查其中是否有任何一个是平衡的。 检查单个字符串已经涉及指数数量的子序列，并且可能修改的字符串的数量也与更改的位置数量成指数关系。 即使进行了修剪，这种方法也会在输入限制下崩溃。 

关键的简化来自于认识到任何`'('`和任何`')'`立即保证有效的子序列`"()"`。 这意味着“所有非空子序列都是不平衡的”整个属性相当于最终字符串不能同时包含两种类型的括号的更简单的条件。 

因此目标配置受到极大限制：最终字符串必须是 all`'('`或全部`')'`。 任何混合物都会自动失效。 

一旦理解了这一点，问题就简化为选择我们想要的统一字符串并计算必须翻转多少个位置才能实现它。 我们尝试两种可能性并采取最小值。

 | 使字符串均匀 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 数一数有多少个`'('`和`')'`出现在字符串中。 这给出了字符串距离均匀程度的完整描述。 
2. 计算将整个字符串转换为所有字符串的成本`'('`。 这个成本正好是`')'`字符，因为每个字符都必须翻转。 
3. 计算将整个字符串转换为所有字符串的成本`')'`。 这个成本正好是`'('`人物。 
4. 返回这两个值中的最小值，因为任一目标配置都保证没有子序列可以包含两种类型的括号。 

仅选择统一字符串背后的原因是任何混合配置都不可避免地包含至少一个`"()"`后续，因为任何发生`'('`在 a 的左侧`')'`已经形成一个有效的平衡子序列。 

### 为什么它有效

 字符串一旦包含两种括号类型就无法满足要求，因为子序列由一个组成`'('`以及后来的一件事`')'`已经是平衡的了。 因此，消除所有此类子序列相当于确保其中一种括号类型完全不存在。 任何统一字符串都满足此条件，任何非统一字符串都会违反此条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n = int(input().strip())
    s = input().strip()

    open_cnt = s.count('(')
    close_cnt = n - open_cnt

    # make all '(' or all ')'
    print(min(open_cnt, close_cnt))
```该解决方案仅依赖于计算字符频率。 关键的实现细节是避免任何模拟子序列或转换的尝试。 答案完全由全局计数决定，因此一次线性扫描（甚至是 Python 的内置扫描）`count`）就足够了。 

唯一的微妙之处是确保考虑两个转换方向。 在大多数角色不同的情况下，仅选择一个方向可能是错误的。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        s = input().strip()
        open_cnt = s.count('(')
        close_cnt = n - open_cnt
        out.append(str(min(open_cnt, close_cnt)))
    return "\n".join(out)

# provided samples
assert run("2\n3\n())\n7\n)()()((\n") == "1\n3"

# single imbalance mix
assert run("1\n2\n()\n") == "1"

# already uniform
assert run("1\n5\n(((((\n") == "0"

# alternating structure
assert run("1\n6\n()()()\n") == "3"

 ## 边缘情况

 一个关键的边缘情况是当字符串在括号之间交替时，例如`"()()()"`。 尽管它的结构可能看起来很复杂，但它仍然包含两者`'('`和`')'`，所以有一个子序列`"()"`永远存在。 该算法对两种类型进行计数，并正确返回完全消除一种类型所需的翻转次数。 

另一种边缘情况是当字符串已经统一时，例如`"(((("`。 在这里，计数显示零右括号，因此答案为零。 该算法自然地处理这个问题，无需特殊的分支。 

最后，最小的非平凡输入`"()"`强调为什么结构并不重要：即使它是一个有效的完整序列，它也会立即被取消资格，因为它包含长度为 2 的平衡子序列。 计算仍然减少到一次翻转，与统一化逻辑相匹配。
