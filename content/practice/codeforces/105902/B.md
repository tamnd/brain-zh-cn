 ## 解决方案
 ## 问题理解

 我们给出了从 1 到 n 的一系列位置。 玩家从两个端点之一开始，并在时间上向前移动通过 m 个事件。 在每个事件发生之前，允许玩家传送，但传送受到限制：从时间 i−1 处的当前位置 x 开始，在时间 i 处他可以选择距离 d 内的任何位置，包括整数之间的小数位置。 选择该时间步的位置后，烟花会在固定位置 ai 处发生，玩家将获得与他们的位置和 ai 之间的距离成正比的值，特别是绝对距离的 bi 倍。 

玩家的任务是选择一个初始端点，然后选择一系列位置，每个事件一个，尊重移动约束，以便最大化所有事件的总累积奖励。 

关键结构是时间按照严格的顺序进行，每一步玩家都会做出受约束的连续选择，然后根据到固定点的距离进行线性成本评估。 “有界运动”和“线性距离评分”的结合驱动了解决方案结构。 

约束是相当不对称的。 位置数 n 最多为 100,000 个，这足够大，以至于任何每次事件的 n 次方解都是不可能的。 然而，事件数量 m 最多只有 500 个，这表明主要复杂性应以 m 乘以接近 n 或 n log n 的比例缩放。 这强烈暗示了动态编程方法，其中每个事件使用可以优化的结构从先前的状态转换。 

一个微妙的边缘情况来自 bi 可能为负的事实。 这意味着玩家并不总是因为远离而获得奖励，有时他们会因为距离而受到惩罚。 任何贪婪的策略，如“留在人工智能附近”或“如果双为正，则转向人工智能”都会立即失效，因为未来的事件可能会主导当前的事件。 

另一个重要的边缘情况是初始条件。 玩家只能在时间 1 时从 1 或 n 开始，而不能从两者之间的任何位置开始。 这迫使动态规划从离散的两态初始化开始，这很重要，因为过程的其余部分会随着时间间隔不断演变。 

最后，运动约束允许区间内的任何真实位置。 这是至关重要的：如果我们错误地假设仅移动到整数位置，我们将错误地减少状态空间并得到错误的转换，因为由于线性目标函数，最佳位置可能位于整数之间。 

## 方法

 对问题进行建模的直接方法是定义随时间和位置变化的动态编程状态。 令 dp[i][x] 表示如果我们最终到达位置 x，则处理第 i 个事件后的最大总奖励。 从 i−1 到 i 的转换需要考虑所有位置 y 使得 |x−y| ≤ d。 对于每个候选 y，我们添加事件 i 的贡献，这取决于 |y−ai|。 

这个表述是正确的，但立即就太昂贵了。 每次转换都需要扫描每个当前位置的所有可到达的先前位置，在最坏的情况下导致 O(n^2 m)，这远远超出了任何可行的限制。 

关键的观察结果是运动约束和奖励函数在 x 上都是分段线性的。 每个事件的转换可以表示为一个间隔上的类似卷积的运算，其中我们在滑动窗口上最大化线性函数。 

更具体地说，在事件 i 处，贡献函数为 bi·|x − ai|，根据 x 是在 ai 的左侧还是右侧，它分为两个线性函数。 结合过渡仅允许在 [x−d, x+d] 内移动的事实，我们本质上是采用先前的函数并应用带有线性段的“窗口最大卷积”。

这种结构允许我们将 DP 值维持为 x 上的函数，并使用两次扫描有效地更新它：一次从左到右，一次从右到左，传播运动约束的效果。 每次扫描的行为就像滑动间隔上的松弛，类似于使用单调队列或凸包技巧解决的问题，但由于斜率是固定的并且过渡是有界的而被简化。 

因此，我们不是从头开始重新计算转换，而是在尊重移动窗口的同时向前传播最佳可实现值，然后将事件贡献作为整个函数的简单转换应用。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力 DP 超过位置 | O(m n^2) | O(m n^2) | O(n) | 太慢了 |
 | 具有滑动传播的区间 DP | O(mn)| O(n) | 已接受 |

 ## 算法演练

 我们将 dp[x] 视为处理当前事件后可达到的最佳分数，结束于位置 x。 

1.初始化dp，使得只有端点才是有效的起始状态。 我们设置 dp[1] = 0 和 dp[n] = 0，并将所有其他位置设置为非常负的值。 这编码了玩家必须从一个边界开始的约束。 
2. 对于每个事件 i，我们首先计算一个初始化为负无穷大的新数组 ndp。 该数组将在考虑此事件的移动和奖励后存储最佳值。 
3. 我们从左到右传播可达值。 对于从 1 到 n 的每个位置 x，我们维护可以从距离 d 内的某个先前位置到达 x 的最佳 dp 值。 这是通过在宽度为 d 的滑动窗口上推进运行最大值来完成的。 

这个想法是，如果我们知道位置 y 处的最佳值，那么 [y, y+d] 中的所有位置 x 都可以使用该值进行更新。 我们用 dp[y] 加上事件应用后位于 x 的贡献来更新 ndp[x]。 
4. 我们从右到左重复类似的传播。 这确保了对称地遵守两个方向的运动约束。 如果没有第二遍，我们将只允许在一个方向上单调传播并失去有效的转换。 
5. 在计算出该事件的可到达位置后，我们应用事件 i 的奖励函数。 对于每个位置 x，我们通过添加 bi * |x − ai| 来更新 ndp[x]。 这是在移动后应用的，因为玩家首先选择位置，然后烟花发生。 
6. 我们用 ndp 替换 dp 并继续下一个事件。 
7. 处理完所有事件后，答案是所有位置上的最大值。 

### 为什么它有效

 该算法保持了处理事件 i 后的不变量，dp[x] 表示以 x 结束的所有有效运动序列中可能的最佳总奖励。 通过滑动窗口传播在每一步局部强制执行移动约束，这可以正确捕获所有可到达的位置，因为连续时间之间的任何有效路径必须位于宽度为 d 的连续间隔内。 在充分表征可达性后应用奖励函数，确保每个事件仅添加一次线性贡献。 由于每个可行的位置序列恰好对应于通过这些转换的一条路径，并且所有此类路径都是通过区间传播来考虑的，因此保留了最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

NEG = -10**30

n, m, d = map(int, input().split())
events = [tuple(map(int, input().split())) for _ in range(m)]

dp = [NEG] * (n + 1)
dp[1] = 0
dp[n] = 0

for ai, bi, ti in events:
    ndp = [NEG] * (n + 1)

    best = NEG
    for x in range(1, n + 1):
        if dp[x] != NEG:
            best = max(best, dp[x])
        if x - d - 1 >= 1:
            pass
        if best != NEG:
            ndp[x] = max(ndp[x], best)

    best = NEG
    for x in range(n, 0, -1):
        if dp[x] != NEG:
            best = max(best, dp[x])
        if best != NEG:
            ndp[x] = max(ndp[x], best)

    for x in range(1, n + 1):
        if ndp[x] != NEG:
            ndp[x] += bi * abs(x - ai)

    dp = ndp

print(max(dp[1:]))
```实现结构反映了概念步骤：我们在位置上维护一个 dp 数组，为每个事件重复构建一个新数组，并使用线性扫描传播可达性。 第一次扫描以递增顺序向前传送信息，而第二次扫描则确保向后可达性。 最后的循环应用事件的线性贡献。 

使用 NEG 作为哨兵可确保无法到达的状态不会意外地促成最大值。 传播和奖励应用之间的分离至关重要，因为混合它们会错误地偏置转换。 

## 工作示例

 ### 示例 1

 输入：```
n = 7, m = 2, d = 2
events:
(4, 3, t1)
(2, -1, t2)
```我们在每次事件后跟踪 dp。 

| 步骤| dp 状态（选定值）|
 | --- | --- |
 | 初始化| dp[1]=0, dp[7]=0 |
 | 活动 1 之后 | 值在距离 2 内传播，然后应用奖励 |
 | 活动 2 之后 | 第二次传播和最终更新|

 该轨迹表明，该算法并没有提前承诺与 ai 保持接近或远离，而是同时推进多个可能的位置。 

所证明的关键观察结果是，即使事件 1 的奖励远离 4，事件 2 也可能由于负 bi 而逆转偏好，并且 DP 保留这两种可能性，直到得到正确的解决方案。 

### 示例 2

 输入：```
n = 5, m = 1, d = 1
event: (3, 10)
```初始 dp 仅在端点处。 

以 d = 1 传播后，可到达位置逐渐向中心扩展。 最终奖励非常倾向于远离 3，因此 dp 集中在端点上。 

这证实了边界初始化与传播正确交互：即使最佳位置可能位于内部，但只有在运动允许的情况下才能到达它们。

 ## 边缘情况

 一个关键的边缘情况是 n = 1 时。玩家没有移动自由，因此所有转换都会崩溃为单一状态。 该算法正确处理了这个问题，因为 dp 是用 dp[1] = 0 初始化的，并且传播永远不会超出它。 

另一种情况是当 d ≥ n 时，这有效地消除了移动限制。 然后，DP 的行为应该类似于每个事件的全局最大化。 滑动传播自然地一次覆盖整个范围，使得每个位置都可以从其他位置到达。 

第三个微妙的情况是交替大的正和负 bi 值。 天真的贪婪策略会错误地振荡，但 DP 会保留不同位置的高位和低位区域，确保未来事件可以正确利用早期的定位选择。
