 ## 解决方案
 ## 问题理解

 我们得到一个树状的状态结构，以节点 1 为根，其中每个新状态都附加到一个较早的状态，并且连接带有一个小写字母。 每个节点对应于通过沿着从根到该节点的唯一路径读取字母而形成的字符串。 

对于每个节点，我们查看它的根到节点字符串并询问它是否具有最小周期。 如果可以通过重复长度为 P 的较短字符串至少两次来构建字符串，则周期 P 是有效的。 在所有节点中，我们想要最大的最小周期。 

因此，任务不是以简单的方式独立计算每个节点的周期，而是找到所有根到节点字符串中最大的“重复结构强度”。 

输入约束最多允许 100000 个节点，因此每个节点的字符串长度也可以是 O(n)。 任何以 O(length) 为单位从头开始重新计算每个节点的字符串周期的解决方案都会降级为 O(n^2)，这远远超出了典型的 1 到 2 秒限制下可接受的范围。 

当字符串几乎是周期性的但不是完全周期性的时，就会出现一个微妙的问题。 例如，像“abababa”这样的字符串具有很强的结构，但无法完全重复，而“ababab”则完全是周期性的。 仅检查前缀后缀相等而不强制执行完整平铺的简单方法会错误地接受仅部分字符串匹配的情况。 

另一个极端情况是，根据此问题的定义，单字母或非重复字符串应贡献 0 的周期性。 例如，形成“abc”的路径没有有效的重复，因此它不能影响最大值。 

## 方法

 蛮力方法很简单。 对于每个节点，我们从根构建其完整路径字符串，并测试从 1 到字符串长度一半的所有可能的周期长度 P。 对于每个候选 P，我们验证每个字符是否与之前的一个 P 位置匹配。 如果存在有效的 P，我们记录最小的 P。 

这是正确的，因为它直接检查周期性的定义。 然而，构建每个字符串每个节点的累计成本为 O(n)，检查所有周期又是另一个 O(n)，因此每个节点的成本为 O(路径长度)。 对所有节点求和，就变成了 O(n^2)，当 n 达到 100000 时，这太慢了。 

关键的观察是我们实际上不需要具体化完整的字符串。 每个节点的字符串都是树中根到叶路径的前缀，因此问题是关于路径上的周期性结构。 我们不是从头开始重新计算结构，而是增量地维护信息。 

重要的结构见解是，周期性取决于相隔固定距离的匹配位置，并且这些位置沿着祖先链。 如果我们可以在任意深度有效地比较两个节点上的字符，我们就可以将周期性视为树上的动态属性。 

这建议在根路径上使用滚动哈希或前缀哈希。 通过散列，可以在 O(1) 中检查子字符串的相等性，从而允许我们通过验证节点的前缀是否等于其自身的移位版本来测试节点的周期性。 

一旦我们可以快速测试“这个节点的字符串是否具有周期P”，我们仍然需要找到最小的有效P。我们不是天真地扫描所有P，而是利用任何有效周期必须在结构意义上划分字符串长度的事实，并且我们可以重用类似于前缀函数（KMP）的失败链接式推理。 这使我们能够为每个节点维护其字符串的最长正确边界并从中导出周期性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 |---|---|---|---|
 | 蛮力 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 太慢了|
 | 最优（树+前缀函数/散列）| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以递增的顺序处理节点，因为每个节点都依赖于较小的父索引。

1. 对于每个节点，我们使用滚动哈希和深度跟踪定义其根到节点字符串的表示。 这让我们可以在恒定时间内比较以节点结尾的任何前缀。 

2. 我们为每个节点的字符串计算一个 KMP 样式的前缀函数，但我们不是从头开始重新计算，而是将父节点的结构扩展了一个字符。 这是可行的，因为添加一个节点只会将一个字符附加到其父节点的字符串中。 

3. 对于节点 i，我们计算其前缀函数值 pi[i]，它给出最长的正确前缀，也是其字符串的后缀。 该值捕获字符串中最大的自重叠。 

4. 从 pi[i]，我们得出候选周期长度为 len[i] - pi[i]。 这是对齐前缀和后缀结构的最小移位。 

5. 我们检查该周期是否有效，因为它至少重复两次。 这相当于检查 len[i] 是否能被候选周期整除或者重复结构是否完全重建了字符串。 如果有效，我们更新全局最大周期。 

6. 我们对所有节点继续这个过程，始终沿着树传播前缀函数状态。 

### 为什么它有效

 prefix 函数对字符串的所有边界信息进行编码，这意味着捕获每个可能的前缀（同时也是后缀）。 任何有效周期都必须对应于边界重复模式，因为周期性意味着按周期移动可以保持整个字符串的相等性。 

由于每个节点的字符串与其父节点的字符串仅在末尾附加一个字符不同，因此所有边界关系都会在本地更新。 这确保我们永远不会错过候选周期，并且前缀函数保证最小的重复结构始终源自最长的边界。 结果，每个有效的周期字符串都被正确检测到，并且所有节点上的最大值是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    parent = list(map(int, input().split()))
    s = input().strip()

    g = [[] for _ in range(n)]
    for i, p in enumerate(parent, start=1):
        g[p - 1].append(i)

    pi = [0] * n
    ans = 0

    stack = [(0, 0, 0)]  # node, prefix function value, depth

    # We simulate DFS carrying KMP state
    def dfs(u, p, cur_pi):
        nonlocal ans
        depth = cur_pi
        if u != 0:
            c = s[u - 1]
            j = p
            while j > 0 and s[j] != c:
                j = pi[j - 1]
            if j < u and s[j] == c:
                j += 1
            pi[u] = j
            depth = j

            length = u + 1
            if depth > 0:
                period = length - depth
                if length % period == 0:
                    ans = max(ans, period)

        for v in g[u]:
            dfs(v, u, depth)

    dfs(0, 0, 0)
    print(ans)

if __name__ == "__main__":
    solve()
```该代码构建树并在形成时计算每个节点的根路径字符串的前缀函数值。 DFS 确保每个节点继承其父节点的匹配状态，这允许 KMP 转换在每条边的摊销常数时间内发生。 

在计算前缀函数之后立即执行周期性检查，因为该值已经确定了最大边界，从而确定了候选重复结构。 

一个常见的陷阱是尝试使用原始父索引而不是 KMP 故障链来重新计算前缀值。 这将打破线性时间保证并过度计算不匹配。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 |---|---|---|
 | 时间 | O(n) | 每个节点都处理一次，并且 KMP 转换在边缘上摊销 |
 | 空间| O(n) | 树存储和前缀函数数组 |

 该解决方案非常适合 n 高达 100000 的限制，因为每次转换都是恒定的摊销工作，并且不执行完整的字符串重建。

 ## 边缘情况

 对于单节点树，路径字符串的长度为1，无法形成重复序列，因此前缀函数保持为零，算法正确返回0。 

对于像“aaaaa”这样的完全统一的链，每次扩展都会将前缀函数增加到当前长度减一，在每个节点产生周期 1。 DFS 会连续维护这一点，因为每个字符都与前一个边界匹配，因此每一步都会更新最大值。 

对于诸如“ababab”之类的交替模式，前缀函数会振荡，但会在均匀深度处收敛到稳定边界。 该算法无需显式检查所有除数即可捕获这一点，因为边界长度已经对重复结构进行了编码。
