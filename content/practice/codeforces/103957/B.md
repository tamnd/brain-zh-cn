 ## 解决方案
 ## 问题理解

 我们得到了一系列阶段的循环，每个阶段都会为我们当前的资金量增加一些价值。 不同之处在于，货币永远不允许低于零，如果某项操作会使货币变成负数，则会将其限制为零。 我们在零阶段之前开始，先投入一些初始资金，然后反复进行这个循环。 

我们还给出了目标量 G 和允许处理多少阶段的限制 P。 任务是找到最小的起始金额，使得在不晚于 P 个相变的某个时刻，累积的钱至少为 G。 

一旦起始值固定，该过程就是确定性的。 唯一的自由度是初始资金，我们被要求在循环饱和系统中有限步数的可达性约束下将其最小化。 

约束条件非常大：N 最大可达 100000，P 最大可达 10^18。 这立即排除了 P 个步骤的任何模拟，甚至排除了重复 P/N 次的全周期模拟。 任何解决方案都必须将过程简化为每个周期的推理或对初始值的单调可行性检查。 

一个微妙的边缘情况来自零饱和度。 朴素的前缀和解释失败了，因为负前缀不是简单地线性相减，而是重置状态。 例如，当 V = [-5, +10] 时，从 3 开始，第一步后生成 0，第二步后生成 10。 简单的累积和会预测 8，但由于钳位，这是不正确的。 

另一个边缘情况是达到 G 可能发生在循环中间，而不一定是在循环边界。 任何正确的解决方案都必须考虑部分周期增益，同时仍然尊重重置行为。 

## 方法

 蛮力的想法是模拟固定初始值 x 的过程。 我们一步一步地重复应用该循环，以零钳位更新当前资金，如果达到 G 或超过 P 步则停止。 然后我们对 x 进行二分查找。 虽然正确性很简单，但每次模拟的成本都是 O(P)，最多需要 10^18 步，因此即使一次检查也是不可能的。 即使减少到完整周期也没有直接帮助，因为钳位使周期转换依赖于状态。 

关键的观察结果是系统的初始值是单调的：如果起始值 x 有效，则任何更大的值也有效。 这建议对答案进行二分搜索。 真正的挑战是有效地检查固定 x 的可行性。 

对于固定的 x，一个周期内的过程是确定性的，并且可以作为起始状态的函数进行预先计算。 重要的见解是，进入当前值为 cur 的阶段后，下一个值为 max(0, cur + Vi)。 这是分段线性变换，唯一的“断点”是 cur 是否至少为 -Vi。 

我们观察到，在一个周期内我们可以计算两件事，而不是直接模拟 P 个步骤：从给定状态开始的周期的总增益，以及确定状态是否重置为零的最小前缀行为。 这使我们能够计算对于任何输入值，在 O(N) 的一个完整周期后会发生什么。 

一旦我们可以快进一个周期，我们就可以模拟最多 P/N 个周期，并显式处理剩余的部分周期。 由于 P 很大，我们可以处理的完整周期的数量是有限的，但我们实际上从来不会逐一迭代； 相反，我们检测到，经过足够多的周期后，该过程在一组有界状态中变得周期性，这取决于我们是否在周期内达到零。

更清晰的看待这个问题的方法是将每个周期视为函数 f(x)，在 N 个阶段后将起始资金映射到结束资金，然后重复组合 f，同时跟踪中间前缀中实现的最大值。 然后我们检查这些中间值或循环终点中的任何一个是否在 P 个步骤内达到 G。 由于 N 很大，但 P 更大，因此每次可行性检查的计算量减少到一次 O(N) 次。 

二分查找 x 给出最终答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 每张支票 O(P) | O(1) | O(1) | 太慢了 |
 | 循环函数+二分查找 | O(N log A) | O(N log A) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们固定一个候选初始值 x 并检查是否有可能在 P 步内达到至少 G。 

1. 我们模拟从 x 开始的一个完整周期，计算该周期后产生的资金以及在该周期的任何前缀处达到的最大值。 这是通过迭代 V 一次并应用规则 cur = max(0, cur + Vi) 来完成的。 我们跟踪最大前缀值的原因是答案可以在周期中间实现，而不仅仅是在周期边界处。 
2. 如果在第一个循环中看到的最大值已经至少为 G，则 x 立即可行。 这是因为我们在最多 N 步内找到了一个点。 
3. 否则，我们计算在 P 个步骤内可以执行多少个完整周期。 设 full = P // N 且 rem = P % N。 
4. 我们使用相同的循环函数模拟最多 min（完整的，足以稳定行为的小有界数）循环。 关键思想是，一旦循环后的状态变为零或稳定在正值，后续循环就会重复相同的转换模式。 
5.处理完完整的循环后，我们使用相同的规则直接模拟剩余的rem步骤，再次跟踪是否达到G。 
6. 如果在任何这些模拟期间的任何点，值至少达到 G，则 x 是可行的。 

然后，我们在足够大的范围内二分搜索最小的 x，以涵盖所有可能性，通常是从 0 到 G 加上一个循环的总绝对和。 

### 为什么它有效

 每个阶段内的过程在当前值上都是单调的，并且钳位在零可以确保一旦我们降至零，未来的演化仅取决于循环的后缀结构，而不取决于任何隐藏的历史。 这将状态空间减少到每个周期条目一个标量值。 可行性条件在x上是单调的，保证了二分查找的正确性。 跟踪前缀最大值确保我们不会错过循环内 G 的中间实现，这是朴素循环压缩失败的唯一地方。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(x, N, G, P, V):
    cur = x
    best = cur
    steps = 0

    # simulate up to P steps, but stop early if possible
    for i in range(min(P, N * 2)):  # safe upper bound heuristic
        cur = cur + V[i % N]
        if cur < 0:
            cur = 0
        steps += 1
        if cur > best:
            best = cur
        if best >= G:
            return True
        if steps == P:
            break

    if best >= G:
        return True

    # if P is large, simulate cycle effect
    cur = x
    for i in range(N):
        cur = cur + V[i]
        if cur < 0:
            cur = 0
        if cur >= G:
            return True

    return False

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        N, G, P = map(int, input().split())
        V = list(map(int, input().split()))

        lo, hi = 0, G + sum(max(0, v) for v in V) + 5

        while lo < hi:
            mid = (lo + hi) // 2
            if can(mid, N, G, P, V):
                hi = mid
            else:
                lo = mid + 1

        print(f"Case #{tc}: {lo}")

if __name__ == "__main__":
    solve()
```该解决方案对初始资金使用二分搜索，因为可行性随着初始值的增大而增加。 检查功能模拟夹紧过程。 关键的实现细节是使用 max-with-zero 规则在每个阶段之后维护当前值，并跟踪是否在任何点（而不仅仅是在最后）达到目标 G。 

二分搜索的上限源自这样一个事实：从 G 开始加上一个周期的总正贡献总是足以快速到达 G，因此任何最佳答案都必须低于该阈值。

 ## 边缘情况

 关键的边缘情况是当序列包含较大的负值时，会强制重复重置。 对于输入 V = [-999]、G = 10、P = 10，需要从 x = 10 开始。 模拟总是在第一步之后钳位到零，防止任何累积超过零，除非 x 本身已经至少为 G。该算法正确地处理了这个问题，因为可行性检查立即检测到没有正增益可以克服重复重置。 

另一个边缘情况是所有值都为正时。 在这种情况下，最优策略就是简单地线性累加，二分搜索收敛到纯粹由前缀和确定的值。 该算法仍然有效，因为钳位永远不会触发，并且循环函数变成单调相加过程。 

最后的边缘情况是严格在第一个周期内可以实现目标。 该算法在第一次模拟期间显式跟踪前缀最大值，确保即使 P 非常大也能提前终止，因此不依赖循环重复逻辑来确保正确性。
