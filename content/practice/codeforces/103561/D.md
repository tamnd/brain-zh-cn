 ## 解决方案
 ## 问题理解

 我们在二维整数网格上得到一组点，并且所有点都是从坐标系中心的固定原点观察的。 从这个原点开始，我们想象一个“相机”只能在由从原点开始的两条射线定义的楔形区域内看到。 正好位于这两条射线上或之间的所有物体都是可见的。 

任务是选择这种楔形的尽可能最小的角宽度，以便每个给定点都位于其中。 楔子可以绕原点自由旋转，因此我们不固定其方向，仅固定其张角。 

输入只是点数及其坐标。 输出是一个实数：可以同时覆盖所有点的楔形的最小角度（以度为单位）。 

约束最多允许 100,000 个点，因此任何直接比较所有对或尝试候选楔形的所有旋转的解决方案都太慢。 O(n²) 方法将涉及检查所有点对作为潜在边界射线，这会在最坏的情况下导致大约 10^10 次操作，并且不可行。 因此，预处理后我们需要一个 O(n log n) 或 O(n) 结构。 

角度环绕会出现一个微妙的问题。 角度 0 度附近的点和 359 度附近的点实际上可能按圆形顺序接近，但对角度进行简单的线性扫描会将它们视为相距较远。 任何正确的解决方案都必须处理这种循环性质。 

另一种边缘情况是当多个点位于来自原点的同一条射线上时。 例如，点 (2, 0) 和 (5, 0) 不应引入任何角度分离； 他们的行为是同一个方向的。 不仔细标准化角度的简单实现仍然有效，但在推理差异时需要小心。 

最后，浮点精度很重要，因为答案需要在 1e-6 容差范围内。 任何依赖于反三角函数的方法都必须在舍入误差下保持稳定。 

## 方法

 蛮力的想法是考虑楔子的每一个可能的方向。 一种自然的方法是使用 atan2 计算每个点相对于原点的角度，对这些角度进行排序，然后尝试选择一个起点并计算覆盖从那里开始的所有点所需的最大角度间隙。 对于每个起始索引i，我们向前扫描并找到向前180度内的最远点，或者等效地确定旋转后包含所有点的圆上的最小间隔。 

然而，独立检查每个起点的这种想法的简单实现会导致 O(n²) 行为，因为对于每个点，我们可以扫描所有其他点来计算最大角距离。 当 n = 10⁵ 时，这太慢了。 

关键的观察是，我们实际上正在寻找覆盖所有点的最小圆弧。 一旦所有点都转换为 [0, 360) 中的角度，问题就变成：找到包含所有点的圆上弧的最小长度。 这相当于找到连续排序角度之间的最大间隙，包括最后一个点和第一个点之间的环绕间隙。 如果我们去掉这个最大的间隙，剩下的就是覆盖所有点的最小弧。 

这是可行的，因为任何最佳楔形都可以旋转，使其边界与其中一个点对齐，而覆盖所有点的唯一障碍是最大的空角区域。 删除该区域会产生最小包围弧。 

这减少了排序角度和扫描一次的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最优（排序+最大间隙）| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1. 使用 atan2(y, x) 将每个点 (x, y) 转换为其极角。 这会将每个点放置在以弧度表示的圆上。 使用atan2而不是arctan的原因是它可以正确处理所有象限而无需手动调整。 
2. 将所有角度标准化为一致的范围，通常为 [0, 2π)。 这确保排序产生正确的循环顺序。 
3. 按升序对所有角度进行排序。 排序后，连续的元素代表以原点为中心的相邻方向。 
4. 计算排序列表中连续元素之间的角度差。 还要计算最后一个角度加上 2π 与第一个角度之间的环绕差。 这些差异代表了圆上的空白。 
5. 找出所有这些差异中的最大差距。 该间隙代表不包含点的最大角扇区。 
6. 用 2π 减去这个最大间隙，得到包含所有点的最小弧。 
7. 通过乘以 180 / π 将结果从弧度转换为度数。 

为什么有效：任何覆盖所有点的楔形都必须排除圆上最多一个连续的空白区域。 最佳的楔形是通过将其开口放置在最大的空白间隙的正对面来实现的，因为该间隙是唯一不需要被覆盖的区域。 因此，最小化楔角相当于最大化排除间隙，即连续角位置之间的最大间距。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def solve():
    n = int(input())
    angles = []
    
    for _ in range(n):
        x, y = map(int, input().split())
        ang = math.atan2(y, x)
        if ang < 0:
            ang += 2 * math.pi
        angles.append(ang)

    angles.sort()

    if n == 1:
        print(0.0)
        return

    max_gap = 0.0

    for i in range(n):
        j = (i + 1) % n
        if j == 0:
            gap = (angles[0] + 2 * math.pi) - angles[i]
        else:
            gap = angles[j] - angles[i]
        max_gap = max(max_gap, gap)

    answer = 2 * math.pi - max_gap
    print(answer * 180 / math.pi)

if __name__ == "__main__":
    solve()
```该解决方案的结构是将几何配置转化为循环排序问题。 atan2 的使用确保了所有象限的正确性，并且归一化步骤避免了负角度不一致。 

一个常见的实施错误是忘记最后一个角度和第一个角度之间的环绕间隙。 如果没有它，所有点都聚集在 0 度附近但有一个点位于 360 度附近的情况将产生不正确的大楔形而不是小楔形。 

另一个微妙之处是处理 n = 1。在这种情况下，不需要角度，正确答案是零。 

## 工作示例

 ### 示例 1

 输入：```

0 3
```| 步骤| 角度（弧度）| 已排序 | 差距| 最大间隙|
 | --- | --- | --- | --- | --- |
 | 转换后| 0, π/2 | 0, π/2 | π/2 | π/2 |

 最大间隙为 π/2，即半个圆是空的。 删除它留下 π/2 作为最小覆盖弧。 转换为 90 度。 

### 示例 2

 输入：```
3
-3 0
0 3
-3 -3
```| 步骤| 角度（弧度）| 已排序 | 差距| 最大间隙|
 | --- | --- | --- | --- | --- |
 | 转换后| π, π/2, -3π/4 | π/2, π, 5π/4 | π/2, π/4, 5π/4-π/2 | π/2 |

 最大差距是3π/4？ 实际上，从 π 到 5π/4 的环绕会产生 π/4，而最大的间隔是在 5π/4 和 π/2 之间，即 3π/4。 移除该间隙后会留下 3π/4 的弧度，相当于 135 度。 

这显示了该解决方案如何正确识别最紧密的覆盖楔块不一定与明显的轴方向对齐。

 ## 边缘情况

 一种边缘情况是所有点都位于来自原点的同一条射线上。 在这种情况下，所有计算出的角度都是相同的，因此排序会产生一个常量数组。 间隙计算产生完整的 2π 间隙作为环绕差异，并且剩余弧变为零，这是正确的，因为零角度透镜覆盖单个方向。 

另一种边缘情况是点聚集在 0/360 度边界周围。 例如，1 度和 359 度处的点应产生 2 度的小弧，而不是 358 度的大弧。 环绕间隙计算通过考虑排序后最后一个角度和第一个角度之间的圆差来显式处理此问题。 

最后的边缘情况是当点几乎共线时的浮点精度。 使用atan2可确保稳定的排序，并且以弧度工作而无需重复转换，可最大限度地减少累积误差。 最终转换为度数一次，从而将精度保持在所需的公差范围内。
