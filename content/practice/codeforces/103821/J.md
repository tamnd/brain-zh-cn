 ## 解决方案
 ## 问题理解

 我们得到了一组球，其中每个球都有一种颜色，因此输入本质上是颜色的多重集。 除此之外，我们还被要求将所有球精确地放入 K 个盒子中。 这些盒子是独特且有序的，因此交换两个盒子会产生不同的排列。 然而，在盒子内部，只有多组颜色很重要，这意味着盒子内球的顺序无关紧要。 

每个盒子必须至少包含一个球。 仅当每个框在相同位置具有完全相同的多组颜色时，两个分区才被视为相同。 

因此，任务是计算我们可以采用多少种方式获取一组彩色项目并将其分成 K 个标记的非空组，其中只有每个组内的颜色才重要。 

约束告诉我们 N 最多为 1000，并且测试用例的总工作量以 N 乘以 K 为单位大约为 10^6。这强烈表明每个测试的 O(NK) 或 O(NK log N) 解决方案是可以接受的，但 N 或 K 中的任何指数都是不可能的。 在简单的组合搜索中迭代盒子子集或单独分配球的解决方案将远远超出限制。 

一个微妙的点是球不是不同的物体。 如果我们忽略这一点并将每个球视为唯一的，那么在多个球共享相同颜色的情况下，我们会严重过量计算。 另一个微妙的问题是每个盒子的非空约束，它耦合了不同颜色的独立分布。 

一些边缘场景清楚地说明了这些陷阱。 如果所有球都有相同的颜色，比如 N 个相同的球和 K 个盒子，那么问题就简化为将相同的物品分配到有序的非空盒子中，这是一个经典的组合计数问题。 每个颜色分布的幼稚产品会错误地允许空框。 

如果 K 等于 N 并且所有颜色都不同，则每个盒子必须恰好包含一个球，答案就是 N 的阶乘。 任何忘记盒子顺序或将球视为无法区分的方法在这里都会失败。 

## 方法

 如果我们忽略结构，最直接的想法就是考虑将每个球放入 K 个盒子之一，然后计算有效的分配。 这立即给出了 K^N 的可能性。 然而，这忽略了相同颜色的球是无法区分的，因此许多作业代表相同的结果。 它还会生成许多带有空框的配置，这些配置是无效的。 

尝试解决这个问题会直接导致我们考虑按颜色分配相同的物品。 对于频率为 f 的固定颜色，将这些相同的球分配到 K 个标记的框中是标准的星形和条形计算，给出 C(f + K - 1, K - 1)。 如果我们对每种颜色独立执行此操作并相乘结果，我们就会正确计算将颜色计数分配到框中的所有方法，但我们完全忽略了总体上没有框可以为空的约束。 一个盒子可能会收到每种颜色的零个球，这是无效的。 

因此，如果我们将两层分开，结构就会变得更加清晰。 首先，我们为每种颜色指定每个盒子中有多少该颜色的球。 这给出了每种颜色的独立组合。 然后我们强制每个框的所有颜色的总和为正。 第二个条件是跨颜色的全局条件，并且正是盒子上的包含排除变得自然的地方。 

我们将盒子视为约束：每个盒子必须非空。 我们通过选择允许为空的框的子集并相应地减去或添加配置来应用包含-排除。 如果我们修复 m 个盒子被迫为空，那么我们只分布到 K - m 个活动盒子中，现在每种颜色的分布再次是独立的。 这恢复了可分离性。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | ---|
 | 暴力作业 | O(K^N) | O(K^N) | O(1) | O(1) | 太慢了 |
 | 仅按颜色分布 | O(NK) | O(K) | 错误 |
 | 盒子上的包含-排除 | O(K * 颜色数量) | O(K) | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 计算输入数组中每种颜色的频率。 这将问题从球压缩到颜色类，因为仅对有效配置中的每种颜色进行计数。 
2. 使用模运算预先计算高达 N + K 的阶乘和逆阶乘。 这允许快速计算二项式系数，这些系数在分配相同项目的公式中重复出现。 
3. 对于固定数量的活动框 B，计算将每种颜色独立分配到这些 B 框的方式有多少种。 对于频率为 f 的颜色，为 C(f + B - 1, B - 1)。 将其乘以所有颜色即可得到 B 的总方式数。 
4. 对空盒子使用包含-排除。 对于从 0 到 K 的每个 m，将 m 解释为被迫为空的盒子的数量。 选择这些框的方式数量为 C(K, m)，剩余的活动框为 B = K - m。 
5. 根据 m 交替进行加法和减法。 如果m是偶数，则加上贡献； 如果是奇数，则减去它。 每一项贡献 C(K, m) 乘以步骤 3 中计算的颜色乘积，其中 B = K - m。 
6. 将所有贡献以 1e9+7 为模求和以获得最终答案。 

### 为什么它有效

 对于将颜色计数固定分配到框中的情况，请考虑最终为空的一组框。 如果没有框为空，则包含-排除为该配置分配净系数 1，否则为 0。 发生这种情况是因为每个无效配置在包含其空框的子集中都以正号和负号进行同等计数，从而导致取消。 有效配置仅出现在没有排除任何框的术语中，因此它们仅存活一次。 

颜色之间的独立性之所以成立，是因为一旦盒子结构固定，分配每种颜色的相同项目就不会与其他颜色相互作用。 颜色之间唯一的耦合是非空约束，它完全通过包含-排除来处理。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
MAXN = 2005

fact = [1] * (MAXN)
invfact = [1] * (MAXN)

def modexp(a, e):
    r = 1
    while e:
        if e & 1:
            r = r * a % MOD
        a = a * a % MOD
        e >>= 1
    return r

for i in range(1, MAXN):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN - 1] = modexp(fact[MAXN - 1], MOD - 2)
for i in range(MAXN - 2, -1, -1):
    invfact[i] = invfact[i + 1] * (i + 1) % MOD

def C(n, k):
    if n < 0 or k < 0 or k > n:
        return 0
    return fact[n] * invfact[k] % MOD * invfact[n - k] % MOD

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        arr = list(map(int, input().split()))

        freq = {}
        for x in arr:
            freq[x] = freq.get(x, 0) + 1

        colors = list(freq.values())

        ans = 0

        for m in range(0, k + 1):
            B = k - m
            if B == 0:
                continue

            ways = 1
            for f in colors:
                ways = ways * C(f + B - 1, B - 1) % MOD

            cur = C(k, m) * ways % MOD

            if m % 2 == 0:
                ans = (ans + cur) % MOD
            else:
                ans = (ans - cur) % MOD

        print(ans % MOD)

if __name__ == "__main__":
    solve()
```阶乘预计算是必要的，因为二项式系数在包含-排除循环内被评估多次。 核心循环迭代可能的空盒子数量，并针对每种情况计算剩余的活动盒子可以独立接收彩球的方式。 

二项式项 C(f + B - 1, B - 1) 编码将 f 个相同的项目分布在 B 个标记框中。 颜色相乘是有效的，因为一旦盒子标识固定，颜色就不会相互作用。 

交替的加法和减法实现了对盒子集合的包含-排除。 跳过 B = 0 的情况可以避免不存在框的情况下的无效组合。

 ## 边缘情况

 关键的边缘情况是当 K 很大但在中间包含-排除项中许多框必须为空时。 该算法通过跳过 B = 0 的情况来处理这个问题，因为将球分配到零个盒子中没有意义，并且对于 N ≥ 1 没有任何贡献。 

另一个边缘情况是所有球都具有相同的颜色。 那么只有一个频率 f = N，并且每种颜色的分布简化为 C(N + B - 1, B - 1)。 包含-排除仍然正确地强制执行非空框，并简化为将整数的组合计数为 K 个有序部分。 

最后，当所有颜色都不同时，每个 f = 1 并且公式简化为 C(B, B - 1) = B 的乘积，其折叠为与有序框之间的排列一致的阶乘结构。
