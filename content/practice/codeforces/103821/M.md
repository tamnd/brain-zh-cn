 ## 解决方案
 ## 问题理解

 我们给出了从 1 到 N 的数字排列，我们通过从左到右扫描排列来定义分数。 每次我们放置一个值 P[i] 时，我们都会查看所有之前的值 P[j]，并在 P[j] 除 P[i] 时加 1。 排列的分数是除数在排列中出现较早的所有有序对中此类除数关系的总数。 

任务不是计算一种排列的分数。 相反，我们必须考虑大小 N 的每种可能的排列，计算其分数，并将这些分数相加。 

输入由多个测试用例组成，每个测试用例给出的值 N 最大为 100000。由于最多可以有 100000 个测试用例，因此任何解决方案都必须有效地预先计算答案并在恒定时间内回答每个查询。 

这些约束意味着任何涉及枚举排列的方法都是不可能的，因为 N！ 增长速度极快。 即使迭代所有排列中的所有对也将远远超出可行的限制。 这促使我们以组合方式计算贡献，避免显式构造排列。 

一个微妙的边缘情况来自于取决于排列顺序的定义。 例如，对于 N = 1，答案通常是 0，因为不存在对。 对于 N = 2，两种排列的贡献不同，但在所有排列的期望中是对称的，并且最终的总和必须考虑两种排序。 任何处理单个排列或假设固定排序结构的方法都会失败。 

## 方法

 蛮力解释很简单。 我们迭代大小为 N 的每个排列，通过检查所有对 i < j 来计算其分数，并验证 P[i] 是否整除 P[j]。 这是正确的，但从根本上来说是不可行的。 有N个！ 排列，每个排列都需要 O(N^2) 个检查，给出 O(N!·N^2)，即使对于 N = 10，这也是一个天文数字。 

关键的观察结果是，分数仅取决于对 (a, b) 的相对顺序，其中 a 除 b。 我们不构造排列，而是提出一个不同的问题：对于一个固定有序的值对 (a, b)，且除以 b，有多少个排列将 a 放在 b 之前？ 

一旦我们固定了（a，b）对，所有其他元素就可以自由排列了。 其中N个！ 排列，恰好一半将 a 放在 b 之前，一半将 b 放在 a 之前。 这种对称性之所以成立，是因为在任何排列中交换 a 和 b 的位置都会产生唯一的配对排列。 

因此，每个有效对 (a, b) 的贡献正好（a 出现在 b 之前的排列数），即 N！ /2 为总分。 那么分数就是：

 对所有 a < b 和 a | 对进行求和 N 中的 b! / 2.

 我们可以将其重写为：

 F(N) = (N! / 2) * (a, b) 对的计数，1 ≤ a < b ≤ N，使得 a 整除 b。 

整个问题简化为计算 [1, N] 范围内的除数关系。 

我们可以预先计算，对于每个 a，它上面存在多少个 a 的倍数。 对于每个 a，有效的 b 是 k·a 的倍数，其中 k ≥ 2。这样的 b 的数量是下限（N / a）- 1。对所有 a 求和给出有效有序对的总数。 

这将问题转化为除数的调和求和，可以使用标准除数计数累加以 O(N log N) 或 O(N) 的形式进行计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(N!·N^2) | O(N!·N^2) | O(N) | 太慢了 |
 | 组合 + 除数计数 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们首先观察到每个贡献对仅由值定义，而不是由它们在特定排列中的位置定义。 这使我们能够将排列计数与除数结构分开。

1. 对于从 1 到 N 的每个值 a，确定所有有效的 b，使得 b 是 a 的倍数且 b > a。 这些正是稍后可能出现在排列中的值，并与 a 形成除数关系。 
2、对于一个固定的a，统计这样的b有多少个。 这是floor(N / a) - 1。减法消除了简单的情况 b = a，因为在严格的对意义上，数字不算作自身除法。 
3. 将所有计数相加 这给出了使得 a 除 b 的有效有序对 (a, b) 的总数。 
4. 将总对数乘以 N！ / 2. 这说明了在所有排列的一半中，a 出现在 b 之前，而在另一半中，a 出现在 b 之后。 
5. 由于重复需要阶乘值，因此在测试用例中预先计算阶乘直至最大 N。 所有计算均以 1e9 + 7 为模进行，因此使用 2 的模逆而不是除法。 
6. 对于每个查询 N，使用预先计算的除数计数计算对计数，乘以阶乘 [N]，然后乘以 inv2。 

关键思想是排列结构只影响排序概率，而不影响除数对的存在。 

### 为什么它有效

 用 b 来固定任何一对不同的值 (a, b)。 在所有排列中，a 出现在 b 之前的排列和 b 出现在 a 之前的排列之间存在完美的双射，这是通过交换它们的位置而获得的。 这种对称性保证了所有排列中正好有一半的排列对这一对贡献了分数。 由于贡献对于值对而言是独立的，因此适用求和线性，并且总分是独立对贡献的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

MAXN = 100000

# precompute factorials
fact = [1] * (MAXN + 1)
for i in range(2, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

inv2 = (MOD + 1) // 2

# precompute divisor-pair counts up to N
cnt = [0] * (MAXN + 1)
for a in range(1, MAXN + 1):
    for b in range(2 * a, MAXN + 1, a):
        cnt[b] += 1

# prefix sum of valid (a,b) pairs per N
pairs = [0] * (MAXN + 1)
for i in range(1, MAXN + 1):
    pairs[i] = pairs[i - 1] + cnt[i]

t = int(input())
out = []
for _ in range(t):
    n = int(input())
    ans = pairs[n] * fact[n] % MOD
    ans = ans * inv2 % MOD
    out.append(str(ans))

print("\n".join(out))
```阶乘数组支持直接计算 N! 在模运算下。 cnt 数组用于计算每个数字从其真除数接收到多少个较小的除数，通过对 b 求和来有效地计算有效的 (a, b) 对。 前缀和对[n]存储数字1到n中除数边的总数。 

然后，每个查询都使用之前导出的公式：总分等于有效对的数量乘以 N！ / 2.

 在模算术中，乘以 inv2 取代了除以 2，这是必要的，因为正好有一半的排列贡献于每对。

## 测试用例```python
import sys, io

MOD = 10**9 + 7

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    MAXN = 100000

    fact = [1] * (MAXN + 1)
    for i in range(2, MAXN + 1):
        fact[i] = fact[i - 1] * i % MOD

    inv2 = (MOD + 1) // 2

    cnt = [0] * (MAXN + 1)
    for a in range(1, MAXN + 1):
        for b in range(2 * a, MAXN + 1, a):
            cnt[b] += 1

    pairs = [0] * (MAXN + 1)
    for i in range(1, MAXN + 1):
        pairs[i] = pairs[i - 1] + cnt[i]

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        ans = pairs[n] * fact[n] % MOD
        ans = ans * inv2 % MOD
        out.append(str(ans))

    return "\n".join(out)

# minimal cases
assert solve("1\n1\n") == "0"
assert solve("1\n2\n") == str((1 * 2 // 2) * 1 % MOD)

# small hand cases
assert solve("1\n3\n") is not None

 ## 边缘情况

 对于 N = 1，除数对计数为零，因为不存在不同的元素。 该算法生成pairs[1] = 0，以及factorial[1] = 1，因此最终答案为0，与定义匹配。 

对于像 2 或 3 这样的小 N，内部除数循环仅正确地累加正确的倍数。 对于 N = 2，cnt[2] 从 a = 1 开始递增一次，给出一对有效的 (1,2)，公式将其乘以 2！ / 2 = 1。 

对于较大的 N，排列的对称性假设至关重要。 该算法依赖于这样一个事实：交换任何对 (a, b) 都会以相反的顺序进行排列，从而确保精确的一半贡献。 该实现从未显式地构造此映射，而是依赖于它的正确性，并且算术通过乘以 inv2 直接反映它。
