 ## 问题理解

 我们有一个一维走廊$N$细胞排列成一行。 机器人从中间单元开始，特别是在索引处$\lceil N/2 \rceil$，并且最初面向右侧。 机器人一步步行走，它的运动规则完全由它是第一次进入一个单元还是再次访问一个单元来驱动。 

从当前方向开始，机器人不断地逐个单元地移动。 当它踏上以前从未访问过的单元格时，它会立即翻转方向。 如果它走到之前访问过的单元格上，它就会继续朝同一方向前进。 该过程持续进行，直到该行中的每个单元格都被至少访问过一次。 我们被要求计算机器人在过程结束之前进行的单位移动总数。 

重要的隐藏结构是，机器人正在有效地执行从起点向外增长的区间的确定性遍历，但具有由“首次访问时打开”规则引起的特定的来回运动模式。 

约束条件允许$N$最多$10^9$直至$10^5$测试用例，因此任何逐步移动的模拟都是不可能的。 即使每个测试用例的线性扫描也已经超过$10^{14}$在最坏的情况下进行操作，这远远超出了任何可行的极限。 这迫使每个测试用例进行封闭式推理，最好是恒定时间。 

值较小时会出现微妙的边缘情况。 为了$N = 1$，机器人从唯一的单元开始，并且已经“访问了所有内容”，所以答案是零步。 为了$N = 2$，中点定义选择单元格 1（因为$\lceil 2/2 \rceil = 1$），机器人立即向右行走一次并停止，产生非常小的不对称行为。 任何依赖对称性的解决方案都必须明确考虑这些微小的情况，因为一般模式只能在更大的情况下稳定。$N$。 

另一个不明显的陷阱是假设机器人只是简单地向左走到尽头，然后向右走到尽头一次。 方向翻转仅在首次进入未访问的单元格时触发，而不是在触及边界时触发，因此该运动不是简单的弹跳。 

## 方法

 强力模拟保留已访问单元格的布尔数组，并沿当前方向重复移动一步。 每次移动后，它都会检查单元格是否是新的，如果需要则翻转方向，并计算步数，直到访问完所有单元格为止。 这是正确的，但非常昂贵。 在最坏的情况下，机器人会探索$N$但由于重复回溯，可能会多次遍历长段，从而导致二次或接近二次的行为，具体取决于实现。 和$N$最多$10^9$，连存储数组都是不可能的，连概念步骤模拟都不可能。 

关键的观察结果是机器人永远不会任意重访。 访问集始终是单个连续段。 一旦机器人访问了一个区间$[L, R]$，它始终位于其一端或内部，并且下一次扩展总是根据方向和奇偶校验在任一侧将此间隔恰好扩展一个单元格。 在到达新的边界单元之前，每个扩展步骤都会在当前访问的段内引起可预测量的来回移动。 

这将问题简化为跟踪间隔如何从中间向外增长。 每次机器人将访问范围扩展一个新单元格时，它都会执行与访问间隔的当前大小成比例的确定性移动量。 将这些贡献相加得出一个仅基于以下条件的封闭式表达式：$N$和初始位置，消除了任何模拟的需要。 

最后的简化是，答案仅取决于起始单元的左侧和右侧发生了多少次扩展以及它们发生的交替模式。 这会产生一个可以在恒定时间内计算的线性算术结构。

 ## 算法演练

 核心思想是将这个过程重新解释为围绕起始位置扩展访问区间。 

设起始位置为$s = \lceil N/2 \rceil$。 将左侧的单元格数量定义为$L = s - 1$，向右为$R = N - s$。 

我们将该过程建模为每次重复扩展一个单元格访问的段。 每个扩展对应于一个“首次访问”事件，并且每个此类事件贡献的可预测移动次数等于距扩展边界的当前距离的两倍加上一个转换步骤。 

## 算法演练

 1. 计算起始索引$s$并推导出$L$和$R$。 这将问题分解为机器人必须在每个方向上扩展多远。 
2. 观察机器人最终必须访问所有$L + R + 1 = N$细胞，这意味着确切地说有$N - 1$从初始单元格开始扩展。 
3.保持概念间隔$[l, r]$开始于$[s, s]$。 每个步骤都会扩展$l$或者$r$向外一。 
4. 当向一侧扩展时，机器人遍历整个当前区间以到达新的边界。 该遍历的成本恰好是当前间隔长度（以步长为单位）。 
5. 到达新单元格后，机器人翻转方向，导致下一次遍历以确定性模式交替进行。 
6. 不是模拟交替，而是将扩展分组为左序列和右序列，分别对算术贡献求和。 
7. 使用每个边扩展贡献一系列递增遍历长度的事实来求总成本：$1, 2, 3, \dots$每侧，取决于订购。 

### 为什么它有效

 每时每刻，所访问的单元格都会形成一个连续的段，而机器人唯一有意义的状态是它正在向下一个扩展的端点。 “首次访问时翻转方向”规则保证一旦到达新的边界，机器人必须再次遍历整个已知段才能到达相反的一侧。 这迫使确定性的交替模式仅取决于有多少单元位于起点的左侧和右侧，而不取决于内部路径历史。 由于每次移动要么扩展该线段，要么完全遍历该线段，因此总功是扩展期间线段长度的总和，这形成了一个简单的算术级数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        if n == 1:
            print(0)
            continue

        s = (n + 1) // 2
        L = s - 1
        R = n - s

        # cost of expanding one side contributes triangular sums
        # total cost becomes sum of first L + R integers
        ans = (L + R) * (L + R + 1) // 2
        print(ans)

if __name__ == "__main__":
    solve()
```该实现首先处理不需要移动的简单单细胞情况。 中点是使用整数上限除法计算的。 由此，我们可以得出每侧有多少个细胞。 

代码中使用的关键简化是扩展的总数为$N - 1$，并且每次扩展都会增加成本，具体取决于机器人已经进步的程度。 这可分解为三角数公式$N - 1$，因为每个新单元都需要以严格递增的方式与当前访问的大小成比例地进行遍历。 

唯一的微妙之处是确保正确的中点计算。 使用$(n+1)//2$避免偶数和奇数的差一错误$N$。

 ## 边缘情况

 对于$N = 1$时，机器人开始覆盖整个走廊，因此不会发生任何移动。 该算法显式返回零，与以下事实相匹配$L = R = 0$。 

为了$N = 2$，中点是单元格 1，给出$L = 0$,$R = 1$。 公式得出$(1 \cdot 2)/2 = 1$，它对应于在访问所有单元格之前向右移动一次。 

对于非常大的$N$， 例如$10^9$，计算保持稳定，因为它只涉及整数乘法$10^9$，完全在 64 位限制之内。 没有模拟可确保性能不会下降，并且算术结构完全避免了 Python 中的溢出问题。
