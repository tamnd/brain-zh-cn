 ## 解决方案
 ## 问题理解

 我们在网格上得到两行水平的点。 一队站在位置的底部边缘$(1,0)$通过$(n,0)$，另一队则站在正上方$(1,n)$通过$(n,n)$。 底部的每个玩家都会向相应的列笔直向上发射垂直激光。 

这两排之间有水平墙段。 每面墙都有固定的高度$y$并跨越 x 坐标区间$[x_1, x_2]$。 这些墙不重叠，也从不相互接触。 

关键机制是当垂直激光击中墙段时会发生什么。 光束不会简单地停止。 相反，它在命中点处被切割，并被从该段的两个端点开始的两个新光束取代，并在相同的垂直方向上继续向上。 这会产生级联效果，其中单个光束在与上方的多个墙壁相互作用时可以分裂成多个光束。 

目标是确定在所有可能的分裂事件应用于所有初始后，至少一束激光到达了多少个不同的顶排玩家$n$横梁。 

这些约束意味着总$n$跨测试用例最多$2 \cdot 10^5$，并且墙的数量最多为$n$。 任何明确模拟光束分裂的解决方案都会太慢，因为一根光束可以重复分支，可能会在密集配置中产生二次或更糟糕的行为。 预期的解决方案必须将重复分裂的效果压缩为可以在近线性时间内有效更新和查询的结构。 

一个微妙的边缘情况来自于墙壁的排列，使得光束重复地分裂到多个层次，即使从单柱开始，也可能覆盖顶部的广泛范围。 幼稚的模拟可能会错误地假设每根光束停留在其柱内或每面墙仅分裂一次，这将错过递归传播。 

## 方法

 直接模拟开始于$n$垂直梁并逐层处理。 每次梁撞击墙段时，我们都会在端点处用两根梁替换该梁。 在最坏的情况下，单个光束可以击中每面墙壁，并且每次击中都会使活动光束的数量加倍。 和$q$墙壁，这导致梁的数量呈指数增长，总加工成本很快变得不可行。 

失败点在于我们正在跟踪单个光束，而许多光束在它们下一步去向方面是多余的。 连续 x 范围内的所有光束都表现相同：它们总是击中相同的相邻墙壁结构并产生相同的传播模式。 这表明我们应该停止单独跟踪光束，而是跟踪哪些 x 坐标间隔是“活动的”。 

关键的观察结果是，墙只会创建或破坏相邻活动段之间的边界。 每面墙有效地将一个片段分割成最多两个独立的片段，并且这些片段随后独立进化。 这意味着我们可以将 x 轴动态划分为活动间隔，并模拟这些间隔如何通过按高度排序的墙壁演变。 

我们按升序处理墙壁$y$。 在每面墙上，我们查看其间隔内存在多少个活动梁$[x_1, x_2]$。 该间隔中的每个活动光束都会在端点处产生新的光束，因此如果该间隔以前是活动的，则效果相当于激活端点。 由于墙壁不重叠或共享端点，因此这些更新永远不会以不明确的方式进行干扰。 

为了有效地维护活动段并计算哪些顶部位置可以到达，我们可以使用差异数组或段激活结构。 该过程简化为标记范围并向上传播端点激活。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(2^q) 最坏情况 | O(n) | 太慢了 |
 | 间隔传播（扫描 + DSU/阵列）| O(n + q) | O(n) | 已接受 |

 ## 算法演练

 我们将这个过程重新解释为沿 x 坐标传播可达性，其中墙壁仅在其端点引入新的可达性来源。 

1. 通过增加 y 坐标对所有墙进行排序。 

这确保了我们模拟梁的向上运动的顺序与它们物理上遇到墙壁的顺序相同。 
2. 维护数组`reach[i]`指示是否位置$i$底行可以生成最终向上穿过结构的梁。 

最初，每个底部位置都是活跃的，因为每个玩家都会发射光束。 
3. 对于每面墙$[x_1, x_2]$，确定该段是否“激活”，即当前至少有一个光束通过它。 

如果没有光束到达该段，则对系统没有影响。 
4. 如果该段已激活，则标记两个端点$x_1$和$x_2$作为主动源。 

这模拟了分裂规则：任何撞击该段的光束都会在两个端点处产生继续向上的光束。 
5. 继续按顺序处理墙，允许新激活的端点为后面的墙做出贡献。 
6. 最后，计算有多少个顶级位置可以从至少一个活动源到达。 

关键的减少是，我们不模拟光束路径，而是仅传播由可到达的墙段引起的端点的激活。 

### 为什么它有效

 该系统完全取决于哪些 x 位置可以将光束发送到更高的水平。 光束仅在撞击墙壁时才会改变行为，并且该事件仅取决于其 x 坐标是否位于墙壁段内。 由于同一 x 位置的所有光束行为相同，并且墙壁从不重叠，因此该过程分解为独立的区间激活。 每个分割仅引入端点，并且不会在现有片段之外创建新的内部行为。 这保证了跟踪端点激活足以重建所有未来的波束路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    walls = []
    for _ in range(q):
        x1, x2, y = map(int, input().split())
        walls.append((y, x1, x2))
    
    walls.sort()
    
    active = [False] * (n + 1)
    for i in range(1, n + 1):
        active[i] = True
    
    changed = True
    
    while changed:
        changed = False
        for y, l, r in walls:
            # check if any active point exists in [l, r]
            if any(active[i] for i in range(l, r + 1)):
                if not active[l]:
                    active[l] = True
                    changed = True
                if not active[r]:
                    active[r] = True
                    changed = True
    
    print(sum(active[1:]))

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该实现直接遵循通过墙端点传播激活的思想。 数组`active`表示当前哪些 x 位置生成可以继续向上的光束。 对于每面墙，我们扫描其间隔以查看它是否被任何活动光束击中。 如果是这样，我们激活两个端点。 

重复的`changed`循环是必要的，因为新激活的端点可能使先前不活动的墙变得相关。 这反映了问题中描述的级联分裂行为。 

一个微妙的实现问题是范围检查`any(active[i] for i in range(l, r + 1))`，每面墙都是线性的，如果不小心实施，可能会变得很慢。 预期的解决方案将用线段树或差分结构替换它，但逻辑保持不变：检测活动集和墙间隔之间的交集。 

## 工作示例

我们在每次迭代后跟踪活跃位置。

 这显示了所有位置仍然可达的情况，因此最终答案是 5。 

现在考虑$n = 5$, 墙壁$[2,3]$和$[4,5]$，但初始激活仅在 2 处。 

| 步骤| 活动集| 墙面处理| 效果|

 ## 边缘情况

 例如，一种边缘情况是一堵几乎覆盖整个范围的墙$n = 5$， 墙$[1,5]$。 每根光束都会撞击这堵墙，因此两个端点都会变得活跃。 由于在这种情况下端点已经处于活动状态，因此不会发生任何变化，答案仍然是 5。将分裂视为删除内部光束的简单模拟可能会错误地减少可到达的位置。 

另一个边缘情况是一连串嵌套的墙，例如$[1,3]$,$[2,4]$,$[3,5]$。 进入中间区域的光束会触发重复的端点激活，所有位置最终都会激活。 任何在没有传播顺序的情况下独立处理墙的方法都无法捕获这种闭合效应。 

第三种情况是在 x 范围内分隔的隔离墙。 例如$[1,2]$和$[4,5]$。 它们独立运行并且不共享激活，因此最终的可达集只是它们端点闭包的并集。 正确的解决方案必须避免由于全局传播假设而意外合并它们。
