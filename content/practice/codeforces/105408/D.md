 ## 解决方案
 ## 问题理解

 我们得到了一个排列，描述了雪貂如何在圆形排列的位置上移动。 在任何时刻，圆上的每个位置都恰好被一只雪貂占据，并且应用排列会将所有雪貂同时推进到下一个位置。 重复此操作形成一个离散时间动力系统，其中每个雪貂沿着排列周期独立移动。 

对于每个查询，我们都会被询问两只特定的雪貂是否会在某轮的_start_处的圆上相邻。 由于“一轮的开始”意味着在应用排列一定次数之后，问题是是否存在时间步$k \ge 0$这样在应用排列之后$k$有时，所选的两只雪貂的位置占据了圆上相邻的索引。 

关键的微妙之处在于邻接是在标签的固定圆上定义的$1$通过$n$，而运动是由排列定义的。 我们不是沿着排列周期跟踪距离，而是询问在相同排列幂下的图像是否落在两个连续的模标签上$n$。 

这些约束迫使我们为每个测试用例提供接近线性的解决方案。 总和为$n$和$q$所有测试用例最多$5 \cdot 10^5$，所以任何二次解$n$甚至$q \log n$每个查询都会失败。 我们应该期望这样的结构，每个测试用例预处理一次排列，并在恒定或接近恒定的时间内回答每个查询。 

简单的模拟会立即失败。 每个雪貂都位于排列的一个循环中，循环长度可以大到$n$，因此模拟至$2024!^{2024!}$步骤是不可能的。 如果我们尝试直接同步两只雪貂，即使将问题简化为循环周期，仍然会给我们留下潜在的巨大的最小公倍数。 

第二个天真的想法是模拟每个查询的完整排列周期并记录所有邻接事件。 那将花费$O(n^2)$每个测试用例在最坏的情况下都无法幸免于约束。 

## 方法

 关键的观察是排列分解为独立的循环。 每只雪貂沿着其周期移动，每步有固定的偏移量。 如果我们确定开始时间$k$，雪貂的位置完全由其周期内的索引加上$k$对周期长度取模。 

这将问题转化为两个循环行走之间的同步问题。 对于雪貂$a$和$b$，假设它们处于循环中$A$和$B$长度$L_A$和$L_B$。 时$k$，它们在周期内的位置移动$k \bmod L_A$和$k \bmod L_B$。 我们想知道是否存在一个时间$k$使得它们的实际标签形成固定圆的边缘。 

我们不从时间角度思考，而是颠倒视角。 考虑圆上的固定边缘，例如位置之间$u$和$v$。 为了$u$和$v$同时被占据$a$和$b$在某个时间$k$，偏移量必须满足两个模方程。 这就创建了一个同余系统$k$，当且仅当某个留数条件以循环长度的最大公约数为模时，该问题才可解。 

这将问题简化为检查是否存在至少一对相邻的圆$(u, v)$使得它们的周期偏移的差异与$a$和$b$, 取模$\gcd(L_A, L_B)$。 

暴力方法会尝试每个查询的所有圆边。 这太慢了，因为有$O(n)$边缘和$O(q)$查询。 

关键的结构见解是每个圆边连接两个排列循环，对于每一对这样的循环，我们只需要存储它们的连接边中出现的偏移差的残差类别。 由于圆的每个原始边都对这样的对贡献一次，因此存储的记录总数与$n$。 

这让我们可以将所有边预处理为由循环对索引的映射，并为每对存储一组以相应 gcd 为模的可行残差。 然后，每个查询都会减少为该结构中的一次查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(n \cdot q)$|$O(n)$| 太慢了 |
 | 循环对残基预处理 |$O(n + q)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 将排列分解为循环，并为每个节点分配其循环标识符及其在该循环内的位置。 

这是必要的，因为重复应用排列下的移动只是循环内的模算术。 
2. 迭代圆的每个邻接边，包括之间的边$n$和$1$。 

每条边连接两个节点$u$和$v$，属于某些循环$C_u$和$C_v$。 
3. 对于每个这样的边，计算 gcd$g = \gcd(|C_u|, |C_v|)$。 

该值确定两个独立循环移位何时可以对齐，因为两个循环可能以不同的周期重复。 
4. 计算残值$$r = (pos[u] - pos[v]) \bmod g$$并将其存储在由该对索引的结构中$(C_u, C_v)$。 

这表明该特定边缘对于与两个周期一致的某些时间对齐是可以实现的。 
5. 对于每个查询$(a, b)$，确定它们的周期$C_a$和$C_b$, 计算$g = \gcd(|C_a|, |C_b|)$，并计算$$\delta = (pos[a] - pos[b]) \bmod g.$$6. 通过检查是否回答查询$\delta$出现在该对的存储残基中$(C_a, C_b)$。 如果存在则输出1，否则输出0。 

### 为什么它有效

 每个周期的行为就像一个模块化时钟，每一步都会将位置前进一位。 仅当一次时，两只雪貂才会与一对固定的圆圈位置对齐$k$同时满足两个模块化约束。 当约束条件与循环长度的 gcd 模一致时，这样的系统就有精确的解。 

每个圆邻接边都编码一个这样的约束。 存储每个循环对的所有约束准确地保留了可实现的对齐集，并且不可能有其他交互，因为排列循环独立演化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def find_cycles(p):
    n = len(p) - 1
    vis = [False] * (n + 1)
    cid = [0] * (n + 1)
    pos = [0] * (n + 1)
    cycles = []

    for i in range(1, n + 1):
        if vis[i]:
            continue
        cur = []
        v = i
        while not vis[v]:
            vis[v] = True
            cid[v] = len(cycles)
            pos[v] = len(cur)
            cur.append(v)
            v = p[v]
        cycles.append(cur)

    return cid, pos, cycles

def solve():
    T = int(input())
    out_lines = []

    for _ in range(T):
        n, q = map(int, input().split())
        p = [0] + list(map(int, input().split()))

        cid, pos, cycles = find_cycles(p)
        cyc_len = [len(c) for c in cycles]

        store = {}

        def add_edge(a, b):
            ca, cb = cid[a], cid[b]
            if ca == cb:
                return
            if ca > cb:
                ca, cb = cb, ca
                a, b = b, a

            g = gcd(cyc_len[ca], cyc_len[cb])
            key = (ca, cb)
            if key not in store:
                store[key] = set()
            store[key].add((pos[a] - pos[b]) % g)

        from math import gcd

        for i in range(1, n + 1):
            j = i + 1 if i < n else 1
            add_edge(i, j)

        res = []
        for _ in range(q):
            a, b = map(int, input().split())
            ca, cb = cid[a], cid[b]

            if ca == cb:
                res.append('1')
                continue

            if ca > cb:
                ca, cb = cb, ca
                a, b = b, a

            g = gcd(cyc_len[ca], cyc_len[cb])
            key = (ca, cb)

            if key in store:
                delta = (pos[a] - pos[b]) % g
                res.append('1' if delta in store[key] else '0')
            else:
                res.append('0')

        out_lines.append(''.join(res))

    print('\n'.join(out_lines))

if __name__ == "__main__":
    solve()
```该代码首先将排列分解为循环，为每个节点分配一个循环 id 及其在该循环内的索引。 这种表示形式使得时间演化可以简化为模算术。 

这`add_edge`函数处理原始圆上的每个邻接。 它规范化循环排序，以便每个循环对都有一个规范密钥，计算循环长度的 gcd，并存储观察到的残差。 这是记录邻接信息的唯一位置。 

在查询期间，如果两个节点位于同一周期中，则可以轻松实现邻接，因为单个周期随时间统一覆盖所有状态。 否则，我们计算所需的残差并检查预先计算的集合中的成员资格。 

## 工作示例

 考虑一个小情况，其中排列分成两个通过圆边相互作用的循环。 我们跟踪边缘如何引起残差约束。 

| 步骤| 边 (u,v) | 循环（Cu、Cv）| 克| 添加残留物|
 | --- | --- | --- | --- | --- |
 | 1 | (1,2) | (A,B) | 计算| r1 |
 | 2 | (2,3) | (B,C) | 计算| r2 |
 | 3 | 查询 (a,b) | (A,C)| g'| 三角洲 |

 该表显示了每条边如何独立地对循环对约束做出贡献，并且查询只是测试这些约束之间是否存在所需的对齐。 

这表明邻接不依赖于全局模拟，而仅依赖于边缘引起的局部约束。

 ## 边缘情况

 一个关键的边缘情况是两个被查询的雪貂位于相同的排列周期中。 在这种情况下，它们围绕该周期同步移动，并且由于周期的每个状态都是可到达的，因此邻接仅取决于该周期是否将它们放置在相邻的圆位置上。 该算法通过在循环标识符匹配时立即返回 true 来处理此问题，从而避免不必要的 gcd 计算。 

当没有圆边连接两个循环时，会出现另一种微妙的情况。 这意味着它们之间不存在邻接约束，因此不可能进行同步。 该算法通过检查映射中是否存在循环对密钥来正确处理此问题。 

最后的边缘情况是之间的环绕邻接$n$和$1$。 将圆视为线性将完全错过此连接，但算法明确包含此边缘，确保约束结构完整。
