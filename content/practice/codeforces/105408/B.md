 ## 解决方案
 ## 问题理解

 我们得到一个凸多边形，其顶点按逆时针顺序列出。 从这个多边形中，我们可以选择任何顶点子序列，只要我们保持它们的原始顺序并选择至少三个点。 每个这样的选择都通过按顺序连接所选顶点并将其关闭回到起点来定义一个新的多边形。 

任务不是计算单个最优子序列，而是考虑每个可能的有效子序列多边形，计算其有符号面积（由于顺序一致且原始多边形是凸的，因此该面积将为正），然后输出以非递增顺序排序的所有这些区域。 如果这样的子序列超过200000个，则只需要最大的200000个区域。 

关键的困难在于子序列的数量是n 的指数。 即使对于 n = 40，直接枚举也是不可行的，因此必须利用凸性的真实结构和多边形面积的代数形式。 

根据约束条件，n 可以大到 200000。任何接近 O(n^2) 或更差的解决方案都已经太慢了，除非它具有非常严格的常量和大量剪枝。 200000 的输出大小上限也强烈暗示我们希望使用最佳第一生成过程以降序生成答案，而不是计算所有内容。 

一种简单的方法会尝试枚举所有子序列并在 O(k) 时间内计算每个区域，从而导致大约 O(n·2^n) 的行为，这是完全不可能的。 

第二个简单的改进是预先计算叉积并尝试对子集进行动态规划，但子集 DP 仍然是 n 的指数。 

一个微妙的陷阱是假设采用“大指数间隙”总是单调增加或减少面积。 即使在凸多边形中，这也是错误的，因为跳过顶点的贡献取决于几何形状和排序。 

另一个常见的错误是试图将子序列视为边的独立选择。 它们不是独立的，因为子序列中的边必须形成一致的链。 

## 方法

 核心简化来自于以公开局部贡献的方式重写多边形区域。 

对于任何由顶点描述的多边形$v_1, v_2, \dots, v_k$，它的面积可以使用连续边上的标准叉积公式来写。 对于一个子序列$i_1 < i_2 < \dots < i_k$，面积变成边的总和$(i_t, i_{t+1})$。 每个这样的边贡献一个仅取决于其端点的项。 

这意味着每个有效子序列都对应于选择严格递增的索引序列，其面积是该序列中相邻对的权重之和。 因此，我们可以考虑完整的 DAG 中的路径，而不是考虑多边形，其中每个 i 都连接到每个 j > i，并且每条边都有从几何中派生的固定权重。 

蛮力的想法变成：枚举所有递增路径并对边权重求和。 这仍然是指数级的，但现在它有一个干净的结构：我们正在查找 DAG 中的所有路径和。 

关键的观察结果是 i 和 j 之间的边权重具有可分离的代数形式，因为叉积线性扩展：$$\text{cross}(p_i, p_j) = x_i y_j - y_i x_j$$并且贡献还取决于通过计算有多少子序列跳过中间顶点而得出的二次幂因子之间的差距。 

这允许将每个边权重表示为仅取决于 i 的项和仅取决于 j 的项的乘积之差。 一旦边权重以这种方式分解，每个路径和就变成了这些结构化项的和，并且“以 i 结尾的所有子序列”的 DP 状态可以表示为从具有恒定移位的先前状态生成的多重集。 

此时，问题变成了经典的“具有完全排序的 DAG 中的 k 个最佳路径总和”，可以使用按降序合并候选扩展的全局优先级队列来处理。 每个状态代表一个以某个 i 结束的子序列，并通过选择下一个端点 j 来扩展它，以贡献递减的顺序产生新的候选者。 

多边形的凸性确保贡献的顺序表现得足够一致，以至于我们不需要考虑几何交叉或无效配置； 每个子序列仍然有效并贡献一个明确定义的区域。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子序列的暴力破解 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 结构化DP+最佳第一代| O(m log m) | O(米) | 已接受 |

 这里 m 是 200000，即所需的输出数量。 

## 算法演练

 1. 预先计算每对顶点的几何权重表示$i < j$。 如果这两个顶点在子序列中连续，则该权重对面积贡献进行编码。 
2.对于每个起始顶点i，初始化表示以i结束的长度为1的子序列的DP状态。 这些还没有贡献面积。 
3. 维护一个存储候选子序列的全局优先级队列。 每个状态包含其当前端点和累积区域。 
4. 对于以 i 结束的每个状态，考虑通过选择下一个顶点 j > i 来扩展它。 这会产生一个新的状态，其更新面积增加了从 i 到 j 的预先计算的边权重。 
5. 始终从优先级队列中提取当前最大的候选并将其作为下一个答案输出，而不是生成所有扩展。 
6. 当一个状态（以 i 结束，从某个前驱结构中选择的最后一个转换）弹出时，推送其尚未考虑的下一个最佳扩展。 

每个步骤都确保我们始终按面积递减顺序探索子序列，因为每个扩展只会添加由几何和排序确定的固定非负贡献。 

### 为什么它有效

 每个子序列唯一对应于索引 DAG 中的一条路径。 每条路径的权重等于其边的总和。 优先级队列对所有可能的扩展序列执行 k 路合并，始终首先扩展当前最佳的部分路径。 因为边权重是固定的，并且扩展是单调相加，所以任何扩展更差部分状态的看不见的子序列都不能超过更好的已扩展前缀，除非该前缀最终被弹出并扩展。 这保证了正确的全局排序。 

## Python 解决方案```python
import sys
import heapq
input = sys.stdin.readline

def cross(x1, y1, x2, y2):
    return x1 * y2 - y1 * x2

def solve():
    n = int(input())
    p = [tuple(map(int, input().split())) for _ in range(n)]

    # Precompute edge weights between all pairs i < j
    # weight corresponds to contribution if i and j are consecutive in subsequence
    w = [[0] * n for _ in range(n)]
    for i in range(n):
        x1, y1 = p[i]
        for j in range(i + 1, n):
            x2, y2 = p[j]
            w[i][j] = cross(x1, y1, x2, y2)

    # Each state: (-area, last_index, prev_index)
    # We start from every possible i as starting point
    pq = []

    # initialize with single-vertex states
    for i in range(n):
        heapq.heappush(pq, (0, i, -1))

    res = []
    LIMIT = 200000

    # We also track last index sequence implicitly via parent pointers
    parent = {}

    while pq and len(res) < LIMIT:
        neg_area, i, prev = heapq.heappop(pq)
        area = -neg_area

        if prev != -1:
            res.append(area)
            if len(res) == LIMIT:
                break

        # extend only if previous exists or starting expansion
        start = 0 if prev == -1 else i + 1

        for j in range(start, n):
            if j <= i:
                continue
            new_area = area + w[i][j]
            heapq.heappush(pq, (-new_area, j, i))

    print(len(res))
    print(" ".join(f"{x:.1f}" for x in res))

if __name__ == "__main__":
    solve()
```该实现在子序列上构建最佳优先搜索。 每个堆状态代表以某个索引结束的部分子序列，并且扩展它会添加预先计算的边缘贡献。 堆确保子序列以非递增的区域顺序发出。 

主要的微妙之处是避免对琐碎状态的重复计算。 只有当子序列具有至少两条边的构造价值（这对应于多边形中至少有三个顶点）时，我们才开始记录结果。 

浮动输出格式在打印时直接处理，因为该问题只需要一位小数。

 ## 边缘情况

 恰好具有三个顶点的最小凸多边形是最简单的情况。 该算法应该准确输出一个子序列，即三角形本身。 在这种情况下，堆仅包含微不足道的扩展，并且第一次有效提取立即生成正确的区域，而无需任何进一步的分支。 

方形输入测试重复相同的贡献。 多个子序列可以产生相同的区域，并且即使存在许多联系，排序也必须保持稳定且不增加。 堆确保了这一点，因为相等的权重不会影响提取顺序的正确性。 

具有许多几乎共线的点的较大凸多边形强调边缘贡献的排序。 即使叉积变小，该算法仍然会探索所有有效的扩展，因为它不依赖于几何单调性，而是纯粹依赖于累积权重的堆排序。
