 ## 解决方案
 ## 问题理解

 我们得到了一个加权的、无向的、连通的城市图。 Jorge 住在城市 0。每条边代表一条正长度的道路，最短路径定义任意两个城市之间的距离。 

Jorge 有 k 个朋友，每个朋友独立占据一个不同的城市。 我们不知道他们占领了哪些城市； 相反，我们考虑 k 个不同城市与剩余 n-1 个城市的每一种可能选择，所有可能性都是相同的。 对于每个这样的选择，我们使用最短路径计算从城市 0 到每个选定城市的距离，然后取这 k 个距离中的最小值。 任务是计算这个最小距离的期望值。 

关键的困难在于我们对非线性统计量进行平均，即随机子集上的最小值，而不是对独立距离求和。 

这些约束将我们推向几乎线性算术图解。 当 n 达到 100000，m 达到 1e6 时，任何根据选择重新计算最短路径的方法都是不可能的。 即使存储所有成对距离也是不可行的。 我们必须计算一次距源 0 的所有距离，然后对排序值进行组合推理。 

当多个城市共享与 0 相同的最短距离时，就会出现微妙的边缘情况。例如，如果多个城市的距离为 5，则最小值的概率结构会逐步变化，如果我们没有正确地将相等距离分组，则对各个城市进行简单平均会失败。 另一种边缘情况是当 k 非常接近 n − 1 时。在这种情况下，最小值几乎肯定是几乎所有节点中的最小距离，并且组合概率仍然必须表现正确，特别是对于从不影响最小值的最远节点。 

## 方法

 强力解释将枚举 n − 1 中 k 个城市的所有子集，计算每个子集的最小距离，并对结果求平均值。 即使忽略最短路径计算，子集的数量为$\binom{n-1}{k}$，即使对于中等的 n，它也会变得天文数字般大。 这立即排除了直接枚举。 

我们首先将图形组件与组合组件分开。 可以使用 Dijkstra 算法计算从城市 0 到其他所有城市的距离$O(m \log n)$。 之后，问题就变成了纯粹的概率问题：给定一个 n − 1 距离的数组，我们统一选择 k 个元素而不进行替换，并希望得到预期的最小值。 

关键的观察是根据生存概率重新构建期望。 我们不直接询问最小值是多少，而是考虑最小值至少是阈值 d 的概率。 当所有 k 个选定的城市都位于距离 ≥ d 的城市之间时，就会发生这种情况。 如果我们对所有距离进行排序，我们可以计算有多少个城市的距离 ≤ d 的前缀计数，并从中推导出有多少个城市的距离 ≥ d。 

这会将期望转换为排序的唯一距离的总和，其中每个值根据最小值等于该值的概率做出贡献。 这个概率用组合来表达：我们统计选择距离阈值更近的节点集之外的 k 个城市的方式数量，并类似地减去下一个阈值。 整个结构简化为排序距离上的前缀组合。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集的暴力破解 | O(C(n, k) · k + m log n) | O(C(n, k) · k + m log n) | O(n) | 太慢了 |
 | Dijkstra + 排序距离上的组合数学 | O(m log n + n log n) | O(m log n + n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将解决方案分为图预处理阶段和组合期望阶段。 

1. 从城市 0 运行 Dijkstra 以计算到所有城市的最短距离。 

这是必要的，因为以后的每次计算仅取决于这些距离，并且一旦计算出最短距离之外的任何路径结构都是无关紧要的。 
2. 将城市 0 排除在考虑范围之外，因为它是源城市并且不能被选为友邦城市。 
3. 将所有距离存储到一个数组中，并按非降序排序。 

需要排序是因为随机子集上的最小值最容易根据从最小到最大的阈值来推理。 
4. 预先计算模 1e9+7 的二项式系数的阶乘和逆阶乘直至 n。 

我们需要快速计算$\binom{a}{b}$因为概率取决于子集的计数。 
5. 定义一个函数 C(a, b) 如果 b > a 返回 0，否则计算$\frac{a!}{b!(a-b)!}$。 

该函数在概率表达式中重复使用。 
6. 迭代排序距离数组中的每个索引 i，将 dist[i] 视为最小值可能首先出现的候选值。 

对于固定的 i，我们计算有多少个大小为 k 的子集的最小值恰好等于 dist[i]。 
7. 设 i+1 为距离 ≤ dist[i] 的城市数量。 严格接近或等于此阈值的城市数量决定了有多少个选择会使 dist[i] 作为最小值无效。 
8. 计算从索引 i..n-1 中选择 k 个城市的方法数（所有距离 ≥ dist[i] 的城市）。 

这是 C(n-1 - i, k)。 
9. 从索引 i+1..n-1 中减去选择 k 个城市的方式数，该索引对应于最小值严格大于 dist[i] 的子集。 

这给出了 dist[i] 恰好是最小值的子集数。 
10.将此概率权重乘以dist[i]并累加为最终答案。 

### 为什么它有效

 该算法根据距城市 0 的最小距离对城市的所有 k 个子集进行划分。每个子集在排序顺序中恰好有一个最小元素，并且组合差值 C(n-1 - i, k) − C(n-1 - i-1, k) 精确计数按排序顺序第一个选择的元素是位置 i 的子集。 这确保了每个子集恰好对一个距离桶贡献一次，因此加权和恰好是最小距离的期望。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def dijkstra(n, adj):
    import heapq
    dist = [10**30] * n
    dist[0] = 0
    pq = [(0, 0)]
    while pq:
        d, u = heapq.heappop(pq)
        if d != dist[u]:
            continue
        for v, w in adj[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

def modinv(x):
    return pow(x, MOD - 2, MOD)

def nCk(n, k, fact, invfact):
    if k < 0 or k > n:
        return 0
    return fact[n] * invfact[k] % MOD * invfact[n - k] % MOD

def solve():
    n, m, k = map(int, input().split())
    adj = [[] for _ in range(n)]
    for _ in range(m):
        a, b, w = map(int, input().split())
        adj[a].append((b, w))
        adj[b].append((a, w))

    dist = dijkstra(n, adj)
    arr = dist[1:]
    arr.sort()

    maxn = n + 5
    fact = [1] * maxn
    invfact = [1] * maxn
    for i in range(1, maxn):
        fact[i] = fact[i - 1] * i % MOD
    invfact[maxn - 1] = modinv(fact[maxn - 1])
    for i in range(maxn - 2, -1, -1):
        invfact[i] = invfact[i + 1] * (i + 1) % MOD

    total = nCk(n - 1, k, fact, invfact)

    ans = 0
    n1 = n - 1

    for i, x in enumerate(arr):
        if n1 - i < k:
            continue
        # subsets where minimum >= x
        ge = nCk(n1 - i, k, fact, invfact)
        # subsets where minimum > x
        if i + 1 <= n1:
            gt = nCk(n1 - (i + 1), k, fact, invfact)
        else:
            gt = 0
        cnt = (ge - gt) % MOD
        ans = (ans + cnt * x) % MOD

    inv_total = modinv(total)
    print(ans * inv_total % MOD)

if __name__ == "__main__":
    solve()
```实现首先构建邻接表并使用 Dijkstra 和堆计算单源最短路径。 这产生了概率阶段所需的所有距离。 

由于需要重复使用二项式系数，因此阶乘和逆阶乘数组预先计算一次。 模逆是使用费马定理计算的，这是安全的，因为模数是素数。 

数组`arr`包含从节点 0 到所有其他节点的距离，经过排序以便我们可以推断阈值。 对于每个位置`i`，我们计算有多少个 k 子集的最小值恰好等于`arr[i]`使用两个二项式计数的差值。 这避免了显式跟踪子集。 

最后，我们使用模逆将累积加权和除以 k 子集总数，从而产生期望值。 

一个常见的实施陷阱是忘记城市 0 被排除在选择池之外。 另一种是错误地对待等距离而不进行分组； 组合差异公式自动避免了这个问题。

 ## 边缘情况

 一种边缘情况是当 k = n − 1 时。在这种情况下，每个节点都会被选择，因此最小值始终是最小距离。 该算法处理这个问题是因为除了第一个索引之外的所有 i 的 C(n − 1 − i, k) 都变为零，只留下最小的距离起作用。 

另一个边缘情况是重复距离。 假设多个节点具有相同的最短路径值。 独立处理每个节点的简单方法会重复计算概率质量，但减法公式 C(n − 1 − i, k) − C(n − 1 − i − 1, k) 会自动将相等的值折叠为正确的频率加权贡献。 

最终的边缘情况是 k = 1。那么答案就是所有最短路径距离的平均值。 该算法正确退化，因为每个节点都是以相等的概率选择的，并且组合差异减少到所有距离上的统一权重。
