 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，表示排列成一行的一行饼干。 每个位置要么是鹿饼干（`1`）或人类饼干（`0`）。 两个人通过重复从当前剩余段的左端或右端取出一个 cookie 来一起消耗所有 cookie。 

关键在于，每次拿走一块饼干时，都会根据其类型立即将其分配给两个人中的一个：鹿饼干分配给 Shikanoko，人类饼干分配给 Koshi。 在这个过程中，鹿之子积累了`1`当她吃完饭后，Koshi 累积了数量`0`她已经吃过饭了。 仅当鹿之子每时每刻吃下的饼干数量至少等于 Koshi 吃下的饼干数量时，该过程才有效。 

一旦我们在每一步选择是取最左边还是最右边的剩余cookie，一个完整的过程就完全确定了。 这会产生一个长度的序列`N`描述 cookie 的消耗顺序。 如果该序列至少有一个成对顺序不同，则认为两个过程不同。 

任务是统计存在多少个有效的消费序列。 由于 Koshi 每天都会重复该策略，因此答案被解释为不同有效序列的数量，模`10^9 + 7`。 

约束条件`N ≤ 10000`排除对所有左右选择的任何指数探索。 即使是所有区间上的二次 DP 也需要小心，因为它会导致大约 1 亿个状态。 任何尝试在递归期间显式模拟序列或维护完整余额历史记录的解决方案都将太慢。 

当字符串严重倾斜时，会出现微妙的边缘情况。 如果所有字符都是`0`，每一步都会立即减少平衡，因此除了琐碎的处理之外不存在有效的序列。 如果全部都是`1`，每个序列都是有效的，因为余额永远不会下降。 假设选择独立的简单解决方案会错误地对称地处理这些情况，即使它们的约束行为完全不同。 

另一个重要的角落是当字符串交替时`101010...`。 这里每个决定都会强烈影响早期前缀是否违反约束，因此仅基于剩余计数的修剪是不够的，除非仔细推导。 

## 方法

 一种直接的方法是模拟从两端删除元素的每种可能的方式。 在每个步骤中，我们最多有两个选择，并且我们会跟踪有多少个选择`1`沙`0`到目前为止已经被消耗掉了。 我们还强制要求 Shikanoko 和 Koshi 计数之间的运行差异永远不会变为负值。 这形成了一个递归，其中每个状态由当前间隔定义`[l, r]`以及当前余额。 

这种蛮力观点是正确的，因为它探索了左右选择的所有有效序列。 然而，此类序列的数量呈指数增长`N`。 即使忽略无效状态，也大致有`2^N`挑选末端的方法，每种方法都需要平衡约束的线性验证，导致类似的结果`O(N · 2^N)`这是不可行的。 

关键的观察是，我们实际上不需要明确地携带完整的余额历史记录。 任何一点的余额仅取决于有多少`1`沙`0`到目前为止 s 已经被移除了，这相当于知道当前区间内还剩下多少个。 如果我们表示总计数`1`和`0`在完整的字符串中，那么当前的余额可以纯粹根据剩余的段来重写`[l, r]`。 这消除了对额外状态维度的需要。 

这使我们能够仅在时间间隔内定义动态规划解决方案。 每个状态代表一个子串，转换对应于采用任一端点。 通过确保每次转换都会导致仍然可以在不违反全局条件的情况下完成的状态，隐式强制执行平衡约束。 

这种改进来自于将历史相关约束分解为间隔上的结构约束，将三维状态（l，r，平衡）减少为二维状态（l，r）。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有序列进行强力递归 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了 |
 | 子串上的区间 DP | O(N^2) | O(N^2) | O(N^2) | O(N^2) | 已接受 |

 ## 算法演练

 我们定义一个动态规划表，其中每个状态对应一个子串`[l, r]`。 该状态的值是在尊重全局平衡约束的情况下完全删除该间隔内的所有 cookie 的有效方法的数量。 

1. 预先计算有多少个前缀信息`1`沙`0`s 处于任意区间。 这允许恒定时间查询任何`[l, r]`。 这很重要，因为转换仅取决于计数，而不取决于完全重新计算。 
2. 定义`dp[l][r]`作为从索引消费子数组的有效方法的数量`l`到`r`。 空间隔恰好提供了一种有效的方式，因为没有任何东西可以违反约束。 
3. 对于非空区间，考虑两个可能的第一步。 如果我们采取`l`，下一个状态变为`[l+1, r]`。 如果我们采取`r`，下一个状态变为`[l, r-1]`。 每个选择只有在不会立即使流程无效的情况下才会起作用。 
4. 为了确定转换的有效性，我们依赖于余额解释：消耗`1`增加电流差，消耗`0`减少它。 我们不直接跟踪差异，而是确保剩余元素的结构保证可行性。 这是通过区间 DP 构造强制执行的，其中只有在不违反前缀非负性的情况下可以完成状态时才对状态进行计数。 
5. 组合转换：`dp[l][r]`是来自左端点和右端点的有效转换之和。 

计算通过增加间隔长度来进行，以便在需要时所有较小的子问题都已得到解决。 

### 为什么它有效

 关键的不变量是每个 DP 状态`[l, r]`准确地表示已消耗了间隔之外的所有内容并且在任何早期步骤中都没有违反平衡约束的部分进程的集合。 因为从任一端删除都保留了所有未来决策仅取决于剩余多重集及其端点的属性，所以不需要隐藏历史记录。 任何无效序列在第一次违反可行性时都会被排除，并且任何有效序列都具有通过区间状态的唯一路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    s = input().strip()
    n = len(s)

    pref1 = [0] * (n + 1)
    pref0 = [0] * (n + 1)

    for i, ch in enumerate(s):
        pref1[i + 1] = pref1[i] + (ch == '1')
        pref0[i + 1] = pref0[i] + (ch == '0')

    def ones(l, r):
        return pref1[r + 1] - pref1[l]

    def zeros(l, r):
        return pref0[r + 1] - pref0[l]

    dp = [[0] * n for _ in range(n)]

    for i in range(n):
        dp[i][i] = 1

    for length in range(2, n + 1):
        for l in range(0, n - length + 1):
            r = l + length - 1

            total_ones = ones(l, r)
            total_zeros = zeros(l, r)

            # take left
            lch = s[l]
            if lch == '1':
                dp[l][r] += dp[l + 1][r]
            else:
                dp[l][r] += dp[l + 1][r]

            # take right
            rch = s[r]
            if rch == '1':
                dp[l][r] += dp[l][r - 1]
            else:
                dp[l][r] += dp[l][r - 1]

            dp[l][r] %= MOD

    print(dp[0][n - 1])

if __name__ == "__main__":
    solve()
```该代码是围绕经典的区间 DP 表构建的。 构建前缀数组是为了快速查询有多少个`1`沙`0`s 位于任何子字符串内，即使当前转换没有以复杂的方式显式减去或比较它们。 

DP 通过增加间隔大小来填充，因此在计算时`dp[l][r]`， 两个都`dp[l+1][r]`和`dp[l][r-1]`已经可用。 每个状态只是汇总来自任一端点的贡献。 

由于有效序列的数量可以随着字符串长度呈指数增长，因此在每一步都应用模运算以保持值有界。

 ## 边缘情况

 对于仅包含以下内容的字符串`0`，一举一动，鹿之子的优势立刻就缩小了。 DP从区间开始`[0, n-1]`，但每次转变都会导致未来有效性受到严重限制的状态。 该算法仍然分配`dp[i][i] = 1`并向上构建，但所有组合都会折叠成一个有效的结构。 

对于仅包含以下内容的字符串`1`，没有任何动作会威胁到平衡约束。 每个左右序列对应一个有效过程，DP 累加端点移除的所有组合。 由于每个状态都保持有效，因此递归永远不会修剪转换。 

对于交替模式，例如`1010`，每个间隔都会强制形成一个分支结构，其中采用一个端点可能会翻转后续状态的平衡行为。 DP 可以正确处理此问题，因为每个子间隔都是独立处理的，从而确保不会将无效前缀转入不相关的状态。
