 ## 解决方案
 ## 问题理解

 我们有偶数个参与者，每个参与者都有一个代表其效率的非零整数值。 我们必须将它们配对，以便每个参与者都恰好在一对中使用。 配对的得分定义为该配对中两个值的乘积，目标是选择使所有配对乘积总和最大化的配对。 

关键的困难在于结对决策在全球范围内相互作用。 单个元素可以与任何其他元素配对，早期的糟糕选择可能会阻碍更好的全局配置。 由于值可以是负值或正值，因此符号之间的相互作用与大小一样重要。 

输入大小最多可达 10^6 个元素。 任何比 O(n log n) 更糟糕的解决方案都有超时的风险，甚至 O(n^2) 的构造也是完全不可行的。 内存限制还迫使我们避免存储超出线性空间的大型辅助结构。 

当涉及负数时，会出现微妙的边缘情况。 例如，将两个负数配对会产生正贡献，而将负数与正数配对会产生负贡献，从而减少总和。 将相邻元素或任意对配对的天真贪婪方法可能会严重失败。 

考虑输入：```
-5 -1 2 4
```像 (-5, 2) 和 (-1, 4) 这样的简单配对给出 -10 + -4 = -14。 

但是 (-5, -1) 和 (2, 4) 配对得到 5 + 8 = 13，这是最优的。 

这表明符号感知分组至关重要。 

另一种边缘情况是所有数字均为正数或全部为负数。 在这两种情况下，最佳配对的结构都得到了简化，但通用算法仍然必须统一处理它们。 

## 方法

 暴力方法会尝试所有可能的配对配置。 这可以看作是生成 n 个元素的所有完美匹配并计算每个元素的乘积之和。 此类匹配的数量为 (n - 1) × (n - 3) × ... × 1，并且呈超指数增长。 即使 n = 20，这也是不可行的。 

暴力破解之所以正确，是因为它会将所有有效分区成对探索并直接评估目标。 然而，它失败了，因为配对的数量以组合方式爆炸，并且没有修剪可以避免重新审视等效的结构选择。 

关键的观察是，一对的贡献仅取决于值本身，而不取决于它们的位置。 这表明我们应该自由地重新排序元素。 一旦排序，最佳配对的结构就变得确定了：大的正数应该配对在一起，大的负数应该配对在一起，除非强制，否则混合符号通常是有害的。 

为了形式化这一点，对数组进行排序表明，最佳策略是按排序顺序将相邻元素配对。 这是有效的，因为将任何交叉配对重新排列为非交叉排序对永远不会减少总和，这是标准交换参数。 直观上，配对极端可以稳定贡献：负面因素会积极地相互放大，而正面因素在分组时会保持大小。

 ## 算法演练

 1. 对数组进行非降序排序。 

这确保了具有相似符号和大小的元素是相邻的，这对于出现最佳配对结构是必要的。 
2. 将运行总和初始化为零。 
3. 以两步迭代已排序的数组，获取连续的对。 

每对 (a[i], a[i+1]) 相加为 a[i] × a[i+1]。 

这种配对规则是安全的，因为任何与邻接的偏差都会引入交叉结构，可以在不减少总和的情况下进行交换。 
4. 将所有配对产品累加为最终答案。 
5. 输出累计和。 

### 为什么它有效

 正确性依赖于配对的交换论证。 假设使用元素 a ≤ b ≤ c ≤ d 但以交叉方式形成两对：(a, c) 和 (b, d)。 替代配对 (a, b) 和 (c, d) 永远不会产生更小的总和，因为：

 在排序顺序下，(a·b + c·d) − (a·c + b·d) = (a − d)(b − c) ≥ 0。 

这表明任何交叉配对都可以局部改进为相邻配对，而不会减少总和。 重复应用此转换会消除所有交叉，从而形成按排序顺序配对的连续元素。 因此，贪婪的相邻配对是全局最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort()
    
    ans = 0
    for i in range(0, n, 2):
        ans += a[i] * a[i + 1]
    
    print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 排序是实现贪婪配对的关键预处理步骤。 循环以 2 为增量前进，以确保每个元素仅使用一次。 由于 Python 中整数的乘法是无界的，因此不存在溢出问题。 

一个微妙的实现细节是输入大小可以达到一百万个元素，因此使用 sys.stdin.readline 是必要的。 排序在复杂性中占主导地位，因此循环仍然是线性开销。

 ## 边缘情况

 对于仅包含负数的输入，例如`-4 -3 -2 -1`, 排序产量`-4 -3 -2 -1`并将相邻元素配对给出`( -4, -3 ) = 12`和`( -2, -1 ) = 2`，总计 14。任何替代配对（例如交叉配对）都会减少乘积和，因为幅度较近的配对总是会产生较少的负相互作用。 

对于混合符号输入，例如`-10 -1 2 3`，排序后的配对产生`(-10, -1) = 10`和`(2, 3) = 6`，共 16 个。交叉配对，如`(-10, 2)`和`(-1, 3)`产量`-20 + -3 = -23`，显示不正确的混合对目标的损害程度。 

对于具有交替符号的输入，例如`-2 1 -1 2`，排序为`-2 -1 1 2`确保算法首先对负数进行分组，避免符号交替的不稳定。
