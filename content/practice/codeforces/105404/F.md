 ## 解决方案
 ## 问题理解

 我们从一个大小为空的网格开始$n \times m$，其中每个单元格最初都是白色的。 修改网格的唯一方法是选择整行或整列并将它们涂成黑色。 一旦一行被绘制，该行中的每个单元格都会变成黑色； 这同样适用于列。 

执行任意数量的此类操作后，我们希望剩余的白细胞数量恰好为$k$。 每个测试用例都给我们$n$,$m$，以及这个目标值$k$，我们必须决定它是否可以实现。 

关于操作结构的一个关键观察结果是，最终配置仅取决于绘制哪些行和哪些列，而不取决于操作的顺序。 如果我们绘制一组行$R$和一组列$C$，然后每个单元格$R$或者$C$变黑。 剩余的白细胞恰好是两组之外的白细胞，形成一个干净的矩形大小块$(n - |R|) \times (m - |C|)$。 

这意味着白细胞的最终数量始终是以下形式$$(n - r)(m - c)$$对于某些整数$0 \le r \le n$,$0 \le c \le m$。 

所以问题不再是运算的问题，而是我们能不能代表的问题$k$作为与网格尺寸对齐的某些子矩形的面积。 

从约束条件来看，$t$最多可以达到 30000，所以每次测试必须非常快，理想情况下$O(1)$或者$O(\sqrt{k})$最坏的情况是。 自从$n$,$m$， 和$k$可能很大，任何涉及网格模拟或枚举操作的事情都是不可能的。 

常见的失败案例来自于“绘画序列”而不是最终几何的推理。 例如，人们可能错误地认为行和列操作之间的重叠会以一种复杂的方式改变计数。 实际上，重叠并不重要：如果选择了单元格的行或列，则单元格就是黑色的，因此结果纯粹是乘法。 

另一个微妙的错误是治疗$k$任意，但忘记它必须等于网格内的矩形区域。 例如，如果$n = 3$,$m = 3$， 和$k = 5$，没有办法形成$3 \times 3$网格区域正好有 5 个剩余的白色单元格，尽管 5 个单元格通过部分绘画直觉看起来“可以达到”。 

## 方法

 强力透视将模拟所有绘制的行和列的选择。 对于行和列的每个子集，我们计算白细胞的最终数量。 这立即呈指数增长$n + m$，因为有$2^n \cdot 2^m$可能的状态。 即使对于小网格，这也会爆炸，并且无法使用。 

关键的简化来自于认识到所有有效的最终状态仅取决于完全删除的行数和列数。 如果我们删除$r$行和$c$列，剩余的白色区域是一个完美的矩形大小$(n-r) \times (m-c)$。 这将问题转化为纯粹的因式分解检查：我们询问是否存在整数$x \le n$和$y \le m$这样$x \cdot y = k$。 

所以任务简化为寻找是否$k$有一个除数$x$适合在$n$，与相应的商$y = k/x$适合在$m$。 

我们不再需要考虑运算，只需要考虑除数$k$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（行/列子集）|$O(2^{n+m})$|$O(1)$| 太慢了|
 | 检查除数$k$|$O(\sqrt{k})$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 1.观察选择要绘制的行和列后，剩余的白色单元格形成了一个大小为$(n-r) \times (m-c)$。 这减少了选择两条边长的问题。 
2. 将任务重新表述为寻找整数$x, y$这样$x \cdot y = k$， 和$x \le n$和$y \le m$。 这里$x$表示剩余未绘制的行和$y$剩余未上漆的柱子。 
3. 迭代所有可能的除数$x$的$k$。 对于每个$x$，检查是否可除$k$干净地。 这确保我们只考虑有效的矩形尺寸。 
4. 对于每个有效除数$x$, 计算$y = k / x$，然后检查是否$x \le n$和$y \le m$。 
5. 如果没有这样的一对有效，则得出结论：$k$无法在网格内形成有效的矩形区域。 

### 为什么它有效

 每个有效的操作序列都准确对应于选择要删除的一组行和列，并且该选择总是产生一个矩形的剩余白色区域。 相反，每个大小的矩形$x \times y$网格内可以通过留下来获得$x$行和$y$柱子未上漆，其余部分上漆。 这在可行配置和因子对之间创建了一对一的对应关系$k$受网格尺寸的限制，因此检查所有除数既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def possible(n, m, k):
    x = 1
    while x * x <= k:
        if k % x == 0:
            y = k // x
            if x <= n and y <= m:
                return True
            if y <= n and x <= m:
                return True
        x += 1
    return False

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n, m, k = map(int, input().split())
        out.append("SI" if possible(n, m, k) else "NO")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```功能`possible`执行除数扫描直至$\sqrt{k}$。 对于每个除数$x$，它检查矩形的两个方向，因为$x$可以代表剩余的行或剩余的列。 

对称检查是必要的，因为我们事先不知道是否$x$应对应于$n$- 维度或$m$-方面。 缺少这种交换是错误答案的常见来源。 

## 工作示例

 ### 示例 1

 输入：```
2 50 81
```我们试图表达$81$作为$x \cdot y$和$x \le 2$,$y \le 50$。 

| x| y = 81/x | x ≤ 2 | y ≤ 50 | 有效 |
 | ---| ---| ---| ---| ---|
 | 1 | 81 | 81 是的 | 没有| 没有|
 | 3 | 27 | 27 没有| 是的 | 没有|
 | 9 | 9 | 没有| 是的 | 没有|

 没有因子对符合网格约束，因此答案是否定的。

 输入：```

```我们搜索 546 的约数。 

| x| y = 546/x | x ≤ 29 | y ≤ 22 | 有效 |
 | ---| ---| ---| ---| ---|
 | 2 | 273 | 273 是的 | 没有| 没有|
 | 3 | 182 | 182 是的 | 没有| 没有|

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(t \sqrt{k})$| 每个测试扫描除数$k$最多$\sqrt{k}$|
 | 空间|$O(1)$| 仅使用常量变量 |

 虽然$\sqrt{k}$理论上可以很大，操作是简单的整数算术，并且检查总数在实践中保持在 30000 个测试用例的典型限制内。

 ## 边缘情况

 一个极端的情况是$k = 1$。 尽管问题表明$k \ge 2$，对其进行推理澄清了正确性：仅$1 \times 1$矩形是有效的，这意味着网格必须允许至少有一个未触及的行和列。 

另一个微妙的情况是当$k$是素数。 例如，$k = 13$和$n = 3$,$m = 10$。 唯一的因式分解是$1 \times 13$，除非一维可以容纳 13，否则该算法会失败。该算法正确地拒绝所有其他值，因为不存在除 1 以外的除数。 

更有趣的情况是两种分解都存在但只有一种适合。 为了$n = 5$,$m = 20$,$k = 50$，我们有对$(1,50)$,$(2,25)$,$(5,10)$,$(10,5)$。 仅有的$(5,10)$和$(10,5)$是相关的，并且至少有一个满足约束，因此算法一旦遇到它就会立即接受。
