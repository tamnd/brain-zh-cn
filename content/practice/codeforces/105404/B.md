 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例都提供一个正整数列表。 任务是确定列表中是否存在至少一个元素可以以简单的方式划分同一列表中的每个元素（包括其自身）。 

更具体地说，对于每个数组，我们正在搜索一个索引$i$这样对于每个$j$，值$a_i$划分$a_j$无余。 如果这样的元素存在，我们回答“SI”，否则我们回答“NO”。 

输入大小迫使我们仔细考虑每个测试用例的效率。 最多可以有 100 个测试用例，每个测试用例最多可以包含 10,000 个数字。 一种简单的方法是检查每个候选元素与所有其他元素的整除性对，需要检查$n^2$每个测试用例的关系，在最坏的情况下达到$10^8$可分性检查。 这仍然是边界，但只有仔细实现才能传递优化的 Python，尽管这是不必要的。 

一个更微妙的约束来自取值范围：每个数字最多可达$10^9$。 这排除了任何频率数组或直接索引方法。 该结构是纯算术的。 

经常打破天真的推理的一种边缘情况是重复值。 例如，在像这样的数组中`[4, 2, 4, 6, 8, 4]`，值 4 出现多次并且是有效的候选者，即使它不会在所有位置整除 2 或 6，除非我们仔细检查所有元素。 另一种边缘情况是最小的数字不是答案。 例如`[2, 3, 6]`有 2 作为候选，但失败，因为 3 不能被 2 整除。 

第二种微妙的故障模式是假设最小元素始终有效。 为了`[2, 3, 4]`，最小值为 2，但 2 不能整除 3，因此正确答案为“NO”。 

最后，具有许多相等元素的数组很棘手。 如果所有数字都相同，则答案始终为“SI”，因为任何元素都可以整除所有其他元素。 

## 方法

 蛮力方法很简单。 对于数组中的每个元素，将其视为候选除数，并检查它是否整除所有其他元素。 这需要两个嵌套循环：一个用于选择候选者，另一个用于验证数组的整除性。 每次检查的时间都是恒定的，因此每个测试用例的复杂度是$O(n^2)$。 和$n = 10^4$，这导致$10^8$每个测试用例的操作，在多个测试用例中变得太慢。 

关键的观察是，有效的候选者必须除以每个元素，包括最大元素。 这立即意味着候选者必须小于或等于它所除的每个元素，因此它必须是数组中的最小值。 然而，除非我们明确地测试整除性，否则仅作为最小值在一般推理中是不够的。 

这极大地减少了搜索空间。 我们不需要测试每个元素，而只需测试数组中的最小值。 如果该值除以每个元素，则它是有效答案。 否则，没有其他元素可以工作，因为任何更大的候选元素都无法除以最小值本身。 

这将问题转化为一次扫描以找到最小值，然后进行第二次扫描以验证整除性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 对于每个测试用例，读取整数数组。 这将独立的场景分开，因此它们之间不会延续任何状态。 
2. 扫描一次数组，找到最小的元素。 选择该候选者是因为任何有效除数必须足够小以除以所有元素。 
3. 再次遍历数组，检查每个元素是否都能被该候选元素整除。 
4. 如果任何元素未通过可分性检查，则立即断定此测试用例不存在有效元素。 
5. 如果所有元素都通过检查，则输出“SI”，否则输出“NO”。 

### 为什么它有效

 如果一个数字整除数组中的每个元素，它必须特别整除最小元素。 由于数组中不存在小于最小值的数字，因此能够承受所有约束的唯一可能的候选者就是最小值本身。 任何更大的数字都不能整除最小值，因此它无效。 这创建了严格的唯一性属性：最多需要检查一个候选者，其有效性完全决定答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))

        m = min(arr)

        ok = True
        for x in arr:
            if x % m != 0:
                ok = False
                break

        out.append("SI" if ok else "NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先读取所有测试用例并独立处理它们。 对于每种情况，它都会使用单遍计算最小值。 

第二个循环检查每个元素是否能被这个最小值整除。 早期中断很重要，因为一旦发现反例，它就可以避免不必要的工作，从而提高早期发生故障的最坏情况输入的性能。 

一个微妙的点是我们不需要测试最小值是否能整除自身，因为`x % m`始终为零，当`x == m`。 这避免了冗余逻辑并保持所有元素的检查一致。

 ## 边缘情况

 一种重要的边缘情况是所有元素都相同。 对于像这样的输入`[7, 7, 7, 7]`，最小值为 7，并且每个元素都能被 7 整除。算法扫描一次，确认所有元素的整除性，并输出“SI”。 这验证了重复值不需要任何特殊处理。 

另一种情况是最小值只出现一次并且仍然不能作为除数，例如`[2, 3, 4, 6]`。 该算法选择 2 作为候选，但在 3 上立即失败，因为`3 % 2 != 0`。 提前终止可以防止不必要的检查。 

第三种情况是，较大的数字看似合理，但实际上并不正确，因为它无法整除最小值。 为了`[4, 2, 4, 6, 8, 4]`，除了 2 或 4 之外的任何候选者都是不可能的。 该算法正确地确定为 2，并根据整个数组对其进行验证，生成“SI”。
