 ## 解决方案
 ## 问题理解

 我们有一排建筑物，每个建筑物都包含一定数量的学生。 弗雷迪总是从第一座建筑物开始，严格向右移动。 当他按顺序访问建筑物时，他会累积迄今为止见过的学生数量。 

对于每个查询值，我们都会被问到一个简单的逻辑问题：Freddy 从一开始必须走多远，才能使他遇到的学生总数至少达到该查询值。 如果即使在访问了所有建筑物之后，总数仍然小于查询，则答案是不可能的。 

关键对象是数组的前缀累加。 每个查询都要求总和达到或超过目标的最小前缀长度。 

这些约束促使我们走向线性或近线性预处理。 对于多达 100000 座建筑物和 100000 个查询，任何重新计算每个查询总和的解决方案都将失败。 在最坏的情况下，双循环将导致 10^10 次操作，这远远超出了 1 秒的限制。 

当查询要求零个学生时，会出现微妙的边缘情况。 在这种情况下，答案应该是零建筑物，因为弗雷迪在进入任何建筑物之前就已经满足了要求。 另一种情况是当建筑物包含零时，这可能会产生前缀和不改变的长距离。 天真的“总和增加就停止”的方法在这里会被打破，因为进展与参观新建筑无关。 

## 方法

 直接方法独立处理每个查询。 对于给定的目标 q，我们模拟从第一栋建筑物步行并不断添加学生计数，直到总和达到 q。 这是正确的，因为它遵循精确的运动规则。 然而，每个查询可能需要扫描几乎所有建筑物。 对于 m 个查询，这变成了 O(nm)，当 n 和 m 都很大时，这太慢了。 

问题的结构是单调的。 当我们向右移动时，前缀和永​​远不会减少。 这种单调性意味着一旦我们预先计算前缀和，每个查询就变成了排序（非递减）数组上的搜索问题。 我们可以通过找到第一个达到所需阈值的前缀和来回答每个查询，而不是重新模拟每个查询。 

这将任务简化为对前缀和数组进行二分搜索。 在一次线性预处理之后，每个查询都会在对数时间内得到回答。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(纳米) | O(1) | O(1) | 太慢了 |
 | 前缀和+二分查找 | O(n + m log n) | O(n + m log n) | O(n) | 已接受 |

 ## 算法演练

 1. 计算建筑数组的前缀和。 第 i 位置的值代表从 1 号楼到 i 号楼的学生总数。 这将问题转化为单调数组，其中每一步都不会减少。 
2、对于每个查询值q，检查是否超过最后一个前缀和。 如果是，则没有可以满足要求的前缀，所以答案为-1。 这避免了不必要的搜索。 
3. 如果 q 为零，则立即返回 0，因为不需要任何建筑物即可到达零个学生。 
4. 否则，对前缀和数组执行二分查找以找到最小索引 i，使得 prefix[i] 大于或等于 q。 这是可行的，因为前缀数组是非递减排序的。 
5. 如果内部使用从零开始的索引，则输出 i + 1，因为答案是根据访问的建筑物数量定义的。 

### 为什么它有效

正确性来自于前缀和的单调性。 一旦前缀总和达到某个值，后面的所有前缀至少都一样大。 这确保了有效答案集形成连续的索引后缀。 二分搜索正是利用了这种结构，通过重复地将搜索空间减半，同时保留任何有效答案必须位于所有总和不足的位置的右​​侧以及已确认足够的所有位置的左侧的不变量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import bisect

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    q = list(map(int, input().split()))
    
    pref = []
    s = 0
    for x in a:
        s += x
        pref.append(s)
    
    total = pref[-1] if pref else 0
    
    out = []
    for x in q:
        if x == 0:
            out.append("0")
            continue
        if x > total:
            out.append("-1")
            continue
        idx = bisect.bisect_left(pref, x)
        out.append(str(idx + 1))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现首先在建筑物上的单次传递中构建前缀和数组。 这保证了 O(n) 预处理。 

每个查询都是独立处理的。 在任何搜索之前，边缘情况 x == 0 都会显式返回为 0，这与不需要建筑物的定义相匹配。 

关键操作是`bisect_left`，它找到前缀和至少为查询值的第一个位置。 加 1 将从零开始的索引转换为建筑物的计数。 

当查询超出所有可到达的学生时，总和检查可以防止不必要的二分搜索。

 ## 边缘情况

 像输入这样的零查询情况`[5, 2, 3]`带查询`0`立即返回`0`因为在访问任何建筑物之前前缀和条件已经满足。 该算法在构建搜索之前处理此问题，因此不执行索引。 

全零数组，例如`[0, 0, 0]`产生前缀和`[0, 0, 0]`。 对于像这样的查询`1`，二分查找返回第一个索引，但由于总和仍然为零，所以预检查`x > total`正确输出`-1`。 这可以避免将重复的零误解为进度。 

等于总和的查询，例如数组`[3, 1, 2]`并查询`6`，恰好落在最后一个前缀处。 二分查找返回最终索引，并且答案是完整长度，因为没有更早的前缀达到总数。
