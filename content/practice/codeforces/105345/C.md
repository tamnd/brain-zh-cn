 ## 解决方案
 ## 问题理解

 我们得到一个代表一排灯笼的二进制字符串，其中每个位置要么是 0 要么是 1。在单个操作中，我们选择任何连续的段并翻转其中的每一位，将 0 变成 1，将 1 变成 0。目标是达到整个字符串变得均匀的配置，要么全 0，要么全 1，使用尽可能少的段翻转。 

关键的一点是我们并不局限于单位置翻转。 每次移动都会影响整个区间，这意味着如果多个不匹配项位于精心选择的段中，则一次操作可以一次修复多个不匹配项。 困难在于选择能够减少未来修正次数而不是创建新边界的细分市场。 

输入大小最多允许 100,000 个字符。 任何尝试所有可能的段甚至评估所有索引对之间的转换的解决方案都将涉及 n2 或 n3 运算的顺序，这远远超出了可行的限制。 需要线性或近线性策略，因为 O(n log n) 或 O(n) 是 1 秒约束下的实际目标范围。 

一个天真的但诱人的想法是从左到右贪婪地模拟翻转操作，始终通过从该位置翻转到稍后的某个点来修复第一个不匹配。 这可能会失败，因为早期决策可能会在先前固定的区域中引入新的不匹配。 

例如，考虑`S = 0101`。 如果我们贪婪地尝试将所有内容都设为 0，那么在没有计划的情况下翻转前缀或最小段可能会导致过度计数操作。 错误的贪婪选择可能会翻转`01`→`10`，然后继续并需要比必要更多的操作，即使最佳是 2 次翻转：`[1..4]`或对齐边界的两个结构化翻转。 

另一个微妙的边缘情况是当字符串已经统一时，例如`00000`或者`11111`。 任何假设至少需要一次操作的算法都会错误地返回正值而不是零。 

最后，交替字符串，例如`010101...`很关键。 它们揭示了解决方案是否对转换数量而不是实际值敏感，因为每个位置都是段之间的边界。 

## 方法

 蛮力的观点是将每个操作视为选择一个区间并翻转它，然后模拟这些区间的所有可能序列，直到字符串变得均匀。 即使将深度限制为 k 个操作也会导致爆炸：每个操作有 O(n²) 个选择，并且序列呈指数增长。 即使对于较小的 n，这也很快变得不可行。 

看待这个问题的另一种方式是停止将间隔作为主要对象来思考，而是关注当我们翻转一个线段时字符串的结构如何变化。 翻转在均匀区域内并不重要，它只改变发生 0 和 1 之间转换的位置。 关键的认识是，重要的不是单个字符，而是连续的不同字符之间的边界。 

如果我们尝试使字符串全为 0，则必须消除每个连续的 1 块。 每个操作都可以根据其选择方式精确消除一个或多个此类块，但最小策略对应于对这些块进行计数。 类似地，如果我们的目标都是 1，我们就会计算 0 块。 由于每次翻转都可以最佳地消除一组连续的错误位，因此答案将成为这两个计数之间的最小值。 

问题归结为扫描字符串并计算相同字符的运行次数。 每次运行代表相同位的最大段。 转换为全 0 会消耗 1 游程的数量，转换为全 1 会消耗 0 游程的数量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（区间模拟）| 指数| O(n) | 太慢了 |
 | 最佳（运行计数）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为计算字符之间的转换，这直接对应于计算连续块。 

1. 计算字符串中连续相同字符的段数。 这是通过从左到右扫描并在当前字符与前一个字符不同时递增计数器来完成的。 这给出了运行的总数。 
2. 分别导出 0 和 1 的游程数。 如果总共有 k 次运行，它们会在 0 和 1 之间交替，因此我们可以根据起始字符计算这两个计数。 
3. 计算有多少次运行对应于1s。 这等于我们想要将所有内容都变成 0 所需的操作数，因为每个 1 块必须通过至少一次翻转操作来消除。 
4. 计算有多少个游程对应于 0。 这就是把一切变成1s的成本。 
5. 输出这两个值中的最小值。 

我们明确评估两个目标的原因是，我们可以自由地以任一统一状态结束，并且根据结构，一个方向可能需要严格更少的翻转。 

### 为什么它有效

 翻转连续段只能更改其边界处的整个运行的状态。 在运行中，翻转不会创建新结构，它只会切换运行的值。 因此，最小翻转次数对应于消除所有不期望值的游程，并且没有任何操作可以在不隐式合并相邻游程的情况下消除多个交替边界结构。 这使得给定值的运行计数成为下限，并且可以通过独立选择每个运行来实现。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    if n == 0:
        print(0)
        return

    # count runs of 0 and 1
    runs0 = 0
    runs1 = 0

    prev = None

    for ch in s:
        if ch != prev:
            if ch == '0':
                runs0 += 1
            else:
                runs1 += 1
        prev = ch

    # cost to make all 0s = number of 1-runs
    # cost to make all 1s = number of 0-runs
    print(min(runs0, runs1))

if __name__ == "__main__":
    solve()
```该代码对字符串执行单次传递，同时跟踪新运行何时开始。 只要当前字符与前一个字符不同，就会检测到一次运行。 我们没有显式存储所有运行，而是直接为 0 运行和 1 运行增加计数器。 

一个微妙的细节是初始化`prev`作为`None`，这确保第一个字符始终开始新的运行。 另一个细节是我们仅在边界处增加运行计数器，这避免了重复计算。

 ## 边缘情况

 对于完全统一的字符串，例如`00000`，循环仍精确计数 1 次 0 和 0 次 1。 最小值变为零，因为不会触发任何转换，也不需要翻转。 

对于完全交替的字符串，例如`010101`，每个字符变化都会触发一次新的运行，从而产生 0 和 1 运行的最大计数。 该算法自然地计算每个边界，最终答案反映了将所有交替块合并为单个统一状态所需的最小段翻转次数。 

For a single character input like`1`，初始的`prev = None`确保正确创建运行，并且由于没有转换，因此结果为零，与字符串已经统一的事实相匹配。
