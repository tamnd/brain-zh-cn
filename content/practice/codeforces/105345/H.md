 ## 解决方案
 ## 问题理解

 我们得到一条从 0 到 n 的整数位置的线段。 一些学生最初占据 1 到 n−1 之间的位置。 每个学生独立选择一个方向，向左朝向 0 或向右朝向 n，每个方向的概率为 1/2。 

一旦过程开始，每个学生都会朝着他们选择的方向每秒移动一步。 当两个学生同时在同一位置相遇时，他们立即反转方向并继续移动。 如果多个学生从同一位置开始，他们只会独立移动，除非他们的路径强制相互作用； 以相同速度向同一个方向移动的学生永远不会相遇。 

该过程继续进行，学生最终可能通过到达位置 0 或位置 n 退出该段。 对于每个查询时间 t，我们被要求提供在时间 t 之前没有学生退出的概率。 保证答案始终为 0 或 2 的负幂，因此我们输出指数 m，使得概率等于 2^{-m}，如果概率为零，则输出 -1。 

约束 n 最大为 10^5，q 最大为 10^5，t 最大为 10^8，排除了随着时间的推移对运动或成对相互作用的任何模拟。 即使在 O(n) 中处理每个查询也已经太慢了。 关键的困难在于碰撞和方向交换会产生交互的错觉，但实际上它们保留了更简单的底层结构。 

当学生被安排为至少一个方向选择总是迫使学生很快退出时，就会出现一种微妙的边缘情况。 例如，如果学生从位置 1 开始，则选择向左会导致在时间 1 立即退出。如果没有替代配置阻止所有此类强制退出，则对于足够大的 t，概率将变为零。 另一个边缘情况是多个学生共享一个位置，因为天真地将它们视为不同的移动实体可能会过多计算交互，而正确的模型仅取决于奇偶性和配对。 

## 方法

 直接模拟将明确跟踪每个学生，逐步移动他们并解决碰撞。 这正确地模拟了规则，因为如果我们忽略身份，则碰撞时交换方向相当于粒子相互穿过。 然而，模拟每个查询的时间 t 是不可能的。 当 t 高达 10^8 且 q 高达 10^5 时，即使每秒每个事件 O(1) 也是不可行的。 

关键的观察结果是，碰撞对于“有人退出了吗”事件并不重要。 如果我们忽略身份，将学生视为不可区分的粒子，则碰撞相当于两个粒子相互穿过。 这个经典技巧将系统转变为独立的步行者：每个学生有效地向左或向右直线移动，而无需交互。 

一旦接受了这种转换，学生避免在时间 t 退出的唯一方法是其选择的方向不会太快地离开该段。 仅当位置 x 的学生没有选择在 t 步内到达 0 或 n 的方向时，它才会在时间 t 时避免退出。 这成为每个学生的本地限制。 

因此，概率是通过计算“被迫”避免立即退出的独立二元选择的数量来确定的。 每个强制约束贡献一个 1/2 的因子，这解释了为什么最终概率始终为 2^{-m} 形式。 如果至少一名学生在 t 内没有有效的方向选择来避免退出，则概率为零。 

我们将问题简化为计算，对于每个学生，两个方向在时间 t 内是否安全，只有一个方向是安全的，或者没有一个方向是安全的。 指数 m 是只有一个安全方向的学生数量。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟 | O(n·t·q) | O(n) | 太慢了|
 | 独立方向减速| O(n + q) | O(n) | 已接受 |

 ## 算法演练

 1. 观察到碰撞不会影响学生是否在时间 t 之前退出，因此我们将每个学生视为在固定选择的方向上独立移动。 这消除了所有交互的复杂性。 
2. 对于位置 x 的学生，计算从左侧退出的时间为 x，从右侧退出的时间为 n − x。 这些是确定性的，不依赖于其他学生。 
3. 对于给定的查询时间 t，根据每个学生是否能够在每个方向下生存到时间 t 对每个学生进行分类。 如果 x > t，则向左移动是安全的；如果 n − x > t，则向右移动是安全的。 
4. 如果两个方向对任何学生来说都是不安全的，那么无论学生在时间 t 之前或在时间 t 退出的随机选择如何，因此不退出的概率为零，我们立即返回 -1。 
5. 如果只有一个方向是安全的，则该学生没有自由：它必须选择安全方向，为概率贡献 1/2 的系数。 
6. 如果两个方向都是安全的，则学生不施加任何约束，因为任一选择都避免在时间 t 内退出。 
7. 对所有位置（或具有多重性的唯一位置）上受约束的学生数量进行求和，并将该计数输出为 m。 

### 为什么它有效

 崩溃碰撞后，每个学生都会进化为独立随机选择方向，退出时间仅取决于其初始位置和选择的方向。 “时间 t 无人退出”事件成为独立事件的交集，每个事件都要求不允许某些方向选择。 独立性确保概率成倍增加，因此每个强制二元选择恰好贡献了 1/2 的因子。 如果任何学生的有效选择为零，则交集为空，概率为零。 指数m正是由约束系统固定的独立二元决策的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    
    # compress counts of positions
    from collections import Counter
    cnt = Counter(a)
    
    # precompute positions list
    positions = list(cnt.items())
    
    for _ in range(q):
        t = int(input())
        m = 0
        ok = True
        
        for x, c in positions:
            left_safe = x > t
            right_safe = (n - x) > t
            
            if not left_safe and not right_safe:
                ok = False
                break
            
            if left_safe != right_safe:
                m += c
        
        if not ok:
            print(-1)
        else:
            print(m)

if __name__ == "__main__":
    solve()
```该代码首先按位置压缩学生，因为同一坐标上的所有学生在生存条件下表现相同。 对于每个查询时间 t，它检查向左或向右移动是否使学生保持在段内。 关键部分是左生存条件 x > t 和右生存条件 n − x > t，它直接编码学生是否会在 t 秒内到达边界。 

变量 m 累加恰好有一个安全方向的学生数量。 如果两个方向对于任何位置都不安全，则 ok 变为 false，答案为 -1。 

一个常见的陷阱是忘记乘以同一位置的学生人数。 由于每个学生都是独立的硬币翻转，因此多重性必须对指数有贡献。 

## 工作示例

 考虑 n = 7，学生处于位置 [2, 2, 2, 3, 3]。 让我们评估 t = 1。 

| 位置 x | 计数 c | 离开安全 (x>1) | 右安全 (n-x>1) | 贡献|
 | ---| ---| ---| ---| ---|
 | 2 | 3 | 是的 | 是的 | 0 |
 | 3 | 2 | 是的 | 是的 | 0 |

 这里没有学生被迫走向单一方向，因此 m = 0，概率为 1。 

现在考虑 t = 2。 

| 位置 x | 计数 c | 离开安全 (x>2) | 右安全 (n-x>2) | 贡献|
 | ---| ---| ---| ---| ---|
 | 2 | 3 | 没有| 是的 | 3 |
 | 3 | 2 | 是的 | 是的 | 0 |

 位置 2 的三名学生必须向右移动，贡献指数 3。其他人自由。 所以m = 3。 

这显示了约束如何纯粹通过消除安全方向选项来累积。

 ## 边缘情况

 当学生开始非常接近边界时，就会出现临界边缘情况。 如果 x ≤ t 且 n − x ≤ t，则两个方向都会在 t 时间内退出，因此生存概率为零。 该算法通过检查两个条件是否为假并立即返回 -1 来正确检测到这一点。 

另一种情况是，当所有学生都处于 x > t 且 n − x > t 均成立的中间区域时。 在这种情况下，不施加任何约束，m 保持为零，产生概率 1。 

最后，当多个学生共享相同位置时，如果受到约束，每个学生都会独立地对指数做出贡献。 基于频率的计数可确保正确计算每个相同位置的学生，而不会出现重复错误。
