 ## 解决方案
 ## 问题理解

 迷宫是一个小网格，其中每个单元的行为就像影响山姆移动方式的地形类型。 有些单元可以自由通过，有些则被封锁，有些则施加资源限制：山姆可能需要收集玉米​​糖才能通过某些障碍。 任务是计算从起始单元格移动所需的最短时间`S`到出口单元`E`，或确定不可能到达出口。 

移动是在网格上标准的四向踩踏，每一次移动都花费一个单位时间。 复杂性来自两种特殊的瓷砖类型。 玉米糖细胞`C`永久增加山姆拥有的糖果数量。 一个南瓜灯`J`仅当山姆当前拥有至少一颗未使用的玉米糖时才能进入，并且进入它会消耗一颗。 南瓜`P`是墙壁。 

这将最短路径问题转化为有状态的最短路径问题。 仅仅这个位置是不够的，因为用不同数量收集的玉米糖到达同一个单元可能会导致不同的未来可能性。 玉米糖总数最多为 8 个的限制是使其易于管理的关键结构限制。 

网格最多为 100 x 100，因此最多有 10,000 个位置。 由于糖果玉米最多为 8 个，因此任何正确的状态表示都将保持相对较小，大约为 10,000 乘以 256 个可能的糖果状态，这对于最短路径搜索来说仍然是可以接受的。 

忽略糖果状态的幼稚方法将会以微妙的方式失败。 例如，考虑 Sam 必须经过一个`J`只有在收集了绕道而行的玉米糖后才能平铺。 如果我们将重新访问单元视为不必要，我们可能会错误地丢弃最佳路径。 

当存在多个糖果玉米但只需要一些糖果玉米来解锁一系列糖果时，就会出现另一种失败情况`J`瓷砖。 没有状态跟踪的贪婪网格中最短路径方法可能会更快地到达出口，但糖果不足，稍后会被卡住。 

一个具体的最小说明：

 输入：```
1 5
S C J E
```正确输出：```
3
```一个忽略糖果状态的天真的 BFS 可能会处理`C`只是又一步并尝试去`S -> J -> E`, 失败是因为`J`收集前不能输入`C`。 

## 方法

 一个简单的尝试是运行 BFS`S`，将每个单元格视为图中的一个节点。 这适用于具有统一成本且没有额外约束的简单网格，因为 BFS 保证未加权图中的最短路径。 

该问题打破了这一假设，因为遍历边缘的能力取决于动态资源，即收集的玉米糖的数量。 这意味着该图不是静态的。 每个网格位置有效地扩展到多个状态，每个状态代表一种可能的糖果数量。 

蛮力思想是将每个状态视为`(row, col, mask or count of candies)`，并允许基于图块类型的转换。 从一个状态开始，我们尝试所有四个动作，如果我们踩到，则更新糖果数量`C`，并且只允许踩踏`J`如果糖果数为正。 这会生成一个分层图，最多可达`n * m * 2^d`州。 

自从`d ≤ 8`，状态数量的界限为 10000 × 256 = 256 万。 每个状态最多有四个转换，因此大约有 1000 万条边。 这已经是临界点，但在具有高效 BFS 的 Python 中是可以接受的。 

更仔细的观察表明，BFS 仍然有效，因为每一步的成本都是相等的。 唯一的变化是我们必须在访问的结构中包含糖果计数，这样我们就不会错误地合并位置看起来相同但资源不同的状态。 

关键的观察结果是，玉米糖的数量很少，并且只会成倍地增加状态空间。 因此，我们可以在扩展状态图上运行标准的 0-1 未加权 BFS。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（网格 BFS 忽略状态）| O(纳米) | O(纳米) | 错误|
 | 状态 BFS（位置 + 糖果数量）| O(nm·2^d) | O(nm·2^d) | 已接受 |

 ## 算法演练

 我们将每个配置建模为由 Sam 的位置和当前持有的玉米糖数量组成的三元组。 目标是带着任何有效的糖果数量到达出口单元。 

1. 找到起始位置`S`并将初始糖果数视为零。 这建立了 BFS 的初始状态。 
2. 初始化队列`(S_row, S_col, 0)`以及所有状态都初始化为无穷大的距离数组。 该距离跟踪到达每个配置的最短时间。 
3. 对状态执行 BFS。 弹出一个状态`(r, c, k)`代表位置和糖果数量。 
4. 对于四个方向中的每一个，计算相邻单元格`(nr, nc)`。 如果出界或者是南瓜`P`，立即跳过，因为它无法通过。 
5.判断是否运动进入`(nr, nc)`根据瓷砖类型是否允许。 如果是的话`J`， 要求`k > 0`。 如果允许，将糖果数量减一。 该模型模拟了玉米糖的消耗。 
6. 如果电池是`C`，进入后增加糖果数量。 这种顺序很重要，因为在考虑该单元的任何潜在收益之前，进入南瓜灯会消耗一颗糖果。 
7. 如果结果状态`(nr, nc, new_k)`没有以更短或相等的距离访问过，更新其距离并将其推入队列。 
8. 继续直到 BFS 完成或直到退出单元格`E`已达到； 答案是所有糖果状态的最小距离`E`。 

### 为什么它有效

 每次移动都有统一的成本，因此 BFS 保证状态按照距离的非递减顺序进行处理。 唯一的复杂之处在于，以不同的糖果数量到达相同的网格单元是根本不同的状态。 通过扩展状态空间以包含糖果计数，我们恢复了经典的 BFS 不变量：一旦状态出队，其距离在到达该精确配置的所有可能路径中是最小的。 由于探索了所有有效的转换，因此不会错过通往任何状态的更短路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]
    
    sr = sc = er = ec = -1
    
    for i in range(n):
        for j in range(m):
            if grid[i][j] == 'S':
                sr, sc = i, j
            if grid[i][j] == 'E':
                er, ec = i, j
    
    max_mask = 1 << 8  # up to 8 candies
    
    dist = [[[10**9] * max_mask for _ in range(m)] for _ in range(n)]
    
    q = deque()
    dist[sr][sc][0] = 0
    q.append((sr, sc, 0))
    
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]
    
    while q:
        r, c, k = q.popleft()
        d = dist[r][c][k]
        
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if nr < 0 or nr >= n or nc < 0 or nc >= m:
                continue
            
            cell = grid[nr][nc]
            nk = k
            
            if cell == 'P':
                continue
            
            if cell == 'J':
                if nk == 0:
                    continue
                nk -= 1
            
            if cell == 'C':
                nk += 1
                if nk >= 8:
                    nk = 7
            
            if dist[nr][nc][nk] > d + 1:
                dist[nr][nc][nk] = d + 1
                q.append((nr, nc, nk))
    
    ans = min(dist[er][ec])
    if ans == 10**9:
        print("SPOOKED!")
    else:
        print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了前面描述的状态扩展。 三度空间`dist`数组确保我们不会合并具有不同糖果数量的状态。 每次转换在修改糖果状态之前都会仔细应用南瓜灯约束。 

一个微妙的实现细节是更新的顺序：我们在输入时减去糖果`J`在考虑之前`C`在同一个单元格中，因为一个单元格不能同时位于该问题定义中。 糖果计数的限制对于正确性来说并不是严格必要的，但可以将状态安全地限制在 0 到 7 之间。

 ## 边缘情况

 一种重要的边缘情况是当糖果积累是必要的但不能立即有用时。 考虑一条所有直接路由都到达的路径`E`通过`J`细胞提早，迫使收集绕路`C`第一的。 BFS 处理这个问题是因为它不假设贪婪的方向选择； 它探索了所有可达的状态，因此自然也包括了弯路。 

另一种情况是山姆可以用不同的糖果数量重新访问同一个牢房。 单纯访问过的数组会错误地阻止重新访问，但基于状态的 BFS 会处理`(r, c, k1)`和`(r, c, k2)`不同的是，允许恢复路径，稍后访问更多糖果可以解锁进度。 

最后，糖果丰富但不必要的情况仍然可以正确运行，因为多余的糖果不会阻止遍历； 糖果数量较高的状态可能会共存，但 BFS 确保最短时间的状态首先到达出口，而不管剩余资源如何。
