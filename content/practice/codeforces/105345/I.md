 ## 解决方案
 ## 问题理解

 我们看到一排排成一排的房子。 每个房子都有一个截止日期，以从开始开始计算的分钟为单位，在此之后爱丽丝仍然可以到达它，但如果她正好在那个时间或更晚到达，则不会收到糖果。 Alice 从位置 k 开始，每分钟她可以沿着线向左或向右移动一步。 

目标是找出爱丽丝是否可以在截止日期之前至少访问每栋房子一次，如果可以，则计算完成所有访问所需的最短总时间。 关键细节是，参观房屋是在抵达后立即进行的，但抵达必须严格在截止日期之前进行。 

从约束的角度来看，n 最多为 2000，这排除了对访问顺序排列的任何指数子集探索。 三次解虽然是边缘解，但如果仔细优化的话可能是可以接受的。 任何尝试显式模拟所有可能的访问顺序或路径的解决方案都无法扩展，因为可能的路线数量随 n 增长得非常快。 

当贪婪地思考时，会出现一个微妙的问题：首先访问附近的房屋可能会将爱丽丝困在一个无法及时到达遥远的紧迫期限房屋的区域。 另一种失败情况来自假设单一扫描方向，因为最佳路径可能需要多次切换方向。 

具体的边缘情况是两端都存在紧迫的期限。 

输入：```
5 3
100 1 100 100 100
```如果爱丽丝贪婪地向左移动到位置 2 处的紧房子，她会浪费时间，并且可能无法在最后期限内到达右侧。 正确的行为需要仔细平衡首先访问哪一侧。 

另一个边缘情况是，当起始位置对于最紧迫的截止日期房子来说不是最佳的时，并且必须首先到达它。 

## 方法

 一种暴力解释是认为爱丽丝选择访问所有房屋的顺序，并为每个顺序模拟她的移动成本。 有n个！ 可能的排列，甚至计算单个排列成本也需要 O(n)，因此这种方法立即不可行。 

问题的结构是一维的，这表明任何最佳路径都不是任意的，而是表现为从起点开始的连续线段扩展。 一旦 Alice 离开某个区域，返回该区域的成本就很高，因此最优解决方案倾向于从 k 向外“扩展”覆盖范围。 

关键的见解是，我们不考虑排列，而是将最终访问集视为随时间扩展的连续区间 [l, r]。 一旦区间中的所有房屋都可以在截止日期内到达，区间内的顺序就无关紧要了，因为连续段内的遍历始终是最佳的来回扫描。 

这将问题转化为检查哪些间隔可以被完全覆盖，并确定从 k 扩展到所有房屋所需的最短时间，同时遵守最后期限。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(n!) | O(n) | 太慢了|
 | 区间扩展 DP/贪心 | O(n²) | O(n) | 已接受 |

 ## 算法演练

 我们将任务重新解释为增长一个始终包含起始位置 k 的线段 [l, r]。 

在任何时刻，如果我们有区间 [l, r]，那么从 k 开始完全覆盖它所需的时间就是从 k 走到一端、扫过并根据结构可能返回的成本。 然而，在一个维度上，固定区间的最优策略是已知的：我们要么先向左走，要么先向右走，以哪个更好，然后完全遍历。 

挑战在于决定我们可以在满足所有最后期限的情况下将间隔延长多远。 

我们使用动态规划来跟踪可行的间隔。 

1. 我们将 dp[l][r] 初始化为是否可以在遵守截止日期的同时覆盖区间 [l, r]。 我们以 dp[k][k] 为真开始，因为 Alice 在时间 0 时已经在那里。 
2. 我们通过从有效区间向左或向右扩展来扩展区间。 如果 dp[l][r] 有效，我们尝试 dp[l-1][r] 和 dp[l][r+1]。 
3. 对于每个候选区间，我们计算从 k 开始访问该区间内所有节点所需的最短时间。 这是通过考虑两种可能的遍历顺序来完成的：先向左然后向右，或者先向右然后向左。 
4. 我们验证这个时间严格小于[l, r]中i的所有a[i]。 如果是，我们将该间隔标记为有效。 
5. 我们继续扩展，直到不再存在有效区间。 
6. 答案是覆盖 [1, n] 的所有有效间隔中的最小时间，如果不可能，则为 -1。 

关键思想是区间有效性是单调的：一旦一个区间可行，如果截止日期允许，它有助于扩展到更大的区间。 

### 为什么它有效

 一条线上的任何有效游览都可以转换为无间隙访问连续路段的游览，因为跳过并返回未访问过的点只会增加不必要的时间。 因此，每个最优解都对应于 k 的某个区间扩展。 DP 确保我们只考虑可以在最严格的截止期限约束内完全遍历的区间，并且区间内的每个可能的排序都由两个单调遍历方向之一主导。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = [0] + list(map(int, input().split()))
    k -= 1  # convert to 0-index

    INF = 10**18

    # dp[l][r] = minimum time to cover [l, r] starting from k
    dp = [[INF] * n for _ in range(n)]

    def cost(l, r):
        # compute minimal time to visit entire segment starting at k
        if l <= k <= r:
            # go to one side then sweep
            left = (k - l) * 2 + (r - k)
            right = (r - k) * 2 + (k - l)
            return min(left, right)
        elif r < k:
            return k - l
        else:
            return r - k

    # initialize single point
    dp[k][k] = 0

    for length in range(1, n + 1):
        for l in range(0, n - length + 1):
            r = l + length - 1
            if l == r:
                continue

            best = INF

            # extend from left
            if l + 1 <= r and dp[l + 1][r] != INF:
                best = min(best, dp[l + 1][r])
            # extend from right
            if l <= r - 1 and dp[l][r - 1] != INF:
                best = min(best, dp[l][r - 1])

            if best == INF:
                continue

            t = cost(l, r)
            ok = True
            for i in range(l, r + 1):
                if t >= a[i + 1]:
                    ok = False
                    break

            if ok:
                dp[l][r] = t

    ans = dp[0][n - 1]
    print(ans if ans != INF else -1)

if __name__ == "__main__":
    solve()
```该代码通过增加长度自下而上构建间隔。 对于每个区间，它尝试从小区间继承可行性。 成本函数在给定起始位置的固定间隔内编码最佳遍历。 

正确性取决于计算线段的最小遍历时间，根据 k 是否位于线段的内部、左侧或右侧，该时间分为三种情况。 截止日期检查确保在截止时间或之后没有房屋被拜访。

 ## 边缘情况

 一栋房子的最小投入：```
1 1
10
```该间隔已经完成，并且成本为零，因为不需要移动。 DP从dp[0][0] = 0开始并立即输出。 

对称紧端箱：```
5 3
2 100 1 100 2
```正确的策略必须优先向中心、谨慎扩张。 DP 确保在位置 3 处违反严格期限的间隔永远不会被接受，因为在到达外部元素之前成本会增长到超过 1。 

期限不对称的情况：```
5 3
100 1 100 1 100
```这里，双方都包含严格的约束，迫使算法仔细选择扩展顺序。 间隔 DP 评估左扩展和右扩展，并且仅接受计算成本低于所有本地截止日期的序列，从而防止无效的早期扩展。
