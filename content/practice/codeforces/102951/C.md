 ## 解决方案
 ## 问题理解

 我们给出两个包含不同顺序的相同元素的序列，通常是两种长度排列$n$。 任务是确定以相同相对顺序出现在两个排列中的最长值序列的长度，而不要求值是连续的。 

具体来说，假设您在一个架子上有一个按顺序排列的标签列表，另一个架子上有相同的标签但重新排列。 您可以从第一个货架上挑选一些标签，保留从左到右的顺序，并且您想要尽可能长的序列，也可以按照相同的顺序从第二个货架上挑选。 

输出是一个整数，即以这种方式可以匹配的元素的最大数量。 

驱动解决方案的关键约束是输入是排列，这意味着每个值在每个数组中只出现一次。 和$n$直到大约$10^5$，任何比较所有位置对的二次方法都需要以下顺序$10^{10}$最坏情况下的操作，这远远超出了 2 秒运行时间的可行限制。 这立即排除了显式计算 LCS 的动态规划解决方案$O(n^2)$。 

一些边缘情况很容易处理不当。 

如果两个排列相同，例如：

 输入：$A = [1, 2, 3, 4]$,$B = [1, 2, 3, 4]$正确答案是 4，因为整个序列都匹配。 仅检查位置相等而不检查顺序的幼稚方法在这里仍然有效，这可能会错误地保证错误的解决方案。 

如果一个排列颠倒过来：

 输入：$A = [1, 2, 3, 4]$,$B = [4, 3, 2, 1]$正确答案是 1，因为没有两个元素保持相对顺序。 假设匹配索引或忘记排序约束的解决方案通常在这里计算过多。 

当值正确但位置未对齐时，会出现另一种微妙的失败情况：

 输入：$A = [2, 3, 1, 4]$,$B = [1, 2, 3, 4]$正确答案是 2，而不是 3，因为只有某些子序列保留两个数组中的顺序。 

## 方法

 思考问题的一个直接方法是尝试第一个排列的所有子序列，并检查每个子序列是否以相同的顺序出现在第二个排列中。 这在原则上是正确的，因为它显式地测试了可能属于公共子序列的每个候选序列。 然而，一个子序列的数量$n$-元素数组是$2^n$，甚至检查第二个数组中的每一个都会花费线性时间，导致指数爆炸，即使对于中等程度的情况也是不可能的$n$。 

结构性突破来自于认识到第二个排列为每个值定义了一个固定位置。 我们可以将第一个排列转换为表示每个元素在第二个排列中出现的位置的索引序列，而不是直接比较值。 一旦完成这个转换，问题就完全改变了：我们不再匹配两个排列，而是在单个索引数组中找到最长的递增子序列。 

这是有效的，因为在两个排列中保留相对顺序相当于在第二个排列中保留位置的递增顺序。 第一个排列的任何子序列都对应于第二个排列中的索引序列，并且当且仅当这些索引严格递增时，该子序列才有效。 

LIS问题可以解决$O(n \log n)$使用带有二分搜索的贪婪结构，这对于$n = 10^5$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解子序列 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 地图 + LIS |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ## 步骤

 1. 为第二个排列构建位置图，存储每个值的索引。 这允许恒定时间查找任何元素在第二个序列中的位置。 
2. 将第一个排列中的每个元素替换为第二个排列中相应的索引，生成一个新的位置数组。 
3. 计算该变换后的数组的最长严格递增子序列的长度。 
4. 维护数组`dp`在哪里`dp[i]`存储长度递增子序列的最小可能结束值`i + 1`。 
5. 对于转换后的数组中的每个位置值，使用二分查找来查找它适合的位置`dp`。 如果它扩展了最长的子序列，则追加它； 否则，替换第一个元素`dp`大于或等于它。 
6. 最终长度`dp`就是答案。 

替换步骤是保持灵活性的关键思想。 即使我们覆盖其中的值`dp`，我们只为给定长度的子序列保留最好的可能尾部，而不是承诺特定的子序列。 

### 为什么它有效

 在任何时候，`dp[k]`表示长度递增子序列的最小可能结束位置`k + 1`。 这个不变量确保如果存在更长的递增子序列，它总是可以构建在这些最小结尾之上。 由于我们总是用较小的有效候选者进行替换，因此我们永远不会失去扩展到最佳解决方案的可能性，并且我们永远不会通过使用不兼容的元素人为地增加子序列长度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from bisect import bisect_left

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    pos = [0] * (n + 1)
    for i, v in enumerate(b):
        pos[v] = i

    seq = [pos[v] for v in a]

    dp = []
    for x in seq:
        i = bisect_left(dp, x)
        if i == len(dp):
            dp.append(x)
        else:
            dp[i] = x

    print(len(dp))

if __name__ == "__main__":
    solve()
```代码的第一部分构造一个直接查找表，以便第一个排列中的每个值都可以在恒定时间内映射到第二个排列中的位置。 这避免了重复扫描。 

转换步骤创建一个对排序约束进行编码的单个数字序列。 一旦完成，原来的问题结构就消失了，我们只对单调性进行推理。 

LIS 计算使用带有二分搜索的贪心策略。 这`dp`array 不存储实际的子序列，但为每个长度维护最小可能的结尾，这足以恢复正确的长度。 

## 工作示例

 ### 示例 1

 输入：$A = [2, 3, 1, 4]$,$B = [1, 2, 3, 4]$职位于$B$:$1 \to 0, 2 \to 1, 3 \to 2, 4 \to 3$变换后的序列：$[1, 2, 0, 3]$| 步骤| x| dp 之前 | dp 之后 | 解释|
 | ---| ---| ---| ---| ---|
 | 1 | 1 | []| [1] | 开始新的子序列 |
 | 2 | 2 | [1] | [1, 2] | 增加延伸 |
 | 3 | 0 | [1, 2] | [0, 2] | 替换第一个元素 |
 | 4 | 3 | [0, 2] | [0,2,3]| 延长最长|

 最终答案是3。 

这演示了替换如何允许从早期的大值中恢复，并且稍后仍然构建更长的子序列。 

### 示例 2

 输入：$A = [1, 2, 3]$,$B = [3, 2, 1]$职位：$3 \to 0, 2 \to 1, 1 \to 2$改造后：$[2, 1, 0]$| 步骤| x| dp 之前 | dp 之后 |
 | ---| ---| ---| ---|
 | 1 | 2 | []| [2] |
 | 2 | 1 | [2] | [1] |
 | 3 | 0 | [1] | [0]|

 最终答案是1。 

这表明该算法可以正确处理完全相反的排序，其中不存在长度超过 1 的递增子序列。

 ## 边缘情况

 单元素排列是最简单的健全性检查。 该算法构建大小为 1 的位置图并生成单元素 LIS，因为没有可供比较的替代结构。 

相同的排列测试 LIS 实现是否保留完整的递增结构而无需不必要的替换。 每个元素都延伸`dp`，所以长度线性增长到$n$，这与正确答案相匹配。 

反向排列强调严格的排序要求。 每个新元素都比转换后的数组中的前一个元素小，因此 LIS 结构不断折叠回长度 1，这证明了对非递增序列的正确处理。 

混合模式证实局部减少不会破坏全局子序列发现。 即使元素分散，替换策略也能确保早期的次优尾部不会阻止后来的有效扩展。
