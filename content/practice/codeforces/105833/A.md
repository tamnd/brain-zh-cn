 ## 解决方案
 ## 问题理解

 我们有一个字符串`S`长度`N + 1`，其中网格对角线上的每个位置都标记为`A`或者`B`。 

令牌开始于`(0, 0)`在一个`(N + 1) × (N + 1)`网格。 玩家交替移动，每次移动都会使行索引或列索引加一。 一旦令牌到达一个单元格，游戏就会停止`(i, j)`和`i + j = N`。 

反对角细胞`(i, N - i)`对应字符`S[i]`。 

如果该字符是`A`，爱丽丝获胜。 如果是的话`B`，鲍勃获胜。 

爱丽丝先行动，双方都发挥最佳状态。 任务不是解决单个游戏实例。 相反，在所有`2^(N+1)`可能的字符串`S`，我们必须计算有多少让爱丽丝获胜。 答案需要模数`10^9 + 3233`。 

关键约束是`N ≤ 100000`。 任何检查所有字符串的解决方案都是不可能的。 甚至整个游戏图上的动态程序也是不必要的。 预期的解决方案是一个封闭形式的计数参数，导致对数时间，因为只需要模幂。 

一个常见的错误是认为字符串的每个字符都很重要。 事实上，最优玩法会将游戏折叠到只有两个中心位置：`N`是奇数，并且只有三个中心位置`N`是均匀的。 字符串的其余部分完全不相关。 

例如，与`N = 1`，字符串有长度`2`。```
AA
AB
BA
BB
```Alice 可以直接选择任一反对角端点。 她在前三个案件中获胜，只输了`BB`。 

为了`N = 2`，只有三重`(S0,S1,S2)`很重要。 获胜的字符串是：```
AAA
AAB
BAA
```与示例答案相匹配`3`。 

## 方法

 暴力方法会枚举所有`2^(N+1)`字符串。 对于每个字符串，我们可以通过网格上的极小极大来解决游戏。 这是正确的，因为它直接遵循游戏定义。 不幸的是，即使对于`N = 50`，字符串的数量已经是天文数字了，所以这个方向是无望的。 

突破来自于对游戏几何结构的理解。 

恰好之后`N`移动，令牌到达反对角线。 实际路径仅确定到达哪个反对角单元格。 最佳游戏让玩家控制行移动和列移动之间的平衡。 

首先考虑一个奇数`N = 2K + 1`。 爱丽丝做出了最后一步。 配对策略表明，在她移动之后，她总是可以将令牌强制到两个中央反对角单元之一上，对应于索引`K`和`K + 1`。 相反，如果这两个单元格都输了，鲍勃可以强制游戏，以便爱丽丝的最后一步必须在这两个失败的单元格之间进行选择。 因此，当且仅当至少满足以下条件之一时，爱丽丝获胜：`S[K]`或者`S[K+1]`是`A`。 

现在考虑偶数值`N = 2K`。 爱丽丝选择第一步，之后鲍勃成为奇数子游戏中的第一个玩家。 如果爱丽丝单向移动，相关的中心对就变成`(S[K-1], S[K])`。 如果她朝相反的方向移动，相关的中心对就会变成`(S[K], S[K+1])`。 鲍勃赢得了一个奇怪的子游戏，除非其中心对中的两个单元格都是`A`。 因此，当这些对中至少有一对完全由以下组成时，爱丽丝就获胜了`A`。 

对于奇数`N`，在中心对的四项任务中，三项获胜：```
AA, AB, BA
```仅有的`BB`输了。 

对于甚至`N`，在中心三元组的八个作业中`(S[K-1], S[K], S[K+1])`，正好三个获胜：```
AAA, AAB, BAA
```计数变为：

 对于奇数`N = 2K + 1`：```
3 * 2^(N-1)
= 3 * 4^K
```对于甚至`N = 2K`：```
3 * 2^(N-2)
= 3 * 4^(K-1)
```两个表达式都简化为```
3 * 4^floor((N-1)/2)
```这正是最终的公式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^(N+1)) | O(2^(N+1)) | O(1) | O(1) | 太慢了 |
 | 最佳| O(log N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 阅读`N`。 
2. 计算`e = floor((N - 1) / 2)`。 
3. 计算`4^e mod (10^9 + 3233)`使用快速模幂。 
4. 将结果乘以`3`。 
5. 对最终值取模`10^9 + 3233`。 
6. 输出答案。 

### 为什么它有效

 对于奇数`N`，最佳发挥仅取决于两个中心反对对角线位置。 爱丽丝获胜，除非两者都被标记`B`，准确给出`3`赢得这两个职位的任务。 

对于甚至`N`，最佳发挥仅取决于三个中心位置。 爱丽丝恰好赢得了三项任务`AAA`,`AAB`， 和`BAA`。 

字符串中所有剩余的字符都不会影响结果，并且可以自由选择。 计算空闲位置后，两种奇偶校验情况都折叠为相同的封闭形式：$$3 \cdot 4^{\lfloor (N-1)/2 \rfloor}.$$该算法精确地计算该公式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 3233

def main():
    n = int(input())
    exp = (n - 1) // 2
    ans = 3 * pow(4, exp, MOD)
    print(ans % MOD)

if __name__ == "__main__":
    main()
```其实现几乎是数学公式的直接翻译。`exp = (n - 1) // 2`计算以封闭形式出现的指数。 Python内置的三参数`pow`以对数时间执行模幂运算，这比重复乘法要快得多。 

乘以`3`之后进行，最后的模运算使结果保持在所需的范围内。 

不存在棘手的边界条件。 最小的输入是`N = 1`，给出指数`0`， 和`pow(4, 0, MOD)`正确返回`1`，产生答案`3`。

## 测试用例```python
# helper: run solution on input string, return output string
import sys, io

MOD = 10**9 + 3233

def solve():
    n = int(input())
    print((3 * pow(4, (n - 1) // 2, MOD)) % MOD)

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout

    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()

    global input
    input = sys.stdin.readline

    solve()

    out = sys.stdout.getvalue()

    sys.stdin = backup_stdin
    sys.stdout = backup_stdout

    return out

# provided sample
assert run("2\n") == "3\n", "sample"

# minimum N
assert run("1\n") == "3\n", "N=1"

# odd N
assert run("3\n") == "12\n", "3 * 4^1"

# even N
assert run("4\n") == "12\n", "3 * 4^1"

# larger value
assert run("5\n") == "48\n", "3 * 4^2"
```| 测试输入| 预期产出 | 它验证了什么 |

 ## 边缘情况

 对于`N = 1`，爱丽丝第一步行动后游戏结束。 

输入：```
1
```相关的对很简单`(S0,S1)`。 只有当两者都满足时，爱丽丝才会输`B`。 有`3`获胜字符串，公式给出：$$3 \cdot 4^0 = 3.$$为了`N = 2`，只有中心三元组很重要。 

输入：```
2
```获胜三连冠是：```
AAA
AAB
BAA
```正好三个作业有效，所以答案是`3`。 

对于较大的奇值，例如`N = 99999`，不可能枚举字符串。 该算法仍然只执行一次模幂运算：$$3 \cdot 4^{49999} \pmod{10^9+3233}.$$运行时间仍然是指数的对数，这很容易足够快。
