 ## 解决方案
 ## 问题理解

 我们得到了一系列能量容器，每个容器都含有一定量的能量。 我们可以在容器之间转移能量，但每次转移效率都很低：如果我们将一定量的能量从一个容器转移到另一个容器，那么我们尝试转移的能量就会在这个过程中丢失一定比例的能量。 

目标是确定最大值$x$这样，在任何顺序的传输之后，每个容器都有可能准确地结束$x$能量单位。 

重新表述该问题的一个有用方法是从“泄漏下的重新分配”的角度进行思考。 我们不会随意改变总能量； 我们只是重新分配它，每次我们在容器之间转移能源时，我们都会付出一定比例的损失成本。 

这些限制足够大，任何尝试模拟传输的解决方案都立即不可行。 容器数量最多可达$10^4$，每个测试都涉及实值传输的算术，所需精度高达$10^{-6}$。 这种组合强烈表明答案是连续的，并且必须使用单调属性而不是直接构造来找到。 转移的强力模拟将涉及对成对运动和可能重复的平衡操作的推理，这将导致组合爆炸。 

一些边缘情况揭示了天真的贪婪重新分配失败的原因：

 如果所有容器已经具有相等的能量，例如输入```
3 50
2 2 2
```正确答案很明显$2$。 任何试图“转移盈余”的策略如果不认识到不需要转移，仍然可能执行不必要的转移并错误地减少可实现的价值。 

如果损失为零，例如```
2 0
1 11
```那么所有能量都完全守恒，答案一定是平均值$(1+11)/2 = 6$。 任何假设定向流约束或增量平衡的方法如果不能简化为总和守恒，则可能会失败。 

如果损失非常高，例如$k = 99$，几乎所有转移的能量都消失了。 在这种情况下，只有局部盈余很重要，超过一个非常小的值就不可能进行全局平均。 假设近乎完美的重新分配的算法会高估答案。 

## 方法

 直接方法将尝试模拟重新分配过程。 人们可以反复识别高于目标水平的集装箱，并将其剩余部分移至低于目标水平的集装箱。 每次传输都会根据损失百分比减少总能量。 虽然概念上很简单，但这在计算上是不可行的。 每个平衡步骤都依赖于其他步骤，并且潜在传输的数量不受保证效率的限制。 在最坏的情况下，这会演变成对所有容器的重复扫描和调整，导致元素数量至少呈二次方行为。 

关键的见解是停止考虑个人转让，而是关注全球可行性。 问题变成：对于建议的目标值$x$，我们是否可以验证是否可以至少以所有容器结束$x$，考虑到我们在转移过程中会损失能量？ 

如果一个容器有超过$x$，它有可以重新分配的剩余能量。 然而，它发送的内容中只有一小部分可以在其他地方使用。 这意味着盈余对其他集装箱赤字的贡献较小。 

这种观察导致了单调结构：如果某个值$x$是可以实现的，那么任何更小的值也是可以实现的。 这使得二分搜索成为一种自然的工具。 

对于固定的$x$，我们计算考虑传输损失后的总有效能量是否足以支持所有集装箱到达$x$。 这将问题减少到对每个候选值进行一次检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 模拟传输 | O(n²) 或更糟 | O(n) | 太慢了 |
 | 二分查找+可行性检查| O(n log A) | O(n log A) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算所有容器的总初始能量。 除了转移造成的损失外，这个总额永远不会改变，因此它充当全球预算。 
2. 修复候选值$x$，代表每个容器中的目标能量。 我们想测试它是否可以实现。 
3. 对于每个容器，考虑它如何与$x$。 如果它有超过$x$，多余的能量是可以转移的。 如果含量较少，则需要能量才能达到目标。 
4. 将所有盈余金额相加，定义为$\max(0, a_i - x)$。 这是可以从高容器中移出的总能量。 
5. 只有一小部分盈余能够在转移后幸存下来。 如果损失百分比是$k$，那么每个发送的单位贡献$(1 - k/100)$单位到接收方。 因此可用的传输能量会成比例地减少。 
6. 计算有效可用能量：$$\text{available} = \text{total\_sum} - \frac{k}{100} \cdot \text{surplus}$$7. 检查可用能量是否至少为$n \cdot x$。 如果是的话，那么就有可能分配能量，使每个容器达到$x$。 
8. 使用二分查找$x$在从$0$到$\max(a_i)$，精炼直至满足精度要求。 

### 为什么它有效

 关键的不变量是，能量只有在移动时才会消失，并且只有高于目标的剩余能量才需要移动。 所有值等于的任何有效的最终配置$x$可以被解释为一个过程，其中上面的盈余恰好是$x$被重新分配，并且所有其他单位保持静止。 这意味着唯一不可挽回的损失与必须转移的盈余金额成正比。 

因为可行性条件单调取决于$x$，增加$x$只会让条件更难满足。 这保证了二分搜索收敛到最大有效值，而不会丢失中间可能性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(x, a, n, k):
    total = 0.0
    surplus = 0.0
    
    for v in a:
        total += v
        if v > x:
            surplus += (v - x)
    
    lost = surplus * (k / 100.0)
    available = total - lost
    
    return available >= n * x

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    lo, hi = 0.0, max(a)
    
    for _ in range(60):
        mid = (lo + hi) / 2
        if can(mid, a, n, k):
            lo = mid
        else:
            hi = mid
    
    print(f"{lo:.10f}")

if __name__ == "__main__":
    solve()
```实现的核心是`can`函数，它将重新分配过程压缩为单个算术检查。 二分查找依靠可行性的单调性，反复细化候选答案。 

一个微妙的点是浮点精度就足够了，因为在小扰动下检查是稳定的，并且所需的误差容限是$10^{-6}$。 二分查找的六十次迭代轻松超出了精度要求。

 ## 边缘情况

 例如，当所有值都相同时`5 5 5 5`，可行性检查立即通过$x = 5$因为剩余为零。 该算法正确地避免了任何人为的减少。 

什么时候$k = 0$，剩余项完全消失。 可行性检查简化为将总和与$n \cdot x$，这正是求平均的条件。 二分查找自然收敛于算术平均值。 

什么时候$k$接近100，几乎所有剩余都消失了。 该算法仍然表现正确，因为可用能量几乎等于总初始能量，并且对于中等程度的可行性很快就会失败$x$。 这可以防止二分搜索高估重新分配能力。
