 ## 解决方案
 ## 问题理解

 我们得到一个一维景观，其中每个位置都有一个整数高度。 将其视为并排放置的一系列不同高度的垂直柱。 下雨时，水可能被困在较高的柱子之间，在较低的柱子上方形成积水“水池”。 

如果一个位置的左侧和右侧都有较高或相等的边界，则该位置属于某个池。 水池的水位由其两个边界边中较低的一侧决定，因为水会首先溢出较弱的一侧。 每个水池的体积等于该边界区域内地形上方滞留的水量。 

在所有这样的水池中，我们被要求找到表面水平最高的一个。 如果多个水池共享相同的最大表面高度，我们会将它们的体积相加。 

输入大小高达 100,000 个位置，因此任何尝试显式扩展每个可能的左右边界对的解决方案都会太慢。 对所有区间进行三次或二次枚举是不可行的，因为即使$O(n^2)$以这个规模来说已经太大了。 

棘手的部分是“池”不仅仅是任何区间，它是由局部最大值约束隐式定义的。 一个幼稚的错误是将每对峰视为形成一个有效的池，而不验证内部结构。 另一个常见的失败是全局计算截留水，然后错误地选择分段，从而丢失问题所需的每个池分组。 

打破朴素逻辑的边缘情况包括严格增加数组，例如`[1,2,3,4]`不存在泳池且答案应为零的地方，或者像这样的平坦高原`[5,5,5,5]`其中边界不明确，但仍然根据解释定义退化池。 另一种微妙的情况是同一最大级别的多个不相交池，必须将其相加而不是仅选择一个。 

## 方法

 强力解释将尝试识别每对有效的左右边界，并计算它们之间截留了多少水。 对于每对索引$l < r$，我们将检查内部每个位置左侧的最大高度是否至少与某个边界条件一样高，右侧也是如此。 如果有效，我们通过扫描内部并求和来计算捕获的体积$\min(\text{left max}, \text{right max}) - h[i]$。 

这种方法原则上是正确的，因为它直接对池的定义进行编码。 不过，需要检查$O(n^2)$候选间隔，并且每个验证本身可能会花费$O(n)$，导致$O(n^3)$在最坏的情况下。 即使优化为$O(n^2)$，它仍然失败$n = 10^5$。 

关键的观察是池不是独立的任意间隔。 每个位置自然受到两侧最近的大于或等于边界的约束。 这表明了一种单调结构：每个索引的左侧和右侧都有最近的“墙”，这些墙定义了不相交或嵌套的水域。 一旦我们有效地计算这些边界，每个水域就变得独立，我们可以在线性时间内计算其贡献。 

这将问题转换为经典的最近更大边界结构，可以使用单调堆栈来计算$O(n)$。 每个索引都贡献于由其左右约束定义的一个最大结构，并且从这些结构中我们可以导出池段及其表面高度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了 |
 | 单调堆栈+区域提取|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题转化为识别由最近的大于或等于边界形成的有界“水容器”。 

1. 对于每个索引，计算左侧最近的大于或等于元素。 这是使用单调递减堆栈来完成的。 我们维护索引，使得堆栈中的高度严格递减。 当我们遇到更高或相等的高度时，我们会弹出，直到恢复不变量。 
2. 对于每个索引，使用相同的想法但从右向左扫描，计算右侧最近的大于或等于元素。 这为我们的每个位置提供了正确的边界。 
3. 现在，每个位置都知道其最近的左右边界，这些边界上方可能含有水。 两个边界都存在的地方就存在一个有效的池。 
4. 对于充当“底部”（即低于两个边界）的每个位置，计算水位为其两个边界高度的最小值。 
5. 根据计算的水位对位置进行分组。 由于该问题要求最高的地表水池，因此我们跟踪遇到的最大水位。 
6. 对于处于最高水位的每个水池，通过对属于该水池的所有位置的水位和高度之间的差值求和来计算其体积。 
7. 输出所有达到最高水位的水池的体积总和。 

### 为什么它有效

 单调堆栈确保对于每个索引，我们都能正确识别保证成立的最近边界。 任何更远的边界都是无关紧要的，因为水会在更近的等于或更高的屏障处更早溢出。 这将阵列划分为独立的候选盆地，其中水位仅由当地约束确定。 由于每个有效池都必须受到此类最近约束的限制，因此不会错过任何有效配置，也不包含无效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    h = list(map(int, input().split()))

    left = [-1] * n
    right = [n] * n

    stack = []

    for i in range(n):
        while stack and h[stack[-1]] < h[i]:
            stack.pop()
        if stack:
            left[i] = stack[-1]
        stack.append(i)

    stack = []

    for i in range(n - 1, -1, -1):
        while stack and h[stack[-1]] < h[i]:
            stack.pop()
        if stack:
            right[i] = stack[-1]
        stack.append(i)

    max_level = 0
    volume = {}

    for i in range(n):
        if left[i] != -1 and right[i] != n:
            level = min(h[left[i]], h[right[i]])
            if level > h[i]:
                max_level = max(max_level, level)
                volume.setdefault(level, 0)
                volume[level] += level - h[i]

    ans = 0
    for lvl, val in volume.items():
        if lvl == max_level:
            ans += val

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先使用两个单调堆栈通道计算最近的大于或等于边界。 左遍找到左边最接近的可以容纳水的边界，右遍对称地做同样的事情。 

边界计算后，每个位置都被视为对盆地有潜在贡献。 每个位置的水位由其两个边界中较弱的一个确定。 只有严格低于该水平的头寸才会对交易量产生影响。 

我们按水位分组累积贡献，以便我们以后可以仅选择最高级别的池。 这符合仅考虑最高表面海拔池并对平局进行求和的要求。 

一个微妙的点是允许等高边界，因此堆栈条件使用`<`而不是`<=`。 使用`<=`会错误地丢弃平坦的高原作为有效的墙壁。

 ## 边缘情况

 对于像这样的单元素输入`[5]`，两个边界阵列至少在一侧仍然无效，因此没有位置满足具有两个围墙。 该算法会跳过所有贡献，正确生成零。 

对于像这样的严格递增序列`[1,2,3,4]`，每个元素的右边界都存在，但左边界崩溃，从而阻止任何有效的池形成。 该堆栈确保不会创建错误配对。 

对于平坦的序列，例如`[3,3,3,3]`，等高元素成为相互边界，但不会产生内部倾斜，因为`level == height`产生零贡献。 条件`level > h[i]`防止错误的体积积累。
