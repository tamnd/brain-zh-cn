 ## 解决方案
 ## 问题理解

 我们在数轴上得到一组放置鸡蛋的位置。 我们还可以放置固定数量的热灯，每个热灯位于一个整数坐标处。 每个鸡蛋都由其最近的灯“服务”，并且配置的质量是通过从任何鸡蛋到其最近的灯的最大距离来衡量的。 

任务最多放置$k$灯，使该最大距离尽可能小。 

思考这个问题的一个有用方法是，每个灯都覆盖其附近的鸡蛋，并且每个鸡蛋必须落在至少一个灯的某个半径范围内。 我们正在尝试最小化允许使用最多覆盖所有鸡蛋的最小半径$k$中心。 

约束条件很大，最多可达$5 \cdot 10^5$位置，这排除了任何尝试评估所有位置或模拟每种配置距离的解决方案。 任何二次或偶数$O(nk)$不会起作用。 我们至少需要$O(n \log n)$， 通常$O(n \log \max a_i)$或者$O(n \log n)$。 

一种天真的方法会尝试在鸡蛋之间或鸡蛋之间放置所有可能的灯并计算覆盖范围，但这会立即失败，因为候选配置的数量会组合增长。 

当鸡蛋间距不均匀时，就会出现贪婪直觉的一个微妙的失败案例。 例如，如果大多数鸡蛋都聚集在一起，但其中一个距离很远，则简单的等分划分可能会导致灯的放置效果不佳。 

输入如下：```
5 2
0 1 2 100 101
```幼稚的“按计数均匀分割”方法可能会错误地将灯放置在索引 2 和 101 簇边界附近，从而错过最佳分组。 正确的解决方案取决于距离，而不是计数。 

## 方法

 蛮力解释是考虑半径$R$并询问我们是否最多可以放置$k$灯让每个鸡蛋都在距离之内$R$一些灯。 如果我们能够有效地测试这一点，我们就可以进行二分搜索$R$。 

对于固定的$R$，我们可以想象每个灯放置在某个位置，覆盖一段长度的区间$2R$。 由于所有鸡蛋都在$R$灯的数量必须位于这样的间隔内，每个灯有效地覆盖了分选鸡蛋位置的连续部分。 

这会导致贪婪检查：从最左边未覆盖的鸡蛋开始，以最佳方式放置灯以在距离内覆盖尽可能多的鸡蛋$R$，然后重复。 这是最佳的，因为在允许的覆盖窗口内尽可能靠右放置灯总是优先于早期的放置。 

尝试所有放置的强力版本是不可行的，因为放置点的子集数量是指数级的。 关键的见解是，对于固定半径，结构变成排序线上的区间覆盖，这在线性时间内是贪婪可解的。 

因此，我们将优化问题转化为决策问题，然后对答案进行二分查找。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力安置 | 指数| O(n) | 太慢了 |
 | 二分查找+贪心检查 | O(n log V) | O(n) | 已接受 |

 ## 算法演练

 ### 第 1 步：对鸡蛋位置进行排序

 排序是必要的，因为覆盖范围只有按顺序排列才有意义。 如果不排序，我们就无法推理连续的段。 

### 第 2 步：定义函数`can(R)`我们测试半径是否$R$最多使用就足够了$k$灯。 

### 步骤3：贪婪覆盖过程

 从最左边尚未覆盖的鸡蛋开始。 

把那个鸡蛋放在位置上$x$。 将灯放在尽可能靠右的位置，同时仍然覆盖$x$，这意味着将其放置在$x + R$。 然后，该灯将覆盖所有鸡蛋，直至$x + 2R$。 

这种选择是最佳的，因为任何覆盖的灯$x$必须位于$[x - R, x + R]$，向右移动只会增加向右的覆盖范围，而不会失去$x$。 

### 步骤 4：跳过覆盖的鸡蛋

 放置一盏灯后，将指针前进到第一个鸡蛋之外$x + 2R$。 这模拟在一次操作中消耗整个段。 

### 第 5 步：计算所用灯的数量

 如果所需灯的数量超过$k$， 然后$R$太小了。 

### 第 6 步：二分查找答案

 搜索最小值$R$从 0 到鸡蛋之间的最大可能距离。 每次检查都以线性时间运行。 

### 为什么它有效

 该算法依赖于单调属性：如果半径$R$有效，任何更大的半径也有效。 这允许二分搜索。 贪婪覆盖确保对于任何固定的$R$，我们通过在每个步骤中始终覆盖尽可能远的位置来最大限度地减少所需的灯的数量。 这可以防止在次优的部分覆盖上浪费灯，并保证如果贪婪策略需要超过$k$灯，没有其他替代放置可以做得更好。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(a, k, r):
    n = len(a)
    i = 0
    used = 0

    while i < n:
        used += 1
        if used > k:
            return False

        start = a[i]
        cover_end = start + 2 * r

        while i < n and a[i] <= cover_end:
            i += 1

    return True

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    a.sort()

    lo, hi = 0, a[-1] - a[0]

    while lo < hi:
        mid = (lo + hi) // 2
        if can(a, k, mid):
            hi = mid
        else:
            lo = mid + 1

    print(lo)

if __name__ == "__main__":
    solve()
```排序步骤确保所有贪婪区间都是连续的。 这`can`函数是核心决策程序，它将鸡蛋贪婪地分组为最小数量的大小区间$2R$。 然后二分查找找到最小的可行半径。 

一个微妙的点是灯的位置不需要显式存储。 我们只需要通过相对于最左边未覆盖的鸡蛋进行最佳放置而产生的覆盖间隔。 

另一个常见的错误是使用$R$直接作为间隔长度而不是$2R$。 灯在位置$x+R$封面$[x, x+2R]$， 不是$[x, x+R]$。 

## 工作示例

 ### 示例 1

 输入：```
3 20
0 5 7
```已排序的数组`[0, 5, 7]`。 

我们测试半径的可行性。 

| 步骤| 我| 位置| 二手灯具| 报道结束|
 | ---| ---| ---| ---| ---|
 | 开始 | 0 | 0 | 1 | 0 + 2R |
 | 移动| 1 | 5 | 1 | 0 + 2R |
 | 移动| 2 | 7 | 1 | 0 + 2R |
 | 结束 | 3 | - | 1 | - |

 对于足够大的$R$，一盏灯就够了。 如果$R = 1$，覆盖范围为 2，因此第一个灯仅覆盖 0，需要另一个灯覆盖 5 和 7，从而提供 2 个灯。 二分查找找到最小值$R$使灯保持在限制范围内。 

### 示例 2

 输入：```
5 2
-2 -1 0 1 4
```排序：`[-2, -1, 0, 1, 4]`尝试小半径，比如说$R = 1$。 

| 步骤| 我| 开始 | 封面结束 | 二手 |
 | ---| ---| ---| ---| ---|
 | 1 | 0 | -2 | 0 | 1 |
 | 2 | 3 | 1 | 3 | 2 |
 | 3 | 4 | 4 | - | 3（失败）|

 我们需要 3 盏灯，所以$R = 1$是不可行的。 

为了$R = 2$：

 | 步骤| 我| 开始 | 封面结束 | 二手 |
 | ---| ---| ---| ---| ---|
 | 1 | 0 | -2 | 2 | 1 |
 | 2 | 4 | 4 | 6 | 2 |

 现在它可以与 2 个灯配合使用，匹配$k$。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n \log V)$| 排序加半径范围内的二分查找； 每个可行性检查都是线性的 |
 | 空间|$O(n)$| 已排序位置的存储 |

 取值范围$V$取决于$2 \cdot 10^9$，但二分查找只需要大约 31 次迭代。 和$n = 5 \cdot 10^5$，这完全符合限制。

 ## 边缘情况

 单个鸡蛋输入如`n = 1`始终返回 0，因为不存在距离。 该算法处理这个问题是因为二分搜索范围崩溃了并且`can(0)`一盏灯立即返回 true。 

所有鸡蛋等距的情况，例如`[1,2,3,4,5]`，强制算法证明分组是严格按距离而不是计数进行的。 为了`k = 2`，贪婪过程形成两个连续的段，并且从不尝试混合相距较远的元素，从而产生最佳分割。 

广泛分离的异常值，例如`[0,1,2,3,100000]`，测试贪婪策略不会在密集区域浪费灯。 第一个灯罩`[0..3]`，第二个仅覆盖异常值，任何将第一个灯向右移动的尝试都会减少左侧的覆盖范围，这是不变量所禁止的。
