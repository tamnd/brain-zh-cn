 ## 解决方案
 ## 问题理解

 我们得到了一组鸡蛋，其中每个鸡蛋都需要特定的时间才能完全煮熟。 雅各布有固定的总时间预算和一个锅，这意味着他一次只能煮一个鸡蛋，不能重叠。 一旦他开始煮一个鸡蛋，他必须在煮另一个鸡蛋之前花费完整的烹饪时间。 

任务是确定如果他以最佳方式安排鸡蛋，他可以完全煮熟多少个鸡蛋。 他拥有的关键自由度是煮鸡蛋的顺序，目标是在总可用时间内最大化完成鸡蛋的数量。 

鸡蛋数量的限制最多为10万个，而时间值和总时间可以高达20亿。 这立即排除了任何显式尝试所有排列或模拟所有子集的方法。 甚至一个$O(n^2)$如果涉及重复扫描或选择，方法就会太慢。 

当人们认为按照输入顺序取出鸡蛋就足够了时，自然就会出现陷阱。 例如，如果输入是：```
t = 10
c = [9, 1, 1]
```按顺序烹饪只能产生 1 个鸡蛋，因为 9 个鸡蛋几乎消耗了所有时间。 但最佳选择是先拿两个小鸡蛋，总共 2 个鸡蛋。 这表明排序是至关重要的。 

另一个微妙的边缘情况是总时间与组合完全匹配：```
t = 6
c = [4, 2, 2]
```正确的策略可以同时烹调所有三个，但如果首先采用 4 个，则天真的按顺序贪婪方法可能会失败。 

因此，真正的问题是选择烹饪时间的子集，其总和尽可能大而不超过$t$，同时最大化所选元素的数量。 

## 方法

 最直接的想法是尝试鸡蛋的每个子集，计算其总烹饪时间，并跟踪适合时间的最大子集。 这是正确的，因为它明确地评估了所有可能性。 然而，子集的数量呈指数增长$2^n$，甚至对于$n = 100000$，这是完全不可行的。 甚至限制尺寸组合$k$没有帮助，因为迭代所有组合仍然会爆炸。 

更结构化的观察来自于注意真正决定一组鸡蛋是否可行的因素：只有它们的烹饪时间总和才重要，而不是它们的顺序。 由于每个鸡蛋对总时间都有积极贡献，因此容纳尽可能多的鸡蛋的最佳方法是最大限度地减少每个所选鸡蛋组尺寸的总成本。 这自然建议首先选择最短的烹饪时间。 

如果我们按非递减顺序对烹饪时间进行排序，那么从最小的向上取鸡蛋可以确保对于任何前缀，我们都可以在尽可能短的总时间内最大化鸡蛋的数量。 任何与此顺序的偏差都会用较大的成本替换较小或相同的成本，这只会减少在时间限制内适合的鸡蛋数量。 

排序后，我们只需从左到右累积时间，直到添加另一个鸡蛋超出预算。 此时达到的前缀长度是最佳的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 |$O(2^n)$|$O(n)$| 太慢了 |
 | 排序+贪心前缀和|$O(n \log n)$|$O(1)$或者$O(n)$| 已接受 |

 ## 算法演练

 1.读取鸡蛋的数量$n$和总可用时间$t$，然后阅读烹饪时间列表。 这就确定了资源约束和个人成本。 
2. 按升序对烹饪时间进行排序。 此步骤确保我们始终首先考虑最便宜的可用鸡蛋，这是最大限度地满足预算的必要条件。 
3. 初始化两个变量：迄今为止使用的总烹饪时间的运行总和，以及已煮熟的鸡蛋数量的计数器。 
4. 从最小到最大迭代排序的烹饪时间列表。 
5. 对于每个鸡蛋，检查添加其烹饪时间是否会超过总时间$t$。 如果没有，则通过将其时间添加到运行总和并递增计数器来将其包含在计划中。 
6. 如果当前鸡蛋的添加量超过限制，请立即停止。 任何其他鸡蛋甚至更大或相等，因此不能在不违反约束的情况下将它们包括在内。 
7. 输出计数器为可煮鸡蛋的最大数量。 

关键的想法是，我们永远不会重新考虑之前的决定，因为分类可以确保一旦无法容纳较大的鸡蛋，那么后面的鸡蛋也无法容纳。 

### 为什么它有效

 正确性依赖于以下事实：所有鸡蛋成本都是独立且可加的，并且我们在单一总和约束下最大化基数。 在任何最佳解决方案中，如果选择成本较高的蛋，而存在成本较小的未使用蛋，则交换它们不会增加总时间，也不会减少蛋的数量。 重复应用此交换参数可将任何最佳解决方案转换为选择排序列表前缀的解决方案。 该前缀正是贪婪扫描所构造的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, t = map(int, input().split())
    c = list(map(int, input().split()))
    
    c.sort()
    
    total = 0
    count = 0
    
    for x in c:
        if total + x <= t:
            total += x
            count += 1
        else:
            break
    
    print(count)

if __name__ == "__main__":
    solve()
```该解决方案首先对烹饪时间进行排序，以便我们始终首先处理最小的剩余成本。 这是使贪婪方法有效的结构转变。 然后循环累积时间直到违反约束。 早期休息很重要，因为一旦我们超出预算，由于排序，所有后续值至少都一样大。 

一个常见的错误是忘记排序，这会立即破坏正确性，因为贪心假设完全依赖于递增顺序。 另一个问题是没有提前停止，这对于正确性来说是不必要的，但会稍微降低性能。

 ## 边缘情况

 一个关键的边缘情况是根本无法煮熟鸡蛋。 对于输入：```
n = 1, t = 3
c = [10]
```排序后，第一个元素已经超出预算。 循环立即中断，而不增加计数器，产生 0。这证实了算法正确地处理了完全不可行的情况。 

另一个边缘情况是所有鸡蛋都相同并且完全符合预算的倍数。 为了：```
n = 3, t = 6
c = [2, 2, 2]
```排序使列表保持不变。 该算法将每个鸡蛋依次相加，正好达到 6 个。由于允许相等，因此它包括所有三个鸡蛋。 这演示了边界相等条件的正确处理，其中总和与限制完全匹配，而没有溢出或过早停止。
