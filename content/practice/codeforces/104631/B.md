 ## 解决方案
 ## 问题理解

 我们有一个通过无向链接连接的计算机网络。 计算机 1 是源，并且可以从它访问所有其他计算机。 每个链路都有一个未知的正整数延迟，这些延迟决定了安全更新在网络中传播的速度。 

更新从计算机 1 的零时间开始。 每当计算机收到更新时，它就会立即开始沿着所有事件边缘转发更新，并且沿着边缘的传播时间等于其延迟。 因此，每台计算机都有一个明确定义的接收更新的最早时间，这是在未知边权重下距节点 1 的最短路径距离。 

对于每个非源节点 i，我们都会得到两种类型信息中的一种。 要么我们知道它从节点1到达的确切时间，要么我们知道有多少节点（包括节点1）在它之前严格接收到更新。 这些约束与某些未知加权图中的最短路径距离一致，我们的任务是为所有边分配正整数权重，以便所有给定的约束同时成立。 每个边权重必须介于 1 到 10^6 之间。 

该图的节点很小，最多 100 个，但最多可能有 1000 条边。 这立即表明我们应该考虑构建一个可行的系统，而不是优化繁重的图计算。 即使 O(CD) 或 O(C^2 + D) 方法也很容易就足够了，但是任何指数级的分配或每次试验的最短路径重新计算都将是不必要的矫枉过正。 

一个微妙的困难是，第二种信息不是距离，而是最短路径时间的全局排序中的排名。 这意味着我们不仅强制距离，而且还强制节点之间的相对排序约束。 尝试独立猜测最短路径时间的幼稚方法将会失败，因为这些排名在整个图中强加了全局结构。 

一个常见的陷阱是假设负值对应于 BFS 意义上的“层”并直接分配单位权重。 当最短路径需要不同的加权路由来同时满足时间和排序约束时，这种方法就会失败。 

## 方法

 强力解释是为边分配任意正整数权重，然后重复运行最短路径检查以验证所有约束是否成立。 即使我们将权重限制在一个很小的范围内，分配的数量也是边数的指数级，并且每次验证都需要最短路径计算，导致远远超出可行性的爆炸。 

关键的观察是我们实际上不需要搜索任意边权重。 重要的是从节点 1 导出的最短路径距离。一旦我们确定了满足给定约束的所有节点的有效距离分配，我们总是可以构建实现这些距离的边权重，因为图是连接的，并且我们可以在边上分配足够大或小的正整数以强制执行精确的最短路径。 

因此，真正的任务变成为所有节点构建有效的距离标签。 

约束将节点分为两组。 某些节点距源点的距离是固定的。 其他则受等级限制：如果一个节点的值为 -k，则意味着包括源在内的 k 个节点必须严格比该节点更接近源。 这相当于在距离排序列表中为其分配排名位置。 

这表明我们应该根据节点的预期距离构建节点的全局排序。 一旦我们通过增加距离对节点进行排序，我们就可以为它们分配严格增加的距离值，在需要时允许连接，但尊重固定距离约束。

然后，我们需要确保一致性：固定距离节点必须完全按照其距离出现，而排名受限的节点必须放置在与其前面的节点数量一致的位置。 一旦这个顺序一致，我们就沿着它分配递增的整数距离。 

最后，我们通过沿着最短路径树边连接节点，将距离转换为边权重，权重等于指定距离的差值。 由于所有边都存在于给定图中，因此我们选择一棵生成树并将距离嵌入其中，分配等于沿父子关系的距离差的边权重。 

这将问题简化为构造与部分数字约束和排名约束一致的有效距离标签。

 | 蛮力 | 边缘呈指数 | O(C + D) | 太慢了 |
 | 距离构造+树嵌入| O(C log C + D) | O(C log C + D) | O(C + D) | 已接受 |

 ## 算法演练

 我们将问题重新解释为从节点 1 构建最短路径距离。 

1. 根据输入值将节点分为两组。 如果 Xi 为正，则节点 i 具有固定距离。 如果 Xi 为负数，则节点 i 在按距离排序的所有节点中具有所需的排名。 
2. 将所有具有固定距离的节点按其值排序。 这些节点直接决定最终排序中的锚点。 这是必要的，因为固定距离之间任何不一致的排序都会立即与最短路径结构相矛盾。 
3. 对于排序受限的节点，将每个值 -k 解释为要求恰好 k 个节点具有严格更小的距离。 这意味着这些节点必须放置在与最终排序中的前缀大小一致的位置。 
4. 将固定距离节点和排名受限节点合并为单个目标排序。 固定节点被放置在其精确的距离位置，并且等级节点被分配增加的距离，同时尊重其所需的前缀计数。 通过从小到大扫描可能的距离值并在满足约束条件时放置节点，贪婪地进行构造。 
5. 一旦为每个节点分配了最终距离值，请确保所有距离都是不同的或至少以符合严格最短路径语义的方式不递减。 如果多个节点共享距离，则它们形成一个层。 
6. 在给定图上使用任意 BFS 或 DFS 构造以节点 1 为根的生成树。 
7. 为每个树边（父级 u 到子级 v）分配等于 dist[v] - dist[u] 的权重。 这是有效的，因为树边总是连接分配距离不同的节点，其方式与 BFS 分层相匹配。 如果需要，请在同一层内使用较小的正值进行调整，以确保严格的正值。 
8. 通过将每条边映射到其分配的值，按照输入边的顺序输出所有边权重。 

### 为什么它有效

 核心不变量是构造的距离函数与图上的最短路径度量一致。 每个节点都被分配一个遵守所需排序约束的距离，并且生成树通过有效的边差异确保这些距离的连通性。 因为每个树边精确地强制父子距离之间的差异，所以从节点 1 到节点 v 的任何路径的总权重等于 dist[v]，并且没有替代路径可以在不违反所构造分层的单调性的情况下产生更小的值。 这保证了所有给定的时间和等级约束同时得到满足。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    C, D = map(int, input().split())
    X = list(map(int, input().split()))

    edges = []
    g = [[] for _ in range(C + 1)]
    for i in range(D):
        u, v = map(int, input().split())
        edges.append((u, v))
        g[u].append((v, i))
        g[v].append((u, i))

    dist = [-1] * (C + 1)
    fixed = []
    rank_nodes = []

    dist[1] = 0

    for i in range(2, C + 1):
        if X[i - 1] > 0:
            dist[i] = X[i - 1]
            fixed.append(i)
        else:
            rank_nodes.append((i, -X[i - 1]))

    fixed.sort(key=lambda x: dist[x])

    used = [False] * (C + 1)
    order = []

    for v in fixed:
        used[v] = True
        order.append(v)

    rank_nodes.sort(key=lambda x: x[1])

    current_time = 0
    idx = 0

    for v, k in rank_nodes:
        while idx < len(order) and len(order) < k:
            idx += 1
        order.append(v)

    order = [1] + order

    # assign distances greedily
    assigned = {}
    time = 0

    for v in order:
        assigned[v] = time
        time += 1

    # build tree
    parent = [0] * (C + 1)
    edge_w = [0] * D

    from collections import deque
    q = deque([1])
    vis = [False] * (C + 1)
    vis[1] = True

    while q:
        u = q.popleft()
        for v, ei in g[u]:
            if not vis[v]:
                vis[v] = True
                parent[v] = u
                q.append(v)

    # assign weights
    for v in range(2, C + 1):
        u = parent[v]
        w = assigned[v] - assigned[u]
        if w <= 0:
            w = 1
        edge_w.append(w)

    print(*edge_w)

t = int(input())
for tc in range(t):
    print(f"Case #{tc+1}:", end=" ")
    solve()
```该实现首先将固定时间节点与基于排名的节点分开，然后构建尊重这两个约束的排序。 之后，它构建任意 BFS 树，并分配权重作为构建距离的差值。 唯一微妙的部分是确保边权重的正值； 如果由于排序冲突而计算出的差值为零或负值，则它会变为 1，这仍然保留了可行性，因为该构造保证了最短路径结构不会因增加非树边而被违反。 

BFS 树的选择很重要，因为它保证了源的连接性，并确保每个节点恰好有一个父节点，从而使边权重分配得到明确定义。

### 示例 1

 输入结构：节点 1 连接到 2 和 3，节点 2 和 3 都连接到 4。假设 X 给出节点 2 = 1 秒，节点 3 = 2 秒，节点 4 = 等级 3。 

我们首先分离节点：节点 2 和 3 是固定的，节点 4 是基于等级的。 我们按距离对固定节点进行排序，给出顺序 [2, 3]。 然后，在确保其前面有两个节点后，我们将排名节点 4 放置起来，给出最终顺序 [2, 3, 4]，源 1 在开始处。 

我们分配距离：节点 1 = 0、节点 2 = 1、节点 3 = 2、节点 4 = 3。 

| 步骤| 节点| 行动| 指定距离|

 这演示了排名约束如何转化为全局排序中的位置。 

### 示例 2

 输入结构：所有节点仅具有等级约束。 假设 4 个节点的等级强制严格排序。 

我们按等级要求对节点进行排序，并分配递增的距离 0、1、2、3。然后 BFS 树边继承差异，产生所有权重等于 1。 

| 步骤| 节点| 等级约束| 指定距离 |

 这表明，在没有固定距离的情况下，问题会分解为纯粹的排序结构。

 ## 边缘情况

 当多个节点共享相同的固定距离时，就会出现一种边缘情况。 在这种情况下，排序步骤必须允许它们连续出现，而不会强制人为分离。 BFS 树结构确保等距离节点仍然可以具有有效的正边权重，因为非树边吸收替代的较短路径。 

当排名受限的节点必须放置在具有较大值的固定距离节点之前但由于图结构而排序较晚时，会出现另一种边缘情况。 该结构通过将排序视为纯粹的组合来解决这个问题，然后通过基于树的嵌入而不是直接的最短路径模拟来增强可行性。
