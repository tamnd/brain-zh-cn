 ## 解决方案
 ## 问题理解

 我们在平面上得到一组点，每个点代表一个孔。 一旦我们选择了球的起始位置和方向，它就会沿着无限直线移动。 每当球击中一个洞时，它都会以以下两种方式之一表现：如果该洞没有与另一个洞配对，则球停在那里； 如果它配对了，它会立即传送到它的伙伴洞并继续朝完全相同的方向移动。 

每个洞最多可以参加一次配对，并且配对是无向的。 我们的自由度很大：我们可以选择球的起始位置（只要它不是正好在一个洞上）、方向以及要连接哪些不相交的洞对。 球接触的每个不同的洞，无论是进入、离开还是停在那里，都会计入一次得分。 目标是最大化这个数字。 

关键的抽象是，一旦方向固定，球就会对其沿该方向相交的所有孔产生线性排序。 虫洞使我们能够按顺序从一个位置“跳”到另一个位置，从而可能创建长链，以受控的方式多次重新访问同一条线。 

这些限制因素非常重要。 最多有 100 个洞，任何试图暴力破解所有配对配置的方法都已经暗示了组合爆炸，因为完美匹配的数量增长大致如下$(N-1)!!$，即使在$N = 20$。 这立即排除了直接枚举所有配对配置的可能性。 

一个微妙的边缘情况是所有孔都位于一条线上。 在这种情况下，方向选择将几何图形折叠成严格的排序，并且问题简化为选择最大化沿路径遍历的配对。 假设固定起始孔的简单模拟在这里可能会失败，因为最佳起始可能在孔之间，从而允许遍历命中排序中不同的第一个元素。 

当不使用配对时，会出现另一种边缘情况。 人们可能会错误地认为没有虫洞，只能触及一个洞，但通过仔细放置起点，如果方向对齐并且起点位于该方向上的第一个洞之前，则可以顺序访问多个共线的洞。 

## 方法

 蛮力的观点是修复一切：选择一个方向，选择一个起始位置，选择一对洞，然后模拟球的运动并计算访问过的洞。 即使我们通过考虑所有成对的孔来离散方向，我们仍然面临$O(N^2)$方向，并且对于每个方向，枚举所有配对都会给出一个超指数因子。 即使模拟单个配置也是如此$O(N)$，所以这种方法很快就变得不可行。 

关键的简化来自于将几何结构与配对结构分开。 一旦我们确定了方向，每个孔都会投影到一条线上，通过投影坐标给出总顺序。 球永远不会按照这个顺序“任意跳跃”； 它总是沿着线向前移动，除非虫洞将其重定向到另一个位置，但仍保持方向。 

这将问题转化为关于沿线序列的推理：如果允许将点配对并立即在配对位置之间跳转，从而有效地将排序顺序的片段缝合在一起，我们希望最大化可以访问的不同点的数量。 

关键的观察是，对于固定方向，最优策略仅取决于投影的顺序，而不取决于实际坐标。 这将几何问题简化为点排列的组合问题。 

对于由一对点引起的每个方向，我们通过孔与该方向向量的点积对所有孔进行排序。 然后，我们在此排序上使用区间 DP 计算最佳可能的遍历，其中将两个孔有效地配对允许我们在位置之间跳转并继续遍历。 该结构类似于带有允许连接可达段的过渡的间隔上的最大匹配。 

我们评估所有$O(N^2)$候选方向，计算每个排序的最佳分数，并取最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | N 中的指数（由于匹配而呈超指数） | O(N) | 太慢了 |
 | 方向+ DP 超过订购 | O(N^3 log N) | O(N^3 log N) | O(N^2) | O(N^2) | 已接受 |

 ## 算法演练

 我们通过枚举由成对的孔定义的候选方向来继续，因为最佳方向总是可以与某些点对对齐。 

对于每个方向，我们将所有孔投影到该方向向量上，并按投影值对它们进行排序，获得线性顺序。 

然后，我们将问题视为选择如何在此有序列表中配对索引以及如何遍历它们以最大化不同的访问节点。 

我们按排序顺序在间隔上定义动态编程状态。 状态表示我们从给定间隔配置开始可以收集的不同孔的最大数量，假设我们从最左边的相关点进入。 

转换考虑跳过一个点，或者将一个点与另一个稍后的点配对，这会创建一个跳跃，允许从配对位置继续，同时保留方向。 每个配对有效地将问题分成不相交间隔上的独立子问题。 

我们计算所有区间的 DP$[l, r]$，从较小的间隔更新到较大的间隔。 

处理完所有方向后，我们取最大的DP结果。 

### 为什么它有效

 固定方向会将问题压缩为一维排序，其中除了虫洞跳跃之外，运动是单调的。 任何有效的遍历都对应于通过配对连接的区间遍历序列，并且任何配对都遵循排序顺序，因为在传送后方向被保留。 这保证了每个可行的行走都对应于 DP 区间的有效分解，相反，每个 DP 构造对应于平面中的可实现轨迹。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        pts = [tuple(map(int, input().split())) for _ in range(n)]

        if n == 1:
            print(1)
            continue

        # generate candidate directions from pairs
        dirs = []
        for i in range(n):
            for j in range(i + 1, n):
                dx = pts[j][0] - pts[i][0]
                dy = pts[j][1] - pts[i][1]
                if dx == 0 and dy == 0:
                    continue
                dirs.append((dx, dy))

        def norm(dx, dy):
            from math import gcd
            g = gcd(dx, dy)
            dx //= g
            dy //= g
            if dx < 0 or (dx == 0 and dy < 0):
                dx, dy = -dx, -dy
            return dx, dy

        seen_dirs = set()
        uniq_dirs = []
        for dx, dy in dirs:
            nd = norm(dx, dy)
            if nd not in seen_dirs:
                seen_dirs.add(nd)
                uniq_dirs.append(nd)

        def solve_dir(dx, dy):
            proj = []
            for i, (x, y) in enumerate(pts):
                proj.append((x * dx + y * dy, i))
            proj.sort()

            order = [i for _, i in proj]
            pos = {order[i]: i for i in range(n)}

            # DP over intervals
            dp = [[0] * n for _ in range(n)]

            for i in range(n):
                dp[i][i] = 1

            for length in range(2, n + 1):
                for l in range(n - length + 1):
                    r = l + length - 1
                    best = 1
                    best = max(best, dp[l + 1][r])
                    best = max(best, dp[l][r - 1])

                    for k in range(l + 1, r + 1):
                        best = max(best, dp[l][k - 1] + dp[k][r])

                    dp[l][r] = best

            return dp[0][n - 1]

        ans = 1
        for dx, dy in uniq_dirs:
            ans = max(ans, solve_dir(dx, dy))

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先读取所有点并生成由点对定义的所有可能的方向向量。 每个方向都经过标准化，以避免重复缩放和符号。 这很重要，因为 DP 仅取决于排序，而不取决于大小。 

对于每个方向，我们使用点积将点投影到方向向量上，然后对它们进行排序。 这会产生与沿该方向的运动一致的线性排序。 

DP表`dp[l][r]`计算此排序的连续段内可实现的最佳访问孔数。 单个元素初始化为 1。更大的间隔是通过从一侧延伸或分割间隔来构建的，反映我们是顺序遍历还是使用虫洞引起的跳跃。 

最后，我们评估所有方向并取最大值。 

关键的实施细节是方向的标准化。 如果没有它，对称方向将被重新计算多次，从而增加运行时间。 另一个微妙之处是投影排序是稳定的，并且为 DP 抽象定义了正确的遍历顺序。 

## 工作示例

 考虑一个在水平线上具有三个共线点的简单情况。 投影顺序已按 x 坐标排序。 

| 步骤| 间隔| dp值|
 | --- | --- | --- |
 | 初始化| [0,0],[1,1],[2,2] | 各 1 个 |
 | 长度 2 | [0,1]| 2 |
 | 长度 3 | [0,2]| 3 |

 这表明，在没有虫洞或简单配对的情况下，如果在第一个点之前选择起始位置，我们仍然可以遍历所有点。 

现在考虑三角形配置。 对于选定的方向，投影可以将点排序为 A、B、C。DP 可以按顺序从 A 到 B 到 C，也可以通过配对进行拆分和重新组合。 最佳价值来自组合子区间，表明虫洞有效地允许合并不相交的遍历段。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(N^4) 最坏情况 | O(N^2) 个方向，每个方向都有 O(N^2) DP |
 | 空间| O(N^2) | O(N^2) | 每个方向的 DP 表 |

 和$N \le 100$，这是边界，但在慷慨的限制下是可以接受的，特别是因为许多方向由于标准化而崩溃。 

该方法很适合，因为在典型的分布中，DP 占主导地位，并且每个测试用例的操作数保持在几百万次之内。

 ## 边缘情况

 对于单个孔，DP 正确初始化并立即返回 1，因为不可能进行区间扩展。 

对于两个孔，投影排序是微不足道的，并且 DP 考虑整个区间 [0,1]，产生 2。这证实了分割转换正确处理简单配对。 

对于共线点，排序保留了自然顺序，并且 DP 合并间隔而不需要虫洞，这表明该解决方案不依赖于配对来进行。 

对于多个方向等效的对称配置，归一化可确保避免冗余 DP 计算，并且结果在相同的排序中保持一致。
