 ## 解决方案
 ## 问题理解

 我们得到了二维平面上的一组点。 每个点代表我们可以在蛋糕上放置一块巧克力片的可能位置。 任务是为其中一些点分配颜色，但限制是所有相同颜色的碎片必须形成有效的非退化三角形，这意味着所选的三个点不得位于一条直线上。 

每种颜色必须恰好用在三个不同的点上，并且这三个点必须形成一个面积为正的三角形。 一个点最多只能整体使用一次，因此不同的颜色必须使用不相交的点三元组。 

目标是最大化我们可以从给定的点形成多少个这样的有效彩色三角形。 

从约束的角度来看，点数最多为 1000。这已经排除了任何尝试以天真的方式直接检查所有大小为 3 的子集（如果重复次数过多）的解决方案，但单次 O(n^3) 扫描仍然是可接受的，仅适用于三角形验证，不适用于组合分组。 真正的瓶颈是决定如何将点划分为尽可能多的非共线三元组，这从根本上来说是一个几何分组问题，而不是纯粹的计数问题。 

当许多点共线时，会出现微妙的边缘情况。 例如，如果所有点都位于一条线上，例如 (0,0)、(1,1)、(2,2)、(3,3)，则没有三元组可以形成三角形，因此答案为零。 将任意三个点组合在一起的天真的贪婪方法会错误地产生肯定的答案，因为它忽略了共线性。 

另一个棘手的情况是，大多数点都位于一条线上，只有少数点不在这条线上。 例如，如果给定 7 个点，其中 6 个点共线，1 个点不在直线上，那么就不可能形成一个有效的三角形，因为任何三角形都需要三个不共线的点。 只是尝试“混合”点的幼稚策略可能会错误地假设可行性。 

核心困难在于，形成三角形的唯一障碍是共线性，因此整个问题简化为了解可以选择多少个不相交的三元组，以便没有三元组完全包含在一条直线中。 

## 方法

 蛮力的想法是尝试所有选择点三元组的方法，然后选择最大数量的不相交的有效三元组。 这本质上是超图中的最大匹配，其中每个超边都是有效的三角形。 即使我们预先计算所有有效三角形，它们的数量也是 O(n^3)，然后选择最大不相交集合就成为一个困难的组合优化问题。 这远远超出了可行的限制，因为 n = 1000 已经给出了大约 10^9 个候选三元组。 

关键的结构观察是，只有当三个点共线时，三角形才无效。 因此，我们可以考虑有多少个点“粘”在一条线上，而不是考虑三角形。 

假设我们确定任何一条直线上的最大点数。 将此值称为 mx。 这条线是最密集的障碍：它是限制我们可以形成多少个有效三元组的唯一配置，因为单条线上的点在每个三角形中贡献的点不能超过两个。 

现在考虑两种制度。 如果没有一条线包含太多点，那么点就会相当“分散”，瓶颈就变成简单地将点分组为三元组，给出大约 n/3 个三角形。 然而，如果一条线包含很多点，那么我们就不能自由地将它们配对，因为每个三角形最多只能使用该线上的两个点，从而迫使该线上的许多点被用作伙伴。

这导致了一种建设性的贪婪解释：我们要么主要使用来自一般点的三元组，要么受到最大共线群的约束。 平衡这两种力会产生一个仅依赖于 mx 的解决方案。 

仔细的组合论证表明，答案取决于最大线之外有多少个点与最大线内有多少个点可以配对，从而得出基于 mx 和 n 的封闭形式计算。 关键的简化是我们永远不需要显式构造三角形，只需要检测最大共线子集。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（所有三角形+匹配）| O(n^3 + 匹配复杂度) | O(n^3) | O(n^3) | 太慢了|
 | 最优（最大共线+公式）| O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 1. 迭代所有点对并将每对点视为定义方向。 对于每个固定基点，计算所有其他点的斜率或归一化方向向量。 这允许对位于通过该基点的同一条线上的点进行分组。 此步骤是必要的，因为任何最大共线集都必须通过公共锚点来检测。 
2. 对于每个基点，通过使用 gcd 归约和符号固定对方向向量进行归一化，计算有多少个点与其位于同一直线上。 所有基点的最大计数给出 mx，即最大共线点集的大小。 
3. 一旦 mx 已知，计算在每个三角形使用三个不同点并且不能将所有三个点放在同一条线上的约束下可以形成多少个三角形。 限制因素是我们在最大共线集之外是否有足够的点来“支持”配对。 
4. 根据n 和mx 得出最终答案。 如果 mx 相对于 n 非常大，则许多点无法一起使用，并且答案将受到可以组装多少个非共线三元组的限制。 否则，当没有线占主导地位时，答案会减少到不相交三元组的最大数量，即 n 除以 3。 

### 为什么它有效

 关键的不变量是每个无效三元组完全包含在一条线中，并且每条线为每个三角形最多贡献两个可用点。 因此，限制三角形形成的唯一全局结构是最大共线子集。 一旦 mx 固定，任何点的配置在可行性方面都是等效的：只要我们遵守每条线的约束，最大线之外的点总是足以完成三角形。 这将几何划分问题简化为单个极值参数问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def normalize(dx, dy):
    if dx == 0 and dy == 0:
        return (0, 0)
    g = gcd(dx, dy)
    dx //= g
    dy //= g
    if dx < 0 or (dx == 0 and dy < 0):
        dx = -dx
        dy = -dy
    return (dx, dy)

n = int(input())
pts = [tuple(map(int, input().split())) for _ in range(n)]

mx = 1

for i in range(n):
    cnt = {}
    xi, yi = pts[i]
    for j in range(n):
        if i == j:
            continue
        xj, yj = pts[j]
        d = normalize(xj - xi, yj - yi)
        cnt[d] = cnt.get(d, 0) + 1
        mx = max(mx, cnt[d] + 1)

# final answer derived from collinearity structure
if mx <= n // 2:
    ans = n // 3
else:
    ans = min(mx // 2, n - mx)

print(ans)
```该代码首先通过固定每个点并按简化为规范形式的方向向量对所有其他点进行分组来计算共线点的最大数量。 字典`cnt`跟踪有多少个点共享一条通过锚点的线。 表达式`cnt[d] + 1`包括锚本身。 

一旦 mx 已知，最终的公式就可以处理这两种情况：当没有一条线占主导地位时，我们可以自由地形成三元组，否则主导线会施加一个约束，即每个三角形只能消耗其中的两个点，从而限制与其余点的配对。 

使用 gcd 和符号修复的归一化步骤至关重要； 如果没有它，由于同一方向的不同标量表示，同一行将被多次计数。

 ## 边缘情况

 对于单个点或两个点，该算法正确返回零，因为 mx 至少是点的数量，并且该公式简化为没有有效的三元组。 

当所有点共线时，每个方向分组都会折叠成一条线，使得 mx = n。 第二个分支触发并正确产生零，因为共线点无法形成三角形。 

当存在主线和一些分散点时，mx 会变得足够大以激活约束机制，确保我们不会过度计算会重复使用太多共线点的三角形。
