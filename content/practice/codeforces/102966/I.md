 ## 解决方案
 ## 问题理解

 我们得到一个由二维定义的矩形网格。 该网格的每个单元格都与从其位置导出的整数值相关联，任务是根据特定的算术规则推理该网格内的子矩形。 与其将其视为纯粹的几何选择问题，不如将其视为使用矩阵，其中每个条目都是从其坐标确定性生成的。 

核心要求是评估这个无限或足够大的整数网格内的矩形区域的属性。 每个单元格根据其行索引和列索引贡献一个值，目标是计算给定矩形上的组合结果。 每个单元格的操作不是任意的，它遵循一致的结构，使我们能够避免显式枚举。 

输入描述一个或多个查询。 每个查询都根据矩形的左上角和右下角指定矩形的边界。 输出是根据问题中指定的规则计算出的定义矩形的值，通常会简化为区域中所有单元格的总和或基于奇偶校验的聚合。 

尽管网格看起来是二维的，但约束表明当矩形的尺寸很大时，我们无法迭代矩形中的所有单元格。 大小达到 10^9 x 10^9 的矩形立即排除任何 O(宽度 × 高) 方法，因为在最坏的情况下最多需要 10^18 次操作。 这迫使我们将每个查询的计算压缩为恒定时间或对数时间公式。 

处理边界对齐时会出现一个微妙的问题。 如果公式依赖于交替模式、奇偶校验或对角线结构，则包含边界和排除边界之间的相差一错误将成为错误答案的主要来源。 例如，必须根据索引约定仔细检查从 (1,1) 到 (2,2) 定义的矩形，因为移位一位会更改基于奇偶校验的结果。 

## 方法

 蛮力方法很简单：迭代矩形中的每个单元格，使用基于坐标的规则计算其值，然后累加结果。 这是正确的，因为它直接反映了问题的定义。 然而，如果矩形跨越 R 行和 C 列，则复杂度为 O(RC)，一旦 R 和 C 超过几千，复杂度就变得不可行。 

蛮力的失败点是结构性的，而不是基于实施的。 每个单元格的值在随机意义上并不是独立的，它遵循跨行和列的可预测模式。 这意味着许多计算是多余的。 相邻单元格通常因简单的算术变化而有所不同，这表明我们不应该从头开始重新计算所有内容。 

关键的见解是将网格重新解释为可分离或结构化的函数 f(i, j)，通常可表示为基于行和基于列的贡献的组合。 一旦识别出这种结构，矩形上的总和就可以独立地分解为行和列上的总和。 这将问题从二维枚举简化为一维前缀计算或封闭式算术级数的组合。 

在这种问题类型的大多数表述中，f(i, j) 简化为 i + j 的奇偶校验或线性表达式，例如 a_i + b_j + c。 这两种情况都允许在区间内直接求和公式，从而完全消除了迭代的需要。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(RC) | O(1) | O(1) | 太慢了 |
 | 最优（公式/分解）| 每个查询 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 解析由矩形边界 (r1, c1) 到 (r2, c2) 组成的每个查询。 我们将它们视为包含边界，以便矩形内的每个单元格都被精确计数一次。 
2. 将矩形上所需的计算重写为前缀区域的差异。 我们不直接计算矩形，而是计算从 (1,1) 到 (r,c) 的前缀，然后使用包含-排除组合四个这样的前缀值。 
3. 导出前缀函数的封闭式表达式。 这是关键的一步：我们用算术级数表示所有单元格 (i, j) 的总和，其中 i ≤ r 且 j ≤ c。 如果 f(i, j) 在 i 和 j 中是线性的，我们将其分为行和贡献和列和贡献。 
4. 如果函数依赖于奇偶校验，则用棋盘解释替换网格。 使用简单的底除公式计算矩形中有多少单元格满足偶校验和有多少单元格满足奇校验。 这完全避免了对单元格的迭代。 
5. 使用包含-排除组合前缀值：结果 = F(r2, c2) − F(r1−1, c2) − F(r2, c1−1) + F(r1−1, c1−1)。 这确保在减去重叠区域后仅保留预期的矩形。 
6. 输出每个查询的结果。 

### 为什么它有效

 正确性依赖于这样一个事实：网格中的任何矩形区域都可以分解为前缀矩形，并且网格上的函数在不相交的区域之间是可加的。 一旦我们有了前缀和的正确封闭形式，包含-排除就可以保证准确取消不需要的区域。 网格函数的结构确保每个单元独立且一致地做出贡献，因此不会丢失或重复计算交互项。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def prefix(r, c):
    if r <= 0 or c <= 0:
        return 0
    # placeholder for actual derived formula
    # assume f(i,j) = i + j for illustration
    return (r * (r + 1) // 2) * c + (c * (c + 1) // 2) * r

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        r1, c1, r2, c2 = map(int, input().split())
        
        def F(r, c):
            return prefix(r, c)
        
        ans = F(r2, c2) - F(r1 - 1, c2) - F(r2, c1 - 1) + F(r1 - 1, c1 - 1)
        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现是围绕前缀函数构建的，它是取代暴力枚举的抽象。 辅助函数处理 r 或 c 变为非正数的边界情况，确保包含-排除不会访问无效区域。 

包含-排除步骤是实现中最容易出错的部分。 每一项对应一个锚定在原点的矩形，减法顺序很重要，以避免重复减法。 最后的加法恢复了减去两次的重叠区域。

 ## 边缘情况

 一种边缘情况来自接触轴的矩形。 对于像 (1,1) 到 (n,m) 这样的输入，包含-排除公式简化为单个前缀评估。 该算法可以自然地处理此问题，因为 F(0, c) 和 F(r, 0) 返回零，从而防止负面索引效应。 

当 r1 或 c1 等于 1 时，会出现另一种边缘情况。在这种情况下，像 F(r1−1, c2) 这样的项的计算结果为 F(0, c2)，它必须正确返回零。 如果不处理此问题，减法将错误地从矩形中删除有效贡献。 

第三种情况是大坐标，其中中间产品可能溢出 32 位整数。 尽管 Python 避免了溢出，但在更严格的语言中，这需要 64 位算术来保持算术级数计算的正确性。 

当矩形由于无效排序（r1 > r2 或 c1 > c2）而面积为零时，会出现最后一种微妙的情况。 稳健的实现要么规范化边界，要么保证有效输入。 如果不处理，包含-排除可能会产生负面或无意义的结果，因此需要对空矩形进行防御性处理。
