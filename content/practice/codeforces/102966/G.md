 ## 解决方案
 ## 问题理解

 我们给出了一个直平台，可以将其视为从 0 到 L 的一维线段。几个 Goombas 严格从该线段内的不同整数位置开始。 每个 Goomba 都有一个初始方向，要么向左移动，要么向右移动，并且它们都以每秒一个单位的相同恒定速度移动。 

该动议具有三个相互作用的规则。 Goomba 持续朝当前方向移动。 如果它到达线段的任一端点，它会立即脱落并被移除。 如果两个 Goombas 在同一位置发生碰撞，两者都会立即反转方向并继续以相同的速度移动。 

任务不是模拟单个 Goomba，而是计算最后一个 Goomba 离开该段的时刻，这意味着系统变空。 

关键的难点来自于碰撞规则。 天真的解读表明我们必须模拟成对相互作用，因为碰撞会改变方向并可能创建未来相互作用的链。 这种解释很快在计算上变得不可行。 

约束表明大约有 10⁴ Goombas 和一个非常大的 L，高达 1016。 任何尝试逐个事件模拟或成对交互跟踪的解决方案都会在密集情况下遇到 O(G²) 行为，这已经是临界点，更重要的是需要仔细的事件管理，而这对于所要求的最终数量来说是不必要的。 

多个 Goombas 在同一点相遇的对称配置会产生微妙的边缘情况。 例如，如果多个 Goombas 同时汇聚到一个位置，那么它们都会同时翻转方向。 幼稚的模拟器可能会以任意顺序处理成对碰撞，并错误地分割同时发生的事件。 

另一个极端情况是，两个 Goomba 恰好在同一时间到达终点，并与另一个 Goomba 发生碰撞。 根据事件顺序，幼稚的模拟可能会错误地让 Goomba 存活或过早消失，即使所有交互在物理上同时发生。 

## 方法

 中心观察是，在所有 Goombas 消失之前，碰撞规则实际上并不重要。 当两个 Goombas 碰撞并反转方向时，如果我们只关心位置和消失时间，这相当于它们穿过彼此。 

这种等价之所以有效，是因为所有 Goombas 都具有相同的速度和对称行为。 如果我们将 Goombas 想象成无法区分的粒子，它们相互穿过而不是弹跳，那么它们的轨迹就会变成独立的直线。 然后，每个 Goomba 只需向左或向右移动，直到到达终点。 

在这种转换下，碰撞仅交换身份，而不交换轨迹。 由于我们只关心平台何时变空，因此身份无关紧要。 

因此，我们不是模拟交互，而是将问题简化为计算每个 Goomba 在给定初始方向的情况下到达终点所需的时间。 答案是这些单独退出时间的最大值。 

暴力方法将显式模拟小时间步长的移动或处理优先级队列中的碰撞事件。 每次碰撞都需要更新位置并可能重新安排未来的事件。 当 G 高达 10⁴ 时，最坏情况下的相互作用密度可以达到 O(G²)，使得这种方法变得太慢并且不必要地复杂。 

最佳方法用单个转换替换交互逻辑：忽略碰撞并将 Goombas 视为独立的步行者。 答案变成了基于方向到最近边界的线性距离上的简单最大值。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(G² log G) | O(G² log G) | O(G) | 太慢了 |
 | 独立轨迹| O(G) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取长度 L 和 Goomba G 的数量。同时读取每个 Goomba 的位置 p 和方向 d。 
2. 对于每个 Goomba，假设它单独移动，确定其退出时间。 如果它向左移动，它的退出时间是 p 秒，因为它在 p 步后到达位置 0。 如果它向右移动，它的退出时间是 L − p 秒，因为它在该时间之后到达 L。 
3. 保持所有计算的退出时间的运行最大值。 
4.处理完所有Goombas后，输出最大值。 

步骤 2 背后的推理是关键的简化。 一旦通过等价忽略碰撞，每个 Goomba 就变成具有固定边界目标的直线运动问题。 

### 为什么它有效

 不变的是，随着时间的推移，碰撞不会改变 Goomba 位置的多重集，只会改变它们的身份。 每次碰撞都可以解释为两个 Goombas 相互穿过并继续沿直线延伸。 由于所有 Goomba 的速度和行为都相同，因此交换身份不会影响位置何时变空。 因此，任何 Goomba 最后一次到达边界的时间正是平台变空的时间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    L, G = map(int, input().split())
    ans = 0

    for _ in range(G):
        p, d = map(int, input().split())
        if d == 0:
            ans = max(ans, p)
        else:
            ans = max(ans, L - p)

    print(ans)

if __name__ == "__main__":
    solve()
```有意将实现最小化，因为转换消除了所有交互复杂性。 唯一的微妙点是正确解释方向：向左移动直接对应于距 0 的距离，而向右移动对应于距 L 的距离。 

不需要排序或事件结构，因为每个 Goomba 都独立地对最终最大值做出贡献。

 ## 边缘情况

 当所有 Goombas 朝同一方向移动时，就会出现极端情况。 例如，如果全部向右移动，系统将表现为独立粒子朝 L 方向移动，最后一个粒子在距右端点最大距离处退出。 该转换仍然有效，因为没有交互改变退出的顺序或时间。 

另一个极端情况是多个 Goombas 在同一时间点发生碰撞。 例如，位置 1 和 L−1 等对称放置相互移动会导致中点发生碰撞。 在独立轨迹模型中，它们只是相互穿过。 退出时间保持不变，最大值仍然仅由边界距离决定。 

最后，Goomba 开始时非常接近端点的情况（例如 p = 1 或 p = L−1）确认该解决方案正确处理了最小行程时间。 这些 Goombas 可能会立即交互，但边界退出时间仍然被正确捕获为 1。
