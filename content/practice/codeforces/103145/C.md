 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供一棵树，然后选择要删除的任意顶点子集。 删除这些顶点后，剩余的顶点仍然形成森林，因为我们只是从树中删除节点。 

如果在剩余图中，每个剩余顶点在剩余顶点中仍然具有至少一个邻居，则删除被称为有效。 换句话说，删除之后，导出的子图中不再剩下孤立的顶点。 

因此，我们正在计算顶点的子集，以便剩余的集合产生一个子图，其中每个顶点在该剩余的集合中的度数至少为一。 

输入由多个测试用例组成，每个测试用例给出一棵最多有100000个节点的树，所有测试用例的节点总数可以达到1000000。每个测试用例的输出是有效删除集的数量模998244353。 

这些约束立即排除了对顶点子集的任何指数枚举。 具有 100000 个顶点的树已经意味着 2^n 个可能的删除集，因此每个测试用例的任何解决方案都必须是线性的或接近线性的，通常为 O(n) 或 O(n log n)。 即使是 O(n sqrt n) 在规模上也太慢了。 

当剩余集合变得非常小时，就会出现微妙的边缘情况。 例如，如果我们只留下一个顶点，则该顶点的度数为零，因此这样的配置是无效的。 另一方面，留下零个顶点总是有效的，因为条件得到了空洞的满足。 

另一个极端情况是星形树。 如果我们尝试只保留叶子，则每个叶子都会变得孤立，因此必须排除此类配置，即使它们看起来是局部独立的。 

## 方法

 直接的蛮力方法将尝试顶点的每个子集，构造导出子图，并检查是否有任何剩余顶点的度数为零。 这将需要扫描每个子集的所有边缘，在最坏的情况下导致大约 O(n · 2^n) 次操作，这远远超出了可行的限制。 

关键的结构观察是有效性完全是局部的：剩余集中的顶点只关心其至少一个邻居是否也在剩余集中。 这表明沿边缘的依赖关系，这正是树动态编程很好地捕捉到的。 

困难在于约束不是简单的子树属性。 一个节点是否有效取决于它是否至少有一个选定的邻居，该邻居可能是有根树中的父节点或子节点之一。 这通过父状态在兄弟子树之间创建了耦合。 

处理这个问题的标准方法是建立树根并执行 DP，其中每个节点跟踪它是否在剩余集合中被选择，以及如何满足其有效性。 选择一个节点时，必须选择其父节点或至少选择一个子节点。 这引入了“至少一个子”约束，可以使用 DP 转换内的总减坏补技巧来处理。 

结果是一个线性DP，其中每个节点聚合两种类型的子树贡献：未被选择的配置，以及被选择并满足邻接要求的配置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了|
 | 具有选择约束的树DP O(n) | O(n) | 已接受 |

 ## 算法演练

 为了方便起见，我们将树建立在任意节点处，即节点 1。对于每个节点，我们计算 DP 值，该值取决于其父节点是否在最终剩余集合中被选择。 

我们定义 DP 状态的方式是区分节点是否包含在剩余集合中以及在给定其子树和父上下文的情况下它是否“内部有效”。 

### 步骤

1. 以节点 1 为树根并固定父子方向。 
2. 对于每个节点 u 和父状态 p，将 dp[u][p][0] 定义为 u 不包含在剩余集合中的路数。 这始终有效，因为删除的节点对其自身没有任何约束。 
3.对于每个节点u和父状态p，将dp[u][p][1]定义为u包含在剩余集合中并且满足其至少有一个邻居也包含在剩余集合中的条件的路数。 
4. 计算 dp[u][p][0] 时，观察到 u 没有被选择，因此每个子 v 都与父状态 p 独立处理。 有效配置的总数是每个子项的有效配置总数与子项的乘积。 
5. 在计算 dp[u][p][1] 时，我们考虑处于选择 u 状态的子节点，因此对于每个子节点 v，父状态变为 1。 
6. 让每个孩子 v：

 T_v 是当子树 v 的父节点被选择时，子树 v 中有效配置的总数，S_v 是 v 未被选择的配置的数量。 
7. 如果 p = 1，则 u 已经满足，因为它的父级已被选择。 因此 dp[u][1][1] 只是 T_v 对所有子节点的乘积。 
8. 如果 p = 0，则必须至少选择一个孩子来满足 u 的要求。 首先计算子项的总无约束乘积，然后减去未选择所有子项的无效情况。 得出 dp[u][0][1] = Product(T_v) − Product(S_v)。 
9. 计算所有节点的 DP 后，答案是 dp[1][0][0] + dp[1][0][1]，因为根没有父节点。 

### 为什么它有效

 不变量是 dp[u][p] 正确计算以 u 为根的子树的所有有效配置，假设当且仅当 p = 1 时选择 u 的父级，并且仅使用本地父子交互强制执行所有约束。 唯一的全局约束（即每个选定的顶点必须有一个选定的邻居）是通过确保每个选定的节点要么由其父节点要么由至少一个子节点支持来完全强制执行的。 由于在这种父子关系中每条边都被考虑一次，因此无效的配置不会被漏掉或被重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

MOD = 998244353

def solve():
    n = int(input())
    g = [[] for _ in range(n + 1)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        g[u].append(v)
        g[v].append(u)

    if n == 1:
        print(1)
        return

    parent = [0] * (n + 1)
    order = []
    stack = [1]
    parent[1] = -1

    while stack:
        u = stack.pop()
        order.append(u)
        for v in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            stack.append(v)

    dp0 = [[1, 0] for _ in range(n + 1)]
    dp1 = [[1, 0] for _ in range(n + 1)]

    for u in reversed(order):
        children = [v for v in g[u] if v != parent[u]]

        # p = 0
        prod_total = 1
        prod_empty = 1
        for v in children:
            prod_total = prod_total * (dp0[v][0] + dp0[v][1]) % MOD
            prod_empty = prod_empty * dp0[v][0] % MOD
        dp0[u][0] = prod_total
        dp0[u][1] = (prod_total - prod_empty) % MOD

        # p = 1
        prod_total = 1
        prod_empty = 1
        for v in children:
            prod_total = prod_total * (dp1[v][0] + dp1[v][1]) % MOD
            prod_empty = prod_empty * dp1[v][0] % MOD
        dp1[u][0] = prod_total
        dp1[u][1] = (prod_total - prod_empty) % MOD

    ans = (dp0[1][0] + dp0[1][1]) % MOD
    print(ans)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```实现首先对树进行生根并构建父数组以避免递归开销，因为在测试中输入大小可以达到一百万个节点。 

我们存储两个 DP 表。 dp0[u]对应于u的父级未被选择的情况，而dp1[u]对应于u的父级被选择的情况。 每个条目都有两个值：索引 0 表示未选择的 u，索引 1 表示选择的 u 并强制执行有效性。 

关键的转变是产品向儿童的转变。 对于每个节点，我们独立地聚合子贡献，因为一旦父状态固定，子树就会不相交。 当未选择父项时，减法步骤会强制执行“至少选择一个子项”条件。 

每次乘法都会应用模算术，以将值限制在 998244353 以下。 

## 工作示例

 ### 示例 1

 考虑一条简单的路径：1 - 2 - 3。 

我们以 1 为根。 

对于节点 3（叶子），dp 值很简单：如果选择了父节点，则可以自由选择或不选择它，但选择始终有效，因为它有潜在的父节点支持。 如果未选择父项，则选择 3 将需要一个不存在的子项，因此该情况​​无效。 

对于节点 2，它聚合节点 3 的结果，并根据是否选择 2 以及是否有父级或子级的支持来组合它们。 

| 节点| p（选择父级）| dp[u][p][0] | dp[u][p][0] | dp[u][p][1] | dp[u][p][1] |
 | ---| ---| ---| ---|
 | 3 | 0 | 1 | 0 |
 | 3 | 1 | 1 | 1 |

 对于节点 2，该组合产生有效的配置，其中要么没有以冲突的方式选择任何节点，要么存在至少一个邻接。 

这证实了孤立的选择（例如仅选取节点 1 或仅选取节点 3）被排除。 

### 示例 2

 考虑一颗星，其中心 1 连接到 2、3、4。 

如果我们尝试仅选择叶子 2、3、4，则每个叶子都会变得孤立，因此必须将其排除。 DP 强制执行此操作，因为选择叶子而不选择中心会导致“坏”子集被减去。 

| 配置| 有效期 |
 | ---| ---|
 | {2,3,4} | 无效|
 | {1,2} | 有效 |
 | {1,2,3,4} | 有效 |
 | {} | 有效 |

 DP 正确地保留每个选定顶点至少有一个选定邻居的配置。

 ## 边缘情况

 单节点树是逻辑上最干净的压力点。 唯一有效的配置是删除节点，留下一个空图，这完全满足条件。 DP 正确返回 1，因为空乘积贡献 1，并且“选定的根”情况变得无效。 

在星形树中，任何选择没有中心的叶子的配置都会破坏邻接规则。 当在没有家长支持的情况下选择中心时，DP 显式地减去“所有儿童未选择”的情况，确保这些无效配置被删除一次。 

在双节点树中，空剩余集和完整剩余集都是有效的，并且任何 DP 公式必须恰好生成 2。转换正确地允许这两种情况，而不会过度计数或丢失交叉依赖性。
