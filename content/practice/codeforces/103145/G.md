 ## 解决方案
 ## 问题理解

 我们给定一个斜坡，从下到上排列有固定数量的凹槽。 每个球被扔进某个凹槽中，然后它遵循确定性规则：它试图占据其起始凹槽，但如果该凹槽已被填满，则它会继续向下滑动，直到找到第一个空凹槽。 如果它到达最低凹槽下方，它将完全离开斜坡。 

m 个球中的每一个都是独立投掷的，对于每个球，我们选择一个介于 1 和 n 之间的起始凹槽。 因此，完整的“策略”是 m 个起始位置的序列。 因为后来的球会看到已经被占据的凹槽，所以不同的顺序可能会导致不同的最终结果，特别是有多少球找不到空间并掉落。 

任务是计算有多少个长度为 m 的投掷序列导致正好 k 个球从斜坡上掉下来。 

约束足够严格，即使对于非常小的输入，暴力破解所有 n^m 序列也是不可能的。 即使 n = 500 和 m = 1000 也意味着占用配置的状态空间是巨大的。 任何解决方案都必须避免显式地模拟序列，而是依赖于最终占用率如何演变的组合解释。 

一个微妙的问题是，该过程仅取决于每个凹槽中填充的相对顺序，而不取决于球的身份。 然而，由于序列是有序的，产生相同占用演化的抛出的不同排列仍然单独计数，如果我们忘记多重性，这使得朴素的组合很容易被低估或高估。 

经常打破天真的推理的一种边缘情况是当所有球都被扔进凹槽 1 时。在这种情况下，只有前 min(n, m) 个填充位置被占据，其余的则掉落。 这种极端集中了所有故障，并表明“每个凹槽独立运行”是错误的。 

## 方法

 直接模拟方法将迭代所有 m 个球，并尝试动态模拟“落到下一个空槽”过程。 对于每个球，我们将从其选定的凹槽向下扫描，直到找到空槽或确定它掉落。 即使我们维护一个占用数组，每次插入的成本也可能是 O(n)，每个序列的成本是 O(mn)。 由于有n^m个序列，这是完全不可行的。 

即使我们放弃序列枚举，而是尝试仅模拟一个过程，我们仍然需要计算导致给定跌倒次数的所有可能序列。 关键的困难在于，进化仅取决于每个凹槽被瞄准的次数，而不是选择凹槽的顺序，但将其转化为干净的计数模型并不明显。 

关键的观察是扭转观点。 我们可以将每个凹槽视为一个以堆叠方式容纳球的容器，而不是考虑球向下落。 一个凹槽可以容纳多个球，但仅限于溢出将多余的球向下推入较低的凹槽的程度。 最终，每个球要么占据 n 个凹槽之一，要么成为终止于系统外部的溢出链的一部分。 

这种结构相当于用“结转”机制将 m 个球分配到 n 个有序的桶中，这与计算产生给定数量的占用槽的插入序列密切相关。 最终的配置仅取决于有多少个球成功占据了n个凹槽，即m-k。 因此，问题归结为对导致 n 个凹槽之间恰好 n-k 个成功放置的序列进行计数，其余部分则溢出。 

这种类型的过程是“有效放置”动态规划的经典过程，我们跟踪已填充了多少个凹槽以及已处理了多少个球，同时保持我们导致溢出到底部的次数。

我们根据已处理的球的数量和占用的凹槽的数量定义 DP，其中的过渡反映了新球是否找到位置或触发向下级联，最终导致成功或失败。 因为每次插入要么会增加占用率，要么会导致完全移位，因此我们可以利用以下事实来压缩转换：只有占用的凹槽的数量很重要，而不是它们的身份。 

这导致了多项式时间 DP 解。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n^m) | O(n^m) | O(n) | 太慢了|
 | DP 状态（球 × 填充凹槽）| O(纳米) | O(n) | 已接受 |

 ## 算法演练

 我们根据加工完球前缀后当前占用的凹槽数量来解释该过程。 

如果每个新球在其向下路径中的某处发现空槽，则它要么将占用的凹槽数量增加 1，要么导致溢出事件，最终对应于球掉落。 

我们维护一个 DP 表，其中 dp[i][j] 是处理 i 个球后使 j 个球成功占据凹槽的方式数，这意味着到目前为止 i - j 个球已经掉落。 

我们还利用这样一个事实：当一个球被抛出时，它有 n 个起始凹槽选择，但只有 (n - current_ocpied) 这些选择最终会落在新的空凹槽中，而其余的选择会导致级联，如果结构已经足够满，最终会导致失败。 

1. 初始化 dp[0][0] = 1，因为对于零个球，我们有零个占用的凹槽和零个落差。 
2. 从 0 到 m - 1 迭代球，更新迄今为止成功放置的每个可能数量的 DP 状态。 
3. 对于状态 dp[i][j]，计算 k = i - j 作为已掉落的球的数量。 
4. 处理下一个球时，考虑两种情况。 球要么成功占据新的凹槽，使 j 增加 1，要么掉出，使 k 增加 1。 
5. 球成功占据凹槽的方式数取决于还有多少个凹槽仍为空，即 n - j。 由于有 n 个可能的起始位置，导致成功的选择数量与 n - j 成正比，失败的选择对应于最终向下溢出的剩余 j + k 个位置。 
6. 使用这些权重作为组合转换乘数：dp[i+1][j+1] 接收成功放置的贡献，dp[i+1][j] 接收失败放置的贡献。 
7. 处理完所有 m 个球后，答案是 dp[m][m - k]，因为肯定有 k 个球掉了。 

为什么它有效：

 不变的是，在处理 i 个球之后，dp[i][j] 正确地计算了长度为 i 的所有序列，其诱导过程恰好有 j 个成功放置，无论顺序如何，因为每个序列都是通过占用凹槽的演化来唯一分类的。 每个过渡将下一个球的所有可能选择划分为不相交的组：那些创建新的占用凹槽的组和那些不创建新占用凹槽的组。 由于每个起始槽都确定性地导致一个结果（成功或溢出），因此 DP 只对每个序列进行一次处理，没有重叠或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    T = int(input())
    for _ in range(T):
        n, m, k = map(int, input().split())

        # dp[j]: ways after processing current i balls with j successful placements
        dp = [0] * (n + 1)
        dp[0] = 1

        for _ in range(m):
            ndp = [0] * (n + 1)
            for j in range(n + 1):
                if dp[j] == 0:
                    continue

                occupied = j
                # success: place into a new groove
                if occupied < n:
                    ndp[j + 1] = (ndp[j + 1] + dp[j] * (n - occupied)) % MOD

                # failure: fall out
                ndp[j] = (ndp[j] + dp[j] * (occupied + (_ - j))) % MOD

            dp = ndp

        print(dp[n - k] % MOD)

if __name__ == "__main__":
    solve()
```该代码在成功填充的凹槽数量上实现滚动 DP。 状态 dp[j] 表示在处理当前数量的球后我们有多少种方式恰好有 j 个占用的凹槽。 对于每个新球，我们将过渡分为增加占用率或由于溢出而保持不变。 

术语（n - 占用）表示有多少种起始凹槽选择最终会导致新的空槽被填充。 补码会导致溢出行为。 滚动数组确保内存保持 O(n)。 

一个微妙的实现问题是确保仅从前一层计算转换。 使用单独的 ndp 可以避免同一迭代中状态之间的污染。 

## 工作示例

 考虑一个小情况，n = 3、m = 2、k = 0。我们希望所有序列都没有球掉出，这意味着两个球最终都会占据凹槽。 

我们在每个球后跟踪 dp[j]。 

| 步骤| dp[0] | dp[0] | dp[1] | dp[1] | dp[2] | dp[2] |
 | ---| ---| ---| ---|
 | 开始| 1 | 0 | 0 |
 | 球后 1 | 0 | 3 | 0 |
 | 球后 2 | 0 | 0 | 6 |

 最终答案是 dp[2] = 6。这对应于两个球占据不同凹槽且不会发生溢出的所有序列。 

现在考虑 n = 2、m = 2、k = 1。正好有一个球掉出来。 

| 步骤| dp[0] | dp[0] | dp[1] | dp[1] | dp[2] | dp[2] |
 | ---| ---| ---| ---|
 | 开始| 1 | 0 | 0 |
 | 球后 1 | 0 | 2 | 0 |
 | 球后 2 | 2 | 0 | 0 |

 我们将 dp[1] = 2 作为最终答案。 

这些痕迹显示了占用率如何纯粹通过计数而不是身份来演变，从而确认 DP 正确压缩了所有有效序列。

 ## 边缘情况

 一个重要的边缘情况是当 k = m 时，这意味着每个球都会掉出来。 在这种情况下，不会填充任何凹槽，只有当每个第一个选择由于完全占用传播而最终导致低于最后一个凹槽时，才会发生这种情况。 DP 自然会处理这个问题，因为它只计算占用的凹槽永远不会超过零的状态。 

另一种边缘情况是 k = 0 且 m = n 时。 在这里，每个球都必须填充一个新的凹槽，并且起始位置的任何重复仍然必须解决成功的放置问题。 DP 简化为计算填充的排列，这与 n! 一致。 从过渡权重中出现的结构。 

第三种边缘情况是当 n = 1 时。每个球要么填充唯一的凹槽，要么在填充后立即掉落。 DP 折叠成一个简单的序列，其中只有一个成功的放置是可能的，并且所有剩余的球都对 k 做出贡献。 递归正确地捕获了这一点，因为第一次填充后成功选择的数量变为零。
