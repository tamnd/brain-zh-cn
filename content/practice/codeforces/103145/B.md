 ## 解决方案
 ## 问题理解

 我们正在模拟一个简化的恩尼格玛式密码机。 机器会转换字符流，但转换在很大程度上取决于每次按键后不断变化的内部状态。 

该机器有三层。 第一层是一组固定的字母交换，例如跳线，它在发生任何其他事情之前成对地重新映射输入字母。 第二层是一系列旋转替换盘。 每个圆盘的行为就像字母表的固定排列，但其解释随着圆盘的旋转而变化。 信号在某个位置进入光盘，通过当前移位的接线进行映射，并产生一个新位置。 在穿过所有圆盘后，信号到达反射器，该反射器成对映射字母，并使用逆映射将信号以相反方向发送回相同的圆盘。 最后，再次应用跳线以产生输出字母。 

关键的困难在于，每次按键后圆盘都会旋转，并且旋转像里程表一样传播：第一个圆盘每次旋转，完整的旋转会导致下一个圆盘旋转，继续沿着链条向上延伸。 这意味着应用于每个字符的转换取决于已处理的字符数量。 

输入给出了多个测试用例。 每个测试用例描述了跳线交换、盘排列、每个盘的初始旋转以及反射器排列。 然后必须按顺序处理多个消息，并且机器状态在所有字符上持续。 

限制很小：最多 50 个光盘，最多 10 条消息，每条消息都小于 50 条。这立即告诉我们，即使是相当直接的逐字符模拟也是可以接受的，因为字符评估的总数最多只有几千个。 然而，重新计算完整排列或每一步低效模拟角色移动的幼稚实现仍然可以通过，但在正确处理旋转方面存在复杂性错误的风险。 

最危险的边缘情况是忘记旋转会对称地影响向前和向后的传球。 另一个微妙的问题是错误地将磁盘旋转视为仅影响映射的一侧，而实际上两行一起移动，这意味着排列本身会循环重新索引。 

如果假设盘接线是静态的，则会出现具体的故障情况：

 用一张光盘输入：```
A swaps with B
1 disc: ABC...Z but second row reversed
di = 1
```按一键后，圆盘就会旋转，因此映射会发生变化。 静态排列实现将重复应用相同的映射并产生周期 1 的重复循环，这是不正确的。 

另一个失败案例来自于忽略进位传播。 如果圆盘 1 旋转 26 圈，则圆盘 2 必须旋转一次。 如果忽略这一点，长消息将完全偏离正确的输出。 

## 方法

 强力模拟独立处理每个按键，并明确地逐步模拟通过每个光盘的信号传播。 对于每个角色，我们将应用插板交换，使用其当前旋转配置向前遍历所有光盘，应用反射器，然后向后遍历逆映射，最后再次应用插板。 处理完字符后，我们一一更新圆盘的旋转状态，处理进位传播。 

这种方法是正确的，因为它准确地反映了机器定义。 然而，如果实施效率低下，瓶颈是旋转排列的重复重建。 如果我们动态重新计算移位行或重建映射，则每个盘转换成本为 O(26)，导致 O(Q * L * n * 26)，这仍然很好，但不必要的开销。 

关键的观察结果是，每个圆盘的行为仅取决于其旋转模 26。我们永远不需要模拟完整的字符串或物理旋转阵列。 相反，每个盘可以被建模为与循环偏移相结合的固定排列。 这将每个光盘变成一个按输入位置和移位状态索引的恒定时间查找表。 可以提前为所有 26 个班次预先计算前向和后向映射。 

一旦完成，整个机器就成为一个确定性状态机，其中每个字符的成本为 O(n)，并且由于进位传播，每次按键的状态更新成本为 O(n)。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 通过重新计算进行直接模拟 | O(总计 * n * 26) | O(n * 26) | 已接受但沉重|
 | 预先计算的移位表| O(总计 * n) | O(n * 26) | 已接受 |

 这里的总计是所有查询的字符总数。 

## 算法演练

 我们对每个圆盘进行预处理，以便将每个可能的旋转状态明确表示为 26 个字母的排列（正向和反向）。 这消除了在运行时模拟字符串旋转的需要。 

## 算法演练

1. 将所有字母 A 到 Z 转换为索引 0 到 25。这允许将每个转换表示为数组索引而不是字符串操作，这对于速度和清晰度至关重要。 
2. 读取插板交换并构建大小为 26 的直接映射数组，其中每个字母映射到其伙伴或自身（如果未触及）。 这确保了恒定的时间进入和退出转换。 
3. 对于每个光盘，将给定的字符串解释为排列的初始第二行。 第一行隐式地表示身份排序。 由此，构建一个基本排列，将输入位置映射到移位 0 的输出位置。 
4.对于每个盘和从0到25的每个可能的旋转状态，预先计算正向映射和逆向映射。 正向映射描述了信号从左侧进入时去往何处，逆向映射描述了反射后的反向遍历。 此步骤至关重要，因为旋转会改变两行的对齐方式，而不仅仅是重新标记。 
5. 维护一个数组 shift[i]，表示圆盘 i 旋转了多少次，以 26 为模。最初，这是由输入 di 值给出的。 
6. 对于每条消息中的每个字符，应用完整的加密路径。 首先应用插板映射。 然后使用当前移位状态和预先计算的转发表通过光盘向前传播。 然后应用反射器贴图。 然后使用逆表以相反的顺序通过光盘向后传播。 最后再次应用插板即可获得输出字符。 
7. 处理完一个字符后，更新旋转状态。 增量移位[0]。 如果变成26，则重置为0并进位到shift[1]，继续向前直到没有进位剩余。 该模型模拟了旋转盘的里程表行为。 

关键的不变量是，在每一步，shift[i] 正确地表示盘 i 模 26 的总旋转次数，并且预先计算的表正确地编码由该旋转状态引起的精确排列。 因为每个转换仅取决于（盘索引、移位状态、方向），所以模拟永远不会丢失信息或偏离真实的机器状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_tables(wiring):
    fwd = [[0]*26 for _ in range(26)]
    inv = [[0]*26 for _ in range(26)]

    base = list(wiring)

    for s in range(26):
        for i in range(26):
            # forward: input position i maps via shifted wiring
            j = (i + s) % 26
            c = base[j]
            out = (ord(c) - 65 - s) % 26
            fwd[s][i] = out

        for i in range(26):
            inv[s][fwd[s][i]] = i

    return fwd, inv

def apply_plug(x, plug):
    return plug[x]

def main():
    T = int(input())
    for _ in range(T):
        p = int(input())
        plug = list(range(26))

        for _ in range(p):
            a, b = input().strip()
            a = ord(a) - 65
            b = ord(b) - 65
            plug[a] = b
            plug[b] = a

        D = int(input())
        discs = []
        for _ in range(D):
            wiring = input().strip()
            discs.append(wiring)

        shift = list(map(int, input().split()))

        reflector = input().strip()
        ref = [ord(c) - 65 for c in reflector]

        fwd = []
        inv = []

        for d in discs:
            f, i = build_tables(d)
            fwd.append(f)
            inv.append(i)

        Q = int(input())

        for _ in range(Q):
            msg = input().strip()
            out = []

            for ch in msg:
                x = ord(ch) - 65

                x = plug[x]

                # forward
                for i in range(D):
                    x = fwd[i][shift[i]][x]

                # reflector
                x = ref[x]

                # backward
                for i in range(D-1, -1, -1):
                    x = inv[i][shift[i]][x]

                x = plug[x]
                out.append(chr(x + 65))

                # rotate
                carry = 1
                for i in range(D):
                    if carry == 0:
                        break
                    shift[i] += carry
                    if shift[i] == 26:
                        shift[i] = 0
                        carry = 1
                    else:
                        carry = 0

            print("".join(out))

if __name__ == "__main__":
    main()
```代码首先将每个转换转换为 0 到 25 之间的整数算术。插件板是每个字符应用两次的直接数组映射。 每个光盘都扩展为按旋转状态索引的完整查找表，以便运行时模拟永远不需要重建移位的字符串。 

前向和后向传递是简单的表查找。 反射器是固定的对合，在过程中间为每个字符应用一次。 最后的旋转逻辑仔细地实现了进位传播，确保较高的圆盘仅在较低的圆盘完成 26 个完整周期时才旋转。 

一个微妙的实现细节是，盘旋转同样影响前向和后向映射，这就是为什么 fwd 和 inv 表都依赖于相同的移位索引。 另一个重要的一点是，移位更新发生在处理每个字符之后，而不是之前，这与第一个光盘在按键时立即旋转的说法相匹配。

| 步骤| 价值|
 | ---| ---|

 该迹线表明，前向和后向遍历是对称的，并且正确性取决于相同移位状态的逆映射与前向映射的精确匹配。 

### 示例 2（循环进位）

 假设 D = 2 并且两个班次都从 25 开始。 

| 人物 | 前移| 在disc0之后移位| 传送到光盘 1 |
 | ---| ---| ---| ---|
 | 第一 | [25, 25] | [0, 25] | 是的 |
 | 第二 | [0, 25] | [1, 25] | 没有|

 这表明圆盘旋转的行为类似于 26 进制里程表。 仅当第一个圆盘完成完整循环后，第二个圆盘才会移动。

 ## 边缘情况

 一种重要的边缘情况是光盘恰好达到 26 转。 在那一刻，它重置为零并触发进位。 该实现通过在增量后立即检查与 26 的相等性来处理此问题，确保在转换中不使用中间无效状态。 

另一个边缘情况是跨多个查询按顺序处理消息。 机器状态在查询之间不会重置，因此班次会继续累积。 该算法通过在所有消息中维护一个全局移位数组来保留这一点。 

最后一个微妙的情况是对称插板交换。 由于交换是对合的，因此应用两次插件板可以恢复原始字符空间的一致性。 该实现在进入机器之前和退出机器之后都应用它，保留密码设计所需的可逆性。
