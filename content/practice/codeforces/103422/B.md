 ## 解决方案
 ## 问题理解

 我们得到了一组值，其中每个值代表分配给队列中的一个人的“工作难度”。 我们可以选择该数组的任何连续段，然后对该段应用非常具体的转换：我们从右向左扫描，在每个位置，我们用当前值和紧邻其右侧的值之间的最小值替换当前值。 这具有向左传播后缀最小值的效果，因此在操作之后，所选段中的每个位置都成为该段内其后缀的最小值。 

任务是选择一个段，使得在对其应用此变换后，所得段的总和尽可能大。 我们必须输出段边界以及变换后的段的结果总和。 

数组大小最大可达 100,000，值最大可达 10^9。 这立即排除了尝试所有 O(n^2) 段并在每个 O(n) 中简单地模拟转换的任何方法，因为在最坏的情况下将达到 O(n^3)。 在典型的一秒约束下，即使 O(n^2) 总工作量也太大，因此解决方案必须将搜索减少到接近线性或线性算数时间。 

一个微妙的边缘情况来自这样一个事实：转换以非常定向的方式破坏了增加的结构。 例如，如果我们采用严格递增的段（如 [1, 3, 5, 7]），则在操作后它会变为 [1, 3, 5, 7] 不变，但如果我们采用递减的段（如 [7, 5, 3, 1]），它会严重崩溃为重复的最小值。 “更长的段总是更好”的天真期望失败了，因为延长段可以减少传播的最小值并大大减少早期元素的贡献。 

另一个边缘情况是出现相同的值。 例如，在 [5, 1, 1, 10] 中，选择包含大 10 的段仍然会导致较早的元素折叠为 1，因此如果高值出现在小值之后，则包含高值并不总是有益的。 

## 方法

 暴力策略是考虑每对端点 l 和 r，模拟 v[l..r] 上的 gorbasort 操作，并计算其总和。 模拟一个片段需要 O(r - l) 时间，因为我们从右向左扫描一次。 对所有段求和，在最坏的情况下这会导致 O(n^3) 行为，因为存在 O(n^2) 段并且每个段都花费线性时间。 对于 n 达到 10^5 来说这太慢了。 

关键的结构观察是该操作将变换后的每个位置定义为所选段内的后缀最小值。 一旦我们确定了右端点 r，当我们向左延伸线段时，我们就可以有效地构建元素的贡献。 如果每个元素小于该段中其右侧的所有元素，则它要么贡献自己的值，要么被“限制”为迄今为止看到的最小值。 

这使得转换后的总和的行为类似于左端点上的分段线性函数。 当我们向左移动时，只有当我们遇到新的最小值时，段的值才会改变，因为只有这样才会改变未来后缀最小值的传播。 在这些点之间，块的贡献是可预测的，并且可以使用当前最小值和覆盖的距离在恒定时间内计算。 

这使我们能够固定右端点 r 并向左扩展，同时保持当前的最小值和累积贡献。 每当我们看到新的较小值时，我们都会更新最小值并相应地调整贡献。 由于对于固定 r，每个元素最多成为新的最小值一次，因此所有 r 的总工作是线性的。

 ## 算法演练

我们迭代所有可能的右端点 r。 对于每个 r，我们将指针 l 从 r 向左展开，从而保持段 [l, r] 的最佳变换和。 

1. 从固定 r 开始，将当前最小值初始化为 v[r]，将当前总和初始化为 v[r]。 段 [r, r] 不会被操作改变，因此这个基本情况是准确的。 
2. 一次向左移动一步。 当我们包含 v[l] 时，我们将其与当前最小值进行比较。 如果 v[l] 大于或等于当前最小值，则转换后 v[l] 恰好变为该最小值，因为其右侧的所有内容都已强制执行更小的后缀最小值。 这意味着它的贡献只是当前的最小值。 
3. 如果v[l]小于当前最小值，则v[l]成为新的最小值。 从这一点开始，从 l 开始的所有位置都将受到这个新值的限制​​，直到出现另一个更小的元素。 我们更新最小值并相应调整运行总和。 
4. 在每一步中，我们将 [l, r] 的计算总和与迄今为止的最佳答案进行比较，并在提高最大值时存储线段端点。 

关键的效率提升是，每次向左移动时，我们只执行恒定的工作，因为我们不会重新计算整个转换后的段。 相反，我们依赖后缀最小值结构将更改折叠为简单的状态更新这一事实。 

经过这些步骤，我们输出最好的 l、r 以及相应的计算和。 

### 为什么它有效

 对于任何固定段，位置 i 处的最终值是 v[i]、v[i+1]、...、v[r] 中的最小值。 这意味着变换后的片段完全由后缀最小值结构决定，当我们向左延伸片段时，后缀最小值结构单调演化。 该算法以增量方式准确地维护此结构，因此每次更新都反映了真实的变换后的段和，而无需重新计算。 由于当新的最小值出现时，后缀最小值配置文件中的每个变化都会立即被捕获，因此不会错误地表示段状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    v = list(map(int, input().split()))

    best_sum = -10**30
    best_l = best_r = 0

    for r in range(n):
        cur_min = v[r]
        cur_sum = v[r]

        if cur_sum > best_sum:
            best_sum = cur_sum
            best_l, best_r = r, r

        for l in range(r - 1, -1, -1):
            if v[l] < cur_min:
                cur_min = v[l]

            cur_sum += cur_min

            if cur_sum > best_sum:
                best_sum = cur_sum
                best_l, best_r = l, r

    print(best_l + 1, best_r + 1, best_sum)

if __name__ == "__main__":
    solve()
```该代码实现右端点循环，然后向左扩展，同时保持运行最小值和转换后的总和。 关键的实现细节是`cur_sum += cur_min`是有效的，因为每个新添加的元素在应用操作后准确地贡献了当前后缀最小值。 

索引在内部以从零开始的形式存储，并且仅在输出时转换为从一开始的形式，与问题的索引约定相匹配。

 ## 边缘情况

 大小为 1 的最小输入直接展示了基态。 该算法在该单个索引处初始化段，因此不会发生扩展并且结果立即正确。 

严格递增的数组（例如 [1, 2, 3, 4]）可确保向左扩展时不会出现新的最小值，因此每个元素都贡献其全部值。 该算法保持 cur_min 不变，并在该段上累加一个简单的算术和。 

严格递减数组（例如 [5, 4, 3, 2, 1]）会频繁更新到当前最小值。 每个新元素都成为主导后缀最小值，并且跟踪显示了早期的大值是如何被重复覆盖的。 该实现正确地反映了这一点，因为每当遇到较小的值时，每个步骤都会更新 cur_min，从而确保保留后缀最小值结构。
