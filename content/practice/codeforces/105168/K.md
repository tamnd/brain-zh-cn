 ## 解决方案
 ## 问题理解

 我们得到了二维平面上的一组点。 任务是准确放置$k$垂直线和$k$水平线，这样没有线穿过任何点，这些线将平面划分为$(k+1)\times(k+1)$矩形区域。 每个区域必须包含完全相同数量的点。 

每个点恰好属于一个区域，该区域由其左侧有多少条垂直线和其下方有多少条水平线确定。 如此有效地，垂直线将点划分为$k+1$垂直条带和水平线独立地划分为$k+1$水平条。 要求是选择一个垂直条带和一个水平条带形成的每个相交单元包含相同数量的点。 

限制非常大：最多$2\cdot 10^5$所有测试中每个测试的分数，以及$k$可以大到$10^9$。 这立即排除了任何尝试显式构造或模拟线路布局的方法。 任何有效的解决方案必须仅依赖于点集的组合结构，而不依赖于几何或坐标搜索。 

一个关键的观察结果是，除了排序之外，确切的坐标是无关紧要的。 只有沿 x 和 y 的相对顺序很重要，因为垂直线可以放置在已排序的 x 值之间的任何位置，对于 y 也是如此。 因此，问题简化为推理 x 和 y 等级的点排列。 

一个微妙的边缘情况是点重叠。 多个点可以共享相同的坐标，并且一条线不得经过任何点，但如果需要，可以将其放置在相等的坐标之间。 任何假设坐标严格排序而不处理重复的解决方案都可能在诸如多个相同点（其中分割必须尊重多重性）的情况下失败。 

另一个边缘情况是$k=0$，其中未绘制线条。 那么整个平面就是一个区域，因此所有点都必然位于一个区域中，这意味着无论分布如何，该条件始终成立。 这种情况经常会破坏假设至少一次拆分的实现。 

## 方法

 如果我们尝试天真地思考，我们可以想象把$k$垂直线和$k$水平线并尝试将点分配给$(k+1)^2$单元格，以便每个单元格获得相同数量的点。 由于坐标是点之间的实值选择，因此天真的想法是按 x 对点进行排序，并尝试所有方法将其拆分为$k+1$连续组，并类似地按 y，然后检查一致性。 

然而，即使忽略线路布局的连续性，可供选择的方式数量$k$之间切位置$n$点是组合的，尝试所有分区是指数的。 即使是单一维度也已经具有$\binom{n-1}{k}$方式，这是不可能的$n=2\cdot 10^5$。 

关键的结构见解是垂直和水平分区必须独立且统一地运行。 每个垂直条带必须包含完全相同数量的点，因为每个条带进一步细分为$k+1$水平线相等的单元格。 如果任何垂直条带的计数不同，则其行的单元格不能全部与其他行匹配。 

这强制了一个强约束：总分$n$必须能整除$(k+1)^2$，因为每个$(k+1)^2$细胞大小相等。 此外，每个垂直条带包含$n/(k+1)$点，并且每个水平条还包含$n/(k+1)$点。 

Now the problem becomes one-dimensional in each axis. 我们需要检查是否可以将排序后的 x 坐标拆分为$k+1$每个大小的连续组$n/(k+1)$，并独立地对 y 坐标执行相同的操作。 如果两者都可行，我们可以在组之间放置线条。 

因此，问题简化为检查当按 x 排序时，每个大小的块是否$n/(k+1)$形成一个有效的条带，对于 y 也是如此。 

微妙之处在于，这与值无关，而仅与按订单统计数据分组有关。 除了排序之外不需要任何几何图形。

 | 暴力分割| 指数| O(n) | 太慢了 |
 | 排序+块验证|$O(n \log n)$| O(n) | 已接受 |

 ## 算法演练

 我们定义$m = k+1$。 如果$n \% (m \cdot m) \neq 0$，我们立即得出不可能的结论，因为均匀分布$(k+1)^2$细胞是不可能的。 

我们按如下方式进行。 

1. 计算$cell = n / (m \cdot m)$。 这是每个区域所需的点数。 
2. 计算$strip = n / m$。 这是每个垂直条带和每个水平条带的点数。 这是因为每个垂直条带恰好包含$m$细胞垂直堆叠。 
3. 按 x 坐标对所有点进行排序。 We scan the sorted list in chunks of size$strip$。 每个块代表一个候选垂直条。 我们验证在每个块中，我们可以在概念上准确地分配$cell$稍后每个水平段的点数。 在这个阶段，我们只保证分组的可行性。 
4. 独立地按 y 坐标对所有点进行排序，并对水平条带执行相同的检查。 
5. 如果 x 方向和 y 方向都可以划分为相同大小的块，则返回 Yes，否则返回 No。 

实现细节是我们不需要显式分配单元格。 该结构保证如果两个轴都允许均匀的块划分，它们的笛卡尔积会自动形成相同大小的单元。 

### 为什么它有效

 由垂直线引起的分区仅取决于 x 的排序，因此每个垂直条都是按 x 排序顺序的连续段。 类似地，水平条对应于按 y 排序顺序的连续段。 由于每个单元格恰好是一个垂直条带和一个水平条带的交集，因此每个单元格的大小由落入每对索引的点数量决定。 

所有单元格的一致性迫使条带之间的一致性，因为对行或列求和必须产生相同的总数。 这将 2D 约束简化为两个独立的 1D 等分约束。 一旦两个投影都接受相同的块分解，笛卡尔积结构就确保所有$(k+1)^2$交集具有相同的基数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok_projection(coords, m, strip):
    coords.sort()
    n = len(coords)
    for i in range(0, n, strip):
        # each block must have exactly 'strip' points
        if i + strip > n:
            return False
    return True

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        pts = [tuple(map(int, input().split())) for _ in range(n)]

        m = k + 1

        if n % (m * m) != 0:
            print("No")
            continue

        strip = n // m

        xs = [p[0] for p in pts]
        ys = [p[1] for p in pts]

        if ok_projection(xs, m, strip) and ok_projection(ys, m, strip):
            print("Yes")
        else:
            print("No")

if __name__ == "__main__":
    solve()
```该解决方案首先强制执行由相等的像元大小所隐含的可分性条件。 然后提取 x 和 y 坐标列表并检查它们是否可以均匀地分为大小连续的块$strip = n/(k+1)$。 需要进行排序，以便块对应于潜在的条带边界。 

一个微妙的实现问题是重复的坐标。 排序仍然有效，因为我们只依赖计数，而不是严格的不平等差距。 块检查不会尝试将切割放置在相同的坐标内； 由于允许在非整数位置进行切割，因此我们始终可以将它们放置在点之间。

 输入：```
n = 4, k = 1
points = [(-2,-2), (-1,2), (1,3), (2,-1)]
```我们有$m=2$， 所以$m^2=4$， 因此$cell=1$,$strip=2$。

 | ---| ---| ---| ---| ---| ---|
 | 初始| [-2, -1, 1, 2] | 待定 | [-2, -1, 2, 3] | 待定 | 计算|
 | 分组| [-2，-1]，[1,2] | 有效 | [-2，-1]，[2,3] | 有效 | 是的 |

 两个投影都清晰地分成大小为 2 的两组，因此可以在组之间放置垂直线和水平线。 

这证实了结构仅取决于排序，而不取决于实际的几何形状。 

### 示例 2

 输入：```
n = 4, k = 1
points = [(-2,-2), (-2,2), (1,3), (-2,0)]
```我们还有$strip=2$。 

| 步骤| 已排序 x | x 块 | 排序 y | y 块 | 结果 |
 | ---| ---| ---| ---| ---| ---|
 | 初始| [-2,-2,-2,1] | 待定 | [-2,0,2,3] | 待定 | 计算|

 ## 边缘情况

 对于$k=0$，没有直线，整个平面是一个区域。 算法正确设置$m=1$,$strip=n$，因此两个投影都会通过，因为每个轴形成包含所有点的单个块。 

对于重复的点，所有相同的坐标，排序会产生一个平面数组。 由于块纯粹由计数定义，因此重复项不会干扰条带形成，并且算法在以下情况下仍然可以正确分区：$n$符合所需的整除条件。 

对于坐标极度不平衡的情况，例如 x 值很多相同而 y 值分散，即使 y 投影成功，x 投影也可能会失败。 该算法正确地拒绝了这种情况，因为两个轴必须独立支持统一块分解，而不仅仅是一个。
