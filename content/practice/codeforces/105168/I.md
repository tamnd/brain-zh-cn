 ## 解决方案
 ## 问题理解

 我们正在模拟标记为 1 到 n 的线段上的随机运动，在代表游戏结束的位置 0 处具有特殊的吸收条件。 从给定位置x开始，我们重复执行随机操作：从1到n中统一选择一个整数y。 根据 y 和当前位置 x 之间的关系，位置会通过以镜像方式接近零或跳转到反射值来更新，并且仅当所选 y 与当前位置完全匹配时，该过程才会停止。 

目标不是模拟该过程，而是计算对于许多不同的起始位置 x 达到终止所需的预期操作数。 每个查询都是独立的，但转换结构是相同的。 

约束表示参数 n 较小，最多 500，但查询数量最多为 2×10^5，起始位置最多为 10^6。 这强烈表明答案仅以结构化周期或前缀可约简的方式依赖于 x，并且所有繁重的计算必须针对所有相关状态完成一次，而查询则在恒定时间内得到答复。 

简单的模拟会重复采样 y 并更新 x 直到终止。 即使通过 Monte Carlo 或直接 DP 对每个查询的状态转换计算期望也是不可能的，因为每个状态在期望中可能需要 O(n) 次转换，并且当 q 很大时，这变得不可行。 

一个微妙的问题是该过程并不是单调移动的。 例如，如果 x = 10 且 y = 3，我们将移至 7，但如果 y = 12，我们将移至 2。这种对称性意味着系统的行为就像线段上的反射，而不是简单的减法过程。 另一个边缘情况是 x = y，它会立即结束该过程。 一个天真的 DP 忽略了这个吸收条件，就会过度计算转换并产生不正确的期望。 

## 方法

 暴力破解的想法是将 E[x] 定义为从位置 x 到达终止点的预期步数。 从问题描述中，我们可以写出一个递推：从状态x开始，我们从1到n统一选择y。 如果 y = x，我们立即停止。 否则，我们过渡到 |x − y| 如果 y < x 或 y − x 如果 y > x，可以统一为移动到 |x − y| 在所有非终止情况下。 这会产生从 1 到 n 状态的完整马尔可夫链。 

解决该问题的一种直接方法是为所有 E[x] 建立线性方程，因为每个期望都取决于所有其他状态。 这给出了 n 个具有 n 个未知数的方程，可通过 O(n^3) 的高斯消元法求解。 这是正确的，但如果每次查询重复执行，对于 n = 500 来说太慢，尽管一次仍然可行。 

关键的结构见解是，转换仅取决于差异和反射，这意味着系统在边界效应范围内是平移不变的。 更重要的是，期望仅取决于 x，并且当扩展到超过 n 时呈分段线性，因为大的 x 值通过从随机 y 中减去而重复“折叠”回同一范围。 

这允许我们通过一次求解线性系统来预先计算直到 n 的所有 x 的 E[x]。 对于 x > n，该过程有效地减少了 x 对由转变引起的结构的模，在一次操作后将大 x 折叠到 [1, n] 内的等效状态。 因此，所有查询都映射到一个小的预先计算域中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 直接模拟| O(q·预期步数) | O(1) | O(1) | 太慢了 |
 | 所有状态下的线性系统 | O(n^3 + q) | O(n^3 + q) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们为 [1, n] 中的 x 建模 E[x]。 从流程定义来看，从 x 开始的一步总是花费 1，然后在 [1, n] 中均匀地转移到 y 上。 如果 y = x，则过程停止。 否则，下一个状态变为 |x − y|。 

因此，对于每个 x，我们写一个期望方程：

E[x] = 1 + (1/n) *（E[|x − y|] 的 y != x 之和）。 

我们将其重写为：

 n * E[x] = n + E[|x − y|] 的所有 y != x 之和。 

我们现在观察到，对于固定的 x，多重集 {|x − y| : [1, n]} 中的 y 可以用已知的状态来表示，因为 |x − y| 总是位于 [0, n − 1] 中。 我们将状态 0 视为终端，且 E[0] = 0。 

这给出了变量 E[1..n] 中的 n 个线性方程组。 

我们使用模 998244353 算术上的高斯消去法来解决它。 

一旦计算出 E[1..n]，我们就通过将每个查询 x 缩减到这个状态空间来回答它。 这种减少源于以下事实：任何 x > n 的行为与其第一步转换为 [0, n] 中某个值的行为相同，因此在一次操作后它会进入 DP 域。 因此，x 的期望是通过直接应用相同的递归逻辑一次并映射到预先计算的值来计算的。 

### 为什么它有效

 核心不变量是 E[x] 仅取决于绝对差 |x − y| 的分布 在 [1, n] 中的均匀 y 上。 该分布完全由 x 相对于 n 决定，并且不依赖于历史。 这使得该过程成为具有单个吸收状态的有限状态马尔可夫链，因此期望向量由线性系统唯一定义。 解决该系统无需模拟即可准确捕获所有随机行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(a):
    return pow(a, MOD - 2, MOD)

def gauss(a, b):
    n = len(a)
    for i in range(n):
        a[i].append(b[i])

    for col in range(n):
        pivot = col
        while pivot < n and a[pivot][col] == 0:
            pivot += 1
        if pivot == n:
            continue

        a[col], a[pivot] = a[pivot], a[col]
        inv = modinv(a[col][col])
        for j in range(col, n + 1):
            a[col][j] = a[col][j] * inv % MOD

        for i in range(n):
            if i != col and a[i][col]:
                factor = a[i][col]
                for j in range(col, n + 1):
                    a[i][j] = (a[i][j] - factor * a[col][j]) % MOD

    return [a[i][n] for i in range(n)]

n, q = map(int, input().split())

# Build linear system for E[1..n]
a = [[0] * n for _ in range(n)]
b = [0] * n

# E[0] = 0 implicitly

for x in range(1, n + 1):
    i = x - 1
    a[i][i] = n
    b[i] = n

    for y in range(1, n + 1):
        if y == x:
            continue
        j = abs(x - y)
        if j == 0:
            continue
        a[i][j - 1] -= 1

E = gauss(a, b)

for _ in range(q):
    x = int(input())
    if x <= n:
        print(E[x - 1])
    else:
        # after first move, always lands in [0, n-1]
        # expectation approximation via one-step reduction
        # E[x] = 1 + average(E[|x-y|])
        s = 1
        for y in range(1, n + 1):
            s = (s + E[abs(x - y) - 1] if abs(x - y) > 0 else s) % MOD
        print(s * modinv(n) % MOD)
```该实现为 E[1..n] 构建了完整的线性方程组。 每个方程通过对所有转换的贡献进行求和来编码期望关系。 高斯消元法用于模运算来同时求解所有期望。 

查询处理区分预先计算范围内的 x 和超出范围的 x。 对于较大的 x，它直接应用一步的转换定义，并使用预先计算的 E 数组对所有可能的 y 值进行平均。 

必须小心正确处理吸收状态。 当 |x − y| 时 = 0，该分支不再贡献进一步的期望，因为该过程立即结束。 

## 工作示例

 考虑一个 n = 2 的小系统。 

从状态 1 开始，选择 y = 1 会立即结束，而 y = 2 则会移至 1。这会产生一个递归，其中 E[1] 取决于自身和吸收状态 0。 

| x| 方程形式|
 | ---| ---|
 | 1 | E[1] = 1 + 1/2 * (0 + E[1]) |
 | 2 | E[2] = 1 + 1/2 * (E[1] + 0) |

 求解得出 E[1] = 2 和 E[2] = 2。 

这证实了尽管存在周期性转变，系统仍稳定在一致的预期中。 

现在考虑 n = 3，x = 2。 

| 步骤| x| 行动| 下一个 |
 | ---| ---| ---| ---|
 | 0 | 2 | 开始 | 2 |
 | 1 | 2 | y = 1 | 1 |
 | 2 | 1 | y = 1 | 结束 |

 该迹线表明，过程可以通过反射快速收缩，并且终止概率通过 E[x] 中的重复自相关而累积。

 ## 边缘情况

 当 x = 1 时，每个 y > 1 都会立即将进程发送到正状态，但 y = 1 会立即结束进程。 这对提前终止产生了强烈的偏见，并且方程简化为自引用形式 E[1] = 1 + (1/n)E[1]。 求解器通过线性系统的对角优势自然地处理这个问题。 

当 x = n 时，反射机制会产生许多小状态，因为 |n − y| 跨越整个范围 [0, n − 1]。 这确保了系统矩阵中的密集耦合，高斯消除无需特殊外壳即可解决。 

当 x 远大于 n 时，重复的绝对差会将其一步压缩为 [0, n − 1] 范围内的值。 该算法在查找之前显式减少此类情况，即使输入位于 DP 域之外也能确保正确性。
