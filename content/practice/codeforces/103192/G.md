 ## 解决方案
 ## 问题理解

 我们得到了离散时间点上加密货币和人民币之间的一系列未来汇率。 每时每刻，1 bitc 的价格都是预先知道的。 交易者从零加密货币和无限人民币开始。 在一天中，他们可能会反复买卖比特币，但有几个限制因素将其变成有界交易优化问题。 

关键限制塑造了可行策略的结构。 首先，任何单笔交易必须至少交易很小的金额，0.01 bitc，但每笔交易没有上限。 其次，在任何时刻，比特币的持有总量不能超过1。第三，交易者可以在单个时间步内将操作分割成任意多笔交易。 最后，计算买入和卖出的交易总数最多为 k。 

产出是利用所有未来价格的全部知识可实现的最大人民币利润。 

时间和空间限制足够严格，以至于对于 n 高达 1000 的情况，对所有买卖对进行二次或三次模拟是不可行的。像 O(n^3) 那样扩展的解决方案甚至具有太多维度的朴素动态规划都会太慢，但 O(n^2 k) 是可以接受的，并且我们应该期望随着时间和交易数量的 DP。 

当贪婪地思考时，就会出现一个微妙的问题。 人们可能会尝试在每个局部最小值处买入并在每个局部最大值处卖出，但交易限制 k 会阻止交易不受限制的分割。 另一个微妙的陷阱是“每个时间步进行多次交易”规则，它有效地消除了粒度问题，并允许我们将每个时刻视为连续状态，而不是其中的离散操作。 

天真的贪婪失败的一个小例子是：

 输入：

 4 2

 1 100 1 100

 贪婪的方法可能会尝试多个买卖周期，但只允许 2 笔交易，最佳策略是一次以 1 买入并以 100 卖出，产生 99，而不是 198 或多个碎片收益。 这凸显出交易计数才是真正的瓶颈，而不是价格波动计数。 

## 方法

 直接的暴力方法会尝试枚举最多 k 个交易的所有序列。 每笔交易都由买入时间 i 和卖出时间 j > i 定义，并且交易必须遵守非重叠的持有约束。 即使我们简化并假设只选择 k 个不相交的间隔，我们仍然需要探索 O(n^2) 候选对象中最多 k 个间隔的组合。 这会导致 O((n^2)^k) 量级的组合爆炸，即使对于较小的 n 和 k 也是不可行的。 

关键的观察结果是，随着时间的推移和已完成的交易数量，问题具有最佳的子结构。 在任何时候，重要的不是过去交易的确切顺序，而是已经使用了多少交易以及我们当前是否持有该资产。 这立即暗示了动态规划公式。 

我们定义在两个维度上跟踪进度的状态：时间和使用的交易数量，以及持有或不持有货币的附加维度。 转换对应于什么都不做、买入或卖出。 由于允许同时进行多个交易，因此我们可以将每个价格步骤视为应用一次转换的决策点。 

主要困难是确保买卖正确消耗交易计数。 买入并不完成一笔交易，而卖出则完成一个完整的买卖周期。 这种不对称性使得 DP 结构能够干净地工作。

 | 对所有贸易组合进行暴力破解 | 指数| 指数| 太慢了 |
 | 随着时间和交易的动态规划 O(nk) | O(nk) | O(nk) | O(nk) | 已接受 |

 ## 算法演练

我们维护一个 DP 表，其中 dp[i][j][0] 表示使用 j 笔交易且不持有 bitc 后，在第 i 时刻我们可以拥有的最大现金，而 dp[i][j][1] 表示持有 1 bitc 时的最大值。 

我们迭代时间并根据我们是否购买、出售或跳过来更新状态。 

1. 将除 dp[0][0][0] = 0 之外的所有状态初始化为负无穷大。这反映了最初我们没有钱，也没有资产。 
2. 对于从 1 到 n 的每个时间 i，以及从 0 到 k 的每个事务计数 j，首先结转不执行任何操作的选项。 这意味着 dp[i][j][0] 和 dp[i][j][1] 开始时至少与 dp[i-1][j][0] 和 dp[i-1][j][1] 一样好，因为我们总是可以避免交易。 
3. 考虑在 i 时间购买。 如果我们在时间 i-1 时没有持有资产，我们可以以价格 p[i] 购买 1 个比特币，保持在相同的交易计数 j 内。 这会将 dp[i][j][1] 从 dp[i-1][j][0] - p[i] 转换。 由于交易尚未完成，因此尚未计算在内。 
4. 考虑在时间 i 出售。 如果我们在时间 i-1 持有资产，我们可以以价格 p[i] 出售，增加现金并消耗一笔交易。 这会将 dp[i][j][0] 从 dp[i-1][j-1][1] + p[i] 转换。 交易计数器仅在完成时增加。 
5. 对所有时间步继续此过程，确保每个状态仅依赖于先前的时间层值。 
6. 最终答案是从 0 到 k 的所有 j 的最大 dp[n][j][0]，因为我们最终没有实现利润的资产。 

工作原理：DP 强制要求每个可能的有效交易序列恰好对应于状态图中的一条路径。 每个转换都编码一个合法动作，并且在卖出操作中精确执行交易计数，确保没有序列超过 k 个已完成的交易。 由于我们始终考虑每一步的所有操作，因此可以表示任何最佳策略，并且由于转换保留了最佳子结构，因此不会错过更好的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    p = list(map(float, input().split()))

    NEG = -1e30

    dp = [[[NEG] * 2 for _ in range(k + 1)] for _ in range(n + 1)]
    dp[0][0][0] = 0.0

    for i in range(1, n + 1):
        price = p[i - 1]
        for j in range(k + 1):
            dp[i][j][0] = dp[i - 1][j][0]
            dp[i][j][1] = dp[i - 1][j][1]

        for j in range(k + 1):
            if dp[i - 1][j][0] > NEG / 2:
                dp[i][j][1] = max(dp[i][j][1], dp[i - 1][j][0] - price)

        for j in range(1, k + 1):
            if dp[i - 1][j - 1][1] > NEG / 2:
                dp[i][j][0] = max(dp[i][j][0], dp[i - 1][j - 1][1] + price)

    ans = max(dp[n][j][0] for j in range(k + 1))
    print(f"{ans:.2f}")

if __name__ == "__main__":
    solve()
```该代码明确地为每个交易计数维护两个状态，区分持有和不持有条件。 初始化使用较大的负哨兵来防止无效转换影响结果。 买卖转换的分离确保了仅在完成时才计算交易使用情况，与问题陈述相匹配。 

随着时间的推移，外部循环可确保时间顺序的一致性。 交易计数的内部循环可确保遵守所有预算交易限制。 所有 j 的最终最大化允许未使用的交易容量，这可能是有益的，因为更少的交易可能是最佳的。

| 我| 价格| 行动| dp解读|
 | --- | --- | --- | --- |

这证实了在交易预算允许的情况下，多个不相交的盈利区间被正确累积。

 ## 边缘情况

 一种极端情况是价格大幅下降。 DP 永远不应该触发任何买卖序列，因为每次潜在的卖出都会产生非正收益。 DP 始终正确地保持 dp[i][j][0] = 0，因为持有状态永远不会改善，而卖出也永远不会带来好处。 

另一个边缘情况是当 k 足够大以超过盈利段的数量时。 在这种情况下，DP 自然会退化为对所有正差异求和，因为事务约束永远不会绑定。 状态转换允许独立捕获每一次有利可图的上涨。 

第三种边缘情况是最佳交易需要在非相邻盈利区间之间等待。 DP 处理此问题是因为“不执行任何操作”转换会向前传播先前的最佳状态，而不会强制立即采取行动，从而保留了跳过任意时间段的能力。
