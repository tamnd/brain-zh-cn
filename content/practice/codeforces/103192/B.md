 ## 解决方案
 ## 问题理解

 该任务模拟操作系统中的经典页面替换场景。 我们收到一系列页面请求，其中每个请求都从固定的页面范围中请求一个页面。 内存的容量有限，因此一次只能容纳少量的页面。 每当所请求的页面当前不在内存中时，就会发生页面错误，如果内存已满，则必须逐出现有页面以腾出空间。 

我们对相同的请求序列模拟三种不同的驱逐策略：FIFO删除在内存中时间最长的页面，LRU删除最长时间未使用的页面，OPT删除下次使用时间最远或不再使用的页面。 对于每个策略，我们都会计算发生的页面错误数量。 输出要求我们报告哪种策略实现了最少的故障数，并输出该最小值。 如果多项政策并列，则可以打印其中任何一项。 

输入大小限制足够大，必须仔细实施每个驱逐选择的简单模拟。 请求长度可达十万，而不同页面的数量足够小，可以使用辅助数据结构进行直接模拟。 这种组合强烈表明每个算法必须在每个策略的近线性时间内实现，避免对内存状态进行任何重复的完整扫描。 

OPT 模拟中有一个微妙的陷阱。 在每次未命中时向前扫描以查找内存中所有页面的下一个使用的幼稚方法在最坏的情况下会降级为立方行为。 另一个常见的错误是忘记某个页面可能永远不会再次使用，这应该使其立即成为最佳驱逐候选者。 

边缘情况包括所有请求都相同、内存大小为 1 以及所有页面都不同（因此每次访问都是错误）的序列。 在这些情况下，FIFO、LRU 和 OPT 以可预测的方式表现不同，并且对“未再次找到”的不正确处理或不正确的时间戳更新将产生错误的比较。 

## 方法

 每个政策的直接模拟在概念上都很简单。 我们维护一个代表当前内存内容的列表或集合，并一一处理请求。 一旦命中，我们就会根据策略更新元数据。 如果未命中，我们会增加错误计数器，如果存在空间则插入页面，或者根据规则逐出页面。 

FIFO 是最简单的：我们按插入顺序维护一个页面队列。 每次错过都会在后面插入新页。 如果内存已满，我们从前面删除。 这是可行的，因为 FIFO 仅取决于到达时间，不需要跟踪插入顺序之外的使用模式。 

LRU 通过跟踪每个页面的最后访问时间来扩展这个想法。 每次访问时，我们都会更新其时间戳。 当需要驱逐时，我们选择最后访问时间最短的页面。 对内存中所有页面的简单搜索已经足够快了，因为内存大小 k 最多为 100，因此每次未命中扫描 k 个元素是可以接受的。 

OPT 是唯一需要未来知识的算法。 对于当前内存中的每个页面，我们需要知道下次何时使用它。 每次驱逐时在请求列表中向前扫描每个页面的简单重新计算会太慢。 关键的观察是，对于序列中的每个位置，我们可以使用向后扫描或通过维护下次使用指针来预先计算每个页面的下一次出现。 然后，可以通过比较 O(k) 中预先计算的下次使用索引来决定每次驱逐。 

一旦计算完所有三个计数器，答案就是其中的最小值。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 先进先出模拟 | O(n) | O(m + k) | 已接受 |
 | LRU 模拟 | O(nk) | O(nk) | O(m + k) | 已接受 |
 | 具有下次使用预处理功能的 OPT | O(nk) | O(nk) | O(n + m) | 已接受 |

 ## 算法演练

 我们在同一请求序列上独立计算 FIFO、LRU 和 OPT。 

### 先进先出模拟

 1. 初始化一个空队列和一组用于成员跟踪的队列。 
2. 对于每个请求，如果页面已在集合中，则除了继续之外不执行任何操作。 
3、如果不在set中且有剩余空间，则插入队列并set。 
4. 如果不在集合中并且内存已满，则删除队列的前面并将其从集合中删除，然后插入新页面。 

这样做的原因是 FIFO 的决策仅取决于到达顺序，因此队列不变量直接代表逐出优先级。 

### LRU 模拟

 1. 维护一个字典，存储当前内存中每个页面的最后访问时间。 
2. 对于每个请求，如果已经在内存中，则将其上次访问时间更新为当前索引。 
3. 如果它不在内存中并且有空间，则将其与其当前索引一起插入。 
4. 如果不在内存中且内存已满，则扫描内存中所有页面，找到最后访问时间最小的并将其逐出，然后插入新页面。 

正确性来自于这样的事实：最小的时间戳总是对应于全局序列顺序下最近最少使用的页面。 

### OPT模拟

 1. 预先计算一个数组`next_pos[i]`表示位置 i 处的页面将再次出现的下一个索引，如果不再出现则为无穷大。 
2. 维护当前内存中的一组页面。 
3. 维护每个页面的下一个出现索引。 
4. 对于每个请求，使用预先计算的表更新或分配其下一个出现的请求。 
5. 未命中时，如果内存已满，则迭代内存中的页面并选择下一个出现索引最大的页面。 

关键思想是未来的知识被简化为每个位置的单个预计算值，因此每次驱逐都是对 k 值的局部最大查询。 

### 为什么它有效

 在任何时间点，FIFO按插入顺序对页面进行排序，LRU按上次访问时间对页面进行排序，OPT按下一次使用位置对页面进行排序。 每个算法都保持一致的排序标准，不依赖于未来的更新，除了 OPT 之外，这是通过预处理解决的。 驱逐决策总是在策略定义的排序下选择极值，因此模拟状态仍然等于每个策略的理论定义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

def simulate_fifo(req, k):
    from collections import deque
    q = deque()
    s = set()
    faults = 0

    for x in req:
        if x in s:
            continue
        faults += 1
        if len(s) < k:
            s.add(x)
            q.append(x)
        else:
            old = q.popleft()
            s.remove(old)
            s.add(x)
            q.append(x)

    return faults

def simulate_lru(req, k):
    mem = {}
    faults = 0

    for i, x in enumerate(req):
        if x in mem:
            mem[x] = i
        else:
            faults += 1
            if len(mem) < k:
                mem[x] = i
            else:
                victim = min(mem, key=mem.get)
                del mem[victim]
                mem[x] = i

    return faults

def simulate_opt(req, k):
    n = len(req)
    nxt = [n] * n
    last = {}
    for i in range(n - 1, -1, -1):
        x = req[i]
        nxt[i] = last.get(x, n)
        last[x] = i

    mem_next = {}
    faults = 0

    for i, x in enumerate(req):
        if x in mem_next:
            mem_next[x] = nxt[i]
        else:
            faults += 1
            if len(mem_next) < k:
                mem_next[x] = nxt[i]
            else:
                victim = max(mem_next, key=mem_next.get)
                del mem_next[victim]
                mem_next[x] = nxt[i]

    return faults

def main():
    n, m, k = map(int, input().split())
    req = list(map(int, input().split()))

    fifo = simulate_fifo(req, k)
    lru = simulate_lru(req, k)
    opt = simulate_opt(req, k)

    best = min(fifo, lru, opt)

    if best == fifo:
        print("FIFO")
    elif best == lru:
        print("LRU")
    else:
        print("OPT")

    print(best)

if __name__ == "__main__":
    main()
```FIFO 部分依赖于与一组配对的队列，因此成员资格检查保持恒定的时间。 LRU 部分使用以页面值作为键的字典，并以时间戳作为值，这已经足够了，因为内存大小很小，因此找到最小值仍然有效。 

OPT 实现由预处理通道驱动，该预处理通道反向计算下一个出现位置。 这将原本昂贵的“向前扫描”操作转变为每个请求的恒定时间查找。 

## 工作示例

 考虑一个小序列，其中内存大小为 2，请求为`1 2 3 1 2`。 

### FIFO 跟踪

 | 步骤| 请求| 内存| 被驱逐| 故障|
 | --- | --- | --- | --- | --- |
 | 1 | 1 | [1] | - | 1 |
 | 2 | 2 | [1,2]| - | 2 |
 | 3 | 3 | [2,3]| 1 | 3 |
 | 4 | 1 | [3,1]| 2 | 4 |
 | 5 | 2 | [1,2]| 3 | 5 |

 FIFO 行为严格按照到达顺序循环页面，忽略重用模式。 

### LRU 跟踪

 | 步骤| 请求| 内存（上次使用）| 被驱逐| 故障|
 | --- | --- | --- | --- | --- |
 | 1 | 1 | {1:1} | - | 1 |
 | 2 | 2 | {1:1,2:2} | - | 2 |
 | 3 | 3 | {2:2,3:3} | 1 | 3 |
 | 4 | 1 | {3:3,1:4} | 2 | 4 |
 | 5 | 2 | {1:4,2:5} | 3 | 5 |

 LRU更倾向于驱逐最长时间未使用的页面，这与时间戳排序相匹配。 

### OPT 追踪

 | 步骤| 请求| 下一步用途 | 内存| 被驱逐| 故障|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 1 | 未来4 | [1] | - | 1 |
 | 2 | 2 | 未来5 | [1,2]| - | 2 |
 | 3 | 3 | 从来没有| [1,3]| 2 | 3 |
 | 4 | 1 | 完成 | [1,3]| - | 3 |
 | 5 | 2 | 从来没有| [2,3]| 1 | 4 |

 OPT 始终避免驱逐即将需要的页面，而是丢弃那些将来不会使用或重用程度最高的页面。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(nk) | O(nk) | FIFO、LRU、OPT 中的每一个每次未命中最多扫描 k 个页面，并且 k ≤ 100 使得此可行 |
 | 空间| O(n + m) | OPT 预处理存储下次使用信息，所有模拟跟踪当前内存状态 |

 这些约束保证即使是三重模拟也仍然足够快。 内存大小限制 k 是无需高级数据结构即可实现简单实现的关键因素。

 ## 边缘情况

 一个关键的边缘情况是某个页面再也不会出现在 OPT 中。 例如，内存大小为 2 且请求顺序`1 2 3 4`,处理页面时`3`， 两个都`1`和`2`没有未来的用途。 该算法将它们的下一次使用价值分配为无穷大，驱逐可以选择其中之一。 这与 OPT 定义相匹配，因为任何没有未来参考的页面同样是最佳删除选项。 

另一个边缘情况是重复访问模式，例如`1 2 1 2 1 2`内存大小为 2。 FIFO 和 LRU 都避免了预热后的故障，OPT 也产生零额外故障。 这证实了所有策略都集中在稳定的重用模式上，任何差异都将表明时间戳更新不正确或无法更新下次使用指针。
