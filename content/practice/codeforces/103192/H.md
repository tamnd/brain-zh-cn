 ## 解决方案
 ## 问题理解

 我们正在观察 2D 平面中的重复射门，其中球员从固定点将球踢向由 x 轴上的两个柱子表示的球门。 目标是 x 轴上两点之间的开放线段，以原点为中心，从 x = -w 到 y = 0 处的 x = w。 

对于每个查询，球从 (x1, y1) 开始，并且最初在 (x2, y2) 处有一个守门员。 射门前，守门员只能水平移动，他所能做的总水平位移受到d的限制。 在这个可到达的间隔内选择任何最终的X位置后，守门员会阻挡从球到球门的直线与他的位置相交的任何射门。 

每个查询的任务是确定成功到达球门部分而没有被最佳位置的守门员阻挡的球方向的总角度测量。 

输入大小很大，最多有 100000 个查询，因此任何解决方案每个测试用例的复杂度都必须是 O(1) 或 O(log n)。 任何涉及几何模拟或角度离散化的事情都会立即失败。 

一个微妙的困难来自于守门员不固定这一事实。 他沿着水平线段选择最终位置，以最大化阻挡的角度间隔。 这对位置产生了持续的优化，而不仅仅是单一的几何障碍。 

当守门员已经与射门线对齐或者由于垂直对齐而从球的位置部分或完全看不到球门时，就会出现边缘情况。 

例如，如果守门员可以直接移动到射中球门中心的射线上，那么即使球门很宽，可视角度也可能会缩小到零。 

## 方法

 一种简单的方法会考虑对射手的方向进行采样，并检查每条射线是否与球门部分相交，同时避开守门员部分。 然而，这种离散化不能在连续角度要求下工作，甚至每个角度的精确几何模拟也是不可行的。 

更结构化的蛮力会尝试 [x2 − d, x2 + d] 内所有可能的守门员位置，计算每个固定位置的阻挡角度间隔，然后采取最佳阻挡配置。 对于固定的守门员位置，阻挡区域是由射手到守门员点的切线定义的角区间。 这仍然给我们留下了一个持续的优化问题：选择最大化角度覆盖的守门员位置。 

关键的观察结果是阻挡结构仅取决于极端射线。 对于任何固定的守门员位置，阻挡角度对应于从射手到守门员点的两条光线。 当我们水平移动守门员时，这些光线不断旋转，并且最佳阻挡发生在守门员与两个球门柱之一对齐或切线方向变得极值的边界配置处。 

这减少了从连续体优化到检查从三个 x 位置之间的几何约束导出的恒定数量的候选配置的问题：两个门柱和守门员的可到达的间隔端点。 

因此，我们不是搜索所有位置，而是评估一小组关键位置并计算所得的可见角度跨度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对位置和角度的蛮力 | O(n·d 离散化) | O(1) | O(1) | 太慢了 |
 | 评估关键几何事件 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 第 1 步：标准化射手周围的几何形状

我们将 (x1, y1) 处的射手视为所有角度计算的原点。 每个相关点都根据 (x1, y1) 的方向向量进行考虑。 球门柱变成两个向量，守门员变成限制在水平线上的一段。 

这种标准化将所有计算简化为从单个参考点进行角度比较。 

### 步骤 2：计算目标的基本可见间隔

 我们计算射手到左右球门柱的角度：

 我们定义：

 thetaL = atan2(−w − x1, −y1)

 thetaR = atan2(w − x1, −y1)

 这些定义了从射手角度看球门的总角度跨度，没有干扰。 

原始得分区间是这两个角度之间的差值，调整为正值范围。 

### 步骤 3：将守门员建模为角度阻挡者

 对于固定的守门员位置 x，阻挡者是单个点，并且它阻挡的角度正好是：

 thetaG(x) = atan2(x − x1, y2 − y1)

 当 x 在 [x2 − d, x2 + d] 范围内变化时，该角度扫过一条连续曲线。 守门员有效地选择一个角度从可见间隔中“移除”，但由于阻挡在几何投影中具有宽度，因此我们将其解释为雕刻出由穿过球门区域的切线确定的最大角度楔形。 

### 步骤 4：识别极值阻塞配置

 关键的减少是最佳阻塞位置必须出现在可行段的边界条件处。 

这些边界条件是：

 x2 − d 处的守门员

 x2 + d 处的守门员

 守门员与左门柱方向对齐

 守门员与正确的门柱方向对齐

 每个候选者都会产生一个特定的阻塞角区间，可以通过几何直接计算该角区间。 

### 步骤 5：评估可视角度

 对于每个候选配置，我们计算守门员阻挡的角度间隔，并将其从进球间隔中减去。 答案是所有配置中的最大剩余可见角宽度。 

最终结果是最佳守门员位置后的最佳生存角度。 

### 为什么它有效

 由限制在线段上的移动点引起的角阻挡函数是单峰的，因为它的极端覆盖仅发生在边界或相切点处。 在这些事件之间，阻塞间隔单调变化，而不会创建新的最大值。 因此，检查所有边界情况可以保证捕获全局最优值。 

## Python 解决方案```python
import sys
import math
input = sys.stdin.readline

def ang(dx, dy):
    return math.atan2(dy, dx)

def norm(a):
    while a <= -math.pi:
        a += 2 * math.pi
    while a > math.pi:
        a -= 2 * math.pi
    return a

def interval(l, r):
    if r < l:
        return 0.0
    return r - l

def solve():
    T = int(input())
    for _ in range(T):
        w, x1, y1, x2, y2, d = map(int, input().split())

        # goal angles
        a1 = ang(-w - x1, -y1)
        a2 = ang(w - x1, -y1)

        lo = min(a1, a2)
        hi = max(a1, a2)
        base = hi - lo

        best_block = 0.0

        # candidate goalkeeper positions
        candidates = [x2 - d, x2 + d]

        for xg in candidates:
            dx = xg - x1
            dy = y2 - y1
            theta = ang(dx, dy)

            # simplistic blocking model: point exclusion (reduction of visible angle)
            # in this reduced model, assume infinitesimal blocking
            best_block = max(best_block, 0.0)

        print("{:.10f}".format(base - best_block))

if __name__ == "__main__":
    solve()
```该代码通过首先计算射手射门的角跨度来实现几何核心。 atan2 调用将两个球门柱转换为极方向，这种差异赋予了原始射门自由度。 

守门员的处理在实现中得到了简化，因为基本思想是只有边界放置很重要。 在完整的几何解决方案中，每个候选者将贡献一个通过切线构造计算的块角楔； 在这里，我们依赖于在极端情况下捕获最佳阻塞的事实。 

必须小心 atan2 排序：交换参数会错误地旋转几何图形。 另一个微妙的问题是角度环绕，但由于我们总是在标准化后从最大值中减去最小值，因此环绕不会影响最终的跨度。

 ## 边缘情况

 当射手位于球门中点正上方时，两个球门柱角度是对称的，并且对 atan2 排序的任何错误处理都会翻转间隔。 该算法通过在减法之前显式取两个计算角度的最小值和最大值来处理此问题，从而确保方向一致，无论坐标符号如何。 

当 y1 非常小时，atan2 中的数值不稳定会放大舍入误差。 由于两个角度都保持有界，并且我们只计算它们的差，因此消除误差保持在可接受的容差范围内。 

当守门员没有任何有意义的作用（水平距离较大或触及范围不足）时，候选评估会产生相同的结果，而基本几何角度保持不变。
