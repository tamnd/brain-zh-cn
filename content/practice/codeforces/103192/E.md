 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，描述从最内层到最外层的“嵌套结构”。 字符串中的每个字符代表一层：第一个字符对应于中心，每个下一个字符用更大的封闭框架包裹前一个结构。 

任务是构建该结构的 2D ASCII 绘图。 最内层是一个非常小的“核心”，由两个水平单元组成。 每层对称地向外扩展，增加边界矩形，并且根据相应的二进制数字是0还是1，每层都使用特定的字符进行绘制。其中一个字符代表肉，另一个代表面包，但对于构​​建过程来说，重要的是每一层都有自己的填充/边框字符。 

几何图形是问题的关键部分：每个新层都会通过在所有边上用一个单元格厚的边框包裹当前图形来增加当前图形的大小，但该结构在左侧是开放的，并且右侧边界明确地用竖线字符标记。 这会产生一个嵌套的“开放式三明治”形状，其中每一层都明显大于前一层。 

约束允许字符串长度最大为 1000。这意味着尝试逐个单元模拟所有内容的简单构造在大约 O(n^2) 或 O(n^2) 个单元总数中仍然可行，因为最终网格大小在两个维度上都随 n 线性增长，导致大约 O(n^2) 总输出大小。 任何比字符串长度的二次方更糟糕的东西都是不安全的。 

有一些边缘情况值得明确说明。 如果字符串的长度为 1，我们只打印两个单元格的基本核心，根本不进行包装。 如果所有字符都相同，输出仍然具有多个不同的层，但视觉上结构变得统一。 另一个微妙的情况是左侧不受可见边框字符的限制，因此任何错误地将右侧边界镜像到左侧的解决方案都会产生对称矩形，这是不正确的。 

## 方法

 思考这个问题最直接的方法就是从中心向外构建结构。 我们从最内层开始，它始终是两个相同字符的一小段水平部分。 然后，对于字符串中的每个附加字符，我们用一个更大的矩形包裹当前网格，该矩形的边框使用与该层关联的字符。 

在强力模拟中，我们实际上可以维护一个 2D 网格，并为每个新层分配一个新的更大网格并将以前的内容复制到中心。 复制后，我们填充新的边界。 这是有效的，因为嵌套结构被显式定义为递归扩展。 然而，这种方法在每一步都会重新计算大量内存移动。 如果最终网格的大小与 n × n 成正比，并且我们将其重建 n 次，则由于重复复制，在最坏的解释中，总工作量将变为 O(n^3)。 

关键的观察是我们永远不需要从头开始重建整个网格。 相反，我们可以先直接计算最终尺寸，分配一次最终网格，然后将每一层的边界放置在正确的位置。 每层对应一个矩形框，其坐标可以根据距中心的距离得出。 这减少了从重复构建到单层绘图过程的问题。 

因此，最佳解决方案变成了一个坐标问题：确定中心，计算每个层如何扩展边界框，然后将每个层的边界绘制一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 逐层重建| O(n^3) | O(n^3) | O(n^2) | O(n^2) | 太慢了 |
 | 直接坐标构建 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

## 算法演练

 我们通过预先计算每一层的位置来一次构建最终的网格。 

1.我们首先确定最终结构的尺寸。 最内层占据一小段长度为二的水平段，每增加一层，结构的所有边都向外扩展一个单位。 这意味着高度和宽度都随着层数线性增长，因此我们可以直接计算与 n 成正比的最终尺寸。 
2. 我们分配一个最初填充占位符值的二维网格。 该网格最终将包含完整的绘图，因此我们避免稍后调整大小。 
3. 我们将中心区域定义为最内层。 我们将第一个字符的表示（两个相同的符号）放置在中心行，水平居中。 这奠定了所有外层扩展的基础。 
4. 对于字符串中的每个后续字符，我们确定其层索引并计算其边界矩形。 每层 i 对应一个矩形，该矩形在每个方向上都比层 i−1 大一个单位。 
5. 我们用图层的字符填充当前图层的顶部和底部边界。 这创建了框架的水平结构。 
6. 我们从概念上填充图层的左边界，但由于结构在左侧是开放的，因此我们不会在那里绘制完整的垂直边框。 相反，我们只确保内部由先前的层正确界定。 
7. 我们用竖线字符填充层的右边界，这是该结构唯一严格的垂直封闭。 这可确保嵌套形状在视觉上保持固定在右侧。 
8. 我们重复此过程，直到放置完所有层。 

绘制完所有图层后，将逐行打印网格。 

### 为什么它有效

 每一层都对前一层形成严格的包含，并且扩展规则保证第 i 层完全包围第 i−1 层，而不会出现重叠歧义。 由于每个图层都是使用距中心的固定几何偏移量绘制的，因此不会有两个图层发生错误干涉。 该构造实际上是层上的归纳：假设第 i−1 层正确放置，第 i 层在其周围添加一个单元格边框，从而保持正确性并保持嵌套不变式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    # Each layer expands the shape; final height and width are linear in n
    h = 2 * n
    w = 2 * n

    grid = [[' ' for _ in range(w)] for _ in range(h)]

    mid_row = h // 2
    mid_col = w // 2

    # base layer: two characters in the center
    base_char = '*' if s[0] == '0' else '-'
    grid[mid_row][mid_col - 1] = base_char
    grid[mid_row][mid_col] = base_char

    top = mid_row
    bottom = mid_row
    left = mid_col - 1
    right = mid_col

    # expand layer by layer
    for i in range(1, n):
        c = '*' if s[i] == '0' else '-'

        top -= 1
        bottom += 1
        left -= 1
        right += 1

        # top and bottom borders
        for j in range(left, right + 1):
            grid[top][j] = c
            grid[bottom][j] = c

        # left side is conceptually open, so we do not draw a full border

        # right boundary is closed with '|'
        for j in range(top, bottom + 1):
            grid[j][right] = '|'

        # ensure interior is preserved implicitly

    # print result (trim empty rows)
    for row in grid:
        line = ''.join(row).rstrip()
        if line:
            print(line)

if __name__ == "__main__":
    solve()
```该实现从中心向外构建结构。 变量`top`,`bottom`,`left`， 和`right`跟踪活动图层的当前边界框。 每次迭代都会将此框在每个方向上扩展一个单位，与递归包装的定义相匹配。 

最微妙的设计选择是我们从不重建以前的层。 相反，我们只扩展当前边界并绘制新层的边缘。 右边界明确写为`|`，而左侧则通过不强制垂直的墙壁而在视觉上保持开放。 

## 工作示例

 考虑一个简短的输入，例如`01`。 第一个字符产生一个小的两单元中心。 第二个字符用一个更大的矩形包裹它，其顶部和底部填充有`-`，其右边缘标记为`|`。 

| 步骤| 层 | 顶部 | 底部| 左| 对| 行动|
 | --- | --- | --- | --- | --- | --- | --- |
 | 0 | 0 | 0 | 0 | c | c+1 | 放置底座“**”|
 | 1 | 1 | -1 | 1 | c-1 | c+2 | 添加边框，设置右墙 |

 该痕迹显示了结构如何在保留中心核心的同时对称生长。 

现在考虑`001`。 前两层是肉，产生嵌套`*`区域，最后一层切换为面包字符`-`，产生一个在视觉上主导所有内部结构的外部框架。 这表明层标识是独立的，并且每个层仅完全覆盖其自己的边界而不修改内部内容。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n^2) | O(n^2) | 每层绘制一个完整的矩形边界，所有层的总面积之和是 n | 的二次方。 
| 空间| O(n^2) | O(n^2) | 最终的网格存储扩展结构的所有字符 |

 约束条件允许 n 最大为 1000，因此二次构造可以轻松满足时间和内存限制。 由于每个单元格被写入固定次数，因此常数因子很小。

 ## 边缘情况

 对于像这样的单字符字符串`0`，算法立即放置基本的两单元核心并且不执行扩展。 由于循环从索引 1 开始，因此不会发生边界调整，因此输出保持最小且正确。 

对于像这样的字符串`0000`，每一层都使用相同的字符。 每次迭代仍然扩展边界，但视觉上的结果是一系列相同的帧。 该算法仍然有效，因为它不依赖于层之间的字符差异，仅依赖于几何扩展。 

对于交替输入，例如`010101`，每次扩展都会切换绘图字符。 由于每一层仅完全覆盖自己的边界，因此层与层之间不会发生干扰，并且内层保持不变的不变性确保了正确性。
