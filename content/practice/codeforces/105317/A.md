 ## 解决方案
 ## 问题理解

 我们有两个整数。 一个代表一个三明治的成本，另一个代表胡安有多少钱。 胡安只想把钱花在完整的三明治上，然后送给朋友，每个朋友只收到一个三明治。 

任务是确定他能买得起多少个完整的三明治，这相当于他可以请多少个朋友。 由于每个三明治的成本是固定的，因此问题就简化为将可用资金总额除以每个三明治的价格并仅计算完整购买量。 

约束条件达到$10^9$对于这两个值，这立即排除了一次花费一个三明治的任何模拟。 重复减去成本的循环最多需要$10^9$最坏情况下的迭代次数，远远超出了 Python 中一秒所能容纳的范围。 唯一可行的方法是恒定时间算术计算。 

不存在像数组或图这样棘手的结构边缘情况，但围绕整数除法语义仍然存在微妙的正确性问题。 结果必须是完整三明治的数量，这意味着必须放弃部分负担能力。 例如，如果成本为 5，金钱为 4，则正确答案为 0，而不是由于舍入错误而导致的 0.8 或 1。 任何使用浮点除法的实现都面临着附近大输入的精度问题。$10^9$。 

## 方法

 这种天真的做法直接反映了故事。 我们不断地从胡安的钱中减去一个三明治的成本，直到他再也买不起另一个三明治。 每减少一次就相当于购买一个三明治并增加服务的朋友数量。 这是正确的，因为每个步骤都会减少有效离散单位的剩余预算。 

然而，当成本较小且预算较大时，这种方法会变得极其缓慢。 如果成本为 1，金钱为$10^9$，我们将执行$10^9$迭代，每次都在不断地工作。 这已经超出了典型的时间限制几个数量级。 

关键的观察结果是，这种重复的减法恰好是整数除法。 我们可以直接计算成本符合预算的次数，而不是模拟每次购买。 这将整个过程简化为单个算术运算。 

问题的结构保证了购买之间的独立性，没有折扣，没有累积效应，并且没有将购买联系在一起的约束。 这使得商$m \div c$足以描述最终答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（重复减法）| O(m/c) | O(1) | O(1) | 太慢了 |
 | 最优（整数除法）| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取两个整数$c$和$m$来自输入。 这些代表每个三明治的成本和可用资金总额。 
2. 使用整数除法计算预算中有多少个完整的三明治$m // c$。 此操作会自动丢弃任何剩余部分，这对应于无法购买完整三明治的无法使用的剩余资金。 
3. 将计算出的值输出为Juan可以接待的好友数量。 

### 为什么它有效

 每个三明治的消耗量都是准确的$c$货币单位，并且无法将不同三明治的部分货币组合起来。 任何有效的购买序列都必须消耗大量资金$c$。 因此，适合的有效块的最大数量$m$正好是$m$除以$c$。 任何安排都不能超过这个，因为这意味着超出总预算，也没有任何安排可以更小，因为我们总是可以贪婪地购买三明治，直到剩余的钱少于$c$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

c, m = map(int, input().split())
print(m // c)
```该解决方案直接应用整数除法。 使用`//`至关重要，因为它确保了楼层划分，从而正确地模拟了仅计算完整三明治的情况。 

不需要循环或额外的内存。 整个计算是解析输入后的单个恒定时间操作。 

## 工作示例

 ### 示例 1

 输入：```
3 10
```我们跟踪计算：

 | 步骤| c | 米 | 米//c| 迄今为止的输出 |
 | ---| ---| ---| ---| ---|
 | 开始| 3 | 10 | 10 - | - |
 | 计算| 3 | 10 | 10 3 | 3 |

 结果是 3，因为 10 包含三个大小为 3 的完整组，还有 1 个货币单位未使用。 

这证实了剩下的钱并不能用来购买额外的三明治。 

### 示例 2

 输入：```
5 4
```| 步骤| c | 米 | 米//c| 迄今为止的输出 |
 | ---| ---| ---| ---| ---|
 | 开始| 5 | 4 | - | - |
 | 计算| 5 | 4 | 0 | 0 |

 这里的预算连一个三明治都不够，所以答案是 0。这表明该算法正确处理了以下情况：$m < c$。

 ## 边缘情况

 一种边缘情况是预算小于成本。 用于输入`c = 10, m = 3`，算法计算`3 // 10 = 0`。 这是正确的，因为买不到完整的三明治，整数除法自然会强制执行此操作，无需特殊处理。 

另一个边缘情况是预算可以被成本整除，例如`c = 4, m = 12`。 计算结果`12 // 4 = 3`，并且没有余数。 该算法正确计算所有可用购买，而不会过度计算。 

最后一种情况是两个值都处于其最大尺寸时，例如`c = 10^9, m = 10^9`。 结果是`1`，并且计算仍然安全，因为 Python 的整数算术本身可以处理大整数，并且在竞争性编程环境中，除法在这种规模下是常数时间。
