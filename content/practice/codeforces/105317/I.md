 ## 解决方案
 ## 问题理解

 我们得到一根弦，我们可以精确地选择一个切割位置，将其分成左部分和右部分。 对于每次切割，我们都会查看得到的两个字符串，并询问每个字符串是否可以重新排列成回文。 我们不需要直接形成回文，只需检查字符的某种排列是否可以形成回文。 

当至多一个字符具有奇数频率时，字符串可以排列成回文。 这个条件是将问题从组合重排问题转变为频率计数问题的核心属性。 

任务是计算有多少个分割位置产生两个都满足回文置换条件的子串。 

输入长度可以达到100000，这排除了任何为每个分割从头开始重新计算字符频率的解决方案。 在最坏的情况下，对所有分割点进行二次扫描并重新计算每个分割将需要大约 10^10 次操作，这远远超出了时间限制。 任何可行的解决方案都必须重用前缀信息并以恒定或接近恒定的时间更新每个位置的状态。 

当解决方案仅检查拆分的一侧或错误地假设如果完整字符串“良好”，则所有分区的行为类似时，就会出现微妙的失败情况。 例如，考虑字符串`aabbb`。 完整的字符串不相关，但有一些分割，例如`aa | bbb`之所以有效，是因为双方至多有一个奇数频率特征。 幼稚的全局检查将完全忽略对分割位置的依赖。 

如果在没有缓存的情况下重复重新计算前缀频率，则会出现另一个问题。 对于像这样的字符串`abababab...`，每个分割的重复计数退化为重复的完整扫描，它默默地通过了小测试，但在约束下失败了。 

## 方法

 蛮力的想法很简单。 对于每个可能的剪切位置，计算左子串和右子串的字符频率，然后检查两个频率表中的“最多一个奇数计数”条件。 这是正确的，因为它直接独立地在每个分割处应用定义。 问题在于它的成本。 每次分割需要扫描左侧最多 O(n) 个字符，右侧最多扫描 O(n) 个字符，总共产生 O(n^2) 工作量。 

关键的观察结果是，没有必要从头开始重新计算频率。 一旦我们知道了整个字符串中每个字符的频率，我们就可以在从左到右扫描时保持前缀频率。 后缀频率隐含地确定为总频率与前缀频率之间的差。 这使得每次分割在恒定时间内更新双方的奇数计数信息成为可能。 

关键的结构是回文条件仅取决于频率奇偶性，而不取决于精确计数。 这意味着我们可以维护和更新一个小型固定大小的数组（典型约束中为 26 个字母）并跟踪当前有多少字符在前缀和后缀中具有奇数频率。 分割点的每次移动都会将一个字符的贡献从后缀翻转到前缀，从而使我们能够更新奇偶校验计数器而无需重新扫描。

 | 蛮力 | O(n^2) | O(n^2) | O(26) | 太慢了 |
 | 最佳| O(26n) | O(26n) | O(26) | 已接受 |

 ## 算法演练

 我们将字符串视为小字母表（通常是小写字母）上的序列。 

1. 计算整个字符串中每个字符的总频率。 这表示进行任何拆分之前的后缀状态。 
2. 用全零初始化前缀频率数组，因为最初前缀是空的。 此时，所有字符都属于后缀。 
3. 跟踪两个计数器：一个用于记录前缀中出现奇数频率的字符数量，另一个用于记录后缀。 最初，前缀的奇数计数为零，后缀的奇数计数可以从总频率数组中计算出来。 
4. 从左向右扫动分割位置。 对于每个字符，我们从概念上将其从后缀移动到前缀。 这是通过减少后缀中的计数并增加前缀中的计数来完成的。 
5. 更新当前字符的计数后，调整前缀和后缀的奇数计数器。 由于每一步只有一个字符发生变化，因此只有该字符可以影响奇偶校验，因此所有更新都是 O(1)。 
6. 对于每个分割位置，检查前缀和后缀是否最多有一个奇数频率的字符。 如果是这样，那么这种分割是有效的并且有助于找到答案。 
7. 继续直到最后一个有效的分割位置，即最后一个字符之前。 

我们可以增量处理每个分割的原因是奇偶校验更改是本地的。 移动一个字符只会切换该字符在前缀和后缀中的奇偶校验，因此可以动态维护全局有效性条件。 

### 为什么它有效

 正确性取决于回文排列条件仅取决于奇偶校验计数的事实。 在每次分割时，前缀和后缀频率向量完全取决于每个字符从后缀转移到前缀的次数。 由于每一步只修改一个角色的成员资格，因此状态转换保留了奇偶校验跟踪的完全正确性。 因此，检查每一步的条件相当于从头开始重新计算，但没有重复的成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)
    
    if n <= 1:
        print(0)
        return

    # assuming lowercase English letters
    total = [0] * 26
    for ch in s:
        total[ord(ch) - 97] += 1

    pref = [0] * 26

    def count_odd(arr):
        c = 0
        for x in arr:
            c += x & 1
        return c

    suffix = total[:]
    pref_odd = 0
    suff_odd = count_odd(suffix)

    ans = 0

    for i in range(n - 1):
        idx = ord(s[i]) - 97

        # remove from suffix
        if suffix[idx] & 1:
            suff_odd -= 1
        suffix[idx] -= 1
        if suffix[idx] & 1:
            suff_odd += 1

        # add to prefix
        if pref[idx] & 1:
            pref_odd -= 1
        pref[idx] += 1
        if pref[idx] & 1:
            pref_odd += 1

        if pref_odd <= 1 and suff_odd <= 1:
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现维护两个频率阵列和两个奇偶校验计数器。 微妙的部分是正确更新奇数计数器：在更改频率之前，我们删除其先前的奇偶校验贡献，然后应用更新，然后添加其新贡献。 这避免了从头开始重新计算奇偶校验。 

循环遍历所有有效的剪切位置，停止于`n-1`因为两边都必须非空。 每次迭代都会在固定的字母大小上执行恒定的工作量。

 ## 边缘情况

 对于像这样的单字符字符串`a`，没有有效的分割位置，因此答案为零。 算法立即返回，不进入循环，符合不存在非空分区的事实。 

对于统一的字符串，例如`aaaaa`，每次分割都会产生两个字符串，每个字符串仅包含一个不同的字符，并且双方都满足回文排列条件。 在执行过程中，后缀奇数计数在每一步都保持为零，前缀在零和一之间切换，但永远不会超过允许的阈值，因此每个分割都会正确计数。 

对于交替模式，例如`abab`，奇偶校验每一步都会振荡。 该后缀早期会快速累积多个奇数计数，导致大多数拆分失败，并且只有最终配置同时满足这两个约束。
