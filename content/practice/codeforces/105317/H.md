 ## 解决方案
 ## 问题理解

 我们被要求构造一个长度的数组$n$，其中每个元素都位于$1$和$m$，对其子数组有全局限制。 该限制与顺序或独特性无关，而是与前缀和有关：不允许数组的连续段具有可被整除的和$m$。 

同样地，如果我们定义前缀和$s_i = a_1 + a_2 + \dots + a_i$，那么对于任意两个索引$i \le j$，我们必须避免$s_j \equiv s_{i-1} \pmod m$。 条件“没有子数组和可被整除”$m$” 与说没有两个前缀和具有相同的余数模完全相同$m$，除了我们隐含地包括$s_0 = 0$。 所以约束变成：所有前缀和模$m$必须是独特的。 

输入给出多个独立的测试用例，每个测试用例指定一对$(n, m)$，对于每种情况，我们要么构造一个有效的数组，要么报告这是不可能的。 

约束允许最多$10^5$测试用例总计$n$总结为$3 \cdot 10^5$。 这迫使任何解决方案在所有组合的测试用例上基本上以线性时间运行。 任何尝试显式检查子数组的操作都是立即不可行的，因为单个数组的大小$n$有$O(n^2)$子数组。 

微妙的故障模式来自于将条件与本地事物混淆。 例如，人们可能会尝试确保没有任何单个元素可以被$m$，或者仅在最后没有部分和为零。 两者都不够，因为违规可能发生在两个内部位置之间。 

例如，如果$m = 5$，数组$[2, 3, 5]$看起来元素方面无害，但是子数组$[2, 3]$总和为$5$，可除以$m$。 在这种情况下，忽略前缀交互的天真贪婪将会失败。 

另一个常见的错误是假设因为值的边界是$m$，我们总是有足够的灵活性来避免碰撞。 实际上，一旦前缀求和模$m$重复，构造就会永久失败，因为重复是不可逆的。 

## 方法

 暴力方法将尝试逐步构建数组，并在每个位置测试来自的所有候选值$1$到$m$，检查添加它是否会创建一个禁止的子数组。 为了检查有效性，我们需要跟踪所有先前的前缀和模$m$并保证不重复发生。 这可以通过前缀余数的哈希集来完成。 

然而，关键的困难在于，对于每个位置，我们可能会尝试达到$m$值，并且对于每次尝试，我们都会更新前缀并检查成员资格。 即使使用散列，这也会变成$O(nm)$每个测试用例在最坏的情况下，远远超出了限制$m$可以达到$10^6$。 

当通过前缀和模查看时，问题的结构变得更加清晰$m$。 我们本质上是试图建立一个序列$n+1$前缀残基$s_0, s_1, \dots, s_n$这样所有的模都不同$m$。 但只有$m$可能的残留物总数。 既然我们已经有了$s_0 = 0$，我们可以拥有的额外不同残基的最大数量是$m-1$。 这立即暗示了一个必要条件：$n \le m-1$。 

一旦我们认识到这个界限，构建就变得自然了。 我们只需要遍历不同的余数模$m$，从不重复。 最简单的方法是通过固定步长不断增加前缀和，通常$1$，并避免对模进行换行$m$太早了。 然而，使用步长$1$直接导致按顺序完全覆盖残基，从而给出干净的确定性结构。 

我们定义$a_i = 1$为所有人$i$，产生前缀和$1, 2, 3, \dots, n$。 这些都是不同的模数$m$只要$n < m$。 如果$n \ge m$，那么根据鸽巢原理，两个前缀和必须模全等$m$，因此不能存在有效的数组。 

因此，问题分解为简单的可行性检查，然后是恒定时间的构造。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 阅读$n$和$m$。 我们首先确定是否可以进行有效的构造。 由于前缀和模$m$必须全部是不同的并且只有$m$残留物，我们需要$n < m$。 如果不满足这个条件，则任何施工都不能满足要求。 
2.如果$n \ge m$，立即输出“NO”。 这是应用于前缀和模的鸽巢原理的直接结果$m$。 中没有值的排列$[1, m]$可以避免重复模状态。 
3.如果$n < m$，通过将每个元素设置为来构造数组$1$。 这会产生前缀和$s_i = i$, which are strictly increasing as integers.
 4. 自从$s_i = i$，所有前缀和都是不同的模$m$只要之间没有两个整数$0$和$n$相差数倍$m$。 因为$n < m$，这是不可能的。 
5. 输出“YES”，后跟构造的数组。 

### 为什么它有效

 关键的不变量是前缀和在每一步都严格增加 1，因此没有两个前缀和可以作为整数重合。 因为模相等意味着整数差是$m$，前缀和之间的所有差异在于$[1, n]$，无差异可达到$m$。 这保证了没有前缀和重复模数$m$，这相当于说没有子数组和可以被整除$m$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
out = []

for _ in range(t):
    n, m = map(int, input().split())
    
    if n >= m:
        out.append("NO")
        continue
    
    out.append("YES")
    out.append(" ".join(["1"] * n))

sys.stdout.write("\n".join(out))
```实现完全遵循算法的结构。 可行性检查`n >= m`是唯一的决策点，直接从前缀残基的鸽巢论证中得出。 该结构使用恒定的数组，这避免了任何模块化重复的风险。 

输出被缓冲以避免重复的 I/O 开销$10^5$测试用例。

 ## 边缘情况

 当$n = 1$，如果$m \ge 2$并输出一个单元素数组`[1]`。 The prefix sum is`1`，它是非零模数$m$，因此任何子数组都不能违反条件。 

什么时候$n = m$，算法拒绝。 例如，与$n = 3, m = 3$，任何数组都会产生 4 个前缀和（包括 0），但只存在 3 个残差类。 手动跟踪确认无论选择什么值，重复都是不可避免的。 

什么时候$m$相对于而言非常大$n$， 例如$(n, m) = (10^5, 10^6)$，该构造输出一长串。 前缀和从 1 到$10^5$，所有不同的模$m$，因此条件成立，因为在该范围内永远不会发生环绕。
