 ## 解决方案
 ## 问题理解

 我们得到了 2D 平面上的点的集合，我们希望将它们全部包含在一个矩形内，该矩形的边与坐标轴对齐。 在所有这些轴对齐的矩形中，我们被要求找到面积尽可能最小的一个。 

轴对齐矩形完全由四个边界确定：左 x 坐标、右 x 坐标、底部 y 坐标和顶部 y 坐标。 每个点都必须位于这些边界之间。 任务本质上是选择这四个值，以便覆盖所有点，并使所得区域（宽度乘以高度）最小化。 

约束允许最多 54321 个点，坐标大小最大为 10^9。 这意味着我们不能以任何组合方式考虑点对或点子集。 任何尝试检查许多候选矩形的解决方案都会变得二次或更糟，并立即失败。 我们需要一种能够在线性或近线性时间内处理所有点的方法。 

当只有一个点时，会出现一种微妙的边缘情况。 在这种情况下，封闭的矩形会折叠成一个点，因此宽度和高度都为零，并且面积也必须为零。 另一种情况是所有点都位于垂直或水平线上。 例如，如果所有 x 坐标都相等，则宽度为零，并且无论 y 分布如何，答案都为零，因为面积是宽度乘以高度。 如果严格假设宽度和高度为正值，这些情况通常会被错误处理。 

## 方法

 暴力方法会尝试通过选取两个点来定义左右边界以及两个点来定义底部和顶部边界来构造矩形。 对于每个这样的选择，我们都会检查所有点是否都位于内部。 这已经意味着从 n 中选择两个点作为 x 边界，两个点作为 y 边界，从而产生大约 O(n^4) 个候选矩形。 即使进行剪枝，检查遏制的成本也为 O(n)，因此对于 n 高达 50000 的情况，总复杂性远远超出了可行的范围。 

关键的观察是我们实际上没有选择最佳矩形边界的自由。 任何有效的封闭轴对齐矩形必须将其边精确放置在点集的极端坐标处。 如果左侧从最小 x 坐标向内移动，则会排除至少一个点。 如果向外移动，面积只会增加。 同样的逻辑适用于右、上、下边界。 这迫使最佳矩形完全由所有点中的最小和最大 x 值以及最小和最大 y 值定义。 

因此，问题简化为一次计算数据集的四个值，然后将宽度和高度相乘。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^5) | O(n^5) | O(1) | O(1) | 太慢了 |
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们在维护极值坐标的同时处理所有点。 

1. 初始化四个变量：min_x 为+无穷大，max_x 为-无穷大，min_y 为+无穷大，max_y 为-无穷大。 这些跟踪迄今为止看到的所有点的边界框。 
2. 对于每个点 (x, y)，使用 x 更新 min_x 和 max_x，并使用 y 更新 min_y 和 max_y。 这确保了在处理所有点之后，这些变量表示包含每个点的最小轴对齐间隔。 
3. 处理完所有点后，将宽度计算为 max_x - min_x，将高度计算为 max_y - min_y。 
4. 将宽度乘以高度来计算面积。 
5. 将结果输出为浮点数。 

我们可以将整个几何图形压缩为四个数字的唯一原因是轴对齐消除了超出 x 轴和 y 轴投影的点之间的任何相互作用。 

### 为什么它有效

矩形必须包含每个点，因此其 x 间隔必须包含集合中的每个 x 坐标，并且其 y 间隔必须包含每个 y 坐标。 包含一组数字的行上的最小可能间隔始终由其最小值和最大值给出。 与这些边界的任何偏差都会排除至少一个点或增加间隔长度，因此无法改善面积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    min_x = float('inf')
    max_x = float('-inf')
    min_y = float('inf')
    max_y = float('-inf')

    for _ in range(n):
        x, y = map(float, input().split())
        min_x = min(min_x, x)
        max_x = max(max_x, x)
        min_y = min(min_y, y)
        max_y = max(max_y, y)

    width = max_x - min_x
    height = max_y - min_y
    print(width * height)

if __name__ == "__main__":
    solve()
```该实现完全依赖于维护运行最小值和最大值。 唯一的微妙之处是将坐标读取为浮点数，因为问题陈述和样本包含十进制值。 使用 float 就足够了，因为所需的精度是 1e-6。 

一个潜在的陷阱是错误地初始化极值。 如果所有坐标均为负值或全部为正值但不跨越零，则使用 0 将失败。 使用无穷大可以避免任何偏差。 

另一个微妙的点是我们永远不需要存储点本身，只需要存储它们的投影。 这就是保持内存使用恒定的原因。

 ## 边缘情况

 单点输入演示了退化几何情况。 有输入：```
1
5 5
```该算法将极值初始化为该点本身，导致 min_x = max_x = 5 和 min_y = max_y = 5。宽度和高度均为零，从而正确生成面积为零。 

垂直线的情况例如：```
3
2 1
2 5
2 10
```在整个处理过程中保持 min_x 和 max_x 均等于 2。 即使 y 极值扩展，宽度仍保持为零。 最终面积为零，因为任何轴对齐的矩形都会退化为 x 方向上的线段。 

水平线表现对称。 为了：```
3
1 3
4 3
10 3
```y 极值是相同的，高度为零，因此面积也为零。 

这些案例证实，该算法不假设正维度，并且纯粹通过极值算术正确处理简并性。
