 ## 解决方案
 ## 问题理解

 我们得到一个非常小的网格，最多 10 x 10，其中每个单元格都包含一个小写字母。 我们可以从任何单元格移动到其 8 个邻居中的任何一个，包括对角线，并且可以任意多次重新访问单元格。 对移动的唯一限制是一步必须实际改变单元格，因此不允许停留在原地。 

每个查询都会给出一个短字符串，我们需要确定网格上是否存在行走，以便访问的单元格字母序列准确地拼写该字符串。 

一个关键细节是同一个单元可以重复使用任意多次，因此该结构不是具有访问约束的简单路径问题。 相反，这是关于网格是否包含与序列匹配的标记步行。 

这些约束极大地影响了解决方案。 网格最多有 100 个单元格，而每个查询字符串的长度最多为 13。最多有 100000 个查询。 这种组合意味着我们可以在网格上负担相当重的每个查询动态程序，只要每个查询保持接近大约几千个操作。 任何涉及在没有记忆的情况下对路径进行指数搜索的操作都会立即失败，因为在最坏的情况下，在 8 个方向上分支 13 个步骤会产生最多 8^13 种可能性。 

一个微妙的一点是，单元格的重用消除了通常的“简单路径”结构。 不记忆状态的简单深度优先搜索可以多次重新访问相同的配置，特别是在所有字母都匹配的统一网格上。 

第二个微妙之处是每个查询都是独立的。 我们无法跨查询携带状态，因为字符串不同并且有效性取决于精确的字符序列对齐。 

## 方法

 直接解释建议从与模式的第一个字符匹配的每个单元格开始，然后执行深度优先搜索，在每一步尝试所有 8 个移动，同时匹配下一个字符。 这是正确的，因为它明确枚举了所有可能的行走。 

然而，这种方法多次重复相同的子问题。 从字符串中的固定单元格和固定索引开始，剩下的问题始终是相同的：我们可以从这里继续匹配字符串的后缀吗？ 如果没有记忆，递归会探索指数级的许多冗余路径，特别是因为允许重新访问单元格并且循环会产生无限的重新计算。 

问题的结构允许更清晰的重新表述。 对于每个查询字符串，将状态定义为由网格位置和字符串中的索引组成的对。 从一个状态开始，转换会转到字母与下一个字符匹配的任何相邻单元格。 由于字符串长度最多为 13，网格最多有 100 个位置，因此每个查询的状态总数最多为 1300 个。每个状态最多有 8 个转换，因此在该状态空间中，记忆化的完整探索是线性的。 

关键的观察是，一旦我们记住状态，网格中的循环就不再重要。 即使步行重新访问单元格，我们也不需要重新计算该状态下的后缀是否可能。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 无需记忆的强力 DFS | 每次查询 O(8^L) | O(L) 递归 | 太慢了 |
 | 每个查询的（单元格、索引）状态上的 DP | 每次查询 O(nm·L·8) | 每次查询 O(nm·L) | 已接受 |

 ## 算法演练

 我们使用网格上的动态编程或记忆 DFS 公式独立处理每个查询。 

1. 我们将每个单元格视为该模式的潜在起点。 仅当起始单元格的字母与字符串的第一个字符匹配时，起始单元格才有效。 此限制立即消除了不必要的工作，因为不匹配的开始永远不会导致有效的匹配。 
2. 我们定义一个状态函数`(r, c, i)`意味着我们目前在牢房`(r, c)`我们已经匹配了前缀`s[0..i]`，所以下一个要匹配的字符是`s[i+1]`。 目标是确定是否可以达到索引`len(s) - 1`。 
3.来自一个州`(r, c, i)`， 如果`i`已经是字符串的最后一个索引，我们已经成功匹配整个模式并返回成功。 
4. 否则，我们尝试移动到所有 8 个相邻单元格`(nr, nc)`。 仅当字符位于`(nr, nc)`等于`s[i+1]`。 对于每个有效邻居，我们递归地评估状态`(nr, nc, i+1)`。 
5. 为了避免多次重新计算相同的状态，我们将结果存储在索引为的备忘录表中`(r, c, i)`。 一旦计算出状态，每当再次遇到时就会重用它。 
6. 如果任何起始位置导致字符串成功完成，则查询的答案为“YES”，否则为“NO”。 

关键的实现细节是记忆是针对每个查询的，而不是全局的。 每个字符串定义不同的状态空间，因此跨查询重用会混合不相关的计算。 

### 为什么它有效

 正确性来自于这样一个事实：网格中的每个有效行走完全对应于状态图中定义的一系列转换`(cell, index)`。 网格中的每个可能的移动都在转换步骤中表示，并且每个有效的字符序列必须对应于该状态图中的某个路径。 

记忆化不会删除任何有效路径，因为它只消除了相同状态的重新计算，而不是转换。 由于每个状态都完全捕获了继续所需的所有信息，因此重新计算不能改变结果。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

dx = [-1,-1,-1,0,0,1,1,1]
dy = [-1,0,1,-1,1,-1,0,1]

def solve_one(grid, s, n, m):
    L = len(s)
    memo = [[[-1] * L for _ in range(m)] for _ in range(n)]

    def dfs(x, y, i):
        if memo[x][y][i] != -1:
            return memo[x][y][i]

        if i == L - 1:
            memo[x][y][i] = 1
            return 1

        nx_char = s[i + 1]
        for k in range(8):
            nx = x + dx[k]
            ny = y + dy[k]
            if 0 <= nx < n and 0 <= ny < m:
                if grid[nx][ny] == nx_char:
                    if dfs(nx, ny, i + 1):
                        memo[x][y][i] = 1
                        return 1

        memo[x][y][i] = 0
        return 0

    for i in range(n):
        for j in range(m):
            if grid[i][j] == s[0]:
                if dfs(i, j, 0):
                    return True
    return False

def main():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]
    q = int(input())

    for _ in range(q):
        s = input().strip()
        print("YES" if solve_one(grid, s, n, m) else "NO")

if __name__ == "__main__":
    main()
```网格遍历是通过8方向偏移数组实现的，这使得每步的邻居生成时间恒定。 备忘录表是三维的，由行、列和字符串中的位置作为键。 每个查询都会重建此表，因为以前的结果不相关。 

DFS 立即通过字符相等性过滤转换，这是在实践中保持解决方案快速的主要修剪因素。 如果没有这种检查，无论可行性如何，都将探索每个邻居，每一步都会将工作量增加八倍。 

## 工作示例

 考虑网格：```
ab
ju
```和查询字符串`auabj`。 

我们追踪 DP 如何从细胞匹配开始探索可能的状态`'a'`。 

| 步骤| 职位| 索引 | 下一个字符 | 行动|
 | ---| ---| ---| ---| ---|
 | 1 | (0,0) | (0,0) | 0 | 你| 尝试邻居|
 | 2 | (0,1)| 1 | 一个 | 向右移动|
 | 3 | (1,1) | 2 | 乙| 对角移动/有效邻居 |
 | 4 | (0,0) | (0,0) | 3 | j | 允许重访 |

 该跟踪表明，重新访问单元格是自然处理的，无需特殊标记，因为状态标识包括字符串中的索引。 

现在考虑一个失败的模式，其中任何步骤都无法匹配字符。 假设下一个所需的字符不存在于给定深度的任何可达状态的任何邻居中。 记忆确保一旦某个状态的所有路径都失败，就不会再次重新考虑该状态，从而防止重复的死胡同探索。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(q·n·m·L·8) | 每个查询最多探索所有`(cell, index)`状态一次，每个状态最多有 8 个转换 |
 | 空间| O(n·m·L) | O(n·m·L) | 每个查询的备忘录表存储所有状态三元组的结果 |

 和`n, m ≤ 10`和`L ≤ 13`，每个查询的成本约为几千次操作，即使对于`q = 100000`在优化实现中，尤其是在编译语言中。

 ## 边缘情况

 充满相同字符的网格强调重用和循环行为。 对于像这样的图案`"aaaaaaaaaaaaa"`，每个单元格都是有效的开始，并且每一步都保持有效。 该算法仍然正确终止，因为备忘录表确保每个`(cell, index)`即使概念行走的数量巨大，对也只评估一次。 

单细胞网格演示了边界处理。 如果模式长度大于 1，则不可能进行任何转换，因为移动总是会更改单元格，因此唯一有效的答案是与单个字符匹配的长度为 1 的字符串。 

第一个字符不存在于网格中的模式将在 DFS 开始之前立即过滤。 这可以防止不必要的状态创建，并确保算法在这种情况下根本不会进入递归。
