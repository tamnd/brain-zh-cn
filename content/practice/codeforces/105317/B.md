 ## 解决方案
 ## 问题理解

 我们在字符串中得到了一组非常小的不同字符$T$，和一个更大的字符串$S$包含与以下完全相同的多组字符$T$，只是重复数量。 我们可以自由地排列两个字符串。 

关键的概念是当一对$(A, B)$被认为是兼容的。 细绳$B$就像一个骨架：每个角色$B$可以拉伸成一个重复副本的块，但是不同字符的顺序$B$必须保存。 换句话说，如果$B = abc$，然后任意字符串$aaabcc$或者$aabbcccc$是可以接受的，但是类似的东西$acb$不能重新排列成有效的扩展$abc$因为块的相对顺序被违反了。 

给定两个排列$S_0$和$T_0$，我们将成本衡量为转换所需的最小相邻交换数量$S_0$到一些有效的扩展$T_0$。 由于相邻交换测量反转距离，因此该成本本质上是当前排列和最佳可能的块结构排列之间的反转计数。 

任务是对抗性的：我们选择两种排列$S_0$和$T_0$来最大化这个成本。 

限制因素很重要。 大字符串最多可以有$3 \cdot 10^5$字符，但涉及的字母$T$大小最多为 5。这立即表明任何依赖于字符排列的解决方案都是可行的，但任何二次方$|S|$仅当是线性扫描而不是成对模拟时才可以接受。 

一个常见的失败案例来自于忽略了只有字符组的相对顺序才重要。 例如，如果我们处理单个字符而不是聚合计数，我们可能会尝试模拟整个字符串的交换，这是不必要的，而且速度太慢$3 \cdot 10^5$。 

另一个微妙的问题是假设一旦我们修复了$T_0$，结构$S_0$是任意的。 事实上，最优的安排是$S_0$对于固定顺序$T_0$完全确定：字符的所有出现都应该分组在一起，否则我们会失去潜在的反转贡献。 

最后一个边缘情况是$|T| = 1$。 任何排列都会产生零成本，但假设至少两个字符的粗心实现可能会尝试无效的对逻辑。 

## 方法

 蛮力观点将从选择所有排列开始$T_0$。 对于每个这样的顺序，我们将尝试所有排列$S_0$，计算要转换的最小掉期$S_0$转化为有效展开式，并取最大结果。 这一下子就炸了：$S_0$有$3 \cdot 10^5$位置，因此即使表示排列也是不可行的，并且重复计算反转距离会花费$O(n \log n)$每次尝试，乘以巨大的搜索空间。 

关键的观察是我们实际上不需要考虑排列$S_0$根本不。 一旦字符的顺序$T_0$是固定的，最大化反转距离的最佳方法是将所有出现的最后一个字符放在前面，然后是前一个字符，依此类推。 这会创建一个配置，其中每对不同的字符都会尽可能反转，从而最大化交换。 

这将整个问题简化为对最多五个字符的排序的小型组合优化。 对于任何固定顺序，可以仅使用频率来计算成本：每对字符根据其相对顺序贡献其计数的乘积。 

因此，任务变成选择使加权求逆和最大化的字符排列。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解字符串 | 不可行 | 不可行 | 太慢了 |
 | 仅排列字符 |$O(k! \cdot k^2)$|$O(k)$| 已接受 |

 这里$k \le 5$，所以枚举是微不足道的。 

## 算法演练

 1.统计每个字符出现的频率$S$。 仅存在于中的字符$T$很重要，他们的计数充分描述了问题。 
2. 从中提取不同字符的集合$T$。 这最多给出了我们需要排列的五个符号。 
3. 枚举这些字符的每个排列。 每个排列代表一个候选顺序$T_0$。 
4. 对于每个排列，计算所有有序对的贡献。 如果一个字符$a$出现在之前$b$在排列中，则所有出现的$a$放置在之后$b$在$S_0$贡献反转。 因为我们稍后会安排$S_0$最佳情况下，贡献变成它们频率的乘积。 
5. 选择产生最大总成对贡献的排列。 
6. 构建$T_0$作为这个最优排列。 
7. 构建$S_0$通过以相反的顺序放置字符$T_0$，每个根据其频率重复$S$。 

之所以如此有效，是因为两个块之间的反转成本仅取决于它们的相对顺序和计数。 一旦我们修复了块的顺序$S_0$，每对不同的角色要么完全贡献，要么根本不贡献，分组确保块内不会浪费任何贡献。 因此，整个优化减少为在最多五个节点上排序加权完整图。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from itertools import permutations

def solve():
    T = input().strip()
    S = input().strip()

    freq = {}
    for ch in S:
        freq[ch] = freq.get(ch, 0) + 1

    chars = list(dict.fromkeys(T))  # preserve order but unique

    best_cost = -1
    best_order = None

    for p in permutations(chars):
        pos = {c: i for i, c in enumerate(p)}
        cost = 0
        for i in range(len(p)):
            for j in range(i + 1, len(p)):
                cost += freq[p[i]] * freq[p[j]]
        if cost > best_cost:
            best_cost = cost
            best_order = p

    T0 = ''.join(best_order)

    S0_list = []
    for ch in reversed(best_order):
        S0_list.append(ch * freq[ch])
    S0 = ''.join(S0_list)

    print(best_cost)
    print(S0)
    print(T0)

if __name__ == "__main__":
    solve()

的建设$S_0$相反的顺序可以保证最大的反转贡献。 如果我们交错字符而不是对它们进行分组，则字符串内的交换不会转化为额外的反转增益。 

## 工作示例

 考虑一个情况$T = "abc"$和$S$包含每个字符的多个副本，例如两个 a、一个 b 和三个 c。 

尝试像这样的排列$abc$将成本分配为$2 \cdot 1 + 2 \cdot 3 + 1 \cdot 3$，取决于订购。 如果我们改为使用$cba$，结构$S_0$变成$c c c b a a$，并且每个高频字符都被放置在所有其他字符之前，从而最大化反转累积。 

| 步骤| 排列| 结对贡献 | 总计 |
 | ---| ---| ---| ---|
 | 1 | ABC | a-b、a-c、b-c | 计算总和 |
 | 2 | 篮球协会| c-b、c-a、b-a | 较大金额 |

 这表明，当与最佳分组相结合时，反转高频结构往往会增加反转潜力。 

另一种情况是单个字符，例如$T = "x"$，仅产生一种排列。 成本始终为零，因为没有要反转的对，并且输出字符串是任意重复$x$。

 ## 边缘情况

 当$T$仅包含一个字符，排列循环退化为单个候选字符。 所构建的$S_0$变成单个块，并且没有跨字符反转可计数，因此成本保持为零。 

当频率高度倾斜时，例如一个字符占主导地位$S$，最佳排序仍然将该字符置于排列的一个极端。 该算法自然地处理了这个问题，因为成对乘积非常倾向于将高频字符放置在一侧。 

当所有字符具有相同的频率时，多个排列可能会并列。 该算法仍然返回有效的最大值，因为在这种情况下所有排列都会产生相同的总和，并且$S_0$通过颠倒所选顺序来保持一致。
