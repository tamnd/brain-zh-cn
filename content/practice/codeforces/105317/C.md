 ## 解决方案
 ## 问题理解

 我们正在使用一个表示一行表格的二进制字符串，其中每个位置要么翻转，要么不翻转。 字符串仅通过直接点更新进行更改：类型 1 的查询将单个位置从 0 切换到 1 或从 1 切换到 0。 

独立于这些更新，我们还考虑不修改字符串的随机过程。 在一个“实验”中，我们重复执行以下操作：在所有子数组中均匀随机选择一个段，并翻转该段中的每一位。 我们重复此操作 k 次，每次都再次独立选择段。 对于类型 2 的查询，我们需要输入此随机过程对当前字符串应用 k 次后的预期 1 数量。 

一个关键的微妙之处在于随机过程始终是概念性的。 不影响以后的查询。 类型 2 的每个查询都会从当前字符串快照开始评估期望。 

字符串大小的约束很大，最多 100000，查询数量也最多 100000，但 k 很小，最多 200。这立即表明任何解决方案都必须避免显式模拟字符串演化或随机翻转过程。 即使对一个实验进行一次模拟也是不可能的，因为分段翻转的时间复杂度为 O(n)，并且 k 足够大，以至于重复模拟会太慢。 

输出精度要求意味着我们必须使用浮点算术或仔细推导的有理公式而不是蒙特卡罗模拟来计算精确的期望。 

一个天真的但诱人的想法是独立模拟每个位置的概率，但这失败了，因为段翻转引入了跨位置的强相关性。 例如，翻转线段 [l, r] 会同时翻转 i 和 j（如果它们位于内部），因此它们的状态不是独立的。 

第二个微妙的陷阱是假设步骤之间呈线性，而不考虑重复的合成。 一次手术后，期待很容易。 在 k 次操作之后，状态通过重复应用线性变换而演变，并且每个查询每一步的简单重新计算会导致 O(nkQ)，这太慢了。 

## 方法

 我们首先考虑单个随机操作中会发生什么。 固定位置i。 当且仅当随机选择的段覆盖 i 时，它才会被翻转。 包含 i 的段数为 i × (n − i + 1)，段总数为 n(n + 1)/2。 所以位置 i 在一次操作中被翻转的概率是一个简单的有理值。 

让 p[i] 表示这个概率。 如果某位为 0 或 1，则在一次随机操作后，其期望值将发生如下变化。 翻转会切换该位，因此如果 X 是当前值，则在一次操作后，它会以 1 − p[i] 的概率变为 X，以 p[i] 的概率变为 1 − X。 取期望给出每个位置独立期望的线性递推。 

这已经揭示了关键结构：每个位置在期望中独立演化，因为线性翻转的期望仅取决于其自身当前的期望，而不取决于邻居。 当取期望时，相关性消失。 

因此，如果 E[i] 是 t 次操作后位置 i 的期望值，我们会得到 E[i] ← (1 − p[i])E[i] + p[i](1 − E[i]) 形式的递推。 这简化为 E[i] ← E[i](1 − 2p[i]) + p[i]。 

这是重复应用的线性仿射变换。 对于每个 i，这是一个一维线性动力系统。 经过 k 步后，我们应用相同的变换 k 次。 这可以明确地解决为几何级数。 

令 a[i] = 1 − 2p[i] 且 b[i] = p[i]。 那么k步后的E[i]为：

 当 a[i] ≠ 1 时，E[i] = a[i]^k * 初始 [i] + b[i] * (1 − a[i]^k) / (1 − a[i])。 

这将问题简化为计算数字的 k 次方并对 i 求和。

然而，我们仍然需要最多 100000 个查询的答案，每个查询可能具有不同的 k，以及更改 initial[i] 的更新。 因此每个查询重新计算能力太慢。 

关键的观察结果是 k ≤ 200。这允许我们预先计算所有 i 和所有 t 直到 200 的幂 a[i]^t。每次更新仅更改一个位置，因此我们可以有效地保持其贡献。 最终答案是所有头寸的总和，因此我们维护期望值的全局总和。 

因此，每个查询类型 2 变为 O(1)，并且更新调整 O(k) 预先计算的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟翻转| O(Q·k·n) | O(n) | 太慢了 |
 | 每个位置的封闭式重新计算 | O(Q · n) | O(n) | 太慢了 |
 | 预计算高达 k ≤ 200 的转换功率 | O(n · 200 + Q · 200) | O(n · 200) | 已接受 |

 ## 算法演练

 1. 计算n并预先计算段总数，即n(n + 1)/2。 这是确定每个位置的翻转概率所必需的。 
2. 对于每个位置 i，计算 p[i] = i(n − i + 1) /total_segments。 这是在单个随机段操作中 i 被翻转的概率。 
3. 将每个位置转换为线性变换 E ← a[i]E + b[i]，其中 a[i] = 1 − 2p[i] 且 b[i] = p[i]。 这隔离了重复随机翻转对期望的影响。 
4. 预先计算从 0 到 200 的所有 t 的幂 a[i]^t。这允许快速评估重复变换，而无需为每个查询重新计算幂。 
5. 对于每个查询，维护从初始字符串初始化的当前预期贡献数组。 总的预期答案是所有 E[i] 的总和。 
6. 对于类型 1 查询，翻转 S[i]。 位置 i 的期望值变为 1 − 当前值，因此通过减去旧的贡献并加上新的贡献来更新运行总计。 这使得全局总和在 O(1) 内保持一致。 
7. 对于带有参数 k 的类型 2 查询，使用带有指数 k 的预先计算公式计算每个位置的最终预期贡献，并对它们求和。 

其核心思想是每个位置在同一个线性算子下独立演化，因此整个系统是独立的一维过程的总和。 

它起作用的原因来自于期望的线性和每个操作的段选择的独立性。 每个位置的期望值仅取决于随机段是否覆盖它，并且该事件是特定于位置的。 尽管翻转在单个操作中跨位置相关，但期望可以完全分解，因为总和的期望等于期望的总和，并且每个期望仅取决于每步的单个伯努利事件。 重复应用构成线性递推，因此闭式求幂是有效的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = list(input().strip())
    n = len(s)

    total = n * (n + 1) / 2

    p = [0.0] * n
    a = [0.0] * n
    b = [0.0] * n

    for i in range(n):
        l = i + 1
        r = n - i
        p[i] = (l * r) / total
        a[i] = 1.0 - 2.0 * p[i]
        b[i] = p[i]

    E = [1.0 if c == '1' else 0.0 for c in s]

    kmax = 200
    pow_a = [[1.0] * (kmax + 1) for _ in range(n)]
    for i in range(n):
        for k in range(1, kmax + 1):
            pow_a[i][k] = pow_a[i][k - 1] * a[i]

    # precompute geometric sums
    geom = [[0.0] * (kmax + 1) for _ in range(n)]
    for i in range(n):
        for k in range(1, kmax + 1):
            geom[i][k] = geom[i][k - 1] * a[i] + 1.0

    Q = int(input())
    for _ in range(Q):
        t = input().split()
        if t[0] == '1':
            i = int(t[1]) - 1
            E[i] = 1.0 - E[i]
        else:
            k = int(t[1])
            ans = 0.0
            for i in range(n):
                ak = pow_a[i][k]
                if abs(1.0 - a[i]) < 1e-12:
                    ans += E[i]
                else:
                    ans += ak * E[i] + b[i] * (1.0 - ak) / (1.0 - a[i])
            print(f"{ans:.6f}")

if __name__ == "__main__":
    solve()
```该实现预先计算每个位置的参数 p、a 和 b。 数组 E 存储每个位置的当前确定性值，因为二元状态的期望与其在此模型下为 1 的概率相匹配。 

幂表 pow_a[i][k] 允许对 a[i]^k 快速求幂，而无需重新计算。 在查询类型 2 期间，我们直接应用封闭式表达式。 尽管实际上 a[i] 仅在简并边界情况下才等于 1，但对 a[i] 接近 1 的特殊情况检查可避免除法不稳定。 

一个微妙的点是我们不模拟随机过程的中间步骤。 查询直接将 k 步变换应用于当前状态。 

## 工作示例

 考虑一个小字符串 S =“10”。 然后 n = 2。每个位置的段概率不同：位置 1 包含在段 [1,1]、[1,2]、[2,2] 中，因此 p[1] = 2/3。 位置 2 也是对称的，p[2] = 2/3。 

没有更新且 k = 1 的查询序列产生：

 | 我| 初始 E[i] | p[i] | p[i] 一个[我] | 1 步后 E |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 2/3 | -1/3 | 1/3 | 1/3
 | 2 | 0 | 2/3 | -1/3 | 2/3 |

 总和为 1。 

现在考虑从相同状态开始 k = 2。 每个位置都是通过重复应用相同的线性图而演变的。 位置 1 从 1 开始并向其固定点 p[i] = 2/3 移动，因此其值从 1 向 2/3 减小。 位置 2 从 0 增加到 2/3。 总和变得接近4/3。 

| 我| E[0] | E[1] | E[2] |
 | ---| ---| ---| ---|
 | 1 | 1 | 1/3 | 1/3 5/9 |
 | 2 | 0 | 2/3 | 4/9 |

 总和为 1。 

该迹线显示了朝向固定值 p[i] 的收缩行为，这证实了重复应用在每个位置上独立收敛。

 ## 边缘情况

 当 n = 1 时，会出现极端情况。唯一的段是 [1,1]，因此每个操作都会确定性地翻转单个位。 概率 p[1] 等于 1，因此 a[i] 变为 -1。 递归恰好在 0 和 1 之间交替，并且 k 步期望取决于 k 的奇偶性。 该算法可以处理这个问题，因为 a[i]^k 正确地变为 (-1)^k 并且闭合形式减少为交替值。 

如果在推理中隐式解释（即使没有查询），当 k = 0 时，会出现另一种边缘情况。 正确的行为是返回当前的数量而不进行任何转换。 由于 a[i]^0 = 1，该公式退化为恒等式。 

最后一个微妙的情况是长时间的更新反复影响同一位置。 由于每次更新都会翻转 E[i]，因此在应用任何查询之前，重复的切换必须准确反映在存储的状态中，否则期望会发生漂移。
