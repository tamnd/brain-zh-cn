 ## 解决方案
 ## 问题理解

 该级别被建模为无限数轴。 角色从位置 0 开始，想要到达位置 n。 在每一时刻，角色可以向左或向右移动一个单位，每单位步支付一秒的固定成本，或者执行类似传送的跳跃，向左或向右精确移动 d 个单位，每次跳跃支付 b 秒。 路径不限于保持在 0 和 n 之间，因此如果有助于减少总时间，则允许超调或变为负数。 

任务是计算精确结束于位置 n 所需的最短总时间。 

约束足够小，n 最多为 1000，并且所有成本和跳跃长度也以 1000 为界。这立即表明整数状态上的最短路径是可行的。 由于状态空间很小，因此对所有可能的位置和转换进行简单的动态编程已经足够快了。 

一个微妙的问题是跳跃可能会过度，然后通过行走来纠正。 这就造成了最佳策略首先故意偏离目标的情况。 例如，如果步行成本高昂，但跳跃成本低且规模大，则最好采用负数或超过 n 来调整跳跃奇偶性或减少所需的小步数。 

当跳跃距离 d 大于 n 时，会出现另一种边缘情况。 在这种情况下，每次跳跃都会超出目标，因此最佳解决方案可能包括几次跳跃，然后返回。 天真的贪婪的“如果靠近就总是向前跳跃”的方法可能会失败，因为当一系列跳跃比混合的单个步骤更好地对齐时，离开可能会降低总成本。 

## 方法

 强力解释将每个整数位置视为图中的一个节点，并将每个移动或跳跃视为一条边。 从每个位置 x，我们可以以成本 a 到达 x+1、x−1，并以成本 b 到达 x+d、x−d。 在该图上运行 Dijkstra 算法将找到最短路径。 

这是正确的，因为所有移动都形成加权边缘，并且我们正在最小化总成本。 然而，如果我们天真地允许位置从负无穷大到正无穷大，那么图形将是无限的。 只有在注意到最佳路径永远不需要远远超出 [0, n] 周围的有界窗口之后，暴力方法才变得有意义。 由于 n ≤ 1000 并且每次移动都会改变位置至少 1，因此任何最佳路径都可以限制在 [-2000, 3000] 之类的范围内，而不会失去最佳性。 这给出了大约几千个节点的有限图，使 Dijkstra 可行，但仍然比必要的重。 

关键的见解是状态空间足够小，即使是简单的最短路径计算，甚至是所有可达位置上的直接 DP 也能有效地工作。 每个位置仅取决于恒定数量的邻居，因此我们可以运行复杂度为 O(n) 或 O(n log n) 的标准最短路径算法，并且仍然在限制范围内。 由于除了成本之外，该图在结构上是未加权的，因此具有优先级队列的 BFS 变体是简单且稳健的。 

另一个简化是，由于所有转换都有固定成本，这本质上是线图上的最短路径，每个节点的边数恒定，因此除了标准 Dijkstra 之外不需要高级优化。

 ## 算法演练

 我们将每个整数位置视为一个节点，表示到达该位置所需的最短时间。

1. 我们首先定义一个合理的持仓界限。 由于目标是 n 并且每次移动最多改变位置 d 或 1，因此我们将自己限制在安全包含所有有用状态的范围内，通常从 -2n 到 2n。 这确保了任何有益的绕行仍然是可代表的。 
2. 我们初始化一个无穷大的距离数组，并设置 dist[0] = 0，因为我们从位置 0 开始，没有任何成本。 
3. 我们使用优先级队列运行 Dijkstra 算法，始终以最小的已知成本扩展状态。 这确保我们始终按照最优顺序处理状态。 
4. 从每个位置 x 开始，我们放松最多四个转换：成本为 a 的 x+1、成本为 a 的 x−1、成本为 b 的 x+d 和成本为 b 的 x−d。 如果找到更便宜的路线，则每次放宽都会更新目的地位置的已知成本。 
5. 我们继续，直到处理完所有可达状态。 
6. 答案是dist[n]，到达目标位置的最小成本。 

其工作原理是该过程精确地模拟了加权图上的最短路径计算。 每个位置都是一个状态，每次移动都是具有非负成本的边，Dijkstra 保证非负权重下的最优性。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n, a, b, d = map(int, input().split())

    # Reasonable bound: enough to cover detours
    LIM = 2 * n + d + 10

    INF = 10**18
    offset = LIM
    size = 2 * LIM + 1

    dist = [INF] * size
    start = offset
    dist[start] = 0

    pq = [(0, start)]

    def relax(nx, nd):
        if 0 <= nx < size and nd < dist[nx]:
            dist[nx] = nd
            heapq.heappush(pq, (nd, nx))

    while pq:
        cur, x = heapq.heappop(pq)
        if cur != dist[x]:
            continue

        pos = x - offset

        # move +1
        relax(x + 1, cur + a)
        # move -1
        relax(x - 1, cur + a)
        # jump +d
        relax(x + d, cur + b)
        # jump -d
        relax(x - d, cur + b)

    print(dist[offset + n])

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现在有界整数段上构建隐式图。 偏移技巧用于通过将所有内容移动到非负数组索引空间来处理负位置。 优先级队列确保我们始终首先扩展最便宜的可达位置。 

一个常见的错误是低估所需的界限。 如果范围太窄，暂时向左或向右移动的有效最佳路径将被切断。 所选择的大约 2n + d 的限制是安全的，因为任何有用的绕行都不能要求超出该范围的重复发散，而不需要支付将主导解决方案的不必要的步行成本。 

另一个微妙的点是检查`cur != dist[x]`，这避免了处理优先级队列中的陈旧条目并使算法保持在时间限制内。

 ## 边缘情况

 当 n 为零时，算法立即返回零，因为起始状态已经与目标匹配，并且不需要转换。 

当跳跃距离大于目标时，最短路径计算自然会避免过度跳跃，因为重复的超调会积累不必要的成本。 Dijkstra 过程仍然考虑这些状态，但很快发现步行或有限的跳跃占主导地位。 

当步行比跳跃便宜得多时，松弛步骤总是倾向于 ±1 转换，并且算法的行为就像折线图上的标准最短路径，而不依赖于跳跃边缘。
