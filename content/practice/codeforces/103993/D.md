 ## 解决方案
 ## 问题理解

 我们正在处理 6 位数字的密码，其中每个密码都是从 0 到 9 的数字序列。密码的结构受到高度限制：它必须使用两个不同的数字，并且每个数字必须恰好出现 3 次，因此密码始终是 AAA BBB 之类的排列，其中 A 和 B 是不同的数字。 

除了这个结构规则之外，我们还得到了一组禁止的数字。 如果任何候选密码使用这些数字之一，则该密码无效。 

因此，该任务纯粹是组合性的：通过选择两个允许的数字并排列它们以使每个数字恰好出现三次，计算可以形成多少个长度为 6 的序列。 

输入由一小部分禁止数字组成。 未列出的一切都是允许的。 由于数字范围仅从 0 到 9，因此有效宇宙非常小，这已经表明对数字对进行暴力破解就足够了。 

这些限制意味着搜索空间非常小。 最多存在 10 位数字，我们正在选择允许的数字对。 这最多给出 C(10, 2) = 45 个候选对，这是很小的。 对于每一对来说，计数排列是一项持续的工作。 这立即排除了任何高级优化或预处理的需要。 

边缘情况主要是关于禁止的数字如何缩小可用的集合。 

一个微妙的情况是可用数字少于两位。 例如，如果 9 位数字被禁止，只剩下一位数字，那么就不可能形成有效的密码，因为我们需要两个不同的数字。 另一个极端情况是当正好剩下两个数字时，这会产生一个有效的数字对，但会产生许多排列。 

贪婪思维失败的例子：如果允许的数字是 {1, 2}，人们可能会错误地认为只有一个密码“111222”，但答案实际上是这个多重集的不同排列的数量，即 6!/(3!3!) = 20。缺少这个组合因子是最常见的错误。 

## 方法

 暴力破解的想法是生成所有长度为 6 的数字 0 到 9 的序列，并检查每个序列是否满足约束：恰好两个不同的数字，每个数字出现 3 次，并且没有禁止的数字。 这样的序列有 10^6 个。 每次检查的时间复杂度为 O(6)，因此这种方法每次测试需要大约 600 万次操作，这已经是临界点，但在某些语言中仍然可以接受，尽管没有必要。 

关键的观察是，有效密码的结构完全是通过选择这两个数字来确定的。 一旦数字 A 和 B 固定，有效序列的数量就纯粹是组合的：我们将 3 个 A 副本和 3 个 B 副本排列在长度为 6 的序列中。 这样排列的个数就是多项式系数6！ / (3!3!) = 20，与选择的数字无关。 

这将问题简化为计算存在多少对允许的数字。 如果允许 k 个数字，我们选择任何无序对 (A, B)，给出 C(k, 2) 种可能性，并且每个都提供恰好 20 个密码。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有序列进行暴力破解 | O(10^6) | O(10^6) | O(1) | O(1) | 太慢/不必要|
 | 选择数字对+组合 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 读出一组禁止数字并标记它们。 此步骤定义了哪些数字在构造中仍然可用。 
2. 通过扫描 0 到 9 并收集那些未标记为禁止的数字来构建允许的数字列表。 这隔离了有效字母表。 
3. 计算允许的位数，称其为k。 该值确定是否可以存在任何有效密码。 
4. 如果 k 小于 2，则立即返回 0，因为我们无法选择两个不同的数字。 这是结构上的不可能。 
5. 否则，计算从 k 中选取两个不同数字的方法数，即 k * (k - 1) / 2。每个这样的对对应于密码中使用的固定多组数字。 
6. 将对的数量乘以 20，因为对于任何选定的对（A，B），具有三个 A 和三个 B 的有效 6 长度排列的数量是 6！ / (3!3!) = 20。 

### 为什么它有效

 每个有效的密码都必须使用两个不同的数字，因此它会产生唯一的无序数字对。 相反，任何无序的允许数字对都唯一地定义了一系列有效密码，其中包含每个数字的三个副本的多重集的所有排列。 这些族是不相交的，因为不同的数字对不能产生相同的序列。 由于每个家族都有相同的固定规模，因此总数就是对的数量乘以一个家族的规模。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    forbidden = set(map(int, input().split())) if n else set()

    allowed = [d for d in range(10) if d not in forbidden]
    k = len(allowed)

    if k < 2:
        print(0)
        return

    pairs = k * (k - 1) // 2
    ans = pairs * 20
    print(ans)

if __name__ == "__main__":
    solve()
```代码首先构造允许的数字集，这是确保禁止的数字永远不会进入任何候选构造所必需的。 支票`k < 2`处理不存在有效对的退化情况。 

表达式`k * (k - 1) // 2`有效计算无序数字对的数量，而无需显式生成它们。 乘以 20 应用固定组合计数来排列每个数字的三个副本。 

一个常见的实施错误是忘记顺序在最终序列中很重要，但在数字的选择中并不重要。 这就是为什么我们将配对选择（组合选择）与排列计数（固定排列计数）分开。

 ## 边缘情况

 如果几乎所有数字都被禁止并且仅保留一个数字，则算法会正确生成 k = 1 并立即返回 0，因为无法形成对。 例如，输入`9`带数字`0 1 2 3 4 5 6 7 8`只留下数字 9，因此 k = 1，并且函数按预期返回 0。 

当没有禁止数字时，k = 10，因此对的数量为 45。乘以 20 得到 900。算法通过相同的公式干净地处理这个问题，无需特殊的大小写。 

如果正好剩下两位数字，k = 2，产生一对，因此正好有 20 个有效密码。 这是最小的非零场景，并确认组合因子应用正确。 

在所有情况下，计算的结构都确保禁止的数字仅影响允许的集合的大小，并且从不需要除计数之外的任何特殊分支。
