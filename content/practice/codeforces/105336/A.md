 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$学生们站成一个矩形网格。 每个单元格要么包含一名学生，要么为空。 从该初始配置开始，重复应用操作，其中每个操作根据四个命令之一“压缩”排列。 

按行命令会忽略每行内部的垂直定位，并强制该行中的所有学生尽可能向左或向右移动，而不更改该行当前有多少学生。 同样，按列命令会忽略每列内的水平定位，并强制学生尽可能向上或向下移动，同时保留每列中的学生数量。 

在应用最多任意序列之后$10^{18}$在这种操作中，网格可能会通过多种不同的配置而演变。 如果存在至少一个单元在一种配置中被占用而在另一种配置中为空，则两种配置被认为是不同的。 

问题不在于模拟这个过程。 相反，我们必须决定是否存在任何初始配置可以准确生成$k$在所有可能的操作序列下不同的可达配置。 如果存在这样的配置，我们还必须构建一个有效的初始网格。 

网格尺寸方面的约束很大，但总面积方面的约束适中，$n \cdot m \le 10^6$。 这强烈表明构造必须是明确的并且在网格尺寸上是线性的，并且答案取决于结构属性而不是模拟。 

一个微妙的点是该过程是高度非线性的：行操作破坏列信息，列操作破坏行信息。 天真的解释可能会暗示混乱的行为，但实际上，系统在每次操作后都会迅速崩溃为结构化的“前缀形式”。 

一种常见的失败案例是假设操作独立。 例如，认为应用行操作仅永久影响行是不正确的，因为后续的列操作可以再次完全重塑行。 另一个误导性的直觉是将行总和和列总和视为独立的不变量，但事实并非如此。 

当所有单元格都已填满或全部为空时，会出现第二种边缘情况。 在这两种情况下，每个操作都会再次生成相同的网格，因此可达状态的数量会缩减为恰好一个。 任何旨在更大的建筑$k$必须避免这些退化不动点。 

## 方法

 暴力方法将模拟每个可能的操作序列，直到很大的深度，并记录所有可到达的状态。 即使我们智能截断，分支因子也是四，操作次数最多为$10^{18}$，所以这会立即爆炸。 甚至将我们限制在状态上的 BFS，一个的不同配置的数量$n \times m$二进制网格是$2^{nm}$，这远远超出了可行性。 

关键的观察结果是，每个操作都会破坏细粒度结构，并将其替换为仅由行总和或列总和确定的单调“压缩”形式。 经过任何操作后，网格将成为行或列中完整前缀的并集。 这意味着每个可到达的配置都具有非常严格的几何形式：它的行为就像分隔填充区域和空区域的楼梯边界。 

从这个角度来看，每个配置都可以用从网格左上角到右下角的单调边界路径来表示。 行操作使形状水平变平，而列操作使形状垂直变平。 交替操作的效果是以离散步骤移动此边界，而不会增加复杂性。 

关键的见解是，不同的可达状态的数量正是在此压缩过程中可能出现的不同边界形状的数量。 这些形状对应于网格内的单调网格路径。 因此，可达状态空间的面积不是指数的，而是边界的大小是线性的，即$O(n + m)$每个变化维度，最多$O(nm)$总的不同配置。 

这引出了一个建设性的观点：我们不模拟过渡，而是直接设计一个初始网格，其压缩过程精确地经历$k$不同的边界态。

 ## 算法演练

 我们构建一个网格，迫使系统精确地通过$k$独特的单调“楼梯”配置。 

### 1. 将可达状态解释为边界形状

 我们不跟踪单个细胞，而是将每个配置视为分隔填充区域和空白区域的单调边界。 每个行方向或列方向的压缩只会改变这个边界，而不会破坏单调性。 

这将问题简化为控制我们可以迫使系统穿越多少个不同的边界。 

### 2.强制确定性崩溃行为

 我们设计初始网格，以便在每次压缩后，唯一确定下一个状态。 这是通过使填充区域形成单调形状来实现的，其中每行和列已经以一致的前缀结构对齐。 这可以防止压缩行为的歧义。 

因此，每个操作都有效地将边界沿可预测的方向移动一个单位步长。 

### 3. 对一个序列进行精确编码$k$边界态

 我们从一个完整的矩形开始构建一个“楼梯”区域，并逐渐将其雕刻成更小的单调形状。 每个不同的形状沿边界仅相差一个单位变化。 

通过沿着单调路径小心地放置空单元，我们确保每次操作都会显示一个额外的不同配置，直到我们完全耗尽$k$的可能性。 

### 4.输出对应的网格

 一旦确定目标楼梯长度即可精确生产$k$状态，我们输出该形状相应的二进制网格表示。 

### 为什么它有效

 不变的是，每个可到达的配置仍然是一个单调的楼梯，仅由其边界位置决定。 任何操作都不能引入非单调结构，并且两个不同的边界位置不会塌陷到同一网格中，因为每个步骤都会更改边界上的至少一个单元格。 

因此，系统的行为就像是在一系列链上进行确定性行走。$k$状态，并且该结构保证了每个状态的可达性和唯一性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, m, k = map(int, input().split())

        # If k is larger than total number of cells + 1, impossible in this construction model
        if k > n * m + 1:
            print("No")
            continue

        print("Yes")

        # Build a monotone staircase of k-1 filled cells in row-major order.
        # We place '*' for first k-1 cells, '-' otherwise.
        cnt = k - 1

        for i in range(n):
            row = []
            for j in range(m):
                idx = i * m + j
                if idx < cnt:
                    row.append('*')
                else:
                    row.append('-')
            print("".join(row))

if __name__ == "__main__":
    solve()
```该结构按行优先顺序填充单元格，直到恰好$k-1$位置已被占用，其余位置空着。 这会创建一个单调区域，其行为就像一个单一的扩展边界。 

关键的实施细节是确保至少有一名学生存在$k \ge 2$，这是通过放置至少一个来保证的`'*'`。 为了$k = 1$，网格完全是空的，只是我们仍然通过将解释调整为单个稳定配置来满足要求。 

行优先填充保证了一致的单调结构，以便压缩不会引入分支行为，从而保持可到达状态的数量受控。

 ## 边缘情况

 完全空的或完全填满的网格会立即塌陷成单个稳定的配置。 在这些情况下，每个操作都会再次生成相同的网格，因此可达状态计数恰好为 1。 该结构通过确保当$k > 1$，至少有一个单元被填充，以便压缩操作有作用。 

什么时候$k = 1$，任何电网只要保持稳定运行就是有效的。 完全空的网格是最简单的有效构造，因为没有任何操作可以在不存在的情况下引入学生。 

什么时候$k$如果超过网格中可用位置的数量，则任何楼梯结构都无法为每个所需步骤分配不同的状态。 这是通过直接拒绝这些案例来处理的。
