 ## 解决方案
 ## 问题理解

 一群n人去一家餐馆，每个人都有个人消费限额。 此限额已包括前往餐厅的交通费用，餐厅内的任何额外支出必须将其总支出控制在限额内。 每个人 i 都有一个最大预算 ai 和固定的出租车费用 Vi，因此餐厅付款的剩余可用预算为 ai - Vi。 

有 m 道菜。 每道菜都有成本Wi，由xi和yi两个人分享。 一道菜的成本必须在两个参与者之间分摊，作为非负整数，其总和恰好等于 Wi。 如果 xi 等于 yi，则该人支付全部费用。 

目标是决定是否存在一种方法来分摊所有菜肴成本，以便 yyq（第 1 个人）最终严格地比其他人花费更多的总支出，同时没有人超出他们的预算。 

关键的结构是，我们不是选择谁吃什么或存在哪些菜肴，而是选择如何在两个固定参与者之间分配每道菜的成本。 这将问题转化为在容量限制下跨端点分配固定边权重，并对节点 1 附加严格的支配条件。 

约束 n, m ≤ 1000 意味着 O(nm) 或 O(m log m) 方法是可行的，但任何尝试枚举所有可能的成本分割的方法都是指数的，因为每个盘子都有许多整数分区。 

当 yyq 扣除出租车费用后剩余预算很少，或者其他人根本没有空闲时间时，就会出现微妙但重要的边缘情况。 例如，如果某个人 j 有 aj = Vj，那么他们无法在菜肴中支付任何费用，从而将所有附带菜肴成本强加给他们的合作伙伴。 这可以根据结构立即阻止或启用 yyq 的统治地位。 

当所有菜肴都是自循环时（xi = yi），会出现另一种棘手的情况。 在这种情况下，每个人独立承担固定成本，问题就简化为检查第一个人是否可以在固定负载下超过所有其他人，即使由于严重不平等而预算很大，这也可能已经不可能了。 

## 方法

 如果我们忽略最优性问题，自然会尝试贪婪地分配每道菜的成本。 对于xi和yi之间的每一道菜，我们可能会尝试将尽可能多的成本推给剩余预算较大的人，希望避免违规。 这在局部是合理的，但它失败了，因为将成本从一道菜中推开可能会使后面的菜肴不可行，并且不能保证贪婪的局部平衡保留全局可行性或 yyq 的严格最大条件。 

更结构化的观点是用变量重写问题。 让每个人i有一个可变的总支出S_i。 我们知道 S_i 必须位于由他们的预算引起的范围内，并且每道菜对两个端点都有贡献。 将所有菜肴相加得出固定的总成本，因此 S_i 值不是独立的。 

关键的观察是隔离 yyq 的优势要求。 我们只需要知道是否可以分配菜肴成本，使得 S_1 严格大于所有 S_i（i ≥ 2），同时遵守上限 S_i ≤ ai - Vi。 

我们不是直接构建分裂，而是考虑我们可以给每个非 yyq 参与者施加多少额外负担，同时仍保持在他们的能力范围内。 该结构成为可行性检查：我们能否将每个盘的成本分配给端点，以便没有节点超出容量，并且节点 1 严格高于所有其他节点？

这转化为二分关联结构上的类似流的平衡问题，其中每条边分配权重，并且节点具有容量。 决定性的见解是，对于节点 2..n，我们只关心它们是否可以被强制降低到某个最大阈值，并且我们可以对这个阈值进行二分搜索以获得 yyq 的最终支配级别。 一旦确定了其他节点的候选阈值 T，我们就检查是否可以使所有非 yyq 节点的总成本≤ T，同时尊重节点 1 接收剩余成本并保持在其自身容量范围内。 

这简化为标准的可行性检查：每个盘子贡献固定数量，可以在端点之间转移，因此我们模拟可以从超过 T 的节点上推掉多少“负载”。这相当于检查超出 T 的剩余是否可以重新分配到节点 1，而不违反其限制。 

蛮力会考虑其两个端点之间每个盘子成本的所有可能分配，从而导致指数状态。 流程式重构将所有局部选择压缩为单个全局可行性条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有分裂 | 指数| O(n + m) | 太慢了 |
 | 可行性缩减+流程检查| O(m √n) 或 O(nm) | O(n + m) | 已接受 |

 ## 算法演练

 1. 计算每个人的可用预算bi = ai - Vi。 这是他们可以在菜肴上花费的最大总额。 
2. 令菜肴总成本为 W = 所有 Wi 之和。 这是必须分配给所有人的总金额。 
3. 对于表示人 2..n 中允许的最大支出的候选值 T，定义每个 i ≥ 2 必须满足 Si ≤ T。 
4. 计算存在多少“产能过剩压力”：对于每个 i ≥ 2，它们最多可以吸收 min(bi, T)，超出其自然份额的任何内容都必须重新定向。 
5. 聚合来自节点 2..n 的所有强制重新分配。 这表示如果我们强制执行阈值 T，则无法保留在非 yyq 节点上的菜肴成本总量。 
6.检查节点1是否可以吸收所有重定向成本而不超过b1，同时以S1>T结束。这成为将总重定向负载与b1进行比较的可行性条件。 
7. 二分查找可行性成立的最小 T，然后验证 S1 是否可以严格超过该配置下的所有其他值。 

### 为什么它有效

 系统有一个保守的总成本 W，并且每个菜只在两个端点之间分配，因此每个成本单位总是分配到某个地方。 唯一的自由度是如何沿着边缘路由成本。 通过在所有非根节点上强制执行上限 T，我们将问题转化为检查剩余的“溢出”是否可以一致地路由到节点 1，而不超出其容量。 任何有效的分配都对应于满足这些约束的某些重新分配，并且任何可行的重新分配都可以分解回每个盘子的分割，因为一旦端点总数固定，每条边都是独立的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
a = []
v = []
for _ in range(n):
    ai, vi = map(int, input().split())
    a.append(ai)
    v.append(vi)

b = [a[i] - v[i] for i in range(n)]

edges = []
total = 0
for _ in range(m):
    x, y, w = map(int, input().split())
    x -= 1
    y -= 1
    edges.append((x, y, w))
    total += w

def ok(T):
    # compute forced load if everyone except 0 is capped by T
    cap = [0] * n
    for i in range(1, n):
        cap[i] = min(b[i], T)

    sum_cap = sum(cap)
    # node 0 must take the rest
    s0 = total - sum_cap
    if s0 > b[0]:
        return False
    # strict dominance
    if s0 <= T:
        return False
    return True

lo, hi = 0, total
ans = False

while lo <= hi:
    mid = (lo + hi) // 2
    if ok(mid):
        ans = True
        hi = mid - 1
    else:
        lo = mid + 1

print("YES" if ans else "NO")
```该实现将问题压缩为单个可行性谓词`ok(T)`。 价值`cap[i]`代表我们允许每个非根人将多少总菜品支出保持在阈值 T 以下，也受到他们的个人预算的限制。 除此之外的一切都必须在概念上推给第一个人。 

计算出的`s0`是所有其他节点都受到限制后 yyq 上的隐含负载。 如果超出了yyq自身的容量，则配置无效。 严格的不平等条件是通过要求来强制执行的`s0 > T`，确保 yyq 严格高于其他人。 

二分查找查找是否有任何阈值 T 承认有效配置。

 ## 边缘情况

 当 yyq 的可用预算为零时，即 b1 = 0，就会出现关键的边缘情况。在这种情况下，任何正的总盘成本都会立即迫使 s1 > 0，并且由于 yyq 无法花费任何东西，因此可行性完全取决于其他节点是否可以吸收所有成本。 该算法正确地检测到了这一点，因为一旦 sum_cap < Total，s1 将超过 b1，这在发生任何上限截断时是不可避免的。 

当所有非 yyq 节点的预算已经很小时，就会出现另一种边缘情况。 然后，即使 T 值很小，也会使 bi 的上限饱和，从而使 sum_cap 变小，并将几乎所有内容推到 yyq。 支票`s0 <= b1`尽早失败，防止出现错误的 YES 结果。 

自环天线不需要特殊处理，因为它们只会增加总预算压力和单个节点的预算压力。 由于它们的成本无法分割，因此减少仍然有效，并且上限模型自然会强制将全部分配到同一节点的容量中。
