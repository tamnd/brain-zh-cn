 ## 解决方案
 ## 问题理解

 我们得到了一个由小写字母组成的网格$n$行和$m$列。 从这个网格中，我们有兴趣选择任何两个连续的行和任何两个连续的列。 每个这样的选择都定义了一个$2 \times 2$子矩阵。 目标是计算其中有多少个$2 \times 2$块匹配特定模式：$$\begin{matrix}
c & c \\
p & c
\end{matrix}$$换句话说，块的顶行必须是两个相同的字符，都等于`'c'`，而底行必须有`'p'`在左边和`'c'`在右侧。 

输出只是位置的数量$(i, j)$使得由行组成的子矩阵$i, i+1$和列$j, j+1$满足这个模式。 

限制因素$2 \le n, m \le 500$至多暗示$500 \times 500 = 250000$可能的左上角$2 \times 2$子矩阵。 这足够小，以至于$O(nm)$扫描就足够了。 然而，网格访问模式很重要，因为必须在恒定的时间内检查每个候选者。 

这里有时会发生一个天真的误解，即独立扫描所有行和列对，并以更重的方式重新计算子矩阵。 对于这个问题规模来说，这仍然有效，但没有必要。 

边缘情况很少但值得注意。 如果网格不包含`'c'`无论如何，答案显然是零。 如果每个细胞都是`'c'`，答案仍然是零，因为左下角一定是`'p'`，它永远不会出现。 另一个微妙的情况是当`'p'`出现但不在相对于 valid 的左下位置`'c'`对，这会导致部分但无效的匹配。 

## 方法

 蛮力方法很简单：迭代每个可能的左上角$2 \times 2$submatrix，提取它的四个字符，并检查它是否与所需的模式匹配。 由于每次检查都是恒定时间，因此运行时间为$O(nm)$，最多为 250,000 次操作。 即使我们考虑 Python 索引的开销，这也足够快了。 

不需要预处理或高级数据结构。 关键的观察结果是每个候选子矩阵都是独立的，验证只需要直接的字符比较。 

人们可以尝试预先计算`'c'`或者`'p'`，但这并没有有意义地降低复杂性，因为瓶颈已经只是扫描网格一次。

 | 蛮力 |$O(nm)$|$O(1)$| 已接受 |
 | 最佳|$O(nm)$|$O(1)$| 已接受 |

 ## 算法演练

 我们只需要检查每一个可能的$2 \times 2$子矩阵一次。 

## 算法演练

 1. 迭代所有可能的左上角位置$(i, j)$在哪里$0 \le i < n-1$和$0 \le j < m-1$。 这确保了子矩阵保持在边界内。 
2. 对于每个位置，检查四个单元格：

 左上角$(i, j)$, 右上角$(i, j+1)$, 左下角$(i+1, j)$，以及右下角$(i+1, j+1)$。 
3. 检查这些是否满足条件：

 顶行必须都是`'c'`，左下角必须是`'p'`，右下角必须是`'c'`。 
4. 如果条件成立，则增加答案计数器。 
5. 扫描完所有位置后，输出计数器。 

我们直接检查每个候选而不是分组或预处理的原因是每个子矩阵完全由四个字符确定，因此不存在值得利用的重叠计算。 

### 为什么它有效

 每个有效答案都对应于左上角的一个选项$(i, j)$。 该算法枚举了所有此类角点，没有遗漏或重复。 对于每个角，它应用与所需模式匹配的确定性谓词。 由于每个候选谓词只评估一次，并且仅取决于这四个单元，因此不会错过任何有效配置，也不会计算无效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]

    ans = 0

    for i in range(n - 1):
        row1 = g[i]
        row2 = g[i + 1]
        for j in range(m - 1):
            if row1[j] == 'c' and row1[j + 1] == 'c' and row2[j] == 'p' and row2[j + 1] == 'c':
                ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现反映了前面描述的枚举。 网格存储为字符串列表，以便字符访问$O(1)$。 我们避免重复索引`g[i][j]`在可能的情况下，将行引用缓存到`row1`和`row2`，这减少了 Python 中的开销。 

循环边界`n - 1`和`m - 1`确保我们在形成时永远不会访问超出范围的索引$2 \times 2$堵塞。

 ## 边缘情况

 一个关键的边缘情况是网格均匀时。 例如：```
2 2
cc
cc
```该算法仅检查一个子矩阵$(0,0)$，但不满足条件，因为左下角不是`'p'`。 迭代仍然正确运行，结果为零。 

另一种情况是当`'p'`存在但从未与有效的对齐`'cc'`其上方：```
3 3
ccc
cpc
ccc
```每检查一次$2 \times 2$窗口将无法满足顶行或右下角的条件。 扫描仍然评估所有四个候选者，但没有一个同时满足所有约束，按预期产生零。
