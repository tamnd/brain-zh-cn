 ## 解决方案
 ## 问题理解

 我们得到了一个高度列表，并可以将它们任意重新排序成一行。 对于任何固定的安排，每个连续的段所贡献的成本等于该段内最高和最矮的人之间的差异。 线路的总不适度是所有线段上这些值的总和。 

任务是找到一种元素的排序，以最大限度地减少这种总体不适，并计算有多少种不同的排列达到了该最小值。 

尽管输入大小最多只有一千，但目标函数是在所有子数组上定义的，这已经意味着段数是二次方的，并且每个段都涉及一个最大值和最小值。 任何显式评估所有排列的解决方案都是远远不可行的，因为 n! 生长速度极快。 即使尝试简单地评估单个排列也会花费 O(n^2) 或更糟的成本，因此问题的结构必须强烈限制最佳排列的样子。 

一个微妙的点是存在相同的高度。 当值重复时，交换相同的元素会产生不同的排列，但不会更改任何段的最大或最小行为。 这些都必须计入最终答案，尽管它们在成本方面是无法区分的。 

一个常见的陷阱是假设元素的顺序并不重要，因为无论如何都会考虑所有子数组。 实际上，放置会改变哪些元素成为分段中的局部最大值或最小值，这完全改变了每个元素在整个结构中的贡献。 

例如，使用输入`[3, 1, 2]`，不同的排列会产生不同的范围总和，甚至很小的反转也会同时影响许多子数组。 事实证明，最优结构极其严格，并且计数部分仅在相同值的等价类中存在。 

## 方法

 一种直接方法会尝试每种排列并计算每个排列的子数组范围的总和。 对于固定排列，有 O(n^2) 个子数组，并且使用预先计算的前缀结构计算每个范围仍然需要推理最大值和最小值。 即使我们优化范围查询，评估所有 n！ 排列是不可能的。 一旦 n 甚至超过 10，此操作就会立即失败。 

关键的观察结果是，目标仅取决于值的相对顺序，并且与全局排序结构的任何偏差都会引入反转，从而扩展子数组集，其中较大元素出现得太早或太晚，从而增加对最大最小和的贡献。 直观上，一旦较大的元素出现在较小元素的左侧，它就成为许多不应该占主导地位的段中的最大值，同样，较小的元素也经常成为最小值。 这会产生无法在其他地方补偿的额外范围贡献。 

这导致这样一个事实：最佳排列只是值的排序排列。 一旦排序，每个子数组的结构就以单调的方式固定：在任何段中，最大值始终是最右边的元素，最小值始终是最左边的元素。 这将问题分解为单个规范配置。 

一旦知道最佳排列，就可以通过扫描所有子数组并维护运行最小值和最大值，在 O(n^2) 中直接计算最小成本。 最佳排列的数量完全由重复项决定，因为可以在不改变排序结构的情况下自由排列相等的值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!·n^2) | O(n!·n^2) | O(n) | 太慢了 |
 | 排序 + 计算 + 计算重复项 | O(n^2 + n log n) | O(n^2 + n log n) | O(n) | 已接受 |

 ## 算法演练

 ### 1. 对数组进行排序

 我们首先按非降序对所有高度进行排序。 这会产生一个规范的排列，其中所有反转都被删除。 任何最优解都可以转化为这种形式，而不会增加成本。 

### 2. 计算排序数组的最小总不适度

 我们迭代子数组的所有起始索引。 对于每个起始位置，我们逐步扩展右边界，同时保持当前的最小值和最大值。 每个扩展都会将它们之间的当前差异贡献给答案。 这直接在 O(n^2) 内累积所有子数组的范围总和。 

### 3. 计算值的重数

 我们计算每个不同高度出现的次数。 这些频率决定了有多少排列可以保留排序结构直至无法区分的交换。 

### 4. 计算最优排列的数量

 排序的配置在值顺序方面是固定的，但相同的值可以在它们之间进行排列。 因此有效排列的数量就是多项式系数

 嗯！ 除以每个不同值的频率阶乘的乘积，以 998244353 为模计算。 

### 为什么它有效

 关键的结构属性是，两个值之间的任何反转都会创建额外的子数组，其中较大的元素出现得太早或较小的元素出现得太晚，从而增加了最大最小跨越不必要距离的段数。 删除反转总是会减少或保留贡献，因为它恢复了值顺序和位置顺序之间的局部性。 因此，全局排序的排列是局部和全局最优的，并且所有最优排列必须保留仅由相等值引起的划分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort()
    
    # compute minimal cost
    ans = 0
    for l in range(n):
        mn = a[l]
        mx = a[l]
        for r in range(l, n):
            if a[r] < mn:
                mn = a[r]
            if a[r] > mx:
                mx = a[r]
            ans += mx - mn
    
    # count permutations (multiset permutations)
    from collections import Counter
    cnt = Counter(a)
    
    fact = [1] * (n + 1)
    for i in range(1, n + 1):
        fact[i] = fact[i - 1] * i % MOD
    
    res = fact[n]
    for v in cnt.values():
        res = res * pow(fact[v], MOD - 2, MOD) % MOD
    
    print(ans, res)

if __name__ == "__main__":
    solve()
```该代码首先对数组进行排序以强制执行规范的最佳结构。 然后双循环枚举所有子数组； 运行的最小值和最大值是增量维护的，因此每次扩展都是 O(1)。 这避免了从头开始重新计算范围极值。 

计数部分构造最多 n 的阶乘，并对每个频率组应用模逆。 这直接实现了多项式系数公式。 

一个微妙的实现细节是，由于 n 很小，计算阶乘一次就足够了，并且给定素数模 998244353，模幂是安全的。

import sys, io

MOD = 998244353

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    n = int(input())
    a = list(map(int, input().split()))
    
    a.sort()

        mn = a[l]
        mx = a[l]
        for r in range(l, n):
            mn = min(mn, a[r])

    res = fact[n]

 ## 边缘情况

 单元素数组不产生子数组范围贡献，因此最小成本为零。 该算法处理此问题是因为双循环仅累积 l = r = 0 的贡献，其中 max 等于 min。 

全相等数组还为每个子数组产生零贡献。 在这种情况下，频率图折叠为单个组，并且多项系数正确返回 1，因为所有排列在值结构中都是等效的。 

严格递增或递减的数组会产生非零贡献，但排序变换使它们保持不变，从而确认该算法不依赖于单调性之外的任何特殊排序。
