 ## 解决方案
 ## 问题理解

 我们得到了一个具有单一进化状态的回合制游戏：战斗强度值和累积伤害值。 最初强度是某个固定值$a_0$，并且伤害从零开始。 有$n$回合，并且在每一轮中我们必须恰好选择两个动作之一。 

一个动作就是一次攻击，会增加当前的力量$a$计入总伤害$d$。 另一个动作是训练步骤，它将力量增加固定量$a_i$与该轮相关的。 

毕竟$n$回合，假设我们使用了攻击动作$x$次数及训练动作$y$次。 最终的分数不仅仅是伤害，而是加权的结果：$$\text{score} = d \cdot (x \cdot k_1 + y \cdot k_2),$$其中每个查询提供不同的常量$k_1, k_2$。 任务是选择操作序列以及隐含的训练增量，以最大化每个查询的分数。 

限制条件很大：$n$和$q$达到$10^5$，并且值可以大到$10^6$和$10^9$。 这排除了任何模拟每个查询的序列或尝试训练操作的所有子集的方法。 甚至$O(nq)$立刻就太慢了。 

一个关键的微妙之处在于，行动的顺序对伤害很重要，因为力量会随着时间的推移而增加。 另一个微妙之处是不同的查询完全改变了目标函数，因此任何解决方案都必须将预处理与每个查询的评估分开。 

当人们认为顺序无关紧要时，就会出现一种常见的失败情况。 例如，如果我们忽略排序：

 输入：```
n = 2, a0 = 1
a = [100, 1]
```如果我们总是假设“首先进行所有训练”，我们可能会忽略，根据结构，仅首先选择大增量可以占主导地位。 正确答案取决于证明排序的最佳结构，而不是猜测。 

另一种失效模式是治疗$d$独立于其中$a_i$被选中。 由于训练既选择子集又影响排序，因此忽略其中任何一个都会导致评分不正确。 

## 方法

 直接的暴力解决方案将枚举每个序列$n$动作，决定哪些轮次是训练，哪些轮次是攻击，并选择哪个子集$a_i$值用作训练增量。 即使我们修正计数$x$和$y$，选择训练位置的方法数为$\binom{n}{y}$，以及所选的顺序$a_i$价值进一步增加了可能性。 这很快就会超出可行性，达到指数级的复杂性。 

第一个结构简化是理解排序。 假设我们修复了哪个$y$我们使用的训练增量。 然后力量只会增加，如果攻击发生得晚，那么攻击总是会受益于更高的力量。 如果攻击发生在训练操作之前，交换它们会使该攻击以更高或相同的强度发生，并且不会减少未来的力量增长。 这意味着所有训练操作都可以在所有攻击之前移动，而不会减少伤害。 因此，对于训练值的任何固定子集，最佳顺序是“首先进行所有训练，然后进行所有攻击”。 

在这种结构下，如果我们选择$y$训练值与总和$S_y$，则攻击前的强度变为$a_0 + S_y$，伤害变为：$$d = x \cdot (a_0 + S_y), \quad x = n - y.$$唯一剩下的决定是哪一个$y$值来选择。 因为只有总和才重要，所以我们总是取最大的$y$数组中的值。 

现在问题变成了一个一维选择$y$：对于每个$y$，我们计算一个确定性分数，然后在每个查询中最大化它。 

剩下的困难是查询引入了之间的非线性耦合$d$,$x$， 和$y$。 展开表达式表明每个$y$对应于 2D 空间中的一个点，每个查询都要求使用从以下公式导出的方向向量获得最佳点积$k_1, k_2$。 这将问题转化为静态凸包最大点积查询。

 ## 算法演练

 ### 1. 对训练增益进行排序并构建前缀和

 我们对数组进行排序$a_i$按降序排列并计算前缀和$S_y$， 在哪里$S_y$是最大的总和$y$训练值。 这保证了对于任何固定数量的训练操作，我们始终使用最优子集。 

### 2. 将损坏表示为函数$y$对于固定的$y$，我们有$x = n - y$，强度变为$a_0 + S_y$。 由于所有训练都在攻击之前进行，因此伤害为：$$d_y = (n - y)(a_0 + S_y).$$这将整个顺序过程减少为每个值一个值$y$。 

### 3. 将每个状态的分数重写为线性形式

 分数变为：$$\text{score}(y) = d_y \cdot ((n-y)k_1 + yk_2).$$扩展：$$(n-y)k_1 + yk_2 = nk_1 + y(k_2 - k_1),$$所以：$$\text{score}(y) = nk_1 \cdot d_y + (k_2 - k_1) \cdot (y d_y).$$每个$y$贡献一个固定对$(d_y, y d_y)$，每个查询都是这些点的线性最大化。 

### 4.解释为凸包最大点积

 各州$y$对应一个点：$$P_y = (d_y, y d_y).$$每个查询定义一个方向向量：$$(K_1, K_2) = (nk_1, k_2 - k_1),$$我们希望最大化：$$P_y \cdot (K_1, K_2).$$这是一个经典的静态凸包问题。 

### 5. 构建点的凸包

 我们计算所有点的上凸包$P_y$。 任何线性目标的最佳答案都位于船体顶点处，并且随着方向的变化，最佳顶点沿着船体单调移动。 

### 6. 使用二分搜索回答查询

 因为船体是有序的，所以具有固定方向的点积沿着它是单峰的。 我们二分搜索最好的顶点$O(\log n)$每个查询。 

### 为什么它有效

 所有运营的重新安排都归结为以下选择：$y$，并且对于每个$y$，最优内部排序是固定的。 这将问题转化为评估一维索引上的确定性函数。 该函数自然地提升为凸几何表示，其中线性查询对应于支持超平面。 凸性确保最大值始终出现在船体上，因此限制对船体顶点的关注永远不会失去最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_convex_hull(points):
    def cross(o, a, b):
        return (a[0] - o[0]) * (b[1] - o[1]) - (a[1] - o[1]) * (b[0] - o[0])

    hull = []
    for p in points:
        while len(hull) >= 2 and cross(hull[-2], hull[-1], p) <= 0:
            hull.pop()
        hull.append(p)
    return hull

def dot(p, k1, k2):
    return p[0] * k1 + p[1] * k2

def query_hull(hull, k1, k2):
    l, r = 0, len(hull) - 1
    while r - l > 3:
        m1 = l + (r - l) // 3
        m2 = r - (r - l) // 3
        if dot(hull[m1], k1, k2) < dot(hull[m2], k1, k2):
            l = m1
        else:
            r = m2
    best = 0
    for i in range(l, r + 1):
        best = max(best, dot(hull[i], k1, k2))
    return best

def solve():
    n, a0 = map(int, input().split())
    a = list(map(int, input().split()))
    a.sort(reverse=True)

    pref = [0]
    for v in a:
        pref.append(pref[-1] + v)

    points = []
    for y in range(n + 1):
        x = n - y
        d = x * (a0 + pref[y])
        points.append((d, d * y))

    hull = build_convex_hull(points)

    q = int(input())
    out = []
    for _ in range(q):
        k1, k2 = map(int, input().split())
        k1n = k1 * n
        k2k1 = k2 - k1
        out.append(str(query_hull(hull, k1n, k2k1)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现首先对训练增益进行排序，以便任何前缀都对应于固定数量的训练动作的最佳可能选择。 前缀和将子集选择转换为恒定时间查找$y$。 

每个$y$然后映射到一个几何点，对损伤及其如何随训练动作的数量进行缩放进行编码。 这是消除对序列结构依赖的关键转换。 

凸包是使用标准单调堆栈构建的。 这是有效的，因为点是按增加的顺序添加的$y$，并且该结构消除了非凸转弯。 

每个查询都转换为线性目标，并且在外壳上使用三元搜索，因为凸多边形上的点积沿其边界是单峰的。

 ## 边缘情况

 当所有训练增益为零时，就会出现关键的边缘情况。 在这种情况下，无论顺序如何，强度都不会增加，因此最佳策略纯粹是选择要执行的攻击次数。 该算法可以正确处理这个问题，因为所有前缀和都保持为零，并且得分函数退化为简单的二次形式$y$，仍然可以通过相同的转换正确评估。 

另一个边缘情况出现时$k_1 = k_2$。 在这里，乘数变得独立于攻击和训练之间的划分，并且问题简化为仅最大化伤害。 凸包仍然可以正确评估，因为查询方向折叠为固定向量，并且最佳点仍然是与最佳损伤累积相对应的包顶点。 

最后一个微妙的情况是$n = 1$。 只有一个决定，船体包含两个点对应$y=0$和$y=1$。 该算法自然地评估两者并返回正确的最大值，而不需要特殊的外壳。
