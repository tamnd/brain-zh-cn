 ## 解决方案
 ## 问题理解

 我们在数轴上有两组点。 一组代表行李物品，每个行李物品都位于某个整数位置。 另一组代表站在传送带前的人，也是整数位置。 每一秒，所有行李的位置都会增加一个，因此所有东西都会以单位速度向右漂移，而人们保持不动。 

每个人最多可以匹配一件行李，但前提是该行李从他们的左侧开始，因为右侧已经存在的任何东西都已被有效地“看到”并且不可能是他们的。 有效的配置是一种将一些行李物品分配给某些人的方法，这样就不会将任何行李分配给多个人，并且没有人会收到多个行李物品。 

对于每个有效的配置，我们都会查看某人首次成功拿到行李的时间。 如果一个人与从位置 a 开始的包匹配，而该人位于位置 b，则包会在时间 b – a 到达他们，因为两者相对于彼此刚性移动。 配置的感兴趣值是所有匹配对中这些时间的最小值。 

我们没有被要求枚举配置。 相反，我们必须计算所有有效配置的最短时间之和，以 998244353 为模。 

坐标很小，最多500个，最多500人，500件行李物品。 这立即表明位置比索引更重要：几何结构密集但有界，因此任何解决方案都应将问题减少到在较小的值范围内进行计数，而不是直接迭代较大的组合状态。 

一个微妙的点是，配置仅由谁接收哪个行李来定义，而不是由任何及时订购来定义。 另一个重要的约束是分配在两侧都是单射的，因此我们正在计算由“人的行李左侧”条件引起的二部图中的部分匹配。 

一个天真的解释错误是认为每个人独立地选择左边的行李物品。 这忽略了两个人选择相同行李的碰撞，这是禁止的。 另一个容易犯的错误是假设我们只关心匹配是否存在； 相反，我们对所有匹配项进行统计汇总。 

## 方法

 蛮力方法将枚举所有方法，为每个人分配没有行李或在他们左边的行李中分配一件行李，然后过滤掉两个人挑选相同行李的分配。 即使我们忽略可行性检查，状态空间也已经是巨大的：每个人最多有 500 个选择，所以原始产品大约是$500^{500}$，远远超出任何计算范围。 

一旦我们将视角从二分图上的分配转移到匹配，真实的结构就会变得更加清晰。 配置只是人和行李之间的匹配，其限制是仅当行李位于人的左侧时才存在边缘。 我们每次匹配所需的数量是所选边上的最小值$b_j - a_i$。 

我们没有直接计算匹配的最小值，而是颠倒了观点。 对于任何阈值时间$t$，我们可以问至少有多少个匹配全部选择了有延迟的边$t$。 这意味着我们只允许边缘满足$b_j - a_i \ge t$。 如果我们定义$F(t)$作为仅使用此类边的匹配数量，则每个匹配对所有匹配的贡献恰好为 1$F(t)$为了$t$直至其最小边重。 总结$t$重建所需的最小值之和。 

因此，答案变成了一系列二部图的累积计数，参数化为$t$，其中边缘消失为$t$增加。 由于仓位以 500 为界，因此只有 500 个相关阈值。 

对于固定的$t$，该图具有特殊的结构。 如果我们对行李位置进行排序，每个人都会连接到行李物品的前缀，因为条件$a_i \le b_j - t$是单调的$a_i$。 因此，每个人都有一个允许携带行李的区间，该区间是按排序顺序排列的前缀。 在这样的“前缀嵌套”二部图中计算匹配是很容易处理的，因为约束是单调的，并且可以按照增加前缀长度的顺序进行处理。 

对于这些图表，我们按照人们可以拿取的行李物品数量进行排序。 当处理一个人时$k$假设有可用的行李物品$i-1$人们已经被分配了不同的行李。 有$k - (i-1)$剩余未使用的合格行李物品。 该人可以选择其中任何一个，也可以保持不匹配。 只要可行性条件满足，这就会产生每人的简单乘法贡献$k \ge i-1$被维持。 

这改变了每一个$F(t)$转化为从前缀大小派生的线性项排序后的乘积，使每个评估$O(m \log m + n \log n)$。 自从$t$范围最多为 500 个值，总的复杂性仍然是可控的。

 ## 算法演练

 我们首先对行李位置进行排序，这样我们就可以推断出“有多少物品位于左侧”作为前缀计数。 对于每个人，我们稍后会将他们的约束转化为一个整数$k$，表示在给定阈值下有多少件行李物品符合资格$t$。 

然后我们迭代所有可能的阈值时间$t$从 0 到最大可能距离。 对于每个$t$，我们构建有效图。 

对于固定的$t$，我们计算每个人有多少行李物品可以满足$a_i \le b_j - t$。 这给出了一个值$k_j$，该人的前缀邻域的大小。 

接下来我们通过增加对人进行排序$k_j$。 这种排序至关重要，因为它确保当我们处理一个人时，所有较早的人都没有更大的可行集，并且我们可以将使用过的行李视为以一致的方式消耗这些前缀。 

我们维护一个运行索引$i$，代表我们处理了多少人。 处理时$i$按此顺序的第一个人，我们检查在之前的作业可能消耗完之后还剩下多少个选择$i-1$可用池中的行李物品。 自由选择的个数是$k_j - (i-1)$。 该人可以选择其中之一或保持不匹配，这有助于$k_j - (i-1) + 1$到总计数。 

我们将这些贡献乘以所有人以获得$F(t)$。 求和$F(t)$全面的$t$给出了最终答案。 

### 为什么它有效

 在任何固定阈值$t$，每个有效的分配都是二分图中的匹配，其中每个人的邻接列表是已排序行李数组的前缀。 按前缀大小对人员进行排序使得匹配约束的行为就像对不可区分的容量进行贪婪分配。 关键的不变量是处理后$i$人，最多$i$行李物品可以被消耗，并且对于后来的人来说所有剩余的有效选择完全由剩余的前缀大小决定。 这消除了对已选择的行李箱物品的具体身份的任何依赖，只留下计数，这使得产品公式精确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    a.sort()
    b.sort()

    max_a = max(a)
    min_b = min(b)

    # possible t values are bounded by coordinate range
    max_t = 500

    ans = 0

    for t in range(max_t + 1):
        # compute k_j for each person: number of a_i <= b_j - t
        k = []
        for bj in b:
            limit = bj - t
            # count of a_i <= limit
            # binary search
            l, r = 0, n
            while l < r:
                mid = (l + r) // 2
                if a[mid] <= limit:
                    l = mid + 1
                else:
                    r = mid
            k.append(l)

        k.sort()

        ways = 1
        for i, ki in enumerate(k):
            # number of ways: choose unused or stay unmatched
            choices = ki - i + 1
            if choices <= 0:
                ways = 0
                break
            ways = (ways * choices) % MOD

        ans = (ans + ways) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对两个数组进行排序，以便前缀计数变得有意义。 对于每个阈值$t$，它使用对排序的行李位置进行二分搜索来计算每个人可以到达的行李数量。 然后对这些计数进行排序以匹配贪婪处理顺序。 

乘法循环直接实现了在前缀约束下计算匹配的派生公式。 表达式`ki - i + 1`代表剩余能力加上使该人不匹配的选项。 如果该值变为非正数，则该阈值不存在有效匹配。 

最后，所有跨阈值的贡献都会累积到最终答案中。 

## 工作示例

 ### 示例 1

 输入：```
2 2
1 2
3 4
```对于每个$t$，我们计算可达前缀。 

在$t = 0$，两个人都可以看到 2 个行李物品，所以$k = [2, 2]$。 排序后，贡献为：

 | 我的人 | 基 | ki - i + 1 |
 | --- | --- | --- |
 | 0 | 2 | 3 |
 | 1 | 2 | 2 |

 所以$F(0) = 6$。 

在$t = 1$，约束收紧并且存在更少的匹配，产生更小的$F(1)$。 总结一切$t$给出最终答案5。 

这显示了贡献如何随着阈值的增加而减少，因为剩余的边缘更少。 

### 示例 2

 输入：```
1 1
1
3
```在$t = 0$，存在一条有效边，所以$F(0) = 2$（匹配或不匹配）。 在$t = 1$，仍然有效，因此添加了另一个贡献。 在$t = 2$，边缘消失并且没有配置贡献。 

跨阈值的累积直接反映了单个可能的匹配存活的时间。

 ## 边缘情况

 当一个人没有达到给定阈值的行李时，就会出现极端情况。 例如，如果所有行李在减去后都严格位于一个人的右侧$t$， 然后$k_j = 0$。 在这种情况下，公式产生$k_j - i + 1 \le 0$，这正确地迫使$F(t) = 0$，因为没有有效的分配可以包括处于该阈值的该人。 

另一种情况是当许多人共享相同的位置时，导致相同的前缀大小。 排序步骤确保它们仍然以一致的顺序进行处理，并且可用选择的线性递减正确地模拟了对相同有限前缀池的竞争。 

最后一个案例很大$t$，所有边都消失。 每一个$k_j = 0$，因此除非没有人，否则乘积立即变为零，这与当无法到达行李时无法进行分配的事实相匹配。
