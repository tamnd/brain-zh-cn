 ## 解决方案
 ## 问题理解

 我们得到了一个按时间顺序排列的问题名称列表，其中每个名称由两个单词组成。 我们需要验证这个序列是否可以通过特定的选择规则产生。 

该规则描述了如何构建序列。 第一个问题可以是任何问题。 对于每个下一个位置，所选问题必须与之前选择的问题“相似”，或者，如果不存在未使用的相似问题，则可以选择任何未使用的问题。 

如果两个问题共享相同的第一个单词或相同的第二个单词，则认为它们相似。 因此，除了单词匹配之外，相似性在结构上并不对称：它纯粹基于二维标签中一个坐标的相等性。 

我们被要求检查给定的序列是否可以在该规则下产生，假设每个问题只使用一次。 

约束很大：每个测试最多 10^5 个名称，最多 10^4 个测试，全局总和为 10^5。 这会立即强制每个测试用例使用线性解决方案，因为任何重复扫描未使用的候选者或检查每个步骤的所有剩余项目的方法都会降级为二次行为并超出限制。 

一个微妙的困难是该规则是存在性的而不是确定性的。 在每一步中，如果存在至少一个未使用的相似问题，则法官可以选择其中的任何一个。 这意味着我们不需要重建唯一的路径； 我们只需要在每一步中验证先前的选择在可用的情况下是否有效。 

一个幼稚的错误是贪婪地假设如果存在任何类似的未使用问题，则下一个元素必须与前一个元素相似。 这是不正确的，因为当不存在类似的未使用的时，序列可以任意“跳跃”。 例如，如果在某个步骤中前一个问题没有共享剩余的第一个单词或第二个单词匹配，则下一个项目可能不相关，并且这仍然有效。 

另一个棘手的情况是，存在多个相似的候选者，但只有一些未被使用。 我们必须仔细跟踪哪些节点仍然可用。 

边缘情况包括：

 在一个序列中，相似性始终是可能存在的，但输入却打破了它。 

输入：```
2
a b
c d
```输出：```
Yes
```这是有效的，因为在第一个之后，不存在类似的问题，因此允许任何转换。 

不正确的推理会拒绝这一点，因为“a b”与“c d”不相似，但当不存在类似的未使用项目时，该规则允许任意选择。 

另一种边缘情况是存在相似性但序列不使用它时：

 输入：```
3
a x
a y
b x
```这里“a x”和“a y”相似，因此在第一个之后，第二个必须与“a x”相似，这是正确的。 在“a y”之后，剩余未使用的是“b x”，这也与第二个词x类似，因此存在有效的转换。 这是有效的。 

关键点是，有效性取决于每一步中至少存在一个未使用的相似候选者，而不是选择特定的一个。 

## 方法

 暴力模拟将保留一组未使用的问题。 在每一步中，我们都会扫描所有剩余未使用的项目，并检查是否至少有一个项目与当前问题共享第一个或第二个单词。 如果存在这样的项目，我们继续； 否则，我们检查不存在此类项目并允许任何下一步操作。 

这种简单的模拟是正确的，但每个步骤都需要扫描最多 O(n) 剩余项目，在最坏的情况下每个测试用例产生 O(n^2) 行为。 当 n 达到 10^5 时，这是不可能的。 

关键的观察是，我们永远不需要知道存在哪个相似元素，只需要知道是否至少存在一个。 这将问题转变为维护按第一个单词和第二个单词分组的剩余项目的计数。 

如果我们跟踪有多少未使用的问题共享每个第一个单词和每个第二个单词，我们可以在 O(1) 中检查当前节点是否存在“类似的未使用”候选者。 唯一的微妙之处是我们不能对当前节点本身进行重复计数，因此我们维护的计数在消耗元素时会递减。 

这减少了从搜索动态集到维护频率表的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力扫描每一步| O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 按单词频率计数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 我们读取所有词对并构建两个频率图：一个用于第一个单词，一个用于第二个单词。 这表示每个字坐标有多少未使用的项目可用。 
2. 我们按顺序迭代序列，将每个问题视为“当前”选择的节点，并且由于它被使用，我们从两个频率图中减少它的贡献。 
3. 在从位置 i 移动到 i+1 之前，我们检查根据规则是否可以合法选择下一个元素。 关键条件基于前一个元素：在过渡时刻是否存在至少一个与其类似的未使用问题。 
4. 对于位置 i 处的当前元素，我们计算有多少未使用的问题共享其第一个单词或第二个单词。 这是通过以下方式获得的：

 TotalSimilar = freqFirst[first[i]] + freqSecond[second[i]] - 重叠调整

 重叠调整是必要的，因为共享两个单词的元素会被计算两次。 我们仔细跟踪配对频率或递减以避免重复计算。 
5.如果i不是最后一个元素，并且totalSimilar为零，则意味着该规则将强制进行“自由选择”转换，但我们不能保证与给定序列结构的一致性。 我们返回“否”。 
6. 否则我们继续。 
7. 如果我们成功处理所有元素，我们将返回“Yes”。 

核心决定是，每当规则说“存在类似的未使用问题”时，在我们前进的那一刻，这必须是真的。 如果当序列隐式要求它为 true 时它为 false，则该序列无效。 

### 为什么它有效

 在每一步中，算法都会维护按第一个和第二个单词分组的剩余候选者的正确计数。 这确保了我们可以准确地确定序列中该点是否存在有效的“相似移动”。 不变的是，处理位置 i 后，频率图准确地表示序列中未使用的后缀。 因此，位置 i 处的任何检查都是针对真实的剩余集执行的，并且不会使用陈旧或不一致的信息做出转换决策。 这保证了只有当每个所需的“相似存在”条件始终成立时，序列才会被接受。

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        first = []
        second = []
        freq1 = {}
        freq2 = {}

        for _ in range(n):
            a, b = input().split()
            first.append(a)
            second.append(b)
            freq1[a] = freq1.get(a, 0) + 1
            freq2[b] = freq2.get(b, 0) + 1

        used1 = {}
        used2 = {}

        ok = True

        for i in range(n):
            a = first[i]
            b = second[i]

            freq1[a] -= 1
            freq2[b] -= 1

            # remaining similar candidates to current node
            similar = freq1[a] + freq2[b]

            # current node removed, so no need to subtract overlap explicitly:
            # counts already exclude current position after decrement

            if i < n - 1 and similar == 0:
                ok = False
                break

        out.append("Yes" if ok else "No")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现为第一个和第二个单词构建频率图。 当我们迭代时，我们从两个映射中减少当前元素，以便它们代表剩余的未使用的项目。 

删除后执行关键检查：如果我们不在最后一个元素，并且没有剩余元素与当前元素共享任一坐标，则规则将不允许强制“相似延续”步骤，因此序列无效。 

微妙的一点是，我们不需要显式地跟踪对或重叠校正，因为一旦删除当前元素，它就不再有助于相似性计数。

 ## 边缘情况

 一种重要的边缘情况是相似性仅通过一个坐标存在，但由于消费顺序而消失。 考虑：```
3
a x
b x
c y
```处理“a x”后，仍然有“b x”，因此存在相似性。 在“b x”之后，没有剩余项目共享第一个单词b或第二个单词x，因此下一个转换必须是自由选择。 该算法正确地检测到相似度恰好在该点变为零，并且仅当它是最后一个元素时才允许继续。 

另一种边缘情况是两个坐标之间存在多个重叠。 例如：```
3
a b
a b
a b
```尽管该语句不允许重复，但这证明了一个原则：每个步骤总是具有相似性，并且计数器永远不会提前降至零，因此该序列被接受。 

最后的边缘情况是最后一次强制转换的晚期断开连接：```
3
a b
a c
d e
```处理完第二个元素后，没有剩余项与“a c”的任一坐标匹配，因此如果存在第三个元素，则会违反规则。 该算法检测到这一确切条件并拒绝该序列。
