 ## 解决方案
 ## 问题理解

 我们被要求在小写英文字母上构造三个非空字符串，使得它们之间的成对距离匹配三个给定的整数。 该距离是编辑距离，表示将一个字符串转换为另一个字符串所需的插入、删除或字符替换的最小次数。 

因此，与其从“字符串”的角度思考，不如从编辑距离几何的角度思考更有用：我们希望将三个点放置在由编辑距离引起的度量空间中，并且给出这些点之间的精确的成对距离。 任务是确定这样的三元组是否存在，如果存在，则实际构造一个。 

对 a、b 和 c 的约束很小，最多为 1000。这排除了直接暴力破解字符串的任何尝试，因为即使是长度为 5000 超过 26 个字母的单个字符串也已经提供了天文数字般的搜索空间。 问题完全在于识别哪些度量三元组可以在 Levenshtein 度量中实现，然后显式地嵌入它们。 

一个关键的结构观察是 Levenshtein 距离的行为类似于字符串操作上的 L1 度量，对于精心构造的字符串，我们可以强制距离的行为几乎类似于树形结构中的路径长度。 这强烈表明可行性条件由三角不等式加上来自共享结构的类似奇偶校验的约束控制。 

如果我们假设三角不等式就足够了，就会出现一个幼稚但重要的失败案例。 例如，(a, b, c) = (1, 1, 2)满足三角不等式，但它实际上是可构造的，而像(1, 1, 1)这样的东西也是可构造的。 然而，(0, 0, 1) 显然失败了，因为两个相同的字符串不能同时与第三个字符串距离为 1，同时又彼此相同。 这已经暗示零距离的相互作用是不同的：如果两个距离为零，则相应的字符串必须相同，这会破坏自由度。 

因此，核心困难不仅仅是不等式检查，而是理解何时可以将三个度量约束嵌入到由字符串编辑生成的非常结构化的度量空间中。 

## 方法

 强力解释将尝试生成三个长度最大为 5000 的字符串，并测试它们的成对编辑距离是否与目标值匹配。 即使我们限制长度，候选字符串的数量也是指数级的，计算编辑距离本身就是 O(nm)，这太慢了。 这种方法立即失败。 

关键的见解是停止考虑任意字符串，而是以受控的“基础+装饰”形式构造字符串，其中可以根据我们设计的重叠和不匹配准确计算编辑距离。 Levenshtein 构造问题的标准技巧是使用由相同字符的长块组成的字符串，这些字符串经过排列以便距离分解为独立的贡献：共享前缀、后缀和强制替换或插入的不相交字符块。 

这将问题简化为从公共基本字符串构建三个路径，其中每个成对距离对应于独立段差异的总和。 我们通过分配不相交的字符区域来表示每个边缘贡献，有效地将加权三角形嵌入到字符串结构中。 

一旦我们采用这个观点，问题就变成了一个可行性检查，即边长为 a、b、c 的三角形是否可以分解为“星形”表示：选择中心字符串 s，并将 t 和 u 构造为 s 的修改，以使距离对齐。

我们选择 s 作为基本字符串，并通过插入和替换不相交的块来表示 t 和 u。 基本约束是成对距离必须与将每条边分割为 t 和 u 相对于 s 中引入的差异的贡献一致。 

这导致了已知的特征：当且仅当三角不等式成立并且 a + b + c 之和为偶数时，构造才是可能的。 奇偶校验条件的出现是因为仔细对齐的结构中的每次替换都会对称地影响两个成对的距离，迫使总距离和相等。 

一旦知道可行性，构造就变得明确：我们分配与三个字符串之间的共享和独占编辑相对应的段。

 ## 算法演练

 我们通过显式模拟如何使用不相交的字符块形成它们的成对距离来构造三个字符串。 

1. 检查三角不等式：确保a + b ≥ c、a + c ≥ b、b + c ≥ a。 如果出现任何问题，则无法进行施工。 这是必要的，因为编辑距离是一个度量。 
2. 检查奇偶校验：计算(a + b + c) mod 2。如果为1，则输出“No”。 此约束确保替换和插入的对称贡献可以在三个成对距离上平衡。 
3. 不失一般性地假设我们构造一个基串 s 并通过受控修改导出 t 和 u。 我们根据共享结构来解释距离：t 和 u 与 s 不同的部分独立地影响成对距离。 
4. 将距离分解为共享贡献和独占贡献。 让：

 x = (a + b - c) / 2,

 y = (a + c - b) / 2,

 z = (b + c - a) / 2。 

这些表示每个成对距离中有多少是由与第三个字符串的偏差“解释”的。 
5. 使用不同的字符块构建长度为 x + y + z 的基本字符串 s。 分配：

 区分 s 和 t 的 x 位置，

 区分 s 和 u 的 y 位置，

 z 位置通过 s 间接区分 t 和 u。 

我们使用三个不同的字符，例如“a”、“b”和“c”，但由于只允许使用小写字母，因此我们要小心地循环或重用。 
6. 将 s 构造为中性字符的重复，例如“a”。 然后通过修改 x 位置为新字符来构造 t，通过修改 y 位置来构造 u，并确保 t 和 u 之间的 z 位置一致。 
7. 输出构造好的字符串。 

### 为什么它有效

 分解确保对之间的每个距离单位都被恰好计算一次。 每个段独立地贡献恰好一个或两个成对距离，具体取决于它的分配方式。 三角形不等式保证段长度非负，而奇偶校验保证分解为半和得到整数。 由于每个修改都局限于受控段，因此生成的编辑距离恰好加起来为 a、b 和 c，而段之间不会发生干扰。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, c = map(int, input().split())

    if a + b < c or a + c < b or b + c < a:
        print("No")
        return

    if (a + b + c) % 2:
        print("No")
        return

    x = (a + b - c) // 2
    y = (a + c - b) // 2
    z = (b + c - a) // 2

    # Construct three strings using disjoint segments
    # We use 'a', 'b', 'c' as markers
    s = []
    t = []
    u = []

    # x contributes to s-t difference
    for _ in range(x):
        s.append('a')
        t.append('b')
        u.append('a')

    # y contributes to s-u difference
    for _ in range(y):
        s.append('a')
        t.append('a')
        u.append('b')

    # z contributes to t-u difference
    for _ in range(z):
        s.append('a')
        t.append('b')
        u.append('c')

    print("Yes")
    print("".join(s))
    print("".join(t))
    print("".join(u))

if __name__ == "__main__":
    solve()
```该实现直接对分解进行编码。 每个循环对应于变量 x、y、z 之一，附加字符，以便每个块恰好增加一对距离。 最后一个块使用第三个字符来强制 t 和 u 之间的分隔，而不影响 s 的对称性。 

最微妙的部分是确保每个部分独立做出贡献。 这是通过使所有块独立的单个位置来实现的，因此编辑距离减少到计数不匹配，因为在构造中不需要插入或删除。

 ## 边缘情况

 一种边缘情况是两个距离为零，例如 a = b = 0 和 c = 1。这迫使 s = t 且 s = u，这意味着 t = u，与 c = 1 相矛盾。该算法通过三角不等式捕获了这一点，因为 0 + 0 < 1 立即失败。 

另一种边缘情况是当一个值相对于其他值极大时，例如 (1, 1, 1000)。 即使三角不等式失败，简单的构造方法仍然可能尝试贪婪地分配不匹配，从而产生不一致的重叠。 在这种情况下，检查可以防止进入施工。 

像 (2, 2, 2) 这样的对称情况充分利用了所有三个段 x, y, z。 该构造仍然有效，因为每个部分的贡献相等，并且没有段变为负值，从而确认分解在相等边界上仍然有效。
