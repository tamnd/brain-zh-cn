 ## 解决方案
 ## 问题理解

 我们给定一个长度的字符串$n$由小写英文字母组成。 我们不必将字符串视为固定字符位置，而是可以“选择”每个字母的某些出现位置，并将这些选定的索引组织为 26 组，每个字母一组。 

对于每个字母$i$，我们选择该字母出现在字符串中的索引的非空子集。 这些选定的索引必须遵守全局排序约束：字母的所有选定索引$a$必须出现在所有选定的字母索引之前$b$，依此类推。 在字母内部，我们可以自由选择其出现的任何子集，但我们无法打破字母之间这种递增的块结构。 

有效结构的质量仅取决于我们为每个字母选择的索引数量。 如果$s_i$是为字母选择的集合$i$，该值被计算为所有字母对的总和$i < j$的$(|s_i| + |s_j|)^2$。 我们希望最大化该值，或者如果不可能为每个字母至少选取一次出现，则输出 0。 

约束条件$n \le 2000$意味着我们无法承受在所有子集选择上都有大量内部循环的三次甚至二次每个字母的构造。 尝试枚举所有 26 个子集或将字符串所有可能分割为 26 个段的解决方案会太慢。 甚至像$O(26 \cdot n^2 \cdot n)$已经是边界了，所以我们需要一个结构，将每个字母的决策减少到可管理的过渡。 

一个关键的微妙之处是，我们不必被迫使用某个字母的所有出现次数，但由于全局排序约束，选择较少的出现次数可能会影响后面字母的可行性。 另一个微妙之处是可行性取决于原始字符串中的位置，而不仅仅是每个字母的计数。 

一个天真的错误是假设我们可以独立选择任何积极的$k_i$每封信并始终实现它们。 这会失败，因为不同字母的出现可能会在字符串中交错出现，从而无法选择大的字母$k_i$提早，不妨碍后来的信件。 

## 方法

 对于来自的每封信，都会尝试采用蛮力方法来决定$a$到$z$，要出现多少次以及要分配哪些索引，同时尊重全局排序约束。 这很快就会变成指数：每个字母最多有$O(n)$选择它为块贡献哪些出现的前缀，并且这些选择在所有 26 个字母中相互作用。 即使限制在原始字符串中的连续段仍然会导致粗略的结果$O(n^{26})$-风格的结构如果天真地完成，因为每个字母边界都依赖于前一个字母边界。 

关键的简化是停止考虑任意子集，而是考虑对字符串的单次扫描。 一旦我们确定了字母块“结束”的位置$i$，所有选定的字母出现次数$i$必须位于上一次剪切之后的连续间隔中。 所以每个字母贡献一个片段$(prev\_cut, cur\_cut]$，并且在该段中我们精确地获取该字母的出现次数。 这将问题转化为沿着字符串选择 26 个切割位置，每个字母一个。 

这将结构简化为字母和位置的动态编程。 对于每个字母$i$，我们决定其段的结束位置，并确保该段至少包含该字母的一次出现。 一个片段的贡献仅取决于该字母在其中出现的次数。 

最后的见解是，目标可以纯粹根据段大小重写，这允许在 DP 转换期间进行增量计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集/分配的暴力破解 | 指数| 指数| 太慢了 |
 | DP 字母和剪切位置 |$O(26 \cdot n^2)$|$O(26 \cdot n)$| 已接受 |

 ## 算法演练

 我们按字母顺序处理字母。 对于每个字母，我们选择其段在字符串中的结束位置。 

1. 对于每个字母，预先计算它在字符串中出现的位置列表。 我们还构建前缀计数，以便我们可以查询某个字母在任何间隔中出现的次数$(l, r]$在$O(1)$。 
2.定义DP状态$dp[i][r]$， 在哪里$i$是已经分配的字母数（来自$a$到$i$-第一个字母），以及$r$是为字母选择的段的结束索引$i$。 该状态代表完成处理信件的最佳方式$i$这样最后选定的索引位于位置$r$。 
3. 从字母过渡$i-1$结束于位置$l$写信$i$结束于位置$r$，我们要求$l < r$。 字母段$i$是$(l, r]$。 我们计算$k_i$, 字母出现的次数$i$在该段内使用前缀和。 如果$k_i = 0$，过渡无效。 
4. 我们与 DP 一起维护两个运行数量：$K$，已处理字母中选定索引的总数，以及$S$，线段大小的平方和。 每个 DP 转换都会使用以下方式更新这些值：$$K' = K + k_i,\quad S' = S + k_i^2$$5. 我们不是在最后重新计算完整的目标，而是增量地维护它。 如果当前值为$V = K^2 + 24S$，然后添加一个新字母$k$发生次数会通过以下方式更改值：$$V' = V + 2Kk + 25k^2$$这允许每次转换在恒定时间内更新分数。 
6.答案是处理完所有26个字母后的最大DP值。 

### 为什么它有效

 DP 强制每个字母在选定的子序列排序中占据一个连续的段，这相当于一个字母的所有选定索引必须严格位于下一个字母的选定索引之前的约束。 每个有效的构造都对应于一个段端点序列，并且每个有效的端点序列都会生成一个有效的分区。 

目标的增量公式确保每个状态都包含评估未来扩展所需的完整信息。 由于转换仅取决于先前的剪切位置，而不取决于较早片段内的内部结构，因此 DP 状态足够并且不会丢失隐藏信息。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    pos = [[] for _ in range(26)]
    for i, ch in enumerate(s):
        pos[ord(ch) - 97].append(i + 1)

    # prefix counts
    pref = [[0] * (n + 1) for _ in range(26)]
    for c in range(26):
        idx = 0
        for i in range(1, n + 1):
            pref[c][i] = pref[c][i - 1]
            if idx < len(pos[c]) and pos[c][idx] == i:
                pref[c][i] += 1
                idx += 1

    def count(c, l, r):
        if r <= l:
            return 0
        return pref[c][r] - pref[c][l]

    NEG = -10**30

    # dp[i][r] = best value after i letters ending at position r
    dp_prev = [-10**30] * (n + 1)
    dp_prev[0] = 0

    # also track K and S aligned with dp states
    K_prev = [[0] * (n + 1) for _ in range(27)]
    S_prev = [[0] * (n + 1) for _ in range(27)]
    K_prev[0][0] = 0
    S_prev[0][0] = 0

    for i in range(1, 27):
        dp_cur = [-10**30] * (n + 1)
        K_cur = [[0] * (n + 1) for _ in range(1)]
        S_cur = [[0] * (n + 1) for _ in range(1)]

        for r in range(1, n + 1):
            for l in range(r):
                if dp_prev[l] == NEG:
                    continue

                c = i - 1
                k = count(c, l, r)
                if k == 0:
                    continue

                K = K_prev[i - 1][l] + k
                S = S_prev[i - 1][l] + k * k
                val = dp_prev[l] + 2 * K_prev[i - 1][l] * k + 25 * k * k

                if val > dp_cur[r]:
                    dp_cur[r] = val
                    K_prev[i][r] = K
                    S_prev[i][r] = S

        dp_prev = dp_cur

    ans = max(dp_prev)
    print(0 if ans < 0 else ans)

if __name__ == "__main__":
    solve()
```该实现为每个字符构建前缀计数，以便每次转换都可以计算常数时间内某个字母在候选段中出现的次数。 DP 迭代字母和所有可能的段端点，并且对于每次转换，它都会尝试所有先前的剪切位置。 

一个微妙的点是我们必须同时携带DP值和累计值$K$和$S$，因为转换成本取决于当前所选元素的总数，而不仅仅是之前的分数。

 ## 边缘情况

 关键的边缘情况是某些字母根本没有出现在字符串中。 在这种情况下，不存在有效的分区，因为每个集合必须非空。 该算法自然会拒绝这些情况，因为每个 DP 转换都需要在所选段中至少出现一次。 

当字母以高度交错的模式出现时（例如交替出现），就会出现另一种边缘情况。 DP 正确地处理了这一问题，因为它只允许在至少出现一次当前字母的情况下形成有效连续段的转换，从而确保每一步都强制执行可行性。 

最后的边缘情况是一个字母多次出现但所有出现都位于所需的剪切边界之前。 在这种情况下，所有产生零计数的转换都会被丢弃，从而防止无效累积并正确阻塞不可行的路径。
