 ## 解决方案
 ## 问题理解

 我们间接获得了每日解决计数的序列。 原始数组包含每天解决了多少个问题，但只提供了最近几天的总和。 对于第一个`k`天，该值表示从月初到当天已解决的问题总数。 之后，每个值都代表前一个值的总和`k`日子在那一天结束。 

任务是恢复有关原始每日计数的足够信息，以找到每天都有正解决计数的最长连续间隔。 

这些约束迫使我们寻求线性解决方案。 所有测试用例的总天数最多为`3 * 10^5`，所以一个`O(n)`或者`O(n log n)`方法很现实。 检查每个可能的间隔或重复重建总和的解决方案将需要大约`O(n^2)`最坏情况下的操作，远远超出了2秒的限制。 

一些边缘情况可能会破坏粗心的实现。 什么时候`k = n`，整个数组只是前缀和，因此将每个值视为正常的滑动窗口和会给出不正确的差异。 例如：```
Input:
1
3 3
1 2 2
```原来的日子是`[1, 1, 0]`，所以答案是`2`。 错误的仅滑动窗口方法可能会失败，因为在第一个窗口之后没有完整的窗口`k`天。 

另一个棘手的情况是滑动总和减少时。 减少并不直接意味着当天的解决方案为零，因为旧的正日可能会离开窗口。 例如：```
Input:
1
5 2
1 2 1 0 0
```原来的日子是`[1, 1, 0, 0, 0]`，所以答案是`2`。 仅查看连续值是否增加或减少可能会错误地对天进行分类。 

第三种情况是所有日子都是正数时：```
Input:
1
5 2
1 2 2 2 2
```答案是`5`。 搜索给定数组中的变化而不是重建每日计数的方法会错误地提前停止连续计数。 

## 方法

 直接的方法是重建原始的每日计数。 设隐藏数组为`x`。 对于第一个`k`天，给定的值是前缀和，因此可以通过减去前一个前缀和来找到每一天。 天后`k`，当前值是窗口总和。 删除前一个窗口并添加新的一天给出以下关系：`a[i] - a[i-1] = x[i] - x[i-k]`所以：`x[i] = a[i] - a[i-1] + x[i-k]`这让我们可以在恒定时间内计算每个隐藏值。 

暴力解决方案可以首先重建完整的阵列，然后通过从每个起始日开始扩展来测试每个可能的连胜。 这是正确的，因为每个可能的间隔都会被检查，但在最坏的情况下它会执行`O(n^2)`工作。 和`n = 300000`，这可能意味着大约 900 亿次操作。 

每个隐藏日仅取决于值的观察结果`k`较早的立场完全改变了问题。 我们不需要重复搜索区间。 在重建天数的同时，我们可以保持当前的连胜长度。 每当恢复值为正时，条纹就会增长。 每当它为零时，连胜就会结束，我们就会更新最佳答案。 由于每天都处理一次，所以整个问题变成线性的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 创建数组`days`这将存储每天恢复的已解决问题的数量。 以后几天的重复需要以下值`k`几天前，因此保留以前的值可以使转换成为可能。 
2.恢复第一个`min(n, k)`使用前缀和的天数。 第一天等于`a[0]`。 此前缀区域中的每一天都是`a[i] - a[i-1]`。 这些值代表实际的每日求解，因为给定值是累积总数。 
3. 第一天之后的每一天`k`天，使用计算隐藏值`a[i] - a[i-1] + days[i-k]`。 减法表示离开窗口的旧日，加法表示`days[i-k]`恢复被删除的贡献。 
4.每天在恢复的同时，更新当前的连胜记录。 如果恢复的值大于零，则增加当前条纹并更新最大值。 如果为零，则重置当前连续记录，因为零解决日会中断连续解决。 
5. 打印处理完所有天后找到的最大条纹。 

为什么有效：重建公式完全遵循每个滑动窗口的定义。 对于第一个`k`天，前缀差异给出了唯一可能的每日值。 对于以后的日子，两个相邻窗口总和之间的差异告诉我们删除旧一天并添加新一天所引起的变化。 由于保证数组有效，因此重建的值正是原始的每日计数。 然后，连续计算直接测量最长的正值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    ans = []

    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))

        days = [0] * n
        best = 0
        cur = 0

        for i in range(n):
            if i == 0:
                days[i] = a[i]
            elif i < k:
                days[i] = a[i] - a[i - 1]
            else:
                days[i] = a[i] - a[i - 1] + days[i - k]

            if days[i] > 0:
                cur += 1
                if cur > best:
                    best = cur
            else:
                cur = 0

        ans.append(str(best))

    print("\n".join(ans))

if __name__ == "__main__":
    solve()
```数组`days`存储重建的每日解决计数。 它是从零开始索引的，所以`days[i-k]`是计算日期时离开滑动窗口的日期`i`。 

第一个分支单独处理第零天，因为没有先前的前缀和。 第二个分支涵盖输入的前缀和部分，其中相邻值的减法给出了准确的每日计数。 最后一个分支使用滑动窗口递归。 

Python 整数是自动使用的，因此可能的值很大`a[i]`不要造成溢出问题。 循环中的操作顺序也很重要：必须通过加回即将到来的一天的贡献来纠正先前的窗口差异。

 ## 边缘情况

 当`k = n`，每个值都是一个前缀和。 为了：```
1
3 3
1 2 2
```该算法仅使用前缀重建分支。 它获得天数`[1, 1, 0]`，将连胜更新为`2`，然后清零后复位。 输出是`2`。 

当窗口减小时，算法不会假设新的一天为零。 为了：```
1
5 2
1 2 1 0 0
```恢复的天数是`[1, 1, 0, 0, 0]`。 重现正确地解释了离开窗口的旧日，所以答案仍然存在`2`。 

当每一天都是积极的时候，就没有重置。 为了：```
1
5 2
1 2 2 2 2
```每个重构日等于`1`，所以当前的连续增长从`1`到`5`最终的答案是`5`。 该算法处理此问题是因为它检查重建的每日值而不是输入数组中的模式。
