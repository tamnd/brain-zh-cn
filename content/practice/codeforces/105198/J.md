 ## 解决方案
 ## 问题理解

 棋盘是无限的，因此唯一重要的信息是起始方格和所选目标方格之间的相对位置。 对于每个测试用例，我们都会获得骑士初始位置的两个坐标和目标位置的两个坐标。 我们需要确定是否存在一种方法可以让马使用奇数步移动到达目标。 如果存在这样的路径，获胜者就是蒙克。 否则，土豆获胜。 

坐标可以大到$10^6$，测试用例数量可以达到$10^5$。 这立即排除了任何探索棋盘的模拟，因为即使是围绕一个位置的小型广度优先搜索也会访问许多方块并重复该过程$10^5$时间将超过可用的操作。 我们需要直接根据坐标进行恒定时间观察。 

棘手的部分是棋盘是无限的，因此天真的方法可能会尝试推理最短的骑士路径或构建搜索图。 答案并不取决于确切的距离。 它仅取决于骑士图的着色属性。 

一个常见的错误是只检查一步是否可以达到目标。 例如，输入`1 1 4 5`是一种情况，目标距离马不远，但答案仍然是蒙克，因为马可以用奇数步到达目标。 另一个错误是假设以偶数次移动到达一个正方形意味着也可能存在奇数路线。 为了`1 1 1 3`，坐标颜色相同，因此每条可能的路线长度都是偶数。 正确的输出是`Potato`。 

## 方法

 直接的暴力解决方案是从起始方块开始执行图搜索。 每个方格都是一个节点，骑士的每一步移动都会创建一条边。 广度优先搜索可以找到到目的地的最短距离，如果该距离是奇数，则 Monke 获胜。 由于未加权图中的最短路径是通过 BFS 找到的，因此这种方法是正确的。 

问题在于图表的大小。 尽管我们只需要一个目的地，但无限板上的 BFS 可能必须检查大量方格。 和$10^5$测试用例中，任何基于搜索的方法都是不可能的。 在最坏的情况下，如果搜索访问$K$每个测试用例的平方，总工作量变为$O(10^5K)$，远远超出了极限。 

关键的观察来自观察棋盘正方形的颜色。 正方形可以通过奇偶校验来着色$x+y$。 骑士的每一个动作都会发生变化$x+y$奇数，因为移动将一个坐标更改 1，另一个坐标更改 2。添加这些更改始终会给出奇数。 结果，马的每一步移动都会改变方块的颜色。 

这意味着骑士图是二分图。 移动次数为奇数的路径必须以相反的颜色结束，移动次数为偶数的路径必须以相同的颜色结束。 剩下的问题是每个相反颜色的方块是否都是可达的。 马可以到达无限棋盘上的每个方格，因此色差是唯一重要的条件。 

暴力破解之所以有效，是因为它直接遵循图定义，但由于图太大而失败。 着色属性将整个图压缩为单个奇偶校验。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(K)$每个测试用例，其中$K$是探索过的方块的数量 |$O(K)$| 太慢了 |
 | 最佳 |$O(1)$每个测试用例|$O(1)$| 已接受 |

 ## 算法演练

 1. 使用以下命令计算起始方块的颜色$(x_1+y_1)\bmod 2$。 不需要总和的精确值，只需要它是奇数还是偶数，因为棋盘着色每两个方格重复一次。 
2. 使用以下命令计算目标方块的颜色$(x_2+y_2)\bmod 2$。 这给出了骑士必须完成的颜色类别。 
3. 比较两种颜色。 如果不同，则马走了奇数步，所以打印`Monke`。 如果它们相同，则每个可能的路径的长度都是偶数，因此打印`Potato`。 

为什么有效：骑士的每一步举动都会改变$x+y$。 一次移动后颜色会翻转，两次移动后颜色会恢复，并且这对于所有路径长度都会持续。 目标颜色完全决定了到该方块的每个可能路径长度的奇偶性。 由于无限骑士棋盘上的所有方格都可以到达，因此相反的颜色恰好对应于奇数次移动中可以到达的位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    ans = []

    for _ in range(t):
        x1, y1, x2, y2 = map(int, input().split())

        if (x1 + y1) % 2 != (x2 + y2) % 2:
            ans.append("Monke")
        else:
            ans.append("Potato")

    sys.stdout.write("\n".join(ans))

if __name__ == "__main__":
    solve()
```输入一次处理一个测试用例，即使有很多用例，也能保持内存使用量恒定。 该程序仅在将输出字符串打印在一起之前存储它们，从而避免了许多单独输出操作的开销。 

表达式`(x1 + y1) % 2`is the entire core of the solution. 由于只有奇偶性很重要，因此较大的坐标值不会造成任何困难。 Python 整数还避免了溢出问题，尽管这里的最大可能总和对于其他语言中的标准整数类型来说已经足够小了。 

比较是直接在两个正方形颜色之间进行的。 无需计算距离、生成骑士移动或处理特殊坐标情况，因为无限棋盘消除了边界效应。 

## 工作示例

 使用示例案例：

 | 开始| 目标| 开始奇偶校验| 目标奇偶校验| 结果 |
 | --- | --- | --- | --- | --- |
 | (1,1) | (4,5) | 0 | 1 | 蒙克 |
 | (100,2)| (9,11) | (9,11) | 0 | 0 | 马铃薯|
 | (1,1) | (1,2) | 0 | 1 | 蒙克 |

 第一个案例表明，答案并不是让一个骑士离开。 颜色不同，因此存在一些奇数长度的路线。 

第二种情况表明，具有相同颜色的方块不能通过奇数步移动来连接。 马的每次移动都会翻转颜色，因此返回到相同颜色需要翻转偶数次。 

第三种情况是最简单的反色情况。 马可能需要多次移动，但奇偶条件保证了奇数长度路径的存在。

 ## 边缘情况

 对于本案`1 1 1 1`，两个平方都有奇偶性$(1+1)\bmod 2=0$。 该算法比较相同的颜色并输出`Potato`。 假设零移动计数为有效奇数路径的粗心实现将错误地打印`Monke`。 

对于本案`1 1 1 3`，两个平方也有偶校验。 马不能以奇数步到达，因为每一步都会翻转颜色，而奇数次翻转不能恢复到原来的颜色。 该算法立即检测到相等奇偶校验并输出`Potato`。 

对于本案`1000000 1000000 1 2`，起始方格具有奇偶性$0$，而目的地有奇偶校验$1$。 这些值很大，但只有坐标和的最后一位很重要。 该算法找到不同的颜色并输出`Monke`不依赖于电路板尺寸。
