 ## 解决方案
 ## 问题理解

 每次测试都会给出在 7 段显示屏上以亮起的段测量的目标“能量单位”量。 每个十进制数字点亮时消耗固定数量的段。 例如，数字`1`很便宜，因为它只使用两个段，而数字`8`价格昂贵，因为它使用了全部七个。 

任务是构造一个有效的整数，其数字总共消耗恰好`N`点亮的段。 在通常的数字意义上，该数字必须尽可能小。 这意味着我们按值比较整数，因此较少的数字通常会有所帮助，但并不总是如此，因为以较长的数字开头`1`仍然可以从较大的数字开始击败较短的数字。 

一个关键限制是数字不能有前导零，除非整个数字为零。 由于输入保证`N ≥ 2`，我们实际上从未产生过零，因此第一个数字必须来自`1`到`9`。 

约束允许最多`10^4`测试用例和总数`N`在所有测试中直至`10^6`。 这强烈表明`O(N)`跨查询重用的预处理解决方案。 任何平方项`N`立即被排除，因为它会达到大约`10^12`最坏情况下的操作。 

如果我们在不检查可行性的情况下尝试贪婪的局部选择，就会出现微妙的失败情况。 例如，当剩余的部分无法准确完成时，选择适合每个步骤的最小数字可能会让我们陷入困境。 另一个失败案例是将“最小化位数”作为主要目标。 这种情况之所以会失败，是因为数字成本各不相同，因此数字越少并不意味着价值越小。 

## 方法

 直接的暴力方法将尝试每个可能的数字序列，其分段成本总和为`N`，然后比较结果整数。 由于每个数字最多可以有 10 种选择方式，并且解的长度大约可以达到`N / 2`，这会组合爆炸。 即使是有限的计数也会呈指数增长，远远超出可行的限度。 

关键的观察是数字构造仅取决于剩余的段和，而不取决于先前的结构。 这是一个经典的无界硬币找零可达性问题，其中数字是硬币，分段计数是硬币重量。 

我们首先计算使用任何数字序列可以实现哪些段的总数。 一旦我们知道哪些余数是可行的，我们就可以安全地对每个数字位置做出贪婪的选择：我们选择仍然留下可行余数的最小数字。 

这是可行的，因为可行性与数字顺序无关，并且一旦固定了前缀，后缀问题与原始问题相同，但总和更小，并且不再有前导零限制。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举| 指数| O(N) 递归堆栈 | 太慢了 |
 | DP + 贪心构造| O(N·10) | O(N) | 已接受 |

 ## 算法演练

 我们依赖于数字的标准段成本：`0→6, 1→2, 2→5, 3→5, 4→4, 5→5, 6→6, 7→3, 8→7, 9→6`。 

### 1. 预先计算段和的可行性

 我们构建一个布尔数组`dp[x]`意思是“可以使用任何数字精确地形成 x 段”。 

我们初始化`dp[0] = true`。 然后对于每个总和`i`最多`maxN`，我们尝试添加每一位数字的成本`c`并更新`dp[i] |= dp[i - c]`。 

这是一个无界的背包，因为数字可以任意重复使用。 

### 2. 为每个测试贪婪地构建答案

 对于给定的`N`，我们从左到右构建数字。 

我们维持剩余金额`rem = N`。 

对于第一个数字，我们必须选择`1`到`9`。 我们按升序尝试数字并选择第一个数字`d`这样`rem - cost[d]`是可行的，根据`dp`。 

我们减去它的成本并继续。 

### 3. 构建剩余数字

 在第一个数字之后，允许有零，因此我们再次尝试以下数字`0`到`9`。 在每一步中，我们都会选择使余数保持可行的最小数字。 

我们重复直到`rem`变为零。 

### 为什么它有效

 DP保证每个剩余段总和的全局可行性。 当我们贪婪地选择一个数字时，我们不会冒未来不可能的风险，因为可行性仅取决于剩余的总和，而不是数字序列。 由于我们总是选择保留可行性的最小可能数字，因此任何替代选择都会产生更大的前导数字或违反可行性，这两种情况在字典数字排序下都更糟。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

cost = [6,2,5,5,4,5,6,3,7,6]

MAXN = 10**6

dp = [False] * (MAXN + 1)
dp[0] = True

for i in range(1, MAXN + 1):
    for d in range(10):
        c = cost[d]
        if i >= c and dp[i - c]:
            dp[i] = True
            break

def build(n):
    res = []
    
    # first digit: no leading zero
    for d in range(1, 10):
        c = cost[d]
        if n >= c and dp[n - c]:
            res.append(str(d))
            n -= c
            break

    # remaining digits: allow zero
    while n > 0:
        for d in range(10):
            c = cost[d]
            if n >= c and dp[n - c]:
                res.append(str(d))
                n -= c
                break

    return ''.join(res)

t = int(input())
for _ in range(t):
    n = int(input())
    print(build(n))
```预处理步骤填充`dp`对所有测试用例进行一次，确保每个查询在相对于生成的位数的线性时间内独立处理。 贪心构造之所以有效，是因为在每一步中我们都会明确验证剩余部分是否位于 DP 可达集合内。 

一个常见的实施陷阱是忘记该数字`0`仅在第一个位置被禁止。 另一种是试图贪婪地最小化数字计数而不是数值，这会导致不正确的输出，例如偏好`8`超过`11`在某些段配置中。 

## 工作示例

 ### 示例 1：`N = 7`| 步骤| 剩余| 选择的数字| 成本| 剩余新 |
 | --- | --- | --- | --- | --- |
 | 1 | 7 | 8 | 7 | 0 |

 以个位数消耗 7 个段的唯一可行方法是`8`。 任何其他数字都会留下无法完成的余数，因此 DP 会强制进行选择。 输出是`8`。 

### 示例 2：`N = 16`| 步骤| 剩余| 选择的数字| 成本| 剩余新 |
 | --- | --- | --- | --- | --- |
 | 1 | 16 | 16 1 | 2 | 14 | 14
 | 2 | 14 | 14 8 | 7 | 7 |
 | 3 | 7 | 8 | 7 | 0 |

 第一个数字被强制为`1`因为任何以以下开头的较小成本数字选择`2`或更高最终会阻碍可行性或导致更大的数量。 修复后`1`，其余结构的行为类似于标准划分问题，并且`8`成为后缀的最佳选择。 

该轨迹显示了可行性和字典最小化如何相互作用：早期的贪婪选择是安全的，只是因为 DP 保证了完成。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(N·10) | DP 总和最大为 N，具有 10 位数字转换，加上每次测试的 O（位数）|
 | 空间| O(N) | 布尔 DP 数组存储所有总和的可达性 |

 总计`N`跨测试的边界是`10^6`，因此预处理和每次测试重建都在限制范围内。 DP在最大和上是线性的，每次测试构造一个长度最多为`O(N)`在最坏的情况下，但在所有测试中摊销。

 ## 边缘情况

 第一个边缘情况来自非常小的段计数，其中只能有一个数字。 为了`N = 2`，唯一有效的数字是`1`。 民主党证实`2`只能通过数字访问`1`，所以算法直接输出它。 

另一种边缘情况是，即使存在很大的数字，最佳解决方案也需要多个数字。 为了`N = 16`，选择`8`首先会离开`9`，这是不可能完全完成的，因此 DP 拒绝该分支并强制使用前缀`1`。 随后施工将安全进行。 

第三个边缘情况是前导数字规则和可行性之间的相互作用。 如果最小的可行数字是`0`，在第一个位置将无效。 该算法明确将第一选择限制为数字`1`到`9`，确保不会出现无效的前导零，即使 DP 允许出现这种情况也是如此。
