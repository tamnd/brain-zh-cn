 ## 解决方案
 ## 问题理解

 我们得到一个长度的数组$n$， 在哪里$n$是奇数，我们可以重复应用一个非常不寻常的全局运算：选择一个正整数$x$，并将数组的每个元素与$x$一击。 本次操作不针对个人仓位； 它在按位异或的情况下以锁步方式移动整个数组。 

目标是确定在经过一些此类全局 XOR 移位序列之后，数组是否可以成为以下数字的排列：$1$到$n$。 由于排列要求该范围内的所有整数恰好出现一次，因此最终状态非常严格，而我们允许的操作非常灵活，但也是全局统一的。 

约束条件很大，有$n$最多$10^6$和值高达约$2 \cdot 10^6$。 这立即排除了任何试图显式模拟转换或探索多个 XOR 选择的方法。 甚至枚举可能的值$x$是不可行的，因为$x$可以任意大，并且 XOR 的效果在集合结构方面是高度非线性的。 

一个微妙的边缘情况是当所有元素已经不同但超出范围时$1$到$n$。 例如，如果$a = [100, 101, 102]$和$n = 3$，元素不同是不够的，因为没有单个 XOR 移位可以将它们精确地映射到$\{1,2,3\}$。 另一个误导性的情况是，当数组已经包含排列但不按排序顺序时，答案很简单是“是”，但关于转换的天真的推理可能仍然会尝试不必要的操作。 

关键的困难是理解在重复的全局异或运算下保留了什么结构。 

## 方法

 一种强力的解释是考虑应用 XOR 掩码序列。 每个操作都适用$a_i \leftarrow a_i \oplus x$为所有人$i$，并应用两个操作$x$和$y$相当于单个操作$x \oplus y$。 这意味着整个过程归结为最多选择一个有效的 XOR 掩码：重复操作不会创建超出单个累积 XOR 值的任何内容。 

所以问题就简化为询问是否存在一个整数$x$使得多重集$$\{a_1 \oplus x, a_2 \oplus x, \dots, a_n \oplus x\}$$正是$\{1,2,\dots,n\}$。 

暴力解决方案会尝试所有可能的方法$x$直到某个界限（例如输入范围中存在的最大值），并且对于每个$x$检查对整个数组进行异或是否会产生排列。 每张支票费用$O(n)$，并与$x$可能跨越$2^{21}$，这变得太大了。 

关键的观察结果是 XOR 是可逆的并且充当整数上的双射。 这意味着我们正在寻找的结构不是关于单个值，而是关于在单个异或移位下匹配整个集合。 如果这样一个$x$存在，然后使用相同的值再次应用 XOR$x$返回到原始数组，因此原始数组和目标排列之间的关系是对称的。 

这表明重新构建问题：我们要问的是，集合是否$\{1,2,\dots,n\}$可以异或移位到给定的多重集。 这意味着多重集差异完全可以通过统一的 XOR 掩码来解释。 

一个关键的结构结果是按结构排序后相应元素之间的异或必须一致。 特别是，如果我们对两个集合进行排序，则有效映射意味着匹配元素之间的所有成对 XOR 差异都是相同的。 由于匹配是未知的，我们转而依赖于规范不变量：异或变换下的频率结构意味着多重集必须在具有固定掩码的异或对齐下闭合。 

最终的简化来自于异或移位排列保留了成对异或差异的事实。 在排列中$1..n$，所有成对的 XOR 差异形成一个固定的多重集。 输入必须在全局移位后匹配该结构，这减少了检查我们是否可以对齐一个元素并测试一致性。 

我们通过尝试将目标范围 (1) 中的最小可能值与数组的每个元素对齐来修复候选映射。 对于每个这样的对齐，我们得出$x = a_i \oplus 1$，隐式应用它，并验证变换后的多重集是否完全变为$1..n$。 虽然这听起来像$O(n^2)$，它实际上是通过散列或频率比较来修剪的，并且可以使用集合检查和早期拒绝来优化。 实际上，只有一个有效$x$可以存在，所以一旦我们找到一致的映射，我们就直接验证它。

 ## 算法演练

 关键思想是，如果存在有效的异或移位，它完全取决于我们如何将数组的一个元素映射到排列中的目标值。 

1. 构建输入数组的频率表或集合表示。 这允许在验证期间进行恒定时间的成员资格检查。 
2. 迭代每个元素$a_i$并将其视为该值的候选图像$1$在目标排列中。 据此，计算候选 XOR 掩码$x = a_i \oplus 1$。 此选择修复了整个转换，因为 XOR 是可逆的。 
3. 从概念上应用此候选：对于每个值$a_j$, 计算$a_j \oplus x$并检查是否在范围内$[1, n]$以及它是否在目标多重集中只出现一次。 此验证可确保不会发生冲突或缺失值。 
4. 如有候选人$x$为所有元素生成完美匹配，立即返回 YES。 如果在穷尽所有可能性后没有候选人工作，则返回 NO。 

### 为什么它有效

 有效的转换完全由单个 XOR 掩码决定$x$。 一旦一个元素被映射到其正确的目标位置，所有其他映射都会被强制。 如果出现任何不一致，则无法通过其他操作来修复，因为组合 XOR 操作只会产生另一个 XOR 掩码。 因此，正确性简化为检查输入集和输入集之间是否存在全局一致的双射。$\{1..n\}$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    target = set(range(1, n + 1))
    freq = {}
    for v in a:
        freq[v] = freq.get(v, 0) + 1

    # try mapping a_i -> 1
    for v in a:
        x = v ^ 1
        ok = True
        seen = {}
        
        for u in a:
            w = u ^ x
            if w < 1 or w > n:
                ok = False
                break
            seen[w] = seen.get(w, 0) + 1
            if seen[w] > 1:
                ok = False
                break
        
        if ok and len(seen) == n:
            if all(seen.get(i, 0) == 1 for i in range(1, n + 1)):
                print("YES")
                return
    
    print("NO")

if __name__ == "__main__":
    solve()
```该代码构建输入数组，然后尝试通过将每个元素与值对齐来推断有效的 XOR 掩码$1$。 对于每个候选掩码，它在临时频率图中完全重建变换后的数组，并检查它是否形成完美的排列$1..n$。 内循环内部的早期中断至关重要，因为它们可以防止无效情况下的二次爆炸，特别是当值过早超出有效范围时。 

最终检查确保完整性和唯一性，这都是有效排列所必需的。

 ## 边缘情况

 一个微妙的情况是所有值都相同。 例如，$n = 3$,$a = [7,7,7]$。 任何 XOR 移位都会保留所有元素的相等性，因此数组永远不会成为排列。 该算法会尝试从每个值派生的候选值，但每次转换仍然会产生重复项，因此所有检查都会失败，答案正确为“否”。 

另一种情况是值已经是排列但远远超出目标范围。 例如，$a = [1000,1001,1002]$和$n = 3$。 该算法尝试$x = 1000 \oplus 1$，它将数组映射到候选集，但结果值不能精确形成$\{1,2,3\}$，因此拒绝发生在范围验证步骤中。 

Finally, consider when only one element matches a potential alignment but others do not. 由于 XOR 是全局的，因此单个不匹配会立即强制拒绝，并且提前退出可确保算法不会浪费时间扫描数组的其余部分。
