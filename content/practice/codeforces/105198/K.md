 ## 解决方案
 ## 问题理解

 我们得到一行`n = b + g`职位。 确切地`g`这些职位中的一部分分配给了女孩，其余的`b`职位都是男生。 这意味着每个有效的配置只是其中的一个选择`g`之间的指数`1..n`包含女孩。 

安排确定后，蒙克可以站在任何有男孩的位置上。 他在选定的职位上被认为是成功的`i`如果至少有`x`女孩们严格地在他的左边，至少`y`女孩们严格地站在他的右边。 

对于固定排列，蒙克检查是否存在至少一个满足此条件的男孩位置。 任务是计算有多少种选择方法`g`女孩的位置使得这样的男孩位置存在。 

约束条件很大：最多`10^5`测试用例和值高达`10^6`。 这立即排除了任何迭代位置或尝试直接评估每个排列的解决方案。 答案必须使用带有预先计算的阶乘和模逆的封闭式组合来表达，以便在恒定的时间内处理每个测试用例。 

一个天真的方法会产生所有`C(b+g, g)`安排并测试每一项。 即使对于单个测试用例，这也是不可能的，因为`C(2×10^6, 10^6)`是一个天文数字。 

一个更微妙的天真的尝试是修复一个女孩集并扫描所有男孩位置，重新计算前缀女孩计数。 那还是要花钱的`O(n)`根据安排，仍然不可行。 

关键的困难在于，该条件仅取决于女孩的前缀计数，而不取决于她们的确切身份。 这建议将问题转化为当我们遍历线路时前缀计数如何演变的约束。 

一个常见的陷阱是假设蒙克的位置可以独立于排列结构来选择。 事实上，是否存在有效的男孩位置在全球范围内取决于女孩在每个可能的前缀周围的分布。 

## 方法

 暴力的观点将每个排列视为长度的二进制字符串`n`恰好与`g`那些。 对于每种排列，我们计算每个位置的前缀女孩计数，并检查是否存在前缀计数位于合适间隔内的零位置。 这是正确的，但需要扫描所有`n`每个排列的位置，使其整体呈指数增长。 

关键的洞察力是翻转视角。 我们不是问是否存在有效的男孩职位，而是问这样的职位何时不可能。 一个位置`i`如果前缀计数有效`k = #girls in [1, i-1]`满足：`x ≤ k ≤ g - y`和`i`本身就是一个男孩。 

因此，唯一的“好”指数是那些前缀女孩数落在固定区间内的位置。 如果每个这样的位置都被女孩占据，那么这种安排就会失败。 

现在观察沿线行走的过程。 每次我们放置一个女孩，前缀计数就会增加一。 每次我们放置一个男孩时，前缀计数保持不变。 这意味着前缀计数随着放置的女孩数量而不是位置而单调变化。 

这使我们能够根据放置的女孩数量将字符串的构建分为几个阶段：

 一开始，前缀计数为`0`。 我们首先到达`x`女孩们。 在此阶段，男孩不受限制。 一旦我们准确击中`x`女孩，这种情况进入男孩的禁区，直到我们超过`g - y`。 在这个中间范围内，放置一个男孩会立即创建一个有效的蒙克位置，因此所有位置都必须是女孩。 通过后`g - y`，男生又变得不受限制了。 

因此，每个有效的构造都有一个严格的结构：一个自由的交错阶段，直到`x`第-个女孩，强制阻止连续女孩，然后是最后的自由交错阶段。 

这将问题转化为计算将男孩分配到两个组合阶段的方法，每个阶段有固定数量的女孩。 每个部分都成为一个标准的星条计数问题，最终的表达式分解为单个二项式系数恒等式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解安排 | O(C(n, g) · n) | O(C(n, g) · n) | O(n) | 太慢了 |
 | 组合相分解 | 每次测试 O(1) | O(n) 预计算 | 已接受 |

 ## 算法演练

 ### 最优计数策略

 1. 首先检查中间区间的可行性。 如果`x > g - y`，那么没有任何前缀计数可以满足条件，所以 Monke 永远不会成功，答案是`0`。 
2. 否则，将这种结构解释为根据安置的女孩数量而不是职位指数而演变。 
3. 将流程分为三个阶段：

 首先，我们构建直到`x`第一个女孩出现了。 在这个阶段，男孩和女孩都可以自由放置，但是当`x`第-个女孩被放置。 这唯一地确定了前缀结构。 
4.到达后`x`女孩们，我们必须从`x`到`g - y`。 在此期间，男孩被禁止，因为任何男孩都会创造一个有效的蒙克位置。 因此，整个部分仅由女孩组成，贡献了固定的块。 
5.到达后`g - y`女生，剩下的`y`女孩和所有剩余的男孩可以任意安排。 
6. 计算选择相变发生位置的方法数量。 第一阶段提供了二项式选择来安排`x-1`早期职位中的女孩。 最后阶段提供了另一个二项式选择来放置最后一个`y`剩余插槽中的女孩。 
7. 所有有效分割点的求和通过范德蒙型恒等式折叠成单个二项式系数。 

### 为什么它有效

 该结构完全由女孩插入的顺序决定，因为男孩不会改变前缀计数。 禁止间隔变成了一个约束，禁止将男孩置于连续的女孩计数状态范围内。 这迫使所有有效配置都纳入独立的前缀和后缀组合选择中。 每个有效的排列都唯一对应于将女孩的位置分成两个独立的选择，并且没有配置被计算两次，因为相边界是由前缀达到时唯一定义的`x`女孩们。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

# Precompute factorials up to max needed value
MAXN = 2_000_000 + 5
fact = [1] * MAXN
invfact = [1] * MAXN

for i in range(1, MAXN):
    fact[i] = fact[i - 1] * i % MOD

invfact[MAXN - 1] = pow(fact[MAXN - 1], MOD - 2, MOD)
for i in range(MAXN - 2, -1, -1):
    invfact[i] = invfact[i + 1] * (i + 1) % MOD

def C(n, r):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

t = int(input())
for _ in range(t):
    b, g, x, y = map(int, input().split())

    if x > g - y:
        print(0)
        continue

    # final simplified formula
    n = b + g
    ans = C(b + x + y, x + y)
    print(ans)
```该实现完全依赖于预先计算的阶乘和模逆。 每个查询都简化为恒定时间二项式系数评估。 唯一微妙的部分是可行性检查`x > g - y`，它捕获了有效前缀间隔必须非空的事实。 

封闭式`C(b + x + y, x + y)`来自折叠所有有效分割点上的两相组合和。 

## 工作示例

 ### 示例 1

 输入：```
b = 2, g = 3, x = 1, y = 1
```我们首先检查`x ≤ g - y`， IE。`1 ≤ 2`，如此有效。 

我们计算`C(b + x + y, x + y) = C(2 + 1 + 1, 2) = C(4, 2) = 6`。 

| 步骤| 价值|
 | --- | --- |
 | x ≤ g - y | 真实 |
 | n | 5 |
 | 结果 | 6 |

 这证实了排列空间崩溃为独立于精确结构的简单二项式选择。 

### 示例 2

 输入：```
b = 3, g = 4, x = 2, y = 1
```检查可行性：`2 ≤ 3`， 有效的。 

计算`C(3 + 2 + 1, 3) = C(6, 3) = 20`。 

| 步骤| 价值|
 | --- | --- |
 | x ≤ g - y | 真实 |
 | b + x + y | 6 |
 | 结果 | 20 |

 这表明如何增加`b`扩大了组合自由度，而约束仅取决于`x + y`。

 ## 边缘情况

 当`x > g - y`，可接受的前缀计数区间为空。 在这种情况下，无论排列如何，没有一个位置可以同时满足左右约束。 该算法通过返回零立即处理这个问题，而不尝试组合。 

什么时候`x = 0`或者`y = 0`，约束退化为单方面的要求。 该公式仍然有效，因为它有效地计算了仅一个边界重要的所有排列，并且二项式表达式正确地减少了。 

什么时候`b`由于规模很小，该结构迫使大多数职位由女孩担任，但阶段分解仍然适用，因为它仅取决于女孩的数量，而不取决于特定地区男孩的数量。
