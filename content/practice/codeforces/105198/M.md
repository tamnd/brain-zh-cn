 ## 解决方案
 ## 问题理解

 网格是无限的，每个测试用例都描述一个起始图块和一个目标图块。 一次移动恰好改变一个坐标一个单位。 普通的最短路径就是曼哈顿距离，因为我们只需要计算必须独立改变 x 和 y 坐标的距离。 

额外的规则改变了情况：最多做之后`k`朝一个方向移动，则下一移动不能继续朝同一方向移动。 任务是找到到达目标的最短有效移动序列。 

输入最多包含`10^5`独立的测试用例。 坐标可以大到`10^9`绝对值，因此模拟路径是不可能的。 即使是与点之间的距离成正比的解决方案也会太慢，因为一个测试用例可能需要数十亿次移动。 和`10^5`在这种情况下，解决方案只需要在每种情况下执行恒定数量的算术。 

主要的边缘情况来自最短曼哈顿路径不需要任何校正或一个轴不需要移动的情况。 

考虑开始于`(0,0)`并达到`(0,0)`和`k = 5`。 答案是`0`。 由于移动限制而总是添加一些修正的解决方案在这里会失败。 

考虑`(0,0)`到`(10,0)`和`k = 3`。 答案不是`10`，因为连续十次向右移动是非法的。 即使最终的 y 坐标未更改，移动限制也会强制在另一个轴上进行额外的移动。 

考虑`(0,0)`到`(5,5)`和`k = 5`。 答案是`10`。 粗心的方法可能会将两个方向分成多个块并添加不必要的转弯，但每个方向已经可以在一个块中完成。 

## 方法

 最直接的方法是模拟动作。 我们可以反复向目的地移动，无论何时到达`k`在一个方向上连续移动，在另一个方向上插入移动以重置连胜。 这会产生一条有效的路径，并且我们第一次到达目标时就会给出正确的答案。 

问题在于规模。 坐标差可以表示为`2 * 10^9`，因此对于单个测试用例，模拟可能需要数十亿次迭代。 和`10^5`情况下，最坏的情况是`10^14`操作，远远超出了时间限制。 

有用的观察是，只有两个所需距离中较大的一个才会引起麻烦。 假设较大的距离是`dx`较小的一个是`dy`。 沿较小轴的每次移动都可以分离沿较大轴的移动块。 最好的安排是`k`沿较大轴移动，然后沿较小轴移动，如此重复。 

第一个`dy`在较小的轴上移动创建`dy`分隔符。 还有最后一组最多`k`较大轴在最后一个分隔符之后移动，因此无需额外移动即可覆盖的总较大轴距离为：$$(dy+1) \times k$$如果`dx`最多为这个值，曼哈顿距离是可以实现的。 

什么时候`dx`越大，则较大轴上的剩余距离需要更多分隔符。 每对额外的较大轴块需要在较小轴上进行两次额外的移动：一次远离目标轴，一次向后移动。 由于每对额外的分离器移动都允许另一个`2k`较大轴上的进展，我们可以直接计算校正。

 ## 算法演练

 1. 计算起始坐标和目标坐标之间的绝对距离。 让他们成为`dx`和`dy`。 
2.如果`dx`小于`dy`，交换它们。 经过这次操作后，`dx`代表较大的距离，`dy`代表较小的距离。 轴是对称的，因此公式始终可以按此顺序应用。 
3. 检查仅使用所需的曼哈顿移动可以处理多少较大距离。 可用容量为`(dy + 1) * k`。 如果`dx`不大于这个值，答案很简单`dx + dy`。 
4. 如果`dx`超过该容量，计算剩余的较大轴距离。 每增加一个`2 * k`单位需要两次额外的移动来创建分隔符，因此添加：$$2 \times \left\lceil\frac{dx-(dy+1)k}{2k}\right\rceil$$到曼哈顿的距离。 

1. 输出曼哈顿距离加上计算出的修正值。 

为什么它有效：

 较大轴的移动是唯一可能用完可用分离器的移动。 较小轴的移动已经发生，并且它们中的每一个都可以中断较大轴移动的一个块。 第一个`(dy + 1)`无需任何额外的移动即可完成块。 任何剩余的较大轴距离必须被分成最多 大小的块`k`，并且每一对额外的块都需要在较小的轴上绕行两次。 该公式精确地计算了此类弯路的最少数量，因此产生的价值既是可实现的，也是最小的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    sx, sy, tx, ty, k = map(int, input().split())

    dx = abs(sx - tx)
    dy = abs(sy - ty)

    if dx < dy:
        dx, dy = dy, dx

    ans = dx + dy

    if dx > (dy + 1) * k:
        extra = dx - (dy + 1) * k
        ans += 2 * ((extra + 2 * k - 1) // (2 * k))

    print(ans)

t = int(input())
for _ in range(t):
    solve()
```该代码首先将坐标变化转换为距离，因为只有移动量才重要。 指示标志不影响限制。 

交换使较大的位移始终命名`dx`。 这避免了为水平和垂直移动编写两个单独的情况。 

条件`dx > (dy + 1) * k`是限制影响答案的唯一地方。 如果为假，则所有必需的移动都适合有效的块。 如果为 true，则剩余距离将通过额外的分隔符移动来处理。 

表达式```
(extra + 2 * k - 1) // (2 * k)
```是整数上限除法。 它计算有多少组`2*k`还需要取得剩余进展。 乘以`2`将这些组转换为额外移动的数量。 

Python 整数具有任意精度，因此大坐标值不需要任何特殊处理。

 ## 边缘情况

 对于起点和目标相同的情况，例如：```
0 0 0 0 5
```距离都为零。 交换检查后，`dx`和`dy`保持为零。 容量检查错误，所以答案保留`0`。 不引入人工运动。 

对于仅存在于一个轴上的运动：```
0 0 10 0 3
```我们有`dx = 10`和`dy = 0`。 可用容量为`(0 + 1) * 3 = 3`，所以剩余距离为`7`。 每额外`2 * 3`进步需要两步，给予`ceil(7 / 6) * 2 = 4`额外的动作。 答案是`10 + 4 = 14`。 

对于直接路径已经有效的情况：```
0 0 5 5 5
```较大的距离是`5`，较小的距离是`5`，容量为`(5 + 1) * 5 = 30`。 由于容量超过了更大的距离，所以答案正是`5 + 5 = 10`。 

对于最大坐标范围：```
0 0 1000000000 0 1
```较大的距离是十亿，较小的距离是零。 容量仅为`1`，离开`999999999`覆盖的单位。 天花板划分给出`500000000`额外更正组，添加`1000000000`移动和生产`1999999999`。 该算术仍然安全，因为 Python 整数不会溢出。
