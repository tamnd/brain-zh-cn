 ## 解决方案
 ## 问题理解

 我们给出一个在整数上定义的函数`x`通过它们的二进制表示来解释。 每个整数对应一组位位置，每个数字`y ≤ x`和`y & x = y`正好是一个子掩码`x`，这意味着它只使用已经存在于中的位`x`。 

该函数从基础层开始，其中`f(0, x)`简单来说就是`x`本身。 下一层是通过获取所有子掩码构建的`y`的`x`并对它们之前的值求和`f(k-1, y)`。 因此，每一步都通过位掩码的子集网格向下传播信息，重复累积所有子集的贡献。 

输入给出多个独立的查询，每个查询都询问该函数在给定深度的值`k`和面具`x`，两者都达到`3 × 10^5`，直到`10^5`查询。 

一种天真的解释建议对每个测试和每个级别的所有子掩码进行重复的子集求和`k`。 这已经暗示了麻烦，因为子掩码的数量随着设置位的数量呈指数增长。 即使最多 19 位，每个查询和每个重复此类操作`k`将远远超出可行的限度。 

真正的约束压力来自于大的组合`t`和大`k`。 任何为每个查询重新计算子掩码的方法，或者甚至在没有结构简化的情况下独立地为每个位重新计算的方法，都将超出时间限制。 

如果尝试直接模拟重现，就会立即出现一种微妙的失败情况：

 例如，与`x = 7 (111₂)`和`k = 2`，直接实现会将所有子集枚举两次，从而导致相同子结构的重复重新计算。 甚至缓存每`(k, x)`这是不可能的，因为两个维度都很大并且状态空间以位为指数。 

核心困难是复发看起来是局部的`x`，但实际上是在整个子集格上重复传播。 

## 方法

 递归是重复应用的经典子集和运算符。 转换的一个应用程序替换了一个函数`g(x)`总和`g(y)`在所有子掩码上`y ⊆ x`。 这称为子集 zeta 变换。 问题应用了这种变换`k`时间开始于`g(0, x) = x`。 

强力模拟将显式地重新计算每层的所有子掩码总和。 一项申请已经收费`O(3^n_bits)`使用子集枚举聚合每个掩码，并重复它`k`时代让人绝望。 

关键的结构观察是这种变换在比特上是线性的并且尊重子集独立性。 每个位在子集的形成方式上都是独立的，这使我们能够将过程重新解释为子集网格中的计数链，而不是重复对值求和。 

我们不是直接跟踪值，而是组合地重新解释递归。 变换的每次应用都对应于允许在不同层引入比特，因此该过程变成对子集包含物的序列进行计数。 这将问题转化为计算每个位从 0 到 1 的转换方式`k`步骤，同时根据起始子集的原始值对贡献进行加权。 

一旦以这种方式表达，位之间的独立性就变得可用：贡献因子分解为每个位的选择，并且所有子掩码的总和干净地因子分解。 

这导致了一个封闭式表达式，仅取决于三个量：`x`，设置位数`x`，以及权力`k`和`(1 + k^{-1})`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 以位为单位的指数 × k | O(1) | O(1) | 太慢了 |
 | 最优（位分解）| 每个查询 O(log x) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们通过根据子掩码链重写递归并分离每比特贡献来导出封闭形式。 

1. 将递归的每一步解释为在所有子掩码上展开函数，这对应于在子集晶格中向下移动。 后`k`步骤，每个值取决于所有链的长度`k`结束于`x`。 
2.修复起始子掩码`y ⊆ x`。 的贡献`y`后`k`脚步只取决于我们可以成长的方式有多少`y`进入`x`正好在`k`层。 存在于的每一位`x`但不在`y`必须在某个步骤引入，并且每个这样的位独立地选择其中之一`k`步骤。 
3. 这意味着扩展固定的方法的数量`y`到`x`是`k^(popcount(x) - popcount(y))`。 递归成为所有子掩码的加权和，其中权重仅取决于它们的大小。 
4. 将函数重写为各个位贡献的总和`y`。 每个位独立地贡献于整数值`y`，因此我们通过固定一个位并计算包含它的所有子掩码来分离总和。 
5.对于固定位`i`，其余位形成一个较小的独立子集。 这允许将总和分解为位乘积，从而形成辅助项的紧凑表达式`S(x)`，它成为比特的几何乘积。 
6. 子掩码和`S(x)`简化为`(1 + k^{-1})^(popcount(x))`因为每个位要么不存在（权重 1），要么存在（权重`k^{-1}`）。 
7. 结合因式分解，最终表达式变为：`f(k, x) = x × k^(popcount(x) - 1) × (1 + k^{-1})^(popcount(x) - 1)`。 
8. 预先计算模逆`k`，计算 popcount 和值`x`，并使用模幂有效地评估幂。 

### 为什么它有效

 递归定义了子集上函数的线性运算符，重复应用对应于多次应用相同的线性变换。 由于子集结构对独立位进行因式分解，因此运算符分解为每个位的贡献。 这将看起来像全局组合爆炸的事情变成了每比特独立的几何缩放，原始数值为`x`仅充当附加到每个贡献位的线性权重。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

t = int(input().strip())
for _ in range(t):
    k, x = map(int, input().split())

    pc = x.bit_count()

    if pc == 1:
        print(x % MOD)
        continue

    invk = modpow(k, MOD - 2)

    base = (k % MOD) * ((1 + invk) % MOD) % MOD
    base = modpow(base, pc - 1)

    print(x % MOD * base % MOD)
```实现完全依赖于封闭形式。 唯一重要的部分是计算模逆`k`，因为表达式使用`k^{-1}`在模块化算术设置内。 每个查询仅应用一次求幂，这使解决方案保持高效。 

特殊情况是`popcount(x) = 1`在该公式下是无害的，但明确地编写它可以避免对指数零的任何混淆并使结构更清晰：该函数直接简化为`x`。 

## 工作示例

 考虑一个小案例，其中`x = 3 (11₂)`和`k = 1`。 我们有两个子掩码：`0`和`3`。 一步之后，每个值都成为其子掩码的总和。 

| y | f(0, y) | f(0, y) | 有助于 f(1, x) |
 | --- | --- | --- |
 | 0 | 0 | 0 |
 | 1 | 1 | 1 |
 | 2 | 2 | 2 |
 | 3 | 3 | 3 |

 对所有子掩码求和得出`6`。 公式给出`x × 2^(pc(x)-1) = 3 × 2 = 6`，完全匹配。 

现在考虑`x = 5 (101₂)`和`k = 2`。 位数是`2`。 公式给出：`f(2, 5) = 5 × 2^(1) × (1 + 1/2)^1 = 5 × 2 × 3/2 = 15`。 

从概念上追踪，每个子掩码根据其位在两层子集扩展中存活的次数进行不同的加权，并且封闭形式聚合了这些加权贡献而不枚举它们。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import math

    MOD = 10**9 + 7

    def modpow(a, e):
        res = 1
        while e:
            if e & 1:
                res = res * a % MOD
            a = a * a % MOD
            e >>= 1
        return res

    t = int(input().strip())
    out = []
    for _ in range(t):
        k, x = map(int, input().split())

        base = (k % MOD) * ((1 + invk) % MOD) % MOD
        base = modpow(base, pc - 1)
        out.append(str((x % MOD) * base % MOD))
    return "\n".join(out)

 ## 边缘情况

 当`x`是 2 的幂，递归式会折叠成一条链，因为只有一位。 该算法干净地处理了这个问题，因为`popcount(x) = 1`使指数为零并返回`x`直接匹配每个子掩码链都简化为以下事实：`0`或没有组合分支的完整值。 

什么时候`k = 1`，该公式简化为所有子掩码的经典子集和。 表达式简化为`x × 2^(popcount(x)-1)`，它与所有子集中独立贡献的每个位对齐，确认转换正确地模拟了单个 zeta 扩展，而没有过度计数。
