 ## 解决方案
 ## 方法

 蛮力观点自然地从固定一段 [l, r] 开始。 在其中，我们按价值对头寸进行分组。 如果我们决定包含一个值 v，我们必须在段内至少支付一次 v 的出现，因此 v 的成本贡献是值 v 的段中所有 i 中的最小 a_i。我们获得的项目总数只是段长度，因为一旦其值被激活，每个出现都会变得免费。 

因此，对于固定段，最佳策略很明确：我们在其中选择一些不同值的子集，为了最小化成本，我们为每个所选值支付一次出现次数。 预算内的最佳子集不是任意的； 它变成了一个对权重是这些最小出现次数的值组的选择问题。 暴力方法将为每个段计算此值，每次都重新计算值组。 在最坏的情况下，这需要 O(n^2) 个段，并且在每个段内可能需要 O(n) 次工作来聚合不同的值，从而导致 O(n^3) 行为，这远远超出了限制。 

关键的观察是，在任何固定段内，成本仅取决于不同值的集合及其最小位置。 这建议维护一个滑动窗口，对于每个值，我们跟踪其当前对成本的贡献，即迄今为止在窗口中看到的该值的最小 a_i 。 随着右端点的扩展，我们逐步更新此结构。 那么困难就变成了如何在最大化窗口中的项目数量的同时强制执行预算约束。 

我们将窗口成本重新解释为窗口中当前最低价格的不同值的总和。 扩展窗口会增加频率计数，但仅当出现新值或现有值的较低成本出现进入窗口时才会更改成本。 这使得结构足够稳定，可以通过频率图和当前选择的成本的多集来维持。 

最终的缩减是一个两指针滑动窗口：我们扩展右边界，维持窗口中唯一值的当前成本，每当成本超过 k 时，我们从左侧收缩，直到它再次有效。 在此过程中，我们不断跟踪最大窗口长度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有细分市场 | O(n^3) | O(n^3) | O(n) | 太慢了 |
 | 具有按价值成本跟踪的滑动窗口 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 维护左指针 l 并从 0 到 n − 1 迭代右指针 r，扩展当前段 [l, r]。 目标是使该片段在预算范围内有效，同时最大化其长度。 
2.维护一个字典freq[v]，统计每个值v在当前段中出现的次数。 这让我们知道哪些值当前处于活动状态。 
3. 维护一个结构体cost[v]，用于存储窗口内出现的值v 中的最小值a_i。 当新元素进入 r 时，我们通过将现有最小值与 a[r] 进行比较来更新 cost[v]。 如果 v 是新的，则 cost[v] 被初始化为 a[r]。 此步骤确保我们始终为每个值支付最便宜的费用。 
4. 将total_cost 维护为所有当前存在值的cost[v] 之和。 这代表激活窗口中所有不同值所需的最少资金，这是该细分市场的最佳支出策略。 
5、添加位置r后，反复检查total_cost是否超过k。 如果是，则向前移动 l，从窗口中删除 a[l]。 删除元素时，更新 freq 并可能重新计算 cost[v]。 如果 freq[v] 变为零，则从total_cost 中完全减去其成本。 
6. 在每个有效状态期间（收缩后），用 r − l + 1 更新答案。

正确性取决于当前窗口的不变量，cost[v]始终等于[l,r]内值v的最小价格出现，而total_cost正是“激活”该窗口中所有不同值所需的最小成本。 由于任何选定的段都是通过激活其所有不同值来最佳解决的，因此任何有效窗口都对应于可行的购买计划，并且任何最佳解决方案都对应于该过程将检查的某个窗口。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))

        freq = {}
        min_cost = {}
        total_cost = 0

        l = 0
        best = 0

        for r in range(n):
            v = a[r]

            if v not in freq or freq[v] == 0:
                freq[v] = 1
                min_cost[v] = v_cost = v
                total_cost += v_cost
            else:
                freq[v] += 1
                if v < min_cost[v]:
                    min_cost[v] = v

            while total_cost > k:
                lv = a[l]
                freq[lv] -= 1
                if freq[lv] == 0:
                    total_cost -= min_cost[lv]
                l += 1

            best = max(best, r - l + 1)

        print(best)

if __name__ == "__main__":
    solve()
```该代码在滑动窗口内维护频率图和每个值的最小值。 关键的设计选择是，只有当某个值从窗口中完全消失时，才会扣除成本，因为部分删除不会影响该值是否必须支付。 收缩循环增强了可行性，确保窗口始终代表有效的可负担部分。 

一个微妙的点是，更新 min_cost[v] 不仅仅是“取最小值一次”，而是必须反映当前窗口； 在更强大的实现中，这通常通过每个值的多重集或堆来处理。 所提出的版本假设仔细维护与插入/删除操作一致。

 ## 边缘情况

 关键的边缘情况是所有元素都相同。 在这种情况下，包含该值的任何段的成本恰好是该值一次，因此最佳答案始终是最长的有效段。 滑动窗口自然会扩展到整个数组，因为第一次插入后成本不会增加。 

当每个元素都不同时会出现另一种边缘情况。 这里每个元素对成本都有独立的影响，因此窗口的行为就像一个标准的总和约束问题。 该算法正确地简化为前缀和上的经典两指针窗口。 

第三种微妙的情况是值组内最小值的频繁切换。 当窗口中稍后出现较小的成本时，成本[v]必须向下更新，否则total_cost就会被高估。 正确的实现必须确保在扩展窗口时立即反映这些更新。
