 ## 解决方案
 ## 问题理解

 我们有一个连接的网络$n$加入的地点$n-1$Trails，所以结构是一棵树。 每条路径连接两个位置并带有一个整数标签。 访问者可以沿着任一方向的路径行走，两个位置之间的有效路线就是树中唯一的简单路径。 

任务不是枚举所有路径，而是计算有多少无序顶点对生成一条边标签表现良好的路径：当您从一个端点遍历路径到另一个端点时，遇到的标签必须严格逐步增加。 

因为树中任何两个节点之间的路径都是唯一的，所以每一对$(u, v)$恰好对应于一个边序列。 唯一的问题是能否通过选择正确的遍历方向使这个序列严格递增。 

如果一条路径在一个方向上严格增加，那么它在相反方向上严格减少。 由于我们仅对每个无序对进行一次计数，因此我们有效地对边缘标签沿唯一树路径形成严格单调序列的所有路径进行计数。 

约束条件很大，有$n$最多$2 \cdot 10^5$，这会立即排除任何尝试显式构建或检查每条路径的方法。 甚至$O(n \log n)$或者$O(n \alpha(n))$方法是可以接受的，但是任何关于路径的二次方都是不可能的，因为树中的路径数已经是$O(n^2)$。 

当路径不是全局单调的时，即使在本地看起来“几乎已排序”，也会出现微妙的失败情况。 例如，带有标签的路径$1, 3, 2, 4$即使大多数相邻比较都在增加，也是无效的，因为单个反转破坏了严格的单调性。 仅检查端点或聚合最小值和最大值的简单方法将错误地接受此类情况。 

另一个棘手的情况是标签重复或相等。 由于条件是严格递增的，任何相等的相邻标签都会立即使一个方向无效，即使另一个方向仍然是单调的。 

## 方法

 暴力策略将采用每对节点，使用 DFS 或父指针提取它们之间的唯一路径，收集沿该路径的所有边缘标签，并检查它们是否在任一方向上严格增加。 即使有一个$O(n)$路径提取，这导致$O(n^3)$最坏情况下的行为，因为有$O(n^2)$对并且每条路径可以是$O(n)$。 这对于$2 \cdot 10^5$。 

关键的结构观察是，每条有效路径都由单个“最重要”边缘确定：沿该路径的最大标签。 如果我们修复该边缘，则路径上的其他所有内容都必须位于仅包含较小标签的区域中。 这建议按照标签的递增顺序处理边缘，并逐步构建有效的贡献。 

当我们按排序顺序考虑边时，目前我们处理带有标签的边$w$，所有先前处理的边的标签严格小于$w$。 这些边缘形成了一片森林。 任何有效的递增路径，其最大边恰好是$w$必须使用该边作为其递增顺序的最后一步，并且路径的其余部分必须完全位于通过至少删除带有权重的边形成的两个组件内部$w$。 

这将问题简化为计算有多少种方法可以从当前边的每个端点的“已处理边组件”中选取一个端点，并通过该边连接它们。 该贡献成为终止于每个端点的较小递增路径的计数的乘积。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 排序边 + DP 传播 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们维护一个价值观$dp[x]$，表示以节点结束的有效严格递增路径的数量$x$仅使用到目前为止已处理的边缘，这意味着仅使用比当前正在考虑的标签更小的边缘。 

我们按照标签的升序处理边缘。 

1. 按标签升序对所有边进行排序。 这确保了当我们处理一条边时，所有较小的边都已计入 dp 值中。 
2. 全部初始化$dp[x] = 0$。 此时，当我们考虑从它延伸时，每个节点单独贡献一条长度为零的隐式路径。 
3. 迭代边缘$(u, v, w)$按排序顺序。 对于每条边，我们要计算最大边正是该边的所有新的递增路径。 在更新任何内容之前，我们存储当前值$dp[u]$和$dp[v]$，因为它们表示使用较小边到达每个端点的方法数。 
4. 计算该边的贡献为$(dp[u] + 1) \cdot (dp[v] + 1)$。 这$+1$术语表示选择端点本身作为路径的起点。 乘法结合了边两侧的独立选择，产生了形成一条路径的所有方法，该路径使用该边作为其递增序列中的最大元素。 
5. 将此值添加到全局答案中。 
6. 现在我们更新 dp 以反映该边现在可用。 每条递增路径结束于$u$可以扩展到$v$通过该边缘，反之亦然。 所以我们更新$dp[u]$和$dp[v]$使用之前存储的值：

 我们设定$dp[u] \leftarrow dp[u] + (old\_dp[v] + 1)$和$dp[v] \leftarrow dp[v] + (old\_dp[u] + 1)$。 
7. 继续，直到处理完所有边缘。 

### 为什么它有效

 在处理过程中的任何一点，$dp[x]$准确计算到达的方式数量$x$使用其边严格递增且最后一条边的权重小于任何尚未处理的边的路径。 这确保了当我们处理权重边缘时$w$，所有对其端点有贡献的路径都是有效的“前缀”，可以在不违反单调性的情况下进行扩展。 

每条有效路径都有唯一的最大权重边。 当处理该边时，算法精确计算在该边两侧选择有效递增子路径的方法数量并将它们组合起来。 由于后面的步骤无法重新创建最大值更小的路径，因此每条路径都只计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    edges = []
    for _ in range(n - 1):
        a, b, c = map(int, input().split())
        edges.append((c, a - 1, b - 1))

    edges.sort()

    dp = [0] * n
    ans = 0

    for w, u, v in edges:
        du = dp[u]
        dv = dp[v]

        ans += (du + 1) * (dv + 1)

        dp[u] = du + dv + 1
        dp[v] = dv + du + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现是增量边缘处理逻辑的直接转换。 最重要的细节是保存$dp[u]$和$dp[v]$在更新其中任何一个之前，因为两个更新都依赖于之前的状态。 如果没有这个，第二次更新将错误地使用已修改的值并超出计数扩展。 

表达式$(du + 1)(dv + 1)$在更新之前评估，因为它对应于当前边缘是其递增序列中的最大边缘的路径。 在此步骤之后，dp 将扩展为包含现在使用此边作为未来扩展中的内部段的路径。 

## 工作示例

 考虑一条小链，其中边缘沿路径具有递增的标签。 每条新边都连接两个以前独立的区域，因此随着组件通过更高的权重合并，贡献会成倍增长。 

| 步骤| 边 (u, v, w) | dp[u] 之前 | dp[v] 之前 | 贡献 | dp[u] 之后 | dp[v] 之后 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 1 | 第一个边缘 | 0 | 0 | 1 | 1 | 1 |
 | 2 | 第二边缘| 1 | 1 | 4 | 3 | 3 |

 第一条边恰好贡献一条路径：单条边本身。 第二条边看到两个端点都已经各自支持一条普通路径，因此它生成四种组合，包括通过先前结构的所有扩展。 

现在考虑一个分支结构，其中多个边在一个节点处相交。 dp 值独立于每个分支进行累积，当较高权重的边连接它们时，它会结合所有先前累积的两侧递增路径，从而在答案中产生更大的跳跃。 这表明 dp 正在每个节点积累“前缀丰富度”，而不是跟踪单个路径。

 ## 边缘情况

 关键的边缘情况是多个边缘共享相同的标签。 由于严格增加禁止相等的连续标签，因此这些边不能在自身之间形成更长的增加序列。 该算法可以正确处理这个问题，因为排序将相等的权重分组在一起，并且 dp 传播不允许在扩展中重用相同权重的边。 

另一种情况是标签递减的线性链。 在这种情况下，每条边都按照从最小到最大的顺序进行处理，但在原始方向上不存在长的递增路径； 相反，有效路径仅根据局部结构形成，并且每条边仅贡献其孤立的组合。 

最后一个微妙的情况是节点通过增加权重连接许多组件。 该节点上的 dp 累积增长，但每条边仍然只使用自己更新之前的快照，确保不同最大边之间不会重复计算。
