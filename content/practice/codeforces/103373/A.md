 ## 解决方案
 ## 问题理解

 我们得到了一小部分国家/地区列表，每个国家/地区都由三个整数描述：获得的金牌、银牌和铜牌的数量，后面是其名称。 我们的任务是确定哪个国家在标准奥林匹克排名规则下排名最高。 

该排名按奖牌数的三倍按字典顺序排列。 一个国家获得的金牌越多越好。 如果金牌数相等，则银牌数决定。 如果金牌和银牌相同，则铜牌打破平局。 由于该问题保证存在一个最好的国家，因此我们不需要在最终答案中处理关系。 

尽管输入大小很小，但结构很重要：我们本质上是在三元组的自定义排序下找到最大元素。 

限制非常少，国家数量不到 300 个。 这立即告诉我们，即使二次或基于排序的解决方案也是安全的，但最简单的线性扫描就已经足够了。 我们只需要在读取输入时维护当前的最佳候选。 

一个微妙但重要的边缘情况来自输入格式：国家/地区名称可能包含空格，因此通过空格进行天真的分割可能会意外地将名称分开。 例如，像这样的行`10 5 3 United States of America`必须保留三个整数后面的完整字符串作为名称。 

另一种边缘情况是，除了输入顺序之外，所有奖牌计数都相同。 由于保证了最佳排名的唯一性，因此我们永远不需要打破无法比较的平局； 我们只是跟踪遇到的第一个最大值或持续更新严格更好的元组。 

## 方法

 强力解释会将每个国家与其他国家进行比较，根据词典编排规则检查主导地位。 对于每一对，我们都会比较金牌，然后是银牌，然后是铜牌，并跟踪是否有一个更好。 这可以正常工作，但成本却不必要地昂贵。 对于多达 300 个国家，这将需要大约 90,000 次比较，这仍然可以接受，但逻辑比需要的更复杂。 

关键的观察是排名定义了三元组的总顺序，因此我们不需要成对优势检查。 我们只需要找到字典比较下的最大元素即可。 这将问题减少到单次遍历列表，保持迄今为止最好的状态。 

我们不是比较每一对国家，而是仅将每个新国家与当前最佳候选国家进行比较。 这将问题减少到 O(n)，更简单且不易出错。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力配对比较 | O(n²) | O(1) | O(1) | 正确但不必要 |
 | 单程最大跟踪| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们对每个国家进行一一处理，并保持迄今为止看到的最好的三重。 

1. 读出国家数量n。 这定义了我们将对最佳候选者执行多少次更新。 
2. 初始化变量来存储当前最佳奖牌数和对应的国家名称。 我们从比任何有效输入更糟糕的状态开始，例如奖牌的负无穷大或从第一输入行初始化。 
3. 对于每个国家/地区，将该行解析为三个整数，并将剩余的字符串作为国家/地区名称。 这需要仔细解析，因为名称可以包含空格，因此我们不能假设前三个整数之外的固定数量的标记。 
4. 将当前国家的奖牌三重（g、s、b）与存储的最佳三重进行比较。 我们按字典顺序进行比较：首先是金，然后是银，然后是铜。 如果当前国家/地区严格来说更好，我们将用当前国家/地区替换存储的最佳值。 
5.处理完所有国家后，输出存储的最佳国家名称。 

### 为什么它有效

 该算法保持了这样的不变性：在处理 i 个国家之后，存储的候选国家是按照字典顺序（金、银、铜）的这 i 个国家中最好的。 每个更新步骤都保留了这个不变量，因为我们只有在找到严格更好的三元组时才替换存储的候选者。 由于比较定义了总顺序，因此处理所有条目后的最终候选必须是全局最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def better(g1, s1, b1, g2, s2, b2):
    if g1 != g2:
        return g1 > g2
    if s1 != s2:
        return s1 > s2
    return b1 > b2

def main():
    n_line = input().strip()
    while n_line == "":
        n_line = input().strip()
    n = int(n_line)

    best_g, best_s, best_b = -1, -1, -1
    best_name = ""

    for _ in range(n):
        line = input().rstrip("\n")
        while line == "":
            line = input().rstrip("\n")

        parts = line.split()
        g = int(parts[0])
        s = int(parts[1])
        b = int(parts[2])
        name = " ".join(parts[3:])

        if better(g, s, b, best_g, best_s, best_b):
            best_g, best_s, best_b = g, s, b
            best_name = name

    print(best_name)

if __name__ == "__main__":
    main()
```该解决方案逐行读取输入，并仔细处理格式化输入中可能出现的空白行。 每行都分为标记，其中前三个标记始终是整数，其余的形成国家/地区名称。 

比较逻辑被隔离在辅助函数中，以使字典顺序明确并避免嵌套条件中的错误。 初始化使用 -1 来表示奖牌计数，因为所有有效值都是非负的。 

一个常见的陷阱是在包含空格时错误地分割国家/地区名称。 通过在第三个索引之后连接所有标记，我们可以完全按照要求保留全名。

 ## 边缘情况

 一种边缘情况是相同的奖牌三重。 该算法仍然表现正确，因为它仅在严格更好的情况下替换最佳候选者，并且由于问题保证了唯一的最佳值，因此任何相等的比较都不会导致歧义。 

另一个边缘情况是当金和银相同时，青铜差异很大。 字典序比较确保仅在前两个级别匹配后才考虑青铜，并且辅助函数显式强制执行此排序。 

最后，通过在解析三个整数后从所有剩余标记重建名称来安全地处理包含空格的名称，并保留确切的原始标识符。
