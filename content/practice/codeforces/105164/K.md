 ## 解决方案
 ## 问题理解

 我们有一组骑士放置在一个不同的方格上$N \times N$棋盘。 每个骑士都有标准的国际象棋动作：它可以攻击 L 形图案中最多 8 个潜在的方格。 如果两个骑士可以一次移动到对方，那么两个骑士就会发生冲突。 

任务是移除尽可能少的骑士，这样剩下的一对骑士就不能互相攻击。 换句话说，删除后，剩余的配置必须具有零个攻击对。 

输入不描述完整的棋盘状态，而仅描述占用单元的坐标。 这意味着我们关心的实际图形不是完整的网格，而只是网格的子集$K$占领的位置，边缘代表这些位置之间的骑士攻击。 

输出是一个整数：要删除的骑士的最小数量，以便归纳图没有边。 

限制很小：$N \le 25$和$K \le 625$。 如果不小心实现且开销很大，这会立即排除所有对的二次或更差的情况，但仍然允许$O(K^2)$舒适地设计图表结构。 关键的挑战不是建立优势，而是认识问题的结构。 

一些微妙的情况很重要：

 如果没有两个骑士互相攻击，则答案为零。 如果没有正确检测到图没有边，仍然尝试匹配或删除节点的幼稚实现可能会意外地过度计数删除。 

如果所有的骑士都放置在一个密集的区域，可能会存在很多攻击，但结构仍然受到骑士移动的限制，因此每个节点最多有8条边。 不经骑士移动规则过滤的粗心全对检查会错误地将非攻击对视为冲突。 

另一个极端情况是骑士形成循环攻击。 例如，三个或更多骑士可以创建相互攻击链。 像“以最大程度删除任何骑士”这样的贪婪删除策略在这里可能会严重失败，因为局部选择不能反映全局最优集。 

## 方法

 蛮力的观点是尝试骑士的所有子集并检查子集是否无冲突。 对于每个子集，我们通过扫描子集中的所有对来验证没有两个骑士互相攻击。 有$2^K$子集并检查每个子集的成本高达$K^2$，导致$O(K^2 2^K)$，即使对于中等程度的人来说，这也是不可能的$K$。 

失败点在于约束是全局的：移除一名骑士可以同时解决多个冲突，而暴力无法重用子集之间的结构。 

关键的观察是骑士和他们的攻击形成一个图，任务变成选择顶点之间没有边的最大子集。 这就是最大独立集问题。 

在一般图表上，这很难，但棋盘上的骑士图是二分的。 如果我们按奇偶性对每个正方形进行着色$r + c$，每个骑士的动作总是从一种颜色变为另一种颜色。 这将冲突图变成二分图。 

在二部图上，应用经典恒等式：最大独立集的大小等于顶点数减去最大匹配的大小。 因此，我们不是考虑删除，而是计算可以最佳匹配多少个不相交的攻击对，然后减去。 

这将问题转换为最多 625 个具有稀疏边缘的节点的最大二分匹配问题，可以使用基于 DFS 的增广路径或 Hopcroft-Karp 有效解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(K^2 2^K)$|$O(K)$| 太慢了 |
 | 二分匹配|$O(K \cdot E)$（DFS）|$O(E)$| 已接受 |

 ## 算法演练

 1. 将每个骑士表示为索引为的图中的一个节点$0$到$K-1$。 每个节点都存储其棋盘坐标。 这种抽象允许我们将棋盘问题转换为图问题。 
2. 根据每个节点坐标的奇偶性为每个节点分配一种颜色，使用$(r + c) \bmod 2$。 这保证了马的每个动作都连接相反的颜色，这是使图二分的结构属性。 
3. 对于每对骑士，使用骑士移动规则检查他们是否互相攻击。 如果存在，请在它们之间添加无向边，但仅存储从黑色侧到白色侧的边。 这为二分匹配准备了图表。 
4. 运行最大二分匹配算法。 对于黑色一侧的每个节点，尝试使用 DFS 找到一条增广路径，将其与不匹配的白色节点匹配或重新路由现有匹配。 
5. 匹配的结果是不相交的攻击对的数量。 每条匹配的边恰好对应一个“冲突解决单元”，其中至少必须移除一名骑士。 
6. 将最终答案计算为$K - \text{matching size}$。 这相当于保留最大可能的非攻击骑士子集。 

### 为什么它有效

 关键的不变量是，在匹配过程中的任何一点，每个白色节点最多与一个黑色节点匹配，并且每个增强步骤严格增加匹配对的数量或在重新路由现有匹配时保留可行性。 由于该图是二分图，因此每个攻击关系都表示为相对分区之间的边，因此匹配永远不会违反有效骑士冲突的结构。 最大匹配的补集恰好对应于最小顶点覆盖，而它的补集对应于最大独立集，这就是我们保留的骑士集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can_attack(a, b):
    r1, c1 = a
    r2, c2 = b
    dr = abs(r1 - r2)
    dc = abs(c1 - c2)
    return (dr == 2 and dc == 1) or (dr == 1 and dc == 2)

def solve():
    N, K = map(int, input().split())
    knights = [tuple(map(int, input().split())) for _ in range(K)]

    # build bipartite graph: edges from black -> white
    adj = [[] for _ in range(K)]

    color = [(r + c) & 1 for r, c in knights]

    for i in range(K):
        for j in range(K):
            if i != j and color[i] == 0 and color[j] == 1:
                if can_attack(knights[i], knights[j]):
                    adj[i].append(j)

    match_to = [-1] * K

    sys.setrecursionlimit(10000)

    def dfs(v, seen):
        for to in adj[v]:
            if seen[to]:
                continue
            seen[to] = True
            if match_to[to] == -1 or dfs(match_to[to], seen):
                match_to[to] = v
                return True
        return False

    match = 0
    for i in range(K):
        if color[i] == 0:
            seen = [False] * K
            if dfs(i, seen):
                match += 1

    print(K - match)

if __name__ == "__main__":
    solve()
```代码首先读取所有骑士位置并使用坐标奇偶性分配二分颜色。 这是能够简化为匹配的结构步骤。 

邻接表仅从黑色节点到白色节点构建。 这避免了重复的边并确保 DFS 匹配在正确的二分表示上运行。 

DFS 函数尝试将白色节点分配给黑色节点，如果该白色节点已经匹配，它会递归地尝试重新分配其先前的伙伴。 递归限制增加，因为扩充路径可以通过多次重新分配进行链接。 

最后，我们计算成功的匹配并从中减去$K$，因为每个匹配对都对应于强制至少删除一个的冲突。

 ## 边缘情况

 一种重要的边缘情况是根本不存在边缘。 在这种情况下，匹配过程永远不会找到增广路径，因此匹配大小保持为零，并且答案正确地变为$K$。 除了检查邻接表之外，该算法不会尝试不必要的 DFS 扩展。 

另一种情况是一个完全密集的局部区域，许多骑士互相攻击。 即使如此，每个节点最多只参与 8 个边，因此 DFS 永远不会组合爆炸。 二分着色确保不会考虑无效的同边边缘，从而防止跨相同分区的错误匹配尝试。 

最后一个微妙的情况是多个增广路径相互作用时。 基于 DFS 的匹配依赖于重新访问先前匹配的节点，正确性取决于重新路由匹配的能力。 递归结构会自动处理此问题，因为每次失败的分配都会触发对替代分配的探索，直到找到空闲节点或用尽所有可能性。
