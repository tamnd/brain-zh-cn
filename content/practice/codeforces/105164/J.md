 ## 解决方案
 ## 问题理解

 我们得到一个字符串$S$长度$N$。 我们可以任意更改该字符串中的字符，每次更改都会花费一次操作。 目标不是直接构造回文，而是确保结果字符串中的某个位置存在一个长度为的子序列$K$形成一个回文。 

子序列意味着我们选择$K$按递增索引顺序排列位置，然后按顺序读取这些字符。 我们希望那些选定的字符向前和向后读相同的内容。 

所以任务是：通过修改原字符串中尽可能少的字符，保证至少存在一个长度-$K$回文子序列。 

关键的困难在于我们没有提前确定要使用哪个子序列。 修改后我们可以自由选择任何子序列，因此优化是针对子序列选择和字符编辑的。 

限制条件$N \le 500$和$K \le 500$立即建议二次或三次动态规划解决方案是可以接受的。 任何类似的东西$O(N^3)$小常数可能会通过，但子序列的指数枚举是不可能的，因为有$\binom{N}{K}$选择。 

一个天真的错误是假设我们只需要使整个字符串接近回文，或者贪婪地对齐整个字符串的对称位置。 这会失败，因为最优子序列可能完全跳过“坏”位置。 例如，如果字符串大部分是随机的，但包含一些相距很远的重复字母，则最佳回文子序列可能完全存在于这些重复中，而忽略其余部分。 

另一个微妙的边缘情况出现时$K = 1$。 任何单个字符都是回文，因此答案始终为零。 粗心的 DP 假设配对位置可能会错误地要求至少一场比赛。 

## 方法

 一个蛮力的想法是选择所有可能的长度子序列$K$，并为每个子序列计算使其成为回文所需的最小更改数。 对于固定子序列，这很简单：我们比较对称对并计算不匹配的数量，可能允许重新着色以最小化成本。 然而，枚举所有子序列已经花费了$\binom{N}{K}$，即使对于中等程度的人来说也是不可行的$N$。 

我们不是先选择子序列，而是反转视角。 我们决定回文子序列在结构上是什么样子，然后尝试以最少的编辑将其嵌入到字符串中。 

长度的回文$K$由它的第一个决定$\lfloor K/2 \rfloor$配对位置，如果$K$很奇怪，是一个中心人物。 对于每一对，我们必须选择两个索引$i < j$在表示子序列中镜像位置的字符串中。 如果他们的字符已经匹配，则不需要任何成本； 否则我们必须更改其中之一以使它们匹配，从而花费一次操作。 

这表明对字符串位置以及我们已经形成的对的数量进行动态编程。 关键是在索引空间中从左到右构建子序列，同时从两端强制回文配对。 

我们定义一个DP，在回文前半部分按升序选择位置，并将它们与从右侧选择的位置进行匹配。 每个不匹配都会造成成本 1。 

这将问题简化为索引对之间的结构化匹配，其中每对都根据字符相等性贡献成本。 最佳解决方案成为区间端点上的最短路径式 DP 以及需要形成多少对。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解子序列 |$O(\binom{N}{K} \cdot K)$|$O(K)$| 太慢了 |
 | 间隔DP施工|$O(N^2 K)$|$O(NK)$| 已接受 |

 ## 算法演练

 我们构建了一个 DP，它计算使用字符串前缀中的位置构造回文子序列的最小成本，同时跟踪已放置了多少个回文字符。 

1.我们定义一个DP状态$dp[l][r][t]$， 在哪里$l$和$r$是字符串中的边界，$t$是我们还需要放置多少个回文字符。 我们的想法是，我们从外向内填充回文，从可用间隔中选择匹配对。 这种表示确保所选索引始终遵循递增顺序约束。 
2. 基本情况是$t = 0$。 如果不需要字符，则不需要进一步更改，因此无论怎样，成本都是零$l$和$r$。 这对应于一个空子序列，它通常是一个回文。 
3. 什么时候$t = 1$，我们只需要一个中间字符。 我们可以选择之间的任何位置$l$和$r$，并且不存在配对约束。 成本为零，因为单个字符在可选修改后始终是回文，并且如果需要，我们总是可以对其重新着色而不影响对称性。 
4. 对于$t \ge 2$，我们考虑形成一对镜像。 我们选择两个位置$i$和$j$和$l \le i < j \le r$。 这些成为剩余回文的最外面的字符。 这种选择的成本是$0$如果$S[i] = S[j]$， 否则$1$。 修复这对后，我们递归内部区间$[i+1, j-1]$和$t-2$剩余字符。 
5. 我们计算所有有效分割的最小值$(i, j)$。 这确保我们探索所有可能的方法来放置最外面的回文对，同时保持顺序约束。 
6.最终答案是$dp[0][N-1][K]$，代表整个字符串和完整回文长度要求。 

这样做的原因是任何回文子序列都可以唯一地分解为其最外层的匹配对加上剩余间隔内的较小回文子序列。 DP 在结构上强制执行这种分解，并且每个有效子序列恰好对应于一个 DP 转换序列。 由于每个转换都会计算该对所需的精确最小修改，因此最佳子结构成立。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = input().strip()

    INF = 10**9

    # dp[l][r][t] = min cost to form palindrome subsequence of length t inside s[l:r+1]
    dp = [[[INF] * (k + 1) for _ in range(n)] for _ in range(n)]

    # base: t = 0 costs 0 everywhere
    for l in range(n):
        for r in range(n):
            dp[l][r][0] = 0

    # t = 1: single character always achievable with 0 cost
    for l in range(n):
        for r in range(l, n):
            dp[l][r][1] = 0

    for length in range(2, k + 1):
        for l in range(n):
            r_limit = n - 1
            for l2 in range(l, n):
                r = l2
                if r - l + 1 < length:
                    continue
                best = INF
                for i in range(l, r):
                    for j in range(i + 1, r + 1):
                        cost = 0 if s[i] == s[j] else 1
                        inner = dp[i + 1][j - 1][length - 2] if length >= 2 else 0
                        best = min(best, cost + inner)
                dp[l][r][length] = best

    print(dp[0][n - 1][k])

if __name__ == "__main__":
    solve()
```DP 数组是为所有子串和所有目标回文长度构建的。 最里面的转换显式地尝试每个可能的外部对，然后将问题减少到少两个字符的内部子字符串。 

边界情况$t = 0$和$t = 1$已预先填充以避免无效递归。 成本计算局限于每对，确保每个不匹配只贡献一次。 

一个微妙的实现细节是我们从不重用字符：一次索引$i$和$j$选择时，递归仅在严格内部区间上运行，这保证了子序列顺序的有效性。
