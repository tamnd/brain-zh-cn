 ## 解决方案
 ## 问题理解

 我们正在模拟容器内的细菌种群，从零时间的单个新生有机体开始。 每种细菌都遵循非常严格的生命周期。 它在最初的两分钟处于非繁殖的“不成熟”状态。 一旦成熟，它不会立即繁殖，而是遵循一个周期性循环：每四分钟它通过减数分裂产生恰好三个新细菌。 无论繁殖如何，每种细菌都有二十分钟的固定寿命，之后就会从系统中消失。 

任务是确定在给定的 N 分钟内存在多少细菌，假设所有出生和死亡都按照这些规则随着时间的推移连续发生，并且新产生的细菌在新生儿时立即开始其生命周期。 

输入大小使得随着时间的推移进行暴力模拟是不可能的。 N 可以大到 10^9，这意味着我们无法跟踪每分钟的演化。 任何随时间迭代的解决方案每个测试用例都需要多达十亿步，这远远超出了允许的一秒时间限制。 由于测试用例多达 50 个，因此即使每个步骤的对数开销也需要仔细控制。 

关键的难点在于每种细菌都有延迟激活、周期性繁殖和延迟死亡的情况。 简单的模拟需要跟踪每个细菌的年龄并安排未来的繁殖事件，这些事件呈指数增长并且很快变得不可行。 

一个微妙的边缘情况在最初几分钟出现，目前还无法复制。 例如，当 N = 1、2 或 3 时，只有原始细菌存在，因为它在 2 分钟时尚未成熟。 另一种边缘情况发生在生命即将结束时的 20 分钟，细菌突然消失，即使繁殖正在进行，也会减少种群数量。 正确的模型必须仔细同步出生、繁殖周期和死亡，而不能重复计算已经过期的个体。 

## 方法

 一种简单的方法是单独模拟每种细菌。 我们可以存储每种细菌当前的年龄，并一分钟一分钟地反复推进时间。 在每一步中，我们都会增加年龄，删除那些达到 20 岁的细菌，对于那些成熟并符合其繁殖时间表的细菌，我们会产生三种新细菌。 

这在概念上是有效的，因为它准确地反映了规则，但复杂性却急剧增加。 在最坏的情况下，种群由于繁殖而呈指数级增长，每分钟都需要扫描所有现有的细菌。 这会导致大约 O(population × time) 的行为，即使 N 大约有几百个，这也会变得难以管理。 

关键的观察结果是，除了年龄之外，细菌是无法区分的，并且它们的行为仅取决于年龄范围。 这意味着我们不需要追踪个体，只需要追踪每个年龄段存在多少细菌即可。 该系统在从 0 到 19 的固定数量的年龄状态上成为确定性的状态转换。每一分钟，整个种群都会改变一个年龄，20 岁时死亡会被删除，繁殖会根据当前处于繁殖时间的成熟细菌数量添加新的 0 岁细菌。 

这将问题转换为固定大小的线性状态演化，可以表示为大小为 20 的矩阵转换。由于 N 很大，我们对此转换应用快速求幂，以对数步长直接跳转到时间 N。

 | --- | --- | --- | --- |
 | 暴力模拟| O(人口 × N) | O(人口) | 太慢了 |
 | 年龄状态下的矩阵求幂 | O(20^3 log N) | O(20^3 log N) | O(20^2) | O(20^2) | 已接受 |

 ## 算法演练

 我们使用状态向量对系统进行建模，其中 state[i] 表示年龄为 i 分钟的细菌数量，i 从 0 到 19。

1. 使用 state[0] = 1 和所有其他条目初始化状态向量 0。 这反映了零时的单个新生细菌。 
2. 定义系统在一分钟内如何演化。 每个细菌的年龄都会增加一岁，因此状态[i]移动到状态[i+1]。 细菌在 19 岁时死亡并从系统中清除。 
3.处理复制。 任何成熟的细菌每四分钟即可繁殖一次。 由于成熟从 2 岁开始，因此在 20 分钟的生命周期内，生殖发生在 2、6、10、14 和 18 岁。 这些年龄对应于精确的繁殖阶段。 
4. 对于每个有繁殖资格的年龄，每个细菌在那一分钟内产生 3 个年龄为 0 的新细菌。 因此，新的状态[0]增加了这些繁殖年龄的细菌总数的3倍。 
5. 将这些规则组合成一个 20×20 转换矩阵 T，其中 T[i][j] 编码年龄为 j 的细菌在一分钟后如何影响年龄 i。 
6. 使用快速求幂计算 T^N。 
7. 将所得矩阵乘以初始状态向量以获得时间 N 的最终分布。 
8. 对所得状态向量的所有条目求和以获得存活细菌的总数。 

这样做的原因是整个系统作为固定有限维状态空间上的线性变换而演化。 每分钟都应用相同的确定性映射，因此重复应用正是矩阵供电。 

## 为什么它有效

 关键的不变量是，每分钟之后，状态向量充分总结了系统的所有相关信息。 除了年龄分布之外，不需要隐藏的历史，因为繁殖和死亡只取决于当前的年龄。 由于状态向量中的每个转换都是线性的，重复应用形成线性变换的半群，这保证了转换矩阵的幂可以正确地表示 N 个演化步骤，而不会丢失信息。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

SIZE = 20

def mat_mul(a, b):
    res = [[0] * SIZE for _ in range(SIZE)]
    for i in range(SIZE):
        for k in range(SIZE):
            if a[i][k]:
                aik = a[i][k]
                for j in range(SIZE):
                    res[i][j] = (res[i][j] + aik * b[k][j]) % MOD
    return res

def mat_pow(mat, exp):
    res = [[0] * SIZE for _ in range(SIZE)]
    for i in range(SIZE):
        res[i][i] = 1
    base = mat
    while exp > 0:
        if exp & 1:
            res = mat_mul(res, base)
        base = mat_mul(base, base)
        exp >>= 1
    return res

def solve_case(n):
    if n == 0:
        return 1

    T = [[0] * SIZE for _ in range(SIZE)]

    for i in range(1, SIZE):
        T[i][i - 1] = 1

    repro = [2, 6, 10, 14, 18]

    for r in repro:
        for j in repro:
            T[0][j] = (T[0][j] + 3) % MOD

    mat = mat_pow(T, n)

    state = [0] * SIZE
    state[0] = 1

    final = [0] * SIZE
    for i in range(SIZE):
        for j in range(SIZE):
            final[i] = (final[i] + mat[i][j] * state[j]) % MOD

    return sum(final) % MOD

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        print(solve_case(n))

if __name__ == "__main__":
    solve()
```该代码构建了一个固定的 20 状态转换系统，其中每个状态对应于一个可能的年龄。 移位结构处理衰老，而过渡矩阵的第一行聚集了所有符合繁殖年龄的新生细菌。 

矩阵求幂计算应用此转换 N 次的效果，最终求和计算所有存活的细菌。 

一个微妙的实现点是生育年龄的处理。 它们是根据生命周期限制明确枚举的，确保只有生命周期内的细菌才有助于新生。

 ## 边缘情况

 当 N = 0 时，系统必须返回恰好一种细菌。 状态向量在年龄为零时使用单个条目进行初始化，并且不应用任何转换，因此总和保持为 1。 

对于较小的 N，例如 N = 2 或 N = 3，一定不会发生复制。 转换矩阵正确地避免了未成熟年龄对新生儿的任何贡献，因此人口保持恒定为 1。 

当 N 接近 20 时，细菌开始死亡。 年龄转移结构确保状态[19]在每一步都会被丢弃，因此过期细菌的贡献永远不会被结转，即使繁殖较早活跃，也可以防止过度计数。
