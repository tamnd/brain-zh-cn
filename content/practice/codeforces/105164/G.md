 ## 解决方案
 ## 问题理解

 我们得到了一系列石塔，每个石塔都有一定的正高度。 两名玩家交替执行相同类型的动作。 在一个回合中，玩家选择一个大于零的整数X，并从每个仍然有石头的塔中减去X，但X不能超过当前最短的非空塔的高度。 达到零的塔将被有效地从进一步的游戏中移除。 

因此，移动完全取决于从所有剩余塔的底部剥离多少层的选择。 每一步都会统一减少所有活动塔，直到至少其中一个消失。 

无法做出有效动作的玩家就会失败，这种情况恰好发生在轮到他们之前所有塔都已归零或者不可能有正 X 的情况下。 

这些限制允许建造最多 100 万座塔楼，每座塔楼的高度最多为 100 万座。 这立即排除了任何重复逐层减去的模拟，因为在最坏的情况下，这将涉及多达 10^12 的原始操作。 

当所有塔都相等时，就会出现关键的边缘情况。 例如，如果所有高度都相同，则每次移动都会一次性完全删除所有剩余的塔。 另一个微妙的情况是当只剩下一个塔时，因为这样游戏就简化为从单个堆中重复减去一个正整数，其行为与多塔交互不同，但实际上与相同的结构一致。 

一个天真的错误是假设答案取决于高度的总和或最大值。 例如，对于塔 [1, 100, 100]，可能感觉大塔占主导地位，但最小的塔迫使第一步移动为 X = 1，之后结构以可预测的方式崩溃，这仅取决于存在多少个不同的“级别”。 

## 方法

 直接模拟将每次移动视为从所有塔中减去当前的最小高度并移除耗尽的塔。 这是正确的，因为任何最佳移动都必须始终选择完全允许的 X，否则您将给对手留下一个更好的位置，具有相同的塔结构，但剩余最小值更大。 然而，即使我们总是减去最小值，该过程仍然可能需要 O(max(ai)) 次迭代，当高度很大时，这太慢了。 

重要的观察是，每次移动都会完全移除至少一座塔，因为所选的 X 等于当前的最小高度，因此至少一座塔会变为零。 排序后，这相当于以递减的不同高度级别剥离数组。 每个不同的高度值恰好贡献一个“层移除事件”，并且游戏在处理完所有不同的高度后结束。 

因此，游戏长度仅取决于存在多少个不同的高度，而不取决于它们的大小或数量。 每个不同的值对应于强制移动，其中在移除所有塔后最小值增加等于该最小值。 

获胜者完全由此类减少次数的平价决定。 如果不同高度级别的数量为奇数，则第一个玩家完成最后一步并获胜。 如果相等，则第二个玩家获胜。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 层去除模拟| O(N * 最大(ai)) | O(N) | 太慢了 |
 | 计算不同的高度 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

1. 按非降序对塔高度列表进行排序，以便相同高度变得相邻。 这使我们能够检测移除层后何时发生最小变化。 
2. 遍历排序后的列表，统计该值与上一个相比发生了多少次变化。 每个变化对应于出现的新的不同高度水平。 
3. 将不同值的数量视为游戏中的移动次数。 这是有效的，因为当每个不同的高度变为最小值时，它会在一次强制操作中被消除。 
4. 如果不同高度的数量是奇数，则宣布 Aliius 获胜。 否则，鲍比斯获胜。 

关键的原因是，每次移动后，至少一个完整的高度层会消失，并且任何高度层都不会重新出现或以改变未来结构的方式部分分裂。 因此，游戏状态通过排序的不同值单调崩溃。 

### 为什么它有效

 不变的是，在 k 次移动之后，恰好 k 个不同的高度级别已被完全删除，并且剩余的塔精确对应于排序的唯一高度列表的后缀。 没有决策点允许跳过级别或合并级别，因为 X 被迫等于当前最小值。 这使得移动序列在相同元素的排序上具有确定性，因此整个游戏简化为计算随着时间的推移存在多少个不同的最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n_and_rest = input().split()
    if not n_and_rest:
        return
    n = int(n_and_rest[0])
    arr = list(map(int, n_and_rest[1:]))

    # if input didn't fully come in first line
    while len(arr) < n:
        arr.extend(map(int, input().split()))

    arr.sort()

    moves = 0
    prev = None

    for x in arr:
        if x != prev:
            moves += 1
            prev = x

    if moves % 2 == 1:
        print("Alicius")
    else:
        print("Bobius")

if __name__ == "__main__":
    solve()
```该实现首先确保稳健的输入解析，因为值可能跨越多行。 排序将相同的高度集中在一起，因此可以在单次扫描中检测到过渡。 变量`moves`计算不同的高度组，这直接对应于强制移除的数量。 

一个微妙的点是处理输入，其中第一行可能包含两者`N`和数组的一部分，这就是解析器累积值直到达到 N 的原因。

 ## 边缘情况

 对于像这样的单塔`[7]`，排序给出一个元素，因此只有一个不同的值。 该算法计算出一步棋，结果是奇数，因此阿利修斯获胜。 这符合第一个玩家可以选择 X = 7 并立即结束游戏的事实。 

对于所有相同的塔，例如`[5, 5, 5]`，排序仍然会产生一个不同的值。 仅发生一次移动：减去 5 一次，所有塔都会消失。 该算法输出 Alicius，它匹配最佳玩法，因为第一步也是最后一步。 

对于交替重复项，例如`[1, 1, 2, 2, 3, 3]`，不同计数为 3。 第一个玩家通过结构间接执行前两次归约，由于 3 是奇数，Alicius 获胜。 这表明多样性不会影响游戏长度，只有不同的高度水平才会影响游戏长度。
