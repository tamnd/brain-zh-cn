 ## 解决方案
 ## 问题理解

 我们得到了一小部分卡片，每张卡片都带有两个独立的属性：用于平衡的点值和用于评分的利润值。 该游戏是 Alice 和 Bob 之间的两阶段交互。 首先，Bob 可以选择修改最多 k 张牌，将其点值加倍。 修改后，Bob 必须将纸牌的某个子集划分为两个不相交的组，以使两组的总点数和相等。 任何一组未选择的牌都将被丢弃。 Alice 接收一组，Bob 接收另一组，但从问题的角度来看，两个玩家一起收集分区中使用的所有卡的值。 

目标是最大化最终划分中包含的所有牌的总价值，前提是在应用最多 k 次加倍后存在有效的等和划分。 

输入大小很小，n 最大为 100，点值以 13 为界。这立即表明指数或伪多项式背包式解决方案是合理的，但任何涉及所有修改的完整子集枚举的内容都必须小心控制。 对所有牌子集和所有双倍集选择的简单枚举将涉及最多 2^100 个状态，这是不可行的。 即使在顶部添加分区检查也会增加成本。 

一个微妙的点是，加倍会改变奇偶校验和幅度，并且最终的分区约束仅取决于调整点值的多重集。 粗心的方法常常会失败，因为它会将分区视为整套卡上的标准子集和问题，而忽略了只有选定的子集需要完美平衡的事实，而其他卡可能被完全排除。 另一个常见的错误是假设必须使用所有卡，这会错误地强制整个数组而不是选定的子集相等。 

例如，如果卡片是 [t = 1, 3, 1, 1]，值为 [10, -5, 5, 6]，则不需要将所有四张卡片均分。 如果某些卡阻碍了平衡分区，我们可能会完全丢弃它们。 这种区别至关重要。 

## 方法

 暴力破解的想法从选择要加倍的牌开始，然后对于修改后的数组尝试找到总和为偶数并且可以分成相等的两半的子集。 这本质上是检查是否存在一个子集，其总和等于某个选定子集总和的一半，同时也最大化总价值。 直接暴力将迭代所有 2^n 子集，然后对于每个子集测试所有 2^k 加倍选择，然后运行子集总和 DP 或枚举来验证可分区性。 即使 n = 100，这也会远远超出可行性，因为在最坏的解释中状态空间变为 2^100 × 2^100。 

关键的结构见解是分区条件仅取决于所选项目的最终多重集，而不取决于 Alice 或 Bob 的单独分配。 如果我们修复目标差异解释，问题就变成选择卡片的子集并为每个选定的卡片分配 + 或 - 符号，使得签名和为零。 加倍操作仅影响个体贡献的大小，并且仅限于 k 个选择。 

以这种方式重新构建，每张选定的卡都会贡献+t或-t，并且加倍将t更改为2t。 分区条件变为有符号子集和等于零约束，目标是最大化所选项目的总价值，而不管符号如何。 这将问题变成了由当前余额差异和使用的加倍次数定义的状态上的背包问题。

由于 ti ≤ 13，所有牌的最大总和足够小以限制可能的不平衡。 我们可以将 DP 状态视为 (i, d,used)，其中 i 是已处理索引，d 是两个分区之间的当前差值，used 是双倍卡的数量。 每个转换都会考虑跳过一张牌，将其放入左组或右组，并在允许的情况下选择将其加倍。 天平尺寸相应地移动 t 或 2t。 

因此，问题简化为具有有限差异范围和小 k 的约束多维 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 子集暴力破解+加倍+分区检查 | O(2^n·2^k·2^n) | O(2^n·2^k·2^n) | O(n) | 太慢了 |
 | 指数DP×余额差异×已用倍数| O(n·S·k)，其中 S ≤ 1300 | O(S·k) | 已接受 |

 ## 算法演练

 1. 我们定义一个动态编程状态，用于跟踪列表中的进度、两个分区之间当前存在的不平衡情况以及到目前为止已使用了多少次加倍。 不平衡是分配给两侧的总和之间的有符号差。 
2. 在处理任何卡片之前，我们用单一状态初始化 DP，其中不平衡性为零并且不使用加倍。 这对应于从空分区开始。 
3. 对于每张卡，我们考虑三个结构决策：完全排除该卡、将其分配给第一个分区或将其分配给第二个分区。 排除是必要的，因为最优解决方案可能不会使用所有卡。 
4. 当将卡牌分配到分区时，我们通过添加或减去其点值来更新不平衡性。 如果我们选择将卡加倍，则只有当我们有剩余的加倍预算时，我们才允许这种转换，并且我们使用两倍的值来更新不平衡。 
5. 我们维护一个滚动 DP，以便在处理每张牌后，我们只为每个可能使用的加倍数量保留可达到的不平衡状态。 这避免了指数的指数爆炸。 
6. 处理完所有卡片后，我们查看最终不平衡为零的所有状态，这意味着两个分区的总和相等。 对于每个这样的状态，我们计算在该状态中实际选择的所有卡所贡献的总价值，并取最大值。 

### 为什么它有效

 DP 将每个可能的有效卡分配编码为左、右或未使用的类别，同时遵守加倍约束。 不平衡变量是分区条件的精确不变量：它始终等于任何阶段所构造的两个子集之间的差。 当处理选定的子集后不平衡变为零时，状态恰好达到有效性。 由于每个转换都对应于游戏中的合法动作，并且每个合法配置都可以由某些转换序列表示，因此不会错过任何有效的解决方案，并且不会出现无效的零不平衡最终状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    cards = [tuple(map(int, input().split())) for _ in range(n)]

    # maximum possible sum of t is 100 * 13 = 1300
    MAXD = 1300

    # dp[used][diff] = max value sum achievable
    dp = [[-10**18] * (2 * MAXD + 1) for _ in range(k + 1)]
    offset = MAXD

    dp[0][offset] = 0

    for vi, ti in cards:
        new = [[-10**18] * (2 * MAXD + 1) for _ in range(k + 1)]

        for used in range(k + 1):
            for d in range(2 * MAXD + 1):
                if dp[used][d] < -10**17:
                    continue

                cur = dp[used][d]

                # 1. skip card
                if cur > new[used][d]:
                    new[used][d] = cur

                # 2. take without doubling
                nd = d + ti
                if 0 <= nd <= 2 * MAXD:
                    if cur + vi > new[used][nd]:
                        new[used][nd] = cur + vi

                nd = d - ti
                if 0 <= nd <= 2 * MAXD:
                    if cur + vi > new[used][nd]:
                        new[used][nd] = cur + vi

                # 3. take with doubling
                if used < k:
                    nd = d + 2 * ti
                    if 0 <= nd <= 2 * MAXD:
                        if cur + vi > new[used + 1][nd]:
                            new[used + 1][nd] = cur + vi

                    nd = d - 2 * ti
                    if 0 <= nd <= 2 * MAXD:
                        if cur + vi > new[used + 1][nd]:
                            new[used + 1][nd] = cur + vi

        dp = new

    ans = 0
    for used in range(k + 1):
        ans = max(ans, dp[used][offset])

    print(ans)

if __name__ == "__main__":
    solve()
```该实现使用一个 2D DP，该 DP 通过已使用的加倍次数以及两个分区之间的当前不平衡进行索引。 不平衡会移动一个偏移量，以便负值安全地存储在数组索引中。 每张卡最多转换为五种可能性：跳过、向左分配、向右分配或加倍分配左/右，后者受剩余 k 预算的限制。 

一个微妙的实施细节是，即使不平衡发生变化，我们也会继续实现最佳的可实现价值总和，因为选择一张卡牌总是会贡献其价值，无论它流向哪一边。 这符合所有使用过的卡片都贡献于最终答案的问题要求。

 输入：

 卡：

 (10,1), (-5,3), (5,1), (6,1)

该轨迹表明，不平衡的演变独立于总价值积累，并且只有最终的零差异状态才重要。

 ## 边缘情况

 一种重要的边缘情况是，除非排除某些卡，否则不存在有效分区。 DP 自然地处理这个问题，因为跳跃转换保留状态而不强制包含。 例如，如果所有卡都具有奇数总结构以防止相等，则除非应用排除，否则 DP 将永远不会达到 diff = 0。 

另一种边缘情况是需要加倍才能实现任何解决方案。 在这种情况下，used = k 的状态至关重要，并且不跟踪已使用计数器的解决方案将错误地错过有效分区。 DP 通过使用的计数显式地分离状态，确保遵守恰好 k 的约束并考虑加倍的最佳使用。 

最后的边缘情况是多种配置产生 diff = 0 但值总和不同。 DP 存储每个状态的最大值，确保最后只选择最佳的有效分区，而不是停在第一个可行的配置处。
