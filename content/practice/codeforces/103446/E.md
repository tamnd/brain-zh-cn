 ## 解决方案
 ## 问题理解

 我们给出一个整数序列和一个阈值$k$。 从这个序列中，我们想要选择一个子序列（保留原始索引），使得每对选定值的值“接近”：任何两个选定元素之间的绝对差必须最多为$k$。 目标是最大化在此约束下我们可以选择的元素数量。 

关键限制是整个所选集合的全局限制，而不仅仅是连续元素之间的限制。 如果我们选择三个值$x, y, z$，那么所有三对差异必须在$k$，不仅$|x-y|$和$|y-z|$。 这使得结构仅取决于所选集合中的值的范围。 

输入大小达到$n = 10^5$，所以任何二次或偶数$O(n \log n)$如果结构不仔细，重复比较所有对的解决方案将会很困难。 理想的解决方案应该排序一次，然后线性或使用滑动窗口扫描。 

出现微妙的边缘情况时$k = 0$。 在这种情况下，我们只能选择相同的值。 答案成为任何数字的最大频率。 例如，如果数组是$[1, 2, 2, 3]$和$k = 0$，正确答案是$2$。 尝试基于邻接性扩展子序列的天真贪婪会错误地选择更多元素。 

当所有值都位于一个小区间内时，会出现另一种边缘情况。 例如，如果所有$A_i = 10^9$，那么任何子集都是有效的，答案是$n$。 正确的解决方案不得意外丢弃重复项或依赖于严格的不等式假设。 

## 方法

 暴力的想法是尝试数组的每个子集，并检查它是否满足所有成对差异最多的条件$k$。 对于每个子集，验证有效性需要扫描所有选定的元素并跟踪最小值和最大值。 子集恰好在以下情况下有效：$\max - \min \le k$。 

这导致了子集的指数数量，大致$2^n$，每次验证成本高达$O(n)$。 即使减少到仅隐式检查子集，这种方法也几乎立即变得不可行。 

条件的结构揭示了这种简化：条件仅取决于所选值的范围。 一旦我们对数组进行排序，任何有效的子集都对应于按排序顺序的连续段，其中最小和最大元素之间的差异最多为$k$。 这将问题转化为找到最长的此类线段。 

排序之后，我们就可以使用两指针滑动窗口了。 我们扩展右端点并保持最左边的位置，以使窗口保持有效。 每当差异超过$k$，我们向前移动左指针，直到条件恢复。 在此过程中看到的最大窗口大小就是答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 排序+滑动窗口|$O(n \log n)$|$O(1)$或者$O(n)$| 已接受 |

 ## 算法演练

 ### 步骤

 1. 对数组进行非降序排序。 排序确保任何有效子集都可以被视为此排序中的连续区间，因为扩展范围只会增加最大最小差异。 
2.初始化两个指针$l = 0$,$r = 0$，和一个变量$ans = 1$。 窗户$[l, r]$代表当前的候选集。 
3. 移动$r$从左到右穿过阵列。 在每个步骤中，包括$A[r]$进入当前窗口。 
4. 当当前窗口违反条件时$A[r] - A[l] > k$, 增量$l$。 这将从左侧缩小窗口，直到它再次有效。 原因是任何元素距离太远$A[r]$已排序的数组中不能保留在同一有效集合中。 
5.修复窗口后，更新$ans = \max(ans, r - l + 1)$。 这跟踪了迄今为止看到的最大的有效子集。 

### 为什么它有效

 排序后，任何选定集合的最大和最小元素必须按排序顺序来自该集合的端点。 如果窗口违反了约束，则最左边的元素始终是要删除的最佳候选元素，因为它对范围的贡献最大。 移除任何内部元件并不会有效地减少射程。 该算法维护一个对于当前右端点始终是最大的窗口，因此探索所有可行的最大子集而不会丢失候选者。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    a.sort()
    
    l = 0
    ans = 1
    
    for r in range(n):
        while a[r] - a[l] > k:
            l += 1
        ans = max(ans, r - l + 1)
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先对数组进行排序，这很重要，因为它将全局成对约束转变为范围约束。 排序后，我们只需要跟踪候选集中的最小和最大值，它们对应于连续段的端点。 

两指针循环维护一个有效的窗口。 内层的while循环是左指针唯一移动的地方，每个元素最多被移除一次，保证排序后的线性复杂度。 

一个常见的错误是忘记排序并尝试在原始数组上使用滑动窗口。 这会失败，因为条件取决于值范围，而不是索引距离。

 ## 边缘情况

 当所有元素都相同时，例如输入$[5, 5, 5, 5]$与任何$k \ge 0$，排序后的数组保持不变，窗口永远不会缩小。 算法不断扩展$r$并且总是产生$ans = n$，匹配正确的结果。 

什么时候$k = 0$， 例如$[1, 2, 2, 3, 2]$，排序给出$[1, 2, 2, 2, 3]$。 值 2 的窗口将扩展以包含所有出现的情况，而 1 和 3 由于违规而被立即排除。 该算法自然地隔离了最大频率块。 

当值相差很大时，例如$[1, 100, 200, 300]$和$k = 50$，每个窗户都很小。 算法反复移位$l$向前，确保没有无效的大范围集合对答案有贡献，并正确返回 1。
