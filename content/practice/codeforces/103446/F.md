 ## 解决方案
 ## 问题理解

 我们得到了由递归生成的大量整数多重集。 从概念上讲，可以将其视为一个包含许多球的盒子，每个球都标有一个值。 这些值不是任意输入，而是由确定性生成器生成，因此 Kaiji 在游戏开始之前就完全知道该序列。 

然后，对手在限制下选择两个球：所选择的对不能有任何第三个球的值严格位于它们之间。 换句话说，如果对所有值进行排序，则攻击者只能选择两个相等的值，或者按不同值的排序顺序相邻的两个值。 此限制迫使每个选定的对要么是相同的值，要么是值谱中的“相邻”对。 

Kaiji 没有看到所选的一对。 相反，他首先选择两只手中的一只进行检查，了解所选球的值，然后必须说明另一只手中的隐藏值是更小、相等还是更大。 

目标是在双方最佳玩法下最大化 Kaiji 正确回答的概率：Kaiji 尝试选择保证最佳成功率的策略，而对手则选择最小化成功率的对。 

生成器为每个测试用例生成最多 1000 万个值，并且最多可以有 1000 个测试用例，总共有 1000 万个值。 如果它使用繁重的数据结构或超出线性扫描的每次测试开销，这会立即排除任何以简单方式存储或排序所有测试中的所有值的解决方案。 仅当每个操作都是恒定时间时，每个测试方法的 O(n) 才是可接受的。 

当所有生成的值都相同时，会出现微妙的边缘情况。 在这种情况下，每个有效对都包含相等的值，因此 Kaiji 始终可以以概率 1 正确回答“相等”。任何假设至少两个不同值的方法都会错误地将这种情况降级为概率结果。 

## 方法

 直接模拟游戏是不可行的。 即使我们修复了一对，Kaiji 的决策过程也存在不确定性，即他将检查哪个元素以及该值如何与另一个元素相关。 枚举所有对是不可能的，因为存在 O(n²) 种可能性。 

相反，允许对的结构是关键的简化。 对手只能在不同值的排序列表中选择相同值或相邻值的对。 这意味着，从 Kaiji 的角度来看，对手引入的唯一不确定性来自相邻不同价值观之间的转变。 

如果整个数组中只有一个不同的值，那么游戏就变得微不足道：每一对都相等，因此 Kaiji 总是通过陈述相等来正确回答。 

如果至少有两个不同的值，对手总是可以避免给 Kaiji 任何确定性的信息优势。 每当 Kaiji 观察到一个值时，隐藏值就可以对应于按不同顺序排序的相邻值。 从 Kaiji 的角度来看，每当对手选择一个不相等的对时，情况就会简化为在局部价值链中区分方向，其中最佳可能的成功概率会崩溃为在两个一致的可能性之间进行猜测。 

关键的观察是，一旦存在至少两个不同的值，任何策略都无法将成功概率提高到一半以上。 对手总是可以选择一对相邻的不同值，从而在“这个较小，另一个较大”与相反结构之间产生歧义，具体取决于哪只手被揭示。 由于 Kaiji 在仅看到具有对称不确定性的二元结构的一个端点后被迫做出承诺，因此最优策略简化为二元猜测。 

因此，整个问题简化为检查所有生成的值是否相等。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解对 | O(n²) | O(n) | 太慢了 |
 | 最佳独特检查 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 使用递归生成序列，同时跟踪所有值是否保持相同。 我们将第一个值初始化为参考基线，并将每个后续值与其进行比较。 
2. 如果任何生成的值与基线不同，我们立即知道数组中至少存在两个不同的值。 从此时开始，进一步的生成可以继续，但最终的决定已经确定。 
3. 处理完所有值后，根据是否观察到第二个不同值来决定答案。 
4. 如果没有观察到差异，则输出 1，因为每个有效对都是相等的，并且 Kaiji 总是正确回答。 
5. 否则，输出 998244353 下 2 的模逆，因为最佳保证成功概率恰好是二分之一。 

扫描足够的原因是对手的限制仅取决于值之间的顺序，并且一旦存在多个不同的值，该结构就会完全崩溃。 

### 为什么它有效

 对手的约束确保每个选定的对在排序值顺序中局部一致，但这种局部性不会为 Kaiji 创建超出相等性检测的可利用结构。 如果存在多个不同值，则至少存在一对有效的相邻不同值。 对手总是可以选择这样的一对，并且对于该对，Kaiji 的观察简化为对称的双结果推理问题。 由于 Kaiji 必须在仅观察一侧后做出承诺，因此任何确定性策略都无法在最坏情况下以大于二分之一的概率区分不平等的方向。 唯一的例外是当不存在此类邻接时，这种情况恰好在所有值都相同时发生。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
INV2 = 499122177

def solve():
    it = iter(sys.stdin.read().strip().split())
    t = int(next(it))
    out = []

    for _ in range(t):
        n = int(next(it))
        a0 = int(next(it))
        A = int(next(it))
        B = int(next(it))
        C = int(next(it))
        M = int(next(it))

        first = a0
        prev = a0
        all_same = True

        for i in range(2, n + 1):
            prev = (A * prev * prev + B * prev + C) % M + 1
            if prev != first:
                all_same = False

        if all_same:
            out.append("1")
        else:
            out.append(str(INV2))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现仅保留重复的前一个值，避免任何数组存储。 这是至关重要的，因为 n 可以达到 1000 万。 我们维护的唯一状态是所有生成的值是否与第一个值匹配。 

模块化答案使用 998244353 下预先计算的 2 的倒数，因为这是存在多个不同值时所需的输出。 

## 工作示例

 考虑所有值都相同的情况。 生成器生成类似 5, 5, 5, 5 的序列。扫描永远不会检测到不匹配，因此最终状态仍然是 all_same = True。 

| 步骤| 当前值| 第一个值 | 全部相同 |
 | --- | --- | --- | --- |
 | 1 | 5 | 5 | 真实|
 | 2 | 5 | 5 | 真实|
 | 3 | 5 | 5 | 真实|
 | 4 | 5 | 5 | 真实|

 输出为1，反映了正确性的确定性。 

现在考虑至少有两个不同值的情况，例如 3、3、7、3。当 7 出现时，不变量就被打破。 

| 步骤| 当前值| 第一个值 | 全部相同 |
 | --- | --- | --- | --- |
 | 1 | 3 | 3 | 真实|
 | 2 | 3 | 3 | 真实|
 | 3 | 7 | 3 | 假 |
 | 4 | 3 | 3 | 假|

 一旦 all_same 变为 False，序列的其余部分就无关紧要了。 答案变成1/2。 

这表明只有第二个不同值的存在才重要，而不是它的频率或位置。

 ## 边缘情况

 结构上唯一的特殊情况是所有生成的值都相同。 在这种情况下，算法永远不会翻转 all_same 标志，因此它输出 1。 

对于至少一个生成值不同的任何输入，该标志在第一次分歧时恰好变为 False，并保持为 False。 该算法不依赖于分歧发生的位置，只依赖于分歧的存在。 

代表性执行是输入 2 1 0 1 0 2，产生 1, 2。扫描立即发现不匹配并切换到概率结果 1/2，与对抗性保证分析相匹配。
