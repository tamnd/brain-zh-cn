 ## 解决方案
 ## 问题理解

 每个项目给出一个索引为的函数$i$，由两个参数定义$k_i$和$a_i$。 该函数是周期性的$x$，其形状由变换后的切线表达式决定，涉及$\sec(x-a_i)$。 因为$\sec(x-a_i)=1/\cos(x-a_i)$，该函数在任何时候都会重复奇异行为$\cos(x-a_i)=0$，并且在这些奇点之间它表现得很平稳。 

里面的表情$\arctan$当$|\sec(x-a_i)|$增长，这发生在$x$接近点$\cos(x-a_i)$接近于零。 因此，每个函数都有一个重复的“谷”结构：当它达到最小值时$\cos(x-a_i)$绝对值最大化，并朝着垂直渐近线附近的共同上限增长。 

对于每个索引$i$，我们被问到是否存在一些真正的价值$x_i$这样函数$f_i(x_i)$严格小于后面的每个函数$f_j(x_i)$为所有人$j>i$。 换句话说，我们想知道函数是否$i$可以成为后缀中严格的最小值$\{i, i+1, \dots, n\}$通过在实线上选择一个点。 

输入大小可达$n=10^5$，因此任何在多个点对所有函数成对评估候选者的解决方案将立即变成二次方。 甚至$O(n \log n)$解决方案必须仔细构建，因为每个函数比较不是简单的标量比较，而是取决于连续变量$x$。 

一个微妙的困难是这些功能并不独立于$a_i$，但周期结构意味着移位$x$的倍数$\pi$重复行为。 幼稚的方法可能会错误地假设只有$k_i$事项或订购方式$a_i$就足够了。 例如，两个具有相同功能的函数$k$但不同的转变可以在不同的地区相互主导$x$，因此仅在特殊点进行评估，例如$x=a_i$可能会错过有效证人或产生误报。 

## 方法

 蛮力的观点很简单：对于每个$i$，我们尝试找到一个$x$这样$f_i(x)$小于所有后缀函数。 如果我们通过考虑函数改变行为的所有“重要”位置来离散化候选点，我们最终会检查从每个点派生的许多点$a_i$以及每个渐近线位置。 评估每个候选的所有后缀比较很快会导致$O(n^2)$评估，因为每次检查都需要扫描所有后续功能。 

关键的观察结果是复杂的三角结构的表达具有误导性。 尽管依赖于$a_i$，每个函数共享相同的全局范围行为：它们都在渐近线附近实现相同的上包络线，并且它们的区分强度仅由$k_i$，它衡量了函数达到最小值的“低”程度。 

在任何固定的$x$，函数之间的差异取决于内部乘法因子的大小$\arctan$变成。 转变$a_i$仅确定每个函数在何处达到最佳值，但不会改变可实现的最低水平，即$\arctan(k_i)/\pi$。 这使得$k_i$唯一可以在所有可能的情况下建立永久支配关系的参数$x$。 

一旦我们以这种方式重新解释问题，“存在$x$在哪里$f_i(x)$在后缀中严格最小”减少到后缀优势条件$k_i$。 较大的函数$k$在其最小值时总是会比具有较小值的函数变得更糟$k$，而一个较小的函数$k$永远不可能在其周期结构的所有对齐点上始终低于较大的值。 

这将连续问题折叠成一个简单的后缀最大查询$k_i$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解候选点 |$O(n^2)$|$O(n)$| 太慢了|
 | 后缀最大超过$k_i$|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 从右向左遍历函数，同时保持 的最大值$k$到目前为止所看到的。 这代表了后缀中最强的功能，就击败其最小值的“难度”而言。 
2. 对于每个索引$i$， 比较$k_i$与最大$k$严格位于后缀之后$i$。 如果$k_i$至少与后面的所有值一样大，将其标记为可行。 
3. 否则，将其标记为不可行，因为存在一个后来的函数，其尺度参数在可能发生最小对齐的每个区域中占主导地位。 

这种贪心后缀比较足够的原因是$k_i$完全确定函数可以达到多低，并且没有移位$a_i$当与较大的后缀元素竞争时，可以补偿此参数中严格较小的情况。 

### 为什么它有效

 每个函数都有一个固定的最佳可能值，由下式确定$k_i$，并且所有函数都具有相同的最坏情况上限。 如果后面的函数具有严格更大的$k_j$，它可以被定位（通过周期性最小值的适当对齐）以实现不高于允许的任何配置的值$f_i$最小化。 由于该问题允许选择单个$x$，唯一的办法$i$可以战胜所有后缀元素的条件是，如果后面的函数没有严格更大的缩放能力，可以在其最佳对齐点削弱它。 这将后缀的优势减少到最大$k$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    k = []
    a = []
    for _ in range(n):
        ki, ai = map(int, input().split())
        k.append(ki)
        a.append(ai)

    ans = ['0'] * n
    suffix_max = 0

    for i in range(n - 1, -1, -1):
        if k[i] >= suffix_max:
            ans[i] = '1'
        else:
            ans[i] = '0'
        suffix_max = max(suffix_max, k[i])

    print(''.join(ans))

if __name__ == "__main__":
    solve()
```实现忽略$a_i$完全是因为主导条件简化为仅比较尺度参数。 向后遍历保证了我们处理索引时$i$，我们已经知道最大值$k_j$为所有人$j>i$。 

唯一的微妙之处是允许相等，因为具有 equal 的函数$k$可以对齐以实现匹配的最小行为，并且在任一方向上都不能保证严格的优势。

 ## 边缘情况

 一个关键的边缘情况是当所有$k_i$值是相同的。 在这种情况下，没有一个函数严格支配另一个函数，因此每个索引仍然有效。 该算法可以正确处理此问题，因为后缀最大值永远不会超过当前值，从而生成所有值。 

另一种情况是严格递减序列$k_i$。 这里每个较早的函数都由至少一个较晚的函数控制，因此只有第一个索引幸存下来。 后缀扫描自然会产生这种模式，因为后缀最大值始终大于当前值，直到数组末尾。 

最后一个微妙的情况是$k_i$波动。 该算法仍然有效，因为它仅取决于右侧是否存在更大的值，并且不会尝试对位置效应进行建模$a_i$，这不会影响这种简化解释中的优势条件。
