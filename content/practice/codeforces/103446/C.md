 ## 解决方案
 ## 问题理解

 我们给出一个非常小的网格，最多 8 × 8，其中每个单元格要么固定为 0，要么固定为 1，要么灵活并标记为 2。每个 2 可以独立地变为 0 或 1，因此通过决定所有这些替换来选择最终矩阵。 

固定矩阵后，我们只查看最终为零的单元格。 从这些零单元中，我们必须挑选一个子集S，并且S必须满足结构要求：网格中的每个零单元必须由S中同一行或同一列中的至少一个选定单元“支持”，并且它们之间的矩形区域必须仅包含零。 

具体来说，如果我们在S中选取一个单元格（u，v），那么如果它们共享行或列，它可以覆盖零单元格（i，j），并且由这两个点形成的轴对齐矩形内的每个单元格仅包含零。 如果每个零单元格都被至少一个选定的锚点以这种方式覆盖，则集合 S 是有效的，包括 S 中的单元格本身（它们可以简单地覆盖自己）。 

完整矩阵的值是此类有效 S 的最小可能大小。由于我们可以选择每个 2 如何变为 0 或 1，因此我们希望选择一个能够最小化此最小 S 大小的完整矩阵。 

约束条件非常严格：n 和 m 最多为 8，因此网格最多有 64 个单元格。 这立即表明解决方案可以对整个网格的单元子集或状态进行指数推理。 64 中的任何多项式在较小的维度上具有额外的指数因子仍然是可以接受的，但是所有 64 单元中没有结构的任何指数都必须避免或大量修剪。 

天真的阅读可能会建议尝试对 k 个二元组进行所有 2k 次替换，然后每次都以最佳方式求解矩阵。 这已经引入了第二个指数层。 即使 k 只是中等大，这种方法很快就会变得不可行。 

一个更微妙的陷阱是假设最终优化中的零和一是对称的。 他们不是。 一个充当硬阻挡器：如果覆盖所需的任何矩形穿过一个，则该候选覆盖关系将被破坏。 如果粗心的解决方案忽略了这一点并假设所有 2 都可以安全地设置为零，则会高估连接性并产生错误的答案。 

另一种常见的失败情况是假设每个零单元可以独立选择其最佳覆盖锚点。 这会失败，因为锚点相互作用：在 S 中选择一个单元格可以同时覆盖许多单元格，并且最优 S 是一个全局选择问题，而不是每个单元格的贪婪选择。 

## 方法

 暴力策略是将 2 个单元格的所有可能替换枚举为 0 或 1。对于每个完成的矩阵，我们通过搜索零单元格的子集并检查它们是否满足覆盖规则来计算有效集合 S 的最小大小。 这是正确的，因为它直接遵循定义，但速度太慢。 如果有 k 个灵活单元，这会引入 2k 个矩阵，并且对于每个矩阵，S 会有另外 2n·m 个子集，在最坏的情况下会导致高达 2^128 的爆炸。 

关键的简化来自于这样的观察：将 2 变成 1 对任何基于矩形的覆盖条件没有任何帮助。 1 只会破坏零对之间的可见性，而零只会增加或保持可见性。 因此，为了最小化 S，将每个 2 视为 0 始终是最佳的。这完全删除了第一个指数层并将问题简化为固定的二进制矩阵。 

此后，问题就变成了一组固定的零单元的纯粹组合问题：我们需要选择一个最小子集 S，使得每个零单元都可以在全零矩形可见性规则下被 S 中至少一个选定的锚点“看到”。

我们可以将其重新表述为集合覆盖问题。 每个候选锚小区 s 定义一个覆盖集合，由它在该规则下可以覆盖的所有零小区组成。 我们想要选择其并集覆盖所有零单元格的最小数量的锚点。 由于网格最多有 64 个单元，因此我们可以使用位掩码表示覆盖范围并对候选执行子集 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解分配和子集 | O(2^k · 2^(n·m) · n·m) | O(n·m) | 太慢了 |
 | 位掩码集覆盖覆盖范围集| O(2^(n·m) · n·m) | O(2^(n·m) · n·m) | O(2^(n·m)) | O(2^(n·m)) | 已接受 |

 ## 算法演练

 ### 1. 标准化矩阵

 我们将每个 2 转换为 0。这是安全的，因为将灵活单元格转换为 0 只能增加有效覆盖矩形的数量，并且永远不会使现有的 0 要求失效。 由于我们正在最小化 S 的大小，因此扩大零区域只能帮助或保持答案不变。 

### 2. 识别宇宙

 我们收集标准化后为零的所有单元格。 这些是必须涵盖的要素。 设该集合为U。 

### 3. 预先计算单元格之间的可见性

 对于每个有序的零单元对 (s, p)，我们检查 s 是否可以覆盖 p。 该条件要求它们位于同一行或同一列，并且它们之间的轴对齐矩形中的每个单元格也必须为零。 

此步骤将几何规则编码为简单的布尔关系。 此后，问题的所有结构都包含在类似图形的关系中，而不是网格本身。 

### 4. 构建覆盖掩模

 对于每个零单元 s，我们构造一个位掩码 Cover(s)，其中如果 s 可以覆盖 U 中的第 i 个零单元，则第 i 位为 1。我们还确保每个单元都覆盖自身，因为在 S 中选择 s 应该始终允许它满足自己的要求。 

### 5. 求解这些集合的最小覆盖率

 我们现在需要选择单元的子集 S，使得 S 中 s 上的 Cover(s) 的并集等于 U 的全部。这是最多 64 个元素和最多 64 个集合的经典最小集覆盖。 

我们对所覆盖元素的子集使用动态规划。 每个状态代表哪些小区已经被覆盖，并且转换尝试添加一个锚小区。 

### 为什么它有效

 正确性取决于两个属性。 首先，用 0 替换所有 2 并不会消除任何潜在的最佳解决方案，因为任何使用 2 作为 1 的解决方案只能限制覆盖范围，而永远不会改善它。 其次，一旦矩阵固定，每个有效的 S 都精确对应于预先计算的 Cover(s) 关系下的 U 的一组覆盖。 DP探索了选择此类盖子的所有可能方法，因此不会错过最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [list(input().strip()) for _ in range(n)]

    # treat all '2' as '0'
    for i in range(n):
        for j in range(m):
            if grid[i][j] == '2':
                grid[i][j] = '0'

    cells = []
    idx = [[-1] * m for _ in range(n)]

    for i in range(n):
        for j in range(m):
            if grid[i][j] == '0':
                idx[i][j] = len(cells)
                cells.append((i, j))

    k = len(cells)
    if k == 0:
        print(0)
        return

    # precompute rectangle validity
    def ok_rect(i1, j1, i2, j2):
        if i1 == i2:
            lo, hi = sorted([j1, j2])
            for y in range(lo, hi + 1):
                if grid[i1][y] != '0':
                    return False
            return True
        if j1 == j2:
            lo, hi = sorted([i1, i2])
            for x in range(lo, hi + 1):
                if grid[x][j1] != '0':
                    return False
            return True
        return False

    cover = [0] * k

    for s in range(k):
        x1, y1 = cells[s]
        for p in range(k):
            x2, y2 = cells[p]
            if ok_rect(x1, y1, x2, y2):
                cover[s] |= (1 << p)

    FULL = (1 << k) - 1
    INF = 10 ** 9

    dp = [INF] * (1 << k)
    dp[0] = 0

    for mask in range(1 << k):
        if dp[mask] == INF:
            continue
        for s in range(k):
            new_mask = mask | cover[s]
            if dp[new_mask] > dp[mask] + 1:
                dp[new_mask] = dp[mask] + 1

    print(dp[FULL])

if __name__ == "__main__":
    solve()
```该解决方案首先将所有灵活单元折叠为零，然后构建所有零位置的列表。 它根据两个单元格是否位于同一行或同一列且它们之间的段完全为零来计算成对可达性关系。 该关系被编码为位掩码，以便每个单元准确地知道它可以覆盖哪些其他单元。 

然后，动态编程将每个位掩码视为覆盖状态。 在任何状态下，添加新的锚点都会合并到它可以覆盖的所有单元格中。 最终的答案是达到全覆盖掩模所需的最少锚点数量。 

一个微妙的点是，DP 并不强制要求所选择的锚点在任何结构意义上都是不相交的或最小的。 这是正确的，因为 S 中允许重叠，并且通过最小化自然地消除了冗余。 

## 工作示例

 考虑一个小网格，其中零形成一个干净的行。 

### 示例 1

 输入网格：```
0 0 0
1 0 1
0 0 0
```所有零都是有效的候选者。 中间行单元格可以水平覆盖其他几个单元格。 

| 步骤| 面膜| 选择S | 覆盖|
 | --- | --- | --- | --- |
 | 开始 | 000000000 | {} | {} |
 | 添加中心 | 000111000 | {(1,1)} | 中排|
 | 添加角落 | 111111111 | {(1,1),(0,0)} | 完整|

 这表明一个锚点是不够的，因为它无法覆盖所有断开的区域。 

### 示例 2

 输入网格：```
0 1 0
0 1 0
0 0 0
```这里的垂直连接被一个个阻挡，因此覆盖范围是支离破碎的。 

| 步骤| 面膜| 选择S | 覆盖|
 | --- | --- | --- | --- |
 | 开始 | 000000000 | {} | {} |
 | 选择底行 | 000000111 | {(2,1)} | 底行|
 | 选择左栏| 001001001 | {(2,1),(1,0)} | 完整|

 这表明它们会破坏矩形并强制多个锚点。

 ## 边缘情况

 一种边缘情况是转换 2 秒后网格不包含零。 在这种情况下，没有元素可以覆盖，因此最优 S 为空。 该算法通过检查 k 是否为零并立即返回 0 来直接处理此问题。 

当零存在但完全被 1 分隔时，会出现另一种边缘情况，因此任何两个不同零之间的矩形都无效。 在这种情况下，每个零只能覆盖其自身，迫使 S 包含所有零单元。 DP 自然会达到此目的，因为每个 cover(s) 只包含 s 本身，因此达到完整遮罩的唯一方法是单独选择所有元素。
