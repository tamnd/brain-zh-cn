 ## 解决方案
 ## 问题理解

 我们得到了小行星的线性序列，每个小行星的颜色要么`a`或者`b`。 我们可以选择该序列的任何连续部分，将其删除，然后将其两端粘合在一起以形成一个循环。 在此操作之后，我们希望将生成的循环分成两个连续的一半，使得一半仅包含`a`另一半只包含`b`。 

目标是最大化所选段的长度。 同样，我们想要最长的子串可以变成“完美的二分循环”：连接其两端后，得到的循环串可以分成两个单色的半部分。 

循环条件很重要：一旦我们选择一个子串，它的结尾和开头之间的边界就会变得相邻，因此任何有效的最终结构都取决于该子串的旋转而不是其原始的线性形式。 

约束条件`n ≤ 2 · 10^5`排除子串的任何二次枚举。 尝试所有 O(n^2) 段或检查 O(n) 中的每个段的任何解决方案都将超出限制几个数量级。 

一个天真的陷阱是假设我们只需要相等的数量`a`和`b`。 这是必要的，但还不够。 例如，`abbaab`具有相等的计数，但没有旋转将其分成两个纯粹的两半。 

另一个微妙的问题是认为最好的段始终是整个字符串减去前缀或后缀。 存在最佳段严格位于内部的反例。 

## 方法

 暴力方法会枚举每个子字符串`[l, r]`，模拟使其成为圆形，并测试一些旋转是否将其分成均匀的两半。 对于每个子字符串，我们需要尝试所有旋转或至少验证分割点，成本为 O(length)。 在最坏的情况下，这会导致 O(n^3)，因为有 O(n^2) 个子字符串，并且每次检查的时间复杂度为 O(n)。 即使优化到 O(n^2)，对于 2·10^5 来说仍然太慢。 

关键的观察是，一旦我们固定了一个子串，“旋转后可以分成两个单色半部分”的条件就受到极大的限制。 旋转后，字符串变成两个连续的相同字符块。 这意味着圆弦必须恰好由两次运行组成，其中之一`a`和其中之一`b`，并且两次运行必须是循环中连续的间隔。 

这意味着在所选子串中，字符之间的所有转换必须集中在循环意义上的至多一个位置。 在线性视图中，这意味着如果我们将剪切最佳地放置在过渡边界处，则子串最多可以有两个块。 

因此，问题简化为找到最长的子字符串，该子字符串最多包含按线性顺序排列的两个相同字符，因为我们可以旋转它，以便运行之间的边界成为切口。 

然而，我们还必须确保两个字符都出现，否则除非我们允许零大小，否则不可能分成两个非空单色半部分。 

这自然会减少为运行或字符上的滑动窗口，从而维护一个最多具有两个连续块的窗口。 

我们扩展右指针，跟踪字符之间的转换，并在超过两个块时缩小。 最大有效窗口长度就是答案。

 ## 算法演练

 我们将字符串视为由相同字符组成。 我们维护一个滑动窗口`[l, r]`遍历字符串并跟踪其中存在多少个字符块。 

1.初始化两个指针`l = 0`，并跟踪窗口内的转换数量。 当发生转变时`s[i] != s[i-1]`。 我们维护当前窗口中的转换计数。 
2. 展开`r`从左到右。 每次我们延长`r`，我们检查是否`s[r] != s[r-1]`。 如果是这样，我们就增加转换计数。 这代表进入一个新的字符块。 
3. 如果过渡数量超过 1，则窗口现在至少包含 3 个块，这意味着它无法恰好旋转为两个单色半部分。 我们必须从左侧收缩，直到窗口再次有效。 
4.为了缩小，我们移动`l`当我们删除两个不同字符之间的边界时，向前和递减转换计数。 具体来说，当`s[l] != s[l+1]`，我们正在删除一个转换并减少计数。 
5. 恢复有效性后，将答案更新为`r - l + 1`。 

为什么这与问题相匹配：任何有效的圆形分割都对应于选择一个可以旋转的线段，以便在两者之间恰好有一个边界`a`和`b`是分割点。 仅当子串在最佳剪切放置之后以线性形式包含最多一次字符之间的交替（这对应于最多两次运行）时，这才是可能的。 

### 为什么它有效

 不变量是当前窗口始终最多包含一个在旋转后重要的转换点。 如果一个窗口有两个过渡，那么无论我们如何旋转它，圆圈中仍然至少有三个交替的块，使其不可能分成两个单色的一半。 相反，任何最多具有一个过渡的窗口都可以旋转，以便切口精确地放置在该过渡处，从而产生两个均匀的半部。 这种等价性确保每个最大有效窗口都对应于有效结构，因此滑动窗口最大值是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    l = 0
    transitions = 0
    ans = 0

    for r in range(n):
        if r > 0 and s[r] != s[r - 1]:
            transitions += 1

        while transitions > 1:
            if l + 1 <= r and s[l] != s[l + 1]:
                transitions -= 1
            l += 1

        ans = max(ans, r - l + 1)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案保留一个窗口并仅计算其中的字符更改。 微妙之处在于，仅对相邻字符之间的转换进行计数，并且仅当左指针经过此类边界时才将其删除。 这确保了正确性，而无需从头开始重新计算结构。

 ## 边缘情况

 用于输入`aaaaa`，算法永远不会看到转换，因此窗口从`0`到`n-1`。 答案就变成了`5`，符合整个段可以简单旋转的事实。 

用于输入`abab`，过渡累积得很快。 在`r = 2`，第三次块尝试迫使左指针前进，仅保留大小为 2 的有效窗口。该算法不允许多次交替，因此答案稳定在`2`，对应于旋转产生`aa|bb`或者`bb|aa`取决于切割。
