 ## 解决方案
 ## 问题理解

 我们有偶数个朋友，比如 2n 个，每个朋友都有一个代表披萨口味偏好的整数值。 该小组想要恰好订购 n 个披萨，并且每个披萨必须根据两个不同朋友的喜好制作。 每个偏好值必须仅使用一次，这意味着我们将所有 2n 个数字配对成 n 对。 

每一对形成一个披萨，披萨的成本定义为该对中两个值中的最大值。 目标是选择一种配对策略，使这些最大值的总和最小化。 

因此，问题简化为将 2n 个数字配对成 n 对，从而最小化成对最大值的总和。 

约束允许 n 最多为 100,000，这意味着最多 200,000 个数字。 任何比 O(n log n) 更差的解决方案都有可能太慢。 尝试所有对组合或在子集上使用动态规划的二次配对策略是立即不可能的，因为它将涉及大约 n² 或 2^(2n) 行为，两者都远远超出了可行的限制。 

当值高度不平衡时，就会出现微妙的边缘情况。 例如，如果大多数值都很小，而其中一个值非常大，则错误地附加大值的天真贪婪配对可能会夸大总成本。 考虑像 1 1 1 1 100 这样的输入。如果配对不好，100 可能最终会在间接贪婪方案中增加多对成本，但最佳结构将其精确地隔离在一对中。 

另一种边缘情况是值已经排序或几乎相同。 在这种情况下，不同的配对策略可能在局部看起来是等效的，但只有一种结构可以保证全局最优性。 

## 方法

 暴力破解的想法很简单：尝试所有可能的方法来配对 2n 个元素并计算每个配对的最大值之和。 这是正确的，因为它将数组的每个有效分区成对探索。 然而，配对的数量是 (2n - 1)!!，其增长速度比 n 中的指数增长得更快。 即使对于 n = 20，这也是不可行的，而当 n = 100,000 时，这是完全不可能的。 

为了改进，我们寻找对最大值如何对总数做出贡献的结构。 每对贡献其两个元素中较大的一个。 如果我们对数组进行排序，我们就可以控制哪些元素可能成为最大值。 关键的观察是，为了最大限度地减少浪费的大值，我们希望“仔细配对大元素，以便每个大元素只支付一次，并且不会与不必要的更大元素配对。”

 排序表明，最佳策略是按排序顺序将相邻元素配对。 这确保了在每一对中，较大的元素在剩余的选择中尽可能小。 任何跳过配对邻接的尝试都有可能将一个大元素与稍后更大的元素配对，从而增加总和。 

因此，问题简化为从末尾开始对每个第二个元素进行排序和求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O((2n−1)!!) | O((2n−1)!!) | O(n) | 太慢了 |
 | 最优（排序+贪婪配对）| O(n log n) | O(n log n) | O(1) 或 O(n) | 已接受 |

 ## 算法演练

 我们首先以非降序对 2n 个值的数组进行排序。 

1. 对所有值进行排序。 这创建了一个全局结构，其中局部邻接反映了值的接近程度，这对于最小化每对中的最大值至关重要。 
2. 从最大值向下迭代，每次步进两位。 换句话说，取索引 2n−1、2n−3、2n−5 等等。 
3. 对这些选定元素求和。 其中每一个都对应于按排序顺序由其前面的元素形成的对中的最大值。 
4. 输出总和。 

排序后隐式配对为：(a[0], a[1]), (a[2], a[3]), ..., (a[2n−2], a[2n−1])。 我们没有明确地构造对，因为只有最大值很重要。 

### 为什么它有效

排序后，考虑任何最佳配对。 专注于最大的元素。 在任何有效的解决方案中，它必须与其他一些元素配对。 最小化成本贡献的最佳合作伙伴是最大的剩余元素，该元素尽可能小，同时仍形成有效对。 将其与较小的元素配对并不会降低成本，因为最大值仍然是最大的元素本身。 然而，将大元素配对在一起可以避免浪费第二大元素作为单独的最大值，因为它可以与稍小的元素配对。 归纳地重复这个论点表明，按排序顺序配对连续元素会产生一种排列，其中每对的最大值恰好是选定的“倒数第二个元素”之一，并且没有重新排列可以进一步减少总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    a = list(map(int, input().split()))
    
    a.sort()
    
    ans = 0
    for i in range(2 * n - 1, -1, -2):
        ans += a[i]
    
    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先读取 n 和 2n 值。 排序至关重要，因为它可以对齐元素，从而使最佳配对成为局部决策而不是全局搜索问题。 

该循环以两步从数组末尾开始，直接累加每个最优对中充当最大值的元素。 我们从不明确地建立对，因为只有最大的对最终答案有贡献。 

一个常见的实现错误是错误地迭代索引，尤其是混合 n 和 2n 或在错误的边界处停止。 正确的范围恰好是 2n 个元素，并且步幅必须是 2。

 ## 边缘情况

 当所有值都相同时，排序不会产生有意义的结构变化。 该算法仍然对相邻元素进行配对，并对从末尾开始的每隔一个元素求和，这会产生完全相同的 n 倍的值。 对于输入 4 4 4 4 4 4，排序数组保持不变，并且当 n = 3 时，总和变为 12，匹配任何有效配对。 

当数组已经按升序排序时，例如 1 2 3 4，算法对 (1,2) 和 (3,4) 进行配对。 选择 2 和 4 作为最大值与从末尾开始逐步遍历每个第二个元素是一致的，并且没有替代配对可以减少总和，因为任何交叉配对（如 (1,4)）会强制 (2,3) 并增加总数。 

当值反向排序时，例如 4 3 2 1，排序首先对结构进行规范化。 应用相同的逻辑，并且排序后算法的行为相同，确保输入顺序永远不会影响正确性。
