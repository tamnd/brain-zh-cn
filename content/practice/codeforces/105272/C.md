 ## 解决方案
 ## 问题理解

 我们被赋予了一组$n$人们，在哪里$n$可以被 3 整除，并且这些人已经被分成每组恰好有 3 名成员的小组。 只有当三名成员中至少有两名被说服参加比赛时，每个团队才会同意参加比赛。 

教练可以直接说服个人。 一旦团队中至少有两名成员同意，第三名成员也会自动同意。 目标是尽量减少必须直接说服的人数，以便每个团队最终参与。 

经过重新设计，每三人一组的行为就像一个阈值小工具：说服一个人不足以激活团队，但说服任何两个人会迫使第三个人跟随。 任务是选择最少数量的初始“种子”，以便每个三元组至少有两个被选择的种子或通过传播而被迫。 

约束条件$n \le 100$意味着对子集的暴力破解是不可行的。 甚至$2^{100}$是完全不可行的。 任何解决方案都必须直接推理每个三元组的结构，而不是探索组合。 

当没有结构的贪婪思考时，就会出现一个微妙的陷阱。 例如，人们可能会尝试为每个团队挑选恰好两个人，从而产生$2n/3$，但跨团队效应是否可以进一步减少这一点尚不明显。 另一个天真的想法是，每个团队选择一个人可能会以某种方式级联，但由于团队是独立的，因此级联不会跨越边界。 

## 方法

 蛮力视角是考虑选择一部分人并模拟过程：对于每个团队，检查是否至少选择了两名成员，如果没有，则通过重复传播是否可以说服第三个成员。 这很快就会变得复杂，因为不存在跨团队的传播，并且在团队内规则崩溃为简单的阈值条件。 枚举子集给出了正确性，但有成本$O(2^n \cdot n)$，这远远超出了限制。 

关键的简化是认识到团队是完全独立的。 一个团队的决定不会影响另一个团队。 这意味着我们可以单独优化每组三组并对答案进行求和。 

在一个团队中，我们需要最少数量的最初被说服的成员，以便最终所有三个人都同意。 如果我们说服 0 或 1 名成员，团队永远不会达到两名被说服的成员的门槛，因此该过程无法完成。 如果我们说服 2 名成员，规则会立即激活，然后是第三名成员。 因此，每个团队的最佳成本恰好是 2。 

由于有$n/3$团队，总答案固定为$2 \cdot (n/3)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 每个团队的推理 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算团队数量为$n // 3$。 这是可行的，因为输入保证能被 3 整除，因此不需要余数处理。 
2. 对于每个团队，确定必须直接说服的最少人数，以确保团队参与。 仅当最初选择至少两名成员时，团队才会激活，因为一旦达到阈值，第三名成员就会自动跟随。 
3. 为每个团队分配 2 的成本，因为选择少于 2 个无法触发激活，选择 2 个就足够了。 
4. 将团队数量乘以2以获得最终答案。 

### 为什么它有效

 每个团队都独立运作，并有严格的激活门槛：三分之二的成员。 不存在跨团队交互，因此任何全局策略都会分解为相同的每个团队决策。 在单个三元组中，任何解决方案都必须选择至少两个成员； 否则永远不会满足激活规则。 恰好选择两个就足够了，因为它会立即触发完全接受。 这确定了每个团队的最低限度是固定的，并且在所有团队中都是累加的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    print(2 * (n // 3))

if __name__ == "__main__":
    solve()
```该解决方案读取单个整数并计算存在多少个不相交的三组。 每个组恰好贡献两个所需的直接选择，因此计算是直接乘法。 

该实现完全避免了循环，因为团队之间不存在交互。 唯一的微妙之处是确保使用整数除法。

 ## 边缘情况

 最小的有意义的情况是$n = 3$，其中恰好存在一个团队。 该算法分配成本 2，这符合必须说服至少两名成员才能激活唯一团队的要求。 

在$n = 0$，尽管在约束中没有明确说明，该公式仍然产生 0，与没有团队一致。 

在较大的值，例如$n = 99$，该算法通过计算 33 个团队并分配 66 个确信的成员来线性扩展。 每个团队保持独立，因此没有交互作用可以进一步减少这个数字，并且计算值仍然是最佳的。
