 ## 解决方案
 ## 问题理解

 我们正在研究一个在完全严格的世代中进化的种群。 第一代从一定数量的个体开始，称之为$a$。 任何生殖世代中的每个人都准确地生产$r$孩子，所以每一代都是通过前一代乘以相同的因子得到的$r$。 这个过程持续进行$m$世代繁殖，但在世代$m$，繁殖停止，这一代仍然存在，但不再产生更多的后代。 

因此人口规模呈几何级数：$$a, ar, ar^2, \dots, ar^m$$记录的总人口是所有这些世代的总和：$$n = a(1 + r + r^2 + \dots + r^m)$$我们只被给予$n$，我们必须数出有多少个三元组$(a, r, m)$和$a \ge 1$,$r > 1$,$m \ge 0$可以精确地产生这个总数。 

关键的困难在于不同的几何分解可以产生相同的总和，因此我们不是求解唯一的因式分解，而是计算所有有效的参数化。 

约束条件$n \le 10^9$强烈表明我们可以枚举结构的至少一维或二维，但不能独立地暴力破解所有三元组。 任何尝试所有的解决方案$a, r, m$会爆炸，因为即使$r^m$生长很快，但仍然留下许多组合。 

出现微妙的边缘情况时$m = 0$。 在这种情况下，只有一代，所以总和就是$n = a$。 这意味着每一个$n$总是贡献至少一种有效配置：单个非繁殖代。 

另一个边缘情况是当$r$足够大，几何和几乎不超过两项或三项。 许多幼稚的方法通过对待不同的$(a, r)$对作为独立的，而不检查剩余的总和是否仍然可以由整数约束完全解释。 

## 方法

 一个蛮力的想法是迭代所有可能的$r$和$m$，计算几何和因子$$S(r, m) = 1 + r + r^2 + \dots + r^m$$然后检查是否$a = n / S(r, m)$是一个整数。 如果是的话，我们就算一下。 

这是正确的，但很快就变得不可行。 虽然$m$隐含地受到以下事实的限制：$r^m \le n$，对数$(r, m)$还是很大的。 对于每个$r$，最大$m$大致是$O(\log_r n)$，并对所有内容求和$r$给出的复杂度接近于$O(n)$最坏情况下的除数结构和指数增长。 
关键的观察是结构完全由两个约束决定：总和是几何的，并且$a$只是一个比例因子。 所以不要枚举$a$，我们固定几何和$S$，并要求$S$划分$n$。 一旦我们修好$r$和$m$,$S$是唯一确定的，并且每个有效配置对应于选择有效的几何和因子$n$。 

所以问题简化为枚举所有对$(r, m)$这样：$$S(r, m) \mid n
\quad \text{and} \quad r > 1$$对于每个有效的$S$，我们正好得到一个$a = n / S$。 

计算结构变得易于管理，因为$r^m$呈指数增长，因此对于每个$r$，我们只需要模拟几何和，直到超过$n$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 (a, r, m) |$O(n \log n)$或者更糟|$O(1)$| 太慢了 |
 | 修正 r 并构建几何和 |$O(\sqrt{n})$到$O(n^{1/2})$有效|$O(1)$| 已接受 |

 ## 算法演练

 我们专注于枚举所有可能的几何和并检查它们是否整除$n$。 

1. 迭代所有可能的值$r$从2开始向上。 我们只需要去$r \le n$，但实际上增长使得每个链的停止更早。 
2.对于固定的$r$，增量计算几何级数和。 开始于$sum = 1$，对应于$m = 0$。 
3. 将运行功率项重复乘以$r$，在每一步更新总和。 每次相加后，检查总和是否超过$n$。 如果确实如此，请停止，因为进一步的条款只会增加它。 
4. 对于每个计算的总和$S$，检查是否$n \bmod S = 0$。 如果是这样，则它代表有效的分解并贡献一种配置。 
5. 始终包括琐碎的案例$m = 0$， 在哪里$S = 1$，对应于$a = n$。 
6. 累积所有有效案例$r$。 

本质思想是每个有效结构对应一个唯一的几何前缀和$S(r, m)$，一旦该总和固定，初始种群就被强制。 

### 为什么它有效

 每个有效的人口历史完全通过选择繁殖因子来确定$r$, 深度$m$，然后按比例缩放$a$。 总人口永远是$a \cdot S(r, m)$。 相反，任意一对$(r, m)$定义固定总和$S$，如果$S$划分$n$，我们可以重建一个有效的$a$。 枚举结束$r$不断增长的幂确保我们只生成一次所有可能的几何和，因为固定的部分和序列$r$是严格递增且唯一确定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    # m = 0 case: sum = 1 always
    # contributes exactly one configuration for any n
    ans = 1
    
    # try all r > 1
    # geometric sum S = 1 + r + r^2 + ...
    for r in range(2, n + 1):
        s = 1
        term = 1
        
        while True:
            term *= r
            s += term
            if s > n:
                break
            if n % s == 0:
                ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该代码通过用 1 初始化答案来立即分离简单的单生成案例。这对应于$m = 0$，没有繁殖发生，人口只是$a = n$。 

对于每个候选者的再现率$r$，我们建立的力量$r$逐渐地。 变量`term`曲目$r^k$， 和`s`累加几何和。 一次`s`超过$n$，我们停止，因为任何进一步的条款只会进一步增加它。 

每次`s`划分$n$，我们计算一个有效的配置，因为它意味着一个有效的整数$a = n / s$。 

一个微妙的点是我们没有明确跟踪$m$，因为它是根据我们更新的次数隐式编码的`term`。

 ## 边缘情况

 ### 情况 n = 1

 唯一可能的配置是单个非繁殖代。 算法初始化`ans = 1`，因此它正确返回 1，而无需输入任何有意义的循环迭代。 

### n 附近较大的 r

 当$r = n$，几何和变为$1 + n$，立即超过$n$。 循环在一次迭代后停止，确保我们不会浪费时间探索无效的深度链。 

###素数n

 对于总理$n$，仅几何和等于 1 或恰好等于 1 的配置$n$是可能的。 该算法检测到这一点是因为只有除数检查在琐碎或全和级别上成功，符合不存在中间因式分解的数学限制。
