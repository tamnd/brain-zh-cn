 ## 解决方案
 ## 问题理解

 我们正在观察一个节日，每个参与者都只买了一张门票。 有两种门票类型： 普通门票`v`还有一张打折的月球票，价格正好是它的一半，`v/2`， 在哪里`v`保证是均匀的。 

我们得到了参加者的总数`p`, 收取的总收入`a`，以及全票价格`v`。 一些未知数量的参与者是月球居民，他们总是支付折扣价，而所有其他参与者则支付全价。 任务是确定有多少`p`与会者购买了折扣票。 

约束很大，但结构极其简单：所有值最多为一百万，只有算术关系很重要。 这立即排除了对个人的任何模拟或以一种依赖于迭代的天真的方式搜索可能的计数`p`或者`a`。 任何正确的解决方案都必须将问题简化为从方程导出的常数时间计算。 

如果一个人试图贪婪地推理而不形成方程式，就会出现一种微妙的失败情况。 例如，假设最大化折扣与会者始终有效，可能会破坏与总收入约束的一致性。 另一个常见的错误是忘记折扣正好是一半`v`，不是任意分数，这对于形成清晰的线性关系至关重要。 

## 方法

 暴力解释会尝试所有可能的数字`x`月球参与者来自`0`到`p`，计算所得收入，并检查是否匹配`a`。 这是有效的，因为每个候选人完全决定收入，但它的成本`O(p)`每个测试用例。 和`p`达到一百万，这在孤立的情况下仍然是可行的，但由于关系是线性的并且可以直接求解，因此变得不必要并且在概念上效率低下。 

关键的观察是总收入可以表示为未知变量的线性函数`x`。 如果`x`参加者半价并`p - x`支付全价，那么收入完全由下式决定`x`。 这将问题转化为求解一个变量的单个线性方程。 一旦重新排列该方程，无需任何迭代即可直接获得答案。 

问题的结构保证结果值是整数且非负，因此不需要额外的搜索或舍入逻辑。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(p) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 首先假设`x`人们是支付费用的月球参与者`v/2`，而剩余的`p - x`支付`v`。 

这直接编码了系统中唯一的未知数。 
2. 将总收入表示为捐款总​​和：`a = x * (v/2) + (p - x) * v`。 

此步骤将故事转换为确定性方程。 
3. 展开表达式：`a = x*v/2 + p*v - x*v`。 
4. 涉及的组术语`x`:`a = p*v - x*(v/2)`。 
5. 重新排列以隔离`x`:`x*(v/2) = p*v - a`。 
6. 求解`x`:`x = (2 * (p*v - a)) / v`。 

乘以 2 避免了小数除法，因为`v`是均匀的。 
7. 返回`x`作为月球参与者的数量。 

### 为什么它有效

 收入函数呈线性关系`x`，这意味着每增加一名登月参与者，就会用半价票取代全价票，收入就会减少`v/2`。 由于没有折扣的基准收入是`p*v`， 区别`p*v - a`衡量因参加者折扣而造成的总减少量。 将减少总量除以`v/2`准确地给出了发生了多少次这样的减少，这唯一对应于`x`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, p, v = map(int, input().split())
    # derived formula: x = 2*(p*v - a) / v
    x = (2 * (p * v - a)) // v
    print(x)

if __name__ == "__main__":
    solve()
```该实现直接对导出的公式进行编码。 乘法`p * v`首先完成，它安全地符合 Python 的整数范围。 最后使用整数除法，因为该问题保证结果是整数。 

一个常见的实现错误是以引入浮点除法的方式重新排列公式，这是不必要的，并且可能会在其他语言中导致精度问题。 另一个错误是忘记在除之前先乘以 2`v`，这会失去正确性，因为折扣恰好是全价的一半。

 ## 边缘情况

 当所有参加者支付全价时，收入等于`p * v`。 代入公式得`x = 0`，这正确反映了没有使用折扣票。 

当所有参与者都是月球居民时，收入变为`p * v / 2`。 将其代入方程得出`x = p`，符合每位与会者使用折扣的预期。 

对于最大输入值，中间产品`p * v`可以达到`10^12`，但 Python 可以安全地处理这个问题，并且最终的除法仍然会产生一个有效的整数。
