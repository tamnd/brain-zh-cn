 ## 解决方案
 ## 问题理解

 我们得到一个字符串，它是通过交替在右侧附加字符和在左侧插入字符来重复构建另一个隐藏字符串的最终结果。 该过程从第一个字符开始，然后向外构建，将第二个字符放在当前字符串的右侧，将第三个字符放在当前字符串的左侧，将第四个字符放在右侧，依此类推。 

任务是扭转这个过程。 我们得到的是最终的打乱结果，而不是模拟构造，并且必须在发生任何左右插入之前恢复原始字符序列。 

关键的观察是，虽然在加密过程中在两端插入了字符，但操作顺序是确定的：右、左、右、左、交替。 这意味着最终的字符串不是任意的； 它由放置在相对于移动中心的已知位置的字符组成。 

长度最多为 50 的约束改变了一切。 任何模拟动态结构插入的解决方案都足够快，但我们也可以通过反转 O(n) 的构造顺序来直接重建。 

当长度为奇数与偶数时，会出现微妙的边缘情况。 例如，对于单个字符，没有任何变化。 对于两个字符，第二个字符始终附加在右侧。 对于三个字符，第三个字符位于左侧，因此中间结构发生了变化。 总是从一个固定方向交替而不跟踪步骤奇偶性的天真尝试将无法正确重建。 

另一个常见的陷阱是试图从左到右重建字符串，而没有意识到“中心”不断隐式移动。 正确的重建取决于知道前向过程中每个步骤是插入到左侧还是右侧，这纯粹由索引奇偶性决定。 

## 方法

 暴力解释将模拟所有可能的原始字符串并测试它们的加密是否与给定的字符串匹配。 即使长度为 50，这也是不可行的，因为候选字符串的数量为 26^50，并且每次模拟的成本为 O(n)，导致搜索空间巨大。 

关键的见解是，如果我们考虑如何添加字符，加密是逐步可逆的。 每一步都会在当前字符串的左端或右端插入一个新字符。 这意味着相反，我们可以以确定性模式从任一端剥离字符。 

如果我们向前模拟构造过程，我们将通过在末尾插入来重复修改字符串，这表明是双端队列结构。 但相反，我们已经知道最终的字符串，并且可以通过按相反顺序撤消操作来重建原始字符串：最后一个操作由步骤索引的奇偶校验确定。 

我们通过认识到最后插入的字符取决于 n 是奇数还是偶数来对过程进行逆向工程，并且在重建原始字符串时迭代地从相应末尾删除字符。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(26^n·n) | O(26^n·n) | O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们通过模拟最终字符串的构建方式来重建原始字符串，但顺序相反。

1. 确定给定字符串的长度n。 这定义了加密期间发生的插入步骤数。 
2. 维护代表不断演变的重构结果的结构。 我们将使用双端队列，因为我们需要从两端高效删除。 
3、从最后的字符串t开始作为所有操作后的当前状态。 
4. 我们模拟从步骤 n 到步骤 2 的撤消操作。在每一步 i，我们决定第 i 个字符（在正向构造中）是否放置在左侧或右侧。 这取决于奇偶校验：奇数索引步骤插入右侧，偶数索引步骤插入左侧。 
5. 相反，如果步骤 i 是正确插入，我们删除最后一个字符。 如果是左插入，我们删除第一个字符。 删除的字符属于重构后的原始字符串中的位置 i。 
6. 我们按照重构的相反顺序收集删除的字符，最后组合成原始字符串。 

### 为什么它有效

 每个加密步骤都在当前字符串的已知末尾添加一个字符。 这意味着最终字符串包含原始字符串的所有字符，但它们的位置是一系列确定性末端插入的结果。 由于每项操作仅影响一端，因此逆转该过程永远不需要猜测或回溯。 奇偶校验规则唯一地确定每一步使用哪一端，因此每次反向删除都恰好对应于一个原始字符位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = input().strip()
    n = len(t)

    # simulate reverse construction
    # we track current string as deque via pointers
    l, r = 0, n - 1

    res = []

    # we undo steps from n down to 1
    for i in range(n, 0, -1):
        if i == 1:
            # last remaining character is the first original char
            res.append(t[l])
        else:
            if i % 2 == 0:
                # even step: was left insertion -> undo from left
                res.append(t[l])
                l += 1
            else:
                # odd step: was right insertion -> undo from right
                res.append(t[r])
                r -= 1

    print("".join(res[::-1]))

if __name__ == "__main__":
    solve()
```该解决方案将两个指针保留在最终字符串上，并根据相应的向前步骤是向左插入还是向右插入来删除字符。 关键的实现细节是我们以相反的顺序重建原始字符串，因此我们在末尾反转了收集到的字符。 

i == 1 处的边界条件很重要，因为在所有删除之后，只剩下一个字符，该字符必须是原始字符串的第一个字符。

 ## 边缘情况

 用于输入`a`，算法集`l = r = 0`，并直接附加`t[l]`，生产`a`。 不会发生任何删除，这与加密后单个字符串不会发生变化的事实相匹配。 

用于输入`ab`，相反的过程首先从右删除（因为步骤 2 是奇数），然后从左删除。 重建产生正确的顺序`ab`反转收集的序列后。 

对于交替的小输入，例如`bac`，指针移动确保左右删除正确地模拟插入的反转，从而确认基于奇偶校验的末端选择完全确定了原始排序而没有歧义。
