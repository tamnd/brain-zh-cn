 ## 解决方案
 ## 问题理解

 我们在无限网格上得到了三个不同的单元格，每个单元格代表一块方形土地。 最初一切都被阻止，我们可以“清除”我们想要的任何单元格。 目标是清除一组单元格，以便通过在清除的单元格上进行 4 向移动来连接三个给定的单元格。 连接性意味着从三个特殊单元中的任何一个开始，我们可以仅使用相邻的已清除单元逐步走到其他两个。 

任务不仅仅是决定这是否可能，而是构建一个具有最小可能大小的已清除单元的配置。 换句话说，我们正在寻找包含所有三个给定点的最小连通网格单元集。 

约束很小：坐标最多为 1000，而且只有三个点。 这排除了诸如大型状态上的动态规划或整个网格上的图最短路径之类的问题，因为答案是几何构造而不是搜索问题。 该结构足够严格，因此解决方案必须来自观察最短曼哈顿连接的行为方式。 

一种简单的方法可能会尝试从每个点运行 BFS 并以某种方式合并最短路径，甚至枚举候选连接结构。 这将隐含地探索许多网格单元，并很快变得不必要的杀伤力。 关键的困难在于两点之间存在多条最短路径，我们必须选择它们，以便所有三个点共享一个连接结构，而不浪费额外的单元。 

天真的直觉的一个微妙的失败案例是假设用最短路径独立连接对总是有效的。 例如，使用曼哈顿最短路径连接 A 到 B 和 B 到 C 可能会产生多余的弯路或无法最大限度地减少重叠，特别是当“中间”点没有选择好时。 正确的解决方案必须强制三个连接路径之间的最大重叠。 

## 方法

 暴力思维是考虑包含这三个点的所有可能的连接形状并计算它们并集的大小。 但即使将我们限制在对之间的路径上，两点之间可能的最短曼哈顿路径的数量也会随着它们的距离而组合增长。 即使对于中等距离，探索三对连接的所有组合也是不可行的，因为在水平和垂直排序的选择方面，每对单独可以具有指数级的最短路径。 

关键的结构见解是，在曼哈顿几何中，多个点之间的最佳连接就像斯坦纳树一样，其分支发生在非常特定的位置。 对于三个航站楼，始终存在一种最佳配置，其中所有路径在单个网格对齐的交汇处相遇。 一旦接受这一点，问题就减少到寻找正确的会面点。 

关键的观察结果是，对于轴对齐距离，最佳“连接点”最小化到 x 和 y 维度上独立的三个点的距离总和。 这是通过选择三个点中的中值 x 坐标和中值 y 坐标来实现的。 一旦这个中心被固定，最好的策略是使用单调的水平和垂直线段将每个点连接到这个中心。 这保证了最小的总联合大小，因为每个段都是曼哈顿距离中的最短路径，并且通过共享中心自动最大化重叠。

 | ---| ---| ---| ---|

 ## 算法演练

1. 提取三个给定点的 x 和 y 坐标。 
2. 使用 x 坐标的中值和 y 坐标的中值计算候选交汇点。 此选择是最优的，因为它最大限度地减少了沿每个轴独立地到所有三个点的总曼哈顿距离总和。 
3. 对于这三个点中的每一个，首先水平移动直到 x 匹配，然后垂直移动直到 y 匹配，构建一条到交汇点的路径。 顺序并不重要，但固定一个顺序可以使构造简单且具有确定性。 
4. 标记一组中这三个路径访问过的每个单元格以避免重复。 这确保我们输出所有所需清除图的并集而不会重复。 
5. 输出集合的大小，后跟任意顺序的所有坐标。 

### 为什么它有效

 正确性取决于这样一个事实：连接曼哈顿几何中三个点的任何最佳解决方案都可以转换为所有路径在单个网格点处相交的解决方案，而无需增加所使用的单元数量。 一旦存在这样的交汇点，最小化总路径长度的最佳交汇点正是坐标方向的中值。 使用轴对齐最短路径的构造保证了我们永远不会引入不必要的弯路，并且路径之间的任何重叠只会减少唯一单元的总数，这有利于最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def path(x1, y1, x2, y2):
    cells = []
    x, y = x1, y1

    dx = 1 if x2 > x else -1
    while x != x2:
        cells.append((x, y))
        x += dx
    cells.append((x, y))

    dy = 1 if y2 > y else -1
    while y != y2:
        if (x, y) not in cells:
            cells.append((x, y))
        y += dy
    cells.append((x, y))

    return cells

def solve():
    pts = [tuple(map(int, input().split())) for _ in range(3)]

    xs = sorted(p[0] for p in pts)
    ys = sorted(p[1] for p in pts)

    cx = xs[1]
    cy = ys[1]

    res = set()

    for x, y in pts:
        for cell in path(x, y, cx, cy):
            res.add(cell)

    print(len(res))
    for x, y in res:
        print(x, y)

if __name__ == "__main__":
    solve()
```该解决方案首先读取三个点并计算两个维度的中值坐标。 该中线充当共享连接点。 然后使用简单的 L 形曼哈顿路径将每个点连接到该交叉点。 

辅助函数构建单调路径，而无需重新访问不必要的单元格。 该集合确保对于最优性至关重要的重叠段不会增加计数。 这很重要，因为最佳结构严重依赖于共享段，并且必须删除重复项。 

一个微妙的实现细节是包含端点的处理。 每条路径必须包括两个端点，以便最终的并集正确形成包含所有三个原始点的连接结构。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    pts = [tuple(map(int, sys.stdin.readline().split())) for _ in range(3)]

    xs = sorted(p[0] for p in pts)
    ys = sorted(p[1] for p in pts)
    cx, cy = xs[1], ys[1]

    res = set()

    def path(x1, y1, x2, y2):
        x, y = x1, y1
        res_local = []
        dx = 1 if x2 > x else -1
        while x != x2:
            res_local.append((x, y))
            x += dx
        res_local.append((x, y))
        dy = 1 if y2 > y else -1
        while y != y2:
            res_local.append((x, y))
            y += dy
        return res_local

    for x, y in pts:
        for cell in path(x, y, cx, cy):
            res.add(cell)

    out = [str(len(res))]
    for x, y in res:
        out.append(f"{x} {y}")

 ## 边缘情况

 一个退化但有启发性的情况是所有三个点都位于垂直直线上，例如 (0,0)、(0,1)、(0,2)。 中值构造选择 (0,1)，每个点直接沿同一列连接。 联合体正是这三个细胞，没有出现额外的分支。 该算法自然地消除了冗余结构，因为所有路径沿垂直线段完全重叠。 

另一种情况是点形成直角图案，如 (0,0)、(0,2)、(2,1)。 中位数变为 (0,1)。 一条路径是琐碎的，一条是单步垂直的，一条形成水平然后垂直的段。 追踪它可以确认所有路径都在选定的中心相交，并且没有任何替代交汇点可以在不增加距至少一个点的距离的情况下进一步减少并集。
