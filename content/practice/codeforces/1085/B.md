 ## 解决方案
 ## 问题理解

 我们得到两个整数，一个由乘法表达式构建的目标值，以及一个固定的基本参数，该参数控制我们如何将任何正整数拆分为商和余数。 

对于任意正整数$x$，我们将其拆分为$k$分成两部分：除法的整数部分，计算有多少个完整的大小块$k$适合$x$，以及删除这些块后剩下的部分。 然后我们将这两个值相乘并要求结果等于$n$。 任务是找到最小的正整数$x$则满足这个条件。 

表达式的结构表明$x$不是任意的，而是由其商和余数如何相互作用决定。 自从$k \le 1000$， 每个$x$可以分解为一个小商和一个有界余数，这强烈建议迭代这些组件之一而不是$x$本身。 

范围为$n$上升到$10^6$，因此迭代所有可能的解决方案$x$达到很大的值会太慢。 直接搜索过去$x$最多$10^6 \cdot k$已经是边界了，任何二次或嵌套的东西$x$显然是不可能的。 

当考虑商和余数如何相互作用时，就会出现一个微妙的问题。 许多不正确的方法试图修复$x \bmod k$并计算$x \div k$独立地，但忘记改变一个会通过定义影响另一个$x = (x \div k)\cdot k + (x \bmod k)$。 将它们视为自变量会导致构造无效$x$。 

## 方法

 暴力方法尝试每个正整数$x$，计算两者$x \div k$和$x \bmod k$，将它们相乘，并检查结果是否等于$n$。 这是正确的，因为它直接遵循问题的定义。 然而，在最坏的情况下，我们可能需要检查所有值$x$直到第一个有效的解决方案，并且由于有效的解决方案可能相距很远，所以这很快就变得不可行。 甚至检查到$10^6 \cdot k$对于紧迫的时间限制来说，行动已经太大了。 

关键的观察是停止思考$x$而是根据它的商来思考。 让$q = x \div k$和$r = x \bmod k$。 然后$x = qk + r$和$0 \le r < k$，条件就变成了简单的乘积约束$q \cdot r = n$。 这将问题从搜索整数转变为搜索因子对$n$。 

一旦我们确定了除数$q$的$n$，相应的余数必须是$r = n / q$。 为了使该对有效，余数必须满足$r < k$，因为它来自模数$k$。 每个有效对直接重建一个候选$x = qk + r$。 在所有这些候选者中，我们选择最小的。 

这种减少是强大的，因为$n$足够小，迭代其所有除数的速度很快，并且约束$k \le 1000$使每个候选者的验证变得微不足道。

 ## 算法演练

 1. 迭代所有整数$q$从 1 到$\sqrt{n}$。 这是有效的，因为每个除数都大于$\sqrt{n}$有一个匹配的较小的对应物。 
2. 对于每个$q$，检查是否可除$n$。 如果不是，则跳过它，因为它无法形成有效的产品对。 
3. 计算配对除数$r = n / q$。 在此刻，$(q, r)$是一个候选分解$n$。 
4. 对于每个有效对，考虑两种解释：$q$作为商和$r$作为剩余候选者，反之亦然。 这是必要的，因为因子对的任一侧都可以起到商的作用。 
5. 对于每个解释，验证余数候选是否严格小于$k$。 这确保它可以合法地成为模值。 
6. 如果有效，则重建$x = qk + r$（或者$x = rk + q$）并跟踪所有有效结构的最小值。 

核心思想是每个有效的解决方案都唯一对应于一个因子对$n$，每个因子对最多生成两个候选值$x$。 

### 为什么它有效

 每个整数$x$唯一定义一对$(q, r)$在哪里$x = qk + r$和$0 \le r < k$。 问题中的条件变为$q \cdot r = n$，所以每个有效的解决方案都对应于因式分解$n$。 相反，每个尊重余数约束的因子对都会重建一个有效的$x$。 因为我们枚举了所有因子对，所以我们不会错过任何解决方案，并且因为我们重建了$x$直接根据除法和取模的定义，我们永远不会产生无效值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    ans = None

    q = 1
    while q * q <= n:
        if n % q == 0:
            r = n // q

            # case 1: q is quotient, r is remainder
            if r < k:
                x = q * k + r
                if ans is None or x < ans:
                    ans = x

            # case 2: r is quotient, q is remainder
            if q < k:
                x = r * k + q
                if ans is None or x < ans:
                    ans = x

        q += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该代码直接实现了除数对逻辑。 循环结束$q$枚举候选因子，每次找到有效的除数对时，我们立即测试商和余数角色的可能分配。 重建$x = qk + r$直接遵循整数除法的定义，因此无需额外模拟$x$是必须的。 

在检查两个方向时必须小心，因为未能交换角色会错过有效配置，其中较小的除数充当余数。

 ## 边缘情况

 出现关键边缘情况时$n = 1$。 唯一的因子对是 (1,1)，正确性取决于正确解释两个商余数方向。 为了$n=1, k=2$，我们得到$x = 2 \cdot 1 + 1 = 3$，满足$3 \div 2 = 1$和$3 \bmod 2 = 1$，给出产品 1。 

另一种微妙的情况发生在$k$相对于$n$。 例如，$n=2, k=1000$。 唯一有效的构造是将 2 视为商，将 1 视为余数，产生$x = 2k + 1$。 该算法可以处理这个问题，因为它显式地检查每个因子对的两个方向，确保即使一个分量远小于因子对，也不会跳过任何有效配置。$k$。
