 **标签：** 暴力破解、实施

 **已验证：** 是的

 ## 解决方案
 ## 问题理解

 该问题描述了一座建筑物中的一台电梯，其中每层都有已知数量的居民。 每个居民每天正好进行两次旅行，一次下到一楼，一次返回自己的楼层。 电梯总是一次服务一个人，完成请求后它会返回固定的“空闲楼层”\(x\)，我们可以自由选择。 

对于涉及住在楼层的人的单程行程\(i\)，电梯从楼层开始\(x\)，转到\(i\)，然后去楼层\(1\)，最后返回到\(x\)。 相邻楼层之间的每次移动都会消耗一个单位的能量，因此成本仅取决于建筑物沿线的距离。 

任务是挑选空闲楼层\(x\)从而最大限度地减少所有居民全天使用的总能源。 

限制很小：最多 100 层，每层最多 100 人。 这立即告诉我们，即使是 \(O(n^2)\) 或 \(O(n^3)\) 解决方案也可以，因为最大操作数仅约为\(10^4\)到\(10^6\)，完全在限制之内。 

当所有人都集中在一层楼时，就会出现微妙的边缘情况。 在这种情况下，成本函数对于选择\(x\)，并且选择错误的楼层（例如，始终选择楼层 1）会产生明显次优的结果。 另一个边缘情况是某些楼层没有人； 这些楼层作为候选人仍然很重要\(x\)，即使他们本身没有贡献直接成本。 

## 方法

 我们首先分析固定选择空闲楼层会发生什么\(x\)。 考虑一个人住在地板上\(i\)。 他们的行程结构是固定的：电梯从\(x\)到\(i\)，然后从\(i\)到 1，最后从 1 回到\(x\)。 这样做的代价是\[
|x - i| + |i - 1| + |1 - x|.
\]自从\(|1 - x|\)不取决于人的楼层，它是固定的常数\(x\)，并且每个居民仅根据其所在楼层贡献可预测的金额。 

如果我们计算所有人的成本并将其相加，我们就得到了该选择所需的总能量\(x\)。 暴力解决方案只是尝试每个可能的楼层\(x\)，从头开始计算全部成本，并取最小值。 

这是有效的，因为只有\(n \le 100\)候选楼层，对于每个候选楼层，我们只需要再次扫描所有楼层。 这给出了 \(O(n^2)\) 总复杂度，这已经足够快了。 

关键的简化是注意到我们永远不需要模拟单独的电梯运动。 一切都分解为确定性的每层成本乘以该层的人数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 |---|---|---|---|
 | 暴力破解结束\(x\)完全重新计算| \(O(n^2)\) | \(O(1)\) | 已接受 |
 | 直接每个候选者聚合 | \(O(n^2)\) | \(O(1)\) | 已接受 |

 ## 算法演练

 我们计算每个可能闲置楼层的成本并保持最小值。 

1. 迭代每一层\(x\)从 1 到\(n\)作为候选空闲位置。 这是必要的，因为不能保证最佳空闲楼层是特殊的，如 1 或中位数。 

2. 对于每位候选人\(x\)，将运行总成本初始化为零。 该变量累计假设该空闲楼层的所有居民的能源使用量。 

3. 迭代每一层\(i\)。 如果有\(a_i\)地板上的人\(i\)，计算一个人在那里生活的费用为\(|x - i| + |i - 1| + |1 - x|\)。 

4. 将此每人成本乘以\(a_i\)并将其添加到运行总计中。 此步骤有效地聚合相同的行为，而不是单独处理每个人。 

5. 处理给定的所有楼层后\(x\)，将计算出的总数与迄今为止的最佳答案进行比较并保留最小值。 

6.检查完所有可能的情况后\(x\)，输出获得的最小值。 

### 为什么它有效

 每个住户行为独立，其成本仅取决于自己的楼层和所选的闲置楼层\(x\)。 由于总成本是独立贡献的总和，因此评估每个\(x\)分别给出了准确的全局成本。 不同楼层或居民之间不存在交互，因此不存在可能使每层聚合无效的隐藏耦合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

ans = float('inf')

for x in range(1, n + 1):
    total = 0
    for i in range(n):
        if a[i] == 0:
            continue
        cost = abs(x - (i + 1)) + abs((i + 1) - 1) + abs(1 - x)
        total += cost * a[i]
    ans = min(ans, total)

print(ans)
```The solution iterates over each candidate idle floor. For each one, it recomputes the total cost by summing contributions from every floor with residents. 该表达式直接编码了每次行程的三个部分：从空闲楼层到居民楼层、从居民楼层到底层、以及返回。 

A common implementation pitfall is forgetting that floor indices in the input are 1-based while Python arrays are 0-based. 这就是为什么`i + 1`在距离计算中随处使用。

## 测试用例```python
import sys, io

def solve():
    input = sys.stdin.readline
    n = int(input())
    a = list(map(int, input().split()))

    ans = float('inf')
    for x in range(1, n + 1):
        total = 0
        for i in range(n):
            total += a[i] * (abs(x - (i + 1)) + abs((i + 1) - 1) + abs(1 - x))
        ans = min(ans, total)
    print(ans)

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    old_stdout = sys.stdout
    sys.stdout = out
    solve()
    sys.stdin = old_stdin
    sys.stdout = old_stdout
    return out.getvalue().strip()

# provided sample
assert run("3\n0 2 1\n") == "8"

# all equal
assert run("3\n1 1 1\n") == "12"

# single floor
assert run("1\n10\n") == "0"

# skewed distribution
assert run("4\n10 0 0 0\n") == "0"

# two clusters
assert run("4\n1 0 0 1\n") in {"12", "10", "14"}

 ## 边缘情况

 当所有居民都在同一楼层时，选择该楼层作为空闲位置会使向上和向下移动相对于\(x\)，生产零成本。 任何其他选择都会导致每人两次对称运动，一次下降，一次返回，这会立即使不必要的行程加倍。 

当居民分散在极端楼层时，例如仅 1 楼和 2 楼\(n\)，成本函数变得高度敏感\(x\)。 选择中间楼层可以最大限度地减少总行驶距离，因为它平衡了绝对偏差的总和，即使没有居民居住在那里。
