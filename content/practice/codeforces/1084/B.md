[CF 1084B - 克瓦斯和公平坚果](https://codeforces.com/problemset/problem/1084/B)

 ## 解决方案
 ## 问题理解

 我们得到了几桶克瓦斯，每桶都含有一些初始量。 在一次操作中，我们可以将任何单个小桶减少一升，并且每次这样的操作都会为我们想要“倒出”的总量贡献一升。 目标是准确删除$s$所有小桶的总升数。 这样做之后，剩余的克瓦斯水平很重要：我们关心的是最大化所有小桶中的最小值。 

所以任务不仅仅是检查我们是否可以删除$s$升，但要以一种避免使任何单个桶太小的方式分配这些清除量。 

关键约束是$n \le 10^3$和$s \le 10^{12}$。 小的$n$表明我们可以负担二次或$n \log n$行为，但巨大$s$立即排除任何试图一次减少一升的模拟。 任何解决方案都必须进行聚合推理，而不是逐步操作。 

当总克瓦斯小于时，会出现微妙的边缘情况$s$。 在这种情况下，即使我们清空所有桶，清除的分配也无法达到目标。 另一个棘手的情况是，贪婪地减少本地最大的桶似乎是正确的，但实际上错误地分配了移除量，导致最小值的降低超过了必要的程度。 

## 方法

 蛮力的想法是模拟一升一升地移除。 在每一步中，我们都会选择一个具有当前最大值的小桶，并将其减一。 这使得本地的最小值尽可能大，因为我们避免接触小桶。 后$s$这样的操作，我们计算最小值。 

这是正确的，因为每个操作都会为下一步保留最好的状态。 然而，复杂度是$O(s \log n)$如果我们维护一个堆，并且$s$可以达到$10^{12}$，这使得这种方法完全不可行。 

关键的观察结果是最终答案是一个阈值问题。 我们不模拟操作，而是假设候选最小值$x$。 如果我们强迫每个小桶至少$x$，然后上面任何多余的部分$x$可以自由移除。 固定的可移除总金额$x$变成$$\sum \max(0, v_i - x).$$如果这个总和至少是$s$，那么至少可以达到最小值$x$。 否则是不可能的。 

这将问题转化为寻找最大的$x$使得条件成立。 由于可行性是单调的$x$，我们可以二分查找答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 贪心模拟|$O(s \log n)$|$O(n)$| 太慢了 |
 | 对答案进行二分搜索 |$O(n \log V)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 计算所有小桶的总和。 如果小于$s$，立即返回-1。 没有足够的克瓦斯来去除$s$无论策略如何升。 
2. 设置答案的二分查找范围。 最终可能的最小值为 0，最大值为$\max(v_i)$，因为没有桶可以超过其初始值。 
3. 对于候选值$x$，计算如果每个小桶减少到至少可以去除多少克瓦斯$x$。 对于每个桶，如果$v_i > x$，我们可以删除$v_i - x$，否则为零。 
4. 如果总可移除克瓦斯至少为$s$， 然后$x$是可行的。 这意味着我们可以实现所有小桶至少都在的配置$x$，所以我们尝试增加$x$。 
5. 如果总可移除克瓦斯小于$s$， 然后$x$太大了。 我们无法在去除足够的克瓦斯的同时保持那么高的最小值，所以我们减少$x$。 
6. 运行二分搜索来找到最大的可行值$x$。 返回它作为答案。 

### 为什么它有效

 关键的性质是单调性。 如果一个值$x$是可行的，那么任何较小的值也是可行的，因为降低最小值只会增加可用的可移除 kvass。 这创建了一个单调谓词$x$，这保证了二分查找找到最佳阈值。 可行性检查准确地捕获了分配清除的所有有效方式，因为任何有效的最终配置都对应于每个桶减少多少的某种选择，并且完全由阈值约束来表征。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(v, s, x):
    total = 0
    for a in v:
        if a > x:
            total += a - x
    return total >= s

n, s = map(int, input().split())
v = list(map(int, input().split()))

if sum(v) < s:
    print(-1)
    sys.exit()

lo, hi = 0, max(v)
ans = 0

while lo <= hi:
    mid = (lo + hi) // 2
    if can(v, s, mid):
        ans = mid
        lo = mid + 1
    else:
        hi = mid - 1

print(ans)
```该代码首先对总和进行可行性检查，因为任何安排都不能超过可用的 kvass。 辅助函数`can`如果我们将所有小桶的盖子限制在`x`。 然后，二分搜索在可行时向上调整候选最小值，否则向下调整，最终收敛到最大有效最小值。 

这里的一个常见错误是忘记，一旦我们确定了阈值，从不同桶中的移除是独立的。 另一种方法是尝试贪婪地模拟，而不是将问题转换为单调决策函数。

## 测试用例```python
import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n, s = map(int, input().split())
    v = list(map(int, input().split()))

    if sum(v) < s:
        return "-1"

    def can(x):
        total = 0
        for a in v:
            if a > x:
                total += a - x
        return total >= s

    lo, hi = 0, max(v)
    ans = 0

    while lo <= hi:
        mid = (lo + hi) // 2
        if can(mid):
            ans = mid
            lo = mid + 1
        else:
            hi = mid - 1

    return str(ans)

def run(inp: str) -> str:
    return solve(inp)

# provided sample
assert run("3 3\n4 3 5\n") == "3"

# minimum case
assert run("1 1\n5\n") == "4"

# impossible case
assert run("2 10\n1 2\n") == "-1"

# all equal
assert run("4 4\n5 5 5 5\n") == "4"

# large skew
assert run("3 5\n10 1 1\n") == "3"

 ## 边缘情况

 当总克瓦斯小于$s$，算法立即返回-1。 例如，输入`2 10`带小桶`[1, 2]`产生总和 3，这是不够的。 早期检查可以防止二分查找产生无意义的值。 

当所有小桶都相等时，说`[5, 5, 5]`和$s = 2$，可行性检查表明任何高于 3 的最小值都是不可能的，因为所有小桶中只有 2 个总移除量可用。 二分查找正确地稳定在 3。 

当一个小桶明显大于其他小桶时，例如`[10, 1, 1]`，清除量集中在大桶中。 阈值公式仍然有效，因为它只计算超出的部分$x$，与移除的来源无关。
