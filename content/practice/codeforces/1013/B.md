 ## 解决方案
 ## 问题理解

 我们得到一个整数列表和一个特殊整数`x`。 唯一允许的转换是选择一个索引并用该元素的按位 AND 替换该元素`x`。 此操作只能减少位，因为 AND 只能将 1 位变为 0，具体取决于`x`。 

目标不是使数组在任何全局意义上进行排序或优化。 唯一的要求要简单得多：在应用该操作任意次之后，我们希望至少有一个值出现在至少两个不同的位置。 我们希望强制执行此类重复所需的操作数量最少，或者确定这是不可能的。 

约束条件`n ≤ 100000`迫使任何解决方案接近线性。 任何针对所有值和中间状态的二次配对策略都会太慢。 取值范围可达`100000`意味着我们可以安全地使用频率数组或哈希图，因为状态空间足够小以跟踪精确值。 

一个幼稚的错误来自于认为我们必须模拟对所有索引子集的操作。 例如，尝试所有对`(i, j)`检查是否可以通过 AND 运算序列使它们相等很快就会爆炸，因为每个元素最多可以以一种有用的方式独立转换。 

第二个微妙的问题是假设重复操作可能有助于超出每个元素一个应用程序的范围。 这是不正确的：申请`a & x`两次与应用一次相同，因此每个元素只有两种可能的状态：原始状态或转换后的状态。 

当没有值可以改变时，就会出现最终的边缘情况。 例如，如果`a[i] & x == a[i]`为所有人`i`，那么任何操作都不会改变任何东西。 如果数组最初没有重复项，则答案立即`-1`。 

## 方法

 强力视角将考虑索引的所有子集来应用操作，然后检查是否有任何值出现两次。 对于每个子集，我们将重新计算数组并​​计算频率。 这是指数级的`n`，因为有`2^n`选择，甚至修剪也没有多大帮助，因为每个操作都会改变全局碰撞结构。 它失败的原因是每个元素的操作是独立的，因此不需要全局搜索。 

关键的观察是每个元素都有两个有意义的状态：它的原始值`a[i]`，及其变换后的值`a[i] & x`。 由于我们只需要任何重复值，因此我们只关心是否可以使用每个元素最多一次转换来使某个值至少出现两次。 

这将问题减少到检查三种情况。 首先，如果任何值已经出现至少两次，则需要零操作。 其次，如果转换后的值与现有的原始值匹配，我们可以使用一个操作来创建副本。 第三，如果两个不同的元素都可以转换为相同的值，我们需要两次操作。 

我们评估所有原始值和所有转换值以及跟踪频率。 答案是在组合系统中实现任何频率 ≥ 2 所需的最小操作数，但要小心确保我们不会重复计算已经相等的元素。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 频率+变换检查| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 计算哈希映射或数组中所有原始值的频率。 如果任何值的频率至少为 2，则答案为 0。这直接满足目标，无需执行任何操作。 
2. 为变换后的值构建第二个频率图`b[i] = a[i] & x`。 我们不修改数组； 我们只计算这些假设结果。 
3. 对于每个索引，考虑对其进行变换是否会创建原始频率图中已存在的值。 如果是，这个单一操作会立即创建一个副本，因此我们可以记录候选答案 1。 
4. 接下来，计算所有索引中转换值的频率。 如果任何转换后的值至少出现两次，则对其中两个索引执行操作会创建一个重复值，因此我们可以通过两次操作来实现目标。 
5. 如果可以的话，答案是 0、1 或 2 中的最小值，否则`-1`。 

### 为什么它有效

 每个元素最多贡献两个可能的值，并且每个操作仅将一个元素从其原始值移动到其转换后的值。 因此，任何解决方案都相当于选择索引的子集来“激活”转换。 创建重复项的唯一方法是将两个元素在这两种状态下对齐为相同的值。 由于没有进一步的转换，整个问题简化为计算这两个值集之间的重叠，并且任何最佳解决方案都必须落入所识别的三种情况之一。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, x = map(int, input().split())
    a = list(map(int, input().split()))
    
    freq = {}
    for v in a:
        freq[v] = freq.get(v, 0) + 1
    
    # already has duplicate
    for v in freq:
        if freq[v] >= 2:
            print(0)
            return
    
    # try one operation case
    seen_after = {}
    for v in a:
        nv = v & x
        if nv in freq:
            print(1)
            return
        seen_after[nv] = seen_after.get(nv, 0) + 1
    
    # check two operations case
    for v in seen_after:
        if seen_after[v] >= 2:
            print(2)
            return
    
    print(-1)

if __name__ == "__main__":
    solve()
```第一遍构建原始值的频率表，这让我们可以立即检测数组是否已经满足条件。 第二遍计算每个可能的转换值并检查它是否可以与现有值发生冲突，这对应于单个操作就足够了。 

第三个检查聚合变换后的频率，以查看两个不同的指数在变换后是否收敛到相同的值。 这是当不存在原始重复项并且没有单个转换与现有值对齐时强制重复的唯一方法。 

顺序很重要，因为我们想要最少的操作数，所以我们必须检测`0`前`1`， 和`1`在考虑之前`2`。

 ## 边缘情况

 考虑一个数组，该数组最初不存在重复项，并且每次转换都会产生相同的值。 例如，`a = [5, 7, 3]`和`x = 1`。 变换后的值全部变为`1`。 算法构建`seen_after = {1: 3}`，检测频率≥2，返回`2`。 这符合要求，因为必须转换至少两个索引才能达到相等。 

另一种边缘情况是转换不执行任何操作。 如果`x = 0`， 然后`a[i] & x = 0`为所有人`i`。 如果原始数组不包含重复项，则转换后的值不会以有用的方式与任何原始值匹配，并且`seen_after`无助于与现有价值观发生冲突。 算法正确返回`-1`在单操作和双操作检查均失败后。
