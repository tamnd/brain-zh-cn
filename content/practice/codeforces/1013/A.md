[CF 1013A - 石头堆](https://codeforces.com/problemset/problem/1013/A)

 ## 解决方案
 ## 问题理解

 我们得到了同一石堆系统的两张快照。 在第一个快照中，每堆石子都有一定数量，而在第二个快照中，石堆的数量不同。 在这两个时刻之间，在夜间，两种类型的操作可能会发生任意多次：可以将一块石头从一堆中完全移走，或者可以将一块石头从一堆移动到另一堆。 对于发生多少这样的操作或有多少不同的人执行它们没有限制，因此我们只关心是否可以使用这两个原始操作将第一个配置转换为第二个配置。 

问题不在于构建一系列行动，而只是决定可行性。 我们正在有效地检查是否可以使用“从任何索引中删除 1”和“在索引之间转移 1 个单位”操作将一个整数数组转换为另一个整数数组。 

限制很小，最多 50 个桩。 这立即排除了对繁重数据结构或线性或二次推理之外的优化的任何需求。 任何 O(n²) 甚至 O(n) 的解决方案都足够快，问题实际上在于理解这些操作保留了什么以及它们可以改变什么。 

当石子总数的变化无法仅通过移除来解释时，或者当重新分布不足以匹配目标形状时，就会出现微妙的失败情况。 

例如，如果我们有：```
x = [1, 0]
y = [0, 2]
```这是不可能的，因为我们无法制造额外的石头。 移动仅重新分配或删除棋子。 

另一个例子：```
x = [0, 0]
y = [1, 1]
```出于同样的原因，这也是不可能的：没有任何操作会增加总石头数。 

这些例子表明，核心困难不是每桩行为，而是全球保护和不平等结构。 

## 方法

 暴力解释将模拟所有可能的移动和移除序列。 每一步要么减少石子总数，要么在位置之间转移一个单位。 尽管分支因子在概念上很大，但状态空间却很大，因为堆可以采用许多中间值。 这立即变得棘手，因为即使 n 和堆大小的值很小，也会产生指数级的许多可达状态。 

关键的观察结果是，这些操作可以在全球范围内而不是在本地范围内被理解。 移动一颗石子不会改变石子的总数，而移走一颗石子会使总数恰好减少 1。 这意味着系统中唯一不可逆的影响是减少所有元素的总和。 

由此，我们得到了一个强大的简化：从初始数组开始，我们可以使用移动任意重新分配棋子，因此唯一真正的约束是我们是否有足够的棋子总体来匹配目标配置。 如果目标需要的石头比我们最初拥有的更多，那是不可能的。 如果我们至少有同样多的棋子，我们总是可以丢弃多余的棋子并通过转移重新排列其余的棋子。 

因此，整个问题简化为比较两个数组的总和。

 ## 算法演练

 1. 计算初始配置中的棋子总数。 这代表我们在进行任何操作之前开始使用的所有石头。 
2. 计算最终配置中的棋子总数。 这是所有操作后必须保留的石块量。 
3. 比较两个总和。 如果初始总和小于最终总和，则输出“No”，因为我们无法创建新的石头。 
4. 否则，输出“Yes”，因为任何多余的宝石都可以被移除，并且任何分配差异都可以通过转移来修复。 

### 为什么它有效

 关键的不变量是转移保留总和，而移除则每次操作将总和减少 1。 因此，该过程施加的唯一限制是最终总数不能超过初始总数。 由于我们可以在堆之间自由移动石头，因此一旦我们有足够的总质量，任何重新分配都是可以实现的。 不存在诸如奇偶校验或每个索引约束之类的隐藏结构，因为除了我们无法创建新的石头之外，没有任何操作限制石头的重新排列方式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    x = list(map(int, input().split()))
    y = list(map(int, input().split()))

    if sum(x) >= sum(y):
        print("Yes")
    else:
        print("No")

if __name__ == "__main__":
    solve()
```该解决方案读取两个数组并直接计算它们的总和。 这种比较就足够了，因为所有允许的操作要么保留或减少石子的总数，而不是增加它。 

一个常见的实施错误是尝试单独匹配桩。 这是不必要的并且具有误导性，因为传输允许任意重塑。 只有全球总和才重要。

 我们看到两个总数相等，因此不需要移除石头。 由于转移允许任意重新分配，因此我们可以通过在堆之间移动石子将 x 转换为 y。

 输入：```

```| 步骤| 总和（x）| 总和（y）| 决定|

这些例子表明，平等可以带来充分的灵活性，而赤字会立即阻碍可行性。

 ## 边缘情况

 当所有桩最初为零并且最终也为零时，两个总和都为零，因此算法立即正确地输出“是”。 

对于这样的情况：```
x = [2, 0]
y = [1, 1]
```该算法计算 sum(x) = 2 和 sum(y) = 2，因此返回“Yes”。 尽管分布不同，但单个转移操作会将一颗石子从 1 堆移动到 2 堆，与目标完全匹配。 

对于这样的情况：```
x = [1, 0]
y = [0, 2]
```我们得到 sum(x) = 1 和 sum(y) = 2。该算法正确地返回“否”，因为即使无限转移也无法弥补丢失的石子总数。 

每个边缘情况都证实了相同的不变量：只有石子总数很重要，并且重新分配没有额外的限制。
