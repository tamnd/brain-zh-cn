 ## 解决方案
 ## 问题理解

 我们有一个垂直堆栈$n$冰淇淋层。 每一层都是字母表上的一个字符串$\{R, G\}$，从左到右代表该层中的糖果。 该过程是严格顺序的：我们从第 1 层开始，然后是第 2 层，依此类推，直到第 1 层$n$。 在一层内，我们必须按照给定的顺序消耗糖果，并且不能在层内或层间重新排序。 

如果我们想象通过按顺序连接所有层将它们展平为一个长序列，那么问题就变成：找到仅由一种颜色组成的最长连续段，或者全部由一种颜色组成$R$或全部$G$，但有限制，即该段尊重层边界和遍历顺序。 

因此，我们实际上正在寻找所有给定字符串串联的单色子字符串的最大长度。 

关键的微妙之处在于我们没有显式构建连接字符串，因为总长度可达$2 \cdot 10^5$，这很好，但我们也可以通过跟踪层之间的转换以流形式解决它。 

约束条件$\sum c_i \le 2 \cdot 10^5$意味着$O(N)$或者$O(N \log N)$需要解决方案，但由于结构是线性的，因此单遍就足够了。 任何层或位置上的二次方法都会太慢。 

当运行跨越层边界时，会出现边缘情况。 例如：

 输入：```
2
RRR
RR
```正确的输出是 5，因为运行是跨层继续的。 简单的每层最大值将错误地返回 3。 

另一个案例：```
3
RR
GR
RR
```粗心的方法可能会在每个层边界处重置，并错过可以根据边界字符重新启动或继续运行的情况。 

主要风险是当边界字符匹配时无法合并连续层。 

## 方法

 一种强力解释是将所有字符串连接成一个序列，然后通过扫描来计算最长的相同字符。 这很简单：迭代每个字符，跟踪当前的条纹长度，当字符更改时重置，并更新最大值。 

这是正确的，因为该问题本质上是线性序列上的游程长度问题。 然而，尽管串联在概念上很简单，但显式构建字符串是不必要的，并且如果不仔细处理，在不合并转换的情况下重复处理每层可能会导致不正确的逻辑。 

改进来自于我们从不需要随机访问或回溯的观察。 我们只需要在流输入中保持当前的连续性。 在任何时候，唯一需要的信息是最后看到的字符和当前的条纹长度。 

这将问题简化为对所有层中的所有字符进行一次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（连接+扫描）| O(N) | O(N) | 已接受 |
 | 流媒体单通道 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们逐层处理层，并在每一层内从左到右处理字符，同时保持全局条纹。 

1. 初始化变量`last`如空，`cur`为 0，并且`best`为 0。 

它们跟踪前一个字符、当前的游程长度以及迄今为止看到的最大游程。 
2. 按从 1 到 的顺序迭代每一层字符串$n$。 

这保留了所需的消费顺序。 
3. 对于每个角色$ch$在当前字符串中：

 如果$ch = last$, 增量`cur`1. 否则，重置`cur`为 1 并设置`last = ch`。 

这一步保持了不变式`cur`正好是当前由相同字符组成的最大后缀的长度。 
4.更新后`cur`， 更新`best = max(best, cur)`。 

这确保我们捕获每个位置结束时的最大运行。 
5.处理完所有层后，输出`best`。 

### 为什么它有效

 在展平序列中的每个位置，算法都会维护在该位置结束的最大连续块的长度。 这已经足够了，因为任何有效的答案都必须在某个位置结束，并且每个可能的运行在结束时都会被考虑一次。 将状态压缩为`(last, cur)`就足够了，因为一旦发生角色变化，当前运行之外的所有早期历史都无关紧要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    last = None
    cur = 0
    best = 0

    for _ in range(n):
        s = input().strip()
        for ch in s:
            if ch == last:
                cur += 1
            else:
                last = ch
                cur = 1
            if cur > best:
                best = cur

    print(best)

if __name__ == "__main__":
    solve()
```该实现直接流过所有层，无需构建组合字符串。 变量`last`即使跨层边界也能确保正确检测转换，因为状态在字符串循环之间持续存在。 

一个常见的错误是重置`last`或者`cur`在每个新层，这会错误地破坏跨层的有效运行。

 ## 边缘情况

 一个关键的边缘情况是运行完全跨越层边界。 考虑：```
2
RRR
RR
```算法保持`last = 'R'`完成第一层之后，那么进入第二层的时候，连胜自然会继续下去。 状态变成`cur = 5`，产生正确答案。 

另一种情况是重复重置：```
3
R
G
R
```这里每个角色都改变力量`cur = 1`。 该算法永远不会错误地合并不连续的段，因为`last`始终在全局范围内跟踪前一个角色。 

最后，层内的边界更改与跨层的边界更改的处理方式相同，因为两者都只是单个 s 中的字符比较
