 ## 解决方案
 ## 问题理解

 我们得到一行$N$染料，每种都有一个正整数的美丽值。 从这个序列中，我们想要选择位置的子集，使得原始行中没有两个选定的位置是相邻的。 一个选择的分数是所选择的美丽值的总和，我们希望最大化这个分数。 

用更熟悉的术语来说，这是路径图上的最大权重独立集：每个位置都是一个节点，边连接连续的索引，并且我们无法选择相邻的节点。 

约束条件$N \le 10^5$排除任何指数子集枚举。 一个天真的$O(2^N)$方法是不可能的，甚至$O(N^2)$子数组上的转换会太慢。 我们需要一个线性或近线性的动态规划解决方案。 

一个微妙的边缘情况来自贪婪直觉失败的小数组。 例如，考虑：

 输入：```
3
10 1 10
```如果不小心的话，像选择局部最优元素这样的天真贪婪策略可能会选择两个 10，但它们并不相邻，因此在这里可行。 然而：

 输入：```
4
8 9 8 9
```根据实现顺序，在没有状态跟踪的情况下重复贪婪地选择最大剩余元素可能会失败，而正确答案是 18（选择索引 2 和 4）。 

另一个边缘情况是当$N=1$，其中答案只是单个值，并且任何 DP 公式都必须正确初始化基态。 

## 方法

 强力解决方案将尝试索引的每个子集并检查它是否包含相邻对。 对于每个子集，我们计算其总和并跟踪最大值。 有$2^N$子集，并检查每个子集成本$O(N)$如果直接实施，则给出$O(N2^N)$时间。 即使我们使用位运算优化子集检查，子集的指数数量仍然是瓶颈。 

关键的观察是决策是局部的：在索引处$i$，我们只需要知道我们是否采取了$i-1$。 如果我们跳过索引$i$, 最高得分可达$i$与最多相同$i-1$。 如果我们取索引$i$，我们必须将其价值添加到最佳解决方案中，直到$i-2$。 这创建了一个最佳子结构，其中答案高达$i$仅取决于前两个状态。 

这将问题简化为简单的递归，类似于经典的“非相邻元素的最大和”。 

让$dp[i]$是仅使用第一个可实现的最大总和$i$元素。 然后：$$dp[i] = \max(dp[i-1], dp[i-2] + b_i)$$这立即导致线性时间 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N2^N)$|$O(N)$| 太慢了 |
 | 动态规划|$O(N)$|$O(1)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时仅保留最后两个 DP 状态。 

1. 初始化两个变量来表示空前缀和长度为一的前缀的最佳答案。 空前缀的值为 0，第一个元素的值为$b_1$。 
2. 对于每个位置$i$从 2 到$N$，计算两种可能性：跳过当前元素，这会保留先前的最佳元素，或者采用当前元素，这会添加$b_i$退两步才能达到最佳效果。 这两个状态中较大的一个成为新状态。 
3. 计算新状态后，将先前的状态向前移动，以便“先前”和“两步后退”值始终正确对齐以用于下一次迭代。 
4. 继续，直到处理完所有元素。 最终的答案是最后位置的最佳值。 

这样做的原因是每个最佳解决方案必须包含或排除最后一个元素，并且这两种情况都会减少到在所需约束中不重叠的严格较小的子问题。 

### 为什么它有效

 在每个索引处$i$，任何有效的选择都会分为两个不相交的类：那些排除$i$，这正是第一个的所有有效选择$i-1$元素，以及那些包括$i$，这强制排除$i-1$并减少第一个的有效选择$i-2$元素。 这两种情况涵盖了所有可能性并保留了最优子结构，因此归纳地取最大值保留了全局最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
b = list(map(int, input().split()))

if n == 0:
    print(0)
    sys.exit()

if n == 1:
    print(b[0])
    sys.exit()

prev2 = 0
prev1 = b[0]

for i in range(1, n):
    take = prev2 + b[i]
    skip = prev1
    cur = max(skip, take)
    prev2 = prev1
    prev1 = cur

print(prev1)
```该实现将 DP 数组压缩为两个变量。`prev1`代表$dp[i-1]$， 尽管`prev2`代表$dp[i-2]$。 在每一步，我们直接计算递归。 

基本情况句柄$N=1$明确地这样我们就不会依赖于未定义的 DP 值。 转换顺序很重要：`prev2`必须在计算当前状态后更新，否则我们将覆盖下一次迭代所需的值。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n = int(input().strip())
    b = list(map(int, input().split()))
    
    if n == 0:
        return "0"
    if n == 1:
        return str(b[0])

    prev2 = 0
    prev1 = b[0]

    for i in range(1, n):
        cur = max(prev1, prev2 + b[i])
        prev2 = prev1
        prev1 = cur

    return str(prev1)

# provided sample

# minimum size
assert run("1\n42\n") == "42"

# two elements
assert run("2\n5 100\n") == "100"

# all equal
assert run("5\n7 7 7 7 7\n") == "21"

# alternating high values
assert run("4\n10 1 10 1\n") == "20"

 ## 边缘情况

 对于$N=1$，该算法直接返回单个值，因为完全跳过了循环。 这可以避免访问无效的 DP 状态。 

为了$N=2$，转换正确地比较第一个或第二个元素，而不需要初始化之外的任何特殊逻辑。 

例如：```
2
5 100
```初始化给出`prev2 = 0`,`prev1 = 5`。 在索引 2 处，我们计算`take = 100`,`skip = 5`，因此结果为 100。这与仅选择第二个元素的最佳选择相匹配。 

这表明即使在没有额外分支逻辑的最小配置中，递归也能自然地处理邻接约束。
