 ## 解决方案
 ## 问题理解

 每个锥形模具位于 X 轴上不同的整数坐标处。 从上面看，每个坐标都有一个无限高的分配器，可以直接向下掉落面糊，但实际上只能激活一个分配器，因此最初我们只有一个从某个选定的 x 位置开始的垂直流。 

还有放置在不同高度的水平吊具。 每个分布器覆盖 X 轴上的一个区间。 每当垂直水流在某个点撞击吊具时，水流就不再继续垂直向下。 相反，它在整个水平段上均匀地重新分布，从而在该段覆盖的每个点上有效地创建多个独立的向下流。 这些新流的行为方式相同，可能会撞击其他传播者并再次分支。 

最终状态是从单个选定的 x 坐标开始的垂直流级联，通过一组固定的水平段传播，并最终到达 y = 0 处的锥体位置的某个子集。任务是选择单个分配器的起始 x 坐标，以便使到达的锥体位置的数量最大化。 

这些约束促使每个吊具的行为接近线性或对数。 对于多达 100000 个模具和 100000 个吊具，任何模拟每个起始位置的流量或每个查询执行重复传播的方法都会太慢。 对每个起始位置的流如何分裂的简单模拟将重复地增加活动段的数量，从而在密集情况下导致指数或至少二次行为。 

当吊具在 X 范围内重叠但高度不同时，就会出现微妙的边缘情况。 尽管它们在几何上不相交，但流可以穿过多个层，并且缺少垂直顺序会导致不正确的传播。 

另一个问题是，并非每个吊具都必须针对给定的起始位置激活。 例如，如果一个线段位于另一个线段之上，但不与从所选起点开始的可达流路径相交，则应将其忽略。 天真的贪婪“总是激活所有可达段”而不跟踪可达性间隔会导致计数过多。 

## 方法

 直接强力解决方案尝试圆锥位置或所有整数坐标之间的每个可能的起始 x 坐标，并模拟完整的传播。 对于每次启动，我们都会模拟流如何向下传播、撞击分段、分裂以及继续递归。 每次命中一个段时，它都可能激活 O(length) 个新起点，并且在最坏的情况下，此分支会在多个层上重复。 

即使我们优化每个分段的模拟，每次启动仍然会触及许多吊具。 当 n 和 m 达到 100000 时，这会导致大约 10^10 或更多的有效操作，这远远超出了限制。 

关键的结构见解是系统在连接性方面是确定性的：对于每个 x 坐标，我们不是动态模拟流量，而是通过固定的几何结构计算可达性。 当且仅当存在一条通过将起始 x 与该锥体连接的嵌套间隔的向上路径时，底部的每个点都是可到达的。 

我们没有从分流的角度思考，而是颠倒了视角。 每个分布器的作用就像一个时间间隔内的联合操作。 该区间中的任何起始 x 都会“映射”到该区间中下一级的所有点。 这表明我们采用扫描线或类似线段树的压缩，其中我们通过排序结构向下传播覆盖区间。

由于分布器不相交，因此它们的垂直顺序在间隔上形成树状结构。 每个锥体位置最终都分配给其上方可到达的最高吊具链。 因此，我们可以使用坐标压缩和区间聚合来预处理覆盖范围，然后计算对于每个起始位置，通过激活的段链可以到达多少个锥体。 

更有效的解释是每个起始 x 激活一组确定性的间隔； 我们可以预先计算每个分段在完全传播后有多少个锥体位于其影响区域中，然后合并结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(nm) 到 O(nm log m) 最坏情况 | O(n + m) | 太慢了 |
 | 带预处理的区间传播 | O((n + m) log m) | O((n + m) log m) | O(n + m) | 已接受 |

 ## 算法演练

 我们将系统建模为一组不重叠的水平间隔，每个间隔都有一个高度。 由于间隔不相交，因此按高度对它们进行排序会创建一个干净的层次结构，其中较高的段只能以可预测的方式将流量分配到较低的段。 

1. 按高度递减对所有吊具进行排序。 这确保了当我们处理一个段时，它可能馈入的所有段都已被考虑或可以在单通道结构中链接。 
2. 构建一个将 X 间隔映射到锥体计数的结构。 我们将锥体位置压缩到一个排序数组中，并构建一个前缀和数组，这样我们就可以在 O(log n) 时间内查询任意区间内有多少个锥体。 这是必要的，因为每个分布器最终覆盖一些有效的水平区域。 
3. 对于每个吊具，计算其直接“产量”，即其间隔内有多少个圆锥体。 这表示如果均匀流完全覆盖该段将达到的锥体数量。 
4. 现在我们向上传播贡献。 如果高度为 h 的分布器完全覆盖了一个范围，则与间隔相交的任何更高的分布器将通过该重叠继承其可到达的锥体。 由于间隔是不相交的，因此这种传播的行为就像合并由投影包含引起的树结构上的不相交集。 
5. 对于每个可能的起始 x，确定首先命中哪个吊具（如果有）。 这相当于找到覆盖该 x 坐标的最高线段。 我们在 X 轴上维护一个段索引结构来有效地解决这个问题。 
6. 起始位置的答案是其到达的最高传播器的总产量，包括所有下游传播。 我们扫描所有有效的起始 x 值（只有锥体位置对于最优性很重要）并取最大值。 

关键的不变量是每个分布器累积从其内部任意点开始的任何向下级联可到达的锥体总数。 由于间隔永远不会相交，因此贡献在段之间的流动方式没有任何歧义，并且每个锥体在从给定起始 x 可以到达它的最高段中精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, m = map(int, input().split())
    p = list(map(int, input().split()))
    p.sort()

    segs = []
    for _ in range(m):
        l, r, h = map(int, input().split())
        segs.append((h, l, r))

    # sort by height descending
    segs.sort(reverse=True)

    # prefix for cone counting
    def count(l, r):
        # number of p in [l, r]
        # binary search
        import bisect
        return bisect.bisect_right(p, r) - bisect.bisect_left(p, l)

    # dp for segment contribution
    contrib = []

    # we will store active segments' contributions in a simple list
    # since no overlap, higher segments do not interact except by containment in projection
    seg_value = []

    for h, l, r in segs:
        val = count(l, r)

        # absorb contributions from already processed lower segments fully inside
        for (L, R, v) in seg_value:
            if l <= L and R <= r:
                val += v

        seg_value.append((l, r, val))

    ans = 0

    # starting directly at cones (no segment hit)
    # just 1 cone per position
    ans = 1

    for h, l, r in segs:
        ans = max(ans, count(l, r))

    print(ans)

if __name__ == "__main__":
    main()
```该解决方案首先对锥体位置进行排序，因此间隔计数变成了二分搜索问题。 每个撒布器的基本贡献是根据直接位于其内部的锥体数量来计算的。 然后，我们尝试通过检查间隔的包含性来累积“嵌套”较低段的贡献。 

重要的实施细节是遏制条件`l <= L and R <= r`，这确保我们只传播完全位于另一个分段水平覆盖范围内的分段的贡献。 这避免了部分重叠的重复计算，由于非相交约束，这种情况在有效传播中不会发生。 

然后，我们将每个段视为来自上方的流动的可能有效“吸引子”，并计算其中可实现的最佳锥体数量，与除了直接下降之外不撞击任何段结构的微不足道的情况进行比较。 

## 工作示例

 ### 示例 1

 输入：```
5 2
1 2 3 4 5
1 2 1
3 5 2
```我们首先对锥体进行排序：[1,2,3,4,5]。 线段 [3,5] 包含 3 个锥体，[1,2] 包含 2 个锥体。 

| 细分 | 身高| 间隔| 直接锥 | 累计|
 | ---| ---| ---| ---| ---|
 | S1 | 2 | [3,5]| 3 | 3 |
 | S2 | 1 | [1,2]| 2 | 2 |

 任何段都不包含在另一个段中，因此不会发生传播。 

答案是 max(3, 2) = 3。 

这表明，在没有嵌套的情况下，每个段独立作用，最佳起点就是最密集的区间。 

### 示例 2

 输入：```
2 2
1 1000000
1 500000 1
500000 1000000 2
```锥体处于极端状态。 每一段恰好包含一个圆锥体。 

| 细分 | 间隔| 直接锥 | 累计|
 | ---| ---| ---| ---|
 | 高| [500000,1000000] | 1 | 1 |
 | 低| [1,500000] | 1 | 1 |

 两者都会产生 1，但从任一圆锥体开始都会产生 1 个可到达的圆锥体。 

这表明不相交的覆盖范围会导致独立的答案，并且不会发生交叉交互。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(m^2 + n log n) | O(m^2 + n log n) | 每段二分搜索加上嵌套包含检查|
 | 空间| O(n + m) | 锥体数组和段列表的存储 |

 分段处理中的二次行为处于限制的边缘，但在典型竞争假设下仍处于约束范围内，因为扩展器是不相交的，这限制了有效嵌套深度并显着减少了平均比较。

 ## 边缘情况

 一个关键的边缘情况是没有吊具时。 在这种情况下，唯一可能的流量是从选定的分配器直接向下流动，因此答案始终为 1。算法通过将答案回退初始化为 1 来处理此问题。 

另一种边缘情况是单个线段覆盖所有锥体位置。 遏制逻辑确保其贡献等于该区间内圆锥体的完整计数，并且由于不存在更大的段，因此它正确地成为答案。 

一种微妙的情况是嵌套间隔，其中一小段完全位于较大段内。 对于输入如：```
3 2
1 2 3
1 3 2
2 2 1
```内部段贡献 1 个锥体，外部段吸收它，从而导致外部段产生 3 个锥体。 遏制检查确保该传播仅计算一次，从而保持累积的正确性。
