 ## 解决方案
 ## 问题理解

 我们在平面上得到一组点，每个点代表放置在 x 轴上方某处的洒水，因为所有 y 坐标都严格为正。 每个洒水都有一种颜色，红色或蓝色。 鲍勃固定在原点，并选择一个形状像扇形、顶点位于原点的几何区域。 该区域的半径不受限制，但受限于角度间隔，因此在几何上，它对应于从原点选择两条射线并按逆时针顺序获取它们之间的所有点。 

如果洒水位于该扇区内部或边界上，则收集该洒水。 鲍勃想要选择一个完全不包含蓝点的扇区，同时最大化其中的红点数量。 

输入大小可以达到 200,000 个点，因此任何显式检查所有点对或所有角度间隔的方法都将失败。 在最坏的情况下，二次解将需要大约 400 亿对考虑因素，这远远超出了 2 秒所允许的范围。 这立即表明问题必须简化为一维结构，很可能涉及围绕原点的角度排序。 

一个微妙的问题来自于圆的几何形状。 即使所有点都位于上半平面内，角度也会在负 x 轴处环绕。 任何解决方案都必须仔细处理点按角度的循环排序。 

当蓝点按角度顺序位于红色簇之间时，会出现另一种边缘情况。 在角度排序中简单地采用最大连续红点块的简单方法是错误的，因为它可能在间隔内包含蓝点，或者错过最佳扇区可以在任意点开始和结束的情况。 

第三个陷阱是将扇区视为线性区间而不处理环绕情况。 最佳扇区可能与 x 轴正方向交叉，这意味着可能需要复制角度间隔来模拟圆形连续性。 

## 方法

 一个蛮力的想法是将每对可能的点视为扇区的边界。 对于每个有序的点对，我们想象一个扇区，按逆时针顺序从第一个角度开始，到第二个角度结束，然后计算内部有多少个红点，同时确保不包含蓝点。 这需要检查每对的所有点，导致时间复杂度为 O(N^3)，如果通过预处理优化计数，则时间复杂度为 O(N^2)，但对于 200,000 个点来说仍然太大。 

关键的观察结果是，唯一有意义的扇区边界出现在给定点定义的角度处。 如果我们固定一个边界并旋转另一个边界，“内部没有蓝点”的条件就成为排序角顺序上的滑动窗口约束。 这将问题转化为寻找不包含蓝点并最大化红点的最长有效角度间隔。 

一旦所有点都转换为极角，我们就可以对它们进行排序并复制数组以处理环绕。 那么问题就变成在这个双圆形阵列中选择一个连续的线段，使其不包含蓝点并且红点的数量最大化。 两指针扫描会维护一个不存在蓝点的窗口； 每当一个蓝点进入窗口时，我们就向前移动左指针，直到恢复约束。 

在每个有效窗口内，我们跟踪包含多少个红点并取最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 扇区对上的暴力破解 | O(N^3) | O(N^3) | O(N) | 太慢了 |
 | 排序 + 角度上的两个指针 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 1. 使用atan2(y, x) 将每个点(x, y) 转换为极角。

这确保了围绕原点的正确角度排序，尊重象限结构。 
2. 将每个角度与其颜色配对，并按角度对列表进行排序。 

排序将圆形几何形状线性化为序列。 
3. 通过为每个元素附加 (angle + 2π, color) 来复制已排序的数组。 

这允许将环绕间隔表示为普通子数组。 
4. 在双倍数组的开头初始化两个指针 l 和 r 并维护一个窗口。 

该窗口表示当前候选角度扇区。 
5. 逐步展开r，将点添加到窗口中。 

每次包含一个点时，我们都会更新红色和蓝色点的计数。 
6. 如果窗口中出现蓝色点，则向前移动l，直到没有蓝色点为止。 

这恢复了该扇区必须包含零个蓝点的有效条件。 
7. 每当窗口有效时，计算内部红点的数量并更新答案。 

最佳扇区必须对应于某个最大有效窗口边界对。 
8. 确保窗口长度永远不会超过 N，因为有效扇区不能包含超过完整旋转的唯一点。 

### 为什么它有效

 按角度排序后，每个有效扇区都对应于圆上的某个连续区间。 复制数组会将圆形间隔转换为线性间隔，而不会丢失任何候选扇区。 两指针过程保持当前窗口不包含蓝点的不变量，并且每当蓝点进入时，左边界都会前进到足以将其移除。 这保证了当右指针扩展时，每个最大有效角度间隔被恰好考虑一次，因此捕获所有有效的无蓝色扇区中的最大数量的红点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def main():
    n = int(input())
    colors = input().strip()
    
    pts = []
    for i in range(n):
        x, y = map(int, input().split())
        ang = math.atan2(y, x)
        pts.append((ang, colors[i]))
    
    pts.sort()
    
    arr = pts + [(ang + 2 * math.pi, col) for ang, col in pts]
    
    l = 0
    red = 0
    blue = 0
    ans = 0
    
    for r in range(len(arr)):
        if arr[r][1] == 'r':
            red += 1
        else:
            blue += 1
        
        while blue > 0:
            if arr[l][1] == 'r':
                red -= 1
            else:
                blue -= 1
            l += 1
        
        ans = max(ans, red)
        
        if r - l + 1 > n:
            if arr[l][1] == 'r':
                red -= 1
            else:
                blue -= 1
            l += 1
    
    print(ans)

if __name__ == "__main__":
    main()
```该实现直接遵循角度扫描构造。 atan2 转换至关重要，因为它保留了所有象限的正确顺序。 排序可确保任何有效扇区成为角度空间中的连续区间。
