 ## 解决方案
 ## 问题理解

 每个候选人都属于一个学科，并带来数字贡献，可能是正面的，也可能是负面的。 我们被要求在结构约束下选择一组候选者：选择后，我们还选择一些科目，并且对于每个选定的科目，该科目中选定的候选者数量必须相同。 

此约束迫使解决方案平衡主题之间的计数。 如果我们决定每个选定的主题必须做出准确的贡献$k$人，那么每个选定的科目都必须贡献自己的顶尖$k$因为增加一个较弱的候选人只能减少总和，对可行性没有帮助。 

输出是所有有效选择的最佳可能总和，或者如果每个有效非空结构的总和为负，则输出为零。 

约束条件达到$10^5$候选人和科目。 尝试所有主题子集或所有可能的解决方案$k$每个主题独立的值将立即失败。 任何平方项$n$甚至$n \log^2 n$常数较大的情况下有超时的风险。 

最微妙的边缘情况是当所有候选者都是负面的或者只有某些主题具有正面前缀时。 例如，如果每个值都是负数，则任何非空选择都会产生负和，因此正确答案为零。 另一个微妙的情况是，即使各个受试者具有不同的最佳计数，混合受试者也能改善结果。 一个天真的贪婪者独立地为每个主题选择最好的可能会失败，因为共享的$k$要求耦合所有科目。 

## 方法

 蛮力想法是决定一个数字$k$，然后为每个主题独立选择顶部$k$候选者，如果至少有$k$人们。 之后，我们总结所有主题的贡献并选择最好的$k$。 这是正确的，因为在固定的时间内$k$，每个主题的最优性是独立的。 

然而，成本在于评估每个$k$。 如果我们尝试所有$k$从$1$到$n$，并且对于每个$k$重新计算每个主题的前缀和，我们将花费$O(n)$每$k$，导致$O(n^2)$最坏情况下的操作。 和$10^5$，这是不可能的。 

关键的观察是，每个主题都可以预处理成一个排序的技能列表，并且其前缀和计算一次。 那么对于一个固定的$k$，总贡献是索引处前缀总和的所有主题的总和$k$，如果存在的话。 而不是重新计算每个$k$，我们翻转视角：每个前缀总和对所有前缀总和有贡献$k$直到其长度，因此我们可以通过迭代主题并将其前缀和分配到一个全局数组中来累积贡献，该数组索引为$k$。 

这将问题转化为排序后在线性时间内聚合所有主题的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对 k 和主题的暴力破解 |$O(n^2)$|$O(n)$| 太慢了|
 | 排序+前缀+累加|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 按主题对候选人进行分组。 这是必要的，因为约束独立地应用于每个主题。 
2. 对于每个科目，按降序排列其技能值。 这确保了采取第一个$k$元素对于该主题是最佳的。 
3. 为每个主题构建前缀和。 位置的前缀$i$如果我们精确地取，则表示最好的可能总和$i+1$该科目的候选人。 
4. 对于每个主题，迭代其前缀和。 对于$i$-th 前缀和，将其添加到组大小的全局累加器中$k = i+1$。 这代表在对尺寸有贡献的主题中选择该主题$k$。 
5. 维护全局数组`best[k]`如果每个选定的主题都准确贡献，则表示贡献总和$k$参与者。 
6. 处理完所有科目后，扫描所有$k$并取最大正值。 

步骤 4 起作用的原因是每个受试者独立地对每个可能的统一组大小做出贡献$k$，并且我们正在总结跨学科的兼容贡献。 

### 为什么它有效

 修复具有组大小的有效解决方案$k$。 在该解决方案中，每个选定的主题都贡献了其最高值$k$候选人，否则交换更好的候选人将在不违反约束的情况下提高总和。 因此，每个有效的解决方案都准确地对应于选择一些主题子集并对它们进行求和$k$- 前缀值。 该算法隐式枚举所有此类贡献并评估每个可能的贡献$k$，因此不会遗漏任何有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

n, m = map(int, input().split())

subjects = defaultdict(list)

for _ in range(n):
    s, r = map(int, input().split())
    subjects[s].append(r)

best = [0] * (n + 1)

for vals in subjects.values():
    vals.sort(reverse=True)
    pref = 0
    for i, v in enumerate(vals):
        pref += v
        k = i + 1
        if pref > 0:
            best[k] += pref

print(max(0, max(best)))
```该代码首先按主题对候选者进行分组，然后对每个组进行排序，以便前缀总和代表每个固定组大小的最佳选择。 这`best`数组根据每个主题的人数来累积贡献。 

支票`if pref > 0`确保我们只包含可以改善结果的前缀； 负前缀只会使任何使用该前缀的配置变得更糟$k$。 这与最终的最大化步骤一致，我们也将其钳位为零。

 | 主题 | 排序值 | 前缀和 |

 | 测量| 复杂性 | 说明|

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import defaultdict

    n, m = map(int, input().split())
    subjects = defaultdict(list)

    for _ in range(n):
        s, r = map(int, input().split())
        subjects[s].append(r)

    best = [0] * (n + 1)

    for vals in subjects.values():
        vals.sort(reverse=True)
        pref = 0
        for i, v in enumerate(vals):
            pref += v
            k = i + 1
            if pref > 0:
                best[k] += pref

    return str(max(0, max(best)))

 ## 边缘情况

 完全负面的实例展示了后备行为。 对于像这样的输入`[-1, -2, -3]`跨学科，每个前缀和都是负数，所以没有$k$累积正值，算法正确返回零，因为`max(best)`为负数或零。 

单受试者输入测试算法不需要多个受试者来形成有效的解决方案。 主体的最佳前缀和直接成为答案，并且累积机制仍然有效，因为只有一个主体贡献。 

每个受试者只有一个候选者的稀疏分布确保算法正确处理所有可能的情况$k=1$贡献。 每个主题只贡献一次`best[1]`，并且没有更高的$k$已填充，使最大值正确反映所有正单个条目的总和。
