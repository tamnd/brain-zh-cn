 ## 解决方案
 ## 问题理解

 我们得到一组顶点，并且对于每个顶点，我们被告知最多允许参与多少条边。 我们的任务是实际构建一个简单的无向连通图，该图尊重这些度数限制，并在所有此类图中最大化直径。 

这里的直径是任意两个顶点之间的所有最短路径距离中最长的。 因此，我们正在尝试构建一个连接结构，使某些顶点对保持尽可能远的距离，同时仍然遵守度数上限。 

这些限制足够小，因此可以期望采用基于构造的方法。 对于最多 500 个顶点，任何在仔细实现时大致呈二次甚至稍呈三次方的解都是可以接受的。 被排除的是任何试图在大搜索空间内重复搜索图形配置或计算直径的行为。 

关键的困难在于我们不仅仅是构建任何有效的图表。 我们必须明确地将直径推得尽可能高，这意味着我们需要了解哪些结构选择可以在度数限制下增加最短路径距离。 

当所有顶点的度数限制为 1 时，就会出现微妙的失败情况。在这种情况下，不存在两个以上顶点的连通图，因此答案必定是不可能的。 另一个极端情况是只有两个顶点的度数大于 1 时，因为这样图就会被迫形成分支有限的树状结构，从而严格控制直径的增长。 简单地贪婪地构建任何生成树的简单方法可以轻松生成看起来正确的图，但不一定具有最大直径。 

## 方法

 一个蛮力的想法是尝试所有可能的连通图并检查度数约束是否成立并计算直径。 即使将我们限制在树上也已经给出了指数数量的结构，因为有$n^{n-2}$标记的树木。 每个的计算直径都是线性的，所以即使对于非常小的直径也是完全不可行的$n$。 这种爆炸的核心原因是，除非我们从结构上解释它们，否则约束不会强烈限制拓扑。 

关键的观察结果是，当图尽可能“像路径”时，连通图的直径最大化。 任何分支都会缩短距离，因为它在遥远的区域之间创建了捷径。 然而，只要度数允许，我们就可以添加额外的边，并且这些边可以潜在地增加可达性，而不必减少最长的最短路径。 

这提出了一种策略：从尽可能长的主干路径开始，因为简单的路径为树提供了最大可能的直径。 然后，使用高度顶点以不减小直径的方式附加附加边。 限制条件$a_i$确定每个顶点仍然可以接受多少条边，因此具有较高容量的顶点成为潜在的分支点，但必须谨慎使用分支，因为它有引入捷径的风险。 

更精确的重新表述是首先考虑构建一棵树，因为添加额外的边不能增加树状环境中的直径，通常只能通过创建快捷方式来减少直径。 因此，我们想要一棵在度数限制下最大化直径的树。 最优的此类树始终是“双端贪婪路径构造”：我们首先使用可用的度数为 1 的容量构建一条长路径，同时保持更高度数的顶点可用于扩展末端。 

我们重复地将具有可用度数容量的顶点附加到当前路径的末端。 具有较高度数限制的顶点可以在内部使用或作为分支点，但是当我们最小化分支并最大化链长度时，可以实现最长的直径。 如果在任何时候我们无法进一步扩展路径，因为所有剩余顶点都已耗尽或剩余度数容量不足以附加到端点，我们就会停止。 

如果图在约束下甚至无法连接，这意味着少于两个顶点可以参与边，我们立即返回 NO。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解图表 | 指数| O(n^2) | O(n^2) | 太慢了|
 | 使用度数能力构建贪婪路径 | O(n log n) 或 O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们将每个顶点解释为具有等于其度数限制的初始“预算”。 我们添加的每条边都会消耗两个端点的一个单位。 我们的目标是构建一条尽可能长的单一路径，因为这会直接最大化直径。 

1. 我们首先选择一个具有正度容量的起始顶点。 该顶点将成为我们最终最长路径的一个端点。 选择任何有效的顶点都是有效的，因为目标不是唯一性而是最大扩展。 
2. 我们维护一条由其端点表示的当前路径。 最初，路径只是一个顶点，因此两个端点重合。 
3. 我们尝试通过将新顶点附加到仍具有剩余度数容量的一个端点来扩展路径。 我们总是选择一个尚未使用且仍有剩余容量的顶点。 原因是重用顶点会产生循环或捷径，这只能减少或无法增加直径。 
4. 每次我们将一个顶点附加到一个端点时，我们都会减少两个顶点的度数容量，并将端点更新为新添加的顶点。 这贪婪地增长了一条简单的链。 
5. 我们继续此过程，直到没有具有剩余容量的未使用顶点可以附加到任一端点。 此时，所构建的结构就是约束下的最大路径。 
6. 如果我们已经使用了所有顶点或者在保持连通性的同时无法进一步扩展，我们将检查结果结构是否连通且有效。 如果不是所有顶点都包含在单个连接组件中，我们输出 NO。 
7. 否则，所构造图的直径只是该路径中的边数，因为路径图的直径等于其长度。 

它之所以有效的根源在于这样一个事实：任何额外的分支边要么不会增加最长的最短路径，要么会创建减少最长最短路径的捷径。 路径是一种独特的结构，可以使同一顶点集上所有连接的简单图的端点之间的距离最大化。 在度数限制下，可实现的最长路径正是我们所希望的最大直径，因为路径结构的任何偏差都会减少某些端点对之间的距离，或者无法进一步延长链。 贪婪扩展确保我们始终保留单个简单路径不变式：每个顶点在构造的主干中最多具有二度，并且端点是唯一符合扩展条件的顶点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    # each vertex has remaining degree capacity
    deg = a[:]
    
    # store unused vertices
    unused = set(range(n))
    
    edges = []
    
    # pick a starting node with positive degree
    start = max(range(n), key=lambda i: deg[i])
    if deg[start] == 0:
        print("NO")
        return
    
    unused.remove(start)
    left = right = start
    
    # greedy path extension
    while True:
        extended = False
        
        # try extend from left
        for v in list(unused):
            if deg[v] > 0 and deg[left] > 0:
                edges.append((left, v))
                deg[left] -= 1
                deg[v] -= 1
                unused.remove(v)
                left = v
                extended = True
                break
        
        if extended:
            continue
        
        # try extend from right
        for v in list(unused):
            if deg[v] > 0 and deg[right] > 0:
                edges.append((right, v))
                deg[right] -= 1
                deg[v] -= 1
                unused.remove(v)
                right = v
                extended = True
                break
        
        if not extended:
            break
    
    if unused:
        print("NO")
        return
    
    diameter = len(edges)
    print("YES", diameter)
    print(len(edges))
    for u, v in edges:
        print(u + 1, v + 1)

if __name__ == "__main__":
    solve()
```该解决方案保留一组未使用的顶点，并从两个端点生成一条路径。 每条边都会消耗两个端点的度数容量，确保永远不会违反约束。 对未使用的顶点进行贪婪扫描是可以接受的，因为$n \le 500$，所以即使是一个$O(n^2)$尝试查找扩展名的速度足够快。 

一个微妙的实现细节是我们总是迭代未使用集的快照。 这避免了迭代期间的修改。 另一个重要的一点是，我们只从端点延伸，这保留了图始终保持简单路径的不变性。 

## 工作示例

 ### 示例 1

 输入：```
3
2 2 2
```我们从具有足够容量来支持路径的所有顶点开始。 假设首先选择顶点 1。 

| 步骤| 左| 对| 未使用 | 边缘已添加 |
 | --- | --- | --- | --- | --- |
 | 1 | 1 | 1 | {2,3} | 无 |
 | 2 | 2 | 1 | {3} | (1,2) |
 | 3 | 2 | 3 | {} | (2,3) |

 生成的结构是长度为 2 的单个路径。直径为 2，因为最远的顶点是 1 和 3。 

### 示例 2

 输入：```
4
1 4 1 1
```这里顶点 2 的容量很大，而其他顶点的容量有限。 

| 步骤| 左| 对| 未使用 | 边缘已添加 |
 | --- | --- | --- | --- | --- |
 | 1 | 2 | 2 | {1,3,4} | 无 |
 | 2 | 1 | 2 | {3,4} | (2,1) |
 | 3 | 1 | 3 | {4} | (1,3) |
 | 4 | 1 | 4 | {} | (1,4) |

 我们以路径 2-1-3-4 结束，直径为 3。顶点 2 仅在构造的一端充当中心，保留最大端点距离。 

这些痕迹表明，该算法始终保留单个主干路径，并且从不引入会减少端点距离的捷径。

 ## 边缘情况

 当只有一个顶点具有正度容量时，就会出现一种边缘情况。 例如，如果输入是`3 0 0`，无法形成边并且无法连接。 当算法无法选择有效的起始顶点或无法从端点延伸时，它会立即检测到这一点。 

另一种情况是当所有顶点的度数均为 1 时。构造仍然成功并生成一条路径，因为每个顶点都可以用作内部节点或端点一次。 贪婪扩展确保没有顶点被闲置。 

当高度顶点存在但与构建过程隔离时，会出现最后一种情况。 由于我们总是全局扫描未使用的顶点，因此任何具有可用容量的顶点如果可以参与有效的边，则最终将附加到端点之一，从而防止碎片并确保尽可能形成单个连接的组件。
