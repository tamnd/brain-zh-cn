 ## 解决方案
 ## 问题理解

 给我们一本书，页面从 1 到 n 排列成一行。 Vasya 从第 x 页开始，想要到达第 y 页。 每次按下按钮都会将当前页面向前移动 d 页或向后移动 d 页，但不允许任何离开间隔 [1, n] 的移动。 

任务是计算准确到达第 y 页所需的最少按钮按下次数，或者确定这是不可能的。 

虽然这看起来像是折线图上的最短路径问题，但结构非常严格：从任何页面 i 开始，唯一可能的下一个状态是 i + d 和 i − d（如果它们保持在边界内）。 这意味着图是未加权的并且在转换中是确定性的，但是数轴足够大，当 n 达到 10^9 时，强力遍历是不可行的。 

关键的观察是，我们实际上不需要探索所有页面，只需要探索步长 d 下 x 和 y 之间的算术关系。 

从概念上讲，基于页面的简单 BFS 是可行的，但当 n 很大时它会立即失败。 即使我们限制探索，在最坏的情况下，可达集仍然可能很大。 

当由于算术约束而不是边界而无法移动时，就会出现打破粗心推理的边缘情况。 例如，如果 d = 2，则移动会保留奇偶校验，因此无论 n 如何，都不可能达到不同奇偶校验的目标。 

当最佳路径需要首先走出直接方向时，就会发生另一种微妙的情况，因为边界迫使绕道而行。 例如，如果 x 接近 1 并且 y 较大，则向后步进是无用或不可能的，而天真的贪婪向前步进可能会低估或高估步数。 

## 方法

 强力解决方案将每个页面建模为图中的一个节点，并从 x 运行 BFS 直到到达 y。 每个节点最多有两条边：i + d 和 i − d（如果有效）。 BFS 保证最短路径，因为所有边的成本都是 1。 

然而，这种方法在最坏的情况下探索 O(n) 状态，因为可达组件可以跨越区间的很大一部分。 当 n 达到 10^9 时，即使触及所有节点也是不可能的。 

关键的见解是运动是线性的并且受到算术结构的约束。 每次移动都会改变位置 ±d，因此任何可到达的页面必须满足模条件：所有可到达的位置都与 x 模 d 一致。 这立即给出了一个可行性条件：y 必须满足 (y − x) % d == 0。 

如果这个条件成立，问题就简化为确定需要多少步骤，同时尊重边界。 最佳路径要么直接朝一个方向移动，要么根据需要从边界 1 或 n 反弹。 由于该图是一条具有固定步长的简单直线，因此我们最多只需要考虑两种单调策略：从 x 直接到 y，或者通过一个边界然后到 y。 

这将问题减少到少量恒定数量的算术检查，而不是图形搜索。

 ## 算法演练

 我们独立解决每个测试用例。

1. 首先，检查 y 是否可以从 x 到达，步长为 d。 这是通过验证 x 和 y 之间的差能否被 d 整除来完成的。 如果不是，则不存在可以精确落在 y 上的 ±d 移动序列，因此我们返回 -1。 
2. 如果可达，考虑 x 和 y 之间的直接距离，单位为 d。 这给出了基准步数为 |x − y| /d。 
3. 然而，如果需要走出[1, n]，直接移动可能不可行。 我们必须考虑直线前进是否在每一步都保持在界限内。 
4. 如果直接步进有效，则它是最佳的，因为每次移动都会将距离恰好减少 d。 
5. 如果直接步进被边界阻挡，唯一的选择是在端点处反射。 这引入了通过 1 或 n 的绕行，有效地将路径转换为 ​​x → 1 → y 或 x → n → y，具体取决于有效步骤可到达的边界。 
6. 在有效时计算两条基于边界的路由并取最小值。 

### 为什么它有效

 每次移动都会保留对 d 进行模的残差类，因此状态空间会分裂成独立的算术级数。 在一个级数中，图变成一条可能存在边界截断的线性链。 任何最短路径都必须是单调的，除了边界处可能出现的一次逆转之外，因为额外的振荡会严格增加路径长度，而不会扩大可达性。 这减少了所有最短路径的直接进展或端点处的单个反射。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, x, y, d = map(int, input().split())

        # If direct reach is possible
        if abs(x - y) % d == 0:
            direct = abs(x - y) // d
        else:
            direct = float('inf')

        best = direct

        # via 1
        if (y - 1) % d == 0 and (x - 1) % d == 0:
            via1 = (x - 1) // d + (y - 1) // d
            best = min(best, via1)

        # via n
        if (n - x) % d == 0 and (n - y) % d == 0:
            vian = (n - x) // d + (n - y) // d
            best = min(best, vian)

        print(best if best != float('inf') else -1)

if __name__ == "__main__":
    solve()
```该实现首先使用模 d 检查算术可达性。 直接转换就是绝对差除以 d。 

这两个边界情况对应于以 d 为步长从 x 走到 1 或 n，这只有在两个端点位于相同算术级数中时才有可能。 成本只是此类步骤的数量，因为路径是强制的。 

我们在所有有效策略中采取最小值。 

必须小心整数除法，因为一旦可行性检查通过，所有有效距离都保证可被 d 整除。

 ## 边缘情况

 一个微不足道但重要的情况是 x 等于 y。 该算法正确返回 0，因为直接距离为零且可被任意 d 整除，因此不需要移动。 

另一种情况是当 d 大于 n 时。 在此，除非 x 等于 y，否则不可能进行移动。 模条件正确地消除了除恒等情况之外的所有转换。 

当 x 接近 1 或 n 时，就会出现边界主导的情况，其中最初只有一个方向可行。 该算法通过仅考虑到端点的算术有效路径来处理此问题，确保不假设任何无效的中间步骤。 

最后一种微妙的情况是存在多条有效路线，一条直接，一条通过边界。 取最小值可确保最优性，而无需显式探索路径，因为所有有效路径都减少为线性算术成本。
