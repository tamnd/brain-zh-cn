 ## 解决方案
 ## 问题理解

 我们得到了一组电话号码，每个号码都与波利卡普拨打该号码的频率相关。 每次拨打号码时，他都必须按数字，但电话最多允许$k$特殊的快速拨号按钮。 每个按钮都可以分配一个完整的电话号码，使用时，他可以输入该号码，而无需输入任何数字。 按快速拨号按钮后，不涉及部分打字，因此该呼叫的成本为零数字按键。 

如果号码未分配给快速拨号按钮，则每次呼叫都需要手动键入该号码的所有数字。 目标是分配最多$k$以尽量减少所有呼叫中按数字的总数的方式向这些按钮添加号码，并根据每个号码的使用频率进行加权。 

因此，该结构是字符串子集的选择问题：每个字符串都有一个权重（其频率）和成本（其长度），并将其分配给按钮将其贡献减少到零。 

约束以非常具体的方式严格。 最多有 500 个数字，所有字符串的总长度也最多为 500。按钮的数量最多为 10 个。这立即排除了对所有数字进行任何指数子集选择。 甚至$2^{500}$子集是不可能的，甚至$O(n^k)$太大了，没有结构。 较小的总长度表明我们应该将思维从“每个字符串”决策转变为“每个前缀结构”，因为字符串共同形成一个紧凑的特里树。 

一个微妙的边缘情况来自重叠的前缀。 两个数字就像`001`和`0012`相互作用：分配一个可能会影响分配另一个的边际价值。 总是选择每个字符串中最高节省的天真的贪婪方法会失败，因为节省不是独立的。 另一种故障模式是独立处理字符串并选择顶部$k$经过$m_i \cdot |s_i|$，它忽略共享前缀和未来的分支结构。 

关键的困难在于分配字符串的好处取决于扩展它的所有其他字符串，这意味着决策空间是分层的而不是平坦的。 

## 方法

 直接方法会尝试所有方法来获取$k$分配给快速拨号的号码。 对于每个子集，我们通过求和来计算总成本$m_i \cdot |s_i|$对于未分配的字符串。 这是正确的但不可行，因为它需要评估$\sum_{i=0}^k \binom{n}{i}$，当$n=500$和$k=10$。 

结构上的突破来自于认识到字符串构成特里树。 如果我们查看一个前缀，将该前缀分配为快速拨号条目可以有效地消除其子树中所有号码的成本。 但是，我们不允许分配任意前缀； 我们只能分配实际的电话号码。 这仍然导致树形依赖结构：每个节点（字符串）可以“覆盖”自身和所有后代。 

这表明了特里树上的动态编程公式：我们决定在每个子树内分配多少个快速拨号槽，并计算可实现的最佳节省。 

在每个节点，我们考虑在其子节点以及可能的节点本身之间分配有限数量的“免费快捷方式”。 分配节点的贡献是总加权节省等于其字符串成本乘以频率，但前提是我们选择它作为快捷方式。 

因此，我们将问题转换为树DP，其中每个节点返回最多使用可实现的最佳节省$t$其子树中的快捷方式。 合并孩子变成了一个背包式的卷积，但是自从$k \le 10$，这仍然是可行的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解字符串子集 |$O(\binom{n}{k} \cdot n)$|$O(n)$| 太慢了|
 | 背着背包的特里DP$k \le 10$|$O(N \cdot k^2)$|$O(Nk)$| 已接受 |

 ## 算法演练

 我们首先对所有电话号码建立一个字典树。 每个节点存储一个完整数字在那里结束的次数以及该节点对应的字符串长度。 

1. 从所有字符串构造一个字典树。 每个节点代表一个前缀，终端节点代表完整的电话号码。 储存频率$m_i$在终端节点。 
2. 对于每个节点，计算其基本成本贡献，即如果我们不对该节点使用任何快捷方式，则按下的数字总数。 这是频率乘以深度（弦长度）。 
3、在每个节点定义一个DP函数：`dp[u][t]`表示我们在以 为根的子树中可以实现的最大节省`u`最多使用`t`快速拨号分配。 
4. 初始化`dp[u][0] = 0`因为使用零捷径不会带来任何节省。 
5.如果节点`u`对应于一个完整的数字，我们考虑在这里使用一个快捷方式。 分配它所产生的节省等于其全部成本贡献。 我们更新`dp[u][1]`因此。 
6. 使用背包卷积合并子DP表。 对于每个子树，我们结合当前子树和子子树之间的快捷方式计数的分布，更新最佳节省。 
7.处理完所有节点后，答案是总基础成本减去`dp[root][k]`。 

关键的不变量是`dp[u][t]`始终代表子树中所有数字的打字成本可实现的最佳降低`u`最多使用`t`指定的快捷方式。 由于子树除了通过 trie 结构之外都是不相交的，因此通过背包组合子树可以保持决策的独立性。 

没有快捷方式会被计算两次，因为每次分配都对应于在 trie 中选择一个节点，并且子树合并确保有限资源的不相交分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

class Node:
    __slots__ = ("ch", "end", "freq")
    def __init__(self):
        self.ch = [-1] * 10
        self.end = 0
        self.freq = 0

def add(root, s, m, nodes):
    v = root
    for c in s:
        d = ord(c) - 48
        if nodes[v].ch[d] == -1:
            nodes[v].ch[d] = len(nodes)
            nodes.append(Node())
        v = nodes[v].ch[d]
    nodes[v].end += m
    nodes[v].freq += m

def dfs(u, k, nodes, depth):
    dp = [0] + [-10**18] * k
    base_cost = nodes[u].end * depth

    for d in range(10):
        v = nodes[u].ch[d]
        if v == -1:
            continue
        child_dp, child_cost = dfs(v, k, nodes, depth + 1)

        new_dp = [-10**18] * (k + 1)
        for i in range(k + 1):
            for j in range(k + 1 - i):
                new_dp[i + j] = max(new_dp[i + j], dp[i] + child_dp[j])
        dp = new_dp

        base_cost += child_cost

    best = dp[:]
    if nodes[u].end > 0:
        gain = nodes[u].end * depth
        for i in range(k, 0, -1):
            best[i] = max(best[i], dp[i - 1] + gain)

    return best, base_cost

def solve():
    n, k = map(int, input().split())
    nodes = [Node()]
    total_cost = 0

    for _ in range(n):
        s, m = input().split()
        m = int(m)
        add(0, s, m, nodes)
        total_cost += len(s) * m

    dp, _ = dfs(0, k, nodes, 0)
    print(total_cost - max(dp))

if __name__ == "__main__":
    solve()
```trie 结构对所有字符串进行紧凑编码，因此共享前缀仅存储一次。 DFS 计算每个节点的节省如何向上传播。 

DP阵列以背包方式小心地合并。 内部双循环确保我们分配最多$k$孩子之间的捷径没有重叠。 

在合并子节点之后，可以选择将节点作为快捷方式包含在内，以确保我们仅在知道下面的所有结构贡献后才将快捷方式分配给节点。 

## 工作示例

 考虑示例输入：```
3 1
0001 5
001 4
01 1
```trie 结构将所有字符串放置在单个前缀链上。 

| 节点| 深度| 频率 | 基本成本| dp[0] | dp[0] | dp[1] | dp[1] |
 | --- | --- | --- | --- | --- | --- |
 | 根 | 0 | 0 | 0 | 0 | 5 |
 | 0 | 1 | 0 | 0 | 0 | 5 |
 | 00 | 00 2 | 0 | 0 | 0 | 5 |
 | 000 | 000 3 | 0 | 0 | 0 | 5 |
 | 0001| 4 | 5 | 20 | 0 | 20 |

 最优分配是选择`0001`，为该节点产生完全节省。 剩余的字符串未分配，因此它们的成本仍然存在。 

现在考虑：```
2 1
12 3
123 2
```| 节点| 深度| 基本成本| dp[0] | dp[0] | dp[1] | dp[1] |
 | --- | --- | --- | --- | --- |
 | 12 | 12 2 | 6 | 0 | 6 |
 | 123 | 123 3 | 6 | 0 | 6 |

 分配`123`严格优于分配`12`，因为它涵盖了更多的加权用法。 DP 正确地捕获了这一点，因为子树节省会不断累积。 

这些例子表明，当更深的节点占主导地位的子树时，该算法自然会优先选择更深的节点，并避免冗余分配。

 ## 边缘情况

 一种重要的情况是多个号码共享长前缀。 对于像这样的输入`0001`,`0002`， 和`0003`，为覆盖所有节点的最深节点分配快捷方式比分配中间前缀更有价值，因为节省的成本随着频率和深度而变化。 

当频率差异很大时会出现另一种边缘情况。 经常使用的短数字可能胜过较长但很少使用的更深字符串，并且 DP 必须自然地允许这种权衡，而不是强制深度优先。 

最后一个边缘情况是$k$大于终端节点数。 DP 仍然表现正确，因为未使用的时隙仅具有零边际增益，并且答案简化为分配所有可能的节点。
