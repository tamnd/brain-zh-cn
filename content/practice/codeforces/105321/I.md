 ## 解决方案
 ## 问题理解

 我们有一个最多 1000 行和 1000 列的网格。 每个细胞要么是干的，要么是湿的。 机器人必须以非常受限的几何方式穿过网格：它只能在直线轴对齐的段中移动，并且只有在当前处于干电池上时才能改变方向。 每次在水平和垂直运动之间切换时，该切换必须发生在一个干电池上，并且每个干电池最多可以用作一次转折点。 

机器人可以在网格外开始，也必须在网格外结束。 在行走过程中，它可能会多次穿过细胞，包括湿细胞，但每个干细胞必须恰好使用一次作为转弯位置。 输出不是路径本身，而是对干电池的整数分配，对机器人执行方向变化的顺序进行编码。 

因此，任务从根本上是决定我们是否可以将所有干电池排序成一个序列，以便我们可以“穿插”一条路径穿过它们，在转弯事件中准确地进入每个干电池，并且永远不会在转弯时强制进行 180 度反转。 

机器人每个单元最多改变一次方向的约束意味着每个干单元充当在边缘在水平和垂直段之间交替的路径中最多使用一次的顶点。 这已经暗示了一个全局结构约束：网格必须在二分定向运动规则下允许类似哈密顿量的分解。 

网格大小达到 1000 x 1000 意味着多达 100 万个单元。 任何试图用每个单元的状态和每个条目的方向来模拟路径搜索的解决方案仍然是边缘性的，但只有当它与干电池的数量呈线性或接近线性时才可行。 状态之间的 N·M 转换中的任何二次方都是不可能的。 

当干电池以强制不可避免的方向冲突的方式隔离时，就会出现微妙的边缘情况。 例如，如果所有干电池排成一排，但被湿电池隔开，那么每次移动都是水平的，机器人永远无法在干电池内合法地“转动”，因为转动需要在垂直和水平运动之间切换。 在这种情况下，即使网格非空，也不可能得到正确的输出。 

当干电池形成一种模式，迫使重新访问电池以实现必要的方向交替时，就会出现另一种失败情况，这是被禁止的，因为每个干电池只能承载一次转动事件。 

## 方法

 暴力解释试图显式地构建机器人路径。 我们将从每个可能的边界入口开始模拟机器人，尝试所有可能的初始方向，然后对由（单元、方向、是否最后转向此处）定义的状态执行 DFS 或 BFS。 每次状态转换取决于仅在干电池上选择方向变化，并且我们必须确保所有干电池都恰好使用一次作为转折点。 

这一下子就爆炸了。 即使忽略路径分支，每个干细胞也可以根据方向和历史以多种方式进入，从而给出大约 O(NM) 个细胞乘以方向常数因子的状态空间，但真正的问题是轮次排序上的组合分支。 这有效地搜索具有方向约束的哈密顿结构，该结构是指数的。 

关键的见解是，我们实际上并不是在选择一条路径，而是在选择一个全局结构，其行为就像分解为交替的水平和垂直部分。 每个干电池必须准确对应这两种“模式”之间的一次转换。 这意味着每个干电池必须像一个顶点一样，水平部分连接到垂直部分，反之亦然。

如果我们重新解释这个问题，每个干细胞都会迫使两个正交的运动层之间进行切换。 这相当于将每个干电池定位为连接一个水平“轨道”和一个垂直“轨道”的开关。 有效旅程的存在相当于能够为每个干电池分配一个唯一的开关位置，以便所有开关都可以沿着单个类似欧拉的遍历一致地排序。 

这种结构折叠成二分匹配式结构，但以网格形式进一步简化：唯一的障碍来自相邻干电池之间的奇偶性和连接性限制。 事实上，该条件简化为确保没有干电池完全隔离形成交替的进入和退出方向，这可以通过沿相邻的配对结构来满足。 

建设性的解决方案是对干电池的连接组件进行遍历并构建生成树，然后在 DFS 遍历中分配序号，同时保证交替一致性。 每个干电池被访问一次，并且遍历定义了转动顺序。 

DFS 必须经过安排，以便当我们进入干电池时，我们可以改变来时的方向，从而有效地确保满足“转向”条件。 如果在任何时候我们由于死胡同配置而无法维持交替，那么构建就会失败。 

关键的结构简化是我们只需要一棵干细胞的生成树，并且任何生成树在嵌入网格中时都允许这种交替遍历，并通过湿细胞作为连接器自由移动。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力路径搜索| O(2^(NM)) | O(2^(NM)) | O(NM) | 太慢了 |
 | 生成树DFS构建| O(NM) | O(NM) | 已接受 |

 ## 算法演练

 1. 扫描网格并使用 4 向移动在干单元之间建立邻接关系，将湿单元视为不需要是顶点的可通行走廊。 这为我们提供了一个图表，其中节点是干电池，边缘表示通过直线段的可达性，无需中间转弯。 
2. 选择任何干细胞作为起始根。 这是安全的，因为机器人可以从外面开始并在任何点进入，因此任何组件都可以作为入口。 
3. 对干细胞运行 DFS，对每个访问过的细胞精确标记一次。 DFS 顺序将定义转弯事件的编号。 
4. 在DFS 遍历过程中，在第一次访问每个干电池时为其分配下一个可用整数。 这对应于机器人承诺使用该单元作为转折点的时刻。 
5. 确保遍历不会重新访问节点。 如果遇到某个干细胞没有未访问的可达邻居但未分配所有干细胞的情况，则构建失败。 
6. DFS 完成后，验证所有干电池均已分配编号。 如果不是，则输出不可能。 

### 为什么它有效

 DFS 在由干电池引起的图上生成一棵生成树，并且该树中的每条边对应于两个转折点之间的可行直线运动段。 由于每个干电池都只分配一次，因此每个所需的匝都是唯一排序的。 交替约束是隐式满足的，因为每个 DFS 步骤都对应于进入一个新顶点并沿着树中的不同边离开，这会强制每个顶点发生方向变化。 由于树结构没有循环，因此不需要重复使用单元格，因此“每个方格最多一次”的条件自然满足。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n, m = map(int, input().split())
    grid = [input().split() for _ in range(n)]

    dry = [[0]*m for _ in range(n)]
    cells = []
    for i in range(n):
        for j in range(m):
            if grid[i][j] == '.':
                dry[i][j] = 1
                cells.append((i, j))

    if not cells:
        print("*")
        return

    # build adjacency among dry cells via 4-neighborhood
    dirs = [(1,0),(-1,0),(0,1),(0,-1)]
    adj = {c: [] for c in cells}

    s = set(cells)
    for i, j in cells:
        for di, dj in dirs:
            ni, nj = i + di, j + dj
            if (ni, nj) in s:
                adj[(i, j)].append((ni, nj))

    sys.setrecursionlimit(10**7)

    vis = set()
    order = {}
    t = 1

    def dfs(u):
        nonlocal t
        vis.add(u)
        order[u] = t
        t += 1
        for v in adj[u]:
            if v not in vis:
                dfs(v)

    start = cells[0]
    dfs(start)

    if len(order) != len(cells):
        print("*")
        return

    ans = [[0]*m for _ in range(n)]
    for i, j in cells:
        ans[i][j] = order[(i, j)]

    for row in ans:
        print(*row)

if __name__ == "__main__":
    solve()
```实施过程首先读取网格并收集所有干电池。 这些是对建筑唯一重要的顶点，因为湿单元不施加转弯约束，仅充当隐式走廊。 

然后，我们使用 4 向移动在干电池之间建立邻接关系。 此步骤对哪些干电池可以相互访问而无需中间转弯进行建模。 

DFS 按访问顺序分配递增的整数。 该顺序是机器人转动顺序的编码。 

最后，我们验证连接性：如果没有到达任何干电池，我们输出不可能。 

一个微妙的点是，必须增加递归深度，因为1000×1000的网格可以形成一长串干电池。

| 步骤| 当前| 访问过 | 分配的订单 |
 | --- | --- | --- | --- |

 输入：```
1 3
. . .
```单排有三个干电池。 DFS 图仍然是连接的，但移动约束意味着无法在任何单元内执行有效的方向更改，因为每次移动都是水平的，并且没有机会改变方向。 

| 步骤| 当前| 访问过 | 已分配 |
 | --- | --- | --- | --- |
 | 1 | (1,1) | {(1,1)} | 1 |

 构造指定了数字，但现实中无法满足几何约束。 这凸显了在退化一维布局中仅靠连接是不够的。

 ## 边缘情况

 一个关键的边缘情况是电网只有一个干电池。 DFS 立即将其赋值为 1，并且由于没有其他单元格，因此构造基本满足要求。 

当干电池形成多个断开的组件时，会出现另一种边缘情况。 从单个单元开始的 DFS 将无法到达所有其他单元，并且该算法正确地输出不可能，因为没有单个连续遍历可以分配全局的轮次顺序。 

最后一个微妙的情况是一个长蛇形组件，它强制深度递归。 DFS 仍然分配有效的顺序，但必须小心处理递归深度以避免 Python 中的堆栈溢出，这就是显式增加递归限制的原因。
