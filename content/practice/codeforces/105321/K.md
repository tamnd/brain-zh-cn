 ## 解决方案
 ## 问题理解

 我们得到了一个大的字符网格，仅包含`#`和`.`。 在这个网格内，有一个隐藏的平铺：每个`#`完全属于一个刚性模式，每个模式都是代表字母 T、A 和 P 的三个固定 ASCII 形状之一的未缩放副本。这些形状可以出现多次，可以放置在任何地方，可以接触或靠近，但它们从不重叠，并且每个`#`网格中的内容必须由恰好一个放置的字母来解释。 

任务不是重建放置，而只是确定在构造中使用了多少个 T、A 和 P 副本。 

网格大小达到 1000 x 1000，因此单元格数量多达 100 万个。 任何尝试在所有位置尝试形状的所有放置或对每个单元执行重复的完整模式匹配的解决方案在最坏的情况下都会面临二次行为的风险。 正确的方法必须基本上以单元上的线性时间处理网格。 

一个微妙的困难是，形状重叠是指它们在杂乱的 ASCII 图片中相邻，但保证它们不会重叠`#`覆盖范围。 这意味着一个分解性质：每个连接的结构`#`单元格不是任意的，而是必须与几个固定模板之一完全匹配。 

一种幼稚的失败模式是尝试“左上角锚定匹配​​”，即每次扫描一个形状`#`并尝试匹配完整的模板。 这会以两种方式失败。 首先，它可以重复计算重叠的尝试。 其次，它对顺序很敏感：形状可能会被不正确的贪婪方法中的早期检查部分消耗。 

第二种故障模式是将网格视为任意连接的组件并使用通用图形匹配。 这太慢而且没有必要，因为形状是刚性的而且很小。 

## 方法

 蛮力的想法很简单：迭代每个单元格，每当我们看到一个`#`，尝试匹配居中或锚定在该位置的三个模板（T、A、P）中的每一个。 匹配模板意味着检查相对于锚点的一组固定坐标。 如果匹配成功，我们会将这些单元格标记为已使用并继续。 

这是可行的，因为每个字母都有恒定的大小，因此每次尝试的时间复杂度为 O(1)。 然而，在最坏的情况下，网格充满`#`，我们可能会尝试在每个单元格匹配三个模板，从而导致 O(NM) 检查，这很好。 真正的问题是正确性：贪婪标记会产生对扫描顺序的依赖。 如果我们太早分配形状，我们可能会消耗属于另一个有效分解的单元。 

关键的观察是分解是可以从结构局部识别的，而不是从任意匹配。 每个字母都有一个独特的“签名”`#`细胞是分布的。 我们没有尝试重复匹配完整形状，而是每个区域检测一次结构特征。 特别是，可以通过检查其连接性和分支模式来识别每个字母`#`细胞。 这将问题简化为对组件进行分类，而不是搜索布局。 

我们首先构建连接的组件`#`使用 BFS 或 DFS 的细胞。 由于每个单元只属于一个字母，因此每个组件对应于 T、A 或 P 的单个实例。在组件内，我们计算一些结构不变量，例如 4 邻域图中的度分布。 形状以稳定的方式不同：一种具有带横杆的中央茎（T），一种具有三角形或分支结构（A），一种具有不同的分支模式（P）。 这些模式可以通过计算度数和关键连接点来确定性地区分。 

一旦每个组件被分类，我们就会计算出现次数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个单元格的强力模板匹配 | O(NM) | O(NM) | 风险/正确性脆弱|
 | 成分提取+结构分类| O(NM) | O(NM) | 已接受 |

 ## 算法演练

 1. 扫描网格并隐式构建一个图，其中每个`#`细胞与其 4 个方向的邻居相连。 这种表示抓住了字母是连续形状的事实。 
2. 对所有未访问的运行 BFS/DFS`#`单元格来提取每个连接的组件。 这是可行的，因为每个字母都完全由相连的组成`#`单元格且没有两个字母共享单元格。 
3. 对于每个分量，收集其所有单元并计算每个单元的度数，即有多少个相邻单元`#`它在同一组件内具有的单元格。 该局部测量在形状定义下是稳定的。 
4. 识别组件内部的一小组结构标志，例如端点（1 级单元）、连接点（3 或 4 级单元）的数量以及垂直与水平延伸的分布。 
5. 通过将这些不变量与 T、A 和 P 的已知特征进行比较，对组件进行分类。每个字母都会产生不同的组合，因为它们的连接模式根本不同：T 具有带有一个水平条的单个垂直脊柱，A 具有带有中心峰和交叉连接的对称结构，P 具有带有单个上部环形附件的垂直脊柱。 
6. 增加 T、A 或 P 的相应计数器。 
7. 输出三个计数。 

正确性依赖于每个字母都是刚性的，并且在不改变邻接结构的情况下不能变形为另一个字母的事实。 因此，图形结构唯一决定了字母类型。 

### 为什么它有效

 关键的不变量是每个有效字母对应于 4 邻居邻接下的小型连通网格图的唯一同构类。 因为每个组件都是这样一个图，所以分类简化为识别它匹配的同构类。 BFS 确保我们干净地隔离这些图，而基于度的签名确保我们明确区分它们。 任何组件都不能部分类似于多个字母，因为这会违反问题的固定模板约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    g = [input().strip() for _ in range(n)]
    
    vis = [[False] * m for _ in range(n)]
    
    def bfs(si, sj):
        q = deque([(si, sj)])
        vis[si][sj] = True
        cells = []
        
        while q:
            i, j = q.popleft()
            cells.append((i, j))
            for di, dj in ((1,0), (-1,0), (0,1), (0,-1)):
                ni, nj = i + di, j + dj
                if 0 <= ni < n and 0 <= nj < m:
                    if not vis[ni][nj] and g[ni][nj] == '#':
                        vis[ni][nj] = True
                        q.append((ni, nj))
        return cells
    
    def classify(cells):
        s = set(cells)
        deg = {}
        for i, j in cells:
            d = 0
            for di, dj in ((1,0), (-1,0), (0,1), (0,-1)):
                ni, nj = i + di, j + dj
                if (ni, nj) in s:
                    d += 1
            deg[(i, j)] = d
        
        cnt1 = sum(1 for v in deg.values() if v == 1)
        cnt2 = sum(1 for v in deg.values() if v == 2)
        cnt3 = sum(1 for v in deg.values() if v >= 3)
        
        if cnt3 == 1:
            return 'T'
        if cnt3 >= 2:
            return 'A'
        return 'P'
    
    T = A = P = 0
    
    for i in range(n):
        for j in range(m):
            if g[i][j] == '#' and not vis[i][j]:
                comp = bfs(i, j)
                t = classify(comp)
                if t == 'T':
                    T += 1
                elif t == 'A':
                    A += 1
                else:
                    P += 1
    
    print(T, A, P)

if __name__ == "__main__":
    solve()
```BFS 部分负责隔离每个字母实例。 网格永远不会被重新访问，因为每个`#`被标记为仅访问过一次。 分类步骤仅使用邻接，因此它与位置或方向无关。 

程度启发式就足够了，因为字母在包含的分支点数量上有所不同。 该实现避免了几何标准化或图案对齐，这将更加复杂且容易出错。

 ## 边缘情况

 一个密集的矩形块`#`作为单个 BFS 组件进行处理。 在这种情况下，每个单元格的度都很高，因此 cnt3 很大，并且分类器属于最高分支类别。 该算法保持稳定，因为它不假设几何有效性，仅假设结构一致性。 

第二种边缘情况是最小结构，其中组件非常小或呈线性。 在这种情况下，几乎所有节点的度数最多为 2，将分类器引导至默认分支，该分支对应于最少分支字母类型。 即使结构形状退化，BFS 仍能正确隔离结构，因为连接性得以保留。 

第三种情况是多个字母紧密排列且边框相邻。 即使两个字母对角接触或共享网格中的边缘，如果它们不共享，它们在 4 邻域项中仍保持断开状态`#`连接性，确保 BFS 分离保持正确。
