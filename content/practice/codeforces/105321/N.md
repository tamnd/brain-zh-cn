 ## 问题理解

 我们给出了可能的正整数长度的列表。 从这个列表中，我们必须选择三个值，允许重复，并将它们解释为空心矩形框的三个维度。 如果选择的尺寸是$a$,$b$， 和$c$，盒子的售价定义为$a^2 + b^2 + c^2$，而制造成本为$ab + bc + ca$。 单箱的利润就是这两个表达式的差值，目标是选择$a$,$b$， 和$c$从列表中最大化该利润。 

所以任务减少到最大化$$(a^2 + b^2 + c^2) - (ab + bc + ca)$$超过所有三元组$a, b, c$从给定的集合中获取。 

约束条件$N \le 5000$值高达$10^6$立即排除天真$O(N^3)$所有三元组的枚举，这将涉及以下顺序：$125$上限为 10 亿个组合。 甚至$O(N^2)$方法需要仔细的结构才能及时适应。 

一个微妙的点是允许重复。 这很重要，因为最佳解决方案可能涉及多次使用相同的值，例如$a = b = c$，它简化了表达并且可以主导混合选择。 

第二个不明显的边缘情况是最佳解决方案来自高度不平衡的值。 由于该表达式混合了平方项和交叉项，因此我们是否更喜欢大分布值还是均匀值并不能立即明显看出。 诸如“取最大的三个值”或“取极端值”之类的天真的贪婪直觉可能会失败，具体取决于两者之间的权衡$a^2$成长和$ab$处罚。 

## 方法

 暴力解决方案尝试每一个三元组$(a, b, c)$从列表中直接计算利润。 这是正确的，因为约束没有定义除集合中的成员资格之外的其他结构。 然而，它需要$O(N^3)$评价，即关于$1.25 \times 10^{11}$操作时$N = 5000$，远远超出任何合理的时间限制。 

我们现在重写目标函数以公开结构：$$a^2 + b^2 + c^2 - ab - bc - ca$$该表达式在所有变量中都是对称的，这建议对数组进行排序并推理极端配置。 一个关键的观察是对于任何固定对$(b, c)$，该表达式是二次方$a$:$$a^2 - a(b + c) + (b^2 + c^2 - bc)$$系数为$a^2$是正数，所以对于固定$b, c$，函数在$a$是凸的。 这意味着最优$a$必须发生在允许集合的极端时$b$和$c$是固定的。 

这减少了有效搜索空间：而不是尝试所有$a$，我们只需要考虑边界附近的候选者。 对于已排序的数组，对于任何固定的$b, c$， 最好的$a$是最小或最大元素，因为凸性确保没有内部值可以击败两端。 

因此，我们将问题简化为迭代所有对$(b, c)$，并且对于每一对检查恒定数量的候选者$a$，通常只是最小值和最大值。 这将复杂性降低到$O(N^2)$。 

我们可以通过观察对称性进一步细化评估：表达式在$a, b, c$，所以我们不需要太仔细地区分角色。 相反，我们可以修复两个索引并使用极值评估最佳完成度。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^3)$|$O(1)$| 太慢了 |
 | 最佳 |$O(N^2)$|$O(1)$| 已接受 |

 ## 算法演练

 我们对数组进行排序，以便可以快速引用最小值和最大值并推断极值行为。 

1. 对数组进行排序$V$按非递减顺序。 排序使我们能够直接访问全局极值，这是凸完成步骤所需的唯一候选者。 
2. 初始化变量`ans`到一个非常小的值。 这将跟踪所有配置中发现的最大利润。 
3. 迭代所有索引对$(i, j)$, 治疗$V[i]$和$V[j]$作为盒子尺寸的两个。 此步骤探索所有结构对相互作用，这是表达中大部分变异性的来源。 
4. 对于每对$(i, j)$，计算第三维的两个候选完成：$V[0]$和$V[N-1]$。 这些代表最小和最大可用值，由于每个变量的凸性，这些值就足够了。 
5. 对于每位候选人$k \in \{0, N-1\}$，计算利润：$$V[i]^2 + V[j]^2 + V[k]^2 - V[i]V[j] - V[j]V[k] - V[k]V[i]$$并更新`ans`如果这个值更大。 

1.处理完所有pair后，输出`ans`。 

### 为什么它有效

 当其他变量固定时，该表达式在每个变量中都是对称且凸的。 凸性确保对于任何固定对，一旦变量排序，使表达式最大化的第三个变量位于允许集合的边界之一。 由于排序使边界可全局访问，因此仅检查第三个坐标的最小值和最大值就足够了。 对称性确保修复任何对都不会失去通用性，因此通过迭代所有对来覆盖所有最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    v = list(map(int, input().split()))
    v.sort()

    INF = -10**30
    ans = INF

    # precompute extremes
    mn = v[0]
    mx = v[-1]

    for i in range(n):
        a = v[i]
        a2 = a * a
        for j in range(n):
            b = v[j]
            b2 = b * b

            # try c = mn
            c = mn
            val = a2 + b2 + c * c - a * b - b * c - c * a
            if val > ans:
                ans = val

            # try c = mx
            c = mx
            val = a2 + b2 + c * c - a * b - b * c - c * a
            if val > ans:
                ans = val

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先对输入进行排序，以使边界访问变得简单。 双循环枚举所有有序对，这是安全的，因为公式是对称的，因此排列不会改变正确性保证。 

在循环内，我们仅测试第三维的最小值和最大值。 这些被预先计算为`mn`和`mx`，避免重复索引。 该表达式是直接使用整数算术计算的，它可以安全地处理高达约$10^{18}$Python 中没有溢出。 

一个微妙的实施细节是我们不强制执行$i < j$。 这是故意的，因为允许重复的尺寸并且对称性确保正确性不会损失。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from math import prod  # dummy import to ensure environment stability

    n = int(input())
    v = list(map(int, input().split()))
    v.sort()

    ans = -10**30

        a = v[i]
        for j in range(n):
            b = v[j]

 ## 边缘情况

 单元素输入，例如$N = 1$产生一个所有尺寸都相等的盒子，因此利润变为$3a^2 - 3a^2 = 0$。 该算法可以处理此问题，因为两个循环都运行一次，并且仅使用其自身和相同的边界值计算相同的值。 

第二种边缘情况是最佳解决方案对所有三个维度使用相同的值。 例如，对于重复的最优结构，算法仍然评估$a = b = c = mn$和$mx$，确保不会错过统一的解决方案。 

第三种情况是最佳解决方案混合了极值。 由于该算法始终测试涉及每对第三维的全局最小值和最大值的组合，因此任何依赖于极值的配置都会至少直接评估一次，即使在最优值高度倾斜时也能保持正确性。
