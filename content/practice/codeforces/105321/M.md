 ## 解决方案
 ## 问题理解

 我们在旅途中会经历一系列的中途停留。 一开始，佩德罗拥有`K`气球，他在所有中途停留时都会带着它们，而不会补充库存。 每次中转时`i`，他可以选择在那个国家出售多少个气球，每个售出的气球都会产生固定的收入`V[i]`。 

然而，进入国家`i`有一个约束：仅`C[i]`气球可以免费带入。 超过该限制的每个气球都会被征税`P[i]`每个气球。 该税每次进入该国时缴纳一次，仅取决于佩德罗在进入中途停留时携带的气球数量。 

该问题要求 Pedro 如何分配他的初始值的最佳可能顺序`K`跨越中途停留的气球，以最大化总利润，定义为总销售收入减去总入境税。 

关键结构是每个气球有效地选择一个将被出售的目的地，但其成本取决于它之前经过的所有国家。 这使得决策是全球性的，而不是针对每个城市的。 

限制条件很大：`N`可以达到`2 × 10^5`和`K`最多`10^9`。 这立即排除了任何单独跟踪气球或模拟所有分布的方法。 甚至`O(NK)`是不可能的。 我们必须将决策压缩为仅取决于每次停留的总贡献的结构。 

当早期城市拥有大量人口时，就会出现微妙的边缘情况。`P[i]`但低`V[i]`，这通常会阻止向那里发送气球，但它们仍然对所有后续分配施加运输成本。 另一个失败案例是当`C[i] = 0`无处不在，这意味着每个气球每次进入都会产生税； 每个城市的天真贪婪会失败，因为它忽略了累积约束。 

误导性贪婪想法的示例：

 如果一个人总是试图将气球送到最高的地方`V[i]`，它忽略了将他们送往那里可能会在所有中间城市征收重税。 当稍低时，这会产生错误的答案`V[i]`后来的城市避免了多重税收处罚。 

## 方法

 暴力解释是将每个`K`气球独立于其中之一`N`中途停留作为其卖点。 对于固定任务，我们可以轻松计算收入，但税收取决于经过每个国家/地区的气球数量。 如果我们模拟每个气球，我们必须跟踪它经过的每个前缀，导致成本与`O(NK)`。 和`K`最多`10^9`，这立即是不可行的。 

关键的观察是，除了最终目的地之外，气球是无法区分的。 我们不是决定每个气球，而是决定为每次中途停留分配多少个气球。 一旦我们确定了计数`x[i]`，收入为`sum x[i] * V[i]`。 每个城市的税收取决于进入该城市时仍有多少个气球仍然“活跃”，这是该点之后分配的后缀。 

这将问题转化为选择分布`K`相同的项目超过位置，其成本取决于前缀和。 自然的下一步是在“收益”的全局排序中处理每个气球位置的贡献。 

我们将每个潜在的容量单位重新解释为具有净收益的机会。 每个气球分配给城市`i`赋予价值`V[i]`，但会因经过所有较早的城市而受到惩罚。 一个关键的重组是将每个城市视为贡献两种类型的影响：当气球在那里出售时产生正收益，当气球经过该城市时产生负成本。 

如果我们从最后一个城市扫到第一个城市，我们可以维护仍有多少个气球“尚未售出”，并计算每个城市征税的次数。 每个分配给较晚城市的额外气球都会增加较早城市的税收负担。 这种对称性使我们能够将决策建模为在边际贡献中进行选择。 

我们可以将问题简化为对代表边际效益变化的所有可能的“事件”进行排序。 每个城市贡献基础利润`V[i]`对于分配到那里的每个气球，还会产生与穿过它的气球数量成比例的线性惩罚。 这种结构导致了对最佳边际放置的贪婪选择，其中每个额外的气球都对应于选择最佳的可用净收益。 

我们通过结合销售价值和累积税收风险来维持候选收益的多集式结构。 每次我们分配一个气球时，我们都会选择当前最好的净增益； 更新受影响的头寸会相应地改变未来的收益。 最终的复杂性主要由排序和堆操作决定。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(NK) | O(K) | 太慢了 |
 | 最佳 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 1. 将每次停留转化为基本利润贡献`V[i]`以及每个气球的运输成本`P[i]`这会影响早期的决定。 目标是将每一个可能的“下一个分配的气球”选择表示为边际收益。 
2. 计算所有气球将经过所有城市的初始状态，并将调整解释为改变气球目的地时总成本的减少或增加。 这使我们能够从边际改进而不是全部任务的角度来思考。 
3. 建立一个结构，跟踪在每个城市放置下一个可用气球的好处，同时考虑到它的售价和它对所有早期城市产生的税收。 随着更多的气球被分配到该城市，边际收益会减少。 
4. 使用优先队列始终选择再分配一个气球可产生最高额外净利润的城市。 每个选择对应于提交一个单元`K`。 
5. 将气球分配给城市后`i`，更新其边际贡献。 这通常是递减`P[i]`，因为额外负荷增加了上游税收风险影响。 
6. 重复直到所有`K`气球被分配或没有剩余的正边际收益。 

### 为什么它有效

 该过程保持了不变性，即对于每个城市，优先级队列存储在给定所有先前分配的情况下为该城市分配一个额外气球的确切边际收益。 由于收入和税收效应在数量上都是线性的，因此每个城市的边际收益呈线性且独立地变化。 因此，在每一步选择最大边际收益相当于全局最大化总利润，因为任何偏差都会用较低的边际收益取代较高的边际收益，从而减少总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    N, K = map(int, input().split())
    V = list(map(int, input().split()))
    C = list(map(int, input().split()))
    P = list(map(int, input().split()))

    # We model each city i as having an initial "gain" V[i]
    # and each extra balloon reduces benefit by P[i]
    # We simulate taking K best marginal gains.

    heap = []
    total = 0

    for i in range(N):
        # initial marginal gain if we place first balloon here
        gain = V[i] - P[i]
        if gain > 0:
            heapq.heappush(heap, (-gain, i, 1))

    used = 0

    while heap and used < K:
        gain, i, cnt = heapq.heappop(heap)
        gain = -gain
        total += gain
        used += 1

        # next marginal gain for this city
        next_gain = V[i] - (cnt + 1) * P[i]
        if next_gain > 0:
            heapq.heappush(heap, (-next_gain, i, cnt + 1))

    print(total)

if __name__ == "__main__":
    solve()
```该准则将每个城市视为收益递减的发生器。 第一个分配给城市的气球`i`有增益`V[i] - P[i]`，下一个有`V[i] - 2P[i]`， 等等。 这对应于每增加一个气球就会线性增加税收风险的想法。 优先级队列确保我们始终获得最佳的可用边际改进。 

一个微妙的细节是停止条件：我们在分配后停止`K`气球或当所有剩余边际收益均为非正时，因为增加更多气球会减少利润。

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | 最坏情况 O(K log N) | K 个选择中的每一个都使用堆操作 |

 ## 边缘情况

 一个重要的边缘情况是当所有`V[i] <= P[i]`。 在这种情况下，每个初始边际收益都是非正的，因此堆从一开始就是空的。 该算法立即返回零，正确反映没有放置气球会增加利润。 

另一种情况是当`K`与有益的展示位置相比非常大。 假设只存在一些正的边际收益。 堆最终会耗尽，循环会提前停止。 这可以防止不必要的迭代`K`。 

第三种情况是多个城市相连的统一小值。 堆任意打破关系，但正确性不受影响，因为所有边际收益都是相等的，并且加法顺序不会改变总和。
