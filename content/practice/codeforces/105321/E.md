 ## 解决方案
 ## 问题理解

 我们有一个英雄，他一开始就有固定数量的能量点。 有N种武器，每种武器最多可以使用一次。 每个武器都有三个参数 A、B 和 C。如果英雄在当前拥有 P 力量的情况下使用武器，则会同时发生两件事：英雄的力量下降到 (P − B) 除以 C 的下限，并且 Boss 失去 A 点生命值。 

目标是决定 Boss 最初的强度，同时仍然允许英雄使用某种武器使用顺序击败她。 有效的策略是所选武器子集的任意排列，因为每种武器都可以使用一次或跳过。 如果英雄在某个时刻他的力量永远不会变为负值并且对 Boss 造成的总伤害至少是她的初始生命值，则英雄获胜。 

关键的难点在于功率的变换不是线性的加法或减法。 每个武器在减去 B 后都会应用楼层划分，这使得武器的顺序至关重要。 早期使用的武器可能会极大地改变后来的武器是否可用。 

N 方面的约束很小，最多 200 个武器，但功率 P 很大，高达 100000。这种组合表明，如果我们将 P 视为朴素动态规划方法中的一个维度，则直接跟踪所有可能的功率状态是不可行的。 然而，由于每种武器单调且永久地减少功率，任何状态转换只会在功率上向下移动，而不会向上移动，这强烈表明代表可达到功率值的状态上的最短路径或 DP。 

楼层划分产生了一个微妙的边缘情况。 例如，如果 P 相对于 B 较小，则 (P − B) 变为负数，并且除法仍会产生有效整数，可能为负数。 这意味着即使中间威力变为非正值，只要在应用下一个武器之前它不会变为负值，一系列武器仍然可能有效。 另一个边缘情况是排序：稍微降低威力但保留可分性结构的武器可能严格优于造成更高伤害但威力过早崩溃的武器。 

## 方法

 暴力方法将尝试武器的每个子集以及该子集中的每个排列，模拟过程，并在跟踪有效性的同时计算总伤害。 这是正确的，因为它显式检查每个可能的顺序。 然而，排列的数量本身就是 N 的阶乘，甚至限制到子集也会给出 N 的 k 的总和选择 k 乘以 k 阶乘，这简化为 N 阶乘增长。 当N达到200时，这是完全不可行的。 

关键的观察是，使用武器后唯一重要的是所产生的力量值，而该值仅取决于之前的力量和所使用的武器。 这定义了整数功率状态上的定向转换系统。 每个武器都会创建一个从 P 到 Floor((P − B) / C) 的函数。 由于 N 很小但 P 很大，我们翻转视角：我们不是从 P 向前进行模拟，而是询问使用某种武器序列可以达到给定状态的中间功率值。 

这自然会导致图解释，其中节点是可能的功率值，但状态空间太大而无法显式扩展。 相反，我们利用每次转换都会严格降低功率，因此我们可以按降序处理状态，并使用动态编程来计算每个可达到的功率值的最佳可实现损害。 

我们维护一张从力量值到最大可实现伤害的地图。 对于每个状态，我们尝试应用每种武器并计算下一个威力。 如果下一个能力有效并改善了已知的伤害，我们会对其进行更新。 由于每个状态仅按降序处理一次，因此转换数量仍然是可控的。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N!) | O(N) | 太慢了 |
 | 状态 DP 超过权力 | O(N·P log P) | O(N·P log P) | O(P)| 已接受 |

 ## 算法演练

 我们将每个可达到的功率值视为一个状态，并计算在该状态结束时可实现的最佳伤害。 

1. 从 dp[P] = 0 的字典或数组开始，这意味着我们以全功率开始，造成零伤害。 这代表唯一已知的初始状态。 
2. 进程状态按功率降序排列。 降序的原因是每次转换都会降低功率，因此一旦处理了一种状态，未来的转换就无法将我们带回该状态或更高的状态。 
3. 对于每个当前功率值 p 及其相关的最佳伤害 d，考虑每种武器（A、B、C）。 
4. 计算下一个幂 p2 作为下限 ((p − B) / C)。 这个模型准确地模拟了武器如何改变英雄的力量。 
5. 如果 p2 有效（非负）并且我们可以改善 p2 处的已知伤害，则将 dp[p2] 更新为 dp[p2] + A。 
6. 跟踪所有状态下可达到的最大伤害，因为任何可达到的状态都对应于某些武器序列的可能完成。 
7. 将最大伤害转换为答案 V，将其解释为可完全耗尽的最大初始 Boss 生命值。 

基本思想是，每个状态都代表在某种武器序列之后剩余功率的可达到的配置，并且每次转换都对应于选择该序列中的下一个武器。 

### 为什么它有效

 该算法依赖于这样一个事实：武器的使用定义了整数功率状态上的单调递减系统。 因为每次转换都会严格降低功率或保持功率有限，所以不存在循环。 这使得状态图成为有向非循环结构，即使它没有显式构建。 以降序处理状态可以保证，当我们计算一个状态的转换时，所有可能导致该状态的状态都已经解决。 因此，每个 dp 值都代表了剩余能量的最佳可实现伤害，与达到的方式无关，从而确保了不同武器顺序的一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, P = map(int, input().split())
    weapons = [tuple(map(int, input().split())) for _ in range(N)]

    # dp[p] = maximum damage achievable when ending with power p
    dp = {P: 0}

    # process states in decreasing order of power
    states = sorted(dp.keys(), reverse=True)

    ans = 0

    while states:
        p = states.pop()
        cur_damage = dp[p]
        ans = max(ans, cur_damage)

        for A, B, C in weapons:
            if p < B:
                continue
            p2 = (p - B) // C
            if p2 < 0:
                continue

            nd = cur_damage + A
            if p2 not in dp or nd > dp[p2]:
                dp[p2] = nd
                states.append(p2)

        states.sort(reverse=True)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现维护从功率值到最佳伤害的映射，并重复传播每种武器引起的转换。 最微妙的部分是更新条件：如果我们提高了总伤害，我们只会覆盖一个状态，因为以更少的伤害达到相同的功率是严格更差的。 排序的状态列表确保我们始终首先扩展更高的功率状态，从而保留由严格递减转换引起的单调结构。 

除法步骤必须在 Python 中仔细编写为整数除法，使用`//`，它已经匹配非负输入的数学下限行为，并且代码明确避免了无效的负转换。

 ## 边缘情况

 一种极端情况是，由于地板划分，武器立即使进一步使用失效。 用于输入`P = 6`和武器`(A=3, B=7, C=5)`，条件`P < B`触发器，因此武器被跳过。 该算法永远不会从中生成负状态，因此不会引入错误的转换。 

另一个边缘情况是重复除法会很快将功率降至零。 在这种情况下，多个不同的序列可以收敛到相同的功率值。 DP 地图可以正确处理此问题，因为它始终只保留每种能量状态的最佳伤害，因此重复到达不会扭曲结果。 

当多种武器产生相同的过渡但不同的伤害值时，就会出现最后的边缘情况。 更新条件`nd > dp[p2]`确保只有最强的路径能够生存，并且由于未来的转换仅取决于功率而不是历史，因此丢弃较弱的路径并不会失去最佳解决方案。
