 ## 解决方案
 ## 问题理解

 我们得到一个长度为隐藏的字符串$N$，但我们永远无法直接看到它。 相反，我们可以询问有关子字符串的查询，每个查询都会询问所选子字符串是否$t = s[L..R]$是整个字符串的有效周期。 

如果整个字符串可以通过重复该子字符串整数次（至少重复两次）来形成，则该子字符串被视为句点。 换句话说，如果我们采取$t$并将其连接多次，我们必须准确地重建完整的字符串$s$，没有不匹配，也没有剩余字符。 

我们的任务不是恢复字符串。 我们只需要决定在最坏的情况下需要多少个查询来确定未知字符串是否是周期性的，并输出一组达到这个最小值的具体查询。 

每个查询有效地测试整个字符串是否是通过重复候选块构建的。 关键的困难在于，我们必须提前选择所有查询，而不会看到任何答案，并且仍然保证在收到响应后我们始终可以确定是否存在有效期限。 

从复杂性的角度来看，$N$可以大到$10^6$。 这立即排除了任何依赖于探索所有子串或模拟交互的事情。 该解决方案必须完全取决于长度的结构特性$N$，因为我们正在设计一个独立于实际字符串的固定查询集。 

一个微妙的边缘情况是$N$是素数。 在这种情况下，唯一可能的周期长度是$1$，整个决策简化为所有字符是否相同。 另一个边缘情况是当$N$是高度复合的，因为存在许多候选周期长度，但其中大多数在信息方面是冗余的。 

## 方法

 一个自然的起点是考虑弦的周期性意味着什么。 如果一个字符串是由重复的块组成的，那么存在一些最小的块长度$p$使得字符串恰好是其长度前缀的重复$p$。 任何有效期必须划分$N$，因为重复必须均匀地平铺字符串。 

这将隐藏结构减少为约数$N$。 每个除数$p < N$代表潜在的周期长度。 如果我们知道哪些除数是有效周期，我们可以通过检查它们中的任何一个是否有效来立即回答该字符串是否是周期性的。 

暴力策略是查询每个可能代表句点的子字符串。 由于任何有效周期都必须是完整的重复，因此测试以下形式的子字符串就足够了$[1, p]$对于每一个$p$划分$N$。 每个查询都会告诉我们该字符串是否完全由该前缀的重复组成。 这是正确的，但此类除数的数量可能很大，最多可达约$10^6$在不正常的情况下，查询预算可能会很昂贵。 

关键的观察结果是，测试所有除数不仅是足够的，而且在最坏的情况下也是必要的。 如果我们省略任何除数$p$，我们可以构造一个字符串，其唯一有效周期恰好是$p$，并且所有其他测试长度均失败。 这迫使任何正确的策略将每个候选除数作为单独的查询包含在内，因为每个候选除数都可以唯一区分不同的隐藏配置。 

如果字符串有句点，则也存在冗余$p$，那么它也有周期$k \cdot p$对于任意整数$k$，但这无助于减少查询集，因为我们不知道较小的除数是否成立。 因此，跳过任何除数都有可能失去完整性。 

因此，最佳策略是为每个真除数精确查询一个前缀$N$，并使用答案来确定其中至少一个是否是有效期限。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 尝试所有子字符串 | 指数| O(1) | O(1) | 不可能|
 | 尝试将所有除数作为查询 | O(√N) 列出除数 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为枚举所有有效的候选周期长度，它们恰好是$N$小于$N$。 

## 算法演练

 1. 迭代所有整数$d$从 1 到$N$，但只考虑那些相除的$N$。 每一个这样的$d$表示候选周期长度，因为有效的周期结构必须均匀地平铺字符串。 
2. 对于每个除数$d < N$，对子字符串构造查询$[1, d]$。 这检查整个字符串是否由其第一个字符串的重复组成$d$人物。 我们限制前缀子字符串，因为任何有效周期都会强制第一个块与模式完全匹配。 
3. 收集所有此类查询。 查询次数正好是真因数的个数$N$。 
4. 输出所有查询。 顺序并不重要，因为每个查询都是独立的，并且我们不会根据答案进行调整。 

重要的结构点是，我们并不是试图重建字符串，只是为了测试每个可能的平铺长度的一致性。 每个除数充当关于隐藏字符串结构的独立假设。 

### 为什么它有效

 字符串的任何有效周期都必须有长度$p$划分$N$，并且必须匹配字符串的前缀。 因此，每个有效的候选者都包含在我们的查询集中。 

相反，如果字符串不是周期性的，则这些除数都不会通过测试。 如果是周期性的，则至少会经过一个与真实周期相对应的除数。 由于我们在除数集上详尽地测试了所有可能性，因此我们不会错过正确的结构，并且任何不正确的结构都不会伪造所有除数的一致性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    divisors = []
    i = 1
    while i * i <= n:
        if n % i == 0:
            if i < n:
                divisors.append(i)
            j = n // i
            if j != i and j < n:
                divisors.append(j)
        i += 1

    divisors.sort()
    
    k = len(divisors)
    print(k)
    for d in divisors:
        print(1, d)

if __name__ == "__main__":
    solve()
```该实现完全专注于有效地枚举除数。 循环至$\sqrt{N}$确保我们无需扫描整个范围即可找到所有因子对。 我们明确排除$N$因为一个周期必须至少包含两次重复，使得$N$作为候选长度无效。 

每个除数都转换为以下形式的查询$(1, d)$，表示长度的前缀子串$d$。 

## 工作示例

 考虑$N = 4$。 

4 的约数是$1, 2, 4$。 我们丢弃 4，留下 1 和 2。 

| 步骤| 除数 | 行动|
 | --- | --- | --- |
 | 1 | 1 | 查询 (1,1) |
 | 2 | 2 | 查询 (1,2) |

 所以$K = 2$，我们输出长度为 1 和 2 的查询。 

如果隐藏字符串是“aaaa”，则两个查询都返回 yes。 如果是“abca”，则两者都返回 no。 如果是“abab”，则只有长度为2的查询才返回yes。 

现在考虑$N = 6$。 

除数是$1,2,3,6$，所以我们测试$1,2,3$。 

| 步骤| 除数 | 行动|
 | --- | --- | --- |
 | 1 | 1 | 查询 (1,1) |
 | 2 | 2 | 查询 (1,2) |
 | 3 | 3 | 查询 (1,3) |

 如果字符串是“abcabc”，则仅$p=3$成功了。 如果是“aaaaaa”，则全部成功。 如果不是周期性的，则不会成功。 

这些例子表明，每一种可能的周期结构都被至少一个经过测试的除数所捕获。

 ## 边缘情况

 对于素数$N$， 例如$N = 7$，算法仅输出除数$1$。 这正确地反映了唯一可能的周期是长度 1，因此唯一有意义的测试是所有字符是否相同。 除数枚举自然地将问题归结为单一假设。 

对于高度复合数，例如$N = 12$，算法生成所有除数$1,2,3,4,6$。 尽管其中一些在周期结构方面是多余的，但它们对于最坏情况的区分是必要的。 如果我们删除其中任何一个，我们就可以构造一个唯一有效周期与缺失除数匹配的字符串，从而使策略不完整。 

为了$N = 1$，不会产生任何查询，因为不存在需要至少两次重复的有效周期。 这与定义一致，因为单字符字符串不能分解为长度较小的重复块。 

这些情况中的每一个都直接来自基于除数的构造，确保一致性而无需特殊情况逻辑。
