 ## 解决方案
 ## 问题理解

 我们得到了单个球员所参加的网球比赛的时间记录。 每场比赛的结果要么是胜利，要么是失败，编码为二进制数组，其中 1 代表胜利，0 代表失败。 

评分系统有两个独立的组成部分。 第一个很简单：每次胜利贡献+1分，每次失败贡献-1分。 第二个取决于连胜：每当一场比赛结束并且玩家当前处于至少三连胜（包括这场比赛）时，他们会在那一刻获得额外的 +1 分。 

关键的微妙之处在于，这种连胜奖金是在每场比赛后评估的，而不仅仅是在比赛结束时评估的。 因此，四连胜会带来多项奖励，其中每场比赛结束的后缀长度至少为 3 的位置都会获得一项奖励。 

输入尺寸很小，N 最大为 100，这可以立即允许直接线性扫描，而无需考虑超出 O(N) 的优化。 即使 O(N^2) 模拟也是可以接受的，但问题的结构显然允许 O(N)。 

一个幼稚的错误是只检查是否存在长度至少为三的条纹，并为每个条纹添加一个奖励。 例如，对于输入`1 1 1 1`，正确的奖励是 2（在位置 3 和 4），但简单的实现可能会错误地仅添加 1。 

另一个常见的错误是计算全局连续数而不是每个位置的连续数。 例如，在`1 1 1 0 1 1 1`，第二次连续再次独立于第一次贡献多个奖金。 

正确的解释是严格局部的：在每场比赛 i 之后，我们检查最后三场比赛（i-2 到 i）是否全部获胜，如果是，我们添加一个奖励分。 

## 方法

 思考这个问题的一种强力方法是从字面上模拟评分规则。 对于每个位置 i，我们通过向后扫描直到遇到损失来计算以 i 结束的连续获胜的当前连胜。 如果该条纹长度至少为 3，我们会增加奖励。 

这是可行的，因为它直接匹配定义，但在最坏的情况下，当所有比赛都获胜时，它可能会降级为 O(N^2)。 每个位置都会回扫 O(N) 步。 

然而，改变一切的观察结果是我们实际上并不需要完整的条纹长度。 奖金条件仅取决于最后三场比赛是否获胜。 如果位置 i、i−1 和 i−2 存在并且均为 1，则玩家在位置 i 处恰好获得一份奖金。 这将潜在的无限向后扫描转换​​为每个位置的恒定时间检查。 

因此，问题简化为二进制数组上大小为 3 的简单滑动窗口。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力向后扫描 | O(N^2) | O(N^2) | O(1) | O(1) | 太慢（不必要）|
 | 最佳滑动检查 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理比赛，同时保持总分。 

1. 将答案初始化为零。 该变量会累积赢/输点数和连胜奖金。 
2. 从 0 到 N−1 迭代每个匹配 i。 对于每场比赛，如果结果获胜则添加 +1，如果失败则添加 -1，更新得分。 这直接编码基本评分规则。 
3. 如果 i ≥ 2，则检查 Ri、Ri−1 和 Ri−2 是否都等于 1。此检查确定当前比赛是否完成以 i 结束的至少三连胜。 
4. 如果条件成立，则为答案再加 +1 分。 这完全符合每个合格端点独立贡献一份奖金的规则。 

处理完所有比赛后，累加值就是最终得分。 

### 为什么它有效

 该算法依赖于这样一个事实：奖励规则仅取决于在位置 i 结束的长度为 3 的窗口是否完全由获胜组成。 从第三次胜利开始，任何更长的连胜都会在每个后缀端点自动触发此条件。 因此，对每个有效的长度为 3 的窗口精确计数一次相当于对所有符合条件的条纹端点进行计数。 基本分数与此条件无关，因此将两个贡献分开可以保持正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    ans = 0
    
    for i in range(n):
        if a[i] == 1:
            ans += 1
        else:
            ans -= 1
        
        if i >= 2 and a[i] == 1 and a[i-1] == 1 and a[i-2] == 1:
            ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案保留单个运行累加器。 循环内的第一次更新直接处理线性评分规则。 第二个条件块是连续三场胜利的滑动窗口检查，仅在至少处理了三场比赛时应用。 

关键的实现细节是顺序：我们首先应用当前比赛的基本分数，然后评估在该位置结束的连胜。 这确保了连续检查始终包括当前匹配。

 ## 边缘情况

 对于最小的输入`N = 1`，说输入`1`，算法将基本分数设置为 +1，并且由于 i < 2 而从不进入条纹检查。输出为 1，这是正确的，因为不存在长度为 3 的条纹。 

对于像这样的完整损失序列`0 0 0 0`，每个位置贡献-1并且没有条纹。 分数变为 -4，符合规则，因为奖励条件从未被触发。 

对于长期连胜，例如`1 1 1 1 1`，该实现正确地在索引 2、3 和 4 处添加奖金。明确追踪，在 i = 2,3,4 处，三窗口条件成立，恰好产生三个奖金增量。 这证实了重叠的连续贡献是按位置独立处理的，而不是分组到单个事件中。
