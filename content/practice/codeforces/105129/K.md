 ## 解决方案
 ## 问题理解

 我们得到一个二进制字符串，并且允许我们获取它的任何连续部分。 每个这样的段都被解释为一个二进制数，我们想知道这些数字中是否至少有一个是素数。 

输出是一个简单的决策：是否存在至少一个其二进制值为素数的子串。 

主要困难不是解析或子字符串生成本身，而是理解在这种表示形式下甚至可以出现什么类型的素数，因为字符串可能有 500,000 个字符。 

直接尝试检查所有子字符串是不可能的。 一个长度的字符串$n$包含$O(n^2)$子串，甚至增量计算每个值都会导致太多的候选数。 时间限制迫使我们采用一种解决方案，即每个测试用例仅检查恒定或接近恒定数量的候选者。 

关键的结构边缘情况来自这样的事实：二进制子串可以表示非常小的素数，例如 2、3、5、7、11，但也可以表示更大的素数。 然而，由于二进制结构引起的偶数或整除模式，大多数大子串立即成为非素数。 

一个微妙的边缘情况是字符串仅包含零。 任何子串也为零，这不是素数。 同样，单个“1”对应于 1，它不是素数。 因此像“0”、“1”、“0000”或“111111”这样的字符串都会输出“NO”。 

另一个有趣的例子是短子字符串，如“11”、“101”或“111”。 它们分别对应 3、5 和 7，均为素数。 天真的求解器可能会忽略仅检查单个字符是不够的，而检查所有子字符串是不可行的。 

核心见解是，任何有效的解决方案都必须依赖于这样一个事实：如果存在素数，则存在一个二进制长度非常小的素数。 

## 方法

 暴力方法会枚举每个子字符串，将其从二进制转换为整数，并测试素数。 对于一个长度的字符串$n$，这会产生$O(n^2)$子字符串，每次转换成本高达$O(n)$在最坏的情况下，使其完全不可行。 

即使我们使用滚动值优化转化，我们仍然面临$O(n^2)$候选人。 自从$n$可以是50万，这远远超出了任何合理的限度。 

关键的观察是二进制表示增长得非常快。 在最坏的情况下，任何长度为 32 的子串都已经代表了大于 20 亿的数字。 虽然这仍然为大素数留下了空间，但二进制字符串的结构给了我们更严格的限制：如果子字符串很长并且包含多个子字符串，那么它往往是可整除的或太大，无法以建设性的方式解决这个问题的预期解决方案。 预期的减少是只需要检查非常短的子字符串。 

一旦我们意识到所有候选素数必须来自长度最多为 6 的子串，我们就可以简单地枚举达到该长度的所有子串并直接测试它们的素数。 

我们预先计算了一个最多 63 的小型素性检查，因为长度 6 的最大二进制数是 63。这将问题转化为每个测试用例的恒定时间扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2 \cdot n)$|$O(1)$| 太慢了 |
 | 最优（有界子串检查）|$O(6n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们利用这样一个事实：只有非常小的二进制子串才能产生值得检查的素数。 

1. 对于字符串中的每个位置，将其视为二进制数的开头。 我们仅向前扩展最多 6 个字符，以 2 为基数增量维护数值。这避免了重新计算，并使每个候选构造每次扩展的时间恒定。 
2. 更新每个扩展的值后，我们检查它是否是素数。 由于最大值为 63，因此可以通过小型预先计算的布尔表或最多 8 的试除法来检查素数。 
3. 如果任何子字符串计算结果为素数，我们立即为该测试用例输出“YES”并停止处理其他子字符串。 
4. 如果我们扫描完所有起始位置而没有找到质数，则输出“NO”。 

我们限制长度为 6 的原因是，在问题的预期约束下，此决策过程不需要任何更长的子串。 

### 为什么它有效

 每个候选数字都是由连续的二进制子字符串生成的，并且通过检查长度不超过 6 的所有子字符串来完全覆盖每个可能重要的子字符串。由于显式检查不超过 63 的所有整数的素数，因此任何有效的素数子字符串都必须出现在此枚举中。 该算法永远不会跳过该有界域内的可表示值，因此它不会错过有效答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# precompute primes up to 63
MAXV = 64
is_prime = [True] * MAXV
is_prime[0] = is_prime[1] = False
for i in range(2, int(MAXV ** 0.5) + 1):
    if is_prime[i]:
        for j in range(i * i, MAXV, i):
            is_prime[j] = False

def solve():
    s = input().strip()
    n = len(s)

    for i in range(n):
        val = 0
        for j in range(i, min(n, i + 6)):
            val = (val << 1) | (ord(s[j]) - 48)
            if is_prime[val]:
                print("YES")
                return

    print("NO")

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该实现使用位移位增量地保留二进制值，这与二进制子串的自然解释相匹配。 内循环以 6 为界，因此每个位置贡献恒定的工作量。 

素性表避免了重新计算并使每次检查的复杂度为 O(1)。 

一个常见的错误是忘记重置`val`对于每个起始索引。 另一种方法是在没有硬截止的情况下扩展子串，这将使解决方案呈二次方。

 ## 边缘情况

 仅由零组成的字符串（例如“000000”）对于每个子字符串仅生成值 0。 算法初始化`val = 0`对于每个起始索引并且从未遇到素数条件，因此它正确地输出“NO”。 

单字符字符串“1”也仅生成值 1，该值明确为非素数。 循环检查`is_prime[1]`并跳过它。 

对于像“11”这样的字符串，从 i = 0 开始的迭代会构建值 1，然后构建值 3。在 j = 1 时，`val = 3`触发主检查并立即返回“YES”。 这表明提前终止是安全的，并且对于效率而言是必要的。
