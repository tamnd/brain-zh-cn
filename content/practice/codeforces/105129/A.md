 ## 解决方案
 ## 问题理解

 对于每个测试用例，我们都会给出一个正整数`n`。 我们必须计算有多少个正整数在范围内`1`到`n`有奇数个小数位。 

的价值`n`可以大到`10^18`。 这立即排除了任何单独检查每个数字的解决方案。 即使对前十亿个数字进行循环也已经太慢了，而最大的输入包含五千万个可能的值。 该算法必须仅取决于数字的位数`n`，这最多是`19`。 

十进制表示自然地按位数对数字进行分组。 每个具有相同位数的数字都形成一个连续的区间，因此我们可以计算整个数字长度范围，而不是计算单个数字。 

一个容易犯错误的地方是`n`位于奇数长度范围内而不是其末端。 

例如：```
n = 234
```正确答案是：```
9 + (99 - 10 + 1) + (234 - 100 + 1)
= 9 + 90 + 135
= 234
```仅计算完整数字范围的粗心实现将仅计算一位数字并返回`9`，缺少部分三位数范围。 

另一个常见错误恰好发生在十的幂处。 

例如：```
n = 100
```答案是：```
9 + 1 = 10
```数量`100`是第一个三位数，因此必须包含它。 使用`<`而不是`<=`计算最后一个范围时会错误地返回`9`。 

最大的投入也值得关注。 

例如：```
n = 10^18
```这是一个 19 位数字。 虽然这样的数字最多只有一个`10^18`，它仍然对答案有贡献，因为 19 是奇数。 具有固定大小整数的语言在计算能力为 10 时可能会溢出，但 Python 整数会自动增长。 

## 方法

 最直接的解决方案是迭代每个整数`1`到`n`，计算其小数位数，并在该位数为奇数时增加答案。 这种方法显然是正确的，因为每个候选者都被检查一次。 

其运行时间为`O(n)`。 和`n`达到`10^18`，这将需要大约一千万次迭代，使其完全不可行。 

关键的观察结果是具有相同位数的数字总是形成一个连续的区间。 

间隔为`d`- 数字是：```
[10^(d-1), 10^d - 1]
```除了一位数开始于`1`。 

每当`d`是奇数，该区间内的每个数字都对答案有贡献。 如果整个区间都低于`n`，我们只需添加它的大小。 如果`n`落在该区间内，我们只添加以结尾的前缀`n`。 

由于数量高达`10^18`最多有`19`的数字，只有`19`要检查的可能的数字长度。 这将每个测试用例的工作量减少到一个微小的恒定量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n) | O(1) | O(1) | 太慢了 |
 | 最佳| O(log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取值`n`。 
2. 计算小数点后的位数`n`。 这决定了需要考虑多少位数字长度。 
3. 将答案初始化为零。 
4. 迭代每个数字长度`d`从`1`最多位数为`n`。 
5. 跳过偶数值`d`，因为只有奇数位长度有贡献。 
6. 计算最小的数`d`数字。 这是`1`什么时候`d = 1`，否则就是`10^(d-1)`。 
7. 计算该数字长度应计算的最大数字。 它是较小的`n`和`10^d - 1`。 
8. 如果计算出的上限至少是下限，则添加间隔大小`upper - lower + 1`到答案。 这可以正确处理完整范围和最终部分范围。 
9. 输出累计答案。 

### 为什么它有效

 每个正整数恰好属于一位数字长度的区间。 该算法恰好访问每个奇数位长度一次，并精确计算该间隔中不超过的数字`n`。 没有省略任何数字，因为每个符合条件的间隔都会被处理。 没有数字会被计算两次，因为不同的数字长度对应于不相交的间隔。 最终的总和恰好是以下整数的个数`1`到`n`其十进制表示形式的位数为奇数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        digits = len(str(n))
        ans = 0

        for d in range(1, digits + 1, 2):
            if d == 1:
                low = 1
            else:
                low = 10 ** (d - 1)

            high = min(n, 10 ** d - 1)

            if high >= low:
                ans += high - low + 1

        print(ans)

solve()
```解决方案首先阅读所有测试用例。 对于每个值`n`，它确定必须检查多少个十进制数字长度。 

循环每次前进两位，因此仅处理奇数位长度。 这样就避免了不必要的工作，直接体现了问题需求。 

对于每个奇数位长度，代码计算该长度的数字占据的间隔。 下界需要对一位数字进行特殊处理，因为它们从`1`而不是`10^0`。 

上限是以下较小者`n`以及具有那么多位数的最大数字。 什么时候`n`位于当前区间内，这自然只计算有效的前缀。 

条件`high >= low`防止添加负间隔大小。 发生这种情况时`n`位数少于当前范围。 

Python 整数自动支持超出的值`10^18`，所以十的计算能力是完全安全的。

import sys
import io

def solve():
    input = sys.stdin.readline
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        digits = len(str(n))
        ans = 0

        for d in range(1, digits + 1, 2):
            low = 1 if d == 1 else 10 ** (d - 1)
            high = min(n, 10 ** d - 1)
            if high >= low:
                ans.append if False else None
                ans += high - low + 1

        out.append(str(ans))

    print("\n".join(out))

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout
    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()
    solve()
    res = sys.stdout.getvalue()
    sys.stdin = backup_stdin
    sys.stdout = backup_stdout
    return res

# minimum input
assert run("1\n1\n") == "1\n"

# boundary at first two-digit number
assert run("1\n10\n") == "9\n"

# boundary at first three-digit number

# end of three-digit range
assert run("1\n999\n") == "909\n"

# maximum constraint

 ## 边缘情况

 考虑过渡到新的奇数位长度。```
Input
1
100
```该算法首先处理一位数字并添加`9`。 对于三位数，间隔变为`[100, 100]`，多贡献一份价值。 最终的答案是`10`，正确包含第一个三位数。 

现在考虑奇数长度区间内的值。```
Input
1
234
```一位数间隔贡献`9`。 三位数间隔变为`[100, 234]`，其大小为`135`。 添加它们会产生`144`。 该算法自动截断最后一个间隔`n`，因此不会计算更大的三位数。 

最后，考虑最大可能的输入。```
Input
1
1000000000000000000
```该算法计算从一位数字到十七位数字的每个完整奇数数字间隔。 然后它处理十九位数字的间隔，这很简单`[10^18, 10^18]`，添加一个最终值。 由于 Python 整数具有任意精度，因此所有算术都保持正确而不会溢出。
