 ## 解决方案
 ## 问题理解

 我们有两个序列，每个序列都是同一组整数从 1 到 n 的排列。 任务是找到出现在两个排列中且严格递增的最长子序列的长度。 

子序列意味着我们选择元素而不改变它们在每个数组内的相对顺序。 增长意味着价值观本身必须严格增长。 因此，我们正在寻找同时满足两个约束的值序列：它必须按顺序出现在两个排列中，并且所选值必须递增。 

重新解释该问题的一个有用方法是将一个排列视为定义一个顺序，将第二个排列视为一个序列，在该序列中，当投影到第一个排序时，我们希望提取最长的递增模式。 

约束允许每个测试用例 n 最多为 100000 个，并且有多个测试用例。 这立即排除了对所有位置对的任何二次方法。 每个测试用例的解决方案必须接近线性或 n log n。 诸如对索引对进行动态编程或检查所有子序列之类的任何操作都会太慢。 

一个幼稚的错误是首先将其视为最长公共子序列，然后再强制执行递增顺序。 这将忽略两个数组都是排列的结构。 另一种常见的错误方法是使用另一种排列的值直接计算 LIS，而没有正确映射位置，这会破坏对齐条件。 

## 方法

 蛮力观点从经典的最长公共子序列开始。 我们可以定义一个 DP，其中 dp[i][j] 是前缀 a[1..i] 和 b[1..j] 的答案。 这适用于一般序列，但每个测试用例的成本为 O(n^2)，当 n 为 100000 时，这远远超出了限制。 

关键的简化来自于两个数组都是排列的事实。 这意味着每个值在每个数组中只出现一次。 我们可以将问题转换为单个序列问题，而不是考虑匹配位置。 

我们将数组 b 固定为参考顺序。 我们将 a 中的每个值映射到 b 中的位置。 然后我们将 a 转换为数组 p，其中 p[i] 是 a[i] 在 b 中的索引。 现在 a 的任何子序列都对应于 p 的子序列，并且两个排列共有的要求意味着我们选择 b 中位置递增的值。 如果我们从遵循 b 排序的子序列中按顺序选择它们，则已自动满足所选值的值增加的条件。 这将问题简化为寻找映射数组 p 的最长递增子序列。 

整个问题变成了长度为 n 的数组上的标准 LIS 问题，可以使用带有二分搜索的贪婪尾部数组在 O(n log n) 中解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 LCS DP | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 太慢了 |
 | 位置测绘+LIS | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。

1. 构建一个数组 pos，其中 pos[x] 是排列 b 中值 x 的索引。 这为我们提供了一种快速比较 b 中排序的方法。 
2. 将每个元素a[i]替换为pos[a[i]]。 经过这个转换，我们得到一个数字序列 p，表示 a 的每个元素在 b 中出现的位置。 现在问题简化为在 p 中找到最长的递增子序列。 
3. 维护一个数组 tails，其中 tails[len] 存储迄今为止找到的长度为 len 的递增子序列的最小可能的最后一个值。 这种结构确保我们保留最容易扩展的子序列。 
4. 对于 p 中的每个值 x，使用二分搜索查找 tails 中 x 可以扩展或替换现有尾部的第一个位置。 如果 x 大于所有当前尾部，则扩展序列； 否则我们会更新适当的位置。 
5. 最终尾部的大小是最长递增子序列的长度，这就是答案。 

二分查找有效的原因是尾部始终保持排序。 每个位置代表该长度的子序列的最佳可能结束值，因此我们永远不会失去潜在的最佳扩展。 

### 为什么它有效

 任何公共递增子序列都对应于 b 中位置严格递增的值序列。 由于我们通过 b 中的位置对每个值进行编码，因此我们准确地选择了 p 中的递增子序列。 相反，p 中的任何递增子序列都对应于在两个排列中以递增顺序出现的值。 因此，p 上的 LIS 准确捕获了 LCIS 长度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))

        pos = [0] * (n + 1)
        for i, x in enumerate(b):
            pos[x] = i

        p = [pos[x] for x in a]

        tails = []

        for x in p:
            lo, hi = 0, len(tails)
            while lo < hi:
                mid = (lo + hi) // 2
                if tails[mid] < x:
                    lo = mid + 1
                else:
                    hi = mid
            if lo == len(tails):
                tails.append(x)
            else:
                tails[lo] = x

        print(len(tails))

if __name__ == "__main__":
    solve()
```第一部分为 b 构建逆位置映射，这很重要，因为它将值比较转换为索引比较。 如果没有这一步，我们就无法将这两个排列统一为一个序列问题。 

第二部分构造p，它是a在b坐标系中的变换表示。 后面的每一步都取决于此减少。 

LIS 实现使用标准耐心排序思想。 二分搜索确保我们有效地维护最佳尾部，并且更新尾部而不是存储完整序列可以保持内存线性。 

## 工作示例

 考虑一个简单的情况，其中 a = [1, 3, 2, 4] 且 b = [1, 2, 3, 4]。 那么 pos 就变成了 {1:0, 2:1, 3:2, 4:3}，p 就变成了 [0, 2, 1, 3]。 

| 步骤| x| 之前的尾巴| 行动| 尾巴之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 0 | []| 附加| [0]|
 | 2 | 2 | [0]| 附加| [0, 2] |
 | 3 | 1 | [0, 2] | 替换索引 1 | [0, 1] |
 | 4 | 3 | [0, 1] | 附加| [0, 1, 3] |

 最终答案是3。这对应于子序列[1,2,4]。 

现在考虑相反的情况 a = [4, 3, 2, 1], b = [1, 2, 3, 4]。 那么 p = [3, 2, 1, 0]。 

| 步骤| x| 之前的尾巴| 行动| 尾巴之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 3 | []| 附加| [3] |
 | 2 | 2 | [3] | 替换| [2] |
 | 3 | 1 | [2] | 替换 | [1] |
 | 4 | 0 | [1] | 替换 | [0]|

 最终答案是 1，因为不存在递增结构。 

第一个跟踪显示本地替换如何提高未来的可扩展性，这是 LIS 背后的核心机制。 第二个表明严格递减的转化序列崩溃为长度一。

 ## 边缘情况

 一个关键的边缘情况是当一个排列完全颠倒时。 在这种情况下，映射会产生严格递减的序列，并且 LIS 算法必须正确地将其缩减为长度 1，而不会出现任何意外扩展。 尾部数组不断被替换而不是扩展，以保持正确性。 

另一种情况是两个排列相同。 然后 pos 将每个元素映射到它自己的索引，产生一个已经递增的序列，因此每个元素都会扩展尾部。 在这种情况下，算法不得触发替换。 

第三种微妙的情况是最佳子序列在两个数组中都不连续。 LIS 转换自然地处理了这个问题，因为它仅对 b 中的相对排序进行操作，而不是邻接，从而确保仍然可以正确发现此类模式。
