 ## 解决方案
 ## 问题理解

 我们正在模拟一个小型培训管理系统，用于跟踪学生及其累积的奖励积分。 每个学生都有一个固定的身份，由从 1 到 n 的索引和关联的名称给出。 最初，所有学生都从零分开始。 

初始化后，我们处理一系列命令。 有两种命令。 一个命令可以为特定学生添加奖励积分，但前提是给定的密码与系统密码完全匹配。 另一个命令根据学生的累积分数打印当前的排名。 

排名规则很简单：学生按照总分降序排列，如果两个学生的分数相同，则id较小的学生先出现。 打印记分牌时，我们仅显示当前得分非零的学生，并根据此排序顺序分配排名。 分数相同则排名相同。 

限制很小，最多有 100 名学生和 100 个操作。 这立即表明每个查询使用 O(n log n) 甚至 O(n²) 方法就足够了。 瓶颈不在于算法复杂性，而在于排序、联系和过滤的正确处理。 

主要的微妙之处在于，排名是根据当前状态的需要重新计算的，而不是增量维护的。 另一个微妙的问题是密码检查必须准确且区分大小写； 即使是很小的不匹配也应该拒绝更新。 

边缘情况出现在我们计算排名的方式中。 如果我们天真地按排序顺序将排名分配为 1、2、3，那么当分数相等时，我们将产生不正确的结果。 例如，如果两个学生都有 10 分，则他们的排名必须相同。 

另一个极端情况是处理零分学生。 即使它们存在于系统中，它们也绝不能出现在记分板输出中。 

最后，按 id 排序的决定性因素至关重要。 如果没有它，排序就会变得不稳定，并且结果可能会因实现而异。 

## 方法

 一种直接的方法是维护一个大小为 n 的数组来存储每个学生的当前分数。 对于奖励命令，我们检查密码； 如果匹配，我们会在恒定时间内更新分数。 当我们执行记分板命令时，就会出现复杂性瓶颈：我们必须按分数和 ID 对所有学生进行排序，然后计算排名并打印非零的学生。 

这种蛮力策略已经很适合了。 对 n ≤ 100 个元素进行排序的成本为 O(n log n)，即使我们对每个查询都进行排序，q ≤ 100，总工作量也可以忽略不计。 

关键的观察是不需要像平衡树或堆这样的复杂数据结构。 数据集足够小，因此从头开始重新计算顺序比增量维护它更简单且不易出错。 每个记分板查询都是独立的，因此我们可以根据需要重建排序视图。 

唯一真正的设计决策是如何正确分配等级。 排序后，我们扫描列表并将相同的排名分配给相同的分数，否则适当增加排名计数器。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（重新计算每个查询的排序）| O(q·n log n) | O(q·n log n) | O(n) | 已接受 |
 | 最优（由于约束而与蛮力相同）| O(q·n log n) | O(q·n log n) | O(n) | 已接受 |

 ## 算法演练

 我们维护两个数组：一个用于学生姓名，另一个用于当前分数，两者均按学生 ID 进行索引。

1.读取n和系统密码，然后读取所有学生姓名。 我们将姓名存储在数组中，这样我们就可以通过 id 直接访问学生的姓名。 
2. 用零初始化大小为 n 的分数数组，因为尚未授予任何奖金。 
3. 对于每个命令，我们解析它的类型。 如果是奖励命令，我们提取目标 ID、奖励值和密码字符串。 
4. 我们将提供的密码与存储的密码进行比较。 如果不完全匹配，我们将打印拒绝消息并跳过任何更新。 
5. 如果密码匹配，我们将奖金添加到学生的分数中并打印成功消息。 
6. 如果命令是记分板，我们将构建所​​有学生的列表及其 ID、姓名和分数。 
7. 我们通过降低分数然后增加 id 对此列表进行排序。 即使分数相同，这种排序也能确保确定性排名。 
8. 我们过滤掉所有得分为零的学生，因为他们不应该出现在记分牌中。 
9. 我们在扫描排序列表时分配排名。 第一个学生获得排名 1。对于接下来的每个学生，如果他们的分数等于前一个学生的分数，他们将获得相同的排名；否则，他们将获得相同的排名。 否则，它们的排名将成为显示条目中的当前索引位置。 
10. 我们将每个条目打印为“rank id name Score”。 

经过这些步骤后，系统状态已一致并准备好执行下一个命令。 

### 为什么它有效

 在任何时刻，分数数组都准确地表示所有接受的奖励操作的总和。 记分板始终直接从此数组派生，因此排序会产生正确的全局排序。 由于排名仅取决于相邻排序分数的相等性，因此共享相等值的排名可以保留所需定义下的正确性。 过滤零不会影响排序，因为所有排除的元素不会影响正分数之间的排名。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, password = input().split()
    n = int(n)

    names = [""] * (n + 1)
    for i in range(1, n + 1):
        names[i] = input().strip()

    score = [0] * (n + 1)

    q = int(input())
    out = []

    for _ in range(q):
        parts = input().split()

        if parts[0] == "bonus":
            idx = int(parts[1])
            val = int(parts[2])
            pw = parts[3]

            if pw == password:
                score[idx] += val
                out.append("Updated successfully")
            else:
                out.append("Wrong password please try again")

        else:
            arr = []
            for i in range(1, n + 1):
                if score[i] > 0:
                    arr.append((score[i], i, names[i]))

            arr.sort(key=lambda x: (-x[0], x[1]))

            rank_out = []
            current_rank = 0
            prev_score = None
            seen = 0

            for s, idx, name in arr:
                seen += 1
                if prev_score is None or s != prev_score:
                    current_rank = seen
                prev_score = s
                rank_out.append(f"{current_rank} {idx} {name} {s}")

            if rank_out:
                out.extend(rank_out)

        out.append("---")

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案将姓名和分数存储在直接基于索引的数组中，以便进行 O(1) 访问。 对于每个奖励命令，它都会执行恒定时间的密码检查并更新分数（如果有效）。 对于记分板命令，它会重建活跃学生列表，使用强制分数优先级和 ID 打破平局的元组键对它们进行排序，然后通过跟踪分数变化来分配排名。 

一个微妙的点是排名分配：我们使用运行计数器`seen`表示排序的过滤列表中的位置。 每当分数发生变化时，排名就设置为该位置； 否则，将被重复使用。 这确保相同的分数共享相同的排名值。 

分隔线根据需要附加在每个命令之后，包括记分板输出之后。

 ## 边缘情况

 一个关键的边缘情况是所有学生的分数都是零。 在这种情况下，记分板命令必须不产生任何排名线，而仅产生分隔符。 该算法处理这个问题是因为我们显式过滤`score[i] > 0`排序之前。 

另一个极端情况是许多学生重复得分相同。 例如，如果所有学生都达到 10 分，则他们都必须共享排名 1。排名分配逻辑根据新分数的第一次出现来设置排名，因此在出现较低分数之前不会发生增量。 

连续奖励命令的密码不匹配也很重要。 由于除非密码匹配，否则我们不会修改状态，因此重复的错误尝试会使分数数组保持不变，并且后来的记分板输出仅反映有效的更新。
