 ## 问题理解

 我们得到一个非负整数序列，并且我们可以选择对任何元素应用数字反转操作。 每个数字都可以独立反转，但只能反转一次。 执行这些选择后，我们获得了最终的数组，并且我们希望最大化它的 mex。 

mex 是未出现在结果数组中的最小非负整数。 因此，目标不是最大化值或重新排列元素，而是从策略上确保小整数尽早出现在可达值集中。 

关键的困难在于每个数字向系统贡献两个可能的值：它的原始值和它的数字反转值。 我们不是选择排列或顺序，只是决定每个元素采用哪种表示形式。 

这些约束意味着 n 最大可达 100000，值最大可达 10^18。 任何尝试所有反向或非反向分配的方法都会以 n 为指数，并且立即不可能。 即使天真地检查每个配置的 mex 也会远远超出限制。 

当反转产生新的小数字时，会出现微妙的边缘情况。 例如，120 变成 21，这可能会突然产生最初不存在的较低范围内的数字的可用性。 另一个边缘情况是当不同的数字在反转后折叠到相同的值时，这可能会影响我们是否可以“覆盖”目标 mex 之前的所有整数。 

一个天真的错误是认为我们只需要考虑原始集合。 例如，如果数组为 [10]，则 mex 为 0，但反转为 1，这会更改小值的可用性。 这表明反转可以直接改变从零开始的初始段的覆盖范围。 

## 方法

 暴力方法会尝试是否反转每个数字的所有 2^n 个选择，计算结果数组，并计算其 mex。 每个 mex 计算都是 O(n)，因此总复杂度变为 O(n · 2^n)，即使对于 n = 30，这也是不可行的。 

更结构化的观点是翻转视角。 我们不是决定最终的配置，而是询问在选择方向后我们可以保证将哪些从 0 开始的整数包含在数组中。 为了最大化 mex，我们希望确保从 0 到 k − 1 的每个整数都可以由至少一个元素（使用其原始形式或反转形式）形成，并且 k 本身无法形成。 

这将问题转化为整数覆盖问题：每个数组元素贡献一小组候选者，最多两个数字。 我们为每个元素选择一个选项，并且希望最大化从零开始的初始覆盖整数段的长度。 

关键的观察是，只有从零开始的连续范围内的数字才是重要的。 如果我们无法构造某个整数 x，则 mex 正是 x，并且 x 以上的任何内容都是无关紧要的。 

我们按升序处理整数。 对于每个目标值 x，我们检查是否存在可以直接或通过反转表示 x 的未使用的数组元素。 如果是，我们分配一个这样的元素来覆盖 x 并将其从进一步的考虑中删除。 如果没有，我们就停下来； x 是墨西哥。 

这种贪婪策略之所以有效，是因为每个整数 x 在需求上都是独立的：一旦我们决定 x 必须出现，则未来值不再需要用于 x 的任何元素，并且延迟分配只会降低可用的灵活性。

 ## 算法演练

1. 对于数组中的每个数字，计算其反转数字形式。 我们将这两个值存储为与该元素关联的选项。 这是必要的，因为每个元素只能贡献一个值，但我们必须知道两种可能性。 
2. 将所有元素插入到一个结构中，使我们能够快速查找值 x 是否可以被某些尚未使用的元素覆盖。 常见的选择是使用哈希映射或多集按值对索引进行分组。 
3、从x=0开始，向上移动。 对于每个 x，尝试找到任何可以生成 x 的未使用元素，无论是原始值还是反转值。 我们只需要存在，但我们还必须能够将所选元素标记为已使用。 
4. 如果没有元素可以产生 x，我们立即停止并输出 x 作为 mex。 
5. 如果存在这样的元素，我们将其分配给 x 并将其标记为已删除，以便以后的值无法重用它。 

这种贪婪赋值有效的原因是，一旦我们提交一个元素来表示某个整数 x，将其用于任何更大的值将永远不会帮助我们恢复 x，因为 mex 首先严格依赖于丢失的小值。 

### 为什么它有效

 该算法保持一个简单的不变量：处理 x − 1 后，[0, x − 1] 中的所有整数都保证可以使用不相交元素实现。 每个步骤要么将此前缀扩展一个，要么在第一个间隙处停止。 由于 mex 被定义为第一个缺失的整数，因此任何最佳解决方案也必须在相同的第一个间隙处失败，因为跳过 x 将使其永久缺失。 在原始形式和反转形式之间进行选择的能力仅扩大了每个元素的候选集，但并没有改变每个元素在构造前缀时最多可以使用一次的事实。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def rev(x: int) -> int:
    return int(str(x)[::-1])

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    from collections import defaultdict, deque

    # map value -> deque of indices that can produce it
    pos = defaultdict(deque)
    used = [False] * n

    for i, x in enumerate(a):
        pos[x].append(i)
        rx = rev(x)
        pos[rx].append(i)

    x = 0
    while True:
        if x not in pos:
            break

        # find an unused element that can produce x
        found = -1
        dq = pos[x]
        while dq:
            i = dq.popleft()
            if not used[i]:
                found = i
                break

        if found == -1:
            break

        used[found] = True
        x += 1

    print(x)

if __name__ == "__main__":
    solve()
```该解决方案构建从每个值到索引的映射，可以直接或通过反转生成它。 这很重要，因为当我们尝试实现目标整数 x 时，我们只关心某些未使用的元素是否可以以任一形式映射到它。 

贪心循环逐步增加x。 对于每个 x，我们尝试分配一个可以代表 x 的新元素。 一旦分配，该元素将通过使用的数组从所有未来的考虑中删除。 这确保我们永远不会在不同的 mex 位置上重复使用元素。 

一个微妙的点是我们不会重复重新计算反转。 相反，我们预先计算两个方向一次。 这使得每次查找的转换均摊为 O(1)。

 ## 边缘情况

 一个极端的情况是反转仅有助于产生中间值而不是所需的前缀。 对于像 [10, 20, 30] 这样的输入，反转会产生 [1, 2, 3]，这突然使 mex 完全取决于我们是否可以先顺序分配 0。 由于 10、20、30 以及它们的反转都不会给出 0，因此算法正确地在 x = 0 处停止。 

另一个边缘情况是重复的数字。 对于[10, 10, 1]，两个10都可以通过反转产生1，但只能使用一个。 贪婪结构确保一旦分配了一个 10，如果后续值需要，第二个 10 仍然可用，从而保持前缀构造的正确性。 

最后的边缘情况是同一个 x 存在多个表示。 该算法安全地丢弃额外的候选者，因为 mex 只要求存在，而不计算多重性。
