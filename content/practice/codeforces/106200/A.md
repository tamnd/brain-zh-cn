 ## 解决方案
 ## 问题理解

 我们给出了几个轴对齐的矩形，每个矩形的左下角固定在原点，而右上角给出为$(w_i, h_i)$。 因此每个输入对定义一个覆盖所有点的矩形$(x, y)$这样$0 \le x \le w_i$和$0 \le y \le h_i$。 

所有这些矩形都使用并集进行组合，这意味着如果一个点位于至少一个矩形中，则该点是允许的。 任务是计算该联合覆盖的总面积。 

一个关键的观察结果是所有矩形在以下位置共享相同的角：$(0,0)$，所以它们都“锚定”在同一个地方。 这使得几何体高度结构化：并集不是任意重叠的矩形，而是从原点扩展的矩形的集合。 

约束允许最多$10^5$矩形和坐标最多$10^9$。 这立即排除了任何逐点检查覆盖范围或离散化整个网格的方法。 如果我们试图对称地对待两个维度，那么即使天真地扫过坐标也是不可能的。 

重叠推理中出现了一个微妙的问题。 如果一个矩形是$(3, 10)$另一个是$(10, 3)$，两者都完全包含另一个，因此天真的“取最大宽度和最大高度”是不正确的。 除非所有对在两个维度上都具有可比性，否则联合不是单个矩形。 

第二种边缘情况是许多矩形部分支配其他矩形。 例如，$(5, 100)$,$(100, 5)$,$(6, 6)$。 除非我们仔细推理结构，否则这种联合并不明显。 

正确的输出是一个整数：至少一个矩形覆盖的总面积。 

## 方法

 暴力方法会尝试对飞机进行建模并标记覆盖范围。 一种方法是想象一个网格$10^9 \times 10^9$，这立即是不可能的。 即使压缩坐标仍然剩下$10^5$不同的 x 和 y 边界，并检查由它们形成的网格中的每个单元格会导致$O(n^2)$或更糟糕的行为。 

另一个残酷的想法是对矩形进行排序并尝试贪婪地合并它们。 但是，矩形不是按全序嵌套的，因此仅基于宽度或高度的贪婪合并会失败。 

关键的结构见解是形式的矩形的并集$[0, w_i] \times [0, h_i]$可以有不同的解释。 我们可以沿着 x 轴逐列思考，而不是直接以 2D 方式思考。 

修复一些x。 在此 x 坐标处，矩形贡献的覆盖范围可达高度$h_i$当且仅当$x \le w_i$。 所以在位置x处，并集的高度最大$h_i$在所有在 x 上至少延伸那么远的矩形中。 

这将问题转化为一维扫描。 每个矩形贡献一个区间$[0, w_i]$有价值$h_i$。 对于每个 x，我们希望获得活动间隔中的最大高度，然后对 x 进行积分。 

我们通过降序对矩形进行排序$w_i$。 当我们从大 x 移动到小 x 时，更多的矩形变得活跃。 两个连续宽度之间的当前答案段仅取决于迄今为止看到的最大高度。 

这减少了在扫描排序端点时维持运行最大值的问题。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²) 或更糟 | O(n) | 太慢了|
 | 按宽度扫 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个矩形重新解释为 x 轴上从 0 到$w_i$，具有恒定的高度$h_i$。 我们按排序顺序处理此覆盖范围的开始和结束位置。 

1. 将所有矩形按降序排序$w_i$。 这确保了当我们在 x 轴上从左向右移动（从大 x 到小 x）时，我们逐渐“激活”更多矩形。 
2. 初始化变量`max_h = 0`和一个变量`prev_w = 0`跟踪最后处理的 x 边界。 我们将积累面积`ans`。 
3. 按排序顺序遍历矩形。 对于每个矩形$(w_i, h_i)$， 对待$w_i$作为新的影响区间开始的断点。 
4. 更新前`max_h`，添加之间的线段贡献的面积`w_i`和`prev_w`。 该段的宽度为`prev_w - w_i`，其高度为`max_h`。 所以我们添加`(prev_w - w_i) * max_h`到答案。 这是有效的，因为在此范围内，活动矩形集不会改变，因此最大高度保持不变。 
5. 更新`max_h = max(max_h, h_i)`因为这个矩形在以下时间变为活动状态$x \le w_i$。 
6. 设置`prev_w = w_i`并继续。 
7. 处理完所有矩形后，除了最小的矩形之外什么都没有剩下$w_i$，因此不需要额外的贡献，因为我们实际上从 x = 0 开始。 

### 为什么它有效

 在任何 x 坐标处，活动矩形的集合正是那些具有$w_i \ge x$。 排序后，这对应于处理后的矩形的前缀。 该算法保持该前缀的最大高度。 连续排序之间$w_i$，活动集不会改变，因此并集高度是恒定的。 这将创建 x 的正确分段常数函数，并且总面积是该函数的积分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    rects = []
    for _ in range(n):
        w, h = map(int, input().split())
        rects.append((w, h))

    rects.sort(reverse=True)

    max_h = 0
    prev_w = rects[0][0]
    ans = 0

    for w, h in rects:
        ans += max_h * (prev_w - w)
        if h > max_h:
            max_h = h
        prev_w = w

    print(ans)

if __name__ == "__main__":
    solve()
```解决方案首先对矩形进行排序，以便我们处理递减的宽度。 变量`max_h`存储宽度至少为当前 x 位置的所有矩形中当前最佳可达高度。 关键是积累`ans += max_h * (prev_w - w)`，它计算高度不变的垂直条带的面积。 

一个微妙的点是初始化`prev_w`。 我们从最大宽度开始，因为在此公式中不存在超出它的区域。 另一个重要细节是更新`max_h`在添加当前段的区域后发生，确保我们不会过早地将矩形包含在不应处于活动状态的区域中。

 ## 边缘情况

 一种关键的边缘情况是只有一个矩形。 算法初始化`prev_w`到该矩形的宽度，并且永远不会进入有意义的段，从而产生零面积。 这与扫描解释相匹配，其中没有要积分的宽度间隔。 

另一种情况是，在较宽的较短矩形之后出现一个非常高但窄的矩形。 扫描确保其高度仅影响其左侧的区域，而不会错误地膨胀较早的部分，因为`max_h`分部贡献后更新。 

最后，当多个矩形共享相同宽度时，它们在相同边界之间不贡献任何水平段。 该算法自然会将它们折叠起来，因为`(prev_w - w)`变为零，因此重复项不会影响面积计算。
