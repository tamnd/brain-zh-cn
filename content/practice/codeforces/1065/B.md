 ## 问题理解

 我们有一个简单的无向图，具有固定数量的顶点和边。 该图没有自环，也没有重复的边。 该结构在其他方面不受限制，这意味着我们可以自由想象任何精确使用的有效配置$m$之间的边缘$n$标记的顶点。 

如果一个顶点不与任何边相交，则该顶点称为孤立的。 任务不是构造一个图，而是推理极端情况：在所有有效的图中$n$顶点和$m$对于边，我们需要尽可能少的孤立顶点数和尽可能多的孤立顶点数。 

关键的张力在于边如何“消耗”顶点以及我们如何自由地将边集中在小顶点子集中。 

约束条件允许$n$最多$10^5$，这立即排除了任何尝试模拟图构造或枚举边的解决方案。 答案必须来自于关于边缘如何分布的结构推理。 

天真的误解常常源于认为边总是“用完”两个新顶点的想法。 这是错误的，因为边可以共享顶点，而事实上，共享是导致孤立顶点的最小数量减少的原因。 

出现微妙的边缘情况时$m = 0$。 在这种情况下，每个顶点都是孤立的，因此最小值和最大值都是$n$。 另一个是当$m$足够大，即使我们试图避免隔离，我们也可以“触及”几乎所有顶点，但仍然不足以保证连通性。 

## 方法

 暴力视图是想象构建图并计算每个配置的孤立顶点。 对于每个边子集，可以计算至少一条边中出​​现多少个顶点，然后推断出孤立的顶点。 这立即爆炸，因为图的数量呈指数增长$m$和$n$，即使对于微小的输入也无法进行枚举。 

正确的观点是把问题反过来看。 我们不是任意放置边，而是问使用以下方法可以使多少个顶点成为非隔离的：$m$边，以及我们可以避免接触多少个顶点。 

对于最大数量的孤立顶点，我们希望将边尽可能集中在尽可能小的顶点集内。 如果我们选择$k$顶点并放置所有$m$它们内部的边缘，然后是剩余的$n-k$顶点是孤立的。 最小的$k$可以承载$m$Edges 是满足以下条件的最小整数：$$\binom{k}{2} \ge m$$如果$m = 0$， 然后$k = 0$。 否则，$k$必须至少为 2。 

所以最大孤立顶点是$n - k$， 在哪里$k$是可以支持的顶点集的最小大小$m$边缘。 

对于最小数量的孤立顶点，我们希望扩展边，以便它们接触尽可能多的不同顶点。 每条边最多可以引入两个新顶点，但是一旦顶点被重用，边际增益就会下降。 最优策略是最大化边覆盖的顶点数量，其限制为$2m$，但不能超过$n$。 所以孤立顶点的最小数量是：$$\max(0, n - 2m)$$这个界限可以通过在可能的情况下构建类似匹配的结构来实现，否则可以重用顶点来饱和覆盖。 

这两个极端是独立的：一种将边缘紧紧地包裹起来，另一种将边缘尽可能地展开。

 ## 算法演练

 1. 通过估计有多少个顶点可以被边“覆盖”来计算孤立顶点的最小数量。 每条边最多可以贡献两个新顶点，因此被覆盖的顶点数最多为$2m$，上限为$n$。 孤立计数为$n - \min(n, 2m)$，这简化为$\max(0, n - 2m)$。 
2. 通过查找容纳所有边所需的最小顶点数来计算孤立顶点的最大数。 开始于$k = 0$，并增加它直到$\binom{k}{2} \ge m$。 这$k$表示可以容纳所有边的最小顶点集。 
3. 剩余的$n - k$顶点可以完全隔离，因为在最佳构造中没有边被迫接触它们。 
4. 输出对：首先输出最小孤立顶点，然后输出最大孤立顶点。 

关键思想是两种计算都是相同边缘预算的独立极值打包。 

### 为什么它有效

 边接触的顶点数量决定了隔离度。 对于最小情况，我们最大化覆盖范围，并且每条边最多贡献两个不同的顶点，从而给出严格的上限。 对于最大情况，我们通过将边集中到密集子图中来最小化覆盖范围，并且$k$-顶点简单图正是$\binom{k}{2}$。 这两个相反的包装参数充分描述了所有可行的配置。

 ## 边缘情况

 对于$n = 1, m = 0$，算法将最小孤立顶点设置为$1 - 0 = 1$。 为最大，$k = 0$，所以结果是$1$。 这符合没有边的单个顶点必须被隔离的事实。 

为了$m = 0$与大$n$，公式得出$min = n$和$max = n$，反映出任何建设都不能减少隔离。 

对于非常大的$m$, 接近于$\binom{n}{2}$，三角搜索发现$k = n$，因此最大孤立顶点变为$0$。 最低限度也是$0$因为所有顶点都必须在稠密图中使用。
