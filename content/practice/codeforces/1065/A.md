 ## 解决方案
 ## 问题理解

 Vasya 拥有固定金额，他希望在重复促销中最大限度地增加他最终获得的巧克力棒数量。 每根金条都有固定的价格，所以他的初始购买力就是他可以直接购买多少根金条。 不同之处在于，商店提供了奖励：每当他购买一定数量的金条时，他就会免费获得额外的金条。 只要他不断购买足够的金条，就可以重复触发此促销活动。 

任务是针对每个测试用例，计算 Vasya 在表现最佳的情况下可以获得的总条数，结合付费购买和所有可能的免费奖励。 

约束条件达到$10^9$，这排除了一次购买一根金条或在天真的循环中迭代促销周期的任何模拟。 任何单独处理每次购买或每次促销激活的方法都会太慢，因为在最坏的情况下，迭代次数可能达到数十亿次。 

当考虑贪心模拟时，会出现一个微妙的问题。 人们可能会尝试反复模拟购买$a$酒吧, 收藏$b$免费酒吧，并重复直到钱用完。 这种天真的形式是不正确的，因为它隐含地假设我们在每个周期后低效地跟踪剩余资金，或者忘记剩余资金仍然有助于在完整捆绑包之外进行额外购买。 

当促销弱于成本比率时，就会出现另一种极端情况，例如当$b = 0$。 在这种情况下，解决方案应该彻底降级为按价格简单划分资金，而不是尝试“应用”无用的促销活动。 

## 方法

 暴力破解的想法是直接模拟 Vasya 的行为。 我们计算他可以用他的钱购买多少条金条，然后重复应用该规则：每次他积累足够多的购买金条时，我们将转换成组$a$购买的金条进入$b$额外的免费酒吧，其本身可以有助于进一步的促销活动。 这很快就会变得低效，因为每个促销应用程序都会更改状态，并且周期数可能与柱总数成正比，在最坏的情况下，柱总数是巨大的。 

关键的观察结果是问题的结构是线性的和乘法的。 每一组$a$购买的金条有效生产$a + b$总条数。 这意味着我们可以将整个过程压缩为计算有多少个完整的组，而不是模拟重复的交换$a$Vasya 可以购买的金条，并将这些组乘以每组的总产量。 

未完成整组的剩余条形$a$对促销没有任何贡献，但仍计入最终总数。 这种分成完整组和剩余组的方式消除了所有动态行为。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 模拟 | O（总条数）| O(1) | O(1) | 太慢了 |
 | 分组数学| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 计算 Vasya 可以用他的钱直接购买多少块巧克力棒，除以$s$经过$c$。 这给出了付费金条的数量，因为每个金条的成本$c$卢布。 
2. 将这些购买的金条分成完整的尺寸组$a$。 每个完整组只会触发一次促销。 
3. 将完整组的数量乘以$a + b$，因为每组$a$购买的金条产生那些$a$酒吧加$b$额外的酒吧。 
4. 添加未形成完整组的剩余购买金块，因为它们不贡献任何奖金，但仍计入总数。 
5. 输出最终的和。 

关键思想是促销不会在部分组之间交互，因此分组依据$a$完全捕获所有可能的奖励激活。 

### 为什么它有效

 购买的每根金条都有助于形成一个完整的块$a$或在促销方面未使用。 每个完整的块都确定性地产生$a + b$条无论购买顺序如何，并且块之间不存在依赖关系。 由于 Vasya 的购买能力固定为$s // c$，分解为完整块和余数保留了过程的所有结构，而不会丢失或重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        s, a, b, c = map(int, input().split())

        bought = s // c

        groups = bought // a
        rem = bought % a

        total = groups * (a + b) + rem
        print(total)

if __name__ == "__main__":
    solve()
```该解决方案首先将金钱转换为可购买的金条数量。 然后，它将这些酒吧分成完整的促销块和剩余部分。 乘法步骤是批量应用促销而不是增量应用的地方。 其余部分保持不变，因为它无法触发报价。 

必须小心整数除法； 楼层除法和取模都是必不可少的。 由于较大的限制，使用浮点运算会引入精度误差。 

## 工作示例

 ### 示例 1

 输入：```
10 3 1 1
```我们计算 Vasya 可以购买的金条数量：$10 // 1 = 10$。 

现在我们将它们分组：

 | 买了| 团体 | 余数 | 总计算量|
 | ---| ---| ---| ---|
 | 10 | 10 3 | 1 | 3 * (3 + 1) + 1 = 13 | 3 * (3 + 1) + 1 = 13 |

 结果是 13，符合每个 3 条完整块总共产生 4 条的想法。 

这证实了剩余的金块不参与促销活动。 

### 示例 2

 输入：```
1000000000 1 1000000000 1
```在这里，每个购买的金条都会立即触发促销。 

| 买了| 团体 | 余数 | 总计算量|
 | ---| ---| ---| ---|
 | 1000000000 | 1000000000 | 0 | 1000000000 * (1 + 1) = 2000000000 |

 然而，由于每个组产生的自由条的大小等于该组本身，因此最终值与总贡献的预期线性缩放相匹配。 

这表明当$a = 1$，每次购买都会直接导致双倍的结果，使得分组变得微不足道。

 ## 边缘情况

 当出现一种边缘情况时$a >$购买的金条数量。 例如，如果 Vasya 只能购买 2 根金条，但需要 5 根才能触发奖金，则算法会正确地将组设置为零并仅返回购买的金条，因为不适用促销。 

另一个边缘情况是当$b = 0$。 该公式减少为仅购买金条的数量，因为每个组都没有贡献额外的收益。 分组步骤仍然可以正确工作，因为它乘以$a + 0$，无需特殊处理即可保持正确性。 

最后的边缘情况是$a = 1$，其中每个购买的条形都形成自己的组。 该算法正确地加倍（或缩放）$1 + b$) 每个栏的贡献，与促销的预期重复应用相匹配。
