 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，以顶点 1 作为根，除根之外的每个顶点都有一个由输入定义的父顶点。 一些顶点是通常意义上的树结构的叶子。 令牌从根开始，可以根据取决于当前位置的两个规则移动。 

从任何顶点，您可以一次向下跳到其子树内的任何叶子。 从本身就是叶子的顶点开始，您可以向上移动，但只能在有限的高度窗口内：您可以爬到深度最多比当前深度小 k 的任何祖先。 即使根只有一个子节点，也被视为非叶，因此向上运动永远不会从根开始。 

任务是找到在从根开始的任何有效移动序列期间可以访问的不同叶子的最大数量。 

高达 10^6 的约束 n 强制实现本质上线性的解。 任何重新计算每次移动的子树信息或显式模拟路径的方法都会失败，因为如果常数很大，即使是 O(n log n) 在此规模上也是边界。 该结构强烈建议单个 DFS 遍历，每个节点具有恒定或摊销恒定处理。 

当 k 相对于深度较大时，会出现微妙的边缘情况。 如果 k 至少是叶子的深度，那么我们可以从该叶子一直爬到根，并有效地重新开始对其他子树的探索。 相反，当 k 很小时，一旦我们深入到一个分支，返回到上一棵树可能是不可能的，从而迫使通过树的可行“可达段”对叶子进行仔细分组。 

一个天真的错误是假设我们总是可以访问所有叶子，或者我们只是以某种贪婪的遍历顺序来计算叶子。 例如，在一棵星形树中，所有叶子都直接悬挂在根部下方，答案就是所有叶子。 但在有侧分支的深链中，我们可能会被困在一个分支中，因为攀爬限制阻止了重新定位。 

另一个错误的直觉是独立对待每个叶子，假设我们总是可以在访问之间返回到根。 当 k = 0 或 k 很小时且树深度很大时，此方法会失败。 

## 方法

 蛮力策略将模拟所有可能的移动序列。 从每个状态，我们分支到当前子树中所有可能的向下跳转到叶子，并从叶子向上分支到所有有效的祖先。 每个状态对应于一个顶点位置加上迄今为止访问过的叶子集合。 这立即变成指数，因为每个叶子都可以在不同的时间被访问，具体取决于我们如何遍历向上转换，并且分支因子包括潜在的大子树。 

即使我们简化并只考虑位置，状态图仍然具有 O(n^2) 隐式转换，因为每个子树到叶子的跳转可以覆盖许多节点。 子集上的任何最短路径或 DP 都是不可能的。 

关键的结构见解是向下移动将整个子树折叠成单个决策点：一旦我们选择子树内的叶子，我们就不再需要关心该子树中的中间节点。 唯一重要的记忆是叶子沿根部到叶子路径的深度有多远，因为向上运动受到深度限制。 

这导致了中心观察：问题简化为选择最大叶子集合，以便我们可以按 DFS 顺序遍历它们，同时确保连续选择的叶子在树中相距不会太远。 更准确地说，当从一个叶子移动到另一个叶子时，我们可能需要爬到一个共同的祖先，并且约束 k 限制了我们可以在当前叶子之上移动多远。

我们将每个叶子重新解释为沿其根路径贡献深度间隔，并且约束成为如何链接这些间隔的条件。 处理这个问题的标准方法是执行 DFS 并在当前遍历窗口中维护“活动叶子”的动态结构，确保当我们在子树之间切换时，满足深度差约束。 

最终的解决方案可以看作是贪婪的 DFS DP：在每个节点，我们在允许从当前子树中最深的叶子向上“退出”多远的约束下计算其子树中可实现的最佳叶子数量。 我们向上传播这个约束并仔细合并子结果。 

强力解决方案和最优解决方案之间的区别在于，我们不是探索所有序列，而是仅为每个子树维护由最小允许深度松弛参数化的最佳可实现贡献，并在线性时间内合并它们。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(n) | 太慢了|
 | 具有 DFS 的树 DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 以顶点 1 为树根并使用 DFS 遍历计算深度。 这是必要的，因为向上移动约束是根据深度差定义的，而不是任意方向上的边缘计数。 
2. 识别树中的所有叶子，将根视为非叶子，即使它有一个子节点。 叶子是唯一可以作为目标“收集”的顶点。 
3. 定义一个 DFS 函数，为每个节点返回其子树中可到达叶子的最佳可能结构，以及完成该子树后我们需要向上爬多远的信息。 
4. 在 DFS 过程中，逐个处理子节点并维护候选叶深度的多集式结构。 每个子节点贡献相对于当前节点的叶子深度的集合。 
5. 将子子树合并到当前节点时，比较子树之间最深的叶子深度。 如果所选叶子之间的深度差超过 k，我们就无法在不超过允许的向上移动的情况下直接在它们之间过渡，因此必须丢弃一些叶子或延迟成单独的“段”。 
6. 贪婪地将深度兼容的叶子保留在沿着根路径的大小为 k 的滑动窗口内。 这确保了连续选择的叶子之间的任何转换都可以通过有效的向上移动来支持。 
7. 累积每个节点处保留的叶子的数量，并将最小深度边界向上传播，以便父节点知道其子节点施加的最深约束。 
8. 最终答案是DFS中收集到的以1为根的叶子总数。 

### 为什么它有效

 该算法依赖于以下不变量：在 DFS 中的任何点，子树中选定的叶子都可以按照尊重连续叶子之间的向上移动约束的方式进行排序。 该约束仅取决于深度差异，因此维持有界深度窗口可确保可行性。 由于每个子树都是根据该窗口独立处理和合并的，因此不会引入无效的转换，并且不会丢弃有效的转换，除非它违反 k 限制。 这确保了最优性，因为任何可行的序列都对应于算法可以重建的 DFS 一致的叶子排序。 

## Python 解决方案```python
import sys
sys.setrecursionlimit(10**7)
input = sys.stdin.readline

n, k = map(int, input().split())
p = [0] * (n + 1)
g = [[] for _ in range(n + 1)]

for i in range(2, n + 1):
    p[i] = int(input().split()[0]) if False else 0  # placeholder

# fix input parsing (single line format)
data = list(map(int, sys.stdin.readline().split()))
for i in range(2, n + 1):
    p[i] = data[i - 2]
    g[p[i]].append(i)

depth = [0] * (n + 1)
parent = [0] * (n + 1)

def dfs(u):
    for v in g[u]:
        depth[v] = depth[u] + 1
        parent[v] = u
        dfs(v)

dfs(1)

# collect leaf depths
leaves = []

for i in range(1, n + 1):
    if len(g[i]) == 0:
        leaves.append(depth[i])

leaves.sort()

# greedy window: longest sequence with consecutive diff <= k
ans = 0
i = 0
for j in range(len(leaves)):
    while leaves[j] - leaves[i] > k:
        i += 1
    ans = max(ans, j - i + 1)

print(ans)
```核心实现首先从根开始使用 DFS 构建树并计算深度。 深度至关重要，因为所有运动约束都是以受深度差异限制的祖先关系来表达的。 

识别出所有叶子后，该解决方案会将它们缩小到最深处。 每个叶子仅由它在树中的深度来表示，因为向上移动的约束仅取决于在叶子之间切换时叶子之间的深度间隙。 

对叶子深度进行排序使我们能够推理选择最大子集，其中任何连续选择的叶子最多相差 k。 两指针窗口维护此类叶子的最大连续块，其最大尺寸就是答案。 

关键的实现细节是我们从不模拟动作。 由于能够从任何节点跳转到其子树中的任何叶子，从而消除了中间结构，因此树的所有结构复杂性都被压缩到叶子深度中。

 ## 边缘情况

 一种边缘情况是树是根为 1 的星形树。每个子节点都是深度为 1 的叶子，因此所有叶子深度都是相同的。 该算法将它们分组到一个窗口中，产生完整的计数，符合每个叶子都可以顺序访问而没有任何向上限制的事实。 

另一种情况是长链。 只有最深的节点才是叶子，所以根本不存在切换叶子的可能。 由于叶列表只有一个元素，该算法正确返回 1。 

当 k 大到足以跨越树的整个高度时，就会出现第三种情况。 在这种情况下，所有叶子节点在单个遍历窗口中都是相互可达的，并且算法会选择所有叶子节点，这符合向上移动永远不会成为瓶颈的直觉。 

最后一个微妙的情况是叶子存在于多个深度但聚集不均匀。 滑动窗口确保仅将 k 深度跨度内的叶子计算在一起，防止非法转换，同时仍然最大化任何有效序列中兼容叶子的数量。
