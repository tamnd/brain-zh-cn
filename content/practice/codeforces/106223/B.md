 ## 解决方案
 ## 问题理解

 场景是一个矩形农场，分为$N \times M$网格。 每个单元格都包含由整数表示的水果类型，蜜蜂使用标准的四向网格步骤穿过网格，始终沿着最短路径行进。 

每个查询的关键思想是这样的：给你两种水果类型$u$和$v$，并且您想要考虑其中一个细胞包含水果的所有细胞对$u$另一个含有水果$v$。 对于每个这样的对，一只蜜蜂沿着它们之间的最短网格路径行进，并且该路径覆盖多个单元格。 在所有有效对之间的所有最短路径中$u$-细胞和$v$-cells，要求您确定此类路径的最大可能长度。 

同样，对于每个查询，您正在使用两组网格点，并且您希望第一组中的任何点与第二组中的任何点之间的最大曼哈顿距离。 

输入大小足够大，以至于对每个查询的所有位置对进行简单扫描会太慢。 如果有多达$10^5$由于单元格和许多查询，每个查询的二次配对已经很棘手，甚至将查询中所有出现的情况相加也表明必须避免重复重新计算距离。 这直接指向预处理每种水果类型的位置。 

当一种或两种水果类型仅出现一次时，就会出现微妙的边缘情况。 例如，如果网格仅包含一个类型为$u$和类型之一$v$，答案很简单，就是它们之间的曼哈顿距离。 假设多次发生并尝试采取极端而不检查存在性的粗心实现将会崩溃或产生垃圾。 

另一个失败案例来自于忘记两个网格点之间的最短路径不是唯一的。 例如，之间$(1,1)$和$(2,2)$，有两条最短路径，但它们的长度都相等$2$。 该问题并不要求计算路径或构建它们，只要求计算它们的长度，因此这里的任何混淆通常会导致不必要的模拟尝试，从而导致时间复杂度爆炸。 

最后，一个常见的错误是认为答案取决于路径上的中间单元。 事实并非如此。 只有端点很重要，因为网格中每条最短路径的长度都等于曼哈顿距离，与路线无关。 

## 方法

 对于每个查询，直接的暴力方法将$(u,v)$,收集所有位置的水果$u$以及水果的所有位置$v$，然后计算每对的曼哈顿距离。 如果$c_u$和$c_v$是它们的频率，这成本$O(c_u \cdot c_v)$每个查询。 在水果类型多次出现的密集网格中，每个查询很快就会变成二次方，在最坏的情况下会退化为$O((NM)^2)$，远远超出任何可行的极限。 

关键的观察结果是曼哈顿距离分为独立的 x 和 y 分量：$$|x_1 - x_2| + |y_1 - y_2|$$对于固定集，可以在不枚举对的情况下减少在所有对上最大化此表达式。 无需检查所有组合，只需跟踪极值坐标组合即可：最小和最大值$x+y$,$x-y$，以及编码曼哈顿距离的类似线性变换。 

对于每种水果类型，我们可以预处理并存储四个值：最小值和最大值$x+y$，以及最小值和最大值$x-y$其发生的所有事件。 这些总结充分体现了该集合在曼哈顿几何中可以延伸多远。 

一旦这些被预先计算，每个查询就通过结合两种水果类型之间的最佳可能的极端值而成为恒定时间计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | ---|
 | 暴力破解（将每个查询的所有单元格配对）|$O(c_u \cdot c_v)$每个查询 |$O(1)$额外 | 太慢了|
 | 预先计算每个水果的极值 |$O(NM + Q)$|$O(K)$| 已接受 |

 ## 算法演练

 1. 遍历网格一次，按水果类型对坐标进行分组。 这将为每种水果构建一个位置列表。 目标是避免在查询期间重复扫描网格。 
2. 对于每种水果类型，计算其位置的四个聚合值：最小值和最大值$x+y$，以及最小值和最大值$x-y$。 这些值以足以满足曼哈顿距离查询的方式总结了水果类型的空间分布。 
3. 对于每个查询$(u,v)$，检索两种水果类型的预先计算值。 
4. 使用以下事实计算两个集合之间的最佳曼哈顿距离：$$|x_1-x_2|+|y_1-y_2| = \max((x_1+y_1)-(x_2+y_2), (x_2+y_2)-(x_1+y_1), (x_1-y_1)-(x_2-y_2), (x_2-y_2)-(x_1-y_1))$$这减少了检查极值之间的差异：$$\max(
\max(x+y)_u - \min(x+y)_v,
\max(x+y)_v - \min(x+y)_u,
\max(x-y)_u - \min(x-y)_v,
\max(x-y)_v - \min(x-y)_u
)$$5. 输出这四个值中的最大值作为查询的答案。 

### 为什么它有效

 曼哈顿距离可以看作两种线性形式中的最大值：$x+y$和$x-y$下标志翻转。 两组上的任何成对最大值总是在这些线性投影的边界点处实现。 由于我们只为每个投影保留极值，因此每个可能的最佳对都由存储的极值组合之一表示。 没有内点可以改善结果，因为任何非极值都受至少一个线性投影中的边界值支配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def add_point(stats, c, r):
    # stats: [min_xp, max_xp, min_xm, max_xm]
    xp = c + r
    xm = c - r
    if stats[0] is None:
        stats[0] = stats[1] = xp
        stats[2] = stats[3] = xm
    else:
        stats[0] = min(stats[0], xp)
        stats[1] = max(stats[1], xp)
        stats[2] = min(stats[2], xm)
        stats[3] = max(stats[3], xm)

def solve():
    n, m, q = map(int, input().split())
    grid = []
    for _ in range(n):
        grid.append(list(map(int, input().split())))

    # assuming fruit values up to K (compress if needed)
    # use dict for sparsity
    stats = {}

    for i in range(n):
        for j in range(m):
            v = grid[i][j]
            if v not in stats:
                stats[v] = [None, None, None, None]
            add_point(stats[v], i, j)

    out = []
    for _ in range(q):
        u, v = map(int, input().split())

        su = stats[u]
        sv = stats[v]

        ans = 0

        # x+y dimension
        ans = max(ans,
                  su[1] - sv[0],
                  sv[1] - su[0])

        # x-y dimension
        ans = max(ans,
                  su[3] - sv[2],
                  sv[3] - su[2])

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```网格被扫描一次，每种水果类型只保留四个数字。 查询阶段纯粹是算术，因此没有对单元列表的迭代。 

一个常见的实现错误是计算时交换行/列索引$x+y$和$x-y$。 只要保持一致性，要么$(i,j)$或者$(row,col)$之所以有效，是因为曼哈顿距离在坐标重新标记下是不变的。 

另一个问题是忘记初始化未见过的水果类型的统计数据。 由于查询可以包含网格中存在的任何水果 ID，因此每个查询类型都必须存在于预处理映射中。 

## 工作示例

 考虑一个小网格：

 输入：```
3 3 2
1 2 3
3 2 1
1 1 2
1 2
2 3
```我们计算每种水果类型的统计数据。 

对于类型 1：$$(x+y): \{2,4,4,3\}, (x-y): \{0,2,0,-1\}$$对于类型 2：$$(x+y): \{3,3,5\}, (x-y): \{1,-1,1\}$$对于类型 3：$$(x+y): \{4,4\}, (x-y): \{2,0\}$$查询 1：(1,2)

 | 步骤| 价值|
 | ---| ---|
 | 最大 (x+y)₁ - 最小 (x+y)2 | 4 - 3 = 1 | 4 - 3 = 1
 | 最大 (x+y)2 - 最小 (x+y)₁ | 5 - 2 = 3 | 5 - 2 = 3 |
 | 最大 (x-y)₁ - 最小 (x-y)2 | 2 - (-1) = 3 | 2 - (-1) = 3 |
 | 最大 (x-y)2 - 最小 (x-y)₁ | 1 - (-1) = 2 |

 答案 = 3

 查询2：(2,3)

 | 步骤| 价值|
 | ---| ---|
 | 最大 (x+y)2 - 最小 (x+y)₃ | 5 - 4 = 1 | 5 - 4 = 1
 | 最大 (x+y)₃ - 最小 (x+y)2 | 4 - 3 = 1 | 4 - 3 = 1
 | 最大 (x-y) 2 - 最小 (x-y) ₃ | 1 - 0 = 1 |
 | 最大 (x-y)₃ - 最小 (x-y)2 | 2 - (-1) = 3 | 2 - (-1) = 3 |

 答案 = 3

 这些痕迹表明，正确的对总是由极值投影确定，而不是由特定的中间点确定。

 ## 边缘情况

 每个水果类型的单个出现会被自然地处理，因为极值折叠到相同的值，因此所有四个计算表达式都会减少到两个唯一点之间的相同曼哈顿距离。 

当两种水果类型占据重叠区域时，该算法仍然有效，因为它不假设不相交。 即使坐标重合，极值差异也会正确地产生零。 

分布高度倾斜的网格，例如一个水果出现在长对角线上，另一个水果聚集在角落，正是需要极值投影方法的情况，因为强力配对太慢，但最大值仍然来自这些结构的端点。
