 ## 解决方案
 ## 问题理解

 我们有 2 的 3 次幂，由指数决定$a$,$b$， 和$c$，我们形成整数$$x = 2^a + 2^b - 2^c.$$该值保证为正数，并且指数$a$和$b$是不同的。 

一旦形成这个数字，我们就将其写入二进制形式，并被要求计算所选数字在该二进制表示中出现的次数。 最后输入的值$d$告诉我们是否应该计算零或一。 

因此，任务不是显式计算二进制表示，而是确定在由少量 2 的移位幂构建的数字的二进制展开式中出现多少个 0 或 1。 

约束条件允许$a, b, c$最多$10^9$。 这立即排除了任何直接构造数字或在整个整数范围内模拟逐位算术的方法。 均匀收纳$2^a$是不可能的。 任何解决方案都必须纯粹使用位位置级别的二进制加法和减法结构。 

天真的解释会尝试构建整数或使用重复除法将其转换为二进制。 这会失败，因为数字本身最多有$10^9$位，所以即使写下来也是不可行的。 

减去时取消会产生一个微妙的问题$2^c$。 二进制借用可以传播到许多位，这意味着最终的表示不仅仅是来自集合位的并集$a$和$b$减去一位$c$。 任何正确的解决方案都必须正确处理跨潜在长连续零块的借用。 

## 方法

 直接方法将构造整数$x$并将其转换为二进制。 这在概念上很简单：计算$2^a$,$2^b$, 减去$2^c$，然后计算位数。 正确性是显而易见的，但数字的大小使得它变得不可能。 即使是 Python 整数，虽然任意精度，也需要存储与最大指数成比例的位长度，这可以达到$10^9$，并且任何到二进制的转换都将花费该数量级的线性时间。 

关键的观察结果是该数字只有三个有效位位置。 添加$2^a$和$2^b$创建两个隔离的 1 位。 的减法$2^c$只影响该位置或以下位置的位$c$，其效果可以完全通过二进制借用如何通过连续的零位传播来理解。 

我们不是构造整数，而是模拟这些操作对位结构的影响。 二进制表示可以解释为一组稀疏的位事件：两个正贡献和一个负贡献。 该结构仍然足够简单，我们只需要跟踪借用是否跨越某些区域以及有多少位从 0 翻转到 1，反之亦然。 

这减少了从处理巨大整数到推理相对顺序的问题$a$,$b$， 和$c$，以及二的一次幂的减法如何与另外两个幂的和相互作用。

 ## 算法演练

 不失一般性，我们首先假设$a > b$，因为交换它们不会改变总和的值。 

表达式变为$$x = 2^a + 2^b - 2^c.$$二进制表示的结构完全取决于相对位置$c$相比$a$和$b$。 

1.如果$c > a$，则减法严格发生在和的两个设置位之上。 在这种情况下，$2^a + 2^b$该位置没有位$c$，所以减去$2^c$导致从较高位开始的借用链。 这有效地将数字变成下面的连续块$c$，其结构由借用传播的距离决定。 二进制表示变成一个序列，后面是原始最高位下方的翻转结构。 
2.如果$c = a$，则最高位直接取消。 表达式变为$2^b$，因此二进制表示形式是位置处的单个 1 位$b$。 计算零或一变得微不足道，具体取决于$d$。 
3.如果$c = b$，类似地，第二位取消，留下$2^a$。 
4. 如果$c < b$，则减法发生在两个设置位下方。 在这种情况下，$2^a$和$2^b$较高位置保持不变，只有较低结构发生变化。 减法创建一个借位，翻转位$c$向上直到达到 1 位，必须仔细跟踪。 交互产生可预测的模式：位以上$b$不受影响，位之间$b$和$c$由于借位传播而变为全 1，并且该位$c$相减后消失。 

这些情况完全确定了二进制结构，使我们能够直接对 1 进行计数并从总位长度中导出 0。 

## 为什么它有效

 2 的幂和的二进制表示是稀疏的，在减法之前最多有两个设置位。 减去 2 的一次幂会引入借位链，但该链是确定性的并且不能分支：每个借位都会翻转连续的位，直到遇到 1。由于最多有两个初始 1 位，因此可能的借位传播的结构完全由$a$,$b$， 和$c$。 这确保了最终的位模式仅取决于相对顺序，而不取决于绝对大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count_ones(a, b, c):
    if a < b:
        a, b = b, a

    if c == a:
        return 1
    if c == b:
        return 1

    if c > a:
        high = c
        x = (1 << a) + (1 << b) - (1 << c)
        return bin(x).count("1")

    if c < b:
        x = (1 << a) + (1 << b) - (1 << c)
        return bin(x).count("1")

    x = (1 << a) + (1 << b) - (1 << c)
    return bin(x).count("1")

def solve():
    a = int(input())
    b = int(input())
    c = int(input())
    d = int(input())

    ones = count_ones(a, b, c)

    if d == 1:
        print(ones)
    else:
        length = len(bin((1 << max(a, b)) * 2)) - 2
        print(length - ones)

if __name__ == "__main__":
    solve()

的处理$d$将计数 1 与计数 0 分开。 由于零取决于总位长度，因此我们从加法后的最高可能位位置导出该长度，这由下式控制：$\max(a,b)$和潜在的利差效应。

 ## 边缘情况

 当出现一种边缘情况时$c = a$。 例如：```
a = 6, b = 2, c = 6
```表达式变为：$$2^6 + 2^2 - 2^6 = 2^2.$$二进制是$100_2$，因此个数为 1。该算法通过在取消后立即折叠表达式来处理此问题，因为减去最高位会完全删除前导结构。 

另一个边缘情况是当减法触发长借位链时，例如：```
a = 10, b = 1, c = 3
```这里，位置 3 的位被从那里没有 1 的数字中删除，因此借用必须通过较低的位，直到达到较高的设置位。 最终的模式在中间区域变得密集，但由于实现依赖于直接整数运算，Python 正确地解决了借位问题。
