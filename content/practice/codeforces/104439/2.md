 ## 解决方案
 ## 问题理解

 该图片包含绘制在方形单元格上的一棵树。 

树干始终只有一个单元格宽，并垂直贯穿整棵树。 树有美丽`n`，这意味着正好有`n`树干两侧有树枝。 最低的分支有长度`n`，下一个有长度`n - 1`，依此类推，直到顶部分支，其长度为`1`。 

每对相邻分支之间都有一个没有分支的干细胞。 最低分支下方还有一个额外的主干细胞，最高分支上方还有一个额外的主干细胞。 

任务是计算有多少网格单元属于整个绘图。 

唯一的投入就是美丽`n`。 输出是总占用面积。 

的价值`n`可以大到`2 · 10^9`，所以构建图片是不可能的。 即使是循环的算法`n`次需要数十亿次迭代。 答案必须直接用数学公式计算。 该语句还警告结果不适合 32 位整数，因此实现必须依赖 64 位算术。 Python 整数已经支持任意精度。 

错误的常见来源是错误地计算树干高度。 例如，当`n = 1`，主干由底部单元、分支行和顶部单元组成，总共`3`细胞，不是`2`。 

另一个容易犯的错误是忘记每个分支都延伸到树干的两侧。 例如，当`n = 2`，分支行贡献`(2 + 2) + (1 + 1) = 6`分支细胞，不`3`。 

最小的输入也值得检查。 用于输入```
0
```根本没有分支机构。 只剩下单个干细胞，所以正确答案是```
1
```## 方法

 直接模拟将构建绘图的每一行，计算占用的单元格，并对它们求和。 这种方法是正确的，因为它完全遵循定义。 其运行时间为`O(n)`因为有`2n + 1`行，使其完全不切实际`n`达到二十亿。 

该图的结构非常规则，因此每个贡献都可以独立计算。 

树干在每一行中占据一个单元格。 有`n`分支行，`n - 1`连续分支之间的空行，最低分支下方一行，最高分支上方一行。 那是`n + (n - 1) + 2 = 2n + 1`trunk cells.

 分支贡献其两侧长度的总和：`2 × (1 + 2 + ... + n)`。 

使用等差数列公式，`2 × n(n + 1) / 2 = n(n + 1)`。 

将两部分相加得到`(2n + 1) + n(n + 1) = n² + 3n + 1`。 

整个计算被简化为评估一个多项式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取数值`n`。 
2. 计算干细胞数量为`2 * n + 1`。 这是树的高度，因为每一行都包含一个树干单元。 
3. 计算分支单元的数量为`n * (n + 1)`。 这等于总和的两倍`1 + 2 + ... + n`。 
4. 将两个数量相加并打印结果。 

### 为什么它有效

 每个被占用的单元格都属于两个类别之一。 

第一类是行李箱。 每一行只贡献一个主干细胞，因此主干贡献`2n + 1`细胞。 

第二类是分支机构。 长度的分支`k`贡献`k`左侧的单元格和`k`在右侧。 对所有分支长度求和产生`n(n + 1)`细胞。 

除了主干本身之外，主干和分支永远不会重叠，因为分支长度不包括主干单元，因此仅计算一次。 每个占据的单元格都被精确计数一次，因此总面积为`n² + 3n + 1`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
print(n * n + 3 * n + 1)
```其实现就是上面导出的封闭式公式。 Python自动处理大于64位的整数，因此即使是最大输入也不存在溢出问题。 

将表达式写为`n * n + 3 * n + 1`完全避免浮点运算。 每个操作都保留在整数算术中，消除了舍入问题。

 ## 边缘情况

 当输入为```
0
```该算法计算`0² + 3·0 + 1 = 1`。 不需要特殊处理，因为分支贡献自动变为零。 

当输入为```
1
```该算法计算一个分支行、一个底部主干单元和一个顶部主干单元，给出`3`干细胞。 各分支机构贡献`2`更多的细胞，产生`5`。 这解决了仅计算两行树干的常见错误。 

当输入为```
2
```分支贡献是`2 × (1 + 2) = 6`，这变成`2 × 3 = 6`通过公式`n(n + 1)`。 总计为`5 + 6 = 11`，确认树干两侧的树枝均已计数。
