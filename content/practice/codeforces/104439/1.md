 ## 解决方案
 ## 问题理解

 我们给出了三个长度。 其中两个描述了建筑中使用的原木的长度：一种原木具有长度`a`另一个有长度`b`， 和`a < b`。 第三个数字`c`是必须切割成块的单根木梁的总长度。 

构建过程严格按照重复层进行构建。 一层由五根原木组成：两根长度相同的长原木`b`和三个长度较短的原木`a`。 每一层都会使建筑物的高度增加一个单位。 这些层以交替模式堆叠，但问题保证每个完整层始终具有相同的成分。 该结构必须在顶部以完整的三层短原木结束，这意味着该结构必须由整数个完整层组成。 

每根原木必须从单根梁上切割而成`c`，并且切割消耗的长度恰好等于所需件数的总和。 层与层之间没有重复使用或剩余连接，因此唯一的限制是是否剩余足够的总长度来切割所有所需的原木。 

任务是确定可以从一根梁构建的完整层的最大数量。 

约束条件达到`10^18`，这立即排除了对单个单元的任何模拟或每单位长度的贪婪切割。 任何正确的解决方案都必须在每个测试用例的恒定或对数时间内工作，仅依赖于算术关系。 

一个微妙的边缘情况是，存在可以完成部分原木集但不是完整层的剩余材料。 例如，如果剩余长度足以容纳两个长原木但不足以容纳三个短原木，则该层无效并且必须被丢弃。 如果试图贪婪地交替切割而不考虑每层消耗固定的材料总量，则会出现另一种失败情况。 

## 方法

 一种简单的方法是逐层模拟构造。 对于每一层，我们从剩余长度中减去两个长原木和三个短原木的成本，如果成功，则增加高度计数器。 每层的成本是固定的：`2b + 3a`。 这使得正确性变得简单，因为每一层都是独立的并且成本相同。 

然而，如果我们认识到每一层消耗完全相同数量的材料，这种简单的模拟就变得不必要了。 如果我们定义`cost = 2b + 3a`，那么最大层数就是该成本的多少倍`c`。 关于顶层是三个短原木的最终约束不会引入不同的成本结构； 它只描述了组合顺序，而不是不同的消费规则。 

因此，问题简化为整数除法：计算可用梁长度适合多少个完整的五个对数组。 

关键的见解是层之间没有交互，不同类型的日志之间没有剩余的重用。 一旦我们观察到这一点，问题就会分解为单个除法运算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(c / (2b + 3a)) | O(c / (2b + 3a)) | O(1) | O(1) | 太慢了 |
 | 直接配方| O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取三个整数`a`,`b`， 和`c`。 这些代表短原木、长原木和总可用材料的固定尺寸。 
2. 计算构建一个完整层的总成本为`cost = 2 * b + 3 * a`。 这反映了单个完整结构单元的确切材料消耗。 
3. 计算通过整数除法可以形成多少个这样的层：`height = c // cost`。 需要整数除法，因为部分层不影响高度。 
4. 输出`height`作为答案。 

### 为什么它有效

 每个有效层都有相同且固定的组成，并且在跨层重新排列或部分重用材料时没有灵活性。 这将光束严格划分为大小不相交的段`2b + 3a`。 任何小于此数量的剩余材料都无法形成有效的层，因为即使缺少一根原木也会破坏结构要求。 因此，层数恰好是尺寸的完整段数`cost`适合`c`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a = int(input())
    b = int(input())
    c = int(input())
    
    cost = 2 * b + 3 * a
    print(c // cost)

if __name__ == "__main__":
    solve()
```该解决方案直接读取三个输入并计算每层成本。 关键操作是整数除法`c // cost`，这强制要求只计算完整的层。 

不需要循环或模拟，因为每一层都是独立的并且资源消耗相同。 

## 工作示例

 ### 示例 1

 输入：```
a = 3, b = 5, c = 29
```我们计算`cost = 2 * 5 + 3 * 3 = 10 + 9 = 19`。 

| 步骤| 剩余c | 行动| 层 |
 | --- | --- | --- | --- |
 | 1 | 29 | 29 构建一层（成本 19）| 1 |
 | 2 | 10 | 10 无法构建另一层 | 1 |

 剩下的10个单位不足以组成另一套完整的两长三短原木。 答案是`1`。 

### 示例 2

 输入：```
a = 1, b = 2, c = 100
```我们计算`cost = 2 * 2 + 3 * 1 = 4 + 3 = 7`。 

| 步骤| 剩余c | 行动| 层 |
 | --- | --- | --- | --- |
 | 1 | 100 | 100 构建层 | 14 | 14
 | 2 | 2 | 停止| 14 | 14

 自从`100 // 7 = 14`，我们获得 14 个完整层。 

这证实了剩余材料不会影响部分高度。

 ## 边缘情况

 常见的失败情况是剩余材料接近但不足以形成完整层。 例如：

 输入：```
a = 1, b = 2, c = 6
```这里`cost = 7`。 该算法计算`6 // 7 = 0`，正确拒绝形成部分结构的可能性。 尝试单独放置日志的贪婪实现可能会错误地放置两个长日志并在没有意识到完整层的情况下停止。 

另一个边缘情况是当`c`非常大，近`10^18`。 由于计算仅使用乘法和整数除法，Python 可以在本地处理此问题而不会溢出，而固定宽度整数语言必须确保 64 位算术。
