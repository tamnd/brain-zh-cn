 ## 解决方案
 ## 问题理解

 我们得到了一个简化的二十一点式游戏，涉及两个玩家：João 和 Maria。 每个玩家一开始都有两张牌，然后一张一张地展示一系列常见的牌。 每张牌的数值为 1 到 13，其中 1 计为 1 分，2 到 10 计为自身，11、12、13 各计为 10 分。 

在任何时刻，玩家的分数都是他们的两张初始牌加上迄今为止显示的所有普通牌的总和。 如果玩家的分数超过23分，则立即被淘汰。 如果玩家恰好达到 23，他们将立即获胜。 如果某一轮后只有一名玩家保持活跃，则该玩家获胜。 

我们对下一张通用卡感兴趣，该卡尚未公布。 我们必须确定下一张牌的最小可能值，以便玛丽亚在它被揭开时立即赢得游戏，或者确定不存在这样的值。 

关键是只有下一张牌才重要。 它后面的一切都是无关紧要的，因为这个问题要求一个条件，保证玛丽亚在抽到那张额外的牌后立即成为赢家。 

限制很小：最多已亮出 8 张普通牌，而且只有两名玩家。 这意味着对所有可能的下一张牌值及其效果进行强力推理是完全足够的，因为状态空间很小。 

一个微妙的边缘情况出现了，玛丽亚无法通过达到 23 分来获胜，而是在若昂超过 23 分后通过淘汰而获胜，而她仍然有效。 另一种情况是当两个玩家同时达到23时，在这种情况下玛丽亚仍然算作获胜者，所以我们不能错误地丢弃平局。 

一个幼稚的错误是只检查 Maria 是否能够准确达到 23，而忽略淘汰条件。 例如，如果 João 已经有 22，Maria 有 21，而下一张牌是 3，则 João 变为 25（被淘汰），Maria 变为 24（也被淘汰），这意味着双方都没有获胜。 所以我们必须仔细模拟两个玩家。 

## 方法

 暴力方法是尝试下一张牌的所有可能值，从 1 到 13。对于每个候选值，我们模拟揭示该牌的效果：更新两个玩家的总和，然后检查游戏结束规则。 

这是有效的，因为游戏状态完全由添加到两个玩家的总和中的单个整数决定。 对于每个候选值，我们只需要常量时间检查：João 或 Maria 是否超过 23，其中之一是否恰好达到 23，以及其中一个或两者是否保持活动状态。 

候选者总数为 13，每次模拟时间复杂度为 O(1)，因此速度相当快。 

关键的观察是，我们不需要考虑下一张牌之外的任何未来牌。 游戏结束条件仅取决于抽签后的即时总和。

 ## 算法演练

 我们首先使用两个玩家的两张初始牌和所有已经公开的普通牌来计算当前的总和。 我们将人头牌 11、12、13 视为值 10，将所有其他牌视为其数值。 

然后我们尝试从 1 到 13 的每个可能的下一张牌值。 

1. 计算如果将这张卡添加到 João 和 Maria 的总数中，她们的总数会是多少。 

同一张牌对两个玩家的影响是相同的，因为它很常见。 
2. 检查 João 是否被淘汰，即他的总和是否超过 23。 
3. 检查Maria是否被淘汰，即她的总和是否超过23。 
4. 检查 Maria 是否刚好达到 23。 
5. 确定在这种情况下玛丽亚是否是赢家：

如果玛丽亚没有被淘汰，并且她达到 23 岁，或者若昂在玛丽亚仍然活跃时被淘汰，则她获胜。 
6. 在所有满足玛丽亚获胜的值中，输出最小的该值。 如果都不起作用，则输出-1。 

重要的微妙之处在于必须同时检查消除和达到 23 点。 将两名玩家都推到 23 以上的值对玛丽亚没有帮助。 

### 为什么它有效

 翻开下一张牌后，如果满足获胜条件，游戏立即结束。 由于两名玩家的分数以相同的附加值确定性地演变，因此结果仅取决于每个总和是否超过两个阈值：23 和等于 23。彻底测试所有可能的卡牌值可以保证我们找到最小的有效卡值或正确地得出不存在的结论。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def card_value(x):
    return x if x <= 10 else 10

n = int(input())

joao = list(map(int, input().split()))
maria = list(map(int, input().split()))
commons = list(map(int, input().split()))

j = sum(card_value(x) for x in joao + commons)
m = sum(card_value(x) for x in maria + commons)

ans = -1

for c in range(1, 14):
    j2 = j + card_value(c)
    m2 = m + card_value(c)

    if m2 > 23:
        continue

    maria_wins = False

    if m2 == 23:
        maria_wins = True
    elif j2 > 23:
        maria_wins = True

    if maria_wins:
        ans = c
        break

print(ans)
```该解决方案首先将所有卡值标准化为其等值积分。 然后，它会计算两名玩家当前的总点数，包括已经亮出的普通牌。 每个候选下一张卡都按升序进行测试，因此第一个有效的卡自动是最小的。 

条件`m2 > 23`立即丢弃玛丽亚破产的无效案例。 获胜条件直接编码了 Maria 获胜的两种方式：恰好达到 23 分或在 João 破产时幸存。 

## 工作示例

 考虑这样一种情况，若昂 (João) 即将崩溃，而玛丽亚 (Maria) 稍稍落后。 

假设在下一张牌之前，João 的总点数为 22，而 Maria 的总点数为 20。 

我们模拟候选值：

 | 下一张卡 | 若昂总计 | 玛丽亚总计 | 若昂半身像| 玛丽亚半身像| 玛丽亚获胜 |
 | ---| ---| ---| ---| ---| ---|
 | 1 | 23 | 23 21 | 21 没有 | 没有 | 没有 |
 | 2 | 24 | 22 | 22 是的 | 没有 | 是的 |
 | 3 | 25 | 25 23 | 23 是的 | 没有 | 是的 |

 第一个有效值为 2，因为 João 超过 23 而 Maria 还活着。 

这证实了我们正确地捕获了基于消除的获胜条件，而不仅仅是完全相等。 

现在考虑两个玩家都接近的情况：

 | 下一张卡 | 若昂总计 | 玛丽亚总计 | 若昂半身像| 玛丽亚半身像| 玛丽亚获胜 |
 | ---| ---| ---| ---| ---| ---|
 | 3 | 25 | 25 24 | 是的 | 是的 | 没有 |

 即使 João 破产，Maria 也破产，所以这是无效的。 这确保我们正确拒绝不安全的获胜场景。

## 测试用例```python
import sys, io

def solve():
    import sys
    input = sys.stdin.readline

    def val(x):
        return x if x <= 10 else 10

    n = int(input())
    j0 = list(map(int, input().split()))
    m0 = list(map(int, input().split()))
    c = list(map(int, input().split()))

    j = sum(val(x) for x in j0 + c)
    m = sum(val(x) for x in m0 + c)

    for x in range(1, 14):
        j2 = j + val(x)
        m2 = m + val(x)
        if m2 > 23:
            continue
        if m2 == 23 or j2 > 23:
            print(x)
            return
    print(-1)

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    sys.stdout = out
    solve()
    return out.getvalue().strip()

 ## 边缘情况

 当 Maria 正好达到 23，但 João 在同一步骤中也超过 23 时，就会出现一种边缘情况。 例如，如果玛丽亚是22，若昂是22，下一张牌是1，玛丽亚就变成23，若昂就变成23。在这种情况下，玛丽亚仍然会立即获胜，因为无论若昂的状态如何，到达23就足够了。 

另一种边缘情况发生在 Maria 幸存但 João 破产时，即使 Maria 没有达到 23。如果 Maria 是 20，João 是 22，下一张牌是 3，Maria 变成 23，João 变成 25。这是一场胜利，它也与精确 23 条件重叠，但逻辑必须允许两种解释一致。 

第三种边缘情况是两名玩家都出局。 如果玛丽亚是22，若昂是22，下一张牌是2，则都超过23。即使若昂被淘汰，这也不能算是胜利，因为玛丽亚也同时被淘汰。
