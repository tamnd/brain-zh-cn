 ## 问题理解

 我们正在处理排列成圆圈的一系列元素。 循环移位操作将所有元素向左或向右移动，环绕两端，这样就不会丢失任何内容。 在执行几次这样的转换后，我们被要求确定序列的最终排列，或者回答有关某些元素最终在哪里的查询。 

在大多数变体中，输入由初始数组和一些移位操作或随时间累积的总移位距离组成。 输出是应用所有移位后的最终数组，或者是所有转换后查询位置的值。 

关键的结构观察是循环移位不会改变元素的相对顺序，只会改变它们的绝对索引。 这立即意味着整个过程可以简化为跟踪单个偏移值，而不是物理移动元件。 

从复杂度的角度来看，如果数组大小达到 10^5 或 10^6，并且可能有多达 10^5 次操作，那么任何实际执行每个操作的数组旋转的解决方案都太慢。 每次旋转的时间复杂度为 O(n)，最坏情况下为 O(nq)，这远远超出了可行的极限。 因此，我们被迫隐式地表示数组。 

当总移位计数大于数组大小时，会出现常见的边缘情况。 例如，如果我们将长度为 5 的数组向右旋转 12 个位置，则有效移位为 12 mod 5 = 2。不减少模 n 的简单实现可能会错误地过度旋转或浪费时间执行冗余循环。 

当允许负偏移时，会出现另一种微妙的情况。 例如，左移 k 相当于右移 n - (k mod n)。 符号标准化处理不正确会导致索引错误或差一错误。 

## 方法

 蛮力方法明确地模拟每个循环移位。 对于每一个操作，它都会逐个移动元素或使用切片重建数组。 这是正确的，因为它确实遵循操作的定义。 然而，每个班次的成本为 O(n)，并且进行 q 次操作后，总成本变为 O(nq)。 对于大输入，这会导致大约 10^10 次操作，这是不可行的。 

关键的见解是循环移位在加法模 n 下形成群结构。 每个移位都可以表示为一个整数位移，多个移位只是累加。 我们不修改数组，而是维护一个整数偏移量来描述数组当前的“开始”位置。 移位后访问第 i 个元素就变成了使用模运算将其映射回原始索引的问题。 

这减少了从重复数据移动到恒定时间索引算术的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(nq) | O(n) | 太慢了 |
 | 最佳（偏移跟踪）| O(n + q) | O(n) | 已接受 |

 ## 算法演练

 我们假设一个长度为 n 的数组和一系列移位操作。

1. 读取数组并将其存储在标准列表中。 此阶段不应用任何转换，因为我们希望保留原始排序作为参考状态。 
2. 将偏移变量初始化为零。 该变量表示数组的逻辑起点相对于原始数组中的索引 0 旋转了多远。 右移会增加此偏移，而左移会减少此偏移。 
3. 对于每个循环移位操作，相应地更新偏移量。 如果向右移动 k，我们将 k 添加到偏移量中。 如果它向左k，我们减去k。 每次更新后，我们使用模 n 标准化偏移量，使其始终保持在 [0, n-1] 范围内。 这种归一化确保偏移量不会无限制地增长，并且作为索引移位仍然有意义。 
4. 处理完所有操作后，数组物理上没有修改，但逻辑起点发生了移动。 最终数组中逻辑上位于位置 i 的元素对应于原始位置 (i - offset + n) mod n。 
5. 如果问题要求完整的最终数组，我们通过迭代从 0 到 n-1 的所有位置 i 并使用计算公式将每个位置映射回原始数组来构造它。 
6. 如果问题提出查询，则通过应用相同的索引转换在 O(1) 内回答每个查询。 

### 为什么它有效

 正确性依赖于偏移量始终代表应用于阵列的总净旋转的不变性。 每个循环移位与先前的移位相加，并且模运算确保环绕下的等效性。 由于数组元素永远不会显式重新排序，因此最终索引和原始索引之间的映射函数在所有操作中保持一致。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))

    offset = 0

    for _ in range(q):
        typ, k = input().split()
        k = int(k)

        if typ == 'R':
            offset = (offset + k) % n
        else:
            offset = (offset - k) % n

    res = []
    for i in range(n):
        res.append(a[(i - offset) % n])

    print(*res)

if __name__ == "__main__":
    solve()
```实现的核心是offset变量。 我们没有物理地旋转列表，而是将其视为固定的循环结构，并通过算术重新解释索引。 即使偏移量变为负值或超过 n，模运算也能确保正确的环绕。 

仅当输出需要完整阵列时才需要最后的重建步骤。 在基于查询的变体中，每个查询直接应用相同的映射公式。 

一个常见的实现陷阱是忘记 Python 的负模仍然会产生有效的正余数，但只有在仔细放置括号之后`(i - offset) % n`。 另一个问题是未能在每次操作后标准化偏移，这可能会导致较大的中间值，但如果一致应用模数，则不会影响正确性。

 ## 边缘情况

 一种重要的边缘情况是移位幅度远大于数组大小。 在这种情况下，在没有自动标准化的语言中，未能减少模 n 会导致不必要的积累和可能的逻辑错误。 

另一种边缘情况是当 n 等于 1 时。在这种情况下，每个循环移位实际上都是空操作，并且任何不考虑模行为的不正确索引算术仍然可能在不太仔细的实现中产生错误结果。 

第三种情况是重复交替移位，例如重复向右移动k，然后重复向左移动k。 偏移量应返回零，这证实了该算法在模算术下正确地将移位视为加法逆元。
