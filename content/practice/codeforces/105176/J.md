 ## 解决方案
 ## 问题理解

 我们得到了一组石头，每块石头都有一个正整数重量。 该过程反复取出两块最重的石头，将它们摧毁，如果它们的重量不同，则会产生一块新石头，其重量等于两者的差值。 这样一直持续到最多剩下一块石头为止。 任务是确定最终剩余的重量，如果没有留下石子，则重量为零。 

输入表示初始多重石块重量集。 输出是一个整数，描述重复应用“粉碎两个最大”操作后最后幸存的权重。 

问题的结构立即表明，主要困难不是算术，而是重复选择两个最大元素。 如果最多有 n 个石头，则每次扫描整个列表以查找两个最大元素的简单模拟每次操作的成本为 O(n)。 由于每次操作都会将石子数量减少 1 个，因此操作总数为 O(n)，这在最坏情况下会导致 O(n²) 时间。 当 n 很大（例如 10⁵）时，这太慢了，此时二次行为将远远超出可行的限制。 

当最终结果为零时，会出现微妙的边缘情况，因为所有棋子都会相互抵消。 例如，如果输入为 [2, 7, 4, 1, 8, 1]，则该过程以一颗重量为 1 的石子结束。但如果输入为 [1, 1]，则两块石子都会被销毁，没有任何剩余，因此正确答案为 0。在这种情况下，如果不检查空性而返回最后计算的差值，则可能会错误地输出未初始化或过时的值。 

## 方法

 暴力模拟维护石子列表并重复扫描它以找到两个最大的元素。 删除它们后，它要么插入它们的差异，要么什么都不插入。 这种方法是正确的，因为它完全反映了所描述的过程。 失败点是性能：每次提取前两个元素的成本为 O(n)，并且这种情况发生 O(n) 次，导致总工作量为 O(n²)。 对于大量输入，这是不可行的。 

关键的观察是该过程只需要快速访问当前的最大元素。 除了重复提取最大值之外，不需要维持顺序。 这正是最大堆（优先级队列）的用例。 通过将所有石头存储在堆中，我们可以在 O(log n) 时间内提取两个最大的元素，并在 O(log n) 时间内插入一个新值。 这将总复杂度降低到 O(n log n)，这足以满足典型的约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力扫描 | O(n²) | O(1)-O(n) | O(1)-O(n) | 太慢了 |
 | 最大堆模拟 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们使用最大堆对系统进行建模，这使我们始终能够有效地检索当前最大的两个石头。 

1. 将所有石头重量插入最大堆中。 如果使用最小堆实现，我们会反转值，因为大多数标准库仅提供最小堆。 
2. 当堆中有多于一块石头时，移走最大的石头 x。 这代表了当时最重的可用石头。 
3. 移走第二大石头 y。 此时，我们正在模拟两个最强候选者所需的成对操作。 
4. 如果 x 和 y 不相等，则计算它们的差值 x − y 并将其推回堆中。 如果它们相等，则两块石头都会被破坏，并且不会重新插入任何东西。 
5. 继续，直到堆的大小为 0 或 1。如果剩余一个元素，则该值就是答案。 如果为空，则答案为 0。

这种贪婪提取有效的原因是该问题从不依赖于非最大元素的相对顺序。 只有两个最大值会影响每个步骤的结果，并且删除它们不需要重新考虑之前的决策。 堆确保我们在每一步都严格按照当前全局最大值的定义来应用操作。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    n_and_rest = sys.stdin.read().strip().split()
    if not n_and_rest:
        return
    n = int(n_and_rest[0])
    arr = list(map(int, n_and_rest[1:]))

    # use max heap via negatives
    heap = [-x for x in arr]
    heapq.heapify(heap)

    while len(heap) > 1:
        x = -heapq.heappop(heap)
        y = -heapq.heappop(heap)

        if x != y:
            heapq.heappush(heap, -(x - y))

    if heap:
        print(-heap[0])
    else:
        print(0)

if __name__ == "__main__":
    solve()
```实现依赖于Python`heapq`，这是一个最小堆，因此所有值都存储为负数以模拟最大堆。 循环条件`len(heap) > 1`确保我们只在配对存在时才尝试提取它们。 最后的检查显式处理空堆情况，这对于所有石头抵消的输入是必需的。

 ## 边缘情况

 一个关键的边缘情况是所有棋子完全抵消。 用于输入`[1, 1]`，算法将两个值推入堆中，弹出它们，发现它们相等，并且不推回任何值。 堆变空，最终输出正确变为零。 

另一个微妙的情况是，当中间的还原重复产生较小的石头时，这些石头稍后会再次相互作用。 例如`[5, 4, 3]`首先产生`5-4=1`， 然后`[3, 1]`产生`2`，最后答案是`2`。 堆确保这些新创建的值始终在正确的时间重新考虑，因为它们重新进入相​​同的全局结构并与现有的石头平等竞争。
