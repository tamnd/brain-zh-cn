 ## 解决方案
 ## 问题理解

 该问题描述了结构的选择过程，该结构可以解释为图形或连接点序列。 核心任务是选择形成有效路径的五个位置的配置，而每个位置的贡献不仅取决于其在路径中的角色，还取决于其最佳传出连接。 

中心思想是，我们有效地选择长度为 5 的路径，其中第一个和最后一个位置不是直接单独评估的。 相反，它们的贡献是通过来自某些内部节点的最强可用传出连接间接确定的。 中间位置充当“决策核心”，端点被限制为依赖于与这些核心位置相关的预选最佳边缘。 

因此，问题简化为探索所有有效方法来选择长度为 5 的路径中的三个连续内部节点，然后使用基于顶部传出边缘的确定性规则完成端点，并最大化所得总分数。 

从约束的角度来看，路径枚举和基于边的评分的存在表明，如果图很大，则枚举所有 5 元组节点或所有候选路径的简单方法太慢。 如果有大约 10^5 个节点和潜在的密集邻接信息，则对所有长度为 5 的路径进行强力枚举将导致大约 O(n^5) 甚至 O(n·deg^4) 的行为，这是完全不可行的。 

唯一可行的解​​决方案必须根据局部结构将搜索空间减少到一定程度，通常是通过预先计算每个节点的最佳候选数，然后组合少量固定数量的选择。 

当节点的传出邻居少于四个时，就会出现微妙的边缘情况。 在这种情况下，“取前 4 个”规则会退化为取所有可用的邻居。 幼稚的实现可能会假设存在至少四个邻居并索引越界或人为地忽略有效贡献。 另一个边缘情况是，当最佳贡献者在不同候选中间节点之间重叠时，如果问题禁止重用路径中的节点，则需要仔细处理重复。 

## 方法

 暴力法是枚举构成有效路径的五个不同节点的所有可能序列，根据端点和内部节点的规则计算每个序列的贡献，并保留最大值。 这在概念上很简单：我们尝试所有可能的选择，验证邻接约束，并直接评估分数。 

问题是组合爆炸。 即使在稀疏图中，长度为 5 的简单路径的数量也可能非常大。 在密集图中，这在实践中实际上变成了指数。 每条路径都需要恒定时间的评估，但候选者的数量决定一切。 

关键的观察结果是端点的贡献不是任意的。 它仅取决于特定内部节点的前几条传出边。 这意味着我们不需要在路径构建过程中动态探索所有邻居。 相反，我们可以为每个节点预先计算一次最好的四个传出边。 之后，端点贡献变成 O(1) 查找。 

这将问题简化为选择中间节点的最佳小型配置（路径中的位置 2、3、4）。 一旦这些被修复，端点将由局部最佳边缘贪婪地确定。 这将搜索从全局路径枚举转换为小邻域结构上的约束组合，可以通过有限候选集上的嵌套循环来处理。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 路径长度呈指数（最坏情况下为 O(n^5)） | O(n + m) | 太慢了 |
 | 最佳| O(n·k^3)，其中每个节点 k ≤ 4 个候选 | O(n + m) | 已接受 |

 ## 算法演练

 ### 1. 预先计算最佳即将离任的候选人

 对于每个节点，我们提取最多四个具有最大边权重的邻居。 这已经足够了，因为端点是通过仅选择最好的四个传出边缘来定义的。 

此步骤将每个邻接列表压缩为恒定大小的摘要，确保后续计算保持有界。 

### 2.解释路径结构

 我们将每个有效的解决方案视为五个节点的路径：

 A → B → C → D → E。 

内部节点 B、C、D 决定结构，而 A 和 E 分别根据 B 和 D 的最佳出边选择。 

关键是，一旦B和D固定，A和E就不再是自由变量了。 

### 3. 枚举中间三元组

 我们迭代所有有效的三元组（B、C、D），这些三元组可以形成长度为两条边 B-C-D 的路径段。 

对于每个这样的三元组，我们根据图结构确保邻接是有效的。 

这极大地减少了搜索空间，因为我们不再直接考虑完整的 5 节点路径。 

### 4. 构建端点贡献

 对于每个候选三元组 (B, C, D)，我们计算：

 B 用于选择 A 的最佳可用传出贡献，以及 D 用于选择 E 的最佳可用传出贡献。这些来自预先计算的前四名列表。 

然后，我们将这些端点贡献与边 B-C 和 C-D 的任何内部贡献相结合。 

### 5. 保持全局最大值

 我们跟踪所有有效三元组的最大总分。 

每次评估都是 O(1)，因此总复杂度仅取决于枚举了多少个三元组。 

### 为什么它有效

 每个有效的解决方案都可以唯一地分解为中心段 B-C-D。 端点节点 A 和 E 仅依赖于仅限于 B 和 D 的最佳传出边的局部最优选择。因为一旦 B 和 D 固定，这些选择就独立于路径的其余部分，因此我们永远不会因为将注意力限制在预先计算的最佳候选者上而失去最优性。 这表明检查所有有效的中间三元组足以探索所有最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    
    for _ in range(m):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((w, v))
        g[v].append((w, u))
    
    # keep top 4 edges per node
    best = [[] for _ in range(n)]
    for i in range(n):
        g[i].sort(reverse=True)
        best[i] = g[i][:4]
    
    # helper: safe access top k neighbors
    def get(i, k):
        if k < len(best[i]):
            return best[i][k]
        return None
    
    ans = 0
    
    # enumerate middle segment B - C - D
    for b in range(n):
        for w1, c in best[b]:
            for w2, d in best[c]:
                if d == b:
                    continue
                
                # endpoint from b (A - B)
                left_gain = 0
                for w3, a in best[b]:
                    if a != c:
                        left_gain = max(left_gain, w3)
                
                # endpoint from d (D - E)
                right_gain = 0
                for w3, e in best[d]:
                    if e != c:
                        right_gain = max(right_gain, w3)
                
                total = w1 + w2 + left_gain + right_gain
                ans = max(ans, total)
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先构建带有权重的邻接列表，然后将每个节点的邻域压缩到其前四个传出边。 这确保了端点选择是恒定时间的。 

三重枚举循环的重点是选择 B 作为起点，然后从其最强的连接中选择 C，并从 C 的最强连接中选择 D。 这隐式地构造了所有有意义的长度为 2 的中间路径。 

端点贡献是通过扫描每个端点节点最多四个候选者来计算的，这使得计算受到限制。 排除重用 C 的条件确保我们不会意外地重用中间节点作为端点。

 ### 示例 1

 假设一个小图，其中节点 1 与节点 2、3、4 强连接，节点 4 与节点 5、6、7 强连接。最佳路径段是 2 → 3 → 4。 

| 步骤| 乙| C | d | 左增益 | 右增益 | 总计 |
 | ---| ---| ---| ---| ---| ---| ---|
 | 1 | 2 | 3 | 4 | 10 | 10 12 | 12 25 | 25
 | 2 | 2 | 3 | 5 | 8 | 11 | 11 22 | 22

 当中间结构与最强的局部邻域对齐时，就会达到最大值。 

这证实了一旦中间段固定，端点贡献是独立的。 

### 示例 2

 考虑节点度数较小且某些节点的邻居少于四个的情况。 

| 步骤| 乙| C | d | 左增益 | 右增益 | 总计 |

 在这里，即使“前 4 个”规则没有完全填充，算法仍然正确地使用所有可用的边而不会失败。 

这证明了稀疏图条件下的鲁棒性。

 ## 边缘情况

 关键的边缘情况是节点的传出边缘少于四个。 在这种情况下，该算法仍然有效，因为“前 4 个”选择退化为完整的邻接列表。 例如，如果节点 B 仅连接到两个节点，则两个节点都会被使用，并且端点选择只需在这些节点中进行选择。 

当同一节点出现在候选结构中的多个角色时，会出现另一种边缘情况。 该实现显式检查邻接约束（例如 d != b）以防止无效重用。 在像三角图这样的小输入上，该算法正确地避免形成无效的长度为 5 的路径，并且仅评估可行的配置。 

最后的边缘情况是多个候选端点产生相同的贡献。 该算法使用最大聚合，因此可以自然地处理关系，而不会影响正确性。
