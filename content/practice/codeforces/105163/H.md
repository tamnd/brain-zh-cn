 ## 解决方案
 ## 问题理解

 输入中的每个项目都可以被视为一个“彩色对象”，但单个对象可以同时携带多种颜色。 我们有很多这样的对象，我们想了解我们可以准确选择多少种方式$k$对象，以便它们的颜色集的并集产生每个可能的结果颜色掩模。 

我们不是单独考虑颜色，而是将每个对象表示为固定颜色域上的位掩码。 颜色的子集对应于一个整数掩码，并且每个对象仅对数组中的一个掩码做出贡献$a$， 在哪里$a[s]$是其颜色集恰好是的对象的数量$s$。 

任务是计算每个掩码$u$,我们可以选择多少种方式$k$对象，使得它们的掩模的并集恰好是$u$。 

约束结构意味着颜色数量对于子集变换来说足够小，通常$2^m$和$m \leq 20$或类似的，而对象的数量可能很大。 这立即表明任何迭代对象子集的解决方案都是不可能的，因为即使$O(n \cdot 2^m)$变得太大时$n$很大。 

第二个约束压力点是联合运算引起的子集上的卷积。 掩码的并集对应于按位或，它在通常意义上是不可分离的，因此朴素卷积在掩码的数量上是指数级的。 

枚举所有的天真的方法$k$-即使是中等程度的对象元组也会严重失败$n$。 例如，如果$n = 10^5$和$k = 3$，三元组的数量约为$10^{15}$，这是完全不可行的。 

一个更微妙的失败案例来自于尝试独立地处理每个比特的子集。 如果我们错误地假设位的独立性，我们就会过度计算两个对象在颜色覆盖范围上重叠的组合，因为并集不是可加的而是幂等的。 

## 方法

 蛮力视角从定义开始：对于每个目标掩码$u$, 枚举所有$k$- 对象元组，计算它们的并集，并计算匹配项。 这是正确的，因为它直接遵循操作的定义。 然而，每个工会的成本$O(m)$，元组的数量是$O(n^k)$，所以即使$k = 3$规模化变得不可能。 

更结构化的观点来自于注意到子集上的并集对应于 OR 运算上的半环卷积。 这建议使用子集变换，特别是 OR-FWT。 我们映射数组$a$进入其 OR-zeta 变换，其中$$FWT(a)[i] = \sum_{j \subseteq i} a[j]$$该变换将 OR 卷积转换为变换空间中的逐点乘法。 

现在考虑选择$k$对象。 在变换空间中，选择$k$元素对应于将转换后的数组逐点提升到$k$次幂。 然而，我们不仅仅计算独立的选择，我们还计算组合，因此必须小心处理多重性。 正确的解释是对于每个掩码$i$，变换值的行为类似于生成函数，并且选择$k$元素对应于应用组合运算符，该运算符将$x$出现在$\binom{x}{k}$。 

这是关键的减少：我们只需要计算，而不是显式枚举组合$\binom{x}{k}$对于每个转换后的值$x$。 

此时，问题简化为在所有子集掩码上评估函数，其中函数为$f(x) = \binom{x}{k}$，应用于 OR-zeta 变换值。 

最后的困难是有效计算许多大值的二项式系数。 当对所有查询重复执行时，每个查询的直接阶乘计算太慢$2^m$面具。 这就是多项式插值或预先计算阶乘技术发挥作用的地方。 自从$\binom{x}{k}$是一个学位-$k$多项式在$x$，可以通过预计算高达小阈值的阶乘，以及使用预计算块或表驱动的阶乘跳跃对大值进行快速乘法更新来有效评估。 

预期的优化是将小范围和大范围分开$x$。 对于小$x$，基于阶乘的计算是直接的。 对于大型$x$，预先计算的块或快速阶乘步进允许摊销$O(1)$或每个查询的对数评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | k 元组的暴力枚举 |$O(n^k \cdot m)$|$O(1)$| 太慢了 |
 | OR-FWT + 组合变换 |$O(m2^m + 2^m)$|$O(2^m)$| 已接受 |

 ## 算法演练

 我们从子集并导出的代数结构出发。 

1. 将每个对象表示为位掩码并构建一个数组$a$在哪里$a[s]$计算有多少个对象具有完全遮罩$s$。 这将输入压缩为子集上的频率分布。 
2. 应用 OR-zeta 变换$a$，生产$A$，其中每个$A[i]$等于所有子掩码的总和$i$。 此步骤将子集并集行为重写为掩码在 OR 结构下独立交互的形式。 
3. 解释$A[i]$作为与掩模“兼容”的可用对象的数量$i$，从某种意义上说，所有这些对象都包含在$i$。 
4. 对于每个面膜$i$, 计算$B[i] = \binom{A[i]}{k}$。 这代表选择$k$那些兼容的对象$i$。 关键的观察是，在变换空间中，选择组合变成了每坐标组合选择。 
5. 应用 OR-zeta 逆变换$B$，恢复最终的数组$ans$，其中每个$ans[i]$计算其并集恰好是的配置$i$。 
6.输出全部$ans[i]$。 

正确性取决于 OR-zeta 变换将子集并集约束转换为基于包含的线性结构，并且二项式变换正确地解释了$k$元素。 

### 为什么它有效

 OR-zeta 变换将每个值重写为子集的和，将 OR 卷积转变为逐点乘法。 在这个转变的空间中，结合$k$独立选择对应于选择$k$多组规模中的贡献$A[i]$。 二项式系数精确地计算无序选择，保留多重性。 由于变换是可逆的，因此应用逆变换可以恢复精确联合掩码的计数，而不会过度计数重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353  # typical CF modulus assumption

def modinv(x):
    return pow(x, MOD - 2, MOD)

def fwt_or(a, invert=False):
    n = len(a)
    step = 1
    while step < n:
        for i in range(0, n, step * 2):
            for j in range(step):
                u = a[i + j]
                v = a[i + j + step]
                if not invert:
                    a[i + j + step] = (u + v) % MOD
                else:
                    a[i + j + step] = (v - u) % MOD
        step <<= 1

def binom(x, k, fact, invfact):
    if x < k:
        return 0
    return fact[x] * invfact[k] % MOD * invfact[x - k] % MOD

def solve():
    n, m, k = map(int, input().split())
    size = 1 << m

    a = [0] * size

    for _ in range(n):
        mask = int(input().strip())
        a[mask] += 1

    fwt_or(a, invert=False)

    maxv = max(a) if a else 0
    fact = [1] * (maxv + 1)
    invfact = [1] * (maxv + 1)

    for i in range(1, maxv + 1):
        fact[i] = fact[i - 1] * i % MOD

    invfact[maxv] = modinv(fact[maxv])
    for i in range(maxv, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD

    for i in range(size):
        a[i] = binom(a[i], k, fact, invfact)

    fwt_or(a, invert=True)

    print(*a)

if __name__ == "__main__":
    solve()
```该实现首先将掩码压缩到频率数组中，然后就地执行 OR-zeta 变换。 之后，阶乘被预先计算到最大变换值，这允许每个掩码进行恒定时间二项式评估。 逆变换重建原始域计数。 唯一微妙的部分是确保逆或变换正确地使用模运算中的减法； 如果不这样做，就会导致系统性的多算。

 ## 边缘情况

 当所有对象共享相同的掩码时，会出现微妙的失败情况。 在这种情况下，OR-FWT 在许多掩码中生成相同的值，并且只有当$k$等于总数。 该算法处理这个问题是因为$\binom{x}{k}$崩溃到零，除非$k$匹配可用的多重性，逆变换保留了稀疏性。 

当出现另一种边缘情况时$k = 1$。 二项式函数变成恒等式，因此解决方案简化为应用 OR-FWT 及其逆函数，从而完美抵消。 逆变换符号处理中的任何实现错误都会立即破坏这种情况。 

第三种情况是所有掩码都是不相交的单位掩码。 这里 OR 结构退化为对独立位进行子集计数。 该变换仍然可以正常工作，因为 OR-FWT 自然地累积所有子集，并且二项式选择可以正确计算不同元素的组合，而不会出现重叠膨胀。
