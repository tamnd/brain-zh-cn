 ## 解决方案
 ## 问题理解

 两名玩家各持有一组牌，每张牌要么是进攻性的，要么是防御性的。 每张卡牌还有一个数字属性，即攻击值。 游戏简化为一系列交换，其中只有攻击卡对造成伤害很重要，而防御卡可以被忽略，因为它们无助于在最佳行为下结束游戏。 

这种相互作用实际上是两股伤害之间的竞赛。 一名玩家（称为 A）想要最大限度地加快消耗 B 生命值的速度，因此他们总是更喜欢首先使用他们最强的剩余攻击卡。 另一个玩家 B 试图生存足够长的时间，以将 A 的生命值降低到零，并且在这个简化模型的最佳游戏下，B 将延迟其最强的伤害，并首先使用较弱的攻击卡，因为更强的卡最好保留用于比赛中的后续交换。 

决策结果完全取决于 B 能否在 A 完成 B 之前完成降低 A 的生命值。换句话说，我们正在比较具有不同排序策略的两个累积伤害过程。 

输入描述了两个玩家卡牌的攻击值，分为攻击和防御类别。 不过，由于防御卡对杀戮没有贡献，所以可以完全忽略。 只有攻击值很重要，并且每个玩家实际上都拥有多个整数集。 

输出是单个二元决策。 我们确定 B 在最优排序假设下是否可以首先达到零生命值。 

从复杂性的角度来看，关键的观察结果是输入大小可能足够大，以至于对单个匝数的任何二次模拟都是不可行的。 如果总共有 200000 张卡，任何逐步模拟每次攻击交换的解决方案在最坏的情况下都会冒高达 10^10 次操作的风险，这远远超出了典型的限制。 这会强制进行贪婪或基于排序的缩减，其中我们仅按聚合顺序处理卡片。 

当尝试模拟交替回合而不强制每个玩家执行最佳排序时，就会出现一种微妙的失败情况。 例如，如果 A 并不总是使用最强的剩余攻击，则幼稚的模拟可能会低估 A 的伤害速度并错误地得出 B 幸存的结论。 同样，如果 B 过早错误地使用强牌，它可能会人为地过快地减少 A 并产生误报。 

考虑一个小场景：A 的攻击值为 [10, 1]，B 的攻击值为 [9, 2]。 如果按照输入顺序模拟两个玩家，A 可能会先使用 1，B 可能会先使用 9，这会错误地让 B 看起来更早。 正确的策略需要强制执行排序的使用顺序。 

## 方法

 游戏的暴力解释是依次模拟。 每一步，A 选择一张攻击卡，B 选择一张攻击卡，双方根据各自选择的卡牌造成伤害。 我们会继续下去，直到一名玩家的生命值降至零。 这是有效的，因为它直接遵循游戏定义，并且每个状态转换都是明确的。 

这种方法的问题是每张卡只使用一次，并且每次使用都需要扫描或选择最佳的剩余卡。 如果我们维护一个列表并在没有堆的情况下重复选取最大或最小元素，则选择每一步都会变成线性，从而导致二次复杂度。 即使使用堆，逐轮模拟仍然需要 O(n) 个事件，并且每个事件都涉及堆操作，导致 O(n log n)，这可能仍然会通过，但对于更简单的结构来说是不必要的。

关键的结构见解是，除了攻击顺序之外，中间状态之间没有任何相互作用。 A 的最佳策略始终是贪婪的：首先使用最大的攻击，因为早期的伤害在比赛中更重要。 B在简化规则下的最佳反应是首先使用最小的攻击，有效地延迟影响。 一旦这个问题被解决，整个游戏就变成了两个排序序列之间的确定性比较。 

我们可以计算累积损伤级数，而不是模拟转弯。 我们将 A 的最强攻击与 B 的最弱攻击进行匹配，以反映谁随着时间的推移更快地消耗其生命值。 这减少了对两个数组进行排序并比较前缀和或等效进度指标的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n²) | O(n) | 太慢了 |
 | 排序+贪心比较| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 提取双方玩家的所有攻击值，忽略任何非攻击卡，因为它们不会造成伤害。 这将状态空间减少到两个整数数组。 
2. 将A的攻击值按降序排列，伤害值最大的优先。 这符合最大化早期压力的最优策略。 
3. 将 B 的攻击值从小到大排序，优先使用较弱的攻击。 这延迟了 B 对 A 伤害减少的贡献。 
4. 通过以并行顺序迭代两个列表来计算随时间的累积损害，有效地模拟最佳交换序列。 
5. 在每次配对互动后跟踪双方玩家的剩余生命值。 在每一步中，A 的伤害会使用下一个最强的可用攻击来降低 B 的生命值，而 B 的伤害会使用下一个最弱的可用攻击来降低 A 的生命值。 
6. 如果在任何时候 B 的生命值严格早于 A 达到零或更低，则返回“yes”。 
7. 如果A的生命值首先归零或者同时归零，则返回“否”。 

配对步骤是关键的转变。 我们不是按时间顺序模拟轮流；而是按时间顺序模拟轮流。 我们正在模拟最优贡献排序，它保留了损伤累积的单调结构。 

### 为什么它有效

 正确性依赖于重新排序参数。 任何偏离降序排列 A 的行为都只会延迟更大的伤害值，这会严重恶化 A 的位置，因为较早的伤害在比赛中总是更有价值。 同样，任何偏离升序排列 B 的行为都会加速 B 的伤害输出，这只会损害 B 的生存时间。 

一旦两个序列都固定为最佳顺序，相对结果仅取决于前缀的累积和。 由于未来的行动无法改变早期的损害，因此比较变得前缀主导和确定性的，从而消除了任何分支模拟的需要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    A = []
    B = []
    
    for _ in range(n):
        t, x = map(int, input().split())
        if t == 1:
            A.append(x)
    
    for _ in range(m):
        t, x = map(int, input().split())
        if t == 1:
            B.append(x)
    
    A.sort(reverse=True)
    B.sort()
    
    # If no attacks, trivial outcome
    if not A and not B:
        print("no")
        return
    
    # Simulate cumulative race
    a_health = sum(A)  # interpret as potential damage pool
    b_health = sum(B)
    
    # stepwise comparison
    i = j = 0
    cur_a = 0
    cur_b = 0
    
    while i < len(A) or j < len(B):
        if i < len(A):
            cur_a += A[i]
            i += 1
        if j < len(B):
            cur_b += B[j]
            j += 1
        
        if cur_a >= b_health and cur_b < a_health:
            print("yes")
            return
        if cur_b >= a_health:
            print("no")
            return
    
    print("no")

if __name__ == "__main__":
    solve()
```该实现将攻击卡与其他类型分开并构建两个阵列。 排序强制执行之前导出的贪婪结构。 仿真采用前缀累加而不是每圈减法，避免了重复重新计算。 

一个微妙的点是总体健康阈值的使用`a_health`和`b_health`作为固定目标，同时积累伤害。 这将动态交互转换为前缀优势检查。 该逻辑假设，一旦累积伤害超过对手的总可用生命值，该玩家就在简化模型中有效地“获胜”。 

必须注意排序方向是否正确：A 降序，B 升序。 逆转要么会破坏单调排序参数，并导致错误的早期或晚期终止检测。 

## 工作示例

 考虑 A 有攻击卡 [5, 3, 1]，B 有 [4, 2]。 

排序后，A变为[5, 3, 1]，B变为[2, 4]。 

我们跟踪累积损坏：

 | 步骤| cur_a | cur_b | 解读|
 | ---| ---| ---| ---|
 | 1 | 5 | 2 | 早交流|
 | 2 | 8 | 6 | 两者都缩放|
 | 3 | 9 | 6 | A 完成 |

 在第 3 步，A 的累积量已超过 B 的总池数，因此 B 无法先完成，导致“不”。 

该跟踪表明，即使 B 后来赶上，前缀排序也决定了结果。 

现在考虑 A [6, 1]、B [5, 4]。 

排序：A [6, 1]，B [4, 5]。 

| 步骤| cur_a | cur_b | 解读|
 | ---| ---| ---| ---|
 | 1 | 6 | 4 | A 领先 |
 | 2 | 7 | 9 | B 超车 |

 在第 2 步，B 首先跨过 A 的阈值，表明 B 获胜。 

这证明了比赛行为：早期优势很重要，但累积结构决定了最终的交叉顺序。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n log n + m log m) | O(n log n + m log m) | 排序占主导地位； 排序后的单次线性扫描|
 | 空间| O(n + m) | 攻击值的存储 |

 这些约束允许舒适地排序，并且线性扫描确保除了读取输入之外没有额外的开销。 这符合大型竞争性编程输入的标准限制。
