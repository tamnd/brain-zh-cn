 ## 问题理解

 该问题描述了涉及两名玩家（乔伊和格雷）的概率纸牌过程，其中最终预期的纸牌数量取决于花色的初始组成和结果之间的递归交互。 任务不是直接模拟抽牌，而是计算从乔伊手的不同“状态”之间的线性关系系统得出的期望值。 

从概念上讲，乔伊的手牌包含四种类型的牌，但其中只有两种对最终贡献很重要：黑桃和梅花。 剩下的牌只影响系统的基线期望，而不影响乔伊的边际贡献。 我们将黑桃的贡献建模为值 x，将梅花的贡献建模为值 y。 这些值不是独立的常数； 它们取决于过程的概率转换以及乔伊没有相关卡片时的预期基本结果。 

关键导出量是 E，即 Joey 没有手部效应时的预期基线值，在声明中给出的公式中可简化为 (a + c − 1)。 这充当重复中的固定偏移。 

然后，系统通过抽取不同花色的概率定义连接 x 和 y 的两个线性方程。 每个术语对应于过渡到不同的状态：保持在黑桃状状态，进入梅花状状态，或重置为增加期望成本的基线状态。 

最终答案是这些贡献的线性组合，并根据 Joey 初始手牌中黑桃和梅花的数量进行加权：

 最终 = E + x ×（黑桃数）+ y ×（梅花数）。 

约束没有明确说明，但由于存在具有线性方程的期望值系统，预期的解决方案必须在每个测试用例的恒定时间内运行。 任何对卡牌状态的模拟或 DP 都是不可行的，因为状态空间随着卡牌数量的增加而增长。 这强烈意味着需要直接求解线性系统。 

当线性系统退化时，就会出现边缘情况。 特别是，如果系数导致除以零或产生负的预期贡献，则问题陈述明确指出该解决方案应被视为无限。 在这种情况下，如果 Joey 没有黑牌，答案就塌陷为 E，否则就变成无穷大。 这反映出在不稳定的概率结构下预期收益的无限增长。 

简单的方法可能会尝试迭代期望传播或转换模拟，这会失败，因为系统具有循环依赖性并且在浮点迭代下不会可靠地收敛。 一个更微妙的错误是假设 x 和 y 之间独立，忽略将俱乐部价值转化为黑桃价值的耦合项。 

## 方法

 强力解释将尝试模拟卡片转换的过程。 每个状态都代表乔伊手牌的配置，并且转换将对具有相关概率的抽牌或解决牌进行建模。 虽然这在概念上很简单，但可能的状态数量随着卡片数量的增加而组合增长，使得这种方法即使对于中等输入大小也是不可行的。 如果有n张卡，配置的数量是指数级的，并且每次转换都需要重新计算期望值，导致运行时不可行。

关键的见解是该系统并不是真正的组合而是线性的。 每种牌类型都会对期望做出附加贡献，黑桃和梅花之间的相互作用可以完全通过两个未知数 x 和 y 来捕获。 一旦认识到这种减少，问题就变成了用从概率导出的系数求解 2×2 线性系统。 语句中的递归已经是一个定点方程，这意味着我们可以重新排列项并直接使用代数求解。 

唯一的复杂性是处理简并性。 如果系统的行列式为零或者求解产生负值，则过程不稳定，并且根据该语句，预期贡献被视为无限。 这将解决方案转换为具有后备情况的条件线性代数问题。

 ## 算法演练

 我们首先计算基线期望 E，它仅由问题中给出的非 Joey 贡献结构确定。 这个值代表了当乔伊没有受到特殊套装影响时的预期结果。 

接下来，我们导出两个未知数 x 和 y。 这些分别对应于黑桃和梅花的边际预期贡献。 我们将语句中给出的递推关系重写为标准线性形式：

 步骤 1：将两个方程乘以它们的分母以消除分数。 这将概率期望转换为代数约束。 

步骤 2：重新排列术语，使所有出现的 x 和 y 出现在左侧。 这产生了以下形式的线性系统：

 A₁x + B₁y = C₁

 A2x + B2y = C2

 步骤 3：使用标准消元法求解系统。 计算行列式 D = A₁B2 − A2B₁。 如果 D 为零，则系统是退化的，我们立即切换到无限情况规则。 

步骤 4：如果 D 非零，则使用 Cramer 规则计算 x 和 y。 这提供了代表预期贡献的精确有理值。 

步骤5：检查解决方案的有效性。 如果 x 或 y 为负，则将系统视为不稳定系统并切换到无限情况规则。 

步骤 6：如果有效，则将最终答案计算为 E + x·黑桃 + y·梅花。 

这样做的原因是，期望对于卡类型的不相交贡献是线性的，并且系统对状态之间的所有依赖关系进行编码。 一旦求解，无论排序如何，x 和 y 都保持一致，因为它们是基础马尔可夫期望方程的不动点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = list(map(int, input().split()))
    
    # unpack based on problem structure
    # a, b, c, d represent counts of different suits
    a, b, c, d = data
    
    # baseline expectation
    E = a + c - 1
    
    # coefficients derived from rearranging equations
    # equation system:
    # x = b/(a+b+c)x + (a+c)/(a+b+c)(1+E) + b/(a+b+c+d)x + d/(a+b+c+d)y
    # y = b/(a+b+c)x + (a+c)/(a+b+c)(1+E)
    
    s1 = a + b + c
    s2 = a + b + c + d
    
    # Avoid division by zero edge cases
    if s1 == 0 or s2 == 0:
        print(E)
        return
    
    # convert to linear system Ax = C
    # x - b/s1 x - b/s2 x - d/s2 y = (a+c)/s1 * (1+E)
    # y - b/s1 x = (a+c)/s1 * (1+E)
    
    A1 = 1 - b / s1 - b / s2
    B1 = -d / s2
    C1 = (a + c) / s1 * (1 + E)
    
    A2 = -b / s1
    B2 = 1
    C2 = (a + c) / s1 * (1 + E)
    
    det = A1 * B2 - A2 * B1
    
    # degeneracy check
    if abs(det) < 1e-12:
        print(E)
        return
    
    x = (C1 * B2 - C2 * B1) / det
    y = (A1 * C2 - A2 * C1) / det
    
    # invalid system handling
    if x < 0 or y < 0:
        print("INF")
        return
    
    # assume we are given counts of spades/clubs separately
    spades = b
    clubs = d
    
    ans = E + x * spades + y * clubs
    print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循代数约简。 第一步是提取计数并计算基线期望。 然后，通过将所有变量移至一侧，将递归转换为线性系统。 使用浮点运算是因为系统较小并且精度问题由简并检查来控制。 

一个微妙的点是决定性阈值。 使用小 epsilon 代替精确的零比较，因为当概率接近时可能会发生浮点抵消。 另一个重要细节是处理无效解：负x或y表示期望值模型在概率结构下崩溃，触发无限输出条件。

 ## 边缘情况

 一种重要的边缘情况是耦合 x 和 y 的所有系数都消失。 在这种情况下，系统简化为独立期望，行列式变为零。 该算法通过行列式检查检测到这一点，并回落到基线 E，防止被零除。 

当概率高度倾斜时，例如当 b 为零时，会出现另一种边缘情况。 在这种情况下，方程显着简化，x 完全依赖于 y。 求解器仍然可以正确处理这个问题，因为即使某些系数为零，线性系统约简仍然有效。 

最后的边缘情况是当计算的期望由于数值不稳定而变为负值时。 该算法明确地检查这一点并切换到无限情况，匹配问题的期望无限增长的定义。
