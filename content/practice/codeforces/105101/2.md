 ## 解决方案
 ## 问题理解

 给定整数坐标平面上的一组点，我们需要计算这些点有多少个不同的三元组形成直角三角形。 只要点不共线，每个三重点就定义了一个三角形，并且我们只对其中一个角度恰好为 90 度的三角形感兴趣。 

输入尺寸较小，最多 100 个点。 这已经表明，检查所有点的三元组是可行的，因为当 N 等于 100 时，三元组的数量最多约为 161700 个。任何 N 为三次方的解决方案仍将在限制内轻松运行。 

问题的微妙部分不是组合学而是几何学。 当且仅当其两条边垂直时，三角形才是直角三角形。 在坐标方面，这意味着我们需要检测共享公共端点的点对之间的垂直向量。 

一个天真的错误是尝试使用浮点算术或余弦比较来显式计算角度。 这就引入了精度问题。 另一个错误是检查所有三元组并尝试在没有一致结构的情况下计算点积，这会导致重复计数或在退化情况下丢失直角条件。 

朴素浮点角度检查的具体失败案例是形成完美直角但在计算或舍入中坐标略有扰动的点，例如期望 (0,0)、(1,0)、(0,1)。 斜率的浮点比较在这里仍然可以工作，但是推广到像 10^9 这样的大坐标会使斜率比较由于溢出或除以零处理而存在风险。 

第二个问题是计数过多：可以从三个顶点中的每一个来识别同一个三角形，因此粗心的方法可能会多次计数每个三角形。 

## 方法

 蛮力的想法很简单。 我们尝试所有三元组并检查三角形是否是直角。 对于每个三元组 (i, j, k)，我们计算点对之间的平方距离，并在所有三种可能的排列中应用毕达哥拉斯条件。 这是可行的，因为对于任何直角三角形，边的一个顺序恰好满足 a^2 + b^2 = c^2。 

这种方法是正确的，但其成本由三元组的数量决定，大约为 N^3 / 6。对于 N = 100，大约需要 160,000 次检查，并且每次检查都涉及持续的工作。 这已经是临界点，但在 Python 中仍然可以接受； 然而，我们可以通过关注固定顶点的直角来更清晰地构建解决方案，并避免重复的几何重新计算。 

关键的观察结果是直角是顶点的局部角。 如果三角形在 A 点有直角，则向量 AB 和 AC 垂直，这意味着它们的点积为零。 我们可以固定直角顶点 A 并计算有多少对其他点形成与 A 垂直的方向向量，而不是全局枚举三元组。 

对于每个固定的 A，我们计算所有其他点的向量，然后检查这些向量的所有对。 点积为零的每一对恰好贡献一个以 A 为直角顶点的直角三角形。 对所有 A 求和即可得到总计数，并且不会重复计算任何三角形，因为每个直角三角形都恰好有一个直角顶点。

 ## 算法演练

 我们依赖这样一个事实：每个直角三角形都有一个唯一的顶点，即直角所在的位置。

1. 迭代每个点 A 作为潜在的直角顶点。 这确保我们只计算 A 是形成 90 度角的角的三角形。 
2. 对于当前 A，计算从 A 到每隔一个点 B 的所有向量。每个向量表示为 (dx, dy) = (Bx - Ax, By - Ay)。 这将几何问题转换为向量代数。 
3. 对于每对向量 (dx1, dy1) 和 (dx2, dy2)，计算它们的点积 dx1 * dx2 + dy1 * dy2。 如果结果为零，则它们之间的角度为 90 度，这意味着 A、B、C 形成一个直角三角形，其中 A 为直角。我们对每个这样的对进行计数。 
4. 累加 A 的所有选项的计数。 

这样做的原因是欧几里得几何中的垂直度恰好是通过点积为零来捕获的。 通过将计算锚定在每个 A 点，我们将全局三角形条件转换为局部成对条件。 

### 为什么它有效

 每个直角三角形都有一个直角顶点。 对于该顶点 A，两条相邻边对应于两个不同的点 B 和 C，使得矢量 AB 和 AC 垂直。 点积测试准确地捕获了这些对，而不捕获其他对。 由于每个三角形在其直角顶点只贡献一次并且在其他地方从未计算过，因此总和是精确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    
    ans = 0
    
    for i in range(n):
        ax, ay = pts[i]
        vecs = []
        
        for j in range(n):
            if i == j:
                continue
            bx, by = pts[j]
            vecs.append((bx - ax, by - ay))
        
        m = len(vecs)
        for j in range(m):
            x1, y1 = vecs[j]
            for k in range(j + 1, m):
                x2, y2 = vecs[k]
                if x1 * x2 + y1 * y2 == 0:
                    ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```解以潜在的直角形式循环遍历每个点。 对于每个这样的点，它构造到其他点的所有方向向量。 向量对上的嵌套循环检查所有组合并使用点积条件来检测垂直度。 

一个微妙的实现细节是，我们只考虑对 j < k 以避免重复计算来自同一顶点的光线对。 另一个重要的一点是对点积使用整数运算，这完全避免了浮点精度问题。

 ## 边缘情况

 一个退化但重要的情况是所有点都位于一条线上。 在这种情况下，根本不存在三角形，因此答案必定为零。 该算法自然地处理这个问题，因为来自任何顶点的所有向量都是共线的，并且没有一对可以具有零点积，除非两个向量都是相反方向的标量倍数，这对于形成垂直的不同点来说不会发生。 

另一种情况是最小输入大小。 如果少于三个点，则无法形成三角形。 外部循环仍然运行，但内部向量列表的大小最多为 1，因此不存在对，并且答案仍然为零。 

最后一种情况是对称配置，例如正方形。 对于一个正方形，四个顶点中的每一个都恰好贡献一个直角三角形，对应于它的两条相邻边。 该算法在每个顶点独立地计算这些贡献中的每一个，产生正确的总数而不重复，因为每个三角形仅与其直角相连。
