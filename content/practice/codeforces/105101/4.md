 ## 解决方案
 ## 问题理解

 我们有几个独立的网格。 每个网格代表一个矩形蛋糕，其中每个单元格都包含一个整数值来描述其味道贡献。 我们必须从每个网格中选择一个正方形子矩阵（行数和列数相同，网格内的任何位置），以便该正方形内所有值的总和尽可能大。 

每个网格的输出正是可实现的最大平方和。 

关键的困难在于正方形的大小和位置都是未知的。 一个正方形的范围可以从 1 × 1 到 min(N, M) × min(N, M)，我们必须考虑所有的布局。 

这些限制以非常具体的方式发挥作用。 每个网格的大小可以达到 500 x 500，但所有网格的总高度也以 500 为界。这意味着，只有在我们小心处理常量的情况下，我们才能在最坏的情况下负担得起每次测试大约为 O(N³) 的算法，但我们不能承受像 O(N³M) 这样的算法，或者在没有预处理的情况下天真地枚举所有方块。 

一种简单的方法会尝试每个正方形大小、每个左上角位置，并从头开始计算其总和。 在 500 x 500 的情况下，这立即变得太慢了，因为仅正方形的数量就是 O(N²M²)，这远远超出了可行的范围。 

一些边缘情况在概念上很重要。 

除了一个正细胞外，网格可能大部分为负细胞。 例如：```
1 1
-5 -5
-5 10
```答案是 10，因为最好的正方形是 1 × 1。仅考虑大正方形的方法可能会忽略这一点。 

另一种情况是，即使完整网格包含许多正值，最佳平方不是完整网格：```
2 2
1 -100
1 -100
```如果允许使用矩形，最好的正方形还是 1 x 1 或垂直条带，但由于只允许使用正方形，因此我们必须正确比较所有尺寸。 

最后，混合符号网格需要仔细处理前缀和，因为即使逻辑正确，重复重新计算和也会溢出时间限制。 

## 方法

 暴力法很简单。 我们选择一个从 1 到 min(N, M) 的正方形大小 k。 对于每个 k，我们在网格上滑动一个 k × k 窗口，计算其单元格的总和，并跟踪最大值。 

为了简单地计算每个平方和，我们将对 k² 个单元求和，因此每次放置的成本为 O(k²)。 由于存在 O(NM) 种布局和 O(min(N, M)) 大小，如果直接完成，在最坏情况下总复杂度约为 O(NM·min(N, M)³)，这对于 500 x 500 网格来说是完全不可行的。 

关键的观察是我们需要的是快速子矩阵求和查询。 一旦可以在 O(1) 中计算任何矩形和，枚举所有平方就变得容易管理。 这正是 2D 前缀和所提供的：一个预处理步骤，将每个网格转换为一个结构，其中任何轴对齐的矩形和都在恒定时间内计算。 

对于前缀和，每个平方和为 O(1)，因此总工作量变为 O(NM·min(N, M))，这是临界值，但可以接受，因为所有测试用例的总 N 限制为 500。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每个尺寸为 O(NM·k²)，最高为 O(NM·min(N,M)³) | O(1) | O(1) | 太慢了 |
 | 前缀和+枚举| O(NM·分钟(N,M)) | O(NM) | 已接受 |

 ## 算法演练

 我们独立处理每个网格。 

1. 在网格上构建一个 2D 前缀和数组。 每个条目存储从 (1,1) 到 (i,j) 的子矩阵之和。 这允许我们在 O(1) 内查询任意矩形和。 这很有用的原因是所有候选方块都可以简化为矩形查询。 
2. 对于从 1 到 min(N, M) 的每个可能的正方形大小 k，我们考虑所有适合 k × k 正方形的左上角 (i, j)。 目标是快速评估该平方和。 
3. 对于每个这样的位置，使用前缀和公式计算总和。 这避免了迭代所有 k² 元素并降低了常数时间的成本。 
4. 追踪所有方格的最大值。 由于即使单个单元格也是有效的正方形，因此初始化必须通过从非常小的数字或第一个计算的正方形开始来安全地允许负值。 
5. 检查所有尺寸和位置后，输出最大值。 

关键的效率增益来自于用恒定时间查询替换重复求和，从而允许完全枚举几何图形而无需重新计算内容。 

### 为什么它有效

 每个方形子矩阵都唯一对应于一对参数：其左上角和边长。 前缀和结构确保任何此类平方和的计算都准确无误，没有近似或遗漏。 由于所有可能的方格都被枚举一次，因此该算法会评估每个可行的候选方格，因此不会错过最佳方格。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(n, m, grid):
    ps = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(n):
        row_sum = 0
        for j in range(m):
            row_sum += grid[i][j]
            ps[i + 1][j + 1] = ps[i][j + 1] + row_sum

    def get(x1, y1, x2, y2):
        return (ps[x2][y2]
                - ps[x1 - 1][y2]
                - ps[x2][y1 - 1]
                + ps[x1 - 1][y1 - 1])

    best = -10**18

    limit = min(n, m)
    for k in range(1, limit + 1):
        for i in range(1, n - k + 2):
            i2 = i + k - 1
            for j in range(1, m - k + 2):
                j2 = j + k - 1
                best = max(best, get(i, j, i2, j2))

    return best

def main():
    T = int(input())
    out = []
    for _ in range(T):
        n, m = map(int, input().split())
        grid = [list(map(int, input().split())) for _ in range(n)]
        out.append(str(solve_case(n, m, grid)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先构建标准的二维前缀和表。 每个条目 ps[i][j] 表示从左上角到 (i, j) 的矩形的总和。 该构造使用行累积来避免重复重新计算水平总和。 

get 函数对子矩阵和应用包含-排除公式。 这是边界处理唯一重要的地方。 从 1 开始的索引避免了对负索引的重复检查。 

枚举循环遍历所有正方形大小和所有有效位置。 嵌套循环是安全的，因为跨测试的总网格大小是有界的，因此累积迭代次数保持在限制范围内。 

一个常见的错误是错误地重新计算行总和或忘记包含正方形边界，这会将所有计算区域移动一个单元格。

 ## 边缘情况

 完全负网格可以正确处理，因为该算法仍然评估每 1 x 1 方格。 例如：```
2 2
-1 -2
-3 -4
```前缀和仍然计算出正确的值，并且最大值从各个单元格更新，产生 -1。 

处理最佳平方不是最大的网格，因为所有尺寸都是独立迭代的。 例如：```
3 3
10 -100 10
-100 -100 -100
10 -100 10
```该算法评估 1 x 1 角（值 10）和较大的正方形（强负值），并正确选择 10。 

由于前缀和是基于 1 的，边界处理是安全的。 当查询顶部或左侧边缘的正方形时，减法项会自动消失，因为前缀数组中的索引为零，而前缀数组已显式初始化为 0。
