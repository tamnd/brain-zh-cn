 ## 解决方案
 ## 问题理解

 魔术师的技巧将识别 1 到 100 之间的隐藏数字的任务简化为回答七个是或否问题的固定序列。 七张卡片中的每一张都包含该范围内的数字子集，观众为每张卡片指示他们选择的数字是否出现在上面。 

从求解器的角度来看，输入只是按顺序排列的七个二进制响应，每个响应对应一张预定义的卡片。 隐藏数字由包含该数字的卡片组合唯一确定，因此输出是与所有七个答案一致的单个数字。 

关键的结构约束是只有 100 种可能的结果，但每种结果都对应于卡片上不同的 7 位成员资格模式。 这立即意味着卡片必须将数字编码为二进制表示形式或近似变体。 只有 7 个“是”或“否”的响应，模式空间最多为 2^7 = 128，这可以轻松覆盖 1 到 100 的范围。这已经表明每个数字都映射到卡片的唯一子集，而重建是一个直接解码问题。 

天真的解释可能会尝试测试从 1 到 100 的每个数字并模拟它是否与给定的答案匹配。 这在限制范围内是安全的，因为它只有 100 次检查，但这是不必要的。 预期的结构是直接解码：每个“是”都为最终答案贡献固定的权重。 

假设任意卡片内容会产生微妙的边缘情况。 如果错误地假设这些集合是随机的，那么重建就会变得不明确。 例如，如果两个数字共享相同的成员资格模式，则输出将是未定义的。 该问题保证了正确的编码，因此每个模式都精确地映射到一个数字，从而避免了这种歧义。 

## 方法

 蛮力的想法很简单。 对于从 1 到 100 的每个数字，我们检查其在七张卡片上的存在模式是否与输入响应相匹配。 具体来说，对于每个候选数字，我们将模拟七张牌并与给定的是或否序列进行比较。 这需要扫描最多 100 个号码，每个号码检查最多 7 个条件，每次测试最多执行 700 次操作。 即使扩展，这仍然是微不足道的，但它隐藏了技巧的结构。 

关键的见解是七张卡以二进制形式对数字进行编码。 每张卡对应一个固定的位位置，“是”表示该位已设置在隐藏数字中。 我们没有检查所有候选者，而是通过将与“是”答案相关的两个的幂相加来直接重建数字。 

这将问题从搜索转变为解码。 输入的结构已经代表了答案的二进制表示，因此重建是对七个值的单次传递。

 ## 算法演练

 1. 为七个位置中的每一个分配一个权重，对应于一个二进制数字。 第一张牌代表 2^0，第二张牌代表 2^1，依此类推。 这反映了这样一个事实：每张卡都以隐藏数字的二进制表示形式编码一位。 
2. 将累加器变量初始化为零。 这将存储解释位时重建的数字。 
3. 按顺序迭代七个响应。 
4. 对于每个响应，如果为“是”，则将相应的 2 的幂添加到累加器中。 如果为“否”，则不执行任何操作，因为该位不存在于数字中。 
5. 处理完所有七个响应后，输出累加器作为重构的数字。 

### 为什么它有效

从 1 到 100 的每个数字都有一个最多使用七位的唯一二进制表示形式。 每张卡都精确对应于该表示中的一位位置。 “是”表示该位为 1，“否”表示该位为 0。由于二进制表示形式是唯一的，因此对活动位求和即可毫无歧义地重建原始数字。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    responses = input().split()
    
    ans = 0
    for i, r in enumerate(responses):
        if r == "Yes":
            ans += (1 << i)
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案读取七个响应并按顺序处理它们。 每个响应的索引确定其位位置。 当响应为“是”时，使用位移位将相应的2的幂相加。 最后的总和作为答案打印出来。 唯一微妙的细节是确保索引一致：第一个响应必须对应于最低有效位，否则重建的数字将不正确。

 ## 边缘情况

 一种重要的边缘情况是所有回答都是“否”。 该算法将累加器初始化为零，并且永远不会添加任何 2 的幂，从而产生 0。由于有效范围从 1 开始，这种情况通常表示无效输入或特殊编码情况，但计算保持一致。 

另一种情况是所有回答都是“是”。 累加器变为 1 + 2 + 4 + 8 + 16 + 32 + 64，等于 127。这验证了 7 位编码的上限是否已正确处理。 

第三种情况涉及分散的位，例如交替“是否是否是否是”。 该算法仅正确累加选定的 2 的幂，结果反映了精确的二进制组合，位置之间没有干扰。
