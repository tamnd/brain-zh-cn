 ## 解决方案
 ## 问题理解

 我们有一个简单的封闭栅栏，由其角点按顺时针顺序描述。 栅栏形成凸多边形，因此每个内角都小于 180 度，并且两个内点之间的每条线段都位于该区域内。 

在这个围栏区域内，我们想放置两座相同的塔。 每个塔必须远离围栏边界，我们用从塔到多边形最近边缘的欧几里得距离来衡量安全性。 如果一座塔距离栅栏任何一侧的距离都小于某个值 r，则认为它太近了。 

同时，两座塔不能距离太近。 他们的欧氏距离必须至少为 D。 

目标是选择两个有效的内部点，满足它们之间的距离约束，同时最大化从任一塔到栅栏边界的最小距离。 

输入按顺序给出多边形顶点和所需的间距 D。输出是一个实数：最大可能的安全距离 r，以便我们可以在多边形内部放置两个点，每个点距离边界至少为 r，并且仍然保持它们至少相距 D。 

约束 N 高达 100000 意味着任何尝试显式测试所有点对或所有候选放置的解决方案都是不可行的。 顶点上的二次方法或离散化内部是不可能的。 即使 O(N log N) 也是可以接受的，但是任何每对顶点重复大量几何图形的操作都会失败。 

当考虑几何收缩时，会出现一个微妙的问题。 如果我们将点向内推距离 r，则可行区域的形状会不断变化。 一种幼稚的方法可能假设我们可以独立选择原始多边形中的两个远点，然后减去 r，但这忽略了某些点在收缩后变得无效。 

另一种故障模式来自于忽略了收缩多边形可能会断开或严重减小其直径。 即使原始多边形中存在两个相距较远的点，在安全半径要求扩大后，它们也可能都距离边界太近。 

一个小的说明性陷阱是一个细长的三角形。 当 r = 0 时，两个远端点有效，但即使 r 很小，也会消除大部分可用区域，从而大大减少可实现的分离。 

## 方法

 最直接的想法就是猜测两座塔的位置，并尝试最大化它们的边界间隙。 对于多边形内的任何一对点，我们可以计算它们到边界的距离，并检查它们的相互距离是否至少为 D。这立即变得不可行，因为候选点的集合是无限的，甚至限制到顶点也是错误的，因为最佳点通常不是顶点。 

稍微更好的暴力视图是沿着边缘离散化候选者或对多边形内部进行网格化，但这会导致状态爆炸。 即使有 10^5 个顶点，将它们配对也会产生 10^10 种组合，这远远超出了限制。 

关键的结构观察是将问题分为两层。 首先，确定候选安全半径r。 那么有效区域就变成了到多边形边界的距离至少为r的所有点的集合。 对于凸多边形，该区域又是通过将每条边向内偏移距离r而获得的凸多边形。 

一旦我们有了这个缩小的多边形，问题就变成了纯粹的几何问题：这个多边形是否包含距离至少为 D 的两个点？ 这正是至少为 D 的多边形的直径。可以使用旋转卡尺在线性时间内计算凸多边形的直径。

这将问题转化为 r 上的单调决策问题。 如果半径r可行，则任何更小的半径也是可行的。 这种单调性允许对 r 进行二分搜索。 每次检查的成本为 O(N)，并且我们重复大约 50 到 60 次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力点对 | O(N^2) 或更糟 | O(1) | O(1) | 太慢了|
 | 二分查找+凸收缩+直径检查 | O(N 对数精度) | O(N) | 已接受 |

 ## 算法演练

 1. 不排序，因为多边形已经按顺时针顺序给出并且是凸的。 在构建偏移边时，我们将直接依赖该结构。 
2. 定义一个函数来检查给定的安全半径 r 是否可以实现。 该函数通过将每条边向内移动距离 r 来构造多边形。 每条原始边变成一条平行线，相邻的移位线相交形成新的顶点。 这是有效的，因为在凸多边形中，所有向内偏移都保持明确定义并保留凸性，直到形状退化。 
3. 计算连续移动线的所有交点以获得缩小多边形的顶点。 这些顶点保持顺序。 如果多边形折叠或变得无效，则半径不可行。 
4. 使用旋转卡尺计算缩小多边形的直径。 通过在扫过船体时保持对映点对，可以找到线性时间内凸多边形任意两点之间的最大距离。 
5. 如果计算出的直径至少为 D，则可以将两个点放置在缩小多边形中并保持足够的间隔，因此 r 是可行的。 
6. 否则，r太大，必须减小。 
7. 在从 0 到安全上限（通常是多边形的最大坐标比例）的连续范围内二分搜索 r。 使用固定的迭代次数或精度阈值（如 1e-7）。 
8. 返回找到的最大可行r。 

### 为什么它有效

 r 中的可行性条件是单调的。 增加 r 只会缩小有效区域，而不会扩展它，因此任何对 r 有效的配置对于较小的值仍然有效。 缩小的区域保持凸形，其直径完全捕获两个内点之间的最佳可能分离。 因此检查直径≥D相当于检查是否存在两个有效的塔位置。 

## Python 解决方案```python
import sys
import math
input = sys.stdin.readline

EPS = 1e-10

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def sub(a, b):
    return (a[0] - b[0], a[1] - b[1])

def dot(a, b):
    return a[0] * b[0] + a[1] * b[1]

def line_intersection(p1, d1, p2, d2):
    # p1 + t d1 intersects p2 + s d2
    # solve p1 + t d1 = p2 + s d2
    # cross(d1, d2) != 0
    den = cross(d1[0], d1[1], d2[0], d2[1])
    if abs(den) < EPS:
        return None
    dp = sub(p2, p1)
    t = cross(dp[0], dp[1], d2[0], d2[1]) / den
    return (p1[0] + d1[0] * t, p1[1] + d1[1] * t)

def build_offset(poly, r):
    n = len(poly)
    lines_p = []
    lines_d = []

    for i in range(n):
        x1, y1 = poly[i]
        x2, y2 = poly[(i + 1) % n]
        dx, dy = x2 - x1, y2 - y1

        length = math.hypot(dx, dy)
        nx, ny = dy / length, -dx / length

        cx = (x1 + x2) / 2
        cy = (y1 + y2) / 2
        if dot((nx, ny), (cx - 0, cy - 0)) < 0:
            nx, ny = -nx, -ny

        p = (x1 + nx * r, y1 + ny * r)
        lines_p.append(p)
        lines_d.append((dx, dy))

    new_poly = []
    for i in range(n):
        p1, d1 = lines_p[i], lines_d[i]
        p2, d2 = lines_p[(i + 1) % n], lines_d[(i + 1) % n]
        inter = line_intersection(p1, d1, p2, d2)
        if inter is None:
            return []
        new_poly.append(inter)

    return new_poly

def diameter(poly):
    n = len(poly)
    if n <= 1:
        return 0.0

    j = 1
    ans = 0.0

    for i in range(n):
        ni = (i + 1) % n
        while True:
            nj = (j + 1) % n
            if abs(cross(poly[ni][0] - poly[i][0],
                         poly[ni][1] - poly[i][1],
                         poly[nj][0] - poly[j][0],
                         poly[nj][1] - poly[j][1])) > abs(
                cross(poly[ni][0] - poly[i][0],
                      poly[ni][1] - poly[i][1],
                      poly[j][0] - poly[i][0],
                      poly[j][1] - poly[i][1])
            ):
                j = nj
            else:
                break

        dx = poly[i][0] - poly[j][0]
        dy = poly[i][1] - poly[j][1]
        ans = max(ans, dx * dx + dy * dy)

    return math.sqrt(ans)

def ok(poly, D, r):
    new_poly = build_offset(poly, r)
    if len(new_poly) < 3:
        return False
    return diameter(new_poly) >= D

def main():
    N, D = map(int, input().split())
    poly = [tuple(map(int, input().split())) for _ in range(N)]

    lo, hi = 0.0, 1e7
    for _ in range(60):
        mid = (lo + hi) / 2
        if ok(poly, D, mid):
            lo = mid
        else:
            hi = mid

    print(lo)

if __name__ == "__main__":
    main()
```该实现将几何变换与可行性测试分开。 偏移构造为每条边构建平行线并与连续的线相交以重建内部多边形。 直径函数使用旋转卡尺，这是凸包直径的标准优化方法。 

二分搜索循环运行固定次数的迭代以避免浮点不稳定问题。

 ## 边缘情况

 当偏移多边形在失去直径有效性之前变得退化时，就会出现关键的边缘情况。 在细三角形中，即使半径很小，也会消除一个顶点的贡献，并将多边形折叠到少于三个点。 在这种情况下，算法会正确返回该半径的不可行结果，因为没有剩余有效的内部区域。 

另一个边缘情况是当 D 非常小时。 那么 r 本质上仅受我们在多边形崩溃之前可以向内推动多远的限制，并且直径检查几乎总是成功。 二分搜索仍然正确收敛，因为可行性仍然是单调的。 

当两个连续的边几乎共线时，最后的边缘情况是数值不稳定。 相交计算可能会放大浮点误差，因此 EPS 检查可防止除以几乎为零的行列式，并避免产生会破坏直径计算的无效顶点。
