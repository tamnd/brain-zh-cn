 ## 解决方案
 ## 问题理解

 我们得到一连串刚性段，首尾相连地放置在平面上。 每个片段都有固定的长度，连续的片段通过允许旋转的关节连接。 最初，整个链条是笔直且垂直的，这意味着每个段都沿着正 y 方向指向上方。 

然后我们应用一系列旋转命令。 每个命令都会选择段 S 和 S+1 之间的一个关节，并将从段 S+1 开始的整个后缀作为刚体围绕该关节旋转。 前面的部分保持固定在适当的位置，而所有后面的部分一起旋转，保留其内部结构。 

处理完所有旋转后，任务是计算最后一段端点的最终坐标。 

这些约束允许最多 100,000 个段和 100,000 个操作。 每次旋转后重新计算每个段位置的解决方案将需要每次操作更新 O(N) 个段，导致最坏情况下的 O(NK) 工作量，这远远超出了可行的限制。 这迫使我们采用一种支持前缀稳定性和后缀轮换更新的数据结构。 

当多次旋转影响重叠后缀时，会出现一个微妙的问题。 一种幼稚的方法可能会从头开始反复重新计算角度并累积浮点误差，最终明显漂移。 另一个错误是将旋转视为独立的局部变换，而没有为每个片段保持一致的全局角度表示。 

## 方法

 直接模拟以角度形式保留一系列线段方向。 每个操作都会将所有段从 S+1 向前旋转 A 度，因此我们将在每次查询时更新 O(N) 条目，然后重新计算位置。 这是正确的，但太慢了，因为最坏的情况会达到 10^10 次更新。 

关键的观察结果是，每个段对最终端点的贡献与其方向向量呈线性关系。 如果我们知道每个线段的角度，则端点就是方向 theta_i 上长度为 L_i 的向量的累加和。 困难在于，当我们应用旋转时，theta_i 会改变整个后缀。 

我们没有单独跟踪每个段，而是坚持这样的想法：链中的每个位置都会经历影响其之前关节的所有操作的累积旋转。 这表明逆转视角：我们不是更新所有受影响的段，而是为每个关节维护对其后的所有段应用多少额外旋转。 “旋转增量”上的芬威克树或线段树让我们可以存储角度增量，以便每个线段的最终角度是所有适用更新的总和。 

在构建支持范围添加和点查询的结构后，我们通过迭代一次段来计算最终位置，保持当前的全局角度并添加每个段的向量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(NK) | O(N) | 太慢了|
 | 具有角增量的 Fenwick/线段树 | O((N+K) log N) | O((N+K) log N) | O(N) | 已接受 |

 ## 算法演练

 我们将角度视为沿着链条累积的值。 每个线段 i 都有一个基本方向，最初指向上方，对应于标准笛卡尔坐标中的 90 度。 

我们在代表关节的索引上维护一棵芬威克树，存储从每个位置开始应用的额外旋转量。

1. 初始化大小为 N+1 且全为零的 Fenwick 树。 该结构存储影响后缀的累积角度偏移。 这是必要的，因为每次更新都适用于连续的段后缀。 
2. 将第一段的初始方向设置为 90 度。 这与标准坐标中的初始垂直向上方向相匹配。 
3. 对于每个旋转命令（S、A），将其转换为增量更新。 我们从位置 S+1 开始添加 A 度，因为从 S+1 开始的所有线段一起旋转。 如果 S 为 0，我们从段 1 开始应用旋转。此转换将后缀旋转减少为范围更新。 
4. 在 Fenwick 树上从索引 S+1 开始执行范围添加 A。 这确保了任何线段 i ≥ S+1 都将在其累积角度中包含该旋转。 
5. 处理完所有命令后，通过从段 1 迭代到 N 来计算最终几何形状。我们维护一个运行角度变量。 
6. 对于每个段 i，查询位置 i 处的 Fenwick 树以获得应用于该段的总额外旋转。 将此值添加到运行角度，该角度表示线段 i 的绝对方向。 
7. 使用 x 的余弦值和 y 的正弦值将角度转换为方向向量，乘以 L_i，然后添加到当前端点坐标。 

工作原理：每次旋转都会影响一个连续的后缀，并且每个段的最终方向恰好是应用于其之前或处的关节的所有旋转的总和。 芬威克树存储这些旋转的差异表示，确保每个线段精确接收正确的累积角度。 由于矢量加法是线性的，因此按最终角度顺序处理片段可以重建精确的端点，而无需显式模拟中间状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0.0] * (n + 2)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0.0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

def solve():
    N, K = map(int, input().split())
    L = list(map(int, input().split()))

    fw = Fenwick(N + 2)

    for _ in range(K):
        S, A = map(int, input().split())
        fw.add(S + 1, A)

    x = 0.0
    y = 0.0
    angle = 90.0

    for i in range(N):
        angle += fw.sum(i + 1)
        rad = math.radians(angle)
        x += L[i] * math.cos(rad)
        y += L[i] * math.sin(rad)

    print(f"{x:.10f} {y:.10f}")

if __name__ == "__main__":
    solve()
```芬威克树将角度增量存储为差异结构。 每个命令在 S+1 处应用，因为段索引直接对应于旋转的前缀累积。 关键的微妙之处在于我们从不存储每个片段的完整角度； 相反，我们在向前行走时即时计算它们。 

从角度到弧度的转换仅在矢量计算时发生，避免了更新过程中重复的三角累积误差。

 ## 边缘情况

 关键的边缘情况是当 S = 0 时，这意味着旋转适用于整个链。 在这种情况下，更新必须从索引 1 开始，而不是从 0 开始。如果此转换执行不正确，则段 1 将被排除，整个链将错误地漂移。 例如，将两段链旋转 90 度应将两段旋转为水平对齐； 未能包含第一段会立即破坏结构。 

另一种边缘情况涉及多次完整的 360 度旋转。 角度累积超过一整圈，但余弦和正弦保持稳定，因为它们是周期性的。 然而，如果尝试在每次更新时积极地标准化角度，则如果做得不一致，可能会出现中间取消错误。 正确的方法完全避免归一化，并依赖于评估时的三角周期性。 

最终的边缘情况是具有许多小旋转的大 N。 如果重复重新计算角度，直接按段更新将严重累积浮点误差。 通过仅存储增量并在每个段应用一次，我们将误差累积减少到每个段的单个评估，从而将数值稳定性保持在容差范围内。
