 ## 解决方案
 ## 问题理解

 我们得到了放置在 N 个位置上的从 1 到 N 的数字的排序。 将数组 a 视为 Larry 的“理想布局”，其中位置 i 理想地需要值 a[i]。 

Harry 可以将同一组值 1 到 N 重新排列成另一个排列 b。 重新排列的成本以一种稍微不寻常的方式定义：对于每个位置 i，我们使用绝对差 |a[i] − b[i]| 来测量 Harry 的选择与 Larry 的期望的偏差程度，然后我们在所有位置上取这些值中的最小值。 Harry 希望使这个最小值尽可能大。 

因此，任务是构建一个排列 b，将 a 中的每个位置推离其首选值，同时最大化最坏情况的接近度。 

约束达到 N = 10^5，这会立即排除任何尝试所有排列的解决方案，甚至排除任何二次方（例如检查所有交换）。 我们需要一个以线性时间运行的构造，因为 O(N log N) 也很好，但没有必要。 

当 N = 1 时，会出现一种微妙的边缘情况。只有一种可能的排列，因此答案被迫为 0，因为 |a1 − b1| 始终为 0。 

当 N ≥ 2 时，有趣的部分开始了。 一个幼稚的想法是尝试避免精确匹配 a[i]，因为这给出了距离 0，但即使我们避免精确匹配，我们也希望避免像 a[i] ± 1 这样的接近匹配，因为它们仍然会减少最小值。 

## 方法

 如果我们忽略结构，蛮力方法将尝试每个排列 b 并计算 O(N) 的分数。 既然有N个！ 即使对于小 N 来说，这也是完全不可行的。甚至限制像回溯这样的更智能的搜索也会失败，因为约束耦合了全局的所有位置。 

关键的观察结果是，每个位置 i 恰好禁止一个值，即 b[i] = a[i] 是不可取的，因为它给出了绝对最小可能贡献 0。因此，我们尝试在 a 定义的对角线上构造一个具有禁止位置的排列。 这成为一个经典的“类似混乱”的赋值问题，其中每个索引恰好排除一个值。 

一旦这样看，结构就变得简单了：由于每个位置只禁止一个值，并且有 N 个可用值，所以我们总是可以循环移位分配。 数组 a 的循环移位保证没有位置收到其原始值，因为每个元素都会移动到不同的索引，并且所有值都是不同的。 

这立即意味着，对于 N ≥ 2，我们总是可以实现最小距离至少为 1，因为获得距离 0 的唯一方法是相等，而我们已经消除了相等。 

一般来说，我们也不能做得比 1 更好。 假设我们尝试强制执行至少为 2 的最小距离。这将要求每个位置 i 不仅要避免 a[i]，还要避免 a[i] − 1 和 a[i] + 1（当它们存在时）。 在边界附近，像 1 或 N 这样的值几乎没有有效的选择，并且当 N 很大并且考虑任意排列时，这会在全局范围内产生不可避免的冲突。 始终成功 ≥2 的干净结构并不存在，在最坏的情况下，该结构迫使我们降至 1。 

因此，当 N = 1 时，最优答案为 0，否则为 1，我们只需构造相对于 a 没有固定点的任意排列即可。 

a 的简单循环移位可以直接实现这一点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(N!) | O(N) | 太慢了|
 | 循环移位施工| O(N) | O(N) | 已接受 |

 ## 算法演练

1. 读取数组 a，它表示值的原始放置位置。 
2. 如果N等于1，立即输出0并返回唯一可能的排列。 这是强制的，因为没有重新排列会改变单个位置的值。 
3. 将 a 向左移动一个位置，构造一个新数组 b，这意味着每个元素取 a 中下一个位置的值，最后一个位置取第一个元素。 
4. 输出 1 作为所达到的最小距离。 
5. 输出构造好的数组b。 

选择这种转变的原因是它保证每个位置 i 接收到一个原本属于不同位置的值，并且由于 a 中的所有值都是不同的，所以相等是不可能的。 

### 为什么它有效

 该构造保证对于每个 i < N，b[i] = a[i+1]，这与 a[i] 不同，因为 a 中的所有元素都是不同的。 同样，b[N] = a[1]，也与a[N]不同。 因此，没有位置满足 b[i] = a[i]，因此每个绝对差至少为 1。由于不同整数之间的绝对差是整数，因此最小可能值恰好为 1，这是最佳的，因为仅在 N = 1 的情况下才不可避免地出现 0。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    if n == 1:
        print(0)
        print(a[0])
        return

    b = a[1:] + a[:1]

    print(1)
    print(*b)

if __name__ == "__main__":
    solve()
```实现直接遵循循环移位思想。 切片操作 a[1:] + a[:1] 执行排列的单次旋转。 这保证了所有值都在 1 到 N 范围内恰好一次，因此 b 是有效的。 

输出值 1 是固定的，因为该构造确保 a[i] 和 b[i] 之间不完全匹配，因此最小绝对差值不能降至 0。

 ## 边缘情况

 对于 N = 1，算法直接返回 0 和单个值。 没有构建不同排列的自由，因此这是唯一有效的结果。 

对于任何 N ≥ 2，循环移位可确保每个位置 i 将其值移离其原始位置。 即使输入已经是一个循环或高度结构化，该移位仍然保证所有 i 的 b[i] ≠ a[i]，这强制执行最佳最小距离 1。
