 ## 解决方案
 ## 问题理解

 我们有一个矩形网格，两个玩家被放置在两个不同的单元格中。 网格没有障碍物，因此始终可以在四个基本方向上移动。 玩家轮流一次移动一步，每次移动时，玩家必须移动到相邻的单元格。 当一个玩家移动到另一个玩家当前占据的单元格时，游戏立即结束，并且移动的玩家获胜。 

关键在于两名玩家都发挥最佳状态，唯一的目标是强制获胜或避免失败，具体取决于轮到谁。 我们需要根据初始位置以及哪位玩家先移动来确定获胜者。 

这些约束允许网格在 2000 之前达到 2000 个，这意味着任何尝试模拟游戏图所有可能状态的解决方案都太慢。 简单的状态搜索会将每对位置视为一个状态，在最坏的情况下会放弃大约 40 亿个状态，这远远超出了时间和内存的可行限制。 这立即表明解决方案必须将状态空间折叠成一个更简单的不变量，很可能基于距离。 

当玩家非常接近时，就会出现微妙的边缘情况。 例如，如果他们相邻，则第一个玩家可以立即获胜。 另一个边缘情况是当他们被长度为 2 的路径分开时，回合顺序决定了第一个玩家是先到达对手还是先被拦截。 这些案例暗示距离和转弯顺序的奇偶性比确切的路径结构更重要。 

## 方法

 暴力方法将游戏建模为状态图，其中每个状态由两个玩家的位置以及轮到谁组成。 在每个状态中，我们都会尝试主动玩家的所有四个动作，并递归地评估它是否会导致胜利。 这是一个典型的巨大隐式图上的极小极大游戏。 虽然概念上是正确的，但状态的数量约为$R^2 C^2$，因为每个玩家独立占据一个单元格，并且每个转换最多向四个方向分支。 即使使用记忆功能，可到达的配置数量对于 1 秒的限制来说仍然太大。 

关键的观察结果是，两名球员的移动力量是对称的，并且都在努力最小化他们之间的曼哈顿距离。 在空网格上，两个单元之间的最短路径正是它们的曼哈顿距离。 如果两个玩家都发挥最佳，他们总是会沿着某个最短路径接近对方，因为任何偏差只会延迟会议并恶化他们的位置。 

这将游戏简化为一个量：玩家之间的曼哈顿距离。 每次移动都会将该距离精确地减少 1，因为移动的玩家总是可以更接近对手。 当玩家回合的距离达到 0 时，游戏结束，这意味着玩家刚刚踏入对手的牢房。 

因此，我们不跟踪位置，而是只跟踪需要多少步才能消除距离，以及哪个玩家获得了最后一步。 这将复杂的游戏状态问题转换为奇偶校验问题。

 ## 算法演练

 1. 使用公式计算两个玩家之间的曼哈顿距离$D = |r_s - r_c| + |c_s - c_c|$。 这代表一个玩家在空网格中到达另一个玩家所需的最少移动次数。 
2. 观察到，在最佳玩法下，任一玩家的每一步移动都会将该距离缩短 1。 这是因为每个玩家总是向对手靠拢，并且不存在迫使绕道的障碍。 
3. 得出游戏恰好在步数处结束的结论$D$，自从之后$D$总移动距离变为零，最后一个移动者已踏入对手的单元格。 
4. 确定哪位玩家走棋数$D$。 如果第一个玩家是 Sam，则 Sam 在第 1、3、5 回合移动，依此类推。 如果克莱德先移动，则克莱德将占据奇数回合。 
5. 比较奇偶性$D$以先行者的身份。 如果$D$奇数时，先行者最后一步； 如果$D$是偶数，第二个推动者成功。 

### 为什么它有效

 不变量是，最佳游戏迫使双方玩家每次移动都将曼哈顿距离精确地减少 1。 延迟或采取更长的路径没有任何好处，因为对手可以反映最佳移动。 这将游戏变成了确定性倒计时$D$为零，唯一剩下的自由度是哪个玩家获得最后的减量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    R, C = map(int, input().split())
    rs, cs, rc, cc = map(int, input().split())
    first = input().strip()

    dist = abs(rs - rc) + abs(cs - cc)

    if first == 'S':
        # Sam moves on odd turns
        if dist % 2 == 1:
            print("S")
        else:
            print("C")
    else:
        # Clyde moves on odd turns
        if dist % 2 == 1:
            print("C")
        else:
            print("S")

if __name__ == "__main__":
    solve()
```该实现将整个网格游戏简化为计算单个曼哈顿距离并检查奇偶性。 唯一微妙的部分是将奇偶性与先行者的身份正确地结合起来。 一个常见的错误是假设第一个玩家总是对应相同的奇偶性，而不调整它是山姆还是克莱德。 代码显式地在起始字符上分支以保持对齐。

 ## 边缘情况

 当玩家开始相邻时，曼哈顿距离为 1。第一个玩家总是立即获胜，因为他们可以直接移动到对手的单元格中。 该算法计算$D = 1$，并且由于 1 是奇数，因此它将胜利分配给先动者，这与实际的游戏玩法相匹配。 

当距离均匀时，例如两步间隔，第二个移动者有效地获得了最后一步。 例如，如果山姆开始并且距离为 2，那么山姆首先将其减少到 1，然后克莱德将其减少到 0 并获胜。 奇偶校验规则正确地捕获了这一点，而无需模拟转弯。 

当克莱德开始比赛时，奇偶性解释会翻转角色，但同样的不变量仍然成立。 计算不依赖于网格大小或形状，仅依赖于曼哈顿距离和转弯顺序对齐。
