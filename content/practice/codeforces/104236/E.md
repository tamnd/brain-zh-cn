 ## 解决方案
 ## 问题理解

 我们在二维平面上得到一组点，每个点代表国家公园中的一个观测站。 每个站都配备了具有统一功率水平的无线设备$P$。 这个功率决定了通信半径：两个站点的欧几里得距离最大时可以直接通信$\sqrt{P}$。 通信不限于直接链路，因为消息可以通过中间站中继，因此连接性是在图论意义上定义的。 

任务是选择最小的可能值$P$这样所有的站都通过这个通信网络连接起来。 

重构后，我们在点上构建一个完整的加权图，其中两个节点之间的权重是欧几里得距离的平方。 我们想要最小的阈值$P$这样如果我们最多只保留权重的边$P$，结果图是连通的。 

约束条件$N \le 500$意味着对边进行完整的成对考虑是可行的。 最多有大约$125{,}000$边缘，所以算法$O(N^2 \log N)$或者$O(N^2)$行为是可以接受的。 任何三次或更差的东西也只能以优化的形式通过，但在这里是不必要的。 

一些边缘情况很重要。 如果所有点都相同，则答案为零，因为不需要连接。 如果点与一个极远的异常值形成一条线，则答案由沿最佳可能跨越结构的单个最长所需连接决定，而不是由绝对最远的点对决定。 

一个天真的错误是假设答案是任意两点之间的最大距离。 例如，如果三个点形成一个三角形，其中两个点相距很远，但都靠近中心点，则通过中心实现连接，因此不需要直接使用最大边。 

## 方法

 问题本质上是关于距离阈值下的连接性。 如果我们固定一个候选值$P$，我们可以构造一个图，其中连接平方距离最多为的所有点对$P$，然后检查图是否使用 BFS 或 DSU 连接。 如果我们搜索可能的结果，这会给出一个正确但昂贵的解决方案$P$价值观。 

蛮力的想法变成：计算所有成对的平方距离，对它们进行排序，并按升序尝试阈值，每次都检查连通性。 在最坏的情况下，我们可能会重新计算连接$O(N)$次，每次成本$O(N^2)$，导致$O(N^3)$行为，这是不必要的，但仍然是可以容忍的$N=500$仅在高度优化的代码中。 

关键的观察是，我们正在有效地寻找最小阈值，以使图变得连通，这正是最小生成树（MST）的定义。 如果我们使用平方距离作为权重来构建完整图的 MST，则答案就是 MST 中的最大边权重。 这是可行的，因为 MST 是连接所有节点最便宜的方式，并且任何连接阈值必须至少允许所有 MST 边缘。 

因此，我们不尝试阈值，而是直接使用 Prim 的算法计算 MST$O(N^2)$，因为我们可以在没有显式边缘存储的情况下维持密集图中的距离。 最终的答案是MST中使用的最大边。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力阈值检查 |$O(N^3)$|$O(N^2)$| 太慢了|
 | Prim MST（密集图）|$O(N^2)$|$O(N)$| 已接受 |

 ## 算法演练

 我们将每个点视为完全连接图中的一个节点，其中边权重是欧几里得距离的平方。 

1. 从任意节点开始，为方便起见，从节点 0 开始，并将其标记为包含在 MST 中。 这代表了我们开始从单个站点构建连接结构的事实。 
2. 维护数组`min_dist[i]`存储距节点的最小平方距离`i`到 MST 中已有的任何节点。 最初，这是仅从节点 0 计算的。 
3. 重复选择尚未进入MST且最小的节点`min_dist`。 这种选择确保我们始终使用最便宜的可用连接来扩展当前连接的组件。 
4.将该节点添加到MST中，并将答案更新为最大值`min_dist`在所有选定的边中。 跟踪该最大值是因为它代表跨越结构中的瓶颈边缘。 
5.添加节点后，更新`min_dist`对于所有剩余节点，通过将它们的当前值与到新添加节点的距离进行比较。 这保持`min_dist`与当前部分MST一致。 
6. 继续，直到包含所有节点。 最终的答案是在此过程中选择的最大边权重。 

### 为什么它有效

 在每一步中，Prim 的算法都保持不变，即当前所选节点集使用该集的可能总成本最小的边进行连接。 将该组件连接到图的其余部分的任何边的权重必须至少为当前的权重`min_dist`所选节点的。 由于我们总是选择最小的可用边，因此我们正在有效地构建最小瓶颈生成树。 该结构中的最大边是仍然允许整个图连接的最小可能阈值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    if n == 1:
        print(0)
        return

    in_mst = [False] * n
    min_dist = [10**30] * n

    in_mst[0] = True
    for i in range(1, n):
        dx = pts[i][0] - pts[0][0]
        dy = pts[i][1] - pts[0][1]
        min_dist[i] = dx * dx + dy * dy

    ans = 0

    for _ in range(n - 1):
        v = -1
        best = 10**30
        for i in range(n):
            if not in_mst[i] and min_dist[i] < best:
                best = min_dist[i]
                v = i

        in_mst[v] = True
        ans = max(ans, best)

        for i in range(n):
            if not in_mst[i]:
                dx = pts[i][0] - pts[v][0]
                dy = pts[i][1] - pts[v][1]
                dist = dx * dx + dy * dy
                if dist < min_dist[i]:
                    min_dist[i] = dist

    print(ans)

if __name__ == "__main__":
    solve()
```该实现是直接密集 Prim 算法。 这`min_dist`数组始终反映将每个外部节点连接到当前 MST 的最便宜的方式。 我们使用平方距离来避免浮点运算，因为比较$\sqrt{P}$距离相当于比较平方值。 

一个微妙的细节是正确初始化距节点 0 的距离； 如果不这样做会导致错误的无限值并破坏选择步骤。 另一个重要的点是跟踪所选边缘权重的最大值而不是对它们求和，因为问题需要阈值而不是总成本。

 ## 边缘情况

 对于单站来说，MST没有边，因此算法初始化`ans`为零并立即返回它。 永远不会进入选择循环，这与不存在连接要求的事实相匹配。 

当所有点重合时，每个计算的平方距离为零，因此每个`min_dist`更新保持为零。 每个选定的边的权重为零，因此最终的最大值也为零，与预期输出匹配。 

在存在一个遥远异常值的情况下，该算法首先在具有小边缘的近距离点之间构建一个簇，然后最终使用尽可能小的链接将异常值连接到该簇。 由于所有中间选项已经用尽，因此最终边缘一定很大，并且 MST 正确地将其捕获为瓶颈。
