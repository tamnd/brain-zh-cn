 ## 解决方案
 ## 问题理解

 我们得到了一系列排成一排的动物。 每只动物都有两个属性：物种标签和技能值。 任务是考虑该线的每个连续段并计算每个段的分数。 分段的分数取决于两个量：分段内技能值的总和，以及该分段中出现的不同物种的数量。 一个片段的贡献是技能总和与不同物种数量的一个非常大的幂（特别是 3366 次幂）的乘积。 

输出是所有可能子阵列上这些贡献的总和。 

这些限制立即塑造了问题。 动物数量最多为 100000 只，因此枚举所有子数组（将是二次数组）太慢。 即使我们可以快速计算一个片段中不同物种的数量，在最坏的情况下，仅片段的数量就约为 50 亿，这已经排除了任何 O(N^2) 方法。 

一个关键的结构细节是物种 ID 以 50 为界。这种小的字母大小是使快速解决方案成为可能的主要杠杆，因为它建议对物种存在进行位掩码或状态压缩。 

一种简单的方法是枚举每个子数组，维护频率图来跟踪不同的物种，维护技能的运行总和，并计算贡献。 这会失败，因为更新每个扩展的频率图的时间复杂度为 O(1)，但对所有 O(N^2) 段执行此操作仍然太大。 

第二个微妙的陷阱是将指数 3366 视为直接为每个段计算的东西。 尽管模幂计算很快，但对每个子数组重复它会增加本来不可能的复杂性。 

## 方法

 蛮力方法很简单：固定左端点，扩展右端点，维护物种的频率数组，计算不同的物种，并维护技能的运行总和。 每个扩展都会更新答案。 这正确地计算了所需的值，但它执行了大约 N^2 次更新，在最坏的情况下大约是 10^10 次操作，远远超出了限制。 

主要观察来自两个事实。 首先，物种值很小，最多只有 50 个，因此不同物种的数量最多为 50 个。其次，取幂使得直接处理不同的计数变得很困难，但我们可以按照不同物种集对子数组进行分组来思考。 

我们不是直接对所有子数组求和，而是反转视角。 固定一组物种 S。考虑其不同物种集恰好为 S 的所有子数组。对于每个这样的子数组，其贡献是（子数组中的技能总和）乘以 |S|^3366。 

由于 |S| 仅取决于集合大小，而不取决于实际位置，我们可以将问题分解为计算，对于每个可能的子集大小 k，其不同物种集大小为 k 的所有子数组的技能总和乘以 k^3366。 

因此，问题简化为计算，对于从 1 到 50 的每个 k，其不同物种计数恰好为 k 的所有子数组的子数组和的总和。 

这仍然很重要，因为计算具有恰好 k 个不同元素的子数组并不是直接相加的。 然而，我们可以使用“至多 k 个不同”计数切换到标准包含-排除结构。 

将 F(k) 定义为不同物种数最多为 k 的所有子数组的总和。 那么精确 k 的答案就是 F(k) 减去 F(k-1)。 剩下的任务是有效地计算所有 k 的 F(k)。

为了计算 F(k)，我们在数组上使用滑动窗口，维护一个最多包含 k 个不同物种的窗口。 对于每个右端点，我们找到最小的左端点，使得窗口最多包含 k 个不同的物种。 对于每个固定的右端点 r，所有以 r 结尾的有效子数组都是从 l 到 r 的子数组，其中 l 是最小的有效左边界。 我们需要这个范围内所有子数组总和的总和。 

我们维护技能值的前缀和，以便可以在 O(1) 内计算子数组和。 然后，对于每个 r，我们使用前缀和算术在常数时间内累积来自所有有效 l 的贡献。 

我们对从 1 到 50 的每个 k 重复此操作。因为 k 很小，所以总复杂度变得可以接受。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(N^2) | O(N^2) | O(50) | 太慢了|
 | 最佳 | O(50·N) | O(N) | 已接受 |

 ## 算法演练

 我们定义一个函数，对于固定限制 k，计算最多包含 k 个不同物种的所有子数组的总贡献。 

1. 预先计算技能值的前缀和，以便在常数时间内获得任意子数组和。 这允许快速聚合范围内的技能贡献。 
2. 对于固定的 k，维护一个滑动窗口 [l, r] 和窗口内物种的频率数组。 逐步扩展 r，更新频率并跟踪当前内部有多少不同物种。 
3. 如果添加新元素使不同物种的数量超过 k，则向前移动 l，直到窗口再次有效。 每次 l 移动时，更新频率并在物种计数降至零时删除物种。 
4. 对于每个位置 r，一旦窗口有效，计算以 r 结束并从 l 到 r 的任何索引开始的所有子数组的贡献。 每个这样的子数组的总和都可以使用前缀和来表示，并且我们使用前缀和公式的差异来聚合它们，而不是迭代所有的开始。 
5. 将该计算的结果存储为 F(k)。 
6. 计算 1 到 50 范围内所有 k 的 F(k) 后，导出恰好 k 个不同物种的贡献，即 F(k) - F(k-1)。 
7. 将每个贡献乘以 k^3366 modulo MOD 并累加为最终答案。 

它为何有效与两个不变量有关。 首先，滑动窗口始终表示将不同物种的数量保持在限制 k 内的最小左边界，因此以 r 结尾的每个有效子数组都被精确计数一次。 其次，前缀和聚合确保对于每个有效子数组，其技能和仅包含在该 (k, r) 对的总累加中一次。 将这些结合起来可以保证 F(k) 恰好是具有最多 k 个不同物种的所有子阵列的总和，没有遗漏或重复。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def mod_pow(a, e):
    return pow(a, e, MOD)

def compute_at_most_k(c, s, k):
    n = len(c)
    freq = [0] * 51
    distinct = 0
    l = 0

    # prefix sums of s
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + s[i]

    total = 0

    # We also maintain prefix of prefix sums for fast range sum of subarray sums
    pref2 = [0] * (n + 1)
    for i in range(n):
        pref2[i + 1] = pref2[i] + pref[i + 1]

    for r in range(n):
        x = c[r]
        freq[x] += 1
        if freq[x] == 1:
            distinct += 1

        while distinct > k:
            y = c[l]
            freq[y] -= 1
            if freq[y] == 0:
                distinct -= 1
            l += 1

        # sum of subarray sums for all l' in [l, r]
        # sum_{i=l..r} sum_{j=i..r} s[j]
        # equals pref2[r+1] - pref2[l] - (r-l+1)*pref[l]
        length = r - l + 1
        contrib = (pref2[r + 1] - pref2[l]) - length * pref[l]
        total += contrib

    return total

def solve():
    n = int(input())
    c = []
    s = []
    for _ in range(n):
        ci, si = map(int, input().split())
        c.append(ci)
        s.append(si)

    max_k = 50
    F = [0] * (max_k + 1)

    for k in range(1, max_k + 1):
        F[k] = compute_at_most_k(c, s, k)

    ans = 0
    for k in range(1, max_k + 1):
        exact = F[k] - F[k - 1]
        ans = (ans + exact * mod_pow(k, 3366)) % MOD

    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该代码首先构建前缀和以允许恒定时间范围和查询。 功能`compute_at_most_k`实现一个滑动窗口，强制执行最多 k 个不同物种的约束。 微妙的部分是双前缀数组`pref2`，它将许多子数组总和的总和转换为封闭式表达式。 如果没有它，计算将退化为每个 r 的二次循环。 

最后的循环计算最多 k 个值之间的差异，以隔离精确的不同计数，然后应用所需的功率加权。

 | 子数组| 总和|

 | [2] | 1 |
 | [3] | 3 |

 所以 F(2) = 19。

 一个关键的边缘情况是所有动物属于同一物种。 在这种情况下，每个子阵列都恰好有一个不同的物种，因此只有 k = 1 起作用。 滑动窗口永远不会扩展到单个物种之外，并且 k ≥ 1 时的 F(k) 变得相同。 F(k) - F(k-1) 之差正确地隔离了 k = 1，并且所有更高项都消失了。 

另一个边缘情况是所有物种都是不同的。 那么子数组中不同物种的数量等于其长度，因此较小的 k 值仅计算短子数组。 滑动窗口急剧缩小，每个 r 仅对小 k 贡献 O(1) 有效起始，这与 F(k) 的预期行为完全匹配。
