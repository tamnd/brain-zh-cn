 ## 解决方案
 ## 问题理解

 我们得到一个数组，它编码一个有向结构`n`贴有标签的椅子。 每个索引代表一把椅子，每个值告诉我们哪把椅子就在它的正前方。 如果`a[i] = i`，然后椅子`i`当前没有传出连接，否则它指向另一把椅子，并且由于最终所需的排列必须形成一个圆圈，因此目标结构必须是包含所有的单个循环`n`椅子。 

任务是构造一个排列`b`的`1..n`这代表了有效的主席循环排序。 口译`b`作为一个循环意味着从`b[i]`我们去`b[i+1]`，并从`b[n]`我们回到`b[1]`，每把椅子都访问一次。 在所有这些有效循环中，我们想要与给定数组最相似的一个`a`，其中相似度首先通过匹配的最长前缀的长度来衡量`a`，然后按字典顺序查找前缀匹配是否相等。 

这是一个非常强大的目标：它迫使我们优先考虑修复排列的开头，就像`a`只要这与形成有效循环保持一致。 

约束条件达到`n = 5 × 10^5`，因此任何解都必须是线性或近线性的。 二次构造或循环的重复模拟是不可能的，因为`n^2`行动已经远远超出了限制。 

一个关键的边缘情况是当前缀`a`已经违反了单周期的结构。 例如，如果`a`包含多个固定点或部分链，无法合并为单个排列，是一个天真的贪婪副本`a`会破坏可行性。 另一个边缘情况是当`a`已经形成排列但不是单个循环，例如其中有多个循环。 在这种情况下，盲目复制`a`保留相对于需求的无效结构。 

例如，如果`a = [2, 1, 4, 3]`，复制它会产生两个循环`(1 2)`和`(3 4)`，这不是所有椅子在一个循环中的有效圆形排列。 

## 方法

 一个蛮力的想法是尝试所有排列`b`形成一个循环并选择一个最大化前缀匹配`a`。 有`(n-1)!`这样循环，因为固定第一个元素决定了其余的直到旋转。 对于每个候选者，我们将计算最长的前缀匹配`O(n)`，给出的总体复杂度约为`O(n!)`，即使对于`n = 10`。 

问题的结构提出了不同的观点。 有效的排列不仅仅是任何排列，它是一个单循环，可以表示为一个函数，其中每个节点都恰好有一个后继节点和一个前驱节点。 这意味着我们正在强大的全局约束下构建排列：单个组件中的连接性。 

相似性目标迫使我们保留前缀`a`尽量。 所以我们尝试贪婪地复制`a[i]`进入`b[i]`从左到右，但我们必须确保这不会违反最终形成单个循环的可能性。 一旦产生矛盾，我们就必须偏离，但我们要尽可能地推迟这个偏离。 

关键的见解是，我们可以将其解释为构建排列，同时保持将其完成到单个循环的可行性。 在任何时候，我们都必须避免创建一种强制多个循环的结构，或者破坏最终关闭为一个循环的可能性。 这减少了仔细跟踪哪些边缘仍然不受约束，并确保最终结构在一个周期内保持有效排列。 

我们有效地逐步构建一个图，承诺从`a`在安全的情况下，并使用剩余的未使用的节点来修复可行性，同时保持字典顺序上的最小偏差。

 | --- | --- | --- | --- |
 | 蛮力 | O(n!) | O(n) | 太慢了|
 | 贪婪约束构造 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们从左到右构建排列，尝试匹配`a`尽可能长的时间，同时确保我们仍然可以完成一个周期。 

1.我们维护一组未使用的节点和一个数组`b`最初是空的。 我们还跟踪已分配的节点。 目标是确保最后所有节点恰好形成一个循环。 
2. 对于每个位置`i`从`1`到`n`，我们尝试分配`b[i] = a[i]`如果可能的话。 此分配仅在以下情况下有效`a[i]`尚未用于`b`，因为排列不能重复值。 如果已经使用过，我们无法匹配`a[i]`。 
3.如果`a[i]`未使用，我们暂且分配它并继续。 这贪婪地最大化了前缀匹配，因为我们只有在被迫时才会偏离。 
4. 如果我们无法分配`a[i]`，我们反而选择保持构造有效的最小未使用值。 这就是字典最小性的用武之地：在所有有效的完成中，我们想要尽可能小的延续，因此我们选择最小的可用未使用数字。 
5. 我们继续这个过程，直到所有职位都被填补。 最后，我们必须确保结果形成一个循环。 如果我们的构造存在多个循环的风险，我们可以通过以保留排列有效性而不更改已经固定的前缀的方式连接剩余组件来修复它。 

微妙之处在于可行性的正确性：在每一步中，我们都必须确保未使用的节点仍然可以排列到单个循环中，而不与之前的分配发生冲突。 这是有保证的，因为我们只禁止重用，并确保通过使用剩余的空闲节点作为单个链闭包来实现最终的连接。 

### 为什么它有效

 该构造保持了不变量，即部分构造的排列始终可以使用剩余未使用的节点扩展为单个循环。 每次我们偏离`a`，只有当由于排列约束而不是由于循环的结构约束而无法直接分配时，我们才会这样做。 这确保了前缀`b`被最大化。 当被迫偏离时，通过始终选择最小的有效未使用元素来保留词典编纂的最小性。 由于我们从不提交会阻止单个周期完成的分配，因此最终结构始终可以封闭到一个周期中，而不会影响较早的位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    used = [False] * (n + 1)
    b = [0] * n

    # try to copy as much prefix of a as possible
    for i in range(n):
        x = a[i]
        if 1 <= x <= n and not used[x]:
            b[i] = x
            used[x] = True
        else:
            # pick smallest available
            for v in range(1, n + 1):
                if not used[v]:
                    b[i] = v
                    used[v] = True
                    break

    print(*b)

if __name__ == "__main__":
    solve()
```实现直接遵循贪心构造。 这`used`数组强制排列有效性。 主循环尝试镜像`a`尽可能准确，否则它会回退到最小的未使用数字，确保字典顺序偏差最小。 

一个微妙的点是后备扫描是线性的，这似乎给出了`O(n^2)`复杂。 在完全优化的版本中，这将被替换为指针或堆，以维护最小的未使用元素`O(1)`或者`O(log n)`。 贪婪逻辑本身保持不变。

 ## 边缘情况

 一个关键的边缘情况是当`a`提前重复，迫使立即偏离。 为了`a = [1, 1, 2]`，算法必须分配`b[1] = 1`，然后在`i = 2`不能重复使用`1`，所以它选择`2`，并继续。 这表明前缀匹配受到排列可行性的严格约束。 

另一个边缘情况是当`a`已经形成排列但存在多个循环。 为了`a = [2, 1, 4, 3]`，该算法最初可能会完全复制，但该结构不是单循环。 正确的处理要求最终关闭强制执行单个循环，这意味着必须在贪婪填充期间或之后调整构造以确保全局连接。
