 ## 解决方案
 ## 问题理解

 我们得到一个有向加权图，其中节点代表游泳池中的指定点，边代表它们之间的直接游泳路线。 每条边都有一个行程时间。 参与者沿着这些边沿路径移动，但还有一个额外的全局规则：每当参与者到达节点时，他们必须准确等待`x`他们在被允许离开之前的几秒钟，并且他们还在最终节点支付等待成本。 

每个查询提供一个起始节点、一个结束节点和一个值`x`。 对于那个具体的`x`，我们必须计算从起点到终点的最短可能时间，其中路径的成本是边权重加上`x`对于每个访问过的节点，包括两个端点。 

关键的结构困难在于图是固定的，但节点成本取决于查询，因此必须针对许多不同的值评估最短路径`x`高效。 

约束表明`n`最多为 500 条，边数最多为`2n`，所以图非常稀疏。 但是，查询数量最多可达 100000 个，这会立即排除每个查询运行最短路径算法的可能性。 即使 Floyd-Warshall 每个查询也是不可能的，因为它会`O(n^3 q)`。 

自查询和断开连接会产生微妙的边缘情况。 如果`u == v`，答案不为零，但仍包含节点处的等待时间，因此至少变为`x`。 如果没有路径，我们必须输出`-1`即使中间节点存在但在有向意义上不可到达。 

另一个陷阱是忘记等待发生在每个访问的节点上，而不仅仅是中间节点。 例如，如果路径是`u -> v`直接来说，成本就是`w(u,v) + 2x`，不仅仅是`w + x`。 

## 方法

 暴力破解的想法很简单：对于每个查询，从起始节点运行 Dijkstra 算法，每次遍历一条边时，我们都会添加其权重，并且还包含节点等待成本。 然而，节点等待取决于有多少节点被访问，这表明标准最短路径状态必须包含迄今为止访问的步骤或节点的数量。 这导致了扩展的状态定义`(node, k)`在哪里`k`是访问节点的数量，使转换取决于`x`。 在这个公式中，每次边转移都会增加距离和节点数，并且成本变为`distance + k * x`。 

这种每个查询的强力方法变得太昂贵，因为我们要么运行具有扩展状态的完整 Dijkstra，要么在每个查询的转换图中重新计算所有对的最短路径。 在最坏的情况下，扩展状态 Dijkstra 大约为`O(n^2 log n)`每个查询都在密集状态空间中进行，这对于`q = 1e5`。 

关键的见解是将结构分为两个部分：一个部分仅依赖于图形，另一个部分线性依赖于`x`。 如果我们修复一条路径，它的成本可以写为：`sum of edge weights + x * (number of nodes in path)`设路径中的边数为`k`，则节点数为`k+1`，因此成本变为：`edge_sum + x * (k+1) = (edge_sum + x) * k + x`这表明我们应该将每条边视为具有修改后的权重，具体取决于`x`，但更重要的是，我们可以以分离线性依赖的形式预先计算所有对最短路径`x`。 

我们将成本重写为：`edge_sum + x * nodes = edge_sum + x + x * edges = (edge_sum + x*edges) + x`因此，对于每一对`(u,v)`，我们想同时知道边数和总边权重的最短路径。 这是路径长度上最小加卷积的经典设置，可以通过维护两个 DP 层来处理：一个跟踪最小成本`k`边和另一个聚合边计数。 

自从`n ≤ 500`，我们可以使用修改后的 Floyd-Warshall 来维护两个矩阵：`dist[i][j]`对于最小边和和`cnt[i][j]`实现该总和结构的最小边数。 然后每个查询都会计算一个函数：`ans(u,v,x) = dist[u][v] + x * (cnt[u][v] + 1)`重要的简化是，对于最短路径，最小化`edge_sum`单独就足够了，因为所有边都有非负权重，因此在相等的边和中，最小边数是一致的。 因此，我们可以运行标准 Floyd-Warshall 以获得最短路径总和，并保持等成本路径中的最小边数。 

这会产生一个在 O(1) 时间内回答每个查询的预计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（每个查询最短路径）| O(q·n² log n) | O(q·n² log n) | O(n²) | 太慢了|
 | 最优（带有边缘跟踪的 Floyd-Warshall）| O(n3 + q) | O(n²) | 已接受 |

 ## 算法演练

 1.初始化距离矩阵`dist`具有无穷大和边数矩阵`cnt`无穷大，除了`dist[i][i] = 0`和`cnt[i][i] = 0`。 这表示停留在同一节点在边上没有任何成本，并且使用零边。 
2. 对于每个有向边`u -> v`有重量`w`， 放`dist[u][v] = min(dist[u][v], w)`和`cnt[u][v] = 1`。 这表明最佳直接连接仅使用一条边。 
3. 在中间节点上运行Floyd-Warshall`k`，更新所有对`(i, j)`使用节点`k`作为一座桥梁。 对于每个三元组`(i, k, j)`，计算候选路径组合`i -> k`和`k -> j`。 
4. 组合路径时，计算总边权重和边数。 如果新的边权重小于当前的最佳边权重，则将两者替换`dist[i][j]`和`cnt[i][j]`。 如果相等，保留较小的边数。 
5. 预处理后，每个查询`(u, v, x)`使用公式回答`dist[u][v] + x * (cnt[u][v] + 1)`，因为节点数等于边加一。 
6.如果`dist[u][v]`仍然是无穷大，输出`-1`因为不存在路径。 

### 为什么它有效

 每个有效路径从`u`到`v`对应于一系列边。 该算法确保对于每对节点，我们存储尽可能小的总边权重，并且在这些权重中我们保持一致的表示结构，以保留边的数量。 由于所有边权重均为非负，因此 Floyd-Warshall 可以正确计算最短路径总和。 等待成本仅取决于访问节点的数量，而这恰好由所选路径中的边的数量决定。 因此，一旦固定了最佳边权重路径，查询成本就变成了一个简单的线性函数`x`应用于预先计算的常量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

def solve():
    n, m, q = map(int, input().split())
    
    dist = [[INF] * (n + 1) for _ in range(n + 1)]
    cnt = [[INF] * (n + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        dist[i][i] = 0
        cnt[i][i] = 0
    
    for _ in range(m):
        u, v, w = map(int, input().split())
        if w < dist[u][v]:
            dist[u][v] = w
            cnt[u][v] = 1
    
    for k in range(1, n + 1):
        for i in range(1, n + 1):
            if dist[i][k] == INF:
                continue
            for j in range(1, n + 1):
                if dist[k][j] == INF:
                    continue
                
                nd = dist[i][k] + dist[k][j]
                nc = cnt[i][k] + cnt[k][j]
                
                if nd < dist[i][j]:
                    dist[i][j] = nd
                    cnt[i][j] = nc
                elif nd == dist[i][j] and nc < cnt[i][j]:
                    cnt[i][j] = nc
    
    out = []
    for _ in range(q):
        u, v, x = map(int, input().split())
        if dist[u][v] == INF:
            out.append("-1")
        else:
            out.append(str(dist[u][v] + x * (cnt[u][v] + 1)))
    
    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先构建两个矩阵：一个用于最短路径权重，另一个用于跟踪有多少条边达到最佳权重。 Floyd-Warshall 一致地更新两者，以便每一对都存储规范的最短路径表示。 然后，每个查询使用预先计算的边数计算一个简单的线性表达式。 

一个常见的错误是尝试重新计算每个查询的最短路径或忽略节点访问和边计数之间的依赖性。 另一个微妙之处是确保仅当路径严格更好或权重相同但边缘计数更好时才更新边缘计数。 

## 工作示例

 考虑一个具有三个节点和边的小图：`1 -> 2 (2)`,`2 -> 3 (2)`,`1 -> 3 (10)`。 

询问`(1, 3, x)`。 

| 步骤| 距离[1][3] | cnt[1][3] | cnt[1][3] | 选择的道路|
 | --- | --- | --- | --- |
 | 直接初始化| 10 | 10 1 | 1→3 |
 | 通过 2 | 4 | 2 | 1→2→3 |
 | 决赛| 4 | 2 | 1→2→3 |

 为了`x = 0`，答案是`4`。 为了`x = 5`，答案是`4 + 5*3 = 19`。 

这表明该算法正确地选择了具有较小边权重的路径，即使它使用更多边。 

现在考虑两条路径具有相同权重的情况：`1 -> 2 (1)`,`2 -> 3 (1)`,`1 -> 3 (2)`。 

| 步骤| 距离[1][3] | cnt[1][3] | cnt[1][3] | 选择的道路|
 | --- | --- | --- | --- |
 | 直接初始化| 2 | 1 | 1→3 |
 | 通过 2 | 2 | 2 | 1→2→3（抢七）|
 | 决赛| 2 | 2 | 1→2→3 |

 这里，两条路径的总权重相等，均为 2，但仅当权重相等时，算法才会选择边数较少的一条； 由于两者都是有效的，因此它保留边数较少的一个，以确保正确的节点成本评估。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n3 + q) | Floyd-Warshall 超过 n ≤ 500 加上每次查询 O(1) |
 | 空间| O(n²) | 两个 n×n 矩阵用于距离和边数 |

 立方预处理是可以接受的，因为`n = 500`，提供大约 1.25 亿个转换，这符合经过修剪的优化 Python 的时间限制。 查询处理是线性的`q`，这是必要的，因为`q`可以很大。

 一种边缘情况是当`u == v`。 算法初始化`dist[i][i] = 0`和`cnt[i][i] = 0`，所以答案就变成了`0 + x * (0 + 1) = x`，正确考虑起点和终点为同一节点的等待。 

另一种情况是存在多条边权重相同但边数不同的最短路径。 Floyd-Warshall 中的平局决胜确保我们始终选择边数最少的路径，这一点至关重要，因为节点成本取决于边数。 

最后，断开的对保持无限距离，我们正确输出`-1`不尝试应用公式，防止溢出或无效算术。
