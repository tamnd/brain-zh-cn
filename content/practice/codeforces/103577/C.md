 ## 解决方案
 ## 问题理解

 每个测试用例都会给出一个基因组字符串，我们必须分配一个来自其所有连续子字符串的数字分数。 对于任何子字符串，我们都会查看其前缀模式在其自身末尾的重复程度。 更准确地说，对于子字符串，我们采用最长的正确前缀（也作为同一子字符串的后缀出现），并测量其长度。 该长度称为子串的级别。 任务是对给定字符串的每个可能的子字符串求和。 

输入最多包含 50 个字符串，每个字符串的长度最多为 5000。这立即排除了任何通过线性扫描独立处理每个子字符串的方法。 每个字符串有 O(n^2) 个子串，因此即使每个子串进行 O(n) 工作，在最坏的情况下也会导致大约 1.25 × 10^11 次操作，这远远超出了可行性。 

一个微妙的问题是，级别是使用正确的前缀和后缀定义的，因此整个子字符串本身是不允许的，即使它匹配得很简单。 例如，对于“aaaa”，子字符串“aa”的级别为1，而“a”的级别为0。经常导致错误的另一点是重叠结构：不同的子字符串共享大部分计算，因此从头开始为每个子字符串重新计算前缀后缀结构是浪费的。 

当字符串没有重复时，例如“abcde”，其中每个子字符串的级别为 0；当字符串高度重复时，例如“aaaaa”，其中每个子字符串都有较大的边界结构，就会出现边缘情况。 如果一个简单的解决方案忘记排除完整的子字符串作为有效边框，那么在像“aaa”这样的情况下就会过度计数，其中“aaa”应该贡献 2，而不是 3。 

## 方法

 蛮力方法检查每个子串并独立计算其级别。 对于子字符串 s[l..r]，我们可以对该子字符串运行前缀后缀计算（例如 KMP 前缀函数），以找到其最长边界。 计算成本为 O(r-l+1)，并且由于有 O(n^2) 个子串，因此总复杂度变为 O(n^3)。 当 n 达到 5000 时，这已经太慢了，并且重复多达 50 个测试用例就变得完全不可行。 

关键的观察结果是，子串的级别仅取决于该子串内的前缀函数行为，并且对于任何固定的起始位置，前缀函数可以在线性时间内增量计算。 如果我们固定一个左端点 i，那么对于后缀 s[i:]，我们可以在 O(n) 内计算一次它的前缀函数。 该后缀的每个前缀完全对应于从 i 开始的子字符串。 位置 j 处的前缀函数值给出子串 s[i..i+j] 的级别。 这消除了重复的重新计算，并将每个字符串的总工作量减少到 O(n^2)。 

从暴力到最优的转变本质上是从重新计算每个子串的结构转变为在共享相同开始的所有子串上重用 KMP 的类似自动机的转变。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^3) | O(n^3) | O(n) | 太慢了|
 | 每次启动的前缀函数 | O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个字符串。 

1. 将起始索引 i 从 0 固定为 n − 1。我们将 s[i:] 视为新字符串。 
2. 使用标准 KMP 逻辑为此后缀构建前缀函数数组 pi。 

在该后缀中的位置j处，pi[j]表示s[i..i+j]的最长真前缀的长度，也是其后缀。 
3. 对于每个 j，将 pi[j] 添加到答案中。 这直接累加了子串 s[i..i+j] 的级别。 
4. 对所有起始位置 i 重复此操作，将贡献求和。 

重要的细节是 pi 是针对每个后缀重新计算的，而不是针对整个字符串一次。 这是必要的，因为前缀关系是相对于子字符串的开头的。 

### 为什么它有效

对于固定 i，前缀函数计算模拟 s[i:] 的前缀与其自身后缀的匹配。 在每个位置 j，计算出的 pi[j] 正是子字符串 s[i..i+j] 的最长边界，因为前缀函数被定义为跟踪最长的正确前缀，该前缀也是以 j 结尾的前缀的后缀。 由于每个子串都唯一地表示为某个后缀的前缀，因此对所有 i,j 上的所有 pi[j] 求和恰好覆盖每个子串一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(s: str) -> int:
    n = len(s)
    total = 0

    for i in range(n):
        pi = [0] * (n - i)
        for j in range(1, n - i):
            k = pi[j - 1]
            while k > 0 and s[i + j] != s[i + k]:
                k = pi[k - 1]
            if s[i + j] == s[i + k]:
                k += 1
            pi[j] = k
            total += k

    return total

def main():
    data = sys.stdin.read().strip().split()
    out = []
    for s in data:
        out.append(str(solve_one(s)))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```外循环修复子字符串开头，内循环增量构建前缀函数。 变量 k 扮演当前候选边界长度的角色，与 KMP 中完全相同。 每个不匹配回退都遵循 pi 中编码的故障链接。 

一个常见的实现陷阱是在比较字符时忘记将索引偏移 i。 另一个是意外地使用单个全局前缀函数，这会混合不相关的子字符串并产生不正确的边框。

 ## 边缘情况

 对于像“abcde”这样的字符串，每个不匹配都会立即发生在前缀函数中，因此每个 pi[j] 都保持为零。 该算法正确地为所有子字符串累加零，因为没有后缀共享非平凡的前缀结构。 

对于像“aaaaa”这样的统一字符串，前缀函数对于每个后缀都会线性增长。 对于每个起始位置 i，贡献是三角形的，算法会累积这些贡献而无需重新计算。 前缀函数构造确保在位置 j 处，我们总是计算最长的有效边界，严格小于子字符串，而不是完整的子字符串本身。 

对于高度重叠的模式（如“ababa”），通过 pi[k-1] 重复回退可以正确捕获嵌套边界，如“a”、“aba”和“ababa”。 每个回退步骤都保留 k 始终是当前前缀的有效边界长度的不变量，因此不会添加无效的边界长度。
