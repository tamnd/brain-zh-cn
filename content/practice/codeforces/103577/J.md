 ## 解决方案
 ## 问题理解

 我们得到了一个简单的多边形，绘制在单位正方形的矩形网格的顶部。 多边形的每个顶点都位于整数坐标上，并且多边形边是连续顶点之间的直线段。 

任务是确定必须切掉多少个单位网格方格，以便多边形完全包含在删除的区域中，但限制是每个单位方格要么被完全截取，要么根本不被截取。 换句话说，伊万不允许部分切割正方形。 如果正方形的任何部分与多边形重叠，则必须删除该正方形，否则该多边形将无法完全包含在删除的集合中。 

因此输出是至少以非空方式与多边形相交的网格单元的数量。 

网格很大，每个维度最多有 100,000 个，但多边形本身很小，最多有 50 个顶点。 这种不对称性是关键的结构提示：我们不会迭代所有网格单元，而是根据网格结构处理多边形。 

天真的解释会尝试测试多边形边界框中的每个单位正方形并检查与多边形的交集。 如果边界框的边长达到 100,000，则意味着多达 10^10 个单元，即使对每个单元进行恒定时间几何测试，这也是完全不可行的。 

当多边形边完全沿着网格边界延伸或穿过网格顶点时，会出现更微妙的边缘情况。 例如，如果一条边恰好位于 x = k 或 y = k 线上，则粗心的相交规则可能会重复计数或错过整个单元条带。 另一个问题是仅在顶点处接触网格线的多边形：不一致地处理顶点相交将导致扫描线计数中出现差一错误。 

## 方法

 这个问题相当于计算一个简单的多边形“接触”了多少个单位单元。 如果多边形与其内部或边界相交，则对像元进行计数。 由于顶点是整数坐标，因此多边形和网格之间的每个相关交互都发生在整数对齐的水平线和垂直线上。 

暴力破解的想法很简单：迭代多边形边界框中的每个网格单元，并检查多边形是否与该正方形相交。 每个单元格的多边形内点或线段正方形相交测试是正确的。 然而，边界框可以大到 10⁵ x 10⁵，从而产生 10^10 个候选单元，这远远超出了任何可行的运行时间。 

关键的观察是我们不需要独立地推理单个细胞。 相反，我们可以逐行处理网格。 对于 y 和 y + 1 之间的固定水平条带，该行中的所有单元格对应于 x 轴上的单位间隔。 如果我们知道多边形与水平线相交的 x 间隔，我们可以直接计算有多少个整数单位线段与其重叠。 

这将问题转化为扫描线计算：对于每条水平线 y + 0.5（表示一行单元格的中间），计算多边形与该线的交集，产生一个或多个 x 区间。 每个区间根据其覆盖的整数段数量贡献多个完整单位单元。 

由于 n 至多为 50，h 至多为 10⁵，因此我们可以为每一行独立地重新计算这些交集。 每行都需要检查多边形的所有边，总共大约有 5 × 10⁶ 边检查，这是可以接受的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有细胞进行暴力破解 | O(wh · n) | O(wh · n) | O(1) | O(1) | 太慢了|
 | 多边形边缘上的行扫描线 | O(h·n log n) | O(h·n log n) | O(n) | 已接受 |

 ## 算法演练

我们逐行处理网格。 每行对应于水平线 y + 0.5，该线穿过该行中所有单元格的中心。 

1. 将行索引 y 从 0 固定为 h − 1。我们将计算该行中有多少个单位正方形与多边形相交。 
2. 对于该行，计算多边形边与高度 y + 0.5 处水平线之间的所有交点。 对于每条边，我们检查它是否严格地在其端点之间穿过这条水平线。 如果是，我们使用线性插值计算交点的 x 坐标。 
3. 收集所有交叉点的 x 坐标。 因为多边形很简单，所以这些交点形成了一个可以排序的序列，并且多边形在进入和离开扫描线之间交替。 排序后，连续的对定义多边形内部的 x 间隔。 
4. 对于每个区间 [l, r]，我们计算有多少个整数单元线段 [x, x + 1] 与其相交。 如果该行中的单位单元的间隔与多边形投影重叠，则包含该单位单元。 这减少了对整数 x 的计数，使得 x < r 且 x + 1 > l，可以计算为 max(0, Floor(r − ε) − ceil(l) + 1)。 
5. 将所有时间间隔内的贡献相加并累积成最终答案。 
6. 对所有行重复并输出总计。 

关键的实现细节是对接触扫描线的边缘的一致处理。 每条边都应准确地计入其两个端点行之一，以避免重复计算。 这是通过将 y 的半开区间上的边视为活动边来处理的。 

### 为什么它有效

 该算法依赖于固定的扫描线不变量：对于任何水平线 y + 0.5，多边形在不相交间隔的集合中与其相交，并且这些间隔准确地描述了在标准奇偶规则下该线上的哪些点位于多边形内部。 由于 y 行中的每个单位正方形都包含点 (x + 0.5, y + 0.5)，因此计算有多少个这样的点落在多边形内相当于计算相交的单元格。 从连续间隔到整数计数的转换是精确的，因为每个晶胞对应于该扫描线上的唯一整数 x 间隔。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, w, h = map(int, input().split())
    poly = [tuple(map(int, input().split())) for _ in range(n)]

    # close polygon
    poly.append(poly[0])

    ans = 0

    for y in range(h):
        y_line = y + 0.5
        xs = []

        for i in range(n):
            x1, y1 = poly[i]
            x2, y2 = poly[i + 1]

            # check if edge crosses the horizontal line
            if y1 == y2:
                continue

            # ensure y1 < y2
            if y1 > y2:
                x1, x2 = x2, x1
                y1, y2 = y2, y1

            if y1 <= y_line < y2:
                t = (y_line - y1) / (y2 - y1)
                xs.append(x1 + t * (x2 - x1))

        xs.sort()

        # pair up intersections
        for i in range(0, len(xs), 2):
            l = xs[i]
            r = xs[i + 1]
            if r <= l:
                continue

            L = int(l + 1e-12)
            R = int(r - 1e-12)

            if R >= L:
                ans += R - L + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该代码遵循每行扫描线的方法。 对于每个水平条带，它从所有多边形边收集 x 交点，对它们进行排序，并将它们成对处理以形成内部线段。 每个段的最终整数计数是通过将浮动间隔转换为整数覆盖的单位单元来计算的。 

一个微妙的实现细节是半开区间条件`y1 <= y_line < y2`，这可以防止共享顶点处的重复计算。 如果没有此规则，两条边共享的顶点将产生重复的交点，从而破坏区间配对。 

从浮点区间端点到整数边界的转换使用小的 epsilon 移位来稳定浮点边界情况。 这是必要的，因为由于精确的算术结构，交点坐标可能非常接近整数。

 ## 边缘情况

 关键的边缘情况是多边形顶点恰好位于扫描线 y + 0.5 上。 在这种情况下，天真地包含两个入射边将产生重复的 x 交点，从而导致不正确的配对。 半开区间规则 y1 <= y_line < y2 确保每个顶点恰好对一条扫描线有贡献。 

另一种情况是水平多边形边。 水平边不产生交叉点，因为它们完全位于扫描线边界内并且不定义进入或退出事件。 包括它们会人为地增加交叉口数量。 

最后，精确沿着网格线运行的多边形需要仔细的浮点处理。 如果没有一致的 epsilon 策略或稳健的整数算术，在将连续间隔转换为整数单元格范围时，交集端点可能会产生相差一的错误。
