 ## 解决方案
 ## 问题理解

 我们得到一个或多个无向多重图。 每条边连接两个顶点并带有正整数权重。 该任务在图表上定义了一个函数，看起来我们可以选择边的子集，但是所选子集的贡献是乘法的：对于每个选定的边，我们乘以它的权重，除了一些边是“禁止的”并且贡献为零。 

更准确地说，是有重量的边缘$w$贡献$w \times p(w)$， 在哪里$p(w)$仅当$w$是奇数，否则为0。 因此，如果包含任何重量均匀的边缘，则会立即杀死整个产品。 这意味着任何最佳子集都不会包含偶数权重边，因为它们迫使乘积为零。 

因此，问题简化为选择一些具有奇数权重的边子集，最大化它们的权重的乘积。 如果我们选择无边，则按照惯例，乘积为 1（空乘积）。 

输入大小最多允许$10^5$节点和$10^5$每个测试用例都有边，因此任何解决方案在边数上都必须接近线性。 二次或偶数$O(m \log m)$具有大量常量是可以的，但是任何尝试枚举子集或运行成对组合的操作都是不可能的。 

微妙的边缘情况是所有边缘都是均匀的。 那么每个非空子集的乘积都是 0，因此选择空集的正确答案是 1。 将答案初始化为 0 或始终乘以至少一个边的简单实现在这里将失败。 

另一种边缘情况是同一节点之间有多个边缘。 由于边在产品中是独立的，因此重复在结构上并不重要，但它们在数字上很重要，并且如果是奇数则必须全部考虑。 

## 方法

 蛮力解释很简单：尝试每个边的子集，计算$w(e)$覆盖权重为奇数的边，并忽略包含偶数权重边的子集，因为它们的计算结果为零。 这是正确的，因为定义明确地乘以所有选定的边。 然而，子集的数量是$2^m$，这对于$m = 10^5$是完全不可行的。 

关键的简化来自于观察到图结构是不相关的。 除了乘法之外，边之间没有交互作用。 唯一的决定是是否包含每个奇数权重边，并且每个包含项独立地将乘积增加其权重的一个因子。 由于所有权重都是大于或等于 1 的正整数，因此包含每个奇数权重边永远不会减少乘积。 没有任何限制阻止将它们全部拿走。 

因此，最佳策略是将所有奇数权重相乘。 如果没有奇数边，则结果为 1。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^m)$|$O(m)$| 太慢了|
 | 最佳 |$O(m)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理每个图。 

1. 初始化累加器`ans`到 1。这表示到目前为止所有选定的有效边的乘积。 从 1 开始可确保跳过所有边产生正确的标识值。 
2. 逐条读取每条边。 对于有重量的边缘`w`，检查是否`w`很奇怪。 
3.如果`w`是均匀的，完全忽略边缘。 它没有贡献任何有用的东西，因为它的贡献为零，并且包含它的任何子集都会迫使乘积为零，这总是比跳过它更糟糕。 
4.如果`w`是奇数，乘以`ans`。 这对应于在最佳子集中选择该边。 
5.处理完所有边后，输出`ans`。 

关键的推理步骤是，由于乘法是结合性和交换性的，并且所有选定的值都是正数，因此排除奇数边永远不会有任何好处。 

### 为什么它有效

 每个可行的解决方案都对应于选择边的子集。 包含偶数权重边的任何子集的乘积为零，这严格受空子集支配，除非所有奇数边乘积也为零，但它们并非如此，因为奇数权重至少为 1。因此，最佳子集仅由奇数边组成。 在奇数边中，每条边都贡献一个大于或等于 1 的乘法因子，因此包含所有奇数边可以使乘积最大化。 边之间不存在依赖关系，因此贪心选择是全局最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    it = iter(data)
    out = []

    while True:
        try:
            n = int(next(it))
        except StopIteration:
            break
        m = int(next(it))

        ans = 1

        for _ in range(m):
            w = int(next(it))
            u = int(next(it))
            v = int(next(it))

            if w % 2 == 1:
                ans *= w

        out.append(str(ans))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案批量读取所有输入，以避免重复的 I/O 开销。 每个边沿均在恒定时间内进行处理，并进行简单的奇偶校验。 节点`u`和`v`与计算无关，只读以正确推进输入流。 

一个常见的实现错误是尝试构建邻接表或推理连通性，但图结构在目标函数中不起任何作用。 另一个微妙的问题是忘记将答案初始化为 1 而不是 0，这会错误地将所有输出强制为零。

 ## 边缘情况

 一个关键的边缘情况是每条边的权重相等。 对于输入如：```
3 2
2 1 2
4 2 3
```算法初始化`ans = 1`，然后跳过两条边，保持结果不变。 这与所需的空子集值匹配。 

另一种情况是单个较大的奇数权重，例如：```
2 1
999999937 1 2
```该算法将其直接乘以累加器，产生正确的结果，而不会出现 Python 中由于任意精度整数而导致的溢出问题。 

最后一种情况是具有所有奇数边缘的大输入尺寸。 该算法仍然对每个边缘执行一次乘法，保持线性行为。
