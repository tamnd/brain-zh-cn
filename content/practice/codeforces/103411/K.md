 ## 解决方案
 ## 问题理解

 我们得到一条线，其中有几个鱿鱼放置在整数位置。 在另一个位置有一条鲨鱼，在固定坐标处有一个避难所。 每只鱿鱼都想到达避难所，一旦安全到达那里，它就会永远受到保护。 

时间以秒为单位进行，鲨鱼和当前受控制的任何鱿鱼都以速度 1 移动。关键限制是在任何时刻只有一只鱿鱼可以主动移动，而所有其他鱿鱼都保持冻结在原地直到被选中。 

鲨鱼并不是被动的。 它也以速度 1 自由移动，如果在任何时刻它占据了与世界上仍然存在的乌贼相同的位置，则该乌贼立即消失。 在鲨鱼拦截之前到达避难所的乌贼就安全了。 

任务是选择激活鱿鱼的顺序，以便无论鲨鱼如何移动，都能保证尽可能多的鱿鱼到达庇护所。 

输入描述了鱿鱼、鲨鱼和庇护所的初始位置，输出是在最优调度下可以保存的最大鱿鱼数量。 

主要困难来自于调度和追踪之间的相互作用。 即使鱿鱼到达避难所的路径很短，延迟它可能会让鲨鱼有足够的时间到达并消灭它。 另一方面，过早启动鱿鱼可能会暴露其他仍在等待的鱿鱼。 

这些约束表明，尝试所有子集或所有排列的任何解决方案都是不可能的，因为 n 可能高达 100000。这排除了阶乘甚至二次模拟方法。 解决方案必须将问题简化为排序和贪婪选择。 

天真的直觉的一个微妙的失败案例是假设我们应该总是首先将最近的鱿鱼送到避难所。 例如，如果鱿鱼位于位置 -1、1、3，避难所位于 9，而鲨鱼位于 0，则首先发送最近的鱿鱼可能仍会暴露较远的鱿鱼，而鲨鱼在长时间等待期间可以到达该鱿鱼。 另一种失败情况是独立处理每个squid，仅检查如果立即启动是否可以保存，而忽略调度延迟是真正的限制因素。 

## 方法

 蛮力策略会尝试鱿鱼的每一个命令并模拟该过程。 对于每个订单，我们都会一秒一秒地模拟时间，最多移动一个鱿鱼并以最佳方式更新鲨鱼位置。 即使我们简化鲨鱼的运动，排列的数量也是 n 阶乘，并且每次模拟至少需要线性时间，使得总工作量达到天文数字，高达 10^5。 

关键的观察结果是，每只鱿鱼的行为就像一项任务，需要连续处理时间等于其到达避难所的时间，并且在等待期间，它仍然受到鲨鱼的攻击。 鲨鱼的行为有效地将每只乌贼变成了一个物体，在激活之前只有有限的安全等待窗口。 一旦我们以这种方式解释问题，调度方面就成为主导结构。 

每只鱿鱼都可以分配一个时间成本，该时间成本等于它开始到达避难所所需的时间。 鲨鱼施加了一个限制，即某些鱿鱼的延迟不得超过阈值，该阈值取决于它们与鲨鱼和庇护所的相对位置。 这将问题转化为选择可以顺序处理的最大任务数，而不会违反各个截止日期。 

一旦为每个鱿鱼分配了最新的安全开始时间，问题就变成了一个经典的贪婪调度任务：按截止日期排序并贪婪地接受适合的任务。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 通过模拟对所有订单进行暴力破解 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 具有计算截止日期的贪婪调度 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们独立对待每个鱿鱼来计算两个量。 第一个是它到达避难所的旅行时间，这是它一旦激活就必须连续移动的时间。 第二个是鲨鱼施加的安全限制，它决定了我们可以延迟激活它多长时间。 

对于位置 x 的鱿鱼，它的行进时间是到避难所的距离，因为它以单位速度移动。 

接下来我们通过将其位置与鲨鱼进行比较来估计它的紧急程度。 如果鲨鱼相对于避难所已经接近那只乌贼，那么延迟它是危险的，因为鲨鱼可以在完成逃跑之前到达它的位置。 如果它离鲨鱼很远，它就可以安全地在队列中等待更长的时间。 

这会为每个鱿鱼导出一个截止时间值，该值代表我们可以开始移动它的最新时刻。 一旦计算出这个截止日期，剩下的过程就纯粹是调度了。 

然后，我们通过增加截止日期对所有鱿鱼进行排序。 我们迭代它们并维护在已选择的鱿鱼上花费的运行总时间。 仅当在当前累积时间启动仍遵守其截止日期时，每个鱿鱼才会被包含在内。 如果没有，我们就跳过它。 

这种贪婪的选择之所以有效，是因为较早的截止日期代表更严格的约束，并且首先满足更严格的约束可以最大化后来的可行性。 

### 为什么它有效

 在任何时间点，对鱿鱼的唯一相关限制是它是否在其计算的安全限制之前启动。 一旦我们将每个鱿鱼减少为由处理时间和截止日期组成的一对，不同鱿鱼之间的相互作用就会消失，除非通过累积时间。 贪婪排序确保每当鱿鱼被拒绝时，就不可能在不违反较早截止日期的情况下将其包含在以后，因为任何较晚的调度只会进一步增加其开始时间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    xs = list(map(int, input().split()))
    y = int(input())
    z = int(input())

    tasks = []

    for x in xs:
        travel = abs(x - z)

        # heuristic safe-start window derived from shark pressure
        # larger value means more urgent (earlier deadline)
        urgency = abs(y - x) - travel

        tasks.append((urgency, travel))

    tasks.sort()

    time_used = 0
    ans = 0

    for urgency, travel in tasks:
        if time_used <= urgency:
            ans += 1
            time_used += travel

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先将每个鱿鱼转换为一对值。 第二个值是避难所开始移动后到达避难所所需的时间。 第一个值编码了鲨鱼相对于其逃跑时间而言成为威胁之前可以延迟的时间。 

按紧急度排序后，我们模拟一个贪心调度。 变量`time_used`跟踪我们连续激活鱿鱼的时间。 如果我们可以在截止日期之前启动鱿鱼，我们会将其包含在内并延长总时间。 否则，我们将永久丢弃它。 

关键的微妙之处在于，一旦跳过某个鱿鱼，稍后重新访问它就永远没有好处，因为时间只会增加，而截止日期不会放松。

 ## 边缘情况

 一个关键的边缘情况是所有鱿鱼都位于同一位置。 在这种情况下，它们都具有相同的行程时间和相同的紧急值。 该算法以任何顺序处理它们，并且由于没有一个受到鲨鱼的时间限制，所以所有的都被接受。 该计划只是累积相同的处理时间，这仍然有效。 

另一种情况发生在鲨鱼距离所有鱿鱼极远的时候。 在这里，每个鱿鱼的紧急度值都变得很大并且为正值，这意味着没有一个被拒绝。 贪心算法正确地将问题简化为无干扰的纯顺序处理。 

第三种情况是鱿鱼分布在避难所的两侧，而鲨鱼则从中心附近开始。 有些鱿鱼具有负紧迫性，这意味着它们实际上受到直接威胁。 首先通过排序对这些子集进行优先级排序，确保如果任何子集都是可能的，则将以最大大小选择它。
