 ## 解决方案
 ## 问题理解

 我们被赋予了一个网络$n$恒星系统由精确连接$n-1$双向高速公路，形成一棵树。 任何两个系统之间都存在一条简单路径。 这一结构最初是固定的，但我们可以在任何两个以前未连接的系统之间添加一条额外的高速公路。 

每个星系统贡献了许多本地定义的“结点”（该语句称为分叉）：如果一个节点有度$d$，那么它贡献了$\binom{d}{2}$，因为每对事件道路都可以通过系统内部的隧道唯一连接。 网络的总分是$\binom{\deg(v)}{2}$在所有顶点上。 

任务是选择一条新边添加，以便在相应更新度后，这些局部贡献的总和最大化，并输出所选边。 

约束条件$n \le 2 \cdot 10^5$排除所有节点对上的任何二次方。 任何解决方案都必须在线性或近线性时间内处理树。 重新计算每个候选边缘分数的解决方案需要$O(n^2)$检查，这是不可能的。 

一个微妙的点是，添加一条边会增加恰好两个节点的度数。 其他一切保持不变。 因此，整个优化减少为选择两个端点。 

一个天真的错误是假设连接两个高度节点始终是最佳的。 但这失败了，因为增益不仅取决于度数，还取决于这些度数如何通过二次表达式相互作用$\binom{d}{2}$，其中边际增益线性取决于$d$。 

## 方法

 暴力方法会尝试每对节点$u, v$，计算在它们之间添加边缘的效果，并选择最佳的。 对于每一对，我们更新两个度并重新计算贡献的全部总和，成本$O(n)$每对。 这导致$O(n^3)$总体来说，已经远远超出了极限。 

我们可以通过注意到所有节点的基线贡献是固定的，并且只有新边的端点发生变化来进行改进。 如果我们增加节点的度数$x$1 时，其贡献变化为$\binom{d}{2}$到$\binom{d+1}{2}$，给出增量$d$。 因此连接的总增益$u$和$v$简直就是$\deg(u) + \deg(v)$，独立于其他一切。 

这将问题分解为在原始树中选择两个不相邻的节点来最大化度数之和。 “尚未通过边连接”约束仅排除现有边，因此我们必须避免选择原始邻居。 

现在结构变成纯粹的组合：我们想要两个具有最大度数和的节点，不包括原始边。 按度数对节点进行排序表明最佳候选节点位于度数最高的顶点中，但我们必须确保所选对不是现有的边。 

我们可以维护邻接集并尝试贪婪地配对高度节点，检查可行性。 因为只有少数几个最重要的候选者很重要，所以我们不需要检查所有对，只要足以保证我们找到最佳的非边缘即可。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 带过滤的度配对 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 通过扫描所有边计算每个节点的度。 这是必要的，因为任何端点的增益仅取决于其当前程度。 
2. 构建邻接结构（每个节点的哈希集或排序列表），以便我们可以在常数或对数时间内检查边是否已存在。 此约束很重要，因为并非所有对都被允许。 
3. 按度数降序对所有节点进行排序。 直觉是，最优对必须来自高度节点，因为增益是度数相加的。 
4. 考虑从顶部形成的候选对$K$在此排序中的节点，其中$K$足够小以保持计算成本低，但又足够大以保证正确性。 在实践中，检查顶级中的所有对$K$（和$K \approx 100$或者$200$) 就足够了，因为任何最优对必须在最高度数的顶点中包含至少一个节点。 
5. 对于每个候选对$(u, v)$，如果它们之间没有原始边，则计算分数$deg[u] + deg[v]$，并跟踪最好的一对。 
6. 输出最佳对以及添加到原始总和中的最大可实现增益。 

### 为什么它有效

 添加边后的总贡献仅取决于其端点的度数，并且每个端点独立贡献。 因此，最大化总分相当于最大化$deg(u) + deg(v)$在这样的约束下$(u, v)$不是现有的边。 由于度是唯一的参数，任何最佳解决方案都必须涉及靠近树的最大度区域的节点，并且扫描排序度列表的足够大的前缀可以保证包含最佳端点。 邻接约束仅删除无效对，而不改变合意性的顺序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    adj = [set() for _ in range(n)]
    deg = [0] * n

    edges = []
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        adj[u].add(v)
        adj[v].add(u)
        deg[u] += 1
        deg[v] += 1
        edges.append((u, v))

    order = sorted(range(n), key=lambda x: -deg[x])

    base = 0
    for d in deg:
        base += d * (d - 1) // 2

    K = min(n, 200)
    best_gain = -1
    best_pair = (0, 1)

    for i in range(K):
        u = order[i]
        for j in range(i + 1, K):
            v = order[j]
            if v not in adj[u]:
                gain = deg[u] + deg[v]
                if gain > best_gain:
                    best_gain = gain
                    best_pair = (u, v)

    total = base + best_gain
    print(total)
    print(best_pair[0] + 1, best_pair[1] + 1)

if __name__ == "__main__":
    solve()
```该代码首先构建树，同时计算度数和邻接集。 邻接集对于快速拒绝无效候选边至关重要。 

基值计算分叉的初始数量，但优化步骤实际上并不需要它来选择边缘，仅用于输出。 

顶部的双环$K$节点是启发式约简的核心。 的选择$K$是在假设最佳端点位于高度节点之间的情况下保持算法在实践中线性的同时保持正确性的原因。 

一个常见的陷阱是忘记只有端点才重要，而不是全局结构。 另一种方法是尝试在每个假设的边缘之后重新计算贡献，这是不必要的。

按度数排列的顶部节点为 2 和 3（1 索引）。 它们已经连接，所以我们跳过它们。 下一个最佳有效对是 (1,3) 或 (1,2) 样式，具体取决于邻接性，最佳非边是 (1,3)。 

| 步骤| 你| v | 度[u] + 度[v] | 有效 | 最佳|
 | --- | --- | --- | --- | --- | --- |
 | 1 | 2 | 3 | 4 | 没有| - |

这说明了为什么最高程度的邻接必须被拒绝，即使它给出了最大的原始和。

 ## 边缘情况

 一个关键的边缘情况是两个最高度节点已经连接。 在这种情况下，天真的贪婪方法会错误地选择无效边。 该算法通过在接受候选对之前显式检查邻接性来避免这种情况。 

另一种情况是多个节点共享相同的最大度数。 在星形结构中，许多叶子的度数为 1，只有中心不同。 该算法仍然有效，因为它只关心成对和而不是绝对同一性。 

最后，在非常小的树中，例如$n=3$，除了一对之外的所有对都是无效的，并且算法正确地回退到唯一可用的非边缘，因为邻接过滤消除了非法选择。
