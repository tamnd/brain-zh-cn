 ## 解决方案
 ## 问题理解

 我们得到了字母表 {A, C, G, T} 上的一个字符串，代表一条 DNA 链。 每个字符都有一个固定的补码：A 与 T 配对，C 与 G 配对。如果我们取一个 DNA 字符串，将其反转，并用其补码替换每个字符，我们就得到了语句中所谓的反向补码字符串。 

任务是检查原始字符串与其反向补码是否相同。 换句话说，对于从左边开始的每个位置 i，i 处的字符必须与位置 n − 1 − i 处的字符的补码匹配。 

输入大小最多可达 10^6 个字符，这会立即排除任何二次方。 如果是线性的，任何直接比较所有位置对的解决方案仍然没问题，但是任何重复字符串构造或重复反转内部循环的解决方案都将超出时间限制。 

一个常见的失败案例是意外地只检查通常意义上的回文性，而不应用补码规则。 例如，字符串“ATAT”是有效的，因为 A ↔ T 且反转保留了映射后的模式，但“AAAA”即使是正常的回文，也会失败。 另一个微妙的错误是显式构造反向补码字符串； 这仍然是 O(n)，但如果在紧密循环中不小心完成，可能会引入不必要的开销或内存重复。 

## 方法

 一种直接的方法是显式构建反向补码字符串并将其与原始字符串进行比较。 我们将反转字符串，通过补码函数映射每个字符，然后检查相等性。 这是正确的，并且以线性时间运行，但它会为转换后的字符串执行额外的内存分配，并在早期检测到不匹配时进行不必要的完整构造。 

我们可以通过观察我们实际上不需要完整转换的字符串来改进这一点。 条件纯粹是成对的：在对一侧求补后，位置 i 必须与位置 n − 1 − i 匹配。 这意味着我们可以从两端向内扫描并即时验证情况。 

关键的结构见解是，问题简化为对索引对的对称约束，因此我们只需要 O(1) 额外空间和对一半字符串的单次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 构建反向补码字符串 | O(n) | O(n) | 已接受 |
 | 两指针比较 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们使用两个指针处理字符串，一个从左端开始，一个从右端开始。 在每一步中，我们都会检查左侧的字符是否等于右侧字符的补码。 

1. 初始化两个索引，l = 0 和 r = n − 1。它们表示字符串中必须满足反向补码条件的对称位置。 
2. 当 l ≤ r 时，将 s[l] 与 s[r] 的补集进行比较。 如果它们不同，我们可以立即断定该字符串不是 DNA 回文，因为该镜像对违反了定义条件。 
3. 如果它们匹配，则通过设置 l += 1 和 r -= 1 将两个指针向内移动。这保留了所有先前检查的对称对满足条件的不变式。 
4. 如果在指针交叉之前所有对都有效，则字符串满足反补对称性，我们输出 YES。 

关键的想法是，每个角色都以其镜像位置参与一个约束，因此不需要回溯或全局结构。 

### 为什么它有效

在任何时刻，该算法都会强制所有已处理的对 (i, n − 1 − i) 满足 s[i]=complement(s[n − 1 − i])。 由于每个索引都恰好参与其中一个这样的对，因此覆盖所有对可以保证正确性。 不匹配的提前终止是有效的，因为单个违反的约束会使整个条件无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

comp = {
    'A': 'T',
    'T': 'A',
    'C': 'G',
    'G': 'C'
}

n = int(input())
s = input().strip()

l, r = 0, n - 1

while l <= r:
    if comp[s[r]] != s[l]:
        print("NO")
        sys.exit(0)
    l += 1
    r -= 1

print("YES")
```补码映射在字典中预先计算以进行 O(1) 查找。 两指针循环检查对称位置，而不构造任何附加字符串。 终止通过`sys.exit(0)`确保我们在检测到违规后立即停止，这在最坏的情况下（即早期发生不匹配）是最佳的。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    comp = {'A':'T','T':'A','C':'G','G':'C'}

    n = int(input())
    s = input().strip()

    l, r = 0, n - 1
    while l <= r:

        r -= 1

 ## 边缘情况

 对于像“A”这样的单个字符，该算法将 s[0] 与补码 (s[0]) 进行比较。 由于补码(A) 是T，因此检查立即失败，正确返回NO。 这表明长度为 1 的字符串在此定义下永远无效。 

对于像“ATAT”这样的偶数长度字符串，每对都被检查一次，并且满足所有约束，因此算法继续进行而不会提前退出并返回 YES。 

对于出现在字符串早期的不匹配对称对，例如“AAA”，算法在第一次比较时立即终止。 这证实了提前退出不会影响正确性，因为单个违反的约束足以取消字符串的资格。
