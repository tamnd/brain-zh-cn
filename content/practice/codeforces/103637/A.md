 ## 解决方案
 ## 问题理解

 我们得到了从 1 到 n 的数字排列，目标是将其转换为恒等排列，其中每个位置 i 都包含值 i。 

可以进行两种操作。 一次操作让我们以固定成本 a 交换任意两个元素。 另一个操作以固定成本 b 完全随机化排列，产生所有 n! 的均匀随机排列。 的可能性。 

我们可以按任何顺序应用这些操作，包括多次重复洗牌或混合交换和洗牌。 任务不是找到确定性的最小成本，而是在最优策略下达到身份排列的最小预期成本。 

约束很小，n 最大为 20。这立即表明排列的状态空间（n！）在概念上是相关的。 尽管n！ 对于 n = 20 来说是一个天文数字，转换的结构（交换或随机重置）表明我们不希望显式枚举状态，而是将状态描述压缩为排列结构下不变的东西。 

当排列已经是恒等时，会出现微妙的边缘情况。 在这种情况下，由于不需要任何操作，预期成本为零。 另一个边缘情况是交换比预期的洗牌更昂贵，在这种情况下，重复洗牌而不是在本地修复任何内容可能是最佳选择。 例如，如果 n = 2 并且排列已经正确，则交换将产生不必要的成本，而洗牌仍会引入随机性，但可以忽略，因为立即停止是最佳的。 

## 方法

 暴力方法会将每个排列视为大小为 n! 的图中的一个状态。 从每个状态，我们可以通过支付 b（洗牌）转换到任何其他状态，或者通过以成本 a 交换任何索引对来转换到特定的邻居。 目标是身份排列，我们希望达到它的预期成本最小。 

这立即成为巨大状态空间上的期望值动态规划问题。 即使为每个排列写出方程也是不可行的。 

关键的观察结果是，身份结构仅取决于有多少元素已经处于正确位置，而不取决于它们的确切排列。 然而，交换可以根据排列中的循环一次修复多个元素，并且改组会均匀地破坏所有结构。 洗牌操作是简化的力量：它使状态再次统一，这意味着洗牌之后，无论之前的配置如何，预期的剩余成本都是相同的。 

这表明了一种战略观点，而不是逐个州的DP。 在任何状态下，我们都有两种选择：要么仅使用交换直接修复排列，要么支付 b 重置为均匀随机排列并以最佳方式再次尝试。 如果我们致力于某种策略，那么该过程就变成了确定性“从当前状态求解”成本和概率性重启循环之间的决策。 

令 E 为随机排列的预期成本。 如果我们选择立即洗牌，我们支付 b 并返回相同的期望 E，这样分支贡献 b + E。如果我们选择使用交换而不进行洗牌，我们需要最小数量的交换来对排列进行排序，比如 k，每个成本为 a，因此成本为 k·a。 

因此，从初始状态开始，我们比较两种策略：直接使用掉期完成，或者支付 b 并重新启动整个过程。 最优期望值满足定点方程：

 E = min(k·a, b + E)

 这种形式意味着如果 k·a ≤ b + E，那么我们更喜欢直接完成。 否则，重复的洗牌将占主导地位，并且方程会崩溃为几何重新启动过程。

重新排列第二种情况给出 E = b + E，这是不可能的，除非我们从不选择该分支。 正确的解释是，如果我们选择依赖洗牌，我们实际上是在重新启动，直到我们遇到一个排列，其中通过交换完成比另一个洗牌更便宜，这导致尝试的几何分布。 

解决这个问题的正确方法是计算 k，即所需的最小交换次数，然后将确定性完成成本 k·a 与重复洗牌直至成功的预期成本进行比较。 每次洗牌都会给出新的排列，因此概率为 1，最终我们会处于决定完成的状态，但预期洗牌次数为 1/p，其中 p 是随机排列“足够便宜以完成”的概率。 

关键的简化是，唯一有意义的区别是我们是否选择洗牌。 如果我们从不洗牌，答案是 k·a。 如果我们选择洗牌，最佳策略是洗牌，直到我们获得身份排列本身，因为身份是唯一保持零成本的状态。 任何非同一状态在对称性下都具有相同的结构，因此等待不会获得部分优势。 

因此，最优策略归结为比较：

 直接成本=k·a

 洗牌直至身份成本 = 预期洗牌次数直至身份乘以 b = n！ · 乙

 所以答案很简单：

 min(k·a, n!·b)

 我们仍然需要计算 k，即将排列转换为恒等的最小交换次数，众所周知，它是 n 减去排列中的循环数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 | O(n!·n) | O(n!·n) | O(n!) | 太慢了|
 | 循环分解+比较| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 通过跟踪来自每个未访问索引的指针，将排列分解为不相交的循环，直到返回到开头。 每个元素恰好属于一个循环，因为排列是双射。 
2. 计算存在多少个循环。 每个长度为 L 的循环都需要 L − 1 次交换来修复，因为我们可以将元素逐个旋转到位。 
3. 计算 k = n − Cycle_count，这是达到同一性所需的最小交换总数。 
4. 将确定性成本计算为 k·a，表示仅使用交换直接修复所有问题。 
5. 将仅洗牌的预期成本计算为 (n 阶乘)·b，表示重复随机洗牌，直到遇到同一性一次，因为同一性以概率 1/n 出现！ 每次洗牌。 
6. 输出这两个值中的最小值。 

推理步骤是，交换操作确定性和最佳地减少无序性，而洗牌充当完全重置，不会提供部分进展，因此除了这种比较之外，策略不会以任何有益的方式交织。 

### 为什么它有效

 交换下的排列结构完全由循环分解捕获，交换在循环内独立运行，循环之间没有交互。 这使得 k = n − 周期成为确定性分辨率的最小成本。 另一方面，洗牌会擦除所有结构并每次产生统一的状态，因此任何涉及洗牌的策略都会减少到重复的伯努利过程，其中只有身份状态是吸收的。 由于没有中间结构能够在洗牌后幸存下来，因此对部分正确性进行调节没有任何好处，并且该过程会崩溃为简单的预期重新启动，直到身份确定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, a, b = map(int, input().split())
    p = list(map(int, input().split()))
    p = [x - 1 for x in p]

    visited = [False] * n
    cycles = 0

    for i in range(n):
        if not visited[i]:
            cycles += 1
            cur = i
            while not visited[cur]:
                visited[cur] = True
                cur = p[cur]

    swaps = n - cycles

    direct = swaps * a

    # compute n! carefully
    fact = 1
    for i in range(2, n + 1):
        fact *= i

    shuffle = fact * b

    print(min(direct, shuffle))

if __name__ == "__main__":
    solve()
```循环计数部分遵循标准排列遍历模式。 每次我们找到一个未访问的索引时，我们都会沿着 p 行走，直到关闭一个循环，标记所有访问过的节点。 这确保每个周期都被精确计数一次。 

阶乘计算是迭代完成的，因为 n 很小，并且值适合 Python 整数而不会出现溢出问题。 

最后的比较反映了将问题简化为两个不相交的最佳策略：直接校正与重复随机重置。

### 示例 2

 n = 2，a = 1，b = 2

 有一个长度为 2 的循环，需要进行一次交换。 直接成本为 1。洗牌策略的预期成本为 2!·2 = 4，因此直接交换更好。 

该跟踪显示了循环结构如何直接决定确定性成本，而洗牌完全忽略结构。

 ## 边缘情况

 对于已经恒等的排列，例如 n = 4 且 p = [1, 2, 3, 4]，循环计数为 4，交换次数为零，因此直接成本为零。 该算法正确地输出零，因为它不考虑洗牌，除非它改善了结果，并且任何洗牌都会引入正的预期成本。 

对于像 n = 5、p = [2, 3, 4, 5, 1] 这样的单个大周期，周期计数为 1，因此交换 = 4。直接成本为 4a。 在关闭循环之前，遍历只访问每个节点一次，确保不会出现过度计数。 洗牌成本仍然与结构无关，并且仍然是 n!·b，因此无论循环形状如何，比较仍然有效。 

对于非常大的 b，该算法从不首选 shuffle，因为 n!·b 占主导地位。 排列的结构变得无关紧要，并且解决方案纯粹简化为循环分解，算法在没有任何分支或概率推理的情况下处理循环分解。
