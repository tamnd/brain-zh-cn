 ## 解决方案
 ## 问题理解

 我们有两个整数。 其中之一，$n$，确定存在多少个盒子，具体来说$2n$总共盒子。 第二个值，$a$，描述每个盒子中有多少不同的物品，并且每个盒子都是独立于其他盒子的。 

我们必须从每个盒子中准确挑选一件物品。 由于每个盒子都包含$a$选择和所有框都是独立的，做出完整选择的方法总数是所有框选择的乘积。 然后将该原始计数减少模数$2n + 2$，余数就是所需的输出。 

关键的难点是两者$n$和$a$可以大到$10^9$，所以盒子总数也很大，直接值$a^{2n}$是一个天文数字，无法明确计算。 

最重要的观察是，我们不需要完整的值，只需要它的余数模与指数结构紧密相关的数字，即$2n + 2$。 这从根本上使问题与模幂相关，模数取决于指数本身。 

如果尝试直接计算功率，则会立即出现简单的故障情况。 例如，如果$n = 10^9$和$a = 2$，值为$2^{2 \cdot 10^9}$，这是不可能计算甚至存储的。 即使是不注意结构的模幂方法在复杂性上仍然很好，但我们必须小心模数，因为它不是固定的。 

当出现另一个微妙的问题时$a$大于模数。 粗心的实现可能会假设提前减少是无害的，但因为模数取决于$n$，指数和模以一种不平凡的方式相互作用，我们必须在模幂规则下保持正确性。 

## 方法

 暴力解释很简单：对于每个框，乘以选择的数量$a$, 重复$2n$次。 这给出了$a^{2n}$。 计算出该值后，我们对其取模$2n + 2$。 

从概念上讲，这是可行的，因为每个框都贡献一个独立的乘法因子。 然而，计算本身是不可行的，因为通过重复乘法求幂需要$2n$乘法，最多可达$2 \cdot 10^9$，远远超出任何合理的时间限制。 

一个自然的改进是切换到快速求幂。 这将求幂成本从线性降低到对数$n$。 然而，更深入的认识是模数不是任意的。 正是$2n + 2$，它始终是偶数且与指数密切相关$2n$。 这种结构表明，重复平方与模约简相结合就足够了，不需要像欧拉定理这样的进一步的数论技巧，因为模数不一定是素数并且$a$不保证是互质的。 

因此，最佳方法就是用模数进行二进制求幂$2n + 2$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n)$乘法|$O(1)$| 太慢了|
 | 二进制求幂|$O(\log n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 读取整数$n$和$a$。 盒子数量为$2n$，模数定义为$m = 2n + 2$。 
2.减少基数$a$模数$m$。 这不会改变最终结果，因为模乘保留了等价性，并且使中间值保持较小。 
3. 设置指数$e = 2n$。 我们正在计算$a^e \bmod m$。 
4. 将结果初始化为$res = 1$。 这代表一个空乘积，它是乘法的中性元素。 
5. 虽然$e > 0$，一点点处理指数。 如果最低位$e$为 1，将当前结果乘以$a$模数$m$。 这确保我们准确地包含与指数的二进制分解相对应的幂。 
6.将底座摆正$a = a^2 \bmod m$。 这使我们得出下一个二的幂。 
7. 将指数右移一位，有效地将其除以 2。 

每次迭代都会以指数方式减小指数大小，这就是为什么该方法即使对于非常大的输入也是有效的。 

### 为什么它有效

 在每一步，算法都保持不变式：$res \cdot a^e$（在哪里$a$是当前的基础）与原始一致$a^{2n}$模数$m$。 指数的二进制表示确保当其相应位被设置时，每个 2 的幂只贡献一次。 由于乘法在模运算下是结合的，并且每次约简都保持同余，因此最终结果与期望值匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def mod_pow(a, e, mod):
    res = 1
    a %= mod
    while e > 0:
        if e & 1:
            res = (res * a) % mod
        a = (a * a) % mod
        e >>= 1
    return res

def solve():
    n, a = map(int, input().split())
    mod = 2 * n + 2
    exp = 2 * n
    print(mod_pow(a, exp, mod))

if __name__ == "__main__":
    solve()
```功能`mod_pow`实现二进制求幂。 基数减少模数`mod`立即避免平方过程中溢出生长。 该循环逐位处理指数，只要当前位有助于指数的分解，就会乘以结果。 

主函数计算模数$2n + 2$，将指数设置为$2n$，并将计算委托给模幂例程。 

一个常见的错误是忘记减少基模`mod`在开始循环之前。 虽然没有严格要求正确性，但它可以防止中间值变得不必要的大。 另一个微妙的问题是混淆指数和模数，因为两者都依赖于$n$。

 ## 边缘情况

 一种边缘情况是当$a = 1$。 在这种情况下，每次乘法都是中性的，无论结果如何，结果都必须始终为 1$n$。 算法初始化`res = 1`并且永远不会改变它，因为每次乘法都是以 1 为模$m$。 

另一个边缘情况很小$n$， 尤其$n = 1$，其中模数是$4$。 对于任何$a$，算法计算$a^2 \bmod 4$。 即使中间平方导致减少到零，二进制求幂也能正确处理这个问题，如示例所示，其中$2^2 \equiv 0 \pmod 4$。 

第三种情况是当$a$已经大于模数了。 该算法立即将其减少为`a %= mod`，确保后续操作保持有界且正确。
