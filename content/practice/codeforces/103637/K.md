 ## 解决方案
 ## 问题理解

 我们得到一个数组$n$整数，每个最多表示$m$位。 我们可以选择口罩$x$，也是一个$m$-位数，但有最多包含的限制$k$设置位。 选择后$x$，每个数组元素都被独立地转换，用它的当前值和它的 XOR 中的较大者替换​​它$x$。 目标是最大化此变换后数组的总和，如果多个掩码达到相同的最大和，我们必须输出最小的这样的$x$。 

该转换是每个元素本地的，但通过选择进行耦合$x$。 每一点$x$可能会以结构化的方式翻转整个数组的贡献，并且由于最大操作，效果并不是简单地逐位相加。 

这些限制强烈地塑造了问题。 我们有$n \le 10^5$，因此任何尝试所有元素子集或所有候选掩码的解决方案都是不可能的。 位宽$m \le 30$表明按位推理或每位贪婪/DP 是预期的。 约束条件$k \le m$也暗示我们正在选择位的子集$x$，因此该结构是最多 30 个位置的组合，但与数组的相互作用使得暴力破解变得不可行。 

一个天真的想法是尝试所有$2^m$可能的面具$x$，计算变换后的数组，并取最好的。 这已经是边界$m = 30$， 自从$2^{30}$大约有10亿候选人，每次评估成本$O(n)$，导致$10^{14}$运营。 

一个稍微不那么幼稚的尝试是将口罩限制在最多有$k$位，但即使这样，组合的数量也是$\sum_{i=0}^k \binom{m}{i}$，当$k \approx m/2$。 

关键的困难在于每一位$x$不独立贡献最终收益； XOR 改变位，最大操作意味着翻转位的好处取决于元素的当前值。 

当翻转一点会减少一些值但增加其他值时，会出现微妙的边缘情况，并且 max 运算符有选择地保留更好的结果。 例如，如果所有$a_i = 0$，那么任何非零$x$产生$a_i \oplus x = x$，所以每个元素变成$x$。 总和变为$n \cdot x$，最好的解决方案是最大化$x$在位约束下，这已经表明较高位占主导地位。 

## 方法

 蛮力观点对待$x$任意。 对于每个候选掩码，我们扫描所有元素，计算$a_i \oplus x$，将其与$a_i$，并累加总和。 这是正确的，因为它直接模拟操作。 问题是成本：$2^m \cdot n$，这远远超出了限制。 

我们需要了解单个位是如何$x$贡献于最终总和。 固定一点位置$j$。 如果我们切换这个位$x$，每个元素要么获得价值，要么失去价值，具体取决于在最大比较后翻转 XOR 中的该位是否会改善它。 关键的观察是每一位的贡献$x$如果我们正确解释它，就可以独立评估。 

对于每一位$j$，如果我们设置，我们可以计算总和的变化量$x_j = 1$而不是 0，假设所有其他位都是固定的。 这将问题转化为选择最多$k$具有最大正增益的位。 然而，存在一个复杂性：增益不是独立于其他所选位的固定数字，因为异或在每个数字内的各个位之间相互作用。 

解决此问题的标准方法是根据每个位在切换时如何影响整个数组来单独评估每个位。 对于固定位$j$，考虑设置该位的影响$x$。 对于每个$a_i$，我们比较$a_i$和$a_i \oplus (1 << j)$。 如果翻转增加了值，我们就获得了差值； 否则，由于最大操作，我们将一无所获。 总结所有元素得出选择位的净收益$j$。 这给出了每比特的权重。 

现在问题最多归结为选择$k$具有最大总重量的比特。 然而，我们还需要字典顺序最小的$x$在最佳答案中，因此在出现平局的情况下，我们首先选择较小的位。 

我们按权重降序对位进行排序，但是当权重相等时，我们更喜欢保留较小的位位置$x$最小。 然后我们去$k$贪婪地一点。

 | ---| ---| ---| ---|
 | 蛮力 |$O(2^m \cdot n)$|$O(1)$| 太慢了|
 | 最佳 |$O(nm)$|$O(m)$| 已接受 |

 ## 算法演练

 我们预先计算独立切换每个位的好处，然后在基数约束下选择最佳的位子集。 

1. 对于每个位位置$j$从 0 到$m-1$，通过扫描数组来计算其贡献。 对于每个元素$a_i$，我们计算$b = a_i \oplus (1 << j)$并检查是否$b > a_i$。 如果是这样，则增益为$b - a_i$。 我们将其累积为$gain[j]$。 此步骤测量如果该位可用于在$x$。 
2.计算完所有增益后，我们将每一位视为具有价值的项目$gain[j]$。 因为我们最多可以使用$k$总共位，我们要选择最多$k$具有最大总增益的位。 
3. 我们通过减小增益对位位置进行排序，但如果两个位具有相同的增益，我们首先选择较小的索引。 这种排序确保当多个解决方案产生相同的总和时，我们构造尽可能小的$x$。 
4.我们初始化$x = 0$。 我们迭代排序后的位，并且对于每个位$j$， 如果$gain[j] > 0$我们还有剩余配额$k$，我们设置位$j$在$x$, 递减$k$，然后继续。 具有非正增益的位永远不会被选择，因为它们不会提高总和​​。 
5.最后输出$x$。 

关键的隐藏步骤是认识到每个位的效果可以根据对最终总和的改进来独立评估，因为最大运算确保我们只保留改进，并且固定位的异或不依赖于其他位$x$当评估该位是否对某个元素有帮助时。 

### 为什么它有效

 当被视为候选包含时，每个位都会为总和贡献独立的非负边际增益$x$。 该变换确保对于每个元素，通过比较原始值和翻转值来完全确定单个位翻转的改进，因此不同位之间的交互$x$除了简单的增益相加之外，不会相互抵消或放大。 这使得全局目标相当于在大小约束下选择具有最大独立权重总和的位子集，并且贪婪地采用最佳加权位保留了最优性，同时按位位置打破平局确保了最小$x$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    gain = [0] * m
    
    for j in range(m):
        bit = 1 << j
        g = 0
        for x in a:
            y = x ^ bit
            if y > x:
                g += (y - x)
        gain[j] = g
    
    bits = list(range(m))
    bits.sort(key=lambda j: (-gain[j], j))
    
    x = 0
    used = 0
    
    for j in bits:
        if used == k:
            break
        if gain[j] > 0:
            x |= (1 << j)
            used += 1
    
    print(x)

if __name__ == "__main__":
    solve()
```该解决方案首先计算每个位的有用程度（如果我们决定将其包含在其中）$x$。 位和数组元素上的嵌套循环是核心计算，成本$O(nm)$。 

在计算增益之后，我们对位进行排序，以便我们总是首先优先选择较高的收益，并且为了相同的收益，我们更喜欢较低的位索引。 这种排序至关重要，因为该问题需要尽可能少的$x$在最佳解决方案中，这意味着当增益平局时更喜欢较小的位位置。 

然后我们贪婪地选择最多$k$具有正增益的位。 每个选定的位都直接插入到最终的掩码中。

该迹线表明，在约束较小的情况下，可以选择两个位，因为使用多个位不会造成任何损失。

 ## 边缘情况

 一种边缘情况是当$k = 0$。 该算法计算所有增益但不选择任何位，因此$x = 0$。 用于输入`5 3 0 / 7 7 7`，每个增益计算都是无关紧要的，因为选择步骤强制为零位，并且输出保持为 0。 

当所有增益均为非正值时，会出现另一种边缘情况。 例如，如果每个$a_i$对于其位结构来说已经是最大值，翻转任何位只会减少值。 在这种情况下，增益数组仅包含零，排序仍然放置所有位，但选择条件`gain[j] > 0`防止任何插入，产生$x = 0$，这是正确的，因为任何翻转都会减少或不会提高总和​​。 

最后一个微妙的情况是打破平局。 如果两个位产生相同的增益，例如跨位位置的值的对称分布，则排序确保首先选择较小的位索引。 这保证了在等和解中，得到的二进制数是最小的，因为较低的位对数值的贡献较小$x$并尽可能受到优先考虑。
