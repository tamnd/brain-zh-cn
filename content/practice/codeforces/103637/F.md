 ## 解决方案
 ## 问题理解

 我们得到一个类似排列的序列$p = (1, 2, \dots, n)$。 从这个序列中我们重复采样$m$元素，其中每个位置都是独立且统一地从$n$位置，因此允许重复，并且生成的多重集不一定是不同的。 这个抽样列表称为$q$。 

对于任意数组$a$，我们定义$\text{nth}(a, b)$作为$b$- 中的第一个最小元素$a$。 所以如果我们排序$a$，我们选择索引处的元素$b$按排序顺序。 

我们还给出了两个固定索引$k$和$d$。 任务是：对于每一个$m$从$d$到$n$，计算概率$k$- 整个数组中的第一个最小值$p$严格小于$d$- 随机样本中的第一个最小值$q$。 答案必须以模形式给出$998244353$。 

自从$p$是恒等排列，它的排序形式很简单。 价值$\text{nth}(p, k)$简直就是$k$。 因此，整个问题简化为比较一个固定整数$k$具有由均匀独立采样形成的多重集的随机顺序统计量$\{1, \dots, n\}$。 

约束条件允许$n$最多$3 \cdot 10^5$，因此任何独立地重新计算每个概率的解决方案$m$在线性或二次时间里是不可能的。 甚至$O(n^2)$排除了对样本大小的推理。 我们必须以大致线性或$n \log n$时间。 

一个微妙的边缘情况是$d = 1$。 然后我们在比较$k < \min(q)$，并且概率对于所有采样元素是否超过$k$。 另一个边缘情况是$d = m$，我们比较$k < \max(q)$，这会将解释翻转为至少一个采样元素大于$k$。 

一种简单的方法是模拟每个样本的采样和排序$m$，但即使通过多重集的枚举精确计算概率也会发生组合爆炸。 

## 方法

 一个直接的尝试是考虑生成所有$m$-长度序列超过$[1,n]$，计算它们的排序$d$-th 元素，并计算它超过的频率$k$。 这立即导致$n^m$状态，即使对于很小的国家来说也是完全不可行的$n$。 

一种稍微结构化的蛮力方法是修复$m$，并计算分布$d$的阶统计量$m$i.i.d. 制服取自$[1,n]$。 这是一个经典的顺序统计问题：我们可以对有多少个采样值进行求和$\le k$，或直接导出排名的二项式分布。 但是，为每个独立执行此操作$m$仍然需要花费$O(n^2)$如果认真执行，因为每个$m$需要求和至$m$州。 

关键的观察结果是该事件$\text{nth}(q,d) > k$仅取决于采样元素的数量$\le k$。 如果至多$d-1$元素是$\le k$，那么$d$-最小的必须超过$k$。 因此，问题简化为样本中“好”元素计数的二项式尾部问题。 

每个样品尺寸$m$得出二项式分布，其中每次抽签都是“好”（在$[1,k]$）有概率$k/n$。 所以好元素的数量是$X \sim \text{Bin}(m, k/n)$。 条件变为$X \le d-1$。 因此，每个答案都是二项式分布的前缀概率。 

剩下的挑战是我们需要所有的概率$m$从$d$到$n$，这表明动态复发$m$，而不是从头开始重新计算二项式和。 

我们可以迭代地维持二项式分布：当增加$m$到 1 时，我们通过添加新的“好”或“坏”元素来更新概率。 这会导致 DP 滚动$m$和$x$，但我们只需要高达$d-1$，这使状态保持可控。

 ## 算法演练

 我们根据计数重新构建问题。 每个$m$采样的元素在集合中是独立的$[1,k]$有概率$k/n$，并在$[k+1,n]$否则。 让$X_m$是采样元素的数量$\le k$。 

我们想要的事件是$\text{nth}(q,d) > k$，这相当于最多说$d-1$元素是$\le k$， 所以$X_m \le d-1$。 

### 步骤

 1. 预先计算模逆$n$，并定义$p = k/n$和$q = (n-k)/n$在模算术中。 

这允许将每个样本视为伯努利试验。 
2.维护DP阵列$dp[x]$， 在哪里$dp[x]$是处理当前电流后的概率$m$， 确切地$x$采样元素是$\le k$， 为了$x \le d-1$。 

我们从不跟踪上述值$d-1$因为它们与最终条件无关。 
3. 初始化$m = 0$:$dp[0] = 1$，所有其他状态为零。 

这代表一个空样本。 
4. 对于每个$m$从$1$到$n$，更新DP：

 每个先前的状态$dp[x]$有助于：

 搬到$x+1$有概率$p$，或停留在$x$有概率$q$。 

我们向后处理这个转变$x$以避免覆盖所需的状态。 
5.每次更新后$m$，计算答案$m$作为总和$dp[0] + \dots + dp[d-1]$。 

这正是最多的概率$d-1$采样的元素落入$[1,k]$。 
6.输出所有人的答案$m \ge d$。 

### 为什么它有效

 DP 对有多少采样元素低于或等于的完整分布进行编码$k$。 由于里面的相对顺序$[1,k]$没关系，只有计数影响是否$d$-第最小的超过$k$。 递归与独立的伯努利试验完全匹配，因此 DP 在每一步都保留了真实的二项式分布。 截断于$d-1$是有效的，因为超出它的任何状态都不会影响我们查询的事件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

n, d, k = map(int, input().split())

inv_n = modinv(n)
p = k * inv_n % MOD
q = (n - k) * inv_n % MOD

dp = [0] * (d + 1)
dp[0] = 1

res = []

for m in range(1, n + 1):
    ndp = [0] * (d + 1)

    for x in range(d):
        if dp[x] == 0:
            continue
        ndp[x] = (ndp[x] + dp[x] * q) % MOD
        if x + 1 <= d:
            ndp[x + 1] = (ndp[x + 1] + dp[x] * p) % MOD

    dp = ndp

    if m >= d:
        res.append(sum(dp[:d]) % MOD)

print("\n".join(map(str, res)))
```该实现保持滚动 DP，其中每一层对应于增加的样本大小。 该转换小心地使用新的数组以避免混合新旧概率，这会破坏分布。 

边界处理在$x + 1 = d$是安全的，因为超出的任何状态$d-1$与最终的概率无关。 我们明确避免需要高于此阈值的值。 

## 工作示例

 ### 示例 1

 输入：```
5 2 3
```这里$n=5$,$k=3$,$d=2$。 我们正在对序列进行采样并检查是否最多有一个采样元素在$\{1,2,3\}$。 

我们追踪$dp[x]$超过$x \in \{0,1\}$。 

| 米 | dp[0] | dp[0] | dp[1] | dp[1] | 总和 dp[0..1] |
 | ---| ---| ---| ---|
 | 1 | 问 | p| 1 |
 | 2 | q^2 | q^2 2pq | q^2 + 2pq |
 | 3 | ... | ... | 计算值|

 在每一步中，总和代表最多出现一个“好”元素的概率。 最终的输出是从开始的这些累积概率$m=2$。 

这表明 DP 正在准确地跟踪在以下位置截断的二项式分布：$d-1$。 

### 示例 2

 输入：```
4 1 1
```这里我们检查最小采样元素是否大于1，这意味着没有采样元素等于1。 

| 米 | dp[0] | dp[0] | dp[1+] | 答案|
 | ---| ---| ---| ---|
 | 1 | 3/4 | 1/4 | 1/4 3/4 |
 | 2 | (3/4)^2 | 休息| (3/4)^2 |

 此案例显示了以下情况下的特殊行为：$d=1$，其中只有零计数状态很重要。 DP 随样本大小减少为简单的几何衰减。

 ## 边缘情况

 当$d=1$，事件变为$X_m \le 0$，意味着没有采样元素落入$[1,k]$。 DP正确地只保留$dp[0]$，每一步都将其乘以$q$，生产$q^m$。 这与每次抽奖的概率相匹配$[k+1,n]$。 

什么时候$k=n$，每个元素都是“好”的，所以$X_m = m$。 条件$X_m \le d-1$变成$m \le d-1$，这对所有人来说都是不可能的$m \ge d$，所以所有输出都为零。 民主党反映了这一点，因为$p=1$，所以质量完全转移到状态$x=m$，超出了跟踪范围。 

什么时候$k=0$，没有任何元素是好的，所以$X_m=0$总是。 DP 将所有概率保留在$dp[0]$，并且每个输出都变为 1，与以下事实相匹配$\text{nth}(q,d)$总是大于$0$。
