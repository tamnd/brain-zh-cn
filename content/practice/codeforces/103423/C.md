 ## 解决方案
 ## 问题理解

 我们有几堆硬币，每堆代表一堆具有固定初始高度的硬币。 两名玩家从这些牌堆开始玩回合制游戏。 在每个回合中，玩家选择一些堆栈并从中取出硬币。 唯一的限制是，在该移动过程中必须总共移除至少一枚硬币，但涉及的堆叠数量或从每个选定堆叠中取出的硬币数量没有限制。 

完整的游戏完全由每次移动后所有堆栈的状态顺序决定，从初始配置开始，最终达到全零配置。 游戏的长度是直到所有东西都变空为止的移动次数。 任务是计算有多少个不同的有效移动序列产生了恰好以 K 步结束的游戏。 

每次移动都会将堆栈高度向量从一个非负整数向量更改为至少一个坐标中的另一个严格较小的向量，并且每个中间配置必须保持非负。 由于不同的中间配置代表不同的游戏，我们有效地计算将初始向量分解为 K 个连续的“递减层”的方法数量，其中每一层都将每一堆减少一些非负量，并且每一层至少严格减少一堆。 

这些约束意味着 N 和 K 都可以足够大，以至于不可能对游戏状态进行任何指数探索。 即使对于中等值，可能的移动序列的数量也会与硬币总数一起增长，因此任何分支到所有可能的移动或状态的解决方案都将立即超出时间限制。 

当某些堆栈从零开始时，会出现微妙的边缘情况。 例如，如果堆栈最初有零枚硬币，它永远不会改变并且实际上从游戏中消失。 任何正确的解决方案都必须将此类堆栈视为固定约束，而不是移动的主动贡献者，否则它将过多计算无效分解，其中考虑负或不必要的减少。 

## 方法

 直接的暴力方法将模拟所有可能的游戏。 从当前状态开始，我们将枚举从每个堆栈中减去非负数的各种方法，确保至少减少一个堆栈，并递归直到所有堆栈变为零。 即使在单一状态下，可能的移动次数也约为 (a_i + 1) 的乘积，因为每个堆栈都独立地影响我们减少它的数量。 在 K 层上，这会创建一个分支过程，其大小在 K 和硬币总数中呈指数关系。 即使对于很小的输入，这也很快变得不可行，因为每一步都会急剧增加状态数量。 

关键的结构观察是可以按列而不是按状态查看游戏。 我们不是考虑随着时间的推移而减少的硬币堆，而是考虑每个硬币堆中的每个单独的硬币被分配一个“轮数”，在该“轮数”时将其移除。 每个硬币必须恰好被移除一次，并且每次移动都对应于移除堆叠中的一组硬币。 每次移动至少移除一枚硬币的限制转化为要求每一回合都是非空的。 

这将问题转化为将所有硬币分配到 K 个有序层中，其中层 j 恰好包含在移动 j 中移除的硬币。 每个堆栈 i 必须将其 a_i 硬币分配为 K 个非负部分，其总和为 a_i。 在所有堆栈中，每一层都必须从某个地方接收至少一枚硬币。 因此，我们正在计算选择 K 个向量的方法数量，每次移动一个向量，以便坐标总和与初始配置匹配，并且没有向量完全为零。

这种结构是典型的“排除空列的星形和条形”情况，但每个堆栈独立应用，然后通过每层非空的要求进行耦合。 解决这种耦合的简洁方法是首先忽略非空约束并对所有分布进行计数，然后使用包含空移动来减去无效情况。 这导致了一个 DP 或组合公式，其中我们对每个堆栈计算其硬币如何在 K 个移动中分割，然后通过对回合子集的标准包含排除来强制没有全局为空的移动。 

蛮力之所以有效，是因为它直接探索所有移动序列，但会失败，因为每个移动都会产生巨大的分支因子。 观察到每个堆栈在 K 次移动中独立地贡献其值的组合，从而将问题简化为计算组合，然后跨列强制执行全局非空。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | K 和 a_i 之和的指数 | 指数| 太慢了 |
 | 分布上的组合 DP + 包含-排除 | O(NK + K^2) 或 O(NK) 取决于实现 | O(K) | 已接受 |

 ## 算法演练

 我们重新解释该过程，为每个堆栈中的每个单位硬币分配一个从 1 到 K 的移动索引。对于堆栈 i，选择每次移动中删除多少硬币对应于选择 a_i 到 K 个部分的弱组合。 

对于每个堆栈，将 a_i 个相同硬币分配到 K 个标记箱中的方法数量为 C(a_i + K - 1, K - 1)。 然而，这计算了所有堆栈中某些容器可能为空的分布，这对应于某些移动不执行任何操作的无效游戏。 

为了强制每次移动在全球范围内至少移除一枚硬币，我们对这组移动应用包含-排除。 

我们按如下方式进行。 

1. 对于每个堆栈 i，计算类似多项式的贡献，其中通过将 a_i 项分配到 K 个箱中的组合隐式捕获 K 个移动的配置系数。 
2. 通过将所有堆栈的贡献相乘（在卷积意义上）与分配给每次移动的硬币数量相乘来合并所有堆栈。 这构建了我们在不强制执行非空移动的情况下将硬币分配给 K 个移动的方式的全局计数。 
3. 对移动子集应用包含-排除。 如果我们将 t 步的子集固定为空，我们就可以有效地将 K 减少到 K - t 步。 我们用交替符号对所有 t 求和，乘以 C(K, t)，因为我们选择哪些动作是空的。 
4. 对于每个简化的 K - t，将跨堆栈的分布数计算为 C(a_i + (K - t) - 1, (K - t) - 1) 的 i 的乘积。 
5. 汇总所有贡献以 1e9+7 为模。 

关键的计算任务变成了对许多值的二项式系数进行有效评估，这是使用预先计算的阶乘和模逆来处理的。 

### 为什么它有效

 每个有效游戏完全对应于将每个单位硬币分配给 {1..K} 中的移动索引，并限制每个索引至少使用一次。 每个堆栈的分配都是独立的，因此将它们堆栈在一起会给出一个产品结构。 包含-排除通过删除某些移动索引未使用的分配来纠正过度计数。 由于移动的空性是跨堆栈的全局属性，因此移动子集上的包含-排除干净地分离了堆栈之间的耦合。 这保证了有效游戏和计数作业之间的一一对应。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    maxv = max(a + [0])
    # we only need factorials up to maxv + k
    
    maxn = maxv + k + 5
    
    fact = [1] * (maxn)
    invfact = [1] * (maxn)
    
    for i in range(1, maxn):
        fact[i] = fact[i - 1] * i % MOD
    
    invfact[maxn - 1] = modinv(fact[maxn - 1])
    for i in range(maxn - 2, -1, -1):
        invfact[i] = invfact[i + 1] * (i + 1) % MOD
    
    def C(n, r):
        if n < 0 or r < 0 or r > n:
            return 0
        return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD
    
    # inclusion-exclusion over empty moves
    ans = 0
    for empty in range(0, k + 1):
        sign = -1 if empty % 2 else 1
        ways_choose_empty = C(k, empty)
        kk = k - empty
        if kk == 0:
            continue
        
        ways = ways_choose_empty
        for x in a:
            ways = ways * C(x + kk - 1, kk - 1) % MOD
        
        ans = (ans + sign * ways) % MOD
    
    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该代码从阶乘预计算开始，以支持快速二项式系数评估，因为包含-排除中的每个项都需要多个组合。 C 函数会仔细保护无效参数，以避免负索引或无效组合解释。 

主循环迭代有多少步被强制为空。 选择哪些空动作贡献 C(K, 空动作)，剩余的 K - 空动作必须接收所有硬币分配。 每个堆栈通过星条项 C(a_i + kk - 1, kk - 1) 独立贡献，并且它们会成倍增加，因为堆栈是独立的硬币来源。 

一个常见的微妙故障点是忘记跳过 kk = 0 的情况，因为只有当所有 a_i 都为零时，将硬币分配到零步才有效，这已经被隐式处理了。

 | 0 | 2 | 1 | C(3,1)=3 | C(3,1)=3 | C(4,1)=4 | C(4,1)=4 | 12 | 12

 | 2 | 0 | 1 | 无效| 无效| 0 |

 现在包含-排除得到 12 - 2 = 10。 

这证实了算法正确地区分了全局范围内没有收到硬币的情况。 

### 示例 2

 输入：

 1 3

 2

 我们在 3 个动作中分配 2 个硬币，所有动作都非空。 

| 空动作| kk | 选择空 | 方式|
 | --- | --- | --- | --- |
 | 0 | 3 | 1 | C(4,2)=6 | C(4,2)=6

 ## 边缘情况

 当所有筹码都为零时，唯一有效的游戏的步数为零。 该算法可以处理此问题，因为除非 kk = 0，否则所有组合项 C(0 + kk - 1, kk - 1) 都会变为零，并且包含-排除会折叠为单个有效配置。 

当 K 为 1 时，唯一可能的顺序是一次完全移除。 该公式简化为一项，没有空动作，并且每个堆栈贡献 C(a_i, 0) = 1，恰好给出一个有效游戏。 

当 K 超过硬币总数时，大多数配置都会消失，因为星条形项强制为零，并且包含排除正确地消除了不可能的分配，其中移动不可能都是非空的。
