 ## 解决方案
 ## 问题理解

 该页面是用一组固定的水平引导线（等距一厘米）和一组放置在任意 x 坐标处的垂直引导线绘制的。 水平线和垂直线的每个交点都是一个格点，我们被要求计算可以形成多少个轴对齐的正方形，这些正方形的角都位于这些交点上。 

通过选择两条水平线和两条垂直线来确定正方形，使得所选水平线之间的水平间距等于所选垂直线之间的垂直间距。 由于水平线间距相等，因此正方形的垂直边长完全可以通过在两条水平线之间选取恰好 k 个连续厘米的间隙来确定。 一对垂直线之间的距离必须具有相同的 k。 

因此，问题归结为将水平线间隙与垂直线间隙配对：对于每个整数距离 k，我们需要知道有多少对垂直线正好被 k 分开，以及有多少对水平线具有垂直间隔 k。 最终的答案是兼容选择的总数。 

这些约束意味着在最坏的情况下直接枚举所有垂直线对是不可能的。 所有测试用例中的垂直线多达 200 万条，对的数量达到 10^12 的数量级，这立即排除了二次解。 如果不利用结构，即使构建所有成对差异的完整频率表也是不可行的。 

当存在垂直距离但超过水平间隙的数量时，会出现微妙的边缘情况。 例如，如果 m 很小并且两条垂直线相距很远，则该对根本无法形成正方形，因为没有足够的水平线来匹配该间距。 任何在不考虑此截止值的情况下计算所有垂直差异的解决方案都会过度计算。 

另一种失败模式是将问题纯粹视为“计算所有差异”并盲目乘以 m，忽略大间隙贡献为零的平方而不是负贡献。 

## 方法

 一种天真的方法是尝试每一对垂直线。 对于每对 (i, j)，计算它们的距离 d = a[j] - a[i]，然后计算有多少个水平对的间距正好为 d，如果 d < m，则为 m - d，否则为零。 这在原则上是正确的，因为对于从 1 到 m-1 的每个间隙大小 k，水平线正好形成一对，并且有 m-k 个这样的对。 

然而，枚举所有对已经花费了 O(n^2)。 当 n 总共达到 200 万时，这变得完全不可行，在最坏的情况下需要大约 10^12 次操作。 

关键的观察是垂直位置是排序的，因此对贡献仅取决于差异 a[j] - a[i]。 我们可以以滑动窗口的方式处理对，而不是显式地计算每个差异。 对于固定的左端点 i，只有那些具有 a[j] - a[i] < m 的 j 才可以做出任何贡献。 此限制将完整的对集转变为一组活动窗口，并且由于两个指针仅向前移动，因此总工作量与 n 呈线性关系。 

在每个窗口内，我们仍然需要 (m - (a[j] - a[i])) 形式的表达式之和。 扩展它允许我们将贡献分成窗口中的元素计数和它们的值的总和，这两者都可以使用前缀和来维护。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解对 | O(n²) | O(1) | O(1) | 太慢了 |
 | 带有前缀和的两个指针 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先对垂直坐标进行排序，因为只有有序差异才重要。

我们维护两个指针 i 和 r，其中 r 是最远的索引，使得从 a[i] 到 a[r] 的垂直距离仍然严格小于 m。 这定义了 i 的潜在合作伙伴的有效窗口。 

我们还维护数组的前缀总和，以便可以在 O(1) 内查询任何段的总和。 

1. 对垂直线数组进行排序并计算其位置上的前缀和。 
2. 初始化右指针r = 0。 
3. 对于从 0 到 n - 1 的每个左索引 i，前进 r 直到 a[r] - a[i] 不再小于 m。 这确保 (i, r] 中的所有索引都可以与 i 形成正方形。
 4. 对于固定的 i，考虑 (i, r] 中的所有有效 j。每个这样的对贡献 m - (a[j] - a[i]) 平方。总贡献是按总体计算的，而不是按对计算的。
 5. 将 (i, r] 中 j 的总和重写为元素计数及其值总和的组合。这允许使用前缀和对每个 i 进行恒定时间计算。
 6. 累积所有 i 的贡献。 

核心思想是我们从不显式枚举对。 相反，我们会扫描一次数组，同时保持有效合作伙伴的活跃范围。 

### 为什么它有效

 固定 i 后，算法会精确考虑那些 j，使得垂直距离是有效的正方形边长候选。 每个有效方块都由一对 (i, j) 唯一表示，其中 i < j，因此不会重复计算或遗漏任何对。 

贡献公式分解了每个有效垂直间隙的水平选择总数，并且由于水平结构仅取决于间隙大小，因此通过窗口聚合可以保留正确性，而不会丢失每对的区别。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        m, n = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort()

        # prefix sums
        pref = [0] * (n + 1)
        for i in range(n):
            pref[i + 1] = pref[i] + a[i]

        ans = 0
        r = 0

        for i in range(n):
            if r < i:
                r = i
            while r + 1 < n and a[r + 1] - a[i] < m:
                r += 1

            cnt = r - i
            if cnt <= 0:
                continue

            sum_seg = pref[r + 1] - pref[i + 1]

            # sum over j in (i, r]:
            # (m - (a[j] - a[i]))
            # = cnt*m - sum(a[j]) + cnt*a[i]
            ans += cnt * m - sum_seg + cnt * a[i]

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案依赖于排序，以便所有有效对都显示为每个 i 的连续段。 前缀和数组用于计算这些段的总和，而不需要迭代它们。 

一个常见的错误是忘记段从 i+1 开始，而不是 i，因为对必须满足 i < j。 另一个微妙的点是保持 r 单调； 它应该只向前移动，不能向后重置，否则复杂性会降低。

 ## 边缘情况

 当垂直线非常接近时，每对都位于大 m 的有效窗口中。 该算法正确地将 r 扩展到末尾并使用前缀和计算贡献，避免了二次枚举。 

当垂直线相距很远时，每个 i 的窗口几乎立即变空，产生零贡献，而无需不必要的工作。 

当所有点都聚集在一起时，对于较小的 i，r 保持在 n 附近，但 r 的单调移动确保总复杂性保持线性而不是二次。
