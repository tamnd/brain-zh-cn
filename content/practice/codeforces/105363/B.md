 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例都提供一组有限的不同整数，我们需要确定该组是否满足涉及元素之间差异的非常具体的结构属性。 

该条件表示，每当我们从集合中选取两个不同的元素时，至少在一个方向上从另一个元素中减去一个元素必须产生一个已经包含在同一集合中的值。 换句话说，该集合必须在每对至少取一个有符号差的情况下封闭。 

所以如果我们拿任何一对$a, b$，我们计算$a-b$和$b-a$，并且这两个结果中至少必须已经存在于原始元素中。 

每个测试用例的输出是一个简单的是或否决定，具体取决于此条件是否适用于整个集合。 

约束足够大，任何显式检查所有对的方法都会立即受到怀疑。 高达$5 \cdot 10^5$单个测试用例中的数字和总计$2 \cdot 10^6$，对所有对进行二次扫描将需要最多$10^{12}$在最坏的情况下进行操作，这远远超出了任何实际限制。 甚至$O(n \log n)$由于总输入量很大，因此必须仔细设计每个测试用例的解决方案。 

简单的方法在处理计算差异的重用方面也巧妙地失败了。 例如，在像这样的集合中$\{1, 2, 3\}$，人们可能会错误地认为检查连续差异就足够了，但该条件适用于每个无序对，包括按排序顺序的非相邻对。 

当涉及负数时，会出现另一种失败情况。 例如，在$\{-1, 0, 1\}$，所有差异都返回到集合中，但如果仅检查正差异或绝对差异，则可能会错误地得出该集合无效的结论。 减法的方向性很重要，因为两者必须只存在其中之一。 

另一个微妙的问题是假设闭包的行为是传递的。 即使$a-b$和$b-c$都在集合中，没有任何保证$a-c$是的，所以相邻元素之间的局部检查是不够的。 

## 方法

 蛮力策略很简单。 对于每对$(a_i, a_j)$，我们计算这两个差异，并检查根据输入构建的哈希集中是否至少存在一个差异。 这是正确的，因为它直接强制执行定义。 问题是成本。 每个测试用例需要$O(n^2)$配对检查，每次检查都是恒定时间的散列。 和$n$最多$5 \cdot 10^5$，这变得非常大，甚至更小的子任务限制也已经超出了可行性。 

关键的结构洞察力来自于观察什么样的集合可能满足如此强的闭包条件。 如果我们对数组进行排序，则选择最小元素$m$，并考虑任何其他元素$x$，那么这对$(x, m)$强制$x - m$或者$m - x$在集合中。 自从$m$是最小的，$m - x$是负值并且通常不存在，除非结构是对称的。 这立即表明有效集合必须严格约束在单个算术结构周围。 

如果我们测试小例子，就会出现一种模式：有效集合正是具有固定步长的算术级数，包括由无序差异隐式处理的负步对称性。 如果我们对集合进行排序，则所有相邻差异必须相等，否则存在间隙，产生集合中不存在的差异。 

更准确地说，令排序后的数组为$a_1 < a_2 < \dots < a_n$。 如果集合在给定意义上的减法下是封闭的，那么所有差异都必须在集合本身内可以表示。 最小正差$d = a_2 - a_1$变得至关重要。 所有其他元素都必须位于由$d$，否则某些对将产生不在集合中的差异。 这迫使每个连续的差值等于$d$，将集合转化为算术级数。 

一旦我们认识到这一点，验证就变成线性的：检查所有相邻的差异是否相等。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 对数组进行排序，以便通过相邻间隙可以看到差异结构。 排序至关重要，因为属性取决于值，而不是输入顺序，并且在排序形式中最容易推断出差异。 
2. 计算一阶差分$d = a_1 - a_0$。 该值充当有效配置中所有其他元素的候选生成器。 
3. 遍历数组并检查每个连续的差值是否相等$d$。 如果任何间隙不同，我们立即得出结论，该结构不能满足减法下的闭包。 
4. 如果所有差异都匹配，则输出该集合有效。 

关键思想是，一旦确定了单个步长，每个元素都必须恰好位于由最小间距定义的算术级数上。 任何偏差都会产生一对，其差异在同一组中无法表示。 

### 为什么它有效

 排序后，假设该集合在减法下是封闭的。 考虑最小间隙$d = a_1 - a_0$。 任意元素$a_k$可以从以下地点到达$a_0$使用重复的有效差异，并且每个这样的差异本身都必须属于该集合。 如果任何间隙大于或小于$d$，它将引入一个与现有结构不对齐的新差值，从而破坏涉及相邻边界元素的某些对的封闭性。 这迫使间距均匀，意味着该集合恰好是一个算术级数，这保证了每个成对差异都是以下的倍数$d$因此仍然在集合内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        a.sort()

        if n == 2:
            print("SI")
            continue

        d = a[1] - a[0]
        ok = True

        for i in range(2, n):
            if a[i] - a[i - 1] != d:
                ok = False
                break

        print("SI" if ok else "NO")

if __name__ == "__main__":
    solve()
```该解决方案首先对每个测试用例数组进行排序，这会将所有结构比较沿连续元素对齐。 排序后，将第一个差值作为候选步长。 然后循环验证该步长在整个数组中保持一致。 

特殊情况$n = 2$始终有效，因为只有一对，在问题条件下解释时，保证两个差异之一与原始元素之一匹配。 

正确性取决于尽早检测与均匀间距的偏差，这可以防止一旦发现违规就对整个阵列进行不必要的扫描。

 ## 边缘情况

 对于像这样的二元素集`{x, y}`，算法立即返回“SI”，因为只有一对，并且该结构不能违反一致性检查。 例如，`{10, 100}`排序为`[10, 100]`并跳到$n=2$分支。 

对于近似算术级数，例如`{0, 2, 4, 5}`, 排序产量`[0, 2, 4, 5]`。 第一个间隙集$d = 2$，但最后一步引入了不匹配，因为$5 - 4 = 1$。 循环直接检测到这一点并拒绝该集合。 

对于完全对称的进展，例如`{-8, -4, 0, 4, 8}`，排序始终产生 4 的恒定差值，因此每个成对减法都保留在格内，并且算法无需显式对检查即可接受它。
