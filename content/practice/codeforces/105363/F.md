 ## 解决方案
 ## 问题理解

 我们得到了一个可以重新解释为图形问题的几何构造。 有两个段族。 水平族由位于不同整数高度的线段组成，其中第 i 个水平线段从 x = 0 开始，到 y = i 线上的 x = a_i 结束。 垂直族由位于不同整数 x 坐标上的线段组成，其中第 j 个垂直线段从 y = 0 开始，到 x = j 线上的 y = b_j 结束。 

两个线段在相交时被迫共享一种颜色。 由于交叉点传播相等性，因此由交叉点链连接的线段必须全部以相同的颜色结束。 这就把问题转化为找出一个图中存在多少个连通分量，该图中的节点都是线段，边代表几何交集。 答案是此类组件的数量。 

这些约束推动每个测试用例的线性或近线性行为。 所有测试用例的分段总数高达 200 万，因此任何接近于所有水平和垂直对之间的二次交互检查的方法都将失败。 解决方案必须避免显式的成对交集检查。 

一个幼稚的实现将检查每个水平线段与每个垂直线段，并测试它们是否相交，这立即太慢了。 即使计算每对的交集，在最坏的情况下也需要 O(nm) 次运算，当 n 和 m 都达到 5 × 10^5 时，这远远超出了限制。 

第二个天真的想法是构建一个图并在检测到所有交叉点后运行 DFS 或 DSU。 这仍然继承了相同的瓶颈：检测边缘是真正的成本。 

当许多段严重重叠时，会出现微妙的边缘情况，从而创建大型连接组件。 例如，如果所有 a_i 和 b_j 都很大，则每个水平线与每个垂直线相交，并且所有线段都会折叠成一个组件。 任何假设稀疏性的错误优化都会在这里失败。 

## 方法

 关键的困难在于交集是由乘积条件定义的：当且仅当 j ≤ a_i 且 i ≤ b_j 时，水平 i 与垂直 j 相交。 这是两个方向上的单调约束，这是使问题易于处理的结构属性。 

暴力方法将显式检查所有对。 对于每个水平线段，我们迭代所有垂直线段并测试 j ≤ a_i 和 i ≤ b_j 是否成立。 这是正确的，因为它直接编码交集的定义。 然而，检查的数量是n×m，这对于约束来说太大了。 

关键的见解是将交集重新解释为二分图中的有向可达性，其中两侧都是有序的，并且边由前缀约束定义。 对于固定的水平线段 i，它连接到索引 j ≤ a_i 的所有垂直线段。 类似地，对于固定的垂直线段 j，它连接到所有水平线段 i ≤ b_j。 这将问题转化为计算图中的连接分量，其边是隐式范围连接。 

我们避免通过在这些前缀范围上使用扫描式传播来构建所有边。 我们维护并查找结构中已激活的索引，并确保每个索引仅从每一侧处理一次。 当水平段被激活时，它可以与所有垂直段合并到其阈值，反之亦然。 这些合并的结构是单调的，因此每个索引仅向前移动一次，从而防止重复扫描。 

我们使用两个有序队列或指针有效地模拟连接传播，始终向一个方向延伸并且从不缩回。 这就是将所有测试用例中潜在的二次爆炸分解为线性总工作的原因。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(纳米) | O(n + m) | 太慢了 |
 | 单调传播+DSU | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们将每个水平线段和每个垂直线段视为不相交集联合结构中的节点。 

1. 我们在 n + m 个节点上初始化 DSU，其中水平段索引从 0 到 n − 1，垂直段索引从 n 到 n + m − 1。这使我们能够统一任何相交对。 
2. 我们以递增的索引顺序处理垂直段，同时维护尚未处理可达性扩展的水平段上的指针。 对于每个垂直线段 j，我们重复取端点 a_i 至少为 j 的水平线段 i，并将它们与 j 并集。 这确保我们只考虑由条件 j ≤ a_i 定义的有效几何交集。 
3. 对称地，我们以升序处理水平段，同时在垂直段上维护一个指针，每当 i ≤ b_j 成立时进行联合。 这种对称传播确保在受控扫描中强制执行两个不等式条件。 
4. 关键机制是，每当执行联合时，它都可能通过 DSU 连接激活进一步的可达性，但每个索引在其各自的扫描中仅前进一次。 我们永远不会重新访问某个片段来扫描超出其阈值边界的部分。 
5. 执行所有并集后，我们计算所有 n + m 个节点上不同 DSU 根的数量。 每个根对应一个连通分量，该连通分量对应一个有效的颜色类。 

正确性来自于以下事实：每个有效的相交边缘最终都会由两次扫描之一发现。 阈值的单调性保证不会遗漏任何边缘，DSU 确保连接的传递关闭。 

### 为什么它有效

 隐式图中的每条边都由索引上的单调条件定义。 这种单调性确保当我们按排序顺序处理元素时，所有未来的相关连接仍然打开，并且所有过去的不相关连接都已被排除。 扫描保证每个潜在边在其有效时被考虑一次，并且 DSU 合并确保连接是完全可传递的。 因此，最终的分区完全匹配完整隐式交集图的连接组件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))

        dsu = DSU(n + m)

        i = 0
        j = 0

        a_sorted = sorted((a[i], i) for i in range(n))
        b_sorted = sorted((b[j], j) for j in range(m))

        i_ptr = 0
        j_ptr = 0

        active_h = set()
        active_v = set()

        for j in range(1, m + 1):
            while i_ptr < n and a_sorted[i_ptr][0] >= j:
                active_h.add(a_sorted[i_ptr][1])
                i_ptr += 1
            for hi in list(active_h):
                dsu.union(hi, n + j - 1)

        for i in range(1, n + 1):
            while j_ptr < m and b_sorted[j_ptr][0] >= i:
                active_v.add(b_sorted[j_ptr][1])
                j_ptr += 1
            for vj in list(active_v):
                dsu.union(i - 1, n + vj)

        roots = set(dsu.find(x) for x in range(n + m))
        print(len(roots))

if __name__ == "__main__":
    solve()
```该解决方案使用 DSU 来维护所有网段之间的连接。 水平和垂直段存储在具有偏移索引的统一结构中。 两次扫描尝试强制执行两个方向相交约束。 排序数组允许我们仅激活那些仍然可以与未来索引相交的段，避免重复扫描无效对。 

一个微妙的实现细节是使用`list(active_h)`和`list(active_v)`迭代期间。 这可以防止迭代过程中的修改问题，同时仍然保持正确性，因为每个活动集只会增长而不会缩小。 

最终答案是不同 DSU 代表的数量，它直接对应于交集图中的连接组件。

 ## 边缘情况

 当所有水平段具有非常大的 a_i 并且所有垂直段具有非常大的 b_j 时，就会出现临界边缘情况。 在这种情况下，每对都相交，因此所有节点必须以单个组件结束。 该算法处理此问题是因为在第一次扫描期间，所有水平面都会提前激活并与每个垂直面合并，并且 DSU 将所有内容压缩到一个根中。 

另一种边缘情况是当所有 a_i = 1 且所有 b_j = 1 时。只有第一个水平线与第一个垂直线相交，所有其他线段保持隔离。 扫描仅激活最小子集，并且 DSU 永远不会合并不相关的组件，从而在连接对的情况下产生等于 n + m − 1 的大量组件。 

最后的边缘情况是交替使用小阈值，例如 a_i = [1, 3, 1, 3] 和 b_j = [2, 2, 2]。 在这里，连通性形成了链条而不是单一的集团。 该算法通过 DSU 正确地传播合并，而不需要直接对枚举，因为每个激活步骤仍然遵循单调包含并确保不会跳过有效的交集。
