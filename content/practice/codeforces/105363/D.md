 ## 解决方案
 ## 问题理解

 我们有一个由道路连接的村庄网络，其中每条道路最初都无法使用，只有在一定的小时数后才可以使用。 所有道路都根据自己的时间表并行“解锁”。 道路一旦解锁，就可以永久使用。 

问题是确定仅使用已经解锁的道路完全可以相互到达村庄的最早时间。 用图表术语来说，我们想要最小的时间$T$这样如果我们只保留边缘$h_i \le T$，结果图是连通的。 

每个测试用例都给出一个加权无向图，其中权重代表激活时间。 我们实际上被要求找到使图连接的边权重的最小阈值。 

约束条件很大：最多$10^5$顶点和$5 \cdot 10^5$每个测试用例的边，总和$10^6$。 这会立即排除为每个候选时间从头开始重新计算连接的任何情况，例如在每个不同的所有边上重复运行 BFS/DFS$h_i$。 如果天真地完成，即使对所有边进行排序并模拟每个阈值的多个完整图重建也会太慢。 

我们还需要小心一个微妙的失败情况：如果我们没有正确地按升序处理边，那么连接性在部分处理的意义上就不是单调的。 

例如，考虑一个三角形：```
0 - 1 (10)
1 - 2 (10)
0 - 2 (100)
```幼稚的方法可能会错误地认为我们需要等待边 (0,2)，但实际上节点已经使用前两条边在时间 10 连接。 

正确答案取决于连接所有节点的最小方式的结构，而不取决于固定对之间的任何特定路径。 

## 方法

 蛮力策略是模拟时间从 1 向上增加，并在每一步检查是否由所有边形成的图$h_i \le T$已连接。 每次连接检查成本$O(n + m)$使用DFS或BFS。 在最坏的情况下，我们可能会扫描最多$10^6$如果权重又密又大，则不同的时间值会令人望而却步$O(m \cdot \max h)$甚至$O(m^2)$- 风格行为取决于实现。 这已经远远超出了极限。 

关键的观察是我们实际上不需要每次都检查。 我们只关心存在足够的边来连接所有顶点的时刻。 这正是我们使用尽可能小的最大边权重选择生成树的时刻。 

这将问题转化为经典的最小瓶颈生成树问题：在所有生成树中，我们想要最小化所使用的最大边权重的生成树。 答案是最小生成树中的最大边权。 

一旦我们认识到这一点，解决方案就变得简单了。 我们按权重对边进行排序，并使用不相交集并集 (DSU) 构建生成树。 一旦我们连接了所有顶点，当前的边权重就是答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(m \cdot (n + m))$|$O(n + m)$| 太慢了 |
 | 克鲁斯卡尔 + DSU |$O(m \log m)$|$O(n)$| 已接受 |

 ## 算法演练

 我们使用 Kruskal 的想法，但一旦实现连接，我们就会提前停止。 

1. 按开放时间对所有道路进行排序$h_i$。 这确保我们始终首先考虑最早可能的连接，这是必要的，因为我们希望最小化图连接的时间。 
2. 初始化 DSU 结构，其中每个村庄都从自己的组件开始。 最初，没有任何村庄相连。 
3. 按升序迭代排序后的边$h_i$。 对于每条边$(u, v, h_i)$，尝试合并包含的组件$u$和$v$。 如果它们已经位于同一组件中，则边缘不会更改连接性。 
4. 每次成功合并后，跟踪剩余的连接组件数量。 每一次成功的联合都会将该计数减少一。 当组件的数量变成1时，所有村庄都连接起来。 
5. 答案是当前边的权重$h_i$，因为这是图此时仅使用边完全连接的第一个时刻。 

关键思想是，我们按照边权重增加的顺序有效地构建一个生成森林，而当它变成一棵树的那一刻正是实现连接的时候。 

### 为什么它有效

 随时$T$，可用边的集合恰好是权重最多的所有边$T$。 Kruskal 的算法按升序处理边，并维护该前缀的最小生成森林。 当 DSU 将所有内容合并为一个组件时，我们就找到了连接所有顶点的边的最小前缀。 

如果连接早于当前边权重是可能的，则意味着仅使用较小边的生成树存在，这与 Kruskal 始终构造最小瓶颈生成树的事实相矛盾。 因此，我们第一次到达单个组件必定是最佳答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n
        self.components = n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.size[a] < self.size[b]:
            a, b = b, a
        self.parent[b] = a
        self.size[a] += self.size[b]
        self.components -= 1
        return True

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n, m = map(int, input().split())
        edges = []
        for _ in range(m):
            u, v, w = map(int, input().split())
            edges.append((w, u, v))

        edges.sort()

        dsu = DSU(n)

        ans = 0
        for w, u, v in edges:
            if dsu.union(u, v):
                ans = w
                if dsu.components == 1:
                    break

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```当我们按递增顺序处理边时，DSU 保持动态连接。 这`components`计数器至关重要，因为它避免了需要完整的图遍历来测试连接性。 每个成功的并集都会减少连接组件的数量，一旦达到一个，我们就知道该图是完全连接的。 

一个微妙的实现细节是我们将最后使用的边权重存储为`ans`。 这是有效的，因为边是按非递减顺序处理的，因此完成连接的最后一个成功的并集给出了正确的阈值。

 ## 边缘情况

 一种边缘情况是图已经几乎连接并且仅缺少一条边来完成连接。 例如：```
3 3
0 1 100
1 2 1
0 2 50
```DSU 首先在时间 1 连接 (1,2)，然后在时间 50 连接 (0,2)，并在那里停止。 答案是 50，因为在此之前节点 0 是孤立的。 

另一种情况是所有边具有相同的权重。 该算法仍然以该权重处理所有边，但一旦发生足够多的并集，就会实现连接。 为了：```
4 3
0 1 5
1 2 5
2 3 5
```每个并集发生在权重 5 处，最终合并在权重 5 处完成连接。该算法正确输出公共权重，因为排序将所有相等的边保留在一起，并且 DSU 合并直到保留单个组件。
