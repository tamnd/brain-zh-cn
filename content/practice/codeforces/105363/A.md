 ## 问题理解

 我们得到了几个短字符串，对于每个短字符串，我们必须决定是否可以将其重新排列以形成单词“hola”。 重新排列意味着我们可以自由地排列字符，但不能添加或删除任何字符。 因此，任务简化为检查输入单词中的字母多重集是否与“hola”中的字母多重集完全匹配。 

每个查询都是独立的，并且字符串非常小，因此每种情况的决策纯粹是该字符串的本地决策。 每个案例的输出是基于此比较的简单接受或拒绝。 

每个字符串的长度最多为 5 的约束显着改变了视角。 任何对每个字符执行恒定或接近恒定工作的算法都足够快。 即使是稍微多余的东西，比如对每个字符串进行排序，仍然可以忽略不计。 这使得解决方案远离复杂性问题，转向正确性和简单性。 

一些边缘情况自然会出现。 具有重复字母的字符串（例如“holaa”）必须被拒绝，因为它包含额外的字符，即使它包含所有必需的字符。 诸如“ola”之类的较短字符串也必须被拒绝，因为它缺少字母。 另一种情况是正确的排列，如“ohal”，其中字母完全匹配但被打乱。 

仅检查所有必需字符是否至少出现一次的简单方法将会失败。 例如，“holi”包含 h、o、l、i，因此如果不考虑精确的频率匹配，简单的存在检查可能会错误地接受它。 

## 方法

 思考这个问题的一种强力方法是生成输入字符串的所有排列，并检查它们是否等于“hola”。 由于字符串的长度最多为 5，因此最大排列数为 120，因此每个测试用例在技术上都是可行的。 然而，重复生成排列是不必要的工作，并且从概念上讲，它掩盖了问题的更简单的结构。 正确性来自匹配的字符计数，而不是明确地探索重新排列。 

关键的观察是，当且仅当两个字符串包含每个字符的频率相同时，这两个字符串才是彼此的排列。 我们可以直接比较字符数，而不是生成重新排列。 由于目标词是固定的并且非常小，因此我们可以对两个字符串进行排序或计算频率并进行比较。 

这里的排序特别干净。 如果我们对输入字符串进行排序并将其与“hola”的排序版本（即“ahlo”）进行比较，我们会立即得到相等测试。 或者，超过 26 个小写字母的频率数组也同样有效，但对于如此小的约束，排序就足够了，而且可以说更简单。 

蛮力方法花费时间探索不必要的结构，而最佳视图将问题分解为每个字符串的直接规范表示。

 ## 算法演练

 我们将使用基于排序的方法，因为它是最直接的。

1. 预先计算“hola”的排序参考字符串，它变为“ahlo”。 这为我们提供了一种规范形式来与每个输入进行比较。 
2.读取测试用例T的数量，因为每个用例都是独立的，必须单独处理。 
3. 对于每个输入字符串 s，计算其排序版本。 排序可确保相同字母的任何排列都会折叠成相同的表示形式。 
4. 将排序后的字符串与“ahlo”进行比较。 如果它们相同，则输入正是“hola”的排列，因此我们输出“SI”。 否则，我们输出“NO”。 
5. 对所有测试用例重复此过程，每行生成一个答案。 

### 为什么它有效

 排序为任何多组字符定义了确定性规范形式。 当且仅当两个字符串的排序表示相同时，它们才是彼此的排列。 这在排列下的字符串的等价类与其排序形式之间创建了一对一的映射，因此排序字符串的相等对于正确性来说既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    target = sorted("hola")
    target = "".join(target)

    t = int(input().strip())
    for _ in range(t):
        s = input().strip()
        if "".join(sorted(s)) == target:
            print("SI")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```该解决方案首先构造目标词“hola”的排序形式，即“ahlo”。 这避免了手动重新计算或硬编码字符比较。 

对于每个测试用例，输入字符串都使用 Python 的内置排序进行排序，考虑到最大长度为 5，这种排序是高效且足够的。比较是一个简单的字符串相等性检查，它直接编码两个字符串是否具有相同的字符多重集。 

使用`sys.stdin.readline`确保快速的输入处理，尽管由于小的限制，性能在这里并不重要。 逻辑本身完全以通过排序的规范表示为中心。

 ## 边缘情况

 比“hola”短的字符串，例如`h`，通过排序为单字符字符串来处理。 由于这无法匹配四字符引用，因此比较立即失败并生成“NO”。 

包含重复字母的字符串，例如`holaa`排序为`aa hlo`（概念上`aahlo`），这不同于`ahlo`，所以被拒绝。 这证实了无需显式计算字符即可正确检测到频率不匹配。 

完全有效的排列，例如`loah`排序为`ahlo`，与目标完全匹配。 这证实了排序步骤完全标准化了顺序差异。
