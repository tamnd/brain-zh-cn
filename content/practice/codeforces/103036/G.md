 ## 解决方案
 ## 问题理解

 我们给出了从 1 到 n 的整数排列，我们模拟从左到右的处理过程。 在每一步中，我们只查看当前位置之前已经出现的值。 

对于当前值 x，我们在之前看到的值中找到两个参考点。 1 是先前看到的仍小于 x 的最大值。 另一个是先前看到的仍然大于 x 的最小值。 如果一侧不存在，我们将区间扩展到值范围的自然边界。 

一旦确定了这两个边界，x 的贡献就是严格位于它们之间的整数的数量。 最终答案是整个排列中这些贡献的总和。 

解释这一点的一个有用方法是，我们在数轴上维护一组从 1 到 n 的动态可见值。 每个新值“跨越”该集合中其最近的现有邻居之间的间隙，并且我们计算该间隙内有多少未使用的整数。 

约束 n 高达 2 · 10^5 立即排除通过每次扫描可见集来重新计算邻居。 对于每个元素，在不断增长的数组中左右搜索的天真的方法会降低到二次时间并超出限制。 这推动我们采用一种支持对数时间内动态前驱和后继查询的数据结构。 

当当前元素是所见值中最小或最大时，会出现微妙的边缘情况。 例如，如果我们在排列中首先处理 1，则没有前驱，因此间隔实际上从 1 开始。类似地，如果我们提前处理 n，则没有后继，间隔会扩展到 n。 粗心的实现如果无法正确地对这些边界扩展进行建模，就会低估这些情况下的贡献。 

## 方法

 蛮力的想法很简单。 我们维护着一个不断增长的已见值列表。 对于每个新值 x，我们在值域中向左扫描以查找最近的较小可见值，并向右扫描以查找最近的较大可见值。 一旦我们确定了这两个值，我们就可以通过迭代范围或直接计算来计算严格位于它们之间的整数数量。 这是正确的，因为它完全遵循定义，但在最坏的情况下每次扫描可能需要 O(n) 时间，当 n 很大时，导致总共操作 O(n^2) 。 

瓶颈是在不断增长的无序集合中重复搜索前驱和后继。 关键的观察是，我们不需要集合的完整结构，只需要顺序统计：对于每个 x，我们需要低于它和高于它的最接近的可见值。 这建议将集合维护在支持快速前驱和后继查询的结构中。 

我们可以在值域 [1, n] 上使用 Fenwick 树对看到的值进行建模，其中每个位置指示一个值是否已经出现。 通过这种结构，我们可以有效地计算有多少个可见元素位于前缀中，并且我们可以对前缀计数进行二分搜索以找到第 k 个最小的可见元素。 这使我们能够在 O(log n) 时间内恢复 x 的前驱和后继。 一旦我们有了它们，贡献就变成了边界之间的简单算术差异。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力扫描| O(n^2) | O(n^2) | O(n) | 太慢了|
 | 具有阶次统计的 Fenwick 树 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

1. 维护一个索引为 1 到 n 的 Fenwick 树，其中每个索引指示该值是否已经出现。 这种结构使我们能够快速计算任何前缀中有多少个可见值。 
2. 从左到右迭代排列。 对于每个当前值 x，首先使用 Fenwick 树上的前缀和查询确定有多少个可见值严格小于 x。 该计数为我们提供了前任的排名位置（如果存在）。 
3. 如果低于 x 的可见值的计数大于零，我们通过查找第 k 个最小可见元素（其中 k 等于该计数）来定位前趋元素。 如果不存在这样的元素，我们将前驱元素视为 0，代表边界。 
4. 同样，计算有多少个可见值小于或等于 x。 如果该值严格小于所见元素的总数，我们将后继定位为第 (k+1) 个最小的所见元素。 如果不存在，我们将其视为n+1。 
5. 一旦确定了前驱 l 和后继 r，将 x 的贡献计算为 r - l - 1。这会直接计算有多少整数严格位于值空间中最近看到的邻居之间。 
6. 将 x 标记为在 Fenwick 树中看到的位置并继续。 

正确性取决于这样一个事实：在任何时刻，可见集合将价值线划分为间隙，并且每个新元素准确地贡献了它所桥接的间隙的大小。 

### 为什么它有效

 在每一步中，看到的值形成 [1, n] 的有序子集。 对于任何新值 x，其在该有序集合中的前任值和后继值恰好是约束 x 周围所有看不见的整数的最接近边界。 l 和 r 之间的任何整数当前都是不可见的，并且在处理连接该间隔的第一个元素时将只计算一次。 因此，该算法将整个价值线随着时间的推移分解为不相交的区间，并且每个贡献测量在该步骤中“激活”的区间的大小。 这确保了计数中没有重叠或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, i, v):
        while i <= self.n:
            self.bit[i] += v
            i += i & -i

    def sum(self, i):
        s = 0
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def kth(self, k):
        idx = 0
        bitmask = 1 << (self.n.bit_length())
        while bitmask:
            nxt = idx + bitmask
            if nxt <= self.n and self.bit[nxt] < k:
                k -= self.bit[nxt]
                idx = nxt
            bitmask >>= 1
        return idx + 1

n = int(input())
a = list(map(int, input().split()))

fw = Fenwick(n)
seen = 0
ans = 0

for x in a:
    left_count = fw.sum(x - 1)
    right_count = fw.sum(x)

    if left_count == 0:
        l = 0
    else:
        l = fw.kth(left_count)

    if right_count == seen:
        r = n + 1
    else:
        r = fw.kth(right_count + 1)

    ans += (r - l - 1)

    fw.add(x, 1)
    seen += 1

print(ans)
```芬威克树既用作动态存在数组，也用作顺序统计结构。 这`sum`函数提供前缀计数，它确定所见元素之间的排名。 这`kth`函数执行二分提升搜索以恢复与排名相对应的实际值，这对于有效查找前驱和后继至关重要。 

一个常见的实现陷阱是将前缀计数与实际值混淆。 树通过值索引存储存在，因此所有逻辑必须在值空间而不是排列中的位置空间中操作。

 ## 边缘情况

 当第一个元素是最小或最大可能值时，前驱或后继缺失，因此算法正确使用边界 0 和 n+1。 例如，在输入中`1 3 2`, 处理时`1`，没有前任，也没有后继，因此贡献变为 3 - 0 - 1 = 2，与此时 2 和 3 在右侧仍然无界的事实相匹配。 

当元素按排序顺序到达时，每一步仅缩小一侧的剩余间隔。 该算法仍然表现正确，因为前驱或后继总是反映最近的可见边界，确保每个间隙在首次被新插入“关闭”时被精确计数一次。
