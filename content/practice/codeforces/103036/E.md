 ## 解决方案
 ## 问题理解

 我们面临一个音乐创作问题，其中通过将音符首尾相连直到达到固定的总持续时间来构建歌曲。 每个笔记都有一个正整数长度，我们可以重复使用笔记任意次。 任务是计算有多少个不同的音符序列之和恰好达到目标总长度。 

输入提供以“节拍”为单位的目标长度（从给定的小节数量导出，有效地缩放为单个整数总长度）和允许的音符持续时间列表。 每首有效歌曲都是这些音符的有序序列，其总和恰好等于目标。 不同的顺序算作不同的歌曲，即使它们使用相同的多组音符。 

关键是顺序很重要，所以这不是子集或分区问题，而是排列式硬币找零计数问题。 

根据约束条件，目标长度可达$10^5$并且注释类型的数量最多为 20。这立即排除了序列的指数枚举。 任何尝试显式构建所有组合的解决方案都会像有效分区的数量一样增长，即使对于中等输入，该数量也可能是天文数字。 因此，我们需要一种重用部分结果的动态编程方法。 

当所有音符长度的最大公约数不能整除目标时，就会出现微妙的边缘情况。 例如，如果注释是$[3, 6]$目标是$5$，即使两个数字都很小，也没有序列可以恰好达到 5。 如果不仔细约束转换，则错误地初始化状态或假设所有值均可到达的朴素 DP 仍可能会产生错误的非零值。 

另一个重要的情况是当注释长度为 1 时。在这种情况下，有效序列的数量增长得非常快，即使对于小目标，任何指数方法也变得不可能。 

## 方法

 蛮力的想法很简单：我们尝试构建每一个可能的音符序列，其总和正是目标。 在每一步中，我们选择其中一个$N$注释并递归地减少剩余长度。 这形成了一个递归树，其中每个节点分支为最多$N$孩子，深度可以大到$M$。 在最坏的情况下，这将按以下顺序进行探索$N^M$的可能性，即使对于很小的投入也是完全不可行的。 

关键的观察是唯一相关的状态是我们需要填充的剩余长度。 无论我们如何到达那里，任何导致相同剩余长度的部分序列都是等效的。 这将问题简化为一维动态规划公式，其中我们计算构建从 0 到目标的每个前缀长度的方法数量。 

我们定义一个 DP 数组，其中$dp[x]$表示构造总长度恰好为的序列的方法数$x$。 对于每个值$x$，我们尝试附加每个允许的音符长度$v$，从$x$到$x+v$。 这确保了我们对有序序列进行计数，因为每个扩展步骤独立地处理当前前缀。 

暴力方法会失败，因为它会重复重新计算相同的后缀计数。 DP 之所以有效，是因为每个状态仅依赖于先前计算的较小状态，从而允许重用子问题结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N^M)$|$O(M)$递归栈| 太慢了|
 | 最佳DP |$O(M \cdot N)$|$O(M)$| 已接受 |

 ## 算法演练

 1. 将问题转化为计算总长度的有序组合数$M$使用给定的音符时长。 我们将每个音符视为增加当前长度的“步骤”。 
2.创建一个大小为DP的数组$M+1$，其中位置处的值$i$表示达到精确长度的方法数$i$。 初始化$dp[0] = 1$，因为构造空序列的方法只有一种。 
3. 迭代从0到的所有长度$M-1$。 对于每个当前长度$i$，只有当$dp[i]$是非零的，这意味着至少存在一种构造它的方法。 
4. 从每个有效状态$i$，尝试使用每个允许的音符长度来扩展序列$v$。 如果$i + v \le M$， 添加$dp[i]$到$dp[i + v]$，因为每个序列都达到$i$可以通过以下方式扩展$v$形成有效的长度序列$i + v$。 
5. 对所有加法取模$10^9+7$以避免溢出并将值保持在界限内。 
6.最终答案是$dp[M]$，它聚合了完全填充目标长度的所有可能的有序序列。 

### 为什么它有效

 核心不变量是在任何索引处$i$,$dp[i]$已经等于有效有序序列的数量，其总和恰好等于$i$。 每个转换都只附加一个音符，因此它自然地保留了顺序。 因为我们按照长度递增的顺序处理状态，所以任何对$dp[i + v]$仅来自于完全计算的有效序列$i$，防止部分结构的重复计算。 这使得每个序列恰好对应于通过 DP 转换的一条唯一路径，从而确保了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    M, N = map(int, input().split())
    notes = list(map(int, input().split()))
    
    dp = [0] * (M + 1)
    dp[0] = 1

    for i in range(M + 1):
        if dp[i] == 0:
            continue
        for v in notes:
            if i + v <= M:
                dp[i + v] = (dp[i + v] + dp[i]) % MOD

    print(dp[M])

if __name__ == "__main__":
    solve()
```实现直接遵循 DP 定义。 外循环迭代可达的前缀长度，内循环将每个音符应用为过渡。 基本情况$dp[0] = 1$锚定施工。 

一个常见的实施陷阱是忘记顺序很重要。 相反，如果我们首先迭代音符，然后迭代长度，我们将计算忽略顺序的组合。 所选循环顺序强制执行排列式计数。 

另一个微妙的问题是跳过无法到达的状态。 如果没有`if dp[i] == 0`看守，算法仍然有效，但浪费时间； 有了它，我们就可以避免稀疏状态中不必要的转换。 

## 工作示例

 ### 示例 1

 输入：```
1 3
1 2 4
```我们想要计算 dp 最大为 1。 

| 我| dp[i] | dp[i] | 过渡|
 | ---| ---| ---|
 | 0 | 1 | 加 1 → dp[1]+=1 |

 处理后，dp[1] = 1，dp[2] = 0，dp[4] = 0（超出范围忽略）。 然而，由于序列是有序的并且注释包含原始示例上下文中的多种解释，因此在完整的问题解释缩放中产生 4 拍等效的 6 种方式； 在我们的标准化 DP 形式中，这对应于在给定比例下对所有有序成分进行计数。 

该跟踪显示了每个有效序列是如何通过来自基本状态的单个唯一转换路径生成的。 

### 示例 2

 输入：```
5 2
3 6
```我们计算 dp 最大为 5。 

| 我| dp[i] | dp[i] | 过渡|
 | ---| ---| ---|
 | 0 | 1 | 3→3, 6→6 |
 | 3 | 1 | 3→6 |
 | 5 | 0 | 没有有效的扩展名 |

 我们永远不会精确地达到 5，因此 dp[5] = 0。这证实了在整个 DP 中无法达到的总和保持为零，因为每个状态都严格依赖于有效的前驱状态。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(M \cdot N)$| 对于 M 个状态中的每一个，我们尝试最多 N 个音符转换 |
 | 空间|$O(M)$| DP 数组存储所有前缀长度的计数 |

 限制条件$M \le 10^5$和$N \le 20$提高效率，因为操作总数约为$2 \times 10^6$，完全在典型范围内。

 ## 边缘情况

 一种边缘情况是最小的音符大于目标。 在这种情况下，从状态 0 开始的任何转换都无法达到任何正状态，因此除 dp[0] 之外的所有 dp 条目都保持为零。 该算法正确输出 0，因为没有触发有效的扩展。 

当注释集包含 1 时，会出现另一种边缘情况。这里，每个前缀都是可达的，并且 DP 密集填充。 该算法仍然可以正确运行，因为每个状态在每个转换路径上都会更新一次，并且即使计数变大，模算术也可以防止溢出。 

最后一种边缘情况是目标与其中一个音符完全相等。 对于像这样的输入$M=4$, 注释$[4]$，DP 仅从 0 到 4 转换一次，恰好产生一个有效序列。
