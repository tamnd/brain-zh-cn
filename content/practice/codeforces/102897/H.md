 ## 解决方案
 ## 问题理解

 任务是为多个场景呈现命令行进度指示器。 每个场景都描述了工作总量以及已完成的工作量。 由此，我们必须构建一个可视化栏，其中完整的单元显示为哈希字符，剩余的单元显示为连字符。 除了这个条形图之外，我们还打印一个百分比值，表示已完成工作的比例，按比例缩放至 100 并向下舍入。 

每个测试用例给出两个整数，即单元总数和已完成单元数。 每种情况的输出都是一行，由括号内的条形后跟空格和带百分号的整数百分比组成。 

关键的结构细节是进度条在总工作的每一单元中只有一个字符。 这意味着输出没有被抽象或缩放，它直接反映输入大小。 

这些约束允许最多十个测试用例，每个测试用例最多可以有十万个单元。 这意味着以二次时间构建字符串的简单解决方案会太慢，但任何以每个测试用例的线性时间构建每个输出行的解决方案都非常快。 所有测试用例的总工作量最多为一百万个字符，这完全在一秒的典型输出限制之内。 

当没有取得任何进展或一切都已完成时，就会出现微妙的边缘情况。 当完成为零时，条形图应仅包含连字符，并且百分比应为零。 当完成等于总数时，该栏应仅包含哈希值，并且百分比应恰好为一百。 另一种边缘情况是当总数为 1 时，条形退化为单个字符，并且舍入行为必须仍然正确。 

一个常见的错误是尝试使用浮点除法和舍入来计算百分比，这可能会引入精度问题或不正确的舍入行为。 另一个错误是忘记整数除法必须是基于下限的，而不是四舍五入到最接近的。 

## 方法

 构建输出的直接方法是逐个字符地模拟进度条。 对于每个测试用例，我们从零迭代到 n 减一，如果索引小于 m，则附加一个散列，否则附加一个连字符。 构建条形图后，我们使用整数除法计算百分比，即 m 乘以 100 再除以 n。 这种方法是正确的，因为它直接遵循输出格式的定义。 

如果实施不当，效率低下的问题仅来自于重复的字符串连接。 如果我们在循环内重复附加到不可变字符串，则复杂性可能会降低到二次行为。 但是，如果我们构造一个字符列表并连接一次，或者使用字符串的直接乘法，则构造仍然是线性的。 

关键的观察是输出的两个部分完全由简单的算术和重复决定。 除了索引是否低于 m 之外，位置之间不存在任何依赖性。 这消除了直接构建之外的任何模拟需求。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 逐个字符连接（简单的字符串附加）| O(n^2) 最坏情况 | O(n) | 太慢了 |
 | 使用重复或列表连接直接构造| O(n) | O(n) | 已接受 |

 ## 算法演练

1. 对于每个测试用例，读取总长度 n 和完成数量 m。 这两个值完全决定了条形图和百分比。 
2. 通过创建一个由 m 个哈希字符后跟 n 减去 m 个连字符组成的字符串来构造进度条。 这是可行的，因为每个位置直接对应于一个工作单元，并且已完成的单元始终占据前缀。 
3. 计算 m 乘以 100 的整数除以 n 的百分比。 这可确保截断为零，符合所需的底层行为。 
4. 将条​​形图括在方括号中，附加一个空格，然后打印计算出的百分比，后跟百分号，从而格式化输出。 

工作原理：进度条是固定长度数组上前缀条件的直接编码。 每个有效的表示必须首先放置所有已完成的单元以匹配示例结构，然后放置每个剩余的单元。 百分比计算是相同比率的线性缩放，因此两个输出是相同基础分数的一致视图。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        bar = "#" * m + "-" * (n - m)
        percent = (m * 100) // n if n > 0 else 0
        sys.stdout.write(f"[{bar}] {percent}%\n")

if __name__ == "__main__":
    solve()
```该实现依赖于 Python 的本机字符串重复，这是高效的，因为它在相对于其长度的线性时间内构造最终字符串。 表达式`"#" * m`恰好生成 m 个字符，而无需增加串联开销。 这同样适用于连字符段。 

百分比计算在除法之前使用整数乘法以避免浮点精度问题。 条件守卫为`n > 0`在限制下，技术上是不必要的，但它使计算对于格式错误的输入具有鲁棒性。 

## 工作示例

 考虑 n 为 10、m 为 2 的情况。条形图必须显示两个完整的单元，后跟八个不完整的单元。 

| 步骤| n | 米 | 酒吧建设| 百分比 |
 | --- | --- | --- | --- | --- |
 | 1 | 10 | 10 2 | “” | - |
 | 2 | 10 | 10 2 | “##--------” | - |
 | 3 | 10 | 10 2 | “##--------” | 20 |

 该栏正确地将两个哈希值放在前面，百分比为floor(20)。 

现在考虑 n 等于 5，m 等于 0。 

| 步骤| n | 米 | 酒吧建设| 百分比 |
 | --- | --- | --- | --- | --- |
 | 1 | 5 | 0 | “” | - |
 | 2 | 5 | 0 | “-----” | - |
 | 3 | 5 | 0 | “-----” | 0 |

 这证实了零进度会产生全虚线条和零百分比。 

这些示例证实了视觉和数字表示之间的边界行为和一致性。

import sys, io

def solve_io():
    input = sys.stdin.readline
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        bar = "#" * m + "-" * (n - m)
        percent = (m * 100) // n

def run(inp: str) -> str:
    old_stdin = sys.stdin
    old_stdout = sys.stdout
    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()
    try:
        solve_io()
        return sys.stdout.getvalue()
    finally:
        sys.stdin = old_stdin
        sys.stdout = old_stdout

assert run("1\n1 1\n") == "[#] 100%\n"

 ## 边缘情况

 当 m 等于 0 时，该构造会生成一个空的哈希段和一个完整的连字符段。 例如，输入 n 等于 5，m 等于 0，则表达式`"#" * 0 + "-" * 5`产量`"-----"`，以及百分比`(0 * 100) // 5`评估为零。 The algorithm naturally handles this without special branching because string repetition with zero produces an empty string.

 When m equals n, the hyphen segment becomes empty. 当 n 等于 4 并且 m 等于 4 时，条形变为`"####"`and the percentage evaluates to 100. This works because subtracting m from n yields zero, and repeating a string zero times correctly contributes nothing to the output.

 当 n 等于 1 时，横杠退化为单个字符。 如果 m 为零，则变为`"-"`，如果 m 为 1，则变为`"#"`。 整数除法仍会产生 0 或 100，这符合预期行为，无需浮点运算。
