 ## 问题理解

 我们得到了几个独立的建筑描述。 每个建筑物由多个楼层组成，每个楼层都由一个字符串表示。 字符串中的字符描述了某个位置是否包含垃圾以及垃圾的类型。 点字符代表空位，其他字符代表垃圾项，不同字符对应不同的垃圾类型。 

对于每个楼层，我们需要计算该楼层出现了多少种不同的垃圾类型。 然后我们将建筑物中所有楼层的该值相加。 最后，我们输出每个测试用例的总数。 

关键点是，清晰度是按楼层独立测量的。 如果相同的垃圾类型出现在多个楼层，则它会在出现的每个楼层分别产生影响。 如果一个楼层包含多次出现相同的字符，它仍然只对该楼层贡献一次。 

这些约束意味着线性扫描解决方案。 测试文件中所有楼层的字符总数最多可达 10^7。 这立即排除了任何按字符昂贵的结构，例如对子字符串进行排序或重复的集重新初始化，每个查询的开销很大。 我们需要一种方法，基本上只处理每个字符一次，并且不断更新。 

当人们错误地将整个建筑物而不是每层楼的垃圾类型聚集在一起时，就会出现微妙的失败案例。 例如，考虑：

 一楼：`a.a`2楼：`aa.`正确解读：

 1楼有`{a}`所以贡献1。 

2楼还有`{a}`所以贡献1。 

答案是2。 

错误的全局集方法会将其视为一种类型总计和输出 1。 

另一个微妙的情况是同一楼层的重复字符。 例如：`####`这应该贡献 1，而不是 4。 

## 方法

 一个简单的解释是独立处理每个楼层，构建该字符串中所有非点字符的集合，并将其大小添加到答案中。 这是正确的，因为集合自然地强制每层垃圾类型的唯一性。 

然而，天真的担忧是性能。 如果我们从字面上创建一个新集合并插入每个字符，我们仍然会接触每个字符一次，因此复杂性与总输入大小呈线性关系。 唯一潜在的开销是哈希集操作，但在 Python 中，在总字符数为 10^7 的约束下，这仍然是可以接受的。 

我们可以通过尽可能避免重复分配或繁重的对象创建来完善实现。 我们可以为每个字符类型维护一个布尔标记数组或最后看到的时间戳数组，而不是显式存储每个楼层的完整集合。 对于每一层，我们仅重置我们使用的标记，这避免了清除大型结构。 

关键的观察是，我们不需要知道哪个楼层使用了某个角色，只需要知道它是否出现在当前楼层。 因此我们可以在 O(1) 标记操作中跟踪每层看到的字符。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每层套 | O(总字符数) | O（字母大小）| 已接受 |
 | 时间戳/布尔数组| O(总字符数) | O（字母大小）| 已接受 |

 ## 算法演练

 1.读取测试用例的数量。 每个测试用例都是独立的，因此我们重置每个测试用例的累积答案。 
2. 对于每个测试用例，初始化一个全局结构`seen`记录当前楼层是否已计入某个角色。 根据约束，这可以是字典或数组。 
3. 迭代每个楼层字符串。 在处理新楼层之前，仅清除或重置与该楼层相关的状态。 重要的要求是前一层的信息不能泄漏到当前一层。 
4. 逐个字符扫描字符串。 如果字符是一个点，请立即跳过它，因为它不代表垃圾。 
5. 对于非点字符，检查该字符是否已被计入本层。 如果没有，请将其标记为已看到，并将楼层贡献增加 1。这可确保每种垃圾类型在每个楼层最多贡献一次。 
6. 完成发言后，将其贡献添加到总答案中。 
7. 移动到下一层并重复。 

### 为什么它有效

 每个楼层都被视为一个独立的宇宙，我们在其中计算非点字符中的不同符号。 标记结构确保每个符号每层计数一次，并且楼层之间的重置保证独立性。 因为每个字符只被处理一次并且最多贡献一个增量，所以最终的总和完全符合“每层不同垃圾类型的总和”的定义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        total = 0
        
        seen = set()
        
        for _ in range(n):
            s = input().strip()
            seen.clear()
            cnt = 0
            
            for ch in s:
                if ch == '.':
                    continue
                if ch not in seen:
                    seen.add(ch)
                    cnt += 1
            
            total += cnt
        
        print(total)

if __name__ == "__main__":
    solve()
```该解决方案独立处理每个测试用例并逐层累积答案。 这`seen`每个楼层都会清理垃圾，确保垃圾类型不会跨楼层污染。 每个字符都会检查一次，插入到 Python 集中可以保证在摊销常数时间内进行唯一性跟踪。 

一个常见的陷阱是忘记清除`seen`楼层之间，这会错误地合并不同楼层的垃圾类型并导致计数膨胀。

 ## 边缘情况

 例如，一种边缘情况是只有点的地板`"....."`。 算法清零`seen`，扫描每个字符，没有找到有效的垃圾，并贡献 0。集合保持为空，并且贡献正确地为零。 

另一种边缘情况是由相同字符组成的极长单层输入，例如`"aaaaaaaa..."`。 第一次出现插入到`seen`，并且所有后续字符都将被忽略，无论长度如何，都会产生 1 的贡献。 

第三种情况是具有重叠符号的交替楼层，例如：```
a.b
b.a
```一楼，`seen`变成`{a, b}`并贡献2。在二楼，`seen`被清除并再次重新计算为`{a, b}`，再次贡献2。 重置可确保不会遗留问题，从而保持各楼层的正确性。
