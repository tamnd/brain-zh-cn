 ## 解决方案
 ## 问题理解

 我们给出了起始年份和整数偏移。 移位应用于年份，但结果不直接使用。 相反，结果年份中超出 9999 上限的任何部分都会以问题描述的特定方式删除，从而产生最终的调整年份间隔。 最后的任务是计算该间隔内有多少个闰年（含）。 

闰年遵循标准的公历规则。 如果一年能被 400 整除，或者能被 4 整除但不能被 100 整除，那么年份就是闰年。 

每个测试用例都是独立的。 对于每一个，我们必须在应用“上限为 9999 并溢出折叠”规则后重建最终间隔，然后计算该间隔中的闰年。 

这些限制本质上是很小的：最多大约一百个测试用例，并且涉及的所有年份在最坏的情况下保持在几万个以内，然后再减少回 1 到 9999 的范围。 这立即排除了诸如在不进行预处理的情况下为每个查询在大范围内逐年重新计算跳跃状态之类的情况。 每个测试用例每年一次的简单扫描仍然没问题，因为域最多只有 9999，但任何渐近比 O(100 × 10000) 更差的东西仍然会轻松通过，而像在没有结构的动态范围上重复重新计算之类的东西将是不必要的开销。 

一个微妙的点就是溢岁月的“折叠”。 当移位年份超过 9999 时，超出 9999 的部分不会被简单地忽略； 相反，它是从 9999 中减去它以形成最终端点。 这会在 9999 附近创建类似对称反射的效果，很容易被误认为是简单的剪切。 

一个常见的错误是将最终间隔始终视为 [Y, Y + A] 剪辑为 [1, 9999]。 当加法超过 9999 时，就会失败，因为溢出会导致端点的减少。 

另一种边缘情况是负 A。间隔方向可以翻转，因此必须仔细规范化端点。 例如，Y = 10 和 A = -20 导致幼稚终点为 -10，这是无效的，但该问题保证调整后最终构建的年份为正。 尽管如此，我们必须确保我们始终在排序间隔内进行计数。 

## 方法

 暴力解决方案首先会严格按照所描述的方式模拟转换，以获得最终的区间端点 L 和 R。一旦获得区间，我们只需迭代从 L 到 R 的每一年，并使用整除规则检查每年是否是闰年。 

这是可行的，因为跳跃检查是 O(1)，并且最大间隔长度以 9999 为界。在最坏的情况下，每个查询扫描近 10000 年，在所有测试用例中进行大约 10^6 次检查，这是微不足道的。 

然而，这仍然每年独立执行重复的模块化检查。 问题的结构提出了一种更好的方法：闰年遵循周期为 400 的周期性模式。我们可以预先计算最多 9999 年的闰年前缀计数，并在区间重建后以 O(1) 的时间复杂度回答每个查询，而不是单独检查每一年。 

关键思想是我们可以将计数问题转换为静态二进制数组上的范围求和查询，其中每个位置指示一年是否是闰年。 一旦我们构建了一次前缀和数组，每个查询就会减少为两个前缀值的减法。 

这就把问题变成了简单的区间重构加上常数时间查询。

 | 蛮力 | O(T × 10000) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们首先预处理从 1 到 9999 的所有年份，并标记哪些是闰年。 由此我们构建一个前缀和数组。 

1. 计算数组`is_leap[i]`使用闰年规则从 1 到 9999 的所有年份。 
2. 构建`pref[i]`在哪里`pref[i] = pref[i-1] + is_leap[i]`。 这存储了到第 i 年为止存在多少个闰年。 
3. 对于每个测试用例，读取 Y 和 A 并计算原始端点`X = Y + A`。 
4. 如果 X ≤ 9999，则根据需要对端点进行排序后，区间只是 [Y, X]。 
5. 如果 X > 9999，则计算溢出`excess = X - 9999`并将端点设置为`R = 9999 - excess`。 
6、最终的间隔是在两个端点之间，所以设置`L = min(Y, R)`和`R = max(Y, R)`。 
7. 输出`pref[R] - pref[L-1]`。 

重要的推理步骤是，即使发生溢出，变换也始终会在 [1, 9999] 内产生有效端点，因此在归一化后区间始终是明确定义的。 

### 为什么它有效

 前缀和数组对固定有序域上的加性测量进行编码。 由于闰年条件与查询无关，仅取决于年份值，因此我们可以安全地预先计算一次。 每个查询都简化为要求一个函数在一段时间内的总和，并且前缀差异精确地表示该总和。 问题的唯一动态部分是确定间隔端点，一旦这些端点正确，计数就变成纯粹的算术。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXY = 10000

def is_leap(y: int) -> int:
    if y % 400 == 0:
        return 1
    if y % 100 == 0:
        return 0
    if y % 4 == 0:
        return 1
    return 0

# precompute prefix of leap years
pref = [0] * (MAXY)
for i in range(1, 10000):
    pref[i] = pref[i - 1] + is_leap(i)

def solve():
    T = int(input())
    out = []
    for _ in range(T):
        Y, A = map(int, input().split())
        X = Y + A

        if X <= 9999:
            L, R = Y, X
        else:
            excess = X - 9999
            R = 9999 - excess
            L = Y

        if L > R:
            L, R = R, L

        if L < 1:
            L = 1
        if R > 9999:
            R = 9999

        out.append(str(pref[R] - pref[L - 1] if L > 1 else pref[R]))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码首先构建闰年的直接指示器，然后将该信息压缩到前缀和数组中。 每个查询都根据自定义溢出规则简化为计算端点，然后对前缀数组进行恒定时间减法。 

一个微妙的部分是处理顺序：间隔端点可以交换，具体取决于调整后的端点是大于还是小于起始年份。 排序可确保正确性，而不需要单独的案例。 

另一个微妙之处是前缀数组中的索引。 由于我们使用基于 1 的年份，因此 pref[0] 充当中立基，因此从第 1 年开始的查询必须避免访问 pref[-1]。

 ## 边缘情况

 一种边缘情况是变换后区间因溢出而崩溃。 例如，Y = 9999，A = 1，生成 X = 10000，多余 = 1，因此 R = 9998。区间变为 [9999, 9998]，排序后变为 [9998, 9999]。 该算法通过在 L > R 时显式交换端点来处理此问题，从而在查询前缀数组之前确保有效性。 

另一种情况是归一化后下界变为 1。 由于前缀和依赖于访问 pref[L-1]，因此实现通过单独处理 L = 1 并避免负索引来保护这一点。
