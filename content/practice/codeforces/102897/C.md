 ## 解决方案
 ## 问题理解

 我们间接得到一个有根树结构，不是通过边，而是通过每个节点的深度列表。 节点 1 固定为根，每个节点 i 都带有一个整数 ai ，该整数以路径上的节点数（包括其自身）来描述它距根的距离。 因此根的深度为 1，其子节点的深度为 2，依此类推。 

但并没有给出真实的亲子关系。 许多不同的有根树可以匹配相同的深度分配，因为深度 d 的节点可以通过多种方式附加到深度 d − 1 的节点。任务是计算有多少个不同的有根树与给定的深度数组一致。 

如果至少一个节点具有不同数量的子节点，则认为两棵树是不同的。 因此，我们不仅仅通过结构来区分，而是通过每个节点的精确子计数来区分，这仍然减少了对不同的有效父分配的计数。 

约束 n ≤ 100000 迫使任何解决方案趋于线性或接近线性时间。 任何涉及显式检查所有父级选择或枚举树结构的操作都会发生组合爆炸。 即使 O(n log n) 也可以，但在节点上进行 O(n^2) 构造或在所有分配上进行朴素 DP 是不可能的。 

当深度约束不一致时，就会出现关键的边缘情况。 例如，如果我们有一个深度为 3 的节点，但不存在深度为 2 的节点，则无法形成树，因此答案必须为 0。另一种微妙的情况是，当多个节点共享深度 2，但深度 1 处只有一个可能的父节点时，就会形成星形结构。 幼稚的方法可能会错误地增加选择，而不考虑每个节点必须在前一深度级别中准确选择一个父节点。 

无效案例示例：

 输入：

 n = 3

 a = [1, 3, 2]

 如果节点索引严格排序，则深度为 3 的节点在深度 2 处没有可能的父节点。 正确答案是0。 

## 方法

 强力解释是通过为每个节点（根除外）分配前一个深度级别的节点中的父节点来构建树。 如果深度 d 处有 kd 个节点，深度 d − 1 处有 kd−1 个节点，则深度 d 处的每个节点都可以选择 kd−1 个父节点中的任何一个，从而导致该级别有 (kd−1)^{kd} 种可能性。 跨级别相乘即可得出候选人数。 

这已经暗示了解决方案的结构，但是蛮力会显式构造或模拟所有分配，或者尝试递归地构建树。 这很快就变得不可行，因为在最坏的情况下，比如两层结构，分配的数量随着 n 呈指数增长。 

关键的观察是深度完全决定了有效的父候选者：深度 d 的每个节点都必须连接到深度 d − 1 的某个节点，其他选择都不重要。 由于只有每个节点的子节点数量对于区分树很重要，因此我们不需要跟踪确切的分配，只需要跟踪级别 d 的节点可以在级别 d − 1 的节点之间分配自己的方式。每个节点独立选择其父节点，因此配置的总数是权力级别的乘积。 

还有一个隐藏的约束：深度 d − 1 的每个节点必须至少有一个更深层次的节点，如果它不是孤立的，可以附加到它上面，但这是自动处理的，因为节点可以自由选择父节点； 我们不需要确保满射性。 

因此，问题简化为按深度对节点进行分组并逐级乘以贡献。

 | 深度计数 | O(n) | O(n) | 已接受 |

 ## 算法演练

 ## 算法演练

1. 按深度值对节点进行分组，并计算每个深度存在多少个节点。 这为我们提供了每个深度 d 的频率图 kd 。 此步骤将问题转换为使用层而不是单个节点。 
2. 检查深度 1 是否恰好包含一个节点。 如果不是，则该结构不能表示以节点 1 作为根的有根树，因此答案为零。 这强制执行固定根约束。 
3. 验证深度是否从 1 开始连续且无间隙。 如果出现某个深度 d > 1 但缺少深度 d − 1，则没有节点可以附加到有效的父节点，因此答案为零。 
4. 按升序对所有不同的深度进行排序，以便我们以有效的父到子方向处理级别。 
5. 将答案初始化为 1。我们将乘以每个深度转换的贡献。 
6. 对于从 2 开始的每个深度 d，将答案乘以 (k_{d−1})^{k_d}。 这反映了深度 d 处的每个节点在深度 d − 1 处的所有节点中独立选择一个父节点，从而为每个节点提供 k_{d−1} 个选择。 
7. 对所有乘法以 998244353 为模。这样可以将数字保持在界限内，同时保持模算术的正确性。 

### 为什么它有效

 在每个深度层，唯一的结构自由在于为前一层的每个节点选择父节点。 一旦节点选择了其父节点，它就会向该父节点贡献一条出边，并且除了通过计数之外，这种选择不会限制其他节点。 由于该问题仅通过子项计数来区分树，因此父项的任何不同分配都会产生不同的有效树。 跨节点的选择是独立的，跨层的选择也是独立的，因此总计数干净地分解为深度转换的幂的乘积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def mod_pow(a, e):
    return pow(a, e, MOD)

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    from collections import Counter
    cnt = Counter(a)

    # root must be unique
    if cnt[1] != 1:
        print(0)
        return

    depths = sorted(cnt.keys())

    # check continuity of depths
    for i in range(1, len(depths)):
        if depths[i] != depths[i - 1] + 1:
            print(0)
            return

    ans = 1

    for i in range(1, len(depths)):
        prev_d = depths[i - 1]
        cur_d = depths[i]
        ans = (ans * pow(cnt[prev_d], cnt[cur_d], MOD)) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先计算每个深度出现的节点数量。 这将问题压缩为多个级别的频率图。 根验证确保只有一个节点位于深度 1，因为根是固定的。 

然后我们对深度进行排序并确保它们形成连续的链。 这可以防止级别没有有效父级别的无效情况。 

最后，我们迭代深度转换。 对于每对连续深度，更深层次的每个节点都会在前一层的所有节点中选择其父节点，贡献一个幂项。 乘法累加所有独立选择。 

模幂运算是使用 Python 的内置 pow 进行处理的，对于 n 到 100000 的范围来说，它的效率足够高，因为指数值受 n 限制。 

## 工作示例

 考虑一个小的有效案例：

 输入：

 n = 4

 a = [1, 2, 2, 3]

 深度计数：

 | 步骤| 深度| 计数 |
 | --- | --- | --- |
 | 1 | 1 | 1 |
 | 2 | 2 | 2 |
 | 3 | 3 | 1 |

 从深度 1 到 2 的过渡贡献 1^2 = 1，因为深度 2 的两个节点必须附加到单个根。 从深度 2 到 3 的过渡贡献 2^1 = 2，因为深度 3 的节点可以选择两个深度 2 节点中的任何一个作为父节点。 最终答案是2。 

现在考虑一个更自由的情况：

 输入：

 n = 5

 a = [1, 2, 2, 2, 3]

 深度计数：

 | 步骤| 深度| 计数 |
 | --- | --- | --- |
 | 1 | 1 | 1 |
 | 2 | 3 | 3 |
 | 3 | 1 | 1 |

 这里：

 转换 1 → 2 贡献 1^3 = 1。 

转换 2 → 3 贡献 3^1 = 3。 

所以答案是3。 

这些痕迹显示了每一层如何独立贡献，以及求幂如何编码独立的父选择。

 ## 边缘情况

 一种边缘情况是根不唯一。 用于输入`n = 3, a = [1, 1, 2]`，算法立即返回 0，因为根级别有多个节点。 在处理过程中，cnt[1] = 2 违反了根约束，因此不会进行进一步的计算。 

另一个边缘情况是断开的深度序列，例如`a = [1, 2, 4]`。 深度图包含 1、2 和 4，但缺少 3。 当从 2 移动到 4 时，排序深度列表未通过连续性检查，并且算法在尝试求幂之前正确输出 0。 

第三种情况是完全有效的链，例如`a = [1, 2, 3, 4]`。 这里每个深度只有一个节点，因此每次转换贡献 1^1 = 1，并且答案始终保持为 1，这与存在与深度一致的线性树结构这一事实相匹配。
