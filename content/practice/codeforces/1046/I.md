 ## 解决方案
 ## 问题理解

 我们给出了随时间在平面上移动的两个点，但它们的运动仅在离散时间戳处指定。 在连续的时间戳之间，每个朋友都以匀速直线移动，因此任意中间时间的位置都是通过线性插值获得的。 

通过这个连续的动作，我们可以追踪两个朋友之间的距离。 当距离变小时，特别是达到或低于阈值时，他们会“打招呼”$d_1$。 不过，问候语也不是可以随意重复的。 问候发生后，只有当朋友们在某个时刻距离足够远（这意味着他们的距离超过了更大的阈值）时，才允许新的问候$d_2$，然后他们再次接近。 

因此，任务是模拟两个移动点之间的平方距离的连续演化，并计算有多少个不同的“近距离接近事件”满足规则：输入$d_1$-距离区域，但仅当自上一次问候以来，轨迹至少有一个时刻超出距离时，才将其计为新问候$d_2$。 

输入大小达到$N = 100{,}000$，这排除了任何尝试对时间进行精细采样或检查每个段的许多中间点的距离的解决方案。 由于运动是分段线性的，因此时间戳之间的每个间隔内的距离函数变成时间的二次函数。 这意味着所有相关事件都发生在二次方程的根处，因此$O(N)$或者$O(N \log N)$需要基于事件的扫描。 

一种简单的方法会评估每个段的许多时间步长的距离或模拟非常小的增量。 这很快就变得不可能了，因为每个部分都需要多次评估来检测交叉点$d_1$和$d_2$，导致最坏情况的复杂性远远超出可接受的限度。 

当朋友们在阈值附近振荡时，会出现更微妙的失败案例。 例如，他们可以输入$d_1$区域多次但不超过$d_2$之间。 在这种情况下，计算每次进入$d_1$会过多地计算问候，因为“必须超过$d_2$自上次问候以来”条件充当重置门。

 另一个棘手的情况是当距离恰好接触时$d_1$或者$d_2$在段边界处。 由于运动是连续的，但采样是离散的，因此仅检查端点的解决方案会错过二次在间隔内低于或高于的情况。 

## 方法

 强力解释会将时间细分为每个间隔内非常小的步骤，并检查每个步骤的距离。 从概念上讲，这是可行的，因为距离是连续的，因此足够小的步长可以检测到所有阈值交叉。 然而，每个$N$分段可能需要许多采样点才能正确检测低于或高于阈值的抛物线。 在最坏的情况下，这会退化为密集模拟，时间复杂度与样本数量成正比，很容易超过$10^8$或更多操作。 

关键的观察是，在时间戳之间的任何固定间隔内，两个点都线性移动，因此它们的相对位置也是线性的。 距离的平方成为时间的二次函数。 二次方程最多可以跨越阈值两次，这意味着每个段的所有有趣事件都可以通过求解每个阈值最多两个方程来完全确定。 

这将问题转化为跟踪区间，其中二次函数如下$d_1^2$以及它超过的地方$d_2^2$。 然后我们扫描时间，维护我们当前是否“有资格”计算问候语，这取决于函数是否超出了$d_2$自上次计数事件以来。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 强力采样|$O(NK)$（每段大 K）|$O(1)$| 太慢了|
 | 二次事件处理 |$O(N)$|$O(1)$| 已接受 |

 ## 算法演练

 我们独立处理连续时间戳之间的每个间隔，将每个段内的本地时间视为$t \in [0,1]$。 

1. 对于每个段$i$, 表示相对位置$P(t) = A(t) - B(t)$作为线性函数$t$。 这是有效的，因为两个端点都线性移动。 由此，计算平方距离$f(t) = |P(t)|^2$，这是一个二次多项式$t$。 
2. 对于每个阈值$T \in \{d_1^2, d_2^2\}$，求解不等式$f(t) \le T$或者$f(t) > T$。 这简化为寻找二次方程的根并确定哪些子区间满足不等式。 
3. 从$d_1$不平等，提取朋友足够亲密可以打招呼的区间。 
4. 从$d_2$不等式，检测距离超过重置阈值的间隔。 最后问候语之后的任何此类间隔都会将系统标记为“符合重置条件”。 
5. 扫过时间从$t=0$到$t=N-1$，维护一个状态变量，指示自上次计数问候以来我们是否看到了“远”情节。 
6. 每当我们进入$d_1$- 有效间隔和重置标志处于活动状态，增加答案并清除标志。 

关键思想是问候语与进入的过渡完全一致$d_1$-系统经历了至少一次到外部的旅行后的区域$d_2$-地区。 

### 为什么它有效

 正确性取决于距离函数的结构。 自从$f(t)$在每个线段上都是二次的，所有阈值交叉都由其根捕获。 在根之间，该函数对于不等式是单调的，因此“近”或“远”区域中的成员资格除了在计算的事件点外不能改变。 状态机确保每次有效的重新进入$d_1$区域只计算一次，并且$d_2$条件强制连续计数之间的分隔。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    d1, d2 = map(int, input().split())
    d1 = d1 * d1
    d2 = d2 * d2

    A = []
    B = []
    for _ in range(n):
        ax, ay, bx, by = map(int, input().split())
        A.append((ax, ay))
        B.append((bx, by))

    def get_quad(p0, p1):
        x0, y0 = p0
        x1, y1 = p1
        dx = x1 - x0
        dy = y1 - y0
        # position: p(t) = p0 + t*(p1-p0), t in [0,1]
        # squared norm becomes quadratic coefficients
        return (x0*x0 + y0*y0,
                2*(x0*dx + y0*dy),
                dx*dx + dy*dy)

    def interval_leq(a, b, c, T):
        # (a + b t + c t^2) <= T
        a -= T
        if c == 0:
            if b == 0:
                return [(0.0, 1.0)] if a <= 0 else []
            t = -a / b
            if b > 0:
                seg = (0.0, min(1.0, t))
            else:
                seg = (max(0.0, t), 1.0)
            if seg[0] <= seg[1]:
                return [seg]
            return []
        disc = b*b - 4*c*a
        if disc < 0:
            return [(0.0, 1.0)] if a <= 0 else []
        import math
        sd = math.sqrt(max(0.0, disc))
        t1 = (-b - sd) / (2*c)
        t2 = (-b + sd) / (2*c)
        if t1 > t2:
            t1, t2 = t2, t1
        res = []
        if c > 0:
            l = max(0.0, t1)
            r = min(1.0, t2)
            if l <= r:
                res.append((l, r))
        else:
            if t1 > 0:
                res.append((0.0, min(1.0, t1)))
            if t2 < 1:
                res.append((max(0.0, t2), 1.0))
        return res

    ans = 0
    had_far = False

    for i in range(n - 1):
        ax0, ay0 = A[i]
        ax1, ay1 = A[i+1]
        bx0, by0 = B[i]
        bx1, by1 = B[i+1]

        dx0 = ax0 - bx0
        dy0 = ay0 - by0
        dx = (ax1 - ax0) - (bx1 - bx0)
        dy = (ay1 - ay0) - (by1 - by0)

        a = dx0*dx0 + dy0*dy0
        b = 2*(dx0*dx + dy0*dy)
        c = dx*dx + dy*dy

        close = interval_leq(a, b, c, d1)
        far = interval_leq(a, b, c, d2)

        ptr_f = 0
        in_far = False

        for l, r in far:
            if r > 0:
                in_far = True

        had_far |= in_far

        if close and had_far:
            ans += 1
            had_far = False

    print(ans)

if __name__ == "__main__":
    solve()
```该实现使用相对运动构建每段的二次距离函数。 每个段贡献系数$a, b, c$为平方距离多项式。 辅助函数计算该多项式低于阈值的位置，分别处理线性和简并情况。 

变量`had_far`存储系统是否经历过任何间隔超过$d_2$自上次计数问候以来。 如果“远”区域非空，每个段都会更新此标志。 当出现有效的“关闭”间隔时，仅当该标志处于活动状态时才会触发问候语，然后将其重置。 

在处理运动恒定或二次函数简化为线性函数的简并性时必须小心。 这些情况在帮助器中明确分隔，以避免被零除和不正确的根排序。

 ## 边缘情况

 当距离多项式仅接触时，会出现关键的边缘情况$d_1$在某一瞬间，没有在其下方形成完整的区间。 在这种情况下，二次方程有双根，粗心的区间提取可能会完全忽略它。 正确的处理将相等视为有效，确保即使是切向接触也算作进入闭合区域。 

另一种情况是当距离超过$d_2$由于二次峰值，仅在单个瞬间发生。 即使这是零度量，它仍然会触发重置条件，因为定义取决于时刻的存在而不是持续时间。 由于等式包含在已求解的区间中，因此二次根检测可以正确捕获这一点。 

最后一个微妙的情况是当两个阈值在单个段内以交替顺序交叉时。 由于两个不等式都是独立解决的，因此状态机以一致的时间顺序处理它们，确保在计算下一个可能的问候语之前更新重置标志。
