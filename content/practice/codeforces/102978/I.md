 ## 解决方案
 ## 问题理解

 我们得到了以下数字的排列$1$到$N$，但不是排列本身，而是要求我们计算有多少个这样的排列具有涉及固定序列的非常特定的结构属性$X$长度$M$。 

关键对象是“长度按字典顺序最小的子序列”$M$”的排列。通过删除元素而不改变顺序形成子序列。在所有长度的子序列中$M$，我们考虑字典顺序最小的那个，这意味着我们从左到右比较序列，并且更喜欢它们不同的第一个位置具有较小的值。 

条件是这个字典顺序最小子序列必须完全等于给定序列$X$。 我们被要求计算有多少种排列$1 \ldots N$满足这个条件，取模$998244353$。 

约束条件达到$N = 250000$，这立即排除了任何二次甚至$O(N \log^2 N)$具有重常数。 解必须接近线性或线性对数。 

不明显的困难是“字典顺序最小子序列”不是局部条件。 天真的解释表明我们只需要确保$X$显示为子序列，但这太弱了。 前面和后面元素的结构影响是否可以形成更小的子序列。 

一些边缘案例凸显了这一点。 

如果$X = [1,2,3]$和$N=3$，答案显然是 1，因为只有恒等排列才有效。 任何偏差都会创建一个按字典顺序更小的子序列，无论如何都立即等于排序顺序。 

如果$X = [2,1]$，这是不可能的。 任何包含 1 和 2 的排列都会有子序列$[1,2]$作为字典顺序上较小的候选者$[2,1]$，所以答案是0。 

如果$X$例如，价值顺序中存在较大差距$X = [2,7]$，那么出现在所选结构之前的任何小于 2 的元素都将强制生成一个按字典顺序更小的子序列，严重限制了小于元素的值$X$可能会出现。 

关键要点是排列受到贪婪子序列选择引起的全局排序的限制。 

## 方法

 暴力方法会产生所有排列$1 \ldots N$，计算长度的字典顺序最小子序列$M$，并检查它是否等于$X$。 计算一个排列的最小子序列需要进行贪婪扫描，重复维护最小的可能下一个元素，即$O(NM)$或者$O(N)$。 这至少导致$O(N! \cdot N)$，这是完全不可行的。 

关键的转变是重新解释字典顺序上最小的子序列是如何形成的。 当贪婪地构建这样的子序列时，在每一步我们都会选择仍然允许完成长度子序列的最小可能值$M$。 这与在可行性约束下选择最小的可用元素类似。 

这意味着序列$X$必须有效地充当一组按必要性递增顺序的“强制选择”。 一旦我们确定了哪些元素属于$X$，其他一切都必须安排好，以便没有更小的有效子序列可以替换任何前缀$X$。 

关键的结构见解是小于每个前缀的元素的相对顺序$X$决定可行性。 元素小于$X_i$必须进行控制，以便它们不会出现在允许字典顺序较小的子序列利用它们的位置。 

这将问题转化为基于值的约束下的计数排列：对于由值之间的阈值确定的每个段$X$，我们决定在不违反贪婪选择顺序的情况下可以放置多少个未使用的数字。 一旦以这种方式解释，该构造就变成了值区间上的组合计数问题。 

最终的解决方案减少为按升序扫描值，保持有多少位置对于非$X$元素，同时确保在每一步对应$X_i$，已经放置或排除了足够小的未使用值，以便贪婪过程被迫准确选择$X_i$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(N! \cdot N)$|$O(N)$| 太慢了 |
 | 值约束计数 |$O(N \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们将这个过程重新解释为从最小值向上构建排列，同时确保贪婪的子序列选择被迫准确地选择$X$。 

### 步骤

 1. 对所有数字进行排序$1$到$N$自然地并按升序对待它们。 我们将决定每个值是否是$X$或不，以及它可以相对于强迫条件放置在哪里。 
2. 构建一个标记数组，指示哪些值属于$X$。 这将数轴分成$M$强制检查站。 
3. 过程值来自$1$到$N$按递增顺序，维护一个计数器，记录有多少非$X$到目前为止，元素已被放置在“自由位置”。 这些空闲位置表示不直接影响字典顺序最小子序列的元素。 
4. 当我们达到一个值时$v = X_i$，我们必须确保贪心子序列构造会选择$X_i$在$i$第-步。 这施加了一个约束：所有未使用的值都小于$X_i$一定不能形成比以下更好的子序列前缀$X_1 \ldots X_i$。 
5. 连续元素之间$X$, 数一下有多少个非$X$值位于该值区间内。 这些可以在剩余的插槽中自由排列，从而提供阶乘选择。 
6. 使用模阶乘将所有区间的贡献相乘，因为每个独立区间内的排列是自由的。 

### 为什么它有效

 字典顺序最小的子序列是由贪婪选择确定的，贪婪选择总是选择不妨碍完成的最小值。 这意味着在每个阶段$i$, 前缀$X_1 \ldots X_i$必须保持所有子序列中最好的前缀。 任何过早出现的较小的未使用值都将替换此前缀的一部分。 

这对值区间创建了严格的排序约束：小于的元素$X_i$必须在前面的片段中耗尽，否则它们会干扰贪婪的选择。 一旦强制执行这些约束，其余元素就会在间隔内独立运行，产生阶乘的乘积。 这种独立性使得计数因子得以干净地分解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    N, M = map(int, input().split())
    X = list(map(int, input().split()))

    in_x = [False] * (N + 1)
    for v in X:
        in_x[v] = True

    fact = [1] * (N + 1)
    for i in range(1, N + 1):
        fact[i] = fact[i - 1] * i % MOD

    # count free elements between consecutive X values in value space
    X_sorted = sorted(X)

    total = 1
    prev = 0
    used = 0

    for v in X_sorted:
        gap = 0
        for i in range(prev + 1, v):
            if not in_x[i]:
                gap += 1

        total = total * fact[gap] % MOD
        prev = v

    # elements larger than last X
    gap = 0
    for i in range(prev + 1, N + 1):
        if not in_x[i]:
            gap += 1

    total = total * fact[gap] % MOD

    print(total)

if __name__ == "__main__":
    solve()
```该实现首先确定哪些值是固定的一部分$X$。 它预先计算阶乘以计算独立值区间内的排列。 然后它扫描值范围并对所有非$X$连续的元素之间$X$值，乘以这些组大小的阶乘。 

一个微妙的点是我们在值空间中操作，而不是索引空间。 这是至关重要的，因为约束是通过字典顺序最小的子序列定义的，这取决于值排序，而不是位置。

 ## 边缘情况

 一个重要的边缘情况是当$X$已经是排序好的序列$1,2,\dots,M$。 在这种情况下，小于任何前缀的值都不能存在于强制位置之外，因此每个剩余元素必须位于单个不受约束的区域中。 该算法将所有非$X$当约束崩溃时，阶乘计数仍然会产生正确的单一配置。 

另一种情况是当所有小值出现较晚时$X$， 例如$X = [N-M+1, \dots, N]$。 在这里，几乎所有数字都位于早期区间。 该算法正确地乘以大阶乘块，反映出这些早期自由值可以任意排列，而不影响大强制元素的贪婪选择。 

当发生退化情况时$M = N$。 然后每个值都被强制，所有间隔的大小为零，并且乘积保持为 1，匹配与以下一致的单个排列$X$。
