 ## 解决方案
 ## 问题理解

 我们被要求计算有多少长度-`n`我们可以使用构建序列`k`符号，其中每个位置都是测验答案选择。 该约束禁止任何四个相同的连续值的运行。 换句话说，序列允许重复一个值，但它永远不能包含像这样的块`x x x x`。 

还有第二个动机：教授更喜欢长串相同的答案，尤其是三元组，但最终的要求纯粹是组合。 我们不是在优化分数；而是在优化分数。 我们正在计算所有有效序列。 

所以任务是：计算所有长度的序列`n`超过一个字母表大小`k`这样就没有字符连续出现四次。 

约束条件达到`n, k ≤ 10^6`，这会立即排除任何尝试显式构建序列或使用每个状态具有 O(nk) 甚至 O(n) 转换的 DP 的解决方案。 我们需要一个封闭的形式，或者最坏的情况是一个可以在 O(n) 中评估的线性递归。 

例如，简单的动态编程方法将通过位置和当前游程长度来定义状态`dp[i][len]`， 在哪里`len`是 1 到 3。这给出了 O(n) 状态，但仍然是 O(n) 转换，这在理论上很好，但前提是转换是恒定时间并且独立于`k`。 然而，转换取决于选择相同或不同的下一个值，因此我们需要确保可以将所有内容压缩为恒定数量的递归变量。 

边缘情况在这里很重要。 为了`n = 1`，任何一个`k`符号有效。 为了`n = 2`和`n = 3`，每个序列都是有效的，因为不可能形成四个连续的相同值。 为了`n = 4`，唯一无效的序列是所有四个位置都相等的序列，它们恰好是`k`序列。 任何解决方案都必须正确地减少这些小情况，否则重现可能会被错误建模。 

## 方法

 暴力方法将生成所有`k^n`序列并检查每个序列是否违反四个连续的相等值。 这显然是指数级的，甚至对于`n = 20`。 发生失败的原因是我们反复重新检查几乎相同的前缀结构。 

关键的观察结果是，有效性仅取决于最后几个元素，而不是整个历史。 具体来说，唯一禁止的模式是长度为 4 的游程，因此我们只需要跟踪最后一个值的当前游程长度，上限为 3。 

这表明了动态编程压缩：我们不跟踪完整序列，而是跟踪有多少有效序列以长度为 1、2 或 3 的相同值结束。 任何新元素要么扩展运行，要么以不同的值开始新的运行。 “不同价值”的贡献总是乘以`k-1`，独立于历史，这是至关重要的简化。 

这将问题简化为具有线性递推的恒定大小状态系统`n`，即使对于`10^6`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(k^n) | O(k^n) | O(n) | 太慢了|
 | 具有运行状态的 DP | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们为每个位置定义三个数量`i`:`a1[i]`：有效长度序列的数量`i`以恰好 1 个相同值的运行结束`a2[i]`：以 2 个完全相同的值结束`a3[i]`：以 3 个完全相同的值结束

 所有序列都分为这三类。 

1. 初始化为`i = 1`。 任何一个`k`值形成长度为 1 的游程，因此`a1[1] = k`， 尽管`a2[1] = a3[1] = 0`。 
2. 对于每个位置`i > 1`，考虑长度序列`i-1`可以通过附加新值来扩展。 
3. 计算`a1[i]`，我们在该位置附加一个与序列最后一个元素不同的值`i-1`。 任意长度的序列`i-1`可以扩展为`k-1`方式，无论其结束游程长度如何。 所以`a1[i] = (a1[i-1] + a2[i-1] + a3[i-1]) * (k-1)`。 
4. 计算`a2[i]`，我们必须附加与最后一个元素相同的值，但仅限于以恰好 1 的游程结束的序列。否则我们将超过 3 的游程。所以`a2[i] = a1[i-1]`。 
5. 计算`a3[i]`，我们通过再次重复相同的值来扩展长度为 2 的游程，所以`a3[i] = a2[i-1]`。 
6.最终答案是`a1[n] + a2[n] + a3[n]`。 

这种分解有效的原因是，每个以长度为 1、2 或 3 的游程结束的序列唯一地确定如何在不违反“无四个连续”规则的情况下扩展它，并且不需要其他隐藏状态。 

### 为什么它有效

 整个约束仅取决于最后游程长度是否达到 4。具有相同最后游程长度的任何两个序列在扩展下表现相同：附加不同的值始终会重置游程，而附加相同的值会将其增加 1。 由于游程长度不允许超过 3，因此这三个状态完全捕获所有有效历史。 这按最终状态创建了有效序列的完整划分，并且每个转换都保持有效性而没有重叠或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n, k = map(int, input().split())
    
    if n == 1:
        print(k % MOD)
        return
    
    a1 = k % MOD
    a2 = 0
    a3 = 0
    
    for _ in range(2, n + 1):
        total = (a1 + a2 + a3) % MOD
        
        na1 = total * (k - 1) % MOD
        na2 = a1 % MOD
        na3 = a2 % MOD
        
        a1, a2, a3 = na1, na2, na3
    
    print((a1 + a2 + a3) % MOD)

if __name__ == "__main__":
    solve()
```该实现直接反映了三态模型。 这`total`变量表示以位置结束的所有序列`i-1`，用于计算转换为`a1`。 过渡到`a2`和`a3`是游程长度的严格变化，这意味着它们仅取决于先前的游程结构，而不取决于`k`。 

一个微妙的点是处理`k-1`什么时候`k = 1`。 在那种情况下，`a1`在第一步之后，转换总是变为零，这正确地反映了仅存在单个常量序列，并且一旦长度超过 3，它就会变得无效。

 ## 边缘情况

 对于`n = 1`，算法初始化`a1 = k`，它直接计算所有有效的单题测验。 由于未执行任何转换，因此输出保持正确。 

为了`k = 1`，每个序列都被迫保持不变。 递归仍然有效，因为`k-1 = 0`， 所以`a1`第一次转换后变为零，而`a2`和`a3`在崩溃之前进化了整整三步。 为了`n ≥ 4`，结果正确地变为零，因为唯一可能的序列违反了约束。 

为了`n = 4`，DP 生成非零计数，但排除所有元素都相同的单个无效序列。 这与每个符号选择都存在一个禁止配置的事实相匹配，产生`k^4 - k`有效序列，递归通过不允许转换到游程长度 4 来隐式强制执行该序列。
