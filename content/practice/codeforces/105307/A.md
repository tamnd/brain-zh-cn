 ## 解决方案
 ## 问题理解

 我们得到了一系列牌组，每组牌只包含两种类型的牌：红色和蓝色。 每轮中，当前庄家选择一张未使用的牌，将牌洗好，另一位玩家统一随机挑选一张牌。 如果抽到的牌是红色的，则下一轮的角色保持不变。 如果它是蓝色的，则角色互换。 

当所有牌组按一定顺序使用一次后，最后发牌的玩家获胜。 两名玩家都是完全理性的，并且知道所有牌组的组成，因此唯一的随机性是抽到的牌的颜色。 

关键的隐藏结构是，每副牌都会贡献一个概率“角色翻转事件”，其概率仅取决于其蓝色分数，并且牌组的使用顺序很重要，因为当前玩家可以最佳地选择下一张牌。 

输出是初始庄家在所有回合后最终成为庄家的概率，在大质数模数下减少为模分数。 

约束最多允许一百万副牌，因此任何解决方案都必须是线性或近线性的。 任何对或排列的二次推理都是不可能的。 即使 O(n log n) 也必须仔细构造，但 O(n) 或 O(n log n) 的排序是可以接受的。 

对牌组顺序排列的简单模拟是不可能的，因为 n 很大，甚至对子集的动态编程也是不可行的。 

当所有牌组的概率效应相同时，就会出现微妙的边缘情况，例如，对于所有 i，p = 1，q = 1。 在这种情况下，无论顺序如何，每副牌都会以 1/2 的概率翻转角色，并且任何有关顺序的贪婪假设仍然必须正确地简化为对称乘积形式。 如果不考虑当前玩家的选择权来对待每一轮，错误的方法常常会失败。 

## 方法

 如果我们忽略最佳玩法，游戏就会变得简单：最终的角色仅取决于蓝色抽牌的数量是偶数还是奇数，但每次抽牌的概率取决于所选择的牌组。 然而，最佳玩法改变了一切，因为庄家选择接下来使用哪副牌，而这种选择会影响未来角色转换的概率。 

一个蛮力的想法是考虑所有可能的选择套牌的顺序。 对于每个序列，通过乘以各轮的转移概率来计算最终角色是庄家的概率。 因为有 n! 序列，即使 n = 10，这也是立即不可行的。 

关键的观察是，每副牌 i 都可以通过使用时切换角色的概率来概括：

 pi = q_i / (p_i + q_i)。 每一轮都是概率为 pi 的随机状态翻转，但至关重要的是，当前玩家选择下一步应用哪个 pi。 这是乘法效应的经典最优排序问题。 

我们将牌组的效果重写为两种状态的线性变换：处于“荷官优势保留”状态与“交换”状态的概率。 每副牌都贡献一个 2×2 的转换矩阵，矩阵的乘积取决于阶数。 游戏变成选择矩阵的顺序以最大化应用于初始向量的乘积的左上角条目。 

直接矩阵视角揭示了一种简化：每个牌组对应于一个分数线性变换，将它们组合起来会产生一个结构，其中每个状态只有一个标量参数很重要。 最佳玩法简化为通过蓝色与总比率的单调函数对牌组进行排序，因为提前或晚使用牌组的好处仅取决于它减少剩余状态的“确定性质量”的程度。

这产生了一个贪婪策略：根据其比率计算每个套牌的价值权重，然后按排序顺序处理套牌，以最大限度地保留当前角色优势的预期。 最终概率成为按最佳顺序独立贡献的乘积。 

这将问题简化为排序加线性累加。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解订单| O(n!) | O(n) | 太慢了|
 | 最优贪心排序+累加| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 ### 密钥转换

 1. 对于每副牌，计算总牌数 t_i = p_i + q_i 和蓝色概率 r_i = q_i / t_i。 这代表了这套牌在使用时翻转角色的概率。 
2. 将状态重写为当前玩家是原庄家的概率x。 最初 x = 1。每副牌根据是否发生翻转将 x 转换为新值。 由于翻转对称地交换角色，所以变换在 x 上是线性的。 
3. 观察应用一副牌 i 会导致形式的转变：

 x → x * (1 - r_i) + (1 - x) * r_i = x * (1 - 2r_i) + r_i

 这表明每个甲板都是斜率 (1 - 2r_i) 的仿射变换。 
4. 最终概率取决于按所选顺序组合这些仿射函数。 合成顺序很重要，因为斜率会倍增，常数会累积。 
5. 庄家的最优选择相当于从x = 1开始选择仿射变换的顺序来最大化最终值。 
6. 对于仿射函数 f_i(x) = a_i x + b_i，组合顺序影响 a_i 与 b_i 项的加权和的乘积。 最佳排序是通过从 (b_i / (1 - a_i)) 导出的单调比较进行排序来确定的，该比较简化为 r_i 的函数。 
7. 这通过减少 r_i / (1 - r_i) 来简化对牌组的排序，这相当于当两者都为正时按 q_i / p_i 排序，并且对零情况进行一致的处理。 
8. 排序后，依次应用变换来计算最终概率模 P。 

### 为什么它有效

 每副牌都会对概率状态进行线性分数更新，并且此类更新的组合是关联的但不可交换。 优化简化为对仿射图进行排序，以从固定的初始条件开始最大化最终值。 决策标准仅取决于每个地图如何缩放当前的不确定性与其注入的恒定质量。 这会产生单调排序，因为成对交换分析表明排序顺序的任何反转都无法改善最终结果，从而建立了贪婪排序的最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n = int(input())
    decks = []
    
    for _ in range(n):
        p, q = map(int, input().split())
        t = p + q
        r_num = q
        r_den = t
        decks.append((r_num, r_den))
    
    # sort by q/(p+q) equivalently by q * (p2+q2) vs q2 * (p+q)
    def cmp(a):
        q, t = a
        return q * modinv(t)
    
    # avoid floating: sort by q/t decreasing => cross multiply
    def key(a):
        q, t = a
        return q * pow(t, MOD - 2, MOD)
    
    # sort descending by q/t
    decks.sort(key=lambda x: x[0] * modinv(x[1]), reverse=True)
    
    x = 1
    
    for q, t in decks:
        r = q * modinv(t) % MOD
        x = (x * (1 - 2 * r) + r) % MOD
    
    print(x % MOD)

if __name__ == "__main__":
    solve()
```实现的核心是通过降低蓝色概率对牌组进行排序后顺序应用仿射更新。 逆模用于计算模下的每个 r_i。 

排序步骤至关重要，因为使用错误的顺序会破坏仿射变换的最佳组合。 变换更新线直接对导出的递归进行编码。 

## 工作示例

 ### 示例 1

 输入：```
1
1 2
```我们有一副 r = 2/3 的牌。 

| 步骤| x 之前 | r | 转型| x 之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 2/3 | x → x(1-2r)+r | 1*(1-4/3)+2/3 = 1/3 | 1*(1-4/3)+2/3 = 1/3 |

 输出为1/3。 

这证实了对于一副牌，结果与直接概率计算相匹配。 

### 示例 2

 输入：```
2
1 1
1 1
```两副牌的 r = 1/2，因此无论 x 如何，每次变换都会变为 x → 1/2。 

| 步骤| x 之前 | r | 转型| x 之后 |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 1/2 | 1/2 x → 1/2 | 1/2 | 1/2
 | 2 | 1/2 | 1/2 1/2 | 1/2 x → 1/2 | 1/2 | 1/2

 最终结果是1/2。 

这显示了重复相同随机变换下的稳定性。

 ## 边缘情况

 当一副牌只有红牌、p > 0 且 q = 0 时，就会出现临界边缘情况。转换变为 x → x，因此它不应影响排序或值。 该算法可以正确处理这个问题，因为 r = 0 会导致恒等变换。 

对于纯蓝色牌组，p = 0 且 q > 0，变换变为 x → 1 - x。 在这种情况下，r = 1，仿射形式正确地简化为完全翻转。 应用该公式会产生 x → x(1 - 2) + 1 = 1 - x，匹配行为。 

另一个微妙的情况是许多牌组具有相同的比率。 它们之间的排序变得任意，但由于具有相同斜率的仿射变换可以达到相同的效果，任何内部排列都会产生相同的结果，因此保持了稳定性。 

最后的边缘情况是当 t_i 很大时模逆的数值处理。 由于 t_i ≤ 10^6 并且 MOD 是素数，因此所有有效输入都存在模逆，并且每次计算在模算术下仍然是安全的。
