 ## 解决方案
 ## 问题理解

 我们在 2D 平面上得到了从原点开始的一系列位置。 圣诞老人的雪橇不再自由选择方向。 相反，它重复执行由四个基本方向组成的字符串给出的固定运动模式。 

我们概念性地想象圣诞老人站在位置 (0, 0) 并从左到右应用字符串。 弦结束了一次，他立刻又重复了一遍，如此无限地持续下去。 每次重复都会产生一个确定性的位移矢量，因此运动在结构上是周期性的。 

问题是，在无限行走中的任何一点，圣诞老人是否会准确地落在目标坐标 (x, y) 上。 这不限于字符串的完整循环的结束，而是可以发生在任何重复内的任何中间步骤。 

这些限制有两个方面的影响。 坐标范围达到10^18比例，因此直接模拟步骤是不可能的。 字符串长度可达 100,000，因此即使单次传递也已经足够大，以至于任何 O(n^2) 或每个周期的重复模拟都太慢。 唯一可行的方向是将一个完整的字符串执行视为一种转换，并对重复的应用程序进行代数推理。 

当一整串后的净位移为零时，会出现一种微妙的边缘情况。 在这种情况下，雪橇被困在有界轨道内，并且只有在第一个周期中访问过的位置才重要。 如果我们忽略这一点并假设周期性移动，我们可能会错误地声称从未实际访问过的坐标可达。 

当仅在循环的中间段而不是循环边界可到达目标坐标时，会发生另一种边缘情况。 例如，如果字符串是“UR”，则访问的位置是(0,0)、(0,1)、(1,1)，然后重复。 像 (0,1) 这样的目标必须在循环内检测到，而不仅仅是通过净位移推理。 

## 方法

 暴力破解的想法很简单：逐步模拟雪橇，存储每个访问过的坐标，并检查 (x, y) 是否出现。 这是正确的，因为运动是确定性的和周期性的，所以最终我们要么看到目标，要么看不到。 问题在于路径是无限的，因此暴力必须在经过一定数量的步骤后人为停止。 

自然的截止是模拟多达一些大量的循环，例如 O(n * C) 个步骤，其中 C 足够大以“覆盖”所有可能的偏移。 然而，由于坐标可以大到 10^18，因此对于可能需要的周期数没有安全限制。 即使几百万个周期也会超出时间限制。 

关键的见解是每个全串应用都会贡献一个固定的位移矢量。 令该位移为(dx,dy)。 经过 k 个完整周期后，位置为 k * (dx, dy) 加上当前周期内的位置。 这减少了检查点的算术级数的无限行走。 

然后我们扭转视角。 我们不是询问 (x, y) 是否被访问过，而是询问字符串中是否存在循环索引 k 和前缀位置 i，使得 i 处的前缀位移加上 k 乘以全循环位移等于 (x, y)。 这将问题转化为检查一小组前缀状态上的线性丢番图条件。 

如果(dx,dy)不是(0, 0)，那么对于每个候选前缀k是唯一确定的，我们只需要验证k是否是两个坐标一致的非负整数。 如果(dx,dy)为(0, 0)，那么循环之间不会发生移位，因此只有一个循环重要，我们直接检查目标是否出现在前缀遍历中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(无限或有界 O(n·C)) | O(n) | 太慢/不完整|
 | 前缀+循环分解| O(n) | O(n) | 已接受 |

## 算法演练

 1. 计算字符串中从 (0, 0) 开始的每个字符后面的雪橇前缀位置。 这给出了一个周期内每个步骤之后的确切位置。 这是必要的，因为目标可能在周期中期达到。 
2. 处理整个字符串后，取最终的前缀位置，计算一个完整周期的总位移为(dx,dy)。 这捕获了整个图案在一次重复后如何移动空间。 
3. 对于字符串中索引 i 对应的每个前缀位置 (px, py)（包括初始的 (0, 0)），考虑在一定数量的完整循环后目标 (x, y) 是否可以匹配此状态。 这意味着我们检查对于某个整数 k ≥ 0，(x - px, y - py) 是否可以表示为 k * (dx, dy)。 
4. 如果 dx 非零，则将 k 计算为 (x - px) / dx 并验证其是否为整数。 还要检查将其应用于 y 是否具有一致性：(y - py) 必须等于 k ​​ dy。 这可以防止 dx 和 dy 未对齐时缩放比例不匹配。 
5. 如果 dx 为零但 dy 不为零，则仅使用 y 坐标进行对称检查，因为水平运动在周期之间永远不会改变。 
6. 如果 dx 和 dy 都为零，则循环根本不会移动雪橇。 在这种情况下，只有第一个循环很重要，因此我们只需检查任何前缀位置是否等于目标。 
7. 如果有任何前缀位置满足条件，则返回“Yes”。 否则返回“否”。 

它之所以有效，是基于这样一个不变量：无限行走中的每个访问点都可以唯一地分解为一个周期内的前缀状态加上整数个全周期移位。 前缀枚举涵盖了所有周期内的可能性，而 (dx, dy) 的线性缩放捕获了所有周期间的运动。 除了这些组合之外，轨迹中不存在其他点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    x, y = map(int, input().split())
    s = input().strip()

    # prefix positions
    px = py = 0
    pref = [(0, 0)]

    for c in s:
        if c == 'U':
            py += 1
        elif c == 'D':
            py -= 1
        elif c == 'L':
            px -= 1
        else:
            px += 1
        pref.append((px, py))

    dx, dy = pref[-1]

    for i, (cx, cy) in enumerate(pref):
        tx = x - cx
        ty = y - cy

        if dx == 0 and dy == 0:
            if tx == 0 and ty == 0:
                print("Yes")
                return
            continue

        if dx == 0:
            if tx == 0 and dy != 0 and ty % dy == 0:
                k = ty // dy
                if k >= 0 and cy + k * dy == y:
                    print("Yes")
                    return
            continue

        if dy == 0:
            if ty == 0 and dx != 0 and tx % dx == 0:
                k = tx // dx
                if k >= 0 and cx + k * dx == x:
                    print("Yes")
                    return
            continue

        if tx % dx == 0:
            k = tx // dx
            if k >= 0 and cy + k * dy == y:
                print("Yes")
                return

    print("No")

if __name__ == "__main__":
    solve()
```前缀构造直接编码所有可能的循环内端点。 最终位移`(dx, dy)`捕获重复结构。 每个条件块处理简并性，其中一个或两个轴在周期内不改变，否则会导致被零除或无效的整数假设。 关键的实现细节是始终验证两个坐标的一致性，而不仅仅是一个坐标的一致性。 

## 工作示例

 考虑输入：```
2 2
UR
```前缀状态是：

 | 我| 移动| 位置|
 | ---| ---| ---|
 | 0 | 开始 | (0, 0) | (0, 0) |
 | 1 | 你| (0, 1) | (0, 1) |
 | 2 | 右 | (1, 1) |

 循环位移为(1, 1)。 目标 (2, 2) 与前缀 (0, 0) 匹配 k = 2 个周期，因此可达。 

这表明可达性可以恰好发生在循环边界处，并且必须始终考虑前缀 (0, 0)。 

现在考虑：```
1 2
RU
```前缀：

 | 我| 移动| 位置|
 | --- | --- | --- |
 | 0 | 开始 | (0, 0) | (0, 0) |
 | 1 | 右 | (1, 0) | (1, 0) |
 | 2 | 你| (1, 1) |

 循环位移为(1, 1)。 目标(1, 2)不能表示为任何前缀加上(1, 1)的整数倍，因此不可达。 

这说明了为什么仅检查距离大小或朴素周期性会失败，因为 x 和 y 之间的方向耦合很重要。

 ## 边缘情况

 一种重要的边缘情况是整个弦的位移矢量为零。 在这种情况下，雪橇永远不会在循环之间移动，因此唯一可到达的点是字符串第一次遍历时的点。 该算法通过立即将问题减少为单周期前缀检查来解决此问题，确保跨周期不会出现错误的外推。 

当 dx 或 dy 为零时，会出现另一种边缘情况。 如果 dx 为零，则水平位置在周期之间永远不会改变，因此尝试除以 dx 将无效。 该算法明确地分离了这种情况，并且仅对实际发生变化的轴执行有效性检查。 

最后一个微妙的情况是仅在开始时 (0, 0) 才能到达目标。 这是因为前缀列表包括任何移动之前的初始位置，因此算法可以正确识别立即成功，而不需要任何循环推理。
