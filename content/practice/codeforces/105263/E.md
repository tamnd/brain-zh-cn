 ## 解决方案
 ## 问题理解

 我们得到一行$n$石头，并且每块石头必须使用其中之一进行绘制$c$可用颜色。 唯一的限制是相同颜色的运行：我们不允许有任何长度的块$k$或更多，其中所有宝石具有相同的颜色。 换句话说，如果一种颜色连续出现，它的条纹长度必须始终严格低于$k$。 

任务是计算每个测试用例存在多少个有效着色，并输出对给定素数取模的结果$p$。 测试用例的数量可能很大，并且每个测试用例可能涉及很大$n$和$c$，所以我们需要一种避免显式枚举着色的方法。 

这些约束已经完全消除了暴力。 即使对于中等$n$，序列总数为$c^n$，并且这超出了任何可行的计算。 的存在$n$最多$10^6$强制每个测试用例采用线性或近线性方法。 附加约束$k \ge 2$表明限制是局部的，并且仅取决于连续结构，这通常表示动态规划或组合递归。 

出现微妙的边缘情况时$k > n$。 在这种情况下，约束实际上是无关的，因为没有长度游程$k$可以存在于一段长度内$n$顺序。 答案应该很简单$c^n$。 另一个边缘情况是$k = 2$，这禁止任何两个相邻的石头具有相同的颜色，将问题变成一个经典的“不相等相邻”计数问题并给出答案$c \cdot (c-1)^{n-1}$。 任何通用解决方案都必须正确降级到这些特殊情况。 

## 方法

 一个天真的尝试是构建从左到右的序列并递归地计算所有有效的分配。 在每一个岗位上，我们都竭尽全力$c$颜色，并且我们只会拒绝某个选择，如果它会产生禁止的长度$k$。 为了强制执行此操作，状态必须记住最后一种颜色的当前游程长度。 这会导致 DP 状态，例如位置和当前条纹长度，其转换取决于我们是否继续相同的颜色或切换。 

这个公式是正确的，但如果直接实施，就会立即变得太大。 游程长度可达$k-1$，所以状态空间是$O(nk)$。 和$n$最多$10^6$，甚至存储或迭代这样的 DP 也是不可能的。 

关键的简化是停止跟踪精确的颜色，而是仅跟踪运行的结构。 重要的观察是约束只关心游程是否达到长度$k$，而不是关于运行的颜色。 这意味着所有颜色都是对称的，我们只需要计算序列可以由多少种长度组合而成$1$到$k-1$，每次运行选择与前一次运行不同的颜色。 

如果我们从运行的角度思考，每个序列都是块的串联。 每个块都有长度$1$到$k-1$。 第一个块可以是任何颜色，因此它贡献了一个因素$c$。 每个后续块必须选择与前一个块不同的颜色，从而贡献一个因素$c-1$，并独立选择其长度。 

这导致了标准线性递归$f[i]$， 在哪里$f[i]$是长度有效序列的数量$i$。 对于每个位置，我们要么延长当前的运行（只要它没有达到长度$k$），或者使用不同的颜色开始新的运行。 困难在于朴素 DP 仍然需要跟踪游程长度，但这可以使用最后一个的前缀和来消除$k-1$州。 

转换变成滑动窗口递归：形成在位置结束的序列$i$，我们考虑最后一次运行可能从位置开始的所有方式$i-1, i-2, \dots, i-(k-1)$。 每个这样的可能性对应于固定最后的游程长度并选择其颜色，同时确保先前的位置已经结束了有效的配置。 

这将每个测试用例的计算量减少到$O(n)$，因为每个状态都可以使用滚动窗口总和进行更新。

 | ---| ---| ---| ---|
 | 暴力 DP 超出运行状态 |$O(nk)$|$O(nk)$| 太慢了 |
 | 滑动窗口DP复发|$O(n)$|$O(n)$或者$O(1)$| 已接受 |

 ## 算法演练

 我们定义$f[i]$作为第一个有效着色的数量$i$石头。 

我们还维护一个前缀和数组$pref[i] = f[0] + f[1] + \dots + f[i]$, 计算模数$p$，以允许快速范围求和。 

递归来自于考虑长度着色中的最后一个块$i$。 假设最后一个块的长度$t$， 在哪里$1 \le t < k$。 然后是上一个$i-t$位置形成任何有效序列，最后一个块提供与前缀的最后一个块不同的颜色选择。 

1. 初始化$f[0] = 1$，代表空序列。 
2. 对于每个位置$i$从$1$到$n$，我们想要计算所有以$i$。 
3.如果$i < k$，目前还没有限制，所以每个扩展都是有效的。 我们可以计算$f[i] = c \cdot (c-1)^{i-1}$直接或通过复发。 
4. 对于一般$i \ge k$，我们按最后一个块的长度进行分割$t$，范围从$1$到$k-1$。 对于每个$t$，前缀贡献$f[i-t]$，然后我们乘以$c-1$因为最后一个块的颜色必须与前一个块的颜色不同。 
5. 使用前缀和有效地计算窗口上的总和：$$f[i] = (c-1) \cdot (f[i-1] + f[i-2] + \dots + f[i-k+1])$$6. 我们维护前缀和，以便每个$f[i]$可以在常数时间内计算出来。 

最终的答案是$f[n]$。 

这样做的原因是每个有效的长度着色$i$由最后一次运行开始的位置唯一确定。 该起点必须在最后一个点内$k-1$位置，否则运行将违反约束。 起始位置的每个选择恰好对应于有效前缀和新运行的一次分解，并且所有此类分解都是不相交的。 这给出了解决方案空间的完整划分，这正是滑动窗口和捕获的内容。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, c, k, p = map(int, input().split())

        if k > n:
            # no restriction applies
            res = pow(c, n, p)
            print(res)
            continue

        # f[i] = number of valid arrays of length i
        f = [0] * (n + 1)
        pref = [0] * (n + 1)

        f[0] = 1
        pref[0] = 1

        for i in range(1, n + 1):
            # window sum f[i-1] ... f[i-k+1]
            left = i - 1
            right = i - k

            total = pref[left]
            if right >= 0:
                total = (total - pref[right] + p) % p

            f[i] = total * (c - 1) % p
            pref[i] = (pref[i - 1] + f[i]) % p

        print(f[n])

if __name__ == "__main__":
    solve()
```该实现维护一个用于序列计数的 DP 数组和一个前缀求和数组以允许恒定时间范围求和。 转换直接编码了最后一次运行的长度必须最多的想法$k-1$，所以我们只对之前有效的切割位置进行求和。 乘以$c-1$反映了与之前运行的颜色不同的颜色选择。 

一个微妙的实现细节是计算窗口和时模减法的处理。 由于前缀和可以相减，因此我们必须添加$p$在取模之前以避免负值。 

特殊情况$k > n$使用快速求幂单独处理，因为递归退化为不受限制的着色。

 ## 边缘情况

 当$k > n$，DP 窗口条件永远不会激活。 例如，与$n=5, k=10$，否则循环将计算空范围。 该算法显式切换到$c^n$，因此它避免了未定义的前缀减法并匹配每个着色都是有效的事实。 

什么时候$k=2$，窗口大小变为1，因此每个状态仅取决于$f[i-1]$。 递归正确地折叠为$f[i] = (c-1) f[i-1]$，并且该实现自然会产生指数行为，无需特殊处理。 

什么时候$c=1$，唯一可能的序列是恒定颜色。 如果$k \le n$，这违反了约束，并且 DP 正确地产生零，因为因子$c-1$变为零，迫使所有$f[i]$为了$i \ge 2$消失。
