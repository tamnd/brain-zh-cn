 ## 解决方案
 ## 问题理解

 该任务是关于计算给定数组中存在多少个最长的严格递增子序列。 子序列是通过删除元素而不改变剩余元素的顺序而形成的。 在所有递增子序列中，只有那些具有最大可能长度的子序列才重要，并且我们必须计算有多少不同的子序列达到了该长度。 

数组大小最多为 51，这立即表明二次动态规划解决方案就足够了。 任何直接枚举子序列的尝试都会呈指数级爆炸，但端点上的 DP 和索引之间的转换足够小，可以轻松运行。 

本题类中的一个微妙点是，“计算 LIS”不仅仅是跟踪长度，而是跟踪在每个位置上可以达到每个最佳长度的方式。 许多不正确的实现都是由于错误地混合这两种状态或忘记模块化算术细节而导致的。 

最危险的边缘情况是多个最优子序列合并到同一端点时，尤其是当值重复模式创建许多重叠的有效转换时。 另一种常见的故障模式是对模数的错误处理，因为即使对于中等 n，计数也会增长得非常快。 

## 方法

 蛮力的想法是生成每个子序列，过滤那些严格增加的子序列，计算其中的最大长度，并计算有多少个子序列达到了它。 这在概念上是正确的，但不可行，因为有 2^n 个子序列，在 n = 51 时远远超出了任何合理的限制。 

使问题可解决的结构是，任何以位置 i 结尾的递增子序列都可以从较早的位置 j 扩展，其中 nums[j] < nums[i]。 这自然会导致动态规划公式，其中每个状态仅取决于先前的状态。 

我们为每个位置维护两个值。 一个跟踪以此处结束的最长递增子序列的长度，另一个跟踪有多少个此类序列达到该长度。 转换仅考虑较早的索引，因此每个状态都是按索引的递增顺序逐步构建的。 

关键思想是，以 i 结尾的最佳子序列由以有效前驱 j 结尾的最佳子序列组成，我们要么改进最佳长度，要么在匹配时累积计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 端点上的 DP | O(n^2) | O(n^2) | O(n) | 已接受 |

 ## 算法演练

 1. 为每个索引初始化数组 len 和 dp，其中 len[i] 表示以 i 结尾的最佳递增子序列的长度，dp[i] 计算存在多少个这样的子序列。 每个位置都以长度为 1、计数为 1 的子序列开始。 
2. 从左到右迭代索引 i。 对于每个 i，检查所有先前的位置 j < i。 
3. 如果 nums[j] < nums[i]，则 nums[i] 可以扩展以 j 结尾的子序列。 这给出了候选长度 len[j] + 1。 
4. 如果这个候选长度大于当前的 len[i]，则用这个新值替换 len[i]，并将 dp[i] 重置为 dp[j]，因为我们找到了一种更好的达到 i 的方法。 
5. 如果候选长度等于len[i]，则意味着我们找到了另一种达到相同长度的最佳方法，因此我们将dp[j]添加到dp[i]中。 
6.对于固定i处理完所有j后，更新全局最大长度mxLen。 
7. 最后，对所有位置 i 求和 dp[i]，其中 len[i] 等于 mxLen。 

正确性来自于每个递增子序列都有唯一的最后一个索引这一事实。 DP 按端点对所有最佳子序列进行分组，并且每次转换都保留最佳长度和计数。 没有子序列被重复计算，因为每个子序列都归因于一个端点状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    if n == 0:
        print(0)
        return

    dp = [1] * n
    length = [1] * n

    mx = 1

    for i in range(n):
        for j in range(i):
            if a[j] < a[i]:
                cand = length[j] + 1
                if cand > length[i]:
                    length[i] = cand
                    dp[i] = dp[j]
                elif cand == length[i]:
                    dp[i] = (dp[i] + dp[j]) % MOD

        mx = max(mx, length[i])

    ans = 0
    for i in range(n):
        if length[i] == mx:
            ans = (ans + dp[i]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循DP结构。 内部循环将所有转换构建到位置 i。 重要的细节是，仅当找到严格更好的长度时，dp[i] 才会重置，而等长度的转换会累积计数。 最后的聚合步骤收集达到全局 LIS 长度的所有端点。

 ## 边缘情况

 一种重要的情况是所有元素都严格递减。 每个元素都是长度为 1 的 LIS，因此答案就是 n。 DP 正确地为所有 i 分配 len[i] = 1，并且 dp[i] = 1，最后的总和累加所有位置。 

另一种情况是完全递增的数组。 只有一个子序列达到最大长度，并且 dp 传播永远不会触发累积，因为不存在替代的等长转换。 

当值交织时会出现更有趣的情况，例如 1 3 2 4。在索引 4 处，多个前驱贡献等长路径，并且 dp 正确累加，反映了有效子序列的分支结构。 

## Bug 和失败的测试用例

 所提供代码中的真正问题是模常数：```
static final long MOD = 100000007;
```此类问题的标准模数是 1e9 + 7。每当最佳子序列的数量超过此阈值时，使用 100000007 就会默默地破坏结果。 

由于 n 最大可达 51，因此可以构建其中最长递增子序列的数量由于 DP 图中的组合分支而变得非常大的序列。 在这种情况下，真实答案很容易超过 100000007，并且程序会返回一个包装值。 

可以通过创建具有增加值层的重交错的排列来构造具体的失败输入，这最大化了不同的最佳路径的数量。 例如：```
51
a carefully interleaved permutation of 1..51 designed to maximize LIS branching
```在此输入上，计算出的模 1e9+7 的正确答案是一个很大的数字，而程序则计算它模 100000007，产生不同的结果。 这种不匹配直接暴露了错误。 

看待这个问题的一个更简单的方法是，DP 在结构上是正确的，但每次 dp[i] 累加时，它都会在错误的模数下减少，因此即使是适度的增长最终也会偏离正确的值。 

解决方法是将模数替换为：```
1000000007
```
