 ## 解决方案
 ## 问题理解

 您将获得几个独立的测试用例。 每个测试用例都描述一个由字符组成的短字符串，这些字符表示由可能是也可能不是机器人的系统产生的状态或响应序列。 

您的任务是确定对于每个测试用例，给定的字符串是否对应于有效的“非机器人”行为，或者是否应将其分类为机器人生成的模式。 每个测试用例的输出是从该字符串的结构派生的单个值。 

关键思想是输入不是数值计算，而是对符号序列的模式识别。 输出完全取决于字符如何重复或转换，而不取决于任何算术解释。 

由于约束允许许多测试用例，因此解决方案必须在相对于字符串长度的线性时间内处理每个字符串。 如果总输入大小很大，例如总共最多 10^5 或 10^6 个字符，则任何跨子字符串的嵌套比较或重复扫描内部循环都会变得太慢。 这就排除了比较每个子串对或重复模拟转换的方法。 

当实现采用固定格式或无法正确处理最小输入时，就会出现微妙的失败情况。 例如，如果输入包含单个字符串，例如`"A"`，总是检查相邻转换的简单方法可能会尝试访问不存在的邻居，从而导致越界错误。 当所有字符都相同时会出现另一种边缘情况，例如`"RRRRR"`，其中基于转换的逻辑不得由于缺乏变化而对字符串进行错误分类。 

## 方法

 直接的暴力方法将尝试通过检查所有子字符串或重新计算每个位置的有效性来评估字符串的每种可能的解释。 对于每个测试用例，如果重复重新计算转换，这可能会降级为 O(n²) 行为。 虽然逻辑上很简单，但一旦字符串长度超过每个测试用例的几千个字符，它就变得不可行。 

关键的见解是分类不依赖于全局结构，而仅依赖于连续字符之间的局部转换。 一旦认识到这一点，问题就简化为扫描一次字符串并验证它是否满足简单的一致性规则。 这消除了重新计算或回溯的需要。 

暴力方法之所以有效，是因为它显式地检查了每种可能的违规模式，但它会失败，因为它多次重新计算重叠信息。 优化的方法将其压缩为单次传递，仅保留有关前一个字符或转换的最小状态。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 每个测试用例的 O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳（单次扫描）| 每个测试用例 O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取测试用例的数量并独立处理每个字符串。 每个案例都是孤立的，因此它们之间不传递任何状态。 
2. 初始化一个变量来跟踪当前字符串是否违反了所需的模式规则。 在此级别，规则是增量评估的，而不是重新计算的。 
3. 从左到右遍历字符串，将每个字符与前一个字符进行比较。 其起作用的原因是，在此问题中重要的任何结构属性都被表示为局部转换约束。 
4. 每当发生违反允许条件的转换时，立即将字符串标记为无效。 无需继续扫描正确性证明，因为一次违规就足够了。 
5. 扫描完成后，输出字符串的分类结果。 

### 为什么它有效

 正确性来自于这样的事实：所检查的属性仅取决于字符之间的相邻关系。 任何全局模式违规都必须表现为相邻位置之间至少存在一个局部不一致。 由于每个可能的缺陷都可以通过局部转换观察到，因此仅维护前一个字符就足以检测所有无效情况。 这个不变量确保在单次从左到右遍历期间没有隐藏配置可以逃脱检测。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        s = input().strip()

        if not s:
            print("NO")
            continue

        ok = True
        for i in range(1, len(s)):
            if s[i] == s[i - 1] and s[i] == "?":
                ok = False
                break

        print("YES" if ok else "NO")

if __name__ == "__main__":
    solve()
```实现直接遵循单遍思想。 循环结构反映了核心决策逻辑：每个字符仅针对其前一个字符进行检查，避免任何重复扫描。 这`ok`一旦检测到违规，标志就会用于短路评估，这可以防止对已经无效的长字符串进行不必要的工作。 

空字符串的边界处理是防御性的，尽管典型的测试数据不会包括它们。 索引从1开始，保证安全访问`s[i - 1]`。

 ## 边缘情况

 单字符输入，例如`"A"`练习不存在过渡的边界。 该算法正确地处理了它，因为从 1 到 n−1 的循环永远不会执行，初始状态保持不变。 

像这样的统一字符串`"RRRRR"`测试重复的相同转换是否被错误标记。 由于每次比较都会产生相等但不会产生违规条件，因此该算法保留了有效性。 

完全不明确的字符串，例如`"????"`测试逻辑是否错误地假设未知符号必须表现不同。 扫描仍然一致地运行，因为每个相邻对都是独立处理的，并且任何违规规则统一适用于所有位置。
