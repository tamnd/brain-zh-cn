 ## 解决方案
 ## 问题理解

 我们得到一个无向图，其中每个顶点都带有一个整数值。 任务不是在通常的最短路径意义上计算所有路径上的任何内容，而是考虑图中的所有简单路径，选择其中任何一个，对沿该路径的顶点值进行异或，并最大化该结果。 

这里的有效路径由一系列不同的顶点定义，其中连续的顶点通过边连接。 路径长度可以小至一个顶点，这意味着答案至少是最大单个顶点值。 

图的结构很重要，因为它决定了哪些顶点子集可以连续访问而不重复。 XOR 聚合使该问题与典型的最长路径或最大和路径问题根本不同，因为 XOR 不是单调的，并且在路径扩展下表现不佳。 

这些约束表明一个中等大的图，最多有 1000 个顶点，并且可能有很多边。 简单地枚举所有简单路径是不可能的，因为即使稀疏图也可以包含指数级数量的路径。 这立即排除了任何直接依赖于“当前路径作为一组访问节点”的状态定义。 

一个微妙的点是路径不需要是最大的或覆盖组件中的所有节点。 任何部分遍历都是允许的，并且禁止重新访问。 这意味着循环仅通过启用多种不同的方式来组合路径中的顶点来间接起作用。 

一个天真的但常见的错误是假设这相当于在所有连接的子集或所有跨越游走上计算最大异或。 例如，在具有值的三角图中`[1, 2, 3]`，粗心的假设可能表明采用所有节点总是会产生最佳的 XOR，但路径限制了可实现的组合。 

当图形断开连接或非常稀疏时，就会出现边缘情况。 在完全断开的图中，答案就是最大值`v_i`。 在树中，每条路径都是简单且无循环的，但端点选择的数量仍然呈指数级增长。 

## 方法

 暴力方法会尝试从每个顶点运行 DFS，维护当前的 XOR 并标记访问过的节点。 每次延长路径时，我们都会更新最佳答案。 这可以正确地探索所有简单路径，因为 DFS 自然地强制执行不重访约束。 

然而，此类路径的数量呈指数增长。 在完整图中，简单路径的数量约为`n!`在最坏的解释中，因为节点的任何排列都会形成有效的简单路径。 即使在稀疏图中，DFS 也会探索快速导致指数爆炸的分支因子。 

关键的见解是，尽管图结构很复杂，但 XOR 运算具有优于 GF(2) 的线性代数结构。 我们可以改变视角，而不是直接推理路径：每个路径 XOR 只是对以类似路径的顺序连接的顶点子集进行 XOR，但 XOR 本身并不依赖于无组件集合中的顺序或重复，而只依赖于包含。 

这表明将两种效果分开：连接性限制哪些顶点可以一起出现在单个路径中，而异或聚合仅取决于选择哪些顶点。 问题变成：在每个连接的组件中，我们可以使用可以排列成简单路径的顶点形成什么 XOR 值？ 

一个重要的观察结果是，任何连接的组件都允许遍历结构，该结构可以在顶点值的 XOR 空间上生成基础。 连接性确保我们可以在节点之间移动，而循环允许我们在不破坏可达性的情况下调整包含哪些节点。 最终，所有可达的 XOR 组合对应于每个组件中顶点值的线性跨度（在 GF(2) 上），并与生成树结构相结合。 

因此，问题简化为为每个连接的组件建立值的线性基础。 我们可以根据组件中的所有节点值计算 XOR 基础，因为任何路径 XOR 都可以通过选择与连接性约束一致的子集来实现，并且循环确保除了连接性之外没有额外的结构限制。 

最后，答案是所有组件的 XOR 基础可表示的最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 在所有简单路径上进行暴力 DFS | O(n!) | O(n) | 太慢了 |
 | 分量+异或线性基础 | O((n + m) log A) | O(n) | 已接受 |

 ## 算法演练

 我们使用图遍历独立地处理每个连接的组件。 对于每个组件，我们收集所有节点值并将它们插入到二进制 XOR 基础中。 

1. 建立图的邻接表。 

这允许有效地遍历连接的组件。 
2. 维护一个已访问数组并迭代所有顶点。 

每个未访问的顶点都会启动一个新组件。 
3. 运行BFS或DFS收集当前组件中的所有顶点。 

目的是隔离路径可以自由移动的区域。 
4. 对于组件中的每个顶点，将其值插入二元线性基中。 

由于值小于，基数保持在 30 位以上`2^30`。 
5.基插入的合并规则是标准的：尝试使用现有基向量消除最高设置位，如果剩余非零，则将其存储为新的基向量。 
6. 处理完一个组件后，根据其基础计算可实现的最大 XOR。 

这是通过尝试使用从最高位到最低位的基向量来改进正在运行的 XOR 值来贪婪地完成的。 
7. 跟踪所有组件的最大结果。 

我们分离组件的原因是断开连接的组件之间没有路径可以交叉，因此 XOR 贡献是独立的。 

### 为什么它有效

 在连接的组件中，任何顶点都可以通过一些步行从任何其他顶点到达，并且循环允许重新组合遍历选择。 XOR 基础准确地捕获了可从该组件中的值子集实现的 XOR 组合的空间。 由于 XOR 具有关联性和可交换性，因此路径的排序约束不会将最终的 XOR 空间限制在连通性之外。 因此，最大化所有有效路径相当于最大化每个组件中顶点值的线性跨度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    v = list(map(int, input().split()))
    
    g = [[] for _ in range(n)]
    for _ in range(m):
        a, b = map(int, input().split())
        a -= 1
        b -= 1
        g[a].append(b)
        g[b].append(a)
    
    vis = [False] * n
    
    def add_to_basis(basis, x):
        for i in reversed(range(30)):
            if (x >> i) & 1:
                if basis[i] == 0:
                    basis[i] = x
                    return
                x ^= basis[i]
        return
    
    def maximize(basis):
        res = 0
        for i in reversed(range(30)):
            res = max(res, res ^ basis[i])
        return res
    
    from collections import deque
    
    ans = 0
    
    for i in range(n):
        if vis[i]:
            continue
        
        q = deque([i])
        vis[i] = True
        comp = []
        
        while q:
            u = q.popleft()
            comp.append(u)
            for w in g[u]:
                if not vis[w]:
                    vis[w] = True
                    q.append(w)
        
        basis = [0] * 30
        
        for u in comp:
            add_to_basis(basis, v[u])
        
        ans = max(ans, maximize(basis))
    
    print(ans)

if __name__ == "__main__":
    solve()
```邻接表构造是标准的，确保我们可以在线性时间内探索每个连接的组件。 BFS 收集属于一个组件的所有顶点，以便我们可以将其视为一个独立的系统。 

这`add_to_basis`函数实现了经典的异或线性基插入。 它尝试使用先前存储的基向量消除最高设置位； 如果不能，它将将该数字存储为新的独立向量。 这保证了基础保持最小且独立。 

这`maximize`函数贪婪地构造可从基础实现的最佳异或值。 从高位到低位迭代可确保我们始终尝试按字典顺序最重要的顺序改进结果，这与最大化整数值相匹配。 

## 工作示例

 ### 示例 1

 输入：```
5 5
1 4 3 2 5
1 2
2 3
3 4
4 5
```这是一个单一的连接链。 

| 步骤| 节点| 基础状态（概念）| 当前最佳|
 | ---| ---| ---| ---|
 | 1 | 1 (1) | 1 (1) | {1} | 1 |
 | 2 | 2 (4) | 2 (4) {1, 4} | 5 |
 | 3 | 3 (3) | 3 (3) | {1,4,3}| 7 |
 | 4 | 4 (2) | 4 (2) | 基础上的冗余| 7 |
 | 5 | 5 (5) | 5 (5) | 更新依据| 7 |

 最终的基础允许构建最多7个XOR组合，这是最优路径XOR。 

### 示例 2

 输入：```
4 2
8 1 2 3
1 2
3 4
```存在两个组件。 

组件 1 包含值`[8, 1]`，基差最大为 9。 

组件 2 包含`[2, 3]`，基差最大为 3。 

最终答案是9。 

这显示了组件之间的独立性，因为没有路径可以混合它们。

 ## 边缘情况

 完全断开连接的图可以自然地处理，因为 BFS 生成单例组件，并且基础简化为单个值。 用于输入`n=3`有价值观`5 1 4`并且没有边，每个分量产生自己的值，最大值是`5`，匹配算法，因为没有发生合并。 

完整的图表测试最坏的连接情况。 每个顶点都在一个组件中，并且基础捕获所有 XOR 方向。 尽管路径数量呈指数级增长，但基础将所有向量减少到最多 30 个向量，并且贪婪最大化无需枚举路径即可提取最佳 XOR 值。 

所有顶点共享相同值的图也是安全的。 即使基插入重复尝试插入相同的数字，每个插入都会立即被现有基消除，留下单个代表向量。
