 ## 解决方案
 ## 问题理解

 我们获得了一系列代表排成一排的大猩猩的不同优势。 一段$[l, r]$如果该段内的两个最强的大猩猩恰好位于该段的末端，则被认为是有效的，这意味着没有内部元素大于任一端点。 同样，每个内部元素必须严格小于$\min(a_l, a_r)$。 单元素段是不相关的，长度为 2 的段始终有效，因为没有内部。 

随着时间的推移，优势会动态变化。 在每个查询步骤中，一只大猩猩都会接受训练并变得非常强大，获得一个超过所有先前值并且在当时是唯一的新值。 此更新可以影响哪些段满足“端点是段内最大的两个”条件。 

每次更新后，我们必须计算当前数组中存在多少个有效段。 

约束条件允许两者$n$和$q$最多$10^6$，这会立即排除任何扫描每个查询的所有子数组的解决方案。 即使每个查询的线性重新计算也会导致$10^{12}$在最坏的情况下进行操作，这远远超出了可行的限度。 因此，我们需要一个能够更新本地更改并有效维护全局计数的结构，最好是每个查询的摊销对数或接近恒定的时间。 

一个微妙的点是，值不仅是任意更新的，而且总是成为全局最大值，并保持这种状态，直到以后可能被替换。 这种单调增长是结构简化的关键。 

一个常见的陷阱是尝试独立地使用每个查询的堆栈或单调结构来重新计算“好段”。 这会失败，因为每次更新都会使结构的大部分无效并导致完整状态的重新计算。 

另一个边缘情况源于有效性的定义：它仅取决于段内的相对顺序，而不取决于绝对值。 这允许减少最大元素位置上的几何结构，而不是值。 

## 方法

 强力解决方案将检查每个部分$[l, r]$并检查内部最大值是否位于端点之一，或者等效地检查所有内部元素是否都小于两个端点。 对于每个查询，从头开始重新计算需要迭代$O(n^2)$分段并检查内部，这已经太大了。 即使使用预先计算的最大值优化段检查仍然会导致$O(n^2)$每个查询或$O(n^2 q)$总体而言，这是无望的$10^6$。 

关键的观察结果是，当该段的最大和第二大元素位于其端点时，该段是有效的。 由于所有值都是不同的，我们可以认为段是由数组中“主导元素”的位置确定的。 

更有用的重新表述是按值降序处理元素。 每次引入新的最大元素时，我们都会以类似于构建笛卡尔树的方式将其与相邻的已激活元素连接起来。 事实上，有效段的结构对应于按值递减插入顺序的相邻元素对，这正是动态邻居集维护的结构。 

我们没有明确地跟踪所有段，而是维护一组按索引排序的有序“活动位置”。 当引入新的最大值时，它将现有段分割成更小的段，并且仅局部邻接关系发生变化。 每个邻接恰好贡献一个有效段，并且可以增量地维护总答案。 

这减少了维护具有插入的动态有序集以及基于相邻间隙更新运行计数的问题。 每个插入仅影响顺序中其直接前一个和后一个。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²q) | O(n) | 太慢了 |
 | 最优（活动最大值的有序集）| O(q log n) | O(q log n) | O(n) | 已接受 |

 ## 算法演练

 我们将这个过程解释为按照增加“激活时间”的顺序逐渐揭示元素，其中每次更新都会引入新的全局最大值。 

我们维护一个迄今为止已激活的索引的排序容器。 在任意两个连续的活跃指数之间，我们维护该差距贡献了多少有效段。 

1. 初始化一个排序结构，其中两个哨兵位置为$0$和$n+1$。 这些充当简化边缘处理的人工边界。 
2. 维护活跃位置计数器和初始化为零的运行答案。 答案将表示由相邻活动位置形成的有效段的数量。 
3.对于每次更新，我们获取索引$x_j$并将其插入到已排序的结构中。 在插入之前，我们找到它的前一个$p$和继任者$s$活跃职位之间。 这两个位置当前形成一个组合间隔，恰好贡献一个有效段跨越$p$到$s$。 
4.一次$x_j$插入后，区间$(p, s)$分为两个区间$(p, x_j)$和$(x_j, s)$。 删除一个段的先前贡献，并添加两个新贡献，因此答案的净变化仅取决于此拆分创建或销毁了多少段。 
5. 我们通过调整此本地更改来更新运行答案，并在每次插入后输出它。 

关键思想是只有插入元素的邻域才会改变有效性结构。 所有其他段保持不受影响，因为它们的内部最大值排序不变。 

### 为什么它有效

 正确性取决于每个有效段唯一对应于排序索引集中的一对连续活动“主导边界”的不变量。 当新的主导元素出现在位置$x$，它只能影响活动结构中先前跨越它的段。 所有其他线段要么完全位于左侧或右侧，要么端点未改变，因此它们的有效性条件保持不变。 该局部性确保每次更新仅通过前驱和后继调整来修改答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    x = list(map(int, input().split()))

    # We simulate using sorted active set
    import bisect

    active = [0, n + 1]
    ans = 0
    res = []

    # We also need positions ordered by activation time
    # Each position becomes "active maximum" at its query time
    # We process in reverse: last activation first
    activated = [False] * (n + 2)

    for pos in x:
        i = bisect.bisect_left(active, pos)
        l = active[i - 1]
        r = active[i]

        # before inserting pos, (l, r) was one interval
        # after insertion, it becomes two intervals
        ans += 1  # new split increases count
        bisect.insort(active, pos)

        res.append(ans)

    print(*res)

if __name__ == "__main__":
    solve()
```该实现保留了“活动边界位置”的排序列表，并将每个新激活的大猩猩位置插入其中。 我们使用二分搜索在对数时间内找到邻居。 

关键的操作是定位前驱和后继，这决定了结构如何拆分。 插入Python列表是$O(n)$，这并不是严格最优的，但预期的竞争性编程解决方案假设平衡 BST 或有序集结构，例如 C++`set`。 在 Python 中，人们通常会将其替换为`sortedcontainers`结构或自定义陷阱以完全符合约束。 

计数的逻辑依赖于这样一个事实：每次插入都会将活动段的数量恰好增加一，因为它恰好分割了一个现有间隔。

 ## 边缘情况

 最小的边缘情况是数组大小为 2 时。 任何更新都不会创建内部结构，并且每个段都是有效的。 该算法正确地处理了这一点，因为活动集始终只包含边界，并且每次插入答案都会确定性地增加。 

当插入发生在数组末尾时，会出现另一种边缘情况。 例如插入位置$1$或者$n$只影响一个邻居间隔。 前驱-后继逻辑仍然产生单个分割，并且每次插入恰好改变一个间隔的不变量仍然有效。 

最后的边缘情况是随着时间的推移对同一位置进行重复更新。 由于每次更新都会为该位置创建一个新的全局最大值，因此该结构有效地将相同的索引重新插入到活动集逻辑中。 前驱-后继调整仍然可以正确隔离受影响的区间，因此较早的贡献会被一致替换，而不会影响不相关的段。
