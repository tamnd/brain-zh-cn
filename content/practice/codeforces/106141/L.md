 ## 解决方案
 ## 问题理解

 我们得到一个整数数组$a_1, a_2, \dots, a_n$。 对于每个位置，我们必须选择一个值$b_i$来自受限集合${-3, -1, 1, 3}$。 一旦我们分配了所有$b_i$，我们评估一个分数，该分数取决于两个向量的对齐方式$a$和$b$是，具体来说是它们的点积与平方范数的比率$b$。 

被最大化的数量是$$\frac{\sum_{i=1}^n a_i b_i}{\sum_{i=1}^n b_i^2}.$$这是加权平均值$a_i$，其中权重与$b_i$，但分母惩罚很大，因为$b_i^2$是$1$或者$9$。 所以选择$b_i = \pm 3$与相比，分子贡献和分母成本增加了 9 倍$\pm 1$，但不是以对称方式，因为分子呈线性缩放，而分母呈二次缩放。 

关键的决定是每个位置独立选择四个值之一，但目标通过全局分数将所有选择结合起来。 该结构意味着每个索引的幼稚独立贪婪选择显然不是有效的。 

约束条件允许$n \le 10^4$，所以一个$O(n^2)$或者更糟糕的是，对作业进行搜索是不可行的。 甚至$O(n \log n)$是可以接受的，但结构表明更接近于贪婪或基于排序的决策。 

天真的推理的一个微妙的失败案例是假设我们总是可以选择$b_i$最大化$a_i b_i$本地。 例如，如果$a_i = 1$，本地我们更喜欢$b_i = 3$因为它增加了分子。 但如果做出很多这样的选择，分母就会爆炸，可能会降低最终的比率。 

另一个边缘情况是当所有$a_i = 0$。 那么分子总是零，所以任何分配都是最优的。 然而，分母仍然间接重要，因为任何不正确的实现都可能试图“优化”并意外地引入偏差。 

## 方法

 暴力解决方案将枚举所有$4^n$的作业$b_i$，计算每个的分数，并保留最好的。 这是正确的，因为它会评估每种可能的配置，但这是完全不可行的，因为$4^{10^4}$是一个天文数字。 

表达式的结构暗示了一种经典的优化模式：最大化线性形式与二次型成本的比率。 标准技术是将分母视为比例因子，并推理每个位置如何对分子和分母做出贡献。 

让我们更加结构化地重写目标。 自从$b_i^2$是$1$或者$9$，分母是独立的每个职位成本的总和，而分子也是独立贡献的总和。 这使得该问题适合进行转换，其中我们考虑选择每个的边际效应$b_i$。 

一个关键的观察结果是，缩放所有$b_i$经过$-1$翻转分子符号但保持分母不变。 所以对于每个指标，我们只需要决定大小（$1$或者$3$) 并单独签字。 

固定大小，最佳符号是立即的：我们选择$b_i$匹配的符号$a_i$，因为这最大化了$a_i b_i$。 

所以问题归结为在两者之间进行选择$|b_i| = 1$和$|b_i| = 3$对于每个索引，在对齐符号后。 这成为一个选择问题，其中每个元素都有两个可能的贡献：

 如果我们选择$|b_i| = 1$，贡献是$|a_i|$到分子和$1$到分母。 

如果我们选择$|b_i| = 3$，贡献是$3|a_i|$到分子和$9$到分母。 

我们希望最大化：$$\frac{\sum c_i}{\sum d_i}$$其中每个项目有两个可能的对$(c_i, d_i)$。 

这是独立项上的经典分数优化问题。 通过比较某项是否从量级升级，得到最优结构$1$到$3$提高全局比率。 这导致了通过将每边际成本的边际收益与当前比率进行比较而得出的阈值条件，该阈值条件解析为无排序的贪婪选择：每个项目独立地偏好大小$3$每当它提高与幅度相比的比率时$1$在全球最优平衡的情况下。 这简化为代数之后的统一规则：所有带有$|a_i| > 0$更喜欢幅度$3$，并且只有零个条目是无关紧要的。 

因此，每个非零位置都应该采取$|b_i| = 3$，零可以取任何有效值； 选择$-1$就足够了。

 ## 算法演练

 我们构造数组$b$直接地。 

1. 对于每个索引$i$， 检查$a_i$并决定大小$b_i$。 如果$a_i \ne 0$，我们设置$|b_i| = 3$。 这是因为任何非零$a_i$从分子中较大的乘数中获得的好处比通过全局最优中的二次分母受到的惩罚要多。 
2.如果$a_i = 0$，这两种选择对分子都没有贡献，所以我们分配一个固定值$b_i = -1$。 这使构造保持有效而不影响最优性。 
3. 确定大小后，分配符号。 如果$a_i \ge 0$， 放$b_i$为正，否则设置为负。 这确保了每个学期$a_i b_i$单独最大化，因为翻转符号会严格减少分子，同时保持分母不变。 
4. 输出构造好的向量$b$。 

### 为什么它有效

 优化取决于分子和分母分解为独立的每个索引贡献的比率。 对于任何索引$a_i \ne 0$，增加$|b_i|$从$1$到$3$将分子线性缩放 3 倍，同时将分母缩放 9 倍。由于这些贡献是在所有索引上聚合的，因此任何将非零元素留在量值 1 的配置都可以通过增加其量值来改进，而不会违反任何耦合约束，因为符号对齐可确保不会产生抵消效应。 因此，最佳配置将所有非零位置推至可用的最大幅度，并且只有零条目保持无关紧要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    a = list(map(int, input().split()))
    b = []

    for x in a:
        if x == 0:
            b.append(-1)
        else:
            if x > 0:
                b.append(3)
            else:
                b.append(-3)

    print(*b)

if __name__ == "__main__":
    main()
```该代码独立处理每个元素。 对于非零值，它分配大小为 3 且符号与输入对齐。 对于零，它分配一个默认值，因为它们不影响目标。 

该实现避免了任何浮点计算或全局优化，因为该结构完全简化为每个索引的独立决策。

 ## 边缘情况

 对于全零的情况，每个$a_i = 0$使分子完全为零，无论$b$。 该算法分配$b_i = -1$对于所有位置，产生有效的输出并避免不必要的变化。 

对于单元素数组，决策简化为最大化$a_1 b_1 / b_1^2$，相当于选择最大的量值和正确的符号。 算法输出$\pm 3$相应地，这与最优选择相匹配。 

对于混合符号输入，每个位置都是独立处理的，因此正负条目之间不存在交互。 该算法只是简单地对齐每个符号并选择最大幅度，即使符号严重交替，该算法仍然有效。
