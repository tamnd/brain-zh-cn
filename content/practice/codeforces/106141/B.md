 ## 解决方案
 ## 问题理解

 我们有一排独立的转子。 每个转子都存储一个环绕的值，因此转子`i`其行为类似于圆上的数字`0`到`m_i - 1`。 系统从每个转子为零开始，我们希望达到目标配置`b`。 

唯一允许的操作是选择一个连续的段`[l, r]`并将该段中的每个转子向前旋转一步，并根据其自身的模量进行环绕。 每个操作均等地影响所有选定的位置，但每个位置的模量不同，因此相同的操作在每个转子上具有不同的“有效行为”。 

任务是计算将全零状态转换为目标数组所需的此类段旋转的最小数量。 

关键的困难在于操作重叠：应用一个段会同时增加多个位置，并且不同的段可能会以复杂的方式重叠。 尝试模拟所有可能的段操作序列的简单方法很快就变得不可行，因为操作数量和可能的段数量都呈二次方增长。`n`。 

约束足够小，以至于`O(n^2)`或者`O(n^3)`风格动态规划对于每个测试集都是合理的，但我们必须小心，因为总和`n`跨测试只有 500，这意味着我们应该大致目标`O(n^2)`或者总体上更好。 任何显式枚举所有段操作序列的方法都是远远不可行的。 

一个微妙的边缘情况是目标值强制“携带”不同方向。 例如，如果一个转子需要大量以一个小值为模的增量，而下一个转子则需要几乎为零的增量，那么看起来我们应该大量构建分段，但实际上我们总是可以通过让值环绕来“撤消”多余的增量，因为只有模约束很重要。 总是增加直接匹配目标的天真贪婪会过度计算操作。 

## 方法

 直接建模是考虑每个位置总体增加了多少次。 让`x_i`是覆盖位置的操作总数`i`。 然后是position处的最终值`i`简直就是`x_i mod m_i`，因为每次操作都会将其加一。 

所以我们需要选择一个整数序列`x`这样`x_i ≡ b_i (mod m_i)`为所有人`i`，我们想要构建`x`作为使用尽可能少的段的段增量的总和。 

现在解释一下段操作的作用。 每个操作都会增加一个值`1`在连续范围内，所以`x_i`是覆盖索引的段数`i`。 如果我们看看差异`x_i - x_{i-1}`，每次这个差异为正时，它对应于在位置开始新的段`i`，而负差异对应于段结束。 

从这个角度来看，操作数正是我们启动一个段的总次数，等于正增量的总和：$$\text{cost} = \sum_{i=1}^{n} \max(0, x_i - x_{i-1})$$和`x_0 = 0`。 

问题变成选择有效的代表`x_i = b_i + k_i m_i`来最小化这个成本。 每个位置独立地允许有效值的无限算术级数，并且成本耦合连续的位置。 

暴力解决方案将尝试每个选项的所有有效选择`x_i`，形成分层最短路径，每层最多 500 个选择。 这给出了关于`500 * 500 * 500`最坏情况下的转换太慢了。 

关键的观察是我们永远不会从制造中受益`x_i`不必要的大。 如果我们增加`x_i`除了需要的范围之外，我们可以创建额外的段起始点，这可以通过选择同一残基类别的较小代表来避免。 这允许贪婪的转换：在每个位置，我们选择有效值`x_i`尽可能大但不超过`x_{i-1}`。 只有当不可能时，我们才会转向上面的最小有效值`x_{i-1}`。 

这将问题简化为一次从左到右的传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 DP 全部有效`x_i`价值观 | 每次测试 O(n·m²) | O(米) | 太慢了 |
 | 贪婪残留追踪 | 每次测试 O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时保持可实现的最佳值`x_{i-1}`。 

1. 初始化`x_0 = 0`。 这表示在第一个位置之前没有应用任何增量。 
2. 对于每个位置`i`，考虑残基类别`b_i (mod m_i)`。 每个有效`x_i`必须采用以下形式`b_i + k * m_i`。 
3. 尽量选择最大有效值`x_i`仍然小于或等于`x_{i-1}`。 这是通过计算有多少个完整的步长来完成的`m_i`适合之间`b_i`和`x_{i-1}`，然后相应地后退。 如果存在这样的值，选择它可以避免增加`x_i`，这避免了在这个位置支付成本。 
4. 如果即使是最小的有效值`b_i`大于`x_{i-1}`，那么我们就被迫增加。 在这种情况下，设置`x_i = b_i`并支付增加的费用`x_i - x_{i-1}`。 
5. 随时累计总成本`x_i > x_{i-1}`，并更新`x_{i-1} = x_i`用于下一次迭代。 

### 为什么它有效

 该过程构造一个序列`x`满足所有模块化约束，同时最小化连续位置之间的向上转换的数量。 任何解决方案都可以转化，以便在每一步`i`，所选择的`x_i`减少模数`m_i`在相同的残留类别内而不破坏可行性，因为只有`x_i mod m_i`关系到正确性。 一旦我们将注意力限制在最大可行的范围内`x_i ≤ x_{i-1}`，延迟增加永远没有帮助，因为任何未来的需求都可以通过稍后在需要的确切位置增加来满足。 这意味着推迟增长会严格减少或保留段开始的总数，因此每一步的贪婪选择都是全局最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        m = list(map(int, input().split()))
        b = list(map(int, input().split()))

        x_prev = 0
        ans = 0

        for i in range(n):
            mi = m[i]
            bi = b[i]
            r = bi

            if x_prev >= r:
                k = (x_prev - r) // mi
                x_i = r + k * mi
            else:
                x_i = r

            if x_i > x_prev:
                ans += x_i - x_prev

            x_prev = x_i

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码保持了当前最好的可能`x_prev`，它表示前一个转子已有效增加了多少次。 对于每个新转子，它会计算不超过该值的残留类别的最佳代表。 划分步骤`(x_prev - r) // mi`是直接跳转到算术级数中最佳可行值的关键压缩步骤。 

只有当我们被迫增加时才计算成本`x_i`，它完全对应于开始新的段。

 ## 边缘情况

 一个关键的边缘情况是当序列在构造中反复迫使减少时`x_i`。 例如，如果某个位置允许的残基类别远小于先前的值，则算法会选择以下最大的可行代表`x_{i-1}`，生产减少，成本为零。 这可以避免对段开始进行过度计数。 

另一个边缘情况是当`b_i`总是在附近`m_i - 1`。 在这种情况下，算法会重复向上跳转，但只有在绝对必要时，因为只有当下面不存在有效代表时才会触发每次跳转`x_{i-1}`。 这可以防止意外创建额外的段。 

最后，当没有机会跨位置重用段时，单元素情况可确保正确性。 该算法正确地将每个此类增加视为新的段开始，匹配最佳结构。
