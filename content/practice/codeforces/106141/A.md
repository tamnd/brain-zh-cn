 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$网格中的一些单元格包含石头，并且恰好有两个特殊单元格包含香蕉。 网格不仅仅是一张静态图片：石头可以移动，但最多只能移动$k$石头可能会被重新安置，并且$k \le 5$，这是关键的限制。 

在我们选择要移动的石头之后，每个移动的石头都会被重新定位到一个空单元格中，然后这些移动后存在的所有石头都会“燃烧”它们的单元格，从而有效地从网格中删除这些单元格。 剩下的是由共享边缘连接的完整细胞的集合。 网格可以分裂成多个连接的组件，因为烧毁的电池充当孔。 

目标是确保两个香蕉电池在燃烧后最终位于剩余网格的同一连接组件中。 我们最多被允许移动$k$石头，并且必须构造这样的配置或报告不可能性。 

连接是 4 向的，因此对角邻接没有帮助。 这很重要，因为即使在看似密集的网格中，它也会产生许多微妙的断开连接。 

输入大小为$n \le 1000$，所以网格最多有$10^6$细胞。 然而，$k \le 5$非常小。 这立即表明我们不是在搜索网格状态，而是在搜索极少数的“关键修改”。 

一个天真的误解是认为我们需要在任意棋子移动后模拟连通性。 这将涉及重复重新计算连接的组件，这已经是边缘问题，但不是真正的问题。 真正的困难在于石头可以_阻挡路径_，而我们只能稍微调整阻挡结构。 

一个关键的边缘情况是，即使没有任何移动，香蕉也已经断开连接。 例如，如果所有中间单元都被封锁在一条笔直的走廊中，并且我们无法打开它，因为$k=0$，答案立即是“否”。 

另一个微妙的情况是当香蕉位于相邻的对角单元格中时：```
1 0
0 1
```尽管它们“视觉上接触”，但它们并没有联系。 这提醒我们连接性是严格正交的。 

最后，一个重要的边缘条件是当移动的石头引入新的孔来隔离香蕉时。 只尝试“连接”地区而不确保全球连通性的粗心策略很容易破坏其他地方的结构。 

## 方法

 蛮力方法会尝试模拟所有向上移动的方式$k$石头，然后重新计算香蕉是否在生成的网格中连接。 对于每个选择最多$k$石头之间最多$10^6$位置，我们也会选择目标单元格。 即使忽略目的地的复杂性，仅选择宝石就已经是$\binom{10^6}{k}$，即使对于$k=2$。 

即使每个配置进行基于 BFS 的连接检查也会花费成本$O(n^2)$，所以这种方法立即崩溃。 

重要的观察是$k$很小，所以我们应该考虑有多少石头实际上与两个香蕉位置之间的连接相关。 网格中香蕉之间的路径仅取决于狭窄的单元走廊； 如果该走廊被堵塞，则它会被沿某些切口的少量关键石头堵塞。 因为我们最多可以移动五块石头，所以我们实际上是在问是否存在一种方法来消除两点之间的小分离结构。 

这将问题转化为对阻塞网格的有界“修复”：我们尝试在网格中找到从一根香蕉到另一根香蕉的路径，在该路径中我们可以转换最多$k$通过移开石头将细胞阻塞为空细胞。 每个移动的石头都会释放一个单元格并占据另一个单元格，但由于我们控制放置并避免香蕉和重复使用的单元格，因此只要我们可以将石头放置在不相关的地方，我们就可以将操作视为“翻转一个被阻止的单元格以释放”。 

因此，本质的归约是：通过最多转换来确定我们是否可以连接两个香蕉电池$k$阻塞细胞变成游离细胞。 

这是具有加权障碍解释的网格中的经典最短路径，其中移动到空单元的成本为 0，进入石单元的成本为 1（因为我们必须花费一次移动来清除它）。 我们正在有效地计算两个香蕉位置之间的 0-1 BFS 距离。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解动作| 指数| 大| 太慢了 |
 | 网格上 0-1 BFS |$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们将每个单元建模为网格图中的一个节点。 穿过一个空的单元格的成本为 0，而穿过一个石单元格的成本为 1，因为我们必须花费一次允许的移动来重新定位该石头并使该单元格可穿过。 

我们使用 0-1 BFS 计算从第一个香蕉到第二个香蕉的最小成本路径。 

1. 为所有单元格初始化一个无穷大的距离矩阵，除了起始香蕉单元格设置为 0。这表示我们需要移除零个石头才能站在起始点。 
2. 使用双端队列进行0-1 BFS。 将起始单元推入双端队列。 
3. 当双端队列不为空时，从前面弹出并放松其四个邻居。 转移成本取决于相邻单元格最初是否包含石头。 
4. 如果邻居是空的，我们尝试以相同的成本放松并将其推到双端队列的前面。 如果它包含石头，我们以成本+1放松并将其推到后面。 此顺序可确保较早处理移除石头较少的路径。 
5. 继续，直到处理完所有可到达的单元格或到达第二个香蕉。 
6. 如果第二个香蕉的计算距离超过$k$，输出“否”。 
7. 否则，使用父指针重建从第二个香蕉回到第一个香蕉的路径。 
8. 沿着这条重建的路径，识别造成成本 1 转换的单元。 这些是必须移动的石细胞。 
9. 对于每个这样的石单元，为其分配一个当前为空且不属于路径或禁止位置的目标单元。 自从$k \le 5$，我们可以贪婪地将移动的石头放置在任何足够远的空单元格中。 
10.输出动作列表。 

### 为什么它有效

 BFS 状态精确地编码了为在香蕉之间创建有效走廊而必须移除的最小石头数量。 任何有效的最终配置都对应于一条路径，其中每个经过的石单元最多被清除一次。 因为每次移动只能清除一个障碍物，并且我们从不重复使用移动的石头，所以 BFS 中的成本与操作数量相匹配。 0-1 BFS 的最优性保证了如果任意解存在于$k$移动时，计算的距离不会超过$k$，重建的路径直接识别出必须解决哪些障碍。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

INF = 10**9

def solve():
    n, k = map(int, input().split())
    x1, y1 = map(int, input().split())
    x2, y2 = map(int, input().split())
    x1 -= 1; y1 -= 1
    x2 -= 1; y2 -= 1

    grid = [input().strip() for _ in range(n)]

    dist = [[INF] * n for _ in range(n)]
    parent = [[None] * n for _ in range(n)]

    dq = deque()
    dist[x1][y1] = 0
    dq.append((x1, y1))

    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while dq:
        x, y = dq.popleft()
        for dx, dy in dirs:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < n:
                cost = dist[x][y] + (grid[nx][ny] == '1')
                if cost < dist[nx][ny]:
                    dist[nx][ny] = cost
                    parent[nx][ny] = (x, y)
                    if grid[nx][ny] == '1':
                        dq.append((nx, ny))
                    else:
                        dq.appendleft((nx, ny))

    if dist[x2][y2] > k:
        print("No")
        return

    path = []
    cx, cy = x2, y2
    while (cx, cy) != (x1, y1):
        path.append((cx, cy))
        cx, cy = parent[cx][cy]
    path.append((x1, y1))
    path.reverse()

    path_set = set(path)

    moves = []
    used = [[False]*n for _ in range(n)]

    # collect stones on path
    stones = []
    for i in range(n):
        for j in range(n):
            if grid[i][j] == '1' and (i, j) in path_set:
                stones.append((i, j))

    # find free cells for placement
    free = []
    for i in range(n):
        for j in range(n):
            if grid[i][j] == '0' and (i, j) not in path_set:
                free.append((i, j))

    fi = 0
    for sx, sy in stones:
        fx, fy = sx, sy
        px, py = free[fi]
        fi += 1
        moves.append((fx+1, fy+1, px+1, py+1))

    print("Yes")
    print(len(moves))
    for a, b, c, d in moves:
        print(a, b, c, d)

if __name__ == "__main__":
    solve()
```BFS 部分是核心正确性引擎。 步入石头和步入空单元之间的成本差异精确地编码在双端队列排序中，这确保了最短路径的正确性，而无需优先级队列。 

重建步骤依赖于父指针，这保证了我们检索实际可行的走廊，而不仅仅是距离值。 一旦我们知道哪些单元位于走廊上并且是石头，我们就将它们视为必须移除的确切障碍。 

放置策略使用不在路径上的任意空闲单元。 自从$k \le 5$，只要我们选择不同的空单元格，总是有足够的灵活性来避免冲突。

| 步骤| 细胞| 成本| 行动|
 | ---| ---| ---| ---|
 | 1 | (3,1) | 0 | 开始 |

| 步骤| 细胞| 成本| 行动|

 ## 边缘情况

 一种重要的情况是，当香蕉已经位于同一个连接组件中而忽略石头时。 在这种情况下，BFS 距离为零，因为我们从未踏入石室。 该算法在零移动的情况下正确输出“是”，因为不需要修改并且连接性已经得到满足。 

另一种情况是，一块石头堵塞了两个区域之间唯一的走廊。 BFS 路径必然经过该单元，从而产生成本 1。重建步骤确保我们准确识别该阻塞单元，并且由于$k \ge 1$，我们可以将其重新定位到其他地方而不破坏路径。 

当存在多个最短路径时，就会出现一种微妙的情况。 BFS 可以选择其中任何一个，但都具有最小的成本。 由于所有有效的最短路径都对应于有效的最小障碍物集，因此任何重建的路径都会产生一组正确的要移动的石头。
