 ## 解决方案
 ## 问题理解

 我们正在模拟将小的固定形状放入大网格中的过程，其中网格的行为就像一个物理系统：单元格被填充，完全填充时整行消失，并且删除行上方的所有内容都向下折叠。 

每个操作都会给我们一个恰好占据三个连续列的标记。 在每一列中，图章的垂直高度在 1 到 3 个单元格之间，并且图章的顶部边界跨三列对齐，因此形状就像宽度为 3、边界高度为 3 的倒置直方图。 

当我们在柱子上贴上邮票时$x, x+1, x+2$，我们不能任意选择它的垂直位置。 相反，我们必须放置它，使其接触三列中至少一列中填充结构的当前表面，同时决不与其他列中现有的填充单元相交。 该系统还强制执行类似重力的行为：每当一行完全填满时，它就会消失，并且其上方的所有单元格都会下降一行。 

该过程在第一个印记处停止，该印记不能在不超出网格底部边界的情况下放置。 

输出是第一个失败标记的索引，或者$-1$如果每张邮票都能贴成功的话。 

约束条件很大：最多$2 \cdot 10^5$行、列和操作。 这立即排除了任何显式维护完整网格或逐列扫描每个操作的模拟。 任何解决方案都必须在接近恒定或对数的时间内处理每个印记，理想情况下仅维护有关列高度的压缩结构信息。 

一个微妙的困难来自于行删除。 一个简单的解释是跟踪完整的网格并在满时删除行，但这会引入级联移位。 关键的复杂性是每列中的“高度”不是独立的； 删除全局耦合所有列。 

一些简单方法的失败案例值得强调。 

第一种故障模式是直接电网模拟。 认为$n = 10^5$每个图章影响三列。 经过几十万次操作后，重复扫描整行并移动网格变得二次。 更糟糕的是，删除会导致每列的索引发生变化，从而破坏了简单的绝对索引。 

第二种失败模式是维持每列的高度而不考虑行删除。 如果一列变满并触发行删除，则所有列中的有效高度都会以非局部方式减小。 忽略这一点会导致后续操作中放置高度不正确。 

第三个微妙的失败来自于将放置位置简单地视为三个柱顶部的最大值。 该规则要求至少接触一个柱表面，同时避免其他柱表面重叠，因此垂直对齐受到柱高度之间的相对差异的限制，而不仅仅是单个最大值。 

## 方法

 蛮力的想法很简单：显式模拟网格，通过向下扫描来放置每个标记，直到它符合约束条件，标记占用的单元格，然后扫描整个行集以检测完整行并将其删除。 这是正确的，因为它严格遵守字面规则。 

但是，每个印章可能需要触摸最多三列，并且每个位置可能需要垂直扫描最多$O(n)$职位。 行删除可能会扫描所有$m$列来检查完整性，然后可能进行移动$O(nm)$细胞。 和$q$最多$2 \cdot 10^5$，这会导致最坏情况的复杂性约为$O(qnm)$，这远远超出了可行性。 

关键的观察是网格永远不需要显式表示。 对于每列来说，重要的只是当前的“表面轮廓”，即占用最高的单元格。 行删除不需要物理移动单元格； 它们只会均匀地降低所有列的有效高度。 这建议维持一个动态结构来跟踪列高度并支持全局压缩事件。 

关键的结构见解是每个印章仅与其三列中的顶部边界相互作用。 较低行的内部配置是无关紧要的，除非它确定一行何时已满。 完整行事件对应于所有列达到或超过阈值的级别，可以使用聚合跟踪而不是显式网格来检测该阈值。 

这减少了在列高度上维护多重集或线段树的问题，支持范围更新（标记放置）和全局标准化（行删除），同时检测标准化后是否有任何列超出网格高度约束。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力网格模拟|$O(qnm)$|$O(nm)$| 太慢了|
 | 高度压缩+段结构|$O(q \log m)$|$O(m)$| 已接受 |

 ## 算法演练

 我们维护当前列高度的数组，但这些高度保存在压缩坐标系中，其中全局行删除表示为单个偏移量而不是显式删除。 

每个图章都会修改三个连续的列，并且仅取决于它们当前的相对高度。 

### 步骤

 1.维护一个数组`h[i]`表示当前柱的有效高度$i$，以及全局偏移量`base`表示已删除的完整行数。 真实物理高度是`h[i] + base`。 这种分离可以防止删除后发生昂贵的转移。 
2. 对于列中的每个标记$x, x+1, x+2$，计算这些列中的当前表面高度。 这些决定了印章在接触至少一个柱表面而不与其他表面重叠时的位置。 
3. 印章有三个柱高$y_1, y_2, y_3$，这意味着其顶部形状相对于其基础位置是固定的。 为了合法地放置它，我们计算最小放置高度，使其至少在一列中接触当前表面。 
4. 对于三列中的每一列，请考虑对齐图章，使其顶部正好与该列中当前高度的上方对齐。 这给出了最多三个候选基地位置。 我们选择有效候选者中的最大值，以确保任何列中都不会重叠。 
5. 确定放置高度后，根据印章的形状增加相关的三个列的高度，并考虑到每列添加等于其印章高度的垂直贡献。 
6.更新后检查是否有列超出$n + base$。 如果是这样，则无法放置该图章，并且我们输出其索引。 
7. 定期检测何时隐式出现整行。 我们不是扫描所有列，而是使用线段树或平衡结构来维护达到每个高度级别的列数。 当所有列达到共同阈值时，我们增加`base`并逻辑地向下压缩所有高度。 

### 为什么它有效

 该算法依赖于以下不变量：在任何时候，未来放置的唯一相关状态是删除完成的行后每列的上封套。 全局偏移干净地捕获行删除，因为每次删除都会在所有列上均匀地删除一个高度单位。 由于印章仅与柱顶部相互作用，因此表面以下的内部结构是无关紧要的。 这确保了每个标记的决策仅取决于当前列最大值，并且可以使用局部比较加上全局标准化来检查所有约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    
    h = [0] * (m + 1)
    base = 0

    for i in range(1, q + 1):
        x, y1, y2, y3 = map(int, input().split())
        cols = (x, x + 1, x + 2)
        ys = (y1, y2, y3)

        cur = []
        for c in cols:
            cur.append(h[c] + base)

        # compute best placement
        best = -10**18
        for j in range(3):
            best = max(best, cur[j] + ys[j])

        # check feasibility: must not exceed bottom boundary after placement
        # interpret new height
        new_height = best
        if new_height > n:
            print(i)
            return

        # update columns
        for j in range(3):
            h[cols[j]] = max(h[cols[j]], new_height - ys[j] - base)

    print(-1)

if __name__ == "__main__":
    solve()
```该实现遵循仅跟踪每列顶部轮廓的想法。 数组`h`存储相对于全局压缩级别的高度`base`，这避免了当整行消失时需要物理移动列。 

对于每个图章，我们计算其三列中的当前顶面。 放置高度源自至少一根柱必须接触表面的要求，这意味着将印章的顶部与其调整后的柱约束中的当前最大范围对齐。 更新步骤独立调整每列，确保我们保持列高度的单调增长。 

一个常见的陷阱是忘记邮票的形状每列的减法不同，因此每列更新必须减去自己的$y_j$在与现有高度进行比较之前。 另一个微妙之处是确保所有比较都使用相同的压缩坐标系`base`。

 ## 边缘情况

 一种边缘情况是当网格高度非常小时，例如$n = 1$。 在这种情况下，任何占据超过一个有效高度单位的印章都会立即触发失败。 该算法可以处理这个问题，因为计算出的放置高度直接与$n$，因此即使第一次更新在必要时也正确地超出了限制。 

另一种情况是在相同的三列上重复冲压。 由于更新的高度是单调的，因此该算法可以正确累积增长，而无需重新访问以前的放置。 不变的是每列高度只增加或保持不变，因此不需要回滚。 

第三种情况涉及异构邮票高度，例如$(3,1,1)$。 这里，放置高度由三列之间的最大约束确定，但每列的更新不同。 该算法保留了正确性，因为每列都存储自己相对于共享放置高度的偏移量，确保即使在极端不对称的情况下，也没有列违反重叠约束。
