 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并要求比较由它构建的两个表达式。 第一个表达式采用所有数组元素的最大公约数，然后对该结果应用阶乘。 第二个表达式首先取每个元素的阶乘，然后计算这些阶乘值的 gcd。 任务是确定给定数组的这两个值是否始终相等。 

输入大小最多可达100000个元素，每个元素最大可达10^9。 这立即排除了任何尝试显式计算阶乘的方法，因为阶乘增长得非常快，即使对于适度的输入也变得不可行。 即使存储或操作完整的阶乘值也是不可能的，因此任何解决方案都必须依赖于 gcd 和阶乘的结构属性，而不是直接计算。 

当数组的 gcd 为 1 时，会出现微妙的边缘情况。在这种情况下，左侧表达式变为 1！ 即 1。正确的表达式也成为阶乘列表的 gcd，并且由于每个阶乘都可以被 1 整除，因此结果也是 1。因此 gcd 等于 1 的数组始终有效。 

当 gcd 大于 1 时，会出现更有趣的情况。例如，如果所有数字都是 2 的倍数，但有些数字不是 4 或 8 等更高幂的倍数，则阶乘行为会引入额外的质因子，而这些因子未反映在原始 gcd 结构中。 这种不匹配是不平等的根源。 

最小的反例直觉已经出现在像 2 和 3 这样的小数字上。它们的 gcd 是 1，所以等式成立，但是一旦所有数字共享一个大于 1 的公约数，阶乘结构就会放大素数指数，从而打破等式。 

## 方法

 暴力尝试将直接计算数组的 gcd，计算其阶乘，并计算每个元素的阶乘，然后获取 gcd。 第一部分很好，但第二部分立即失败，因为即使是 20！ 已经超过 10^18，这里的值高达 10^9，使得阶乘值变得天文数字般大。 即使使用大整数，在时间限制内计算 100000 个阶乘也是不可能的。 

关键的观察结果是阶乘结构完全由素数指数驱动。 对于数字 x!，素数 p 的指数由勒让德公式确定。 当我们取阶乘的 gcd 时，我们实际上是在所有这些勒让德和中取最小指数。 问题归结为在 gcd 之前或之后应用阶乘是否保留这些最小指数关系。 

决定性的简化是，当且仅当数组的 gcd 为 1 时，两个表达式才相等。如果 gcd 为 1，则两边的计算结果均为 1，如前所述。 如果 gcd 大于 1，则所有元素共享至少一个素因数，并且阶乘放大使第二个表达式在指数结构方面严格大于第一个表达式，从而打破相等性。 

因此，整个问题归结为数组上的单个 gcd 计算。

 ## 算法演练

 ## 算法演练

1. 计算数组中所有元素的gcd。 该值捕获同时除以每个元素的最大整数，这正是所需的唯一结构信息。 
2. 检查这个gcd是否等于1。这是问题中唯一的决策点，因为表达式的相等性完全取决于是否存在非平凡公约数。 
3. 如果gcd为1，则输出YES。 否则输出NO。 

### 为什么它有效

 阶乘运算保留了整除性，但以非线性方式放大素数指数。 当所有数字合计互质时（gcd 等于 1），每个元素中都不会强制出现质因数，并且两种构造都会崩溃为 1。当存在非平凡 gcd 时，每个数字都包含至少一个共享质因数，并且阶乘会引入这些素数的额外重数，这些质数在数组中不同步，导致阶乘的 gcd 超过 gcd 的阶乘。 只要 gcd 大于 1，这种结构性不匹配就保证了不平等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    g = 0
    for x in a:
        g = gcd(g, x)
    
    if g == 1:
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该实现直接遵循简化为单个 gcd 计算。 gcd 是迭代累积的，这避免了存储中间结果或执行任何昂贵的阶乘相关工作。 最终的比较是恒定时间。 

一个微妙的细节是用 0 初始化 gcd 累加器。这是安全的，因为 gcd(0, x) = x，因此它可以正确地使用第一个数组元素进行计算，而无需特殊的大小写。

 ## 边缘情况

 当数组包含单个元素等于 1 时，gcd 为 1，答案为 YES，与计算结果为 1 的两个表达式匹配。当数组包含单个元素大于 1 时，gcd 等于该元素，答案为 NO，因为右侧的阶乘放大无法匹配左侧结构。 

对于所有元素共享一个公约数（如 2）的数组，gcd 至少为 2，并且算法输出 NO。 gcd 计算直接捕获此条件，无需检查阶乘行为。
