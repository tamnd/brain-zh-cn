 ## 解决方案
 ## 问题理解

 我们被赋予了一个系统$n$组件，每个组件都必须分配一个非负整数值$d_i$。 这些值不是独立的。 组件对之间存在定向约束。 每个约束都有形式$u \to v$带参数$a, b$，并且它强制执行一个下限$d_u$按照$d_v$： 的值$u$必须至少是$a \cdot d_v + b$。 

除了这些依赖规则之外，还有一个全局预算：所有选定值的总和$d_i$不得超过$X$。 在满足依赖约束和总和约束的所有分配中，我们希望最大化所有分配中的最小值$d_i$。 如果不存在有效的分配，我们必须报告不可能性。 

因此，该结构是一个线性下界不等式系统，总和有一个全局上限。 目标不是最大化总和或单个值，而是在保持可行的情况下将最小变量推得尽可能高。 

约束允许最多$2 \cdot 10^5$每个测试的变量和依赖项，最多$10^4$测试。 这会立即排除每次测试中任何二次甚至三次的情况。 任何解决方案在总输入大小上都必须基本上是线性的或接近线性的，可能$O((n + m) \log X)$或更好。 

幼稚的解释会尝试贪婪地赋值或传播约束一次。 但这会失败，因为约束可以形成链和循环，通过系数乘法放大值$a_i$。 即使很小的周期也会强制产生大的定点行为。 

一些失败场景说明了其中的微妙之处：

 考虑两个相互约束的变量：$1 \to 2: d_1 \ge d_2$,$2 \to 1: d_2 \ge d_1 + 1$。 这是不可能的，因为这意味着$d_1 \ge d_1 + 1$。 

一个更微妙的情况：$1 \to 2: d_1 \ge 2 d_2$,$2 \to 1: d_2 \ge 2 d_1$。 这会迫使两个方向呈指数增长，除非所有值都为零，否则这是不可能的，但零可能会违反取决于常数的约束。 

另一种边缘情况是当约束形成一条链并且小幅增加向后传播时，导致一个节点主导整个系统。 局部贪婪分配会失败，因为更改一个节点会使先前的假设失效。 

## 方法

 暴力方法会尝试猜测目标值$T$, 强制执行$d_i \ge T$对于所有节点，重复传播约束直到到达固定点，然后检查是否满足总和约束。 这已经很昂贵了，因为每次传播可能多次遍历所有边，并且我们需要尝试许多值$T$。 在最坏的情况下，这会变成$O(nm)$每次检查并乘以二分搜索对大值变得不可行。 

关键的观察结果是问题在目标最小值处是单调的。 如果我们确定一个候选人$T$，我们可以检查可行性：强制执行$d_i \ge T$，然后重复应用约束$d_u = \max(d_u, a d_v + b)$直至稳定。 如果这个过程超出预算$X$，或产生矛盾，该候选人无效。 

这将问题转化为最大线性不等式系统的可行性检查。 每个约束向前传播下限，并且系统收敛，因为每次更新仅增加值，并且值通过可行性或溢出检测来限制。 

然后我们二分查找最大可行最小值$T$。 对于每次检查，我们计算与以下一致的最小可行分配$d_i \ge T$和依赖约束，然后验证总和。 

这是有效的，因为增加$T$只增加全部$d_i$在最终的闭包中，永远不会减少它们，因此可行性是单调的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个猜测的暴力传播|$O(nm \log X)$|$O(n + m)$| 太慢了 |
 | 二分查找+约束传播 |$O((n + m) \log X \log C)$|$O(n + m)$| 已接受 |

 这里$C$表示传播过程中值的大小，其边界为$10^{18}$规模。 

## 算法演练

 我们将该问题视为在固定下界约束下重复求解“最小可行分配”$T$。 

1. 固定候选值$T$。 初始化每个$d_i = T$。 这强制要求最小值至少为$T$。 
2. 将所有节点放入队列中。 我们将使用类似 BFS 的过程来传播约束松弛。 
3.趁队列不为空时，取一个节点$v$并尝试放松所有影响它的传入约束。 对于每条边$u \to v$带参数$a, b$，我们检查是否$d_u$必须增加才能满足$d_u \ge a \cdot d_v + b$。 如果是这样，请更新$d_u$并推$u$进入队列。 

我们以这种方式处理更新的原因是，每个节点的值是迫使其向上的所有约束的最大值，因此重复松弛会收敛到最小不动点。 

1. 在传播过程中，如果任何值超出安全范围（例如，$X$），我们可以提前停止并声明这一点$T$不可行的。 任何进一步的增加只会恶化可行性。 
2. 稳定后，计算所有的总和$d_i$。 如果超过$X$， 这$T$是不可行的。 否则是可行的。 
3. 二分查找结束$T$从$0$到足够大的上限（通常$10^{18}$）。 跟踪最佳可行的任务。 
4. 重新运行传播以获得最终最优值$T$来重建实际的$d_i$价值观。 

为什么它有效：

 关键的不变量是，在每个松弛步骤之后，形式的每个约束$d_u \ge a d_v + b$对于涉及更新节点的所有已处理边都满足，并且值仅在必要时增加。 这确保了一旦队列清空，任何约束都不能进一步增加任何变量，而不会违反先前值的最小值。 得到的向量是上面的最小不动点$T$。 由于约束和目标都是单调的$T$，二分查找正确找到最大可行最小值。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

def check(n, edges, X, T):
    d = [T] * n
    q = deque(range(n))
    inq = [True] * n

    while q:
        v = q.popleft()
        inq[v] = False
        dv = d[v]

        for u, a, b in edges[v]:
            # constraint: d_u >= a * d_v + b
            nd = a * dv + b
            if nd > d[u]:
                d[u] = nd
                if d[u] > X:
                    return None
                if not inq[u]:
                    q.append(u)
                    inq[u] = True

    total = sum(d)
    if total > X:
        return None
    return d

def solve():
    t = int(input())
    for _ in range(t):
        n, m, X = map(int, input().split())
        edges = [[] for _ in range(n)]

        for _ in range(m):
            u, v, a, b = map(int, input().split())
            u -= 1
            v -= 1
            edges[v].append((u, a, b))

        lo, hi = 0, 10**18
        best = None

        while lo <= hi:
            mid = (lo + hi) // 2
            res = check(n, edges, X, mid)
            if res is not None:
                best = res
                lo = mid + 1
            else:
                hi = mid - 1

        if best is None:
            print(-1)
        else:
            print(*best)

if __name__ == "__main__":
    solve()
```该实现以反向邻接形式对约束进行编码，以便每个节点只需要在其值发生变化时检查传入的影响。 BFS 队列确保任何增加都会传播直至稳定。 

功能`check`计算固定最小阈值的最小可行向量。 当值超过时的早期停止条件$X$防止不必要的传播。 然后，二分搜索确保我们找到最大可能的最小值。 

一个微妙的点是我们总是只从更新的节点传播。 如果没有`inq`守卫，队列可能会过度增长并重复处理未更改的节点，这会显着降低性能。 

## 工作示例

 考虑一个具有三个节点和简单线性依赖关系的小型系统。 

输入：```
3 2 100
1 2 1 0
2 3 1 0
```我们将约束解释为：$d_1 \ge d_2$,$d_2 \ge d_3$，有总金额限制。 

我们二分查找$T$。 

为了$T = 10$:

 | 步骤| 队列| 更新节点 | 值 (d1,d2,d3) |
 | ---| ---| ---| ---|
 | 初始化| 0,1,2 | 无 | (10,10,10) |
 | 放松2→3 | 2 | d2=10≥10| (10,10,10) |
 | 放松1→2 | 1 | d1=10≥10| (10,10,10) |

 总和为30，可行。 

为了$T = 40$:

 | 步骤| 队列| 更新节点 | 值 (d1,d2,d3) |
 | ---| ---| ---| ---|
 | 初始化| 0,1,2 | 无 | (40,40,40) |

 总和为 120 > 100，不可行。 

这表明二分搜索正确区分了基于全局总和的可行性。 

该轨迹证实，一旦阈值固定，传播就不会改变相对结构，只会改变绝对缩放。

 ## 边缘情况

 具有矛盾的增长约束的纯循环是最微妙的情况。 考虑两个节点：```
2 2 100
1 2 2 0
2 1 2 0
```从任意开始$T$，传播导致两个值无限制地乘法增长，除非上限为$X$。 该算法通过重复更新来检测这一点，最终超过$X$，返回足够大的不可行性$T$。 然后二分搜索正确地下降到不稳定状态以下。 

第二个边缘情况是具有大系数的长链。 即使值仍然有限，它们也可以迅速升级到接近$10^{18}$，所以超过时提前截止$X$对于避免溢出和不必要的传播至关重要。
