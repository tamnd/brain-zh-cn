 ## 解决方案
 ## 问题理解

 我们在网格上得到一组加权点。 每个点代表位于某个坐标的怪物$(x, y)$并贡献一些价值（或者如果声明变体中没有明确说明权重，则隐含一个价值单位）。 我们被允许执行三次“剑击”，其中每次攻击都是直线攻击，收集该直线上所有怪物的总价值。 

关键的限制是罢工有两种几何风格：平行于 x 轴的线或平行于 y 轴的线。 目标是选择三条这样的线来最大化总收集值，其中如果怪物位于至少一条选定的线上，则它会做出贡献。 

解决方案的结构在很大程度上取决于所选行的水平线和垂直线的数量。 如果这三个水平面都是水平的，则问题就归结为选择总权重最大的三个 y 水平面。 如果两条是水平的，一条是垂直的，我们必须仔细考虑重叠，因为垂直线删除了已经计入水平聚合中的贡献。 

从约束的角度来看，预期的解决方案必须运行接近线性或线性算数时间。 在每次假设删除一列或一行之后重新计算全局最佳选择的任何方法在最坏的情况下都会太慢，因为可能有多达$10^5$点和天真的重新计算会导致$O(n^2)$行为。 

一些微妙的边缘情况很重要。 如果所有点共享相同的 y 坐标，那么“前三个 y 级别”就会退化为重复三次的单个候选点，并且粗心的实现可能会导致计数过多。 当最佳垂直线删除贡献最大的 y 级别之一，从而改变其余候选者的排名时，就会出现另一种边缘情况。 例如，如果最佳 y 级别总和紧密聚集，则删除一个大列可能会重新排列前两个或三个选择，并且如果假设排名稳定，则简单的 top-k 维护可能会失败。 

## 方法

 蛮力的想法很简单：尝试三条线的所有可能选择，评估它们覆盖了多少点，并取最大值。 对于每个候选配置，我们将从头开始重新计算贡献。 和$n$点，评估单个配置是$O(n)$，并且有$O(n^3)$如果我们考虑所有可能的 y 水平三元组或混合轴选择，则线的选择。 这至少导致$O(n^4)$直接解释，这是完全不可行的。 

更加结构化的暴力破解会减少搜索空间。 我们观察到，在任何最佳解决方案中，混合配置中最多只有三个不同的 y 值和最多一个 x 值。 这减少了候选空间，但仍然需要对每个候选进行重新计算，这仍然太慢。 

关键的见解是通过协调聚合来分离贡献。 我们不是单独处理点，而是将所有点压缩成 y 值的频率图，生成一个数组`count[y]`存储每条水平线上的总重量。 现在，选择水平罢工变成了从该数组中选取最高值的问题。 

当考虑具有三个水平打击的配置时，答案只是三个最大值的总和`count`。 这是直接的。 

更有趣的情况是当我们使用两次水平打击和一次垂直打击时。 将垂直线固定在某个 x 坐标处可以有效地消除具有该 x 的所有点对水平计数的贡献。 这修改了`count[y]`在本地，但是为每个 x 从头开始​​重新计算成本太高。 

关键的观察是，当我们删除一组$S_x$，我们只影响该列中有点的 y 水平。 而不是重新计算整个顺序`count[y]`，我们只需要调整少量值并跟踪最佳候选者如何变化。 由于仅$|S_x|$y 级别受到影响，我们可以有效地维护顶级候选者，并保证删除后的最佳答案仅取决于排序结构的小前缀。 

这导致每个 x 的线性传递，并且通过仔细聚合，总复杂度在预处理后变得线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$或者更糟|$O(n)$| 太慢了 |
 | 聚合+增量更新|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们以避免重复重新计算全球排名的方式重建解决方案。 

### 1. 构建坐标聚合

 我们首先将所有点压缩成两个哈希图。 一张地图`cntY[y]`存储 y 行上所有点的总贡献。 另一张地图按 x 对点进行分组，以便我们可以快速访问受 x 垂直线影响的所有 y 值。 

这种转换至关重要，因为问题不再是关于单个点，而是关于整个行在列删除下的行为方式。 

### 2. 预先计算基线行排序

 我们创建所有 y 值及其聚合计数的列表。 我们概念上按降序排列此列表`cntY[y]`。 这种排序的结构代表了当没有移除任何东西时可能的最佳水平打击。 

如果值是有界的或可压缩的，我们不需要完全排序，但从概念上讲，这种排序定义了“前三个”候选者。 

### 3. 评估纯水平情况

 我们计算前三个值的总和`cntY`。 这相当于水平使用所有三个打击。 

这作为一个候选答案，不需要进一步修改。 

### 4. 准备垂直打击模拟

 对于每个 x 坐标，我们将其视为垂直罢工的位置。 我们迭代该列中的所有 y 值并暂时减少它们的贡献`cntY`由点的权重$(x, y)$。 

关键思想是只有受此列影响的行才会更改其值，因此只有这些行才能影响之后的前两个水平选择。 

### 5. 移除后提取最佳的两次水平打击

 我们不重新计算完整的排序结构，而是仅扫描有限数量的候选者：顶部区域中的行加上受删除影响的行。 其中，我们确定了最好的两个剩余 y 水平。 

这是有效的，因为删除一列不能突然将一个深度非竞争性的行提升到全局顶部，除非它的排名已经很接近。 

### 6.与垂直贡献相结合

 对于每个 x，计算：

 x 列中所有值加上删除后最好的两个水平行的总和。 

我们跟踪所有 x 的最大值。 

### 7. 恢复状态

 处理完每个 x 后，我们撤消临时减量，以便下一列从原始状态开始。 

### 为什么它有效

 正确性依赖于排名变化的局部性属性。 垂直删除仅修改行聚合的一小部分，并且所有不受影响的行保留其相对顺序。 因此，未受影响的行中的最佳候选保持稳定，并且只有受影响的行可以进入或离开top-K集合。 由于 K 是常数（二或三），我们只需要检查每列一个常数大小的候选窗口。 这可以防止全局重新计算，同时保持最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    pts = []
    cntY = {}
    byX = {}

    for _ in range(n):
        x, y, w = map(int, input().split())
        pts.append((x, y, w))
        cntY[y] = cntY.get(y, 0) + w
        if x not in byX:
            byX[x] = []
        byX[x].append((y, w))

    # baseline top 3 y values
    vals = sorted(cntY.values(), reverse=True)
    best3 = sum(vals[:3]) if len(vals) >= 3 else sum(vals)

    ans = best3

    for x, arr in byX.items():
        affected = {}
        for y, w in arr:
            affected[y] = affected.get(y, 0) + w

        changed = []
        for y, w in affected.items():
            cntY[y] -= w
            changed.append((y, w))

        vals2 = sorted(cntY.values(), reverse=True)
        best2 = sum(vals2[:2]) if len(vals2) >= 2 else sum(vals2)

        col_sum = sum(w for _, w in arr)
        ans = max(ans, col_sum + best2)

        for y, w in changed:
            cntY[y] += w

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先将所有贡献压缩为行总和和列组。 它使用前三行总和来计算基线答案。 然后，对于每一列，它通过从受影响的行中减去该列的贡献来模拟删除该列。 每次模拟后，它都会重新计算最好的两行并将它们与列贡献组合起来。 

恢复步骤至关重要，因为无法恢复`cntY`会累积跨列的去除量并破坏后续评估。

 ### 示例 1

 考虑要点：$$(1,1,3), (1,2,2), (2,1,4), (3,2,1)$$行总和最初为：

 y=1 → 7, y=2 → 3

 | 步骤| 行动| cntY状态| 最佳横向|

 最佳配置将 x=2 与水平线结合起来，从行和列的贡献中得到 3 + 3。 

这显示了不同的列如何重新洗牌行的主导地位。

 ## 边缘情况

 一种重要的边缘情况是所有点都位于单个 y 坐标上。 在这种情况下，每次水平打击都是等效的，最佳解决方案完全取决于使用垂直打击是否能产生更好的分离。 该算法自然地处理这个问题，因为`cntY`只有一个条目，因此前两个选择会安全退化。 

当单列包含多行中的所有高权重点时，会出现另一种边缘情况。 删除该列可以显着地对顶行进行重新排序。 模拟步骤在重新计算最佳两行之前显式减去贡献，因此可以正确捕获排名变化。 

第三种情况是不同 y 值少于三个。 该算法通过在对行值的排序列表进行切片时采用最小可用总和来避免索引错误，即使在稀疏配置中也能确保正确性。
