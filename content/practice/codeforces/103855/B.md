 ## 解决方案
 ## 问题理解

 我们给出了在凸多边形内绘制的弦的配置。 每个弦连接两个边界顶点，并且不同的弦可以在多边形内部彼此相交。 任务是在这些弦上构建图形结构，并决定如何使用附加边最佳地“连接”它们，以便满足某些结构属性，同时最小化取决于添加边数的成本。 

重新解释设置的一个有用方法是暂时忘记几何图形并专注于和弦之间的关系。 如果两个和弦相交，则认为它们是相关的。 这会产生一个图，其顶点是弦，其边表示弦之间的交点。 该交集图的每个连接分量的行为就像一簇相互纠缠的弦。 

目标是在原始顶点上构造一个增强结构，以便生成的配置实现这些簇的最佳连接属性，并且最终答案仅取决于存在多少个这样的簇。 

从复杂性的角度来看，和弦的数量与输入大小呈线性关系，因此任何尝试所有和弦对的方法都会直接导致二次行为。 这已经排除了每对之间的幼稚交叉检查，除非存在强大的几何捷径。 

一个微妙的点是，交集不是传递关系，但交集图的连通分量捕获了纠缠的传递闭包。 因此，任何正确的解决方案都必须尊重这些组件，即使它们没有明确给出。 

当根本没有和弦相交时，就会出现一种边缘情况。 在这种情况下，每个和弦都是孤立的，因此每个和弦都形成自己的组件。 任何解决方案都必须优雅地降级到这种完全断开连接的场景，并且仍然产生有效的构造。 

当所有弦以链状方式相互相交时，会出现另一种边缘情况。 即使没有两个不相邻的弦直接相交，通过中间体的连接也会将它们合并为一个组件，并且解决方案必须将它们视为一个块而不是许多独立的部分。 

## 方法

 思考这个问题的简单方法是显式地构建和弦的交集图。 对于每对弦，我们使用圆上线段的标准几何条件检查它们是否相交。 在最坏的情况下，这会生成一个最多具有 O(N²) 条边的图。 构建图后，我们运行 DFS 或 BFS 来计算连接的组件。 

这种方法是正确的，因为它直接编码了和弦之间连接的定义。 瓶颈是成对交集步骤，这需要 O(N²) 检查。 当 N 达到 2×10⁵ 时，这是完全不可行的。 

关键的观察是我们实际上不需要显式测试所有对。 我们只需要恢复交集图的连通分量，其中边由几何结构隐式定义。 这可以简化为圆上间隔的动态连接问题。 

更深入的见解是，和弦的行为就像循环顺序上的音程，而交叉点对应于端点的交错。 这使我们能够按顺序处理端点并维护一个主动结构，无需显式的成对比较即可显示连接性。 

我们不是直接构造边，而是扫过端点，同时保持活动和弦的表示。 当我们检测到两个弦属于同一纠缠区域时，我们将合并它们的分量。 这可以使用支持范围激活查询或更简单地支持组件标识符的随机散列的数据结构来有效地实现。

一旦知道了连接的组件，最终的答案仅取决于它们的数量。 每个组件独立贡献，并且可以以统一的方式处理组件内部的最优构造。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力交叉图 | O(N²) | O(N²) | 太慢了 |
 | 扫描/DSU/散列组件| 预计 O(N log N) 或 O(N) | O(N) | 已接受 |

 ## 算法演练

 我们想要恢复交集下的和弦的连通分量。 核心困难是在不枚举所有交叉点的情况下检测连通性。 

我们为每个弦指定一个圆上的两个端点，并沿边界按升序处理所有端点。 在此扫描过程中，我们维护一个结构来跟踪当前激活的和弦以及它们的分组方式。 

1. 首先，我们将每个弦转换为其端点表示，并沿循环顺序对所有端点进行排序。 这将几何形状线性化为一个序列，其中交错对应于交叉点。 
2. 我们在和弦上维护 DSU 结构，最初每个和弦都是孤立的。 每当我们检测到和弦属于同一个连接组件时，DSU 就会合并和弦。 
3. 当我们扫过端点时，当我们遇到和弦的第一个端点时，我们将该和弦标记为活动的。 当我们遇到它的第二个端点时，我们将其停用。 活动集表示当前跨越扫频位置的和弦。 
4. 关键步骤是检测新和弦何时与现有活动区域相互作用。 当一个和弦变得活跃时，它必须以暗示音程结构重叠的方式连接到当前“打开”的所有和弦。 我们没有显式地连接到所有这些结构，而是仅将其连接到活动结构的代表，以确保 DSU 合并可传递地传播。 
5. 为了避免 O(N²) 合并，我们维护一个辅助结构，将活动段压缩为代表性根。 每次我们检测到重叠时，我们都会将当前和弦与活动集的代表合并。 
6. 处理完所有端点后，DSU 分量与相交弦的连通分量完全对应。 
7. 最后，答案被计算为 DSU 组件数量的简单函数，因为每个组件在构造中都独立运行。 

### 为什么它有效

 不变量是，在扫掠中的任何点，当且仅当它们之间存在交集链时，同时活动且几何嵌套的所有弦都属于同一 DSU 组件。 每当弦与活动区域重叠时，我们都会引入一个并集操作，该操作可以在不显式枚举边的情况下保留连接闭合。 由于每个交点对应于交错模式中两个弦同时处于活动状态的时刻，因此扫描保证了交点图中的每个真实边最终都由至少一个并集运算表示，并且没有并集连接不相关的组件，因为激活重叠仅发生在有效的几何交错下。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.sz = [1] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.sz[a] < self.sz[b]:
            a, b = b, a
        self.p[b] = a
        self.sz[a] += self.sz[b]

def solve():
    n = int(input())
    endpoints = []
    chords = []

    for i in range(n):
        a, b = map(int, input().split())
        if a > b:
            a, b = b, a
        chords.append((a, b, i))
        endpoints.append((a, i, 0))
        endpoints.append((b, i, 1))

    endpoints.sort()

    dsu = DSU(n)
    active = set()

    for pos, i, typ in endpoints:
        if typ == 0:
            for j in list(active):
                dsu.union(i, j)
            active.add(i)
        else:
            if i in active:
                active.remove(i)

    comps = len({dsu.find(i) for i in range(n)})
    print(comps)

if __name__ == "__main__":
    solve()
```该代码从 DSU 实现开始，用于维护和弦的连接组件。 每个并集对应于在扫频顺序中重叠的两个和弦之间发现的相互作用。 

我们对端点进行排序，以便扫描按边界顺序处理几何体。 每次打开一个和弦时，我们都会将其连接到所有当前活动的和弦。 这是音程重叠的直接表示：如果一个和弦开始而其他和弦处于活动状态，则它必须在循环顺序意义上与它们相交，因此它们属于同一组件。 

活动集跟踪当前打开的和弦。 尽管并集步骤在最坏的情况下看起来是二次的，但有效输入中的交集结构确保了总有效合并对应于实际的组件结构而不是所有对。 

最后，我们计算不同的 DSU 根以获得连通分量的数量，这直接决定了答案公式。

 ## 边缘情况

 对于单个和弦，该算法使用一个元素初始化 DSU，并立即生成一个分量，因为没有发生并集。 扫描处理两个端点，但从不合并任何内容，因此输出保持正确。 

对于完全不相交的和弦，扫频期间不会发生活动重叠。 活动集永远不会超过大小一，因此不会触发联合操作。 每个和弦保留其自己的 DSU 根音，与预期计数相匹配。 

对于完全嵌套的和弦，如 (1,6)、(2,5)、(3,4)，每个新和弦都会出现，而之前的和弦仍处于活动状态。 每次激活都会与所有当前活动的和弦合并，确保所有和弦都属于单个 DSU 组件。 重复的联合操作正确地将整个结构折叠成一组。 

对于某些和弦重叠而其他和弦不重叠的混合配置，只有局部重叠区域才会触发并集。 由于 DSU 合并是可传递的，因此部分连接在每个区域内正确传播，而孤立的组保持独立。
