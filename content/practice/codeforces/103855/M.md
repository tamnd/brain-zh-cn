 ## 解决方案
 ## 问题理解

 该问题从一系列数字开始，要求我们计算所有元素对的全局表达式。 对于输入中出现的每个数组，我们有效地聚合了一个依赖于元素之间的成对差异的函数。 乍一看，这看起来像是所有对的双倍和，这表明对所有指数进行二次计算。 

关键的困难在于该表达式涉及值之间的绝对差异，这通常会妨碍代数简化。 直接解释需要迭代所有对和计算术语，例如$|p_i - p_j|$，这立即表明$O(N^2)$方法。 

然而，这种类型的 Codeforces 问题所隐含的约束通常允许大约$10^5$元素，这排除了任何二次成对迭代。 任何超出粗略范围的事情$10^8$在严格的时间限制下，Python 中的操作变得不安全。 这促使我们寻求一种将成对相互作用减少为线性或近线性的公式。 

当所有值都相同时，会出现微妙的边缘情况。 在简单的实现中，人们仍然可以正确计算成对贡献，但在简化表达式时很容易意外地过度计算或错误处理对称性。 当值较大或为负时，会出现另一种边缘情况，因为删除绝对值的代数重排必须准确保留符号处理。 如果试图错误地对数组进行排序或转换而不考虑对贡献如何在索引之间分布，就会出现第三个问题。 

## 方法

 我们从暴力解释开始。 该表达式基本上是涉及绝对差的所有有序或无序元素对的总和。 最直接的方法是迭代每一对$(i, j)$，计算该对的贡献，并累加结果。 这在概念上是简单且正确的，因为它准确地反映了定义。 问题是它执行$N^2$每个数组的操作，当$N$很大。 为了$N = 10^5$，这需要大约$10^{10}$的操作，远远超出了可接受的范围。 

第一个简化来自于去除一维设置中的绝对值。 一旦序列排序完成，符号$p_i - p_j$根据索引顺序变得确定性。 这使我们能够根据前缀贡献重写成对总和。 每个元素对其后面的所有元素都有正向贡献，对其之前的所有元素有负向贡献，从而产生按位置加权的线性组合。 这将双倍和折叠成单遍公式，其中每个元素$p_i$乘以仅取决于其索引的系数。 

第二个关键观察结果是，问题实际上并不是一维的。 该表达式扩展到坐标对$(p_i, q_i)$，感兴趣的数量成为绝对坐标差的最大值。 这正是切比雪夫距离，坐标系旋转45度后可以分解为曼哈顿距离。 

通过引入变换后的坐标$a_i = p_i + q_i$和$b_i = p_i - q_i$，切比雪夫距离分裂为两个独立的一维绝对差问题。 其中每一个都可以使用之前导出的相同线性缩减技术进行评估。 这将整个问题简化为计算同一函数两次并以代数方式组合结果。 

最后，仔细的代数表明，原始表达式可以重构为原始坐标贡献减去变换贡献的一半的线性组合。

 | ---| ---| ---| ---|
 | 蛮力 |$O(N^2)$|$O(1)$| 太慢了 |
 | 最佳 |$O(N \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 该算法围绕将所有成对绝对差减少为单个加权和进行。 

1. 读取数组$p$和$q$，每个长度$N$。 这些表示点的坐标，其中每个索引对应于 2D 空间中的单个点。 
2.构造两个辅助数组$a_i = p_i + q_i$和$b_i = p_i - q_i$。 此变换旋转坐标系，使得切比雪夫距离可分离为一维绝对差。 
3. 对于每个序列$p$,$q$,$a$， 和$b$，计算定义为排序元素的加权线性组合的函数值。 这是通过对数组进行排序，然后使用从成对扩展导出的基于索引的系数公式累积贡献来完成的。 
4. 使用恒等式合并结果$\text{answer} = value(p) + value(q) - \frac{value(a) + value(b)}{2}$。 

这来自将最大绝对差结构重写为旋转坐标贡献并纠正重复计算。 
5. 输出最终结果。 

关键的计算步骤是对每个数组进行排序并应用相同的线性扫描来有效地计算其贡献。 

### 为什么它有效

 正确性依赖于两个结构事实。 首先，一维的绝对差之和可以精确地重写为排序元素的线性函数，因为每个元素的贡献仅取决于有多少元素小于或大于它。 这将二次相互作用转变为确定性系数和。 

其次，点之间的切比雪夫距离相当于两个独立旋转轴的最大值，应用变换后可以表示为曼哈顿距离$(p+q, p-q)$。 由于曼哈顿距离本身分解为每个坐标轴上的绝对差，因此整个二维问题变成了四个独立的一维问题的组合。 最终的公式只是这些分解贡献的代数重组。 

由于每个变换都以所需的形式精确保留成对距离，因此不会引入近似值，并且每对在两种表示中的贡献相同。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def value(arr):
    arr.sort()
    n = len(arr)
    res = 0
    for i, x in enumerate(arr, 1):
        res += (2 * i - n - 1) * x
    return res

def solve():
    n = int(input())
    p = list(map(int, input().split()))
    q = list(map(int, input().split()))

    a = [p[i] + q[i] for i in range(n)]
    b = [p[i] - q[i] for i in range(n)]

    vp = value(p[:])
    vq = value(q[:])
    va = value(a)
    vb = value(b)

    ans = vp + vq - (va + vb) // 2
    print(ans)

if __name__ == "__main__":
    solve()
```核心辅助函数计算排序后序列的线性化贡献。 系数$(2i - n - 1)$捕获排序数组中当前位置两侧有多少元素，这正是取代成对绝对差值展开的方法。 

主函数构造旋转坐标数组并在所有四个序列上计算相同的函数。 一个微妙的问题是，整数除以 2 是安全的，因为由于成对展开的对称性，组合的变换贡献总和为偶数。

 ## 边缘情况

 一种边缘情况是当所有值都在$p$或者$q$是相同的。 在这种情况下，每个成对差异为零，并且算法在所有变换后也必须产生零。 排序后的数组保持不变，因此每个元素接收的系数完全抵消。 变换后的数组$a$和$b$also become constant, preserving the same cancellation.

 另一种边缘情况涉及负数$p - q$，填充数组$b$。 由于该算法仅依赖于排序后的排序，因此符号并不重要。 即使值严重为负，排序仍然可以正确地放置它们，并且系数公式继续准确地计算左右贡献。 

第三种情况是当$p$和$q$规模很大但结构合理，因此$p+q$和$p-q$32 位算术溢出。 使用 Python 整数可以避免此问题，但在类型化语言中，这需要 64 位或更高的精度才能保持转换的正确性。
