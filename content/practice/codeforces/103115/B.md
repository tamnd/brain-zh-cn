 ## 解决方案
 ## 问题理解

 我们有一个过程，从大量相同的玩家开始，所有玩家都位于虚拟状态，写为$(0,0)$。 一个状态$(a,b)$代表获胜的玩家$a$比赛和输球$b$游戏。 游戏以回合形式进行，玩家总是只与处于完全相同状态的其他玩家配对。 每场比赛都会增加获胜者的胜利计数和失败者的失败计数，因此状态$(a,b)$分为两个可能的下一个状态：$(a+1,b)$和$(a,b+1)$。 

一旦达到边界条件，玩家就会被移除：要么达到$n$胜利或受苦$m$损失。 一旦达到这样的界限，他们就会立即退出系统。 

最初，有$2^{n+m}$球员在$(0,0)$，并且该过程将继续，直到所有玩家都退出。 对于多个查询，我们被问到：有多少玩家最终在给定的边界状态下准确退出$(a,b)$，其中任一$a=n$或者$b=m$。 

关键的约束洞察是$n$和$m$可以大到$2 \cdot 10^5$，并且最多有$2 \cdot 10^5$查询。 这立即排除了对过程或每个玩家的跟踪的任何模拟。 即使从图遍历的角度思考也太慢了，因为隐式状态空间是$O(nm)$，这远远超出了可行的范围。 

正确的方法必须将整个过程压缩为封闭形式的组合表达式。 

当考虑提前退出时，会出现一种微妙的边缘情况。 例如，一名玩家到达$(n,b)$或者$(a,m)$立即停止，因此人们可能怀疑流入更深层次的状态受到了影响。 然而，这个过程是完全对称和质量守恒的：每个内部状态将其群体完全分裂到下一层，并且终止仅删除边界处的节点，而不影响到达它们的数量。 

例如，考虑一个小案例$n=2, m=1$。 如果我们尝试进行模拟，我们很快就会发现，没有公式的路径计数会变得混乱，并且状态上的朴素 DP 在缩放时仍然会爆炸。 

## 方法

 暴力解释将模拟整个国家扩张。 各州$(a,b)$每场比赛将其人口分成两半。 由于状态数量以网格形式增长$(0,0)$向外，跃迁总数在深度上呈指数级增长$n+m$，这是完全不可行的。 

关键的见解是停止从个人角度思考，而是从玩家可以达到某种状态的方式来思考$(a,b)$。 每个玩家对应一系列的胜利和失败，其中每次胜利都会递增$a$并且每次损失都会增加$b$。 达到$(a,b)$总共需要恰好$a+b$步骤，并且这些步骤的顺序可以自由选择。 

所以不同路径的数量$(a,b)$只是一个二项式系数$\binom{a+b}{a}$。 这个过程中的每一步也会将人口减半，因为每个群体都会均匀地分为赢家和输家。 这就引入了一个因素$2$在每个深度级别进行缩放。 

如果我们仔细跟踪，每一步都会使路径的权重减少一个因子$1/2$，并且由于路径$(a,b)$有长度$a+b$，总缩放变为$2^{-(a+b)}$。 开始于$2^{n+m}$，我们得到一个干净的封闭形式。 

因此，答案变成了直接的组合公式而不是模拟。

 ## 算法演练

 1. 预先计算阶乘和逆阶乘$n+m$，因为我们需要计算二项式系数$\binom{a+b}{a}$有效地进行模运算。 这避免了每个查询重新计算组合。 
2. 对于每个查询$(a,b)$，将其解释为过程停止的边界状态。 我们不模拟转变； 相反，我们计算有多少条不同的进化路径达到这个确切的状态。 
3. 计算二项式系数$\binom{a+b}{a}$，计算有多少次胜利和失败的序列导致$(a,b)$。 
4. 将其乘以比例因子$2^{n+m-a-b}$，这说明了初始人口是$2^{n+m}$每一步都有效地将质量均匀地分布在两个分支之间。 
5. 输出结果取模$10^9+7$。 

### 为什么它有效

 关键的不变量是每个内部状态将其全部人口均匀分布到其两个子状态中，并且除了边界终止处之外没有质量损失。 这意味着系统的行为就像所有长度达到的序列的完整二进制扩展$n+m$，其中每条路径均等且独立地对一个终端边界态做出贡献。 由于每条路径都是由获胜和失败的序列唯一确定的，因此对路径进行计数就足够了，并且分裂对称性通过 2 的幂确保了统一的权重。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modexp(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    n, m, q = map(int, input().split())
    N = n + m

    fact = [1] * (N + 1)
    invfact = [1] * (N + 1)

    for i in range(1, N + 1):
        fact[i] = fact[i - 1] * i % MOD

    invfact[N] = modexp(fact[N], MOD - 2)
    for i in range(N, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD

    def C(x, y):
        if y < 0 or y > x:
            return 0
        return fact[x] * invfact[y] % MOD * invfact[x - y] % MOD

    pow2 = [1] * (N + 1)
    for i in range(1, N + 1):
        pow2[i] = pow2[i - 1] * 2 % MOD

    for _ in range(q):
        a, b = map(int, input().split())
        steps = a + b
        ways = C(steps, a)
        ans = ways * pow2[n + m - steps] % MOD
        print(ans)

if __name__ == "__main__":
    solve()
```阶乘预计算支持快速二项式系数查询，而双表的强大功能可有效处理重复求幂。 关键细节是使用$a+b$作为组合路径的深度，然后通过调整$n+m-(a+b)$以匹配系统的初始缩放。 

一个常见的错误是尝试对从$(0,0)$没有意识到该过程是对称的，并且路径计数已经捕获了所有有效的演化。

 ## 边缘情况

 一个关键的边缘情况是当$a=0$或者$b=0$，意味着纯粹通过一种结果达到状态。 在这种情况下，$\binom{a+b}{a}=1$，答案完全简化为二次方缩放因子。 该算法自然地处理了这个问题，因为基于阶乘的二项式计算对于这些极值返回 1。 

另一个边缘情况是当$a+b = n+m$，意味着达到最大深度的状态。 这里指数变为零，所以答案正是$\binom{n+m}{a}$，反映出不再有进一步的分裂。 

最后，边界状态$a=n$或者$b=m$统一处理，无需特殊分支，因为该公式不区分超出坐标的获胜边界和损失边界状态。
