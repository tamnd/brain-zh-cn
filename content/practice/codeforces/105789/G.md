 ## 解决方案
 ## 问题理解

 我们得到一个一维板，其中每列都有一个当前高度。 一系列操作放置影响连续列段的片段。 每个操作的行为就像一个段上的“下降”：该段中的高度被统一修改，作为一个块增加。 

决定移动是否安全的关键规则仅取决于局部一致性。 当一个片段应用于一系列列时，我们只需要关心该范围内的所有列当前是否具有相同的高度。 如果它们相等，则应用该部件会保留干净的分层结构。 如果它们不同，则在应用该操作后，至少有一列在填充的单元格下方将有一个内部间隙，这会破坏结构并使状态不安全。 

因此，每个查询都减少为对数组的两个操作：检查某个范围内的所有值是否相同，如果检查通过，则将该范围内的所有值加一。 

输入大小足够大，在该问题族的典型版本中，列数和操作数都可以达到 200000 左右。 这立即排除了任何扫描每个查询范围的解决方案，因为简单的检查将花费每个操作的线性时间，并在最坏的情况下导致二次行为。 

如果我们尝试在坐标较大或稀疏时显式扩展棋盘，则直接模拟也会遇到内存问题。 预期的困难是在概念上保持简单的结构上有效地处理许多范围检查和范围增量。 

一种微妙的边缘情况来自长交替图案。 如果高度频繁变化，则未正确分割边界的幼稚间隔合并方法将默默地传播错误的均匀性检查。 另一个边缘情况是重复的全范围更新，其中不正确的合并逻辑可能会折叠不同的段并在以后错误地报告一致性。 

## 方法

 蛮力解释很简单。 我们存储一个高度数组。 对于一个段上的每个操作$[l, r]$，我们首先扫描该范围内的所有值以检查它们是否相等。 如果它们不相等，我们拒绝该操作。 如果它们相等，我们将范围内的所有值加一。 

这是有效的，因为它完全遵循问题定义。 正确性是立即的，因为我们直接验证每个受影响列的条件。 失败点是性能。 一次操作即可完成$O(n)$时间，并与$m$操作，最坏的情况变成$O(nm)$。 当两者都达到200000时，这就远远超出了可行的极限。 

关键的观察是数组的结构在实践中并不是任意的。 我们唯一测试的是一个段是否一致，并且我们唯一做的就是统一增加一个段。 这意味着数组以常量值块的形式演变，并且操作仅拆分或合并这些块。 我们永远不需要知道统一段内的各个值，只需要了解段边界及其值。 

这建议将数组划分为最大的段，其中每个段存储恒定的高度。 然后，范围更新变成这些片段的局部分割和合并。 或者，具有惰性传播的线段树可以维护范围增量并支持使用最小和最大跟踪的均匀性查询。 

两种观点是等价的：一种是显式区间集，另一种是隐式平衡二元分解。

 ## 算法演练

 我们描述基于区间的解决方案，因为它最直接地反映了问题的结构。 

1. 我们维护一组覆盖整个棋盘的不相交间隔，其中每个间隔存储左边界、右边界和恒定的高度值。 

这种表示是有效的，因为任何时候两个相邻区域共享相同的高度，它们都会立即合并，从而确保最大分段。 
2.对于一个段上的每一个操作$[l, r]$，我们首先找到与该范围相交的所有区间。 

此步骤是必要的，因为更新可能会穿过现有的统一块，并且正确性取决于在边界处精确地分割它们。 
3. 任何部分重叠的区间$[l, r]$最多分为三部分：左侧剩余部分、中间受影响部分和右侧剩余部分。 

分割可确保更新区域与间隔边界对齐，因此我们永远不会将更新应用于未对齐的段。 
4. 分割后，我们检查受影响的区域是否恰好由一个区间或多个区间组成。 

如果有多个区间覆盖$[l, r]$，那么高度在范围内的某个地方有所不同，因此操作不安全，我们跳过更新。 
5. 如果区域是均匀的，我们将其高度加一，替换表示的区间$[l, r]$，然后尝试与相邻间隔合并（如果它们现在共享相同的值）。 

合并可恢复最大分段并防止不必要的碎片。 
6. 每个操作仅涉及恒定数量的间隔，并且所有搜索和更新都使用有序结构操作来执行。 

### 为什么它有效

 在任何时候，该结构都会将数组维护为具有相同高度的最大连续段。 任何询问范围是否统一的查询都相当于检查该范围是否完全包含在单个段内。 如果它跨越多个线段，则该范围内至少存在一个边界，这意味着必须存在两个不同的高度。 更新保留了这种不变性，因为分割将段精确地与查询边界对齐，而合并确保我们永远不会保留冗余的相邻相等段。 这既保证了一致性检查的正确性，又保证了增量更新的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Seg:
    __slots__ = ("l", "r", "v")
    def __init__(self, l, r, v):
        self.l = l
        self.r = r
        self.v = v

def solve():
    n, q = map(int, input().split())
    
    segs = []
    segs.append(Seg(1, n, 0))

    def find(pos):
        lo, hi = 0, len(segs) - 1
        while lo <= hi:
            mid = (lo + hi) // 2
            s = segs[mid]
            if s.l <= pos <= s.r:
                return mid
            if pos < s.l:
                hi = mid - 1
            else:
                lo = mid + 1
        return -1

    def split(idx, x):
        s = segs[idx]
        if s.l == x:
            return idx
        if s.r == x:
            return idx + 1
        if s.l < x < s.r:
            left = Seg(s.l, x - 1, s.v)
            right = Seg(x, s.r, s.v)
            segs[idx] = left
            segs.insert(idx + 1, right)
            return idx + 1
        return idx

    def normalize(i):
        if i > 0 and segs[i].v == segs[i - 1].v:
            segs[i - 1].r = segs[i].r
            segs.pop(i)
            return i - 1
        if i + 1 < len(segs) and segs[i].v == segs[i + 1].v:
            segs[i].r = segs[i + 1].r
            segs.pop(i + 1)
        return i

    for _ in range(q):
        l, r = map(int, input().split())

        i = find(l)
        i = split(i, l)

        j = find(r)
        j = split(j, r + 1)

        # now segments fully align with [l, r]
        vals = segs[i:j + 1]

        ok = True
        base = vals[0].v
        for s in vals:
            if s.v != base:
                ok = False
                break

        if not ok:
            continue

        segs[i].v += 1
        segs[i].r = segs[j].r

        for _ in range(j - i):
            segs.pop(i + 1)

        normalize(i)

    print(len(segs))

if __name__ == "__main__":
    solve()
```该代码维护一个按位置排序的最大段列表。 这`find`函数定位哪个段包含给定索引，该索引在分割边界之前使用。 这`split`函数确保查询端点成为精确的段边界，这一点至关重要，因为有关一致性的所有推理都取决于与查询范围对齐的段。 

对齐后，区间范围对应于段列表的一个连续切片。 该代码检查该切片中的所有段是否共享相同的值。 如果不是，则忽略该操作。 否则，它将切片合并为具有递增值的单个段。 

这`normalize`函数在更新后通过与具有相等值的邻居合并来恢复最大值，从而防止随着时间的推移产生碎片。 

一个微妙的实现细节是，在检查一致性之前必须在两端进行分割。 如果仅分割一个边界，则查询范围可能会意外覆盖部分重叠的段并产生不正确的多段检测。

 ## 边缘情况

 一种重要的情况是更新与两个段之间的边界完全匹配。 例如，如果结构是$[1,2,0],[3,5,0]$我们更新$[2,3]$，分裂产生$[1,1,0],[2,2,0],[3,3,0],[4,5,0]$。 中间区域跨越多个段，因此该操作被正确拒绝。 

另一种情况是错误地重复合并折叠结构。 如果更新后我们得到$[1,2,1],[3,4,1]$，并且稍后的操作将它们合并到一个单独的段中，它确保了未来的一致性检查$[1,4]$正确返回true。 如果不合并，算法将错误地检测边界并拒绝有效操作。 

第三种情况是单点更新，例如$[p,p]$。 这些在很大程度上依赖于正确的分裂行为。 如果分割无法隔离该点，则更新可能会错误地影响更大的段并破坏后续检查。
