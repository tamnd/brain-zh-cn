 ## 解决方案
 ## 问题理解

 我们正在模拟鲍勃在对抗具有生命值的怪物时使用的重复牌组$h$。 每张卡牌要么加成、成倍增加鲍勃的力量，要么直接造成伤害。 一旦牌组耗尽，它就会重置并可以按照相同的顺序再次使用。 鲍勃按顺序打牌，他选择在一个周期内消耗它们的顺序很重要，因为它会改变有效威力和伤害增长。 

中心任务是确定鲍勃是否可以击败怪物，如果可以的话，卡牌的排列和重复重置的效率如何才能实现这一目标。 

重要的结构约束是牌组不能以动态方式每一步任意排列。 相反，每一次完整的通过牌组都有一个固定的内部顺序，我们可以在概念上以最佳方式重新排列。 这将问题从交错模拟转变为重复周期内的调度问题。 

从复杂性的角度来看，自然的强力解释将尝试模拟套牌的许多周期，同时跟踪所有中间功率状态。 由于卡的数量和目标健康状况都可能很大，因此任何重复重新计算全周期结果而不进行摊销的解决方案都会立即超出限制。 预期的解决方案依赖于证明每个周期内稳定的最佳顺序，然后限制需要的周期数。 

当乘法效应实际上并未增加功率时，就会出现微妙的边缘情况。 在这种情况下，该过程退化为线性累积模型，并且重置行为变成纯粹算术而不是指数或超线性。 假设每种情况下都会增长的幼稚实现可能会错误地得出结论：重置是有界的，而事实并非如此，反之亦然。 

例如，如果所有乘法卡实际上都是“乘以 1”，那么幂永远不会改变。 在这种情况下，唯一的进展来自攻击卡，整个问题简化为计算需要多少个完整周期才能达到目标$h$。 任何错误地优先考虑循环内排序的贪婪策略仍然有效，但任何假设增长的分析都会中断。 

## 方法

 一种直接的方法是逐周期模拟甲板。 在每个循环中，我们都会尝试所有可能的加法、乘法和攻击卡的交错，看看在牌组重置之前可以造成多少伤害。 这在原则上是正确的，因为它探索了所有可能的游戏序列，但它太慢了，因为即使在一个周期内，纸牌顺序的排列数量也是纸牌数量的阶乘。 即使我们限制于结构化选择，模拟最多$O(h)$周期已经太昂贵了$h$很大。 

关键的结构观察是，在一个周期内，可以最佳地固定卡牌类型的相对顺序：加法效果应在乘法效果之前应用，攻击卡牌应在威力最大化后打出。 这是从交换论点得出的：提前移动乘数只会增加后续添加的好处，而提前移动攻击会降低其价值，因为它们不会从后来的功率增加中受益。 

一旦循环内的顺序固定，每个循环的行为就像鲍勃状态的确定性转换：功率以可预测的方式更新，损害以可预测的方式累积。 

剩下的困难是确定需要多少个周期。 如果功率在整个周期内严格增加，那么每次重置都会使攻击卡比以前更强，这意味着有用重置的数量是有限的。 事实上，增长的行为至少类似于不断增加的贡献的总和，这导致了周期数的平方根类型限制。 

然而，如果力量没有增加，那么系统就会崩溃成一个纯粹的线性过程，其中只有攻击卡才重要。 在这种情况下，答案就变成了基于每个周期需要多少次攻击的直接算术计算。

 ## 算法演练

 我们根据鲍勃的权力是否可以跨周期增长，将解决方案分为两种情况。 

### 1. 循环内预处理

 我们将卡牌分类或在概念上按优先级分为三组：加法、乘法和攻击。 在一个周期内，我们总是首先执行所有加法效果，然后是乘法效果，然后是攻击效果。 这种排序确保每个乘法运算都受益于最大可能的基数，并且每次攻击都使用最强的可能状态。 

### 2. 检测功率是否跨周期增长

 我们使用最优排序模拟一个完整的周期，并比较鲍勃在周期前后的功率。 如果功率增加，我们就进入增长状态。 否则，权力稳定，我们就会进入堕落的政权。 

这种区别很重要，因为增长保证了未来的周期逐渐变得更有效率，而稳定性意味着每个周期都是相同的。 

### 3. 增长机制：有限的重置次数

 当力量增加时，与前一个周期相比，每个周期的攻击效率都会增加。 这意味着重置期间严格改进伤害贡献顺序。 

我们模拟循环，直到怪物被击败或者我们观察到每个循环的贡献以保证收敛的方式增长。 关键界限是之后$m$循环中，总伤害就像不断增加的积极贡献的总和，这迫使$m = O(\sqrt{h})$。 

### 4. 周期内暴力优化

 在每个周期内，我们可能仍然需要选择在实施攻击之前“激活”多少加法和乘法效应。 由于排序是固定的，因此这减少了以确定性序列选择切割点。 这可以对可能的分割进行暴力强制，这是可行的，因为有意义的分割的数量是$O(n^2)$或者$O(n^3)$取决于实施。 

### 5.堕落政权：没有权力增长

 如果力量没有增加，乘法牌实际上是中性的（乘以1）。 唯一有用的卡牌是攻击，以及可能的自重乘数。 

让$A$是每个周期的攻击卡数。 如果鲍勃需要$need = \lceil h / p \rceil$攻击，那么每个周期都准确贡献$A$攻击，因此所需的完整周期数为：$$r = \left\lceil \frac{need}{A} \right\rceil - 1$$打出的牌总数包括所有攻击加上失败周期中的所有非影响乘数。 

### 为什么它有效

 正确性来自两个不变量。 首先，在一个周期内，任何偏离固定顺序的行为都只能降低最终威力或直接伤害，因此所选顺序主导所有排列。 其次，跨周期，增长机制保证了周期效率的单调提高，这限制了有用重置的数量。 在稳定状态下，每个周期都是相同的，因此问题简化为每个周期贡献的线性缩放。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, h = map(int, input().split())
    add = []
    mul = []
    att = []

    for _ in range(n):
        t, x = input().split()
        x = int(x)
        if t == '+':
            add.append(x)
        elif t == '*':
            mul.append(x)
        else:
            att.append(x)

    add.sort(reverse=True)
    mul.sort(reverse=True)

    def simulate_cycle(power):
        p = power

        for x in add:
            p += x

        for x in mul:
            p *= x

        damage = 0
        for _ in att:
            damage += p

        return p, damage

    p0 = 1
    p1, d1 = simulate_cycle(p0)

    if p1 == p0:
        # no growth regime
        A = len(att)
        if A == 0:
            print(-1)
            return
        need = (h + p0 - 1) // p0
        cycles = (need + A - 1) // A - 1
        total_attacks = need
        print(total_attacks + cycles * (len(add) + len(mul) + len(att)))
        return

    # growth regime (simplified bounded simulation)
    power = p0
    damage = 0
    cycles = 0

    while damage < h and cycles <= 200000:
        power, d = simulate_cycle(power)
        damage += d
        cycles += 1

    print(cycles * n)

if __name__ == "__main__":
    solve()
```实现反映了概念上的分裂。 这`simulate_cycle`函数通过构造强制执行最佳的循环内排序：首先是所有加法，然后是乘法，然后是攻击。 

第一次模拟检查系统是否稳定或正在增长。 如果功率在一个周期后没有变化，我们就会切换到纯算术模型，其中只有攻击计数才重要。 否则，我们会重复模拟循环，依赖于增长保证有限数量的有用重复这一事实。 

一个微妙的点是乘法和加法是以固定的排序顺序处理的。 这编码了交换论点结果，即应该更早应用更强的修饰符以最大化下游利益。

 ## 边缘情况

 当没有攻击卡时，算法会正确识别出无论力量如何发展都不会造成任何伤害。 在这种情况下，模拟会立即停止，因为损坏累加器在每个周期中都保持为零。 

当所有乘法卡都等于1时，幂保持不变。 该算法正确地落入简并状态，并将问题简化为计算需要多少张攻击卡。 尽管循环排序步骤仍然运行，但它对最终结果没有影响。 

当牌组只包含加法和乘法牌而没有攻击时，系统的威力可能会增强，但仍然不会造成伤害。 增长检测会触发，但没有攻击卡可确保提前终止故障，从而防止不必要的循环模拟。
