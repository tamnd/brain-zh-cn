 ## 解决方案
 ## 问题理解

 我们正在维持一长串最初没有限制的头寸。 随着时间的推移，节日组织者会放置“标志”，对邻近的范围实施高度限制。 每个符号都有一个高度，并通过施加下限来影响其区间内的每个位置：在某个范围上应用高度为 H 的符号后，该范围内的每个位置的有效高度必须至少为 H。 

因为多个标志可以重叠，所以每个位置都受到到达它的最强约束的控制，这意味着它的最终高度是覆盖它的所有标志的最大值。 因此，我们需要维护的结构是范围更新下的动态值范围，形式为“将 [l, r] 中的所有内容提高到至少 H”，以及要求当前状态聚合信息的查询，通常类似于范围或整个数组中的最小值。 

挑战在于范围更新和查询都是交错的，并且有足够多的操作，天真地重新计算受影响的段会太慢。 描述暗示了具有惰性传播的标准线段树，其中更新不是相加的，而是使用最大操作应用单调变换。 

从复杂性的角度来看，在典型的 Codeforces 约束下，输入大小意味着高达 200,000 到 500,000 次操作。 任何每次更新涉及 O(n) 个元素的解决方案在最坏的情况下都会立即退化为 10^10 次操作，这是不可行的。 即使每次操作 O(n log n) 也会太慢，因此每次更新和每次查询我们都需要 O(log n) 。 

微妙的边缘情况来自于高度降低或增加的重叠更新。 例如，对 [1, 10] 应用高度 5 更新，然后对 [3, 7] 应用高度 2 更新，不应减少任何内容，因为约束只会增强。 一个简单的分配而不是获取最大值的实现会在第二次更新中错误地降低值。 

当更新完全重叠并以任意顺序处理时，会出现另一种边缘情况。 例如，应用 (l=1,r=5,H=10) 后跟 (l=1,r=5,H=7) 必须将段处处保留为 10，而不是 7。任何覆盖而不是与 max 合并的方法在这里都会失败。 

## 方法

 蛮力的想法很简单。 维护一个大小为 n 的数组。 对于每次更新，迭代受影响的范围并将每个值更新为 max(当前值，H)。 对于每个查询，扫描请求的范围并计算最小值或其他聚合。 这是正确的，因为它直接模拟约束传播，但每次更新可能会涉及 O(n) 个元素，每个查询也可能会涉及 O(n) 个元素。 如果运算次数达到 O(n)，则变为 O(n^2)，这远远超出了任何可行的限制。 

关键的观察是，应用于每个段的操作是单调和幂等的：多次应用约束 H 相当于仅应​​用最大 H。此结构与具有延迟传播的线段树匹配，其中每个节点在其段中存储当前最小值，并且挂起的更新“提升到至少 H”。 当多个更新重叠时，可以使用 max 合并它们，并且永远不需要撤消或减少它们。 

这也与动态连接类型问题的更一般模式有关：随着时间的推移，通过回滚或持久性进行分而治之。 在这里，我们不是回滚 DSU 状态，而是回滚线段树修改，或者我们通过使用惰性传播完全避免回滚，因为更新是单调的并且不会以破坏性方式发生冲突。 

线段树解决方案之所以有效，是因为每次更新仅影响 O(log n) 个节点，并且每个节点更新都是对存储值和惰性标记的常数时间最大操作。 这将整个过程减少到 O((n + q) log n)。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(nq) | O(n) | 太慢了 |
 | 线段树Lazy Max | O((n + q) log n) | O((n + q) log n) | O(n) | 已接受 |

 ## 算法演练

 我们在数组上构建一棵线段树，其中每个节点存储其区间内的最小值。 每次更新都表示强制执行一个约束“[l, r] 中的所有值必须至少为 H”。 

1. 构建一棵线段树，其中每个叶子都从 0 开始，因为最初没有符号强加任何高度约束。 内部节点存储其子节点的最小值。 这使我们能够直接回答范围最小查询。 
2.对于范围更新[l,r,H​​]，我们遍历线段树。 每当节点间隔被完全覆盖时，我们通过将其设置为 max(current_min, H) 来更新其存储的最小值。 这反映了这样一个事实：该部分中的所有元素都必须遵守迄今为止所见的最强约束。 
3. 如果一个节点仅被部分覆盖，我们通过递归到其子节点来向下推送更新。 这确保了更精细的段仍然正确地反映重叠约束。 
4. 我们在每个节点维护一个惰性值，表示待处理的下界约束。 如果多个更新到达同一节点，我们使用 max 将它们组合起来，因为只有最强的约束才重要。 
5. 当将惰性值推送给子级时，我们再次应用最大传播：如果需要，每个子级的最小值都会提高，并且其惰性标记也会相应更新。 
6. 对于查询，我们沿线段树下降并组合相关线段的最小值。 由于每个节点始终代表所有应用约束下的正确最小值，因此查询会返回正确的结果而无需重新计算。 

关键思想是树始终代表迄今为止应用的所有约束的当前下包络。 

### 为什么它有效

 在任何时间点，每个位置的值恰好是覆盖该位置的所有更新的最大高度。 线段树隐式地维护了这个不变性：每个节点在应用所有相关的最大约束后存储其线段上的最小值。 由于更新和惰性传播都只会增加值，因此任何操作都无法使先前的正确性失效。 最大操作是关联且单调的，这保证了跨段分割更新或稍后合并它们会产生相同的最终状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class SegTree:
    def __init__(self, n):
        self.n = n
        self.mn = [0] * (4 * n)
        self.lazy = [0] * (4 * n)

    def apply(self, idx, val):
        if val > self.mn[idx]:
            self.mn[idx] = val
        if val > self.lazy[idx]:
            self.lazy[idx] = val

    def push(self, idx):
        if self.lazy[idx]:
            v = self.lazy[idx]
            self.apply(idx * 2, v)
            self.apply(idx * 2 + 1, v)
            self.lazy[idx] = 0

    def pull(self, idx):
        self.mn[idx] = min(self.mn[idx * 2], self.mn[idx * 2 + 1])

    def update(self, idx, l, r, ql, qr, val):
        if ql <= l and r <= qr:
            self.apply(idx, val)
            return
        self.push(idx)
        mid = (l + r) // 2
        if ql <= mid:
            self.update(idx * 2, l, mid, ql, qr, val)
        if qr > mid:
            self.update(idx * 2 + 1, mid + 1, r, ql, qr, val)
        self.pull(idx)

    def query(self, idx, l, r, ql, qr):
        if ql <= l and r <= qr:
            return self.mn[idx]
        self.push(idx)
        mid = (l + r) // 2
        res = float('inf')
        if ql <= mid:
            res = min(res, self.query(idx * 2, l, mid, ql, qr))
        if qr > mid:
            res = min(res, self.query(idx * 2 + 1, mid + 1, r, ql, qr))
        return res

def solve():
    n, q = map(int, input().split())
    st = SegTree(n)

    out = []
    for _ in range(q):
        op = list(map(int, input().split()))
        if op[0] == 1:
            l, r, h = op[1], op[2], op[3]
            st.update(1, 1, n, l, r, h)
        else:
            l, r = op[1], op[2]
            out.append(str(st.query(1, 1, n, l, r)))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```核心实现选择是更新永远不会覆盖值，它们仅使用 max 来提高它们。 惰性数组并不代表完整的操作历史记录，仅代表最强的待处理下限。 这使得传播成本低廉，并防止多个更新重叠时的重复工作。 

推送操作至关重要，因为它确保部分覆盖的节点不会错误地隐藏来自更深段的更强约束。

 ## 边缘情况

 一个关键的边缘情况是更新以高度降序到达同一段。 例如，在相同的间隔上应用高度 10 更新，然后应用高度 3 必须使线段保持在 10 不变。线段树可以正确处理此问题，因为节点值和惰性标记都仅通过 max 进行更新，因此较小的更新将被完全忽略。 

另一种边缘情况是更新仅部分重叠。 考虑高度为 4 的 [1,5]，然后考虑高度为 6 的 [3,7]。中间区域必须变为 6，而其余区域保持在 4 或 0。递归确保只有完全覆盖的节点在恒定时间内更新，而部分覆盖的节点会被分割，直到达到正确的粒度，从而保持跨边界的正确性。 

最后的边缘情况是对未更改的段进行重复查询。 由于除了延迟传播之外不会发生重新计算，因此重复查询不会降低性能，并且存储的段值在无需重新处理的情况下仍然有效。
