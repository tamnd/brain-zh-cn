 ## 解决方案
 ## 问题理解

 我们得到了一系列的每日温度和一件只能在有限的时间内使用的外套。 一旦连续佩戴几天，它就会变脏，必须清洗后才能再次使用。 您可以自由选择洗涤日期，包括提前洗涤，但在您不穿外套的任何一天，您都会暴露在天气中。 

我们的目标不是尽量减少不穿外套的天数或任何类似的添加剂。 相反，我们关心所有不穿外套的日子中最糟糕的一天。 我们希望安排穿着和洗涤时间，以便我们在没有穿外套的情况下面对的最冷的一天尽可能温暖。 

查看目标的另一种方法是最大化温度阈值 T，这样我们就可以确保：温度低于 T 的每一天都可以安全地被外套覆盖，而温度等于或高于 T 的日子只允许在遵守洗涤约束的受控突发情况下不穿外套。 

一个关键的结构性限制来自外套使用限制 C。任何连续的不穿外套天数超过 C 的时间都是不可能维持的，因为这意味着连续穿外套 C 天以上而不清洗或中断。 

输入只是 N 天的温度列表和描述最大连续涂层使用量的整数 C。 输出是一个整数，表示在最佳策略下无涂层天数中可达到的最佳最低温度。 

主要困难是我们没有独立选择个别日子。 这种限制是连续几天的，因此关于无外套日的当地决定可能会迫使全球范围内不可行。 

当一个人试图贪婪地将所有“糟糕”（温暖）的日子标记为不穿外套而不检查连续结构时，就会出现一个微妙的失败案例。 

例如，考虑 N = 6、C = 2 和温度`[5, 5, 5, 5, 5, 5]`。 如果我们决定阈值 T = 5，所有日子都变得无外套，形成长度为 6 的单个片段。由于 C = 2，这是无效的。 仅计算不穿外套总天数的幼稚方法会错误地接受此配置。 

当无外套日被分成多个部分时，就会出现另一种边缘情况。 例如，`[10, -100, 10]`C = 1 且阈值 T = 10 产生无涂层图案`[1, 0, 1]`，这是有效的，因为即使无涂层总数为 2，也没有段超过长度 1。任何忽略连续性的解决方案都会错误地评估可行性。 

这些观察结果表明，可行性完全取决于连续的游程长度，而不是全局计数。 

## 方法

 一种直接的方法是猜测阈值 T 并检查是否可以强制所有无涂层日的温度至少为 T，而不违反连续无涂层伸展的限制。 对于固定的 T，我们将温度至少为 T 的每一天标记为无外套。 然后我们扫描数组并测量这些天的最长连续运行时间。 如果该运行超过 C，则阈值是不可能的。 

这种强力方法之所以有效，是因为条件仅取决于每天是否超过阈值，因此每个候选 T 都会对数组进行二元分类。 然而，尝试所有可能的 T 值需要最多 101 次检查（因为温度限制在 -50 到 50 之间），每次扫描 N 天。 这导致大约 100N 次操作，对于 N 达到 10^5 来说这是可以接受的。 

如果温度不受限制，我们可以对唯一值或二分搜索 T 进行排序。可行性检查是单调的：如果阈值 T 有效，则任何较低的阈值也有效，因为变得无涂层的天数更少，永远不会增加运行长度。 这种单调性允许二分查找的时间复杂度为 O(N log N)。 

不同的视角完全避免了门槛。 我们可以在数组上滑动一个大小为 C + 1 的窗口。 任何这样的窗口都保证在任何有效的调度策略下，其中至少一天必须不穿外套，因为外套不能连续穿着超过 C 天。 对于每个窗户，我们计算其最高温度，如果该窗户强制白天不穿外套，则该温度代表最糟糕的暴露程度。 答案成为这些最大值的所有窗口中的最小值。 这将问题转换为经典的滑动窗口极大最小值结构，可通过单调双端队列解决。 

另一种看待该结构的方式是从一个完全没有外套的世界开始，然后逐渐“保护”最冷的日子。 当我们重新引入从最低温度向上的保护时，我们跟踪连续的无涂层部分。 当所有段收缩至最大 C 长度时，我们就达到了最佳截止温度。 

所有这些观点都是等效的。 滑动窗口和阈值模拟是最直接、最容易实现的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解超过阈值 | O(100N) | O(1) | O(1) | 已接受 |
 | 滑动窗口（双端队列）| O(N) | O(N) | 已接受 |

 ## 算法演练

 我们使用滑动窗口解释，避免显式阈值枚举并直接计算限制结构。 

### 步骤

 1. 考虑数组中长度为 C + 1 的每个连续段。 采用此窗口大小的原因是，任何违反外套约束的行为都必须出现在某个区域内，其中外套使用量将超过 C，这恰好对应于 C + 1 连续无外套天数。 
2. 对于每个这样的窗口，计算其内部的最高温度。 如果整个窗口被迫包含至少一天无涂层的时间，则该值代表“最差暴露点”。 
3. 在所有这些窗口最大值之上保持运行最小值。 这种聚合捕获了最佳的全局保证，因为每个有效的调度都必须遵守每个局部约束窗口。 
4. 使用单调双端队列有效计算窗口最大值，该单调双端队列按温度降序存储元素索引。 当我们滑动窗口时，我们会删除过时的索引并将前面的索引保持为最大值。 
5.处理完所有窗口后，输出最小记录的最大值。 

每个步骤都是由这样的观察驱动的：可行性是由最严重的连续违规控制的，而不是全球分布。 

### 为什么它有效

任何有效的策略都必须确保在长度为 C + 1 的每个段中，至少有一天是无外套的。 否则的话，这件大衣会连续使用C天以上，这是不可能的。 这迫使每个这样的部分“暴露”至少一个温度。 某个部分中最差的暴露温度由其最大值决定，因为我们关心的是不可避免的最高暴露温度。 最小化所有段可确保我们选择一个阈值，以避免阵列中任何地方不可避免的过冷暴露。 

这将全局调度问题转换为局部窗口约束问题，其中限制因素是最差窗口。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, c = map(int, input().split())
    a = list(map(int, input().split()))
    
    if c >= n:
        print(max(a))
        return

    k = c + 1
    dq = deque()
    best = float('inf')

    for i, x in enumerate(a):
        while dq and dq[-1][1] <= x:
            dq.pop()
        dq.append((i, x))

        while dq and dq[0][0] <= i - k:
            dq.popleft()

        if i >= k - 1:
            best = min(best, dq[0][1])

    print(best)

if __name__ == "__main__":
    solve()
```该实现为每个大小为 C + 1 的滑动窗口维护一个候选最大值的双端队列。每个元素都与其索引一起存储，因此我们可以丢弃落在当前窗口之外的元素。 双端队列保持温度降序排列，因此前面始终给出当前窗口的最大值。 

特殊情况 C >= N 单独处理。 在这种情况下，外套永远不会因为连续使用限制而变得无法使用，因此我们可以有效地选择不以限制性方式清洗，并且答案会降低到阵列中的最高温度，因为除非我们选择其他方式，否则我们可以避免无涂层。 

最微妙的部分是窗户维护：状况`i >= k - 1`确保我们仅在第一个完整窗口形成后才开始记录有效窗口，并且索引剪枝确保滑动时的正确性。

 ## 边缘情况

 一种极端情况是 C 为零，这意味着您根本无法连续几天穿着这件外套。 用于输入`N = 6, C = 0, A = [1, 2, 3, 4, 5, 6]`，该算法使用窗口大小 1。每个窗口最大值只是元素本身，因此所有窗口的最小值为 1。双端队列独立处理每个元素，并且不会出现邻接问题，因为每个窗口都是隔离的。 这与每天都成为单独的强制无衣暴露这一事实相符。 

另一种边缘情况是当 C 非常大时，例如`N = 5, C = 10, A = [3, 1, 4, 1, 5]`。 在这里，条件永远不会绑定，因此数组内部不完全存在大小为 C + 1 的窗口。 算法直接返回最高温度，即5。特殊检查`if c >= n`确保我们不运行空窗口逻辑。 

第三种情况是统一数组，例如`A = [7, 7, 7, 7]`与任何 C。每个窗口最大值为 7，因此运行最小值仍为 7。双端队列始终折叠为单个代表性索引，并且无论分段如何，输出都保持稳定。
