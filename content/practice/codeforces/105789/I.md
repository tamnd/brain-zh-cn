 ## 解决方案
 ## 问题理解

 我们给出了一个动态排列$P$大小的$K$和另一个数组$A$。 排列定义了无限周期序列$P^\infty$，通过重复形成$P$不休。 数组$A$也是无限延伸的，但与$P$，它不一定按值模式重复； 相反，它的周期性扩展$A^\infty$是使用原始问题的结构规则来定义的。 

对于每个查询，我们从概念上比较这两个无限序列，并询问在两个无限序列中出现相同的最长连续段的长度$P^\infty$和$A^\infty$。 查询的答案是此类公共子串的最大长度。 

关键的困难在于这两个对象都是无限的，因此直接模拟是不可能的。 重复结构$P$很简单，因为它是一个排列，但是$A$仅受周期性条件的约束$K$，这不是立即本地的。 

典型的 Codeforces 动态排列问题所隐含的约束足够大，以至于任何迭代所有子字符串或直接比较无限展开的解决方案都是不可行的。 对所有子串的长度进行简单比较$n$每个查询都是二次的，这是远远超出可接受的范围的。 

当两个无限序列以完美的周期性方式对齐时，就会出现微妙的边缘情况。 在这种情况下，公共子串是无界的，并且答案在概念上是无限的。 总是计算有限最大值的简单实现会错误地限制这种情况。 

当元素位于$A$不存在于$P$。 在这种情况下，匹配段必须立即中断，并且任何将匹配扩展到此类不匹配之外的尝试都会导致不正确的过度计数。 

## 方法

 直接方法将尝试扩大两者$P^\infty$和$A^\infty$并比较从每个位置开始的所有子字符串。 即使限制对有界长度的子字符串的关注，这也会导致爆炸：对于每个起始索引，扩展比较可能会花费$O(K)$，并且在所有位置上重复会导致每个查询的二次或更差的行为。 

关键的观察来自周期性结构。 排列$P$在其无限重复中引发严格的循环：位置相差数倍$K$始终包含相同的值。 同样，在$P$，由于所有元素都是不同的，因此值的位置唯一确定其在循环中的下一个值。 

上$A$从侧面看，结构比看起来更受限制。 如果我们观察到足够长的匹配子串$A^\infty$和$P^\infty$，我们正在有效地强制价值认同和循环顺序的对齐。 一旦匹配超过长度$K$，它迫使循环结构完全对齐。 这导致了二分法：要么由于完全周期兼容性而答案是无限的，要么它受到以下限制：$K$。 

这种减少是至关重要的简化。 我们不需要推理无限序列，而只需要考虑最多长度的子串$K$。 任何更长的有效匹配都意味着完全的结构兼容性，可以通过循环一致性进行检查。 

一旦有界，我们可以将问题转换为滑动窗口式计算，通过双倍表示$A$，表示为$A_2$。 加倍可确保每个有效窗口的长度达到$K$无限序列中的 出现为有限数组中的连续段。 

对于每个右端点$i$在$A_2$，我们计算最左边的有效开始$L_i$，其中子字符串结尾于$i$与结构保持一致$P$。 这减少了在排列循环中维持邻接连续性以及在对齐中断时使段无效的任务。 

最好不是通过索引而是通过后继关系来处理排列。 对于每个值，我们维护其在循环中的位置$P$，所以我们可以测试是否有连续的元素$A$对应于循环中的连续元素。 

暴力破解会失败，因为它会反复重新检查实际本地确定的结构。 有效性仅取决于是否存在连续元素的观察结果$A$遵循循环顺序将问题简化为具有恒定时间检查的线性扫描。

 | ---| ---| ---| ---|
 | 蛮力 |$O(nK)$每个查询|$O(1)$| 太慢了 |
 | 最佳 |$O(n)$每次更新/查询摊销 |$O(n)$| 已接受 |

 ## 算法演练

 我们通过扫描有限表示来处理每个查询$A_2$，其中包含$A$与自身连接。 这确保了长度达到的任何有效子字符串$K$在无限扩展中完全出现在数组内部。 

1. 构建位置图$P$，其中每个值都映射到其在排列循环中的索引。 这允许循环邻接的恒定时间比较。 
2. 维护数组$L$， 在哪里$L[i]$存储结束于的有效子字符串的最左边的起始索引$i$。 
3. 迭代索引$i$在$A_2$。 如果当前元素$A[i]$不存在于$P$，那么没有有效的子字符串可以结束于$i$，所以我们重置$L[i]$到$i+1$。 这反映了不匹配会立即中断任何延续。 
4.如果$A[i]$是在$P$，将其在循环中的位置与$A[i-1]$。 如果它们是连续模数$K$，则子串可以扩展，所以设$L[i] = L[i-1]$。 这保留了结束于的最长有效窗口$i-1$。 
5. 否则，循环中的邻接关系被破坏，因此我们在以下位置开始一个新的有效段$i$， 环境$L[i] = i$。 
6. 跟踪最大窗口长度$i - L[i] + 1$跨越所有职位。 
7. 如果在分析过程中我们发现整个结构在$P$和$A$，我们返回无穷大，否则返回最大计算值。 

### 为什么它有效

 正确性依赖于这样一个事实：在一个排列循环中，邻接性完全决定了相对顺序。 由于所有元素都是不同的，一旦我们知道一个元素在循环中的位置，就可以唯一确定下一个有效元素。 中的任何有效子字符串$A^\infty$匹配$P^\infty$必须在每一步都保留这种邻接结构。 因此，每个有效段完全对应于沿着循环的连续行走$P$，任何邻接中断都会立即破坏可扩展性。 这使得滑动窗口表征准确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    A = list(map(int, input().split()))
    P = list(map(int, input().split()))

    pos = {}
    for i, v in enumerate(P):
        pos[v] = i

    A2 = A + A

    ans = 0
    L = -1

    for i in range(len(A2)):
        if A2[i] not in pos:
            L = i + 1
            continue

        if i > 0 and A2[i - 1] in pos:
            if (pos[A2[i - 1]] + 1) % k == pos[A2[i]]:
                # extend
                pass
            else:
                L = i
        else:
            L = i

        if L != -1:
            ans = max(ans, i - L + 1)

    print(ans if ans <= k else "infinity")

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现构建了从值到它们在排列中的位置的直接映射，这是检查循环邻接的核心工具。 构造双倍数组是为了避免显式处理环绕逻辑，因为任何长度最多的有效子字符串$K$必须完全位于两份副本之一内。 

变量$L$跟踪当前有效窗口的开始。 每当发生不匹配时，无论是由于缺少值还是循环邻接中断，我们都会重置$L$到下一个位置。 最大窗口长度不断更新。 

最终检查将最佳有限段与$K$，因为任何更长的值都意味着无限周期性对齐情况。

 ## 边缘情况

 当出现临界边缘情况时$A$包含不存在于的元素$P$。 在这种情况下，任何包含此类元素的窗口都必定是无效的。 该算法通过在遇到缺失值时立即重置左边界来处理此问题，从而防止过度扩展。 

另一个边缘情况是当$A$正好是一个旋转$P$。 在这种情况下，每个连续的对都满足循环邻接性，因此窗口永远不会重置。 该算法产生的最大长度超过$K$，这会触发无限答案条件。 

最后一个边缘情况是$A$在有效和无效转换之间交替。 例如，一个序列匹配几个步骤的循环，然后中断，强制多次重置$L$，并且实现正确地隔离每个有效段，而不会错误地合并它们。
