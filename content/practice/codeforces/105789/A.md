 ## 解决方案
 ## 问题理解

 我们正在使用一个有向图，其边缘用字符标记。 任务不是回答通常意义上的可达性，而是发现可以通过其边标签序列形成回文的游走连接的所有顶点对。 

回文条件耦合路径的两端。 如果我们知道有一个回文步行`u`到`v`，那么该遍历的第一个和最后一个字符必须匹配，并且删除它们会在下一对向内的顶点之间留下另一个回文遍历。 这表明回文连接不是单个路径的本地连接，而是通过两端的对称扩展进行传播。 

输入图可能足够大，以至于不可能迭代所有路径。 即使枚举所有顶点对也已经是二次的，并且语句中描述的算法显式地构造了对顶点对的过程`(u, v)`而不是单个顶点或边。 这立即表明状态空间是顶点对的集合，这可能是`O(n^2)`。 

一个天真的解释是尝试显式构建所有回文行走或在状态上运行 BFS`(u, v)`其中过渡取决于匹配的边缘标签。 这已经暗示了最坏情况的爆炸，因为每一对都可以生成许多新的对。 

一种微妙的边缘情况来自自循环和琐碎路径。 每个顶点`(u, u)`最初是有效的，因为空行走是回文。 另一种边缘情况是多个边缘共享相同的标签，因为它们可以生成多个相同的转换。 粗心的实现可能会重复工作或重新添加已处理的对，从而导致崩溃。 

例如，如果我们有一个有两个顶点的图`1 -> 2`贴上标签`a`和`2 -> 1`贴上标签`a`， 然后`(1, 1)`和`(2, 2)`都是微不足道的，并且`(1, 2)`和`(2, 1)`立即连接。 从那里开始，扩展规则将通过传入和传出边缘重复确认相同的配对，但我们必须确保不会重新处理已经看到的对。 

输出只是不同顶点对的数量`(u, v)`和`u != v`通过这个关闭过程发现的。 

## 方法

 蛮力观点是对待每一对`(u, v)`作为一种状态并尝试直接模拟该过程。 我们从一切开始`(u, u)`和所有边缘`(u, v)`作为初始状态。 然后我们反复尝试扩展：对于每个当前对`(u, v)`，我们查看每个传入的边`u`和每个传出边缘`v`。 如果标签匹配，我们生成一个新的对`(a, b)`。 

这是正确的，因为它准确地反映了回文行走的递归分解：匹配外部字符将问题减少为内部较小的回文行走。 然而，这种模拟可以重复扫描同一对的相同邻接信息，并且存在潜在的问题`O(n^2)`对。 每对扩展成本`deg_in(u) * deg_out(v)`，这在密集图中会导致灾难性的重复。 

关键的观察是该算法本质上是对顶点上的二元关系的闭包计算。 每对`(u, v)`最多插入一次，当处理它时，我们只尝试生成新的对。 繁重的工作在于将传入和传出边缘与相同的标签进行匹配。 我们不考虑路径，而是考虑将两个半步结合起来：向后退一步`u`并向前迈出了一步`v`，通过标签相等性同步。 

这将问题转化为按标签分组的图边上的两指针扩展。 每条边仅通过其度贡献参与配对，并且在所有对中，总工作受到所有顶点总和的限制`indeg(u)`次`outdeg(v)`聚集在整个结构中，折叠成`O(M^2)`在最坏的情况下，但在实践中是通过计算边对交互而不是对状态来控制的。 

关键的结构转变是认识到我们不是枚举路径，而是枚举兼容的边对，并且每个成功的匹配对应于闭包中的唯一生成事件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 无忧无虑地暴力破解 BFS | O(n²·deg²) | O(n²·deg²) | O(n²) | 太慢了 |
 | 通过边缘配对优化对闭合 | O(平方米) | O(n²) | 已接受 |

 ## 算法演练

 我们维护一个顶点对队列`(u, v)`已知它们通过回文步行连接。 我们还维护一个布尔表`seen[u][v]`以确保每对都被处理一次。 

1. 用所有对初始化队列`(u, u)`。 这些对应于空回文，因为停留在顶点形成有效的平凡行走。 
2.同样用所有有向边初始化队列`(u, v)`。 任何单边都形成长度为一的回文，因此这些边立即有效。 
3. 当队列不为空时，弹出一对`(u, v)`。 我们将此视为确认存在从`u`到`v`。 
4. 对于这一对，迭代每个传入边`(a -> u)`带标签`c1`。 同时，迭代每个传出边`(v -> b)`带标签`c2`。 
5. 每当`c1 == c2`，我们可以形成一个新的回文步行`a`到`b`通过包装现有的回文行走`(u, v)`两端具有匹配的边缘。 这会产生一个候选对`(a, b)`。 
6.如果`(a, b)`以前没有见过，将其标记为已见过并将其推入队列。 
7. 继续，直到无法生成新的对。 最终答案是不同对的数量`(u, v)`和`u != v`那些曾经被标记为见过的。 

### 为什么它有效

 每个回文行走都会自然分解为外部匹配边缘和内部回文子行走。 如果步行从`a`到`b`，它的第一条边`(a -> u)`必须匹配最后一条边`(v -> b)`在标签和方向上。 删除这些边缘会留下有效的回文步行`u`到`v`。 这表明每个有效对都是通过重复扩展已知的有效对来生成的。 相反，每个生成的对都对应于一个有效的包装步骤，因此不会引入无效的对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import deque, defaultdict

def solve():
    n, m = map(int, input().split())
    
    in_edges = [[] for _ in range(n)]
    out_edges = [[] for _ in range(n)]
    
    edges = []
    
    for _ in range(m):
        u, v, c = input().split()
        u = int(u) - 1
        v = int(v) - 1
        in_edges[v].append((u, c))
        out_edges[u].append((v, c))
        edges.append((u, v))
    
    seen = [[False] * n for _ in range(n)]
    q = deque()
    
    for i in range(n):
        if not seen[i][i]:
            seen[i][i] = True
            q.append((i, i))
    
    for u, v in edges:
        if not seen[u][v]:
            seen[u][v] = True
            q.append((u, v))
    
    while q:
        u, v = q.popleft()
        
        for a, c1 in in_edges[u]:
            for b, c2 in out_edges[v]:
                if c1 == c2:
                    if not seen[a][b]:
                        seen[a][b] = True
                        q.append((a, b))
    
    ans = 0
    for i in range(n):
        for j in range(n):
            if i != j and seen[i][j]:
                ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```实现直接反映了关闭过程。 邻接被分为传入列表和传出列表，以便每个扩展步骤都可以有效地配对边缘。 这`seen`矩阵可以防止重复处理相同的状态，这一点至关重要，因为如果没有它，同一对可能会通过不同的扩展路径重复生成。 

最后的计数步骤不包括对角线对，因为`(u, u)`是微不足道的，不包含在输出定义中。 

## 工作示例

 考虑一个具有三个顶点和标记边的小图：`1 -> 2 (a)`,`2 -> 3 (a)`， 和`3 -> 2 (a)`。 

我们初始化`(1,1)`,`(2,2)`,`(3,3)`和直接边缘`(1,2)`,`(2,3)`,`(3,2)`。 

| 步骤| 对 (u,v) | 行动| 新双 |
 | ---| ---| ---| ---|
 | 初始化| (1,1),(2,2),(3,3) | 对角线入队| 无 |
 | 初始化| (1,2),(2,3),(3,2) | 入队边缘| 无 |
 | 流程| (2,3) | 通过标签 a | 匹配传入/传出 (1,2) 通过 1->2 和 3->2 |
 | 流程| (3,2) | 对称展开| 没有新的|
 | 流程| (1,2) | 没有更多匹配 | 无 |

 这演示了回文结构如何同时向后和向前传播，链接不是直接边缘的路径。 

现在考虑一个图，其中所有边共享相同的标签`a`：完整的双向链`1 <-> 2 <-> 3`。 

通过重复环绕，每对都变得可达，因为任何传入边都可以匹配任何传出边。 队列很快就饱和了`n^2`对，展示了为什么算法必须依赖重复数据删除。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(平方米) | 每次扩展都会比较传入和传出的边缘； 总的成对边相互作用受到二次边组合的限制 |
 | 空间| O(n²) | 布尔表存储每个顶点对是否已被看到 |

 该算法的结构确保每对最多入队一次，并且每次入队触发的工作与局部度积成正比。 由于边交互的总数受到边组合的限制，因此整个过程保持在边数的二次边界内。

 ## 边缘情况

 一种边缘情况是仅通过初始化隐式自循环的图。 该算法首先插入所有`(u, u)`即使不存在边缘。 由于输出不包括对角线对，因此图`n`孤立的顶点应该返回`0`。 队列进程`(u, u)`但找不到匹配的传入和传出边缘，因此不会生成新的对。 

另一种边缘情况是多个边缘共享相同的端点和标签。 例如，如果有许多平行边`u -> v`贴上标签`a`，每个都可以重复触发扩展。 这`seen`表确保`(u, v)`仅处理一次，防止重复边缘的重复二次放大。 

最终的边缘情况是一个强连接图，其中所有边缘共享相同的标签。 在这种情况下，每个传入边都可以在每个步骤中与每个传出边配对。 该算法仍然正确终止，因为每对`(u, v)`仅插入一次，对的总数受以下限制`n^2`，尽管中间匹配工作很密集。
