 ## 解决方案
 ## 问题理解

 我们在严格的结构规则下计算二叉树的形状。 每个节点要么有两个子树，要么没有，重要的不是键或标签，而是每个子树中有多少个节点。 

仅当在每个节点上，其左子树和右子树本身都是有效树并且它们的大小几乎相等时，才允许存在一棵树，这意味着节点数之差最多为 1。 此约束是递归应用的，因此有效的树强制将其总节点数非常严格地分解为左右部分。 

对于每个测试用例，值 n 描述整个树必须包含多少个节点。 任务是计算对于该确切的 n 存在多少种不同的有效树形状。 答案需要模 2^64，这实际上意味着算术是通过无符号 64 位溢出行为执行的。 

这些约束很不寻常，因为 n 是一个 64 位整数，最大约为 10^19，并且测试用例的数量可能高达一百万。 这会立即排除任何尝试构建最多 n 个表或针对每个查询线性迭代最多 n 个所有值的方法。 即使是对数的每状态 DP 也必须仔细构建，因为状态空间本身只能根据需要进行探索。 

即使对于较小的 n，试图枚举所有树的天真的解释也会失败，因为二叉树形状的数量呈指数增长。 另一个常见错误是尝试在从 0 到 n 的所有大小上使用标准 DP，当 n 本身可能很大并且跨查询稀疏时，这是不可能的。 

值非常小时会出现微妙的边缘情况。 当 n = 0 时，树为空并被定义为有效。 对于 n = 1，只有一棵树由单个节点组成。 任何重复都必须尊重这些基本情况，否则它将错误计算所有较大的值。 

## 方法

 直接的暴力方法会尝试构造每个具有 n 个节点的二叉树，并检查它是否满足平衡条件。 对于每棵树，验证有效性需要自下而上计算子树大小，并且具有 n 个节点的形状数量已经是指数级的，大约是 Catalan 数。 即使 n 约为 20，这也会很快超出可行性。 

关键的观察是根部的条件完全决定了树的分裂方式。 如果一棵树有 n 个节点，则删除根叶子 n − 1 个节点，在左子树和右子树之间分裂。 该约束强制这两个子树大小几乎相等，因此根据 n − 1 的奇偶校验，只有一到两个有效的分割。 

这将问题变成了整数大小的递归问题，而不是结构枚举问题。 每个状态 n 仅取决于恒定数量的较小状态，特别是对应于尽可能均匀地分割 n - 1 的两个值。 这使得递归树的深度呈对数，因为每一步都会将问题大小大约减少一半。 

困难从组合转移到计算递归定义函数的许多值，其中每个值都取决于以前未见过的值。 由于 n 很大且分散，因此使用哈希图进行记忆就成为自然的工具。

 | --- | --- | --- | --- |

 | n | 上的记忆递归 每个状态 O(log n)，摊销 O(T log n) | O(#不同状态) | 已接受 |

 ## 算法演练

 我们将函数 f(n) 定义为具有 n 个节点的有效超平衡树的数量。

1. 建立基础案例。 一棵空树提供一个有效结构，因此 f(0) = 1。单个节点也只有一种配置，因此 f(1) = 1。 
2. 对于任何大于 1 的 n，考虑根节点。 删除它留下 n − 1 个节点，这些节点必须分为左尺寸 L 和右尺寸 R，使得 L + R = n − 1 和 |L − R| ≤ 1。 
3. 计算 s = n − 1 并确定其奇偶性。 如果 s 是偶数，则唯一可能的分割是 L = R = s / 2。树的总数是 f(L) 乘以 f(R)，因为左子树和右子树是独立的。 
4. 如果 s 是奇数，则写为 s = 2k + 1。唯一有效的平衡分割是 (k, k + 1) 和 (k + 1, k)。 这两种情况在结构上是对称的，但在有序树中是不同的，因此总计数变为 2 × f(k) × f(k + 1)。 
5. 由于 n 可能非常大，因此请延迟计算 f(n)。 当请求值时，仅递归计算所需的子问题并将结果存储在字典中以避免重新计算。 
6. 执行所有模 2^64 的乘法。 在 Python 中，这是通过在每次算术运算后使用 (1 << 64) − 1 进行掩码来实现的。 

### 为什么它有效

 每棵有效树都由其根分为左子树和右子树来唯一确定。 平衡约束消除了除最多两种分割剩余节点的方法之外的所有方法。 由于两个子树必须独立地满足相同的规则，因此总计数会分解为独立子问题的乘积。 递归严格减少n，并且由于记忆，每个状态都计算一次，因此不会错过有效配置，也不会引入无效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD_MASK = (1 << 64) - 1
sys.setrecursionlimit(10**7)

memo = {}

def f(n):
    if n in memo:
        return memo[n]
    if n == 0 or n == 1:
        memo[n] = 1
        return 1

    s = n - 1
    if s % 2 == 0:
        k = s // 2
        res = (f(k) * f(k)) & MOD_MASK
    else:
        k = s // 2
        res = (2 * f(k) * f(k + 1)) & MOD_MASK

    memo[n] = res
    return res

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append(str(f(n)))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```解决方案围绕函数 f(n) 的记忆递归。 每个调用要么返回一个缓存值，要么将 n 分成两个严格较小的子问题，这些子问题是通过尽可能均匀地分割 n − 1 得出的。 使用 MOD_MASK 进行掩码强制执行 64 位溢出语义，这是必需的，因为否则 Python 整数会变得任意大。 

一个微妙的点是递归深度遵循 n 的重复减半，因此即使对于最大 64 位输入，深度仍然很小。 这使得增加递归限制后递归变得安全。 

## 工作示例

 考虑结构仍然可见的小输入。 

当 n = 0 时，函数根据定义立即返回 1。 对于 n = 1，它也返回 1，因为只有一棵单节点树。 

对于 n = 3，我们计算 f(3)。 这里 s = 2，是偶数，所以 k = 1。结果变为 f(1)^2 = 1。 

对于 n = 4，我们计算 s = 3，因此 k = 1。有效分割为 (1,2) 和 (2,1)，因此 f(4) = 2 × f(1) × f(2)。 由于 f(2) = 1，因此结果为 2。 

| n | s = n−1 | 分裂| 计算|
 | --- | --- | --- | --- |
 | 3 | 2 | (1,1) | f(1)^2 = 1 | f(1)^2 = 1 |
 | 4 | 3 | (1,2),(2,1) | (1,2),(2,1) | 2 × f(1) × f(2) = 2 |

 这些痕迹表明奇偶校验完全决定了分支结构，并且递归很快崩溃到基本情况。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(T log n) 摊销 | 每个不同的 n 都会触发递归减半，直到达到基本情况，并且记忆可以防止重新计算 |
 | 空间| O(D) | D 是所有查询中遇到的不同状态的数量，每个状态在备忘录表中存储一次 |

 递归深度是 n 的对数，因为每一步都将问题大约减少一半。 对于多达一百万个查询，性能取决于中间状态的重用，这是记忆化所保证的。 内存占用仍然很小，因为只存储实际访问的值。

 ## 边缘情况

 当n = 0时，算法直接返回1，不进行递归。 这可以避免访问负子树大小并锚定重复。 

对于n = 1，由于首先命中基本情况，因此永远不会触发拆分逻辑，从而确保单节点树不会被错误分解。 

对于奇数 n − 1 值，必须对两个对称分割都进行计数。 在这种情况下，算法显式地乘以 2，并且这里缺少的因子会默默地将每个奇数状态的所有答案减半。 

对于非常大的 n，递归会反复将参数减半。 例如，像 10^18 这样的值会通过 10^18 → 5×10^17 → 2.5×10^17 等链快速减少，直到达到 0 或 1。备忘录表确保不同测试用例之间的共享子问题被重用而不是重新计算，从而防止指数爆炸。
