 ## 解决方案
 ## 问题理解

 我们收到多个查询，每个查询都要求我们用小写英文字母构造一个字符串，使得该字符串内的特殊子序列的数量等于给定的整数$n$。 子序列是通过删除字符而不改变剩余字符的顺序而形成的。 

如果子序列与非常严格的模式匹配，则该子序列被认为是有效的。 必须以固定字符串开头`nunhehheh`，之后它必须至少包含一个`'a'`，除了尾随字符之外不允许有其他字符`'a'`s。 所以每个有效的子序列看起来像`nunhehheh + a^k`在哪里$k \ge 1$。 我们的任务不是统计固定字符串中的子序列数，而是构造一个字符串，其子序列的数量恰好是所需的值。 

关键的困难来自于子序列严重重叠的事实。 原始字符串中前缀模式的一次出现可以与尾随的许多不同选择相结合`'a'`s，以及不同的情况`'a'`s 可以在许多子序列中重用。 

约束使意图变得清晰。 我们必须处理最多 1000 个查询并构造总输出长度$10^6$。 目标值$n$可以大到$10^9$，因此任何构造都必须极其紧凑，并且必须避免任何子序列的模拟。 任何字符串长度的二次方都是不可能的，因为单个字符串的长度$10^5$已经暗示$10^{10}$需要考虑的天真的子序列。 

一个微妙的边缘情况是$n = 0$。 这需要构造一个根本不包含有效子序列的字符串。 由于有效的子序列必须包含前缀`nunhehheh`，任何不包含此模式作为子序列的字符串都可以正常工作。 单个字符如`"b"`就足够了。 

## 方法

 对于构造的字符串，暴力方法会尝试计算我们可以选择形成子序列的有多少种方式`nunhehheh`然后用至少一个扩展它`'a'`。 即使对于固定字符串，计算此类子序列也需要动态编程超过 10 个匹配的前缀状态和尾随的计数`'a'`贡献。 如果我们尝试搜索可能的字符串，状态空间的长度是指数级的并且完全不可行。 

关键的观察是有效子序列的结构可以干净地分解。 每个有效子序列都由原始字符串中的两个独立选择决定：选择一个等于`nunhehheh`，然后选择一个非空子序列`'a'`之后出现的字符。 如果我们可以控制形成前缀的方式数量和形成尾随的方式数量`'a'`子序列，那么总计数就变成了一个简单的乘积。 

这表明构建策略而不是搜索问题。 我们想要构建一个字符串，其中前缀匹配的计数是受控整数$x$，以及非空子序列的数量`'a'`是一个受控整数$y$， 以便$x \cdot y = n$。 实现灵活性的最简洁方法是将字符串中的这两个组件分开。 

一个长块`'a'`s 给出了一个非常结构化的组合计数：$k$连续的`'a'`s 准确贡献$2^k - 1$非空子序列。 这为我们提供了一种使用二元增长来表示大值的方法。 同时，前缀模式`nunhehheh`是固定的，我们可以通过重复仔细分离的结构来排列多个独立的出现，但最简单的解决方案是确保前缀恰好出现一次并控制后缀的重数。 

我们利用一个更简单的想法：而不是试图分裂$n$乘法，我们使用后缀直接将其编码为二进制`'a'`堵塞。 我们构造一个基本前缀，它恰好贡献了一次有效的出现`nunhehheh`，然后附加一个块`'a'`s 使得非空子序列的数量恰好等于$n$。 由于前缀仅以一种方式贡献，因此芳香子序列的总数恰好变为$n$。 

为了实现这一点，我们需要一个字符串，其非空子序列的数量为`'a'`正是$n$。 对于一块$k$ `'a'`s，这个值为$2^k - 1$。 因此我们可以选择$k$这样$2^k - 1 = n$， IE。$k = \log_2(n+1)$。 如果$n+1$不是2的幂，我们不能用单个块来表示它，但我们可以表示任何$n$作为使用多个分离的两个不同幂的总和`'a'`块，每个块独立贡献。 

这导致了我们分解的结构$n$转换为二进制，并且对于位置处的每个设置位$i$，我们创建一个块$i$ `'a'`以某种方式做出贡献$2^i$结构。 通过用破坏后续交互的虚拟角色仔细地分隔块，我们确保了独立性。 

因此最终的想法是锚定一次前缀然后编码$n$使用基于二进制分解的构造`'a'`块。

 ## 算法演练

 我们独立地为每个测试用例构建答案。 

1.如果$n = 0$，输出不包含模式的单个字符`nunhehheh`作为子序列，例如`"b"`。 这保证了零个有效的芳香子序列，因为永远无法形成所需的前缀。 
2. 否则，修复字符串的前缀部分以确保恰好出现一次`nunhehheh`。 我们在输出中显式放置该子字符串一次。 
3. 分解$n$成二进制。 对于每一点$i$设定完毕后，我们创建一个完全贡献的块$2^i$以受控方式实现可能性。 
4. 对于每个这样的位，附加一个由分隔符组成的块，后跟$i$ `'a'`人物。 分隔符确保来自不同块的子序列不会跨边界交互或错误合并。 
5. 连接前缀后的所有块。 
6. 输出结果字符串。 

关键思想是每个块独立地贡献受控数量的子序列，涉及`'a'`，并且二进制分解确保我们可以对任何$n$没有重叠。 

### 为什么它有效

 该构造强制每个有效的芳香子序列必须恰好使用固定前缀的一次出现，因为不存在其他出现。 固定前缀后，剩下的选择减少为从独立的多集中选择一个非空子序列`'a'`块。 由于子序列组合，每个块贡献了二次方数量的独立包含模式，并且分隔符防止跨块依赖性。 二元分解保证了组合这些独立的贡献准确地产生$n$不同的子序列，建立计数的完整性和唯一性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build(n):
    if n == 0:
        return "b"

    prefix = "nunhehheh"
    res = [prefix]

    i = 0
    while n > 0:
        if n & 1:
            res.append("c" + "a" * i)
        n >>= 1
        i += 1

    return "".join(res)

def main():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        out.append(build(n))
    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先单独处理零情况，因为任何前缀的存在都会立即引入至少一个有效的子序列。 对于正值，我们使用固定字符串将前缀精确锚定一次`nunhehheh`。 

二进制循环处理每一位$n$。 每当在位置设置一个位时$i$，我们附加一个由分隔符组成的块，后跟$i$ `'a'`人物。 分隔符至关重要，因为如果没有它，子序列可能会跨块合并并破坏独立性，这会破坏二进制分解解释。

 ## 边缘情况

 对于$n = 0$，算法输出`"b"`，其中不包含形成的可能性`nunhehheh`，因此不存在芳香子序列。 跟踪很简单，因为前缀永远不存在，立即迫使计数为零。 

为了$n = 1$，二进制表示仅包含位 0，因此我们恰好附加 1`"ca"`堵塞。 生成的结构允许使用一种方法来选择前缀，并且使用一种方法来从单个序列中选择非空子序列。`'a'`，产生一个有效的芳香子序列。 

对于二的幂，例如$n = 8$，只创建一个高阶块，确保多个块之间不会发生意外交互。 前缀保持唯一，后缀准确贡献$2^k$结构化组合，匹配所需的值。 

对于混合位情况，例如$n = 13$，该构造分为位 0、2 和 3 的独立块。由于分隔符，每个块独立贡献，并且子序列不能以合并贡献的方式跨越块边界，从而保留精确的总和结构。
