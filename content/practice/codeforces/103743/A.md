 ## 解决方案
 ## 问题理解

 我们得到了一场比赛中按时间顺序排列的击杀事件日志。 每个事件都规定一名玩家淘汰另一名玩家。 虽然受害者会立即重生并且可以再次被杀死，但我们只关心随着时间的推移谁杀死了谁的顺序。 

任务是确定是否存在至少一名玩家在比赛中的某个时刻实现了“五杀”。 这意味着在同一玩家连续五次击杀中，受害者都是不同的。 击杀事件在该玩家的行动中必须在时间上是连续的，而不一定是涉及所有玩家的连续的全局事件。 

输出是一个单一的判决：对于任何玩家来说是否存在这样的序列。 

限制很小，总杀死事件最多为 1000 个。 这立即表明任何达到二次时间的解决方案都是安全的，但问题的结构允许直接线性扫描，每个事件的工作量恒定。 

一个微妙的点是解释“连续”。 天真的解释可能会考虑事件的全球相邻性，但这会错误地混合不同的杀手。 正确的解释是针对每个玩家的：我们只查看单个玩家按时间顺序执行的击杀顺序。 

第二个边缘案例是受害者的重复出现。 玩家可能会连续杀死五次，但会重复一个受害者，这会使五次杀死无效。 例如，像 A 杀死 X、A 杀死 X、A 杀死 Y、A 杀死 Z、A 杀死 W 这样的序列应该失败，因为 X 在五个连续杀死中出现了两次。 

另一个边缘情况是五次杀戮可以发生在任何地方，不一定从第一次杀戮开始或在最后一次杀戮结束。 

## 方法

 暴力方法是，对于每个玩家，按顺序收集他们所有的杀戮，然后检查每个大小为 5 的窗口，验证五个受害者是否不同。 如果一个玩家有 k 次杀戮，则每个玩家的成本为 O(k * 5)，这与他们的杀戮计数实际上是线性的。 对于所有玩家来说，这仍然是 O(n)，因为每个事件都会被处理固定次数。 

更幼稚的全局方法将尝试检查完整事件列表中长度为 5 的每个连续片段，并验证所有杀戮是否由同一玩家完成并具有不同的受害者。 这在概念上是失败的，因为“连续”条件并不指全局邻接，而且也更难正确概括。 

关键的观察是，我们永远不需要存储超过每个玩家最近五次击杀的数据。 一旦玩家完成了至少五次击杀，只有最近的五次击杀才能检测当时有效的五次击杀结束。 这将问题转化为为每个玩家维护一个滑动窗口并检查一个小的固定大小的条件。 

因为 n 最多为 1000，所以我们可以安全地维护一个从玩家姓名到最近受害者列表（或双端队列）的字典。 每个新的kill都会在O(1)时间内更新该结构并检查最多五个值。

 | --- | --- | --- | --- |
 | 每个玩家窗口的暴力破解 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们为每个玩家维护他们迄今为止杀死的受害者的序列，但我们只保留最后五个条目，因为任何较早的条目都无法促成稍后结束的新的有效五杀。

1.按时间顺序一一阅读击杀事件。 每个事件都会产生一个杀手和一个受害者。 这个顺序至关重要，因为“连续”取决于时间。 
2. 对于每个事件，将受害者添加到凶手最近杀人的个人列表中。 如果列表增长超过五个元素，请删除最旧的一个。 这确保我们只跟踪其历史记录中最相关的后缀。 
3. 更新列表后，如果杀手现在至少有 5 次杀人记录，请检查最后 5 名受害者是否全部不同。 这是确认在此事件中结束五杀所需的唯一条件。 
4. 如果任何玩家满足最近五次击杀的明确性条件，我们可以立即断定答案是肯定的并停止处理进一步的事件。 
5. 如果我们处理完所有事件而没有找到这样的玩家，则答案是否定的。 

我们只检查最后五个条目的原因是，任何有效的五次击杀都必须在该玩家的当前事件中结束，并且任何较早的候选窗口在结束时都已经被检查过。 

### 为什么它有效

 对于任何玩家来说，每个可能的五次击杀都对应于按时间顺序排列的连续五次击杀块。 通过维持大小为 5 的滚动窗口，我们确保每个这样的块都被检查一次，此时它成为玩家历史的后缀。 由于我们从不丢弃未来窗口所需的信息，并且始终保留最近的五个事件，因此不能跳过任何有效序列。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    recent = {}  # player -> list of last kills
    
    for _ in range(n):
        a, b = input().split()
        
        if a not in recent:
            recent[a] = []
        
        recent[a].append(b)
        
        if len(recent[a]) > 5:
            recent[a].pop(0)
        
        if len(recent[a]) == 5:
            if len(set(recent[a])) == 5:
                print("PENTA KILL!")
                return
    
    print("SAD:(")

if __name__ == "__main__":
    solve()
```核心结构是一个以玩家姓名为键的字典，它只存储最后几个受害者。 这避免了存储完整历史记录的需要。 

唯一微妙的实现细节是滑动窗口修剪步骤。 当列表超过五个元素时，我们总是从前面删除，以确保大小恒定。 使用列表就足够了，因为最大大小是固定的并且很小，因此 O(1) 摊销操作就足够了。 

我们还依靠集合构造来检查唯一性。 由于窗口大小最多为 5，因此该检查的时间为常数。

 ## 边缘情况

 一种边缘情况是五次杀戮正好发生在日志的末尾。 该算法仍然会检测到它，因为每个事件都会在更新窗口后触发检查。 例如，如果玩家的最后五个事件都是不同的，则最终迭代会立即捕获它并输出成功。 

另一种情况是，当玩家击杀次数超过五次时，并且有效的五次击杀发生在其历史记录的后期。 由于我们始终只保留最后五个击杀，因此较旧的序列会自动丢弃，但这是安全的，因为任何有效序列都必须在其结束时刻在最后五个击杀中结束。 滑动窗口确保我们永远不需要更早的历史记录。 

最后一个微妙的情况是多个玩家可能实现五杀。 该算法在第一次检测时停止，这是正确的，因为输出只需要存在，而不需要身份或计数。
