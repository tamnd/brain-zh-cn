 ## 解决方案
 ## 问题理解

 我们得到一个一维的游戏世界，其位置位于整数线上。 坐标从0到n都有柱子，其中i柱子有宝藏值ai，i≥1，而0号柱子是起点，没有宝藏。 在位置 n+1 之外是一个安全的无限平台，到达该平台上的任何位置即可成功结束游戏。 

一次移动包括严格向右跳跃，从当前位置到坐标更大的另一个位置，并且每次跳跃的长度至多为p。 玩家从位置 0 开始，只能落在柱子上或最终落在最终平台上，目标是到达平台。 

不同的是，每个查询都定义一个级别 x，并且在级别 x 中，玩家只能踩在索引为 x 倍数的柱子上。 因此，我们不考虑所有支柱，而是只保留索引 x、2x、3x 等，直到 n。 在这些允许的支柱中，我们希望有一个从 0 开始的有效递增序列，尊重跳跃限制 p，并最终到达平台。 分数是所有访问过的柱子的人工智能总和。 

每个查询都要求该级别的最大可能分数，或者如果没有有效路径可以到达平台，则报告不可能性。 

约束 n 和 q 最大为 10^6，因此任何通过独立模拟所有允许的跳转来处理每个查询的解决方案都会失败。 在最坏的情况下，即使每个查询的 O(n) 也已经超过 10^12 次操作。 这迫使我们对每个起始除数结构进行有效的预处理，并确保所有查询的总工作保持接近线性或接近线性。 

一些边缘情况很容易被忽视。 

如果x大于p，那么从0到支柱x的第一次跳跃已经不可能，因为它超出了跳跃限制。 例如，如果 p = 3 且 x = 5，则根本无法启动任何序列，因此答案立即不可能。 

另一个微妙的情况是到达平台。 即使我们通过选定的支柱计算出一条好的路径，我们仍然需要最后一个支柱 j 来满足 n + 1 − j ≤ p。 例如，如果 n = 10 且 p = 2，到达支柱 8 是不够的，除非 11 − 8 ≤ 2 成立，否则我们无法退出。 

最后，有效位置的结构在很大程度上取决于 x。 对于不同的查询，允许的节点集完全改变，因此在不利用算术结构的情况下，所有节点上的全局DP不能直接重用。 

## 方法

 每个查询的直接模拟都会构建 x 的倍数序列并对其运行动态程序。 从倍数 i = kx 中，我们尝试所有先前的倍数 j = tx，其中 j < i 且 i − j ≤ p，取最佳可达总和。 这正确地对问题进行了建模，但对于每个查询，它可能会扫描 O(n/x) 个节点，并且在每个节点内扫描最多 O(p/x) 个前辈，从而在最坏的情况下导致二次行为。 

关键的简化来自于观察，在固定级别 x 内，所有允许的节点都是均匀分布的：x、2x、3x 等。 如果我们根据该序列中的索引 k 重写位置，则从 k 移动到先前的有效位置 k′ 对应于坐标中的距离 (k − k′)x。 跳跃约束 (k − k′)x ≤ p 变为 k − k′ ≤ ⌊p / x⌋。 这将转换转变为由 k 索引的 DP 数组上的固定大小滑动窗口。 

因此，每个查询都成为具有滑动窗口最大值的最长直线问题，可以在 x 倍数的线性时间内使用单调双端队列来维护该问题。 

我们仍然需要将其与查询有效地连接起来。 由于每个查询都是独立的，但使用相同的结构，因此我们使用此优化的 DP 预先计算从 1 到 n 的每个 x 的答案。 总功变为谐波：n/1 + n/2 + n/3 + …，即 O(n log n)。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的暴力破解 | 最坏情况 | O(q · n) O(n) | 太慢了 |
 | 带滑动窗口的 Per-x DP | 总共 O(n log n) | O(n) | 已接受 |

 ## 算法演练

 对于每个级别 x，我们将自己限制为允许的支柱 x、2x、3x、...、mx 的序列，其中 mx ≤ n。 

1. 将原始约束转换为序列索引。 我们将第 k 个节点定义为位置 kx，值为 a[kx]。 这将问题从整数的稀疏图简化为简单的链。 
2. 确定是否可以开始。 第一步是从 0 到 x，因此只有当 x ≤ p 时才有可能。 如果 x > p，我们立即知道该级别不存在有效路径。 
3. 构建一个基于 k 的 DP 数组。 令 dp[k] 表示到达 kx 支柱时收集到的最大硬币。 
4. dp[k] 的转换仅考虑一次跳转可到达 kx 的先前位置。 距离约束为 (k − j)x ≤ p，因此 j 必须位于 [k − W, k − 1] 中，其中 W = ⌊p / x⌋。 我们在此范围内保持滑动最大值。 
5. 使用单调双端队列存储最后 W 个位置的候选 dp 值。 当我们向前移动 k 时，我们会推送 dp[k − 1] 并弹出超出窗口的过时索引。 
6. 如果 x ≤ p，则将 dp[1] 初始化为 a[x] 加 0。 如果 x > p，则完全跳过处理。 
7. 填完dp后，识别哪些k可以成功到达平台。 如果 kx ≥ n + 1 − p，我们可以从支柱 kx 退出。 
8. 取所有有效退出位置的最大 dp[k]。 如果不存在或者所有状态都不可达，则答案是不可能的。 

### 为什么它有效

 DP 状态完全捕获到达每个允许的支柱的所有有效方式，因为任何有效路径都必须按升序访问支柱并且仅在允许的子集中。 滑动窗口限制与按 x 重新缩放后的跳跃约束完全匹配，因此考虑每个可行的前驱，并且不包括无效的前驱。 单调双端队列确保我们始终计算每个状态的最佳转换值，从而保持最大子结构的正确性。 

最后的过滤步骤确保最后跳转到平台的可行性，因此DP不会过多计算实际上无法完成的路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, q, p = map(int, input().split())
a = [0] + list(map(int, input().split()))

# store answers for all x
ans = [None] * (n + 1)

for x in range(1, n + 1):
    m = n // x
    if m == 0:
        ans[x] = None
        continue

    # cannot even start
    if x > p:
        ans[x] = None
        continue

    W = p // x

    dp = [0] * (m + 1)
    dq = []  # store indices, maintain decreasing dp

    dp[1] = a[x]

    dq.append(1)

    for k in range(2, m + 1):
        # remove out of window
        while dq and dq[0] < k - W:
            dq.pop(0)

        best_prev = dp[dq[0]] if dq else 0
        dp[k] = a[k * x] + best_prev

        # maintain monotonic deque
        while dq and dp[dq[-1]] <= dp[k]:
            dq.pop()
        dq.append(k)

    # compute best reachable ending
    limit = n + 1 - p
    best = None

    for k in range(1, m + 1):
        pos = k * x
        if pos >= limit:
            if best is None or dp[k] > best:
                best = dp[k]

    ans[x] = best

for _ in range(q):
    x = int(input())
    if ans[x] is None:
        print("Noob")
    else:
        print(ans[x])
```该代码独立处理每个可能的级别，但使用双端队列来确保每个 DP 转换在恒定的摊销时间内计算。 关键的实现细节是使用 W = p // x 从坐标距离到索引距离的转换，这就是将几何约束转变为简单的滑动窗口的原因。 

最后的过滤步骤仅扫描 DP 阵列的最后一个有效段，以确保端点可以到达平台，如果只关注最大化总和，这一点很重要且很容易被忽略。 

## 工作示例

 ### 示例 1

 输入：

 n = 5，p = 4

 a = [2, 5, -6, -4, 3]

 查询 x = 1

 我们考虑所有位置 1、2、3、4、5，因此 k 直接对应于索引。 

| k | 位置| dp[k] | dp[k] | 最大使用窗口 |
 | --- | --- | --- | --- |
 | 1 | 1 | 2 | 0 |
 | 2 | 2 | 7 | 2 |
 | 3 | 3 | 1 | 7 |
 | 4 | 4 | 3 | 7 |
 | 5 | 5 | 6 | 7 |

 最好的结束位置是位置 ≥ 6 − 4 = 2 的位置，因此 k ≥ 2。最大 dp 为 7，在 k = 2 时实现。这符合直觉，即在负值减少增益之前，采用 1 → 2 是最好的。 

### 示例 2

 输入：

 n = 10，p = 8

 a = [5, 4, -6, 8, -11, 5, -6, 4, -7, 3]

 查询 x = 2

 允许的位置为 2、4、6、8、10。 

| k | 邮政 | dp[k] | dp[k] | 最佳前一窗口 |
 | --- | --- | --- | --- |
 | 1 | 2 | 4 | 0 |
 | 2 | 4 | 12 | 12 4 |
 | 3 | 6 | 17 | 17 12 | 12
 | 4 | 8 | 21 | 21 17 | 17
 | 5 | 10 | 10 24 | 21 | 21

 退出条件要求 pos ≥ 11 − 8 = 3，因此所有 k 都有效。 最佳答案是24。 

该轨迹显示了滑动窗口 DP 如何在不重新考虑早期决策的情况下稳定地积累最优选择，因为窗口约束强制执行所有有效的跳跃。

 ## 边缘情况

 一种重要的边缘情况是 x 超过 p 时。 在这种情况下，DP 甚至永远不会启动，因为从 0 到 x 的第一次跳转是无效的。 该算法明确地检查这一点并立即分配一个不可能的结果，从而避免不必要的计算。 

另一种情况是当 DP 中存在有效的退出位置但没有一个满足平台条件时。 即使 dp 值很大，我们也必须确保 kx ≥ n + 1 − p。 该算法使用此阈值过滤候选者，因此它不会错误地接受无法完成的路径。 

当 p < x 但 x 仍然整除 n 时，会出现最后一个微妙的情况。 尽管存在许多倍数，但起始约束占主导地位。 该算法会尽早正确地短路这些情况，从而防止错误的 DP 初始化，否则会产生误导性的总和。
