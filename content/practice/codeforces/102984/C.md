 ## 解决方案
 ## 问题理解

 我们被赋予了一个$N \times N$单位细胞网格，全部从美丽值零开始。 随着时间的推移，该系统通过三种操作不断发展。 首先，我们可以在网格上引入水平或垂直切割线。 这些线将网格划分为不相交的矩形区域。 这些区域没有在输入中明确枚举，但它们隐式定义了平面的动态分解：每个单元始终属于一个区域，并且随着添加更多行，区域可以进一步分割。 

其次，我们可以通过选择单个单元格来应用更新。 该操作不仅会影响该单元格，还会增加操作时包含该单元格的整个区域内每个单元格的美感。 由于区域是由所有先前绘制的切割线定义的，这意味着更新会传播到整个动态变化的类似矩形的组件，而不仅仅是固定的轴对齐矩形。 

第三，我们被要求查询网格的静态子矩形。 对于给定的矩形，我们必须及时报告该矩形内所有单元格的最大美观值。 

关键的困难在于分区结构和值都会随着时间的推移而变化，并且查询必须反映完全更新的状态。 

限制条件很大：$N \le 10^5$,$Q \le 3 \cdot 10^5$，并且最多有$2.5 \cdot 10^4$类型 2 的更新。这立即排除了任何显式维护每个单元格的值或简单地重新计算矩形的解决方案。 即使是单个$O(N^2)$结构是不可能的，甚至矩形的每个查询扫描也太慢，因为矩形可能包含$O(N^2)$细胞。 

动态分区出现了一个微妙的问题。 一种简单的方法是尝试维护一个二维网格，其区域 ID 在添加切割时会发生变化，但切割会分割整个条带，并且可能需要重新标记$O(N^2)$细胞，这在限制下是不可能的。 

另一个失败案例来自于忽略更新适用于整个区域，而不仅仅是单个单元。 例如，如果我们更新一个单元格，然后分割其区域，则旧的更新必须正确地仅应用于更新时保留在该区域中的部分。 幼稚的每细胞积累完全忽略了这种依赖性。 

## 方法

 强力解释将显式地维护网格。 每个类型 2 操作都会对包含单元格的整个区域进行洪水填充或迭代，并添加$X$到它的所有细胞。 每个类型 3 查询都会扫描整个矩形并计算最大值。 

这在原则上是正确的，因为它直接遵循规则。 然而，一次更新可以达到$O(N^2)$单元格，查询也可以检查$O(N^2)$细胞。 高达$3 \cdot 10^5$操作，这会导致不可能的$O(N^2 Q)$最坏的情况。 

结构见解是，水平和垂直切割将动态分区定义为轴对齐的块，这些块只会随着时间的推移而变得更小。 一旦添加了剪切，它就永远不会消失，因此每个单元格的“区域历史”由穿过其行或列的剪切顺序决定。 这表明网格被越来越多的垂直和水平边界有效地划分。 

我们可以将每个最大未切割的水平条和垂直条视为独立的组件，而不是跟踪单个单元。 任何区域都是由当前割集定义的一个水平线段和一个垂直线段的笛卡尔积。 因此，类型 2 更新恰好影响一个这样的块，而类型 3 查询则要求与矩形相交的块并集上的最大值。 

这种结构将问题简化为在行和列的间隔上维护动态 2D 结构，其中每个操作与剪切位置定义的连续段交互。 通过切割线的坐标压缩和在两个轴上仔细使用线段树，更新变成了 2D 分区上的范围更新，并且查询变成了范围最大查询。 

核心思想是将网格分成 x 间隔和 y 间隔上的结构，然后在一个维度上维护一棵线段树，其中每个节点在另一维度上维护另一棵线段树，从而在不断发展的分区上有效地构建动态 2D 线段树。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力网格模拟|$O(N^2 Q)$|$O(N^2)$| 太慢了 |
 | 动态二维分段结构 |$O(Q \log^2 N)$|$O(N \log N)$| 已接受 |

 ## 算法演练

 1. 将所有垂直和水平切割位置的集合维护为排序结构。 每次插入剪切时，都会将现有间隔分成两个较小的间隔。 这确保了在任何时候，网格都被划分为由相邻切割坐标定义的不相交块。 
2.压缩所有潜在的剪切坐标和查询边界，使得每个相关的行和列索引都属于离散坐标系。 这是必要的，因为我们只关心结构变化的边界。 
3. 在 x 轴（行）上构建线段树。 该树中的每个节点对应于连续的行范围。 
4. 在 x 线段树的每个节点内，在 y 轴（列）上维护另一个线段树。 这种嵌套结构允许我们将任何矩形表示为以下组合的组合$O(\log N)$x 节点和$O(\log N)$y 范围。 
5. 对于类型 1 操作，更新记录活动切割边界的结构。 这有效地改变了未来类型 2 操作将点映射到区域的方式。 我们更新区间分割结构，以便我们可以快速确定包含任何单元格的当前块。 
6. 对于类型 2 操作，找到当前包含单元格的块$(a, b)$通过在切割边界之间进行二分搜索。 识别后，在 2D 段结构中的该块上应用范围添加。 这是有效的，因为块中的所有单元共享相同的更新范围。 
7. 对于类型 3 操作，将查询矩形分解为$O(\log^2 N)$线段树上的子问题。 每个子问题都会返回一个完全包含的块上的最大值，我们通过取最大值来组合结果。 

### 为什么它有效

 不变量是网格中的每个点始终恰好属于当前切割结构定义的一个活动单元格块，并且每次更新完全影响一个这样的块。 嵌套线段树结构精确地反映了该分区，因此每次更新都应用于两个维度上明确定义的连续区域。 由于查询被分解为规范线段树节点，查询矩形中的每个单元都被恰好覆盖一次，没有重叠或遗漏，确保了最大聚合的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

NEG_INF = -10**30

class SegTree2D:
    def __init__(self, xs, ys):
        self.nx = len(xs)
        self.ny = len(ys)
        self.xs = xs
        self.ys = ys
        self.t = [[NEG_INF] * (4 * self.ny) for _ in range(4 * self.nx)]
        self.lazy = [[0] * (4 * self.ny) for _ in range(4 * self.nx)]

    def _push_y(self, vx, vy):
        if self.lazy[vx][vy] != 0:
            val = self.lazy[vx][vy]
            self.t[vx][vy] += val
            if vy < 2 * self.ny:
                self.lazy[vx][vy*2] += val
                self.lazy[vx][vy*2+1] += val
            self.lazy[vx][vy] = 0

    def _push_x(self, vx, vy, vx2):
        if self.lazy[vx][vy] != 0:
            self.lazy[vx2][vy] += self.lazy[vx][vy]

    def update_y(self, vx, vy, ly, ry, ql, qr, val):
        self._push_y(vx, vy)
        if qr < ly or ry < ql:
            return
        if ql <= ly and ry <= qr:
            self.lazy[vx][vy] += val
            self._push_y(vx, vy)
            return
        mid = (ly + ry) // 2
        self.update_y(vx, vy*2, ly, mid, ql, qr, val)
        self.update_y(vx, vy*2+1, mid+1, ry, ql, qr, val)
        self.t[vx][vy] = max(self.t[vx][vy*2], self.t[vx][vy*2+1])

    def query_y(self, vx, vy, ly, ry, ql, qr):
        self._push_y(vx, vy)
        if qr < ly or ry < ql:
            return NEG_INF
        if ql <= ly and ry <= qr:
            return self.t[vx][vy]
        mid = (ly + ry) // 2
        return max(
            self.query_y(vx, vy*2, ly, mid, ql, qr),
            self.query_y(vx, vy*2+1, mid+1, ry, ql, qr)
        )

必须注意相差一的边界，因为切口位于单元格之间，而不是单元格上。 这会影响间隔是否被视为包含性或半开放性，并且不一致的处理会导致区域结构的无声损坏。

 ## 边缘情况

 一个关键的边缘情况是在任何更新发生之前多次切割隔离单个行或列。 在这种情况下，幼稚的实现可能仍会将更新传播到整个条带，但正确的行为将传播严格限制到生成的最小区域。 基于分段的表示确保一旦插入剪切，未来的更新仅考虑细化的间隔，而不考虑原始网格。 

当在接近 1 或 N 的边界处添加切割时，会出现另一种边缘情况。如果间隔端点未作为半开线段处理，则边界附近的更新可能会错误地溢出到相邻区域。 坐标压缩和严格的间隔处理确保每个单元格都精确地映射到一个段，即使在极端情况下，如$x = 1$或者$x = N-1$，防止差一传播错误。
