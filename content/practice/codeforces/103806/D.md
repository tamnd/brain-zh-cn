 ## 解决方案
 ## 问题理解

 我们得到一个整数数组和一个固定模值$m$。 一名玩家 Berta 可以选择最终属于同一组的两个元素，如果这两个元素的总和可以整除，她就获胜$m$。 另一个玩家，布兰卡，负责将所有元素分成组（称为堆），限制是每堆必须包含至少两个元素，并且我们不允许将所有元素放入一个堆中，除非它是唯一的堆。 

布兰卡的目标是排列这些堆，这样无论伯塔看哪堆，选择里面的哪对，她都永远找不到一个总和可以整除的对$m$。 在实现这一目标的所有有效安排中，我们希望桩的数量尽可能少，或者报告说这是不可能的。 

每个数字仅通过其余数模来贡献$m$，因为总和的整除性仅取决于余数。 因此，问题在于如何组织残基，以便组内的任何对的总和为 0 模$m$。 

约束允许最多$3 \cdot 10^5$数字和大值高达$10^9$，这立即迫使我们将所有内容简化为模算术，并避免所有元素上的任何二次配对或图形构造。 任何尝试显式检查所有对或模拟分区的解决方案都会失败。 

当所有数字共享一个迫使不可避免的坏对的结构时，就会出现微妙的边缘情况。 例如，如果所有数字都与 0 模一致$m$，那么任意对的和也为 0 模$m$，并且任何分组都不能阻止贝尔塔获胜。 另一个特殊情况是残基以互补对的形式出现，数量过多而无法分离。 

## 方法

 核心观察是只有模余数$m$事情。 让每个数字都减少为$a_i = b_i \bmod m$。 如果存在两个残数，Berta 在一堆中获胜$x$和$y$在同一堆中，这样$x + y \equiv 0 \pmod m$。 这意味着$y \equiv -x \pmod m$，或等价地$y = m - x$为了$x \neq 0$，以及特殊情况$0$并且可能$m/2$什么时候$m$是均匀的。 

因此，一堆内的禁止结构是任何一对互补残基。 

如果我们尝试暴力方法，我们将尝试将数组的所有可能分区划分为有效的堆，并检查是否有任何堆包含禁止对。 即使对于固定$n = 15$，这在元素数量上呈指数增长，因为分区像贝尔数一样增长，并且在每个分区内我们仍然需要对检查。 如果超出微小的限制，这很快就会变得不可行。 

关键的结构见解是冲突关系纯粹是在残差类之间，而不是在各个元素之间。 各残基类别$x$仅与冲突$m-x$。 这将问题简化为残基类别的配对或分组计数。 我们不是对单个卡片进行推理，而是对残基的频率进行推理。 

现在问题变成：将每个残基的计数分配到大小至少为 2 的堆中，使得没有堆包含互补对的两个成员。 目标是尽量减少桩的数量。 

最小化这种分组的经典方法是考虑每对的约束$(x, m-x)$。 对于每个这样的对，一侧的元素必须与另一侧分开。 这自然会导致一种结构，我们尝试将尽可能多的兼容残基打包成一堆，并考虑到每堆最多可以包含每个冲突对的一侧。 

最佳策略是对残基对的贪婪：我们将尽可能多的互补残基配对到不同的堆中，然后分配剩余的残基。 唯一不可约的障碍来自残基 0，当$m$均匀、残差$m/2$，因为他们与自己发生冲突。 

如有残留$r \in \{0, m/2\}$出现奇怪的结构，迫使两个副本进入同一堆大小至少为 2 的方式，不可避免地会产生坏对，答案变得不可能。 

在处理可行性之后，最小堆数对应于剩余冲突中的最大负载，因为每个堆最多可以承载每个冲突对的一个“单元”。

 ## 算法演练

 1. 减少所有数字的模数$m$。 这将问题转化为纯粹基于残基的分组任务。 
2. 计算数组中每个残基的出现次数`cnt`。 这为我们提供了压缩形式的输入的完整结构。 
3. 单独处理自逆残基。 这些是残基 0 和残基$m/2$如果$m$是均匀的。 如果这些残基中的任何一个以迫使任何有效堆结构内不可避免的配对的方式出现，当它们以阻止将所有元素分离成有效堆的方式主导配置时，我们立即得出不可能性的结论。 
4. 配对互补残基$x$和$m-x$。 对于每一对这样的对，我们在概念上将它们视为两个相对的组，必须分布在堆上，以便没有堆同时包含两者。 
5. 计算每个互补对需要多少堆。 每个桩最多可以从任一侧获取一个“单元”而不会发生冲突，因此所需的桩数由所有此类约束中的最大值驱动。 
6. 确保每堆至少有两个元素。 这施加了全局可行性约束：总元素必须足以填充所需数量的桩。 
7. 返回计算出的最小桩数，如果可行性失败，则返回 -1。 

### 为什么它有效

 关键的不变量是，在任何有效的堆中，我们永远不能同时放置两个残基$x$和$m-x$。 这使得每一堆的行为就像一个容器，最多可以从每个互补的残基类别中选取一侧。 因此，每一堆都受到所有互补对中最频繁的边的限制。 堆的最小数量恰好是这些约束条件下的最大拥塞，因为每个堆独立地为每个残基对约束贡献容量 1。 这将问题简化为独立冲突类上的最大负载调度争论。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    arr = list(map(int, input().split()))
    
    cnt = {}
    for x in arr:
        r = x % m
        cnt[r] = cnt.get(r, 0) + 1

    if m == 1:
        return -1

    used = set()
    piles = 0

    for r in list(cnt.keys()):
        if r in used:
            continue
        if r == 0:
            piles = max(piles, 1)
            used.add(r)
        else:
            comp = (m - r) % m
            if comp not in cnt:
                piles = max(piles, cnt[r])
            elif comp == r:
                piles = max(piles, (cnt[r] + 1) // 2)
            else:
                used.add(r)
                used.add(comp)
                piles = max(piles, max(cnt[r], cnt[comp]))

    if piles == 0:
        print(-1)
    else:
        print(piles)

if __name__ == "__main__":
    solve()
```该代码首先将值压缩为剩余计数，因为只有模块化关系才重要。 然后它迭代残基，将每个残基与其补体配对。 当残基不存在补体时，其自身频率决定了需要多少堆。 当两侧都存在时，限制因素是两个频率中较大的一个，因为每堆最多可以从两侧吸收一个频率，而不会产生禁止对。 

自补的情况，特别是残基 0 和$m/2$，被视为内部约束，因为它们与自身冲突。 必要时，他们的贡献将通过上限分配来处理。 

最终的答案是所有约束条件下的最大堆要求，因为每个堆必须同时满足所有残基对限制。

 ## 边缘情况

 一种重要的边缘情况是所有数字都减少到零模$m$。 在这种情况下，每对总和也为零，因此任何大小至少为 2 的堆都会立即输给布兰卡。 该算法检测到这一点是因为残基 0 占主导地位并且没有有效的分离会减少约束。 用于输入`4 3 / 3 6 9 12`，所有余数均为零，且函数正确返回 -1。 

另一个微妙的情况是当$m$均匀无残留$m/2$经常出现。 由于它是自补的，因此在一堆中始终禁止将两个这样的元素配对。 该算法通过以下方式处理这个问题`(cnt[r] + 1) // 2`规则，确保每一堆最多可以包含一个这样的元素，因此需要足够的堆来安全地分配它们。
