 ## 解决方案
 ## 问题理解

 我们得到了几个独立的场景，每个场景都描述了一群朋友去剧院。 每个朋友已经被分配了一个座位，由代表行和列的一对整数指定。 

座位布局的不同寻常之处在于列的编号方式。 列的排列不是从左到右单调增加，而是使列 1 和列 2 位于行的中间，彼此相邻。 从那里开始，列对称地向外延伸：奇数以远离中心的递增顺序向一侧扩展，偶数以远离中心的递增顺序向另一侧扩展。 然而，对于问题本身，该结构仅在通过列号定义邻接关系时才有意义。 

对于每个测试用例，我们必须确定是否所有朋友都可以被视为“在一起”，这意味着他们都坐在同一排并占据一组在列位置上连续的座位。 邻接要求很严格：对一行内的列进行排序后，它们必须形成一个没有间隙的连续块。 

每个测试用例最多可以包含大量朋友，所有测试用例的总数可以达到 100 万。 这立即排除了每个测试用例的朋友数量呈二次方的任何解决方案，因为即使 10^6 log 10^6 操作也可以，但任何类似于成对检查的操作都会太慢。 

当朋友共享同一行但他们的列不连续时，就会出现微妙的边缘情况。 例如，如果三个朋友位于同一行的第 1、2 和 4 列，则答案必须为“否”，因为缺少第 3 列。 另一个边缘情况是当所有朋友都在不同的排时； 即使列完美对齐，答案仍然是否定的，因为它们没有坐在同一行中。 

主要困难不是奇怪的列编号，而是认识到只有行相等和间隔连续性才重要。 

## 方法

 推理该问题的一个直接方法是检查每对朋友并验证两个条件：所有行必须相等，并且在收集所有列后，该集合必须形成连续的序列。 这种暴力的想法在概念上很简单：对于每个测试用例，将所有行与第一行进行比较，然后检查最小列和最大列之间的每个整数是否出现。 

然而，如果简单地实施，这种方法就会变得低效。 检查所有好友对的时间复杂度为 O(n²)，在最坏的情况下这将导致大约 10^2 次操作，远远超出了任何可行的限制。 由于内存重置或较大的坐标范围，即使重复存储和扫描每个测试用例的布尔存在数组也会出现问题。 

关键的观察是我们实际上不需要知道座位的完整结构。 每个测试用例我们只需要三条信息：所有行是否相同、最小列和最大列。 如果所有朋友都在同一行，那么当且仅当不同列的数量等于最大列减去最小列加一时，它们才是连续的。 这避免了除了简单聚合之外的任何排序或频率数组的需要。 

这将问题简化为每个测试用例一次线性扫描，跟踪一些变量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力配对检查 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳线性扫描 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

1. 对于每个测试用例，读取朋友的数量并初始化行一致性、最小列、最大列和条目数的跟踪变量。 我们将该行初始化为第一个朋友的行以建立参考点。 
2. 遍历所有友元，读取每对（行、列）。 对于每个朋友，将他们的行与参考行进行比较。 如果发生任何不匹配，我们已经知道它们不能全部坐在一起，但我们仍然继续读取输入以保持正确性。 
3. 迭代时，不断更新最小和最大列值。 这捕获了该排占用座位的全部分布。 
4. 统计好友的数量。 这是必要的，因为我们稍后会将其与最小和最大列定义的间隔长度进行比较。 
5. 处理完所有好友后，检查所有行是否与参考行匹配。 如果不是，答案立即是否定的。 
6. 如果行一致，通过检查 max_column - min_column + 1 是否等于好友数量来验证列是否形成连续段。 

### 为什么它有效

 如果所有朋友共享同一排，那么他们的座位就简化为整数位置上的一维问题。 当且仅当一组整数的最大值减去最小值等于大小减一时，一组整数形成一个连续的块。 任何缺失的列都会严格增加范围而不增加占用的座位数，从而打破了这种平等。 由于我们精确跟踪最小值、最大值和计数，因此不需要排序假设，也不会遗漏任何间隙。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        n = int(input())

        first_row = None
        same_row = True

        min_col = 10**30
        max_col = -10**30

        for i in range(n):
            f, c = map(int, input().split())

            if i == 0:
                first_row = f

            if f != first_row:
                same_row = False

            if c < min_col:
                min_col = c
            if c > max_col:
                max_col = c

        if not same_row:
            out.append("NO")
        else:
            if max_col - min_col + 1 == n:
                out.append("SI")
            else:
                out.append("NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案依赖于在单次传递期间仅维护参考行并更新列的边界。 变量`same_row`捕获行中是否出现任何偏差。 最小-最大跟踪确保我们无需排序即可重建跨度。 

一个常见的实现错误是尝试将所有列存储在列表中并对它们进行排序，这仍然是正确的但没有必要。 另一个错误是在行不匹配时提前退出而不消耗剩余的输入行，这会破坏输入解析。

 ## 边缘情况

 关键的边缘情况是所有列形成完美间隔但行不同。 例如：```
1
3
1 1
2 2
1 3
```扫描正确设置`same_row`在第二个元素处设置为 False，但仍继续更新 min 和 max。 尽管 min 和 max 会建议连续的范围，但行不匹配会强制“否”，这符合坐在一起需要单排的要求。 

另一种边缘情况是 n = 1。只有一个朋友时，min 等于 max，并且条件 max - min + 1 == n 不成立。 该算法正确返回 SI，因为行一致性和连续性均得到满足。 

第三种情况涉及高达 10^9 的大坐标值。 由于我们只跟踪最小值和最大值，因此不会发生溢出或大数组分配，并且无论坐标范围如何，解决方案都保持恒定空间。
