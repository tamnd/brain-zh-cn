 ## 解决方案
 ## 问题理解

 我们给出了一个在无向连通图上演化的过程，其中$n$顶点。 该过程取决于整数序列$a_1, a_2, \dots, a_m$，我们可以将其视为“距离随时间跳跃”。 

选择起始顶点并将其标记为在第 0 天绘制。然后，对于每一天$i$，每个顶点到任何已绘制顶点的最短路径距离恰好是$a_i$变成画的。 一旦顶点被绘制，它就会永远保持绘制状态，因此绘制的顶点集只会增长。 

问题不是关于特定的图表。 相反，我们会被问到，如果允许我们为该图选择最佳的起始顶点，那么这个距离规则序列是否足够强大以保证每个连通图的完全覆盖。 如果答案是肯定的，我们打印“SI”。 否则，我们必须构建一个连通图$n$使得无论选择哪个起始顶点，该过程都无法绘制所有顶点。 

限制很小：$n \le 100$,$m \le 100$，所有测试用例的总和最多为 1000。这意味着我们可以从结构图属性和简单构造的角度进行思考。 任何事物$O(n^2)$或者$O(nm)$每个测试用例的速度都非常快。 

一个微妙的一点是，该过程仅由距离驱动。 图的结构仅通过最短路径距离起作用，因此对手可以自由地构建在图度量约束内表现得像任意距离度量的图。 

主要困难在于理解距离层的序列是否可以在所有可能的图中“模拟”完整的类似 BFS 的扩展，或者是否存在某些图形几何结构永久地阻止它。 

一些边缘案例有助于阐明该过程的性质。 

如果$a = [1]$，然后从任何顶点开始，我们立即绘制所有邻居，然后在接下来的步骤中，我们继续从整个绘制的边界向外扩展一条边。 这正是 BFS 扩展，在任何连通图上它最终都会达到一切。 

如果相反$a = [2]$，并且该图是一条简单路径，从一个端点开始仅绘制距起点均匀距离的顶点。 相反的奇偶校验层永远不会出现，因此一半的图形保持未绘制状态。 

这些例子表明价值$1$起着特殊的作用，因为它是唯一直接沿边缘传播并保证完全可达性的距离。 

## 方法

 暴力解释将尝试固定图的每个可能的起始顶点并逐步模拟绘画过程。 每次模拟都需要计算从当前绘制集到所有顶点的距离，这可以通过类似 BFS 的更新或多源 BFS 每天来完成。 这已经花费了大约$O(mn^2)$在一个简单的实现中，每个起始顶点，然后如果我们尝试进行全局推理，则乘以所有图。 这远远超出了概念上的需要。 

真正的转变来自于观察到该条件在所有连通图中都是通用的。 这意味着我们不是在分析单个图表；而是在分析单个图表。 相反，我们要问的是，即使在对抗性距离几何条件下，序列是否也会强制最终覆盖。 

关键的观察是值$1$与所有其他距离有着根本的不同。 一旦绘制了一个顶点，就会执行一个操作$a_i = 1$立即将绘画传播到每个绘制顶点的所有邻居。 从那时起，该过程就成为从初始种子集开始的标准 BFS 扩展。 由于图是连通的，任何非空集合的 BFS 最终都会到达整个组件。 

这意味着如果序列至少包含一个$1$，无论图表如何，该过程总是成功的。 初始步骤可能会绘制一个复杂的子集，但是第一次出现$1$将过程转换为全图泛洪。 

如果没有$1$存在，每一步至少使用距离$2$。 在这种情况下，我们可以构建图，其中过程与图的某些部分永久“异相”。 一个简单而健壮的结构是路径图。 在路径上，距离的表现就像一条线上的绝对差。 没有距离$1$，该过程无法以保证覆盖所有起点的方式可靠地从部分填充的集合传播到相邻层。 人们总是可以放置顶点，以便某些层从任何可能的开始都不会被激活。 

因此，决策简化为一个简单的条件：是否$1$出现在序列中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟|$O(n^3 m)$每图推理|$O(n)$| 太慢/不必要|
 | 检查是否存在 1 |$O(m)$|$O(1)$| 已接受 |

 ## 算法演练

 ### 最优策略

 1. 阅读$n$和顺序$a_1, \dots, a_m$。 的价值$n$仅与在需要时构建反例有关，与决定答案无关。 
2. 扫描序列并检查是否有值等于$1$。 这是对该过程唯一重要的结构属性。 
3. 如果$1$存在，输出“SI”。 不需要构造，因为一旦出现单位距离传播步骤，每个连通图都会被完全绘制。 
4.如果没有$1$存在，构造一个简单的连通图来打破这个过程。 一条路径$n$顶点就足够了。 输出其边缘$(0,1), (1,2), \dots, (n-2,n-1)$。 

### 为什么它有效

 不变量是所有没有距离的传播步骤$1$保持至少为“距离粒度”$2$。 在任何图中，这都可以防止从部分绘制的区域保证邻接级别的扩散。 此刻a$1$出现后，这种不变量就会消失，因为每个绘制的顶点都成为其直接邻居的源，并且连接性迫使最终完全覆盖。 

因此，当序列包含单位步骤时，它通常是成功的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        a = list(map(int, input().split()))
        
        if 1 in a:
            print("SI")
        else:
            print("NO")
            # construct a simple path graph
            print(n - 1)
            for i in range(n - 1):
                print(i, i + 1)

if __name__ == "__main__":
    solve()
```该实现仅对每个测试用例执行线性扫描。 当一个$1$一经发现，我们立即接受该方案。 否则，我们会发出一个路径图，保证它是连通的并且是最小的。 

这里的一个常见错误是试图模拟绘画过程。 这是不必要的，因为答案仅取决于单位距离传播是否存在，而不取决于其他距离的详细相互作用。

 考虑一个情况$n = 6$,$a = [4, 1, 2, 3, 5]$。

| 步骤| 顺序检查 | 决定|

 我们输出一个路径图$0-1-2-3-4$。 从任何起始顶点开始，单位距离传播的缺失会阻止该过程可靠地填充所有配置中的所有顶点。

 ## 边缘情况

 如果序列仅包含大于 1 的值，则算法始终输出路径。 例如，与$n = 4$和$a = [2,2]$，输出是一个链$0-1-2-3$。 从任何顶点开始，缺乏单元传播会阻止该过程保证所有图的完全覆盖，因此返回一个固定的简单连通结构就足够了。 

如果$n = 2$时，路径构造仍然正常工作，产生单边。 这是最小连通图，只要不存在，它就可以正确地作为反例$1$存在。 

如果$1$出现在较长序列中的任何位置，例如$[3, 7, 1, 5]$，算法仍然立即接受。 后面的值并不重要，因为一旦单位距离传播可用，连接性将确保最终完全覆盖任何非空绘制集。
