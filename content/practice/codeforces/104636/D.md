 ## 解决方案
 ## 问题理解

 我们有一个管道系统$n$出口，每个出口都有一个尺寸$s_i$。 阿尔卡季倒入固定量的水$A$进入系统，但他可以在此之前阻止任何插座子集。 然后，水仅按照未堵塞的出水口的大小按比例分配。 

关键规则是，如果畅通的网点总大小$S$，然后出口$i$收到$\frac{s_i}{S} \cdot A$水的单位。 我们的目标是确保第一个出口至少收到$B$水的单位。 我们被要求尽量减少为实现这一条件而封锁的网点数量。 

第一个重要的观察结果是，阻塞仅改变分数的分母，而不改变第一个出口的分子。 第一个出口收到的金额仅取决于我们在保持第一个出口可用的情况下可以将剩余尺寸的总和缩小到多小。 

自从$n$可以大到$100{,}000$，任何尝试所有堵塞孔子集的解决方案都是不可能的。 即使检查所有子集也需要$2^n$操作，这远远超出了任何可行的限制。 即使对子集进行排序或重复重新计算总和也会导致至少$O(n^2)$最坏情况下的行为，也太慢了。 

的限制$A$和$B$很小（$\le 10^4$），但它们不是主要限制因素。 真正的瓶颈是孔的数量，这迫使我们陷入困境$O(n \log n)$或者$O(n)$解决方案。 

当第一个孔已经提供了足够的水而不阻塞任何东西时，就会出现微妙的边缘情况。 例如，如果$s_1$与其他相比非常大，或者如果$B \cdot \sum s_i \le A \cdot s_1$，那么答案为零。 幼稚的方法可能仍然会尝试删除不必要的元素。 

另一个边缘情况是我们必须阻止几乎所有内容。 如果除第一个孔之外的所有其他孔都很大，则最佳策略是仅保留 1 号孔并堵塞其余孔。 任何忽略分母中大洞影响的不正确的贪婪排序都会在这里失败。 

## 方法

 蛮力的想法很简单：尝试要阻塞的所有可能的孔子集，计算剩余大小的总和，并检查第一个孔是否至少接收到$B$水。 对于每个子集，我们重新计算总大小$S$，然后评估是否$\frac{s_1}{S} \cdot A \ge B$。 这是有效的，因为它直接模拟问题中描述的过程。 

然而，有$2^n$子集，甚至评估一个子集也需要$O(n)$如果天真地这样做。 这导致指数算法对于以下情况完全不可行$n = 100{,}000$。 

关键的观察结果是，堵住第一个洞以外的洞只会有助于减少分母$S$。 为了最大化第一个孔接收的水，我们希望最小化$S$，这意味着我们应该首先移除最大的孔。 每个移除的孔在每次移除操作中都会尽可能地减少分母。 

这将问题转化为选择我们删除多少个最大的非第一个元素，以使不等式成立。 一旦我们按降序对其他洞进行排序，我们就会贪婪地将它们一一删除，直到满足条件为止。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了|
 | 最优（排序+贪婪去除）|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们隔离第一个孔的大小，因为它始终被保留并直接决定最终分数的分子。 

我们计算所有孔尺寸的初始总和，因为这是阻塞任何东西之前的起始分母。 

我们通过验证是否不需要阻塞来检查是否不需要阻塞$A \cdot s_1 \ge B \cdot S$。 如果这已经成立，我们可以立即返回零，因为任何阻塞只会减少$S$，但我们已经很满意了。 

我们收集除第一个孔之外的所有孔，并按大小降序对它们进行排序。 这种顺序确保了当我们移除洞时，我们总是尽可能积极地减少分母。 

然后，我们迭代排序后的列表，一次从总和中删除一个洞，并计算我们执行的删除次数。 每次移除后，我们都会检查条件是否$A \cdot s_1 \ge B \cdot S$很满意。 当它第一次成为真时，我们停止并返回移除的孔的数量。 

### 为什么它有效

 该过程保持了一个明显的不变量：在任何一步，我们都精确地删除了最大的$k$所有非第一个孔中的孔。 这保证了当前总和$S$是通过删除任何可能实现的最小总和$k$洞。 由于条件仅取决于$S$，最小化$S$对于固定的移除次数是最优的，因此第一次不等式成立对应于所需的最小移除次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, A, B = map(int, input().split())
    s = list(map(int, input().split()))
    
    s1 = s[0]
    others = s[1:]
    
    total = sum(s)
    
    if A * s1 >= B * total:
        print(0)
        return
    
    others.sort(reverse=True)
    
    removed = 0
    for x in others:
        total -= x
        removed += 1
        if A * s1 >= B * total:
            print(removed)
            return

    print(n - 1)

if __name__ == "__main__":
    solve()
```该解决方案首先读取输入并将第一个孔与其余孔分开。 这很重要，因为第一个孔永远不会被堵塞，因此它的贡献在整个过程中保持不变。 

我们计算一次总和并将其用作不断变化的分母。 当初始配置已经满足要求时，早期退出检查可以避免不必要的排序和迭代。 

按降序对剩余的孔进行排序可确保每次移除都能最大限度地减少分母。 在迭代过程中，我们不断更新总数并检查叉乘形式的不等式，以避免浮点错误。 

最后的后退`n - 1`only triggers if even removing all other holes is insufficient, which corresponds to the case where only the first hole remains.

 去掉 1 个尺寸为 2 的孔后，总数变为 6，条件成立。 所以答案是1。 

这证实贪婪地去除最大的可用孔就足够了。

 ## 边缘情况

 一种边缘情况是不需要阻塞。 对于像这样的输入$n=1$,$s_1$是唯一的孔，分母永远不会改变，因此条件检查一次并立即返回零。 该算法通过早期的不平等检查来处理这个问题。 

另一种边缘情况是所有其他孔与第一个孔相比都非常大。 在这种情况下，贪婪循环将删除所有这些。 例如，如果$s = [1, 100, 100, 100]$，初始总和为 301，条件失败。 删除 100 三次后，总和变为 1，并且条件基本满足。 该算法正确返回 3，这是最佳值，因为任何较小的移除次数都会留下较大的分母。 

最后一种边缘情况是，如果直接实现，可能会出现浮点错误。 该解决方案通过将不等式两边相乘来完全避免这种情况，确保所有比较都使用整数完成，即使在最大输入大小的情况下也能保持逻辑稳定。
