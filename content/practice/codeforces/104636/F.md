 ## 问题理解

 我们给定一个长度的字符串`n`代表部分解码的基因组。 每个位置是四个核苷酸之一`A`,`C`,`G`,`T`，或未知字符`?`必须更换。 

最终目标是将这个不完整的字符串转变为完全指定的基因组，其中四个核苷酸中的每一个出现的次数完全相同。 由于总长度是固定的，这意味着解码后，每个`A`,`C`,`G`， 和`T`必须准确出现`n / 4`次。 如果`n`不能被4整除，这个条件从一开始就是不可能的。 

任务是确定这样的完成是否存在，如果存在，则通过替换每个完成来构造任何有效的完成`?`。 

输入尺寸非常小，`n ≤ 255`。 这立即排除了任何繁重的事情，例如指数搜索或子集上的复杂动态编程。 线性扫描解决方案就足够了。 

当字符串已经包含过多的某些核苷酸时，就会出现微妙的边缘情况。 例如，如果`n = 8`并且字符串已经包含五个`A`s，那么即使所有`?`都变成了非`A`字母，我们无法减少计数，所以答案是不可能的。 另一个边缘情况是当数量太少时`?`平衡所有四个角色的赤字，即使没有一个角色的代表性过高，这也会导致失败。 

## 方法

 暴力方法会尝试每个可能的替代方案`?`，分配每一个`A`,`C`,`G`， 或者`T`。 如果有`k`问号，这导致`4^k`的可能性。 即使对于中等`k`，这几乎立即超出了可行性。 和`n = 255`，最坏的情况变得天文数字般大。 

关键的观察是我们不需要独立探索选择。 重要的只是每个角色的最终计数。 既然目标已经确定，`n / 4`，我们可以首先计算每个核苷酸已经存在的数量。 这准确地确定了我们还需要多少个。 

如果任何核苷酸已经超过`n / 4`，这个任务是不可能完成的。 否则，我们将分发`?`贪婪地排列位置：用还有剩余配额的核苷酸一一填充。 由于时隙总数与总赤字完全匹配，因此当存在时，这总是会产生有效的解决方案。 

问题的结构从组合搜索简化为简单的计算：每个`?`只是分配给四个计数器之一的容量单位，直到所有计数器达到其所需值。

 | 蛮力 | O(4^k) | O(4^k) | O(n) | 太慢了|

 ## 算法演练

 我们通过跟踪我们已经拥有的每种核苷酸的数量以及我们仍然需要的数量来解决这个问题。 

1.首先检查是否`n`能被4整除。如果不能被4整除，我们立即输出`===`因为四个符号之间的平均分配是不可能的。 
2. 计算出现次数`A`,`C`,`G`， 和`T`在给定的字符串中，忽略`?`。 这给出了基因组的当前部分状态。 
3. 将目标计数计算为`target = n / 4`。 这是最终字符串中每个核苷酸必须达到的确切数量。 
4. 对于每个核苷酸，将其缺陷计算为`target - current_count`。 如果任何缺陷为负，意味着核苷酸已经出现太多次，则输出`===`。 这种情况无法修复，因为我们只能通过替换来添加字符`?`，不删除现有的。 
5. 从左到右迭代字符串。 每当我们遇到一个`?`，将其贪婪地分配给任何仍具有正赤字的核苷酸。 像这样的自然顺序`A`,`C`,`G`,`T`作品。 
6. 分配后，减少相应的赤字。 这确保我们永远不会超过任何核苷酸所需的数量。 
7. 处理完所有字符后，输出结果字符串。 

贪心选择是安全的，因为所有替换都是独立的单位，唯一的约束是匹配精确的总数。 不存在职位限制或职位之间的相互作用。 

### 为什么它有效

 该算法保持一个不变量，即在处理字符串的任何前缀后，分配的字母数量加上剩余的缺失恰好等于每个核苷酸所需的目标计数。 每个`?`仅分配给仍具有剩余容量的核苷酸，因此我们永远不会超过任何目标。 由于总数`?`等于赤字总和，每个赤字最终都会得到满足，保证存在一个完整的有效分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = list(input().strip())

    if n % 4 != 0:
        print("===")
        return

    target = n // 4
    cnt = {c: 0 for c in "ACGT"}

    for ch in s:
        if ch != '?':
            cnt[ch] += 1

    for c in "ACGT":
        if cnt[c] > target:
            print("===")
            return

    need = {c: target - cnt[c] for c in "ACGT"}

    for i in range(n):
        if s[i] == '?':
            for c in "ACGT":
                if need[c] > 0:
                    s[i] = c
                    need[c] -= 1
                    break

    print("".join(s))

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 最初的可行性检查确保我们在解决方案在结构上不可能时永远不会尝试施工。 这`need`字典跟踪每个字符的剩余容量。 

更换过程中，扫描`A`,`C`,`G`,`T`按固定顺序确保确定性，但任何顺序都有效。 关键是我们只在以下情况下分配`need[c] > 0`，防止任何类别填满。

 ## 边缘情况

 一种边缘情况是当`n`不能被 4 整除。例如，输入`5\nA?G?C`立即失败。 该算法在任何处理之前都会检查这一点，因此它会正确输出`===`无需扫描字符串。 

另一个极端情况是代表性过高。 考虑`4\nAAAA`。 这里的计数为`A`已经大于`1`，目标。 该算法检测到`cnt['A'] > target`并提前终止。 不`?`尝试进行处理，这可以避免不正确的掩蔽。 

第三种边缘情况是所有字符都`?`， 例如`4\n????`。 赤字都等于 1，并且每个`?`被贪婪地分配。 该不变量确保在四次分配后，所有缺陷精确地达到零，从而产生有效的基因组。
