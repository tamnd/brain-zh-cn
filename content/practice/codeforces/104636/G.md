 ## 解决方案
 ## 问题理解

 我们得到一串拉丁字母，其中可能出现大写和小写字符。 任务是确定该字符串是否是一个全字母组，这意味着来自`'a'`到`'z'`忽略大小写差异，在字符串中的某个位置至少出现一次。 

更具体地说，我们扫描单个字符序列，并想要检查存在的不同字母集是否涵盖了英语字母表的所有 26 个字母。 大小写并不重要，所以`'A'`和`'a'`出于检查覆盖范围的目的，它们被视为相同的符号。 

输入的大小很小，最多 100 个字符。 这立即排除了除了单次通过字符串之外的任何优化需要。 甚至一个$O(n \cdot 26)$或者$O(n^2)$这种方法在技术上是可行的，但自然的解决方案是线性时间和恒定的附加内存。 

主要的边缘情况来自字母的标准化方式和完整性检查方式。 一个天真的错误是将大写和小写视为不同的字符，这会错误地标记字符串，例如`"TheQuickBrownFoxJumpsOverTheLazyDog"`如果大小写未标准化，则不包含所有字母。 另一个微妙的问题是错误地假设字符串长度保证了覆盖范围，这是错误的，因为重复并不意味着多样性。 

## 方法

 解决该问题的一种暴力方法是，对于来自的每个字母`'a'`到`'z'`，扫描整个字符串并检查该字母是否以小写或大写形式出现。 这是正确的，因为它直接验证了全语法的定义。 然而，对于这 26 个字母中的每一个，我们最多可以扫描 100 个字符，从而导致最多 2600 个字符检查。 这在实践中已经是微不足道的了，但与必要的结构相比，该结构的效率很低。 

更好的方法是扭转观点。 我们不是问“每个字母都出现在字符串中吗？”，而是处理字符串一次并记录我们看到了哪些字母。 每个字符都被标准化为小写并插入到一个集合中或标记在一个大小为 26 的布尔数组中。最后，我们只需检查是否所有 26 个位置都被标记。 这是可行的，因为我们关心的属性是大小为 26 的固定宇宙中的成员资格，因此不需要重复扫描。 

关键的观察是字母表的大小是恒定的。 一旦我们认识到感兴趣的域是固定的且很小，我们就可以将问题简化为单遍累积问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(26·n) | O(26·n) | O(1) | O(1) | 已接受 |
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们逐个字符地处理字符串并维护一个跟踪出现过的字母的结构。 

1.初始化布尔数组`seen`大小为 26，全部设置为 false。 每个索引对应一个字母`'a'`到`'z'`。 这为我们提供了持续的会员跟踪。 
2. 迭代输入字符串中的每个字符。 对于每个字符，将其转换为小写，以便大写和小写版本映射到相同的索引空间。 这种标准化确保我们不会根据大小写重复计算字母。 
3. 计算字符索引为`ord(c) - ord('a')`。 这张地图`'a'`到 0，`'b'`到 1，依此类推，直到`'z'`25岁。马克`seen[index] = true`。 这记录了这封信至少被遇到过一次。 
4.处理完所有字符后，检查每个条目是否`seen`是真的。 如果任何位置仍然为假，则字符串中至少缺少一个字母，因此该字符串不是全语法。 
5、输出`"YES"`如果所有条目都为真，则输出`"NO"`。 

正确性取决于每个字母都是独立处理的，只有它的出现才重要，而不是频率。 

### 为什么它有效

 该算法保持了处理前 k 个字符后的不变量，`seen[i]`当且仅当对应于索引 i 的字母出现在这 k 个字符中时才为 true。 这一不变量成立是因为每个字符只更新一个索引，并且一旦设置值就不会再有任何操作会清除该值。 扫描结束时，状态准确地表示字符串中所有字母的集合，因此检查所有条目是否为 true 相当于检查字符串是否至少包含所有 26 个字母一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    seen = [False] * 26

    for c in s:
        if c.isalpha():
            idx = ord(c.lower()) - ord('a')
            seen[idx] = True

    if all(seen):
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该解决方案读取长度和字符串，但长度对于正确性来说并不是严格必需的。 核心逻辑是跟踪字母覆盖范围的固定大小布尔数组。 

致电给`lower()`确保处理时不区分大小写。 这`all(seen)`check 是每个字母必须至少出现一次这一要求的直接翻译。 

一个微妙的一点是，我们从不依赖频率，只依赖存在。 这就是为什么布尔数组足以代替计数器数组的原因。

 ## 边缘情况

 一个微妙的边缘情况是区分大小写。 如果算法没有规范化字符，那么`"AaBbCc...Zz"`会错误地显示为缺少许多字母，因为大写和小写将映射到不同的索引。 在我们的方法中，`lower()`确保两种形式映射到同一个槽，所以`"A"`和`"a"`两者都设置索引 0。 

另一个边缘情况是大量重复输入，例如`"aaaaaaaaaaaaaaaaaa"`。 此处，仅标记了一个索引。 布尔数组可以防止过度计数误导算法，因为重复更新不会改变正确性。 

最后的边缘情况是最小覆盖输入，例如`"abcdefghijklmnopqrstuvwxyz"`与缺少一个字母，例如`"abcdefghijklmnopqrstuvwxy"`。 在第一种情况下，所有索引都只填充一次，结果为`"YES"`。 在第二个例子中，索引 25 仍然为假，最终检查正确产生`"NO"`。
