 ## 解决方案
 ## 问题理解

 我们得到了一系列的咖啡馆访问序列，其中每个数字代表 Vlad 在该时刻访问的咖啡馆的索引。 咖啡馆可以重复，这意味着 Vlad 可能会多次访问同一家咖啡馆，而我们只关心访问顺序。 

任务是识别序列中出现的所有咖啡馆中最早访问的咖啡馆。 简单来说，对于每个不同的咖啡馆，我们都会查看它上次访问的时间，并且我们想要最后一次出现的咖啡馆距过去最远。 

这些限制允许最多 200,000 次访问，因此任何重复扫描阵列以查找每个不同咖啡馆的解决方案都会太慢。 二次方法需要扫描每个咖啡馆的整个阵列，在最坏的情况下可能会降低到大约 4 × 10^10 次操作，这在 2 秒内是不可行的。 这立即将我们推向线性或近线性解决方案。 

当多个咖啡馆仅出现一次时，就会出现微妙的边缘情况。 在这种情况下，答案是数组中最早出现的咖啡馆，因为它的最后一次访问也是该出现的咖啡馆。 当所有值都相同时，就会出现另一种边缘情况，答案就是那家咖啡馆。 

一个常见的错误是试图跟踪第一次出现而不是最后一次出现。 例如，在 2 1 2 3 这样的序列中，cafe 1 出现一次但不一定相关，而cafe 2 出现两次且最后一次出现较晚。 正确答案仅取决于最后的位置，而不取决于第一次出现。 

## 方法

 暴力解决方案会考虑每个不同的咖啡馆，扫描整个数组，并记录它最后出现的位置。 在计算完所有最后出现的情况后，我们选择其中的最小值。 虽然正确，但这种方法会对每个独特的咖啡馆重复进行全面扫描。 如果有 k 个不同的咖啡馆，最坏情况的时间复杂度变为 O(nk)，当所有值都不同时，它会退化为 O(n^2)。 

关键的观察是我们不需要重复重新计算最后出现的情况。 每次我们在数组中看到一家咖啡馆时，我们都可以覆盖其存储的位置。 扫描结束时，每个咖啡馆的存储值正是其最后出现的索引。 这将问题简化为一次通过数组，然后对记录的位置进行简单的选择步骤。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了|
 | 最佳（最后一次出现跟踪）| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 创建字典或数组`last`将每个咖啡馆索引映射到序列中的最新位置。 我们将其初始化为空，因为还没有看到咖啡馆。 
2. 从左到右迭代访问序列。 对于每个位置`i`， 店铺`last[a[i]] = i`。 这确保了每次我们看到一家咖啡馆时，我们都会覆盖它之前的记录，因此只保留最近的位置。 
3. 处理整个序列后，我们现在有了每个至少出现一次的咖啡馆的最后出现索引。 
4. 扫描所有记录的咖啡馆并选择存储索引最小的一家。 这对应于最近一次访问的咖啡馆是过去最远的。 
5. 输出该咖啡馆的索引。 

我们最后扫描所有记录的咖啡馆而不是维持运行最小值的原因是我们只在处理完整序列后才知道最终的最后位置。 

### 为什么它有效

 在迭代中的任何一点，`last[x]`正是cafe最近的位置`x`到目前为止已处理的前缀。 循环结束后，该值将成为整个数组中真正最后出现的值。 由于最终答案仅取决于最后出现的这些，因此选择其中的最小索引可以正确识别最后一次访问时间最早的咖啡馆。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

last = {}

for i, x in enumerate(a):
    last[x] = i

ans = None
best_pos = 10**18

for x, pos in last.items():
    if pos < best_pos:
        best_pos = pos
        ans = x

print(ans)
```第一次遍历数组会构建从咖啡馆到其最后看到的索引的映射。 枚举索引`i`自然就代表了访问时间。 每个作业都会覆盖之前的位置，这对于确保正确性至关重要。 

第二个循环比较存储的最后位置。 我们跟踪最佳位置和相应的咖啡馆。 使用较大的初始值可确保第一次比较始终成功。 

一个常见的陷阱是使用基于 1 和基于 0 的索引不一致。 这里我们在内部使用基于 0 的索引，但由于我们只比较相对顺序，因此无论索引基数如何，最终答案仍然正确。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | 一次记录最后出现的情况，一次传递不同的键 |
 | 空间| O(n) | 每个不同的咖啡馆最多存储一个条目 |

 该解决方案完全符合约束条件，因为对于 Python 中的线性传递来说，200,000 次操作是微不足道的，并且字典操作是按常量时间摊销的。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)

 ## 边缘情况

 ### 单次访问

 输入：```
1
10
```该算法存储`last[10] = 0`。 不存在其他条目，因此最小值一般为 10。输出是正确的，因为根据定义，唯一的咖啡馆既是第一个也是最后一个。 

### 所有咖啡馆都相同

 输入：```
5
3 3 3 3 3
```每次迭代都会覆盖`last[3]`，但值仍为 3，最终位置为 4。由于字典中只有一个键，因此会自动选择它。 这证实了重复覆盖不会影响正确性。 

### 所有不同的咖啡馆

 输入：```
4
8 1 6 2
```最后一张地图是`{8:0, 1:1, 6:2, 2:3}`。 最小索引为 0，因此选择咖啡馆 8。 这表明解决方案正确地简化为在不存在重复时查找最早出现的情况。
