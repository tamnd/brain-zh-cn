 ## 解决方案
 ## 问题理解

 我们得到了一份管家优势列表，我们需要决定琼恩·雪诺将支持哪些管家。 仅当存在至少一名强度严格较小的管家和至少一名强度严格较大的管家时，才支持一名管家。 

此条件独立应用于数组中的每个位置。 我们不是选择子集或修改值，我们只是检查每个元素是否“夹在”整个集合中某个较小值和较大值之间。 

输入大小可以达到10万个元素，每个值可以大到10^9。 这立即排除了将每个元素与所有其他元素进行比较的任何解决方案。 在最坏的情况下，二次方法将执行大约 10^10 次比较，这远远超出了两秒限制。 

当所有值都相同时，会出现微妙的边缘情况。 在这种情况下，没有任何元素同时具有严格较小和严格较大的邻居。 尽管有很多元素，但没有一个满足条件。 另一个极端情况是仅存在两个不同的值。 那么只有较大的值才有较小的邻居，只有较小的值才有较大的邻居，但没有任何元素可以同时满足这两者。 

## 方法

 检查每个管理员的条件的直接方法是扫描整个数组并搜索较小的值和较大的值。 对于每个索引，我们将运行两次扫描：一次是检测是否有任何值较小，另一次是检测是否有任何值较大。 这可以正确工作，因为它显式验证了定义中的条件。 

然而，这会导致总共 O(n^2) 次检查。 当 n 达到 100,000 时，这大约变成 10^10 次比较，这太慢了。 

关键的观察是值的条件仅取决于数组中的全局极值。 如果某个值大于全局最小值，则该值只能具有严格较小的元素。 类似地，如果它小于全局最大值，则它只能具有严格更大的元素。 这减少了识别哪些元素严格位于数组的最小值和最大值之间的问题。 

一旦知道最小值和最大值，就可以在恒定时间内对每个元素进行分类，从而将整个解决方案简化为单个线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将“具有至少一个较小的元素和至少一个较大的元素”这一条件转化为使用最小值和最大值的全局特征。 

1. 计算数组中的最小值。 这代表了可能的最小强度，因此任何等于它的元素都不能有严格更小的邻居。 
2. 计算数组中的最大值。 这代表了最大可能的强度，因此任何等于它的元素都不能有严格更大的邻居。 
3. 迭代所有元素并计算有多少个元素满足条件 value > 最小值且 value < 最大值。 这直接编码了数组中某处存在较小和较大元素的要求。 
4. 返回计数作为答案。 

关键思想是任何较小元素的存在等价于严格大于全局最小值，而任何较大元素的存在等价于严格小于全局最大值。 这消除了任何成对比较的需要。 

### 为什么它有效

正确性来自全局极值如何编码所有可能的比较。 如果某个元素等于最小值，则数组中任何位置都不存在严格小于的元素。 如果它大于最小值，则最小值本身保证存在更小的元素。 同样的推理对称地适用于最大值。 因此，满足两个不等式就相当于满足原来的条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    a = list(map(int, input().split()))
    
    mn = min(a)
    mx = max(a)
    
    ans = 0
    for x in a:
        if mn < x < mx:
            ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先读取数组并在线性时间内计算其最小值和最大值。 这两个值是评估每个元素所需的唯一全局信息。 

然后循环根据严格的不等式检查每个元素。 将条件仔细编写为 mn < x < mx 以确保排除边界元素。 一个常见的错误是使用非严格比较，这会错误地包含等于最小值或最大值的元素。 

整体结构故意简单：所有繁重的计算都被推入单次传递中，并且分类是在每个元素的恒定时间内完成的。

 ## 边缘情况

 当所有元素相同时，最小值和最大值都是相同的值。 在这种情况下，条件 mn < x < mx 对于每个元素都失败，因为没有数字可以严格位于相等的边界之间。 该算法正确返回零，因为两次比较都排除了每个元素。 

当只有两个不同的值时，例如像 [1, 1, 1, 5, 5] 这样的数组，最小值为 1，最大值为 5。1 和 5 都不严格位于它们之间，因此同样不计算任何元素。 这符合以下事实：任何选定的元素至少有一方面未能满足要求。 

当数组只有一个元素时，最小值和最大值都等于该元素，并且严格不等式会立即将其消除。 该算法自然返回零，无需任何特殊处理。
