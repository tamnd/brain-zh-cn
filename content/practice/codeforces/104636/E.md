 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 每个测试用例都由一个非常短的字符串组成，只有三个字符。 任务是当我们忽略字母大小写时，判断这个字符串是否代表单词“YES”。 允许任何大小写字母的混合，因此所有变体如“yes”、“Yes”或“YEs”都应该被接受。 

输出也是每个测试用例的。 对于每个输入字符串，如果在不区分大小写的比较下匹配“YES”，我们会打印一条确认信息，否则我们拒绝它。 

约束非常小：每个字符串的长度为 3，并且最多有 1000 个测试用例。 这意味着即使是最直接的逐字符处理在性能方面也是微不足道的。 任何为每个测试用例持续工作的解决方案就足够了，甚至稍微浪费的方法（例如重复转换字符串或与多个变体进行比较）仍然可以顺利通过。 

在长度或结构方面不存在棘手的隐藏边缘情况，因为每个输入都保证恰好是三个字母字符。 唯一的微妙之处是大小写标准化。 对于像“yes”或“yEs”这样的有效输入，像检查是否与文字字符串“YES”相等而不调整大小写这样的简单比较将会失败。 另一个潜在的错误是部分检查字母，例如仅比较第一个字符或使用假定统一大小写的词典编排技巧。 

一个有代表性的失败案例是：

 输入：```
yEs
```正确输出：```
YES
```一个简单的实现来检查`s == "YES"`会错误地打印 NO，因为即使字母在概念上匹配，情况也会有所不同。 

## 方法

 思考这个问题的强力方法是枚举混合大小写的单词“YES”的所有有效表示。 由于这三个字符中的每一个都可以独立地为大写或小写，因此有$2^3 = 8$可能的变体：

 是的，是的，是的，是的，是的，是的，是的，是的。 

我们可以存储这个集合并检查每个查询的成员资格。 这是正确的，并且每个测试用例以恒定的时间运行，但对于如此小的转换问题来说，它是不必要的间接。 它还在构建或存储集合时引入了可避免的开销。 

更清晰的观察是，在应用单个标准化步骤后，所有有效字符串都变得相同：将每个字符转换为相同的大小写。 如果我们将输入字符串转换为小写或大写，那么每个有效变体都将完全变为“yes”或“YES”。 这将问题简化为单个字符串比较。 

因此，我们不再处理多个模式，而是将每个测试用例减少为：

 将 s 转换为小写，然后与“yes”进行比较。 

这是关键的简化：目标词的结构是固定的，大小写是唯一的变化来源。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举所有变体 | 每次测试 O(1) | O(1) | O(1) | 已接受 |
 | 标准化案例并比较 | 每次测试 O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取测试用例的数量。 这定义了我们将执行多少次独立检查。 
2. 对于每个测试用例，读取 3 个字符的字符串。 由于长度是固定的，因此不需要预处理。 
3. 将字符串转换为规范化形式，通常为小写形式。 这消除了所有大小写区别，以便逻辑上相同的字符串映射到相同的表示形式。 
4. 将规范化字符串与目标“yes”进行比较。 如果它们完全匹配，则输入必须是“YES”的某种大小写变体，因此我们输出“YES”。 
5. 否则，输出“NO”。 

### 为什么它有效

 该算法依赖于这样一个事实：大小写转换是一种多对一映射，可以保留字母标识，同时消除格式差异。 每个有效的输入字符串都由与“YES”相同的字母以某种大小写顺序组成，因此在规范化之后，它们都会崩溃为单一规范形式。 任何字母不同的字符串在标准化后都不能变成“yes”，因此永远不会产生误报。 

这创建了一个干净的不变量：在步骤 3 之后，当且仅当原始字符串是“YES”的大小写排列时，字符串等于“yes”。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        s = input().strip()
        if s.lower() == "yes":
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```该解决方案独立处理每个测试用例。 这`.lower()`call 是唯一应用的转换，确保统一比较。 必须进行剥离才能从输入中删除换行符； 忘记这一点是错误比较的常见原因。 

打印语句使用大写输出，但根据问题陈述，任何大小写都是可接受的。 逻辑本身严格基于规范化的平等。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    output = []
    t = int(sys.stdin.readline())
    for _ in range(t):
        s = sys.stdin.readline().strip()
        if s.lower() == "yes":
            output.append("YES")
        else:
            output.append("NO")
    return "\n".join(output) + ("\n" if output else "")

 ## 边缘情况

 一种边缘情况是输入格式已正确但大小写混合，例如“Yes”。 算法将其转换为“yes”并匹配成功。 踪迹是：

 输入：“Yes”→小写变为“yes”→等于目标→输出YES。 

另一个边缘情况是完全不相关的字符串，例如“XyZ”。 标准化后它变成“xyz”，与“yes”不匹配，因此它正确地产生NO。 

不存在涉及长度或空输入的结构边缘情况，因为该问题保证了固定长度的字符串，因此正确性完全取决于规范化后的字符匹配。
