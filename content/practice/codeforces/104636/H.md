 ## 解决方案
 ## 问题理解

 我们正在模拟一家非常小的酒店，有十个房间，索引从 0 到 9。每个房间可以是空的，也可以只入住一名客人。 随着时间的推移，客人要么从两个入口之一到达，要么从特定房间离开。 

抵达时的行为是关键规则：如果客人从左侧入口进入，他们总是会入住最左边当前空着的房间。 如果他们从右边的入口进入，他们总是会选择最右边当前空着的房间。 离开是明确的：当我们看到数字 x 时，当前占用房间 x 的客人离开，使其再次空出。 

输入是这些事件的时间顺序。 我们必须按顺序模拟它们并维持十个房间的占用状态。 最终输出是一个 10 个字符的二进制字符串，其中每个位置表示该房间末尾是否被占用。 

事件计数方面的限制很大，最多可达 100,000 个操作。 这立即告诉我们任何解决方案都必须在恒定时间内处理每个事件。 由于状态空间很小（只有 10 个房间），我们不受内存或结构大小的限制，只需避免每个事件的重复扫描工作，这些工作将乘以 10 × 100,000 之类的东西，这仍然很好，但任何更复杂或间接的东西都是不必要的。 

天真的错误通常来自尝试使用复杂的数据结构动态重建任务或跟踪“谁在哪个位置”，或者在没有仔细结构的情况下对每个事件使用完整扫描重新计算最近的空房间。 尽管扫描 10 个位置是恒定的，但重复不必要的逻辑或对出发时的状态更新管理不善可能会导致不正确的结果。 

一种微妙的边缘情况是离开后重新使用房间。 例如，如果一个房间被释放，并且稍后另一个到达者应该根据邻近规则重新使用它，则算法必须正确地将其重新考虑为立即可用。 

另一个潜在的陷阱是误解，认为出发并不取决于到达顺序，而是直接取决于房间指数。 错误的方法可能会尝试跟踪每个入口队列中的客人，但这是不必要的且容易出错。 

## 方法

 简单的模拟之所以有效，是因为系统非常小。 我们维护一个大小为 10 的数组，表示每个房间是否被占用。 对于每次到达，我们从左到右或从右到左扫描以找到第一个空槽并分配它。 对于每次出发，我们只是将给定房间标记为空。 

这种强力方法在实践中已经是最佳的，因为扫描 10 个元素的时间是恒定的。 即使有 100,000 次操作，我们最多也执行大约 1,000,000 次原始检查，这是微不足道的。 

我们可以尝试通过为每侧最近的空闲房间维护单独的结构来“优化”，但这会增加复杂性而没有任何好处，因为重新计算 10 多个位置已经可以忽略不计。 

关键的见解是认识到状态空间是固定的且很小，因此直接模拟是预期的解决方案。 该问题旨在测试仔细的实现而不是算法优化。

 ## 算法演练

 我们维护一个列表`rooms`大小为 10，最初全为零。 

### 步骤

 1.初始化一个数组`rooms`长度为 10，所有值均设置为 0。 

这表示一开始所有房间都是空的。 
2. 按顺序处理输入序列中的每个字符。 

正确性取决于保留准确的时间顺序。 
3. 如果角色是`'L'`，从房间 0 向上扫描，直到找到第一个空房间。 

将其指定为已占用并立即停止。 

这样做的原因是该规则明确优先考虑最左边的可用插槽。 
4.如果角色是`'R'`，从房间 9 向下扫描，直到找到第一个空房间。 

将其指定为已占用并立即停止。 

这强制执行对称的最右边选择规则。 
5. 如果字符是数字`'0'`到`'9'`，将其转换为整数 x 并标记`rooms[x] = 0`。 

这直接模拟出发，释放指定的房间，无论之前的状态如何。 
6. 处理完所有事件后，将数组输出为 0 和 1 的字符串。 

### 为什么它有效

 每一步，`rooms[i]`准确反映是否有房间`i`在处理完之前的所有事件后被占用。 到达时始终选择所需方向上的第一个可用房间，出发时仅清除特定时段而不影响其他时段。 由于除了当前占用状态之外没有任何事件依赖于任何事物，因此维持此不变性可以保证最终配置的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    s = input().strip()

    rooms = [0] * 10

    for c in s:
        if c == 'L':
            for i in range(10):
                if rooms[i] == 0:
                    rooms[i] = 1
                    break
        elif c == 'R':
            for i in range(9, -1, -1):
                if rooms[i] == 0:
                    rooms[i] = 1
                    break
        else:
            idx = ord(c) - ord('0')
            rooms[idx] = 0

    print(''.join(map(str, rooms)))

if __name__ == "__main__":
    main()
```实现直接遵循仿真模型。 唯一微妙的部分是通过正确方向的线性扫描来处理到达。 由于数组大小固定为 10，因此每个事件的时间保持恒定。 

数字处理依赖于 ASCII 转换，这避免了任何解析开销并确保每次出发直接映射到房间索引。

 ## 边缘情况

 一个重要的边缘情况是同一房间的重复填充和清空。 考虑输入`L0L`。 第一次到达后，0号房间已满。 数字`0`再次释放它，最后`L`必须正确地重用房间 0 作为最左边的可用插槽。 该算法自然地处理这个问题，因为每次到达总是从数组的开头进行扫描，并且释放的槽立即可见。 

另一个边缘情况是交替到达，不断改变两端的占用率，例如`LRLRLR...`。 系统决不能因为状态过时而“跳过”空闲房间。 由于每一步都直接从重新计算可用性`rooms`，没有做出历史假设，因此重用始终反映当前配置。 

第三种情况是满员后离开。 即使所有房间都已满，数字事件也会立即释放一个房间，并且下一个到达者必须根据更新的状态正确选择。 由于每个操作都会更新相同的共享数组，因此不存在可能延迟可用性更新的隐藏状态。
