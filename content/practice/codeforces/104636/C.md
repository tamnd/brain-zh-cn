 ## 解决方案
 ## 问题理解

 我们得到一个学生列表，每个学生都由从 1 到 n 的整数 id 标识。 学生 1 是托马斯。 每个学生都有四项考试成绩，他们的整体表现是通过这四项成绩的总和来衡量的。 

所有学生均按总分降序排列。 如果两个学生的总分相同，则先选择 id 较小的学生来打破平局。 任务是确定托马斯在此排序中的最终位置。 

约束 n ≤ 1000 和得分值高达 100 意味着每个总得分以 400 为界。这立即表明任何 O(n^2) 或 O(n log n) 解决方案都非常快，因为即使 10^6 比较在 1 秒内也是微不足道的。 

主要的微妙之处在于打破平局的规则。 一个常见的错误是只严格计算较高的分数，而忘记相同的分数仍然需要比较 id。 另一种失败情况是仅按分数排序，而没有正确编码 id 排序，这会导致存在重复项时位置不正确。 

一个具体的边缘情况是多个学生与托马斯的总分相同。 例如，如果 Thomas 的总分是 390，另外两个学生也有 390，那么只有分数严格更高的学生才应该在他之前，而在相同分数中，由于 id = 1，Thomas 应该排在第一位。 如果不小心将“分数 ≥ Thomas 分数”计为较高排名，则会错误地将 Thomas 压低。 

## 方法

 一种直接的方法是计算每个学生的总分，然后按照所需的排序规则对学生进行排序，最后找到学生1在排序列表中的位置。 

这是有效的，因为排名定义实际上是一个全局排序问题：计算完所有总数后，任务就简化为按字典顺序对 (score, id) 对进行排序，其中分数递减，id 升序。 

一种蛮力的替代方法是将托马斯与其他所有学生进行比较，并计算有多少人排名在他之前。 如果学生的分数严格大于托马斯的分数，或者分数相等但他们的 id 较小，则该学生领先。 由于托马斯的 id 最小，因此没有任何相同分数的学生可以超过他，因此只有严格较大的分数才重要。 这一观察结果大大降低了逻辑性。 

蛮力思想已经在 O(n) 中运行，但是将其推广到任何学生将需要 O(n^2) 比较。 基于排序的方法更加统一，并且更接近排名系统的通常实现方式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 已接受但没有必要 |
 | 排序 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 ### 最佳方法

 1. 读取所有学生的成绩，并通过将四门科目相加来计算出他们的总成绩。 

这将问题转化为处理每个学生的单一可比较值。 
2. 将每个学生存储为一对，包含他们的总分和 ID。 

我们保留 id 因为打破平局取决于它。 
3. 按两个键对学生列表进行排序：首先按总分递减，然后按 id 递增。 

这直接将排名规则编码到排序比较器中。 
4. 扫描已排序的列表并找到 id 等于 1 的位置。 

此排序顺序中的索引就是答案，调整为从 1 开始的索引。 

### 为什么它有效

 计算总数后，学生之间的每次比较仅取决于两个值：score 和 id。 排序顺序与问题的排名规则完全匹配，这意味着排序后的数组是有效的最终排名。 由于排序产生的总顺序与所需的比较函数一致，因此保证学生 1 的位置与其排名相匹配。 不需要后期调整，因为所有平局决胜在排序过程中已经解决。 

## Python 解决方案

```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    students = []
    
    for i in range(1, n + 1):
        a, b, c, d = map(int, input().split())
        total = a + b + c + d
        students.append(( -total, i ))  # negative for descending sort
    
    students.sort()
    
    for idx, (_, sid) in enumerate(students, start=1):
        if sid == 1:
            print(idx)
            return

if __name__ == "__main__":
    main()
```该实现使用负分数来避免编写自定义比较器：Python 默认按升序排序，因此翻转符号即可实现降序排列。 id 保持原样，以便关系自然地首先解析为较小的 id。 

对已排序学生的循环是安全的，因为 n ≤ 1000，因此线性扫描可以忽略不计。

 ## 边缘情况

 一个重要的情况是所有学生的总分数相同。 在这种情况下，排序完全依赖于 id 排序。 由于学生 1 的 id 最小，因此排序顺序从 Thomas 开始，算法正确返回排名 1。 

另一种情况是托马斯得分最低，例如全零，而其他人得分最高。 排序将他排在最后，因为其他每个学生都有更高的键（-score 更负），因此他的排名变为 n，这与预期一致。 

最后一个微妙的情况是，多个学生共享托马斯的分数，但没有一个学生超过它。 由于 id 排序，排序可确保 Thomas 出现在所有人之前，因此即使存在许多并列，他的排名仍为 1。
