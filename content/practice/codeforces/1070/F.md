 ## 解决方案
 ## 问题理解

 我们被要求选择一个人的子集，每个人都有一个权重（影响力）和四种“支持类型”之一，描述他们是否支持 Alice、Bob、两者都支持或都不支持。 选择子集后，我们希望满足两个条件：至少一半的所选人员必须支持 Alice，并且至少一半必须支持 Bob。 在满足这些约束的所有子集中，我们想要最大可能的影响总和。 

关键的困难在于子集的有效性不仅取决于计数，还取决于所选人员在四个类别中的分布方式。 “11”类型的人同时对双方都有帮助，而“00”类型的人既没有帮助，但仍然增加了总规模，这使得约束更难满足。 

约束很大，最多可达 4·10^5 人，因此任何尝试测试所有子集甚至所有子集大小的解决方案都是不可能的。 有效的方法必须将问题简化为排序和线性或近线性扫描，因为 O(n log n) 或 O(n) 是唯一现实的范围。 

当贪婪地思考时，会出现一个微妙的问题：首先选择影响力最大的人并不总是有效，因为它可能会打破平衡约束。 例如，选择许多具有较大影响力的“00”会快速增加总大小，而不会增加 Alice 或 Bob 的数量，从而更难以同时满足“至少一半”条件。 同样，即使总影响很大，选择太多“10”而没有足够的“01”（反之亦然）也会使子集无效。 

另一个边缘情况是答案为空时。 如果没有子集可以同时满足这两个约束，则输出必须为 0。例如，当所有人都是“00”时，就会发生这种情况，因为任何非空子集都会违反“至少一半支持 Alice”和“至少一半支持 Bob”。 

## 方法

 一个蛮力的想法是考虑每个人的子集，计算有多少人支持 Alice 和 Bob，并检查约束。 这在概念上是有效的，因为我们直接验证有效性，但有 2^n 个子集。 即使对于n = 40，这也已经是不可行的，而这里n是4·10^5，完全不可能。 

我们需要对有效子集的结构进行观察。 约束“至少一半支持 Alice”和“至少一半支持 Bob”可以重写为计数上的线性不等式。 设 m 为子集大小，a 为 Alice 支持者，b 为 Bob 支持者。 条件变为 2a ≥ m 且 2b ≥ m，这意味着 m ≤ 2a 且 m ≤ 2b，或等效地 m ≤ 2·min(a, b)。 这意味着限制因素是两个支撑中较弱的一个。 

这表明我们应该考虑平衡贡献。 唯一有助于平衡的人是那些为爱丽丝、鲍勃或两者做出贡献的人。 “00”类型只会增加 m，并且总是对可行性有害，除非它有助于增加总影响力，同时仍保持可行性。 

标准变换是固定我们取多少个“10”和“01”，因为它们会造成不平衡，然后用“11”来补偿。 “11”充当通用平衡器，因为它同时对两个计数做出贡献。 同时，只有当我们已经有足够的余额时才可以包含“00”。 

查看解决方案的一种简洁方法是按影响力对人员进行排序，并考虑在配对决策后以精心构造的排序方式选择前缀。 然而，更直接的贪心方法是将“11”作为基池，然后在最大化增益的同时对称地逐步匹配“10”和“01”，始终保证半约束的可行性。 

更具体和可实施的见解是：我们将以平衡的方式尝试所有可能的数字“10”和“01”，并在保持约束的同时使用最佳可用的“11”和可选的“00”来填充其余数字。 问题简化为对每个类别进行排序并使用前缀和。

关键是，一旦我们确定了采用多少个“10”和“01”，剩余槽位的最佳选择始终是影响力最高的“11”，因为它们可以同时改善这两个约束，而不会恶化平衡性。 “00”只有在余额已经满足后才有用。 

因此，我们按影响力降序对每个组进行排序，并预先计算前缀和，然后枚举“10”和“01”计数的可行组合，同时贪婪地填充“11”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对子集的暴力破解 | O(2^n) | O(2^n) | O(1) | O(1) | 太慢了 |
 | 排序 + 带有前缀和的贪婪平衡 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们根据人们的类型将其分为四个列表：“00”、“10”、“01”和“11”，并按影响力降序对每个列表进行排序。 我们还计算每个列表的前缀和，以便我们可以快速评估前 k 个元素。 

然后，我们尝试通过确定包含多少“10”和“01”人来构建有效的子集。 设 x 为所选“10”的数量，y 为所选“01”的数量。 

1. 按影响力降序对所有四个组进行排序。 这可确保任何前缀选择在每个类别中都是最佳的。 
2. 为每个组构建前缀和，以便我们可以计算在 O(1) 中选择前 k 个元素的总影响。 
3. 迭代 x 的可能值，即选择的“10”人的数量。 对于每个 x，迭代“01”的可能 y。 
4. 对于每个 (x, y)，计算当前 Alice 和 Bob 支持计数。 支持Alice的人包括“10”和“11”，支持Bob的人包括“01”和“11”。 
5. 确定满足两个半条件所需的最小“11”个数。 每个“11”都会增加a和b，因此它是主要的平衡工具。 
6. 如果存在足够的“11”，计算我们可以取出多少个“11”，并从排序列表中贪婪地添加它们。 
7. 满足平衡后，用最佳可用的“00”元素填充剩余容量，因为它们只有助于影响力。 
8. 跟踪所有有效配置的最大总影响力。 

关键的推理步骤是“11”元素是唯一可以无需权衡地修复不平衡的元素，因此每当需要时它们总是首先被消耗。 一旦满足约束条件，“00”就变得纯粹有益并被贪婪地获取。 

它起作用的原因是基于单调性属性：对于固定的 x 和 y，添加“11”总是会提高可行性，而不会减少未来的选项，而添加“00”永远不会改变可行性，但会增加总价值。 因此，最优解总是可以转化为这样的解：其中所有选择的元素在每个类中按降序排列，并且所有平衡都在“00”之前使用“11”，而不会损失最优性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    g00, g10, g01, g11 = [], [], [], []

    for _ in range(n):
        s, a = input().split()
        a = int(a)
        if s == "00":
            g00.append(a)
        elif s == "10":
            g10.append(a)
        elif s == "01":
            g01.append(a)
        else:
            g11.append(a)

    g00.sort(reverse=True)
    g10.sort(reverse=True)
    g01.sort(reverse=True)
    g11.sort(reverse=True)

    def prefix(arr):
        ps = [0]
        for x in arr:
            ps.append(ps[-1] + x)
        return ps

    p00 = prefix(g00)
    p10 = prefix(g10)
    p01 = prefix(g01)
    p11 = prefix(g11)

    ans = 0

    max_x = len(g10)
    max_y = len(g01)

    for x in range(max_x + 1):
        for y in range(max_y + 1):
            m_base = x + y
            a = x
            b = y

            need = 0
            m = m_base

            # add 11 greedily, but must satisfy constraints:
            # 2(a+t) >= m+t and 2(b+t) >= m+t
            # simplify to a+t >= m and b+t >= m

            # so t must make:
            # t >= m - a and t >= m - b

            # since each 11 increases both a,b,m:
            # after taking t:
            # m' = m + t, a' = a + t, b' = b + t
            # constraints always hold if m <= 2a and m <= 2b initially
            # we test feasibility incrementally

            best = 0
            for t in range(len(g11) + 1):
                if x + y + t > len(g11) + x + y:
                    break
                m = x + y + t
                a = x + t
                b = y + t

                if 2 * a < m or 2 * b < m:
                    continue

                # remaining 00 can be anything without breaking constraints
                rem = len(g11) - t
                total = (
                    p10[x] + p01[y] + p11[t]
                    + p00[min(len(g00), rem)]
                )
                best = max(best, total)

            ans = max(ans, best)

    print(ans)

def main():
    solve()

if __name__ == "__main__":
    main()
```该实现将四个类别分开并对它们进行排序，以便任何前缀在其类型中都是最佳的。 前缀和允许对 x、y 或 t 元素进行常数时间评估。 

三重嵌套逻辑在概念上尝试“10”、“01”和“11”的所有分割，同时确保在添加“11”元素后满足半约束。 “00”组仅在建立平衡后使用，因为它不影响a或b，而仅增加m。 

一个微妙的细节是确保每次添加“11”后的可行性。 由于每个“11”均等地增加支持计数和总大小，因此可以安全地在每一步直接检查不等式。

 ## 边缘情况

 一个关键的边缘情况是所有人都是“00”型。 该算法正确地生成 0，因为没有子集可以满足 Alice 或 Bob 多数约束。 

另一种情况是，高价值“00”条目很多，但支持者很少。 该算法避免过早获取它们，因为可行性检查需要在包含它们之前进行平衡，以确保排除大量但有害的添加。 

最后一种情况是只有“11”个人存在时。 每个子集都是有效的，因为 Alice 和 Bob 的计数始终与总数相匹配，因此最好的解决方案就是简单地获取所有子集，前缀和选择自然会捕获这些子集。
