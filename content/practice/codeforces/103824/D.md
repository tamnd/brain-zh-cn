 ## 解决方案
 ## 问题理解

 我们得到了一系列不同的整数，代表即将到来的比赛的评分变化。 我们可以任意重新排序这些值，并将它们提供给一个模拟玩家最终实际参加多少场比赛的过程。 

该过程从计划的 2 场比赛开始。 播放器从第一个元素开始执行重新排序的序列。 在完成第 i 场比赛后，计划的总比赛次数可能会增加 2，但仅限于非常特定的局部模式：当前值必须为正值，先前值也必须为正值，并且当前正值必须严格大于先前值。 当这种情况发生时，计划的长度就会增加，有可能允许玩家继续超出最初计划的时间。 否则，什么都不会改变。 

至关重要的是，这个过程是连续的并且是自动停止的。 只要当前索引不超过当前计划长度，播放器就会继续。 一旦 i 达到计划限制，即使仍有未使用的元素，执行也会停止。 

我们的任务是排列数组，以使所有实际播放值的总和最大化。 

每个测试用例的约束 n 最多为 100,000，总和也有界，这意味着每个测试用例至少需要线性或接近线性的基于排序的行为。 任何二次或涉及多种排列的模拟都是立即不可行的。 

一个微妙的困难是停止点取决于排列本身。 我们不是简单地最大化前缀和，而是控制动态扩展的前缀，其长度取决于我们早期创建的增加的相邻正对的数量。 

打破天真的直觉的一种边缘情况是积极因素与消极因素交织在一起。 

例如，如果我们尽早排列大的正数，但按降序排列，例如 [5, 4, 3, 2]，则不会发生扩展，因为条件需要严格增加正数。 该过程将在长度为 2 时立即停止，失去大部分增益。 天真的贪婪的“把大数字放在第一位”彻底失败了。 

另一个失败案例是假设我们应该始终最大化扩展。 扩展增加了可达指数，但可能会强制包含有害的负值。 所以一味地最大化有效正链的数量也不是最优的。 

## 方法

 强力方法将尝试所有排列并模拟每个排序的过程。 对于每个排列，我们从左到右扫描，保持当前计划的长度，并应用扩展规则。 每次排列的成本为 O(n)，并且有 n！ 排列，即使 n = 10 也是不可能的。 

关键的观察结果是，只有积极价值观之间的局部结构对于延长这一过程很重要。 否定项从不参与扩展条件，它们仅通过在停止之前是否包含来影响答案。 这将问题分为两个相互竞争的目标：我们希望尽早消耗所有积极因素，并且我们希望避免由于过度扩展而引入太多消极因素。 

扩展规则仅取决于相邻的递增正对。 这表明，如果我们控制正数的排列方式，我们就可以直接控制过程扩展的次数。 

现在考虑一下我们真正想要这个过程做什么。 每次扩展都会使总长度增加 2。如果我们设法确保执行顺序中所有正值出现在任何负值之前，那么每个有助于扩展的播放位置都是有益的。 一旦正数用尽，进一步的扩展只会将负数带入播放的前缀中，这是有害的。

这导致结构简化：我们希望将所有正值放在第一位，并且其中我们希望最大化有效相邻递增对的数量，同时保持结构稳定和可预测。 保证正数中有效对的最大数量的最简单方法是按升序对它们进行排序。 每对相邻的正数都将满足扩展条件。 

负数最好以任意顺序放置在末尾，因为它们的内部顺序根本不影响扩展。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 + 模拟 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 对正项进行排序，附加负项 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们以区分正值和非正值的方式构建排列。 

1. 将数组分为正数和非正数。 原因是只有正值才能参与扩展触发，因此混合它们只会引入不必要的中断。 
2. 按升序对正值进行排序。 这确保了每对相邻的正数都满足触发扩展的两个条件：都是正数，并且第二个大于第一个。 这以受控方式最大化有效触发器的数量。 
3. 对剩余值（零和负数）任意排序，通常按升序排列，以便在达到这些值时，危害最小的值会更早出现。 
4. 首先输出所有正数，然后输出所有非正数。 

为什么这种排序有效需要了解该过程是如何演变的。 执行始终从正块开始，并且当它保留在该块内时，每个相邻对都有助于延长计划长度。 由于所有正数都是连续且排序的，因此该过程不断扩展，直到它有效地消耗了整个正数段。 

一旦过程进入非正段，就不会发生进一步的扩展，因为条件需要正值。 此时，该过程只是线性运行，直到达到最终计划的长度或耗尽数组。 

关键的结构不变性是所有扩展机会完全集中在排序的正前缀内，并且不会因负数的中断而错过有益的触发因素。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        pos = []
        neg = []

        for x in a:
            if x > 0:
                pos.append(x)
            else:
                neg.append(x)

        pos.sort()
        neg.sort()

        res = pos + neg
        print(*res)

if __name__ == "__main__":
    solve()
```实施直接反映了构造。 我们通过符号分隔值，因为只有正数才能有助于延长计划的匹配数量。 对正数进行排序可确保每个连续的对都可以有效触发扩展。 

负数块被附加在最后，因为在用尽所有正数之前包含负数只会减少总和，而不会创造新的扩展机会。 

一个微妙的实现细节是我们根本不需要模拟该过程。 最优排列的结构保证了动态停止规则总是以可预测的方式展开，因此显式模拟只会增加开销。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import defaultdict

    def solve():
        t = int(input())
        for _ in range(t):
            n = int(input())
            a = list(map(int, input().split()))
            pos = [x for x in a if x > 0]
            neg = [x for x in a if x <= 0]
            pos.sort()
            neg.sort()

    old_stdout = sys.stdout
    sys.stdout = io.StringIO()
    solve()
    out = sys.stdout.getvalue().strip()
    sys.stdout = old_stdout
    return out

# all negatives

# single positive dominant structure
assert run("1\n5\n5 1 2 3 -1") == "1 2 3 5 -1"

 ## 边缘情况

 一种边缘情况是没有正值。 在这种情况下，不会发生任何扩展，因此该过程只需获取前两个元素并停止即可。 对底片进行分类可确保我们最大限度地减少早期损失。 

另一种情况是所有值均为正值。 该算法按升序对所有内容进行排序，这迫使每个相邻对重复触发扩展。 这使得流程得到最大程度的扩展，确保没有任何价值被闲置。 

像 [100, -1, 2, 3, 4] 这样的混合大小写说明了分离的重要性。 如果我们排在第一，我们就会提前失去所有的延期机会。 通过将正数设置为 [2, 3, 4, 100]，我们确保了受控的链接，同时仍然最大化可达总和，并且负数被安全地推迟。
