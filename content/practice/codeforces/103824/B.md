 ## 解决方案
 ## 问题理解

 我们有一个带有尺寸的网格$n \times m$。 每个操作都允许 Rabbit 通过删除一些正数的行或一些正数的列来缩小网格。 关键点是操作不会一一删除任意行或列，它只是在单个步骤中将高度或宽度减少任何选定的量。 每次操作后，网格仍保持完整的矩形。 

重复该过程，直到剩余矩形的面积在 40 到 60 之间（含 40 和 60）。 任务是确定所需的此类收缩操作的最小数量，或者报告这是不可能的。 

尽管这些规则听起来像是我们“删除行或列的集合”，但效果更简单：每个操作都让我们替换$x$中的任意整数$[1, x-1]$，或替换$y$中的任意整数$[1, y-1]$。 所以最终的矩形可以是任何$x' \times y'$这样$1 \le x' \le n$和$1 \le y' \le m$，每次我们触及一个维度，我们就支付一次操作。 

约束条件很大，最多可达$10^6$测试用例和尺寸高达$10^9$，因此任何尝试逐步模拟收缩的解决方案都是立即不可能的。 该结构表明我们必须直接跳转到候选最终维度而不是模拟操作。 

一个微妙的点是每个操作只改变一个维度。 所以达到最后一对$(x', y')$仅取决于我们是否更改过行以及是否更改过列，而不取决于中间状态。 

一个天真的误解是认为我们必须逐渐缩小矩形，但这是不必要的，因为一个操作可以直接跳转到任何更小的值。 

因此，输出纯粹是选择一个可达的最终矩形，其面积为$[40, 60]$这最大限度地减少了我们修改的维度。 

当初始矩形已经有效、仅需要调整一个维度以及必须更改两个维度时，就会出现边缘情况。 

例如，如果$n = 10, m = 5$，面积为 50，因此答案为 0。如果$n = 10, m = 4$，面积为 40，因此又为 0。如果$n = 10, m = 6$，面积是 60 所以 0。但是如果$n = 10, m = 7$，面积为 70，我们必须至少缩小一维。 

一个常见的错误是假设我们总是可以通过直接调整面积在一次操作中获得有效的矩形。 这是错误的，因为单独改变一个维度会迫使另一个维度保持固定，并且生成的区域可能完全跳过目标范围。 

## 方法

 强力解释是模拟行和列减少的所有可能序列。 从$(n, m)$，每一步都允许过渡到任何$(x', m)$或者$(n, y')$，然后递归地探索所有可能性。 这很快就会爆发，因为每个州都分支成$O(n + m)$的可能性，即使数值缩小，分支因子仍然巨大。 这种方法在限制条件下立即失败。 

关键的观察是每个维度的操作都是独立的。 如果我们决定最终高度$x$，我们支付一次操作如果$x \ne n$。 同样，选择最终宽度$y$花费一次操作如果$y \ne m$。 所以每个候选最终矩形的成本为$\{0, 1, 2\}$取决于有多少尺寸与原始尺寸不同。 

这将问题简化为小型有限搜索：我们只关心面积在 40 到 60 之间的矩形。由于面积范围很小，我们可以枚举所有可能的目标区域以及这些区域的所有因子对。 

对于每个有效分解$t = x \cdot y$，我们检查是否$x \le n$和$y \le m$，然后计算成本：

 成本是$[x \ne n] + [y \ne m]$。 

我们采用所有有效配置中的最小值。

 ## 算法演练

 ### 算法演练

 1. 将有效目标区域范围固定为 40 到 60（含），因为只有这些最终状态才重要。 
2. 对于每个整数$t$在此范围内，枚举所有因子对$(x, y)$这样$x \cdot y = t$。 这可以通过迭代来完成$\sqrt{t}$。 每对代表一个可能的最终矩形。 
3.对于每个因子对，检查是否可以嵌入到当前网格中，意思是$x \le n$和$y \le m$。 如果没有，请将其丢弃。 
4. 如果有效，计算达到该目标的成本：

 如果$x = n$，不需要行操作； 否则需要进行一行操作。 

如果$y = m$，不需要列操作； 否则需要一列操作。 
5. 跟踪所有有效对的最低成本。 
6. 如果不存在有效对，则输出-1。 否则输出最小成本。 

### 为什么它有效

 每个操作仅更改一个维度，但可以在一次移动中将其设置为任何较小的值。 这使得每个维度在操作计数方面都是独立的：它要么保持不变，要么只修改一次。 因此，每个可达的最终矩形完全对应于选择是保留还是更改每个维度，成本等于更改的维度数。 由于每个可行的最终状态必须具有固定的小范围内的面积，因此枚举所有候选状态可以保证我们不会错过任何最佳配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    LOW, HIGH = 40, 60
    
    # Precompute all candidate (x, y) pairs for areas 40..60
    candidates = []
    for t in range(LOW, HIGH + 1):
        for x in range(1, int(t ** 0.5) + 1):
            if t % x == 0:
                y = t // x
                candidates.append((x, y))
                if x != y:
                    candidates.append((y, x))
    
    for _ in range(T):
        n, m = map(int, input().split())
        
        ans = 10**9
        
        for x, y in candidates:
            if x <= n and y <= m:
                cost = 0
                if x != n:
                    cost += 1
                if y != m:
                    cost += 1
                if cost < ans:
                    ans = cost
        
        print(-1 if ans == 10**9 else ans)

if __name__ == "__main__":
    solve()
```该解决方案预先计算面积在允许范围内的所有可能的最终矩形形状。 这避免了为每个测试用例重新计算因子对。 

对于每个测试用例，我们只需检查哪些候选矩形适合原始网格。 成本计算直接反映了我们是否需要执行行缩减、列缩减或两者都执行。 

一个常见的实现陷阱是忘记包含因子对的两个方向，因为$x \times y$和$y \times x$代表不同的网格形状。 另一个微妙的问题是确保我们正确地将“维度上无操作”视为成本为零，这恰好发生在该维度与原始维度匹配时。 

## 工作示例

 ### 示例 1

 输入：```
n = 20, m = 20
```我们测试有效的目标区域。 一个可能的目标是$4 \times 10 = 40$，但是 10 超过 20？ 实际上有效，因为 10 ≤ 20, 4 ≤ 20。所以它是可达的。 然而成本是 2，因为两个维度都发生了变化。 

另一位候选人是$5 \times 8 = 40$，对于成本 2 也有效。 

我们检查 [40,60] 中的所有对，至少有一对适合，所以答案是 2。 

| 步骤| (x, y) | 有效的？ | 成本|

 | 40 = 4×10 | 是的 | 是的 | 2 |
 | 50 = 5×10 | 50 = 5×10 | 是的 | 是的 | 2 |
 | 60 = 5×12 | 是的 | 是的 | 2 |

 最终答案：2

 这表明该算法不是在寻找单一分解，而是在所有可能的分解中寻找最便宜的分解。

```我们检查面积为 40-60 的矩形是否适合。 受约束的最大可能区域是 49，因此只有 49 以内的候选区域才重要。 

我们测试 40-49，除了 49 = 7×7 之外，没有发现两边都 ≤ 7 的因子对。

 这种情况证实了当原始矩形已经匹配有效目标区域时，不需要进行任何操作。

 ## 边缘情况

 当起始矩形已经有有效范围内的面积时，算法立即找到候选矩形$(n, m)$如果它位于[40,60]，则产生成本为0。这避免了不必要的减少考虑。 

当只有一个维度需要调整时，例如$100 \times 1$，该算法识别出有效的分解，例如$50 \times 1$或者$40 \times 1$如果可能，并正确分配成本 1，因为只有一个维度不同。 

当不存在符合界限的因式分解时，例如$6 \times 6$，每个候选人都被拒绝$x \le n, y \le m$检查，算法正确返回 -1，无需对操作进行更深入的推理。
