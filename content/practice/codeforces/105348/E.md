 ## 解决方案
 ## 问题理解

 我们有一棵树，对于每个节点，我们想要测量如果我们强制该节点位于路径上的某个位置，路径会变得有多大。 

更准确地说，修复一个节点`i`。 我们查看每对可能的节点`(u, v)`在树中，这样唯一的简单路径来自`u`到`v`经过`i`。 在所有这些对中，我们取最大可能的距离`u`和`v`，其中距离是路径上的边数。 该最大值就是节点的答案`i`。 

因此，对于每个节点，我们并不是要求它在整棵树中的偏心率，而是要求约束通过它的最佳类似直径的路径。 

输入量大：最多有1000个测试用例，所有测试的节点总数最多为`10^5`。 这立即排除了任何每个节点二次甚至每个测试用例二次的解决方案。 任何尝试重新计算所有对距离或从每个节点重复运行 BFS 的操作都会失败。 每个测试用例需要一个线性或近线性的解决方案。 

一个天真的陷阱是假设每个节点的答案只是树的直径。 当节点不在任何直径路径上时，这是错误的。 例如，在以节点 1 为中心的星形树中，直径为 2，但对于叶节点`i`，穿过它的最佳路径是叶子到中心到另一叶子，仍然是2，所以匹配。 然而，在更倾斜的树中，远离直径端点的节点具有严格更小的限制直径。 

另一个常见的错误是尝试为每个节点计算整个树中最远的节点，然后使用该距离两次。 失败是因为最佳路径的两个端点`i`必须位于不同的分支`i`，在有根意义上不处于同一子树方向。 

一个小的说明性失败案例：

 输入：```
1
4
1 2
2 3
3 4
```预期输出：```
3 2 2 3
```一种天真的“全局直径无处不在”方法将输出所有 3，但节点 2 或 3 无法使用相对侧的两端来实现通过它们的长度为 3 的路径。 

## 方法

 蛮力的想法很简单。 对于每个节点`i`，考虑每一对`(u, v)`,检查它们之间的路径是否经过`i`，并计算最大距离。 在树中，检查是否`i`位于之间的路径上`u`和`v`可以使用 LCA 逻辑或通过检查距离来完成，但即使我们优化该检查，枚举所有对也是如此`O(N^2)`每个节点，导致`O(N^3)`总体而言是最糟糕的解释。 即使是更仔细的每个节点 BFS 方法仍然需要成本`O(N^2)`每个测试用例对于`10^5`总节点数。 

关键的观察是修复节点`i`将树分成几个连接的组件`i`被删除。 任何经过的有效路径`i`必须从这些组件之一开始并以不同的组件结束。 所以对于节点`i`，通过它的最佳路径必须连接两个不同的分支`i`, 向上经过`i`作为唯一的桥梁。 

这将问题简化为理解，对于每个邻居子树`i`，当远离时，该子树内可到达的最远节点是多少`i`。 如果我们知道，对于每个相邻方向，从`i`，那么答案为`i`成为两个不同方向的最佳总和。 

这本质上是一个“带重根的树DP”问题。 我们首先计算向下的深度，然后向上传播贡献，以便每个节点知道来自各个方向的最佳值，而不仅仅是其自己的子树。 

我们最终得到每个节点的所有邻居方向中的两个最佳深度，并且限制直径是它们的总和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 最坏情况 O(N3) | O(N) | 太慢了 |
 | 具有重新生根功能的树 DP | 每个测试用例 O(N) | O(N) | 已接受 |

 ## 算法演练

 我们任意地对树进行根，通常在节点 1 处，并计算两个 DP 状态：从节点到其子树的最佳向下路径，然后重新根来自父端的最佳贡献。 

1. 以节点 1 为树根并运行 DFS 进行计算`down[u]`，最大距离`u`进入其子树。 对于每个孩子`v`,`down[u] = max(down[u], down[v] + 1)`。 这捕获了完全包含在每个子子树中的最佳链。 
2. 在同一个 DFS 或第二遍期间，还跟踪其中两个最大值的每个节点`(down[child] + 1)`超过所有的孩子。 这些代表从以下位置开始的最佳两个向下分支`u`。 
3. 现在运行一个重新根DFS来计算`up[u]`，最佳距离是从`u`向上穿过其父级进入树中不存在的部分`u`的子树。 当从父级转移到子级时，我们必须重新计算贡献，排除该子级的子树，这就是为什么我们需要前两个子级贡献。 
4. 对于每个节点`u`，答案是通过组合来自的最佳两个“分支深度”获得的`u`。 这些分支来自它的两个子分支（通过`down`）及其父方（通过`up`）。 我们将所有候选分支长度收集到一个小列表中，并取两个最大值的总和。 
5. 输出每个节点的总和。 

我们保留前两个贡献而不仅仅是最大值的原因是通过节点的最佳路径必须使用两个不同的分支。 使用同一子树两次是不可能的，因为路径必须在节点处分叉。 

### 为什么它有效

 在任意节点`u`，去除`u`将树分解为不相交的组件，每个邻居一个。 任何经过的有效路径`u`必须选择两个不同的组件，并在每个组件中选取尽可能最长的链。 民主党表示`down`和`up`精确编码每个组件中可用的最大链长度。 由于每个组件的最佳贡献都是独立捕获一次，因此将两个最大的贡献结合起来会产生最佳对。 没有路径可以超过这个，因为任何路径通过`u`被限制为恰好选择两个不同的相邻组件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        g = [[] for _ in range(n + 1)]
        for _ in range(n - 1):
            u, v = map(int, input().split())
            g[u].append(v)
            g[v].append(u)

        parent = [0] * (n + 1)
        order = []
        
        stack = [1]
        parent[1] = -1
        
        while stack:
            u = stack.pop()
            order.append(u)
            for v in g[u]:
                if v == parent[u]:
                    continue
                parent[v] = u
                stack.append(v)

        down = [0] * (n + 1)

        for u in reversed(order):
            best1 = 0
            best2 = 0
            for v in g[u]:
                if v == parent[u]:
                    continue
                val = down[v] + 1
                if val > best1:
                    best2 = best1
                    best1 = val
                elif val > best2:
                    best2 = val
            down[u] = best1

        up = [0] * (n + 1)
        ans = [0] * (n + 1)

        children = [[] for _ in range(n + 1)]
        for v in range(2, n + 1):
            children[parent[v]].append(v)

        stack = [1]
        up[1] = 0

        while stack:
            u = stack.pop()

            best1 = best2 = -1
            best_child = [-1] * (len(g[u]) + 1)

            for v in g[u]:
                if v == parent[u]:
                    continue
                val = down[v] + 1
                if val > best1:
                    best2 = best1
                    best1 = val
                elif val > best2:
                    best2 = val

            for v in g[u]:
                if v == parent[u]:
                    continue
                use = best1
                if down[v] + 1 == best1:
                    use = best2
                up[v] = max(up[u] + 1, use + 1 if use != -1 else 0)
                stack.append(v)

            candidates = []
            candidates.append(up[u])
            for v in g[u]:
                if v == parent[u]:
                    continue
                candidates.append(down[v] + 1)

            candidates.sort(reverse=True)
            ans[u] = candidates[0] + (candidates[1] if len(candidates) > 1 else 0)

        print(*ans[1:])

if __name__ == "__main__":
    solve()
```该解决方案的结构分为两次遍历树。 第一遍计算子树深度（`down`）使用反向 DFS 顺序。 第二遍向上传播贡献（`up`）同时确保当我们将信息传递给子级时，我们使用预先计算的前两个子级值排除该子级自己的子树贡献。 

每个节点的最终答案是通过收集该节点上可用的所有有向分支长度形成的，其中包括其父端贡献和所有子端贡献。 排序或选择前两个值会给出最佳的不相交方向对。 

一个微妙的点是确保计算时“排除子”逻辑正确`up[v]`。 如果不维护第二好的子贡献，我们会意外地重用相同的子树，这会破坏正确性。

 ## 边缘情况

 单节点树是最简单的边界。 算法初始化`down[1]=0`并且没有孩子，因此候选列表仅包含`up[1]=0`，产生答案 0，这与除了平凡节点之外不存在路径的事实相匹配。 

星形树检查是否正确处理多个相同的分支。 每片叶子向中心贡献的分支长度为 1，最好的两个总是 1 和 1，从而限制中心直径为 2，叶子的直径也为 2。 重新生根过程确保叶子正确继承中心贡献作为其向上路径。 

长链确保深度传播在两个方向上起作用。 每个内部节点都会看到一个向上的长分支和一个向下的长分支，并且算法一致地选择它们，而不混合来自同一侧的贡献。
