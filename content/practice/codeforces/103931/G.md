 ## 解决方案
 ## 问题理解

 我们得到了一个由两个参数描述的单一武器模型和一个比赛片段的重播。 武器最多交易$B$每颗子弹的伤害和射速为$R$每分钟转数。 由此我们可以推断出子弹的发射频率。 回放告诉我们玩家累计了总伤害值$D$在一段时间间隔内$S$秒，从第一颗子弹发射的那一刻到最后一颗子弹落地的那一刻。 

任务不是模拟战斗，而是确定所报告的伤害在武器的射击限制下是否可能发生。 如果伤害超过任何有效的射击序列可以产生的伤害，我们必须输出玩家肯定在作弊。 

关键的限制是发射率。 如果$R > 0$，两发子弹之间的最短时间为$60 / R$秒。 这意味着在给定时间窗口内可以发射的最大子弹数。 如果$R = 0$，武器根本无法开火，因此任何正伤害都立即意味着作弊，除非伤害为零。 

微妙之处在于如何解读时间窗口$S$。 由于它被定义为从第一颗子弹到最后一颗子弹的时间，所以射击次数$k$满足最后一次射击发生的时间$S$，但第一枪的时间是$0$。 因此，$k$子弹需要$k - 1$间隔，每个至少$60 / R$， 意义：$$(k - 1) \cdot \frac{60}{R} \le S$$所以：$$k \le \left\lfloor \frac{S \cdot R}{60} \right\rfloor + 1$$边缘情况非常重要。 如果$R = 0$， 然后$k = 0$除非没有造成任何伤害。 如果$S = 0$，该窗口只允许单次射击，因为第一颗子弹和最后一颗子弹在时间上重合。 

试图枚举所有可能的射击序列或模拟每毫秒射击的幼稚方法是不必要的并且容易出错，但更重要的是，它将难以正确解释浮点冷却时间。 

另一个常见的陷阱是忽略每颗子弹的伤害上限为$B$，但每颗子弹的实际伤害可能会更少。 然而，由于我们只是检查是否可以造成伤害，因此我们假设最好的情况：每颗子弹都会造成$B$伤害，因此所需的最少子弹数量为：$$\left\lceil \frac{D}{B} \right\rceil$$所以问题简化为检查是否：$$\left\lceil \frac{D}{B} \right\rceil \le \text{max bullets allowed by firing rate and time}$$对零情况进行特殊处理。 

## 方法

 暴力解释将尝试模拟射击时机。 我们可以以小增量迭代时间，在冷却时间允许的情况下安排射击，并累积伤害。 这在概念上是正确的，但它引入了不必要的复杂性和精度问题。 和$S$高达 2000 秒和 RPM 高达 2000，模拟可能需要考虑每个测试用例数千个潜在的点火事件，并且最多$10^3$测试用例这变得低效且脆弱。 

关键的观察是我们根本不需要模拟时间。 射击约束直接转化为一个间隔内射击次数的简单限制。 类似地，伤害约束降低到所需射击的下限。 一旦两个量都表示为整数，问题就变成了直接比较。 

因此，我们不是对游戏玩法进行建模，而是将物理原理转换为两个整数：最大可能的子弹和最小所需的子弹。 如果所需数量超过最大数量，则肯定存在作弊行为。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 模拟 | O(T·S·R)| O(1) | O(1) | 太慢/脆弱|
 | 直接数学| O(T)| O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 阅读$B, R, D, S$。 这些定义了每颗子弹的伤害、射速、总伤害和时间窗口。 
2.如果$R = 0$，武器无法开火。 在这种情况下，如果$D > 0$，输出“卦！” 因为任何损坏都是不可能的。 否则输出“ok”。 
3.如果$B = 0$，每颗子弹都不会造成伤害。 如果$D > 0$，无论怎样射击都不可能，所以输出“卦！”。 如果$D = 0$，它始终有效。 
4. 计算达到伤害所需的最少子弹数量$D$。 由于每颗子弹最多贡献$B$， 这是：$$\text{need} = \left\lceil \frac{D}{B} \right\rceil$$这确保了我们假设每颗子弹的伤害是最佳的。 

1. 计算射速允许的最大子弹数量$S$秒。 每次拍摄都需要$60 / R$第一个之后的冷却秒数。 重新排列给出：$$\text{max} = \left\lfloor \frac{S \cdot R}{60} \right\rfloor + 1$$这说明了零时间处的第一个项目符号。 

1.如果$\text{need} > \text{max}$，输出“gua!”，否则输出“ok”。 

### 为什么它有效

 该算法将所有有效的射击序列压缩为单个约束：射击次数受时间和速率限制，与确切的计时无关。 任何有效序列都对应于具有固定最小间隙的射击时间的非递减序列，并且此类序列完全由间隔内有多少镜头来表征。 另一方面，通过分配完整的伤害积累来最大化$B$每次射击都会造成伤害，因此可行性仅取决于是否存在足够的射击来达到目标$D$。 由于这两个约束都成为严格的整数界限，因此比较它们足以确定可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        B, R, D, S = map(int, input().split())

        if D == 0:
            print("ok")
            continue

        if R == 0 or B == 0:
            print("gua!")
            continue

        need = (D + B - 1) // B
        max_shots = (S * R) // 60 + 1

        if need > max_shots:
            print("gua!")
        else:
            print("ok")

if __name__ == "__main__":
    solve()
```实现直接遵循派生的约束。 吊顶划分`(D + B - 1) // B`无需浮点运算即可安全地计算所需的项目符号。 最大射击公式全程使用整数运算，避免了精度问题。 特殊情况$R = 0$和$B = 0$尽早处理以防止无效算术。 

## 工作示例

 考虑第一个样本输入：$B = 38, R = 156, D = 152, S = 1$。 

我们将所需的项目符号计算为$\lceil 152 / 38 \rceil = 4$。 射速允许：$$\left\lfloor \frac{1 \cdot 156}{60} \right\rfloor + 1 = 2 + 1 = 3$$由于 4 超过 3，所以输出为“gua!”。

 | --- | --- |
 | d | 152 | 152
 | 乙| 38 | 38

 这显示了一种经典的违规行为，其中伤害意味着在时间窗口内的射击次数超出了物理上可能的次数。 

现在考虑一个有效的案例：$B = 99, R = 51, D = 9, S = 10$。 

所需子弹：$$\lceil 9 / 99 \rceil = 1$$最大拍摄次数：$$\left\lfloor \frac{10 \cdot 51}{60} \right\rfloor + 1 = 8 + 1 = 9$$| 步骤| 价值|

 这证实了当损坏相对于每次射击的容量较小时，约束条件是可以满足的。

 ## 边缘情况

 当$R = 0$，算法立即将任何正面损坏归类为不可能的。 这可以避免除以零或错误地假设某些隐式触发功能。 例如，输入$B=99, R=0, D=1, S=10$直接导致“卦！”，因为无法发射子弹。 

什么时候$S = 0$，该公式仍然可以正确运行，因为$(S \cdot R) // 60 + 1 = 1$，这意味着时间轴中只能存在一颗子弹。 如果$D$需要不止一颗子弹，不平等自然就会失败。 

什么时候$B = 0$，除非$D = 0$，因为没有有限数量的零伤害子弹可以达到正目标。 早期检查可以防止无效的上限划分。 

这些案例表明，所有病态输入都可以干净地分解为整数比较，而不需要特殊的模拟逻辑。
