 ## 解决方案
 ## 问题理解

 我们得到一个长度为隐藏的字符串$n$，其中每个字符都是左括号或右括号。 我们不能直接看到字符串，但是我们得到了一组区间约束。 每个约束指定一个子串$[l, r]$并告诉我们“该子字符串中出现的‘(’比‘)’多了多少个”。 

同样地，如果我们赋值$+1$到 '(' 和$-1$到')'，那么每个查询都会给出这个未知数组的一段的总和。 任务是重建与所有这些和一致的任何有效括号序列，同时确保最终序列是正确的平衡括号序列。 如果约束相互矛盾或者不能对应于任何有效的括号序列，我们必须输出失败。 

隐藏字符串被限制为有效，这意味着前缀余额永远不会变为负数并且总和为零。 这增加了超出任意性的结构$\pm 1$作业：我们不仅求解线性系统，而且求解具有前缀可行性的约束系统。 

约束条件可达$n = 3000$，所以任意$O(n^3)$推理可能仍然可以接受，但更糟糕的是太慢了。 更重要的是，问题结构表明我们需要保持许多区间和的一致性，这自然会转化为前缀和和差异约束，而不是暴力破解字符串。 

当查询局部同意但全局与有效括号序列相矛盾时，就会出现微妙的失败情况。 例如，如果我们过早强制前缀太正，那么我们以后可能无法在保持非负平衡的同时关闭括号。 另一种失败情况是重叠间隔总和不一致，例如：

 输入：```
2 2
1 1 1
1 2 0
```第一个查询强制$s_1 = +1$，表示位置 1 处的“(”。第二个力$s_1 + s_2 = 0$， 所以$s_2 = -1$，这在本地是一致的。 但如果我们有：```
2 2
1 1 1
1 2 2
```这是不可能的，因为长度为 2 的子字符串在有效的括号字符串中不能有总和 2。 必须发现此类矛盾。 

核心困难在于我们必须满足前缀和的线性约束，并强制前缀和的行为类似于有效的括号路径。 

## 方法

 如果我们忽略有效性约束，问题就简化为赋值问题$a[i] \in \{+1, -1\}$这样对于每个查询$[l, r]$，前缀和满足：$$p[r] - p[l-1] = c$$在哪里$p[i]$是前缀和$a$。 

这是整数线性方程组。 一个蛮力的想法是尝试所有$2^n$括号序列并检查所有查询，但这即使对于小型查询也是完全不可行的$n$。 

更结构化的暴力破解会将其视为回溯问题：为每个位置分配一个括号并增量维护所有查询总和。 每个分配影响最多$O(n)$查询，并且有$2^n$作业，所以这又爆炸了。 

关键的观察是所有约束在前缀和中都是线性的。 我们不是推理单个字符，而是推理前缀值$p[i]$。 每个查询都成为一个差异约束：$$p[r] - p[l-1] = c$$这是一个经典的具有差异或图约束系统的并查找。 我们可以将每个前缀索引解释为一个节点，将每个查询解释为强制执行固定差异的边。 这使我们能够为所有前缀分配一致的值或检测矛盾。 

一旦确定了全局移位的前缀和，我们仍然需要确保该序列对应于有效的括号序列。 自从$a[i] = p[i] - p[i-1]$，每个值都是强制的，因此可行性降低为检查所有值$a[i]$在$\{\pm 1\}$并且前缀和永远不会变为负数并以零结束。 如果由于组件断开连接而存在多个有效分配，我们可以选择强制有效性的根规范化。 

完整的解决方案变为：构建前缀和的约束图，使用具有势或 BFS 的 DSU 一致地分配值，检测矛盾，导出字符，最后验证括号有效性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(2^n \cdot n)$|$O(n)$| 太慢了 |
 | 最优（DSU/图势）|$O(n + q)$|$O(n + q)$| 已接受 |

 ## 算法演练

 我们将前缀和视为未知变量$p[0], p[1], \dots, p[n]$， 和$p[0] = 0$。 

### 步骤1：建立前缀节点之间的约束

 对于每个查询$(l, r, c)$，我们编码：$$p[r] = p[l-1] + c$$这成为节点之间的加权边$l-1$和$r$。 

这将子串约束转化为图一致性问题。 

### 步骤 2：维护前缀索引的加权 DSU

 我们使用并查找结构，其中每个节点存储其父节点以及与父节点的权重差。 重量代表$p[x] - p[parent[x]]$。 合并两个节点时，我们强制执行它们的根之间所需的差异。 

当处理约束时，如果两个节点已经连接，我们检查隐含的差异是否与给定的匹配$c$。 如果不是，则系统不一致。 

此步骤确保所有前缀和对于连接的组件都是唯一定义的。 

### 步骤 3：分配实际前缀值

 处理完所有约束后，每个连接分量都具有相对值，但具有任意绝对偏移。 我们为每个组件固定一个代表并为其分配值 0，然后通过 DSU 关系传播值。 

这会产生一个完整的前缀和数组$p[i]$。 

### 步骤 4：恢复括号字符

 对于每个位置$i$，计算：$$a[i] = p[i] - p[i-1]$$我们验证每个$a[i]$是+1 或-1。 如果出现任何其他值，则系统无效。 

我们将+1 映射到“(”，将-1 映射到“)”。 

### 步骤 5：验证括号序列

 我们在构建的序列上模拟前缀平衡。 我们确保：$$p[i] \ge 0 \quad \forall i \in [1, n], \quad p[n] = 0$$如果违反，我们输出失败。 

### 为什么它有效

 具有差异的 DSU 强制所有区间和约束的精确一致性，因为每个约束都是前缀差异上的线性方程。 任何有效的括号序列都会产生满足所有边的唯一前缀和分配，因此 DSU 永远不会排除可行的解决方案。 相反，DSU 中检测到的任何矛盾都意味着不可能存在线性方程组，因此不存在分配。 

重建后，强制将差异转换回字符，因此有效性降低为检查结果行走是否为 Dyck 路径。 如果不是，则没有一致的分配可以同时满足查询和括号有效性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.diff = [0] * n  # value[x] - value[parent[x]]

    def find(self, x):
        if self.parent[x] == x:
            return x
        px = self.parent[x]
        root = self.find(px)
        self.diff[x] += self.diff[px]
        self.parent[x] = root
        return root

    def get(self, x):
        self.find(x)
        return self.diff[x]

    def union(self, x, y, w):
        rx = self.find(x)
        ry = self.find(y)
        if rx == ry:
            return self.get(x) - self.get(y) == w

        # attach rx under ry
        vx = self.get(x)
        vy = self.get(y)
        self.parent[rx] = ry
        self.diff[rx] = vy + w - vx
        return True

def solve():
    n, q = map(int, input().split())
    dsu = DSU(n + 1)

    for _ in range(q):
        l, r, c = map(int, input().split())
        if not dsu.union(l - 1, r, c):
            print("?")
            return

    val = [0] * (n + 1)

    for i in range(n + 1):
        dsu.find(i)

    for i in range(n + 1):
        val[i] = dsu.diff[i]

    # validate and build brackets
    res = []
    for i in range(1, n + 1):
        d = val[i] - val[i - 1]
        if d == 1:
            res.append('(')
        elif d == -1:
            res.append(')')
        else:
            print("?")
            return

    bal = 0
    for ch in res:
        if ch == '(':
            bal += 1
        else:
            bal -= 1
        if bal < 0:
            print("?")
            return

    if bal != 0:
        print("?")
        return

    print("! " + "".join(res))

if __name__ == "__main__":
    solve()
```DSU 结构以压缩形式维护前缀和差。 并集操作强制执行精确的方程满足，查找操作在累积前缀差异的同时执行路径压缩。 

解决约束后，我们从 DSU 增量中提取绝对前缀值。 关键的微妙之处在于我们只使用 DSU 的相对一致性； 绝对值的重建是一致的，因为每个节点都是相对于其根来表示的。 

最终的转换步骤是确定性的：一旦前缀和固定，每个字符都会被强制。 任何偏离$\pm 1$立即证明不一致。 

最终扫描确保前缀永远不会低于零，这是括号序列有效性所需的唯一结构条件。

 ## 边缘情况

 一种边缘情况是约束未完全连接所有前缀节点。 例如：```
4 1
1 2 0
```节点 0..4 分为组件。 DSU 分配相对值，缺失的组件会得到任意偏移。 该算法仍然有效，因为在形成字符时只有差异才重要。 

另一个边缘情况是单个联合操作内部的直接矛盾：```
3 2
1 1 1
1 1 2
```第一个设置 p1 = 1，第二个要求 p1 = 2。在联合期间，DSU 检测到同一根上的不一致差异并返回失败。 

最终的边缘情况是有效的线性约束，在重建后产生非括号有效的序列。 例如，即使代数一致，约束也可能强制前缀下垂为负。 最终的前缀扫描捕获了这一点：```
p = [0, 1, 0, -1]
```这在第三个位置失败，因为余额变为负数，因此输出被正确拒绝。
