 ## 解决方案
 ## 问题理解

 我们得到一个源字符串$S$和一个目标字符串$F$。 任务是剪掉一段连续的$S$，表示一个子串，这样$F$仍然可以在该段中作为子序列找到。 在所有这些有效段中，我们想要长度尽可能最小的一个。 

关键的限制是$F$保证已经是$S$，因此至少始终存在一个有效子串，即整个字符串$S$。 

字符串很长：$S$可以达到$10^5$每个测试用例的字符数，最多有$10^4$测试用例的组合长度可达$5 \cdot 10^5$。 这立即排除了任何二次方$|S|$。 任何显式尝试所有子字符串或每个候选窗口从头开始重复扫描的方法都将失败。 

一个微妙的边缘情况来自于子序列匹配在索引选择方面是灵活的这一事实。 中的同一个角色$F$可以在不同位置进行匹配$S$，不同的匹配可能会导致不同的窗口长度。 从左侧开始的天真的贪婪前向匹配可能会锁定到一个糟糕的结束位置，从而产生一个非最小子串。 

例如，考虑$S =$ `abac`,$F =$ `ac`。 前向贪婪匹配可能会选择`a(1) -> c(4)`, 给予窗口`[1,4] = abac`，而最佳窗口是`[3,4] = ac`。 这种不匹配来自于尽早发生`a`。 

当一个字符多次出现时，会出现另一种失败情况。 为每个字符选择第一个可能的匹配$F$最大化结束索引，但不保证窗口最小化。 

## 方法

 暴力方法会尝试每个子串$S$，并针对每一项检查是否$F$是一个子序列。 检查单个子串的长度$m$反对长度模式$|F|$成本$O(m)$。 有$O(n^2)$子串，因此总复杂度变为$O(n^3)$，这远远超出了任何限制$n = 10^5$。 

即使我们优化子序列检查$O(|F|)$，我们仍然得到$O(n^2 \cdot |F|)$，这仍然是不可能的。 

关键的观察是我们实际上不需要尝试所有子字符串。 相反，我们可以修复子序列匹配在内部的结束位置$S$，然后询问允许有效子序列在此结束的最佳可能起始位置。 

这导致了两阶段匹配的想法。 首先，我们计算，对于每个位置$F$，最早的索引$S$我们可以将前缀匹配到该字符。 其次，我们从右边开始计算，对于$F$，最新可能的起始索引$S$可以匹配从该位置开始的后缀。 将这些组合起来给出每个对齐位置的候选窗口$F$，我们取最小值。 

我们本质上是预先计算子序列匹配的前向可达性和后向可达性。 自从$|F| \le 100$，我们有能力做$O(|S| \cdot |F|)$预处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | (O(n^2 \cdot | F | )) |
 | 最佳 | (O(n \cdot | F | )) |

 ## 算法演练

 我们将匹配视为跟踪位置$S$对应于每个前缀和后缀$F$。 

### 前传

 1.初始化一个数组`L`尺寸的$|F|$， 在哪里`L[i]`将存储最早的索引$S$我们可以在哪里完成匹配$F[0..i]$。 

我们开始扫描$S$从左到右，同时将指针保持在$F$。 
2. 对于每个字符$S$，如果它与当前字符匹配$F$，我们将该位置指定为下一个匹配并将指针移入$F$向前。 
3. 一旦我们完成了一个角色$F[i]$，我们将当前索引存储在`L[i]`。 

这保证了`L[i]`是 length 前缀最左边可能的结束位置$i+1$，因为我们总是消费$S$贪婪地从左到右。 

### 向后传递

 1、同样构造一个数组`R`在哪里`R[i]`是最新的起始索引$S$我们可以从中匹配$F[i..]$。 
2.我们扫描$S$从右到左，从末尾维护一个指针$F$落后。 
3. 每当字符匹配时，我们都会记录当前位置作为该后缀的有效开始。 

这确保了`R[i]`是后缀最右边可能的开头$F[i..]$。 

### 合并结果

 1. 对于每个分割点$i$在$F$，考虑一个开始于`R[i]`并结束于`L[i]`。 
2. 最佳答案是所有有效答案中的最小值$i$的`L[i] - R[i] + 1`。 
3. 从中提取子字符串$S$。 

### 为什么它有效

 对于任何包含的有效子字符串$F$作为一个子序列，存在一个映射$F$进入$S$。 设位置为$F[i]$被映射定义一个 split: 前缀$F[0..i]$匹配在某个位置结束，并且后缀$F[i..]$从某个位置开始匹配。 前向和后向传递独立地计算这些端点的最佳可能极值选择。 由于任何有效的嵌入都必须遵守前缀端点和后缀起始点，因此最佳窗口必须出现在这些分割点之间。 这将子串上的全局优化问题减少为线性扫描$F$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    S, F = input().split()
    n, m = len(S), len(F)

    # L[i] = earliest end position in S for F[0..i]
    L = [-1] * m
    j = 0
    for i in range(n):
        if j < m and S[i] == F[j]:
            L[j] = i
            j += 1
            if j == m:
                break

    # R[i] = latest start position in S for F[i..]
    R = [-1] * m
    j = m - 1
    for i in range(n - 1, -1, -1):
        if j >= 0 and S[i] == F[j]:
            R[j] = i
            j -= 1
            if j < 0:
                break

    best_len = n + 1
    best_l = best_r = 0

    for i in range(m):
        if L[i] != -1 and R[i] != -1 and R[i] <= L[i]:
            cur_len = L[i] - R[i] + 1
            if cur_len < best_len:
                best_len = cur_len
                best_l, best_r = R[i], L[i]

    print(S[best_l:best_r + 1])

if __name__ == "__main__":
    T = int(input())
    for _ in range(T):
        solve()
```前向扫描通过贪婪地消耗字符来构建前缀匹配的最早完成点。 向后扫描对称地构建后缀匹配的最新可能起点。 

这里的一个常见错误是假设前向贪婪匹配单独决定了答案。 这只会修复一个端点，并忽略移动子序列的开头可以缩小窗口。 向后数组使我们能够探索替代对齐方式，而无需显式枚举它们。 

最后的循环检查所有分割位置$F$，这是安全的，因为$|F| \le 100$，所以即使是一个$O(|F|)$每个测试用例的扫描是微不足道的。

 ## 边缘情况

 一种边缘情况是当$F$由重复字符组成，并且存在多个有效匹配$S$。 该算法通过仅存储极值位置而不是单个贪婪路径来处理此问题。 为了`S = aaaaa`,`F = a`，两次扫描都将唯一的字符标记为最早和最新，从而生成正确的单字符答案。 

另一种情况是最佳窗口需要跳过早期出现的情况以实现更严格的界限。 在`S = abac`,`F = ac`，仅前向扫描就会匹配`a`在索引 0 处和`c`在索引 3 处，但向后扫描允许配对`a`在索引 2 处`c`在索引 3 处，产生最优`ac`。 基于拆分的组合确保对两种可能性进行评估。 

最后一个微妙的情况是给定分割的前向和后向端点不一致。 条件`R[i] <= L[i]`防止后缀在前缀结束后开始的无效窗口。 这保证了只有可行的子序列嵌入才能对答案做出贡献。
