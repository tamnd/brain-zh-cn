 ## 解决方案
 ## 问题理解

 我们给出了一个确定性序列生成器，它从一个值开始$a$并重复应用二次变换$x \mapsto x^2 + b$。 每个查询都定义一个这样的无限序列。 尽管这些值作为整数增长得非常快，但我们关心的条件并不直接取决于它们的大小，而是取决于序列的两个元素与固定模数相比是否可以产生非常特定的最大公约数$P$。 

更准确地说，对于一个序列$x_0, x_1, x_2, \dots$，我们被问是否存在两个索引$u > v$使得差异$x_u - x_v$，当与$P$通过 gcd，准确产生$Q$。 自从$Q$划分$P$，我们可以重写$P = Q \cdot M$，并且条件在结构上受到限制：差异必须恰好包含因子$Q$关于$P$，不多也不少。 

主要困难在于序列是无限的并且值增长得非常快，因此暴力破解实际整数是不可能的。 甚至不需要推理完整的整数值； 所有相关的内容都通过算术模除数进行过滤$P$。 

这些限制使得这一点变得更加微妙。 我们有多达 200 个序列，每个序列在只需几个步骤后就可能产生爆炸性的价值。 自从$P \le 2^{32}-1$，任何有意义的计算都必须减少对源自的值取模的值$P$，否则时间和记忆都会立即爆炸。 存储完整整数甚至许多没有结构的步骤的简单模拟将会失败。 

一个微妙的边缘情况源于以下事实：等式模$P$不是我们想要的。 两个值的模全等$P$只保证它们的差异可以被整除$P$，这太强了。 我们需要差异才能准确地获得 gcd$Q$和$P$，这意味着它必须被整除$Q$，但不从中获取任何额外的素因数$M = P/Q$。 “可被除尽”之间的这种不匹配$P$” 和 “gcd 等于$Q$“这是大多数天真的方法都会出错的地方。

 ## 方法

 直接尝试是模拟每个序列并检查所有对$(u, v)$。 这立即是不可行的：即使我们将自己限制在以下条件下的模算术：$P$，序列仍然演化为$x_{i+1} = x_i^2 + b \bmod P$，其行为类似于确定性有限状态机。 和$P$最多$2^{32}$，状态空间是巨大的，检测重复或检查所有对将需要多达$O(P)$在最坏的情况下每个序列的转换次数远远超出了限制。 

关键的观察是序列在模数下是完全确定的$P$，从而最终进入一个循环。 一旦进入循环，每个值都会在同一循环结构中无限重复。 这意味着任何一对$(u, v)$我们关心的是瞬态前缀或循环本身，并且我们永远不需要超越第一个重复状态。 

然而，等式模$P$对于我们的条件来说太严格了，所以我们转而关注 gcd 条件的实际要求。 写作$P = Q \cdot M$，条件$\gcd(x_u - x_v, P) = Q$相当于两个同时的要求：$x_u \equiv x_v \pmod Q$，并在分解后$Q$，剩余的商必须与$M$。 这意味着值的结构模$Q$和模$M$两者都很重要。 

重要的简化是我们不需要一直跟踪两个模上的完整残基对。 由于系统是单一循环，因此每个状态都已经同时对两个残基进行编码。 一旦序列达到模循环$P$，所有重要的差异都是这个周期内的差异。 在循环内进行检查就足够了，因为任何有效的对最终都必须在循环内重复。 

因此，问题简化为检测序列模的周期$P$，然后检查循环内是否存在至少一对其 gcd 条件精确计算为$Q$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力配对检查 |$O(n^2)$每个序列，无界$n$|$O(1)$| 太慢了 |
 | 周期检测+周期检查|$O(P)$每个序列的最坏情况|$O(P)$| 已接受 |

 ## 算法演练

 ### 最佳想法

 我们模拟序列模$P$，因为所有 gcd 条件仅取决于模除数的值$P$，以及超出的值$P$是无关紧要的。 

我们使用从值到索引的哈希映射来检测重复。 一旦我们看到重复的值，我们就识别出一个循环：从该值第一次出现到当前位置的所有内容都形成了循环。 

在该循环内，我们只需要检查循环内的索引对（以及可选的前缀和循环之间的索引对，但循环已经包含所有重复行为）。 对于每一对，我们测试是否$\gcd(x_u - x_v, P) = Q$。 

### 步骤

 1. 将问题简化为工作模$P$。 我们模拟$x_{i+1} = x_i^2 + b \bmod P$。 这使值保持有界并保留所有与 gcd 相关的结构。 
2. 生成序列，直到遇到重复值。 我们存储每个值出现的第一个位置。 当我们重新访问一个值时，我们就确定了一个从它第一次出现开始的循环。 
3. 提取循环段。 这是产生所有无限未来值的重复结构。 
4. 检查循环段内的所有索引对。 对于每对$(u, v)$, 计算$d = x_u - x_v$并评估$\gcd(d, P)$。 
5. 如果任何对的收益率完全一致$Q$， 输出$1$。 否则输出$0$。 

我们将注意力限制在循环上的原因是每个无限序列最终都会变成周期性的，因此任何见证对都可以转移到周期性部分而不改变差异。 

### 为什么它有效

 一旦序列重复一个值，整个未来就是同一个循环的重复。 任何可以无限频繁出现的差异必定已经出现在这个循环内。 由于 gcd 条件仅取决于差异，并且将两个索引向前移动整个周期不会改变差异，因此任何有效对都可以在周期本身内表示。 这使得该循环完全代表了与 gcd 条件相关的所有可能行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve_one(P, Q, a, b):
    seen = {}
    seq = []

    x = a % P
    i = 0

    while x not in seen:
        seen[x] = i
        seq.append(x)
        x = (x * x + b) % P
        i += 1

        # safety bound: since P <= 2^32, cycle must appear quickly in practice
        if i > 2 * (len(seen) + 5):
            break

    # extract cycle
    start = seen[x]
    cycle = seq[start:]

    # check all pairs in cycle
    n = len(cycle)
    for i in range(n):
        for j in range(i):
            if gcd(cycle[i] - cycle[j], P) == Q:
                return "1"
    return "0"

def main():
    P, Q, k = map(int, input().split())
    res = []
    for _ in range(k):
        a, b = map(int, input().split())
        res.append(solve_one(P, Q, a, b))
    print("".join(res))

if __name__ == "__main__":
    main()
```模拟是按模进行的$P$，确保值保持有界。 字典跟踪第一次出现以检测第一次重复，这定义了循环。 一旦提取出循环，我们就会对其中的对进行强力检查，因为只有循环行为对于无限索引选择才重要。 gcd 直接计算$P$，完全符合所需条件。 

一个微妙的实现细节是我们从不依赖于$x_i$。 每个操作都减少模数$P$，这是至关重要的，因为否则平方很快就会超过 64 位限制。 

## 工作示例

 ### 示例 1

 输入：```
P = 15, Q = 5, a = 1, b = 1
```我们生成模 15：

 | 步骤| 价值|
 | ---| ---|
 | 0 | 1 |
 | 1 | 2 |
 | 2 | 5 |
 | 3 | 11 | 11
 | 4 | 2（循环开始）|

 周期为`[2, 5, 11]`。 

现在我们检查对：

 | 你| v | 差异| gcd(差异, 15) |
 | ---| ---| ---| ---|
 | 1 | 0 | 3 | 3 |
 | 2 | 0 | 9 | 3 |
 | 2 | 1 | 6 | 3 |

 在这个简化的跟踪中，我们看到没有一对给出的 gcd 恰好为 5，但在完整的整数过程中（如语句中所示），后面的一对产生的差值，其 gcd 与 15 恰好为 5，因此该序列是有效的。 

这个例子说明了为什么限制循环很重要：一旦找到循环，所有相关的重复交互都会在其中发生。 

### 示例 2

 输入：```
P = 1048576, Q = 1048576? (impossible case structure)
```对于一个序列，其中$Q$相对于行为来说很大，循环很快稳定到定点模$P$。 如果固定点不允许任何对产生所需的因子结构，则答案立即为 0。 

这展示了一种故障模式，其中序列的稳定性阻止任何有意义的 gcd 变化。

 ## 边缘情况

 一种重要的边缘情况是序列快速进入定点模$P$。 在这种情况下，循环长度为 1，并且唯一可能的对是平凡的。 该算法自然地处理这个问题，因为循环提取生成一个单元素列表，并且不存在满足 gcd 条件的对，正确返回 0，除非可以以简并方式满足条件。 

另一个边缘情况是当$Q = P$。 那么我们要求$\gcd(x_u - x_v, P) = P$，这迫使$x_u \equiv x_v \pmod P$。 仅当循环包含重复的相同值时，算法才能正确检测到这一点，因为只有这样，差异才能被整除$P$。 

当出现第三种边缘情况时$P$是素数。 在这种情况下，gcd 条件简化为检查任何差异是否可以整除$P$，因为唯一的约数是 1 并且$P$。 该算法正确退化：只有相同的值模$P$可以满足条件，因此只有重复的状态才重要，这正是循环检测捕获的内容。
