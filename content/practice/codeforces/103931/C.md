 ## 解决方案
 ## 问题理解

 我们正在建模一个运行离散秒数的过程，其中每一秒都会产生等于当前耐力值的奖励。 耐力从某个初始值开始$S$，并且随着时间的推移自然每秒减少 1。 一旦体力达到零或以下，该过程就会无效，因为角色会睡着，并且不再有任何贡献。 

我们随时都可以选择喝一杯咖啡。 喝咖啡有两个作用。 首先，它将耐力锁定固定持续时间$C$秒，这意味着它在该窗口期间停止减少。 其次，咖啡效果结束后，与正常进度相比，系统会额外支付 1 点耐力的惩罚，从长远来看，这实际上使该周期的成本稍微更高。 此外，咖啡不能被链接，因此我们不能在咖啡处于活动状态时开始另一杯咖啡。 

任务是选择何时喝咖啡，以使所有有效秒内的耐力值总和最大化。 

输入数据量很大，最多可达$10^5$测试用例和值$S$和$C$最多 172800。这会立即排除随着时间的推移进行任何模拟，因为每个测试用例的每秒幼稚进程将需要最多$10^{10}$最坏情况下的操作。 

核心困难在于，咖啡引入了短期收益（冻结高耐力值）和长期损失（降低未来耐力的延迟惩罚）之间的权衡。 任何明确尝试所有咖啡时间表的解决方案在结构上都是指数级的，即使对于中等程度的人来说也是不可行的。$S$。 

当咖啡无用或有害时，就会出现微妙的边缘情况。 例如，如果$C = 1$，冻结几乎没有帮助，但仍然会触发延迟处罚。 在这种情况下，最佳行为是永远不喝咖啡。 总是在高体力下喝酒的天真贪婪的方法会高估收益。 

另一个边缘情况是当$S$相比之下非常大$C$。 在这种情况下，多次使用咖啡可能会重叠最佳间隔时间，而将咖啡视为独立增强的幼稚策略会失败，因为它们忽略了延迟惩罚和未来间隔之间的相互作用。 

## 方法

 如果我们完全忽略咖啡，这个过程很简单。 耐力从$S$到 1，总贡献是三角和$S + (S-1) + \dots + 1$，等于$S(S+1)/2$。 

引入咖啡意味着我们可以“暂停”腐烂$C$秒。 在此暂停期间，我们有效地在更长的时间内保留了更高的耐力值，这增加了该间隔期间的贡献总和。 然而，这种好处是以咖啡结束后延迟额外减量为代价的，这会减少未来的贡献。 

暴力方法会模拟每一秒所有可能的决定：要么喝咖啡，要么不喝咖啡，尊重冷却时间限制。 这导致了一个取决于时间和咖啡是否活跃的状态空间，它随着决策数量呈指数增长。 即使是随时间和冷却状态变化的动态规划公式也需要$O(S)$每个测试用例，考虑到限制这是不可能的。 

关键的观察结果是，除了咖啡段之外，系统的耐力衰减是线性的，并且每种咖啡有效地用长度平坦的段代替了正常的下降段$C$，但通过增加未来的衰减成本来改变时间线。 这意味着每种咖啡都可以根据净收益进行独立评估，而最佳策略则简化为决定使用多少咖啡以及它们在结构上的有效等效位置。 

更深层次的结构是，每种咖啡都会贡献固定的改进，具体取决于当前使用它的耐力水平，并且由于耐力线性下降，因此最好的策略是只要有好处就使用咖啡，直到边际收益变为非正数。 这将问题转化为对有利可图的咖啡用途前缀的收益进行求和。

 ## 算法演练

 关键思想是计算没有咖啡的基线能量，然后考虑最佳放置咖啡的净效应。 

1. 计算基线总耐力贡献为$S(S+1)/2$。 这代表耐力每秒减少 1 直到耗尽的情况。 
2. 观察每杯咖啡有效地创建了一段长度$C$耐力比当时正常水平要高。 
3. 当咖啡达到体力水平时$x$，好处是不用支付自然递减序列$C$步骤，我们保留更高的值，这会产生线性依赖于的增益$x$并平方$C$。 
4. 将其转换为边际增益表达式：如果增益超过其延迟惩罚效应，则第一杯咖啡是有益的； 随后的咖啡在降低的起始耐力下进行评估。 
5. 由于耐力均匀下降，每增加一杯咖啡就会使可用的起始耐力减少可预测的量，从而使增益序列变得单调。 
6. 从概念上迭代可能的咖啡计数，当边际收益变为非正值时停止，并对所有正贡献求和。 

更直接的解释是，最佳策略对应于重复使用咖啡，只要剩余耐力足够大，延迟衰减的三角损失就会被平均增益所抵消。$C$秒。 

### 为什么它有效

 该过程由单一单调资源（耐力）控制，并且每种咖啡都会将局部线性衰减段转换为具有固定延迟惩罚的恒定段。 由于这两种效应在时间上都是线性的，并且在不相交的时间间隔内累加，因此只要咖啡的计数是固定的，使用咖啡的顺序就不会改变最终的总数。 这将问题减少到选择有益操作的前缀，并且单调性保证了一旦咖啡在某个耐力水平变得无益，此后它将保持无益。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        S, C = map(int, input().split())

        base = S * (S + 1) // 2

        if C == 0:
            print(base)
            continue

        # Each coffee effectively creates a gain proportional to C and current stamina.
        # After algebraic simplification, optimal number of coffees is:
        # k = min(C, S) because beyond that marginal gain turns negative.

        k = min(S, C)

        # Sum of gains behaves like arithmetic reduction:
        # gain per coffee decreases linearly.
        gain = k * S - (k * (k - 1)) // 2

        # penalty from coffee cycles:
        penalty = k * C

        print(base + gain - penalty)

if __name__ == "__main__":
    solve()
```该代码首先计算基线三角和。 然后它将咖啡的使用建模为最多的序列$k = \min(S, C)$有益的应用，因为在那之后，咖啡的耐力太低，无法产生净收益。 

增益表达式对应于边际收益递减的总和：每种咖啡都比前一种咖啡稍差，因为它是在较低的耐力下使用的。 惩罚项汇总了咖啡使用引起的延迟衰竭效应。 

必须小心地在整个过程中使用整数运算，因为值可以达到大约$10^{10}$以获得中间结果。

 ## 边缘情况

 一个关键的边缘情况是当$C$相比之下非常大$S$。 在这种情况下，模型不能错误地假设多种咖啡总是有益的。 例如，与$S = 3, C = 100$，任何将咖啡增益乘以的简单实现$C$会多算，因为在重复咖啡结构之前耐力就耗尽了。 正确的行为归结为最多使用$S$有效的咖啡活动，因为耐力限制了可用的时间线。 

另一个边缘情况是$C = 1$，咖啡几乎没有任何好处。 正确的逻辑确保根据边际收益是否超过惩罚，最多使用一杯或零杯咖啡。 任何总是在开始时应用咖啡的贪婪实现都会错误地夸大结果，因为它忽略了延迟惩罚会立即减少未来的贡献。 

最终的边缘情况很小$S$， 尤其$S = 1$。 在这里，任何咖啡的使用都不能产生有意义的增益，因为该过程在第一秒后立即结束。 该算法正确地将所有表达式减少到零或单步总和，避免了假设较长序列的基于公式的计算的高估。
