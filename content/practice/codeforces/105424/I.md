 ## 解决方案
 ## 问题理解

 我们有一个尺寸为的一维板$N$，其中令牌从单元格 1 开始并想要到达单元格$N$。 在这些端点之间，一些单元格包含“蛇”，每当令牌落在那里时，它们就会将令牌向左推一个固定的量。 

移动不仅仅是单一的确定性转变。 相反，每一轮都以从 1 到 6 的骰子掷骰开始。令牌向右移动该数量，但被夹紧，因此它永远不会越过单元格$N$。 落地后，如果单元格中有蛇，则令牌会根据单元格的值反复向左拉，直到到达没有蛇的单元格。 这种连锁反应可以在同一着陆位置内多次发生，但只能在同一回合内发生。 

特殊之处在于，掷出 6 并不一定会结束回合。 如果出现 6，玩家可以立即继续再次滚动并应用相同的机制，所有这些都在一个回合内完成。 任何其他滚动在处理所得的运动链后立即结束转弯。 

目标是确定到达电池所需的最小匝数$N$，假设最佳玩法，甚至允许在无用的情况下忽略“不幸”的骰子序列。 如果没有办法永远到达$N$，答案是$-1$。 

电路板尺寸可达$2 \cdot 10^5$，因此任何明确模拟骰子结果或探索每回合和每掷状态的解决方案都会太慢。 解决方案必须将骰子的随机性压缩为状态的确定性优化。 

一个微妙的困难是，落在蛇上可能会发生多次级联。 另一个是滚动 6 允许在单回合内进行链接移动，从而有效地合并多个转换，而无需增加回合计数器。 

一个幼稚的错误是将每个单元视为一个带有边缘的简单节点$i+1 \dots i+6$。 这忽略了两个钳位$N$以及强制向左蛇传播，这会极大地改变可到达的位置。 

另一种失败模式是将每个骰子掷骰视为单独的 BFS 边缘成本 1。这错误地计算了回合而不是掷骰序列，并且它还忽略了特殊的“继续 6”规则，这使得某些序列可以自由扩展当前回合。 

## 方法

 直接模拟视点将每个状态视为由位置和回合结构组成的对。 从一个位置开始，我们将尝试所有可能的骰子结果，模拟正确的移动，然后重复应用蛇形过渡直到稳定。 然后我们将根据骰子是否为 6 来决定回合是否结束或继续。 

该模型提出了一个图，其节点是棋盘位置，但边缘因回合分组而变得复杂。 如果我们显式地扩展所有内容，则每个状态转换都对应于一次掷骰子，并且我们需要区分我们是否仍在同一回合内或已经开始新的回合。 这使得状态空间加倍并创建了$6N$每层的过渡，导致$O(6N)$每转层 BFS 仍然太大，因为转数是无界的，并且每个状态转换的成本不统一。 

关键的观察结果是，除了在停止之前是否至少发生一个非 6 之外，转弯内滚动的确切顺序是无关紧要的。 回合的行为类似于最多连续几次移动的爆发，其中只有第一个非 6 掷才会结束爆发。 由于我们想要最小的回合数，因此我们只关心在固定的突发次数内是否可以达到某个状态，而不是确切的骰子历史记录。 

这使我们能够将问题重新解释为位置上的最短路径，其中每个位置转换对应于完整的转弯结果。 在一个回合内，我们可以模拟任意多个 6 的效果，然后是从 1 到 5 的最后一次掷骰（或者根据解释，在最终确定后结束回合的 6），但关键的简化是，在一个回合内，我们实际上可以执行由骰子 6 链接的多个向右移动，每个中间位置立即解决蛇。 

因此，每一回合相当于选择一系列移动，通过一系列确定性的“解决然后再次滚动”步骤来推进令牌，其中影响回合成本的唯一分支是我们是继续还是终止。 这种结构自然由 BFS 处理，其中每个节点都是一个单元，边代表一圈内可到达的所有可能的最终位置。 

为了有效地计算这些边，我们使用 0-1 样式扩展在一个回合内进行模拟：通过 6 移动不会花费新的回合，而非 6 则结束该回合。 我们可以将其建模为每匝层的多源 BFS，或者更清晰地建模为最短路径，其中对应于 6 的边的成本为 0，对应于 1-5 的边的成本为 1（在次要意义上），但最终我们通过扩展匝边界内可到达的状态，将所有内容折叠为“最小匝”BFS。 

主要技术瓶颈是有效解决蛇链。 由于每个单元格都指向左侧，重复应用形成一个仅具有向后边缘的函数图。 我们可以在使用路径压缩式记忆或迭代折叠完全解决蛇之后预先计算最终着陆位置，确保每个单元在接近 O(1) 的时间内解决。 

一旦每个动作都减少到确定性的“蛇解决后”位置，转换就会变得干净：从一个位置，掷骰子 k 将我们发送到$resolve(\min(i+k, N))$。 然后，我们将滚动 6 视为允许在同一回合内继续探索，这意味着通过 k=6 转换序列可到达的所有节点在零成本边缘下形成连接组件，并且 1..5 中的任何 k 都会在移动后触发回合增量。 

这将问题简化为 BFS，其中我们首先扩展零成本边（6 步链），然后在 1-5 步的 +1 成本边中分层。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 指数| O(N) | 太慢了|
 | 0-1 BFS 状态与蛇压缩 | O(N) | O(N) | 已接受 |

 ## 算法演练

 1. 预先计算一个函数`go[i]`这给出了在 i 上着陆并重复应用蛇跳后的最终位置。 这确保了每一步都可以被视为单个确定性转换。 原因是蛇链永远不会分支，因此重复应用总是会收敛到一个固定点。 
2. 构建从每个单元格 i 到最多六个目的地的转换：`go(min(i+k, N))`k 从 1 到 6。这些代表掷骰子并立即解决所有强制运动的效果。 
3. 将第 6 卷转变解释为零成本边，因为它们不会结束回合，而第 1 卷到第 5 卷对应于结束回合的边，因此将回合计数增加 1。这种分离捕获了只有非 6 次回合才能终止回合的规则。 
4. 从距离为 0 的单元格 1 开始运行 0-1 BFS。使用双端队列，其中零成本转换 (k=6) 被推到前面，成本 1 转换 (k=1..5) 被推到后面。 这确保了状态按照轮次递增的顺序进行处理。 
5. 当到达单元格 N 时停止。 如果无法访问，则返回-1。 

正确性取决于以下事实：任何掷骰子序列都可以分解为以第一个非 6 结尾的片段，并且在每个片段内，所有中间过渡都是零成本扩展。 BFS 确保我们始终首先处理最少数量的此类段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    N = int(input())
    p = [0] * (N + 1)
    arr = list(map(int, input().split()))
    for i in range(2, N):
        p[i] = arr[i - 2]

    # resolve snake chains
    resolved = [-1] * (N + 1)

    sys.setrecursionlimit(10**7)

    def get(x):
        if resolved[x] != -1:
            return resolved[x]
        if x == 1 or x == N:
            resolved[x] = x
            return x
        if p[x] == 0:
            resolved[x] = x
        else:
            resolved[x] = get(x - p[x])
        return resolved[x]

    for i in range(1, N + 1):
        get(i)

    dist = [-1] * (N + 1)
    dist[1] = 0
    dq = deque([1])

    while dq:
        v = dq.popleft()
        if v == N:
            print(dist[v])
            return

        for k in range(1, 7):
            nv = v + k
            if nv > N:
                nv = N
            nv = resolved[nv]

            nd = dist[v] + (0 if k == 6 else 1)

            if dist[nv] == -1 or nd < dist[nv]:
                dist[nv] = nd
                if k == 6:
                    dq.appendleft(nv)
                else:
                    dq.append(nv)

    print(-1)

if __name__ == "__main__":
    solve()
```预处理步骤计算所有蛇效应后每个单元的最终着陆位置。 这可以防止 BFS 期间重复重新计算，并确保每次转换都是 O(1)。 

BFS使用双端队列来实现0-1 BFS逻辑。 k=6 的移动被视为零成本边并被推到前面，而所有其他移动都会增加回合计数并被推到后面。 

一个微妙的点是夹紧`v + k`到`N`，符合运动不能超过最终单元格的规则。 另一个重要的细节是在夹紧之后解决，而不是之前，因为蛇适用于最终的着陆位置。

 ## 边缘情况

 一个关键的边缘情况是形成深向后指针结构的长蛇链。 在这种情况下，简单的重复模拟将重复地重新计算中间状态并退化为二次行为。 已记下的`get(x)`确保每个单元都被解析一次，将链折叠成单个端点，无论深度如何。 

另一个边缘情况是当令牌重复落在$N$由于过冲夹紧。 由于夹紧发生在蛇解析之前，因此靠近末端的任何大滚动仍然必须正确解析为$N$即使中间细胞含有蛇。 BFS 在调用之前显式钳位`resolved`，保证正确性。 

最后一个微妙的情况是蛇将令牌发送回自己或形成一个循环。 因为`p[i] < i`，所有边都严格向左走，因此循环不可能存在。 这保证了解析函数的终止，并证明无需跟踪访问状态即可进行记忆。
