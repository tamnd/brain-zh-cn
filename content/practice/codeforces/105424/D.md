 ## 解决方案
 ## 问题理解

 我们有一个由以下材料组成的结构$N$独立的部分。 每个零件都有规定的标准重量$s_i$，当前配置有权重$a_i$。 配置的成本不是总的绝对差异，而是标准化偏差：每个部分都贡献$\frac{|a_i - s_i|}{s_i}$，总偏差是这些贡献的总和。 

我们可以执行单位操作，其中每个操作增加或减少一个$a_i$1.每次操作花费1分钟。 目标是在转换数组时花费尽可能少的操作$a$进入一些最终数组$a'$使得总偏差为$a'$至多是$K$。 

关键的困难在于我们不需要使所有零件完全等于其目标。 我们只需要将总体加权偏差降低到阈值以下，并且我们希望通过最小的总单位调整来实现这一点。 

约束条件$N \le 10^5$迫使任何解决方案接近线性或$N \log N$。 由于每个操作都可能影响不同的组件，因此不可能对每个单元进行模拟。 

一个微妙的点是$s_i \le 10$，这使得偏差公式中的权重成为小整数。 这个有界分母成为解决方案的结构杠杆。 

一个天真的错误是假设我们应该总是减少每一个$a_i$朝向$s_i$。 当某些部件在偏差方面改进“昂贵”时，这不一定是最佳的。 另一个陷阱是试图贪婪地减少最大的绝对差异，而不考虑标准化$s_i$，它以非统一的方式改变优先级。 

作为具体的故障场景，假设$s = [1, 10]$,$a = [100, 11]$， 和$K = 0$。 一个幼稚的策略可能会均匀地减少两个值以达到其目标，但就每个操作的偏差减少而言，减少第一部分的效率远低于首先修复第二部分，因为归一化差异很大。 正确的解决方案必须评估每次操作的边际效益，而不是原始差异。 

## 方法

 暴力解释是逐步模拟该过程。 每分钟，我们选择一个索引，并将其向最能减少总偏差的方向调整 1。 每次更新后，我们都会重新计算完整偏差。 

这在概念上是有效的，因为每个状态转换都是有效的，并且我们始终跟踪真正的目标。 然而，每次重新计算偏差成本$O(N)$，我们可能会执行到$\sum |a_i - s_i|$运营。 自从$a_i$可以达到$10^9$，最坏情况下的总操作量可以达到$10^{14}$，使得这种方法完全不可行。 

关键的观察是索引上的每个单元操作$i$以非常结构化的方式改变该指数的偏差。 如果$a_i > s_i$，减少$a_i$准确减少偏差$\frac{1}{s_i}$每单位步长，直到达到$s_i$。 如果$a_i < s_i$，增加它可以减少每步相同量的偏差。 这意味着每个指数都贡献具有固定斜率大小的线性“偏差预算”$1/s_i$，但仅限于达到目标。 

因此，每次操作相当于花费 1 个单位时间，根据所选指标将总偏差减少一个已知常数。 因此，最佳策略是始终将操作花费在单位成本降低幅度最大的索引上，这恰好是最小的$s_i$在可用指数中，因为$\frac{1}{s_i}$较大时，$s_i$较小。 

然而，我们不能自由地从单个索引中选择无限多个归约：每个索引只允许$|a_i - s_i|$可用的操作。 这将问题转化为多个“归约单位”，每个归约单位都有值$1/s_i$，我们要选择最小数量的单元，直到总偏差降到最大$K$。 

自从$s_i \le 10$，我们只有 10 种可能的效率类型，这允许按$s_i$并贪婪地处理。

 ## 算法演练

 我们重新构建偏差，以便我们可以推断可以将其减少多少。 

1. 计算初始总偏差$D = \sum \frac{|a_i - s_i|}{s_i}$。 这是目前与可行性的“距离”。 我们希望将其减少到最多$K$，因此所需的减少量为$R = D - K$。 如果$R \le 0$，无需任何操作。 
2. 对于每个索引$i$，计算其绝对失配$d_i = |a_i - s_i|$。 这是可以提高该指数的最大单元操作数。 
3.观察指数每一个单位的调整$i$准确减少总偏差$\frac{1}{s_i}$，只要我们还没有达到$s_i$。 这意味着每个指数都贡献相同“福利项目”的线性序列。 
4. 不要逐个迭代，而是按索引对所有索引进行分组$s_i$。 自从$s_i \le 10$，维护一个频率数组，其中每个组$g[x]$存储总可用操作和每个操作的总偏差减少量为$1/x$。 
5. 我们现在决定从每组中进行多少次操作。 规模较小的团体$s_i$每次操作都会带来更大的收益，因此我们按升序处理组$s_i$。 
6.针对固定群体$s$，我们考虑需要从中进行多少次操作。 如果采取该组中的所有可用操作将剩余所需的减少量减少到零或更低，我们只采取部分金额并完成。 
7. 累积运算，直到所需的减少变为非正数。 累计计数就是答案。 

核心思想是每个操作仅根据其组贡献固定增益，因此我们正在解决每个类别具有统一项目值的贪婪背包。 

为什么它有效：

 在任何时刻，对具有较小索引的操作$s_i$总是提供至少与更大的操作一样多的偏差减少$s_i$，并且这个顺序在这个过程中永远不会改变，因为每一步的减少$\frac{1}{s_i}$是恒定的。 因此，任何最优的操作顺序都可以重新排列，使得所有更高收益的操作优先，而不影响可行性，这保证了贪婪排序$s_i$是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    s = list(map(int, input().split()))
    a = list(map(int, input().split()))

    groups = [[] for _ in range(11)]

    total_deviation = 0.0

    for i in range(n):
        diff = abs(a[i] - s[i])
        if diff == 0:
            continue
        total_deviation += diff / s[i]
        groups[s[i]].append(diff)

    need = total_deviation - k
    if need <= 0:
        print(0)
        return

    ops = 0

    for val in range(1, 11):
        if need <= 0:
            break
        if not groups[val]:
            continue

        total_ops = sum(groups[val])

        reduction_per_op = 1.0 / val
        max_reduction = total_ops * reduction_per_op

        if max_reduction >= need:
            ops_needed = int(need * val + 1e-12)
            ops += ops_needed
            need = 0
        else:
            ops += total_ops
            need -= max_reduction

    print(ops)

if __name__ == "__main__":
    solve()
```该解决方案首先从直接计算定义的初始偏差开始。 每个指标独立贡献，所以我们总结$|a_i - s_i| / s_i$。 

然后我们计算必须消除多少偏差才能达到阈值$K$。 如果这已经满足，则答案为零。 

分组依据$s_i$至关重要，因为它将问题转化为 10 个独立的桶。 桶中的每个单元操作都会减少固定量的偏差$1/s_i$，所以我们只需要每个桶的可用操作总数。 

贪心循环从最小的开始处理桶$s_i$到最大。 对于每个桶，我们要么完全消耗它，要么部分消耗它，具体取决于它是否满足剩余要求。 乘法$need * val$将所需的偏差减少量转换回该存储桶的操作数量。 

一个微妙的实现问题是在偏差和操作计数之间进行比较和转换时的浮点精度。 该代码通过在转换为整数时使用较小的 epsilon 来稳定这一点。

 ## 边缘情况

 一种边缘情况是初始偏差已在允许的阈值内。 例如，$N=3$,$s=[1,2,1]$,$a=[1,2,1]$,$K=0$。 计算出的偏差为零，因此算法立即返回零，而不进入任何贪婪循环。 

另一个边缘情况是当只有最小的$s_i$群体足以跨过门槛。 在这种情况下，算法会停止中间组，并使用乘法将剩余的所需归约转换为部分运算数$s_i$。 正确性依赖于这样一个事实：在一个组内，每个操作都具有相同的值，因此部分选择始终是最优的且定义明确的。 

第三种边缘情况是需要所有组时。 例如，当大的偏差存在时$s_i$，算法耗尽所有桶。 由于每个组都已完全消耗，因此答案只是所有组的总和$|a_i - s_i|$，这与必须使用所有可能的单位调整这一事实相匹配。
