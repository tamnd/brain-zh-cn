 ## 解决方案
 ## 问题理解

 IPv4 地址可以看作是一个 32 位整数，通常写为用点分隔的四个十进制数。 CIDR 块如`192.168.88.0/24`描述一组地址：前24位固定为网络前缀，其余8位可以自由变化。 每个查询都会询问给定的 IPv4 地址是否属于此固定前缀范围。 

因此，该任务简化为由二进制前缀约束定义的集合中的成员资格测试。 我们会得到一个这样的前缀描述和最多 1000 个候选 IP 地址。 对于每个候选者，我们必须确定其前 k 位是否与前缀匹配，其中 k 在斜杠后面给出。 

约束足够小，即使每个查询进行直接按位或基于整数的检查也足够了。 将每个 IP 转换为 32 位整数并比较屏蔽前缀是恒定时间的工作，因此即使对所有查询进行线性扫描在性能方面也是微不足道的。 

棘手的部分不是计算性的，而是表征性的。 一个常见的陷阱是单独处理每个八位位组，而没有正确重建完整的 32 位结构。 另一个是对 CIDR 掩码如何应用的误解：比较不是针对 IP 的完全相等，而是仅在掩码后的前缀部分上相等。 

当前缀长度为 0 或 32 时，会出现微妙的边缘情况。`/0`网络包含每个IP地址，而一个`/32`网络只包含一个地址。 除非通过位掩码显式处理，否则简单的字符串比较在这里会失败。 

另一个问题是由前导零或点分小数解析不一致引起的。 例如，`010.0.0.1`仍应被解释为 10.0.0.1，但基于字符串的简单比较会以不同方式对待它们。 

## 方法

 暴力心理模型是将 CIDR 块解释为生成其范围内的所有可能的 IP，然后对照此生成的集合检查每个查询 IP。 如果前缀是`/k`，我们会枚举所有`2^(32-k)`可能的后缀，构造完整的 IP，并将它们存储在哈希集中。 每个查询都成为一次成员资格测试。 

这是正确的，因为它显式地构造了子网的定义。 然而，由于效率原因，它立即失败。 即使在像这样的温和情况下`/24`，有超过 1600 万个可能的地址，并且对于`/16`超过40亿。 构建或存储它在时间和内存上都是不可行的。 

关键的观察是子网成员资格从根本上来说是一个按位条件。 我们可以将子网和每个 IP 表示为 32 位整数，并应用仅保留前缀位的掩码，而不是枚举所有后缀。 当且仅当两个地址的掩码值匹配时才属于同一子网。 这完全删除了枚举，并将每个查询减少为恒定时间的位操作。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（枚举）| O(2^(32-k) + n) | O(2^(32-k) + n) | O(2^(32-k)) | O(2^(32-k)) | 太慢了 |
 | 最佳（位屏蔽）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 将 CIDR 输入解析为两部分，基础 IP 和前缀长度 k。 基本IP定义了网络锚点。 
2. 将基础IP从点分十进制转换为32位整数。 每个八位位组贡献 8 位，移位到其​​位置。 
3. 构建一个 32 位掩码，其中前 k 位为 1，其余 32-k 位为 0。这隔离了网络前缀。 
4. 将掩码应用于基础 IP，以获得规范的网络代表。 这消除了输入中可能存在的主机位中的任何歧义。 
5. 对于每个查询IP，使用相同的解析方法将其转换为32位整数。 
6. 对查询IP 应用相同的掩码。 
7. 将屏蔽查询值与屏蔽基值进行比较。 如果匹配，则该 IP 属于该子网； 否则它不会。 
8. 相应地输出YES 或NO。 

### 为什么它有效

 屏蔽仅在前缀位上强制等效，这正是 CIDR 定义的。 同一子网中的任何两个 IP 共享相同的前 k 位，并且掩码消除了剩余位中的所有可变性。 由于子网基址和查询 IP 的掩码相同，因此掩码后的相等性相当于属于同一前缀定义的等价类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ip_to_int(s: str) -> int:
    a, b, c, d = map(int, s.strip().split('.'))
    return (a << 24) | (b << 16) | (c << 8) | d

def build_mask(k: int) -> int:
    if k == 0:
        return 0
    return ((1 << k) - 1) << (32 - k)

def solve():
    line = input().strip()
    ip_part, k_part = line.split('/')
    k = int(k_part)

    base = ip_to_int(ip_part)
    mask = build_mask(k)

    base_net = base & mask

    n = int(input())
    out = []

    for _ in range(n):
        ip = ip_to_int(input().strip())
        if (ip & mask) == base_net:
            out.append("YES")
        else:
            out.append("NO")

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现的核心是将点分十进制转换为32位整数。 每个八位位组都被移动到正确的位置，完全按照内存中的显示方式重建二进制表示形式。 

掩模结构值得关注。 当k为0时，子网包含所有IP，因此掩码为零，掩码后每个IP都映射为零。 当 k 为 32 时，掩码变为全 1，强制完全相等。 

按位 AND 后，其他所有内容都简化为整数比较，这是 CIDR 语义的直接计算翻译。

 ## 边缘情况

 一个`/0`子网是最宽松的情况。 掩码变为零，因此与其进行“与”操作的每个 IP 都变为零，与基本网络值匹配。 该算法自然会为所有查询返回“是”，而无需掩码构造之外的特殊外壳。 

一个`/32`子网是最严格的情况。 掩码变为全 1，因此不会忽略任何位。 该比较简化为查询 IP 和基本 IP 之间的完全整数相等，这与定义完全匹配。 

像这样的例子`0.0.0.0/0`任何查询，例如`8.8.8.8`产生相同的掩码值 0，因此它正确返回 YES。 反过来，`192.168.1.1/32`只匹配自身，因为任何位的任何差异都会改变屏蔽值。
