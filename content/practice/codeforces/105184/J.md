 ## 解决方案
 ## 问题理解

 我们每天都会得到一组从 0 到 9 的十进制数字。提供了每个数字的计数，并且可用数字的总数可能非常大。 从这些数字中，我们必须准确地选出$m$数字，然后将它们排列成一个$m$- 没有前导零的数字。 目标是使所得数字的值尽可能小。 

一旦形成了尽可能小的数字，我们就不会被要求直接输出它。 相反，我们必须计算其值模$10^9 + 7$。 重要的微妙之处在于，我们并没有最小化取模后的余数；而是将取模后的余数最小化。 我们必须首先最小化实际整数，然后才应用模运算。 

这些约束意味着任何将数字显式构造为长度可达的字符串的解决方案$10^9$是不可能的。 即使对所有数字进行线性时间模拟也会失败，因为每次测试的数字总数可能很大，并且最多有$10^4$测试用例。 这迫使解决方案的工作时间与数字类型的数量成正比，而不是与数字的数量成正比。 

处理前导零时会出现一个幼稚的错误。 如果我们简单地对所有选定的数字进行排序并将它们连接起来，我们可能会先放置零，这是无效的。 另一个常见的错误是假设局部最小化数字也会最小化模后的数值，但由于模数仅在最后应用，因此我们必须保留最小数字的精确位置结构。 

例如，如果我们有数字`{0: 3, 1: 1}`和$m = 2$，正确的数字是`10`， 不是`01`。 天真的排序会产生`01`，由于前导零规则而无效。 

另一个微妙的问题是误解选择步骤。 我们不能随意选择任意数字而忽略计数； 我们必须尊重可用性，同时确保字典顺序上尽可能最小的有效数量。 

## 方法

 蛮力策略是生成所有可能的选择方式$m$从可用的多重集中选择数字，然后排列每个选择以形成尽可能小的数字，最后取最小值。 即使忽略排列，仅选择子集在组合上也是巨大的：方式的数量在位数上呈指数级增长，并且每个选择的排列都会使情况变得更糟。 即使对于很小的输入，这也会很快变得不可行，更不用说当计数达到$10^9$。 

关键的观察结果是最佳数字具有非常严格的结构。 由于我们想要尽可能小的整数，因此我们总是更喜欢更早的数字。 然而，第一位数字有一个特殊的限制：它不能为零。 固定第一位数字后，所有剩余位置都可以按升序用最小的可用数字贪婪地填充。 

这将问题从组合选择简化为确定性构造：我们只需要确定第一个数字，然后按升序贪婪地消耗剩余的数字。 

一旦数字序列固定下来，剩下的挑战就是计算其数值模$10^9 + 7$无需显式构建它。 这是通过按顺序处理数字同时保持滚动位置权重来处理的。

 ## 算法演练

 ### 逐步构建

 1. 识别 1 到 9 中计数非零的最小数字，并将其用作第一位数字。 将其计数减一。 这是必要的，因为任何有效数字都不能以零开头，并且在所有有效选择中，最小的非零数字给出可能的最小主值。 
2. 将剩余数字视为大小的多重集$m-1$。 从这一点开始，零是被允许的，并且实际上是有益的，因为它会降低较早放置的值。 
3. 从数字 0 到 9 贪婪地构建剩余序列。对于每个数字，获取尽可能多的副本，但不能超过剩余的槽位。 
4. 在构造数字时，逐步计算其值。 从最重要、最有权重的位置开始$10^{m-1}$。 放置完每个数字后，将当前重量乘以 10 的模倒数，向右移动一个位置。 
5. 通过在每一步将数字×当前重量相加来累加结果，始终取模$10^9 + 7$。 

### 为什么它有效

 该结构是最优的，因为第一个数字主导整数的字典顺序，并且必须选择最小的有效非零数字来最小化它。 修复该约束后，所有剩余数字都不受前导限制，因此按升序对它们进行排序会产生尽可能小的后缀。 任何将较大数字放在前面的偏差都会严格增加字典和数字意义上的数字，并且无法在以后进行补偿。 

模计算是正确的，因为它与数字的位置定义完全匹配。 10 的滚动幂可确保每个数字根据其最终位置进行加权，并且通过乘以 10 的倒数来正确地转移权重而无需重新计算幂。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
INV10 = pow(10, MOD - 2, MOD)

def solve():
    T = int(input())
    for _ in range(T):
        arr = list(map(int, input().split()))
        m = arr[0]
        cnt = arr[1:]

        # pick smallest non-zero leading digit
        first = -1
        for d in range(1, 10):
            if cnt[d] > 0:
                first = d
                cnt[d] -= 1
                break

        # remaining slots
        rem = m - 1

        # build sequence implicitly and compute value
        power = pow(10, rem, MOD)
        ans = 0

        # place first digit
        ans = (ans + first * power) % MOD
        power = power * INV10 % MOD
        rem -= 1

        # fill remaining digits greedily
        for d in range(10):
            if rem == 0:
                break
            if cnt[d] == 0:
                continue
            take = min(cnt[d], rem)
            # sum of take digits all equal to d
            # contribution: d * (10^rem + ... + 10^(rem-take+1))
            # geometric series in modular form
            cur = power
            for _ in range(take):
                ans = (ans + d * cur) % MOD
                cur = cur * INV10 % MOD
            power = cur
            rem -= take

        print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现直接遵循构造逻辑。 首先通过扫描数字 1 到 9 选择前导数字。其余数字按升序处理，确保最终数字按字典顺序最小化。 模运算使用 10 的滚动幂，因此我们永远不需要存储或构造完整的数字。 

一个微妙的实现细节是更新`power`每次安置后。 这确保了每个数字都乘以正确的位置权重，而无需重复重新计算大幂。

 ## 边缘情况

 当零占主导地位的可用数字时，就会出现一种边缘情况。 认为$m=3$和数字是`{0:5, 2:1}`。 该算法首先选择`2`作为前导数字，然后用零填充剩余位置。 这产生`200`，这是正确的，因为任何其他排列要么从零开始，要么使用更大的前导数字。 

另一种情况是只有一种非零数字类型可用。 如果数字是`{0:100, 7:2}`和$m=2$，结果一定是`77`。 该算法正确地选择 7 作为前导数字，减少其计数，然后放置另一个 7，因为它仍然可用。 

最后一个微妙的情况是$m=1$。 这里的答案只是可用的最小非零数字。 该算法自然地处理这个问题，因为在选择第一个数字之后，不存在剩余位置并且构建阶段被跳过。
