 ## 问题理解

 我们得到一个仅由小写英文字母组成的字符串。 我们可以执行选择两个字母的操作`x`和`y`，然后替换每次出现的`x`在整个字符串中`y`一次全球扫荡。 该操作可以重复任意次。 

目标是转换整个字符串，使每个字符都变成字母`'i'`，并且我们希望最小化所需的操作次数。 

重要的是每个操作都不是本地的，它同时影响所选字符的所有出现。 这就产生了一个问题，即字母如何逐渐“合并”到其他字母中，直到所有内容都塌陷成`'i'`。 

约束条件`|s| ≤ 10^4`意味着我们不需要为每个角色的每个操作模拟任何内容。 仅当操作数量非常小且受字母表大小限制时，任何重复扫描或模拟整个字符串多个步骤的转换的解决方案仍然会通过。 由于只有 26 个可能的字母，因此我们应该期待一个取决于字母类型而不是字符串长度的解决方案。 

一个微妙的点是我们并不被迫直接使用`'i'`作为操作期间的中间符号。 即使`'i'`最初并不存在于字符串中，我们可以将任何字母映射到`'i'`。 

值得隔离的边缘情况来自小字母：

 如果字符串是`"i"`，答案是`0`因为不需要任何改变。 

如果字符串是`"abc"`，最佳行为是映射每个`a`,`b`， 和`c`进入`'i'`，进行 3 次操作。 一个粗心的想法可能会尝试首先将所有内容映射到一个字母中，然后再映射到`'i'`，但这永远不会提高计数。 

如果字符串是`"aii"`， 仅有的`a`需要改变，所以答案是`1`。 

这些例子表明，成本仅取决于有多少个不同的字母与`'i'`。 

## 方法

 暴力解释是考虑逐步应用操作，每次都维护完整的字符串。 在每一步中，我们都可以尝试所有对`(x, y)`并模拟替换`x`和`y`，然后递归或搜索达到所有字符都已存在的状态的最少步数`'i'`。 这是所有可能字符串上的最短路径问题，其中每个状态转换都是一个全局替换。 

国家空间是天文数字般巨大。 尽管每个字符串的长度可达`10^4`，可能的不同字符串的数量是`26^n`，即使我们忽略结构并仅考虑字母分区，转换仍然形成指数状态上的图。 这立即使得暴力破解变得不可能。 

关键的观察结果是该操作仅影响字母身份，而不影响位置。 每个包含相同字母的位置永远表现相同。 因此，状态是通过字符串中当前存在的字母来完整描述的。 

从这个角度来看，每次操作都会删除一个字母符号`x`并将其替换为`y`。 这严格减少了不同字母的数量，除非`x`和`y`已经一样了。 既然我们希望一切都成为`'i'`，唯一重要的字母是原始字符串中除`'i'`。 每个这样的字母都必须至少被“消除”一次，因为除了将其映射到另一个符号之外，没有其他方法可以使其消失，并且最终所有内容都必须崩溃为`'i'`。 

我们总是可以通过直接映射每个非`'i'`信入`'i'`一次。 任何间接链，例如`a -> b -> i`并没有减少操作次数，因为`a -> b`无助于减少不同字母的数量，除非`b`已经是`'i'`。 

所以答案变成了纯粹的组合：计算字符串中出现了多少个不同的字母，如果出现则减一`'i'`就在其中。

 ## 算法演练

 我们将问题简化为跟踪字符串中存在哪些字母。 

1. 扫描一次字符串并记录出现的字母。 

这很重要，因为只有不同字符集会影响操作数量； 职位无关紧要。 
2. 计算总共有多少个不同的字母。 
3. 检查是否`'i'`就在这些字母之中。 
4. 如果`'i'`存在，则答案是`distinct_count - 1`， 自从`'i'`不需要删除或转换。 
5. 如果`'i'`不存在，答案是`distinct_count`，因为每个字母都必须直接转换为`'i'`。 

步骤 4 和 5 中的决策反映了是否`'i'`是字符串初始字母表的一部分，或者必须通过转换引入。 

### 为什么它有效

 每个操作恰好消除一个源字母`x`通过将其合并到某个目标字母中`y`。 该过程一直持续到仅`'i'`遗迹。 任何操作都不能从系统中消除多个不同的字母，因为只有一个`x`根据操作选择。 因此，每个不同的字母都不同于`'i'`需要至少一项操作。 从每个这样的字母直接转换为`'i'`达到这个下限，使其成为最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    
    present = [False] * 26
    for ch in s:
        present[ord(ch) - ord('a')] = True

    distinct_count = sum(present)
    i_idx = ord('i') - ord('a')

    if present[i_idx]:
        print(distinct_count - 1)
    else:
        print(distinct_count)

if __name__ == "__main__":
    solve()
```该实现仅跟踪字母的存在，避免任何替换模拟。 关键的微妙之处是确保我们对待`'i'`特别是当它存在时，因为它不需要被转化掉。

 ## 边缘情况

 如果字符串只包含`'i'`，该算法仅标记一个不同的字母并减去一个，因为`'i'`存在。 计算值变为零，与不需要任何操作的事实相匹配。 

对于像这样的字符串`"zzzz"`，不同的集合只包含`'z'`，并且自从`'i'`不存在，答案是一。 该算法正确执行单个概念转换`z -> i`。 

对于包含每个字母的字符串，包括`'i'`， 例如`"abci"`，不同计数为 4 并且`'i'`存在，所以答案是 3。算法正确地避免了计数`'i'`作为一封需要删除的信。
