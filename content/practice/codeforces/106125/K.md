 ## 解决方案
 ## 问题理解

 两名玩家即将根据密封的出价进行一次交换。 一个玩家，老麦克唐纳，已经在杯子里藏了一些硬币，你确切地知道有多少硬币，称之为$c$。 你还有$n$可用硬币。 您必须选择一个号码$x$，代表您的出价。 

两个出价公布后，硬币就会交换：您收到麦当劳$c$硬币，他收到你的$x$硬币。 然后，谁出价多，谁就奖励一头牛。 如果出价相同，则不会有奶牛易手。 

你的目标是按字典顺序分为两层。 首先，您希望在单次交互后最大化最终获得的奶牛数量。 仅当多项选择$x$给同样的牛结果，你更喜欢在交换后给你留下更多硬币的那个。 由于交易所总是交换出价，因此交易后您的最终代币数量为$n - x + c$，所以在相同的奶牛结果中，您更喜欢较小的$x$。 

输出是一个整数$x$，您选择的出价。 

约束条件允许$c, n \le 10^9$，这意味着任何解决方案每个测试用例的时间都必须恒定。 没有空间对一系列可能的出价进行模拟，但博弈的结构足够简单，可以通过分析得出最佳决策。 

一些边缘情况很重要。 

如果你准确出价$c$，结果是平局并且没有牛被交换。 幼稚的策略可能会错误地将其视为中性或等同于输赢，但它是一个独特的结果类别。 

如果您出价超过$n$，该举动无效，因为你不能花费你没有的金币。 这使得决策边界位于$x \le n$至关重要的。 

如果$c = 0$，麦克唐纳没有做出任何贡献，任何积极的出价都会立即赢得牛，但最小化出价成为次要目标。 

## 方法

 暴力方法会尝试所有可能的出价$x$从$0$到$n$。 对于每个候选人，我们模拟结果：比较$x$和$c$，确定我们是否获得或失去一头牛，并计算最终的硬币$n - x + c$。 然后我们根据两级目标选择最好的。 

这可以正常工作，因为问题只有$n+1$可能性，并且每次评估都是常数时间。 然而，随着$n$最多$10^9$，迭代所有值将需要每次测试十亿次操作，这远远超出了 1 秒的限制。 

关键的观察结果是，结果仅取决于之间的比较$x$和$c$，而不是超出该阈值的幅度。 每个有效的出价都属于以下三个类别之一：$x < c$,$x = c$， 或者$x > c$。 在每个类别中，奶牛的结果是固定的，因此唯一有意义的优化是在次要硬币目标下选择最佳代表。 

如果$x > c$，你赢了牛。 在所有这些选择中，最终的硬币减少为$x$增加，因此最佳中标出价是严格大于的最小可能值$c$，即$c+1$，前提是不超过$n$。 

如果$x = c$，你没有牛，但保留了硬币$n$。 从奶牛数量来看，这总是比获胜更糟糕，所以只有当不可能获胜时才重要。 

如果$x < c$，你失去了牛。 其中，最大化您选择的硬币$x = 0$，但这仅在不可能获胜时才重要。 

因此，解决方案取决于是否存在中标，即是否存在$c+1 \le n$。

 ## 算法演练

 1. 比较$c + 1$和$n$。 这决定了是否存在严格击败麦克唐纳但仍然负担得起的出价。 
2.如果$c + 1 \le n$， 选择$x = c + 1$。 这是可能的最小中标出价，它保证了牛获胜，同时最大限度地减少了硬币的花费。 
3.否则，任何出价都不能严格超过$c$在预算之内，所以获胜是不可能的。 
4. 在这种情况下，选择$x = 0$。 这完全避免了损失硬币，同时承认牛丢失或被绑的情况是不可避免的。 

### 为什么它有效

 游戏的结构将所有结果降低到单调阈值$c$。 以上任意出价$c$产生与赢得一头牛相同的分类结果，但成本严格随着出价而增加。 因此，最优获胜策略是高于阈值的最小可行值。 如果不存在这样的值，则所有获胜状态都无法达到，我们会回退到最佳可能的失败或平局配置，这是通过最小化出价来实现的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    c, n = map(int, input().split())
    if c + 1 <= n:
        print(c + 1)
    else:
        print(0)

if __name__ == "__main__":
    solve()
```该代码直接实现了之前导出的阈值逻辑。 条件`c + 1 <= n`检查最小中标投标的可行性。 如果为真，我们输出该值。 否则，我们默认为零，因为任何积极的出价都无法击败麦克唐纳，只会减少最终的硬币。 

一个微妙的点是我们永远不需要考虑精确的出价$c$。 与失败或获胜相比，这种情况永远不会改善奶牛的结果，并且相对于失败或获胜，它也不会改善硬币平衡$x=0$在失败的政权中。

 ## 边缘情况

 当$c = n$，条件$c+1 \le n$失败，所以我们输出$0$。 用于输入`25 25`，算法选择$x = 0$。 1 到 25 之间的任何出价都会导致奶牛结果平局或损失，更高的出价是不可能的。 这个决定正确地避免了不必要的硬币损失。 

什么时候$c = 0$， 例如`0 10`，条件成立，我们选择$x = 1$。 这是最小的积极出价，保证牛获胜。 选择$x = 0$只会导致平局，而这在奶牛数量上是更糟糕的。 

什么时候$n = 0$，我们不能提出任何积极的出价。 用于输入`5 0`，条件失败，我们输出$0$，这是唯一有效的举动。 即使我们失去了这头牛，在有限的条件下也没有其他选择。
