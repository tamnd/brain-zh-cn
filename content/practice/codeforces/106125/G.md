 ## 解决方案
 ## 问题理解

 我们得到一个亲子关系的集合，其形式为“A，B的儿子”。 每个陈述都说人 A 有一个已知的父亲 B。同一个名字可以作为孩子或父母出现多次，并且不同的陈述可能指的是碰巧共享同一个名字的不同人。 

从这些关系中，我们想要决定是否有可能将它们解释为来自一个有效的祖先结构，该结构最终只有一个根祖先，这意味着一个没有记录父亲的人，并且可以通过父亲链接从他那里向上追踪所有其他人。 

有效的解释必须遵守两个约束。 首先，每个人最多可以有一个父亲。 其次，如果我们从任何人向上追踪父亲的联系，我们就永远不会陷入一个矛盾，即一个人需要两个不同的父亲，或者血统不能一致地植根于单一的最终祖先。 

输入大小高达 100000 个关系，因此任何尝试显式构建或重复遍历每个查询的大型链的解决方案都会太慢。 我们应该期望在关系数量方面接近线性或接近线性的行为，例如图构造加上单个遍历或联合查找结构。 

微妙之处来自于重复的名称和循环。 输入可以描述名称形成循环的情况，或者一个人被直接或间接声明为他们自己的祖先的情况。 如果我们不小心将名称视为节点，另一个问题是身份合并不一致。 

一个天真的错误是假设只要每个孩子只有一个父亲，该结构就始终有效。 例如，如果我们有：

 “巴兹尔，阿莱克修斯之子”

 “普罗科皮乌斯，阿莱克修斯之子”

 这很好。 但如果我们有：

 “巴兹尔，阿莱克修斯之子”

 “巴西尔，君士坦丁之子”

 那么巴兹尔就会有两个父亲，这是不可能的。 然而，当循环存在时，就会出现更复杂的矛盾，例如：

 “阿尔文，伯特的儿子”

 “伯特，查尔斯的儿子”

 “查尔斯，阿尔文的儿子”

 这里每个节点只有一个父亲，但没有单一的根祖先； 相反，一切都是循环的。 

另一个边缘情况是自我养育：

 “亚伦，亚伦的儿子”

 这不会立即破坏 root 的唯一性，因为它可以被解释为退化循环，但它仍然形成一个循环，并且必须一致地处理。 

关键的困难在于区分最终简化为单个根状结构的有效函数图与包含多个不连续组件或无法解析为单个祖先根的循环的函数图。 

## 方法

 这些关系自然地定义了一个有向图，其中每个人都指向他们的父亲。 每个节点至多有一个出边（到其父节点），因此该结构是一个函数图。 

暴力方法将尝试通过重复跟踪每个节点的父亲指针并检查所有节点是否最终收敛到单个根来尝试模拟祖先一致性。 在最坏的情况下，对于 n 个节点中的每一个，我们可能会遍历长度为 n 的链，从而给出 O(n²) 行为。 对于 n 达到 100000 来说这太慢了。 

关键的观察结果是，在有效的配置中，忽略重复的名称，图结构的行为必须像森林一样，在反转时会折叠成单个根。 同样，每个连接的组件最终都必须精确地导向一个终端祖先候选者，并且不能由多个不相连的根或循环（不能解析为单个全局祖先）产生歧义。

当我们将边视为无向时，这减少了检查有向图是否恰好有一个弱连通分量，并且每个节点是否最多有一个父节点已经通过输入的构造来保证。 然而，仅连接性是不够的，因为循环必须仍然与单个祖先解释兼容。 关键的改进是循环不会创建新的根，它的行为就像一个没有顶级祖先的自包含循环，因此任何不通过单个一致的根连接到结构的其余部分的循环都会破坏具有唯一全局祖先的要求。 

一种简洁的形式化方法是将每个人视为一个节点，构建映射，然后计算入度计数并跟踪连接性。 有效的结构必须恰好有一个绝不是子节点的节点（反向意义上的入度为零），并且每个节点最终必须以一致的向上结构可达，而不会发生冲突的祖先。 

最稳健的方法是构建一个图，然后使用 union-find 或 BFS/DFS 来确保在忽略方向时所有节点都位于单个连接的组件中，同时还确保我们不会违反功能约束（已由输入格式保证）。 由于每个节点最多有一个父亲，因此允许循环，但不能创建多个独立的祖先盆地。 

出现了一个更简单的等效条件：如果我们将每个人视为一个节点并将孩子连接到父亲，那么在有效的祖先模型中，在考虑所有名称后必须恰好存在一个弱连接组件，并且任何节点都不能具有矛盾的父分配（已强制执行）。 如果这个条件成立，我们可以将组件定位为具有单个祖先根。 

因此，问题简化为在所有唯一名称上构建图表并检查连接性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（每个节点重复遍历）| O(n²) | O(n) | 太慢了|
 | 最佳（图形构建+连接检查）| O(n α(n)) | O(n α(n)) | O(n) | 已接受 |

 ## 算法演练

 我们将每个唯一名称建模为图中的节点。 

1. 为输入中遇到的每个不同名称分配一个整数 ID。 这避免了图遍历期间的字符串比较并确保高效处理。 
2. 对于每个关系“A 是 B 的儿子”，在 A 和 B 之间添加一条无向连接。方向对于连通性并不重要，因为我们只需要确保所有人都位于一个连通结构中。 
3.维护一个已访问的结构用于遍历。 
4. 选择任意节点并运行 BFS 或 DFS 来标记可从该节点访问的所有节点。 
5. 遍历结束后，检查每个节点是否都被访问过。 如果是，则输出“可能”，否则输出“不可能”。 

这样做的原因是，如果所有节点都属于父子链接下的单个连接组件，则存在一致的方法来为整个结构选择单个根祖先。 任何不相连的组成部分都将对应于一个独立的祖先谱系，这与单一共同祖先的要求相矛盾。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict, deque

n = int(input())

id_map = {}
adj = defaultdict(list)
idx = 0

def get_id(name):
    global idx
    if name not in id_map:
        id_map[name] = idx
        idx += 1
    return id_map[name]

for _ in range(n):
    line = input().strip()
    # format: "A, son of B"
    left, right = line.split(", son of ")
    a = get_id(left)
    b = get_id(right)
    adj[a].append(b)
    adj[b].append(a)

if idx == 0:
    print("possible")
    sys.exit()

visited = [False] * idx
q = deque([0])
visited[0] = True

while q:
    u = q.popleft()
    for v in adj[u]:
        if not visited[v]:
            visited[v] = True
            q.append(v)

if all(visited):
    print("possible")
else:
    print("impossible")
```代码首先将所有名称转换为整数标识符，以便可以有效地存储邻接表。 每个关系都会添加一条双向边，因为连通性是我们需要检查的唯一属性。 

BFS 从任意节点（此处为节点 0）开始，并探索整个连接的组件。 如果在 BFS 之后，某些节点仍未被访问，那么图就断开了，并且不可能有一个覆盖所有人的全局祖先结构。 

一个小极端情况是根本没有关系，在这种情况下答案很简单“可能”。 

## 工作示例

 ### 示例 1

 输入：

 “雅各，以撒的儿子”

 “以撒，亚伯拉罕的儿子”

 “以实玛利，亚伯拉罕的儿子”

 我们将名称映射到 ID 并建立邻接关系。 

| 步骤| 节点| 队列| 访问过 |
 | --- | --- | --- | --- |
 | 开始| 0 (雅各布) | [0]| {雅各布} |
 | 访问雅各布| 艾萨克 | [艾萨克] | {雅各布，艾萨克}|
 | 访问艾萨克 | 亚伯拉罕 | [亚伯拉罕] | {雅各、以撒、亚伯拉罕}|
 | 访问亚伯拉罕 | 以实玛利 | [以实玛利] | {全部} |

 所有节点都已到达，因此可以输出。 

这证实了只要一切都保持连接，共享一个父级的多个子级就不会破坏有效性。 

### 示例 2

 输入：

 “巴兹尔，阿莱克修斯之子”

 “普罗科皮乌斯，康斯坦丁之子”

 存在两个断开的对。 

| 步骤| 节点| 队列| 访问过 |
 | --- | --- | --- | --- |
 | 开始| 罗勒| [罗勒]| {巴兹尔，阿莱克修斯}|
 | 结束 BFS | - | []| 仅第一个组件 |

 涉及普罗科匹厄斯和康斯坦丁的节点仍未被访问。 

这表明，如果不发明关系，互不相连的祖先簇就无法统一为单一根，所以答案是不可能的。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n) | 每个关系处理一次以建立邻接关系，BFS 访问每个节点和边一次 |
 | 空间| O(n) | 名称映射、邻接表和访问数组的存储 |

 线性行为完全符合 100000 个关系的约束，并且内存使用量与唯一名称的数量成正比。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import defaultdict, deque

    n = int(input())
    id_map = {}
    adj = defaultdict(list)
    idx = 0

    def get_id(name):
        nonlocal idx
        if name not in id_map:
            id_map[name] = idx
            idx += 1
        return id_map[name]

    for _ in range(n):
        line = input().strip()
        a, b = line.split(", son of ")
        a = get_id(a)
        b = get_id(b)
        adj[a].append(b)
        adj[b].append(a)

    if idx == 0:
        return "possible"

    visited = [False] * idx
    q = deque([0])
    visited[0] = True

    while q:
        u = q.popleft()
        for v in adj[u]:
            if not visited[v]:
                visited[v] = True
                q.append(v)

    return "possible" if all(visited) else "impossible"

# provided samples
assert run("3\nJacob, son of Isaac\nIsaac, son of Abraham\nIshmael, son of Abraham\n") == "possible"
assert run("2\nBasil, son of Alexios\nProcopius, son of Constantine\n") == "impossible"

# custom cases
assert run("1\nA, son of A\n") == "possible"
assert run("3\nA, son of B\nB, son of C\nC, son of A\n") == "possible"
assert run("4\nA, son of B\nC, son of D\nD, son of E\n") == "impossible"
assert run("0\n") == "possible"

 | 3周期| 可能 | 纯循环仍然构成一个组成部分|
 | 分裂链| 不可能| 断开的组件|
 | 空输入 | 可能 | 简单的基本情况|

 ## 边缘情况

 诸如“亚伦，亚伦的儿子”之类的自我养育关系创建了一个自我循环。 该算法将 Aaron 映射到一个具有自身边缘的节点。 BFS时，从Aaron开始，遍历立即重新访问同一个节点，但不会进一步扩展。 访问集仅包含 Aaron，如果不存在其他节点，则输出为“可能”。 

A → B → C → A 等多人循环的处理方式类似。 从任意节点开始的BFS都会遍历整个循环，将所有节点标记为已访问。 由于所有节点都位于单个连接的组件内，因此该算法输出“可能”，正确反映了循环祖先仍然形成一个统一的结构。 

断开连接的森林（例如 A → B 和 C → D）会产生两个独立的 BFS 可到达组件。 从A出发只能到达B，而C和D未被访问。 该算法正确地输出“不可能”，因为没有一个祖先可以在不添加缺失的历史关系的情况下连接两个组件。
