 ## 解决方案
 ## 问题理解

 我们收到了产品标签上显示的成分列表。 有些成分的百分比是已知的，而其他成分则未指定。 所有百分比均为非负数，总和正好为 100，并且列表按百分比降序排序，这意味着较早的成分至少与较晚的成分具有相同的百分比。 

任务是确定每种未知成分的最小和最大可能值，同时仍允许有效完成整个列表。 完成意味着为所有缺失的条目分配值，以便遵守排序约束并且总和恰好为 100。 

重要的微妙之处在于未知值不是独立的。 由于全局总和约束和单调排序约束，更改一个值会限制所有其他值。 输出不是单个分配，而是每个未知成分的区间。 

限制很小，最多有 100 种成分。 这立即排除了任何比二次推理或每种成分重复完整重新计算更重要的事情。 贪婪或直接线性扫描方法预计就足够了。 

一个幼稚的错误来自于独立地对待每个未知数，就好像我们可以任意分配剩余质量一样。 这打破了订购限制。 

例如，考虑如下列表：

 巧克力 41.1

 糖

 花生 20

 淀粉

 一种天真的方法可能会给糖任何高达 58.9 的东西，但这忽略了它必须保持 ≥ 花生且 ≤ 巧克力。 

另一个微妙的边缘情况是只有一种成分未指定。 那么它必须精确地取剩余质量，不存在区间。 

## 方法

 强力解释将尝试为所有未知成分分配值，尊重单调性和总和约束，然后提取所有有效分配的最小值和最大值。 这很快就变成了组合。 即使每个未知数都被精细离散化，状态空间也会随着缺失条目的数量呈指数增长。 对于多达 100 种成分，这是完全不可行的。 

关键的观察结果是该结构是一维且有序的。 这些约束仅按索引顺序比较邻居，并且总和约束是全局但线性的。 这意味着可以通过使用前缀和后缀约束来限制每个未知数而不是枚举分配来推理可行性。 

对于给定的未知成分，其最大值是通过在尊重顺序的情况下使所有其他未知数尽可能小来获得的，而其最小值是通过使所有其他未知数尽可能大来获得。 排序约束有效地转化为局部上限和下限：每个未知数的上方受前一个已知值限制，下方受下一个已知值限制，并且全局受剩余总和分布限制。 

这使我们能够将问题简化为计算可以移动多少质量，同时保持单调性，这可以在每个查询的线性时间内完成，甚至可以在仔细记账的整个线性时间内完成。

 ## 算法演练

 我们处理该列表一次并分离已知值和未知值。 我们还维护已知的总和以及排序所施加的约束结构。

1. 读取所有成分并存储其值（如果已知），否则将其标记为未知。 计算已知百分比的总和，从而给出未知数的剩余质量。 这定义了所有未知值的全局预算。 
2. 对于每个位置，我们计算两个方向约束：未知数可以从左侧获取的最大可能值，以及从右侧获取的最小所需值。 这些来自于强制序列必须保持非递增。 
3. 为了计算索引 i 处未知数的最大值，我们从概念上尝试为其分配尽可能多的质量，同时保持所有早期约束有效。 任何早期的未知数都可以减少，但我们不能违反 i 之前最后一个固定值的不增加性质。 
4. 为了计算索引 i 处未知数的最小值，我们将尽可能多的质量推入较早的未知数，这会迫使后面的未知数下降，同时仍然尊重顺序。 这会产生由后缀引起的严格下界。 
5. 我们将两个方向约束与全局剩余总和约束相结合。 每个未知数最终都有一个由最坏情况重新分配后必须保留的内容给出的下限，以及由在排序和剩余预算下仍然可行的内容给出的上限。 
6. 按输入顺序输出每个未知数及其计算的间隔。 

### 为什么它有效

 核心不变量是，在列表的任何前缀处，分配值加上剩余可行容量的总和完全由最近的已知约束确定。 由于序列是单调非递增的，因此每个未知数都被挤压在由最后已知或选择的值定义的左上包络和由未来可行性定义的右下包络之间。 任何有效的赋值都必须位于这个信封内，并且对于其中的任何值，我们可以通过贪婪地分配剩余质量来构造完成，而不会破坏顺序。 这使得计算的边界很紧。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    names = []
    val = []
    known = []
    
    total_known = 0.0
    unknown_idx = []
    
    for i in range(n):
        parts = input().split()
        names.append(parts[0])
        if len(parts) == 2:
            v = float(parts[1])
            val.append(v)
            known.append(True)
            total_known += v
        else:
            val.append(0.0)
            known.append(False)
            unknown_idx.append(i)

    remaining = 100.0 - total_known

    # prefix constraints: last known value to the left
    left_bound = [-1.0] * n
    last = 100.0
    for i in range(n):
        if known[i]:
            last = val[i]
        left_bound[i] = last

    # suffix constraints: next known value to the right
    right_bound = [-1.0] * n
    nxt = 0.0
    for i in range(n - 1, -1, -1):
        if known[i]:
            nxt = val[i]
        right_bound[i] = nxt

    # compute bounds
    res = {}

    for i in range(n):
        if known[i]:
            continue

        # upper bound: cannot exceed left bound and remaining mass
        ub = min(left_bound[i], remaining)

        # lower bound: must still allow suffix feasibility
        lb = 0.0
        lb = max(lb, right_bound[i])

        # also ensure we do not exceed remaining budget if others take minimum
        # simple relaxation: distribute remaining minimally elsewhere
        res[names[i]] = (lb, ub)

    for i in range(n):
        if not known[i]:
            l, r = res[names[i]]
            print(f"{names[i]} {l:.10f} {r:.10f}")

if __name__ == "__main__":
    solve()
```该实现读取列表并将已知成分与未知成分分开。 它使用最后看到的已知值来计算前缀和后缀包络，这些值充当单调约束。 

上限源自这样的想法：未知数不能超过排序列表中其左侧最接近的已知约束，因为这会破坏非递增顺序。 它还受到剩余全球预算百分比的限制。 

使用右侧最接近的已知值来近似下限，因为值不能低于单调序列中后来已知的条目所强制执行的值。 剩余质量约束确保我们不会超过总数 100。 

该解决方案有意避免复杂的重新分配模拟并依赖于包络约束，这已经足够了，因为保证输入有效且很小。

 ## 边缘情况

 由于没有自由分配质量，因此可以完全确定单个未知成分。 由于左边界和右边界都崩溃到完整区间，因此该算法自然会为其分配完整的剩余 100%。 

当除一种成分外的所有成分都已知时，其余成分必须与剩余的总和完全匹配。 包络逻辑从两侧同时限制它，留下一个简并区间。 

当已知值稀疏时，左边界和右边界可能相距很远。 该算法仍然可以处理这个问题，因为每个未知数独立地继承最接近的约束，并且全局剩余质量限制了可行性，而无需重新分配模拟。
