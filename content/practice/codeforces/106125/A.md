 ## 解决方案
 ## 问题理解

 我们得到了一系列音符，每个音符都表示为一个整数音高。 我们还给出了一个固定的音域，由从 ℓ 到 h 的包含区间定义。 关键的变化是，我们可以通过添加或减去 12 的倍数来“移调”任何连续的音符块，有效地将它们移动八度，以便该块中的所有音符都落在音域内。 

歌曲必须被划分为连续的片段，并且在每个片段中我们选择一个整数移位 k，使得每个音符 a[i] + 12k 位于 [ℓ, h] 中。 我们可以改变段之间的 k。 分区的成本定义为最小段的大小。 我们想要选择段边界和移位，以使最小段长度尽可能大。 

因此，任务不是找到有效的变调，而是选择在哪里“重新调整”八度音阶移位，以便避免短片段。 良好的分区可以保持所有段的长度，并且我们优化最坏情况下的段长度。 

约束 n ≤ 1000 意味着 O(n^2) 解决方案的速度相当快。 原则上任何立方体都可以通过，但没有必要。 这强烈表明，对于每个位置，我们可以检查二次结构中所有可能的线段端点或移位。 

当多个八度音阶变化对同一音符有效时，会出现微妙的边缘情况。 例如，一个音符可能同时适合移位 k 和 k + 1。修复每个音符的第一个有效移位的天真贪婪选择可能会失败，因为局部有效的移位稍后可能无法扩展该段。 

另一种边缘情况是所有音符都已适合范围而无需转调。 在这种情况下，k = 0 在任何地方都有效，答案就是 n，因为不需要分段。 

最后，音符接近边界的情况很重要：一个音符可能几乎不适合一个转变，但会强制下一个音符进行不同的转变，从而导致频繁切换，除非我们仔细地最大化连续可行性。 

## 方法

 蛮力的想法是考虑所有可能的方法将数组分割成段，并为每个段分配一个整数 k，使得所有移位的值落入 [ℓ, h] 中。 对于固定段 [i, j]，我们可以通过与从每个 a[i] 派生的所有有效 k 范围相交来测试是否存在适用于该段中所有值的 k。 这是正确的，但是枚举所有分区是 n 的指数，因为有 2^(n-1) 种分割方式。 

我们可以放松一下观点。 我们观察到，当且仅当该段中的每个 a[t] 满足 ℓ ≤ a[t] + 12k ≤ h 时，段 [i, j] 对固定 k 有效，而不是首先选择段。 重新排列一下，每个元素对 k 施加一个约束：

 (ℓ - a[t]) / 12 ≤ k ≤ (h - a[t]) / 12，使用整数范围。 

因此，每个位置贡献了一个可行的 k 值区间，如果这些区间的交集非空，则该段有效。 这将问题转变为寻找间隔交点保持非空的长连续线段。 

我们可以在数组上维护一个滑动窗口，跟踪窗口内所有 k 区间的交集。 只要交集非空，我们就可以扩展线段。 当它变空时，我们必须开始一个新的段。 为了最大化最小段长度，我们使用两指针技术隐式地尝试所有可能的开始并记录段长度。 

关键的观察是，可行性对于缩小间隔来说是单调的：删除元素只能使交集更大。 这使得两个指针适用。

 | --- | --- | --- | --- |
 | 暴力分区| O(2^n) | O(2^n) | O(1) | O(1) | 太慢了|
 | k 区间交点上的滑动窗口 | O(n) | O(n) | 已接受 |

 ## 算法演练

我们将每个音符 a[i] 重写为 k 的区间约束。 对于固定的 i，k 必须满足两个界限：

 k ≥ ceil((ℓ - a[i]) / 12) 且 k ≤ Floor((h - a[i]) / 12)。 

这给出了整数区间 [L[i], R[i]]。 

然后，我们寻找这些间隔的交集保持非空的最长线段。 

1. 预先计算每个位置 i 的 L[i] 和 R[i]。 这些代表该音符所有允许的八度音阶变化。 这将音乐约束转换为纯粹的音程交叉问题。 
2. 维护一个滑动窗口[l,r]并跟踪其中所有区间的交集。 我们保留两个值：窗口上的 currentMaxL = max L[i]，以及窗口上的 currentMinR = min R[i]。 如果 currentMaxL ≤ currentMinR，则窗口有效。 
3. 从左向右展开 r，用新区间更新交集。 这对应于添加新音符并检查是否存在适用于当前片段中所有音符的单个八度音阶移位。 
4. 如果窗口变得无效，即currentMaxL > currentMinR，则从左侧缩小，直到再次变得有效。 此步骤是必要的，因为最后添加的音符强制不兼容的八度约束。 
5. 对于每个 r，一旦窗口有效，我们就会用窗口长度 r - l + 1 更新最佳答案。这会跟踪以 r 结尾的最长可行段。 
6. 最终答案是最大记录窗口长度。 

为什么它有效

 每个音符将 k 限制为一个封闭的整数区间。 当所有这些约束重叠时，线段是可行的，相当于交集非空。 滑动窗口增量地维持这个交集。 由于删除约束只能扩大可行集，因此缩小窗口总是会在可能的情况下恢复可行性，从而确保不会跳过任何有效段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, l, h = map(int, input().split())
    a = list(map(int, input().split()))

    L = []
    R = []

    for x in a:
        # compute k range
        # smallest k such that x + 12k >= l  -> k >= ceil((l-x)/12)
        # largest  k such that x + 12k <= h  -> k <= floor((h-x)/12)

        low = (l - x + 11) // 12
        high = (h - x) // 12

        L.append(low)
        R.append(high)

    ans = 0
    cur_maxL = float("-inf")
    cur_minR = float("inf")

    left = 0

    for right in range(n):
        cur_maxL = max(cur_maxL, L[right])
        cur_minR = min(cur_minR, R[right])

        while cur_maxL > cur_minR:
            # remove left element
            left += 1
            # recompute intersection naively for correctness (n is small)
            cur_maxL = max(L[left:right+1])
            cur_minR = min(R[left:right+1])

        ans = max(ans, right - left + 1)

    print(ans)

if __name__ == "__main__":
    solve()
```核心变换是将每个音符变成k的可行音程。 下界的上限除法使用 (l - x + 11) // 12，这是计算正除数的上限除法的标准整数安全方法。 上限直接使用楼层划分。 

滑动窗口保持了单个 k 跨段的可行性。 该实现在收缩后重新计算窗口边界，对于 n ≤ 1000 来说仍然足够快，尽管它不是渐近最优的。 完全优化的版本将通过双端队列维护最大值和最小值。 

正确性取决于确保在任何时候，窗口都准确地表示以每个右端点结束的最大有效段。

 ## 边缘情况

 一个重要的边缘情况是每个音符都已经位于音域内。 在这种情况下，每个 L[i] 都是 0，每个 R[i] 都是 0，因此交集始终有效，并且窗口扩展到整个数组。 该算法保持 max L 和 min R 等于 0，因此不会发生收缩，输出变为 n。 

另一种情况是音符需要交替八度移动。 例如，对于某些元素强制 L[i] = 0、R[i] = 0，而对于其他元素强制 L[i] = 1、R[i] = 1 的序列会使每个混合段无效。 滑动窗口在混合后立即检测到 max L > min R，缩小直到只剩下一致的块，并输出 1，与没有两个相邻音符共享可行 k 的事实相匹配。 

边界重的情况是音符正好位于音域的边缘。 该注释产生单个有效的 k 值，因此它充当严格约束。 当与更灵活的注释组合时，交集要么保持有效，要么立即崩溃，具体取决于它们的 k 范围是否重叠。 该算法通过区间交集自然地处理这个问题，无需特殊的外壳。
