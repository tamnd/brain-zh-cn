 ## 解决方案
 ## 问题理解

 我们给定一个长度的二进制数组$n \le 1000$，最初全为零。 我们不直接设置位。 相反，我们重复选择一个连续的段$[l, r]$，系统将该部分“暴露”给随机性，用独立的公平硬币翻转替换其中的每一位。 

每次操作后，我们都会看到整个数组以及当前的百分比。 一旦数组中至少 70% 的元素变为 1，该过程就会立即停止。 我们最多允许进行 125 次操作。 

因此，我们真正的控制不是对位值的控制，而是对数组的哪些部分以及按什么顺序重新随机化的控制。 每个操作都会破坏所选段中以前的信息，并用新的无偏随机性替换它。 

约束条件$n \le 1000$而且只有 125 次操作意味着我们无法承受多次尝试确定性地修复每个位置之类的事情。 相反，该解决方案必须利用重复随机重采样在重叠段上的行为方式的结构。 

一个微妙的边缘情况是随机性不是累积的。 如果多次随机化一个段，则只有影响每个索引的最后一个操作很重要。 例如，如果我们随机化$[1, n]$两次，第一个结果被完全覆盖。 因此，任何假设随机性累积的策略都是不正确的。 

另一个极端情况是过度关注期望。 每个重新采样的位始终有 50% 的机会为 1，因此对整个数组的任何简单重复采样都会使全局分布保持在 50% 的中心位置。 如果没有结构，达到 70% 的可能性是天文数字上不可能的。 

## 方法

 暴力心理模型是重复随机化整个数组，并希望单个样本能够跨越 70% 的阈值。 每个完整操作都会生成一个均匀随机的二进制数组，因此一次尝试成功的概率是二项式$(n, 1/2)$变量超过$0.7n$。 为了$n = 1000$，这是指数级的小，因此在 125 次尝试内它基本上永远不会成功。 

关键的观察是我们并不局限于全局重采样。 我们可以有选择地仅对数组的一部分进行重新采样。 这使我们能够在本地“改进”配置，而不是重新启动所有内容。 这个想法是将数组视为小段的集合，并仅对当前表现不佳的那些段进行重复重新随机化，同时保留已经包含许多段的部分。 

这将这个过程变成了一个随机爬山系统。 每个操作都会为一个分段提供一个新的随机候选，并且我们利用这样一个事实：在多个随机样本中，我们可能会看到比该分段的当前配置更好的配置。 通过在许多细分市场中重复应用这一原则，总体密度会向上漂移。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 完全随机抽样|$O(125 \cdot n)$|$O(n)$| 概率太慢 |
 | 分段随机改进 |$O(125 \cdot n)$|$O(n)$| 高概率被接受|

 ## 算法演练

 我们将数组划分为几个大小适中的连续块，通常大约有 8 到 15 个元素。 目标是通过反复重新采样来独立改进每个块，并保持改善全局数量的效果。 

1. 将数组拆分为$k$大小相同的块（最后一个块可能更小）。 选择块大小是为了我们可以在 125 次操作内多次重新访问块。 
2. 反复选择一个还不够“足够好”的块，这意味着它包含的块数少于稳定性所需的块数。 
3. 通过发出查询对该块应用一项操作$[l, r]$对应的块范围。 这会用新的随机位替换整个块。 
4. 每次操作后，检查更新后的数组并计算全局百分比。 如果百分比与之前相比有所增加，我们会在概念上将更改保留为进度； 如果没有，我们仍然继续，但在后续步骤中优先考虑重新访问较弱的块。 
5. 继续这个过程，始终偏向当前数量落后的区块。 随着时间的推移，反复重新采样的块往往会积累更好的随机结果，这仅仅是因为我们不断在坏片段上“重试”。 
6. 如果全局百分比达到至少 70，则提前停止，否则使用所有 125 个操作。 

重要的机制是较小的块集中了随机性。 一块大小$B$在单次重采样中产生高密度块的概率非常大，并且跨块重复此操作会放大许多块同时变强的机会。 

### 为什么它有效

 每个块重采样都会产生一个独立的样本$B$位随机向量。 虽然期待依然存在$B/2$，方差很高，重复采样提供了多次获得异常良好实现的机会。 因为我们只关注仍然较弱的块，所以我们不断用新鲜的独立样本替换低质量区域。 这会产生漂移效应，其中薄弱区域会更频繁地“重新滚动”，直到它们达到有利的配置。 经过足够多的操作，足够多的块在本地跨越 70% 以上，这使得全局数组超过了 70% 的阈值。 

不变性并不是特定位单调改进，而是算法始终将计算工作集中在低密度区域，同时保留已经成功的区域免受进一步破坏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input().strip())
    a = [0] * n

    ops = 0

    def apply(l, r):
        nonlocal ops, a
        print(l + 1, r + 1, flush=True)
        ops += 1

        a = list(map(int, input().split()))
        _ = input().strip()  # percentage line ignored

    B = max(5, min(12, n))
    blocks = []
    for i in range(0, n, B):
        blocks.append((i, min(n, i + B)))

    while ops < 125:
        total_ones = sum(a)
        if total_ones * 10 >= 7 * n:
            return

        # pick a block with fewer ones than half its size if possible
        chosen = None
        best_deficit = 0

        for l, r in blocks:
            ones = sum(a[l:r])
            size = r - l
            deficit = size - ones
            if deficit > best_deficit:
                best_deficit = deficit
                chosen = (l, r)

        if chosen is None:
            chosen = blocks[ops % len(blocks)]

        l, r = chosen
        apply(l, r)

if __name__ == "__main__":
    main()
```每次交互后，实现都会维护当前的数组状态。 该阵列被划分为小块，以便每个操作都有显着改变局部密度的有意义的机会。 

每个步骤都会选择一个当前看起来最弱的块并对其进行重新采样。 这将有限数量的操作集中在改善最差区域上，而不是随机地将工作分散到整个阵列上。 

必须注意每次查询后刷新输出，因为这是一个交互式问题。 另一个微妙之处是，每次操作后，输入中的整个数组都会被替换，因此我们必须始终覆盖本地副本。

 ## 边缘情况

 对于非常小的$n$， 例如$n = 1$，任何单次重采样在第一次操作后都会立即成功，概率为 1，因为数组变为 0 或 1，并且阈值为 70%。 

对于大型$n$，主要风险是随机更新过于集中在已经很好的区域，导致一些块永久变弱。 块选择策略通过始终优先考虑密度最低的段来防止这种情况，确保不会无限期地忽略任何区域。 

对于大多数位已经是 1 的几乎一致的数组，算法自然会提前终止，因为一旦达到 70%，全局百分比检查就会停止进一步的操作。
