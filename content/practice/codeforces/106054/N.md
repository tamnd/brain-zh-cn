 ## 解决方案
 ## 问题理解

 我们在二维网格上得到一组点，每个点代表必须保护的树的位置。 政府希望建造一个与轴线对齐的简单封闭边界的围栏，这意味着其侧面与坐标轴平行。 栅栏必须完全包围其内部的所有点。 

还有一个额外的几何约束：每棵树必须严格距离栅栏至少一个单位。 因此栅栏不能直接穿过点的紧密边界框。 它必须向外扩展，以便在任何一棵树和栅栏之间的所有方向上都有一个单位缓冲区。 

目标是最小化这个矩形栅栏的总长度。 

输入最多可提供 100000 个点，坐标最多可达 1 亿个。 这立即排除了任何二次或三次几何方法，例如尝试所有点对或枚举候选矩形。 即使 O(n log n) 也是可以接受的，但是任何涉及点的成对组合的事情都太慢了。 

当只有一棵树时，就会出现微妙的失败情况。 在这种情况下，一个简单的边界框将具有零宽度和零高度，并且人们可能会错误地得出栅栏长度为零的结论。 然而，栅栏保持至少一个单位的要求迫使栅栏扩展成非退化矩形。 

例如，如果输入是单个点 (2, 3)，则正确的栅栏是以它为中心、边长为 2 的 2 x 2 正方形，周长为 8。任何忘记缓冲区约束的解决方案都将错误地输出 0。 

## 方法

 暴力方法会尝试通过在所有可能的整数坐标中选取左、右、下和上边界来显式选择矩形。 对于每个候选矩形，我们将检查所有点是否都位于内部且至少有一个单位的边距。 即使我们将候选边界限制为现有点附近的值，我们仍然会得到 O(n^4) 或最多 O(n^2) 种极值组合，这远远超出了 100000 个点的可行范围。 

关键的观察是最佳栅栏的形状完全由点集的极值坐标决定。 由于栅栏是轴对齐的并且必须包含具有统一边距的所有点，因此左边界被迫位于最小 x 坐标左侧 1 个单位，右边界位于最大 x 坐标右侧 1 个单位，对于 y 也是如此。 一旦这些极端被解决，就没有自由了，因为缩小任何一边都会违反可行性，而扩大任何一边只会增加周长。 

因此，问题简化为计算最小和最大 x 和 y 值并应用固定变换。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^4) | O(n^4) | O(1) | O(1) | 太慢了|
 | 最佳（最小/最大扫描）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们在跟踪极端坐标的同时处理这些点一次，然后计算扩展的矩形及其周长。

1. 初始化四个变量来跟踪极值：最小 x、最大 x、最小 y、最大 y。 这些从第一点开始，因此我们始终保持有效的界限。 这确保我们永远不会依赖人为的哨兵值。 
2. 迭代所有点。 对于每个点 (x, y)，更新两个坐标的最小值和最大值。 此步骤有效地构建仍然包含所有点的最紧密的可能边界框。 
3. 处理完所有点后，计算边界框的宽度为 max_x 减去 min_x，高度为 max_y 减去 min_y。 它们表示包含所有树的最小轴对齐矩形，而不考虑所需的缓冲区。 
4. 将两个维度扩大 2 个单位。 这是由于将每一侧向外移动 1 个单位：左侧减少 1，右侧增加 1，底部减少 1，顶部增加 1。因此，每个维度恰好增加 2。 
5. 使用宽度和高度之和的 2 倍计算展开矩形的周长。 这给出了栅栏的总长度。 

最终表达式为 2 * ((max_x - min_x + 2) + (max_y - min_y + 2))。 

### 为什么它有效

 可行栅栏必须包含距离其边界至少为 1 的每个点。 在最佳情况下，这迫使矩形的每一边都位于极值点之外恰好一个单位。 任何向内的运动都会在极端点立即破坏可行性。 任何向外的移动都会增加周长，但不会提高可行性。 因此，最优解由边界框在各个方向上均匀扩展一个单位来唯一确定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    min_x = min_y = 10**18
    max_x = max_y = -10**18

    for _ in range(n):
        x, y = map(int, input().split())
        if x < min_x:
            min_x = x
        if x > max_x:
            max_x = x
        if y < min_y:
            min_y = y
        if y > max_y:
            max_y = y

    width = max_x - min_x
    height = max_y - min_y

    width += 2
    height += 2

    print(2 * (width + height))

if __name__ == "__main__":
    solve()
```该实现是边界框参数的直接转换。 唯一微妙的一点是用足够大的哨兵初始化极值，以便任何有效的输入点都能正确替换它们。 另一个重要的细节是，在计算周长之前必须进行 2 的扩展，否则很容易出现差一的错误。

 ## 边缘情况

 单点情况是最微妙的。 用于输入`1 2 3`，该算法将所有极值设置为相等，在扩展之前产生零的宽度和高度。 添加所需的缓冲区后，两个维度都变为 2，产生周长 8。任何忘记扩展步骤的解决方案都将错误地输出 0，违反距离要求。 

另一种边缘情况是所有点都位于一条直线上。 边界框的宽度或高度为零，但扩展仍会产生有效的矩形。 该算法自然地处理这个问题，因为它在应用缓冲区之前不假设正区域。
