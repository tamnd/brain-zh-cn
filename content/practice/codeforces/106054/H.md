 ## 解决方案
 ## 问题理解

 给定一个未知整数的除数多重集$X$，但缺少一个除数。 总共，$X$正好有$N+1$正因数，我们得到$N$其中。 每个提供的数字都保证能被整除$X$，并且整个除数集中只有一个除数不存在。 任务是重建原始数字$X$和缺失的除数，或者确定存在多个有效的重建。 

除数集具有严格的结构：除数总是成对出现，其乘积为$X$。 此配对属性是连接所有给定数字的中心约束。 缺失的除数必须与某个已知的除数配对，并且该配对确定了$X$。 

约束允许最多$2 \cdot 10^5$数量，每个最多$10^{18}$。 这立即排除了任何尝试候选值的方法$X$或重复对大整数进行因式分解。 操作必须本质上是线性的或接近线性的$N$，并且任何依赖于 128 位中间值的排序和算术的解决方案都是可以接受的。 

当出现重复或不一致的配对候选时，就会出现一个微妙的问题。 如果有多个不同的值$X$是可能的，或者如果不同的配对意味着相互冲突的结果，则答案是不明确的，我们必须输出`*`。 另一个边缘情况是缺失的除数是$1$或者$X$，因为它们总是彼此配对，并且缺少其中一个会改变结构的不对称性。 

如果假设最大的数字总是$X$。 例如，如果缺失的除数很大，则提供的最大值可能小于$X$，使得这种贪婪的假设不正确。 

## 方法

 蛮力视角从除数的配对性质开始。 如果我们以某种方式猜到$X$，我们可以通过检查每个给定的数字是否能整除它以及缺失的除数是否存在来验证它。 然而，尝试所有候选人$X$是不可能的，因为$X$可以达到$10^{18}$，候选生成将取决于输入值的组合。 

一个更结构化的想法是利用这样一个事实：如果我们知道$X$, 每个除数$d$必须与$X/d$。 由于我们恰好缺少一个除数，因此我们可以尝试使用给定列表“模拟”这些配对。 如果我们对除数进行排序，很自然的尝试是将最小的与最大的配对，看看它们的乘积是否一致。 当不缺少除数时，此方法有效，但这里缺少一个元素，因此配对稍微损坏：一对将不完整。 

关键的见解是避免直接猜测缺失的除数，而是将其视为配对不匹配问题。 如果我们假设一个候选值$X$，它必须等于$a_i \cdot a_j$对于某些涉及缺失元素的配对。 这意味着每个有效$X$必须采用以下形式$a_i \cdot a_k$， 在哪里$a_k$要么是已知的除数，要么是缺失的除数。 

这导致了正确的结构观察：如果我们对所有给定的除数进行排序，那么真实的$X$必须等于$a_0 \cdot a_{n-1}$（如果没有缺失元素破坏配对），或者它必须来自涉及由缺失除数引起的一个“未配对”元素的乘积。 我们可以测试从配对端点派生的候选者，并在线性时间内验证它们。 

核心方法变成：尝试一小组看似合理的候选者$X$，导出隐含的缺失除数，并验证所有元素的一致性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（尝试所有 X） | 指数/不可行 | O(1) | O(1) | 太慢了|
 | 基于配对的验证 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们首先对给定的除数进行排序，以便配对结构变得可见。 排序是必要的，因为有效数字的除数是对称的$\sqrt{X}$。 

接下来，我们尝试构建候选值$X$。 自然的第一候选人是$a_0 \cdot a_{n-1}$，假设缺失的除数不会破坏最外层的配对。 我们还考虑到缺失的除数可能会迫使其中一个端点错误配对，因此我们通过交换配对结构一次来测试第二个候选端点。 

对于每位候选人$X$，我们通过使用两指针技术模拟除数配对来验证它。 我们将指针维护在最小和最大剩余值处。 如果这两个值的乘积等于$X$，我们都向内移动。 否则，我们将较小的值视为可能需要缺失的伙伴，并将隐含的缺失除数计算为$X / a_i$。 我们记录这个候选缺失值并继续验证。 如果我们遇到多个不一致的缺失值或除法不精确的不匹配，则该候选值$X$无效。 

检查完所有候选后，我们可能有零个、一个或多个有效重建。 如果恰好有一对有效$(X, \text{missing})$存在，我们输出它。 否则，我们输出`*`。 

关键的一点是，每个候选者都在一次线性扫描中得到验证，即使对于大量输入也能确保效率。 

### 为什么它有效

 除数配对属性强制每个除数恰好属于一个互补对，其乘积为$X$。 删除一个除数就恰好破坏了一对这样的除数。 所有其他对保持完整并且仍然必须完美匹配。 因此，任何有效的$X$必须保持除一对之外的所有配对的一致性，并且算法显式检查此条件。 任何不正确的候选值要么违反整除性，要么产生多个不兼容的缺失值，从而确保拒绝。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(arr, X):
    n = len(arr)
    i, j = 0, n - 1
    missing = None

    while i <= j:
        if i == j:
            # single leftover must be sqrt(X)
            if arr[i] * arr[i] == X:
                i += 1
                j -= 1
                continue
            if missing is None:
                missing = arr[i]
                i += 1
                j -= 1
                continue
            return None

        if arr[i] * arr[j] == X:
            i += 1
            j -= 1
        else:
            if missing is not None:
                return None
            if X % arr[i] != 0:
                return None
            missing = X // arr[i]
            i += 1

    if missing is None:
        return None
    return missing

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    a.sort()

    candidates = set()

    candidates.add(a[0] * a[-1])
    candidates.add(a[0] * a[1])
    candidates.add(a[-1] * a[-2])

    best = None

    for X in candidates:
        if X <= 0:
            continue
        missing = check(a, X)
        if missing is None:
            continue

        # verify consistency
        full = a + [missing]
        if len(full) != len(set(full)):
            # duplicates are fine, but must still form valid divisor structure
            pass

        full.sort()
        ok = True
        for i in range(len(full)):
            if X % full[i] != 0:
                ok = False
                break
            if full[i] * full[-1 - i] != X:
                ok = False
                break

        if ok:
            if best is None:
                best = (X, missing)
            else:
                print("*")
                return

    if best is None:
        print("*")
    else:
        print(best[0], best[1])

if __name__ == "__main__":
    solve()
```该解决方案首先对除数列表进行排序，以便候选配对变得有意义。 候选集故意很小，从端点派生，因为有效$X$必须与任何接近正确的配对结构中的极值相关。 

这`check`函数模拟除数配对，同时允许与缺失的除数对应的恰好一个中断。 它确保以受控方式解释每一个不匹配，并且不会引入超过一个缺失值。 

生成候选缺失除数后，我们重建完整的除数列表并验证严格的配对对称性。 此最终检查可防止误报，即局部一致的配对仍无法形成有效的除数系统。 

一个常见的陷阱是忘记缺失的除数会影响全局的配对结构，而不仅仅是端点的本地配对结构。 重建步骤确保全局一致性。

 ## 边缘情况

 One critical edge case is when the missing divisor is either 1 or$X$。 In that situation, the smallest and largest elements no longer form a correct pair. 该算法可以处理此问题，因为配对模拟会立即在端点处检测到不匹配，并相应地分配缺失值。 如果$1$缺失，重建的除数将显示为$X$，反之亦然，但验证将强制一致性。 

另一种边缘情况发生在$X$是一个完全平方数，缺少的除数是它的平方根。 在这种情况下，在两指针扫描期间，中间元素单独出现。 该算法显式检查条件$a[i]^2 = X$，确保处理这个单例时不会强制执行额外的缺失分配。 

最后的边缘情况是当多个候选值$X$可以通过本地验证，但无法实现全局一致性。 最终的重建检查通过在插入缺失值后强制执行全除数对称来捕获此问题，从而防止不正确的模糊输出。
