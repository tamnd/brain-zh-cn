 ## 解决方案
 ## 问题理解

 我们得到一个凸多边形，代表一个有围栏的花园。 有一个顶点是特殊的：山羊用一根长度相同的绳子绑在这个顶点上$L$。 山羊可以在多边形外自由移动，但不能穿过栅栏，绳子本身也不能穿过栅栏。 因此，可到达区域不仅仅是一个半径的圆$L$，因为该圆的某些部分可能会被多边形的边缘遮挡。 相反，山羊可以将绳子“缠绕”在多边形边界周围，从而有效地增加它可以从起始顶点扫过的角度区域。 

任务是回答多个查询：对于每个绳索长度，计算山羊可以到达的区域的确切面积。 

输入约束很大：最多$10^5$顶点和$10^5$查询。 这立即排除了任何根据查询从头开始重新计算几何图形或模拟连续移动的方法。 任何解决方案都必须对多边形进行一次预处理，并在对数或恒定时间内回答每个查询。 

一个微妙的困难是，可到达区域不仅仅是“被边缘剪裁的圆的扇区”。 如果绳子足够长，山羊可以沿着边界行走并到达远离起点的顶点，并且从每个这样的顶点它可以再次以圆弧扩展。 这将创建沿边界前缀链以多个多边形顶点为中心的圆扇区的并集。 

一个幼稚的错误是仅在欧几里得距离中假设单调性。 例如，在凸多边形中，距离起点最远的可见点不一定沿着笔直的径向方向； 可能需要先沿着边缘行走。 另一个微妙的问题是精度：通过计算角度`acos`是不稳定的，并且问题明确表明`atan2`。 

## 方法

 直接的蛮力想法是模拟给定绳索长度下山羊可以沿着多边形边界到达多远$L$。 从顶点 0 开始，我们沿着边行走，累积周长，直到剩余的绳子长度不够。 在每个可到达的顶点，我们将计算半径等于剩余绳索长度（被相邻边剪裁）的圆的扇区。 

这种方法很快就变得不可行。 对于每个查询，我们可能会遍历$O(N)$顶点并计算角度贡献，给出$O(NQ)$时间。 和$10^5$顶点和查询，这是$10^{10}$操作，远远超出极限。 

关键的结构观察是多边形是凸的，并且顶点从附着点开始按逆时针顺序给出。 这使得两个重要的属性成立。 首先，从起点沿着边界行走对应于围绕起点顶点的单调角度扫描。 其次，当我们沿着边界按顺序包含更多顶点时，可达区域不断增长。 

这使我们能够重新解释问题：我们不考虑绳索的连续运动，而是考虑每个边界顶点对最终区域的贡献。 每个顶点贡献一个扇形，其半径为到达该顶点后剩余的绳索长度，其角度为沿边界累积的外转角。 

我们可以预先计算前缀边长和前缀转角。 然后，每个查询都简化为查找我们可以沿着多边形边界走多远，并对前缀上的平滑函数求和，这可以使用前缀积分和二分搜索来加速。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟|$O(NQ)$|$O(N)$| 太慢了|
 | 前缀+二分查找+预计算|$O((N+Q)\log N)$或者$O(N+Q)$|$O(N)$| 已接受 |

 ## 算法演练

 1. 将起始顶点固定为索引 0，并按逆时针顺序计算沿凸多边形的所有边向量和边长度。 这使得从系绳点开始沿着边界自然行走。 
2. 计算边长的前缀和。 这让我们回答，对于任何顶点$i$，从起点开始沿边界有多远。 这一点至关重要，因为行走所花费的绳索长度正是行走的边界距离。 
3. 使用以下方法预先计算每个边缘方向的角度`atan2`。 从连续的边计算每个顶点的外部转动角度。 因为多边形是凸多边形，所以这些角度都是正的并且总和为$2\pi$。 
4. 建立这些转角的前缀和。 这使我们能够知道山羊到达顶点时累积的总角跨度$i$。 这个角度跨度对应于一旦绳索可以“缠绕”到该顶点，围绕起点的圆的可触及范围就可以达到。 
5.查询绳子长度$L$，首先确定山羊可以沿着边界走多远。 对前缀边长度使用二分搜索，找到最大索引$k$使得从顶点 0 到顶点的距离$k$至多是$L$。 
6.到达顶点后剩余的绳子$k$是$r = L - \text{dist}[k]$。 该剩余长度定义了一个以起始顶点为中心的圆形扇区，该扇形进一步向外延伸超出顶点$k$。 
7. 总面积由两部分组成。 第一部分是完全到达的顶点的贡献之和$0$通过$k$，每个贡献由转动角度确定的扇形面积。 第二部分是顶点剩余绳索的最终部分扇区贡献$k$，正比于$r^2$乘以超出该顶点的可用角度跨度。 
8. 使用前缀和来计算全部贡献$O(1)$定位后$k$，并直接计算最终的部分项。 

### 为什么它有效

 多边形的凸性保证边界行走对应于围绕起始顶点的严格排序的角度扫描。 这确保了每次山羊到达新顶点时，新暴露的区域恰好是添加到先前可到达区域的圆形扇区，而不会重叠或丢失楔形。 顶点的前缀充分表征了可达区域的几何形状，因此将问题简化为前缀距离和前缀角度和，从而保留了精确的面积，而无需重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def atan2(y, x):
    return math.atan2(y, x)

N, Q = map(int, input().split())
pts = [tuple(map(int, input().split())) for _ in range(N)]
L = list(map(int, input().split()))

# edges
dx = []
dy = []
dist = [0] * N

for i in range(1, N):
    x1, y1 = pts[i-1]
    x2, y2 = pts[i]
    dx.append(x2 - x1)
    dy.append(y2 - y1)
    dist[i] = dist[i-1] + math.hypot(dx[-1], dy[-1])

# angle of edges
ang = [math.atan2(dy[i], dx[i]) for i in range(N-1)]

turn = [0.0] * N
for i in range(1, N-1):
    z1x, z1y = dx[i-1], dy[i-1]
    z2x, z2y = dx[i], dy[i]
    a1 = math.atan2(z1y, z1x)
    a2 = math.atan2(z2y, z2x)
    d = a2 - a1
    if d < 0:
        d += 2 * math.pi
    turn[i] = d

pref_turn = [0.0] * N
for i in range(1, N):
    pref_turn[i] = pref_turn[i-1] + turn[i]

def solve(Li):
    # binary search on farthest reachable vertex
    lo, hi = 0, N-1
    while lo < hi:
        mid = (lo + hi + 1) // 2
        if dist[mid] <= Li:
            lo = mid
        else:
            hi = mid - 1
    k = lo

    r = Li - dist[k]

    # sector-like accumulation
    area = 0.0
    area += 0.5 * pref_turn[k] * (Li ** 2)

    # remaining part
    if k < N - 1:
        area += 0.5 * (r ** 2) * (2 * math.pi - pref_turn[k])

    return area

out = []
for x in L:
    out.append(str(solve(x)))

print("\n".join(out))
```该实现首先沿着多边形边界构建前缀距离。 这使得能够快速二分搜索绳子可以沿着边缘“行走”多远。 转动角度的计算方法为`atan2`，避免不稳定的反余弦计算。 

每个查询都使用二分搜索来定位最后一个完全可达的顶点。 剩余的绳索长度定义了该顶点之外还有多少可用的附加圆形区域。 最终的面积表达式被分为前缀角度贡献和剩余扇区贡献。 

一个微妙的部分是角度标准化：每个边缘到边缘的转弯都被迫进入$[0, 2\pi)$。 如果没有这种校正，负角度差将破坏前缀积累。

 ## 边缘情况

 当绳索长度足够大以穿过整个多边形边界时，就会出现关键的边缘情况。 在这种情况下，二分查找返回最后一个顶点，剩余的绳子贡献了一个完整的圆形扇区。 该算法仍然有效，因为前缀角和变为$2\pi$，剩余项变为零。 

当绳索几乎到达第一个边缘时，会出现另一种边缘情况。 然后$k = 0$，前缀和为零，仅使用部分扇区公式。 该实现自然地处理了这个问题，因为前缀数组在索引 0 处用零初始化。 

第三种情况是角度环绕时的数值稳定性$2\pi$。 如果没有标准化，负角度差将错误地累积。 这`if d < 0: d += 2*pi`校正可确保前缀保持单调，因此即使是长多边形也不会产生漂移或符号错误。
