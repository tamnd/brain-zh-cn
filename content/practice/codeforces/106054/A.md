 ## 解决方案
 ## 问题理解

 我们得到一个固定长度为 10 的字符串，仅由大写拉丁字母组成。 我们可以从这个字符串中删除字符，但不允许重新排列剩下的字符。 问题是删除一些字符后能否得到子序列“TAP”。 

所以任务不是在任何地方独立地找到“T”、“A”和“P”。 它们必须按此顺序出现在原始字符串中，中间可能有其他字母，并且我们可以忽略这些额外的字母。 

约束非常小，因为字符串长度始终恰好是十。 这立即告诉我们，即使是二次或三次方法也太过分了。 线性扫描就足够了，甚至对子序列进行暴力破解在性能方面也是可以接受的。 

有一些边缘情况值得明确指出。 包含所有三个字母但顺序错误的字符串应该会失败。 例如，“PATXXXTXXX”包含所有字符，但顺序不正确，因此无效。 另一种情况是，所需字母之一完全丢失，例如“CONTRACETO”，它应该立即失败，因为没有“P”。 最后，重复的字母可能会产生误导。 像“TAAAPPPPPP”这样的字符串是有效的，因为我们可以选择第一个有效的“T”，然后选择后面的“A”，然后选择后面的“P”，保持顺序。 

## 方法

 思考这个问题的一种强力方法是考虑每个长度为三的子序列并检查它是否等于“TAP”。 由于字符串的长度为 10，因此最多有 C(10, 3) = 120 个这样的子序列，并且检查每个子序列的时间为常数。 这在一定范围内已经可以正常工作并且很容易推理。 

然而，这种方法显式地枚举了组合，这是不必要的。 问题的结构只问是否可以按顺序找到“T”，然后是“A”，然后是“P”。 一旦我们认识到顺序是唯一的约束，我们就可以将问题简化为单遍贪婪扫描。 

关键的观察是我们只需要通过固定的模式来跟踪进度。 我们开始寻找“T”。 一旦找到它，我们就不再关心前面的字符并开始寻找“A”。 一旦找到“A”，我们就寻找“P”。 如果我们设法到达模式的终点，答案是肯定的。 

这是有效的，因为模式的每个字符都必须按顺序匹配，并且向前跳过永远不会阻止有效的未来匹配。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解子序列 | O(120) | O(1) | O(1) | 已接受 |
 | 贪心扫描 | O(10) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 初始化一个指针`i = 0`在目标字符串“TAP”上，这意味着我们当前正在尝试匹配第一个不匹配的字符。 
2. 迭代每个字符`c`在输入字符串中从左到右。 
3.如果`c`等于`TAP[i]`， 进步`i`一个。 这意味着我们已经成功地按顺序匹配了下一个所需的字符。 
4. 如果在任何时候`i`变成3，我们已经匹配了“TAP”的所有字符，所以我们可以提前停止。 
5. 扫描完整个字符串后（或提前停止），检查是否`i == 3`。 如果是，则输出“S”，否则输出“N”。 

只在比赛中晋级的原因是我们必须维持秩序。 跳过不匹配的字符可确保我们有效地搜索子序列而不是子字符串。 

### 为什么它有效

 该算法保持不变式，即在处理输入中的每个字符之前，`i`表示可以使用输入字符串的已处理前缀的子序列形成的“TAP”的最长前缀的长度。 每次我们匹配下一个需要的字符时，我们都会将该前缀扩展一。 因为我们只在输入和模式中前进，所以我们永远不会使之前的选择无效，并且任何有效的子序列都必须对应于某些成功前进的序列`i`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
target = "TAP"

i = 0
for c in s:
    if i < 3 and c == target[i]:
        i += 1
        if i == 3:
            break

print("S" if i == 3 else "N")
```该代码读取单个输入字符串并维护一个指针`i`进入模式“TAP”。 每个字符都会根据当前所需的字符进行检查。 一旦所有三个字符都匹配，我们就会提前终止，因为进一步的扫描无法改变结果。 最终条件检查完整模式是否匹配。 

一个微妙的点是提前退出。 虽然对于正确性来说没有必要，但它保证扫描一旦成功就会停止，尽管长度为 10 这主要是风格上的。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    s = sys.stdin.readline().strip()

    for c in s:

 ## 边缘情况

 一种微妙的情况是所有必需的字母都存在但交错不正确。 对于输入“PTAATPXABCD”，扫描首先找到“T”，但其后没有有效的“A”，从而导致正确进程中的“P”。 指针在第二阶段卡住，算法正确地拒绝了它。 

另一种情况是大量重复。 对于“TTTAAAAPPPP”，算法不会尝试选择最佳出现，它只是采用第一个有效的级数：第一个 T，之后的第一个 A，之后的第一个 P。 这保证了正确性，因为任何有效的子序列仍然允许至少一个单调前进通过该模式。
