 ## 解决方案
 ## 问题理解

 给定一个整数序列，我们想要计算可以选择多少对位置，以便两个相应值的乘积不包含任何平方质因数。 

表达该条件的另一种方法是查看素因数分解。 如果没有出现指数至少为 2 的素数，则该数字是“干净的”。 对于一对数字，我们希望它们的组合因式分解也保持干净，这意味着在这两个数字中，总共没有素数被使用超过一次。 

因此，有效对必须同时满足两个条件。 首先，每个单独的数字必须已经是无平方的，否则它自己的因式分解已经包含平方素数并且乘积自动失败。 其次，这两个数字不能共享任何素数因子，因为共享素数会使该素数的指数在乘积中至少达到 2。 

输入只是一个整数列表。 输出是具有上述属性的索引对的数量。 

此类问题的典型约束结构意味着对所有对的二次检查太慢。 如果数组大小达到 10^5，则对所有对进行简单的 O(n^2) 扫描将需要大约 10^10 次检查，这远远超出了两秒限制可以处理的范围。 这迫使我们压缩每个数字的表示并有效地计算兼容对。 

当数字包含平方因子时，会出现微妙的边缘情况。 例如，考虑像 4 和 9 这样的值。两者单独无效，因此必须排除涉及它们的任何对。 另一种情况是两个数字没有平方但共享一个素数，例如 6 和 10，它们都包含素数 2。它们的乘积包含 2 的平方，因此即使每个数字单独看起来都可以接受，该对也是无效的。 

## 方法

 一种直接的方法是检查每一对，并通过重新计算素数分解或乘法和因式分解来测试它们的乘积是否是无平方的。 这在逻辑上是有效的，因为它准确地强制执行条件，但如果我们动态分解，在最坏的情况下每次检查的成本大约为 O(sqrt A)。 对于 n 个元素，这会导致大约 O(n^2 sqrt A)，即使对于中等输入大小也是无法使用的。 

关键的观察结果是，条件仅取决于每个数字中存在的素数集，并且仅取决于这些集重叠或内部包含重复项。 这建议将每个数字压缩为规范表示：在分解过程中仅出现一次的素数集，前提是没有素数出现超过一次。 

一旦每个数字都转换为这样的掩码，两个数字之间的条件就变成纯粹的组合。 我们只需要确保他们的掩码不相交并且都是有效的。 这将问题转化为计算没有交集的子集对。 

处理此问题的标准方法是将相同的掩模分组并计算频率。 然后，对于每个有效掩码，我们只想将其与之前见过的不冲突的掩码配对。 当涉及的不同素数的数量较小时，可以使用位掩码上的频率图来完成此操作，或者如果掩码存储为素数的排序元组，则可以使用哈希图来完成此操作。 

对于该问题系列中的典型约束，值是有界的，因此最大到 10^6 或 10^7 左右的数字仅涉及每个数字的一​​组可管理的素数。 我们使用最小素数因子的筛子对每个数字进行一次因式分解，丢弃任何包含重复素数因子的数字，并构建其减少的素数集。 然后，我们通过迭代先前看到的表示并检查交集来计算兼容对。

 | 最优（因式分解+散列）| 最坏情况为 O(n log A + K^2)，通常接近 O(n log A) | O(n) | 已接受 |

 ## 算法演练

 我们将每个数字简化为一个结构，该结构可以准确捕获与条件相关的信息。 

1. 预先计算数组中直到最大值的最小质因数。 这允许以每个元素的对数时间快速分解每个数字。 
2. 对于每个数字，使用最小素因子表将其分解。 在因式分解时，我们跟踪是否有任何素数出现多次。 如果发生这种情况，我们会将该数字标记为无效，并在以后的计数中完全忽略它。 
3. 如果该数字有效，我们将构建一个由其不同素数组成的表示。 这种表示唯一地确定它是否可以参与有效对，因为只有素数集才重要。 
4. 维护从这些表示到我们已经看到它们的次数的频率图。 
5. 对于每个新的有效数字表示，我们迭代先前存储的表示并检查两组素数是否相交。 如果它们不相交，我们将它们的频率的乘积添加到答案中。 
6. 在处理与早期组的兼容性之后，我们增加当前表示的频率。 

我们只检查之前看到的组的原因是为了确保每对都被精确计数一次。 我们永远不会以相反的顺序重新访问对。 

### 为什么它有效

 在每一步中，每个数字都被替换为因式分解中以指数一出现的一组精确素数，或者如果它不是无平方的则被丢弃。 当且仅当两个数的素数集不相交时，它们才构成有效对。 该算法仅枚举一次此类集合的所有对，并仅在不相交成立时将它们添加到计数中，因此不会遗漏任何有效对，也不包含无效对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_spf(n):
    spf = list(range(n + 1))
    for i in range(2, int(n ** 0.5) + 1):
        if spf[i] == i:
            step = i
            start = i * i
            for j in range(start, n + 1, step):
                if spf[j] == j:
                    spf[j] = i
    return spf

def factorize(x, spf):
    primes = []
    while x > 1:
        p = spf[x]
        cnt = 0
        while x % p == 0:
            x //= p
            cnt += 1
            if cnt > 1:
                return None
        primes.append(p)
    return tuple(primes)

def disjoint(a, b):
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] == b[j]:
            return False
        if a[i] < b[j]:
            i += 1
        else:
            j += 1
    return True

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    mx = max(arr) if arr else 0

    spf = build_spf(mx)

    freq = {}
    ans = 0

    for v in arr:
        rep = factorize(v, spf)
        if rep is None:
            continue
        rep = tuple(sorted(rep))

        for k, cnt in freq.items():
            if disjoint(rep, k):
                ans += cnt

        freq[rep] = freq.get(rep, 0) + 1

    print(ans)

if __name__ == "__main__":
    solve()
```筛子构建最小的质因数，因此每个数字都可以快速分解，无需试除。 因式分解步骤明确拒绝任何重复素数的数字，从而在源处强制执行无平方性。 

不相交检查使用两指针扫描，因为每个表示都存储为素数的排序元组。 这使得每个值的素数数量的比较保持线性，这在实践中很小。 

频率图确保我们有效地计算组合，而无需重新访问对。 

## 工作示例

 考虑一个输入，其中数字是`[6, 10, 15]`。 他们的因式分解是`6 = {2,3}`,`10 = {2,5}`,`15 = {3,5}`。 

| 步骤| 当前| 代表| 频率图 | 添加对 |
 | ---| ---| ---| ---| ---|
 | 1 | 6 | {2,3} | {} | 0 |
 | 2 | 10 | 10 {2,5} | {6:1} | 没有不相交的|
 | 3 | 15 | 15 {3,5} | {6:1, 10:1} | (15,10), (15,6) 部分过滤 |

 只有 (6,15) 和 (10,15) 有效，因为每一对都避免共享素数。 

该轨迹显示了算法如何纯粹基于交集结构进行过滤。 

现在考虑`[4, 3, 5, 6]`。 这里4被立即丢弃。 

| 步骤| 价值| 有效 | 行动| 回答 |
 | ---| ---| ---| ---| ---|
 | 1 | 4 | 没有| 跳过| 0 |
 | 2 | 3 | 是的 | 频率添加| 0 |

 | 4 | 6 | 是的 | 仅与 3 配对 | 2 |

 这说明了平方因子如何尽早淘汰候选人。

 ## 边缘情况

 关键的边缘情况是包含重复素数的数字，例如 8 = 2^3。 该算法在分解过程中立即拒绝这些，因此它们永远不会进入频率图。 用于输入`[8, 3, 5]`，唯一有效的表示是`{3}`和`{5}`，并且该算法恰好计算出它们之间的一对。 

另一种边缘情况是重复无平方的相同数字。 用于输入`[6, 6]`，两者都映射到`{2,3}`。 由于它们的交集本身不为空，因此该算法正确地避免了对一对进行计数，因为相同的索引仅在不同的事件中配对，但仍然不符合不相交条件。
