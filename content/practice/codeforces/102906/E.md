 ## 解决方案
 ## 问题理解

 我们得到了一系列数字，但我们不应该将它们视为单独的值，而应该将它们视为一个多重集，允许我们重复执行非常具体的转换。 任务是确定是否可以仅使用这些转换将一个给定数字转换为另一个给定数字。 

操作很简单：我们可以将固定向量C相加任意次数，也可以将当前向量顺时针旋转90度任意次数。 从向量 A 开始，我们想知道是否可以到达向量 B。 

从几何角度来说，这意味着我们正在从 A 开始的平面上行走。在每一步，我们要么移动 C 要么移动 C 的旋转版本，并且我们可以在添加 C 之前保持旋转 C。 问题是 B 是否位于通过这些移动的任何序列均可到达的所有点的集合中。 

输入由三个 2D 向量 A、B 和 C 组成。输出是一个单一决策：B 是否可以表示为 A 加上通过旋转和添加 C 生成的一些有限运算序列。 

这些约束允许坐标的绝对值高达 10^8，这立即表明我们必须避免对操作序列进行任何强力模拟。 任何一步步枚举序列或探索状态的方法都会呈指数级爆炸，因为每个步骤都会分支成多种可能性（旋转或不旋转、添加或不添加）。 我们需要 O(1) 或至多 O(log value) 代数表征。 

当 C 为零向量时，会出现微妙的边缘情况。 在这种情况下，旋转不会执行任何操作，并且每个操作都是相同的，因此唯一可到达的点是 A 本身。 例如，如果 A = (1, 2)、B = (1, 2)、C = (0, 0)，则答案为“是”，但如果 B 不同，则答案为“否”。 任何代数方法都必须明确处理这种简并性，否则就有除以零或假设可逆性的风险。 

当 C 位于轴上或共线旋转时，会出现另一种极端情况。 例如，如果 C = (1, 0)，则其旋转产生 (0, 1)、(-1, 0)、(0, -1)，跨越整个整数网格。 相反，如果 C = (0, 0)，则旋转完全崩溃。 这两个极端的行为根本不同，因此解决方案必须在推理可达性之前对 C 进行分类。 

## 方法

 天真的想法是模拟从 A 开始的所有可能的操作序列。每个步骤都允许添加 C 或添加其旋转版本之一。 在 k 步骤之后，我们将探索大约 4^k 种可能性，因为旋转和加法选择复合。 即使我们修剪重复的状态，可达集也会在没有结构的情况下增长。 最坏的情况会达到半径不断增长的所有整数格点，使得这种方法在 k 甚至达到 50 之前就变得不可行。 

关键的观察是顺序并不重要。 每个操作只是添加一个向量，该向量是 C 的四个旋转之一。因此，整个过程简化为构造有限向量集的整数线性组合：C、R(C)、R²(C) 和 R³(C)，其中 R 是 90 度旋转。 

这将问题转化为检查向量 B − A 是否位于这四个向量的整数范围内。 但这些向量并不是独立的。 事实上，在二维中，向量的四个旋转版本最多生成一个二维晶格，并且它们的跨度折叠成可以用两个基向量描述的结构。 

更清晰的理解方法是处理 C = (x, y)。 那么它的旋转是（x，y），（−y，x），（−x，−y）和（y，−x）。 任何可达向量都是具有非负系数的这些向量的总和，但由于我们可以按任何顺序应用操作，因此我们可以任意重新排列它们。 这将问题转变为检查整数线性系统是否有解。

关键的简化是可达集形成由两个向量生成的网格：(x, y) 和 (−y, x)。 如果我们可以将 B − A 表示为这两者的线性组合，那么答案是肯定的。 

这将问题简化为求解 2×2 整数系统，这可以通过检查行列式一致性并确保整数解存在来完成。

 ## 算法演练

 1. 计算位移向量 D = B − A。我们只关心相对运动，因为所有操作都是通过平移作用于 A 的。 
2. 处理C = (0, 0) 的特殊情况。 在这种情况下，根本不可能移动，因此仅当 D = (0, 0) 时答案为“是”，否则为“否”。 
3. 设 C = (x, y)。 构造两个基向量 v1 = (x, y) 和 v2 = (−y, x)。 它们对应于平面中两个独立的旋转方向。 
4. 检查是否存在整数 a 和 b 使得：

 D = a * v1 + b * v2。 

展开坐标给出系统：

 Dx = a x − b y

 Dy = a y + b x
 5. 使用行列式推理求解该系统。 计算 det = x² + y²。 如果 det 为零，则意味着 C 为零并且已被处理。 否则，计算：

 a = (Dx x + Dy y) / det

 b = (Dy x − Dx y) / det
 6. 验证a 和b 是否均为整数。 如果其中一个不是整数，则变换是不可能的。 如果两者都是整数，则答案为“是”。 

原理：每个允许的操作都相当于将 C 的四个旋转副本之一相加，这四个向量生成一个格子，其基可以作为 C 及其 90 度旋转。 任何操作序列都会分解为这些基向量的整数组合，因此可达性正是该格中的成员资格。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    ax, ay = map(int, input().split())
    bx, by = map(int, input().split())
    cx, cy = map(int, input().split())

    dx = bx - ax
    dy = by - ay

    if cx == 0 and cy == 0:
        print("YES" if dx == 0 and dy == 0 else "NO")
        return

    det = cx * cx + cy * cy

    a_num = dx * cx + dy * cy
    b_num = dy * cx - dx * cy

    if a_num % det != 0 or b_num % det != 0:
        print("NO")
        return

    print("YES")

solve()
```该代码首先将问题简化为纯粹的位移检查，消除了对起点的依赖。 立即处理简并零向量情况以避免无效除法。 

行列式`cx*cx + cy*cy`之所以出现，是因为它是向量 C 的长度平方，并且在投影到旋转基础上时用作归一化因子。 的表达式为`a_num`和`b_num`来自使用 C 及其垂直向量形成的旋转矩阵的逆来求解 2×2 线性系统。 

一个常见的实现陷阱是忘记我们实际上不需要计算浮点值。 一切都必须保持整数，整除性检查是验证可行性的唯一可靠方法。 

## 工作示例

 ### 示例 1

 输入：

 A = (0, 0)，B = (1, 1)，C = (0, 1)

 我们计算 D = (1, 1)。 由于 C 不为零，因此 det = 1。 

现在：

 a_num = 1·0 + 1·1 = 1

 b_num = 1·0 − 1·1 = −1

 两者均除以 det = 1，得到 a = 1，b = −1，所以答案是 YES。 

这证实了即使系数在解释中为负，系统也允许通过不同的操作序列进行取消。 

### 示例 2

 输入：

 A = (0, 0)，B = (2, 2)，C = (1, 1)

 我们计算 D = (2, 2), det = 2。 

a_num = 2·1 + 2·1 = 4

 b_num = 2·1 − 2·1 = 0

 我们得到 a = 2，b = 0，所以是。 

这演示了只需要一个方向的情况； 单独重复添加 C 就足够了。

 ## 边缘情况

 当 C 为 (0, 0) 时，每次旋转都会折叠到同一点，因此可达集仅包含 A。该算法在任何代数之前显式检查这一点，确保不会发生无效除法并保证所有零移动场景的正确性。 

当 C 非常大或具有很大坐标时，像 cx * cx 或 dx * cx 这样的中间产品可能会超过 32 位限制。 使用 Python 可以完全避免溢出，但在 C++ 中这需要 64 位整数。 无论大小如何，使用基于行列式的整除性的公式仍然有效，因为它仅取决于精确的算术。
