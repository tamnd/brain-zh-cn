 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的字符串。 任务是构造无法作为给定字符串的子序列获得的最短可能字符串。 

这里的子序列意味着我们可以从原始字符串中删除字符而不改变剩余字符的顺序。 目标是在同一字母表中找到一个不符合此属性的字符串，并且在所有此类字符串中，我们需要最小可能的长度。 

更具体地说，将原始字符串想象为具有位置的字母序列。 如果我们可以从左到右挑选匹配的字符，并在原始字符串中向前移动，则候选答案字符串是有效的。 我们被要求找到一个不可能执行此过程的字符串，并且我们想要最短的此类字符串。 

约束结构通常允许字符串长度达到 10^5 左右，这排除了任何尝试枚举所有可能候选字符串的解决方案。 即使检查子序列的有效性也是 O(n)，因此暴力破解所有长度为 k 的字符串将会爆炸为 26^k 个候选者。 这立即表明我们必须以动态或贪婪的方式对字符串中的位置进行推理，而不是对构造的字符串进行推理。 

当字母表中的每个字符都出现在字符串中时，就会出现微妙的边缘情况。 例如，如果字符串是`"abcabc"`，那么每个字符都是一个子序列，但也有一些较长的字符串，例如`"aa"`或者`"zzz"`可能是也可能不是子序列，具体取决于重复和顺序。 另一种边缘情况是某些字符根本不会出现，例如`"abac"`， 在哪里`"z"`立即是长度为 1 的有效答案。假设我们总是需要多个字符的粗心方法将在这里失败。 

## 方法

 蛮力的想法很简单。 我们尝试构造长度为 1 的所有字符串，然后构造长度为 2 的字符串，依此类推，对于每个候选字符串，我们检查它是否是给定字符串的子序列。 子序列检查与字符串长度成线性关系，因此对于每个候选者，我们要付出 O(n) 的代价。 候选者的数量随着长度呈指数级增长，因此在最坏的情况下，我们在找到答案之前会探索多达 26^k 种可能性。 即使对于较小的 k，这也很快变得不可行。 

关键的观察是我们不需要显式构造候选字符串。 相反，我们可以问一个补充问题：从字符串中的任何位置开始，我们无法匹配的最短字符串是什么。 如果我们知道对于每个后缀，最短的不可能子序列，我们可以通过动态规划构建整个字符串的答案。 

这项工作的结构是子序列匹配的单调过程。 一旦我们修复了第一个字符，剩下的问题就仅限于匹配后字符串的后缀。 这会产生重叠的子问题：不同的路径通常会落在相同的后缀位置上。 这表明 DP 高于位置，其中状态代表我们从索引 i 开始可以做的最好的事情。 

我们定义一个 DP，其中 dp[i] 是最短字符串的长度，该字符串不是从位置 i 开始的后缀的子序列。 从位置 i 开始，如果某个字符 c 根本没有出现在后缀中，那么答案就是 1，因为字符串“c”已经立即失败。 否则，后缀中的每个字符都可用，因此任何单字母字符串都是有效的，并且我们必须考虑更长的结构。 我们尝试所有字符，跳转到下一个出现的字符，然后添加一个步骤。 

我们可以使用标准的 next 数组预先计算下一个出现位置，这允许 O(1) 内的转换。 这将问题减少到 O(n * 26)。

 | 方法| 时间复杂度| 空间复杂度| 判决|
 | ---| ---| ---| ---|
 | 蛮力 | O(26^k·n) | O(26^k·n) | O(1) | O(1) | 太慢了 |
 | 后缀上的最优 DP | O(n·26) | O(n·26) | 已接受 |

 ## 算法演练

 我们从右到左处理字符串，同时维护有关每个字符接下来出现的位置的转换信息。 

1. 我们构建一个下一次出现表 next[i][c]，它存储 i 处或 i 之后字符 c 出现的第一个位置，如果没有出现，则存储哨兵值。 这使我们能够模拟子序列匹配，而无需每次都扫描字符串。 这是必要的原因是子序列转换仅取决于下一个有效位置，而不取决于中间字符。 
2. 我们将 dp[i] 定义为从位置 i 开始不能形成子序列的最短字符串的长度。 直觉是 dp[i] 衡量“转义”从 i 开始的后缀的难度。 
3. 如果我们在位置 i 处，存在一个字符 c，使得 next[i][c] 无效，则 dp[i] 变为 1。这是因为我们可以立即选择 c，从而使第一步的子序列匹配失败。 
4. 否则，每个字符都存在于后缀中的某个位置。 在这种情况下，对于每个字符 c，我们移动到 next[i][c] + 1 并将答案加一。 我们对所有字符取最小值。 这反映了尝试通过选择第一个字母然后从下一个状态继续最佳地构建最短的不可能字符串。 
5. 我们从字符串末尾向后计算 dp，以便在需要时已知所有转换。 答案是dp[0]。 

### 为什么它有效

DP 依赖于 dp[i] 正确表示不能匹配为从后缀 i 开始的子序列的字符串的最小长度这一不变量。 每个转换完全对应于固定候选字符串的下一个字符并前进到下一个可行的匹配位置。 由于子序列匹配本质上是贪婪的，总是采用最早的可能匹配，因此下一个表完全捕获所有有效的延续。 这确保不会遗漏较短的无效字符串，因为任何短于 dp[i] 的候选字符串都将对应于通过下一个转换的有效路径，这与 dp[i] 的定义相矛盾。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)
    alphabet = 26
    A = ord('a')

    next_pos = [[n] * alphabet for _ in range(n + 1)]

    last = [n] * alphabet

    for i in range(n - 1, -1, -1):
        last[ord(s[i]) - A] = i
        for c in range(alphabet):
            next_pos[i][c] = last[c]

    dp = [0] * (n + 1)
    dp[n] = 1

    for i in range(n - 1, -1, -1):
        best = float('inf')
        for c in range(alphabet):
            j = next_pos[i][c]
            if j == n:
                best = 1
                break
            best = min(best, 1 + dp[j + 1])
        dp[i] = best

    print(dp[0])

if __name__ == "__main__":
    solve()
```该实现首先构造一个下一个出现表，以便可以在恒定时间内回答每个字符转换。 这避免了在检查后续可行性时重复扫描。 

dp 数组是向后计算的，因此当我们评估状态 i 时，所有状态 j > i 都是已知的。 关键的微妙之处在于处理后缀中缺少字符的情况。 在这种情况下，我们立即返回 1，因为单字符字符串已经失败。 

另一个细节是哨兵值n，它代表“未找到”。 我们始终将 j = n 视为失败状态，这简化了边界处理。 

## 工作示例

 考虑字符串`abac`。 

我们从末尾开始计算转换。 

| 我| dp[i] 推理 |
 | ---| ---|
 | 4 | 空后缀，任何字符立即失败，dp[4] = 1 |
 | 3 (“c”) | 从这里开始出现a、b、c 中的任何一个吗？ i 处仅存在 c，因此缺少 a 或 b 会导致 dp[3] = 1 |

 结果显示单个字母如`"z"`就足够了，因为它从未出现在字符串中。 

现在考虑`abcabc`。 

这里每个字符都出现在每个合理的后缀中，因此没有一个字母会失败。 DP 迫使我们考虑更长的结构。 该算法将探索转换并最终确定在某个深度无法匹配重复结构，从而产生大于 1 的答案。 

| 我| dp[i] | dp[i] |
 | ---| ---|
 | 6 | 1 |
 | 5 | 1 |

 跟踪证实，只有当所有字符都出现在各处时，我们才需要扩展构造的字符串。

 ## 边缘情况

 对于像这样的字符串`"aaaa"`，位置 0 处的 DP 立即检测到除`"a"`后缀中缺少。 转换规则设置 dp[0] = 1，因为选择`"b"`在第一步就已经失败了。 这表明该算法可以正确处理高度重复的字符串，而无需更深层次的递归。 

对于像这样的字符串`"abcdefghijklmnopqrstuvwxyz"`，每个后缀仍然包含所有字母。 DP 永远不会触发立即失效条件，因此它会探索更长的构造。 在每个状态下，转换总是成功，并且算法通过重复的后缀跳转正确构建最小不可能的字符串长度，确认该结构正确处理最坏情况的全字母表覆盖。
