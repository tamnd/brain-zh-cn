 ## 解决方案
 ## 问题理解

 我们得到了多组小写字母。 我们可以将这些字母重新排列成我们想要的任何顺序，形成一个相同长度的新字符串。 

对于任何结果字符串，我们查看每个连续的子字符串并检查它是否是回文。 每个子字符串都按重数进行计数，这意味着如果相同的字符序列出现在不同的位置，则它会贡献多次。 

任务是对字母进行排列，以使回文子串的总数尽可能大。 

约束 n 高达 100000 意味着任何解决方案都必须以线性或接近线性的时间运行。 任何涉及显式检查所有子字符串的操作都是不可能的，因为存在 O(n²) 个子字符串，甚至在最坏的情况下检查每个子字符串的回文性也会将其推至 O(n³)。 即使像 Manacher 这样更高效的子串回文技术也只能帮助评估固定字符串，而不能优化所有排列，因此关键的困难在于结构而不是计算。 

一种天真的方法会尝试所有排列甚至贪婪的局部交换来改善回文结构。 两者都失败了，因为排列的数量是阶乘的，并且局部优化不能捕获回文密度的全局结构。 

当字符串具有许多不同的字符时，会出现微妙的边缘情况。 例如，如果所有字符都不同，则每种排列都恰好有 n 个回文子串，因为只有单个字符是回文。 在这种情况下，任何尝试“构建对称性”的方法都不会改善答案，并且错误地偏向对称性甚至可能会降低混合分布的灵活性。 

另一个边缘情况是所有字符都相同。 那么每个子串都是回文，任何排列都是最优的。 正确的解决方案不得使这种情况过于复杂，并且仍必须产生有效的重新排列。 

## 方法

 关键的观察是回文子串很大程度上依赖于重复结构。 如果子串的端点匹配且内部也是对称的，则该子串是回文。 当字符串包含大量相同字符和允许许多对称中心的重复模式时，回文子串的数量会增加。 

蛮力的想法是尝试多重集的每个排列并计算每个排列的回文子串的数量。 对于固定字符串，我们可以使用线性时间回文计数方法，例如 Manacher 算法。 然而，生成所有排列是不可行的，因为最多有 n! 即使 n = 20，这也会变得天文数字般大。 

相反，我们需要推理什么结构可以最大化回文子串。 一个关键的简化是，完全位于相同字符块内的子串始终是回文子串，并且它们的贡献呈二次方：长度为 k 的块本身贡献 k(k+1)/2 个回文子串。 除非结构高度对称，否则跨边界子串很难形成回文，这很难用任意多重集来实现。 

这表明最大化相同字符的长连续块是有益的。 为了最大化总贡献，我们希望尽可能地将相同的角色放置在一起，理想情况下每种角色类型形成一个大块。 由于我们可以自由排列，所以我们能做的最好的就是将所有相同的字母组合在一起。 

因此，最佳结构是以任何顺序排列字符，其中相同的字符是连续的。 这些块的任何排序都会产生每个块相同的内部回文贡献，并且跨块回文的性能不会优于最大化块大小所带来的收益。

因此，我们只需将每个字符的所有出现次数组合在一起输出即可。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（所有排列）| O(n!·n²) | O(n!·n²) | O(n) | 太慢了 |
 | 最佳（按频率分组）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 统计输入字符串中每个字符的出现频率。 这是必要的，因为最终的排列仅取决于多样性，而不取决于原始顺序。 
2. 通过按字母顺序迭代字符并附加按其频率重复的每个字符来构建结果字符串。 这确保每个字符形成一个连续的块。 
3. 输出构造好的字符串。 

### 为什么它有效

 关键属性是，当该块尽可能大时，相同字符的连续块内的回文子串的贡献最大化。 将相同的字符拆分到多个位置可以二次减少内部回文子串的数量，因为 k(k+1)/2 在 k 中是严格凸的。 将相同字符重新分配到多个块中会减少这些贡献的总和。 

跨字符回文无法弥补一般多重集中的这种损失，因为它们需要不同字母之间严格的对称约束，而这无法在任意频率分布下进行全局优化。 因此，集中相同的字符可以保留所有最大的局部贡献并产生最佳的全局配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    freq = [0] * 26
    for ch in s:
        freq[ord(ch) - 97] += 1

    res = []
    for i in range(26):
        if freq[i]:
            res.append(chr(i + 97) * freq[i])

    print("".join(res))

if __name__ == "__main__":
    solve()
```该代码首先计算所有小写字母的频率表。 然后，它通过根据其计数重复排序的顺序连接每个字符来构造答案。 最佳解决方案中字符的顺序是任意的，因此字典顺序用于确定性。 

除了仔细读取输入之外，不需要边界处理，因为 n ≥ 1 保证至少存在一个字符。

 ## 边缘情况

 对于“a”这样的单字符串，算法统计频率{a:1}，直接输出“a”。 回文子串计数为 1，这是最优且不可避免的。 

对于像“aaaa”这样的字符串，频率分组会生成“aaaa”。 每个子串都是一个回文，分组保留了这种最大结构。 该算法输出相同的字符串而不进行修改。 

对于像“abc”这样的字符串，每种排列都会产生正好三个回文子字符串，每个字符一个。 算法输出“abc”，该值有效且符合最优值。 

对于像“aabccc”这样的混合大小写，分组会产生“aabccc”。 任何排列都具有相同的回文单字符子串总数，并且分组不会减少此计数，同时保留所有可用结构。
