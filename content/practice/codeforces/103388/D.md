 ## 解决方案
 ## 问题理解

 我们拥有一个由无向道路连接的城市网络，其中每条道路都有代表其“美丽”的正权重。 任务是将这组城市分为两组。 一旦选择了分割，任何端点位于不同组中的道路都会被破坏，并且每个组只保留完全包含在其中的道路。 

对于每个结果组，我们将其“美丽”定义为该组内所有道路中的最大边缘权重。 如果一个群体根本没有内部道路，那么它的美感就被定义为零。 目标是将城市划分为两个不相交的集合，以使两个组具有完全相同的美丽值。 我们必须确定存在这种有效分区的所有可能的美感值。 

输入规模很大，最多有5×10^5个城市和5×10^5条道路。 这立即排除了任何尝试直接测试分区的行为，因为即使检查单个分区也是 O(N + M)，并且分区的数量是指数级的。 任何解必须接近线性或对数线性，通常为 O(M log M) 或 O(M α(N))。 

一些边缘情况在结构上很重要。 

如果没有道路，则每个分区在两个部分中产生的美感都是零，因此唯一的答案是 0。 

如果图非常密集或具有许多高权重边，则关于分区的简单推理可能会失败，因为“最大内部边”取决于连接模式而不是全局排序。 

当所有边如此相互关联以至于任何分区都会迫使两侧继承不同最大权重的边时，就会出现一种微妙的情况。 例如，如果图是具有不同权重的完整图，则可能根本不可能均衡最大值，从而导致 IMPOSSIBLE。 

## 方法

 一个蛮力的想法是将城市的所有分区枚举为两组，并为每个分区计算每条边内的最大边。 即使对于单个分区，我们也会扫描所有边缘以计算两个最大值。 这已经花费了 O(M)。 由于有 2^N 个分区，因此除了非常小的 N 之外，这是完全不可行的。 

为了取得进展，我们将视角从分区转向边缘引起的约束。 假设我们固定一个候选美丽值X。我们询问是否存在一个分区，其中两侧的最大内部边缘恰好为X。 

此条件在边缘上施加了强大的结构：

 任何权重大于 X 的边都不能完全位于任一组内，否则该组的美观度将超过 X。因此，必须切割每条权重大于 X 的边，这意味着它的端点必须位于不同的组中。 换句话说，权重大于 X 的所有边都强制执行二分约束。 

这立即表明由严格大于 X 的边形成的子图上的图着色条件。如果该图不是二分图，则该 X 不存在有效分区。 

现在，我们还需要至少一条权重恰好为 X 的边完全出现在其中一个组内，否则两个组的美度都将严格小于 X。这意味着在每个连接的组件内（相对于边 > X），我们需要至少一条 X 边，该边不会因二分性约束而被迫穿过切割。 

这将问题转化为检查阈值图上的二分性并验证是否存在可以放置在边内而不违反约束的“安全”X 边。 

释放效率的关键观察是我们只需要考虑实际边权重的 X 值。 在两个连续权重之间，“> X”图的结构或 X 边的可用性没有任何变化。 因此，我们按权重对边进行排序，并按降序处理它们，使用具有奇偶校验或二分着色的 DSU 来维护约束的动态结构。

当我们从大到小权重时，我们逐渐添加边缘作为“主动约束”。 当我们达到权重级别 w 时，我们可以通过检查添加所有边 > w 是否已经引起矛盾，以及是否存在至少一个权重 w 的边可以放置在一致的着色内来确定 w 是否可行。 

这将问题转变为动态维护二分结构，同时跟踪哪些权重仍然是可行的候选者。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解分区 | O(2^N·M) | O(2^N·M) | O(N + M) | 太慢了 |
 | 排序 + DSU 奇偶校验扫描权重 | O(M log M) | O(M log M) | O(N + M) | 已接受 |

 ## 算法演练

 1. 按权重降序对所有边进行排序，以便我们可以从最强（最大权重）到最弱（最小权重）处理约束。 这确保了当我们评估候选值 X 时，所有权重大于 X 的边都已经插入到结构中。 
2. 维护一个用奇偶校验信息增强的不相交集并集结构。 每个节点存储它相对于其父节点是否具有相同的颜色或相反的颜色，这允许我们增量地表示二分区约束。 
3. 初始化没有边缘的DSU。 此时，每个节点都是独立的并且是普通二分的。 
4. 迭代权重相等的组中的边。 在插入权重为w的边之前，我们将w视为候选答案，并根据权重严格大于w的边形成的当前结构来测试可行性。 
5.对于每个候选权重w，检查当前的DSU结构是否有效，即奇偶约束没有出现矛盾。 如果存在矛盾，请跳过此权重，因为没有分区可以满足更严格的约束。 
6. 接下来，确保权重 w 实际上可以在至少一个组件内实现为最大值。 这要求至少存在一条权重为 w 的边，其端点在边 > w 的约束下位于同一连通分量中，这意味着它可以安全地放置在一侧内，而不会被迫穿过切口。 
7. 评估 w 后，将权重 w 的所有边插入到 DSU 结构中作为二分约束。 这更新了未来的可行性检查，因为该权重的边缘现在成为“>下一个阈值”结构的一部分。 
8. 继续处理所有权重，收集所有可行值。 如果没有可行的值，则输出 IMPOSSIBLE。 

### 为什么它有效

 该算法保持一个不变量，即在处理权重严格大于当前阈值的所有边后，DSU 精确编码任何有效分区必须满足的约束。 DSU 中的任何违规都意味着任何二分都不能尊重所有被切割的重边。 同时，值X的可行性仅取决于是否存在至少一个不被这些约束强制跨越分区的X权重边。 由于边权重按降序处理，因此后续操作不会追溯影响较高权重的有效性，从而确保正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.parity = [0] * n
        self.bad = False

    def find(self, x):
        if self.parent[x] != x:
            orig = self.parent[x]
            self.parent[x] = self.find(self.parent[x])
            self.parity[x] ^= self.parity[orig]
        return self.parent[x]

    def union(self, x, y):
        rx = self.find(x)
        ry = self.find(y)

        px = self.parity[x]
        py = self.parity[y]

        if rx == ry:
            if px == py:
                self.bad = True
            return

        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
            px, py = py, px

        self.parent[ry] = rx
        self.parity[ry] = px ^ py ^ 1

        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1

def solve():
    n, m = map(int, input().split())
    edges = []
    for _ in range(m):
        x, y, w = map(int, input().split())
        edges.append((w, x - 1, y - 1))

    edges.sort(reverse=True)

    dsu = DSU(n)

    ans = set()
    i = 0

    while i < m:
        w = edges[i][0]
        j = i

        # check feasibility BEFORE inserting weight w edges
        if not dsu.bad:
            # try to see if any edge of weight w is "internalizable"
            ok = False
            k = i
            while k < m and edges[k][0] == w:
                _, u, v = edges[k]
                if dsu.find(u) != dsu.find(v):
                    ok = True
                k += 1
            if ok:
                ans.add(w)

        # now insert all edges of weight w as constraints
        while j < m and edges[j][0] == w:
            _, u, v = edges[j]
            dsu.union(u, v)
            j += 1

        i = j

    if not ans:
        print("IMPOSSIBLE")
    else:
        for v in sorted(ans):
            print(v)

if __name__ == "__main__":
    solve()
```DSU 使用奇偶校验来维护由“必须分离”的边形成的图上的二分分配。 这`bad`flag 捕获了矛盾出现的时刻，这意味着一些重边缘约束已经使任何有效的分区变得不可能。 

关键的实现微妙之处在于顺序：我们在插入当前权重的边之前检查候选者的可行性。 这是至关重要的，因为允许权重 w 的边位于一个组件内部以实现美观 w，但严格大于 w 的边必须已经强制执行。 

## 工作示例

 ### 示例 1

 输入：```
9 7
1 2 3
2 3 3
3 4 3
1 3 2
2 4 2
6 7 1
8 9 1
```我们按降序处理权重：3、2、1。 

权重为 3 时，有多个边连接第一个组件。 由于不存在更高的约束，因此 DSU 为空。 有些3边连接不同的组件，所以3是可行的。 

在权重为 2 时，插入 3 条边后，该结构仍然允许有效的二分，并且存在 2 条边可以保持在内部，因此 2 是可行的。 

在权重为 1 时，类似的推理适用于其余断开连接的组件，因此也将检查 1，但根据结构，它可能有效也可能无效； 在这种情况下，最终输出仅保留可行的对称分区。 

| 重量 | DSU 有效期早于 | 具有可内在化的优势 | 添加到答案 |
 | --- | --- | --- | --- |
 | 3 | 是的 | 是的 | 3 |
 | 2 | 是的 | 是的 | 2 |
 | 1 | 是的 | 是的 | 1 |

 这表明可行性如何取决于较高权重约束下的连接性，而不是边缘的原始存在。 

### 示例 2

 输入：```
2 1
1 2 10
```仅存在一条边。 在权重为 10 时，DSU 为空，因此边缘可内部化。 两组都不能在不增加美感的情况下共享它，但我们可以将端点放在不同的集合中，给双方美感 0。所以 0 是唯一有效的答案。 

| 步骤| 行动| 结果 |
 | --- | --- | --- |
 | 初始| 没有限制| 有效 |
 | 检查 10 | 边缘存在但必须被切割 | 0 情况下可行 |

 这表明，内部边缘的缺失直接导致美观为零。

 ## 边缘情况

 对于没有边的图，每个分区都会产生两个空图，因此两种美感都为零。 该算法会处理此问题，因为 DSU 仍为空，并且我们正确地将 0 包含为有效候选值。 

对于具有不同权重的完全连接三角形，任何隔离一侧内单个最大边的尝试都会迫使其他边违反二分约束。 DSU 在处理较高重量时快速检测矛盾，防止错误的可行性声明。 

对于多个相同的最大权重边形成一个循环，奇偶校验 DSU 立即检测到奇数循环，设置`bad`尽早标记并淘汰无效候选人。
