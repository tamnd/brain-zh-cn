 ## 解决方案
 ## 问题理解

 我们得到三个正整数，代表三根棍子的当前长度。 在一步操作中，我们可以精确地选取一根棍子并将其长度增加一个单位。 目标是达到这三个长度可以形成非退化三角形的情况，这意味着所有三对边都严格满足三角形不等式。 

仅当任意两条边的总和严格大于第三条边时，一组三个长度才能形成有效的面积为正的三角形。 由于我们总是可以对边重新排序，因此关键条件变成检查在一些增量之后，最大的边是否严格小于其他两个边的总和。 

约束非常小：每个长度最多为 100。这立即表明，即使对所有合理状态进行强力模拟也是可行的，因为总搜索空间很小。 然而，该问题的结构使我们能够完全避免模拟。 

当棍子已经满足三角不等式时，当它们恰好位于简并边界上，以及当一根棍子明显大于其他两根棍子的总和时，就会出现边缘情况。 例如，输入如下`3 4 5`已经形成一个有效的三角形，所以答案是零。 输入如`1 2 3`不要形成三角形，因为不允许相等； 我们至少需要一个增量。 输入如`1 1 10`需要许多增量，因为最大的棍子支配着其他棍子的总和。 

尝试以任意方式增加棒的简单方法很容易错过增量的最佳分布，因为每一步应该增加哪根棒并不明显。 关键的困难在于，增加非最大边往往比增加最大边更有用，但这需要全局推理，而不是一步步贪婪地推理。 

## 方法

 思考该问题的一个直接方法是模拟在三根棍子之间分配增量的所有可能方式，直到出现有效的三角形。 每分钟我们选择三根棍子中的一根并将其增加一根，每一步都分为三种可能性。 这形成了一个巨大的状态空间，其中每个状态都是长度的三倍，并且转换增加一个坐标。 

即使我们限制总长度的增长，这种方法很快就会变得不可行，因为状态数量与操作数量组合增长。 例如，经过 50 次操作后，已经有以下数量级了$3^{50}$可能的序列，即使许多序列导致相同的最终状态。 

关键的观察是只有相对顺序和最大的边才是重要的。 假设我们对木棍进行排序，以便$a \le b \le c$。 三角形条件变为$a + b > c$。 如果不满足，我们需要增加总和$a + b$或减少$c$相对于它。 由于我们只能增加棍棒，所以唯一有用的操作要么增加两个较小边之一，要么以受控方式增加最大边，这仍然提高了可行性。 

更直接的观点是问：对于一个固定的排序，最少需要多少个增量才能满足$a + b > c$？ 如果仔细思考的话，我们根本不需要考虑在这个过程中改变最大元素的身份，因为任何最优序列都可以重新排列，这样我们在最后排序后总是将最终的最大边视为原始三边之一。 

因此问题减少为增加总和$a + b$直到超过$c$。 每个操作恰好将三个值之一增加一，因此最好的策略始终是增加两个较小边之一，因为增加最大边会使不等式更难以满足。 因此最小成本正是我们需要筹集的资金$a + b$变得严格大于$c$。 

这直接导致封闭式答案：我们计算赤字$c - (a + b)$。 如果这是负数，我们已经有了一个三角形。 如果是非负数，我们需要将总和增加$c - (a + b) + 1$单位。

 ## 算法演练

 1. 读取三个棒的长度。 这些是固定的初始值，除了概念上的推理之外，在计算过程中不会重新排序。 
2. 对这三个值进行排序，以便我们能够以稳定的方式推理三角形条件。 排序后，最大的值被明确识别为潜在违规方。 
3. 通过检查两个较小值的总和是否严格大于最大值来计算三角形是否已存在。 如果成立，则无需进行任何操作。 
4. 如果条件不成立，则计算距离满足不等式还有多远。 赤字是两个较小边的总和未能超过最大边的金额。 
5. 将赤字转换为所需增量的数量。 由于每次增量恰好增加一棒一个单位，因此每次操作都会增加左侧$a + b$如果我们选择最佳，最多会增加 1，因此我们准确计算需要多少个单位才能使不等式变得严格。 

### 为什么它有效

 排序后，对三角形有效性重要的唯一约束是$a + b > c$。 任何增加的操作$c$只会使这种不等式更难以满足，因此最佳序列永远不会在当前最大的棍子上花费移动。 这将每个最佳策略简化为在任何分布中重复增加两个较小的棍子，这相当于每次操作将它们的总和增加一。 由于总和必须严格超过$c$，最少的运算次数恰好是最小的整数$k$这样$a + b + k > c$，产生唯一的最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

a, b, c = map(int, input().split())
a, b, c = sorted([a, b, c])

if a + b > c:
    print(0)
else:
    print(c - (a + b) + 1)
```该代码首先对三个棒长度进行排序，以便可以以规范形式检查三角形条件。 这消除了关于当前哪一方最大的任何歧义，并避免了大小写区别。 

条件`a + b > c`直接检查有效三角形是否已存在。 如果是这样，则不需要增量。 

否则，我们计算需要多少增量才能使两个较小边的总和超过最大边。 表达式`c - (a + b) + 1`来自将严格的不平等要求重新排列为单位增加的整数。 

这里的一个常见错误是忘记了不平等的严格性。 使用`c - (a + b)`仅靠它只能达到相等，但仍然不能形成有效的三角形。

 ## 边缘情况

 对于输入`1 2 3`, 排序产量`a=1, b=2, c=3`。 算法检查`1 + 2 > 3`，这是错误的。 赤字计算为`3 - 3 + 1 = 1`，输出变为`1`。 这符合将任何一根棍子增加一个单位以使不等式严格化的需要。 

为了`1 1 10`，排序给出`a=1, b=1, c=10`。 总和是`2`，远低于`10`。 公式返回`10 - 2 + 1 = 9`。 然而，仔细推理表明，只需要 8 步就可以达到`1 + 9 > 10`，但是由于我们可以在两个较小的边上最优地分配增量，所以正确的解释是每个操作将总和增加一，所以我们需要`10 - 2 + 1 = 9`增量以使其严格更大。 这证实了计算符合整数步长的严格不等式要求。
