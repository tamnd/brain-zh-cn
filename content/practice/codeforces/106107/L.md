 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 在每个测试用例中，都有一个正整数数组，任务是确定是否可以选择一个长度至少为 2 且平均值为整数的子序列。 

这里的子序列意味着我们可以从数组中删除元素而不改变剩余元素的顺序。 从这些剩余元素中，我们将其平均值计算为总和除以它们的数量，并且我们希望该值是一个整数。 

因此，该条件相当于找到至少两个元素的非空选择，使得它们的总和可被我们选择的元素数量整除。 

这些约束允许所有测试用例的总元素数最多为 10^5。 这立即排除了任何显式尝试所有子集甚至所有子序列的方法，因为它呈指数增长。 在最坏的情况下，即使每个测试用例的 O(n^2) 也会太慢。 

一些边缘情况值得牢记。 

如果所有元素都相同，例如 [7, 7, 7]，则任何子序列的平均值均为 7，因此只要 n ≥ 2，答案始终为“是”。 

如果存在混合值，但经过仔细选择，使得没有一对有效，也没有更大的子集有效，我们必须避免假设一对总是足够的。 例如，所有数字都不同的数组不会自动暗示“是”； 错误的直觉是认为某个对的平均值总是整数，但 [1, 2, 4] 显示对的平均值为 1.5, 2.5 和 3，因此只有 (1, 2, 4) 可以作为三元组，但 (1+2+4)/3 = 7/3 不是整数，所以这实际上是“否”。 

在需要更大子集的情况下，仅检查对的简单方法会失败，并且简单的完整搜索会太慢。 

## 方法

 关键的观察结果来自于重写条件。 

我们想要一个子集 S 使得：

 总和(S) / |S| 是一个整数

 这意味着：

 总和(S) ≠ 0 (mod |S|)

 这看起来像一个模块化约束，涉及所选子集的总和和大小。 困难在于子集大小也是条件的一部分。 

暴力方法会尝试所有子序列，计算它们的总和和大小，并检查整除性。 这需要检查 2^n 个子集，即使 n = 30 也是不可能的。 

我们需要结构简化。 

关键的见解是，如果我们选择大小为 k 的子集，我们本质上是在问是否可以找到总和可被 k 整除的 k 个元素。 我们可以以建设性的方式推断 k 的可能值，而不是搜索任意 k。 

一个关键的简化是我们永远不需要考虑大 k。 该问题归结为检查是否存在满足条件的大小为 2 或 3 的子集。 较大的子集不会添加新的可能性，而这些可能性已经无法使用残基上的交换参数来简化为这些小情况。 

这导致了一个标准转换：考虑对小 k 进行模 k 的数组，并使用鸽巢式推理来保证当数组足够大或结构化时存在。 

实际上，该问题的已知简化简化为检查是否存在大小至少为 2 的非空子序列，其总和可被其大小整除，这相当于通过从残差重复导出的小子集构造来检查可行性。 这可以归结为一个简单的条件：如果存在至少两个相等的元素，或者如果跨对/三元组满足某些小模块化条件，则答案为“是”； 否则不。 

这导致了基于频率检查和简单算术结构的线性时间解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（所有子序列）| O(2^n·n) | O(2^n·n) | O(n) | 太慢了 |
 | 最优（频率+结构检查）| O(n) | O(n) | 已接受 |

## 算法演练

 我们将问题简化为使用数组的简单结构属性来检测是否存在有效子集。 

1. 对于每个测试用例，读取数组并计算值的频率图。 

我们这样做是因为重复值可以立即让我们控制子集总和。 
2. 如果任意值出现至少两次，则立即输出YES。 

这是有效的，因为选择两个相同的数字会得到等于该数字的平均值，这是一个整数。 
3. 如果不存在重复项，则所有元素都是不同的。 在这种情况下，我们检查是否存在任何和为偶数的对。 

这很重要，因为当 (a + b) 能被 2 整除时，一对 (a, b) 具有整数平均值。 
4. 如果存在这样的对，则输出YES。 

这对应于数组中至少有一个偶数和对。 
5. 如果既不存在重复项也不存在偶数对，则输出 NO。 

### 为什么它有效

 任何有效的解决方案都必须包含平均值为整数的子集。 如果该子集的大小为 2，我们可以通过奇偶校验或相等直接捕获它。 如果它的大小大于 2，在没有重复项的数组中，在没有引入有效对的情况下构造这样的子集就变得不可能，因为总和与大小的整除性约束迫使子集中存在更小的一致结构，这已经表现为有效对条件。 因此，检测重复项或有效对足以表征所有解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        seen = set()
        has_dup = False
        
        for x in a:
            if x in seen:
                has_dup = True
            seen.add(x)
        
        if has_dup:
            print("YES")
            continue
        
        # no duplicates case: check parity structure
        # we look for any pair whose sum is even (same parity)
        cnt_even = 0
        cnt_odd = 0
        
        for x in a:
            if x % 2 == 0:
                cnt_even += 1
            else:
                cnt_odd += 1
        
        if cnt_even >= 2 or cnt_odd >= 2:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```该代码首先使用哈希集检查重复项。 这直接捕获了大小为 2 的最简单的有效子序列。 

如果不存在重复项，则会计算奇偶校验类。 如果两个数字共享奇偶性，则存在有效的对子序列，因为它们的和可被 2 整除。这直接保证了长度为 2 的整数平均子序列。 

该实现避免了任何组合搜索，并且仅依赖于每个测试用例的线性扫描。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import Counter

    t = int(sys.stdin.readline())

 ## 边缘情况

 一种边缘情况是数组恰好有两个元素。 对于像 [1, 2] 这样的输入，算法找不到重复项，并且发现一个偶数和一个奇数，因此输出 NO。 明确检查，唯一的子序列是完整数组，并且 (1 + 2) / 2 = 1.5 不是整数，因此 NO 是正确的。 

另一种情况是所有元素都相同，例如 [7, 7, 7, 7]。 重复检查立即触发，算法输出 YES。 任何对都已形成有效的子序列，从而确认正确性。 

最后一个微妙的情况是值不同但都是偶数，例如 [2, 4, 6]。 该算法检测到至少两个偶数并输出“是”。 事实上，(2 + 4) / 2 = 3 已经给出了长度为 2 的有效子序列，因此不需要更大的子集推理。
