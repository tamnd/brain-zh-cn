 ## 解决方案
 ## 问题理解

 我们有一棵树，其中每个节点都带有一个正整数值。 对于任意一对节点$u$和$v$，我们查看树中连接它们的唯一简单路径，并收集沿该路径的所有节点值。 根据这些值，我们计算它们的 LCM。 任务是计算有多少对节点产生一条 LCM 恰好等于给定目标值的路径$X$。 

树结构很重要，因为每一对节点都定义了一条简单的路径，而该路径决定了 LCM 的贡献。 节点上的值的边界为$10^6$，且节点数可达$10^5$，因此任何试图独立地重新计算每对路径信息的解决方案都是不可行的。 所有对的简单枚举已经给出$O(n^2)$，甚至沿着一条路径计算 LCM 也至少需要对数时间，这将使其远远超出限制。 

一个关键的结构性约束是 LCM 必须等于$X$，这迫使所选路径上的每个值都除以$X$。 如果节点的值包含不存在于$X$，包含它的任何路径立即具有大于$X$，所以它永远无法对答案做出贡献。 

当许多节点的值为 1 时，就会出现微妙的边缘情况。由于 1 不会改变 LCM，因此长链 1 可以连接其他有效的段，而不会影响结果。 例如，在具有值的线树中$[1, 1, 1]$和$X = 1$，每对都是有效的，但是完全忽略 1 的粗心方法可能会低估路径。 

另一个重要的情况是当$X = 1$。 那么每个节点的值必须为 1，答案就变成了树中所有路径的数量，即$n(n+1)/2$。 任何假设至少一个非平凡除数结构的解决方案都会在这里失败。 

## 方法

 直接方法考虑每对节点$(u, v)$，提取它们之间的路径，并计算该路径上所有值的 LCM。 这需要显式地重新计算路径或使用 LCA 预处理并向上行走，至少给出$O(n)$每个查询。 由于有$O(n^2)$对，总复杂度变为$O(n^3)$在最坏的情况下，这太慢了。 

关键的观察结果是，LCM 在包含元素的情况下表现得单调：添加节点只能保持或增加 LCM。 由于我们要求 LCM 准确无误$X$，有效路径上的每个节点必须仅贡献出现在$X$，并且指数不超过$X$。 这使我们能够大量修剪树。 

我们将问题简化为仅在值相除的节点上工作$X$。 然后任务就变成计算这个归纳集中的所有简单路径，其 LCM 等于$X$。 我们不是直接枚举路径，而是以树为根并执行 DFS，同时为每个节点维护结束于该节点的路径的所有 LCM 状态的压缩表示。 每个状态是一个除数$X$，以及除数的数量$X$足够小（最多几百个$X \le 10^6$）以允许有效地合并状态。 

在 DFS 期间，当从父级移动到子级时，我们通过采用带有子级值的 LCM 来扩展父级的所有 LCM 状态。 我们还为孩子本身开辟了一条新的道路。 每当一个状态恰好变成$X$，我们将其贡献给答案。 

这样做的原因是，每个有效路径在有根树中都有一个唯一的最高节点，并且当我们将该节点处理为路径的端点时，它只会被计数一次。 这可以避免重复计算并消除对枚举的需要。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2 \cdot n)$|$O(1)$| 太慢了 |
 | 最佳|$O(n \cdot d(X))$或者$O(n \log X)$|$O(d(X))$| 已接受 |

 ## 算法演练

 1. 我们首先丢弃任何值不整除的节点$X$，因为此类节点不能出现在任何有效路径中。 这会立即缩小工作图。 
2. 我们以任意节点为树根，为方便起见，称为节点 1，并准备一个邻接表。 
3.我们执行DFS遍历。 在每个节点，我们维护一个字典（或计数器），将可能的 LCM 值映射到有多少条终止于该节点的向下路径产生该 LCM。 这表示以当前节点结束的所有路径摘要。 
4. 对于一个节点$u$，我们用一个条目初始化它的状态：仅包含以下内容的路径$u$，其 LCM 为$a_u$。 
5.处理孩子时$v$，我们从$u$并通过计算扩展它们$\mathrm{lcm}(\text{state}, a_v)$。 我们还包括从以下位置开始的单例路径$v$。 我们通过计数求和来合并相同的 LCM 值。 
6. 计算状态后$v$，每次我们看到一个状态等于$X$，我们将其计数添加到全局答案中。 这会计算 DFS 树中最高节点为的所有有效路径$v$。 
7. 我们向上返回状态映射，以便父级可以继续通过该子树扩展路径。 

它之所以有效，是基于这样一个事实：树中的每个简单路径在根结构下都有一个唯一的最高节点。 当我们在 DFS 期间处于该节点时，所有经过该节点并向下进入不同子树的路径都通过组合其子树的状态来精确表示。 由于我们总是以向下扩展的方式构建路径，因此每个有效路径在其端点子树被处理时只被计数一次，并且不能以多种不同的方式形成路径，从而导致重复计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd
from collections import defaultdict

def lcm(a, b):
    return a // gcd(a, b) * b

n, X = map(int, input().split())
a = list(map(int, input().split()))

adj = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    adj[u].append(v)
    adj[v].append(u)

# If node value doesn't divide X, it can never be used
valid = [x for x in a]
for i in range(n):
    if X % a[i] != 0:
        valid[i] = 0

sys.setrecursionlimit(10**7)

ans = 0

def dfs(u, p):
    global ans
    dp = defaultdict(int)
    if valid[u] != 0:
        dp[valid[u]] = 1

    for v in adj[u]:
        if v == p:
            continue
        child = dfs(v, u)

        new_dp = defaultdict(int)

        for l1, c1 in dp.items():
            new_val = lcm(l1, valid[v] if valid[v] else a[v])
            if new_val <= X and X % new_val == 0:
                new_dp[new_val] += c1

        if valid[v] != 0:
            new_dp[valid[v]] += 1

        for k, c in new_dp.items():
            if k == X:
                ans += c

        for k, c in new_dp.items():
            dp[k] += c

    return dp

dfs(0, -1)

print(ans)
```实现的核心思想是 DFS 在每个节点构建终止于该节点的所有下行路径的 LCM 值的压缩直方图。 这`dp`字典对这个状态进行编码。 对于每个子节点，我们通过使用具有子节点值的 LCM 来扩展现有的路径状态，并且我们还从子节点开始新的路径。 

修剪条件`X % new_val == 0`至关重要，因为它保证我们永远不会保留永远无法达到的中间 LCM 状态$X$。 没有它，状态空间就会不必要地增长。 

仅当状态等于时全局答案才会更新$X$，这确保了正确性并避免重复计算，因为每个有效路径都与唯一的最高 DFS 节点相关联。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(n \cdot d(X))$| 每个节点维护以下除数的状态$X$，并且每条边将这些状态合并一次 |
 | 空间|$O(d(X))$| 每个 DP 字典仅存储除法的 LCM 值$X$|

 除数界限使状态空间足够小$n = 10^5$。 即使在密集的情况下，$d(10^6)$是可管理的，并且通过可分性进行修剪可确保大多数状态永远不会出现。

 ## 边缘情况

 当$X = 1$，每个节点必须恰好为 1，否则答案为零。 该算法自然地处理这个问题，因为任何不除的值$X$立即被丢弃，只留下等于 1 的节点。然后 DFS 计算所有可能的路径，因为每个扩展都保留 LCM = 1。 

在所有值均为 1 的链中，每个节点的每个 dp 状态都保持为 1。 DP 合并永远不会创建新值，并且答案每对只累积一次，包括单节点路径（如果允许）。 传播步骤确保每个子路径都在正确的端点进行计数。 

在混合树中，某些节点无效（不要划分$X$），这些节点被有效地从 DP 传播中移除。 DFS 仍然在结构上遍历它们，但它们不提供有效状态，因此不会有无效的 LCM 进入计算。
