 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 在每个测试用例中都有一组正方形，每个正方形都有整数边长。 从这个集合中，我们必须计算有多少种方法来选择三个不同的正方形，以便这三个正方形可以边缘对齐放置，以精确地平铺一个矩形，没有间隙或重叠。 

每个选定的方块必须完整使用，不需要旋转，因为方块是对称的，并且我们不允许破坏或合并方块。 问题纯粹是关于三个给定的正方形瓷砖是否可以完美地形成一个更大的矩形。 

这些约束允许所有测试用例总共有 200,000 个方格，因此任何直接尝试所有三元组的解决方案都是不可能的。 每个测试用例的三次甚至二次方法将无法生存。 预期的解决方案必须将问题简化为对边长频率上的模式进行计数。 

一个微妙的点是，我们并不是在平面上任意排列正方形，而只是检查某种排列是否存在。 这意味着我们正在对有效的几何平铺进行分类，而不是模拟放置。 

当假设任意三个正方形的面积之和等于矩形面积时，就可以将它们组合起来，这是一个天真的陷阱。 例如，边长为 1、2、3 的正方形的总面积为 14，这对于轴对齐的正方形而言，在任何有意义的平铺意义上都不是完美的矩形面积，但仅基于面积的推理是不够的且具有误导性。 

另一个常见的错误是假设有两个相等的正方形就足够了。 例如，边 2、2、3 不会形成有效的矩形平铺，即使重复可能表明对称。 

核心困难是确定三个正方形的有限数量的有效平铺配置。 

## 方法

 暴力方法会尝试每一个三元组的索引，并检查这三个正方形是否可以形成一个矩形。 对于每个三元组，人们将尝试所有几何放置或等效地测试所有平铺配置。 这导致大约$O(n^3)$每个测试用例的三倍，这远远超出了可行的限制$n$达到$2 \cdot 10^5$。 

即使我们优化几何检查，瓶颈仍然是三元组的数量。 关键的观察结果是，只有三个正方形的有效平铺结构受到极大限制。 

如果我们想象用正方形构建一个矩形，那么只有两种基本方法可以将矩形划分为三个轴对齐的正方形。 一种是将三个相同的方块排成一行或一列。 另一种是将两个相同的较小方块放置成一条带，其中一个较大的方块占据另一个维度的剩余空间。 

这一观察结果将问题从几何问题转化为边长上的频率计数问题。 我们不需要测试三元组，而只需要计算多重值集中的两个特定模式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了 |
 | 频率计数 |$O(n \log n)$或者$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 对于每个测试用例，计算每个正方形边长出现的次数。 这将问题从单个方块减少到相同大小的频率。 
2. 数出三个正方形边长相同的三元组。 如果一个值$x$出现$f$次，那么它就贡献了$\binom{f}{3}$有效的三元组。 这对应于将矩形形成为由相同正方形组成的 3×1 条带。 
3. 计算两个正方形都有边长的三元组$a$一个正方形有边长$2a$。 对于每个值$a$， 如果$2a$存在，那么选择这样的三元组的方法数是$\binom{f[a]}{2} \cdot f[2a]$。 这对应于一个$2 \times 1$一维平铺并用一个正方形填充剩余区域。 
4. 将所有值的贡献相加并输出测试用例的结果。 

### 为什么它有效

 任何使用恰好三个正方形的矩形的有效平铺都必须以形成三个相等正方形的单行或一列的方式对齐正方形，或者形成两层结构，其中一个正方形跨越整个维度，而另外两个正方形对称地填充剩余区域。 无法在不产生间隙或强制非正方形区域的情况下将矩形划分为三个不相等的轴对齐正方形。 这将所有有效配置限制为恰好两个计数的情况，从而使计数完整且准确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import Counter

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        freq = Counter(a)
        ans = 0
        
        for x, fx in freq.items():
            if fx >= 3:
                ans += fx * (fx - 1) * (fx - 2) // 6
            
        for a_val, fa in freq.items():
            b_val = 2 * a_val
            if b_val in freq:
                ans += fa * (fa - 1) // 2 * freq[b_val]
        
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将输入压缩为频率图。 这是至关重要的，因为所有有效的构造仅取决于每个边长出现的次数，而不是它们的位置。 

相同三元组的组合公式直接计算无序索引选择。 第二项仔细地将一个值固定为较大的平方，并选择两个相同的较小的值。 使用组合可以避免排序问题，因为三元组是由索引集定义的$i < j < k$。 

一个微妙的实现细节是迭代频率图而不是原始数组，确保每个测试用例的线性行为。

 ## 边缘情况

 当所有正方形都相同时，就会出现极端情况。 用于输入`1 1 1`，只有一种方法可以选择所有这三个，并且它形成一个有效的矩形。 该算法完全通过组合项来处理这个问题$\binom{f}{3}$，它可以正确计算所有索引三元组，无需额外的逻辑。 

另一种边缘情况是仅存在混合模式时，例如`2 2 4`。 这里的算法避免计算任何无效的排列，因为它只考虑严格的关系$b = 2a$，防止意外包含不相关的值。 

最后一个微妙的情况是大频率，例如许多相同的正方形。 组合公式增长很快，但由于整数很大，在 Python 中仍然安全，并且计算仍然以线性时间运行，因为它避免了显式枚举三元组。
