 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并且最多可以对特定类型执行一次全局修改。 在该操作中，我们选择一些位置子集，并将相同的整数值添加到该子集中的每个元素。 在这个可选操作之后，我们查看整个数组的最大公约数并希望它大于一。 

任务是确定我们必须修改的子集的最小可能大小，以便可以实现严格大于 1 的全局 gcd。 如果数组的 gcd 已经大于 1，则无需进行任何操作，答案为零。 

核心难点在于该操作并没有统一移动所有元素，而只是移动子集。 这会在选定元素和未选定元素之间产生交互，因为最终的 gcd 同时取决于两个组。 

这些约束意味着输入大小最多为一百万个元素，值最多为十亿。 任何明确检查所有对或尝试候选子集的解决方案都是不可能的。 即使每个候选值的 O(n log n) 也已经太慢了，因此解决方案必须依赖于聚合算术结构，通常涉及 gcd 属性和频率分析而不是模拟。 

一些边缘情况很重要。 

如果所有元素都已经是某个大于一的整数的倍数，则答案为零。 例如，`[6, 10, 14]`已经有了gcd 2，所以不需要任何操作。 

如果所有元素都等于 1，则任何子集运算只会将所选元素移动 X，并且我们必须确保所有最终值共享大于 1 的公约数。 简单的方法可能会尝试随机 X 值，但正确的推理仅取决于模块化结构。 

如果只修改一个元素，人们很容易认为它总是足以使 gcd 变大，但这忽略了作为限制 gcd 的固定残基的未修改元素。 

## 方法

 我们首先考虑暴力观点。 我们尝试每个子集 S 和每个整数 X，模拟操作，并计算结果数组的 gcd。 对于每个子集，我们将测试是否存在某个 X 使 S 中的所有数字及其补码对齐到一个公约数结构中。 这已经变得不可行，因为需要考虑 2^n 个子集和无限多个 X 值。 

即使我们修复 X，检查子集也需要重新计算 n 个元素的 gcd，从而导致 O(n 2^n) 行为，这完全超出了范围。 

关键的结构见解是，我们并不是真的试图强迫所有数字变得相等，而只是让它们都可以被某个素数 p 整除。 这改变了视角：我们不是直接跟踪 gcd，而是分析模 p 的整除性。 

如果运算后所有数字共享一个素因数 p，则每个元素必须满足 a_i == 0 (mod p) 或 (a_i + X) == 0 (mod p)。 这创建了一个划分条件：未修改的元素必须已经是 0 mod p，而修改的元素必须在添加 X 后共享相同的模 p 余数。这迫使所有选定的元素位于模 p 的单个余数类中。 

对于固定的 p，我们可以计算有多少个元素已经可以被 p 整除。 其余元素必须通过运算可被 p 整除。 对于这些，我们需要选择一个子集，使得它们都对模 p 全等，因为对于所有选定的 i，单个 X 必须满足 X ≡ -a_i (mod p)。 

因此，对于每个素数 p，我们能做的最好的事情就是保持所有已被 p 整除的元素，并选择共享模 p 相同余数的最大不可整除元素组。 其他所有内容都必须包含在子集中。 

因此，对于每个 p，我们计算：

 不能被 p 整除的元素数量减去不能被 p 整除的元素中 a_i mod p 的最大频率。 

我们希望最小化数组或其因子中出现的所有素数。 

这将问题简化为基于因子的频率计数而不是子集枚举。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集和值 | O(n·2^n) | O(n·2^n) | O(n) | 太慢了|
 | 基于素数的分组和频率| O(n √A) 摊销 | O(n) | 已接受 |

 ## 算法演练

 我们专注于从数组中提取候选素数并评估每个素数的成本。 

1. 计算整个数组的gcd。 

如果大于1，则立即返回0。 

这是可行的，因为如果所有数字已经共享一个除数，则不需要进行任何操作。 
2. 从所有数组元素中收集候选素数。 

我们对每个元素进行因式分解并存储其不同的素因数。 

这些素数是修改后可以成为最终 gcd 的唯一可能值，因为 gcd 必须一致地除以至少一个修改或未修改的结构。 
3. 对于每个候选素数 p，计算两个结构：

 可被 p 整除的元素的计数，以及不可被 p 整除的元素的残基 a_i mod p 的频率图。 

可整除的元素已经满足条件，不需要包含在子集中。 
4. 对于不可整除的元素，按余数对 p 取模进行分组。 

我们保留的最佳子集是所有已被 p 整除的元素加上最大的残差组。 

所有其他不可整除的元素必须包含在要调整的子集中。 
5. p 所需的子集大小为：

 总元素减去（可除以 p + 最佳残基组大小）。 
6. 取所有素数的最小值。

### 为什么它有效

 修复一个素数 p 来除以最终的 gcd。 运算后每个元素都必须能被 p 整除。 未选择的元素无法更改，因此它们必须已经满足 a_i == 0 mod p。 选定的元素都接收相同的移位 X，这意味着所有选定的值必须落在以 p 为模的相同残基类中。 这会强制所有选定的索引共享相同的 a_i mod p 值。 因此，最好的策略是在 p 的非倍数中选择最常见的残基类别。 没有其他结构可以进一步减少修改元素的数量，因此这实现了该 p 的最小子集大小。 取最好的 p 给出全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def factorize(x):
    res = set()
    d = 2
    while d * d <= x:
        while x % d == 0:
            res.add(d)
            x //= d
        d += 1
    if x > 1:
        res.add(x)
    return res

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    g = 0
    for x in a:
        g = g or x if g == 0 else __import__("math").gcd(g, x)

    if g > 1:
        print(0)
        return

    primes = set()
    for x in a:
        primes |= factorize(x)

    if not primes:
        print(n)
        return

    best = n

    for p in primes:
        freq = defaultdict(int)
        cnt_div = 0
        for x in a:
            if x % p == 0:
                cnt_div += 1
            else:
                freq[x % p] += 1

        best_res = max(freq.values()) if freq else 0
        need = n - (cnt_div + best_res)
        best = min(best, need)

    print(best)

if __name__ == "__main__":
    solve()
```实施从完整的 gcd 减少检查开始。 这是整个数组被压缩为单个值的唯一时刻，并且它立即处理琐碎的零答案情况。 

因式分解步骤收集所有可能充当最终 gcd 的素数。 这避免了考虑不相关的模数。 每个素数的频率计算将已经有效的元素与需要修改的元素分开，并确定最佳的残基对齐。 

一个微妙的问题是，仅针对不能被 p 整除的元素计算留数。 在残差分组中包含可除元素会扭曲频率结构，因为它们已经有效并且不应该被强制进入任何类别。

质因数：{2,3,5}。

所以子集大小 = 4 - (2 + 2) = 0。

```所有数字都是不同的质数，gcd 为 1。 

尝试 p = 3：

 除3倍数外，所有元素都是不可整除的，留数都不同，所以最好的组是1。

所有素数的行为都相似，所以答案是 4。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n √A + 总分解成本) | 每个数字都被分解一次，然后在数组 | 上计算每个素数
 | 空间| O(n + P) | 频率图加上素数集|

 约束允许最多一百万个元素，但因式分解是摊销的，因为值最大为 1e9，并且每个数字平均只贡献几个素数。 每个素数扫描是线性的，但在实践中，所有数字中不同素数的数量有足够的界限，使得这种方法可以在优化的 Python 或编译语言下通过。

 ## 边缘情况

 一个单元素数组，例如`[7]`初始化后立即触发 gcd 检查，因为 gcd 等于元素本身。 该算法返回零，因为不需要修改。 

完全偶数数组，例如`[2, 4, 6]`开头有gcd 2。 gcd 在任何因式分解之前检查短路，防止不必要的计算。 

全为 1 的数组在分解过程中不会产生素数，从而使素数集为空。 该算法返回 n，这意味着如果需要任何非平凡的 gcd，则每个元素都需要包含在子集中。 

一组不同的素数，例如`[2, 3, 5, 7]`产生许多候选素数，但对于每个素数，残基结构会崩溃为大小为 1 的孤立组。 根据评估，计算出的答案变为 n-1 或更大，这反映出几乎所有元素都必须调整为在单一模数结构下对齐。
