 ## 解决方案
 ## 问题理解

 我们得到一个数字数组和一个相同长度的二进制字符串。 我们从左到右处理索引。 在位置`i`，如果二进制字符是`1`，我们从数组中获取前缀`1`到`i`并将其反转到位； 否则我们什么也不做。 

关键的困难在于每个操作都会影响所有后续操作的数组结构。 稍后索引处的前缀反转可能会以不明显的方式撤消或与早期反转组合，因此直接模拟会强制对越来越大的前缀进行重复工作。 

约束允许最多`5 · 10^5`跨测试用例的总数组长度，最多`10^5`测试用例。 这立即排除了任何执行显式前缀反转的解决方案`1`以一种简单的方式。 单一反转是`O(n)`，并且在最坏的情况下，复杂度变成二次方。 

当交替反转相互作用时，就会出现微妙的失败情况。 例如，如果数组是`[1,2,3,4]`我们逆转`i=3`再次在`i=4`，朴素模拟产生：

 之后`i=3`:`[3,2,1,4]`后`i=4`:`[4,1,2,3]`尝试“仅计算反转”的简单优化会失败，因为前缀反转不会以简单的加法方式交换。 效果取决于顺序和前缀边界，而不仅仅是总操作的奇偶性。 

## 方法

 蛮力方法是当我们看到一个时，逐字执行每个前缀反转`1`在二进制字符串中。 每次长度反转`i`成本`O(i)`时间，所以最坏的情况是每个角色都`1`，导致`1 + 2 + ... + n = O(n^2)`每个测试用例的工作。 总共有`n`最多`5 · 10^5`，这已经远远超出了极限。 

关键的观察是我们不需要中间数组。 每个元素的最终位置完全取决于有多少个活跃的前缀反转影响它，但直接跟踪该效果仍然很重要，因为每个反转仅翻转前缀的方向。 

我们不再模拟数组，而是从指针的角度重新解释这个过程。 想象一下，将当前的“逻辑数组”维护为一个带有两个指针的段：左边界和右边界，以及指示我们是从左到右还是从右到左读取它的方向标志。 索引处的前缀反转`i`有效地翻转第一个的方向`i`元素。 如果我们处理索引`n`下降到`1`，我们可以使用此结构决定每个元素如何映射到最终配置，而无需物理上反转任何内容。 

这种向后视角之所以有效，是因为每个位置的最终贡献取决于它是否位于包含它的奇数或偶数个前缀反转内。 通过从末尾开始处理，我们避免了重复的变异，而是模拟元素如何“消耗”到结果中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 两指针反向模拟| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1.初始化两个指针`l = 1`和`r = n`和一个布尔标志`rev = False`。 这些表示数组的当前活动段以及逻辑上是否反转。 这种抽象取代了物理上反转的子数组。 
2.从index开始遍历二进制字符串`n`下降到`1`。 我们向后退，因为位置处的每个前缀操作`i`只影响第一个`i`元素被解释，并且从最后开始工作使我们能够决定最终的位置，而无需重新审视之前的修改。 
3.如果`b[i] == '0'`，我们在结构和概念上不做任何事情来跳过它，因为它没有引入方向或边界的变化。 
4.如果`b[i] == '1'`，我们认为现阶段第一个`i`元素颠倒了。 在指针模型中，这意味着段`[l, r]`必须像前缀长度反转一样进行调整`i`已应用于当前逻辑数组。 我们通过从适当的一端提取下一个元素来模拟这一点，具体取决于`rev`，然后更新对应的指针。 
5. 我们从后到前构造答案数组。 每一步，取决于`rev`，我们选择`a[l]`或者`a[r]`，将其附加到结果中，并向内移动指针。 如果我们在概念上“应用”一步反转，我们会切换`rev`。 
6. 最后，反转构建结果，因为我们以与提取相反的顺序构建了它。 

关键思想是我们从不执行任何实际的前缀反转。 所有效果均通过指针移动和方向切换来表示。 

### 为什么它有效

 数组的每个元素在被使用时都会被删除一次`l`或者`r`。 这`rev`flag 编码当前剩余段是否由于奇数个前缀反转影响而被逻辑翻转。 由于每个操作仅翻转前缀的方向，因此对其余后缀的影响始终可以表示为“从左读取”或“从右读取”。 这个不变量确保在每一步中，下一个提取的元素与完整模拟接下来放置在最终结构中的元素完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        b = input().strip()

        l, r = 0, n - 1
        rev = 0
        res = []

        # We process from right to left
        for i in range(n - 1, -1, -1):
            if b[i] == '1':
                rev ^= 1

            if not rev:
                res.append(a[l])
                l += 1
            else:
                res.append(a[r])
                r -= 1

        res.reverse()
        print(*res)

if __name__ == "__main__":
    solve()
```实现直接遵循指针模型。 循环运行自`n-1`到`0`，每当`1`遇到。 这`l`和`r`指针跟踪剩余的可用段。 什么时候`rev`为 false，从左侧取出下一个输出元素； 当为 true 时，从右侧开始。 

反转标志进行异或运算（`rev ^= 1`）因为每个前缀反转都会翻转方向。 最终的逆转`res`是必需的，因为我们以相反的消费顺序构造答案。

 ## 边缘情况

 具有任何二进制字符串的单元素数组始终返回相同的元素。 该算法通过切换来处理它`rev`但两个指针始终指向相同的索引，因此输出保持稳定。 

完全为零的字符串永远不会切换`rev`，因此元素严格按顺序从左侧取出。 指针逻辑退化为简单的线性遍历、匹配恒等变换。 

一根完整的弦在每一步都会翻转方向。 每个切换仅反转解释的不变性确保元素在从右和左获取之间交替，从而在无需显式数组操作的情况下产生完全反转效果。
