 ## 解决方案
 ## 问题理解

 我们有以下数字的排列`1`到`n`。 一个值`x`必须通过在此未排序排列上运行的二分搜索过程找到。 该过程不寻找位置`x`直接地。 只根据中间元素是否最多移动左右边框`x`或大于`x`，当它停止时，它返回当前的左边框。 目标是在搜索开始之前执行最多两次交换，以便返回的位置包含`x`。 

输入包含几个独立的案例。 对于每种情况，我们都知道排列的大小、所需的值和当前的排列。 输出是交换列表。 接受零次、一次或两次交换的任何有效序列。 

所有排列的总大小最多为`200000`。 这排除了尝试许多可能的交换或重复模拟每个位置的过程的任何情况。 每个测试用例的线性解决方案就足够了，因为它给出了`O(200000)`总工作量，而检查所有位置对等方法将需要最多`O(n^2)`运营。 

问题的微妙部分是二分搜索不需要整个数组进行排序。 只有二​​分搜索路径访问的位置才重要。 一个常见的错误是移动`x`某处并假设搜索路径保持不变而无需证明。 另一个错误是在二分搜索模拟中使用从零开始的索引，而问题描述的是从一开始的位置。 

例如，考虑：```
1
4 1
2 3 4 1
```二分查找开始于`l = 1`,`r = 5`。 它检查位置`2`，看到`3 > 1`，并移动`r`到`2`。 它停止于`l = 1`，但位置`1`包含`2`。 正确的输出是：```
1
1 4
```一个粗心的解决方案，假设答案始终是原来的位置`x`会失败，因为算法返回的位置是通过比较确定的，而不是通过位置确定的`x`。 

另一个边缘情况是当`x`已经在二分查找返回的位置。```
1
5 3
1 2 3 4 5
```正确的输出是：```
0
```在许多情况下，始终执行交换的解决方案仍然会被接受，但它可能会意外移动`x`偏离正确的位置并产生错误的答案。 

## 方法

 最直接的方法是尝试可能的交换并检查结果。 单个交换有`O(n^2)`可能性，两次交换会创造更多的组合。 即使我们只模拟二分查找`O(log n)`每次尝试后，操作次数都远远超出了可能的范围`n`最多`200000`。 

关键的观察是我们不需要修复排列。 我们只需要把`x`进入当前二分查找结束的位置。 设该位置为`l`。 

对原始排列运行一次给定的二分搜索。 如果`p[l]`已经是`x`，无需更改。 否则，交换元素`x`就位`l`。 

唯一关心的是是否搬家`x`更改搜索进行的比较之一。 交换改变了两个位置：最终位置`l`和原来的位置`x`。 

位置`l`算法停止后不再检查。 原来的位置是`x`更有趣。 如果搜索检查了该位置，则与该值进行比较`x`，因此搜索将左边框移动到该位置。 左边框的任何后续移动只能发生，因为另一个检查值也最多`x`。 因此最终的位置`l`始终最多包含一个值`x`在这种情况下。 交换后，旧值来自`l`移动到原来的位置`x`，仍然满足相同的比较结果。 如果原来的位置为`x`从未被检查过，更改它根本不会影响搜索。 

所以一次交换总是足够的。 该问题允许两次交换，但最优构造只需要零次或一次。

 | --- | --- | --- | --- |
 | 蛮力 | O(n²) 或更糟 | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 查找当前位置`x`在排列中。 我们需要这个索引，因为唯一可能的修改是移动`x`到另一个地方。 
2. 模拟语句中描述的精确二分查找。 维持`l`和`r`作为基于一的边界并使用当前排列值更新它们。 当循环结束时，`l`是算法在任何交换之前返回的位置。 
3.如果位置的值`l`已经是`x`，输出零交换。 当前的排列已经满足所需的条件。 
4. 否则，交换位置处的值`l`和`pos[x]`。 这个地方`x`正是二分搜索结束的地方。 
5. 输出该单个交换。 

为什么有效：交换之前的二分搜索路径决定最终位置`l`。 掉期看跌期权`x`那里。 唯一可以提前检查的变化位置是旧位置`x`。 如果检查过，该值会从`l`保证最多是`x`，其给出与原始比较相同的方向`x`。 其他每次比较都保持不变，因此算法仍然达到`l`并返回包含的位置`x`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    ans = []

    for _ in range(t):
        n, x = map(int, input().split())
        p = list(map(int, input().split()))

        pos = 0
        for i, v in enumerate(p):
            if v == x:
                pos = i
                break

        l = 0
        r = n
        while r - l > 1:
            m = (l + r) // 2
            if p[m] <= x:
                l = m
            else:
                r = m

        if p[l] == x:
            ans.append("0")
        else:
            ans.append("1")
            ans.append(f"{l + 1} {pos + 1}")

    print("\n".join(ans))

if __name__ == "__main__":
    solve()
```该数组内部使用从零开始的索引，而所需的输出使用从一开始的位置。 仅在打印交换时才进行转换。 

模拟使用与语句相同的边框约定。`l`是当前有效的左边框，`r`是独占权边界，所以条件`r - l > 1`完全符合停止条件。 

寻找`pos[x]`是线性的。 二分查找模拟需要对数时间，因此整个解决方案以扫描一次排列为主。

 ## 边缘情况

 对于以下情况`x`已经是返回的位置，算法立即停止。 

输入：```
1
5 3
1 2 3 4 5
```二分查找到达位置`3`， 和`p[3]`已经是`3`，所以输出是：```
0
```不需要交换，因为每次比较都已经遵循预期路径。 

对于以下情况`x`不在返回位置，算法仅移动`x`。 

输入：```
1
4 1
2 3 4 1
```搜索检查位置`2`，向左移动，并停在位置`1`。 存在的价值是`2`。 交换位置`1`和`4`产生：```
1 3 4 2
```运行相同的二分搜索仍然停止在位置`1`因为唯一检查的值是`3`，大于`1`。 返回的位置包含`1`，这样就构建成功了。 

重要的边界是当`x`在二分查找过程中会对其本身进行检查。 上面的证明涵盖了这一点，因为移动到该位置的值来自最终的左边界，并且该值必须通过`<= x`创建边框时的比较。 搜索方向不变，所以最终结果保持不变。
