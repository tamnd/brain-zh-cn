 ## 问题理解

 该路线由一个字符串描述，该字符串包括`'0'`,`'1'`， 和`'?'`。 

标记的一段`'0'`必须穿凉鞋行走，有一段标记`'1'`必须穿着靴子行走，并标记一段`'?'`可以分配任何类型的鞋类。 从凉鞋改为靴子的成本`a`，同时从靴子改为凉鞋的成本`b`。 旅行者可以穿着任何一种鞋子开始，也可以穿着任何一种鞋子结束，因此只有连续路线段之间的变化才重要。 

任务是分配每个`'?'`任何一个`'0'`或者`'1'`以便所有鞋类更换的总成本尽可能小。 

路径长度最多为`200000`，所以任何二次方都是不可能的。 即使扫描所有可能的作业也是无望的，因为每个`'?'`可能性的数量加倍。 一个`O(n)`或者`O(n log n)`算法很容易足够快。 

棘手的部分是每个`'?'`属于一个块，并且在一个块内分配不同的值永远不会有帮助。 例如：```
0??1
```如果我们将其指定为`0011`，有一个转变。 如果我们将其指定为`0101`，共有三个转变。 块内的每一次额外更改只会增加成本。 

另一个边缘情况是整个字符串仅包含`'?'`。```
???
```正确答案是`0`，因为我们可以为每个位置分配相同的值。 假设第一个已知字符始终存在的解决方案将会失败。 

一个更微妙的情况是当`'?'`块仅接触已知的一侧。```
??111
```将块分配给`'1'`根本不产生任何过渡。 尝试独立优化每个块而不考虑丢失的邻居的解决方案可能会引入不必要的更改。 

## 方法

 暴力解决方案会尝试每个任务的每个任务`'?'`。 如果有`k`问号，这需要`2^k`作业。 每个作业都可以在线性时间内检查，给出`O(n · 2^k)`复杂性，即使是几十个问号也是不可能的。 

关键的观察结果是，只有相邻字符之间的转换才能得出答案。 连续的一个块`'?'`仅影响其左右边界。 由于在块内引入变化永远不会减少边界转换的数量，因此每一个最佳分配都会用一个相同的字符填充整个块。 

这使我们可以忽略确切的块长度。 相反，删除每个`'?'`从概念上讲，仅查看固定字符的序列。 每当两个连续的固定字符不同时，就必须发生一种不可避免的鞋类更换。 如果过渡是`0 → 1`，花费`a`。 如果是的话`1 → 0`，花费`b`。 

每一个`'?'`只要有可能，块总是可以被填充以匹配相邻的一侧，从而消除任何人为的过渡。

 ## 算法演练

 1. 读取路由字符串和值`a`和`b`。 
2. 从左到右扫描字符串，直到第一个不是的字符`'?'`。 这成为之前的固定字符。 
3、如果不存在固定字符，则输出`0`，因为每个位置都可以分配相同的值。 
4. 继续扫描字符串的其余部分。 
5.忽略每一个`'?'`，因为它的值将在稍后选择。 
6. 每当找到另一个固定字符时，将其与前一个固定字符进行比较。 
7. 如果不同，请添加`a`为了过渡`0 → 1`或添加`b`为了过渡`1 → 0`。 
8.更新之前的固定字符并继续。 
9. 输出累计成本。 

### 为什么它有效

 每个连续的块`'?'`只有两个相邻的固定字符。 用混合值填充块只能在块内引入额外的转换，而永远不会降低总成本。 用单个值填充整个块总是至少一样好。 

如果两个相邻的固定字符相等，则为整个块分配该值，从而不创建转换。 

如果它们不同，则无论发生在块内的哪个位置，都不可避免地会发生一次转换。 成本仅取决于固定字符的顺序，而不取决于块长度。 

全部删除后`'?'`，连续固定字符之间的每次转换都是不可避免的鞋类更换，因此将这些成本相加即可得出最小可能的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    a, b = map(int, input().split())

    prev = None
    ans = 0

    for ch in s:
        if ch == '?':
            continue
        if prev is not None and prev != ch:
            if prev == '0':
                ans += a
            else:
                ans += b
        prev = ch

    print(ans)

if __name__ == "__main__":
    solve()
```该实现仅保留先前的固定字符和当前答案。 

问号会立即被跳过，因为它们不会直接影响答案。 每当两个连续的固定字符不同时，就会添加相应的转换成本。 

第一个固定字符初始化状态而不增加任何成本，因为旅行者可以穿着任何一种鞋开始。 

如果字符串只包含`'?'`,`prev`遗迹`None`，不计算任何转换，并且答案正确地保持为零。

import sys
import io

def solve():
    input = sys.stdin.readline
    s = input().strip()
    a, b = map(int, input().split())

    prev = None
    ans = 0

    for ch in s:
        if ch == '?':

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout
    sys.stdin = io.StringIO(inp)
    out = io.StringIO()
    sys.stdout = out
    solve()
    sys.stdin = backup_stdin
    sys.stdout = backup_stdout
    return out.getvalue()

 考虑输入```

```不存在固定字符。 扫描从不分配`prev`，所以累积的答案仍然是`0`。 将每个位置分配给凉鞋或将每个位置分配给靴子确实不需要更换鞋子。 

现在考虑```

```第一个固定字符是`1`。 扫描过程中会忽略每个较早的问号，因为稍后可以将其分配给`1`。 不会遇到不同的固定字符，因此算法输出`0`。 

最后，考虑```

```扫描观察固定字符`0`和`1`。 它们有一次不同，所以它增加了`2`。 无论它们之间有多少个问号，每项作业都必须包含从凉鞋到靴子的准确过渡，使得`2`最佳答案。
