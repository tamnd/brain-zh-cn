 ## 解决方案
 ## 问题理解

 我们得到了一个部分已知的密码字符串，长度为`n`。 每个位置都已经限制了最终的密码字符。 有些位置固定为特定字符，有些位置在小写字母和大写版本之间灵活，有些位置完全未知，并且可以成为任何数字或任何字母。 

除了这些局部约束之外，最终字符串还必须满足三个全局条件：它必须包含至少一个大写字母、至少一个小写字母和至少一个数字。 此外，在最终分配后，不允许两个相邻字符相同。 

任务是计算有多少个完整密码同时满足位置限制和全局有效性规则，以 998244353 为模。 

长度约束`n ≤ 10^5`立即排除任何尝试枚举所有有效字符串的方法。 即使每个位置有 52 个选择的分支也会导致指数爆炸。 该结构建议对字符串进行动态编程方法，因为邻接约束强制局部状态依赖，而大写/小写/数字要求引入了在构造期间必须跟踪的全局状态。 

大小写灵活的小写字母与邻接限制之间的相互作用产生了微妙的边缘情况。 例如，如果`s[i] = 'a'`和`s[i+1] = 'A'`，将它们视为独立选择可能会在大小写转换后意外地允许禁止的相等，因为两者可能映射到相同的大写字母。 另一个极端情况是当许多位置都处于`'?'`，除非将角色分组到类别中，否则幼稚的 DP 转换会变得昂贵。 

## 方法

 暴力策略将尝试通过递归地为每个位置选择一个字符来生成字符串的所有有效补全`s`。 在每个位置，我们都会尝试每个允许的字符并在最后检查有效性。 这在概念上是有效的，因为它直接尊重所有约束，但分支因素是巨大的：每个位置最多 62 个选择，导致大约`62^n`最坏情况下的可能性。 即使通过邻接进行修剪也不会改变搜索的指数性质。 

关键的观察是，过去扩展字符串所需的唯一信息是最后选择的字符以及我们是否已经看到至少一个大写字母、小写字母和数字。 位置约束可以预处理为每个索引允许的字符集，并且转换仅取决于这些集和邻接不等式。 

这将问题简化为位置上的动态编程，其中状态跟踪三个布尔标志和前一个字符。 由于直接存储完整的前一个字符太大，我们将字符压缩为 62 个符号（26 个小写字母，26 个大写字母，10 个数字）的有限字母表，并在转换中显式处理邻接。 

DP 计算每个位置存在多少种方法可以达到由下式定义的状态：`(position, last_char, mask)`在哪里`mask`编码到目前为止我们是否见过大写、小写和数字。 每个步骤都会扩展为与以下字符不同的有效下一个字符`last_char`并且是当前位置约束所允许的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(62^n) | O(62^n) | O(n) | 太慢了|
 | DP 超过位置、最后一个字符、掩码 | O(n·62·3) | O(62·8) | 已接受 |

 ## 算法演练

 我们将字符编码为大小为 62 的统一字母表。每个字符属于三种类型之一：大写、小写或数字。 我们还维护一个 3 位掩码，其中位 0 表示我们至少看到了一个大写字母，位 1 表示小写字母，位 2 表示数字。 

### 1. 预先计算每个位置允许的字符

 对于每个索引`i`，我们构造一个可能的字符列表：

 如果`s[i]`是`'?'`，我们允许所有 62 个字符。 

如果`s[i]`是数字或大写字母，固定为单个字符。 

如果`s[i]`是一个小写字母，它可以是该小写字母或其对应的大写字母。 

此步骤将问题转换为固定候选集上的统一DP。 

### 2. 在位置 0 处初始化 DP

 对于第一个位置，我们尝试每个允许的字符。 每个选择都会根据字符类型初始化掩码。 此步骤没有先前的字符限制。 

### 3. 职位转换

 对于每个位置`i > 0`，我们计算一个新的 DP 表。 对于每个州`(last_char, mask)`从之前的位置开始，我们尝试用每个允许的字符扩展它`c`在位置`i`。 我们跳过过渡`c == last_char`。 我们通过 OR-ing 的类型更新掩码`c`。 

邻接约束在本地强制执行，所有全局约束通过掩码累积。 

### 4. 积累结果

 处理完所有位置后，我们对掩码等于的所有 DP 状态求和`111`以二进制表示，这意味着我们至少看到了一个大写字母、小写字母和数字。 

### 为什么它有效

 每个有效密码都对应于通过 DP 状态的一条路径，因为每个位置选择都被明确表示，并且在每次转换时都会强制执行邻接关系。 相反，每个 DP 路径都会构造一个有效的字符串，因为允许的字符集强制执行内存约束，转换强制执行邻接约束，而最终掩码强制执行全局字符类型要求。 没有无效的字符串可以进入有效的终止状态，并且没有有效的字符串被排除，因为所有符合规则的选择都被枚举了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def char_type(c):
    if '0' <= c <= '9':
        return 1 << 2
    if 'a' <= c <= 'z':
        return 1 << 1
    return 1 << 0

def expand_char(c):
    if c == '?':
        chars = []
        for i in range(26):
            chars.append(('a', i))
            chars.append(('A', i))
        for i in range(10):
            chars.append(('0', i))
        return list(range(62))
    if 'a' <= c <= 'z':
        return [ord(c) - ord('a'), ord(c.upper()) - ord('A') + 26]
    if 'A' <= c <= 'Z':
        return [ord(c) - ord('A') + 26]
    return [ord(c) - ord('0') + 52]

def ctype(idx):
    if idx < 26:
        return 1 << 1
    if idx < 52:
        return 1 << 0
    return 1 << 2

def solve():
    s = input().strip()
    n = len(s)

    allowed = []
    for i in range(n):
        if s[i] == '?':
            allowed.append(list(range(62)))
        elif 'a' <= s[i] <= 'z':
            a = ord(s[i]) - ord('a')
            b = ord(s[i].upper()) - ord('A') + 26
            allowed.append([a, b])
        elif 'A' <= s[i] <= 'Z':
            allowed.append([ord(s[i]) - ord('A') + 26])
        else:
            allowed.append([ord(s[i]) - ord('0') + 52])

    dp = [[0] * 62 for _ in range(8)]

    for c in allowed[0]:
        dp[ctype(c)][c] += 1

    for i in range(1, n):
        ndp = [[0] * 62 for _ in range(8)]
        for mask in range(8):
            for last in range(62):
                val = dp[mask][last]
                if not val:
                    continue
                for c in allowed[i]:
                    if c == last:
                        continue
                    nmask = mask | ctype(c)
                    ndp[nmask][c] = (ndp[nmask][c] + val) % MOD
        dp = ndp

    ans = 0
    for last in range(62):
        ans = (ans + dp[7][last]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案依赖于将所有字符压缩为 62 个状态，因此邻接是一个简单的整数比较。 DP 表存储掩码和最后一个字符的每个组合的计数，确保我们永远不会重新计算子问题。 掩码更新是一个简单的按位或，这使得跟踪所需的字符类型每次转换的时间恒定。 

一个常见的陷阱是忘记小写字母不是固定的：它们引入了小写和大写形式之间的分支，必须在预处理中而不是在 DP 转换期间处理。 另一个微妙之处是在位置 0 处初始化，该位置不适用邻接关系，因此所有允许的字符必须独立播种。 

## 工作示例

 考虑一个简单的输入`a?0`。 

在位置 0 处，`'a'`可以成为`a`或者`A`。 在位置 1 处，`'?'`允许所有字符，但必须与之前的选择不同。 在位置 2 处，`'0'`是固定的。 

| 步骤| 职位| 最后一个字符 | 面膜| 行动|
 | --- | --- | --- | --- | --- |
 | 初始化| 0 | 一个或一个| 小写或大写| 种子状态|
 | 1 | 1 | 取决于| 更新 | 展开除等于之外的所有有效 |
 | 2 | 2 | 取决于| 决赛| 只允许使用数字 |

 该跟踪显示了邻接如何立即修剪过渡，而不是在最后修剪。 

现在考虑`??1`。 

在位置 0 处，所有 62 个字符都是可能的。 在位置 1，每个都必须与第一个不同。 在位置 2 处，仅允许数字 1，因此转换将所有状态汇集到数字完整掩码中。 

这演示了 DP 如何逐渐积累约束，同时保留所有有效的部分构造。

 ## 边缘情况

 一个关键的边缘情况是当两个相邻字符`s`通过不同的情况限制相同的字母。 例如，`s = "aA"`强制两个位置代表相同的底层字母`a/A`，但邻接不允许相等。 DP 处理此问题是因为两个位置都扩展为相同的编码索引集，并且转换显式地阻止相同的索引，从而在必要时导致零有效扩展。 

另一种情况是由通配符包围的一长串数字。 由于数字仅贡献数字掩码位，因此 DP 确保其他地方必须满足大小写要求； 否则这些状态永远不会达到最终的掩模`111`。 这可以防止对满足邻接但错过所需字符类的字符串进行过度计数。 

最后，当所有角色都完成后`'?'`，DP 探索完全的自由，但仍然在本地强制执行邻接，确保即使字母表很大，也不会计算连续的相同字符。
