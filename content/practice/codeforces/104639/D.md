 ## 解决方案
 ## 问题理解

 我们给出一个无向简单图，其中一些顶点对已经通过边连接。 我们可以执行的操作是添加新边，但不能删除现有边，也不能引入平行边或自循环。 

最终目标是将这个图转换为具有强大结构属性的图：每当两个顶点可以通过某种边序列相互到达时，它们之间必须已经有一条直接边。 换句话说，可达性和邻接性在最终图中必须一致。 

这种情况会产生非常严格的后果。 在图的任何连接区域内，每对顶点最终都必须直接连接，否则它们之间将存在一条没有边的路径。 因此，每个连接的组件必须成为最终图中的一个派系。 

任务是计算实现此条件所需添加的最小边数。 

约束最多为一百万个顶点和一百万个边。 这立即表明任何试图直接考虑顶点对的解决方案都是不可能的。 如果我们不注意组件的处理方式，即使每个组件的二次行为也太慢。 线性或接近线性的图遍历是唯一可行的方向。 

一个微妙的一点是，我们不允许通过廉价地合并组件来“修复”图形。 如果我们连接两个先前断开连接的组件，则组合结构中的每个顶点都可以通过新边从其他每个顶点到达，这迫使我们也跨合并集添加所有缺失的边。 这很快就会变得比单独处理组件更昂贵。 

一个天真的错误是认为我们可以将所有东西连接成一个大集团并从那里计算缺失的边。 这忽略了在组件之间添加单个边会全局更改可达性结构并强制附加所需边的级联。 

例如，考虑尺寸为 2、2 和 2 且没有内部边缘的三个组件。 如果我们将它们全部连接到一个组件中，我们最终必须形成一个大小为 6 的团，这需要 15 条边，而保持它们分开只需要每个组件总共 3 条边，因此添加了 9 条边。 这种差异非常显着，并且说明了为什么合并不是最佳的。 

## 方法

 一种强力方法是模拟反复修复违规行为的过程。 我们可以搜索同一连接组件中不直接连接的任何一对顶点，添加该边，然后继续，直到实现闭合。 每次检查都需要 BFS 或 DFS 来验证可达性和邻接性，并且每次添加都会再次更改结构。 

这种方法的问题在于，大小为 n 的单个组件可能需要 n² 个缺失边，并且每次插入都可能触发连接性的重新计算。 这导致每个组件的最坏情况是三次方或至少是二次方开销，这远远超出了 n 高达 10⁶ 的可行限制。 

关键的观察是每个连接组件内的最终结构是完全确定的：它必须恰好在这些顶点上成为一个完整的图。 我们不需要模拟一条一条添加边的过程。 我们只需要计算完整图中缺失了多少条边。 

一旦我们认识到这一点，问题就简化为分解任务。 我们找到原始图的连通分量。 对于每个组件，我们计算需要多少条边才能使其成为一个团。 如果一个组件有 s 个顶点并且当前有 e 个边，那么完整的图将包含 s·(s−1)/2 个边，因此我们必须为该组件精确添加 s·(s−1)/2 − e 个边。 

将所有组件的总和得出最终答案。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 最坏情况 O(n3) | O(n + m) | 太慢了|
 | 连接组件计数 | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们依赖于连接组件的结构，因此算法专注于发现它们并测量它们的大小和边数。 

1. 构建图的邻接列表表示并准备所有顶点的已访问数组。 这使我们能够有效地遍历每个连接的组件。 
2. 从每个未访问的顶点运行 DFS 或 BFS，以一次识别一个连接的组件。 遍历时，收集属于该组件的所有顶点，并计算从这些顶点遇到的边数。 

边计数必须小心处理，因为每个无向边在邻接列表中出现两次，因此我们确保计数一致或稍后适当划分。 
3. 对于每个组件，计算其大小 s。 完整图中必须包含的边数为 s·(s−1)/2。 将其与该组件中当前存在的边数进行比较。 
4. 将所需边与现有边之间的差异添加到答案中。 
5. 输出所有分量的总和。 

### 为什么它有效

 关键的不变量是连接的组件对于传递性要求是独立的。 如果两个顶点位于不同的组件中，则它们之间没有路径，因此该条件对这些组件之间的边没有要求。 在组件内部，一旦两个顶点之间存在路径，最终的图形必须包含它们之间的直接边。 这迫使每个组件演变成一个派系，并且没有任何解决方案可以避免在不违反条件的情况下精确添加该派系缺失的边。 这使得组件分解既充分又必要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n, m = map(int, input().split())

adj = [[] for _ in range(n)]
for _ in range(m):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    adj[u].append(v)
    adj[v].append(u)

visited = [False] * n

def dfs(start):
    stack = [start]
    visited[start] = True
    nodes = 0
    edges = 0

    while stack:
        u = stack.pop()
        nodes += 1
        for v in adj[u]:
            edges += 1
            if not visited[v]:
                visited[v] = True
                stack.append(v)

    return nodes, edges // 2

ans = 0

for i in range(n):
    if not visited[i]:
        sz, e = dfs(i)
        ans += sz * (sz - 1) // 2 - e

print(ans)
```该实现使用迭代 DFS 来避免大型链上的递归深度问题。 在遍历过程中，每条边都会被计数两次，一次是从每个端点开始计算，所以在计算每个组件的最终边计数时，我们会除以二。 

一个常见的陷阱是忘记边缘必须按组件而不是全局进行计数。 简单地对所有边求和并从全局 n·(n−1)/2 中减去会错误地假设最终图必须是一个单独的团，而这不是必需的。

 ## 边缘情况

 单个孤立顶点是它自己的大小为一的组件。 公式 s·(s−1)/2 立即给出零，因此不添加边缘。 该算法自然会处理此问题，因为 DFS 标记顶点并返回该组件的零边。 

完全连接的组件已经满足派系条件。 在遍历期间，每个分量的每条边都被精确计数一次，并且由于它已经匹配 s·(s−1)/2，因此减法结果为零。 不存在过度计数的风险，因为边缘计数是针对每个组件的。 

长路径图测试 DFS 深度和边缘计数正确性的最坏情况。 即使每个顶点都是连接的，但只存在 n−1 条边，并且该公式正确计算了完成团所需的缺失边的数量。
