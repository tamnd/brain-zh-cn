 ## 解决方案
 ## 问题理解

 我们得到了两个单独的编程竞赛的有序结果。 每场比赛都会对各个团队进行排名，但最终重要的是大学的表现，而不是各个团队的表现。 

每场比赛中，一所大学都会派出多支队伍，但只有表现最好的队伍才重要。 因此，这两项比赛的第一步都是通过保留每所大学在比赛列表中出现的最佳（最早）位置，将团队级别的排名压缩为大学级别的排名。 

一旦我们有了两个大学排名（每个竞赛一个），我们就需要将它们合并为一个最终排序。 合并规则的行为就像两个排序列表的稳定合并：我们根据大学在每次比赛中的排名反复比较大学。 如果一所大学在第一次竞赛排名中早于另一所大学，则被认为是更好的大学； 如果他们在比赛中排名相同，则优先考虑第一场比赛的排名。 

合并后，会出现重复项，因为大学都出现在两个列表中。 我们只保留合并序列中每所大学的第一次出现，以保持顺序。 

输出是最终的大学去重排名。 

这些限制允许每次比赛最多有 10^4 支球队，因此简单的 O(n^2) 比较或重复扫描最佳排名已经是临界点，但在 Python 中仍然可以接受。 然而，合并步骤在预处理后必须是线性的，否则如果通过重复列表删除或重复列表中的成员资格检查来实现，它将出现 TLE。 

如果试图直接合并团队列表而不首先崩溃到大学排名，就会出现微妙的失败案例。 这会错误地将同一所大学的多个团队视为单独的竞争对手。 

当一所大学只参加一场比赛时，就会出现另一种边缘情况。 它仍然必须出现在最终排名中，并且应该正确合并，不得人为处罚。 

## 方法

 一种直接的方法是从字面上模拟整个过程。 对于每场比赛，扫描排名列表，并为每所大学存储第一次出现的索引作为其分数。 这会生成两个映射大学排名的字典。 

然后我们重复构建一个合并序列。 在每一步中，我们都会比较所有尚未排名的剩余大学，并根据比较规则选择最好的大学，将其追加，并将其从考虑中删除。 这相当于在自定义排序下重复选择最小值。 

问题是这个选择步骤的成本很高。 如果有 U 所大学，选择次优需要扫描 O(U) 个候选者，重复 O(U) 次，从而得到 O(U^2)。 当 U 达到 10^4 时，这将变成大约 10^8 次操作，这在 Python 中太慢了。 

关键的观察结果是，将每场比赛转换为独特大学的排名列表后，两个列表都已按所需顺序排序。 最终规则正是两个排序序列的稳定合并，其中比较是基于每个列表中的排名位置的字典顺序，并且平局有利于第一个列表。 

这将问题简化为使用两个指针合并两个预排序列表，然后使用集合进行重复数据删除。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力选择 | O(U^2) | O(U^2) | O(U) | 太慢了|
 | 两指针合并 | O(n + m) | O(U) | 已接受 |

 ## 算法演练

 我们首先将每场比赛的队伍名单压缩成一个独特的大学排名。

1. 从上到下阅读第一届比赛名单。 对于每所大学，如果没有见过，就按顺序记录下来。 这将产生第一个排名列表。 第二场比赛也是如此。 此步骤是有效的，因为只有每所大学的最佳团队才重要，并且最早出现的对应于最佳排名。 
2. 将这两个列表视为有序序列。 我们现在使用两个指针合并它们，一个指针指向每个列表。 在每一步中，我们都会比较两个列表中的当前候选人。 
3. 如果两个指针都指向不同的大学，我们会在最终排序中附加应该出现在前面的大学。 排序规则是第一个列表中较早的位置占主导地位； 否则我们比较第二个列表中的位置。 实际上，这相当于基于对（rank_in_first，rank_in_second）进行合并，并将缺失值视为无穷大。 
4. 如果一个列表已用完，我们将附加另一个列表中的剩余元素。 
5. 在合并期间，我们保留一组已经输出的大学。 如果一所大学已经输出，我们就跳过它。 这是必要的，因为一所大学可能出现在两个列表中，并且必须在最终输出中删除重复项。 
6. 继续，直到完全处理完两个列表。 

经过这些步骤后，输出列表以正确的合并顺序仅包含每所大学一次。 

### 为什么它有效

 每个竞赛列表都已按排名升序排序。 合并规则根据任意两所大学在这两个排序序列中的相对位置来定义它们之间的一致排序。 由于比较是传递的并且与排名对的字典顺序一致，因此两指针过程产生的合并序列保留了全局排序。 重复数据删除不会影响正确性，因为一旦大学出现在合并顺序中，任何后续出现的情况都代表着更差的位置，可以安全地忽略。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build_ranking(arr):
    seen = set()
    res = []
    for x in arr:
        if x not in seen:
            seen.add(x)
            res.append(x)
    return res

def solve():
    n, m = map(int, input().split())
    a = [input().strip() for _ in range(n)]
    b = [input().strip() for _ in range(m)]

    r1 = build_ranking(a)
    r2 = build_ranking(b)

    pos1 = {u: i for i, u in enumerate(r1)}
    pos2 = {u: i for i, u in enumerate(r2)}

    i = j = 0
    used = set()
    out = []

    while i < len(r1) or j < len(r2):
        while i < len(r1) and r1[i] in used:
            i += 1
        while j < len(r2) and r2[j] in used:
            j += 1

        if i >= len(r1) and j >= len(r2):
            break
        if i >= len(r1):
            u = r2[j]
            j += 1
        elif j >= len(r2):
            u = r1[i]
            i += 1
        else:
            u1 = r1[i]
            u2 = r2[j]

            if pos1.get(u1, float('inf')) < pos1.get(u2, float('inf')):
                u = u1
                i += 1
            elif pos1.get(u1, float('inf')) > pos1.get(u2, float('inf')):
                u = u2
                j += 1
            else:
                if pos2.get(u1, float('inf')) <= pos2.get(u2, float('inf')):
                    u = u1
                    i += 1
                else:
                    u = u2
                    j += 1

        if u not in used:
            used.add(u)
            out.append(u)

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先压缩每个比赛列表中重复的大学，仅保留第一次出现的大学，因为它们代表了最佳团队排名。 然后它会构建位置图，以便可以在恒定时间内完成比较。 合并循环小心地推进指针，同时跳过已经使用的大学以避免冗余处理。 当大学具有相同的相对顺序时，平局决胜条件确保了确定性行为。

 ## 边缘情况

 一种极端情况是，一所大学多次出现在单个竞赛列表中。 例如，如果输入是`A A A A B`，压缩步骤仅确保`A B`遗迹。 在执行过程中，算法会忽略后来发生的情况，因为`seen`立即设置过滤器。 如果没有此步骤，合并逻辑将错误地将重复出现的情况视为单独的条目并产生重复项。 

当一所大学只参加一场比赛时，就会出现另一种边缘情况。 例如，如果`X`排名第一但不在第二，其位置`pos2`默认为无穷大。 在比较过程中，这确保了它完全按照第一次比赛顺序排列，这符合缺席比赛不应对其进行处罚的预期规则。 

最后一种情况是比赛之间的全面逆转。 即使两个排名相反，成对比较规则也能保证确定性排序，因为每次比较都是基于排名对的一致字典顺序进行解析，并且通过优先选择第一个比赛来打破平局。
