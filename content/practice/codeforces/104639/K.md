 ## 解决方案
 ## 问题理解

 我们得到一个凸多边形，它代表平面上的安全区域。 对于每个空投查询，我们还会得到一个由其直径端点定义的圆。 每个空投均匀随机落在该圆圈内的任何位置。 

对于每个查询，我们必须选择多边形内的一个点。 该点充当固定着陆参考。 选择的成本是该所选点与圆内均匀随机点之间的欧氏距离平方的期望值。 任务是最小化多边形内点的所有选择的期望，并输出每个圆的最小值。 

多边形是固定的，而圆则因查询而异。 约束允许最多 5000 个顶点和 5000 个查询，坐标最大为 1e9。 

一个天真的解释是每个查询都需要优化凸多边形上的连续函数，这已经表明几何和凸性属性将是中心。 

一个关键的结构观察是，期望仅取决于所选点与圆中随机点之间的相对位置，而不取决于多边形形状，除非所选点的可行性。 

一个微妙的边缘情况是圆的最佳点位于多边形之外。 例如，如果多边形非常小并且圆距离很远，则最佳无约束点可能是圆心，但多边形会强制投影到其边界上。 这意味着我们不仅计算圆的几何属性，而且计算凸集的约束优化。 

## 方法

 首先忽略多边形约束。 对于固定圆，假设我们选择一个点 P，随机点 X 均匀分布在圆盘内。 目标是最小化 E[|P − X|²]。 

将平方展开得到 E[|P − X|²] = |P|² − 2P·E[X] + E[|X|²]。 关键的简化是期望仅取决于 P 与分布均值的距离。 对于圆盘上的均匀分布，平均值恰好是圆的中心。 因此，当 P 尽可能接近圆心时，表达式变得最小，并且在 P 等于圆心时实现最小无约束值。 

因此，问题简化为将圆心投影到凸多边形上。 最优P是多边形中以欧几里得距离最接近圆心的点。 一旦我们有了投影点 Q，答案就是 E[|Q − X|²]，它可以展开为 |Q − C|² + E[|X − C|²]，其中 C 是圆心。 

因此，问题分为两部分：仅取决于圆的几何常数项，以及从圆心到多边形的平方距离。 

剩下的任务是为每个查询点 C 计算从 C 到凸多边形的平方距离。 当 n 最大为 5000，q 最大为 5000 时，O(nq) 多边形内的点或到边缘的距离扫描就足够了。 由于多边形是凸多边形并且顶点是有序的，因此距外部点最近的点位于顶点或垂直投影落在线段内的边上。 这可以在每个查询的线性时间内进行检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询在所有边缘上的强力投影 | O(nq) | O(n) | 已接受 |
 | 最优与上述凸几何简化相同 | O(nq) | O(n) | 已接受 |

 不需要更高级的数据结构，因为约束足够小，并且一旦正确重写期望，几何结构就很简单。 

## 算法演练

1. 预先计算每个查询的圆心和半径平方。 中心是直径端点的中点，半径的平方是端点之间距离平方的四分之一。 这隔离了所有特定于查询的几何数据。 
2. 对于每个查询，将问题视为找到从圆心到凸多边形的最小平方距离。 这是合理的，因为期望分解为一个常数，仅取决于圆加上从所选点到中心的平方距离。 
3. 对于给定的中心点 C，迭代所有多边形边。 对于每条边 AB，计算 C 到线 AB 上的投影。 
4. 如果投影参数位于线段内，则计算从 C 到投影点的平方距离。 否则计算到 A 和 B 中最近端点的平方距离。这确保我们计算到该边缘段上最近点的距离。 
5. 保持所有边和顶点的最小平方距离。 这给出了从 C 到凸多边形的平方距离。 
6. 加上从圆盘中的随机点到其中心的平方距离的常数期望值（等于 R²/2），并输出总和。 

第 6 步起作用的关键原因是，对于均匀圆盘，二维的方差在轴上均匀分布，给出 E[|X − C|²] = R²/2。 

### 为什么它有效

 期望 E[|P − X|²] 可以分解为 |P − C|² + E[|X − C|²]，其中 C 是圆心。 第二项与 P 无关，因此最小化期望精确地减少为最小化从 P 到 C 的平方距离。由于可行区域是凸的，因此最近的可行点是明确定义的并且位于顶点或边投影上。 该算法枚举了所有此类候选点，因此它必须包括真正的最近点，从而实现全局最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def dot(ax, ay, bx, by):
    return ax * bx + ay * by

def dist2(ax, ay, bx, by):
    dx = ax - bx
    dy = ay - by
    return dx * dx + dy * dy

def clamp01(t):
    if t < 0.0:
        return 0.0
    if t > 1.0:
        return 1.0
    return t

def solve():
    n, q = map(int, input().split())
    poly = [tuple(map(int, input().split())) for _ in range(n)]

    for _ in range(q):
        x1, y1, x2, y2 = map(int, input().split())

        cx = (x1 + x2) / 2.0
        cy = (y1 + y2) / 2.0

        dx = x2 - x1
        dy = y2 - y1
        r2 = (dx * dx + dy * dy) / 4.0

        best = float('inf')

        for i in range(n):
            x3, y3 = poly[i]
            x4, y4 = poly[(i + 1) % n]

            ex = x4 - x3
            ey = y4 - y3

            vx = cx - x3
            vy = cy - y3

            e2 = ex * ex + ey * ey
            t = 0.0
            if e2 != 0:
                t = (vx * ex + vy * ey) / e2
                t = clamp01(t)

            px = x3 + t * ex
            py = y3 + t * ey

            best = min(best, dist2(cx, cy, px, py))

        ans = best + r2 / 2.0
        print(f"{ans:.10f}")

if __name__ == "__main__":
    solve()
```该解决方案首先将每个直径转换为圆心和半径的平方。 中点公式直接给出期望中心，而半径计算是纯代数的。 

多边形循环使用投影计算每条线段上的最近点。 夹紧步骤确保突出部分保持在段内； 否则端点占主导地位。 这是线段上的标准欧几里得投影。 

最后，答案是将圆心到多边形的距离平方与固定方差项 R²/2 结合起来。
