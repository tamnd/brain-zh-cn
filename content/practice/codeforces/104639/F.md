 ## 解决方案
 ## 问题理解

 我们得到一个整数数组和一个在其上定义的两人游戏。 Alice 首先移动，在每次移动中，玩家选择数组中的任意两个位置，并将这两个值替换为一对新的整数，保留它们的总和，同时严格减少它们的绝对差。 游戏继续进行，直到任何一对都无法进行此类移动，无法移动的玩家就输了。 

在游戏开始之前，我们可以删除任意数量的元素，只要剩下三个元素即可。 对于每个测试用例，我们必须计算最后三个索引中有多少个选择导致爱丽丝在最佳游戏下强制获胜的起始位置。 

每个测试用例对 n 的约束最多为 5×10^5，总 n 最多为 3×10^6。 这立即排除了任何尝试模拟每个三元组游戏的解决方案，甚至排除了每个测试用例比大约 O(n log n) 或 O(n) 更糟糕的解决方案。 由于输出是对所有三元组的计数，因此问题的结构必须将三个数字的游戏简化为简单的分类，然后允许通过频率聚合进行计数。 

当值非常接近时，会出现微妙的边缘情况。 例如，对于像 [5, 5, 6] 这样的三元组，不可能进行任何移动，因为任何对都具有差值 0 或 1，并且在保持整数值并保留总和的同时无法进一步减少差值 1。 所以这个配置是终极的，对于起始玩家来说肯定是失败的。 另一方面，像 [1, 3, 10] 这样的三元组显然允许移动，并且直观上玩家可以不断减少大的间隙，直到达到最终状态。 关键的困难在于准确识别哪些三元组是终结的，哪些三元组是获胜的。 

## 方法

 暴力破解的想法很简单：枚举每个索引三元组，根据这三个值模拟游戏，并确定 Alice 是否获胜。 这是立即不可行的，因为有 O(n^3) 三元组，并且每次模拟都涉及多个分支游戏状态。 即使进行大量修剪，每个三元组的配置数量仍然会快速增长，因为每次移动都会创建一个新的对状态。 这种方法在约束变得很大之前就失败了。 

关键的观察结果是，该操作只会减少两个选定数字之间的差异，同时保留它们的总和。 这意味着任何差值至少为 2 的对都可以严格“紧缩”，而差值为 0 或 1 的对已经处于最压缩的整数形式。 对于一组三个数字，如果没有一对数字的差异至少为 2，则游戏立即结束。这完全表征了终止状态。 

因此，我们不需要分析完整的博弈树，而只需确定一个三元组是否包含任何相差至少 2 的值对。如果不存在这样的对，则位置是终结的，并且第一个玩家会失败。 否则，该位置就获胜。 

这将任务简化为计算三元组，其中最大值减去最小值最多为 1，然后从所有可能的三元组中减去它们。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n^3 · 游戏) | O(1) | O(1) | 太慢了|
 | 频率+组合学| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题简化为对三元组值进行分类。

1. 计算数组中每个不同值的频率。 这使我们能够在不枚举索引的情况下推理三元组。 
2. 计算从数组中选择任意三个元素的方法总数。 这是 C(n, 3)。 这代表了所有可能的最终选择。 
3. 识别“坏”三元组，即爱丽丝输掉的三元组。 当三元组中的最大值和最小值之差最多为 1 时，三元组就会失败。这意味着所有三个值必须位于单个值或两个连续整数值中。 
4. 计算所有三个值都相等的三元组。 对于每个值 x，这贡献 C(cnt[x], 3)。 
5. 计算值仅来自两个相邻值 x 和 x+1 的三元组。 该组合桶中的三元组总数为 C(cnt[x] + cnt[x+1], 3)，但这包括已经计数的“全部相等”情况，因此我们减去 C(cnt[x], 3) 和 C(cnt[x+1], 3)。 
6. 对所有 x 求和，得到失败三元组的总数。 
7. 从总三元数中减去失败的三元数即可得到答案。 

### 为什么它有效

 该操作总是允许减少任何差值至少为 2 的对。因此，只要三元组包含这样的对，就至少存在一个移动。 相反，如果所有值都在大小为 1 的范围内，则在保留整数值的同时无法进一步减少对，因此该位置是终端。 这使得失败条件相当于“三元组中的所有值都在一个单位间隔内”，这正是计数公式所捕获的内容。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def C3(x):
    if x < 3:
        return 0
    return x * (x - 1) * (x - 2) // 6

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    freq = defaultdict(int)
    for x in a:
        freq[x] += 1

    total = C3(n)

    bad = 0
    keys = sorted(freq.keys())

    for x in keys:
        bad += C3(freq[x])

    for x in keys:
        if x + 1 in freq:
            bad += C3(freq[x] + freq[x + 1]) - C3(freq[x]) - C3(freq[x + 1])

    print(total - bad)

if __name__ == "__main__":
    solve()
```该代码首先将数组压缩为频率图，以便可以对三元组进行组合计数。 函数 C3 安全地计算三个元素的组合。 我们计算总三元组，然后减去由单个值或两个相邻值形成的所有丢失配置。 

邻接循环通过减去已包含的纯值贡献来小心地避免重复计算。 

## 工作示例

 考虑一个数组`[1, 1, 2, 2]`。 

我们有频率：`1 -> 2`,`2 -> 2`。 三元组总数为 C(4, 3) = 4。 

我们计算坏三元组：

 所有相等的三元组对于两个值都贡献 C(2,3)=0。 

对于 (1,2) 对，我们计算 C(4,3)=4 减去 C(2,3) 减去 C(2,3)，得到 4。 

所以 bad = 4，答案是 0。每个三元组都会失败，因为所有元素都在大小为 1 的范围内。 

现在考虑`[1, 3, 10]`。 

所有频率均为 1。三元组总数为 1。没有值重复，因此不存在 C(3) 贡献。 也不存在相邻对，所以 bad = 0。答案是 1，意味着这个三元组获胜。 

这符合大间隙确保至少存在一对可还原的直觉。

 ## 边缘情况

 当所有元素都相同时，每个三元组显然没有有效的移动，因为每对的差值为 0。该算法将其完全计入 C(cnt[x], 3) 项内，并从总三元组中完全减去它，按预期产生零。 

当值在两个连续整数之间交替时，例如`[4,4,5,5,5]`，每个三元组仍然限制在大小 1 的范围内。组合的 C(cnt[x]+cnt[x+1],3) 项捕获所有选择，内部组合的减法确保不会过度计数，正确地将所有三元组标记为失败。 

当数组包含广泛分离的值时，例如`[1, 100, 200]`，不会计算坏三元组，因为不存在相邻值条件且不存在单值重复。 每个三元组都获胜，并且公式简化为总 C(n,3)，符合至少一个可约对始终存在的事实。
