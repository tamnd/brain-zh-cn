 ## 解决方案
 ## 问题理解

 我们给出一个参考程序，其运行时间固定为$T$，以及一个列表$n$其他具有已知运行时间的程序。 我们被要求选择一个整数乘数$k \ge 2$这样的时间限制$k \cdot T$足以完成列表中的每个程序，这意味着每个$t_i \le k \cdot T$。 在所有有效值中，我们必须输出尽可能小的值$k$。 

重写条件，每个程序都施加一个下限$k$：对于给定的$t_i$，我们需要$k \ge \frac{t_i}{T}$。 自从$k$必须是整数，真正的约束变为$k \ge \left\lceil \frac{t_i}{T} \right\rceil$。 因此，最终答案是这些每个程序要求的所有程序的最大值，并附加以下约束：$k \ge 2$。 

限制条件$n \le 10^5$和$t_i \le 10^9$建议我们必须在线性时间内计算答案。 任何涉及排序的方法都是不必要的，并且会比所需的慢，但仍然可以接受； 然而，一次通过就足够了。 

当所有程序与其他程序相比都非常快时，就会出现一个微妙的情况$T$。 然后每个比率都小于 1，因此计算出的最大值变为 1，但问题明确强制$k \ge 2$。 例如，如果$T = 1000$和所有$t_i \le 500$，即使所有程序都已通过，正确答案是 2$k = 1$。 

当某些情况发生时，会出现另一种边缘情况$t_i$远大于$T$，这可以将答案推至$10^9$规模。 由于乘法在 64 位整数范围内，因此在 Python 中是安全的，但在固定宽度语言中需要小心。 

## 方法

 思考问题的一个直接方法是尝试所有可能的值$k$，从2开始，检查所有程序是否满足$t_i \le kT$。 对于每个$k$，这需要扫描所有$n$程序，因此每次检查成本$O(n)$。 在最坏的情况下，$k$可以长到大约$10^9$，这使得这种方法完全不可行。 

关键的观察是可行性是单调的$k$。 如果某个$k$有效，然后任何更大的$k$也有效。 这将问题简化为寻找最小整数$k$满足一组下界，可以直接从每个程序独立导出。 

每个程序都有一个要求$k \ge \lceil t_i / T \rceil$。 而不是搜索一遍$k$，我们直接计算这些要求并取其最大值。 这将问题从间隔内的搜索压缩为单遍聚合。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解 k |$O(n \cdot k)$|$O(1)$| 太慢了|
 | 直接最大比率 |$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 阅读$n$和$T$。 这些定义了所有比较的缩放基线。 
2. 初始化变量$k\_req = 2$，因为无论数据如何，该问题都强制要求最小允许乘数为 2。 
3. 每个节目时间$t_i$，计算使其通过的最小乘数。 这是$\left\lceil \frac{t_i}{T} \right\rceil$。 
4. 更新$k\_req = \max(k\_req, \lceil t_i / T \rceil)$。 这确保了迄今为止看到的所有约束仍然得到满足。 使用最大值是因为每个程序独立地约束相同的全局$k$。 
5.处理完所有程序后，输出$k\_req$。 

### 为什么它有效

 每个程序都施加独立的不平等$kT \ge t_i$，可以重新排列为$k \ge t_i / T$。 由于所有约束必须同时成立，$k$必须满足他们之中最强者的要求。 上限将分数要求转换为整数可行性约束。 对所有程序取最大值恰好保留了最严格的约束，因此最终值既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, T = map(int, input().split())
    arr = list(map(int, input().split()))
    
    k_req = 2
    
    for t in arr:
        # compute ceil(t / T)
        k = (t + T - 1) // T
        if k > k_req:
            k_req = k
    
    print(k_req)

if __name__ == "__main__":
    main()
```实现的核心是整数上限计算`(t + T - 1) // T`，这避免了浮点运算并确保大值的正确性。 初始化为 2 至关重要，因为即使所有比率都建议为 1，约束条件也不允许这样做。

import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline
    
    n, T = map(int, input().split())
    arr = list(map(int, input().split()))
    
    k_req = 2
    for t in arr:
        k_req = max(k_req, (t + T - 1) // T)

 ## 边缘情况

 当所有程序时间远小于$T$，计算出的上限始终为 1。例如：```
2 100
1 2
```该算法计算$k = 1$对于两者，但由于初始化是$k\_req = 2$，最终输出仍然是 2。循环永远不会覆盖它，这正确地强制了问题的约束。 

当一个程序非常大时：```
2 10
1 100000
```计算过程如下。 第一个值给出$k = 1$， 所以$k\_req = 2$。 第二个值给出$k = 10000$， 所以$k\_req$变为 10000。这表明单个约束完全决定了答案，之前的值仅作为下限。
