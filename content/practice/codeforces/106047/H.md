 ## 解决方案
 ## 问题理解

 我们得到一个无向图，其中每条边带有 60 位权重。 两个顶点之间的路径不是通过求和或最小化权重来评估的，而是通过对该路径上的所有边权重进行按位与计算。 这意味着仅当路径上的每条边都设置了该位时，路径值中的每个位才会保留。 

对于每个查询的顶点对，我们必须确定它们之间是否存在按位与值至少为给定阈值的路径$V$。 “至少”在这里很重要，因为我们将最终的 AND 结果作为整数进行比较，而不是孤立地逐位进行比较。 

关键的结构性限制是$n$和$q$都可以达到$10^5$，边数可以达到$5 \times 10^5$。 任何试图探索每个查询的路径，甚至每个查询运行类似最短路径的搜索的方法都会太慢。 即使每个查询只有一个 BFS 也会导致$O(q(m+n))$，这是完全不可行的。 该解决方案必须对图表进行预处理，以便在几乎恒定的时间内回答每个查询。 

当两个顶点之间存在边但在 AND 约束下不存在有效路径时，就会出现微妙的失败情况。 例如，如果$V = 6$我们有优势$1-2$重量为 7 且$2-3$权重为 1，那么即使$1$连接到$3$在底层图中，路径$1 \to 2 \to 3$具有 AND 值$7 \& 1 = 1$，即下面的$V$，所以正确答案是“否”。 

另一个边缘情况是当$V = 0$。 在这种情况下，每条路径都满足条件，因为任何 AND 结果始终至少为 0。因此答案简化为完整图中的标准连接。 

## 方法

 暴力破解的想法是独立处理每个查询。 对于查询$(u, v)$，我们尝试从中找到任何路径$u$到$v$并计算沿该路径的边缘的按位与。 跟踪当前 AND 值的朴素 DFS 或 BFS 会在所有可能的路径上进行分支。 在最坏的情况下，图中简单路径的数量是指数级的，甚至限于 BFS 状态，我们需要跟踪以下形式的对$(node, current\_and)$，这会导致状态空间爆炸。 这很快就变得不可能了$q = 10^5$。 

关键的见解来自于理解按位 AND 在路径上的行为方式。 一旦某个位在任何边缘变为 0，它就永远不会在以后的边缘中返回。 这意味着最终的 AND 至少是$V$，设置的每一位$V$必须在路径中的每一个边缘上生存。 即使路径上的一个边缘丢失了所需的位，该位也会永久丢失。 

这将问题从路径优化问题变成了简单的边缘过滤问题。 我们只关心与以下内容兼容的边$V$，表示其权重包含中设置的所有位的边$V$。 一旦我们丢弃所有不兼容的边，剩余图中的任何路径都会自动具有至少 AND 值$V$，因为每条边都保留了所有必需的位。 

因此，每个查询都简化为过滤图中的纯粹连接性检查。 这可以使用一次构建的不相交集并集结构来有效地处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 每个查询呈指数 | O(n + m) | 太慢了|
 | 最佳（DSU 过滤）| O(m α(n) + q) | O(n) | 已接受 |

 ## 算法演练

 ### ## 算法演练

 1. 读取所有边，通过检查if判断每条边是否可用$(w_i \& V) = V$。 

该条件确保了每个所需的位$V$存在于边缘权重中。 
2. 初始化一个不相交的并集结构，其中每个顶点都从​​其自己的组件开始。 
3. 对于满足条件的每条边，合并其在 DSU 中的端点。 

这仅使用有效边构建连接的组件。 
4. 对于每个查询$(u, v)$，检查是否$u$和$v$属于同一个DSU组件。 

如果是，则输出“Yes”，否则输出“No”。 

### 为什么它有效

 考虑完全由满足以下条件的边形成的任何路径$(w \& V) = V$。 对于设置的每一位$V$，沿路径的每个边都设置了该位，因此该位在整个 AND 操作中仍然存在。 因此路径值始终包含$V$，这意味着至少是$V$。 

相反，如果一条路径甚至使用了一个违反条件的边沿，则至少需要的一位将被永久删除，从而使 AND 结果无法达到$V$。 因此，每条有效路径都必须完全位于过滤后的图中，并且该图中的连通性准确地表征了答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n + 1))
        self.size = [1] * (n + 1)

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]

def main():
    n, m, q, V = map(int, input().split())
    dsu = DSU(n)

    for _ in range(m):
        u, v, w = map(int, input().split())
        if (w & V) == V:
            dsu.union(u, v)

    out = []
    for _ in range(q):
        u, v = map(int, input().split())
        out.append("Yes" if dsu.find(u) == dsu.find(v) else "No")

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    main()
```DSU结构纯粹用于在过滤边缘后压缩连接信息。 路径压缩可确保每次操作的平摊时间接近恒定，这是必须放弃的$10^5$查询。 

关键的实现细节是边缘过滤条件`(w & V) == V`。 这比检查更严格`w >= V`，这是不正确的，因为数字排序不保留位包含。 只有按位包含才能保证 AND 下的稳定性。

 ## 边缘情况

 当$V = 0$，条件$(w \& V) = V$对于每条边总是成立，因为两边都为零。 因此，DSU 根据完整图进行合并，并且每个查询都简化为标准连接，这正确匹配了任何 AND 结果始终至少为零的事实。 

当图按原始意义连接但过滤器下不存在有效边时，每个节点都会变得孤立。 对于像这样的输入$V = 63$并且所有边都具有随机低位，DSU 不会合并任何顶点，因此即使原始图中直接相连的节点之间的查询也返回“否”，符合有效路径中的每条边必须满足位约束的要求。 

当多条边连接同一对节点时，它们被独立处理。 如果至少有一个满足条件，DSU 会将它们合并，但由于联合运算的幂等性，重复合并不会影响正确性。
