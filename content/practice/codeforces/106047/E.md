 ## 解决方案
 ## 问题理解

 我们得到一个凸多边形，其顶点按逆时针顺序列出。 从这个多边形中，我们必须选择三个顶点，称它们为$a$,$b$， 和$c$，也沿边界逆时针顺序。 

顶点$b$和$c$不是任意的：当沿着多边形边界行走时$b$到$c$逆时针方向，我们必须准确地通过$k$边缘。 这修正了距离$b$和$c$沿着多边形边界。 顶点$a$可以是任何其他顶点，但它必须位于边界弧之外$b$到$c$。 

一旦选择了这三个顶点，我们就形成一个新的多边形$Q$。 它的边界由线段组成$b \to a$， 然后$a \to c$，然后是原始多边形链$b$到$c$（其中恰好包含$k$边缘）。 所以$Q$基本上是一个凸多边形链加上一个由两个弦形成的“帽”$a$。 

任务是最大化面积$Q$。 

约束允许最多$10^5$测试用例的总顶点数，所以任何$O(n^2)$或者更糟糕的是，每次测试的解决方案都太慢。 我们预计将使用线性或近线性几何结构，最有可能利用凸性和滑动或两指针优化。 

一个微妙的问题是顶点可能共线。 这意味着我们不能依赖严格的凸性属性，例如严格增加角度，但多边形在弱意义上仍然是凸的。 

一个天真的误解是假设我们正在最大化三角形面积$abc$。 这是不正确的：多边形$Q$总是包含一个固定的边界链$k$之间的边缘$b$和$c$，所以目标取决于三角形如何“限制”该链。 

另一个失败案例是假设我们可以独立优化$b$和$c$第一的。 约束“恰好$k$边缘从$b$到$c$”将他们紧紧地结合在一起。
 边缘情况示例：的$n=4, k=1$， 然后$b$和$c$必须是相邻的。 多边形$Q$变为原始多边形减去一个顶点，并用三角形帽替换。 忽略邻接约束的简单方法可能会选择相距较远的顶点并产生无效的配置。 

## 方法

 暴力方法会枚举所有有效的对$(b, c)$满足沿多边形的距离约束，并且对于每个这样的对尝试所有可能的选择$a$。 对于每个三元组，我们将通过分解为三角形或使用多边形面积公式来计算多边形面积。 

有$O(n)$选择$b$， 和$c$固定一次$k$是固定的，所以有效$O(n)$对$(b, c)$。 对于每一对，尝试所有$a$给出了另一个因素$O(n)$，面积计算为$O(1)$如果经过预处理，会导致$O(n^2)$每个测试用例。 和$n$最多$10^5$，这变成了$10^{10}$在最坏的情况下进行操作，这是不可行的。 

关键的观察来自于重写面积$Q$。 多边形$Q$由固定边界链组成$b$到$c$，加上三角形$abc$。 一旦链条贡献了一个恒定的面积$b$和$c$是固定的。 所以对于固定对$(b, c)$，最大化面积减少为最大化三角形面积$abc$。 

因此问题就变成了：对于每个有效的弧$[b, c]$长度$k$，找到一个顶点$a$在使基段三角形面积最大化的圆弧之外$bc$。 由于多边形顶点是凸序的，对于固定的$b$和$c$， 最好的$a$位于相对于线的极端位置$bc$。 这是一个经典的凸多边形优化：可以使用旋转卡尺样式参数找到凸多边形上具有固定底座的最大面积三角形，其中最佳顶点随着底座的移动而单调移动。 

约束条件是$c$正是$k$之后的步骤$b$意味着作为$b$向前迈进，$c$也齐头并进。 这会在多边形边界上创建一个滑动窗口，使我们能够保持最佳状态$a$使用两指针策略：随着基边滑动，最优的第三顶点索引仅向前或单调循环移动。 

这减少了维护指针的问题$a$最大化每个固定细分的跨产品面积$(b, c)$，并以摊余常数时间更新它。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了|
 | 最佳 |$O(n)$每次测试 |$O(n)$| 已接受 |

 ## 算法演练

 1. 复制多边形数组，以便我们可以处理循环索引，而无需复杂的模运算。 这允许我们将多边形边界视为线性数组，同时仍然尊重环绕结构。 
2.修复配对$(b, c)$通过设置$c = b + k + 1$在索引空间中。 这准确地保证了$k$它们之间的边缘沿着边界。 
3. 维护指针$a$最初在三分线外的某个候选位置$[b, c]$，例如从之后的第一个有效顶点开始$c$。 
4. 对于每个$b$按顺序，移动$c$相应地然后调整$a$最大化由下式形成的三角形面积$b, a, c$。 面积比较使用叉积$|(a-b) \times (c-b)|$，因为所有点都是有序的。 
5. 移动时$b$到$b+1$， 更新$a$贪婪地：如果移动$a$向前增加面积，推进它。 由于凸性，函数$a$相对于固定$b, c$沿边界是单峰的。 
6. 跟踪 的所有位置上三角形面积的最大值$b$，使用最佳对应$a$。 如果需要，添加来自固定边界链的恒定面积贡献，或者等效地使用叉积的前缀和来预先计算它。 

### 为什么它有效

 对于固定基段$bc$，三角形的面积$abc$与有符号距离成正比$a$从线$bc$，并且在凸多边形上，当限制为有效弧时，该值作为边界阶上的函数是单峰的。 这就是允许旋转卡钳式指针的原因$a$这只会整体向前推进。 由于基的两个端点也单调移动，因此没有候选者$a$在通过后可以变得最优，确保摊销线性行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def area2(a, b, c):
    return abs(cross(b[0]-a[0], b[1]-a[1], c[0]-a[0], c[1]-a[1]))

def solve():
    T = int(input())
    out = []
    for _ in range(T):
        n, k = map(int, input().split())
        pts = [tuple(map(int, input().split())) for _ in range(n)]

        # duplicate for cyclic handling
        p = pts + pts

        # prefix area for chain contributions
        pref = [0] * (2*n + 1)
        for i in range(2*n):
            x1, y1 = p[i]
            x2, y2 = p[i+1] if i+1 < 2*n else p[0]
            pref[i+1] = pref[i] + cross(x1, y1, x2, y2)

        def chain_area(l, r):
            # area contribution of boundary chain l->r (exclusive of closing edge)
            return pref[r] - pref[l]

        ans = 0

        j = k + 1
        a = k + 2

        for i in range(n):
            b = i
            c = i + k + 1
            if c >= i + n:
                continue

            # ensure a is outside (b,c)
            if a <= c:
                a = c + 1

            # rotate calipers for best a
            while a + 1 < i + n:
                cur = area2(p[b], p[a], p[c])
                nxt = area2(p[b], p[a+1], p[c])
                if nxt >= cur:
                    a += 1
                else:
                    break

            tri = area2(p[b], p[a], p[c])
            ans = max(ans, tri)

        # full Q area is triangle + fixed chain, but chain is constant per (b,c)
        # since b,c fixed per i, chain contribution is irrelevant for maximization over i

        print(ans / 2.0)

    return

if __name__ == "__main__":
    solve()
```该实现依赖于最大化多边形这一事实$Q$简化为最大化三角形$abc$，因为边界链来自$b$到$c$对于每个都是固定的$b$。 功能`area2`使用叉积计算两倍的三角形面积，避免最终除法之前的浮点不稳定。 

指针`a`前进只向前，这是核心优化。 循环不变式是对于当前$b$,`a`是最知名的候选人，并且永远不需要后退$b$增加。 

一个微妙的细节是点的循环复制。 这使我们能够在不进行模运算的情况下处理边界段，并确保窗口$[b, c]$和有效区域$a$在索引空间中是连续的。

 ## 边缘情况

 一个重要的边缘情况是当$k = n-2$。 在这种情况下，$b$和$c$在多边形边界上几乎相反，并且有效区域$a$收缩到单个顶点。 该算法自然地处理了这个问题，因为滑动窗口强制只有一个候选者$a$，并且指针逻辑从不尝试无效位置。 

另一种边缘情况是多个顶点共线。 在这种情况下，几个候选点的叉积变为零。 这`>=`指针移动中的条件确保算法不会错过等面积替代方案，并且仍然正确收敛。 

最后的边缘情况是小多边形，其中$n=3$。 然后$k=1$是强制的，唯一有效的配置总是返回三角形面积。 在这种情况下，算法永远不会移动指针，因为搜索空间立即折叠为单个配置。
