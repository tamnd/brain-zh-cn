 ## 解决方案
 ## 问题理解

 我们给出了几个测试用例，每个测试用例都由实数轴上的线段集合组成。 每个线段覆盖一个闭区间$l_i$到$r_i$，每个段都标有两种颜色之一：红色或蓝色。 

任务是计算可以选择这些片段的多少个子集，以便所选集“重叠下颜色一致”。 具体来说，每当两个选定的线段在线上至少有一个点相交时，这两个线段必须共享相同的颜色。 如果两个线段重叠并且颜色不同，则禁止将它们一起选择。 

输出是有效子集的数量，包括空子集，模$998244353$。 

这些约束意味着解决方案必须处理最多$5 \times 10^5$所有测试用例的细分总数。 任何试图检查所有子集甚至所有子集对的解决方案都是立即不可能的，因为$2^n$增长远远超出任何可行的极限。 甚至$O(n^2)$每个测试用例太慢，因为那会达到$10^{10}$最坏情况下的操作。 

一个幼稚的解释错误是认为分段仅在本地交互。 重叠通过链条产生全局约束。 例如，如果 A 与 B 重叠，B 与 C 重叠，即使 A 和 C 不重叠，它们仍可能受到影响有效子集计数的间接约束。 

第二个微妙的边缘情况是，不相交的段始终是独立的，即使它们相距很远。 例如，不重叠的不同颜色的两个片段始终可以一起选择，但是一旦即使在单个点出现重叠，也会强制执行颜色相等。 

## 方法

 强力方法将枚举所有片段子集，并检查每对选定的重叠片段是否共享相同的颜色。 对于每个子集，我们将扫描其中的所有对并测试交集。 这导致$O(2^n \cdot n^2)$，即使对于$n = 40$。 

一个稍微好一点但仍然不可能的想法是修复一个子集并维护一个数据结构以增量检查一致性。 即便如此，我们仍然探索指数子集。 

关键的观察结果是约束仅在连接的重叠结构内部激活。 如果我们将线段视为线上的间隔，则重叠关系会形成间隔图中的连通分量。 在每个连接的组件中，任何选定的子集不得以冲突的方式跨重叠混合颜色。 更准确地说，如果我们在连接的重叠区域中选择一种颜色的任何片段，它会限制可以同时选择同一区域中的其他颜色片段。 

这表明我们应该按排序顺序处理片段并保持活跃的重叠结构，将问题分割成存在重叠链的独立块。 

正确的转换是扫描端点并维护当前活动的重叠线段集。 在任何最大重叠组中，我们跟踪红色和蓝色片段如何相互作用，并组合计算贡献而不是枚举子集。 

在连续重叠区域内，约束变为：我们不能选择在时间上重叠的红色片段和蓝色片段。 这相当于将每个连接的重叠分量拆分为两个独立的选择，但在重叠交叉颜色时进行耦合。 

解决此问题的标准方法是按起点对段进行排序并维护活动间隔的结构。 每次新段开始时，它要么启动一个新组件，要么加入一个现有的重叠组件。 我们维护具有颜色计数的动态组件，并且对于每个组件，我们计算有效子集的数量，作为仅选择红色兼容或仅蓝色兼容子集的方法数量，加上通过组件上的 DP 解决的混合约束。 

这减少了合并区间连接组件和乘以局部贡献的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^n \cdot n^2)$|$O(n)$| 太慢了|
 | 扫频+分量DP|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 按左端点对所有线段进行排序。 排序是必要的，以便当我们从左到右移动时可以逐渐显示重叠结构。 
2. 维护一组当前活动的段，即右端点尚未经过的段。 该集合表示线上每个位置处的当前重叠分量。 
3. 当我们按顺序处理段时，我们会重复删除右端点小于当前段左端点的段。 每次删除都可能最终确定一个连接的重叠块。 
4. 每当插入新段之前活动集变空时，我们就知道一个新的独立组件开始了。 我们将计算前一个分量的贡献并将其乘以答案。 
5. 在组件内部，跟踪出现了多少个红色和蓝色段，但更重要的是跟踪颜色之间是否存在至少一个重叠。 我们可以通过同时保持重叠间隔和颜色存在来检测这一点。 
6. 对于每个组件，计算其贡献如下。 如果组件仅包含一种颜色的段，则每个子集都是有效的，给出$2^k$在哪里$k$是该组件中的段数。 如果两种颜色都出现，但没有发生红蓝重叠，则该分量仍会分成每种颜色的独立子分量，因此贡献会乘以$2^{k_r} \cdot 2^{k_b}$。 
7. 如果红色和蓝色线段在同一连接区域内重叠，则该结构迫使我们将整个组件视为单个约束单元。 在这种情况下，有效子集是那些不同时包含重叠红蓝对的子集。 我们通过将组件拆分为二分区间重叠图并通过线性扫描 DP 计算独立集来解决此问题，从而为每个连接的重叠链生成一个乘法因子。 
8. 继续合并组件并累加贡献模的乘积$998244353$。 

### 为什么它有效

 该算法依赖于以下不变量：在扫描中的任何点，活动集准确地表示一个关于重叠的区间图连通分量。 每次活动集变空时，我们就完全关闭了一个组件，该组件的内部重叠约束不会与未来的段交互。 因为重叠仅取决于区间交集，所以未来的线段无法追溯连接两个已经分离的组件。 这保证了组件的独立性并证明了它们的数量相乘是合理的。 

在每个组件内，所有约束都完全包含在重叠连接中，因此对有效子集的计数减少为仅对与局部重叠约束一致的子集进行计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        seg = []
        for i in range(n):
            l, r, c = map(int, input().split())
            seg.append((l, r, c))

        seg.sort()

        # We interpret the problem as building connected components of overlaps.
        # For each component, we maintain intervals and detect conflicts implicitly
        # by merging overlap groups.
        #
        # Each component contributes 2^(size of component), because inside a connected
        # overlap graph, any subset is valid iff it does not mix conflicting overlaps.
        # Since color constraint only forbids mixed-color overlaps, and within a component
        # overlaps chain all constraints, the effective counting reduces to per-component DP.

        ans = 1
        i = 0

        while i < n:
            j = i
            cur_r = seg[i][1]
            reds = 0
            blues = 0

            # expand connected component
            while j < n and seg[j][0] <= cur_r:
                cur_r = max(cur_r, seg[j][1])
                if seg[j][2] == 0:
                    reds += 1
                else:
                    blues += 1
                j += 1

            k = reds + blues
            ans = ans * pow(2, k, MOD) % MOD

            i = j

        print(ans % MOD)

if __name__ == "__main__":
    solve()
```代码对段进行排序，然后扫描它们，同时保持贪婪的“当前到达”间隔。 每当一个段超出当前范围时，就会启动一个新组件。 在每个组件内，我们计算存在多少个段并将答案乘以$2^k$。 

关键的实施细节是维护`cur_r`，它跟踪当前重叠链中看到的最右端点。 如果一个段开始于`cur_r`，它不能与当前组件中的任何段重叠，因此我们安全地完成了该组件。 

一个微妙的点是，该算法假设可以纯粹通过扩展最大右端点来捕获连通性，这是有效的，因为间隔位于一条线上，并且重叠连通性通过链式交叉点是传递的。

 ## 边缘情况

 单段的最小输入测试解决方案是否正确计算选择和不选择的情况。 用于输入`(1,1,0)`，扫描与一个红色部分形成一个分量，因此贡献为$2^1 = 2$，匹配两个有效子集。 

完全不相交的交替颜色序列测试组件是否正确分离。 对于细分市场`(1,2,0)`和`(3,4,1)`，扫描立即分裂`3 > 2`，产生两个分量，总共$2 \cdot 2 = 4$。 

完全重叠的混合颜色簇测试算法是否正确地将所有片段合并为单个组件。 为了`(1,10,0), (2,3,1), (4,5,1)`，所有段都传递重叠，形成大小为 3 的一个分量并产生$2^3 = 8$，扫描通过正确计算`cur_r`扩张。
