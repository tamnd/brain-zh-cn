 ## 解决方案
 ## 问题理解

 我们有一组放置在数轴上的士兵。 每个士兵都有固定的位置和具有一定威力的武器。 当怪物出现在某个位置时，所有士兵都会向该位置射击。 子弹只有在从士兵到怪物的距离后仍然有足够的剩余威力时才能贡献命中。 子弹行进的距离越远，它的线性强度就越弱。 

对于放置在该位置的怪物$d$与屏蔽宽度$s$, 一名士兵在阵地$a_i$有力量$p_i$如果距离惩罚没有将其威力降低到护盾要求以下，则成功贡献一击。 这个条件简化为不等式$$|a_i - d| \le p_i - s.$$因此对于任何固定的屏蔽宽度$s$，以及任何位置$d$，我们可以统计还有多少士兵还能击中怪物。 怪物在该位置存活$d$如果这个计数最多是$k$。 目标是选择尽可能小的整数$s$这样无论怪物站在数轴上的哪个位置，命中的次数都不会超过$k$。 

这些限制因素推动$O(n \log n)$或者$O(n)$每个测试用例的解决方案。 高达$10^5$所有测试用例中的总士兵数，任何平方数$n$会太慢，因为它需要最多$10^{10}$最坏情况下的操作。 

一些微妙的边缘情况立即出现。 如果所有士兵的力量为零，那么任何正的护盾宽度都会轻易地阻止所有攻击，但答案仍然必须是非负的。 如果$k = n$，那么怪物就可以被所有士兵击中，所以即使是零护盾也可能足够了。 如果士兵密集地聚集在同一点且威力较高，那么甚至可能需要大盾牌来减少重叠间隔。 

最重要的隐藏困难是没有询问我们固定的位置$d$，但关于最坏的可能$d$，这迫使我们推理全局重叠结构而不是逐点评估。 

## 方法

 思考问题的一个直接方法是固定一个屏蔽宽度$s$然后模拟每一个可能的怪物位置。 对于每个位置$d$，我们会计算有多少士兵满足$|a_i - d| \le p_i - s$。 这相当于检查每个点覆盖了多少个区间，因为每个士兵都定义了一个以$a_i$有半径$p_i - s$。 蛮力方法将评估之间所有整数位置的覆盖范围$1$和$10^9$，这显然是不可行的。 

即使我们限制自己只检查关键点，重新计算每个点的覆盖范围$s$仍然需要花费$O(n^2)$在最坏的情况下，因为每次检查都涉及扫描所有士兵。 

关键的观察是对于固定的$s$，每个士兵贡献一个对称区间：$$[a_i - (p_i - s),\ a_i + (p_i - s)].$$我们要确保没有任何点位于超过$k$这样的间隔。 这是一个经典的“最小最大重叠”问题。 

而不是修复$s$并检查重叠，我们反转视角：我们问最小值是多少$s$使得这些收缩区间的最大重叠最多为$k$。 作为$s$增加，间隔均匀缩小，因此重叠只能减少。 这种单调性允许二分搜索$s$。 

对于固定候选人$s$，我们计算所有间隔、排序端点和扫描线以计算最大重叠$O(n \log n)$。 然后我们二分查找最小的$s$满足条件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力（模拟所有位置）|$O(n \cdot 10^9)$|$O(n)$| 太慢了|
 | 检查+二分查找+扫线|$O(n \log n \log P)$|$O(n)$| 已接受 |

 这里$P$是最大功率范围。 

## 算法演练

 ### 1. 将每个士兵解释为一个缩小的区间

 对于固定屏蔽$s$，计算每个士兵的有效半径$r_i = p_i - s$。 如果$r_i < 0$，该士兵根本不贡献任何间隔。 否则它会覆盖该段$[a_i - r_i, a_i + r_i]$。 这将问题转换为间隔重叠计数问题。 

这种变换起作用的原因是，原始条件正是点位于以中心为中心的半径内的条件。 

### 2. 定义防护罩的可行性

 我们定义一个函数`check(s)`如果线上没有点被超过的覆盖，则返回 true$k$间隔。 这直接符合怪物在任何地方都能生存的要求。 

单调行为至关重要：增加$s$只会减少所有$r_i$，所以间隔只会缩小，重叠不会增加。 

### 3. 计算固定的最大重叠$s$评估`check(s)`，我们生成所有有效间隔并将它们转换为事件：左端点处+1，右端点后-1。 对这些事件进行排序允许进行维护每个坐标处的活动间隔数量的扫描。 扫描期间的最大值是同时命中的最大数量。 

我们将这个最大值与$k$。 

### 4.二分查找最小有效值$s$由于可行性是单调的，我们二分搜索$s$从 0 到$\max p_i$。 每个步骤都会运行扫描线检查。 

### 为什么它有效

 每个士兵定义一个区间族，这些区间均匀收缩为$s$增加。 函数“区间最大重叠”是单调非增的$s$，所以有效的集合$s$value 是整数的后缀。 二分查找正确找到最小的有效值。 扫描线正确捕获最坏情况的重叠，因为最大覆盖范围的任何点都必须出现在该构造中的间隔端点处。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(s, a, p, k):
    events = []
    for ai, pi in zip(a, p):
        r = pi - s
        if r < 0:
            continue
        l = ai - r
        rgt = ai + r
        events.append((l, 1))
        events.append((rgt + 1, -1))
    
    events.sort()
    
    cur = 0
    best = 0
    for x, v in events:
        cur += v
        if cur > best:
            best = cur
    
    return best <= k

def solve():
    t = int(input())
    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))
        p = list(map(int, input().split()))
        
        lo, hi = 0, max(p)
        ans = hi
        
        while lo <= hi:
            mid = (lo + hi) // 2
            if check(mid, a, p, k):
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1
        
        print(ans)

if __name__ == "__main__":
    solve()
```核心结构是围绕可行性检查的二分搜索。 这`check`函数构建间隔事件并运行扫描线来计算最大重叠。 

一个微妙的点是使用`rgt + 1`闭幕式。 这确保了正确的整数覆盖语义：恰好位于右端点的点仍然包含在间隔中。 

另一个重要的细节是跳过士兵在哪里$p_i - s < 0$，因为他们无法提供任何有效的覆盖范围。

 ## 边缘情况

 当所有的幂都为零时，每个士兵都会为任何正的半径产生空的或负的半径间隔$s$。 扫描线没有接收到任何事件，因此最大重叠为零，并且算法正确接受$s = 0$。 

什么时候$k = n$，每个配置即使在$s = 0$。 检查函数计算的最大重叠永远不会超过$n$，所以二分查找立即收敛到零。 

当士兵都处于同一位置且威力较大时，初始重叠为$n$在$s = 0$。 检查功能正确报告故障，直到$s$减少所有半径足以使间隔缩小和分裂，减少下面的重叠$k$。
