 ## 解决方案
 ## 问题理解

 我们正在处理一个由奇数学生组成的教室，每个学生都秘密地持有一个非零整数分数。 恰好有一个学生的分数最高，而且这个分数严格高于其他所有人。 

获取信息的唯一方法是询问一名学生另一名学生的分数。 当学生时`i`被问及学生的情况`j`，回复要么是真实的，要么可能会翻转符号，具体取决于是否`i`是“诚实”还是“不诚实”。 不同的是，诚实与学生是否`i`有一个积极的分数。 积极的学生总是正确地报告。 消极的学生可能会颠倒他们所报告内容的符号。 

这创建了一个部分观察系统：查询不同的来源给出了可能不一致的答案，并且可靠性取决于未知的迹象。 唯一保证的全局结构事实是数组中的中位数分数为正，这意味着超过一半的学生具有正值。 

任务是最多使用约来识别唯一最大元素的索引`1.5n`查询。 

限制条件`n ≤ 1000`意味着任何解决方案都可以提供线性或近线性交互策略，但任何二次方都立即不可行，因为每个查询都是昂贵的外部操作。 我们不允许模拟所有对之间的比较。 

一个天真的误解是将每个答案都视为绝对的。 但这会立即失败，因为负面记者可以任意翻转标志，产生矛盾的比较。 当通过不同中介进行链接比较时，会出现另一个微妙的失败情况：单个负中介可以反转结果并使较小的值看起来大于最大值。 

例如，如果学生`i`为负数，则查询`i j`回报`-a[j]`而不是`a[j]`。 因此，根据询问的对象，较大的正值可能会显示为负值。 

核心挑战是在对抗性符号翻转的情况下提取一致的排序信号。 

## 方法

 暴力策略会通过仔细解决每个符号的歧义来尝试确定每个学生的真实价值。 人们可以在两个方向上重复查询每一对学生，并尝试推断哪些学生是积极的或消极的。 一旦确定了符号，我们就可以恢复实际值并选择最大值。 然而，可靠地解决符号一致性问题需要在许多对之间进行重复的交叉验证。 在最坏的情况下，这会变成二次方`n`，要求顺序为`n^2`查询次数远远超出了允许的限制。 

关键的观察是我们实际上不需要完整的排序，甚至不需要所有的符号。 我们只需要找到最大元素即可。 这表明使用在符号不确定性下稳定的比较来将问题简化为寻找“主要候选者”。 

当我们比较两个候选者时，就会出现一个有用的结构`a`和`b`通过询问第三个学生`k`。 如果`k`为正，我们得到真实的比较； 如果`k`为负数，两个值都会进行符号翻转，但它们的相对顺序保持一致。 这是关键的不变量：查询相同的内容`k`保留任意两个目标之间的顺序，即使`k`谎言。 

因此，如果我们固定足够大的“锚”集或重复使用精心选择的参考，我们可以执行类似于锦标赛的淘汰：两两比较候选者，丢弃较弱的，然后继续。 

中位数阳性保证确保在任何相当大的子集中，我们总是可以经常找到一个阳性响应者，足以保持比较的可靠性。 这使我们能够模拟确定性消除过程，其中无法消除真正的最大值。 

因此，我们将问题简化为线性锦标赛，其中我们维护当前的最佳候选者，并使用受控查询将其与其他候选者进行比较，确保每次比较都使用一致的参考，因此符号翻转不会扭曲结果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（解析所有值）| O(n²) 查询 | O(1) | O(1) | 太慢了 |
 | 通过一致的查询消除锦标赛 | O(n) 次查询 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们构建了一个流程，维护当前的最佳候选者，并使用精心构造的查询迭代测试其他学生。 

1. 选择一个初始候选人，例如学生`1`。 这仅作为起点，而不是可信的参考。 
2. 迭代所有其他学生`i`从`2`到`n`。 
3. 对于每个候选者当前最好的比较`best`和`i`，使用执行查询`best`作为响应者：`? best i`。 这将返回一个值，该值要么是 true`a[i]`如果`best`是正数，或者`-a[i]`如果`best`是负数。 
4. 同样，我们需要一条对称的信息来消除方向的歧义。 我们查询`? i best`，它返回`a[best]`或者`-a[best]`。 
5. 从这两个响应中，我们通过确保两者在相同的符号上下文下进行解释来比较一致的幅度。 关键事实是两个响应的乘积反映了之间的真实顺序`a[i]`和`a[best]`达到固定的符号一致性，因此我们可以决定哪个索引应保留为候选索引。 
6. 如果`i`被确定为大于当前`best`， 更新`best = i`。 
7. 继续，直到处理完所有学生。 
8. 输出`best`。 

这样做的原因是每次比较要么发生在一致的符号环境中，要么发生在完全对称的交换环境中。 在这两种情况下，两个比较值之间的相对顺序都会被保留。 中位数正保证确保我们不会陷入所有比较在步骤之间不一致反转的情况，因为大多数诚实的反应可以防止锦标赛中的对抗性漂移。 

### 为什么它有效

 在任何步骤中，该算法都会在查询引发的一致比较模型下维护一个永远不会比任何先前被丢弃的学生更差的候选者。 尽管个别答案可能会根据响应者的不同而发生符号翻转，但两个固定候选者之间的每次比较都是内部一致的，因为两者都是通过相同的确定性交互模式进行评估的。 这创建了对真实值的传递主导关系，确保真实的最大值永远不会被消除，并最终保留为最终候选值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(i, j):
    print("?", i, j)
    sys.stdout.flush()
    return int(input())

def solve():
    n = int(input())
    best = 1

    for i in range(2, n + 1):
        x = ask(best, i)
        y = ask(i, best)

        # If best is positive, x = a[i], y = a[best]
        # If best is negative, x = -a[i], y = -a[best]
        # So x and y always share the same sign transformation,
        # and comparing x vs y preserves true ordering.
        if x < y:
            best = i

    print("!", best)
    sys.stdout.flush()

if __name__ == "__main__":
    solve()
```该实现维护一个正在运行的最佳候选，并将其与每个新索引进行比较，每次比较仅使用两个查询。 关键的实现细节是我们从不尝试解释绝对值； 我们只比较配对的响应`(x, y)`它们总是由相同的未知符号因子进行转换，从而使比较有效。 

每次查询后都需要刷新，因为交互取决于立即响应传播。 缺少同花将会阻碍法官的判断。

 ## 边缘情况

 关键的边缘情况是当前最佳候选者是否定的。 在这种情况下，它的所有响应都会发生符号翻转，这可能会使大的真实值看起来更小。 对称查询`ask(i, best)`确保双方经历相同的转换，因此比较仍然有效。 

例如，如果`best = 2`是负数并且`i = 5`是真正的最大值：

 查询`? 2 5`产量`-a[5]`， 尽管`? 5 2`产量`a[2]`被否定了。 比较这两者仍然正确地表明`a[5] > a[2]`，所以算法正确更新`best`。 

这种对称性可以防止对抗性符号翻转破坏锦标赛结构。
