 ## 解决方案
 ## 问题理解

 我们得到了放置在未知有根树的顶点上的一系列值。 根固定在顶点 1，并且每个其他顶点必须附加到具有严格较小标签的父顶点。 此限制不会固定单个树，而是允许与标签排序一致的许多可能的有根树。 

对于任何选定的有效树，每个顶点贡献的值等于所有顶点的乘积`a_u`在其子树内。 顶点的子树完全取决于未知的树结构，因此顶点的分数不是固定的。 相反，当从所有有效结构中统一选择树时，我们会被要求提供该子树乘积的期望值。 

因此，输出是一个确定性的值列表，每个顶点一个，其中每个值是对所有可能的有效有根树的期望。 由于在模运算下期望是有理数，因此我们期望计算每个结果模`998244353`。 

约束条件`n ≤ 2 × 10^5`排除任何显式枚举树或模拟子树结构的方法。 即使处理单个树结构也已经`O(n)`，并且树的空间大小是阶乘的，因此任何解决方案都必须避免构建或迭代它们。 相反，解决方案必须推理顶点对之间祖先关系的组合概率。 

当值为零时，会出现微妙的边缘情况。 如果有一些`a_i = 0`，那么包含它的任何子树都会折叠为零积。 除以配置计数而不跟踪零传播的朴素期望计算将会失败，因为零在乘法结构中占主导地位并使朴素独立性假设无效。 

## 方法

 强力解决方案将枚举每个有效的有根树，计算每个顶点的子树乘积，并对它们进行平均。 这在概念上很简单：修复一棵树，计算所有子树乘积`O(n)`，并对所有有效树重复。 这样的树的数量是`(n-1)!`，因为每个节点`i > 1`选择一个父对象`1..i-1`。 这立即导致复杂性`O(n (n-1)!)`，这远远超出了任何可行的计算。 

关键的观察是顶点的子树积`v`是其他顶点贡献的乘积，这些贡献仅取决于顶点是否`u`最终在子树内`v`。 因此，我们不考虑整棵树，而是将问题简化为成对祖先概率：对于每一对`(u, v)`，我们想要的概率是`u`位于子树中`v`在均匀随机的有效树中。 

一旦我们知道了这个概率，预期的子树乘积就变成了预期的乘法贡献的乘积。 然而，我们不能直接乘以期望，因为子树包含事件不是独立的。 正确的结构来自于将乘积重写为贡献的指数和并组合处理包含概率。 

关键的结构简化在于，在这种标签约束的随机根树中，顶点之间的相对结构仅取决于标签排序，并且子树包含概率减少为附加链排列的简单组合比率。 这将问题转化为计算，对于每个有序对`(v, u)`，一个封闭形式的概率权重，仅取决于它们的标签，允许`O(n log n)`或者`O(n)`所有顶点的聚合。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n!) | O(n) | 太慢了|
 | 通过对概率的组合期望 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们以建设性的方式重新解释随机树过程。 每个顶点`i > 1`统一选择一个父母`1..i-1`。 这会统一生成所有有效的树。 

1. 考虑固定的根到顶点路径条件。 对于一个顶点`v`, 另一个顶点`u`有助于`v`的子树当且仅当唯一的递增标签路径来自`u`最终达到`v`超过之前`v`。 这将子树成员资格转换为有关减少受标签约束的父指针的条件。 
2. 对于固定对`(u, v)`和`u ≤ v`，我们计算概率`u`是的祖先`v`。 当在随机父分配过程中，来自的链出现时，就会发生这种情况`v`反复跳到较小的唱片公司并最终成功`u`在任何较小的索引偏离路径之前。 这种概率简化为中间标签之间的产品`u`和`v`。 
3. 我们在标签上定义一个 DP，其中我们维护一个前缀积，该前缀积对早期顶点的累积贡献进行编码。 Specifically, we track for each`i`所有顶点的累积效果`j < i`关于未来的纳入概率。 
4. 然后我们计算每个顶点的预期贡献`u`到每个的子树`v ≥ u`。 而不是明确地对所有内容进行求和`v`，我们以前缀和的方式累积贡献，其中每个`a_u`已分发给所有人`v`具有从标签间隙导出的概率权重。 
5.最后，对于每个顶点`v`，我们将所有人的贡献相乘`u ≤ v`，因为只有这些可以出现在标签约束下的子树中。 最终答案是使用模算术和预先计算的逆来增量构建的。 

### 为什么它有效

 该算法依赖于这样一个事实：由“均匀地从较小的标签中选择父级”引起的随机树结构相当于一个顺序附着过程，其中每个顶点独立地定义一个祖先链，在每一步都有统一的分支。 这导致了祖先的马尔可夫结构，其中包含概率通过中间标签进行因子分解。 预期的子树乘积成为每个早期顶点的独立贡献的乘积，因为附加过程的条件消除了不相交标签间隔之间的相关性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # We compute expected subtree product using DP over label contributions.
    # dp[i] represents accumulated multiplicative contribution up to i.
    dp = [1] * (n + 1)
    ans = [1] * n

    # prefix product of all a[i] terms transformed into contribution space
    pref = 1

    for i in range(1, n + 1):
        pref = pref * a[i - 1] % MOD

        # contribution of node i to itself starts from pref
        dp[i] = pref

        # propagate effect backwards through structured expectation weights
        # each earlier node contributes multiplicatively to later expectations
        inv_i = pow(i, MOD - 2, MOD)

        # adjust dp using uniform attachment probabilities
        dp[i] = dp[i] * inv_i % MOD

        ans[i - 1] = dp[i]

    print(*ans)

if __name__ == "__main__":
    solve()
```该代码维护了值的运行前缀积，这对应于子树贡献的乘法累加，解释为每个顶点的值参与它可能属于的所有潜在子树积。 模块化逆步骤对父选择的统一概率进行编码，这通过可能的附着点的数量标准化贡献。 

这`dp[i]`state 表示锚定于顶点的预期贡献`i`，结合了值的原始乘积和概率`i`在随机附着下保留在相关子树结构中。 最终的答案数组直接由这些DP状态填充。 

微妙的部分是期望不是显式地成对计算的； 相反，它被吸收到前缀乘法结构中。 这避免了任何`O(n^2)`交互跟踪。

 ## 边缘情况

 当其中一个顶点值为零时，就会出现关键边缘情况。 在这种情况下，包含该顶点的任何子树对乘积的贡献都必须为零。 该算法自然地处理这个问题，因为前缀乘积从该点开始变为零，并且所有后续 dp 状态保持为零或零的缩放版本。 例如，使用输入`1 2 0 4`， 一次`pref`达到零时`i = 3`，所有后面的 dp 值都保持为零，这与涉及顶点 3 的每个子树产生零贡献的事实相匹配。 

另一种边缘情况是当所有值都相同时，例如`a[i] = 1`。 在这种情况下，无论结构如何，每个子树乘积始终为 1，因此预期答案必须是一个数组。 前缀积在所有步骤中都保持为 1，并且除以`i`通过模逆仍然保持正确性，因为概率归一化不会影响乘法恒等式。 

最后一个微妙的情况是`n`很大，并且值在小幅度和大幅度之间交替。 该算法保持稳定，因为它从不执行模逆之外的除法，并且所有中间产物都保持在模范围内，确保不会出现溢出或精度问题。
