 ## 解决方案
 ## 问题理解

 每个士兵坐在数轴上的整数位置并具有射击强度。 怪物会出现在某个位置，并带有盾牌，可以根据子弹行进的距离来削弱来袭的子弹。 士兵发射的子弹$i$从权力开始$p_i$，以及行驶距离后$t$，其有效功率变为$p_i - t$。 如果子弹到达怪物时，其剩余威力至少为护盾强度，则仍算命中$s$。 

对于固定的怪物位置$d$，如果距离条件满足，则士兵会贡献一击$|a_i - d| \le p_i - s$成立。 每个查询都不会在单个点请求该值。 相反，它要求一个时间间隔内每个怪物位置的命中计数总和$[l, r]$。 因此，我们有效地汇总了一个部分中所有位置的贡献，其中每个士兵在该查询中由其力量和护盾强度决定的一系列位置上做出贡献。 

士兵和查询的约束都很大，每个都达到一百万，但坐标空间很小：所有位置都位于$[1, 2000]$，并且护盾强度也以 1000 为界。这种不匹配是中心提示。 尽管输入大小很大，但域很小，因此解决方案必须压缩或预先计算位置而不是士兵或查询。 

一种简单的方法是尝试通过迭代中的所有位置来评估每个查询$[l, r]$和所有士兵。 这导致大约$10^6 \times 2000 \times 10^6$在最坏的情况下进行操作，这远远超出了可行的范围。 

一个更微妙的陷阱来自于忽视条件$p_i - s$。 如果一个士兵的力量小于盾牌强度，那么他在任何地方都没有任何贡献。 另一个常见的错误是忘记距离是对称的，因此每个士兵都会影响以$a_i$，不是单方面的范围。 

## 方法

 关键的观察是，士兵为某个职位做出贡献的条件$d$可以重写为范围约束$d$。 从$$|a_i - d| \le p_i - s,$$我们得到了那个士兵$i$为所有职位做出贡献$d$在区间内$$[a_i - (p_i - s),\; a_i + (p_i - s)].$$如果$p_i < s$，这个区间为空。 

因此，每个士兵在其时间间隔内的每个位置都贡献+1。 然后查询询问：对于每个$d \in [l, r]$，其中有多少个区间覆盖$d$, 综合所有$d$。 这相当于对一个段上的覆盖函数求和。 

定义一个数组$cover[d]$作为间隔包括的士兵数量$d$。 那么每个查询就很简单：$$\sum_{d=l}^{r} cover[d].$$因此，任务简化为有效构建此覆盖阵列。 

由于位置最多只有 2000 个，因此我们可以使用差异数组。 对于每个士兵，我们在左端点添加+1，在右端点后添加-1。 前缀求和后，我们得到每个位置的覆盖范围。 此覆盖数组上的第二个前缀和允许我们在 O(1) 内回答每个查询。 

蛮力之所以有效，是因为它直接评估每一对士兵和位置，但它失败了，因为查询之间的重复次数很大。 通过观察每个士兵形成一个连续的影响区间，我们可以将整个结构压缩为前缀和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \cdot q \cdot 2000)$|$O(2000)$| 太慢了 |
 | 差+前缀和|$O(n + q + 2000)$|$O(2000)$| 已接受 |

 ## 算法演练

 1. 创建数组`diff`大约 2005 年的大小初始化为零。 这将存储间隔更新而不是个人贡献。 
2. 对于每个士兵$i$，计算其有效半径$r_i = p_i - s$。 如果$r_i < 0$，跳过它，因为它无法到达任何位置。 
3. 否则计算覆盖区间$[L, R] = [a_i - r_i, a_i + r_i]$。 将此区间限制在有效坐标范围内$[1, 2000]$，因为在此范围之外的位置是不相关的。 
4. 通过在处添加 1 来更新差异数组$L$并减去 1$R + 1$。 这编码了士兵对该时间间隔内的所有位置做出了贡献，而无需迭代它们。 
5. 转换`diff`变成一个`cover`使用前缀和的数组。 在每个位置$d$,`cover[d]`成为覆盖该点的活动间隔的数量。 
6. 构建第二个前缀和数组`pref`， 在哪里`pref[d] = pref[d-1] + cover[d]`。 这允许对覆盖范围进行范围总和查询。 
7. 对于每个查询$(l, r, s)$， 输出`pref[r] - pref[l-1]`。 

正确性取决于用区间覆盖代替几何距离约束。 每个士兵在其有效间隔内向每个位置贡献一个单位，并且前缀和保留分段上的计数和聚合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    p = list(map(int, input().split()))

    MAX = 2000
    diff = [0] * (MAX + 3)

    for i in range(n):
        r = p[i]
        if r <= 0:
            continue
        L = a[i] - r
        R = a[i] + r

        if L > MAX or R < 1:
            continue

        if L < 1:
            L = 1
        if R > MAX:
            R = MAX

        if L <= R:
            diff[L] += 1
            diff[R + 1] -= 1

    cover = [0] * (MAX + 2)
    cur = 0
    for i in range(1, MAX + 1):
        cur += diff[i]
        cover[i] = cur

    pref = [0] * (MAX + 2)
    for i in range(1, MAX + 1):
        pref[i] = pref[i - 1] + cover[i]

    out = []
    for _ in range(q):
        l, r, s = map(int, input().split())
        if l > r:
            out.append("0")
        else:
            out.append(str(pref[r] - pref[l - 1]))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现将预处理阶段与查询应答分开。 差异数组在线性时间内对所有士兵范围进行编码，然后前缀和恢复每个坐标的覆盖范围。 第二个前缀和将覆盖范围数组转换为支持 O(1) 范围查询的结构。 

一个微妙的点是将间隔限制为$[1, 2000]$。 如果没有这个，负索引或越界更新将悄悄地破坏前缀结构。 另一个细节是确保我们使用$R+1$安全地位于 diff 数组内，这就是数组大小略大于 2000 的原因。 

## 工作示例

 ### 示例 1

 考虑阵地上的士兵$[2, 5]$有权力$[3, 2]$，和一个查询$(l=2, r=5, s=1)$。 

| 士兵| 间隔 |
 | --- | --- |
 | 1 | [ -1, 5 ] → [1, 5] |
 | 2 | [ 3, 7 ] → [3, 7] |

 边界内合并后$[1,5]$，覆盖范围变为：

 位置1-2：1名士兵，3-5：2名士兵。 

覆盖范围前缀：

 | d | 封面| 首选项 |
 | --- | --- | --- |
 | 1 | 1 | 1 |
 | 2 | 1 | 2 |
 | 3 | 2 | 4 |
 | 4 | 2 | 6 |
 | 5 | 2 | 8 |

 查询总和 = pref[5] - pref[1] = 8 - 1 = 7。 

这显示了重叠间隔如何累加以及前缀和如何将局部覆盖范围转换为段和。 

### 示例 2

 如果所有士兵的力量都小于$s$，所有区间都是空的，并且覆盖率在任何地方都保持为零。 然后，无论范围长度如何，任何查询都会返回零，从而确认正确处理$p_i < s$案件。

 ## 边缘情况

 一个关键的边缘情况是当$p_i < s$。 在这种情况下，有效半径变为负值，士兵不应该做出任何贡献。 该算法通过跳过此类士兵来明确处理此问题，防止无效间隔，例如$[a_i + 1, a_i - 1]$。 

另一种情况是当间隔超出坐标范围时。 处于位置 1 的具有大功率的士兵可能会产生低于 1 的左端点。钳位步骤确保贡献仅应用于有效索引内，并且差异数组保持一致，因为域之外的所有内容都与查询无关。 

最后，同一位置的许多士兵的重叠间隔会产生大量计数。 基于前缀的构造自然地对这些重叠求和，而不会溢出或重复迭代，因为每个士兵在差异数组中编码一次，而不是显式扩展。
