 ## 解决方案
 ## 问题理解

 我们被要求为鲍勃构建任何有效的标记集$n$课程，每个分数是 0 到 2047 之间的整数，有一个额外的约束：所有分数的按位异或必须等于给定值$x$。 在所有此类有效作业中，我们希望最大化平均分。 自从$n$是固定的，最大化平均值与最大化所有分数的总和完全相同。 

所以真正的任务是纯粹的组合：选择$n$有限范围内的数字，强制它们的异或$x$，并使它们的总和尽可能大。 

上限 2047 至关重要。 这意味着每个数字都是 11 位整数，因此从位操作的角度思考是很自然的。 另外，自从$n \le 1000$并且有多达500个测试用例，任何解决方案都必须$O(t)$或者在最坏的情况下$O(t \log n)$。 任何涉及搜索的事情$n$- 大状态上的元组或动态编程会太慢。 

当尝试贪婪构造而不尊重全局 XOR 约束时，会出现微妙的失败情况。 例如，如果我们只是尝试将所有值设置为 2047，这会在本地最大化总和，但 XOR 会变得固定并且可能不匹配$x$。 独立调整多个元素的简单修复可能会意外地减少不必要的总和。 

另一个边缘情况是当$x = 0$。 那么当以下情况时，最优解可能就是所有值都等于 2047：$n$是均匀的，或者稍微调整一下$n$是奇数，取决于 XOR 中的奇偶校验效果。 

## 方法

 蛮力解释很简单，但毫无希望。 我们可以尝试所有$n$-length 值数组$[0, 2047]$，检查它们的异或是否等于$x$，并计算最大总和。 这探讨了$2048^n$可能性，即使对于$n = 2$，所以它立即不可行。 

关键的观察结果是，当每个元素尽可能大（即接近 2047）时，总和会最大化。因此，我们从每门课程都获得 2047 的配置开始，然后最小化修改足以修复 XOR 约束的值。 

设初始数组全为2047。其异或通过奇偶校验固定：if$n$是偶数，异或为0，否则为2047。我们将其与需要的异或进行比较$x$。 它们之间的不匹配是一个值$\Delta$，它代表我们需要“纠正”XOR 的程度。 

现在问题变成了：我们要调整一些元素远离2047，这样异或校正正好是$\Delta$，同时最小化损失总和。 如果我们将值从 2047 更改为某个值$a$，总和的减少量为$2047 - a$。 由于与 2047 的 XOR 行为类似于 11 位的按位求补，因此我们得到了一个清晰的恒等式：更改值的成本正是应用于该元素的 XOR 差值。 

所以我们将问题简化为表达$\Delta$作为一些选定的“差异值”的异或，同时最小化它们的总和。 关键的简化是，使用多次修正永远无法将成本降低到低于使用单次修正值的水平$\Delta$本身。 任何分解为多个 XOR 组件都无法在总成本方面击败直接表示。 

因此，最佳策略是应用单次尺寸修正$\Delta$，或者不执行任何操作，如果$\Delta = 0$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2048^n)$|$O(n)$| 太慢了 |
 | 最佳 |$O(1)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 1. 计算全 2047 数组的基本异或。 如果$n$是偶数，则为 0，否则为 2047。这代表在进行任何调整之前的最佳起始配置。 
2. 计算失配值$\Delta = \text{base\_xor} \oplus x$。 这是达到目标所需的精确异或校正。 
3. 计算调整前可能的最大总和，即$2047 \cdot n$。 这相当于为每门课程分配 2047。 
4. 如果$\Delta = 0$， 返回$2047 \cdot n$。 不需要调整，所以这已经是最佳的。 
5. 否则，减去$\Delta$求总和并返回结果。 这对应于执行修复 XOR 的单个最小成本校正。 

### 为什么它有效

 该构造之所以有效，是因为通过记录每个元素与 2047 的偏差量，任何有效的解决方案都可以转换为全 2047 基线。这些偏差的行为就像 XOR 组件一样，精确地组合成所需的校正$\Delta$。 由于按值更改元素$d$正好减少总和$d$，优化变成最小化 XOR 固定的偏差值之和。 最好的方法是使用等于$\Delta$，因为将其分成多个部分并不能减少总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, x = map(int, input().split())
        
        base_xor = 2047 if n % 2 else 0
        delta = base_xor ^ x
        
        ans = 2047 * n - delta
        print(ans)

if __name__ == "__main__":
    solve()
```实施直接遵循减少到单个校正值。 唯一微妙的部分是使用奇偶校验正确计算基线的异或$n$，因为重复相同数字的异或仅取决于计数是奇数还是偶数。 

我们从不显式构造数组。 每个测试用例的一切都被简化为恒定时间算术。 

## 工作示例

 考虑一个情况$n = 3$和$x = 5$。 

基线异或$[2047, 2047, 2047]$是 2047，因为计数是奇数。 所以$\Delta = 2047 \oplus 5 = 2042$。 

| 步骤| 基础异或| 目标 x | 达美航空 | 结果 |
 | --- | --- | --- | --- | --- |
 | 初始| 2047 | 2047 5 | 0 | 0 |
 | 计算| 2047 | 2047 5 | 2042 | 2042 2047·3 - 2042 | 2047·3 - 2042 |

 这表明我们只需要应用一次大小为 2042 的单一校正。 

现在考虑$n = 4$,$x = 0$。 

| 步骤| 基础异或| 目标 x | 达美航空 | 结果 |
 | --- | --- | --- | --- | --- |
 | 初始| 0 | 0 | 0 | 2047·4 |
 | 计算| 0 | 0 | 0 | 没有变化|

 这里，全 2047 个配置已经满足 XOR 约束。

 ## 边缘情况

 当$n$是偶数且$x = 0$，基线 XOR 已经为零，因此不需要校正。 该算法计算$\Delta = 0$并返回$2047 \cdot n$，这对应于为所有课程分配最高分。 

什么时候$n$是奇数并且$x = 2047$，基线 XOR 已经是 2047。同样$\Delta = 0$，因此不进行任何调整，解仍保持最佳状态。 

什么时候$n = 2$，结构是最小的，任何修正都会立即影响 XOR 和总和约束。 该公式仍然可以正确计算是否需要单次校正，并在恒定时间内应用它，而无需特殊的套管。 

什么时候$x$如果很大，接近 2047，XOR 校正仍然限制在 11 位以内，因此从最大和中减去不会溢出或需要特殊处理。
