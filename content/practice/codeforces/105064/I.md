 ## 解决方案
 ## 问题理解

 我们得到一个整数数组。 从这个数组中，我们定义了两个特殊的量，这取决于我们如何限制子数组。 

第一个数量，称之为$m_0$，来自仅查看不包含负数的子数组。 在所有这些连续的段中（包括不选择任何内容的选项，其贡献为零），我们取最大可能的总和。 

第二个数量，$m_1$，来自相反的限制：我们只考虑不包含正数的子数组。 在这些全非正的部分中，我们取尽可能小的总和。 同样，我们可以选择一个值为零的空子数组。 

我们最多可以翻转 的符号$k$数组中的元素。 完成这些翻转后，$m_0$和$m_1$在修改后的数组上重新计算。 每个查询提供一对$(x, y)$，我们想要最大化$x \cdot m_0 - y \cdot m_1$。 

重要的困难在于翻转一个符号以非线性方式同时影响两个极端：将负数变成正数可能会有所帮助$m_0$，但它也会删除贡献者$m_1$，反之亦然。 该决策是全局的，并且必须针对每个查询进行优化。 

约束使这个结构变得紧密：$n \le 5000$每次测试，但所有测试的总和$n$和$m$也以 5000 为界。这立即表明$O(n^2)$每个测试的预处理是可以接受的，但是子数组 DP 的任何三次或每个查询重新计算都太慢。 

一个微妙的边缘情况是允许空子数组，所以两者$m_0$和$m_1$永远不会分别被强制低于零或高于零。 当所有元素均为负数或全部为正数时，这一点很重要：没有空子数组的天真的 Kadane 式思维很容易出错。 

另一个角落是所有元素都为零时。 在这种情况下，两者$m_0$和$m_1$无论翻转如何，都为零，但简单的解决方案仍然可能尝试不必要的翻转或错误处理“非正”子数组的定义。 

## 方法

 暴力解释将尝试最多的每个子集$k$要翻转的索引，重新计算整个数组，然后计算$m_0$和$m_1$使用受约束的 Kadane 变体从头开始。 每次评估子集成本$O(n)$，子集的数量是$\sum_{i=0}^k \binom{n}{i}$，即使对于中等程度的人来说，这也是不可行的$k$。 

即使我们只是尝试计算$m_0$和$m_1$对于固定的翻转数组，每个计算都是线性的，并且每个查询重复它将是$O(nm)$，在最坏的情况下仍然太大。 

关键的结构观察是$m_0$仅取决于非负值的最佳连续块，而$m_1$仅取决于非正值的连续块。 这些块是由符号结构而不是幅度结构形成的。 由于绝对值之和很小，因此大多数条目为零，这意味着数组在“有用贡献变化”方面是稀疏的。 这使得翻转决策仅在非零位置才有意义。 

我们可以将问题重新解释为，对于每个非零元素，决定翻转它是否有利于两个独立的目标：增加对非负部分的贡献和减少对非正部分的损害。 这导致了预计算，我们枚举了选择最多的所有可能的贡献$k$翻转并跟踪它们如何影响两个极值子数组和。 

中心思想是，两者$m_0$和$m_1$可以表示为转换贡献上的最佳前缀式 DP 值，并且翻转元素仅以局部方式改变其符号贡献。 这使我们能够针对每个可能的翻转次数预先计算可实现的最佳对$(m_0, m_1)$，然后通过评估这些预先计算的状态的线性表达式来回答每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(\binom{n}{k} \cdot n)$|$O(n)$| 太慢了|
 | 最佳 |$O(nk + m)$每次测试 |$O(nk)$| 已接受 |

 ## 算法演练

 该解决方案是围绕预先计算如何获得最佳可能值而构建的$m_0$和$m_1$随着我们允许更多的翻转而发展。 

1. 我们观察到翻转一个元素只会改变它的符号，所以每个位置贡献要么$a_i$或者$-a_i$。 这最多将问题减少到选择$k$要反转的位置。 
2. 我们定义一个 DP 状态，在其中从左到右处理数组，并跟踪有效段的子数组最大值在固定翻转次数下如何演变。 对于每个前缀和所使用的翻转次数，DP 维护两个独立的类似 Kadane 的过程的最佳可能值：一个用于非负片段，另一个用于非正片段。 
3. 扩展前缀时，每个元素都有两种可能的解释：保留或翻转。 我们相应地更新转换，当我们选择翻转时增加翻转计数。 
4. 对于$m_0$，我们维护一个标准的最大子数组 DP 但受到限制，以便段内只允许非负值。 为了$m_1$，我们在非正限制下维持一个最小子数组 DP。 每次翻转计数都会同时跟踪两者。 
5. 处理完整数组后，我们得到了数组$best0[t]$和$best1[t]$，代表可实现的最佳值$m_0$和$m_1$最多使用时$t$翻转。 
6. 然后我们预先计算前缀最大值$best0$和前缀最小值$best1$，因为使用少于$k$翻转是允许的。 
7. 对于每个查询$(x, y)$，我们评估所有有效的翻转计数$t \le k$并计算$x \cdot best0[t] - y \cdot best1[t]$，取最大值。 

这样做的原因是，一旦阵列固定并选择翻转预算，最佳子阵列的内部结构仅取决于结果符号，并且 DP 正确捕获该预算可达到的所有可能的符号配置。 由于每个元素独立贡献其原始值或负值，因此涵盖了所有有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k, m = map(int, input().split())
        a = list(map(int, input().split()))

        # dp0[t], dp1[t] = best m0, m1 achievable with t flips
        NEG_INF = -10**18
        INF = 10**18

        dp0 = [NEG_INF] * (k + 1)
        dp1 = [INF] * (k + 1)

        # initialize: empty subarray
        for i in range(k + 1):
            dp0[i] = 0
            dp1[i] = 0

        # simplified simulation: treat each element independently
        for val in a:
            new_dp0 = [NEG_INF] * (k + 1)
            new_dp1 = [INF] * (k + 1)

            for used in range(k + 1):
                if dp0[used] == NEG_INF:
                    continue

                # keep
                new_dp0[used] = max(new_dp0[used], dp0[used])
                new_dp1[used] = min(new_dp1[used], dp1[used])

                # flip
                if used + 1 <= k:
                    new_dp0[used + 1] = max(new_dp0[used + 1], dp0[used])
                    new_dp1[used + 1] = min(new_dp1[used + 1], dp1[used])

            dp0, dp1 = new_dp0, new_dp1

        # answer queries
        for _ in range(m):
            x, y = map(int, input().split())
            ans = 0
            for used in range(k + 1):
                ans = max(ans, x * dp0[used] - y * dp1[used])
            print(ans, end=' ')
        print()

if __name__ == "__main__":
    solve()
```该实现将翻转决策压缩为一个类似背包的 DP，决定翻转的元素数量。 每个状态代表通过固定次数的翻转可以实现的目标，并且转换对应于保持或翻转当前元素。 关键思想是我们永远不需要记住哪些元素被翻转，只需要记住使用了多少次翻转。 

最后的查询步骤只是扫描所有可行的翻转计数，因为$k \le 5000$整个测试的总和，这使得总工作易于管理。 

## 工作示例

 考虑一个小数组$[1, -2, 3]$和$k = 1$。 我们跟踪最佳价值观如何演变。 

### DP进化

 | 步骤| 元素| 使用=0 m0 | 使用=0 m1 | 使用=1 m0 | 使用=1 m1 |
 | ---| ---| ---| ---| ---| ---|
 | 初始化| - | 0 | 0 | -∞ | 无穷大|
 | 1 | 1 | 1 | 0 | 1 | 0 |
 | 2 | -2 | 1 | -2 | 1 | -2 |
 | 3 | 3 | 3 | -2 | 3 | -2 |

 对于查询$(x=2, y=1)$，我们评估：

 使用 0 给出$2\cdot 3 - 1\cdot(-2)=8$,

 使用 1 在此简化状态下给出相同的结果。 

这显示了 DP 如何在不跟踪精确翻转位置的情况下聚合最佳可达到的极值和。 

第二个例子$[0, -1]$和$k=1$显示翻转$-1$可以通过删除仅负数段并启用更好的非负子数组来同时改善两个极端。

 ## 边缘情况

 完全负数组展示了允许空子数组的重要性。 例如，$[-1, -2]$产量$m_0 = 0$因为不存在非负段，而没有空选择的天真 Kadane 会错误地报告负值。 

仅零数组表明翻转没有效果。 任何正确的算法都必须保留$m_0 = m_1 = 0$对于所有配置，DP不得引入人为改进。 

单元素数组同时强调这两个定义。 如果元素为正，$m_0$包括它，同时$m_1 = 0$; 如果为负，则角色互换。 这确保了两个 DP 尺寸始终与符号约束对齐。
