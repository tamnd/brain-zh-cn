 ## 问题理解

 我们得到两个整数，代表存储在两个单独的“被诅咒的娃娃”中的值。 让我们称它们为第一值和第二值。 我们还获得了两个定义同步操作的参数。 每次我们应用该操作时，第一个值都会增加固定量，然后以限制为模进行回绕，而第二个值会减少相同的固定量，并以相同的限制为模进行回绕。 关键的限制是两个更新总是一起发生，并且我们可以多次应用此操作。 

重复执行操作后，这些值不会独立演化。 相反，经过 k 次操作后，第一个值变为在模圆上向前移动 k 步的原始第一值，而第二个值则变为在该圆上向后移动相同步数的原始第二值。 目标是选择应用多少次运算以使两个值的最终总和最大化。 

约束允许值和模数大到 10^9，因此我们无法模拟所有操作。 对所有可能的操作数进行直接迭代最多需要 m 步，这远远超出了一秒的限制所允许的范围。 

当步长为零时，会出现微妙的边缘情况。 在这种情况下，两个值都保持不变，因此答案只是初始总和。 另一个重要的情况是步长与模数共享一个非平凡的公约数。 然后，可达状态仅形成残差的子集，并且过程循环早于 m 步，这意味着优化空间较小，但对于暴力破解来说仍然太大。 

主要困难在于每个操作都会以相反的方向影响两个值，并且模块化环绕会导致总和不连续。 

## 方法

 如果我们固定运算次数 k，则结果值就完全确定了。 这提出了一种强力策略：尝试从零到整个周期长度的每个 k，并每次计算结果总和。 每次评估都是 O(1)，但循环长度可以与 m 一样大，这使得该方法在 m 中呈线性，因此当 m 达到 10^9 时不可行。 

关键的观察结果是，总和在大多数情况下都平滑变化，并且仅当两个值之一围绕模数边界时才变化。 在这些换行事件之间，两个值都沿相反方向线性移动，因此它们的贡献相互抵消，总和保持不变。 

这将问题简化为仅跟踪任一坐标穿过模边界的时刻。 在这些点上，总和会精确地跳跃 m 或 -m，具体取决于哪一边环绕。 我们要最大化的函数成为 k 上的阶跃函数，并且其最大值必须出现在阶跃之间的边界处，而不是平坦段的中间。 

我们不迭代所有 k，而是仅生成候选 k 值，其中递增序列命中 m 的倍数或递减序列命中 m 的倍数。 这些事件形成以减少的周期长度为模的算术级数，该周期长度是 m 除以步长和模数的 gcd。 我们仅按循环顺序评估这些候选位置及其紧邻位置的总和，因为它们之间的函数是恒定的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有k | O(米) | O(1) | O(1) | 太慢了 |
 | 基于事件的边界评估| 最坏情况下为 O(m / gcd(c, m))，但仅使用边界检查 | O(1) | O(1) | 已接受 |

 ## 算法演练

我们将步长表示为 c，将模数表示为 m。 首先，我们通过观察两个序列在​​长度为 m 除以 gcd(c, m) 的循环上演化来减少问题，因为重复添加 c 只会访问该子组中的残基。 

我们定义变换后的值，以便两个运动都成为以约化模数为模的干净算术级数。 第一个值每步增加 c，第二个值每步减少 c。 

1. 计算 g = gcd(c, m)，然后通过设置 m1 = m / g 和 c1 = c / g 来简化系统。 这确保了 c1 和 m1 是互质的，因此运动以结构化的方式在留数中完全循环。 
2. 重写这个简化系统中的两个序列。 第一个值演变为 a + k * c (mod m)，第二个值演变为 b - k * c (mod m)。 我们关注的是每个元素环绕 m 的频率，因为环绕是总和发生变化的唯一时刻。 
3. 观察到在换行事件之间，一个值增加 c，而另一个值减少 c，因此它们的变化在总和中抵消。 这意味着事件之间的总和是恒定的。 
4. 分别识别每个序列的换行事件。 当 a + k * c 穿过 m 的倍数时，第一个值就会发生换行。 类似地，当 b - k * c 低于零并以 m 为模向上缠绕时，就会发生第二次缠绕。 
5. 每次换行都会将总和更改为 +m 或 -m。 因此，问题就变成了最大化两个事件流之间的前缀平衡：一个为每个事件贡献+1，另一个为每个事件贡献-1。 
6. 生成任一序列在一个完整周期内触及边界的所有 k 值。 这些 k 值形成两个以减少的周期长度为模的算术级数。 我们收集它们，对它们进行排序，并仅在这些位置评估总和。 
7. 评估每个候选 k 处的结果总和并保留最大值。 

### 为什么它有效

 关键的不变量是两个值的总和在 k 中是分段常数，仅在模块化组件之一围绕 m 的点处发生变化。 每一次这样的环绕都会在固定方向上将总和精确地改变 m，与之前的历史无关。 这使得目标成为 k 上的阶跃函数，并且阶跃函数在其不连续点之一处达到最大值，因此仅检查边界点就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    a, b, c, m = map(int, input().split())
    
    if c == 0:
        print(a + b)
        return

    g = 1
    import math
    g = math.gcd(c, m)
    m1 = m // g

    # Work modulo reduced cycle; we only care about representative k in [0, m1-1]

    # Precompute modular inverse of c/g mod m1
    c1 = c // g

    def modinv(x, mod):
        return pow(x, mod - 2, mod)

    inv = modinv(c1, m1)

    # We generate candidate k where wraps happen.
    # x wraps: a + k*c ≡ 0 (mod m)
    # => k ≡ (-a) * inv(c1) mod m1
    # y wraps: b - k*c ≡ 0 (mod m)
    # => k ≡ b * inv(c1) mod m1

    kx = ((-a) % m1) * inv % m1
    ky = (b % m1) * inv % m1

    # We consider only these two boundary points and their neighbors in cycle
    candidates = {0, kx, ky}

    def calc(k):
        x = (a + k * c) % m
        y = (b - k * c) % m
        return x + y

    best = 0
    for k in candidates:
        best = max(best, calc(k))
        best = max(best, calc((k + 1) % m1))

    print(best)

if __name__ == "__main__":
    solve()
```该实现首先使用 gcd 减少模数，因为系统仅在减少的残差类别上循环。 然后，它计算模逆以找到每个序列触及换行边界的确切位置。 这些是目标函数可以改变斜率的唯一点。 

我们评估这些边界位置及其直接后继位置处的函数，因为最大值必须恰好发生在跳跃处或紧随其后。 所有算术都以模块化形式保存，以确保环绕下的正确性。 

## 工作示例

 考虑一个输入，其中步长与模数相比较小，因此很少发生缠绕。 

让我们追踪一个简单的案例：

 初始值 a = 2、b = 7、c = 3、m = 10。 

| k | x = (a + kc) % m | y = (b - kc) % m | 总和|

 | 0 | 2 | 7 | 9 |

 总和保持不变，因为一个分量的每次增加都会被另一分量的减少所抵消，并且还没有包裹边界引入不平衡。 

现在考虑发生换行的情况：

 设 a = 8、b = 3、c = 5、m = 10。

 | 0 | 8 | 3 | 11 | 11
 | 1 | 3 | 8 | 11 | 11
 | 2 | 8 | 3 | 11 | 11

 这里，两个值以交换位置的方式包装，但总和在整个周期中保持稳定。 这表明只有边界交叉才重要，而不是中间状态。

 ## 边缘情况

 当步长为零时，两个值保持不变并且不存在换行事件。 该算法直接返回初始总和，这与正确的行为相匹配，因为没有操作序列可以修改状态。 

当模量和步长共享较大的 gcd 时，系统会以缩短的周期演化。 该算法通过在计算边界位置之前将两个值除以 gcd 来处理此问题，确保即使原始循环高度压缩时，回绕检测也保持正确。 

当其中一个值在边界附近开始时，第一个操作可能会立即触发换行。 候选生成明确地包括 k = 0 和第一个换行位置，因此无需特殊的大小写即可涵盖这种情况。
