 ## 问题理解

 我们得到一个三角形数组`n`行。 排`i`恰好包含`i`整数。 该过程从顶行开始，逐行重复销毁，直到只剩下底行。 

当水晶有价值时`x`被移除时，它会选择其正下方的两个晶体中较大的一个。 假设左孩子有价值`a`正确的孩子有价值`b`。 如果`a > b`，左孩子变为`a - x`。 否则右孩子会变成`b - x`。 另一个孩子保持不变。 

当除了最后一行之外的所有行都消失后，我们必须输出底行中晶体的值。 

三角形包含$$1+2+\dots+n=\frac{n(n+1)}2$$数字。 自从`n ≤ 1000`，元素总数最多为 500500。任何处理每个晶体恒定次数的算法都足够快。 细胞数量的二次方大致意味着$2.5 \times 10^{11}$的操作，远远超出了极限。 

一个微妙的点是，晶体会立即修改其子级之一，并且以后的决策必须使用更新的值，而不是原始值。 

考虑```
2
5
1 10
```顶级水晶选择`10`， 自从`1 ≤ 10`，并将其更改为`10-5=5`。 

答案是```
1 5
```在任何地方使用原始值都会产生错误的结果。 

错误的另一个来源是平局。 相等的值不选择左孩子。 

为了```
2
3
7 7
```条件`a ≤ b`适用，所以正确的晶体变成`7-3=4`。 

正确答案是```
7 4
```使用`>=`而不是`>`会错误地修改左边的水晶。 

负数也很重要。```
2
-5
2 1
```自从`2 > 1`，左边的晶体被选择并成为$$2-(-5)=7.$$答案是```
7 1
```以错误的顺序减去会得到`-7`。 

## 方法

 最直接的想法就是完全按照描述来模拟仪式。 从第一行开始，我们检查每个晶体并比较其下面的两个值。 根据哪个较大，我们立即修改该子项。 

这种模拟已经很有效，因为除了最后一行中的晶体之外​​的每个晶体都被处理一次。 处理的细胞总数等于$$1+2+\dots+(n-1)=\frac{n(n-1)}2,$$大约有五十万次操作`n=1000`。 

人们可能会想象一种更复杂的方法，它可以跟踪路径或递归地跟踪能量流动的地方。 这种对所有可能路径的强力攻击会呈指数级增长，因为每个晶体可能有两种选择。 幸运的是，这个过程本身决定了独特的选择，并且每个水晶只影响一个孩子。 不需要回溯。 

关键的观察结果是三角形可以简单地就地更新。 一次排`i`已处理完毕，不再使用。 所有未来的决策仅取决于行中已更新的值`i+1`。 这允许我们直接覆盖值而不需要任何辅助结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 枚举所有可能的路径 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | 就地模拟 | O(n²) | O(n²) | 已接受 |

 ## 算法演练

 1. 将整个三角形读入二维数组。 
2. 从上到下处理行。 排`i`影响行`i+1`，所以完成行后`i`我们再也不需要它了。 
3. 对于每个位置`j`在行中`i`， 让`x`是该晶体的当前值。 
4.比较两个孩子，`triangle[i+1][j]`和`triangle[i+1][j+1]`。 
5. 如果左孩子严格大于，则将其替换为$$\text{left}-x.$$否则将右孩子替换为$$\text{right}-x.$$比较必须使用当前值，因为同一行的先前晶体可能已经修改了它们。 

1. 继续，直到处理完倒数第二行。 
2. 输出最后一行。 

### 为什么它有效

 处理后行`i`，每行晶体引起的变化`0,1,\dots,i`已经合并到 row 中`i+1`。 未来的操作不会影响任何较早的行。 

不变的是在开始行之前`i`，它上面的所有行都已经完全按照规则被完全销毁，并且行`i`包含仪式中当时存在的价值观。 处理每个晶体应用与语句完全相同的操作，因此不变量仍然成立。 当倒数第二行完成时，底行恰好包含最终的能量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
triangle = [list(map(int, input().split())) for _ in range(n)]

for i in range(n - 1):
    for j in range(i + 1):
        x = triangle[i][j]
        if triangle[i + 1][j] > triangle[i + 1][j + 1]:
            triangle[i + 1][j] -= x
        else:
            triangle[i + 1][j + 1] -= x

print(*triangle[-1])
```三角形的存储方式与输入中显示的完全相同。 由于后续计算总是需要更新的值，因此就地修改数组是很自然的。 

外循环从上到下处理行。 排`i`包含`i+1`元素，因此内部循环会遍历所有元素。 

比较使用`>`而不是`>=`。 相等的值必须选择右子元素，因为该语句仅在左子元素严格大于时才指定左子元素。 

Python 整数自动处理大值，因此溢出不是问题。 对于另一种语言中的固定大小整数，64 位类型是必要的。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n²) | 除最后一行之外的每个晶体都被处理一次 |
 | 空间| O(n²) | 整个三角形被存储 |

 最大可能的三角形包含 500500 个元素。 处理每个单元一次很容易满足时间限制，而存储 50 万个整数则适合 256 MB。

import sys, io

def run(inp: str) -> str:

    n = int(input())
