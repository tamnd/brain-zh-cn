 ## 解决方案
 ## 问题理解

 我们正在与二维整数网格上的隐藏点进行交互。 一开始有一点$(x_0, y_0)$我们不知道。 系统定义了一个函数$f(x, y)$，它表示僵尸从$(x, y)$在一个非常不寻常的运动规则下到原点：僵尸以轴对齐的分段移动，每个分段都有一个整数长度$d$, 成本$d^2$时间单位，每段结束后必须旋转 90 度。 

而不是看到$f$直接，我们可以查询当我们移动僵尸时它如何变化。 一个查询$(\Delta x, \Delta y)$将向量添加到当前位置，我们收到值$$f(x + \Delta x, y + \Delta y) - f(x, y).$$应答后，系统相应更新当前位置。 我们必须确定原始隐藏坐标。 

关键的困难在于查询是自适应的。 每个查询都会改变状态，因此后面的答案取决于之前的移动。 我们限制最多 100 个查询，并且坐标可以大到$10^9$绝对值，因此暴力破解或扫描是不可能的。 

一个微妙的边缘情况是每个查询都会永久移动隐藏点。 例如，如果我们问$(1,0)$，所有后续答案都相对于$(x_0+1,y_0)$，不是原来的位置。 任何假设查询是独立的方法都会产生不一致的方程。 

另一个陷阱是假设我们可以直接恢复$f(x,y)$。 我们从不观察绝对值，只观察变化状态下的差异。 这迫使我们利用差分公式中的代数结构，而不是重建函数本身。 

## 方法

 蛮力的想法是尝试“探测”该函数$f$通过探索多个方向并尝试局部推断曲率或重建值。 由于每个查询仅在移动点处给出差异，因此这很快就会变得不一致：每次查询后，参考点都会发生变化，因此系统的行为就像一个漂移的预言机。 即使尝试近似偏导数也需要在同一状态下重复采样，这是不可能的。 

转折点是认识到僵尸成本函数具有封闭的二次形式。 之间的任何最佳路径$(x,y)$并且原点最多可以减少到两个轴对齐的段，因为将固定位移分成多个段只会增加平方和。 这意味着：$$f(x,y) = x^2 + y^2.$$一旦知道这一点，每个查询就变成纯粹的代数。 扩展：$$f(x+\Delta x, y+\Delta y) - f(x,y)
= 2x\Delta x + \Delta x^2 + 2y\Delta y + \Delta y^2.$$这是一个线性表达式$x$和$y$加上已知常数。 即使每次查询后状态发生变化，两个精心选择的查询也足以求解这两个变量，因为每个查询都干净地隔离了一个坐标。

 ## 算法演练

 我们利用每个查询都会揭示当前坐标中的线性方程这一事实。 

1.我们首先查询$(1, 0)$。 答案是$$(x+1)^2 + y^2 - (x^2 + y^2) = 2x + 1.$$这立即允许恢复原来的$x$坐标为$(\text{res}_1 - 1)/2$。 
2. 查询完成后，系统将位置更新为$(x+1, y)$。 我们不会试图挽回这一点； 相反，我们利用它来发挥我们的优势。 
3.然后我们查询$(0, 1)$从这个新的状态。 答案是$$(x+1)^2 + (y+1)^2 - ((x+1)^2 + y^2) = 2y + 1.$$这隔离了当前$y$值，仍然是原来的值$y$，所以我们计算$y = (\text{res}_2 - 1)/2$。 
4.此时我们已经重建了原来的$(x, y)$完全可以直接输出。 我们不使用两次移动后该点的最终位置； 我们仅将其用作中间工件。 

关键思想是，尽管每次查询后状态都会发生漂移，但每个坐标仍然可以被隔离，因为二次形式干净地分成独立的 x 和 y 分量。 

### 为什么它有效

 功能$f(x,y)=x^2+y^2$跨坐标可分离，并且每个查询仅引入当前状态下的线性交叉项。 因为我们总是查询单位轴向量，所以每个响应仅取决于一个坐标加上一个已知常数。 状态转移不会混合坐标，因此每个方程一次对单个变量保持有效。 这保证了两个查询唯一地确定原始点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(dx, dy):
    print("?", dx, dy)
    sys.stdout.flush()
    return int(input().strip())

# Query 1: shift by (1,0)
r1 = ask(1, 0)
x = (r1 - 1) // 2

# Query 2: shift by (0,1)
r2 = ask(0, 1)
y = (r2 - 1) // 2

# After queries, current position is (x+1, y+1),
# but we must output original position.
print("!", x, y)
sys.stdout.flush()
```第一个查询提取$x$来自直接二次展开式，其中所有$y$- 条款取消。 第二个查询在状态转移后评估，但结构$f$确保同样的取消发生$y$。 

一个常见的实施错误是忘记查询会永久移动重点。 这在这里是无害的，只是因为每个查询都旨在隔离一个坐标并且不需要重新访问先前的状态。 

另一个微妙之处是整数除法：两个结果都保证是奇数，因为$2x+1$和$2y+1$总是出现，所以减法和除法是精确的。

该迹线表明，即使在状态发生变化后，每个坐标也是孤立的，因为二次结构仍然与轴对齐。

 ## 边缘情况

 当一个坐标为零时，就会出现边界情况。 例如，开始于$(0, 5)$，第一个查询产生$1$, 给予$x=0$。 第二个查询仍然隔离$y$状态转换后正确。 不会出现除法问题，因为两个响应仍然是奇数。 

另一个边缘情况是大幅度坐标，例如$10^9$。 二次展开仍然会产生 64 位范围内的值，并且整数算术可以安全地处理计算。 

最后一个微妙的情况是负坐标。 由于该公式仅取决于$2x+1$和$2y+1$，符号被自动保留，整数除法可以正确地重建负值，无需任何特殊处理。
