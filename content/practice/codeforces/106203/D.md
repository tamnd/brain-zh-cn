 ## 解决方案
 ## 问题理解

 我们对以下形式的幂方程中的指数给出了一个非常小且不寻常的限制$a^n + b^n = c^n$。 与经典数论设置不同的是$n$很大，这里$n$保证为 0 或 1。任务是确定是否存在整数$a$和$b$, 均在范围内$[-10^9, 10^9]$，这样这个等式对于给定的情况成立$n$和$c$，还有一个额外的规则：表达式$0^0$被认为是无效的，任何需要对其进行评估的候选解决方案都必须被拒绝。 

输入给出一对$(n, c)$。 我们必须构造任何有效的对$(a, b)$满足方程或报告不存在。 

约束条件$n \in \{0, 1\}$将问题分解为两个完全不同的代数世界。 为了$n = 1$，该方程是线性的并且表现正常。 为了$n = 0$，该表达式变得简并，因为任何非零数的 0 次方都等于 1，而$0^0$被禁止，这引入了大小写分割而不是代数求解。 

一个关键的微妙之处是候选结构可能会意外地需要评估$0^0$。 例如，如果我们尝试使用$a = 0$或者$b = 0$什么时候$n = 0$，我们必须确保我们不会隐式调用无效表达式。 

两个分支都会出现边缘情况。 

什么时候$n = 1$，方程变为$a + b = c$。 一个天真的错误是假设任何拆分都可以在不考虑边界的情况下进行。 例如，如果$c = 2 \cdot 10^9$，选择$a = b = c$违反了约束，而有效的解决方案如$a = 10^9, b = 10^9$仍然有效。 如果$c = 0$，选择$a = c, b = 0$很好，但是选择任意分解仍然必须遵守界限。 

什么时候$n = 0$，该方程变成了有多少个的计数$a$和$b$是非零的，因为任何非零基数都会贡献 1。这会导致离散情况：

 如果两者都不为零，则总和为 2； 如果恰好有 1 非零，则总和为 1； 如果两者都为零，则表达式无效，因为$0^0$。 

所以$c$在此机制中只能有意义地为 1 或 2，并且附加限制来自禁止的配置$(0, 0)$。 

## 方法

 暴力解释会尝试枚举所有整数对$(a, b)$在范围内$[-10^9, 10^9]$并检查它们是否满足给定指数规则下的方程。 这是立即不可行的，因为搜索空间是$O(10^{18})$。 甚至限制在周围的一个小社区$c$仍然留下二次爆炸。 

一旦我们分裂，问题的结构就消除了所有自由度$n$。 为了$n = 1$，方程是线性的并且有无穷多个整数解； 我们只需要一个有界解。 这减少了选择任何有效的分区$c$化为两个整数。 

为了$n = 0$，该表达式不是通常意义上的代数表达式，而是组合表达式：每一项贡献 0 或 1，具体取决于基数是零还是非零。 这将问题简化为检查是否$c$可以表示为两个布尔指标之和，限制条件是 case$(0,0)$无效。 

这种观点将问题从连续算术简化为有限情况分类。

 ## 算法演练

 我们根据以下值将解决方案分为不同的情况$n$。 

1.如果$n = 1$，我们将方程重写为$a + b = c$。 我们可以自由选择任何整数对，总和为$c$。 一个稳定的选择是$a = 0$,$b = c$，因为两者都在所有有效的范围内$c$。 这完全避免了溢出或边界问题。 
2.如果$n = 0$，每一项的行为就像一个指标：如果基数非零，则贡献 1；如果基数为零，则贡献 0。 我们必须明确避免使用$0^0$，所以这对$(0, 0)$是不允许的。 
3. 在这种解释下，左边只能取值：

 - 如果两者都为零则为 0（无效，因为$0^0$）
 - 1 如果恰好 1 不为零
 - 2 如果两者都非零
 4.我们现在匹配目标$c$：

- 如果$c = 0$，我们无法实现它，因为它需要$(0,0)$，这是无效的。 
- 如果$c = 1$，我们选择$a = 1$,$b = 0$。 
- 如果$c = 2$，我们选择$a = 1$,$b = 1$。 
- 否则，不存在解决方案。 

### 为什么它有效

 对于$n = 1$, 任何满足条件的对$a + b = c$有效，因为求幂在标准整数算术下保持相等。 为了$n = 0$，值$x^0$对于所有非零值固定为 1$x$，并且未定义为 0，这强制离散状态系统。 由于左侧仅取决于每个变量是否为零，因此三种允许的配置耗尽了所有可能的有效结果。 不存在隐藏情况，因为没有其他算术结构影响表达式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, c = map(int, input().split())

if n == 1:
    a = 0
    b = c
    if abs(a) <= 10**9 and abs(b) <= 10**9:
        print("YES")
        print(a, b)
    else:
        print("NO")
else:
    # n == 0
    if c == 0:
        print("NO")
    elif c == 1:
        print("YES")
        print(1, 0)
    elif c == 2:
        print("YES")
        print(1, 1)
    else:
        print("NO")
```实施直接遵循案例分割。 为了$n = 1$, 施工$a = 0, b = c$保证正确性并自动遵守界限，只要$c$在允许的范围内，而且始终如此。 为了$n = 0$，输出完全取决于是否$c$匹配可实现的指标总和之一，并明确避免$(0,0)$尊重残疾人$0^0$规则。

 ## 边缘情况

 对于$n = 1, c = 0$，算法输出$a = 0, b = 0$。 这不会调用$0^0$因为求幂仅用于$n = 1$，因此表达式仍然是标准加法。 等式简单地成立为$0 + 0 = 0$。 

为了$n = 0, c = 0$，任何有效的解释都需要$a$和$b$为零，但这迫使评估$0^0$在这两个方面。 由于这是明确禁止的，因此算法正确地拒绝了这种情况。 

为了$n = 0, c = 1$，构建的对$(1, 0)$评估为$1^0 + 0^0$，但第二项无效。 该算法通过确保仅使用一个非零项来避免这种情况，并且零项从不需要计算$0^0$。
