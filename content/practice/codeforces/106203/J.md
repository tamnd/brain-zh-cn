 ## 解决方案
 ## 问题理解

 我们得到一个长度的数组`n`其元素是范围内的整数`[0, k]`。 该数组被视为多重集，其顺序与条件无关，因为唯一重要的属性是元素之和以及元素的乘积。 

任务是构造另一个任意长度的数组`m`并且值限制为`[0, k]`，这样它与给定的数组不同，但具有完全相同的和和相同的乘积。 如果不存在这样的数组，我们必须输出`-1`。 

一个关键的微妙之处是目标数组不需要与原始数组具有相同的长度。 只有总和和乘积必须匹配。 这使得问题成为一个受约束的整数分解问题：我们试图使用有界字母表中的数字来表示固定的和和乘积。 

限制因素`n, k ≤ 10^5`这意味着任何试图枚举候选数组或直接分解乘积的解决方案都是不可行的。 即使是对大型候选空间的线性扫描或任何指数构造也被排除。 

主要困难来自于产品状况。 虽然通过重新分配很容易保留总和，但乘积却极其严格，尤其是当出现零或一时。 

一些边缘情况立即很重要。 

如果数组至少包含一个零，则乘积为零。 任何有效答案还必须至少包含一个零。 单独重新排列非零元素的天真尝试可能会失败，因为删除零或更改其计数可能会破坏乘积相等。 

示例：输入`1 2 [0]`。 乘积为零，总和为零。 任何包含单个零的数组都可以，但如果尝试错误`[1, -1]`或者`[0, 0]`，它可能会违反边界或长度限制。 

如果所有元素都是`1`，则总和等于`n`和产品等于`1`。 保存产品的唯一方法`1`就是只使用那些。 任何不同的数组都需要一个非 1 元素，这会立即更改乘积，因此这种情况在以下情况下是不可能的：`n = 1`，但有可能时`n > 1`通过重新安排结构？ 实际上任何排列都是相同的，因此不存在不同的多重集。 这使得全一数组完全不可能。 

如果值大于`1`并且不存在零，产品增长很快并且变得极其有限。 唯一可行的转换通常涉及拆分或合并数字，同时保留总和和乘积，这很少可能，除非在结构化情况下。 

## 方法

 暴力解释将尝试构造长度达到的所有可能的数组`3n`值在`[0, k]`，计算它们的和和乘积，并与原来的进行比较。 这立即爆炸了，因为即使对于`n = 20`，数组的数量是`(k+1)^n`，这远远超出了可行的限度。 

即使限制为固定长度仍然会留下指数搜索空间。 乘积约束也使得修剪变得困难，因为小的变化会以乘法方式传播。 

关键的观察结果是产品保存施加了强大的结构性约束。 特别是，一旦乘积非零，每个元素必须至少`1`，并且任何修改数组的尝试都必须准确保留乘法分解。 这实际上意味着我们正在处理乘积固定的整数的多重集分解问题。 

当零存在时，唯一的灵活性就会出现，因为零会消除乘积。 在这种情况下，任何至少有一个零且总和相同的数组都是有效的。 这将问题简化为非负约束下的仅求和重排问题。 

当不存在零时，所有元素均为正。 那么乘积是正的且固定的，这迫使质因式分解的多重集保持相同。 由于我们的整数限制为`k`，我们唯一可以使用的数字是乘积的除数。 任何变换都必须同时保留和和乘法结构，除非我们不以非平凡的方式改变多重集，否则这是不可能的。 

这导致了最终的结构：要么数组包含零，在这种情况下我们可以构造具有相同和和乘积的不同排列，要么不包含零，在这种情况下唯一可能的重新排列是微不足道的，因此是不允许的。 

当零存在时，就会出现建设性的策略。 我们可以隔离一个零，并将剩余的总和重新分配给一组固定的值，通常使用`1`和`k`边界以保持有效性并确保乘积保持为零。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力枚举| O((k+1)^n) | O((k+1)^n) | O(n) | 太慢了 |
 | 建设性的零处理 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们根据数组是否包含零来划分推理。 

### 情况 1：数组至少包含一个零

 1. 计算总和`S`数组并确认输入数组中至少有一个零。 因此数组的乘积是`0`。 
2.如果`S == 0`，那么每个元素都必须为零。 任何有效的数组也必须完全由零组成，因此不能存在不同的数组。 我们输出`-1`。 不可能的原因在于总和约束强制所有元素，没有留下改变结构的自由。 
3. 否则，我们构造一个新数组，该数组仍包含至少一个零以保存乘积`0`，但改变了剩余金额的分配。 我们明确地放置一个零。 
4. 分配剩余金额`S`仅使用正整数。 一个简单的安全构造是使用`S-1`一个和一个`1`调整后的元素，但为了确保有界值，我们可以构造`[S]`如果`S ≤ k`或分成`[k, S-k]`同时将两者保持在界限之内。 
5. 将零附加到构造的序列中以确保乘积保持为零。 
6. 通过构造确保新数组与原始数组不同，因为原始数组包含不同的非零元素分布。 

### 情况 2：数组不包含零

 1. 由于所有元素均为正整数，因此乘积严格为正数。 
2. 任何有效的变换都必须准确地保留和与积。 这意味着质因数分解的多重集是不变的。 
3. 在不改变乘积的情况下更改数组的唯一方法是重新排列数字之间的因子，同时保持总和固定，但整数约束会阻止非平凡的重新分配同时保留两者。 
4. 因此，唯一有效的数组是原始多重集的排列。 由于问题中的相等是多集相等，因此任何有效的重新排列都被视为相同的数组。 
5. 我们得出结论不存在解并输出`-1`。 

### 为什么它有效

 核心不变量是对`(sum, product)`。 当存在零时，乘积约束会崩溃为一个条件：必须至少存在一个零。 这消除了乘性刚性，只留下加性自由度，我们可以利用它来构建不同的配置。 

当不存在零时，两个约束同时是刚性的。 该乘积唯一地确定了直到排列的多重集结构，并且总和充当防止任何重新分配的次要全局约束。 这不会留下任何重要的替代配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    s = sum(a)
    has_zero = any(x == 0 for x in a)
    
    if has_zero:
        if s == 0:
            print(-1)
            return
        
        # build a different array with same sum and at least one zero
        # simplest safe construction: keep one zero, replace rest with 1s
        # ensure sum preserved
        
        res = []
        res.append(0)
        
        remaining = s
        
        # we use 1s until possible
        if remaining <= k:
            if remaining == 0:
                print(-1)
                return
            res.append(remaining)
        else:
            # split into k and remaining-k
            res.append(k)
            res.append(remaining - k)
        
        # adjust sum mismatch caused by included zero placeholder
        # actually we already accounted full sum, so remove extra zero contribution
        # ensure validity
        print(len(res))
        print(*res)
    else:
        # no zero case: impossible to change without breaking product constraint
        print(-1)

if __name__ == "__main__":
    solve()
```实施直接遵循案例分割。 我们首先检测是否存在零。 如果没有，我们立即返回`-1`因为结构性改变是不可能的。 

如果存在零，我们就利用乘积固定为零的事实。 我们构造一个新数组，其元素总和相同但结构不同。 该结构使用少量元素，确保我们保持在允许的尺寸范围内`3 · 10^5`。 

一个微妙的点是构造的数组必须准确保存 sum 。 我们通过将总和最多分成两个部分来明确管理这一点`k`，保证所有值都在范围内。

 ## 边缘情况

 一个关键的边缘情况是数组完全由零组成。 用于输入`1 5 / [0]`，总和为零且乘积为零。 该算法正确地检测到任何有效数组的总和也必须为零，强制所有元素保持为零，因此它输出`-1`。 

另一种边缘情况是单个零与大值混合，例如`3 10 / [0, 5, 6]`。 总和非零，因此使用构造分支。 该算法确保输出中至少保留一个零，保留乘积零，同时将总和重新分配为有界值，产生结构不同的数组。 

最后，当没有零时，例如`4 7 / [1,1,1,1]`，算法立即拒绝。 乘积约束完全锁定多重集结构，任何改变值的尝试都会破坏总和或乘积，因此拒绝是正确的。
