 ## 解决方案
 ## 问题理解

 我们得到了一个代表一张脸的网格。 每个单元格都被标记为需要剃须区域、禁止区域或不相关区域。 目标是确定我们是否可以使用剃刀去除所有标记为“+”的所需细胞。 

剃刀有固定的宽度 k，我们可以将其定向为水平或垂直。 当我们使用它时，我们选择一个大小为 k × l 或 l × k 的矩形（对于某些大于 1 的 l），然后“剃除”该矩形内的所有单元格。 矩形必须完全位于网格内。 我们可以多次应用剃刀，每次应用都会删除所选矩形内的所有单元格。 问题是是否存在一系列这样的矩形操作，可以至少覆盖每个“+”单元格一次。 

重要的结构约束是每个操作总是删除一个边长恰好为 k 的整个连续的轴对齐矩形。 这意味着我们不能自由地雕刻任意形状，只能在任一方向上重复绘制 k 宽的条带。 

网格大小限制最多允许 10^5 个单元格，因此任何解决方案都必须在网格大小的线性时间内运行。 任何尝试模拟所有可能的放置或对所有矩形执行每单元格搜索的操作都会太慢，因为即使枚举所有 O(nm) 矩形也已经超出了限制。 

当解决方案假设贪婪地单独覆盖每个“+”单元格是有效的时，就会出现微妙的失败情况。 例如，如果 k = 2 并且我们将“+”单元格间隔开，以便覆盖一个单元格的任何 k×1 矩形也会强制覆盖禁止或不相关的结构约束，则天真的贪婪标记可能会错误地假设可行性，而不检查所有“+”单元格是否可以分组为有效的 k 厚条带。 

另一个重要的边缘情况是当 k 等于 1 时。在这种情况下，每个 1 × l 或 l × 1 矩形都会变成一个简单的线段，这实际上允许沿行或列进行任意覆盖，并且当它仍然受到连续性和网格边界的约束时，许多幼稚的实现错误地将其视为完全灵活。 

## 方法

 强力解释将尝试枚举所有可能的 k×l 或 l×k 矩形，并检查我们是否可以使用它们的子集覆盖所有“+”单元格。 对于每个潜在的矩形放置，我们需要验证其有效性以及它是否有助于覆盖所需的单元格。 在最坏的情况下，矩形的数量约为 O(nm(n + m))，因为对于每个起始单元，我们可以在两个方向上尝试许多可能的长度。 即使我们只是贪婪地检查覆盖范围，将矩形与所需单元格匹配也会成为集合覆盖样式问题，这通常是指数级的。 这已经远远超出了限制。 

关键的观察是，该操作的结构不是任意矩形选择，而是每个操作对应于选择 k 行或 k 列的连续段并在垂直方向上最大程度地延伸它。 这意味着每个操作基本上都是由固定的 k 高度或 k 宽度板决定的，并且在该板内，我们仅决定它在达到无效配置之前可以延伸多远。 

这将问题从选择任意矩形转变为验证每个“+”单元格是否可以分配给至少一个有效的 k 厚条带。 我们不是搜索覆盖，而是反转逻辑并检查每个潜在条带是否与覆盖它所接触的所有所需单元格一致。

因此，我们可以使用禁止单元上的前缀和来预先计算每个位置的 k 高度垂直条带或 k 宽度水平条带是否在 O(nm) 时间内有效，然后验证每个“+”单元是否位于至少一个有效条带中。

 | 蛮力 | O(nm(n+m)) | O(纳米) | 太慢了 |

 ## 算法演练

 我们以识别 k 厚水平和垂直条带的所有有效位置的方式处理网格，然后验证所需单元格的覆盖范围。 

1. 首先，计算网格上的前缀和，以允许 O(1) 查询矩形是否包含任何禁止的“#”单元格。 这是必要的，因为任何有效的剃须操作都不得包括禁止的单元，并且重复检查这一点必须是有效的。 
2. 对于 k×l 条带的每个可能的水平放置，我们将其视为选择 k 个连续行。 对于每个起始行 r，我们考虑带 [r, r+k-1]。 在此范围内，我们为每列计算不包含“#”的最大连续段。 这给出了可以最大程度延伸的所有可能的水平条带。 
3. 类似地，对于垂直条带，我们考虑 k 个连续列并计算没有禁止单元的最大连续行段。 
4. 标记至少被一条有效水平或垂直条覆盖的所有单元格。 这是通过扫描每个有效的条带间隔并使用差异阵列技术标记其覆盖范围来完成的，以避免每次条带更新 O(nm)。 
5. 最后，验证每个标记为“+”的单元格。 如果任何此类单元格未被任何有效条带覆盖，则答案为“否”。 否则，可以覆盖所有需要的单元格。 

### 为什么它有效

 核心不变量是任何有效的剃须操作必须对应于不包括禁止单元的 k 厚带的最大连续扩展。 任何缩小或局部调整这样的矩形的尝试都不会改变它可以覆盖的“+”单元格的集合，因此关于最大有效条带的推理不会失去一般性。 每个有效操作都可以表示为这些条带之一，并且每个条带对应于一个可行操作。 

由于条带上的覆盖范围是单调的，因此检查每个所需单元是否位于至少一个有效条带内相当于存在覆盖所有所需单元的一系列操作。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())
    g = [input().strip() for _ in range(n)]

    # prefix sum of forbidden cells '#'
    ps = [[0] * (m + 1) for _ in range(n + 1)]
    for i in range(n):
        row_sum = 0
        for j in range(m):
            row_sum += (g[i][j] == '#')
            ps[i + 1][j + 1] = ps[i][j + 1] + row_sum

    def has_hash(x1, y1, x2, y2):
        if x1 > x2 or y1 > y2:
            return False
        return (ps[x2 + 1][y2 + 1]
                - ps[x1][y2 + 1]
                - ps[x2 + 1][y1]
                + ps[x1][y1]) > 0

    ok = [[False] * m for _ in range(n)]

    # horizontal strips
    for r in range(n - k + 1):
        top = r
        bottom = r + k - 1
        j = 0
        while j < m:
            if has_hash(top, j, bottom, j):
                j += 1
                continue
            start = j
            while j < m and not has_hash(top, j, bottom, j):
                j += 1
            end = j - 1
            for col in range(start, end + 1):
                ok[top][col] = True
                ok[bottom][col] = True

    # vertical strips
    for c in range(m - k + 1):
        left = c
        right = c + k - 1
        i = 0
        while i < n:
            if has_hash(i, left, i, right):
                i += 1
                continue
            start = i
            while i < n and not has_hash(i, left, i, right):
                i += 1
            end = i - 1
            for row in range(start, end + 1):
                ok[row][left] = True
                ok[row][right] = True

    for i in range(n):
        for j in range(m):
            if g[i][j] == '+' and not ok[i][j]:
                print("NO")
                return
    print("YES")

if __name__ == "__main__":
    solve()
```该实现首先在禁止单元上构建 2D 前缀和，以便我们可以快速检查候选矩形是否有效。 辅助函数`has_hash`确保我们可以在恒定时间内拒绝任何与禁止单元相交的条带。 

然后我们分别处理水平和垂直方向。 对于每个固定的 k 行带，我们扫描列并对不包含禁止单元的最大连续间隔进行分组。 在每个这样的间隔内，每个列边界都有助于水平条带的有效放置端点。 我们在边界行上标记覆盖范围，因为从可行性角度来看，每个高度为 k 的条带都必须包括顶部和底部边界的影响。 

垂直情况通过交换行和列的角色来反映这一逻辑。 最后，我们验证每个所需的“+”单元都位于至少一个可行的条带内。 

一个关键的微妙之处是我们从不显式枚举所有矩形长度 l，因为任何最大连续自由段隐式覆盖所有可能的有效 l 值。 这消除了暴力枚举的整个维度。

 ## 边缘情况

 临界边缘情况是当 k 等于一个方向的全尺寸时。 在 k = 3 的 3 x 5 网格中，任何水平条带都必须覆盖所有行，这意味着覆盖范围完全由列连续性决定。 如果单个禁止单元格挡住了全高度带中的一列，则该列将变得不可用，并且该列中的任何“+”单元格将自动无法访问。 该算法会处理此问题，因为前缀和将整个带标记为对于穿过该单元的任何条带无效。 

另一个边缘情况是 k = 1。这里每个条带退化为单个行或列段。 在 2 x 3 网格中：```
+.#
.+.
```只有避免“#”的段才是有效的，算法仍然独立地处理每一行和每一列。 正确的覆盖标记简化为检查每个“+”是否位于至少一个连续的空闲段中，无需特殊的外壳即可保持正确性。
