 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，该数组通过点更新随时间变化。 每次更新后，我们必须能够回答数组中有多少位置具有使用 XOR 定义的特定“平衡”属性。 

对于一个索引$i$，考虑严格在左侧的所有元素的异或$i$，以及严格在右侧的所有元素的异或$i$。 如果这两个 XOR 值相等，则认为索引是特殊的。 

查询要么更改数组的一个元素，要么询问特殊索引的当前数量。 

数组长度和查询次数均达到$2 \cdot 10^5$，这会立即排除每个查询从头开始重新计算前缀和后缀 XOR。 每次更新后进行全面扫描$O(nq)$，这远远超出了可接受的范围。 

边界处出现微妙的边缘情况。 为了$i = 0$，左侧 XOR 为空，因此为零。 为了$i = n-1$，右侧的 XOR 为空且为零。 任何正确的解决方案都必须一致地处理这些定义，否则前缀或后缀处理中的离一逻辑将在单元素或两元素数组上失败。 

例如，如果数组是$[5]$，左右异或都为零，因此索引 0 始终有效。 如果幼稚的实现忘记了空的 XOR 约定，它可能会错误地排除这种情况。 

## 方法

 直接方法为每个查询重新计算前缀 XOR 和后缀 XOR。 每次更新后，我们重新计算两个数组并计算满足条件的索引。 每次重新计算成本$O(n)$，导致$O(nq)$总工作量。 和$2 \cdot 10^5$操作，这太慢了几个数量级。 

关键的观察结果是，可以以完全不需要前缀或后缀结构的方式重写条件。 

让$P$是整个数组的异或。 让$pref[i]$是元素的异或$0$到$i$。 然后对索引进行左异或$i$是$pref[i-1]$，正确的异或是$P \oplus pref[i]$。 条件变为：$$pref[i-1] = P \oplus pref[i]$$现在使用身份$pref[i] = pref[i-1] \oplus a[i]$。 代入得到：$$pref[i-1] = P \oplus (pref[i-1] \oplus a[i])$$取消$pref[i-1]$在两边使用 XOR 属性得到：$$0 = P \oplus a[i]$$简化为：$$a[i] = P$$因此，当索引的值等于整个数组的异或时，索引才是有效的。 

这将整个问题简化为维护更新中的两件事：数组的全局异或和每个值的频率。 每个类型 2 的查询都变成一次查找：当前有多少元素等于全局 XOR。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 强力重新计算每个查询的 XOR |$O(nq)$|$O(1)$| 太慢了 |
 | 维护总异或+频率图|$O(q)$|$O(n)$| 已接受 |

 ## 算法演练

 我们维护两个状态：整个数组的异或和数组中值的频率图。 

1. 通过计算初始数组中每个元素的出现次数来初始化频率图，并计算所有元素的 XOR。 这为所有未来查询设置基线状态。 
2. 对于位置的更新查询$i$，首先从频率图和全局 XOR 中删除旧值的贡献。 这是必要的，因为在应用新值之前数组状态必须保持一致。 
3. 在位置应用新值$i$，相应地更新频率图和异或。 在此步骤之后，维护的状态再次准确地反映当前数组。 
4. 对于要求平衡索引数量的查询，返回当前全局异或值的频率。 这是有效的，因为我们证明了当索引的值等于总异或时索引是有效的。 

### 为什么它有效

 正确性取决于根据前缀异或重写左右异或表达式，然后完全消除前缀变量。 最终条件简化为每个元素与单个全局标量（整个数组的异或）之间的逐点相等。 由于该标量始终在更新时精确维护，因此计算有效索引相当于计算该标量在数组中的出现次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n = int(input())
    a = list(map(int, input().split()))
    q = int(input())

    freq = {}
    total_xor = 0

    for v in a:
        total_xor ^= v
        freq[v] = freq.get(v, 0) + 1

    out = []

    for _ in range(q):
        tmp = input().split()
        if tmp[0] == '1':
            i = int(tmp[1])
            x = int(tmp[2])

            old = a[i]
            if freq[old] == 1:
                del freq[old]
            else:
                freq[old] -= 1

            total_xor ^= old

            a[i] = x

            total_xor ^= x
            freq[x] = freq.get(x, 0) + 1

        else:
            out.append(str(freq.get(total_xor, 0)))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案保留一个显式数组，以便可以在恒定时间内应用更新。 XOR 累加器通过删除旧值并添加新值来更新，这是有效的，因为 XOR 是它自己的逆。 频率字典跟踪有多少元素与任何给定值匹配，查询答案是简单的字典查找。 

一个常见的错误是在覆盖旧值之前忘记减少旧值的频率，这会导致多次更新后计数膨胀。 

## 工作示例

 考虑数组$[1, 2, 3]$。 

我们将总异或计算为$1 \oplus 2 \oplus 3 = 0$。 没有元素等于 0，所以答案是 0。 

| 步骤| 数组| 总异或| 频率图 | 回答 |
 | ---| ---| ---| ---| ---|
 | 初始| [1,2,3]| 0 | {1:1,2:1,3:1} | 0 |

 现在将索引 1 更新为值 1，给出$[1, 1, 3]$。 

| 步骤| 数组| 总异或| 频率图 | 回答 |
 | ---| ---| ---| ---| ---|
 | 更新后| [1,1,3]| 3 | {1:2,3:1} | 1 |

 这里只有等于 3 的索引值才有效，并且只有一个这样的元素。 

该跟踪表明该算法从不需要前缀计算，只需要一致维护单个聚合不变量。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(n + q)$| 每次更新都会修改映射和 XOR 累加器中恒定数量的条目 |

 该解决方案非常适合约束，因为两个限制都与输入大小成线性关系，并且所有操作平均时间都是恒定的。

 ## 边缘情况

 对于像这样的单元素数组$[x]$，该算法将总异或计算为$x$。 频率图恰好包含一次出现$x$，所以如果$x = 0$，答案为 1，否则为 0。实现可以正确处理此问题，因为空前缀和后缀隐式编码在 XOR 标识中，而不是特殊大小写中。 

对于同一索引的重复更新，删除和插入步骤可确保频率图不会重复计数值。 每次更新首先减去旧值，然后再添加新值，从而保持映射始终准确反映当前数组的不变性。 

对于所有值均为零的数组，总异或在所有更新过程中保持为零。 每个索引始终有效，频率图始终返回$n$，匹配平衡头寸的定义。
