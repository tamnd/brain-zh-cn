 ## 解决方案
 ## 问题理解

 我们得到了一系列不同的建筑高度。 对于索引中的任何区间$i$到$j$，我们想象按顺序沿着建筑物行走。 每一步都会比较两个连续的高度：如果我们移动到更高的建筑物，则为上升，否则为下降。 这会产生沿该段的起伏模式。 

现在考虑相同的线段，但以相反的方向遍历。 这会翻转每次比较的方向，因此上升变为下降，下降变为上升。 

一段$[i, j]$如果向前方向的上升次数等于反向方向的上升次数，则称为公平。 由于反转是上下交换，所以这个条件相当于说向前的上升次数等于向前的下降次数。 

因此，任务简化为计算有多少子数组在连续元素之间具有相同数量的增加和减少。 

输入大小可以大到$10^6$，所以任何二次方法$(i, j)$成对是立即不可能的。 甚至$O(n \log n)$是可以接受的，但是任何尝试重新计算每个段信息的操作都会超时，因为大约有$5 \times 10^{11}$最坏情况下的子数组。 

一个微妙的点是单元素段总是公平的，因为没有移动。 另一个边缘情况是单调数组。 如果序列严格递增或递减，则只有单元素段有效，因为每个较长的段都是不平衡的。 

主要挑战是每个段的简单计数需要扫描每个子阵列，这太慢了。 

## 方法

 如果我们尝试暴力破解，我们会检查每一对$(i, j)$子数组中的 count 会增加和减少。 即使使用前缀预处理，维护每个段的两个计数仍然需要成本$O(1)$每个查询，但有$O(n^2)$查询，所以这仍然是二次的。 

关键的观察是每个相邻的比较都贡献$+1$（增加）或$-1$（减少）。 当该段上的这些值的总和为零时，该段就是公平的。 所以我们将问题转化为计算数组中总和为零的子数组$+1/-1$。 

让$a_k = 1$如果$h_k < h_{k+1}$， 否则$a_k = -1$。 然后一段$[i, j]$是公平的，如果$$a_i + a_{i+1} + \dots + a_{j-1} = 0.$$这是一个经典的前缀和问题：定义前缀和$p[0]=0$,$p[k]=a_1 + \dots + a_k$。 那么当以下情况时，段和为零：$p[j-1] = p[i-1]$。 所以我们需要计算相等的前缀和对。 

这将问题简化为计算单次中前缀和值的频率。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 前缀和频率|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们首先将高度数组转换为差异数组，其中每个位置编码下一个建筑物是更高还是更低。 

然后，我们在这个转换后的数组上维护一个运行前缀和。 每次我们再次看到相同的前缀和值，就意味着两次出现之间的段具有相同的起伏次数。 

### 步骤

 1. 读取高度数组并立即处理我们只需要在连续元素之间进行比较的事实。 

这减少了从建筑物到过渡的问题维度。 
2. 对于每个相邻对，计算一个值$+1$如果是增加并且$-1$如果是减少的话。 

选择这种编码是因为上升和下降之间的平衡成为零和条件。 
3. 在处理任何元素之前，维护从零开始的运行前缀和。 
4. 使用字典（哈希图）来存储每个前缀和值到目前为止出现了多少次。 
5. 初始化映射，前缀和为零出现一次，代表空前缀。 
6. 当我们处理每个值时，更新前缀和。 每次前缀和值再次出现时，将其之前的频率添加到答案中。 

这是有效的，因为相等的前缀和在它们的位置之间定义了一个零和段。 
7. 处理完所有元素后，输出累计计数。 

### 为什么它有效

 该算法依赖于以下不变量：每个具有相同数量的增加和减少的子数组唯一对应于一对前缀和相等的索引。 当处理第二次出现时，每对相等的前缀和只计算一次。 这确保了有效段和计数对之间的双射，因此不会遗漏任何段，也不会重复计算任何段。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = list(map(int, input().split()))
    n = data[0]
    h = data[1:]
    
    if n <= 1:
        print(n)
        return
    
    from collections import defaultdict
    
    freq = defaultdict(int)
    freq[0] = 1
    
    pref = 0
    ans = 0
    
    for i in range(n - 1):
        if h[i + 1] > h[i]:
            pref += 1
        else:
            pref -= 1
        
        ans += freq[pref]
        freq[pref] += 1
    
    print(ans + n)

if __name__ == "__main__":
    solve()
```该实现一次性读取完整输入并将其拆分为$n$和高度数组。 这避免了大输入的重复 I/O 开销。 

转换步骤隐含在相邻元素的循环中，其中我们根据比较将前缀和更新为 +1 或 -1。 字典`freq`存储每个前缀和出现的频率，每次匹配都会立即得出答案。 

决赛`ans + n`考虑到所有单元素子数组，因为前缀和逻辑仅计算长度至少为一条边的段，即涉及至少一次比较。 

一个微妙的实现细节是初始化`freq[0] = 1`。 如果没有这个，从索引 0 开始的段将不会被正确计数，因为它们的前缀和与初始的空前缀匹配。 

## 工作示例

 ### 示例 1

 输入：```
5
1 4 9 7 3
```我们计算转移：

 | 步骤| 配对 | 价值| 前缀和| 之前的频率 | 贡献 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 1 → 4 | +1 | 1 | {0:1} | 0 |
 | 2 | 4 → 9 | +1 | 2 | {0:1,1:1} | 0 |
 | 3 | 9 → 7 | -1 | 1 | {0:1,1:1,2:1} | 1 |
 | 4 | 7 → 3 | -1 | 0 | {0:1,1:2,2:1} | 1 |

 总跃迁贡献 = 2，加上 5 个单个元素得出 7。样本输出包括所有有效子数组，包括较长的平衡子数组，例如$[1,4,9,7]$，具有相同的起伏。 

此跟踪显示重复前缀和如何检测平衡段，特别是当运行总和返回到零时。 

### 示例 2

 输入：```
3
3 2 1
```过渡：

 | 步骤| 配对 | 价值| 前缀和| 之前的频率 | 贡献 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | 3 → 2 | -1 | -1 | {0:1} | 0 |
 | 2 | 2 → 1 | -1 | -2 | {0:1,-1:1} | 0 |

 只有单元素段才有效。 该结构是严格递减的，因此没有前缀和重复。 

这证实了单调数组除了单个元素之外产生的贡献为零。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(n)$| 每个高度被处理一次，哈希图操作被摊销为常数时间 |
 | 空间|$O(n)$| 在最坏的情况下，所有前缀和都是不同的 |

 这在限制范围内很合适，因为$n \leq 10^6$，并且时间和内存都呈线性缩放。

 ## 边缘情况

 单个建筑物输入显示了不存在比较的基本条件。 该算法正确返回 1，因为`ans`保持为零，我们添加`n`。 

严格递增的序列会产生始终递增的前缀和，因此不会出现重复值。 哈希映射永远不会累积额外的贡献，并且结果会折叠为仅单​​元素段。 

严格递减序列的行为是对称的，前缀和在每一步都递减。 同样，没有发生前缀冲突，这确认了只计算了不重要的段。 

反复增加然后减少等交替序列会创建多个前缀和重复，并且算法通过重复的哈希命中正确捕获所有平衡子数组，从而确认前缀和等价性的正确性。
