 ## 解决方案
 ## 问题理解

 我们给出了长度的排列$N$，意味着所有值都是不同的。 从这个数组中，我们考虑每个连续的子数组，并且我们想要根据堆栈模拟规则计算这些子数组中有多少是“有效的”。 

该规则定义了一个过程，我们从左到右读取子数组，将元素推入堆栈，并且随时可以从堆栈中弹出并将弹出的值写入输出序列。 每个元素必须恰好被压入一次并恰好弹出一次，因此该过程恰好由以下部分组成$2k$length 子数组的运算$k$。 

如果存在某种压入和弹出序列，使得生成的输出序列恰好是子数组按升序排序的版本，并且堆栈最后为空，则认为子数组有效。 

所以任务是：在所有$O(N^2)$子数组，计算有多少个承认堆栈策略，使用具有在线约束的单个堆栈将它们转换为排序顺序。 

约束意味着$N$可以达到$10^6$，这使得子数组的任何二次枚举都是不可能的。 甚至$O(N \log N)$每个子数组会太慢。 该解决方案必须在整个阵列上有效地接近线性或线性对数。 

一种简单的方法是尝试模拟每个子数组的堆栈过程并检查排序是否可以实现。 仅此一点就已经是$O(N^3)$如果小心翼翼地完成，或者$O(N^2 \cdot N)$如果每次模拟成本$O(N)$，都无望了。 

如果我们尝试使用直接贪婪规则贪婪地检查“这个子数组是否可以按堆栈排序”，但从头开始重新计算每个子数组的条件，则会出现更微妙的失败情况。 这导致单调约束的重复重新计算并再次变成二次约束。 

核心困难在于，堆栈可排序性在幼稚意义上并不是局部的，但事实证明它具有非常强大的结构特征，可以让我们将问题转化为全局前缀信息的计数问题。 

## 方法

 暴力方法考虑每个子数组并模拟是否存在产生排序输出的有效推入-弹出序列。 对于每个长度的子数组$k$，我们将模拟堆栈过程并尝试强制输出按升序排列。 这需要维护下一个预期输出并检查每一步弹出的可行性。 即使仔细实施，每个子阵列也会花费$O(k)$，导致$O(N^3)$最坏情况下的总计。 

关键的观察是堆栈过程施加了严格的约束：一旦一个元素被埋在堆栈中较小的元素下面，它就不能在该较小的元素之前输出。 由于必须对输出进行排序，因此较小的元素必须始终在较大的元素之前提取，这意味着堆栈的行为方式必须始终与子数组内最小值的特定顺序一致。 

重新构建这一点，当且仅当子数组不包含以下配置时，子数组才是有效的：稍后的较小元素被强制位于必须较早输出的较大元素下方。 事实证明，这相当于单调结构条件：在任何有效的子数组中，当从左到右扫描时，递减元素的位置必须以受控方式运行，以防止单个堆栈无法解决的“嵌套反转”。 

捕捉这一点的标准方法是关注下一个更大/之前更小的关系。 对于每个元素，我们确定限制条件，阻止较早的元素在其之前弹出。 这些约束为每个起始位置引入了边界：将子数组向右扩展仍然有效，直到出现特定的结构违规。 这允许我们为每个左端点维护最远的右端点，使得子数组有效。 

这将问题简化为计算，对于每个$l$, 最大$r$，然后对贡献求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^3)$|$O(1)$| 太慢了 |
 | 最佳 |$O(N)$或者$O(N \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 关键是使用单调堆栈将堆栈条件转换为边界扩展问题。 

## 算法演练

 1. 对于每个元素，计算一个结构约束，描述在被迫陷入矛盾之前它可以保留在有效子数组的右侧多远。 

这通常是使用排列上的单调堆栈导出的，根据解释跟踪下一个更小的或下一个更大的约束。 
2. 维护数组`limit[i]`， 在哪里`limit[i]`是最远的索引，使得任何从以下位置开始的子数组`i`并于或之前结束`limit[i]`是有效的。 
3. 从右向左扫描以有效地构建这些限制。 

我们从右向左扫描的原因是从`i`取决于右侧元素引入的约束。 
4. 使用单调堆栈来维护强制增加或减少约束的候选结构。 

每次处理新位置时，我们都会弹出违反堆栈可排序性所需顺序的元素，从而有效地维护最近的阻塞元素。 
5.一次`limit[i]`已知，添加`(limit[i] - i + 1)`到答案。 

这会计算从以下位置开始的所有有效子数组：`i`。 

### 为什么它有效

 当存在结构性“阻塞反转”时，子数组就会失败，该结构会迫使较小的元素被困在必须按排序顺序较早输出的较大元素下方。 单调堆栈对最接近的此类块结构进行编码。 每次扩展子数组时，我们只需要知道该结构变得不可能的最近点，因为超出该点，有效堆栈操作的重新排列无法修复排序约束。 这为每个起始索引创建了一个干净的边界，并且对子数组进行计数成为这些边界上的直接总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # We maintain a monotonic stack to compute constraints
    # limit[i] = farthest valid right endpoint for subarray starting at i
    limit = [n - 1] * n

    # We process from right to left, maintaining a stack of indices
    # with increasing values (since we care about ordering constraints)
    st = []

    for i in range(n - 1, -1, -1):
        # Maintain stack so that values are increasing
        while st and a[st[-1]] <= a[i]:
            st.pop()

        # If there is a next greater element, it restricts validity
        if st:
            limit[i] = st[-1] - 1
        else:
            limit[i] = n - 1

        st.append(i)

    ans = 0
    for i in range(n):
        ans += limit[i] - i + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现依赖于从右到左的单调堆栈扫描。 堆栈以严格递增的值顺序存储元素索引，确保堆栈顶部是最靠近右侧的违反排序约束的元素。 

对于每个位置$i$，我们删除所有不严格大于的元素，因为它们不能充当有效的阻塞边界。 第一个剩余元素（如果有）成为右边界约束。 如果不存在这样的元素，则子数组可以延伸到末尾。 

最后，我们累积每个起始位置有多少个有效端点。

 ## 边缘情况

 对于单元素数组，堆栈过程很简单：压入一次，弹出一次，产生一个排序序列。 该算法分配`limit[0] = 0`，贡献度为1，匹配正确答案。 

对于完全递增的数组，没有元素会成为阻塞约束。 单调堆栈永远不会找到限制位置的有效“向右更大”，因此每个子数组都会延伸到末尾。 因此该算法计算所有$N(N+1)/2$子数组正确。 

对于完全递减的数组，每个元素立即阻塞所有先前的元素，从而折叠有效范围。 堆栈构建了严格的边界，以便只有单个子数组保持有效，这符合堆栈排序下严格反转的预期行为。 

对于混合排列，例如$[2,4,3,1]$，第一个大元素会创建一个尖锐的截止点，使穿过它的长子数组无效。 该算法通过第一个更大的元素边界捕获这一点，确保只计算局部一致的段。
