 ## 解决方案
 ## 问题理解

 我们得到一行整数。 在一步中，我们选择两个相邻的位置并在它们之间转移任意数量的值：我们添加一些整数$k$到一侧并减去相同的$k$从另一边。 这保留了阵列的总和，但允许在相邻位置之间任意重新分配质量。 

任务是确定所需的最小移动次数，以便每个位置最终得到一个非负值。 如果不可能，我们必须报告这一事实。 

约束条件$N \le 10^5$立即排除任何尝试模拟实际传输或搜索状态的解决方案，因为即使每步的线性操作数也会太慢。 行动的结构表明我们不是在优化幅度，而是在决定“重新分配行动”必须发生在哪里。 

考虑该操作的一个有用方法是，它允许我们跨两个邻居之间的边界移动任意数量，但每次选择的边界使用需要支付 1 的成本。 同一对上的多个应用程序永远没有帮助，因为它们的效果可以通过求和合并到单个操作中$k$。 这意味着真正的决定是我们激活哪些相邻对，而不是传输有多大。 

有一些边缘情况值得隔离。 

如果所有值都已经非负，则答案显然为零，​​因为不需要重新分配。 

如果数组的总和为负数，则任何移动序列都无法修复该数组，因为总和永远不会改变，而所有最终值必须至少为零，这会强制得出非负总数。 

一种更微妙的情况是，当总和为非负值但数组早期存在深负谷时。 尝试“立即修复局部负数”的天真贪婪可能会失败，因为从右侧借用可能需要协调使用多个相邻边，并且错误地计算每个值单位而不是每个边界激活的操作会导致错误的答案。 

## 方法

 暴力的观点是明确地模拟重新分配。 人们可以想象反复找到一个负面位置并将价值从其邻居之一推入其中，依次调整邻居。 每个这样的调整都可以通过数组传播，并且每个传播步骤对应于一个操作。 虽然原则上是正确的，但这种方法太慢，因为单个错误配置可能需要在长链上移动大量价值，并且每个移动决策都可能级联，导致二次或更糟糕的行为。 

关键的观察是，我们不关心移动了多少值，只关心是否使用过边界。 每个操作在单个边缘上的容量不受限制，因此同一相邻对上的所有传输都会分解为一个操作。 问题变成了计算必须使用多少个不同的“必要边界”来消除所有负赤字。 

这将问题转化为跟踪当我们从左到右扫描时必须修复负前缀不平衡的程度。 每当运行前缀总和降至零以下时，数组的左侧部分就会累积赤字，必须通过从右侧跨越至少一个边界发送值来补偿这一赤字。 每当我们遇到这样的新赤字，超出了之前的补偿所能覆盖的范围时，我们就被迫激活一个新的操作，而这些激活正是我们所计算的。

 | ---| ---| ---| ---|
 | 暴力模拟|$O(N^2)$|$O(1)$| 太慢了|
 | 带有赤字计数的前缀扫描|$O(N)$|$O(1)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时维护一个运行的前缀和，该前缀和表示当前位置当前可用的总值。 

1. 初始化变量`prefix`归零和计数器`ops`为零。 
2. 从第一个元素到最后一个元素迭代数组，将每个值添加到`prefix`。 
3. 如果在任何时候`prefix`变为负数，意味着左侧部分的赤字多于盈余。 这一缺陷必须通过从右侧拉取值来补偿，这必然需要激活至少一个跨越剩余后缀的边界操作。 我们递增`ops`减一并在概念上重置`prefix`回到零，因为我们想象一个操作用于抵消这个边界水平的赤字。 
4. 继续扫描阵列，对每个新段重复此过程。 每次运行总和再次低于零时，它代表了一个新的、独立的操作要求。 
5. 处理完整数组后，如果所有元素的总和为负，则输出 -1，因为没有重新分配可以修复全局赤字。 Otherwise, output`ops`。 

关键原因是，每次我们遇到新的负前缀区域时，以前的操作就不再有帮助了，因为它们只能在本地移动值，并且无法在不引入额外边界使用的情况下同时解决多个单独的赤字事件。 

### 为什么它有效

 前缀总和表示已处理前缀中的净可用质量。 负前缀和表示某个后缀必须通过至少一个相邻边界向该前缀贡献值。 一旦我们在概念上分配了一项行动来解决这一赤字，我们实际上就是在向未来的细分市场“借用”。 

在此点之后任何进一步低于零的下降都对应于新的独立借用要求，在不增加整个线路的流复杂性的情况下，无法通过重复使用相同的边界激活来满足该要求。 因此，每当前缀和达到超出已补偿的新的赤字状态时，就会强制进行新的操作，并且没有解决方案可以使用比这些强制事件更少的操作。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    if sum(a) < 0:
        print(-1)
        return
    
    prefix = 0
    ops = 0
    
    for x in a:
        prefix += x
        if prefix < 0:
            ops += 1
            prefix = 0
    
    print(ops)

if __name__ == "__main__":
    solve()
```该实现遵循上述扫描。 初始总和检查处理不可能性条件：由于操作保留总和，负总和使目标无法到达。 

这`prefix`变量跟踪扫描时有多少剩余可用。 每当它降至零以下时，我们将其解释为强制操作，从后缀中提取足够的值以取消赤字，并且我们将其重置，因为假设该操作完全补偿了当前的违规。 

重置步骤是微妙的部分：它不是模拟精确的传输，而是表示一次操作足以消除当前的赤字边界条件。

 ## 边缘情况

 对于单元素数组，如果它为非负数，则算法立即输出 0，否则输出 -1，因为没有邻居就不可能进行重新分配。 

对于总和为负的数组，早期检查在任何扫描之前都会正确拒绝这种情况，因为没有任何操作序列可以增加总质量。

 对于具有重复的小负数与正数交错的数组，前缀扫描正确地触发多个赤字重置，反映了从后缀引入外部值的重复必要性。
