 ## 解决方案
 ## 问题理解

 我们得到了一系列建筑物，每座建筑物都有不同的高度。 对于索引中的任何建筑物间隔`i`到`j`，我们考虑两种遍历：从左到右移动和从右到左移动。 当我们移动时，我们记录每一步的高度是向上还是向下，从而产生一串由 U 和 D 组成的移动。 

一对`(i, j)`如果从以下位置行走时看到的起伏模式被称为有效`i`到`j`与从 步行时看到的模式相同`j`到`i`。 任务是计算存在多少个这样的区间。 

输入大小可以大到一百万栋建筑物，这会立即排除任何检查所有对的二次方法。 解必须接近线性或线性算数，因为即使`O(n^2)`在最坏的情况下意味着大约 10^12 次比较，这远远超出了可行的限制。 

一个微妙的点是，定义比较序列，而不仅仅是上升和下降的计数。 两个间隔可能具有相同的上升和下降次数，但仍然会失败，因为顺序不同。 另一个重要的边缘情况是单个建筑间隔`(i, i)`始终有效，因为两个遍历方向都会产生空序列。 

一个天真的错误是假设对称意味着“相同数量的增加和减少”，这是不正确的。 另一个错误是将反转视为自动匹配，当 U/D 编码中的模式不是回文时，这种匹配会失败。 

## 方法

 直接方法将枚举所有`(i, j)`对，模拟遍历`i`到`j`，记录U/D序列，然后从`j`到`i`并进行比较。 每次模拟需要`O(j-i)`时间，因此总成本变为`O(n^3)`在最坏的情况下，如果按字面意思或`O(n^2)`即使小心重复使用。 和`n = 10^6`，这是不可能的。 

关键的观察结果是 U/D 序列仅取决于相邻元素之间的比较。 对于固定间隔，前向模式由符号数组确定`s[k] = +1 if h[k] < h[k+1] else -1`。 反向遍历会翻转方向，因此我们有效地将序列与其反向版本进行比较，但方向反转引起了符号翻转。 

两次遍历产生相同的 U/D 序列的条件转化为强大的结构约束：区间内的比较模式在反转下必须保持不变。 只有当每个边缘比较的方向在反转下一致时才有可能，这迫使高度排序中的结构非常严格。 

这种刚性导致了重新表述：有效区间恰好对应于比较序列形成单个“山状”单调分段结构的区间，该结构围绕其极值对称。 更具体地说，如果我们将数组映射为一系列斜率，则有效区间是序列相对于其最大值或最小值对称的区间，并且方向变化从两端完美对齐。 

这可以简化为以每个位置为中心的计数间隔，其中向外的扩展保留了严格的交替结构。 由于高度不同，我们可以使用两指针扩展或单调堆栈结构来识别每个元素周围的最大有效跨度。 

最佳解决方案计算对于每个位置，我们可以向左和向右延伸多远，同时保留所需的对称比较结构。 然后以线性时间从这些跨度中计算每个有效间隔。

 ## 算法演练

 我们依赖这样一个事实：有效性完全取决于从区间两端观察时比较方向的表现。 

### 步骤

 1. 将height数组转换为sign数组，其中每个位置记录步数是否来自`i`到`i+1`是向上还是向下。 这将问题压缩为关于方向变化而不是绝对值的推理。 序列变成字符串`{U, D}`。 
2. 观察一下，当我们反转一个区间时，方向顺序也反转了，同时也反转了。 为了使间隔保持相同，序列在这种组合变换下必须保持不变。 
3. 这种不变性迫使结构的方向变化只能以围绕枢轴的单一对称方式发生。 该枢轴对应于区间内的唯一局部极值。 
4. 对于每个位置`k`，将其视为潜在的对称中心并尝试向外扩展。 我们延长`l`和`r`只要诱导的比较方向与镜像模式保持一致即可同时进行。 
5. 在扩展过程中，我们比较`(k - d, k + d)`并保证反转规则下左右两侧的坡度方向正确对应。 如果发生任何不匹配，扩展就会停止。 
6. 每一次成功的扩张`k`定义一系列有效间隔，我们计算有多少个`(i, j)`对由每个有效的对称半径生成。 

### 为什么它有效

 核心不变量是有效区间在反转下必须保持相同的邻接关系。 因为所有高度都是不同的，所以每个局部比较严格来说都是两种状态之一，因此坡度方向上的任何不对称都会立即破坏可逆性。 通过强制围绕枢轴对称并仅在镜像比较一致时进行扩展，我们确保每个计数间隔都满足两个方向上 U/D 序列所需的相等性，并且不会形成无效间隔，因为任何不匹配都会违反第一个不同对的对称性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    h = list(map(int, input().split()))
    
    if n == 1:
        print(1)
        return

    # sign array: 1 if up, 0 if down
    s = [1 if h[i] < h[i+1] else 0 for i in range(n-1)]

    # expand around each center
    ans = n  # all singletons
    
    for center in range(n):
        l = center
        r = center
        
        while l > 0 and r < n-1:
            # compare slope to the left and right
            if s[l-1] != s[r]:
                break
            l -= 1
            r += 1
        
        ans += (center - l)  # number of valid expansions ending at center
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将输入压缩为二进制斜率表示，以便每个相邻对都减少为单个方向位。 这消除了对实际值的依赖并仅保留结构。 

主循环将每个索引视为对称中心并向外扩展，同时强制左侧斜率镜像右侧斜率。 条件`s[l-1] == s[r]`确保向外迈出的方向在反转时保持相同的方向模式。 

贡献`(center - l)`计算有多少左边界产生终止于该中心位置的有效对称间隔。 单元素区间单独处理。

 ## 边缘情况

 单元素数组是最简单的有效情况。 该算法单独处理它并返回 1，因为不存在斜率比较。 扩展逻辑永远不会被触发，这避免了越界访问。 

严格单调数组也很重要。 在这种情况下，所有斜率都是相同的，因此每个区间在反转下都是平凡对称的，因为两个方向都会产生均匀的序列。 扩展条件永远不会失败，并且所有间隔都一致地计数为有效的单向运行。 

高度交替的序列暴露了朴素对称假设的失败模式。 在这些情况下，当围绕大多数中心扩展时，斜率不匹配会立即发生，并且只有非常短的对称段能够幸存。 该算法在第一次不匹配时正确停止扩展，确保不计算无效间隔。
