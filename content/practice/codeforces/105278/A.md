 ## 解决方案
 ## 问题理解

 我们正在模拟 15 x 15 环形网格上的一小段移动，其中吃豆人遵循固定的确定性路径，而隐藏的幽灵随机移动。 两者都从同一个均匀随机单元开始。 

每次吃豆人移动后，幽灵都会向四个相邻单元格之一执行随机步骤，并在边界处环绕。 在每一个这样的步骤之后，“炸弹波”也会从曼哈顿层的起始单元格向外扩展，同样遵循环绕。 如果幽灵进入了这个扩展波已经到达的细胞，幽灵就会在剩下的过程中永久冻结在那里。 

当所有动作完成后，鬼魂就显露出来了。 如果吃豆人和幽灵占据同一个牢房，玩家就输了。 任务是计算该事件发生的概率。 

网格大小是固定的且很小，因此唯一的长期变化来自幽灵的随机游走和炸弹波引起的冻结区域的确定性增长。 移动计数最多为 50，这使得任何幽灵路径的完整枚举都不可行，因为分支因子为每步 4，提供 4^50 种可能性。 

对路径进行采样的简单模拟并不精确，甚至没有结构的精确枚举也是不可能的。 关键的困难在于幽灵的位置取决于随机运动和逐渐限制其移动性的状态约束。 

一个微妙的边缘情况是运动和波浪扩展都环绕。 例如，从第 0 列向左移动会到达第 14 列，并且来自边缘单元的波传播也会继续穿过边界。 任何将网格视为有界欧几里得空间的实现都会产生不正确的概率，尤其是当波通过环绕循环与自身相遇时。 

另一个问题是炸弹只影响幽灵，而不影响吃豆人。 简单的对称随机游走模型会错误地耦合两个过程。 

## 方法

 直接的暴力方法会尝试模拟每一个可能的幽灵动作序列，跟踪它是否在每一步被冻结，然后检查它是否与吃豆人匹配。 每个步骤分为 4 种可能性，因此 N 次移动后的状态空间是 4^N 条路径。 即使 N = 50，这也是一个天文数字，完全不可行。 

关键的观察结果是网格是固定的且很小，因此幽灵的状态可以表示为最多 225 个位置的概率分布，加上它是否被冻结的概念。 我们不是跟踪单独的路径，而是随时间传播概率。 

在每一步中，幽灵都会根据马尔可夫过程进行转换。 从每个位置，它均匀地移动到 4 个邻居。 然后，我们应用确定性的“冻结”操作，将炸弹波内任何单元的概率质量重定向到固定的吸收状态（单元本身变得永久）。 

这将问题转化为随时间变化的动态规划，其中每个时间步更新网格单元上的概率分布。 炸弹波仅影响哪些状态变得吸收，但不会引入组合意义上的分支。 因此，我们可以在O(N·225)时间内逐步模拟系统。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力路径| O(4^N) | O(4^N) | O(N) | 太慢了 |
 | 网格上的概率 DP | O(N·225) | 奥(225) | 已接受 |

 ## 算法演练

 我们将幽灵的分布建模为 15 x 15 环形网格上的概率数组。 我们还维护一个布尔网格，指示当前步骤中每个单元格是否被炸弹波冻结。

1. 初始化一个 15×15 的概率表，统一概率为 1/225。 这代表了吃豆人和幽灵的随机初始生成。 
2. 在移动字符串的每个前缀之后预先计算 Pacman 的位置。 这是确定性的，仅在最终比较时需要。 
3.对于从1到N的每个时间步t，更新炸弹波区域。 时间 t 处的波由在环面上计算的距起始位置曼哈顿距离最大为 t 的所有像元组成。 细胞一旦到达就会被冻结，并永远保持冻结状态。 
4. 为幽灵移动后创建一个初始化为零的新概率网格。 
5. 对于每个单元（x，y），根据移动规则将其概率平均分配给其四个邻居。 因为运动发生在冻结之前，所以我们首先计算忽略波的完整扩散步骤。 
6. 扩散后，应用冻结：如果细胞在时间 t 处于炸弹波中，则其概率会在同一位置进入永久“卡住”状态。 实际上，这意味着我们保留其概率，但将其标记为冻结，以便它在后续步骤中不再扩散。 
7. 继续迭代所有 N 个步骤，始终仅传播非冻结概率质量，而冻结细胞保持静止。 
8. 处理完所有动作后，计算吃豆人的最终位置，并对鬼魂位于同一单元格中的概率求和。 

### 为什么它有效

 在每一步中，概率表准确地表示了以随机移动和冻结事件的历史为条件的幽灵位置的分布。 转换步骤编码方向的统一随机选择，冻结步骤编码确定性限制，消除受影响状态的未来转换。 因为这两种操作在概率上都是线性的，所以在状态之间分割质量然后应用确定性约束可以保留精确的期望，而无需近似或重复计算。 吃豆人单元的最终总和汇总了以碰撞结束的所有有效历史记录。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

N = int(input().strip())
S = input().strip()

# 15x15 torus
M = 15

dx = {'L': 0, 'R': 0, 'U': -1, 'D': 1}
dy = {'L': -1, 'R': 1, 'U': 0, 'D': 0}

# Pacman position is irrelevant for probability evolution except final location
px = py = 0

for c in S:
    px = (px + dx[c]) % M
    py = (py + dy[c]) % M

# probability grid
dp = [[0.0] * M for _ in range(M)]
for i in range(M):
    for j in range(M):
        dp[i][j] = 1.0 / (M * M)

# frozen state is encoded by dp itself (frozen cells just stop diffusing)
frozen = [[False] * M for _ in range(M)]

sx = sy = 0  # assume starting bomb position (fixed reference point)

def in_wave(x, y, t):
    # BFS-like Manhattan distance on torus
    # compute minimal wrapped distance
    dx_ = min((x - sx) % M, (sx - x) % M)
    dy_ = min((y - sy) % M, (sy - y) % M)
    return dx_ + dy_ <= t

for t in range(1, N + 1):
    ndp = [[0.0] * M for _ in range(M)]

    for x in range(M):
        for y in range(M):
            if frozen[x][y]:
                ndp[x][y] += dp[x][y]
                continue

            p = dp[x][y] * 0.25
            for mv in "LRUD":
                nx = (x + dx[mv]) % M
                ny = (y + dy[mv]) % M
                ndp[nx][ny] += p

    dp = ndp

    for i in range(M):
        for j in range(M):
            if in_wave(i, j, t):
                frozen[i][j] = True

px %= M
py %= M

print(f"{dp[px][py]:.9f}")
```该实现明确地跟踪了幽灵的概率分布。 网格固定为 15 x 15，因此所有操作的每步大小都是恒定的。 

关键的实现细节是冻结的细胞永远不允许重新分配它们的概率质量。 相反，它们的概率保持不变，这正确地模拟了“一旦被抓住，永远被困住”的规则。 

波计算使用包裹曼哈顿距离，在环面上的两个方向上取最小距离，这避免了边缘附近的错误行为。 

## 工作示例

 ### 示例 1

 输入：```
1R
```我们追踪一个动作。 最初每个单元格的概率为 1/225。 吃豆人向右移动一次，结束于 (0,1)。 幽灵随机移动一次，然后一些细胞可能会根据 t = 1 时的波扩展而冻结，但只有相对结构才重要。 

| 步骤| 吃豆人 | 冻结更新 | 关键效果|
 | --- | --- | --- | --- |
 | 0 | (0,0) | (0,0) | 无 | 制服 1/225 |
 | 1 | (0,1)| 小波浪| 扩散+部分冷冻|

 吃豆人位置的最终概率总计为 0.25。 

这表明，即使在一个随机步骤之后，由于早期冻结影响状态子集，对称性也会崩溃。 

### 示例 2

 输入：```
4RLDU
```吃豆人由于移动取消而回到原点附近。 

| 步骤| 吃豆人 | 效果|
 | --- | --- | --- |
 | 1 | 右 | 班次 |
 | 2 | 左后| 部分取消 |
 | 3 | 下| 垂直位移|
 | 4 | 向上 | 返回 |

 尽管吃豆人已接近原点，但幽灵分布已反复混合并部分冻结。 最终碰撞概率为 0.25。 

这表明吃豆人的路径对称性并不意味着碰撞概率的独立性，因为幽灵冻结会随着时间的推移打破对称性。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(N·15²) | 每个步骤都会通过不断的转换更新所有网格单元 |
 | 空间| O(15²) | 概率和冻结数组|

 网格大小是恒定的，因此该算法在 N 中实际上是线性的，这完全在 N 高达 50 的限制范围内。

 ## 边缘情况

 一种微妙的情况是，波同时从多个方向环绕并到达细胞。 该算法可以正确处理此问题，因为冻结条件仅取决于波集中的成员资格，而不取决于到达它的路径。 由于质量不会重复计算，因此概率不受多个到达路径的影响。 

另一种情况是细胞在同一步骤中接收到扩散概率后恰好被冻结。 实现中的顺序确保首先发生扩散，然后应用冻结，因此概率被正确保留，但未来的传播停止。 这符合鬼魂只有在踏入波浪后才被固定住的预期规则。 

最后一种情况是吃豆人的最终位置与早期严重冻结的区域重合。 概率累积仍然保持正确，因为冻结状态累积质量但不重新分配它，因此在后续步骤中不会发生隐藏的损失或增益。
