 ## 解决方案
 ## 问题理解

 我们得到一个正整数数组和许多查询。 每个查询都会选择该数组的一个连续段。 对于该部分，两个玩家有效地在其元素的选定子序列上玩游戏。 

游戏中的一步棋选择一个大于1的元素，并用其大于1的真因数之一代替它。这严格地减少了值，所以每个元素只能减少到变成1。当玩家要移动时，所有数字都已经是1，他不能移动，立即获胜。 

对于每个查询段，玩家 A 可以选择该段的非空子序列作为起始游戏数组。 我们必须计算有多少个这样的子序列可以保证玩家 A 在最佳玩法下获胜（假设 A 首先移动）。 

所有查询的约束总大小都很小，每个测​​试文件的数组长度和查询的总和限制为 1000。 这立即告诉我们，$O(n^2)$甚至$O(n^2 \log n)$每个测试用例的风格解决方案是可以接受的，我们不应该期望任何涉及大型结构的大量每个查询重新计算的事情。 

一个微妙的点是真正决定胜利者的因素。 每个数字的行为都是独立的：每一步都会减少一个元素。 当所有选择的元素都变成 1 时游戏结束，最后移动的玩家获胜。 所以这个游戏相当于一个堆游戏，每个数字贡献一定的移动次数，等于它在达到1之前可以减少多少次。 

如果我们定义一个值$g(x)$作为次数$x$可以用适当的除数代替，直到变为1，然后每个选定的元素都有贡献$g(x)$移动。 总游戏长度是所选子序列上这些值的总和。 由于玩家轮流进行，当且仅当总步数为奇数时，玩家 A 获胜。 

因此，任务简化为：对于每个查询段，计算其权重总和的子序列$g(b_i)$很奇怪。 

一个天真的误解是认为因式分解序列的结构很重要。 它并不重要：只有有效除数减少步骤的最大数量很重要，因为每次移动总是减少单个独立元素。 

当元素为 1 时，会出现边缘情况。例如，如果段中的所有元素均为 1，则每个$g(1)=0$，因此每个子序列的总和为 0，玩家 A 在所有情况下都输了。 正确答案是 0。 

另一种边缘情况是当段只有一个元素时$x>1$。 那么答案是 1 如果$g(x)$是奇数，否则为 0，因为唯一的子序列是选择该元素。 

## 方法

 蛮力的想法很简单。 对于每个查询段，枚举每个非空子序列，计算总和$g(b_i)$，并检查是否为奇数。 这是正确的，但非常昂贵：一段长度$m$有$2^m - 1$子序列，所以即使对于中等$m$这会爆炸。 

关键的简化来自于将结构与奇偶校验分开。 我们不需要精确的总和，只需要它是否是奇数。 这意味着每个元素贡献 0 或 1 模 2，具体取决于是否$g(b_i)$是偶数还是奇数。 一旦我们将每个元素转换为二进制奇偶校验权重，问题就变成了计算二进制数组中具有奇数和的子序列。 

这是一个经典的组合计数问题。 让$k$是段中奇偶校验为 1 的元素的数量，并且$m-k$奇偶校验为 0 的数字。奇偶校验为 0 的元素不会影响任何子序列和的奇偶校验，但它们确实会将子序列的数量乘以因子$2^{m-k}$。 

其中$k$相关元素，恰好一半的子集具有奇数和，当$k>0$。 这是通过将每个子集与其第一个奇偶校验元素的补码翻转配对得出的。 

所以答案就变成了：$$2^{m-k} \cdot 2^{k-1} = 2^{m-1} \quad \text{if } k>0$$和 0 如果$k=0$。 

因此，答案仅取决于该段是否包含至少一个具有奇数的元素$g(x)$。 我们可以预先计算$g(x)$所有值的奇偶性直到$10^7$，并且对于每个查询，只需检查段中的任何位置是否具有奇奇偶校验。 有了前缀和，这就变成了$O(1)$每个查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^m)$每个查询 |$O(1)$| 太慢了 |
 | 最佳 |$O(n + q)$每次测试 |$O(n)$| 已接受 |

 ## 算法演练

 我们首先需要明白什么$g(x)$是。 每次我们选择一个大于1的除数，我们就会减少这个数，最终达到1。移动的次数正是$x$以重数来计算。 例如，$12 = 2^2 \cdot 3$给出$g(12)=3$。 每一步都会有效地消除一个素因数。 

所以奇偶性为$g(x)$简单来说就是质因数的总数是否$x$很奇怪。 

一旦我们将数组转换为二进制数组，其中每个元素如果具有奇数素因数计数则为 1，否则为 0，每个查询都会减少为对具有奇数和的子序列进行计数。 

现在我们计算答案如下。 

1. 预先计算所有值的质因数数量的奇偶性，直到最大可能值$b_i$。 我们使用类似筛子的方法来累积素因子计数。 
2. 将输入数组转换为奇偶校验数组$p[i]$， 在哪里$p[i]=1$如果$g(b_i)$为奇数，否则为0。 
3. 构建前缀和数组$p$这样每个查询段都可以快速判断它是否包含至少一个索引$p[i]=1$。 
4. 对于每个查询$[l, r]$，使用前缀和计算段中 1 的数量。 如果为零，则输出0。 
5. 否则，计算$2^{(r-l+1)-1}$模数$998244353$，因为奇偶校验元素的任何非空集合都保证一半的子集贡献奇数和。 

### 为什么它有效

 整个游戏减少到总移动数的奇偶性，并且每个元素独立地对该奇偶性做出贡献。 一旦我们将每个元素压缩为单个位，子序列就成为包含或排除元素的独立选择，并且奇偶校验与 XOR 呈线性关系。 唯一重要的因素是是否存在至少一个奇怪的贡献者； 如果不存在，则每个子序列和都是偶数，否则对称性保证所有子序列的恰好一半产生奇数和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
MAXV = 10**7

# Precompute smallest prime factor up to max value seen in constraints
# We only need parity of number of prime factors (with multiplicity)
# We'll compute omega parity using sieve

spf = list(range(MAXV + 1))
is_prime = [True] * (MAXV + 1)
is_prime[0] = is_prime[1] = False

primes = []
for i in range(2, MAXV + 1):
    if is_prime[i]:
        primes.append(i)
        if i * i <= MAXV:
            for j in range(i * i, MAXV + 1, i):
                is_prime[j] = False

# compute omega parity
omega_parity = [0] * (MAXV + 1)
for i in range(2, MAXV + 1):
    if is_prime[i]:
        omega_parity[i] = 1
    else:
        # factor using primes
        x = i
        cnt = 0
        for p in primes:
            if p * p > x:
                break
            while x % p == 0:
                cnt ^= 1
                x //= p
        if x > 1:
            cnt ^= 1
        omega_parity[i] = cnt

t = int(input())
out = []

for _ in range(t):
    n, q = map(int, input().split())
    b = list(map(int, input().split()))

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + omega_parity[b[i]]

    inv2 = (MOD + 1) // 2

    for _ in range(q):
        l, r = map(int, input().split())
        ones = pref[r] - pref[l - 1]
        length = r - l + 1

        if ones == 0:
            out.append("0")
        else:
            out.append(str(pow(2, length - 1, MOD)))

print("\n".join(out))
```该实现首先使用筛选和试除法构建质因数奇偶校验表。 这是可以接受的，因为测试中的总约束很小。 然后，使用前缀和和模幂在恒定时间内回答每个查询。 

一个微妙的细节是，我们实际上不需要知道游戏中如何选择因素，只需要知道因素去除步骤总数的奇偶性。 这就是将游戏分解为简单的组合计数问题的原因。 

## 工作示例

 考虑一个数组$[2, 3, 4]$和一个查询$[1, 3]$。 假设奇偶校验分类给出$p = [1,1,0]$。 前缀和告诉我们有两个。 

我们隐式计算所有子序列。 任何子序列必须包含三个位置的元素。 有$2^3 = 8$总子序列。 由于至少有一个奇偶校验一元素，所以组合配对后所有子序列的恰好一半贡献奇数和，所以答案是$2^{3-1}=4$。 

| 步骤| 细分 | 奇偶校验计数 | 长度 | 回答 |
 | --- | --- | --- | --- | --- |
 | 1 | [2,3,4]| 2 | 3 | 4 |

 这表明奇偶校验位的精确分布除了非零之外并不重要。 

现在考虑一个所有值都是 2 的幂的段，例如$[2,4,8]$。 每个都有偶数或奇数奇偶校验，具体取决于指数计数，但假设所有映射都为 0。那么没有子序列可以产生奇数和，因此答案为 0。 

| 步骤| 细分 | 奇偶校验计数 | 长度 | 回答 |
 | --- | --- | --- | --- | --- |
 | 1 | [2,4,8]| 0 | 3 | 0 |

 这证实了所有值在奇偶校验上都是中性的边缘情况。

 ## 边缘情况

 当段仅包含 1 时，每个元素贡献零移动，因此每个子序列具有偶数总奇偶校验。 该算法正确地将前缀和计算为零并立即输出 0。 

当段中恰好有一个元素具有奇数素因子奇偶校验时，前缀和为 1，因此答案变为$2^{0}=1$，匹配单个有效的获胜子序列。 

当段有许多零和几个一时，前缀和仍然正确地检测到至少一个贡献元素的存在，并且组合因子自动考虑剩余的中性元素，而无需显式地迭代它们。
