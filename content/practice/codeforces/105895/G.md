 ## 问题理解

 我们得到一个整数序列，目标是使用特定操作使所有元素相等。 每个操作都是通过选择一个索引来执行的$i$，然后选择三种效果之一。 仅第一个效果发生变化$a_i$通过增加或减少 1。第二个效果看起来在右侧$i$，并且每个元素严格大于$a_i$减少 1。第三个效果看起来在左侧$i$，并且每个元素都严格小于$a_i$增加了 1。 

重要的一点是，每个操作在一个方向上都是全局的，但取决于与所选枢轴值的比较$a_i$。 任务是确定使每个测试用例的所有元素相等所需的此类操作的最小数量。 

输入大小限制允许最多$10^5$elements in total across all test cases. 这立即排除了对索引对或三元组的任何二次或三次推理。 任何尝试简单地模拟操作或重新计算全局效应的解决方案都会失败，因为每个操作本身都可能触及$n$元素，甚至重复$O(n)$时间变得太大。 

一个微妙的困难是操作并不纯粹是本地的。 一次移动可能会根据相对顺序改变许多值，这使得贪婪模拟变得不可靠。 

常见的失败案例来自于假设操作仅影响所选索引。 例如，如果除一个之外所有元素都相等，则在本地修复异常值似乎是最佳选择，但实际上操作可能会无意中改变许多其他值。 

## 方法

 蛮力方法将直接模拟该过程。 从任何状态，我们都可以尝试每个索引和每个操作类型，将其应用到数组，并在状态上运行最短路径搜索。 这在理论上是正确的，因为每个操作都有单位成本，因此数组空间上的 BFS 或 Dijkstra 会找到最小步数。 然而，状态空间是巨大的，因为每个数组条目可能变化很大，使得状态数量在值范围内呈指数增长。 即使存储单个状态也已经$O(n)$，所以这是完全不可行的。 

关键的见解是停止将序列视为独立值，而是将操作解释为相对于所选阈值移动质量。 每个操作本质上都与数组的排序结构交互，而不仅仅是绝对值。 如果我们想象选择一个目标最终值$x$，那么每个元素都必须调整以达到$x$，并且成本成为需要多少“向上”和“向下”移动的函数。 

关键的观察是，正确的答案仅取决于值的分布，而不取决于它们的位置。 当结构正确时，定向操作有效地允许我们取消增加和减少调整之间的贡献。 在重新表述操作的效果之后，问题就简化为计算需要多少总调整以及我们如何有效地通过数组结构传播该调整。 

一旦根据相对于候选最终值的贡献进行重写，最优策略就变成了一个平衡问题：我们实际上为净增加和减少付出了代价，但其中一些可以通过第二次和第三次操作在头寸之间共享。 一旦值被标准化和分组，这就会产生线性扫描解决方案。

 ## 算法演练

 我们通过固定候选最终值来重新构建问题$x$。 每个元素都必须成为$x$，所以每个$a_i$贡献了所需的调整$|a_i - x|$，但成本不仅仅是这个总和，因为操作可以以结构化方式影响多个元素。 

1. 对数组进行排序，以便我们可以推断全局结构而不是位置。 排序使我们能够解释单调序列中流动时的增加和减少。 
2. 考虑目标值$x$。 将数组拆分为小于以下的元素$x$并且大于$x$。 总的向上需求是$x - a_i$超过下面的元素$x$，向下的需求是$a_i - x$超过上面的元素$x$。 这两个量代表必须转移多少质量。 
3. 观察每个操作都可以被解释为向上或向下推动值，同时影响枢轴一侧的所有元素。 这意味着单个操作可以同时服务多个单元调整，但前提是这些调整方向一致。 
4. 结构简化的关键在于选择时实现成本最优$x$来平衡这些流量。 最好的$x$是现有数组值之一，因为在间隙之间移动目标只会增加不平衡，而不会改善共享取消。 
5. 对于每位候选人$x$，在考虑定向传播产生的重叠后，将成本计算为所需的向上和向下调整的最大值。 最佳答案是所有候选答案中的最小值。 
6. 预先计算排序数组上的前缀和，以便每个候选者$x$可以评估为$O(1)$，导致总体$O(n \log n)$由于排序的解决方案。 

### 为什么它有效

 该算法依赖于一个不变量：任何操作都在一个方向上精确贡献一个有效校正单位，但如果多个局部不平衡位于所选枢轴的同一侧，则可以同时减少多个局部不平衡。 通过排序和聚合，我们确保所有导致相同方向不平衡的元素都被分组，因此操作的共享效果可以通过前缀和完全捕获。 由于成本函数在可能的目标值上是凸的，因此限制为数组值可以保留全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        a.sort()
        prefix = [0] * (n + 1)
        for i in range(n):
            prefix[i + 1] = prefix[i] + a[i]
        
        total = prefix[n]
        ans = float('inf')
        
        for i in range(n):
            x = a[i]
            
            left_cost = x * i - prefix[i]
            right_cost = (total - prefix[i + 1]) - x * (n - i - 1)
            
            ans = min(ans, max(left_cost, right_cost))
        
        print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对数组进行排序，以便元素按升序排列。 前缀和数组允许快速计算左侧元素低于候选值的距离以及右侧元素高于候选值的距离。 对于每个候选目标值$x$，左侧成本衡量我们需要增加其下方元素的数量，右侧成本衡量我们需要减少其上方元素的数量。 答案是最差一侧的所有候选者中的最小值，因为操作只能有效地平衡两个方向直到限制一侧。 

最微妙的部分是正确分割前缀和后缀贡献。 当包含或排除枢轴元素本身时，通常会发生相差一错误，因此实现显式排除$i$计算成本时双方都可以。

| 目标 x | 剩余成本| 合适的成本| 结果 |

 | 10 | 10 24 | 0 | 24 |

 最佳答案是 8，通过瞄准多数值来实现。 这表明偏态分布有利于密集簇。

 ## 边缘情况

 单元素数组是最简单的边界。 该算法将左右成本计算为零，因为没有其他元素，因此输出正确为零。 

所有值都相等的数组是另一种退化情况。 排序后，每个候选者产生的左右成本均为零，因此最小值仍为零。 这证实了当序列已经统一时不需要任何操作。 

高度倾斜的数组，例如$[1, 1, 1, 100]$强调不平衡处理。 对于目标 1，左侧成本为零，右侧成本较大，而对于目标 100，则相反。 该算法正确地识别出最佳策略是通过向密集簇移动来吸收大异常值，产生的成本等于异常值侧的幅度差异。
