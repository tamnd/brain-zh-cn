 ## 解决方案
 ## 问题理解

 我们得到一个正整数数组。 每个查询都集中在该数组的连续段上，对于该段，我们可以选择任何子序列（不一定是连续的，但保持顺序）来形成“游戏数组”。 

所选择的子序列用于两人游戏。 玩家从 A 开始交替移动。在一次移动中，玩家选择任何严格大于 1 的元素，并将其替换为大于 1 的非平凡除数之一。最终元素可以减少，直到所有元素都变为 1，此时无法移动，即将移动的玩家失败。 

问题不是直接模拟游戏。 对于每个片段，我们必须计算在最佳玩法下有多少个非空子序列使玩家 A 获胜，并输出该计数模 998244353。 

约束很大，测试用例中的总数组长度和总查询数最多为 10^6。 这会立即排除任何构建或评估每个查询子序列的内容。 即使每次查询 O(n log n) 也会太慢； 预处理后每个查询需要 O(1) 或 O(log n) 。 

一个微妙的问题是子序列是由索引选择定义的，因此如果两个子序列选择不同的位置，即使值一致，它们也是不同的。 此外，不允许使用空子序列。 

当所有选定元素都等于 1 时，就会出现关键边缘情况。在这种情况下，游戏中不可能进行任何移动，因此第一个玩家会立即失败。 忽略这种情况的幼稚计数方法将错误地包括无效的获胜配置。 

## 方法

 对于每个查询，暴力解决方案将枚举该段的所有子序列并模拟游戏结果。 即使忽略游戏模拟，长度为 m 的段也有 2^m 个子序列，而且 m 可能很大，所以这是不可能的。 

第一个简化来自于对游戏本身的理解。 每次移动都对大于 1 的单个元素进行操作，并通过选择除数来减少它。 不管除数如何选择，唯一重要的事实是该元素严格递减并最终变为1。每个元素只能“使用”直到它变为1，之后就不再相关了。 

这意味着每个大于 1 的元素有效地贡献至少一个移动机会，而等于 1 的元素则不贡献任何内容。 更重要的是，由于玩家总是可以在一次移动中将所选元素直接减少到 1，因此因式分解的内部结构不会影响最佳游戏：每个有用元素的行为就像一个可以一次性消耗的单个令牌。 

因此，游戏简化为简单的回合制过程：玩家交替消耗大于 1 的元素。获胜者仅取决于所选子序列中存在多少个此类元素。 

如果我们定义子序列，则令 k 为其中大于 1 的元素数量。 游戏由 k 个动作组成，因此当且仅当 k 为奇数时 A 获胜。 

现在问题变成纯粹的组合问题。 对于给定的段，我们对子序列进行计数，使得所选“好”元素（值大于 1）的数量为奇数。 

设段中有c个大于1的元素和z个等于1的元素。 每个子序列都是通过独立选择或不选择每个元素而形成的。 z 元素仅贡献乘法自由度，而 c 元素决定奇偶校验。 

在 c 个元素中，当 c > 0 时，奇数大小的子集数量恰好为 2^(c-1)，当 c = 0 时，奇数子集数量为 0。 z 元素的每次选择都会使有效子序列的数量加倍，贡献因子 2^z。 

因此，对于 c > 0，答案变为 2^z * 2^(c-1) = 2^(段长度 - 1)。 如果c = 0，则所有元素均为1，并且不存在有效的获胜子序列。

因此，每个查询都简化为检查该段是否至少包含一个大于 1 的值。

 | 值的前缀计数 > 1 | O(n) 预处理后每个查询的 O(1) | O(n) | 已接受 |

 ## 算法演练

 ### 1.对每个元素进行分类

 我们扫描数组一次，如果每个元素大于 1，则将其分类为“活动”；如果等于 1，则将其分类为“非活动”。只有活动元素才会影响游戏奇偶性。 

这种分类就足够了，因为内除数运算永远不会影响元素在变为 1 之前保持可播放的事实。 

### 2.构建前缀信息

 我们计算一个前缀和数组，其中每个位置存储有多少个活动元素出现在该索引处。 这允许我们查询任何段在恒定时间内包含多少个活动元素。 

这样做的原因是子序列的选择与值无关，因此我们只需要计数，而不需要位置。 

### 3. 使用分段分类回答每个查询

 对于查询 [l, r]，我们使用前缀差异计算该范围内的活动元素 c 的数量。 如果 c 为零，则答案立即为零，因为游戏根本没有任何动作。 

否则，段长度为 len = r - l + 1。每个非空子序列在结构方面的权重相等，并且活动元素上的子集恰好有一半具有奇数大小。 非活动元素仅将计数乘以 2^（非活动元素数量），因此最终答案仅取决于 len。 

我们输出 2^(len - 1) 模 998244353。 

### 为什么它有效

 不变的是，只有所选子序列中大于 1 的元素数量的奇偶性才能决定获胜者。 除数运算永远不会改变这种与奇偶性相关的结构，因为每个活动元素在最佳发挥下恰好贡献一个不可避免的“消耗单元”。 由于子序列独立地处理元素，因此计数减少到奇偶校验约束的子集计数，并且不活动元素不会影响奇偶校验，而只会缩放配置数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modexp(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

t = int(input())
for _ in range(t):
    n, q = map(int, input().split())
    b = list(map(int, input().split()))

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + (1 if b[i] > 1 else 0)

    for _ in range(q):
        l, r = map(int, input().split())
        c = pref[r] - pref[l - 1]
        if c == 0:
            print(0)
        else:
            length = r - l + 1
            print(modexp(2, length - 1))
```前缀数组`pref`跟踪数组段中有多少元素严格大于 1。每个查询减去两个前缀值以获得 O(1) 的计数。 

求幂步骤使用二进制求幂有效地计算两个模 998244353 的幂。 由于指数最多为 10^6，因此对于所有查询来说这都足够快。 

一个微妙的点是我们从不显式计算 z，即 1 的数量，因为它在最终表达式中被抵消。 这就是使解决方案能够扩展到大输入的原因。 

## 工作示例

 ### 示例 1

 考虑数组：`[1, 1, 4, 16, 2, 8, 3, 7]`取完整段 [1, 8]。 

我们计算有多少个元素大于 1：

 | 我| 价值| >1？ | 前缀 |
 | --- | --- | --- | --- |
 | 1 | 1 | 0 | 0 |
 | 2 | 1 | 0 | 0 |
 | 3 | 4 | 1 | 1 |
 | 4 | 16 | 16 1 | 2 |
 | 5 | 2 | 1 | 3 |
 | 6 | 8 | 1 | 4 |
 | 7 | 3 | 1 | 5 |
 | 8 | 7 | 1 | 6 |

 所以 c = 6，长度 = 8。由于 c > 0，答案是 2^(8 - 1) = 2^7 = 128。 

这表明，尽管大量数字的分布看起来很复杂，但一旦存在至少一个活动元素，只有长度才重要。 

### 示例 2

 取一个仅由 1 组成的片段，例如`[1, 1, 1, 1]`。 

这里 c = 0。游戏中不能使用任何元素，因此 A 对于每个子序列都会立即失败。 答案是0。 

忽略“无移动”条件的简单子集计数方法会错误地输出 2^4 - 1 = 15，但这些子序列不满足游戏条件。

 ## 边缘情况

 当段仅包含 1 时，算法会正确返回零，因为前缀差 c 变为零，从而触发特殊情况。 这可以避免错误地计算没有实际游戏动作的子序列。 

当一个段恰好包含一个大于 1 的元素时，包含该元素的每个有效子序列都会纯粹基于包含奇偶性贡献一个获胜条件，并且公式 2^(len-1) 仍然成立。 前缀检查确保我们不会错误地将全 1 段视为有效。 

当段很大但活动元素稀疏时，算法仍然可以正确运行，因为它从不依赖于活动元素的位置，只依赖于它们的存在。
