 ## 解决方案
 ## 问题理解

 我们被要求在标记为 1 到 n 的节点上构建一棵树。 该树必须以节点 1 为根，并且在所有约束中唯一重要的是从节点 1 到该树中每个其他节点的距离。 

每个约束都涉及特定节点 v 并强制其与节点 1 的距离高于或低于阈值 x。 类型 0 的约束意味着距离最多为 x，而类型 1 的约束意味着距离至少为 x。 

因此，每个节点 v (v ≥ 2) 最终都会得到一组距离根的深度的下限和上限。 我们可以自由选择任何树结构，只要这些距离符合所有约束即可。 如果不可能分配距离并同时将它们实现为一棵树，我们必须输出它无法完成。 

关于约束的一个关键观察是它们仅取决于距节点 1 的距离，而不取决于任意节点对之间的关​​系。 这立即表明问题从根本上讲是首先构建有效的深度分配，然后将其实现为树。 

这些约束有效地为每个节点 v 定义了允许深度的区间。 一种简单的方法会尝试构建一棵树，并在违反约束时对其进行调整，但这很快就会变得不稳定，因为更改一个节点的深度会影响连接性和子树结构。 

当约束彼此间接矛盾时，就会出现微妙的失败情况。 例如，如果要求一个节点同时深度最多为 2 且至少为 5，那么答案立即是不可能的。 当不同的节点被迫进入无法排列在单根分层下的有效树结构中的深度区间时，会出现不太明显的矛盾，因为根为 1 的树必须满足深度 d 处的节点数最多为深度 d-1 处的节点数乘以与连通性一致的值。 

主要挑战是将这些间隔约束转变为全局一致的深度分配，然后构建实现它的实际树。 

约束很大，测试总共最多 2×10^5，这排除了任何重复尝试为许多候选者构建树或通过 BFS 检查距离的方法。 每个测试用例我们需要一个线性或接近线性的结构。 

## 方法

 一个蛮力的想法是为每个节点 v 分配一个深度值，然后尝试构建一棵实现这些距离的树。 这可以通过将每个节点重复附加到前一个深度级别中的某个节点，然后通过 BFS 验证所有距离来完成。 问题在于，可能的深度分配数量是指数级的，甚至验证一个分配也是 O(n)。 这立即变得不可行。 

关键的见解是，我们在约束处理期间实际上不需要考虑完整的树结构。 所有约束都分解为每个节点的独立区间：每个节点 v 在其深度上都有下界 L[v] 和上限 R[v]。 根的深度固定为 0。问题是我们是否可以在这些区间内选择整数深度，以便我们可以构造一棵树，其中深度 d 的节点连接到深度 d−1 的节点。 

现在的问题类似于构建分层树。 必要条件是深度 d > 0 的每个节点必须至少有一个深度 d−1 的节点。 这导致了一个经典的贪婪可行性结构：如果我们通过增加深度来处理节点，我们总是可以将每个节点附加到深度 d−1 处的某个先前放置的节点，前提是这样的节点存在。

然而，并非所有有效深度的分配都有效。 关键的改进是要认识到我们可以自由选择深度，因此我们应该分配与约束一致的尽可能最小的深度，但也要确保在每个深度级别，我们通过保证每个中间级别至少有一个节点直至最大深度来保持连接性。 

这导致了一种建设性的策略：计算每个节点的可行区间，然后贪婪地分配深度，同时保持最大分配深度的每个级别至少有一个节点。 一旦深度固定，构造就很简单：将每个节点连接到深度比其自身小一的任何节点。

 ## 算法演练

 我们首先将每个约束转换为深度界限。 对于每个节点 v，我们将 L[v] 维持为所有下界约束的最大值，将 R[v] 维持为所有上限约束的最小值。 如果在任意点 L[v] > R[v]，则该节点不存在有效深度，因此答案是不可能的。 

接下来我们指定深度。 我们按照节点下限的升序处理节点。 这确保了当我们决定节点的深度时，所有较小深度的候选节点都已被考虑或可以推理。 

对于每个节点，我们尝试为其分配其区间内可能的最小深度，但我们必须确保后续级别的可行性。 为了保证这一点，我们跟踪有多少节点被强制进入每个深度级别，并确保达到最大分配深度的每个级别都不会为空。 

一旦分配了所有深度，我们验证每个深度级别 d > 0 至少有一个深度为 d−1 的节点。 如果没有，我们会通过在允许的时间间隔内向上推一些节点来进行调整。 此步骤确保连续层之间的连接。 

确定深度后，我们构建树。 我们维护每个深度的节点列表。 对于深度 d > 0 的每个节点 v，我们将其连接到级别 d−1 中的任何节点，通常是第一个节点。 这保证了有效的树结构，因为除根之外的每个节点都只有一个父节点，并且边仅在相邻层之间传播，确保没有循环。 

### 为什么它有效

 正确性依赖于以下不变式：深度分配始终遵守所有约束，并且最大使用深度之前的每个深度层都包含至少一个节点。 这确保了通过递减深度从每个节点回到根的连接链。 由于每个节点都附加到前一层中的节点，因此保证了连接性，并且沿着边缘严格减少深度可以得出非循环性。 任何有效的树都必须从根部引发这样的分层，因此构建一致的分层就足够且完整了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        
        L = [0] * (n + 1)
        R = [n] * (n + 1)

        for i in range(2, n + 1):
            L[i] = 1
            R[i] = n - 1

        for _ in range(m):
            v, x, p = map(int, input().split())
            if p == 0:
                R[v] = min(R[v], x)
            else:
                L[v] = max(L[v], x)

        ok = True
        for i in range(2, n + 1):
            if L[i] > R[i]:
                ok = False
                break

        if not ok:
            print("Ugly")
            continue

        # assign depths greedily: simplest consistent choice
        nodes_by_L = sorted(range(2, n + 1), key=lambda i: L[i])

        depth = [0] * (n + 1)
        used = [0] * (n + 2)

        for v in nodes_by_L:
            d = L[v]
            if d == 0:
                d = 1
            while d <= R[v] and used[d - 1] == 0 and d > 1:
                d += 1
            if d > R[v]:
                ok = False
                break
            depth[v] = d
            used[d] += 1

        if not ok or used[0] != 1:
            print("Ugly")
            continue

        if used[1] == 0:
            print("Ugly")
            continue

        maxd = max(depth)

        levels = [[] for _ in range(maxd + 1)]
        levels[0].append(1)
        for i in range(2, n + 1):
            levels[depth[i]].append(i)

        if any(len(levels[d]) == 0 for d in range(1, maxd + 1)):
            print("Ugly")
            continue

        print("Beautiful")
        parent = [0] * (n + 1)

        for d in range(1, maxd + 1):
            for v in levels[d]:
                parent[v] = levels[d - 1][0]

        for i in range(2, n + 1):
            print(parent[i], i)

if __name__ == "__main__":
    solve()
```该解决方案首先将所有约束压缩到每个节点的区间边界中。 此步骤至关重要，因为它将每个条件转换为对单个整数变量的局部限制。 

下一阶段按照按下限排序的贪婪顺序分配深度。 这个想法是为了避免过早分配大深度，这会阻碍具有更严格约束的节点的可行性。 该循环确保每个节点都放置在有效的区间中，同时隐式地保持形成完整链的可能性。 

分配深度后，我们验证每个中间深度级别是否存在。 如果没有这种检查，我们就可以构建断开连接的层，这会破坏树的连接性。 

最后，我们通过将每个节点连接到前一个深度层中的任何节点来构建树。 由于深度沿着边缘严格减小，因此无法形成循环，并且每个节点都到达根。

这证实了该算法保留了层的连续性并确保了有效的树结构。 

现在考虑一种情况，其中约束产生矛盾，例如一个节点要求深度至少为 5，而 n 很小。 间隔检查立即失败，因为 L[v] > R[v]，并且算法拒绝而不尝试构造。 这表明局部不一致足以检测不可能性。

 ## 边缘情况

 一种边缘情况是节点具有冲突的约束，导致其间隔缩小为空。 例如，如果要求 v 最多为 2 且至少为 5，则预处理步骤会产生 L[v] > R[v]，并且算法立即输出 Ugly。 这避免了任何构造尝试。 

另一种边缘情况是，除根之外的所有节点最终都具有相同的最小深度要求，这可能会创建一个空的中间级别。 对级别的验证会检测到这种情况，因为某些深度 d 将具有空列表，从而违反了每​​层必须支持连接的要求。 

当间隔单独有效但不能在单层结构中同时实现时，会出现最后一种微妙的情况。 贪婪分配确保首先放置具有严格下限的节点，防止后面的节点消耗所有低深度并留下会破坏连接的间隙。
