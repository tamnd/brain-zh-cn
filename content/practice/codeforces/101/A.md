[CF 101A - 家庭作业](https://codeforces.com/problemset/problem/101/A)

 **标签：** 贪婪
 **求解时间：** 2m 5s
 **已验证：** 是的

 ## 解决方案
 ## 问题理解

 我们以小写字符串开头，最多可以删除`k`其中的字符。 其余字符必须保持其原始顺序，因为删除字符会创建子序列。 我们的目标不是最大化剩余长度，而是最小化仍然出现的不同字母的数量。 

如果某个字母在最终字符串中出现一次，该字母仍算作一个不同的字符。 要从结果中完全删除某个字母，我们必须删除该字母的所有出现位置。 

这将问题从“我们应该删除哪些位置？”转变为问题。 变成“我们应该完全删除哪些字符类型？”。 如果某个字符出现 7 次，则删除该字符需要 7 次删除。 如果出现一次，删除它只需 1 次删除。 

字符串长度可以达到`10^5`，这立即排除了任何指数或二次`n`。 我们不能尝试删除所有字符子集，也不能在嵌套循环中重复重建大字符串。 一个`O(n log n)`或者`O(n)`解决方案是正确的目标。 

有几种边缘情况很容易被错误处理。 

考虑：```
aaaaa
4
```我们最多可以删除 4 个字符，但删除所有 5 个字符就会超出限制。 答案仍然是 1 个不同的字符，而不是 0。粗心的实现可能会贪婪地删除尽可能多的字符，并意外删除整个字符串。 

现在考虑：```
abc
10
```在这里我们可以删除每个字符，因为`k`大于字符串长度。 正确答案是 0，结果字符串为空。 某些解决方案错误地强制保留至少一个字符。 

另一个微妙的情况是：```
aabbbcccc
5
```频率是`2, 3, 4`。 如果我们删除`'a'`和`'b'`，我们恰好进行了 5 次删除，并将答案减少为 1 个不同的字符。 首先删除最大组的天真的贪婪策略在这里会严重失败，因为删除`'c'`花费 4 并且留下两个不同的字母而不是一个。 

关键的观察结果是，删除字符类型的固定成本等于其频率。 为了最大限度地减少剩余不同字母的数量，我们应该首先删除最便宜的字符类型。 

## 方法

 暴力解决方案将尝试不同字符的每个子集，并检查删除这些字符的所有出现的成本是否最高`k`。 如果字母子集有效，我们会计算剩余的不同字母数并保留最小值。 

这是有效的，因为选择基本上是关于角色类型，而不是位置。 一旦我们决定哪些字母完全消失，所得到的子序列就通过保留所有其他字符来唯一确定。 

问题是子集的数量。 尽管字母表中只包含小写英文字母，但仍然有`2^26`子集，大约有 6700 万种可能性。 这对于 2 秒的限制来说太大了。 

重要的结构特性是每种字符类型都有独立的删除成本。 去除`'a'`永远不会影响拆除成本`'b'`。 我们正在有效地选择尽可能多且频率符合预算的角色类型`k`。 

这变成了一个贪心问题。 假设两种字符类型的频率为 2 和 5。如果我们只能负担其中一种，则删除频率为 2 的字符总是至少与删除频率为 5 的字符一样好，因为两者都将不同计数精确地减少了 1，但其中一种更便宜。 

该观察完全决定了策略：

 1. 计算频率。 
2. 按出现频率对字符类型进行排序。 
3. 当我们还剩下足够的删除时，首先删除最小的组。 
4. 仅使用剩余的字符类型重建答案。 

字母表大小只有 26 个，因此排序频率本质上是恒定时间。 主要工作是扫描字符串。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^26 · 26 + n) | O(2^26 · 26 + n) | O(26) | 太慢了 |
 | 最佳 | O(n + 26 log 26) | O(n + 26 log 26) | O(26) | 已接受 |

 ## 算法演练

 1.读取字符串`s`和整数`k`。 
2. 计算每个字符在字符串中出现的次数。 

我们需要这些频率，因为删除一个字符完全会花费它的频率。 
3. 存储全部`(frequency, character)`配对并按频率升序对它们进行排序。 

应首先考虑最便宜的字符类型，因为每个完全删除的字符都会使不同计数恰好减少 1。 
4. 遍历已排序的列表。 

如果当前频率小于或等于剩余删除预算`k`，删除整个字符类型：

 - 从中减去它的频率`k`- 将角色标记为已删除

 否则停止删除字符。 

一旦我们无法承受当前的最小频率，我们也无法承受任何以后的频率，因为列表是排序的。 
5. 通过仅保留未删除的字符来构建最终字符串。 

这会自动保留原始顺序，因此结果是有效的子序列。 
6. 计算剩余的不同字符数并打印：

 - 剩余不同字符的数量
 - 结果子序列

 ### 为什么它有效

 每个删除的字符类型都会带来完全相同的好处，将不同计数减少 1。选择之间的唯一区别是它们的成本，即它们的频率。 

假设最佳解决方案以频率 5 删除一个字符，但保留另一个频率为 2 的字符。交换这些选择不会增加使用的总删除量，但仍会删除一个不同的字符。 重复此交换参数会将任何最佳解决方案转换为始终首先删除最小频率的解决方案。 

由于该属性，贪婪策略是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
k = int(input())

freq = [0] * 26

for ch in s:
    freq[ord(ch) - ord('a')] += 1

pairs = []

for i in range(26):
    if freq[i] > 0:
        pairs.append((freq[i], chr(i + ord('a'))))

pairs.sort()

removed = set()

for cnt, ch in pairs:
    if cnt <= k:
        k -= cnt
        removed.add(ch)
    else:
        break

result = []

for ch in s:
    if ch not in removed:
        result.append(ch)

answer = ''.join(result)

print(len(set(answer)))
print(answer)
```第一部分使用长度为 26 的固定大小数组来计算频率。这比使用字典更快、更简单，因为字母表是预先知道的。 

这`(frequency, character)`对进行排序，以便最便宜的字符类型首先出现。 在预算允许的情况下，贪婪循环会删除整个字符组。 

停止条件很重要。 一旦频率太大，后面的每个频率也会太大，因为数组是排序的。 继续循环不会有任何结果。 

最后的重建步骤扫描原始字符串并跳过删除的字符。 这会自动保留相对顺序，因此结果仍然是一个子序列。 

一个微妙的问题是空字符串的情况。 如果删除所有字符类型，`answer`变成`""`， 和`len(set(answer))`正确评估为`0`。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(n + 26 log 26) | O(n + 26 log 26) | 计数重建扫描一次字符串，最多排序26个频率 |
 | 空间| O(26) | 频率存储和删除字符跟踪 |

 由于字母表大小是固定的，因此排序成本实际上是恒定的。 该解决方案主要是对字符串进行线性扫描，这很容易满足以下限制：`n = 10^5`。 

## 测试用例```python
# helper: run solution on input string, return output string
import sys, io

def solve():
    input = sys.stdin.readline

    s = input().strip()
    k = int(input())

    freq = [0] * 26

    for ch in s:
        freq[ord(ch) - ord('a')] += 1

    pairs = []

    for i in range(26):
        if freq[i] > 0:
            pairs.append((freq[i], chr(i + ord('a'))))

    pairs.sort()

    removed = set()

    for cnt, ch in pairs:
        if cnt <= k:
            k -= cnt
            removed.add(ch)
        else:
            break

    ans = ''.join(ch for ch in s if ch not in removed)

    print(len(set(ans)))
    print(ans)

def run(inp: str) -> str:
    backup_stdin = sys.stdin
    backup_stdout = sys.stdout

    sys.stdin = io.StringIO(inp)
    sys.stdout = io.StringIO()

    solve()

    out = sys.stdout.getvalue()

    sys.stdin = backup_stdin
    sys.stdout = backup_stdout

    return out

# provided sample
assert run("aaaaa\n4\n") == "1\naaaaa\n", "sample 1"

# delete one entire character group
assert run("aaaabbb\n4\n") == "1\naaaa\n", "remove b"

# delete everything
assert run("abc\n10\n") == "0\n\n", "empty result"

# minimum size input
assert run("a\n0\n") == "1\na\n", "single character"

# exact budget match
assert run("aabbbcccc\n5\n") == "1\ncccc\n", "remove a and b"

# no deletions allowed
assert run("abac\n0\n") == "3\nabac\n", "unchanged string"

 ## 边缘情况

 考虑不可能删除所有字符的情况：```
aaaaa
4
```频率列表仅包含`(5, 'a')`。 自从`5 > 4`，贪心循环不会删除任何内容。 重建步骤保留了每个角色，产生`"aaaaa"`有 1 个不同的字母。 这证实了该算法永远不会超出删除预算。 

现在考虑相反的极端：```
abc
10
```排序后的频率为`(1, 'a')`,`(1, 'b')`,`(1, 'c')`。 

该算法删除所有三组：

 | 人物 | 成本| 剩余 k |
 | --- | --- | --- |
 | 一个 | 1 | 9 |
 | 乙| 1 | 8 |
 | c | 1 | 7 |

 重建后的字符串为空。 重复计数变为 0，这是有效的，因为删除所有字符只需花费 3。 

另一个微妙的场景是多种字符类型共享相同的频率：```
aabbcc
2
```所有频率均为 2。算法可能会删除其中任何一个，具体取决于排序顺序。 如果它删除`'a'`，结果变为`"bbcc"`有 2 个不同的字母。 

这仍然是最佳的，因为每个删除选择的成本都是相同的。 该问题接受任何有效答案，因此关系并不重要。
