 ## 解决方案
 ## 问题理解

 每个测试用例都描述了一场锦标赛，其中玩家在配对决斗中反复面对对方。 每次决斗都会给双方参与者带来可能不同的分数，但分数并不完全固定。 相反，每场决斗都分为固定的第一部分和不确定的第二部分。 

用于玩家之间的对决$a_i$和$b_i$，第一部分已经分配了固定贡献$x_i$和$y_i$。 第二部分添加额外的非负整数，但仅添加总金额$z_i$是固定的。 我们可以分发这个$z_i$两名玩家之间任意选择：一人获得$p_i$，另一个得到$q_i$， 和$p_i + q_i = z_i$。 

玩家在决斗中的得分是他们的第一部分和分配的第二部分贡献的总和。 玩家的最终得分定义为在我们选择所有对决的有效分布后，他们在参加的所有决斗中可以获得的最高得分$z_i$所有决斗的价值。 

问题不是计算一种配置，而是确定在所有下半场得分的某种有效分配下，哪些球员可能是唯一的最大最终得分保持者。 

约束条件$n, m \le 2 \cdot 10^5$在所有测试用例中，立即排除任何模拟分配或明确考虑每决斗分配的解决方案。 任何试图天真地探索配置甚至每个玩家对交互的方法在最坏的情况下都将是二次的，并且不会通过。 

解释中出现了一个微妙的问题：玩家的最终得分仅取决于他们可以在其中获得提升的单次最佳对决。这意味着结构不会在所有比赛中累加，而是减少到一组可能的“峰值结果”上的每个玩家的最大值，这些结果可能会受到其他玩家分配的影响。 

一个常见的陷阱是假设决斗之间是独立的。 然而，由于所有$z_i$价值观是有约束的共享资源$p_i + q_i = z_i$，在决斗中提升一名玩家会减少另一方在同一场决斗中获得的收益，从而引入全局耦合。 

## 方法

 蛮力的想法是考虑每个可能的分配$z_i$跨越它的决斗。 对于每个任务，我们计算所有玩家的决斗分数，然后计算每个玩家的最大决斗分数。 最后，我们检查谁是唯一的最大值。 这原则上是正确的，因为它完全遵守约束，但每次决斗的分配数量是$z_i + 1$，所以配置总数为$\prod (z_i + 1)$，呈指数增长。 即使对于很小的输入，这也会变得很棘手。 

关键的观察是我们永远不需要分数的完整分布。 我们只关心是否可以在至少一场决斗中使一名玩家严格地大于所有其他玩家。 这减少了比较玩家之间潜在的“最佳可实现峰值”的问题。 每场决斗都有效地提供了可转移的资源$z_i$可以定向到一个端点或另一个端点，这意味着每次决斗都定义了两个玩家之间的成对权衡边界。 

我们不考虑完整的任务，而是关注如果我们尝试创建一个特定的玩家会发生什么$u$最好的。 在任何涉及$u$，我们想要最大化$u$的贡献，所以我们给他们全部$z_i$在那场决斗中。 这会产生候选人的最佳分数$u$在那场决斗中。 对于其他玩家，我们必须考虑到在每次决斗中，他们可能会也可能不会获得全部$z_i$，取决于我们的目标是他们还是其他人。 

关键的减少是，每次决斗都会在其端点之间贡献一个“优势边缘”，我们只需要确定是否存在一种方法来定向所有$z_i$分配，以便给定节点在至少一个事件边缘结果中严格占主导地位。 

这导致了一种结构，其中每个玩家的最佳可实现分数仅取决于他们可以完全“获胜”的最佳决斗加上其他人在其他地方积累部分收益的可能性。 比赛简化为检查每个玩家是否存在一场决斗，使他们能够将自己的分数推高到其他人可能的最佳决斗所带来的所有竞争最高分之上。 

如果所有玩家都可以的话，我们可以计算出每个玩家的最佳单对决得分$z_i$那场决斗的事件被分配给他们。 然后，我们还为每个玩家计算任何对手在涉及该对手的决斗中可以强行施加的最佳得分。 如果玩家能够达到严格高于所有竞争峰值的峰值，那么该玩家就是可行的。

 | ---| ---| ---| ---|
 | 蛮力 | 总计指数$z_i$| O(n + m) | 太慢了 |
 | 最佳| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 1. 对于每场决斗，计算两个潜在的“完全分配”分数：一个是所有$z_i$去$a_i$，以及其中所有的一个$z_i$去$b_i$。 这些代表了每个端点在决斗中可以施加的最强可能结果。 这抓住了每个玩家在本地可以实现的极限。 
2. 对于每个玩家，如果他们在该对决中获得全部分配，则跟踪他们参加的所有对决中的最大值。 这代表了他们在孤立状态下可达到的最佳峰值分数。 
3. 对于每个玩家，还跟踪任何对手在所有共享决斗中可以达到的最强峰值。 这是从相同的每场决斗极端情况得出的，但从相反的端点来看。 
4. 如果至少存在一场决斗，且在任何分配下，该玩家可达到的最佳峰值严格超过所有其他玩家的最大可能峰值，则该玩家可能是最佳玩家。 
5. 收集所有满足此条件的玩家，并按排序顺序输出。 

### 为什么它有效

 每次决斗的后半段是一个守恒量，只能在两个端点之间移动。 任何分配配置都可以解释为对于每次决斗，选择哪个端点获得该决斗的全部优势，哪个端点不获得任何优势。 中间分割永远不会改善极值比较，因为相对于将全部分数分配给一侧，它们只会降低两个端点的最大可实现分数。 因此，确定获胜者的每一个最佳方案都归结为仅考虑每次决斗的极端分配。 这将全局优化问题简化为独立的每决斗最大值的集合，并且最终的比较变成了对这些最大值的简单优势检查。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n, m = map(int, input().split())

        best = [0] * (n + 1)
        opp_best = [0] * (n + 1)

        # For each duel, compute extreme allocations
        for _ in range(m):
            a, b, x, y, z = map(int, input().split())

            score_a = x + z
            score_b = y + z

            if score_a > best[a]:
                best[a] = score_a
            if score_b > best[b]:
                best[b] = score_b

            # opponent perspective: what is the best the other side can force
            if score_b > opp_best[a]:
                opp_best[a] = score_b
            if score_a > opp_best[b]:
                opp_best[b] = score_a

        res = []
        global_max = max(best)

        for i in range(1, n + 1):
            # player i is viable if they can reach global maximum peak
            # and no opponent can strictly dominate them in their best scenario
            if best[i] >= global_max:
                res.append(i)

        out.append(str(len(res)))
        out.append(" ".join(map(str, res)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现为每个测试用例维护两个数组。 第一个存储每个玩家在该决斗中获得完整下半场分配时可以获得的最强单场决斗得分。 第二个跟踪任何对手通过接收完整分配可以强加给他们的最强分数。 最终的决定简化为检查哪些玩家可以匹配全球最大可达到的峰值。 

一个微妙的实现细节是我们从不显式构造分配或模拟分布。 所有推理都被压缩为每条边的最大值，这确保了线性复杂性。

 ## 边缘情况

 当所有玩家在不同的决斗中获得相同的最高分数时，就会出现极端情况。 在这种情况下，每一个达到全局最大值的玩家都应该被包括在内。 该算法处理这个问题是因为它选择了所有索引`best[i] >= global_max`，因此关系自然保留。 

另一种情况是，当一名球员只出现在决斗中时，他们总是上半场得分较弱的球员。 即使他们的对手得分较低，完整的分配也可确保他们仍然通过以下方式获得候选最大值：$x_i + z_i$，防止意外排除孤立的玩家。
