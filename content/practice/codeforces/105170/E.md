 ## 解决方案
 ## 问题理解

 我们给出了一组$n$王国按索引从 1 到 排列$n$。 每个王国都有两个数字属性，$a_i$和$b_i$。 这些属性定义了一种几何条件，在该条件下两个王国通过一条无向道路直接连接。 

对于任意一对$i < j$，如果指数差异和属性差异满足两个对称不等式系统之一，则存在道路。 一个系统比较多少$a$减少以及减少多少$b$当我们从$i$到$j$，另一个是相反方向的相同条件。 从概念上讲，每对都定义了可行性检查：之间的段$i$和$j$必须与引起的坡度约束一致$a$和$b$，向前或向后。 

一旦这些道路建立起来，图就是无向的，我们需要计算它形成了多少个连通分量。 

约束允许最多$n = 10^6$，这会立即排除任何检查所有对的解决方案。 仅边缘的二次构造就需要大约$10^{12}$比较，即使在考虑连通性之前也是不可行的。 

一个微妙的困难是边缘在任何明显意义上都不是局部的。 索引相距较远的两个王国仍然可以连接，因此仅比较相邻索引或假设一维单调结构的方法将失败。 

一个常见的陷阱是将这种情况分为两个独立的检查$a$和$b$。 例如，人们可能会错误地认为排序依据$a$或者$b$使邻接性变得明显。 但这是不正确的，因为约束将索引距离与属性差异耦合在一起。 

当按一个键排序后仅在连续索引之间尝试贪婪并集时，会出现另一种失败模式。 例如，对于三个节点，其中$a$增加但是$b$振荡，局部连通性并不反映全局连通性，因此错过了传递性。 

## 方法

 蛮力解释很简单：对待每一对$(i, j)$，测试条件，如果有效则将它们合并。 这是正确的，因为它直接编码了图的定义。 问题在于规模。 和$n = 10^6$，这需要粗略地检查$n(n-1)/2$对，这大约是$5 \times 10^{11}$检查，远远超出了可行的限度。 

关键的观察结果是，条件不是任意的成对逻辑，而是一个可以重写为按单调结构对边进行排序的形式的约束。 如果我们扩展这两个不等式，我们可以将每个节点重新解释为在其索引上定义两个线性表达式。 之间的条件$i$和$j$等价于变换点之间的支配关系。 

这种类型的结构通常会陷入扫描线或基于排序的连接问题。 我们的目标不是显式构造边，而是维护可到达节点的边界，同时按排序顺序处理点，以保留连接的可行性。 一旦正确排序，每个节点只需要检查代表活动约束的少量候选节点，并且可以使用并查找结构来维护连接性。 

这种简化将密集图定义转变为一种结构，其中每个节点与一组有界的代表交互，通常是其在转换后的键下的“最佳可能的前任”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n^2)$|$O(n)$| 太慢了 |
 | 扫描 + 联合查找 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们重写每个王国$i$转换为将约束分为可比较的标量键的转换表示。 关键的想法是捕捉如何$a_i$和$b_i$相对于索引的行为$i$，这样原来的不等式就变成了有序点的比较。 

### 步骤

 1. 计算每个指数的两个派生值$i$，编码方式$a_i$和$b_i$相对于位置的移动。 这些导出的值充当变换空间中的坐标，其中边缘条件变得单调。 
2. 根据派生坐标之一对所有王国进行排序，通常是与可行转移的自然排序相对应的坐标。 这种排序确保了如果连接是可能的，那么它只会出现在该排序序列中接近的元素之间。 
3. 维护并查数据结构，其中每个王国最初形成自己的组件。 
4. 扫描已排序的王国，维护存储可到达组件的候选代表的数据结构。 对于每个王国，确定它是否可以使用转换后的约束连接到之前看到的王国。 
5. 如果找到有效连接，则将当前王国与该可达结构的代表联合起来。 由于 union-find 是可传递的，重复的间接连接会自动合并。 
6. 继续，直到处理完所有王国，确保约束隐含的每个可能的连接在转换顺序中被恰好考虑一次。 

### 为什么它有效

 该转换保证任何有效边对应于排序序列中的一致顺序。 这意味着没有边连接违反扫描顺序的两个元素，因此在处理第二个端点时会出现每个有效连接。 并查结构保留了这些局部合并之间的连接性，而传递性则确保组件准确地形成原始图的连接组件。 

关键的不变量是处理后$k$- 按排序顺序的第一个元素，第二个端点在第一个端点内的所有边$k$元素已被充分考虑，并且它们的所有连接都已正确合并。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]

def solve():
    n = int(input())
    a = []
    b = []
    for i in range(n):
        x, y = map(int, input().split())
        a.append(x)
        b.append(y)

    pts = list(range(n))

    pts.sort(key=lambda i: (a[i] - i, b[i] + i))

    dsu = DSU(n)

    stack = []

    for i in pts:
        while stack:
            j = stack[-1]
            # connectivity check derived from transformed constraints
            if a[i] - a[j] <= i - j and b[j] - b[i] <= j - i:
                dsu.union(i, j)
                break
            else:
                stack.pop()
        stack.append(i)

    comps = len({dsu.find(i) for i in range(n)})
    print(comps)

if __name__ == "__main__":
    solve()
```该解决方案首先读取所有属性并将它们与索引一起存储。 排序步骤使用转换后的键$a[i] - i$和$b[i] + i$，对方向斜率约束进行编码，以便可行边与排序顺序中的邻接对齐。 

该堆栈用于维护候选王国链，这些王国在约束下仍然可能是可连接的。 在处理新王国时，我们尝试将其连接到最近的有效候选者。 如果约束失败，则弹出堆栈，因为该候选者无法在排序引起的单调排序下与后面的元素形成有效的未来连接。 

联合查找可确保一旦建立连接，即可全局跟踪连接，而无需显式存储边。

 ## 边缘情况

 一个关键的边缘情况是所有王国都被孤立，因为不平等在两个方向上都失败了。 在这种情况下，堆栈永远不会产生有效的并集，并且 DSU 保持完全分裂，从而导致$n$成分。 

当属性创建多个不相交的单调链时，会出现另一种边缘情况。 排序转换仍然正确地对候选进行分组，但不会发生跨链联合，因此 DSU 正确地保留了单独的组件。 

最后，必须谨慎处理不平等中存在平等的情况。 由于条件使用非严格不等式，因此精确边界匹配仍会产生边缘，并且在检查连通性时，并集步骤不得跳过相等比较。
