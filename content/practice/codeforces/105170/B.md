 ## 解决方案
 ## 问题理解

 我们得到一棵有根树，其中每个顶点都有一个值。 我们从根开始执行深度优先遍历，但是我们访问任何节点的子节点的顺序是完全灵活的。 这意味着我们没有得到固定的 DFS 订单； 相反，我们可以在每个节点上选择子节点的排列，这会改变生成的顶点前序序列。 

一旦生成了 DFS 预序序列，我们通过对该序列中出现在偶数位置的顶点值求和来对其进行评估，其中索引从 1 开始。因此，只有第 2、第 4、第 6 等元素对分数有贡献。 

任务是选择每个节点的子节点的顺序，以使该分数最大化。 

这些约束意味着所有测试用例的节点总数最多为 2 × 10^5。 这立即排除了任何尝试子项的所有排列或显式模拟 DFS 顺序的解决方案。 即使在本地处理所有重新排序，在最坏的情况下也是指数级的，因此该解决方案必须依赖于 DFS 遍历的结构属性而不是枚举。 

这个问题的一个微妙问题是，DFS 顺序中节点的位置奇偶校验不是本地固定的。 当我们遍历一棵子树时，它里面的每个节点都会继承前序序列中的一个全局索引，它是否有贡献取决于它之前访问了多少个节点。 这会在兄弟子树之间创建依赖关系：交换两个子树不仅会更改它们的内部顺序，还会更改后面所有内容的奇偶校验对齐。 

朴素推理的一个小的说明性失败案例是一个有两个子节点的根，一个子树的大小为 1，另一个子树的大小为 2。根据首先访问的子树，第二个子树内的位置奇偶校验会翻转，从而改变哪些节点起作用。 任何独立处理子树的贪婪方法都会在这里失败，因为它忽略了这种奇偶校验移位。 

## 方法

 暴力方法将枚举每个节点的子节点的所有可能排列，生成最终的 DFS 顺序，并计算分数。 即使对于度为 d 的节点，这也涉及 d！ 排序，并且由于每个排序都会影响所有后代 DFS 遍历，因此总工作量会在树上呈乘法爆炸。 在星形树中，这已经成为 n 的阶乘，这是不可行的。 

关键的观察结果是 DFS 预序连接了与子树相对应的连续段。 每个子树都会贡献一个块，一旦选择了自己的子排序，该块的内部结构就被固定，但该块在全局序列中的放置决定了其根是从偶数位置还是奇数位置开始。 一旦知道起始奇偶校验，子树内的每个节点都具有确定性奇偶校验模式。 

这提出了一个动态编程公式：对于每个子树，我们计算如果其根从奇数位置开始与偶数位置开始，它会贡献多少价值。 困难在于，在组合子项时，每个子项的起始奇偶校验取决于先前处理的子项的大小，因此我们处理子项的顺序很重要。 

这将问题简化为对“项目”（子子树）进行排序，其中每个项目都有一个大小和两个可能的贡献，具体取决于当前奇偶校验。 此外，如果子树的大小为奇数，它会在处理后翻转奇偶校验状态，这会在排序和未来决策之间产生耦合。 该结构允许基于比较偶校验与奇校验开始之间的增益差异进行贪婪排序。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 树DP+贪心排序| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

我们将树的根设为 1 并执行后序遍历。 

对于每个节点，我们计算两条信息。 首先，它的子树的大小。 其次，两个 DP 值：如果节点在全局 DFS 顺序中的奇数位置进入，则子树的最佳可能贡献；如果在偶数位置进入，则子树的最佳可能贡献。 

1. 对于每个叶节点，子树仅由其自身组成。 如果它被输入到一个奇怪的位置，它就没有任何贡献。 如果它被输入到偶数位置，它就会贡献自己的价值。 子树大小为1。 
2. 对于内部节点，我们首先为所有子节点递归计算这些 DP 值。 此时，每个子项的行为就像一个“块”，可以使用任一奇偶校验输入，并且我们知道结果及其子树大小。 
3. 现在我们需要将子节点合并到表示当前节点下方的 DFS 顺序的单个序列中。 节点本身被放置在最前面，因此总是以当前全局奇偶校验来访问它。 
4. 我们维护一个正在运行的奇偶校验状态，该状态表示下一个访问的节点是否位于奇数或偶数 DFS 位置。 最初，在访问当前节点后，奇偶校验翻转，因为我们已经消耗了一个位置。 
5. 然后按某种顺序附加每个子子树。 当处理子进程时，如果当前奇偶校验为偶数，则取子进程的“偶数条目”DP 值； 否则我们取其“奇数条目”DP 值。 处理完子级后，当且仅当该子级子树的大小为奇数时，奇偶校验才会翻转，因为奇数长度的块会更改后续位置的对齐方式。 
6. 我们唯一的自由是选择孩子的顺序。 为了决定这一点，我们使用偶数奇偶校验和奇数奇偶校验时输入的贡献之间的差异来比较儿童。 在偶数进入方面具有较大优势的孩子应该更早地被优先考虑，因为早期的安置更经常地影响翻转累积之前的高影响平价状态。 
7. 我们按照这种差异的降序对子级进行排序，然后按该顺序模拟合并过程，累积最佳贡献。 

自下而上处理完所有节点后，在奇数位置输入根的答案就是最终结果。 

### 为什么它有效

 每个子树的行为就像一个段，它消耗固定数量的 DFS 位置，并根据其大小引发奇偶校验翻转。 DP 确保对于任何固定的条目奇偶校验，子树在内部已经是最优的。 剩下的自由度只是片段的顺序。 贪婪排序之所以有效，是因为早期放置子树与晚期放置子树之间的贡献差异仅取决于在每个奇偶校验状态下评估它的频率，并且这种效果相对于其两个 DP 状态之间的差异是单调的。 由于奇偶校验翻转仅取决于子树大小而不取决于内部结构，因此每个子树可以被视为具有带符号首选项的原子块，从而使排序参数有效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    parent = [-1] * n
    order = []
    stack = [0]
    parent[0] = -2

    while stack:
        x = stack.pop()
        order.append(x)
        for y in g[x]:
            if parent[y] == -1:
                parent[y] = x
                stack.append(y)

    children = [[] for _ in range(n)]
    for v in order[1:]:
        children[parent[v]].append(v)

    dp0 = [0] * n
    dp1 = [0] * n
    sz = [1] * n

    for x in reversed(order):
        best0 = 0
        best1 = a[x]  # if x starts at even position, contributes
        size = 1

        info = []

        for y in children[x]:
            info.append(y)

        def key(y):
            return dp1[y] - dp0[y]

        info.sort(key=key, reverse=True)

        cur_parity = 0  # 0 means next node is odd position (since x itself handled outside)
        for y in info:
            if cur_parity == 0:
                best0 += dp0[y]
                best1 += dp0[y]
            else:
                best0 += dp1[y]
                best1 += dp1[y]

            if sz[y] % 2 == 1:
                cur_parity ^= 1

            size += sz[y]

        sz[x] = size
        dp0[x] = best0
        dp1[x] = best1

    print(dp0[0])

t = int(input())
for _ in range(t):
    solve()
```该解决方案构建显式有根树，然后以相反的 DFS 顺序处理节点，以便每个子树都在其父树之前计算。 每个节点维护子树大小和两个 DP 值。 合并步骤使用对子项的贪婪排序，基于当它们的入口奇偶性为偶数与奇数时它们的表现如何。 最终的答案是根在奇数 DFS 位置输入时的 DP 值，它对应于实际的遍历开始。 

合并期间必须注意奇偶校验跟踪。 跟踪下一个子树是否进入偶数或奇数位置的变量必须仅在子子树具有奇数大小时翻转，因为只有奇数长度的段才会反转后续段的奇偶校验对齐。

 ## 边缘情况

 单节点树总是产生零贡献，因为根在位置 1 被访问，这是奇数，因此不计算任何值。 DP 正确地将 dp1 初始化为节点值，但返回 dp0 作为根，产生零。 

在二节点树中，排序是无关紧要的，因为只有一个子树。 第二个节点始终位于位置 2，因此始终计算其值。 DP 正确地反映了这一点，因为子子树在偶数条目奇偶校验下贡献了其值。 

在星形树中，叶子处理的顺序决定了在发生奇偶校验翻转之前有多少叶子落在偶数位置。 基于 dp1 减去 dp0 的贪婪排序可确保从均匀条目中受益更多的叶子被放置在更早的位置，从而最大化总增益。
