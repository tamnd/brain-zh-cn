 ## 解决方案
 ## 问题理解

 我们给定一个用二进制写成的非常大的整数，称之为`n`。 而不是以通常的方式将其解释为数字并从 1 循环到`n`，我们被要求直接使用它的二进制表示。 

对于每个整数`i`从 1 到`n`，我们看一下 1 位的数量`i`，这是 的 popcount`i`。 调用这个值`g(i)`。 然后我们通过斐波那契数列映射该计数，其中`f(1) = 1`,`f(2) = 1`，对于较大的指数，每一项都是前两项之和。 最终答案是`f(g(i))`全面的`i`在`[1, n]`，取模`10^9 + 7`。 

所以问题实际上并不在于迭代数字。 它是关于在由二进制限制定义的大范围内聚合 popcount 函数。 

主要的困难在于`n`最多可以有 10^7 位。 这立即排除了任何迭代值最多的方法`n`，甚至构造所有中间整数。 即使存储所有数字也是不可能的。 唯一可用的结构是二进制字符串本身。 

关键的观察结果是`g(i)`仅取决于哪些位位置被选为 1`i`。 所以问题就简化为计算有多少个整数`i ≤ n`正好有`k`设置位，对于每个`k`，然后求和`f(k)`由这些计数加权。 

这将问题转化为二进制前缀上的约束组合问题。 

一些边缘情况很重要：

 当`n = 1`，唯一的值是`i = 1`，所以答案是`f(1) = 1`。 任何假设至少两位或从 0 开始的解决方案都将被关闭。 

什么时候`n = 2`（二进制`10`），有效数字是`1`和`2`，带有弹出次数`1`和`1`，所以答案是`f(1) + f(1) = 2`。 

一种常见的失败模式是尝试将范围视为所有固定长度的二进制数等于`|n|`，其中错误地包含大于的数字`n`。 

## 方法

 蛮力方法在概念上很简单。 我们会迭代`i`从 1 到`n`，计算其弹出计数，评估该计数的斐波那契值，并累加结果。 这是正确的，但完全不可行。 即使我们可以在 O(1) 内计算 popcount，迭代次数也约为`n`， 和`n`本身是一个最多10^7位的二进制整数，因此其数量级是天文数字。 

发生故障是因为我们迭代的是值而不是位模式。 我们实际需要的结构不是数字顺序，而是位位置子集上的组合结构。 

关键的洞察力是转换观点：每个数字`i ≤ n`对应于选择位位置的子集，使得生成的二进制数不超过`n`。 对于固定的子集大小`k`，贡献是`f(k)`乘以大小的有效子集的数量`k`。 

这将问题转化为二进制表示的数字动态规划问题`n`，我们计算有多少前缀在给定数量的选定数字的情况下产生有效数字。 

我们维护，对于每个前缀`n`，我们可以用给定的 popcount 来形成数字，分为两种情况：我们是否已经严格小于`n`，或者仍然与前缀完全匹配。 这是二进制字符串上的标准“紧密”DP，还有一个附加维度跟踪迄今为止选择的数字的数量。 

一旦我们准确地知道有效整数的数量`k`，最终的答案只是斐波那契值的加权和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解 i | O(n) | O(1) | O(1) | 太慢了 |
 | 二进制前缀上的数字 DP | O(L²) 最坏情况结构化 DP | O(L²) 或优化 O(L) | 已接受 |

 这里`L`是二进制字符串的长度。 

## 算法演练

 我们从左到右处理二进制字符串，计算我们可以形成多少个小于或等于前缀的数字`n`。 

1. 预先计算斐波那契值`L`，因为任意数字的 popcount 最多为`L`位不能超过`L`。 这确保我们可以立即将任何计数映射到其斐波那契值。 
2.建立一个DP表，其中`dp[tight][k]`表示我们可以使用经过处理的前缀形成数字的多少种方式，以便准确地`k`到目前为止已经选择了一些，并且`tight`指示我们是否仍然完全匹配前缀`n`。 
3. 初始化`dp[1][0] = 1`，这意味着在处理任何位之前，我们只有一个与前缀匹配的空结构。 
4. 对于二进制串中的每一位`n`，更新DP：

 我们考虑在当前位置放置 0 或 1，但如果处于紧密状态，则必须尊重前缀施加的约束。 如果当前位`n`为 0，我们不能将 1 置于紧态。 如果是 1，两种选择都是可能的，但选择 1 可能仍然使我们保持紧张或打破紧张，具体取决于确切的匹配。 
5. 每个转换还会更新所选转换的数量，增加`k`当我们放置 1 时。 
6. 处理完所有位后，我们得到了按 popcount 分组的有效数字的计数，这些数字在紧状态和非紧状态下累积。 
7. 通过求和计算出最终答案`dp[0][k] + dp[1][k]`乘以`f(k)`全面的`k`。 

### 为什么它有效

 每个整数`i ≤ n`恰好对应于该数字 DP 中的一条有效路径：我们逐位决定是否`i`比赛`n`到目前为止或已经变得更小，我们跟踪我们使用了多少个。 严格的约束保证我们永远不会超过`n`，并且状态分割确保每个有效数字只被计算一次。 由于 popcount 完全由所选的数量捕获，因此分组`k`就足够了，并且最终的总和正确地聚合了范围内所有整数的贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

s = input().strip()
L = len(s)

# Fibonacci up to L
fib = [0] * (L + 2)
fib[1] = 1
for i in range(2, L + 2):
    fib[i] = (fib[i - 1] + fib[i - 2]) % MOD

# dp[tight][k]
# tight = 0 or 1
dp0 = [0] * (L + 1)
dp1 = [0] * (L + 1)

dp1[0] = 1  # start tight, 0 ones used

for ch in s:
    bit = int(ch)
    ndp0 = [0] * (L + 1)
    ndp1 = [0] * (L + 1)

    for k in range(L + 1):
        val1 = dp1[k]
        if val1:
            # place 0
            if bit == 1:
                ndp0[k] = (ndp0[k] + val1) % MOD
            else:
                ndp1[k] = (ndp1[k] + val1) % MOD

            # place 1
            nk = k + 1
            if nk <= L:
                if bit == 1:
                    ndp1[nk] = (ndp1[nk] + val1) % MOD
                # if bit == 0, cannot place 1 in tight state

        val0 = dp0[k]
        if val0:
            # already smaller, can place freely
            ndp0[k] = (ndp0[k] + val0) % MOD
            ndp0[k + 1] = (ndp0[k + 1] + val0) % MOD if k + 1 <= L else ndp0[k + 1]

    dp0, dp1 = ndp0, ndp1

ans = 0
for k in range(L + 1):
    cnt = (dp0[k] + dp1[k]) % MOD
    ans = (ans + cnt * fib[k]) % MOD

print(ans)
```该解决方案是围绕二进制表示形式的数字 DP 构建的。 斐波那契数组预先计算一次，每次 DP 状态转换都会根据当前的 popcount 更新构造数字的计数。 分裂成`tight`和`non-tight`各州强制执行由以下定义的上限`n`。
