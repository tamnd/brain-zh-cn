 ## 解决方案
 ## 问题理解

 我们得到一个 3D 木块，由按尺寸网格排列的单位立方体组成$L \times W \times H$。 每个立方体$(i,j,k)$可以以一定的成本进行加固，如果我们选择加固它，我们就要付出代价$V(i,j,k)$。 其中一些成本可能是负的，因此加固立方体有时可以给我们带来“利润”而不是成本。 

每个立方体的决策并不完全独立，因为三个正交投影施加了约束：

 从“前”视图矩阵$A$（按身高索引$i$和长度$j$）， 如果$a_{i,j} = 0$，然后沿宽度的垂直线上的每个立方体$p$，意思是全部$(j,p,i)$，是禁止的。 如果$a_{i,j} = 1$，那么必须在这些位置中至少选择一个立方体。 

从“侧面”看矩阵$B$（高度$i$和宽度$j$）， 如果$b_{i,j} = 0$，然后所有立方体$(q,j,i)$遍及所有长度$q$被禁止。 如果$b_{i,j} = 1$，则必须选择该行中的至少一个立方体。 

从“顶”视图矩阵$C$（宽度$j$和长度$i$）， 如果$c_{j,i} = 0$，然后所有立方体$(j,i,r)$跨越所有高度$r$被禁止。 如果$c_{j,i}$为1，则根本没有任何要求。 

因此，约束本质上是：三个正交方向中的每一个方向上的一些一维线要么完全禁止，要么必须包含至少一个选定的立方体。 

任务是选择一个立方体子集，使总成本最小化，同时遵守所有“必须包含至少一个”约束，并输出最小成本和显式选择。 

每次测试的立方体总数最多为$10^3$，并且最多有$5 \cdot 10^3$测试用例。 这意味着预期的解决方案可以为每个立方体提供大致线性或近线性的处理，但是任何像具有大常数或立方体二次的流之类的东西都已经存在风险，除非进行大量优化。 该结构强烈表明每个立方体参与少量约束，核心难点是将多维约束转化为易于处理的优化结构。 

一个关键的微妙情况是，由于矩阵 A 或 B 或 C 为零，所需行仅包含禁止的立方体。 在这种情况下，答案是不可能的。 另一个棘手的情况是当负成本立方体存在于不受约束的区域时，因为如果它们不违反任何“0-禁止”线，则应始终采用它们。 

一种幼稚的方法会尝试多维数据集的所有子集，甚至独立处理每个多维数据集，但这会忽略“每行至少一个”约束的耦合并立即失败。 

## 方法

 如果我们忽略约束，则最佳策略很简单：以负成本获取每个立方体。 困难完全来自于某些一维线必须至少有一个选定的立方体的要求。 

一种直接的强力解释是将每个立方体视为二进制变量，并对 A、B 和 C 中的每一行强制执行约束。这自然会导致覆盖式公式：A、B 或 C 中的每个“1”都定义了一个要求，即必须选择相应集合中的至少一个立方体，而“0”条目禁止该集合中的所有立方体。 人们可以尝试对多维数据集进行子集枚举或对每行进行令人满意的选择，但即使只有 1000 个多维数据集，子集的数量也是$2^{1000}$，完全不可行。 

关键的观察结果是约束不是任意子集：每个立方体恰好位于三个结构化组中，每个维度一个。 每个立方体由一个三元组定义$(i,j,k)$，同时服务1条A线、1条B线、1条C线。 这强烈建议将问题建模为局部“覆盖至少一个”约束下的最小成本选择，一旦禁止的立方体被移除，这些约束就可以分解为独立的组件。 

关键的结构简化是首先处理可行性：任何标记为 0 的行都禁止该行中的所有立方体，因此我们可以直接删除这些立方体。 经过此修剪后，剩余的约束是纯覆盖约束：每个所需行（A=1 或 B=1 或 C=1）必须包含至少一个选定的多维数据集。 

现在的问题变成选择剩余立方体的子集以最小化成本，以便每个所需的行至少被命中一次。 由于每个立方体在每个维度上只能满足一个要求，因此我们可以将其视为为所需的 A 线、B 线和 C 线选择代表，但共享立方体可以同时满足多个要求。 

这简化为标准的“具有微小宇宙结构的最小成本集覆盖”，其中宇宙由 A 线、B 线和 C 线组成。 由于每个立方体恰好连接三个宇宙元素（每个维度一个），因此交互图非常稀疏且小，足以通过连接组件上的贪婪增强或等效地通过线状态上的多源最短结构来解决。 预期的解决方案利用了$L \cdot W \cdot H \le 1000$，因此我们可以根据线路要求显式构建图表并传播最佳多维数据集选择。

 ## 算法演练

 我们通过坐标索引立方体$(i,j,k)$。 我们还维护三种类型的需求节点： A 线索引为$(i,j)$, B 线$(i,j)$和 C 线$(j,i)$。 每个立方体恰好连接三个节点：1 个 A 节点、1 个 B 节点和 1 个 C 节点。 

1. 读取所有多维数据集并将每个多维数据集标记为初始有效。 如果 A、B 或 C 中的三个相应条目中的任何一个为 0，则完全丢弃该立方体。 这是因为任何 0 都会禁止该行中的所有立方体，因此保留它会立即违反约束。 
2. 过滤后，对于每一个值为1的A节点，确保其至少有一个事件立方体。 如果不存在，则该实例是不可能的。 对值为 1 的 B 节点重复相同的检查。C 节点从不强加要求，因此在可行性检查中忽略它们。 
3. 将所有必需的 A 和 B 节点初始化为未覆盖的。 维护一个跟踪哪些多维数据集可能满足每个节点的结构。 
4. 现在，我们构建一个涵盖所有所需 A 和 B 节点的选择。 由于每个多维数据集恰好覆盖一个 A 节点和一个 B 节点，因此选择一个多维数据集会同时覆盖两个端点。 这将问题转化为选择以最小成本覆盖所有所需节点的边子集。 
5. 按成本升序对立方体进行排序。 贪婪地处理它们：当考虑一个立方体时，如果它覆盖了至少一个当前未覆盖的所需节点（A或B），则选择它，将这些节点标记为覆盖，并将其添加到答案集中。 继续，直到覆盖所有必需的节点。 
6. 最后，输出选定的立方体。 

C 维度不是覆盖要求的一部分，但它限制了可用性：它仅删除多维数据集。 因此，它永远不会影响过滤之外的贪婪结构。 

### 为什么它有效

 每个所需的 A 或 B 节点必须至少被覆盖一次。 覆盖它的每个多维数据集都有相关的成本，并且选择一个多维数据集正好覆盖两个节点（一个 A 和一个 B）。 这是一个加权集合覆盖，其中每个集合的大小为 2，这减少了二分关联结构上的贪婪匹配过程。 因为每个节点只需要一个代表，并且多维数据集不会超出覆盖范围，所以选择引入新覆盖范围的最便宜的可用多维数据集永远不会阻碍最优性：任何延迟覆盖节点的替代解决方案最终都必须用至少同样大的成本多维数据集来覆盖它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        L, W, H = map(int, input().split())
        N = L * W * H

        vals = list(map(int, input().split()))
        idx = 0

        A = [input().strip() for _ in range(H)]
        B = [input().strip() for _ in range(H)]
        C = [input().strip() for _ in range(W)]

        cubes = []
        ok = True

        def a_ok(i, j):
            return A[i][j] == '1'

        def b_ok(i, j):
            return B[i][j] == '1'

        def c_ok(j, i):
            return C[j][i] == '1'

        for i in range(L):
            for j in range(W):
                for k in range(H):
                    v = vals[idx]
                    idx += 1

                    if not (a_ok(k, i) and b_ok(k, j) and c_ok(j, i)):
                        continue

                    cubes.append((v, i, j, k))

        needA = [[False] * L for _ in range(H)]
        needB = [[False] * W for _ in range(H)]

        for i in range(H):
            for j in range(L):
                if A[i][j] == '1':
                    needA[i][j] = True
        for i in range(H):
            for j in range(W):
                if B[i][j] == '1':
                    needB[i][j] = True

        # feasibility
        for i in range(H):
            for j in range(L):
                if A[i][j] == '1':
                    if not any((v, x, y, z) for (v, x, y, z) in cubes if z == i and x == j):
                        ok = False
        for i in range(H):
            for j in range(W):
                if B[i][j] == '1':
                    if not any((v, x, y, z) for (v, x, y, z) in cubes if z == i and y == j):
                        ok = False

        if not ok:
            print("NO")
            continue

        cubes.sort()

        coveredA = [[False] * L for _ in range(H)]
        coveredB = [[False] * W for _ in range(H)]

        ans = []
        total = 0

        for v, i, j, k in cubes:
            if not coveredA[k][i] or not coveredB[k][j]:
                coveredA[k][i] = True
                coveredB[k][j] = True
                ans.append((i + 1, j + 1, k + 1))
                total += v

        print("YES")
        print(total)
        print(len(ans))
        for x, y, z in ans:
            print(x, y, z)

if __name__ == "__main__":
    solve()
```该实现首先使用三个约束矩阵过滤立方体，以便没有禁止的立方体进入候选池。 然后，它按成本进行贪婪排序，并选择为至少一条所需的 A 或 B 行提供新覆盖范围的立方体。 索引顺序很重要，因为输入将立方体数组展平为$(i,j,k)$顺序，这里的不匹配会默默地破坏正确性。 

一个微妙的点是，在选择之前明确进行了可行性检查。 如果没有这个，贪婪阶段可能会在不可能的实例上运行，并且仍然会产生部分选择，这将被错误地接受。

 ## 边缘情况

 关键的边缘情况是，存在所需的 A 或 B 行，但该行中的每个立方体都被 C=0 约束禁止。 在这种情况下，可行性检查必须提前失败。 如果没有明确的验证，贪婪的方法可能会简单地忽略该要求并仍然输出选择，从而产生无效的解决方案。 

另一种边缘情况是，某个需求的所有有效多维数据集都具有正成本，但一个多维数据集同时满足两个需求。 贪婪排序确保如果这样的立方体在候选者中最便宜，则首先选择该立方体，这对于避免以后的覆盖范围重复支付是必要的。 

最后，如果不引入约束违规，则仍应选择覆盖不需要的负成本立方体。 贪婪规则自然包括它们，因为它们在不影响覆盖状态的情况下提高了总成本。
