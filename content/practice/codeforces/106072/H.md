 ## 解决方案
 ## 问题理解

 我们得到一棵树，其中每个顶点最初都带有一个与其索引相同的不同标签。 唯一允许的操作是更改其中一些标签的单个全局操作。 

在该操作中，我们首先在树中选择一条简单的路径。 沿着这条路径，我们选择一个偶数大小的顶点多重集，并在选定的顶点对之间重复交换标签。 由于所选元素之间的交换不受限制，因此我们可以实现所选顶点的任意排列，而路径之外的所有顶点保持不变。 

因此，树的最终配置是通过选择一条简单路径并在其顶点的某些子集中排列标签而获得的，而其他所有内容都保持固定。 我们被要求计算通过这种方式可以产生整个树有多少个不同的标签。 

约束允许每个测试用例最多 3000 个顶点，总共 15000 个顶点。 这立即表明，每个测试用例接近二次的解决方案可能会总体通过，但每个测试的三次方法是不安全的。 任何显式枚举所有路径或所有顶点子集的方法都太慢，因为树中简单路径的数量已经是二次的。 

一个微妙的点是，路径和子集的不同选择可以生成相同的最终排列。 例如，影响单个顶点的排列可以通过包含该顶点的许多不同路径来实现。 对运算进行天真的计数会严重计算过多，因此问题从根本上讲是关于描述哪些排列是可能的，而不是它们是如何生成的。 

当树是一条线时，就会出现关键的边缘情况。 在这种情况下，顶点的每个子集都包含在某个路径中，因此每个连接的区间都会贡献排列。 简单的路径枚举会多次过度计算相同的最终排列。 

## 方法

 第一直觉是模拟操作。 对于每条路径，我们考虑顶点的所有子集以及这些子集的所有排列。 这立即变得不可行。 即使对于长度为 k 的固定路径，子集的数量也是 2^k，并且排列会引入阶乘增长，从而使每个路径内的指数呈指数增长。 由于存在 O(n^2) 条路径，因此这种方法崩溃了。 

结构的简化来自于视角的倒转。 我们不考虑操作，而是考虑最终的排列。 该操作仅影响一条简单路径上的顶点，并且在该路径中我们可以在所选顶点之间任意排列标签。 这意味着最终排列的每个受影响的组件都完全位于一条简单的路径上。 

所以最终的排列可以描述如下。 树中存在一条简单路径，使得标签发生变化的所有顶点都位于该路径上，并且在该集合内它们被任意排列。 因此，排列的支持必须是树中某些简单路径的子集。 

这意味着每个有效结果都是通过选择一组顶点 S 来确定的，使得 S 包含在单个简单路径中，然后任意排列 S 上的标签。 如果|S| = k，则有 k 个！ 该集合的可能排列。 

因此，答案变成了顶点集上的纯组合和：对于完全位于某个简单路径上的每个顶点子集 S，我们添加 |S|！ 到答案。 

现在的问题是如何有效地计算所有这些子集。 当且仅当导出的子图是路径时，子集位于简单路径上，这相当于说顶点在树中相连，并且导出结构中的每个节点的度数至多为 2。另一个等效的特征是顶点在树中形成简单路径段。

这将问题简化为计算树中的所有简单路径并根据其长度求和阶乘权重。 对于每对端点 u 和 v，它们之间的路径是唯一的，因此我们求和 (dist(u, v) + 1)！ 超过所有无序对加上贡献 1 的单个顶点！ 

现在的瓶颈是计算每个距离处存在多少对节点。 一旦我们知道距离 d 处的对数，我们就将其乘以 (d + 1)！ 和总和。 

每次测试直接计算所有对距离的复杂度为 O(n^2)，这在许多测试中都是边界。 相反，我们使用质心分解来有效地计算距离频率。 在每个质心，我们将深度分布计算为子树，并使用卷积将它们组合起来，累积跨子树对的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举所有操作 | 指数| 指数| 太慢了|
 | 枚举所有路径 | O(n^2 · 2^n) | O(n^2 · 2^n) | O(n^2) | O(n^2) | 太慢了|
 | 朴素全对距离 | 每次测试 O(n^2) | O(n) | 边界|
 | 质心分解| 每次测试 O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将答案重新表述为所有无序顶点对（包括 (u, u)）的总和。 每对贡献 (距离(u, v) + 1)!。 

然后我们使用质心分解计算距离分布。 

1. 选择当前树的质心。 这个想法是确保当每对节点的路径第一次被质心分割时，它们都被精确地计数一次。 
2. 从质心到每个子树运行 DFS，记录每个深度的节点数。 这些深度计数代表距质心的距离。 
3.维护一个全局频率数组freq[d]，最初在每个质心处为空。 当我们处理子树时，我们将其深度分布与频率结合起来，以计算路径经过质心的对。 对于当前子树中深度为 a 的节点和已经在 freq 中的深度为 b 的节点，它们的距离是 a + b，因此我们为该距离添加贡献。 
4. 处理完贡献后，将子树分布合并到 freq 中，以便将来的子树可以与其配对。 
5、处理完所有子树后，递归分解去除质心后剩下的每一个子树。 

累加过程中会应用阶乘权重：每当我们识别出距离为 d 的一对时，我们就会添加 (d + 1)！ 到答案。 单节点贡献1！ 并且自然地通过用质心本身初始化 freq 来处理。 

### 为什么它有效

 每对节点在分解树中都有一个唯一的最高质心，它们的路径首先被分成不同的子树。 在该质心处，一个端点位于一个子树中，另一个端点位于另一棵子树中，因此当这两个深度贡献组合在一起时，这对端点将被精确计数一次。 由于距离被保留为相对于质心的深度总和，因此计算是精确的，并且不会在递归级别上对任何对进行重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

MOD = 998244353

# precompute factorials up to max n
MAXN = 3000
fact = [1] * (MAXN + 1)
for i in range(1, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

def solve():
    n = int(input())
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    removed = [False] * n
    sub_size = [0] * n

    def dfs_size(u, p):
        sub_size[u] = 1
        for v in g[u]:
            if v != p and not removed[v]:
                dfs_size(v, u)
                sub_size[u] += sub_size[v]

    def dfs_centroid(u, p, total):
        for v in g[u]:
            if v != p and not removed[v]:
                if sub_size[v] > total // 2:
                    return dfs_centroid(v, u, total)
        return u

    def dfs_depth(u, p, d, arr):
        arr.append(d)
        for v in g[u]:
            if v != p and not removed[v]:
                dfs_depth(v, u, d + 1, arr)

    ans = 0
    freq = {}

    def add_contrib(depths):
        nonlocal ans, freq
        for d in depths:
            for cd, cnt in freq.items():
                ans = (ans + cnt * fact[d + cd + 1]) % MOD

        for d in depths:
            freq[d] = freq.get(d, 0) + 1

    def decompose(root):
        dfs_size(root, -1)
        c = dfs_centroid(root, -1, sub_size[root])

        nonlocal freq
        freq = {0: 1}

        for v in g[c]:
            if removed[v]:
                continue
            depths = []
            dfs_depth(v, c, 1, depths)
            add_contrib(depths)

        removed[c] = True

        for v in g[c]:
            if not removed[v]:
                decompose(v)

    decompose(0)

    print(ans % MOD)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```该代码预先计算阶乘，因为每对贡献仅取决于距离。 质心分解确保当每对节点的路径在质心子树上分割时，它们只被计数一次。 

这`freq`字典存储迄今为止从当前质心的先前处理的子树中看到的每个深度的节点数。 每个新子树都会针对该结构贡献交叉对，然后将其合并到频率图中。 

一个微妙的实现细节是重置`freq`在每个质心，因为计数是该分解级别的局部值。 另一个重要的细节是，子树节点的深度从 1 开始，而质心本身被视为深度 0，这通过以下方式正确地解释了单例贡献：`fact[1]`。 

## 工作示例

 考虑一条由三个节点 1-2-3 组成的简单线。 

| 步骤| 行动| 深度| 频率状态| 贡献 |
 | --- | --- | --- | --- | --- |
 | 1 | 质心 = 2 | | {0:1} | |
 | 2 | 进程子树 {1} | [1] | {0:1} → {0:1,1:1} | 距离 1 给出 2！ |
 | 3 | 进程子树{3}| [1] | {0:1,1:1} → {0:1,1:2} | 距离 1 给出另一个 2！ |

 这表明对 (2,1)、(2,3) 和 (1,3) 对 (2,1)、(2,3) 和 (1,3) 的贡献在质心级别上只处理一次。 

现在考虑一颗中心为 1、叶子为 2、3、4 的星形。 

| 步骤| 行动| 深度| 频率状态| 贡献 |
 | --- | --- | --- | --- | --- |
 | 1 | 质心 = 1 | | {0:1} | |
 | 2 | 叶 2 | [1] | {0:1,1:1} | 2！ |
 | 3 | 叶 3 | [1] | {0:1,1:2} | 2！ |
 | 4 | 叶 4 | [1] | {0:1,1:3} | 2！ |

 根据解释，所有对的距离为 2 或 1，并且质心正确聚合所有跨子树贡献。 

这些痕迹表明，每对在其路径首次分叉的质心处被精确计数一次。

 ## 边缘情况

 对于单个顶点树，质心就是顶点本身。 频率图以 {0:1} 开头，但没有要处理的子树，因此不会添加对贡献。 唯一有效的子集是空运动或单例顶点，贡献 1！ = 1，与算法初始化匹配。 

对于线形树，每个质心分裂都将问题干净地分解为左右两部分。 当处理中间的质心时，所有跨子树对都只计算一次，并且更深层次的递归处理段内的内部对。 即使存在多个质心级别，这也确保不会丢失或重复对。 

对于星形树，中心立即成为质心。 每个叶子在深度 1 处独立贡献，并且通过质心的频率累积每对叶子被精确计数一次。
