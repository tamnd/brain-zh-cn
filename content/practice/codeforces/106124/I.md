 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每个顶点代表一个人，每个有向边代表一个“跟随”关系。 如果有一条边从$u$到$v$，那么人$u$跟随人$v$。 

对于每个顶点$v$，我们想要衡量“名人目标”的强度有多大。 一群人$S$有效期为$v$如果每个人在$S$如下$v$， 和$v$没有人跟随$S$。 名人中心地位$CC(v)$是这样一个组的最大可能大小$S$。 

所以对于一个固定的$v$，我们本质上是在寻找有多少新加入的关注者$v$可以收集，但我们只允许统计那些没有被关注的粉丝$v$。 任何有一条边的顶点$v$不能被纳入该组。 

任务是计算$CC(v)$对于每个顶点并返回具有最大值的顶点，通过选择最小的索引来打破平局。 

输入数据量很大，最多可达$10^5$顶点和$10^6$边缘。 这立即排除了任何通过边缘上的嵌套扫描重新计算每个顶点信息的解决方案，因为这会偏向$O(NM)$最坏情况下的行为。 需要一种边数更接近线性的解决方案。 

当存在相互边缘时，就会出现微妙的边缘情况。 如果$u \leftrightarrow v$， 然后$u$有助于$CC(v)$仅当$v \nrightarrow u$，在这种情况下是假的，所以不能被计算在内。 另一个极端情况是孤立的顶点，其中没有传出边的顶点自动接受所有传入的追随者。 

一个天真的错误是简单地计算每个顶点的入度并假设是$CC(v)$。 当存在双向边时，此操作会失败。 例如，如果我们有$1 \to 2$和$2 \to 1$，则入度(1) = 1 但$CC(1) = 0$因为 1 跟随 2，所以 2 不能属于该组。 

## 方法

 一种直接的方法是处理每个顶点$v$独立。 对于固定的$v$，我们可以迭代所有顶点$u$，检查是否$u \to v$存在，并验证$v \nrightarrow u$。 如果两个条件都成立，我们就计算$u$。 这给出了准确的定义$CC(v)$。 

正确性是立竿见影的，但是成本太大。 以简单的方式检查邻接性需要扫描边缘列表或使用矩阵。 对于邻接表，每次检查的成本可达$O(\deg(u))$，导致最坏情况接近$O(NM)$，这对于$10^5$和$10^6$。 

关键的观察是$CC(v)$几乎是$v$，除非我们必须减去那些也有传出边缘的传入邻居$v$。 这种“互边缘过滤器”是唯一的障碍。 我们可以将所有边预处理为快速隶属结构，而不是重新计算每个顶点的关系，以便我们可以测试是否存在$v \to u$在恒定的时间内。 

我们存储邻接集以进行快速查找。 然后对于每个顶点$v$，我们遍历它的传入邻居并只计算那些$u$这样$v \to u$缺席。 这将问题简化为边缘线性扫描。

 ## 算法演练

 我们重新组织计算，以便显式维护传入和传出的邻接信息。 

1. 为每个顶点的出边建立邻接集。 这允许恒定时间检查是否存在有向边。 
2. 为每个顶点构建传入邻居列表。 这是通过在输入解析期间反转方向直接从边缘导出的。 
3. 对于每个顶点$v$, 迭代所有顶点$u$在其传入列表中。 
4. 对于每一个这样的$u$，仅当以下情况时才将其包含在计数中：$u$不在出邻接集中$v$。 
5. 将计算出的计数记录为$CC(v)$。 
6. 跟踪具有最大值的顶点，并在平局的情况下保留最小索引。 

基本的推理步骤是传入的邻居是包含在该组中的唯一候选者。 每个有效的团体成员都必须遵守$v$，因此它们必须出现在传入邻接列表中。 唯一的额外限制是删除那些共同追随者。 

### 为什么它有效

 该算法永远不会错过有效的组成员，因为每个候选者都必须出现在传入的邻接列表中$v$。 它永远不会计算无效成员，因为任何顶点$u$这样$v \to u$使用邻接集显式过滤掉存在。 因为每一对$(u, v)$在传入的遍历中仅被考虑一次$v$，计算出的计数恰好是最大有效组的大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    out_adj = [set() for _ in range(n + 1)]
    in_adj = [[] for _ in range(n + 1)]
    
    for _ in range(m):
        u, v = map(int, input().split())
        out_adj[u].add(v)
        in_adj[v].append(u)
    
    best_v = 1
    best_val = -1
    
    for v in range(1, n + 1):
        cnt = 0
        for u in in_adj[v]:
            if v not in out_adj[u]:
                cnt += 1
        if cnt > best_val or (cnt == best_val and v < best_v):
            best_val = cnt
            best_v = v
    
    print(best_v, best_val)

if __name__ == "__main__":
    solve()
```传出邻接被存储为一个集合，以保证在过滤相互边时进行恒定时间的成员资格检查。 传入的邻接被存储为一个简单的列表，因为我们只按顺序迭代它。 

顶点循环计算$CC(v)$通过仅扫描真正的候选者，这避免了对不相关边缘的任何重复扫描。 平局决胜逻辑在同一遍中处理，以避免存储所有结果。 

一个常见的实施陷阱是错误地反转邻接关系。 传入列表必须使用目标端点（而不是源）构建。 另一个错误是使用列表而不是集合作为传出边缘，这会将成员资格检查降低为线性时间并破坏性能。

 ## 边缘情况

 关键的边缘情况是每个传入边缘都与传出反向边缘配对。 考虑两个节点上的完全双向图：$1 \leftrightarrow 2$。 1 的传入列表是$[2]$，但由于 1 也跟随 2，因此候选者被删除并且$CC(1)=0$。 这同样适用于顶点 2。 

另一种情况是纯sink结构，比如$2 \to 1, 3 \to 1, 4 \to 1$。 这里顶点 1 有传入的邻居$[2,3,4]$，并且由于它不跟随任何一个，所以所有的都被计算在内。 该算法处理每个候选者并确认输出集中不存在，产生$CC(1)=3$。 

具有孤立顶点的稀疏图也表现得很自然。 如果顶点没有传入边，则其传入列表为空并且其得分为零。 该算法不需要对这种情况进行特殊处理，因为循环不执行迭代。
