 ## 解决方案
 ## 问题理解

 我们得到了位置上的隐藏排列$1 \ldots N$。 这个排列定义了长度的二进制字符串如何$N$被变换：每一步只是根据排列对位进行重新排序。 重复应用排列最终会将任何固定的二进制字符串恢复到自身，因为排列分解为不相交的循环，并且每个循环仅旋转位。 

对于每个二进制字符串$x \in \{0,1\}^N$，我们将其周期定义为将字符串返回到其原始状态所需的最小排列应用数。 我们没有得到排列。 相反，我们会得到一组周期值，并且对于每个周期$p$，有多少个二进制字符串恰好具有该周期。 

任务是重建任何可以精确产生这种周期分布的排列，或者报告不存在这样的排列。 

就以下方面而言，限制很小$N$， 和$N \le 100$，但是字符串的数量可能是天文数字，最多可达$2^{100}$。 这立即表明该解决方案无法模拟字符串或枚举状态。 一切都必须简化为循环结构上的算术。 

一个关键的结构事实是，排列完全确定了二进制串的周期，而这些周期仅取决于循环分解。 所以问题实际上是：重建与给定聚合计数一致的循环结构。 

当人们假设周期与周期长度直接对应时，就会出现一种微妙的失败情况。 这是错误的，因为二进制串可以同时在多个周期下具有周期性，并且其周期是周期贡献的最小公倍数。 任何忽略周期之间相互作用的方法都会出现错误计数。 

例如，如果排列具有长度为 2 和 3 的循环，则在两个循环上恒定的字符串具有周期 1，而根据对齐方式混合位会不同地产生周期 2、3 或 6。 因此，天真的“一个周期等于一个周期类”的推理立即就被打破了。 

## 方法

 蛮力的想法是枚举所有排列$N$，计算它们的周期分解，并为每个排列模拟所有二进制字符串以计算它们的周期。 这在概念上很简单：对于每个字符串，重复应用排列直到它返回。 然而，有$2^N$字符串，最多$N!$排列，使得这完全不可行。 即使对于单个排列，所有字符串的计算周期也是指数级的。 

关键的观察是我们永远不需要查看单个字符串。 排列分为不相交的循环，每个循环的行为都是独立的，除了在涉及多个循环时采用最小公倍数之外。 二进制串的周期由每个周期的周期性决定，特别是由每个周期内轮换下比特的重复方式决定。 

在一个长度周期内$L$，二进制字符串的限制行为类似于循环字符串，其周期除以$L$。 更准确地说，每个周期贡献一个由旋转控制的除数结构，并且全局周期是该周期引起的局部周期的最小公倍数。 

这将问题简化为构造循环长度的多集，其导出的算术与给定的周期直方图相匹配。 我们不是推理字符串，而是推理由循环引起的除数和最小公倍数。 

关键的简化在于，每个循环对乘法结构都有独立的贡献，并且给定周期的字符串总数可以通过循环结构上的包含-排除来表示。 这将问题转化为重建整数的多重集，其除数相互作用与给定计数匹配，这可以通过候选周期除数的莫比乌斯求逆来处理。 

我们本质上是猜测循环结构并使用循环长度除数的算术恒等式来验证一致性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N! \cdot 2^N \cdot N)$|$O(N)$| 太慢了|
 | 使用除数运算重建循环 |$O(N^2 \log N)$|$O(N)$| 已接受 |

 ## 算法演练

 我们首先重新解释输入。 我们被赋予一个函数$F(p)$，周期恰好为的二进制串的数量$p$。 由此我们得出一个累积函数$G(p)$，其句点被整除的字符串的数量$p$。 这是通过对所有的求和得到的$F(d)$超过除数$d \mid p$。 

这种累积形式更为自然，因为“周期除以$p$” 对应于通过应用排列来固定$p$次。 

然后我们利用字符串被固定的事实$f^p$恰好当每个长度周期$L$受到尊重$p$旋转，仅取决于 gcd 结构$p$和$L$。 对于一个周期长度$L$，它贡献$2^{\gcd(L,p)}$固定任务。 乘以周期即可得出总固定计数。 

所以$G(p)$等于循环的乘积：$$G(p) = \prod_{cycles\;L} 2^{\gcd(L,p)} = 2^{\sum \gcd(L,p)}$$以 2 为底的对数将乘法转换为加法：$$\log_2 G(p) = \sum \gcd(L,p)$$现在问题变成：找到循环长度的多重集，使得这个 gcd-sum 恒等式适用于所有$p$。 

我们使用除数变换求逆来解决这个问题。 

## 算法演练

 1. 转换给定的精确周期计数$F(p)$转化为累计计数$G(p)$通过对除数求和。 这将数据与排列幂下的定点计数对齐。 
2. 取所有值的以 2 为底的对数$G(p)$。 由于通过构造，值是 2 的幂，这会产生表示的整数$\sum \gcd(L,p)$。 如果任何值不是 2 的幂，则不存在排列。 
3. 设置数组$A(p) = \log_2 G(p)$。 这通过 gcd 相互作用编码了循环长度的贡献。 
4. 使用除数上的莫比乌斯求逆来提取与精确循环长度相对应的贡献。 我们计算一个函数$C(L)$代表多少个周期的长度$L$必须存在，以便他们的 GCD 贡献能够重建$A(p)$。 
5. 验证所有$C(L)$是非负整数和所有的总和$L \cdot C(L)$等于$N$。 这确保了有效的排列结构。 
6. 通过连接循环构建排列：对于每个长度$L$， 创造$C(L)$大小不相交的循环$L$按标签递增顺序。 

### 为什么它有效

 核心不变量是置换幂下的定点计数通过 gcd 算法唯一确定循环结构。 每个周期都会额外贡献$\log_2 G(p)$通过$\gcd(L,p)$，莫比乌斯反演隔离了每个周期长度的贡献，而不受其他长度的干扰。 这种分离是精确的，因为 gcd 可以在除数格上进行乘法，从而可以完美恢复循环多重性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    N, K = map(int, input().split())
    p = list(map(int, input().split()))
    m = list(map(int, input().split()))

    F = {}
    for pi, mi in zip(p, m):
        F[pi] = mi

    # build divisors map up to max p
    maxp = max(p)

    def get_divs(x):
        divs = []
        i = 1
        while i * i <= x:
            if x % i == 0:
                divs.append(i)
                if i * i != x:
                    divs.append(x // i)
            i += 1
        return divs

    G = {}
    for pi in p:
        s = 0
        for d in get_divs(pi):
            if d in F:
                s += F[d]
        G[pi] = s

    # check powers of two
    A = {}
    for pi in p:
        v = G[pi]
        if v <= 0 or (v & (v - 1)) != 0:
            print("impossible")
            return
        A[pi] = v.bit_length() - 1

    # transform by Möbius-like inversion on divisor poset
    C = {}

    for i in sorted(p, reverse=True):
        val = A[i]
        for j in p:
            if j < i and i % j == 0:
                val -= C.get(j, 0)
        C[i] = val
        if val < 0:
            print("impossible")
            return

    # build permutation
    res = list(range(1, N + 1))
    idx = 0
    for length in sorted(C):
        cnt = C[length]
        for _ in range(cnt):
            if idx + length > N:
                print("impossible")
                return
            cycle = res[idx:idx + length]
            for t in range(length):
                cycle[t] = cycle[(t + 1) % length]
            res[idx:idx + length] = cycle
            idx += length

    if idx != N:
        print("impossible")
        return

    print(*res)

if __name__ == "__main__":
    main()
```第一阶段将给定周期计数聚合为除数的累积固定计数。 这是必要的，因为排列幂行为自然地用重复应用下的固定点而不是精确的最小周期来表达。 

对数步长至关重要，因为循环贡献会在定点计数中成倍增加。 如果任何值不是 2 的幂，则该结构无法对应每个周期的独立二进制赋值，因此实例不一致。 

反转步骤从较大的指数向下进行，因此在处理值时，严格除数的所有贡献都已被考虑在内。 这反映了除数格上的经典莫比乌斯求逆。 

最后，排列构造简单地布置了规定长度的不相交循环。 正确性依赖于这样一个事实：一旦循环计数固定，任何不相交循环的排列都会产生等效的排列。 

## 工作示例

 考虑一个简化的情况$N=2$，身份排列。 

我们期望两个二进制字符串的周期都是 1。 

| 步骤| p=1 计数 | 累积 G(1) | 日志2 |
 | --- | --- | --- | --- |
 | 初始化| 2 | 2 | 1 |

 这会为每个元素产生长度为 1 的一个循环贡献，从而产生同一性。 

现在考虑 2 周期排列。 

所有字符串的句点为 1 或 2，具体取决于等式约束。 

| 步骤| p = 1 | p = 2 | G(1) | G(2) | G(2) |
 | --- | --- | --- | --- | --- |
 | 初始化| 2 | 2 | 2 | 4 |

 | 步骤| log2 G(1) | log2 G(1) | log2 G(1) log2 G(2) | log2 G(2) | log2 G(2)
 | --- | --- | --- |
 | 价值| 1 | 2 |

 这表示一个长度为 2 的循环，产生交换排列。 

这些痕迹显示了循环结构如何纯粹以定点计数进行编码，以及对数线性化如何揭示底层分解。

 ## 边缘情况

 一种边缘情况是所有质量都集中在周期 1。在这种情况下，所有字符串在一个应用程序下都是固定的，这迫使排列是恒等的。 该算法检测到这一点是因为所有累积值都是 2 的最小幂，仅产生长度为 1 的周期。 

另一个边缘情况是对数值不一致。 如果有的话$G(p)$不是 2 的幂，反转步骤立即失败。 这可以正确处理无法对应于独立循环贡献的输入，例如违反乘法结构的人为夸大的计数。 

第三种情况是在求逆过程中除数一致性失败。 如果候选循环长度在减去较大除数的贡献后意味着负重数，则表明输入分布中存在过度计数，并且算法正确地拒绝它。
