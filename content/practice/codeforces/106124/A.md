 ## 解决方案
 ## 问题理解

 我们得到一个整数`s`，我们需要将它分成两个整数`a`和`b`使得它们的总和等于`s`。 两个都`a`和`b`必须为非零，并且两者都必须在三位数整数范围内，即介于 −999 和 999 之间（包括 −999 和 999）。 

任务不是优化长度或绝对值之类的东西。 只要遵守约束，任何有效的分解都是可以接受的。 输出只是打印在一行上的两个整数。 

约束窗口非常小。 每个有效的候选者都位于一个大小约为 2000 x 2000 的有限盒子中。 这立即意味着，即使是对所有可能性的简单搜索，在 Python 中仍然足够快。 然而，问题的结构表明我们应该完全避免搜索。 

唯一微妙的失败案例来自约束边界。 一个天真的想法，就像总是使用`a = 0`和`b = s`立即失败，因为零是被禁止的。 另一个常见的错误是忽略三位数界限，例如生成`a = s`和`b = 0`，或生成一个超出 ±999 的值`s`接近边界并且调整不正确。 

一个具体的问题案例是`s = 0`。 如果我们尝试`a = 0`,`b = 0`，两者均无效。 另一个是`s = 1000`，其中一个朴素的对称分割就像`a = 500`,`b = 500`有效，但是`a = 0`,`b = 1000`同时违反这两个约束。 

## 方法

 暴力方法会枚举所有整数`a`在有效范围内并计算`b = s - a`，然后检查两者是否有效且非零。 这是有效的，因为搜索空间最多有 1999 个候选者`a`，并且每次检查都是常数时间。 所以总共大约有两千次操作，速度相当快。 

然而，这是不必要的，因为问题的结构保证了有效的构造总是很容易直接形成。 由于我们只需要两个非零数字，因此我们可以故意先选择一个数字，然后通过减法定义另一个数字。 

关键的观察是我们只需要避免一个坏值：零。 如果我们选择`a = 1`， 然后`b = s - 1`。 唯一的风险是当`b = 0`，这恰好发生在`s = 1`。 在这种情况下我们可以选择`a = 2`,`b = -1`。 两者均非零且仍在范围内。 因为`s`限制为±999，这些固定选择绝不会超出允许的范围。 

因此，该解决方案简化为恒定时间条件构造，而不是任何搜索。

 ## 算法演练

 我们通过小心地避免零来直接构建答案。 

1.读取整数`s`。 这是我们需要分成两个非零部分的总和。 
2.如果`s`不等于1，设置`a = 1`。 这个选择是安全的，因为`a`是非零且在界限内，并且它简化了第二个数字的构造。 
3. 计算`b = s - a`。 自从`a`是固定的，这直接强制执行所需的总和条件。 
4.如果`b`结果为零，通过设置来调整结构`a = 2`和`b = s - 2`。 这保证了两个数字都非零，同时保留总和。 
5、输出`a`和`b`。 

我们明确避免的唯一例外情况是第一次尝试产生零时，我们通过第二次有效分解来修复它。 

### 为什么它有效

 我们始终执行`a + b = s`通过建设，自`b`定义为`s - a`。 唯一可以违反的约束是`b ≠ 0`，只有当`a = s`。 通过选择默认值`a = 1`，我们为所有人避免这种情况`s ≠ 1`。 什么时候`s = 1`，修正`a = 2, b = -1`提供允许范围内的有效分解。 由于允许的间隔足够宽以容纳这些固定的回退值，因此不会发生边界违规。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = int(input())

a = 1
b = s - a

if b == 0:
    a = 2
    b = s - 2

print(a, b)
```实施完全遵循建设性战略。 我们从最简单的非零值开始，`1`，并导出合作伙伴价值。 唯一的修正是在极少数情况下，这会迫使`b`变为零，这只发生在`s = 1`。 在这种情况下，我们切换到第二个固定对。 

除了这个单一修正之外，没有循环，没有搜索，也没有边缘情况分支。

 ## 边缘情况

 对于`s = 0`，算法集`a = 1`，生产`b = -1`。 两个值均非零且在范围内，因此无需校正。 

为了`s = 1`，初始选择给出`b = 0`，这违反了约束。 该算法检测到这一点并切换到`(2, -1)`，恢复有效性。 

对于较大的正值或负值，例如`999`或者`-999`，固定结构仍然使两个数字保持在允许的范围内，因为调整仅从`s`。
