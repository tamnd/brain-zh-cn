 ## 解决方案
 ## 问题理解

 我们正在模拟驾驶员穿过一系列路标，其中每个路标要么设置特定的速度限制，要么删除当前的限制并恢复原始的国家速度限制。 

关键细节是，我们不知道国家限速，只是它是十的倍数，并且严格高于任何明确的限速标志。 这意味着每当我们看到“重置”标志时，我们都无法直接恢复固定的数字。 相反，我们必须推断出与迄今为止观察到的所有情况一致的最大可能速度。 

任务是有顺序的。 在每个标志之后，我们必须输出保证在该点之前看到的所有约束条件下合法的最高速度。 

输入是操作流。 每个操作要么是一个数字限制，立即将最大允许速度限制为该值，要么是指示重置的斜杠符号，这会删除所有明确的限制并使我们返回到未知的国家速度限制制度。 

限制很小，最多有 100 个标志。 这意味着二次时间以内的任何解决方案都已经是安全的，但问题的结构足够简单，线性模拟就足够了。 

重置周围会出现微妙的故障情况。 如果人们错误地假设国家速度限制是固定的或试图“记住”猜测值，那么逻辑就会崩溃。 

例如，考虑：

 输入：```
/
```这是不可能的，因为第一个标志必须是速度限制，但它突出了一个问题：重置后，我们没有具体的数字，除非我们从先前的限制中得出它。 

更现实的边缘情况是：

 输入：```
50
80
/
```正确推理：

 50 后，上限为 50。 

80 后，限制变为 50（因为我们总是遵守迄今为止看到的最小限制）。 

重置后，该限制将成为未知的国家速度限制，该限制必须严格大于任何可见的限制，因此至少为 90（因为它必须是 10 的倍数）。 这样答案就变成了90。 

简单地“忘记约束”并在重置后打印无界值或零的简单实现是不正确的。 

核心困难在于重置需要从历史中重建一致的上限，而不仅仅是清除状态。 

## 方法

 强力解释将尝试通过重新考虑所有先前的约束来在每个符号之后重新计算有效的最大速度。 每次重置后，我们都会向后扫描以找到迄今为止看到的最大速度限制，然后假设国家速度限制是严格大于它的十倍的最小倍数。 这在概念上是有效的，因为国家限制必须与之前的所有限制一致。 

然而，如果扩展到大输入，这种方法就会变得低效，因为每次重置都可能触发对所有先前符号的完整扫描，从而在最坏的情况下导致二次行为。 尽管这里 N 很小，但这会促使简化状态。 

关键的观察是我们永远不需要从头开始重新计算。 唯一重要的信息是自上次重置以来看到的最大速度限制。 该值本身就决定了国家可能的最低速度限制：严格大于该速度限制的十的最小倍数。 因此，我们不是扫描历史记录，而是维护一个运行最大值，只要出现斜杠，该最大值就会重置。 

这将整个过程简化为单次传递，每个标志都有恒定时间的更新。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N^2) | O(N^2) | O(1) | O(1) | 已接受但没有必要 |
 | 最佳 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们维护一个状态：自上次重置以来看到的最高明确速度限制。 

1. 初始化变量`mx`到 0。这代表当前活动的最强约束。 
2.读取第一个符号，保证是数字，然后设置`mx`到那个值。 立即输出。 
3. 对于后续的每个标志，按顺序进行处理。 
4.如果符号是数字`L`， 更新`mx`成为`max(mx, L)`。 输出`mx`因为法定速度始终受到迄今为止最严格的限制。 
5. 如果标志是`/`，将国家速度限制计算为严格大于的十的最小倍数`mx`。 这可以完成为`((mx // 10) + 1) * 10`。 输出这个值，但重要的是不要重置`mx`为零； 相反，我们从概念上切换到“除国家限制外没有任何限制”模式，这意味着未来的明确限制再次从该基线开始重新建立约束。 
6. 继续，直到处理完所有标志。 

关键的想法是`mx`始终跟踪重置之间当前段中最强的显式约束。 斜线不会为了正确性而删除历史记录，它只会改变我们解释下一段的方式。 

### 为什么它有效

 在任何没有重置的情况下，合法性完全取决于自上次重置以来看到的所有速度限制的最小值。 该最小值正是当前的`mx`。 当重置发生时，唯一缺失的值是国家速度限制，但问题保证它是严格高于任何观察到的限制的十倍。 最小的值完全由下式决定`mx`，因此我们可以安全地输出它，而无需猜测其他任何内容。 没有其他历史信息影响当前的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    mx = 0
    first = True
    
    for _ in range(n):
        s = input().strip()
        
        if s == "/":
            # restore national speed limit
            mx = ((mx // 10) + 1) * 10
            print(mx)
        else:
            val = int(s)
            if first:
                mx = val
                first = False
            else:
                mx = max(mx, val)
            print(mx)

if __name__ == "__main__":
    solve()
```该实现保持运行最大值`mx`这代表当前段中最严格的显式限制。 当我们遇到数字符号时，我们会更新这个最大值。 当我们遇到斜线时，我们会向上舍入到下一个十的倍数，将当前最大值转换为隐含的国家限制。 

这`first`flag 确保初始状态被干净地处理，因为问题保证第一个标志始终是速度限制。 

一个常见的陷阱是重置`mx`归零`/`。 这会错误地暗示没有限制，而正确的行为是从先前的最大值推断出具体的界限。

 ## 边缘情况

 关键的边缘情况是重置前的最大值已经是十的倍数。 例如：

 输入：```
60
/
```我们计算`((60 // 10) + 1) * 10 = 70`，因此输出变为 70。重置后简单返回 60 的简单方法将违反国家速度限制必须严格高于迄今为止看到的任何明确限制的要求。 

另一种情况是小限制之后重置：

 输入：```
10
20
/
```我们追踪`mx = 20`，然后在复位输出 30 之后。这表明复位不会恢复为固定常数，而是完全取决于累积的历史记录。 

最后，没有新限制的重复重置确认了稳定性：

 输入：```
10
/
/
```步骤 1 给出 10，步骤 2 给出 20，步骤 3 给出 30。每次重置都会继续导出严格递增的序列，这符合国家限制始终高于先前观察到的限制的规则。
