 ## 解决方案
 ## 问题理解

 我们得到一条有 n 个位置的环形线，其中每个位置的值都在 1 到 m 之间。 我们还给出了一个确定性更新规则 f，它采用该圆上的三个连续值并生成中间位置的第二天值。 

换句话说，如果今天的隐藏状态是数组 a[1..n]，那么明天观察到的数组 s0 完全由以下规则决定：对于每个索引 i，i 处的值变为 f(a[i−1], a[i], a[i+1])，索引循环取。 

问题不是模拟正向，而是反向检查一致性。 我们给出了一个提议的状态 s0，并且必须决定是否存在至少一个先前的配置 a，以便应用该规则恰好产生 s0。 我们不需要重建它，只需判断是否存在有效的前驱即可。 

就字母大小而言，约束很小，但在长度上则不然：n 最多为 200，m 最多为 10。转换函数 f 是为所有 m^3 输入三元组完全指定的，因此它的行为就像一个固定的查找表。 

关键的困难在于每个位置都依赖于三个连续的未知值，这意味着朴素的独立性假设立即被打破。 每个单元格约束与相邻单元格约束严重重叠，形成紧密耦合的循环约束系统。 

天真的尝试可能会尝试从左到右贪婪地分配值。 但这会失败，因为每个选择都会同时影响两个方向的未来可行性，而最终的环绕约束将结束与开始耦合起来。 

当局部一致性在除边界环绕之外的任何地方都保持时，就会出现微妙的边缘情况：

 输入：

 n = 4，m = 2

 s0 = [1, 2, 1, 2]

 贪婪构造可能会找到位置 1 到 3 的局部有效序列，但位置 4 的值无法同时满足索引 3 处的局部规则和索引 4 处的换行约束。 正确的答案是“是”或“否”，取决于是否存在完整的循环分配，而不是部分构造是否成功。 

当存在多个局部选择但只有一个全局循环有效时，会出现另一种失败情况。 局部启发式可能会过早丢弃唯一的全局一致分支。 

## 方法

 我们可以将问题重新解释为在大小为 m 的字母表上搜索赋值 a[1..n]，这样每个索引 i 都会对三重窗口 (a[i−1], a[i], a[i+1]) 施加约束，该窗口必须在 f 下精确计算 s0[i]。 这是具有三元约束的循环上的约束满足问题。 

一种暴力方法会枚举前驱状态的所有 m^n 种可能的分配，并验证每个分配是否产生 s0。 这在概念上很简单：对于每个候选数组，检查 O(n) 中的所有 n 个约束，给出 O(n · m^n)。 m达到10，n达到200，这是一个天文数字，完全不可行。 

一旦我们观察到约束是局部的并且跨度长度仅为 3，该结构就变得易于管理。这表明对连续对进行动态规划。 我们不决定完整的前缀，而是维护前驱数组的最后两个值。 这两个值足以确定有效的扩展，因为位置 i 处的每个约束完全取决于 (a[i−1], a[i], a[i+1])。 

这将问题简化为在传播可行对的同时遍历循环。 然而，圆形边界引入了一个复杂性：前两个选择的值必须与环绕循环的最后一个约束一致。 

我们通过固定前两个值并对剩余位置执行线性 DP 来解决此问题，同时显式强制执行最终的环绕约束。 由于 m 很小，我们可以尝试所有可能的起始对并为每个运行 DP，如果任何配置成功，则提前停止。

这为每个起始对提供了 O(n · m^2) 的可管理状态空间，其中 m^2 开始，这是可以接受的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对完整阵列进行暴力破解 | O(n·m^n) | O(n·m^n) | O(n) | 太慢了|
 | 与固定启动配对 DP | O(m^2 · n · m^2) = O(n · m^4) | O(m^2 · n · m^2) = O(n · m^4) | O(m^2) | 已接受 |

 ## 算法演练

 我们将前驱数组视为未知变量 a[1..n]。 目标是测试至少一个循环分配是否满足所有局部约束。 

1. 修复起始对（a[1], a[2]）。 我们迭代所有 m^2 可能性。 这锚定了 DP，因为以后的每个状态都取决于这些初始值。 
2. 构建一个转换表，告诉我们给定 (a[i−1], a[i]) 和所需的输出 s0[i]，a[i+1] 的哪些值是允许的。 这直接来自于检查所有 m 个候选者并保留那些满足 f(a[i−1], a[i], x) = s0[i] 的候选者。 
3. 在位置 2 到 n−1 上运行 DP。 DP状态为(a[i−1],a[i])。 从每个状态，我们尝试与 s0[i] 兼容的所有可能的 a[i+1]，产生到 (a[i], a[i+1]) 的转换。 这是可行的，因为当我们扩展到位置 i+1 时，每个约束都已完全确定。 
4.到达位置n后，我们并没有立即停止。 我们必须在索引 n 处强制执行最终循环约束，该约束取决于 (a[n−1], a[n], a[1])。 我们检查位置 n 处的任何可达状态是否满足 f(a[n−1], a[n], a[1]) = s0[n]。 
5. 如果任何起始对导致至少一个满足换行约束的有效结束状态，我们可以立即返回 yes。 如果所有起始对都失败，答案是否定的。 

正确性依赖于这样一个事实：DP 枚举前驱数组的所有局部一致的部分赋值，而最终检查强制执行循环引入的唯一非局部约束。 

### 为什么它有效

 在每一步 i，DP 状态 (a[i−1], a[i]) 捕获决定 a[i+1] 合法值所需的所有信息。 任何可以进一步扩展的前驱配置都必须出现在这个状态空间中，因为除了最后两个元素之外，没有较早的值会影响未来的转换。 这使得 DP 完成所有有效的部分分配。 

唯一未在本地处理的约束是索引 n 处的环绕依赖关系，它连接 a[n]、a[n−1] 和 a[1]。 通过在开始处修复 (a[1], a[2]) 并显式验证这个最终约束，我们确保每个循环依赖项都被检查一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    f = [[[0] * m for _ in range(m)] for _ in range(m)]
    vals = []
    for _ in range(m * m * m):
        vals.append(int(input()))
    
    idx = 0
    for x in range(m):
        for y in range(m):
            for z in range(m):
                f[x][y][z] = vals[idx] - 1
                idx += 1
    
    s0 = list(map(int, input().split()))
    s0 = [x - 1 for x in s0]
    
    for a1 in range(m):
        for a2 in range(m):
            dp = [[False] * m for _ in range(m)]
            dp[a1][a2] = True
            
            for i in range(1, n - 1):
                ndp = [[False] * m for _ in range(m)]
                target = s0[i]
                
                for x in range(m):
                    for y in range(m):
                        if not dp[x][y]:
                            continue
                        for z in range(m):
                            if f[x][y][z] == target:
                                ndp[y][z] = True
                
                dp = ndp
            
            for x in range(m):
                for y in range(m):
                    if not dp[x][y]:
                        continue
                    if f[x][y][a1] == s0[n - 1]:
                        print("yes")
                        return
    
    print("no")

if __name__ == "__main__":
    solve()
```该实现将函数 f 编码为由从零开始的值索引的 3D 表。 DP表存储在处理到位置i之后一对(a[i-1],a[i])是否可达。 

每次迭代都会通过尝试与所需输出 s0[i] 一致的所有可能扩展来前进一步。 最后的循环使用原始起始值 a[1] 检查环绕条件。 

一个常见的微妙之处是索引：DP 从位置 2 开始运行，因为前两个值是固定的，并且使用 s0[n] 单独检查最终约束。 

## 工作示例

 考虑一个小实例，其中 n = 4 且 m = 2，具有简单规则 f 和候选 s0 = [1, 2, 1, 2]。 假设我们测试起始对 (a1, a2) = (0, 1)。 

| 步骤| DP 状态对 (x,y) | 行动|
 | --- | --- | --- |
 | 开始 | (0,1)| 初始化|
 | 我=2 | {(y,z) 对 s0[1] 有效} | 使用 f | 扩展
 | 我=3 | 过滤后可达对 | 传播|
 | 决赛| 检查 f(x,y,a1) = s0[3] | 换行约束 |

 此跟踪显示 DP 如何逐渐构建仅本地有效的配置，而不存储完整的数组。 

现在考虑不存在前驱的情况，例如 s0 以与 f 不兼容的方式交替。 DP 在某一层崩溃为空集，并且所有起始对都失败。 

| 步骤| DP尺寸| 结果|
 | --- | --- | --- |
 | 开始 | m^2 状态 | 所有可能的开始|
 | 中| 0 个州 | 矛盾达成|
 | 决赛| 跳过| 没有有效的前任 |

 这表明了对不可行的分支的早期修剪。

 ## 边缘情况

 一种边缘情况是本地存在多个前驱选择，但只有一个关闭循环。 DP 单独保存所有部分对，因此它永远不会太早提交，并且仅在最后验证闭合，从而保持正确性。 

另一个边缘情况是正确的前驱需要一个看起来局部不是最优的特定起始对。 由于尝试了所有 m^2 起始对，因此不排除任何有效配置。 

最后的边缘情况是直接的死胡同，在第一步之后不可能进行扩展。 在这些情况下，DP 表很快就会变空，并且算法会正确拒绝该起始对，而不会影响其他对。
