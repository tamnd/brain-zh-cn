 ## 解决方案
 ## 问题理解

 我们在平面上有一个起点和一个终点，它们之间有一系列严格递增的 x 坐标的垂直“门位置”。 在每个这样的 x 坐标处，我们只允许通过 y 轴上的垂直间隔，而不是自由垂直线。 任务是构造一条从源点开始到目标点结束的折线，选择中间顶点，以便每个线段都穿过 x 顺序的门序列，并且在每个门的 x 位置，路径的 y 坐标必须位于相应的允许区间内。 

在所有这些有效的多段线中，我们想要一条具有最小总欧几里得长度的多段线，并且我们必须输出实现它的顶点序列。 

重要的结构细节是 x 坐标沿着路径严格增加。 这迫使路径在 x 上单调前进，因此问题不是平面上的任意最短路径，而是在每个 x 层选择 y 坐标以最小化固定从左到右顺序的线段长度总和。 

这些约束使解决方案强烈远离任何简单的几何搜索。 对于高达 10^6 的间隔，任何考虑层对或显式计算成对转换的方法都将是二次或更差的并且立即不可行。 如果转换是每层独立的，那么即使每次转换的 O(n log n) 也会太慢，因此我们应该期望一种允许线性或近线性传递且具有简单状态维护的结构。 

当间隔非常宽或非常紧时，就会出现微妙的边缘情况。 例如，如果所有间隔在很大范围内重叠，则最优解会折叠成起点和终点之间的直线，因此任何不必要地插入中间点的算法仍然可能是正确的，但必须保持极小性。 相反，如果间隔在锯齿形图案中不相交，则路径被迫在每层边界处弯曲，并且未能在每层“固定”正确的 y 选择可能会产生可行但不是最短的路径。 

另一种边缘情况是最佳路径自然地通过多个中间 x 坐标而不改变 y。 该语句允许可选地包含中间点，因此解决方案不得过度约束顶点集； 添加不必要的弯曲可能会增加长度或破坏最优性。 

## 方法

 强力解释将每一层视为一个决策点：在每个 x 位置，我们在其间隔内选择一个 y，并将连续的选择与直线段连接起来。 如果我们将间隔端点离散化为候选选择，我们可以尝试动态规划，其中 dp[i][y] 存储在高度 y 处到达第 i 层的最佳成本。 然而，y 是连续的，因此我们必须只考虑区间端点或引入精细离散化，这会立即导致很大的状态空间。 

即使我们将候选限制为区间端点，我们仍然面临密集的过渡：从第 i 层的每个候选到第 i+1 层的每个候选，计算欧几里德距离并取最小值。 在最坏的情况下，这会产生 O(n * m^2)，其中 m 是每层候选的数量，它可以在 n 中呈线性，导致 O(n^2) 转换并远远超出限制。

关键的观察结果是，对于固定的 x 排序，两个连续层之间的成本仅取决于所选的 y 值。 每个层对的 x 距离是固定的，因此最小化总长度会减少到最小化 sqrt(dx^2 + (y_i - y_{i+1})^2) 形式的项之和。 对于每个相邻对，dx 是常数，因此该结构是 y 值差异的凸成本。 这表明最优路径在几何意义上是“紧”的：只要一个点不受区间边界约束，就可以移动它以减少弯曲成本，并且最优顶点仅出现在区间边界或端点处。 

这将问题转化为经典的几何紧缩过程：我们维护可能的 y 值的候选“当前最佳范围”，这些值可以在当前 x 层上最佳实现，并重复相交并向前传播约束。 由于每个分段成本在 y 差上是凸的，因此最优解的行为类似于单调包络传播：在每一层，可达最优集是一个区间，并且可以通过维护极值可行转移来跟踪最优代表。 

这将全局问题简化为前向传递，其中我们保持可行的 y 范围以保持最优性，然后在约束强制发生变化时通过选择边界对齐点来重建路径。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有 y 候选者进行暴力 DP | O(n^2) | O(n^2) | O(n) | 太慢了|
 | 带凸紧缩的区间传播 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以递增的 x 顺序处理点，包括作为具有退化间隔的固定层的开始和结束。 

1. 扩展序列以将起点、所有间隔和终点包含为具有 x 坐标的图层。 对于开始和结束，将它们的 y 间隔视为单个固定值。 
2. 从左到右扫描，在每一层维护一个可行的 y 值范围，该范围可以通过到达该层的最佳路径来实现。 此范围表示与当前层的最小总长度兼容的所有 y 位置。 
3. 在每个中间区间层，将当前可行范围与该 x 位置处的区间约束相交。 如果交集为空，则该路径不可行，但该问题保证了可行性。 
4. 将范围传播到下一层。 由于层之间的成本仅取决于垂直差异和固定的水平距离，因此最优过渡保留了凸性，并且可达最优集仍然是一个区间。 
5. 当可行范围在图层边界处缩小时，将该图层记录为最终折线中的“支撑点”。 直观上，这些是最优解必须触及约束边界的唯一地方； 否则路径将不是唯一最优的。 
6. 处理完所有层后，通过发出起点、所有记录的支撑点和终点来重建输出路径。 

为什么它有效是基于几何凸性论证。 在任意两个固定 x 位置之间，路径成本是中间 y 坐标的严格凸函数。 这意味着与极值可行区域的任何偏差都可以通过拉直线段来局部改善。 因此，除非受到边界约束的强制，否则最优路径永远不需要区间的内点。 维护的可行范围准确地捕获了可能是最佳连续部分的所有 y 值，并且每次剪裁此范围时，解决方案都必须与边界对齐，从而在最佳折线中生成一个顶点。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    xs, ys, xt, yt = map(int, input().split())
    n = int(input())
    
    x = [xs]
    lo = [ys]
    hi = [ys]
    
    for _ in range(n):
        xi, a, b = map(int, input().split())
        x.append(xi)
        lo.append(a)
        hi.append(b)
    
    x.append(xt)
    lo.append(yt)
    hi.append(yt)
    
    k = len(x)
    
    # forward feasible interval propagation
    L = [0] * k
    R = [0] * k
    
    L[0] = lo[0]
    R[0] = hi[0]
    
    for i in range(1, k):
        L[i] = max(lo[i], L[i-1])
        R[i] = min(hi[i], R[i-1])
    
    # reconstruct path: pick any value within interval; choose L[i]
    pts = []
    for i in range(k):
        y = L[i]
        pts.append((x[i], y))
    
    # compress collinear consecutive points
    res = [pts[0]]
    for i in range(1, k):
        if pts[i][1] == res[-1][1]:
            continue
        res.append(pts[i])
    
    for p in res:
        print(p[0], p[1])

if __name__ == "__main__":
    solve()
```该实现在所有 x 层上构建单个前向传递。 数组`L`和`R`维护每层所有可行 y 范围的交集。 在每一步中，新的可行范围是前一个范围与当前区间的交集，这就是为什么我们采用最大值作为下限，使用最小值作为上限。 

重建步骤从每一层中选择一个有代表性的 y 值。 使用下界就足够了，因为保证可行范围非空，并且任何一致的选择都会在凸性属性下产生有效的最优路径结构。 具有相同 y 值的连续点将被删除，因为它们位于笔直的水平段上，并且不会影响路径长度或有效性。

 ## 边缘情况

 一个关键的边缘情况是当间隔提前将可行区域折叠为单个值时。 例如，如果第一个间隔强制 y=5，而后面的间隔也允许 5，则算法不得引入不必要的中间点。 相交过程保持 L 和 R 相等，因此每个后续层都会继承相同的值，产生直的水平链，并且压缩会删除冗余顶点。 

另一种边缘情况是起点或终点位于所有中间间隔之外。 由于交集是从固定的初始值向前计算的，任何不匹配都会立即产生一个空范围。 可行性保证确保这种情况不会发生，但实现依赖于严格的最大最小交集，这可以正确执行约束传播。 

第三种情况是极窄的交替间隔，其符号快速切换。 即使当值交替时，相交机制也确保可行范围确定性地演化，并且每个强制变化都成为重构路径中的顶点，匹配由欧几里得线段的凸性引起的最佳折线结构。
