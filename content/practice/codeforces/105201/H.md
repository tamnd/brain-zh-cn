 ## 解决方案
 ## 问题理解

 我们有一行`n * k`位置，按距入口的距离编号。 每个位置最初包含一只鞋，每个人都拥有`k`鞋。 数组`a`告诉我们当前放置在每个位置的鞋子的所有者。 

一天由两个随机事件组成。 每个人统一选择一双属于自己的鞋子，然后拿走。 随后，人们按照随机顺序返回。 取出的鞋子按照到达顺序放入空位置，首先填充最近的空位置。 

对于每个查询，我们需要在给定天数后属于特定人的所有鞋子的预期平均位置。 答案是精确有理值的模块化表示。 

职位总数最多为`500000`，但查询次数也是`500000`，天数可以大到`10^18`。 模拟无法发挥作用，因为即使一个查询需要很多天，也需要太多的转换。 对于 50 万个查询，我们在预处理后每个查询需要接近恒定的时间，这根据每个查询的天数或鞋子总数排除了任何内容。 

主要的边缘情况来自转换公式。 什么时候`k = 1`，每个人只有一只鞋子，所以每天都会把所有鞋子脱下来，然后重新随机放置。 递归中的乘数变为零，将其视为正常的模除法将会失败。 例如：```
Input:
2 1
1 2
2
1 0
1 1
```人 1 的初始平均位置为`1`。 一天后，有两只鞋子，它们的主人被随机分配到位置`1`和`2`，所以人 1 的预期平均位置是`3/2`。 计算幂的粗心实现`(k-1)/k`无需处理`k = 1`会尝试反转零。 

另一个边缘情况是`d = 0`。 没有发生随机操作，因此答案必须是初始平均值。 例如：```
Input:
2 2
1 1 2 2
1
1 0
```答案是`3/2`，因为人1的两只鞋子已经在位置1和2。直接应用长期公式而不检查`d = 0`可能会意外地用固定值替换初始状态。 

最后一个微妙的情况是仓位是单索引的。 为了：```
Input:
2 2
1 2 1 2
1
1 0
```人 1 拥有位置 1 和 3，因此平均值为`2`。 将第一个槽视为距离零会产生错误的结果。 

## 方法

 直接的做法就是保留鞋子的整体排列，每天进行模拟。 每天，我们会为每个人随机选择一只鞋子，确定空的位置，将取下的鞋子打乱，然后放回去。 在每个模拟日之后，我们可以计算所请求人员的平均位置。 这是正确的，因为它完全遵循所描述的过程。 

问题是天数没有限制。 单个查询可能会询问`10^18`天，所以即使是处理一天的模拟`O(nk)`时间是不可能的。 在最坏的情况下，需要大约`10^18 * 5 * 10^5`运营。 

有用的观察是我们永远不需要知道个别鞋子的位置。 我们只需要属于每个人的预期职位总和。 

假设一个人有预期的鞋子位置总和`S`。 他们的每一位`k`鞋子有概率`1/k`被移除，因此留下的鞋子贡献了预期的总和`(k-1)/k * S`。 

移除的鞋接收空位置之一。 所有人身上，空出的位置正是脱下鞋子的位置。 因为每只鞋都有概率`1/k`被移除后，空仓的预期总和是总仓位总和除以`k`。 总仓位总和为：$$\frac{nk(nk+1)}{2}$$因此空仓的预期总和为：$$\frac{n(nk+1)}{2}$$有`n`被移除的鞋子，并且每只被移除的鞋子同样有可能获得任何空位置。 因此，一个人脱掉鞋子的预期新职位贡献为：$$\frac{nk+1}{2}$$这给出了线性递推：$$S_{t+1}=\frac{k-1}{k}S_t+\frac{nk+1}{2}$$该递归的固定点是无穷多天后的平均位置：$$\frac{nk+1}{2}$$对于平均值而不是总和，相同的公式变为：$$A_d=C+\left(\frac{k-1}{k}\right)^d(A_0-C)$$在哪里`A0`是初始平均位置并且`C`是`(nk+1)/2`。 

整个问题简化为存储每个人的初始平均值并计算每个查询的一个模幂。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(天 * n * k) | O(nk) | O(nk) | 太慢了 |
 | 最佳 | O(nk + q log d) | O(nk + q log d) | O(n) | 已接受 |

 ## 算法演练

 1. 计算初始排列中每个人的位置总和。 将每个总和除以`k`获得初始平均位置`A0`每个人的。 
2. 预先计算递归使用的模常数。 固定值为：$$C=\frac{nk+1}{2}$$乘数为：$$r=\frac{k-1}{k}$$所有除法均以模逆的乘法形式执行。 

1. 对于每个查询`(x, d)`，检索存储的人的初始平均值`x`。 
2.如果`d`为零，则立即输出初始平均值，因为没有发生转换。 
3. 否则计算：$$r^d$$使用二进制求幂。 什么时候`k = 1`,`r`为零，因此每个正天数都会直接达到固定值。 

1. 使用以下方法合并值：$$C+r^d(A_0-C)$$并打印结果模`10^9+7`。 

这样做的原因是每个人的预期平均位置完全由单个值描述。 日常随机选择只影响鞋子是停留还是移动，而期望的线性意味着每只鞋子的预期贡献可以独立求和。 由于重复率仅取决于先前的预期平均值，因此为每个人保留这一值就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10 ** 9 + 7

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    sums = [0] * (n + 1)
    for i, x in enumerate(a, 1):
        sums[x] += i

    inv_k = pow(k, MOD - 2, MOD) if k != 1 else 0
    inv_2 = (MOD + 1) // 2

    if k == 1:
        c = (n + 1) * inv_2 % MOD
        initial = [0] * (n + 1)
        for i in range(1, n + 1):
            initial[i] = sums[i] % MOD
    else:
        c = (n * k + 1) % MOD * inv_2 % MOD
        initial = [0] * (n + 1)
        for i in range(1, n + 1):
            initial[i] = sums[i] % MOD * inv_k % MOD

    if k == 1:
        multiplier = 0
    else:
        multiplier = (k - 1) % MOD * inv_k % MOD

    q = int(input())
    ans = []

    for _ in range(q):
        x, d = input().split()
        x = int(x)
        d = int(d)

        if d == 0:
            ans.append(str(initial[x]))
        elif k == 1:
            ans.append(str(c))
        else:
            power = pow(multiplier, int(d), MOD)
            value = (c + power * (initial[x] - c)) % MOD
            ans.append(str(value))

    print("\n".join(ans))

if __name__ == "__main__":
    solve()
```代码的第一部分存储每个所有者的初始头寸总和。 因为每个主人都有确切的`k`鞋子，除以`k`给出初始平均值。 

递归常数计算一次。 的模逆`k`仅当`k`不是一个，所以特殊情况单独处理。 什么时候`k = 1`，一天足以丢失有关初始放置的所有信息，使答案等于每个正数的固定值`d`。 

对于正常情况，`pow(multiplier, d, MOD)`在对数时间内执行快速求幂。 Python 整数不会溢出，但会减少每次运算的模数`MOD`保持值较小并匹配所需的模运算。

 对于一个小安排：```

 第一天，预期平均值从初始值向固定值移动一半。 这证实了递归不会直接跳至平衡。

 ## 边缘情况

 当`k = 1`，该算法从不尝试反转`k`或无效循环乘数的计算能力。 为了：```
2 1
1 2
2
1 0
1 1
```第一个查询返回初始位置`1`。 第二个查询返回：$$\frac{2+1}{2}=\frac32$$这是`500000005`模数`10^9+7`。 

什么时候`d = 0`，算法完全跳过递归。 为了：```
2 2
1 1 2 2
1
1 0
```存储的总和是`3`，并除以`k = 2`给出`3/2`。 答案仍然是原来的安排，因为没有几天过去。 

对于单索引位置，代码使用以下方式构建总和`enumerate(a, 1)`。 在：```
2 2
1 2 1 2
1
1 0
```人 1 拥有位置 1 和 3，平均产生`(1+3)/2 = 2`。 该实现与物理距离定义相匹配，因为它永远不会将位置移动一位。
