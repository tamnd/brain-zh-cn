 ## 解决方案
 ## 问题理解

 我们得到一个代表一行战场单元的二进制字符串。 每个单元格要么是空的，要么包含一个敌人。 目标是使用两种行动消灭所有敌人。 一道雷击正好消灭了一个选定的敌人。 火球会移除一个选定的敌人，也会移除其直接邻居（如果存在）。 限制是我们在整个过程中最多只能使用一个火球术，而闪电可以使用任意次数。 

任务是最小化消灭所有敌人所需的行动总数。 每个敌人必须直接通过闪电或间接通过火球效果移除。 

关键的困难是决定是否以及在哪里使用单个火球，因为它的效果在很大程度上取决于局部结构。 它可以消除最多三个连续的敌人，但如果它与敌人稀疏的区域重叠，它的放置也可能会造成浪费。 

这些约束允许测试用例的总长度最多为 3·10^5。 这立即意味着任何解决方案对于每个测试用例都必须是线性或接近线性的，因为每个用例的 O(n^2) 扫描在最坏的情况下会退化到大约 10^10 次操作，这是不可行的。 

一个天真的想法是尝试火球的所有可能位置并重新计算剩余的敌人数量。 这将重复扫描字符串，给出 O(n^2) 行为。 另一种幼稚的方法是贪婪地从左到右移除而不考虑火球放置，但这失败了，因为最佳火球位置取决于全局结构，而不仅仅是局部密度。 

一些边缘情况清楚地暴露了结构。 

如果字符串全为零，则答案为零，因为不需要执行任何操作。 

例如，如果字符串全是`11111`，仅闪电策略需要五次操作。 仅当仔细考虑边界时，放置在位置 3 的单个火球才会删除所有五个，但实际上它最多删除三个，还剩下两个，因此最佳答案是 3 个动作，而不是 1 或 2 个。 

如果敌人像这样被孤立`10101`，火球几乎没用，因为没有位置覆盖多个单独的位置； 如果中心放置减少重叠，则最佳仍然是五个闪电或略少，但增益很小。 

这些案例表明，火球术仅在密集的连续片段中有用，并且其最佳效果始终来自于将其应用在单个连续的片段中。 

## 方法

 如果我们忽略火球约束，问题就很简单：每个敌人都需要一个闪电，所以答案就是闪电的数量。 

引入单个火球会改变事情，因为它可以在一次操作中消灭最多三个连续的敌人。 更重要的是，它可以通过降低局部密度来合并连续块中的节省。 

一种强力方法是尝试将 s[i] = '1' 的每个索引 i 作为火球中心。 对于每个这样的选择，我们模拟删除 i 及其邻居，然后计算剩余的。 此重新计算的成本为 O(n)，对所有候选者执行此操作的成本为 O(n^2)。 

关键的观察结果是字符串分解为连续字符串的独立块。 任何火球只能以有意义的方式影响单个块，因为零分隔组件并防止跨块交互。 在长度为 k 的块内，如果没有火球，我们需要 k 个闪电。 使用一个火球，我们最多可以减少 2 个所需的操作，因为最好的可能结果是在一个操作中删除三个连续的火球，而不是三个单独的闪电。 

因此，整个问题简化为找到放置火球可产生最大减少量的连续片段。 在长度为 k 的块中，如果 k ≥ 1，则增益恰好为 1；如果 k ≥ 3，则增益为 2；而 3 是不可能的，因为邻接重叠会阻止每个火球删除超过 3 个不同的火球。 

由于我们在全球范围内只允许发射一个火球，因此我们需要一个可以最大限度地节省闪电的最佳方块。 然而，全局最优策略进一步简化：我们不需要显式选择一个块，我们只需计算总数并减去可能的最佳节省，最多为 2。 

更准确地说，通过找到块内三个连续模式的任何模式来实现最佳节省。 如果存在这样的模式，我们可以节省 2 次操作。 如果不存在这样的模式，但至少存在一对相邻模式，则保存 1。否则，我们无法保存任何内容。 

这将问题减少到扫描一次字符串。

 | ---| ---| ---| ---|
 | 蛮力火球位置| O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 扫描块并计算最佳增益 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们处理字符串并跟踪三个值：总数、是否存在一对连续的以及是否存在三个连续的。 

1. 计算字符串中 1 的总数。 如果我们只使用闪电攻击，这是基准成本。 
2. 从左到右扫描字符串并检查是否有连续的字符串。 每当我们看到`s[i] = s[i-1] = '1'`，我们标记至少可以节省 1 点，因为火球可以比两个单独的闪电更有效地消灭至少两个相邻的敌人。 
3.每当我们看到`s[i] = s[i-1] = s[i-2] = '1'`，我们标记出 2 的更强豁免是可能的，因为单个火球可以代替三个单独的闪电攻击。 
4. 扫描后，确定可实现的最佳保存方式。 如果任何地方存在三元组，则节省为 2。否则，如果存在对，则节省为 1。否则，节省为 0。 
5.输出total_ones减去saving。 

这样做的原因是任何火球总是影响最多三个位置的连续区域，并且唯一有意义的改进来自替换多个连续的闪电。 除了这些情况之外，较长的运行不会提高每个火球的效率，因为单个火球不能覆盖三个以上的位置。 

### 为什么它有效

 该算法依赖于这样一个事实：单个火球的收益完全由局部邻接结构决定。 每个最佳放置都可以简化为连续运行的配置。 在这样的运行中，最佳可能的改进取决于有多少个连续的可以以火球位置为中心。 这将全局优化问题限制为常数因子局部模式搜索，确保这些模式之外的任何配置都不会产生更好的改进。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n = int(input())
    s = input().strip()

    ones = s.count('1')

    best = 0

    for i in range(n):
        if s[i] == '1':
            if i > 0 and s[i-1] == '1':
                best = max(best, 1)
            if i > 1 and s[i-1] == '1' and s[i-2] == '1':
                best = 2

    print(ones - best)
```该解决方案首先计算所有敌人，因为每个敌人通常都需要闪电攻击。 变量`best`跟踪使用单个火球我们可以降低多少成本。 

在扫描过程中，我们检测邻接模式。 一对连续的保证至少一个火球可以通过用一个火球加上其他地方剩余的闪电替换两个闪电来节省一次操作。 三个 1 可以保证最多节省 2 个，因为三个相邻的敌人可以被一个火球代替，而不是三个单独的闪电。 

该实现使用简单的从左到右扫描，一旦找到三元组，我们就可以安全地设置`best = 2`因为没有任何配置可以超越这个提升。 

在边界处需要小心，因为索引 i-2 只能在有效时访问。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        s = input().strip()

        ones = s.count('1')
        best = 0

        for i in range(n):
            if s[i] == '1':
                if i > 0 and s[i-1] == '1':
                    best = max(best, 1)
                if i > 1 and s[i-1] == '1' and s[i-2] == '1':
                    best = 2

        out.append(str(ones - best))

    return "\n".join(out)

# edge: no enemies
assert run("1\n5\n00000\n") == "0"

# edge: all ones small
assert run("1\n3\n111\n") == "1"

# edge: separated ones
assert run("1\n5\n10101\n") == "5"

# edge: mixed blocks
assert run("1\n6\n110111\n") == "2"
```| 测试输入| 预期产出 | 它验证了什么 |
 | ---| ---| ---|
 | 00000 | 0 | 无需任何操作|
 | 111 | 111 1 | 火球术的最大好处|
 | 10101 | 10101 5 | 没有邻接优势|
 | 110111 | 2 | 混合块三重节省|

 ## 边缘情况

 对于像这样的字符串`00000`，该算法立即计算零并且从不触发任何邻接条件。 输出正确为零，因为无法保存。 

为了`111`，扫描同时检测到一对和三元组。 在索引 2 处，三重条件最好设置为 2，导致答案 1，这对应于用单个火球替换三个闪电，并且该部分中没有剩余的敌人。 

为了`10101`，不存在相邻的，因此最好保持为零。 每个敌人都是孤立的，每个敌人都需要单独的闪电。 

为了`110111`，第一个块给出一对，第二个块给出一个三元组，因此最佳变为 2。该算法仅在提供最大压缩的情况下正确应用火球，忽略块是单独的，因为只有一个火球可用。
