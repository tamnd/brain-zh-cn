 ## 解决方案
 ## 问题理解

 我们正在维护一个从空开始并通过一系列边缘插入不断演变的图。 每个插入的边都有两种颜色之一：红色或蓝色，但图的底层连接性会忽略颜色。 每次插入后，我们必须计算在同时满足两个条件的情况下可以删除多少条边。 

首先，剩余的图必须准确地保留连接性，这意味着在删除之前连接的任何顶点对在之后必须仍然保持连接。 在图方面，我们只允许删除不需要的边来保留当前的连接组件。 这立即表明删除与循环有关，因为生成森林内的树边缘是连接严格必需的唯一边缘。 

其次，存在全局平衡约束：移除的红色边缘的数量必须等于移除的蓝色边缘的数量。 这将两个颜色类别结合在一起，并防止我们简单地单独删除每种颜色的所有非桥边。 

每次查询后的输出是在两个约束下可以删除的最大边数。 

约束最多为 3·10^5 个顶点和 3·10^5 个查询，因此任何在每次更新后从头开始重新计算连通性或重新计算生成林的方法都会立即变得太慢。 即使每个查询使用 DFS 或 DSU 重新计算组件也会导致 O(nq)，这远远不可行。 我们需要一个增量结构，以对数或接近恒定的摊销时间更新连接和循环结构。 

微妙的边缘情况来自自循环和多边。 自循环永远无法帮助连接，因此它始终是可删除的，但它仍然会导致颜色不平衡。 类似地，已连接组件之间的多条边是冗余的，但会影响红蓝差异约束。 仅跟踪组件而不计算冗余边缘的简单方法将会失败。 

## 方法

 核心困难在于，我们不仅要计算冗余边缘，还要计算按颜色分割的冗余边缘，同时仍然尊重全局连接性。 

一个蛮力的想法是在每次查询后维护完整的图并重新计算忽略颜色的最大跨度森林。 对于固定图，我们可以运行 DSU 或 DFS 来选择生成林，然后计算其外部的边数。 这些边正是连接性方面可删除的边。 然而，这忽略了删除必须平衡红色和蓝色计数的要求。 

要合并颜色，我们需要考虑所有冗余边，然后选择其中的子集进行删除，使得红色删除的数量等于蓝色删除的数量。 如果我们让 R 为冗余红边，B 为冗余蓝边，那么我们最多可以删除 min(R, B) 对，贡献 2·min(R, B)，加上这些对之外的结构上强制删除的所有边。 关键是冗余纯粹是由连接性定义的：如果一条边在到达时其端点已经连接，则该边是冗余的。 

这导致了中心观察。 如果我们使用 DSU 按插入顺序处理边，则每次看到一条边时，我们都可以检查它是否连接两个不同的组件。 如果是，则它是生成林的一部分，无法删除。 如果不是，则它是多余的，并且根据其颜色对 R 或 B 做出贡献。 连接结构本身从不依赖于颜色。 

因此，我们将问题简化为在 DSU 下随着时间的推移维护两个冗余边计数，并在每次查询后使用简单的算术公式进行回答。

暴力破解会失败，因为每个查询重新计算连接的时间为每步 O(n + q)。 基于 DSU 的增量方法以摊余逆阿克曼时间处理每条边，使整个过程呈线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 重新计算每个查询的连接 | O(q·n) | O(n) | 太慢了 |
 | DSU增量计数| O((n+q) α(n)) | O((n+q) α(n)) | O(n) | 已接受 |

 ## 算法演练

 我们在顶点和两个计数器上维护一个不相交的集合并结构，一个用于冗余的红色边缘，一个用于冗余的蓝色边缘。 我们还维护到目前为止看到的边总数。 

1. 初始化 DSU，其中每个顶点都是其自己的父顶点，并将两个冗余计数器设置为零。 如果需要清晰起见，我们还会保留一个全局边缘计数器。 
2. 对于每个查询，读取边 (u, v) 及其颜色。 
3. 检查u 和v 是否属于同一个DSU 组件。 这是使用查找操作完成的。 DSU 代表当前的跨连接森林。 
4. 如果u和v在不同的组件中，我们将它们合并到DSU中。 该边对于连通性是必需的，因此在不更改连通性的情况下无法将其删除。 
5. 如果 u 和 v 已经在同一个组件中，则该边在连接性方面是冗余的。 我们增加与其颜色（红色或蓝色）相对应的冗余计数器。 这是因为该边缘位于环上。 
6. 处理完边后，计算答案。 所有冗余边都是删除的候选边，但我们必须删除相同数量的红色和蓝色边。 如果 R 是冗余红色边缘的数量，B 是冗余蓝色边缘的数量，那么我们可以将每种颜色的 min(R, B) 条边缘配对进行删除。 每对贡献两个缺失，因此平衡缺失贡献 2·min(R, B)。 不能使用剩余的未配对冗余边，因为它们会违反等式约束。 
7. 输出值2·min(R,B)。 

它之所以有效的原因来自于连接约束的结构。 DSU 维护当前图的生成森林，因此不在森林中的每条边都必然是循环的一部分。 删除循环边的任何子集不会改变连通性，并且每条冗余边都位于当前图中的至少一个循环上，该循环由较早的森林边增强。 由于森林边缘永远不会被移除，因此连接结构得以保留。 因此，唯一的自由在于选择冗余边缘，唯一的全局限制是跨颜色的匹配计数。 DSU 确保我们始终将边正确分类为必要边和冗余边，这使得计数稳定且与顺序无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        ra = self.find(a)
        rb = self.find(b)
        if ra == rb:
            return False
        if self.r[ra] < self.r[rb]:
            ra, rb = rb, ra
        self.p[rb] = ra
        if self.r[ra] == self.r[rb]:
            self.r[ra] += 1
        return True

def solve():
    n, q = map(int, input().split())
    dsu = DSU(n + 1)

    red_redundant = 0
    blue_redundant = 0

    for _ in range(q):
        t, u, v = map(int, input().split())

        if not dsu.union(u, v):
            if t == 1:
                red_redundant += 1
            else:
                blue_redundant += 1

        ans = 2 * min(red_redundant, blue_redundant)
        print(ans)

if __name__ == "__main__":
    solve()
```DSU 纯粹用于决定边缘是否有助于连接。 当端点已连接时，并集操作会返回 False，这会识别冗余边。 然后我们按颜色将它们分开。 

一个常见的微妙之处是，我们永远不需要跟踪哪些确切的边形成循环，只需要跟踪它们的计数。 另一个微妙的点是，我们永远不会减少任何东西：即使图在概念上不断发展，边缘也永远不会被删除，因此 DSU 随着时间的推移仍然有效。

 ## 边缘情况

 一个关键的边缘情况是自循环。 自环永远不会改变连接性，因此它总是立即被归类为冗余。 例如，输入`1 1 1`使红色冗余计数器增加 1，但答案保持为零，直到出现匹配的蓝色冗余边缘。 DSU 正确处理了这个问题，因为 find(1) 等于 find(1)，从而触发了冗余分支。 

另一种情况是已经连接的顶点之间的重复边。 如果顶点 1 和 2 通过链连接，则无论颜色如何，任何附加边 1-2 都是冗余的。 DSU 保证 union(1,2) 返回 false，因此所有此类边都会累积到正确的计数器中。 

最后一个微妙的情况是早期不平衡，例如许多红色冗余边缘出现在任何蓝色边缘之前。 公式 2·min(R, B) 确保不会过早地计数删除，因为在两侧都可用之前，不匹配的边无法对平衡删除集做出贡献。
