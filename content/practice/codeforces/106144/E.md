 ## 解决方案
 ## 问题理解

 我们得到一个代表一条有障碍物的道路的网格，其中每个单元格要么是空的，要么是被阻挡的。 固定高度 k 的垂直汽车最初位于第一列，占据从 1 到 k 的行。 汽车永远不会旋转，因此在任何时刻它总是占据单列中长度为 k 的连续垂直段。 

一次移动，小车可以向左或向右移动一列，或者向下移动一排。 仅当移动后汽车占据的每个单元都没有障碍物时才允许移动。 任务是确定汽车可以到达网格下方多远，这意味着在任何有效的移动序列期间汽车的任何位置覆盖的最大行索引。 

状态空间本质上是网格内 k 长度垂直段的所有有效放置，以及相邻列之间的过渡或向下移动。 困难在于，水平移动可以用来“绕行”被阻塞的单元格，而列之间的这种相互作用阻碍了简单的逐行模拟。 

这些约束意味着所有测试用例的总网格大小最多为 10^6 个单元格，因此任何解决方案都必须仅处理每个单元格恒定的次数。 任何接近 O(nm log nm) 或更差的值都有超时的风险。 这强烈表明采用摊销处理的线性或近线性扫描。 

当需要水平移动绕过垂直障碍链时，会出现微妙的故障情况。 

例如，考虑 k = 2：```
....
.xx.
.xx.
....
```仅跟踪每列垂直可行性的简单方法会得出第二列阻碍第 2 行前进的结论。但是，通过在遇到障碍物对齐之前转移到第一列或第三列，汽车可以继续向下行驶。 这表明可达性取决于跨列的全局连接，而不是独立的列检查。 

当障碍物形成“交错墙”时，会出现另一种边缘情况：```
....
.x..
..x.
....
```局部看来，每根柱子都可以单独通过，但汽车必须协调水平运动，以避免与移动的障碍物重叠。 任何仅独立检查 k 长度垂直窗口的方法在这里都会失败。 

## 方法

 一个蛮力的想法是将每个有效的汽车放置视为图中的一个节点。 从由左上角坐标 (x, y) 定义的状态开始，如果 k 个单元格空闲，我们可以尝试向上、向左和向右转换。 这成为 O(nm) 状态的可达性问题。 每次状态检查需要扫描k个单元，因此最坏情况复杂度变为O(nm·k)。 在极端情况下 k 高达 10^6，这显然是不可行的。 

关键的观察是我们永远不需要显式枚举所有汽车位置。 汽车总是占据一个垂直段，因此在任何行 x 我们只关心哪些列可以通过终止于 x + k − 1 的高度为 k 的有效垂直窗口到达。一旦列在某个高度变得可到达，它就可以在未来的行中重复使用，除非被滑动窗口内的障碍物阻挡。 

这建议为每一行维护 k 高度窗口有效的列集。 剩下的问题是如何在不经常重新访问单元格的情况下水平传播可达性。 该结构成为分层可达性问题，其中每一行仅依赖于前 k 行。 

我们预先计算每个单元格结束的垂直段是否无障碍。 然后我们将每一行视为列的图形层。 水平转换可以在行内进行，垂直转换从行 i 移动到 i + 1，但仅限于有效窗口的列内。 

这将问题简化为在大小为 n × m 的网格上类似 BFS 的传播，其中每个单元最多被处理一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力状态图 | O(nmk) | O(纳米) | 太慢了 |
 | 有效窗口上的分层 BFS | O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们将问题转换为由代表汽车有效顶部位置的网格单元定义的状态的可达性。 

1. 对于每个单元格 (i, j)，计算汽车是否可以将底部停在第 i 行第 j 列处。 如果从 (i − k + 1, j) 到 (i, j) 的所有单元格均为空，则情况成立。 这可以使用每列中障碍物的前缀和来检查。 
2. 构建布尔网格`valid[i][j]`指示在第 i 行结束的垂直放置是否可以在第 j 列。 
3. 我们想要找到从第 k 行第 1 列开始的所有可达状态。如果该状态有效，则使用该状态初始化 BFS 或队列。 
4. 使用队列探索状态。 如果有效且尚未访问，则尝试从每个状态 (i, j) 水平移动到 (i, j − 1) 和 (i, j + 1)。 如果在新行有效，还尝试向下移动到 (i + 1, j)。 
5. 维护一个包含 (i, j) 状态的已访问数组，以便每个状态都被处理一次。 
6. 跟踪所有访问状态中 i 达到的最大行索引。 

BFS 确保我们探索所有可到达的位置而无需重新计算。 

工作原理：每个有效的汽车配置都对应于一个状态（行、列），其中汽车的底部边缘位于该行。 任何有效的移动序列都对应于该状态图中的一条路径，并且每条边都准确对应于原始问题中的合法移动。 由于 BFS 探索了这些转换下所有可到达的状态，因此它捕获了所有可能的方法来推进汽车。 因此，任何访问状态达到的最大行数就是正确答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    t = int(input())
    for _ in range(t):
        n, m, k = map(int, input().split())
        grid = [input().strip() for _ in range(n)]

        # prefix sum per column: obstacles
        pref = [[0] * m for _ in range(n + 1)]
        for i in range(n):
            row = grid[i]
            for j in range(m):
                pref[i + 1][j] = pref[i][j] + (1 if row[j] == 'x' else 0)

        def ok(i, j):
            # check window [i-k+1, i] in column j
            if i < k - 1:
                return False
            return pref[i + 1][j] - pref[i - k + 1][j] == 0

        # visited states (i, j)
        vis = [[False] * m for _ in range(n)]
        dq = deque()

        start_row = k - 1
        if ok(start_row, 0):
            dq.append((start_row, 0))
            vis[start_row][0] = True

        ans = 0

        while dq:
            i, j = dq.popleft()
            ans = max(ans, i)

            # left/right
            if j > 0 and not vis[i][j - 1] and ok(i, j - 1):
                vis[i][j - 1] = True
                dq.append((i, j - 1))

            if j + 1 < m and not vis[i][j + 1] and ok(i, j + 1):
                vis[i][j + 1] = True
                dq.append((i, j + 1))

            # move down
            if i + 1 < n and not vis[i + 1][j] and ok(i + 1, j):
                vis[i + 1][j] = True
                dq.append((i + 1, j))

        print(ans + 1)

if __name__ == "__main__":
    solve()
```该代码首先构建按列的前缀和，以便检查垂直线段是否包含障碍物的时间复杂度为 O(1)。 功能`ok(i, j)`验证汽车是否可以占据第 i 行结束的第 j 列。 

BFS 从第 k − 1 行开始，该行对应于初始放置结束行。 水平转换保持在同一行索引上，而向下转换则增加行索引。 答案跟踪访问的最大底行索引，最后转换回基于 1 的索引。 

主要的微妙之处在于索引：BFS 状态代表汽车的底部，因此第 i 行对应于输出方面的实际网格行 i + 1。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import deque

    def solve():
        t = int(input())
        for _ in range(t):
            n, m, k = map(int, input().split())
            grid = [input().strip() for _ in range(n)]

            pref = [[0] * m for _ in range(n + 1)]
            for i in range(n):
                for j in range(m):
                    pref[i + 1][j] = pref[i][j] + (grid[i][j] == 'x')

            def ok(i, j):
                if i < k - 1:
                    return False
                return pref[i + 1][j] - pref[i - k + 1][j] == 0

            vis = [[False] * m for _ in range(n)]
            dq = deque()

            start = k - 1
            if ok(start, 0):
                dq.append((start, 0))
                vis[start][0] = True

            ans = 0
            while dq:
                i, j = dq.popleft()
                ans = max(ans, i)
                for nj in (j - 1, j + 1):
                    if 0 <= nj < m and not vis[i][nj] and ok(i, nj):
                        vis[i][nj] = True
                        dq.append((i, nj))
                ni = i + 1
                if ni < n and not vis[ni][j] and ok(ni, j):
                    vis[ni][j] = True
                    dq.append((ni, j))

            return str(ans + 1)

    return solve()

 ## 边缘情况

 一个关键的边缘情况是初始列对于汽车的整个高度不可行。 在这种情况下，不会将任何 BFS 状态排入队列，并且答案正确地保持为 0，因为即使最初也无法放置汽车。 

另一种情况是，只有在较高行水平移动后，列才有效。 BFS 自然地捕捉到了这一点，因为水平移动是在每一层的垂直移动之前或同时探索的。 

最后一种情况是障碍物形成交替图案，需要在每一行重复进行左右调整。 由于每个（行、列）状态都被访问一次，因此算法不会无限振荡，并且所有此类模式都可以安全地探索而无需重复。
