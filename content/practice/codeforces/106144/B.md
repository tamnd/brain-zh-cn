 ## 解决方案
 ## 问题理解

 我们给出了平面上的一系列点，并且这些点的顺序是固定的。 从这个序列中，我们要选择一个连续的线段，比如从索引 l 到 r，并检查按顺序访问这些点形成的多边形在按该顺序闭合时是否是严格凸多边形。 

仅当线段包含至少三个点时才有效。 对于一个线段来说，其内每三个连续的点必须转向相同的方向，并且不允许三个连续的点位于同一条直线上。 换句话说，每个内部转弯必须严格一致且非零，因此多边形永远不会局部“变平”。 

每个测试用例的输出是此类连续凸段的最大可能长度。 如果没有长度至少为 3 的线段满足条件，则答案为零。 

这些约束迫使我们在每个测试用例的线性或近线性时间内进行思考。 所有测试用例的总点数最多为 2 · 10^5，因此每个测试用例具有二次行为的任何解决方案都将立即失败。 这已经排除了显式检查所有 O(n^2) 间隔的可能性，因为即使是简单的凸性检查也是 O(n) ，导致最坏情况为 O(n^3) 。 

一个关键的微妙之处在于这些点是按固定顺序给出的。 我们不会对点重新排序以形成凸包，我们只是检查给定顺序的连续子序列。 

简单的解决方案可能会忽略一些失败案例：

 由于单个共线三元组，线段可能无法凸性。 例如，点(0,0)、(1,1)、(2,2)、(3,0)大多是“形状形成”，但前三个点是共线的，因此任何包含它们的线段都是无效的，即使其余部分看起来是凸的。 

另一个问题是凸性是有方向的。 即使没有三个点共线，左右交替转动的序列也是无效的。 

最后，必须显式检查长度为 3 的短线段，因为它们代表一个三角形，如果不共线，则该三角形始终为凸三角形。 粗心的实现可能会跳过它们或错误地处理方向一致性。 

## 方法

 蛮力的想法很简单。 对于每个区间 [l, r]，我们测试由点 l 到 r 形成的多边形是否按顺序严格凸。 对于每个间隔，我们计算所有三元组 (i, i+1, i+2) 的方向并确保它们全部非零且相同。 这需要每个时间间隔进行 O(r − l) 工作，并且有 O(n^2) 个时间间隔，在最坏的情况下总计为 O(n^3)，这对于 n 达到 2 · 10^5 来说太慢了。 

为了改善这一点，我们需要避免从头开始为每个区间重新计算凸性。 关键的观察结果是，凸性是连续三元组的局部属性：它仅取决于向量 (p[i+1] − p[i]) × (p[i+2] − p[i+1]) 叉积的符号。 如果我们固定一个起始点l，那么当我们向右延伸r时，我们只需要保证所有新的转弯与初始方向保持一致即可。 

这将问题转化为寻找最长的连续子数组，其中导出的方向符号序列是恒定且非零的。 一旦我们将点转换为转弯标志，任务就减少为找到所有连续标志都相等且已定义的最长线段。 

我们预先计算每个三元组的方向，生成一个长度为 n − 2 的数组。然后我们在这个由相同的非零值组成的派生数组中搜索最长的连续段。 每个这样的段对应于原始点序列中的最大凸区间，移动两个索引。

 | 蛮力 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

1. 对于每个测试用例，按顺序阅读点列表。 顺序是固定的，不得更改。 
2. 计算每个连续三重点的方向。 对于索引 i、i+1、i+2，计算向量 (p[i+1] − p[i]) 和 (p[i+2] − p[i+1]) 的叉积。 该值表明转动是顺时针、逆时针还是共线。 
3. 如果叉积为零，则将该位置标记为无效，因为严格凸性禁止三个连续共线点。 这立即打破了通过这个三元组的任何间隔。 
4. 将叉积序列转换为符号数组，其中每个元素为+1或-1表示有效回合，0表示无效回合。 
5. 扫描该数组并计算仅包含相同非零符号的最长连续段。 这是通过维持当前条纹的运行长度并在符号改变或变为零时重置来完成的。 
6. 如果方向数组中的最佳条纹长度为 k，则原始点数组中相应的凸区间的长度为 k + 2。跟踪所有此类条纹的最大值。 
7. 如果不存在有效条纹，则答案为 0。 

### 为什么它有效

 由有序点定义的多边形的凸性相当于要求所有连续的三元组具有相同的方向符号并且没有简并性。 叉积捕获局部转弯方向，并且该符号的一致性确保多边形不会在左转弯和右转弯之间切换。 任何零交叉乘积都会立即违反严格的凸性。 由于每个区间的有效性仅取决于相邻的三元组，因此将问题压缩到符号数组中可以保留所有必要的信息。 因此，最长的有效间隔恰好是映射回点索引的相同非零方向符号的最长运行。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        p = [tuple(map(int, input().split())) for _ in range(n)]
        
        if n < 3:
            print(0)
            continue
        
        def cross(a, b, c):
            # (b-a) x (c-b)
            return (b[0] - a[0]) * (c[1] - b[1]) - (b[1] - a[1]) * (c[0] - b[0])
        
        best = 0
        cur_len = 0
        cur_sign = 0
        
        for i in range(n - 2):
            val = cross(p[i], p[i+1], p[i+2])
            if val == 0:
                cur_len = 0
                cur_sign = 0
                continue
            
            s = 1 if val > 0 else -1
            
            if s == cur_sign:
                cur_len += 1
            else:
                cur_sign = s
                cur_len = 1
            
            best = max(best, cur_len)
        
        # cur_len is length in triple-array, convert to points
        print(best + 2 if best > 0 else 0)

if __name__ == "__main__":
    solve()
```实现的核心是连续三元组的叉积。 它是直接根据坐标计算的，无需显式构建向量。 扫描保持相同方向符号的当前条纹。 每当出现零交叉积时，条纹就会被重置，因为包含该三元组的任何区间都不能是凸的。 

最终答案加 2，因为连续 k 个有效三元组对应于原始序列中的 k + 2 个点。

 ## 边缘情况

 序列中任何位置的共线三元组都会立即打破通过它的任何凸区间。 对于 (0,0)、(1,1)、(2,2)、(3,3) 等输入，每个三元组的叉积为零，因此算法在每一步都会重置，并且永远不会构建有效的条纹，从而产生输出 0。 

原本是凸的但包含单个平转的序列也会失败。 例如，除了一个直线段之外，形成凸链的点会将方向条纹分成两部分，并且算法正确地仅采用较长的有效边。 

大小为 3 的最小有效凸区间是自然处理的，因为第一个非零叉积开始了长度为 1 的条纹，在加 2 后映射回 3 个点。 

左转和右转的交替顺序可以防止任何超过 1 的条纹，因此仅当存在单个一致的三元组时，答案才变为 3； 否则为 0。
