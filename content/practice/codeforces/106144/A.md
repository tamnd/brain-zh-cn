 ## 解决方案
 ## 问题理解

 我们给出了一个可以在两个非常具体的删除规则下缩小的数组。 第一条规则允许我们删除数组中当前存在的单个最小值，并且当多个位置包含该最小值时，我们可以自由选择删除哪个位置。 第二条规则更加结构化：如果数组当前以两个相等的值开头，我们可以立即删除它们。 

目标是使用尽可能少的操作完全擦除数组。 每个操作都会减少长度，但效果是不对称的：一个规则恰好删除从全局最小集中选择的一个元素，而另一个规则则删除前面的固定相邻对，但前提是这两个元素匹配。 

所有测试用例的总大小最多为 2·10^5 的约束意味着我们每个测试套件需要一个接近线性或线性算术的解决方案。 任何使用嵌套扫描重复模拟动态数组删除的方法都会太慢，特别是因为在简单循环中重复“查找最小值”或“检查前缀相等”操作会降低到 O(n^2)。 

一个微妙的问题出现在前缀操作如何与最小元素的删除交互中。 删除最小值可以暴露之前被该最小值分隔开的新的相邻相等对。 如果忽略提前删除某些最小值可能会解锁多个未来对删除，那么总是尽可能应用前缀操作的天真贪婪策略可能会失败。 

例如，考虑像 [2, 1, 1, 2] 这样的数组。 如果我们急切地从中间删除最小的 1，我们可能会破坏以后两次应用配对删除的机会。 最佳策略取决于保留结构，以便对可以级联。 

另一种边缘情况是当数组已经完全由相邻的相等对组成时，例如 [3, 3, 2, 2, 1, 1]。 在这里，最佳播放完全避免了不必要的单个删除，并且任何优先考虑最小删除而不考虑配对潜力的策略都会过度计算操作。 

## 方法

 暴力策略将模拟每个状态下的所有有效操作：在每个步骤中，尝试删除所有可能位置中的最小元素，或者如果前缀允许，则删除前两个元素。 这形成了数组状态的搜索树。 即使使用记忆化，不同状态的数量也可能会爆炸，因为删除相同最小值的不同出现会导致未来不同的邻接结构。 在最坏的情况下，每次删除都会分支为多个下一个配置，从而产生指数行为。 

关键的观察是第二个操作纯粹是结构性的：仅当数组以两个相等的元素开头时，它才会删除一对，并且总是立即消耗这两个元素。 这意味着只要我们正确对齐边界，相等元素的任何最大前缀都会表现得像一堆强制归约。 

第一个操作是唯一与排序进行全局交互的操作。 然而，它也受到限制：它总是以当前最小值为目标。 这表明大于最小值的值永远不会被直接删除，除非通过删除较小的元素来暴露。 

重要的结构性转变是停止根据立场思考，而是根据按大小排序的价值观思考。 每次我们决定删除最小元素时，我们实际上是在选择是否“破坏”未来潜在的对减少或保留支持批量删除的结构。

这导致了贪婪压缩的观点：如果我们按升序处理值，我们可以推断每个值必须单独删除多少次，以及通过相邻配对级联可以消除多少次。 数组自然地分解成段，一旦删除中间的较小元素，相等的值就会成对消失。 

最佳解决方案简化为在重复模拟删除所有较小值的效果后跟踪有多少“活动块”幸存下来，并计算需要删除多少次才能解决剩余的不匹配元素。 

形式化这一点的一种方便方法是扫描值并维护一个结构，该结构跟踪配对从左侧崩溃后剩余的每个值的元素数量，一旦较小的阻塞消失，将第二个操作视为相等相邻段的贪婪堆栈取消。

 ## 算法演练

 我们将这个过程重新解释为维护数组的简化表示，其中相邻的相等元素在删除较小的值后变得相邻时可以成对取消。 

我们从左到右处理数组，同时在应用所有可能的对删除后维护代表当前有效前缀的值堆栈。 

1. 初始化一个空堆栈，该堆栈将代表所有可能的前缀对删除后的当前缩减数组。 
2. 从左到右遍历数组中的每个元素。 对于每个元素，尝试将其合并到当前结构中。 
3. 如果栈非空且栈顶等于当前元素，则移除栈顶而不是压入当前元素。 这对应于隐式应用第二个操作，因为两个相邻的相等元素一旦在有效状态下变得连续就会抵消。 
4. 如果栈顶不同，则压入当前元素。 这代表当前无法配对的边界。 
5. 处理整个数组后，堆栈中包含无法单独通过对取消删除的元素。 每个剩余元素对应于在过程的某个阶段强制使用第一个操作。 
6. 答案是最终堆栈的大小，因为每个剩余的不匹配元素都必须在某个时刻作为最小元素单独删除，而所有可取消的结构已经被消除。 

这样做的原因是，第二个操作的每次有效使用都恰好对应于在完全缩减状态下删除一对相等的相邻元素。 通过贪婪地模拟所有此类取消，我们确保不存在人为障碍。 剩下的正是在任何有效操作顺序下都无法配对的元素，并且每个这样的元素都会在任何最佳策略中强制执行一个最小删除事件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    st = []
    for x in a:
        if st and st[-1] == x:
            st.pop()
        else:
            st.append(x)
    
    print(len(st))

t = int(input())
for _ in range(t):
    solve()
```该解决方案维护一个立即折叠相邻相等元素的堆栈。 弹出操作完全对应于以最激进的形式应用第二个允许的操作，因为任何时候两个相等的值在有效序列中变得相邻时，都可以将它们删除而不影响未来的可行性。 

打印最终堆栈大小是因为它表示所有可能的对消除后的不可约余数。 每个剩余元素在结构上都是隔离的，并且在允许的操作下不能与任何邻居配对，从而强制使用第一个操作进行单独的删除步骤。 

一个微妙的点是我们从来没有明确地模拟“最小删除”。 该操作被吸收到不变量中：任何无法通过配对移除的东西最终都必须被一一移除，并且堆栈准确地捕获该残留物。

 ## 边缘情况

 一种边缘情况是单元素数组，例如 [7]。 栈一开始为空，元素被压入，最终栈大小为1，正确反映了需要删除一次。 

另一个边缘情况是完全对称的配对结构，如 [4, 4, 3, 3, 2, 2]。 堆栈在每一步都会重复取消对，最终变空。 这表明该算法可以正确处理链式取消，而不需要任何显式的全局推理。 

第三种情况是取消取决于中间结构，例如 [1, 2, 2, 1]。 第一个 2 被推送，第二个 2 取消它，然后 1 和 1 取消。 堆栈模型正确地捕获了这种非本地交互，因为邻接始终是在当前缩减的前缀上评估的，而不是在原始数组上评估的。
