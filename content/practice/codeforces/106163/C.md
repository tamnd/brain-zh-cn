 ## 解决方案
 ## 问题理解

 我们有一棵树$n$节点，对于每个节点，我们知道它在树中的度。 然后每度减一，所以每个值$d[i]$现在代表$\deg(i) - 1$。 计算这些值后不再需要直接使用树结构； 一切都归结为这个整数数组。 

对于任何固定的子集大小$i$，我们考虑精确的所有子集$i$节点并查看它们修改度的总和。 一个值$p$被称为$i$- 如果存在某些尺寸子集，则很有趣$i$其度数总和等于$p$。 对于每个查询$(x, y)$，我们必须计算有多少个整数$x$- 有趣或$y$-有趣的。 

约束条件很大：最多$2 \cdot 10^5$节点和查询。 这立即排除了任何枚举子集甚至独立计算每个大小的所有子集总和的方法。 甚至像$O(n^2)$每个查询都是不可能的，因为这已经超过了$10^{10}$运营。 

一个微妙的点是度数来自树并且全部移动$-1$。 这意味着所有的总和$d[i]$是固定的：$$\sum d[i] = \sum (\deg(i)-1) = 2(n-1) - n = n-2.$$这种全局约束是取代考虑树本身的需要的关键结构。 

经常打破朴素推理的边缘情况涉及小树和极端分布。 例如，当$n=1$，我们有一个度数为 0 的节点，所以$d[1] = -1$。 子集和的行为很奇怪，因为允许负值，并且许多天真的子集和假设都被打破。 另一个角是星形树：一个节点的度数较大，所有其他节点的度数为 1，因此移位后大多数值变为 0，而一个值变大。 在这种情况下，许多子集和会收缩到一个小范围内，如果假设一般的子集和行为，则很容易低估这个范围。 

## 方法

 如果我们忽略效率，最直接的解释就是计算，对于每一个$k$，大小的所有子集总和$k$。 这是一个经典的组合爆炸：对于每个$k$， 有$\binom{n}{k}$子集，并对每个子集成本求和$O(k)$，所以总数大约是$$\sum_{k=0}^{n} \binom{n}{k} \cdot k = O(n2^n),$$除了微小之外这是完全不可行的$n$。 

即使我们尝试对子集大小进行动态规划，例如背包式 DP，其中我们为每个基数维护可达总和，我们很快就会达到以下状态空间：$O(n^2)$或更糟糕的是，每次转换都会改变所有总和，再次导致二次或三次行为。 

关键的结构观察是，我们不要求精确的子集，而只要求对于固定的子集大小是否可以实现总和。 这将问题变成了“可能的总和范围”问题，而不是计算子集。 关键的简化来自于认识到对于任何固定的$k$，可实现的总和的集合是连续的。 发生这种情况是因为这些值是从按常数移位的树度序列导出的，这意味着强大的交换属性：将子集中的一个节点与另一个节点交换会以可预测的增量更改总和，而不会在可实现的值中产生间隙。 

一旦我们知道对于每个$k$，所有可达到的总和形成一个区间$[L_k, R_k]$，每个查询的问题就变得微不足道：我们只需要计算两个区间的并集中有多少个整数，或者如果它们重叠则有一个区间。 

剩下的任务是有效地计算这些端点。 大小的最大总和$k$是通过选择来实现的$k$最大的$d[i]$，并且通过选择获得最小总和$k$最小的$d[i]$。 对数组进行排序给出了前缀和，因此我们可以计算两个边界$O(1)$每$k$预处理后。 

这将整个问题简化为构建两个数组$L_k$和$R_k$为所有人$k$，然后回答区间并集查询。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集 DP |$O(n2^n)$|$O(2^n)$| 太慢了|
 | 排序 + 极端前缀 |$O(n \log n + n + q)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 读取树并计算每个节点的度数。 然后将每个值变换为$d[i] = \deg(i) - 1$。 此步骤之后不再需要树结构本身，因为所有约束都编码在这些值中。 
2. 对数组进行排序$d$按非递减顺序。 排序是必要的，因为我们希望通过有效地选择最小或最大元素来构造极值和。 
3.构建前缀和数组$pref$， 在哪里$pref[i]$是第一个的总和$i$已排序数组的元素。 这允许对极端子集的总和进行恒定时间计算。 
4. 对于每个子集大小$k$，计算最小可能的总和$L_k$作为$pref[k]$，因为取最小的$k$元素使总和最小化。 
5. 对于每个子集大小$k$，计算最大可能的总和$R_k$作为$pref[n] - pref[n-k]$，因为这对应于取最大的$k$元素。 
6. 预先计算所有区间$[L_k, R_k]$为了$k = 0$到$n$。 每个区间准确地表示大小子集可实现总和的范围$k$。 
7. 对于每个查询$(x, y)$，计算间隔并集的大小$[L_x, R_x]$和$[L_y, R_y]$。 如果重叠，则合并； 否则将它们的长度相加。 

### 为什么它有效

 正确性来自于排序值的交换参数。 大小的任意子集$k$可以转化为$k$通过重复地将选定的大元素与较小的未使用元素交换来计算最小元素，每次交换都会严格地减少或增加总和一个受控的量，而不跳过任何中间可实现的总数。 这确保了固定基数的最小和最大可实现总和之间没有“漏洞”，因此可达集恰好是一个连续整数区间。 

由于这种连续性，只有极端子集对于确定所有可能性很重要。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    deg = [0] * n
    
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        deg[u] += 1
        deg[v] += 1

    d = [x - 1 for x in deg]
    d.sort()

    pref = [0] * (n + 1)
    for i in range(n):
        pref[i + 1] = pref[i] + d[i]

    total = pref[n]

    for k in range(n + 1):
        L[k] = pref[k]
        R[k] = total - pref[n - k]

        if r < l:
            return 0
        return r - l + 1

    def union_length(a, b, c, d):

        right = max(b, d)
        overlap = max(0, min(b, d) - max(a, c) + 1)

        if overlap > 0:

        return (b - a + 1) + (d - c + 1)

    q = int(input())
    out = []

            l1, r1 = L[x], R[x]
            l2, r2 = L[y], R[y]

            overlap = max(0, min(r1, r2) - max(l1, l2) + 1)
            out.append(str((r1 - l1 + 1) + (r2 - l2 + 1) - overlap))

    print("\n".join(out))

```实现首先将树转换为度导出数组，这是之后唯一重要的结构。 对数组进行排序可以构建前缀和，从而可以恒定时间访问任何大小的最小和最大子集和。 

一个常见的实现陷阱是忘记子集和是经过变换的值$d[i]$，不是原始学位。 另一个是错误地处理负值，因为即使原始度数非负，移位后的度数也可能为负。 

查询逻辑纯粹是区间算术：每个$k$对应于可能和的连续区间，并且每个查询询问两个这样的区间的并集的大小。 

## 工作示例

 ### 示例 1

 假设我们有一棵产生值的小树$d = [0, 1, 2]$改造后。 

排序后的数组是$[0,1,2]$，前缀和是$0,1,3,6$。 

| k | L_k | R_k |
 | --- | --- | --- |
 | 0 | 0 | 0 |
 | 1 | 0 | 2 |
 | 2 | 1 | 3 |
 | 3 | 3 | 3 |

 对于查询$(1,2)$，我们取间隔$[0,2]$和$[1,3]$。 他们的工会是$[0,3]$，因此从 0 到 3 的所有整数至少可以在两个子集大小之一中实现。 

这显示了重叠如何将两个范围折叠成一个连续的块。 

### 示例 2

 考虑一个星形结构产生$d = [-1,0,0,0]$。 

排序后的数组是$[-1,0,0,0]$，前缀和是$-1,-1,-1,0, -0$。 

为了$k=1$，区间为$[-1,0]$。 为了$k=2$，区间为$[-1,0]$再次。 这表明不同的子集大小可以产生相同的可实现范围，这是预期的，因为许多元素在转换后是相同的。 

对此类相同间隔的查询确认联合逻辑正确地避免了重复计算。

 ## 边缘情况

 单节点树产生$d[1] = -1$，唯一的子集大小是 0 和 1。该算法正确形成区间$[0,0]$和$[-1,-1]$，因此查询表现为退化区间的并集。 

在星形树中，大多数节点都有$d[i]=0$且有一个节点有较大的正值。 排序会导致许多相同的前缀和，从而折叠许多间隔。 该算法仍然有效，因为它仅依赖于前缀极值，即使在大量重复的情况下也仍然有效。 

在路径图中，度数在 1 和 2 之间变化，因此移位后，值大部分为 0，少数为 1 和 -1。 区间构造自然会产生重叠范围，并且联合逻辑正确地合并它们而不会丢失中间值。
