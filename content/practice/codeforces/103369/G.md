 ## 问题理解

 我们得到一个整数数组和一个最多可以使用一次的允许操作。 该操作让我们可以选择任意位置子集并添加相同的正值$k$到所有选定的元素。 在选择性地应用此操作一次之后，我们希望结果数组变得非递减。 

换句话说，我们可以通过均匀移位向上“提升”一些元素，并且我们想知道是否可以仅使用应用于所选子集的单个全局增量来修复由原始排序引起的所有反转。 

关键的难点在于操作受到极大的限制：我们无法独立调整元素，只能将数组分成两组，一组接收$+k$以及那些没有的。 

约束意味着$n$最多$2 \cdot 10^5$在所有测试用例中，因此每个测试用例的任何解决方案都必须是线性或接近线性的。 对所有子集或对进行二次推理是不可能的。 

天真的解释可能建议尝试所有索引子集和所有可能的$k$，但是子集选择是指数的，并且是无限的$k$。 即使限制于结构模式仍然会导致太多的情况，除非我们利用强单调性。 

当数组已经非递减时，会出现微妙的边缘情况。 在这种情况下，正确的答案是立即的：我们可以选择根本不应用该操作。 当数组“几乎已排序”（除了多个递减段之外）时，会出现另一种边缘情况。 例如，在像这样的数组中$[3, 1, 2, 4]$，如果所需的校正在反演过程中不一致，则单个均匀提升无法修复排序。 

## 方法

 蛮力策略会将索引的每个子集视为“提升集”，并尝试确定是否存在$k$这样添加后$k$对于该子集，数组已排序。 For a fixed subset, determining feasibility reduces to checking whether all violated order constraints can be satisfied by a single consistent shift. 然而，有$2^n$子集，甚至检查每个子集都需要$O(n)$，导致$O(n2^n)$，这远远超出了可行的范围。 

关键的观察结果是该操作创建了数组位置的二进制分区：每个索引要么保持不变，要么移动相同的量。 一旦我们修复了有效的最终排序数组，每个元素都必须属于两个“级别”之一：原始值或原始值加$k$。 这意味着在最终的排序序列中，每当我们从非移位元素转换为移位元素时，差异结构必须全局一致。 

关键的见解是查看数组的排序版本并将其与原始版本进行比较。 如果我们想象对数组进行排序，最终数组中的每个元素要么对应于其原始值，要么对应于其原始值减去$k$。 因此，如果我们将原始数组与其排序版本对齐，则相应元素之间的差异最多只能采用两个不同的值：$0$和$k$。 任何第三个差异都会立即使转变变得不可能。 

这将问题简化为检查原始数组和排序数组之间的差异多重集是否最多包含一个正的不同值。

 ## 算法演练

 ### 1. 对数组的副本进行排序

 我们构造输入数组的排序版本。 这代表了我们想要实现的目标非递减顺序。 

### 2. 将每个原始元素与其排序位置进行比较

 对于每个索引，计算排序值与原始值之间的差异。 这种差异告诉我们该元素需要增加多少才能匹配其排序位置。 

如果我们看到负差异，则意味着原始元素已经大于其排序位置，这不能仅通过添加值来修复，因此答案立即不可能。 

### 3.收集所有正差异

 我们收集所有积极的差异。 这些表示必须增加才能到达其排序位置的元素。 

如果我们只允许使用一个单一的操作$k$，那么所有这些正差必须相等，因为每个选定的元素都会收到完全相同的增量。 

### 4. 验证所需增量的一致性

 如果正差集包含多个不同值，则不可能选择单个值$k$同时固定所有需要的位置。 

如果恰好存在一个明显的正差异，则该值就是候选值$k$。 

### 5. 决定可行性

 如果没有出现冲突并且满足所有条件，则可以通过一次操作或零次操作（如果已经排序）来修复数组。 

### 为什么它有效

 该操作施加了严格的结构：每个修改的元素都移动完全相同的量。 当与排序目标进行比较时，每个元素独立地暗示所需的移位值。 如果必须向上移动的元素之间需要多个移动值，则无法将它们与单个全局协调$k$。 相反，如果所有所需的班次都一致，我们可以准确选择这些元素并应用它$k$，生成排序后的数组。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    b = sorted(a)
    
    diffs = set()
    
    for x, y in zip(a, b):
        if y < x:
            print("No")
            return
        if y > x:
            diffs.add(y - x)
    
    if len(diffs) <= 1:
        print("Yes")
    else:
        print("No")

t = int(input())
for _ in range(t):
    solve()
```该实现直接遵循将数组与其排序版本对齐的想法。 关键细节是我们将任何增加的要求视为需要统一转变的证据。 如果出现多个明显的正向变化，我们会立即拒绝。 

支票`y < x`处理不可逆反转情况，其中排序需要减少元素，这是操作不允许的。 

套装`diffs`强制所需增量的全局一致性。

 ## 边缘情况

 一种重要的边缘情况是数组已经排序。 该算法处理此问题是因为所有差异均为零，因此集合保持为空并且答案被接受。 

另一种情况是只有一个元素需要移动。 例如，$[1, 3, 2]$产生一个单一的正差异模式，该模式仍然是一致的，因此算法正确地接受它。 

更微妙的情况是多个元素需要增加但增加量不同。 例如$[1, 4, 2, 6]$产生两个不同的所需增量，因此无法选择一个$k$。 该算法正确地拒绝了这一点，因为差异集的大小大于 1。
