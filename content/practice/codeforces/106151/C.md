 ## 解决方案
 ## 问题理解

 我们得到一个有向图，其中每个顶点代表豪宅中的一个房间，每个有向边代表房间之间的单向门。 每扇门的成本为 0 或 1，具体取决于它是否已解锁或需要使用徽章才能通过。 有效地使用徽章意味着我们可以穿过一扇锁着的门，但它总是会给总旅行时间增加 1 的成本。 

任务是找到从房间 1 到房间 N 的最小总成本，或者如果没有有向路径则确定这是不可能的。 

这些约束允许最多 100,000 个房间和 100,000 个门，这立即排除了任何尝试从头开始为每条边重新计算最短路径或使用朴素 O(NM) 松弛方法的解决方案。 我们需要接近线性或线性时间的东西。 

将图表视为未加权时会出现常见的失败情况。 例如，如果我们运行忽略权重的标准 BFS，我们会错误地假设每条边的成本相等。 

考虑一个小案例：

 输入：

 1 2 1

 1 3 0

 3 2 0

 从 1 到 2 的最佳路径是 1 → 3 → 2，成本为 0，而直接边 1 → 2 的成本为 1。普通 BFS 可能会根据邻接顺序错误地优先考虑直接边，并错过更便宜的路径。 

另一个微妙的问题来自于将 Dijkstra 与二进制堆一起使用，但将权重视为一般整数。 虽然正确，但有点矫枉过正，因为权重只有 0 或 1，我们可以做得更好。 

## 方法

 暴力破解的想法是使用标准的最短路径算法，例如 Dijkstra。 每次我们放松边缘时，我们都会将更新的距离推入优先级队列。 这是正确的，因为所有边权重都是非负的，因此 Dijkstra 保证了最优性。 

然而，复杂度变成了 O(M log N)，这在这里是可以接受的，但它忽略了问题的结构：权重只有 0 或 1。该结构允许我们完全避免对数因子。 

关键的观察结果是，每次松弛最多只会将距离增加 1。这意味着我们不需要完整的优先级队列排序； 我们只需要区分“相同成本”和“成本+1”。 这自然会导致基于双端队列的 BFS 变体，其中成本 0 的边缘位于前面，成本 1 的边缘位于后面。 

这将问题转化为 0-1 加权图中的最短路径，可使用 0-1 BFS 在 O(N + M) 中求解。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | Dijkstra 与堆 | O(M log N) | O(M log N) | O(N + M) | 已接受 |
 | 0-1 BFS | O(N + M) | O(N + M) | 已接受 |

 ## 算法演练

 我们使用双端队列处理图，保持靠近前面的节点始终具有较小当前距离的不变性。 

1. 构建一个邻接表，其中每条边都存储其目的地和权重。 
2. 为除节点 1 之外的所有节点初始化一个无穷大的距离数组，将其设置为 0。这表示我们从房间 1 开始，不会产生任何成本。 
3. 将节点 1 推入双端队列。 双端队列的行为就像一个优先级结构，其中更便宜的状态首先在不使用堆的情况下扩展。 
4. 当双端队列不为空时，从前面弹出一个节点。 这确保了我们始终扩展当前已知的最便宜的状态。 
5. 对于当前节点的每条传出边，将候选距离计算为当前距离加上边权重。 如果该值小于邻居存储的距离，则更新它。 
6. 如果边权重为0，则将邻居推到双端队列的前面。 如果边权重为1，则将其推到后面。 此顺序可确保在同一探索级别的任何成本 1 转换之前处理所有零成本转换。 
7. 继续，直到双端队列为空。 
8. 答案是到节点 N 的距离，如果保持无限大则为 -1。 

### 为什么它有效

该算法保持了节点按距离非递减顺序处理的特性。 任何时候我们放松权重为0的边缘，都不会增加成本，因此必须尽快处理结果状态，因此它被推到前面。 权重 1 的任何边都会增加成本，因此它必须落后于当前所有的零成本扩张。 

由于所有边权重仅为 0 或 1，因此任何最短路径都可以分解为一个序列，其中转换按此双端队列规则正确排序。 这保证了当一个节点第一次被最终确定（弹出）时，它的距离已经是最小的，并且以后永远不会得到改善。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

n, m = map(int, input().split())
g = [[] for _ in range(n + 1)]

for _ in range(m):
    u, v, t = map(int, input().split())
    g[u].append((v, t))

INF = 10**18
dist = [INF] * (n + 1)
dist[1] = 0

dq = deque([1])

while dq:
    u = dq.popleft()
    for v, w in g[u]:
        nd = dist[u] + w
        if nd < dist[v]:
            dist[v] = nd
            if w == 0:
                dq.appendleft(v)
            else:
                dq.append(v)

print(-1 if dist[n] == INF else dist[n])
```邻接表完全按照给定的方式存储有向边，从而保留方向性。 距离数组初始化得足够大，以避免重复添加 0 或 1 时出现溢出问题。 

双端队列操作是关键的实现细节：appendleft仅用于零权重边以确保立即传播，而append用于权重1边以适当延迟处理。

 | ---| ---| ---| ---|
 | 1 | 1 | 0 | 距离[2]=0，距离[4]=1 |

 最佳路径是 1 → 2 → 3 → 5，成本为 1。 

该跟踪显示了如何对 0 成本边进行优先级排序，从而允许节点 2 在节点 4 之前得到处理，即使两者都是较早发现的。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(N + M) | 每次成功松弛，每个节点最多被推入双端队列一次，并且每条边都被处理一次 |

 ## 边缘情况

 一种边缘情况是，当存在具有混合成本的多条路径时，幼稚的 BFS 会错误地锁定更糟糕的早期路由。 例如：```
4 4
1 2 1
2 4 1
1 3 0
3 4 0
```正确的路径是 1 → 3 → 4，成本为 0。由于边权重为 0，算法首先将节点 3 推到前面，因此它在扩展更高成本路径之前完全通过零成本链传播。 在来自节点 2 的 1 成本路由最终确定之前，以距离 0 到达节点 4。 

另一个边缘情况是完全断开连接。 如果节点 N 从未到达，则其距离保持无穷大，算法正确输出 -1，无需在最终比较之外进行显式可达性检查。
