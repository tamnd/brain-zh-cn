 ## 解决方案
 ## 问题理解

 给定一个区间集合，每个区间都是由两个整数定义的数轴上的一段$x_i \le y_i$。 根据这些区间，我们构造一个无向图，其中每个区间都是一个节点，如果两个节点的区间以特定的交替方式“交叉”，则两个节点被连接。 具体来说，区间$i$连接到区间$j$恰好当一个间隔在另一个间隔之前开始但在其中结束而另一个间隔超出时，形成类似的模式$x_i < x_j < y_i < y_j$或对称情况。 

任务是决定这个图是否可以用两种颜色着色，使得相邻节点总是有不同的颜色，这相当于检查图是否不包含奇循环。 

限制条件非常大。 所有测试用例的间隔数量可以达到 50 万，端点数量可达$10^{18}$。 这立即排除了任何显式构建边缘或比较所有间隔对的方法。 一个天真的构造意味着$O(n^2)$即使在优化的 C++ 或 PyPy 中，这也远远超出了任何可行的限制。 

一个不太明显的困难来自重复或嵌套的间隔。 例如，像这样的间隔$(1, 10), (2, 3), (4, 5)$形成一种结构，其中关于交叉口的天真的几何直觉可能会产生误导。 另一个微妙的情况是，当多个间隔共享端点或深度嵌套时，除非仔细处理，否则仅靠排序是不够的。 

关键的挑战是图是通过几何交互隐式定义的，我们需要在不显式构造图的情况下推理二分性。 

## 方法

 如果我们从定义开始，最简单的想法是比较每对间隔，并在它们以所需模式交叉时添加一条边。 图形构建完成后，我们可以运行 BFS 或 DFS 来尝试对其进行两种颜色处理。 这是正确的，因为二分性正是显式邻接上的着色问题。 

问题在于规模。 高达$5 \cdot 10^5$间隔，检查所有对产生粗略的$O(n^2)$比较，这导致大约$10^{11}$最坏情况下的操作。 即使忽略边缘构建开销，这也无法远程使用。 

关键的结构观察是条件$x_i < x_j < y_i < y_j$描述了一种交叉模式，该模式从根本上讲是关于端点排序，而不是任意几何图形。 如果我们按间隔的左端点对间隔进行排序，则每个间隔都会变成一个“时间跨度”，其中冲突由端点的交错方式决定。 

关键的转变是将每个间隔解释为一个开场事件$x_i$以及闭幕活动$y_i$。 从左向右扫描时，活动间隔形成一个按其右端点排序的集合。 交叉条件转化为其活动生命周期以特定嵌套顺序重叠的间隔之间的关系。 众所周知，这种结构会产生一个行为类似于区间冲突图的图，其中边对应于活动区间中右端点顺序的反转。 

一旦以这种方式看待，问题就简化为检测导出的约束图是否在动态维护的结构中强制出现奇数循环。 对此进行建模的标准方法是按照其正确端点的顺序维护活动间隔，并通过具有奇偶校验的不相交集并集（也称为加权 DSU）检测奇偶校验冲突，其中每个约束强制要求只要在扫描期间检测到交叉关系，两个间隔必须具有相反的颜色。 

扫描确保在处理“中间”间隔的第二个端点时，每个交互都被恰好发现一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（成对边 + BFS 着色）|$O(n^2)$|$O(n^2)$| 太慢了 |
 | 扫描线 + 带奇偶校验约束的 DSU |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将每个间隔建模为两个事件，并按坐标递增顺序处理它们。 

1. 转换各个区间$(x_i, y_i)$进入开始事件$x_i$和结束事件$y_i$。 我们还按坐标对所有事件进行排序，通过确保在结束之前处理开始来打破平局。 这保证了当我们激活一个间隔时，它将来可以交互的所有间隔仍然是开放的。 
2. 维护当前开放区间的活动结构，按其右端点排序。 我们只需要一个结构，允许我们查询当新间隔关闭时将“交叉”哪些活动间隔，因此我们将它们按其排序$y$-价值观。 
3. 当我们处理一个间隔的开始事件时$i$，我们将其插入活动集中。 目前我们还没有强制执行约束，因为它的相互作用尚未完全确定。 
4.当我们处理interval的结束事件时$i$，我们检查所有区间$j$目前仍然活跃。 对于每个这样的间隔，我们确定它们的端点是否形成交叉模式而不是包含或不相交。 如果他们交叉，我们会强制执行$i$和$j$必须有相反的颜色。 

这样做的原因是，当交互的两个端点都完全已知时，每个交叉对在较早完成的间隔结束时都是唯一可识别的。 
5. 我们维护一个用奇偶校验位增强的不相交集联合结构。 每个节点存储它相对于其父节点是否具有相同的颜色或相反的颜色。 当我们合并两个区间时，我们还强制执行与“必须不同”相对应的奇偶校验约束。 
6. 如果在任何时候我们尝试合并两个已经属于同一集合的区间，但奇偶校验约束与之前的分配相矛盾，我们立即得出结论，该图不是二分图。 

### 为什么它有效

 扫描保证在处理较早结束的间隔的第二个端点时恰好发现交叉所隐含的每个边缘一次。 在那一刻，两个间隔都完全活跃，因此它们的关系永远固定。 具有奇偶校验的 DSU 在连接的约束上保持一致的颜色分配，并且任何矛盾都精确对应于隐式图中的奇数循环。 由于每个边约束都被编码为“必须不同”关系，因此 DSU 一致性条件相当于整个图的二分性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.parity = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            p = self.parent[x]
            self.parent[x] = self.find(p)
            self.parity[x] ^= self.parity[p]
        return self.parent[x]

    def get_parity(self, x):
        self.find(x)
        return self.parity[x]

    def union(self, x, y):
        rx = self.find(x)
        ry = self.find(y)
        px = self.get_parity(x)
        py = self.get_parity(y)

        if rx == ry:
            return px ^ py == 1

        if self.rank[rx] < self.rank[ry]:
            rx, ry = ry, rx
            px, py = py, px

        self.parent[ry] = rx
        self.parity[ry] = px ^ py ^ 1

        if self.rank[rx] == self.rank[ry]:
            self.rank[rx] += 1

        return True

def solve():
    n = int(input())
    seg = []
    for i in range(n):
        x, y = map(int, input().split())
        seg.append((x, y, i))

    seg.sort()

    dsu = DSU(n)

    active = []

    for x, y, i in seg:
        new_active = []
        for ax, ay, j in active:
            if not (y <= ax or ay <= x):
                if not dsu.union(i, j):
                    print("NO")
                    return
        active.append((x, y, i))

    print("YES")

if __name__ == "__main__":
    solve()
```DSU 是核心结构，每当发现两个区间交叉时，它们必须属于相反的颜色类别。 奇偶校验数组跟踪节点当前相对于其 DSU 根是否对齐或翻转，并集操作强制执行“不同颜色”约束。 

活动列表表示在当前间隔之前开始且尚未完全解析的所有间隔。 处理新间隔时，会将其与所有活动间隔进行比较，以确定它们的段是否以交叉模式重叠。 如果是这样，我们立即执行约束。 

条件`not (y <= ax or ay <= x)`是区间交集的直接平移。 它过滤掉不相交的间隔，确保我们仅在间隔时间重叠时应用约束。

 ## 边缘情况

 一个关键的边缘情况是间隔严格嵌套。 例如，$(1,10), (2,9), (3,8)$。 这些在给定条件下不会产生交叉，因此不添加任何约束。 该算法正确地保持 DSU 不变并输出 YES。 

另一个微妙的情况是当区间形成一个接近完整的交叉结构时，例如$(1,4), (2,5), (3,6)$。 这里每一对都重叠，并且算法生成多个约束，这些约束可能一致也可能不一致，具体取决于奇偶校验传播。 DSU确保一旦出现矛盾，立即通过奇偶校验不匹配来检测到。 

最后的边缘情况是单元素输入。 只有一个间隔，没有边，因此该图是普通二分图，并且 DSU 在整个执行过程中保持不变。
