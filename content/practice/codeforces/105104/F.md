 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供了四个轴对齐的矩形。 每个矩形都可以旋转，这意味着我们可以自由交换其边。 任务是决定这四块是否可以不重叠地放置，以恰好填满一个完美的正方形，不留任何间隙，也不延伸到边界之外。 

关键的限制是我们不允许切割或变形矩形，只能平移和旋转它们。 因此，最终的形状必须是使用四个矩形块完美地平铺正方形。 

就每次测试工作而言，输入大小适中，但测试用例数量可能很大。 多达 1000 个测试用例，每个用例只有四个矩形，即使是$O(4!)$或每个案例的小常数因子指数搜索是可以接受的。 任何比每个测试用例几千个配置更糟糕的配置都已经是不必要的了，但是大参数中的任何多项式都不会出现在这里，因为结构被固定为四部分。 

天真的推理中的一个微妙的失败模式是假设“匹配总面积”就足够了。 它不是。 例如，矩形 (1, 1)、(1, 1)、(1, 1)、(1, 3) 的总面积为 6，这无论如何都不是正方形面积，但即使总数与正方形匹配，几何图形仍然可能失败。 

另一个常见的错误是只检查单个布局，例如“一行或一列中的所有矩形”。 这会错过有效的配置，例如将正方形分成两个水平条，每个条由两个并排的矩形填充。 

核心困难在于矩形必须沿着共享边缘完美对齐，这对边长如何在组之间匹配产生了强大的结构约束。 

## 方法

 暴力方法会尝试将四个矩形放置在假设的正方形内的所有可能位置。 这需要选择一个正方形边长，然后尝试使用连续几何图形或回溯放置来打包矩形。 即使只有四个矩形，连续放置也会引入许多自由度：每个矩形都可以放置在许多坐标处，并且检查重叠变得非常重要。 在最坏的情况下，探索布局会退化为几何回溯问题，其连续搜索空间很大，远远超出了必要的范围。 

关键的观察结果是，任何使用四个矩形的正方形的有效平铺都必须遵循非常严格的结构。 由于所有切割都是轴对齐的并且矩形不能重叠，因此正方形只能分解为少量的组合图案。 对于四个矩形，每个有效的配置都可以分解为三种结构形式之一。 

第一种形式是单个水平条或垂直条分解，其中所有矩形在一个方向上对齐。 在这种情况下，所有矩形共享一个等于正方形边长的尺寸，其他尺寸之和等于边长。 

第二种形式将正方形分成两个水平（或垂直）行，每行恰好包含两个并排放置的矩形。 在此配置中，每行必须具有统一的高度，并且每行内的宽度必须总和为正方形边长。 此外，两个行高的总和必须等于正方形边长。 

第三种形式与旋转对称下的第二种形式没有本质上的区别，因此它不会引入额外的情况。 

这种从几何放置到结构化分区的减少减少了检查矩形组中恒定数量的组合分配以及方向选择的问题。

 | ---| ---| ---| ---|

 ## 算法演练

 我们将问题简化为尝试所有一致的方法来将四个矩形定向和划分为有效的正方形平铺。 

1. 枚举每个矩形的所有旋转。 每个矩形有两种可能的状态，因此有$2^4 = 16$方向配置。 这是必要的，因为在一个方向上不适合的矩形在交换其边后可能会适合。 
2. 对于每个方向配置，考虑四个矩形的所有排列。 这给出了 24 次重新排序，确保分组决策不会因输入顺序而产生偏差。 
3. 对于每种排列，根据结构模式计算候选平铺。 
4. 首先检查单排配置。 将所有四个矩形并排放置在一个水平条中。 仅当所有矩形具有相同的高度并且它们的宽度之和等于正方形的边长（其中边长是公共高度）时，这才有效。 
5.然后对称检查单列配置。 所有矩形必须共享相同的宽度，并且它们的高度之和必须等于该宽度。 
6. 接下来检查两行配置。 将四个矩形分成两组，每组两个。 对于每个可能的分割，验证每个组是否可以形成一行：在一个组内，所有矩形必须具有相同的高度，以便它们水平对齐。 每组中的宽度之和必须为相同的值，即正方形边长。 最后，正方形的高度成为两行高度的总和，并且它必须与之前确定的宽度相匹配。 
7. 如果任何配置满足这些约束，则答案为“是”。 如果没有，则输出“No”。 

### 为什么它有效

 具有四个轴对齐矩形的正方形的任何有效平铺都会导致将正方形划分为由矩形边界定义的水平或垂直条带。 由于只有四块，因此划分图在任一方向上最多有两级细分。 这会强制将所有有效布局分成一个由四个矩形组成的条带或两个由两个矩形组成的条带。 由于旋转被显式枚举，因此消除了任何方向不匹配，并且由于尝试了所有分区，因此不会遗漏任何结构布置。 这种对恒定大小的配置空间的详尽性保证了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from itertools import permutations, product

def can(rects):
    for perm in permutations(rects):
        a, b, c, d = perm

        # single row
        h = a[1]
        if b[1] == h and c[1] == h and d[1] == h:
            if a[0] + b[0] + c[0] + d[0] == h:
                return True

        # single column
        w = a[0]
        if b[0] == w and c[0] == w and d[0] == w:
            if a[1] + b[1] + c[1] + d[1] == w:
                return True

        # two rows: (a,b) and (c,d)
        # check pairing (0,1)|(2,3)
        pairs = [((a, b), (c, d)), ((a, c), (b, d)), ((a, d), (b, c))]

        for (r1, r2) in pairs:
            (x1, y1), (x2, y2) = r1
            (x3, y3), (x4, y4) = r2

            if y1 == y2 and y3 == y4:
                h1 = y1
                h2 = y3
                if x1 + x2 == x3 + x4 and x1 + x2 > 0:
                    s = x1 + x2
                    if h1 + h2 == s:
                        return True

    return False

def solve_case(rects):
    for mask in range(16):
        oriented = []
        for i in range(4):
            w, h = rects[i]
            if mask & (1 << i):
                oriented.append((h, w))
            else:
                oriented.append((w, h))
        if can(oriented):
            return True
    return False

def main():
    t = int(input())
    for _ in range(t):
        rects = [tuple(map(int, input().split())) for _ in range(4)]
        print("Yes" if solve_case(rects) else "No")

if __name__ == "__main__":
    main()
```该解决方案首先将方向处理与结构验证分开。 这`solve_case`函数使用位掩码迭代所有旋转状态，确保在两个可能的方向上考虑每个矩形。 

这`can`然后函数尝试矩形的所有排列以避免对输入顺序的依赖。 在其中，代码检查三种结构可能性：所有矩形在一行中、所有矩形在一列中以及两行分割。 

两行情况是通过在所有三种可能的配对中显式配对矩形来处理的。 通过在对内强制执行相同的高度并验证两行的宽度总和一致，将每对作为潜在行进行测试。 最终的正方形边长是根据行宽度得出的，并根据总高度进行验证。 

正确性取决于这样一个事实：一旦方向固定，任何有效的平铺都必须对齐到这些离散分区模式之一。

 ## 边缘情况

 一种特殊情况是所有矩形都是相同的正方形。 在这种情况下，每个方向都会导致有效的平铺，但算法不能依赖于单一配置。 旋转循环确保即使输入顺序不利，仍然会发现有效的排列。 

另一种情况是矩形面积匹配但边长不兼容，例如 (1, 6)、(2, 3)、(3, 2)、(6, 1)。 尽管总面积是一致的，但任何行或列对齐都无法使组之间的尺寸相等。 在执行期间，每个排列要么不满足行中的等高约束，要么不满足列中的等宽约束，因此该函数正确地拒绝这种情况。 

第三种情况是不对称但有效的平铺，其中正方形被分成具有不同高度的两行，每行包含两个宽度不同但高度相等的矩形。 两行检查明确强制每对内的高度相等，因此只要分组正确对齐，仍然可以检测到此类配置。
