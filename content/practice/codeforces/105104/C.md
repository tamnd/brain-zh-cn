 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供一个数组，我们需要计算有多少子数组满足特定的奇偶校验条件：在所选子数组内，至少有一个值出现奇数次。 

同样，如果子数组的频率分布不全均匀，则该子数组被视为有效。 子数组失败的唯一原因是子数组中的每个不同值出现偶数次。 

输出是所有测试用例中此类子数组的数量。 

限制在一个非常具体的方面是很大的。 所有测试用例的总长度最多为10^6，并且测试用例的数量也可能很大。 这排除了任何 O(n^2) 子数组枚举。 除非非常轻量级，否则即使每个测试用例的 O(n log n) 也将是临界值。 

检查每个子数组并计算频率的简单方法将需要 O(n^2) 个子数组，并且每个子数组最多需要 O(n) 次工作，这远远超出了限制。 即使优化频率维护仍然留下 O(n^2) 子数组。 

当所有元素都相同时，会出现微妙的边缘情况。 例如，如果数组为 [1, 1, 1, 1]，则每个奇数长度的子数组都有效，因为只有这样计数才会变为奇数。 粗心的方法可能会错误地假设所有子数组都是有效的，或者尝试根据全局奇偶校验错误地进行捷径。 

另一个边缘情况是交替模式，如 [1, 2, 1, 2]。 这里许多子数组具有全偶频率计数，并且有效子数组的结构取决于跨段的奇偶校验取消，而不仅仅是本地检查。 

关键的困难在于条件是频率上的全局条件，而不是元素上的局部条件。 

## 方法

 强力解决方案枚举每个子数组 [l, r]，维护频率图，并检查所有频率是否均匀。 对于每个子阵列，我们将增量更新频率，但仍然重新计算或维持奇偶校验状态。 即使使用增量更新，也有 O(n^2) 个子数组，并且每次更新都会影响 O(1) 状态，但检查有效性需要了解所有计数是否均匀，这仍然取决于跟踪可能所有不同值的奇偶校验。 

在最坏的情况下，这导致每次测试大约需要 O(n^2) 次操作，总规模变成 10^12 次操作，这显然是不可能的。 

关键的观察是我们实际上不需要跟踪完整的频率。 我们只关心频率的奇偶性。 每个值在子数组中贡献偶数或奇数计数。 我们想要计算至少一个值具有奇奇偶校验的子数组。 

这相当于总子数组减去每个值具有偶数频率的子数组。 所以我们翻转问题：计算频率全偶数的子数组。 

现在我们将其转化为前缀奇偶校验的角度。 对于每个前缀，我们维护一个奇偶校验状态，描述哪些值到目前为止出现了奇数次。 当子阵列的两个前缀状态相同时，它具有全偶频率。 这是因为异或相等奇偶校验状态会取消一切。 

因此，问题简化为计算相等前缀 XOR 状态对。 如果我们将前缀奇偶校验表示为可散列状态，则好子数组（全偶数）的数量是出现组合状态的总和。 那么答案就是子数组总数减去该值。 

我们维护奇偶校验更新的滚动式异或式签名。 由于值很大（最多 10^6），因此我们无法存储完整的位集。 相反，我们对每个值使用随机哈希，并将它们异或为运行的前缀哈希。 这将奇偶校验状态压缩为单个整数，同时以高概率保留相等比较。 

因此，我们将问题简化为计算相等的前缀哈希值。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 前缀奇偶校验哈希 | O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 对于每个测试用例，将运行的前缀 XOR 哈希初始化为 0。这表示没有值出现的空前缀的奇偶校验状态。 
2. 为每个不同值分配一个随机 64 位整数。 该值代表其对奇偶校验切换的贡献。 
3. 从左到右遍历数组，通过异或当前元素分配的随机值来更新前缀哈希。 这模拟了切换奇偶校验：第一次出现将其打开，第二次出现将其关闭，依此类推。 
4. 维护一个频率字典，用于计算每个前缀哈希到目前为止出现的次数。 
5. 每次达到前缀哈希值时，我们都会将该哈希的当前计数添加到“坏”子数组（所有频率均为偶数的子数组）的运行总数中。 这是有效的，因为两个相同的前缀状态定义了一个具有偶数计数的子数组。 
6. 处理完所有前缀后，计算子数组总数为 n(n+1)/2。 
7. 从总子阵中减去全偶数频率子阵的数量。 结果是至少有一个值出现奇数次的子数组的数量。 

核心思想是前缀相等对区间内所有奇偶校验贡献的取消进行编码。 

### 为什么它有效

 每个前缀哈希表示直到该索引的所有元素的奇偶校验向量。 当 r 处的奇偶校验状态等于 l − 1 处的奇偶校验状态时，子数组 [l, r] 具有所有偶数频率。XOR 编码可确保相同的多组切换产生相同的哈希值。 因此，计算相等前缀状态相当于计算零奇偶校验子数组，并且补码给出所需的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    rng = 10**6 + 5

    # deterministic pseudo-random using splitmix64 style
    def splitmix64(x):
        x += 0x9e3779b97f4a7c15
        x = (x ^ (x >> 30)) * 0xbf58476d1ce4e5b9
        x &= (1 << 64) - 1
        x = (x ^ (x >> 27)) * 0x94d049bb133111eb
        x &= (1 << 64) - 1
        return x ^ (x >> 31)

    # precompute random hashes for values
    h = [0] * (rng)
    seed = 123456789
    for i in range(rng):
        seed = splitmix64(seed + i)
        h[i] = seed

    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        pref = 0
        freq = {0: 1}
        total_even = 0

        for x in a:
            pref ^= h[x]
            total_even += freq.get(pref, 0)
            freq[pref] = freq.get(pref, 0) + 1

        total_subarrays = n * (n + 1) // 2
        print(total_subarrays - total_even)

if __name__ == "__main__":
    solve()
```该解决方案首先为每个可能的值构建稳定的哈希值。 splitmix64 函数用于避免实践中的冲突，并且是快速确定性哈希的竞争性编程的标准。 

对于每个测试用例，我们使用这些哈希值上的 XOR 来计算前缀奇偶校验状态。 字典`freq`跟踪每个前缀状态出现的频率。 每个重复的前缀状态形成一个有效的“全偶频”子数组，因此我们累积这些计数。 

最后，我们从总子数组中减去。 

一个常见的实现陷阱是忘记初始化`freq[0] = 1`，它考虑从索引 0 开始的子数组。另一种方法是使用 Python 的内置哈希，它在每个进程中都是随机的，并且在运行中不稳定。 

## 工作示例

 ### 示例 1

 输入：```
n = 3
a = [1, 2, 3]
```我们计算前缀状态：

 | 我| 一个[我] | 前缀哈希 | 之前的频率 | 添加偶数子数组 |
 | ---| ---| ---| ---| ---|
 | 0 | 1 | h1 | {0:1} | 0 |
 | 1 | 2 | h1^h2 | h1^h2 | {0:1,h1:1} | 0 |
 | 2 | 3 | h1^h2^h3 | h1^h2^h3 | ... | 0 |

 没有前缀重复，所以`total_even = 0`。 子数组总数为 6。答案为 6。 

这表明当所有元素都不同时，每个子数组都包含奇数出现。 

### 示例 2

 输入：```
n = 4
a = [1, 1, 2, 2]
```前缀演变：

 | 我| 一个[我] | 前缀 | 更新频率 | 甚至添加了子数组 |
 | ---| ---| ---| ---| ---|
 | 0 | 1 | h1 | +0 | 0 |
 | 1 | 1 | 0 | +0 | 1（h1 重复）|
 | 2 | 2 | 小时2 | +0 | 1 |
 | 3 | 2 | 0 | +1 | 2 |

 这里，前缀 0 重复两次，对应于 [1,1,2,2] 等平衡段。 这些正是所有计数均为偶数的子数组。

 ## 边缘情况

 对于像[7]这样的单元素数组，前缀状态是{0, h7}。 没有前缀重复，因此不存在全偶子数组。 子数组总数为 1，因此答案为 1。算法正确地对其进行了计数，因为前缀哈希会立即更改并且从不重复。 

对于像 [5, 5, 5, 5] 这样的完全相同的数组，前缀哈希在 0 和 h5 之间交替。 状态 0 出现在索引 0、2、4 处，产生多个重复的前缀匹配。 这些与计数为偶数的子数组完全对应，例如 [1,2] 样式配对取消。 减法步骤精确地删除了这些间隔，只留下那些至少有一个奇数计数的间隔。
