 ## 解决方案
 ## 问题理解

 我们得到了一个选区列表，每个选区仅由一个整数描述，表示它包含多少票。 在每个选区中，两名候选人之一赢得该选区的所有选票，因此每个选区的行为就像一个“区块”，将其全部价值贡献给一方或另一方。 

隐藏的问题不是模拟选举。 相反，它询问这些投票块的结构是否足够严格，以至于任何将它们分成两组的合法方式都会导致独特的整体结果模式。 同样，我们想知道是否存在两种不同的符号分配，一种分配对应于“候选人 A 赢得该选区”，另一种分配对应于“候选人 B 赢得该选区”，使得所得加权和在所有合法变换下都表现得无法区分。 声明中的例子暗示了这一点：不同的符号分配可以产生相同的总体效果，这意味着确定选举结果的模糊性。 

输入大小最多为 100000 个值，每个值大到 2^31 − 1。这立即排除了任何尝试枚举子集或比较所有可能分区的方法。 即使是对选区对进行二次方法也会太慢。 该解必须基本上是线性的或线性算数的。 

当许多值相等或小值可以由较大值组成时，就会出现一个微妙的问题。 在这些情况下，不同的符号分配可能会以意想不到的方式相互抵消。 例如，如果数组包含像 1、2、3、4 这样的值，我们可以通过多种方式组合符号来创建相等的总计，这对应于歧义。 “不同的数字保证唯一性”的天真的直觉失败了，因为符号上的线性组合仍然可能发生冲突。 

另一个边缘情况是所有值都是 2 的幂或遵循严格递增的结构。 在这种情况下，唯一性通常成立，但前提是每个值与所有先前值的总和相比足够大。 否则，较早的值可以通过取消来模拟较晚的值。 

## 方法

 强力解释将尝试检查是否存在产生相同选举结果的两个不同的符号分配。 考虑它的一种方法是考虑选区的所有子集并计算所有可能的签名和。 如果每个子集都产生不同的值，那么系统就是明确的。 然而，这立即导致 2^n 种可能性，即使 n = 40 这也已经是不可行的。 

另一种尝试是对值进行排序并尝试使用动态编程来检测子集和之间的冲突。 这仍然会爆炸，因为总和范围很大，高达 n * 2^31，并且 DP over sums 在时间或内存上都不可行。 

关键的见解是将问题重新解释为多重集子集和的唯一性条件，其中每个元素都可以使用加号或减号。 这相当于询问该集合是否在非常有限的意义上“与 {−1, 0, 1} 中的系数线性无关”。 关键的观察是，当较小元素的某些子集可以组合起来匹配较大元素时，就会出现歧义。 

一旦我们对数组进行排序，条件就大大简化了。 如果在任何时候最大的剩余元素并不严格大于迄今为止考虑的所有较小元素的总和，那么我们可以构造两个冲突的不同符号分配。 相反，如果每个元素都严格大于所有先前元素的总和，则每个元素都会引入一个新的尺度，该尺度无法通过早期元素的组合来复制，从而确保唯一性。 

这将问题转化为排序后的贪婪线性扫描。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解符号分配 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | 排序贪婪前缀和检查 | O(n log n) | O(n log n) | O(1) 额外 | 已接受 |

 ## 算法演练

 ### 算法演练

 1. 按非降序对数组进行排序，这样我们总是会考虑较小的值是否可以构建较大的值。 排序是必要的，因为关键属性取决于较小元素的累积可达性。 
2. 将运行总和变量初始化为 0。该总和表示可以使用已处理元素的组合构造的最大值。 
3. 从最小元素到最大元素迭代排序数组。 
4. 对于每个元素ai，检查ai 是否小于或等于当前运行总和。 如果这个条件成立，我们立即得出结论，存在歧义并停止。 
5. 如果 ai 严格大于当前总和，我们通过添加 ai 来更新运行总和，因为该元素扩展了可构造值的范围。 
6. 如果循环完成而没有触发失败条件，我们得出结论，符号分配之间不可能发生冲突。 

### 为什么它有效

 在每一步中，运行总和代表可以使用先前处理的元素的有符号组合生成的完整值范围。 如果一个新元素至多是这个和，则意味着它可以表示为先前元素与适当符号的组合，因此它不会引入新的独立方向。 这意味着两个不同的符号分配可以产生相同的总体效果，从而破坏唯一性。 如果每个新元素都超出了当前的可达范围，则它无法被较早的元素复制，因此每个元素都会贡献一个新的独立比例，从而防止整个集合中发生任何基于取消的冲突。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    a.sort()

    s = 0
    for x in a:
        if x <= s:
            print("NO")
            return
        s += x

    print("YES")

if __name__ == "__main__":
    solve()
```其实现完全遵循贪心构造。 排序确保我们始终以递增的数量级测试可行性。 运行总和`s`跟踪先前元素已经可以形成的内容。 当我们遇到不超过此边界的元素时，我们会检测到可表示性中的潜在冲突并提前终止。 

一个常见的实施错误是使用`<`而不是`<=`。 相等一定会失败，因为等于当前总和的元素可以通过早期元素的有符号组合精确复制，从而产生歧义。 另一个微妙之处是我们只维护一个总和； 无需分别跟踪正范围和负范围，因为对称性隐含在构造中。

 ## 边缘情况

 最小的边缘情况是单个选区。 有输入`1`，任何值都会形成一个独特的系统，因为无法进行比较。 该算法将 sum 初始化为零，如果第一个值是正数，则立即接受该值。 

用于输入`2\n1 1`, 排序产量`[1, 1]`。 第一个元素将 sum 设置为 1。第二个元素等于当前总和，从而触发失败条件。 这反映了这样一个事实：两个相同的块可以在双方之间交换，而不改变整体结果，从而产生歧义。 

用于输入`3\n1 2 4`，运行总和演变为 1, 3, 7。每个元素都严格大于其之前的总和，因此没有元素可以由较早的元素表示。 该算法正确地确认了唯一性。 

用于输入`5\n1 2 3 8 16`，处理1、2、3后前缀和达到6。下一个元素8仍然是安全的，但16最终以仍然保持严格增长的方式超过总结构； 但是，如果我们稍微修改以包含 6 而不是 8，则会立即出现故障，因为 6 等于 1+2+3，这演示了即使值不相等，子集可组合性也会破坏唯一性。
