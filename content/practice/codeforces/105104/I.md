 ## 解决方案
 ## 问题理解

 我们给出了几个测试用例，每个测试用例都包含从 0 到 n − 1 的数字排列。对于此排列的任何连续段，我们查看未出现在该段内的最小非负整数，即该段的 mex。 对于从 0 到 n 的每个可能值 k，我们必须计算有多少个子数组的 mex 完全等于 k。 

测试用例的输出是一个由 n + 1 个数字组成的序列，其中第 k 个数字表示有多少个间隔产生等于 k ​​的 mex。 

这些约束意味着所有测试用例的总长度最多为 10^6。 在最坏的情况下，任何每个测试用例都是二次的解决方案都将涉及大约 10^12 次操作，这远远超出了两秒内可以运行的范围。 即使每个测试用例的 O(n log n) 方法也会有风险，除非它非常严格。 这推动我们整体上采用 O(n) 或摊销 O(n) 解决方案。 

当对 mex 进行本地推理而不考虑排列的全局结构时，会出现一个微妙的失败案例。 

如果尝试独立计算每个子数组的 mex，即使使用滑动窗口，更新的成本也太高了。 

另一个常见的错误是假设对 mex k 有贡献的区间的行为与大于 k 的值无关，但对于 mex 来说，可以安全地忽略大于 k 的值，而值 k 本身是唯一的阻碍因素。 这种不对称性是关键的结构特性。 

## 方法

 直接方法考虑每个区间 [i, j]，通过扫描或维护频率结构来计算其 mex，并增加该 mex 值的答案。 这是正确的，因为它明确遵循定义，但每个间隔需要 O(n) 工作来维护 mex，导致 O(n^3) 或最多 O(n^2) 优化，这对于 n 高达 10^6 的总体来说太慢了。 

关键的观察结果来自于对 mex 等于 k ​​的段的含义的重新表述。 这样的段必须包含从 0 到 k − 1 的每个数字，并且不能包含 k。 由于我们正在处理排列，因此每个值只出现一次，因此每个数字都有固定的位置。 

对于固定的k，0到k-1的位置定义了强制覆盖段。 任何有效区间都必须包含所有这些位置，因此其左端点必须位于其中的最小位置或之前，而其右端点必须位于其中的最大位置或之后。 同时区间必须避开k的位置。 

这将每个 k 的问题减少为计算有多少个间隔覆盖固定段，同时避免单个禁止点。 如果我们增量地维护有关最小和最大位置的前缀信息，则可以在每 k 恒定时间内评估该结构。 

暴力破解之所以有效，是因为它显式地检查每个间隔，但它会失败，因为重新计算 mex 的成本很高。 mex 约束仅依赖于小值位置的观察结果将问题转化为一条线上的几何计数任务。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) 到 O(n^3) | O(1) 到 O(n) | 太慢了 |
 | 最佳 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先存储排列中每个值的位置，以便 pos[x] 给出值 x 的索引。 

当 k = 0 时，我们计算 mex 为 0 的子数组。这意味着该子数组不能包含 0。每个排除 0 位置的子数组都是有效的。 子数组总数为n(n + 1)/2，无效的是包含pos[0]的子数组，可以通过选择该位置左侧的左端点和右侧的右端点来计数。

对于 k ≥ 1，我们在增加 k 的同时维持两个运行值：值 0 到 k − 1 之间的最小和最大位置。这些定义了必须完全包含在任何有效区间中的 обязатель 段。 

对于每个 k，我们进行如下操作。 

1. 使用 pos[k − 1] 更新当前最小和最大位置。 这会跟踪包含所有小于 k 的值的最小段。 
2.设L为0至k-1中的最小位置，R为0至k-1中的最大位置。 
3.设x为pos[k]。 如果 x 位于 [L, R] 内部，则每个同时包含 L 和 R 的区间将自动包含 k，因此没有区间可以恰好包含 mex k，且答案为零。 
4. 否则，对 mex k 有贡献的区间必须覆盖 [L, R] 并且必须避免 x。 无限制覆盖 [L, R] 的区间数量为左端点​​的 L 个选择和右端点的 (n − R + 1) 个选择。 
5. 从这些中减去也包含 x 的区间。 由于 x 在 [L, R] 之外，因此有效交集区域变为开始不晚于 min(L, x) 且结束不早于 max(R, x) 的区间，为左端点提供 min(L, x) 选择，为右端点提供 (n − max(R, x) + 1) 选择。 
6. 最终的答案是这两个计数之间的差值。 

对于 k = n，仅当区间包含从 0 到 n − 1 的所有数字（即整个数组）时，mex 才等于 n，因此答案为 1。 

### 为什么它有效

 在每个 k 处，必须出现在有效区间中的值集合正是集合 {0, 1, ..., k − 1}。 在排列中，每个值都对应一个唯一的位置，因此区间的可行性仅取决于它是否覆盖了这些所需元素的极端位置。 覆盖这些极值的任何区间都会自动包含所有必需的值。 mex 成为 k 的唯一障碍是 k 本身的存在，并且由于 k 的位置也是唯一的，因此它充当单个禁止点。 这将条件简化为简单的区间包含约束，因此每个有效区间都只计算一次，并且不包含无效区间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []

    for _ in range(t):
        n = int(input())
        p = list(map(int, input().split()))

        pos = [0] * n
        for i, v in enumerate(p):
            pos[v] = i

        total = n * (n + 1) // 2

        ans = [0] * (n + 1)

        x0 = pos[0]
        left = x0
        right = x0

        ans[0] = total - (x0 + 1) * (n - x0)

        for k in range(1, n):
            pk = pos[k - 1]
            if pk < left:
                left = pk
            if pk > right:
                right = pk

            x = pos[k]

            if left <= x <= right:
                ans[k] = 0
                continue

            base = left * (n - right)

            lo = min(left, x)
            hi = max(right, x)
            bad = lo * (n - hi)

            ans[k] = base - bad

        ans[n] = 1

        out.append(" ".join(map(str, ans)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现首先将排列转换为位置数组，以便可以在 O(1) 内访问每个值。 这是至关重要的，因为算法在增加 k 的同时重复查询值的位置。 

对于 k = 0，我们计算所有子数组，并使用左右选择的直接组合计数减去包含 0 位置的子数组。 

对于 k ≥ 1，我们维护一个动态区间 [left, right]，它始终表示从 0 到 k − 1 的值的极端位置。当扩展到 k 时，我们更新这个范围，然后评估 pos[k] 是否位于其中。 如果是，我们立即分配零，因为覆盖所需元素的每个区间都必须包含 k。 

否则，我们使用相同的端点计数逻辑计算覆盖 [left, right] 的区间数，并减去那些也包含 pos[k] 的区间数。 

最后一种情况 k = n 是单独处理的，因为它始终对应于完整数组。 

## 工作示例

 考虑排列 [0, 1, 2, 3]。

对于 k = 2 和 k = 3，相同的逻辑进行扩展，每次扩展所需的段。 

现在考虑一个更有趣的排列 [2, 0, 1, 3]。 

对于 k = 1，我们只需要值 0，因此 left = right = pos[0] = 1。由于 pos[1] = 2 位于右侧，因此有效间隔必须覆盖位置 1 但避免 2。

 ## 边缘情况

 当最小元素 0 位于数组的一端时，k = 0 的贡献变得最大，因为几乎所有间隔都避免它或以可预测的方式包含它。 计算仅依赖于其索引，因此即使像 0 或 n − 1 这样的边界位置也可以通过相同的组合公式正确处理，无需特殊分支。 

当 pos[k] 恰好位于当前 [L, R] 内部时，算法会正确返回零，因为覆盖所有所需元素的任何区间都不可避免地包含 k。 这可以避免对满足覆盖范围但违反 mex 条件的间隔进行过度计数。 

当排列已经排序时，每个新的 k 都会逐渐扩大间隔，产生一组高度结构化的答案，其中只有 k = 0 和 k = 1 才会产生重要的计数。 [L, R] 的增量维护可确保在线性时间内处理这种退化情况，而无需重新计算。
