 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 每个测试用例描述了放置在一条线上的一组点。 每个点都有一个坐标和附加的附加值。 我们需要构建一条从点 1 开始，到点 n 结束的路线，并且对每个点恰好访问一次。 路线的总成本是连续访问点之间的成对转移成本之和。 

两点之间的转移成本仅取决于它们的坐标和值，但仅在外观上不对称。 如果我们重写表达式，成本就会简化为更加结构化的形式。 对于两个点 i 和 j，在它们之间移动总是会贡献变换坐标的绝对差，其中每个点 i 可以用 yi = xi + vi 表示。 转移成本变为 |yi − yj|。 

因此，问题简化为在一条线上的 n 个点上找到一条最小成本哈密顿路径，其中距离是 yi 值的绝对差，并附加约束，即该路径必须从索引 1 开始并在索引 n 结束。 我们还需要计算有多少不同的访问订单达到了这个最低成本。 

所有测试用例的约束总规模都很小，所有用例的 n 总和以 5,000 为界。 这强烈表明每个测试用例采用 O(n²) 甚至 O(n log n) 方法就足够了，但任何超过 n 的三次或指数方法都会太慢。 

一种简单的方法会尝试尝试 n 个点的所有排列，计算每个点的成本，然后选择最佳的。 这立即是不可行的，因为 n 最多为 5000 使得 n! 排列如天文数字般巨大。 

一种更微妙的失败模式出现在总是选择最近的未访问点的贪婪方法中。 虽然这在某些度量 TSP 变体中有效，但在这里可能会失败，因为全局结构比局部邻近性更重要。 

另一个常见的陷阱是忘记成本仅取决于 yi = xi + vi。 任何试图独立地直接使用 (xi, vi) 的解决方案都会使结构过于复杂并错过关键的减少。 

## 方法

 第一步是简化成本函数。 观察定义，i 和 j 之间的成本变为 xi + vi 和 xj + vj 的绝对差。 这将问题转换为每个节点单个值的一维几何问题。 

一旦完成此减少，我们在数轴上留下 n 个点，每个点都有坐标 yi，并且我们需要一条从固定起始节点 (1) 到固定结束节点 (n) 的哈密顿路径，从而最小化总行进距离。 

如果没有固定的端点，最佳路径就很简单。 按 yi 对所有点进行排序并从一个极端走到另一个极端的总成本等于 max(yi) − min(yi)，因为排序遍历中的每条边都恰好贡献了连续点之间的间隙。 

之所以会出现这种情况，是因为我们被迫从点 1 开始，到点 n 结束，而该点可能位于排序顺序中的任何位置。 这迫使我们可能会穿越这条线，走向一个极端，然后扫向另一个极端。 

关键的观察是任何最佳路径仍然必须覆盖整个区间 [min(yi), max(yi)]。 唯一的灵活性是我们访问相对于固定端点的两个极端的顺序。 有意义的策略只有两种：先到最左边的点再向右扫，或者先到最右边的点再向左扫。 如果端点正确对齐，每个策略都会生成一条从 1 开始到 n 结束的有效哈密顿路径。 

这减少了评估两个候选成本的解决方案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!) | O(n) | 太慢了 |
 | 优化线路减少 | O(n) | O(n) | 已接受 |

 ## 算法演练

### 最佳构造

 1. 计算每个点的 yi = xi + vi。 这将成本函数转换为直线上的纯绝对距离问题。 
2. 确定所有yi 中的最小值和最大值。 这些代表必须完全遍历的线段的端点。 
3. 设 s = y1 和 t = yn 为变换空间中的固定起点和终点。 
4. 计算覆盖整个区间的成本 max_y − min_y。 这是覆盖所有地点所需的最小不可避免的行程。 
5. 考虑两种将端点与区间极值对齐的方法。 一种选择是先从 s 到 min_y，然后扫描到 max_y，并在 t 结束。 另一种是先从 s 到 max_y，然后扫描到 min_y，最后在 t 结束。 
6. 添加因强制每个选项的开始和结束对齐而产生的额外成本：

 第一个选项添加 |s − min_y| + |t − max_y|。 

第二个选项添加 |s − max_y| + |t − min_y|。 
7. 取两个总成本中的最小值。 
8. 计算这两个选项中有多少个达到了最低成本。 如果两者相等，则两个方向都是有效的最优结构。 

### 为什么它有效

 将问题简化为线度量后，任何最优路径都必须遍历最小和最大 yi 之间的整个跨度。 由于绝对值的三角不等式，任何跳过并返回中间点的偏差只会增加总距离。 因此，每个最优解都对应于一种策略，该策略选择访问两个极端的顺序，然后单调地扫描排序的点。 由于在尊重固定端点的同时只有两种可能的访问极端的顺序，因此检查两者足以表征所有最佳解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    MOD = 998244353
    for _ in range(t):
        n = int(input())
        x = list(map(int, input().split()))
        v = list(map(int, input().split()))

        y = [x[i] + v[i] for i in range(n)]

        min_y = min(y)
        max_y = max(y)

        s = y[0]
        tval = y[n - 1]

        base = max_y - min_y

        cost1 = base + abs(s - min_y) + abs(tval - max_y)
        cost2 = base + abs(s - max_y) + abs(tval - min_y)

        best = min(cost1, cost2)

        cnt = 0
        if cost1 == best:
            cnt += 1
        if cost2 == best:
            cnt += 1

        print(best, cnt)

if __name__ == "__main__":
    solve()
```该代码首先执行关键转换 yi = xi + vi，从而线性化成本结构。 然后，它提取变换后坐标的全局极值，因为每个最优解必须完全覆盖该区间。 起点和终点被视为点 1 和 n 的变换值，因为路径约束是在索引上定义的，而不是在任意顶点上定义的。 

两个候选成本恰好对应于一条线上的两个可能的扫描方向。 该实现小心地避免重建实际路径，因为只需要成本和计数。 计数是通过直接比较两个候选表达式来完成的。 

一个微妙的点是，即使 s 或 t 等于间隔的端点，也必须考虑两个候选者。 在这些情况下，一个选项可能会退化，但另一个选项仍然为计算最佳结构提供有效的比较基线。

 ## 边缘情况

 当点 1 或点 n 已经位于变换后的坐标集的任一极端时，两个候选策略之一会折叠为直线扫描。 在这种情况下，公式仍然可以正确计算，因为绝对偏差项之一变为零。 

如果两个端点与两个极值一致，则两个候选成本变得相同。 该算法正确地计算了两个最佳结构，对应于两个可能的扫描方向，即使两者产生相同的成本结构。 

当所有 yi 值相等时，min_y 等于 max_y，因此基础跨度为零。 两个候选值也都评估为零，并且最佳序列的数量变成两个，反映出任何方向都是有效的，但两种结构都崩溃为等效的零成本遍历。
