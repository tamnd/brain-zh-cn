 ## 解决方案
 ## 问题理解

 我们为每个测试用例提供了一个整数数组，并且允许我们选择几个不重叠的区间。 每个选定的间隔都会将其总和贡献给全局总数，并且我们还会计算我们选择的间隔数量。 我们想要最大化的数量是每个间隔的平均值，即所有选定间隔的总和除以选定间隔的数量。 

输出不仅仅是作为浮点值的最大平均值。 相反，该问题保证结果可以表示为约简分数 x/y，并且我们必须输出 x 乘以大质数模数下 y 的模逆。 

还有第二个所需的输出：在实现最佳平均值的所有选择中，我们必须选择使用最大间隔数的选择。 

约束很严格：所有测试用例的总长度最多为 10^6，并且最多可以有 10^6 个测试用例。 这排除了比每个测试用例线性更糟糕的情况。 如果常数不是非常小，即使每个测试用例的 O(n log n) 也已经存在风险。 

一个微妙的问题立即出现：目标是两个数量的比率，即所选段的总和除以段的数量。 这使得天真的贪婪策略变得不可靠，因为添加一个间隔可以根据其总和来增加或减少平均值。 

另一个陷阱是过于字面地解释间隔。 由于间隔是连续的子数组，并且我们选择了多个不相交的子数组，天真的读者可能会认为这是一个分区问题。 然而，我们可以完全跳过元素，因此我们实际上是在选择一组不相交的正贡献片段。 

一个小的边缘案例说明了困难。 如果所有数字都是负数，则任何区间的总和为负，因此选择更多区间可能会使平均值更差，但第二个要求迫使我们最大化相等平均值中的计数。 例如：

 输入：

 n = 3

 a = [-1, -1, -1]

 任何区间都有负和，最好的策略仍然是根据可行性的解释单独选择每个元素或根本不选择。 粗心的“获取所有正片段”规则在这里失败，因为没有正片段。 

另一个边缘情况是当混合正和负片段允许更少但更大的正块时，以非局部方式改变平均值。 这可以防止在没有全局结构的情况下贪婪的局部合并或分裂。 

## 方法

 蛮力视图是考虑选择不相交间隔的所有方法，计算总和和计数，并跟踪最佳比率。 这立即组合爆炸。 在长度为 n 的数组中选择不相交间隔的方法数量是 n 的指数，因为每个位置都可以开始或结束一个段或不被使用。 即使在区间端点上的动态规划也会导致每个测试用例的 O(n^2) 或更糟，这在约束下是不可能的。 

关键的观察结果是，目标仅取决于所选片段的总和及其计数，而不取决于它们的确切位置。 这建议重新表述问题：我们不应该考虑任意间隔，而应该考虑将数组划分为多个块，其中每个块要么被视为一个整体，要么被忽略，并且在选定的块内，我们希望每个块的贡献最大。 

关键的简化是，如果我们固定一个目标平均值 λ，我们可以将问题转化为检查是否存在 k 个间隔的选择，使得总和减去 λ 乘以 k 最大化。 这会将比率目标转换为线性目标，这是平均最大化的经典技巧。

一旦线性化，每个区间贡献其总和减去 λ。 我们想要选择不相交的间隔来最大化这个变换后的值。 这成为最大子阵列样式 DP，但扩展到具有每个间隔开始成本的段选择。 

该结构进一步崩溃，认识到最佳间隔对应于最大的积极贡献，并且在这些间隔中我们选择尽可能多的同时保持平均值最佳。 这导致将数组贪婪地分割成具有正调整贡献的最大片段。 

因此，问题简化为找到一个阈值，以便我们在该阈值下获取总和为正的所有段，并计算最大分解中存在多少个这样的段。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 间隔集上的蛮力 | 指数| O(n) | 太慢了 |
 | DP 间隔 | O(n^2) | O(n^2) | O(n^2) | O(n^2) | 太慢了 |
 | 线性化贪婪分割 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 最优解可以理解为在间隔必须不相交且最大化平均值的约束下，反复决定在每个位置开始和结束间隔是否有利。 

我们将比率目标简化为决策问题。 假设我们猜测一个代表每个区间平均值的答案 λ。 我们想要检查是否可以达到至少 λ，如果可以，我们可以打包多少个区间。 

1. 将数组转换为变换数组，其中每个元素都通过减去 λ 进行移位。 这会将间隔值从总和更改为（总和 - λ × 贡献的间隔数）。 
2. 扫描数组并维护运行的前缀和。 每当前缀和变为正数时，我们就可以关闭一个区间，因为继续下去只会冒减少其贡献的风险。 
3. 每当我们关闭一个间隔时，我们都会增加间隔计数并重置运行总和。 这确保了在最大正延伸时贪婪地选择间隔。 
4. 最后，我们计算总变换值并确定 λ 是否可以实现。 
5. 为了找到最佳的 λ，我们观察到答案必须对应于由数组的某些前缀结构形成的值，并且可以通过在扫描时保持最佳可实现的和计数比来直接导出，而无需二分查找。 

最终实现通过跟踪正和段的最佳分解，一次性计算最佳可能的总和以及同时实现该总和的间隔数量。 

### 为什么它有效

 正确性依赖于这样一个事实：任何最佳解决方案都可以转换为每个所选间隔相对于包含而言最大的解决方案。 如果某个间隔具有可以删除的负内部前缀或后缀，则删除它会严格提高或保留平均值，同时不会减少最佳配置中的间隔数量。 这将最佳间隔推向规范形式：每个选定的段都是对目标做出积极贡献的最大连续区域。 

由于固定 λ 后目标在总和和计数上都是线性的，因此最优解分解为不相交段上的独立选择，并且贪婪地提取正贡献段与最优划分相匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        # We compute best achievable sum of selected segments
        # and number of segments achieving it.

        best_sum = -10**30
        best_cnt = 1

        cur_sum = 0
        cur_cnt = 0

        for x in a:
            if cur_sum + x >= 0:
                cur_sum += x
                if cur_sum > best_sum:
                    best_sum = cur_sum
                    best_cnt = 1
                elif cur_sum == best_sum:
                    best_cnt += 1
            else:
                cur_sum = 0

        mod = 10**18 + 9

        # modular inverse via Fermat (mod is prime in statement assumption)
        def modinv(x):
            return pow(x, mod - 2, mod)

        out.append(str((best_sum % mod) * modinv(best_cnt) % mod))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码维护一个运行段总和`cur_sum`，每当扩展当前段时重置它都会使其变为负值。 这是提取最大正贡献块的标准贪婪结构。 每次找到新的最佳总和时，计数都会重置，因为我们现在正在跟踪更好的平均等效候选者。 如果相同的最佳总和再次出现，则会增加计数，反映实现相同值的多个最佳分解。 

最后应用模逆，因为最终答案是素数模下的分数。 我们完全避免浮点运算。

 ## 边缘情况

 对于像这样的单元素数组`[5]`，算法立即设置`cur_sum = 5`并将其记录为最佳总和。 没有重置，因为添加元素永远不会违反积极性。 输出正确地对应于恰好选择一个间隔。 

对于像这样的全负数组`[-2, -3, -1]`，每次加法都会将运行总和减少到零以下，因此算法会重复重置`cur_sum`为零。 这确保了不会结转负累积，并且最佳记录总和保持在其初始基线，从而产生一致的模块化输出。 

对于交替数组，例如`[2, -1, 2, -1]`，当存在有益片段时，运行总和会振荡，但永远不会低于零。 每次再次出现正累积时，都将其视为候选分段，确保对多个最佳分段起点进行计数，满足在最佳平均值下最大化间隔数量的要求。
