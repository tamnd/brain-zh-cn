 ## 解决方案
 ## 问题理解

 我们有一个具有多行和多列的矩形网格。 该网格中的每个单元格要么包含一个气球，要么包含一个炸弹。 两个代理在网格中逐行移动，并在每一行中选择该行中的位置来收集气球。 炸弹的存在限制了行中的哪些部分可以在某些转换中安全使用。 

任务是计算在动态移动规则下可以在所有行中收集的气球的最大数量，该规则取决于我们是“保持”跨行固定位置还是在炸弹包围的安全间隔内“移动”活动位置。 

每行的行为就像一条分段线：炸弹将行分成独立的连续安全间隔。 在每个间隔内，可以独立计算运动和贡献，但跨行的过渡必须尊重炸弹引起的结构。 

解决方案结构隐含的约束是网格尺寸足够大，以至于$O(nm)$或者$O(nm \log m)$预期的解决方案是，任何对列的每行二次方法都会太慢。 这立即排除了对连续行之间的所有位置对进行简单的重新计算。 

当一行被炸弹完全封锁（除了孤立的单元格之外）时，就会出现微妙的边缘情况。 在这种情况下，有效间隔长度可以缩小到单个位置，并且转换必须正确重置贡献，而不是向前推进无效范围。 

当炸弹位于边界时会出现另一种边缘情况。 例如，如果第一列或最后一列包含炸弹，则安全间隔边界会发生变化，并且计算这些边界时的任何差一错误都会导致不正确的 DP 转换。 

## 方法

 暴力方法将尝试模拟两个代理跨行移动的所有可能方式，同时保持所有有效的位置配置。 这很快就会爆炸，因为即使我们每行只跟踪一个“枢轴”位置，每个状态都会转变为$O(m)$下一行的可能性。 超过$n$行这变成$O(nm^2)$，这对于大型网格来说是不可行的。 

关键的观察是我们实际上不需要明确地跟踪两个代理。 在任何时刻，一个位置都可以被视为锚定过渡的枢轴，而另一个位置则基于其周围的最佳可能部分做出贡献。 这会将状态空间折叠到每行一个位置，产生 DP 状态$dp[i][j]$，表示枢轴位于列时的最佳分数$j$在行中$i$。 

从这里开始，出现了两种类型的转变。 首先，我们可以保持枢轴固定并垂直延伸，从包含的行中的最大安全水平段收集贡献$j$。 其次，我们可以切换枢轴，这会根据我们在无炸弹段内水平移动的距离引入过渡。 第二个转换的结构仅依赖于线性表达式$j$，它允许使用前缀/后缀最大值进行优化。 

暴力破解很慢，因为它会重新计算每对列的转换。 优化的解决方案之所以有效，是因为在无炸弹区间内，对最大化形式表达式的贡献减少了$dp[i-1][k] - k + j$，可以使用运行最大值来维持。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(nm^2)$|$O(nm)$| 太慢了 |
 | 具有前缀最大值的优化 DP |$O(nm)$|$O(m)$| 已接受 |

 ## 算法演练

 我们逐行处理网格，在列上维护 DP 数组。 

1. 初始化第一行的 DP 数组。 对于每一列，单独计算该单元格的最佳贡献，将其视为第一个主元。 这设置了不存在先前行约束的基本状态。 
2. 对于每个后续行，首先确定安全间隔。 我们扫描行并计算每一列$j$，左边和右边最近的炸弹。 这定义了一个段$[l, r]$这样任何有效的运动涉及$j$必须保持在这个区间内。 
3. 计算“垂直延伸”过渡。 如果我们将枢轴保持在柱上$j$，我们扩展之前的值并添加周围最大的连续无炸弹段的贡献$j$。 这仅取决于步骤 2 中确定的间隔长度。 
4. 计算每个安全段内从左到右的转换。 我们维持运行最大值$dp[i-1][k] - k$。 当我们移动时$j$在段内从左到右，我们更新这个最大值并计算表格的候选值$max(dp[i-1][k] - k) + j$。 这捕获了我们切换枢轴位置的情况。 
5. 类似地计算从右到左的转换。 这确保了最好的前任位于右侧的情况$j$也被覆盖，再次使用线性化形式$dp[i-1][k] + k + j$适当转变。 
6. 处理完两个方向后，最终确定$dp[i][j]$作为垂直延伸和枢轴开关转换的最大值。 
7. 对所有行重复此操作，并返回最后 DP 行中的最大值。 

### 为什么它有效

 DP 状态将所有配置压缩为每行一个活动列，因为任何第二个位置都可以解释为通过炸弹界定的连续间隔进行贡献。 在无炸弹段内，所有有效转换都简化为列索引中的线性函数。 这确保了转换的最佳选择始终位于极值，或者可以通过前缀或后缀最大值来捕获。 由于每行都是根据这些最大值独立处理的，因此不会丢失全局结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    dp = [0] * m

    for i in range(n):
        left_bomb = [-1] * m
        right_bomb = [m] * m

        last = -1
        for j in range(m):
            if grid[i][j] == '#':
                last = j
            left_bomb[j] = last

        last = m
        for j in range(m - 1, -1, -1):
            if grid[i][j] == '#':
                last = j
            right_bomb[j] = last

        new_dp = [0] * m

        j = 0
        while j < m:
            if grid[i][j] == '#':
                j += 1
                continue

            l = j
            while l > 0 and grid[i][l - 1] != '#':
                l -= 1
            r = j
            while r < m - 1 and grid[i][r + 1] != '#':
                r += 1

            best = 0

            max_left = -10**18
            for k in range(l, r + 1):
                max_left = max(max_left, dp[k] - k)
                best = max(best, max_left + k + (r - k + 1))

            max_right = -10**18
            for k in range(r, l - 1, -1):
                max_right = max(max_right, dp[k] + k)
                best = max(best, max_right - k + (k - l + 1))

            for k in range(l, r + 1):
                best = max(best, dp[k] + max(k - l + 1, r - k + 1))

            for k in range(l, r + 1):
                new_dp[k] = best

            j = r + 1

        dp = new_dp

    print(max(dp))

if __name__ == "__main__":
    solve()
```实现遵循逐行 DP 结构。 数组`left_bomb`和`right_bomb`被计算但主要用于帮助理解分段； 为了简单起见，实际的分割是按时间间隔在本地重新计算的。 提取每个连续的无炸弹片段，并在其中评估三个贡献：从左到右的枢轴切换、从右到左的枢轴切换和垂直延伸。 

一个微妙的实现细节是，我们通过从每个未访问的单元扩展来直接重新计算安全段边界。 这可以避免合并间隔中的错误，并确保即使存在多个孤立段也能保持正确性。 

线性最大值计算`dp[k] - k`和`dp[k] + k`直接对应于转换后的转移公式，确保有效地考虑每个候选主元。 

## 工作示例

 考虑一个有两行且没有炸弹的小网格。 这是每个单元都连接的最简单的情况。 

| 行| 细分 | 最大 (dp[k] - k) | 最大 (dp[k] + k) | 结果 |

 这表明，在没有炸弹的情况下，解决方案的行为类似于标准间隔 DP，其中所有位置均可到达，并且最大值会全局传播。 

现在考虑用炸弹将其分开的一行：

 行：`..#..`区间分解得到`[0,1]`和`[3,4]`。 

| 细分 | 行为 |
 | --- | --- |
 | [0,1]| DP 独立计算 |
 | [3,4]| DP 独立计算 |

 这表明没有过渡跨越炸弹边界，从而保持了独立段处理的正确性。

 ## 边缘情况

 完全轰炸的行测试 DP 是否正确避免无效转换。 在这种情况下，每一列都变得无法访问，并且任何不重置或隔离段的实现都将错误地传播值。 

第 0 列或 m-1 列的边界炸弹测试区间扩展中的差一误差。 正确的行为是安全间隔适当缩小并且不会尝试延伸到网格之外。 

跨越整行的单个长段检查前缀/后缀最大值是否正确捕获交叉位置转换而无需重复计数。
