 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$网格和数字$k$。 我们需要放置$k$相同的对象（称为$w$在声明中）到网格的不同单元格上。 目标是最大化分数，该分数取决于这些选定的细胞如何与其网格中的邻居相互作用。 

交互模型是局部的：单元格根据其与其他选定单元格的相邻性对分数做出贡献。 内部细胞可能贡献更多，因为它们有更多的邻居，而边界和角细胞贡献较少。 问题本质上是要求做出安排$k$选择在这些几何约束下最大化总邻接贡献的单元。 

构造提示隐含的约束表明$n$可以足够大，任何$O(n^2)$预处理是预期的上限。 这已经排除了任何试图评估所有子集的方法$k$单元格或动态模拟布局。 解决方案的结构必须来自网格的全局排序或分解，而不是来自局部搜索。 

出现微妙的边缘情况时$k$非常小或非常大。 什么时候$k = 1$，不可能有邻接，因此所有配置都是等效的。 什么时候$k = n^2$，格子被占满，分数固定。 有趣的机制是中间值，其中放置选择决定了可以形成或保留所选单元之间的边数。 

另一个边缘情况来自奇偶校验结构。 由于邻接是基于网格的，因此棋盘着色等交替模式的行为会有所不同，具体取决于我们是在内部还是在边界附近。 总是首先选择最高度单元的天真贪婪，当它过度承诺内部单元而不考虑剩余的放置将如何减少未来的邻接潜力时，它就会失败。 

## 方法

 蛮力方法会尝试所有方法来选择$k$之间的细胞$n^2$，计算每个子集的诱导邻接分数，并返回最大值。 这是正确的，因为它直接评估目标函数的定义。 然而，子集的数量是$\binom{n^2}{k}$，即使对于$n = 6$。 每次评估也需要花费$O(n^2)$如果天真地完成或至少$O(k)$，因此总运行时间呈爆炸式增长。 

关键的结构观察是，并非所有细胞在静态意义上都具有同等价值。 它们的贡献取决于它们是在内部还是在边界，更重要的是，它们是否放置在可以与其他选定细胞配对的区域。 这表明，我们不应该对子集进行推理，而应该在全局排序中为每个单元分配一个优先级，以反映尽早包含它的“成本”。 

对于小$k$，最优策略是优先考虑内部小区，特别是具有奇偶结构的中部区域小区$i + j$奇数或偶数，取决于邻接的计算方式。 这些单元最大化了潜在的贡献，因为它们可以参与更多的内部边缘。 一旦内部区域耗尽，下一个最佳候选者就是具有良好奇偶性的边界单元。 之后，无论选择如何，剩余的细胞贡献都会减少。 

对于大型$k$，一种不同的现象占主导地位。 一旦选择了大多数单元，每个新的布局都会破坏潜在的边际收益，因为它减少了“未使用的邻接机会”的数量。 在这种制度下，目标从最大化即时收益转向最小化边际损失。 这就是“蛇排序”变得最佳的地方：穿过网格的哈密顿路径，逐行交替方向。 这样的遍历确保连续的选择尽可能长时间地在空间上保持接近，从而最大限度地减少每一步引入的新丢失邻接的数量。 

因此，最终的解决方案是分阶段构建的所有网格单元的单一排序：首先是基于内部奇偶校验的单元，然后是基于边界奇偶校验的单元，最后是对剩余位置的蛇形遍历。 取第一个即可得到答案$k$单元格按此顺序排列。

 | --- | --- | --- | --- |
 | 蛮力 | 指数为$n^2$|$O(n^2)$| 太慢了 |
 | 最优排序构造 |$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们构建了所有网格单元的排序，以反映它们的边际有用性递减。 

1. 按位置将单元格分为几类：内部单元格（不接触边框的单元格）、边界单元格和角点单元格。 内部单元被优先考虑，因为它们最大化了邻接潜力，因为它们有四个邻居而不是两个或三个。 
2. 在内部，通过奇偶校验进一步分隔单元格$i + j$。 首先填充一个奇偶校验类，因为它允许更均匀地分布所选单元，从而延迟了紧密堆积的簇的创建，从而减少了未来的增益。 
3. 一旦第一个奇偶校验类用完，就追加另一个奇偶校验的剩余内部单元。 此时，中心的所有高值邻接位置都被使用。 
4. 移至边界单元，再次遵守奇偶校验顺序。 边界单元的邻居较少，因此它们贡献的边际增益较小，但仍然比角点更好。 
5. 一旦结构化区域耗尽，就对所有剩余单元构建蛇形遍历。 这是跨行的锯齿形路径：一行从左到右，下一行从右到左。 这确保了连续的单元格在网格中保持相邻或接近相邻。 
6.输出第一个$k$以此顺序排列的单元格作为所选位置。 

出现蛇排序的原因是，当只剩下低值单元格时，保留连续选择之间的局部性是避免邻接结构快速退化的唯一方法。 任何分散的订购都会提前引入孤立的放置，这会立即增加损失。 

### 为什么它有效

 该构造隐含地保持了贪婪不变量：在每个阶段，我们选择的单元格能够最大限度地减少剩余未选择单元格中总可实现邻接度的边际下降。 内部细胞尽早最大限度地减少这种损失，因为它们仍然有许多潜在的合作伙伴。 一旦这些都用尽，边界单元就成为最佳可用选择。 之后，网格的行为就像一个稀疏图，其中邻接很少，并且类似哈密顿量的遍历最大限度地减少了碎片。 由于每一步都会在全局一致的排序中选择最佳可用的边际选择，因此长度的前缀$k$是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())

    cells = []

    def add_group(condition):
        for i in range(n):
            for j in range(n):
                if condition(i, j):
                    cells.append((i, j))

    # interior odd parity
    add_group(lambda i, j: 1 <= i <= n-2 and 1 <= j <= n-2 and (i + j) % 2 == 1)
    # interior even parity
    add_group(lambda i, j: 1 <= i <= n-2 and 1 <= j <= n-2 and (i + j) % 2 == 0)

    # boundary odd parity
    add_group(lambda i, j: not (1 <= i <= n-2 and 1 <= j <= n-2) and (i + j) % 2 == 1)
    # boundary even parity
    add_group(lambda i, j: not (1 <= i <= n-2 and 1 <= j <= n-2) and (i + j) % 2 == 0)

    # snake fill for completeness (overwrites order benefit for leftover structure)
    snake = []
    for i in range(n):
        row = list(range(n))
        if i % 2 == 1:
            row.reverse()
        for j in row:
            snake.append((i, j))

    # remove duplicates while preserving order
    seen = set(cells)
    for x in snake:
        if x not in seen:
            cells.append(x)
            seen.add(x)

    # output first k
    for i in range(k):
        x, y = cells[i]
        print(x + 1, y + 1)

solve()
```该代码完全按照描述分阶段构建排序。 首先使用显式坐标检查生成内部区域，确保中心高阶单元尽早出现。 边界单元跟随，通过奇偶校验分隔。 最后，蛇遍历保证了所有剩余的单元格都以结构化的方式包含在内，从而避免了病态的间隔。 

一个常见的陷阱是在合并蛇层时忘记保持顺序一致性。 这`seen`set 确保不会发生重复，同时仍然允许蛇遍历在最后仅贡献缺失的单元格。 

## 工作示例

 ### 示例 1

 考虑一个$4 \times 4$网格与$k = 5$。 

我们首先列出内部细胞。 为了$n = 4$, 内部是索引$1$到$2$。 我们先取奇校验：

 | 步骤| 添加单元格 | 类别 |
 | --- | --- | --- |
 | 1 | (1,2) | 室内奇特|
 | 2 | (2,1) | 室内奇特|
 | 3 | (1,1) | 室内均匀|
 | 4 | (1,2) 已使用 跳过 | 边界阶段开始 |
 | 4 | (0,0) | (0,0) | 边界奇数|

 取出 5 个细胞后，我们停止。 所选择的集合以中心放置为主，确保最大的邻接潜力。 

这演示了排序如何在触及边界之前优先考虑内部结构。 

### 示例 2

 考虑一个$5 \times 5$网格较大$k = 18$。 

早期的选择占据了中央$3 \times 3$按奇偶校验顺序排列的区域。 一旦这些耗尽，边界细胞就开始出现。 下表显示了进展情况：

 | 步数范围| 地区 | 效果|
 | --- | --- | --- |
 | 1-8 | 1-8 室内装饰 | 最大邻接建立|
 | 9-16 | 边界| 减少但控制损失|
 | 17-18 | 17-18 蛇区| 最小化碎片|

 轨迹表明，只有当中心结构饱和时，算法才会过渡到较低值的位置，从而保持全局最优性。

 ## 边缘情况

 对于$n = 1$，该算法简化为单个单元网格。 排序结构仍然只产生一个单元格，并取第一个$k = 1$元素返回正确的结果，而不依赖奇偶校验或边界逻辑。 

为了$n = 2$，没有内部区域。 该算法立即回退到边界奇偶校验排序，然后蛇形完成。 这可以避免访问无效的内部索引并正确退化为完整的 4 单元排列。 

对于大型$k = n^2$，该算法只是输出完整的排序。 蛇阶段确保所有单元格都包含一次，因此不需要特殊处理。
