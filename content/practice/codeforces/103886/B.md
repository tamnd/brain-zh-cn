 ## 解决方案
 ## 问题理解

 我们正在研究离散网格上的几何优化问题。 想象一个类似于 2D 教室的布局，其中一些单元包含“大厅监视器”，网格的外边界就像一堵硬墙，也限制了移动。 我们可以将一个名为 escape-scanner 3000 的设备放置在任何有效的网格单元上，并且对于每个放置，我们测量它距被检测到的距离有多远。 

对于任何选定的单元，检测风险由最近的障碍物决定，其中障碍物要么是放置在网格中某处的霍尔监视器，要么是网格本身的边界。 距离是使用标准几何距离公式计算的，我们关心的是从所选位置到任何障碍物的最小距离。 我们的目标是以最大化该最小距离的方式放置扫描仪。 

因此，输出是一个数字：通过将扫描仪最佳地放置在网格中的任何位置，可以实现最佳的“最安全距离”。 

尽管该声明提到了蛮力观点，但实际结构很重要：我们在所有网格位置上最大化到最近特征的距离函数，其中特征既是内部点（监视器）又是外部矩形边界。 

从复杂性的角度来看，预期的解决方案是对所有候选扫描仪位置进行明确的暴力破解，检查每个位置的所有监视器和边界。 如果我们假设一个大小为 h × k 和 n 个霍尔监视器的网格，则直接解决方案会评估 h·k 个候选位置，并为每个位置计算到 n 个监视器的距离以及恒定数量的边界检查。 这导致总工作量为 O(hkn)，这是可以接受的，因为约束条件很宽松。 

主要的边缘情况源于边界处理和大厅监视器的空置或极端分布。 如果粗心的实现忽略了边界作为有效的“最近障碍物”，它将高估边缘附近单元的距离。 例如，在没有监视器的 3×3 网格中，最佳位置是中心单元，距离应为 1（或 1.414，取决于度量解释），因为边界与边缘单元直接相邻。 仅检查监视器的简单解决方案会根据初始化错误地返回无穷大或零。 

另一个微妙的问题是距离对称性。 由于距离是欧几里德距离，因此忘记一致求平方根或混合平方值和非平方值将产生不正确的比较。 正确的方法必须与用于比较和最终输出的指标保持一致。 

## 方法

 暴力解决方案几乎已经嵌入到问题描述中。 我们考虑逃生扫描器 3000 的每个可能的位置，迭代所有网格单元，并且对于每个位置计算其到每个大厅监视器以及到网格边界的距离。 放置的值由最近的此类障碍物定义，因此我们采用所有监视器和边界点之间的最小距离。 最后，我们取所有展示位置的最大值。 

这是可行的，因为没有允许修剪的结构约束：每个单元都可能是最佳的，具体取决于监视器的放置位置。 正确性很简单，因为我们明确评估了每个可能候选者的定义。 

这变得有趣的原因纯粹是计算性的。 蛮力评估 h·k 放置，每个放置检查 n 个监视器加上恒定的边界检查，因此总工作量与 h·k·n 成正比。 在最坏的情况下，如果所有维度都很大，这仍然是可以管理的，只是因为问题明确指出边界足够宽松以允许它。 不存在会减少搜索空间的隐藏优化（例如凸性或单调性）。

关键的观察结果是，“逃逸距离”函数完全是局部的，并且不承认全局捷径：了解单元格值的唯一正确方法是与所有障碍物进行比较。 这意味着预期的解决方案不是渐进地降低复杂性，而是仔细地实现距离计算和边界处理。

 ## 算法演练

 1.迭代网格中的每个单元作为逃逸扫描器3000的候选位置。每个单元被独立评估，因为分数仅取决于距该固定点的距离。 
2. 对于每个候选单元，计算其到所有大厅监视器的距离。 我们保持跑步的最小距离，因为单元的安全性是由最近的障碍物而不是任何平均或总体措施来定义的。 
3. 对于同一候选单元格，还计算到网格最近边界的距离。 这是通过考虑单元格距顶部、底部、左侧和右侧边缘的距离，并将其转换为到最近的墙点的欧几里得距离来完成的。 
4. 取该像元的所有计算距离中的最小值。 该值表示特定放置的安全程度，因为检测是从最近的源进行的。 
5. 跟踪所有单元格中该最小距离的最大值。 这确保了我们在所有最坏情况的邻近度中选择最好的位置。 
6. 评估所有单元格后，输出找到的最大值。 

### 为什么它有效

 正确性来自于直接匹配问题定义：对于每个候选位置，我们精确计算问题定义的值，即到任何障碍物的最小距离。 由于我们详尽地评估所有可能的位置，因此这些正确计算的局部值的全局最大值必定是答案。 没有引入近似或启发式方法，因此不会跳过更好的放置位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    h, k, n = map(int, input().split())
    monitors = []
    for _ in range(n):
        x, y = map(int, input().split())
        monitors.append((x, y))

    def dist_to_boundary(x, y):
        # distance to nearest side (treated as Euclidean distance to border line)
        top = x
        bottom = h - 1 - x
        left = y
        right = k - 1 - y
        return min(top, bottom, left, right)

    best = 0.0

    for i in range(h):
        for j in range(k):
            best_here = float('inf')

            # distance to monitors
            for mx, my in monitors:
                dx = i - mx
                dy = j - my
                best_here = min(best_here, (dx * dx + dy * dy) ** 0.5)

            # distance to boundary
            best_here = min(best_here, dist_to_boundary(i, j))

            best = max(best, best_here)

    print(best)

if __name__ == "__main__":
    solve()
```该代码完全遵循暴力结构。 嵌套循环枚举每个可能的扫描仪位置。 对于每个位置，我们使用标准平方距离公式后跟平方根来计算到每个监视器的欧几里德距离。 我们还通过测量到任何边缘的最短轴对齐距离来计算到边界的距离。 

一个微妙的点是初始化`best_here`作为无穷大。 这确保了第一次障碍物比较始终设置有效的基线。 另一个重要的细节是，边界距离被视为直接轴对齐距离，而不是迭代边界点，这避免了不必要的开销。

 ## 边缘情况

 空格子的盒子是最具有欺骗性的。 如果没有大厅监视器，解决方案就完全减少到最大化与边界的距离。 对于 3×3 网格，中心单元给出距离 1，并且实现正确地捕获了这一点，因为监视器循环没有贡献任何内容，并且只有边界距离保持活动状态。 

第二种边缘情况是监视器恰好放置在边界相邻的单元上。 例如，在监视器位于 (0,1) 的 3×3 网格中，顶行变得极其受限。 该算法可以正确处理这个问题，因为监视器距离和边界距离都是统一评估的，并且最小值正确地捕获了主导约束。 

最后一个微妙的情况是多个监视器竞争时。 例如，位于对角的监视器迫使最佳位置朝向几何中心。 该算法自然地解决了这个问题，因为它总是在所有距离上取最小值，确保即使其他监视器距离较远，也不会忽略任何单个监视器。
