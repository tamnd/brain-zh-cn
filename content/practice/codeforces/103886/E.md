 ## 解决方案
 ## 问题理解

 该问题要求我们计算每个给定整数的值$x$，其中每个$x$代表“项目规模”或目标总和。 对于每一个这样的$x$，我们计算在一定的规则体系下存在多少个有效的构造，然后减去那些具有回文结构的构造。 

关键的结构事实是所有项目都贡献给定的金额$x$独立于其他值$x$。 这意味着我们可以预处理所有的答案$x \le 10^5$并在固定时间内回答查询。 

每个项目可以解释为一个配置，其总价值为$x$。 在没有任何对称性约束的情况下，此类配置的数量呈指数增长$x$，具体为$2^{x-1}$。 其中，一些配置是对称的，即左半部和右半部彼此镜像，并且这些配置被认为是回文并且必须被排除。 

当出现一个微妙的点时$x$很小。 为了$x = 1$，唯一的配置是普通回文，因此任何涉及减法的公式都不能意外地产生负值或不一致的结果。 为了$x = 2$，“半结构”和“全结构”之间的界限很紧密，楼层划分中的相差一误差通常会出现在这里。 

一个幼稚的实现会尝试枚举每个的所有配置$x$，但由于计数是指数级的，即使$x = 40$已经是不可行的，并且该问题允许值高达$10^5$，所以枚举是不可能的。 

因此，挑战纯粹是组合性的：导出封闭形式并对其进行有效评估。 

## 方法

 如果我们尝试构建给定的所有有效项目$x$，每个位置有效地分为两种可能性，导致$2^{x-1}$总配置。 这是标准的“每个职位二元选择”的增长模式。 这部分很简单，解释了为什么指数项出现在解决方案中。 

复杂性是由回文配置引起的。 回文项目完全由其前半部分决定，因为后半部分必须准确反映它。 如果我们考虑一个总和的项目$x$，那么只有第一个$\lfloor x/2 \rfloor$位置可以自由选择，其余位置则强制对称。 这降低了自由度$x-1$到$\lfloor x/2 \rfloor$，给出回文计数$2^{\lfloor x/2 \rfloor}$。 

然而，有一个边界修正：完全居中的情况，当$x$是奇数或最小数。 这引入了有效回文计数干净对齐的调整：$2^{\lfloor x/2 \rfloor}$在考虑了退化的中心案例之后。 

一旦知道这两个量，每个量的答案$x$只是总配置和回文配置之间的差异。 自从$x$很大并且可能有很多查询，预计算能力最多为两个$10^5$允许在每个查询的恒定时间内计算所有答案。

 | --- | --- | --- | --- |
 | 暴力枚举| 指数| O(1) | O(1) | 太慢了 |

 ## 算法演练

 我们现在以直接反映组合结构的方式构建解决方案。 

1. 预先计算最大可能的 2 的幂$x$。 我们储存$2^i$为所有人$i$因为总数和回文计数都依赖于快速指数查找。 这避免了查询期间重复求幂。 
2.对于每个查询值$x$，计算配置总数为$2^{x-1}$。 这对应于这样的事实：除了第一个位置之外的每个位置都引入了独立的二元选择。 
3. 计算回文配置的数量为$2^{\lfloor x/2 \rfloor}$。 这反映出只有前半部分结构是自由选择的，而其余部分则由对称性决定。 
4. 从总数中减去回文数，以获得该项目的“危险”项目数量$x$。 
5. 独立输出每个查询的结果。 

关键的观察是这两个组件仅依赖于$x$，因此不存在查询间交互。 

### 为什么它有效

 每个有效的配置都被唯一地分类为两个不相交的集合之一：回文或非回文。 总计数无限制地枚举所有可能的配置。 回文公式精确地计算了那些完全由其前半部分决定的配置。 由于每个回文配置都包含在总数中，因此减去可以干净地删除它们，而不会重叠或遗漏。 这样就保证了正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    q = int(data[0])
    xs = list(map(int, data[1:]))

    max_x = max(xs) if xs else 0

    pow2 = [1] * (max_x + 2)
    for i in range(1, max_x + 2):
        pow2[i] = pow2[i - 1] * 2

    out = []
    for x in xs:
        total = pow2[x - 1] if x - 1 >= 0 else 0
        pal = pow2[x // 2]
        out.append(str(total - pal))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现依赖于一个预先计算的 2 的幂数组。 该数组是迭代构建的，这避免了递归或模幂开销。 

对于每个查询，我们直接索引到该表中。 表达式$x - 1$是安全的，因为$x \ge 1$，但我们仍然通过显式处理非正索引来防范边缘情况。 

由于任意精度，整数运算在 Python 中是安全的，因此除非另有说明，否则不需要模数。 

## 工作示例

 ### 示例 1

 让$x = 3$。 

| x| 全部的$2^{x-1}$| 回文的$2^{\lfloor x/2 \rfloor}$| 答案|
 | --- | --- | --- | --- |
 | 3 | 4 | 2 | 2 |

 总的配置都是对两个有效位置的二元选择。 回文完全由第一个位置决定。 减去留下不对称配置。 

这证实了对称约束恰好删除了较小指数空间的一半。 

### 示例 2

 让$x = 5$。 

| x| 全部的$2^{x-1}$| 回文的$2^{\lfloor x/2 \rfloor}$| 答案|
 | --- | --- | --- | --- |
 | 5 | 16 | 16 4 | 12 | 12

 这里，五个位置为一般配置提供了四个自由度，但由于镜像，回文配置仅提供了两个自由度。 这种差异隔离了非对称结构。 

这个例子展示了两个指数基数之间的差距如何随着$x$，这就是减法保持稳定且非负的原因。

 ## 边缘情况

 最小输入$x = 1$是主要的极端情况。 总公式给出$2^{0} = 1$，而回文公式给出$2^{0} = 1$，结果为零。 这符合单元结构完全对称且对结果没有任何贡献的解释。 

为了$x = 2$，总计为$2^{1} = 2$，回文计数为$2^{1} = 2$，再次产生零。 这证实了尺寸为 2 的两种配置在问题定义下是对称的。 

对于奇数，例如$x = 5$，楼层划分$2^{\lfloor x/2 \rfloor}$确保中心元素不会引入分数行为，保持指数一致并避免不对称中间位置的过度计数。
