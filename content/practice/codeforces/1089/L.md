 ## 解决方案
 ## 问题理解

 我们有一群工人，每个人都已经选择了他们想做的工作。 正好有$k$不同的工作，每个工人都指向其中一项。 由于工人的数量至少与工作的数量一样多，因此我们最多可以为每个工作分配一名工人，但最终我们必须只覆盖每个工作一次。 

复杂的是，当前的选择可能会让一些工作空缺，而其他工作则由多名工人选择。 为了解决这个问题，我们可以说服工人改变他们选择的工作。 说服$i$-th 工人成本$b_i$，经过说服我们可以给他们分配不同的工作。 

目标是确保每一项工作$1$到$k$恰好被分配给一名工人，从而最小化总说服成本。 

约束条件达到$n = 10^5$。 任何尝试模拟分配或重复尝试组合的解决方案都会超出$O(n^2)$。 甚至$O(n \log n)$是实际目标，因为排序或堆使用是可以接受的。 这强烈表明我们应该按工作分组对待工人，并在每个组内做出贪婪的决定。 

当一项工作最初由一名工人选择时，就会出现一种微妙的边缘情况。 除非我们故意调动该工人，否则该工人将被迫从事该工作，但这样做只会产生不必要的成本，因为其他工作可能会变得更难承担。 另一种极端情况是根本没有选择工作； 那么我们必须从其他工作组“引进”一名工人，这迫使我们选择以具有成本效益的方式转移哪名工人。 

例如，如果除一名工人选择工作 2 外，所有工人都选择工作 1，则工作 3（如果$k \ge 3$）必须通过将某人从工作 1 或 2 调动来填补。最佳选择完全取决于重新分配哪个工人的成本最便宜。 

一种幼稚的方法可能会尝试按顺序贪婪地分配作业，而不考虑全局成本权衡，当局部最优分配消耗其他地方所需的低成本灵活性时，这种方法就会失败。 

## 方法

 思考这个问题的一种直接方法是想象我们最终必须为每项工作配备一名代表工人。 由于每个工人最初“属于”一个工作，每个工作都会形成一组候选者，我们必须决定哪些工人保留在原来的工作中，哪些“牺牲”以进行重新分配。 

对于每项工作，暴力方法会考虑哪些工人留下来，哪些工人被调动，同时确保所有工作最终只有一名工人。 这很快就会变成组合分配问题。 如果一份工作有$t$工人，选择留下哪一个会与所有其他工作的选择相互作用，因为每个被调动的工人都可以从事另一份工作。 在最坏的情况下，这会导致指数组合或至少$O(nk)$如果小心地进行转换，这对于$10^5$。 

关键的观察结果是，在每项工作中，只有一名工人可以保持“免于”说服成本，因为只有其中一人会在最终任务中真正担任该工作。 该职位的所有其他工人都是重新分配的候选人。 如果我们决定一项工作保留其最好的工人，那么该组中的所有其他工人都将成为“可用供应”，可用于填补其他地方缺失的工作岗位。 

这重新定义了问题：我们必须确保每项工作都至少有一名工人，并且我们希望最大限度地减少我们被迫转移的工人的总成本。 同样，对于每项工作，我们希望以零成本保留一名工人，并将所有其他工人视为重新分配的候选人。 每个工作保留哪名工人的最佳选择是保留说服成本最大的工人，因为那是我们最不想“支付解雇费用”的工人。 

因此，解决方案简化为对所有工人成本求和，并减去每个工作组中最大成本的总和，同时确保我们有足够的保留分配来覆盖所有工作。 由于每项工作至少贡献一名保留的工人，因此可行性是自动的。

 | 最佳|$O(n \log n)$或者$O(n)$|$O(k)$| 已接受 |

 ## 算法演练

 我们根据员工最初选择的工作对他们进行重组，因为决策是在每个工作组内进行的。 

1. 按所有工人选择的工作对他们进行分组。 每个小组都代表有可能担任该职位的候选人，并且只有其中一名必须保留在那里。 
2. 对于每个工作组，找到说服成本最大的工人$b_i$。 该工人的重新分配成本最高，因此我们更愿意将其保留在原位。 
3. 计算所有工人的说服成本总和。 这就是我们要说服所有人的成本。 
4. 从此总数中减去最大值$b_i$来自每个工作组。 这模拟了这样一个事实：我们为每项工作保留一名工人，但他们不被说服，选择最大限度地节省成本。 
5. 输出结果值，该值代表确保覆盖所有作业所需的最短说服时间。 

我们只需要为每个工作保留一名工人的原因是，每个工作最终都必须分配恰好一名工人，因此每个组都贡献恰好一个不需要说服的“锚定”任务。 

### 为什么它有效

 每项工作必须保留一名工人。 在一个工作组中，除了我们分配给该工作的工人之外的任何工人都必须被调动。 由于搬家费用$b_i$，最小化成本相当于最大化我们通过不移动某人而避免支付的成本。 避免搬迁的最佳人选始终是拥有最大的工人$b_i$在这个群体中，因为保留任何其他工人都会迫使我们失去更高价值的储蓄。 这为每个作业创建了一个不会跨作业交互的本地选择，因此独立最优选择的总和是全局最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    max_b = [0] * (k + 1)
    total = 0
    
    for i in range(n):
        job = a[i]
        cost = b[i]
        total += cost
        if cost > max_b[job]:
            max_b[job] = cost
    
    ans = total - sum(max_b[1:])
    print(ans)

if __name__ == "__main__":
    solve()
```该实现维护所有说服成本的运行总计，同时跟踪每项工作分配给该工作的工人的最大成本。 最后的减法步骤将每份工作精确地删除一名工人，使其不被算作“被说服”。 

关键细节是我们从不显式模拟重新分配。 我们只计算保留哪些工人最便宜。 索引来自$1$到$k$直接匹配职位标签，避免了一对一的复杂情况。

import sys, io

def solve():
    input = sys.stdin.readline
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    max_b = [0] * (k + 1)
    total = 0
    for i in range(n):
        total += b[i]
        max_b[a[i]] = max(max_b[a[i]], b[i])
    
    print(total - sum(max_b[1:]))

 ## 边缘情况

 当每个工人都选择相同的工作时，所有其他工作都是空的，必须通过从密集组中重新分配工人来完全填补。 该算法通过选择该组中成本最高的工人留下来处理此问题，同时将所有其他工人计为可移动成本，从而通过保留最昂贵的工人来正确地最小化总说服时间。 

当每个作业已经只有一个工人时，每个组都有一个元素，因此最大值等于该组的总数。 减法消除了一切，产生零成本，与不需要说服的事实相匹配。
