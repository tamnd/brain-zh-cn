 ## 解决方案
 ## 问题理解

 我们每周都有一份课程表，每 7 天重复一次。 一周中的每一天都被标记为客座学生课程的活动或非活动状态。 除了这个时间表之外，我们还给出了目标上课天数，表示为$k$。 

我们想要在日历上选择一个连续的天数块，从一周中的任何一天开始，这样在该块内的课程天数恰好是$k$。 在所有这些有效的区块中，我们必须最小化区块的总长度，即我们在伯兰停留的日历天数。 

关键细节是一周无限重复，因此周六之后周日又会以相同的模式出现。 我们选择的块可以在相对于每周周期的任何对齐处开始，这很重要，因为改变开始可以改变课程日累积的速度。 

这些约束迫使我们采用非常具体的计算形状。 测试用例的数量可以多达10,000个，并且$k$可以达到$10^8$。 这立即排除了任何每天模拟每个测试用例的方法。 即使是线性扫描$k$在最坏的情况下太慢了，因为这意味着最多$10^8$每个测试的步骤，这是不可行的。 

唯一可用的结构是周期性。 由于时间表每 7 天重复一次，因此每整周的上课天数是固定的，长的部分必须分解为完整的周期加上短的剩余部分。 该解决方案必须依赖于这种周期性计数而不是显式模拟。 

当每周计划的活跃天数很少时，就会出现微妙的边缘情况。 例如，如果只有一天是活动的，并且这一天是星期日，那么实现连续上课日需要仔细调整周； 假设稳定积累的天真贪婪扫描可能会高估或低估所需的窗口，因为它忽略了跨周边界的对齐。 

另一个边缘情况是所有 7 天都处于活动状态。 在这种情况下，每天都有贡献，答案应该很简单：$k$，但是任何尝试以周为单位进行推理的错误解决方案都可能会错误地添加额外的填充。 

## 方法

 蛮力策略将模拟每个可能的开始日期并向前扩展，计算包括多少个上课日，直到达到$k$。 对于 7 天周期中的每个起始偏移，我们可以日复一日地向前走，维护上课天数的计数器，并在到达时停止$k$。 然后我们记录窗口长度。 

这是正确的，因为它直接构造每个有效段并对其进行测量。 然而，其成本却令人望而却步。 在最坏的情况下，如果$k = 10^8$我们有很多测试用例，每次模拟可能会遍历$O(k)$天。 即使只有 10,000 次测试，这也远远超出了任何可行的时间限制。 

关键的观察结果是该结构每周都会重复。 我们不应该考虑个别日子，而应该考虑整周有多少个上课日，以及部分周的贡献如何。 一旦我们确定了循环的起点，上课/非上课日的序列就变成周期性的，因此上课日的累积成为重复块上的线性函数。 

这建议将答案分成整周加上剩余的前缀。 如果我们知道一个完整周期中有多少个上课日，我们可以在恒定时间内跳跃几周，并且最多只模拟一个额外的部分一周。 对于一周中每个可能的起始日（只有 7 种可能性），我们可以使用这种分解贪婪地计算最小窗口。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(7k)$每次测试|$O(1)$| 太慢了 |
 | 最佳|$O(7)$每次测试|$O(1)$| 已接受 |

 ## 算法演练

 我们对每周模式进行预处理并计算一整周内存在多少个上课日。 

1. 计算一周中的总活跃天数。 该值决定了我们在整个 7 天周期中获得的上课天数。 它是让我们高效前进的基本单位。 
2. 对于一周中从 0 到 6 的每个可能的起始日，模拟累积上课天数的过程，但不要在任意长的时间上逐日步进，而是尽可能重复使用整周。 这避免了时间的线性增长。 
3. 对于固定的起始偏移量，计算在保持低于目标的情况下我们可以包括多少个整周$k$。 每个整周都有固定的上课天数，因此我们可以批量减去。 此步骤可能会减少问题的规模$10^8$最多执行恒定数量的操作。 
4. 消耗整周后，最多再向前模拟 14 天。 这个上限是足够的，因为任何最佳片段都可以调整为在一个周期边界内开始，超过两周的超调只会重复已经看到的模式。 这确保了我们捕捉到的准确点$k$上课日到了。 
5. 对于每个起始位置，计算精确到达所需的总段长度$k$上课日，并取所有 7 个起始偏移量中的最小值。 

### 为什么它有效

 该算法依赖于两个结构特性：周期性和有界调整。 周期性确保每 7 天后，系统的状态准确地重复一次，因此班级积累仅取决于我们采取了多少个完整周期。 有界调整确保最佳窗口边界可以由一个周期内的开始和最多一个附加周期内的结束来表示； 否则，将窗口向后移动整周将产生更短或相等的解决方案，与最优性相矛盾。 这保证了仅检查 7 个起始位置和整周跳跃后的恒定长度模拟就足以找到全局最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        k = int(input())
        a = list(map(int, input().split()))
        
        total_week = sum(a)
        
        # duplicate for easy circular traversal
        b = a * 2
        
        ans = float('inf')
        
        # try each starting position in the week
        for start in range(7):
            need = k
            length = 0
            
            # if no classes in a week (shouldn't happen per constraints)
            if total_week == 0:
                continue
            
            # take full weeks if beneficial
            full_weeks = 0
            if need > total_week:
                full_weeks = (need - 1) // total_week
                need -= full_weeks * total_week
                length += full_weeks * 7
            
            # now simulate at most 14 days
            cur = start
            while need > 0:
                if b[cur] == 1:
                    need -= 1
                cur += 1
                length += 1
            
            ans = min(ans, length)
        
        print(ans)

if __name__ == "__main__":
    solve()
```该实现首先计算每周有多少个上课日。 对于开始日的每个可能的对齐方式，它贪婪地删除尽可能多的整周，将问题转换为更小的余数问题。 然后直接在双倍数组上模拟余数以避免模运算。 最终答案是所有起始对齐的最小值，它捕获了每周周期的最佳可能变化。 

一个微妙的实现细节是使用以下方法计算整周$(k - 1) // total\_week$。 这确保了我们不会过度调整$k$可以被每周总数精确整除，从而保持余数模拟的正确性。

 ## 边缘情况

 一种关键的边缘情况是只有一个工作日处于活动状态。 例如，如果只有星期六有效，则达到$k$上课日需要整周间隔。 该算法通过将结构折叠为整周跳跃来处理此问题，确保不会尝试强制不存在的相邻日期。 

另一个边缘情况是所有日子都处于活动状态。 在这种情况下，每周总数等于 7，并且该公式立即消耗整周。 每个单位的上课日恰好对应一个日历日，因此该算法自然会简化为直接线性结果。 

第三种边缘情况是当$k = 1$。 该算法评估所有起始仓位，并立即找到有效的起始日，产生长度 1。正确绕过整周逻辑，因为不需要批量减法。 

这些案例证实，稀疏性和密度的两个极端都可以通过相同的周期性分解逻辑来统一处理。
