 ## 解决方案
 ## 问题理解

 我们有一个多层地牢。 每个级别都是一个单元格网格，所有级别都通过特殊的出口单元按顺序连接。 玩家从第一层左上角的单元格开始，初始力量等于 1。 

地牢中的每个单元格的行为都类似于地形类型。 有些单元格是空的，什么也不做。 有些是无法进入的墙壁。 有些包含具有正力量值的敌人。 有些包含通往下一层的出口。 

玩家移动时，只能穿过未被阻挡的单元格。 重要的规则是，当玩家进入一个有敌人的牢房时，只有当他们当前的力量至少等于敌人的力量时，他们才可以击败它。 如果他们获胜，他们的力量就会根据敌人的价值而增加。 

目标是计算玩家达到最终关卡时可实现的最大功率，假设他们始终以最佳方式穿过地牢。 

从约束条件来看，各级单元格总数最多为 1e6。 这立即排除了任何根据功率值或每个敌人从头开始重复重新计算可达性的解决方案。 任何尝试独立模拟所有路径的解决方案都太慢，因为指数路径探索甚至每次更新重复的 BFS 都会超出时间限制。 

关键的微妙之处在于，地下城不仅仅是一个最短路径或可达性问题。 玩家进入牢房的能力取决于当前的力量，而当前的力量取决于已经击败了哪些敌人。 这在图遍历和状态增长之间创建了反馈循环。 

一些边缘情况很重要。 

首先，考虑这样一种情况：强大的敌人阻挡了进入包含弱小的敌人的区域的通道，而弱小的敌人需要获得足够的力量。 总是与第一个可用的敌人作战的幼稚贪婪策略可能会失败，因为它会阻止你以后更好的积累。 

其次，考虑敌人背后不连贯的“口袋”。 如果弱敌背后有强敌，你可能需要在其他地方增强实力后才返回。 幼稚的单遍 BFS 会错误地假设这些单元永久无法到达。 

第三，级别之间的退出转换可能会产生误导。 即使可以提前达到退出水平，但如果当前水平仍然存在更强劲的增长，那么立即退出也可能不是最佳选择。 

## 方法

 暴力解释是将其视为一个状态图，其中每个状态都是`(cell, current power)`。 从每个状态，您可以移动到相邻的单元格，并且当遇到敌人时，仅在满足功率限制的情况下进行转换，并相应地更新功率。 

这是正确的，但完全不可行。 可能的功率值的数量随着每个敌人被击败而增加，并且在最坏的情况下，每个单元可能会以不同的功率状态被多次访问。 这会导致状态爆炸，轻松超过 1e9 概念转换。 

关键的观察结果是功率单调增加。 一旦达到一定的实力，就再也回不去了。 这意味着我们不需要跟踪每个单元的多个状态。 相反，我们应该始终尝试在给定当前力量的情况下接触尽可能多的“当前可击败”的敌人，并随着力量的增加不断扩大可触及的空间。 

这自然意味着对地下城图的最佳优先遍历，但按敌人的强度优先。 如果我们维持所有可到达的单元并始终消耗我们当前可以击败的最弱的敌人，我们就模拟了最佳的生长过程。 

这将问题转化为一个单一的全局过程：扩大可到达区域，收集所有可到达的敌人，并始终以受控方式选择下一个最强的可到达增益。 优先级队列或多集结构成为自然的工具。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 状态 BFS over（小区、功率）| 指数| O(LNM × 功率状态) | 太慢了 |
 | 可达性+贪婪优先处理| O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 ### 1. 将地下城建模为单元格图

 我们将每个非阻塞单元视为图中的一个节点，通过每个级别内的 4 向邻接和通过出口单元的垂直连接进行连接。 这将运动转换为标准的图遍历。 

### 2. 维护一组可达单元格

 我们从幂为 1 的初始位置开始，执行 BFS/DFS 来标记所有可以物理到达的单元格，而不考虑太强大的敌人。 这些代表当前可访问的边界。 

之所以安全，是因为移动限制与力量无关，除非被敌人阻挡。 

### 3.维持“可用敌人”的结构

 在所有可到达的单元格中，我们收集所有强度小于或等于当前功率的敌方单元格。 这些代表我们现在可以采取的行动。 

我们将它们存储在以敌人强度为关键的最小堆中，因此我们总是首先处理最弱的可用敌人。 这种顺序很重要，因为它确保我们在不跳过必要的中间步骤的情况下逐步扩大权力。 

### 4.反复提取并处理敌人

 虽然至少存在一个可击败的敌人，但我们反复：

 1. 从敌人堆中找出我们当前可以击败的最弱的敌人。 
2. 增加力量的价值。 
3.从这个敌人所在的位置，再次扩大可达性，因为更高的力量现在可能会解锁之前被封锁的区域。 

每次功率增加，可到达的区域只会扩大，而不会缩小。 这证明从新解锁的边界重新运行 BFS 而不是全局重新计算是合理的。 

### 5. 继续，直到无法取得进一步进展

 一旦堆中不包含强度≤当前功率的敌人，并且没有新的可到达扩展揭示此类敌人，该过程就会稳定。 当前功率是可达到的最大功率值。 

### 为什么它有效

 不变的是，在每一个时刻，我们都准确地维护当前功率下可到达的单元集，并且在这些单元中我们识别出当前可以击败的所有敌人。 因为力量只有在敌人被击败时才会增加，所以任何以前无法到达的单元格只有在获得一定的力量后才会出现，并且永远不会因为重新扩张而被错过。

贪婪的选择是安全的，因为所有未来的选择都只取决于权力的增加。 就可达性而言，击败任何可触及的敌人永远不会有害； 它严格扩展或保留可达区域。 优先级排序确保我们永远不会跳过必要的小步骤，以便以后获得更大的收益。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import deque
import heapq

def solve():
    L, N, M = map(int, input().split())
    
    grid = []
    starts = []
    
    for _ in range(L):
        level = [list(map(int, input().split())) for _ in range(N)]
        grid.append(level)

    visited = [[[False]*M for _ in range(N)] for _ in range(L)]
    
    dq = deque()
    dq.append((0, 0, 0))
    visited[0][0][0] = True

    power = 1
    pq = []

    def add_cell(z, x, y):
        if visited[z][x][y]:
            return
        visited[z][x][y] = True
        val = grid[z][x][y]
        if val == -9:
            return
        dq.append((z, x, y))

    while dq:
        z, x, y = dq.popleft()
        val = grid[z][x][y]

        if val > 0:
            if val <= power:
                heapq.heappush(pq, (val, z, x, y))

        # move in same level
        for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
            nx, ny = x+dx, y+dy
            if 0 <= nx < N and 0 <= ny < M and not visited[z][nx][ny]:
                if grid[z][nx][ny] != -9:
                    visited[z][nx][ny] = True
                    dq.append((z, nx, ny))

        # level transition
        if val == -1 and z+1 < L:
            if not visited[z+1][x][y]:
                visited[z+1][x][y] = True
                dq.append((z+1, x, y))

    # process enemies greedily
    while True:
        while pq and pq[0][0] <= power:
            val, z, x, y = heapq.heappop(pq)
            if grid[z][x][y] != val:
                continue
            power += val

            # after gaining power, re-expand from this point
            dq = deque([(z, x, y)])
            while dq:
                z0, x0, y0 = dq.popleft()
                for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
                    nx, ny = x0+dx, y0+dy
                    if 0 <= nx < N and 0 <= ny < M and not visited[z0][nx][ny]:
                        if grid[z0][nx][ny] != -9:
                            visited[z0][nx][ny] = True
                            dq.append((z0, nx, ny))

                if grid[z0][x0][y0] == -1 and z0+1 < L:
                    if not visited[z0+1][x0][y0]:
                        visited[z0+1][x0][y0] = True
                        dq.append((z0+1, x0, y0))

        break

    print(power)

if __name__ == "__main__":
    solve()
```该实现将两个问题分开：可达性扩展和敌人处理。 类似 BFS 的扩展确保我们永远不会错过可访问的单元，而堆则确保我们始终以受控的顺序应用功率增益。 微妙的一点是，每次力量的增加都会引发新的扩张，因为新的区域可能会变得可用。

 ## 边缘情况

 一个关键的边缘情况是，一个弱敌人被无法到达的区域包围，直到另一个弱敌人在其他地方被击败。 该算法可以处理此问题，因为每次功率增加时都会触发可达性扩展，从而确保仍然捕获延迟的可达性。 

另一个边缘情况是，可以提前到达下一个级别，但不会带来最佳功率增益。 BFS不强制转换； 它只在物理上可以到达的情况下才允许，所以留在当前水平以获得更多收益是自然而然的。 

最后，多个敌人具有相同强度的情况是安全的，因为相等值之间的堆排序不会影响正确性； 一旦达到，它们都是独立有效的功率增量。
