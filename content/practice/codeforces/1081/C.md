 ## 解决方案
 ## 问题理解

 我们正在画一条线$n$砖块，每块砖块选择其中之一$m$可用颜色。 绘画完成后，我们会查看与之前的砖相比颜色发生变化的位置。 形式上，我们从左到右扫描并计算有多少个索引$i \ge 2$满足$a_i \ne a_{i-1}$。 该计数必须准确无误$k$。 

任务是计算整个序列准确产生了多少个完整着色$k$这样的“改变点”，并返回结果模$998244353$。 

解释该结构的一种有用方法是将数组视为被分割成相同颜色的连续块。 每次我们有一个变化点，一个新的块就会开始。 所以如果有$k$颜色与前一个不同的位置，则该数组恰好由$k+1$单色段。 

限制条件$n, m \le 2000$建议$O(n^2)$或者$O(n^2 m)$动态规划方法是合理的，而任何对所有着色呈指数增长的方法都是不可行的，因为着色的数量是$m^n$，即使对于中等程度的增长也太快$n$。 

出现微妙的边缘情况时$k = 0$。 在这种情况下，所有砖块必须具有相同的颜色，这会产生完全相同的结果$m$有效的配置。 另一个边界情况是$k = n-1$，每对相邻的颜色都不同，强制颜色严格交替，这严重限制了过渡，但仍然允许多种选择，具体取决于$m$。 

一个天真的错误是对待每一个$k$独立地改变位置，但约束是全局的：一旦选择了一种颜色，未来的选择取决于之前的颜色，因此独立性假设被打破。 

## 方法

 暴力解决方案将枚举所有$m^n$着色并数出有多少个$k$过渡。 这在概念上很简单：生成每个数组并计算其中的变化$O(n)$，给出总复杂度$O(n m^n)$。 这会立即失败，因为即使$m=2, n=2000$是一个天文数字。 

关键的观察结果是，结构仅取决于我们是继续一个片段还是开始一个新片段。 我们永远不需要跟踪完整的着色历史，只需要跟踪前一块砖的颜色以及到目前为止发生了多少次过渡。 这自然会导致对位置的动态规划。 

让$dp[i][j]$是第一个着色方式的数量$i$砖这样完全$j$转变已经发生。 当我们放置砖块时$i$，我们要么匹配以前的颜色（没有新的过渡），要么选择一种不同的颜色（一个新的过渡）。 关键点是，在“不同颜色”的情况下，我们有$m-1$无论之前的颜色是什么，都可以进行选择，这使我们能够消除对实际颜色值的依赖。 

这将问题从跟踪全颜色状态减少到仅跟踪过渡计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(m^n \cdot n)$|$O(n)$| 太慢了|
 | DP 过渡 |$O(nk)$|$O(nk)$| 已接受 |

 ## 算法演练

 我们定义一个 DP 表，其中$dp[i][j]$代表第一种着色方式的数量$i$砖块正好$j$过渡。 

1. 初始化第一块砖的基本情况。 有$m$给它上色的方法，并且没有发生任何过渡，所以$dp[1][0] = m$。 每个有效的着色都必须从某种颜色选择开始。 
2. 对于每个位置$i$从$2$到$n$，我们考虑扩展所有有效配置$i-1$。 
3.如果我们分配砖块$i$与砖块颜色相同$i-1$，那么转移次数不会改变。 对于计入的每个配置$dp[i-1][j]$，只有一种方法可以在不增加转换计数的情况下扩展它，因此这有助于$dp[i-1][j]$到$dp[i][j]$。 
4.如果我们分配砖块$i$与砖的颜色不同$i-1$，然后我们将过渡计数增加 1。由于之前的颜色是固定的，因此正好有$m-1$新颜色的有效选择。 这有助于$(m-1) \cdot dp[i-1][j-1]$到$dp[i][j]$。 
5. 我们计算所有状态$dp[n][k]$，这就是最终的答案。 

其递归式为：$$dp[i][j] = dp[i-1][j] + (m-1)\cdot dp[i-1][j-1]$$仔细处理无效索引。 

### 为什么它有效

 DP 状态就足够了，因为为了将来的扩展目的，任何部分着色都完全由两条信息表征：已经发生了多少次转换以及最后的颜色是什么。 最后一个颜色依赖性在过渡公式中消失，因为所有颜色都是对称的，并且切换到任何不同的颜色总是会产生精确的结果$m-1$选择。 这种对称性确保对所有最后颜色的聚合不会丢失信息，并且不会重复计算配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

n, m, k = map(int, input().split())

# dp[j] = ways for current prefix with j transitions
dp = [0] * (k + 1)

# base: first brick, 0 transitions, m choices
dp[0] = m

for i in range(2, n + 1):
    ndp = [0] * (k + 1)
    for j in range(0, k + 1):
        # same color transition count unchanged
        ndp[j] = (ndp[j] + dp[j]) % MOD

        # different color increases transitions
        if j + 1 <= k:
            ndp[j + 1] = (ndp[j + 1] + dp[j] * (m - 1)) % MOD

    dp = ndp

print(dp[k])
```该实现将 DP 压缩到单个维度，因为每一层仅依赖于前一层。 关键细节是，两个转换都是从相同的先前状态应用的，因此我们不能就地更新，否则我们会在同一迭代中污染未来的状态。 

基础初始化`dp[0] = m`捕捉到第一块砖块可以自由选择其颜色的事实。 每个后续步骤都在此基础上构建，无需跟踪实际的颜色标识。

 ## 边缘情况

 对于$n=1$，没有可能的转换，所以答案总是$m$什么时候$k=0$和$0$否则。 DP 隐式地处理这个问题，因为位置循环永远不会运行并且基本状态保持不变。 

为了$m=1$，所有砖块必须是相同的颜色。 唯一有效的配置具有零转换。 用DP术语来说，$m-1 = 0$，因此所有创建过渡的路径都消失了，只留下$dp[n][0] = 1$，这是正确的。 

为了$k = n-1$，每个相邻的对必须不同。 DP 强制形成一条链，其中每一步都乘以$m-1$， 导致$m \cdot (m-1)^{n-1}$，这符合自由选择第一种颜色然后总是避免使用前一种颜色的组合解释。
