 ## 解决方案
 ## 问题理解

 我们有两个长度的数组$n-1$。 各位置$i$描述两个未知连续值之间的关系$t_i$和$t_{i+1}$，其中每个$t_i$是范围内的整数$[0, 3]$，这意味着我们可以将每个值视为 2 位数字。 

对于每个相邻的对，我们被告知该对的按位或和该对的按位与。 任务是判断一个数组是否存在$t_1, t_2, \ldots, t_n$同时满足所有这些约束，如果存在，则构造任何有效的约束。 

关键结构是每个约束仅涉及两个相邻变量，因此问题的行为就像一系列局部兼容性条件。 每个$t_i$受到其邻居的严格约束，但我们仍然有自由，因为多个位模式可以产生相同的 OR 和 AND。 

限制条件$a_i, b_i \in [0,3]$意味着我们每个位置仅使用两位。 这很重要，因为它将局部搜索空间限制为每个变量的四种可能性，这使得在线性时间内可以管理转换。 

自从$n$可以达到$10^5$，任何独立尝试每个位置的所有可能性或对完整作业执行回溯的解决方案都将失败。 一个朴素的指数构造$4^n$状态是完全不可行的，甚至$O(n \cdot 4^2)$没有结构的暴力转换很好，但必须仔细组织以避免隐藏的重新计算。 

当约束局部有效但全局不一致时，就会出现微妙的边缘情况。 例如，一对约束可能会对同一个中间元素施加相互矛盾的要求。 只检查立即一致性而不保留下一步可行性的贪婪分配很容易在以后陷入困境，即使早期的选择看起来有效。 

## 方法

 直接暴力方法将处理每个位置$t_i$作为四个值中的一个选择，并在检查约束时尝试按顺序分配它们。 在每一步中，我们都会尝试所有可能的值$t_{i+1}$与当前满足 OR 和 AND 关系$t_i$。 这导致了一个分支过程，其中每个状态都可以转换到几个有效的下一个状态。 

这种蛮力是正确的，因为每个有效的解决方案都必须对应于通过这些局部转换的路径。 然而，在最坏的情况下，部分分配的数量呈指数级增长，因为每一步仍然可能有多种选择，导致粗略地$4 \cdot 4^{n-1}$最坏的分支情况下的可能性。 

关键的观察是，由于每个位置仅取决于前一个位置，因此我们不需要探索所有路径。 我们只需要知道哪些值$t_i$在位置上是可行的$i$，然后向前传播可行性。 这将问题简化为链上的动态规划，每个位置的状态空间大小为 4。 

我们维持哪些价值观$t_i$是可能的，对于每个可能的值，我们存储一个前驱值来重建序列。 因为转换仅取决于相邻约束，所以每个步骤可以在每个状态的恒定时间内计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(4^n)$|$O(n)$| 太慢了|
 | 状态上的最优 DP |$O(4n)$|$O(4n)$| 已接受 |

 ## 算法演练

 我们解读每一个$t_i$作为 2 位掩码。 对于任意相邻对$(x, y)$，约束条件变为：

 x \,|\, y = a_i,\quad x \,&\, y = b_i

 这两个方程完全确定了位之间的关系$x$和$y$。 

### 步骤

 1. 对于每个位置$i$，考虑所有值$t_i$在$\{0,1,2,3\}$。 这些代表所有可能的 2 位状态，我们将在整个链上传播有效的转换。 
2. 对于每对$(x, y)$，预先计算是否满足约束对$(a_i, b_i)$。 这是通过直接使用按位运算检查两个方程来完成的。 由于域大小是固定的，因此该步骤每对都是恒定时间的。 
3. 在连续位置的状态之间构建转移图。 对于每个有效对$(x, y)$, 标记一下$y$可以跟随$x$。 这对所有本地一致性规则进行编码。 
4. 从左到右运行分层动态规划过程。 在位置 1，最初允许所有值。 对于每个下一个位置$i$，我们只保留值$y$至少可以从一个有效的$x$在上一层中。 
5. 为每个可达状态存储一个父指针$y$在位置$i$，记录哪个状态$x$导致了它。 这允许重建最终序列。 
6. 处理完所有位置后，检查该位置是否有状态可达$n$。 如果没有一个可达，则约束系统不一致并且不存在序列。 
7. 否则，选择任何有效的最终状态并通过向后跟踪父指针来重建序列。 

### 为什么它有效

 DP在处理位置后保持不变$i$，状态集恰好代表了这些值$t_i$可以扩展到满足所有约束的有效前缀$i-1$。 由于每个转换都对单个边的 OR 和 AND 约束进行编码，因此不会存在无效的部分分配。 相反，任何全局有效的解决方案都必须在每个过滤步骤中幸存下来，因为它的每个前缀都是本地有效的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(x, y, a, b):
    return (x | y) == a and (x & y) == b

n = int(input())
a = list(map(int, input().split()))
b = list(map(int, input().split()))

dp = [[False] * 4 for _ in range(n)]
parent = [[-1] * 4 for _ in range(n)]

for v in range(4):
    dp[0][v] = True

for i in range(1, n):
    for cur in range(4):
        for prev in range(4):
            if dp[i-1][prev] and ok(prev, cur, a[i-1], b[i-1]):
                dp[i][cur] = True
                parent[i][cur] = prev
                break

end = -1
for v in range(4):
    if dp[n-1][v]:
        end = v
        break

if end == -1:
    print("NO")
    sys.exit()

res = [0] * n
res[n-1] = end

for i in range(n-1, 0, -1):
    res[i-1] = parent[i][res[i]]

print("YES")
print(*res)
```转换检查直接对每个边强制执行按位约束。 DP 表记录每个位置处每个值的可行性，而父表存储单个有效的前驱表以允许重建。 向后传递在线性时间内重建一个一致的分​​配。 

一个微妙的点是，我们在找到第一个有效的前驱后就中断了。 这是安全的，因为我们只需要存在，而不需要所有的解决方案。 DP 保证任何记录的前驱都会导致有效的延续。 

## 工作示例

 ### 示例 1

 输入：```
n = 4
a = [3, 3, 2]
b = [1, 2, 0]
```我们跟踪可达状态。 

| 我| 可达状态|
 | --- | --- |
 | 1 | {0,1,2,3} |
 | 2 | {1,2,3} |
 | 3 | {0,2,3} |
 | 4 | {0,1,2,3}（示例以任何有效状态结束）|

 一种重建路径产生：$t = [1, 3, 2, 0]$。 

这说明了如何在每个步骤中保持多个状态，但不可行的转换逐渐被消除。 

### 示例 2

 输入：```
n = 3
a = [0, 3]
b = [0, 3]
```在第一个约束中，只允许具有相同值的对，从而强制$t_1 = t_2$在$\{0,1,2,3\}$。 第二个约束要求$t_2 = t_3$同样，但也强制 OR 和 AND 都为 3，这只发生在$t_2 = t_3 = 3$。 这是一致的，但如果我们稍微修改一下：```
a = [1, 2]
b = [0, 3]
```然后没有对同时满足这两个约束，并且 DP 在位置 2 处清空，产生 NO。

 ## 边缘情况

 一种重要的边缘情况是约束完全固定相邻值。 例如，$a_i = b_i = 3$力量$t_i = t_{i+1} = 3$。 DP 的处理方法是只允许状态 3 在该位置生存，所有其他状态自然消失。 

另一种情况是当约束允许多次转换时，例如$a_i = 3, b_i = 0$，它允许任何一对不同的值覆盖 OR 中的所有位，但 AND 中不重叠。 DP 正确地保留多个状态，并且不会过早地提交单个值，这避免了以后出现死胡同。 

当早期选择仍然允许本地匹配但消除了未来的兼容性时，就会出现贪婪方法的失败情况。 DP 通过在最终重建步骤之前从不永久固定值来避免这种情况，从而保留所有全局有效的延续。
