 ## 解决方案
 ## 问题理解

 我们正在使用大小为的矩形网格$w \times h$，其中每个单元格可以被视为板上的一个单位正方形。 我们在这个网格上反复画出镀金的“环”。 第一个环覆盖整个矩形的外边框。 第二个环在其内部绘制一层，但在每个方向上从外部边界开始两个单元，并且这种模式继续。 

更准确地说，$i$-th 环是将原始网格缩小后保留的矩形的边框$2(i-1)$从每一侧层。 这意味着环的内部矩形$i$有尺寸$(w - 4(i-1)) \times (h - 4(i-1))$，我们只取它的边界单元。 

任务是计算放置后有多少不同的单元被镀金$k$这样的戒指。 

约束条件非常小，有$w, h \le 100$。 这立即告诉我们，网格上的任何直接计算甚至简单的每环公式都是安全的，因为总工作受到常数因子的限制。 除了每个环的恒定时间算术之外，不需要渐近优化的数据结构或巧妙的预处理。 

主要的微妙之处是概念上的而不是计算上的：每个环都是一个缩小的矩形的边界，我们必须小心，不要重复计算或误解有多少个单元格属于矩形边界。 

当人们假设每个环总是有一个完整的矩形边界公式时，就会发生一个典型的错误$2(w + h) - 4$。 这仅适用于外环。 对于内环，尺寸会缩小，并且小矩形会退化。 

例如，如果$w = h = 3$和$k = 1$，答案是 8。幼稚的实现可能会错误地减去角点，或者错过即使在最小网格中所有边界单元仍然有效的事实。 

当矩形收缩后变得非常小时，就会出现另一种边缘情况。 例如，如果一个环对应于一个$1 \times m$或者$2 \times m$矩形，由于顶部和底部（或左侧和右侧）重叠，边界公式发生变化。 如果处理不好这个问题就会导致计数过多。 

## 方法

 一种强力方法是显式模拟网格。 对于每个环，我们计算其边界矩形并在布尔网格中标记其边界上的所有单元格。 处理完所有环后，我们对标记的细胞进行计数。 这是正确的，因为它直接遵循定义。 

然而，这不必要地沉重。 虽然$w, h \le 100$，模拟仍然可以达到$k \cdot (w \cdot h)$在最坏的情况下工作。 虽然在技术上仍然可以接受，但它隐藏了问题的结构并且容易出现实施错误。 

关键的观察结果是，每个环只是一个矩形的周长，其尺寸线性减小。 因此，我们可以直接计算每个环的周长并对它们求和，而不是标记单元格。 唯一需要注意的是当高度或宽度变为 1 或 2 时正确处理退化矩形。 

对于一个矩形$a \times b$，边界细胞计数为：

 - 如果$a = 1$，所有单元格都在一行上：$b$- 如果$b = 1$，所有单元格都在一列上：$a$- 否则的话，就是$2a + 2b - 4$我们将此公式应用于与每个环对应的每个内部矩形。

 ## 算法演练

 我们通过迭代每个环并对其边界大小求和来计算答案。 

1. 从答案等于 0 开始，因为最初没有对任何单元格进行计数。 我们将独立累积各个环的贡献。 
2. 对于每个环索引$i$从 0 到$k-1$，计算内部矩形的尺寸。 每个环都会将原始矩形缩小$2i$从每一面看，所以剩下的尺寸是：$a = w - 2i \cdot 2$,$b = h - 2i \cdot 2$，这简化为$a = w - 4i$,$b = h - 4i$。 
3. 计算该矩形的边界单元数。 如果$a = 1$，环是单排并贡献$b$细胞。 如果$b = 1$，它是一个单列并贡献$a$细胞。 不然就贡献了$2a + 2b - 4$。 这种区别是必要的，因为角减法假设一个适当的二维矩形。 
4. 将此贡献添加到运行总计中。 每个环都与其他环不相交，因为每个环都位于一个严格较小的内部矩形上。 
5.全部处理完毕后$k$环，输出累加和。 

### 为什么它有效

 每个环完全对应于唯一定义的内部矩形的边界，并且这些边界在不同的值之间不重叠$i$。 收缩过程确保每个单元最多属于一个环。 由于我们计算每个矩形的边界单元的确切数量，因此总和仅对每个镀金单元进行一次计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

w, h, k = map(int, input().split())

ans = 0

for i in range(k):
    a = w - 4 * i
    b = h - 4 * i

    if a <= 0 or b <= 0:
        break

    if a == 1:
        ans += b
    elif b == 1:
        ans += a
    else:
        ans += 2 * a + 2 * b - 4

print(ans)
```该解决方案迭代每个环并使用直接公式重新计算当前矩形大小。 重要的细节是收缩系数$4i$，这是通过在每个环层的每一侧向内移动两个单元来实现的，每步有效地将宽度和高度减少 4 个。 

边界计算仔细地分离了退化情况。 如果没有这些检查，像这样的矩形$1 \times b$会错误地使用$2a + 2b - 4$，这会低估。

## 测试用例```python
import sys, io

def solve():
    import sys
    input = sys.stdin.readline
    w, h, k = map(int, input().split())

    ans = 0
    for i in range(k):
        a = w - 4 * i
        b = h - 4 * i
        if a <= 0 or b <= 0:
            break
        if a == 1:
            ans += b
        elif b == 1:
            ans += a
        else:
            ans += 2 * a + 2 * b - 4
    print(ans)

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    solve()
    return sys.stdout.getvalue().strip()

# provided samples
assert run("3 3 1") == "8"

# minimum grid
assert run("3 3 1") == "8"

# multiple rings shrinking to single line
assert run("5 1 2") == "5"

# rectangular case
assert run("6 4 1") == "16"

# two rings
assert run("7 7 2") == "24"

 ## 边缘情况

 一个重要的边缘情况是矩形在收缩后折叠成单行或单列。 在这种情况下，周长公式不得减去角点。 例如，在一个$1 \times 5$矩形，所有 5 个单元都属于边界，并使用$2a + 2b - 4$会错误地给出$2 \cdot 1 + 2 \cdot 5 - 4 = 8$，这是错误的。 条件处理确保正确的线性计数。 

当内部矩形变得无效（非正尺寸）时，会发生另一种边缘情况。 例如，与$w = 5, h = 5, k = 3$，第三个环将产生负尺寸。 循环终止条件防止对不存在的环进行计数，确保我们只累积有效的贡献。
