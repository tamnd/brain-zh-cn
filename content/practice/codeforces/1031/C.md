 ## 解决方案
 ## 问题理解

 Lesha 有两份不同的时间预算，一份是今天的，一份是明天的。 每个讲座笔记的固定阅读成本等于其索引：笔记 1 需要 1 小时，笔记 2 需要 2 小时，依此类推。 他可以选择任何笔记子集，但每个所选笔记必须在一天内完整阅读，并且每个笔记在两天内最多可以使用一次。 

目标是最大化他阅读的不同笔记的总数，将它们分为两组：分配给今天的一组，其总成本不超过`a`，以及分配给明天的另一组，其总成本不超过`b`。 

尽管票据的索引是无限期的，但实际上只有小索引可用，因为成本线性增长。 关键结构是较小的索引严格来说更便宜，因此它们在最大化计数方面总是更有效。 

约束条件允许`a, b`最多 10^9，这会立即排除任何模拟背包状态或动态尝试子集的尝试。 任何解决方案都必须直接推理最佳选择的结构，而不是枚举可能性。 

一个幼稚的错误是在没有全局协调的情况下尝试日复一日地贪婪地分配笔记。 例如，如果`a = 3`和`b = 3`，选择`{1,2}`今天用了 3 个小时，除了大​​件物品之外，没有为明天留下任何有用的东西。 但交换分配或以不同方式分割可以提高总数。 另一种微妙的失败模式是每天独立挑选尽可能最小的音符，而不确保全局唯一性； 这两天可能会尝试使用相同的廉价纸币。 

## 方法

 蛮力的想法是尝试所有可能的方法来分割两天之间的音符前缀。 对于固定的最大音符索引`k`，我们可以选择任何子集`{1..k}`，将每个元素分配给第一天或第二天，并检查两个总和的可行性。 这已经是指数级的了`k`，因为每个音符都有三种选择：未使用、第一天或第二天。 即使我们封顶`k`在最大值处，其中`k(k+1)/2 ≤ a+b`，作业数量增长得太快。 

关键的观察是，由于成本严格增加，任何最佳解决方案将始终包含类似前缀的选择。 如果使用编号较高的纸币，则所有较便宜的未使用纸币始终是首先包含的更好候选者。 这将问题简化为在两个背包上分配最小的数字。 

我们没有从分配的角度思考，而是相反：我们首先尝试记下尽可能多的最小笔记，忽略分割，然后我们决定如何在两天之间分配它们。 如果我们记下注释 1, 2, 3, …, k，它们的总成本是`k(k+1)/2`。 我们选择最大的`k`使得这个总和适合`a + b`。 之后，剩下的任务就是先分割这些`k`两个容量限制之间的数字。 

一旦前缀固定，贪婪地从最大到最小分配是最佳的：较大的项目更难放置，因此应首先将它们放置在可以容纳它们的日期中。

 | --- | --- | --- | --- |
 | 暴力子集 | 指数| 指数| 太慢了|
 | 前缀+贪婪分割| O(k) | O(k) | 已接受 |

 ## 算法演练

 ### 第 1 步：找到最大前缀

 我们计算最大的`k`这样`1 + 2 + ... + k ≤ a + b`。 这确保了我们永远不会尝试使用一张我们根本买不起的纸币。 

推理是任何使用的最佳解决方案`k`注释必须使用第一个的某些子集`k`索引，因为跳过较小的音符而选择较大的音符永远不会有好处。 

### 步骤 2：初始化剩余容量

 我们对待`a`和`b`作为两个独立垃圾箱的剩余容量。 最初两者都已满。 

### 步骤 3：分配从 k 到 1 的音符

 我们迭代自`k`下降到`1`。 对于每个注释`i`，我们尝试将其放在适合的日期。 

如果可能的话，我们更愿意将其放在第一天。 如果没有，我们会尝试第二天。 

我们向下的原因是较大的纸币受到更多限制。 如果我们延迟它们，即使存在有效的分配，我们也可能会失去可行性。 

### 第 4 步：存储分配

 我们维护两个列表：一个用于第一天，一个用于第二天。 每次我们分配一个音符时，我们都会从相应的容量中减去它的成本。 

### 步骤 5：输出

 我们打印这两个组。 

### 为什么它有效

 正确性依赖于单调交换论证。 任何最佳解决方案都可以转换为使用整数前缀而不减少元素数量的解决方案。 一旦前缀固定，首先分配较大的元素永远不会阻止可行的解决方案，因为放置大元素的任何失败都意味着两个容器都太小而无法容纳它，这与包含它的任何有效分区的可行性相矛盾。 

因此，从最大到最小的贪婪放置保留了可行性并最大化了所有可用数字的使用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def max_k(total):
    k = 0
    cur = 0
    while True:
        if cur + (k + 1) > total:
            break
        k += 1
        cur += k
    return k

a, b = map(int, input().split())
total = a + b

k = max_k(total)

a_rem, b_rem = a, b
day1 = []
day2 = []

for i in range(k, 0, -1):
    if i <= a_rem:
        day1.append(i)
        a_rem -= i
    else:
        day2.append(i)
        b_rem -= i

print(len(day1))
print(*day1)
print(len(day2))
print(*day2)
```第一个函数计算适合组合容量的最大前缀和。 这避免了考虑超出全球可能范围的数字。 

贪心循环处理自`k`向下。 每个数字都被放入第一个适合它的容器中。 减法更新确保我们永远不会超出容量。 

首先尝试第一天的选择是任意的； 任一顺序都有效，因为最终分配不是唯一的。 

## 工作示例

 ### 示例 1：a = 3，b = 3

 我们首先计算最大值`k`这样`k(k+1)/2 ≤ 6`。 这给出了`k = 3`自从`1+2+3 = 6`。

 总数是 7，所以`k = 3`自从`1+2+3 = 6`。

## 复杂度分析

 | 测量| 复杂性 | 说明|

 | 时间 | O(k) | 单次传递所选前缀 |
 | 空间| O(k) | 存储分配的笔记|

 的价值`k`以最大整数为界，使得`k(k+1)/2 ≤ 2×10^9`，约为 60000，因此该解决方案很容易满足限制。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    a, b = map(int, input().split())
    total = a + b

    k = 0
    cur = 0
    while cur + (k + 1) <= total:
        k += 1
        cur += k

    a_rem, b_rem = a, b
    day1 = []
    day2 = []

    for i in range(k, 0, -1):
        if i <= a_rem:
            day1.append(i)
            a_rem -= i
        else:
            day2.append(i)
            b_rem -= i

    out = []
    out.append(str(len(day1)))
    out.append(" ".join(map(str, day1)))
    out.append(str(len(day2)))
    out.append(" ".join(map(str, day2)))
    return "\n".join(out)

# sample
assert run("3 3") is not None

# minimum case
assert run("0 0") == "0\n\n0\n"

# only one day
assert run("10 0") is not None

# asymmetric split
assert run("5 2") is not None

# large balanced

 ## 边缘情况

 当其中之一`a`或者`b`为零，算法自然地将所有内容放入另一天，因为贪婪检查首先对空的一侧失败，并且所有项目都进入唯一可用的容量。 

对于非常大的相等输入，前缀计算在大约 60000 处停止，因此不会出现溢出或性能问题。 贪婪循环仍然只在这个前缀上运行，确保线性时间行为与输入幅度无关。 

一个微妙的情况是容量高度倾斜，例如`a = 1`和`b = 1000000000`。 该算法仍然首先根据总容量确定前缀，然后将较小的值先分配到大容器中，从而在没有任何特殊情况逻辑的情况下保留了可行性。
