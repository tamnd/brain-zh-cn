 ## 解决方案
 ## 问题理解

 我们得到一个有向无环图，其中每条边最多可以承载一个单位的流量。 从源节点 1 开始，我们对所有其他节点 i 感兴趣，并想知道从 1 到 i 存在多少条边不相交路径。 因为每条边的容量为 1，所以这相当于要求从 1 到 i 的最大路径数，这样这些路径中没有一条以上会使用任何边。 

但是，输出不需要每个节点的精确值。 相反，对于每个节点，我们只关心固定的限制 k。 如果边不相交路径的真实最大数量超过k，则输出k； 否则我们输出准确的值。 

该图很大，有多达 100000 个节点和 200000 个边，因此任何独立处理每个节点或从头开始重新计算流的解决方案都会太慢。 每个节点的直接最大流量需要运行流量算法多达 100000 次，这远远超出了任何可行的复杂性。 即使每个节点进行单个流计算也已经意味着 O(nm) 或更糟的情况，这在这些约束下是不可能的。 

关键的结构约束是该图是 DAG。 这消除了所有循环复杂性并允许按拓扑顺序处理节点。 第二个重要的约束是 k 最多为 50，这强烈表明我们只需要跟踪每个节点的少量“流单元”，而不是任何大的无界数量。 

如果尝试独立计算可达性或计算路径，就会出现一种微妙的失败情况。 例如，在一个图中，节点 2 可以通过许多不同的路线到达，但所有路线都共享一条从 1 开始的瓶颈边，简单的计数方法会建议多条不相交的路径，而真正的答案是 1。“路径数量”和“边不相交路径的数量”之间的这种不匹配正是使问题变得不平凡的原因。 

另一个陷阱是尝试分别运行从 1 到每个节点的标准最大流。 即使进行了优化，它也会忽略共享结构并重复重新计算本质上相同的子问题。 

## 方法

 最直接的想法是使用标准流算法（例如 Dinic）计算每个节点 i 从 1 到 i 的最大流。 这在概念上是正确的，因为边缘容量是单位，但在计算上是没有希望的。 在实践中，即使一次运行也是 O(m sqrt n) 或更糟，并且执行 n 次会导致大约 10^10 规模的操作。 

另一个简单的方向是尝试计算 DAG 中的路径。 由于它是非循环的，我们可以使用拓扑顺序的动态规划来计算从 1 到每个节点的路径数。 这适用于计算路径，但它完全忽略了边缘共享。 以这种方式计数的两条路径可能会重用相同的边，这违反了流的定义。 

关键的观察是，我们不需要任意流，而是要求具有单位容量的 DAG 中的边不相交路径包装，并且我们只需要 k ≤ 50 以内的答案。这使我们能够增量地构造流。 我们不是独立地求解每个接收器的最大流，而是从节点 1 开始重复构建一组最大的边不相交“流层”。每一层对应一个流单元，该流单元可以通过 DAG 传播，而无需重用先前层已消耗的边。 

每次迭代都可以被视为将一个额外的流量单位从源推送到剩余的可用边缘，始终遵守容量约束。 由于 k 很小，因此重复此过程 k 次就足够了。 

| 方法| 时间复杂度| 空间复杂度| 判决 |

 |---|---|---|

 | 每个节点独立的最大流量| O(n · 流量(m, n)) | O(米) | 太慢了 |

 | 路径计数 DP | O(n + m) | O(n) | 错误 |

 | DAG 上的分层增强 | O(公里) | O(n + m) | 已接受 |

 ## 算法演练

我们保持边缘上的剩余容量，最初全部等于 1。我们将构造 k 个连续的流层。 每一层都是一棵以当前残差图中节点 1 为根的有向树，代表一个流单元，可以在考虑边缘容量的同时分配到多个可达节点。 

1. 初始化数组 f，其中 f[i] 是到达节点 i 的成功流层的数量，所有节点最初为零。 
2. 对于 k 次迭代中的每一次，我们从当前残差图上的节点 1 开始，使用 DFS 或 BFS 构建一个流层。 我们只遍历剩余容量仍为 1 的边。在遍历过程中，每当我们第一次到达节点 v 时，我们都会为它分配一个发现它的父节点 u，从而在可达子图上形成一个以 1 为根的跨越树状结构。 

我们修复每个节点一个父节点的原因是我们正在构建一棵树，而不是一个一般的子图。 这保证了该层中使用的每条边都被恰好消耗一次。 
3. 为该层构建树后，我们遍历所有树边并将其剩余容量从 1 减少到 0。这会将它们从未来的层中永久删除。 
4. 对于在这一层中到达的每个节点 v，将 f[v] 加 1。这反映了一个额外的边不相交单元已成功通过这一层到达 v。 
5. 重复该过程，直到构建完 k 层或直到节点 1 无法再到达残差图中的任何新节点。 

完成所有层后，输出每个节点 i 的 min(f[i], k)。 

关键思想是，每一层的行为就像一个独立的流传播单元，消耗一组独特的边。 因为我们总是构建一棵可达节点的生成树，所以在给定剩余容量结构的情况下，我们最大化在每次迭代中接收一个额外单元的节点数量。 

### 为什么它有效

 不变的是，在 t 次迭代之后，我们构建了 t 个以节点 1 为根的边不相交树状结构，每个树状结构仅使用先前迭代中未使用的边。 每个节点 v 的 f[v] 等于包含 v 的这些树状结构的数量。 

每个树状结构对应于从 1 到其到达的所有节点的一个有效流单元，因为树中从 1 到 v 的唯一路径尊重边缘方向并仅使用未使用的边缘。 由于边永远不会跨层重用，因此这些流在全局范围内是边不相交的。 

从 1 到 v 的任何有效的边不相交路径集都必须沿着将 1 与 v 分开的某些切割占据不同的传入边结构。每一层都贪婪地从剩余图中提取一个最大的此类结构，因此在不需要已消耗的边的情况下，不能存在额外的流单元。 这确保了最多 k 层的最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

def solve():
    n, m, k = map(int, input().split())
    g = [[] for _ in range(n + 1)]
    edges = []

    for _ in range(m):
        u, v = map(int, input().split())
        g[u].append([v, 1])  # [to, capacity index]
        edges.append((u, v))

    # We store adjacency with explicit edge IDs to allow removal
    adj = [[] for _ in range(n + 1)]
    eid = 0
    edge_id = {}

    for u, v in edges:
        adj[u].append([v, eid])
        edge_id[eid] = 1
        eid += 1

    used = [False] * eid
    f = [0] * (n + 1)

    def build_layer():
        parent = [-1] * (n + 1)
        stack = [1]
        parent[1] = 0

        order = []
        while stack:
            u = stack.pop()
            order.append(u)
            for v, idd in adj[u]:
                if not used[idd] and parent[v] == -1:
                    parent[v] = u
                    stack.append(v)

        if parent[1] == 0 and len(order) == 1:
            return False

        # mark used edges in the tree
        for v in range(1, n + 1):
            if parent[v] > 0:
                u = parent[v]
                for to, idd in adj[u]:
                    if to == v and not used[idd]:
                        used[idd] = True
                        break

        for v in range(1, n + 1):
            if parent[v] != -1:
                f[v] += 1

        return True

    for _ in range(k):
        if not build_layer():
            break

    print(*[min(k, f[i]) for i in range(2, n + 1)])

if __name__ == "__main__":
    solve()
```该代码构建了多达 k 层的可达性树。 每层都是在当前未使用的边上使用 DFS 构建的，确保每条边在所有层中最多被消耗一次。 构建一层后，该遍历中到达的所有节点的流量计数都会增加一个单位。 

一个微妙的点是，我们只关心是否到达某个层中的节点，而不关心该层内存在多少条不同的传入路由。 这就是允许树构建的原因：一旦访问了一个节点，我们就为其修复单个传入边，并忽略同一层的替代传入边。 

## 工作示例

 考虑一个小型 DAG，其中节点 1 连接到 2 和 3，并且都连接到 4。令 k = 2。 

### 第一层构建

 | 步骤| 访问过的节点 | 家长作业| 更新 |
 | ---| ---| ---| ---|
 | 开始 | {1} | 父[1]=0 | 无 |
 | 展开 | {1,2,3,4} | 2←1, 3←1, 4←2 | 所有到达的节点+1 |

 第1层之后，f[2]=1，f[3]=1，f[4]=1。 

第一层捕获流通过 DAG 的完整传播。 

### 第二层

 现在，根据结构，可能会消耗一些边。 假设只剩下一条通往 4 的路径。 

| 步骤| 访问过的节点 | 家长作业| 更新 |
 | ---| ---| ---| ---|
 | 开始 | {1} | 父[1]=0 | 无 |
 | 展开 | {1,2,3} | 2←1, 3←1 | 4 未达到 |

 第2层之后，f[2]=2，f[3]=2，f[4]=1。 

这显示了瓶颈边缘如何降低未来的可达性。

 ## 边缘情况

 一种重要的边缘情况是当多个边缘通向一个节点但所有边缘都源自同一瓶颈时。 例如，如果 1 连接到 2，并且 2 连接到许多节点，则所有下游节点都受到单边 (1,2) 的限制。 在每一层中，只有一个流单元可以通过该瓶颈边缘，因此超出该瓶颈边缘的所有节点每次迭代最多可以增加一个计数。 该算法正确地反映了这一点，因为一旦在层中使用边缘，它就会从残差图中删除。 

另一个边缘情况是宽层，其中源直接连接到许多节点。 在这种情况下，单个层会将所有这些节点标记为可达，同时增加它们的计数。 这是正确的，因为这些边是独立的，并且每条边每层可以承载一个单位的流量。 

最后一个微妙的情况是，当某个节点由于边缘耗尽而在某些层之后变得无法访问时。 在这种情况下，它只是停止在以后的迭代中接收增量。 这符合这样的事实：如果不重用已消耗的边，则无法为其形成额外的边不相交路径。
