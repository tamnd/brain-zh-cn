 ## 解决方案
 ## 问题理解

 我们被给予$n$人和固定的初始座位安排$i$从座位开始$i$。 每个座位都是独一无二的，每个人都只占据一个座位。 

每个人还有一个首选座位$a_i$，位于较大座位池中的某个位置$1 \ldots 2n$。 对于每个人，我们可以让他们留在原来的座位上$i$，或将他们移到他们喜欢的座位上$a_i$。 在最终的安排中，任何两个人都不能占据同一个座位。 

目标是最大限度地增加最终坐在自己喜欢的座位上的人数。 

输出只是在这些限制下可以成功安排到他们喜欢的座位上的最大人数。 

约束条件允许$n$最多$10^5$，因此任何尝试所有子集或通过重复冲突解决来模拟分配的解决方案都会太慢。 任何二次或涉及每个候选选择的重复图形传播的事情都不会通过。 

一个微妙的困难是，选择一个人坐在他们喜欢的座位上可能会迫使其他人离开他们原本保留的座位。 例如，如果两个人想要同一个座位，则只有其中一个人可以坐，另一个人必须回到原来的座位。 然而，如果其他人已经将其声明为首选座位，则该后备座位本身可能会被阻止。 

关键的边缘情况来自碰撞：

 例如，如果多人想要同一个座位，则只有一个人可以从中受益$a = [5, 5, 5]$座位 5 只允许一次成功。 

如果偏好形成像这样的链条$1 \to 2 \to 3 \to 1$，所有这些仍然可以同时满足，因为它们彼此之间进行排列。 

这两种行为表明，冲突仅在多个边缘针对同一席位时才重要，而在依赖关系形成循环时则无关紧要。 

## 方法

 暴力方法会尝试为每个人决定是否让他们保持在$i$或将它们移至$a_i$，同时确保没有座位被使用两次。 这本质上是搜索所选动作的所有有效子集。 状态空间是指数的，即使是贪婪的模拟也会反复解决通过席位依赖结构传播的冲突，从而在发生级联时导致潜在的二次行为。 

关键的简化来自于观察冲突实际上是如何产生的。 每个人恰好贡献一个候选人目标席位$a_i$。 选择这些动作的唯一真正限制是，一个座位不能分配给多个选择其首选选项的人。 

如果多人指向同一个座位，最多只有一个人能够成功。 如果我们选择其中一个，该座位就会被占用并迫使其原始所有者移动，但这并不会减少我们在其他地方可以实现的满意人数。 

这意味着每个座位$v$最多可以贡献一个满意的人$i$这样$a_i = v$。 因此，我们能做的最好的事情就是计算偏好列表中出现了多少个不同的座位。 

剩下的问题是这个上限是否总是可以实现的。 这是。 我们可以为每个不同的座位分配$v$，正是一个人想要它，然后解决强制向外移动。 这些强制移动永远不会减少计数，因为它们只决定未选择或后备分配的去向，而不是我们已经固定的首选座位分配数量。 

因此，答案减少为不同值的数量$a$。

 | ---| ---| ---| ---|
 | 暴力搜索作业 | 指数| O(n) | 太慢了 |
 | 计算不同的偏好 | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们想要计算所有首选座位中出现了多少个不同的座位。 

1.读取所有值$a_1, a_2, \ldots, a_n$。 这些代表人们想要占据的目标席位。 
2. 将每个值插入集合中或将其标记在布尔数组中。 这会自动删除重复项，确保每个座位仅计算一次。 
3. 处理所有值后该集合的大小就是答案，因为每个不同的座位最多可以贡献一个满意的人，并且这个上限总是可以实现的。 

该实现不需要构建最终的座位或模拟运动，因为冲突的结构分解为简单的唯一性计数。 

### 为什么它有效

 每次成功分配到首选座位时，该座位都会被占用一次。 如果两个或两个以上的人瞄准同一个座位，则只能选择其中一个，因此重复的人不能将答案增加到每个座位一个以上。 

同时，为每个不同的席位选择一个代表人物绝不会产生阻碍性矛盾，从而减少所选优先席位分配的数量。 任何因释放原始席位而导致的强制调整只会影响后备布局，与目标无关。 

因此，满意人数的最大数量等于不同首选座位的数量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
a = list(map(int, input().split()))

seen = set()
for x in a:
    seen.add(x)

print(len(seen))
```该解决方案读取所有首选项并将它们存储在一个集合中。 该集会自动确保重复项不会影响计数。 打印该套装的最终尺寸。 

唯一微妙的实现细节是避免任何不必要的座椅运动模拟。 尽管这个故事暗示了座位分配时人与人之间存在依赖性，但这些依赖性并不影响可以分配多少个首选座位。

 答案是 2，因为只有 2 号和 3 号座位是目标。 这符合每个席位最多可以贡献一个满意的人的想法。 

现在考虑一个没有重复的情况：

 输入：$n = 4$,$a = [5, 6, 7, 8]$| 步骤| 加工值| 看过集|

 所有的喜好都是不同的，所以每个人都可以得到满足。 

这些痕迹表明，只有碰撞才会减少可实现的计数，并且每次碰撞都会压缩为每个座位一个可用的槽位。

 ## 边缘情况

 例如，当所有偏好都相同时$a = [7, 7, 7, 7]$，只有一个人能够满足。 该算法生成一个包含一个元素的集合，正确地捕获了这一限制。 

例如，当所有偏好都不同时$a = [1, 2, 3, 4]$，每个人都可以分配自己想要的座位。 该集合包含所有值，答案等于$n$。 

当首选项混合重复值和唯一值时，例如$a = [1, 2, 2, 3, 3]$，每个重复的席位仅贡献一次。 集合构造自然会折叠重复项，确保不会过度计数，也不需要显式的冲突解决逻辑。
