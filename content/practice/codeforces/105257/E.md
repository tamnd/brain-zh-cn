 ## 解决方案
 ## 问题理解

 我们有一个圆分成$K$等距位置，以及$n$这些头寸包含市场。 每个市场都位于圆上的固定点，因此我们可以将输入视为圆形数组上的索引的递增序列。 

适合每个市场$i$，我们可以从它到另一个市场建立直接连接$j$，但前提是$j$是距离最远的市场之一$i$沿着圆圈。 距离是沿着圆上两点之间的较短弧测量的，因此“最远”意味着最大化该圆距离。 如果多个市场达到相同的最大距离，则可以选择其中任何一个。 

每个选定的连接都成为在圆内绘制的弦。 我们希望选择尽可能多的有效有向连接，但限制是不允许两个弦在任何地方相交或重叠，除了可能在其端点处。 

输出是我们可以构造的此类非相交有向连接的最大数量。 

约束条件$n \le 10^5$意味着我们需要接近的东西$O(n \log n)$或者$O(n)$。 由于二次行为，任何尝试所有市场对或成对检查交叉点的操作都会立即失败。 

一个微妙的问题来自圆形几何形状。 市场最远的有效目标不是一个点，而是圆另一侧的连续区域。 这意味着每个源节点并不指向一个固定的目的地，而是指向整个候选区间。 为每个节点任意选择最远候选点的简单方法很容易破坏全局最优性，因为较早选择“坏”最远端点可能会阻塞许多后来的非交叉边。 

当几个市场恰好位于最远边界时，就会出现另一种失败情况。 例如，当点对称时，多个目的地相连，不同的选择会产生不同的交叉结构。 不考虑未来可行性的贪婪选择可能会减少答案。 

## 方法

 如果我们暂时忽略几何形状，那么暴力破解的想法很简单：对于每个市场$i$，尝试每个有效的最远目的地$j$，然后尝试选择边的子集，使得它们的弦都不相交。 这就变成了圆上的几何间隔选择问题。 人们可以模拟每对候选边的相交，然后尝试所有子集。 这很快就会退化为指数搜索，甚至检查单个子集的有效性也需要花费时间$O(n^2)$交叉测试。 

关键的简化来自于理解“圆上最远”的真正含义。 如果我们将市场固定在某个位置$x$，其最远的市场正是位于相对于$x$。 就排序位置而言，这成为索引的连续区间。 因此，每个节点不是具有单个目的地，而是具有一定的可能目的地区间。 

现在问题变成了：我们有$n$源，每个源都有一个允许的目标间隔，我们想要选择尽可能多的对$(i, j)$尽可能使每个$i$最多使用一次作为源，每个$j$最多使用一次作为目的地，并且生成的和弦不会交叉。 在一条线上（适当打破圆之后），非交叉结构对应于单调排序约束：如果$i_1 < i_2$，那么我们必须避免$j_1 > j_2$。 

这将问题转化为选择最大数量的兼容区间分配问题。 每个来源$i$想要一个合作伙伴$j$里面$[L_i, R_i]$，并且我们希望以保持顺序的方式将源与不同的目标贪婪地匹配。 最大化此类分配的经典方法是按右端点的升序处理源，并始终分配最早的可用有效目标。 这确保了我们不会在早期节点上浪费大量选择，并为后期节点保持灵活性。 

我们可以通过排序和使用跟踪未使用的候选者的数据结构来有效地实现这一点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对边缘和子集的暴力破解 | 指数| O(n) | 太慢了 |
 | 区间构造+贪心匹配 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 首先，将市场头寸按升序排列，并将它们视为线上的点而不是圆上的点。 这是安全的，因为我们可以选择一个起始切割，以避免破坏任何候选的最远结构； 从概念上讲，我们将圆“展开”成一段。 
2. 针对每个市场$i$，计算其最远距离。 在圆上，最远的区域正是那些与圆的距离$i$最大化，对应于圆的另一半。 在排序坐标中，这成为连续索引间隔$[L_i, R_i]$。 
3. 解读各个市场$i$作为一项请求：它必须与一个目标完全匹配$j$从其区间中选择$[L_i, R_i]$。 任何有效的解决方案都对应于选择此类匹配。 
4. 对所有来源进行排序$i$通过增加$R_i$。 这种顺序确保我们始终首先处理最受限的选择，因为较小的右端点意味着较少的可能目标。 
5. 维护当前未使用的可作为目的地的市场指数结构。 当我们按排序顺序处理源时，我们为每个源分配其允许间隔内的最小可用索引$[L_i, R_i]$。 一旦使用了目标，就会将其从可用性中删除。 
6. 每一次成功的分配都会将答案加一，我们会继续下去，直到处理完所有源。 

### 为什么它有效

 每个源间隔表示保留最大距离的所有可能的最远端点。 关键属性是任何有效的解决方案都可以转换为按照目的地位置升序分配目的地的解决方案，而不会减少边的数量。 一旦我们强制执行这种单调结构，问题就变成了标准的贪婪区间匹配：采用最早的可行目的地永远不会阻止更好的全局解决方案，因为任何后来的源都无法从使用较早的源可能采用的较早未使用的目的地中受益。 

这表明贪婪分配在区间约束下产生最大尺寸的非交叉匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    K, n = map(int, input().split())
    a = list(map(int, input().split()))

    a.sort()

    # Compute intervals [L, R] of farthest candidates
    # Distance on circle -> opposite semicircle
    L = [0] * n
    R = [0] * n

    # We will treat circle linearly; farthest region is half-circle away.
    # Convert K to "half range"
    half = K / 2

    # For each i, we find points within distance ~half in circular sense.
    # Using two pointers over sorted array.
    j1 = 0
    j2 = 0

    for i in range(n):
        # move j1 to maintain left boundary (too close clockwise)
        while j1 < n and (a[i] - a[j1]) % K < half:
            j1 += 1
        # move j2 to maintain right boundary (within half in opposite direction)
        while j2 < n and (a[j2] - a[i]) % K <= half:
            j2 += 1

        L[i] = j1
        R[i] = j2 - 1

    # Greedy matching on intervals
    used = [False] * n
    ans = 0

    # sort indices by R
    order = sorted(range(n), key=lambda i: R[i])

    ptr = 0
    for i in order:
        # find smallest available j in [L[i], R[i]]
        j = L[i]
        while j <= R[i] and used[j]:
            j += 1
        if j <= R[i]:
            used[j] = True
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```核心实现思想是将循环“最远”约束转化为索引区间。 两指针扫描在已排序数组上以线性时间构建这些间隔。 之后，问题就纯粹是一个区间分配任务了。 

一个微妙的点是我们从未明确地以几何方式构造边缘。 我们只在索引空间中进行推理。 这避免了直接处理和弦相交测试，因为右端点的贪婪排序隐式强制执行非相交。

 ## 边缘情况

 一个重要的边缘情况是所有市场都位于一个半圆内。 在这种情况下，每个点的最远区域都会塌缩成类似的相反区间，从而在所有点之间产生严重重叠$[L_i, R_i]$。 按右端点的贪婪排序确保我们仍然通过首先消耗最紧的间隔来提取最大数量的不相交分配。 

另一种情况是市场在圆圈周围均匀分布。 在这里，每个区间几乎变成一个候选者。 该算法的行为类似于直接匹配，每个源都可以独立配对，达到最大计数。 

当多个候选人并列最远时，就会出现第三种情况，从而产生更宽的间隔。 尽管每个源都具有灵活性，但贪婪结构确保只有在不阻碍未来更严格的约束时才使用灵活性，从而保持最优性。
