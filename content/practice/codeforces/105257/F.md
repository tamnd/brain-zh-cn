 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每一个都有一个整数列表，代表不同代码提交的分数。 玩家必须提交代码两次，最终得分不是总和或最大值，而是两次所选提交得分的按位与。 相同的提交可以被选择两次，这意味着我们可以将索引与其自身配对。 

对于每个测试用例，任务是选择两个索引 i 和 j（可能相等），以便 gi 和 gj 的值尽可能大。 

输入大小很大：所有测试用例的总数最多为 2×10^5。 这排除了任何显式检查所有对的二次配对策略，因为在最坏的情况下这将导致大约 4×10^10 次操作。 我们需要一个在每个测试用例的线性时间或更好的时间内工作的解决方案。 

“可以提交相同的代码两次”规则产生了一个微妙的边缘情况。 如果忽略这一细节，人们可能会错误地认为需要 i ≠ j 并尝试找到两个不同的元素。 例如，如果数组是 [7, 1]，错误的解释可能会表明答案是 7 & 1 = 1，而正确的解释允许选择 7 两次，得到 7 & 7 = 7。 

另一种边缘情况是除了一个大值之外所有数字都为零。 任何与零的配对都会破坏位，但自配对会保留原始值。 

## 方法

 暴力破解的想法很简单：尝试所有对 (i, j)，计算 gi 和 gj，并跟踪最大值。 这是正确的，因为它直接评估问题的定义。 然而，每个测试用例需要检查 n^2 对，当 n 总共达到 2×10^5 时，这会变得太慢。 

关键的观察是，按位 AND 对于相同的元素表现得单调：对于任何数字 x，将其与自身配对会产生 x，并且这总是至少与将其与任何其他数字配对一样大，因为 AND 只能关闭位。 

这立即改变了问题的结构。 我们不需要寻找最佳配对，而只需要考虑最佳结果是否来自自身配对。 由于每个元素都是通过 (i, i) 的有效候选者，因此答案不能小于数组中的最大元素。 另一方面，任何对 (i, j) 都会产生一个按位包含在 gi 和 gj 中的值，因此它永远不会单独超过它们中的任何一个。 这意味着没有一对可以击败最大元素本身。 

因此，最优策略简化为寻找每个测试用例中的最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 最佳（最大扫描）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 ### 1.独立处理每个测试用例

 每个测试用例都是孤立的，因此我们分别计算每个数字列表的答案。 

### 2. 扫描所有值，同时保持运行最大值

 我们保留一个变量`best`初始化为 0。对于数组中的每个数字 x，我们更新`best = max(best, x)`。 

这是可行的，因为每个元素都可以通过选择两次来获得有效结果，因此答案必须至少是最大元素。 

### 3.输出找到的最大值

 扫描完所有元素后，我们打印`best`作为该测试用例的答案。 

### 为什么它有效

 对于任意两个索引 i 和 j，值 gi & gj 不能超过 gi 也不能超过 gj，因为按位 AND 只清除位。 因此，每个可能的对结果都以数组中的最大元素为上限。 由于选择 i = j 即可实现 gi，因此始终可以实现最大元素。 这会将答案精确地固定为列表中的最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n = int(input())
    arr = list(map(int, input().split()))
    print(max(arr))
```该解决方案只需读取每个测试用例，计算最大值，然后打印它。 关键的实现细节是使用Python的内置`max`，它以线性时间运行，并且对于该问题规模而言是最佳的。 不需要额外的预处理，也不需要显式地考虑对。

## 测试用例```python
import sys, io

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    input = sys.stdin.readline

    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        out.append(str(max(arr)))
    return "\n".join(out)

# provided sample-like cases
assert solve("2\n3\n0 1 2\n4\n10 10 5 4\n") == "2\n10"

# minimum size, single element
assert solve("1\n1\n7\n") == "7"

# all equal values
assert solve("1\n5\n3 3 3 3 3\n") == "3"

# mixed values
assert solve("1\n4\n1 8 3 6\n") == "8"

# includes zeros
assert solve("1\n3\n0 0 0\n") == "0"
```| 测试输入| 预期产出 | 它验证了什么 |

 | 一切平等| 相同的值 | 重复项下的稳定性 |

 ## 边缘情况

 ### 单元素数组

 输入：```
1
1
42
```算法初始化`best = 0`并将其更新为 42。输出为 42，这与唯一有效操作是将元素与其自身配对这一事实相匹配。 

### 全零

 输入：```
1
4
0 0 0 0
```每次更新都会保留`best = 0`。 任何对都会产生 0，因此输出是正确的。 

### 价值观的广泛传播

 输入：```
1
5
1 2 4 8 16
```扫描在 16 处结束。即使配对不同的数字会减少值，但自配对会保留每个候选数字，确保最大的数字保持不变。
