 ## 问题理解

 我们得到一个长字符串`S`和两个短图案`s1`和`s2`。 对于每个子串`T`的`S`，我们看多少次`s1`出现在`T`作为子序列以及多少次`s2`出现在`T`作为子序列。 这些计数不是通常意义上的连续出现次数，而是按顺序形成模式的索引选择的计数。 

对于每个子串`T`，我们将这两个值相乘，并将结果称为“不愉快”。 任务是将这个值在所有子串上求和`S`。 

重要的困难是两者`s1`和`s2`是子序列计数模式，而不是子字符串，因此每个子字符串在如何选择子序列方面已经具有潜在的指数内部结构。 对子串和子序列嵌入的直接枚举是完全不可行的。 

这些限制使得这一点变得精确。 字符串`S`长度可达十万，因此任何试图显式检查所有子串的方法都已经暗示了二次行为，这太大了。 不过，这两个图案都非常小，最多也就二十个字符。 这种不对称性是关键信号：解决方案必须将模式视为固定的自动机状态，并将大字符串视为唯一需要仔细迭代的维度。 

一个微妙的边缘情况来自这样一个事实：子序列计数可能为零或很大，具体取决于重复的字符。 另一个极端情况是两种模式严重重叠时`S`，因为该乘积耦合了它们的组合。 最后，在有关子字符串的某些推理中隐式允许空子字符串，但由于非空模式的子序列计数为零，因此没有任何贡献。 

## 方法

 最直接的方法就是枚举每个子串`S[l..r]`，并为每一个运行一个动态程序来计算有多少种方式`s1`和`s2`可以作为子序列嵌入。 单个字符串的标准子序列 DP 运行于`O(|T| * |s|)`时间，因此每个子字符串执行两次会导致粗略的结果`O(n^3)`最坏情况下的总复杂度。 即使进行了优化，所有子字符串的外循环也已经强制`O(n^2)`迭代次数，远远超出了一秒的处理能力`10^5`。 

关键的观察结果是，两者`s1`和`s2`足够短，以至于它们的子序列结构可以表示为有限自动机，每个自动机最多有二十个状态。 当我们通过附加字符来扩展子字符串时，子序列 DP 转换会确定性地更新。 这表明，我们不应该从头开始重新计算每个子字符串，而应该在扩展正确的端点时逐步构建贡献。 

突破口是修复正确的端点`r`并考虑所有以`r`。 每个这样的子字符串仅由其起始位置确定`l`，当我们从`r-1`到`r`，每个子串都扩展一个字符。 这使我们能够为每个起始位置维护方法的数量`s1`和`s2`可以在当前子字符串内部形成，并以结构化方式更新这些值。 

因为这两种模式都很短，所以我们可以为每个可能的前缀长度维护`s1`和`s2`以当前位置结尾的所有子字符串的运行总计。 这将问题转化为右端点和两个小自动机维度上的分层 DP。

 ## 算法演练

 我们从左到右处理字符串，将每个位置视为以该位置结尾的所有子字符串的右边界。 

对于每个位置`i`，我们维护一个DP表`dp[a][b]`。 该表的含义是对以索引结尾的所有子串的总贡献`i`， 在哪里`a`代表有多少个字符`s1`我们已经作为子字符串内的子序列进行了匹配，并且`b`代表同样的`s2`。 换句话说，每个以`i`将其当前子序列状态计数贡献到该表的一个单元格中，并且该表聚合所有起始位置。 

1. 初始化`dp`对于位置`0`除空结构外全部为零，因为尚不存在子字符串。 
2. 迭代每个字符`S[i]`从左到右。 
3. 启动新的DP表`ndp`初始化自`dp`，因为每个现有的子字符串都以`i-1`仍然是结束于的子字符串`i`在应用扩展之前。 
4. 对于每个州`(a, b)`，模拟扩展每个结尾的子字符串`i-1`通过添加`S[i]`。 这意味着我们更新这两种模式的子序列进度。 如果`S[i]`匹配下一个必需的字符`s1`，然后子序列匹配该前进`a`也可以形成； 同样对于`s2`。 
5. 处理转换后，我们还考虑到每个位置`i`可以开始一个仅包含以下内容的新子字符串`S[i]`。 这有助于基态`(0, 0)`用于在该单字符子字符串内形成子序列。 
6.一次`ndp`构建完成后，所有以结尾的子串的贡献`i`通过对所有状态求和，乘以已完成的比赛数来获得`s1`和`s2`通过转换隐式编码。 我们将其累积成最终答案。 
7. 更换`dp`和`ndp`并继续。 

关键的不变量是处理后的位置`i`，DP 表正确表示了所有以 结尾的子串的聚合子序列状态分布`i`。 每个子串结束于`i`是结束于的子字符串的扩展`i-1`，或新开始的子字符串`i`，并且两种情况都只计算一次。 由于子序列转换仅取决于当前状态和下一个字符，因此除此之外没有任何历史信息`(a, b)`是必须的。 

这保证了每个端点的每个子串都被精确计数一次，并且在每个子串中，每个有效的子序列嵌入`s1`和`s2`对其状态演化做出了正确的贡献。 产品结构得以保留，因为我们在同一状态空间中同时跟踪两种模式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

S = input().strip()
s1 = input().strip()
s2 = input().strip()

n = len(S)
m1 = len(s1)
m2 = len(s2)

# dp[a][b] = number of ways (over all substrings ending at current i)
# where a chars of s1 and b chars of s2 have been matched as subsequences
dp = [[0] * (m2 + 1) for _ in range(m1 + 1)]

ans = 0

for ch in S:
    ndp = [[0] * (m2 + 1) for _ in range(m1 + 1)]

    for a in range(m1 + 1):
        for b in range(m2 + 1):
            val = dp[a][b]
            if not val:
                continue

            # extend without using ch
            ndp[a][b] = (ndp[a][b] + val) % MOD

            # try to match s1
            if a < m1 and ch == s1[a]:
                ndp[a + 1][b] = (ndp[a + 1][b] + val) % MOD

            # try to match s2
            if b < m2 and ch == s2[b]:
                ndp[a][b + 1] = (ndp[a][b + 1] + val) % MOD

    # start new substring at current position
    ndp[0][0] = (ndp[0][0] + 1) % MOD

    dp = ndp

    # count fully matched states
    ans = (ans + dp[m1][m2]) % MOD

print(ans % MOD)
```该代码遵循这样的思想：每个子字符串都是通过扩展先前的子字符串或重新开始来构建的。 DP 状态跟踪两个子序列自动机中的同时进展。 对每一步的答案有贡献的唯一状态是两个模式都完全匹配的状态，因为它对应于每个子串内的一对有效的子序列嵌入。 

一个微妙的实现点是“开始新的子字符串”转换。 没有显式注入`(0,0)`每个位置一次，从以下位置开始的子字符串`i`永远不会被计算在内。 另一个微妙之处是，必须将转换从先前的 DP 应用于新的数组，因为就地更新将错误地允许单个角色在同一层内推进多个子序列步骤。 

## 工作示例

 考虑一个存在重叠子序列的小例子。 让`S = ababa`,`s1 = aba`,`s2 = ba`。 

我们跟踪每个位置的 DP 层，仅关注聚合含义。 

在`i = 0`, 仅子串`"a"`存在。 它无法完成任一模式，因此贡献为零。 

在`i = 1`, 子串`"ab"`和`"b"`考虑结束位置。`"ab"`部分匹配两种模式，但仍然没有完全完成，因此贡献仍然为零。 

在`i = 2`, 子串`"aba"`出现。 这是第一个位置`s1`可以作为子序列进行匹配。`s2`还可以通过多种方式进行搭配。 现在，DP 为两个自动机均已达到其最终位置的状态累积非零贡献，这些贡献有助于所有以 结尾的子串的答案`2`。 

| 我| 新角色 | dp[m1][m2] 贡献 | 运行 ans |
 | ---| ---| ---| ---|
 | 0 | 一个 | 0 | 0 |
 | 1 | 乙| 0 | 0 |
 | 2 | 一个 | 1 | 1 |
 | 3 | 乙| 2 | 3 |
 | 4 | 一个 | 3 | 6 |

 该轨迹显示了后面的位置如何积累更多有效的子序列配对，因为两种模式的额外完成在较长的子串内变得可能。 

第二个没有重叠的例子，例如`S = cccc`,`s1 = ab`,`s2 = ba`，使所有状态的 DP 保持为零。 这证实了该算法正确地避免了在字符不足时生成虚假子序列匹配。

 ## 边缘情况

 一个关键的边缘情况是两个模式相同。 在这种情况下，每个子字符串的贡献变成子序列计数的平方，这会严重放大重复字符。 DP 自然地处理这个问题，因为两个自动机都遵循相同的转换，因此它们的状态同步演变，而不需要任何特殊的外壳。 

另一种情况是长度为一的非常短的子串。 这些总是将 DP 重置为基本状态，并且仅当两个模式的长度均为 1 并且匹配相同的字符时才起作用。 显式的“开始新子字符串”转换可确保这些子字符串只计算一次。 

最后的边缘情况涉及高度重复的字符串，例如`"aaaaa..."`，其中子序列计数组合增长。 DP 没有明确列举组合； 相反，它通过状态转换来聚合它们，因此指数增长被压缩为每个位置的多项式状态更新。
