 ## 问题理解

 每个输入行描述了一个以紧凑数字形式编码的类似 Unix 的权限配置。 系统没有直接授予文件权限，而是使用一个三位数，其中每个数字独立地描述三个用户类别之一的访问权限：所有者、组和其他。 

每个数字都在 0 到 7 之间，应被解释为 3 位掩码。 这三位按固定顺序对应于读、写和执行权限。 如果设置了一个位，则相应的字符出现在最终的权限字符串中，否则使用破折号。 

每个测试用例的输出是一个 9 个字符的字符串。 它是通过将三个数字中的每一个扩展为三个字符的块，并将这些块按照所有者、组、其他人的顺序连接起来而形成的。 

限制很小，最多有 100 个测试用例，并且每个用例都需要持续工作。 这立即排除了每个输入行线性处理之外的任何内容，尽管即使二次行为仍然会通过。 问题的结构是定宽解码，所以性能并不是主要难点。 

一个常见的错误来自于对位顺序的误解。 最高有效位对应于读取，中间位对应于写入，最低有效位对应于执行。 例如，数字 6 对应于二进制 110，这意味着读取和写入已启用，但执行未启用，从而生成字符串“rw-”。 另一个常见错误是颠倒每个三元组内的字符顺序，这会导致错误的排列，例如“wr-”或“-rw”。 

## 方法

 暴力破解的想法是将每个数字视为 0 到 7 之间的数字，并使用重复的模数或二进制转换逻辑显式测试每个权限。 对于每个数字，可以重复除以 2 或逐个提取位，从而在临时数组中构建字符串。 这是正确的，因为每个数字恰好编码三个独立的布尔标志。 

由于只有三位，这种方法已经在每个数字上以恒定的时间运行，但它往往以更冗长且容易出错的方式编写。 如果不小心使用字符串操作或循环内重复串联来实现，它可能会变得不必要的缓慢或混乱，即使理论上的复杂性仍然微不足道。 

关键的观察是每个数字直接映射到三个固定位位置。 我们可以使用按位运算来代替模拟二进制提取。 这将解决方案简化为直接查找这三个位中的每一个是否都被设置。 由于每个数字只有八个可能的值，因此映射完全是确定性的且时间恒定。

 ## 算法演练

 1. 读取测试用例的数量并迭代每个输入字符串。 
2. 对于字符串中的每个字符，将其转换为整数。 
3. 对于每个数字，按固定顺序确定权限字符：读、写、执行。 
4. 对于每个权限，使用按位与检查数字中的相应位。 
5. 如果设置了该位，则附加相应的字符，否则附加破折号。 
6. 连接所有数字的三字符块并输出生成的 9 字符字符串。 

这种排序起作用的原因是该问题定义了位和权限类型之间的严格位置映射。 每个数字都是独立的，因此解码可以在本地完成，而所有者、组和其他人之间没有任何交叉依赖性。 

### 为什么它有效

每个数字都编码三元素集的子集，其中每个元素对应于一种权限类型。 数字的二进制表示正是该子集的特征向量。 由于从位到字符的映射是固定且单射的，因此通过独立检查每个位来重建字符串可以毫无歧义地保留所有信息。 连接解码的块可以唯一地重建完整的权限字符串。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def decode_digit(d):
    r = 'r' if d & 4 else '-'
    w = 'w' if d & 2 else '-'
    x = 'x' if d & 1 else '-'
    return r + w + x

t = int(input())
out = []

for _ in range(t):
    s = input().strip()
    res = []
    for ch in s:
        d = ord(ch) - ord('0')
        res.append(decode_digit(d))
    out.append(''.join(res))

print('\n'.join(out))
```实现的核心是`decode_digit`函数，它使用按位 AND 隔离三个权限位。 常数4、2、1分别对应读、写、执行位置的二进制权重。 

主循环只是将该转换独立地应用于每个字符并连接结果。 Using a list for accumulation avoids repeated string concatenation overhead, which is unnecessary but still a good habit in Python competitive programming.

 ## 边缘情况

 对于像这样的输入`0`，该数字没有设置位，因此所有三个权限都映射到破折号。 该算法评估`0 & 4`,`0 & 2`， 和`0 & 1`，所有这些都是假的，为每个组生成“---”。 重复此操作三次会生成九个破折号的完整字符串，这与预期的空权限状态匹配。 

对于像这样的输入`7`，所有位均已设置。 支票`7 & 4`,`7 & 2`， 和`7 & 1`全部评估为真，产生“rwx”。 这种行为在所有数字中都是一致的，并且由于每个数字都是独立处理的，因此组之间不存在干扰，即使在统一或极端情况下也能确保正确性。
