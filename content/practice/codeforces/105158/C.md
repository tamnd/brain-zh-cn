 ## 解决方案
 ## 问题理解

 我们得到一个长度的数组$n$，其中每个元素都是范围内的整数$[1, n]$。 我们可以应用由函数定义的转换$f$，它将每个值映射到$[1, n]$到同一范围内的另一个值。 应用此函数后，数组变为$B$，其中每个元素都独立地替换为其映射值。 

目标是使结果数组$B$非减少。 选择函数的成本是值的数量$x$为此$f(x) \neq x$。 换句话说，域中我们从其原始身份“更改”的每个值都会贡献一个成本单位，无论它在数组中出现多少次。 

因此，问题不在于直接修改数组位置，而在于选择值的重新标记，并为每个改变标签的值付费。 

一个关键的结构约束来自排序如何与值映射交互。 如果一个值$x$出现在数组的前面和另一个值$y$稍后出现，那么为了保持最终数组的排序，我们必须确保$f(x) \le f(y)$。 这是对出现次数的全局约束，而不仅仅是相邻元素。 

一个天真的误解是认为我们只需要在映射后比较相邻位置。 这是不够的，因为相距很远的事件之间可能会出现错误的顺序。 

例如，考虑像这样的序列$A = [1, 2, 1]$。 如果我们映射$f(1)=2$,$f(2)=1$，那么结果数组就变成了$[2, 1, 2]$，它没有排序，即使如果只考虑相邻结构，局部推理可能会错过冲突。 

当两个值交错时，会出现第二种微妙的情况。 如果$x$和$y$以交替顺序出现，例如$x, y, x$，那么两个约束$f(x) \le f(y)$和$f(y) \le f(x)$是被迫的，这意味着$f(x)=f(y)$。 这会产生一种强制合并效应，而天真的贪婪方法会错过这种效应。 

自从$n$可以达到$2 \times 10^5$，任何接近值对二次行为的解都是不可能的。 我们必须将结构压缩成更接近线性或$n \log n$。 

## 方法

 暴力方法会尝试分配一个值$f(x)$对于每一个$x$，然后检查转换后的数组是否已排序。 由于每个$f(x)$最多有$n$选择，不同值的数量呈指数级增长，甚至检查有效性也需要扫描整个数组。 这立即爆炸超出了任何可行的限制。 

更结构化的视图来自于注意到唯一重要的是由数组中的出现引起的值之间的相对顺序约束。 各值$x$有效地代表了一组位置。 如果我们定义一个值在最终顺序中何时必须出现在另一个值之前，我们就会得到可以使用间隔来表达的约束。 

对于每个值$x$，令其第一次出现为$L_x$最后一次出现的是$R_x$。 然后我们观察到一个关键事实：如果$x$出现在某些事件发生之前$y$， 然后$L_x < R_y$允许约束$f(x) \le f(y)$。 同样，如果$L_y < R_x$，我们得到$f(y) \le f(x)$。 如果两个条件都成立，则两个值相互约束，强制相等。 

这意味着值的行为就像线上的间隔，并且重叠结构决定了强制相等组。 重叠间隔的每个连接区域必须映射在一起。 

一旦对值进行分组，就可以为每个组分配一个输出值。 一组内尺寸$k$，我们至少必须改变$k-1$值，因为如果我们选择自己的标签作为代表，最多可以保持一个值不变。 

因此，问题就简化为找出区间重叠图中存在多少个连通分量。

 ## 算法演练

 我们将每个不同的值转换为由其在数组中的第一次和最后一次出现定义的间隔。 

1. 计算$L_x$和$R_x$对于每个值$x$。 

这捕获了推理排序约束所需的所有位置信息。 
2. 按左端点对所有值进行排序$L_x$。 

这使我们能够处理从左到右扫描的间隔。 
3. 扫描已排序的间隔，同时保持当前活动组的最大右端点。 

如果下一个间隔在此最大右端点之前或处开始，则它与当前组重叠并且必须属于同一组件。 
4. 当某个区间与当前组不重叠时，我们关闭当前组件并开始一个新组件。 
5. 计算形成了多少个组件。 
6.答案是$n - \text{number of components}$，因为每个分量的大小$k$准确贡献$k-1$被迫改变。 

### 为什么它有效

 核心不变量是，在任何时刻，活动组代表一组最大的值，其区间直接或传递重叠。 重叠意味着存在一系列交错，强制组件中的所有值之间存在相等约束。 相反，如果两个组件被间隙分开，则约束无法强制它们共享值，因此可以独立对待它们。 这确保了每个组件都是最小的强制平等结构，并且独立地折叠每个组件会产生最佳成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    INF = 10**18
    first = [INF] * (n + 1)
    last = [-1] * (n + 1)

    for i, x in enumerate(a):
        if first[x] == INF:
            first[x] = i
        last[x] = i

    intervals = []
    for x in range(1, n + 1):
        if last[x] != -1:
            intervals.append((first[x], last[x]))

    intervals.sort()

    components = 0
    cur_r = -1

    for l, r in intervals:
        if l > cur_r:
            components += 1
            cur_r = r
        else:
            cur_r = max(cur_r, r)

    print(n - components)

if __name__ == "__main__":
    solve()
```该实现首先将每个值压缩到其出现间隔中。 然后，它对这些间隔进行排序并执行标准合并扫描以对重叠下的连接组件进行计数。 微妙的部分是通过运行最大右边界将重叠视为传递连接，这避免了显式构建图。 

## 工作示例

 考虑一个值强烈交错的输入：

 输入：```
6
1 2 1 3 2 3
```| 步骤| 处理间隔| 当前右界 | 组件|
 | --- | --- | --- | --- |
 | 1 | (1:[0,2]) | (1:[0,2]) | 2 | 1 |
 | 2 | (2:[1,4]) | 4 | 1 |
 | 3 | (3:[3,5]) | 5 | 1 |

 所有间隔传递重叠，形成单个分量，所以答案是$6 - 1 = 5$。 

这演示了由于交织而将所有值强制归入一组的情况。 

现在考虑一个分离的结构：

 输入：```
6
1 1 2 2 3 3
```| 步骤| 处理间隔| 当前右界 | 组件|
 | --- | --- | --- | --- |
 | 1 | (1:[0,1]) | (1:[0,1]) | 1 | 1 |
 | 2 | (2:[2,3]) | 1 → 3 | 2 |
 | 3 | (3:[4,5]) | 3 → 5 | 3 |

 每个块都是隔离的，产生三个分量和答案$6 - 3 = 3$。 

这证实了非重叠段的行为是独立的。

 ## 边缘情况

 一个微妙的边缘情况是两个值几乎不通过共享边界发生接触。 例如，在类似的序列中`1 2 3 1`，值1有区间`[0,3]`值 2 有`[1,1]`。 即使 2 完全出现在 1 的范围内，它也会强制合并，因为 1 和 2 以通过中间位置创建相互排序约束的方式交错。 由于它们的间隔重叠，该算法正确地合并了它们。 

另一种情况是当间隔链没有直接重叠时，例如`[1,4]`,`[4,7]`,`[7,10]`。 即使每对仅接触边界，扫描也会将它们合并为单个组件，因为运行的右端点会传递连接性。 这确保弱重叠仍然产生单个强制平等组。
