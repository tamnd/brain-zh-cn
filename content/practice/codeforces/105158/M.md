 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 在每个测试用例中，都有两个相同长度的整数数组。 对于每一个指标，我们都可以“调整”该位置的值，但调整不是任意的。 我们为位置选择的最终值$i$必须保持在以原始值为中心的对称区间内$a_i$，并且该区间的半宽与全局参数成正比$k$乘以$b_i$。 

形式上，对于每个索引$i$，我们可以选择一个最终值$x_i$使得与原始值的距离为$k \cdot b_i$。 选择完所有这些之后$x_i$，我们希望所有最终值变得相等。 目标是找到最小的非负整数$k$这是可能的。 

关键的难点是每个位置都有不同的灵活性尺度$b_i$，因此每个元素的可达范围以不同的速率增长：$k$增加。 我们实际上是在问：何时存在一个同时位于所有可调整区间内的整数点？ 

约束很大：所有测试用例的总长度达到$3 \cdot 10^5$。 这立即排除了尝试每个可能的目标值或模拟每个目标值的所有选择的任何方法。$k$。 每个测试用例的任何二次方都会失败，甚至$O(n \log A)$必须仔细构建以避免大范围内的重复扫描。 

当间隔几乎不重叠或仅在单个整数点相交时，会出现微妙的边缘情况。 例如，如果所有$a_i$相等，那么$k = 0$是有效的。 相反，如果一个元素的值极小$b_i$，它可能成为迫使更大的瓶颈$k$，即使其他元素非常灵活。 

另一个重要的情况是，当交集仅在远离所有值的大值处才变为非空时$a_i$。 “让一切都接近平均值”的天真想法会失败，因为最终值并不局限于任何平均值附近，它只需要位于所有扩展的区间内。 

## 方法

 暴力解释是修复$k$并检查是否存在公共整数$x$这样对于每个$i$,$x$位于区间内$[a_i - k b_i, a_i + k b_i]$。 这将问题简化为测试是否相交$n$间隔非空。 

对于固定的$k$，这个检查很简单：计算所有左端点的最大值和所有右端点的最小值，然后验证左端点的最大值是否不大于右端点的最小值。 这以线性时间运行。 

然而，尝试一切$k$是不可能的，因为$k$可以大到$10^9$，检查每个值会将工作乘以$10^9$。 甚至二分查找$k$需要$O(n \log 10^9)$，这是可以接受的，但是我们实际上可以通过观察可行性条件的结构来完全避免二分搜索。 

关键的观察是可行性取决于一组不等式：$$a_i - k b_i \le x \le a_i + k b_i$$对于所有人$i$，这相当于：$$\max_i (a_i - k b_i) \le \min_i (a_i + k b_i)$$左边是凸增函数$k$在分段线性意义上，右侧是凸递减函数。 最小的$k$不等式成立的地方正是最严格的约束对满足的地方。 而不是搜索一遍$k$，我们可以将条件重新表述为检查约束之间的成对一致性，从而减少为仅考虑极端差异。 

更具体地说，答案是由最差的一对决定的$(i, j)$，因为在最优情况下$x$，每个元素都必须能够“到达”它，最难的限制来自于平衡两个元素，其斜率$b_i$相对于它们的距离差异最大$a$-空间。 这导致了表达式：$$k \ge \max_{i,j} \frac{|a_i - a_j|}{b_i + b_j}$$直觉是，如果两个元素必须在同一点相遇，则它们的可达间隔必须重叠，这恰好强制执行此约束。 

因此，我们将问题简化为计算变换后的成对比率的最大值。 这可以使用已知的技巧在每个测试用例的线性时间内完成：我们排序$a_i$，然后只有按排序顺序的相邻对才能定义最大比率，因为任何非相邻对都有一个产生更严格约束的中间点。

 ## 算法演练

 1. 按以下值对索引进行排序$a_i$。 这会排列元素，以便使差异$a$是本地结构的，使得邻居之间出现最严重的约束。 
2. 扫描排序数组中的相邻对。 对于每对$(i, i+1)$，计算候选值$$k_{i} = \frac{a_{i+1} - a_i}{b_i + b_{i+1}}$$这是因为强制要求它们的间隔在某个点处接触。 
3. 跟踪所有此类候选者的最大值。 这个最大值就是最小值$k$这允许每个相邻对具有重叠的可达范围。 
4. 输出这个最大值，四舍五入到最接近的整数$k$必须是整数。 

我们只检查相邻对的原因是，按排序顺序相距较远的任何两个元素已经具有主导其直接约束的中间约束。 如果距离较远的一对是最坏的情况，则意味着与区间扩展引起的三角形结构相矛盾。 

### 为什么它有效

 在最佳解决方案中，所有最终值都会崩溃到一个点$x$。 每个索引定义一个以$a_i$有半径$k b_i$。 可行性意味着所有区间都相交。 

当排序时$a_i$，对全局交叉点的最紧密阻碍必须出现在该排序中的一些相邻间隔之间。 如果两个不相邻的间隔定义瓶颈，则中间元素的间隔已经更紧密地限制重叠。 这确定了所需的最大$k$完全由相邻的相互作用决定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        b = list(map(int, input().split()))

        arr = sorted(zip(a, b))
        ans = 0.0

        for i in range(n - 1):
            a1, b1 = arr[i]
            a2, b2 = arr[i + 1]
            if a2 == a1:
                continue
            ans = max(ans, (a2 - a1) / (b1 + b2))

        out.append(str(int((ans + 1e-12) // 1 + (1 if ans % 1 > 0 else 0))))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现首先对每个测试用例进行排序$a_i$，这对于将问题减少到本地检查至关重要。 对相邻对的扫描计算从间隔接触条件导出的限制比。 

计算使用浮点运算来估计比率，然后向上舍入到最接近的整数，因为$k$必须是完整的。 小 epsilon 用于避免值非常接近整数时的精度错误。 

一个微妙的问题是处理平等$a_i$。 在这种情况下，该对没有任何约束，因为分子为零，并且任何$k$在本地工作。 

## 工作示例

 ### 示例 1

 输入：```
n = 3
a = [1, 2, 3]
b = [2, 2, 2]
```排序后的数组保持不变。 

| 配对| 差异 | 总和| 比率|
 | ---| ---| ---| ---|
 | (1,2) | 1 | 4 | 0.25 | 0.25
 | (2,3) | 1 | 4 | 0.25 | 0.25

 最大比率为 0.25，因此$k = 1$。 

这表明，即使值均匀分布，所需的灵活性也受到最差局部间隙的控制。 

### 示例 2

 输入：```
n = 4
a = [3, 1, 4, 5]
b = [1, 3, 1, 1]
```排序：```
(1,3), (3,1), (4,1), (5,1)
```| 配对| 差异 | 总和| 比率|
 | ---| ---| ---| ---|
 | (1,3)-(3,1) | (1,3)-(3,1) | 2 | 4 | 0.5 | 0.5
 | (3,1)-(4,1) | 1 | 2 | 0.5 | 0.5
 | (4,1)-(5,1) | (4,1)-(5,1) | 1 | 2 | 0.5 | 0.5

 最大值为 0.5，所以$k = 1$。 

这显示了多个相邻对如何同时紧张，而答案是由整个链上一致的瓶颈驱动的。

 ## 边缘情况

 一个重要的边缘情况是当所有$a_i$是平等的。 例如：```
a = [7, 7, 7]
b = [1, 100, 1]
```每个区间都以同一点为中心，所以$k = 0$作品。 该算法发现相邻排序元素之间的差异为零，因此所有比率都为零，并且答案仍然为 0。 

另一种边缘情况是当$b_i$非常小。 例如：```
a = [1, 100]
b = [1, 1000000]
```小的$b_i$力量很大$k$，因为一个元素几乎不移动。 相邻比率直接捕获了这一约束，并且算法正确地优先考虑紧密对，而不是被其他元素的巨大灵活性所误导。 

最后的边缘情况涉及到巨大的差距$a$但也很大$b$，天真的直觉可能表明没有约束。 该算法仍然正确地捕获了权衡，因为该比率明确地平衡了距离与组合灵活性。
