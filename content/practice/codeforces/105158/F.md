 ## 问题理解

 我们得到了一个字符串集合，我们需要计算其中有多少满足非常特定的结构模式。 如果字符串的长度正好为 5、第三个和第五个字符相同且前四个字符彼此不同，则该字符串被认为是有效的。 

因此，对于每个输入字符串，我们不会对其进行转换或将其与其他字符串进行比较。 我们只检查它是否独立地满足这些约束，然后计算有多少个满足。 

输入大小很大：最多 100000 个字符串，所有字符串组合起来的总长度最多为 200000。这立即告诉我们必须在基本恒定的时间内处理每个字符串。 任何使用大量操作扫描每个字符串或进行与字符串数量成比例的额外分配的解决方案，只有在总体保持线性的情况下仍然会通过。 

天真的解释可能建议检查排列或相互比较子字符串，但约束是局部的和固定长度的。 每个字符串的长度要么是 5，要么不是，所以我们永远不需要考虑每个字符串超过 5 个字符。 

如果人们认为“不同”意味着所有五个字符都不同而不仅仅是前四个字符，则会出现微妙的失败情况。 例如，像“abaca”这样的字符串具有重复的字符，但重复可能出现在与条件无关的位置。 另一个陷阱是忘记只有前四个位置必须是不同的，而第五个字符仅与第三个位置相关，不包含在独特性要求中。 

一个具体的例子：

 输入：

 “abc”

 这是无效的，因为第三个字符“c”不等于第五个字符“e”。 一个粗心的实现，只检查前四个之间的区别，会错误地计算它。 

另一个例子：

 “阿巴巴”

 此处，第三个和第五个字符匹配 ('a')，但前四个字符并不全部不同（'a' 重复）。 仅验证相等条件的简单检查会错误地计算它。 

## 方法

 蛮力方法很简单。 对于每个字符串，我们首先检查它的长度。 如果不是五个，我们立即拒绝。 否则，我们检查字符：验证位置三等于位置五，然后检查前四个字符是否成对不同。 这可以通过比较前四个位置中的所有对或将它们插入到集合中并检查其大小来完成。 

这种方法已经足够有效，因为每个字符串都有恒定的长度。 即使我们对每个字符串执行 O(1) 工作，总复杂度也是 O(n)。 如果我们重复使用像集合这样的数据结构，唯一潜在的低效率来自不必要的开销，但即使如此，由于固定大小很小，仍然在限制范围内。 

关键的观察结果是该结构是固定的并且非常小。 我们永远不需要对长子字符串进行散列或对大域进行频率计数。 每个检查都以常量（最多五个字符）为界，因此问题简化为每个字符串常量时间谓词评估。 

这消除了全局推理或预处理的需要。 每个字符串都可以独立地进行验证。

 ## 算法演练

 我们一一处理每个字符串并应用固定的验证例程。

1. 读取一个字符串并立即检查其长度。 如果它不正好是五个，我们就丢弃它。 此步骤可防止对无效候选者进行不必要的字符检查。 
2. 如果长度为5，则比较第三个和第五个字符。 如果它们不同，则该字符串无效，因此我们继续。 这是最早可能的拒绝条件，因为它取决于固定位置。 
3. 如果位置相等成立，我们现在检查前四个字符。 我们必须确保所有四个都是不同的。 由于长度是恒定的，我们可以使用一个小的布尔数组或一个集合来记录看到的字符。 如果发生任何重复，我们将拒绝该字符串。 
4. 如果两个条件都满足，我们将增加答案计数器。 

检查的顺序对于效率很重要。 长度检查和位置相等检查是恒定时间且廉价的，并且它们避免了在检查剩余字符之前对无效字符串进行不必要的工作。 

### 为什么它有效

 每个字符串都是独立处理的，验证逻辑直接反映了“优秀”字符串的定义。 第三和第五个字符上的条件仅强制执行一次，并且在预期的位置子集上验证独特性条件。 由于所有检查均已完成且未进行近似，因此通过算法的任何字符串都满足这两个约束，并且至少不满足一个条件的任何字符串都会被拒绝。 字符串之间没有交互，因此正确性保持每个实例并扩展到完整计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_good(s: str) -> bool:
    if len(s) != 5:
        return False
    if s[2] != s[4]:
        return False
    # check first four characters are all distinct
    seen = set()
    for i in range(4):
        if s[i] in seen:
            return False
        seen.add(s[i])
    return True

def main():
    n = int(input())
    ans = 0
    for _ in range(n):
        s = input().strip()
        if is_good(s):
            ans += 1
    print(ans)

if __name__ == "__main__":
    main()
```该解决方案将定义直接编码到辅助函数中。 首先进行长度检查以避免索引假设。 条件`s[2] != s[4]`在构建任何数据结构之前进行检查，因为不匹配会立即取消字符串的资格。 

基于集合的唯一性检查仅覆盖前四个字符，精确匹配问题要求。 我们故意不在该检查中包含第五个字符，因为它受单独条件的约束。

 ## 边缘情况

 一种边缘情况是字符串短于或长于五个字符。 例如，“abc”或“abcdef”。 该算法在长度检查时立即拒绝这些，而不访问索引，从而防止越界假设。 

另一个边缘情况是第三个和第五个字符匹配，但前四个字符严重重复，例如“aaaaa”。 在通过位置相等性测试后，该算法在独特性检查期间正确地失败了。 

第三种情况是前四个字符都不同，但第三个和第五个字符不匹配，例如“abcde”。 该算法在位置检查的早期就拒绝了它，这表明这两个条件都是独立执行的，并且都不是多余的。
