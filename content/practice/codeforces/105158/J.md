 ## 问题理解

 我们得到一个五位整数，其中所有数字都不同。 我们可以从这五位数字中任意重新排列它们的顺序，但结果数字仍然必须是有效的五位整数，这意味着它不能从零开始。 

对于每个测试用例，任务是确定至少一个这样的重新排列是否形成合数。 如果存在这样的重新排列，我们必须输出任何一种有效的复合排列。 如果每个有效排列都是素数，则答案为 -1。 

关键结构是每个输入恰好贡献 5 个不同的数字，因此可能的重新排列数固定为 120 减去排列后从 0 开始的数。 这个有限的大小强烈表明，对每个测试进行强力排列是可行的，但测试用例的数量可以达到 100000，因此使用平方根检查对素性进行简单的每次测试重新计算总的来说会太慢。 

当零是其中一位数字时，会出现微妙的边缘情况。 例如，如果数字是 10234，则某些排列（例如 01234）是无效的，必须被忽略，即使它们是数字多重集的有效排列。 另一个边缘情况是当所有排列都是素数时，在这种情况下我们必须正确输出 -1 而不是意外返回无效或未经过滤的候选者。 

第二个微妙的问题是性能：独立检查每个排列的素性将导致多达 120 × 100000 = 1200 万次素性检查，并且每次检查最多 sqrt(99999) 操作，这会变得太慢。 这迫使我们使用预计算将生成与素性测试分开。 

## 方法

 直接暴力策略独立考虑每个测试用例。 我们枚举五位数字的所有排列，跳过那些带有前导零的排列，将每个排列转换为整数，并通过测试其平方根的整除性来检查它是否是合数。 这是正确的，因为它详尽地检查了有效重新排列的完整搜索空间。 问题是计算成本。 在最坏的情况下，我们每次测试执行大约 120 次排列，最多进行 100000 次测试，给出 1200 万个候选数字。 平方根素性检查每个数字大约需要 300 次运算，总共需要数十亿次运算，这远远超出了限制。 

改进来自于分离关注点。 可能值的空间很小，以 99999 为界，因此可以使用筛子预先计算该范围内每个数字的素数。 之后，每个排列只需要一个常数时间的查找来确定它是质数排列还是合数排列。 这将问题转化为固定数字集上的纯组合生成，每个候选者进行 O(1) 分类。 

我们还通过使用 itertools 在每个测试中直接生成排列来避免重复重新计算排列，因为 120 个候选者很小并且可以接受。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（每个排列的试除法）| O(T·120·√N)| O(1) | O(1) | 太慢了 |
 | 筛选+排列枚举| O(100000 + T·120) | O(100000) | 已接受 |

 ## 算法演练

1. 使用筛子预先计算 99999 以内的所有数字的素数。 这允许稍后对任何候选号码进行即时分类。 
2. 对于每个测试用例，将输入数字的五位数字提取到列表中。 使用数字可以避免重复的字符串解析。 
3. 生成这五个数字的所有排列。 
4. 对于每个排列，检查第一位数字是否为零。 如果是，请立即丢弃它，因为它不能形成有效的五位数。 
5. 将剩余的排列转换为整数。 
6. 使用预先计算的筛子检查该整数是否为合数。 如果是复合的，立即输出并转到下一个测试用例。 
7. 如果没有排列产生合数，则输出-1。 

提前停止背后的原因是，该问题只要求存在一个有效的复合排列，因此一旦找到有效答案，我们就不需要探索完整的排列空间。 

### 为什么它有效

 每个有效的数字重新排列在排列枚举中只出现一次。 筛子保证每个候选数的正确分类。 由于我们接受遇到的第一个合数，并且所有候选数都根据完整的有效集进行检查，因此我们不会错过有效答案。 如果没有候选者是合数，则根据定义，所有有效排列都是素数，因此 -1 是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MAXV = 100000

is_prime = [True] * MAXV
is_prime[0] = is_prime[1] = False

for i in range(2, int(MAXV ** 0.5) + 1):
    if is_prime[i]:
        step = i
        start = i * i
        for j in range(start, MAXV, step):
            is_prime[j] = False

from itertools import permutations

t = int(input())
for _ in range(t):
    s = input().strip()
    digits = list(s)

    found = False

    for p in permutations(digits, 5):
        if p[0] == '0':
            continue
        val = 0
        for ch in p:
            val = val * 10 + (ord(ch) - 48)

        if not is_prime[val]:
            print(val)
            found = True
            break

    if not found:
        print(-1)
```顶部的筛子构建一个全局表，以便以后的每个素性查询都减少为单个数组查找。 这是防止重复重新计算整除性检查的关键优化。 

在测试循环内，数字被提取一次并重复使用。 排列循环足够小，Python 的内置生成器就足够了。 整数构造是手动完成的，以避免内部循环内字符串到整数的开销。 

在转换之前应用前导零检查，以便无效排列永远不会到达筛选查找阶段。
