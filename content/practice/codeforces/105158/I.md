 ## 解决方案
 ## 问题理解

 给我们一个由小写字母组成的字符串，最多允许我们改变$k$人物。 目标是确定我们是否可以将弦变成非常刚性的周期结构。 

定义“美丽”字符串的条件比普通的周期性更强。 我们必须找到一个时期$p$这样每个字符都完全匹配$p$前面的位置，只要字符串内有可能。 换句话说，一个周期一次$p$选择后，字符串必须在所有对齐位置模数上保持一致$p$，并且只有第一个$p$字符决定整个字符串。 

这将问题从“我们能否全局修复字符串”转变为“我们能否强制字符串进入重复的长度模式”$p$进行有限的编辑”。

 正是这些限制使问题变得有趣。 测试用例的总长度可达$3 \cdot 10^5$， 尽管$k$很小，最多 100。这立即表明任何解决方案取决于$n^2$甚至$n \cdot k^2$很好，但是任何涉及每个测试用例二次工作的事情都不是。 关键信号是我们可以对每个候选结构进行线性扫描，但不能对所有可能性进行大量的每个字符优化。 

一个微妙的边缘情况是当字符串对于某些大的已经是周期性的时$p$。 例如，如果字符串已经是统一的，例如`aaaaaa`， 任何$p$作品。 一种幼稚的方法，只检查小的$p$会失败的。 另一个棘手的情况是，当最佳解决方案需要选择一个从局部重复中不明显的非平凡周期时，例如`ababac`修复几个字符会揭示一个小时期。 

## 方法

 蛮力方法尝试每一个可能的周期$p$从$1$到$n$。 对于每个$p$，我们强制一致性：所有索引$i$和$i+p$必须匹配，因此每个残基类模$p$形成一组独立的位置，它们必须具有相同的最终特征。 

对于固定的$p$，我们检查每个组$r \in [0, p-1]$。 组内$r$，我们计算字符频率并计算需要多少更改才能使所有字符相等。 该组的成本是组的大小减去最常见的字符数。 对所有组进行求和即可得到该期间所需的总编辑量。 

这是正确的，因为一旦固定了一个周期，各组都是独立的：为一个残基类别选择最后一个字母不会影响另一个残基类别。 

暴力成本是$O(n^2)$在最坏的情况下，因为对于每个$n$我们扫描所有的时期$n$人物。 

关键的观察是$k$很小，所以我们只关心完全不匹配最多的解决方案$k$。 这允许修剪并且还激励有效地计算不匹配结构，而不是以繁重的方式从头开始为每个周期重新计算。 由于每个组都独立贡献，因此我们可以通过每个周期迭代一次字符来累积不匹配，这在给定约束的情况下已经足够最优。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有时期的暴力破解并完全重新计算 |$O(n^2)$|$O(1)$额外 | 太慢了 |
 | 通过分组计数进行经期检查 |$O(n^2)$最糟糕的是，$O(nk)$约束下实用|$O(26)$| 已接受 |

 ## 算法演练

 中心思想是测试每个可能的周期并计算必须更改多少个字符才能使字符串在该周期下保持一致。 如果成本超过，我们会提前停止$k$。 

1. 对于每个候选期$p$从 1 到$n$，假设字符串是通过重复长度的块形成的$p$。 我们将验证是否可以在短时间内实现$k$编辑。 
2. 在此期间，按余数对索引进行分组$p$。 每个组必须以最终字符串中的单个重复字符结束。 
3. 对于每组，计算每个字符出现的次数。 该组的最佳选择是最常见的角色，因为我们希望尽量减少变化。 
4. 组的成本是组大小减去最大频率。 将此成本添加到当前的运行总计中$p$。 
5. 如果在任何时候总成本超过$k$，我们提前停止处理该期间，因为它无效。 
6.如果处理完所有组后总成本最多为$k$，我们立即返回“是”。 
7. 如果没有任何期限，请返回“否”。 

它起作用的原因来自于结构分解：一旦一个周期固定，每个位置仅受其剩余类别的约束。 这些类不交互，因此可以针对每个类独立计算最佳编辑。 该算法搜索由周期选择引起的所有可能的分解，并通过计算每次分解的最小编辑来准确检查可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(n, k, s):
    s = s.strip()
    
    for p in range(1, n + 1):
        changes = 0
        
        for r in range(p):
            freq = [0] * 26
            cnt = 0
            
            for i in range(r, n, p):
                c = ord(s[i]) - 97
                freq[c] += 1
                cnt += 1
            
            if cnt == 0:
                continue
            
            changes += cnt - max(freq)
            if changes > k:
                break
        
        if changes <= k:
            return True
    
    return False

def main():
    t = int(input())
    out = []
    for _ in range(t):
        n, k = map(int, input().split())
        s = input().strip()
        out.append("Yes" if solve_one(n, k, s) else "No")
    print("\n".join(out))

if __name__ == "__main__":
    main()
```实施直接遵循期间枚举策略。 内部循环通过步幅步进对索引进行分组$p$，确保每个残留类别在每个周期精确处理一次。 大小为 26 的频率数组使每个字符的字符计数时间保持恒定。 

早间休息的时候`changes > k`对于性能至关重要。 如果没有它，即使已经无效，我们也总是会处理完整的周期。 

一个常见的错误是忘记每个残基类别都是独立的，从而导致错误地尝试混合组。 另一个是在嵌套循环内低效地重新计算频率； 这里每个周期它保持线性。

        res = []
        for _ in range(t):
            n, k = map(int, input().split())
            s = input().strip()

                        return True

 ## 边缘情况

 一个关键的边缘情况是当$k = 0$。 在这种情况下，算法必须拒绝在某个时期内尚未完全一致的任何字符串。 例如，输入`ab`和$k=0$只适用于$p=2$，因为任何较小的周期都会导致不匹配。 该算法可以正确处理这个问题，因为每个重要的分组都会立即贡献正成本。 

另一个边缘情况是最佳周期是$p = n$。 在这种情况下，每个字符形成自己的组，因此无论字符串如何，成本始终为零。 该算法总是立即接受这一点$k \ge 0$，这是正确的，因为我们可以选择任何$p \le n$和$p=n$不施加任何限制。 

第三种情况是高度重复的字符串，例如`aaaaaa`。 每个周期的成本为零，因此算法在第一次有效检查时接受。 这证实了基于频率的成本计算不依赖于实际的字符多样性，并且在简并分布下保持稳定。
