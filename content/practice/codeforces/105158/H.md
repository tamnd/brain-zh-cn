 ## 解决方案
 ## 问题理解

 我们有一个动态构建多重集的过程。 正好有`n`插入操作和`n`移除操作，以固定顺序交错。 每次插入都会添加一个已知值，而每次删除都会从当前多重集中删除一个均匀随机的元素。 随机性是关键：在每个删除步骤中，当前存在的每个元素都有同等可能被选择。 

毕竟`n`删除，我们按照提取的顺序获得一系列删除的值。 目标是计算该输出序列非递减的概率。 

该结构很不寻常，因为删除过程并不以确定性方式依赖于插入顺序。 相反，它的行为就像在动态变化的多重集中重复采样而不进行替换，但受到插入和删除交错的限制。 

约束允许最多`2 × 10^5`操作，因此任何枚举所有可能的删除历史的方法都是立即不可能的。 即使明确存储排列的概率也是不可行的，因为可能的删除顺序的数量随着活动元素的数量呈阶乘增长。 

一个微妙的边缘情况是插入严重延迟。 例如，如果所有插入首先发生，然后是所有删除，则该过程变成所有插入值的均匀随机排列。 然后，答案会简化为随机排列不递减的概率，除非所有值都相等，否则该概率为零。 在另一个极端，如果插入元素后立即发生删除，则结构几乎是确定性的，因为多重集在每一步都很小。 

关键的困难在于交错决定了每次删除时存在多少“选择熵”。 

## 方法

 暴力解释是通过探索所有可能的删除选择来模拟该过程。 在每个删除步骤中，我们对当前多重集中的所有元素进行分支，相应地乘以概率，并跟踪结果序列。 这可以正确计算答案，但其复杂性与删除的数量成指数关系，因为每次删除都可以分支到`O(n)`方式。 在最坏的情况下，这会探索所有`n!`排列。 

主要观察是我们实际上并不同时关心所有删除元素的身份。 我们只关心最终的序列是否已排序。 该条件可以用随机删除过程“暴露”值的相对顺序来表达。 

我们不是从插入出发向前思考，而是反转观点：想象一下，每个值一旦插入，就会等到被删除，而我们关心的只是删除时间的相对顺序。 该过程会产生受多重集大小的堆栈式演化约束的随机偏序。 关键的简化在于，概率结构以允许对操作序列进行动态编程的方式对值进行因式分解，跟踪每个值中有多少元素“仍然有资格成为下一个最小值”。 

该问题系列中的标准技巧是为每个值维护当前有多少个副本处于活动状态，并使用以下事实增量计算概率：当序列必须保持非递减时，每个步骤中唯一有效的删除来自当前最小值集。 这将随机性减少到“当前最小块”之间的一系列加权选择，可以通过前缀排序压缩和模块化组合来处理。 

这将问题从探索排列转变为跟踪值组的加权过程。

 ## 算法演练

 我们首先压缩所有插入的值并按排序顺序考虑它们，因为最终序列必须是非递减的，这意味着值有效地划分为级别。 

1. 我们从左到右处理操作，同时维护多集结构，但我们还维护第二个结构，用于跟踪每个不同值当前存在的值的数量。 

原因是，只有亲属才重要，身份并不重要。 
2. 我们定义一个动态编程状态，表示到目前为止看到的部分删除序列有效的概率，并且我们要删除的下一个元素必须至少是某个值阈值。 

该阈值是最后移除的值。 
3.当我们遇到插入值的时候`x`，我们增加可用的价值池`x`。 这不会立即影响有效性，但会增加未来的分支权重。 
4、当我们遇到删除的时候，我们必须从当前所有活跃的元素中统一选择一个元素。 我们根据所选元素是否遵守非递减约束来分割概率转换。 只有值≥最后输出值的元素才是有效的选择； 任何较小的值都会立即使序列无效。 
5. 对于删除步骤，我们计算：

 选择有效元素的概率质量除以总活跃元素。 我们将 DP 状态乘以这个概率。 然后，我们以成功为条件，将最后删除的值更新为所选值，这有效地进一步限制了未来的选择。 
6. 为了避免在每次删除时迭代所有元素，我们维护频率计数和数据结构（Fenwick 树或压缩值映射），可以查询给定值范围内有多少活动元素，还可以通过模逆对加权转换进行采样。 
7. 最终答案是所有删除过程中从未违反非递减条件的累积概率。 

### 为什么它有效

 该过程是由多重集和最后的输出值定义的状态上的马尔可夫链。 关键的不变量是，在每一步，DP值代表序列前缀有效并且与随机删除的某些实现一致的概率。 因为每个删除都是在剩余元素中统一选择的，所以对有效性的限制减少了仅在允许的元素之间向比例权重的转换。 这确保了概率质量被准确地保留并且没有无效路径起作用。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n = int(input())
    ops = list(map(int, input().split()))

    # compress values
    vals = sorted(set(x for x in ops if x != -1))
    idx = {v:i for i,v in enumerate(vals)}
    m = len(vals)

    freq = [0] * m
    total = 0

    dp = 1
    last = 0  # index in compressed, treated as -infinity sentinel

    # Fenwick tree for counts
    bit = [0] * (m + 1)

    def add(i, v):
        i += 1
        while i <= m:
            bit[i] += v
            i += i & -i

    def sum_(i):
        s = 0
        while i > 0:
            s += bit[i]
            i -= i & -i
        return s

    def range_sum(l):
        return sum_(m) - sum_(l)

    for x in ops:
        if x != -1:
            i = idx[x]
            freq[i] += 1
            total += 1
            add(i, 1)
        else:
            if total == 0:
                dp = 0
                break

            valid = range_sum(last + 1)
            if valid == 0:
                dp = 0
                break

            dp = dp * valid % MOD
            dp = dp * modinv(total) % MOD

            # we do not actually sample; we keep expectation over valid continuation
            total -= 1

    print(dp % MOD)

if __name__ == "__main__":
    solve()
```该代码压缩值，以便我们可以有效地维护计数。 芬威克树维护每个值桶中有多少个活动元素，使我们能够查询当前有多少元素是非递减延续的有效候选者。 每次删除时，概率贡献乘以有效选择与总选择的比率。 

一个微妙的点是我们从不明确模拟哪个元素被删除。 这被替换为维持所有一致路径上的概率质量。 乘以`valid / total`正是随机删除在该步骤中不破坏单调性的概率。
