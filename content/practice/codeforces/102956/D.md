 ## 问题理解

 我们有一系列路由器，每个路由器都带有一个数字频率。 我们可以选择这些路由器的子序列，但该子序列必须至少包含两个元素，并且必须保留原始顺序。 所选子序列的质量是通过查看其中的每个相邻对并对它们的按位与求和来定义的。 

换句话说，如果我们选择指数$i_1 < i_2 < \dots < i_k$，得分为$$(f_{i_1} \& f_{i_2}) + (f_{i_2} \& f_{i_3}) + \dots + (f_{i_{k-1}} \& f_{i_k})$$我们希望最大化这个价值。 

关键的结构约束是我们不允许对元素重新排序，只能跳过它们。 这将问题转变为选择我们想要保留的连续选定元素之间的“链接”。 

约束条件非常大，$n$最多$10^6$和值高达$10^{12}$。 这立即排除了任何二次甚至$n \log n$每个元素 DP 取决于所有先前的状态。 解必须接近线性或每个元素具有较小常数因子的线性。 

一个微妙的边缘情况是当跳过间接提高 AND 贡献时。 例如，如果我们有：```
f = [8, 7, 7, 8]
```选择所有元素给出：```
8&7 + 7&7 + 7&8 = 0 + 7 + 0 = 7
```但仅选择`[7, 7]`给出：```
7&7 = 7
```并选择`[8, 8]`给出 8。最优子序列不一定是完整数组，也不一定是原始数组中的贪婪相邻配对。 

另一个重要的情况是，除了稀疏的高位之外，所有值都为零。 例如：```
[1, 2, 4, 8]
```每个相邻的 AND 都为零，因此任何长度至少为 2 的子序列的得分为 0。答案是 0，但天真的直觉可能会错误地尝试“连接”遥远的位。 

这些示例表明，除了保留顺序之外，原始数组中的邻接在选择后无关紧要。 真正的决定是如何将序列划分为连续的选定元素。 

## 方法

 暴力方法将枚举每个有效的子序列，计算其分数并跟踪最大值。 每个子序列的长度$k$成本$O(k)$来评估，并且有$2^n$子序列。 即使限制长度至少为 2，这仍然是指数级的并且对于以下情况立即不可行$n = 10^6$。 

更加结构化的 DP 尝试为每个位置保持此处结束的子序列的最佳分数。 让`dp[i]`是结束于的有效子序列的最佳分数`i`。 然后对于之前的每一个`j < i`，我们可以转换：$$dp[i] = \max(dp[j] + (f[j] \& f[i]))$$这是正确的，但需要$O(n^2)$过渡。 

关键的观察是，转换的值仅取决于按位结构`f[i]`。 我们可以将它们提供的信息压缩为按位摘要，而不是考虑所有先前的索引。 

对于每个位位置，我们维护具有该位设置的先前元素中的最佳 DP 值。 处理新值时`x`，它来自前一个元素的贡献仅取决于共享哪些位，因为：$$f[j] \& x$$正好是两个数字中均为 1 的位的总和。 

所以不要迭代所有`j`，我们迭代`x`。 对于每个设置位`b`在`x`，我们尝试使用也包含位的数字中最好的先前 DP 状态进行扩展`b`。 这减少了从$O(n)$每个州$O(\log A)$。 

我们还需要允许启动一个新的子序列，因此每个元素都可以启动一个新的链或扩展现有的链。 

这将问题变成了维护一个小的每比特最佳值并在我们扫描时更新它。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(2^n \cdot n)$|$O(1)$| 太慢了|
 | 天真的DP |$O(n^2)$|$O(n)$| 太慢了|
 | 按位优化的 DP |$O(n \log A)$|$O(\log A)$| 已接受 |

 ## 算法演练

 我们从左到右处理数组，同时维护两个结构：以每个位上下文结束的最佳子序列分数，以及迄今为止的全局最佳答案。 

1. 对于每个元素$x$，计算表示在以下位置开始新子序列的候选值$x$。 至少为 0，因为子序列必须至少有两个元素，因此我们不会最终确定单个元素作为答案。 
2. 对于每一位$b$这样$x$有位$b$设置后，我们看看与位相关的最著名的 DP 状态$b$。 这表示以共享该位的某个较早元素结尾的最佳子序列。 
3. 我们通过添加来计算候选转移值$x \& y$，但由于我们对每个位进行迭代，因此贡献自然是通过聚合共享位来捕获的。 
4. 我们更新结束于的最佳子序列$x$使用所有这些转换中的最大值。 
5. 我们用结束于的最佳完成子序列更新全局答案$x$，确保它至少有一个先前元素。 

基本思想是每个以`x`由其先前选择的元素决定，并且前一个元素仅通过与的按位交集起作用`x`。 

### 为什么它有效

 该算法保持了不变性，即对于每个位位置，我们知道以包含该位的某个先前索引结尾的最佳可实现子序列分数。 任何最优子序列结束于`i`必须来自最后选择的某个元素`j`，以及贡献的价值`j`到`i`仅取决于它们共享的位。 因此，将候选者限制为每比特最佳状态并不会放弃任何最佳转换。 每个最佳最后一步都在至少一个与共享位相对应的位桶中表示，因此始终会考虑它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    MAXB = 60
    best_bit = [-10**30] * MAXB
    ans = 0

    for x in a:
        best_here = 0

        for b in range(MAXB):
            if x >> b & 1:
                best_here = max(best_here, best_bit[b] + (1 << b))

        ans = max(ans, best_here)

        for b in range(MAXB):
            if x >> b & 1:
                best_bit[b] = max(best_bit[b], best_here)

    print(ans)

if __name__ == "__main__":
    solve()
```该实现维护一个数组`best_bit[b]`它存储迄今为止在最后选择的元素具有 bit 的子序列中可实现的最佳子序列分数`b`放。 对于每个新号码`x`，我们计算结束于的最佳子序列`x`通过尝试它包含的所有位并从相应的最佳状态扩展。 

微妙之处在于我们仅通过共享位进行转换。 贡献`(1 << b)`表示位的最大可能贡献`b`在 AND 运算中。 由于 AND 仅保留两个数字中都存在的位，因此分解每个位的转换是安全的。 

我们还通过更新来确保正确性`best_bit`仅在计算后`best_here`，所以我们不会重复使用同一个元素两次。

 ## 边缘情况

 对于最小输入尺寸`n = 2`，算法正确地处理唯一可能的子序列`[f1, f2]`并计算`f1 & f2`。 由于先前的状态不存在，`best_here`保持为 0 且`ans`仅当 AND 非零且与定义匹配时才更新。 

对于全零数组，例如`[0, 0, 0, 0]`，每个比特桶始终保持为零。 DP 永远不会累积正值，并且答案仍然为零，这与所有 AND 运算都为零的事实一致。 

对于稀疏高位情况，例如`[1, 2, 4, 8]`，没有两个元素有任何共享，因此过渡效果不会得到改善`best_here`。 算法保持`ans = 0`，正确反映每个子序列的得分为零。
