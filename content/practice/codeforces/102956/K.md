 ## 解决方案
 ## 问题理解

 我们有一堆垂直的架子，每个架子都有一个耐用性阈值。 从顶部算起的第 i 个架子只能容纳有限数量的球通过级联过程间接放置在其上。 我们反复将相同的球扔到选定的架子上。 如果书架上积累的负载量超过其阈值，它就会破裂，并且导致破裂的球会根据固定规则向下“溢出”，该规则会分割负载并将部分负载进一步推向书柜下方。 

重要的是，单个球的放置不仅仅影响一个架子。 较高货架上的故障会向下传播，可能会导致连锁反应破坏多个连续的货架。 我们可以选择每个球最初放置的位置，并且我们希望使用尽可能少的球来保证前 k 个架子最终会破裂。 

输出是一个包含 n 个值的序列。 第 k 个值是所需的最小球数，以便通过最佳放置策略，在某个时刻前 k 个架子全部被打破。 

约束 n ≤ 70 足够小，子集或状态上的指数结构原则上是合理的，但前提是状态被严重压缩。 ai 值也很小（≤ 150），这表明该过程受相对较小的整数容量控制，并且重复的加倍或减半行为可能是相关的。 

幼稚的解释可能建议模拟所有可能的放置序列并跟踪生成的级联。 这会立即失败，因为即使对于较小的 k，可能的放置序列的数量也会随着球的数量呈指数增长。 另一种诱人但不正确的想法是独立对待货架，但级联规则使独立性变得错误：打破一个货架会改变到达所有较低货架的有效负载。 

当假设负载分布的单调性时，会出现微妙的故障情况。 例如，人们可能会认为将所有球放在架子 1 上始终是最佳选择。 一般来说，情况并非如此，因为在较低的货架上进行有针对性的放置可以减少浪费的溢出并提高打破多个货架的效率。 

## 方法

 蛮力策略会尝试将每个球的放置情况建模为一个决策：选择一个架子，模拟级联，并跟踪所有架子的最终状态。 每一步都会改变整个配置，因此状态空间变得巨大。 即使我们将自己限制在 k 个最上面的架子上，球在架子之间分配的方式也会随着球的数量呈指数级增长，并且每次模拟本身都可以触发一系列破坏事件。 在最坏的情况下，如果我们尝试 O(2^n) 种放置模式，甚至为 m 个球尝试 O(n^m) 种分布，这是不可行的。 

关键的观察结果是，该过程具有非常强的结构简化：当架子破裂时，向下传递的负载量不是任意的，而是由架子的阈值确定性地确定。 每个架子的行为就像一个阈值放大器，将输入负载转换为固定的输出负载。 这表明我们不应该模拟单个球，而应该跟踪触发 k 个连续断裂所需的最小“有效负载”。 

关键的见解是将系统视为转换的组合。 每个货架在损坏时都会将传入的负载 x 转换为新的负载地板 (x/2)。 由于所有货架的行为相似但阈值不同，因此系统成为分层转换链。 问题简化为找到最小初始负载，在重复阈值触发变换后，保证前 k 层按顺序激活。

这自然会导致对货架前缀的动态规划公式。 对于每个前缀长度 k，我们维护强制级联所需的最小球数，该级联从顶部开始恰好破坏那些 k 个架子，并考虑到向下传播的剩余负载。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 球中的指数 | O(n) | 太慢了 |
 | 带有级联建模的前缀DP | O(n^2) 或 O(n^2 log A) | O(n) | 已接受 |

 ## 算法演练

 我们从上到下处理货架，并为每个前缀 k 计算确保所有前 k 个货架最终在最佳放置下断裂所需的最小初始负载。 

1. 我们将状态 dp[k] 定义为所需的最小球数，以便可以使前 k 个架子完全突破某些放置顺序并引发级联。 这个定义是全局性的策略，不依赖于固定的放置顺序。 
2. 我们初始化 dp[0] = 0，因为打破零架子不需要球。 
3. 我们从 1 到 n 迭代 k，并尝试通过考虑如何强制在已解决的大小为 k-1 的前缀之上打破第 k 个架子来确定 dp[k]。 
4. 对于每个候选先前状态 j < k，我们考虑放置球的效果，以便货架 j+1 到 k 涉及级联故障链。 从 j 扩展到 k 的成本取决于必须向货架 j+1 注入多少负载，使其在通过中间中断反复减半后达到阈值 a_k。 
5. 关键的过渡是，要使货架 k 在从 k-1 向下的一系列中断之后中断，我们必须确保第 k 个中断之前的传入负载至少为 a_k，但是通过先前的中断重复将楼层划分为 2，该负载可能会减少。 这导致了相反的加倍结构：较高货架所需的负载随着距目标货架的距离呈指数增长。 
6. 我们使用这些加倍效应计算候选成本，累积所有可能的先前断点的最小值。 
7. 每个 k 的答案是 dp[k]，按顺序打印。 

### 为什么它有效

 不变的是，在处理 dp[k] 后，我们捕获了最小可能的初始配置，该配置强制在前 k 个架子上进行完全级联的中断。 任何有效的策略都必须引发一系列货架故障，并且每次故障都会以仅取决于当前货架的确定性方式转换传入负载。 由于这些转换是组合性且单调的，因此任何最佳策略都可以重新排列成规范形式，其中货架按自上而下的顺序分解，而不失一般性。 这样就无需考虑交错或混合放置策略，从而确保 DP 捕获所有最佳行为。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # dp[k] = minimal balls to break first k shelves
    dp = [0] * (n + 1)

    # We interpret dp incrementally
    for k in range(1, n + 1):
        best = float('inf')

        # try last "activation point"
        for j in range(k):
            cost = dp[j]

            # compute cost to force shelf j+1..k cascade
            # effective load amplification over (k-j) layers
            x = a[k-1]
            for _ in range(k - j - 1):
                x = 2 * x + 1

            cost += x
            if cost < best:
                best = cost

        dp[k] = best

    print(*dp[1:])

if __name__ == "__main__":
    solve()
```该代码通过前缀逐步构建答案。 对于每个 k，它考虑将过程拆分为先前解决的前缀 j 和从 j+1 到 k 的新级联段。 内部循环通过反转每个货架断裂的减半效应来构造该段顶部的最小所需负载，这解释了递推式 x = 2x + 1。这种递推式来自于级联负载的楼层划分效应的反转。 

转换成本 dp[j] + x 反映了我们首先最优地打破前 j 个架子，然后独立触发剩余段的级联。 j 的最小化捕获了最终级联结构的所有可能的分解点。 

该实现依赖于 n ≤ 70 的事实，因此具有 O(n) 内部重建的 O(n^2) 循环就足够了。

 ## 边缘情况

 一个关键的边缘情况是所有货架都具有相同的阈值。 在这种情况下，任何将多个中断级联在一起的尝试都不会降低成本，因为每个架子仍然需要独立激活。 DP正确地分别评估每个分割j，发现没有多层放大可以改善结果。 

当最后一个架子的阈值比所有先前架子的阈值大得多时，就会出现另一种边缘情况。 该算法自然地隔离了最后一个转换点，因为通过较小阈值的任何级联都会不必要地放大所需的负载。 

最终的边缘情况是 n = 1。算法立即返回 a[1]，因为唯一有效的策略是直接用其阈值数量的球直接打破单个架子。
