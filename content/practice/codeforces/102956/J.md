 ## 解决方案
 ## 问题理解

 我们有一个通过无向电缆连接的计算机网络，我们需要选择一个计算机子集在两个同时规则下“激活”。 首先，没有两台选定的计算机通过电缆直接连接，因此选定的组在图形方面必须是独立的。 其次，每根电缆必须至少接触一个选定的端点，因此每条边都必须被选定的顶点覆盖。 在满足这两个约束的所有集合中，我们想要一个具有尽可能少的所选计算机数量的集合，并且我们必须输出它的大小。 如果不存在这样的集合，则答案是不可能的。 

约束达到三十万个顶点和边，这立即排除了任何二次甚至接近的东西。 任何解决方案在图的大小上都必须基本上是线性的，因为即使像 n log n 这样的小因素也是可以接受的，但任何以嵌套方式重复探索边缘的东西都会超时。 

定义本身隐藏着一个微妙的问题：这样的集合总是存在，或者当它存在时它具有简单的结构，这一点并不明显。 另一个潜在的陷阱是假设可以贪婪地选择集合而不考虑边缘的全局一致性。 

一些边缘情况阐明了结构：

 如果图有一个三角形，例如1-2-3-1，那么任何独立集合最多只能包含一个顶点，但任何单个顶点都无法覆盖所有边。 例如，选择顶点 1 会使边 2-3 未被覆盖。 该输入必须返回-1。 

如果图根本没有边，则每个集合都是独立的并且顶点覆盖是空的。 然而，该问题要求所选集合非空，因此正确答案变为 1，而不是 0。 

如果图是像 1-2-3-4 这样的简单链，那么选择顶点 {1, 3} 或 {2, 4} 都满足条件，并且最小大小取决于我们如何划分图。 

这些例子已经表明，该结构与两党性密切相关。 

## 方法

 暴力尝试将尝试所有顶点子集并检查这两个条件。 对于每个子集，我们将通过检查所有边缘来验证独立性，并通过再次扫描所有边缘来验证覆盖范围。 这导致大约 O(2^n · m)，对于 n 高达 300,000 来说这是完全不可行的。 

重写这两个条件可以得到更结构化的观察。 如果集合 S 是独立的，则没有边在 S 中具有两个端点。如果 S 也是顶点覆盖，则每条边在 S 中至少有一个端点。结合这两个语句强制每条边在 S 中恰好有一个端点。这意味着每条边必须位于 S 及其补集 V \ S 之间。 

这立即意味着该图是二分图，其中 S 形成一个颜色类，V \ S 形成另一个颜色类。 相反，在任何二分图中，二分的任一侧都是独立的，并且也覆盖所有边。 因此，问题简化为检查图是否是二分图，如果是，则在每个连接的组件中选择较小的一侧。 

每个连接的组件都可以用两种颜色着色。 在组件内，我们可以选取颜色 0 的所有顶点或颜色 1 的所有顶点。为了最小化总大小，我们为每个组件独立选择较小的颜色类。 唯一剩下的复杂性是该问题禁止空集，因此如果图根本没有边，我们仍然必须返回 1 而不是 0。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(2^n·m) | O(2^n·m) | O(n) | 太慢了|
 | 二分着色 | O(n + m) | O(n) | 已接受 |

 ## 算法演练

 我们使用 BFS 或 DFS 分别处理每个连接的组件，同时分配两种颜色。

1. 我们迭代所有顶点。 如果一个顶点还没有被访问过，我们就从它开始一个 BFS。 
2. 在 BFS 期间，我们分配起始顶点颜色 0 并沿边传播交替颜色。 如果我们看到邻居已经用相同的颜色着色，则该图不是二分图，我们立即返回 -1。 这是必要的，因为这种冲突意味着某些边缘迫使两个端点进入同一侧，从而破坏了所需的结构。 
3. 对于每个连接的组件，我们维护有多少个着色为 0 的顶点和有多少个着色为 1 的顶点。 
4. 完成一个组件后，我们将 min(count0, count1) 添加到答案中，因为我们可以自由选择更便宜的一侧作为该组件的选定集。 
5. 处理完所有组件后，如果总答案为零，我们将其替换为 1，因为该问题需要非空选择集。 

### 为什么它有效

 着色强制相邻顶点始终位于相反的集合中。 这保证了每个颜色类别的独立性。 当一条边的两个端点需要相同的颜色时，就无法对图进行分区，以使每条边在相对侧都有端点，这意味着不存在有效的集合。 一旦找到有效的二分，每条边都会在每个颜色类中自动拥有一个端点，因此选择整个颜色类可确保顶点覆盖属性，同时保持独立性。 最小化每个组件就足够了，因为组件是断开连接的，因此决策不会交互。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    color = [-1] * n
    ans = 0

    for i in range(n):
        if color[i] != -1:
            continue

        q = deque([i])
        color[i] = 0
        cnt = [1, 0]

        while q:
            u = q.popleft()
            for v in g[u]:
                if color[v] == -1:
                    color[v] = color[u] ^ 1
                    cnt[color[v]] += 1
                    q.append(v)
                elif color[v] == color[u]:
                    print(-1)
                    return

        ans += min(cnt[0], cnt[1])

    if ans == 0:
        ans = 1

    print(ans)

if __name__ == "__main__":
    solve()
```邻接表有效地存储图以支持线性遍历。 BFS 分配颜色并同时计算组件中每个分区的大小。 BFS 内部的冲突检查是关键的正确性门，因为它会立即检测非二分结构。 

对空答案的最终调整处理没有边缘的退化情况，因为在这种情况下，每个顶点都是孤立的，并且所有组件对最小值的贡献为零。

 ## 边缘情况

 对于空边图，该算法将所有组件生成为孤立的顶点，每个组件对答案的贡献为零。 由于累加值变为零，因此最终调整会强制输出为 1，满足即使不存在边，所选集合也不能为空的要求。 

对于单个奇数循环（例如三角形），BFS 着色不可避免地会在遍历中的某个点为两个相邻顶点分配相同的颜色。 当访问循环的结束边缘时，会立即检测到冲突，并且算法返回 -1，而不需要处理图形的其余部分。 

对于断开的二部图，每个组件都是独立处理的。 即使一个组件很大而另一个组件是单边，其中一个组件的决策也不会影响另一个组件，因为边缘不交叉组件，因此局部最小化是全局最优的。
