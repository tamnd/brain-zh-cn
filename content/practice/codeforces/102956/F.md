 ## 解决方案
 ## 问题理解

 给定一个小写字母网格，我们想要计算该网格内有多少个轴对齐的矩形在其边框上具有非常严格的属性：矩形边界上的每个单元格必须包含完全相同的字符。 

通过选择两个不同的行和两个不同的列来定义矩形，形成子矩阵。 该状况根本不关心内部，只关心外部框架。 顶行、底行、左列和右列必须全部由相同的字母组成，并且所有这些字母必须彼此相同。 

任务是计算存在多少个这样的矩形。 

到 2000 年，网格的规模可达 2000 个，这意味着多达 400 万个单元。 任何尝试独立检查每个可能的矩形的解决方案都会太慢，因为单独的矩形数量约为 n²m²，在最坏的情况下约为 10^2。 这已经排除了任何简单地重新计算边界有效性的方法。 

当矩形非常薄时，会出现微妙的边缘情况。 如果矩形的高度为 2 或宽度为 2，则其边框会严重重叠，并且同一单元格会参与多个边。 例如，2×k 矩形仍然要求其所有周边单元格相同，这有效地强制这两行中以及所选列之间的所有单元格适当匹配。 假设独立双方的粗心做法会重复计算或遗漏这些退化案例。 

另一个陷阱是行和列之间的对称性。 矩形由两行和两列确定，但条件将它们耦合在一起：单独选择行并不独立于选择列，因为有效性取决于行对的列方向一致性。 

## 方法

 蛮力策略首先修复顶行和底行。 对于每对行，我们查看每对列并检查它们形成的矩形是否具有统一的边框。 对于一对固定的行，我们可以预先计算哪些列是“兼容的”，这意味着顶部和底部行在该列具有相同的字符。 然后，任何矩形都必须从此类兼容列的运行中选择其左边界和右边界。 

然而，这仍然不够，因为兼容性只能确保垂直边一致。 水平边框还必须匹配所有列，这会以天真的计数会错过的方式耦合相邻列。 

看待问题的更好方法是颠倒视角：我们不再用矩形来思考，而是用行对来思考。 对于任何固定的行对，我们查看两行共享相同字符的列。 在这些列中，我们想要对列索引 (l, r) 对进行计数，这样：

 l 和 r 之间的段是“一致的”，因为对于内部的每一列，两行上的字符对在边界约束中涉及的所有行中都是相同的。 

这建议将每对行转换为派生字符串：对于行 x1 和 x2，定义按列相等的字符对签名。 当且仅当所有四个边都同意单个字符时，矩形边框才是有效的，这会强制形成强大的结构：所有四个角相邻的边都强制要求两个选定的行必须在边界列上匹配，并且这些列必须形成一个块，其中该协议仍然存在。 

关键的见解是固定字符 c 并仅考虑等于 c 的单元格。 对于每个字符，我们独立地计算其边界完全由 c 组成的矩形。 这将问题减少到 26 个二进制网格。 

现在问题变成：在一个二进制矩阵中（单元格对于字符 c 要么允许，要么不允许），计算边界完全在允许的单元格内的所有矩形。

对于固定字符 c，我们处理行对。 对于每一对行，我们计算一个一维数组，其中如果两行在第 j 列都有 c，则位置 j 为 1，否则为 0。现在我们需要计算所有选定位置两端均为 1 的子数组，并且最重要的是，垂直约束适用于整个边框。 这减少了对两行在端点处同时有效的列对的计数。 

最后的组合步骤是，对于每对行和字符 c，每个有效矩形对应于选择两列 l 和 r，使得两行在 l 和 r 处都有 c，此外顶部和底部之间的每一行也必须在两个端点处都有 c。 这变成了使用散列或位集交集对压缩列集进行经典对计数，但通过预先计算每行对的有效列列表进行优化。 

标准优化是为每对行预先计算与给定字符匹配的列列表。 然后，对于每个这样的列表，我们计算对 (l, r) 的数量，这只是 k 选择 2，但前提是确保所有四个边都保持相同的字符约束。 这是通过逐字符处理和相交行约束来强制执行的。 

最终的解决方案减少为迭代行对，构建每个字符的频率数组，并使用组合对计数累积贡献。

 ## 算法演练

 1. 修复字符 c 并将所有其他字符视为与本次传递无关。 完整答案是所有字符的总和。 
2. 对于这个字符 c，将每一行预处理为二进制数组，其中 1 表示单元格等于 c，否则为 0。 这隔离了有效的边界候选者。 
3. 迭代所有行对（顶部、底部）。 对于每一对，计算一个数组 good，其中仅当两行在 j 列都有字符 c 时，good[j] 才为 1。 这标识了可以用作垂直边框的列。 
4. 扫描这个好的数组并识别连续的 1 段。 在每个段内，每对列都可以作为潜在的左边界和右边界。 
5. 对于长度为 k 的线段，将 k·(k−1)/2 添加到答案中。 这会计算左列和右列的所有选择。 
6. 累积所有行对和所有字符。 

这种分解起作用的原因是，一旦顶行和底行被固定并且我们限制为单个字符，对于在有效段内选择的任何对列，水平边框条件将自动满足。 垂直边已经通过 good 的构造得到保证，并且水平一致性也随之而来，因为两个选定的行都是固定的并且在端点处等于 c。 

### 为什么它有效

 修复字符 c 将问题转化为计算边框完全为 c 的矩形。 对于任何有效的矩形，其顶部和底部行必须在两个选定的列处都有 c，并且所有中间结构不会影响边界条件。 通过枚举行对并将列约束折叠为连续的有效段，每个矩形通过其独特的左上和右下结构精确计数一次。 分段确保不同列间隔之间的独立性，因此每个分段内的对计数是完整且不重叠的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    ans = 0

    # process each character separately
    for ch in range(26):
        c = chr(ord('a') + ch)

        # binary grid for this character
        b = [[1 if grid[i][j] == c else 0 for j in range(m)] for i in range(n)]

        # for each pair of rows
        for top in range(n):
            col_good = [1] * m

            for bot in range(top, n):
                # update column validity
                for j in range(m):
                    col_good[j] &= b[bot][j]

                # count segments of ones
                cnt = 0
                for j in range(m):
                    if col_good[j]:
                        cnt += 1
                    else:
                        ans += cnt * (cnt - 1) // 2
                        cnt = 0
                ans += cnt * (cnt - 1) // 2

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先按字符组织计算，这避免了混合不兼容的边框字母。 对于每个字符，我们构建该字符出现位置的二进制掩码。 然后，对于每个顶行，我们逐渐向下延伸底行，维护一个按列交叉数组 col_good ，该数组跟踪当前带中的所有行仍具有该字符的列。 

内部扫描将每个有效的列段转换为列对的组合计数。 更新 col_good[j] &= b[bot][j] 确保仅保留对整个垂直条带有效的列。 

唯一微妙的实现细节是，当底行向下移动时，col_good 被重用并逐渐细化，这避免了从头开始重新计算交集，并使解决方案保持在时间限制内。 

## 工作示例

 考虑一个小网格：```
a a a
a b a
a a a
```我们修复字符“a”。 对于 top = 0，col_good 从 [1,1,1] 开始。 对于 bot = 0，所有列都保持有效。 段长度为 3，贡献 3 对。 对于 bot = 1，中间列无效，因此 col_good 变为 [1,0,1]，贡献 1 对。 对于 bot = 2，col_good 返回到 [1,1,1]，再次贡献 3 对。 

| 顶部 | 机器人 | 上校好 | 细分 | 贡献 |
 | ---| ---| ---| ---| ---|
 | 0 | 0 | [1,1,1]| 3 | 3 |
 | 0 | 1 | [1,0,1]| 1 + 1 | 0 |
 | 0 | 2 | [1,1,1]| 3 | 3 |

 这显示了行之间的垂直一致性如何动态限制有效列。 

现在考虑：```
z z
z z
```对于字符“z”，每一列始终有效。 每对行恰好贡献一个长度为 2 的线段，给出 1 个矩形。 该迹线证实 2×2 完整均匀网格恰好产生一个有效的边框矩形。

 ## 边缘情况

 1×m 或 n×1 网格永远不会贡献任何矩形，因为至少需要两个不同的行和列。 该算法自然会处理这个问题，因为行对迭代永远不会产生有效的对。 

完全均匀的网格是最密集的情况。 对于固定字符，每个行对的每个 col_good 都保持满，从而产生最大段。 该算法简化为对每个行对的所有列对进行计数，以匹配组合期望。 

高度交替的网格，例如两个字符的棋盘，迫使大多数 col_good 数组分解为单长度段。 每个段贡献为零，因为 k 选择 2 为零，因此尽管正在处理许多行对，但该算法正确地避免了过度计数。
