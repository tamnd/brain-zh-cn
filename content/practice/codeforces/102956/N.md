 ## 解决方案
 ## 问题理解

 我们从一排参与者开始，每个参与者都有一个初始强度值。 该过程通过一系列相邻的决斗而演变。 每场决斗都会选择两名相邻的玩家，较弱的一名被淘汰，获胜者的实力增加一分。 删除后，该行会闭合，因此邻接关系始终得以保留。 恰好之后$n-1$如此淘汰，只剩下一名球员。 

不确定性不在于机制，而在于赛程安排和决胜局。 您不会被告知在每一步中选择哪一对相邻的对，并且当两个玩家具有相同的实力时，任何一个都可以获胜。 因为每次胜利都会增强力量，所以早期的结果会放大后来的行为，这意味着不同的决斗序列可能会导致不同的最终幸存者。 

任务是确定在某些有效的选择序列下哪些初始索引可能是最终的幸存者。 

限制是极端的：最多$10^6$实力高达$10^9$。 这立即排除了任何显式处理匹配的模拟。 即使每一步重复的线性扫描也已经是二次的了。 唯一可行的解​​决方案是将整个锦标赛减少为几个可在线性时间内计算的全局聚合。 

一个微妙的困难是，该过程不像标准淘汰赛那样是固定的二叉树。 邻接约束意味着消除会在本地传播，获胜者会改变位置，因此原则上任何玩家都可以通过合并链与许多其他玩家间接交互。 

一个天真的错误是假设只有局部最大值才重要。 例如，在数组中$[1, 100, 2]$，人们可能会认为指数 2 一定会获胜，因为它一开始就是全球最强的。 然而，重复的强制配对和通过胜利而成长可以让其他结构占据主导地位，具体取决于淘汰的顺序。 

另一个常见的错误假设是，初始强度单独决定可行性。 事实上，较弱的玩家可以在遇到强大的对手之前，通过反复击败更弱的邻居来获得力量。 

## 方法

 直接模拟尝试逐步对过程进行建模。 在每次操作中，我们都会维护当前的数组，选择相邻的对，解决获胜者，增加其强度，并删除失败者。 这已经花费了$O(n)$每一步的操作数，并且有$n-1$步骤，导致$O(n^2)$。 和$n = 10^6$，这是完全不可行的。 

真正的困难在于玩家的“力量”不是固定的。 每当玩家成为最后的幸存者时，每次获胜的决斗都会累积+1。 这意味着玩家的有效强度取决于其通过淘汰在阵列中移动的距离。 

如果一名球员在这个位置$j$最终变得与位置相邻$i$，它必须准确获胜$|i - j|$一路上的决斗。 每次胜利都会增加+1，因此到达时其强度$i$变成：$$a_j + |i - j|$$这将问题从局部交互转变为全局“线性奖励可达性”结构。 

我们现在重新解释这个表达式：

 对于固定位置$i$，左边的任何玩家都会贡献一个潜在价值：$$a_j + (i - j) = (a_j - j) + i$$右边的任何球员贡献：$$a_k + (k - i) = (a_k + k) - i$$所以对于每个位置$i$，来自左侧的最强可能的传入威胁仅取决于$a_j - j$，并且从右边开始仅取决于最大值$a_k + k$。 

这将整个问题简化为计算两个前缀/后缀变换。 

一个位置$i$只有当它不被来自任何一方的更强大的候选人严格控制时，它才能成为最后的幸存者。 该条件归结为检查它是否在两个变换后的坐标系中同时最优。 

我们比较方法：

 | 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(n²) | O(n) | 太慢了 |
 | 前缀/后缀转换| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 计算数组$L[i]$存储的最大值$a[j] - j$全面的$j \le i$。 这捕获了可以到达位置的最强可能的竞争对手$i$通过移动积累收益后从左侧开始。 
2. 计算数组$R[i]$存储的最大值$a[j] + j$全面的$j \ge i$。 这捕获了从右侧到来的最强大的竞争对手。 
3. 对于每个位置$i$，计算两个潜在的外部威胁：

 最强左派抵达$i$是$i + L[i]$，最强的右到达是$R[i] - i$。 
4. 通过以下方式确定每个职位的全球最大可实现“影响力”：$$\max(i + L[i],\; R[i] - i)$$5. 职位$i$如果其自身的转换值与当时可实现的最佳结构相匹配，则作为可能的最终获胜者是有效的。 具体来说，我们检查它是否在这种主导格局中达到最优，这意味着它并不严格地比两个方向上所有竞争的转换值更差。 
6. 收集满足该条件的所有索引并按升序输出。 

关键思想是，锦标赛动态可以线性化为两个坐标系中类似几何的优势问题，一个向左移动，一个向右移动。 

### 为什么它有效

 每一次淘汰都会为获胜者贡献一个单位的力量，而该单位恰好对应于沿着阵列移动的一步。 这将每个候选者变成一条线，其斜率由位置决定。 影响任何指数的最佳竞争对手完全由变换坐标中的前缀和后缀极值确定。 由于任何比赛序列只能重新排列这些累积的顺序，而不能改变其总贡献结构，因此这两个系统中的主导地位充分表征了生存能力。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    # 1-indexed transformations via arrays
    left_best = [-10**30] * n
    right_best = [-10**30] * n

    # compute max(a[j] - j)
    cur = -10**30
    for i in range(n):
        cur = max(cur, a[i] - i)
        left_best[i] = cur

    # compute max(a[j] + j)
    cur = -10**30
    for i in range(n - 1, -1, -1):
        cur = max(cur, a[i] + i)
        right_best[i] = cur

    res = []
    for i in range(n):
        left_val = i + left_best[i]
        right_val = right_best[i] - i

        # i is valid if it is not strictly dominated by both sides
        if a[i] >= max(left_val, right_val):
            res.append(i + 1)

    print(len(res))
    print(*res)

if __name__ == "__main__":
    solve()
```该实现构建了两次线性扫描，一次从左到右，一次从右到左。 左侧扫描保持最佳值$a[j] - j$，它编码候选人在向右移动时可以积累多少力量。 右侧扫描对向左移动进行对称计算。 

最终条件将每个指数自身的强度与来自两个方向的最强假设到达进行比较。 在两次比较中幸存下来的指数正是那些可以通过一定的决斗序列安排成为最后剩下的参与者的指数。 

一个微妙的实现细节是所有计算都必须在 64 位范围内完成，因为$a[i]$可以很大并且偏移量可达$10^6$累积成中间表达式。

 ## 边缘情况

 单元素数组很简单，因为不会发生决斗，并且唯一的参与者自动成为获胜者。 

全平等数组更加微妙，因为每次决斗都可以任意解决，允许任何索引都有可能积累胜利并根据调度占据主导地位。 

严格单调数组暴露了左右累积之间的不对称性。 在递增序列中，最右边的元素可以通过吸收所有其他元素来积累最有利的胜利，而在递减序列中，最左边的元素占主导地位。 

这些情况与变换后的优势解释一致：在每种情况下，前缀或后缀最大值完全确定生存性，而不需要内部索引之间的任何复杂交互。
