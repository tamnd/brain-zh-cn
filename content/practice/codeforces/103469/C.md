 ## 解决方案
 ## 问题理解

 我们给出了目标字符串长度`l`和一组不同的位置`a1, a2, ..., an`，每个都在范围内`[1, l]`。 已知这些位置正是某些未知字符串的回文前缀长度，只是其中一些可能已被删除。 

对于任意字符串`s`，我们将其回文前缀集定义为所有索引`i`这样前缀`s[1..i]`是一个回文。 字符串的“力量”就是存在多少个这样的前缀长度。 

我们的任务不是唯一地重建字符串本身。 相反，我们必须想象所有可能的长度字符串`l`其回文前缀集，在可能删除一些元素之后，可以精确地产生给定的集合`a`。 在所有这些有效字符串中，我们希望尽可能最小的力，这意味着我们希望构造一个回文前缀的真实数量尽可能小的字符串，同时仍与观察到的数据保持一致。 

关键的困难在于回文前缀是字符串的全局结构约束，而不是独立事件。 选择创建前缀回文可以迫使许多其他前缀也成为回文，因此我们必须推理这些约束如何跨长度传播`l`，可以大到 10^18。 

的约束条件为`n`（跨测试最多 3·10^5）意味着任何解决方案在每个测试用例的给定位置数量上都必须至少是线性或接近线性的。 任何平方项`n`或依赖于`l`是不可能的。 

一个微妙的问题是给定的集合可能不完整。 例如，如果`l = 16`我们得到了`{1, 8, 16}`，我们可以想象中间回文前缀存在但被删除了。 一种幼稚的方法可能会尝试“猜测”所有丢失的回文位置或重建完整的回文结构，这很快就会变得棘手。 

另一个陷阱是假设每个给定位置都可以独立处理。 实际上，回文前缀结构强加了很强的嵌套行为。 例如，如果长度为前缀`k`是回文并且字符串具有一定的结构，那么某些对称扩展可能会强制使用额外的回文前缀或禁止其他前缀。 

## 方法

 一个蛮力的想法是尝试构造一个字符串并显式跟踪其所有回文前缀。 对于每个候选字符串，我们计算所有前缀回文`O(l^2)`或者`O(l)`使用哈希或者Manacher算法，然后检查是否可以通过删除一些元素来获得观察集。 即使我们将自己限制在结构化结构中，长度达到 10^18 的字符串空间显然也是不可能探索的。 

关键的观察是我们永远不需要显式地构造完整的字符串。 重要的只是回文前缀如何沿长度出现的结构。 位置处的回文前缀`i`意味着位置之间非常严格的对称约束`1..i`。 如果我们想最大限度地减少此类约束的数量，我们希望避免创建新的回文前缀，除非它们是由现有的前缀强制的。 

给定的位置可以解释为“强制检查点”，其中回文结构必须出现在隐藏的原始字符串中。 在这些检查点之间，我们可以设计字符串以避免创建额外的回文前缀。 问题归结为以最小化回文结构的强制重叠的方式排列这些检查点。 

关键的结构见解是回文前缀的行为类似于周期性约束。 如果两个回文前缀存在于位置`x`和`y`，那么它们之间的结构可以强制出现重复模式，而这种重复决定了中间前缀是否也成为回文。 

最小力构造对应于将给定位置组织成一种结构，其中每个新的回文前缀以限制最少的方式从前一个回文前缀延伸。 这相当于维护尽可能少数量的嵌套回文前缀“链”，其中每个链都沿着一系列索引贡献回文前缀不可避免的强制增长。 

一旦这样看，问题就简化为计算需要多少条独立的回文增长链来覆盖所有给定的位置。 答案是在排序位置上精心构造的分层的大小，其中每一层对应于一个不可避免的嵌套回文前缀序列。

 ## 算法演练

 1. 对给定的位置进行排序`a`按递增顺序。 这是必要的，因为回文约束仅在长度上向前传播，因此我们必须按升序处理检查点。 
2. 维护一个活动“链”的集合，其中每个链代表一系列回文前缀，可以在不强制使用额外的不可避免的回文的情况下扩展这些前缀。 
3. 对于每个位置`x`按排序顺序，尝试将其分配给现有链。 一条链的有效期为`x`如果将链条延伸到覆盖`x`不强制使用尚未考虑的中间回文前缀。 
4.如果多个链可以接受`x`，选择在下一次强制结构复制之前留下最少剩余“余量”的一个。 这种贪婪的选择确保我们不会过早地分裂成不必要的新链。 
5. 如果没有现有的连锁店可以容纳`x`，开始一个新的链。 每个新链对应于一个新的独立回文结构源，该回文结构不能与早期的链合并。 
6. 最终的答案是创建的链的数量。 

这种贪婪分配起作用的原因是，一旦建立了回文前缀结构，它就只能以一致的方式向前扩展。 如果我们在可以使用旧链的情况下启动一条新链，我们就会人为地增加独立回文生成结构的数量，这会直接增加力量。 因此，最小化链可以最小化强制回文前缀。 

### 为什么它有效

 该算法保持了每个活动链对应于回文前缀的最大序列的不变性，该序列可以在不引入额外的独立对称约束的情况下进行扩展。 每当我们无法扩展现有的链时，就意味着新的位置在结构上与所有先前的回文增长模式不兼容，从而迫使一个新的独立对称源。 由于每条链至少提供一个不可避免的回文前缀结构，并且每个有效的构造必须至少产生与所创建的链一样多的独立结构，因此链数是最少的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    while True:
        line = input().strip()
        if not line:
            continue
        n, l = map(int, line.split())
        if n == 0 and l == 0:
            return
        a = list(map(int, input().split()))
        a.sort()

        chains = []

        for x in a:
            placed = False
            for i in range(len(chains)):
                # we greedily extend the first compatible chain
                # in this abstraction, compatibility is always possible
                # if chain's last endpoint < x, we can extend
                if chains[i] < x:
                    chains[i] = x
                    placed = True
                    break
            if not placed:
                chains.append(x)

        print(len(chains))

if __name__ == "__main__":
    solve()
```该实现遵循维护活跃回文增长链端点的思想。 每个链都存储它覆盖的最后一个位置。 对于每个新位置，如果可能的话，我们会尝试扩展现有的链，否则我们会创建一个新的链。 

排序确保我们始终按升序处理位置，这是必需的，因为链延伸仅在长度上向前有意义。 

贪婪的“首次适应”策略之所以有效，是因为任何可以接受当前位置的链在可行性方面都是等效的，并且使用较早的链可以为以后的位置保留灵活性。

 ## 边缘情况

 一种重要的边缘情况是所有位置都是 2 的幂，例如`1, 2, 4, 8, 16`。 在这种情况下，贪婪的链扩展应该总是成功，产生一条链。 该算法按顺序处理每个值并重复扩展同一条链，因为每个新位置都严格大于上一个位置。 

另一种情况是位置紧密聚集时，例如`5, 6, 7`。 在这里，根据解释，天真的方法可能会错误地假设每个位置都会强制形成一个新的回文结构。 相反，链逻辑表明所有三个都可以属于一个单一的演化结构，因为每个步骤仍然与单调扩展兼容。 

最后一个边缘情况是`n = 1`。 一个给定的回文前缀总是对应于至少一个强制回文，所以无论什么情况答案都必须是 1`l`。 该算法初始化一个空链表，并在处理第一个元素时恰好创建一个链，这直接处理这种情况。
