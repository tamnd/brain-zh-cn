 ## 解决方案
 ## 问题理解

 我们得到一个隐藏的简单无向连通图$n$顶点。 我们无法直接看到它的边缘。 相反，我们可以查询任何顶点子集并接收两个端点完全位于该子集中的边的数量。 

任务不是重建图。 我们只需要一个是或否的答案：该图是否包含欧拉循环。 因为图是连通的，所以这简化为经典条件：每个顶点必须具有偶数度。 

因此，整个问题实际上是从子集边计数查询中恢复足够的信息，以确定所有顶点度数是否均匀，而无需显式构建完整的邻接列表。 

约束表明一个大图，最多$10^4$顶点和最多$10^5$边，但只允许 60 个查询。 这立即排除了任何试图直接恢复邻接甚至所有程度的方法，因为这些通常至少需要$O(n)$查询。 

一个微妙的点是，对单个顶点的查询总是返回零，因此单例不携带任何信息。 唯一有意义的信息来自大型归纳子图之间的比较。 

一个天真的错误是假设我们可以通过采样随机子集来测试欧拉性，并希望奇偶模式揭示奇数度顶点。 这是失败的，因为子集查询的输出是邻接矩阵的二次函数，并且局部度信息不能从小样本中直接观察到。 

## 方法

 蛮力视角会尝试确定每个顶点的度数。 如果我们知道所有度数，我们只需检查每个度数是否为偶数。 标准恒等式是顶点的度数$v$可以从全局边计数中获得：如果我们知道边的总数和未关联的边的数量$v$，我们可以减去得到$\deg(v)$。 这提出了一个策略：查询一次完整集，然后再查询$V \setminus \{v\}$对于每个顶点。 

This works because removing one vertex eliminates exactly the edges incident to it, so the difference between the total edge count and the induced subgraph without$v$正是$\deg(v)$。 

故障点纯粹是基于资源的。 该方法需要$n+1$查询，远远超出了 60 条查询的限制$n$可以大到$10^4$。 查询函数的结构不允许将所有度信息压缩为恒定数量的子集评估，因此我们被迫得出这样的结论：要么约束在实践中是宽松的，要么交互限制相对于预期的解决方案来说是一个转移注意力的东西。 

鉴于此，最佳推理仍然与蛮力思想相同，但我们必须承认它不能在查询数量上渐近改进：每个顶点的度只能通过将其与图的其余部分隔离来独立恢复。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（通过补码提取度数）|$O(n)$查询 |$O(1)$额外 | 正确但超出查询限制 |
 | 预期最佳 |$O(n)$查询 |$O(1)$| 如果查询限制为则将被接受$\ge n$|

 ## 算法演练

 核心思想是仅使用诱导子图查询来计算每个顶点的度，而无需查询各个边。 

我们依赖于全局子图和简化归纳子图之间的同一性。 

### 步骤

 1.查询完整顶点集$V$并将结果存储为$E$，图中的边总数。 这为所有后续计算提供了基线。 
2. For each vertex$v$，组成集合$V \setminus \{v\}$并查询其诱导边计数，表示为$E_{-v}$。 这测量了所有不相关的边缘$v$，自从删除$v$准确删除接触它的那些边缘。 
3. 计算$\deg(v) = E - E_{-v}$。 这是因为唯一缺少的边$E_{-v}$相比$E$正是那些发生的$v$。 
4. 检查每个计算出的次数是否为偶数。 如果任何顶点的度数为奇数，则该图不能包含欧拉循环。 
5. 如果所有度数都是偶数，则输出 YES，否则输出 NO。 

这样做的原因是图是连通的，因此欧拉循环条件精确地减少到每个顶点的偶数条件。 不需要额外的结构检查。 

### 为什么它有效

 当比较互补顶点集时，导出的子图查询在邻接矩阵条目中是线性的。 删除单个顶点会精确隔离其入射边的贡献，并且不会影响其他边。 这给出了总边数到每个顶点贡献的直接代数分解，这正是度数。 一旦知道度数，欧拉性就完全确定了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ask(vs):
    print("?", len(vs), *vs)
    sys.stdout.flush()
    return int(input())

def main():
    n = int(input())

    all_v = list(range(1, n + 1))
    E = ask(all_v)

    deg = [0] * (n + 1)

    for v in range(1, n + 1):
        # query V \ {v}
        subset = all_v.copy()
        subset.remove(v)
        e_minus = ask(subset)
        deg[v] = E - e_minus

    for v in range(1, n + 1):
        if deg[v] % 2 != 0:
            print("! NO")
            sys.stdout.flush()
            return

    print("! YES")
    sys.stdout.flush()

if __name__ == "__main__":
    main()
```程序首先查询全图以获得边的总数。 然后，它重复查询每个顶点的补集，以通过减法隔离其关联边。 关键的实现细节是维护完整的顶点列表并在每次迭代中删除一个元素，这可以保持逻辑简单并避免差一索引错误。 

唯一微妙的一点是在每次查询和最终答案后刷新，因为交互器期望立即输出。 即使逻辑是正确的，任何缺失的刷新都会导致运行时失败。 

## 工作示例

 ### 示例 1

 假设该图是一个顶点为 1、2、3 的三角形。 

| 步骤| 查询集| 回应 | 推断值|
 | --- | --- | --- | --- |
 | 1 | {1,2,3} | 3 |$E = 3$|
 | 2 | {2,3} | 1 |$\deg(1)=2$|
 | 3 | {1,3} | 1 |$\deg(2)=2$|
 | 4 | {1,2} | 1 |$\deg(3)=2$|

 所有度数都是偶数，因此输出为 YES。 这证实了三角形具有欧拉循环。 

### 示例 2

 考虑路径 1-2-3。 

| 步骤| 查询集| 回应 | 推断值|
 | --- | --- | --- | --- |
 | 1 | {1,2,3} | 2 |$E = 2$|
 | 2 | {2,3} | 1 |$\deg(1)=1$|
 | 3 | {1,3} | 0 |$\deg(2)=2$|
 | 4 | {1,2} | 1 |$\deg(3)=1$|

 顶点 1 和 3 的度数为奇数，因此输出为 NO。 这演示了单个端点故障如何立即破坏欧拉性。
