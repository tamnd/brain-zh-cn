 ## 解决方案
 ## 问题理解

 我们从从 1 到 n 的一系列标签开始，按升序排列。 唯一允许的操作是在当前序列中选取两个相邻元素，将它们都删除，并支付取决于两个删除元素的原始标签的成本。 删除一对后，数组会缩小并形成新的邻接关系，因此未来的选择在很大程度上取决于之前的删除。 

这个过程一直持续到什么都没有剩下为止。 由于每个操作恰好删除两个元素，因此总共有 n/2 个操作。 完全删除策略的质量不是成本的总和，而是所有选定对中的最大成本。 任务是最小化所有可能的有效删除顺序的最大边缘成本。 

输入对可以变得相邻的对上的完整成本结构进行编码。 一个关键的结构保证是在此过程中只有相反奇偶性指数才能相邻。 此限制将成本图减少为类似二分的结构，并且是输入仅列出这些对的核心原因。 

约束 n ≤ 4000 意味着二次或三次动态规划解决方案是可行的，但任何具有重常数或对匹配的指数状态探索的三次动态规划解决方案都是有限的。 对所有删除顺序的简单模拟是阶乘的，并且立即不可能。 

当局部便宜的货币对阻碍了全局必要的结构时，贪婪直觉的一个微妙的失败案例就会出现。 例如，尽早删除最便宜的相邻对可能会以一种强制稍后进行非常昂贵的合并的方式分割数组。 贪婪失败的一个小例子是全局最优答案需要首先跳过可用的低成本对以保留以后配对的灵活性。 

## 方法

 直接方法将尝试所有可能的删除顺序。 在每一步中，我们选择一个相邻的对，将其删除，并对结果序列进行递归。 状态的数量就像邻接约束下元素配对方式的数量一样增长，这本质上是动态邻接更新下路径中完美匹配的数量。 即使忽略成本评估，这也会随着 n 呈指数增长，大致相当于类似加泰罗尼亚结构的数量级，并且很快就会变得不可行，超过 n 大约 30 或 40。 

关键的观察结果是，问题不在于删除的顺序，而在于删除引起的最终配对结构。 每个有效过程都对应于初始索引的完美匹配，但有一个附加约束：每个匹配对在被删除时必须变得相邻。 这将问题转化为选择路径的匹配，其中边缘不固定，但必须在收缩下可实现。 

这种类型的“可行结构上的最大边缘最小化”自然会受到区间动态规划的攻击。 关键的结构事实是，如果我们查看任何段，该段中的第一个元素最终必须与同一段中的某个其他元素配对，并且删除该对会将段拆分为独立的子段。 这建议在间隔上进行 DP，其中我们尝试左边界的所有可能伙伴并确保其余分区的兼容性。 

一旦我们接受这种分解，目标就变成了：我们想要对索引进行配对，以使最大选择的边成本最小化。 这相当于检查所有配对是否可以限制在成本≤X的边上，然后找到这样的最小值X。这将问题转化为对X的单调可行性检查。

对于固定阈值 X，我们只允许成本 ≤ X 的对。我们需要决定是否可以仅使用允许的相邻对操作来完全删除数组。 这成为经典的“我们能否在约束下完全匹配结构”的区间DP。 

完整的解决方案是对 X 进行二分搜索，其中每次检查都会运行一个区间 DP 来验证可行性。

 ## 算法演练

 我们将问题重新表述为决定给定阈值 X 是否允许完全删除，然后使用对成本排列的二分搜索来最小化 X。 

### 1. 预处理成本查找

 我们存储有效奇偶校验兼容对的成本（i，j）。 这允许在 DP 转换期间进行 O(1) 访问。 这很重要的原因是 DP 将重复查询许多候选配对，并且这里的任何对数开销都会使解决方案超出限制。 

### 2. 对答案进行二分搜索

 我们维护一个候选值 X 并询问完全删除中使用的所有对是否可以具有最多 X 的成本。由于 X 中的可行性是单调的，二分搜索是有效的：如果我们可以使用成本 ≤ X 进行删除，那么我们也可以对任何更大的阈值这样做。 

### 3.区间DP定义

 令 dp[l][r] 为一个布尔值，表示仅使用当前阈值 X 下允许的操作是否可以完全删除从 l 到 r 的子数组。我们只考虑偶数长度的段，因为删除一次会删除两个元素。 

此状态捕获了基本结构：一旦边界固定，片段上的任何有效删除序列都必须独立于其余片段完全消除该片段。 

### 4. 基本情况

 长度为 0 的段是可以简单删除的。 当且仅当两个端点可以配对并且它们的成本 ≤ X 时，长度为 2 的段才是可删除的。这锚定了递归。 

### 5. 通过为左端点选择伙伴进行转换

 对于段 [l, r]，假设我们将 l 与某个 k 配对，其中 l < k ≤ r 且 (l, k) 在阈值 X 下允许。如果我们选择这样的配对，剩余元素将分为两个独立的子段：[l+1, k-1] 和 [k+1, r]。 两者都必须完全可删除，选择才有效。 

因此，如果存在有效的 k 使得两个子问题都为真，则 dp[l][r] 为真。 

### 6.以递增的间隔长度填充DP

 我们按照段长度递增的顺序计算 dp，确保子问题在查询之前准备就绪。 这是必要的，因为每个转换都依赖于严格较小的间隔。 

### 7.答案提取

 我们二分查找 dp[1][n] 为 true 的最小 X。 

### 为什么它有效

 每个有效的删除过程都可以解释为重复选择最终变得相邻的一对并将其删除，这对应于将区间递归分解为不相交的子区间。 DP 准确地捕捉了这种分解结构：选择 l 的第一个伙伴定义了将问题划分为独立的子问题。 任何有效的删除序列都不能违反此结构，并且每个 DP 构造都对应于一个有效的删除序列。 可行性检查确保我们仅使用阈值 X 以下的边缘，因此二分搜索在所有可实现的匹配中隔离最小最大边缘成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    
    # initialize cost matrix with large values
    INF = 10**18
    cost = [[INF] * (n + 1) for _ in range(n + 1)]
    
    # read triangular parity-structured input
    for i in range(1, n):
        row = list(map(int, input().split()))
        idx = 0
        if i % 2 == 1:
            j = i + 1
            step = 2
            for v in row:
                cost[i][j] = v
                j += step
        else:
            j = i + 2
            step = 2
            for v in row:
                cost[i][j] = v
                j += step
    
    # collect all costs for binary search domain
    vals = []
    for i in range(1, n + 1):
        for j in range(i + 1, n + 1):
            if cost[i][j] < INF:
                vals.append(cost[i][j])
    
    vals.sort()
    
    # DP feasibility check
    def can(x):
        dp = [[False] * (n + 2) for _ in range(n + 2)]
        
        for i in range(1, n + 1):
            dp[i][i - 1] = True
        
        for length in range(2, n + 1, 2):
            for l in range(1, n - length + 2):
                r = l + length - 1
                if length == 2:
                    if cost[l][r] <= x:
                        dp[l][r] = True
                    continue
                for k in range(l + 1, r + 1, 2):
                    if cost[l][k] <= x and dp[l + 1][k - 1] and dp[k + 1][r]:
                        dp[l][r] = True
                        break
        return dp[1][n]
    
    # binary search over sorted costs
    lo, hi = 0, len(vals) - 1
    ans = vals[-1]
    
    while lo <= hi:
        mid = (lo + hi) // 2
        if can(vals[mid]):
            ans = vals[mid]
            hi = mid - 1
        else:
            lo = mid + 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```输入解析使用基于奇偶校验的索引重建隐藏成本矩阵。 这一点至关重要，因为在原始动态过程中只有同奇偶校验邻接才有效，因此 DP 只会查询这些对。 

可行性函数构建全区间DP表。 选择步长为 2 的迭代长度可确保奇偶校验一致性，因为只有偶数长度的段才能被完全删除。 该转换尝试左端点的所有可能的配对伙伴，这是标准区间匹配分解。 

二分搜索将最终目标从最小化结构上的最大值减少到重复的可行性检查，每个检查都是多项式的。

 我们考虑一个小的 4 元素链，其中存在多个配对路线。

```我们测试 X = 4。 

| 细分 | 选择| 有效子段 | 结果 |
 | --- | --- | --- | --- |
 | [1,4]| 对 (1,2) 无效 | - | 没有|

 ## 边缘情况

 当局部最优配对阻止必要的远程配对时，就会出现一种边缘情况。 例如，即使 (2,3) 非常便宜，尽早使用它也可能会阻止配对 (1,4)，而这在最佳阈值解决方案中可能是必需的。 DP 处理这个问题是因为它从不贪婪地提交，它会为每个左端点尝试所有可能的合作伙伴，从而保留所有全局配置。 

当由于成本限制而仅存在一种配对模式时，会出现另一种边缘情况，从而有效地强制进行唯一的分解。 在这种情况下，dp 退化为单个有效的递归分割序列，并且该算法仅在该强制结构中的最大边缘处正确识别可行性。
