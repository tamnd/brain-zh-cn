 ## 解决方案
 ## 问题理解

 我们被赋予一对值$(a, b)$在以素数为模的有限域上$p$。 每个查询都要求转换起始对所需的最少操作数$(a_i, b_i)$成目标对$(c_i, d_i)$，其中每个操作应用两个确定性变换之一。 

每个变换以线性方式混合两个坐标模$p$。 一个操作将一个坐标加倍并使用减法调整另一个坐标，另一个操作对第二个坐标执行对称操作。 重要的是，操作的顺序很重要，因为这些转换不会交换。 

输入尺寸很大：最多$10^5$查询和$p$可以大到$10^9+7$。 这会立即排除所有每个查询的图遍历$p^2$州。 即使在状态空间上使用单个 BFS 也是不可能的，因为图有大小$p^2$，这远远超出了计算极限。 

因此，关键要求是将状态空间压缩为一维或代数结构，以便可以在对数或常数时间内回答每个查询。 

一个微妙的限制是$a+b \not\equiv 0 \pmod p$对于所有输入。 这个条件被证明是主要的结构提示：它防止退化为一个特殊的不变子空间，在该子空间中动力学变得模糊。 

一种简单的方法会模拟所有可能的转换$(a,b)$并尝试 BFS 到达$(c,d)$。 这立即失败：甚至探索$10^6$每个查询的状态远远超出限制，并且有$10^5$查询。 

第二个天真的想法是观察变换是线性的并尝试对矩阵求幂。 然而，矩阵求幂并没有直接帮助，因为我们没有重复应用相同的运算； 我们在每一步都在两个不同的线性图之间进行选择，从而产生指数数量的可能组合。 

需要注意的关键边缘情况是，尽管这两个操作看起来都是二维的，但它们保留了一个非常强的不变量，可以解决问题。 

例如，如果我们尝试较小的值：$(a,b) = (1,2)$，两个操作都保留$a+b = 3$。 这并非偶然，而是结构简化的关键。 

## 方法

 暴力解决方案将对待每个状态$(a,b)$作为具有两个出边的有向图中的节点。 从每个查询的开始状态运行 BFS 将给出最短路径。 这是正确的，但不可能，因为每个 BFS 都会探索$p^2$状态并且有$10^5$查询。 

关键的观察结果是这两个操作都保留了总和：$$(a+b) \to (2a + (b-a)) = a+b$$和$$(a+b) \to ((a-b) + 2b) = a+b.$$因此每个可达状态必须位于同一条仿射线上$a+b = S$， 在哪里$S$每个查询都是固定的。 

这将 2D 问题折叠为 1D 问题：一次$a$已知，$b$被确定为$b = S - a$。 我们现在只追踪进化$a$。 

每个操作都成为一个函数$a$：

 - 第一次操作：$a \to 2a$- 第二次操作：$a \to 2a - S$现在问题变成：从$a$， 抵达$c$使用这两个仿射函数的最少应用次数。 

这些操作的结构很重要。 两种运算共享相同的线性部分（乘以 2），仅在常量移位方面有所不同。 这意味着任何操作序列都会产生以下形式的转换：$$a \mapsto 2^k a - S \cdot T$$在哪里$T$是 2 的不同幂的总和，由序列中使用第二个操作的位置确定。 

这将问题变成了表示问题：我们必须决定在缩放约束下是否可以使用二进制加权子集和来表示一个值。 

从这个结构来看，解决方案简化为检查是否存在小的$k$这样：$$\frac{2^k a - c}{S}$$是范围内的整数$[0, 2^k - 1]$。 最小这样的$k$就是答案。

 | 代数约简+二分查找$k$|$O(\log p)$每个查询 |$O(1)$| 已接受 |

 ## 算法演练

 我们使用不变性和仿射约简独立地解决每个查询。 

1. 计算不变和$S = a + b \bmod p$。 如果目标对不满足$c + d \equiv S \pmod p$，立即返回$-1$。 

这是因为这两种操作都保留$a+b$，因此不匹配使得转换不可能。 
2. 通过处理将问题简化为单个变量$b$作为依赖者：$b = S - a$。 对目标做同样的事情，所以我们只需要变换$a \to c$。 
3. 定义两个操作$a$：$$f(a) = 2a, \quad g(a) = 2a - S.$$此步骤将 2D 系统转换为单个坐标上的仿射系统。 
4. 观察之后$k$操作，任何序列都会产生：$$a_k = 2^k a - S \cdot T,$$在哪里$T$是 2 的不同幂的子集和$k$位。 
5. 重新排列目标方程：$$S \cdot T = 2^k a - c.$$计算：$$T = (2^k a - c) \cdot S^{-1} \bmod p.$$6. 检查是否$T$是有效的二进制子集和：

 它必须满足$0 \le T < 2^k$。 如果是这样，$k$是可行的。 
7. 迭代增加$k$（自此以来最多约 60$2^k$超过$p$快速）并返回最小的有效值$k$。 

### 为什么它有效

 不变和将系统简化为由具有相同线性部分的两个映射生成的一维仿射半群。 每个操作序列都唯一对应于选择哪些步骤应用翻译$-S$，并且这些翻译以二进制权重累积，因为每个应用程序都会通过后续的加倍进行缩放。 这对系数强制采用二进制结构$T$，使得可达性等于 2 的幂的有界子集和。 由于每个有效的表示都是唯一的$k$，检查存在性减少为验证简单的算术条件而不是搜索状态空间。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    p, q = map(int, input().split())
    inv = lambda x: pow(x, p - 2, p)

    for _ in range(q):
        a, b, c, d = map(int, input().split())

        S = (a + b) % p
        if (c + d) % p != S:
            print(-1)
            continue

        if S == 0:
            # impossible under constraints (given input guarantees S != 0)
            print(0 if (a, b) == (c, d) else -1)
            continue

        invS = pow(S, p - 2, p)

        pow2 = 1
        ans = -1

        for k in range(0, 70):
            if k > 0:
                pow2 = (pow2 * 2) % p

            val = (pow2 * a - c) % p
            t = (val * invS) % p

            if t < pow2:
                # additional consistency check in integer sense
                # lift to canonical integer representative already in [0, p)
                ans = k
                break

        print(ans)

if __name__ == "__main__":
    solve()
```实现首先强制执行不变性$a+b$。 然后，它将每个查询简化为检查仿射模型中的可行性。 循环结束$k$建立两个模的幂$p$，并且对于每个$k$计算所需的平移系数$T$。 关键检查是该系数是否在二进制范围内$[0, 2^k)$，它编码它是否可以从第二个操作的有效序列中产生。 

必须小心模运算：所有中间计算都是模数完成的$p$，但最终的可行性检查依赖于代表的自然整数排序$[0,p)$。

 ## 边缘情况

 一种边缘情况是不变量立即阻止变换。 如果$a+b \neq c+d$，算法返回$-1$无需进一步计算，反映状态空间分裂成不相交的不变线。 

另一个边缘情况是当转换很微不足道时（$k=0$）。 算法正确检查$T=0$在$k=0$，对应于没有应用任何操作。 

当出现第三种边缘情况时$S$模数很小或接近于零$p$。 自从$S \neq 0$保证，模逆始终有效，防止仿射约简中的除法问题。
