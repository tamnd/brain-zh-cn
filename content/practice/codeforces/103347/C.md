 ## 解决方案
 ## 问题理解

 我们有一个圆形花园$n$花坛标记为$1$到$n$，之后$n$我们回到$1$。 朱丽叶从床上开始$1$。 时间以离散的分钟为单位进行，并且在分钟内$i$，她的移动步长正好是$i$，每分钟都在增加。 

更准确地说，如果她处于位置$c$在分钟开始时$i$，她向前走去$i$沿着圆圈顺时针移动并在该位置结束该分钟$c + i \pmod n$。 在短暂停留后，她在下一分钟从那个位置继续。 

问题是，在无限长的时间内，朱丽叶是否会在每个花坛上至少停一次。 换句话说，我们想知道访问过的停止位置的集合是否最终覆盖了所有模余数$n$。 

关键的观察是该过程是确定性的并且纯粹是算术性的。 后$k$分钟，她的位置是第一个的总和$k$自然数模$n$，这是一个三角形数：$$S_k = 1 + 2 + \dots + k = \frac{k(k+1)}{2}$$所以问题就简化为询问序列是否$S_k \bmod n$最终达到每个残基类模$n$。 

约束条件$n \le 10^4$立即排除对许多步骤的任何模拟，并结合天真地检查所有访问过的状态的大范围，因为我们可能需要推理最多$O(n)$或更多不同的残基，并且该过程是无限的。 直接模拟也会产生误导，因为序列呈二次方增长，因此天真的思考没有结构的周期性可能会失败。 

较小时会出现微妙的边缘情况$n$行为与较大的不同。 例如，$n = 2$两个残基之间简单地交替，但是$n = 3$尽管移动量很大，但仍无法覆盖所有残留物。 这表明隐藏的数论结构而不是遍历问题。 

## 方法

 强力解释将模拟朱丽叶的大量步骤的位置，将访问过的残基存储在一个集合中，直到所有$n$被看到或检测到重复模式。 每一步都是$O(1)$，但所需的步骤数不受以下限制$n$，因为序列模$n$取决于三角数并且可以以长周期重复。 在最坏的情况下，我们可能会模拟远远超出$O(n^2)$检测结构之前的步骤，在 1 秒限制下不可靠。 

关键的见解是停止考虑“圆周上的运动”，而是将访问过的位置视为二次多项式模的值$n$。 顺序$S_k = k(k+1)/2$由模运算控制，并且可达残数的集合由该二次多项式是否是满射模来确定$n$。 

然后问题就崩溃为一个经典结构：行为仅取决于是否$n$是二的幂。 什么时候$n$具有奇数素因子，二次留数结构会破坏覆盖范围，并且某些留数变得无法到达。 什么时候$n$是 2 的幂，增量中的加倍结构确保随着时间的推移完全覆盖残基。 

因此，我们不进行模拟，而是考虑$n$并检查它是否包含任何大于的奇数因子$1$。 如果是，答案是否定的。 否则，答案是肯定的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(T)，其中 T 无界，可能非常大 | O(n) | 太慢/不可靠 |
 | 质因数/二次方检查 | O(log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1.读取整数$n$，代表圆形花园的大小。 目标是对是否所有位置都可以作为停止点进行访问进行分类。 
2、不断分割$n$经过$2$当它是均匀的时候。 这在因式分解中隔离了 2 的所有幂。 此步骤相当于提取模数的 2-adic 结构，这是此过程中唯一允许完全覆盖的结构。 
3. 去除所有2的因数后，检查剩余的值是否为$1$。 如果它大于$1$， 然后$n$有一个奇怪的因素。 
4. 如果存在奇数因素，立即断定不可能完全覆盖并输出NO。 否则，如果减少的值是$1$，输出YES。 

这样做的原因是三角数序列模$n$其行为类似于有限环上的二次映射。 在具有奇数因子的模上，该映射会折叠成一个受限子集，因为乘以$1/2$定义明确并引入了阻止均匀覆盖的结构。 超过 2 的幂，缺乏可逆性$2$在残基中创建足够的混合，使序列最终能够到达每个类别。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    # remove all factors of 2
    while n % 2 == 0:
        n //= 2
    
    # if anything other than 1 remains, there is an odd factor
    if n == 1:
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该实现直接反映了因式分解逻辑。 消除两个因数的循环是安全的，因为$n \le 10^4$，所以至多$O(\log n)$发生迭代。 最后的检查将 2 的纯幂与所有其他整数区分开来。 

一个常见的错误是试图直接模拟朱丽叶的动作。 这种方法失败了，因为位置呈二次方增长，并且周期在小模拟窗口内不明显。

 ## 边缘情况

 对于$n = 1$，算法立即输出 YES，因为只有一个花坛，所以条件很容易满足。 

为了$n = 2$，重复减半将其减少为$1$，因此它正确输出 YES，匹配覆盖两个残数的交替三角和。 

为了$n = 3$，没有任何因数 2 可以被删除，因此算法立即输出 NO，正确捕获了模 3 的三角留数不覆盖所有类。 

为了$n = 8$, 重复除法得到$1$，确认二次幂下的完全覆盖。 该算法无需模拟即可正确识别这一点。
