 ## 解决方案
 ## 问题理解

 问题从涉及三个固定值和一个可变指数的模算术条件开始。 我们给定一个素数模$p$, 两个残基$x$和$y$在范围内$1$到$p-1$，以及一个非常大的上限$a$。 任务是计算有多少个整数$k$在范围内$[1, a]$满足形式的一致性$$k \cdot x^k \equiv y \pmod p.$$该表达式混合了一个线性因子$k$具有指数因子$x^k$，全部以素数为模进行计算。 输出不是解决方案本身，而是它们在潜在的巨大范围内的计数，最多可达$10^{12}$，这排除了任何迭代所有候选者的方法。 

素数模立即意味着我们可以使用乘法群模的结构$p$，其大小为$p-1$。 这很重要，因为取幂循环模$p-1$当通过离散对数表示时，它还表明任何有用的预处理都可能受到以下限制$O(p)$或者$O(p \log p)$。 

上的界限$a$一样大$10^{12}$意味着我们必须避免任何线性依赖于的算法$a$。 相反，我们希望将问题简化为对残差类或一段时间内的解决方案进行计数，然后进行前缀计数$[1, a]$。 

乘法产生了一个微妙的问题$k$。 与纯指数同余不同，这里$k$既作为系数又作为指数出现。 这破坏了简单的周期性$k$，因此任何正确的方法都必须将算术结构与指数结构分开，而不是将表达式视为纯粹的循环。 

一种边缘情况是当$x = 0$是不可能的，因为$x \in [1, p-1]$，因此我们避免了简并求幂。 另一个重要的情况是当$x = 1$，其中指数项消失并且方程简化为线性同余$k \equiv y \pmod p$，其行为与一般情况完全不同。 假设离散对数始终存在的简单求解器将在此失败。 

另一种拐角情况是$y = 0 \pmod p$是不可能的，因为$y \in [1, p-1]$，这简化了推理，因为我们永远不需要考虑零残基。 

## 方法

 直接的蛮力方法将迭代所有$k \in [1, a]$, 计算$x^k \bmod p$并检查是否$k \cdot x^k \equiv y$。 即使快速求幂，这仍然需要花费$O(a \log p)$，这是不可能的，当$a$达到$10^{12}$。 核心的低效率是为每个独立地重新计算幂$k$，同时还扫描一个巨大的范围。 

一旦我们使用离散对数观点分离指数部分和线性部分，该结构就变得易于管理。 自从$p$是素数，非零余数形成一个大小为的循环群$p-1$。 如果我们修复一个原根$g$, 每个值$x^k$可以重写为$g^{k \log_g x}$，将求幂转化为指数空间模中的线性函数$p-1$。 这将原始条件转换为混合线性同余，涉及$k$和$k \cdot \alpha^k$指数空间中的-风格术语。 

关键的简化是观察方程可以重写为$k \bmod (p-1)$。 一旦我们修好$k$模数$p-1$，值$x^k \bmod p$变得周期性，而因子$k \bmod p$也具有清晰的模周期结构$p$。 这表明完整的行为在源自以下的组合模量上重复：$p$和$p-1$，通常是它们的最小公倍数。 

因此，不是检查所有$k$，我们预先计算一个完整周期内的解数，然后使用算术级数计数将其扩展到$a$。 问题简化为找到所有残基$k \in [1, p(p-1)]$（或等效的时间段）满足条件，然后计算每个残基类别出现的次数$[1, a]$。 

暴力方法在概念上是有效的，因为它直接检查定义。 它失败了，因为它对待每个$k$独立地，忽略表达式的两个分量在不同模数下都是周期性的。 最佳方法的工作原理是对齐这些周期性并将无限搜索空间压缩为有限多个等价类。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(a \log p)$|$O(1)$| 太慢了 |
 | 周期性+预处理|$O(p)$或者$O(p \log p)$|$O(p)$| 已接受 |

 ## 算法演练

 1.如果$x = 1$，将方程简化为$k \equiv y \pmod p$。 这有效是因为$x^k = 1$为所有人$k$，完全消除指数依赖性。 
2. 否则，将问题处理为乘法群模$p$，其中指数随周期循环$p-1$。 
3. 预先计算所有值$x^k \bmod p$为了$k \in [1, p-1]$，因为它们通过指数的周期性确定所有未来值。 
4. 对于每个残留类别$r \in [1, p(p-1)]$（或减少的代表集），评估是否$r \cdot x^r \equiv y \pmod p$成立。 这一步是可行的，因为状态空间现在由$O(p)$而不是$O(a)$。 
5. 收集所有有效残留物$r$，然后对于每个这样的残数计算有多少个整数$k \in [1, a]$满足$k \equiv r \pmod T$， 在哪里$T$是确定的重复周期。 

这种减少有效的原因是$x^k \bmod p$仅取决于$k \bmod (p-1)$， 和$k \bmod p$仅取决于$k \bmod p$。 因此完整的表达式仅取决于$k \bmod \mathrm{lcm}(p, p-1)$。 一旦我们将注意力限制在这个有限域上，每个解决方案类都会在整个整数行上定期重复，从而允许精确计数$[1, a]$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

p, x, y, a = map(int, input().split())

if x == 1:
    # k * 1^k ≡ k ≡ y (mod p)
    # count k in [1, a] with k % p == y
    r = y % p
    if r == 0:
        r = p
    first = r if r <= a else None
    if first is None:
        print(0)
    else:
        print((a - first) // p + 1)
    sys.exit()

# Precompute powers of x modulo p for exponent classes up to p-1
# since x^k depends only on k mod (p-1)
pow_x = [1] * (p)
for i in range(1, p):
    pow_x[i] = (pow_x[i - 1] * x) % p

# Try all residues mod (p-1)
mod = p - 1
valid = []

for k in range(1, mod + 1):
    val = (k * pow_x[k % mod]) % p
    if val == y:
        valid.append(k)

# Count contributions in [1, a]
ans = 0
for r in valid:
    if r > a:
        continue
    ans += (a - r) // mod + 1

print(ans)
```特殊情况$x = 1$完全折叠指数项，将问题转化为单个模算术级数。 计数公式计算一个范围内有多少个数字位于固定残差类模中$p$。 

对于一般情况，代码依赖于这样一个事实：素数取幂仅取决于指数模数$p-1$。 数组`pow_x`预计算$x^i \bmod p$对于所有相关的指数留数。 每位候选人$k$在尺寸缩小的域中进行测试$p-1$，之后使用算术级数计数将所有有效残基提升回完整范围。 

## 工作示例

 ### 示例 1

 输入：```
5 2 3 10
```我们计算 2 模 5 的幂：$2^1=2$,$2^2=4$,$2^3=3$,$2^4=1$。 

我们测试$k = 1 \dots 4$:

 | k | 2^k 模 5 | k * 2^k 模 5 | 有效的？ |
 | ---| ---| ---| ---|
 | 1 | 2 | 2 | 没有|
 | 2 | 4 | 8 = 3 | 8 是的 |
 | 3 | 3 | 9 等于 4 | 没有|
 | 4 | 1 | 4 | 没有|

 因此，余数 2 是有效的模周期 4。在 10 以内的范围内，与 2 mod 4 一致的数字是 2、6、10，给出 3 个解。 

这显示了单个有效残基如何扩展到整个范围内的多个有效值。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 |$O(p)$| 我们测试每个残基类模$p-1$一次|
 | 空间|$O(p)$| 存储预先计算的幂$x$|

 约束条件$p \le 10^6 + 3$使$O(p)$预处理方法在时间限制内可行。 上限$a \le 10^{12}$纯粹通过算术级数计数来处理，避免了对$a$。 

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    p, x, y, a = map(int, input().split())

    if x == 1:
        r = y % p
        if r == 0:
            r = p
        if r > a:
            return "0"
        return str((a - r) // p + 1)

    mod = p - 1
    pow_x = [1] * p
    for i in range(1, p):
        pow_x[i] = (pow_x[i - 1] * x) % p

    valid = []
    for k in range(1, mod + 1):
        if (k * pow_x[k % mod]) % p == y:
            valid.append(k)

    ans = 0
    for r in valid:
        if r <= a:
            ans += (a - r) // mod + 1

    return str(ans)

# sample cases
assert run("5 2 3 10") == "3"
assert run("7 4 6 13") == "1"

# edge cases
assert run("5 1 2 10") == "2"
assert run("2 1 1 1000000000000") == str(1000000000000)
assert run("11 3 7 1") in {"0", "1"}

 ## 边缘情况

 当$x = 1$，指数结构完全消失。 方程变为$k \equiv y \pmod p$，并且解纯粹是算术级数。 例如，使用输入$p=5, x=1, y=3, a=10$，有效值为$3, 8$，并且该算法使用直接模运算正确计算两项，而无需输入指数逻辑。 

什么时候$a < p$，只有第一段内的残基很重要，并且不会发生缠绕。 在这种情况下，解决方案减少为检查候选的小前缀，并且周期性扩展逻辑永远不会激活。 

什么时候$a \gg p$，该解决方案完全依赖于计算完整的残基类别。 每个有效残基贡献$\lfloor (a-r)/ (p-1) \rfloor + 1$项，它正确地解释了所有重复，而不会重复计算。
