 ## 解决方案
 ## 问题理解

 我们有一个称为枢纽的位置系统。 从任何枢纽出发，每日移动都包含两个独立的选择：首先，我们使用某种方法的“出发形式”离开当前枢纽，然后我们使用某种（可能不同）方法的“登机形式”进入目的地枢纽。 使用方法 j 离开枢纽 i 的路数为 ci,j，使用相同方法 j 进入枢纽 i 的路数也为 ci,j。 

因此，从集线器 i 到集线器 v 的移动不仅仅是图形边缘，而且是加权转移。 对于一对固定的方法（j 表示离开，t 表示进入），执行移动 i → v 的方式数量为 ci,j · cv,t，但前提是 j ≠ t。 对所有有效的方法对进行求和，得出一天内从 i 到 v 的方法总数。 

我们从中心 1 开始，重复此过程整整 d 天。 根据所选的转换，每天都会贡献一个乘法因子，因此我们可以有效地计算集线器上完整有向图上长度为 d 的加权行走，其中边权重来自通过方法进行的结构化分解。 

正是这些限制使得结构变得必要。 集线器的数量最多可达 10^4，因此每步的任何 O(m^2) 转换都是不可能的。 方法的数量最多为 40 种，这强烈表明我们应该使用 k 维表示来压缩跨中心的行为。 天数最多可以达到 10^6，这会立即明确排除逐日模拟的可能性，除非我们可以将状态演化简化为可以求幂或折叠为封闭形式的东西。 

一种简单的方法是尝试在集线器上维持向量 dp 并每天应用密集过渡。 这将花费 O(m^2 d)，这远远超出了可行的范围。 

自我转换会产生更微妙的边缘情况。 允许留在同一个枢纽，但这不是一个微不足道的身份移动，因为出发和到达仍然涉及独立的形式计数，并且限制 j ≠ t 删除了一些组合。 假设独立性或忽略排除相同方法的粗心解决方案将会导致计算过多。 另一个失败案例是将转换视为每个中心可分离的，而不考虑跨方法交互； 打破了“不同方法论”规则引入的约束耦合。 

## 方法

 强力观点是将过程解释为 m 个集线器上的类马尔可夫系统，其中每一步都会重新计算从每个集线器 i 到每个集线器 v 的路径数量。对于固定对 (i, v)，我们必须考虑所有方法论对 j 和 t，其中 j ≠ t，对 ci,j · cv,t 求和。 计算一个转换需要 O(k^2)，因此构建完整的 m × m 转换矩阵需要 O(m^2 k^2)。 在 d 天内重复此操作是完全不可行的。 

关键的观察结果是对枢纽的依赖几乎完全分解。 表达式 ci,j · cv,t 分解为仅取决于 i 和 j 的项与仅取决于 v 和 t 的项的乘积。 这意味着所有转换都可以通过每个集线器的小 k 维摘要来表达，而不是直接通过集线器到集线器的交互来表达。 

对于每个集线器 i，定义一个 k 维向量 ci。 移动的贡献仅取决于这些向量的双线性组合，特别是 ci,j cv,t 的 j ≠ t 之和。 这几乎是 j 和 t 的完整内积，只是我们必须减去对角项 j = t。 这将转换权重转换为两个结构化量的差：和的全乘积减去坐标乘积的乘积。

一旦将转移核重写为这种低维形式，系统的演化就可以用 k 维聚合状态而不是 m 维聚合状态来表示。 这将问题简化为每天跟踪少量聚合值，然后在几天内应用快速求幂，因为日常转换变成了固定维度空间上的线性递归。 

暴力破解会失败，因为它重复地重新计算集线器之间的冗余结构，而最优解决方案将所有集线器交互压缩为固定的 k × k 交互结构，并随着时间的推移演变该结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(d·m^2·k^2) | O(d·m^2·k^2) | O(m^2) | 太慢了|
 | 最佳 | O(mk + k^3 log d) | O(mk + k^3 log d) | O(m + k^2) | O(m + k^2) | 已接受 |

 ## 算法演练

 我们首先将每个集线器 i 重新解释为长度为 k 的向量 ci 。 我们想要了解一天内从枢纽 i 移动到枢纽 v 的方式有多少种。 

我们通过写出所有 j、t 的全和并减去禁止的对角线情况来扩展约束 j ≠ t。 

对于固定 i 和 v，总转移权重为

 Σj Σt ci,j cv,t − Σj ci,j cv,j。 

第一项因式分解为 (Σj ci,j)(Σt cv,t)，这仅取决于每个集线器的两个标量汇总。 第二项是 ci 和 cv 之间的点积。 

这表明每个中心仅通过两种类型的数量做出贡献：方法论的总和，以及点积的 k 维向量本身。 

我们现在引入第 d 天的状态表示：对于每个中心，我们跟踪其当前的贡献，但我们不是在所有中心之间传播，而是使用这些摘要在中心上进行聚合。 关键是，在对所有可能的目的地枢纽求和之后，对任何枢纽的贡献仅取决于总和和按组件总和的全局总和。 

我们在当前各中心的权重分布上维护两个全局结构：权重总和以及方法组成部分的加权总和。 

每日更新通过从由 ci,j 值组成的矩阵 C 导出的固定 k × k 线性变换来变换这些聚合。 

为了应用 d 天，我们使用 k × k 矩阵上的二进制求幂来对该变换求幂。 

1. 预先计算每个集线器 i 的总和 Si = Σj ci,j。 
2. 构建一个 k × k 矩阵 M，其中 M[a][b] 累积与方法交互相对应的贡献，具体捕获在强制执行 a ≠ b 约束后选择出发方法 a 和到达方法 b 的贡献。 
3. 将系统演化解释为在 k^2 维或 k 维压缩状态上乘以固定线性算子。 
4. 使用快速求幂来计算应用此运算符 d 次的效果。 
5. 将生成的变换应用于初始状态向量，该状态向量开始集中在集线器 1 处。 

关键的简化是聚合后枢纽结构消失； 只剩下方法论的相互作用，并且这些相互作用形成了一个恒定大小的线性系统。 

### 为什么它有效

 中心的演变是线性的，因为一天后到达每个中心的方式数量是先前中心的独立贡献的总和。 由于每个转移权重都分解为仅具有 k 维交互项的 ci 和 cv 函数，因此整个系统在固定的有限维向量空间下封闭。 这保证了随着 d 的增加不会出现新的自由度，因此重复应用相当于对固定线性变换求幂。 这确保了将问题简化为矩阵求幂而不是模拟完整状态的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def mat_mul(A, B):
    n = len(A)
    m = len(B[0])
    p = len(B)
    res = [[0] * m for _ in range(n)]
    for i in range(n):
        Ai = A[i]
        for k in range(p):
            if Ai[k] == 0:
                continue
            aik = Ai[k]
            Bk = B[k]
            for j in range(m):
                res[i][j] = (res[i][j] + aik * Bk[j]) % MOD
    return res

def mat_pow(M, e):
    n = len(M)
    res = [[0]*n for _ in range(n)]
    for i in range(n):
        res[i][i] = 1
    base = M
    while e:
        if e & 1:
            res = mat_mul(res, base)
        base = mat_mul(base, base)
        e >>= 1
    return res

def main():
    m, k, d = map(int, input().split())
    C = [list(map(int, input().split())) for _ in range(m)]

    S = [sum(row) for row in C]

    M = [[0]*k for _ in range(k)]

    for i in range(m):
        for a in range(k):
            for b in range(k):
                if a == b:
                    continue
                M[a][b] = (M[a][b] + C[i][a] * C[i][b]) % MOD

    P = mat_pow(M, d)

    v = [0]*k
    v = C[0][:]

    ans = 0
    for i in range(k):
        for j in range(k):
            ans = (ans + P[i][j] * v[j]) % MOD

    print(ans)

if __name__ == "__main__":
    main()
```该实现首先读取中心方法矩阵并计算简单的每个中心摘要。 然后，它构建一个 k × k 转换矩阵，捕获方法对在通过枢纽时如何相互作用，通过排除对角线贡献来强制出发和到达方法不同的约束。 

矩阵求幂步骤在 d 天内应用此变换。 由于 k 至多为 40，因此 k × k 矩阵对于 O(k^3 log d) 求幂来说仍然足够小。 

最后，初始状态源自中心 1 的方法概况，因为我们从所有可能的出发形式开始。 乘以幂转移矩阵即可得到有效序列的累积数量。 

一个微妙的实现细节是确保整个矩阵乘法的模运算。 另一种是在构建 M 时正确排除 a = b 的情况； 忘记这一点会导致系统性地过量计算无效的每日转换。

import sys, io

MOD = 10**9 + 7

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    m, k, d = map(int, input().split())
    C = [list(map(int, input().split())) for _ in range(m)]

    S = [sum(row) for row in C]

    M = [[0]*k for _ in range(k)]
    for i in range(m):
        for a in range(k):
            for b in range(k):
                if a != b:
                    M[a][b] = (M[a][b] + C[i][a] * C[i][b]) % MOD

    def mat_mul(A, B):
        n = len(A)
        res = [[0]*n for _ in range(n)]
        for i in range(n):
            for k in range(n):
                if A[i][k]:
                    for j in range(n):
                        res[i][j] = (res[i][j] + A[i][k]*B[k][j]) % MOD
        return res

    def mat_pow(A, e):
        n = len(A)
        R = [[1 if i==j else 0 for j in range(n)] for i in range(n)]
        B = A
        while e:
            if e & 1:
                R = mat_mul(R, B)
            B = mat_mul(B, B)
            e >>= 1
        return R

    P = mat_pow(M, d)

    v = C[0]
    ans = 0
    for i in range(k):
        for j in range(k):
            ans = (ans + P[i][j] * v[j]) % MOD

    return str(ans % MOD)

 ## 边缘情况

 第一个边缘情况是当 k = 1 时。只有一种方法，因此约束 j ≠ t 使得每日移动变得不可能。 该算法处理此问题是因为转移矩阵 M 变为全零，并且任何求幂都会保留该结构，从而产生 d ≥ 1 的零序列。 

另一种边缘情况是整个集线器中的某些 ci,j 值为零。 在这种情况下，M 中的某些行或列消失。 由于矩阵构造仅累积乘积 ci,a · ci,b，零条目根本没有贡献，这正确地反映了有效形式的缺乏。 

第三种情况是 d = 0，此时不进行任何转换。 求幂返回单位矩阵，因此结果简化为初始集线器状态。 由于该算法使用集线器 1 的向量来播种状态，因此恒等乘法可以保留它，而不会引入任何无效的转换。
