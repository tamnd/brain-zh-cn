 ## 解决方案
 ## 问题理解

 我们有一系列 GPU，每个 GPU 都以固定负载开始。 我们可以通过添加整数调整来独立修改每个 GPU，但每个调整的幅度都受到一个公共值的限制$k$。 同时，修改后每个GPU仍必须有至少1的负载。 

除此之外，还有几家公司。 每个公司都拥有一个连续的 GPU 段，并对该段的最终总负载施加限制：其间隔内的总和必须位于给定的下限和上限内。 间隔可能重叠，因此每个 GPU 同时影响多个约束。 

任务不是构造任何有效的配置，而是找到尽可能小的配置$k$这样就存在一些允许的每 GPU 调整集，满足所有间隔总和约束和安全条件。 

关键结构是每个约束仅取决于一行上的前缀和，而每个变量都是独立有界的。 这种组合表明了一个链上的线性约束系统，而不是任意的线性方程。 

限制因素$n, m \le 1000$表明一个$O(n^2)$甚至$O(nm)$可行性检查是可以接受的，特别是如果与对数搜索相结合$k$。 尝试直接枚举所有配置的解决方案是不可能的，因为每个 GPU 可以在很大的整数范围内变化，最多可达$10^9$。 

一种幼稚的方法会尝试贪婪地或独立地为每个间隔分配值，但重叠会使局部决策不安全。 稍微好一点的尝试可能会尝试按顺序满足间隔，但由于共享 GPU，早期的选择可能会在以后产生矛盾。 

一些微妙的边缘情况揭示了局部推理失败的原因。 考虑两个严重重叠但总和范围不兼容的区间。 即使每个区间都可以单独满足，共享区域也可能强制进行不可能的分配。 

另一个边缘情况是当$a_i = 1$。 那么安全约束力较低$\Delta_i \ge 0$，消除了减少负载的能力。 任何依赖于向下调整的解决方案都会变得无效，即使区间总和表明它是有帮助的。 

最后一种边缘情况是间隔边界非常严格，例如单个间隔覆盖所有 GPU。 那么问题就简化为检查固定数组的有界扰动是否可以达到特定的总和范围，这是一个全局可行性约束而不是局部调整问题。 

## 方法

 暴力方法会尝试直接分配每个$\Delta_i$之内$[-k, k]$, 强制执行$\Delta_i \ge 1 - a_i$，然后验证所有区间总和。 对于固定的$k$，每个变量都有一个小的整数范围，因此人们可能会想象搜索所有赋值或使用回溯。 

这会立即失败，因为即使对于$n = 1000$，分配的数量是指数级的。 即使基于部分和的修剪也无法有效地处理重叠间隔，因为约束在整个前缀结构中全局相互作用。 

关键的观察是从考虑单个 GPU 值转向考虑前缀和。 如果我们定义$S[i]$作为调整后的负载的前缀和，则每个间隔约束成为两个前缀值之间的简单差异约束。 同时，每个元素的边界也成为连续前缀值之间的约束。 

这将问题转化为以下形式的不平等系统$S[v] \le S[u] + w$和$S[v] \ge S[u] + w$，这正是一个差异约束系统。 可以使用图上的最短路径算法来检查此类系统的可行性。 

自从$k$是单调的，如果给定$k$作品，任何更大的$k$也有效。 这允许二分搜索$k$，并且每个可行性检查都简化为运行图形一致性测试。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(n) | 太慢了|
 | 前缀和+差约束+二分查找| O((n + m) n log K) | O((n + m) n log K) | O(n + m) | 已接受 |

 ## 算法演练

 我们固定一个候选值$k$并测试是否可以满足所有约束条件。 

首先，我们将每个 GPU 约束转换为其调整范围。 自从$|\Delta_i| \le k$和$a_i + \Delta_i \ge 1$，我们得到一个下界$\Delta_i \ge 1 - a_i$和一个上限$\Delta_i \le k$，因此有效范围是这些约束的交集。 

其次，我们定义前缀和$S[i]$作为截至索引的调整值之和$i$。 每个位置的调整约束$i$成为之间的约束$S[i]$和$S[i-1]$， 自从$\Delta_i = S[i] - S[i-1] - a_i$直至不断变化。 这会产生直接的不等式限制连续前缀值之间的差异。 

三、各公司区间$[L, R]$具有所需的总和界限$[A, B]$被转换为对前缀和的约束。 由于区间内的总和等于$S[R] - S[L-1]$，我们将上限和下限重写为两个前缀节点之间的线性不等式。 

第四，我们构建一个有向图，其中每个不等式$S[v] \le S[u] + w$变成有向边$u \to v$有重量$w$。 

第五，我们引入一个超级源，连接到具有零权重边的所有前缀节点。 这使得所有变量都可以访问，并确保我们可以检测系统中任何地方的不一致之处。 

第六，我们使用贝尔曼-福特式松弛进行最短路径可行性检查。 如果之后还有距离可以改善的话$n+1$迭代时，存在负循环，这意味着约束系统对此不一致$k$。 

第七，我们二分查找一遍$k$。 每次可行性检查要么确认所有约束都可以满足，要么证明不可能性，并且单调性保证了搜索的正确性。 

正确性取决于前缀值仍然受到编码为边不等式的所有活动约束的约束的不变性。 如果存在总权重为负的环，则表示所要求的不等式存在矛盾，这意味着前缀和的分配不能同时满足所有条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**30

def feasible(n, a, segs, k):
    # nodes: 0..n, S[0]=0..S[n]
    N = n + 1

    edges = []

    # S[i] - S[i-1] in [Li, Ui]
    for i in range(1, n + 1):
        lo = max(1 - a[i-1], -k)
        hi = k

        # S[i] <= S[i-1] + hi
        edges.append((i-1, i, hi))
        # S[i-1] <= S[i] - lo  => S[i-1] <= S[i] + (-lo)
        edges.append((i, i-1, -lo))

    # segment constraints
    pref = [0]
    for x in a:
        pref.append(pref[-1] + x)

    for l, r, A, B in segs:
        base = pref[r] - pref[l-1]

        lo = A - base
        hi = B - base

        # S[r] - S[l-1] <= hi
        edges.append((l-1, r, hi))
        # S[l-1] - S[r] <= -lo
        edges.append((r, l-1, -lo))

    # add super source 0 connected to all with 0
    # we use Bellman-Ford from 0
    dist = [0] * N

    for i in range(N):
        edges.append((0, i, 0))

    # Bellman-Ford
    for _ in range(N):
        changed = False
        for u, v, w in edges:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                changed = True
        if not changed:
            return True

    # one more pass => negative cycle
    for u, v, w in edges:
        if dist[v] > dist[u] + w:
            return False

    return True

def main():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))

    segs = []
    for _ in range(m):
        l, r, A, B = map(int, input().split())
        segs.append((l, r, A, B))

    if any(a[i] < 1 for i in range(n)):
        return print(-1)

    lo, hi = 0, 10**9

    # quick feasibility check at large k
    if not feasible(n, a, segs, hi):
        print(-1)
        return

    ans = hi
    while lo <= hi:
        mid = (lo + hi) // 2
        if feasible(n, a, segs, mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    main()
```该实现为每个候选者构建一个约束图$k$。 最微妙的部分是将区间约束转换为前缀不等式，其中上限和下限都必须小心地转换为有向边。 一个常见的错误是忘记了根据前缀差异重写时下界会翻转方向。 

可行性检查采用贝尔曼-福特式松弛。 超级源确保所有前缀变量都包含在松弛过程中，即使它们没有通过约束直接连接。 检测后进一步改进$n$迭代对应于检测约束图中的不一致循环。 

## 工作示例

 考虑一个具有三个 GPU 的小型案例：$$a = [2, 3, 2]$$一家公司要求所有 GPU 的总数在 8 到 10 之间。 

我们测试$k = 1$。 每个 GPU 的调整是有限的，因此每个值只能轻微移动。 前缀约束变得足够严格，以至于总和无法达到所需的范围。 贝尔曼-福特松弛未能稳定，表明不可行。 

| 步骤| 类型 | 添加约束 | 效果|
 | --- | --- | --- | --- |
 | 1 | 基地| 前缀构造 | 建立S数组|
 | 2 | 间隔 | [6, 8] 中的 S[3] - S[0] | 严格的全局约束|
 | 3 | BF检查| 放松失败| 发现矛盾|

 这表明即使是单个全局间隔也可以以局部边界无法解析的方式限制可行性。 

现在考虑：$$a = [5, 5, 5], \quad intervals: [1,3] \text{ must be } [14, 18]$$为了$k = 1$，总调节范围太小，达不到15-18个范围的变化。 增加$k$允许调整在所有指数中累积，并且一旦$k = 2$，由于前缀灵活性足够，系统变得可行。 

| 步骤| 类型 | 约束| 结果 |
 | --- | --- | --- | --- |
 | 1 | 初始化 | S[0]=0 | S[0]=0 开始状态|
 | 2 | 边缘构建 | 每个索引的边界 | 灵活性有限|
 | 3 | 检查 k=2 | 充分放松成功| 存在可行的分配 |

 这些痕迹表明，可行性取决于局部调整限制与全局区间总和之间的相互作用，而不是孤立的。

 ## 边缘情况

 当每个$a_i = 1$。 在这种情况下，约束$\Delta_i \ge 1 - a_i$强制所有调整为非负值。 任何假设对称的解$[-k, k]$自由将错误地允许减少，并且可能错误地满足可行性模型中的区间约束。 该算法正确地防止了这种情况，因为下界变为零，消除了约束图中的所有负边。 

当单个区间覆盖整个数组时，会出现另一种边缘情况。 约束变成全局前缀差异上的单个不等式$S[n] - S[0]$。 该算法将其作为图中的单边干净地处理，并且可行性减少为检查总调整能力是否可以将总和转移到所需的范围内。 

最后的边缘情况是约束不一致，即使在很大的情况下$k$。 在这种情况下，二分搜索通过初步检查尽早检测到不可行性，因为即使无约束的调整也无法满足矛盾的区间界限。
