 ## 解决方案
 ## 问题理解

 我们得到了几个由小写字母组成的短字符串，对于每个字符串，我们希望在同一组中找到在特定距离度量下“最接近”的另一个字符串。 该距离不是通常的替换编辑距离，而只是插入和删除距离。 因此，将一个字符串转换为另一个字符串的成本恰好是必须从一个字符串中删除多少个字符并将其插入到另一个字符串中，并且不允许进行替换。 

这个距离有一个更简单的解释：如果我们让`lcs(a, b)`是最长公共子序列的长度，那么之间的距离`a`和`b`是`|a| + |b| - 2 * lcs(a, b)`。 这个恒等式至关重要，因为它将问题从字符串编辑转变为最长公共子序列最大化问题。 

对于每个字符串`si`，我们必须选择另一个字符串`sj`最小化这个距离，如果多个候选者达到相同的最小值，我们必须在其中选择字典顺序最小的字符串。 

字符串数量方面的限制很严格，每个测试用例最多 20000 个，但每个字符串都非常短，长度最多为 8。这种组合立即表明，如果使用昂贵的字符串比较逻辑来完成所有对的暴力破解太大，但任何利用微小字母长度的方法都是可行的。 

一个简单的解决方案是比较每一对并计算每对的 LCS。 尽管每个 LCS 由于长度为 8 而很便宜，但最坏的情况仍然是每个测试用例大约 20000 个平方对，这太大了。 

一个更微妙的问题是打破平局。 如果多个字符串具有相同的距离，我们不能只保留第一个找到的最小值。 我们必须显式地比较字典顺序，如果我们只存储距离，很容易被错误处理。 

边缘情况包括：

 一种情况是许多相同的前缀但不同的结尾，例如`["aaaaaa", "aaaaba", "aaabaa"]`。 幼稚的方法可能会错误地假设相同的前缀占主导地位并选择任意匹配，但 LCS 差异在后缀位置中仍然很重要。 

另一种情况是多个字符串具有相同的最小距离，例如`["abc", "abd", "aac"]`为了`"abc"`两者都在哪里`"abd"`和`"aac"`可能会在基于 LCS 的距离下平局。 正确的答案必须选择字典顺序最小的字符串索引，而不是单独的最小距离。 

最后，像长度 1 这样的非常小的字符串需要小心处理，因为距离根据相等性变为 0 或 2。 

## 方法

 暴力法很简单。 对于每个字符串`si`，我们将它与其他所有字符串进行比较`sj`。 我们计算它们的 LCS，得出距离，并跟踪最小值。 由于每个字符串的长度最多为 8，因此在实践中通过 DP 计算 LCS 是常数时间，每对大约 64 次操作。 然而，对于 20000 个字符串，这会导致在最坏的情况下每个测试用例进行大约 4e8 次比较，这太慢了。 

关键的观察结果是，由于弦非常短，LCS 结构是由小的组合模式决定的。 每个字符串最多有 2^8 = 256 个子序列。 我们可以用每个字符串的所有子序列来表示每个字符串，并使用散列或位掩码签名，而不是直接比较字符串。 

我们可以预先计算每个字符串的所有子序列并映射它们。 两个字符串之间的LCS是最大长度`k`这样它们就有一个共同的长度子序列`k`。 这表明，如果我们处理按子序列签名分组的字符串，我们可以搜索最大化重叠的候选者。 

更直接、更有效的方法是迭代可能的 LCS 长度，从 8 到 0。对于每个字符串，我们生成所有长度递减的子序列，并在预先计算的从子序列到候选字符串的映射中查找它们。 第一个非空匹配给出最佳 LCS，因此给出最小距离。 

由于每个字符串仅产生 256 个子序列，因此总预处理约为 500 万次操作，这是很容易实现的。

 ## 算法演练

 我们通过将每个字符串转换为其所有子序列的紧凑表示来解决该问题，然后使用这些表示来有效地推断最佳匹配。 

1. 对于每个字符串，使用其字符上的位掩码生成所有子序列。 每个子序列都存储在以子序列字符串或压缩哈希为键控的字典中。 除了每个键之外，我们还存储包含它的所有字符串索引。 此步骤构建从子序列到候选字符串的反向索引。 
2. 对于每个字符串`si`，我们尝试通过从 length 开始按长度降序考虑子序列来找到最佳匹配`|si| - 1`降至 0。这种排序的原因是较长的公共子序列会产生较小的距离。 
3. 对于固定的子序列长度`k`，我们枚举所有的子序列`si`长度`k`。 对于每个子序列，我们使用预先计算的映射查找哪些字符串包含它。 
4. 如果我们找到任何候选人`sj`那不是`i`，我们使用以下方法计算距离`|si| + |sj| - 2k`。 我们保持迄今为止找到的最佳距离，如果我们发现更小的值或具有字典顺序更小的字符串的相等值，则更新答案。 
5. 一旦我们在给定的位置找到任何有效的候选人，我们就会提前停止`k`，因为不更小`k`可以改进答案。 

为什么它有效

 正确性来自于 LCS 中距离是单调的这一事实。 较大的 LCS 总是产生较小或相等的距离。 通过从大到小搜索子序列长度，我们第一次找到公共子序列保证了集合中某个字符串的最大可能的LCS。 因为枚举了每个可能的子序列，所以我们不会错过任何实现该 LCS 的候选序列。 在候选人评估过程中明确处理平局打破，确保在相等距离中按字典顺序最小选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def gen_subseqs(s):
    n = len(s)
    res = []
    for mask in range(1 << n):
        t = []
        for i in range(n):
            if mask & (1 << i):
                t.append(s[i])
        res.append("".join(t))
    return res

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        s = [input().strip() for _ in range(n)]

        # map subsequence -> list of indices
        mp = {}

        subs = []
        for i, st in enumerate(s):
            all_subs = gen_subseqs(st)
            subs.append(all_subs)
            for sub in all_subs:
                if sub not in mp:
                    mp[sub] = []
                mp[sub].append(i)

        ans_idx = [-1] * n
        ans_dist = [10**9] * n

        for i in range(n):
            best_d = 10**9
            best_j = -1

            # try subsequences by decreasing length
            # group by length
            by_len = {}
            for sub in subs[i]:
                by_len.setdefault(len(sub), []).append(sub)

            for k in range(len(s[i]) - 1, -1, -1):
                if k not in by_len:
                    continue
                found = False
                for sub in by_len[k]:
                    for j in mp.get(sub, []):
                        if j == i:
                            continue
                        d = len(s[i]) + len(s[j]) - 2 * k
                        if d < best_d or (d == best_d and s[j] < s[best_j] if best_j != -1 else True):
                            best_d = d
                            best_j = j
                if best_j != -1:
                    break

            ans_idx[i] = best_j + 1
            ans_dist[i] = best_d

        print(*ans_idx)
        print(*ans_dist)

if __name__ == "__main__":
    solve()
```该解决方案首先枚举每个字符串的所有子序列。 这是安全的，因为长度最多为 8，因此每个字符串 256 个子序列是恒定规模的工作。 这些子序列存储在反向索引中，允许快速查找哪些字符串包含给定的子序列。 

对于每个字符串，我们按长度重新组织其子序列，以便我们可以首先尝试更长的匹配。 在搜索过程中，一旦找到给定子序列长度的任何有效匹配，我们就会停止，因为没有更短的公共子序列可以改善距离。 

在候选更新期间，通过在距离相等时比较字符串值来处理字典顺序的平局。

 ## 边缘情况

 一种重要的边缘情况是许多字符串共享相同的子序列但最终字符不同。 例如，`"aaaaaa"`,`"aaaaab"`， 和`"aaaaba"`。 该算法生成相同的高长度子序列，但平局决断确保`"aaaaba"`被选择超过`"aaaaab"`如果它产生相等的距离并且按字典顺序更小。 

当字符串长度为 1 时，会出现另一种边缘情况。这里，子序列仅是`""`和字符本身，因此距离为 0 或 2。当不存在公共字符时，算法会正确回退到长度 0 LCS。 

最后的边缘情况是多个字符串在不同的子序列长度上都是同样的最佳候选者。 该算法从不假设第一个找到的匹配的唯一性，并在相同的 LCS 级别内继续进行比较，即使在严重平局的情况下也能确保正确性。
