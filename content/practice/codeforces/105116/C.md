 ## 解决方案
 ## 问题理解

 我们正在模拟一条蛇在固定的 n × m 网格上移动，该网格始终遵循确定性的“蛇状”路径：它从左到右扫描每一行，当它到达一行的末尾时，它会跳转到下一行的开头，从右下角回到左上角。 

蛇从左上角单元格中的单个片段开始。 时间以离散的秒数进行，蛇每秒都会查看该固定遍历路径上的下一个单元格。 如果该单元格为空，蛇就会向前移动一步，将所有身体部位向前移动。 如果该细胞包含一个苹果，蛇就会生长：头部移动到苹果细胞中，并且身体不会以通常的方式移动，从而有效地将长度增加一倍。 

有q个苹果相继出现。 第 i 个苹果只有在第 (i−1) 个苹果被吃掉后才会出现。 任何时刻都会有一个“活跃”的苹果，但有一点不同：如果一个苹果出现在已经被蛇占据的单元格中，那么它会立即被吃掉，而不会发生任何移动或时间流逝，并且会立即生成下一个苹果。 这可能会在任何实际移动发生之前导致瞬时消耗链。 

任务是计算所有苹果被吃完之前的总秒数，包括移动秒数和由蛇内产卵事件引起的任何直接连锁反应。 

约束很大，所有维度和 q 都达到 10^5，因此任何逐个单元模拟运动的解决方案都会立即变得太慢。 随着时间的推移，完整的模拟可能会重复遍历多达 n·m 个单元，这是不可能的。 

关键的微妙之处在于，许多苹果在生成时立即被消耗掉，而无需提前时间，并且在实际移动之间，蛇的行为沿着网格的固定线性遍历是完全确定的。 这表明我们永远不应该模拟一步一步的运动，而应该根据沿着这个线性顺序的位置进行推理。 

在像一条长蛇覆盖大部分网格的情况下，天真的实现会失败，其中许多生成的苹果落入其体内并立即被吃掉。 另一种失败情况是重复坐标导致一秒钟内多次即时消耗，这将破坏任何假设每秒处理一个苹果的实现。 

## 方法

 直接的暴力方法是一秒一秒地模拟蛇。 在每一步中，我们都会计算类似哈密顿量遍历的下一个单元格，检查它是否包含当前的苹果，相应地移动或生长，并处理生成。 这正确地模拟了所有规则，包括即时链，但每秒需要 O(1) 工作，在最坏的情况下，蛇可能需要 O(n·m + q) 次移动，仅移动就需要最多 10^5 步，并且每次苹果交互都可能触发级联检查。 虽然这看起来是线性的，但隐藏的问题是，正确处理蛇体状态需要维护和更新长达 10^5 长度的动态结构，并且重复访问/更新在实践中可能会降低性能，尤其是在严格常数的情况下。 

更重要的是，蛮力忽略了结构的简化：蛇的运动不是任意的，它是所有网格单元上的固定循环顺序。 这意味着我们可以将网格展平为大小为 N = n·m 的一维数组，其中每个单元格在遍历顺序中都有一个索引。 蛇总是沿着这个索引模 N 向前移动。 

一旦我们压平网格，蛇就变成了圆形阵列上的一段。 它的头部位置是一个前进的指针，增长事件隐式地扩展了它的尾部边界。 每个苹果事件都会成为其位置与当前蛇间隔之间的比较。

关键的见解是，只有当蛇实际移动到未被占用的单元格时，时间才会前进。 所有即时进食均在当前时间发生，无需消耗秒数。 所以我们只需要模拟有意义的事件之间的转换，而不是每一步。 

我们维护当前的头索引和表示蛇在循环上占据的范围的结构。 对于每个苹果，如果它位于当前蛇体内，我们首先在其生成位置处理所有即时消耗，重复触发下一个苹果。 只有当苹果位于当前占据区域之外时，我们才会计算头部必须移动多少步才能到达它，这直接使用线性化网格上的模运算来增加时间。 

这将问题简化为具有动态间隔扩展的循环数组上的一系列跳转，根据表示形式，可以在 O(q) 或 O(q log n) 中处理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n·m + q + 蛇维护开销) | O(n·m) | 太慢了 |
 | 扁平化循环+区间推理 | O(q) | O(n·m) 或 O(q) | 已接受 |

 ## 算法演练

 我们首先将每个单元 (i, j) 映射到线性索引 pos(i, j) = (i − 1)·m + (j − 1)。 蛇通过以 N = n·m 为模递增该索引来向前移动。 

我们维护三个主要值：当前头部位置、当前尾部位置和当前时间。 

我们还维护一个结构，用于快速检查某个位置是否位于循环上当前蛇体区间内。 因为蛇从不分支，只生长或向前移动，所以它所占据的细胞总是在循环顺序上形成连续的区间。 

我们一颗一颗地加工苹果，小心翼翼地处理即时连锁店。 

1. 将下一个苹果坐标转换为其线性索引 p。 
2. 当 p 在当前蛇区间内时，苹果立即被吃掉。 我们立即前进到下一个苹果，而不增加时间。 每个此类事件也会将蛇延长一段，从而相应地更新间隔。 此步骤可以连续重复多次，因为每个新苹果也可能落在更新的蛇内。 
3. 一旦我们找到不在蛇内部的苹果位置 p，我们就计算蛇头必须向前走多少步才能沿着循环到达 p。 这是从 head 到 p 的模块化距离。 
4. 我们将该距离添加到总时间中，并将头部向前移动该距离。 
5. 到达p后，蛇将其吃掉并生长，这意味着头部一侧的间隔扩大了1。 
6.我们继续下一个苹果。 

重要的结构特性是，蛇所占据的单元始终按循环顺序形成单个连续段，因此即使在环绕状态下，成员资格查询也可以减少为间隔检查。 

### 为什么它有效

 在任何时刻，蛇在循环遍历顺序上都占据一个连续的区间，因为它总是向前移动而不分裂。 生长仅延伸头部，而运动仅使两端均等地向前移动。 因此，任何新头寸要么严格超出此间隔，要么触发立即消耗而无需时间进展。 这保证了每个苹果都按顺序处理一次，并且所有即时链在任何移动之前都得到解决，从而保留了正确的时间计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, q = map(int, input().split())
    N = n * m

    apples = []
    for _ in range(q):
        x, y = map(int, input().split())
        apples.append((x - 1) * m + (y - 1))

    # snake interval on circle: [l, r] in mod N, initially at 0
    l = r = 0
    head = 0
    time = 0

    def inside(x):
        if l <= r:
            return l <= x <= r
        else:
            return x >= l or x <= r

    i = 0
    while i < q:
        p = apples[i]

        # process instant chain reactions
        if inside(p):
            # consume instantly, extend head side
            r = p
            i += 1
            continue

        # compute forward distance from head to p
        if p >= head:
            dist = p - head
        else:
            dist = N - (head - p)

        time += dist
        head = p

        # grow snake
        r = p

        i += 1

    print(time)

if __name__ == "__main__":
    solve()
```该代码首先将网格展平为单个循环数组，这消除了所有几何运动的复杂性。 这`inside`函数实现了循环区间中的隶属度，这是让我们无需模拟即可检测即时苹果消耗的关键不变量。 

主循环按顺序处理苹果。 如果一个苹果已经在蛇区间内，它会立即被消耗掉，并且区间会扩大而不会增加时间。 否则，我们计算从当前头部位置到苹果的前进距离，并将时间提前该量。 这直接模拟沿着固定遍历路径的运动，而不需要逐步迭代。 

一个微妙的点是，头指针总是直接移动到下一个消耗的苹果位置，因为一旦我们将移动压缩为距离，中间单元对于时间计算就不再重要。 间隔更新确保蛇的身体在生长事件下保持一致。

 ## 边缘情况

 一个关键的边缘情况是当多个苹果出现在蛇当前占据的区间内时。 在这种情况下，算法不会提前时间，只会反复扩大间隔。 该实现通过循环 while 来处理这个问题`inside(p)`是的，确保在任何移动发生之前解决所有链式消费。 

另一个边缘情况是从最后一个单元回到第一个单元的环绕移动。 距离计算显式地处理模算术，即使当头索引接近循环结束时也确保正确的前向遍历。 

最后的边缘情况是蛇已经占据了几乎整个网格。 在这里，几乎所有苹果的生成都是即时的，只有少数实际运动发生。 由于该算法从不迭代体细胞，因此即使间隔跨越大部分周期，它也保持线性并且不会降级。
