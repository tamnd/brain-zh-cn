 ## 解决方案
 ## 问题理解

 我们被要求决定是否可以选择两种类型的烘焙食品的混合，以便同时满足两个约束。 Vasилиса 必须为她的 N 位客人每人购买一件商品，因此商品总数固定为 N。同时，她有一张促销优惠券，要求购买的总费用恰好为 K 卢布。 

可用的产品类型只有两种：一种为每件商品 A 卢布，另一种为每件商品 B 卢布，其中 A 与 B 不同。任务是确定是否存在非负整数 x 和 y，其中 x 是 A 价商品的数量，y 是 B 价商品的数量，x + y 等于 N，总成本 Ax + By 等于 K。如果存在这样的对，我们必须输出任何有效的对； 否则，我们输出-1。 

约束允许的值最大为 10^9，这会立即排除尝试所有可能的 x 或 y 值的解决方案。 在最坏的情况下，对 N 个可能性的线性扫描最多需要 10^9 次迭代，这远远超出了 1 秒的限制。 这推动我们寻求代数解决方案而不是组合搜索。 

当人们尝试贪婪的选择时，例如尽可能多地购买便宜的物品或尽可能多地购买昂贵的物品，就会出现一个常见的陷阱。 这些策略失败是因为目标成本 K 是精确固定的，而正确的解决方案可能需要精确的平衡而不是极端的配置。 当使用整数除法而不检查整除性时，会出现另一个微妙的失败情况，导致小数计数被默默地接受。 

## 方法

 强力方法会迭代从 0 到 N 的所有可能的 x 值，将 y 计算为 N 减去 x，并检查 Ax + By 是否等于 K。这是正确的，因为它显式测试了每个有效的计数组合。 然而，这种方法执行 N 次迭代，并且每次迭代都涉及不断的工作。 当 N 达到 10^9 时，这在计算上变得不可行。 

关键的观察是这个问题并不是真正的二维问题。 一旦我们确定了 x，y 的值就立即由 N 确定，这将系统简化为单个变量。 更重要的是，成本方程在 x 上变成线性。 这意味着我们可以使用代数而不是枚举直接求解 x。 

根据 x + y = N，我们将 y = N - x 代入成本方程。 这将问题转化为 x 的单个线性方程。 因为只有一个未知数，所以我们可以在常数时间内隔离它。 剩下的任务是检查计算值是否是整数并且位于有效范围内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(N) | O(1) | O(1) | 太慢了 |
 | 最优代数解 | O(1) | O(1) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将 x 表示为 A 成本项目的数量，将 y 表示为 B 成本项目的数量。

1. 从约束 x + y = N 开始，将 y 表示为 N - x。 这会从系统中删除一个变量，并将所有内容减少为一个未知的 x。 
2. 将 y 代入成本方程 Ax + By = K，得到 Ax + B(N - x) = K。这一步将问题转换为仅涉及 x 的线性方程。 
3. 展开方程得到 Ax + BN - Bx = K，然后对涉及 x 的项进行分组，得到 x(A - B) + BN = K。这将所有对 x 的依赖隔离在一个系数中。 
4. 重新排列以求解 x：x(A - B) = K - BN。 这将 x 表示为两个已知量的比率。 
5. 计算分子 K - BN 和分母 A - B。此时，x 必须等于 (K - BN) / (A - B)。 仅当分子可被分母整除时，该值才有效。 
6. 检查计算出的 x 是否为整数且满足 0 ≤ x ≤ N。如果是，则将 y 计算为 N - x 并输出两个值。 否则，得出结论：不存在有效选择。 

### 为什么它有效

 该变换将两个未知数的两个线性方程组简化为一个变量的单个线性方程。 由于所有约束都是线性的，并且除了非负性之外不存在不等式，因此每个可行解都必须完全满足这个导出表达式。 任何有效的解都必须产生相同的 x 值，因为该方程在实数中具有唯一解，并且完整性加界是唯一剩余的限制。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    K = int(input())
    N = int(input())
    A = int(input())
    B = int(input())

    numerator = K - B * N
    denom = A - B

    if denom == 0:
        print(-1)
        return

    if numerator % denom != 0:
        print(-1)
        return

    x = numerator // denom
    y = N - x

    if x < 0 or y < 0:
        print(-1)
        return

    print(x, y)

if __name__ == "__main__":
    solve()
```代码直接实现了代数求导。 第一步计算 x 表达式的重新排列的分子和分母。 可除性检查确保我们不接受分数解。 边界检查强制两个计数都是总和为 N 的有效非负整数。 

一个微妙的实现细节是 A - B 的符号。由于 A 和 B 可以以任意顺序出现，因此分母可能为负数，但只要在除法之前检查整除性，Python 中的整数除法仍然可以正确工作。 

## 工作示例

 ### 示例 1

 输入：

 K = 11，N = 3，A = 5，B = 1

 我们计算：

 | 步骤| 表达| 价值|
 | --- | --- | --- |
 | 分子| K-B·N | 11 - 3 = 8 | 11 - 3 = 8
 | 分母| 甲 - 乙 | 5 - 1 = 4 |

 | y | N - x | 1 |

 结果有效，因为 x 和 y 均为非负且总和为 N。这演示了解决方案如何平衡昂贵和便宜的物品以匹配确切的目标成本。 

### 示例 2

 输入：

 K = 2、N = 1、A = 3、B = 0

 | 步骤| 表达| 价值|
 | --- | --- | --- |
 | 分子| 2 - 0·1 | 2 |
 | 分母| 3 - 0 | 3 - 0 3 |

 这里 2 不能被 3 整除，因此不存在整数 x。 这显示了该方法如何在不尝试枚举的情况下正确拒绝不可能的分数解。

 ## 边缘情况

 当 A 和 B 非常接近或交换时，就会出现一种重要的边缘情况，例如 A = 1 且 B = 2。在这种情况下，分母 A - B 变为负值，处理不慎可能会导致符号错误。 该算法通过在除法之前严格依赖整数整除性来避免这种情况，因此符号不会影响正确性。 

当 K 小于最小可能成本或大于给定 N 个项目的最大可能成本时，会出现另一种情况。 例如，如果所有物品的成本至少为 5 并且 N 为 3，则任何低于 15 的 K 都是不可能的。 该公式自然会拒绝这些情况，因为计算出的分子生成的 x 值超出了 [0, N] 范围。 

当即使线性方程有实数解也不存在整数解时，就会出现第三种边缘情况。 例如，如果计算出的 x 等于 2.5，则整除性检查失败，并且算法会正确报告不可能性，而无需尝试舍入或近似。
