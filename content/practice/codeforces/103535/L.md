 ## 解决方案
 ## 问题理解

 我们得到一个小写字母的字符串。 对于每个字母和每个子字符串，我们查看该字母在子字符串中出现的次数，计算该值的平方，然后将所有内容相加。 

更准确地说，对于每个子字符串，我们获取从“a”到“z”的每个字符，计算它在该子字符串中出现的次数，对计数进行平方，然后将结果累加到所有子字符串上。 

因此，任务不是关于单独的子字符串，而是关于所有子字符串范围内字符出现的全局贡献。 

输入大小很大，测试用例的总字符串长度高达数百万。 任何显式检查所有子字符串的解决方案都会立即太慢，因为存在 O(n^2) 个子字符串，甚至仅计算每个子字符串内的频率在最坏的情况下也会导致 O(n^3) 行为。 

这迫使我们根据单个字符出现的贡献来重新解释问题，而不是重新计算子串统计数据。 

当字符串均匀时，会出现微妙的边缘情况。 例如，对于`aaaa`，每个子串都有一个很大的平方频率，并且朴素的子串枚举将由于重复重新计算计数而过度计数。 另一个极端情况是交替字符，例如`ababab`，其中重叠很重要，如果结构不仔细，对每个字符的子字符串进行简单计数会导致重复计算。 

主要挑战是将“频率平方子串之和”转换为每个出现或一对出现独立贡献的结构。 

## 方法

 蛮力方法很简单。 枚举每个子字符串，计算其中每个字符的频率，对其求平方，然后添加到答案中。 这是正确的，因为它直接遵循定义。 然而，对于长度为 n 的字符串，大约有 n(n+1)/2 个子字符串，如果简单地完成，每个子字符串都需要 O(n) 计数工作，从而给出 O(n^3) 复杂性。 即使使用前缀频率数组将每个子串的计算量减少到每个字符的 O(1)，我们仍然会得到 O(26·n^2)，当 n 在测试用例中达到 10^5 或更多时，这远远超出了限制。 

关键的见解是停止从子串的角度思考，而是从事件的贡献角度思考。 对于固定字符 c，每个子字符串贡献其包含的 c 数量的平方。 扩大正方形揭示了一种结构，该结构分为单次出现贡献和多次出现之间的成对相互作用。 这将问题转换为计算有多少子字符串在特定位置包含一次或两次出现。 

这种转变之所以有效，是因为子串约束成为索引上的区间包含约束，一旦位置已知，每个事件就可以以 O(1) 的方式组合计数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^3) | O(n^3) | O(1) | O(1) | 太慢了|
 | 最佳 | O(26·n) | O(26·n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个字符，然后对结果求和。 

对于固定字符c，设其在字符串中出现的位置为p1，p2，...，pk。 

1. 将问题转换为每个字符的出现次数。 我们隔离一个字母，因为不同字母的贡献在正方形中不会相互作用。 
2. 对于此字符，将任何子字符串 [l, r] 解释为包含 cnt(c, l, r) 出现次数。 我们需要 cnt^2 的所有子串的总和。 
3. 将cnt^2 展开为cnt + 2·C(cnt, 2)。 这将单个事件和成对事件分开。 
4. 计算单次贡献。 位置 pi 处的每次出现都包含在从 pi 或之前开始并在 pi 或之后结束的所有子字符串中。 此类子串的数量为 pi·(n − pi + 1)，因此每次出现都恰好贡献了该数量。 
5. 计算配对贡献。 对于 i < j 的一对出现 (pi, pj)，如果开头最多为 pi 并且结尾至少为 pj，则两者都包含在子字符串中。 这为开始提供了 pi 个选择，为结束提供了 (n − pj + 1) 个选择。 每个这样的子串对 cnt^2 扩展贡献 2，因此总对贡献为 2 · pi · (n − pj + 1)。 
6. 使用后缀累积有效计算对贡献。 我们不是迭代所有对，而是从右到左扫描位置，同时保持运行总和 (n − pj + 1)。 对于每个 pi，我们添加 2·pi 乘以未来项的后缀和。 
7. 将所有字符的两个部分相加。 

### 为什么它有效

 关键的不变量是每个子字符串都由它包含的出现次数唯一地解释，并且 cnt^2 的代数展开确保每个出现次数和每个有序的出现次数对最终总和仅贡献一次。 有效子串边界的组合计数与索引的包含条件完全匹配，因此不会发生多计数或少计数的情况。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    pos = [[] for _ in range(26)]
    for i, ch in enumerate(s):
        pos[ord(ch) - 97].append(i + 1)

    ans = 0

    for c in range(26):
        p = pos[c]
        k = len(p)
        if k == 0:
            continue

        suf = [0] * (k + 1)

        for i in range(k - 1, -1, -1):
            suf[i] = suf[i + 1] + (n - p[i] + 1)

        for i in range(k):
            pi = p[i]
            ans += pi * (n - pi + 1)
            ans += 2 * pi * suf[i + 1]

    MOD = 998244353
    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该实现首先按字符对索引进行分组，因为每个字母在最终总和中都是独立的。 后缀数组`suf[i]`存储所有后续出现的总贡献，即有多少个子字符串可以在它们之后结束。 这允许在每个位置的 O(1) 中计算对贡献，而不是枚举对。 

关键的微妙之处在于索引：位置是从 1 开始的，因此子字符串计数公式如`pi * (n - pi + 1)`保持干净并避免边界处的差一错误。

为了`b`在位置 2，单个贡献为 2 × (2 − 2 + 1) = 2。不存在对。 

总数为 4。

 | 一个 | 一个 | [1] | 2 | 0 | 2 |
 | 乙| 乙| [2] | 2 | 0 | 2 |

 这证实了子字符串纯粹是通过包含出现次数来计算的。 

### 示例 2：`aaa`位置为 [1, 2, 3]。 

单项贡献：

 1 贡献 1×3 = 3

 2 贡献 2×2 = 4

 3 贡献 3×1 = 3

 结对贡献：

 (1,2): 2 × 1 × (3 − 2 + 1) = 4

 (1,3): 2 × 1 × (3 − 3 + 1) = 2

 (2,3): 2 × 2 × (3 − 3 + 1) = 4

 总数为 20。 

这与直接枚举相匹配，并显示了配对项如何捕获单独计数无法表示的重叠效应。

 ## 边缘情况

 单字符串是最简单的边界。 用于输入`a`，唯一的子串是`[1,1]`，频率平方为 1。该算法正确计算了一次出现，其贡献为 1 × (1 − 1 + 1) = 1，并且不存在对项，因此输出是正确的。 

完全统一的字符串，例如`aaaaa`强调配对计数。 每个子串都有不同的频率，幼稚的实现经常会错误地计算重叠的贡献。 在这里，每对位置通过子串包含多次贡献，并且后缀累积确保通过子串开始和结束的边界约束精确地计算每对位置。 

该算法自然地处理这两种情况，因为两者只是相同位置贡献公式的特化，零对或最大对取决于 k。
