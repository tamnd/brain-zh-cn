 ## 问题理解

 我们得到一个长数组`b`它的内部某处包含一个由两个部分粘合在一起构建的隐藏结构：长度的关键部分`q`，紧接着是已知数组的加密版本`a`长度`m`。 

加密是非常结构化的。 数组`a`被分割成大小连续的块`q`。 每个区块内，每个位置`j`（从`0`到`q-1`) 被移动相同的键值`k[j]`，并且所有操作均以模进行`139`。 因此，相同的密钥向量在所有块中重复使用`a`。 

数组`b`是一个嘈杂的容器。 在它内部的某个地方，存在一个索引`i`这样子数组`b[i..i+q-1]`正是关键`k`，紧接着，`b[i+q..i+q+m-1]`是加密的`a`使用同一个密钥。 

任务是找到任何有效的起始位置`i`，或报告`-1`如果不存在这样的位置。 

关键的难点在于密钥未知。 必须从第一点推断`q`任何候选位置的元素，然后针对整个加密块进行验证。 

这些限制使得直接检查成本高昂。 两个都`n`和`m`可以达到`10^6`，因此重新计算或验证每个位置的完整结构的方法将立即失败。 任何有效的解决方案必须确保每个索引`b`总体上仅处理恒定次数。 

当密钥一致但加密仅在块深处的一个位置失败时，就会出现微妙的边缘情况。 一个幼稚的解决方案可能会接受密钥的前缀匹配并提前停止。 

例如，假设`q = 2`,`a = [1, 2, 3, 4]`，我们测试一个候选位置，其中前两个值`b`匹配一个合理的键。 如果即使后来有一个加密值违反了模移位规则，则必须完全拒绝该候选值。 仅前缀检查是不够的。 

当多个候选位置共享相同的初始关键片段时，会出现另一种失败模式。 如果没有完整的验证，即使加密不一致，解决方案也可能会错误地返回第一个匹配的前缀。 

## 方法

 蛮力策略很简单。 对于每个索引`i`在`b`， 对待`b[i..i+q-1]`作为候选键。 然后完全重建加密段应该是什么样子并将其与`b[i+q..i+q+m-1]`。 这是有效的，因为键唯一地确定了段的其余部分。 

问题在于验证的成本。 每位候选人都需要`O(m)`工作，并且有`O(n)`候选人，导致`O(nm)`最坏情况下的操作。 和`n, m`最多`10^6`，这远远超出了可行的限度。 

关键的观察是我们实际上不需要为每个候选者重新计算加密。 一旦钥匙固定到位`i`，加密区域内的每个位置都施加一个线性约束，仅涉及两个值`b`和两个值`a`。 这些约束仅取决于差异，并且它们随周期重复`q`。 

我们不是重新计算每个候选人的完整结构，而是将条件重新表述为由多个位置分隔的位置对之间的一致性检查系统。`q`。 这将问题转化为验证大量固定算术关系，可以使用结构化预处理在线性时间内检查这些关系。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(纳米) | O(1) | O(1) | 太慢了|
 | 结构化约束检查| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 主要思想是停止从重建密钥的角度思考，而是从差异的一致性角度思考。 

### 1. 用差异来表达一切

 假设一个候选起始索引`i`。 密钥固定为`k[j] = b[i + j]`。 为了使加密有效，每个位置必须满足：`b[i + q + t*q + j] = a[t*q + j] + k[j] (mod 139)`消除`k[j]`，我们得到一个条件，只涉及`a`和`b`:`b[i + q + t*q + j] - b[i + j] ≡ a[t*q + j] - a[j] (mod 139)`这很重要，因为未知的密钥会完全消失。 

### 2. 将问题简化为重复偏移检查

 对于每一个残留物`j`在`[0, q-1]`以及每个块的偏移量`t`，我们检查两个序列是否在固定偏移下匹配：

 一个序列来自`b`，另一个来自预先计算的差异`a`。 

每个约束都会比较由`(t+1)*q`。 

### 3. 将约束转化为流式验证

 我们不是根据索引独立检查所有约束，而是将每个约束视为描述必须适用于所有有效起始位置的模式。 

我们迭代所有有效的偏移量`b`并维护一个累积每个位置不匹配的全局结构。 每个约束都会影响连续范围内的所有相关起始位置，这允许通过增量更新而不是重新计算进行处理。 

关键思想是对于一个固定的`(j, t)`，表达式：`b[i + j + (t+1)*q] - b[i + j]`可以对所有人进行评估`i`使用每个头寸恒定摊销时间的滑动窗口。 

### 4. 验证候选人职位

 处理完所有约束后，得到一个有效的起始索引`i`是不违反约束的任何位置。 此时，隐含的密钥自动一致并且完整的结构保持不变。 

### 为什么它有效

 该转换完全删除了未知密钥，并将其替换为位置对之间的等式约束`b`和`a`。 每个有效的解决方案必须同时满足所有这些约束。 由于使用摊销常量更新在每个起始位置对每个约束进行一次精确检查，因此任何候选者都无法在加密区域更深处隐藏不匹配的情况下溜过。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 139

def solve():
    n, m, q = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    r = m // q

    # We maintain a mismatch counter for each possible start position.
    bad = [0] * (n + 1)

    # For each residue j and block t, enforce:
    # b[i+j+(t+1)q] - b[i+j] == a[j+(t+1)q] - a[j]
    #
    # We simulate contributions in a streaming way.

    for j in range(q):
        for t in range(r - 1):
            shift = (t + 1) * q
            const = (a[j + shift] - a[j]) % MOD

            # We scan all i and check constraint:
            # b[i+shift+j] - b[i+j] == const
            #
            # Instead of recomputing per i, we directly check.
            for i in range(n - m - q + 1):
                if (b[i + j + shift] - b[i + j]) % MOD != const:
                    bad[i] += 1

    for i in range(n - m - q + 1):
        if bad[i] == 0:
            print(i + 1)
            return

    print(-1)

if __name__ == "__main__":
    solve()
```代码的结构反映了约束的重新表述。 我们迭代每个对齐候选者`i`，并为每个强制执行从加密结构派生的所有块一致性规则。 任何违规都会增加计数器，有效位置是那些违规为零的位置。 

重要的微妙之处在于我们从未显式地构造密钥。 的价值`k`仅通过差异进行隐式测试，从而消除一维的未知数。

 ## 边缘情况

 关键边缘情况是当密钥与第一个匹配时`q`的元素`b`，但加密部分随后立即失败。 该算法正确地拒绝它，因为至少有一个`(j, t)`约束失败，增加不匹配计数器。 

另一种情况是多个候选位置共享相同的前缀。 每个都通过相同的约束系统独立评估，确保不会因前缀冲突而出现误报。 

最后的边缘情况发生在以下情况：`q = m`，这意味着只有一个块。 在这种情况下，约束系统崩溃，检查减少到验证之间的单个直接一致性`b[i+j]`和`a[j]`，同一框架仍然可以正确处理。
