 ## 解决方案
 ## 问题理解

 学期被建模为单个连续的时间线。 每个实验室占据一个连续的时间段，并且这些时间段不重叠。 如果第一个实验室持续下去$a_1$天，然后跨越天$1$通过$a_1$。 第二个实验随后立即开始并运行$a_2$天，依此类推，直到$n$-th 实验室完成了这个学期。 

一位名叫弗拉德（Vlad）的学生只在特定的日子参加：每天$k$- 从 day 开始的第天$k$，所以活跃天数是$k, 2k, 3k, \dots$。 每当这些日子中的某一天落在实验室的时间间隔内时，弗拉德就会为该实验室贡献至少一个已解决的问题。 任务是计算有多少个实验室间隔包含至少一个倍数$k$。 

关键的结构是我们没有被要求每天进行模拟。 学期的总天数可以是$10^{18}$，因此任何需要数天迭代的方法都是不可能的。 唯一可行的方法必须使用区间算术。 

当实验室间隔完全位于两个连续的倍数之间时，就会出现微妙的边缘情况。$k$。 例如，如果$k = 10$，从第 11 天运行到第 19 天的实验室不包含任何活动日。 相反，即使是长度为 1 的非常短的 lab，如果它包含多个$k$，例如实验室覆盖第 20 天，当$k = 10$。 

## 方法

 蛮力解释很简单。 我们计算每个实验室的精确天数间隔，然后迭代所有倍数$k$直到学期结束，检查每个倍数是否属于每个区间。 这导致了一个嵌套结构：对于每个实验室，我们可能会扫描多个实验室$k$，或者等效地扫描所有日期并检查会员资格。 由于总天数是所有天数的总和$a_i$，可以达到$10^{18}$，这立即是不可行的。 

关键的观察是翻转视角。 而不是询问一个实验室是否包含多个$k$，我们问有多少个倍数$k$落在时间线的前缀内。 每个实验室对应于全局日轴上的前缀间隔。 的倍数的个数$k$最多一天$x$简直就是$\lfloor x / k \rfloor$。 因此，对于一个跨越$[L, R]$，其中的活跃天数为$\lfloor R/k \rfloor - \lfloor (L-1)/k \rfloor$。 如果该差异大于零，则实验室做出贡献。 

这减少了在评估每个实验室的简单算术条件时维持实验室持续时间的运行前缀和的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 数天或数倍的暴力破解 |$O(\sum a_i)$或者更糟|$O(1)$| 太慢了|
 | 前缀间隔+除法检查|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 维护一个运行变量`cur`代表前一个实验室的最后一天。 最初`cur = 0`。 这为我们提供了一种计算每个实验室间隔的直接方法。 
2. 对于每个实验室$i$，计算其间隔为$[cur + 1, cur + a_i]$。 然后更新`cur += a_i`。 这会将分段结构转换为显式数字间隔而不存储它们。 
3. 对于每个间隔$[L, R]$, 计算有多少个倍数$k$使用整数除法位于其中：`R // k - (L - 1) // k`。 
4. 如果结果为正，则将答案加一。 这直接检查该时间间隔内是否至少存在一个有效的日子。 
5. 处理完所有实验后，输出最终计数。 

关键的想法是$k$均匀分布并且在任何前缀间隔内对它们进行计数可以简化为常数时间算术表达式。 

### 为什么它有效

 每个实验室对应于数轴上的一个连续线段。 有效参与天数的顺序也是固定的等差数列。 计算线段和算术级数之间的交集减少为比较端点处的前缀计数。 表达式$\lfloor R/k \rfloor - \lfloor (L-1)/k \rfloor$精确计算该表格的项数$t \cdot k$位于$[L, R]$，因此条件“Vlad 参与该实验室”相当于该计数非零。 由于每个实验室都独立处理一次，因此总计数是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    cur = 0
    ans = 0
    
    for x in a:
        L = cur + 1
        R = cur + x
        cur = R
        
        if R // k - (L - 1) // k > 0:
            ans += 1
    
    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案仅保留正在运行的端点`cur`，因此它从不显式存储或迭代几天。 每个实验都使用算术在恒定时间内进行处理。 

表达式`(L - 1) // k`对于正确处理边界至关重要。 不减 1，情况`L`本身是一个倍数`k`将会被错误计算，因为楼层划分会错误地移动边界。

 ## 边缘情况

 一个重要的边缘情况是，实验室恰好以以下倍数开始：$k$。 例如，与$k = 5$，必须立即对从第 10 天开始的实验室进行计数。 

对于输入：```
1 5
10
```间隔为$[1, 10]$。 我们计算：$10 // 5 = 2$,$(1 - 1) // 5 = 0$，因此计数为 2，这意味着 5 和 10 的倍数都在实验室内，并且实验室计数正确。 

另一个边缘情况是当实验室恰好以以下倍数结束时$k$但在前一个之后开始。 为了$k = 4$， 间隔$[5, 8]$仍然包含 8 作为有效日期。 计算：$8 // 4 = 2$,$4 // 4 = 1$，给出计数 1，正确标记参与。 

这些案例证实了边界处理$(L-1)$对于正确性至关重要。
