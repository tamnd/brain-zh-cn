 ## 解决方案
 ## 问题理解

 我们得到两个整数序列，每个序列代表两个单独架子上奖杯的顺序。 在每个货架中，所有值都是不同的，但相同的值可能出现在两个货架中。 目标是构建一个包含两个原始序列作为子序列的单个序列，这意味着我们可以删除元素，但不能对剩余的元素进行重新排序。 在所有这些有效的合并序列中，我们想要一个具有最小可能长度的序列。 如果多个序列具有最佳长度，我们会选择第一个元素尽可能小的序列。 

这正是两个序列的最短公共超序列问题，但在结果序列的第一个元素上有一个额外的平局决胜局。 

这些约束允许每个序列最多 200000 个元素，因此在整个状态空间上进行任何二次动态编程都是不可能的。 尝试直接计算最短超序列的简单方法将需要 O(nm) DP 或显式枚举交错，这两者都远远超出了可行的限制。 

关键的结构约束是每个值在每个序列中最多出现一次。 这显着改变了重叠的性质：公共元素可以唯一匹配，并且问题简化为在共享值集上对齐两个排列。 

当两个序列没有共同元素时，就会出现微妙的边缘情况。 在这种情况下，每个有效的超序列必须包含所有元素，答案就是尊重内部顺序同时最小化第一个元素的合并。 当 LCS 为空时，基于 LCS 的简单重建一定不会中断，因为那时就没有锚点了。 

当第一个元素显着不同时，会出现另一种边缘情况。 字典顺序最小的第一个元素要求强制全局偏好，但这不能违反两个序列仍然是子序列的要求。 一个粗心的贪婪合并总是选择最小的可用头而不考虑未来的对齐在这里可能仍然是正确的，但它必须通过最短超序列的结构来证明是合理的。 

## 方法

 暴力方法将尝试通过尝试两个序列的所有交错同时保持顺序来构造超级序列。 即使我们通过仅保留最短候选来进行修剪，可能的交错数量也会以 n 和 m 为指数。 这会失败，因为合并序列中的每个位置都是从第一个序列或第二个序列中获取的二元选择，导致最坏情况下的可能性为 O(2^(n+m))。 

最短公共超序列的标准观察是，一旦我们知道了 LCS，它的长度就固定了。 如果 LCS 的长度为 L，则最佳答案的长度为 n + m − L。因此，真正的任务是有效计算 LCS，然后重建有效的超序列。 

由于每个序列中的元素都是唯一的，因此 LCS 可以转化为最长递增子序列问题。 我们将第一个序列中的每个值映射到其索引，然后将第二个序列转换为第一个序列中出现的值的索引序列。 LCS 与该映射序列的LIS 完全对应。 

一旦我们知道了 LCS 匹配位置，我们就可以通过遍历两个序列并在尊重这些匹配锚点的同时合并它们来重建答案。 在连续的匹配元素之间，两个序列都贡献独立的块，必须在保留内部顺序的同时合并这些独立的块。 由于这些块的任何交错都会保留有效性，因此我们可以贪婪地选择最小化最终序列的第一个元素。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n + m) | 太慢了|
 | 基于LIS的LCS+合并重建| O(n log n) | O(n log n) | O(n + m) | 已接受 |

 ## 算法演练

 1. 为第一个序列构建位置图，存储每个值出现的位置。 这允许恒定时间查找第二序列中的元素是否是公共的。 
2. 将第二个序列转换为第一个序列中的位置数组，但仅保留两个序列中都存在的元素。 除此之外，存储它们的实际值。 
3. 计算位置数组上的 LIS。 我们维持经典的耐心排序结构并存储父指针来重建 LIS。 这为我们提供了实际值的 LCS，而不仅仅是指数。 
4.通过LIS父指针回溯，以正确的顺序恢复LCS的匹配元素。 
5. 现在重建最短公共超序列。 我们在两个序列上维护两个指针 i 和 j，并迭代 LCS 匹配。 
6. 对于每个匹配的值，我们首先合并 a[i:pi] 和 b[j:pj] 中的所有内容，其中 pi 和 pj 是两个序列中当前匹配元素的位置。 在此合并过程中，我们始终采用 a[i] 和 b[j] 中较小的当前头，因为任何选择都会保留有效性，并且较早选择较小的值会最小化最终序列的第一个元素。 
7. 消耗完两个前缀后，我们将匹配的元素追加一次，然后将两个指针都向前推进。 
8. 处理完所有匹配的元素后，我们以同样的贪心方式合并两个序列的剩余后缀。 

工作原理：LCS 将两个序列划分为通过强制匹配分隔的独立块。 在每个块内，两个序列中都没有出现任何值，因此我们可以自由地任意交错，同时保留顺序约束。 任何交织都会产生有效的超序列，并且首先采用较小头的贪婪选择保证了尽可能小的起始元素，而不影响完成剩余结构的可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def lis_with_parent(seq):
    import bisect
    n = len(seq)
    if n == 0:
        return []

    tails = []
    tails_idx = []
    parent = [-1] * n

    for i, x in enumerate(seq):
        pos = bisect.bisect_left(tails, x)
        if pos == len(tails):
            tails.append(x)
            tails_idx.append(i)
        else:
            tails[pos] = x
            tails_idx[pos] = i

        if pos > 0:
            parent[i] = tails_idx[pos - 1]

    # reconstruct LIS
    k = tails_idx[-1]
    lis = []
    while k != -1:
        lis.append(k)
        k = parent[k]
    lis.reverse()
    return lis

def solve():
    n, m = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))

    pos = {v: i for i, v in enumerate(a)}

    b_idx = []
    b_val = []
    for v in b:
        if v in pos:
            b_idx.append(pos[v])
            b_val.append(v)

    lis = lis_with_parent(b_idx)
    lcs_vals = [b_val[i] for i in lis]

    i = j = 0
    ans = []

    for x in lcs_vals:
        while i < n and a[i] != x and (j >= m or a[i] < b[j]):
            ans.append(a[i])
            i += 1
        while j < m and b[j] != x and (i >= n or b[j] < a[i]):
            ans.append(b[j])
            j += 1

        while i < n and a[i] != x and j < m and b[j] != x:
            if a[i] < b[j]:
                ans.append(a[i])
                i += 1
            else:
                ans.append(b[j])
                j += 1

        while i < n and a[i] != x:
            ans.append(a[i])
            i += 1
        while j < m and b[j] != x:
            ans.append(b[j])
            j += 1

        ans.append(x)
        i += 1
        j += 1

    while i < n or j < m:
        if j == m or (i < n and a[i] < b[j]):
            ans.append(a[i])
            i += 1
        else:
            ans.append(b[j])
            j += 1

    print(len(ans))
    print(*ans)

if __name__ == "__main__":
    solve()
```该解决方案首先使用 LIS 约简计算 LCS。 映射步骤确保仅考虑两个序列共有的值，并且 LIS 保证我们保留相对顺序约束。 

重建阶段是结构化合并。 每次我们到达匹配的元素时，我们就处于同步点，两个序列都必须包含该值。 它之前的所有内容在两个序列之间都是独立的，因此我们安全地贪婪地将它们交错。 最终的后缀合并遵循相同的逻辑，但没有进一步的同步约束。 

一个常见的实现陷阱是无法处理一个指针在另一个序列之前到达匹配项的段。 合并循环内的显式检查确保我们永远不会跳过所需的匹配元素。

 ## 边缘情况

 当两个序列没有交集时，LCS 为空，算法会跳过所有同步步骤。 合并循环直接产生所有元素的有效交错，同时保留顺序，并且贪心比较确保第一个元素尽可能最小。 

当两个序列中的所有元素都匹配时，每个元素都成为同步点。 该算法按顺序将每个元素恰好附加一次，产生与两个输入相同的序列，这是最小可能的答案。 

当第一个元素显着不同时，初始贪婪合并可确保最小的元素首先出现，除非它阻碍了可行性。 由于可行性在比赛之间的每个块中独立保留，因此这种贪婪的选择不会使完成超级序列的能力失效。
