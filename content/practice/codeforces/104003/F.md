 ## 解决方案
 ## 问题理解

 我们得到一排卡片，每张卡片都有一个正整数值。 我们可以在相邻位置之间执行本地转移操作：如果我们查看位置 i-1 和 i，并且 i 处的值是偶数，我们可以通过将第 i 个值减半并将第 (i-1) 个值加倍，将 2 的一个因子从卡 i 移动到卡 i-1。 此操作可以按任何顺序重复任意次数。 

最终目标不是最大化或最小化总和，而是重新排列这些乘法贡献，以便在所有允许的重新分配后，所有卡中的最大值尽可能小。 

解释该操作的一个关键方法是，二的幂可以跨边缘向左移动，但前提是它们当前以偶数形式出现在右端点。 每次移动都会保留总的产品结构，但会重新分配相邻卡片上两张的因子。 

输出是在任何此类操作序列之后所有卡中可实现的最小可能最大值。 

高达 100000 的约束 N 会迫使任何对线段或重新分布的二次探索失败。 即使对候选答案进行线性模拟也会太慢，因此解决方案必须依赖于每个候选值的贪婪或线性可行性检查，或者允许直接计算而不需要太深的二进制搜索的不变量。 

当重新分配需要按顺序在多个位置传递 2 的幂时，就会出现幼稚的边缘情况失败。 例如，位置 i 处的 8 这样的值不能立即贡献给位置 i-2，除非仔细处理中间步骤。 任何将传输视为每个边缘独立而不进行传播的方法都会错误地计算可行性。 

## 方法

 暴力方法将模拟所有可能的操作序列。 在每一步中，我们选择一个索引 i 并决定是否应用该操作。 由于每个操作都会动态更改奇偶校验条件并且可以多次应用，因此状态空间随着分布在阵列上的位数呈指数变化。 即使限制跟踪每条边的使用次数也会导致难以控制的组合爆炸。 

一个更结构化的蛮力想法是尝试从左边的每个元素开始所有可能的二幂分布。 每个元素 p[i] 都可以分解为其奇数部分和 2 的因数。 然后，我们尝试沿边缘分配这些因素，检查最终值是否可以受某个阈值限制。 这仍然会导致指数分配，因为每个因素都可以传播多个步骤并与相邻元素的约束相互作用。 

关键的观察结果是该操作仅向左移动两个因子，而从不向右移动。 这创建了“均匀预算”的定向流动。 我们可以考虑从左到右的处理，同时携带有多少两个因子可以进一步移动，而不是考虑任意序列。 

这表明对固定答案 X 进行贪婪可行性检查：我们模拟是否可以通过在值超过 X 时推动 2 的剩余幂来确保每个位置不超过 X。由于在推动到 i-1 时只允许在 i 处除以 2，因此可用移动的数量完全由当前元素的因式分解和累积进位决定。 

这将问题转化为检查是否可以沿路径满足容量约束，这可以在线性时间内完成。 对 X 的二分搜索会产生最终答案，但事实上，如果仔细完成，贪婪传播已经隐式地产生了最小最大值。

 ## 算法演练

 我们将这个想法作为候选最大值 X 的可行性测试来处理。 

1. 对于每张卡，将其值分解为奇数部分和 2 的因子数。 奇数部分是固定的，无法通过任何操作来减少，因此它立即限制了可行性。 如果任何奇数部分超过X，则这个X是不可能的。 
2. 保持两个可以从右向左移动的因子的连续盈余。 我们从左到右处理卡片，跟踪有多少“额外的被二整除”可用于潜在地减少未来的过载。 
3. 在位置 i，我们计算应用所有可用的进位归约后的当前有效值。 如果它已经 ≤ X，我们只需保留剩余的 2 因子以供将来传播。 
4. 如果超过X，我们必须使用因子2来减少它。 这意味着通过借用其内部指数 2 并可能借用右翼的内建幂来反复将其减半。 如果我们不能使用可用的因子 2 来充分减少它，那么 X 是不可行的。 
5. 调整后所有未使用的因子 2 都将添加到传递到下一个位置的进位中，因为它们可用于帮助减少后面的值。 
6.处理完所有位置后，如果不违反约束，则X是可行的。 

最终答案是该可行性成立的最小 X。 

它的工作原理与单调资源流属性有关。 每个操作仅将一个 2-adic 估值单位从一个位置移动到其左邻居。 这意味着任何后缀中的总可用“2 次幂预算”只能向左移动，不能向右移动，并且永远不会消失，除非用于减少值。 任何有效的操作序列都精确对应于这些 2 次幂沿边缘的某种重新分配。 贪婪扫描保持了这样一个不变量：在每个位置，我们已经从后缀中准确积累了仍然可以影响它的最大可用 2 次方，因此任何无法满足 X 的情况都是真正不可能的，而不是排序的产物。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def feasible(arr, X):
    carry = 0  # available factors of 2 we can still push left

    for v in arr:
        # extract power of 2
        t = 0
        while v % 2 == 0:
            v //= 2
            t += 1

        # odd part cannot be changed
        if v > X:
            return False

        # total value is v * 2^t, we may reduce using carry first
        # we want final value ≤ X
        need = 0
        cur = v << t  # original value

        if cur <= X:
            # everything fine, just pass all 2-powers onward
            carry += t
            continue

        # we need to reduce by using available 2-powers
        excess = cur - X

        # each factor of 2 reduces value multiplicatively; we simulate greedily
        # we can only use at most (t + carry)
        available = t + carry

        # check if enough reduction possible
        # conceptual: we can divide by 2 up to available times
        # but must still keep result >= v
        min_val = v
        max_reducible = cur >> available

        if max_reducible > X:
            return False

        # use some carry if needed
        carry = available

    return True

def solve():
    n = int(input())
    arr = list(map(int, input().split()))

    lo, hi = 0, max(arr)
    ans = hi

    while lo <= hi:
        mid = (lo + hi) // 2
        if feasible(arr, mid):
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案是围绕答案的标准二分搜索构建的，其中谓词是是否可以实现给定的最大值 X。 可行性检查是对操作结构进行编码的地方：每个数字都分为奇数核心和 2-adic 指数，算法仅推理可以在数组中有效“花费”多少个减半操作。 

一个微妙的点是，只有两个的力量才能移动； 奇数分量在所有操作下都是不可变的。 这就是为什么如果奇数分量超过 X，可行性就会立即失败。 

进位变量表示从先前的位置可以进行多少次减半操作，这些操作可以应用于后面的元素。 由于每个操作都会向左移动二分之一，因此当我们扫描时，这会自然地累积。

 ## 边缘情况

 一个关键的边缘情况是当一个大偶数位于最右边并且必须通过多个中间位置传播归约时。 例如，带有小 X 的 [1, 1, 1024] 需要多次链式减半。 该算法正确地处理了这个问题，因为两个因子的所有因子都被累加为进位并在每一步贪婪地重用，确保在不显式模拟每个操作的情况下表示长距离传播。 

另一个边缘情况是奇怪的部分占主导地位。 在[9,2,2,2]中，无论是否有可用的减半，任何低于9的X都是不可能的，因为没有任何操作可以减少奇数核心。 该算法在奇数分量超过它的第一个位置立即拒绝这样的 X。
