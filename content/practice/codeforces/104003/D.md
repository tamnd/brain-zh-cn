 ## 解决方案
 ## 问题理解

 威廉不断地为随时间到来的朋友们提供甜点。 关键的限制是，在每时每刻，当新的一组到达后，甜点必须被切成一定数量的相等的切片，以便每个当前在场的人都可以得到整数数量的切片。 

这个过程是动态的。 最初没有切片。 当第一批人到达时，威廉可能会将整个蛋糕切成一些相等的块，以便所有这些人都可以平均分配。 后来，当更多的人到达时，威廉可以拿走所有现有的切片，并将每个切片进一步细分为相同数量的较小的相等块。 此操作将切片总数乘以整数因子。 限制是，在每次这样的细化之后，切片总数必须能被当前总人数整除，以便仍然可以进行平均分配。 

输出不是关于如何将切片分配给个体，而是关于所有组到达后最终切片的最小数量，假设 William 总是选择最佳切割以最小化最终细分计数，同时尊重每个阶段的规则。 

输入是一组大小的序列，在每一步中，我们都会维护必须服务的累积人数。 

约束很小：组的数量最多为 10 个，每个组的大小最多为 10 个。这立即排除了对每步常数时间算术之外的重度优化或渐近的任何需要。 即使对合理国家使用暴力也会被通过。 真正的挑战是认识重复可分性约束的不变结构。 

一个天真的解释可能会尝试通过在每一步探索不同的乘数来模拟切割决策，但如果我们尝试任意的细化选择，分支因子会快速增长。 另一种错误的方法是只考虑当前组的大小而不考虑累积总数，一旦多个组累积，这种情况就会崩溃。 

当人们认为足以使切片数量独立地被每个组大小整除时，就会出现一种微妙的失败情况。 例如，如果组为 2、3 和 4，则分别强制被 2、然后 3、然后 4 整除，忽略实际约束累计为 2、然后 5、然后 9。 

## 方法

 暴力模型将尝试跟踪每组之后的切片数量，并枚举每个步骤的所有可能的整数乘数。 在步骤 i，如果我们当前有 s 个切片和 t 个人，我们必须选择一个整数 k，使得 k·s 可以被 t 整除。 尝试所有此类 k 值会导致爆炸，因为 k 可能很大，并且最多 10 个步骤的选择序列会不必要地增加可能性。 尽管限制很小，但这种方法模糊了结构。 

关键的观察是状态完全由单个整数描述：切片的数量必须始终是当前人数的倍数，并且我们希望最小的此类数字与所有先前的约束一致。 每次人数从 t 增加到 t + a_i 时，当前切片计数必须调整为新总数的倍数。 由于我们只能将当前切片计数乘以整数，因此最佳策略始终是将其增加到新要求所隐含的最小公倍数结构。 

这将问题简化为维持一个运行值，该运行值始终是可被当前累计人数整除的最小数字，但限制是我们只能在每一步中按整数因子向上扩展。 因为每一步都强制被新的前缀和整除，所以可实现的最小最终数只是组大小的所有前缀和的最小公倍数。

这是有效的，因为每次更新都会对当前切片总数强制执行整除性约束，并且一旦一个数字可以被所有前缀和整除，它在进一步的乘法步骤下仍然有效。 该结构塌陷为重复的 LCM 积累。

 | --- | --- | --- | --- |
 | 蛮力 | 指数| O(1) | O(1) | 太慢了|
 | 最优（前缀的 LCM）| O(N log M) | O(N log M) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们按顺序处理组，同时维护两个值，当前人数和代表最小有效切片计数的当前答案。 

1. 从零人和一份开始。 切片计数从 1 开始，因为我们希望在应用任何约束之前有一个中性乘法恒等式。 这允许对第一组进行一致的处理。 
2. 对于每个小组规模，将其添加到总人数中。 这个累计值就是这批人到达后必须均匀服务的人数。 
3. 更新当前切片计数，使其可以被新的总人数整除。 由于我们只能通过整数乘法进行缩放，因此我们计算当前切片计数的最小倍数，该倍数也可以被新总数整除。 这相当于用 lcm(current_slices, current_people) 替换切片计数。 
4. 对每个组继续此过程，始终强制被完整前缀和整除。 每个步骤都保留了所有先前组的可行性，因为先前的约束将更新值除以构造。 
5. 处理完所有组后，输出最终的切片计数。 

关键的不变量是，处理完第 i 个组后，当前切片计数是可被前 i 个组大小之和整除的最小数，并且可以通过与先前状态的连续整数相乘获得。 这确保不存在更小的有效配置，因为任何有效配置都必须能被每个中间前缀和整除，并且该过程总是升级到最小​​的这样的倍数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def lcm(a, b):
    return a // gcd(a, b) * b

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    people = 0
    slices = 1

    for x in a:
        people += x
        slices = lcm(slices, people)

    print(slices)

if __name__ == "__main__":
    solve()
```该实现直接反映了算法。 这`people`变量跟踪与会者的前缀总和，这是每个步骤的主动可分性要求。 这`slices`变量跟踪最小可行的切片数。 

唯一重要的操作是 LCM 更新。 使用`a // gcd(a, b) * b`避免溢出并保持计算稳定。 仔细选择乘法顺序以避免中间增长超出 Python 在典型竞争约束下可以有效处理的范围。

 ## 边缘情况

 一种边缘情况是所有组大小均为 1 时。在这种情况下，累积总数每一步增加 1，并且算法连续采用 1、2、3 等的 LCM。 用于输入`1 1 1 1`，计算按照切片 = 1、然后 2、然后 6、然后 12 进行。每个步骤都是强制的，因为每个新的人数都会引入一个新的除数要求，如果不增加切片计数就无法满足该要求。 

另一种情况是当一个组包含所有人时，例如`10`独自的。 该算法立即将 people 设置为 10 并将切片更新为 lcm(1, 10)，产生 10。由于没有中间约束，因此不需要进一步的乘法，并且结果已经是最小的。 

当组大小互质时，会出现更微妙的情况，例如`2, 3, 5`。 前缀和变为 2、5 和 10，算法正确构建 2 → 10 → 10。一旦吸收所有约束，最终值就会稳定，这表明后面的约束可以通过共享整除结构包含早期的约束。
