 ## 解决方案
 ## 问题理解

 我们得到了平面上的一组点。 我们必须放置一个非常具体的三角形，以便每个点都位于其内部或边界上。 三角形不是任意的，它被限制为等腰并与坐标轴对齐，因为它的两条相等的边沿着坐标轴。 任务是选择这样一个具有最小可能“短边长度”的三角形，同时仍然覆盖所有点。 

从几何角度来看，这减少了我们的自由度：一旦三角形被定位，它的形状基本上就由一个参数固定，即等边的长度。 增加该长度会扩大覆盖区域，减少该长度会缩小覆盖区域。 目标是找到该参数的最小值，以便覆盖所有点。 

输入大小高达 100,000 个点，因此任何尝试模拟每个候选点的三角形放置或以简单嵌套方式检查几何覆盖范围的解决方案都将不起作用。 二次扫描需要大约$10^{10}$最坏情况下的操作，远远超出1秒的限制。 这立即表明答案必须取决于点集的某些聚合属性，而不是成对的相互作用。 

边缘情况大多是几何简并。 如果所有点都一致，则答案应该是最小的。 如果点仅在一个方向上相距极远，则答案将由该分布决定。 一个常见的失败是尝试推理凸包结构或三角形方向选择，这在这里是不必要的，并且会导致过于复杂而不会改善核心边界。 

## 方法

 思考这个问题的一种强力方法是固定候选三角形的大小，然后检查所有点是否都适合该三角形的某个位置。 这将需要尝试许多可能的放置或至少验证不同锚点位置的覆盖范围。 每次验证都会花费$O(n)$，如果我们搜索许多候选尺寸或配置，总复杂度至少会变成二次或更糟。 困难在于三角形受到约束但仍可连续放置，因此简单的模拟不会自然地离散搜索空间。 

关键的观察结果是三角形在结构上是轴对齐的，这意味着它的边界约束有效地减少到 x 和 y 坐标上的独立条件。 我们可以将覆盖条件重新解释为将点限制在由极端坐标值确定的形状内，而不是全局考虑几何形状。 

由于三角形随其边长均匀扩展，因此限制因素始终由最远点位于由三角形方向引起的坐标系中的距离来确定。 将几何形状转换为轴约束后，问题就简化为找到一个值，该值在特定的线性组合中同时主导水平和垂直方向上的扩展。 这将几何优化分解为对变换坐标的极值的简单计算。 

因此，我们不需要尝试所有的放置，而只需要从点集中计算少量的全局统计数据，特别是从坐标导出的某些线性表达式的最小值和最大值。 答案成为这些派生范围的最大值。

 ## 算法演练

 我们首先将几何约束重写为仅依赖于坐标极值的约束。 每个点都对三角形边必须有多大提供约束，以便将其包含在覆盖区域中。 

1. 读取所有点并跟踪最小和最大的 x 坐标，以及最小和最大的 y 坐标。 这些值捕获点集的完整水平和垂直分布。 
2. 计算水平跨度为$x_{\max} - x_{\min}$。 这表示点从左到右相距多远。 
3. 计算垂直跨度为$y_{\max} - y_{\min}$。 这代表了上下价差。 
4. 根据轴对齐等腰三角形的几何形状组合这两个值。 限制因素不是纯粹的水平或垂直，而是最坏情况的方向，这迫使三角形足够大以同时覆盖两个极端。 
5. 返回所需的最大跨度，以确保满足两个方向约束。 

### 为什么它有效

 关键的不变量是三角形的任何有效放置都必须足够大，以同时容纳极端 x 分隔点和极端 y 分隔点。 由于三角形沿其定义轴均匀扩展，因此任何巧妙的移动都无法降低这些极端情况所带来的要求。 每个点集都有一对点，它们在正交方向上独立确定必要的最小尺寸，最佳放置只能对齐这些约束，但不能缩小它们。 因此，该解正确地简化为仅坐标范围的函数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    xmin = ymin = 10**18
    xmax = ymax = -10**18

    for _ in range(n):
        x, y = map(int, input().split())
        xmin = min(xmin, x)
        xmax = max(xmax, x)
        ymin = min(ymin, y)
        ymax = max(ymax, y)

    print(max(xmax - xmin, ymax - ymin))

if __name__ == "__main__":
    solve()
```该解决方案仅在读取输入时跟踪全局极值。 这避免了存储所有点并保证线性时间。 最终答案是直接从这些极值计算出来的，因为任何可行的三角形都必须跨越诱导坐标系中点集的水平和垂直直径。 

一个微妙的点是初始化：极值必须从足够大的正哨兵和负哨兵开始，才能正确处理单点输入。 另一个是所有算术都保持在整数范围内，因为坐标达到$10^9$，并且差异仍然在$10^9$规模。

 ## 边缘情况

 对于单个点，例如`(5, 5)`，极值永远不会改变，并且两个跨度都为零，因此算法正确返回 0。三角形可以退化为点大小的覆盖。 

对于像这样的水平线`(1,1), (5,1), (10,1)`，垂直跨度为零，而水平跨度为 9。算法正确地识别出只有水平跨度重要，产生 9。 

对于像这样的垂直线`(2,1), (2,4), (2,8)`，水平跨度为零，垂直跨度为 7，因此答案变为 7。这证实了该方法对称地对待轴。 

对于完整的方形边界，所有极值均处于活动状态，并且结果由边界框的最大边长驱动。 这证实了没有任何隐藏的几何配置可以将所需的尺寸减小到极值所强制的尺寸以下。
