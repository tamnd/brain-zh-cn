 ## 解决方案
 ## 问题理解

 我们得到一个整数$n$，我们需要将其表示为三个正整数之和$a, b, c$。 额外的限制是这三个数字都不能被 3 整除。任何有效的三元组都是可接受的，因此除了满足约束之外没有优化标准。 

从结构的角度来看，这是一个构造性分解问题。 我们不是在寻找最小值或最大值，而是在模块化约束下寻找一个可行的分区。 这立即表明解决方案应该依赖于模算术而不是搜索。 

输入约束$3 \le n \le 10^9$排除了三元组的暴力枚举。 即使我们尝试修复两个变量并导出第三个变量，搜索空间也是二次的$n$，太大了。 因此，问题必须简化为常数时间构造。 

当出现微妙的边缘情况时$n$很小。 例如，$n = 3$强制唯一可能的分区$1,1,1$，这是有效的，因为没有一个可以被 3 整除。另一个有趣的情况是当$n \equiv 0 \pmod{3}$，因为像平均分成这样的天真的尝试$n/3, n/3, n/3$立即失败，因为每个部分都可以被 3 整除。 

## 方法

 暴力策略将枚举所有对$(a, b)$这样$a \ge 1, b \ge 1$, 计算$c = n - a - b$，并检查所有三个是否都是正数且不能被 3 整除。这在逻辑上是有效的，因为它直接测试问题的定义，但它需要$O(n^2)$在最坏的情况下进行检查，这是不可能的$n$最多$10^9$。 

关键的观察结果是，约束仅取决于 3 的整除性，这完全是周期性模 3。我们可以使用避免残数 0 mod 3 的固定模式来构造它们，而不是搜索有效值。 

不能被3整除的整数恰好是1或2模3全等的整数。所以我们只需要保证每个$a, b, c$落入这两个残基类别，同时仍然求和$n$。 由于我们在选择值方面有完全的自由，因此我们可以固定其中两个并导出第三个，然后在其余部分违反约束时稍微调整。 

标准构造是选择两个小的非 3 的倍数，例如 1 和 2，并相应地调整第三个值。 如果我们设置$a = 1$,$b = 1$， 然后$c = n - 2$。 现在唯一的问题是是否$c$能被 3 整除。如果是，我们稍微改变早期的选择之一，以避免落在 3 的倍数上。因为模 3 的残差类很小并且在加法下是封闭的，所以恒定数量的调整总是足够的。 

这将问题从无界搜索减少到恒定数量的算术情况。

 ## 算法演练

 我们直接使用模块化推理来构造答案。 

1.从修复开始$a = 1$。 这保证了$a$是正数且不能被 3 整除。 
2.修复$b = 1$出于同样的原因。 至此，我们一共使用了2个单位。 
3. 计算$c = n - 2$。 这确保了总和条件$a + b + c = n$完全成立。 
4. 检查是否$c$能被 3 整除。如果不能被 3 整除，那么$(1, 1, c)$已经有效。 
5.如果$c$能被3整除，通过替换调整结构$b = 2$和$a = 1$, 给予$c = n - 3$。 这种转变保留了总和，同时改变了残基类别$c$，确保它不再能被 3 整除为有效$n$。 

这些小调整就足够了，因为移位 1 或 2 以受控方式改变了模 3 的残差类别，而我们只需要避免残差 0。 

### 为什么它有效

 每个整数恰好落入以 3 为模的三个残差类别之一。我们明确避免选择 0 类别中的值$a$和$b$，并且我们确保$c$源自$n$通过减去一个小常数。 由于通过不同的残差减去 2 或 3 个周期，因此这些常量偏移中至少有一个会产生有效的$c$不能被 3 整除。这保证了解决方案的存在，并且我们的构造总是能找到一个解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())

a = 1
b = 1
c = n - 2

if c % 3 == 0:
    a = 1
    b = 2
    c = n - 3

print(a, b, c)
```代码直接实现了建设性的想法。 我们从最简单的分解开始，并且仅在第三个组件落在禁止的残基类别上时进行调整。 

唯一微妙的一点是确保积极性。 为了$n \ge 3$， 两个都$n - 2$和$n - 3$保持至少 1，因此两个分支都是安全的。 条件检查保证我们在不破坏积极性约束的情况下避免唯一有问题的残留情况。

 ## 边缘情况

 对于$n = 3$，算法产生$a = 1, b = 1, c = 1$。 这些都不能被3整除，因此结果立即正确，无需进入调整分支。 

为了$n = 6$，我们计算$c = 4$，这是有效的，因为 4 不能被 3 整除。条件不会触发，所以我们输出$1,1,4$。 所有值都是正值并且满足约束条件。 

调整分支仅在以下情况下触发$n - 2$能被 3 整除。在这种情况下，切换到$n - 3$确保第三个数字移动残基类并避免 0 模 3，同时保留所有有效的正值$n \ge 3$。
