 ## 解决方案
 ## 问题理解

 我们得到了一个一维景观，由沿线的高度表示。 每个位置都有一个高度，我们要确定哪些位置是“最佳视点”，即在自然视线规则下，至少可以从一个方向无障碍地看到它们。 

如果当我们从某个方向扫描时，某个位置严格高于所有先前看到的位置，则该位置被认为从该方向可见。 换句话说，如果该扫描中没有较早的点以相同或更大的高度阻挡该点，则该点将变得可见。 

任务是在考虑数组两端的可见性时计算有多少位置可见。 如果从左侧扫描或右侧扫描可见，则该位置被认为是优秀的。 

输入是单个高度数组。 输出是一个整数，表示从至少一个方向可见的索引数量。 

此类问题的典型约束结构意味着数组可能很大，因此任何解决方案都必须以线性时间或近线性时间运行。 当 n 增长到超过几万时，将每个元素与所有其他元素进行比较的二次解显然会失败，因为它将执行 n 平方比较的数量级。 

当所有元素都相等时，会出现微妙的边缘情况。 例如，如果数组为 [5, 5, 5, 5]，则只有左侧扫描的第一个元素可见，只有右侧扫描的最后一个元素可见，因此答案应该是 2，而不是 n。 将“可见”视为“不严格小于所有先前位置”的天真解释会错误地将所有位置标记为可见。 

另一个边缘情况是严格递减数组，如 [5, 4, 3, 2, 1]。 从左侧看，只能看到第一个，从右侧看，只能看到最后一个。 正确答案还是 2，而不是 n。 这有助于确认可见性取决于前缀最大值，而不是本地比较。 

## 方法

 蛮力的想法是独立模拟每个方向的可见性。 对于左侧扫描，我们迭代数组并保持迄今为止看到的最大高度。 每当我们遇到严格大于此最大值的值时，我们都会将其标记为可见并更新最大值。 我们从右侧重复相同的过程。 

这种方法是正确的，因为它直接对可见性的定义进行建模。 然而，它在结构上也是最优的，因为每次扫描都是 O(n)，而且我们只执行两次扫描。 

真正的强力替代方案是对照每个位置左侧的所有元素（与右侧类似）检查每个位置，验证是否存在任何阻塞元素。 这将需要每个位置执行 O(n) 次工作，从而导致总操作次数为 O(n²)。 当 n 很大时，这很快就变得不可行，因为它在最坏情况的输入中执行数百亿次比较。 

关键的观察是我们永远不需要完整的历史记录，只需要每个方向的运行最大值。 一旦我们意识到可见性仅取决于一个点是否超过该扫描中的所有先前值，问题就会分解为两个前缀最大计算。 

我们从左侧计算前缀最大数组，从右侧计算后缀最大数组。 任何与其位置处的前缀最大值匹配的索引都从左侧可见，后缀也类似。 最终的答案是这两个集合的并集的大小。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（成对检查）| O(n²) | O(1) | O(1) | 太慢了|
 | 前缀/后缀最大值 | O(n) | O(n) | 已接受 |

 ## 算法演练

1. 计算一个前缀最大数组，其中每个位置存储从开始到该索引的最大值。 这捕获了左侧的可见性，因为如果一个点等于其位置上的运行最大值，则该点是可见的。 
2. 计算后缀最大数组，其中每个位置存储从末尾到该索引的最大值。 这可以在相同的逻辑下捕获右侧的可见性。 
3. 迭代所有索引，如果某个位置的值等于该索引处的前缀最大值或等于该索引处的后缀最大值，则将该位置标记为优秀。 这直接对至少一个方向的可见性进行编码。 
4. 统计所有这样的位置并输出结果。 

前缀和后缀最大值足够的原因是可见性完全由定向扫描中的优势决定。 任何从给定一侧到其位置不是最大值的点都必须在该方向之前被更高或相等的元素阻挡，从而使其从该侧不可见。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    a = list(map(int, input().split()))

    pref = [0] * n
    suff = [0] * n

    cur = -10**18
    for i in range(n):
        if a[i] > cur:
            cur = a[i]
        pref[i] = cur

    cur = -10**18
    for i in range(n - 1, -1, -1):
        if a[i] > cur:
            cur = a[i]
        suff[i] = cur

    ans = 0
    for i in range(n):
        if a[i] == pref[i] or a[i] == suff[i]:
            ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```该实现直接反映了算法。 前缀传递从左侧保持运行最大值，后缀传递从右侧执行相同的操作。 一个常见的微妙之处是在更新最大值时使用严格比较； 使用`>`而不是`>=`确保相等的高度不会产生多个可见的峰，从而保持平坦段的正确性。 

最后一个循环检查任一可见性集中的成员资格。 这种联合逻辑很重要：我们不会重复计算双方可见的位置。

 ## 边缘情况

 对于全平等的情况，例如：```
4
2 2 2 2
```前缀最大值变为 [2, 2, 2, 2]，后缀最大值相同。 只有索引 0 和 3 符合在其扫描方向上第一次出现最大值的条件。 该算法精确计算这两个端点，因为每个内部元素都被两个方向上较早的相等值阻塞。 

对于严格递减的数组，例如：```
5
5 4 3 2 1
```前缀最大值为 [5, 5, 5, 5, 5]，因此从左侧开始只有索引 0 符合条件。 后缀最大值为 [5, 4, 3, 2, 1]，因此每个元素仅在成为后缀最大值时才从右侧符合资格。 然而，由于后缀最大值需要严格更新，因此从右侧扫描时只有索引 4 是真正的新最大值，因此最终答案是 2。这证实了可见性与方向峰值相关，而不仅仅是单调后缀的一部分。
