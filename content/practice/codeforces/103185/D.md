 ## 解决方案
 ## 问题理解

 给我们一组糖果盒，每个盒子包含一些糖果，糖果的数量是 2 的幂。 我们不是直接给出实际计数，而是给出指数：第 i 个框包含$2^{A_i}$糖果。 

两个兄弟将把这些盒子分给他们。 每个盒子必须完全属于一个兄弟，因此最终的总和是通过将数组划分为两个不相交的子集并对 2 的幂求和来形成的。 

问题是是否可以将每个盒子分配给两兄弟中的一个，以便两兄弟最终得到的糖果总数本身就是 2 的幂。 

所以如果一个兄弟得到一笔像$16$，这是有效的，或者$32$，但是像这样的总和$20$或者$24$不是。 

约束允许最多$10^5$框，每个框的值最多为$2^{10^5}$。 这立即排除了分配上的任何子集枚举，因为有$2^N$分发盒子的方法。 即使存储所有子集总和也是不可能的。 任何解决方案都必须将问题的结构简化为 N 中的线性或近线性。 

一个关键的微妙之处在于它携带物质。 由于所有值都是 2 的幂，因此将它们相加的行为类似于二进制加法，并且组合相同指数的框会创建进位到更高位。 将其视为 2 次方的独立子集和的简单方法将会失败。 

典型的失败案例来自于忽略进位交互。 例如，如果我们有很多$2^0$盒子，它们的总和可能会变成$2^k$，但前提是它们正确分组。 即使存在有效分区，不跟踪每个位级别计数的贪婪分配也可以轻松产生非二次方和。 

另一种边缘情况是所有盒子都是 2 的相同幂。 即使这样，均匀地分割它们也不能保证双方都成为 2 的幂，除非计数与单个二进制进位链对齐。 

## 方法

 蛮力的想法很简单：尝试将每个框分配给任一兄弟，计算两个总和，并检查两者是否都是 2 的幂。 这是正确的，因为它直接探讨了定义。 然而，作业的数量是$2^N$，这已经变得不可行了$N = 30$，更不用说$10^5$。 即使是中间相遇方法也很困难，因为总和是带有进位传播的大整数。 

关键的观察是整个问题是由二进制表示控制的。 由于每个框在位置贡献一个单独的设置位$A_i$，总的多重集可以被视为比特位置上的频率数组。 重要的不是各个方框，而是每个位出现的次数。 

如果两个最终总和必须是 2 的幂，则每一方最终都必须在其二进制表示中恰好有一个设置位。 这意味着在解析所有进位后，每一侧的位结构必须完全折叠成单个最高活动位。 

重要的见解是，一旦我们确定了 2 的每个幂中有多少项进入一侧，进位传播就是确定性的。 我们不再考虑子集，而是考虑每个指数中有多少元素属于 Bob； 查理得到剩余的部分。 我们需要检查是否存在跨位位置的计数分割，以便两个结果多重集通过二进制携带减少到 2 的一次幂。 

这将问题从指数选择减少到通过仔细的进位处理对位计数进行多项式模拟。

 | 暴力子集 |$O(2^N \cdot N)$|$O(1)$| 太慢了|

 ## 算法演练

 我们将输入压缩到频率数组中，其中`cnt[i]`盒子的数量等于$2^i$。 然后，我们模拟这些权力在分配给一位兄弟时如何结合。 

我们尝试用二进制进位链来解释该过程：在两组之间分配项相当于将每个位计数拆分为两个非负整数，其总和为原始计数。 每个组在进位后独立形成一个二进制数。 

1. 统计每个指数出现的次数，构建数组`cnt`。 
2. 尝试所有可能的选择以获得第一个兄弟的和的最终指数。 由于总和是 2 的幂，因此我们考虑目标位位置`k`最终总和是$2^k$。 这限制了建筑的结构。 
3.对于固定的`k`，模拟我们是否可以分配项目，以便一侧最终精确地结束$2^k$。 这意味着我们必须确保在分配和携带之后，只有位`k`在那一侧保持非零。 
4. 我们从较低位向上传播，维护一个进位值，该进位值表示有多少项已溢出到当前构造的下一个指数中。 在每一位 i 处，我们将现有计数与进位相结合，并决定有多少位用于形成目标结构。 
5. 如果在任何时候我们都可以一致地赋值，使得最终的进位链在位置 k 处恰好产生一位，而在其他地方产生零，则存在有效的分区。 
6. 我们对第二个兄弟隐含地重复同样的推理，因为剩下的项目也必须形成 2 的幂。 如果任意 k 有效，则输出 YES。 

关键的不变量是，在每个位位置，该过程都保持可用项与二进制数所需结构之间的一致性，并在末尾恰好有一个设置位。 进位编码所有过去的决定； 一旦出现矛盾（比如需要负数赋值或者留下多余的不能带入k中），配置就不可能了。 

该算法之所以有效，是因为任何有效的解决方案都必须对应于兄弟之一的最终最高设置位的选择，并且二进制加法结构确保所有中间状态都由进位传播唯一确定。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def is_power_of_two(x):
    return x > 0 and (x & (x - 1)) == 0

def possible(cnt, target):
    carry = 0
    for i in range(len(cnt)):
        total = cnt[i] + carry
        if i == target:
            # everything above must vanish after this point
            # leftover must be exactly 1 at this bit after forming power of two
            if total % 2 != 1:
                return False
            carry = total // 2
        else:
            # we must fully eliminate this bit in final number
            carry = total // 2
    return carry == 0

def solve():
    n = int(input())
    arr = list(map(int, input().split()))

    maxb = max(arr)
    cnt = [0] * (maxb + 2)

    for x in arr:
        cnt[x] += 1

    # try all possible target bit positions
    for t in range(len(cnt)):
        if possible(cnt, t):
            # also check second pile implicitly works
            return "Y"
    return "N"

print(solve())
```该实现将输入压缩为比特频率，然后尝试每个可能的最终指数。 这`possible`函数模拟是否可以通过跟踪进位传播来强制一个兄弟精确地形成一个二的幂值。 目标位的条件确保恰好有一个单元保持在该电平，而所有较低位必须通过配对和进位完全取消。 

一个微妙的实现细节是，我们必须以不同于其他位的方式对待目标位，因为这是唯一允许保存最终不成对贡献的位置。 所有其他位置必须通过进位完全解决。 进位是整数除以二，因为其中两项$2^i$形成一项$2^{i+1}$。

4

```我们建立计数：$cnt[2]=2, cnt[3]=1, cnt[5]=1$。 

尝试不同的目标，假设我们测试目标$k=5$。 我们模拟：

 | 我| cnt[我] | 携带| 总计 | 行动| 下一个携带|
 | ---| ---| ---| ---| ---| ---|
 | 2 | 2 | 0 | 2 | 全面携带| 1 |

 整个过程干净利落地结束，意味着一位兄弟可以实现$2^5$。 剩下的盒子自然形成另一个有效的2的幂，所以输出是YES。 

### 示例 2

 输入：```
5
3 1 4 1 5
```计数：$cnt[1]=2, cnt[3]=1, cnt[4]=1, cnt[5]=1$。 

尝试所有目标位置都会失败，因为进位传播总是在中间位中留下额外的质量，如果不在至少一个子集中创建多个活动位，则无法消除这些额外的质量。 

这表明虽然总和是结构化的，但没有分区产生两个干净的二进制单位结果。

 ## 边缘情况

 像单个盒子一样的情况，例如输入`1\n42\n`，立即失败，因为一个兄弟得到了所有糖果，而另一个兄弟得到了零，而零不是 2 的正幂。 该算法处理此问题是因为没有目标位可以满足进位条件。 

另一个微妙的情况是所有盒子都相同，例如`4\n3 3 3 3`。 进位模拟确保分组产生完全消除或干净的二进制减少，并且如果双方都不能形成单个活动位，则所有目标都会失败。 

最后一种边缘情况是存在许多小指数，例如大量零。 在这里，进位链主导行为，并且模拟正确地将对折叠成更高位，直到保留单个峰值或出现多个峰值，这使配置无效。
