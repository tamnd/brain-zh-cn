 ## 解决方案
 ## 问题理解

 我们得到了 cookie 大小的集合，本质上是一个整数数组。 从这个数组中，我们想要选择一个 cookie 子集，使得所选元素位于一个狭窄的值范围内：最大的所选值减去最小的所选值必须最多`k`。 在所有这些有效的子集中，我们被要求最大化我们可以采用的元素数量。 

输入大小可以大到一百万，并且值本身是任意的，最大可达 1e9。 的约束条件为`k`很小，最多 9，这立即表明任何严重依赖于值差异的解决方案都可以使用排序或局部扫描而不是全局组合来优化。 

内存限制非常严格，为 1 MB，这有效地排除了值域上的频率数组以及与`n`存储数据的多个副本。 这推动我们以最少的额外存储进行就地处理或排序流行为。 

当人们试图将问题视为每个值的独立频率聚合时，就会出现一个幼稚的陷阱。 例如，如果 cookie 是`[1, 100, 101, 102]`和`k = 2`，如果不小心强制执行连续的数字间隔约束，则天真的分组可能会通过混合来自不相交区域的计数而过度计数。 

当所有值都相同时，会出现另一种微妙的边缘情况。 例如，`[5, 5, 5, 5]`与任何`k`应该返回`4`，并且任何错误地重置重复窗口的滑动逻辑都会在此处失败。 

## 方法

 蛮力的想法很简单：考虑每个可能的子集，检查其最大值减去最小值是否在范围内`k`，并保留最大的有效值。 即使限制为按排序顺序连续的子集，人们仍可能尝试枚举所有开始和结束对。 排序后，这减少了检查所有间隔的条件`[i, j]`这样`a[j] - a[i] <= k`，并取最大长度。 

这立即表明`O(n^2)`扫描所有对。 和`n`直到 1e6，这是完全不可行的，因为它需要大约 10^12 次比较。 

关键的观察是，排序后，任何最佳子集都必须在排序数组中形成连续的段。 一旦排序，扩大窗口只会增加最大值，缩小窗口只会增加最小值，因此有效性自然表示为滑动区间条件。 这将问题转化为寻找最长子数组，其中端点之间的差异最多为`k`。 

该结构正是两指针技术捕获的结构。 我们保持左边界并尽可能延伸右边界，同时保留约束，仅在必要时缩小左边界。 每个元素最多进入和离开窗口一次，排序后给出线性时间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 排序+两个指针| O(n log n) | O(n log n) | O(1) 额外（除了排序） | 已接受 |

 ## 算法演练

 ## 算法演练

 1. 按非降序对 cookie 大小数组进行排序。 这将问题转化为处理连续值区间，其中任何有效子集都必须在此排序中显示为连续段。 
2.初始化两个指针`l = 0`和`r = 0`，代表所选 cookie 的当前窗口。 在任何时刻，这个窗口都代表一个候选子集。 
3. 展开右侧指针`r`步步。 每次扩容后，检查是否`a[r] - a[l] <= k`。 如果成立，则当前窗口有效并且可能会更新答案。 
4. 如果违反条件，则通过增加从左侧缩小窗口`l`直到条件再次有效。 这是有效的，因为增加`l`只是减少窗口内的最小值，直接恢复可行性。 
5. 过程中，保持最大窗口尺寸`r - l + 1`。 这代表了迄今为止看到的最佳有效子集。 
6. 继续直到`r`到达数组末尾，确保所有可能的右端点都被考虑一次。 

### 为什么它有效

 排序后，任何满足约束的子集都可以按排序顺序映射到连续的段，而不失一般性，因为在有效范围内包含任何跳过的元素都不会违反最大减最小条件。 两个指针不变的是，在每一步，窗口`[l, r]`是保持该段对当前有效的最小左边界`r`。 这保证了没有有效的段结束于`r`会被错过，因为任何较小的左索引都会违反约束，而任何较大的左索引只会不必要地减小大小。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())
a = list(map(int, input().split()))

a.sort()

l = 0
ans = 1

for r in range(n):
    while a[r] - a[l] > k:
        l += 1
    ans = max(ans, r - l + 1)

print(ans)
```该解决方案首先对数组进行排序，以便有效子集对应于值空间中的连续范围。 然后两个指针循环维护一个窗口`[l, r]`总是满足约束条件`a[r] - a[l] <= k`。 每当约束中断时，左指针就会前进，直到恢复有效性。 使用当前窗口长度在每一步更新答案。 

这里的一个常见错误是忘记收缩必须在循环中发生，而不是在单个条件步骤中发生，因为`l`在再次满足约束之前可能需要。

 ## 边缘情况

 一种边缘情况是所有元素都相同。 该算法不断扩展右指针而不收缩，因为`a[r] - a[l]`始终为零。 窗口成为整个数组，正确生成`n`。 

另一个边缘情况是当`k = 0`但价值观重复。 只有相同的值才能共存，并且该算法自然地将相同的元素分组在一起。 当遇到不同的值时，左指针前进，直到窗口再次仅包含该值，以确保正确性。 

最后的边缘情况是严格增加小序列`k`。 窗户不断地折叠成一两号。 例如，与`[1,2,3,4]`和`k = 1`，窗口永远不会超出相邻对，并且算法正确返回`2`而不是错误地允许更长的跨度。
