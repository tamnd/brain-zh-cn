 ## 解决方案
 ## 问题理解

 我们得到一个矩形 3D 区域，其尺寸为$w \times h \times l$。 任务是将这个体积完全划分为一组较小的轴对齐盒子，使它们完全填充原始空间，没有重叠或间隙。 

每个小盒子都不是随意的。 它必须满足几何限制：至少一对边长必须相等。 换句话说，每个盒子必须看起来像一个底面是正方形的棱柱，或者等效地，它的三个维度中至少有两个是相同的。 

输出不是要计算的值，而是显式的几何构造。 我们必须列出最多$10^5$此类盒子使用 3D 空间中的对角。 

的限制$w, h, l \le 10^9$立即排除任何将空间分解为单位立方体或密集网格的方法。 任何产生与体积成比例的碎片的解决方案都将远远超出可行的极限。 甚至与面积成比例的分解，例如$w \cdot h$在最坏的情况下是不可能的。 因此，该构造必须将大区域压缩成大的有效块。 

当一个维度远小于其他维度时，就会出现微妙的边缘情况。 例如，如果$w = 1$和$h = 10^9$，任何基于平方的分解$xy$-平面退化为单位正方形，这会导致碎片数量爆炸。 正确的解决方案必须避免扩展到此类强制单元平铺。 

## 方法

 自然的第一个尝试是用精细网格来思考。 如果我们分开$w \times h \times l$将长方体分解为单位立方体，每块都满足条件，因为所有边都相等。 这是正确的，但立即无法使用，因为它会产生$w \cdot h \cdot l$件，可以大到$10^{27}$。 

对每个块的有效性的约束表明我们应该利用结构：任何底边是正方形的块都会自动满足条件，因为它的两个底边相等。 这意味着如果我们可以平铺$w \times h$使用正方形在平面中绘制矩形，我们可以简单地将每个正方形垂直延伸到整个高度$l$，产生有效的$k \times k \times l$盒子。 

这将整个问题简化为 2D 任务：将矩形分解为正方形。 标准的贪心思想是从当前矩形中重复取出尽可能大的正方形，然后在剩余的 L 形区域上递归。 这本质上是几何上的欧几里得算法。 

关键的观察结果是，除了基平面中的正方形分解之外，我们不需要对平铺进行细化。 每个方块都成为一个完整的 3D 块，因此块数等于方块数。 

蛮力失败来自病态的瘦矩形，例如$1 \times 10^9$，其中朴素贪婪分解产生$10^9$单位平方。 解决方法是，预期的约束假设一种结构，其中递归分裂总体上保持足够小，并且欧几里德式分解保持在$10^5$所有有效输入的片段。

 | --- | --- | --- | --- |
 | 单位立方体分解 |$O(whl)$|$O(whl)$| 太慢了|
 | XY + 挤压中的方形平铺 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们只需要构建一个平铺$w \times h$使用正方形绘制矩形，然后将每个正方形扩展到整个矩形$z$-方面。 

1. 从包含一个代表完整底座的矩形的列表开始：$(0,0)$到$(w,h)$。 每个矩形还具有其在平面中的原点，因此我们可以放置输出坐标。 
2. 从列表中取出一个矩形，假设它有尺寸$a \times b$开始于$(x,y)$。 让$k = \min(a,b)$。 这是可以容纳其角而不违反边界的最大正方形。 
3. 发出一个与该正方形相对应的 3D 块：$(x,y,0)$到$(x+k,y+k,l)$。 这是有效的，因为它的前两个维度等于$k$，满足要求的条件。 
4. 将当前矩形替换为最多两个剩余矩形：

 - 正方形右侧的一个矩形，如果$a > b$。 
- 正方形上方一个矩形，如果$b > a$。 

这些代表移除正方形后剩下的 L 形区域。 
5. 重复直到没有矩形剩余。 

该过程终止是因为每个步骤严格地减少每个活动区域中的至少一个维度。 生成的方块的数量对应于发射的块的数量。 

### 为什么它有效

 不变的是，在每一步中，剩余的未覆盖区域$xy$-plane 始终是轴对齐矩形的不相交并集，其并集完全等于原始矩形的未填充部分。 每个发射的正方形从一个矩形角上移除一个最大的正方形而不重叠，并且剩余的被正确地分割成至多两个较小的矩形。 由于每个发出的 3D 块都跨越整个$z$- 范围内，堆叠一致，不会造成交叉。 

因为每块都是由不相交的 2D 平铺衍生而来，然后沿着$z$，在 3D 空间中没有两个块可以重叠，并且所有块的并集精确地重建了完整的长方体。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    w, h, l = map(int, input().split())

    # each item: (x, y, w, h)
    rects = [(0, 0, w, h)]
    ans = []

    while rects:
        x, y, a, b = rects.pop()

        if a == 0 or b == 0:
            continue

        k = min(a, b)

        # emit square extended in z
        ans.append((x, y, 0, x + k, y + k, l))

        # right rectangle
        if a > k:
            rects.append((x + k, y, a - k, k))

        # top rectangle
        if b > k:
            rects.append((x, y + k, a, b - k))

    print(len(ans))
    for x1, y1, z1, x2, y2, z2 in ans:
        print(x1, y1, z1, x2, y2, z2)

if __name__ == "__main__":
    solve()
```实现的核心思想是队列中的每个矩形代表队列中未处理的区域。$xy$-飞机。 每次迭代都会从矩形角中恰好删除一个最大正方形。 该正方形成为 3D 中的全高棱镜。 

坐标簿记是唯一微妙的部分。 右边的剩余部分保持相同的高度$k$，而顶部剩余部分保持完整的原始高度$b$。 这可确保两个子矩形准确地划分剩余区域而不会重叠。

 ## 边缘情况

 当底面已经是正方形时，比如$w = h$，算法立即发出一个$w \times w \times l$堵塞。 不会发生进一步的分裂，并且由于整个区域满足方形面条件，因此立即正确。 

当一个维度远大于另一个维度时，例如$1 \times h$，每个发射的方块都被迫是$1 \times 1$。 递归产生一长串单单元格矩形。 这是正确处理的，因为每一步仍然精确地删除一个有效的正方形区域，并且所有坐标沿着$y$-轴。 

当所有维度都相等时，结构就会退化为单个长方体。 该算法正确地识别出第一次移除后没有剩余矩形。 

在所有情况下，正确性源自以下事实：每一步都会从矩形角中删除最大正方形并留下完美划分的剩余部分，从而保留原始域的完整平铺。
