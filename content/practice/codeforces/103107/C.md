 ## 解决方案
 ## 问题理解

 我们有两个简单的多边形，每个多边形都由其顶点按逆时针顺序描述。 你可以把它们想象成凸面饼干被打碎后躺在平面上的两块“破碎的饼干片”。 任务是确定是否存在一种方法来平移其中一个块，以便当将两个块放置在一起时，它们的并集形成一个凸多边形，并且边缘对齐方式不会出现几何不一致。 

关键条件不仅是形状重叠得很好，而且它们的边界必须以一致的方式边缘到边缘匹配。 特别是，不能出现多边形的一条边与另一条边的多条边对齐的情况，也不能出现任何类似的破坏边界线段一一对应关系的简并性。 平移后，组合的外边界必须追踪单个凸多边形。 

从计算的角度来看，每个多边形最多有 2000 个顶点，因此尝试所有边对或顶点对之间的所有平移的简单几何对齐测试已经是边界。 三次甚至二次边缘配对可能会遇到大约 10^6 到 10^7 次几何检查，并且每次检查都涉及段比较或方向测试，这仍然是可以接受的。 然而，任何有效尝试所有顶点对对齐并从头开始重新计算凸性的解决方案都会将时间推向 O(n^2 m) 或更糟，这太慢了。 

主要的边缘情况是几何简并，其中：

 多边形已经是凸多边形，但有许多共线的连续点。 在这种情况下，多个边位于同一线段上，天真的“边匹配”逻辑可能会错误地将它们视为不同的或错误地重叠。 

另一种情况是，两个多边形都是凸多边形，但其中一个多边形相对于另一个多边形旋转，使得仅存在支撑线的部分重叠。 例如，如果两个三角形共享两条平行边，但它们的第三条边不兼容，则仅基于并行性的朴素匹配可能会错误地接受。 

另一个微妙的情况是多边形共享一条长共线链。 例如，如果一个多边形具有由许多线段组成的长直边界，而另一个多边形仅匹配其中的一部分，则朴素线段匹配可能会违反语句中描述的一对一约束。 

## 方法

 强力解释首先尝试将第一个多边形的每个顶点与第二个多边形的每个顶点对齐，并使用该对作为平移向量。 翻译后，我们将尝试检查联合边界是否形成凸多边形以及所有边界边是否以有效方式匹配。 

这种方法原则上是正确的，因为任何有效的平移都必须将一个多边形的至少一个顶点映射到另一个多边形的某个顶点，或者至少以一种在考虑多边形边的离散结构后可以简化为顶点对齐的方式对齐支撑边。 然而，问题是成本。 尝试所有 O(nm) 转换，然后重新计算合并的外壳或验证凸性，每次尝试的成本为 O(n + m) 或更糟，导致 O(n^2 m + nm^2) 复杂度，当 n、m 达到 2000 时，这远远超出了限制。 

关键的观察结果是平移不会改变边缘方向或长度。 重要的是每个多边形周围的边向量的序列。 如果两个多边形可以通过平移合并成凸多边形，那么它们的边界循环必须以保留全局凸性的方式交织。 这将问题转化为边缘向量循环序列的比较：我们本质上是检查两个循环序列是否可以连接（可能具有循环移位）以形成单个凸多边形边界，其中所有转弯都处于同一方向并且边缘一致匹配。

这将问题简化为处理边缘方向序列，并确保当我们组合它们时，所得序列形成有效的凸包边界。 对此进行推理的标准方法是将每个多边形标准化为其边缘向量序列，然后尝试“合并”这些序列，同时保留凸转弯结构。 这与Minkowski和边界结构和凸多边形卷积思想密切相关。 

一旦通过这个镜头来看，问题就变成了检查边缘向量的两个循环序列是否可以对齐，使得它们的组合角顺序是单调的并且不会出现方向冲突。 这可以通过将边缘转换为极角、排序并检查两个序列是否位于不以冲突方式包裹的连续角度间隔内，并确保方向一致（顺时针或逆时针一致性）来完成。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（尝试所有翻译+重新计算凸包）| O(n^2 m) 或更糟 | O(n + m) | 太慢了 |
 | 边向量角度合并（Minkowski 式推理）| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 1. 通过减去连续的顶点来计算两个多边形的边向量，将最后一个顶点视为连接回第一个顶点。 这将每个多边形转换为有向线段的循环序列。 这一步的原因是平移不影响边缘方向，因此边缘是形状的不变结构。 
2. 使用atan2 将每个边缘向量转换为角度表示。 这使我们能够在全局角度空间而不是坐标空间中推理多边形的转动行为。 
3. 标准化每个多边形的边缘角度序列，以便循环处理，这意味着我们在概念上允许起点旋转。 这是必要的，因为起始顶点是任意的。 
4. 检查每个多边形在其边角序列中是否单独形成凸链。 这确保了每个片段在合并后已经兼容形成凸边界。 
5. 尝试将两个循环角度序列合并为单个循环序列，其中角度以 2π 为模单调递增。 这对应于检查是否存在旋转，使得两个多边形的所有边都可以放置在单个凸包排序中。 
6. 验证没有角度间隔以冲突的方式环绕。 实际上，这简化为检查两个角度集的并集是否位于长度 π 或更小的某个半开区间内，以确保所得边界的凸性。 
7. 如果存在这样的对齐，则输出“yes”，否则输出“no”。 

### 为什么它有效

 凸多边形的特征在于其边缘方向角的单调性。 当两个多边形平移并合并时，它们的组合边界仍然必须满足这种单调角度排序。 如果存在有效的平移，则存在一致的方式将两个边缘方向序列交织成单个循环序列而不引入凹转弯。 相反，如果不存在这样的角度顺序，则任何放置都将强制产生方向冲突，从而产生反射角或边缘不匹配，从而违反凸性或一对一边缘对应条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import math

def read_polygon(n):
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    edges = []
    for i in range(n):
        x1, y1 = pts[i]
        x2, y2 = pts[(i + 1) % n]
        dx, dy = x2 - x1, y2 - y1
        edges.append(math.atan2(dy, dx))
    return edges

def is_convex_angles(angles):
    n = len(angles)
    # find max gap
    a = sorted(angles)
    a.append(a[0] + 2 * math.pi)
    max_gap = 0
    for i in range(n):
        max_gap = max(max_gap, a[i + 1] - a[i])
    # convex if all angles lie in an arc of length <= pi
    return (2 * math.pi - max_gap) <= math.pi

n = int(input())
A = read_polygon(n)

m = int(input())
B = read_polygon(m)

angles = A + B

print("yes" if is_convex_angles(angles) else "no")
```此代码将每个多边形减少到其边缘方向，然后检查所有方向是否可以组合成角度空间中的半圆。 该条件相当于能够将合并边界定向为凸多边形。 

微妙的一点是我们从不明确尝试翻译。 平移仅影响位置，而不影响方向，因此整个可行性问题归结为方向约束是否兼容。 排序步骤至关重要，因为它将循环排序问题转换为线性区间问题。

 ## 边缘情况

 一种重要的边缘情况是多边形具有许多共线边缘。 在这种情况下，多个连续的边会产生相同的角度，而基于唯一性的简单方法会错误地缩小角度集并低估真实的跨度。 该算法可以正确处理此问题，因为重复项不会影响计算的最大角度间隙。 

另一种边缘情况是当一个多边形几乎是一条具有前后小偏差的直线时。 边缘角度将紧密地聚集在半圆边界周围，如果总跨度不超过 π，算法会正确地将其视为潜在可合并的。 

最后一种边缘情况是角度围绕 -π 到 π 边界。 通过在排序数组中附加第一个角度加上 2π，该算法可以正确处理循环连续性并避免换行点处的错误间隙。
