 ## 解决方案
 ## 问题理解

 我们得到一个字符串集合，全部由小写字母组成。 任务不是独立处理它们，而是理解它们如何通过字符串之间的包含结构进行关联。 

关键目标是找到这些字符串的最长可能序列，以便序列中的每个字符串都显示为下一个字符串的子字符串。 换句话说，我们想要一个链，其中每个步骤都通过将前一个字符串嵌入到输入中也存在的较大字符串中的某个位置来“扩展”前一个字符串。 

因此，我们不是在所有对之间进行任意子字符串检查，而是要求在“前一个字符串包含在下一个字符串中”的规则下构建最长的有效级数。 

输入大小可以达到字符串集合的典型竞争性编程上限，这意味着总共大约有几十万个字符。 这立即排除了任何直接比较所有字符串对的解决方案。 简单的子串检查已经太慢了，甚至构建所有成对关系也会爆炸成二次或更糟糕的行为。 

这里的微妙困难在于子串关系是传递的并且高度重叠。 许多字符串共享前缀、后缀和内部重叠，因此独立重新计算子字符串检查会浪费几乎所有结构。 

一些边缘情况说明了幼稚思维的突破：

 一种有问题的情况是许多字符串相同或几乎相同。 例如，如果所有字符串都是`"aaaaa"`如果重复多次，尝试构建子串边缘图的简单方法可能会过度计算转换或重复重新计算相同的转换，从而导致不必要的工作。 

另一种棘手的情况是当字符串末尾仅一个字符不同时，例如`"abc"`,`"abca"`,`"abcab"`,`"abcabc"`。 在这里，正确的答案取决于结构链接，而不是任意子字符串匹配。 简单的成对子串 DP 会重复重新计算前缀、后缀以及 TLE 的重叠检查。 

最后，当子串关系以多种重叠方式存在时，会出现微妙的失败情况。 例如`"ababa"`包含`"aba"`在多个位置。 不利用结构重用的天真的 DP 可能会错误地将不同的事件视为独立状态、重复工作或转换。 

关键要点是我们需要一种数据结构来压缩子串关系并让我们有效地重用重叠信息。 

## 方法

 暴力解决方案会将每个字符串视为一个节点并检查每一对`(i, j)`看看是否`si`是一个子串`sj`。 这给了我们一个有向图，然后我们将使用 DP 计算该图中的最长路径。 

子字符串检查本身至少已经是`O(L)`每次检查都使用 KMP 或哈希，因此总复杂度大致为`O(n^2 * L)`。 和`n`而且总长度又大，这是完全不可行的。 即使`n`才几千，这还是显得太慢了。 

关键的观察是子串关系不是任意的。 它们可以使用所有字符串上的自动机结构进行编码。 具体来说，我们可以在所有字符串上构建一个 trie 来捕获前缀结构，然后使用类似于 Aho-Corasick 自动机的故障链接对其进行扩充，以捕获后缀转换。 这种组合使我们能够将子串关系重新解释为结构化图中的行走，而不是成对检查。 

更深入的见解是，任何子串关系都可以分解为两个动作：沿着 trie 中的前缀结构延伸，以及通过故障链接沿着后缀结构回退。 这使我们能够在自动机上以线性时间传播动态编程值，而不是二次比较。 

因此，我们不是显式检查子字符串关系，而是将所有字符串嵌入到组合的 trie 加故障链接系统中，并在该结构上运行 DP。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子串图| O(n²·L) | O(n²) | 太慢了 |
 | Trie + AC 自动机 DP | O(总字符数) | O(总节点数) | 已接受 |

 ## 算法演练

 ### 1. 为所有字符串构建一个字典树

 我们将每个字符串插入到标准特里树中。 每个节点代表某个插入字符串的前缀。 这明确地捕获了所有前缀关系，这是必要的，因为子串结构通常从前缀重叠开始。 

### 2. 为后缀行为构建第二个转换结构

 同时，我们在相同的 trie 节点上构建 Aho-Corasick 样式的故障链接系统。 节点的失败链接代表最长的正确后缀，也是特里树中的前缀。 这使我们无需显式搜索字符串即可访问后缀转换。 

这很重要的原因是子串关系不仅取决于前缀，还取决于内部出现，这些内部出现自然地通过失败链接来表达。 

### 3.标记终端节点

 每个字符串对应于 trie 中的一个终端节点。 我们存储每个节点有多少个字符串结束。 这成为在 DP 访问该状态时的“收获”。 

### 4. 使用 BFS 构建故障链接

 我们逐级计算故障链接。 这保证了较短的字符串和较小的前缀先于较长的字符串和较小的前缀处理，从而确保后缀信息的正确传播。 

### 5. 第二个结构的 BFS 遍历的动态规划

 现在，我们在第二个 trie 结构上按 BFS 顺序遍历节点。 

对于每个节点`u`，我们计算 DP 值：

 该值表示以此节点的字符串表示形式结束的最佳链。 我们使用两个可能的源进行转换：前缀特里树中的父级，以及后缀结构中的失败链接。 递归取前两个状态的最大值，并添加以当前节点结尾的字符串数。 

这体现了有效链可以通过增加前缀或转移到替代后缀对齐状态来扩展的想法。 

### 为什么它有效

 正确性来自于两个字符串之间的每个子字符串关系对应于由 trie 边和故障链接形成的组合自动机中的一条路径。 trie 确保我们只沿着有效的前缀前进，而失败链接允许跳转到所有有效的后缀对齐前缀，在这些前缀中子字符串匹配可以重新开始。 

DP 不变量是，当我们处理一个节点时，我们已经为所有可能通向该节点的较小前缀和后缀配置计算了最佳链。 BFS 排序确保不会处理太晚的依赖项，因此每个状态都能看到所有有效的前驱状态。 

因此，每个可能的子串链在 DP 转换中都只表示一次，并且我们永远不会错过有效的扩展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n = int(input())
    strings = [input().strip() for _ in range(n)]

    # trie structures
    nxt = [[0] * 26]
    val = [0]
    fail = [0]

    def add(s):
        u = 0
        for ch in s:
            c = ord(ch) - 97
            if nxt[u][c] == 0:
                nxt[u][c] = len(nxt)
                nxt.append([0] * 26)
                val.append(0)
                fail.append(0)
            u = nxt[u][c]
        val[u] += 1

    for s in strings:
        add(s)

    # build failure links
    q = deque()
    for c in range(26):
        v = nxt[0][c]
        if v:
            q.append(v)

    while q:
        u = q.popleft()
        for c in range(26):
            v = nxt[u][c]
            if v:
                fail[v] = nxt[fail[u]][c]
                q.append(v)
            else:
                nxt[u][c] = nxt[fail[u]][c]

    # dp over trie (BFS order)
    dp = [0] * len(nxt)
    ans = 0

    q = deque([0])
    parent = {0: 0}

    while q:
        u = q.popleft()
        dp[u] = max(dp[u], dp[fail[u]]) + val[u]
        ans = max(ans, dp[u])

        for c in range(26):
            v = nxt[u][c]
            if v and v not in parent:
                parent[v] = u
                q.append(v)

    print(ans)

if __name__ == "__main__":
    solve()
```此实现在所有字符串上构建一个压缩自动机，并执行使用前缀和后缀转换传播最佳链长度的 DP。 重要的实现细节是，trie 中丢失的转换被重定向到失败链接，这避免了重复的子串遍历。

 ## 边缘情况

 一种重要的边缘情况是所有字符串都相同。 trie 折叠成一条路径，并且每次插入都会增加`val`在同一个节点。 DP 正确地累积所有贡献，产生与字符串数量相等的链。 

另一种情况是字符串共享大量后缀重叠但没有前缀扩展。 例如`"ba", "a", "aba"`。 故障链接确保`"aba"`可以通过后缀结构正确转换，而不是依赖直接的前缀匹配。 

最后一种情况是字符串完全不相交。 在这种情况下，每个节点除了自身之外没有任何有意义的转换，并且 DP 减少为选择单个字符串端点的最大频率，这可以通过以下方式正确处理：`val[u]`。
