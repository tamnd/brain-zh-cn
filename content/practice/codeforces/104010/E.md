 ## 解决方案
 ## 问题理解

 我们站在无限数线上的零位置，想要到达某个目标坐标$x$，可以是正数、负数或零。 一步中，我们选择一个非负整数$k$，然后向左或向右跳跃恰好$2^k$。 因此，可用的移动长度是 2 的幂，每次移动都允许我们独立选择方向。 

任务是计算准确着陆所需的最小跳跃次数$x$。 每个测试用例都是独立的，最多可以有100000个查询，而$x$可以大到$10^{18}$在幅度上。 

的约束条件为$t$迫使我们在大致恒定或对数时间内回答每个查询。 任何试图模拟路径或搜索状态的解决方案都是立即不可行的，因为即使是中等的分支因子也会呈指数级爆炸。 的大小$x$建议采用二进制表示结构，因为$10^{18}$适合大约 60 位。 

当出现微妙的边缘情况时$x = 0$。 在这种情况下，不需要任何移动，但如果基于二进制表示的天真推理总是尝试分解为 2 的幂，则可能会错误地分配至少一个移动。 

另一个极端情况是负面的$x$。 由于我们可以自由选择每次移动的方向，对称性表明只有$|x|$很重要，但这必须通过运营结构来证明，而不是假设。 

## 方法

 强力解释会将每个状态视为线上的坐标，并将每次移动视为分支到两个新状态：加或减$2^k$对于任何$k \ge 0$。 从零到的最短路径搜索$x$从概念上讲是正确的，因为所有移动都会花费一步。 然而，状态空间是无限的、高度连通的，甚至将坐标限制为$[-10^{18}, 10^{18}]$留下了太多的州。 由于任意的原因，每个状态也有无限多个传出转换$k$，所以即使枚举转换也是不可能的。 这种方法会立即失败，因为分支因子是无界的，并且最优解的深度可以达到 60 左右。 

关键的观察结果是，每次移动都对应于切换坐标的二进制表示中的单个位位置，因为$2^k$仅影响位$k$。 然而，与标准二进制加法不同，我们可以在每次移动时独立选择符号，这意味着我们不限于以通常的方式进行进位传播。 相反，问题变得等同于表示$x$作为二的有符号幂的总和，最大限度地减少项数。 

这正是整数的非相邻形式 (NAF) 表示的经典概念，其中我们以 2 为基数重写数字，但允许使用数字$-1, 0, 1$，最小化非零数字的数量。 每个非零数字对应一个跳转，最小化跳转相当于最小化此有符号二进制展开中的非零数字。 

NAF 的贪婪构造一点点起作用，决定是否使用$\pm 1$在当前位或在奇偶校验强制时进位到下一位。 这消除了二进制表示中的长链，这正是导致朴素二进制分解效率低下的原因。

 ## 算法演练

 我们独立处理每个数字，将其减少到其绝对值，因为符号无关。 

1.如果$x = 0$，立即返回0。 不需要跳跃，因为我们已经在原点了。 
2. 更换$x$经过$|x|$。 这是有效的，因为每次移动都可以选择方向，因此任何移动序列都可以到达$x$可以镜像到达$-x$具有相同的长度。 
3. 将操作计数器初始化为零。 这将表示我们最终使用的非零符号位的数量。 
4. 虽然$x > 0$，检查是否$x$是奇数还是偶数。 如果$x$是偶数，我们除以 2 并移动到下一位。 这对应于当前位置的有符号二进制表示中的零位。 
5.如果$x$是奇数，我们决定在该位加1还是减1。 在二元 NAF 构造中，如果$x \bmod 4 = 1$，我们使用数字$+1$，否则如果$x \bmod 4 = 3$，我们使用数字$-1$。 选择后，我们增加操作计数器并更新$x = (x - d)/2$， 在哪里$d \in \{+1, -1\}$。 
6. 重复此操作，直至处理完所有位。 

第 5 步的关键思想是选择$\pm 1$确保我们避免在表示中产生连续的非零位，否则会增加所需跳转的数量。 

### 为什么它有效

 在每一步中，我们都构建一个表示$x$以 2 为基数，数字为$\{-1, 0, 1\}$。 每次减或加 1 对应于提交大小的跳跃$2^k$。 贪婪选择确保每当以不利的方式设置一个位（在标准二进制中创建两个相邻的非零数字）时，我们可以通过将其翻转为有符号数字并向前推进来立即解决它。 这保证了不存在两个相邻的非零数字，这是最小长度有符号二进制表示的定义属性。 由于每个非零数字恰好对应一次移动，因此最小化它们会直接最小化跳跃次数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_one(x):
    if x == 0:
        return 0
    x = abs(x)
    ops = 0

    while x > 0:
        if x & 1 == 0:
            x >>= 1
        else:
            if x == 1:
                ops += 1
                break
            if x & 3 == 1:
                x -= 1
            else:
                x += 1
            ops += 1
            x >>= 1

    return ops

t = int(input())
for _ in range(t):
    x = int(input())
    print(solve_one(x))
```该实现直接镜像签名的二进制结构。 该循环从最低有效侧开始逐位处理该数字。 当当前位为零时，我们只需移位即可。 当它是 1 时，我们根据编码的下一位上下文来决定是减还是加 1$x \bmod 4$，这可以防止未来出现非零数字的相邻情况。 

特殊情况$x = 1$被明确处理，因为除了计算一个最终操作之外不需要进位调整。

 ## 边缘情况

 对于$x = 0$，算法在进入循环之前立即返回零，这避免了错误地计算二进制处理循环中的幻像操作。 

为了$x = 1$，循环以单个操作终止，并且不再进行移位，因为没有剩余高位。 这避免了额外的不必要的调整步骤，否则会尝试访问不存在的较高位。 

对于 2 的大幂，例如$x = 2^{60}$，该算法仅执行移位直到最后一位，仅产生一个操作，符合该大小的单次跳转就足够的直觉。
