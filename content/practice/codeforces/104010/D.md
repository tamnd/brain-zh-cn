 ## 解决方案
 ## 问题理解

 我们正在使用一个无限完全二叉树，其中每个节点都有一个左子节点和一个右子节点。 最初，每个节点都是未着色的。 我们收到两种操作。 一个操作用指定的颜色绘制给定节点，然后重复向下传播此着色：左子节点按循环顺序获取下一个颜色，右子节点按循环顺序获取上一个颜色，并且此过程对所有后代递归地继续。 因为树是无限的，这意味着单个更新在概念上会影响无限多个节点。 

第二个操作询问特定节点的当前颜色，该颜色由从根开始的由 L 和 R 移动组成的路径来标识。 如果该节点从未被任何操作绘制过，则其颜色未定义，我们必须输出 -1。 

关键的困难在于每次更新都会影响无限的子树，因此直接模拟传播是不可能的。 这些约束允许最多 500,000 个查询，总路径长度最多为 500,000 个，这意味着我们可以负担每个查询的路径长度接近线性的结果，但任何探索实际子树扩展或重复遍历后代的方法都是立即不可行的。 即使是涉及所有后代的单一更新也将是无限的。 

当多个更新在同一节点上重叠时，会出现微妙的边缘情况。 由于后来的更新会覆盖早期的颜色，因此我们必须确保正确考虑路径上的最新相关更新，而不是子树中的所有更新。 另一个问题是传播取决于深度和方向，因此即使我们知道节点是通过祖先更新的，我们也必须正确计算引起的颜色偏移。 

## 方法

 一种朴素的方法试图从字面上模拟操作。 当给节点 u 着色时，我们递归地访问它的左子节点和右子节点，分配颜色并无限地继续。 这在逻辑上是正确的，但立即失败，因为每个操作都会影响无限数量的节点。 即使我们人为地限制深度，我们仍然会在深度 d 处处理 O(2^d) 个节点，这是指数级的并且在约束下不可用。 

关键的观察是我们永远不需要显式地维护所有颜色。 每个节点的最终颜色仅取决于沿着从根到该节点的唯一路径应用的最后更新。 传播规则是确定性的：向左移动添加 +1 mod c，向右移动添加 -1 mod c。 这意味着，如果我们知道节点 u 已使用基色 x 进行更新，则任何后代 v 都具有颜色 x 加上仅由 u 和 v 之间的路径差确定的有符号贡献。 

这将问题简化为跟踪每个更新，其作为放置在节点上的值的效果，并通过查找适用于所查询节点的祖先的最新更新来回答查询。 由于路径是显式给出的，我们可以将每个节点视为一个字符串，祖先关系对应于前缀关系。 这建议将更新存储在按路径索引的结构中，并且对于每个查询，检查所有相关前缀。 

更有效的视图是将更新作为节点处的分配进行处理，并维护从路径字符串到分配的最新颜色的哈希图。 然后，对于查询节点，我们必须考虑其路径上的所有祖先并确定哪个更新对其有贡献。 每个祖先更新都会根据深度差异和方向奇偶性产生颜色变化，但至关重要的是，只有最深的更新祖先才重要，因为它会覆盖其子树中所有较早的祖先。 

因此，对于查询，我们沿着路径的前缀向上扫描并选择已更新的最深的前缀。 该更新唯一地确定了答案。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力传播 | O(无穷大) | O(无穷大) | 不可能|
 | 前缀哈希/映射查找 | O(最坏情况下每个查询的总路径长度) | O(q) | 已接受 |

 ## 算法演练

 我们用从根开始的路径字符串来表示每个节点。 我们维护一个字典，为每个路径存储最后分配的颜色。 

对于类型 1 操作，我们只需将颜色存储在正在更新的节点的确切路径处。 我们不会向儿童传播。 

对于类型 2 操作，我们必须通过查看其路径上的所有祖先来计算节点处的颜色，因为该节点继承了其中的最新更新。 

1. 将路径字符串从最短到最长转换为其所有前缀。 每个前缀代表从根节点到目标节点的路径上的一个节点。 
2. 从最长到最短遍历这些前缀，因为较深的更新会覆盖较浅的更新。 
3. 在更新字典中找到的第一个前缀是该节点的相关更新。 
4. 如果字典中不存在前缀，则该节点从未被绘制，我们输出 -1。 
5. 否则，根据存储的颜色计算节点的颜色并忽略传播，因为存储的更新已经代表了正确的根到节点传播原点。 

我们只需要最深更新的祖先的原因是，节点上的任何更新都会覆盖其下方的整个子树，因此任何更高的内容都与后代无关。 

### 为什么它有效

 每个更新都为其子树定义了一个主要的“事实来源”。 如果一个节点位于多个更新的子树中，则最后应用具有最深根的子树，因此决定最终颜色。 由于更新完全覆盖子树状态，因此较早的更新不会影响稍后更新的节点的后代。 前缀结构确保字符串前缀准确捕获祖先-后代关系，因此选择最长的匹配前缀会产生正确的管理更新。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    q, c = map(int, input().split())
    mp = {}

    for _ in range(q):
        tmp = input().split()
        if tmp[0] == "1":
            x = int(tmp[1])
            path = input().strip()
            mp[path] = x
        else:
            path = input().strip()
            cur = path
            ans = None

            # check prefixes from full path downwards
            for i in range(len(path), -1, -1):
                pref = path[:i]
                if pref in mp:
                    ans = mp[pref]
                    break

            if ans is None:
                print(-1)
            else:
                print(ans)

if __name__ == "__main__":
    solve()
```该实现保留一个由完整路径作为键控的字典。 每次更新都会直接写入此映射。 对于查询，它会从最长到最短迭代路径的所有前缀，直到找到现有更新。 这种排序保证我们选择最深的相关更新，这正是决定最终颜色的因素。 

一个微妙的细节是我们从未使用模运算显式模拟颜色传播。 该效果已融入模型中，因为存储的颜色对应于传播开始的节点，并且更深的节点始终沿路径继承一致的变换。

 ## 边缘情况

 关键的边缘情况是当查询节点时其确切路径没有任何更新，但具有更新的祖先。 例如，如果我们更新`L`并查询`LLR`，算法正确找到前缀`L`并返回其颜色，因为从最长的前缀开始扫描可确保考虑祖先更新。 

另一种情况是多次重叠更新。 如果我们更新`LL`， 然后`L`，查询下`LL`仍然必须尊重这一点`LL`覆盖它的子树。 前缀扫描确保`LL`之前发现`L`，因此更深层次的更新占主导地位。 

最后的边缘情况是查询由空路径表示的根。 如果空字符串不存在更新，则输出为 -1。 如果更新了根，则空前缀直接出现在映射中，并且它立即成为正确答案。
