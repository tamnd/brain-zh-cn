 ## 解决方案
 ## 问题理解

 我们的晋升是循环进行的。 如果您购买一定数量的冰淇淋，例如$X$，公司给你$Y$免费额外单位。 此规则重复适用：每次您积累另一整批$X$付费单位，您会收到另一个$Y$免费单位。 

每个冰淇淋单位的固定价格为 3 美元，但只有您实际支付的单位才算成本。 免费单位会增加您最终获得的冰淇淋总数，但不会影响价格。 

对于每个测试用例，我们希望确定所需的最小付费单位数量，以便收到的冰淇淋总数，包括所有已完成的免费赠品$X$- 规模的采购，至少是$N$。 最终答案是付费单位数的3倍。 

关键的困难是自由单位取决于有多少个完整的组$X$我们买。 如果我们购买的数量少于$X$单位，我们没有奖金； 一旦我们到达$X$，我们突然获得了额外的$Y$，并且这种不连续性每次都会重复$X$单位。 

限制足够小，我们可以负担得起$O(N \log N)$甚至$O(N^2)$每个测试用例的风格检查。 和$N \le 15000$最多 15 个测试用例，甚至每个用例几十万次评估也是安全的，但只有当每次评估都是恒定时间时，独立尝试所有可能的购买策略组合的任何操作仍然是可以接受的。 

当贪婪思维应用不当时，就会出现微妙的失败案例。 例如，与$X = 4$,$Y = 7$， 和$N = 22$，人们很容易认为购买足够多的完整组总是最佳的。 但部分剩余部分很重要：在完成一些完整的组之后，购买一些额外的付费单元（无需完成另一个完整的组）$X$）可能是有效跨越门槛所必需的。 忽略这一点会导致逐段错误，从而低估答案。 

另一种边缘情况发生在$X = 1$。 然后每个付费单位立即生产$Y$自由单位，将问题转化为简单的线性方程。 假设至少需要一个完整块的简单实现可能会使这种情况变得过于复杂或错误处理。 

## 方法

 最直接的想法就是模拟购买过程。 我们尝试增加付费单位的数量$b$，并为每一个计算我们收到的免费单位数。 如果$b$包含$\lfloor b / X \rfloor$满块，那么空闲单元的数量正好是$Y \cdot \lfloor b / X \rfloor$，冰淇淋总数变为$b + Y \cdot \lfloor b / X \rfloor$。 我们停在最小的地方$b$该总数至少达到$N$。 

这种强力扫描是正确的，因为每种可能的购买策略都对应于一定数量的付费单位$b$，总收益完全由下式决定$b$。 问题是效率：在最坏的情况下我们可能会检查所有$b$从 0 到$N$，每次检查都是恒定时间，给出$O(N)$每个测试用例。 虽然这对于这些约束来说已经是可以接受的，但我们可以使用二分搜索更清晰地构建解决方案，因为冰淇淋的总数在$b$。 

关键的观察是单调性。 如果我们增加付费单位的数量，我们永远不会减少获得的冰淇淋总数。 即使该函数的跳转次数为$X$，它永远不会减少。 这使我们能够二分查找最小的$b$使得条件成立。 

我们定义一个检查函数，用于计算给定的冰淇淋总数$b$，然后二分查找最小的有效值$b$在$[0, N]$。 上限$N$是安全的，因为购买$N$单位总是至少给出$N$冰淇淋总数，无论免费赠品如何。

 ## 算法演练

 我们独立处理每个测试用例。 

1. 定义一个函数，计算购买冰淇淋的总数量$b$付费单位。 完整组数为$b // X$，每组贡献$Y$免费单位。 总计为$b + (b // X) \cdot Y$。 
2. 设置二分查找范围$b$从 0 到$N$。 从成本意义上来说，下限是基本可行的，上限是安全的，因为购买$N$单位始终保证至少$N$总共冰淇淋。 
3. 执行二分查找。 对于中点$mid$，使用步骤 1 中的函数计算冰淇淋总数。 
4. 如果总数至少为$N$，答案可以是$mid$或更小，所以我们将右边界移动到$mid$。 
5. 否则，$mid$不够，所以我们将左边界移动到$mid + 1$。 
6. 收敛后，左边界表示所需的最小付费单位数。 

正确性来自这样一个事实：冰淇淋总数是以下函数的函数：$b$是单调非减的。 尽管它以数倍的步长增加$X$，当$b$增加。 这保证了二分搜索永远不会错误地丢弃有效区域，并且始终收敛到最小的可行区域$b$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        X, Y, N = map(int, input().split())

        def total(b):
            return b + (b // X) * Y

        lo, hi = 0, N

        while lo < hi:
            mid = (lo + hi) // 2
            if total(mid) >= N:
                hi = mid
            else:
                lo = mid + 1

        print(lo)

if __name__ == "__main__":
    solve()
```该解决方案将逻辑包装到每个测试循环中。 辅助函数`total(b)`直接捕捉促销的结构。 二分查找保持了答案始终位于其中的不变量`[lo, hi]`。 什么时候`total(mid)`满足要求，我们安全地缩小上限，因为任何较大的值`b`不可能比`mid`。 否则我们增加`lo`因为所有较小的值也会失败。 

一个常见的错误是试图通过仅考虑完整组的大小来计算答案$X$并忽略其余部分。 二分搜索通过评估每个候选者的精确公式来完全避免这种情况。

 ## 边缘情况

 当$X = 1$，每个购买的单位立即生成$Y$免费单位。 该函数变为$b \mapsto b(1+Y)$，二分查找简化为求解一个简单的线性阈值。 例如，与$Y = 1$和$N = 3$，最小的$b$是 2 因为$2 \cdot 2 = 4$，已经满足要求了。 该算法自然地处理了这个问题，因为总函数在以下方面保持单调和连续：$b$。 

什么时候$Y = 0$，促销消失，答案就变成了$b = N$。 函数简化为恒等式，二分查找通过反复测试mid是否至少收敛$N$。 不需要特殊的外壳。 

什么时候$N$刚好高于$X + Y$，最佳解决方案通常需要将完整数量的已完成块与少量余数混合。 例如，$X = 4, Y = 7, N = 22$表明停在两个完整的块上是不够的，需要部分块。 评估公式正确地解释了这一点，因为它重新计算了每个候选人的楼层划分项$b$，确保块完成之间不会遗漏边界。
