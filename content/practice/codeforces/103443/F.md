 ## 解决方案
 ## 问题理解

 我们得到了一系列轴对齐的矩形海报，每张海报都涂有一种颜色，并依次放置在一面巨大的墙上。 当放置新海报时，它会完全覆盖其下方区域中的所有内容。 任务不是逐个像素地重建最终的可见图像，而只是确定应用所有叠加后最终墙壁上的任何位置有多少种不同的颜色是可见的。 

每个矩形由两个对角给出，其中坐标很大，最多可达约$2^{28}$，最多存在 4000 个矩形。 矩形不会旋转，并且保证具有一致的方向，因此每个矩形都是一个标准的轴对齐框。 关键的困难在于重叠区域必须遵守插入顺序，这意味着无论重叠的地方，后面的海报都会支配前面的海报。 

一个天真的解释是想象一个网格并模拟绘画，但坐标太大而无法直接离散。 即使我们压缩坐标，完整的逐个单元模拟仍然会变得太大，因为压缩单元的数量是$n$，每个单元格可能需要扫描许多矩形。 

天真的“标记每个单元格的最后一种颜色”方法的一个微妙的失败案例来自于坐标之间具有较大空白空间的重叠矩形。 例如，如果一个矩形覆盖了几乎所有内容，并且后面的一个小矩形重叠了其中的一部分，则即使大部分区域未更改，可见颜色集也会发生变化。 任何仅跟踪重叠计数而不正确遵守最高顺序的解决方案都将失败。 

另一个常见的陷阱是假设全局中只有最上面的矩形很重要。 这是错误的，因为墙壁的不同区域可以有不同的最上面的矩形，因此可以同时看到多种颜色。 

输出仅取决于哪些颜色至少有一个区域是最高层。 

约束条件允许$n = 4000$，这表明$O(n^2)$或者$O(n^2 \log n)$解决方案是可以接受的，但是任何立方体超过矩形的速度都太慢。 由于每个矩形可以与许多其他矩形进行几何交互，因此我们必须避免按单位面积进行模拟，而是对矩形边界引起的结构进行推理。 

## 方法

 直接暴力方法离散化所有 x 和 y 坐标，形成最多$2n \times 2n$坐标压缩后的单元格。 然后将每个矩形绘制在它覆盖的所有单元格上，并按顺序进行处理。 这是正确的，因为它完全模拟了覆盖过程。 然而，每个矩形最多可以覆盖$O(n^2)$细胞在最坏的情况下，并这样做$n$矩形导致$O(n^3)$的行为，远远超出了人的极限$n = 4000$。 

关键的观察是，我们永远不需要每个区域的完整几何形状，只需要每个基本单元中最顶部矩形的标识。 我们可以反转视角，而不是迭代每个单元格的每个矩形：扫描 x 间隔并维护活动矩形，并在每个 x 条带内扫描 y 间隔以确定当前哪个矩形位于顶部。 

这将 2D 可见性问题转换为 y 上重复的 1D“最顶层间隔”查询，其中当我们沿着 x 移动时矩形进入和退出。 事件的结构是稀疏的，因为矩形边界是活动集发生变化的唯一地方。 

我们最终维护了一组按插入时间（z 索引）排序的动态活动矩形，并且对于连续 x 坐标之间的每个垂直板，我们查询每个 y 间隔的最上面的矩形。 这可以通过支持插入、删除和检索最大 z 索引的平衡结构来完成。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力网格绘画|$O(n^3)$|$O(n^2)$| 太慢了|
 | 具有活动集的 2D 扫描线 |$O(n^2 \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将每个矩形视为 x 轴和 y 轴上的事件，并将问题简化为扫描压缩坐标。 

1.从矩形左右边缘收集所有唯一的x坐标，对它们进行排序，并使用它们来定义垂直条带。 这确保了几何形状恒定的每个 x 区域都被处理一次。 
2. 同样，从顶部和底部边缘收集所有唯一的 y 坐标，对它们进行排序，并使用它们来定义水平条带。 这确保了在每个 x 条带内，y 结构也是分段常数。 
3. 对于每个矩形，将其转换为两个事件：一个事件在其左边界变为活动状态，另一个事件在其右边界变为非活动状态。 这使我们能够维护当前覆盖垂直条带的一组动态矩形。 
4. 从左到右扫描 x 间隔。 在每个 x 条带上，维护一个由 z 索引（插入顺序）作为键控的数据结构，其中包含当前覆盖 x 中该条带的所有矩形。 关键思想是在固定的 x 条带内，潜在可见矩形的集合是固定的。 
5. 对于每个 x 条带，从下到上扫描 y 间隔。 在每个 y 间隔，我们确定哪个具有最高 z 索引的活动矩形覆盖此 y 范围。 该矩形是相应单元格块的可见颜色。 
6. 每当我们在任何 x-y 块中识别出可见矩形时，我们都会将其颜色记录在布尔数组或集合中。 最后，我们计算有多少种颜色被记录为可见。 

核心难点是有效维护“最顶层的活动矩形”。 这是通过支持插入、删除和检索最大 z 索引的结构来处理的，例如具有延迟删除功能的堆或按 z 顺序键控的平衡树。 

### 为什么它有效

 该算法依赖于以下不变量：在由连续 x 坐标和连续 y 坐标定义的任何单元内，覆盖该单元的矩形集是恒定的。 由于矩形仅在边界坐标处开始或停止，因此这些原子区域内不会发生任何变化。 因此，仅检查这些区域就可以保证正确性。 每个区域内 z 顺序最上面的矩形唯一地确定了该区域的可见颜色，并且每种可见颜色必须在至少一个区域中显示为最上面的矩形，因此不会遗漏任何颜色。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq
from collections import defaultdict

def solve():
    n = int(input())
    rects = []
    xs = set()
    ys = set()

    for i in range(n):
        x1, y1, x2, y2, c = map(int, input().split())
        rects.append((x1, y1, x2, y2, c, i))
        xs.add(x1)
        xs.add(x2)
        ys.add(y1)
        ys.add(y2)

    xs = sorted(xs)
    ys = sorted(ys)

    x_id = {x:i for i, x in enumerate(xs)}
    y_id = {y:i for i, y in enumerate(ys)}

    x_events = [[] for _ in range(len(xs))]
    for x1, y1, x2, y2, c, i in rects:
        x_events[x_id[x1]].append(("add", x1, y1, x2, y2, c, i))
        x_events[x_id[x2]].append(("remove", x1, y1, x2, y2, c, i))

    active = set()
    seen_color = set()

    for xi in range(len(xs) - 1):
        for ev in x_events[xi]:
            typ, x1, y1, x2, y2, c, i = ev
            if typ == "add":
                active.add((i, y1, y2, c))
            else:
                active.discard((i, y1, y2, c))

        if not active:
            continue

        active_list = list(active)

        for yi in range(len(ys) - 1):
            y_low = ys[yi]
            y_high = ys[yi + 1]

            best = -1
            best_color = None

            for idx, y1, y2, c in active_list:
                if y1 > y_low and y2 < y_high:
                    continue
                if not (y2 <= y_low or y1 >= y_high):
                    if idx > best:
                        best = idx
                        best_color = c

            if best_color is not None:
                seen_color.add(best_color)

    print(len(seen_color))

if __name__ == "__main__":
    solve()
```该实现压缩坐标，以便每个相关边界变成离散索引。 x 切片上的扫描使用插入和删除事件更新活动矩形集。 

在每个 x 条带内，我们扫描 y 间隔并检查哪些矩形覆盖该区域，选择索引最高的矩形作为可见层。 尽管这里是以一种简单的方式实现的，但概念模型与扫描线的思想相匹配：每个基本区域的最上面的活动矩形都会被检查一次。 

一个常见的微妙之处是矩形覆盖检查必须尊重重叠，而不仅仅是完全包含。 相交的条件必须确保矩形跨越当前 y 单元，否则应忽略。

 ## 边缘情况

 一种边缘情况是矩形仅在边界处接触而没有重叠区域。 例如，一个以 x = 3 结束的矩形，另一个以 x = 3 开始的矩形。这些不应相互干扰，因为它们共享零面积。 扫描线自然地处理这个问题，因为压缩间隔是在不同坐标之间定义的，因此在精确的边界重叠处不存在单元。 

另一种情况是一个大矩形完全覆盖所有其他矩形，后面是分散的小矩形。 该算法仍然记录所有颜色，因为每个小矩形将主导其区域中的至少一个压缩单元。 

最后一个微妙的情况是相同的 z 索引排序行为。 由于矩形是按插入顺序处理的，因此不会出现联系，并且最高索引始终是明确定义的。
