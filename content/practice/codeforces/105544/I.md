 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 每个测试用例都描述一个无向简单图，但不是通常的边列表形式。 相反，该图被间接指定为三角形的集合。 每条线给出三个不同的顶点，形成一个三角形，实际的图形是通过将任何这些三角形中出现的所有边的并集获得的。 

任务是确定该图是否包含长度为 5 的简单循环。 如果存在这样的循环，我们必须按循环顺序输出形成它的任意五个顶点。 否则我们输出-1。 

输入的结构很重要：我们不能随意假设任意稀疏或密集图，而是假设三角形并集的图。 这已经意味着高度的局部聚类：每条边至少属于一个三角形，并且许多边往往参与多个三角形。 

约束足够大，以至于不可能进行简单的循环枚举。 对于多达约 10,000 个顶点和 n^{1.5} 个三角形的数量级，诱导边的数量可以达到数百万条。 任何尝试枚举所有简单 5 元组顶点或所有长度为 4 的路径的方法都会因组合爆炸而立即失败。 即使直接迭代节点对之间的所有长度为 3 的路径也太慢了。 

微妙的边缘情况来自三角形的重复结构。 粗心的实现可能会假设每个三角形都是独立的，并将每个三角形视为一个单独的集团，从而丢失了跨越三角形边界的循环。 例如，三角形 (1,2,3)、(3,4,5)、(5,1,6)、(6,7,2)、(2,4,7) 可以共同形成 5 圈，该 5 圈在任何单个三角形内都不可见。 任何解决方案都必须在联合图上运行，而不是在单个三角形上运行。 

另一个陷阱是假设可以通过独立检查小深度的邻域来检测 5 周期。 5 周期可以轻松地穿过多个三角形，因此对单个三角形甚至成对三角形的局部检查是不够的。 

## 方法

 直接的暴力策略是构建图的完整邻接列表，然后尝试不同顶点的每个有序五元组，检查它们是否形成循环。 这在概念上很简单：选择五个节点并验证五个所需边的存在。 正确性是显而易见的，但是选择的数量是n^5的数量级，这对于n达到10^4来说是完全不可行的。 

稍微结构化一点的强力方法是固定起始节点 u 并使用 DFS 尝试从 u 开始的所有长度为 4 的路径，检查它们是否返回到 u。 即使进行剪枝，这也会在密集区域中每个节点探索大约 O(deg^4) 的结构，这仍然会退化为数亿或数十亿次操作。 

关键的结构观察是我们不需要直接枚举循环。 当且仅当存在一条边 u-v 可以延伸到从 u 到 v 的长度为 3 且避免重复顶点的路径时，才存在 5 循环。 换句话说，5 周期 u-a-b-c-v-u 对应于边端点之间的长度为 3 的路径。 

这将问题从搜索循环转移到搜索相邻节点之间的约束长度为 3 的路径。 这仍然很重要，但现在我们可以利用集合交集和位集表示来加速可达性查询。 

图大小 n ≤ 10^4 强烈建议采用位集技术。 有了位集，邻接查询就变成了字级操作，并且可以快速计算交集。 我们还利用了这样一个事实：我们只需要找到一条见证人路径，而不需要计算所有见证人路径。

我们将邻接表示为位集，并使用它们来检测三步路径的潜在中间段。 然后，对于每个候选边 (u, v)，我们尝试找到一个可通过两步从 u 到达的中间顶点 b，同时连接到 v 的邻居。这减少了从路径到预先计算邻域的交集的搜索空间。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 枚举所有 5 元组 | O(n^5) | O(n^5) | O(1) | O(1) | 太慢了 |
 | DFS 全长 4 路径 | O(n·d^4) | O(n·d^4) | O(n) | 太慢了 |
 | 位集交集+边缘居中 | O(n^2 / w + m · n / w) 摊销 | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们希望将 5 循环条件转换为可以使用邻域交集检测的模式。 

1. 从所有三角形构建图的邻接结构。 对于每个三角形 (x, y, z)，添加边 x-y、y-z 和 z-x。 最后，我们得到完整的无向图。 
2. 将每个顶点的邻接存储为大小为 n 的位集。 这允许在 O(n / word_size) 中检查邻接和交集。 
3. 预先计算一个二级可达性结构 N2[u]，也是一个位集，表示从 u 最多两步可达的所有顶点。 我们通过取 u 的所有邻居的邻居的并集来计算它。 这捕获了从 u 开始的潜在长度为 3 的路径中所有可能的中间顶点。 
4. 迭代每条边 (u, v)。 任何使用该边的 5-cycle 都必须对应于路径 u → a → b → c → v。 
5. 对于固定边 (u, v)，尝试找到位于 N2[u] 中的顶点 b。 这确保存在从 u 到 b 的两步路径，即对于某个 a，u → a → b。 
6. 对于每个这样的候选 b，我们现在尝试将其扩展到 v。我们计算 N(b) 和 N(v) 的交集。 该交集中的任何顶点 c 都满足 b-c 和 c-v，给出潜在 5 循环的最后两条边。 
7. 如果找到这样的 c，我们通过记住一个中间节点 a 来重构整个循环，使得 u-a-b 成立。 我们在构建 N2 期间存储了一个见证父级，以便我们可以恢复实际路径而不仅仅是存在。 
8. 一旦找到有效序列 u-a-b-c-v，我们就输出循环 u、a、b、c、v 并终止该测试用例。 

### 为什么它有效

 每个简单的 5 环至少包含一条边 (u, v)。 固定这条边会将循环分割成其端点之间长度为三的路径。 我们对 N2[u] 的构造保证了这样一条路径的每个有效的第二个顶点都出现在其中。 最后的交集步骤强制将剩余两条边闭合到 v 中。由于所有步骤都准确地保留邻接约束，因此任何检测到的序列都必须是有效的简单 5 循环，并且在处理其第一条边时将发现任何有效的 5 循环。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())

        adj = [set() for _ in range(n)]

        for _ in range(m):
            x, y, z = map(int, input().split())
            x -= 1
            y -= 1
            z -= 1
            adj[x].add(y); adj[y].add(x)
            adj[y].add(z); adj[z].add(y)
            adj[z].add(x); adj[x].add(x)

        # fix accidental self-add if any (safe guard)
        for i in range(n):
            if i in adj[i]:
                adj[i].remove(i)

        # build bitsets
        bit = [0] * n
        for i in range(n):
            b = 0
            for j in adj[i]:
                b |= 1 << j
            bit[i] = b

        # precompute 2-hop neighborhoods (bitset union of neighbors)
        bit2 = [0] * n
        for i in range(n):
            b = 0
            for j in adj[i]:
                b |= bit[j]
            b &= ~(1 << i)
            bit2[i] = b

        found = False

        nodes = list(range(n))

        for u in range(n):
            if found:
                break
            for v in adj[u]:
                if u >= v:
                    continue

                # try to find b in N2[u]
                cand = bit2[u]
                # iterate bits of cand
                bmask = cand
                while bmask:
                    lb = bmask & -bmask
                    b = (lb.bit_length() - 1)
                    bmask -= lb

                    if b == u or b == v:
                        continue

                    # intersection N(b) & N(v)
                    inter = bit[b] & bit[v]
                    inter &= ~(1 << u)
                    inter &= ~(1 << v)

                    if inter:
                        lc = inter & -inter
                        c = (lc.bit_length() - 1)

                        # now find a such that u-a-b
                        for a in adj[u]:
                            if a != b and (bit[a] >> b) & 1:
                                print(u + 1, a + 1, b + 1, c + 1, v + 1)
                                found = True
                                break
                        if found:
                            break
                if found:
                    break

        if not found:
            print(-1)

if __name__ == "__main__":
    solve()
```该实现严重依赖位操作来压缩邻接检查。 最关键的细节是使用位掩码来表示邻域，这允许快速完成两跳扩展和交叉检查。 

重建步骤被有意推迟，直到找到候选中间段。 这避免了存储所有对的完整路径信息，这会太昂贵。 

选择循环顺序是为了首先修复边，然后仅探索中间顶点的结构化候选，这可以防止搜索空间爆炸。

 ## 边缘情况

 一种重要的边缘情况是多个三角形塌陷成高度密集的局部结构而不形成 5 环。 在这种情况下，位集交集会产生许多中间顶点的候选点，但没有一个可以完成返回起始边缘的一致路径，因此算法正确地耗尽所有候选点并返回 -1。 

当有效的 5 循环与许多三角形共享顶点时，会出现另一种边缘情况。 即使邻接集很大，位集交集仍然确保我们仍然只隔离那些同时满足路径结构两半的顶点，从而防止误报。 

最后一个微妙的情况是多个不同的 5 周期共享相同的边。 该算法将检测在候选中间顶点迭代期间遇到的第一个有效重建。 这已经足够了，因为任何有效的周期都是可以接受的。
