 ## 解决方案
 ## 问题理解

 我们得到了一组轴对齐的矩形板，放置在一个更大的矩形花园内。 每个板都有固定的垂直位置，这意味着它的底部和顶部 y 坐标是不可变的，但我们可以水平向左或向右移动板。 

有一个重要的相互作用规则：如果两个板在垂直投影上重叠，那么它们的左右顺序必须与初始配置保持一致。 换句话说，垂直重叠会产生一个约束，禁止交换它们的相对 x 顺序。 此外，如果两个垂直重叠的板在水平方向上靠近，我们必须在它们之间至少保持给定的最小间距。 板可以水平接触，但不能违反所需的间距。 

目标是水平重新定位所有板，保留垂直坐标并尊重顺序约束和间距约束，以便最小化配置的整体边界宽度。 由于整个花园的垂直跨度是固定的，最小化宽度直接最大化了未使用的面积。 

关键的困难在于约束不是简单地全局的。 除非两个板的垂直范围相交，否则它们可能不会直接交互，但通过重叠链的传递性仍然可以在许多板之间强制排序约束。 

根据约束，我们注意到每个测试用例最多有 100 个板，最多有 32 个测试用例。 这可以轻松实现 O(n^2) 或 O(n^2 log n) 解决方案。 所有对的任何三次方仍然可能很好，但没有必要。 对排序进行阶乘或指数探索是不可能的。 

当板形成多个重叠组并间接传播排序约束时，就会出现微妙的失败情况。 例如，板 A 与 B 重叠，B 与 C 重叠，但 A 不与 C 重叠。仅检查直接重叠并忽略传递约束的简单方法可能会错误地允许无效的重新排序。 

另一个问题是假设所有板都在全球范围内交互。 如果板垂直不相交，则它们不会施加顺序约束，并且将它们视为完全连接会导致不必要的限制和不正确的宽度最小化。 

## 方法

 如果我们忽略结构，我们可能会尝试贪婪地分配 x 位置，甚至暴力破解板的所有排列并模拟约束传播。 对于每个排序，我们将从左到右分配位置，确保满足间距约束，并计算结果宽度。 这是正确的，因为任何有效的配置都对应于与所有约束一致的某种总排序。 

然而，有n！ 可能的排列，甚至 n = 20 也已经使这变得不可行。 当n达到100时，这是完全不可能的。 

关键的观察结果是，仅当垂直投影重叠时才存在约束。 这将问题转化为图结构：每个板都是一个节点，如果两个节点的 y 间隔相交，则它们之间存在一条边。 每个连接的组件定义一组必须在内部排序约束下放置的板，而不同的组件在排序上不会相互约束。 

在一个连接的组件中，我们仍然需要尊重由重叠引起的偏序。 重要的是，如果两个板在 y 方向上重叠，它们的相对顺序从初始配置中就固定下来。 这给了我们定向约束：如果板 i 最初位于板 j 的左侧并且它们垂直重叠，则 i 必须保持在 j 的左侧。 

因此，每个连接的组件成为一个有向非循环约束系统。 组件内部的任务变为：找到与这些约束一致的线性排序，以最小化总跨度，其中边缘施加的最小间隙等于板宽度加间距。

这相当于将每个约束转换为 x 坐标上的差异约束后，在 DAG 中找到最长路径。 

我们将每个板转换为具有变量 x[i]（其左坐标）的节点。 对于 j 之前的每个约束 i，我们要求：

 x[j] ≥ x[i] + 宽度[i] + 间距

 这是一个差异约束图。 我们想要尊重所有约束的最小可行分配，这是此约束图中从源开始的最长路径。 

我们向所有节点添加一个边权重为 0 的超级源节点，然后放宽约束。 由于图是通过一致排序构造的 DAG，因此我们可以使用拓扑顺序或倒号的最短路径。 

最后，组件的答案由 x[i] + width[i] 的最大值确定。 最终答案是所有分量中的最大值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!·n) | O(n!·n) | O(n) | 太慢了 |
 | 图约束+最长路径| O(n^2) | O(n^2) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们以结构化的方式构建解决方案。 

1. 我们首先确定哪些板垂直相互作用。 如果两个板的 y 间隔重叠，则它们是连接的。 我们为此目的构建了一个无向图，因为重叠关系是对称的。 此步骤隔离了可以单独求解的独立组。 
2. 我们使用 DFS 或 BFS 计算连通分量。 每个组件都被独立处理，因为没有约束跨越组件。 
3. 在每个组件内部，我们导出定向排序约束。 对于组件中的每一对板 i 和 j，如果它们的垂直范围重叠并且 i 最初位于 j 的左侧，则我们添加一条有向边 i → j，其权重等于 width[i] 加上所需的间距。 这表明 j 必须放置在 i 右侧足够远的位置。 
4. 然后，我们计算满足所有约束的 x 坐标的可行分配。 我们将所有 x[i] 初始化为零并反复放松约束。 从概念上讲，对于每条边 i → j，我们将 x[j] 更新为至少 x[i] + w[i] + d。 这是 DAG 结构上的最长路径传播问题。 
5. 我们对每个组件中的约束图进行拓扑排序，并按该顺序处理节点，更新距离。 这保证了当我们处理一个节点时，所有传入的约束都已得到解决。 
6. 一旦确定了 x 坐标，我们就将组件的跨度计算为 x[i] + width[i] 的所有板的最大值。 由于锚定，最小 x 实际上为零，因此该跨度表示该组件可实现的最小宽度。 
7. 最终答案是组件跨度的总和，或者根据布局解释，组件可以独立并行放置的最大值。 由于组件之间不互相约束，因此它们可以并排排列，而不会影响彼此的内部约束，因此我们将它们的宽度相加。 

### 为什么它有效

 关键的不变量是每条边都对垂直重叠的板之间的间隔编码必要的下界。 任何有效的安排都必须满足所有这些不等式。 通过将这些约束转换为拓扑有序结构上的最长路径计算，我们确保每个节点的位置都是最紧密的，并且仍然尊重所有前辈。 因为约束仅沿着重叠链传播，并且因为每个组件都是独立处理的，所以不会忽略或重复计算任何约束。 这保证了可行性和最小总宽度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict, deque

def solve():
    t = int(input())
    for _ in range(t):
        n, d = map(int, input().split())
        rects = []
        for i in range(n):
            x1, y1, x2, y2 = map(int, input().split())
            rects.append((x1, y1, x2, y2))

        # build vertical overlap graph
        g = [[] for _ in range(n)]
        for i in range(n):
            x1i, y1i, x2i, y2i = rects[i]
            for j in range(i + 1, n):
                x1j, y1j, x2j, y2j = rects[j]
                if not (y2i <= y1j or y2j <= y1i):
                    g[i].append(j)
                    g[j].append(i)

        vis = [False] * n
        comp_id = [-1] * n
        comps = []

        for i in range(n):
            if not vis[i]:
                q = [i]
                vis[i] = True
                comp = []
                while q:
                    u = q.pop()
                    comp.append(u)
                    comp_id[u] = len(comps)
                    for v in g[u]:
                        if not vis[v]:
                            vis[v] = True
                            q.append(v)
                comps.append(comp)

        ans = 0

        for comp in comps:
            idx = {v: k for k, v in enumerate(comp)}
            m = len(comp)

            # build directed constraints
            dag = [[] for _ in range(m)]
            indeg = [0] * m

            for i in comp:
                for j in comp:
                    if i == j:
                        continue
                    x1i, y1i, x2i, y2i = rects[i]
                    x1j, y1j, x2j, y2j = rects[j]

                    if not (y2i <= y1j or y2j <= y1i):
                        if x1i < x1j:
                            u = idx[i]
                            v = idx[j]
                            dag[u].append((v, x2i - x1i + d))
                            indeg[v] += 1

            # topological DP
            q = deque([i for i in range(m) if indeg[i] == 0])
            dist = [0] * m

            while q:
                u = q.popleft()
                for v, w in dag[u]:
                    if dist[v] < dist[u] + w:
                        dist[v] = dist[u] + w
                    indeg[v] -= 1
                    if indeg[v] == 0:
                        q.append(v)

            best = 0
            for i in range(m):
                orig = comp[i]
                x1, y1, x2, y2 = rects[orig]
                best = max(best, dist[i] + (x2 - x1))

            ans += best

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先构建到单独的独立组的垂直重叠连接。 然后将每个组转换为有向约束系统，其中边缘对强制水平分隔进行编码。 拓扑 DP 通过传播所需的最大偏移量来计算最紧密的可行布局。 最后，每个组件贡献其可实现的最小宽度。 

一个微妙的点是，仅针对重叠的垂直间隔创建约束。 这可以防止不必要的边并确保图保持足够稀疏以进行 O(n^2) 处理。 

## 工作示例

 考虑三个板的小型配置，形成垂直重叠链。 板 1 与板 2 重叠，板 2 与板 3 重叠，但板 1 不与板 3 重叠。这将创建传递约束链。 

| 步骤| 处理节点 | 距离阵列| 行动|
 | --- | --- | --- | --- |
 | 1 | 1 | [0, 0, 0] | 开始|
 | 2 | 1 → 2 | [0，w1+d，0] | 强制间距 |
 | 3 | 2 → 3 | [0，w1+d，w1+d+w2+d] | 传播链|

 这显示了即使没有直接重叠，间接约束也是如何累积的。 

现在考虑两个独立的垂直组。 一组的石板集中在花园的下部，另一组的石板集中在花园的上部。 它们从不相互作用，因此它们的距离是独立计算的，并且它们的贡献相加。 

| 步骤| 组件| 结果宽度 |
 | --- | --- | --- |
 | 1 | 下组| W1 |
 | 2 | 上组| W2 |
 | 3 | 总计 | W1 + W2 |

 这表明组件之间的独立性至关重要； 合并它们会错误地引入人为约束。

 ## 边缘情况

 当板仅通过传递链重叠时，就会出现关键的边缘情况。 例如，板 A 与 B 重叠，B 与 C 重叠，但 A 和 C 不重叠。 该算法仍然正确地强制通过约束图进行 A → B → C 传播，确保正确的间距累积。 

另一种边缘情况是没有板垂直重叠。 在这种情况下，每个板都是其自己的组件，不会创建任何边缘，并且所有距离都保持为零。 计算出的答案正确地减少到原始宽度的总和，这意味着不可能进行压缩。 

最后的边缘情况是密集重叠，其中所有板都垂直重叠。 这会生成完整的约束图，但拓扑 DP 仍然有效，因为约束与初始排序一致并且没有引入循环。
