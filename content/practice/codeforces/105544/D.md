 ## 解决方案
 ## 问题理解

 我们得到了一个代表飞机座位布局的网格。 每个单元格要么是空的，要么包含病毒源。 每个病毒座位以两种不同的方式影响附近的座位：直接相邻的座位（上、下、左、右）被分配隔离持续时间$d_1$，而对角相邻座位的隔离时间为$d_2$。 

一个健康席位可能会受到多个病毒席位的影响，但该席位的最终隔离时间不会累加。 相反，我们采用任何相邻病毒造成的最长隔离持续时间。 

任务是计算每个健康座位由所有附近病毒座位引起的隔离持续时间，并输出转换后的网格，其中每个健康座位被其计算的天数替换，而病毒座位保持不变。 

网格大小最多为100×100，最多有1000个测试用例。 这使得对所有单元格的每次测试线性扫描变得可行，但任何比$O(nm)$每次测试总的来说会太慢。 

当一个座位受到不同病毒细胞的直接和对角影响时，就会出现微妙的边缘情况。 在这种情况下，正确的值是所有贡献中的最大值，而不是总和或最后写入的值。 例如，如果有一个座位$d_1 = 7$来自直接邻居和$d_2 = 3$从对角邻居来看，无论处理顺序如何，答案都必须是 7。 

另一个边缘情况是边界处理。 边缘或角落上的座位具有较少的邻居，因此没有边界检查的朴素邻居迭代将访问无效的网格位置并产生不正确的结果或运行时错误。 

## 方法

 思考这个问题的一个直接方法是将每个病毒细胞视为影响源并将其影响传播到邻近细胞。 对于每个病毒细胞，我们检查其周围的八个位置。 对于每个有效邻居，我们根据方向是基数方向还是对角线方向分配候选隔离值。 然后我们用迄今为止看到的最大值更新邻居的结果。 

这种强力解释已经很好地匹配了约束。 每个病毒细胞最多贡献8次更新，最多有$nm$细胞。 因此，每个测试用例的总工作量受一个小的常数因子倍的限制$nm$，这足够有效。 

不需要最短路径、BFS 或多源传播，因为影响不会超出距离一。 关键的观察结果是，问题纯粹是局部的：每个细胞的价值仅取决于其邻近区域，而不取决于感染链。 

因此，最佳解决方案与暴力方法相同，但干净地组织为带有邻居检查的单个网格遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力邻居模拟 | 每次测试 O(nm) | O(1) 额外 | 已接受 |
 | 最佳网格扫描| 每次测试 O(nm) | O(1) 额外 | 已接受 |

 ## 算法演练

 1. 将答案网格初始化为输入的副本，但将每个健康单元格替换为 0 作为起始值。 为了输出的一致性，病毒细胞仍标记为病毒。 
2. 定义四个直接相邻方向：上、下、左、右和四个对角方向。 
3. 迭代网格中的每个单元格。 当遇到病毒细胞时，我们将其视为影响源。 
4. 对于每个病毒细胞，检查其四个直接邻居。 对于网格内的每个有效邻居，如果该邻居是健康席位，则将其值至少更新为$d_1$。 这代表直接邻近影响。 
5. 然后检查其四个对角邻居。 对于每个健康的有效对角邻居，将其值至少更新为$d_2$。 
6.处理完所有病毒细胞后，再次遍历网格，将任意健康细胞的数值替换为字符串形式输出，同时保持病毒细胞不变。 

每个更新步骤都是单元格值的局部松弛。 我们始终保留最大值，因为多个病毒源可能会影响同一个座位。 

### 为什么它有效

 每个健康的座位只能受到其紧邻的 8 细胞附近的病毒细胞的影响。 每种病毒独立贡献$d_1$或者$d_2$，并且影响不会进一步传播。 由于我们对每个病毒-邻近对的所有此类贡献进行一次明确评估，因此考虑了所有可能的感染源。 取最大值可确保重叠影响下的正确性，而不会重复计算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for tc in range(1, t + 1):
        n, m, d1, d2 = map(int, input().split())
        grid = [list(input().strip()) for _ in range(n)]
        
        ans = [[0] * m for _ in range(n)]
        
        dirs4 = [(-1,0),(1,0),(0,-1),(0,1)]
        dirs_diag = [(-1,-1),(-1,1),(1,-1),(1,1)]
        
        # propagate from each virus
        for i in range(n):
            for j in range(m):
                if grid[i][j] == 'V':
                    for dx, dy in dirs4:
                        ni, nj = i + dx, j + dy
                        if 0 <= ni < n and 0 <= nj < m and grid[ni][nj] == '.':
                            ans[ni][nj] = max(ans[ni][nj], d1)
                    for dx, dy in dirs_diag:
                        ni, nj = i + dx, j + dy
                        if 0 <= ni < n and 0 <= nj < m and grid[ni][nj] == '.':
                            ans[ni][nj] = max(ans[ni][nj], d2)
        
        # build output
        out = []
        out.append(f"Airplane #{tc}:")
        for i in range(n):
            row = []
            for j in range(m):
                if grid[i][j] == 'V':
                    row.append('V')
                else:
                    row.append(str(ans[i][j]))
            out.append(''.join(row))
        
        print('\n'.join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先为隔离持续时间构建一个数字网格。 然后每个病毒细胞直接更新其邻居。 使用`max`确保正确解决重叠影响，而无需记录哪些病毒造成了什么影响。 

边界检查至关重要，因为邻居可能会落在网格之外。 两个独立的方向阵列清晰地分离了直接影响和对角影响，避免了两个距离规则之间的混淆。 

最后，输出重建将数字网格与原始结构合并，完全按照需要保留病毒细胞。 

## 工作示例

 ### 示例 1

 输入：```
1
3 3 5 2
V..
...
..V
```我们将所有非病毒单元初始化为 0。现在在 (0,0) 处处理病毒。 它影响 (0,1) 和 (1,0) 为 5，影响 (1,1) 为 2。 

然后在(2,2)处处理病毒。 它再次影响 (2,1)、(1,2) 和 5，并再次影响 (1,1) 和 2。 

| 病毒 | 直接更新 (d1=5) | 对角线更新 (d2=2) |
 | --- | --- | --- |
 | (0,0) | (0,0) | (0,1)=5, (1,0)=5 | (0,1)=5, (1,0)=5 | (1,1)=2 |
 | (2,2) | (2,1)=5, (1,2)=5 | (1,1)=2 |

 最终网格：```
V55
552
25V
```这证实了 (1,1) 处的重叠影响正确解析为 max(2,2) = 2。

 ## 边缘情况

 角病毒提供了最干净的边界处理测试。 考虑一个 2 x 2 的网格，左上角有一个病毒。 仅存在三个邻居，因此算法必须忽略越界索引。 迭代检查确保了这一点的安全。 

第二个微妙的情况是多种病毒从不同方向影响同一个座位。 因为我们总是申请`max`，处理顺序并不重要。 例如，座位可能首先接收对角线值 3，然后接收直接值 7。存储的结果变为 7 并保持稳定。 

最后一种情况是完全没有病毒的网格。 每个单元格都应保持为 0，并且由于不会触发任何更新，因此初始化已与正确的输出匹配，无需任何特殊处理。
