 ## 解决方案
 ## 问题理解

 我们得到非常大的十进制数字，每个数字代表一个钞票序列号。 如果一个数字能被 13 整除，则该数字被认为是有效的。我们不是直接除以整个数字，而是应用特定的转换：将数字从右侧开始分成三组，将每组视为一个整数，然后从最右边的组开始使用交替减法和加法将这些组组合起来。 该交替和的最终标量结果以与原始数字相同的方式确定能否被 13 整除。 

对于每个测试用例，我们必须输出两件事。 首先，计算 3 位数字块上交替和的绝对值。 其次，这个值是否能被13整除，打印为YES或NO。 

关键限制是每个数字的大小，最多 1000 位数字，最多 1000 个测试用例。 这立即排除了将数字解析为整数类型或直接对完整数字执行算术的可能性。 即使是大的整数库也将是不必要的开销，因为该结构是纯粹基于数字的并且是块本地的。 

一个幼稚的错误来自于尝试将数字视为单个整数并重复除以 10 或使用标准整数类型对其进行解析。 例如，1000 位数字无法适合任何内置整数类型，甚至字符串到整数的转换也会溢出。 

另一个微妙的边缘情况是处理少于 3 位数字的前导组。 例如，输入`12345`应分组为`12`和`345`， 不是`012`和`345`。 未能从右侧对齐分组会破坏整个转换。 

## 方法

 暴力方法将显式地从输入字符串重建完整整数，然后应用整除性检查。 这会立即失败，因为在标准固定宽度类型中构造 1000 位整数是不可能的，甚至任意精度算术也是不必要的并且比所需的要慢。 如果我们想象一个大整数实现，加法和除法每次操作的成本为 O(d)，其中 d 是位数，导致 O(t·d) 或更糟糕的重复工作。 

问题的实际结构绕过了所有这些。 该数字只能以三位数字的块形式访问，并且每个块在自定义位置系统中都被独立视为以 1000 为基数的数字。 交替的减法和加法只是这些以 1000 为基数的块的加权和。 这意味着我们永远不需要全局解释完整的数字，只需在本地提取块并将它们组合起来。 

因此，解决方案变成从右到左线性扫描字符串，将数字分组为大小最多为 3 的整数，并累加交替和。 这将每个测试用例的问题减少到 O(n)。

 | --- | --- | --- | --- |
 | 蛮力（全整数算术）| O(n²) 或在实践中不可能 | O(n) | 太慢/不切实际|
 | 最佳块处理 | O(n) | O(1) 额外（除了输入） | 已接受 |

 ## 算法演练

 我们将每个数字作为字符串处理，并从右到左迭代。 

1. 从字符串的最后一位数字开始并向左移动，将数字分成最多三个的块。 每个块都被解释为十进制整数。 此分组保留了预期的以 1000 为基数的数字分解。 
2. 维护一个运行乘数，最右边的组从 -1 开始，因为第一个运算是减法。 处理完每个组后，翻转符号，以便添加下一组，然后减去，依此类推。 这直接对交替操作进行编码。 
3. 对于每个组，将子字符串转换为整数，并将其乘以当前符号，然后将其添加到运行总计中。 这会增量地构建交替总和，而不存储所有组。 
4.处理完所有组后，取结果的绝对值。 该问题明确需要大小，无论符号如何。 
5. 使用简单的模运算检查最终的绝对结果是否能被 13 整除，并相应输出 YES 或 NO。 

### 为什么它有效

 每个 3 位数字块的行为类似于以 1000 为基数的数字。交替的减法和加法定义了这些以 1000 为基数的数字的线性组合。 由于每个块都是独立的并且块之间没有进位传播，因此该变换完全等同于在块表示上评估固定线性形式。 这确保计算值与语句中给出的定义匹配并保留整除性信息。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    s = input().strip()
    
    total = 0
    sign = -1
    
    i = len(s)
    while i > 0:
        j = max(0, i - 3)
        block = int(s[j:i])
        total += sign * block
        sign *= -1
        i = j
    
    ans = abs(total)
    print(ans, "YES" if ans % 13 == 0 else "NO")
```代码独立处理每个测试用例。 主循环将字符串从右到左切成三个字符的块。 每个切片都会安全地转换为整数，即使对于前导部分块也是如此。 

这`sign`变量对交替减法-加法模式进行编码，无需通过索引显式跟踪组奇偶校验。 这可以避免从左侧计算组时经常出现的差一错误。

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    
    t = int(input())
    out = []
    for _ in range(t):
        s = input().strip()
        total = 0
        sign = -1
        
        i = len(s)
        while i > 0:
            j = max(0, i - 3)
            total += sign * int(s[j:i])
            sign *= -1
            i = j
        
        ans = abs(total)
        out.append(f"{ans} {'YES' if ans % 13 == 0 else 'NO'}")
    
    return "\n".join(out)

 ## 边缘情况

 一种边缘情况是长度不是三的倍数的数字。 例如，`12345`变成块`12`和`345`。 该算法通过从右侧切片来正确处理此问题：首先`345`， 然后`12`。 符号交替仍然一致应用，因此不需要特殊的大小写。 

另一种边缘情况是单位数或单块数。 用于输入`7`，该算法产生一个块`7`有标志`-1`，产生`-7`，绝对值变为`7`，正确检查了整除性。 

最后的边缘情况是通过分组隐式创建的零填充块。 由于切片永远不会引入填充并且只采用实际的子字符串，因此值如`001`被正确解释为整数 1，无需额外标准化即可保持正确性。
