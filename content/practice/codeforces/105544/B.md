 ## 解决方案
 ## 问题理解

 该问题要求我们评估以混合十进制形式给出的实数，其中小数扩展的一部分不重复，而另一部分则永远重复。 我们不需要使用无限小数，而是需要将其转换为最低限度的精确分数。 

每个测试用例给出两个字符串。 第一个字符串表示小数点后不重复的有限前缀。 第二个字符串表示无限重复的数字块。 例如，如果输入是`s1 = "10"`和`s2 = "12"`，该数字被解释为 0.10 后跟 121212...，表示 0.1012121212...

 输出必须是一对整数 n 和 d，使得值等于 n/d 并且减少分数，以便 n 和 d 不共享公约数。 

就结构而言，约束非常小：每个测试用例两个字符串的总位数最多为 10，并且最多有 40 个测试用例。 这立即排除了对大整数库或渐近优化的任何需要。 即使是直接代数构造和最大公约数约简也足够了。 

尽管有一些小的限制，但主要的微妙之处在于正确地将混合循环小数转换为分数。 将重复块视为有限或忽略非重复部分和重复部分之间的对齐的天真的尝试将会失败。 

典型的错误方法是假设 0.s1s2s2s2... 简单地等于整数(s1s2) / 10 的某个幂，这是错误的，因为重复部分在非重复前缀之后开始。 

处理 s1 为空或 s2 为空等情况时会出现另一个错误。 如果 s2 为空，则该数字只是一个终止小数。 如果s1为空，则从小数点后立即开始重复。 

## 方法

 蛮力的想法是模拟十进制展开，生成重复序列的许多数字，并将其近似为分数。 然而，将长重复小数转换为精确分数需要模式检测或具有理性重构的任意精度算术。 尽管这里的限制很小，但这种方法在概念上是不必要的并且容易出错。 

关键的观察是这两个部分都是有限字符串，因此我们可以使用代数精确地表示数字。 

设 x 为值：

 x = 0.s1，然后无限重复 s2。 

设a为s1和s2连接而成的整数，b为s1单独形成的整数。 令 k = |s1| 且 m = |s2|。 

我们可以表达：

 0.s1s2s2s2... = b / 10^k + (0.s2) / 10^k + (0.s2) / 10^(k+m) + ...

 重复的尾巴形成一个几何级数：

 (0.s2) / 10^k * (1 + 10^{-m} + 10^{-2m} + ...)

 这简化为：

 (0.s2) / 10^k * 1 / (1 - 10^{-m})

 将分子和分母乘以 10 的幂可以消除小数并产生干净的整数分数。 最终结果可以表示为：

 设 A = int(s1 + s2)、B = int(s1)、P = 10^|s2|。 

然后：

 x = (A - B) / (10^|s1| * (P - 1)) + B / 10^|s1|

 组合术语可得出：

 x = (A - B + B*(P - 1)) / (10^|s1| * (P - 1))

 这会产生精确的整数分子和分母。 之后，我们使用 gcd 减少分数。 

这样做的原因是重复和非重复部分都是有限字符串，因此无限小数变成具有整数比的几何级数，允许封闭式求和。

 ## 算法演练

 我们仅使用整数算术来构造分数。

1. 读取字符串 s1 和 s2，并计算它们的长度 k 和 m。 这些定义了小数点移动的距离和重复周期的长度。 
2. 将 s1、s2 和串联 s1 + s2 转换为整数。 这些表示十进制前缀的缩放版本，没有浮点错误。 
3. 计算 10 的幂：pow_k = 10^k 且 pow_m = 10^m。 这些代表小数点定位和重复标准化的缩放因子。 
4. 将分子构建为 (int(s1 + s2) - int(s1)) + int(s1) * (pow_m - 1)。 该表达式以将所有数字对齐到相同比例的方式分离重复部分和非重复部分的贡献。 
5. 将分母设为 pow_k * (pow_m - 1)。 这捕获了初始小数移位和无限重复标准化因子。 
6. 将分子和分母减去它们的 gcd，使分数成为最低项。 
7. 输出简化的分子和分母。 

重要的结构思想是，循环小数被转换为有限的几何级数，而我们实际上从未将其扩展到代数形式之外。 

### 为什么它有效

 s1和s2表示的数总是可以分解为有限的前缀加上无限的几何尾。 几何尾部的比率为 10^{-m}，因此其和可以精确地表示为分母为 (1 - 10^{-m}) 的有理数。 由于每一步都是在除法之前使用整数执行的，因此不会发生精度损失。 gcd 缩减确保了表示的唯一性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def solve():
    T = int(input())
    for _ in range(T):
        a, b = map(int, input().split())
        s1 = input().strip()
        s2 = input().strip()

        k = len(s1)
        m = len(s2)

        pow_k = 10 ** k
        pow_m = 10 ** m

        if m == 0:
            # purely terminating decimal
            num = int(s1)
            den = pow_k
        else:
            a_int = int(s1 + s2) if s1 + s2 else 0
            b_int = int(s1) if s1 else 0

            num = (a_int - b_int) + b_int * (pow_m - 1)
            den = pow_k * (pow_m - 1)

        g = gcd(num, den)
        num //= g
        den //= g

        print(num, den)

if __name__ == "__main__":
    solve()
```实现直接遵循代数分解。 唯一棘手的部分是将字符串连接安全地转换为整数，这很好，因为总长度最多为 10 位数字。 

我们明确处理 s2 为空时的情况，因为几何级数公式会退化，我们应该回退到标准的终止小数分数。 

gcd 步骤是必要的，因为同一小数的不同分解可以产生非约化分数。

 ## 边缘情况

 一个微妙的边缘情况是重复部分实际上为零，例如 s2 =“0”。 在这种情况下，几何级数公式仍然会产生分母 (10^m - 1)，即变为 9，但分子会折叠为在 gcd 归约后正确简化的倍数。 例如，0.10\overline{0} 只是 0.1，代数生成 10/100，从而减少到 1/10。 

另一种情况是当 s1 为空时。 那么这个数字就纯粹是从小数点后第一位开始重复的。 该公式仍然有效，因为 int("") 被视为 0，并且表达式完全简化为标准重复分数公式。 

最后，当 s1 和 s2 都是最小长度（例如单位数字字符串）时，所有算术都保持稳定，因为最大中间值以 10^10 为界，完全在整数限制内。
