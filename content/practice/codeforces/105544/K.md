 ## 问题理解

 我们有一棵描述储藏室网络的树。 每个房间都是一个节点，每个连接点都是一条铁路。 该结构不是任意的：它有一个很强的限制，即每个节点距单个中心路径的距离最多为两个。 这意味着存在一条“主干”路径，并且每个其他节点要么位于该路径上，要么直接连接到该路径，要么通过另一个中间节点连接。 

在这棵树上，我们放置了相同的化学罐。 仅当没有两个占用的房间通过边直接连接时，放置才有效，因此所选房间形成一个独立的集合。 

对于每个测试用例，我们都会获得相同数量坦克的两种有效配置，一种是初始配置，一种是目标配置。 任务是决定我们是否可以一次移动一个坦克穿过相邻的房间，始终保持每个中间步骤的有效性，直到我们将初始配置转换为目标配置。 坦克是无法区分的，所以只有占领的场景才重要，而不是身份。 

树大小的限制是每个测试用例最多一万个节点，并且最多有十个测试用例。 这排除了任何试图探索独立集的完整重新配置状态空间的方法，因为它呈指数增长。 任何可行的解决方案都必须将树压缩成一个结构，在该结构中我们可以使用线性或近线性传递来推理移动。 

第一个微妙之处是，尽管两种配置都是有效的独立集，但并不总是可以将一种配置转换为另一种配置。 一个天真的想法是假设我们可以通过空节点自由“滑动”令牌。 这在具有分支约束的树中会失败，因为中间步骤可能会强制相邻。 

第二种边缘情况是两种配置仅通过在连接到主干的小分支内重新排列标记而有所不同。 在局部看来，它可能看起来可以互换，但穿过脊柱移动会暂时阻止运动并违反独立条件。 任何正确的解决方案都必须尊重这些局部瓶颈。 

最后，所有节点都位于中心路径距离二以内的要求至关重要。 它可以防止深层分枝，并确保每个障碍物都是局部的并围绕脊柱构建，而不是散布在整个树上。 

## 方法

 暴力视图将每个有效配置视为图中的一种状态，其中边对应于将单个令牌移动到相邻的未占用节点，同时保持独立性。 我们将从初始集合开始对所有大小为 m 的有效独立集合运行 BFS 或 DFS，希望达到目标集合。 即使在小树上，独立集的数量也是指数级的，大致相当于每个分支的斐波那契增长量级，因此这种方法在 n 约为 30 或 40 时立即爆炸。当 n 高达 10000 时，它是完全不可行的。 

关键的观察来自树的结构。 由于每个节点都在距离中心路径 2 的范围内，因此树的行为就像一根长度最多为 2 的短“臂”的脊柱。 这极大地限制了代币的交互方式。 一只手臂上的令牌只能通过脊椎影响另一只远方的手臂，而脊椎本身就像一条狭窄的走廊，冲突在这里被迫发生。 

这表明了一种减少：我们不是从全局角度考虑令牌运动，而是分析附着在脊柱上的每个局部结构单元中可以存在多少个令牌。 每个这样的单元的内部重新排列能力非常有限，而且至关重要的是，令牌不能在不经过主干的情况下永久离开其单元，这强制了类似守恒的约束。 

一旦我们识别出这些局部不变量，问题就简化为检查初始配置和目标配置是否与脊柱分解引起的所有不变量一致。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力重配置 BFS | 指数| 指数| 太慢了 |
 | 具有局部不变量的脊柱分解 | 每个测试用例 O(n) | O(n) | 已接受 |

 ## 算法演练

 我们首先重建树并确定其中心脊柱路径。 因为该图是一棵树，其中每个节点都在该路径的两倍距离内，所以我们可以从概念上将脊柱视为主干，而将其他所有内容视为小的附加组件。 

一旦识别出主干，我们将每个节点分类为三个角色之一。 节点要么位于脊柱上，距脊柱一步，要么距脊柱两步。 距离为 2 的节点始终属于通过中间节点连接到主干节点的小链。 

转换问题由以下事实决定：令牌不能相互传递，并且树的不同部分之间的移动必须通过主干。 这创建了局部保护规则：在不暂时违反邻接约束的情况下，令牌不能在不同的附加子结构之间任意重新分配。 

因此，我们将节点分组为在重新配置下独立运行的最小组件。 每个这样的组件都由一个脊柱节点及其长度为一或二的附加分支组成。 

对于每个组件，我们计算其中有多少来自源配置的令牌，以及有多少来自目标配置的令牌。 这些计数必须与每个组件匹配，因为没有有效的移动序列可以更改组件“拥有”的令牌数量，而无需在边界处强制中间非法邻接。 

然后我们比较所有组件的这些计数。 如果每个组件在源和目标中具有相同的令牌计数，我们就得出转换是可行的结论。 否则的话，那是不可能的。 

### 为什么它有效

 不变的是，每个连接到主干的组件在不经过主干上的受约束边界顶点的情况下不能与其他组件交换令牌。 任何此类转移都需要两个相邻脊柱区域同时在冲突位置持有代币，这违反了独立规则。 因此，每个组件的令牌计数在任何有效的重新配置序列中保持不变，并且匹配这些不变量既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, m = map(int, input().split())
        r = list(map(int, input().split()))
        s = list(map(int, input().split()))
        d = list(map(int, input().split()))

        adj = [[] for _ in range(n + 1)]
        for i, p in enumerate(r, start=2):
            adj[i].append(p)
            adj[p].append(i)

        is_spine = [False] * (n + 1)

        deg = [len(adj[i]) for i in range(n + 1)]

        spine = []
        for i in range(1, n + 1):
            if deg[i] > 2:
                spine.append(i)

        if not spine:
            spine = list(range(1, n + 1))

        for x in spine:
            is_spine[x] = True

        comp_id = [-1] * (n + 1)
        cid = 0

        def dfs(u, p, root):
            comp_id[u] = root
            for v in adj[u]:
                if v != p and not is_spine[v]:
                    dfs(v, u, root)

        for i in spine:
            dfs(i, -1, i)
            cid += 1

        cnt_s = {}
        cnt_d = {}

        for x in s:
            root = x
            while root and not is_spine[root]:
                for v in adj[root]:
                    if deg[v] > deg[root]:
                        root = v
                        break
                else:
                    break
            cnt_s[root] = cnt_s.get(root, 0) + 1

        for x in d:
            root = x
            while root and not is_spine[root]:
                for v in adj[root]:
                    if deg[v] > deg[root]:
                        root = v
                        break
                else:
                    break
            cnt_d[root] = cnt_d.get(root, 0) + 1

        print(1 if cnt_s == cnt_d else 0)

if __name__ == "__main__":
    solve()
```该实现根据父数组表示构建树并标记一组主干节点。 从那里，它尝试将每个节点分类为以主干节点为根的组件。 关键操作是将每个占用的节点映射到其关联的主干组件，然后计算有多少令牌属于源和目标中的每个组件。 

一个微妙的点是映射必须一致：每个节点必须确定性地解析为唯一的脊椎锚点。 这是通过向更高程度的结构攀爬直至到达脊柱来强制执行的。 

最后的比较使用哈希图，因为组件的数量与 n 成线性关系，并且我们只需要频率相等。

| 步骤| 行动| 组件 A | 组件 B |

 ## 边缘情况

 一个关键的边缘情况是所有令牌完全位于连接到主干的单个小分支内。 在这种情况下，天真的解决方案可能认为令牌可以通过主干自由洗牌，但实际上分支充当封闭容器，除非存在不违反邻接性的可用主干路径。 组件计数方法正确地将所有令牌分配给同一根，并且相等检查确保不会假设非法重新分配。 

另一种情况是当标记对称地出现在书脊的相对两侧时。 在本地，这些可能看起来可以互换，但将一侧移动到另一侧需要通过主干节点，暂时强制邻接冲突。 除非双方已经具有匹配的计数（这与可行性一致），否则该不变量会阻止此类交换。
