 ## 解决方案
 ## 问题理解

 我们给出了一个生产系统，可以将其建模为作业的有向无环图。 每项作业都需要固定的时间来处理，从一项作业转移到另一项作业会产生额外的传输时间。 有效的生产计划从一项或多项入口作业开始，遵循作业之间的定向转换，并最终以一项或多项终端作业结束。 计划的总提前期是所有访问作业的处理时间加上所选路线上的所有过渡时间的总和。 

任务是计算从任何有效开始到任何有效结束的最大可能提前时间。 计算出这个最大时间后，我们还需要判断最优生产路线是否唯一。 如果恰好有一个作业序列达到了这一最大值，我们必须输出总时间以及该路线上的作业序列。 如果多条不同的路线达到相同的最大时间，我们会输出总时间，后跟字母 M。 

尽管该声明提到插入虚拟开始和结束作业，但实际结构更简单：作业是节点，转换是有向边，并且保证图没有循环。 这保证了拓扑排序的存在并且最长路径计算是明确定义的。 

限制很小：每个测试用例最多 50 个作业和 100 个转换。 这立即排除了图上线性或近线性动态规划之外的任何内容。 即使 O(n^3) 方法也可以单独通过，但会在多个测试用例中重复，并且会产生开销，O(n + m) 或 O(nm) 中的拓扑 DP 就是预期的结构。 

当存在多个启动节点时，会出现微妙的故障情况。 例如，如果作业 0 和作业 1 都没有传入边，并且都可以以相同的总成本到达作业 2，则存在两条仅在初始段不同的最佳路径。 选择任意起始节点的简单解决方案会错误地得出唯一性的结论。 

当多条路径仅在最终节点连接但较早分叉时，会出现另一个问题。 例如，如果两条不同的路线以相同的成本达到相同的最大最终作业，则两者都必须被视为有效的最优解，并且即使它们的后缀重合，输出也必须是M。 

## 方法

 暴力解决方案将枚举 DAG 中从每个起始节点到每个结束节点的所有可能路径，计算它们的总成本，并跟踪最大值。 由于该图是非循环的，但在最坏的情况下仍然是指数的，因此路径的数量可以随着分支呈指数增长。 即使有 50 个节点，二进制分支结构也可能已经创建了 2^50 条路径，这是不可行的。 

关键的观察结果是该图没有循环，因此最优子结构成立：到达节点的最佳方式仅取决于到达其前辈的最佳方式。 这将问题转化为具有节点权重和边权重的 DAG 上的最长路径问题。 

我们可以按拓扑顺序处理节点，并为每个节点计算结束于该节点的最大可实现提前期。 除此之外，我们还维护有多少种不同的方法可以达到该最大值，但我们将此计数限制为 2，因为我们只关心最优值是否唯一。 

DP 完成后，我们会查看所有终端节点并选择具有最大值的节点。 然后我们使用前驱指针重建路径。 在重建过程中，如果在任何时候一个节点有多个可以产生最佳值的有效前驱节点，我们将答案标记为不明确。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有路径进行暴力破解 | 指数| O(n + m) | 太慢了 |
 | 具有重建和计数功能的 DAG DP | O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

我们将每个作业视为权重等于其处理时间的节点，并将每个转换视为权重等于其传输时间的有向边。 

1. 为图构建邻接表并计算拓扑排序的入度。 这是必需的，因为图结构不是按顺序给出的。 
2. 对所有作业进行拓扑排序。 由于图是非循环的，这种排序保证了当我们处理一个节点时，所有可以到达它的节点都已经被处理过。 
3. 初始化 DP 数组，其中 dp[v] 表示完成作业 v 时可实现的最大提前时间。将 dp[v] 初始设置为所有节点 v 的处理时间，因为路径可以从任何入口节点开始。 
4.维护一个ways数组，其中ways[v]记录有多少条不同的最佳路径到达v，上限为2。这使我们能够检测是否违反了唯一性，而无需计算组合大数。 
5. 按拓扑顺序遍历节点。 对于传输成本为 w 的每个有向边 u → v，考虑将到 u 的最佳路径扩展到 v。候选值为 dp[u] + w +processing[v]。 如果这个值大于dp[v]，我们替换dp[v]，将ways[v]设置为ways[u]，并将u存储为v的前驱。如果相等，我们将ways[u]添加到ways[v]，但上限为2，我们记录v有多个最佳前驱，这将影响以后的重建。 
6.处理完所有边后，在所有没有出边的节点中识别出最好的终端节点。 如果多个终端达到相同的 dp 值，则唯一性已经被打破。 
7. 通过从所选终端节点向后跟随前驱指针来重建路径。 如果在任何时候多个前驱可以产生相同的最佳 dp 值，我们将解决方案标记为不明确。 
8. 如果重建唯一，则反转路径并输出。 否则输出M。 

正确性依赖于这样一个事实：在 DAG 中，每条路径都由最优子路径组成。 如果路径的前缀对于其端点来说不是最佳的，则整个路径不可能是最佳的。 一旦所有前驱已知，这允许独立计算 dp[v]。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(n, m, times, edges):
    adj = [[] for _ in range(n)]
    indeg = [0] * n

    for u, v, w in edges:
        adj[u].append((v, w))
        indeg[v] += 1

    # topological sort
    from collections import deque
    q = deque([i for i in range(n) if indeg[i] == 0])
    topo = []

    while q:
        u = q.popleft()
        topo.append(u)
        for v, w in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)

    dp = times[:]
    ways = [1] * n
    parent = [-1] * n

    for u in topo:
        for v, w in adj[u]:
            cand = dp[u] + w + times[v]
            if cand > dp[v]:
                dp[v] = cand
                ways[v] = ways[u]
                parent[v] = u
            elif cand == dp[v]:
                ways[v] = min(2, ways[v] + ways[u])

    best = max(dp)
    candidates = [i for i in range(n) if dp[i] == best]

    if len(candidates) != 1:
        return str(best) + ",M"

    end = candidates[0]

    if ways[end] > 1:
        return str(best) + ",M"

    # reconstruct
    path = []
    cur = end
    while cur != -1:
        path.append(cur)
        cur = parent[cur]
    path.reverse()

    return str(best) + "," + ",".join(map(str, path))

def main():
    data = sys.stdin.read().strip().splitlines()
    i = 0
    out = []

    while i < len(data):
        if not data[i].strip():
            i += 1
            continue

        n, m = map(int, data[i].split())
        i += 1

        times = list(map(int, data[i].replace(",", " ").split()))
        i += 1

        edges = []
        for _ in range(m):
            u, v, w = map(int, data[i].split())
            edges.append((u, v, w))
            i += 1

        out.append(solve_case(n, m, times, edges))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先构建邻接列表并使用入度跟踪计算拓扑顺序。 这确保了在处理节点时，所有传入的贡献都已经完成。 

dp 数组存储每个作业结束的最佳提前​​时间，每个转换都包括边缘传输时间和目标节点的处理时间。 这很重要，因为节点成本是状态的一部分，而不仅仅是转换。 

父数组用于重建。 仅当最佳过渡唯一时，它才会存储单个前驱。 使用ways数组单独跟踪歧义性，该数组被限制为2，因此我们只区分独特和非独特的情况。 

最后，我们在所有节点中选择最佳端点。 如果多个节点相连，或者最佳端点本身有多个最优路径，我们返回 M。否则，我们通过父指针向后行走来重建路径。 

## 工作示例

 ### 示例 1

 我们考虑一个小图，其中最佳路径是唯一的。 

设节点权重为`[2, 7, 2, 6]`和边定义了一种链状结构，其中只有一条路径实现最大成本。 

| 步骤| 节点| dp 更新 | 方式| 家长 |
 | --- | --- | --- | --- | --- |
 | 初始化| 全部 | dp = 节点权重 | 1 | - |
 | 进程 0 | 0 | 2 | 1 | - |
 | 流程 1 | 1 | 9 | 1 | 0 |
 | 流程 3 | 3 | 17 | 17 1 | 1 |

 最终答案是通向节点 3 的唯一路径。 

该轨迹表明每个节点都有一个最佳前驱节点，因此重建过程不会有任何歧义。 

### 示例 2

 现在考虑两条不同路径在接收器处达到相同最佳值的情况。 

| 路径| 成本|
 | --- | --- |
 | 0 → 1 → 5 | 53 | 53
 | 0 → 2 → 5 | 53 | 53

 两条路由在节点 5 处产生相同的最大 dp。在 DP 期间，dp[5] 两次达到相同的值，因此ways[5] 变为 2。 

即使最终成本已明确定义，这也会迫使输出为 M。

 ## 边缘情况

 一种重要的边缘情况是存在多个起始节点。 在这种情况下，dp 初始化将所有节点视为潜在的起点。 如果两个起点以相同的成本引入一条共享链，则两者都会对同一个汇做出贡献，并且 way 会变得大于 1，从而产生歧义。 

另一种边缘情况是存在多个具有相同最大 dp 值的终端节点。 即使每个终端都有独特的内部结构，多个最佳端点的存在意味着不存在单一的主导生产路径。 

当一个节点正好有一个最佳前驱节点，但该前驱节点本身可以​​通过多个最佳路径到达时，就会出现最后一种微妙的情况。 在这种情况下，父重建仍然产生单链，但方法会更早地检测到非唯一性并正确输出 M，从而防止误导重建。
