 ## 解决方案
 ## 问题理解

 我们被放置在一个大小为 100 x 100 的隐藏二维网格中。在这个网格内最多有十艘船，并且每艘船始终是由五个连续单元格组成的直线段，无论是水平还是垂直。 船永远不会重叠，但它们可以互相接触。 

我们不知道船只在哪里。 相反，我们通过反复选择要射击的细胞来与法官互动。 每次射击都会返回单元格是否为空、包含船段或导致整艘船被摧毁。 一旦所有船只都沉没，互动立即结束。 

微妙之处在于法官具有适应性。 只要响应与某些有效的最终位置保持一致，就可以在交互过程中决定船舶位置。 这消除了网格提前固定的任何假设，但它并没有消除几何约束：每艘船仍然是长度为 5 的直线段，并且最多有 10 艘。 

2500次射击的限制是主要瓶颈。 对网格进行全面扫描是不可能的，甚至探测每个单元两次也已经太昂贵了。 船舶的结构，即长长的刚性部分，是我们拥有的唯一杠杆。 

一个天真的想法是随机射击细胞直到我们被击中，然后尝试从击中扩展。 在对抗性自适应设置中，这可能会严重失败：法官总是可以延迟给出有用的命中，并且随机性没有给出找到所有船只的确定性界限。 

当尝试逐行或逐列密集搜索时，会出现不同的失败模式。 例如，每行扫描 100 列会产生 10000 次射击，即使在考虑扩展之前也远远超出了限制。 

核心挑战是使用一组确定性的小型探测器保证每艘船至少命中一次，同时保持该组足够小以保持在预算范围内。 

## 方法

 蛮力的观点很简单。 我们可以将每个细胞视为候选细胞并对其进行射击，标记命中，然后扩展每个发现的片段，直到找到所有五个部分。 这可以正确识别船只，并且扩展最多需要每艘船恒定数量的额外射击。 问题在于初始发现阶段：100 x 100 已经给出了 10000 个细胞，因此即使每个细胞一次注射也超过了限制的四倍。 

我们需要一种方法来减少搜索空间，同时仍然保证每个可能的长度为 5 的水平或垂直段包含至少一个选定的探测单元。 

这变成了覆盖问题。 我们需要一组网格点，使得任何行或任何列中五个单元格的每个连续块与该组相交。 关键的观察是两个维度的周期性结构都可以加强这种保证。 

如果我们为每个单元格着色`(row mod 5, column mod 5)`，然后关注两个残基匹配的单个对角线类，长度为 5 的每个行段必然跨越所有五个列残基，并且长度为 5 的每个列段跨越所有五个行残基。 这可确保每个有效的船段恰好包含所选类别的一个单元。 

这将搜索集从 10000 个单元减少到大约 10000 / 5 = 2000 个单元，这完全符合限制。 一旦发现命中，我们就可以在本地扩展以恢复整艘船，因为每艘船只有五个单元长。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力网格扫描 | O(10000 + k) 次射击 | O(1) | O(1) | 太慢了 |
 | 模块化击球组+扩展| O(2000 + k) 次射击 | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 构建一组由所有位置组成的目标单元格`(i, j)`这样`i mod 5 == j mod 5`。 这产生了大约五分之一的网格。 选择此模式的原因是五个连续行或列的任何序列都会循环通过所有残基模 5。 
2. 对该组中的每个单元格进行一次射击。 如果响应为“未命中”，我们将忽略它。 如果它被“击中”或“沉没”，我们就发现了至少一个属于一艘船的细胞。 
3. 当发现命中单元时，我们尝试重建整艘船。 由于每艘船都是长度为 5 的直线段，因此只需确定其方向，然后向两个方向延伸即可。 
4. 为了确定方向，我们在四个基本方向上探测相邻的细胞。 如果`(x+1, y)`或者`(x-1, y)`产生撞击，船是垂直的。 否则，如果`(x, y+1)`或者`(x, y-1)`产生撞击，船是水平的。 只有一个方向会产生额外的飞船单元。 
5.一旦知道方向，我们就沿着线行走，总共探测最多五个连续的单元格，当我们到达未命中的位置或收集五个飞船单元格后停止。 每个新发现的单元都会被记录下来，这样我们就不会重新启动同一艘船的处理。 
6. 在所有五个部分均被确认后（通过击中或最终“沉没”），我们将该船标记为完整，并继续扫描击中集中的剩余单元格。 

关键思想是发现和重建是分开的。 命中集保证我们总是找到每艘船的种子细胞，并且重建阶段确定性地将种子扩展为全长 5 段。 

### 为什么它有效

 细胞集`(i, j)`模 5 的相等残基与每个可能的长度为 5 的水平段相交，因为这样的段恰好包含每个列残基类别的一个代表。 相同的对称性适用于跨行残基的垂直段。 这保证了每艘船，无论法官的位置或适应性如何，都必须包含至少一个被探测单元。 

一旦找到单个船舶单元，船舶是固定长度的直线段的约束就会迫使整个结构沿着单行或单列放置，并且不可能出现分支或歧义。 这使得局部扩展具有确定性和有界性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

N = 100

def ask(x, y):
    print(x, y)
    sys.stdout.flush()
    return input().strip()

def expand(x, y, used):
    # try to determine orientation
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]
    vertical = False
    horizontal = False

    for dx, dy in dirs:
        nx, ny = x + dx, y + dy
        if (nx, ny) in used:
            if dx != 0:
                vertical = True
            if dy != 0:
                horizontal = True

    # fallback probing if needed
    if not vertical and not horizontal:
        if ask(x+1, y) in ("hit", "sunk"):
            vertical = True
        elif ask(x-1, y) in ("hit", "sunk"):
            vertical = True
        elif ask(x, y+1) in ("hit", "sunk"):
            horizontal = True
        elif ask(x, y-1) in ("hit", "sunk"):
            horizontal = True

    cells = [(x, y)]
    used.add((x, y))

    if vertical:
        for dx in [1, -1]:
            nx, ny = x + dx, y
            while 1 <= nx <= N:
                if (nx, ny) in used:
                    nx += dx
                    continue
                res = ask(nx, ny)
                if res == "miss":
                    break
                used.add((nx, ny))
                cells.append((nx, ny))
                if len(cells) == 5:
                    break
                nx += dx
    else:
        for dy in [1, -1]:
            nx, ny = x, y + dy
            while 1 <= ny <= N:
                if (nx, ny) in used:
                    ny += dy
                    continue
                res = ask(nx, ny)
                if res == "miss":
                    break
                used.add((nx, ny))
                cells.append((nx, ny))
                if len(cells) == 5:
                    break
                ny += dy

    return cells

def main():
    n, k = map(int, input().split())

    used = set()

    for i in range(1, n+1):
        for j in range(1, n+1):
            if i % 5 != j % 5:
                continue
            if (i, j) in used:
                continue

            res = ask(i, j)
            if res == "miss":
                continue

            if (i, j) not in used:
                ship = expand(i, j, used)
                # ship fully discovered

    return

if __name__ == "__main__":
    main()
```主循环将查询限制为模块化对角线类，这确保了 2000 个镜头的预算。 这`used`set 可防止扩展期间的冗余查询，并避免在多个发现中重复计算船舶单元。 

扩展函数故意保守：只有在确认某个方向包含船舶的一部分后，它才会继续探测该方向。 这可以避免浪费镜头去探索空旷的空间。 

## 工作示例

 考虑一个简化的 10 x 10 网格，其中有一艘水平船`(3,3)`到`(3,7)`。 

我们只拍摄细胞`i mod 5 == j mod 5`， 例如`(1,1)`,`(2,2)`,`(3,3)`,`(4,4)`等等。 第一个相关点击是`(3,3)`。 

| 步骤| 射击| 回应 | 行动|
 | ---| ---| ---| ---|
 | 1 | (3,3) | 击中| 触发扩展|
 | 2 | (3,4) | 击中| 检测水平|
 | 3 | (3,5) | 击中| 继续 |
 | 4 | (3,6) | 击中| 继续 |
 | 5 | (3,7) | 沉没| 船舶完成 |

 该痕迹表明，单次种子命中就足以确定地恢复整艘船。 

现在考虑一艘垂直船`(6,2)`到`(10,2)`。 例如，相同的命中集可保证对这些单元格之一进行采样`(8,2)`。 

| 步骤| 射击| 回应 | 行动|
 | ---| ---| ---| ---|
 | 1 | (8,2) | 击中| 触发扩展|
 | 2 | (7,2) | 击中| 检测垂直|
 | 3 | (6,2) | 击中| 继续 |
 | 4 | (9,2) | 击中| 继续 |
 | 5 | (10,2) | (10,2) | 沉没| 船舶完成 |

 第二条轨迹证实方向检测加上线性膨胀始终终止于最多五个已确认的船舶单元。

 ## 边缘情况

 一种边缘情况是，一艘船完全躺在最初没有被探测到的单元上，直到扫描后期才被探测到。 例如，一艘船位于`(1,5)`到`(1,9)`可能只有在以下情况下才会被发现`(1,5)`是在模块化模式中达到的。 该算法仍然保证发现，因为每个长度为 5 的水平段恰好包含一个匹配的残差单元，因此必须查询这五个位置中的至少一个。 

另一种情况是重叠发现尝试，其中多个探测单元属于同一艘船。 例如，两次点击`(3,3)`和`(3,5)`两者都可能在初始扫描期间被触发。 这`used`set 确保一旦从一个种子扩展一艘船，在扫描过程中会忽略来自同一艘船的后续命中，从而防止双重重建并避免多余的镜头。 

第三种情况涉及边界相邻的船舶，例如`(1,1)`到`(1,5)`。 扩展仍然有效，因为当在网格外或超出船舶端点遇到未命中时，探测就会停止，并且固定长度保证在五次成功命中内完成。
