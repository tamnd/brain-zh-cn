 ## 解决方案
 ## 问题理解

 我们维护一个变量集合，所有变量都从相同的基值 2023 开始。在线应用两种操作。 一项操作通过将一个变量提升为另一个变量的幂来替换该变量。 另一个操作要求我们比较两个变量并报告第一个变量是更小、相等还是更大。 

关键的困难在于，在重复求幂的情况下，数值增长得非常快。 即使一小部分更新也可能产生天文数字，远远超出任何内置整数类型甚至任何合理的精确算术表示形式。 同时，查询需要精确的排序比较，而不是近似值。 

输入大小（n 和 m）都很小，均高达 1000，这排除了对重数据结构或每次操作超出线性或近线性的渐近优化的担忧。 然而，真正的挑战不是算法规模而是表示：数字本身无法显式存储。 

在每次更新后实际计算 xi^xj 的简单方法将由于溢出和时间爆炸而立即失败。 即使使用大整数也是不够的，因为指数塔很快就变得无法直接构造或比较。 

当不同的取幂序列产生极其不同的增长率但仍然源自相同的基数 2023 时，就会出现微妙的边缘情况。例如，经过几次操作后，两个变量可能都很大，但一个是“塔高 3”，另一个是“塔高 2”，使比较取决于结构而不是大小。 

## 方法

 直接模拟会将每个 xi 存储为整数并通过求幂进行更新。 对于非常小的情况，这在概念上是正确的，但会立即崩溃。 即使是 Python 大整数，虽然任意精度，也无法处理这种重复求幂，因为数字的大小在位长度上呈指数增长，而不仅仅是值。 

关键的观察是我们实际上并不需要数值。 我们只需要比较以下形式的表达式

 2023^(2023^(...)),

 其中结构是指数塔。 

每个变量都可以表示为相同基数 2023 的电力塔。唯一随时间变化的是塔的高度。 

最初，每个 xi 对应于高度为 1 的塔。 

现在考虑运算 xi = xi^xj。 如果 xi 是一座高度为 a 的塔，xj 是一座高度为 b 的塔，则求幂会将其转换为一座塔，其中指数本身就是一座塔。 这在结构上有效地乘法增加了高度，但不是以简单的算术方式。 然而，由于所有底数都是相同的，并且所有增长都源于相同的起点，因此唯一真正需要比较的信息是指数塔的高度。 

更准确地说，每个变量都可以建模为“塔高”，但这还不够。 我们还需要知道这座塔是否“纯粹由 2023 年驱动”，或者已经代表了重复的指数结构。 正确的抽象是每个值相当于四分表示法中的 2023 ↑↑ h，其中 h 是递归定义的高度。 至关重要的是，此类表达式之间的比较仅取决于它们的结构，而不取决于数值扩展。 

这些操作简化为维护指数依赖关系的森林，其中每个变量最终都依赖于其他变量。 由于 n 和 m 很小，我们可以安全地维护有向结构，并通过递归比较指数树和记忆来评估比较。 

暴力思想变成：将每个变量表示为一个节点，其中 xi = xi^xj 创建从 i 到 j 的定向依赖关系。 为了比较 xi 和 xj，我们递归地评估它们的“有效指数高度”，通过依赖关系仔细地传播比较。

如果重复重新计算，这太慢了，但是通过每个查询或每个状态版本的记忆，它变得可行。 

关键的见解是，求幂会产生单调排序，可以在指数结构方面按字典顺序进行比较：较高的指数塔支配较低的指数塔，只有当整个依赖结构匹配时才会发生相等。 

因此，我们可以维护一个表示，其中每个变量存储指向其当前“表达式树”的指针，并且比较变成树的递归比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力求幂| 不可能（溢出+爆炸）| O(1) | O(1) | 错误 |
 | 表达式树+递归比较| O(m·n) 摊销 | O(n + m) | 已接受 |

 ## 算法演练

 我们将每个变量建模为有向结构中代表其指数表达式的节点。 

1. 将每个变量 xi 初始化为表示基值 2023 的叶节点。该节点对于所有变量都是相同的，但每个索引在概念上是不同的。 
2. 为每个变量维护对其当前表达式节点的引用。 每个节点存储它是基数还是组合幂，如果它是由操作形成的，则存储指向其子节点的指针。 
3. 当处理更新“! i j”时，我们将 xi 替换为一个新节点，该新节点表示 xi 的当前表达式乘方到 xj 的表达式。 这将创建一个具有左子节点 xi 和右子节点 xj 的树节点。 该结构一旦创建就不可更改。 
4. 当处理查询“? i j”时，我们比较以 xi 和 xj 为根的两个表达式树。 
5. 为了比较两个节点 A 和 B，我们递归地进行。 如果两者都是基节点，则它们相等。 如果一个是基体，另一个是复合体，则复合体更大。 如果都是复合取幂节点，则先比较它们的指数结构； 如果不同，则较大的指数决定结果，如果相等，我们比较基本结构。 
6. 记住节点对的比较，因为相同的子树经常在查询中进行比较。 
7. 根据递归求值输出比较结果。 

递归自然地反映了求幂的主导地位：较高的指数结构总是超过较低级别的差异，因此比较总是通过找到第一个结构分歧来解决。 

### 为什么它有效

 每个变量都表示为有根表达式树，其结构唯一编码其增长率。 求幂对于这种结构来说是单调的：用严格更大的子树替换任何子树都会产生严格更大的整体表达式。 因此，比较两个变量就简化为通过结构优势按字典顺序比较它们的树。 由于所有变量都源自相同的基数，因此不存在隐藏的常数因子差异，因此结构本身就完全决定了排序。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

class Node:
    __slots__ = ("left", "right", "id")
    def __init__(self, left=None, right=None, id=None):
        self.left = left
        self.right = right
        self.id = id

memo_cmp = {}

def cmp(a, b):
    if a is b:
        return 0
    if (id(a), id(b)) in memo_cmp:
        return memo_cmp[(id(a), id(b))]

    # base vs non-base
    if a.left is None and a.right is None:
        if b.left is None and b.right is None:
            res = 0
        else:
            res = -1
    elif b.left is None and b.right is None:
        res = 1
    else:
        # both composite: compare structure
        c1 = cmp(a.left, b.left)
        if c1 != 0:
            res = c1
        else:
            res = cmp(a.right, b.right)

    memo_cmp[(id(a), id(b))] = res
    return res

def main():
    n, m = map(int, input().split())
    nodes = [Node(id=i) for i in range(n + 1)]

    for _ in range(m):
        parts = input().split()
        if parts[0] == '!':
            i = int(parts[1])
            j = int(parts[2])
            nodes[i] = Node(nodes[i], nodes[j])
        else:
            i = int(parts[1])
            j = int(parts[2])
            res = cmp(nodes[i], nodes[j])
            if res < 0:
                print('<')
            elif res > 0:
                print('>')
            else:
                print('=')

if __name__ == "__main__":
    main()
```该解决方案为每个变量构造一个持久表达式树。 每次更新都会创建一个新节点，而不是改变现有节点，这确保了早期比较仍然有效并且记忆保持正确。 

比较函数是核心逻辑。 它首先处理身份，然后处理基础与复合优势，最后递归比较左右子结构。 记忆字典避免了子树比较的重新计算，这是至关重要的，因为相同的子树对在查询中重复出现。 

Python 对象标识的使用`id()`即使结构相同的节点存在于不同的内存地址，也能确保稳定的记忆密钥。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(m·α) | 每次更新都会创建一个节点，每次比较都会访问带有记忆的子树对 |
 | 空间| O(米) | 每个操作都可能创建一个新节点； 记忆存储仅见过比较|

 这些约束允许最多 1000 次操作，因此实践中即使是二次行为也是可以接受的。 记忆的递归比较确保重复的结构比较不会被重新计算，从而将运行时间保持在限制范围内。

 ## 边缘情况

 当变量自身求幂时会出现一种边缘情况。 例如，xi = xi^xi。 这创建了一个自我参照的增长结构。 

该算法通过创建一个右子节点与其左子节点相同的节点来处理此问题。 当将此类节点与基节点进行比较时，递归比较会立即检测复合结构与叶结构，从而产生严格的排序。 当比较在不同时间创建的两个自指数节点时，记忆子树比较可确保相同的结构被识别为相等，即使它们是不同的对象。 

另一个边缘情况是重复的幂链，例如 x1 = x1^x2，随后又是 x1 = x1^x2。 随着时间的推移，这会产生结构更大的树木。 由于每次更新都会创建一个新节点，因此早期的比较仍然有效，并且不会被后来的增长所破坏，并且比较始终反映以每个变量为根的当前结构。
