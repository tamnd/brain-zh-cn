 ## 解决方案
 ## 问题理解

 我们得到了一套充电器，每个充电器都有固定的物理长度，以及一个长电源板，其中包含排列成一行的有限数量的插座。 每个充电器必须使用其两个端点之一恰好插入一个插座，然后根据其方向向左或向右延伸。 关键的几何约束是充电器的行为就像固定在插座上的间隔：一旦插入，充电器就会占据带子的连续部分，并且两个充电器在空间上不能重叠，尽管它们可以在边界处接触。 

任务是选择尽可能多的充电器，并将每个选定的充电器分配到不同的插座位置（因为一个插座只能容纳一个插头），以便它们产生的间隔不会重叠。 

输入大小清楚地表明，暴力分配是不可能的。 由于多达 200,000 个充电器和插座位置可能高达 10^9，任何尝试所有子集或所有布局的方法都会发生组合爆炸。 即使是 O(n²) 贪婪模拟在最坏的情况下也会变得临界，并且任何涉及排序加上每个放置的嵌套扫描的操作都会失败。 

翻转每个充电器的能力会产生一个微妙的问题。 放置在插座位置 p 处的长度为 w 的充电器会产生 [p - (w - 1), p] 或 [p, p + (w - 1)] 区间。 这种对称性使得天真的贪婪方法变得脆弱，因为最佳方向取决于相邻的选择。 

一个常见的失败案例是假设我们总是更喜欢首先放置长充电器。 例如，如果插座靠得很近，那么尽早放置一个长充电器可能会阻挡以后许多较短的充电器，即使翻转它可以避免干扰。 相反，总是先放置短充电器也可能不是最佳选择，因为长充电器的可行放置位置较少。 

## 方法

 强力解释将尝试将充电器分配给插座并测试所有方向，然后验证间隔是否重叠。 即使我们固定 k 个充电器的分配，检查有效性也是 O(k)，并且子集的数量呈指数级增长。 当 n 非常小时，这立即变得不可行。 

一种更结构化的尝试是将每个充电器插座对视为生成两个可能间隔的选择。 然后，我们想要选择一组最大的不重叠间隔，但有每个插座最多使用一次并且每个充电器最多使用一次的约束。 这是几何设置中类似匹配的选择。 

关键的见解是避免考虑任意放置，而是将每个插座解释为我们想要将充电器“安装”到左侧或右侧的决策点。 如果我们按顺序处理插座，唯一重要的是每个所选充电器延伸到已占用空间或未来空间的距离。 

这导致了一种贪婪的策略：我们从左到右分配插座，并且在每个插座上我们尝试选择最合适的充电器，可以放置在不与我们已经承诺的重叠的情况下。 由于所有交互都减少到锚定在固定点的间隔端点，因此我们可以维护一个跟踪可用充电器的结构，并始终选择最佳的可行候选者。 

关键的结构观察是，每个充电器，无论方向如何，都会消耗固定在插座上的连续段，因此可以纯粹通过比较它相对于先前占用的边界延伸的距离来确定可行性。 这将问题转化为重复选择不违反当前边界的最佳可用“扩展”，可以使用贪婪多重集策略来维护该边界。

 | ---| ---| ---| ---|
 | 蛮力 | 指数| O(n) | 太慢了 |
 | 贪婪排序选择 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 按长度降序对充电器进行排序。 直觉是，较长的充电器受到更多限制，因此如果可以放置它们，应首先考虑它们，同时我们在插座分配上仍然具有灵活性。 
2.从左到右维护插座上的指针，并维护跟踪已放置充电器的当前“占用边界”的数据结构。 最初，没有空间被占用。 
3. 对于每个插座位置，尝试分配可放置的最长剩余充电器，且不会与先前分配的充电器重叠。 对于插座 p 处长度为 w 的充电器，其相对于当前边界的最佳放置位置取决于将其向左或向右放置是否保持其间隔在占用区域之外。 
4. 放置充电器时，请选择对未来插座干扰较小的方向。 具体来说，如果向左延伸会与当前的左边界重叠，我们就向右延伸，反之亦然。 
5. 放置充电器后，相应更新占用的区间边界。 这确保了所有未来的放置都尊重已经占用的空间。 
6. 继续操作，直至所有插座均已处理完毕或剩余的充电器无法安装。 

### 为什么它有效

 在任何步骤中，该算法都会维护一个代表已占用空间的连续禁区。 每个仍然可以放置的充电器必须至少在一个方向上完全安装在该区域之外。 由于所有充电器都间隔固定在不同的插座上，因此任何可行的解决方案都可以进行转换，以便始终尽早放置最长的可行充电器，而不会减少总数量。 这种交换论点确保延迟可行的长充电器永远不会增加最终的放置数量，因为它只会减少剩余的可用空间而不会创造新的机会。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, s = map(int, input().split())
    w = list(map(int, input().split()))

    # sort chargers by length descending
    w.sort(reverse=True)

    # we simulate placing chargers greedily on sockets
    # we track occupied boundary as an interval [L, R]
    L, R = 0, -1
    used = 0
    i = 0

    for socket in range(1, s + 1):
        if i >= n:
            break

        # try to place next largest charger
        wlen = w[i]

        # try placing to the left or right
        left_ok = (socket - (wlen - 1)) > R
        right_ok = (socket + (wlen - 1)) < L if L > 0 else True

        if left_ok or right_ok:
            used += 1
            i += 1

            if left_ok and (not right_ok or socket - (wlen - 1) >= 1):
                L = min(L if L > 0 else socket - (wlen - 1), socket - (wlen - 1))
                R = max(R, socket)
            else:
                R = max(R, socket + (wlen - 1))
                if L == 0:
                    L = socket

    print(used)

if __name__ == "__main__":
    solve()
```该实现对充电器进行排序，以便我们始终尝试将最严格的充电器放在第一位。 变量`L`和`R`表示由所有放置的充电器感应的当前占用的条带段。 按升序处理每个插座，对于每个插座，我们检查当前最大的未使用的充电器是否可以在不相交的情况下放置`[L, R]`。 

方向选择被编码在左右可行性检查中。 如果两个方向都无效，我们会跳过该充电器。 如果至少有一个有效，我们将放置它并相应地更新占用的间隔。 该更新保持了所有占用空间都作为单个连续段进行跟踪的不变性，这就是允许恒定时间可行性检查的原因。

 ## 边缘情况

 一个关键的边缘情况是所有充电器都大于插座间距允许的情况。 在这种情况下，每次放置尝试都会失败左右可行性检查，并且算法正确返回零，因为`left_ok`和`right_ok`总是假的。 

另一种情况是，存在许多小型充电器，但由于分拣而首先处理单个大型充电器。 该算法可能会提前放置大型充电器，但由于它只扩展占用的间隔一次，因此如果后续的小型充电器重叠，它们将被正确地拒绝。 正确性依赖于一个不变量，即一旦一个区域被占用，以后的放置就不允许与其相交，从而保持可行性一致性。
