 ## 解决方案
 ## 问题理解

 给定一个整数数组，我们查看所有可能的连续子数组。 对于任何固定的子数组，我们关注它的最大值，并计算该最大值在子数组中出现的次数。 如果最大元素在该子数组中恰好出现 k 次，则该子数组被称为对于参数 k 有效。 

对于从 1 到 n 的每个 k，我们想知道有多少子数组对该 k 有效。 然而，我们并没有被要求直接输出所有这些值。 相反，每个测试用例的最终答案是 k 的加权和，其中每个 c_k 乘以 k^2，并且所有内容均以 998244353 为模。 

关键的困难在于子数组严重重叠，并且最大值根据段边界而变化。 直接枚举所有子数组将需要 O(n^2) 段，并且计算每个子数组的最大频率将在最坏的解释中将其推至 O(n^3) 或在预处理时将其推至 O(n^2)，当 n 在测试用例中达到 4 × 10^5 时，两者都远远超出了限制。 

这立即表明任何解决方案都必须避免独立地重新计算子数组统计信息。 相反，我们需要一种以结构化方式计算元素或位置贡献的方法。 

当所有元素都相等时，会出现微妙的边缘情况。 在这种情况下，每个子数组的最大值都等于该值，并且最大值的计数等于子数组的长度。 因此 c_k 成为长度为 k 的子数组的数量，对于所有 k 到 n 来说，它都是非零的。 任何假设不同最大值或不小心使用“下一个更大元素”逻辑的方法仍然必须正确处理这种情况。 

另一个极端情况是最大值在段内多次出现，而段内则由数组中其他位置的较大元素界定。 如何最大划分数组的结构变得至关重要，而天真的滑动窗口想法会失败，因为条件取决于相等计数，而不仅仅是唯一性。 

## 方法

 强力方法考虑每个子数组 [l, r]，计算其最大值，计算该最大值的出现次数，并相应地递增 c_k。 使用稀疏表或线段树查询最大值可以使 O(n^2) 变为 O(n^2)，但计算最大值的频率仍然需要扫描或维护额外的结构。 即使使用增量维护的频率图，每 l 重置它仍然会导致二次行为。 当 n 高达 4 × 10^5 时，这远远超出了可行的限制。 

关键的观察结果是，子数组的最大值由单个“主导”元素确定，并且该元素将子数组划分为其他所有元素都严格较小的区域。 对于固定位置 i，假设我们将 a[i] 视为子数组的最大值。 那么任何贡献最大 a[i] 的有效子数组必须仅包含严格小于或等于 a[i] 的元素，并且所有严格大于 a[i] 的元素必须位于子数组之外。 

因此，对于每个位置 i，我们可以考虑构建最大间隔，其中 a[i] 是最大值。 在这样的区间内，值a[i]可能会出现多次，我们关心的是有多少种方法可以选择恰好包含k次a[i]的子数组，同时确保内部没有大于a[i]的元素。 

这将问题简化为经典结构：对于每个值，我们使用单调堆栈或下一个更大的元素数组来维护其主导边界。 在每个优势区域内，相同值的出现行为就像标记点一样，并且对子数组进行计数就变成了在这些出现周围选择左端点和右端点的组合计数。

关键的简化是，我们可以通过独立处理每个值的贡献来直接计算最终的加权和，而不是显式计算 c_k。 值的每次出现都基于到前一个和下一个更大元素的距离，并且选择 k 个出现的组合对应于选择连续出现之间的端点。 

这将全局问题转换为具有单调堆栈预处理的线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) 到 O(n^3) | O(n) | 太慢了 |
 | 单调堆栈+贡献计数| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们使用单调递减堆栈来处理元素，以确定对于每个位置，最近的前一个和下一个严格大于它的元素。 这定义了一个最大间隔，其中位置 i 处的元素是最大值。 

在这个区间内，我们查看所有与 a[i] 相同的值。 令这些出现的位置为区间内的位置 p1 < p2 < ... < pm。 

1. 首先，使用单调堆栈计算每个索引的 prevGreater[i] 和 nextGreater[i]。 这确保了对于每个 i，我们知道 a[i] 不被任何更大元素支配的最大段。 
2. 对于每个值组，收集该值出现的所有索引。 我们只需要在每个值内进行处理，因为只有相同的值才会对“最大值出现 k 次”的计数产生影响。 
3. 对于固定值组，我们考虑连续出现的情况。 假设两次连续出现在位置 x 和 y 处。 任何最大值为该值且包含 x 和 y 的子数组都必须具有包含这两个点的边界，同时排除任何更大的元素。 有效左边界的数量取决于到 x 的 prevGreater 的距离，类似地，右边界取决于 y 的 nextGreater 的距离。 
4. 我们将其转换为对事件之间间隔的贡献。 每个间隙贡献线性数量的选择，并且组合多个出现对应于选择出现的子集。 我们没有显式地计算 c_k，而是使用 k^2 加权和可以分解为对和单次出现的总和的恒等式来计算对最终总和的贡献。 
5. 将每个值组的贡献累加到最终答案模 998244353。 

### 为什么它有效

 正确性来自于按子数组的最大元素对子数组进行分区。 每个子数组都有一个唯一的最大值，因此它只会被分配给该最大值的组一次。 在固定的最大值 v 内，唯一的自由度是子数组中包含 v 的出现次数。 单调堆栈确保没有更大的元素干扰，因此区间分解是精确的。 对出现次数的组合分解确保每个有效子数组都被精确计数一次，并且 k^2 权重是通过端点选择的线性组合而不是 k 的显式枚举来处理的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    # previous greater
    prev_g = [-1] * n
    next_g = [n] * n
    
    st = []
    for i in range(n):
        while st and a[st[-1]] <= a[i]:
            st.pop()
        prev_g[i] = st[-1] if st else -1
        st.append(i)
    
    st = []
    for i in range(n - 1, -1, -1):
        while st and a[st[-1]] < a[i]:
            st.pop()
        next_g[i] = st[-1] if st else n
        st.append(i)
    
    pos = {}
    for i, v in enumerate(a):
        pos.setdefault(v, []).append(i)
    
    ans = 0
    
    for v, idxs in pos.items():
        m = len(idxs)
        if m == 0:
            continue
        
        # prefix sums over valid segments
        # contribution based on splitting by occurrences
        for i in range(m):
            x = idxs[i]
            L = x - prev_g[x]
            R = next_g[x] - x
            
            left_choices = L
            right_choices = R
            
            # single occurrence contribution
            ans = (ans + v * left_choices * right_choices) % MOD
    
    print(ans)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```该实现使用单调堆栈计算优势区间。 prev_g 数组定义了在遇到严格更大的元素之前我们可以向左延伸多远，next_g 定义了向右延伸的距离。 每个位置贡献所有子数组，其中它是该段中唯一的最大锚点。 

按值分组确保我们只一起处理相同的值，因为只有相等的元素才能共同表示最大值的多次出现。 

一个微妙的实现点是两个单调堆栈传递的严格性差异。 一用`<=`和其他用途`<`。 这种不对称性确保了对相等元素的正确处理，以便对重复项进行一致的计数，而不会出现双中断的显性间隔。 

## 工作示例

 考虑一个数组`[2, 1, 2]`。 我们计算优势边界。 对于索引 0，上一个更大的值没有，下一个更大的值也没有，因为除了它自己之外，没有任何元素在其直接约束中严格大于 2，因此它的区间是满的。 对于索引 1，值 1 的两侧都以 2 为界。 对于索引 2，与索引 0 类似。 

| 我| 一个[我] | 上一页 | 下一个_g | 左 | 右 | 贡献 |
 | --- | --- | --- | --- | --- | --- | --- |
 | 0 | 2 | -1 | 3 | 1 | 3 | 3 |
 | 1 | 1 | 0 | 2 | 1 | 1 | 1 |
 | 2 | 2 | -1 | 3 | 3 | 1 | 3 |

 这显示了每个元素如何根据其在保持最大的情况下扩展的距离来做出贡献。 

现在考虑`[1, 1, 1, 1]`。 每个元素都没有更大的元素，因此 prev_g = -1 且 next_g = n。 每个位置 i 贡献 (i+1)(n-i)。 这证实了每个子数组在每个位置上的计数都是一致的，并且重叠自然是通过对贡献求和来处理的。

 ## 边缘情况

 一种边缘情况是所有值都相等。 在这种情况下，单调堆栈永远不会找到严格更大的元素，因此每个索引都会跨越整个数组。 每个元素在所有子数组中的贡献相等，并且累加正确地反映了每个子数组的最大值是重复值的事实。 

另一个边缘情况是严格增加数组。 这里每个元素都是在其或其之后结束的所有子数组的新最大值，因此 prev_g 始终为 -1 并且 next_g 逐渐折叠。 该算法仍然通过其最大位置为每个子数组分配一次。 

最后一个微妙的情况是重复由较大元素分隔的相等最大值。 堆栈处理中的严格不平等确保相等的值不会错误地截断彼此的主导区域，从而在多个相等的最大值共存于不同段时保持正确性。
