 ## 解决方案
 ## 问题理解

 我们得到了整数单元数轴上的一系列事件。 每个事件都是在特定时间和位置出现的一枚硬币，并且只存在一秒钟。 两个机器人在时间 1 之前启动，每一秒它们都可以沿线的任一方向移动到固定的速度限制。 移动后，硬币出现，任何站在硬币单元上的机器人都会收集它。 

任务不是模拟固定速度的运动。 相反，我们必须确定可能的最小速度，以便两个机器人可以选择起始位置和移动策略，从而使它们能够收集所有硬币。 

关键的困难在于硬币是有时间顺序的，并且移动限制会产生跨时间的耦合。 机器人无法传送，因此每个硬币都会施加一个约束，将其位置和时间与之前收集的硬币必须能够到达的位置和时间相关联。 

输入大小很大，测试用例中总共最多 10^6 个硬币。 这立即排除了任何对硬币进行推理或逐时间步模拟时间的解决方案。 任何方法都必须在每个测试用例中以基本上线性或接近线性的时间处理硬币。 

一种微妙的边缘情况是，硬币在时空上相距太远，即使是一个机器人也无法到达，但两个机器人仍然可以共同到达。 例如，如果硬币出现在（时间 1，位置 1）和（时间 1，位置 10^9），则如果一个机器人从单个位置开始，无论速度如何，它都无法收集到这两个硬币，但两个机器人可以轻松地分担责任。 任何正确的解决方案都必须正确区分不可行是由于速度不足还是由于基本几何分离所致。 

当硬币随着时间的推移在两个相距遥远的区域之间交替时，就会出现另一种重要的情况。 不考虑跨时间差异的全局可行性而将硬币天真地贪婪地分配给机器人将会失败，因为早期的分配以局部不可见的方式限制了未来的可达性。 

## 方法

 蛮力视角首先确定候选速度 v 并尝试检查可行性。 如果我们可以测试给定的 v，我们就可以二分搜索答案。 

对于固定的v，每个机器人定义了一段时间内可到达的区间：从起始位置开始，t秒后可以到达v·t半径内的任何位置。 这意味着，如果机器人收集一系列硬币，连续收集的硬币之间的距离必须遵守时间和空间限制。 然而，由于有两个机器人，因此每个硬币必须分配给两个序列之一。 

直接蛮力会尝试将所有硬币分配给两个机器人和所有起始位置。 这是 n 的指数并且立即不可行。 

关键的观察是，一旦 v 固定，可行性就变成了时间排序点的两条“路径”上的调度问题。 每个机器人必须能够独立地以递增的时间顺序遍历其分配的子序列，并且对于分配给同一机器人的每个连续对 (ti, ci), (tj, cj)，我们必须有 |cj − ci| ≤ v·(tj − ti)。 该约束充分表征了单个机器人的可行性。 

因此，对于固定 v，问题简化为将序列划分为两个子序列，每个子序列遵循单调可达性约束。 这是偏序集上的经典双路径可行性问题。 

我们通过使用贪婪结构来避免显式搜索分区：我们维护每个机器人最新的可到达“状态”，并将每个硬币分配给任何可以切实获取它的机器人，同时保持未来最灵活。 这可以转化为 O(n) 的可行性检查。 

一旦给定 v 的可行性是可测试的，我们就会在最大 10^9 的范围内对 v 进行二分搜索。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力分配 | 指数| O(n) | 太慢了 |
 | 检查+二分查找+贪心可行性 | O(n log V) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们对最小的 v 进行二分搜索，以便所有硬币都可以分配给两条有效的机器人路线。 

对于固定的 v，我们以递增的时间顺序处理硬币，并为每个机器人维护它收集到的最后一个硬币。 

1. 按时间对硬币进行排序，尽管输入已经保证了这一点。 
2. 对于每个机器人，将最后收集的硬币存储为（时间，位置）。 最初，两个机器人都没有硬币，这意味着可以想象它们在时间 0 时从任何位置开始。 
3. 按时间顺序迭代硬币。 对于当前硬币（t，c），判断机器人A或机器人B能否拿走它。 如果机器人没有先前的硬币，或者距离约束成立，则机器人可以拿走它：

 |c - c_prev| ≤ v·(t − t_prev)。 

这种情况确保机器人能够及时从最后收集的硬币移动到新的硬币。 
4. 如果两个机器人都能拿走硬币，则将其分配给前一个硬币“限制较少”的机器人，即最后位置在时空权衡中更接近当前硬币的机器人。 一个简单有效的规则是将其分配给最后时间较小的机器人，因为该机器人未来有更多的时间松弛。 
5. 如果只有一个机器人可以接受，则将其分配给那里。 
6. 如果两个机器人都无法接受，则当前 v 不可行。 
7. 如果所有币都分配成功，则 v 可行。 

我们对 v 使用从 0 到 10^9 的二分搜索。 可行性检查是单调的：如果速度 v 有效，则任何更大的速度也有效，因为所有可达性约束都会变弱。 

### 为什么它有效

 正确性依赖于这样一个事实：每个机器人收集的硬币形成一个时间递增序列，在空间相对于时间上具有严格的利普希茨约束。 该约束在时间顺序上是凸的，这意味着如果序列可行，则插入较早的约束不会使稍后超出局部违规的可行性无效。 贪婪分配确保当存在有效分配时，我们永远不会在两个机器人上不必要地阻塞一枚硬币，因为任何可行的解决方案都会导致至少一个与这种逐步可行性检查兼容的分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def can(v, coins):
    # each robot: (time, position)
    a_t = a_x = -1
    b_t = b_x = -1

    for t, x in coins:
        can_a = True
        can_b = True

        if a_t != -1:
            if abs(x - a_x) > v * (t - a_t):
                can_a = False
        if b_t != -1:
            if abs(x - b_x) > v * (t - b_t):
                can_b = False

        if not can_a and not can_b:
            return False

        if can_a and not can_b:
            a_t, a_x = t, x
        elif can_b and not can_a:
            b_t, b_x = t, x
        else:
            # both can take it: assign to the more flexible robot
            # heuristic: choose robot with smaller last time gap advantage
            if a_t < b_t:
                a_t, a_x = t, x
            else:
                b_t, b_x = t, x

    return True

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        coins = []
        for _ in range(n):
            t, c = map(int, input().split())
            coins.append((t, c))

        # quick impossibility check
        if n == 0:
            print(0)
            continue

        lo, hi = 0, 10**9
        ans = -1

        while lo <= hi:
            mid = (lo + hi) // 2
            if can(mid, coins):
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1

        print(ans)

if __name__ == "__main__":
    solve()
```该代码将可行性测试与 v 上的搜索分开。可行性函数维护每个机器人最后获取的硬币，并检查每个新硬币是否可达。 关键的实现细节是约束中的时差缩放，它可以正确地模拟运动限制。 

一个微妙的点是机器人状态的初始化时间为-1。 这有效地允许从任何起始位置取出第一个分配的硬币，因为在第一次分配之前没有强制执行距离约束。 

选择两个机器人都可以拿走一枚硬币的贪婪决胜规则是为了避免一个机器人过早挨饿。 基于上次的分配是一种简单的启发式方法，可以保留灵活性。

 (1, 1), (2, 10), (3, 2)

| 硬币| 机器人A | 机器人B | 作业 |
 | --- | --- | --- | --- |
 | (1,1) | 免费| 免费| 一个 |
 | (2,10) | | B能到达，A不能| 乙|
 | (3,2) | A能到达，B不能| | 一个 |

 这表明分裂如何避免一个机器人无法单独完成的大跳跃。 

现在考虑一个不可能的情况：

 (1, 1), (2, 10), (3, 1)

 对于小 v，机器人分配会失败，因为同一个机器人无法在单位时间间隔内在 1 和 10 之间跳转并返回到 1。 可行性函数最终将拒绝所有低于所需阈值的v，二分搜索收敛到最小有效速度。 

这些痕迹证实可行性取决于时间间隔，而不仅仅是空间距离。

 ## 边缘情况

 一个关键的边缘情况是所有硬币同时出现但位置相距很远。 在这种情况下，每个机器人在该时刻只能收集空间中的一个连续区域，并且无论速度如何，任何需要两个以上不相交位置的硬币组都是不可能的。 该算法正确地拒绝了这一点，因为两个机器人都从未定义的位置开始，并且无法在零时间差内传送到多个远端单元。 

另一个边缘情况是硬币在空间中形成锯齿形，单位时间间隙但空间跳跃较大。 尽管每次单独的跳跃可能是可行的，但如果一个机器人被迫在相反的方向上进行两次大的跳跃，交替分配仍然可能会失败。 可行性检查抓住了这一点，因为它强制每个机器人的轨迹在整个分配的子序列上的一致性。 

最后，单个机器人足够但贪婪分配分裂不正确的情况由灵活性规则处理。 如果一个机器人过早受到约束，则两个机器人随后的硬币可能会失败，因此分配策略必须避免过早地将机器人锁定在狭窄的轨迹中。
