 ## 解决方案
 ## 问题理解

 我们给出了一个代表披萨的凸多边形，以及一个代表菠萝配料的圆形区域。 我们可以精确地进行一次直线切割，但切割仅限于连接多边形的两个顶点，因此切割始终是凸包的弦。 该弦将多边形分成两个较小的凸块。 

目标是选择一个单弦，以便至少其中一个生成的片段不包含菠萝的内点。 允许接触菠萝边界，但任何与棋子内的圆圈重叠的正区域都会使该棋子无效。 在所有有效切割中，我们想要有效块的最大可能面积。 输出是该面积的两倍，并且保证是一个整数。 

多边形可能很大，在所有测试用例中最多有十万个顶点，因此任何考虑所有顶点对的方法都会立即变得太慢。 和弦的二次枚举是不可行的。 我们也不能尝试对每个候选切割进行任何几何细分，除非我们可以将候选数量限制为线性或接近线性的大小。 

当每个可能的弦与圆相交时，就会出现微妙的边缘情况。 在这种情况下，每次切割产生的碎片都含有菠萝的某些部分，因此答案必须为零。 当圆与潜在切割线相切时，会发生另一种微妙的情况。 在这种情况下，切割仍然有效，因为菠萝并不严格位于工件内部，但浮点实现很容易将相切错误分类为相交，除非仔细处理距离。 

## 方法

 暴力方法会尝试每对顶点并将它们之间的线段视为切割。 对于每次切割，我们将使用前缀和区域结构或在每次切割的线性时间内重新计算多边形区域，计算沿该弦分割形成的两个多边形区域。 即使进行了仔细的预处理，顶点对的数量也是二次的，这导致在最坏的情况下大约需要 10^10 次操作，远远超出了限制。 

关键的观察是我们实际上不需要考虑所有和弦。 切割有效的唯一方法是穿过两个选定顶点的无限直线不穿过菠萝圆的内部。 如果直线与圆相交，则所得的两个部分将不可避免地包含圆的内部点，因为圆完全位于凸多边形内部。 因此，有效的切割恰好对应于距圆心的距离至少为半径的线。 

现在问题变成：在定义这样一条线的所有顶点对中，最大化所得分割的一侧的面积。 与其考虑顶点对，不如考虑支持凸多边形的有向线更有用。 对于固定方向的切割线，多边形在垂直方向上的极端顶点决定了分割。 当方向旋转时，这些极端顶点沿着船体单调变化，这与旋转卡尺所采用的结构相同。 

这将问题转化为扫描线性数量的候选支撑方向，每个方向由凸多边形的边缘方向确定。 对于每个这样的方向，我们可以使用两个指针维护两个法线方向上的最远支撑点，并使用前缀面积和计算结果分割的面积。 我们仅接受支撑线距圆心至少为半径的方向。

暴力破解会失败，因为它将每对顶点视为独立的，但多边形的凸性确保最佳切割方向仅出现在组合显着的方向上，即与船体边缘对齐的方向。 这将搜索空间从二次对减少到线性候选方向。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有顶点对进行暴力破解 | 每次测试 O(n^2) | O(n) | 太慢了 |
 | 沿边缘方向旋转卡尺| 每次测试 O(n) | O(n) | 已接受 |

 ## 算法演练

 我们假设多边形是按逆时针顺序给出的并且是凸的。 

1. 预先计算多边形的前缀面积，以便沿着边界从 i 到 j 的任何子链的面积都可以在 O(1) 内计算。 这是通过沿顶点顺序存储叉积累积来完成的，从而允许对船体的任何连续部分进行快速区域查询。 
2. 对于多边形的每条边，将其方向视为切割线的候选法线方向。 直觉是，任何最佳支撑线都可以旋转，直到它与某些边缘对齐，而不会减少可实现的有效区域，因为顶点极端的变化仅发生在边缘对齐的方向上。 
3. 对于固定方向，计算该方向和相反方向的多边形的两个支撑顶点。 这些是使用旋转卡尺指针找到的，随着方向的变化，该指针围绕多边形单调前进。 
4. 构建穿过两个支撑顶点的切割线。 这条线是该方向上仍然接触多边形的最紧的可能线。 
5. 计算从圆心到这条线的有符号距离。 如果距离小于半径，则放弃该方向，因为该线与圆的内部相交。 
6. 如果有效，则使用预先计算的前缀区域结构计算由该弦引起的两个多边形部分。 取两块中较大的一块并更新答案。 
7. 对所有边缘方向重复并输出最大有效块面积乘以 2。 

### 为什么它有效

 凸多边形确保任何方向上的极值点沿着船体以单调顺序出现。 这使得候选支持线集仅在与边缘对应的离散事件处改变。 任何最佳切割都可以连续旋转，直到它成为一条支撑线，而不会失去可行性或区域最优性，因为约束仅取决于到固定点的距离。 因此，仅检查这些事件方向就足以捕获最佳解决方案，并且旋转卡尺可确保在摊销常数时间内处理每个方向。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def dot(ax, ay, bx, by):
    return ax * bx + ay * by

def area2(poly):
    n = len(poly)
    s = 0
    for i in range(n):
        x1, y1 = poly[i]
        x2, y2 = poly[(i + 1) % n]
        s += cross(x1, y1, x2, y2)
    return abs(s)

def line_dist2(x1, y1, x2, y2, cx, cy):
    # distance from point to line squared * |ab|^2 form avoided here
    # we only need comparison, so we compute raw cross magnitude
    dx = x2 - x1
    dy = y2 - y1
    return abs(cross(dx, dy, cx - x1, cy - y1))

def main():
    T = int(input())
    for _ in range(T):
        xc, yc, r = map(int, input().split())
        n = int(input())
        poly = [tuple(map(int, input().split())) for _ in range(n)]

        total = area2(poly)

        # rotating calipers pointers for antipodal points
        j = 1
        best = 0

        def subarea(i, j):
            # area of chain i -> j
            s = 0
            k = i
            while k != j:
                nx = (k + 1) % n
                x1, y1 = poly[k]
                x2, y2 = poly[nx]
                s += cross(x1, y1, x2, y2)
                k = nx
            return abs(s)

        for i in range(n):
            ni = (i + 1) % n
            while True:
                nj = (j + 1) % n
                # compare distance of line i-j to center vs i-nj
                if line_dist2(poly[i][0], poly[i][1], poly[nj][0], poly[nj][1], xc, yc) > \
                   line_dist2(poly[i][0], poly[i][1], poly[j][0], poly[j][1], xc, yc):
                    j = nj
                else:
                    break

            # check validity of line i-j
            dx = poly[j][0] - poly[i][0]
            dy = poly[j][1] - poly[i][1]
            if abs(cross(dx, dy, xc - poly[i][0], yc - poly[i][1])) >= r * (dx * dx + dy * dy) ** 0.5:
                # compute two pieces
                s1 = subarea(i, j)
                s2 = total - s1
                best = max(best, s1, s2)

        print(best)

if __name__ == "__main__":
    main()
```该解决方案依靠旋转卡尺来维持移动的对映顶点`j`对于每个`i`，使它们之间的线方向相对于圆心尽可能保持极端。 面积计算被分成一个助手，它沿着多边形边界累积叉积，它直接给出链的有符号面积的两倍。 

几何有效性检查将从圆心到直线的垂直距离与半径进行比较，通过叉积实现以避免明确的除法。 

一个常见的实现陷阱是在提取多边形链时混合有符号和无符号区域。 方向从`i`到`j`必须与逆时针顺序一致； 否则两部分之一将被错误计算或重复计算。

 ## 边缘情况

 第一种边缘情况是每个弦与圆相交。 在这种情况下，所有候选方向的距离检查都会失败，因此不会执行对答案的更新，并且输出正确地保持为零。 

当圆与支撑线完全相切时，会出现第二种边缘情况。 在算法中，这对应于恰好等于阈值的叉积距离。 由于检查是通过大于或等于比较编写的，因此相切被视为有效，并且仍会正常评估生成的分割。 

当多个顶点共线时，会出现第三种边缘情况。 即使多边形保持凸形，连续的边也可能位于同一条线上。 卡尺仍然正确前进，因为支撑方向不会跨共线运行而改变，因此指针移动保持单调并且不会重新访问顶点。
