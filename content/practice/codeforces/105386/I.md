 ## 解决方案
 ## 问题理解

 我们得到一个字符串，并且允许我们循环移位它。 选择移位后，我们查看排列成圆圈的结果字符串，这意味着最后一个字符被视为与第一个字符相邻。 对于旋转后的字符串，我们希望修改尽可能少的字符，以便没有两个相邻字符是相同的。 

通过一次操作，我们可以将任意位置更改为任意小写字母。 对于每个可能的旋转，我们测量需要多少变化才能使旋转的字符串“正确”，然后我们选择提供最少变化的旋转。 

在所有测试用例中，字符串长度最多可达 500,000，因此总大小的任何平方都会失败。 即使是每次旋转的线性时间也是不可能的，因为有 n 次旋转。 这立即排除了每次轮班从头开始重新计算答案的可能性。 

一个天真的误解是认为旋转会改变弦的难度。 例如，人们可能认为在不同点切割字符串可能会破坏或创建长串相同的字符，从而改变答案。 然而，由于在检查邻接性时字符串被认为是循环的，因此每次旋转都会保留相同的循环邻接结构，只是重新索引它。 

常见的边缘情况是完全统一的字符串，例如“aaaaaa”。 每次轮换看起来都是一样的，最好的策略是沿着周期交替进行更改。 另一种是由交替的长块组成的字符串，如“aaabbbccc”，其中每个块独立地贡献成本。 

## 方法

 蛮力策略会尝试每一次轮换。 对于每次移位，我们计算确保没有两个相邻字符匹配所需的最小修改次数。 对于固定字符串计算此值的直接方法是将其视为循环图，其中必须为每个位置分配一个与其相邻位置不同的新字符。 即使我们尝试使用动态编程来优化它，每次旋转仍然需要线性处理，导致总工作量为 O(n^2)。 

关键的观察是旋转字符串不会改变位置之间的循环邻接关系。 循环中的每对邻居在旋转后仍然是邻居，只是索引发生了移位。 这意味着“出现相等相邻对的位置”的结构在旋转过程中是不变的。 

一旦我们修复了旋转，问题就变成局部的：唯一重要的信息是相等的字符如何沿着循环形成连续的段。 在长度为 L 的相同字符段内，最佳策略是修改大约一半的字符，以便没有两个相邻字符保持相等。 这会产生与 L 成比例的成本，与旋转无关。 

由于旋转不会改变这些循环运行，因此每次移位的答案都是相同的。 因此，我们只需要在原始循环串上计算一次成本。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有旋转进行蛮力 | O(n²) | O(1) | O(1) | 太慢了 |
 | 单次循环运行 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将字符串视为循环字符串并将其压缩为连续相同字符的最大段。

1. 遍历字符串并将其概念上合并为一个循环，这意味着最后一个字符和第一个字符也被认为是相邻的。 如果它们相等，则它们属于同一运行，因此我们相应地旋转运行边界而不是分割它。 
2. 识别该循环意义上相同字符的所有最大游程。 每次运行都有一定的长度 L。 
3. 对于每次运行，计算必须更改多少个字符，以便没有两个相邻字符保持相等。 在统一运行中，我们最多可以保持所有其他角色不变，因此运行所贡献的成本是下限（L / 2）。 
4. 将所有运行中的这些贡献相加以获得最终答案。 

唯一微妙的部分是正确处理循环合并。 如果第一个和最后一个字符相等，则它们属于同一运行，并且必须在计算贡献之前合并。 否则，我们会错误地将单个循环运行视为两个单独的线性运行，这会低估所需的更改。 

### 为什么它有效

 该算法依赖于这样一个事实：每个邻接约束都是独立的，因为只有当两个相同的字符相邻时才会发生违规。 每个具有相同字符的最大块形成一个孤立区域，其中约束是密集的，并且最佳修复策略是该块的局部。 由于旋转不会改变哪些位置等于其循环邻居，因此分解为游程及其长度保持不变，这固定了每次旋转的成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve(s: str) -> int:
    n = len(s)
    if n == 1:
        return 0

    # build cyclic runs
    runs = []

    i = 0
    while i < n:
        j = i + 1
        while j < n and s[j] == s[i]:
            j += 1
        runs.append(j - i)
        i = j

    # merge first and last run if cyclicly connected
    if len(runs) > 1 and s[0] == s[-1]:
        runs[0] += runs[-1]
        runs.pop()

    # compute cost
    ans = 0
    for L in runs:
        ans += L // 2
    return ans

def main():
    t = int(input())
    for _ in range(t):
        s = input().strip()
        print(solve(s))

if __name__ == "__main__":
    main()
```该实现首先将字符串压缩为连续的等字符串。 这是在单次扫描中完成的，因此它以线性时间运行。 唯一重要的步骤是当第一次和最后一次运行共享相同字符时合并它们，这确保尊重字符串的循环性质。 

一旦形成游程，每个游程都会贡献其长度的一半，四舍五入到答案。 这直接匹配在统一段内修复相邻重复项的最佳策略。 

## 工作示例

 ### 示例 1

 考虑`s = "aaabbaa"`。 

我们首先形成线性游程：`"aaa"`,`"bb"`,`"aa"`。 由于第一个和最后一个字符都是`'a'`，我们将第一个和最后一个运行合并为`"aaaaa"`和`"bb"`。 

| 步骤| 运行 | 行动|
 | --- | --- | --- |
 | 初始| [3,2,2]| 线性分割|
 | 合并 | [5, 2] | 第一个和最后一个合并 |
 | 成本| 2 + 1 | 楼层(5/2) + 楼层(2/2) |

 最终答案是3。 

这说明循环合并是必要的； 否则我们就会错误地对待结构。 

### 示例 2

 考虑`s = "abcde"`。 

所有字符都是不同的，因此每次运行的长度都是 1。 

| 步骤| 运行 | 行动|
 | --- | --- | --- |
 | 初始| [1,1,1,1,1] | 每个角色都被隔离|
 | 合并 | [1,1,1,1,1] | 无需合并 |
 | 成本| 0 | 所有楼层(1/2) |

 无需更改，因为该字符串已经有效。 

这表明，当不存在邻接违规时，该算法自然返回零。

 ## 边缘情况

 单字符字符串已经非常有效，因为没有相邻对，并且算法正确返回零，因为存在长度为 1 的单次游程。 

完全一致的循环字符串，例如`"aaaaa"`考虑循环邻接后形成一个合并游程。 该算法合并端点并计算下限（5/2），正确生成 2。在这种情况下，忘记循环合并将错误地生成 2 个单独的运行，并低估或误算贡献。 

像这样的字符串`"ababa"`循环中没有相等的相邻对。 游程分解仅产生长度为 1 的游程，因此每个贡献都为零，这与不需要修改的事实相匹配。
