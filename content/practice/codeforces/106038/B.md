 ## 解决方案
 ## 问题理解

 我们得到了一个按时间顺序排序的不同整数列表，其中每个整数代表贝尔纳多去健身房的一天。 目标是确定此列表中出现的最长连续日历日。 当天数相差 1 时，就会形成一条条纹，例如 5、6、7 形成长度为 3 的条纹，并且我们希望所有此类连续序列的最大长度。 

输入大小与记录的访问次数成线性关系，因此任何重复扫描数组或尝试从每个起点扩展间隔的解决方案仍然必须足够高效，能够在一秒钟内处理最多大约 200,000 个元素。 这排除了任何二次行为，因为在最坏的情况下，对已排序数据的嵌套扫描很容易超过 10^10 次操作。 

一种常见的失败模式来自于将问题视为我们需要考虑所有对或独立地为每个索引重新计算条纹长度。 例如，给定输入`1 2 3 4 10 11`，一种幼稚的方法可能从每个索引开始并重复向前检查，多次重新计算重叠的段，这变得多余且缓慢。 

另一个微妙的边缘情况是根本没有连续的元素。 例如，输入`5 10 20`应该返回 1，因为每一天都是独立的，长度为 1。 任何假设至少存在一对连续且错误地初始化答案的解决方案都可能在此失败。 

## 方法

 暴力破解的想法很简单：对于数组中的每个位置，将其视为连续线的起点并向前延伸，而下一个值恰好比当前值大 1。 这可以正确计算从每个索引开始的最长条纹，并在所有开始处取最大值得出正确的答案。 

然而，这种方法会重复扫描重叠的段。 在最坏的情况下，数组本身是单个长连续序列，例如`1 2 3 4 ... n`，第一个索引扫描 n 个元素，第二个索引扫描 n-1 个元素，依此类推，导致大约 n²/2 次比较。 当 n 很大时，这会变得太慢。 

关键的观察结果是，因为数组已经排序，所以连续的序列显示为连续的块。 我们不需要在每个索引处重新启动扫描，而是只需要在从左到右扫描一次时跟踪当前条纹的长度。 当连续元素之间的差值恰好为 1 时，我们就延长条纹； 否则，我们将重置它。 这消除了所有冗余的重新计算。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们一次性处理数组，同时保持当前连续条纹的长度和迄今为止看到的最佳答案。 

1. 初始化变量`best`为 1，因为任何单个元素都会形成长度为 1 的有效条纹。 这处理没有连续对的情况。 
2. 初始化`cur`为 1，表示当前条纹在第一个元素处结束的长度。 
3. 从第二个元素迭代到数组末尾。 在每个索引处`i`， 比较`a[i]`和`a[i-1]`。 
4. 如果`a[i] == a[i-1] + 1`, 增量`cur`增加 1 是因为连胜持续不断。 这直接编码了连续天数的定义。 
5. 否则，重置`cur`变为 1，因为连胜被打破并且新的连胜从位置开始`i`。 
6.更新后`cur`， 更新`best`作为`max(best, cur)`跟踪阵列中任何位置看到的最大条纹。 
7.循环结束后，输出`best`。 

正确性来自于维持每个索引处的不变性`i`,`cur`等于恰好结束于的最长连续条纹的长度`i`。 由于每个条纹都必须在某个位置结束，因此跟踪所有这些端点可以保证我们捕获全局最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = list(map(int, input().split()))
    if not data:
        return

    n = data[0]
    arr = data[1:]

    # In some inputs n may not be needed explicitly if all values are present
    # but we trust format: first number is n, followed by n values.
    if len(arr) < n:
        arr += list(map(int, input().split()))

    if n == 0:
        print(0)
        return

    best = 1
    cur = 1

    for i in range(1, n):
        if arr[i] == arr[i - 1] + 1:
            cur += 1
        else:
            cur = 1
        if cur > best:
            best = cur

    print(best)

if __name__ == "__main__":
    solve()
```该代码执行一次线性扫描。 唯一保留的状态是当前的连胜长度和迄今为止的最佳结果。 比较`arr[i] == arr[i-1] + 1`是整个决策机制，其他一切都是围绕它进行记账。 

一个常见的实施错误是在序列中断时忘记重置当前条纹，这会错误地累积跨间隙的值。 另一个是初始化`best`为 0，当数组至少有一个元素但没有连续的对时，这将失败。

 ## 边缘情况

 一个关键的边缘情况是数组大小为一时。 该算法初始化两者`cur`和`best`为1，所以直接输出1，不进入任何循环迭代。 用于输入`7`，状态保持不变，答案正确为1。 

另一种情况是整个序列严格递增但不连续，比如`1 3 4 6`。 算法重置`cur`在每次中断时，确保不会发生无效合并。 决赛`best`由于配对而变为 2`3 4`，这是正确的最长连续线段。 

最后一个微妙的情况是最长的条纹位于开始或结束时。 自从`best`每一步后都会更新，不仅在休息时更新，在结尾时也会更新`1 2 3 4`或者`10 11 12 13`无需特殊的后处理即可完全捕获。
