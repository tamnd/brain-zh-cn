 ## 解决方案
 ## 问题理解

 我们得到了圆括号的循环排列。 该字符串仅包含`'('`和`')'`，并且左括号和右括号的总数相等。 我们可以获取字符串的前缀并将其移动到末尾，从而有效地旋转字符串。 任务是选择一个旋转，使结果序列成为有效的括号字符串，并输出我们需要从前面移动多少个字符。 

这里有效的括号序列意味着，如果我们从左到右扫描，我们在任何前缀处看到的右括号永远不会多于左括号，并且最后计数匹配，这已经由输入保证了。 

关键约束是字符串长度达到典型的 Codeforces 限制（隐式很大，因此需要线性或近线性解决方案）。 尝试每次旋转并从头开始检查有效性的二次方法会太慢，因为每次检查的时间复杂度为 O(n)，总体上为 O(n²)。 

当多次旋转产生有效序列时，会出现微妙的边缘情况。 例如，`"()()"`已经有效，因此旋转 0 是正确的，但旋转 2 也可以。 该问题允许任何有效答案，因此我们只需要一次正确的旋转。 

另一种边缘情况是弦在平衡方面完全颠倒，例如`"))((()"`。 仅检查总计数的天真检查将会失败，因为中间前缀很重要，而不仅仅是全局计数。 

## 方法

 蛮力策略会尝试所有可能的轮换。 对于每个旋转索引 i，我们构造旋转后的字符串，并通过从左到右扫描并跟踪平衡来检查它是否有效。 每次有效性检查的成本为 O(n)，并且有 n 次旋转，时间为 O(n²)。 这对于大 n 来说太慢了。 

问题的结构表明我们应该避免从头开始重新计算有效性。 重要的观察是，括号序列的有效性仅取决于前缀平衡，并且旋转只是移动我们开始读取相同循环序列的位置。 因此，我们可以考虑双倍或循环视图上的前缀和，而不是重建字符串。 

我们将余额定义为 +1`'('`和 -1 为`')'`。 对于从位置 i 开始的旋转，我们需要段中的所有前缀和`[i, i+n-1]`（循环地）保持非负值。 

一个关键的想法是找到开始遍历最有可能避免负前缀和的位置。 这可以通过跟踪原始阵列上的最小前缀平衡并选择在累积平衡曲线中的最低点之后开始的旋转来识别。 这确保了重新居中时路径不会低于零。 

这将任务简化为单个线性扫描计算前缀平衡并定位最小前缀和的索引。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(n) | 太慢了|
 | 最佳（前缀最小旋转）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 根据括号将字符串转换为 +1 和 -1 值的数组。 这将问题转化为跟踪累积和而不是字符串结构，这使得旋转效应更容易推理。 
2. 从左到右扫描字符串的同时计算前缀和。 保持运行总计，其中每个`'('`增加总和并且每个`')'`减少它。 这表示序列在每个点上高于或低于有效性阈值的程度。 
3. 跟踪前缀和达到最小值的索引。 该点对应于整个序列中平衡的最深点。 直觉是这是循环结构中的最弱点。 
4. 选择旋转，使该最小点正好位于新字符串的开头之前。 具体来说，如果最小前缀出现在索引 i 处，我们将旋转 i+1 个位置。 
5. 输出该旋转索引。 如果存在多个有效选择，则任何达到最小值的位置都有效，因为所有此类轮换都避免了负前缀和。 

### 为什么它有效

 前缀和曲线描述了从零开始到零结束的步行。 有效的括号序列恰好是永远不会低于零的游走。 当我们旋转序列时，我们实际上是在循环行走中选择了一个新的起点。 最佳起点是紧随全局最小值之后的起点，因为这会将整个曲线向上移动，从而使最低点变为零而不是负值。 由于所有其他前缀都高于或等于此最小值，因此旋转版本中的任何前缀都不能为负值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

s = input().strip()
n = len(s)

min_prefix = 0
min_idx = -1

pref = 0

for i, ch in enumerate(s):
    if ch == '(':
        pref += 1
    else:
        pref -= 1

    if pref < min_prefix:
        min_prefix = pref
        min_idx = i

# rotate so that min prefix position moves to end of prefix
print(min_idx + 1)
```该代码在单次计算中计算累积余额，同时跟踪最低值出现的位置。 变量`min_idx`存储前缀和最小的最后一个位置。 在该索引之后旋转可确保序列从平衡始终非负的点开始。 

一个常见的微妙之处是逐一处理：我们旋转`min_idx + 1`， 不是`min_idx`，因为我们希望新字符串立即在最低前缀点之后开始。 

## 工作示例

 ### 示例 1：`"))((()"`我们一步步计算前缀余额。 

| 我| 字符 | 前缀 | 最小前缀 | 最小 idx |
 | --- | --- | --- | --- | --- |
 | 0 | ) | -1 | -1 | 0 |
 | 1 | )| -2 | -2 | 1 |
 | 2 | ( | -1 | -2 | 1 |
 | 3 | ( | 0 | -2 | 1 |
 | 4 | ) | -1 | -2 | 1 |
 | 5 | ( | 0 | -2 | 1 |

 索引 1 处的最小前缀为 -2，因此我们旋转 2。旋转后的字符串为`"((())("`，这是有效的。 

该迹线表明，即使出现多个负倾角，最深的倾角也决定了正确的旋转点。 

### 示例 2：`"()()"`| 我| 字符 | 前缀 | 最小前缀 | 最小 idx |
 | --- | --- | --- | --- | --- |
 | 0 | ( | 1 | 0 | -1 |
 | 1 | ) | 0 | 0 | -1 |
 | 2 | ( | 1 | 0 | -1 |
 | 3 | ) | 0 | 0 | -1 |

 最小前缀始终为 0，因此`min_idx = -1`。 我们输出`0`，意味着不需要旋转。 该字符串已经有效。 

这表明该算法自然地处理已经平衡的序列，而无需特殊的外壳。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    s = input().strip()
    n = len(s)

    min_prefix = 0
    min_idx = -1
    pref = 0

    for i, ch in enumerate(s):
        pref += 1 if ch == '(' else -1
        if pref < min_prefix:
            min_prefix = pref
            min_idx = i

    return str(min_idx + 1)

# provided samples
assert run("))((()\n") == "2"
assert run("()\n") == "0"
assert run("))((\n") == "2"

# custom cases
assert run("()") == "0", "already valid"
assert run(")(()") == "1", "single correction via rotation"
assert run("((()))") == "0", "fully nested already valid"
assert run(")))(((") == "3", "large imbalance symmetric"

 ## 边缘情况

 一种边缘情况是字符串已经有效并且永远不会低于零。 在这种情况下，所有点的最小前缀都为零，因此`min_idx`遗迹`-1`，输出变为`0`。 该算法正确地识别出不需要旋转。 

另一种边缘情况是最小前缀出现在最后一个字符处。 在像这样的字符串中`"((()))"`或任何已经平衡的有效形式，这仍然会导致`min_idx = -1`，因此结果是稳定的并且不会错误地建议旋转。 

更微妙的情况是多个位置共享相同的最小前缀值。 该算法存储最后一次出现的情况，但任何出现的情况都会产生有效的旋转，因为在任何最深的下降之后移动都会产生非负的重新居中行走。
