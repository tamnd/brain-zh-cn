 ## 解决方案
 ## 问题理解

 我们得到了一系列代表交易的整数，这些整数被逐一累积成一个运行总和。 计算该总和的机器具有由参数确定的固定整数范围$k$，因此运行总计必须始终保持在零附近的对称区间内。 如果在任何时候部分和超出此间隔，计算就会溢出并变得无效。 

任务不是计算总和，而是决定是否可以对给定的数字重新排序，以便每个前缀总和保持在允许的范围内。 如果存在这样的顺序，我们必须输出它； 否则我们必须报告不可能。 

输入由位限制参数组成$k$，数组的大小$n$，以及数组本身。 隐式约束是所有中间前缀和必须保持在由下式确定的固定范围内$k$，其行为类似于有符号整数容量。 

关键的困难在于重新排序会极大地改变前缀和。 即使总和有效，错误的排序也会暂时将前缀和推到安全区间之外。 这使得问题从根本上是关于控制部分总和，而不是最终结果。 

幼稚的方法可能会尝试所有排列或模拟没有结构的贪婪选择。 对于大的情况，这会立即中断$n$，因为阶乘增长是不可能的，甚至每次排列的二次检查也会太慢。 

当大的正值和负值同时存在时，就会出现微妙的故障情况。 例如，提前放置所有大的正数可能会超出上限，即使稍后存在大的负数可以进行补偿。 同样，首先放置负数可能会在正数到达之前降至下限以下。 正确的解决方案必须仔细平衡极端情况，而不是将数组视为可自由重新排序。 

## 方法

 暴力策略将生成数组的所有排列，并检查每个排列的前缀和是否在范围内。 这是正确的，因为它直接强制执行约束，但它需要$n!$排列和$O(n)$验证每个排列，导致不可行$O(n \cdot n!)$复杂。 即使是为了$n = 10$，这变得不切实际。 

关键的观察是，只有每一步的运行总和的大小才重要，危险来自极端。 大的正值威胁上限，而大的负值威胁下限。 这表明我们应该始终按照使前缀和尽可能远离边界的顺序放置元素。 

实现此目的的标准方法是将数字分为正数和负数，然后始终选择相对于当前总和最安全的元素。 直观上，当总和为正值或接近上限时，我们应该更喜欢负值来将其拉回，而当总和为负值或接近下限时，我们应该更喜欢正值。 

这将问题从组合搜索转变为受控贪婪构造。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \cdot n!)$|$O(n)$| 太慢了|
 | 贪婪平衡构造|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ## 算法演练

 1. 将数组分为两组：非负数和负数。 我们将根据当前总和在这两个池之间切换来控制前缀总和。 
2. 非负数按降序排序，负数按升序排序。 这确保我们始终首先考虑最“极端”的安全举措，因为极值是最有效地纠正前缀和中的漂移的值。 
3. 将运行总和初始化为零并准备一个空结果列表。 
4. 在每一步中，确定当前总和是否更接近上限或下限。 如果它更接近上限，我们更喜欢取最小的可用负数将其向下拉。 如果它更接近下限，我们更愿意取最大的可用正数将其向上推。 
5. 在选择一个数字之前，请检查该数字的相加是否使总和保持在允许的范围内。 如果首选的选择违反了界限，请尝试另一组。 如果两个选择都无效，则构造失败并且不存在有效的排序。 
6. 继续，直到使用所有元素。 

该决策规则之所以有效，是因为它始终使用可用的最有效的纠正措施。 大数值被保留用于需要防止前缀和漂移到无效区域的情况。 

### 为什么它有效

 该算法保持不变，即在每一步之后，剩余的未使用的数字仍然足以完成序列而不违反界限。 每个选择都是为了通过将总和推离当前更接近的边界来降低下一步的风险。 由于每一步都会优先考虑下一步的可行性，因此任何未能放置元素的情况都意味着当时没有任何排列可以在不违反约束的情况下放置它。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = input().split()
    if not data:
        return
    k = int(data[0])
    n = int(data[1])
    arr = list(map(int, data[2:]))

    limit = 1 << k

    pos = sorted([x for x in arr if x >= 0], reverse=True)
    neg = sorted([x for x in arr if x < 0])

    i = j = 0
    res = []
    s = 0

    for _ in range(n):
        placed = False

        # decide direction
        if s >= 0:
            # try negative first
            if j < len(neg) and limit >= s + neg[j] >= -limit:
                s += neg[j]
                res.append(neg[j])
                j += 1
                placed = True
            elif i < len(pos):
                if -limit <= s + pos[i] <= limit:
                    s += pos[i]
                    res.append(pos[i])
                    i += 1
                    placed = True
        else:
            # try positive first
            if i < len(pos) and -limit <= s + pos[i] <= limit:
                s += pos[i]
                res.append(pos[i])
                i += 1
                placed = True
            elif j < len(neg):
                if -limit <= s + neg[j] <= limit:
                    s += neg[j]
                    res.append(neg[j])
                    j += 1
                    placed = True

        if not placed:
            print("N")
            return

    print("S")
    print(*res)

if __name__ == "__main__":
    solve()
```该解决方案首先按符号分隔值，然后对它们进行排序，以便首先考虑最有影响力的元素。 模拟贪婪地进行，总是尝试根据当前总和最大程度地降低违反界限的风险的移动。 

关键的细节是我们从不盲目消费元素。 每个候选者在被接受之前都会根据溢出范围进行验证，从而始终确保前缀安全。

 ## 边缘情况

 关键的边缘情况是当单个大量级元素主导边界时。 在这种情况下，仅放置顺序就决定了可行性。 该算法正确地处理了这个问题，因为它立即拒绝任何会溢出间隔的步骤。 

当正值和负值的大小交替时，就会出现另一种边缘情况，迫使池之间重复切换。 贪婪规则确保总和始终向区间的安全一侧移动，从而防止过早承诺一个符号。 

第三种情况是数组严重倾斜，例如许多大的正数和一些小的负数。 这里，算法会延迟消耗大的正数，直到实现足够的平衡，从而防止早期溢出，即使总和是安全的。
