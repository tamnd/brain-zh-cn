 ## 解决方案
 ## 问题理解

 我们正在追踪四位固定候选人的投票情况：海龟拉斐尔、莱昂纳多、多纳泰罗和米开朗基罗。 每个候选人已经拥有一定数量的选票，并且还有一些尚未投票的剩余选票。 剩余的每一票都将投给四位候选人中的一位，但我们不知道它们将如何分配。 

只有当所有剩余选票都以对该候选人最有利的方式分配后，候选人最终获得的票数严格超过最终总票数的一半，该候选人才能获胜。 任务是确定哪些海龟仍然有理论路径成为唯一的多数获胜者。 

输入大小在结构上是恒定的，因为总是有四个候选者。 唯一不同的幅度是剩余选票的数量，该数量可能很大。 这立即排除了对复杂数据结构或模拟投票的任何需要。 任何试图枚举剩余选票分布的解决方案本质上都是指数分布的，即使对于中等值也是不可能的，因为每个剩余选票独立地选择四个候选人之一。 

当当前没有候选人明显领先但剩余选票不足以为任何人创造多数时，就会出现微妙的边缘情况。 例如，如果所有选票都已投完，但没有人超过一半，则不可能有获胜者。 另一个特殊情况是，一名候选人已经取得了巨大的领先优势，但如果可以最佳地重新分配剩余选票，该候选人仍有可能被超越。 

考虑这种情况：当前投票数为 10 0 0 0，剩余投票数为 5。 总数变为 15，因此多数需要至少 8 票。 Rafael已经有10个了，所以他基本上满足了条件，但我们仍然必须仔细检查其他人。 莱昂纳多、多纳泰罗和米开朗基罗需要全部剩余的 5 票加上额外的赤字弥补，这是不可能的。 

另一种情况：1 1 1 1，还剩 4 票。 总数变为 8，多数门槛为 5。每位候选人可以获得全部 4 票剩余选票，最终为 5 票，因此所有人仍然有可能获胜。 认为只有现任领导人才重要的天真解释在这里是不正确的。 

## 方法

 蛮力的想法是将剩余的每张选票分配给四名候选人并模拟所有可能的结果。 每票有 4 个选择，所以这会产生$4^r$可能性在哪里$r$是剩余票数。 即使是为了$r = 20$，这已经超过了十亿种配置，使其不可行。 

关键的观察是我们不需要模拟分布。 对于任何固定的候选人，我们只需要知道是否存在某种剩余选票的分配，使他们能够超过最终总数的一半。 对于候选人来说，最好的情况是所有剩余选票都投给他们，因为将选票投给其他人只会让他们的任务变得更加困难。 这将问题简化为对每个候选人进行直接可行性检查。 

我们将最终总票数计算为所有当前票数加上剩余票数的总和。 中奖门槛严格超过这个总数的一半。 对于每个候选人，我们检查他们当前的票数加上所有剩余的票数是否严格大于此阈值。 如果是，他们仍有获胜的能力； 否则，他们将被淘汰。

 ## 算法演练

 我们将问题简化为评估每个候选人的简单不平等。 

1. 读取当前四票数和剩余票数。 我们还通过将四个当前值相加并添加剩余票数来计算所有投票完成后的总票数。 
2. 对于每位候选人，假设他们获得了所有剩余选票，计算出可能的最佳最终得分。 这是对该候选人最有利的情况，并给出了其可实现结果的上限。 
3. 将多数阈值计算为最终总票数的一半，并要求严格大于该值。 这种严格性很重要，因为平局不会产生赢家。 
4. 对于每个候选人，检查他们的最佳情况分数是否超过多数阈值。 如果是这样，我们将他们标记为仍有获胜机会。 
5. 按字母顺序输出所有此类候选者。 如果没有人符合条件，则输出表示没有获胜者的字符串。 

### 为什么它有效

 对于任何候选人来说，将剩余选票分配出去只会减少其最终总票数，而无论分配如何，总票数保持不变。 由于获胜条件仅取决于相对于固定阈值的最终总数，因此候选人的最佳结果是通过给予他们所有剩余的选票来实现的。 如果即使这个最大场景也未能超过多数阈值，那么没有其他分布可以帮助他们，因为任何替代分配都会严格降低他们的分数而不改变阈值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    r, l, d, m, rem = map(int, input().split())
    names = [("Donatello", d), ("Leonardo", l), ("Michelangelo", m), ("Rafael", r)]

    total = r + l + d + m + rem
    threshold = total // 2  # must be strictly greater

    res = []

    for name, cur in names:
        if cur + rem > threshold:
            res.append(name)

    if not res:
        print("sem vencedores")
    else:
        res.sort()
        for x in res:
            print(x)

if __name__ == "__main__":
    main()
```该解决方案首先读取五个整数并将候选人与他们当前的选票分组。 最终总票数只计算一次，因为它不取决于剩余票数的分配方式。 阈值是使用整数除法得出的，我们在检查可行性时依赖于严格的不等式。 

通过将所有剩余选票添加到当前计数中，对每位候选人进行独立评估。 这直接实现了最佳情况推理。 排序可确保按字母顺序输出，因为不能保证输入中的候选顺序与所需的输出顺序匹配。 

一个常见的陷阱是使用`>=`而不是`>`。 该问题严格要求超过一半，因此平等并不构成胜利。

 ## 边缘情况

 一个重要的边缘情况是没有剩余选票的情况。 例如，输入`5 4 3 3 0`导致固定的结果，无法改变分数。 该算法计算出的总数为 15，阈值为 7。由于所有当前分数都低于或等于此阈值，因此没有候选人通过严格条件。 检查正确地将每个候选人评估为`cur + 0 > threshold`，没有产生赢家。 

另一个边缘情况是所有候选人平等并且有足够的剩余选票来形成多数。 为了`1 1 1 1 4`，总数变为 8，阈值为 4。每个候选者都有最佳情况 5，超出阈值，因此全部打印。 该算法自然地处理这个问题，因为它独立评估每个候选者，而不假设当前排名很重要。
