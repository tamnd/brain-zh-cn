 ## 解决方案
 ## 问题理解

 我们得到一个整数对的集合。 从每一对中，我们可以精确地选择一个数字。 在每对做出一个选择后，我们获得了一组选定值。 任务是找到一个大于 1 的整数来整除每个选定的值。 除了找到满足该整除条件的任何有效整数之外，我们并不试图最大化或最小化任何东西。 如果不存在这样的整数，则答案为-1。 

关键的观点是，我们并不是在寻找所有数字的单一全局除数。 相反，对于每一对，我们可以通过第一个或第二个元素“路由”需求。 这种灵活性使得问题变得不那么简单：除数必须至少命中每对中的一个元素，但不一定在所有对中都命中相同的位置。 

高达 150,000 的约束 n 迫使任何解接近线性或线性。 尝试检查所有数字的所有约数或独立因式分解每个数字的策略将会失败，因为数字高达 2×10^9，如果在没有结构的情况下重复，则朴素因式分解可能会太慢。 

当每对做出贪婪选择而不考虑全局一致性时，就会出现微妙的失败情况。 例如，始终选择较小的数字或始终选择第一个元素可能会错过有效的解决方案。 另一个常见的陷阱是尝试计算一列或两列中所有数字的最大公约数，这会失败，因为有效除数可能会在每对列之间切换。 

## 方法

 一个直接的蛮力想法是尝试每个大于 1 的整数并检查它是否可以作为弱公约数。 对于每个候选 x，我们扫描所有对并验证每对中至少有一个元素可被 x 整除。 这是正确的，但不可行。 可能的 x 值的范围是巨大的，检查 n 对之间的整除性会导致在最坏的解释中大致为 O(maxA × n) 行为，这远远超出了限制。 

关键的结构观察是任何有效答案都必须除以每一对中的至少一个数字。 这意味着，如果我们为每对选择一个对除数“负责”的数字，那么答案必须是所有选定数字的除数。 这将问题转化为寻找一致的选择，其中存在大于 1 的共同 gcd。 

这表明了一种贪婪的传播思想：从第一对中选择一个候选者，例如a1或b1，并假设它对最终的gcd有贡献。 然后，对于其他每一对，我们被迫仅保留与该候选者兼容的值，这意味着我们可以通过将其与 ai 或 bi 相交来更新正在运行的 gcd。 如果在任何时候两个选择都破坏了 gcd（变为 1），则候选者失败。 

我们只需要尝试两个初始种子：从 a1 开始或从 b1 开始。 如果存在有效的解决方案，则这些种子中至少有一个将与正确的选择模式保持一致，因为第一对必须将其元素之一贡献给任何有效的构造。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 对所有整数进行暴力破解 | O(U·n) | O(U·n) | O(1) | O(1) | 太慢了 |
 | 尝试两个贪婪的 gcd 传播 | O(n log A) | O(n log A) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们尝试两次构建有效的解决方案，一次假设第一对中的第一个元素是解决方案的一部分，一次假设第二个元素是解决方案的一部分。

1. 从等于所选起始值的候选 gcd 开始。 该值表示必须划分到目前为止所有选定元素的当前约束。 
2. 迭代每对 (ai, bi)。 对于当前对，检查 ai 是否可以被当前 gcd 候选整除，或者 bi 是否可以被它整除。 如果两者可整除，我们可以保持 gcd 不变，并在概念上选择任一侧。 如果只有一种有效，我们就被迫选择那个。 
3.如果ai和bi都不能被当前gcd候选整除，我们必须修改候选。 我们不会立即强制失败，而是通过采用 gcd(candidate, ai) 或 gcd(candidate, bi) 来重新计算候选 gcd，具体取决于哪个选择仍然可能可行。 
4. 更新后，如果两个选项都将兼容性降低到 1，则此起始假设失败，我们提前停止此尝试。 
5. 如果我们处理完所有对并且 gcd 候选值大于 1，我们将其输出。 

我们从第一对的另一个元素开始重复相同的过程。 

### 为什么它有效

 该算法保持当前 gcd 候选者与迄今为止所有处理对中的有效选择一致的不变性。 在每一步中，我们仅转换为仍然可以从当前对中划分有效所选元素的值。 如果两个选项都强制 gcd 降至 1，则任何延续都无法恢复大于 1 的有效除数，因为未来的选择只能进一步限制整除性，而不能创建新的公因数。 这确保了处理所有对后任何幸存的候选者都是有效的弱公约数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import gcd

def check(start):
    g = start
    for a, b in pairs:
        if a % g == 0 or b % g == 0:
            if a % g == 0:
                continue
            else:
                continue
        ga = gcd(g, a)
        gb = gcd(g, b)
        if ga == 1 and gb == 1:
            return 0
        g = ga if ga > 1 else gb
    return g if g > 1 else 0

n = int(input())
pairs = [tuple(map(int, input().split())) for _ in range(n)]

ans = check(pairs[0][0])
if not ans:
    ans = check(pairs[0][1])

print(ans if ans > 1 else -1)
```该解决方案定义了一个模拟函数，该函数假设起始值对最终除数有贡献。 然后它会遍历所有对，同时保持正在运行的 gcd 约束。 

关键的实现细节是处理两个值都不能被当前 gcd 整除的情况。 在这种情况下，我们过渡到 gcd 减少步骤。 这是必要的，因为真正的答案可能是当前候选值的除数而不是候选值本身。 如果没有此后备，算法将错误地拒绝有效配置。 

我们显式地尝试第一对的两个元素，因为任何有效的解决方案都必须使用其中一个作为构造的初始锚点。

 ## 边缘情况

 当真实答案不是对中第一个元素的除数但通过第二个元素仍然有效时，就会出现关键边缘情况。 双启动策略通过尝试两个锚点来处理此问题，确保不会错过任何有效配置。 例如，如果正确的除数是 5 并且第一对是 (6, 10)，则从 6 开始会很快失败，但从 10 开始允许传播。 

另一个边缘情况是需要提前减少 gcd 时。 如果起始值限制太多，直接整除性检查将失败，但 gcd 转换会恢复隐藏的除数。 这是安全的，因为通过 gcd 减少只会删除与未来对不一致的素因数，而不会引入无效因数。 

最后，所有对都是单独互质组合的情况会导致立即终止。 一旦一对的 gcd 减少都达到 1，未来的调整就无法恢复大于 1 的有效除数，因为所有未来的选择都受到相同的约束结构的限制。
