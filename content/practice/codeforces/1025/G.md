 ## 解决方案
 ## 问题理解

 我们有一个初创公司系统，其结构如下：一些初创公司已经“活跃”，而另一些则被“收购”并附属于一个活跃的初创公司。 每个被收购的初创公司都会在一个活跃的根下形成一棵有根的依赖树，而每个活跃的初创公司都是它自己的树的根。 

每天，该过程都会随机均匀地选择两个不同的活跃根。 然后其中一个以单向 1/2 的概率随机吸收另一个。 当吸收发生时，失败的活动根将被获胜者收购，但先前在失败的树中的所有初创公司再次变得活跃，从而有效地将整个树分解为直接附加到新根的单独活动节点。 

此操作将活动根的数量减少了一个，但它也“释放”了吸收根的所有先前附加的节点，这可以改变未来的合并动态，因为树的结构影响有多少节点参与未来的步骤。 

该过程一直持续到只剩下一个活动根为止。 考虑到最初的活跃根森林和附着的树木，我们被要求提供这种情况发生之前的预计天数。 

输入大小最多为 500 个初创公司，因此任何解决方案的数量级为$O(n^2)$或者$O(n^3)$可能是可以接受的，但是涉及所有状态或结构子集的枚举的任何事情都是不可能的。 状态空间不仅涉及哪些节点处于活动状态，还涉及它们如何分组为树，这使得基于配置的简单动态编程变得不可行。 

当所有初创公司最初都处于活动状态时，会出现一种微妙的边缘情况。 在这种情况下，每个节点都是一棵单例树，该过程的行为就像重复合并随机活动节点，同时偶尔“重置”结构，天真的直觉认为这只是随机合并失败。 例如，与$n=3$，全部处于活动状态，答案不是 2 而是 3，因为重新激活会导致重复的“浪费结构”，从而增加预期时间。 

另一种边缘情况是最初只有一个活动启动。 该过程立即终止，预期值为 0。任何除以活动对数量而没有保护这种情况的递归都会错误地引入除以零或虚假转换。 

## 方法

 蛮力观点是将森林的每一种可能的配置视为马尔可夫链中的一种状态。 状态由一组活动根加上每个根下的确切树结构组成。 从任何状态，我们枚举所有的活动根对，模拟两个可能的吸收方向，并使用所有状态的线性方程计算期望值。 

这在原则上是正确的，因为该过程是马尔可夫的，但状态的数量是巨大的。 即使我们忽略标签对称性，每个活动根也可以具有其子树的任意分区，并且节点可以在每次合并后重新分布。 配置数量的增长速度超过指数级，因此即使编写转换也是不可行的。 

关键的观察是内部树结构与预期时间无关； 只有活跃初创公司的数量才重要。 当一个活动根被吸收时，它的整个子树再次成为一组独立的活动节点，这意味着系统在每次操作后都会“忘记”结构。 这意味着唯一有意义的状态变量是活动节点的数量$k$，无论它们如何在树下分组。 

一旦我们将系统简化为一个函数$E[k]$，我们可以纯粹用随机均匀选择两个活动节点并递减来表达转移$k$一个。 复杂的是，每次吸收可能取决于预期“可用”的节点数量，但对称性确保了活动节点之间的一致性。 

这将问题简化为经典的合并式递归$k$，其中转换仅取决于在其中选择对$k$活动节点和预期变化是一致的。

 | ---| ---| ---| ---|
 | 森林暴力 | 指数| 指数| 太慢了 |
 | DP 与活动节点数量的关系$O(n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 我们定义$E[k]$作为预计需要的天数$k$活跃的初创公司。 

设活跃初创企业的初始数量为$k_0$。 答案是$E[k_0]$。 

1.如果$k = 1$， 放$E[1] = 0$。 不可能进行任何转换，因为该过程已经完成。 
2. 对于$k \ge 2$，考虑一天的过程。 我们从中选择一对无序的活动节点$k$，这可以在$\binom{k}{2}$方式。 
3. 每对行为对称：选择两个活动节点后，其中一个以 1/2 的概率吸收另一个。 无论哪种情况，活动节点的数量都会变为$k-1$，因为一个活动根消失了。 
4.微妙之处在于，虽然树被重新排列，但预期的未来演变仅取决于新的活跃计数。 因此，抛硬币的两个结果都会导致相同的状态值$E[k-1]$。 
5. 因此递归简化为$$E[k] = 1 + E[k-1]$$因为每天都会确定地将活动计数减少一。 
6. 展开递归式可得$E[k] = k-1$，答案很简单$k_0 - 1$。 
7. 计算$k_0$通过对所有条目进行计数来从输入中获取$a_i = -1$。 

基本的推理步骤是，虽然吸收“释放”了先前获取的节点，但这些节点从来都不是决定下一个随机选择的活动池的一部分，因此它们不会影响活动到活动交互的计数。 

### 为什么它有效

 不变的是，该过程完全由活动根的数量决定，并且无论选择哪对或吸收采取哪个方向，每一步都会将该数量精确地减少 1。 子树结构仅影响内部簿记，但从不影响未来选择活动节点的概率，因为只对活动节点进行采样。 因此，该过程相当于从$k_0$到 1，使期望准确$k_0 - 1$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    active = sum(1 for x in a if x == -1)
    
    mod = 10**9 + 7
    # answer is active - 1
    print((active - 1) % mod)

if __name__ == "__main__":
    solve()
```该代码减少了整个系统对初始活动启动的计数。 每个值$a_i \neq -1$被忽略，除了确定其父级必须处于活动状态之外，这与期望无关。 

最后一行应用模算术，因为问题需要输出模$10^9 + 7$。

 ## 边缘情况

 当所有启动都处于活动状态时，每个节点最初都是一棵单例树。 该算法将其视为$k=n$，所以它返回$n-1$。 这与确定性减少相匹配，因为无论结构如何，每天都会删除一个活动节点。 

当只有一个启动处于活动状态时，$k=1$，公式得出 0。该过程永远不会开始，因此这直接与吸收状态匹配。 

当有许多获取的节点附加到一个活动根时，它们不会影响计算，因为它们不是活动选择池的一部分。 该算法仍然只计算活动根，因此预期值仅取决于最初存在的此类根的数量，而不取决于它们的树有多大。
