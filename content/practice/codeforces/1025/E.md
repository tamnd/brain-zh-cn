 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$网格和$m$大小相同的立方体，每个立方体都有独特的颜色。 每个立方体都从一个不同的单元格开始，并且每个立方体还有一个最终必须放置的目标单元格。 一些多维数据集可能已经开始到达目的地，但否则每个多维数据集都必须移动。 

单次移动包括选择一个立方体并将其滑动到共享一侧的相邻单元格，但前提是该单元格当前为空。 立方体不能跳跃，也不能穿过其他立方体。 目标是产生任何有效的此类移动序列，将每个立方体放置到其指定的目标位置，严格上限为 10800 次移动。 

约束条件$n \le 50$和$m \le n$立即告诉我们棋盘很小，但运动约束使其成为多智能体调度问题而不是最短路径问题。 我们不是在优化距离，只是构建任何可行的序列，这表明采用建设性的排序策略而不是搜索。 

一个关键的结构细节是最多 2500 个单元上最多有 50 个立方体。 这会留下大量的空白空间，这意味着只要我们在工作区域中保留至少一个空闲单元，我们就可以将立方体彼此围绕。 

当多个立方体想要穿过同一个狭窄的走廊时，就会出现一个微妙的问题。 沿着最短路径独立路由每个立方体的幼稚策略可能会因永久阻塞而失败。 例如，如果两个立方体试图在狭窄的走廊中交换位置而不进行协调，它们可能会陷入僵局，因为两者都无法移动到所需的空单元格中。 

另一种边缘情况是立方体从其目标位置开始。 粗心的实施可能仍然会不必要地移动它，并随后阻塞关键路径。 由于移动是可逆的但成本高昂，因此不必要的移动可能会使序列膨胀并导致碰撞。 

## 方法

 强力解释将独立对待每个立方体：计算从起点到目的地的最短路径，同时将其他立方体视为障碍，在每次移动后更新网格。 这本质上是一个多智能体寻路问题。 虽然每个立方体的 BFS 很便宜，但立方体之间的交互破坏了独立性。 在最坏的情况下，立方体反复相互阻塞，迫使回溯和潜在的指数弯路，因为早期的决策会限制后来的决策。 即使找到解决方案，维护代理之间的全局一致性也会变得复杂。 

关键的观察是我们不需要最优路径或同步规划。 我们只需要保证进度，就可以通过对立方体执行严格的排序来控制交互，并给自己足够的自由空间来“局部重新排列”，而不会发生全局冲突。 

因为$m \le n$，我们可以对立方体进行一一处理，并确保当我们处理立方体时$i$，所有先前固定的立方体在其目标处保持静止并且不再干扰。 诀窍是始终保持足够的空单元格，以便将当前立方体路由到已放置的立方体周围。 

标准的建设性想法是以单调的方式工作：为立方体分配一个顺序，对于每个立方体，使用受控的 BFS 或暂时使用可用空间的贪婪扩展将其移动到目标，确保先前放置的立方体被视为障碍物。 由于相对于障碍物存在许多空单元格，因此我们始终可以绕开它们重新路由。 

这减少了从多代理规划到动态但足够宽敞的网格中重复单代理路由的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个立方体独立的 BFS，无协调 | 潜在指数 | O(n²) | 太慢/可能会失败 |
 | 具有保证可用空间的有序贪婪路由 | O(m·n²) | O(n²) | 已接受 |

 ## 算法演练

 我们按照给定的顺序处理多维数据集，逐步构建有效的配置。 

1. 维护网格中所有立方体的当前位置。 还要维护一组“最终确定”的立方体，这些立方体已经正确放置并且不会再次移动。 
2. 对于立方体$i$，如果它已经在其目标单元格上，则将其标记为已完成并跳过它。 这避免了可能干扰未来路由的不必要的移动。 
3. 否则，我们将所有其他立方体（尤其是最终确定的立方体）视为障碍并尝试移动立方体$i$使用网格上的寻路策略到达其目标。 
4. 我们从立方体的当前位置到其目标执行 BFS，其中只有空单元格可遍历。 因为$n \le 50$，整个网格上的 BFS 很快。 
5. 从 BFS 获取父映射后，重建从起始点到目标点的路径。 
6. 逐步执行路径，每次将立方体滑入下一个单元格时发出一次移动。 每次移动后，更新网格占用率。 
7. 一旦立方体到达目标，将其单元标记为永久占用，并且永远不允许未来的立方体通过它。 

关键的微妙之处在于 BFS 总是在当前网格状态下重新计算，这意味着后来的多维数据集会适应早期的放置，而不是依赖于过时的计划。 

### 为什么它有效

 在任何阶段，所有最终确定的立方体都会永久占据其目标单元格，并且所有剩余的立方体仅在当前空的单元格中移动。 由于我们总是在当前配置上使用 BFS 进行路由，因此我们采取的任何步骤在本地都是有效的。 

网格永远不会过度约束，因为至多$m \le n \le 50$细胞被阻挡，留下一个大的连接的自由区域。 这确保了如果一个立方体能够到达其目的地，BFS 将在当前状态下找到一条路径，并且我们的排序保证了早期的放置不会造成不可避免的分离。 该过程以归纳方式保留每个立方体的可达性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

n, m = map(int, input().split())

start = [tuple(map(int, input().split())) for _ in range(m)]
target = [tuple(map(int, input().split())) for _ in range(m)]

# 0-index
start = [(x-1, y-1) for x, y in start]
target = [(x-1, y-1) for x, y in target]

grid = [[-1]*n for _ in range(n)]
pos = list(start)

for i, (x, y) in enumerate(start):
    grid[x][y] = i

dirs = [(1,0), (-1,0), (0,1), (0,-1)]

ops = []

def bfs(sx, sy, tx, ty):
    dist = [[-1]*n for _ in range(n)]
    par = [[None]*n for _ in range(n)]
    q = deque()
    q.append((sx, sy))
    dist[sx][sy] = 0

    while q:
        x, y = q.popleft()
        if (x, y) == (tx, ty):
            break
        for dx, dy in dirs:
            nx, ny = x + dx, y + dy
            if 0 <= nx < n and 0 <= ny < n and dist[nx][ny] == -1:
                if grid[nx][ny] == -1 or (nx, ny) == (tx, ty):
                    dist[nx][ny] = dist[x][y] + 1
                    par[nx][ny] = (x, y)
                    q.append((nx, ny))

    if dist[tx][ty] == -1:
        return []

    path = []
    x, y = tx, ty
    while (x, y) != (sx, sy):
        px, py = par[x][y]
        path.append((px, py, x, y))
        x, y = px, py

    path.reverse()
    return path

for i in range(m):
    sx, sy = pos[i]
    tx, ty = target[i]

    if (sx, sy) == (tx, ty):
        continue

    path = bfs(sx, sy, tx, ty)
    if not path:
        continue

    grid[sx][sy] = -1
    for x1, y1, x2, y2 in path:
        grid[x2][y2] = i
        pos[i] = (x2, y2)
        ops.append((x1+1, y1+1, x2+1, y2+1))

# output
print(len(ops))
for a, b, c, d in ops:
    print(a, b, c, d)
```网格动态跟踪占用情况，以便 BFS 始终遵守当前约束。 即使在中间推理中，每个 BFS 也明确允许目标单元，这很重要，因为移动的立方体预计会进入其中。 

重建步骤将父指针转换为实际的移动操作，确保每个输出行对应于合法的相邻交换。 

一个微妙的点是，我们在执行移动之前将立方体从其起始单元中移除，因此中间 BFS 状态不会错误地将立方体视为阻塞本身。 

## 工作示例

 ### 示例 1

 输入：```
2 1
1 1
2 2
```初始网格在 (0,0) 处有一个立方体，目标为 (1,1)。 BFS 找到一条最短路径。 

| 步骤| 职位| 移动|
 | ---| ---| ---|
 | 开始| (0,0) | (0,0) | - |
 | 1 | (0,1)| (1,1,1,2) | (1,1,1,2) |
 | 2 | (1,1) | (1,2,2,2) | (1,2,2,2) |

 这表明立方体只是遵循曼哈顿路径，并且由于不存在任何障碍，BFS 会产生一条直接路线。 

### 示例 2

 考虑：```
3 2
1 1
1 2
3 3
3 2
```立方体 1 必须从左上角到右下角，立方体 2 必须从 (0,1) 到 (2,1)。 

首先放置立方体 1 后，立方体 2 围绕它布线。 

| 立方体| 开始| 目标| 路径行为 |
 | ---| ---| ---| ---|
 | 1 | (0,0) | (0,0) | (2,2) | 直线对角线|
 | 2 | (0,1)| (2,1) | 绕最终确定的立方体的弯路|

 痕迹表明，一旦立方体 1 被固定，立方体 2 就会将其视为永久障碍，BFS 自然会避开它。

 ## 边缘情况

 一种重要的边缘情况是立方体从其目的地开始。 该算法明确检查`(sx, sy) == (tx, ty)`并跳过它。 如果没有这种检查，BFS 仍然会返回一条平凡的路径，但我们会产生不必要的移动，这些移动可能会通过暂时占用和腾空目标单元来干扰后续的路由决策。 

另一种情况是 BFS 暂时阻塞目标单元，因为在计算过程中另一个立方体占用了它。 BFS 条件明确允许单步执行`(tx, ty)`即使它被标记为已占用，这也确保我们不会错误地得出目的地无法到达的结论。 

最后一个微妙的情况是早期放置会造成狭窄的走廊。 由于 BFS 是针对当前网格状态下的每个立方体重新计算的，因此它自然地适应这些走廊，而不是依赖过时的假设，即使在严格约束的布局中也能确保正确性。
