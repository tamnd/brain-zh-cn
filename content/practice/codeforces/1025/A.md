 ## 解决方案
 ## 问题理解

 我们得到一个代表一行小狗颜色的字符串，其中每个字符都是从“a”到“z”的颜色。 目标是确定我们是否可以使用非常具体的操作来转换该字符串，以使所有字符变得相同。 

该操作允许我们仅在当前出现至少两次的情况下选择颜色 x，然后将每次出现的 x 重新着色为我们选择的其他颜色 y。 我们可以多次重复此操作。 

重要的不是重新着色的确切顺序，而是我们最终是否能够消除除一种颜色类别之外的所有颜色类别。 

关键的限制是，只有当颜色的频率至少为 2 时，它才可用于重新着色。出现一次的颜色永远不能直接选择作为重新着色的源，因此单例的行为就像“锁定”元素，除非它们被其他操作间接吸收。 

当 n 达到 100,000 时，任何解决方案都必须在线性时间或近线性时间内运行。 这立即排除了每次重新着色后模拟操作或重复扫描字符串的可能性，因为在对抗性情况下这些可能会降级为 O(n²)。 

当所有角色都不同时，就会出现微妙的极端情况。 例如，`abcdef`。 每种颜色都会出现一次，因此无法执行任何操作。 答案显然是“不”。 

另一个极端情况是所有字符都已经相同，例如`aaaaa`。 这里我们根本不需要任何操作，所以答案是“是”。 

一个更有趣的情况是，当只有一种颜色的频率大于一种并且所有其他颜色都是单例时，例如`aab`。 即使一种颜色可用，但有些颜色永远无法消除，因为没有其他颜色有足够的频率被选为重新着色链中的源。 

## 方法

 暴力解释将尝试模拟操作。 在每一步中，我们都会扫描字符串，选择频率至少为 2 的任何颜色，并将所有出现的颜色重新着色为另一种颜色。 每次操作后，我们都会重新计算频率并重复，直到所有字符都相等或无法进行操作。 

原则上这是可行的，因为它直接遵循规则，但每一步都需要重新计算整个字符串的频率，并且我们最多可以执行 O(n) 操作。 每次重新计算的成本为 O(n)，因此最坏情况的复杂度变为 O(n²)，这对于 n = 100,000 来说太慢了。 

关键的观察结果是操作不依赖于位置，只依赖于频率计数。 该过程可以被认为是重复合并颜色类，但只有大小至少为两个的类才能启动合并。 

这导致了一个更简单的不变量：如果存在至少一种颜色出现多次，那么我们可以将它用作“收集器”颜色，并最终将所有其他颜色吸收到其中，因为一旦我们选择了目标颜色 y，它在接收元素时就不需要满足任何约束。 唯一的要求是每一步至少有一种源颜色的频率≥2，直到所有合并完成。 

因此，我们陷入困境的唯一情况是一开始没有颜色的频率≥2。 在这种情况下，无法执行任何操作，因此我们永远无法统一字符串，除非它已经统一。 

因此，问题简化为一项简单的检查：如果字符串已经有一个不同的字符，则回答“是”。 否则，检查是否有任何字符频率至少为2。如果是，则回答“是”，否则回答“否”。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n²) | O(n) | 太慢了|
 | 频率检查 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们继续分析弦的频率结构。 

1. 统计从'a'到'z'每个字符出现的频率。 这捕获了与操作相关的所有信息，因为位置并不重要。 
2. 检查存在多少个不同的字符。 如果正好有一个，则该字符串已经是统一的，所以不需要任何操作，我们可以立即返回“Yes”。 这是有效的，因为目标条件已经满足。 
3. 如果存在多个不同的字符，则扫描是否有字符至少出现两次。 如果这样的字符存在，我们返回“Yes”，因为它可以作为至少一个操作的有效源。 
4. 如果没有字符出现频率≥2，则返回“No”。 在这种情况下，每个角色都是唯一的，因此无法应用任何操作，也无法开始合并过程。 

### 为什么它有效

 关键属性是更改配置的唯一方法是选择频率至少为 2 的源颜色。 如果不存在这种颜色，则系统完全冻结。 如果至少存在一种，我们总是可以用它来吸收其他颜色，因为目标颜色没有限制。 这意味着单个“活动”颜色类别足以驱动系统实现完全统一。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    freq = [0] * 26
    for ch in s:
        freq[ord(ch) - 97] += 1

    distinct = sum(1 for x in freq if x > 0)
    if distinct == 1:
        print("Yes")
        return

    for x in freq:
        if x >= 2:
            print("Yes")
            return

    print("No")

if __name__ == "__main__":
    solve()
```由于仅存在 26 个字符，频率数组将整个输入压缩为恒定大小状态。 支票为`distinct == 1`直接处理已经统一的情况。 第二个循环检测是否可以启动任何有效操作； 没有这样的启动器，任何转变都是不可能的。 

不会出现排序问题，因为我们从不模拟操作，只测试存在条件。

 ## 边缘情况

 对于输入`a`，频率数组包含单个非零条目。 该算法检测到`distinct == 1`并立即返回“是”，正确处理小事。 

为了`abcd`，所有频率均为 1。算法跳过第一个条件，发现没有`freq >= 2`，返回“No”，匹配无法执行任何操作。 

为了`aab`，频率是`a:2, b:1`。 该算法检测到有效源并返回“是”，这反映出单个可复制颜色的存在足以驱动转换过程。
