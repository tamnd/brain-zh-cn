 ## 解决方案
 ## 问题理解

 该问题描述了一个非常小的“会计系统”，其中输入中的每条记录代表一组货币交易，任务是按顺序处理所有这些记录后确定最终的净余额。 

每个输入行对应于影响帐户余额的单个操作。 有些操作会增加余额，其他操作会减少余额，有些操作可能代表中性操作，具体取决于记录的结构。 输出是按顺序应用每个操作后的最终结果余额。 

从计算的角度来看，结构故意简单：我们不要求随着时间的推移优化查询或维护多个帐户，而只是模拟效果的线性累积。 

约束足够小，简单的模拟就足够了。 即使我们假设最多 10^5 次操作，单次求和或更新值仍然很好地保持在典型限制内，因为在 2 秒约束下，使用常数时间更新的 O(n) 工作是微不足道的。 任何预处理或高级数据结构的尝试都将是不必要的开销。 

此类问题的主要微妙之处通常来自于正确解释符号更改或确保不会意外跳过或重复应用任何操作。 常见的边缘情况是正确处理空操作或零效果操作。 

例如，如果输入表示：```
+10
-3
+5
```正确的输出是`12`。 误读符号或错误地修剪输入的幼稚实现很容易产生`18`或者`-18`，特别是如果解析不正确。 

另一种边缘情况是所有操作都取消时，例如：```
+7
-7
```正确的输出是`0`。 任何假设严格为正运行总计的实现都会在这里失败。 

## 方法

 暴力解释是将每一行视为一个事务，并在每次更新后从头开始重复重新计算总数。 这意味着对于每个操作，我们都会再次扫描所有先前的操作，对它们求和以计算当前余额。 这是正确的，因为它直接遵循问题的定义：余额始终是截至该点的所有操作的总和。 

然而，在最坏的情况下，这会导致加法次数的二次方。 对于 n 次操作，我们将执行 1 + 2 + ... + n 次加法，即 O(n²)。 对于 n 大约 10^5 来说，这变得完全不可行。 

关键的观察是每次重新计算总和是多余的。 每个运算只对最终结果贡献一个固定值，加法是结合律。 这意味着我们可以维护一个运行总计并逐步更新它。 我们不需要重新计算所有内容，而是只需应用每个操作一次并累积其效果。 

这将问题从重复聚合减少到对输入进行单遍折叠。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n²) | O(1) | O(1) | 太慢了|
 | 最佳 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们处理每行一次并维护一个代表当前余额的累加器。 

1. 初始化变量`balance = 0`。 这表示应用任何操作之前的帐户状态。 
2. 从输入中一一读取每个操作。 
3. 将运算解析为有符号整数值。 
4. 直接将该值添加到`balance`，更新帐户状态。 
5、所有操作处理完毕后，输出`balance`。 

选择每个步骤都是为了确保我们永远不会存储不必要的中间状态。 运行总和充当整个历史的压缩表示。 

### 为什么它有效

 正确性依赖于这样一个事实：最终结果纯粹是独立操作的相加。 每个操作都将其值准确地贡献给总数，并且除了通过求和之外，没有任何操作依赖于任何其他操作。 这创建了一个不变量：处理前 k 个操作后，`balance`等于这些 k 值的总和。 由于这在每一步都成立，因此在处理完所有 n 个操作之后，该值正是问题所需的总和。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    data = sys.stdin.read().strip().split()
    balance = 0

    for x in data:
        balance += int(x)

    print(balance)

if __name__ == "__main__":
    main()
```该解决方案使用快速批量输入读取，因为该问题自然是可流式处理的。 将输入拆分为标记可以避免逐行开销，并确保即使对于大型输入，解析也保持高效。 

核心设计选择是使用单个整数累加器。 除了这个变量之外，不需要数组或状态跟踪。 每个令牌都会立即转换并合并到结果中，从而确保恒定的内存使用。 

一个微妙的细节正在使用`sys.stdin.read()`而不是重复`readline()`来电。 虽然两者都是 O(n)，但批量读取避免了 Python 级别的循环开销，并且在输入大小较大时更加稳健。

 ## 边缘情况

 一个关键的边缘情况是所有操作完全取消。 对于输入：```
5 -5
```该算法逐步处理：

 余额从 0 开始，在第一个令牌之后变为 5，然后在第二个令牌之后返回到 0。 不变式是`balance`即使中间值波动，等于前缀和也能确保正确性。 

另一种边缘情况是所有值均为负数时：```
-2 -3 -4
```运行总计变为 -2，然后是 -5，然后是 -9。 不需要特殊处理，因为 Python 整数自然支持负累加。 

最终的边缘情况是单元素输入，例如：```
42
```该算法将余额初始化为 0 并立即更新一次，生成 42。前缀和不变量保持不变，因为在这种情况下第一个和最后一个状态是相同的。
