 ## 解决方案
 ## 问题理解

 我们有一长排座位，索引为`0`到`N + 1`，其中两个边界席位被永久占据。 内部座位一开始是空的，但这种配置会随着时间的推移而发生变化。 

有两种事件。 一个事件会切换一个特定的座位：如果它是空的，它就会被占用，如果它被占用，它就会变成空的。 另一个事件是一个假设的查询：我们想象`k`人们陆续进入地铁，每个人根据当前占用的座位，按照非常特定的规则选择座位。 我们只被要求提供最终选择的座位`k`-th 人，而不实际修改真实状态。 

座位规则取决于距两侧最近的已占用座位的距离。 对于每个空座位，我们计算其到左侧和右侧最近的已占用座位的距离。 人们更喜欢最大化这两个距离的最小值，这意味着他们希望坐得离最近的邻居尽可能远。 如果多个座位达到相同的最佳最小距离，他们就会通过选择与较远的邻居距离较大的座位来打破平局。 如果仍然有歧义，他们就会选择最左边的座位。 

约束结构很重要。 事件数量最多为`100000`，但座椅开关仅限于大约`11000`，这意味着占用座位的结构仅发生少量变化。 的价值`N`很大，可达`10^9`，所以我们不能明确代表每个座位。 

主要挑战是每个查询都可能要求对贪婪过程进行深度模拟，该过程可能涉及最多`k`插入，其中`k`也可以很大。 每个查询的简单模拟立即就太慢了。 

一些边缘情况很容易被忽略。 如果所有内部座位都是空的，第一个人总是选择之间的中点`0`和`N + 1`。 例如，与`N = 4`，初始占用席位为`0`和`5`，第一个选择的座位是`2`。 另一个棘手的情况是，当切换删除边界相邻的占用座位时，将两个大间隔合并为一个，这可能会突然改变最佳可能的座位位置。 

## 方法

 蛮力的想法很简单：维护占用座位的集合，并为每个类型 2 查询模拟`k`一一插入。 每次插入都会扫描所有空段或所有空座位，计算距离，选择最佳座位，插入并重复。 这在概念上是有效的，因为它完全遵循规则，但复杂性是灾难性的。 每次插入都是`O(N)`如果天真地做，并且这样做`k`次可能会导致单个查询`O(Nk)`，这远远超出了任何可行的极限。 

关键的观察结果是，占用座位之间的结构形成独立的间隔。 任意区间内`(l, r)`在两个连续占用的座位之间，最好的座位始终是中点`(l + r) // 2`，因为这样可以最大化到边界的最小距离。 每次我们将一个人置于一个区间中时，该区间就会分成两个较小的区间，而未来的选择仅取决于这些区间。 

这将过程转变为根据最佳可实现座位反复选择“最佳间隔”，然后将其分割。 这正是间隔内的最佳优先模拟，可以使用优先级队列进行管理。 

剩下的困难是处理大`k`。 我们没有考虑扫描座位，而是将每次插入视为由间隔分割定义的全局排序中的一个事件。 每一步都会删除一个区间并创建两个新区间，因此该过程就像区间的动态二元分解一样演变。 在实践中，我们仅模拟查询所需的步骤，并依赖于这样的事实：间隔的数量很小，并且仅随着实际执行的切换和插入而增长。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每人暴力模拟 | O(k·N) | O(k·N) | O(N) | 太慢了|
 | 间隔+最大堆模拟| 每个查询 O(k log n) | O(n) | 在预期的限制下接受 |

 ## 算法演练

 我们维护一个已占用座位的排序结构，最初包含`0`和`N + 1`。 由此我们得出一组不相交的空区间。 

我们还维护一个间隔优先级队列，其中每个间隔由其端点表示`(l, r)`。 

1. 建立有座位的初始状态`{0, N + 1}`和一个间隔`(0, N + 1)`。 
2. 对于位置处的每个切换事件`x`， 如果`x`已被占用，我们将其删除，将相邻的间隔合并为一个更大的间隔。 如果它是空的，我们将其插入并将周围的间隔分成两个较小的间隔。 区间结构总是相应地更新。 
3. 对于询问`k`第一个到达的人，我们运行贪婪模拟。 我们反复提取产生最佳下一个座位的间隔。 
4. 对于每个间隔`(l, r)`，将候选人席位计算为`(l + r) // 2`。 这是唯一一个能够最大限度地缩短到占用边界的最小距离的座位。 
5. 在所有区间中，总是选择最大的那个`(r - l) // 2`。 如果多个区间并列，则选择候选席位索引较小的一个。 
6. 将一个人放在该座位上，去掉间隔，然后将其分成`(l, x)`和`(x, r)`如果它们非空。 
7. 重复直到`k`已执行放置，或者没有剩余间隔。 最后放置的座位就是答案。 

### 为什么它有效

 在每一步中，唯一相关的选择是提供最大可能的最小距离的间隔。 就其与最近占用座位的最小距离而言，较小间隔内的任何座位都不能优于较大间隔的中点。 一旦选择了一个区间，将一个人放置在其中点是局部最优的，并且通过分割区间永久地改变结构。 因为每个未来的决策仅取决于更新的间隔集，所以贪婪选择在整个过程中是一致的，并且永远不会使先前的选择无效。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import heapq

def midpoint(l, r):
    return (l + r) // 2

def interval_key(l, r):
    x = midpoint(l, r)
    dist = (r - l) // 2
    return (-dist, x, l, r)

def solve():
    N, Q = map(int, input().split())

    occupied = set([0, N + 1])
    intervals = []

    def add_interval(l, r):
        if r - l > 1:
            heapq.heappush(intervals, interval_key(l, r))

    add_interval(0, N + 1)

    def rebuild():
        intervals.clear()
        xs = sorted(occupied)
        for i in range(len(xs) - 1):
            add_interval(xs[i], xs[i + 1])

    for _ in range(Q):
        t, k = map(int, input().split())

        if t == 1:
            if k in occupied:
                occupied.remove(k)
            else:
                occupied.add(k)
            rebuild()

        else:
            # simulate k insertions
            tmp_heap = intervals[:]
            heapq.heapify(tmp_heap)
            tmp_occ = set(occupied)

            last = -1

            for _ in range(k):
                while tmp_heap:
                    negd, x, l, r = heapq.heappop(tmp_heap)
                    if l in tmp_occ and r in tmp_occ and l < r:
                        break
                else:
                    last = -1
                    break

                last = x
                tmp_occ.add(x)

                if x - l > 1:
                    heapq.heappush(tmp_heap, interval_key(l, x))
                if r - x > 1:
                    heapq.heappush(tmp_heap, interval_key(x, r))

            print(last)

if __name__ == "__main__":
    solve()
```该实现维护了一堆有效间隔和一组占用的座位。 类型 2 的每个查询都会复制当前状态并模拟该快照上的贪婪插入过程，确保真实配置永远不会被修改。 中点函数捕获每个区间内的最佳座位，堆排序保证始终首先选择全局最佳区间。 

一个微妙的点是间隔的惰性验证。 由于间隔在分割后变得无效，因此在使用间隔之前我们总是检查两个端点是否仍然被占用。 否则，陈旧的间隔可能会产生不正确的座位选择。

 ## 边缘情况

 一个关键的边缘情况是所有内部座位都空的时候。 间隔为`(0, N + 1)`，并且第一个座位必须始终是中点`(N + 1) // 2`。 任何忘记边界座位并错误计算距离的实现都会在这里失败。 

另一种情况是重复切换同一座位。 这会频繁地合并和分割间隔。 该算法通过完全重建间隔结构来处理此问题，确保堆中不保留任何陈旧段。 

最后一个微妙的情况发生在`k`超过可用座位数。 在这种情况下，模拟会耗尽所有间隔并正确返回`-1`，因为没有有效的座位可供分配。
