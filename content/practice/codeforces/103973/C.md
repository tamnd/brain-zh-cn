 ## 解决方案
 ## 问题理解

 我们有两个有半径的圆$a$和$b$。 一个圆固定到位，另一个圆从外侧与其相切。 然后，第二个圆围绕第一个圆滚动，始终保持相切，并在绕固定圆完成一整圈后最终返回到起始位置。 

当这种运动发生时，滚圆也绕着自己的中心旋转。 任务是确定移动圆在绕固定圆旋转一整圈期间完成了多少次完整旋转。 答案不一定是整数，因此必须以约分数形式给出。 

输入由许多独立的测试用例组成，每个测试用例只是一对半径。 自从$T$可以大到$10^5$，每个测试用例必须在恒定时间内处理。 半径本身可以达到$10^{18}$，这立即排除了任何涉及运动模拟或沿路径迭代步进的方法。 

一个天真的解释可能会建议模拟滚动运动，沿着路径以微小增量跟踪旋转角度。 这会失败，因为路径是连续的，而且运动的规模需要大约$10^{18}$如果按半径尺度离散化，则为步骤。 

当两个半径相等时，会出现微妙的边缘情况。 例如，$a = b = 1$。 天真的直觉可能会说圆只滚动一次，但几何推理表明它实际上完成了两次完整的旋转。 这已经暗示了涉及比局部滚动速度更全局的因素。 

## 方法

 思考这个问题的强力方法是想象较小的圆沿着大圆的圆周移动并不断更新其旋转。 在沿着外部路径的每次无穷小的运动中，滚动圆的旋转与其所行进的弧长除以其自身的半径成正比。 这需要对一整圈长度的旋转进行积分$2\pi(a + b)$，因为移动圆的中心轨迹为半径的圆$a + b$。 

虽然这种推理原则上是正确的，但在限制下直接模拟或离散化它是不可能的。 关键的低效率在于我们需要处理整个连续路径，有效地进行无限数量的微小更新。 

关键的见解是该动议干净地分解为两个贡献。 首先，运动圆的中心沿着半径为圆的圆运动$a + b$，贡献旋转$\frac{a + b}{b}$。 其次，由于圆围绕另一个曲面而不是直线滚动，因此会产生额外的效果，这会导致中心路径每转一圈额外的完整旋转。 这是经典的“在闭合曲线上滚动而不滑动”现象，其中曲率增加了额外的$1$全轮。 

结合这些效果可得出总旋转：$$\frac{a + b}{b} + 1 = \frac{a + b + b}{b} = \frac{a + 2b}{b}$$然而，这尚未与样本行为一致，更仔细的几何分解表明，正确的不变量是对称的$a$和$b$。 我们观察到每个圆对有效滚动几何形状的贡献均等，而不是不对称地分开贡献。 

正确的最终公式简化为：$$\frac{a + b}{\gcd(a, b)} \Big/ \frac{b}{\gcd(a, b)} = \frac{a + b}{b} \text{ reduced properly via gcd}$$导出它的一种更稳定的方法是注意，在一个完整的旋转过程中，接触点所描绘的弧长对应于$2\pi(a + b)$，动圆每转一周对应弧长$2\pi b$。 因此原始旋转数为：$$\frac{a + b}{b}$$但由于路径闭合，当以分数表示时，实际计数必须减少到最低项，这就是为什么需要以减少形式输出的原因。 

因此，整个任务简化为计算一个分数并使用分子和分母的 gcd 来简化它。

 ## 算法演练

 1. 读取整数$a$和$b$对于每个测试用例。 这些定义了固定圆和滚动圆的几何形状，并完全确定了运动。 
2. 计算原始旋转数为$a + b$。 这代表在一次完整的行程中两个圆对滚动运动的有效总贡献。 
3. 将分母设置为$b$，因为滚动圆的每个完整旋转对应于与其半径成比例的弧长。 
4. 计算$g = \gcd(a + b, b)$。 此步骤通过删除分子和分母之间的任何共享比例因子来确保分数以其不可约形式表示。 
5、输出$\frac{a + b}{g} / \frac{b}{g}$。 这是代表旋转次数的减少分数。 

最后减少的关键思想是长度的几何比率决定旋转，并且只有相对缩放很重要，而不是绝对单位。 

### 为什么它有效

 滚动运动仅取决于弧长之比。 经过一整圈，移动圆的中心描绘出一条闭合曲线，其长度与$a + b$。 滚动圆通过除以自身的圆周尺度将行进的弧长转换为旋转，其与$b$。 这两个量之间的任何公约数都对应于运动中的重复结构，不会影响最终计数，这正是 gcd 去除捕获的内容。 这保证了所得分数既最小又准确。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from math import gcd

def solve():
    t = int(input())
    for _ in range(t):
        a, b = map(int, input().split())
        num = a + b
        den = b
        g = gcd(num, den)
        num //= g
        den //= g
        print(f"{num}/{den}")

if __name__ == "__main__":
    solve()
```该代码独立读取每个测试用例并直接计算约简分数。 唯一重要的操作是 gcd，它确保分数最小。 整数除法是安全的，因为即使相加，所有值也都在 64 位范围内。 

一个常见的错误是忘记减少分数，即使数值正确，这也会导致格式不正确。

def run(inp: str) -> str:

 ## 边缘情况

 对于$a = b$，算法产生分子$2a$和分母$a$，这减少到$2/1$。 gcd 步骤在这里至关重要，因为如果不减少，输出将错误地保留$2a/a$，这不是必需的格式。 

对于非常大的值，例如$a = 10^{18}, b = 1$，分子变为$10^{18} + 1$。 因为这是互质的$1$，最大公约数是$1$，分数保持不变。 该算法处理此问题时不会出现溢出问题，因为 Python 整数自然支持任意精度。 

什么时候$a$和$b$共享一个大的gcd，比如$a = 6, b = 4$，分子是$10$分母是$4$，并且 gcd 缩减正确地将其简化为$5/2$。 这证实了简化步骤不是装饰性的，而是结构上正确性所必需的。
