 ## 解决方案
 ## 问题理解

 我们有几堆石头，每堆都有一个正整数大小。 我们反复合并现有的两堆，直到只剩下一堆。 每次合并都会选择大小为 x 和 y 的两堆，将其删除，然后用大小为 x + y 的单堆替换它们，同时获得 x 乘以 y 的分数。 

任务是选择最大化整个过程中总累积分数的合并顺序。 

输入由堆数 n 和数组 a 组成，其中每个值描述初始堆大小。 输出是一个整数，表示执行 n-1 次合并后可达到的最大总分。 

n 最大为 100000 的约束立即排除了任何尝试模拟所有合并序列的解决方案。 即使区间上的二次动态规划方法也会失败，因为区间状态以 O(n^2) 的形式增长，并且每次转换都将涉及额外的计算。 ai 的值最多为 10000，因此算术运算在 64 位整数中是安全的，但主要困难在于组合。 

一个微妙的陷阱是假设像总是合并两个最小或两个最大堆这样的贪婪策略是有效的。 例如，对于堆 [1, 2, 3, 4]，首先合并 (1, 2) 会产生与首先合并 (3, 4) 不同的中间结构，并且最终分数在很大程度上取决于这些早期选择。 另一个失败案例来自于局部最优合并，它会降低未来的乘法潜力，例如过早地合并中等堆，从而失去了以后将它们与更大的聚合相乘的机会。 

## 方法

 蛮力解释很简单：在每一步中，选择任意一对桩，将它们合并，计算分数，然后在简化的多重集上递归。 这将探索初始数组上所有可能的二叉合并树。 由于有 (n − 1) 个合并步骤，并且在每个步骤中 O(n^2) 个可能的对选择，因此状态数量呈阶乘增长。 即使对多重集进行记忆化，状态空间也会非常大，因为不同的合并顺序会产生不同的中间多重集，并且这些多重集无法有效地压缩为小的 DP 状态。 

关键的见解是停止考虑合并顺序，而是重新解释流程积累的内容。 每次合并都会贡献 x·y，并且每个元素都会参与多次合并，因为它会被吸收到越来越大的堆中。 这建议跟踪每个原始石头在合并过程中与其他石头相互作用的频率。 

更结构化的观点是，该过程相当于构建一棵满二叉树，其叶子是初始堆。 每个内部节点对应一个合并，其贡献是左右子树之和的乘积。 展开这个表达式表明，每对原始桩只贡献一次，乘以它们的值在树结构中组合的次数。 最佳策略是在合并早期最大化大值的“加权共现”。 

这简化为经典的贪婪观察：始终首先合并两个最小的可用堆。 原因是，尽早合并两个小值可以使它们的乘积较小，同时为以后的合并保留较大的值，在以后的合并中，它们将乘以越来越大的累积和。 任何延迟合并小值的偏差都会迫使它们与更大的金额进行交互，从而增加成本，而不会在其他地方创造补偿收益。 

因此，最佳解决方案是重复提取两个最小的堆，合并它们，将它们的乘积添加到答案中，然后将它们的总和推回原处。

 ## 算法演练

 我们维护的结构始终允许我们有效地检索两个最小的桩。 最小堆是实现此目的的自然工具。 

1. 将所有堆大小插入到最小堆中。 这表示发生任何合并之前的当前可用组件。 
2. 当剩下不止一堆时，从堆中提取两个最小的元素 x 和 y。 这种选择是合理的，因为推迟这些最小值中的任何一个只会迫使它稍后与更大的累积值合并，从而增加其贡献。 
3. 计算合并成本 x · y 并将其添加到运行总分中。 这代表了将这两个组件结合起来的直接回报。 
4. 将 x + y 插入堆中。 这个新堆代表可以参与未来合并的合并组件。 
5. 重复，直到剩下一堆。 此时，所有合并都已被计算一次。 

### 为什么它有效

 在任何步骤中，考虑剩余的两个最小的堆 x 和 y。 任何最佳解决方案最终都必须合并所有桩，特别是 x 和 y 都将参与合并，直到它们消失。 如果 x 和 y 现在不合并在一起，则它们中至少有一个必须首先以某个值 z ≥ y 合并。 这会产生至少 x·z 或 y·z 的成本，当与同一合并树结构的重新排列进行比较时，两者都不小于 x·y。 将这种延迟的配对交换为较早的合并永远不会增加未来的成本，并且严格避免中间桩尺寸不必要的膨胀。 重复应用此交换参数会产生一个首先组合最小元素的结构，这正是堆贪婪过程。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    if n == 1:
        print(0)
        return
    
    heapq.heapify(a)
    total = 0
    
    while len(a) > 1:
        x = heapq.heappop(a)
        y = heapq.heappop(a)
        total += x * y
        heapq.heappush(a, x + y)
    
    print(total)

if __name__ == "__main__":
    solve()
```该解决方案是围绕最小堆构建的，该最小堆始终在对数时间内暴露两个最小的堆。 Heapify 在线性时间内构建初始结构。 每次迭代执行两次弹出和一次推送，全部为 O(log n)，并贡献一个合并成本。 

n = 1 的早期返回处理不发生合并的退化情况，确保答案为零而不是尝试无效的堆操作。 

核心循环严格地将堆的数量每次减少 1，保证在 n-1 次迭代后终止。

 ## 边缘情况

 最小边缘情况是 n = 1。对于单个堆，不会发生合并，得分为零。 该算法在堆操作之前显式返回 0，避免无效弹出。 

另一种边缘情况是所有值都相等，例如 [2, 2, 2, 2]。 堆总是返回相同的元素，因此合并顺序无关。 每个步骤都会产生可预测的增长，并且算法会累积一致的产品，而无需打破平局逻辑。 

最后一种微妙的情况是，一个值明显大于所有其他值，例如 [1, 1, 1, 10000]。 贪婪策略首先合并这些，产生大小为 2 和 3 的中间堆，确保 10000 仅在过程后期与相对较大的聚合相乘，这与最大化总贡献相一致。
