 ## 解决方案
 ## 问题理解

 我们在一个无限的棋盘上有两个国王。 一位国王属于独行者（白色），另一位国王属于柳叶（黑色）。 他们从白棋开始交替走棋，每个国王的移动方式就像标准的国际象棋国王一样，这意味着它可以走到八个相邻方格中的任何一个。 

还有一个额外的限制：一个国王不能移动到与另一个国王相邻的任何方格。 实际上，这意味着每次移动后两个国王必须始终保持严格大于 1 的切比雪夫距离。 

怀特的目标不是捕获或到达特定的正方形，而是最终到达具有任意大 x 坐标的某个位置，形式上是 (10^100, y) 形式的单元格。 由于棋盘是无限的，这相当于询问白色是否可以无限期地增加其 x 坐标而不会被阻挡。 布莱克的目标是防止这种情况成为可能。 

每个测试用例都给出了两个国王的初始坐标。 我们必须确定白方是否有强制策略，以最终永远逃到右边，假设双方都发挥最佳状态。 

这些约束允许最多 100,000 个测试用例，坐标的绝对值最多为 10^9。 这立即排除了游戏的任何模拟。 即使单个游戏也可以持续任意多次移动，并且每次移动都取决于全局策略而不是局部贪婪，因此任何解决方案都必须将问题简化为每个测试用例的恒定时间几何条件。 

一个微妙的一点是，移动受到距离的限制。 即使黑色在欧几里得术语中距离很远，如果它能够在白色前进太远之前将自己对齐到白色前面，它仍然可能能够进行干扰。 另一个重要的边缘情况是黑色在 x 方向上开始领先，但在 y 方向上明显偏移。 直觉可能表明黑棋领先时总是会获胜，但事实并非如此，因为垂直分离会阻止黑棋形成有效的阻挡线。 

## 方法

 对游戏的暴力解释将模拟两个国王的所有合法动作，并在无限游戏树上运行极小极大搜索。 每个状态由无限网格上的两个位置组成，并且在每个状态中，两名玩家最多可以进行八次可能的移动，并且由于邻接限制而具有额外的合法性限制。 即使有记忆，状态空间也是无界的，因为坐标没有界线并且国王可以任意漂移。 这使得暴力破解从根本上是不可能的。 

关键的观察是，唯一重要的是两位国王之间的相对几何形状。 白方的目标仅取决于黑方是否能够永久阻止 x 坐标的增加。 由于两个王的移动速度相同且移动能力相同，因此黑棋只有在x轴上始终保持在白棋前面的“阻挡位置”才能控制白棋。 

这将问题转化为理解黑色能否及时到达并保持与白色对齐的位置。 决定性因素是黑色能否减少水平间隙，同时补偿垂直间隙。 如果黑色相对于水平距离垂直距离太远，则在白色滑过之前它无法同时缩小间隙并在白色前面对齐。 

这导致了初始配置的简单几何条件：将水平距离与垂直偏移进行比较。

 ## 算法演练

 ### 1.提取坐标并计算相对位置

对于每个测试用例，我们在 (x1, y1) 处取白色，在 (x2, y2) 处取黑色。 我们计算 dx = x2 - x1 和 dy = y2 - y1。 这将问题转化为分析黑色是否最初位于右侧以及它垂直位移了多远。 

dx 的符号立即确定黑色是否已经在白色后面。 如果黑色在后面或在相同的 x 坐标处开始，它不能在前面建立永久的障碍，因为白色总是可以先向右移动。 

### 2.处理黑棋不领先的情况

 如果 dx <= 0，则白色至少已经与黑色一样远。 由于白棋先走且只关心x的增加，因此黑棋无法在白棋之前形成阻挡墙。 在这种情况下，白棋立即获胜。 

### 3. 当黑方领先时，比较垂直偏移与水平优势

 如果 dx > 0，黑色最初在前面。 现在唯一的问题是黑色能否足够快地与白色垂直对齐以阻止进展。 

我们比较 |dy| 与 dx. 值 dx 表示黑棋在水平方向上领先多少步，这也是白棋可能绕过其 x 位置之前可用的移动次数。 在此期间，黑色还必须纠正 |dy| 的垂直分隔。 将自身对齐到白色前面。 

如果|dy| < dx，黑棋有足够的时间进行垂直调整，并且仍然保持在白棋的前面，使其能够形成有效的屏障。 

如果|dy| >= dx，黑方无法及时缩小垂直差距并保持其水平领先，因此白方总能找到一条滑过去的路径并无限期地继续增加x。 

### 4. 决定获胜者

 如果黑棋领先并且 |dy| < dx，输出柳叶。 否则输出独自行走。 

### 为什么它有效

 关键的不变量是，阻挡白色需要黑色最终占据白色 x 边界上或之前的位置，同时在 y 上保持足够相邻以防止绕过。 由于两个国王每回合移动一步，因此水平进度对于黑棋来说是一个严格的时间预算，以修复垂直错位。 如果预算不足，白色总是可以通过利用剩余的水平松弛来保持 x 的前进进度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        x1, y1, x2, y2 = map(int, input().split())
        dx = x2 - x1
        dy = y2 - y1

        if dx <= 0:
            print("Walk Alone")
        else:
            if abs(dy) < dx:
                print("Salix Leaf")
            else:
                print("Walk Alone")

if __name__ == "__main__":
    solve()
```该解决方案读取每个测试用例并将其减少到两个差异。 第一个决策点检查黑色是否已经在 x 方向领先。 如果不是，白方就微不足道地获胜，因为黑方无法建立前方障碍。 

当黑色领先时，唯一相关的比较是水平领先和垂直间隔之间。 严格的不平等`abs(dy) < dx`至关重要的是：平等对于黑色来说是不够的，因为在边界情况下，白色总是可以保持足够的水平进度以防止完全对齐，从而破坏阻挡策略。 

每个测试用例的所有操作都是恒定时间，因此该解决方案可以轻松处理 100,000 个输入。

 ## 边缘情况

 当黑色开始领先但大小与垂直偏移完全匹配时，例如 (0,0) 和 (2,2)，算法将其分类为白色获胜，因为 abs(dy) 并不严格小于 dx。 在这种情况下，黑棋纠正垂直错位的能力非常严格，无法同时保持向前的阻挡位置，从而让白棋在x方向继续前进。 

当黑色在 x 后面时，例如白色在 (2,0) 处，黑色在 (-1,0) 处，dx 为负值，算法立即返回独行。 尽管黑棋可能会向白棋移动，但它永远不会成为持续的前方阻挡者，因为白棋总是在 x 方向上保持主动。
