 ## 解决方案
 ## 问题理解

 我们有一棵建筑物树。 每栋建筑都是一个节点，每条道路都是一条边，该结构保证任意两个节点之间都存在一条简单路径。 

两人同时开始：一人从节点开始$a$，另一个从节点开始$b$。 每分钟，两者都可以停留或沿着一条边移动到相邻节点，并且它们以相同的速度移动。 如果在同一分钟内，他们处于同一节点或以相反方向穿过同一边缘，则移动的人会立即被抓住，并且该过程会停止。 

其中之一正试图最大限度地增加她在被抓之前设法参观的不同建筑物的数量。 任务是计算她可以访问的不同节点的最大可能数量，假设双方都处于最佳状态。 

关键的结构性约束是$n \le 10^6$，这会立即排除任何二次方，例如所有对最短路径或每个起始位置的重复 BFS 模拟。 任何解本质上都必须是线性或线性对数的。 

当两个玩家沿着之间的独特路径对称移动时，就会出现微妙的失败情况。$a$和$b$。 例如，在简单的一行中$1 - 2 - 3 - 4$， 如果$a = 1$和$b = 4$，天真的想法可能会认为跑步者可以自由地“分支”，但实际上，交汇点约束迫使沿路径进行确定性交互，并且当他们的距离相遇时，捕捉就会发生。 

另一个极端情况是当$a = b$。 在这种情况下，捕获是立即的，并且不可能进行任何移动，因此答案为零。 

最后，因为当它们在同一分钟内以相反方向穿过同一边时也会发生捕获，所以任何正确的推理都必须有效地将它们的移动视为沿着树度量的连续距离减少，而不仅仅是离散节点占用。 

## 方法

 一个蛮力的想法是逐步模拟两个玩家的动作。 在每个时间单位，我们考虑跑步者的所有可能的动作和追逐者的所有可能的响应，跟踪所有可到达的状态。 每个状态都包括位置和迄今为止访问过的节点集。 这很快就变得不可行，因为状态空间爆炸：有$O(n^2)$位置对，甚至忽略访问集的复杂性，转换也会呈指数级扩展。 即使是状态空间上的单个 BFS 也需要大约$n^2$或更多操作，这是不可能的$10^6$。 

关键的观察结果是树结构强制产生非常严格的交互。 之间只有一条路径$a$和$b$，并且追逐者将始终沿着最短路径向跑步者移动。 这将整个交互减少到沿着树度量的单个维度。 

奔跑者只能安全地探索不受追赶者前进“支配”的节点。 如果我们将树根设在$b$，那么每个节点到追逐者的起点都有一个明确定义的距离。 追赶者总是移动以减少与跑步者的距离，因此，在任何时刻，追赶者的触及范围都会有效地从各个方向向外扩展。$b$以速度 1 移动，而奔跑者也以速度 1 移动。奔跑者只能在追逐者到达之前访问节点。 

这将问题转化为树上经典的最早到达比较：计算$b$到所有节点，然后确定跑步者可以沿着从以下位置开始的任何可行探索路径前进多远$a$在被超越之前。 正确的解释是，如果一个节点能够严格早于追赶者到达，那么该节点对于跑步者来说是“安全的”。 

因此我们计算所有距离$b$使用广度优先搜索（BFS）。 然后我们进行第二次 BFS$a$，但仅遍历跑步者到达时间严格小于追逐者到达时间的边缘。 这个受限 BFS 中的可达节点数就是答案。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力状态模拟 |$O(n^2)$或者更糟|$O(n^2)$| 太慢了 |
 | 多源距离BFS剪枝|$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们利用两个玩家在树上以相同的速度移动的事实，因此可达性完全由最短路径距离决定。 

1.计算距节点的最短距离$b$使用 BFS 到每个其他节点。 这给出了追逐者可以占领每个节点的最早时间。 这是有效的，因为树具有单位权重边，因此 BFS 会生成精确的最短路径。 
2.从节点开始初始化第二个BFS$a$。 将跑步者的到达时间设置为$a$为零并将其标记为已访问。 
3、BFS扩容时，考虑从一个节点迁移$u$给邻居$v$。 跑步者将到达$v$在某个时间$dist_a[u] + 1$。 
4. 只允许过渡到$v$如果$dist_a[u] + 1 < dist_b[v]$。 这确保了跑步者严格地在追逐者之前到达，这是必要的，因为同时到达会导致在节点或沿边缘捕获。 
5. 对于此 BFS 中访问的每个有效节点，递增答案。 这包括了跑步者在被抓住之前可以安全到达的所有建筑物。 
6. 继续，直到 BFS 队列耗尽。 

关键的实现细节是使用严格的不平等。 平等必须被拒绝，因为在边缘相遇也是一种捕获事件。 

为什么它有效：BFS 来自$b$为每个节点定义一个全局“威胁时间”。 运行者的 BFS 在到达时间必须严格小于威胁时间的约束下构造可到达的节点的最大前缀。 由于两个过程在树上以单位速度同步演化，因此任何违反此约束的行为都意味着在节点或中间边缘不可避免地发生拦截，因此违反该约束的路径都不能成为最佳安全轨迹的一部分。 相反，任何遵循它的路径都可以通过连续选择最短步移动来执行，而无需与追逐者的到达同步。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

def solve():
    n, a, b = map(int, input().split())
    a -= 1
    b -= 1

    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    INF = 10**18

    dist_b = [INF] * n
    q = deque([b])
    dist_b[b] = 0

    while q:
        u = q.popleft()
        for v in g[u]:
            if dist_b[v] == INF:
                dist_b[v] = dist_b[u] + 1
                q.append(v)

    dist_a = [-1] * n
    q = deque([a])
    dist_a[a] = 0

    ans = 1

    while q:
        u = q.popleft()
        for v in g[u]:
            if dist_a[v] != -1:
                continue
            nd = dist_a[u] + 1
            if nd < dist_b[v]:
                dist_a[v] = nd
                ans += 1
                q.append(v)

    print(ans)

if __name__ == "__main__":
    solve()
```第一个 BFS 计算追逐者的最早到达时间$b$。 第二个 BFS 是从$a$，其中每个步骤都会检查跑步者是否严格早于追逐者到达下一个节点。 答案计算在此约束遍历中成功排队的所有节点，从$a$本身。 

一个常见的错误是忘记了严格的不等式并允许相等，这会错误地计算恰好在到达时间或沿着边缘发生捕获的节点。 另一个微妙的问题是忘记将起始节点的答案初始化为 1，因为$a$总是在任何移动之前首先被访问。

 ## 边缘情况

 当$a = b$，BFS来自$a$从追踪者距离为零的节点开始。 由于条件要求严格不平等，因此不可能进行扩展。 该算法正确返回 1，仅考虑起始节点。 

当树是一条简单路径并且$a$和$b$是端点，距离数组变得对称，并且每个潜在的移动在某个点上都相等。 BFS 来自$a$无法扩展到起始节点之外，这与任何移动都会导致在节点或沿边缘立即拦截的事实相匹配。 

什么时候$n = 1$，两个 BFS 过程都退化为单节点初始化，并且答案仍然是 1，这是一致的，因为唯一的建筑物被平凡地访问过。
