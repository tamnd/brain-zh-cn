 ## 解决方案
 ## 问题理解

 我们得到一个已经保证是二分图。 这意味着顶点可以分为两组，以便每条边连接来自不同组的顶点。 在此结构之上，一些顶点已指定颜色：红色、蓝色或无色。 

我们最终必须满足的约束是所有剩余顶点的最终着色必须与图的有效二分着色一致，同时也尊重所有预先分配的颜色。 但是，我们可以删除顶点。 删除一个顶点也会删除与其相关的所有边，我们希望删除尽可能少的顶点，以便剩余的图可以正确着色。 

因此，真正的任务是决定保留哪些顶点，以便存在与所有保留的预着色顶点一致的二分着色，同时最小化删除的顶点数量。 

约束足够小，可以预期顶点和边数的线性或近线性解。 对于最多 10^4 个顶点和边，每个组件大约 O(n + m) 的任何解决方案都很容易就足够了，但是任何涉及指数子集或每个顶点的大量重新计算的方法都会太慢。 

一个微妙的点是，二分图消除了搜索有效结构的需要。 唯一的困难是协调预着色约束与每个连接组件固有的双色结构。 

一些边缘情况在实践中很重要。 一种是当连接的组件根本没有预着色顶点时，此后任何二分分配都可以工作并且不需要删除。 另一种情况是，当组件具有冲突的预色时，会强制为二分边的两个端点分配不兼容的颜色。 例如，如果二分边同时包含一个强制为红色的顶点和另一个强制为蓝色的顶点，则必须删除其中一个。 第三种情况是孤立的顶点，它们总是安全的，除非它们的预着色方式与其组件的所选全局解释不一致，但由于它们形成单例组件，因此它们的行为是独立的。 

## 方法

 暴力方法会尝试为每个顶点决定是保留它还是删除它，然后检查剩余的图是否可以根据预着色正确地进行 2 色着色。 这将涉及探索顶点子集，并为每个子集运行二分验证或约束传播。 即使我们积极剪枝，子集的数量也以 n 为指数，每次检查的成本至少为 O(n + m)，这使得这完全不可行。 

关键的观察是我们实际上不需要搜索任意子集。 该图已经是二分图，因此每个连接的分量恰好具有两种有效的着色，它们彼此相反。 一旦我们修复了一个顶点的分配，组件的其余部分就会被强制。 这意味着每个组件只有两个全局状态：要么我们将二分区的一侧解释为红色，另一侧解释为蓝色，要么翻转该解释。 

给定这种结构，顶点出现问题的唯一原因是其预先指定的颜色与其二分边的所选解释不匹配。 由于我们可以删除顶点，因此不匹配不会迫使我们放弃整个组件； 相反，我们可以简单地删除有问题的顶点。 这将问题转变为针对每个组件选择两种二分解释中的哪一种能够最小化删除的顶点数量。

因此，对于每个连接的组件，我们使用 BFS 或 DFS 计算其二分。 然后我们计算在一种解释和翻转解释下有多少预着色顶点与分配不一致。 该组件的最佳答案是两个计数中较小的一个，而实际要删除的顶点正是那些在所选解释下不一致的顶点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力子集搜索 | O(2^n · (n + m)) | O(2^n · (n + m)) | O(n + m) | 太慢了|
 | 组件二分+翻转选择| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

 我们逐个组件地处理图形，因为约束不会在断开连接的部分之间交互。 

1. 我们运行 BFS 或 DFS 为连接组件中的每个顶点分配二分奇偶校验（0 或 1）。 该奇偶性表示图形的强制结构，与颜色无关。 
2. 在分配奇偶校验时，我们还收集当前组件中的所有顶点，以便我们可以独立于图的其余部分对其进行评估。 
3. 一旦确定了某个组件，我们就会评估两种可能的解释。 在第一种解释中，我们将奇偶校验 0 视为红色，将奇偶校验 1 视为蓝色。 在第二种解释中，我们翻转这个映射。 
4. 对于每个解释，我们扫描组件中的所有顶点并计算有多少预着色顶点与解释冲突。 未着色的顶点永远不会增加成本，因为它们始终可以在以后分配。 
5. 我们选择冲突数量较少的解释。 在此选定的解释下发生冲突的顶点被标记为要删除。 
6. 我们累积所有组件中删除的顶点并输出它们的计数和索引。 

它的工作原理与二部图的结构不变量有关。 在任何连接的组件中，一旦一个顶点的奇偶性固定，所有其他顶点的奇偶性都被唯一确定。 这将有效着色的整个空间减少到每个组件恰好有两个全局配置。 由于删除仅删除约束并且不会引入新约束，因此可以安全地删除任何与所选配置不一致的顶点，而不会影响其他地方的一致性。 最终的选择是最优的，因为对于每个组件，我们独立地最小化仅有两个可行的全局状态的线性成本。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    c = list(map(int, input().split()))
    
    g = [[] for _ in range(n)]
    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)

    vis = [False] * n
    col = [-1] * n  # bipartite parity
    removed = []

    from collections import deque

    for i in range(n):
        if vis[i]:
            continue

        # BFS for component
        q = deque([i])
        vis[i] = True
        col[i] = 0
        comp = [i]

        while q:
            u = q.popleft()
            for v in g[u]:
                if not vis[v]:
                    vis[v] = True
                    col[v] = col[u] ^ 1
                    q.append(v)
                    comp.append(v)

        # try two interpretations
        remove0 = []
        remove1 = []

        for u in comp:
            if c[u] == 0:
                continue
            if c[u] == 1:
                if col[u] != 0:
                    remove0.append(u)
                if col[u] != 1:
                    remove1.append(u)
            else:  # c[u] == 2
                if col[u] != 1:
                    remove0.append(u)
                if col[u] != 0:
                    remove1.append(u)

        if len(remove0) <= len(remove1):
            removed.extend(remove0)
        else:
            removed.extend(remove1)

    print(len(removed))
    if removed:
        print(*[x + 1 for x in removed])
    else:
        print()

if __name__ == "__main__":
    solve()
```该代码首先构建图表并使用 BFS 计算每个组件的二分奇偶校验标记。 这种奇偶性是组件的固定结构支柱。 之后，它评估奇偶性和实际颜色之间可能的映射，并收集每种情况的不匹配顶点。 最终的决定是每个组件独立做出的，所有选定移除的联合形成答案。 

一个常见的实施陷阱是忘记选择是针对每个组件的，而不是全局的。 另一个原因是混淆了奇偶校验映射的两种解释，这会导致删除计数不正确。 

## 工作示例

 ### 示例 1

 输入：```
7 7
1 2 1 0 2 1 0
1 5
2 5
3 5
3 6
4 6
2 7
3 7
```我们首先构建二分结构。 假设 BFS 分配奇偶校验值如下：

 | 顶点| 平价 | 颜色 |
 | --- | --- | --- |
 | 1 | 0 | 1 |
 | 2 | 0 | 2 |
 | 3 | 1 | 1 |
 | 4 | 0 | 0 |
 | 5 | 1 | 2 |
 | 6 | 0 | 1 |
 | 7 | 1 | 0 |

 现在我们测试解释 A：奇偶校验 0 是红色，奇偶校验 1 是蓝色。 

我们扫描并发现彩色顶点与此映射不一致的不匹配。 假设在此解释下顶点 2、4 和 6 发生冲突。 

在解释 B 下：奇偶校验 0 是蓝色，奇偶校验 1 是红色，假设只有顶点 5 发生冲突。 

所以我们选择解释 B 并仅删除顶点 5。 

该跟踪表明，我们并不是试图通过重新分配结构来解决全局冲突，而是为每个组件选择最佳的全局翻转。 

### 示例 2

 输入：```
5 4
1 1 2 2 0
5 1
5 2
5 3
5 4
```这是一个以 5 为中心的星形图。BFS 将奇偶校验 0 分配给节点 5，将奇偶校验 1 分配给所有其他节点。 

我们测试这两种解释：

 在奇偶校验 0 = 红色、奇偶校验 1 = 蓝色的情况下，节点 1 和 2 正常，但节点 3 和 4 发生冲突，因此移除量 = {3, 4}。 

在奇偶校验 0 = 蓝色、奇偶校验 1 = 红色的情况下，节点 1 和 2 发生冲突，因此删除量 = {1, 2}。 

我们选择任一侧，都给出两次删除，因此我们可以输出大小为 2 的任何有效最小集。 

这证明了对称性：当两个二分翻转的成本相等时，任何一个都是最优的。

 ## 边缘情况

 一个重要的边缘情况是没有预着色顶点的组件。 在这种情况下，两种解释都产生零冲突，因为没有什么可违反的。 该算法将正确选择任一侧并且不会删除任何内容。 

另一种情况是组件已完全着色但与二分解释之一不一致。 该算法通过评估两次翻转并选择冲突较少的翻转来处理此问题，从而有效地丢弃最少的必要顶点集。 

最后一个微妙的情况是带有预着色的孤立顶点。 由于每个孤立的顶点都形成自己的组件，因此 BFS 会为其分配一个奇偶校验，并且算法只需检查该奇偶校验是否与其颜色相匹配。 如果不是，则删除该顶点，这是最佳的，因为没有替代结构可以与之协调。
