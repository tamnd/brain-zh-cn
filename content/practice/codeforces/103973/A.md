 ## 解决方案
 ## 问题理解

 我们正在模拟一款连续战斗游戏，玩家按照固定顺序与怪物战斗。 每个怪物都有固定的攻击阈值，玩家维持一个代表其当前攻击能力的整数状态。 

一开始，玩家的能力为零。 当面对怪物i时，单场战斗的胜负取决于当前能力m与怪物力量ai的比较。 如果能力与怪物完全匹配，则怪物会立即被击败。 如果能力变大，怪物仍然会被击败，但玩家的能力有可能减少一分。 如果能力较小，则怪物存活下来，玩家的能力可能会以一定概率增加一倍，之后必须再次与同一怪物战斗。 在当前怪物被击败之前，玩家无法继续处理下一个怪物，因此每个怪物都定义了一个独立的随机过程，但状态会延续。 

任务是计算击败所有怪物所需的预计战斗总数。 答案是一个有理数，我们必须将其模 998244353 输出。 

约束表明 n 至多为 1000，每个 ai 至多为 1000，而概率则以 0.01 的小有理倍数形式给出。 这立即排除了对随机性或随机过程的枚举路径的任何模拟，因为所有怪物的可能能力值的状态空间很大并且转换是概率性的。 任何解决方案都必须以分析方式计算期望并在怪物之间重用结构。 

一个微妙的困难是，在与单个怪物反复战斗时，能力会发生变化，这会影响所有未来的怪物。 独立对待怪物的幼稚解释是失败的。 

一种重要的边缘情况是当 ai 很大而玩家的能力通常低于它时。 例如，从m = 0和a1 = 100开始，该过程涉及许多重复的不成功的战斗，并且由于重复的概率增量，预期的战斗次数变大。 另一个边缘情况是当概率极端时，例如 xi = 100，当失败时，行为会变得确定性向上漂移。 

## 方法

 暴力方法会将过程模拟为由（当前怪物索引，当前能力）定义的状态上的马尔可夫链。 从每个状态，我们根据概率进行分支并递归累积预期步骤。 这正确地对系统进行了建模，但状态数至少是 m 可能范围的 n 倍，并且在实践中 m 本身可能会漂移到大约 2000。 更糟糕的是，转换会形成循环，因为失败的怪物会保留相同的索引，但概率会发生变化，这意味着除非被记忆，否则朴素递归会无限地重新访问状态。 

如果我们尝试对所有 (i, m) 进行动态规划，我们可以定义每个状态的预期步骤，但固定 i 的转换取决于 m 和 ai，并且还将 (i, m) 连接到 (i, m+1) 或 (i, m-1) 或 (i+1, m)，形成一个稠密的线性方程组。 全局解决这个问题是可能的，但需要处理大约 10^6 个状态，这是临界且不必要的。 

关键的简化是观察到对于每个怪物，该过程仅取决于当前的 m 值，并且与单个怪物战斗时的进化与未来的怪物无关。 这使我们能够为每个可能的起始 m 计算两个量：击败怪物 i 所需的预期战斗次数，以及击败后最终 m 的分布。 然而，直接维护发行版仍然很繁重。

关键的观察结果是，对于每个怪物，该过程是具有单个吸收成功条件的一维随机游走，并且 m 的预期时间和过渡可以使用 m 有界间隔上的线性方程来计算。 由于在最佳状态下 m 永远不需要超过 max(ai) 超过 1，因此我们可以压缩状态空间并在 O(A) 或 O(A^2) 中求解每个怪物的 DP，其中 A 是 max ai。 

我们为每个怪物计算一个期望值系统 E[m]，代表从能力 m 开始击败它的预期剩余战斗。 每个状态满足仅取决于 m、m+1、m-1 和固定阈值 ai 的线性方程。 对每个怪物迭代求解这些方程可以有效地得到答案。

 | 蛮力（状态递归/模拟）| 指数| 大| 太慢了 |

 ## 算法演练

 我们按顺序处理怪物，同时保持每个可能的能力状态 m 的剩余战斗的预期数量。 令 E[m] 表示从当前能力 m 开始，到完成当前索引以后的所有怪物所需的预期战斗次数。 

1. 我们初始化一个 DP 数组 E，其中 E[m] 是从能力为 m 的怪物 1 开始所需的预期战斗次数。 最初 m = 0，因此我们只需要 E[0]，但我们计算所有相关 m 的值。 
2. 对于从 1 到 n 的每个怪物 i，我们根据当前的 E 计算一个新的 DP 数组 nextE。这​​个想法是我们将击败怪物 i 的效果“折叠”到转换中。 
3.对于固定的能力m，我们分析完成怪物i的预期成本。 如果m等于ai，则怪物在一场战斗中被击败，我们直接进入下一个DP状态。 因此 nextE[m] 等于 1 加上转换后计算的 E[m]。 
4. 如果m大于ai，那么在一次战斗中我们总是击败怪物，但能力以概率pi减少1。 这会产生一个循环，其中 nextE[m] 取决于 nextE[m] 本身和 nextE[m−1]，因为击败后我们继续处理下一个可能会降低能力的怪物。 
5.如果m小于ai，我们没有击败怪物，所以我们保持相同的怪物状态并支付一次战斗费用。 随着概率 pi，能力增加 1，否则保持不变。 这创建了一个具有概率向上漂移的自循环，并且期望满足涉及 nextE[m] 和 nextE[m+1] 的线性方程。 
6. 我们以 m 的递增顺序求解这些线性方程，以便在需要时已经计算出依赖性。 这种排序之所以有效，是因为转换仅在 m 的相邻值之间移动。 
7.处理完怪物i的所有m后，我们用nextE替换E并继续处理下一个怪物。 
8. 最终答案是E[0]，即从零能力开始到第一个怪物之前的预期战斗次数。 

### 为什么它有效

 核心不变量是，处理完怪物 i 后，DP 数组 E[m] 正确表示从能力 m 开始完成怪物 i 到 n 所需的预期剩余战斗次数。 每个怪物贡献一个局部随机过程，每次战斗最多只改变 m 一个，因此每个 m 的期望仅取决于邻近状态，不需要当前 DP 层之外的全局历史。 这使得系统可简化为一系列局部线性方程，可以增量求解而不会失去正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def main():
    n = int(input())
    monsters = []
    max_a = 0
    for _ in range(n):
        a, x = map(int, input().split())
        p = x * modinv(100) % MOD
        monsters.append((a, p))
        max_a = max(max_a, a)

    # DP over ability values up to max_a + 2
    LIM = max_a + 5
    dp = [0] * (LIM + 1)

    for a, p in monsters:
        ndp = [0] * (LIM + 1)

        for m in range(LIM, -1, -1):
            if m == a:
                ndp[m] = (1 + dp[m]) % MOD

            elif m > a:
                # always kill, possibly drop m-1
                ndp[m] = (1 + (p * ndp[m - 1] + (1 - p) * dp[m]) % MOD) % MOD

            else:
                # must retry until success
                # expected geometric-like structure
                ndp[m] = (modinv(p) + ndp[m + 1]) % MOD if p != 0 else (1 + ndp[m]) % MOD

        dp = ndp

    print(dp[0] % MOD)

if __name__ == "__main__":
    main()
```DP数组表示完成每个前缀怪物后预计剩余的战斗次数，以当前能力值参数化。 转换逻辑尝试将三种状态直接编码为期望递归。 关键的实现困难是处理 m < ai 情况下的“重试直到成功”行为，该行为会崩溃为几何期望结构，因为每次失败都会以概率 1 - p 独立重复相同的状态，并以概率 p 前进。 

从高 m 向下迭代用于确保在计算 ndp 时，像 m − 1 和 m + 1 这样的依赖关系已经在同一层中得到解决。

 ## 边缘情况

 当玩家开始时远低于怪物的强度时，就会出现关键的边缘情况，例如 m = 0 和 a = 100。在这种情况下，算法会重复应用“重试直到成功”规则，这对应于几何期望而不是有限数量的 DP 转换。 递归折叠成与 1/p 成正比的闭合形式期望值，并且 DP 必须正确考虑重复的自循环而不发散。 

另一种边缘情况是概率为 1 (xi = 100)。 然后，每次失败的尝试都会确定性地增加 m，从而使该过程有效地成为确定性的行走。 DP 处理此问题是因为失败分支消失并且仅保留向上转换，从而确保在每个怪物至多有限数量的增量后终止。 

最后的边缘情况是 m 在怪物的多个阶段完全等于 ai。 DP 确保了一致性，因为相等总是会触发单步转换，而没有概率分支，因此重复的相等阈值不会累积意外的漂移。
