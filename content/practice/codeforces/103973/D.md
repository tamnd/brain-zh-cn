 ## 解决方案
 ## 问题理解

 我们给出了应用于固定位数的按位变换。 无符号整数`x`准确地表示为`k`位，因此每个值都在以下范围内`0`到`2^k - 1`。 我们还获得了一系列操作。 每个操作都会修改当前值`x`通过将其与自身的左移或右移版本进行异或，具体取决于运算值是正数还是负数。 

按顺序应用所有操作后，我们获得最终值。 任务是计算有多少个初始值`x`完全转换后保持不变，这意味着函数返回完全相同`x`我们开始于。 答案必须以 998244353 为模计算。 

关键的困难在于，变换不是一个简单的算术函数，而是一个混合各个位置的位的按位过程，并且它隐式地应用于所有可能的起始状态。 

约束足够严格，可以尝试所有`2^k`可能的值是不可能的，当`k`最大为 1000。即使一次遍历所有状态也已经超出限制，因为`2^1000`是一个天文数字。 同样，模拟每个候选人的功能`x`需要对每个状态应用多达 1000 个操作，这是完全不可行的。 

一个天真的但更结构化的尝试可能会尝试独立地模拟每个位的转换，但这也会失败，因为移位会导致位之间的依赖关系，这意味着每个输出位依赖于多个输入位。 

出现微妙的边缘情况时`n = 0`。 在这种情况下，该函数不执行任何操作，因此每个`x`是一个不动点。 那么答案一定是`2^k mod MOD`。 任何假设至少存在一项操作的解决方案都会错误地返回`0`或者`1`取决于实施。 

另一个重要的边缘情况是当移位将位完全超出范围时。 例如，如果`k = 5`我们转移`4`或者`-4`，只有一小部分位保持有效。 正确处理需要屏蔽`k`每次操作后的位。 

## 方法

 该转换是通过对以下形式的重复操作构建的`x ^= (x << s)`或者`x ^= (x >> s)`。 每个此类操作在域 GF(2) 上都是线性的，因为 XOR 对应于模 2 的加法，而移位对应于位位置的固定线性映射。 这意味着整个函数是一个线性变换`k`GF(2) 上的维向量空间。 

蛮力的想法将显式地模拟每个可能的函数`x`。 对于每个候选者，我们应用所有操作并检查结果是否与原始结果匹配。 这需要`2^k`模拟，每次成本计算`O(nk)`最坏情况下的位操作。 即使忽略常数，这也远远超出了可行的极限。 

关键的观察是我们永远不需要评估所有输入的函数。 线性变换完全由其对基的影响决定。 如果我们保持每个基向量的演化方式，我们可以将变换表示为`k × k`二进制矩阵。 每个操作对应于将该矩阵左乘于由移位异或规则引起的另一个稀疏线性运算符。 

一旦我们有了完整的变换矩阵`T`，不动点的条件是`T(x) = x`，可以重写为`(T - I)x = 0`。 在 GF(2) 上，减法是异或，所以这变成`(T XOR I)x = 0`。 解的数量恰好是该矩阵零空间的大小，等于`2^(k - rank)`。 

因此，我们将问题简化为计算 a 的秩`k × k`二元矩阵，可以通过高斯消去法完成`O(k^3 / 64)`使用位集。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解所有 x | O(2^k · n · k) | O(2^k · n · k) | O(1) | O(1) | 太慢了 |
 | 线性变换+高斯消去| O(n·k^2 / 64 + k^3 / 64) | O(n·k^2 / 64 + k^3 / 64) | O(k^2) | O(k^2) | 已接受 |

 ## 算法演练

 我们将变换解释为 GF(2) 上的矩阵，其中每一列代表基向量的图像。 我们逐步构建这个矩阵，然后求解线性系统。 

1. 初始化一个`k × k`身份转变。 这表示在应用任何操作之前，每个位都映射到其自身。 
2. 将转换表示为位集数组`T`， 在哪里`T[i]`是基向量的图像`i`。 最初，`T[i]`位置有一个 1`i`。 
3. 处理数组中的每个操作。 对于换班`s`，我们定义一个线性算子`L`这样它就可以通过与其移位版本进行异或来转换任何向量。 我们没有将其应用于所有可能的向量，而是直接将其应用于变换矩阵的每一列。 这是可行的，因为将线性运算符应用于线性映射的输出相当于组合线性映射。 
4. 对于每一列`T[i]`，通过应用移位异或规则就地更新它。 这使矩阵与组合变换保持一致。 
5.处理完所有操作后，我们得到完整的变换矩阵`T`。 
6. 构建矩阵`B = T XOR I`，对应于`(T - I)`超过 GF(2)。 该矩阵对不动点的条件进行编码。 
7. 使用位集对 GF(2) 执行高斯消元法来计算`B`。 每个主元都会将解决方案空间的维度减少一倍。 
8. 将答案计算为`2^(k - rank)`以 998244353 为模。 

正确性依赖于每次操作后的不变量，`T`正确地表示了迄今为止所见的所有变换的组成。 由于每个操作都是线性的，因此通过将其应用于所有基础图像来进行合成可以保持正确性。 最终系统`Bx = 0`精确地捕获了全部变换后保持不变的向量，其解空间的维数决定了有效的个数`x`。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def gauss_rank(mat, k):
    rank = 0
    row = 0

    for col in range(k - 1, -1, -1):
        pivot = -1
        for i in range(row, k):
            if (mat[i] >> col) & 1:
                pivot = i
                break
        if pivot == -1:
            continue

        mat[row], mat[pivot] = mat[pivot], mat[row]

        for i in range(k):
            if i != row and ((mat[i] >> col) & 1):
                mat[i] ^= mat[row]

        row += 1
        rank += 1
        if row == k:
            break

    return rank

def apply_op(cols, s, k):
    new_cols = [0] * k
    if s >= 0:
        for i in range(k):
            x = cols[i]
            new_cols[i] = x ^ ((x << s) & ((1 << k) - 1))
    else:
        s = -s
        for i in range(k):
            x = cols[i]
            new_cols[i] = x ^ (x >> s)
    return new_cols

def main():
    n, k = map(int, input().split())
    A = list(map(int, input().split()))

    cols = [(1 << i) for i in range(k)]

    mask = (1 << k) - 1

    for a in A:
        if a >= 0:
            s = a
            for i in range(k):
                cols[i] = cols[i] ^ ((cols[i] << s) & mask)
        else:
            s = -a
            for i in range(k):
                cols[i] = cols[i] ^ (cols[i] >> s)

    mat = cols[:]
    for i in range(k):
        mat[i] ^= (1 << i)

    rank = gauss_rank(mat, k)

    pow2 = 1
    exp = k - rank
    for _ in range(exp):
        pow2 = pow2 * 2 % MOD

    print(pow2)

if __name__ == "__main__":
    main()
```该解决方案按列构建转换，将每一列视为`k`位整数。 每个移位异或运算都直接应用于所有列，这对应于组合线性映射。 之后，我们通过对单位矩阵进行异或运算，将问题转化为求解齐次线性系统，然后使用位集上的高斯消去法计算其秩。 最终的指数是从系统的无效性导出的。 

在屏蔽左移时需要小心，以便超出范围的位`k`位边界不会泄漏到无效位置。 右移自然会丢弃位。 消除步骤从高到低处理列，以确保枢轴选择稳定且高效。

 ## 边缘情况

 当`n = 0`，变换就是恒等式。 矩阵保持不变，所以`T XOR I`becomes the zero matrix. 高斯消去法发现秩为0，答案变为`2^k`，这意味着每个可能的位掩码都是一个固定点。 

当所有操作都超出边界时，例如相对于`k`，屏蔽后左移部分变为零。 该变换简化为更简单的 XOR 结构，但仍然保持线性。 矩阵构造确保永远不会出现超出范围的位，因此最终的排名计算仍然有效。 

什么时候`k = 1`，每个变换都分解为一位运算，并且所有矩阵都简化为 1×1 系统。 消除步骤正确处理了这种退化情况，根据是否保留单个位来生成 0 或 1 个固定点。
