 ## 解决方案
 ## 问题理解

 我们不会被要求在插入后构建最终的排列。 相反，对于给定的`n`，我们必须构造一个排列`p`的`1..n`这样上一个问题“插入`n+1`某处并使所有前缀和取模`n+1`成对不同”，至少有一个有效的插入位置。

 如果这样的排列`p`存在，我们输出其中任何一个。 否则我们输出`-1`。 

全部的总和`n`测试用例最多是`10^6`，因此任何在每个测试用例的线性时间内构建答案的解决方案都足够快。 在最坏的情况下，任何二次方都会太慢。 

微妙的部分是我们不是直接构建`p`。 插入后定义条件`n+1`，所以我们首先需要了解哪些长度排列`n+1`满足不同前缀条件。 

一个常见的错误是寻找复杂的插入策略。 一旦理解了有效的最终排列的结构，生成版本就会简单得多。 

考虑`n = 2`。 唯一的排列是`[1,2]`和`[2,1]`。 插入后`3`，每个结果排列都有长度`3`。 事实证明，没有长度为 3 的排列可以使所有前缀和模数`3`不同，所以正确答案是：```
-1
```任何盲目地为每个输出一些排列的构造`n`会在这里失败。 

另一个容易被忽视的案例是`n = 1`。 

我们可以输出：```
1
```因为插入后`2`在前面我们得到`[2,1]`，其前缀求模求和`2`是`0`和`1`，这是不同的。 

## 方法

 蛮力方法将生成所有`n!`的排列`1..n`，并且对于每个排列尝试所有`n+1`插入位置。 构造出结果排列后，我们可以检查其前缀和是否模`n+1`都是不同的。 

这是正确的，但即使对于`n = 10`已经有`3,628,800`排列。 阶乘增长使这个想法变得不可用。 

关键的观察结果是插入步骤可以颠倒。 

让`m = n + 1`。 假设我们已经有了一个有效的最终排列`q`的`1..m`其前缀和模`m`是两两不同的。 如果我们删除该元素`m`，剩下的序列是一个排列`p`的`1..n`。 重新插入`m`到相同的位置重新创建`q`， 所以`p`自动是一个有效的答案。 

问题变成：

 “对于哪些值`m`是否存在一个排列`1..m`其前缀和模`m`都是不同的吗？”

 这是一种众所周知的结构，有时称为超排列。 这种排列仅存在于`m = 1`或者`m`是均匀的。 对于奇数`m > 1`，它们不存在。 

自从`m = n + 1`，存在等价于：`n`是奇数，或者`n = 0`（它不会出现在输入中）。 

所以每一个偶数`n`是不可能的。 

对于甚至`m = 2k`，一个简单的构造是：```
m, 1, m-2, 3, m-4, 5, ..., 2, m-1
```例如，当`m = 8`:```
8 1 6 3 4 5 2 7
```它的前缀和取模`8`是：```
0 1 7 2 6 3 5 4
```这正是所有模余数`8`，每个出现一次。 

获得所需的排列`p`，我们只需删除领先的`m`。 

剩下：```
1, m-2, 3, m-4, 5, ...
```这是一个排列`1..n`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n!·n²) | O(n!·n²) | O(n) | 太慢了 |
 | 最佳施工| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 阅读`n`。 
2.如果`n`为偶数，输出`-1`。 

什么时候`n`是偶数，`m = n + 1`是奇数且大于`1`。 没有有效的最终长度排列`m`存在，所以没有有效的`p`也存在。 
3.否则设置`m = n + 1`。 
4. 构造通过移除第一个元素而获得的排列：```
m, 1, m-2, 3, m-4, 5, ...
```5. 输出结果序列。 

### 为什么它有效

 对于甚至`m`， 定义：```
q = [m, 1, m-2, 3, m-4, 5, ...]
```第一个前缀和模`m`是`0`。 

添加后`1`，残数变为`1`。 

添加`m-2`将其更改为`-1 (mod m)`，即`m-1`。 

添加`3`将其更改为`2`。 

添加`m-4`将其更改为`m-2`。 

继续这样下去，残数变为：```
0, 1, m-1, 2, m-2, 3, m-3, ...
```每个余数取模`m`只出现一次，因此所有前缀和都是不同的。 

删除元素`m`产生一个排列`p`的`1..n`。 重新插入`m`在前面重建`q`，证明`p`是一个有效的答案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    ans = []

    for _ in range(t):
        n = int(input())

        if n % 2 == 0:
            ans.append("-1")
            continue

        m = n + 1
        res = []

        l = 1
        r = m - 2

        while l <= r:
            res.append(str(l))
            if l != r:
                res.append(str(r))
            l += 2
            r -= 2

        ans.append(" ".join(res))

    sys.stdout.write("\n".join(ans))

solve()
```第一个分支处理不可能的情况。 什么时候`n`是偶数，`m = n + 1`是奇数，并且不存在有效的超排列。 

对于奇数`n`，我们生成：```
1, m-2, 3, m-4, 5, ...
```直接，无需构建更大的排列`q`。 

指针`l`和`r`按升序生成奇数，并按降序生成匹配的大数。 因为`m`是偶数，该序列包含来自的每个整数`1`到`m-1`正好一次。 

条件`if l != r`当两个指针在中间相遇时，避免打印相同的数字两次。

 ## 边缘情况

 对于`n = 2`，我们有`m = 3`，这很奇怪。 该理论表明长度不存在超级排列`3`存在。 该算法立即打印：```
-1
```这是正确的。 

为了`n = 1`，我们有`m = 2`。 该建筑产生：```
p = [1]
```重新插入`2`前面给出：```
2 1
```前缀求模求和`2`是：```
0, 1
```这是不同的，所以答案是有效的。 

对于较大的奇数值，例如`n = 9`，我们有：```
m = 10
p = 1 8 3 6 5 4 7 2 9
```重新插入`10`前面产生长度的超级排列`10`，因此通过构造存在所需的插入位置。
