 ## 解决方案
 ## 问题理解

 我们有一个矩形网格，其行为就像一个时间相关的迷宫。 玩家从网格的右上角开始，想要到达左下角。 移动是一步一步进行的，每一步玩家都可以向四个方向中的任何一个方向移动一个单元格或保持在原地。 

困难来自于移动危险。 每行都有一只水平来回巡逻的老虎。 每只老虎每单位时间移动一步，在边界处反转方向，并且这种运动是完全确定性的。 由于这种弹跳行为，每只老虎的位置随着时间的推移是周期性的，并且整个网格配置在从列数得出的固定周期后重复。 

仅当玩家从未与任何老虎共享单元格时，移动才有效，无论是在移动之后还是在老虎更新下一个时间步的位置之后。 任务是确定是否存在安全到达出口的移动序列。 

输入大小意味着网格最大为 100 x 100，因此最多有 10,000 个位置。 时间也是相关的，因为老虎的位置随着时间的推移而变化，但它们的周期受到网格宽度的一个小函数的限制。 这立即表明随时间变化的直接最短路径是可行的，但任何随时间呈指数变化的路径都是不可行的。 

在没有记忆的情况下模拟所有可能路径的天真尝试将在相同的时间阶段重复重新访问相同的配置，从而导致指数爆炸。 另一个常见错误是忽略时间并将老虎位置视为静态或仅检查当前步骤，这会失败，因为安全单元可能在下一刻变得危险。 

当玩家尝试在当前安全的单元格中“等待”但在下一个周期中被占用时，就会出现微妙的边缘情况。 例如，如果一只老虎周期性地摆动并到达一根柱子，那么如果忽略时间，玩家可能在本地看起来很安全，但在下一步中会被困住。 

## 方法

 蛮力的想法是从起始单元开始探索所有可能的移动序列，模拟老虎每一步的移动。 这原则上是正确的，因为它会尝试每条路径，但它会呈指数分支。 即使每步有 5 种可能的移动并且深度达到 10,000，这也是完全不可行的。 

关键的观察是系统在状态空间中并不是真正无限的。 关于时间的唯一相关信息是老虎当时的位置，并且由于它们的运动是周期性的，因此时间可以以周期长度为模来减少。 这将问题转换为分层图上的最短路径，其中每一层对应于一个时间阶段。 

一旦我们接受了这一点，问题就变成了由位置和时间模周期定义的状态上的 BFS。 每个状态最多有五个转换，因此总的复杂性变得易于管理。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| 指数| O(1)-O(路径) | 太慢了 |
 | (r, c, t mod T) | BFS O(n·m·T) | O(n·m·T) | 已接受 |

 ## 算法演练

 ### 1. 预计算老虎运动

 对于每一行，随着时间的推移模拟老虎的位置，直到模式重复。 由于运动是柱之间的简单反弹，因此周期为`2m - 2`。 我们存储每一行​​的老虎列，并以该周期为模的时间。 

这使我们能够在 O(1) 内回答某个单元在给定时间阶段是否被占用。 

### 2.定义BFS状态空间

 状态表示为`(row, col, time_mod)`。 这捕获了确定未来有效性所需的一切，因为老虎的位置仅取决于`time_mod`。 

我们从`(1, m, 0)`。 

### 3. 检查单元格的有效性

 细胞有时是安全的`t`如果当时没有老虎占领它`t`。 由于每一行都只有一只老虎，因此我们只需要检查该老虎所在的列。 

我们还必须确保这两个时间的安全`t`和`t+1`，因为移动发生在老虎更新之前和之后。 

### 4. BFS 转换

 来自一个州`(r, c, t)`我们尝试五个动作：上、下、左、右、停留。 对于每个候选单元格`(nr, nc)`，我们计算`t_next = (t + 1) % period`。 

仅当满足以下条件时，我们才会接受此举：

 当前单元格`(nr, nc)`时间未被占用`t`并且同一时间没有被占用`t_next`。 

这增强了整个过渡过程中的安全性。 

### 5. 访问追踪

 我们标记`(r, c, t_mod)`以避免重新访问相同的配置。 这可以防止由等待周期引起的无限循环。 

### 6. 终止

 如果我们达到`(n, 1, any time)`我们输出YES。 如果BFS穷尽所有状态，则输出NO。 

### 为什么它有效

 该算法将系统的每个可达配置视为有限图中的节点。 因为老虎的位置在每个周期都会重复，所以每个无限的时间线都会崩溃为一组有限的时间阶段。 BFS 保证在这个有限状态图上进行最短的可达性探索，并且安全条件确保不会发生无效的转换。 由于所有有效路径都对应于该图中的行走，并且所有图形行走都经过探索，因此结果是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    
    # precompute tiger position for each row over time
    period = 2 * m - 2 if m > 1 else 1
    
    tiger = [[0] * period for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        # simulate one full cycle
        path = []
        pos = 1
        dir = 1
        for t in range(period):
            path.append(pos)
            if m > 1:
                if pos + dir > m or pos + dir < 1:
                    dir *= -1
                pos += dir
        tiger[i] = path
    
    def safe(r, c, t):
        return tiger[r][t] != c
    
    sr, sc = 1, m
    tr, tc = n, 1
    
    dist = [[[-1] * period for _ in range(m + 1)] for _ in range(n + 1)]
    q = deque()
    
    dist[sr][sc][0] = 0
    q.append((sr, sc, 0))
    
    moves = [(1,0), (-1,0), (0,1), (0,-1), (0,0)]
    
    while q:
        r, c, t = q.popleft()
        if r == tr and c == tc:
            print("YES")
            return
        
        nt = (t + 1) % period
        
        for dr, dc in moves:
            nr, nc = r + dr, c + dc
            
            if not (1 <= nr <= n and 1 <= nc <= m):
                continue
            
            if not safe(nr, nc, t):
                continue
            if not safe(nr, nc, nt):
                continue
            
            if dist[nr][nc][nt] != -1:
                continue
            
            dist[nr][nc][nt] = 1
            q.append((nr, nc, nt))
    
    print("NO")

if __name__ == "__main__":
    solve()
```该解决方案首先逐行构建每只老虎的确定性运动，并存储每个时间阶段的列位置。 这避免了 BFS 期间的重新计算。 

BFS 状态空间使用 3D 访问数组来确保我们不会重新访问相同的配置。 转换逻辑明确地强制执行边界条件，因此玩家永远不会离开网格。 关键的正确性细节是检查移动之前和之后的时间层，这可以防止进入在转换后立即变得不安全的单元格。

 ## 边缘情况

 一个关键的边缘情况是老虎交替占据目标单元格。 一个忽略未来占用的天真的 BFS 会错误地允许进入`(n, 1)`当抵达时安全但抵达后立即不安全时。 

另一种情况是需要原地等待时。 如果没有“停留”移动，算法在必须延迟移动直到老虎经过的情况下会失败。 

最后，当`m = 1`，周期公式退化。 在这种情况下，没有水平运动并且老虎是静止的，因此周期应被视为1以避免被零除或无效索引。
