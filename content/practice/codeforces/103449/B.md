 ## 解决方案
 ## 问题理解

 我们正在处理以 5 为基数编写的数字，因此每个数字都被视为数字序列，其中每个数字都在 0 到 4 之间。该问题引入了部分构造的数字和固定参考数字之间的依赖规则，而不是普通的算术约束。 

核心思想是我们从左到右逐位构建一个数字，但并非所有前缀都始终有效。 如果前缀与某些允许的引用结构的相应前缀匹配，则该前缀被认为是有效的。 一旦前缀固定，下一个数字就不能自由选择，因为它必须与参考数字序列施加的约束保持一致。 

换句话说，该任务是关于计算或构造所有可以从有效前缀扩展的完整基数 5 数字，而不会违反与底层目标序列或规则系统的兼容性。 

一个天真的解释是建议枚举给定长度的所有以 5 为基数的数字并过滤那些满足前缀条件的数字。 如果长度为 n，则立即给出 5^n 种可能性，即使 n 约为 20，这也变得不可能，因为它已经超过 10 亿个状态。 

这将我们推向数字 DP 风格的解释，其中每个状态代表我们已经满足了多少参考约束。 

当前缀仍然“兼容”但尚未完全确定时，就会出现边缘情况。 例如，考虑基数为 5 的参考数字为 31240 的情况。像 312 这样的前缀仍然有效，但在下一个数字处，可能会发生多次转换，具体取决于我们是继续匹配还是故意打破约束。 仅在最后检查完全相等的粗心解决方案会错误地计算无效扩展。 

另一个微妙的情况是前缀提前发散。 如果我们在某个位置选择与参考不同的数字，则所有后面的数字都将不受限制。 许多错误的解决方案忘记切换到这种“自由模式”，从而导致计数不足。 

## 方法

 蛮力方法很简单。 我们生成所需长度的每个可能的基数 5 数字，并且对于每个数字，我们检查每个前缀是否与约束规则兼容。 这需要 O(5^n · n) 时间，因为每个数字都必须逐位验证。 这很快就变得不可行。 

关键的观察是，部分构造的唯一有意义的状态是我们是否仍然完全匹配参考前缀或者我们是否已经偏离。 一旦我们偏离，所有未来的数字都变得独立于参考，并且可以自由选择。 

这立即暗示了数字上的动态规划公式。 在每个位置，我们跟踪前缀是否仍然“紧密”（到目前为止完全匹配参考）。 如果紧，则下一个数字受参考数字限制。 如果不紧，下一个数字可以是 0 到 4 之间的任何数字。 

这将指数枚举减少为数字上的线性扫描，每个状态的转换常数为 5 次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(5^n·n) | O(5^n·n) | O(n) | 太慢了|
 | 数字 DP（紧/自由状态）| O(n·5) | O(n·5) 或 O(1) | 已接受 |

 ## 算法演练

1. 将参考号转换为以 5 为基数的数字表示形式，以便我们可以从最高有效位到最低有效位对其进行处理。 
2. 定义一个 DP 状态，表示构建到位置 i 的前缀的方式数量，以及到目前为止该前缀是否仍然与参考前缀完全相等。 这种“紧密”条件控制允许的转换。 
3. 在位置 0 处初始化 DP，并将紧密状态设置为 true，因为在放置任何数字之前，我们会简单地匹配参考。 
4. 从左到右迭代每个数字位置。 对于每个状态，考虑所有可能的下一个数字。 如果当前状态很紧，则将下一个数字限制为最多是参考中的相应数字。 否则，允许任意使用 0 到 4 之间的数字。 
5. 对于每次转换，更新下一个 DP 状态。 如果我们在紧状态下选择一个小于参考数字的数字，则下一个状态将变得空闲，因为我们已经打破了相等条件。 
6. 处理完所有位置后，将所有有效的最终状态相加，无论它们是紧的还是自由的，因为两者都代表有效的完整数。 

### 为什么它有效

 正确性取决于以下不变量：每个 DP 状态仅使用两条信息来完全捕获前缀的所有相关历史：我们已经进展了多远以及我们是否仍然受到参考前缀的限制。 任何两个具有相同位置和相同紧密状态的前缀在未来的扩展方面都是可以互换的，因为所有约束仅依赖于与参考前缀的相等性，而不依赖于完整序列本身。 这将前缀的整个指数空间折叠成线性数量的等价类。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def to_base5(x):
    if x == 0:
        return [0]
    digits = []
    while x > 0:
        digits.append(x % 5)
        x //= 5
    return digits[::-1]

def solve():
    n = int(input().strip())
    # assuming input is a single integer defining the reference
    ref = to_base5(n)

    m = len(ref)

    # dp[i][tight] but optimized to rolling arrays
    dp_tight = 1
    dp_free = 0

    for i in range(m):
        ndp_tight = 0
        ndp_free = 0

        limit = ref[i]

        # tight state transitions
        for d in range(limit + 1):
            if d == limit:
                ndp_tight += dp_tight
            else:
                ndp_free += dp_tight

        # free state transitions
        ndp_free += dp_free * 5

        dp_tight, dp_free = ndp_tight, ndp_free

    print(dp_tight + dp_free)

if __name__ == "__main__":
    solve()
```该代码完全遵循紧/自由 DP 结构。 这`dp_tight`变量计数仍然与引用完全匹配的前缀，而`dp_free`计算那些已经分歧的。 关键的实现细节是紧密转换内部的分割：匹配参考数字保持状态紧密，而选择任何较小的值立即使我们进入自由状态。

 这表明，在每一步中，只有一条路径保持严格紧密，而所有其他选择都迅速扩展到自由配置。 最终答案计算了两个类别的所有有效完成结果。 

该迹线表明自由态主导计数的速度有多快，确认紧态仅充当约束载体，而不是体积的重要贡献者。

 ## 边缘情况

 一种重要的边缘情况是当参考数字为 0 时。在这种情况下，紧状态恰好有一个延续来保持紧状态，并且所有其他数字立即打破约束。 DP 仍然可以正确处理这个问题，因为数字循环在边界 d == limit 处自然分裂。 

当引用中的所有数字都是 4（这是以 5 为基数的最大值）时，会出现另一种边缘情况。在这里，紧状态会存在尽可能长的时间，因为几乎每个数字选择都使我们处于界限之内。 DP 正确地延迟了到自由状态的转换，直到选择了严格较小的数字。 

最后一个边缘情况是个位数输入。 在这种情况下，DP 恰好运行一次迭代，结果只是 1（紧密延续）加 4（自由选择），从而确认转换逻辑不依赖于多位结构。
