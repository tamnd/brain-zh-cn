 ## 解决方案
 ## 问题理解

 我们在球体上得到两组点，每个点都用纬度和经度描述。 纬度由行星的几何形状决定，因此两张地图上的纬度保持相同。 两个数据集之间的唯一自由度来自于行星绕其自身轴的旋转，这会均匀地改变所有点的经度。 

任务是确定沿经度方向是否存在单个旋转角度，使得将第一个地图中的每个点移动该角度后，所得坐标集与第二个地图完全匹配。 

因此，该结构是纯粹的几何结构：球体上的两个多重点集，其中纬度不变，经度定义为模 360 度，我们想要测试是否可以通过在经度坐标上应用均匀圆周平移来获得第二组点。 

约束 n 高达 400,000 立即排除所有点对之间的任何二次比较。 即使排序两次也是可以接受的，但是任何涉及在两个集合之间天真地配对点的方法都会失败。 我们需要更接近线性或线性时间的东西，可能依赖于散列或规范表示。 

一个微妙的问题是浮点精度。 坐标最多具有四位小数，因此直接浮动比较是有风险的，除非仔细标准化。 另一个问题是经度在 -180 和 180 处环绕，因此必须以 360 为模处理移位。最后，纬度组中的重复很重要：多个点可能共享纬度但经度不同，因此匹配必须保留每个纬度带内的多重性。 

一种简单的方法可能会尝试将第一组中的一个点与第二组中的一个点对齐，导出候选旋转，并验证所有点。 这在概念上是合理的，但尝试所有配对会太慢。 关键是将轮换候选者减少到从结构派生的一个小集合。 

## 方法

 蛮力的想法始于一个简单的观察：如果我们猜测第一个地图中的哪个点对应于第二个地图中的哪个点，我们可以计算所需的经度偏移作为它们经度之间的差。 然后我们可以将这种转变应用于第一个地图中的所有点，并检查变换后的集合是否等于第二个地图。 

这是正确的，但配对的数量是 n 平方。 每次检查的成本为 n，因此如果简单地完成，总数将变为 O(n^3)，或者如果我们尝试优化检查，则总数将变为 O(n^2)。 当n达到400,000时，即使O(n log n)也是现实的上限，所以这是完全不可行的。 

关键的见解是纬度在旋转下不会改变。 这意味着每个纬度定义了沿经度的点的独立循环排序。 如果两个地图对应于同一个行星，则对于每个固定纬度，经度的多集必须匹配相同的全局偏移。 更重要的是，相同的转变必须同时适用于所有纬度。 

这将问题转化为在每个纬度匹配圆形多重集，所有这些都通过单个全局旋转值对齐。 我们可以计算每个在旋转下不变的映射的规范表示，而不是测试所有旋转。 标准技巧是按纬度对点进行排序，然后在每个纬度组内考虑经度的循环序列并以旋转不变的方式对其进行编码，例如减去参考点并标准化。 

为了避免浮点不稳定，我们将坐标缩放为整数。 由于精度高达 1e-4，因此乘以 10000 会使所有值都是安全整数。

最后，我们将每个纬度组简化为排序的循环序列，并计算与起点无关的哈希值。 然后整个地图被表示为这些按纬度的规范哈希值的多重集。 如果两个映射产生相同的多重集，则答案是相同的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) 或更糟 | O(n) | 太慢了|
 | 分组+规范散列| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将每个点视为缩放纬度和经度后的一对整数。 

我们按纬度对点进行分组。 在每个纬度组中，我们收集所有经度并对它们进行排序。 这给了我们一个模 360 度线上的循环序列。 

对于每个组，我们构建一个旋转不变的签名。 我们通过计算连续排序经度之间的差异来做到这一点，包括从最后一个回到第一个的环绕。 这会产生一个循环结构，与我们从哪里开始读取圆无关。 然后我们将此循环差序列编码为哈希。 

我们对每个纬度组重复此操作，为每个地图生成多个组签名。 最后，我们比较两个多重集。 

1.读取第一张地图和第二张地图的所有点，将坐标缩放为整数。 
2. 使用字典按纬度对每个地图的点进行分组。 
3. 对于每个纬度组，对其经度进行排序。 
4. 对于每个排序的经度列表，计算圆差，包括环绕。 
5. 将所得差异序列哈希为规范值。 
6. 将所有哈希值收集到映射的多重集表示中。 
7. 比较两个多重集是否相等并输出结果。 

我们使用差异而不是绝对经度的原因是全局旋转将所有经度移动相同的常数，但圆上连续点之间的差异保持不变。 

### 为什么它有效

 每个纬度形成一个独立的点的圆形排列。 行星的自转对应于每个此类排列中的均匀循环移位。 循环移位不会改变循环序列中相邻差值的多重集。 因此，每个纬度组在旋转下都被简化为不变的表示。 由于所有组共享相同的旋转，因此所有纬度上的这些不变量的相等对于地图匹配来说既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import defaultdict

def normalize_group(lons):
    lons.sort()
    m = len(lons)
    if m == 0:
        return 0
    if m == 1:
        return 1
    diffs = []
    for i in range(m):
        j = (i + 1) % m
        diff = lons[j] - lons[i]
        if j == 0:
            diff += 3600000  # scaled 360 degrees
        diffs.append(diff)
    diffs.sort()
    h = 1469598103934665603
    for d in diffs:
        h ^= d + 0x9e3779b97f4a7c15
        h *= 1099511628211
        h &= (1 << 64) - 1
    return h

def build_signature(points):
    groups = defaultdict(list)
    for a, b in points:
        la = int(round(a * 10000))
        lo = int(round(b * 10000))
        groups[la].append(lo)

    sig = []
    for la in groups:
        sig.append(normalize_group(groups[la]))
    sig.sort()
    return sig

def solve():
    n = int(input())
    p1 = [tuple(map(float, input().split())) for _ in range(n)]
    p2 = [tuple(map(float, input().split())) for _ in range(n)]

    s1 = build_signature(p1)
    s2 = build_signature(p2)

    print("Same" if s1 == s2 else "Different")

if __name__ == "__main__":
    solve()
```该解决方案首先离散化坐标以避免浮动比较问题。 每个纬度组都是孤立的，因为旋转不会混合纬度。 在每个组内，对经度进行排序将圆形结构简化为线性表示。 环绕差异确保无论起始点如何，循环结构都被保留。 

哈希函数是标准的乘法滚动哈希，用于将差异序列压缩为固定大小的指纹。 对组哈希进行排序可确保我们比较多重集而不是纬度带的有序列表。 

一个微妙的问题是，两张地图上的浮动舍入必须保持一致。 乘以 10000 并四舍五入可确保稳定的整数表示。

 ### 示例 1

 第一张地图的点按纬度分组，第二张地图的配置相同，但按经度旋转。 分组和标准化后，两者都会产生相同的签名。 

| 步骤| 映射 1 组哈希 | 映射 2 组哈希 | 比较|
 | --- | --- | --- | --- |
 | 分组后| [H1，H2] | [H1，H2] | 相同的多重集 |
 | 排序后 | [H1，H2] | [H1，H2] | 等于|

 这表明全局经度偏移不会影响循环差值哈希。 

### 示例 2

 在第二个样本中，至少一个纬度内的经度分布在结构上有所不同，这意味着没有循环移位可以对齐集合。 

| 步骤| 映射 1 组哈希 | 映射 2 组哈希 | 比较|
 | --- | --- | --- | --- |
 | 分组后| [H1，H2] | [H1，H3] | 不匹配|

 这证实，即使总计数匹配，也能检测到内部循环结构差异。

 ## 边缘情况

 一个重要的边缘情况是只有一个点的纬度组。 在这种情况下，不存在有意义的循环结构，因此对于任何单点组来说，散列必须是恒定的。 该算法返回 m = 1 的固定值，确保旋转不会引入错误的区别。 

另一种边缘情况是经度位于 -180/180 边界附近时。 因为我们使用缩放整数并以 3600000 为模处理差异，所以最后的差异计算中明确包含环绕。 例如，-179.9999 和 179.9999 处的点会产生一个小的圆形间隙，而不是一个大的线性间隙，从而保持正确性。 

最后一个微妙的情况是，多个点具有相同的纬度，但经度却相差很大。 排序保证顺序一致，循环差保证旋转下的不变性。
