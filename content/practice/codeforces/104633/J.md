 ## 解决方案
 ## 问题理解

 我们得到一棵代表校园的加权树。 建筑物是节点，人行道是有长度的边。 每条人行道必须使用两台可​​以沿着树推的吹雪机至少清理一次。 每个鼓风机从某个选定的建筑物开始，沿着边缘移动，完成后停在某个地方。 移动时，鼓风机会清理其经过的任何人行道。 关键的自由在于，两个鼓风机都可以再次穿过已经清理干净的人行道，但每个边缘必须至少被其中一个至少覆盖一次。 目标是选择起始位置和移动路线，以使两个鼓风机组合行走的总距离最小化。 

结构很重要：图是一棵树，因此任何两座建筑物之间都存在一条简单路径。 这消除了有关循环的任何歧义，并且意味着任何遍历策略都可以根据边缘覆盖和沿着树路径重复行走来推理。 

约束允许最多 100000 个节点，因此任何解决方案都必须接近线性或对数线性。 任何涉及所有节点之间的成对距离或每个候选端点重复 DFS 的操作都太慢。 即使是二次或 n log n 平方风格的方法也会失败。 

问题中的一个微妙点是，每个鼓风机都可以遍历已经清除的边缘，而这些遍历仍然会花费距离。 因此，任务不仅仅是覆盖边缘，而是设计路径，尽量减少在树的相同部分上的重复行进。 

一个常见的失败案例来自于假设将树分成两部分并为每个鼓风机分配一个子树是有效的。 这是具有误导性的，因为鼓风机可以穿过任何已清理的边缘，因此边缘的“所有权”不会以任何有用的方式限制移动。 

## 方法

 蛮力的想法是将每个鼓风机视为产生覆盖边缘子集的行走，并尝试所有方法在两个鼓风机之间分配边缘，同时计算每个分配的最佳路线。 对于固定分配，计算鼓风机的最佳路线可简化为树遍历问题，其中分配的集中的每个边都必须至少被访问一次。 即使我们假设我们可以快速计算一次分配的成本，边缘分配的数量也会以 n 为指数，因为每条边缘都可以独立属于鼓风机 A、鼓风机 B 或两者。 这已经使得该方法不可行。 

关键的观察是，在树上，单个步行者的任何最佳遍历策略都是很好理解的。 如果单个鼓风机必须覆盖所有边缘，则通过进行 DFS 式遍历来实现最小行走，其中每个边缘都遍历两次，除了沿着选定的“保存路径”的边缘（我们避免回溯）。 经典结果是，如果从一个端点开始并在另一个端点结束，则与将所有边加倍相比，节省的总成本等于这两个端点之间的距离。 

因此，对于单个鼓风机，最佳策略是通过选择使它们的距离最大化的起始节点和结束节点来确定，该距离恰好是树的直径。 

对于两个鼓风机，关键的认识是它们不会限制彼此的移动，因为仍然允许在已经清除的边缘上进行遍历。 这意味着该问题不会成为结构意义上的划分问题。 相反，重要的是通过同时拥有两个遍历源可以减少多少不可避免的“双重遍历成本”。

正确的思考方法是，总成本从所有边权重之和的两倍开始，这对应于在两个方向上遍历每条边。 每次我们设法对齐移动，以便边缘被不同的路径“从两端”有效覆盖时，我们就准确地保存了简单路径所贡献的距离。 树中这种最大可能的节省仍然由直径结构决定，因为最长的简单路径是唯一可以最大化重叠的地方。 

因此，最佳策略最终仅取决于树的总重量及其直径。

 | --- | --- | --- | --- |
 | 暴力边缘分配 | 指数| O(n) | 太慢了|
 | 最佳直径减小 | O(n) | O(n) | 已接受 |

 ## 算法演练

 ### 关键思想：将问题简化为总边缘重量和直径

 我们计算两个量：树中所有边权重的总和以及树的直径。 

1. 计算所有边的总权重。 这代表了基线成本结构，其中每条边都有助于遍历工作。 
2. 使用两次 BFS 或 DFS 遍历计算树的直径。 首先从任意节点运行以找到最远的节点，然后从该节点再次运行以找到最远的距离。 所得距离就是直径。 
3. 使用公式总成本 = 2 * sum_of_edges - 直径合并这些值。 

这种组合背后的直觉是，如果每个鼓风机独立执行完整的遍历策略，则每个边自然会被计算两次。 减少这种重复的唯一方法是沿着最长的路径对齐移动，以便两次遍历以消除沿着该路径的一次完整遍历的方式“相遇”。 直径反映了树中可能的最大减少量。 

### 为什么它有效

 正确性取决于树的结构特性：对双重遍历的任何改进都必须对应于一条简单路径，沿着该路径两次遍历之一可以避免回溯。 由于树在节点之间具有唯一的路径，因此任何此类保存路径都必须是单个简单路径。 最长的此类路径是直径，因为用协调遍历替换该路径上的双向遍历可以准确地节省其长度。 两个步行器的配置无法产生更大的净节省，因为任何额外的重叠都受到路径的相同唯一性的限制。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n)]

total = 0

for _ in range(n - 1):
    a, b, d = map(int, input().split())
    a -= 1
    b -= 1
    g[a].append((b, d))
    g[b].append((a, d))
    total += d

def farthest(start):
    stack = [(start, -1, 0)]
    dist = [0] * n
    order = []

    while stack:
        v, p, _ = stack.pop()
        order.append(v)
        for to, w in g[v]:
            if to == p:
                continue
            dist[to] = dist[v] + w
            stack.append((to, v, w))

    far = max(range(n), key=lambda i: dist[i])
    return far, dist[far]

u, _ = farthest(0)
v, diameter = farthest(u)

print(2 * total - diameter)
```该实现构建邻接列表并一次性累积总边权重。 功能`farthest`执行迭代 DFS 来计算距起始节点的距离。 使用两次以获得直径端点。 

一个常见的实现陷阱是递归深度。 尽管递归 DFS 更简单，但 100000 个节点的约束使得 Python 中的迭代遍历更安全。 

在计算两个所需的全局属性后，直接应用最终公式。

 ## 边缘情况

 单个长链是最敏感的配置，因为所有节省都来自同一路径。 该算法通过确保直径等于边的总和来处理此问题，从而最大程度地消除冗余遍历。 

星形树测试局部分支是否干扰全局距离选择。 在这种情况下，直径始终位于两片叶子之间，并且算法正确地忽略了中心分支，因为它不会导致更长的路径。 

长链条连接到短树枝的高度不平衡树的行为与链条情况类似。 基于 DFS 的直径计算仍然可以识别正确的端点，因为无论分支因子如何，距离累积都会正确传播。
