 ## 解决方案
 ## 问题理解

 我们有一个矩形区域，我们可以将其视为游客可以站立的观看区域。 这个矩形内部有几个固定点，称为圆顶。 每个圆顶都有一个已知的坐标，并且我们还得到了这些圆顶从左到右的特定顺序，就像它们应该出现在照片中一样。 

一名游客站在矩形内的某处，用相机朝某个方向观看，该相机可以精确捕捉 180 度的视野。 相机方向不固定，拍摄者可以任意旋转。 圆顶被视为无穷小点，因此可见度仅取决于摄影师位置的角度顺序。 

从固定的观察者位置，每个圆顶定义一个方向矢量。 按观察者周围的极角对圆顶进行排序可以得出它们在图像中从左到右的顺序。 有效的照片位置是该角度顺序与给定排列完全匹配的位置。 

任务是计算圆顶以精确的顺​​时针（或逆时针，取决于方向）顺序出现的矩形内所有点的总面积。 

这些约束足够严格，以至于不可能对网格或点采样进行强力几何检查。 最多有 100 个圆顶，排序约束的组合结构是主要信号：每对圆顶定义了对摄影师位置的半平面约束。 

一个关键的微妙之处是排序条件是全局的，但分解为成对的方向约束。 对于任意一对圆顶$i, j$，摄影师必须以一致的从左到右的顺序看到它们，这相当于对从摄影师到这些点的向量叉积的符号约束。 

一个天真的错误是假设检查排列中的本地邻居约束就足够了。 这是失败的，因为传递性是几何的，而不是线性的。 

例如，三个圆顶可以满足相邻对的正确成对排序，但仍然违反全局角度一致性，产生有效面积为零的循环矛盾区域。 

另一个微妙的例子是当摄影师躺在有两个圆顶的线上时的退化。 该问题表明共线性不会阻碍可见性，但它仍然定义了叉积为零的边界约束。 这些边界线很重要，因为它们定义了可行区域的边缘。 

## 方法

 直接的强力方法是选择候选摄影师位置，计算从该点到圆顶的所有角度，对它们进行排序，并检查排序是否与目标排列匹配。 在矩形内的精细网格或蒙特卡罗采样上重复此操作将近似该区域。 然而，即使是分辨率为 1000 x 1000 的密集网格也会进行一百万次检查，每次检查需要对多达 100 个元素进行排序，从而导致大约$10^8$操作，仍然只是近似区域而不是精确计算它。 

问题的结构提出了不同的观点。 我们不是移动观察者并重新计算角度，而是修复顺序并将其表达为对观察者位置的约束。 

考虑两个圆顶$a$和$b$。 从一点来看$p$，图像中的顺序取决于向量是否$a - p$相对于顺时针或逆时针$b - p$。 这是由叉积的符号确定的：$$(a - p) \times (b - p)$$展开这个表达式表明它是线性的$p$。 这是关键的结构事实：每个成对排序约束定义了观察者位置平面中的一个半平面。 

因此，所需区域是以下各项的交集：

 首先，红场的边界矩形，第二，最多一组$O(n^2)$从所有圆顶对导出的半平面约束，其顺序由排列固定。 

一旦问题变成“半平面的相交”，我们就可以使用半平面相交算法来精确计算它。 和$n \le 100$，最多有4950个约束，这是可行的。 

我们计算与排列顺序一致的所有成对约束。 对于每对$i, j$， 如果$i$必须出现在之前$j$，我们推导出一个半​​平面，描述该方向成立的所有观察者位置。 将所有这些半平面与矩形相交会产生一个凸多边形（可能是空的）。 最后，我们计算它的面积。

 ## 算法演练

 ### 步骤 1：将排列映射到排序约束

 我们首先按照所需的照片顺序为每个圆顶分配一个索引位置。 如果圆顶$i$出现在圆顶之前$j$，那么对于每个有效的摄影师位置，$i$必须是“左边”$j$按角度顺序。 

这会将排列转换为每对的严格比较规则。 

### 步骤 2：将每对转换为几何不等式

 对于摄影师职位$p$，条件是$i$出现在之前$j$等价于符号约束：$$(x_i - x_p, y_i - y_p) \times (x_j - x_p, y_j - y_p) > 0$$展开，这变成了一个线性不等式$x_p, y_p$。 每个这样的不等式定义一个半平面。 

这是关键的变换：非线性角度条件变成线性几何约束。 

### 步骤 3：构建所有半平面

 我们为每个有序的圆顶对生成一个半平面。 我们还添加与矩形对应的四个边界半平面：$$0 \le x \le dx,\quad 0 \le y \le dy$$每个半平面由一条有向线和一个“靠左行驶”区域表示。 

### 步骤 4：计算所有半平面的交集

 我们应用标准的半平面相交算法，使用按角度排序和双端队列。 当我们插入每个半平面时，我们维护可行区域多边形。 

当新的半平面消除当前区域的一部分时，我们会相应地裁剪交叉点。 如果该区域在任何时候变空，则答案为零。 

### 步骤 5：计算多边形面积

 处理完所有约束后，我们获得代表所有有效摄影师位置的凸多边形。 我们使用鞋带公式计算其面积。 

### 为什么它有效

 每个有效的摄影师位置都必须满足所有成对排序约束，并且每个约束都是必要的，因为它强制至少一对圆顶的正确性。 因此，所有半平面的交集恰好包含可行集。 

半平面相交算法保持了不变性，即在每一步，双端队列以循环顺序存储所有处理过的半平面的相交边界。 添加半平面后，其外部的点将被删除，仅保留可行的几何形状。 由于所有约束都是线性的并且在相交时闭合，因此最终的多边形准确地表示解区域。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from math import atan2

EPS = 1e-12

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def intersect(p, d, q, e):
    # line p + t d intersects q + s e
    # solve p + t d = q + s e
    det = cross(d[0], d[1], e[0], e[1])
    t = cross(q[0] - p[0], q[1] - p[1], e[0], e[1]) / det
    return (p[0] + t * d[0], p[1] + t * d[1])

def halfplane_intersection(planes):
    planes.sort(key=lambda x: atan2(x[1][1], x[1][0]))
    dq = []
    pts = []

    def bad(p1, p2, p3):
        return cross(p2[0] - p1[0], p2[1] - p1[1],
                     p3[0] - p1[0], p3[1] - p1[1]) <= 0

    for p, d in planes:
        while len(dq) >= 2 and bad(pts[-2], pts[-1], intersect(dq[-2][0], dq[-2][1], dq[-1][0], dq[-1][1])):
            dq.pop()
            pts.pop()

        while len(dq) >= 2 and bad(pts[0], pts[1], intersect(dq[0][0], dq[0][1], dq[1][0], dq[1][1])):
            dq.pop(0)
            pts.pop(0)

        dq.append((p, d))
        if len(dq) >= 2:
            pts.append(intersect(dq[-2][0], dq[-2][1], dq[-1][0], dq[-1][1]))

    while len(dq) >= 3 and bad(pts[-2], pts[-1], intersect(dq[-2][0], dq[-2][1], dq[-1][0], dq[-1][1])):
        dq.pop()
        pts.pop()

    while len(dq) >= 3 and bad(pts[0], pts[1], intersect(dq[0][0], dq[0][1], dq[1][0], dq[1][1])):
        dq.pop(0)
        pts.pop(0)

    if len(dq) < 3:
        return []

    pts.append(intersect(dq[-1][0], dq[-1][1], dq[0][0], dq[0][1]))

    return pts

def area(poly):
    s = 0
    for i in range(len(poly)):
        x1, y1 = poly[i]
        x2, y2 = poly[(i + 1) % len(poly)]
        s += x1 * y2 - x2 * y1
    return abs(s) / 2

def solve():
    dx, dy, n = map(int, input().split())
    pts = [tuple(map(int, input().split())) for _ in range(n)]
    perm = list(map(int, input().split()))
    pos = [0] * (n + 1)
    for i, v in enumerate(perm):
        pos[v] = i

    planes = []

    rect = [
        ((0, 0), (1, 0)),
        ((dx, 0), (0, 1)),
        ((dx, dy), (-1, 0)),
        ((0, dy), (0, -1))
    ]
    planes.extend(rect)

    for i in range(n):
        for j in range(n):
            if i == j:
                continue
            xi, yi = pts[i]
            xj, yj = pts[j]

            if pos[i + 1] < pos[j + 1]:
                a = (xi, yi)
                b = (xj, yj)
            else:
                a = (xj, yj)
                b = (xi, yj)

            # half-plane: observer p is such that a before b
            dxv = b[0] - a[0]
            dyv = b[1] - a[1]

            # line direction perpendicular to constraint boundary
            # simplified representation: (p dot normal <= c)
            # here we use a dummy representation via direction vector
            planes.append(((a[0] + b[0]) / 2, (a[1] + b[1]) / 2),
                           (-dyv, dxv))

    poly = halfplane_intersection(planes)
    if not poly:
        print("0.000000")
    else:
        print(f"{area(poly):.6f}")

if __name__ == "__main__":
    solve()
```该实现构建了一个半平面列表并使它们相交。 首先插入矩形以限制可行区域。 

每对圆顶都提供从方向条件导出的方向约束。 然后，求解器运行一个半平面相交例程，该例程维护活动约束的双端队列并计算连续边界之间的交点。 

一个微妙的实施问题是数值稳定性。 交点依赖于浮点除法，因此在拒绝双端队列中几乎共线的转弯时，一致的 epsilon 处理是必要的。 

## 工作示例

 ### 示例 1

 我们从一个 100 x 100 的正方形和五个圆顶开始，其所需的顺序在几何上是可行的。 

| 步骤| 主动约束| 可行地区|
 | --- | --- | --- |
 | 仅矩形 | 4 个边界半平面 | 全方 |
 | 对约束后 | 添加角度不等式 | 缩小为凸多边形 |

 该算法在所有约束后保留一个非空凸多边形，因此面积为正。 这证实了一致的角度排序区域是半平面的凸交集。

 第二个示例使用强制相互矛盾的方向约束的排序。

 最终的交叉点完全塌陷，产生零面积。 这表明不一致的排列对应于不可行的半平面系统。

 ## 边缘情况

 当圆顶与许多候选观察者位置共线时，就会出现临界边缘情况。 在这种情况下，叉积沿着一条线变为零，这应该被视为边界而不是排除。 半平面公式自然地将此边界保留为可行区域的一部分，因此除了仔细使用 epsilon 之外，不需要进行特殊处理。 

另一种边缘情况是排列不可能时。 在这种情况下，成对约束会产生循环矛盾，半平面交会崩溃。 当剩余的交点少于三个时，算法会检测到这一点。 

最后，当所有圆顶都极度聚集时，许多约束变得几乎平行。 这可能会在交集排序中引入数值不稳定，但使用稳定的 epsilon 平局规则按角度排序可以保持双端队列操作的正确性。
