 ## 解决方案
 ## 问题理解

 我们在 d 维欧几里得空间中给出了几个点。 每个点都是一个已知向量，对于每个点，我们还给出了到未知隐藏向量的精确欧几里德距离。 任务是重建 R^d 中与所有这些距离约束一致的任何向量。 

换句话说，我们在 R^d 中得到点 x₁、x2、…、xₙ 和值 e₁、e2、…、eₙ，这样就存在某个未知点 p，其中对于每个 i，欧几里德距离 ‖p − xᵢ‖ 等于 eᵢ。 我们必须输出同时满足所有方程的任何有效 p，直至浮点误差。 

每个约束定义一个以 xᵢ 为中心、半径为 eᵢ 的球体，隐藏点位于所有这些球体的交点处。 任务是在高维中找到一个这样的交点，其中 d 和 n 都达到 500。 

约束足够大，任何将其视为符号代数或枚举候选解决方案的方法都是不可行的。 我们也不能依赖离散结构，因为所有值都是实值并且以高精度给出。 

一个微妙的边缘情况是当所有球体相交于一个点时与当系统是超定的但由于构造而仍然一致时。 另一个极端情况是某些距离为零，这意味着隐藏点与已知向量完全重合。 例如，如果一个约束是 x = (1,2)，且 e = 0，则答案必须与该点完全匹配，并且如果处理不当，基于投影的方法中的任何数值不稳定都可能会失败。 

## 方法

 关键的困难在于每个约束在未知向量 p 的坐标中都是二次的。 直接求解 d 个变量中的 n 个二次方程并不是我们可以通过这种规模的标准代数消元法来完成的。 

一个蛮力的想法是将 p 视为未知，并尝试使用通用非线性求解器迭代求解系统。 例如，我们可以最小化平方误差函数：

 Σᵢ (‖p − xᵢ‖² − eᵢ²)²

 使用梯度下降。 虽然这在概念上是正确的，但它在数值上很脆弱，并且在最坏情况下太慢，因为每次迭代的成本为 O(nd)，并且在有限数量的步骤内不能保证收敛。 当 d 和 n 达到 500 时，即使 10⁴ 迭代也已经变得昂贵。 

重要的结构观察是两个距离方程相减消除了“p”²中的二次项。 扩展：

 ‖p − xᵢ‖² = ‖p‖² − 2 p·xᵢ + ‖xᵢ‖²

 如果我们取约束 i 和约束 1 之间的差值，则 ‖p‖² 项取消，在 p 中留下线性方程：

 2 p·(xᵢ − x₁) = ‖xᵢ‖² − ‖x₁‖² + e₁² − eᵢ²

 这将整个问题转化为求解 d 变量的线性方程组。 我们最多得到 n − 1 个 d 个未知数的线性方程。 由于 d ≤ 500，我们可以使用高斯消元法求解该方程组。 

在几何上，每对球体定义一个包含所有球体交集的超平面。 与所有这些超平面相交即可恢复隐藏点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 直接非线性求解| O(k·n·d) | O(k·n·d) | O(d) | 太慢/不稳定|
 | 线性化+高斯消去| O(d3 + n·d2) | O(d²) | 已接受 |

 ## 算法演练

 我们选择第一个点作为参考，并将所有距离约束转换为未知向量 p 中的线性方程。

1. 固定第一个已知向量x₁及其距离e₁作为参考约束。 
2. 对于从 2 到 n 的每个 i，展开 i 和 1 的两个平方距离方程。减去它们以消除 p 中的二次项。 这会产生 aᵢ · p = bᵢ 形式的线性方程，其中 aᵢ = xᵢ − x₁ 和 bᵢ 根据已知常数计算。 减法是使系统线性化的关键步骤。 
3. 收集最多 n − 1 个这样的方程。 每个方程都存在于 d 维空间中，因此我们将其解释为线性系统 A p = b。 
4. 使用增广矩阵上的高斯消元法求解该系统。 我们逐列进行，选择具有不可忽略系数的主元行，将其交换到位，并从所有其他行中消除该变量。 
5. 消元完成后，回代得到p的所有坐标。 
6. 输出结果向量。 

### 为什么它有效

 每个变换都会保留解集，因为减去两个有效距离方程会删除一个公共项，而不会引入近似值。 满足所有原始球方程的任何点必须满足每个导出的线性方程。 相反，线性系统的任何解都满足从原始系统导出的所有成对一致性约束。 由于系统是由一致的几何配置构建的，因此这些超平面的交点与球体的原始交点重合，这正是隐藏向量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    d, n = map(int, input().split())
    xs = []
    es = []
    
    for _ in range(n):
        *coords, e = map(float, input().split())
        xs.append(coords)
        es.append(e)

    x1 = xs[0]
    e1 = es[0]

    # We build augmented matrix: (n-1 equations) x d variables + RHS
    A = []
    b = []

    for i in range(1, n):
        row = [xs[i][j] - x1[j] for j in range(d)]
        rhs = 0.0

        xi = xs[i]
        for j in range(d):
            rhs += xi[j] * xi[j] - x1[j] * x1[j]
        rhs += e1 * e1 - es[i] * es[i]

        A.append(row)
        b.append(rhs)

    m = len(A)
    aug = [A[i] + [b[i]] for i in range(m)]

    r = 0
    c = 0
    nvar = d

    for c in range(nvar):
        pivot = r
        for i in range(r, m):
            if abs(aug[i][c]) > abs(aug[pivot][c]):
                pivot = i
        if abs(aug[pivot][c]) < 1e-12:
            continue
        aug[r], aug[pivot] = aug[pivot], aug[r]

        div = aug[r][c]
        for j in range(c, nvar + 1):
            aug[r][j] /= div

        for i in range(m):
            if i != r and abs(aug[i][c]) > 1e-12:
                factor = aug[i][c]
                for j in range(c, nvar + 1):
                    aug[i][j] -= factor * aug[r][j]

        r += 1
        if r == m:
            break

    p = [0.0] * d
    for i in range(d):
        p[i] = aug[i][d] if i < len(aug) else 0.0

    print(*p)

if __name__ == "__main__":
    solve()
```第一阶段构建由减去平方距离方程导出的线性系统。 行向量存储系数 (xᵢ − x₁)，右侧编码平方范数的常数差加上距离调整。 

高斯消元法是在增广矩阵上进行的。 枢轴选择使用部分枢轴以避免系数接近零时的数值不稳定。 行被归一化，然后在所有其他方程中消除，这确保我们得到简化的行梯形形式。 

最后的提取步骤假设解是一致的并且系统具有满秩 d。 在发现较少主元的退化情况下，剩余坐标默认为零，但在随机生成模型下，系统几乎肯定是满秩的。

 ## 边缘情况

 当其中一个距离为零时，就会出现临界边缘情况。 在这种情况下，隐藏点必须与该输入向量完全一致。 线性化后，系统仍然可以正确编码，因为减去零距离方程会产生与 p 等于该点一致的约束。 然而，在数值上，如果高斯消除选择不同的主元顺序，浮点噪声可能会使解稍微漂移，因此需要部分主元旋转来保持稳定性。 

另一种情况是当所有点都位于低维仿射子空间中时，使得线性系统存在秩不足。 例如，在 3D 中，如果所有约束都折叠到一个平面上，则消除将产生少于 d 个枢轴。 该算法仍然继续进行，但剩余坐标默认为零或任意值。 在随机构造假设下，这种情况极不可能发生，但实施过程中不得过早假设满秩。 

最后一个微妙的情况是近乎退化的系统，其中两个约束产生几乎相同的行。 如果没有旋转，除以接近零的系数会放大浮点误差。 行交换策略确保始终选择最大的可用主元，从而将数值稳定性保持在所需的 1e-5 容差范围内。
