 ## 解决方案
 ## 问题理解

 我们得到一行块，每个块都有一定数量的树。 还给出了一系列步行，其中每次步行覆盖从一个端点到另一个端点（包括端点）的连续块段。 对于每次步行，克莱尔都会经过该时间间隔内每个街区上的所有树木，任务是计算她在所有步行中经过的树木总数。 

直接阅读该问题表明我们需要反复计算固定数组的许多子数组的值之和。 数组长度最大为200，查询次数可达10000。 每个值都很小，但在多个范围内的积累才是重要的。 

这些约束立即表明，从头开始重新计算每个段的总和是临界的，但可能是可以接受的。 然而，每次扫描最多 200 个元素执行 10,000 次查询会导致大约 200 万次操作，这已经接近 Python 中 1 秒限制下舒适的上限。 任何额外的开销或多个测试用例都会带来风险。 更干净的解决方案应该避免重复遍历。 

一个微妙的点是每个步行端点可以按任何顺序给出。 从块 5 到块 2 的步行与从 2 到块 5 的步行相同。任何假设 A_i 始终小于或等于 B_i 的解决方案都会失败。 

另一个常见的错误是忘记包容性。 由于两个端点都包含在内，因此像 1 到 1 这样的段仍然会贡献该单个块的值，而不是零。 

当步行范围折叠成单个块或所有树木都集中在一个位置时，就会出现边缘情况。 例如，如果我们有 N = 1、T = [7] 和单个查询 (1, 1)，则正确答案为 7。使用半开区间或不正确切片的粗心实现将产生 0。 

## 方法

 最直接的方法是通过迭代其范围内的所有块并对树求和来独立处理每次行走。 这很简单：对于每个查询，我们对端点进行标准化，以便我们始终从较小的索引迭代到较大的索引，然后累积该区间内的值。 

这种做法是正确的，因为它直接遵循问题的定义，没有进行任何变换。 然而，它的成本与每个查询间隔的大小成线性比例。 在最坏的情况下，每个查询几乎跨越整个数组，导致大约 M × N 次操作，在最坏的情况下约为 2 × 10^6 次操作。 这仍然勉强可行，但它不必要地紧张并且做了重复的工作。 

关键的观察是底层数组是静态的。 每个查询都要求对固定数组范围进行求和，并且有很多这样的查询。 这种结构正是前缀和的设计目的。 通过预先计算一次累积和，可以通过减去两个前缀值在恒定时间内回答每个查询。 重复范围求和的整个成本分解为单个预处理步骤。 

蛮力方法之所以有效，是因为数据很小，但在效率裕度和可扩展性方面却失败了。 前缀和变换通过重用先前计算的部分结果来代替重复遍历。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(明尼苏达) | O(1) | O(1) | 严格限制下速度太慢 |
 | 前缀和| O(N + M) | O(N) | 已接受 |

 ## 算法演练

 我们将数组转换为允许恒定时间范围查询的结构。

1. 读取树数组并构建一个前缀和数组，其中每个位置存储从开始到该索引的树总数。 这允许立即检索任何前缀段总和而不是重新计算。 
2. 对于每个查询，读取端点 A 和 B。由于行走的方向并不重要，因此对它们重新排序，使较小的索引排在前面。 这确保我们始终使用有效的递增间隔。 
3. 使用前缀数组计算该区间内树的总和。 从 L 到 R 的总和是通过从 R 处的前缀值减去 L 之前的前缀值获得的。这是有效的，因为前缀和将所有贡献累加到一个点，而减法会删除该间隔之前的所有内容。 
4. 将计算值添加到运行总计中，因为问题要求的是所有组合行走的总和，而不是每个查询输出的总和。 
5. 处理完所有查询后，输出累计总数。 

### 为什么它有效

 前缀和数组对每个位置存储其之前所有元素的总贡献的不变量进行编码。 任何范围总和都可以表示为两个此类累积值的差，因为除了严格在区间内的贡献外，所有重叠贡献都会抵消。 由于每个查询都减少为对预先计算值的恒定时间算术运算，因此不会发生重新计算或重叠歧义，并且每个块对于包含它的每个查询只贡献一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    t = list(map(int, input().split()))
    
    prefix = [0] * (n + 1)
    for i in range(1, n + 1):
        prefix[i] = prefix[i - 1] + t[i - 1]
    
    total = 0
    for _ in range(m):
        a, b = map(int, input().split())
        if a > b:
            a, b = b, a
        total += prefix[b] - prefix[a - 1]
    
    print(total)

if __name__ == "__main__":
    solve()
```该解决方案首先构造一个前缀和数组，其中索引 i 表示从块 1 到块 i 的树总数。 这是通过输入数组的单次线性传递完成的。 

然后独立处理每个查询。 唯一微妙的实现细节是端点的标准化，确保左边界不大于右边界。 减法`prefix[b] - prefix[a - 1]`无需循环或切片即可正确捕获包含区间的总和。 

一个常见的陷阱是忘记前缀索引需要偏移量，因为前缀 [0] 代表空和。 这就是我们访问的原因`a - 1`即使 a 为 1 也是安全的，因为 prefix[0] 被定义为 0。

 ## 边缘情况

 单块输入测试实现是否正确处理零前缀索引。 有输入`1 1 / 7 / 1 1`，前缀数组变为`[0, 7]`，查询计算结果为`prefix[1] - prefix[0]`，产生 7。 

反向查询例如`3 1`验证是否应用了标准化。 如果没有交换，简单的循环或切片将错误地处理范围或产生空段。 交换后，间隔变得有效，并且前缀减法产生全部总和。 

均匀数组强调重复累积的一致性。 由于每个块都具有相同的值，因此每个范围总和与其长度成正比，并且前缀减法可以在多个查询中准确地保留这一点。
