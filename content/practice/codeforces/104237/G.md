 ## 解决方案
 ## 问题理解

 我们获得了一系列课程。 每门课程都有两个附加值：分数和单元数。 我们需要恰好选择 K 门课程。 一旦做出选择，最终成绩的计算方式为所选课程的总分除以这些课程的总单元数。 

目标是选择大小为 K 的子集来最大化该比率。 这不是一个简单的“选择最大分数”或“单独选择最佳比率”问题，因为分母取决于所选的集合，因此集合的值不能以直接的方式相加。 

约束达到 N = 100000，因此任何尝试大小为 K 的所有子集的方法都是不可能的。 即使检查单个子集也是 O(K)，并且枚举组合会发生组合爆炸。 这促使我们寻求一种无需显式构建子集即可评估候选答案的方法。 

有一些边缘情况打破了天真的直觉。 

如果 K = 1，答案就是 v_i / u_i 的最大值。 任何假设项目之间相互作用的算法都会使这种情况变得过于复杂，但它仍然必须自然落下。 

如果 K = N，则答案是固定的，因为必须选择所有项目，因此结果为 sum(v_i) / sum(u_i)。 任何仍然尝试优化的选择逻辑都是多余的，并且如果实施不正确，可能会引入数值不稳定。 

当使用贪婪策略时，会出现一个更微妙的问题。 例如，按 v_i / u_i 排序并取前 K 个会失败，因为组合项目会更改分母结构。 具有巨大单位的高比率项目可能会拖累总数，即使它看起来是局部最优的。 

## 方法

 暴力方法会尝试恰好 K 个课程的每个子集，计算结果比率，并跟踪最佳的。 根据定义，这是正确的，因为它直接评估目标。 然而，子集的数量是$\binom{N}{K}$，即使对于中等的 N，它也会变得天文数字般大。对于高达 100000 的 N，这是完全不可行的。 

更结构化的观点来自于重写问题。 我们试图最大化总和的比率，这建议比较候选答案而不是直接构建子集。 假设我们猜测最优值是某个数字 x。 如果我们可以检查是否存在大小为 K 的子集：

 总和(v) / 总和(u) ≥ x

 我们可以验证x的可行性。 重新整理这个不等式可以得到：

 sum(v) − x * sum(u) ≥ 0

 现在表达式变成了元素的加法。 对于每个课程 i，定义一个转换值：

 分数_i(x) = v_i − x * u_i

 如果我们选择 K 门课程，则总数变为 sum(score_i(x))。 对于固定的 x，大小 K 的最佳可能子集就是具有最大 Score_i(x) 的 K 个项目。 这是因为我们想要最大化总和，并且每个项目的贡献是独立的。 

那么问题就变成了：对于一个候选x，检查K个最大的变换值之和是否非负。 该检查在 x 中是单调的。 如果x太大，即使最好的K项也无法达到零。 如果x足够小，则可行。 这种单调性允许对 x 进行二分搜索。 

这将问题从组合选择转化为在固定公式下对变换后的值进行重复排序，这对于 N = 100000 来说足够有效。

 | --- | --- | --- | --- |
 | 蛮力 | O( C(N, K) · K ) | O( C(N, K) · K ) | O(K) | 太慢了|
 | 最优（二分查找+贪心检查） | O(N log N log R) | O(N log N log R) | O(N) | 已接受 |

 ## 算法演练

 ### 1. 将目标值解释为参数x

 我们假设考生的答案 x 代表最终成绩。 我们不是直接计算最佳比率，而是测试 x 是否可以实现。 

### 2.将每门课程转化为线性贡献

 对于每门课程 i，计算 Score_i = v_i − x * u_i。 这会将比率约束转换为总和约束。 

### 3. 对转换后的值进行排序

 按降序对所有 Score_i 进行排序，以便最佳贡献者位于前面。 

这样做的原因是，对于固定的 x，每个项目都独立贡献，因此选择最佳的 K 个贡献可以使总数最大化。 

### 4. 取出最好的 K 个项目并计算它们的总和

 计算 S = 已排序转换数组中前 K 个元素的总和。 这表示在恰好选择 K 个项目的约束下 sum(v − x u) 的最佳可能值。 

### 5.检查x的可行性

 如果 S ≥ 0，则存在 K 个项目的有效选择，其平均值至少为 x。 否则，x 太大。 

### 6.二分查找最大可行x

 搜索真实值。 下限可以从 0 开始，上限可以设置为 max(v_i / u_i) 或足够大的数字，例如 1e6。 

每次迭代都会细化估计，直到精度达到所需的 1e-6。 

### 为什么它有效

 核心不变量是 x 的可行性是单调的。 如果值 x 是可实现的，则任何较小的值也是可实现的，因为减去较小的 x * u_i 会增加每个变换后的分数。 这保证了二分搜索空间是有序的，并且前 K 个变换值的贪婪选择正确地表示了每个固定 x 的最佳可能子集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check(n, k, u, v, x):
    arr = [v[i] - x * u[i] for i in range(n)]
    arr.sort(reverse=True)
    s = sum(arr[:k])
    return s >= 0

def solve():
    n, k = map(int, input().split())
    u = []
    v = []
    for _ in range(n):
        ui, vi = map(int, input().split())
        u.append(ui)
        v.append(vi)

    lo, hi = 0.0, 1e6
    for _ in range(60):
        mid = (lo + hi) / 2
        if check(n, k, u, v, mid):
            lo = mid
        else:
            hi = mid

    print(f"{lo:.6f}")

if __name__ == "__main__":
    solve()
```函数 check 构建候选平均值 x 的变换值并评估是否存在有效子集。 需要排序是因为固定 x 的最佳子集始终是 K 个最大的变换贡献。 

二分搜索运行固定次数的迭代，以保证精度超过 1e-6。 浮点稳定性就足够了，因为比较仅取决于符号，而不取决于精确的幅度。

 ## 边缘情况

 当 K 等于 1 时，该算法仍然有效，因为前 K 个变换值减少到单个最大元素，并且二分搜索收敛到最佳个体比率。 例如，如果一门课程主导所有其他课程，则其转换值在正确的 x 范围内保持最高，并且可行性恰好按照其比率翻转。 

当 K 等于 N 时，排序选择步骤变得无关紧要，因为所有元素都已被采用。 可行性检查简化为 sum(v) − x sum(u) ≥ 0，直接得出 x = sum(v) / sum(u)。 该算法仍然正确收敛，因为不存在子集竞争。 

当值极度倾斜时，例如一门课程具有非常大的 u 和 v，转换 v − x u 会根据 x 正确地对其进行惩罚或奖励。 即使 v/u 很高，大的 u 也能确保对 x 的敏感度，从而防止在整个搜索范围内出现错误的优势。
