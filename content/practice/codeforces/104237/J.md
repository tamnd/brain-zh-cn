 ## 解决方案
 ## 问题理解

 我们得到一组谷仓，在图中排列为顶点。 每次乔到达那里时，每个谷仓都会给他带来固定的利润。 谷仓之间有定向道路，每条道路都有相关费用。 除了这些道路之外，乔还可以从任何谷仓传送到任何其他谷仓，并支付与指数绝对差成比例的基于距离的成本。 通过任何方式到达谷仓后，乔立即领取该谷仓的奖励。 

问题不在于找到一条最佳路径。 相反，我们被问到是否存在任何方式来旅行并返回到之前访问过的状态，使得整个周期的总增益严格为正。 如果存在这样的循环，乔就可以无限重复，积累无限的利润。 

约束表明谷仓的数量最多为 500 个，而道路的数量最多为 5000 个。明确模拟长移动序列或探索所有路径的完全幼稚的方法是不可能的，因为即使对状态进行三次或四次探索也已经太大了。 然而，$N^2$尺度结构是可以接受的，这表明如果每次迭代都是$O(N^2)$或更好。 

隐形传态产生了一种微妙的边缘情况。 因为传送边存在于每对谷仓之间，所以显式构造所有传送边的粗心实现已经是$O(N^2)$边，这很好，但是如果在没有优化的情况下以贝尔曼-福特风格天真地完成，在所有边上重复运行通用最短路径算法会变得太慢。 

另一个重要的细节是，每次进入谷仓时都会收集奖励，包括在传送之后。 这意味着节点权重属于边的目的地，而不是源。 缺少此细节会导致符号建模不正确。 

最后，目标是检测任何正循环，而不是计算到特定节点的路径。 这将该问题与标准最短路径任务区分开来。 

## 方法

 直接建模方法是将每个谷仓解释为一个节点，将每个移动解释为一条有向边，其权重等于该移动的净增益。 如果乔搬出谷仓$i$到谷仓$j$，他支付移动费用并立即收到$c_j$，所以边权重为：$$w(i \to j) = c_j - \text{cost}(i, j)$$对于道路，给出了成本。 对于隐形传送，成本为$K \cdot |i - j|$。 

一旦这样定义了图，问题就变成了检测是否存在总权重为正的循环。 

一个蛮力的想法是运行最长的路径松弛长达$N$使用所有边进行迭代。 如果有任何价值改善$N$第-次迭代，存在正循环。 这是经典的贝尔曼-福特循环检测思想，但采用最大化形式。 

然而，该图包含每对节点之间的传送边，即$O(N^2)$。 一个天真的贝尔曼福特将花费$O(N^3)$， 大约$500^3 = 125$百万次松弛，考虑到开销，这在 Python 中是临界值。 

关键的观察是传送边缘具有非常结构化的权重形式：$$c_j - K|i - j|$$这使我们能够在每次迭代的线性时间而不是二次时间中计算所有传送松弛。 通过将绝对值分为两种情况，我们重写：$$c_j - K|i - j| = 
\begin{cases}
(c_j + Ki) - Kj & i \le j \\
(c_j - Ki) + Kj & i \ge j
\end{cases}$$这种结构允许从左到右和从右到左进行扫描，同时保留最佳候选者。 

因此每次 Bellman-Ford 迭代都可以简化为$O(N + M)$，使完整的解决方案变得可行。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力贝尔曼-福特在所有边缘 |$O(N^3)$|$O(N^2)$| 太慢了|
 | 通过传送扫描优化 Bellman-Ford |$O(N(N + M))$|$O(N + M)$| 已接受 |

 ## 算法演练

 我们将问题转化为最大权重循环检测任务。 

1. 在谷仓上构建一个有向图，其中每个状态代表刚刚到达谷仓。 初始化一个值数组`dist[i] = c_i`，代表在谷仓结束时可获得的最佳利润$i$在零动作中。 这反映出从谷仓开始立即获得奖励。 
2. 添加所有道路过渡。 对于一条从$a$到$b$有成本$w$，定义松弛规则：$$dist[b] = \max(dist[b], dist[a] + c_b - w)$$这捕获了在目的地获得奖励并支付道路成本。 
3. 由于其完整的图结构，单独处理隐形传态。 对于每个目的地$j$，我们想要：$$\max_i (dist[i] + c_j - K|i - j|)$$这分为两次扫描：

 在从左到右的扫描中，我们保持最佳值$dist[i] + Ki$,

 在从右到左的扫描中，我们保持最佳值$dist[i] - Ki$。 
4. 执行松弛过程$N$迭代。 在每次迭代中，首先应用道路松弛，然后使用两次扫描方法传送松弛。 
5.完成后$N$完整迭代，再执行一次迭代。 如果任何值提高，则存在正循环，答案是`YES`。 
6. 如果额外迭代没有改善，输出`NO`。 

### 为什么它有效

 该过程是贝尔曼-福特的最长路径模拟。 后$k$迭代，`dist[i]`代表最多使用可实现的最佳利润$k$移动。 任何增加利润的循环都必须可以无限期地使用，这意味着它最终会增加一些`dist[i]`即使在所有简单路径都用尽之后。 如果之后仍无法改善$N$步骤，不存在正循环，因为任何循环都可以在不重新访问节点超过$N$大小图中的时间$N$。 传送优化不会改变正确性，因为它计算与显式枚举完全相同的松弛。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M, K = map(int, input().split())
    c = [0] + [int(input()) for _ in range(N)]

    edges = []
    for _ in range(M):
        a, b, w = map(int, input().split())
        edges.append((a, b, w))

    dist = c[:]  # starting reward at each node

    def relax_once():
        nonlocal dist
        updated = False

        new = dist[:]

        # road edges
        for a, b, w in edges:
            val = dist[a] + c[b] - w
            if val > new[b]:
                new[b] = val
                updated = True

        # teleport: left to right
        best = -10**30
        for i in range(1, N + 1):
            best = max(best, dist[i] + K * i)
            val = best + c[i] - K * i
            if val > new[i]:
                new[i] = val
                updated = True

        # teleport: right to left
        best = -10**30
        for i in range(N, 0, -1):
            best = max(best, dist[i] - K * i)
            val = best + c[i] + K * i
            if val > new[i]:
                new[i] = val
                updated = True

        dist = new
        return updated

    for _ in range(N):
        relax_once()

    if relax_once():
        print("YES")
    else:
        print("NO")

if __name__ == "__main__":
    solve()
```该实施保持了每个谷仓的全球最佳利润。 每次迭代都会计算一个新的`new`数组以避免同一轮内混合更新，这对于贝尔曼-福特风格推理的正确性至关重要。 道路放松是简单的边缘更新。 

传送弛豫是使用两次线性扫描来计算的。 前向扫描保留左侧索引的最佳候选，调整为$+Ki$，它正确地解释了绝对距离$i \le j$。 向后扫描反映了另一个方向的相同想法。 这避免了显式迭代所有对。 

最后一个额外的放松步骤是周期检测触发器：如果之后还有什么可以改进的$N$完整的迭代，必然存在正循环。

 ## 边缘情况

 一个关键的边缘情况是，仅远距离传送就创造了唯一有利可图的结构。 如果所有道路都不存在，则完全忽略传送边缘的幼稚方法将错误地得出不存在循环的结论。 事实上，高奖励谷仓之间的传送本身就可以形成一个重复的正循环。 

当贝尔曼-福特流程后期进行改进时，就会出现另一种边缘情况。 如果实现过早检查周期，则可能会错过需要多次转换才能受益的周期。 最后的额外迭代确保正确检测到这种延迟传播。 

第三种边缘情况是当$K$足够大以至于隐形传态总是负的。 在这种情况下，所有传送松弛绝不能覆盖基于道路的结构，并且算法仍然表现正确，因为扫描松弛根本不会改善`dist`。
