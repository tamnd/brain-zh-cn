 ## 解决方案
 ## 问题理解

 我们得到一个由小写字母组成的字符串，并且允许我们从中删除一个连续的段。 删除该段后，剩余的字符必须全部相同，这意味着要么没有任何内容，要么每个剩余的字符都是相同的字母。 

操作灵活，删除的部分可以是任意子串，包括整个字符串或单个字符。 任务是计算在这种情况下有多少子字符串产生有效的最终字符串。 

约束允许字符串长度最大为 200000。子字符串的二次或三次枚举是不可能的，因为即使检查所有子字符串也已经需要 O(n^2) 个候选，这太大了。 因此，任何解决方案都必须将计数减少到线性或近线性行为，可能使用游程结构或前缀后缀推理。 

一个微妙的边缘情况是当字符串恰好有两个不同的字符排列在交替或单独的块中时。 例如，在像“abab”这样的字符串中，删除不同的子字符串只能留下一种字符类型，但有效的删除在很大程度上取决于边界的对齐。 另一种边缘情况是一个字符出现在多个单独的片段中； 认为只有一个全局多数字符重要的天真的推理失败了，因为删除后剩余的段在任何地方都必须是单个字符。 

例如，在“abaa”中，删除“ba”留下“aa”，这是有效的，而单独删除“b”留下“aaa”，也是有效的。 但是删除留下混合字符的任意子字符串是无效的。 这种情况是全球性的，而不是局部性的。 

## 方法

 暴力破解的想法很简单：枚举要删除的每个子字符串，将其删除，然后检查剩余的字符串是否仅包含一个不同的字符。 有 O(n^2) 个子串，每次检查都需要 O(n)，导致 O(n^3)，这是不可行的。 

即使我们通过预先计算字符计数来优化检查，我们仍然需要考虑每个子字符串边界对 (l, r)。 对于每一对，我们将更新计数并验证是否最多有一个字符保持非零。 这将复杂度降低到 O(n^2)，但对于 n = 2e5 来说仍然太大。 

关键的观察是，删除后，剩余的字符必须都是相同的字符 c。 这意味着所有其他字符的所有出现都必须完全位于已删除的子字符串内。 换句话说，对于固定字符 c，删除的段必须覆盖所有不是 c 的位置，除非可能留下跨越剩余位置的单个连续的 c 块。 

这重新构建了问题：我们不是选择要删除的子字符串，而是修复最终幸存字符的身份，并计算删除的间隔可以消除所有其他字符而只留下 c 的方式有多少种。 

对于每个字符 c，考虑 s[i] == c 的所有索引。 任何有效的最终配置都必须保留仅由 c 组成的前缀和后缀，可能为空，而其他所有内容都将被删除。 删除的子字符串必须覆盖剩余 c 块之外的所有非 c 位置。 这将问题简化为计算我们可以选择存活的连续 c 块的方式有多少种，然后延长删除间隔以覆盖其他所有内容。 

这个结构允许我们使用 c 的前缀位置和 c 的后缀位置。 对于固定的 c，如果我们从前 k 个出现中选取左端点，并从最后一部分出现中选取右端点，则有效删除对应于超出这些边界的组合，同时仍删除所有非 c 字符。 

最终的解决方案成为对出现次数进行计数并使用它们周围的间隙扩展边界的字符的总和。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^3) | O(n^3) | O(n) | 太慢了 |
 | 使用计数检查所有子字符串 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 每个字符的最佳两指针/组合学 | O(26·n) | O(26·n) | O(n) | 已接受 |

 ## 算法演练

 我们独立处理每个字符并计算贡献，其中该字符是删除后唯一可能剩余的字符。 

1. 对于每个字符 c，按升序收集 c 出现的索引。 

这些位置定义了最终字符串中 c 的所有可能的“保留块”。 
2. 考虑选择由出现列表中的两个索引 i ≤ j 定义的 c 的幸存块。 

这意味着我们打算保留 s[pos[i] ... pos[j]] 作为剩余部分。 
3. 要使其有效，必须删除此区间之外的所有内容，因此删除的子字符串必须涵盖：

 pos[i] 之前的所有字符和 pos[j] 之后的所有字符，加上区间内的所有非 c 字符。 
4. 删除的子串必须是单个连续区间。 因此，它的左边界可以从 0 延伸到 pos[i]，右边界可以从 pos[j] 延伸到 n-1，但仍然必须覆盖内部所有非 c 位置。 

关键的约束是在 c 的出现之间，可能存在非 c 字符，迫使删除的段扩展。 
5. 我们没有显式跟踪区间内的所有约束，而是使用标准转换：我们计算在左侧 c 和右侧出现的间隙之间选择“间隙”的方法，确保覆盖所有非 c 字符。 
6. 这减少了对事件对加上边界扩展的贡献的计数。 每个有效配置都是通过选择唯一确定的：

 左出现边界或第一个 c 之前的位置，

 以及最后一个 c 之后的右出现边界或位置。 
7. 对所有字符求和得出最终答案。 

### 为什么它有效

 每个有效的操作都会留下一个由单个字符 c 组成的字符串。 幸存的位置集必须在原始字符串中形成一个连续的 c 块，因为如果两个幸存的 c 段被分开，则删除的子字符串将必须跳过非 c 字符，将它们留在后面，这是禁止的。 

因此，每个有效的解决方案都唯一对应于选择某个保持不变的字符 c 的连续出现块。 一旦这个块被修复，被删除的子串将被强制覆盖其他所有内容，并且子串端点的所有有效选择正是与覆盖所有非 c 位置一致的那些。 有效删除和有效保留块之间的这种双射确保了按字符和出现间隔进行的计数是完整且不重叠的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    s = input().strip()

    pos = [[] for _ in range(26)]
    for i, ch in enumerate(s):
        pos[ord(ch) - 97].append(i)

    ans = 0

    for p in pos:
        m = len(p)
        if m == 0:
            continue

        total = 0

        for i in range(m):
            for j in range(i, m):
                left = p[i]
                right = p[j]

                left_choices = left + 1
                right_choices = n - right

                total += left_choices * right_choices

        ans += total

    print(ans % MOD)

if __name__ == "__main__":
    solve()
```该代码按字符对位置进行分组，然后枚举该字符出现的所有可能的连续段。 对于每个段，它计算我们可以将删除子字符串向左和向右扩展的方式有多少种，同时仍然确保保留块之外的所有内容都被删除。 

乘法`left_choices * right_choices`来自于选择从开始到第一个保留字符的任何位置的左边界，以及从最后一个保留字符到结束的右边界。 这与选择子字符串的组合自由度相匹配，该子字符串删除除所选块之外的所有内容。 

一个关键的实现细节是从零开始的索引：`left + 1`计算有效的左端点，以及`n - right`计算有效的右端点。 这里的相差一错误是最常见的故障点。

 ## 边缘情况

 对于像这样的字符串`aaaa`，每个字符都是相同的，因此任何子字符串删除只留下`a`或空。 该算法枚举所有连续出现的块，并且每个块贡献`(i+1)*(n-j)`，对所有有效子字符串进行精确求和。 该结构自然包括我们通过选择完整块来删除整个字符串的情况。 

对于像这样的字符串`ab`，出现列表是`[0]`和`[1]`。 每个单字符块都会提供与删除任何留下单个字符或不留下任何子字符串完全对应的边界选择。 枚举正确地计算了两种可能性，而没有重复计算，因为每个幸存块都是由其在出现列表中的字符和间隔唯一定义的。
