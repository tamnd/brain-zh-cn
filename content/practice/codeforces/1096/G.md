 ## 解决方案
 ## 问题理解

 我们正在使用有限的数字字母表构建固定偶数长度的数字字符串。 该字符串表示票号，但唯一重要的结构规则是每个允许的数字在字符串的每一半中使用了多少次。 

一张长度的票$n$被分成两半长度$n/2$。 我们想要计算可以形成多少个有效的全长字符串，使得左半部分的数字之和等于右半部分的数字之和。 数字可以重复，并且如果零是允许的数字集的一部分，则允许使用前导零。 

从组合的角度来看，问题在于长度序列$n$超过一个小字母表大小$k \le 10$，约束条件是前半部分的总和等于后半部分的总和。 

长度维度的约束很大：$n$可以达到$2 \cdot 10^5$。 这立即排除了任何显式处理每个字符串甚至枚举半字符串的解决方案。 甚至将所有可能的一半和存储在依赖于的朴素 DP 中$n$除非仔细构建，否则最大总和太大。 关键的观察是字母表很小，因此总和范围是可控的，但达到每个总和的方法数量巨大，需要多项式或基于 FFT 的卷积。 

一种天真的方法会尝试分别枚举所有左半部分和右半部分并匹配总和。 这失败了，因为每一半都有$k^{n/2}$的可能性，这是一个天文数字。 即使是 DP 的长度和总和$O(n \cdot 9n)$州太大了，因为$n^2$就在附近$4 \cdot 10^{10}$。 

一个微妙的边缘情况是包含数字 0 时。 它不会改变总和，但会增加组合多重性。 任何忽略数字重数并仅跟踪可达到的总和的解决方案都将严重低估。 另一个陷阱是假设对称性意味着直接分割计数而无需卷积，这会破坏，因为数字的不同分布对相同的总和的贡献不同。 

## 方法

 问题的结构自然地分为两个相同的两半。 如果我们知道，对于每个可能的总和$s$, 有多少种方法可以用数字和形成半长序列$s$，则通过将共享相同总和的两个独立的一半配对来获得完整的答案。 这将问题转化为计算分布与其自身的卷积。 

暴力方法构造所有长度的序列$n/2$并计算它们的总和。 这已经花费了$k^{n/2}$，即使对于小公司来说也是不可能的$n$。 即使我们切换到动态规划，我们也定义$dp[i][s]$作为形成长度的方法的数量$i$与总和$s$。 过渡很简单，但状态数量会增加到$O(n^2)$，以及每次转换成本$O(k)$, 给予$O(n^2 k)$，这仍然太慢了。 

关键的结构见解是一半的 DP 是固定基多项式的重复卷积。 每个位置独立地从允许的集合中贡献一位数字，因此一个位置的生成函数是一个多项式，其中系数$x^d$如果数字为 1$d$是允许的。 生成函数为$n/2$位置是这个多项式的幂$n/2$。 我们只需要它的系数，可以使用分治求幂结合基于 FFT 的模数乘法来计算$998244353$，这是 NTT 友好的素数。 

一旦我们有了系数数组$A[s]$对于一半，答案是所有的总和$s$的$A[s]^2$，因为我们独立地选择具有相同总和的左半部分和右半部分。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力枚举|$O(k^{n/2})$|$O(n)$| 太慢了 |
 | DP 长度和总和 |$O(n^2 k)$|$O(n^2)$| 太慢了 |
 | 分而治之 + NTT |$O(S \log n \log S)$|$O(S)$| 已接受 |

 这里$S$是最大可能的总和，边界为$9n/2$。 

## 算法演练

 我们将问题压缩为计算卷积下的多项式求幂。 

1. 构造一个初始多项式$P(x)$其中每个允许的数字$d$在位置贡献系数 1$x^d$。 该多项式代表半票中的单个位置。 
2. 定义指数$t = n/2$。 我们想要$P(x)^t$，但仅系数达到 度$9t$很重要，因为总和不能超过这个数字。 
3. 对多项式使用分而治之的幂运算。 我们递归计算$P^t$将指数分成两半。 在每一步中，将两个中间多项式相乘。 
4. 每个乘法都是使用 NTT 在模下完成的$998244353$。 即使对于大次多项式，这也能保持卷积高效。 
5. 计算系数数组后$A[s]$的$P(x)^t$， 解释$A[s]$作为用 sum 构建二分之一的方法数$s$。 
6. 将最终答案计算为$\sum_s A[s] \cdot A[s]$，因为左半部分和右半部分是独立的，但必须共享相等的总和。 

卷积出现的关键原因是各个位置的数字选择是独立的，并且和是线性相加的。 重复独立加法与多项式乘法完全对应。 

### 为什么它有效

 中的每个系数$P(x)^t$计算选择的方式数量$t$总和等于固定值的数字。 卷积结构保证所有组合都被精确计数一次，因为每个乘法步骤都会合并不相交位置上的独立选择。 最后的平方步骤通过匹配来自两个独立结构的相同和分布来强制左和右和的相等。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
G = 3

def ntt(a, invert):
    n = len(a)
    j = 0
    for i in range(1, n):
        bit = n >> 1
        while j & bit:
            j ^= bit
            bit >>= 1
        j ^= bit
        if i < j:
            a[i], a[j] = a[j], a[i]

    length = 2
    while length <= n:
        wlen = pow(G, (MOD - 1) // length, MOD)
        if invert:
            wlen = pow(wlen, MOD - 2, MOD)
        i = 0
        while i < n:
            w = 1
            for j in range(i, i + length // 2):
                u = a[j]
                v = a[j + length // 2] * w % MOD
                a[j] = (u + v) % MOD
                a[j + length // 2] = (u - v) % MOD
                w = w * wlen % MOD
            i += length
        length <<= 1

    if invert:
        inv_n = pow(n, MOD - 2, MOD)
        for i in range(n):
            a[i] = a[i] * inv_n % MOD

def multiply(a, b):
    n = 1
    while n < len(a) + len(b) - 1:
        n <<= 1
    fa = a[:] + [0] * (n - len(a))
    fb = b[:] + [0] * (n - len(b))
    ntt(fa, False)
    ntt(fb, False)
    for i in range(n):
        fa[i] = fa[i] * fb[i] % MOD
    ntt(fa, True)
    return fa[:len(a) + len(b) - 1]

def solve():
    n, k = map(int, input().split())
    digits = list(map(int, input().split()))

    base = [0] * (max(digits) + 1)
    for d in digits:
        base[d] = 1

    poly = base[:]
    exp = n // 2

    def poly_pow(p, e):
        if e == 1:
            return p
        if e % 2 == 0:
            half = poly_pow(p, e // 2)
            return multiply(half, half)
        else:
            return multiply(poly_pow(p, e - 1), p)

    res = poly_pow(poly, exp)

    ans = 0
    for x in res:
        ans = (ans + x * x) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```NTT实现用于加速多项式乘法，这是核心瓶颈。 这`multiply`函数确保系数数组扩展到 2 的幂，因为 NTT 需要该结构。 递归`poly_pow`通过对多项式求平方来执行求幂。 

一个微妙的点是我们只将系数保持在一定程度$9n/2$，这可以防止数组不必要的增长。 另一个细节是最终答案对系数进行平方而不是执行另一个卷积，这避免了额外的 NTT 调用。 

## 工作示例

 ### 示例 1

 输入：```
4 2
1 8
```我们构建基础多项式$P(x) = x^1 + x^8$，并将其提高到 2 次方，因为$n/2 = 2$。 

扩展后：

 | 总和 | 方式|
 | --- | --- |
 | 2 | 1 |
 | 9 | 2 |
 | 16 | 16 1 |

 现在我们对系数进行平方：

 | 总和 | 贡献 |
 | --- | --- |
 | 2 | 1 |
 | 9 | 4 |
 | 16 | 16 1 |

 总计为$6$。 

这表明在两半上独立选择的相同总和分布会产生平方贡献。 

### 示例 2

 输入：```
2 1
6
```只允许使用数字 6。 每一半都有一种形成和 6 的方法。 

| 总和 | 左路| 正确的方法 | 贡献 |
 | --- | --- | --- | --- |
 | 6 | 1 | 1 | 1 |

 最终答案是 1，符合只有“66”有效的事实。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(S \log S \log n)$| 每个多项式乘法都使用 NTT 次数$S$，求幂执行$O(\log n)$乘法|
 | 空间|$O(S)$| 我们存储多项式系数直至最大可能的总和 |

 最大总和的界限为$9n/2$，这大约是$10^5$，使得基于 FFT 的卷积在一定范围内可行。 对数幂因子仍然足够小$n$最多$2 \cdot 10^5$。

 ## 边缘情况

 当只有一位数字可用时，多项式就变成单项式。 求幂分解为一项，并且答案变得微不足道的 1，因为两半都是被迫的。 

当包含零时，它不会改变总和，但会增加卷积的多重性。 该算法正确地对其进行了计数，因为它对零阶系数质量有贡献，该系数质量通过求幂传播而不影响和结构。
