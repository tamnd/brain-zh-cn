 ## 解决方案
 ## 问题理解

 我们有一个固定的排列$p$尺寸的$n$。 我们可以选择另一种排列$q$指数的$1$到$n$。 一次$q$是固定的，我们看一下由它构建的两个序列：第一个序列只是按以下给出的顺序排列的索引$q$，第二个序列是$p$以同样的顺序进行。 

对于任何序列，我们计算一个分数，定义为前缀最大值的数量，这意味着我们遇到一个严格大于之前看到的所有元素的频率。 任务是挑选$q$使得索引序列和排列中的前缀最大计数之和$p$- 序列尽可能大，然后输出任意排列$q$达到这个最大值。 

约束允许最多$2 \cdot 10^5$跨测试用例的总元素，因此任何解决方案都必须在每个测试用例的线性时间内运行。 这立即排除了对所有排列的任何二次检查或任何重复模拟许多候选排序的方法。 

一个微妙的点是，前缀最大计数高度依赖于顺序，并且以明显的方式不是线性或相加的。 一个天真的错误是假设改进一个序列会自动改善总数，但相同的排序同时以耦合方式影响两个序列。 

揭示这种相互作用的一个小边缘情况是$p$已经在增加。 如果我们选择$q$作为身份顺序，两个序列都在增加，并且每个序列都有贡献$n$。 相反，如果我们通过一些不相关的规则对索引进行重新排序，我们可能会稍微改进一个序列，但会立即失去另一个序列中的前缀最大值，并且该损失以后无法恢复。 

## 方法

 暴力方法将枚举所有排列$q$，构建两个归纳序列，并计算每个序列的前缀最大值计数。 每次评估费用$O(n)$，并且有$n!$排列，所以即使对于非常小的总数也是不可行的$n$。 

为了减少这种情况，我们首先观察到分数的两个组成部分之一变得几乎微不足道的控制。 如果我们按递增顺序获取索引，则索引序列严格递增，这意味着每个元素都是新的前缀最大值，因此这部分的贡献正好$n$，最大可能值。 任何与递增顺序的偏差都会立即引入至少一个位置，其中较小的索引出现在较大的索引之后，从而破坏了前缀最大属性并减少了这种贡献。 

对称地，如果我们通过增加 的值来对索引进行排序$p[i]$，那么$p$-sequence 增加，其前缀最大计数也增加$n$。 这给出了第二个极端结构，其中第二项被最大化。 

关键的见解是最佳答案必须位于这两个极端顺序之一。 任何混合排序都试图同时增加两个分量，但不可避免地会在两个结构中引入损失，并且不能主导两个极端，因为每个极端已经使两项中的一项饱和$n$。 

因此，解决方案简化为计算两个候选排列并选择更好的一个：

 身份秩序$q = (1,2,\dots,n)$，并且顺序按$p[i]$。 我们评估所得分数并输出哪个更好。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(n!\,n)$|$O(n)$| 太慢了|
 | 两个极端|$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们构建两个候选排列并计算它们的分数。 

1. 构造恒等排列$q^{(1)} = (1,2,\dots,n)$。 

在这种排序中，索引序列严格递增，因此其前缀最大计数为$n$。 总分变为$n + f(p)$，因为第二个序列恰好是$p$按原来的顺序。 
2. 构建第二个排列$q^{(2)}$通过增加以下值对索引进行排序$p[i]$。 

这保证了序列$p[q^{(2)}]$严格递增，因此其前缀最大计数为$n$。 
3. 构建时$q^{(2)}$，按此顺序计算索引的前缀最大计数。 

我们扫描按排序的索引$p$排序并维持迄今为止所见的最大指数。 每次当前索引超过此最大值时，我们都会增加计数。 
4. 将第二个构造的得分计算为$n + f(q^{(2)})$，自从$p$-序列完全增加。 
5. 比较两个分数并输出给出较大值的排列。 

### 为什么它有效

 关键的属性是两个前缀最大函数的上界为$n$，并且每个极端构造都实现$n$恰好是两个序列之一。 任何其他排列都不能超过$n$在任一组件中，任何与极端排序的偏差都只会减少其中一个前缀最大计数，而不能保证另一个组件的补偿性增加超出这些构造的界限。 因此，最优解必须是两个极值排序之一。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def pref_max_count(arr):
    mx = -10**18
    cnt = 0
    for x in arr:
        if x > mx:
            cnt += 1
            mx = x
    return cnt

t = int(input())
for _ in range(t):
    n = int(input())
    p = list(map(int, input().split()))

    q1 = list(range(1, n + 1))
    f_p = pref_max_count(p)
    score1 = n + f_p

    idx = list(range(1, n + 1))
    idx.sort(key=lambda i: p[i - 1])

    f_q2 = pref_max_count(idx)
    score2 = n + f_q2

    if score1 >= score2:
        print(*q1)
    else:
        print(*idx)
```第一个候选直接使用恒等排序，其中索引序列是最大递增的，所以我们只需要计算原始的前缀最大计数$p$。 第二个候选者通过增加对索引进行重新排序$p[i]$，转动$p$-序列转换为完全递增的序列，并将复杂性转移到计算索引上的前缀最大值。 

唯一微妙的实现细节是索引：由于 Python 使用基于 0 的数组，但问题使用基于 1 的索引，因此索引的排序列表必须访问$p[i-1]$。 

## 工作示例

 考虑一个小排列$p = [2,3,1]$。 

对于身份排序$q = [1,2,3]$，索引序列为$[1,2,3]$，给出前缀最大计数$3$。 这$p$- 序列是$[2,3,1]$，其前缀最大值是$2$和$3$， 所以$f(p)=2$。 总分是$3+2=5$。 

现在对索引进行排序$p$, 给予$q = [3,1,2]$因为$p_3=1$,$p_1=2$,$p_2=3$。 这$p$- 序列变为$[1,2,3]$, 贡献$3$。 索引序列为$[3,1,2]$，其前缀最大值出现在$3$只是，所以$f(q)=1$。 总分是$3+1=4$。 身份排序更好，所以我们输出$[1,2,3]$。 

该跟踪表明，即使一个排序使$p$完美增长时，指数结构的损失可能超过收益。

 ## 边缘情况

 当$p$严格递增，恒等排列使两个序列完全递增，因此两个前缀最大计数相等$n$。 任何替代排序要么会破坏索引的单调性，要么会造成混乱$p$-单调性，因此不能超过同一性分数。 

什么时候$p$严格递减，排序依据$p$产生$p[q]$随着增加，给出一个序列分数$n$，而索引序列变得几乎随机。 在这种情况下，算法与恒等构造进行比较，并且仍然正确地选择两个极端中的较好者，因为任何混合排序仅减少两个前缀最大结构之一，而不超过这些界限。
