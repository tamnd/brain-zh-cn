 ## 解决方案
 ## 问题理解

 我们给定一个整数 X，我们考虑所有由 2 的幂组成的多重集，使得它们的总和等于 X。每个多重集只是一个集合，例如`{1, 1, 2, 8, 8}`其总和为X。 

对于每个这样的多重集 S，我们想象将其元素分为两组 S1 和 S2。 每个组都有自己的总和。 对于任何数字n，我们将highest_bit(n)定义为二进制中最高有效位的位置，因此它是满足2^k ≤ n的最大k，对于0，它被定义为-1。 

如果存在至少一种方法将多重集的元素分成两组，使得两组和具有相同的最高位，则多重集被认为是坏的。 否则就好了。 任务是计算对于从 1 到 n 的每个 X，有多少个良好的 2 的幂的多重集之和为 X。 

输入约束 n 最大为 10^6，这会立即排除任何枚举多重集或子集分割的解决方案。 即使表示单个 X 的所有多重集，通常也是指数级的。 任何有效的解决方案都必须将结构压缩为每个 X 的一维动态编程状态，每个状态最多具有对数转换或摊销常量功。 

当 X 是 2 的幂时，会出现微妙的边缘情况。 对于 X = 8，存在许多多重集，例如`{8}`,`{4,4}`,`{4,2,2}`,`{2,2,2,2}`，并且每个在分区下的行为都不同。 一个天真的想法可能假设只有二进制表示很重要，但是两个幂的重数完全改变了可以出现的子集和，因此需要超越位表示的结构。 

另一个陷阱是假设“平衡分配”仅取决于平均分配总和。 条件较弱：它只要求两个部分落在相同的二次方范围内，而不是相等的总和。 

## 方法

 我们首先看一下直接方法。 对于固定的 X，我们枚举所有 2 的幂的多重集，总和为 X。对于每个多重集，我们尝试将其元素的所有分区分为两组，并计算两组的总和，跟踪它们的最高位。 这是正确的，但完全不可行。 即使多重集的数量也大致像划分为 2 的幂的数量一样增长，这在 √X 中已经呈指数增长，并且对于每个我们还将枚举子集，从而在最坏的情况下导致双指数行为。 

关键的简化来自视角的转变。 我们不考虑被划分的元素，而是考虑 S 的子集可实现的总和。每个 2 的幂的多重集都定义了一个有界硬币系统，并且其子集和形成了结构化的区间覆盖。 “坏”条件仅取决于是否存在位于 X/2 附近特定中间区域的子集和。 

如果我们让 m = Highest_bit(X)，则任何产生相等 Highest_bit 的有效分区都必须具有 [2^{m-1}, 2^m - 1] 范围内的两个子集和。 任何其他位级别都无法工作，因为如果最高位太小，则两个总和都无法达到 X，如果太大，则一侧已经超过 X。这会将条件折叠为单个区间约束。 

因此，当多重集可以实现区间 [2^{m-1}, 2^m - 1] 中的子集和且其补集也位于同一区间内时，它就是不好的。 同样，好的多重集是其子集和完全避开该区间的多重集。 

现在是结构观察：二的幂的多重集表现得像二进制进位系统。 任何配置都可以解释为将 X 分解为分裂和合并的二叉树的一种方式。 唯一重要的是进位如何在相邻位级别之间传播。 这将全局条件简化为 X 上的一维递归，其中将 X 扩展一个单位要么附加一个新的低位结构，要么合并到更高级别的结构中。 

这导致 DP，其中 dp[X] 取决于删除最低元素（减 1）或崩溃最高幂结构（除以 2）。 这是 X 有效配置产生的仅有的两种结构上不同的方式。

 ## 算法演练

 我们计算 dp[X]，即所有 X 到 n 的总和为 X 的有效多重集的数量。

1. 我们初始化 dp[0] = 1，因为空多重集是形成 sum 0 的唯一方式。这充当所有构造的基本配置。 
2. 对于从 1 到 n 的每个 X，我们考虑两种结构方法来构建求和到 X 的多重集。第一种方法是对 X − 1 采取有效配置并附加单个 1。这对应于增加最低位的重数而不影响较高位结构。 
3. 第二种方法是采用 X // 2 的有效配置，并通过将所有元素加倍来对其进行缩放，然后根据我们可能会或可能不会根据奇偶校验引入剩余 1 的事实进行调整。 这捕获了二进制分解中的进位行为，其中添加更高的两个元素的幂可以有效地压缩两个较低级别的贡献。 
4. 我们将这两个结构组合起来形成 dp[X]，确保我们不会重复计算归一化后对应于相同多重集结构的配置。 这是自然处理的，因为两个转变代表不相交的结构起源：一个保留奇数残基结构，另一个保留纯偶数缩放结构。 
5. 我们迭代这个过程直到 n，在线性时间内产生所有 dp 值。 

### 为什么它有效

 如果所有元素都是偶数，则可以通过重复删除最低的 1 或分解出全局 2 来唯一地约简每个 2 的幂的多重集和 X。 这定义了从 X 到 0 的唯一分解路径。递归完全颠倒了这两个归约步骤。 由于这些归约是不相交的和确定性的，因此每个有效的多重集都只计数一次，并且每个构造的状态都对应于一个有效的多重集。 “坏分区”条件在两次归约下都被保留，因此 dp 仅累积有效配置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n = int(input())
    dp = [0] * (n + 1)
    dp[0] = 1

    for x in range(1, n + 1):
        # add a '1' element
        dp[x] = (dp[x] + dp[x - 1]) % MOD

        # scale-up structure (binary carry compression)
        if x % 2 == 0:
            dp[x] = (dp[x] + dp[x // 2]) % MOD

    print(*dp[1:])

if __name__ == "__main__":
    solve()
```第一个转换对应于将单位元素插入到大小为 x − 1 的任何有效配置中。第二个转换对应于采用 x/2 的配置并将所有元素加倍，这保留了 2 的幂结构，同时将所有贡献移高一位。 

递归是线性的，因为每个状态仅取决于最多两个先前状态，因此最多 n 的完整数组是在单遍中计算的。 

## 工作示例

 考虑较小的 X 值以了解配置如何构建。 

对于 X = 1，仅`{1}`存在，所以 dp[1] = 1。 

对于 X = 2，我们有`{2}`和`{1,1}`，给出 dp[2] = 2。 

我们追踪 DP 转换。 

| X | dp[x-1] 贡献 | dp[x//2] 贡献 | dp[x] | dp[x] |
 | ---| ---| ---| ---|
 | 1 | dp[0] = 1 | - | 1 |
 | 2 | dp[1] = 1 | dp[1] = 1 | 2 |
 | 3 | dp[2] = 2 | - | 2 |
 | 4 | dp[3] = 2 | dp[2] = 2 | 4 |

 这符合这样的直觉：偶数值继承了缩放带来的额外结构自由度，而奇数值仅通过添加单位元素来扩展。 

该迹线显示了 X 如何系统地获得额外的配置，反映了当所有元素都可以统一移位时可用的额外进位结构。

import sys, io

MOD = 998244353

def solve():
    n = int(sys.stdin.readline())
    dp = [0] * (n + 1)
    dp[0] = 1
    for x in range(1, n + 1):
        dp[x] = dp[x - 1]
        if x % 2 == 0:
            dp[x] = (dp[x] + dp[x // 2]) % MOD
        else:
            dp[x] %= MOD
    print(*dp[1:])

def run(inp: str) -> str:
    old_stdin = sys.stdin
    sys.stdin = io.StringIO(inp)
    old_stdout = sys.stdout
    sys.stdout = io.StringIO()
    solve()
    out = sys.stdout.getvalue().strip()
    sys.stdin = old_stdin
    sys.stdout = old_stdout
    return out

# minimal cases
assert run("1") == "1"
assert run("2") == "1 2"

# small correctness sanity
assert run("3")  # should run without error

# boundary
assert run("10")

 ## 边缘情况

 对于 X = 1，DP 仅从 dp[0] 开始，因此唯一的构造是添加单个 1 元素多重集。 该算法设置 dp[1] = dp[0]，根据需要给出 1。 

即使是 X，例如 2、4 或 8，两种跃迁都有贡献。 对于 X = 4，dp[4] 从“add 1”结构接收 dp[3]，从缩放结构接收 dp[2]，反映了二进制表示的扩展和压缩。 该组合确保对所有有效多重集进行计数而不会重复，因为每个构造路径都是由最后一个结构操作是加 1 还是全局加倍步骤唯一确定的。 

对于奇数 X，例如 3，仅适用“加 1”转换，因为无法通过将整数加倍来获得 X。 该算法自然会排除无效的缩放贡献，因为整数除法截断可以防止错误的传播。
