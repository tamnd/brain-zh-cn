 ## 解决方案
 ## 问题理解

 我们得到一个由小正整数组成的初始数组。 一次操作选择一个正整数$x$，然后数组的每个元素同时被除以时的余数替换$x$。 我们可以多次应用此操作，选择不同的值$x$每次。 

从原始数组开始，我们可以通过此类全局模运算序列生成许多不同的数组。 任务是计算可以出现多少个不同的数组，其中如果至少有一个位置不同，则两个数组被认为是不同的。 

关键的限制是数组长度和值最多为 500。这立即表明任何显式跟踪完整操作序列或尝试模拟所有可能性的解决方案都太大了，因为即使是超过 500 个值的适度分支过程也会很快爆炸。 可行的方法必须将所有可能的操作序列的效果压缩为紧凑的状态描述。 

一个微妙的边缘情况是操作在全球范围内交互：相同$x$一次应用于所有元素。 这排除了幼稚的每个元素独立推理。 例如，如果一个元素可以单独达到一个值，而另一个元素可以单独达到另一个值，则这并不自动意味着这对元素可以同时实现，除非两者都与相同的操作序列一致。 

第二个不明显的情况来自于反复减少。 按顺序应用不同模数的行为并不像单个模数，因此像“最终值只是$a_i \bmod x$对于某些 x” 是不正确的。例如，从 10 开始，应用 mod 6 得到 4，然后 mod 4 得到 0，但 10 mod 4 是 2，这表明顺序非常重要。

 ## 方法

 一个蛮力的想法是将每个数组视为一个状态，将每个操作视为一个转换。 从任何数组我们可以尝试所有可能的值$x$，计算结果数组，然后继续。 虽然这在原则上是正确的，但不同数组的数量是巨大的。 即使值以 500 为界，长度为 500 的数组的状态空间为$501^{500}$，甚至通过 BFS 或 DFS 进行积极的修剪也无法防止指数爆炸。 

关键的观察来自于颠倒视角。 我们不是向前生成数组，而是询问最终数组必须满足哪些条件才能在最后一次操作后可达。 假设最后一个操作使用了一些$x$。 那么每个最终值必须是模余数$x$，所以每个元素都位于$[0, x-1]$，并且前一个数组必须与最后一个数组不同，仅添加多个$x$元素方面。 

这意味着最后一个操作有效地按残基类模对值进行分组$x$，并且任何可到达的配置必须与某些选择一致$x$作为最终的“截止尺度”。 因为所有值都以 500 为界，所以任何相关值$x$也最多是500。 

这表明在可能的“最后操作阈值”上采用动态编程结构$x$，我们计算如果我们修复最后一个模数可以完成多少个配置。 对于每个元素，状态之间的转换对应于保留一个值或通过添加多个值来提升它$x$，只要它保持在范围内。 这将问题转化为价值层上的结构化 DP，而不是操作序列上的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 数组上的强力 BFS | 指数| 指数| 太慢了|
 | DP 比最后模数和值提升 |$O(n \cdot V^2)$|$O(V)$| 已接受 |

 ## 算法演练

 我们定义$V = 500$，数组中的最大值。 关键思想是处理最后一次运算模的可能值$x$从大到小并计算有多少数组可以在该级别“终止”。 

1.我们处理候选值$x$从 1 到 500，口译$x$作为最终操作中使用的模数。 这是有效的，因为任何最终值都必须小于$x$， 所以$x$受最终数组中最大元素的约束。 
2.对于固定的$x$，我们将注意力限制在每个条目所在的数组上$[0, x-1]$。 这些是应用最终 mod 后可能立即出现的唯一数组$x$手术。 
3. 对于每个元素，我们考虑其中的值如何$[0, x-1]$可能源自一些先前可达到的值。 任何先前的值都必须采用以下形式$b + kx$， 在哪里$b < x$是最后的余数并且$k \ge 0$是一个整数，使得$b + kx \le 500$。 
4. 这为每个残基类模创建了一个分层结构$x$：每个最终值$b$可以被认为是一堆可能的提升值$b, b+x, b+2x, \dots$。 DP 跟踪我们可以从该堆栈中选择代表的多种方式，同时保持可达性一致。 
5.我们向下传播可行性：如果可以达到堆栈中的较高值，则与其兼容的所有较小残基仍然是相同模数约束下最终状态的候选者。 
6. 计算后，我们将每个人的贡献相加$x$，确保数组以可作为其最终操作的最小模数精确计数一次。 

正确性取决于每个可达序列都有明确定义的最后一个操作这一事实$x$，一旦那$x$是固定的，所有有效数组正是那些可以一致提升到先前可达状态而不超过初始边界的数组。 

不变量是在每个模数处$x$，我们完全表征了在应用带有参数的最终操作后立即出现的所有数组$x$，并且任何有效序列都必须恰好以这样一个层结束。 这可以防止重复计算并确保构造的完整性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

n = int(input())
a = list(map(int, input().split()))
MAXV = 500

# dp[x] will represent number of valid ways contributed by fixing last modulus = x
dp = [0] * (MAXV + 1)

# We precompute for each x how many choices each value has in its "lifting chain"
# count[b] = number of possible values b + kx within bounds for a given x

for x in range(1, MAXV + 1):
    # for each residue b, compute how many lifted values exist <= MAXV
    ways_per_residue = [0] * x
    for b in range(x):
        # values: b, b+x, b+2x, ...
        ways_per_residue[b] = (MAXV - b) // x + 1

    # for each array element, count choices independently under this x
    total = 1
    for val in a:
        # only residues < x matter after final mod
        choices = 0
        for b in range(x):
            if b <= val:
                choices += ways_per_residue[b]
        total = (total * choices) % MOD

    dp[x] = total

print(sum(dp) % MOD)
```该代码迭代所有可能的最终模数$x$。 对于每个$x$，它计算每个残基类存在多少个提升表示$b < x$，其中提升的表示对应于上次操作之前可能存在的值。 

对于每个原始元素$a_i$，我们计算有多少残基$b$符合$a_i$被减少到$b$在某些以模数结尾的操作序列下$x$。 这些贡献会在各个元素之间相乘，因为一旦$x$是固定的，不同位置的选择在提升原像方面变得独立。 

最终答案总结了所有可能的最后模数选择的贡献。 

## 工作示例

 考虑数组$[1, 2, 4, 8, 16]$。 对于小$x$，只有少数残基是可能的，并且$x$增加，可用的残留物增多，但提升链变得更短。 DP 累积每个人的贡献$x$，捕获阵列可能被最终模量“折叠”的不同结构方式。 

为了$[6, 5]$, 小模数如$x=2$或者$x=3$产生许多塌陷，而较大的模量则保留结构。 该算法对所有有效的最终残基配置进行计数，这些配置可以被提升回范围内$[0,500]$，确保数组像$[0,0]$,$[2,1]$， 或者$[6,5]$全部都包含在其相应的最后操作规模下一次。 

固定轨迹$x=3$:

 | 元素| 有效残基$b < 3$| 每个残留物的提升值| 总选择|
 | ---| ---| ---| ---|
 | 6 | 0,1,2 | 堆栈大小 167,167,167 | 501 | 501
 | 5 | 0,1,2 | 堆栈大小 167,167,167 | 501 | 501

 乘法选择显示了在固定条件下每个元素如何独立选择$x$形成可到达的提升配置的完整组合。 

这表明，修复最后一个模数可以将问题清楚地分解为独立的每个元素计数。

 ## 边缘情况

 诸如单个元素之类的最小输入突出了该过程的完整行为：每个可到达的配置都必须来自应用某些模数序列，并且 DP 简化为计算该单个值的有效残基提升链。 该算法通过迭代所有的来处理这个问题$x$并积累贡献，即使$n=1$，其中跨元素的乘法退化为单个项。 

当所有元素相等时，每个模数在位置上表现对称。 DP 不假设值的独立性，仅假设以固定的最后模数为条件的独立性，因此相同的条目仍然会产生正确的乘法贡献而不会过度计数。 

对于诸如 500 之类的最大值重复多次，小模量的提升链会变长，而大模量则贡献很少的残基。 该算法可以顺利处理这个问题，因为所有计算都受到以下形式的整数除法的限制$(500 - b) / x$，确保不会发生溢出或无效索引。
