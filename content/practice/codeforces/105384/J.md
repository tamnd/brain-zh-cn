 ## 解决方案
 ## 问题理解

 我们给出了长度的排列$n$。 杰西可以将位置分成两个非空组。 一组为黄色，另一组为蓝色。 之后，每个组按值独立排序，然后将排序后的值按索引升序写回到该组的原始位置。 

最终的数组完全由该划分决定：在每个颜色类中，最小的位置获取该类的最小值，第二小的位置获取第二小的值，依此类推。 

分数是索引的数量$i$这样在这个过程之后，位置的值$i$恰好变成$i$。 任务是选择最大化该分数的分区并输出实现该分数的任何有效构造。 

约束允许最多$10^5$测试用例和总数$n$最多$10^6$，因此每个测试用例的解必须是线性的。 任何涉及嵌套扫描、根据猜测重新计算排序结构或子集组合搜索的操作都会立即变得太慢。 

排序如何与位置相互作用会出现一个微妙的问题。 一种幼稚的方法可能假设我们可以独立优化每个位置或每个值，但该操作通过每个颜色组内的排名来耦合索引和值。 另一个常见的错误是假设将值接近的元素放在一起就足够了，而实际上组内的索引顺序也很重要。 

小边缘情况清楚地显示了耦合。 为了$p = [2, 1]$，如果我们将两个位置涂成相同的颜色，则排序后会得到恒等排列，产生两个不动点。 然而，该问题禁止将所有内容都放在一种颜色中，因此我们被迫将它们分开，分数为零。 这表明全局最优性在很大程度上取决于排列结构如何与“必须分成两组”约束相互作用。 

## 方法

 如果我们尝试暴力着色，我们会检查所有$2^n - 2$有效的分区。 对于每个分区，我们模拟每个组内的排序过程，重建数组，并对固定点进行计数。 即使是单一的模拟成本$O(n \log n)$，使得这在大规模上完全不可行。 总工作量将爆炸性超过$O(n 2^n)$。 

关键的结构简化来自于观察操作在组内执行的操作。 在任何组内，最终的分配都是排序索引和排序值之间的完美匹配。 一个位置$i$仅当其在指数中的排名等于值的排名时才正确$i$同一组中的值之间。 这强烈表明全球互动是由群体结构而不是局部选择来调节的。 

关键的观察是将排列视为一组不相交的循环。 在一个周期内，指数和值紧密耦合。 如果循环的所有元素都放置在同一组中，则在对该组进行排序后，循环会在这些位置上折叠成完全排序的块，从而将循环中的每个元素变成固定点。 如果一个循环被分成两个组，那么结构对齐就会被破坏，并且循环不能再保证贡献固定点。 

这将问题简化为一个非常简单的组合决策：除非两种颜色都必须非空的要求强制，否则不应分割循环。 如果至少有两个循环，我们可以将整个循环分别分配给黄色和蓝色，保持每个循环完整并获得完全排序的全局结果。 如果只有一个循环，我们就被迫将其拆分，这会破坏所有潜在的不动点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力着色 |$O(n \cdot 2^n)$|$O(n)$| 太慢了|
 | 基于周期的分配 |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 ### 基于循环的构建

 1. 使用索引上的标准访问将排列分解为不相交的循环。 每个索引都属于一个循环，因为排列定义了一个函数图。 
2. 计算存在多少个周期。 这个数字决定了我们是否可以避免分裂任何循环，同时仍然满足使用两种颜色的要求。 
3. 如果至少有两个周期，则选择一个周期为黄色，并选择至少一个不同的周期为蓝色。 所有剩余的周期都可以任意分配，同时确保两组都保持非空。 这保证了颜色之间的循环不会被破坏。 
4. 如果恰好有一个循环，我们就被迫将其拆分。 一种有效的构造是将一个位置设为黄色，其余位置设为蓝色。 
5. 输出黄色集合及其大小。 

步骤3中的关键点是保持循环完整可以保持每组内排序后索引顺序和值顺序之间的内部一致性。 步骤 4 是不可避免的，因为问题要求两组都非空。 

### 为什么它有效

 在单个组内，排序会创建从第 k 个最小索引到第 k 个最小值的单调映射。 当保持完整时，置换循环可以完美地对齐这两个顺序，因为该循环已经代表了索引值对应的封闭结构。 一旦循环被跨组分割，这种对齐就会被打破，因为排名是在每个组中单独计算的，并且循环的相对结构不再保留。 因此，循环充当原子单元，用于保留对固定点的贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        p = [0] + list(map(int, input().split()))

        vis = [False] * (n + 1)
        cycles = []
        for i in range(1, n + 1):
            if not vis[i]:
                cur = []
                v = i
                while not vis[v]:
                    vis[v] = True
                    cur.append(v)
                    v = p[v]
                cycles.append(cur)

        k = len(cycles)
        yellow = []

        if k == 1:
            cyc = cycles[0]
            yellow.append(cyc[0])
        else:
            yellow.extend(cycles[0])
            # ensure blue is non-empty by not taking all cycles
            # so we leave at least one cycle for blue
            # if only 2 cycles, cycles[1] is blue implicitly

        print(n)
        print(len(yellow))
        print(*yellow)

if __name__ == "__main__":
    solve()
```该实现首先使用标准访问行走提取排列中的循环。 对每个未访问的索引进行排列，直到返回到已访问的节点，形成一个循环。 

循环分解后，决策仅取决于循环数。 当只有一个循环时，我们只选择一个元素作为黄色，以满足具有两种颜色的要求。 当存在多个循环时，我们将一个完整的循环视为黄色，并保留至少一个完整的循环为蓝色，以确保没有循环被分裂。 

输出打印黄色集的大小和索引本身。 

一个微妙的实现细节是我们不需要显式构造蓝色集。 该问题只需要黄色子集； 蓝色隐含着其他一切。 

## 工作示例

 ### 示例 1：排列$[2, 1]$| 步骤| 状态|
 | ---| ---|
 | 循环分解|$(1 \leftrightarrow 2)$|
 | 循环次数| 1 |
 | 黄色选择 |$\{1\}$|
 | 蓝色暗示|$\{2\}$|
 | 最终结果| 无不动点|

 单个循环会强制分裂，因此在一种颜色中不存在完整的循环结构。 排序后，每一组只有一个元素，因此没有任何元素可以对齐到固定位置。 

### 示例 2：排列$[2, 1, 4, 3]$| 步骤| 状态|
 | ---| ---|
 | 循环分解|$(1 \leftrightarrow 2), (3 \leftrightarrow 4)$|
 | 循环次数| 2 |
 | 黄色选择 |$\{1,2\}$|
 | 蓝色暗示|$\{3,4\}$|
 | 最终结果| 所有位置固定|

 每个循环在组内都保持完整，因此对每个组进行排序会将两个循环转换为其索引上的恒等映射。 

这证实了多个循环允许完全恢复已排序的排列。

 ## 边缘情况

 当排列是单个大循环时，算法被迫准确地选择一个元素作为黄色。 运行循环分解会产生一个包含所有索引的循环。 该构造选择单个索引，例如$1$，并将其指定为黄色，其余为蓝色。 由于两个组都包含至少一个元素，因此该构造是有效的。 排序后，两个组都是单例，因此没有位置可以成为固定点。 

当存在多个循环时，即使一个循环比其他循环大得多，将一个循环的所有元素视为黄色并且至少将整个循环视为蓝色可以保持整个循环的完整性。 每个循环都独立地在其索引上分类成完美的恒等映射，因此每个位置在重建后都变得正确。
