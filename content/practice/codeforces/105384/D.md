 ## 解决方案
 ## 问题理解

 我们得到一排代表货架的位置。 每个位置要么是空的，要么被一本书占据。 目标是让每个位置都“干净”，但有一个限制：包含书籍的位置不能直接清洁。 处理一本书的唯一方法是将其移动到相邻的空位置，每次移动都会消耗一个单位的能量。 

当所有的移动完成后，每个位置都必须被清理干净，这隐含着所有的书籍必须被重新安置，使得每个原来占用的位置都变成空的，可以自由地清理。 

关键的交互是本地的：书籍只能通过相邻的空槽移动，而空槽本身可以自由清理。 成本完全取决于书籍通过该结构的次数。 

所有测试用例的约束都很大，总长度高达 2×10^5。 这排除了任何逐步重复移动单本书的模拟，因为在最坏的情况下，即一本书穿过长长的空白空间，这会降级为二次行为。 

当书籍聚集或孤立时，就会出现微妙的边缘情况。 

例如，考虑如下配置`1010`。 人们可能认为独立移动每本书是最佳选择，但交互很重要，因为空槽是共享资源。 

另一个棘手的情况是`111000`。 一种幼稚的方法可能会假设书籍只是以与距离成正比的最小成本“落入”空白空间，但如果没有全局策略，人们可能会过多计算或错过空位置的重用。 

主要的挑战是认识到只有书籍和空位之间的相对顺序才重要，而不是它们孤立的绝对位置。 

## 方法

 暴力解释从字面上模拟了该过程。 我们反复挑选一本仍然存在的书，并尽可能将其移动到附近的空位置，从而减少每一步的成本。 这可以通过扫描阵列、找到可移动的书籍并执行移动直到没有书籍留在无效位置来实现。 

这是正确的，因为它完全遵守移动规则，但当一本书必须穿过交替状态的长走廊时，它的低效率就变得明显。 在最坏的情况下，每次移动都会将一本书移动一个位置，并且每本书可能有 O(n) 个这样的移动，每个测试用例的总工作量为 O(n²)。 

关键的观察是我们永远不需要明确地模拟运动。 每本书最终都会占据一些空位，重要的是有多少空位被“消耗”为相对于书籍排序方式的目的地。 如果我们从左到右查看书架，每次遇到一本书，我们都可以将其视为与其左侧或右侧的下一个可用空位置以最佳匹配的方式进行配对。 该过程简化为跟踪有多少空位可用于“吸收”书籍，并在书籍超出本地可用量时积累必要的班次。 

这将问题转化为单次线性扫描的贪婪记账过程。

 | 暴力模拟| O(n²) | O(n) | 太慢了|

 ## 算法演练

 我们从左到右处理书架，维护一个计数器，表示左侧当前有多少可用的空槽，可以吸收我们以后遇到的书籍。 

1. 初始化变量`empty = 0`和`cost = 0`。 这`empty`计数器跟踪可用的空容量，同时`cost`积累所需的移动力量。 
2. 从左到右遍历字符串，检查每个位置。 
3. 如果当前位置为空，则递增`empty`by 1. 这代表一个新可用的位置，可以作为之前或之后看到的某些书籍的目的地。 
4. 如果当前位置包含一本书并且`empty > 0`，我们将该书分配给可用的空槽之一。 这不会增加成本，因为我们可以在概念上匹配它，而无需除簿记之外的额外移动，因此我们递减`empty`1.
 5. 如果当前位置包含一本书并且`empty == 0`，则这本书左侧没有可用的空槽位可配对。 它必须“推”过去未来的空白空间，所以我们增加`cost`为 1。这代表创建一个延迟的不匹配，将通过未来的空头寸来解决。 
6.继续直到字符串末尾并输出`cost`。 

第 5 步背后的直觉是，每当我们在出现足够的空位之前看到一本书，我们就被迫支付重新排列的费用。 每一个这样的赤字都对应着一个在全球范围内无法避免的移动单位。 

### 为什么它有效

 在扫描的任何前缀处，`empty`代表有多少个免费目的地可以以可逆的方式吸收书籍。 遇到一本书的时候`empty > 0`可以在本地进行匹配，而不会造成全局干扰。 什么时候`empty == 0`，这本书在可用的局部空间方面实际上是“无与伦比的”，迫使结构重新安排，这在任何最佳解决方案中都将花费一个单位。 

这个不变量确保每次我们增加`cost`，我们正在计算前缀结构中书籍数量和可用空位置之间的根本不平衡，并且如果不提前支付至少一个单位，则后续操作无法消除这种不平衡。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        s = input().strip()

        empty = 0
        cost = 0

        for ch in s:
            if ch == '0':
                empty += 1
            else:
                if empty > 0:
                    empty -= 1
                else:
                    cost += 1

        print(cost)

if __name__ == "__main__":
    solve()
```实现直接遵循基于扫描的推理。 这`empty`计数器的行为就像一个可用插槽池，每本书要么消耗一个插槽，要么在没有可用插槽的情况下触发成本增加。 关键的微妙之处在于我们从不尝试明确地移动任何东西； 整个结构被编码在不平衡跟踪中。

 ## 边缘情况

 全零的配置，例如`00000`永远不会触发成本，因为每个位置都已经是可清洁的，并且只是增加了可用容量。 该算法仅递增`empty`， 所以`cost`始终保持为零。 

包含所有这些的配置，例如`11111`产生的成本等于书籍数量减一，因为似乎没有空位可以抵消早期书籍。 扫描增量`cost`在每一个位置上，都符合这样一个事实：除了最后一本书之外，每一本书都必须强制进行结构重新安排。 

混合情况如`100001`展示了早期的空白空间如何能够吸收后来的书籍，但仅限于其容量。 第一个零增加`empty`，最后一本书消耗了它，使得成本完全由初始不平衡（如果在容量建立之前发生任何情况）驱动。
