 ## 解决方案
 ## 问题理解

 我们正在计算有多少对数组$a$和$b$, 两者的长度$n$，可以使用以下值形成$1$到$m$，使得每对位置之间都存在特定的对称条件。 

选择任意两个索引$i < j$。 你看看这两对“交叉”的$(a_i, b_j)$和$(a_j, b_i)$。 从每一对中，您取最小值。 要求是这两个最小值始终相等，无论哪个$i$和$j$你选择。 

因此数组的结构对于每个位置来说并不是独立的。 每个位置通过这种交叉最小值约束与其他位置交互，这强烈限制了值如何跨索引排列。 

输入大小达到$10^6$对于两者$n$和$m$，因此任何检查所有索引对或尝试单独推理每对位置的解决方案都是不可行的。 任何平方项$n$，甚至线性于$n$由于每个元素的繁重工作，已经处于边缘。 预期的解决方案必须将结构简化为仅依赖于$m$，每个值的处理基本上是常数或对数。 

当试图进行本地推理时，会出现一个微妙的陷阱。 例如，人们可能会尝试仅对相邻索引强制执行该条件，或者假设$a$或者$b$。 这失败了，因为条件涉及每一对$(i,j)$，而不仅仅是相邻位置。 

另一个常见的失败是将条件视为每对索引的独立约束。 为了$n=2, m=2$， 有$2^4=16$可能的数组对，但只有 10 个有效。 幼稚的每对约束方法往往会过度计算，因为它忽略了全局一致性效应。 

## 方法

 直接的暴力解决方案将生成所有$m^{2n}$数组对并检查每对索引的条件。 即使检查单个候选人也需要$O(n^2)$比较，因此总复杂度变为$O(m^{2n} \cdot n^2)$，即使对于微小的输入，这也是完全遥不可及的。 该结构显然隐藏着强大的全球排序原则。 

关键的观察结果是，约束仅取决于通过最小值比较不同指数的值。 两个值之间的最小值仅取决于两个值中较小的一个。 这表明重要的不是确切的值本身，而是它们相对于阈值的相对顺序。 

如果我们固定一个阈值$x$, 每个位置$i$可以根据是否$a_i \ge x$以及是否$b_i \ge x$。 这将每个阈值的每个位置减少到四种状态之一。 交叉最小值条件迫使这些状态在所有指数中保持全局一致，从而消除了模式的任意混合。 

当这种一致性同时超过所有阈值时，该结构会崩溃为一个令人惊讶的简单特征：每个索引的行为就好像它具有单个有效标量值，并且整个对$(a_i, b_i)$可以通过变换空间中的单个数字进行编码。 这导致有效对的数量仅取决于对可能值水平的简单幂表达式求和。 

得到的闭合形式是：$$\sum_{x=1}^{m} (2x - 1)^n$$该表达式精确匹配小情况并捕获由约束引起的组合结构。 

我们现在比较方法。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 |$O(m^{2n} \cdot n^2)$|$O(n)$| 太慢了|
 | 封闭式总和 |$O(m \log n)$|$O(1)$| 已接受 |

 ## 算法演练

 目标是评估：$$\sum_{x=1}^{m} (2x - 1)^n \bmod 998244353$$1. 预先计算模数$MOD = 998244353$。 所有算术都是以该值为模进行的，因为结果增长得非常快。 
2. 对于每个整数$x$从$1$到$m$，计算基值$v = 2x - 1$。 这代表了级别转换后的贡献$x$在最终的组合结构中。 
3. 计算$v^n \bmod MOD$使用快速求幂。 这一步至关重要，因为$n$可以大到$10^6$，所以简单的乘法会太慢。 
4. 将结果累加为运行总和。 
5. 输出最终的和模数$MOD$。 

关键的计算负担是求幂。 每一项都是独立的，所以循环$m$是不可避免的，但每次求幂都在$O(\log n)$，使整个解决方案易于管理。 

### 为什么它有效

 交叉最小值约束强制所有索引对在阈值比较下表现一致。 这种一致性将问题分解为计算由值之间的有效“中位水平”索引的贡献$a$和$b$。 每个级别$x$独立贡献，以及配置在级别贡献的方式数量$x$正是$(2x-1)^n$。 对所有级别求和可重建总计数，而不会重复计数，因为每个有效配置都是由其所属的阈值间隔唯一确定的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modpow(a, e):
    res = 1
    while e > 0:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

n, m = map(int, input().split())

ans = 0
for x in range(1, m + 1):
    base = 2 * x - 1
    ans += modpow(base, n)
    ans %= MOD

print(ans)
```直接按照推导公式实现。 唯一重要的组成部分是二进制求幂函数，它避免了重复重新计算幂。 每次迭代都会计算一个新的基础$2x-1$，确保循环步骤之间不存在依赖关系。 

一个常见的错误是试图在不同的领域重复使用中间权力$x$，但基础差异很大，以至于在没有额外多项式机制的情况下，这种重用不会产生干净的递归。

import sys, io

MOD = 998244353

def modpow(a, e):
    res = 1
    while e > 0:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    n, m = map(int, input().split())
    ans = 0
    for x in range(1, m + 1):
        ans += modpow(2 * x - 1, n)
        ans %= MOD
    return str(ans)

 ## 边缘情况

 一种边缘情况是当$m = 1$。 该公式可分解为一项$(1)^n$，所以不管怎样答案总是 1$n$。 该算法自然地处理这个问题，因为循环运行一次并且求幂返回 1。 

另一个边缘情况是当$n = 1$。 表达式变成奇数之和$1 + 3 + \dots + (2m-1)$，等于$m^2$。 该实现正确地计算了这个，因为每个幂运算都是微不足道的，并且循环累加算术级数。 

第三个边缘情况很大$n$。 直接乘法会溢出或超时，但二进制求幂将每一项减少到对数时间，即使在以下情况下也能保持性能稳定：$n = 10^6$。
