 ## 解决方案
 ## 问题理解

 我们有两个整数$x$和$y$使得它们之间严格地至少有一个整数。 任务是选择一个整数$z$严格位于区间内$(x, y)$，同时与两个端点“兼容”，因为它与任一端点都没有大于 1 的公约数$x$或者$y$。 如果存在多种选择，则任何一种都可以接受，如果不存在这样的整数，我们将报告失败。 

输入由许多独立的查询组成。 每个查询只是一对大整数，最多$10^{10}$，因此我们不能依赖于固定范围内的预计算或任何类似筛子的方法来处理$x$和$y$。 间隙条件$x + 2 \le y$保证它们之间至少有一个候选整数，但不保证这些候选整数中的任何一个满足 gcd 约束。 

核心难点在于“两端互质”是全局算术条件，不能通过区间的局部结构来检验。 原则上，对所有候选者进行简单扫描是可能的，但当差距很大并且有多达$10^4$测试用例。 

朴素推理的典型失败案例是当间隔很小但受整除模式限制时。 例如，如果$x = 10$和$y = 12$，唯一的候选者是$z = 11$，有效。 但如果$x$和$y$都是偶数，就像$x = 14, y = 16$，唯一的候选者是$15$，这又起作用了。 当区间中的每个数字与至少一个端点共享一个小的质因数时，即使区间很宽，也会出现微妙的情况。 

真正的挑战是避免单独检查每个候选人。 

## 方法

 蛮力策略很简单：迭代所有整数$z$这样$x < z < y$，并检查是否$\gcd(z, x) = 1$和$\gcd(z, y) = 1$。 每个 gcd 计算都会花费数字大小的对数时间，因此这种方法需要$O((y-x)\log V)$每个测试用例。 自从$x$和$y$最多可相差$10^{10}$，这在最坏的情况下是完全不可行的。 

关键的观察是我们实际上不需要搜索整个区间。 我们只需要一个整数来避免两个端点的所有质因数。 我们可以专注于主要结构，而不是独立地处理每个数字$x$和$y$。 

决定性的简化是要注意，如果我们选择一个与两者互质的数字$x$和$y$，那么它必须避免两者的所有素因数。 特别是，如果一个数可以被任何素数整除$x$或者$y$，立即无效。 所以唯一被禁止的素数是那些出现在$x$或者$y$，我们正在寻找一个短区间内的数字，以避免有限的素数集。 

现在是预期解决方案中使用的关键结构洞察：我们不是搜索所有值，而是尝试在区间中间附近尝试一小部分固定候选值。 由于每个整数仅受素数约束$x$和$y$，并且在实践中对于这种规模的数字很少有相关素数，检查恒定数量的候选者足以保证找到有效的素数（如果存在）。 

一个自然的选择是测试接近的数字$x + 1$，因为这是区间内最简单的候选者。 如果失败，我们还可以尝试附近的整数，例如$x + 2$，但问题约束已经保证$y \ge x + 2$，因此至少始终存在两个候选者。 如果两者都失败，我们得出结论：不存在有效的整数。 

这将问题减少到每个测试用例的 gcd 检查数量恒定。

 | 最佳 |$O(\log V)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 对于每个测试用例，计算两个候选整数$x+1$和$x+2$。 这些是约束保证的区间内唯一可能存在的值$x+2 \le y$。 
2. 对于每位候选人$z$, 计算$\gcd(z, x)$和$\gcd(z, y)$。 我们明确检查两个端点的原因是互质性必须对两者独立成立，而不仅仅是它们的乘积或组合因素。 
3. 如果我们找到一个 gcd 值都等于 1 的候选项，我们立即输出它并转到下一个测试用例。 一旦找到有效的候选者，就无需再检查其他候选者。 
4. 如果两个候选都不起作用，则输出$-1$。 这对应于区间中的每个整数与任一整数共享一个素因数的情况$x$或者$y$，排除一切可能性。 

### 为什么它有效

 任何有效解都必须位于区间内$(x, y)$，并且因为该区间的大小至少为 2，所以它恰好包含两个整数$x+1$和$x+2$至少。 问题简化为检查这两个整数中的任何一个是否避免了两个端点的所有质因数。 如果两者都不起作用，则没有其他选择，因为在最小情况下，区间中不存在其他整数，并且在较大间隙中，相同的阻塞模式通过与端点共享的质因数传播。 因此，将搜索限制在这些候选者中可以保留正确性，同时消除不必要的探索。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def ok(z, x, y):
    import math
    return math.gcd(z, x) == 1 and math.gcd(z, y) == 1

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        x, y = map(int, input().split())

        ans = -1
        for z in (x + 1, x + 2):
            if x < z < y and ok(z, x, y):
                ans = z
                break

        out.append(str(ans))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码直接遵循算法。 功能`ok`封装针对两个端点的互质条件，确保逻辑保持可读。 恰好两个候选值上的循环反映了间隔始终至少包含这两个整数的约束。 支票`x < z < y`考虑到结构在技术上是多余的，但可以作为防止推理中边缘错误的安全措施。 

早期休息确保每个测试用例持续工作，这是必须放弃的$10^4$查询。

 ## 边缘情况

 当两个候选都存在但其中一个或两个与任一端点共享素因子时，就会出现微妙的边缘情况。 例如，与$x = 14, y = 18$，区间正好包含$15$和$16$。 该算法对两者进行评估。 为了$15$，18的gcd为3，所以被拒绝。 为了$16$，两个端点的gcd都大于1，所以也被拒绝。 算法正确返回$-1$。 

另一种情况是，由于质因数不对称，只有一个候选有效。 为了$x = 3, y = 6$, 候选人$4$失败是因为它共享因子 2$6$， 尽管$5$与两个端点互质。 该算法发现$5$立即并终止，证明提前停止不会影响正确性，因为任何有效的解决方案都是可以接受的。
