 ## 解决方案
 ## 问题理解

 我们得到一串小写字母。 任务是查看所有有序的位置对，其中第一个位置包含字符`e`，第二个位置包含字符`f`，以及`e`出现在字符串中的时间早于`f`。 对于每个这样的对，我们测量它们位置之间的距离，定义为索引的差异，并将所有有效对的距离相加。 

换句话说，每当我们看到一个`e`在位置`i`，我们看每一个`f`在位置`j > i`并做出贡献`j - i`到答案。 目标是有效地计算这个总数。 

输入大小约束为`n ≤ 5000`。 这个值足够小，二次解是可以接受的，但又足够大，三次解或任何具有不必要开销的东西在实践中都会太慢。 明确检查所有对的解决方案是边界性的，但在 O(n²) 中仍然可行，而任何重复重新计算内部总和的解决方案都会有超时的风险。 

当字符串不包含任何内容时，就会出现微妙的边缘情况`e`或没有`f`。 在这种情况下，没有有效的对，答案为零。 另一个边缘情况是当所有`e`人物终究出现了`f`字符，它也会产生零，因为没有有效的排序`i < j`存在。 例如，对于`feef`, 没有有效的`(e, f)`自从每`f`在任何之前`e`。 

第三个微妙之处是，贡献取决于位置差异，而不仅仅是计数。 一个简单的计数解决方案，将数量相乘`e`和`f`是不正确的，因为它忽略了距离。 

## 方法

 直接的方法是迭代每个位置`i`含有`e`，并且对于每个这样的位置，扫描所有位置`j > i`并积累`(j - i)`每当`s[j] = 'f'`。 这是简单且正确的，因为它完全遵循定义。 然而，在最坏的情况下，字符串都是`e`紧随其后的是所有`f`，或交替字符，这导致大约`n² / 2`字符检查，这仍然可以接受`n = 5000`，但几乎没有留下低效率的空间。 

关键的改进来自于分离距离表达式`j - i`分成两个独立的部分：`j`和`i`。 如果我们固定一个位置`i`和`s[i] = 'e'`，后来所有人的贡献`f`职位是`sum(j) - count(f) * i`，其中总和和计数涵盖所有`f`右侧的位置`i`。 这建议采用从右到左的预处理策略，维护多少个`f`保留字符及其索引之和。 

一旦我们从右到左处理字符串，我们就可以维护两个运行值：`f`到目前为止看到的字符及其索引的总和。 当我们遇到一个`e`在位置`i`，我们立即知道它对所有未来的总贡献`f`使用存储的聚合值的位置。 这将每个位置的处理量减少到 O(1)。 

暴力破解之所以有效，是因为它显式枚举了所有有效对，但它无法扩展，因为它重复地重新计算相同的后缀信息。 观察一切未来`f`贡献可以通过两个聚合来概括，将嵌套结构转变为线性扫描。

 ## 算法演练

 我们从右到左处理字符串，以便在任何时刻我们都知道有关的一切`f`位于当前位置右侧的字符。 

1.初始化两个变量，`cnt_f = 0`和`sum_f = 0`， 在哪里`cnt_f`商店有多少家`f`到目前为止我们从右边看到的字符，以及`sum_f`存储它们的索引之和。 这些代表了所有有效的压缩摘要`f`未来的职位`e`贡献。 
2.初始化最终答案`ans = 0`。 这将累积总距离总和。 
3. 迭代索引`i`从`n - 1`下降到`0`。 在每个位置，我们根据角色决定如何更新或贡献。 
4. 如果`s[i] == 'f'`，通过递增更新后缀状态`cnt_f`并添加`i`到`sum_f`。 这确保了未来`e`职位可以解释这一点`f`。 
5. 如果`s[i] == 'e'`，使用所有当前已知的计算其贡献`f`职位。 对于每一个这样的`f`在位置`j`，我们要添加`(j - i)`。 扩展这给出`sum_f - cnt_f * i`，所以我们直接将该值添加到`ans`。 
6. 继续直到字符串的开头。 决赛`ans`包含所有有效的总贡献`(e, f)`对。 

### 为什么它有效

 在每个索引处`i`，变量`cnt_f`和`sum_f`准确地代表所有`f`位置严格位于`i`。 这个不变量得以维持，因为我们仅在向左移动经过每个位置时更新这些变量。 当处理一个`e`在位置`i`, 每个有效对`(i, j)`和`j > i`通过表达式仅计算一次`sum_f - cnt_f * i`。 没有一对被错过，因为所有的未来`f`位置包含在后缀状态中，并且不会重复计算任何对，因为每个`e`使用后缀信息的固定快照独立贡献。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    n = len(s)

    cnt_f = 0
    sum_f = 0
    ans = 0

    for i in range(n - 1, -1, -1):
        if s[i] == 'f':
            cnt_f += 1
            sum_f += i
        elif s[i] == 'e':
            ans += sum_f - cnt_f * i

    print(ans)

if __name__ == "__main__":
    solve()
```代码维护了所有的后缀摘要`f`职位。 每次`e`遇到时，它计算其对所有未来的总贡献`f`使用算法部分中导出的恒等式来处理恒定时间的字符。 关键的实现细节是从右到左迭代，以便后缀始终对应于有效的`j > i`职位。 

算术表达式`sum_f - cnt_f * i`直接展开总和`(j - i)`超过所有存储的`j`。 这完全避免了内部循环。

 ## 边缘情况

 当字符串只包含`f`字符后仅跟随`e`字符，该算法正确地产生零，因为没有`e`见过后缀`f`在从右到左扫描期间。 用于输入`ffffeeee`，扫描构建`cnt_f`首先，但每一个`e`毕竟来了`f`已经被消耗了，所以没有添加贡献。 

当字符串是`eeeeffff`， 每一个`e`看到完整的后缀`f`人物。 在加工过程中，`cnt_f`和`sum_f`在遇到任何问题之前就已完全构建`e`，所以每个`e`使用相同的后缀状态独立贡献。 这可以正确计算所有对，而不会重复。 

当字符串是单个字符时，例如`e`或者`f`，该算法恰好执行一个扫描步骤并且从不触发贡献，根据需要产生零。
