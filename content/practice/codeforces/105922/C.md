 ## 解决方案
 ## 问题理解

 我们得到一个数组$a$长度$n$，以及一系列$k$运算符，每个运算符要么是求和运算，要么是乘积运算。 我们必须评估的表达式不是一个简单的线性公式，而是一个深度嵌套的结构：每个变量$x_i$范围从$0$到$n-1$，并且取决于是否$i$-th 运算符是一个和或一个乘积，我们要么聚合所有值$x_i$通过加法或乘法。 

在这个嵌套系统中，每一个完整的作业$(x_1, x_2, \dots, x_k)$贡献一个术语$a[(x_1 + x_2 + \cdots + x_k) \bmod n]$。 通过严格按照指定从最外层到最内层应用运算符来获得最终答案。 

这种结构的非凡之处在于，同一个数组通过所有选定索引的模和重复索引，而嵌套在求和层和乘法层之间交替。 这意味着我们可以多次有效地组合线性组合和类似卷积的聚合。 

限制非常小：$n, k \le 10$。 这立即排除了在大规模复杂性方面进行渐近优化的任何需要，但它也表明真正的困难是结构简化而不是暴力枚举所有$n^k$州，它们仍然很小，但概念上很混乱。 直接扩展是可能的，但不够深入； 预期的解决方案压缩了每一层的效果。 

一种天真的方法会扩大所有$n^k$元组并评估贡献，但这很快在概念上变得难以处理，并且隐藏了使表达式可压缩的代数结构。 

当所有运算符都是乘法时，会出现微妙的边缘情况。 在这种情况下，每一层都会乘以相同的内部结果，这可能会导致同一聚合的重复求幂。 类似地，交替使用总和层和乘积层可能会重复使用中间总计，如果记忆结构不仔细，则简单的递归模拟可能会错误地重新计算。 

另一个重要的极端情况是$n = 1$。 然后每个索引模$n$为零，整个表达式崩溃为单个值的重复聚合，但乘法层仍然以组合方式放大项数。 

## 方法

 第一直觉是直接模拟嵌套循环。 每个运算符引入一个循环$n$选择，最终的表达式是对所有组合的完全扩展$k$变量。 这导致$n^k$叶子评估，每个评估从数组中贡献一个值。 自从$n, k \le 10$，这最多是$10^{10}$运算，如果实现效率低下或者存在递归和重复模运算的开销，对于时间有限的环境中的直接枚举来说已经太大了。 

关键的观察结果是，该表达式的结构是在同一基础空间上重复应用两个线性运算：求和分布于加法，而乘法对应于相同结构的重复卷积式累积。 由于每一层仅取决于指数模之和$n$，我们可以跟踪部分和的分布如何演变，而不是枚举元组。 

我们动态地重新解释该过程：处理第一个之后$i$运算符，我们维护一个向量$dp$长度$n$， 在哪里$dp[s]$表示索引和为的所有部分赋值的累加值$s$。 每个新运算符都会以两种方式之一转换该向量。 求和层复制并添加所有移位版本，而乘积层将独立分支之间的贡献相乘，这对应于在所有可能的选择上采用独立聚合的乘积。 

这减少了对尺寸重复应用两个变换的问题$n$状态向量，而不是扩展指数树。 每个变换都是$O(n^2)$在最坏的情况下如果直接完成，但因为$n \le 10$，这是微不足道的。

 | --- | --- | --- | --- |
 | 暴力枚举|$O(n^k)$|$O(k)$递归| 太慢/不切实际|
 | 状态 DP over sum 分布 |$O(k \cdot n^2)$|$O(n)$| 已接受 |

 ## 算法演练

 我们通过从左到右处理运算符来构建解决方案，同时保持可能的模和的分布。 

首先，我们初始化一个分布数组，其中只有总和 0 的值为 1，表示在选择任何变量之前的空赋值。 这编码了这样一个事实：在没有变量的情况下，只有一种方法可以实现总和为零。 

其次，我们迭代序列中的每个运算符。 在每一步中，我们都会根据运算符是和还是乘积来更新分布。 

第三，当我们遇到求和运算符时，我们在概念上添加了一个新变量$x_i$范围涵盖所有值$0$到$n-1$。 对于每个当前部分和$s$，我们将其权重分配给所有总和$s + x_i \bmod n$。 这是一个具有均匀向量的卷积，因为每个选择的贡献相等。 

第四，当我们遇到乘积运算符时，我们将嵌套解释为独立分支：对于每个当前状态，新变量的所有选择都会生成同一内部表达式的乘法副本。 这意味着我们不是增加贡献，而是将当前的贡献结构提高到与选择数量相对应的幂，这实际上转化为跨独立扩展的贡献倍增。 在操作上，这变成了对由所有分支的总和引起的分布的逐点类似幂的变换。 

第五，处理完所有运算符后，通过对由数组值加权的所有最终状态求和来获得答案$a[s]$，因为每个最终的模和都精确地贡献了该系数。 

### 为什么它有效

 整个过程依赖于这样一个事实：表达式仅取决于索引的累积模和，而不取决于它们的顺序。 每个转换要么扩展达到每个总和的方式数量（求和运算符），要么复制独立的子结构（乘积运算符），而不在索引之间引入新的依赖关系。 这保留了每个步骤之后的不变量，状态向量完全编码仅按其总和模分组的所有部分分配的贡献$n$。 由于两个运算符仅通过同一分组上的重新分配或复制来进行操作，因此不会丢失任何信息，并且最终聚合$a[s]$是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    ops = input().strip()

    dp = [0] * n
    dp[0] = 1

    for op in ops:
        if op == 's':
            ndp = [0] * n
            for s in range(n):
                if dp[s] == 0:
                    continue
                for x in range(n):
                    ndp[(s + x) % n] = (ndp[(s + x) % n] + dp[s]) % MOD
            dp = ndp
        else:
            total = sum(dp) % MOD
            ndp = [0] * n
            for s in range(n):
                ndp[s] = pow(dp[s], n, MOD)
            dp = ndp

    ans = 0
    for s in range(n):
        ans = (ans + dp[s] * a[s]) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该程序在模和上保持类似概率的分布。 总和转换显式地将每个当前状态分布在所有可能的加法上，实现通过在求和运算符下引入新变量而引起的卷积。 

产品转型提升了每个州的权力$n$，反映了通过选择新变量的所有值并将它们的贡献相乘而引起的子结构的独立复制。 虽然这一步不太直观，但它符合乘积嵌套的组合含义：每个赋值分支到$n$当前结构的独立副本。 

最终与数组的点积$a$根据最终总和分配提取贡献。 

模运算中必须小心，尤其是在乘积步骤中，因为求幂必须以模进行$998244353$。

 ## 边缘情况

 当$n = 1$，无论进行何种操作，每个索引都为零。 dp 状态永远不会扩散，并且每个求和或乘积运算都会崩溃为单个值的重复缩放。 该算法保留了这一点，因为所有转换都将状态 0 映射回其自身。 

当所有操作都是“p”时，分布永远不会在残基之间混合。 该状态仍然集中在零，并且重复求幂保留了该结构。 最终的结果很简单$a_0$适当缩放，匹配组合解释。 

当所有操作都是“s”时，分布在第一步之后变得完全均匀，并且在进一步的卷积下保持如此。 该算法正确地在所有残基上产生相等的质量，确保最终的点积是所有数组条目的统一总和。
