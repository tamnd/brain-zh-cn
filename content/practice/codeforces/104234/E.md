 ## 解决方案
 ## 问题理解

 我们得到了从 L 到 R 的连续整数块。将这些数字同时视为“位置”和“项目”。 对于该段中的每个数字 i，我们必须为其分配来自同一段的不同数字 yᵢ，形成区间的排列。 

该约束对于每个位置都是局部的：当 i 与 yᵢ 配对时，它们的最大公约数必须为 1。换句话说，i 及其指定值不得共享任何质因数。 

输出是按从 L 到 R 的顺序编写的排列。如果不存在这样的分配，我们必须报告不可能性。 

测试用例中所有段的总大小最多为 10⁵，因此只要我们处理每个值一次，每个测试用例的 O(n) 方法就足够了。 在最坏的情况下，任何涉及每个值因式分解或针对许多候选者进行 gcd 检查的操作都会太慢。 

当段的大小为 1 时，会立即出现关键边缘情况。如果唯一的数字大于 1，则无法将其映射到其他任何数字，并且 gcd(i, i) = i > 1，因此会失败。 如果唯一的数字是 1，则它有效，因为 gcd(1,1)=1。 

另一种微妙的情况是当段长度为奇数但大于 1 时。“配对邻居”的天真尝试会使一个元素不配对，并且剩余位置将不可避免地映射到自身，除非它恰好为 1，否则这是无效的。 

## 方法

 蛮力想法会尝试贪婪地构建排列。 对于每个 i，我们扫描 [L, R] 中所有未使用的值 y，并选择 gcd(i, y)=1 的一个。 这在原则上是正确的，因为我们总是保持有效的部分匹配，但它太慢了。 n 个位置中的每个位置都可以扫描 O(n) 个候选者，从而导致每个测试用例执行 O(n²) 次操作，总共有 10⁵ 个元素。 

区间的结构表明了一种更简单的模式。 关键的观察是连续整数总是互质的。 对于任何整数 k，gcd(k, k+1) = 1 普遍成立。 这意味着相邻交换自动满足 gcd 条件，无需任何数论计算。 

因此，我们不是搜索兼容值，而是强制结构：我们将段划分为相邻对并交换每对。 这会产生一个排列，其中每个元素都映射到邻居，保证 gcd 约束。 

唯一的障碍是奇偶校验和 1 的存在。如果段长度是偶数，则配对工作正常。 如果是奇数，则只有一个元素保持未配对状态。 剩下的数只有在为 1 时才有效，并且我们选择将其保持固定，因为 1 是唯一与自身互质的数字。 

这导致了行为上的彻底分裂：要么我们将所有东西配对，要么当 1 存在于边界情况时我们将其隔离。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力匹配| O(n²) | O(n) | 太慢了|
 | 相邻配对结构| O(n) | O(1) 额外 | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例。 

1. 计算长度 n = R − L + 1。这确定是否可以配对。 
2. 如果 n = 1，我们只有一个数字。 只有当它为 1 时我们才能输出它，因为否则 gcd(L, L) ≠ 1。如果 L ≠ 1，我们返回 -1。 
3. 如果L = 1且n ≥ 2，我们特殊对待1。 我们输出 1 作为固定点，然后使用相邻交换对剩余的段 [2, R] 进行配对。 这是有效的，因为删除 1 会留下从 2 开始的连续区间，并且所有这些对仍然是互质的。 
4. 如果 L > 1 并且 n 为奇数，我们输出 -1。 没有可用的有效不动点，因为任何不动点 i 都需要 gcd(i, i)=1，这仅适用于 i=1。 
5. 否则，n 为偶数且 L > 1，或 L = 1，偶数余数如上处理。 我们通过以 2 步迭代并交换 (L, L+1)、(L+2, L+3) 等来构建排列。

构造的数组 y 直接按顺序填充，始终以相反的顺序分配两个连续的值。 

### 为什么它有效

 不变的是，每个已处理的大小为 2 的块都形成有效的相互分配，并且所有块都是不相交的。 由于每个块都是一对连续的整数，因此 gcd(i, i+1)=1 保证了本地的正确性。 因为我们对整个范围（可能除了单个 1）进行分区，所以每个索引都只接收一个值，并且每个值都只使用一次，因此结果是一个有效的排列。 

不需要考虑其他 gcd 情况，因为我们从不分配不相邻的值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    L, R = map(int, input().split())
    n = R - L + 1

    if n == 1:
        if L == 1:
            print(1)
        else:
            print(-1)
        return

    res = [0] * n

    if L == 1:
        if n == 2:
            print("1 2")
            return
        res[0] = 1
        # pair 2..R
        i = 2
        idx = 1
        while i <= R:
            res[idx] = i + 1
            res[idx + 1] = i
            idx += 2
            i += 2
        print(*res)
        return

    if n % 2 == 1:
        print(-1)
        return

    idx = 0
    i = L
    while i <= R:
        res[idx] = i + 1
        res[idx + 1] = i
        idx += 2
        i += 2

    print(*res)

def main():
    t = int(input())
    for _ in range(t):
        solve()

if __name__ == "__main__":
    main()
```实现直接遵循配对策略。 关键细节是我们从不显式计算 gcd，因为结构配对已经保证了这一点。 索引处理非常小心：当 L = 1 时，我们首先输出 1，然后使用结果数组中的移位索引从 2 开始配对。

 ## 边缘情况

 当线段恰好为[1, 1]时，算法直接返回1，因为它是唯一满足自身gcd条件的不动点。 

当段为 [L, R] 且 L > 1 且长度为 1 时，算法会正确返回 -1，因为自映射无效。

 当 L = 1 并且长度为奇数时，算法通过显式分隔 1 并将剩余的连续数字配对来避免留下非 1 固定点。 这可确保不会发生无效的自映射。 

当 L > 1 并且长度为奇数时，算法立即拒绝这种情况，因为不存在有效的固定点来吸收奇偶校验不匹配。
