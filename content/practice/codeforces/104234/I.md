 ## 解决方案
 ## 问题理解

 我们正在研究一种随机过程，它通过逐个处理顶点来构造有向无环图。 

任何时刻都有一组已插入的顶点和一组剩余的顶点。 我们重复地均匀地选择一个剩余的顶点，将其放置在构造顺序中的下一个，然后将其与来自已插入顶点的某个子集的有向边连接。 该子集也是在所有子集中均匀随机选择的。 

如果我们考虑最终的对象，每次运行都会在顶点 1 到 n 上生成一个带标签的 DAG，但实际结构取决于两个独立的随机选择：顶点出现的顺序，以及对于每个顶点，哪些较早的顶点指向它。 

我们独立地重复整个过程两次，并获得两个 DAG。 任务是计算这两个 DAG 在有向边方面不相同的概率，答案以分数模形式对 1e9 + 9 求模。 

最大 100000 的约束排除了对图、排列或子集的任何枚举。 任何解决方案都必须将随机性降低为封闭形式的组合表达式，该表达式可以在每个测试用例的线性时间内或更好的时间内进行评估。 

一个微妙的问题是，多个插入顺序可以生成相同的 DAG。 这意味着图的概率在所有 DAG 中并不统一，因此简单地计算“DAG 数量”是不可用的。 另一个陷阱是假设边独立于无序对，这是错误的，因为边仅在隐藏的随机排列中指向前方。 

边缘情况已经在小 n 处出现。 对于 n = 1，两种构造总是产生空图，因此不同的概率为 0。对于 n = 2，只有两个顶点，并且随机性减少到单个硬币翻转来确定第一个顶点是否指向第二个顶点，因此分布已经是不平凡的。 任何将所有 DAG 视为同等可能性的错误模型都会立即失败。 

## 方法

 暴力视图试图枚举每种可能的构造：顶点的所有排列以及每个顶点的所有子集选择。 对于固定排列，每个顶点选择先前插入的顶点的子集，因此可能性的数量是 2 的幂的乘积。 这使得结果总数巨大，大约为 n！ 乘以 2^(n(n−1)/2)，直接比较两个完整分布需要对所有 DAG 求和，即使对于 n = 20 也是不可能的。 

关键的观察是排列和子集选择完全分开。 一旦排列确定，每对前向顶点都会独立确定是否存在边。 这意味着每个排列定义了与该顺序一致的所有有向图上的均匀分布。 对于最终 DAG 来说唯一重要的结构是由可达性给出的诱导偏序。 

由此，DAG 的概率可以写成有多少排列与其拓扑顺序一致。 这将问题简化为对由构造引起的随机偏序的线性扩展的数量的二阶矩计算。 

经过代数简化后，两个独立生成的 DAG 相同的概率会分解为由构造选择总数确定的单个常数项，从而形成仅依赖于 n 的封闭形式。 

然后我们计算补集，因为该任务要求 DAG 不同的概率。

 ## 算法演练

最终的计算简化为评估从前向顶点对总数导出的模幂表达式。 

1. 计算 N = n(n − 1) / 2，这是任何顶点前向排序中可能的有向边的数量。 
2.仅通过模幂逻辑隐式地预先计算阶乘贡献； 我们从不明确地构建排列。 
3. 计算构建结果的总数为 2^N 乘以 n！ 从概念上讲，但只有指数部分在计算上很重要。 
4. 评估两个生成的 DAG 相同的概率，作为可能的构建结果总数的倒数。 
5. 用 1 减去该值即可获得它们不同的概率。 
6. 使用模幂和模逆运算返回模 1e9 + 9 的结果。 

### 为什么它有效

 每个 DAG 生成都可以被视为首先选择一个排列，然后独立决定每个前向边缘的存在。 这使得每个完整的施工结果都有相同的可能性。 由于两次运行是独立的，因此它们完全匹配的概率只是所有结果的概率平方和，它折叠为结果总数的倒数。 该过程的结构保证了构建历史上的统一概率空间，从而消除了对生成的 DAG 的特定形状的任何依赖。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 9

def modpow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

# total forward pairs
MAXN = 100000
fact = [1] * (MAXN + 1)
for i in range(1, MAXN + 1):
    fact[i] = fact[i - 1] * i % MOD

inv_fact = [1] * (MAXN + 1)
inv_fact[MAXN] = pow(fact[MAXN], MOD - 2, MOD)
for i in range(MAXN, 0, -1):
    inv_fact[i - 1] = inv_fact[i] * i % MOD

def solve(n):
    N = n * (n - 1) // 2

    total = fact[n] * modpow(2, N) % MOD

    # probability same = 1 / total
    same = pow(total, MOD - 2, MOD)

    return (1 - same) % MOD

t = int(input())
for _ in range(t):
    n = int(input())
    print(solve(n))
```包含阶乘表是因为概念计数分为排列选择和边子集，但在实现中我们只需要构造结果的最终组合计数。 模逆用于表示域模 1e9 + 9 中的除法。 

一旦顺序固定，指数 N 准确地捕获了存在多少个独立边缘决策。 

## 工作示例

 ### 示例 1：n = 2

 对于两个顶点，存在一对潜在的前向顶点。 构建结果的总数是 2 个排列选择乘以 2 个边缘存在选择，给出 4 个同等可能的结果。 

| 步骤| 价值|
 | ---| ---|
 | N = n(n−1)/2 | N = n(n−1)/2 | 1 |
 | 总计 = 2^N · 2！ | 4 |
 | 相同的概率 | 1/4 | 1/4
 | 不同概率 | 3/4 |

 这表明即使是最小的情况也已经将空间分割成多个同样可能的结构。 

### 示例 2：n = 3

 现在任何排列中都有 3 个正向对。 

| 步骤| 价值|
 | ---| ---|
 | N = n(n−1)/2 | N = n(n−1)/2 | 3 |
 | 总计 = 2^3 · 3！ | 48 | 48
 | 相同的概率 | 1/48 | 1/48
 | 不同概率 | 47/48 | 47/48

 此案例展示了在排列和独立边缘选择的驱动下，构造空间的增长速度有多快。

 ## 边缘情况

 对于 n = 1，构造总是产生空图。 该算法计算 N = 0，total = 1，因此倒数为 1，最终答案变为 0，这与两次运行必须相同的事实相匹配。 

对于 n = 2，前向结构中恰好存在一个可能的边缘方向。 该算法正确地考虑了排列和边缘选择，产生了不平凡的概率，而不是将所有内容折叠成单个确定性图。
