 ## 解决方案
 ## 问题理解

 我们得到一个无向简单图$n$顶点。 从这个图中，我们概念性地生成了可以通过以各种可能的方式重新标记顶点来获得的所有图。 产生不同边集的两次重新标记算作不同的图。 

问题不是枚举它们，而是确定与给定图同构的不同标记图的数量是否最多为$n$。 换句话说，我们采用顶点标签的所有排列，将它们应用于图，删除重复的相同结果边集，并计算出现的唯一图的数量。 我们必须决定这个计数是否受以下限制$n$。 

约束很大，所有测试用例的顶点和边的总数高达$10^5$。 这立即排除了任何涉及显式排列推理或图规范标签的事情。 任何依赖于阶乘增长、强力自同构检查或每个排列的图同构测试的方法在时间限制内都是不可能的。 

关键的困难在于我们所询问的数量取决于图的自同构群。 当两个排列因自同构不同时，它们会完全相同地产生相同的标记图。 所以不同图的数量同构于$G$等于$n! / |\mathrm{Aut}(G)|$。 因此，该任务相当于检查该值是否最多为$n$，但是直接计算自同构群的大小是远远不可行的。 

小图表中出现了一个微妙的问题。 为了$n=1$和$n=2$，几乎任何结构都会简单地分解为很少的几个不同的标记图，因此关于“许多排列”的天真的直觉可能会产生误导。 例如，当$n=2$，空图和单边图在重新标记下都精确地生成一个不同的标记图，即使有$2!$排列。 

## 方法

 直接方法将尝试理解顶点的所有排列并根据它们生成的图对它们进行分组。 从概念上讲，对于每个排列，我们都会重新标记图并将生成的边集插入到哈希集中。 这是正确的，但完全不可行。 有$n!$排列，甚至生成一个重新标记的图成本$O(n + m)$。 即使对于$n = 20$。 

关键的观察是同构类中不同标记图的数量完全由对称性决定。 高度对称的图具有较少的不同重新标记，因为许多排列不会改变边缘结构。 

我们希望这个数字最多是$n$，与$n!$。 这迫使自同构群变得非常大。 事实上，唯一的办法就是减少$n! / |\mathrm{Aut}(G)|$下降到$O(n)$是为了$|\mathrm{Aut}(G)|$顺序为$(n-1)!$或更大，这意味着几乎每个排列都是自同构。 

这种程度的对称性只有在所有顶点在尽可能最强的意义上结构上相同的图中才可能出现。 在简单的无向图中，这种情况仅发生在两种情况下：空图和完全图。 在这两种情况下，顶点的每个排列都保留邻接性，因此$|\mathrm{Aut}(G)| = n!$，因此不同标记图的数量恰好是$1$，这基本上满足了条件。 

任何其他图都会破坏对称性。 一旦存在一对具有不同度或不同邻接模式的顶点，将它们分开的排列就不再是自同构，并且自同构群急剧缩小。 这使得轨道尺寸远远超出$n$，所以答案变为NO。 

因此，问题简化为简单的结构检查：确定图是空的还是完整的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力排列 |$O(n! \cdot (n+m))$|$O(n+m)$| 太慢了|
 | 结构检查（空或完整）|$O(n+m)$|$O(n)$| 已接受 |

 ## 算法演练

 我们将问题简化为检查图是否具有零边或所有可能的边。 

1.读取顶点数$n$和边缘$m$，然后读取所有边。 
2. 计算一个简单无向图中可能的总边数，即$n(n-1)/2$。 
3.如果$m = 0$，该图是空的，这意味着每个排列都保留它，因此它满足条件。 
4. 如果$m = n(n-1)/2$，该图是完整的，这意味着每个排列都保留它，因此它也满足条件。 
5. 否则，该图既不是完全对称也不是最大连通，因此输出NO。 

这样做的原因是只有空图和完整图才具有完全的顶点对称性。 与这些结构的任何偏差都会引入至少一个顶点，其度数不同于另一个顶点或其邻接模式不同，从而破坏完全对称性。 这会立即减少自同构组的大小，并将不同的重新标记图的数量增加到超出允许的阈值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n, m = map(int, input().split())
        for _ in range(m):
            input()
        max_edges = n * (n - 1) // 2
        if m == 0 or m == max_edges:
            out.append("YES")
        else:
            out.append("NO")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现依赖于忽略计算边缘之外的实际边缘结构。 每个测试用例都会消耗其边缘列表而不存储它，因为只有计数很重要。 

的计算$n(n-1)/2$必须使用整数运算，并注意读取所有$m$边缘以保持输入解析对齐。

 ## 边缘情况

 空图的情况是完全对称的最直接的例子。 顶点的每个排列都保留了边的缺失，因此所有$n!$重新标记折叠成一个单独的图表。 该算法通过检查来处理这个问题$m = 0$，这会立即触发 YES。 

从对称性的角度来看，完整的图情况表现相同。 每对顶点都是连接的，因此重新标记不能改变邻接关系。 该算法通过以下方式检测到这一点$m = n(n-1)/2$。 

对于中间图，即使单个缺失或额外的边也会破坏完整的顶点可互换性。 例如，如果一个顶点的度数与另一个顶点不同，则交换它们会改变图结构，因此许多排列不再属于自同构群。 这保证了不同标记图的数量远远超出$n$，这就是为什么该算法在所有非极端情况下都安全地输出“否”。
