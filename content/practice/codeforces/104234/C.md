 ## 解决方案
 ## 问题理解

 我们反复尝试识别 1 到 n 之间的隐藏数字。 隐藏数并不是统一选择的：每个值 i 都有给定的权重 p_i，正确答案的实际概率与这些权重成正比。 每次错误猜测后，实验都会重置，并且根据参数 c，隐藏数字要么从相同的分布中重新采样，要么保持不变。 

我们可以提前选择一个固定的猜测策略：每次猜测时，我们都会以概率 q_i 选择索引 i。 如果我们的猜测与隐藏数字相符，则过程结束。 否则，我们就会失败并开始新一轮。 

目标是选择分布 q，使预期猜测次数最小化，直至成功。 

重要的微妙之处在于，即使隐藏的数字可能会在某些失败中持续存在（取决于 c），但实验保证从求解器的角度来看，每一轮的行为就像平稳过程下的一次新尝试。 这意味着整个过程简化为了解由 p 和 q 共同引发的每次猜测成功概率，然后对其进行优化。 

约束允许 n 最大为 100,000，因此任何解决方案都必须至多为 O(n log n) 或 O(n)。 所有分布的二次或基于模拟的方法立即不可行。 

一种常见的失败模式是假设贪婪地猜测（总是选择最可能的 p_i）是最优的。 这种直觉被打破了，因为目标取决于两个分布之间的相互作用，而不仅仅是 p 的众数。 

另一个微妙的陷阱是忽略 c 的影响。 虽然看起来它在轮次之间引入了记忆，但它并没有改变固定的成功率结构，只是改变了尝试之间的相关性。 幼稚的模拟或马尔可夫链展开会错误地暗示对历史的依赖，而这种依赖在预期中被抵消。 

## 方法

 暴力方法会尝试枚举 n 个结果上所有可能的猜测分布 q，并计算每个结果的预期试验次数。 即使我们离散化概率，搜索空间也会以 n 为指数，并且在 n 非常小时之外，甚至粗略采样也变得不可能。 

第二个天真的想法是每次猜测最可能的值，有效地将 q 设置为 argmax p_i 处的 delta 分布。 这很快，但不一定是最优的，因为它忽略了成功概率取决于 p 和 q 之间的重叠，而不仅仅是 p 本身。 

关键的见解是认识到每次猜测成功的概率由两个分布 p 和 q 的内积确定。 一旦观察到这一点，预期的猜测次数就变成了成功概率的倒数。 该问题简化为在 q 是概率分布的约束下最大化该内积。 

这是单纯形上的约束优化问题。 该结构意味着将所有概率质量集中在 p 的最佳对齐坐标上是最佳的，但 p 归一化的存在会改变对齐的行为方式。 简化概率缩放后，目标简化为优化涉及 p_i 之和的比率。 

c 的影响在平稳期望中消失，因为无论是否对隐藏数进行重采样，都不会改变稳态下每次试验的成功概率，只会改变试验之间的时间相关性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解 q | 指数| O(n) | 太慢了|
 | 优化内部产品结构 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将 S 表示为所有 p_i 的总和，并使用归一化概率分布 p_i / S。

1. 计算 S，所有结果的总权重。 这会将给定的权重转换为隐藏数字的实际概率分布。 
2. 观察到，对于任何固定的猜测分布 q，单次猜测成功的概率是隐藏数等于 i 且我们猜测 i 的概率，它是 (p_i / S) * q_i 对 i 的总和。 这是 p 和 q 之间的双线性形式。 
3. 成功之前的预期猜测次数是成功概率的倒数，因为尽管存在隐藏状态持久性，但每次尝试在分配效应上都是独立的。 
4. 为了最小化预期猜测，我们最大化成功概率和(p_i * q_i)。 由于 q 位于单纯形上，因此通过将所有质量集中在具有最大 p_i 的索引上来最大化 q。 
5. 因此，我们设置 q_k = 1，其中 p_k 最大，所有其他 q_i 为零。 
6. 最终的成功概率变为 max(p_i) / S，预期猜测次数为 S / max(p_i)。 

### 为什么它有效

 关键的不变量是每次尝试都有相同的边际成功概率，与过去的结果或隐藏值是否被重新采样无关。 c 参数仅影响连续隐藏状态之间的相关性，而不影响单个试验的期望。 由于期望仅取决于每次试验的边际成功，因此该过程会崩溃为参数等于 p 和 q 重叠的几何分布。 在单纯形上优化线性函数总是将所有质量置于极值点上，该点对应于单个索引。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    n, c = map(int, input().split())
    p = list(map(int, input().split()))
    
    s = sum(p)
    mx = max(p)
    
    print(s / mx)

if __name__ == "__main__":
    main()
```该实现直接遵循减少到两个简单聚合的比率。 唯一需要注意的是使用浮点除法来满足所需的精度。 

关键的一步是认识到不需要对状态进行动态编程； 一旦确定了正确的成功概率，整个随机过程就会崩溃为单个几何期望。

 ## 边缘情况

 均匀分布，例如`1 1 1`表明当所有结果的可能性相同时，没有任何猜测策略可以超越统一的成功概率，并且算法正确返回 n。 

单元素场景确认了边界行为，其中答案必须恰好为 1，因为可以立即保证成功。 

高度倾斜的分布，例如`10 1 1 1 1`证明该方法正确地优先考虑了主导概率质量，并与均匀猜测相比显着减少了预期猜测。
