 ## 解决方案
 ## 问题理解

 我们有一个矩形网格，其中每个单元格都已经分配了颜色。 该游戏涉及重复应用绘画操作，其中每个操作都用单一颜色绘制整行或整列，覆盖之前的任何内容。 操作的顺序很重要，因为后面的绘制会覆盖前面的绘制。 

关键问题是：哪些颜色可能是最终绘画操作中使用的颜色？ 

关键的观察是最后一个操作不能被其他操作覆盖。 如果最后的移动绘制了一行，那么在该移动之后，该行中的每个单元格都必须与最终颜色匹配，并且之后不会改变它。 如果最后一步绘制了一列，则同样的逻辑对称地适用。 

因此，任务简化为查找所有颜色，以便至少存在一行或一列可能是使用该颜色的最后一次绘制操作。 还有一个额外的限制，即一个特定值（代表“未绘制”或无效的颜色）不能被视为有效的最终颜色，因此必须将其从答案中排除。 

输入代表一个$n \times m$整数网格，输出是可以作为最终绘画颜色的所有有效颜色的列表，按升序排序。 

这种类型的典型网格问题所隐含的约束最多允许$10^5$细胞或更多。 这立即排除了对所有可能的重绘序列的任何三次或重复的深度模拟。 即使检查所有重涂订单本质上也是阶乘并且完全不可行。 

对每个可能的最后一个操作候选的简单扫描必须简化为对行和列的简单结构检查。 

当行或列几乎一致但包含单个不同的单元格时，就会出现微妙的失败情况。 例如，像这样的行$[3, 3, 3, 4]$如果仅检查部分一致性，可能会诱使天真的解决方案接受颜色 3。 但是，该行不能是颜色 3 的最终绘制行，因为最后一个操作将强制所有单元格具有完全相同的颜色。 列也会出现同样的问题。 

另一个陷阱是忘记“最后一次操作”必须完全确定该线上的最终网格状态，而不仅仅是多数一致性。 

## 方法

 强力解释将尝试模拟行和列绘画操作的所有可能序列。 对于每种候选颜色，我们可以想象选择最后绘制的行或列的所有方法，并检查是否可以一致地生成网格。 这会立即组合爆炸，因为每行或每列都可以按多个顺序绘制，并且序列数在总行数中呈阶乘增长。 即使是在最后一次操作中检查所有行和列的简化版本仍然需要在线性时间内针对网格验证每种可能性，从而导致$O((n+m)\cdot nm)$，对于最大网格来说太大了。 

关键的结构见解是最后一个操作完全控制至少一条整线。 如果最后一个操作绘制一行，则该行在最终网格中必须已经是均匀的。 这同样适用于列。 这将问题转化为直接验证任务：对于每一行，检查其所有元素是否相等； 对于每一列，执行相同的操作。 每条这样的统一线都直接标识候选颜色。 

这是有效的，因为任何有效的最终状态都必须包含最后绘制的线，并且该线在最后步骤之后不能包含混合颜色。 因此，统一不仅是必要的，而且对于成为候选人来说就足够了。

 ## 算法演练

 1.读取尺寸网格$n \times m$。 这是应用所有绘画操作后的最终状态。 
2. 独立检查每一行并确定该行中的所有值是否相同。 如果一行是均匀的，则将其颜色记录为候选颜色。 原因是，有效的最后一步可能会绘制整行，之后保持不变。 
3. 对每一列重复相同的检查。 对于每一列，验证所有值是否相同。 如果是这样，也将该列的颜色记录为候选颜色。 
4. 将所有发现的候选颜色存储在一个集合中以避免重复。 颜色可以同时显示为统一的行和统一的列。 
5. 如果问题指定特定值不能被视为有效，则删除禁用颜色。 
6. 对剩余的候选颜色进行排序，并按升序输出。 

### 为什么它有效

 有效的最终操作必须一次性覆盖整行或整列。 在该移动之后，不存在修改任何单元格的后续操作。 这会强制该行中的每个单元格共享相同的最终值，否则它将被不一致地覆盖。 因此，任何有效的最后一步都完全对应于最终网格中完全统一的行或列。 相反，任何统一的行或列都可以被解释为最后一步，只需假设它是在最后绘制的，这也足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    g = [list(map(int, input().split())) for _ in range(n)]

    bad = -1  # if problem defines forbidden color, adjust if needed
    # (in many CF variants this is given; here we keep placeholder logic)

    ans = set()

    for i in range(n):
        ok = True
        for j in range(1, m):
            if g[i][j] != g[i][0]:
                ok = False
                break
        if ok:
            ans.add(g[i][0])

    for j in range(m):
        ok = True
        for i in range(1, n):
            if g[i][j] != g[0][j]:
                ok = False
                break
        if ok:
            ans.add(g[0][j])

    if bad in ans:
        ans.remove(bad)

    res = sorted(ans)
    print(len(res))
    if res:
        print(*res)

if __name__ == "__main__":
    solve()
```行检查遍历每一行并将每个单元格与第一个元素进行比较，这足以确定一致性。 列检查在垂直方向上执行相同的操作。 使用集合可以确保行和列中出现的颜色不会重复。 

一个微妙的一点是我们从不尝试重建绘画序列。 我们纯粹依赖于最终网格的结构约束，这避免了任何指数推理。

 ## 边缘情况

 一种重要的边缘情况是行或列中仅存在一个单元格。 例如，一个网格$n = 1$意味着每一列都是均匀的。 该算法正确地将每一列视为有效，因为单个值始终满足一致性。 反之亦然$m = 1$，其中每一行自动统一。 

另一种情况是网格，其中所有行都不均匀，但某些列是均匀的。 例如：```
3 3
1 2 3
1 2 3
1 2 3
```在这里，没有行是统一的，但所有列都是统一的。 该算法收集 {1,2,3} 作为候选。 这是正确的，因为任何列都可能是有效序列中的最后一个绘制操作，即使没有行符合条件。 

最后一个微妙的情况是禁止的颜色出现在统一的行或列中。 基于集合的方法确保在最后很容易将其过滤掉，而不会干扰扫描过程中的检测逻辑。
