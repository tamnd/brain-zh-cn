 ## 解决方案
 ## 问题理解

 我们正在研究一个规划问题，其中一个人重复执行一项需要权衡的行动：每个工作单元都会产生一定量的“食物价值”，但也会消耗一定量的耐力或努力。 每种动作类型都可以执行多次，我们希望了解获得的总食物和消耗的总耐力之间的最佳权衡。 

从形式上来说，每个“项目”对应一种膳食活动，具有固定的食物质量增益和固定的耐力成本。 您可以选择执行这些活动的次数，并且操作总数隐式地受到耐力或我们关心的最大食物的限制。 预处理后，我们需要回答以下形式的多个查询：如果允许我们最多消耗给定量的体力，我们可以获得的最大食物质量是多少？ 

自然结构是一个背包式的优化问题，但关键困难在于耐力极限和价值尺度都足够大，以至于直接在朴素维度上进行DP是不可行的。 这些约束意味着任何超过完全限制的二次或伪二次状态空间都将失败，因此我们需要通过仔细的状态转换进行一维优化。 

当所有物品的耐力成本都非常小但价值却很大时，就会出现微妙的边缘情况，这使得贪婪推理失败。 当错误地反转 DP 方向导致同一过渡层内的项目重复使用时，就会出现另一种故障模式，当预期模型根据解释而不同时，有效地将问题转化为无界背包。 正确的解决方案必须严格控制转移顺序。 

## 方法

 最直接的表述是定义一个状态，其中 dp[s] 是使用 s 单位耐力可实现的最大食物值。 这是一个经典的背包公式：对于每个项目，我们尝试通过添加该项目来扩展先前的状态。 这是正确的，但立即就会遇到麻烦，因为耐力维度可能非常大，如果我们还需要处理许多查询，则无法有效地迭代所有 s。 

更实用的暴力视图是针对每个查询限制计算该约束下的最佳项目子集。 这导致每个查询一个标准的 0/1 背包，每个查询的成本为 O(nS)，完全不可行。 

关键的观察是，我们实际上并不需要在查询时分别为每个耐力预算提供最佳值。 相反，我们只需要耐力成本和食物价值之间的帕累托边界：对于任何可实现的食物价值，我们需要所需的最低耐力。 这将问题转化为双背包DP问题。 

我们将状态重新定义为 dp[v]，即精确达到 v 单位食物价值所需的最小耐力。 这翻转了视角，并允许我们通过可实现的总价值总和来限制 DP 维度，该总价值通常要小得多或以允许压缩的方式构建。 一旦建立了这个 DP，回答查询就变成了单调搜索问题：给定耐力限制，我们找到最大的 v 使得 dp[v] 在限制内，这可以用二分搜索来回答。 

剩下的技术问题是确保正确的转换顺序。 由于我们正在最大限度地减少耐力，因此必须以一种避免在单次迭代中多次重复使用每个项目的方式进行处理，除非明确允许。 这是通过按值的降序迭代 dp 数组来处理的，确保每个项目每层应用一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 每个查询的耐力指数 DP | O(q·n·S) | O(q·n·S) | O(S)| 太慢了 |
 | 值索引 DP + 二分查找 | O(n·V + q log V) | O(n·V + q log V) | O(V) | 已接受 |

 ## 算法演练

1. 以食品质量为主轴，将问题转化为价值可达性DP。 我们将 dp[v] 定义为精确获取 v 总值所需的最小耐力。这种转变是必要的，因为查询限制在耐力上，所以我们想要反转映射。 
2. 用较大的哨兵值初始化 dp，并设置 dp[0] = 0。这表示实现零食物需要零耐力。 
3. 对于每个具有价值增益 val 和耐力成本成本的食物选项，以 v 的降序更新 dp 数组。对于每个 dp[v] 已经可达的 v，我们尝试转换到 v + val 并更新 dp[v + val] = min(dp[v + val], dp[v] + cost)。 反向迭代保证每个 DP 层中的每个项最多使用一次。 
4. 处理完所有项目后，dp 数组代表完整的帕累托前沿：对于每个可实现的食物值，我们知道所需的最低耐力。 
5. 预处理辅助结构，使 dp 值随着 v 的增加而单调最小化，确保对于任何 v，dp[v] 是至少实现该值的最佳耐力。 此步骤删除非最佳主导状态。 
6. 对于每个查询耐力限制 S，对 v 执行二分搜索以找到最大 v，使得 dp[v] ≤ S。这是有效的，因为 dp[v] 是单调的，即在最佳前沿表示中实现更高的值不能需要更少的耐力。 

为什么它有效

 DP 维护一组代表所有可实现的价值-耐力对的状态，但未来的最佳答案永远不需要主导状态。 任何需要更多耐力才能达到相同或更低值的状态都不是任何查询的最佳解决方案的一部分。 反向迭代确保一次性转换的正确性，保留预期的组合结构。 二分搜索步骤是有效的，因为最终的 DP 在修剪主导状态后编码了单调前沿。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

INF = 10**18

def solve():
    n, q = map(int, input().split())
    
    items = []
    max_val = 0
    
    for _ in range(n):
        v, c = map(int, input().split())
        items.append((v, c))
        max_val += v

    dp = [INF] * (max_val + 1)
    dp[0] = 0

    for v, c in items:
        for cur in range(max_val - v, -1, -1):
            if dp[cur] != INF:
                nv = cur + v
                nd = dp[cur] + c
                if nd < dp[nv]:
                    dp[nv] = nd

    for i in range(1, max_val + 1):
        if dp[i] > dp[i - 1]:
            dp[i] = dp[i - 1]

    def get_best(stamina):
        lo, hi = 0, max_val
        ans = 0
        while lo <= hi:
            mid = (lo + hi) // 2
            if dp[mid] <= stamina:
                ans = mid
                lo = mid + 1
            else:
                hi = mid - 1
        return ans

    for _ in range(q):
        s = int(input())
        print(get_best(s))

if __name__ == "__main__":
    solve()
```核心代码是背包式的DP超过总价值而不是总成本。 反向循环结束`cur`确保每个项目每次迭代应用一次，防止意外重用。 第二个循环强制单调性，因此较高的值永远不会比较低的值人为地具有更好的耐力要求。 

二分搜索之所以有效，是因为在压平主导状态后，dp 成为一个非递增函数，因为更高的可实现值至少需要同样多的耐力。 

## 工作示例

 ### 示例 1

 假设我们有以下项目（价值，成本）：(3, 2), (2, 1)，并且查询要求在耐力限制 1, 2, 3 下获得最大值。 

我们一步步构建dp。 

| 项目 | dp[0] | dp[0] | dp[2] | dp[2] | dp[3] | dp[3] | dp[5] | dp[5] |
 | ---| ---| ---| ---| ---|
 | 初始化| 0 | 信息 | 信息 | 信息 |
 | (2,1) | 0 | 1 | 信息 | 信息 |
 | (3,2) | 0 | 1 | 2 | 3 |

 修剪优势后，dp 显示值 2 消耗 1 耐力，值 3 消耗 2 耐力，值 5 消耗 3 耐力。 

对于耐力 2，二分查找发现值 5 太昂贵，值 3 可行，所以答案为 3。 

这证实，根据成本效率，首先组合较小的项目可能会主导单个大型项目。 

### 示例 2

 项目：(5, 5)、(4, 3)、(2, 2)。 查询体力5.

 | 步骤| 最佳可达值 |
 | ---| ---|
 | (5,5) | 0, 5 |
 | (4,3) | 0、4、5、9 |
 | (2,2) | 0、2、4、6、5、7、9 |

 耐力 5 下的最佳值为 9（通过 (4,3)+(2,2)）。 

这个轨迹说明了为什么我们需要 DP 而不是贪婪：最佳效率组合不一定是最大的单个项目。 

## 复杂度分析

 | 测量| 复杂性 | 说明|
 | ---| ---| ---|
 | 时间 | O(n·V + q log V) | O(n·V + q log V) | DP处理取值范围V上的每一项，查询使用二分查找|
 | 空间| O(V) | DP 阵列存储每个值的最佳耐力 |

 该解决方案是可以接受的，因为 V 受值之和限制，该值明显小于任何朴素耐力维度，并且查询被简化为对数搜索而不是完全重新计算。

import sys, io

def run(inp: str) -> str:

    import sys
    from math import inf

    INF = 10**18

    n, q = map(int, sys.stdin.readline().split())
    items = []
    max_val = 0

    for _ in range(n):
        v, c = map(int, sys.stdin.readline().split())
        items.append((v, c))
        max_val += v

    dp[0] = 0

    for v, c in items:
        for cur in range(max_val - v, -1, -1):
            if dp[cur] != INF:
                nv = cur + v
                dp[nv] = min(dp[nv], dp[cur] + c)

    for i in range(1, max_val + 1):
        dp[i] = min(dp[i], dp[i - 1])

    def best(s):
        lo, hi = 0, max_val
        ans = 0
        while lo <= hi:
            mid = (lo + hi) // 2
            if dp[mid] <= s:
                ans = mid
                lo = mid + 1
            else:
                hi = mid - 1
        return ans

    out = []
    for _ in range(q):
        out.append(str(best(int(sys.stdin.readline()))))
    return "\n".join(out)

 ## 边缘情况

 一个重要的边缘情况是，所有物品单独的耐力都太昂贵，但一起形成一个有效的组合。 例如，在低耐力限制下，项目（3,2）和（4,3）不能被单个贪婪选择替换。 DP 正确地处理了这个问题，因为它枚举了所有可达到的值总和，而不是提前提交。 

另一种极端情况是多个商品具有相同的价值成本比。 在这种情况下，不正确的修剪可能会删除有效的组合。 单调前缀步骤 dp[i] = min(dp[i], dp[i-1]) 确保即使多个状态以不同的成本达到相同的值，也只保留最好的耐力，防止错误的查询答案。 

当不存在任何项目或所有查询的耐力为零时，就会出现第三种边缘情况。 初始化 dp[0] = 0 确保二分查找始终返回至少零值，正确表示空选择状态。
