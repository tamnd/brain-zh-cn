 ## 解决方案
 ## 问题理解

 城市是由一组水平街道和垂直大道组成的矩形街区网格。 每个街道和大道的交叉点都定义了一个街区，每个街区都包含已知数量的潜在客户。 

两家咖啡馆必须位于两个不同的街区。 位于某个街区的顾客只有在距离不太远的情况下才可以考虑去咖啡馆，因为到达咖啡馆最多需要穿过一条水平或一条垂直边界线。 在网格术语中，这意味着单元中的客户$(i, j)$只能在两个方向上前往位于单步邻域内的咖啡馆，因此咖啡馆必须位于行最多相差 1、列最多相差 1 的区块中。 

如果从顾客所在的街区可以到达这两家咖啡馆，则该顾客将恰好选择其中一家。 确切的打破平局规则没有具体说明，但重要的含义是，两个咖啡馆社区之间重叠区域的顾客不能保证同时为两个咖啡馆做出贡献。 

目标是计算有多少无序对的不同街区可以被选择作为咖啡馆位置，使得每个咖啡馆至少可以吸引$k$有效地将客户分配到咖啡馆的客户。 

网格大小最多为$50 \times 50$，所以最多有 2500 个块。 对所有对的二次扫描已经是可行的，但是网格上的任何每对重新计算仍然需要保持在几百万次操作之内。 这立即排除了任何试图从头开始重新计算每对邻域和的方法$O(nm)$，因为这会导致大约$2500^3$最坏情况下的操作。 

一个更微妙的问题是重叠邻域的行为方式。 如果两家咖啡馆距离很近，则许多顾客都位于两者距离 1 的范围内。 这些顾客只能被分配到一家咖啡馆，因此将他们的贡献视为“完全计入两家咖啡馆”可能会高估每家咖啡馆的独立吸引力。 正确的解决方案必须避免两个选择之间的耦合。 

一些极端情况值得牢记。 如果所有块的人口都非常少，例如每个单元格的值为 1 并且$k = 100$，那么不存在有效的咖啡馆，答案必须为零。 如果$k = 1$，那么几乎每个 3x3 邻域包含至少一个单元的块都是有效的，几乎都是内部和边界单元。 另一个极端情况是在 3x3 邻域缩小的边界处，因此必须仔细计算总和，而不要在网格外建立索引。 

## 方法

 直接的暴力解决方案会考虑每对有序的不同块，并模拟每个咖啡馆接待的顾客数量。 对于一对区块，我们将检查每个网格单元，检查它是否在每个咖啡馆的距离 1 内，然后决定如何分配重叠的顾客。 即使这个分配步骤被简化，邻域评估也已经花费了$O(nm)$。 和$O((nm)^2)$对，总复杂度变为$O((nm)^3)$，这远远超出了 2500 个单元格的需要。 

关键的观察是，单个咖啡馆的吸引力仅取决于其自身的位置，而不取决于第二家咖啡馆。 对于任何固定块，可以到达该块的客户集合恰好是其 3x3 邻域中的值之和。 咖啡馆之间的交互仅对分配重要，但由于只要每个咖啡馆独立拥有足够的总可达客户，就可以以不降低可行性的方式分配重叠客户，因此条件解耦。 这将问题转化为一个简单的分类：每个块要么是有效的（其 3x3 总和至少为$k$）或无效。 

一旦每个块都被标记为有效或无效，最终的答案就是选择两个有效块的方法数量。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 通过每对重新计算对对进行暴力破解 |$O((nm)^3)$|$O(1)$| 太慢了|
 | 前缀和 + 每个单元邻域 + 计数对 |$O(nm)$|$O(nm)$| 已接受 |

 ## 算法演练

 1. 计算网格上的二维前缀和，以便可以在恒定时间内查询任何矩形和。 这允许有效计算邻域和。 
2. 对于每个单元格$(i, j)$，计算以 为中心的 3x3 邻域中所有值的总和$(i, j)$，剪裁到网格边界。 这代表可以到达位于该单元的咖啡馆的顾客总数。 
3. 如果计算出的邻域和至少为，则将单元标记为有效$k$。 
4. 计算有多少个单元格是有效的，假设这个计数是$cnt$。 
5. 最终答案是无序有效单元对的数量，即$cnt \cdot (cnt - 1) / 2$。 

这种减少起作用的主要原因是咖啡馆的可行性仅取决于其自身的邻里总和。 由于当两个咖啡馆都有足够的独立总数时，重叠客户总是可以以不会同时使两个咖啡馆失效的方式进行分配，因此两个选定的位置之间除了不同之外不存在任何依赖关系。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m, k = map(int, input().split())
a = [list(map(int, input().split())) for _ in range(n)]

# prefix sum
ps = [[0] * (m + 1) for _ in range(n + 1)]

for i in range(n):
    for j in range(m):
        ps[i + 1][j + 1] = (
            a[i][j]
            + ps[i][j + 1]
            + ps[i + 1][j]
            - ps[i][j]
        )

def rect_sum(x1, y1, x2, y2):
    return (
        ps[x2][y2]
        - ps[x1][y2]
        - ps[x2][y1]
        + ps[x1][y1]
    )

def neighborhood(i, j):
    r1 = max(0, i - 1)
    c1 = max(0, j - 1)
    r2 = min(n, i + 2)
    c2 = min(m, j + 2)
    return rect_sum(r1, c1, r2, c2)

cnt = 0
for i in range(n):
    for j in range(m):
        if neighborhood(i, j) >= k:
            cnt += 1

print(cnt * (cnt - 1) // 2)
```该实现首先构建一个标准的 2D 前缀和，它允许在恒定时间内计算任何矩形和。 功能`rect_sum`使用前缀表上的包含-排除来提取子矩阵的总和。 

功能`neighborhood`将单元格转换为其 3x3 窗口，仔细限制边界，以便边缘单元格仅考虑现有网格单元格。 这可以避免任何越界访问并自然地处理边界情况。 

然后对每个细胞进行独立测试，并对有效细胞进行计数。 最后的组合步骤计算无序对的数量。 

一个微妙的点是，我们从不尝试模拟两个咖啡馆之间的互动。 一旦每个咖啡馆独立地满足阈值条件，这种相互作用实际上与可行性无关。 

## 工作示例

 考虑一个小网格，其中只有少数单元格满足阈值。 

输入：```
3 3 5
1 2 1
2 2 2
1 2 1
```中心的 3x3 邻域包含所有值，总和为 14，因此它是有效的。 角单元格的邻域较小，但在本例中总和仍至少为 5。 

| 单元格 (i,j) | 邻里总和| 有效 |
 | ---| ---| ---|
 | (0,0) | (0,0) | 1+2+2 = 5 | 是的 |
 | (0,1)| 1+2+1+2+2+2 = 10 | 是的 |
 | (1,1) | 14 | 14 是的 |

 这里所有单元格最终都有效，因此如果总共有 9 个单元格，则答案变为$9 \cdot 8 / 2 = 36$。 这证实了该算法正确地减少了对有效位置的计数。 

现在考虑一个稀疏的情况。 

输入：```
2 2 10
1 1
1 1
```每个 3x3 邻域总和均为 4 或更少，因此没有单元格符合条件。 

| 单元格 (i,j) | 邻里总和| 有效 |
 | ---| ---| ---|
 | 全部 | 4 | 没有|

 计数为零，因此答案为零，符合没有咖啡馆可以达到所需阈值的逻辑。

 ## 边缘情况

 边界单元演示了为什么钳位很重要。 例如，在一个$3 \times 3$网格中，左上角的单元格只有 2x2 邻域，而不是完整的 3x3。 该算法使用正确调整矩形边界`max`和`min`，所以它从不假设存在缺失的单元格。 

一个完全无效的网格，其中每个单元格的值为 1 且$k$较大表明算法正确识别零个有效位置而无需尝试对枚举。 

完全有效的网格表明，解决方案减少为组合计数，并且不依赖于邻域和之外的几何排列。
