 ## 解决方案
 ## 问题理解

 我们给出了一系列牛奶批次，每个批次都有两个属性。 第一个是系数$b_i$它代表每单位加热温度 Vova 获得的风味量。 第二个是极限$t_i$它代表该批料可以加热而不产生泡沫的最高温度。 如果选择的全局加热温度是$T$，然后批量$i$仅当满足以下条件时才可饮用$T \le t_i$。 如果它是可以饮用的，它的贡献正是$b_i \cdot T$到完全的享受。 如果$T > t_i$，该批次无法使用并且没有任何贡献。 

任务是选择一个整数温度$T$这适用于所有批次，以便使所有保持无泡沫的批次的贡献总和最大化。 

关键的结构是$T$是全球性的。 我们不是独立优化每个批次，而是选择一个值，该值同时线性增加某些项目的贡献，同时可能排除其他项目。 

输入尺寸$n$最大可达 100000，且值为$t_i$可以大到$10^9$。 这立即排除了任何尝试所有可能温度的方法。 一个蛮力就过去了$T$需要检查至$10^9$候选人，这是不可能的。 由于指数增长，即使迭代所有对或子集也是不可行的。 

一个天真的观察是答案仅取决于值定义的“断点”$t_i$，因为增加$T$仅当超过这些阈值之一时才更改活动批次集。 

天真的推理的一个微妙的失败案例是假设$T = \max t_i$总是最优的。 例如，如果一批有很大$b_i$但很小$t_i$，增加$T$超过该阈值将完全删除它，可能会减少总和。 

另一种失败模式是尝试评估每个$T$独立地不重复使用计算，导致$O(n \cdot \max t_i)$或类似不可能的复杂性。 

## 方法

 蛮力的想法很简单。 对于每个可能的整数温度$T$，我们计算所有的总和$b_i \cdot T$这样$t_i \ge T$。 对于每位候选人$T$，我们扫描所有$n$项目，检查可行性，并积累贡献。 这是正确的，因为它直接模拟了问题的定义。 

然而，这种方法太慢了，因为$T$范围可达$10^9$。 即使我们将候选人限制为独特的$t_i$值，我们仍然有$n$候选人，并且每个人都需要$O(n)$工作，导致$O(n^2)$，对于$10^5$。 

关键的见解是扭转观点。 我们不是固定一个温度并检查哪些批次存活下来，而是固定一个批次并询问它贡献的温度。 批$i$为所有人做出贡献$T \le t_i$，其贡献是线性的$T$。 这意味着总函数是线性函数的总和，但每个函数都在其自身的截止点处被截断。 

我们可以按照$t_i$。 随着我们增加$T$, 批次在何时失效$T$超过了他们的阈值。 这表明扫除可能的$T$值按降序排列$t_i$，维护哪些批次仍处于活动状态。 

更清洁的转型是将贡献视为函数$T$：$$f(T) = T \cdot \sum_{i: t_i \ge T} b_i$$所以问题就变成了寻找最大值$T \cdot S(T)$， 在哪里$S(T)$是$b_i$其中$t_i \ge T$。 作为$T$增加，$S(T)$只会减少，而$T$增加。 最佳结果发生在我们完全包含所有项目的点$t_i \ge T$，对应于排序阈值和检查候选断点。 

我们对对进行排序$(t_i, b_i)$按递增顺序$t_i$。 然后我们从最大的开始处理$t_i$向下，保持运行总和$b_i$。 在每个阈值$t_i$，我们考虑选择$T = t_i$，因为在两个相邻阈值之间，活动集没有变化，并且增加$T$仅提高乘法因子，同时保持活动集固定直到边界。 

因此我们只需要评估$T \cdot (\text{sum of } b_i \text{ with } t_i \ge T)$在这些关键点上。

 | ---| ---| ---| ---|
 | 暴力破解 T | O(n·最大 t) | O(1) | O(1) | 太慢了|
 | 最优排序+清扫 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题转化为在从约束导出的少量有意义的候选点处评估函数。 

1. 将每批配对为$(t_i, b_i)$并对列表进行排序$t_i$按递增顺序。 当温度升高时，这会根据批次停止有效的时刻来组织批次。 
2. 计算所有的总和$b_i$。 这代表了初始值$S(T)$对于非常小的$T$，其中所有批次均有效。 
3.从最小的开始遍历已排序的数组$t_i$到最大。 在每个位置$i$，我们对待$t_i$作为候选温度。 
4. 维持运行后缀之和$b_i$，但随着我们前进，通过减法隐式计算。 当我们传递带有阈值的批次时$t_i$，它对于任何更高的温度都不再有效，因此我们删除它$b_i$来自活动总和。 
5. 在每个不同的阈值$t_i$，将候选答案计算为$t_i \times (\text{current active sum})$。 我们仅在恰好处于边界时或在更新活动集之后进行评估。 
6. 跟踪所有此类候选人评估的最大值。 

我们可以将自己限制在这些点上的原因是，在两个连续阈值之间，活动集$S(T)$是恒定的。 在这样一个区间内，$f(T) = T \cdot S$是一个线性函数$T$，因此其区间内的最大值始终位于右端点。 

### 为什么它有效

 正在优化的函数是分段线性的，其中断点恰好出现在值处$t_i$。 在断点之间，斜率不会改变，因为活动批次集保持固定。 自从$f(T)$在每个区间上都是线性的，它在该区间上的最大值必须位于一个端点。 因此，将评估限制在终点$t_i$足以捕获全局最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    b = list(map(int, input().split()))
    t = list(map(int, input().split()))
    
    pairs = list(zip(t, b))
    pairs.sort()
    
    total_b = sum(b)
    best = 0
    
    i = 0
    n = len(pairs)
    
    while i < n:
        j = i
        # group equal t values
        while j < n and pairs[j][0] == pairs[i][0]:
            j += 1
        
        # temperature candidate is pairs[i][0]
        T = pairs[i][0]
        
        # remove all elements with t == T from active set
        for k in range(i, j):
            total_b -= pairs[k][1]
        
        # now active set is all with t > T
        best = max(best, T * total_b)
        
        i = j
    
    print(best)

if __name__ == "__main__":
    solve()
```该实现依赖于按阈值对对进行排序。 变量`total_b`从所有系数的总和开始，当我们通过每个阈值组时逐渐失去贡献。 删除具有当前阈值的所有批次后，我们计算该确切温度下的贡献，因为如果我们要增加超过此点，这些批次将不再有效。 

分组相等$t_i$值很重要，因为当超过该阈值时，所有这些值都会同时变得无效，并且我们必须确保在评估该级别的候选人之前将它们删除。

 ## 边缘情况

 一个关键的边缘情况是最佳温度是最小阈值。 考虑输入：```
2
100 1
1 100
```如果我们选择$T = 1$，两批均贡献，给出$101$。 如果我们选择$T = 100$，只有第二批做出贡献，给出$100$。 该算法评估两个断点。 在$T = 1$，还没有发生删除，所以分数是$1 \cdot 101$。 在$T = 100$，第一批之前已经被删除了，所以只$1 \cdot 1$留下来，生产$100$。 最大值被正确识别。 

另一个边缘情况是当所有$t_i$是相同的。 然后立即删除所有批次，并且唯一有意义的评估发生在该单个阈值。 该算法处理该组一次，减去所有$b_i$，并评估超出它的零贡献，确保正确性而无需特殊的套管。
