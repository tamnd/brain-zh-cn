 ## 解决方案
 ## 问题理解

 我们得到了放置在 100 x 100 网格上的几位国王的位置。 在一个动作中，我们选择一个国王并开始向八个方向中的任何一个方向逐步移动它，因此每次移动都会将其单元格更改为包括对角线在内的相邻方块之一。 每当这个移动的国王进入已经包含另一个国王的单元格时，该国王就会从棋盘上移除，并且该选定国王的移动过程将继续，直到它执行这样的捕获。 捕获发生后，我们停止此操作，并可以选择任何剩余的国王来重复该过程。 

目标是将国王的集合减少到恰好一个，从而最大限度地减少所有行动中单步国王移动的总数。 

从约束中观察到的关键是最多有 100 个国王。 这足够小，任何尝试考虑所有国王对之间关系的解决方案都是可行的，因为超过 100 个节点的完整图只有 4950 条边。 这立即表明可能涉及全连接结构上的成对距离和全局优化。 

一个微妙的问题是理解“移动”成本的真正含义。 捕获不是免费的； 它花费从起始国王到目标国王所需的国王步数。 这不仅仅是图意义上的邻接，而是 8 个连通网格上的几何最短路径，这正是两点之间的切比雪夫距离。 

天真的误解会将每次捕获视为成本 1，这是错误的，因为全面移动可能需要许多步骤。 另一种失败模式是只贪婪地考虑最近邻居，这没有考虑有效合并多个国王的长期结构。 

## 方法

 蛮力视图是模拟重复选择移动哪一个国王和捕获哪一个目标的整个过程，并为每个可能的序列计算总成本。 这相当于探索合并 k 个对象的所有可能顺序，并在每一步尝试以精确的路径成本合并所有可能的对。 即使我们忽略路径重新计算，序列的数量也是 k 的阶乘，即使 k = 20，这也是完全不可行的，更不用说 100 了。 

关键的结构见解是重新解释该过程。 每次我们移动一个国王来捕获另一个国王时，我们实际上是将两个组件合并为一个，并且合并的成本仅取决于它们当时的位置。 由于最终结果是剩下一个国王，因此除一名国王之外的所有国王都必须被吸收一次。 这正是在初始点集上构建生成树的结构，其中每条边对应一个捕获操作，权重是当时合并的两个国王之间的移动成本。 

无论合并顺序如何，两个国王 (i, j) 之间的距离都是固定的，因为合并的目的地始终是两个原始端点之一的位置。 这将问题转化为在完整的国王图上找到最小生成树，其中每个边的权重是坐标之间的切比雪夫距离。 

一旦这样看，问题就简化为最多 100 个节点上的标准 MST 计算，可以使用 Kruskal 算法有效地解决。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力合并模拟 | k 的指数 | O(k) | 太慢了|
 | 完整图上的最小生成树 | O(k^2 log k) | O(k^2 log k) | O(k^2) | O(k^2) | 已接受 |

 ## 算法演练

 我们将每个国王视为图中的一个节点，并将任意两个国王之间的成本定义为在空棋盘上从一个国王移动到另一个国王所需的移动次数。 

1. 使用切比雪夫距离计算每对国王之间的成本。 这是`max(|r1 - r2|, |c1 - c2|)`因为国王可以沿对角线移动，同时减少行和列的差异。 这定义了一个完整的加权图。 
2. 使用计算出的权重构造所有边 (i, j)。 由于 k 至多为 100，因此边的数量是可控的。 
3. 按权重以非降序对所有边进行排序。 这使我们能够贪婪地选择不会在合并结构中创建循环的最小可用连接。 
4. 在 k 个节点上运行不相交集并集结构。 最初，每个国王都是其自己的组成部分。 
5. 迭代已排序的边。 对于每条边 (i, j)，如果 i 和 j 位于不同的分量中，则将它们合并并将边权重添加到答案中。 继续下去，直到所有国王都连接成一个组件，这正好需要 k − 1 次合并。 

每次合并对应于原始过程中的一次捕获操作，累积的边权重表示所需的单步王移动总数。 

### 为什么它有效

 每个有效的捕获序列都在原始国王上定义了一棵生成树，其中每条边对应于两个先前单独的组件之间的一个捕获步骤。 该步骤的成本正是合并时所选对之间的切比雪夫距离，它与完整图中的固定边权重相匹配。 任何导致剩下一个国王的策略都必须精确执行 k − 1 次合并，因此对应于某个生成树。 

相反，任何生成树都可以通过选择根并模拟沿其边缘的合并来实现，确保每个边缘只被支付一次。 因此，最小化总移动成本相当于找到该图的最小生成树。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        return True

def solve():
    k = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(k)]

    edges = []
    for i in range(k):
        r1, c1 = pts[i]
        for j in range(i + 1, k):
            r2, c2 = pts[j]
            w = max(abs(r1 - r2), abs(c1 - c2))
            edges.append((w, i, j))

    edges.sort()
    dsu = DSU(k)

    ans = 0
    used = 0

    for w, a, b in edges:
        if dsu.union(a, b):
            ans += w
            used += 1
            if used == k - 1:
                break

    print(ans)

if __name__ == "__main__":
    solve()
```DSU 维护哪些国王已经通过选定的合并连接起来，确保我们永远不会形成循环。 每个成功的联合对应于在两个先前单独的组之间选择一个捕获操作。 排序的边保证我们总是首先选择最便宜的可用连接，这正是定义最小生成树的贪婪属性。 

切比雪夫距离计算至关重要，因为它模拟了空网格中国王移动的真实最小数量。 任何其他距离定义都会打破最短移动成本的等价性。

 ## 边缘情况

 最小的边缘情况是恰好有两个国王。 该算法构造一条权重等于切比雪夫距离的单边并立即输出。 这与唯一可能的移动顺序匹配，因此正确性是立即的。 

退化的几何配置是指所有国王都排列在一条直线上或挤成一个紧密的簇。 在这种情况下，许多边具有相等或接近相等的权重。 DSU 确保即使存在多个等权边，也仅选择 k − 1，从而避免计数过多。 

一个遥远的异常值案例说明了为什么贪婪聚类很重要。 如果一个国王距离较远，MST 结构会延迟其连接，直到所有较近的合并都用尽，从而确保昂贵的边仅使用一次，这与最终吸收该孤立国王的必要性相匹配。
