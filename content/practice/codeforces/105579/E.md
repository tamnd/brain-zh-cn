 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times n$代表瓷砖墙的网格。 每个单元格包含一个金块，写为`G`，或银色瓷砖，写为`S`。 我们还得到一个整数$k$，我们想要选择一个连续的$k \times k$该网格的子正方形。 选择该子正方形后，我们可以重新绘制其中的一些瓷砖，以使整个正方形变得均匀，这意味着所有瓷砖都是相同的类型。 

对于任何选择的$k \times k$地区，我们可以决定做一切`G`或一切`S`。 成本是与所选目标类型不匹配的单元格数量。 任务是计算所有可能的最小重绘成本$k \times k$子方块并且都以统一的颜色为目标。 

网格尺寸$n$最多 1000 并且$k$最多为 100。这种分离至关重要。 可能的方格数大致为$(n-k+1)^2$，这大约是$10^6$在最坏的情况下。 每个方格最多包含$k^2 \le 10^4$细胞。 每平方的天真的重新计算大约是$10^{10}$操作量太大，持续 2 秒。 任何有效的解决方案都必须将每平方成本降低到恒定或接近恒定的时间。 

一个微妙的问题是必须考虑两种目标颜色。 一个简单的实现，只尝试使正方形均匀`G`或仅至`S`独立地为每个窗口而不重复使用计算存在重复工作和损失效率的风险。 

另一个边缘情况是当$k = n$。 那么就只有一个候选方格，答案就是最少翻转使整个网格均匀。 正确的解决方案不得假设存在多个窗口。 

## 方法

 蛮力方法很简单。 对于每个可能的左上角$k \times k$正方形，我们扫描所有$k^2$细胞并计算有多少个`G`以及有多少个`S`。 该正方形的成本是$\min(\#G, \#S)$。 我们在所有位置上取最小值。 

这是正确的，因为它直接评估每个候选区域而不进行近似。 故障点是运行时。 有$O(n^2)$位置和每个成本$O(k^2)$，导致$O(n^2 k^2)$。 和$n = 1000$和$k = 100$，这是关于$10^{10}$操作，速度太慢。 

关键的观察结果是，我们不需要从头开始重新计算每个方格的计数。 相反，我们可以预先计算网格上的 2D 前缀和，分别计算有多少个`G`单元格出现在任何矩形中。 一旦我们可以查询数量`G`任何细胞中的$k \times k$地区在$O(1)$，数量`S`细胞只是$k^2 - G$，因此成本立即可知。 

这减少了通过恒定时间查询在所有位置上滑动固定大小窗口的问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2 k^2)$|$O(1)$| 太慢了 |
 | 前缀和|$O(n^2)$|$O(n^2)$| 已接受 |

 ## 算法演练

 我们将网格转换为数字矩阵，其中每个`G`变为 1 并且每个`S`变为 0。然后我们构建一个 2D 前缀和数组。 

1.构造前缀和数组`ps`在哪里`ps[i][j]`存储的数量`G`矩形中的单元格由`(0,0)`到`(i-1,j-1)`。 这种转变简化了边界处理。 
2. 对于每个单元格`(i,j)`在原始网格中，计算：$$ps[i+1][j+1] = ps[i+1][j] + ps[i][j+1] - ps[i][j] + (grid[i][j] == 'G')$$这确保每个前缀单元格都计数`G`s 向上和向左恰好一次。 
3. 迭代所有可能的左上角`(i,j)`的一个$k \times k$正方形。 有效范围是`0 ≤ i ≤ n-k`,`0 ≤ j ≤ n-k`。 
4. 对于每个位置，计算`G`使用包含-排除公式在正方形内平铺：$$G = ps[i+k][j+k] - ps[i][j+k] - ps[i+k][j] + ps[i][j]$$这给出了恒定时间内的计数。 
5. 计算`S = k*k - G`。 制作正方形制服的成本为`min(G, S)`。 
6. 跟踪所有位置的最小成本并将其输出。 

### 为什么它有效

 前缀和数组维护一个不变量，即每个条目都存储精确的计数`G`平铺在其相应的子矩形中。 因为矩形和在包含-排除上是线性的，所以任何$k \times k$查询可以分解为四个前缀值，没有重叠或遗漏。 这保证了每个窗口都以正确的计数进行一次评估，并且由于考虑了每个有效的位置，因此找到了全局最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    ps = [[0] * (n + 1) for _ in range(n + 1)]

    for i in range(n):
        row_sum = 0
        for j in range(n):
            val = 1 if grid[i][j] == 'G' else 0
            ps[i+1][j+1] = ps[i][j+1] + ps[i+1][j] - ps[i][j] + val

    best = k * k

    for i in range(n - k + 1):
        for j in range(n - k + 1):
            g = ps[i+k][j+k] - ps[i][j+k] - ps[i+k][j] + ps[i][j]
            s = k * k - g
            best = min(best, g, s)

    print(best)

if __name__ == "__main__":
    solve()
```该解决方案首先仅在金块上构建 2D 前缀和。 这是支持恒定时间矩形查询所需的唯一预处理步骤。 然后嵌套循环枚举所有有效的$k \times k$位置，并且每个查询使用四个前缀访问，因此不需要对正方形进行内部扫描。 

关键的实现细节是基于 1 的索引`ps`，这避免了应用包含-排除时的边界检查。 如果没有这种转变，每个矩形查询都需要在边缘使用条件逻辑，从而增加出错的风险。 

的初始化`best`作为`k*k`之所以有效，是因为如果所有瓷砖都已经是一种颜色，那么这是最坏情况下的重涂成本； 任何有效的配置都不能超过此值。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 |$O(n^2)$| 前缀和构建加上每个的恒定时间评估$(n-k)^2$职位 |

 这些约束允许最多大约一百万次窗口检查，每次检查都在恒定时间内完成。 前缀和构造还运行大约一百万次操作，使总数保持在限制范围内。

 ## 边缘情况

 当$k = 1$，每个单元格都是一个有效的正方形。 该算法使用前缀和独立评估每个单元格，但每个查询都简化为单元格本身。 最小成本始终为 0，因为单个单元已经是均匀的。 

什么时候$k = n$，只有一个窗口。 前缀和查询覆盖整个网格，算法正确计算是否将所有内容转换为`G`或者`S`更便宜。 

对于具有交替模式的网格（例如棋盘），前缀和仍然可以正确聚合每个​​窗口上的计数。 尽管当地的模式差异很大，但每个$k \times k$区域是独立评估的，并且没有窗口会错过贡献，因为包含-排除保证了精确的矩形和。
