 ## 解决方案
 ## 问题理解

 我们从数字 1 开始，并给出一组小型算术运算。 每个运算都是由四个符号之一和一个 1 到 9 的数字组成的对。我们必须按某种顺序排列这些运算，并将它们按顺序应用于当前值，产生最终结果。 任务是选择使最终值尽可能大的排序，并输出重新排序的列表。 

每个运算都以非常直接的方式转换当前值：加法将其增加固定量，减法将其减少固定量，乘法对其进行缩放，除法将其缩小。 关键的困难在于，操作的效果在很大程度上取决于应用它的时间，因为后面的操作会作用于已经修改的值。 

约束 n ≤ 100 意味着我们可以轻松地考虑 O(n²) 推理或基于排序的策略。 任何指数排列都是不可能的，因为 n! 即使 n = 10，增长也太快。该结构表明解决方案是对操作类型进行排序，而不是尝试所有排列。 

缩放和加法运算之间的相互作用产生了一个微妙的问题。 例如，早期应用的小加法在以后的乘法之后可能会变大，而后期应用的相同加法仅贡献其原始值。 同样，如果稍后进行相乘，早期应用的减法可能会放大为更大的损失。 

暴露朴素策略的边缘情况包括混合不同类型的操作，而不考虑它们的位置影响。 例如，如果我们贪婪地先放置所有加法，然后再放置乘法，我们就会错过乘法应该放大加法的事实，因此相反的顺序更好。 另一个失败案例是独立于顺序处理所有操作，这会立即中断。 例如，与`+ 5`和`* 2`, 申请`+ 5`首先得到 7，然后是 14，而反转则得到 2，然后是 7，这是一个更小的结果。 

最后一个极端情况是除法。 由于除法会降低价值，因此提前放置它会永久缩小后续的所有内容。 任意混合除法的简单方法很容易失去早期乘法器或加法所创造的巨大收益。 

## 方法

 暴力方法会尝试 n 张卡的所有排列，模拟每个排序的过程，并跟踪最大结果。 这是正确的，因为它直接评估每个可能的序列。 然而，它需要 n! 评估，每次评估成本为 O(n)，导致时间为 O(n · n!)，即使 n = 10 也是不可行的。 

关键的观察是我们实际上不需要任意排列操作。 每个操作要么缩放当前值，要么移动它，缩放具有更强的结构效应，因为它改变了后续所有内容的大小。 这意味着操作应按其定性效果进行分组：乘数应首先放大所有内容，加法操作应位于中间，除数应放在最后以避免缩小先前的收益。 

在乘数中，较大的因子在早期更有利，因为它们会放大后续的乘数以及随后的所有加性效应。 在除法中，顺序并不重要，因为除法是乘法运算，但它们都应该尽可能延迟。 加法和减法最好放在乘法器之后，这样它们就已经被放大了，并且在除法之前，这样它们就不会被减少。 

这减少了将操作分类为少量类别并在其中适当排序的问题。

 | --- | --- | --- | --- |
 | 蛮力 | O(n·n!) | O(n·n!) | O(n) | 太慢了 |
 | 最佳 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们通过利用每个操作如何与缩放效应相互作用来构建最佳排序。 

1. 将所有卡片分为四组：乘法、除法、加法和减法。 这是必要的，因为每种类型以根本不同的方式影响值，并且任意混合它们会失去结构。 
2. 将乘法卡按数值降序排列。 较大的乘数应尽早应用，因为它们会放大其后的所有内容，包括其他乘数和所有加性贡献。 
3. 将所有乘法卡放在序列的开头。 此时，我们在引入任何附加失真之前已经最大化了缩放效果。 
4. 将中间部分的所有加法卡和减法卡收集在一起。 在此阶段，它们之后不会发生进一步的乘法，因此它们的内部顺序不会影响乘法器随后对它们进行缩放的方式。 
5. 将所有分区卡附加到末尾。 除法会减少整个累加结果，因此延迟它可以尽可能长时间地保留之前获得的所有值。 
6. 输出串联序列：排序后的乘法器，然后是加法和减法，然后是除法。 

重要的结构决策是将缩放操作与加法操作分开。 一旦乘数固定在前面，所有加法效应都变成线性贡献，一旦除法固定在最后，它们就变成最终的均匀缩放。 

### 为什么它有效

 不变的是，在将乘法器放在前面之后，剩余的操作永远不会影响之前和之后的乘法器之间的相对放大，因为在该块之后没有引入进一步的缩放。 类似地，一旦将除法放在末尾，它们就会对整个结果应用统一的因子，并且不会与中间块内的排序交互。

这意味着加法块内的任何重新排列都不能改善或恶化其放大程度，因为放大已经完全由乘法器块决定。 同样的逻辑也适用于部门，它只应用最终的全局收缩。 唯一重要的排序是在乘数中，按降序因子排序可确保最大程度地传播增长。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())
mul = []
add = []
sub = []
div = []

for _ in range(n):
    op, val = input().split()
    val = int(val)
    if op == '*':
        mul.append((val, op, val))
    elif op == '/':
        div.append((val, op, val))
    elif op == '+':
        add.append((val, op, val))
    else:
        sub.append((val, op, val))

# sort multiplications descending
mul.sort(key=lambda x: x[0], reverse=True)

res = []

for _, op, val in mul:
    res.append((op, val))
for _, op, val in add:
    res.append((op, val))
for _, op, val in sub:
    res.append((op, val))
for _, op, val in div:
    res.append((op, val))

for op, val in res:
    print(op, val)
```实现直接反映了理论分解。 乘法组按强度排序并放在第一位，因为它定义了整个表达式的放大结构。 然后添加加法和减法而不进行内部排序，因为一旦乘数固定，它们的相对顺序不会改变最终的缩放模式。 除法被放在最后，因为无论内部安排如何，它们都会统一减少最终结果。 

一个常见的实现陷阱是过度考虑附加块内的排序。 由于该阶段之后不再存在乘法器，因此交换两个加法或两个减法不会改变最终结果。

 ## 边缘情况

 单个操作输入是最简单的边界。 该算法将其放入其类别中并直接输出，这保留了正确性，因为不存在相对的排序决策。 

当所有运算都是乘法时，降序排序可确保较大的乘数首先出现，从而最大化后续乘数的级联效应。 例如，`* 5, * 2, * 1`正确地确保早期扩增。 

当仅存在除法时，任何排序都是有效的，因为所有除法都是乘法交换的。 该算法仍然输出一致的顺序，保持正确性。 

当加法和减法在没有乘法器的情况下混合时，它们的顺序并不重要，因为在分组步骤之后不会引入缩放差异。 该算法的分组确保它们保持在稳定的块中，从而产生有效的最佳排列。
