 ## 解决方案
 ## 问题理解

 我们得到了 Minecraft 世界的垂直横截面，表示为$h \times w$网格。 每个单元格要么是空的，要么是土块，要么是金块。 底部是熔岩，初始配置形成固体堆叠：没有“浮动”方块，因此每个占用的单元格下方都有支撑。 

玩家从最左侧相关区域中最高的非空单元格上方开始，并且可以在顶部附近的空白区域自由移动。 运动并不是主要的困难。 真正的成本来自于修改世界：开采方块和放置泥土方块。 

采矿可以向下或向右进行。 采矿会移除一个方块，并可能导致重力使其上方的任何物体塌陷。 放置泥土会插入一个方块，该方块会下落直到落在支撑物上。 任务是在到达正确边界的同时移除所有金块，最大限度地减少块创建和销毁操作的数量。 

解释这一点的一个有用方法是，一旦存在空白空间，遍历本质上是自由的。 唯一昂贵的操作是那些永久改变结构的操作：删除块或放置新块。 这意味着问题根本上在于必须移除哪些区块才能使所有黄金变得可用，而不是移动计划。 

当金块埋在柱子内时，会出现一个微妙的问题。 如果我们尝试从上方到达它们，我们必须首先移除它们上方的所有方块。 这立即表明成本是由每列内部的垂直依赖性决定的。 

不存在迫使全球电网重组的病态水平障碍。 由于顶行在整个宽度上都是空的，因此玩家可以在所有结构上方水平移动，而无需花费操作。 这消除了大部分“迷宫”方面，并将问题简化为按列推理。 

一个天真的错误是假设只有金块才重要，而答案只是金细胞的数量。 当黄金不在其柱的顶部时，此操作会失败。 

例如，考虑单个列：```
#
G
#
```如果我们尝试只开采黄金单元，我们就无法直接到达它。 我们必须首先清除顶部的污垢块。 正确的成本是 2，而不是 1。 

另一个失败案例是假设我们必须清除整个列。 当黄金仅出现在顶部附近时，这个数字就被夸大了。 

关键的挑战是准确确定我们需要在每列中向下挖掘多远。 

## 方法

 暴力策略会模拟玩家的动作和每一个可能的操作顺序。 国家需要包括整个网格配置，因为每次采矿活动都会导致重力移动区块，并且每次放置都可能改变未来的可访问性。 即使与$h, w \le 100$，配置数量呈爆炸式增长。 每个操作分支成多个可能的下一个状态，并且分支因子很大，因为玩家可以选择不同的列和不同的动作顺序。 这很快就变得不可行。 

重要的结构简化在于，水平运动实际上是自由的，并且始终可以通过顶部的空白空间进行。 这意味着我们可以根据上面的可访问性独立对待每一列。 一旦我们站在柱子上方，唯一的问题是我们必须移除多少块才能移除该柱子中我们关心的所有金子。 

在单个柱内，重力确保移除上方的块不会产生影响成本的复杂的内部重新排列。 该列始终保持连续的堆栈，因此我们可以考虑从上到下的垂直扫描。 

现在关键的观察是我们永远不需要低于一列中最低的黄金块。 它下面的任何污垢或结构都可以保持不变，因为它不会阻止进入其他地方或出口的剩余黄金。 因此，每一列都是独立贡献的：我们只需要清除该列中从上到下到最低黄金的所有内容。 

这将问题简化为计算每列黄金单元中的最高深度（最接近底部），并对上方（包括该点）的所有块求和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 指数|$O(hw)$| 太慢了 |
 | 逐列最低黄金扫描 |$O(hw)$|$O(1)$额外 | 已接受 |

 ## 算法演练

 1. 对于每一列，从上到下扫描并记录包含金块的最低（最深）行索引。 

这很重要，因为低于该点的任何内容都与完成列中的所有黄金无关。 
2. 如果一列不含黄金，则它的成本为零，因为我们永远不需要修改它。 
3. 否则，计算从列顶部到最低黄金位置有多少个单元格。 

每个这样的单元都必须在某个时刻被移除，以便访问和开采该列中的所有黄金。 
4. 将所有列中的值相加以获得最终答案。 

### 为什么它有效

 关键的不变量是，要删除一个金块，必须首先销毁同一列中其上方的所有块，并且另一列中的任何操作都不能消除此要求。 由于顶部的移动是自由的，因此列之间不存在依赖关系。 一旦我们决定将某个列清除到一定深度，该过程就是线性的并且独立于其他列。 因此，选择每列中最深的黄金定义了该列所需的最小挖掘量，并将这些独立的最小值结合起来产生全局最优值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    h, w = map(int, input().split())
    grid = [input().strip() for _ in range(h)]

    ans = 0

    for j in range(w):
        lowest_gold = -1
        for i in range(h):
            if grid[i][j] == 'G':
                lowest_gold = i

        if lowest_gold == -1:
            continue

        # we must clear from top down to lowest_gold inclusive
        ans += lowest_gold + 1

    print(ans)

if __name__ == "__main__":
    solve()
```实现直接遵循这个想法：对于每一列，我们确定最深的黄金单元格。 该列中所需破坏操作的数量正是从上到下到该位置的单元格数量。 

我们完全避免模拟运动，因为问题结构保证了通过空顶部区域的无限制遍历。 唯一有意义的工作是计算垂直依赖性。

 ## 边缘情况

 没有金的列会被干净地处理，因为我们从不更新`lowest_gold`标记，它的贡献为零。 这可以避免错误地强制挖掘不相关的泥土结构。 

黄金位于最顶部的列可确保算法返回 1，因为我们仍然必须执行一次破坏来删除黄金单元本身。 扫描正确识别`lowest_gold = 0`，导致成本1。 

存在多个金块的一列证实了为什么只有最深的一个才重要。 一旦我们挖掘到最低的金块，所有较高的金块都会自动包含在移除路径中，因此超出该深度不会产生额外的立柱成本。
