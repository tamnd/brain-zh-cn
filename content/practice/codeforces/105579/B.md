 ## 解决方案
 ## 问题理解

 我们有一个固定的 10 x 10 棋盘状网格。 每个单元格要么是空的，要么包含一个敌人。 任务是准确选择一个空单元格作为起始位置。 从这个位置开始，我们考虑国际象棋马一步可以到达的所有单元格。 位于这些骑士可到达的牢房之一的每个敌人都被视为被击败。 

目标是确定在所有空单元格中，通过将骑士放置在那里并执行单次骑士攻击可以捕获的最大敌人数量。 

网格尺寸是恒定的并且非常小。 这立即消除了对渐近有效数据结构或搜索技术的任何需求。 任何检查所有候选位置并评估每个位置恒定移动次数的解决方案都足够快。 

当网格中的空单元格很少（可能只有一个）时，就会出现一种微妙的边缘情况。 在这种情况下，我们仍然评估单个细胞，答案完全取决于它的骑士是否移动到敌人身上。 

另一种值得进行合理性检查的情况是，除了孤立的空单元格之外，敌人都密集。 一个天真的错误是假设你总是可以选择一个中心位置，但约束迫使我们评估每个有效的空单元格。 

## 方法

 一种直接的方法是迭代网格中的每个单元格并将每个空单元格视为候选起始位置。 对于每个这样的位置，我们模拟所有八种可能的骑士移动，并计算棋盘内有多少土地并包含敌人。 

这种强力方法已经接近最佳，因为电路板只有 100 个单元。 对于每个单元格，我们最多执行 8 次检查，因此总工作量约为 800 次操作，可以忽略不计。 

这种方法的正确性很简单：考虑每一个可能的起始位置，并且对于每一个位置，我们准确地评估一次移动中可能受到攻击的所有单元格。 由于不存在其他交互，因此动态编程、BFS 或预处理没有任何好处。 

关键的观察结果是，问题纯粹是局部的。 每个位置都是独立的，并且骑士的招式是固定且大小恒定的。 这消除了任何需要更高级优化的结构。

 ## 算法演练

 1. 将 10 x 10 网格作为字符串列表读入内存。 这允许在检查位置时 O(1) 访问任何单元格。 
2. 将八种可能的骑士移动定义为坐标偏移。 这些代表一次性从任何给定单元格出发的所有合法目的地。 
3. 初始化变量`best`为零。 这将存储从任何空起始位置看到的最大敌人数量。 
4. 迭代网格中的每个单元格 (i, j)。 每个单元格都被视为一个潜在的起始位置。 
5. 如果当前单元格不为空，则跳过它。 我们只能将马放置在空的格子上。 
6. 对于每个空单元格，初始化一个计数器`cnt = 0`。 这将计算骑士移动可以到达的敌人数量。 
7. 将所有八个马移动偏移应用于 (i, j)。 对于每个结果位置，首先检查它是否位于 10 x 10 边界内。 
8. 如果目标单元格在边界内并且包含敌人，则递增`cnt`。 
9.检查完所有八个动作后，更新`best = max(best, cnt)`。 
10.处理完所有单元格后，输出`best`。 

### 为什么它有效

 每个空单元格都经过独立且详尽的评估。 对于任何固定的起始位置，骑士一步只能到达最多 8 个单元格，并且算法会精确检查这些单元格。 由于每个空单元格都被视为候选单元格，因此所有有效计数的最大值必须对应于最佳放置。 任何配置都不会被遗漏，因为不同起始位置之间没有交互。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

grid = [input().strip() for _ in range(10)]

moves = [
    (2, 1), (2, -1), (-2, 1), (-2, -1),
    (1, 2), (1, -2), (-1, 2), (-1, -2)
]

best = 0

for i in range(10):
    for j in range(10):
        if grid[i][j] != '.':
            continue

        cnt = 0
        for dx, dy in moves:
            ni, nj = i + dx, j + dy
            if 0 <= ni < 10 and 0 <= nj < 10:
                if grid[ni][nj] == 'h':
                    cnt += 1

        best = max(best, cnt)

print(best)
```代码直接对算法进行编码。 网格存储为字符串以进行快速索引。 移动列表对骑士的几何形状进行一次编码，避免在循环内重复。 边界检查确保我们不会访问无效索引，这是网格模拟中运行时错误的最常见来源。

 ## 边缘情况

 角重配置是安全的，因为每个移动都经过明确的边界检查。 如果骑士被放置在 (0,0)，则大多数偏移会导致网格之外，并且这些偏移会被完全忽略。 该代码永远不会访问无效索引。 

只有一个空单元格的网格也可以正确处理，因为外循环仍然对其进行评估。 如果该单元格没有有效的骑士捕获，则计数器保持为零，这会正确传播到最终答案。 

具有单个孤立空单元的密集敌方区域表明该算法不假设对称或中心放置。 它评估唯一有效的选择并计算其确切的范围。
