 ## 解决方案
 ## 问题理解

 迷宫可以被视为一个$N \times N$网格，其中每个单元格都有唯一的整数温度。 如果牛头怪愿意，可以从任何单元格移动到四个相邻单元格之一。 它的移动规则是确定性的：它总是查看四个邻居并移动到温度严格最小的一个，但前提是该邻居比当前单元更冷。 如果没有相邻的单元比它更冷，它就会永远停在当前位置。 

每个查询都会给出一个起始单元，任务是确定牛头怪在重复应用此规则后最终停止的单元的温度。 

约束的关键含义是$N \le 500$，因此网格最多有 250,000 个单元格，而查询数量可以高达 20,000 个。 每个查询的简单模拟可能会重复遍历多个单元，并且在最坏的情况下，一条路径可能会遍历几乎整个网格。 如果独立重复，这使得任何每个查询的遍历方法都可能太慢。 

一个微妙但重要的特性是所有温度都是不同的。 这消除了选择下一步行动时的歧义，并保证每一次行动都严格降低温度。 

当在没有记忆的情况下独立模拟每个查询时，会出现一个幼稚的错误。 例如，在形状像下降螺旋的网格中，单个查询可以遍历$O(N^2)$细胞。 重复 20,000 个查询会导致不可行$O(N^2 Q)$最坏的情况。 

如果假设局部最小值很少见或在没有完全传播的情况下尝试贪婪的捷径，则会出现另一种失败情况。 考虑一个不是局部最小但通向长下降链的单元。 贪婪的“如果被半径 1 内的更高值包围则尽早停止”规则将错误地终止。 

正确的模型是每个单元确定性地沿着严格递减的路径流动，直到达到其邻域图意义上的局部最小值。 

## 方法

 暴力方法独立处理每个查询。 从给定的单元格开始，我们反复检查其四个邻居，如果温度低于当前单元格，则选择温度最小的一个，然后移动到那里。 由于每次移动都会严格降低温度并且所有值都是不同的，因此路径长度的边界为$N^2$在最坏的情况下。 

这种做法是正确的，因为它直接模拟了规则。 但是，它会重复重新计算相同的下降路径。 许多起点最终流入同一个汇单元，这意味着网格的大部分被重复遍历。 

关键的观察结果是，如果不是局部最小值，则每个单元格都有一个确定性的“下一步”。 这会在单元上产生一个功能图，其中每个节点都指向其最佳邻居。 因为每条边都从较高的温度到较低的温度，所以循环是不可能的，并且每个连接的组件形成一棵以局部最小值为根的树。 因此，每个查询都会问：“这个节点属于哪个根？”

 这将问题转变为在由局部下降指针定义的有向森林中查找连接的组件。 一旦我们计算出每个单元格的最终目的地（根），每个查询就变成$O(1)$。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的强力模拟 |$O(N^2 Q)$最坏的情况|$O(1)$额外 | 太慢了 |
 | 预计算接收器（DFS / DP /联合式传播）|$O(N^2)$|$O(N^2)$| 已接受 |

 ## 算法演练

 我们通过将运动视为严格较冷的邻居的有向边缘来预先计算每个细胞最终的终点。 

1. 对于每个单元，通过扫描四个相邻位置并在严格小于当前单元的单元中选择温度最小的单元来确定其下一个单元。 如果不存在这样的邻居，则将其标记为接收器。 此步骤为每个单元格构建确定性的“下一个指针”。 
2. 创建一个记忆结构，其中每个单元存储其最终接收器（最终达到的局部最小值）。 最初，所有条目都是未计算的。 
3. 对于网格中的每个单元格，运行 DFS 式解析：如果其最终接收器已知，则立即返回。 否则，递归地跟随下一个指针直到到达接收器，然后沿着递归路径传播该结果。 这有效地将每个下降链压缩为指向接收器的直接指针。 
4. 预处理后，只需返回查询单元格存储的接收器值即可回答每个查询。 

我们明确使用记忆而不是简单遍历的原因是许多路径严重重叠。 一旦知道了单元格的目的地，进入该单元格的每条路径都会继承相同的结果。 

### 为什么它有效

 每个单元都有严格的递减路径，因为每次移动都会降低温度并且所有值都是不同的。 这保证了递归在唯一的局部最小值处终止。 记忆化确保每个单元格的接收器只计算一次。 由于“下一个指针”是确定性的，因此该结构形成了一个以局部最小值为根的有向树森林，并且该算法计算该森林中每个节点的根。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

N = int(input())
grid = [list(map(int, input().split())) for _ in range(N)]

# directions
dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

# next pointer: where each cell would move
nxt = [[None] * N for _ in range(N)]
for i in range(N):
    for j in range(N):
        best = (grid[i][j], i, j)
        for di, dj in dirs:
            ni, nj = i + di, j + dj
            if 0 <= ni < N and 0 <= nj < N:
                if grid[ni][nj] < best[0]:
                    best = (grid[ni][nj], ni, nj)
        if best[1] == i and best[2] == j:
            nxt[i][j] = (i, j)
        else:
            nxt[i][j] = (best[1], best[2])

# memoized sink
sink = [[None] * N for _ in range(N)]

def find(i, j):
    if sink[i][j] is not None:
        return sink[i][j]
    ni, nj = nxt[i][j]
    if (ni, nj) == (i, j):
        sink[i][j] = (i, j)
        return (i, j)
    sink[i][j] = find(ni, nj)
    return sink[i][j]

for i in range(N):
    for j in range(N):
        find(i, j)

Q = int(input())
out = []
for _ in range(Q):
    x, y = map(int, input().split())
    x -= 1
    y -= 1
    i, j = sink[x][y]
    out.append(str(grid[i][j]))

print("\n".join(out))
```网格被直接读入内存，对于每个单元，我们通过扫描其四个邻居来计算其最佳传出边缘。 这建立了确定性下降图。 

这`find`函数执行记忆递归。 单元格要么立即解析（如果已经计算），要么递归解析其下一个单元格并存储结果。 这保证了在预处理期间每个单元都被访问一次。 

查询只需索引预先计算的接收器表并返回存储的温度。 输入解析中减一使问题的基于 1 的索引与 Python 的基于 0 的数组对齐。

 ## 边缘情况

 1×1 网格是最简单的情况。 该算法将唯一的单元标记为其自己的接收器，因为不存在邻居。 查询立即返回该单元的温度，无需递归。 

沿路径严格递减的网格测试长链。 该算法在 DFS 记忆过程中仍然会解析每个单元一次，因此即使路径长度很大，也不会发生重新计算。 

许多单元集中到同一局部最小值的情况证明了缓存的重要性。 如果没有记忆，每个查询都会重复地回溯相同的路径，但这里所有中间节点都重用计算出的接收器，因此重复的查询保持恒定的时间。
