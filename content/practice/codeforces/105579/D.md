 ## 解决方案
 ## 问题理解

 我们得到一行$n$索引从 1 到$n$。 该过程通过选择索引为斐波那契数列的所有头寸，从当前行重复形成一个组。 那些选定的人将从队列中移除，剩余的人关闭，并将相同的规则再次应用于缩短的队列。 任务是确定发生多少次这样的回合，直到没有人留下。 

关键的微妙之处在于斐波那契位置始终是根据当前阵容而不是原始指数来定义的。 每次移除后，剩余的人都会从 1 开始重新索引，因此斐波那契集合会根据新的长度重新计算。 

约束条件$n \le 10^{18}$立即排除对个人的任何模拟。 即使迭代每一轮中的所有位置也是不可能的，因为单次传递就已经是$O(n)$。 任何可行的解决方案都必须完全适应数组不断变化的大小，而不是显式元素。 

当出现不明显的极端情况时$n$很小。 例如，$n = 1$由于位置 1 是斐波那契数，因此立即生成一个组。 为了$n = 2$，位置 1 和位置 2 都是斐波那契，所以该过程也一步结束。 一个天真的假设，即只有位置 1 是斐波那契数，就会导致这里的结果不正确。 

另一个微妙的陷阱是假设删除的元素数量在各轮中是恒定的。 事实上，随着斐波那契截止线随着数组的缩小而变化，它也会发生变化。 

## 方法

 直接模拟保留大小的数组$n$，重复构造设置为当前长度的斐波那契索引，删除那些元素，并计算迭代次数。 这在概念上是简单且正确的，因为它完全遵循所描述的过程。 然而，这在计算上是不可行的。 即使每轮删除一小部分元素，最坏的情况仍然从$10^{18}$元素，使得任何针对元素的操作都变得不可能。 

关键的观察是该过程仅取决于当前长度$m$，而不是元素的实际值或身份。 每轮移除元素的数量等于斐波那契数的个数，不超过$m$。 设该值为$F(m)$。 转换变成整数上的纯递归：$$m \rightarrow m - F(m)$$斐波那契数列呈指数增长，因此计数$F(m)$只是$O(\log m)$。 这意味着每一轮都会删除相对于当前大小的对数数量的元素。 虽然这最初可能表明需要进行大量轮次，但斐波那契分布的结构在实践中使状态迅速崩溃，因为截止点非线性向下移动，如下所示：$m$减少。 

因此，问题简化为重复评估$F(m)$并更新$m$直到达到零。

 ## 算法演练

 我们将整个过程简化为仅跟踪当前阵容的长度。 

1. 预先计算所有斐波那契数$10^{18}$，因为这个界限涵盖了所有可能的中间状态。 这允许恒定时间访问有多少个斐波那契数$\le m$。 
2. 初始化$m = n$和一个柜台$ans = 0$。 变量$m$代表每轮结束后剩余的人数。 
3. 在每一轮中，确定有多少个斐波那契数小于或等于$m$。 这可以通过对预先计算的斐波那契列表进行二分搜索来完成。 
4. 减去该计数$m$，因为这些位置在本轮中被删除。 
5. 将答案加 1，因为每轮形成一个组。 
6. 重复直到$m = 0$。 

正确性取决于这样一个事实：每一轮之后，剩余阵容的结构与其规模无关。 唯一重要的是当前范围内存在多少个斐波那契指数，因为这些位置总是被删除的。 

### 为什么它有效

 在任何阶段，系统都由单个整数完整描述$m$。 选择规则仅取决于斐波那契指数的集合$m$，对于给定的情况是固定的$m$。 删除这些头寸会产生一个较小的阵容，其结构不会影响未来超出其新规模的斐波那契成员资格。 这会在整数上创建一个明确定义的确定性递归，确保过程始终一致地发展直到终止。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Precompute Fibonacci numbers up to 1e18
F = [1, 2]
while True:
    nxt = F[-1] + F[-2]
    if nxt > 10**18:
        break
    F.append(nxt)

def count_fib_leq(x):
    # binary search
    lo, hi = 0, len(F)
    while lo < hi:
        mid = (lo + hi) // 2
        if F[mid] <= x:
            lo = mid + 1
        else:
            hi = mid
    return lo

def solve():
    n = int(input())
    m = n
    ans = 0

    while m > 0:
        cnt = count_fib_leq(m)
        m -= cnt
        ans += 1

    print(ans)

if __name__ == "__main__":
    solve()
```斐波那契列表构造一次并可重复用于所有查询。 由于值呈指数级增长，因此列表大小保持很小（大约 90 个元素）。 二分搜索隔离有多少个斐波那契指数落在当前范围内。 

循环仅更新整数状态$m$，避免对个人进行任何模拟。 答案计算此减少步骤应用了多少次，直到耗尽。 

一个常见的实现错误是每次更新后通过重建序列错误地重新计算斐波那契数； 该解决方案通过预计算一次完全避免了这种情况。

 ## 边缘情况

 对于$n = 1$，斐波那契集合仅包含位置 1。该算法计算$F(1) = 1$, 减少$m$为 0，并返回 1 轮。 跟踪确认立即终止。 

为了$n = 2$，斐波那契位置是 1 和 2，所以$F(2) = 2$。 该算法一步删除两者，再次产生一轮。 

为了$n = 3$，斐波那契位置为 1、2 和 3，因此立即删除整个数组。 递归自然地处理这个问题，因为$F(3) = 3$，在一次迭代中折叠流程。
