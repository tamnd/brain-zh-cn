 ## 问题理解

 我们得到了一些相同的砖块，我们想用它们建造一个楼梯。 每个楼梯都有一个正整数高度，并且楼梯必须严格从一级到下一级递增。 我们的目标是最多使用可用的积木，同时最大化我们可以构建的不同步骤的数量。 

关键输出不是楼梯的精确配置，而是在台阶高度必须增加且使用的砖块总数不能超过给定限制的约束下可能的最大台阶数。 

约束结构很重要：如果最多有$n$对于砖块，一个简单的结构可能会尝试任意增加序列，但是如果我们想最大化步骤数量，严格增加高度的要求会立即迫使我们采用最小成本的结构。 

第一个微妙的一点是，最佳结构中不存在“隐藏的选择”。 如果我们决定建造$k$台阶，当楼梯尽可能紧时，砖块的总数会尽可能少：$1, 2, 3, \ldots, k$。 任何偏离此点的行为都会增加总成本并降低可行性。 

值得考虑的边缘情况包括非常小的输入和三角形数的精确边界匹配。 例如，如果$n = 1$，答案显然是1。如果$n = 3$，我们可以构建$1 + 2 = 3$给出 2 个步骤，但是如果$n = 2$，我们只能搭建一级台阶，因为两级台阶需要 3 块砖。 

一个天真的错误是假设任何递增序列都同样有效，或者尝试任意贪婪增量而不认识到最小结构是强制的。 

## 方法

 蛮力的想法很简单：逐级模拟建造楼梯，始终将下一个楼梯的高度至少增加一，同时跟踪已使用了多少块砖。 在每一步中，我们都会尝试添加下一个楼梯，并在超过时停止$n$。 这是正确的，因为它总是为给定的步数构建尽可能最小的楼梯。 

然而，这直接暴露了效率的低下。 在最坏的情况下，如果$n$很大，我们可以迭代到$k \approx \sqrt{2n}$步骤，并且每个步骤都是持续的工作，因此对于典型的约束来说它已经足够有效。 但从概念上讲，这个残酷的过程隐藏了关键结构：第一个的总和$k$整数。 

观察结果是，最佳楼梯必须始终使用尽可能小的递增序列，这将问题简化为寻找最大的$k$这样：$$1 + 2 + 3 + \cdots + k \le n$$这是一个三角和：$$\frac{k(k+1)}{2} \le n$$所以问题就变成了解决一个简单的不等式。 这可以使用二次公式以数学方式完成，也可以通过累积值迭代直至超过限制来完成。 迭代方法更简单，并且避免了浮点精度问题。

 ## 算法演练

 1. 从尽可能小的楼梯高度开始，即 1。这确保我们最大限度地减少任何固定数量楼梯的砖块总使用量。 
2. 维护三个变量：当前楼梯高度、迄今为止使用的砖块总数以及建造的楼梯数量。 
3. 尝试通过每次将高度增加 1 来添加下一个楼梯。 这是被迫的，因为任何较小的增加都会违反严格的顺序。 
4、添加楼梯前，检查添加楼梯所需的砖块数量是否超出限制$n$。 如果有，请立即停止。 
5. 否则，包括楼梯，更新已用砖块的总数，增加楼梯计数，然后继续。 
6. 重复此操作，直到在不超过可用砖块的情况下无法添加更多楼梯。 

为什么它有效：该构造总是构建给定长度的字典顺序最小的严格递增的正整数序列。 对于任何固定数量的楼梯$k$，这个序列使总成本最小化。 由于成本单调增长$k$，第一个值$k$违反约束的位置正是最大有效解的结束位置。 这证明贪婪增量构建对于最优性来说既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())

cnt = 0
add = 1
total = 0

while total + add <= n:
    total += add
    add += 1
    cnt += 1

print(cnt)
```该解决方案维护三个反映构建过程的变量。 变量`add`表示下一个楼梯的高度，从 1 开始，每次迭代增加 1。 变量`total`跟踪到目前为止已经使用了多少块砖，确保我们永远不会超出预算`n`。 变量`cnt`计算已成功建造了多少楼梯。 

循环条件`total + add <= n`至关重要，因为它确保我们仅在剩余砖块能够完全支撑的情况下才使用楼梯。 这可以避免超调并消除回溯的需要。 

这里一个常见的差一风险是更新`total`before checking feasibility; 正确的做法是先检查，再提交。

 ## 边缘情况

 对于$n = 1$，算法开始于`add = 1`并检查`0 + 1 <= 1`，因此它只构建了一个楼梯，然后停止，因为下一个添加将是 2，这已经超出了剩余预算。 这确认了最小输入的正确处理。 

为了$n = 2$，添加第一个楼梯，给出`total = 1`，下一个候选者是`add = 2`。 自从`1 + 2 > 2`，循环立即停止，恰好生成一个楼梯。 这表明该算法并没有高估可行性。 

对于三角形边界，例如$n = 6$，序列完全构建$1 + 2 + 3 = 6$，下一次尝试将是 4，它正确地失败了。 这表明平等是安全处理的，因为条件使用`<=`，允许包含精确配合。
