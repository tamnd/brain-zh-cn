 ## 解决方案
 ## 问题理解

 给定一个整数数组，唯一允许的操作是选择数组中当前最大值的一次出现并将其减一。 有一个限制可以防止在两个连续操作中选择相同的位置。 

该过程持续进行，直到每个元素都变为零。 任务不仅是决定这是否可能，而且在更难的变体中，输出实现它的所选索引的有效序列，当多个可能时，首选字典顺序最小的序列。 

关键的约束是结构性的而不是数字性的。 该操作始终以最大元素为目标，这迫使数组从上到下“展平”。 这意味着大元素的相对顺序决定了我们是否可以在不违反连续索引限制的情况下替代选择。 

从复杂性的角度来看，数组大小足够大，任何重复扫描最大值的模拟都会太慢。 解决方案必须将问题简化为贪婪的结构检查和受控构造，其中每个元素都在聚合阶段而不是单元操作中进行处理。 

当最大和第二大元素相差太大时，就会出现微妙的失败情况。 如果最大值严格大于第二个最大值至少两倍，那么在一次递减之后，相同的索引仍将保持唯一最大值，迫使它立即再次被选择，这违反了约束。 这是根本的障碍。 

例如，考虑`[5, 1]`。 选择第一个元素后，就变成了`4`，仍然严格大于`1`，所以必须重新选择，违反了规则。 

当值相等或恰好相差一时，会出现第二种类型的边缘情况。 在这些情况下，最大值可以在索引之间“切换”，从而实现交替。 

## 方法

 暴力模拟将重复查找最大值，在关系中选择一个索引，将其递减，并跟踪最后选择的索引以强制执行约束。 这在原则上是正确的，但代价高昂，因为每个步骤都会花费线性时间，并且步骤数等于所有值的总和。 在最坏的情况下，这会变成二次或更糟。 

关键的观察是对数组进行排序揭示了过程的结构。 排序好后，我们就可以分层思考：从最大值往下，逐渐“激活”更多的索引。 该过程保持有效的唯一方法是每个新值不低于前一个值太远，否则单个索引将被重复强制。 

这导致了建设性的解释。 我们始终维护与排序值的最大前缀相对应的当前活动索引集。 只要相邻值最多相差 1，我们就可以一次扩展此活动集一个元素，并以尊重“无连续索引”约束的循环方式在其中分配递减量。 

因此，我们不是模拟个体的减量，而是模拟价值水平。 每次我们的价值下降`v`到`v-1`，我们输出分布在当前活动前缀上的操作块。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(ΣA·N)| O(N) | 太慢了 |
 | 排序层构造 | O(N log N) | O(N log N) | O(N) | 已接受 |

 ## 算法演练

 我们首先将每个值与其原始索引配对，并按值以非降序对这些对进行排序。 我们还概念性地添加了一个虚拟对`(0, 0)`简化连续级别之间的差异。 

然后我们从最大的元素向下工作，一次扩展一个元素的活动集。 

1. 对数组进行排序`(value, index)`配对，并附加`(0, 0)`作为哨兵。 这允许干净地处理最终下降到零。 
2. 维护一个变量`i`表示排序元素的当前前缀为“活动”，从最大元素开始。 
3. 对于从较大值级别移动到下一个较小值级别的每一步，收集当前活动前缀中的所有索引。 这些索引准确地代表了现阶段哪些位置可以安全地用于减量。 
4. 对这些索引进行排序，以确保在可能有多个选择时按字典顺序最小输出。 这种顺序确保当我们重复块时，我们总是从最小的可用索引开始。 
5. 计算此级别转换所需的操作数，作为排序数组中连续值之间的差值。 对于每个这样的操作，按顺序输出整个当前活动集，模拟所有活动最大候选者的完整“一轮”递减。 
6. 移动到排序顺序中的下一个元素，将活动集扩展一个索引，然后重复，直到所有值都达到零。 

正确性来自于这样的事实：在任何阶段，所有当前活动的元素都具有相同的有效高度。 排序值的差异准确地告诉我们在新元素变得相关之前必须删除多少个统一层。 

### 为什么它有效

 在任何时刻，活动前缀都准确包含那些在减去先前应用的层后其值当前与最大值相关的索引。 减去层的过程在所有活动索引中都是统一的，因此不会强制任何索引立即重复。 相邻排序值最多相差 1 的条件确保活动最大集永远不会折叠为违反邻接约束的单个刚性元素。 每个层转换都保留最大值在至少一个替代索引之间共享的不变量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = [(0, 0)]
    
    for i in range(1, n + 1):
        arr.append((int(input()), i))
    
    arr.sort()
    
    if n == 1:
        if arr[1][0] == 1:
            print(1)
        else:
            print(-1)
        return
    
    # feasibility check: consecutive differences must be <= 1
    for i in range(1, n + 1):
        if arr[i][0] - arr[i - 1][0] > 1:
            print(-1)
            return
    
    active = []
    res = []
    
    for i in range(n, 0, -1):
        active.append(arr[i][1])
        active.sort()
        
        cnt = arr[i][0] - arr[i - 1][0]
        for _ in range(cnt):
            res.extend(active)
    
    print(*res)

if __name__ == "__main__":
    solve()
```该解决方案首先以成对的方式读取值，以便排序保留原始索引。 哨兵`(0, 0)`允许统一处理最终下降到零。 

可行性检查强制执行连续排序值之间的间隙不超过 1 的结构条件。 如果没有这个，单个元素将需要减少多次才能使任何其他元素符合条件，从而违反了“无连续索引”规则。 

这`active`列表表示当前共享最大“级别”的索引集。 每次迭代都会从已排序的后缀中添加一个索引，反映当我们向下移动时，新元素如何成为最大平台的一部分。 

对于每个级别，我们精确地重复完整的活动集`arr[i][0] - arr[i-1][0]`次。 每次重复对应于所有活动最大值减少一个单位。 

## 工作示例

 考虑数组`[4, 2, 1, 4, 2]`。 

我们建造`(value, index)`配对并排序：`[(1,3), (2,2), (2,5), (4,1), (4,4)]`加哨兵`(0,0)`。 

| 步骤| 活动集| 当前值| 下一个值 | 差异| 输出块|
 | --- | --- | --- | --- | --- | --- |
 | 1 | [4] | 4 | 2 | 2 | [1]重复2次|
 | 2 | [1,4]| 2 | 2 | 0 | 无 |
 | 3 | [1,2,4]| 2 | 1 | 1 | 全套一次|
 | 4 | [1,2,3,4,5] | 1 | 0 | 1 | 全套一次|

 该跟踪显示了索引如何逐渐添加，以及每个值差异如何直接转换为对当前活动集的重复完整扫描。 

现在考虑一个小边缘情况`[2, 1]`。 

排序：`[(1,2), (2,1)]`。 

| 步骤| 活动集| 差异| 输出|
 | --- | --- | --- | --- |
 | 1 | [1] | 1 | [2] |
 | 2 | [1,2]| 1 | [1,2]|

 这证实了层结构中自然会出现交替。

 ## 边缘情况

 关键的边缘情况是当一个元素明显大于所有其他元素时，例如`[10, 1, 1]`。 排序后，10 和 1 之间的差距超过 1，因此算法立即拒绝它。 任何模拟尝试都会重复选择最大的元素并违反不重复约束。 

另一种情况是所有元素都相等时，例如`[3, 3, 3]`。 这里，活动集立即扩展到所有索引，并且每个减量轮循环遍历所有位置，自然避免连续重复并干净地完成。
