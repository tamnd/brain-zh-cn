 ## 解决方案
 ## 问题理解

 我们得到了一组整数。 允许一次移动：选择任意整数移位值并将其添加到数组的每个元素。 经过统一移位后，我们查看结果数组的 MEX，即移位后未出现的最小非负整数。 

任务是选择移位以使该 MEX 变得尽可能大。 

关键约束是相同的移位应用于所有元素，因此元素之间的相对差异不会改变。 只有它们在整数线上的绝对位置一起移动。 

尽管由于移位可以是任何整数，该操作看起来是连续的，但 MEX 是纯粹的组合。 我们只关心移位后的数组是否可以完全包含某个 k 的前缀集 {0, 1, 2, ..., k−1}。 

如果原始数组在测试中的大小高达约 3⋅10^5，则尝试所有移位或模拟每个可能的目标 MEX 的任何解决方案都太慢。 一种天真的方法会考虑每一个可能的转变并重新计算 MEX，这根据实现的不同，成本为 O(n^2) 或更糟，显然超出了限制。 

当数字为负数或间距过大时，就会出现微妙的失败情况。 例如，如果数组为 [-10, 100]，按不同值移位可以将任一元素与 0 对齐，但永远不会创建长连续块，因此 MEX 永远不会增长超过 2。任何假设我们可以从任意值“构建”0..k−1 而不检查连续性的贪婪尝试都会高估答案。 

## 方法

 一个蛮力的想法是尝试每一个可能的移位 x 并计算移位数组的 MEX。 移位后，我们从 0 向上扫描并检查哈希集中的成员资格。 这是可行的，因为 MEX 定义很简单，但速度太慢。 

瓶颈在于移位范围实际上是无界的，即使我们限制它，每次检查的成本也是 O(n)。 对于最多 10^5 个元素，这是不可行的。 

关键的观察结果是，移位不会改变相对顺序或值之间的间隙。 如果移位后我们希望 MEX 为 k，则移位后的数组必须包含从 0 到 k−1 的每个整数。 这意味着原始数组必须包含一组形成连续序列的 k 个整数，因为移位会保留差异。 

因此，我们不考虑转变，而是扭转观点。 我们问：数组中已经形成连续整数块的最大数字组是多少，无论它位于数轴上的哪个位置？ 如果我们找到这样一个长度为 k 的块，我们可以将其移动，使其恰好变为 {0, 1, ..., k−1}，从而实现 MEX 至少 k。 不可能有更大的 MEX，因为任何有效的答案都需要原始数组中存在这样的连续结构。 

因此，问题简化为在一组不同值中找到最长的连续整数。

 | --- | --- | --- | --- |
 | 尝试所有班次+重新计算 MEX | O(n²) 或更糟 | O(n) | 太慢了 |
 | 最长连续整数游程 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 从数组中删除重复项，因为重复的值无助于在连续块中形成新的整数。 MEX 条件仅取决于存在或不存在。 
2. 对剩余值进行排序。 排序使任何潜在的连续结构彼此相邻，使间隙易于检测。 
3. 扫描已排序的数组，同时维护连续整数的当前条纹长度。 
4. 对于每个元素，将其与前一个不同元素进行比较。 如果正好是前一个 + 1，则延长当前的连胜。 否则，将连胜重置为 1。 
5. 跟踪扫描期间看到的最大条纹长度。 这个值就是答案。

每次我们扩展一条条纹时，我们都有效地确认在适当的移位后这些值都可以映射到前缀段 {0..k−1} 中。 当出现间隙时，它就破坏了形成连续前缀的能力，所以我们重新开始。 

### 为什么它有效

 移位后任何有效的 MEX 值 k 都要求存在 k 个不同的原始值，在应用均匀移位后，其成对差异可以恰好为 1。 仅当这些 k 值已经是原始集合中的连续整数时，这才有可能。 因此，每个可实现的 MEX 都准确对应于已排序的唯一数组中的连续块，并且最佳可能的 MEX 对应于最长的此类块。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    a = sorted(set(a))
    
    if not a:
        print(0)
        return
    
    best = 1
    cur = 1
    
    for i in range(1, len(a)):
        if a[i] == a[i - 1] + 1:
            cur += 1
        else:
            cur = 1
        if cur > best:
            best = cur
    
    print(best)

if __name__ == "__main__":
    t = int(input())
    for _ in range(t):
        solve()
```实现直接遵循这个想法。 这`set`删除重复项，以便条纹不会人为地膨胀。 排序可确保连续的整数相邻。 单遍保持当前运行长度。 

一个常见的错误是忘记首先进行重复数据删除，这不会影响这里条纹检测的正确性，但在推理 MEX 可行性时可能会造成混乱。

这表明负值没有帮助，除非它们参与连续的块。

 ## 边缘情况

 对于所有元素都相同的数组，例如 [7, 7, 7]，排序和重复数据删除会将其减少为 [7]。 扫描没有看到连续的扩展，因此结果是 1，这与移位后只有单个值可以对齐到 0 的事实相符。 

对于包含负值和正值的数组，如 [-1, 0, 1]，排序后仍然可以正确检测到连续块。 该算法识别出长度为 3 的条纹，对应于移位 +1 产生 [0,1,2] 的事实，实现 MEX 3。 

对于像 [100, 1, 50] 这样的稀疏数组，没有两个元素是连续的，因此每个条纹的长度都是 1。任何移位只能将一个值对齐到前缀结构中，因此答案正确地变为 1。
