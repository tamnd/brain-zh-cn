 ## 解决方案
 ## 问题理解

 该任务本质上是处理文本标记流并从中提取非常具体的字符统计信息。 我们不执行任何结构解析或转换，而是读取所有输入字符串并仅关注连字符的出现`-`。 每当该字符出现在任何标记中时，它都会为最终答案贡献一个单位。 

您可以将输入视为一系列以空格分隔的字符串，可能跨越多行，而输出是一个整数，表示所有这些字符串组合中出现的连字符数。 除了字符检查之外，没有任何分组、排序和字符串解释。 

尽管官方描述很少，但隐含的约束是 Codeforces 字符串处理任务的标准。 输入大小总共可以轻松达到数十万或数百万个字符。 这立即排除了任何重复的重新扫描或低效的串联策略。 解决方案必须一次性处理输入，对每个字符精确检查一次。 

输入的终止和结构化方式会产生微妙的边缘情况。 由于读取通常是通过基于令牌的输入或完整流读取直到 EOF 来完成，因此当输入长度变化时，假设固定数量的字符串或行的实现可能会失败。 另一个边缘情况是根本不存在任何连字符，其中正确的输出只是零，并且实现必须确保即使没有找到匹配的字符，它们仍然会产生输出。 

## 方法

 蛮力解释很简单。 我们将所有字符串读入内存，然后对于每个字符串，我们迭代每个字符并在遇到任何字符时递增计数器`-`。 这是正确的，因为它直接反映了任务的定义。 这种方法的成本与所有字符串中的字符总数成正比。 如果我们将总长度表示为 N，则运行时间为 O(N)，这在渐近复杂度方面已经是最优的。 

除此之外没有任何有意义的算法优化，因为每个字符必须至少检查一次以确定它是否是连字符。 任何跳过扫描或批处理字符的尝试仍然会隐式触及每个字符，因此理论上的下限保持线性。 

唯一实际的改进是如何读取输入。 有效地使用 Python 的标准输入方法可确保我们不会因重复的函数调用或字符串连接而产生开销。 因此，最佳解决方案不是降低复杂性，而是以最直接、最有效的方式实现线性扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力扫描每个字符串 | O(N) | O(1) 到 O(N) | 已接受 |
 | 单通道流扫描 | O(N) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 从标准输入读取输入，将其作为空格分隔标记的连续流。 这确保我们正确处理换行符和空格的任何混合，而无需对格式做出假设。 
2. 将计数器初始化为零。 该计数器将累积所有标记中遇到的连字符总数。 
3. 迭代从输入获得的每个标记。 每个令牌都被独立处理，但处理方式相同。 
4. 对于每个 token，迭代每个字符并检查它是否等于`-`。 如果是，则将计数器加一。 这种直接字符检查避免了任何不必要的预处理或转换。 
5. 处理完所有令牌后，输出计数器的最终值。 

关键思想是问题完全归结为流输入下的字符计数，因此正确性仅取决于确保没有字符被跳过并且没有字符被重复计算。 

### 为什么它有效

 该算法保持一个简单的不变量：处理 k 个标记后，计数器等于恰好出现在这 k 个标记中的连字符的数量。 每个标记都会被完全扫描一次，并且每个字符都会被精确评估一次。 由于输入被划分为不相交的标记并且所有标记都被覆盖，因此最终计数器必然等于整个输入中连字符的总数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

data = sys.stdin.read().split()

ans = 0
for token in data:
    for ch in token:
        if ch == '-':
            ans += 1

print(ans)
```该解决方案使用一次读取整个输入`sys.stdin.read()`，这避免了每行读取的开销。 拆分为标记可确保统一处理所有空格分隔的字符串。 

嵌套循环结构是有意且最小化的。 外部循环迭代标记，而内部循环检查每个字符。 条件`ch == '-'`是唯一需要的逻辑，计数器累加结果而无需任何辅助存储。 

这里一个常见的实现错误是使用重复`input()`循环调用，这可能会更慢并且有丢失 EOF 条件的风险。 另一个微妙的问题是，如果尝试错误地使用字符串方法进行优化，则会忘记处理令牌中的所有字符。 所提出的解决方案通过坚持直接迭代来避免这两个陷阱。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    data = sys.stdin.read().split()
    ans = 0
    for token in data:
        for ch in token:
            if ch == '-':
                ans += 1
    return str(ans)

 ## 边缘情况

 关键的边缘情况是输入根本不包含标记。 在那种情况下，`sys.stdin.read().split()`返回一个空列表，并且循环永远不会执行。 计数器保持其初始值零，这是正确的，因为没有字符，因此也没有连字符。 

另一种情况是非常大的连续字符串，没有空格。 由于该算法独立处理每个字符，因此它仍然可以正确执行。 例如，输入类似`a---b---c`被视为单个标记，扫描直接计算所有连字符，产生正确的总数。 

最后的边缘情况是不规则的空白，例如标记之间的多个空格或换行符。 由于拆分会统一折叠所有空白，因此标记的分组与计数的正确性无关。
