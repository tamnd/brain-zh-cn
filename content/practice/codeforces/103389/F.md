 ## 解决方案
 ## 问题理解

 给定一个矩形字符网格，任务是确定通过在两个维度上的周期性重复可以将网格压缩多少。 换句话说，我们想要找到最小的子矩形，以便在水平和垂直方向上重复平铺它可以准确地重建整个网格。 

关键的观察结果是，沿行的重复和沿列的重复不会相互干扰。 如果网格每 k 行、每 d 列重复，则基本重复单元是 k × d 块，答案是 k 乘 d。 

输入可以被认为是一个矩阵，其中每一行都是一个字符串。 输出是一个整数，表示最小重复图块的面积。 

从复杂性的角度来看，网格大小通常最多可达总共约 10^5 个单元格或稍多，具体取决于问题变体。 这立即排除了任何比较所有子矩形或通过对所有候选维度进行强力检查周期性的解决方案。 独立尝试所有行和列周期并通过扫描整个网格来验证它们的简单方法在最坏的情况下会降级为三次行为，这太慢了。 

当只有一个维度是周期性的时，会出现微妙的边缘情况。 例如，如果所有行都相同但列不同，则行周期为 1，而列周期可能为全宽。 类似地，如果每列向下相同但行水平不同，则列周期为 1。假设对称或尝试从另一个维度导出一个维度的简单实现在这些不对称网格上将失败。 

另一个边缘情况是完全统一的网格，其中每个单元格都是相同的。 这里行周期和列周期都缩减为 1，答案为 1。任何未正确计算常量数组的最小前缀函数或边界的实现都可能错误地返回完整维度。 

## 方法

 强力解决方案将独立测试每个可能的行周期 k 和列周期 d。 对于每个候选 k，我们将验证行 i 等于行 i 模 k 对于所有 i，对于列也是如此。 每次验证都会扫描整个网格，因此检查单个对 (k, d) 的成本为 O(nm)。 在最坏的情况下，尝试所有 k 到 n 以及所有 d 到 m 会导致 O(nm(n + m)) 行为，这远远超出了可行的限制。 

关键的见解是每个维度的周期性都简化为经典的字符串或序列周期问题。 如果我们将每一行压缩成一个散列，那么网格就变成了行散列的序列。 最小重复行模式正是该序列的最小周期，可以使用 KMP 中的前缀函数或通过计算也是后缀的最长正确前缀来找到。 

同样的想法也适用于列，将每列视为字符序列并对其进行散列，或者更有效地通过预先计算列散列，使每列成为可比较的单元。 一旦两个最小周期被独立计算，网格的基本重复块就是它们的乘积。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(nm(n + m)) | O(1) | O(1) | 太慢了 |
 | 最佳（KMP / 哈希）| O(纳米) | O(纳米) | 已接受 |

 ## 算法演练

 我们对称地对待行和列并独立计算它们的最小周期。 

### 行方向

 1. 将每一行转换为哈希值。 这会将网格变成长度为 n 的一维数组。 

这一步是必要的，因为我们想要在 O(1) 内比较整个行，而不是逐个字符地比较。 
2. 计算行哈希数组上的前缀函数（KMP 失效函数）。

prefix 函数捕获最长的正确前缀，它也是数组中每个前缀的后缀。 
3. 令k为行序列的最小周期长度。 它由 n 减去前缀函数的最后一个值给出。 

这是有效的，因为完整序列的最长边界定义了顶层存在多少重复。 

### 列方向

 1. 通过将每列视为跨所有行的字符序列来构建列哈希。 

每列都成为一个可比较的单位，类似于行的处理方式。 
2. 对列哈希数组应用相同的前缀函数计算。 

这产生了最小的列周期d。 

###最终答案

 1. 将两个独立的周期 k 和 d 相乘，得到最小重复瓷砖的面积。 

### 为什么它有效

 行重复和列重复是独立的约束，因为任何有效的平铺都必须同时遵守这两个约束。 一旦行中存在有效周期，k 行的每个块都是相同的。 在每个这样的块内，列周期性决定每行如何水平压缩。 这两个独立的最小周期的组合产生最小的矩形，可以平铺整个网格而不会引入不匹配。 

其正确性在于网格等式关系将行序列和列序列的等式考虑在内，并且两者均通过标准边界计算得到最优解。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def prefix_function(arr):
    n = len(arr)
    pi = [0] * n
    j = 0
    for i in range(1, n):
        while j > 0 and arr[i] != arr[j]:
            j = pi[j - 1]
        if arr[i] == arr[j]:
            j += 1
            pi[i] = j
    return pi

def solve():
    n, m = map(int, input().split())
    grid = [input().strip() for _ in range(n)]

    row_hash = []
    for r in grid:
        h = 0
        for c in r:
            h = h * 131 + (ord(c) - 96)
        row_hash.append(h)

    pi_row = prefix_function(row_hash)
    row_period = n - pi_row[-1]
    if n % row_period != 0:
        row_period = n

    col_hash = []
    for j in range(m):
        h = 0
        for i in range(n):
            h = h * 131 + (ord(grid[i][j]) - 96)
        col_hash.append(h)

    pi_col = prefix_function(col_hash)
    col_period = m - pi_col[-1]
    if m % col_period != 0:
        col_period = m

    print(row_period * col_period)

if __name__ == "__main__":
    solve()
```该实现首先将每一行压缩为滚动散列，以便行比较变成恒定时间操作。 同样的想法也适用于列。 之后，将 KMP 前缀函数应用于每个一维表示。 

一个微妙的点是最终的整除性检查。 即使前缀函数建议一个候选周期，它也只有在除以总长度时才形成有效的重复。 否则，我们必须回到完整长度，因为不存在更小的精确平铺。

 ## 边缘情况

 完全恒定的网格可以正确处理，因为行和列哈希数组都变成恒定序列，并且前缀函数返回最大边界。 这会将两个周期强制为 1，从而生成正确的最小图块。 

处理仅在一维重复的网格，因为行和列计算是独立的。 例如，每行相同但列不同的网格将产生行周期 1，而列周期保持全宽，反之亦然。 

没有重复的素数大小的网格会触发后备机制，其中前缀函数返回零边界，从而导致两个维度上的全长周期。 这确保我们不会错误地假设不存在的部分结构。
