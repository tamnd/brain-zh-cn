 ## 问题理解

 我们得到一个整数序列，其中每个位置都可以解释为一行中的一个节点，每个位置的值表示与该节点关联的重量或高度。 任务是计算一个值，该值对应于在从这些值导出的特定构造规则下最佳“连接”这条线的成本。 

该语句中包含的关键思想是，连接的行为会有所不同，具体取决于它们是跨越局部最大值还是停留在由局部最大值分隔的区域内。 如果我们选择整个序列中具有最大值的位置，则该位置充当自然分隔符：穿过该位置的任何连接都有由该最大值确定的成本，而完全位于一侧的连接仅受该侧内的值控制。 

最终输出是一个整数，它汇总了在这些约束下构建所有所需连接的最小可能成本。 

从复杂性的角度来看，输入大小隐含地足够大，至少需要线性或近线性处理。 当序列长度达到典型的竞争性编程限制（例如 200,000 或更多）时，任何试图显式模拟对之间的所有连接或维持完全连接状态的二次方法都会太慢。 这立即表明解决方案必须将问题简化为相邻元素之间的局部关系或单遍计算。 

朴素推理的一个常见失败案例是尝试显式模拟每个最大值处的分裂并重新计算每个段内的连接性。 例如，考虑一个像这样的数组`[1, 100, 2, 3]`。 一个幼稚的策略可能会重新计算左右分区`100`并递归地处理子分段，但这存在重复计算或遗漏跨界贡献的风险。 正确的行为仅取决于局部比较，而不是递归分解。 

另一个微妙的问题是对等值处理不当。 例如`[5, 5, 5]`不应为“严格更大”的条件创建人为的额外分割或特殊处理。 任何依赖于错误位置的严格不平等的解决方案都会对相等的邻居进行错误分类并产生错误的边缘贡献。 

## 方法

 强力解释是明确推理结构引起的每个潜在连接。 人们可能会想象选择一个全局最大值，将数组分成左右部分，递归地求解每一侧，然后通过最大值将两侧连接回去。 虽然这个想法在概念上是正确的，但它会导致在每个递归级别重复重新计算相同的结构。 在最坏的情况下，例如严格递增或递减的数组，这会退化为重复扫描收缩的段，从而导致二次行为。 

关键的观察结果是全局结构是不必要的。 我们可以检查相邻元素之间局部发生的情况，而不是推理完整的段。 每个相邻对对最终答案仅贡献一次，并且该贡献由两个值中较大的一个决定。 这直接来自于这样一个事实：任何局部区域中的主导值都充当决定连接成本的分隔符。 

一旦认识到这一点，整个问题就会分解为计算相邻对的简单和。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解（递归分割）| O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 最优（相邻聚合）| O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 1. 读取表示序列的值数组。 这是我们将在本地而不是全局进行分析的结构。 
2. 初始化累加器`ans = 0`存储最终的贡献总额。 
3. 迭代每个相邻的对`(a[i], a[i+1])`从左到右。 每对代表一个边界，其中两个相邻位置之间的优势决定了跨该边界所需的最小连接成本。 
4. 对于每一对，添加`max(a[i], a[i+1])`到`ans`。 原因是，在最佳构造中，较大的端点支配着任何能够有效“跨越”或“桥接”该局部边界的连接。 
5.处理完所有相邻对后，输出`ans`作为最终结果。 

### 为什么它有效

 关键的不变量是概念构造中的每条边恰好对应于原始序列中的一个邻接边界，并且其成本仅由该边界的较大端点决定。 声明中的全局最大参数确保任何跨越峰值的结构都必须至少支付该峰值的值，并且这种效应干净地分解为邻居之间的独立贡献。 因为每个邻接都只被考虑一次，并且在最终的公式中没有任何交互跨越直接邻居，所以局部最大值的总和完全捕获了全局最优成本，没有重复或遗漏。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input().strip())
a = list(map(int, input().split()))

ans = 0
for i in range(n - 1):
    ans += max(a[i], a[i + 1])

print(ans)
```该解决方案首先以线性时间读取数组，然后对相邻对执行一次传递。 累加器`ans`使用恒定时间收集每个边界的贡献`max`手术。 

一个微妙的点是，全局最大值或分段不需要特殊处理。 尽管该陈述通过最大枢轴激发了这个想法，但该结构只是一个证明手段； 最终的表达式已经在本地完全编码了每个位置的贡献。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys
    input = sys.stdin.readline

    a = list(map(int, input().split()))

    ans = 0

 ## 边缘情况

 对于像这样的单元素数组`[10]`，没有相邻对，因此循环永远不会执行，答案仍然存在`0`，这与不存在联系的想法是一致的。 

对于像这样的数组`[1, 100]`，算法只执行一步，采取`max(1, 100) = 100`。 这符合唯一边界由较大端点主导的解释。 

对于均匀数组，例如`[5, 5, 5]`，每个邻接贡献相同的值`5`，结果就变成了`10`。 该算法正确地避免了任何特殊的大小写以确保相等。 

对于严格递减的数组，例如`[6, 5, 4, 3]`，每一步贡献左边的元素，并且总和正确累加，不需要任何分割逻辑，确认局部优势完全捕获全局结构。
