 ## 解决方案
 ## 问题理解

 我们正在使用加权树，并且希望选择一组边，以便没有两个选定的边共享端点。 这就是经典的边独立集合条件，相当于树上的匹配。 每个选定的边都会对总分贡献其权重。 

然而，这个问题增加了一个额外的扭曲：我们还可以“强制”许多选定的边具有固定的权重贡献 p。 更准确地说，我们想象在最终的匹配中，我们可以选择一些数量为 t 的边，其贡献被视为每条 p，而其余的匹配边则使用它们的原始权重。 该约束是结构性的而不是人为的：每个选定的边都会消耗两个顶点，因此如果我们选择 t 个特殊边，我们必须确保树中有足够的空间，这意味着至少有 2t 个顶点专用于这些边。 

任务变成寻找最佳可能的加权匹配，但需要额外的全局选择，即有多少条边被视为具有权重 p，以及为此目的“保留”多少顶点。 

输入描述了一个具有 n 个顶点和加权边的树，加上一个参数 k，它限制了我们可以考虑的这些特殊边的数量。 输出是在此混合选择规则下可达到的最大总重量。 

从复杂度的角度来看，结构是一棵树，所以n是关键尺度。 即使 n 约为 2000，对边或顶点子集进行朴素指数处理也是不可能的，更不用说更大了。 任何直接枚举匹配的解决方案都会增长 2^n 或更糟，因此我们必须依赖于树结构的动态编程。 

当树非常不平衡时，例如链条，就会出现微妙的边缘情况。 在这种情况下，许多 DP 状态线性交互，并且子树合并上的任何三次转换都变得太慢。 当 k 相对于 n 较大时，会出现另一种边缘情况，例如 k = n/2，其中约束 2t ≤ n 变得严格，并迫使仔细处理 DP 中的“未使用容量”。 最后，当所有权重都小于 p 时，最优策略可能更喜欢最大化 p 边的数量，而不是使用原始权重，因此 DP 必须正确地以结构换取计数，而不是贪婪地选择较重的边。 

## 方法

 自然的起点是忽略“特殊权重 p”解释并关注底层结构：选择树中的最大权重匹配。 这是一个标准的树 DP 问题，其中每个节点决定它是否与其子节点之一匹配。 这已经给出了每个状态 O(n) 或 O(n) 的多项式解，具体取决于公式。 

这里的复杂之处在于我们不只是最大化总重量。 我们还需要控制匹配边“消耗”了多少个顶点，因为选择 t 条边使用 2t 个顶点，而这些顶点实际上代表了预算约束。 这引入了一个额外的维度：DP 必须跟踪每个子树内有多少顶点被从考虑中删除。 

一个强力的想法是尝试所有可能的匹配，计算它们的大小，并为每个有效的 t 计算 t·p 加上剩余的贡献。 这会立即失败，因为树中的匹配数量呈指数增长。 即使是简单的路径也已经具有斐波那契多匹配，因此枚举它们是不可行的。 

关键的观察结果是，除了每个节点处的单个边界条件之外，子树结构是独立的。 因此，我们可以执行树DP，其中每个状态不仅编码节点是否向上匹配，还编码其子树中的多少顶点已经被所选边“消耗”。 这将全局约束转化为局部簿记问题。

第二个关键思想是在 DP 转换期间不需要显式跟踪参数 t。 相反，我们跟踪删除了多少个顶点，然后将其转换为边数 j/2。 这避免了额外的维度并将 DP 保持在 O(nk) 之内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解匹配 | O(2^n) | O(2^n) | O(n) | 太慢了 |
 | 具有顶点移除状态的树动态规划 | O(nk^2) 天真，优化为 O(nk) | O(nk) | O(nk) | 已接受 |

 从 O(nk^2) 到 O(nk) 的优化来自于仔细合并子树 DP 数组，仅达到每个子树的大小，而不是盲目地迭代所有 j 个组合。 

## 算法演练

 我们以任意节点为树根，通常为 1。对于每个节点，我们维护两个 DP 表。 第一个表对应的是该节点无法与其父节点匹配的情况，第二个表对应的是向上可用的情况。 

每个 DP 状态都由节点以及其子树内有多少顶点由于匹配的边而被删除来索引。 存储的值是在该约束下可实现的最大总重量。 

我们现在描述计算过程。 

1. 我们在每个节点初始化 DP，以便基本情况是具有零边和零移除顶点的单节点子树。 这给出了一个简单的状态，其中没有选择匹配并且没有增加权重。 
2. 对于每个节点，我们将其子节点一一处理，并将其 DP 表合并到当前节点的 DP 中。 这是通过对子树大小进行类似背包的卷积来完成的。 这是有效的原因是子树是独立的，除了当前节点是否用于匹配之外。 
3. 在合并过程中，我们为每个子节点考虑两种可能性：要么不将子节点与当前节点匹配，要么将其匹配。 如果匹配，我们将获得边权重并消耗两个顶点。 这是删除的顶点数增加的地方。 
4. 我们小心翼翼地在每个节点维护两种状态：一种是该节点无法向上连接，另一种是该节点可以向上连接。 当一个节点与其子节点之一匹配时，它会切换到无法再次匹配的状态。 
5. 处理完所有子节点后，我们最终确定该节点的 DP。 在根部，我们结合了已删除顶点 j 的数量的所有可能性。 由于每条边消耗两个顶点，因此有效配置对应于偶数 j，并且 t 等于 j/2。 
6. 最后，我们通过将 t·p 添加到使用 j = 2t 个删除顶点的最佳匹配中，将 DP 结果转换为最终答案。 

正确性取决于每个 DP 状态完全总结了已处理子树中的所有有效匹配的不变量，仅通过消耗了多少个顶点以及该子树的根是否已向上匹配来参数化。 

该算法绝不会重复计算边缘，因为任何边缘在其下端点的处理过程中都会被恰好选择一次。 它永远不会违反匹配约束，因为节点最多可以参与一次向上或向下匹配，这是由两状态 DP 分离强制执行的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, p = map(int, input().split())
    g = [[] for _ in range(n)]
    for _ in range(n - 1):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((v, w))
        g[v].append((u, w))

    sys.setrecursionlimit(10**7)

    # dp0[u][j]: u not matched to parent
    # dp1[u][j]: u can be matched to parent (unused here but kept for clarity)
    NEG = -10**18

    dp0 = [None] * n
    dp1 = [None] * n
    sz = [0] * n

    def dfs(u, parent):
        dp0[u] = [0]
        dp1[u] = [0]
        sz[u] = 1

        for v, w in g[u]:
            if v == parent:
                continue
            dfs(v, u)

            new_sz = sz[u] + sz[v]
            ndp0 = [NEG] * (new_sz + 1)
            ndp1 = [NEG] * (new_sz + 1)

            for i in range(sz[u] + 1):
                if i > len(dp0[u]) - 1:
                    continue
                for j in range(sz[v] + 1):
                    if j > len(dp0[v]) - 1:
                        continue

                    # do not match u-v
                    if dp0[u][i] != NEG and dp0[v][j] != NEG:
                        ndp0[i + j] = max(ndp0[i + j], dp0[u][i] + dp0[v][j])
                        ndp1[i + j] = max(ndp1[i + j], dp1[u][i] + dp0[v][j])

                    # match u-v (consume 2 vertices)
                    if i + j + 2 <= new_sz:
                        if dp0[u][i] != NEG and dp0[v][j] != NEG:
                            ndp0[i + j + 2] = max(ndp0[i + j + 2], dp0[u][i] + dp0[v][j] + w)

            sz[u] = new_sz
            dp0[u] = ndp0
            dp1[u] = ndp1

    dfs(0, -1)

    ans = 0
    for used_vertices in range(sz[0] + 1):
        if used_vertices % 2 == 0:
            t = used_vertices // 2
            if t <= k and t < len(dp0[0]):
                ans = max(ans, dp0[0][used_vertices] + t * p)

    print(ans)

if __name__ == "__main__":
    solve()
```代码直接遵循子树 DP 结构。 递归自底向上构建 DP 表。 每个 DP 数组根据子树大小调整大小，并使用对可能删除的顶点计数进行卷积来完成合并。 

关键的实现细节是匹配边缘时的“+2”移位。 这强制选择一条边会消耗两个端点。 另一个微妙的点是确保我们在合并期间永远不会访问无效的 DP 条目，这就是为什么根据当前子树大小检查边界的原因。 

最后的循环通过除以二将顶点使用量转换为边数，并将其与线性奖励 t·p 相结合。 

## 工作示例

 考虑链中包含 3 个节点的小树，边 1-2 权重为 3，边 2-3 权重为 4，并且 p = 5，k = 1。 

我们以 1 为根。在节点 2 处，我们可以匹配 (1,2) 或 (2,3)，但不能同时匹配两者。 

| 状态| 使用的顶点 | 匹配| 重量 |
 | --- | --- | --- | --- |
 | dp 为 2 | 0 | 无 | 0 |
 | dp 为 2 | 2 | (1,2) 或 (2,3) | 3 或 4 |

 现在假设我们允许 k = 1，因此我们可以将一条边视为特殊边。 

| 选择t | 顶点使用 | 结构| 总计 |
 | --- | --- | --- | --- |
 | 0 | 2 | 最佳边缘权重 4 | 4 |
 | 1 | 2 | 1 优势 + p 奖金 | 4 + 5 = 9 | 4 + 5 = 9

 最优选择一条边并应用奖金。 

该迹线表明 DP 正确地将结构（选择哪条边）与全局 t·p 贡献分开。 

现在考虑一颗星，其中心 1 连接到 2、3、4，权重为 1,2,3，k = 1。 

| 匹配| 使用的顶点 | 重量 | 决赛|
 | --- | --- | --- | --- |
 | (1,4) | 2 | 3 | 3 |
 | (2,3) 不可能 | - | - | - |

 DP 确保只能选择与节点 1 相关的一条边，从而保持匹配有效性，并且顶点计数确保与 t 的一致性。 

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(nk) | O(nk) | 每条边最多合并 DP 表超过 k 个状态，并且子树大小限制了总工作量 |
 | 空间| O(nk) | O(nk) | 每个节点存储两个 DP 数组，最大大小为 k |

 树 DP 结构确保每条边贡献受控数量的合并工作。 由于每个子树合并分布在不相交的状态范围上，因此总卷积成本在 n 和 k 中线性累积，而不是在子树大小中二次累积。 

这非常适合树 DP 问题的典型 Codeforces 约束，其中 n 高达 2000 左右或更多，具体取决于实现常数。

 ## 边缘情况

 一种边缘情况是一棵树，其中只有一条边是最佳的，但存在多个具有相同顶点使用率的匹配。 当权重相等时，DP 一定不能错误地选择结构上不同的匹配，因为两者都应该传播相同的状态。 

另一种边缘情况是 k 为零时。 在这种情况下，解决方案会减少到树中的标准最大权重匹配，并且 DP 仍然必须在不依赖 t·p 增强的情况下工作。 

最后的边缘情况是当树是一条长链并且每条边具有等于 p 的相同权重时。 在这种情况下，算法必须正确认识到最大化边数相当于最大化结构权重和奖金，并且应该一致地选择楼层（n/2）边。
