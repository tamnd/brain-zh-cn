 ## 解决方案
 ## 问题理解

 我们给定一个固定的字符串`s`长度`n`，以及多个查询。 每个查询给出一个段`[l, r]`里面`s`。 对于每个段，我们被要求构造一个新的字符串`t`尽可能短，同时仍然无法显示为`s[l..r]`。 

子序列检查以标准贪婪方式完成：从位置开始`l`，我们向右扫描以找到第一个出现的`t1`，然后从那里继续扫描以查找`t2`， 等等。 如果我们成功匹配所有字符而不移动过去`r`， 然后`t`是一个子序列。 否则，如果我们被迫超越`r`， 它不是。 

因此，每个查询都要求最短的字符串，迫使这个贪婪的匹配过程在段内“耗尽”。 

字符串字母表很小且固定，这很重要，因为它允许我们预先计算字符的转换。 

这些约束意味着我们无法独立模拟每个查询和每个候选字符串的子序列匹配。 如果我们尝试暴力构建`t`并对其进行测试`[l, r]`，每个测试已经在段中花费线性时间，并且可能的字符串数量随着长度呈指数增长。 甚至试图贪婪地延伸`t`同时检查会降级为类似的东西`O(n * q)`每次尝试，当两者都时太慢`n`和`q`很大。 

当所有字符都在`s[l..r]`是相同的。 在这种情况下，任何`t`包含不同的字符会立即在一步中失败，而重复的相同字符可能会存活更长时间。 总是在本地附加最频繁的字符的幼稚策略可能会忽略最佳移动取决于全局下一个出现次数，而不是段内的频率。 

当某个位置之后根本没有出现字符时，就会出现另一种边缘情况。 在这种情况下，立即选择它会导致后续扫描跳过`r + 1`，这是最快可能的故障，必须正确捕获。 

## 方法

 暴力破解的思路是直接构造最短的字符串`t`通过模拟每个查询内的子序列匹配过程。 从位置开始`l`，我们尝试所有可能的下一个字符，模拟指针移动的距离，并递归地扩展字符串，直到我们强制失败超出`r`。 这在概念上是有效的，因为我们实际上遵循子序列检查的定义。 

问题是每个扩展都需要在字符串中向前扫描以查找下一个出现的位置，以及可能的扩展分支数量（按字母大小排列）。 即使我们贪婪地选择一个分支，我们仍然需要在内部重复扫描`[l, r]`，这会导致在最坏的情况下每个查询出现二次行为。 对于许多查询，这变得不可行。 

关键的观察是一旦我们确定了一个位置`x`子序列匹配指针当前所在的位置，下一个字符的最佳选择`t`不依赖于`[l, r]`。 这仅取决于`x`。 对于每个角色`c`，我们可以计算出下一个出现位置`c`后`x`。 最好的选择是将下一个位置尽可能向右推的角色，因为这可以最大限度地延长子序列的生存时间。 

This means each position`x`具有单一确定性转变`nxt[x]`，这是选择最佳角色后可到达的最远的下一个位置。 一旦构建完成，构建最难匹配字符串的过程就变成了纯粹的步行：从`l`，重复跳转使用`nxt`直到我们通过`r`。 

现在，每个查询都减少为查找需要多少次跳转才能到达大于的位置`r`。 这是函数图上的一个经典的二元提升问题，其中每个节点只有一个传出边。

 ## 算法演练

 我们首先预处理每个字符的下一次出现。 对于每个职位`x`，我们计算如果我们当前处于位置，每个角色将带我们去哪里`x`在子序列匹配过程中。 

然后我们将其转换为单个最佳过渡`nxt[x]`，它代表我们在构建的一步中可以推动匹配指针的最远距离`t`。 

之后，我们对待`nxt`作为函数图并构建二进制提升表，这样我们就可以一次跳过多个步骤。 

对于每个查询`[l, r]`，我们模拟从`l`直到我们超过`r`，使用升降台来计算需要跳跃多少次。 

### 步骤

 1. 预计算`next_pos[x][c]`作为下一个出现的字符`c`位置后`x`。 这是通过从右向左扫描同时保持每个字符最后看到的位置来完成的。 
2. 对于每个位置`x`, 计算`nxt[x]`通过取所有中的最大值`next_pos[x][c]`。 如果字符不存在，则将其下一个位置视为`n + 1`。 这一步定义了最优的贪婪扩展。 
3. 搭建二元升降台`up[k][x]`在哪里`up[0][x] = nxt[x]`更高的级别是由二段跳组成的。 
4. 对于每个查询`[l, r]`，反复尝试从`l`向上的 2 次幂，同时保持在`r`，计算可能的跳跃次数。 
5. 指针第一次超出`r`，跳跃的次数就是答案。 

### 为什么它有效

 在任何位置`x`, 延伸过程`t`只取决于子序列匹配指针接下来将落在哪里。 由于我们总是选择最大化下一个位置的角色，因此每一步对于延迟失败都是局部最优的。 因为子序列过程在位置上是单调的，所以一旦定义了跳转，它就不会依赖于先前的选择或查询间隔，除非通过停止条件`r`。 这将整个问题简化为重复应用确定性函数，二进制提升可以准确处理该函数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    s = input().strip()
    
    ALPHA = 26
    nxt_pos = [[n] * ALPHA for _ in range(n + 2)]
    
    last = [n] * ALPHA
    
    for i in range(n, 0, -1):
        last[ord(s[i - 1]) - 97] = i
        for c in range(ALPHA):
            nxt_pos[i][c] = last[c]
    
    nxt = [0] * (n + 2)
    for i in range(1, n + 1):
        best = 0
        for c in range(ALPHA):
            best = max(best, nxt_pos[i][c])
        nxt[i] = best
    
    nxt[n + 1] = n + 1
    
    LOG = 20
    up = [[n + 1] * (n + 2) for _ in range(LOG)]
    
    for i in range(1, n + 2):
        up[0][i] = nxt[i]
    
    for k in range(1, LOG):
        for i in range(1, n + 2):
            up[k][i] = up[k - 1][up[k - 1][i]]
    
    out = []
    
    for _ in range(q):
        l, r = map(int, input().split())
        pos = l
        steps = 0
        
        for k in reversed(range(LOG)):
            np = up[k][pos]
            if np <= r:
                pos = np
                steps += 1 << k
        
        out.append(str(steps + 1))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实施从构建开始`nxt_pos`，它存储每个位置的每个字符的下一次出现。 这允许稍后进行恒定时间转换查询。 

然后`nxt[i]`通过选择将子序列指针推得最远的字符来计算。 这是从字符的分支选择到单个确定性转换的关键减少。 

二元升降台`up`是建立在这个函数之上的。 每个条目`up[k][i]`代表应用过渡`2^k`时间开始于`i`。 

对于每个查询，我们贪婪地应用可能的最大跳跃而不超过`r`。 成功跳转的次数加一就是所构造字符串的长度。 

## 工作示例

 考虑一个简单的字符串`s = "abac"`和一个查询`[1, 3]`。 

我们首先计算转换。 从位置 1 开始，接下来出现的是`a -> 1`,`b -> 2`,`c -> 4`，所以最好的移动是到 4。从位置 4 开始，一切都超出了，所以它保持在 5。 

| 步骤| 职位| 选择效果| 下一个位置 |
 | --- | --- | --- | --- |
 | 0 | 1 | 开始 | 1 |
 | 1 | 1 | 最佳跳跃| 4 |
 | 2 | 4 | 停止| 5 |

 指针超过`r = 3`跳一跳后，所以答案是1。 

现在考虑`s = "aaaaa"`带查询`[1, 4]`。 

| 步骤| 职位| 下一个位置 |
 | --- | --- | --- |
 | 0 | 1 | 2 |
 | 1 | 2 | 3 |
 | 2 | 3 | 4 |
 | 3 | 4 | 5 |

 我们需要跳跃 4 次才能超过 4，所以答案是 4。 

这些例子表明，该算法对孤立的字符重复并不敏感，而是跟踪全局子序列指针可以推送多远。

 ## 边缘情况

 当片段由单个重复字符组成时，过渡始终向前移动一步。 该算法通过使`nxt[x] = x+1`，因此重复提升可以精确计算剩余的位置直到`r + 1`。 

当某个字符在某个位置之后不再出现时，它的下一次出现将被视为`n + 1`，成为最大候选者`nxt[x]`。 这正确地模拟了子序列匹配中的立即失败。 

什么时候`l == r`，指针在超出段之前只能进行非常少量的转换，并且二进制提升以对数步长正确地对此进行计数，而不是模拟逐个字符的移动。
