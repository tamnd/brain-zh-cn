 ## 解决方案
 ## 问题理解

 我们得到了一条带有固定有序站点列表的巴士路线。 乘客报告了他们在乘坐公共汽车时观察到的一系列停靠站，但存在一个复杂的情况：公共汽车可能沿路线的前进方向或相反方向行驶。 

任务是确定观察到的停车顺序是否与沿路线向前行驶、沿路线反向行驶、两者一致或两者都不一致。 如果两个方向都可以产生完全相同的观察序列，则无法确定方向，并且答案被认为是不明确的。 

更具体地说，将路线视为一个数组`route[0..n-1]`。 观察到的序列是另一个数组`obs[0..k-1]`。 我们正在检查是否`obs`完全按照正向顺序的连续遍历来匹配路由，或者按照反向顺序的完全连续遍历来匹配路由。 

输出是基于这两种可能性的观察分类。 

尽管问题陈述非常短，但关键的隐藏结构是我们没有重建任何复杂的东西，例如路径或图形。 我们仅将一个序列与两个确定性参考序列进行比较：路线本身及其反向版本。 

由于比较是线性的，因此约束隐式表明`n`和`k`足够大以至于二次或重复扫描方法会太慢。 因此，任何解决方案都必须在线性时间内比较序列。 

一些边缘情况值得强调。 

如果观察到的序列与正向和反向路由相同，则只有当路由长度为 0 或 1，或者所有元素都相同时，才会发生这种情况。 在这种情况下，正确答案是“不确定”。 

如果观察到的序列与两个方向都不匹配，即使是部分匹配，如果仅检查前缀相等而不是完整序列相等，则简单的子字符串检查可能会错误地得出匹配结果。 例如，路线`[1,2,3,4]`和观察`[1,2,4]`在任一方向上都无效，即使它与路线的早期部分匹配。 

另一个微妙的情况是反向路由完全匹配但正向路由不匹配。 仅检查一个方向或假设对称性的粗心实现会对此进行错误分类。 

## 方法

 蛮力的想法很简单：生成两种可能的路线遍历，一种按正向顺序，另一种按反向顺序，并将每种遍历与观察到的序列进行比较。 

对于每个方向，我们逐个元素扫描两个数组。 如果所有元素匹配且长度相等，则该方向有效。 这是正确的，因为任何有效的遍历都必须准确地保留顺序和值。 

然而，即使每次比较都是线性的，简单的实现也可能会重复重新计算反向数组或对多个测试用例执行冗余扫描。 如果有很多测试用例或大量输入，则可能会由于重复的内存分配或不必要的工作而变得效率低下。 

关键的观察是我们不需要显式地构建反转数组。 比较时我们可以直接以相反的顺序索引到原始数组。 这减少了开销，并使解决方案在时间和内存方面保持纯线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力（构建+比较两个数组）| O(n + k) | O(n) | 已被接受但效率低下 |
 | 最优（基于索引的比较）| O(n + k) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们假设我们已经有了路线和观察到的顺序来描述该过程。 

### 1. 读取路线和观察顺序

 我们将路线存储在一个数组中，将观察到的序列存储在另一个数组中。 这是必要的，因为我们需要直接索引访问来进行正向和反向比较。 

### 2. 检查前向有效性

 我们首先检查观察到的序列是否与正向顺序的路由匹配。 我们首先比较长度，因为不匹配会立即排除相等性。 然后我们逐个扫描索引并验证相等性。 

这一步是正确的，因为正向遍历与路由数组完全对应，无需修改。 

### 3. 检查反向有效性

 我们重复相同的比较，但不是比较`route[i]`，我们比较`route[n - 1 - i]`。 这模拟了向后行走的路线，而不构建反向数组。 

这可以避免额外的内存分配，同时保持正确性。 

### 4. 合并结果

 如果正向和反向比较都有效，我们输出`Unsure`。 如果只有forward有效，我们输出`Forward`。 如果只有反向有效，我们输出`Reverse`。 否则，该序列无效。 

### 为什么它有效

 正确性来自于以下事实：路线根据方向准确定义了两个可能的确定性停靠序列。 任何有效的观察都必须与这两个序列之一完全匹配。 由于我们在两个方向上逐元素进行比较，因此我们正在根据两个可能的基本事实检查相等性。 除了两个方向产生相同序列的对称情况外，不存在任何歧义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def check_equal(a, b):
    if len(a) != len(b):
        return False
    for i in range(len(a)):
        if a[i] != b[i]:
            return False
    return True

def check_reverse(route, obs):
    n = len(route)
    k = len(obs)
    if n != k:
        return False
    for i in range(n):
        if route[n - 1 - i] != obs[i]:
            return False
    return True

def solve():
    data = list(map(int, input().split()))
    n = data[0]
    route = data[1:1+n]

    k = int(input())
    obs = list(map(int, input().split()))

    forward = check_equal(route, obs)
    backward = check_reverse(route, obs)

    if forward and backward:
        print("Unsure")
    elif forward:
        print("Forward")
    elif backward:
        print("Reverse")
    else:
        print("Impossible")

if __name__ == "__main__":
    solve()
```该实现将正向和反向检查分为清晰的辅助函数。 前向检查是直接相等扫描。 反向检查避免了构造反向数组，而是从路由末尾开始索引，这样更干净，内存效率更高。 

一个微妙的细节是我们总是首先检查长度。 这可以防止意外的部分比较，即前缀可能匹配但完整序列不匹配。

 ## 边缘情况

 一种重要的边缘情况是所有路线元素都相同。 例如，路线`[5,5,5,5]`和观察`[5,5,5,5]`使正向和反向检查同时通过。 算法正确输出`Unsure`因为方向信息本质上会丢失。 

另一个边缘情况是长度不匹配。 如果观察到的序列与路由的长度不同，则正向或反向比较都不会成功。 该算法会在两项检查中尽早拒绝，以防止任何意外的部分匹配。 

最后的边缘情况是最小输入大小。 当路线只有一个元素时，两个方向根据定义是相同的。 该算法将任何匹配的单元素观察正确分类为`Unsure`如果匹配，或者`Impossible`否则。
