 ## 解决方案
 ## 问题理解

 我们给出了一个场景，涉及两个收集器，每个收集器都拥有无限多个“骨段”：对于每个正整数长度，每个收集器都有两个相同长度的相同段。 从这些集合中，每个收集器必须独立选择一对等长的线段，这意味着他们选择一个长度$a$对于第一个收集器和长度$b$对于第二个收藏家。 

然后，这些选定的长度旨在形成具有周长的矩形的边$P$。 由于矩形有两对相等的对边，因此所选的四个线段必须形成长度相同的边$a$和$b$，因此周长条件变为：$$2(a + b) = P$$或等价：$$a + b = \frac{P}{2}$$任务是计算有多少个有序对$(a, b)$的正整数满足这个条件。 

从输入的角度来看，我们只给出整数$P$，并且我们必须输出可以形成有效矩形的两个线段长度的有效有序选择的数量。 

约束条件$P \le 2 \cdot 10^9$立即告诉我们任何解决方案都必须在恒定时间或最坏对数时间内运行。 任何迭代所有可能的段长度的方法$P$会太慢了。 

出现微妙的边缘情况时$P$很奇怪。 在那种情况下，$P/2$不是整数，因此没有一对整数线段长度可以满足矩形条件，并且答案必须为零。 另一种边缘情况发生在$P = 2$, 给予$a + b = 1$，它没有正整数解。 

## 方法

 直接的暴力方法将迭代所有可能的值$a$，并且对于每一个计算$b = \frac{P}{2} - a$，检查是否$b$是积极的。 这是正确的，因为它将目标和的所有有效分解枚举为两个正整数。 然而，这需要$O(P)$最坏情况下的迭代，这是不可能的$P$可以大到$2 \cdot 10^9$。 

关键的观察是问题的结构纯粹是算术：我们正在计算固定整数的有序分解$S = \frac{P}{2}$化为两个正整数。 每个有效对都是通过选择唯一确定的$a$， 和$b = S - a$。 唯一的要求是$a \ge 1$和$b \ge 1$，翻译为：$$1 \le a \le S - 1$$所以每个整数$a$在此范围内恰好产生一个有效的$b$。 这立即给出了答案：$$S - 1$$这将整个问题简化为检查奇偶校验后的常数时间算术计算。

 ## 算法演练

 1.读取整数$P$。 我们只需要这个单一值，因为问题没有额外的结构。 
2. 检查是否$P$很奇怪。 如果是，则立即输出 0，因为$P/2$不是整数，并且不存在有效的矩形边长。 
3. 计算$S = P / 2$。 这表示所需的两条边长之和。 
4. 计算有效有序对的数量：$S - 1$， 因为$a$范围可以从 1 到$S-1$，并且每个选择都唯一决定$b = S - a$。 
5. 输出计算值。 

### 为什么它有效

 正确性来自于所有有效矩形的直接表征。 每个解对应于选择两个正整数$a$和$b$使得它们的总和固定为$S = P/2$。 两边都必须为正的约束强制执行严格的线性范围$a$，并且该范围内的每个值都会产生不同的有效配置。 除了这个算术条件之外，没有隐藏的几何约束，因此计数解决方案精确地减少为计数有界区间内的整数点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    P = int(input().strip())
    
    if P % 2 == 1:
        print(0)
        return
    
    S = P // 2
    # number of positive integer pairs (a, b) with a + b = S
    # a can be 1..S-1
    if S <= 1:
        print(0)
    else:
        print(S - 1)

if __name__ == "__main__":
    solve()
```直接按照推导公式实现。 奇偶校验至关重要，因为跳过它会错误地允许小数和。 边界检查为$S \le 1$处理微小的输入，例如$P = 2$，其中不存在正分解。

 ## 边缘情况

 对于奇数$P$，算法在执行任何除法之前立即拒绝输入。 这很重要，因为尝试计算$P/2$因为如果不进行检查，整数就会默默地失去正确性。 

为了$P = 2$，我们得到$S = 1$，这会产生一个空的有效范围$a$。 该算法通过在以下情况下返回零来显式处理此问题：$S \le 1$，防止出现负数。 

对于非常大的甚至$P$，计算仍然安全，因为Python中所有操作都是64位范围内的简单整数算术，并且不涉及迭代或递归。
