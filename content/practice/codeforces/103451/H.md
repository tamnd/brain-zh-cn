 ## 解决方案
 ## 问题理解

 我们得到一个整数 n，两个玩家在它上交替移动。 在每次移动时，玩家都会查看当前的数字，然后将其减少一或通过楼层划分将其减半。 问题在于，只有当数字为奇数时才允许“减一”移动，而只要数字为正数，则始终允许除以二。 

当数字变为零时游戏结束，无法移动的玩家失败。 由于双方都发挥最佳状态，因此任务是针对 n 的每个起始值确定哪一方获胜。 

该过程本质上是从 0 到 n 状态的确定性博弈，其中每个状态都会转换为更小的状态。 因为每一步都会严格减少数字，所以游戏形成了一个有限有向非循环结构，这立即暗示了对整数的动态规划解释。 

约束允许 n 最多为 10^18，查询最多为 10^3，因此任何逐步模拟转换的解决方案都是不可能的。 即使每个查询的 O(n) 也是远远不可行的，甚至每个步骤的对数模拟也必须非常小心，因为简单的递归仍然存在重复分割奇数的风险。 

围绕奇数会出现一个微妙的极端情况。 例如，从 3 开始，您可以转到 1 或 1，但从 2 开始，您只能转到 1。这意味着许多状态会崩溃为相同的结果，并且很容易错误地认为游戏的行为就像一个简单的减半过程。 对于连续奇数序列，该假设不成立，其中减一可以延迟下降到 2 的幂。 

诸如“如果可能的话，总是除以二”之类的天真贪婪的想法是错误的。 例如，从 3 开始，除以 1，但先减后除可能会导致不同的奇偶校验路径，这对于最佳游戏至关重要。 

## 方法

 暴力破解的想法是将每个整数视为一个游戏状态，并通过检查所有有效的移动来递归地计算它是否获胜。 如果存在向失败状态的移动，则状态 n 获胜。 这直接定义了 n → n/2 和 n → n−1 上的递归（当 n 为奇数时）。 

这对于小值是正确的，但问题是分支深度与大 n 相结合。 即使每一步都减少了 n，在最坏的情况下，单个查询仍然可以探索 θ(n) 个状态，而当 n 达到 10^18 时，这是完全不可行的。 

关键的观察结果是，跃迁的结构在宇称和二的幂方面是高度规则的。 每个偶数都会立即折叠为 n/2，这意味着所有偶数状态的行为就像其一半大小的对应项一样。 唯一真正的分支复杂性集中在奇数，但即使在那里，减一移动也会将它们转换为偶数状态，这又会立即减少。 

这意味着游戏实际上取决于我们可以将因子去掉多少次以及奇偶性变化如何与这个过程相互作用。 一旦我们重复压缩偶数状态，问题就简化为沿着压缩状态空间跟踪行为，其中转换以 n 为对数。 递归深度变为 O(log n)，并且每个状态使用记忆或直接奇偶校验减少最多评估一次。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有状态进行强力递归 | 每个查询 O(n) | O(n) | 太慢了|
 | 奇偶压缩递归/记忆化归约 | 每个查询 O(log n) | O(1) 或 O(log n) | 已接受 |

 ## 算法演练

 我们定义一个函数 win(x) 来判断当前玩家是否有从 x 开始的获胜策略。

1. 如果 x 为 0，则该仓位会亏损，因为不存在任何移动。 
2. 如果 x 是偶数，我们立即将问题简化为 win(x / 2)。 这是有效的，因为除以 2 始终是允许的，并且严格控制在减半之前首先操纵奇偶校验的任何间接序列。 
3. 如果 x 是奇数，我们考虑两种可能的移动。 一步是 x − 1，这会导致偶数，因此立即减少为 win((x − 1) / 2)。 另一个移动是 x / 2（地板除法），这也会导致严格更小的状态。 
4. 如果 x 的至少一个动作导致失败状态，我们就宣布 x 获胜。 否则，x 就输了。 
5. 在压缩转换后，我们以 x 的递增顺序记忆或迭代计算结果，以避免重复评估。 

关键的压缩步骤是每个偶数 x 立即折叠成 x/2，因此我们永远不需要显式地对偶数的长链进行建模。 每个奇数最多生成两次较小幅度的转变，并且这两个转变都会立即进一步减小。 

它起作用的原因来自于一个结构不变量：每个位置相当于通过重复除以二而获得的较小代表，唯一的决定点是奇数是否会强制奇偶校验移位，从而改变其压缩形式的输赢结果。 由于每次移动都会严格减少压缩状态，因此递归定义了有根据的排序，并且不存在循环依赖。 这保证了输赢分类的正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from functools import lru_cache

@lru_cache(None)
def win(x: int) -> int:
    if x == 0:
        return 0

    if x % 2 == 0:
        return win(x // 2)

    move1 = win(x // 2)          # subtract 1 then halve
    move2 = win(x // 2)          # divide directly

    return 1 if (move1 == 0 or move2 == 0) else 0

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        print(win(n))

if __name__ == "__main__":
    solve()
```实现直接遵循递归定义。 记忆化确保每个不同的状态都被评估一次。 关键的微妙之处在于，在第一次操作后，两个奇数转换都会崩溃到相同的简化状态，因此分支因子实际上不会增加​​复杂性。 

立即应用均匀减少，防止重复除法的长链导致递归深度膨胀。 这使得有效状态空间足够小，最多可以进行 10^3 次查询。

 ## 边缘情况

 当 x = 1 时，游戏立即终止。 唯一可用的棋步会将对手逼至 0，因此该位置微不足道。 该算法在基本情况下直接处理此问题并返回 1。 

对于像 x = 2^k 这样的 2 的幂，重复偶约归约会将状态折叠到 1。由于 1 获胜，所以折叠链的奇偶性决定了结果。 递归正确地保留了这一点，因为每个偶数归约都是均匀应用的，直到到达奇数核心。 

对于较大的奇数（例如 x = 10^18 − 1），第一步始终会产生偶数状态，该状态会立即减少。 该算法通过相同的压缩状态正确评估两个分支，确保尽管在奇数节点有明显的选择，但不会发生指数分支。
