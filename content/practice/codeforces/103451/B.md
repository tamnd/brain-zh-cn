 ## 解决方案
 ## 问题理解

 我们得到一个非负整数数组，以及一个将其重复“扩展”为新值的递归操作。 

在基础级别，当级别为零时，该值只是数组中所有元素的总和。 

对于更高级别，我们查看每个连续的子数组。 对于每个子数组，我们在该子数组的上一层计算相同的函数，然后将所有这些结果相加。 重复此 k 次会产生一个增长非常快的数字。 

因此，任务是获取原始数组，应用此子数组前值求和运算 k 次，并输出对 1e9+7 取模的最终值。 

输入大小允许最多 200,000 个元素，k 最多 200,000 个。 这立即排除了任何显式枚举子数组的方法，即使是一次，因为子数组的数量为 O(n^2)。 即使 k = 1 也已经需要聚合所有子数组，并且 k > 1 会加剧这种爆炸。 任何解决方案都必须避免显式地迭代子数组。 

当 k = 0 时，会出现微妙的边缘情况。在这种情况下，递归永远不会开始，答案只是数组的简单总和。 另一个边缘情况是当 n = 1 时。每个子数组只是单个元素，因此所有 k 的答案都变为相同的值，这对于任何派生公式来说都是有用的健全性检查。 

## 方法

 直接模拟通过迭代所有对 (l, r)、对每个子数组求和，然后在 k 层的结果结构上重复此操作来计算 f(A, 1)。 这已经是 O(n^2) 了，执行 k 次实际上就变成了 O(k n^2)，这远远超出了任何可行的限制。 

关键的观察结果是，整个过程是线性的，因为每个元素都独立地对最终答案做出贡献。 递归仅更改每个原始元素的计数次数，而不更改元素之间的交互方式。 因此，我们不是跟踪子数组，而是跟踪单个固定位置在所有级别的贡献。 

在级别 0 处，每个元素 a[i] 对总和的贡献恰好为 1。 

在第 1 层，我们计算有多少个子数组包含索引 i。 即左端点有 i 个选择，右端点有 (n - i + 1) 个选择，因此系数变为 i(n - i + 1)。 

在更高的级别上，相同的结构会重复，但在每个选定的子数组内，元素的位置会发生变化。 这导致了一个产品结构：在每个深度选择左边界和右边界的方法数量完全分开。 在完成两个级别之后，就会出现一种模式，其中二项式系数自然出现。 

这导致在 k 次操作之后 a[i] 的系数的闭合形式，该形式仅取决于其位置和 n。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对每层子数组进行暴力破解 | O(k n^2) | O(k n^2) | O(n^2) | O(n^2) | 太慢了 |
 | 组合贡献公式| O(n + k) | O(n + k) | 已接受 |

 ## 算法演练

 ### 1. 将过程重新解释为线性贡献问题

 我们假设最终答案可以写成原始元素的加权和，其中每个 a[i] 都有一个仅取决于其索引和 k 的系数。 这是有效的，因为每个步骤都是子数组的总和，这与输入值是线性的。 

### 2. 计算基础系数

 当 k = 0 时，每个元素只贡献一次，因此所有 i 的系数均为 1。 

对于 k = 1，每个元素 i 恰好出现在左边界的 i 个选择和右边界的 (n - i + 1) 个选择中，给出系数 i(n - i + 1)。 

这证实了该系数仅取决于有多少子数组覆盖一个位置。 

### 3. 观察 k = 2 时的结构

 对于第二级，我们再次对所有子数组求和，但现在每个子数组都有自己的内部权重。

如果我们固定原始数组中的一个元素 i，我们会选择包含它的子数组 [l, r]。 在该子数组内部，其局部贡献仅取决于其位置 (i - l + 1) 和剩余长度 (r - i + 1)。 对所有有效的 l 和 r 求和可分为两个独立的和，一个用于左侧，一个用于右侧，从而生成三角和的乘积。 

这会产生：

 i(i + 1)/2 乘以 (n - i + 1)(n - i + 2)/2

 ### 4. 推广到 k 个级别

 每增加一层，组合“三角形”的两边都会增加 1。这会产生二项式系数：

 a[i] 的系数 =

 C(i + k - 1, k) × C(n - i + k, k)

 ### 5. 计算最终答案

 预先计算最多 n + k 的阶乘和逆阶乘。 然后将每个索引的 sum a[i] 乘以上述系数。 

### 为什么它有效

 在每个级别，操作仅独立选择左边界和右边界。 这些选择在递归级别上乘法累积。 由于子数组内元素的位置仅取决于左边界将其移动的距离，因此计数分解为两个独立的组合选择，一个从左侧，一个从右侧。 这种独立性正是将嵌套子数组求和转换为二项式系数乘积的原因。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def build_fact(n):
    fact = [1] * (n + 1)
    invfact = [1] * (n + 1)
    for i in range(1, n + 1):
        fact[i] = fact[i - 1] * i % MOD
    invfact[n] = pow(fact[n], MOD - 2, MOD)
    for i in range(n, 0, -1):
        invfact[i - 1] = invfact[i] * i % MOD
    return fact, invfact

def C(n, r, fact, invfact):
    if r < 0 or r > n:
        return 0
    return fact[n] * invfact[r] % MOD * invfact[n - r] % MOD

def main():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))

    maxv = n + k + 5
    fact, invfact = build_fact(maxv)

    ans = 0
    for i in range(n):
        left = C(i + k, k, fact, invfact)
        right = C(n - i - 1 + k, k, fact, invfact)
        ans = (ans + a[i] * left % MOD * right) % MOD

    print(ans)

if __name__ == "__main__":
    main()
```阶乘表构建至 n + k，因为两个二项式项将索引最多移动 k。 每个系数都是使用预先计算的阶乘在恒定时间内计算的。 

关键的实现细节是索引：公式使用从 0 开始的 i，因此左项变为 C(i + k, k)，右项变为 C(n - i - 1 + k, k)。 一个常见的错误是将基于 1 的推导与基于 0 的实现混合在一起，这会错误地移动所有系数。

 ## 边缘情况

 对于 k = 0，该算法简化为对 a[i] 求和，因为两个二项式项的计算结果均为 1。由于对于所有有效 x，C(x, 0) = 1，因此该公式可以清楚地处理此问题。 

对于 n = 1，左项变为 C(k, k) = 1，右项也变为 C(k, k) = 1，因此每个级别都保持该值不变。 这符合每一级只有一个子数组的直觉，因此不会发生组合爆炸。 

对于较大的 k，系数会快速增长，但由于阶乘预计算，系数仍保持有界模 MOD。 模逆的使用确保即使二项式值很大，它们仍然可以在每个元素的恒定时间内计算。
