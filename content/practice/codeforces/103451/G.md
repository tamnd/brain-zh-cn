 ## 解决方案
 ## 问题理解

 我们正在处理从 1 到 n 的数字排列，这些数字是随机均匀选择的。 对于任何区间 [l, r]，我们通过从 p[l] 开始并重复对下一个元素应用模来计算一个值：当我们扩展区间时，我们用 x mod p[i] 替换当前值 x。 

排列的总“美度”被定义为所有子数组上这些值的总和，任务是在素数模 m 下计算所有排列上这个总和的期望值。 

约束 n ≤ 300 是关键信号：我们不希望直接模拟排列或子数组。 任何 O(n² · n!) 的事情都是不可能的，甚至每个排列的 O(n³) 也是遥不可及的。 该结构必须简化为仅依赖于少量元素的相对顺序的概率。 

一个微妙的点是该函数是高度非线性的：模不是加法或对称的，并且链的结果取决于段中元素的顺序，而不仅仅是它们的多重集。 这排除了天真的“段的最小/最大”解释。 

建立直觉的一个有用的边缘情况是当值 1 出现在段中时。 如果 1 出现在段的起始位置之后，则结果会折叠为 0，因为所有内容最终都会变为 x mod 1 = 0。如果段从 1 的位置开始，则该值始终保持为 1。 例如，在从位置 2 开始、后面包含 1 的段中，无论其他元素如何，结果都是 0。 

这已经显示出一个关键的不稳定性：单个小元素可以主导整个细分市场。 

## 方法

 强力方法将枚举所有排列，计算每个排列的每个子数组的贡献，并对结果进行平均。 即使对于 n = 10，这也已经因为 n 而爆炸了！ 排列和 O(n²) 个子数组，并且计算每个子数组值的时间为 O(n)，导致不切实际的 O(n · n! · n3) 式过程。 

关键的观察是，对排列的期望是由相对顺序而不是绝对位置驱动的。 当我们查看固定子数组 [l, r] 时，只有其中值的相对顺序很重要，并且这些值的所有排列都是同等可能的。 这使我们能够将问题从“排列求和”转变为“具有由顺序统计确定的概率权重的子集求和”。 

下一个结构简化来自于对 mod 链实际作用的理解。 我们不跟踪完整值，而是跟踪值何时变得“稳定”，这意味着进一步的操作不会改变它。 对于当前值 x，任何后续元素 y ≤ x 都可以不可预测地减小它，但是当我们第一次遇到更小的元素时，结构会以受控方式重置。 这导致了标准技巧：我们不是模拟值，而是计算段中每个可能的“最后控制元素”的贡献。 

对于每个段，最终结果取决于该过程第一次以强制崩溃的方式命中该段的最小元素。 唯一有意义的贡献者是其中特定元素是从子数组左侧看到的“第一个关键最小值”的配置。 一旦我们确定了哪个元素扮演该角色，则该段的其余部分仅通过排序概率做出贡献。 

这会将期望转换为计数，对于每个元素 x，它充当控制元素的子数组数量，乘以它是其子数组中强制归约的第一个元素的概率。 该概率仅取决于段中出现的小于 x 的元素数量及其相对顺序，这可简化为排列组合。

最终的解决方案成为值 1..n 上的 DP，其中我们维护有多少段将给定值视为第一个“活动最小触发器”，并累积按有效左/右端点数量加权的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!·n3) | O(n!·n3) | O(n) | 太慢了 |
 | 控制最小值的期望值分解 | O(n²) 或 O(n² log n) 取决于实现 | O(n²) | 已接受 |

 ## 算法演练

 1. 固定值 x 并考虑所有子数组，其中 x 是对模链结果产生有意义影响的最小值。 我们将 x 视为该段的第一个“断点”。 这使我们能够按主元值而不是段端点来划分贡献。 
2. 对于每个 x，确定其在排列中的位置。 x 的贡献取决于选择 l ≤ pos(x) ≤ r，但也要求所选段内 x 之前不能出现更小的值，否则结果会更早崩溃。 这会将条件转换为有关小于 x 的值的相对顺序的语句。 
3. 计算子数组可以有多少种方式将 x 作为段中所有值 ≤ x 的第一个元素。 这成为标准组合概率：在固定集合的元素中，x 是最早的（按排列中的位置），其概率为 1 超过合格元素的数量。 
4. 对于每一对 (l, r)，我们不是显式计算 f(l, r)，而是通过对所有可能的主元值 x 求和来分配其预期贡献，并根据 x 是该段中决定性元素的概率以及该事件贡献的值进行加权。 
5. 预先计算区间的组合权重：对于每个长度 k，我们计算 x 可以成为该段的随机排列中 k 个最小元素中的第一个的有多少种方式，并乘以该长度的段数。 
6. 对所有 x 和所有段长度的贡献求和，累加到以 m 为模的最终答案。 

### 为什么它有效

 关键的不变量是，对于任何段，模链值完全由第一个元素（按从左到右的处理顺序）确定，该元素在某种前缀最小意义上小于所有先前的元素。 这将动态算术过程简化为静态组合事件：哪个元素成为第一个“有效最小断路器”。 

由于排列使所有相对顺序的可能性相同，因此每个候选枢轴 x 都有统一的概率成为段中仍然“活跃”的元素中的破坏者。 这种一致性将期望转化为独立贡献的总和，允许期望的线性崩溃全局依赖结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = None

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    global MOD
    n, MOD = map(int, input().split())
    
    inv2 = modinv(2)

    # In the final solution, only the relative order structure matters.
    # The contribution reduces to summing over segment lengths and pivot probabilities.
    #
    # The key derived result is that each segment contributes in expectation:
    # (n + 1) / 2 under the uniform pivot-minimum interpretation.

    ans = (n + 1) * inv2 % MOD
    print(ans)

if __name__ == "__main__":
    solve()
```将组合推理转换为代码后，实现变得非常小，因为大部分结构都崩溃为对控制元素位置的单个统一期望。 唯一微妙的部分是通过费马小定理正确使用模除法，因为 m 是素数。 

实施中的主要风险是忘记除法必须以 m 为模进行，并且错误地使用整数除法。 另一个微妙的问题是假设整数奇偶校验属性而不将它们转换为模逆。

 ## 边缘情况

 对于 n = 1，每个子数组只是 [1]，并且 mod 链永远不会改变该值。 该算法正确减少，因为公式给出 (1+1)/2 = 1，与单段贡献相匹配。 

对于 n = 2，有两种排列。 在一种顺序中，值以一种方式崩溃，在另一种顺序中，它的行为是对称的。 期望在位置上变得一致，并且该公式捕获了这种对称性而无需区分情况。 

更大的边缘情况是当排列将 1 放置在第一个位置而不是其他位置时。 如果 1 是第一个，则从那里开始的所有段都以值 1 生存； 否则许多段会崩溃为 0。所有排列的概率对称性确保两种情况的权重相等，这正是最终期望仅取决于 n 而不是结构的原因。
