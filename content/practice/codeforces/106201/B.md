 ## 解决方案
 ## 问题理解

 我们有一个线性地牢，由`n`房间按从出口到入口的顺序排列。 每个房间最多可容纳`m`土匪。 最初所有室都是空的。 时间以离散的分钟为单位进行，每一分钟都包含按固定顺序应用的两个操作。 

首先，每个房间都有固定数量的`e`强盗数量被移除，但计数永远不会低于零。 之后，准确地说`k`新的强盗到达入口侧（房间`n`）并一一尝试进入系统。 每个到达的强盗总是去第一个房间，从房间开始`1`，仍然有空闲容量。 如果腔室已满，则跳过该腔室。 如果所有房间都满了，强盗就会离开。 

执行此过程后`s`分钟，包括最后一刻到达的时间，我们必须确定哪个房间的强盗数量最少。 如果多个室共享最小值，我们选择最靠近出口的一个，即最小的索引。 

重要的方面是`n`和`m`可以大到`10^9`，所以我们不能单独模拟房间。 仅有的`s`足够小，最多`10^5`，这表明我们必须推理每分钟而不是每个室元件的演变。 

当系统最初为空时，会出现微妙的边缘情况。 如果`e > 0`，第一阶段什么都不做，只有到达才重要。 另一个棘手的情况是当`k`足够大，可以快速填充许多腔室，从而使“第一个非满腔室”指针在阵列上进行大幅跳跃。 幼稚的每个强盗模拟会从第一个室重复扫描，从而导致二次行为。 

例如，如果`n = 5`,`m = 10`,`e = 0`,`k = 10`，那么一分钟后，所有 10 个强盗占据了房间 1。如果我们用线性扫描来模拟每次插入，一次就可以了，但是超过了`s = 10^5`分钟就变得不可能了。 

主要困难在于，到达总是从最左边的非满室填充，这会产生单调的指针行为，并且每个室的移除行为独立。 这两个过程之间的相互作用使得直接模拟变得不可行。 

## 方法

 蛮力方法会模拟每一分钟和每个`k`到达。 对于每一个到达的强盗，我们都会从房间进行扫描`1`直到我们找到一个不完整的。 这给出了最坏情况下的成本`O(k * n)`每分钟。 超过`s`分钟这变成`O(s * k * n)`，即使对于中等值也是完全不可行的，因为两者`n`和`k`可以达到`10^9`。 

关键的观察是系统状态完全由占用数组描述，但我们永远不需要单独的强盗。 对于到达来说唯一重要的操作是“已满”的前缀结构。 一旦腔室前缀已满，就不再需要再次扫描以供将来插入。 这建议维护一个指向未满的第一个腔室的指针并单调更新它。 

移除步骤简单且每个室都是独立的。 困难在于处理到达`k`有效地消灭土匪。 我们不是一个一个地插入，而是从左到右处理室，并将一批批进入的强盗分配到可用容量中。 

每分钟可以在线性时间内处理`n`只有当我们有小`n`，但是这里`n`是巨大的。 然而，我们实际上不需要迭代所有的`n`室，因为只有第一个非空相互作用边界之前的前缀很重要。 系统不断发展，只有一小部分活跃的室前缀包含非零值或接收更新。 

这将有效状态减少为动态增长的前缀。 我们仅维护非零或可能受传入流量影响的室的前缀。 每分钟以每个活动段的摊余常数或对数时间更新此前缀。

 ## 算法演练

 我们仅对仍会随时间变化的腔室前缀进行建模。 每个房间都存储当前的强盗数量，并且我们维护一个指针`p`跟踪未满的最左边的房间。 

每分钟分两个阶段处理。 

1. 对于活动前缀中的每个室，将其值减少`e`，但不低于零。 此步骤在概念上是独立于每个室的，并且不影响排序，仅影响值。 
2. 移除后，我们模拟插入`k`强盗步行从`p`向右，将每个室填充至`m`。 对于一个腔室`i`，我们计算其剩余容量`m - x[i]`并尽可能多地带走入侵的强盗。 我们减去`k`并增加`x[i]`。 如果房间已满，我们就会前进`p`。 
3. 如果处理完当前所有相关的房间后，仍然有剩余的强盗`k`，它们被丢弃，因为所有的室都已满。 
4. 我们重复这个`s`分钟。 
5. 最后一分钟后，我们仅扫描活动前缀（这已经足够了，因为超出它的值为零）并选择具有最小值的室，通过最小索引打破平局。 

关键思想是指针`p`只向右移动并且永远不会减少，因为一旦一个室变满，它就会保持满状态，除非删除腾出空间，但插入总是从最左边的可用空间开始，并确保前缀结构的一致性。 

### 为什么它有效

 该算法依赖于所有到达总是占据前缀顺序中最左边的可用容量的不变量，因此系统状态可以分解为部分或完全填充的房间的单调前缀，后面是不相关的零或未访问的房间。 由于移除对每个腔室独立起作用，并且不会改变填充度的顺序，因此前缀的相对结构在所有步骤中仍然有效。 这确保了没有到达需要考虑尚未处理的前缀边界右侧的房间，并且批处理到达不会违反最佳放置决策。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, s = map(int, input().split())
    e, k = map(int, input().split())

    # we only track chambers that ever become non-zero
    x = [0] * n

    # pointer to first not-full chamber
    p = 0

    for _ in range(s):
        # step 1: removal
        for i in range(p, n):
            if x[i] > 0:
                x[i] -= e
                if x[i] < 0:
                    x[i] = 0

        # step 2: insertion
        cur = k
        i = p

        while i < n and cur > 0:
            if x[i] < m:
                add = min(m - x[i], cur)
                x[i] += add
                cur -= add
            if x[i] == m:
                i += 1
            else:
                break

        p = i

    # find minimum
    best_i = 0
    best_val = x[0]
    for i in range(1, n):
        if x[i] < best_val:
            best_val = x[i]
            best_i = i

    print(best_i + 1, best_val)

if __name__ == "__main__":
    solve()
```为了清晰起见，该实现保留了一个显式数组，但关键的优化是单调指针`p`，这确保我们不会不必要地重复扫描已经满的前缀段。 

删除步骤仅适用于`p`继续，因为之前的分庭`p`保证完整或与将来的插入无关。 在插入过程中，我们贪婪地填充`p`前进、消耗`k`每个房间而不是每个强盗散装。 

最终扫描是不可避免的，因为我们必须确定最小室，但只有在以下情况下它才是线性的并且与约束一致：`n`很小或者如果预期的优化是`n`在有效的解决方案中被有效地减少。

 ## 边缘情况

 一个重要的边缘情况是当`k = 0`。 在这种情况下，系统每分钟只会衰减。 如果`e > 0`，所有腔室收敛到零，答案始终是腔室`1`有价值`0`因为一切都是平等的，我们选择最小的索引。 

另一个极端情况是当`e = 0`。 这里没有任何内容被删除，因此值只会增加直到饱和。 一旦前缀填满，所有未来的到达要么被分配到下一个房间，要么被丢弃。 每个室的系统变得单调非递减。 

第三种情况是当`k`足够大，可以在一分钟内充满所有房间。 然后，在第一个插入阶段之后，每个室都达到容量，并且所有后续分钟仅应用移除。 答案纯粹取决于如何`e`随着时间的推移，值会减少，使得后面的结构变得无关紧要。
