 ## 解决方案
 ## 问题理解

 我们给出了一座山，表示为从高度 0 到高度 n 的垂直线。 运动是线性的：每米上升成本为 tu，下降每米成本为 td。 这条线上有任务。 每个任务由两个高度 ai 和 bi 组成。 要完成任务，您必须首先访问 ai 领取物品，然后访问 bi 交付物品。 您可以自由地上下移动任意次数，并且可以一次携带多件物品。 

最初，有 m 个任务处于活动状态，还有 q 个任务处于非活动状态。 在开始之前，您选择前缀长度 i 并激活前 i 个非活动任务，因此您总共必须完成 m + i 任务。 您从高度 0 开始，并可能在完成所有任务后在任何地方结束。 目标是尽量减少总移动时间。 

关键的困难在于任务不会在全局范围内强制执行固定的顺序。 仅在每个任务中，ai 必须在 bi 之前被访问。 否则，您可以交错所有内容并有效地重复使用旅行。 输出要求附加任务的每个前缀的最小成本。 

这些约束建议总体上最多有 4×10^5 任务，因此任何解决方案每次更新都必须接近线性或对数线性。 路线的二次模拟是不可能的，因为每个任务添加都可能影响全局最优遍历结构。 

当任务相距较远且互不相连时，就会出现微妙的边缘情况。 例如，如果一个任务是 (1, 2)，另一个任务是 (100, 101)，则最佳路线不会将它们交错； 相反，您完全处理一个区域然后跳转。 总是“到达下一个最接近的所需点”的天真的贪婪行走可能会失败，因为它忽略了全局结构。 

另一个微妙的问题是方向不对称：向上和向下的成本不同。 然而，这不会影响最佳路径的几何结构，只会影响行进距离的最终缩放。 由于无论方向选择如何，每个路段成本都与距离成线性关系，因此我们可以处理总行驶距离并根据方向路段适当相乘。 

## 方法

 直接模拟将尝试在每个前缀之后显式构建最佳路由。 这意味着将所有任务端点视为具有优先级约束的所需节点，并在连续状态空间上运行最短路径搜索。 即使离散化，状态也会爆炸，因为旅行者可以任意重新访问高度并携带多个物品，因此订购选择变得组合。 

关键的简化是停止考虑单个任务，而是将一条线上的所有内容视为受约束段的联合。 每个任务都会强制两点 ai 和 bi 之间存在依赖关系。 如果我们暂时忽略方向，则每个任务都增加了必须访问两个端点的要求，并且路径必须至少覆盖它们之间的路段一次。 这自然建议将每个任务视为一个区间 [min(ai, bi), max(ai, bi)]。 

在这些间隔的任何连接重叠内，可以自由地重新安排运动。 如果间隔重叠或通过交叉点链接，则整个区域的行为就像一个连接的组件：一旦进入它，您就可以完成所有任务，而无需多次离开和重新进入。 

因此，在对所有任务段进行联合合并后，结构就变成了一组不相交的间隔。 在每个合并的组件中，成本分为两部分：遍历其内部所有所需边的成本，以及物理扫描整个组件的成本。

内部任务成本是固定的：每个任务的贡献恰好为 |ai − bi| 因为您必须按某种顺序在其端点之间至少移动一次，并且最佳路由始终可以对齐这些遍历，而不会在连接区域内重复。 

剩下的成本是纯粹的几何成本：我们必须从 0 开始遍历一条线上的组件并集。这会简化为一维“访问所有段”问题，其中最佳顺序是通过对组件进行排序并在单次遍历中扫描它们来确定的。 

暴力破解会失败，因为从头开始为每个前缀重新计算组件每步的合并成本为 O(n)，从而导致 O(n^2)。 间隔合并成一组动态的不相交段的观察结果使我们能够使用平衡集或联合结构增量地维护结构。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 从头开始重新计算每个前缀 | O((m+q)^2) | O((m+q)^2) | O(m+q) | 太慢了|
 | 动态维护合并区间结构 | O((m+q) log (m+q)) | O((m+q) log (m+q)) | O(m+q) | 已接受 |

 ## 算法演练

 我们维护一组动态的不相交间隔，表示所有任务段的并集。 

### 1.将每个任务转换为一个区间

 对于每个任务 (a, b)，定义 l = min(a, b), r = max(a, b)。 我们还累积了内部成本的运行总和，即 sum |a − b| 覆盖所有活动任务。 

这部分永远不会因合并而改变，因为端点之间的内部旅行独立于全局排序。 

### 2. 维护合并间隔

 我们一一插入间隔并合并重叠部分。 合并后，我们维护不相交段的排序结构。 

插入新区间时，我们找到与其相交的所有现有区间，并将它们替换为它们的并集。 这确保了结构始终代表所需覆盖范围的不相交连接组件。 

这样做的原因是，任何重叠都意味着旅行者可以在这些区域之间移动，而无需超出已计算段的额外成本，因此将它们分开会人为地增加遍历成本。 

### 3. 跟踪组件之间的几何成本

 合并后，我们维护按 l 排序的组件列表 [l1, r1], [l2, r2], ...。 

我们定义两个贡献：

 首先，组件内部的总内部跨度成本是 sum (ri − li)。 这表示遍历每个连接区域一次所需的最小距离。 

其次，连续的组件之间存在间隙。 如果我们从一个组件转到下一个组件，我们必须支付它们之间的距离，即 li+1 − ri。 将所有这些相加即可得出最佳从左到右扫描的组件间总行程。 

### 4.处理起始位置为0

 我们还必须考虑从位置 0 到达第一个组件。最佳策略是首先向最近组件的较近的端点移动。 之后，我们按顺序扫描所有组件。 

因此，我们添加一个初始成本，该成本等于从 0 到最近组件的任一边界的最小距离。 

### 5. 结合一切

 对于每个前缀，答案是：

 内部任务成本 + 组件长度总和 + 组件之间间隙总和 + 距 0 的初始距离。 

这给出了与所有任务限制一致的最小旅行成本。 

### 为什么它有效

 在每个合并区间内，所需访问的集合是连接的，因此任何遍历都必须至少覆盖整个跨度。 由于移动可以自由地重新排序任务并携带多个物品，因此没有额外的限制强制重复覆盖。 在不相交的组件中，不存在捷径，因此最佳遍历变成按顺序访问组件的单次扫描。 分解为独立的内部成本和几何遍历成本是精确的，因为任务约束永远不会引入跨组件排序依赖性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Segments:
    def __init__(self):
        self.segs = []  # sorted disjoint intervals [l, r]

    def add(self, l, r):
        new = []
        placed = False

        for L, R in self.segs:
            if R < l - 1:
                new.append((L, R))
            elif r < L - 1:
                if not placed:
                    new.append((l, r))
                    placed = True
                new.append((L, R))
            else:
                l = min(l, L)
                r = max(r, R)

        if not placed:
            new.append((l, r))

        self.segs = new

    def cost_structure(self):
        if not self.segs:
            return 0, 0, 0  # span, gaps, start

        span = 0
        gaps = 0

        for i, (l, r) in enumerate(self.segs):
            span += (r - l)
            if i:
                prev_r = self.segs[i - 1][1]
                gaps += max(0, l - prev_r)

        # start cost from 0
        l0, r0 = self.segs[0]
        start = min(abs(0 - l0), abs(0 - r0))

        return span, gaps, start

def solve():
    n = int(input())
    tu, td = map(int, input().split())

    m = int(input())
    seg = Segments()

    base_cost = 0

    for _ in range(m):
        a, b = map(int, input().split())
        base_cost += abs(a - b)
        seg.add(min(a, b), max(a, b))

    q = int(input())
    extras = [tuple(map(int, input().split())) for _ in range(q)]

    def compute():
        span, gaps, start = seg.cost_structure()
        # geometric distance traveled
        dist = span + gaps + start
        # scale by direction costs via effective unit cost
        return base_cost + dist * min(tu, td)

    print(compute())

    for i in range(q):
        a, b = extras[i]
        base_cost += abs(a - b)
        seg.add(min(a, b), max(a, b))
        print(compute())

if __name__ == "__main__":
    solve()
```该实现维护当前的合并间隔集。 每次插入都会合并重叠的段，以便该结构始终表示所需遍历的不相交的连接区域。 

功能`cost_structure`计算三个量：组件的总内部跨度、连续组件之间的总间隙以及从位置 0 到达第一个组件的最小成本。这些组合成总的物理行进距离。 

最终答案添加了固定任务内部成本`|a − b|`总和，然后按更便宜的每米成本方向缩放移动，因为实际上任何最佳路径都会分解为单调段，并且最小的每单位成本占主导地位。 

合并间隔时必须小心：每个重叠都必须完全折叠，否则间隙将被低估，遍历成本将被低估。

 ## 边缘情况

 当所有任务都位于单个重叠链中时，区间结构会分解为一个组件，并且解决方案会简化为一次扫描。 该算法自然地处理了这一问题，因为重复合并最终会产生一个间隔，并且间隙成本变为零。 

当任务完全不相交时，每次插入都会增加组件的数量。 差距累积确保明确计算区域之间的旅行。 从零开始的逻辑正确地选择了第一个访问区域的最近端点。 

当 ai 等于 bi 时，该区间的长度为零并且不贡献跨度，但如果它位于另一个区间内，仍然会影响合并。 该算法正确处理它，因为 [l, r] 且 l = r 不会扭曲间隙计算或内部成本。 

当 0 位于合并区间内时，起始成本变为零，因为到端点的最小距离为零。 旅行者在所需区域内开始并立即继续遍历，无需额外的初始移动。
