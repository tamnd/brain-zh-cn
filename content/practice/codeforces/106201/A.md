 ## 解决方案
 ## 问题理解

 我们被赋予了一个$n \times m$网格，并且我们可以在该网格的边界上选择一个起始单元。 从选定的单元格开始，一个过程每秒扩展到所有四个相邻单元格，就像网格上的广度优先扩展一样。 到达任何单元格所需的时间正是其与所选起始单元格的曼哈顿式网格距离，因为仅允许在四个方向上以单位成本移动。 

任务是选择一个边界单元，使其与网格中任何单元的最大距离最小。 换句话说，我们选择边界上的源来最小化四向移动下最远单元格的距离。 

输出只是任何最佳边界单元的坐标。 

约束条件允许$n, m \le 10^{18}$，这会立即排除任何模拟或网格遍历。 任何解决方案都必须是常数时间，因为即使在任一维度上都是线性时间也是不可能的。 

一种简单的方法是选择一个候选边界单元并计算其到所有单元的最大距离，这已经需要扫描网格。 这是不可行的，更糟糕​​的是，对所有边界单元执行此操作将使成本成倍增加$O(n + m)$，这仍然是不可能的。 

当出现微妙的边缘情况时$n = 1$或者$m = 1$。 在这种退化网格中，每个单元都位于边界上，距离结构塌陷为一条简单的线。 另一种边缘情况是当网格对称时，例如$n = m$，其中存在多个最佳答案并且其中任何一个都是有效的。 

## 方法

 关键的观察结果是，传播过程的行为类似于多源距离问题，我们在边界上恰好选择一个源。 距离边界起点最远的单元格始终是网格内相对于起始边缘“最深”的单元格。 

如果我们将起始单元固定在顶行，则最坏情况距离将由底行主导。 同样，从底行开始会使顶行成为瓶颈。 同样的对称性在水平方向上也成立。 

因此，最佳策略是选择一个边界单元，使其到最远边缘的垂直和水平距离的最大值最小。 直觉上，我们希望尽可能从边界的中心开始，但又限制在边界上。 

这将问题减少到最小化到最远侧的距离。 如果我们从顶行或底行开始，限制距离是垂直高度减 1。如果我们从左列或右列开始，限制距离是水平宽度减 1。为了最小化最坏情况，我们比较$n$和$m$并选择最大范围较小的一侧。 

如果$n \le m$，最好选择行边界中间列上的单元格，因为垂直深度较小或相等。 如果$m < n$，我们选择列边界中间行的单元格。 

更具体地说，最优点总是以较长维度的轴为中心，同时保持在较短维度的边界上。

 ## 算法演练

 我们推断哪个边界单元最小化到网格中任何单元的最大距离。 

1. 比较$n$和$m$。 这决定了最坏情况距离是垂直还是水平扩展占主导地位。 
2.如果$n \le m$，我们优先考虑最小化水平极值。 我们将起始单元格放置在顶部或底部边界上，并选择尽可能靠近中心的列。 最好的专栏是$j = \lceil m/2 \rceil$，因为它最小化了到左右边缘的最大距离。 
3. 案例中$n \le m$，我们可以选择任意一行$i = 1$或者$i = n$，因为两者都是对称的。 我们挑选$i = 1$为了具体。 
4.如果$m < n$，我们反而优先考虑最小化垂直极限。 我们将起始单元格放在左侧或右侧边界上，并选择尽可能靠近中心的行。 最好的行是$i = \lceil n/2 \rceil$。 
5. 在这种情况下，我们修复$j = 1$，因为左边界或右边界是对称的。 
6. 输出选择的坐标。 

### 为什么它有效

 到最远像元的网格距离仅取决于起点距相对边界的距离。 任何远离长尺寸中心的内部偏移都会严格增加到一侧的最大距离，而不会改善另一侧足以补偿。 由于起点必须位于边界上，因此通过以我们未“锚定”的维度为中心来实现最佳权衡。 这确保了所有边界选择中到最远角的最大距离最小化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())

if n <= m:
    i = 1
    j = (m + 1) // 2
else:
    i = (n + 1) // 2
    j = 1

print(i, j)
```该实现直接遵循观察结果，即只有相对大小$n$和$m$事情。 当行数较少或相等时，我们修复边界行并将列居中。 当列数较少时，我们固定边界列并将行居中。 

表达式$(m + 1) // 2$通过正确处理奇数和偶数宽度正确计算中间列，确保两侧的最大偏差最小。

 ## 边缘情况

 当$n = 1$，网格实际上是一条线。 每个像元都位于边界上，但距任何选定起点最远的像元始终是端点。 该算法选择$i = 1$并将列居中，这正确地最小化了到线两端的最大距离。 

什么时候$m = 1$，情况是对称的。 算法选择$j = 1$并将行居中，确保到顶部和底部端点的距离相等。 

什么时候$n = m$，两个分支都是有效的选择。 该算法始终选择顶行和中心列，这是最佳的，因为任何以边界为中心的选择都具有相同的最大距离。
