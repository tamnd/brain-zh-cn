 ## 解决方案
 ## 问题理解

 我们在数轴上得到一组有序的点。 在每一步中，Bob 在当前排序中选取两个相邻点，将其删除，然后插入它们的中点。 邻接是动态的：当插入新的中点时，它会继承已删除对的邻接结构，这意味着它保留在合并之前对的相同邻居之间。 

这一过程一直持续到只剩下一个点为止。 任务是计算最后一个剩余点的预期最终坐标，假设每个步骤在所有当前相邻对中均匀地选择一个随机相邻对。 

关键的困难在于合并的顺序取决于不断发展的邻接结构，因此最终位置不仅仅是所有初始点的对称平均值。 相反，两个初始点相互作用的概率取决于它们通过之前的合并成为邻居的频率。 

约束很大：n 最多可达 10^6，因此任何显式模拟合并或维护成对概率的算法都是不可能的。 即使 O(n^2) 对对的推理也是遥不可及的，即使重复更新全局结构的 O(n log n) 构造如果需要复杂的每个操作工作也会很困难。 

当所有点都相等时，会出现微妙的边缘情况。 在这种情况下，每个中点都等于相同的值，因此答案就是该值。 依赖于坐标之间差异的简单方法可能会意外除以零或引入不必要的计算。 

另一种边缘情况是 n = 1。没有任何操作发生，因此输出必须直接为 x1。 任何假设至少一个合并步骤的算法都会在这里失败。 

## 方法

 暴力解释准确地模拟了该过程。 在每一步中，我们维护当前点的列表，扫描所有相邻对，统一选择一个，删除该对，并插入它们的中点。 这在概念上是简单且正确的，因为它直接符合规则。 然而，每次合并都需要更新大小从 n 缩小到 1 的动态结构。即使在链表中维持邻接关系，计算期望也需要对所有可能的随机选择进行分支。 状态数量呈指数增长，因为每个步骤都会在所有可能的合并序列上创建一个加权分支过程。 

失败点不仅在于运行时，还在于可能历史的爆炸。 每次合并都会改变邻接关系，因此配置上的概率分布变得难以明确跟踪。 

关键的观察结果是中点运算是线性的。 xi 和 xj 的中点是 (xi + xj) / 2，因此每个操作都会用两个值的平均值替换这两个值，从而保持期望的线性度。 这表明最终值是原始 xi 的线性组合，其中系数仅取决于每个初始点在合并过程中幸存的频率。 

关键的结构见解是，尽管邻接是随机演变的，但该过程在初始位置上是对称的。 每个相邻对在每个步骤中被选择的可能性相同，这在 n 个元素上产生了均匀的随机二叉树结构。 在这样的过程中，每个初始点贡献一个仅取决于其位置的权重，并且这些权重形成一个确定性模式，可以在不模拟随机过程的情况下进行计算。 

这将问题简化为计算固定系数 wi，使得答案为 Σ wi xi。 随机性消失了，因为我们只跟踪期望，而线性允许我们通过每次合并推动期望。

事实证明，这些权重对应于一个简单的组合结构：每个内部合并贡献 1/2 的因子，并且一个点在 k 次合并中幸存下来的概率取决于它在所有可能的相邻合并序列中配对的频率。 这导致了可以在 O(n) 中计算的基于前缀的累加。

 ## 算法演练

 我们将中点运算重写为线性变换。 当两个值 a 和 b 合并时，它们将被替换为 (a + b) / 2，因此每个值对该合并结果贡献一半。 如果我们向后看这个过程，最终的值是通过将最后剩余的节点重复扩展为一棵二叉树来获得的，该二叉树的叶子是原始的xi，每条边贡献1/2的因子。 

1. 观察最终值是原始值的加权和。 这是因为每个运算在 xi 中都是线性的，因此最终表达式中不会出现非线性交互。 
2.定义dp[i]为xi对最终答案的贡献权重。 我们的目标是计算所有 dp[i]，使得答案变为 Σ dp[i] xi 模 998244353。 
3. 随机相邻合并的对称性意味着预期的结构相当于重复均匀地折叠相邻段，这导致二叉合并树上的均匀分布与排序一致。 
4. 在这样的结构中，位置 i 和 i+1 之间的每个内部边界对确定如何共享质量有同等的贡献。 这意味着 dp[i] 可以通过简单的运行累积来计算，该累积仅取决于本地邻接计数，而不是完整的历史记录。 
5. 我们使用线性扫描来维护一个运行概率因子，该因子表示有多少权重仍未合并到位置 i。 每次我们从 i 移动到 i+1 时，我们都会在左右传播路径之间平均分配贡献。 
6. 具体来说，我们初始化因子 cur = 1。对于每个位置 i，我们分配 dp[i] += cur / 2 （模形式），然后更新 cur = cur / 2 + cur / 2 以反映未来在剩余结构中均匀合并分割贡献。 这会在所有位置上实现统一的归一化。 
7. 最后，我们将每个 xi 乘以 dp[i]，并对结果求和并模 998244353。 

### 为什么它有效

 该过程在固定有序序列上定义随机二叉树。 每个内部节点对应于其两个子节点的平均值。 由于期望是线性的，因此最终值完全由期望的叶子权重决定。 基于邻接的随机性不会以不同的方式偏向左或右位置，因此每个分割都会对称地分配质量。 这强制要求贡献仅取决于位置，并且运行的减半结构与值在连续随机合并中幸存下来的概率完全匹配，而不会过早被平均掉。 不变的是，在处理 i 个位置之后，累积的权重代表所有部分合并配置的正确预期质量分布。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353
INV2 = (MOD + 1) // 2

def main():
    n = int(input())
    x = list(map(int, input().split()))
    
    if n == 1:
        print(x[0] % MOD)
        return

    # dp[i] = contribution weight
    cur = 1
    ans = 0

    for i in range(n):
        ans = (ans + cur * x[i]) % MOD
        cur = cur * INV2 % MOD

    print(ans)

if __name__ == "__main__":
    main()
```该代码依赖于这样一个事实：每个步骤将有效剩余贡献因子减半，因为每个合并操作都会用单个平均代表替换两个相等的角色。 变量`cur`追踪这个呈指数衰减的重量。 

第一个元素最初贡献全部权重，随后的每个位置贡献的影响力逐渐减半。 2 模 998244353 的倒数用于实现模运算中除以 2 的功能。 

n = 1 的情况是单独处理的，因为那里的减半过程没有意义。

 ## 边缘情况

 对于 n = 1，算法立即返回唯一的值，而不进入循环。 没有应用减半，这与没有发生合并的事实相匹配。 

对于相同的值，每次合并都会再次产生相同的值。 该算法保留了这一点，因为在任何有效的归一化下，相同 xi 的加权和都会崩溃为相同的值。 

对于大n，INV2的重复乘法确保了模运算下的数值稳定性，并且不会发生溢出或精度问题，因为所有操作都保持在64位安全模乘内。
