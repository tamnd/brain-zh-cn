 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 在每一个中，我们在 2D 平面上接收最多 100 个点。 从这些点中，我们可以选择任何子集，并将所选点形成的凸多边形视为其顶点。 如果所选点全部共线或存在少于三个不同的非共线点，则无法形成具有面积的凸多边形。 

任务是找到可以通过这种方式形成的任何凸多边形的最小可能面积。 如果不存在有效的凸多边形，我们输出-1。 

一个关键的几何重新解释会有所帮助：当我们选择点的子集时，我们得到的多边形就是它的凸包。 因此，问题实际上是要求通过选择给定点的某些子集形成的凸包的最小可能面积，约束条件是凸包必须是具有非零面积的适当多边形。 

由于 n 最多为 100 并且最多有 10 个测试用例，因此 O(n3) 甚至 O(n2 log n) 方法很容易实现。 最坏情况下，每个测试用例的点三元组总数最多约为 170 万个，这是可以接受的。 

主要的边缘情况来自简并。 如果所有点都位于一条线上，则每个凸包都会折叠成一个线段，并且答案为 -1。 同样，如果不同点少于三个，则不存在三角形。 

一种微妙的情况是，除了一个远离的点之外，点几乎共线。 仅检查所有点的凸包的简单方法会失败，因为我们可以选择子集，而不一定是所有点。 

例如，考虑：

 输入：

 (0,0)、(1,0)、(2,0)、(0,1)

 所有点的凸包是面积 > 0 的四边形，但形成三角形的最小子集可能是 (0,0)、(1,0)、(0,1)，其中较小。 所以我们必须搜索子集，而不是全局外壳。 

## 方法

 强力解释将枚举点的每个子集，计算其凸包并测量其面积。 有 2^n 个子集，对于每个子集计算凸包至少需要 O(k log k)。 当 n = 100 时，这将变得天文数字般大，约为 2^100 个子集，这是完全不可行的。 

关键的观察是任何具有最小面积的凸多边形不需要超过三个顶点。 如果我们有一个有 k > 3 个顶点的凸多边形，我们总是可以选取任意三个连续的顶点并形成一个面积严格小于或等于多边形面积的三角形。 这是因为将凸多边形分割成三角形总是会划分区域，并且至少有一个三角形必须不大于平均值。 

因此，问题简化为找到由任意三个给定点形成的最小面积非退化三角形。 如果不存在这样的三角形，则答案为-1。 

这将问题转化为所有三元组点的直接三次枚举。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力子集 + 外壳 | O(2^n · n log n) | O(2^n · n log n) | O(n) | 太慢了 |
 | 尝试所有三元组（三角检查）| O(n^3) | O(n^3) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将问题简化为寻找所有三元组中的最小面积三角形。 

1. 迭代索引 i、j、k 的所有三元组。 

我们这样做是因为任何有效的凸多边形都可以简化为至少一个表示具有非零面积的子集的三角形。 
2. 对于每个三元组，使用叉积公式计算有符号面积：

 面积 = |(xj - xi)(yk - yi) - (yj - yi)(xk - xi)| / 2.

我们在比较过程中避免除法，而是存储双倍的区域。 
3. 如果计算面积为零，则这三个点共线，不能形成有效的三角形，因此我们跳过它们。 
4. 否则，用目前找到的最小三角形面积更新答案。 
5. 处理完所有三元组后，如果没有找到有效的三角形，则输出-1。 否则输出最小面积。 

### 为什么它有效

 仅当至少三个选定点不共线时，由点子集形成的任何有效凸多边形才具有非空内部。 在这些选定的点中，取任意三个不共线的点即可得到一个完全包含在多边形三角剖分中的三角形。 由于多边形面积是任何三角剖分中三角形面积的总和，因此至少有一个三角形不大于多边形平均结构，因此我们可以将注意力限制在仅三角形上。 因此，所有子集上的最小凸多边形面积正是所有三元组中的最小非退化三角形面积。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def tri_area2(x1, y1, x2, y2, x3, y3):
    return abs((x2 - x1) * (y3 - y1) - (y2 - y1) * (x3 - x1))

t = int(input())
for _ in range(t):
    n = int(input())
    pts = [tuple(map(int, input().split())) for _ in range(n)]

    INF = 10**30
    ans = INF

    for i in range(n):
        x1, y1 = pts[i]
        for j in range(i + 1, n):
            x2, y2 = pts[j]
            for k in range(j + 1, n):
                x3, y3 = pts[k]
                a2 = tri_area2(x1, y1, x2, y2, x3, y3)
                if a2 > 0:
                    ans = min(ans, a2)

    if ans == INF:
        print(-1)
    else:
        print(ans / 2.0)
```三重循环直接枚举所有可能的三角形。 辅助函数计算两倍的三角形面积，以避免比较期间出现浮点错误。 只有找到最小值后，我们才除以二以获得输出。 

一个微妙的点是使用大的哨兵值而不是零初始化。 由于简并三元组的面积可以为零，因此不能将零用作初始最小值。

 ## 边缘情况

 当所有点都相同或重复时，每个三元组产生零面积。 在这种情况下，算法永远不会更新最小值，因此它会正确返回 -1。 

当所有点共线但不同时，叉积始终为零，因此同样不会发生更新，结果为 -1。 

当恰好存在三个非共线点时，算法恰好认为一个有效三角形并直接返回其面积。 

当存在许多点但只有一个小子集形成最小三角形时，穷举三重搜索仍然可以找到它，因为它不依赖于任何全局结构（如凸包或排序）。
