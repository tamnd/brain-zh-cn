 ## 解决方案
 ## 问题理解

 我们得到了放置在数轴上的多组点。 在每一步中，我们都会重复选取任意两个现有点，删除它们，然后插入它们的中点。 此过程将一直持续到只剩下一个点为止，然后该过程停止。 关键细节是，每一步的所有可能对中的点对都是随机均匀选择的。 

任务不是模拟这个过程，而是计算最后一个剩余点的预期最终位置。 由于答案保证是有理数，因此我们被要求以模块化形式输出它：$p \cdot q^{-1} \bmod 998244353$， 在哪里$\frac{p}{q}$是简化形式的期望。 

输入大小可达$n = 10^6$，这立即排除了任何模拟或基于状态的马尔可夫链方法。 即使单个操作也会将点数减少一个，并且每个操作都将涉及重新计算成对选择，因此在最坏的情况下，简单的模拟至少是二次的。 这远远超出了可行的限度。 相反，解决方案必须依赖于过程的结构不变量。 

一个微妙的点是随机性仅适用于合并的顺序，而不适用于合并的算术结果。 这通常表明最终期望可能以线性方式独立于随机性。 另一个不明显的问题是答案同时取决于所有点，因此关于本地合并的贪婪推理不起作用。 

## 方法

 直接模拟维护点的多重集并重复选择一对随机点，用它们的中点替换它，然后继续。 每次合并都是$O(1)$，但步数是$n-1$。 困难的部分是随机性：每一步都有$\binom{k}{2}$选择，因此枚举转换或对所有序列求平均值是指数级的$n$。 即使是关于中间状态分布的推理也很快变得棘手。 

关键的观察结果是中点运算是线性的。 如果我们写一个合并$x_i$和$x_j$作为$(x_i + x_j)/2$，那么每个操作都是输入的仿射组合。 由于期望是线性的，我们可以跟踪每个原始点对最终结果的贡献。 

我们不跟踪位置，而是跟踪系数。 最初每个点贡献权重$1$。 当两点有系数$a_i$和$a_j$合并，它们产生一个新点，其系数变为$(a_i + a_j)/2$，分发回原始贡献。 选择的随机性不会影响对称性：每个无序对在每个阶段都有相同的可能性，因此每个点都是可交换的。 这种对称性迫使所有原始点对最终结果具有相同的预期贡献。 

由于最终点是所有初始点的凸组合，并且所有系数的期望相等，因此每个点贡献权重$1/n$在期待中。 因此，预期的最终位置只是所有​​初始坐标的平均值。 

这将整个问题简化为计算模算术平均值。

 ## 算法演练

 1. 计算所有输入值的总和。 这会捕获发生任何合并之前所有点的总质量。 由于所有变换都是线性的，跟踪总和足以恢复期望。 
2. 将总和乘以模逆$n$modulo 998244353。这在模算术中执行除法，对应于对所有点的贡献进行平均。 
3. 输出结果作为期望的最终坐标。 

### 为什么它有效

 每次合并都会替换两个值$x_i, x_j$和$(x_i + x_j)/2$。 如果我们归纳地扩展这个过程，最终值始终是原始输入的加权平均值。 权重取决于随机合并树，但由于均匀随机配对的对称性，标记叶子上的每个二叉合并树的可能性相同。 

这种对称性意味着每个初始元素在最终表达式中具有相同的预期权重。 由于总重量之和必须为 1，因此每个预期重量为$1/n$。 因此，最终值的期望正是输入的平均值，与随机过程无关。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    
    s = sum(arr) % MOD
    inv_n = modinv(n)
    print((s * inv_n) % MOD)

if __name__ == "__main__":
    solve()
```实现是最小的，因为核心洞察消除了任何模拟的需要。 唯一重要的步骤是模除$n$，这是使用费马小定理处理的，因为模数是素数。 

一个微妙之处是确保在乘法之前对总和进行模减，以避免不必要的增长。 另一个是$n$本身可能很大，但模逆计算仍然存在$O(\log MOD)$，在约束条件下可以忽略不计。

 ## 边缘情况

 单元素输入，例如`n = 1`是唯一不发生合并的情况。 该算法很自然地处理这个问题，因为总和等于元素本身，并且 1 的倒数为 1，因此输出保持不变。 

全相等数组在此过程中表现出确定性，因为每个中点都与输入相同。 该算法反映了这一点，因为平均值等于相同的值，并且没有随机性影响结果。 

值接近的大输入$998244353$求和时需要模归约。 该实现立即减少总和，防止溢出并确保整个计算过程中模运算的正确性。
