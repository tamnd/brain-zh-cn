 ## 解决方案
 ## 问题理解

 我们被要求构建一种特殊的大小排列`n`。 这里的排列意味着我们排列数字`1`到`n`每个恰好一次。 

不同的是，排列必须表现得像没有固定点的对合。 换句话说，如果我们查看位置的值`j`，称之为`a[j]`，然后再次应用相同的映射让我们回到：`a[a[j]] = j`。 同时，不允许任何位置映射到自身，因此`a[j] != j`对于每一个`j`。 

这一结构强制每个索引与另一个索引精确配对。 如果`a[x] = y`，然后自动`a[y] = x`，并且两者都不是`x`也不`y`可以相等。 因此，排列完全由不相交的 2 循环组成。 

输入尺寸`n`可以大到`100000`，因此任何解决方案都必须在线性时间内运行。 二次或偶数$O(n \log n)$构造是好的，但是任何涉及重复搜索或匹配的事情都是不必要的并且有风险。 

出现关键边缘情况时`n`很奇怪。 如果每个元素都必须属于一对，则奇数个元素会使一个索引不成对。 这使得条件无法满足。 

例如，当`n = 3`，我们需要像这样的对`(1, 2)`和`(3, ?)`，但是已经没有伙伴了`3`。 任何尝试都会迫使固定点或打破双射要求。 

因此，真正的困难不是在可能的情况下构造排列，而是在结构不可能时识别。 

## 方法

 一个天真的尝试是尝试通过跟踪哪些位置已被使用来构建排列。 我们可以迭代索引`1`到`n`，并且对于每个未使用的索引`i`, 搜索另一个未使用的索引`j`并设置`a[i] = j`,`a[j] = i`。 这保证了正确性，因为我们明确执行条件。 

问题是性能。 对于每个索引，我们可以扫描剩余未使用的索引来寻找合作伙伴。 在最坏的情况下，这大约变成$n + (n-1) + (n-2) + \cdots$，即$O(n^2)$。 和`n = 100000`，这远远超出了可行的限度。 

关键的观察是条件`a[a[j]] = j`和`a[j] != j`强制排列与索引完美匹配。 我们不需要动态地寻找合作伙伴。 我们只需要确保每个元素都与另一个元素完全配对。 

一旦我们接受这种解释，构造就变得立即：如果我们按顺序列出数字并交换相邻对，我们会自动创建有效的 2 周期。 每对`(1,2)`,`(3,4)`,`(5,6)`，依此类推满足这两个条件。 该映射是对称的，并且没有元素映射到其自身。 

如果`n`是奇数，有一个元素未配对，因此不存在有效的构造。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力匹配| O(n²) | O(n) | 太慢了 |
 | 相邻交换| O(n) | O(1) 额外 | 已接受 |

 ## 算法演练

 1. 检查是否`n`很奇怪。 如果是则立即输出`-1`。 这是因为配对需要大小恰好为 2 的组，而奇数计数会使完全配对变得不可能。 
2.初始化一个数组`a`尺寸的`n + 1`（为了方便起见，使用从 1 开始的索引）。 
3. 迭代索引`i`从`1`到`n`的步骤`2`。 
4. 对于每对`(i, i+1)`， 分配`a[i] = i+1`和`a[i+1] = i`。 这会在相邻元素之间创建相互映射。 
5. 处理完所有索引后，输出数组`1`到`n`。 

我们步长为 2 的原因是每个操作恰好消耗两个未使用的索引，并且我们保证对之间没有重叠。 

### 为什么它有效

 构造的排列是 2 个循环的不相交并。 各指标`i`恰好分配了一个合作伙伴`i+1`或者`i-1`，所以映射是双射的。 应用排列两次会返回到原始索引，因为交换两次会恢复原始位置：`a[a[i]] = i`。 由于没有元素映射到其自身，因此也满足定点条件。 不同对之间没有相互作用，因此该属性在全局范围内保持。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    if n % 2 == 1:
        print(-1)
        return
    
    a = list(range(n + 1))
    
    for i in range(1, n + 1, 2):
        a[i], a[i + 1] = i + 1, i
    
    print(*a[1:])

if __name__ == "__main__":
    solve()
```该解决方案首先通过检查奇偶校验来处理不可能性的情况。 这是我们唯一可能失败的情况。 

该数组使用虚拟零索引进行初始化，以便基于 1 的索引与问题定义匹配。 这避免了重复的指数调整并降低了相差一的风险。 

该循环在每对的恒定时间内交换相邻索引。 每次迭代都会永久确定两个位置，因此不需要额外的簿记。 

最后，打印`a[1:]`删除虚拟槽并输出排列。

 ## 边缘情况

 当`n = 1`，单个元素没有可能的伙伴。 算法立即返回`-1`，满足不允许有不动点的要求。 

什么时候`n = 2`，循环运行一次并交换位置`1`和`2`，生产`[2, 1]`。 手动检查，`a[a[1]] = a[2] = 1`对于索引也类似`2`，确认正确性。 

什么时候`n`是任意奇数吗`5`，算法在任何构造开始之前检测奇偶校验。 为了`n = 5`，返回`-1`避免尝试部分配对，例如`(1,2)`,`(3,4)`这将留下索引`5`取消配对并强制固定点或无效映射。
