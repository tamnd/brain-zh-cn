 ## 解决方案
 ## 问题理解

 我们得到一个大小的数组$2n$代表着战斗力$2n$朋友们。 任务是将他们分成两个不相交的团队，以便每个人都属于一个团队，并且两个团队必须包含奇数个成员。 

对于每个团队，其实力被定义为该团队内部排序后其成员的中位数。 我们希望选择分割，使两个中位数之间的绝对差值尽可能小。 

关键的困难在于，分割不仅仅是选择两个子集，还要确保两个子集大小都是奇数，然后了解中位数在任意分区下的表现。 

约束条件意味着$n$可以达到$10^5$， 所以$2n$可以达到$2 \cdot 10^5$每次测试，总计$2 \cdot 10^5$在所有测试中。 这立即排除了任何尝试枚举分区或模拟子集的解决方案。 甚至$O(n^2)$太慢了。 预期的解决方案必须依赖于排序和线性或近线性扫描。 

“奇数尺寸”约束产生了一个微妙的问题。 天真的尝试可能会尝试通过交替元素或平衡计数来贪婪地分裂，但这种方法可能会意外地创建大小均匀的团队或以看起来局部最优但全局无效的方式不匹配中位数。 

例如，考虑一个小数组，如下所示：```
[1, 2, 3, 100]
```如果一个人试图将相似的价值观组合在一起，我们可能会组建团队`[1, 2, 3]`和`[100]`。 这满足奇数尺寸，但中位数是 2 和 100，差异很大。 然而，存在更好的结构，并且天真的贪婪分组并没有系统地探索所有有效的中值对齐。 

另一个陷阱是假设最小化所选元素之间的差异会自动最小化中值差异。 中位数取决于每个子集内的相对顺序，而不仅仅是选择的代表。 

## 方法

 暴力方法会尝试每个有效的分区$2n$将元素分成两个奇数大小的子集，计算两个子集的中位数，并跟踪最小差异。 即使忽略分区的指数数量，计算每个分区的中位数也是昂贵的。 分区的数量约为$2^{2n}$，即使对于小规模来说，这也是完全不可行的$n$。 

关键的结构见解是排序完全决定了中位数的表现。 一旦数组排序完毕，任何中位数都对应于某个其排名全局固定的元素。 真正的问题是选择两个可以同时充当有效奇数大小子集的中值的元素。 

两个子集必须为奇数的约束使得结构变得刚性。 如果一支球队有规模$2x+1$，另一个自动具有大小$2(n-x)-1$。 每个中位数对应于其子集的一个“中心”元素，这意味着子集的一半位于该子集中该中位数的每一侧。 这会强制平衡条件：如果在排名中选择一个中位数$i$，另一个中位数不能任意接近，除非它们之间有足够的元素可以分配到两个团队，同时保留奇数大小。 

这导致了一个强大的重新表述：对整个数组进行排序后，我们在索引空间中对称地配对元素。 如果我们选择两个中间候选人$a[i]$和$a[i+n]$，它们之间的元素恰好足以分布在两个团队中，以满足奇数大小约束，同时保持两个中位数有效。 

因此，最佳答案减少为扫描所有此类有效对并取最小差异。

 ## 算法演练

 1. 对整个数组进行排序。 排序是必要的，因为中位数是按顺序定义的，并且任何有关有效中位数的推理都必须依赖于全局排名而不是子集局部结构。 
2. 观察到我们只需要考虑可以同时充当两个有效奇数大小子集的中值的元素对。 
3. 确定该位置价值较小的团队的候选中位数$i$在已排序的数组中。 为了确保剩余的元素可以形成第二个具有有效中位数的奇数团队，匹配中位数的排名必须足够远。 
4.正确的配对是匹配索引$i$带索引$i+n$。 这个间隙保证了$n$元素位于两个选定的中位数之间，这有足够的灵活性将元素分配到两个奇数大小的组中，同时保持中位数有效性。 
5. 计算差异$a[i+n] - a[i]$对于所有有效的$i$从$0$到$n-1$，并取最小值。 
6. 返回找到的最小值。 

### 为什么它有效

 排序固定全局排名，因此任何中位数都对应于排序顺序中的固定位置。 一旦我们选择了两个候选中位数，就必须分配所有其他元素，以便每个中位数在其自己的团队中具有相同数量的较小和较大的元素。 奇数规模约束迫使每个团队都有一个明确定义的中心元素，同时满足两个中位数而不发生重叠冲突的唯一方法是将它们精确地分开$n$已排序数组中的位置。 这确保了中间区域包含足够的元素来对称分布，同时独立地保留两个中值定义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        a.sort()

        ans = float('inf')
        for i in range(n):
            ans = min(ans, a[i + n] - a[i])

        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先对数组进行排序，以便中值位置对应于连续的索引。 排序后，它仅检查对$(i, i+n)$，它们是唯一能够保持足够分离以形成两个有效奇数大小组的对。 循环运行结束$n$考虑到限制，这是安全的。 

这里的一个常见错误是迭代所有对或尝试显式构建子集。 这是不必要的，因为排序结构已经隐式编码了所有有效的中值配置。

 ## 边缘情况

 像这样的最小情况`[1, 100]`强制两个团队的尺寸均为 1，因此两个中位数都是元素本身。 该算法仅考虑单个有效配对并返回正确的差异。 

当所有元素都相同时，每个可能的分割都会产生相等的中位数。 排序后，每$a[i+n] - a[i]$为零，因此算法自然输出零，无需特殊处理。 

在均匀间隔递增序列的情况下，跨距离配对$n$确保两个中位数都来自数组的平衡部分。 该算法的固定索引间隙保证每个中位数在其各自的子集中两侧都有足够的元素，因此计算出的差异反映了有效的可实现配置，而不是抽象配对。
