 ## 解决方案
 ## 问题理解

 我们得到一个字符串和多个独立的查询。 每个查询指定字符串的一个段，并且我们只能查看该段的内部。 我们的任务是找到一个完全包含在查询段中的单个连续子串，该子串是回文并且具有最大可能长度。 每个查询的答案就是最佳回文的长度。 

因此从概念上讲，每个查询都在字符串上定义一个窗口，在该窗口内我们正在搜索最强的对称模式。 

每个测试用例的约束都很小，字符串长度和查询数量最多为 1000，但所有测试用例的总和可以达到大约一百万个 n 乘以 q 形式的操作。 这立即排除了任何针对每个查询从头开始重新计算回文信息或尝试围绕每个查询的每个中心进行扩展的解决方案。 每个查询大约是二次的解决方案已经太慢了。 

关键的困难在于回文是全局结构，而查询是局部限制。 字符串中存在的回文仅当它完全包含在其间隔内时才对查询有用。 

一些边缘情况值得明确说明。 

如果查询间隔的长度为 1，则答案始终为 1，因为每个字符都是回文。 仅考虑偶数长度回文的简单解决方案可能会错误地返回 0。 

如果字符串包含所有相同的字符，则每个子字符串都是回文，因此每个查询都应返回完整的间隔长度。 任何仅跟踪中心周围最大回文数但无法跨位置聚合的解决方案都会低估这种情况。 

如果最佳回文很大但稍微超出查询范围，则不得计算在内。 例如，在`abacaba`, 回文`abacaba`是全局的，但是查询`[2,6]`应该只考虑`bacab`，不是完整的字符串。 

## 方法

 直接强力解决方案独立处理每个查询。 对于固定查询`[l, r]`，我们枚举该区间内的所有子串，并检查每个子串是否是回文，保持最大长度。 

检查子串是否为回文可以在其长度的线性时间内完成，或者通过动态编程预先计算。 无论哪种方式，对于单个查询，我们都会粗略地查看$O((r-l+1)^3)$如果天真地这样做，或者$O((r-l+1)^2)$与 DP 检查。 对于多达 1000 个查询，在最坏的情况下这会变得太慢。 

使这个问题可解决的结构是，任何子串的回文成员资格都可以预先计算一次$O(n^2)$，然后重复使用。 剩下的挑战不是检测回文，而是在一组预先计算的“回文间隔”上回答范围最大查询。 

我们翻转视角。 我们不再询问“对于每个查询，哪个回文是最好的？”，而是一次性枚举所有回文子串。 每个回文都成为一个值等于其长度的区间。 然后每个查询要求完全包含在中的最大值区间`[l, r]`。 

这将问题转换为间隔内的二维范围最大查询，我们可以将其重组为每开始或每结束聚合，并通过预处理和稀疏表有效地回答。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的暴力破解 |$O(n^3)$每个查询最坏情况|$O(1)$或者$O(n^2)$| 太慢了 |
 | 预计算回文+范围最大结构 |$O(n^2 \log n)$预处理,$O(1)$每个查询|$O(n^2 \log n)$| 已接受 |

 ## 算法演练

 我们分层构建解决方案，从回文检测开始，到快速查询应答结束。 

### 步骤 1：预先计算回文表

 我们建一张桌子`pal[i][j]`告诉子字符串是否`s[i..j]`是一个回文。 这是使用标准 DP 完成的：单个字符是回文，两个字符的子字符串是回文（如果相等），较长的子字符串取决于端点和内部子字符串。 

这让我们充分了解哪些子串是有效的回文。 

### 步骤 2：将回文转换为“最佳结尾为 r”值

 对于每个端点`r`，我们想知道，对于每个可能的起始位置`l`，结束于的最长回文数是多少`r`并开始于或之后`l`。 

我们定义一个数组`bestStart[r][l]`含义：在所有以`r`并开始于`[l, r]`，最大长度是多少。 

为了计算这个，我们扫描`l`从右到左每个固定`r`。 如果`s[l..r]`是一个回文，它贡献候选长度`r-l+1`。 当向左移动时，我们保持运行最大值，以便每个`bestStart[r][l]`检查后可以在常数时间内填写`pal[l][r]`。 

此步骤压缩所有以`r`进入一个可以回答“在左约束下在此结束的最佳回文”的结构。 

### 步骤 3：将查询转换为端点上的范围最大值

 对于查询`[l, r]`，我们选择的回文必须在之间的某个地方结束`l`和`r`。 对于每个可能的端点`e`，我们已经知道最好的回文结尾为`e`开始于或之后`l`，即`bestStart[e][l]`。 

所以查询的答案就变成了：

 最大总和`e`在`[l, r]`的`bestStart[e][l]`。 

现在的问题是每个固定数组的范围最大查询`l`。 

### 步骤 4：预处理范围最大查询

 对于每个固定`l`，我们建立一个稀疏表`e`对于数组`bestStart[e][l]`。 这允许在任何时间间隔内回答最大查询`[l, r]`在恒定的时间内。 

自从`n`最多 1000，构建`n`稀疏表足够高效。 

## 为什么它有效

 每个回文子串都由其端点唯一确定`(i, j)`。 该算法确保每个这样的子字符串在`bestStart[j][i]`，并且不会遗漏任何有效的回文。 

对于任何查询，每个候选回文都按其结束位置分组，并且该结构保证我们仅在有效起始位置上取最大值。 稀疏表确保我们不会跳过任何端点`[l, r]`，因此始终保留所有有效回文的最大值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, q = map(int, input().split())
        s = input().strip()
        s = " " + s  # 1-indexed

        pal = [[False] * (n + 1) for _ in range(n + 1)]
        best_start = [[0] * (n + 2) for _ in range(n + 2)]

        for i in range(1, n + 1):
            pal[i][i] = True

        for i in range(1, n):
            if s[i] == s[i + 1]:
                pal[i][i + 1] = True

        for length in range(3, n + 1):
            for i in range(1, n - length + 2):
                j = i + length - 1
                if s[i] == s[j] and pal[i + 1][j - 1]:
                    pal[i][j] = True

        for r in range(1, n + 1):
            for l in range(r, 0, -1):
                best_start[r][l] = best_start[r][l + 1]
                if pal[l][r]:
                    best_start[r][l] = max(best_start[r][l], r - l + 1)

        LOG = 10
        st = [[[0] * (LOG + 1) for _ in range(n + 1)] for _ in range(n + 1)]

        for l in range(1, n + 1):
            for r in range(1, n + 1):
                st[l][r][0] = best_start[r][l]

            j = 1
            while (1 << j) <= n:
                for r in range(1, n + 1):
                    if r + (1 << j) - 1 <= n:
                        st[l][r][j] = max(st[l][r][j - 1],
                                          st[l][r + (1 << (j - 1))][j - 1])
                j += 1

        def query(l, r):
            length = r - l + 1
            k = (length).bit_length() - 1
            return max(st[l][r][k], st[l][r - (1 << k) + 1][k])

        for _ in range(q):
            l, r = map(int, input().split())
            print(query(l, r))

if __name__ == "__main__":
    solve()
```该代码首先构建一个标准回文 DP 表。 然后它按端点将所有回文压缩为`best_start`。 最后，它为每个固定的左边界构建一个稀疏表，以便每个查询成为端点上的恒定时间范围最大查询。 

这里的一个常见陷阱是尝试直接回答来自`pal[i][j]`不跟踪哪些回文实际上位于其中`[l, r]`。 这就失去了对遏制的依赖。 分离为“基于终端的聚合”使解决方案保持一致。 

## 工作示例

 ### 示例 1

 输入字符串：`abcddcba`， 询问`[1, 8]`我们只展示关键步骤。 

| r | bestStart[r][l] 相关条目 |
 | --- | --- |
 | 5（`d`) | 单个字符|
 | 6（`d`) | 回文`dd`在 l=5 时给出值 2 |
 | 7（`c`) | 单身|
 | 8（`a`) | 完全反向结构有贡献但有限|

 查询检查所有端点 1 到 8，并从中找到最大值 8`abcddcba`。 

该迹线显示了全范围回文在存在时如何占主导地位。 

### 示例 2

 输入字符串：`abcddcba`， 询问`[2, 7]`我们将端点限制为 2..7。 完全在里面的最好的回文是`cddc`长度为 4。 

| 端点 e | 最佳开始[e][2] |
 | --- | --- |
 | 4 | 0 |
 | 5 | 0 |
 | 6 | 2（`dd`) |
 | 7 | 1 |

 最多为 4 个`cddc`。 

这证实了有效的回文必须满足端点和包含约束。

 ## 边缘情况

 对于一串相同的字符，例如`aaaaa`，每个子串都是回文。 该算法标记每个`(l, r)`在 DP 表中有效，并且`best_start`传播全长。 当查询询问时`[2, 4]`，端点聚合保证了最大`3`从子字符串返回`aaa`，确认密集回文区域的正确处理。 

对于单字符查询，例如`[3, 3]`在`abc`，仅对角线条目`pal[i][i]`是真的。 这`best_start`array 在每个位置记录长度为 1，并且在单个端点上查询正好返回 1，符合每个字符都是回文的定义。 

对于最佳回文跨越中心但不跨越边界的查询，例如`abacaba`和`[2, 6]`，完整回文会被自动排除，因为它的端点在区间之外。 只有完全包含在该段中的回文才会有助于`best_start`，因此结果正确地缩减为最大的内部对称子串。
