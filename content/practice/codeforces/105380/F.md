 ## 解决方案
 ## 问题理解

 我们被要求查看从 1 到 n 的每个数字排列，计算每个排列包含多少个反转，然后将所有排列的反转计数相加。 

排列中的反转只是一对位置，其中较大的数字比较小的数字出现得早。 对于每个排列，我们对这些对进行计数，然后在大小为 n! 的整个排列空间中聚合此计数。 

因此，每个测试用例的输出都是一个数字：长度为 n 的所有排列所贡献的反转总数。 

就测试用例数量而言，约束是极端的，最多可达 10^6，并且 n 本身最多可达 10^6。 这立即排除了任何迭代排列甚至每次测试构建阶乘推理的方法。 该解决方案必须在预计算后将每个查询减少到 O(1) 或 O(log n)，否则不会通过。 

这里经常出现的一个天真的误解是试图计算每个排列的反转计数或模拟交换。 即使考虑生成排列也已经太慢了，因为 n! 当 n = 10 时，增长超出了可行性。 

更微妙的边缘情况是 n = 1。只有一个排列且不存在对，因此答案必须为 0。任何除以 n 或假设每对至少有一个反转的公式仍必须遵守此基本情况。 

## 方法

 直接方法将枚举每个排列并单独计算反转。 对于每个排列，标准 O(n^2) 扫描会查找反转。 因为有 n! 排列，这变成了 O(n!·n^2)，即使 n = 10 也会立即爆炸。 

关键的观察是停止考虑全局排列，而是分析一对固定的位置 i 和 j，其中 i < j。 我们问：在所有排列中，这一对产生反转的频率是多少？ 

修复任何一对不同的值 x 和 y。 在所有排列中，对称性告诉我们，在一半排列中，x 出现在 y 之前，而在另一半排列中，x 出现在 y 之后。 这是因为在任何排列中交换 x 和 y 都会创建一个双射，该双射会翻转该对是否形成反转。 

因此，对于每对无序值，对总反转的贡献是相同的并且纯粹是组合的。 

有两种等效的观点。 一是固定头寸：每对指数贡献均等。 另一个是固定值：每对值在排列中表现对称。 基于价值的观点更加清晰。 

对于任何一对不同的元素，例如（a，b），它们在所有排列中以任一顺序出现的频率相同。 有n个！ 总共排列，其中正好有一半将 a 放在 b 之前。 当 a 在 b 之后时，该对贡献一次反转。 

因此，每个无序对都贡献了 n! / 总共 2 次反转。 

这样的对共有 C(n, 2) 个，因此最终答案变为：

 C(n, 2) · n! / 2

 我们可以简化：

 C(n, 2) = n(n−1)/2

 所以答案 = n(n−1)/2 · n！ / 2 = n(n−1)n! /4

 这将问题简化为计算最多 n 的阶乘并与每个查询的简单算术相乘。 

我们在所有测试用例中预先计算最大 n 的阶乘，因为 t 很大并且重复计算会很浪费。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!·n^2) | O(n!·n^2) | O(n) | 太慢了 |
 | 最佳| O(最大 n + t) | O(最大 n) | 已接受 |

 ## 算法演练

 我们现在将组合公式转换为实用的计算策略。

1. 读取所有测试用例并确定其中的最大n。 这确保阶乘仅计算一次直至达到所需的限制。 
2. 使用迭代乘法预先计算从 1 到 max n 的阶乘。 每个阶乘都直接从前一个阶乘构建，这避免了重复的重新计算。 
3. 对于每个测试用例 n，使用公式 n(n−1)n 计算答案！ / 4.
 4. 打印每个测试用例的结果。 

唯一微妙的一点是确保整数算术保持精确。 所有值都是整数，但除以 4 必须精确，因为 n(n−1)n！ 当 n ≥ 2 时，总是能被 4 整除。当 n = 0 或 1 时，我们直接返回 0。 

### 为什么它有效

 由于对称性，每对无序值在所有排列中的行为都是相同的。 任意两个值之间的交换操作将排列划分为大小相等的类，其中该对的顺序翻转。 这保证了每对都贡献了所有排列的一半作为反转。 对所有对求和会产生一个独立于内部排列结构的统一封闭式表达式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = None  # no modulus specified

t = int(input())
ns = []
max_n = 0

for _ in range(t):
    n = int(input())
    ns.append(n)
    if n > max_n:
        max_n = n

fact = [1] * (max_n + 1)
for i in range(1, max_n + 1):
    fact[i] = fact[i - 1] * i

def solve(n):
    if n < 2:
        return 0
    return fact[n] * n * (n - 1) // 4

out = []
for n in ns:
    out.append(str(solve(n)))

print("\n".join(out))
```阶乘数组预先计算一次，这可以避免在最多 10^6 次查询中重新计算。 然后，每个查询都会简化为一个常数时间算术表达式。 

除以 4 是安全的，因为对于 n ≥ 2，其中 n! 排列，对称性确保 n(n−1)n! 可被 4 整除。这可以避免浮点问题并保留精确的整数输出。 

## 工作示例

 我们首先计算阶乘：fact[3] = 6。 

那么答案 = 6 × 3 × 2 / 4 = 36 / 4 = 9。

 这与所有 6 个排列的预期总反转和相匹配。

def run(inp: str) -> str:

 ## 边缘情况

 对于 n = 1，没有索引对，因此反转计数必须为零。 该公式将直接给出 n(n−1)n!/4 = 0，因此实现无需特殊处理即可安全返回 0。 

对于 n = 2，我们有两个排列 [1,2] 和 [2,1]。 只有第二个贡献一次反转。 公式给出 2 × 1 × 2！ / 4 = 4 / 4 = 1，与枚举完全匹配。 

对于较大的 n，阶乘值会快速增长，但 Python 整数可以安全地处理任意精度。 主要风险是性能，通过预先计算一次并在所有查询中重用结果可以避免这种风险。
