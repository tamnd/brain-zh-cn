 ## 解决方案
 ## 问题理解

 我们得到一堆同心凸多边形，一个在另一个里面，其中每个多边形完全包含前一个多边形。 每一层都有一个相关的分数。 如果飞镖落在某个区域，则始终会记入仍然覆盖该点的最里面多边形的分数。 

从几何角度来看，这将平面划分为不相交的“环”：多边形之间的区域$i-1$和多边形$i$。 飞镖落在环内任意位置$j$产生分数$S_j$，无论它向外抛出多远，只要它仍在定义抛出的外边界内即可。 

爱丽丝投掷$M$飞镖。 她首先选择一个“精确区域”多边形$t$。 确切地$K$飞镖在多边形内均匀随机投掷$t$，而剩余的$M-K$飞镖均匀地投掷在最外面的多边形内$N$。 

目标是选择最好的多边形$t$从而使所有飞镖的期望总分最大化。 由于答案需要模数$10^9+7$，最终值表示为具有模逆的分数。 

这些约束意味着我们无法模拟随机点或离散几何形状。 计算面积和期望值必须在多边形上以线性时间完成，并且所有比较必须是精确的，而不是浮点。 

天真的解释会试图评估每个候选人$t$使用几何采样或在多边形上重复积分，这太慢了。 另一个微妙的陷阱是使用浮点区域：由于我们比较期望值的比率，即使很小的精度误差也可能改变所选的最佳多边形。 

当多边形是极薄的壳时，会出现进一步的边缘情况。 例如，如果一个多边形添加了非常小的区域但分数很大，则幼稚的浮动除法可能会错误地对候选者进行排名。 

## 方法

 如果我们修复一个多边形$t$，我们可以通过将区域分解为环来计算在其内部均匀投掷飞镖的预期得分。 每个环$j$与其总面积内的面积成比例地贡献$t$。 这导致了一个干净的期望公式：期望分数是分数的加权平均值，其中权重是所选多边形内每个环的面积比例。 

对于固定的$t$，我们可以在线性时间内计算它包含的所有环的期望。 为所有人做这件事$N$选择给出了$O(N^2)$解决方案。 这是有效的，因为对于每个候选者，我们独立地重新计算环贡献的前缀和。 

瓶颈很明显：如果$N$很大，说$2 \cdot 10^5$， 然后$O(N^2)$操作是不可行的。 

关键的观察是每个候选期望可以写成两个前缀和的比率。 具体来说，对于每个$i$，我们将分子定义为累积的“得分加权面积”，将分母定义为总面积。 期望变为$E_i = \frac{\text{num}_i}{\text{den}_i}$。 

一旦问题简化为比较前缀的比率，我们只需要找到其中的最大分数$N$候选人。 这可以使用交叉乘法在线性时间内完成，完全避免浮点。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(N^2)$|$O(N)$| 太慢了|
 | 前缀+比率最大化|$O(N)$|$O(N)$| 已接受 |

 ## 算法演练

 ### 计算几何构建块

 1. 使用鞋带公式计算每个多边形的双倍符号面积。 我们保留双倍的面积以避免分数，因为只有比率才重要。 这确保所有几何形状保持整数。 
2. 通过减去连续面积，将每个多边形转换为“环贡献”。 多边形与多边形的区别$j$和$j-1$给出环的面积$j$。 这是可行的，因为多边形是严格嵌套的，因此不存在重叠歧义。 

### 构建期望组件

 1. 维护两个前缀累加器。 一店总加权分面积达环$i$，其他门店总面积达环$i$。 分子通过相加而增长$R_j \cdot S_j$，而分母通过添加而增加$R_j$。 

该结构直接编码了多边形均匀落地的期望值$i$，因为每个点都是均匀分布的，并且根据面积贡献比例概率。 

### 寻找最佳精度区域

 1. 对于每位候选人$i$，我们将其期望解释为分数$\frac{\text{num}_i}{\text{den}_i}$。 我们使用交叉乘法来比较候选者：$a > b$当且仅当$\text{num}_a \cdot \text{den}_b > \text{num}_b \cdot \text{den}_a$。 这完全避免了浮点。 
2. 跟踪产生最大分数的索引。 

### 计算最终答案

 1. 一次最佳精度多边形$i^*$找到，计算最终期望分数：$$K \cdot E_{i^*} + (M-K) \cdot E_N$$每个期望都是一个分数，因此我们使用模逆进行转换$10^9+7$。 

### 为什么它有效

 核心不变量是每个多边形$i$对应于不相交环上明确定义的概率分布，并且期望对于这些不相交的贡献是线性的。 因为环精确地划分了空间，所以前缀和完全捕获了多边形内的所有随机性。 因此，每个候选都减少到一个单一的理性值，并且最大化期望变成对从相同累积结构导出的分数的纯粹比较。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def inv(x):
    return pow(x, MOD - 2, MOD)

def shoelace(poly):
    n = len(poly)
    s = 0
    for i in range(n):
        x1, y1 = poly[i]
        x2, y2 = poly[(i + 1) % n]
        s += x1 * y2 - x2 * y1
    return abs(s)

def solve():
    N, M, K = map(int, input().split())
    S = [0] + list(map(int, input().split()))

    polys = []
    for _ in range(N):
        v = int(input())
        poly = [tuple(map(int, input().split())) for _ in range(v)]
        polys.append(poly)

    area2 = [0] * (N + 1)
    for i in range(1, N + 1):
        area2[i] = shoelace(polys[i - 1])

    R = [0] * (N + 1)
    for i in range(1, N + 1):
        R[i] = area2[i] - area2[i - 1]

    num = [0] * (N + 1)
    den = [0] * (N + 1)

    for i in range(1, N + 1):
        num[i] = num[i - 1] + R[i] * S[i]
        den[i] = den[i - 1] + R[i]

    best = 1
    for i in range(1, N + 1):
        if num[i] * den[best] > num[best] * den[i]:
            best = i

    numN, denN = num[N], den[N]
    numB, denB = num[best], den[best]

    ans = (K * numB % MOD) * inv(denB) % MOD
    ans = (ans + (M - K) * (numN % MOD) % MOD * inv(denN)) % MOD

    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先通过鞋带公式计算双倍多边形面积，确保所有几何值保持整数。 然后，它通过区分连续的多边形将这些区域转换为环形区域，从而隔离每个不相交的环形区域。 

构建前缀和是为了维护每个前缀多边形的预期分数的分子和分母。 这是将几何期望转化为比率比较问题的关键简化。 

最佳多边形的选择使用 64 位整数的交叉乘法，完全避免了精度问题。 最后，模运算使用费马逆来重建期望值。

 ### 示例 1

 假设有 3 个带有分数的多边形$S = [1, 10, 3]$和面积（双倍）$A = [2, 6, 12]$。 让$M = 5, K = 2$。 

| 我| R_i | num_i | 丹尼 |
 | --- | --- | --- | --- |
 | 1 | 2 | 2 | 2 |

 我们比较分数：$E_1 = 1$,$E_2 = 7$,$E_3 = 5$。 最好的是$i=2$。 

最终的期望是：$2 \cdot 7 + 3 \cdot E_3$。 

这表明即使外部区域更大，较高的内部分数也可以占据主导地位。

 ## 边缘情况

 关键的边缘情况是当多边形添加非常小的环区域但具有很大的分数时。 在这种情况下，浮点实现通常会高估面积并低估分数峰值。 在这里，由于所有内容都是通过整数前缀和计算并通过交叉乘法进行比较，因此比率保持精确和稳定。 

另一个边缘情况是多个候选人的期望相同。 由于比较使用严格的大于，因此保留第一个最大值。 由于所有分数完全相等，因此任何选择都是有效且一致的。 

最后，当所有分数都相同时，每个前缀比率都会折叠为相同的值。 该算法仍然正确地计算与面积成比例的相等分子和分母，从而准确地保持相等性。
