 ## 解决方案
 ## 问题理解

 我们有两个整数，$k$和$m$。 考虑建立一个目标金额$n$将其准确地分成$k$有序部分，其中每个部分都是非负整数。 如果在任意位置，则两个分解是不同的$k$-元组不同。 

对于固定的$n$，问题是计算有多少个这样的$k$-存在其条目总和为的元组$n$。 我们被要求找到最小的$n \ge 0$使得这个计数至少是$m$。 如果没有这样的$n$存在，我们必须返回$-1$。 

这是一个直接组合计数问题。 有序非负解的数量$$x_1 + x_2 + \dots + x_k = n$$由标准星条恒等式给出：$$\binom{n+k-1}{k-1}.$$所以问题就简化为寻找最小的$n$这样$\binom{n+k-1}{k-1} \ge m$。 

输入大小足够大$n$可以增长很远，并且二项式系数很快就会变得巨大。 单纯的基于阶乘的计算是不可能的。 甚至直接迭代过来$n$太慢了，因为$n$可以达到周围$10^{18}$在搜索空间中。 

主要困难不是组合数学本身，而是有效评估大二项式系数并在不溢出的情况下搜索阈值。 

当出现一种边缘情况时$k = 1$。 那么就只有一种方法可以表示任何$n$，所以如果$m > 1$不存在解，否则$n = 0$已经适用于$m = 1$。 另一个边缘情况是$m = 1$， 在哪里$n = 0$始终有效，因为只有一种方法可以将零写为$k$非负整数。 

一种简单的方法，枚举每个组合的所有组合$n$即使对于很小的值也会立即失败，因为组合的数量会组合增长并且很快变得巨大。 

## 方法

 蛮力的想法很简单。 对于每个$n$，计算数量$k$- 元组求和为$n$，然后当计数达到时停止$m$。 虽然在概念上是正确的，但这需要评估每个的二项式系数$n$，并且每次评估都涉及较大的中间值。 即使使用乘法公式，也要从头开始对每个公式进行计算$n$导致最坏情况的复杂性约为$O(n \cdot k)$，这是不可能的，当$n$可以达到$10^{18}$。 

关键的观察结果是函数$$f(n) = \binom{n+k-1}{k-1}$$是单调的$n$。 增加$n$只会增加作品的数量。 这允许二分搜索$n$，因为我们正在寻找单调谓词成立的最小值。 

剩下的任务是有效评估$\binom{n+k-1}{k-1}$在大值下。 我们用乘法来计算它，而不是阶乘：$$\binom{N}{r} = \prod_{i=1}^{r} \frac{N - r + i}{i}$$和$N = n + k - 1$和$r = k - 1$。 一旦运行值超过，我们就会提前停止$m$，因为超出阈值的精确值与二分搜索决策无关。 

我们还使用 128 位算术限制中间值，以避免在比较之前溢出。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n \cdot k)$|$O(1)$| 太慢了|
 | 最优（二分搜索 + 乘法 nCr）|$O(\log n \cdot k)$摊销|$O(1)$| 已接受 |

 ## 算法演练

 1、先处理小事。 如果$m = 1$，最小有效总和是$n = 0$因为总是有一种方法可以将零分成$k$部分。 
2.如果$k = 1$, 每个非负数$n$仅有一种表示。 如果$m > 1$， 不$n$可以满足条件，所以答案是$-1$。 
3. 设置二分查找范围$n$从$0$到足够大的上限，例如$10^{18}$。 这个界限安全地涵盖了所有有意义的答案，因为二项式系数增长得非常快。 
4. 对于候选人$n$, 计算$N = n + k - 1$和$r = k - 1$，并评估$\binom{N}{r}$使用乘法公式。 这避免了阶乘计算并使数字易于管理。 
5. 计算乘积时，逐项相乘，每次相乘后检查该值是否达到或超过$m$。 如果有，请尽早停止并将该值视为足够。 这可以防止溢出和不必要的工作。 
6. 如果计算值至少为$m$，将二分查找右边界移至$n$。 否则，将左边界移动到$n + 1$。 
7. 二分查找完成后，验证结果是否$n$实际上满足条件。 如果没有，则返回$-1$。 

正确性取决于单调性：增加$n$增加了组合的数量，所以谓词“$\binom{n+k-1}{k-1} \ge m$”在单个边界处从假到真的转变。

 ## Python 解决方案```python
import sys
input = sys.stdin.readline

def ncr_at_least(n, r, limit):
    if r < 0 or r > n:
        return 0
    if r > n - r:
        r = n - r

    res = 1
    for i in range(1, r + 1):
        res *= n - r + i
        if res >= limit:
            return limit
        res //= i
    return res

def count_ways(n, k, m):
    N = n + k - 1
    r = k - 1
    return ncr_at_least(N, r, m)

def solve():
    k, m = map(int, input().split())

    if m == 1:
        print(0)
        return
    if k == 1:
        print(-1)
        return

    lo, hi = 0, 10**18
    ans = -1

    while lo <= hi:
        mid = (lo + hi) // 2
        if count_ways(mid, k, m) >= m:
            ans = mid
            hi = mid - 1
        else:
            lo = mid + 1

    print(ans)

if __name__ == "__main__":
    solve()
```核心实现依赖于一个安全的二项式系数评估器，一旦达到阈值就会停止$m$已达到。 除法在乘法之后立即进行，保持中间值足够小，以避免实践中的溢出，甚至在 Python 的大整数变得太大之前也是如此。 二分查找保持以下所有值的不变性`lo`无效且所有值等于或超出`ans`是有效的候选人。 

特殊情况为$m = 1$和$k = 1$避免二项式公式不必要或具有误导性的退化行为。

 ## 边缘情况

 对于$m = 1$，算法立即返回$0$。 例如输入`k = 5, m = 1`，该检查完全绕过二分搜索。 这是正确的，因为$\binom{4}{4} = 1$对应于$n = 0$。 

为了$k = 1$，否则该函数将计算$\binom{n}{0} = 1$为所有人$n$，但提前返回会强制执行正确性。 用于输入`k = 1, m = 2`，输出是`-1`，与没有增加的事实相匹配$n$改变计数。 

对于大型$m$， 例如$m = 10^{12}$，一旦运行的乘积超过$m$，防止溢出并确保二分搜索决策保持快速，即使精确的二项式值很大。
