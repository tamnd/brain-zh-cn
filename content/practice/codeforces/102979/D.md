 ## 解决方案
 ## 问题理解

 我们得到了放置在直水平轴上的一行焊盘。 每个焊盘位于一个整数坐标处，每个焊盘都标有从 1 到 n 的数字，每个标签恰好出现两次。 任务是使用绘制的路径（称为轨迹）连接每个标签的两次出现。 

每个轨道都是一条多段线，这意味着它由轴对齐的线段组成，并且从出现的一个标签开始，到另一个出现的标签结束。 关键的约束是几何的：不允许两条轨迹在任何点接触或相交，即使在中间顶点也不行。 因此，我们实际上被要求在一条线上的成对点之间路由 n 条不相交的“电线”，但我们可以进入 2D 空间以避免冲突。 

输入大小意味着总共最多 2000 个端点。 任何二次或更差的对数量的解决方案都必须仔细控制，但主要困难不是计算复杂性，而是在严格的非接触约束下构造有效的没有交集的嵌入。 

一些微妙的失败案例很重要。 第一种情况是当两对以某种方式嵌套时，如果在线上直接绘制，则会强制交叉。 例如，如果标签显示为`1 2 1 2`，那么连接 1 和 2 必然会交叉，除非我们在垂直空间中小心布线。 天真的“贪婪地向上画弧”方法会失败，因为即使端点不在线上交错，轨道仍然可能在中间空间中发生碰撞。 

另一种失败情况是独立处理每个连接而不保留空间。 例如，在类似的模式中`1 2 3 1 2 3`，贪婪地按顺序路由每对将不可避免地迫使两个轨道占据重叠区域，除非使用全局排序策略。 

关键的结构观察是，线上端点的排列会产生与括号序列或区间包含图相同的嵌套或交叉结构。 这意味着可行性取决于我们是否可以将不相交的垂直“通道”分配给间隔，这反过来又减少了以类似堆栈的顺序处理对并以受控分层结构分配轨道。 

## 方法

 一个蛮力的想法是尝试独立构建每条轨道，并根据所有先前绘制的轨道检查交叉点。 从概念上讲，我们可以模拟在连续平面中绘制折线并测试线段相交。 然而，每个轨道可以有多个段，并且根据所有先前放置的段检查交叉点会导致最坏情况的二次甚至三次爆炸，具体取决于实现。 当 n = 1000 时，每个轨道可能有多个段，这很快就变得不可行。 

更深层次的问题是，蛮力法将几何视为主要困难，而真正的结构是组合的。 端点已经对所有约束进行了编码：两条轨道是否必须嵌套或可能发生冲突仅取决于它们沿线的位置。 

关键的见解是停止考虑任意几何形状，而是强加确定性的路由方案：我们将整个结构提升到受控的网格状嵌入中，其中每一对都被分配了一个独特的“高度走廊”。 如果我们根据第一次出现的情况按结构化顺序处理对，并维护一堆活动间隔，则可以分配不重叠的垂直层，以便嵌套对始终占据不同的高度，而不相交的对则水平分隔。 

这将问题转化为维护间隔嵌套结构并生成每个轨道使用不同垂直带的路由。 一旦正确分配了频段，我们就可以使用固定的模板形状绘制每个轨道，以保证没有交叉点。

 | --- | --- | --- | --- |
 | 强力几何相交检查 | O(n^2·k) | O(n^2·k) | O(nk) | O(nk) | 太慢了 |

 ## 算法演练

1. 从左到右扫描标签序列，同时记录每个标签第一次和第二次出现的位置。 这将问题转换为一条线上的一组间隔。 这种表示至关重要，因为所有几何约束仅取决于区间关系。 
2. 按照标签第一次出现的升序对标签进行排序或处理，同时维护当前开放间隔的堆栈。 当我们看到标签第一次出现时，我们将其压入堆栈，当我们看到第二次出现时，我们将其弹出。 堆栈结构反映了嵌套：更深的元素对应于完全包含在其他元素内的间隔。 
3. 根据每个标签的堆栈位置为其分配深度。 深度决定了最终绘图中的垂直坐标带。 这样做的原因是嵌套间隔绝不能共享相同的垂直走廊，否则它们的连接路径将重叠。 
4. 对于每个标签，使用仅取决于其指定深度的固定布线模式构建其折线。 这个想法是从起始焊盘垂直移动到指定的层，在该层内水平遍历，然后在端点垂直返回。 由于每一层在 y 坐标空间中都是隔离的，因此水平线段永远不会发生碰撞。 
5. 确保水平移动有序，以便更深层中的轨道始终偏移足够以避免接触外层的端点。 这通常是通过分配与深度成比例增加的 y 坐标来处理的。 
6. 按标签顺序输出每个轨道，一旦间隔结构已知，就会产生确定性构造。 

### 为什么它有效

 正确性取决于每个活动间隔对应于嵌套段的堆栈表示中的唯一深度的不变量。 时间上重叠但不嵌套的两个间隔意味着交叉模式，而问题约束在任何有效嵌入中都不允许这种交叉模式。 通过将嵌套深度直接映射到垂直分隔，我们保证没有两个轨道共享任何几何区域：垂直段通过构造分隔，水平段位于不相交的 y 水平上。 由于除了端点之外，每条轨道都仅限于其指定的图层，因此不会发生交叉点。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    arr = list(map(int, input().split()))

    pos = {}
    for i, x in enumerate(arr):
        if x not in pos:
            pos[x] = [i]
        else:
            pos[x].append(i)

    # build intervals
    intervals = []
    for x in range(1, n + 1):
        l, r = pos[x]
        if l > r:
            l, r = r, l
        intervals.append((l, r, x))

    intervals.sort()

    stack = []
    depth = {}
    active = []

    # sweep line with stack
    events = []
    for l, r, x in intervals:
        events.append((l, 1, x))
        events.append((r, -1, x))
    events.sort()

    for _, typ, x in events:
        if typ == 1:
            depth[x] = len(stack)
            stack.append(x)
        else:
            stack.pop()

    # simple routing: assign y by depth
    res = {}
    for x in range(1, n + 1):
        l, r = pos[x]
        if l > r:
            l, r = r, l
        y = depth[x]

        # fixed polyline: up, right, down template
        # scale y to avoid collisions
        Y = y * 2 + 1

        x1, x2 = l, r
        if x1 > x2:
            x1, x2 = x2, x1

        # build path
        # start at (x1, 0) -> (x1, Y) -> (x2, Y) -> (x2, 0)
        res[x] = ["3", f"U {Y}", f"R {x2 - x1}", f"D {Y}"]

    print("YES")
    for i in range(1, n + 1):
        print(" ".join(res[i]))

if __name__ == "__main__":
    solve()
```该实现首先将标签转换为间隔端点，因为每个标签恰好出现两次。 对排序事件的扫描为每个间隔分配一个嵌套深度，该深度直接决定其垂直层。 

然后，构建步骤对每个轨道使用统一的三段折线。 每个轨道垂直向上进入其层，水平移动到匹配的端点，然后向下返回。 y 坐标间距的倍增因子确保不同层即使在中间顶点也不会接触。 

一个微妙的实现细节是确保深度分配反映真实的嵌套顺序，而不是同一位置的事件的任意顺序。 为了保持正确性，必须一致地对事件进行排序并“先打开后关闭”处理。

 ## 边缘情况

 对于完全嵌套的情况，例如`1 2 3 1 2 3`，算法分配不断增加的深度 0、1、2，确保三个独立的垂直层。 堆栈在最大嵌套时增长到大小 3，并且每个轨道都在自己的带中布线，以防止任何接触。 

对于不相交的间隔，例如`1 1 2 2 3 3`，所有深度都变为 0，因此所有轨道共享相同的水平带但占据不同的 x 范围，这意味着它们的水平段永远不会重叠。
