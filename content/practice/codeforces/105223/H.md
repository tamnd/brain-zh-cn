 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每个测试用例描述了由几堆石子组成的游戏状态。 在每一回合中，玩家可以从任何一个非空堆中移走一颗石子，或者执行组合移动，其中恰好选择三个非空堆，所有堆的当前大小都具有相同的奇偶性，并且从这三堆中的每堆中移走一颗石子。 

玩家交替移动，不能移动的玩家就输了，这种情况只有在所有牌堆都空的时候才会发生。 问题是确定第一个移动的玩家（Besher）在最佳玩法下是否强制获胜。 

约束条件很大，所有测试用例中最多有 200,000 个桩，桩尺寸最大为 10^9。 这立即排除了任何游戏玩法或状态探索的模拟。 任何尝试动态建模状态或搜索棋子的解决方案都是不可行的，因为分支因子很大并且深度可以达到棋子的总数。 

唯一能够真正摆脱这些约束的结构是将整个配置压缩成一个小的不变量，很可能涉及奇偶校验或简单的聚合。 

朴素推理的一个微妙的失败案例是假设桩是独立的，因为“从任何桩中取一个”移动。 例如，用桩`[1, 1, 1]`，独立性建议三个独立的单子游戏，但三重奇偶棋立即允许一次移动所有三个棋子，这显着改变了游戏的长度和结构。 任何正确的解决方案都必须考虑到这种相互作用。 

另一个误导性的案例是认为只考虑堆的数量。 例如，`[2, 2, 2]`和`[1, 1, 1]`在计数方面具有相同的结构，但如果试图过于粗略地推理，则未来的奇偶校验行为会有所不同。 

## 方法

 暴力方法会将状态视为堆大小的完整向量，并尝试使用带有记忆的递归来计算获胜/失败的位置。 每个州将分支到`n`单次删除移动和可能的许多三次删除移动，具体取决于奇偶校验组。 即使我们压缩状态，可达配置的数量也是天文数字，因为每个堆都可以独立地从`10^9`到`0`。 这使得任何基于状态的动态编程都是不可能的。 

关键的观察是，每一步都会使棋子总数减少 1 或 3。两者都是奇数，因此每一步都会翻转棋子总数的奇偶性。 这立即将游戏图限制为基于和奇偶校验的两个交替层。 

一旦我们关注总和的奇偶性，结构就会进一步简化。 在所有检查的情况下，特殊的三重奇偶校验移动的存在不会创建超出总奇偶校验的第二个独立不变量，因为它仍然消耗奇数个棋子并且不会引入任何保留奇偶校验或创建循环优势的操作。 它仅将三个独立的单一移除捆绑到一个动作中，而不改变丢失位置的可达性。 

这导致了核心的简化：游戏结果仅取决于石子总数是奇数还是偶数。

 ## 算法演练

 我们将每个测试用例减少为一个整数，即所有堆大小的总和。 

1. 计算所有堆中所有石子的总和。 这是唯一重要的全局量，因为每次移动都会使总和改变一个奇数，从而翻转奇偶校验。 
2. 检查这个和是奇数还是偶数。 平价完全决定了最佳发挥下的获胜者。 
3. 如果总和是奇数，则第一个玩家获胜。 否则，第二个玩家获胜。 

这样做的原因是，每一个合法的举动都会将游戏转变为总棋子数量相反的状态。 由于唯一的最终状态是和为零，即偶数，因此任何奇和状态都必须至少有一个移动，并且最佳游戏减少为迫使对手每回合都进入相反的奇偶校验状态，直到终止。 

## 为什么它有效

 该游戏可以被视为一个有限有向无环图，其中每条边对应于一个合法的移动，并且每条边翻转总和的奇偶校验。 由于最终位置具有偶数奇偶性（总和为零），因此游戏变成了双向结构，获胜位置严格按照奇偶性层交替。 奇偶限制三重移动不会引入额外的不变量来打破这种交替或在奇偶层内创建一个单独的失败类，因此奇偶性完全表征了获胜条件。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        total = sum(arr)
        if total % 2 == 1:
            print("YES")
        else:
            print("NO")

if __name__ == "__main__":
    solve()
```减少后直接实施。 每个测试用例的唯一计算是对数组求和，这在约束下是安全的。 无需跟踪每堆奇偶校验或模拟转换。 

这里的一个常见错误是试图保持奇数堆和偶数堆的计数或模拟三步条件。 这些都不会影响最终决定，因为全局总和已经编码了所有相关信息。 

## 工作示例

 考虑桩所在的输入`[1, 1]`。 

| 步骤| 桩| 总和| 决定|

 总和是偶数，因此预计第一个玩家会输。 事实上，任何一步棋都会将总和减至 1，为第二位玩家留下一颗棋子，后者可以拿走它并获胜。 

现在考虑`[1, 1, 1]`。 

| 步骤| 桩| 总和| 决定|
 | --- | --- | --- | --- |
 | 开始| [1, 1, 1] | 3 | 奇数|

 总和是奇数，所以第一个玩家获胜。 事实上，他们可以立即使用三平棋来消除所有三颗棋子并完成游戏。 

现在考虑`[1, 1, 2]`。 

| 步骤| 桩| 总和| 搬家选择| 下一个总和 |
 | --- | --- | --- | --- | --- |

 所有可能的移动都会导致奇数状态，即下一个玩家获胜，而起始位置失败。 

这些例子表明，在最优推理下，只有总和的奇偶性是稳定的。

 ## 边缘情况

 对于单堆来说，游戏简化为重复移除一颗石子，因此结果纯粹是通过奇偶校验来交替的。 该算法可以正确处理这个问题，因为总和正是桩的大小，并且奇偶校验与单个 take-1 游戏的已知结果相匹配。 

对于技术上可以实现三重移动的配置，例如`[2,2,2]`，算法仍然正确运行。 总和是偶数，因此预测第一个玩家会失败。 即使存在三重棋，它也只是消除了三颗棋子，并没有改变基于平价的结论。 

对于像这样的混合物`[1,1,1,1]`，在不同阶段可以进行多次三重移动，但每次移动仍然会翻转平价。 总和是偶数，因此第一个玩家输了，并且在逐步追踪时任何一步都保留了此分类的正确性。
