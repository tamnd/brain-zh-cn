 ## 解决方案
 ## 问题理解

 我们有一排学生，每个学生都有固定的初始高度。 学校对任何比他们的邻居都矮的学生都不满意。 第一个和最后一个学生可以豁免，因为他们只有一个邻居。 

我们可以通过添加单位块来增加学生的身高或通过删除单位块来降低学生的身高来修改高度。 每增加一个单位的成本`x`，以及每减少一个单位的成本`y`。 目标是制定最终的高度序列，使得内部位置都不是严格的局部最小值，同时最小化总成本。 

一个关键的观察是每个学生都必须满足一个简单的局部约束：对于每个索引`i`从`2`到`n-1`，我们必须确保`h[i] >= min(h[i-1], h[i+1])`作为坏的严格条件是假的。 同样，我们必须避免`h[i] < h[i-1] and h[i] < h[i+1]`。 

约束允许最多`10^5`每个测试用例的学生和`2 × 10^5`所有测试的总计。 任何二次或偶数$O(n \log n)$如果每个测试用例的解决方案具有大量常量，则可能存在超时风险。 这将我们推向线性或近线性贪婪或 DP 公式。 

当许多连续值相等或接近相等时，就会出现微妙的边缘情况。 另一种情况是当成本不对称时：如果增加比减少便宜得多，反之亦然，则简单的本地修复可能会失败，因为修复一个位置可能会在其他地方引入新的违规行为。 

## 方法

 解决这个问题的一个强力方法是重复扫描数组并修复任何严格局部最小值的位置。 如果`h[i]`比两个邻居都小，我们可以增加`h[i]`最多`min(h[i-1], h[i+1])`，或者将一个或两个邻居减少到`h[i]`，选择成本便宜的。 我们将继续这样做，直到不存在任何违规行为为止。 

这在概念上是有效的，因为每个操作都会直接消除至少一个违规行为。 然而，问题是每次修复都会在附近产生新的违规行为，从而迫使重复进行全面扫描。 在最坏的情况下，单个调整会在阵列中来回传播，从而导致$O(n^2)$行为。 

关键的见解是，每个位置只需要考虑与其两个相邻位置的关系，并且最终配置必须消除所有严格的局部最小值。 我们没有重复模拟变化，而是使用局部 DP 解释来计算每个位置的最佳成本贡献：每个中间位置要么升高，要么降低相邻位置，然后我们选择更便宜的一致组合。 

这将问题简化为计算每个三元组的中心不严格小于两侧的成本，并在整个数组中一致地聚合这些约束。 该结构变为线性，因为每个约束仅与相邻约束重叠。

 ## 算法演练

 我们处理数组，同时强制内部索引不成为严格的局部最小值。 其想法是，对于每个位置，决定是应该向上调整还是应该向下调整相邻位置，并一致地考虑最小成本。 

### 步骤

 1.从左到右迭代所有内部位置，处理每个索引`i`作为潜在违规行为的中心，涉及`i-1`,`i`， 和`i+1`。 我们检查是否`h[i]`严格小于两个邻居，因为只有这样它才违反条件。 
2.如果`h[i]`不是严格的局部最小值，我们什么也不做并继续前进。 这是安全的，因为它已经满足本地所需的条件。 
3.如果`h[i]`是一个严格的局部最小值，我们必须解决它。 有两种可能的策略： 增加`h[i]`至少达到`min(h[i-1], h[i+1])`，或减少一个或两个邻居，以便`h[i]`不再是严格意义上的更小。 
4. 计算融资成本`h[i]`来匹配较小的邻居。 这个成本是`(target - h[i]) * x`， 在哪里`target = min(h[i-1], h[i+1])`。 
5. 计算将两个邻居降低到的成本`h[i]`。 这个成本是`(h[i-1] - h[i]) * y + (h[i+1] - h[i]) * y`。 
6. 选择更便宜的选项并应用其成本。 应用后，从概念上更新受影响的值，以便将来的检查反映修改。 
7. 继续扫描，确保调整产生的任何新违规在到达其中心时得到处理。 

### 为什么它有效

 正确性来自于每个约束都是局部的并且仅取决于相邻元素的三元组这一事实。 任何消除严格的局部最小值的修复都不需要重新考虑远处的位置，因为它不会在邻近区域之外引入新的违规行为。 这创建了一个稳定的传播，其中每个位置在一致的方向上精确解析一次，确保满足所有约束而无需重新访问先前的决策。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, x, y = map(int, input().split())
        h = list(map(int, input().split()))

        cost = 0

        for i in range(1, n - 1):
            if h[i] < h[i - 1] and h[i] < h[i + 1]:
                target = min(h[i - 1], h[i + 1])

                raise_cost = (target - h[i]) * x
                reduce_cost = (h[i - 1] - h[i]) * y + (h[i + 1] - h[i]) * y

                if raise_cost <= reduce_cost:
                    cost += raise_cost
                    h[i] = target
                else:
                    cost += reduce_cost
                    h[i - 1] = h[i]
                    h[i + 1] = h[i]

        print(cost)

if __name__ == "__main__":
    solve()
```该解决方案扫描每个测试用例一次，维护高度的工作副本。 核心决策局限于每个三元组。 当检测到违规时，我们会比较提高中心与降低两个邻居的成本。 立即更新数组可确保后续检查看到正确的结构。 

一个微妙的点是，修改邻居可能会影响早期的位置，但因为我们只会将值移动到消除严格的局部最小值，所以我们不需要明确地重新访问早期的索引。 从左到右的扫描确保了最终的一致性。 

## 工作示例

 ### 示例 1

 输入：```
n = 4, x = 5, y = 5
h = [1, 6, 2, 5]
```我们处理索引 2（值 2），它是 6 到 5 之间的严格局部最小值。 

| 我| h[i-1] | h[i-1] h[i]| h[i+1] | h[i+1] 违规| 提高成本| 降低成本 | 行动|
 | ---| ---| ---| ---| ---| ---| ---| ---|
 | 2 | 6 | 2 | 5 | 是的 | (5-2)*5 = 15 | (6-2)*5 + (5-2)*5 = 35 | 提高|

 我们增加`h[2]`到 5，总成本 = 15。最终数组变为`[1, 6, 5, 5]`。 

这表明选择提高中心比减少两个邻居的成本要低，特别是当与邻居的差距很大时。 

### 示例 2

 输入：```
n = 6, x = 2, y = 4
h = [5, 4, 5, 5, 4, 6]
```我们检查索引 2 和 5。 

| 我| h[i-1] | h[i-1] h[i]| h[i+1] | h[i+1] 违规| 提高成本| 降低成本 | 行动|
 | ---| ---| ---| ---| ---| ---| ---| ---|
 | 2 | 5 | 4 | 5 | 是的 | (5-4)*2 = 2 | (5-4)*4 + (5-4)*4 = 8 | 提高|
 | 5 | 5 | 4 | 6 | 是的 | (5-4)*2 = 2 | (5-4)*4 + (6-4)*4 = 12 | 提高|

 两次加注都是最优的，最终成本都是4。 

这表明，当增加成本明显低于减少成本时，算法始终倾向于向上调整。

 ## 边缘情况

 关键的边缘情况是所有三个值都相等，例如`h = [5, 5, 5]`。 不存在严格的局部最小值，因为该条件要求双方严格不平等。 该算法正确地不执行任何操作并返回零成本。 

另一种情况是先稳定后下降，例如`h = [5, 5, 1, 5, 5]`。 中心值是严格的局部最小值并且将被修正。 如果增加成本较小，则提高； 否则两个邻居都会降低。 不管怎样，在一次操作之后，局部违规就会消失，并且不会在其他地方重新出现，因为平台确保了对称性。 

最后一个微妙的情况是交替的高点和低点，例如`h = [10, 1, 10, 1, 10]`。 每个内部低点都是严格的局部最小值，并且每个低点都必须独立解决。 该算法一致地处理每个三元组，确保每个低值都被恰好处理一次，并且总成本是独立校正的总和，而段之间没有干扰。
