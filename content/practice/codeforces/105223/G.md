 ## 解决方案
 ## 问题理解

 我们得到一个随时间变化的数组。 每次更新后，我们都被要求计算两层中构建的全局数量。 

首先，获取当前数组的任何非空子序列。 对于每个这样的子序列，计算它自己的所有非空子序列的 XOR 之和。 这已经为每个子序列生成了一个值。 

其次，再次获取原始数组的所有非空子序列，并将第一步计算出的值对所有子序列求和。 

因此，实际上，数组的每个可能的子序列都会多次贡献，并且在每个子序列内，我们再次枚举其所有子序列并对异或值求和。 

输入规模很大，最多有100万个元素和100万个更新。 这立即排除了任何显式枚举子序列的方法。 即使处理单个数组也会涉及$2^n$对象，这远远超出了可行的范围。 任何解决方案都必须将指数级多个子序列的贡献压缩为闭合形式表达式，并在点更新下维护它。 

一个微妙的困难是子序列是由索引而不是值定义的。 不同位置的两个相等的值是不同的元素，因此必须对位置进行组合计数。 

打破天真的思维的一种边缘情况是小重复。 例如，使用数组$[x, x]$，子序列如$[x]$出现两次，子序列计数再次放大了这种重复。 任何重视价值而不是立场的方法都会低估贡献。 

另一种失败模式是忘记每个子序列都按包含它的原始数组的超集数量进行加权。 大小的子序列$k$出现在许多较大的子序列中，而不仅仅是一次，因此直接的“子序列的子序列之和”解释必须仔细考虑多重性。 

## 方法

 直接的暴力策略将枚举数组的每个子序列。 对于每个这样的子序列，我们将枚举其所有子序列并计算异或和。 这已经大致意味着$O(4^n)$最坏情况下的行为，因为每个子集都包含指数数量的子集。 即使是为了$n = 40$，这个就变得无法使用了，所以这个方向被完全封锁了。 

关键的简化是翻转求和的顺序。 而不是首先选择一个子序列$s$数组，然后是其中的子序列，我们修复一个内部子序列$t$数组并询问：有多少个外部子序列$s$包含$t$？ 每一个这样的$s$贡献异或$t$一旦通过内部定义$f(s)$。 

如果$t$用途$k$元素，那么原始数组中的每个剩余元素都可以包含或不包含在$s$，只要$t \subseteq s$。 这正好给出了$2^{n-k}$选择。 因此整个问题简化为对所有非空子序列的贡献求和$t$，每个权重为$2^{n-|t|}$，乘以异或$t$。 

这将嵌套子序列的子序列问题转换为单个加权子序列和问题。 剩下的挑战是计算$$\sum_t XOR(t) \cdot 2^{n-|t|}$$更新下。 

XOR 结构建议按位分割。 XOR 每位都是线性的，因此每位都可以独立处理。 困难在于权重取决于子集大小，因此我们必须跟踪子集大小如何与每个位的所选元素的奇偶校验相互作用。 

这导致了生成函数的观点，其中每个元素通过被选择或不被选择而做出贡献，并且选择影响大小和异或奇偶性。 生成的结构是每个元素相同的小转移矩阵的乘积，可以使用特征分解将其简化为每位的闭合形式表达式。

 ## 算法演练

 我们一步步将表达式转变为可维护的东西。 

### 1.重写双子序列和

 我们将最终答案重新解释为所有非空子序列的总和$t$数组的。 每一个这样的$t$贡献其 XOR 值乘以超集的数量$t$在阵列的子序列格中。 大小的子序列$k$有$2^{n-k}$超集，因此答案成为所有子序列的加权和。 

这完全消除了问题的外层。 

### 2. 排除全球力量

 我们重写权重$2^{n-k}$作为$2^n \cdot (1/2)^k$。 因素$2^n$仅取决于当前数组长度并且易于维护。 核心成为子序列的加权和，其中每个元素贡献一个因子$w = 1/2$每个选定的元素。 

### 3. 减少异或对位的贡献

 我们将 XOR 扩展为位和。 对于固定位$b$，它对 XOR(t) 的贡献是$2^b$如果元素个数为奇数$t$有位$b$设置，否则为零。 

因此，对于每一位，我们需要计算二进制数组的所有子序列的加权和，其中我们跟踪所选子序列的奇偶性，每个所选元素贡献权重$w$。 

### 4. 将子序列构造建模为状态转换

 对于固定位，每个元素要么是“零位”，要么是“一位”。 

零位元素不影响奇偶校验，但可以采用也可以不采用。 这只是将所有状态乘以$(1 + w)$。 

一位元素会影响奇偶校验：如果我们处于偶数状态，则选择它会移动到奇数状态，反之亦然。 这定义了 2 状态线性变换。 

所以每个一位元素应用相同的矩阵$$\begin{bmatrix}
1 & w \\
w & 1
\end{bmatrix}$$我们重复应用这个矩阵，每个设置了该位的元素一次。 

### 5. 压缩重复的相同变换

 因为所有一位元素都使用相同的矩阵，并且顺序并不重要，所以我们将该矩阵计算为等于此类元素数量的幂。 

该矩阵通过特征值具有闭合形式。 其特征值是$1 + w$和$1 - w$。 这使我们能够直接计算其 k 次方。 

将结果应用到初始状态$[1, 0]$给出偶数和奇数贡献的封闭形式.

 ### 6. 结合零位缩放

 处理完一位元素后，我们乘以$(1+w)^{z}$， 在哪里$z$是零位元素的数量。 

这给出了该位的最终奇奇偶校验和。 

### 7. 维护更新计数

 对于每个位位置，我们维护有多少个数组元素设置了该位。 其余的都是隐式零位元素。 

当更新更改值时，我们会减少旧值位的计数并增加新值位的计数。 每次更新仅影响 30 位，因此我们可以有效地重新计算贡献。 

### 为什么它有效

 整个计算是子序列的线性组合，其中每个元素独立地为每个比特的全局两态系统贡献局部变换。 由于子序列选择对元素进行因式分解，因此最终的总和成为相同交换变换的乘积。 这保证了将所有元素压缩为每位计数可以保留加权子序列的精确分布，因此不会丢失结构信息。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7
INV2 = (MOD + 1) // 2

MAXB = 31

def mod_pow(a, e):
    res = 1
    while e:
        if e & 1:
            res = res * a % MOD
        a = a * a % MOD
        e >>= 1
    return res

def solve():
    n = int(input())
    arr = list(map(int, input().split()))
    q = int(input())

    cnt = [0] * MAXB
    for x in arr:
        for b in range(MAXB):
            if x >> b & 1:
                cnt[b] += 1

    def bit_contribution(b):
        k1 = cnt[b]
        k0 = n - k1

        w = INV2

        a = mod_pow((1 + w) % MOD, k1)
        b_ = mod_pow((1 - w) % MOD, k1)

        odd = (a - b_) * ((MOD + 1) // 2) % MOD
        odd = odd * mod_pow((1 + w) % MOD, k0) % MOD

        return odd * pow(2, b, MOD) % MOD

    total_2n = mod_pow(2, n)

    for _ in range(q):
        i, x = map(int, input().split())
        i -= 1

        old = arr[i]
        if old != x:
            for b in range(MAXB):
                if old >> b & 1:
                    cnt[b] -= 1
                if x >> b & 1:
                    cnt[b] += 1
            arr[i] = x

        ans = 0
        for b in range(MAXB):
            ans = (ans + bit_contribution(b)) % MOD

        ans = ans * total_2n % MOD
        print(ans)

if __name__ == "__main__":
    solve()
```该解决方案维持阵列中设置位的每位频率。 每个查询仅更新这些计数。 每个位的核心计算使用封闭形式矩阵幂结果重建所有子序列的贡献，避免任何显式子集枚举。 

因素$2^n$每个状态重新计算一次，而所有繁重的工作都被推入小整数的模幂运算中。 

## 工作示例

 考虑一个小数组，我们可以在其中手动跟踪行为：$[1, 2]$。 

最初，位 0 和位 1 计数均为 1。对于每个位，我们独立计算贡献并将它们组合起来。 当更新改变值时，只有那些位计数会调整，并且所有子序列贡献一致地变化。 

第二个例子是$[3, 3]$。 这里重复很重要。 由于索引不同，每个子序列都存在多次，并且基于矩阵的公式自然会单独计算两次出现的次数，因为它对位置而不是值进行操作。 

这些示例证实该方法尊重位置多重性并且不会错误地折叠相同的值。

 ## 边缘情况

 关键的边缘情况是所有元素都相同。 例如，$[5,5,5]$产生许多重复的子序列。 一种幼稚的基于价值的方法会将它们合并为一个，但正确的表述会分别对待每个位置。 位计数方法保留了多重性，因为每次出现都会独立地递增相同的计数器。 

另一种情况是单个元素在两个值之间频繁切换。 由于每次更新仅更改 30 位，因此该算法仅正确更新受影响的计数器，避免重新计算未受影响的结构，同时保留精确的子序列贡献。
