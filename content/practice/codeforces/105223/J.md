 ## 解决方案
 ## 问题理解

 我们有一个轴对齐的正方形，里面有几条直线段，每条直线段要么是水平的，要么是垂直的。 这些线段严格位于正方形内部，但它们可能重叠甚至重合。 我们被要求选择两个不同的线段，这样，如果我们将两个线段延伸成无限条线，那么这两条线将正方形划分为一组较小的轴对齐矩形，并且在这些生成的矩形中，存在一个恰好出现两次的矩形形状值。 

如果两个矩形可以旋转 180 度或翻转，则认为两个矩形是相同的，这意味着只有边长重要，而不是方向。 

从几何角度来看，延伸两个轴对齐的线段意味着我们在正方形内添加两条完整的水平或垂直切割线。 任何一对这样的线都会将正方形最多分为六个或九个矩形区域，具体取决于它们是否相交或平行。 问题中的条件不是关于形成多少个区域，而是关于生成的矩形形状的多重集：我们希望只有一种形状类型出现两次，而所有其他形状最多出现一次。 

输入大小高达 10 万个段，因此任何尝试所有段对的解决方案都立即不可行，因为它需要 n 平方运算，在最坏的情况下约为 100 亿次。 这远远超出了典型的两秒限制。 

关键的困难在于，两个选定线段的效果仅取决于它们的支撑线，而不取决于它们的确切端点。 任何线段都定义垂直线 x = c 或水平线 y = c。 所以多条线段可以对应同一条有效切割线。 这种冗余很重要：选择具有相同坐标的不同线段会产生相同的几何分区。 

一个幼稚的错误是独立处理段而不将它们压缩到唯一的 x 或 y 位置。 另一个微妙的陷阱是假设所有段都是有用的； 位于同一条线上的许多线段是可以互换的，只有它们的线坐标才重要。 

## 方法

 暴力方法会检查每对片段。 对于每一对，我们将它们扩展到整行并计算正方形的结果分区，然后计算矩形形状并检查两个矩形是否具有相同的尺寸。 每次检查都需要计算与正方形边界的交集并导出恒定数量的矩形，因此每对的成本为 O(1)。 然而，有 O(n^2) 对，这导致大约 10^10 次操作，这是不可行的。 

关键的观察结果是，分区的结构仅取决于我们选择垂直线还是水平线以及它们的坐标。 一旦两条线被固定，所产生的矩形尺寸由方形边框和这些线之间的距离确定。 因此，唯一重要的是是否存在两条线，其相对位置产生重复的矩形区域配置。 

该问题简化为一次推理一维中的成对切割。 垂直切割仅取决于其 x 坐标，水平切割仅取决于其 y 坐标。 因此，我们可以将片段分为垂直组和水平组，并独立分析每种类型。 

对于垂直线，对它们的 x 坐标进行排序可以让我们推断出相邻或相同的位置。 这同样适用于水平线。 精确地生成一个重复的矩形形状的条件简化为找到一对相同位置的切口或强制形成一个重复的矩形的非常特定的对称配置。

事实上，要在正方形内两个轴对齐切割的所有结果块中获得完全相同的两个矩形，唯一的方法是当两个选定的线段对应相同的方向并引起对称分割时，当我们选择对齐的两个线段以使它们的投影创建等宽或等高的分区时，就会发生这种情况。 这会分解为搜索共享相同坐标（x 或 y）的两个线段。 如果任何坐标在线段中至少出现两次，则选择具有该坐标的任意两个线段即可满足条件，因为两次切割重合并沿相反轴创建重复的矩形条带。 

因此，问题简化为检测垂直线段的 x 坐标或水平线段的 y 坐标中的重复项。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解对 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |
 | 坐标分组| O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 阅读所有片段并将每个片段分类为垂直或水平。 如果 x1 等于 x2，则线段是垂直的，否则线段是水平的。 这种分类是必要的，因为只有轴对齐方向对于诱导切割线很重要。 
2. 对于垂直线段，提取其 x 坐标。 对于水平线段，提取其 y 坐标。 存储按这些坐标分组的索引。 
3. 插入坐标时，检查之前是否已经出现过任何坐标。 如果我们看到垂直线段重复的 x 或水平线段重复的 y，我们立即得到一个有效的答案，并且可以输出任意两个相应的线段索引。 
4. 如果垂直或水平坐标中不存在重复项，则断定不存在有效对并输出 -1 -1。 

提前停止背后的原因是，第一个重复坐标已经保证了两个不同的段，从而产生相同的全线切割。 

### 为什么它有效

 一旦一段被延伸，只有它的支撑线才重要。 如果两个不同的线段位于同一条垂直线上 x = c，则选择它们会产生两个相同的垂直切割，这不会改变分区，这与在不同表示中选择单个切割两次不同。 这保证了分解后的矩形在水平方向上包含重复的相同区域，满足所有形成的矩形中恰好有两个相等的矩形的要求。 满足条件的任何有效配置都必须引入重复的有效切割线，因为在没有重复切割位置的情况下，所有生成的矩形尺寸通过轴对齐网格的构造而不同。 因此检测任何重复的坐标既是必要的也是充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, xs, ys, xe, ye = map(int, input().split())

    seen_v = {}
    seen_h = {}

    for i in range(1, n + 1):
        x1, y1, x2, y2 = map(int, input().split())

        if x1 == x2:
            x = x1
            if x in seen_v:
                print(seen_v[x], i)
                return
            seen_v[x] = i
        else:
            y = y1
            if y in seen_h:
                print(seen_h[y], i)
                return
            seen_h[y] = i

    print(-1, -1)

if __name__ == "__main__":
    solve()
```该代码首先通过比较端点来分离垂直和水平段。 对于垂直段，它使用将 x 坐标映射到生成它的第一个段索引的字典。 对于水平线段，它对 y 坐标执行相同的操作。 当遇到重复坐标时，它会输出两个线段索引。 

一个微妙的实现细节是我们只存储每个坐标的第一次出现。 这已经足够了，因为该问题只需要任何有效的对。 另一个重要的一点是，线段端点与确定方向和位置无关，因此我们不会对端点进行标准化或重新排序。

 ## 边缘情况

 一个微妙的边缘情况是多个段位于同一行但写入相反的端点。 例如，`1 5 1 9`和`1 9 1 5`仍然代表相同的垂直线 x = 1。算法可以正确处理这个问题，因为它只检查`x1 == x2`，完全忽略排序。 

另一种情况是多次重复：如果三个或更多段共享相同的坐标，则第一次重复检测已经触发有效答案。 例如：```
3 0 0 10 10
2 1 2 9
2 3 2 8
2 4 2 7
```第二段已经与第一段匹配，因此输出是立即的`1 2`，即使存在第三个。 

最后一种情况是所有线段都具有混合方向但仍然不存在重复坐标。 在这样的输入中，算法正确返回`-1 -1`因为没有两个线段定义相同的有效切割线，因此不会出现诱导矩形结构中的重复。
