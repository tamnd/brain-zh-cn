 ## 解决方案
 ## 问题理解

 我们收到一系列字母，其中每个字母都已分配给目的地公司。 同时，我们还有信封，每个信封上都写着固定的公司，匹配相同的顺序结构。 阿米尔拥有的关键自由是，他可以在信封上任意排列字母，只要每个信封都收到一封信即可。 

我们必须遵守的限制是，将信件放入信封后，任何信件都不应放入其公司与信件的目的地公司相匹配的信封中。 换句话说，如果一封信是写给公司的`x`，它不能放入任何贴有标签的信封中`x`。 

因此，问题简化为检查给定的字母多重集是否存在排列，使得每个字母都被放置到标签与其自身目标值不同的位置。 

输入大小可达`n = 100000`，这立即排除了对排列的任何阶乘或指数推理。 任何涉及显式构造或排列检查的事情都是不可能的。 我们需要一个可以仅使用标签分布的计数或结构属性来验证的条件。 

当所有字母都属于同一家公司时，就会出现微妙的边缘情况。 在这种情况下，每个信封也属于同一家公司（因为该公司必须至少出现一次，但信封在位置上相对应），并且每个位置都不可避免地在禁止的意义上是“正确的”。 例如：```
3 1
1 1 1
```输出必须是`NO`，因为每个字母都是相同的，并且每个位置都使它们匹配。 

另一种边缘情况是，有多家公司，但一家公司占据主导地位。 例如，如果一家公司出现了一半以上的职位，我们可能会怀疑完全避免自我放置是不可能的，因为没有足够的“外国”职位来容纳该标签的所有出现。 

## 方法

 暴力方法会尝试将字母分配给信封并检查分配的所有排列。 从概念上讲，我们会将每个字母视为一个项目，将每个信封视为一个槽，然后尝试所有双射并验证是否有任何双射避免固定点。 这有助于正确性，因为它探索了所有可能的安排，但它的增长随着`n!`，这对于`n = 100000`是完全不可行的。 

关键的见解是停止考虑个体排列，而是关注每个公司标签的计数。 给定公司的每个字母都必须放入属于不同公司的信封中，因此标签的每次出现都必须“避免”自己的类别。 限制因素是剩余标签是否提供足够的容量来托管这些避免的展示位置。 

如果一家公司出现太多次，特别是超过所有信件的一半，那么即使我们尝试以最佳方式分配展示位置，也不会有足够的不匹配信封可用于分隔所有出现的情况。 相反，如果没有标签超过一半`n`，那么始终可以通过在不同标签之间以循环或贪婪移位方式重新排列元素来构造有效的类似混乱的分配。 

因此，整个问题归结为检查任何公司标签的最大频率是否最多`n - max_frequency`。 同样，我们只需要确保没有任何标签占据主导地位。 在这个特定的结构中，条件简化为检查最大频率是否至多`n - largest_frequency`，除非单个标签占据所有位置或违反类似混乱的排列所需的平衡，否则它始终成立。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n!) | O(n) | 太慢了 |
 | 频率检查 | O(n) | O(米) | 已接受 |

 ## 算法演练

 我们通过将其简化为频率约束检查来解决该问题。 

1. 计算每个公司在列表中出现的次数。 这告诉我们有多少封信想要发送到每个目的地类别。 
2. 找出所有公司中的最大频率。 这确定了最“要求”的群体，也是最有可能引发冲突的群体。 
3. 将此最大频率与其余字母进行比较。 如果最常出现的公司占据了所有位置，或者同等地，如果不可能将其出现分配到其他公司的信封中，我们就得出失败的结论。 
4. 否则，得出成功的结论，因为剩余的公司提供了足够的位置来排列字母，以便没有字母留在其自己的类别中。 

此步骤背后的原因是，当没有单个类压倒可用替代槽系统时，就存在有效的分配。 

### 为什么它有效

 将每个公司标签视为一种颜色。 我们希望将每个有颜色的项目放置到不是其自身颜色的位置。 唯一的障碍是当一种颜色出现得如此频繁，以至于其他颜色无法提供足够的“安全”位置。 如果最大的组太大，那么即使是最好的重新排列仍然会迫使至少一个元素留在自己的组内。 如果不是太大，我们总是可以将主导群体的元素分布在属于其他群体的位置上，然后类似地完成剩余的放置。 正确性取决于这样一个事实：只有最大频率才会产生瓶颈； 所有其他发行版都可以进行洗牌，而无需创建固定分配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, m = map(int, input().split())
arr = list(map(int, input().split()))

freq = {}
for x in arr:
    freq[x] = freq.get(x, 0) + 1

mx = max(freq.values())

# if one company dominates too much, impossible
if mx > n - mx:
    print("NO")
else:
    print("YES")
```该解决方案首先构建所有公司标签的频率图。 这是必要的，因为问题的结构完全取决于多重性，而不是位置。 计算出最大频率后，我们直接应用从紊乱式约束导出的可行性条件。 

关键的实现细节是我们不尝试构造排列。 任何建设性方法都存在复杂性和边缘情况错误的风险。 相反，我们纯粹依赖组合界限。

## 测试用例```python
import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    import sys as _sys
    input = _sys.stdin.readline

    n, m = map(int, input().split())
    arr = list(map(int, input().split()))

    freq = {}
    for x in arr:
        freq[x] = freq.get(x, 0) + 1

    mx = max(freq.values())
    return "YES\n" if mx <= n - mx else "NO\n"

# provided samples
assert run("3 1\n1 1 1\n") == "NO\n"
assert run("4 4\n4 1 2 3\n") == "YES\n"

# custom cases
assert run("1 1\n1\n") == "NO\n", "single element must fail"
assert run("2 2\n1 2\n") == "YES\n", "swap works"
assert run("5 2\n1 1 1 2 2\n") == "YES\n", "balanced mix"
assert run("6 2\n1 1 1 1 2 2\n") == "NO\n", "dominant class"
```| 测试输入| 预期产出 | 它验证了什么 |
 | --- | --- | --- |
 |`1 1 / 1`| 否 | 最小尺寸，强制自匹配|
 |`2 2 / 1 2`| 是 | 最简单的有效交换 |
 |`5 2 / 1 1 1 2 2`| 是 | 不平凡的可行组合|
 |`6 2 / 1 1 1 1 2 2`| 否 | 主频块解决方案|

 ## 边缘情况

 对于像这样的单字母场景`n = 1`，唯一可能的位置是固定的，因此算法正确返回 NO，因为最大频率等于`n`。 

对于高度倾斜的分布，例如`1 1 1 1 2 2`，频率`1`是 4，而补码只有 2。检查`mx > n - mx`触发器，正确识别没有足够的非`1`槽以避免自行放置。 

对于平衡的情况，例如`1 2 3 4`，所有频率均为 1，所以`mx = 1`和`n - mx = 3`，允许有效的排列。 该算法接受这些情况，而无需构造显式分配，仅依赖于从容量约束导出的可行性条件。
