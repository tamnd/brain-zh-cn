 ## 解决方案
 ## 问题理解

 我们得到一个整数数组，并要求计算有多少个连续的子数组具有特殊属性：如果将子数组内的所有元素相乘，则所得的乘积具有奇数个除数。 

这个条件背后的关键数学事实是，当且仅当一个数字是完全平方数时，它的约数为奇数。 除数通常成对出现，但对于完美平方，一对除数会分解为单个重复因子，从而产生奇数。 

因此，任务简化为计算乘积为完全平方数的子数组。 

数组元素为正数，最多 300 个，测试用例中的总元素数最多为 10^6。 该大小立即排除了任何显式检查所有子数组的解决方案。 每个测试用例的二次枚举将尝试大约 n^2 / 2 个子数组，在最坏的情况下，这会变成 10^10 次操作，远远超出任何可行的限制。 

产品增长的速度会产生一个微妙的边缘情况。 例如，即使像 [2, 2, 2, 2] 这样的小数组也已经产生了许多很大的子数组乘积，但我们实际上并不直接计算这些乘积。 如果重复执行，任何基于乘法的简单检查都会溢出或变慢。 

另一个重要的边缘情况是当所有元素均为 1 时。每个子数组的乘积为 1，它是完全平方数，因此答案变为 n(n+1)/2。 任何正确的解决方案都必须自然地处理这个问题，而无需特殊的外壳。 

## 方法

 暴力解决方案会尝试每个子数组，计算其乘积，并检查该乘积是否是完全平方数。 即使我们通过扩展正在运行的产品来避免从头开始重新计算产品，我们仍然对每个测试用例执行 O(n^2) 更新。 由于元素总数高达 10^6，在最坏的情况下这会导致大约 5×10^11 次操作，这是不可行的。 

关键的见解是我们永远不需要产品的数值。 我们只关心产品是否是完美的正方形。 当一个数因式分解中的每个素数指数都是偶数时，该数就是完全平方数。 这将问题从乘法转移到跟踪质数指数的奇偶性。 

由于每个a[i]最多为300，因此其素数分解只涉及300以内的素数，并且每个数字可以用哪些素数出现奇数次来表示。 我们只跟踪奇偶校验，因此每个元素都可以编码为素数上的位掩码，其中如果该素数在前缀乘积中出现奇数指数，则该位为 1。 

现在，当且仅当这些位掩码在子数组上的异或为零时，子数组的乘积才是完全平方。 这将问题转化为对 XOR 等于 0 的子数组进行计数，这是标准前缀 XOR 频率问题。 

我们维护前缀 XOR 掩码并计算每个掩码出现的频率。 每对相等的前缀掩码定义一个子数组，其 XOR 为零，因此其乘积是完全平方数。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n^2 log A) | O(n^2 log A) | O(1) | O(1) | 太慢了|
 | 前缀异或频率| O(n log A + P) | O(n log A + P) | O(P)| 已接受 |

 这里 P 是不同质数掩码状态的数量，以 n 为界。 

## 算法演练

 我们将每个数字转换为仅捕获质数指数奇偶性的表示，然后使用前缀累加。

1. 预先计算最多 300 个素数，并为每个素数分配一个索引。 这使我们能够构建用于因式分解的紧凑位掩码。 
2. 对于每个数组元素，对其进行因式分解并构造一个位掩码，其中每个位对应于该素数在其因式分解中是否出现奇数次。 由于我们只关心奇偶校验，因此重复除法会切换位而不是递增计数。 
3. 维持初始化为零的运行前缀掩码。 这表示从开始到当前位置的所有编码值的异或。 
4. 使用哈希映射或字典来计算每个前缀掩码出现的次数。 使用频率为 1 的空掩码对其进行初始化。 
5. 当我们处理每个元素时，通过将其与元素的掩码进行异或来更新前缀掩码。 每次我们达到前缀掩码值时，将其当前频率添加到答案中。 这会计算出现相同掩码的所有先前位置，形成以当前索引结尾的有效子数组。 
6. 增加当前前缀掩码的频率。 

步骤 5 正确的原因是两个相等的前缀掩码意味着它们之间的段的 XOR 为零，这意味着所有素数指数抵消为偶数，这正是完美平方积的条件。 

### 为什么它有效

 该算法保持一个不变式，即位置 i 处的前缀掩码表示前 i 个元素的乘积中素数指数的奇偶性。 当 l−1 和 r 处的前缀掩码相同时，从 l 到 r 的子数组乘积恰好是完全平方，因为 XOR 消除确保所有素数指数都是偶数。 因此，计算相同的前缀掩码只会对每个有效子数组进行一次计数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

# Precompute primes up to 300
MAXV = 300
is_prime = [True] * (MAXV + 1)
is_prime[0] = is_prime[1] = False
primes = []
for i in range(2, MAXV + 1):
    if is_prime[i]:
        primes.append(i)
        for j in range(i * i, MAXV + 1, i):
            is_prime[j] = False

prime_index = {p: i for i, p in enumerate(primes)}

def factor_mask(x):
    mask = 0
    for p in primes:
        if p * p > x:
            break
        if x % p == 0:
            cnt = 0
            while x % p == 0:
                x //= p
                cnt ^= 1
            if cnt:
                mask ^= (1 << prime_index[p])
    if x > 1:
        mask ^= (1 << prime_index[x])
    return mask

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        freq = {0: 1}
        pref = 0
        ans = 0

        for v in a:
            pref ^= factor_mask(v)
            ans += freq.get(pref, 0)
            freq[pref] = freq.get(pref, 0) + 1

        print(ans)

if __name__ == "__main__":
    solve()
```因式分解步骤为每个数字构造奇偶校验掩码。 前缀变量累积这些掩码的异或，有效地跟踪数组当前前缀的乘积中素数指数的奇偶校验。 

频率字典存储每个前缀状态出现的次数。 当前缀重复时，每个较早的出现都会形成一个以当前位置结尾的有效子数组。 

一个常见的实现陷阱是忘记将零掩码的频率初始化为 1，这是计算从索引 0 开始的子数组所必需的。

 ## 边缘情况

 对于像这样的输入`[1]`，前缀掩码始终为零，因此频率图立即计数一个子数组。 该算法从频率开始`{0:1}`，因此在处理单个元素时，前缀保持为零，并且恰好贡献一个有效子数组，该子数组与正确答案匹配。 

对于像这样的输入`[2, 2, 2, 2]`，每个元素切换相同的素数位。 前缀掩码在两种状态之间交替。 前缀状态的每次重复都会贡献多个子数组，并且频率表正确地对所有相等状态对进行计数。 例如，当前缀在偶数位置返回到零时，它会计算乘积为完全平方数的子数组，因为 2 的指数在这些段中是偶数。
