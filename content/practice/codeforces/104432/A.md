 ## 解决方案
 ## 问题理解

 我们得到了几个独立的测试用例。 每一个都有一个正整数列表。 我们可以在这个列表中插入一个额外的整数。 目标是选择这个插入的值，使得插入后整个集合的最大公约数恰好为1。在所有有效选择中，我们必须选择最小的非负整数，但限制插入的值不能为1。 

该任务真正要问的是如何使用一个精心选择的数字“修复”整个数组的 gcd，并以最小化该数字的方式执行此操作。 

这些约束允许所有测试用例的元素总数最多为 10^5，值最大为 10^18。 这立即排除了任何尝试针对每个测试用例的多种可能性模拟所有数组元素的候选插入的解决方案。 在数组上计算 gcd 很便宜，但如果不小心的话，在没有结构的情况下重复尝试每个元素的许多候选者会太慢； 在这里我们将看到该结构将问题折叠成一个非常小的搜索。 

有一些微妙的边缘情况很重要。 

如果所有数字都相同且大于 1，则 gcd 就是该数字。 例如，像 [6, 6, 6] 这样的数组的 gcd 为 6。如果我们尝试插入 x = 0，则 gcd 变为 gcd(6, 0) = 6，这没有帮助。 如果我们尝试 x = 2，gcd(6, 2) = 2，仍然不是 1。只有与 6 互质的数字才有效，并且必须找到最小的此类数字。 

如果数组已经有 gcd 1，那么插入 0 是有效的，因为 gcd(1, 0) = 1。这使得 0 成为最小的可能答案，并且它是被允许的，因为只有 1 是被禁止的。 

粗心的方法会尝试独立地为每个候选 x 重新计算整个数组的 gcd。 由于 x 原则上可能会变大，因此这会浪费时间并忽略只有原始数组的 gcd 实际上很重要的事实。 

## 方法

 蛮力的想法很简单。 对于每个测试用例，计算数组的 gcd，然后尝试从 0 开始增加 x 的值，跳过 1，并检查插入 x 是否使总体 gcd 等于 1。每次检查都需要计算 gcd(g, x)，其中 g 是数组的 gcd。 这是有效的，因为整个数组加上 x 的 gcd 恰好是 gcd(g, x)。 

这种粗暴的方法是正确的，但如果我们想象在没有洞察力的情况下扩展它，可能会感觉我们需要测试 x 的许多值并重复与整个数组组合。 如果我们每次都重新计算 n 个元素的 gcd，那么这将变得昂贵。 然而，一旦我们将数组压缩为单个 gcd 值，每次检查就会变成 O(1)，瓶颈就消失了。 

关键的观察是整个数组相当于 gcd 运算下的单个数字 g。 该问题简化为找到最小的 x，使得 gcd(g, x) = 1 且 x ≠ 1。由于 x = 0 仅在 g = 1 时有效，因此其余搜索是在从 2 开始的非常小的整数上进行。 因为实际上 g 相对于其大小最多有几个小的质因数，所以第一个互质数很快就会出现。 

这将问题简化为每个测试用例计算一个 gcd，然后扫描一小部分整数前缀。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对每个候选 x 的数组进行暴力破解 | O(n·X) | O(n·X) | O(1) | O(1) | 太慢了 |
 | 减少到 gcd 然后尝试小 x | O(n + K) | O(1) | O(1) | 已接受 |

 这里 K 是在找到互质值之前检查的小整数的数量，在实践中它受到一个非常小的常数的限制。 

## 算法演练

 我们首先减少数组，然后搜索最小的有效插入值。

1. 计算数组中所有元素的gcd。 这将所有结构压缩为单个数字 g，因为 gcd 是结合律和交换律的。 
2. 如果g等于1，那么数组已经有gcd 1，没有任何插入。 允许的非 1 之外的最小非负整数是 0，插入 0 会使 gcd 保持为 1 不变。 
3. 如果g大于1，我们必须找到从0开始向上的最小整数x，不包括1，使得gcd(g, x)等于1。 
4. 在这种情况下立即跳过 x = 0，因为 gcd(g, 0) = g，当 g > 1 时，它不是 1。 
5. 从 x = 2 开始向上检查。 对于每个候选者，计算 gcd(g, x)。 该 gcd 变为 1 的第一个值就是答案。 
6. 一旦找到这样的 x 就停止，因为我们正在按升序扫描并且需要最小值。 

### 为什么它有效

 整个数组仅通过其 gcd g 做出贡献，因为 gcd(a1, a2, ..., an, x) 简化为 gcd(g, x)。 因此，任何固定结果的 x 选择都必须与 g 互质。 当 g 已经是 1 时，最小的有效整数是 0，或者是从 2 开始且不与 g 共享任何质因数的第一个整数。 因为 gcd 约束仅取决于素数因子，一旦我们达到一个数字 x，它引入了一个不整除 g 的新素数，则该 x 的 gcd 永久变为 1。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
import math

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        arr = list(map(int, input().split()))
        
        g = 0
        for v in arr:
            g = math.gcd(g, v)
        
        if g == 1:
            out.append("0")
            continue
        
        x = 2
        while True:
            if math.gcd(g, x) == 1:
                out.append(str(x))
                break
            x += 1
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案首先将数组折叠成单个 gcd 值。 这是对所有未来推理唯一重要的信息。 g = 1 的分支处理零的特殊角色：它保留 gcd 1 并且是允许的最小数字。 

对于 g > 1，我们从 2 开始执行简单的递增扫描。 gcd 检查是恒定时间且安全的，因为我们不再需要考虑与单个数组元素的交互。 

该循环在代码中故意不受限制，但从数学上讲，它很快终止，因为非常小的整数几乎总是用 g 破坏 gcd 结构。 

## 工作示例

 ### 示例 1

 输入数组：[2,1,4]

 我们一步步计算数组的 gcd。 

| 步骤| 价值| 当前gcd |
 | --- | --- | --- |
 | 开始| - | 0 |
 | 阅读 2 | 2 | 2 |

 | 阅读 4 | 4 | 1 |

 由于最终的 gcd 是 1，所以我们立即输出 0。插入 0 不会改变 gcd，因为 gcd(1, 0) = 1。 

此示例显示了数组已经与 gcd 1“完全兼容”的分支，除了插入允许的最小数字之外不需要任何更正。 

### 示例 2

 输入数组：[9,33,3,11]

 | 步骤| 价值| 当前gcd |
 | --- | --- | --- |
 | 开始| - | 0 |
 | 阅读 9 | 9 | 9 |
 | 阅读 33 | 33 | 33 3 |

 | 阅读 11 | 11 | 11 1 |

 等等，在这种情况下，gcd 变为 1，因此根据规则我们将输出 0。这与预期的第二个样本行为相匹配。 

该迹线表明，即使数字看起来高度结构化，单个相对素数元素也可以将 gcd 折叠为 1，从而立即激活零情况。

 ## 边缘情况

 当数组gcd已经为1时，插入0是有效的并且严格最优。 例如，输入 [3, 5, 7] 立即折叠为 gcd 1，因此算法不得尝试任何正候选。 

当数组 gcd 大于 1 并且偶数时，小偶数候选会立即失败。 例如，如果 g = 8，则 x = 2、4、6、8 都会失败，因为它们共享因子 2。该算法通过递增 x 直到达到与 g 互质的数字（如 3 或 5）来正确地跳过它们。 

当所有数字都相同时，例如 [12, 12, 12]，gcd 仍为 12，算法从 2 向上搜索，直到找到第一个不与 12 共享质因数的整数。在这种情况下，x = 5 会立即工作，因为 gcd(12, 5) = 1，并且扫描提前终止。
