 ## 解决方案
 ## 问题理解

 我们得到一棵树，意味着一个没有循环的连通图。 这棵树的一些顶点是叶子，这些顶点的度数为一。 任务是测量每个顶点距离最近的叶子有多远，然后确定哪个顶点距离任何叶子最远，并将该距离与按自然处理顺序出现该最大值的第一个顶点一起输出。 

从概念上讲，想象每片叶子在零时间开始发出信号，并且该信号沿着树每单位时间传播一个边缘。 每个顶点都会记录信号到达它的最早时间。 一旦传播完成，我们扫描所有顶点并选择记录到达时间最长的顶点，通过选择最小的索引来打破平局。 

输入大小意味着在典型的 Codeforces 约束中树最多具有大约 100,000 个顶点。 任何具有二次行为的解决方案都将涉及 100 亿次运算，并且立即不可行。 甚至$O(n \log n)$在这里是可以接受但不必要的开销。 树的结构与均匀传播过程相结合强烈暗示了线性遍历，例如 BFS 或 DFS。 

当树是一个简单的链时，会出现微妙的边缘情况。 在这种情况下，正好有两片叶子，并且链的中心应该具有最大距离。 仅考虑一个叶子或对每个叶子运行独立搜索的简单方法可能会意外地偏差距离或错误地重新计算。 另一个边缘情况是星形树，其中所有非中心节点都是叶子。 正确的答案是距离为 1 的中心，但忘记多源初始化的粗心实现可能会错误地返回距离为零的叶子。 

## 方法

 一个蛮力的想法是使用从每个叶子开始的 BFS 或 DFS 来计算从每个节点到每个叶子的距离。 对于每个叶子，我们运行最短路径搜索并更新距离。 这是正确的，因为树的边缘未加权并且最短路径是明确定义的。 然而，如果有$L$叶子和$N$节点，每个 BFS 成本$O(N)$，导致$O(NL)$在最坏的情况下。 在一颗星形的树上，$L$是$O(N)$，退化为$O(N^2)$，太慢了。 

关键的观察结果是所有叶子都充当同时源。 我们可以想象将所有叶子放入距离为零的单个 BFS 队列中，然后一起向外扩展，而不是单独处理每个叶子。 因为 BFS 自然地按照距离递增的顺序处理状态，所以我们第一次到达节点时保证是通过来自任何叶子的最短路径。 这将问题转化为树上的标准多源 BFS。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（每片叶子的 BFS）| O(n²) | O(n) | 太慢了 |
 | 多源BFS | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们用一个很大的值初始化一个距离数组，表示还没有到达节点。 我们通过检查哪些节点的度数恰好为 1 来识别所有叶子。 这些叶子被插入距离为零的队列中，因为它们是传播的起点。 

然后我们执行标准的 BFS。 每次弹出一个节点时，我们都会尝试放松其邻居。 如果通过当前节点到达邻居的距离小于其存储值，我们会更新它并将邻居推入队列。 这确保了每个节点按照其到任何叶子的最小距离的递增顺序进行处理。 

BFS 完成后，我们扫描所有节点并选择最大距离。 如果多个节点共享相同的最大距离，我们选择其中最小的索引。 

## 为什么它有效

 BFS 队列保持不变式，即每个节点都是通过来自任何叶子的最短可能路径首先发现的。 由于所有叶子在距离为零时同时初始化，因此每个扩展步骤都对应于逐层增加距最近源的距离。 在树中，任何两个节点之间都只有一条简单路径，因此一旦到达某个节点，以后就没有替代路径可以改善其距离。 这保证了存储的值是到任何叶子的真实最小距离，因此这些值的最终最大值是正确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n = int(input().strip())
    g = [[] for _ in range(n)]
    
    for _ in range(n - 1):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)
    
    dist = [10**18] * n
    q = deque()

    for i in range(n):
        if len(g[i]) == 1 or n == 1:
            dist[i] = 0
            q.append(i)

    while q:
        u = q.popleft()
        for v in g[u]:
            if dist[v] > dist[u] + 1:
                dist[v] = dist[u] + 1
                q.append(v)

    best_dist = -1
    best_node = 0

    for i in range(n):
        if dist[i] > best_dist:
            best_dist = dist[i]
            best_node = i

    print(best_node + 1, best_dist)

if __name__ == "__main__":
    solve()
```实现首先为树构建邻接表。 叶子检测纯粹通过程度检查来完成，即使在以下情况下也是安全的：$n = 1$，其中单个节点既是起始节点又是叶子节点。 所有叶子都被推入 BFS 队列，距离为零，建立多源设置。 

BFS 循环是标准的。 条件`dist[v] > dist[u] + 1`确保仅当找到来自某些叶子的较短路径时才更新每个节点。 因为每条边都有相同的权重，所以这就像没有优先级队列的 Dijkstra 一样。 

最后，我们扫描最大距离。 我们通过仅在出现严格更大的距离时进行更新来存储实现该目标的第一个节点。 

## 工作示例

 考虑一个小链：$1 - 2 - 3 - 4 - 5$。 

我们将叶子初始化为节点 1 和 5。 

| 步骤| 队列| 当前节点| 距离更新 |
 | ---| ---| ---| ---|
 | 初始化| [1, 5] | - | 距离[1]=0，距离[5]=0 |
 | 1 | [5, 2] | 1 | 距离[2]=1 |
 | 2 | [2, 4] | 5 | 距离[4]=1 |
 | 3 | [4, 3] | 2 | 距离[3]=2 |
 | 4 | [3] | 4 | 没有变化|
 | 5 | []| 3 | 没有变化|

 这确认了中心节点 3 的最大距离为 2。 

现在考虑一颗星，其中心 1 连接到 2、3、4、5。 

| 步骤| 队列| 当前节点 | 距离更新 |
 | ---| ---| ---| ---|
 | 初始化| [2,3,4,5] | - | 所有叶子距离 0 |
 | 1 | [3,4,5,1] | 2 | 距离[1]=1 |
 | 2 | [4,5,1]| 3 | 没有变化|
 | 3 | [5,1]| 4 | 没有变化|
 | 4 | [1] | 5 | 没有变化|

 中心节点 1 的距离为 1，这是最大的。

 ## 边缘情况

 单节点树是最简单的陷阱，因为叶子检测通常会排除度为 1 的节点，但这里必须将唯一的节点视为叶子。 该算法通过允许明确地处理这个问题$n = 1$初始化队列，确保距离保持为零。 

在双节点树中，两个节点都是叶子。 多源BFS从两端同时开始，各节点保持零距离。 最终的最大值为零，并且可以选择任一节点，但实现选择第一个出现的节点，这是一致的。 

在路径形树中，BFS 层从两端对称扩展。 中心节点累积最大距离，因为它距两个源最远。 BFS 确保同时考虑两个方向，防止偏向一个端点。
