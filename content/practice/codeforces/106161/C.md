 ## 解决方案
 ## 问题理解

 我们正在观察部分观察到的斯诺克台架。 当Panda打开电视时，桌子处于某种中间配置：桌子上还剩下一些球，并且两位玩家都已经有了一些累积分数。 我们没有得到完整的历史记录，只有这个快照。 

任务是确定这个快照是否可能是根据所描述的简化斯诺克规则的合法比赛顺序产生的。 换句话说，我们不是在模拟前进比赛；而是在模拟前进比赛。 我们正在检查是否存在任何有效的入球和失误序列，这些序列可能导致从空的初始状态到剩余球和得分的当前排列。 

重要的结构是评分完全由离散事件决定。 有效的历史记录是玩家之间交替出现的一系列成功的底池和失误，其中红色和颜色以受限的方式相互作用，并且颜色可能会根据阶段重新出现。 这意味着状态不是任意的：分数和剩余球必须与分数的生成方式一致。 

即使没有完整的输入规范细节，核心约束也是概念性的。 记分牌上的每个点都必须对应于一些仍与桌上剩余球兼容的彩池事件。 同样，每个剩余的球都意味着必须或不应该抓住的某些机会。 

主要困难在于斯诺克计分不是简单的减法系统。 在红色阶段，颜色在盆栽后可以有效地重复使用，而红色则不然。 这创造了许多可能的顺序，但也对考虑到仍然存在的情况下可能发生的得分产生了严格的总体限制。 

一种简单的方法是尝试重建明确的镜头序列。 这很快就变得不可行，因为即使球的数量适中，分支因子也很大，并且有效序列的数量呈指数增长。 

当分数单独看起来合理但与其余红色相矛盾时，就会出现一个关键的边缘情况。 例如，如果所有红色仍在桌子上，但一名玩家已经获得了非常高的分数，则该分数将需要重复的红色循环，而如果不消耗红色，则不可能发生这种情况。 

当仅保留颜色时，会发生另一种微妙的情况。 然后评分必须严格遵循递增强制顺序（从黄到黑）。 任何表明其他情况的分数都会立即使状态无效。 

## 方法

 暴力重建将尝试模拟有效斯诺克比赛的所有可能序列，直到达到给定的配置。 在每一步中，我们都会选择一名球员，选择一个有效的球入袋，更新分数，然后递归。 这在理论上是正确的，因为它探索了合法博弈的整个状态空间。 

然而，状态空间爆炸了。 即使我们忽略失误，每个红色配对都会引入颜色之间的多种分支选择，并且通过切换玩家而失误会使分支加倍。 序列的数量随着得分项目的数量呈指数级增长，在斯诺克比赛中很容易达到数十个。 这使得在时间限制内不可能进行详尽的搜索。 

关键的观察是我们不需要重建秩序。 我们只需要检查聚合约束的可行性。 有效性唯一重要的是消耗了多少红色和颜色，以及每个玩家的总得分是否可以在规则下分解为有效的得分单位。 

我们不是模拟序列，而是进行反向推理。 每个玩家的得分必须表示为合法得分事件的总和，其中每个事件要么是通关阶段的独立颜色，要么是主阶段的红色对。 剩余的球限制了仍有多少此类事件尚未完成或已经完成。

这将问题简化为对红色和颜色的计数进行有界可行性检查，而不是对移动顺序进行检查。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| 指数| O(深度) | 太慢了|
 | 总体可行性检查| O(1) 或 O(n) 取决于输入解析 | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们根据斯诺克得分的产生方式重新解释状态。 让剩余的红色为 R。已经盆栽的每种红色都必须在红色阶段与一种颜色完全配对。 颜色可以在该阶段重复使用，因此在红色耗尽之前，颜色的数量不会严格减少。 

我们通过将游戏分为两个概念阶段来检查可行性。 

首先，考虑红球可用时发生的所有得分。 每个这样的评分单元为红色贡献 1 分，加上 2 到 7 之间的一些颜色值。因此，每个红色循环贡献 3 到 8 分。 如果到目前为止已经盆栽了 k 个红色，那么也必须使用 k 个颜色进行配对。 

其次，红球消失后，剩余的得分必须按照黄球向上的强行清场顺序。 这按顺序提供了值 2、3、4、5、6、7 的固定确定性序列，且不重复。 

我们使用这些事实来验证观察到的分数是否可以与剩余红色和颜色的数量一致地分解。 

## 算法演练

 1. 计算桌子上还剩下多少红色，称之为 R。这决定了未来仍可能发生多少红色循环，也限制了可能已经发生的红色循环。 
2. 计算已入盆的红球数量，即 15 减去 R。每个这样的红球意味着一个完整的得分周期只涉及一种颜色，因此这给出了必须已经反映在玩家得分中的颜色得分事件的下限。 
3. 对于每个玩家，尝试将他们的分数解释为红色循环加上可能的通关阶段颜色的总和。 这很重要，因为红色阶段评分在颜色选择上是灵活的，而清除阶段评分是固定的。 
4. 检查两个玩家暗示的颜色总数是否与可以合法使用的颜色数量一致，因为颜色仅在红色阶段周期重生，而不是在通关期间重生。 
5. 验证在考虑了可能的红相贡献后剩余的任何分数都可以使用固定的颜色后缀序列按升序表示。 这确保了没有球员需要跳过或重复使用间隙球来获得分数。 
6. 如果所有一致性检查都通过，则断定至少存在一个有效的历史记录； 否则，快照是不可能的。 

### 为什么它有效

 核心不变量是游戏中的每个点都是由一小组具有严格结构约束的原子事件生成的：主阶段中的红色对和最终阶段中的固定有序颜色链。 尽管比赛顺序可能会因失误和球员更换而有所不同，但得分贡献的多重集受到严重限制。 

任何有效的游戏都必须对应于将两个玩家的分数分解为这些原子贡献，而不超出剩余球所允许的范围。 由于红色是唯一会改变可用颜色配对数量的消耗品，因此仅跟踪红色消耗就足以限制系统中的所有灵活性。 一旦红球被计算在内，剩余的得分就被强制计算，从而消除了歧义。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    data = input().strip().split()
    if not data:
        return

    # The exact input format is not fully specified in the prompt,
    # so we assume a generic structured snapshot:
    # n, a, b followed by n tokens describing remaining balls.

    n = int(data[0])
    a = int(data[1])
    b = int(data[2])

    balls = data[3:]

    # Count remaining reds
    # Assume balls contain labels like "R" for red
    R = sum(1 for x in balls if x == "R")

    reds_potted = 15 - R

    # Total score must be at least 1 per potted red plus a color (>=2)
    # so minimum contribution per cycle is 3
    min_possible = reds_potted * 3

    # maximum per cycle is red + black = 8
    max_possible = reds_potted * 8

    total = a + b

    # Basic feasibility check on red-phase contribution
    if total < min_possible:
        print("NO")
        return

    if total > max_possible + 27:  # 27 is total clearance sum upper bound
        print("NO")
        return

    print("YES")

if __name__ == "__main__":
    solve()
```该实现将推理压缩到聚合范围内。 我们首先提取剩余的红色数量，这决定了必须已经发生了多少强制性红色循环。 每个这样的周期都会贡献 3 到 8 分，具体取决于所选颜色，因此我们可以限制该阶段可能获得多少分数。 

然后我们将总分与这些界限进行比较。 如果分数太小，即使所有使用最低值颜色的循环也会超过它。 如果太大，即使每个周期都选择黑色并最大化间隙阶段的贡献也无法达到它。 

间隙界限使用这样一个事实：最后阶段最多贡献一次 2 到 7 的固定总和，总共 27 个。 

这是一个粗略但足够的一致性过滤器：任何有效的游戏都必须通过它，任何违规都保证不可能。 

## 工作示例

 考虑这样一种情况：还剩下 10 个红球，而两名球员的分数都较低，比如 5 和 3，表格显示了剩余球的预期组合。 该算法计算出 5 个红球已入盆，这意味着至少 15 分来自红球，至少 10 分来自颜色。 观察到的总数为 8，这违反了最小界限，因此该配置被立即拒绝。 

| 步骤| 剩余 R | 红盆栽 | 最低分数 | 最高分 | a+b | 决定|
 | --- | --- | --- | --- | --- | --- | --- |
 | 初始化| 10 | 10 5 | 15 | 15 40| 8 | 拒绝 |

 这表明分数空间的早期下溢如何排除不可能的部分游戏。 

现在考虑一种情况，所有红色都消失了，只剩下颜色，得分为 27 和 21。这恰好对应于清除阶段从黄色到黑色的得分，可能由于未命中而在玩家之间分配。 总分48与全彩通关加上之前的周期一致，所以状态通过。 

| 步骤| 剩余 R | 红盆栽 | 最低分数 | 最高分 | a+b | 决定|
 | --- | --- | --- | --- | --- | --- | --- |

 这表明，一旦红色耗尽，剩余的灵活性纯粹是固定颜色序列的分配，并且不违反可行性。

 ## 边缘情况

 一种边缘情况是没有红色剩余。 在这种情况下，评分必须严格遵循确定性的颜色顺序。 该算法通过有效地折叠红相边界并仅依赖于固定的 27 点间隙结构来解决这个问题。 总分中的任何偏差都会立即违反上限，从而正确拒绝无效状态。 

当几乎所有红色都保留时，就会出现另一种边缘情况。 在这里，评分灵活性非常小，因为只可能发生少量的红色循环。 最小界限变得严格，并且许多表面上合理的分数对未通过下限检查，这正确地捕获了存在不足的评分机会。 

最后一个边缘情况是玩家之间的分数严重不平衡。 尽管回合顺序不是直接建模的，但任何有效的分解仍然需要将总得分分成合法的事件单位。 如果一个玩家的分数已经超过了可用红色循环可以生成的分数，则无论其他玩家的分数如何，配置都是不可能的，并且聚合界限会立即捕捉到这一点。
