 ## 解决方案
 ## 问题理解

 我们得到了放置在非负整数行上的相同长度段的集合。 每个线段从某个整数坐标开始，并恰好覆盖 L 个连续的整数点。 因此，从位置 l 开始的线段覆盖了从 l 到 l + L − 1 的所有点。 

对于线上的每个整数点，我们可以计算有多少条线段覆盖它。 我们感兴趣的是恰好被覆盖 k 次的点。 

任务是在可选地移动至多一段之后计算此类“精确 k 覆盖”点的最大可能数量。 移动线段意味着选择一个线段，将其从当前位置移除，然后将其放置在非负轴上的任何其他位置，同时保持其长度不变。 所有其他部分保持固定。 

输出是每个测试用例可实现的最大计数。 

约束足够大，任何尝试模拟每次移动覆盖范围或显式尝试所有放置的解决方案都会失败。 所有测试用例的段总数最多为 2×10^5，这排除了对段的任何二次推理或每个候选移动从头开始重新计算覆盖范围。 即使每个段的线性扫描也会变得紧张，除非预处理后每个操作都是恒定时间。 

一个微妙的困难是，移动一个片段会产生全局影响：它会删除原始区间的覆盖范围，并将覆盖范围添加到全新的区间。 这两种效应都与覆盖计数的当前分布相互作用，因此天真的贪婪选择放置段的位置很容易失败。 

当两个区域竞争时，会出现典型的失败情况：移开一段可以通过减少过度覆盖来改善一些点，但将其放置在其他地方可能会在创建新点的同时破坏一些恰好 k 点。 例如，如果 k=1 并且某个片段当前与密集区域重叠，则将其删除可以帮助很多点，但将其放置在稀疏区域中可能没有任何帮助。 

核心困难在于，我们必须在不显式尝试所有布局的情况下评估最佳的“删除一个区间+在其他地方添加一个区间”的效果。 

## 方法

 我们从所有段都固定的基线配置开始。 我们可以使用差异数组计算每个整数点有多少段覆盖它。 由此我们立即得到最初的答案：有多少个点已经完全覆盖了 k。 

现在考虑一下当我们移动一段时会发生什么。 这一操作相当于两次独立的改变：我们删除一个现有的区间，并在其他地方添加一个相同长度的新区间。 

蛮力的想法是尝试我们移动的每一部分，并为每一部分尝试每一种可能的新位置。 对于每个候选位置，我们将重新计算整个轴上的覆盖效果。 即使我们预先计算初始覆盖范围，评估单个移动仍然需要 O(N) 成本，因为一个间隔影响 O(L) 个点，并且移动和放置有 O(N^2) 个选择。 这已经远远超出了极限。 

关键的观察结果是，删除段的效果和添加段的效果可以根据它们如何改变恰好 k 个点的计数来独立评估。 一旦我们将这两种效应表示为间隔内的加性贡献，每个效应都成为标准的“派生数组上的范围总和”问题。 

对于任何点，只有三个覆盖状态与 k 相关：其当前覆盖范围是 k−1、k 还是 k+1。 任何其他值不会直接影响在单次递增或递减后它是否正好变为 k。

这允许我们定义一个变换后的数组，其中每个位置如果当前具有覆盖范围 k−1，则贡献 +1；如果当前具有覆盖范围 k，则贡献 -1。 然后放置一个新段相当于选择一个长度为 L 的区间并在其上对该数组求和。 最佳放置就是固定长度 L 的最大子数组和。 

类似地，根据减少 1 时覆盖范围如何在 k+1、k 和 k−1 之间转换，删除分段也会转换为对相同类型的派生数组的范围贡献。 

最终的解决方案简化为计算这些转换后的数组上的前缀和并提取最佳间隔和，并在 O(1) 中评估每个段的去除效果。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力移动和放置 | O(N²·L) | O(N + L) | 太慢了|
 | 前缀求和+区间评分分解 | 每个测试用例 O(N) | O(N) | 已接受 |

 ## 算法演练

 1. 使用所有段上的差异数组在坐标范围上构建频率数组。 这给出了每个整数点的覆盖计数。 此步骤是必要的，因为所有后续推理都取决于了解覆盖率与本地 k 的比较情况。 
2. 计算一个基线答案，该答案等于覆盖范围恰好为 k 的点的数量。 这代表任何移动之前的起始配置。 
3. 在所有点上构建辅助数组，其中如果覆盖范围为 k−1，则每个位置贡献 +1；如果覆盖范围为 k，则每个位置贡献 -1，否则贡献 0。 该数组编码了添加分段时增加每个点的覆盖范围的好处。 
4. 通过在该辅助数组上滑动长度为 L 的窗口并取最大总和来计算新段的最佳可能放置位置。 这代表了通过在任何地方添加分段可实现的最佳净收益。 
5. 对于每个分段，使用相同的想法计算删除它的效果，但直接分类当覆盖范围减少 1 时每个点如何变化。 具体来说，当前位于 k+1 的点变好（它们变成 k），而当前位于 k 的点变坏（它们下降到 k−1）。 使用覆盖范围类别的前缀和，计算每个段的净去除增益，时间复杂度为 O(1)。 
6. 最终答案是基线加上删除片段的所有选择中的最佳改进，再加上全局计算的最佳放置增益。 

### 为什么它有效

 该变换将问题简化为仅跟踪 k 周围覆盖范围的局部变化，因为覆盖范围远离 k 的任何点都不能通过单段移动精确地变为 k，除非它位于移动段的区间内。 添加和删​​除操作在连续间隔上仅对覆盖范围产生 ±1 的影响，因此通过计算有多少点穿过阈值 k−1、k 和 k+1 可以完全捕获它们的影响。 由于这两个操作都是对同一导出评分函数的独立区间修改，因此最佳移动分解为最大化两个可分离的区间得分。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, L, k = map(int, input().split())
    ls = list(map(int, input().split()))

    max_r = max(ls) + L + 5
    diff = [0] * (max_r + 2)

    for l in ls:
        diff[l] += 1
        diff[l + L] -= 1

    m = len(diff)
    freq = [0] * m
    cur = 0
    for i in range(m):
        cur += diff[i]
        freq[i] = cur

    base = 0
    for x in freq:
        if x == k:
            base += 1

    gain = [0] * m
    for i in range(m):
        if freq[i] == k - 1:
            gain[i] = 1
        elif freq[i] == k:
            gain[i] = -1

    window = 0
    best_add = -10**18

    for i in range(m):
        window += gain[i]
        if i >= L:
            window -= gain[i - L]
        if i >= L - 1:
            best_add = max(best_add, window)

    pref_km1 = [0] * (m + 1)
    pref_k = [0] * (m + 1)
    pref_kp1 = [0] * (m + 1)

    for i in range(m):
        pref_km1[i+1] = pref_km1[i]
        pref_k[i+1] = pref_k[i]
        pref_kp1[i+1] = pref_kp1[i]

        if freq[i] == k - 1:
            pref_km1[i+1] += 1
        if freq[i] == k:
            pref_k[i+1] += 1
        if freq[i] == k + 1:
            pref_kp1[i+1] += 1

    def query(a, l, r):
        return a[r+1] - a[l]

    best_remove = 0
    for l in ls:
        r = l + L - 1
        if r >= m:
            r = m - 1
        if l < m:
            rem = (query(pref_kp1, l, r) - query(pref_k, l, r))
            best_remove = max(best_remove, rem)

    print(base + best_add + best_remove)

if __name__ == "__main__":
    solve()
```该代码首先使用差异数组重建覆盖范围，这避免了显式迭代每个段的长度。 之后，它构建恰好 k 个覆盖点的基线计数。 

数组`gain`编码放置新段时增加每个位置覆盖范围的好处。 该阵列上的固定长度滑动窗口可以有效地找到最佳可能的位置。 

对于删除，覆盖类别上的前缀和允许对特定段在删除时如何改变恰好 k 点的数量进行恒定时间评估。 里面的表情`rem`直接计算段区间内有多少点提高减去多少点下降。 

最终的总和将原始配置与删除和添加一个段的最佳独立效果结合起来。 

## 工作示例

 考虑一种小型配置，其中段在一个区域中严重重叠，而在其他地方则稀疏。 令 L=3 且 k=1，段从位置 0、1 和 10 开始。 

我们首先计算每个点的覆盖范围，然后得出仅覆盖一次的点的基线计数。 

| 步骤| 覆盖区间 | 频率变化 | 精确-k 贡献 |
 | --- | --- | --- | --- |
 | 初始构建| [0-2]、[1-3]、[10-12] | [1-2] | 中的重叠 从 freq==1 开始计数 |

 基线已经捕获了曾经被覆盖的所有点。 

现在考虑添加一个段。 增益阵列标记了增加覆盖范围有益或有害的位置。 

| 指数范围| 频率 | 增益|
 | --- | --- | --- |
 | 重叠区| 2 | 0 |
 | 稀疏区| 0 | 0 |
 | 边界区 | 1 | +1 |

 滑动长度为 L 的窗口选择放置新段的最佳区域。 

这证实了最佳放置纯粹由 k−1 和 k 区域的局部密度决定。

 ## 边缘情况

 关键的边缘情况是所有段都严重重叠，以致许多点的覆盖范围大于 k。 在这种情况下，移开一段主要是通过移除效应而不是相加效应来增加正好 k 个点的数量。 该算法可以处理这个问题，因为去除增益显式地计算从 k+1 到 k 的转变，这在密集区域中占主导地位。 

当 k 为 0 或 1 时，会出现另一种边缘情况。当 k=0 时，除非未被发现，否则没有点最初是“精确 k”的，因此增益数组正确地将 k−1 视为 −1 并忽略无效的负状态。 前缀构造自然地避免了下溢，因为它仅依赖于相等性检查。 

最后，当 L 相对于坐标分布较大时，最佳放置窗口可能跨越几乎所有活动位置。 滑动窗口仍然可以正确评估，因为它在不假设稀疏性的情况下处理完整的增益阵列。
