 ## 解决方案
 ## 问题理解

 每个测试用例都会给出一组论文，每篇论文都由一小组评审员进行评估。 每个审稿人都会给出一个-3到3之间的整数分数，论文的总分就是这些值的总和。 如果一篇论文的总分至少达到阈值，则该论文被视为被接受$k$。 

初步评估后，有一个称为反驳的可选操作。 熊猫最多可以选择$b$不同的论文并对它们应用反驳操作。 该操作以一种非常结构化的方式改变每个审稿人对该论文的评分：正分数减少 1，而非正分数增加 1。 这意味着高分略有减少，低分略有提高，但就总影响而言，这种变化并不对称。 

目标是决定对哪些论文应用反驳，以便在所有更改之后，接受的论文数量最大化。 

关键的约束结构是每篇论文最多有 10 名审稿人，所有测试用例最多有 200,000 篇论文。 这立即排除了任何尝试模拟所选论文的组合或重复重新计算全局状态的方法。 任何解决方案都必须几乎独立地处理每篇论文，并将问题简化为线性或近线性时间内的排序或贪婪选择。 

当一篇论文已经被接受但在反驳后被拒绝时，就会出现一种微妙的边缘情况。 例如，如果一篇论文最初几乎没有超过阈值，但大多数审稿人给出了积极的分数，那么应用该操作可以将总分减少到足以将其降至以下$k$。 “反驳一切可能有帮助的事情”的幼稚策略在这里会失败。 

另一种边缘情况是论文最初被拒绝，但在反驳后被接受。 这是应用该操作有用的唯一情况，而准确识别何时发生这种情况是问题的核心。 

## 方法

 直接的暴力策略将尝试最多的每个子集$b$论文并模拟应用反驳的效果。 对于每个子集，我们将重新计算所有论文分数并计算有多少达到阈值。 这需要评估$\sum_{i=0}^{b} \binom{n}{i}$可能性，即使对于小企业来说也是天文数字$b$，并且每次评估都需要$O(nm)$努力重新计算总数。 这种方法一旦超出微小的输入就会失败。 

关键的观察是，反驳对单篇论文的影响完全独立于所有其他论文。 除了全局约束之外，没有什么可以将论文之间的决策结合在一起$b$可以选择论文。 这将问题从组合全局优化简化为每项增益选择问题。 

对于每篇论文，我们只需要比较两个状态：当前的接受状态和应用固定变换后的状态。 如果应用反驳将论文从拒绝变为接受，这是有益的。 如果它从接受变成拒绝，那是有害的。 如果不改变决定，那就无关紧要。 

一旦我们计算出每篇论文的收益，全局问题就变成了最多选择$b$具有正增益的项目。 由于每篇论文对目标的贡献不是 0 就是 1（或者是负，我们永远不会选择），所以最佳策略就是在允许的情况下采纳尽可能多的有益论文。$b$。

 ## 算法演练

 我们独立计算对每篇论文应用反驳的效果，然后贪婪地选择最佳候选者。 

1. 对于每篇论文，通过将所有审稿人的评分相加来计算其初始总分。 这决定了当前是否被接受。 
2. 对于同一篇论文，计算反驳如何改变其分数。 如果每个审稿人的分数 ≤ 0，则贡献 +1；如果他们的分数 ≥ 1，则贡献 -1。将这些贡献相加即可得出该论文的总分数变化。 
3. 将原始分数与修改后的分数相结合，得到反驳后的最终分数，并确定该状态下论文是否被接受。 
4. 对比反驳前后的接受状态。 如果论文从拒绝变为接受，则将其标记为有益，增益+1。 如果它从接受变为拒绝，则将其标记为-1，尽管此类论文永远不会被选择。 如果保持不变，则增益为 0。 
5. 统计有多少篇论文获得+1。 由于每次选择都是独立的，唯一的约束是最多选择$b$论文，选择最多$b$这些有益的论文。 
6、最终答案为初步录用论文数加上入选有益论文数。 

### 为什么它有效

 每篇论文的行为就像一个二进制项目，其价值只能在一次操作下提高、恶化或保持不变。 由于目标纯粹是跨论文累加的，并且操作计数受到全局限制但不相互作用，因此将一篇有益的论文优先于另一篇论文没有任何好处。 每一篇有用的论文在最终计数中都贡献了一个单位的改进，因此任何大小的子集最多$b$其中是等价的。 这使得贪婪选择成为最优，并防止选择之间存在任何隐藏的依赖关系。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n, m, k, b = map(int, input().split())
        
        base_accepted = 0
        good = 0
        
        for _ in range(n):
            arr = list(map(int, input().split()))
            
            s = sum(arr)
            base_ok = (s >= k)
            
            delta = 0
            for x in arr:
                if x >= 1:
                    delta -= 1
                else:
                    delta += 1
            
            new_ok = (s + delta >= k)
            
            if base_ok:
                base_accepted += 1
            
            if (not base_ok) and new_ok:
                good += 1
        
        out.append(str(base_accepted + min(b, good)))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 对于每篇论文，我们计算原始总和以及反驳引起的净变化。 审稿人的循环很小，因为$m \le 10$，所以这一步实际上是每篇论文的恒定时间。 

关键的实现细节是将论文隐含地分为三类：已接受的、反驳后新接受的以及其他所有类别。 只有从拒绝到接受的转变才有助于最终的改进计数。 我们从不明确存储收益，因为只有正收益的计数才重要。 

## 工作示例

 考虑一个包含三篇论文的小场景。 

论文 A 的分数总和已高于阈值，因此最初被接受。 应用反驳后，假设其分数降至阈值以下。 论文 A 对受益池没有任何贡献。 

论文B最初被拒绝，但在转换后被接受。 在这两种情况下，论文 C 均被拒绝。 

我们跟踪分类：

 | 纸| 初始总和| 初始确定 | 达美航空 | 新总 | 新好 | 增益|

 | 乙| 中等| 0 | +3 | 高| 1 | +1 |

 如果$b = 1$，我们只选择论文 B，额外接受一次。 如果$b = 2$，选择更多并没有帮助，因为只有一篇论文有积极的收益。 

这表明决策完全归结为计算有多少被拒绝的论文可以转为被接受的论文。

 ## 边缘情况

 一种微妙的边缘情况是，论文已经被接受，但在提出反驳后被拒绝。 在这种情况下，增益为负，算法自然会忽略它，因为它只计算拒绝到接受的转换。 例如，如果一篇论文的总和恰好等于$k$大多数评论者给出了积极的分数，增量可以将其推至阈值以下，但由于它从未出现在有益池中，因此它永远不会被选择。 

当操作后纸张没有改善时，会出现另一种边缘情况。 在这种情况下，有益论文的数量为零，并且答案正确地减少到最初接受的论文数量。
