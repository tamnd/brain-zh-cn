 ## 解决方案
 ## 问题理解

 我们被分配了一个最多六名角色的小队，他们在一系列的回合中反复战斗。 每轮开始时都有固定的能量池，每个角色可以使用一次技能或保持闲置。 使用技能会消耗本轮的部分能量，每轮消耗的总能量不能超过可用的能量池。 

每个角色都有一个基本成本和一个固定的伤害值。 不同之处在于，成本在各轮中并不是静态的：如果角色在一轮中使用了他们的技能，他们的成本在下一轮中会暂时变得更高，并且只要他们每轮都继续使用它，就会保持在更高的水平。 如果他们跳过一轮，他们的成本会立即重置为原始值。 

我们想要对每一轮和每个角色进行选择，无论他们是否行动，以便尊重每轮的总能量约束，并使所有轮次的伤害总和最大化。 轮数可能非常大，因此我们无法逐轮模拟。 

约束驱动解决方案的结构。 字符数最多为 6，这强烈表明任何描述“本轮哪个字符子集处于活动状态”的状态都小到足以枚举。 困难不在于子集选择本身，而在于成本增加和重置导致的跨轮依赖性。 

轮数最多可达十亿轮。 这立即排除了任何每轮动态编程或模拟。 任何有效的解决方案都必须将过程压缩为我们可以检测重复或达到稳定循环的形式。 

当角色交替使用以避免成本增加时，就会出现微妙的边缘情况。 例如，一个角色可以每隔一轮使用一次，以保持基本成本，同时仍然造成伤害。 总是在每一轮中独立地选择最具破坏效率的子集的天真的贪婪方法在这里失败了，因为它忽略了长期成本演化。 

另一种失败模式是假设一旦一个角色变得“昂贵”，它就会以有用的方式保持昂贵。 事实上，即使跳过一轮也会完全重置成本，因此最佳策略可以有意“冷却”昂贵的角色，以便以后有效地重用它们。 

## 方法

 暴力解释将每一轮都视为状态转换问题。 对于每一轮，我们知道每个角色是否在上一轮中使用过，这决定了他们当前的成本。 这表明状态由大小为 n 的二进制向量组成，表示上一轮使用了哪些字符。 从这样的状态开始，我们尝试在当前回合中使用的每个角色子集，检查能量限制下的可行性，计算损坏，并转换到新状态。 

这是正确的，因为它直接模拟规则：成本仅取决于以前的使用情况，决策仅取决于当前状态。 然而，每个状态最多有 2^n 种可能性，并且每次转换都会尝试另外 2^n 个子集。 即每轮状态层有 2^(2n) 次转换。 尽管n很小，但R轮数最多可达10^9，因此每轮迭代都是不可能的。 

关键的观察结果是，系统是一个有限状态机，最多有 2^n 个状态，并且转换仅取决于先前的状态。 这意味着我们正在处理一个有向加权图，其中节点是状态，边代表一轮游戏。 每条边的权重等于给定状态下一轮中可达到的最大伤害。 问题变成了在该图中找到 R 个步骤的最大总权重，这是一个经典的最大加矩阵求幂问题。

我们构建一个转换矩阵，其中如果我们从状态 a（确定哪些角色当前处于“提升成本模式”）开始并以状态 b（本轮选择的集合）结束，则条目 T[a][b] 是一轮中可实现的最佳伤害。 然后我们在 max-plus 乘法下对该矩阵进行快速求幂，以有效地计算 R 步。 

因为 n 最多为 6，所以状态空间最多为 64，因此 64 x 64 矩阵是可以管理的。 由于 S = 64，O(S^3 log R) 中的矩阵求幂是可行的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力模拟| O(R·2^n·2^n) | O(R·2^n·2^n) | O(2^n) | O(2^n) | 太慢了 |
 | Max-Plus 矩阵求幂 | O(2^{3n} log R) | O(2^{3n} log R) | O(2^{2n}) | O(2^{2n}) | 已接受 |

 ## 算法演练

 我们将每个状态编码为长度为 n 的位掩码，其中位 i 表示角色 i 是否在上一轮中使用了他们的技能。 该位掩码完全确定每个字符的当前成本。 

对于每个状态，我们计算每个角色的有效成本：如果它们在上一轮中使用过，则它们的成本为 ci + k，否则为 ci。 

对于每个状态，我们预先计算如果从该状态开始，我们在一轮中可以获得的最佳总伤害。 

### 1. 枚举所有状态

 我们迭代从 0 到 2^n - 1 的所有掩码。每个掩码表示哪些字符当前处于“增加成本”状态。 

这是必要的，因为未来只取决于这个二进制记忆，其他历史都不重要。 

### 2. 计算每个州的有效成本

 对于固定状态掩码，我们根据每个字符的位是否设置来为每个字符分配 ci 或 ci + k。 这会将问题转换为一轮标准的类似背包的约束。 

重要的一点是，在单轮内，一旦状态已知，成本就固定了。 

### 3. 枚举本轮的所有字符子集

 对于每个状态，我们都会尝试所有角色子集来决定本轮谁行动。 我们计算总成本和总损害，并丢弃成本超过 m 的子集。 

这一步是有效的，因为 n 很小，所以所有 2^n 子集都是可枚举的。 

### 4. 构建转移矩阵

 对于每个起始状态 a 和每个选定的子集 S，我们计算等于 S 的下一个状态 b，因为只有当前回合中使用的字符才会在下一轮中增强。 我们更新：

 T[a][b] = max(T[a][b], 伤害(S, a))

 这捕获了所有可能的一轮转换。 

### 5. 转移矩阵的快速求幂

 我们现在将 T 解释为最大加邻接矩阵。 矩阵相乘对应于组合回合，其中加法被伤害之和代替，比较被取最大值代替。 

我们使用二进制求幂对 T 求 R 次方。 我们从一个单位矩阵开始，其中保持相同状态而不受到伤害为零。 

求幂后，我们对所有起始状态取最大条目，以获得 R 轮中可实现的最佳总伤害。 

### 为什么它有效

 关键的不变量是，在 k 轮之后，矩阵幂 T^k 正确地存储了在恰好 k 轮的任何状态对之间转换时可实现的最佳可能总损害。 每个乘法步骤组成最优子结构：将 k 轮分成 i 和 k-i 确保任何最优序列都分解为最优前缀和后缀。 由于状态完全捕获了成本演化，因此不会丢失隐藏的历史记录，并且每个有效策略都对应于该状态图中的一条路径。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

NEG = -10**30

def mat_mul(A, B, n):
    C = [[NEG] * n for _ in range(n)]
    for i in range(n):
        Ai = A[i]
        for k in range(n):
            if Ai[k] == NEG:
                continue
            Bik = B[k]
            aik = Ai[k]
            for j in range(n):
                if Bik[j] == NEG:
                    continue
                val = aik + Bik[j]
                if val > C[i][j]:
                    C[i][j] = val
    return C

def mat_pow(mat, exp, n):
    res = [[NEG] * n for _ in range(n)]
    for i in range(n):
        res[i][i] = 0
    base = mat
    while exp:
        if exp & 1:
            res = mat_mul(res, base, n)
        base = mat_mul(base, base, n)
        exp >>= 1
    return res

def solve():
    n, m, k, R = map(int, input().split())
    a = []
    c = []
    for _ in range(n):
        ai, ci = map(int, input().split())
        a.append(ai)
        c.append(ci)

    S = 1 << n
    trans = [[NEG] * S for _ in range(S)]

    for mask in range(S):
        cost = [0] * n
        for i in range(n):
            cost[i] = c[i] + (k if (mask >> i) & 1 else 0)

        best = [-1] * S
        for sub in range(S):
            total_cost = 0
            total_dmg = 0
            for i in range(n):
                if (sub >> i) & 1:
                    total_cost += cost[i]
                    total_dmg += a[i]
            if total_cost <= m:
                best[sub] = total_dmg

        for sub in range(S):
            if best[sub] >= 0:
                trans[mask][sub] = max(trans[mask][sub], best[sub])

    res = mat_pow(trans, R, S)

    ans = 0
    for i in range(S):
        for j in range(S):
            ans = max(ans, res[i][j])

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先对每个状态进行编码并计算每个状态的所有可行子集。 转换矩阵存储状态之间一轮转换的最佳可实现损害。 然后，矩阵求幂将大量轮次压缩为对数步长，最终答案是所有可能的开始和结束状态中的最佳值。 

一个微妙的点是我们不强制执行固定的开始状态。 任何初始配置都是允许的，这就是为什么最终答案采用所有矩阵条目的最大值而不是单个起始行的原因。 

## 工作示例

 考虑一个包含两个字符的小案例。 角色 1 造成 10 点伤害，费用为 2，角色 2 造成 6 点伤害，费用为 3，其中 m = 5，k = 2。 

### 状态转换跟踪

 | 面膜（以前用过）| 成本（c 或 c+k）| 最佳子集| 成本| 伤害|
 | ---| ---| ---| ---| ---|
 | 00 | 00 (2, 3) | {1,2} | 5 | 16 | 16
 | 01 | (2, 5) | {1} | 2 | 10 | 10
 | 10 | 10 (4, 3) | {2} | 3 | 6 |
 | 11 | 11 (4, 5) | { } | 0 | 0 |

 这显示了先前的使用如何直接改变下一轮的可行性。 昂贵的国家会削减某些组合。 

现在假设 R = 2。一轮之后，状态根据选择的子集进行转换，矩阵求幂通过两个步骤组成这些选择。 最佳序列通常每轮都不是贪婪的，因为在第一轮中选择两个角色可能会迫使第二轮进行高成本，从而减少总增益。 

该轨迹表明状态依赖性至关重要，并证明了为什么我们不能独立对待每一轮。

 ## 边缘情况

 一个关键的边缘情况是所有字符在增强状态下都很昂贵，但在基本状态下很便宜。 例如，在 m = 2 下，ci = 1 且 k = 1000 的角色。最佳策略是每隔一轮使用它，在可用和不可用状态之间交替。 该算法可以处理此问题，因为状态转换显式编码是否使用字符，从而允许矩阵自然地表示交替模式。 

另一种边缘情况是当 R = 1 时。在这种情况下，求幂崩溃为一轮子集上的单层最大化，并且矩阵正确地简化为所有有效子集的直接枚举。 

第三种边缘情况是某个状态中没有可行的子集。 该状态根本没有具有有限权重的传出转换，实际上成为一个死节点。 矩阵通过 NEG 值自然地处理这个问题，确保此类路径永远不会贡献最大值。
