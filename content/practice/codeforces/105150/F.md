 ## 解决方案
 ## 问题理解

 我们正在模拟一场比赛，每圈的成本取决于当前轮胎组的磨损程度。 每个轮胎组都以某个初始磨损值开始，并且每在该组上行驶一圈，该圈就精确地获取当前磨损值（以秒为单位），然后磨损增加一。 

有两套轮胎。 第一个是在开始时安装的，有磨损`a`，第二个从磨损开始`b`。 在任何一圈之前，包括第一圈，我们都可以进站，更换活动轮胎组以换取固定的时间处罚`p`。 我们可以根据需要多次切换。 

目标是准确完成`n`总时间最短的圈数，其中时间包括单圈时间和进站处罚。 

重要的结构是每个轮胎组都会确定性地演变：如果您将其用于`k`从磨损开始连续圈数`x`，总成本是一个算术级数。 这意味着连续使用的任何部分都具有封闭形式的成本，这使得尽管有很大的限制，问题仍然易于处理。 

这些限制使得简单的模拟变得不可能。 自从`n`可以大到 10^9，我们无法逐圈模拟，甚至无法保持每圈的 DP。 任何直接迭代几圈的方法都会被立即排除。 该解决方案必须根据每个轮胎的使用情况进行推理。 

当切换总是稍微有益或总是稍微有害时，就会出现微妙的边缘情况。 如果贪婪策略在局部看起来更好时进行切换，则它可能会失败，因为切换会引入固定的惩罚，但会将磨损重置为不同的初始值，这会影响非局部的未来成本。 例如，即使`b < a`，如果进站成本与分散在许多圈中的节省成本相比太大，立即使用第二组可能仍然不是最佳的。 

## 方法

 一个简单的方法是逐圈决定使用哪个轮胎。 在每一圈，我们可以考虑继续使用当前轮胎或换成另一个轮胎。 这导致动态程序超过状态`(i, current_tire)`， 在哪里`i`是完成的圈数。 每次转换的成本为 O(1)，因此总复杂度为 O(n)。 这已经失败了`n = 10^9`。 

一个稍微好一点的观点是，一旦我们切换，我们可能会连续多圈留在轮胎上。 这建议将圈数分组。 如果我们固定一个段序列，则每个段都会贡献确定性算术级数成本加上可能的切换成本。 然而，枚举所有分段仍然是指数级的。 

关键的观察结果是，在任何最佳策略中，每个轮胎始终在连续的块中使用，并且对于选定的块长度，成本是完全确定的。 这将问题简化为决定在交替块中为每个轮胎分配多少圈。 

对于一个长度的块`k`从磨损开始`x`，成本为`k * x + k*(k-1)/2`。 这种凸结构意味着对于固定数量的段，我们可以贪婪地或通过边际增益的二分搜索来计算最佳块大小。 更重要的是，是否更换的决定仅取决于比较继续使用当前轮胎的边际成本与更换并重新启动另一个轮胎加上惩罚的边际成本。 

这导致了累积使用量的凸优化结构。 如果我们决定轮胎 A 用于以下用途，我们可以定义一个函数来计算最小成本`x`总圈数（可能分开），轮胎 B`n-x`圈数，考虑转换成本。 最佳解决方案简化为寻找最佳分割点，这可以通过评估从算术和导出的单调成本函数来有效解决。 

最终的解决方案利用了这样一个事实：使用轮胎的成本`k`圈数是二次方`k`，因此总成本变成凸函数加上线性切换惩罚的总和。 这允许在 O(1) 中评估每个候选转换点，并根据实施策略在 O(log n) 或 O(1) 中搜索最佳分割。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力 DP 圈数 | O(n) | O(1) | O(1) | 太慢了 |
 | 段枚举| 指数| O(1) | O(1) | 太慢了 |
 | 凸分割优化 | O(1) 或 O(log n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们将比赛视为选择圈数`x`在第一个轮胎上运行（可能在切换后），其余的`n-x`在第二个轮胎上，但我们必须正确考虑顺序以及切换可能发生在任一段之前或之间的事实。 

1. 计算轮胎的使用成本`k`从磨损开始连续圈数`x`。 这是`k*x + k*(k-1)/2`。 这为我们提供了封闭式评估而不是模拟。 
2. 考虑一下我们从不切换的情况。 我们计算完成所有工作的成本`n`轮胎 A 和所有轮胎上的圈数`n`在 B 轮胎上跑几圈并取最小值。 当切换没有好处时，这可以处理琐碎的最佳结构。 
3. 考虑只切换一次的情况。 我们选择一个分割点`i`，这意味着我们使用一个轮胎`i`跑几圈然后付费`p`切换并使用另一个轮胎`n-i`圈数。 我们评估两个顺序：A 然后 B，B 然后 A。 
4. 对于固定订单，将总成本表示为以下函数`i`。 这变成了一个二次表达式`i`加上常数。 由于它是凸的，因此最小值出现在单个点处，可以通过分析或通过检查实值最优值周围的相邻整数来找到该点。 
5. 评估最优方案`i`对于两个方向和两个轮胎订单，夹紧到有效范围`[0, n]`，并计算候选答案。 
6. 返回所有候选者中的最小值。 

### 为什么它有效

 任何最佳策略都可以减少到至多一个切换点而不失一般性，因为每个轮胎的边际成本随着使用量线性增加。 如果我们要多次切换，则两个相邻的段可以合并为同一轮胎上的单个段，而不会从分割中获益，因为成本函数在段长度上是凸的，并且切换会增加固定的惩罚。 这迫使最优结构最多有一个过渡点，使得对单个分割的搜索就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cost(x, k):
    return k * x + k * (k - 1) // 2

def solve():
    n, a, b, p = map(int, input().split())

    ans = float('inf')

    def eval_order(start, second):
        nonlocal ans

        def total(i):
            return cost(start, i) + (p if i > 0 and i < n else 0) + cost(second, n - i)

        # continuous minimization: derivative-based optimum
        # cost difference is convex, so check neighborhood around critical point
        # approximate optimal split
        # derivative balance gives i ≈ (second - start + n) / 2
        if n == 0:
            return

        i_opt = (second - start + n) / 2

        for di in range(-3, 4):
            i = int(i_opt + di)
            if 0 <= i <= n:
                ans = min(ans, total(i))

    # no switch
    ans = min(ans, cost(a, n), cost(b, n))

    # one switch A -> B or B -> A
    eval_order(a, b)
    eval_order(b, a)

    print(ans)

if __name__ == "__main__":
    solve()
```功能`cost(x, k)`对固定轮胎段的算术级数和进行编码，无需模拟圈数。 

这`eval_order`函数评估我们将比赛分为两个连续部分的场景。 仅当两个部分都非空时才添加进站成本。 关键的实现细节是我们不会迭代所有`i`，而是通过分析方式计算凸最优并测试一个小邻域，这是有效的，因为目标是一个二次函数`i`。 

我们还明确包括“无切换”情况，因为否则分割公式将始终支付或假设转换结构。

 ## 边缘情况

 当`n = 1`，分体配方一定不能意外充电进站。 该实现处理了这个问题，因为只有当两个段都非空时才会添加切换惩罚。 

什么时候`p = 0`，切换对于平衡磨损率完全有利。 凸公式仍然有效，因为最优分割转向均衡两个序列之间的边际成本，并且围绕解析最优值的邻域搜索捕获了该点。 

什么时候`a`和`b`相等时，最佳解决方案始终是不进行切换，因为任何分割只会增加惩罚，而不会改变圈数成本。 该代码通过以下方式捕获了这一点`all A`和`all B`基线检查，主导所有分裂候选人。
