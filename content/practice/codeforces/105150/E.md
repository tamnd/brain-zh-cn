 ## 解决方案
 ## 问题理解

 我们有一个方形的办公室，但只有左墙和底墙存在。 顶部和右侧是开放的，就像一个连续的入射光源。 太阳光线从开放边界进入办公室，并沿矢量给定的固定方向移动$(x_s, y_s)$，其中两个分量均为非正值，并且至少有一个分量严格为负值，因此光线始终向正方形内部移动。 

办公室内有几个办公桌部分。 从开放边界进入的光线沿直线传播，并且可能与某些桌子相交。 我们可以在部分开放边界上安装窗帘。 放置在边界点的窗帘阻挡了穿过该点的每一束光线，完全阻止其进入办公室。 

目标是阻挡与任何桌面部分相交的所有光线，同时最大限度地减少沿边界使用的窗帘的总长度。 

关键的微妙之处在于每条射线对应一个唯一的边界入口点。 问题不再考虑正方形内的光线，而是选择一组最小长度的边界点，“覆盖”击中任何部分的所有光线。 

这些限制立即表明，简单地模拟光线是不可能的。 最多有$2 \cdot 10^5$线段和坐标最多$10^9$，因此任何跟踪每条射线或每个边界点几何的解决方案都是不可行的。 该结构必须将 2D 几何图形折叠为 1D 区间覆盖问题。 

一个幼稚的错误是独立对待每张桌子并为其分配自己的窗帘间隔。 这会失败，因为多个桌子可能会被重叠的光线组阻挡，如示例中所示，一个窗帘同时覆盖两个部分。 

当线段平行于射线方向时，会出现另一种微妙的故障模式。 在这种情况下，它不会产生任何阻挡要求，因为光线要么完全错过它，要么沿着它通过，而不会强制使用正长度的边界间隔。 

## 方法

 强力解释将模拟从每个边界点进入办公室的每条射线，确定它首先击中哪张桌子，然后将该入口点标记为禁止。 这在几何上是明确定义的，但在计算上是不可能的。 边界是连续的，实际上需要无限采样，甚至精细地离散化也会导致令人望而却步的复杂性。 

关键的观察是射线完全由其与边界的相交决定。 由于所有光线都沿相同方向移动，因此平移光线相当于沿边界滑动其入口点。 对于每个桌子部分，照射到它的光线集沿着边界形成连续的间隔。 一旦我们将每个线段映射到这样的区间，问题就变成了选择覆盖所有区间的线上点的最小总长度，这正是计算区间并集的总长度。 

几何步骤是为每个线段端点计算沿着光线穿过它的边界的参数值。 由于光线随方向移动$(x_s, y_s)$，我们使用垂直投影来转换点$(x, y)$转换为标量坐标：$$t = x \cdot (-y_s) + y \cdot x_s$$该值与光线起源沿边界的位置成正比。 然后每个段产生一个区间$[t_1, t_2]$，然后我们计算并集长度。

 | ---| ---| ---| ---|
 | 暴力射线模拟 | 有效 O(∞) | O(1) | O(1) | 不可能|
 | 区间投影+并集| O(k log k) | O(k log k) | O(k) | 已接受 |

 ## 算法演练

 1. 对于每个线段端点，计算标量投影值$t = x \cdot (-y_s) + y \cdot x_s$。 这会将 2D 几何形状转换为沿与光线正交的边界方向的 1D 排序。 
2. 对于每个段，形成一个区间$[min(t_1, t_2), max(t_1, t_2)]$。 该间隔表示与该线段相交的所有光线原点。 
3. 将所有间隔收集到一个列表中。 
4. 按间隔的左端点对间隔进行排序。 需要排序，因为重叠结构仅按排序顺序可见。 
5. 扫描间隔，同时保持当前合并的覆盖间隔。 如果下一个间隔与当前间隔重叠或接触，则延长它。 否则，将当前间隔的长度添加到答案中并开始新的间隔。 
6. 处理完所有间隔后，添加最后一个活动间隔长度。 

最终答案是所有投影间隔的总并集长度。 

### 为什么它有效

 所有光线都是平行的，因此从边界入口点到任何内部线的映射都是仿射的。 线段恰好与其入口点落在连续范围内的那些光线相交，因此每个线段对应于一维空间中的单个间隔。 任何窗帘放置都对应于删除该一维线的部分，并且阻挡所有线段需要覆盖所有间隔。 因此，最小总幕长正是这些间隔的并集的度量，扫描计算时没有重叠。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, xs, ys = map(int, input().split())
    k = int(input())

    intervals = []

    # projection direction perpendicular to ray direction
    # using a consistent linear form
    dx = -ys
    dy = xs

    for _ in range(k):
        x1, y1, x2, y2 = map(int, input().split())

        t1 = x1 * dx + y1 * dy
        t2 = x2 * dx + y2 * dy

        l, r = (t1, t2) if t1 <= t2 else (t2, t1)
        intervals.append((l, r))

    intervals.sort()

    ans = 0
    cur_l, cur_r = intervals[0]

    for l, r in intervals[1:]:
        if l <= cur_r:
            if r > cur_r:
                cur_r = r
        else:
            ans += cur_r - cur_l
            cur_l, cur_r = l, r

    ans += cur_r - cur_l

    print(f"{ans / (dx*dx + dy*dy) ** 0.5:.10f}")

if __name__ == "__main__":
    solve()
```代码首先将射线方向转换为垂直投影向量$(dx, dy)$。 这确保了相等的投影值对应于沿着相同边界线进入的光线。 每个桌面端点都投影到该一维坐标中，形成间隔。 

扫描步骤合并重叠范围，确保共享覆盖范围被计数一次。 最后，我们通过投影矢量的长度进行归一化，以使坐标尺度与沿边界的实际几何长度相匹配。 

一个常见的实施陷阱是忘记标准化步骤。 投影产生任意线性比例的值，因此如果不除以矢量幅度，结果将偏离一个常数因子，具体取决于$(x_s, y_s)$。 

## 工作示例

 ### 示例 1

 输入：```
6 -2 0
1
2 1 4 2
```这里$dx = 0$,$dy = -2$。 

| 端点 | t值|
 | ---| ---|
 | (2,1) | -2 |
 | (4,2) | -4 |

 间隔变为[-4,-2]。 

无需合并。 

答案是间隔长度除以$\sqrt{dx^2 + dy^2} = 2$，给予：$$\frac{2}{2} = 1$$这表明单个段干净地映射到一个连续的阻塞范围。 

### 示例 2

 输入：```
6 0 -2
1
2 1 4 2
```现在$dx = 2$,$dy = 0$。 

| 端点 | t值|
 | ---| ---|
 | (2,1) | 4 |
 | (4,2) | 8 |

 间隔为 [4, 8]，长度为 4，归一化为$\sqrt{4} = 2$，给出 2。 

这对应于沿边界映射水平覆盖范围的垂直射线。

 ## 边缘情况

 关键的边缘情况是当线段平行于射线方向时。 在这种情况下，两个端点都会投影到相同的值，从而产生零长度间隔。 该算法自然会忽略它，因为它不会增加联合长度。 

另一种情况是许多重叠的片段形成一条链。 扫描正确地将它们合并为一个连续的间隔，从而防止重复计数。 

最后，当片段相距较远时，算法会生成不相交的间隔，其长度独立求和，以满足单独窗帘部分的需要。
