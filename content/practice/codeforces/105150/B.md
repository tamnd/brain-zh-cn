 ## 解决方案
 ## 问题理解

 我们有两个独立的累进税制和固定的总收入$X$。 德米特里和安娜必须将这笔收入分成两部分：德米特里宣称$t$，安娜宣称$X - t$。 每个国家都按括号对收入征税，其中括号内的每一部分收入均按固定百分比征税，收入较高的部分可能按不同税率征税。 

关键的复杂性在于，两个税收函数都是分段线性的，但断点和税率可能不同。 对于任何选择的分割$t$，总税收是两个分段线性评估的总和：一个为$t$和一个用于$X-t$。 任务是选择$t$最小化这个总和。 

约束条件很大：最多$10^5$税收水平和收入值高达$10^{16}$。 这立即排除了任何以简单的单位方式评估税收或在线性收入空间中增量重新计算贡献的方法。 甚至一个$O(X)$扫荡是不可能的，因为$X$可以是$10^{16}$。 

更深层次的结构是，每个税收制度都定义了一个凸的、连续的、分段线性的收入函数。 两个这样的函数的总和，其中一个应用于$t$另一个到$X-t$，仍然是一个凸函数$t$。 这强烈表明可以通过搜索断点而不是所有值来找到最佳点。 

括号边界处出现了一个微妙的问题。 仅考虑断点的简单实现可能会错过最佳值可能位于斜率恒定的段内。 另一个常见的陷阱是由于溢出或错误处理斜率前缀和而错误地计算大区间的税收。 

## 方法

 一个蛮力的想法是尝试每一种可能的分割$t$从$0$到$X$，使用括号定义分别计算德米特里的税和安娜的税，并取最小值。 这是正确的，因为它直接评估所有有效输入的目标函数。 然而，评估单个值的税收需要遍历括号，在最坏的情况下，这是$O(n + m)$。 结合多达$10^{16}$的可能值$t$，这变得完全不可行。 

关键的观察是每个税收制度都是分段线性函数。 在相邻阈值之间的任何间隔内，边际税率都是恒定的。 这意味着总税收函数仅在已知断点位置处发生斜率变化。 如果我们将德米特里的税定义为$f(t)$安娜的税为$g(X-t)$，那么总成本是$F(t) = f(t) + g(X-t)$。 在任一系统的任意两个连续断点之间（针对第二个函数适当移动），两者$f$和$g$是线性的，所以$F$也是线性的。 因此，最优结果必须出现在这些断点之一或边界处$0$和$X$。 

这将问题简化为构建所有候选点，其中德米特里的税收斜率发生变化或安娜的税收斜率发生变化（通过转换$X - b_i$），对它们进行排序，并仅在这些点评估目标。 由于每个系统贡献最多$10^5$断点后，候选人总数是可控的。 

我们仍然需要一种有效的方法来计算给定点的税收。 这可以在$O(\log n)$在阈值加上分段贡献的前缀和上使用二分搜索，或者在$O(n)$如果我们预先计算前缀结构并仔细处理查询。 通过排序阈值，每个分段的税额前缀累加给出了直接的评估公式。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力胜过一切$t$|$O(X(n+m))$|$O(1)$| 太慢了 |
 | 断点+评估|$O((n+m)\log(n+m))$|$O(n+m)$| 已接受 |

 ## 算法演练

 我们将每个税收系统视为一个函数，可以使用分段的前缀和快速计算任何收入的税收。 

1. 预先计算两国每个等级的累计税收贡献。 这使我们能够通过定位其括号并添加前一个括号的全部贡献加上最后一个括号中的部分贡献来评估任何收入的税收。 
2. 构建候选分割点列表。 对于 Dmitry，所有阈值$a_i$是候选人。 对于 Anna，相应的分割点为$t$是$X - b_i$，因为如果安娜的收入超过$b_i$，德米特里的份额必须是$X - b_i$。 
3. 添加候选边界$0$和$X$。 这些都是必要的，因为当所有收入都分配给一侧时，就会出现最佳状态。 
4. 对所有候选点进行排序和去重。 在任意两个连续候选之间，目标函数是线性的，因此没有内部点可以改进端点。 
5. 评估每个候选人的总税额$t$通过计算$f(t) + g(X-t)$，并跟踪最小值。 
6. 返回$t$产生最小值。 

为什么它起作用与凸性有关。 每个税收函数仅在超过阈值时增加斜率，因此两者$f$和$g$是凸且分段线性的。 转变$g(X-t)$翻转但保留分段线性结构。 凸函数之和是凸函数，所以$F(t)$除全局最小值外没有局部最小值，并且只能在断点边界处改变斜率。 因此，仅检查这些点就足够了。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def build(prefix, rates, caps):
    n = len(rates)
    pref = [0] * (n + 1)
    for i in range(n):
        pref[i+1] = pref[i] + caps[i] * rates[i]
    return pref

def tax(x, caps, rates, pref):
    n = len(rates)
    if n == 0:
        return x * rates[0] / 100.0

    if x <= 0:
        return 0.0

    lo, hi = 0, n
    while lo < hi:
        mid = (lo + hi) // 2
        if x > caps[mid]:
            lo = mid + 1
        else:
            hi = mid

    i = lo
    res = 0.0

    if i > 0:
        res += pref[i]

    prev = caps[i-1] if i > 0 else 0
    rate = rates[i]
    res += (x - prev) * rate

    return res / 100.0

n = int(input())
if n:
    a = list(map(int, input().split()))
else:
    a = []
p = list(map(int, input().split()))

m = int(input())
if m:
    b = list(map(int, input().split()))
else:
    b = []
q = list(map(int, input().split()))

X = int(input())

pref_d = build([], p, a)
pref_a = build([], q, b)

cands = {0, X}
for v in a:
    cands.add(v)
for v in b:
    cands.add(X - v)

cands = sorted(cands)

best = None
best_t = 0

for t in cands:
    val = tax(t, a, p, pref_d) + tax(X - t, b, q, pref_a)
    if best is None or val < best:
        best = val
        best_t = t

print(best_t)
```核心实施思想是将税收评估为整个括号加上部分最后一段的前缀和。 二分搜索找到收入属于哪个档次，然后我们添加前一个档次的全部贡献和当前档次的部分贡献。 

候选构建步骤至关重要：将 Anna 的阈值转化为$X - b_i$将她的坡度变化与德米特里的坐标系对齐。 这使得我们能够将问题视为单变量优化。 

一个微妙的点是处理空税制。 什么时候$n = 0$，只有一种固定费率，因此评估必须避免索引到阈值数组中。 另一个微妙的问题是浮点运算。 尽管问题要求整数输出，但中间税值是小数，因此比较是以浮点数进行的，但如果需要，可以用缩放整数替换。

 ## 边缘情况

 一种边缘情况是所有费率都为零。 在这种情况下，税收函数完全为零，因此每次分割都是最优的。 该算法仍然有效，因为所有候选评估都返回零，并且选择了第一个候选。 

另一种情况是当$X$小于所有阈值。 然后，两个税收函数都简化为单个线性段，并且纯粹通过比较零收入下的恒定边际税率来确定最佳解决方案。 候选集仍然包括 0 和 X，因此评估正确的端点。 

最后一个微妙的情况是$X$大于两个系统中的所有阈值。 那么这两个函数在任何地方都处于最后一段，并且目标在整个域上是线性的。 该算法评估该线性线段集的端点，即使内部结构无关紧要，也能保证正确性。
