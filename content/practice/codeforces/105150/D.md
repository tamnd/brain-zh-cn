 ## 解决方案
 ## 问题理解

 我们被要求想象一个由可以写成以下形式的数字构建的无限递增序列$$x = 2^k + 60m$$在哪里$k$和$m$是正整数（或者至少是正整数$k$，并且非负$m$，取决于解释； 重要的是两个组件都可以自由且独立地做出贡献）。 每个有效数字都被放入排序列表中，重复项将被忽略，因为我们只关心不同的整数，并且我们想要$n$- 该集合的第一个最小元素。 

因此，任务不是评估单个数字的公式，而是理解所有可表示数字的全局结构，并从该结构中提取特定的排名统计数据。 

约束条件$n \le 10^{15}$立即排除任何枚举值甚至构造序列的大前缀的方法。 甚至生成$10^7$每秒的候选人数仍远低于所需的数量。 该解决方案必须依赖于集合的结构属性而不是显式构造。 

一个幼稚的错误是尝试生成达到某个界限的所有值，例如通过迭代$k$和$m$并插入到一个集合中。 这会失败，因为两个参数都无限增长，并且映射到小值的对的数量已经足够大，使得排序不可行。 另一个错误的方向是假设任一参数的单调性，例如固定$k$并不断增加$m$，然后合并序列，但无法识别重叠模式，这会导致重复计数和错误排序。 

当人们假设很小的时候，就会发生一种更微妙的失败。$k$总是主导排序。 尽管$2^k$是结构化的，60的倍数相加会导致不同的值之间交错$k$-以一种不平凡的方式分层。 

## 方法

 蛮力视图很简单：枚举所有对$(k, m)$, 计算$2^k + 60m$，插入到集合中，排序，然后取$n$-th 元素。 这在原则上是正确的，因为它直接构造了定义。 问题在于规模。 即使我们将值限制在某个较大的范围内$X$，有效对的数量大致按比例增长$X \log X$，它太大了，无法一一列举$n$最多$10^{15}$。 

关键的洞察力是翻转视角。 我们不生成数字，而是问：给定阈值$X$, 有多少个有效数字$\le X$？ 如果我们能够有效地回答这个计数问题，我们就可以对答案进行二分查找$n$-th 值。 这是“找到$n$-隐式排序集中的第一个元素”来进行前缀计数。

 对于固定的$k$, 表格中的数字$2^k + 60m \le X$对应于所有$m \le \frac{X - 2^k}{60}$。 当$2^k \le X$。 唯一的困难是总结所有可行的$k$，但自从$2^k$相关数量呈指数级增长$k$价值观只是$O(\log X)$。 

因此，我们可以计算每个查询的对数时间计数，然后进行二分搜索$X$。 答案必须位于最小可能值和足够大的上限之间，该上限可以安全地围绕$2^{60} + 60n$在最坏情况的推理中。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | 指数/不可行 | 大| 太慢了 |
 | 二分查找+计数|$O(\log X \cdot \log X)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 定义一个函数`count(X)`计算有多少个这样的数字$2^k + 60m$小于或等于$X$。 该函数驱动整个解决方案，因为它将集合转换为前缀可数结构。 
2. 内部`count(X)`，迭代二的幂。 开始于$2^k = 2$并反复乘以2，直到超过$X$。 每个这样的值代表完整算术级数的固定基偏移量$m$。 循环很短，因为 2 的幂呈指数增长。 
3.对于每个固定$2^k$，计算在保持不变的情况下可以添加多少个 60 的倍数$X$。 这是$\max(0, \lfloor (X - 2^k) / 60 \rfloor)$。 将其添加到运行总计中。 这是有效的，因为对于固定的$k$，有效数字形成一个简单的算术级数，从$2^k$与步骤 60 一起。 
4. 对答案空间执行二分查找。 保持一个下限和上限，其中上限选择得足够大以超过$n$-th 值。 在每一步中，计算 mid 并评估`count(mid)`。 
5. 如果`count(mid) >= n`，答案位于中值或中值以下，因此将上限向下移动。 否则，将下限向上移动。 这是标准的单调谓词搜索。 
6. 收敛后，下界表示最小值，使得至少$n$数字是$\le$它，这正是$n$-th 元素。 

### 为什么它有效

 正确性取决于计数函数的单调性。 如果$X_1 < X_2$，那么每个可表示的数字$\le X_1$也是$\le X_2$， 所以`count(X)`是非减的。 这保证了二分查找的有效性。 

每个固定$k$贡献连续的算术级数$m$，因此固定层内不存在间隙或不规则现象。 对各层进行求和可以保持单调性，并确保不会跨阈值进行过度计数。 二分搜索隔离了前缀计数达到的精确截止点$n$，它必须对应于$n$- 可表示的最小数。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def count(x):
    total = 0
    p = 2
    while p <= x:
        total += max(0, (x - p) // 60)
        p <<= 1
    return total

def solve():
    n = int(input())
    
    lo, hi = 1, 10**18
    
    while lo < hi:
        mid = (lo + hi) // 2
        if count(mid) >= n:
            hi = mid
        else:
            lo = mid + 1
    
    print(lo)

if __name__ == "__main__":
    solve()
```该代码完全按照派生的方式实现计数函数。 一旦值超出搜索范围，二次幂循环就会自然停止。 二分查找范围$10^{18}$就足够了，因为即使$10^{15}$鉴于 60 步间距和指数碱基增长，第 - 个数字不能超过该比例。 

一个微妙的点是两个的起始功率。 最小的有意义$2^k$是$2$， 自从$k$是积极的。 从 1 开始会错误地包含无效表示。 计数步骤中的楼层划分确保我们在以下情况下永远不会计算负贡献：$2^k > x$。

 ## 边缘情况

 关键的边缘情况发生在以下情况：$X$小于最小有效基数$2$。 在此区域中，计数函数必须返回零。 该循环正确地避免了添加任何内容，因为没有任何 2 的幂满足$p \le X$。 

另一个微妙的情况是当$X$位于 2 的两个幂之间，例如$X = 63$。 仅此$2,4,8,16,32$贡献，同时$64$被排除在外。 每个层贡献的数量可能不同，均为 60 的倍数，并且该公式确保每个层都独立评估，无需交互。 这可以防止意外的跨层计数，否则会破坏二分搜索所需的单调性假设。
