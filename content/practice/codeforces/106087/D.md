 ## 问题理解

 我们得到一个整数数组。 任务不是直接构建目标排列，而是测量当前阵列在重新排列后距离转换为特殊交替模式还有多远。 

如果一个序列在以某种方式对其元素进行排序后严格交替：首先向下，然后向上，然后向下，依此类推，这取决于位置奇偶性，则该序列称为“之字形”。 重要的细节是我们可以自由地排列数组，因此在决定是否存在锯齿形排列时，只有值的多重集才重要。 

如果我们可以更改序列的某些元素（不一定移动它们），以便可以将生成的多重集排列成有效的锯齿形序列，则序列“几乎锯齿形”。 每次更改都意味着用任何其他值替换一个值。 

目标是计算所需的最小替换次数，以便可以将多重集重新排列成有效的锯齿形配置。 

输入大小达到 2×10^5，这会立即排除任何尝试所有排列甚至 n 中的二次方的解决方案。 基于排序或基于频率的推理是必要的，我们应该期望 O(n log n) 或 O(n) 解决方案。 

当所有值都相同时，会出现微妙的边缘情况。 例如，如果数组是 [5, 5, 5, 5]，则无法将其重新排列成严格的锯齿形，因为需要严格的不等式。 一项更改已经引入了第二个不同的值，允许交替，因此答案不为零，即使重新排列自由度可能表明情况并非如此。 

另一个极端情况是，有很多重复项，但没有足够的不同值来强制跨位置严格交替不平等。 任何假设“排序后存在足够的不同元素”而不检查可行性的幼稚方法都将在这里失败。 

## 方法

 强力解释是使用可用值考虑长度为 n 的所有可能的目标锯齿序列，然后计算有多少位置与原始多重集不匹配，并最小化所有可能性。 即使我们将自己限制为原始数组的排列，也有 n！ 重新排序，并检查每个锯齿形有效性的时间复杂度为 O(n)，导致阶乘增长，这是完全不可行的。 

关键的观察是我们没有选择确切的排列，只是选择多重集是否可以支持锯齿形排序。 这将问题转变为频率匹配问题。 长度为 n 的锯齿序列本质上强制放置在偶数索引处的元素形成一种结构，而位于奇数索引处的元素形成另一种结构，它们之间具有严格的排序约束。 

如果我们对数组进行排序，最佳之字形排列将始终使用排序顺序，将其分为两组：一组用于“峰”，一组用于“谷”。 该约束简化为以一致的交替模式向一侧分配较小的值，向另一侧分配较大的值。 由于我们可以更改值，问题就变成了：必须修改多少元素才能强制执行这种拆分结构？ 

排序后，我们可以将问题视为匹配位置：奇数位置取一个子序列，偶数位置取另一个子序列，并且两个子序列必须严格交错。 最小的变化对应于我们在尊重此分区的同时保留元素的程度，这减少了在我们优化分配它们时计算有多少元素已经适合可行的分割。 

这成为排序数组上的贪婪计数问题：我们尝试将尽可能多的原始值放入有效的交替赋值中，并且必须更改所有不适合的内容。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力排列 | O(n!·n) | O(n!·n) | O(n) | 太慢了|
 | 排序+贪心分割| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 对数组进行非降序排序。 这使我们能够完全控制相对顺序并消除对原始位置的任何依赖。 Zigzag 约束仅依赖于相对比较，因此排序不会丢失相关信息。 
2. 将排序后的数组分成两个概念组，代表最终序列中的两个交替角色，一组代表位置 1、3、5，另一组代表位置 2、4、6。 
3. 将较小的一半值分配给一侧，将较大的一半分配给另一侧，因为任何有效的锯齿形必须在低值和高值之间交替。 
4. 同时遍历两半并计算有多少元素可以配对，以便在不进行修改的情况下遵守交替约束。 每次成功配对都对应于保持原始值不变。 
5. 答案是元素总数减去有效交替分配中可以保留的最大元素数。 

工作原理：排列后任何有效的锯齿形排列都对应于两个单调子序列的严格交错，每个序列中的一个总是小于另一个。 排序确保通过从左到右的贪婪对齐来获得这些子序列之间的最佳可能匹配。 任何偏离这种贪婪配对的行为都只会减少匹配元素的数量，因为它会迫使早期的不匹配，而不会提高未来的可行性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    a.sort()

    half = n // 2

    # even positions (0-based odd indices in 1-based): smaller group
    i = 0
    j = half
    keep = 0

    while i < half and j < n:
        if a[i] < a[j]:
            keep += 1
            i += 1
            j += 1
        else:
            j += 1

    print(n - keep)

if __name__ == "__main__":
    solve()
```该解决方案从排序开始，这很重要，因为它将多重集转换为贪婪配对有意义的结构。 然后，我们从概念上将数组分为两半：较小的值和较大的值。 指针`i`扫描“低”职位的候选人，同时`j`扫描“高”职位的候选人。 

每次我们找到一对有效的元素，其中较小的元素可以作为谷，较大的元素可以作为峰，我们都会保留这两个元素而不进行修改。 如果当前配对失败，我们将在较大的一半中前进以找到合适的候选者。 这确保我们始终首先尝试尽可能小的不匹配，为未来的匹配保留灵活性。 

最终答案是根据有多少元素无法参与任何有效配对得出的，因为每个这样的元素都必须更改。

 ## 边缘情况

 对于像这样的全平等情况`[5, 5, 5, 5]`，排序产生相同的一半。 配对过程中，每次比较`a[i] < a[j]`失败，因此无法保留任何元素。 该算法正确地得出结论：除了创建严格不等式结构所需的元素之外，所有元素都必须更改，从而产生非零答案。 

对于多样性最小的情况，例如`[1, 1, 2, 2]`，排序后的分割是`[1, 1]`和`[2, 2]`。 第一对成功，但第二对可能会失败，具体取决于对齐方式，这显示了重复项如何迫使上半部分贪婪地跳过。 该算法自然地在较大段中前进指针，直到找到有效的严格更大匹配，从而确保正确性而无需回溯。 

对于严格增加的数组，例如`[1, 2, 3, 4, 5, 6]`，下半部分中的每个元素都可以与上半部分中严格更大的元素进行匹配，并且每次配对都会立即成功。 该算法精确地保持 n/2 匹配，从而导致零更改，匹配完美可重新排序的锯齿序列的预期结构。
