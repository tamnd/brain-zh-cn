 ## 解决方案
 ## 问题理解

 我们被要求计算自然数的有序三元组$(a, b, c)$，每一个之间$1$和$n$，这样可以使用固定结构中的加法或乘法从前两个数字获得第三个数字：$a$和$b$结合形成$c$作为任一$a + b = c$或者$a \cdot b = c$。 

顺序很重要，所以$(1, 2, 2)$和$(2, 1, 2)$是不同的，因为中间操作数改变了算术行为。 这项任务纯粹是组合性的：我们必须计算有多少个三元组满足这两个方程中的至少一个，同时保持在界限内。 

约束条件$n \le 10^7$立即排除所有三元组上的任何三次或二次枚举。 甚至一个$O(n^2)$在这种规模下，采用恒定时间检查的方法会太慢。 该解决方案必须避免迭代所有对$(a, b)$明确地。 

当两个操作同时产生有效结果时，就会出现微妙的边缘情况，这种情况发生在$a \cdot b = a + b$。 这只发生在非常小的值，例如$(2, 2)$， 自从$2 \cdot 2 = 4$和$2 + 2 = 4$。 如果单独计算的话，如果不小心处理的话，这种情况就会被重复计算。 

## 方法

 直接方法将迭代所有$a$和$b$, 计算$c = a + b$和$c = a \cdot b$，并检查是否$c \le n$。 这在逻辑上是正确的，但需要检查所有$n^2$对。 和$n = 10^7$，这是完全不可行的，按以下顺序生成$10^{14}$运营。 

关键的观察是我们实际上不需要选择$c$独立。 一次$a$和$b$是固定的，$c$对于每个操作都是唯一确定的。 所以问题减少到计算有效对$(a, b)$这样$a + b \le n$或者$a \cdot b \le n$。 

加性条件在对网格中形成一个三角形区域，并且可以通过分析计算其计数。 对于每个$a$，有效的$b$值来自$1$到$n - a$，给出一个简单的算术级数。 

乘法条件更加结构化：对于每个$a$， 有效的$b$值满足$b \le \lfloor n / a \rfloor$。 这将问题转化为除数的调和式求和，可以用以下公式计算：$O(n)$时间或使用分组技术进一步优化，但在考虑到约束和常数因素的情况下，直接求和已经足够了。 

唯一剩下的复杂性是重叠：在两种条件下计数的对完全对应于$a + b = a \cdot b$。 这个方程重新排列为$(a - 1)(b - 1) = 1$，所以唯一的正整数解是$a = b = 2$。 这一对贡献一个有效的三元组，并且不得重复计算。

 ## 算法演练

 ### 1. 计算加法三元组

 我们修复$a$并计算所有$b$这样$a + b \le n$。 对于给定的$a$,$b$范围从$1$到$n - a$, 贡献$\max(0, n - a)$有效对。 总结一切$a$产生三角形数字结构。 

### 2. 计算乘法三元组

 我们修复$a$并确定有多少$b$满足$a \cdot b \le n$。 这相当于$b \le \lfloor n / a \rfloor$。 对于每个$a$，我们将此数量添加到答案中。 

这样做的原因是乘法约束对每个固定的第一个操作数都进行了清晰的分解，这与加法减少到线性截止不同。 

### 3. 合并结果

 我们将这两项贡献相加。 此时，每个有效对每次操作都会被计数一次。 

### 4. 消除重复计算

 我们减去两个条件都成立的交集。 解决$a + b = a \cdot b$给出$(a - 1)(b - 1) = 1$，所以唯一有效的自然解是$(a, b) = (2, 2)$。 这恰好贡献了三倍$(2, 2, 4)$，所以我们减去 1 如果$n \ge 4$。 

### 为什么它有效

 每个有效的三元组都根据它是否满足加法或乘法进行唯一分类。 加法计数耗尽所有线性和有效对，乘法计数耗尽所有乘积有效对。 它们的交集是一个单例集，因此包含-排除将整个问题简化为两个独立的一维求和加上常数校正。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n = int(input())

# additive pairs: sum_{a=1..n} max(0, n - a)
add = n * (n - 1) // 2

# multiplicative pairs: sum_{a=1..n} floor(n / a)
mul = 0
for a in range(1, n + 1):
    mul += n // a

# overlap correction: only (2,2) if n >= 4
overlap = 1 if n >= 4 else 0

print(add + mul - overlap)
```加法部分使用封闭式三角数公式计算，避免了迭代。 乘法部分是通过除数的单个循环计算的，这是可以接受的$n \le 10^7$由于简单的整数运算，可以在 PyPy 或优化的 CP Python 下使用。 

重叠的减去是必不可少的，因为两个循环独立地计算该对$(2,2)$，并且没有其他对满足这两个方程。

 ## 边缘情况

 对于$n = 1$， 仅有的$(1,1,2)$或者$(1,1,1)$样式结构可以正确减少，因为乘法和加法约束是严格限制的。 该算法给出加法 = 0，乘法 = 1，无重叠，产生 1。 

对于$n = 3$，没有有效的重叠，因为唯一的交集需要$c = 4$，超出了限制。 该算法正确地避免减去该范围内的任何内容。 

为了$n \ge 4$，这对$(2,2)$在两种结构中都有效。 减法步骤恰好删除了一个重复项，并且由于不存在其他整数解$(a-1)(b-1)=1$，无需进一步修正。
