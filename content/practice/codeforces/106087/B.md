 ## 解决方案
 ## 问题理解

 我们给出了总距离$s$和固定增量$k$。 一系列跑步者（在语句中称为“鸽子”）产生一个正整数列表$a_1, a_2, \dots, a_n$使得第一个值至少为 1，并且每个下一个值至少为$k$比前一个更大。 所有值的总和恰好是$s$。 任务是确定最大可能的长度$n$这样的一个序列。 

换句话说，我们希望将尽可能多的项打包成一个严格递增至少的项。$k$类似算术的序列，同时保持总和固定为$s$。 第一项仅限制为至少 1，并且各个值没有上限，但总和必须匹配$s$。 

这些约束清楚地表明我们不能尝试所有的值$n$通过显式构建序列。 总和$s$可以达到$10^{18}$，因此任何解决方案都必须在每个测试用例的对数或恒定时间内工作。 

出现微妙的边缘情况时$k = 0$是不允许的，但是$k = 1$是。 在这种情况下，序列变成一个具有最小间隙 1 的简单严格递增序列，并且与大结构相比，最优结构在大小上表现不同$k$。 另一个边缘情况是当$s$非常小，例如$s = 1$，无论什么，答案都是 1$k$。 假设至少有两项或从 0 开始构建的天真的尝试将会失败。 

## 方法

 直接的方法是尝试固定长度$n$，构造该长度的最小可能有效序列，并检查其总和是否超过$s$。 对于固定的$n$，通过设置获得最小有效序列$a_1 = 1$,$a_2 = 1 + k$,$a_3 = 1 + 2k$， 等等。 这是最佳的，因为任何较大的起始值只会增加总和。 

这个最小序列的总和是$$S(n) = n + k(0 + 1 + \dots + (n-1)) = n + k \cdot \frac{n(n-1)}{2}.$$如果$S(n) \le s$，然后是一个长度序列$n$是可能的； 否则就不是。 这将问题简化为寻找最大的$n$使得二次不等式成立。 

蛮力的想法会增加$n$直到总和超过$s$，最多可以达到$O(\sqrt{s})$每个测试用例或更糟。 和$s \le 10^{18}$，这仍然可以是边界，但对于$t = 1000$如果天真地实施，在最坏的情况下。 

关键的观察是$S(n)$是单调的$n$。 一旦某个长度无效，所有更大的长度也都无效。 这允许二分搜索$n$。 自从$S(n)$平方增长，最大可能$n$大致是$O(\sqrt{s})$，因此搜索范围小且稳定。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力增加$n$|$O(\sqrt{s})$每次测试 |$O(1)$| 太慢了|
 | 二进制搜索$n$|$O(\log s)$每次测试 |$O(1)$| 已接受 |

 ## 算法演练

 我们将问题简化为检查给定序列长度的可行性$n$。 

1.对于固定的$n$，计算最小可能的总和$S(n) = n + k \cdot n(n-1)/2$。 这对应于最小的有效级数，因为任何项的任何增加只会增加总和。 
2.如果$S(n) \le s$， 然后$n$是可行的。 否则就不是。 
3. 使用二分查找$n$，从 1 开始到安全上限。 自然界限是$2 \cdot 10^9$，但在实践中$\sqrt{2s/k}$就足够了； 我们可以安全地夹在$10^{9}$。 
4. 在二分查找中，使用公式反复检验中点。 如果可行，向右移动尝试更大$n$，否则向左移动。 
5.最终的答案是最大可行的$n$。 

重要的微妙之处在于计算时使用 64 位算术$k \cdot n(n-1)/2$。 虽然$n$实际上很小，中间乘法可能会溢出 32 位类型。 

### 为什么它有效

 对于任何有效的长度序列$n$，替换每个$a_i$与约束一致的最小可能值会产生具有相同长度且没有更大总和的序列。 这种转变严格减少或保留总和，因此可行性完全由最小构造决定。 由于最小总和是单调的$n$，二分查找正确识别最大可行长度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def feasible(n, s, k):
    return n + k * n * (n - 1) // 2 <= s

def solve():
    t = int(input())
    for _ in range(t):
        s, k = map(int, input().split())

        lo, hi = 1, 10**9
        ans = 1

        while lo <= hi:
            mid = (lo + hi) // 2
            if feasible(mid, s, k):
                ans = mid
                lo = mid + 1
            else:
                hi = mid - 1

        print(ans)

if __name__ == "__main__":
    solve()
```功能`feasible`对导出的最小和公式进行编码。 除法是安全的，因为它始终使用整数运算。 二分查找保持了所有值的不变量`ans`都是可行的，并且任何超出的事情`hi`不是。 

的上限$10^9$是安全的，因为即使$k = 1$，总和增长得像$n^2/2$，这超过了$10^{18}$很久以前$10^9$。

 ## 边缘情况

 对于$s = 1$， 不管$k$，只能存在一个元素，因为即使是长度为 2 的最小序列也至少需要 sum$1 + (1 + k) > 1$。 该算法处理这个问题是因为$S(2) = 2 + k > 1$，因此二分查找永远不会超出 1。 

对于非常大的$k$， 例如$k \ge s$光是第二期就已经超出了预算。 例如，$s = 10, k = 20$。 然后$S(2) = 2 + 20 = 22$，所以只有$n = 1$是可行的。 可行性检查立即拒绝$n = 2$，答案仍然是 1。 

对于$k = 1$，生长最慢，产生最大可能的长度。 该算法仍然有效，因为它简化为三角形数，并且二分搜索正确地找到了最大的$n$满意的$n(n+1)/2 \le s$。
