 ## 解决方案
 ## 问题理解

 我们从 0 层开始，想要到达目标楼层 D。每次移动时，电梯都允许两种可能的操作：上 3 层楼或下 2 层楼。 每个动作都算作按下一个按钮。 任务是确定精确达到 D 所需的最少按钮按下次数。 

这是整数状态无限行上的最短路径问题。 每层都是一个节点，从每个节点 x 我们可以以 1 的成本到达 x + 3 或 x − 2。目标范围很小，D 在 −1000 到 1000 之间，因此状态空间足够小，甚至相当直接的图搜索也是可行的。 

主要的微妙之处在于达到 D 并不是单调的。 使用 +3 向 D 移动可能会出现过冲，需要使用 -2 进行修正，反之亦然。 像“总是靠近”这样的贪婪策略会失败，因为以小周期为模的接近度比原始距离更重要。 

一个幼稚的错误是尝试确定性公式，例如始终仅使用 +3 步或仅使用 −2 步，或线性组合，例如求解 3a − 2b = D 并最小化 a + b，而不确保具有最小非负解的整数的可行性。 这些方法很快就会失效。 

例如，如果 D = 1，人们可能会认为“+3 then -2 在 2 步中得到 1”，这是正确的，但对于 D = 2，贪婪思维可能会建议 +3 then -1 调整是不可能的，人们必须探索组合。 该结构不是线性优化，它是两步图上的最短路径。 

因为 D 以 1000 为界，所以 k 步中从 0 开始的完整可到达区域大约在 ±3k 范围内，因此 k 达到几百就足够了。 这建议采用 BFS 式解决方案或状态动态规划。 

边缘情况包括：

 如果 D = 0，则答案为 0，因为我们已经到了那里。 

如果D为负，则只能通过+3和-2的组合才能达到，因此我们必须首先允许超调正侧。 

如果 D 很小，例如 1 或 2，则最佳解决方案涉及混合步骤而不是直接移动。 

## 方法

 暴力方法是将其视为无限整数图上的未加权最短路径问题，并从 0 开始运行 BFS，通过应用 +3 和 -2 扩展状态，在我们第一次到达 D 时停止。这是正确的，因为每次移动的成本相等，因此 BFS 保证最小步数。 

然而，在没有约束的情况下，图是无限的，因此简单的 BFS 可以永远持续下去。 关键的观察结果是，尽管图是无限的，但到达 [−1000, 1000] 范围内任何 D 的最佳路径永远不需要探索稍大的有界区间之外的地方。 每次移动最多改变 3 个位置，因此在 k 步中我们在 [−3k, 3k] 范围内。 为了达到 1000 以内的任何 D，我们不需要超过 1000 左右的 k，因此大约 [−3000, 3000] 之外的位置是不必要的。 这使得 BFS 对于访问集来说是可行的。 

一个更结构化的观察是，这是一个具有单位成本边缘的线性丢番图可达性问题，并且 BFS 正在有效地按总步骤的递增顺序探索 +3 和 -2 的组合。 我们第一次点击 D 的时候就保证是最优的。 

暴力 BFS 可以工作，但如果无限制地实施，就会造成浪费。 优化版本与 BFS 相同，但具有固定的合理状态窗口和提前停止。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 无界BFS | O(∞) | O(∞) | 无法使用|
 | 有界 BFS | O(V) 其中 V ≈ 6000 | O(V) | 已接受 |

 ## 算法演练

 我们将每个楼层建模为图中的一个节点，并从 0 开始执行 BFS。

1. 初始化一个队列，从位置 0 开始，距离为 0。这表示在大厅中尚未按下任何按钮。 
2. 维护已访问过的集合以避免再次访问楼层。 这可以防止因在 +3 和 -2 之间循环而导致的无限循环。 
3. 当队列不为空时，提取当前楼层以及到达该楼层所需的步数。 这确保 BFS 按递增的步骤顺序处理状态。 
4. 如果当前楼层等于D，则立即返回当前步数。 BFS 保证这是可能的最小按下次数。 
5. 否则，生成两个邻居：current + 3 和 current − 2。对于每个邻居，如果它尚未被访问并且位于目标周围的合理范围内，则将其推入步骤计数 + 1 的队列中。 
6. 如果 BFS 在没有提前返回的情况下完成（这在正确的边界下不应发生），则目标无法到达，但在这个问题中，它始终是可到达的，因为 gcd(3, 2) = 1。 

为什么它有效：每次移动都有相同的成本，并且 BFS 从一开始就按照距离递增的顺序探索状态。 由于所有转换都有单位权重，因此我们第一次到达某个状态时保证会通过最短的操作序列。 访问集确保每个状态都被处理一次，从而保持正确性而无需冗余扩展。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    D = int(input().strip())

    if D == 0:
        print(0)
        return

    # safe exploration bounds
    LIM = 3000

    q = deque()
    q.append((0, 0))
    visited = set([0])

    while q:
        x, d = q.popleft()

        if x == D:
            print(d)
            return

        for nx in (x + 3, x - 2):
            if -LIM <= nx <= LIM and nx not in visited:
                visited.add(nx)
                q.append((nx, d + 1))

    # theoretically unreachable
    print(-1)

if __name__ == "__main__":
    solve()
```该代码直接在整数位置上实现 BFS。 队列存储当前楼层以及到达该楼层所用的按钮按下次数。 访问集确保我们永远不会重新处理楼层，从而防止指数爆炸在 +3 和 -2 之间重复循环。 

选择绑定 LIM 是为了确保 ±1000 范围内到达目标的任何最佳路径都不需要进一步漂移。 这是安全的，因为任何超出此范围的绕行都将需要额外的步骤，而这些步骤无法改进未加权图中已经最短的路径。

 ## 边缘情况

 当 D = 0 时，算法在进入 BFS 之前立即返回 0。 初始检查显式处理此问题，因此不需要队列操作。 

对于像 D = 2 这样的小正值，BFS 会探索 0 → 3 → 1 → 4 → 2，并正确识别出达到 2 需要两步。 贪婪的 +3 方法会错过这一点，并错误地表明过度调节是不可避免的。 

对于像 D = -5 这样的负目标，BFS 自然会通过重复的 -2 步移动扩展到负区域，但也会在需要时探索临时的正位置。 访问集确保诸如 0 → 3 → 1 → 4 → 2 → 0 之类的循环不会被重复访问，因此搜索仍然是有限的，并且仍然找到最短路径。
