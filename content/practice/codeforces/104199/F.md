 ## 解决方案
 ## 问题理解

 我们有$n$朋友们站在编号为 1 到 1 的房间中$n$。 每个朋友最初都持有一个包裹，必须将其准确地交付给另一个朋友，并且每个朋友既是发送者又是接收者。 所以数据描述了一个有向图$n$每个节点都只有一个出边的节点。 

地板下有一个与房间对齐的输送系统，但它比建筑物的占地面积长。 它有$3n$职位：$n$位置延伸到第一个房间的左侧，$n$位置位于房间正下方，并且$n$位置向右延伸。 全局移位操作将每个包向左或向右移动一位。 当包裹在某个时刻恰好位于目的地朋友的房间下方时，即被视为已送达。 

目标是应用一系列左移和右移，以便每个包裹至少到达其目的地位置一次，同时永远不会离开输送机范围。 

重要的观察结果是每个包裹同步移动。 我们不是独立地运送物品，而是将包含所有包裹的严格线路转移到一起。 这将问题转化为寻找一系列全局翻译，将每个源位置与其目标位置同时对齐至少一次。 

限制条件$n \le 100{,}000$意味着任何解决方案都必须本质上是线性或近线性的。 对朋友的二次推理立刻就太慢了。 任何试图模拟对之间的所有转变或所有交互的方法都将失败。 

一个微妙的边缘情况是，如果单独处理，多个包可能需要不兼容的转换。 例如，如果 1 发送到 2，2 发送到 1，则一个包需要右移，另一个需要左移才能对齐，因此简单的每边对齐不起作用。 

另一个重要的情况是长度超过 2 的循环结构，其中最佳移位必须重用中间对齐，而不是独立处理每个边缘。 贪婪的局部对齐方法在这里破裂，因为满足一对的转变可能会破坏其他对的对齐。 

## 方法

 一个蛮力的想法是模拟一段时间内所有可能的传送带移动，并检查所有包裹何时与目的地对齐。 由于输送机具有$3n$位置，每次位移为一个单位移动，总位移范围为$O(n)$，可能的状态数已经是$O(n)$，并且每个状态都需要检查所有$n$包。 这导致$O(n^2)$或更糟糕的行为，这对于$n = 10^5$。 

关键的结构洞察力是停止思考绝对位置，而是着眼于源和目的地之间的相对位移。 每包$i$必须经历使其偏离原来位置的转变$i$定位$a_i$。 该要求相当于说在某个时候，全局偏移等于$a_i - i$。 

因此，每个包都会对全局移位值施加约束。 我们并不试图永远同时满足他们，每个包裹至少只满足一次。 这将问题转化为分析在向左或向右扫动传送带时必须经过多少个不同的移位值。 

关键的简化在于，最优策略的移位值是单调的。 我们只需要从某个最左边的所需移位移动到某个最右边的所需移位，并且其间的每个整数移位都可以通过连续移动来实现。 总成本变成了我们必须遍历的区间的大小，加上由于循环引起的包裹状结构而在单个区间中未连接约束时所需的额外移动。 

我们将问题转化为收集所有所需的班次$d_i = a_i - i$，然后计算在从 0 开始并且可以向左或向右移动的整数线上单次行走访问所有这些值所需的最小单位移动数，同时保持在防止掉落的范围内$3n$输送带。 由于传送带足够宽，边界不会渐近绑定； 他们只确保可行性。 

对相等的移位进行分组后，问题简化为以最小的遍历覆盖所有所需的整数，这是通过对唯一值进行排序并以相当于总跨度遍历的方式对间隙求和来实现的。

 | ---| ---| ---| ---|
 | 轮班强力模拟 |$O(n^2)$|$O(n)$| 太慢了|
 | 差集+区间遍历 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 ### 关键思想

 各位朋友$i$要求在某个时刻，传送带移动$x$满足$i + x = a_i$， 所以$x = a_i - i$。 因此，我们将问题简化为访问所有所需的移位值。 

### 步骤

 1. 计算一系列位移要求$d_i = a_i - i$为所有人$i$。 

这将转换“何时打包$i$对齐？” 进入全局移位的单个整数条件。 
2. 对所有值进行排序$d_i$。 

排序揭示了整数线上所需对齐的几何结构。 
3. 将排序后的值视为数轴上的点，并计算从 0 开始访问所有这些点的最小游走。 

最佳行走方式是先走向一个极端，然后再扫向另一个极端。 
4. 确定从 0 开始位于点簇的左侧还是右侧，并计算到最近端点的距离，然后添加全跨度长度。 

这说明了这样一个事实：我们必须首先达到所需的状态集，然后才能遍历所有状态。 
5. 返回总距离作为答案。 

关键的实现细节是，答案仅取决于集合的最小值和最大值，加上从 0 到最近一侧的距离，因为在单调扫描期间会访问中间值。 

### 为什么它有效

 所有约束都是单个全局变量上的等式$x$。 每个包恰好贡献一个所需的值$x$，而成功意味着至少访问每个此类值一次。 任何有效的移动序列都对应于整数线上的行走，并且重新访问值无助于降低成本，因为移动成本与位移成线性关系。 因此，最优策略始终是覆盖所需点的凸包的最短路径，并扩展到包括起点 0。这保证了没有替代排序可以减少总移动。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    d = [a[i] - (i + 1) for i in range(n)]
    d.sort()
    
    # include starting point 0
    left = min(d[0], 0)
    right = max(d[-1], 0)
    
    # we must cover full interval
    print(right - left)

if __name__ == "__main__":
    solve()
```核心变换是计算位移数组$d_i$。 这是对图结构进行编码的唯一位置。 之后的一切将问题简化为一维区间问题。 

排序仅用于识别极值，但实际上仅用于最终公式的最小和最大事项。 为了清晰和安全起见，该实现使用排序，但可以简化为线性扫描。 

包含 0 至关重要，因为传送带启动时没有移位。 如果不考虑它，计算出的间隔将错误地假设我们在所需区域内开始。

 | 2 | 3 | 1 |
 | 3 | 2 | -1 |

 已排序$d$:$[-3, -1, 1, 1]$我们从 0 开始。覆盖所有点和 0 的区间是从 -3 到 1。 

| 步骤| 左| 对| 行动|
 | ---| ---| ---| ---|
 | 初始化| 0 | 0 | 开始 |
 | 点后| -3 | 1 | 包括所有约束|

这表明，即使值是聚集的，起点也会迫使区间扩大。

这证实了答案仅取决于极端情况，而不取决于多重性。

 ## 边缘情况

 对于最小的情况$n=2$，说输入：```
2
2 1
```我们得到$d = [1, -1]$。 所需的区间范围从 -1 到 1（包括 0），因此答案为 2。该算法正确地包括 0 作为起始约束，防止我们仅使用 max-min 时出现低估情况$d$。 

对于所有平移均为正的情况，例如：```
3
2 3 1
```由于最后一个元素，我们仍然计算负位移，因此间隔自然延伸到 0。这确保始终正确考虑起始位置，并且行走不会假设我们在目标区域内开始。
