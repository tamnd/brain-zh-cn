 ## 解决方案
 ## 问题理解

 我们有一排门，每扇门都涂有一种颜色，每种颜色恰好出现两次。 搬运工从第一扇门前开始，想逃到最后一扇门后的右侧。 移动是通过穿过门来进行的，但奇怪的是，每次他穿过某种颜色的门时，他都会立即传送到另一扇相同颜色的门，根据进入的方向位于相邻门之间。 

因此，每个动作不是简单的从左到右行走，而是“走进一扇门”，然后根据该颜色的配对出现重新定位到序列中的其他位置，然后继续。 

任务是找到越过最后一扇门所需的最少门道数量。 

输入大小高达 200000，这会立即排除所有可能状态的二次模拟。 一个关于位置和“进入侧”的朴素 BFS 将有多达 2n 个状态，并且转换在本地看起来很简单，但仍然会产生许多冗余的探索。 即使是 O(n²) 间隔推理也太慢了。 

一个微妙的困难是传送取决于方向。 从门的左侧或右侧进入会将您带到相对于配对门的不同位置，这会改变下一个有效的移动。 一个天真的错误是忽略方向并将每个颜色对视为索引之间的简单跳跃。 例如，对于两个出现都接近末端的配置，忽略方向会产生不正确的最短路径，因为它合并了实际上导致不同延续的两个不同状态。 

另一个失败案例是假设向右单调前进。 传送后有可能向左走，所以贪婪的“一直向前”策略会立即失效。 

## 方法

 关键的挑战是，每扇门的行为就像一条线上位置的两个“侧面”之间的连接器，并且每种颜色都会创建一个取决于方向的可逆链接。 如果我们天真地扩展问题，我们可以想到将每个索引拆分为两种状态：从左侧进入或从右侧进入。 在每个状态中，我们模拟穿过一扇门，跳到它的那对门，然后继续。 

这给出了一个包含大约 2n 个节点和每个节点 O(1) 次转换的图，因此 BFS 是正确的并且会找到最短路径。 然而，仔细地编写转换是很棘手的，更重要的是，我们可以简化结构。 

关键的观察是，每个颜色对在最佳遍历中恰好产生一个“有用的捷径”，并且方向规则确保任何最佳路径都可以解释为穿过由这些成对连接形成的一系列间隔。 我们可以使用类似扫描的动态编程以及跟踪到达每种颜色第一次出现的最佳方式的结构，按顺序处理门并保持到位置的最佳已知距离，而不是显式地建模左/右状态。 

关键的简化是，该过程的行为就像图中的最短路径，其中节点位于门之间的位置，每种颜色在其两次出现之间创建边缘，可以在两个方向上以成本 1 松弛。因为该图本质上是一条带有额外双向“传送边缘”的线，所以位置上的 BFS 就足够了，但我们可以通过认识到转换始终落在索引之间的边界来进一步压缩状态。 

因此，我们在代表“i 和 i+1 之间”的位置上运行 BFS，当我们在位置 i 穿过颜色为 c 的门时，我们立即跳转到配对的出现并将生成的边界状态排队。 每个边界最多被访问一次，给出线性复杂度。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力状态 BFS over（位置，方向）| O(n) 状态，O(n) 转换 | O(n) | 已接受但实施繁重 |
 | 具有颜色跳跃的边界 BFS | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将门之间的每个位置建模为一个节点。 有 n+1 个这样的位置：第一扇门之前、任意两扇相邻的门之间以及最后一扇门之后。 我们对这些头寸运行 BFS。 

1. 将每种颜色映射到数组中它的两个索引。 这是必要的，因为每次穿过门的动作都会触发跳转到相同颜色的另一个事件。 
2. 隐式而不是显式地建立邻接关系。 从边界位置 i 开始，我们可以尝试向右穿过门 i+1 或向左穿过门 i，具体取决于我们是否在边界内。 每一次这样的举动都会花费一次转变。 
3. 当我们穿过索引为 j 的门时，我们识别出它的颜色 c 和它的配对索引 k。 我们计算传送后的新边界位置：如果我们从 j 的左侧进入，我们落在 k 和 k+1 之间； 如果从右侧开始，我们落在 k-1 和 k 之间。 这是直接从问题的几何描述得出的。 
4. 从边界 0（第一扇门的左侧）开始使用 BFS。 初始化距离0。 
5. 对于每个弹出的边界状态，尝试两种可能的门交叉（左和右）。 如果穿越有效，则在传送后计算结果边界，如果没有访问过则将其推入。 
6. 当我们到达边界n（最后一扇门的右侧）时停止，这代表逃生。 

正确性取决于将每个交叉点视为未加权图中的单位成本边。 

### 为什么它有效

 关键的不变量是每个边界状态代表位于两扇门之间的独特配置，并且从一个边界到另一个边界的每个有效移动恰好对应于一个门穿越事件。 因为所有的转换都具有相同的成本，并且我们按照 BFS 顺序进行探索，所以我们第一次到达边界时保证是到达那里所需的最少交叉次数。 传送不会带来额外的费用； 它只重新连接下一个边界状态，因此它在此隐式图中保留了最短路径结构。 

## Python 解决方案```python
import sys
from collections import deque

input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))

    pos = {}
    for i, c in enumerate(a):
        if c not in pos:
            pos[c] = [i]
        else:
            pos[c].append(i)

    dist = [-1] * (n + 1)
    q = deque()

    dist[0] = 0
    q.append(0)

    def relax(u, v):
        if v < 0 or v > n:
            return
        if dist[v] == -1:
            dist[v] = dist[u] + 1
            q.append(v)

    while q:
        i = q.popleft()

        if i < n:
            c = a[i]
            x, y = pos[c]

            # crossing from left boundary i to right through door i
            # lands around paired occurrence depending on direction
            if i == x:
                j = y
            else:
                j = x

            # entering j from left puts us between j and j+1 -> boundary j
            relax(i, j)

            # entering j from right puts us between j-1 and j -> boundary j-1
            relax(i, j - 1)

        if i > 0:
            c = a[i - 1]
            x, y = pos[c]

            if i - 1 == x:
                j = y
            else:
                j = x

            relax(i, j)
            relax(i, j - 1)

    print(dist[n])

if __name__ == "__main__":
    solve()
```该代码构建了从每种颜色到其两个位置的映射，从而允许恒定时间查找传送目的地。 BFS 状态是边界索引，我们维持距离`dist`。 

功能`relax`确保我们只推送看不见的状态并正确增加距离。 每个边界都考虑穿过紧邻其右侧的门和紧邻其左侧的门，对应于搬运工可以从任一侧进入门的所有可能方式。 

边界正确性被仔细处理：我们只允许在`[0, n]`，和边界`0`和`n`代表迷宫外面每一侧。

 ## 边缘情况

 一个重要的边缘情况是两种颜色的出现相邻。 例如：

 输入：```
2
1 1
```从边界 0 开始，穿过第一个门会立即传送到第二个门并允许退出。 该算法通过位置图对称地处理这两种情况，BFS 一步达到边界 2。 

另一个边缘情况是隐形传送将搬运工送回较早的边界。 在这种情况下，BFS 队列已经确保我们不会重新访问已处理的状态。 即使颜色跳跃导致索引变小，被访问的数组也能防止无限循环并保持正确性，因为 BFS 保证第一次访问是最优的。
