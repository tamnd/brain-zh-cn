 ## 解决方案
 ## 问题理解

 我们得到了一家有几个房间的酒店，每个房间在理想的计划中都有所需数量的椅子。 事实上，酒店共有$N$椅子必须分布在$K$房间。 这些限制不仅仅是“符合目标”：每个房间必须至少配备一把椅子，并且所有房间的短缺（定义为每个房间与其理想要求相比缺少多少椅子）必须相同。 

所以如果一个房间最初期望$a_i$椅子和我们放置$x_i$椅子，然后$x_i \ge 1$和所有值$a_i - x_i$必须是相同的常数$d$，在每个房间共享。 这意味着每个房间都被均匀地填充不足$d$，同时仍然尊重每个房间至少有一把椅子。 

目标是选择有效的制服短缺$d$以及满足该要求的椅子的分配，同时最大化未使用的椅子数量。 同样，我们希望尽量减少必须放置的椅子数量。 

约束条件很大$N$但中等$K$。 自从$K$可以达到$10^5$，任何解都必须是线性或接近线性的$K$。 如果成本高昂，那么尝试所有可能的短缺或为每个候选人重复重新分配椅子的解决方案就会太慢$O(K^2)$或者更糟。 

当某些房间的容量非常小时，就会出现关键的隐藏边缘情况。 如果一个房间有$a_i = 1$，那么唯一有效的分配力$x_i = 1$，因此该房间的短缺量固定为零，从而限制了全局短缺量。 任何假设所有房间都可以共享大量制服赤字的幼稚尝试都会在这里立即失败。 

## 方法

 思考问题的一个直接方法就是猜测共同的不足$d$。 如果我们修复$d$，那么每个房间必须收到$x_i = a_i - d$椅子。 这立即提出了两个可行性条件：首先，$x_i \ge 1$， 所以$d \le a_i - 1$对于每个房间，意味着$d \le \min(a_i) - 1$。 其次，使用的椅子总数必须准确$\sum (a_i - d)$，等于$\sum a_i - Kd$，并且这不得超过$N$。 

所以对于一个固定的$d$，可行性简化为检查是否$Kd \ge \sum a_i - N$。 结构变得单调：增加$d$减少了使用的椅子数量，更容易保持在预算之内，但也受到最小房间尺寸的限制。 

蛮力方法会尝试所有可能的值$d$从$0$最多$\min(a_i) - 1$，计算每次所需的椅子总数，并跟踪最佳可行配置。 每张支票费用$O(K)$，并且最多可以有$1000$的不同值$d$，大致导致$10^8$最坏情况下的操作。 这是临界点，但没有必要。 

关键的观察是可行性仅取决于线性表达式$d$。 而不是迭代$d$，我们可以计算最大有效值$d$直接使用算术约束：$$Kd \le \sum a_i - N$$和$$d \le \min(a_i) - 1.$$所以最优的$d$是这两个上限的最小值。 

一次$d$是固定的，实际放置的椅子数量是强制的，所以保存的椅子数量就是$N - \sum (a_i - d)$，或等价地$N - (\sum a_i - Kd)$。 

这将整个问题简化为计算数组上的一些聚合。

 | --- | --- | --- | --- |
 | 暴力破解结束$d$|$O(K \cdot \min a_i)$|$O(1)$| 太慢了 |
 | 最优算术 |$O(K)$|$O(1)$| 已接受 |

 ## 算法演练

 ### 1. 计算全局统计数据

 我们计算所有房间容量的总和以及其中的最小容量。 这两个值完全控制任何制服短缺的可能性。 

总和决定了理想配置中存在多少把椅子，而最小值决定了在不违反“每个房间至少一把椅子”约束的情况下允许的最大统一赤字。 

### 2. 从预算中得出最大可能的短缺

 我们将总使用约束重写为：$$\sum a_i - Kd \le N$$这给出：$$d \ge \frac{\sum a_i - N}{K}$$然而，由于我们想要可行性和最大程度的节省，所以我们计算最大的$d$这并不违反这个不等式：$$d \le \frac{\sum a_i - N}{K}$$我们采用该值的下限，因为$d$必须是整数。 

### 3. 应用每个房间的可行性约束

 每个房间仍必须至少配备一把椅子，因此：$$a_i - d \ge 1 \Rightarrow d \le a_i - 1$$最严格的约束来自最小的房间，因此：$$d \le \min(a_i) - 1$$### 4. 组合约束

 有效的$d$是预算衍生界限和结构界限的最小值。 我们还确保$d \ge 0$。 

### 5. 计算最终答案

 曾经$d$是固定的，总共放置的椅子是$\sum a_i - Kd$。 保存的椅子是：$$N - (\sum a_i - Kd)$$### 为什么它有效

 关键的不变量是任何有效的配置都对应于选择一个整数$d$这样所有的房间分配都是准确的$a_i - d$。 没有独立改变房间的自由，因为条件迫使相同的缺陷。 这减少了搜索空间$K$对单个标量参数的维赋值。 每个可行的解决方案都由一些唯一的表示$d$，以及每个有效的$d$产生有效的配置，因此优化$d$相当于对所有作业进行优化。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    a = list(map(int, input().split()))
    
    total = sum(a)
    mn = min(a)
    
    # upper bound from "at least one chair per room"
    d1 = mn - 1
    
    # upper bound from total chairs constraint
    d2 = (total - n) // k
    
    d = min(d1, d2)
    if d < 0:
        d = 0
    
    used = total - k * d
    print(n - used)

if __name__ == "__main__":
    solve()
```该实现首先读取输入并在一次遍历房间的过程中计算聚合值。 价值`d1`强制没有房间变空，因为减去超过`a_i - 1`会违反最小约束。 价值`d2`强制分配的椅子总数不超过以统一赤字表示的可用供应量。 

选择最佳可行方案后`d`，我们计算实际使用了多少张椅子。 最终的答案是可用椅子和已用椅子之间的区别。 

一个微妙的点是夹紧`d`为零。 如果没有这个，情况下`total < n`即使没有短缺可以有意义地应用，也会产生负赤字。

 ## 边缘情况

 当只有一个房间时，条件会强制将所有椅子放置在该房间中，因此无法保存，除非$N$超出其容量限制结构允许的范围。 该算法计算$d1 = a_1 - 1$和$d2 = (a_1 - N)$，并正确夹紧到可行值，从而实现一致的放置。 

当所有房间容量相同时，最小约束成为主导因素，任何增加统一赤字的尝试都会立即受到限制。 该算法简化为检查总主席预算是否高于或低于理想总和，以及计算出的$d$正确地反映了这种平衡。 

当一个房间的容量非常小（例如 1）时，它会强制$d = 0$，因为不允许有赤字。 该算法通过以下方式捕获这一点：$d1 = 0$，确保所有其他计算遵守最严格的结构约束，无需特殊外壳。
