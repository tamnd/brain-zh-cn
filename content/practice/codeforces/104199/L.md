 ## 解决方案
 ## 问题理解

 我们在活动大厅中保持着充满活力的宾客队伍。 每位客人都有一个唯一的数字标识符。 该生产线支持三种类型的操作，不断重塑其订单。 

一位客人到达时可以向另一位客人声明“朋友推荐”。 如果该朋友当前正在排队，则到达的客人必须直接插在他们后面。 如果朋友缺席，则新客人会添加到行尾。 随着时间的推移，访客可能会出现多次，因此该结构必须支持重新插入。 

客人也可以随时离开队列，并且离开不得破坏其他人的相对秩序。 

最后，我们被要求重复报告当前行的前面，或者报告该行是空的。 

关键的困难在于插入不仅仅在末尾，而且可能在任意现有元素之后，而删除和查询必须在最多 200,000 次操作下保持高效。 任何扫描每个操作行的方法都将立即失败，因为每个查询的完整扫描在最坏的情况下会导致二次行为。 

第二个微妙之处是，“如果存在的话插入朋友后面”需要快速检测一个人当前是否在结构内，如果是，则直接访问他们的位置。 

当相对于经常移动的人重复插入时，会出现幼稚故障模式。 例如，如果我们每次总是线性搜索好友，那么像移动前端元素后面的许多链式插入这样的情况就会退化为重复的完整扫描，即使每个单独的操作看起来都很便宜，也会产生隐藏的二次爆炸。 

另一个边缘情况是删除。 如果我们从基于数组的队列中物理删除元素，则每次删除后移动元素都会变得每次操作都是线性的，当删除频繁时，再次导致二次行为。 

因此，核心要求是支持以下结构：

 快速检查客人的存在情况，

 快速插入到已知节点旁边，

 快速删除已知节点，

 并快速进入当前战线。 

## 方法

 直接暴力模拟将队列存储为 Python 列表。 对于每个`in x y`，我们扫描列表来查找`y`，然后插入`x`就在它之后。 如果`y`没有找到，我们追加。 为了`out x`，我们再次扫描定位`x`并将其删除。 为了`check`，我们返回第一个元素。 

这在逻辑上是正确的，因为它确实模拟了规则。 然而，每个操作可能需要扫描最多 O(n) 个元素。 如果执行多达 200,000 次操作，总工作量将变为 O(nq)，在最坏的情况下可能会超过 10^10 次操作，远远超出了可行的限制。 

关键的观察是，行顺序始终是具有局部修改的链接结构。 每个客人的位置仅通过在已知节点旁边插入或删除来改变。 这建议用指针操作替换数组移位。 

我们可以将队列建模为双向链表。 每个客人对应一个节点。 我们维护一个从 guest id 到节点指针的字典，因此我们可以在 O(1) 内访问任何 guest。 插入变成了指针重新连接：如果友元存在，我们就在它们后面拼接新节点； 否则我们追加在尾部。 通过使用其邻居取消链接节点，删除也是 O(1)。 

这将所有昂贵的线性扫描转换为恒定时间的字典查找和指针更新。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解（列表+扫描）| O(q·n) | O(q·n) | O(n) | 太慢了|
 | 最优（哈希图+链表）| O(q) | O(n) | 已接受 |

 ## 算法演练

 我们维护一个具有显式头指针和尾指针的双向链表，以及从访客 id 到节点的哈希映射。 

1. 创建节点结构存储`value`,`prev`， 和`next`。 这代表排队中的一位客人。 这样可以实现恒定时间的移除和插入，而无需移动元件。 
2. 维护`head`和`tail`当前行的前部和后部的指针。 这些给了 O(1) 访问第一个元素的时间`check`。 
3.维护字典`pos[x]`将每个访客 ID 映射到其节点。 这确保我们永远不会线性搜索列表。 
4.对于一个`in x y`操作时，首先创建一个新节点`x`。 然后检查是否`y`存在于`pos`。 如果存在，我们立即插入新节点`y`的节点通过更新四个指针：新节点的链接和周围的邻居。 如果不存在，我们使用尾指针将节点附加到末尾。 这保留了所需的排序规则。 
5. 对于一个`out x`操作，从中检索节点`pos[x]`并将其从列表中删除。 我们重新连接它的前一个和下一个节点。 如果是头或尾，我们更新相应的指针。 最后，删除`x`从字典中。 
6. 对于`check`，我们输出`head.value`如果列表非空，否则输出-1。 

为什么有效：链表始终代表客人当前的确切顺序。 字典保证我们可以在恒定的时间内找到任何引用的客人。 每次插入仅通过修改局部邻接来保留相对顺序，并且每次删除都保留剩余节点之间的顺序。 由于所有操作仅修改或查询恒定的局部结构，因此永远不会违反全局排序不变量。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class Node:
    __slots__ = ("val", "prev", "next")
    def __init__(self, val):
        self.val = val
        self.prev = None
        self.next = None

def main():
    q = int(input().strip())

    pos = {}
    head = None
    tail = None

    out = []

    for _ in range(q):
        cmd = input().split()

        if cmd[0] == "in":
            x = int(cmd[1])
            y = int(cmd[2])

            node = Node(x)
            pos[x] = node

            if y in pos:
                ny = pos[y]

                node.prev = ny
                node.next = ny.next

                if ny.next:
                    ny.next.prev = node
                ny.next = node

                if tail == ny:
                    tail = node
            else:
                node.prev = tail
                node.next = None

                if tail:
                    tail.next = node
                tail = node

                if head is None:
                    head = node

        elif cmd[0] == "out":
            x = int(cmd[1])
            node = pos.pop(x)

            if node.prev:
                node.prev.next = node.next
            else:
                head = node.next

            if node.next:
                node.next.prev = node.prev
            else:
                tail = node.prev

        else:
            if head is None:
                out.append("-1")
            else:
                out.append(str(head.val))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    main()
```该实现依赖于手动双向链表，因为 Python 的内置列表不支持 O(1) 中间插入或删除。 字典`pos`是避免遍历的关键组件。 

插入逻辑仔细区分好友是否存在。 当在一个节点之后插入时，我们必须处理友元是尾部的情况，因为`ny.next`是 None，我们必须更新`tail`。 类似地，当插入到空列表时，头和尾都必须初始化。 

删除必须对称地处理边界情况。 拆下头部需要更新`head`，并且删除尾部需要更新`tail`。 字典删除可确保过时的引用不再出现。

 ## 边缘情况

 一种重要的边缘情况是当引用的朋友是当前尾部时插入。 在这种情况下，插入逻辑必须更新尾指针。 例如：

 输入：```
3
in 1 1
in 2 1
in 3 2
```处理后列表变为`1 2 3`。 插入时`3`后`2`, 节点`2`当时是尾部，所以更新失败`tail`会使结构不一致并破坏以后的插入或检查。 该算法明确地检查这一点并分配`tail = node`。 

另一个边缘情况是删除 head 元素。 如果第一个元素离开，头指针必须向前移动。 为了：```
3
in 1 1
in 2 1
out 1
```该列表变为`2`。 删除逻辑检测到`node.prev is None`和更新`head = node.next`。 如果没有这个调整，`check`仍会返回已删除的元素。 

最后的边缘情况是重复重新插入相同的 id。 由于每个`in x y`创建一个新节点并更新`pos[x]`，任何先前出现的情况都必须已被删除。 该字典确保仅跟踪当前节点，因此无法访问过时的指针。
