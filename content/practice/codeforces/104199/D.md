 ## 问题理解

 我们有一排建筑物，每座建筑物都有固定的高度。 您从建筑物外的地面开始，您的目标是检索位于特定建筑物屋顶上的物品，索引为`m`。 运动受到两个独立机制的限制：使用便携式梯子的垂直运动，以及相邻屋顶之间的水平运动。 

要从地面到达任何建筑物的屋顶，您携带的梯子必须至少与该建筑物一样高。 一旦到达屋顶，您可以使用固定连接移动到邻近的屋顶，但有一个限制：携带重型设备会改变您的移动方式。 特别是下降时，有效移动取决于梯子限制和高度差，即使有设备，跨越等高相邻建筑物的移动也不受限制。 

关键的隐藏结构是，到达目标屋顶并随后返回地面只能通过相邻建筑物的顺序，但每次从地面进入都取决于您最初必须爬上的最大建筑物高度，而每次建筑物之间的转移取决于在“携带设备下降”限制下高度过渡是否可行。 

我们被要求找到最小梯子长度，以便存在有效的移动序列，允许：

 首先，到达目标屋顶，然后取回设备，最后在所有移动限制下安全返回地面。 

输入大小高达 100,000 座建筑物，这排除了对所有可能路径或成对过渡的任何二次探索。 任何试图模拟建筑物之间所有可能路线的解决方案都会太慢。 因此，我们正在寻找一种解决方案，将问题简化为高度数组的结构属性的推理，最好是在线性或近线性时间内。 

当人们认为只有目标建筑高度重要时，就会出现一种常见的失败情况。 例如，如果我们只考虑建筑物的高度`m`，我们可能认为答案很简单`h[m]`。 这是错误的，因为您可能会从较高的建筑物开始，然后向侧面移动并使用等高过渡来到达目标，从而有效地绕过直接爬上建筑物的需要`m`。 

另一个微妙的错误是假设所有相邻建筑物的单调移动总是可能的。 关于可自由穿越的相同高度的限制引入了允许间接布线的平台，这显着影响了所需的最小梯子高度。 

## 方法

 暴力解释会尝试所有可能的梯子长度`L`并检查是否可以完成整个操作。 对于固定的`L`，我们将标记所有建筑物的高度最多`L`从地面可到达。 从每个这样的起始建筑物开始，我们将模拟沿着相邻建筑物的移动，遵守携带设备下降和平等遍历的规则，并检查是否可以到达目标以及是否可以返回。 

这种方法在概念上是正确的，因为它直接编码问题约束。 然而，对于每个候选梯子长度，我们可能需要完全遍历建筑图，这会花费`O(n)`。 自从`L`范围可达`10^9`，一个简单的二分搜索仍然需要`O(n log H)`，这是临界点，但可能会过去； 然而，真正的问题是，简单的可行性检查通常最终会重复地重新探索阵列的大部分，而不重用结构。 

关键的观察结果是，移动规则有效地将结构简化为由相邻建筑物之间的边缘形成的连接组件，这些组件在给定高度阈值下是“安全的”。 等高充当零成本桥梁，仅在承载设备时才限制从高处到低处的过渡。 这意味着实际可行性不取决于单独的路径，而是取决于目标是否位于从最大高度不超过梯子长度的某个起始位置可到达的组件中。 

一旦我们以这种方式重新解释问题，我们就不再需要模拟运动。 相反，我们只需要了解组件如何在阈值下形成以及目标组件是否连接到至少一个有效的起点。 

这导致了贪婪的结构解决方案：随着允许的梯子高度的增加，更多的建筑物可用作入口点，并且相邻组件会合并。 我们只需要跟踪通过有效的高度转换可以将目标周围的连接扩展多远。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟| O(n² log H) | O(n² log H) | O(n) | 太慢了|
 | 阈值+扩展| O(n) | O(n) | 已接受 |

 ## 算法演练

 我们将这一过程重新解释为从目标建筑向外探索，通过有效的过渡进行扩展，同时跟踪每次扩展所需的最小梯子高度。 

1.从目标索引开始`m`并将其视为初始可达区域。 

如果我们直接站在上面，梯子至少必须与这座建筑物的高度相匹配。 
2. 从目标向左和向右扩展，只要相邻建筑物允许在与承载设备相关的平等或非增加约束下移动。 

此扩展包捕获了所有可以穿越的建筑物，而无需比必要的更高的梯子。 
3. 扩展时，保持当前可到达段遇到的最大高度。 

该最大值表示最初访问该连接区域的任何部分所需的最小梯子高度。 
4. 如果我们遇到一个边界，其中的运动被严格增加的高度障碍所阻碍，并且根据规则无法跨越，我们就会停止向该方向扩展。 

这是因为跨越这样的边界需要一个至少与较高一侧一样高的梯子，这已经在最大跟踪中得到考虑。 
5. 继续下去，直到两个方向都不再可能进一步扩展为止。 

生成的片段表示与在约束下达到和离开目标相关的完全连接的组件。 
6. 答案是最终可到达段内的最大高度。 

这确保了初始访问和所有所需的转换都是可行的。 

### 为什么它有效

 该过程在目标周围构建最大连续区域，其中在移动规则下所有转换都是可行的。 任何延伸到这个区域之外的尝试都需要跨越一个高度障碍，无论如何都会迫使更大的梯子，这意味着这样的延伸不会减少答案。 保持的不变性是，该段内的每个建筑物都可以使用长度等于迄今为止看到的最大高度的梯子到达，并且在不增加该最大值的情况下不能包括任何外部建筑物。 因此，最终的最大值是支持整个所需遍历的最小可能梯子长度。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    h = list(map(int, input().split()))
    m -= 1

    left = m
    right = m
    ans = h[m]

    while True:
        changed = False

        while left > 0 and h[left - 1] <= h[left]:
            left -= 1
            ans = max(ans, h[left])
            changed = True

        while right < n - 1 and h[right + 1] <= h[right]:
            right += 1
            ans = max(ans, h[right])
            changed = True

        if not changed:
            break

    print(ans)

if __name__ == "__main__":
    solve()
```该实现维护一个窗口`[left, right]`表示从目标开始的当前可达段。 扩展规则直接对约束进行编码，即如果不需要增加超出当前边界的有效梯子使用量，则只能遍历到相邻建筑物。 每次扩展时，我们都会更新看到的最大高度，因为这决定了最初访问该段所需的最小梯子长度。 

一个微妙的点是，两个方向都是独立但对称地扩展的，确保我们不会错过由等高过渡形成的可到达的平台。 循环一直持续到无法扩展为止，从而保证可达区域的最大闭合。

 ## 边缘情况

 对于单个建筑物，算法会立即返回其高度，因为不可能进行扩展。 初始段已经是最大的，答案是唯一存在的高度。 

对于所有高度都相等的平坦序列，左扩展和右扩展在一次扫描中在整个数组上进行。 最大值保持不变，证实梯子长度仅取决于统一的高度，而不取决于位置。 

对于严格先增后减的数组，从峰值或接近峰值开始可确保扩展受到最高值的约束。 即使遍历继续进入较低区域，该算法也会正确地将峰值传播到最终答案中。 

对于距离目标较远存在高层建筑的情况，只有单调条件允许遍历时，扩展最终才会到达目标，并且当遍历时，最大值也会相应更新。 这确保了不会错过远处的高约束。
