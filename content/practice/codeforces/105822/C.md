 ## 问题理解

 我们给定一个长度为的二进制字符串$N$，其中每个字符描述了如何$i$-th 项必须与从一组固定编号元素中提取的值配对。 每个位置都对我们如何从池中选择未使用的数字对施加了结构性约束。$2N$有序元素。 目标是为每个位置分配一对数字，以便所有数字仅使用一次，并且分配遵循字符串编码的模式。 

关键的隐藏要求是构造必须在所有位置上保持全局一致，而不仅仅是每个字符局部有效。 这将任务转变为构建一个分区$2N$将元素排序为$N$方向约束下的对。 

这些约束意味着任何解决方案都必须是线性或近线性的。 自从$N$可能很大，任何尝试所有配对或回溯分配的方法都是立即不可行的，因为它会随着阶乘增长$2N$。 这将我们推向一种贪婪的结构，其中每个决策都做出一次，仅使用诸如有序多重集或指针之类的簿记结构。 

当字符串均匀或严重倾斜时，会出现微妙的边缘情况。 例如，如果所有角色都强制使用相同的配对类型，则始终消耗相同奇偶校验池的天真的贪婪策略可能会因过早耗尽一个类别而破坏后续步骤。 当局部最优配对增加了即时灵活性但违反了全局排序要求，导致后续步骤陷入死胡同时，就会出现另一种故障模式。 

## 方法

 强力解释将尝试独立地为每个位置分配对，同时检查与先前分配的一致性。 从概念上讲，这意味着维护一组未使用的数字，并为每个索引递归选择两个元素，验证所得的部分结构仍然可以完成。 每一步都有分支$\Theta(N^2)$可能的对，这爆炸成$(2N)!/(2^N N!)$最坏情况下的可能性，远远超出任何可行的极限。 

暴力破解失败的原因是它没有利用数字的严格排序结构。 所有元素都不是对称的，因为较小的指数与未来约束的相互作用不同。 关键的观察结果是，该解决方案并不依赖于任意对选择，而是依赖于维持奇数指数和偶数指数之间的受控平衡，同时确保所选值的单调增长。 

其见解是将可用数字分成两个有序池，然后始终从最小的未使用元素中消耗。 我们不是任意决定配对，而是根据当前角色和池之间的剩余余额强制执行确定性规则。 这将问题转化为贪婪扫描，其中状态由剩余的奇数和偶数索引完全捕获。

 ## 算法演练

 我们隐式维护两个有序集合：一个包含未使用的奇数索引数字，另一个包含未使用的偶数索引数字。 在每一步中，我们都会根据当前字符分配一对。 

1. 初始化表示可用奇数和偶数的两个序列。 这些是自然排序的，因为我们总是采用最小的剩余元素。 这种排序至关重要，因为正确性依赖于单调消耗。 
2. 从左到右迭代字符串。 在每个位置，我们根据字符是“O”还是“E”来决定如何形成一对。 该决定纯粹是局部的，但受到平衡使用的全局不变量的约束。 
3. 如果当前字符是“O”，我们总是取一个奇数和一个偶数，从每个池中选择最小的可用数。 这会强制执行交叉配对，从而在整个结构中保持奇偶校验平衡。 
4. 如果当前字符是“E”，则比较剩余的奇数和偶数个数。 如果相等，则取两个奇数； 否则，我们取两个偶数。 这一有条件的选择确保我们避免过早耗尽一个池，同时保留未来步骤的可行性。 
5. 记录每个选择的对并将这些数字标记为已使用，从而缩小相应的池。 
6. 继续，直到处理完所有位置。 最后，所有数字必须仅使用一次，并且每个位置都有一个有效的对。 

正确性来自单调结构：我们总是使用最小的可用元素，并且只有当对称或不平衡需要时，我们才会在“E”情况下偏离。 这保证了后来的决策永远不会依赖于任意的早期选择。 

### 为什么它有效

 该构造保持了一个隐藏的不变量：在处理字符串的任何前缀后，剩余未使用的数字始终可以配对以满足后缀约束。 贪婪的选择确保我们永远不会隔离其中一个奇偶校验类变得不可用的配置。 “E”决策充当保持可行性的受控平衡操作，而“O”则强制执行稳定的交叉消费。 由于所有选择在值方面都是单调的，因此当存在较小的有效元素时，未来的步骤不会被先前选择的大元素阻塞。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    s = input().strip()

    odds = list(range(1, 2*n + 1, 2))
    evens = list(range(2, 2*n + 1, 2))

    oi = 0
    ei = 0

    res = []

    for c in s:
        if c == 'O':
            a = odds[oi]
            b = evens[ei]
            oi += 1
            ei += 1
            res.append((a, b))
        else:
            if len(odds) - oi == len(evens) - ei:
                a = odds[oi]
                b = odds[oi + 1]
                oi += 2
            else:
                a = evens[ei]
                b = evens[ei + 1]
                ei += 2
            res.append((a, b))

    out = []
    for a, b in res:
        out.append(f"{a} {b}")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现使用两个指针索引，而不是从列表中删除元素，这使每步的操作保持 O(1) 。 奇数和偶数数组按排序顺序预先计算，确保始终可以通过当前指针访问“最小的未使用”。 唯一微妙的部分是保持正确的指针增量：“O”使两个指针前进，而“E”则根据平衡情况将一个或另一个指针前进两个。

 ## 边缘情况

 一个关键的边缘情况是所有字符都是“E”。 在这种情况下，算法必须确保奇数-奇数和偶数-偶数配对之间的决策不会过早耗尽一个池。 为了$N = 3$和`EEE`，执行过程如下：首先两个池相等，所以我们取`(1,3)`，使赔率降低。 下一个不平衡有利于均匀配对，所以`(2,4)`被采取。 最后将剩下的数字配对`(5,6)`。 贪婪规则自动交替消耗以保持可行性。 

另一个边缘情况是“O”的长前缀，后跟“E”。 例如，`OOOOEEE`首先强制严格交叉配对，均匀耗尽两个池。 当切换到“E”时，池是平衡的，因此算法安全地切换到奇奇配对，确保连续性。 两个池在“O”段期间同步收缩的不变性保证了在转换之前不会引入不可逆的不平衡。
