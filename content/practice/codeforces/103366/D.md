 ## 解决方案
 ## 问题理解

 我们得到了一组整数，并且可以将它们重新排列成任意顺序。 重排后，我们必须保证至少有一个值的存在，称之为$x$，使得所有出现的$x$相距足够远：任何两个位置包含$x$必须至少相差$d$。 其他值没有任何限制。 

我们的任务不仅是找到一个有效的重排，而且在所有有效的重排中，我们必须输出字典顺序最小的数组。 如果没有任何布置可以满足任何选择的间距要求$x$，答案是$-1$。 

约束条件$n \le 10^6$跨所有测试用例迫使我们每个测试用例的时间大致呈线性或接近线性。 任何尝试测试所有排列或重复模拟每个候选值的放置的方法都会立即失败，因为即使$O(n \log n)$当对许多测试进行求和时，每个测试用例可能会变得紧张，并且任何二次的都是不可能的。 

一个微妙的问题是特殊值$x$不是提前固定的。 我们可以自由选择哪个值成为“间隔”值。 这是关键的结构自由，也是最不正确的贪婪想法的根源：选择错误$x$可以使可解的实例显得不可能，相反，选择错误的构造顺序可能会破坏词典编排的最优性。 

一个常见的失败案例来自于在贪婪布局期间忽略可行性约束。 例如，如果我们过于激进地贪婪地放置小值而不跟踪所选值是否$x$仍然可以在间距限制下完成，我们最终可能会处于以下状态：$x$无法再放置。 

另一个失败来自于选择$x$纯粹通过频率。 频率本身并不能决定词典编排的可行性； 它仅确定间隔模式原则上是否可行。 

## 方法

 蛮力的想法很简单：尝试每个不同的值作为候选值$x$，并且对于每一个，尝试构建数组的有效排列，同时对该值强制执行间距约束，然后采用字典顺序最小的有效结果。 

即使我们修复$x$，构建有效的排列需要回溯或检查每个位置的未来可行性。 朴素的构造会考虑所有排列甚至所有贪婪放置而不进行修剪，这会在最坏的情况下导致阶乘或指数行为，因为每个位置可能会分支到许多选择。 

关键的观察是，一旦我们确定了候选人$x$，唯一的约束是相同项目的间距约束。 这是一个经典的调度结构：$x$必须放置在位置$p_1 < p_2 < \dots < p_k$这样$p_{i+1} \ge p_i + d$。 这将问题转化为放置$k$相同长度的标记$n$具有最小距离的数组。 

由此，我们可以得出一个可行性条件：如果$x$出现$k$时代，我们需要$p_1 = i$， 然后$p_k = i + (k-1)d \le n$。 所以可行性的充要条件是$x$放置是我们可以分配间隔为$d$。 

一旦理解了可行性，字典极小性就表明了一种贪婪的从左到右的结构：在每个位置，我们放置尽可能小的值，不会破坏完成排列的可能性，同时还确保如果我们选择特殊值$x$在某个位置，我们仍然有足够的剩余空间来将其剩余副本放置在间距限制下。 

我们实际上不需要决定$x$如果我们将其视为“构建期间的跟踪约束”，则提前：我们可以假设选择一个$x$并动态执行可行性。 然而，为了使这一确定性，我们选择一个有效的$x$首先，然后贪婪地构建尊重它的字典顺序最小的数组。

 |---|---|---|

 | 蛮力 |$O(n! \cdot n)$|$O(n)$| 太慢了|

 | 具有可行性跟踪的最优贪婪 |$O(n \log n)$或者$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 我们首先将数组压缩为频率，并考虑可以选择哪些值作为特殊值$x$。 一个值$x$仅当我们可以将其所有出现的间隔至少为$d$，这相当于$k_x$足够小以适应$n$有间距的位置$d$。 

然后我们选择一名可行的候选人$x$。 一种自然的选择是选择最小的此类值，以使词典编排结构更容易，但如果结构正确，任何可行的值都可以工作。 

修复后$x$，我们从左到右构建答案，同时维护每个值的剩余出现次数并跟踪$x$仍然需要放置。 

在每个位置，我们尝试按升序排列值，并初步决定放置该值是否保留剩余的实例$x$可行的。 对于除以下以外的值$x$，不存在结构性约束，因此只要它们存在，它们就总​​是安全的。 

当考虑放置时$x$在位置$i$，如果有$r$剩余的副本$x$在此放置之后，最新可能的最终放置仍必须满足$i + (r-1)\cdot d \le n$。 如果这个条件不成立，我们就无法放置$x$即使它是最小的可用值。 

我们总是选择当前位置通过可行性的最小值，减少其计数，然后继续。 

### 为什么它有效

 该结构保持一个不变式：填充前缀后$1 \dots i$，至少存在一个后缀的完成，使得所有剩余的出现$x$可以根据距离放置$d$。 每个决定只接受保留这种不变性的动作，因此我们永远不会进入死状态。 字典顺序极小性遵循，因为在每个位置，我们选择不破坏可行性的最小值，任何较小的拒绝选择都会使完成变得不可能。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    for _ in range(T):
        n, d = map(int, input().split())
        a = list(map(int, input().split()))

        freq = {}
        for v in a:
            freq[v] = freq.get(v, 0) + 1

        # feasibility limit for spacing
        # k elements need (k-1)*d + 1 positions
        def ok(k):
            return (k - 1) * d + 1 <= n

        candidates = [v for v, c in freq.items() if ok(c)]
        if not candidates:
            print(-1)
            continue

        x = min(candidates)

        remaining = freq.copy()

        # we greedily construct lexicographically smallest array
        res = []

        # remaining count of x
        rx = remaining[x]

        for i in range(1, n + 1):
            for v in sorted(remaining.keys()):
                if remaining[v] == 0:
                    continue

                # try place v
                remaining[v] -= 1

                if v == x:
                    rx_after = rx - 1
                    if rx_after <= 0:
                        res.append(v)
                        rx = rx_after
                        break
                    # check feasibility of future x placements
                    # we placed one x at position i, so remaining rx_after must fit
                    if i + (rx_after - 1) * d <= n:
                        res.append(v)
                        rx = rx_after
                        break
                else:
                    res.append(v)
                    break

                remaining[v] += 1
            else:
                print(-1)
                break
        else:
            print(*res)

if __name__ == "__main__":
    solve()
```该代码首先计算频率并过滤可能用作特殊间隔值的值。 然后，它选择最小的可行候选者，使构造偏向词典编排最优性。 

贪心循环逐个位置构建答案位置。 对于每个位置，它按升序扫描值，暂时减少它们的频率，并检查选择它们是否使配置保持有效。 唯一重要的检查是$x$，我们确保剩余的副本仍然可以按间距放置$d$在剩余的后缀中。 

一个微妙的点是我们更新了剩余的计数$x$确认可行性后才可以。 如果检查失败，我们恢复频率并继续搜索下一个候选者。 

## 工作示例

 考虑输入：```
n = 4, d = 3
a = [1, 2, 1, 2]
```我们有频率：1 出现 2 次，2 出现 2 次。 为了$d = 3$，两者都是可行的，因为$(2-1)\cdot 3 + 1 = 4 \le 4$。 我们选择$x = 1$。 

| 我| 选择 v | 剩余频率 (1,2) | 接收 | 行动|
 | ---| ---| ---| ---| ---|
 | 1 | 1 | (1,2) | 1 | 地点 1 |
 | 2 | 2 | (1,1) | 1 | 地点 2 |
 | 3 | 2 | (1,0)| 1 | 地点 2 |
 | 4 | 1 | (0,0) | (0,0) | 0 | 地点 1 |

 结果是`1 2 2 1`，这在有效排列中按字典顺序是最小的。 

该轨迹显示了算法如何优先考虑小值，同时保留完成约束值放置的能力。 

现在考虑：```
n = 4, d = 4
a = [1, 1, 2, 2]
```两个值都出现两次，但可行性要求$(2-1)\cdot 4 + 1 = 5 > 4$，所以没有任何值可以作为$x$。 算法正确输出`-1`。 

这表明在任何贪婪的建设开始之前，可行性过滤是必不可少的。

 ## 边缘情况

 当唯一可行的候选者出现时，就会出现一种重要的边缘情况$x$存在但不是数组中的最小值。 在这种情况下，天真的字典贪婪总是试图将最小的数字放在前面，可能会意外地破坏可行性$x$。 该算法通过仅检查所选选项的可行性来避免这种情况$x$，不适用于所有值。 

另一个边缘情况是当$d = 1$。 该约束变得空洞，因为每对相同值自动满足距离至少为 1，因此任何排列都是有效的。 贪婪构造退化为简单地对数组进行排序。 

最后的边缘情况是频率几乎不可行，例如$k_x = \frac{n+1}{d}$。 在这种情况下，早期的贪婪布局$x$必须仔细控制，因为早期放置可能会消除以后唯一有效的间距模式。 不平等检查$i + (r-1)\cdot d \le n$通过在每一步强制执行全局可行性来防止这种确切的故障模式。
