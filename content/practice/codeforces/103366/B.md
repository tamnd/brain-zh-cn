 ## 解决方案
 ## 问题理解

 给定一个正有理数，表示为约简分数 x/y，并要求我们将其表示为有限连分数。 这里的连分数是一个嵌套表达式，其中的值被写为整数部分 a0 加上由剩余系数 a1、a2 等形成的另一个连分数的倒数，直到 an。 

用更具操作性的术语来说，我们希望将 x/y 分解为整数序列，这样如果我们从底部开始并重复反转和相加，我们就可以准确地重建原始分数。 输出不是单个值，而是唯一描述此分解的一系列系数。 

值大小方面的约束很大，x 和 y 最多为 10^9，测试用例最多为 1000 个。 这强烈表明任何解决方案都必须以每个测试用例的对数时间运行，因为即使是 O(x) 或 O(y) 风格的方法也是不可能的。 标准欧几里得算法的运行时间为 O(log min(x, y))，速度相当快，并且在结构上与连分数相关，这是关键提示。 

当缩减步骤之后其中一个数字是另一个数字的倍数时，就会出现微妙的边缘情况。 例如，如果 x = 5 且 y = 1，则分数已经是整数。 连分数应该只是 [5]，而不是更长的东西。 另一个极端情况是除法步骤快速产生零余数，例如 114/1 或 1/114。 在这些情况下，序列变得非常短，并且总是尝试附加相反步骤的粗心实现可能会错误地附加额外的零或无效项。 

## 方法

 思考问题的一种天真的方法是直接模拟定义。 人们可以重复计算 x/y 的整数部分，减去它，然后反转剩余的小数部分，一直持续到分数变成整数为止。 这在概念上是干净的：每个步骤提取 ai = Floor(x/y)，然后用 (y, x - ai * y) 替换 (x, y)。 这是有效的，因为它准确地反映了连分数的数学定义。 

然而，这个过程不仅仅是一个技巧，它本质上是用不同形式编写的欧几里得算法。 关键的观察是减法步骤 x - ai * y 恰好计算除法中的余数，交换 (x, y) 对应于移动到下一个除数-余数对。 这意味着每个连分式系数直接对应于欧几里得算法中的一个商。 

蛮力的观点建议对潜在的大中间分数进行重复算术运算。 但每一步都严格地将其中一个数字减少到小于前一个除数的余数，因此步骤数受到欧几里德迭代次数的限制，该迭代次数是对数的。 

一旦我们认识到这种等价性，问题就简化为重复执行整数除法和余数提取，直到余数变为零。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 连分数的直接模拟 | O(log min(x, y)) | O(1) | O(1) | 已接受 |
 | 欧几里得算法解读| O(log min(x, y)) | O(1) | O(1) | 已接受 |

 两种方法本质上是相同的，但是通过欧几里得来查看它可以使正确性立即实现并且实现简单。 

## 算法演练

 我们重复应用整数除法，同时保持当前对 (x, y) 始终表示我们仍需要扩展的剩余分数的不变量。

1. 计算a = x // y，并将其记录为下一个连分数系数。 这会提取当前阶段分数的整数部分。 
2. 计算余数 r = x % y。 这表示从 x 中删除 y 的完整整数部分后剩下的内容。 
3. 如果余数 r 为零，我们就停止。 此时 x/y 恰好是一个整数，因此连分数以最后一个系数结束。 
4. 否则，我们将 (x, y) 替换为 (y, r)。 这对应于反转小数余数，因为剩下的分数是 r/y，取倒数给出下一阶段的 y/r。 
5. 重复该过程直至终止。 

每次迭代都会严格减少欧几里得意义上的第二个值 y，确保该过程必须完成。 

### 为什么它有效

 在每一步中，我们都将分数 x/y 表示为 a + r/y，其中 a = Floor(x/y)。 这正是连分数的第一项。 然后将剩余部分 r/y 反转以继续分解。 变换 (x, y) → (y, r) 保留了连分数构造下的分数值，因为它对应于将 r/y 重写为 1 / (y/r)。 由于欧几里得算法中的每一步恰好是一个除法步骤，因此商序列是唯一确定的，并且必须在余数变为零时终止。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_case(x, y):
    res = []
    while True:
        a = x // y
        res.append(a)
        r = x % y
        if r == 0:
            break
        x, y = y, r
    return res

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        x, y = map(int, input().split())
        cf = solve_case(x, y)
        out.append(str(len(cf) - 1) + " " + " ".join(map(str, cf)))
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```核心功能`solve_case`实现欧几里得除法循环。 每次迭代都会附加商`a = x // y`，它直接对应于连分数中的系数。 余数决定我们是终止还是继续。 

关键的实现细节是交换`(x, y) = (y, r)`，如果人们用分数而不是欧几里得算法来思考，很容易被误读。 如果没有这种交换，该过程将无法正确地终止。 

输出格式需要打印系数之间的转换次数，这就是我们输出的原因`len(cf) - 1`而不是`len(cf)`。 

## 工作示例

 ### 示例 1：x = 105，y = 38

 我们跟踪欧几里得步骤：

 | x| y | a = x//y | r = x%y |
 | --- | --- | --- | --- |
 | 105 | 105 38 | 38 2 | 29 | 29
 | 38 | 38 29 | 29 1 | 9 |
 | 29 | 29 9 | 3 | 2 |
 | 9 | 2 | 4 | 1 |
 | 2 | 1 | 2 | 0 |

 收集的系数为 [2, 1, 3, 4, 2]。 

这显示了完整的欧几里得下降，其中每个余数变得严格更小，确认每个系数对应于一个除法步骤。 

### 示例 2：x = 114，y = 1

 | x| y | a = x//y | r |
 | --- | --- | --- | --- |
 | 114 | 114 1 | 114 | 114 0 |

 我们在一步之后立即终止，产生[114]。 

这演示了整数情况，其中连分数没有嵌套结构。

 ## 边缘情况

 一种边缘情况是当 x < y 时，立即产生 a0 = 0。 例如，x = 1, y = 5 生成一个从 0 开始的序列。算法自然地处理这个问题，因为 x // y 变为 0，而求余步骤将分数翻转为 (y, x)，继续正确地进行。 

追踪 x = 1, y = 5：

 第一步给出 a = 0，余数 1。然后我们移动到 (5, 1)，产生 5 并终止。 最终的序列是[0, 5]，这是正确的连分数形式。 

另一个边缘情况是 x = y 时。 在这种情况下，a = 1 并且余数立即为零。 输出为[1]，无需进一步处理。 该算法正确停止，不会产生额外的交换或无效系数。
