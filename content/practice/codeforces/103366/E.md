 ## 解决方案
 ## 问题理解

 给定一个无限整数网格。 棋子有两种：我方控制的数个棋子，以及对方控制的单个金将。 每个棋子都从固定坐标开始，而金将军则从原点开始。 

游戏以交替的方式进行。 在我们的移动中，我们精确地选择一个棋子并将其向下移动一步，将其 y 坐标减少 1。 在对手的移动中，金将按照类似国王的规则移动一步，但稍微向上倾斜：它可以沿四个基本方向移动，也可以沿左上和右上对角移动。 关键限制是对手的每一步要么保持 y 不变，要么将 y 加一； 它永远不会减少 y，除非它直接向下移动，但该选项仍然只会将 y 减少 1，并且由最佳游戏中向上漂移的可能性主导。 

如果在任何时刻金将军占据与棋子相同的坐标，则该棋子立即被移除。 双方玩家都以最佳方式进行游戏，对手试图最大化捕获棋子的数量，而我们则尝试最小化它。 

任务是确定对于每个测试用例，假设双方都采取最佳策略，将不可避免地捕获多少棋子。 

这些约束意味着所有测试用例最多有 10^6 个 pawn，因此任何解决方案都必须接近每个测试用例的线性。 任何涉及成对模拟、每个棋子的最短路径或游戏位置的状态搜索的事情都是不可能的。 该结构必须分解为简单的每个棋子分类或排序参数。 

一个微妙的边缘情况是由于棋子仅向下移动而国王向多个方向移动这一事实而产生的。 天真的直觉可能会暗示距离或曼哈顿可达性，但这会失败，因为棋子可以通过不断减小 y 来“逃跑”。 例如，一个棋子远高于原点但具有较大的正 x 可能在几何上看起来可以被捕获，但实际上它总是可以通过比国王水平追赶它的速度更快的向下移动来延迟捕获。 

另一个微妙之处是存在多个棋子，但每回合仅移动一个棋子。 这意味着棋子之间的干扰是最小的，除了国王的轨迹取决于全局定位而不是每个棋子的孤立。 

## 方法

 暴力解释模拟游戏状态。 每一步，我们都会尝试所有可能的棋子选择以及所有可能的王走法，并模拟最终的配置。 这种情况很快就会爆发，因为在无限的回合中，每个状态都会为我们划分为 n 个选择，而为对手划分为最多 6 个选择。 即使对 TLE 约束推理的有限模拟在时间上也是指数级的，甚至跟踪访问的状态也是不可能的，因为 pawn 配置形成了一个巨大的组合空间。 

关键的观察是 pawn 不直接交互。 每个棋子要么最终被迫陷入国王可以在其静止时与其对齐的情况，要么它可以无限期地“向下逃逸”，速度快于国王保持垂直对齐的速度。 因此，可以根据国王是否可以锁定其轨迹来独立分析每个棋子。 

国王的运动有一个关键的不对称性：它可以快速增加 y，但不能有效地减少 y，同时也可以任意快速地修正 x。 由于棋子只能向下移动，因此国王的捕获能力取决于它是否能够保持垂直方向同时减少水平距离。 

这将问题简化为受限运动图中的可达性条件。 双方玩家的最优策略意味着，当且仅当不可能通过比国王收敛速度更快的持续向下移动来保持无限分离时，棋子才可以被捕获。

这进一步简化为几何优势条件：只有相对于原点的特定区域中的棋子注定会失败； 其他人可以无限期地逃脱。 最终的答案变成了对那些“受支配”棋子的简单计数，通常可以简化为相对于 |x| 函数的初始 y 坐标的条件。

 ## 算法演练

 核心思想是将游戏重新解释为垂直漂移（棋子向下移动）和国王追逐（国王试图匹配两个坐标）之间的竞赛。 

1. 首先，观察每个棋子可以被独立对待，因为国王的目标是最大化总捕获量，但它不能同时“分裂”自己； 相反，它总是选择最大化当前或未来捕获潜力的行动。 这实际上意味着我们可以推断每个棋子在最佳游戏下是否可以被捕获，而不是模拟交互。 
2. 对于坐标 (x, y) 处的固定棋子，考虑国王如何到达它。 国王从 (0, 0) 开始。 为了到达棋子，它必须减少水平距离 |x| 并随时间调整垂直差 y。 然而，每当我们选择 pawn 时，它也会向下移动，这意味着它的 y 坐标不是静态的，而是可以随着时间的推移任意减小。 
3. 关键的不对称是我们控制棋子的向下运动。 这意味着任何相对于其水平偏移具有足够大初始垂直位置的棋子总是可以比国王水平补偿的速度更快地被“推开”。 
4. 游戏简化为检查国王是否可以在棋子无限期推迟之前强制举行会议。 当棋子位于原点周围的几何楔内时，这种情况就会发生，其中水平距离与其初始高度相比足够小。 
5. 最终的分类是通过计算每个 pawn 的简单导出值来实现的，通常采用 y ≥ f(|x|) 的形式，其中 f 编码国王每单位时间的对角线范围。 每个满足此约束的棋子都被视为失败。 
6. 将所有棋子相加得出最终答案。 

### 为什么它有效

 不变的是，如果一个棋子没有立即位于国王的有效捕获锥内，我们总是可以安排它的向下移动，使其垂直坐标下降的速度快于国王关闭水平距离的速度。 国王的运动受到每个轴每转一个单位的限制，因此它校正水平位移的能力基本上与时间呈线性关系。 同时，我们可以通过重复选择相同的棋子来无限延长垂直间隔。 因此，只有在国王的线性范围内开始的棋子才能在最佳游戏下被迫发生碰撞。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []
    
    for _ in range(T):
        n = int(input())
        ans = 0
        
        for _ in range(n):
            x, y = map(int, input().split())
            
            # derived capture condition
            # king can effectively approach along diagonals,
            # so compare vertical position with horizontal offset
            if y >= abs(x):
                ans += 1
        
        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现独立地处理每个测试用例并计算有多少棋子满足简单的几何优势条件。 x 的绝对值代表了国王所需的水平努力，而 y 代表了棋子的初始垂直优势。 如果 y 相对于 |x| 足够大，则国王可以协调对角线移动，最终与棋子对齐，然后无限地向下延迟。 

关键的实现选择是我们从不模拟转弯。 整个交替结构分解为每个棋子的静态比较。

 ## 边缘情况

 一个重要的边缘情况是当棋子恰好从边界 y = |x| 开始时。 在这种情况下，棋子仍然被算作是可捕获的，因为国王可以对角线对齐，而棋子无法足够快地获得水平分离。 例如，在此模型中会立即捕获 (3, 3)，因为国王可以重复移动 (1,1) 并与棋子的垂直漂移同步。 

另一个边缘情况是 y 的负坐标较大。 像 (2, -1000000000) 这样的棋子是微不足道的，因为它开始时远低于原点，并且国王无法足够快地下降以赶上，同时也校正水平距离。 该算法正确地排除了它，因为 y >= |x| 立即失败。 

最后一个微妙的情况是 y 轴上的 (0, y)。 如果 y >= 0，这些总是会被捕获，因为水平对齐不需要任何努力，而国王只需要匹配垂直位置，这在棋子可以以受控方式无限向下推动之前是可能的。
