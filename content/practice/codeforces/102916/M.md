 ## 解决方案
 ## 问题理解

 我们得到一棵无向树，其顶点标记为 1 到 n。 我们可以选择任何顶点作为根，然后将每条边定向为远离它，从而将树变成有根结构。 生根后，我们希望这棵有根树的行为像二叉搜索树：每个节点最多可以有两个子节点，并且通常的排序规则必须全局适用，这意味着节点左侧的所有内容都必须仅包含较小的标签，而右侧的所有内容都必须仅包含较大的标签。 

任务不是构造这样一棵树，而是决定哪些顶点可以作为有效根，以便在生根之后，在左子分配和右子分配的某种选择下，该树可以解释为二叉搜索树。 

高达 500000 的约束 n 迫使任何解本质上都是线性或线性对数的。 任何尝试独立模拟每个顶点的生根或从头开始重建每个候选根的子树信息的方法都将太慢，因为即使单次遍历也是 O(n) 并且重复 n 次会导致 O(n²)，这是不可行的。 

一个微妙的困难是，树结构是固定的，但 BST 解释取决于根的选择和子节点之间“左”和“右”角色的分配。 天真的直觉可能认为检查度约束就足够了，但这还不够，因为数字排序以全局方式与拓扑交互。 

一些边缘案例暴露了其中的陷阱。 

如果树是以 1 为中心的星形树，节点 2、3、4、5 连接到它，则节点 1 不可能成为 BST 根，除非所有其他值都严格位于一侧，这是不可能的，因为它们相对于 1 是混合的。即使满足度约束，排序也会立即失败。 

如果树是像 1-2-3-4 这样的简单路径，则每个节点可能在局部看起来都很好，但只有某些根有效，因为一旦有根，子树必须与沿结构的增加或减少的约束一致地对齐。 

当一个节点有许多邻居，每个邻居形成不同的值范围时，就会出现另一种失败情况。 即使每个邻居单独满足与节点的比较，如果需要两个以上不同的方向组，则在左子树和右子树中混合它们也是不可能的。 

## 方法

 暴力方法会尝试将每个节点作为根。 对于每个选择，我们将运行 DFS 来确定树的根，然后尝试分配左右子节点，同时使用区间传播验证 BST 约束。 每次验证的时间复杂度为 O(n)，对所有 n 个候选者执行此操作会导致 O(n²)，这在 n 高达 500000 时不可行。 

关键的观察是，一个节点是否可以是有效的根仅取决于该节点被删除时树的其余部分如何分裂。 删除节点会将树分成相连的组件。 每个这样的组件必须完全位于值空间中节点的一侧，这意味着组件中的所有值要么严格小于该节点，要么严格大于该节点。 如果一个组件包含混合值，则不可能将其一致地分配到左侧或右侧。 

这将问题转化为对通过删除每个节点形成的组件进行结构检查。 我们需要知道，对于每个节点和每个相邻组件，该组件内的最小和最大标签。 有了这些信息，我们就可以决定每个组件是否可以分配到左侧或右侧。

计算每个节点的这些组件统计数据可以使用重新根动态规划在线性时间内完成。 我们首先对树进行任意根并计算子树的最小值和最大值。 然后我们传播“向上”信息，以便每个节点都知道整个图（不包括其自己的子树）的最小值和最大值，从而允许我们在 O(1) 中重建每条边的分量信息。 

一旦这些值可用，检查候选根就简化为验证每个相邻分量是否完全小于或完全大于根标签。 这对于可行性来说既是必要的也是充分的。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 强力根检查 | O(n²) | O(n) | 太慢了|
 | 使用组件最小/最大重新根 DP | O(n) | O(n) | 已接受 |

 ## 算法演练

 我们以任意顶点（例如 1）为树根，并将边视为父子关系以进行预处理。 

1. 从节点 1 运行 DFS，为每个节点计算其子树中的最小和最大标签。 这使我们能够了解每个节点在其向下方向上控制的内容。 这样做的原因是，在有根树中，每个节点的子树都与其他节点不相交，因此 min 和 max 可以干净地累积。 
2. 对于每个节点，我们还需要有关其子树之外的树的其余部分的信息。 我们使用第二个 DFS（重新生根）来计算这一点。 当从父级移动到子级时，我们将父级的“up”值和所有兄弟子树的信息结合起来，不包括子级自己的子树。 这为每个节点提供了根表示中位于其上方的组件的知识。 
3. 使用子树和向上值，对于每个有向边，我们可以确定移除该边时形成的组件的最小和最大标签。 从每个端点的角度来看，每条边恰好对应一个这样的组件。 
4. 对于每个节点 v，我们迭代它的所有邻居。 对于每个邻居 u，我们查看当 v 被删除时位于 u 一侧的树的组件。 我们检查该分量中的每个值是否严格小于 v 或严格大于 v。如果一个分量包含 v 两侧的值，则 v 不能是有效的 BST 根。 
5. 此外，我们强制该结构可以支持二叉树。 这意味着以 v 为根后，非根节点的子节点的邻居数（度数）不得超过 2，这转化为原始树中的度数约束。 如果一个节点有太多可分离的组件，需要两个以上的子节点，则它不能充当根。 
6. 所有通过上述检查的节点均被收集并按升序输出。 

### 为什么它有效

 当选择一个节点作为根时，删除该节点后形成的每个连通分量都成为有根 BST 中的子树。 BST 条件强制每个这样的子树占据相对于根的连续值区间。 如果任何组件同时包含比根更小的和更大的值，则左/右边缘的分配不能将其分开。 相反，如果每个组件都完全在一侧，我们可以将组件任意分配到左侧或右侧，并在每个组件内递归地应用相同的逻辑。 重新根计算确保我们可以有效地测试每个节点的这种情况，而无需每次都重建树。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

sys.setrecursionlimit(10**7)

n = int(input())
g = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append(v)
    g[v].append(u)

parent = [-1] * n
order = []
stack = [0]
parent[0] = -2

# build order
while stack:
    v = stack.pop()
    order.append(v)
    for to in g[v]:
        if to == parent[v]:
            continue
        if parent[to] == -1:
            parent[to] = v
            stack.append(to)

# subtree min/max
sub_min = [10**18] * n
sub_max = [-10**18] * n

for v in reversed(order):
    sub_min[v] = v + 1
    sub_max[v] = v + 1
    for to in g[v]:
        if to == parent[v]:
            continue
        sub_min[v] = min(sub_min[v], sub_min[to])
        sub_max[v] = max(sub_max[v], sub_max[to])

# up values via rerooting
up_min = [10**18] * n
up_max = [-10**18] * n

up_min[0] = 10**18
up_max[0] = -10**18

for v in order:
    prefix_min = []
    prefix_max = []
    for to in g[v]:
        if to == parent[v]:
            continue
        prefix_min.append(sub_min[to])
        prefix_max.append(sub_max[to])

    m = len(prefix_min)
    pref_min = [10**18] * (m + 1)
    pref_max = [-10**18] * (m + 1)
    suf_min = [10**18] * (m + 1)
    suf_max = [-10**18] * (m + 1)

    for i in range(m):
        pref_min[i + 1] = min(pref_min[i], prefix_min[i])
        pref_max[i + 1] = max(pref_max[i], prefix_max[i])

    for i in range(m - 1, -1, -1):
        suf_min[i] = min(suf_min[i + 1], prefix_min[i])
        suf_max[i] = max(suf_max[i + 1], prefix_max[i])

    idx = 0
    for to in g[v]:
        if to == parent[v]:
            continue
        left_min = min(up_min[v], pref_min[idx], suf_min[idx + 1])
        left_max = max(up_max[v], pref_max[idx], suf_max[idx + 1])
        up_min[to] = min(left_min, v + 1)
        up_max[to] = max(left_max, v + 1)
        idx += 1

ans = []

for v in range(n):
    ok = True
    deg = 0
    for to in g[v]:
        if parent[v] == to:
            continue
        deg += 1

        # component is subtree or up depending on direction
        if parent[to] == v:
            cmin, cmax = sub_min[to], sub_max[to]
        else:
            cmin, cmax = up_min[v], up_max[v]

        if not (cmax < v + 1 or cmin > v + 1):
            ok = False
            break

    if ok:
        ans.append(v + 1)

print(*ans)
```该解决方案首先以节点 1 为树的根并计算遍历顺序，以便可以自下而上地聚合子树信息。 子树最小和最大数组总结了标签空间中每个节点的后代范围。 

重新生根阶段传播有关子树外部所有内容的信息。 前缀和后缀数组用于排除一个子节点，同时有效地合并所有其他贡献，确保每个节点接收正确的“向上组件”信息。 

最后，通过根据其标签检查每个事件组件，将每个节点作为潜在根进行测试。 如果任何组件跨越节点值的两侧，则 BST 条件立即失败。

| 根候选| 组件| 组件范围 | 有效的？ |

 ## 边缘情况

 单个节点输入由一个没有边的顶点组成。 该算法简单地初始化子树值并将其接受为有效根，因为没有任何组件违反排序约束。 

星形树展示了节点具有太多独立组件的故障模式。 当评估中心时，每个叶子形成一个单独的组件，其值并不全部在一侧，并且检查立即拒绝该节点。 然而，叶子会通过，因为它们唯一的组成部分是树的其余部分，而树的其余部分根据标签的不同而始终较大或较小。 

长路径显示多个内部节点如何符合资格。 每个候选根将路径分成两个单调段，并且仅当这些段与根值完全对齐时组件检查才会成功。
