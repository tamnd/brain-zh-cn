 ## 解决方案
 ## 问题理解

 我们正在组织一场由 n 位玩家参加的完整循环赛国际象棋锦标赛，这意味着每对玩家必须恰好相遇一次。 这就创建了一组固定的 n(n−1)/2 场比赛，而我们唯一的灵活性是如何随着时间的推移安排它们。 

限制是只有 k 个棋盘可用，因此在任何给定的时间段内我们最多可以并行运行 k 个游戏。 “回合”就是这样一个时间段，我们选择一些不相交的玩家对，并将每对玩家分配到一个棋盘上，但限制是没有玩家在同一回合中出现在超过一场游戏中。 

任务是构建一个完成所有比赛的时间表，同时最小化轮数。 

输入只是 n 和 k。 输出是一系列轮次，其中每轮列出最多 k 个不相交的游戏。 

从复杂度的角度来看，n 最多为 200，因此匹配总数最多为 19900。任何显式跟踪所有匹配的解决方案都是可行的。 然而，真正的约束不是计算而是构造：我们需要一种系统的方法将完整图的边打包到比赛日中，每天的容量为 k 个不相交的边。 

一个天真的时间表会贪婪地模拟回合，重复扫描所有剩余的比赛并挑选 k 个不相交的比赛。 如果不小心实施，这种方法将面临 O(n^4) 行为的风险，但更重要的是，除非精心构建，否则它不能保证最优性。 

一个关键的结构边缘情况是当 k 非常小时。 如果 k = 1，则每一轮只包含一场比赛，因此答案被迫为 n(n−1)/2 轮。 在另一个极端，如果 k ≥ n/2，我们通常可以使轮次接近完美匹配，但前提是我们可以确保不相交的配对结构。 

主要挑战是任意贪婪的配对可能会让玩家频繁“受阻”，从而不必要地增加回合数。 

## 方法

 这个问题可以从图论上重新表述。 我们给定一个完整的图 K_n，并希望将其边划分为组，其中每个组的大小最多为 k 的匹配。 每组为一轮。 我们希望最大限度地减少使用的匹配数量，但每个匹配的大小都有容量限制。 

下界是直接的：每轮最多可以包含 k 个边，因此至少需要 ceil(n(n−1)/(2k)) 轮。 然而，仅此界限不足以直接构建最佳调度。 

考虑构造的一个有用方法是关注顶点度数而不是边缘。 每个玩家的度数为 n−1，玩家每次参与一轮，都会消耗其剩余度数的一个单位。 由于每一轮最多允许每个玩家参与一次，因此瓶颈是在各轮之间分配这些度数单位，同时遵守每轮 k 个边的全局上限。 

关键的观察是，我们实际上不需要仔细选择哪些边在全局范围内连接在一起。 相反，我们可以贪婪地构建时间表，同时保持一个简单的不变量：我们总是尝试在每一轮中填充尽可能多的未使用的玩家，最多 k 个边，按顺序扫描玩家并将他们与下一个可用的对手配对。 

因为 n ≤ 200，我们可以维护一个简单的布尔矩阵或邻接结构来标记比赛是否已安排。 每次开始一轮时，我们都会迭代玩家，并贪婪地为每个未使用的玩家选择第一个可用的未使用的伙伴，最多填充 k 场比赛。 由于每条边仅使用一次，因此该过程在 O(n^3) 时间内完成。

正确性来自于这样一个事实：我们永远不需要“提前计划”哪个特定的边进入哪一轮； 只要我们尊重不相交并最终覆盖所有边缘，任何匹配结构都是可以接受的。 容量 k 仅限制我们每轮打包的边数，而不限制哪些边必须分组在一起。

 | ---| ---| ---| ---|
 | 扫描暴力贪婪匹配 | O(n^3) | O(n^3) | O(n^2) | O(n^2) | 已接受 |
 | 最优结构贪心构造 | O(n^3) | O(n^3) | O(n^2) | O(n^2) | 已接受 |

 ## 算法演练

 我们维护一个矩阵来跟踪一对玩家是否已经被安排。 最初，所有配对均未安排。 

1. 开始一轮一轮地构建，直到所有对都用完。 每轮开始时都是空的，没有分配任何比赛。 这确保我们系统地将完整的边集分解为有效的匹配。 
2. 对于每一轮，我们迭代从 1 到 n 的所有玩家。 对于玩家 i，我们尝试为他们分配一个搭档 j，这样比赛 (i, j) 尚未安排，并且 i 和 j 都没有在当前回合中使用。 这保证了回合内的匹配属性。 
3. 当我们找到这样的一对 (i, j) 时，我们将这场比赛标记为全局使用，并将两名玩家标记为当前回合已占用。 我们继续扫描以找到更多对。 这种贪婪的填充确保我们在不违反限制的情况下尽可能多地使用回合的容量。 
4. 当我们已经达到 k 场比赛或者找不到更多有效的不相交对时，我们停止向当前回合添加比赛。 然后该轮结束并输出。 
5. 我们重复这个过程，直到所有 n(n−1)/2 场比赛都安排完毕。 

我们以固定顺序扫描的原因是为了避免病理性饥饿，即由于配对选择不一致而导致某些边缘保持未使用状态。 确定性扫描确保当两个端点在某一轮中都空闲时，最终将选择每个可用边缘。 

### 为什么它有效

 该算法保持两个不变量。 首先，每个预定的对都是唯一的并且永远不会重用，因为我们明确地将边标记为已消耗。 其次，在每一轮中，没有玩家出现超过一次，因为我们在选择后立即将两个端点标记为已占用。 

由于每轮要么调度新的边，要么由于可用边耗尽而终止，并且边的数量有限，因此该过程必须终止。 调度的完整性源于以下事实：每当一条边保持未使用时，两个端点最终将在空闲的一轮中被考虑，因为没有全局约束阻止它们在已使用的边之外进行配对。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

n, k = map(int, input().split())

used = [[False] * (n + 1) for _ in range(n + 1)]
remaining = n * (n - 1) // 2

out = []
while remaining > 0:
    taken = [False] * (n + 1)
    round_games = []

    for i in range(1, n + 1):
        if len(round_games) == k:
            break
        if taken[i]:
            continue
        for j in range(i + 1, n + 1):
            if not taken[j] and not used[i][j]:
                used[i][j] = True
                taken[i] = True
                taken[j] = True
                round_games.append((i, j))
                remaining -= 1
                break

    out.append(round_games)

print(len(out))
for r in out:
    print(len(r))
    for a, b in r:
        print(a, b)
```该代码维护一个全局邻接矩阵`used`这可以防止任何对被安排两次。 变量`remaining`跟踪剩余的边数，允许在分配所有匹配项后提前终止。 

在每一轮比赛中，`taken`确保没有玩家参与超过一场游戏。 嵌套循环结构是有意为之的：对于每个玩家 i，我们向前扫描以找到第一个有效的伙伴 j。 这种贪婪的选择是安全的，因为任何有效的配对在可行性方面都是等效的，并且我们不会尝试在单轮内优化超出填充能力的任何内容。 

当一个微妙的点被打破时`len(round_games) == k`。 如果没有这个，我们可能会溢出一轮并违反董事会限制。 

## 工作示例

 ### 示例：n = 4，k = 2

 我们从所有未安排的对开始：(1,2)、(1,3)、(1,4)、(2,3)、(2,4)、(3,4)。 

在第 1 轮中，我们扫描 i = 1。我们选择 (1,2)。 然后取 i = 2，i = 3 与 4 配对，所以我们得到 (3,4)。 第 1 轮有 2 场比赛。 

| 圆形| 我扫描| 已选对 | 剩余边缘|
 | ---| ---| ---| ---|
 | 1 | 1→4 | (1,2), (3,4) | (1,2), (3,4) | 4 |
 | 2 | 1→4 | (1,3), (2,4) | (1,3), (2,4) | 2 |
 | 3 | 1→4 | (1,4), (2,3) | 0 |

 这演示了贪婪扫描如何自然地将舍入填充到容量，同时仍然恰好覆盖所有边缘一次。 

### 示例：n = 5，k = 2

 从 10 条边开始。 每轮最多可以有 2 条不相交的边。 

| 圆形| 已选择对 | 剩余|
 | ---| ---| ---|
 | 1 | (1,2), (3,4) | (1,2), (3,4) | 8 |
 | 2 | (1,3), (2,5) | 6 |
 | 3 | (1,4), (2,3) | 4 |
 | 4 | (1,5), (2,4) | (1,5), (2,4) | 2 |
 | 5 | (3,5) | 0 |

 该跟踪表明，即使没有明确的全局规划，贪婪策略也会不断找到有效的不相交匹配，直到完成。

 ## 边缘情况

 当 k = 1 时，每轮只能包含一场比赛。 该算法将始终在每轮中准确挑选一对未使用的对，因为在选择 (i, j) 后，两个端点都将被占用，并且不可能进一步配对。 调度退化为所有边的简单枚举。 

当k很大时，例如k≥n/2，每轮通常可以容纳近乎完美的匹配。 贪婪扫描倾向于快速挑选不相交的对，因为许多玩家在每轮开始时仍然空闲，因此内部循环尽早找到有效的伙伴并有效地填充容量。 

当 n 为奇数时，每一轮必然会留下至少一名玩家未使用，这自然由`taken`大批。 不需要特殊的套管，因为当没有伙伴可用时，内部循环会简单地跳过不匹配的顶点。
