 ## 解决方案
 ## 问题理解

 我们在数轴上得到一组线段，每个线段跨越两个偶数。 任务是将一组点放置在同一条线上，以便每个线段恰好包含一个选定的点，同时还确保每个选定的点至少位于一个线段内。 

这会在段和点之间创建紧密耦合。 每个航段必须独占“获得”一个积分，但只有当这些航段最终仍各自获得一分时，才允许积分服务于多个航段。 困难在于平衡覆盖范围和排他性，因为在一个区域中放置一个点可能会满足多个重叠的段，从而违反“每个段恰好一个”规则。 

坐标空间的范围很小，从 0 到 4n − 2，但 n 本身可以大到 200000。这排除了任何试图独立探索候选点位置或检查所有线段子集的解决方案。 任何 n 的二次方都会太慢，因为 2e5 的平方已经超出了可行的限制几个数量级。 端点的结构是偶数，这暗示我们可以将连续整数视为离散槽，从而有效地将几何直觉简化为区间上的组合。 

当段完全嵌套时，会出现微妙的失败情况。 例如，考虑段 [0, 10]、[2, 8]、[4, 6]。 最小线段内的任何单个点都位于所有三个线段中，这立即打破了每个线段仅包含一个点的要求。 这表明仅仅选择线段内的任意点是不够的； 我们必须小心避免将多个线段堆叠到同一点上，除非重叠符合“恰好一个”约束。 

当片段以链状方式重叠但不能一致地分配不同的代表点时，就会出现另一种边缘情况。 例如，如果两个段相同，例如 [0, 4] 和 [0, 4]，则这是不可能的，因为两者都需要唯一点，但 [0, 4] 中的任何点都属于两个段，从而违反了每​​个段的唯一性。 

## 方法

 直接的尝试是为每个段独立分配一个点。 对于每个线段，我们可以尝试其中的每个可能的坐标，并检查在那里放置一个点是否保持所有先前的约束有效。 这很快就会退化为检查所有先前放置的点的兼容性，因为新点可能会意外地位于多个较早的段中并导致它们获得额外的分数。 

在最坏的情况下，每次放置都需要扫描所有现有的段或点以确保可行性，从而给出 O(n²) 过程。 对于 200000 个段，这变得完全不可行。 

关键的观察结果是，可行性从根本上来说与排序约束有关。 一旦放置了一个点，它就会分割线并隐式确定哪些线段已经满足。 我们可以考虑将片段与离散的候选位置相匹配，而不是考虑任意放置。 

因为所有端点都是偶数，所以我们可以将线解释为整数槽并使用贪婪匹配策略：按右端点的升序处理段，并将不违反先前分配的段的最左侧可用有效位置分配给每个段。 这将问题转化为具有分配约束的间隔调度的变体。 

这样做的原因是，选择最早的有效位置可以为后续段保留灵活性。 如果我们延迟在某个段内的放置，我们可能会冒着阻止可用范围更小的未来更紧密的段的风险。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力 | O(n²) | O(n) | 太慢了|
 | 右端点贪婪 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 按右端点升序对所有段进行排序。 这确保了当我们处理一个段时，所有较早的段结束的时间不晚于该段。 排序至关重要，因为必须首先满足较早结束的约束。 
2. 维护一个数据结构，用于跟踪线上的哪些位置已被用作点。 由于坐标是离散且有界的，因此这可以是简单的布尔数组或不相交集样式的“下一个可用位置”结构。 
3. 按排序顺序迭代段。 对于每个段[L,R]，搜索[L,R]中尚未使用的最小位置x。 
4. 如果不存在该位置，则立即返回失败。 这意味着该段中的每个候选位置已经被分配给前一个段的点占据，因此该段将被迫共享一个点，违反了“每个段恰好一个”的要求。 
5. 如果存在这样的位置 x，则在 x 处分配一个点并将其标记为已使用。 
6. 继续，直到处理完所有段，然后输出所有选定的点。 

关键的实现细节是有效地找到范围中的下一个未使用的位置。 简单的扫描时间复杂度为 O(n²)，因此我们使用并查找结构，其中每个位置都指向下一个空闲槽。 一旦使用了一个位置，我们就将其与下一个位置合并，从而在将来的查询中有效地跳过它。 

### 为什么它有效

 贪心选择确保每个段尽早被分配到不同的代表位置。 由于片段是通过增加右端点来处理的，因此为片段选择的任何位置都尽可能位于左侧，为后面的片段留下最大的空间。 并查找结构保证我们永远不会重复使用位置，因此没有两个段共享相同的点。 如果一个段在其范围内找不到空闲位置，则意味着所有可能的代表都已提交给较早的段，并且任何分配都会强制重复，从而导致配置无法进行。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def find(x, parent):
    if parent[x] != x:
        parent[x] = find(parent[x], parent)
    return parent[x]

def union(x, parent):
    parent[x] = find(x + 1, parent)

def solve():
    n = int(input())
    seg = [tuple(map(int, input().split())) for _ in range(n)]
    
    seg.sort(key=lambda x: x[1])
    
    maxv = 4 * n + 5
    parent = list(range(maxv))
    
    res = []
    
    for l, r in seg:
        x = find(l, parent)
        if x > r:
            print(-1)
            return
        res.append(x)
        union(x, parent)
    
    print(len(res))
    print(*res)

if __name__ == "__main__":
    solve()
```代码首先按段的右端点对段进行排序，以便我们始终首先处理最受约束的段。 通过父数组实现的并查找结构跟踪给定坐标处或给定坐标之后的下一个可用位置。 

这`find`函数执行路径压缩，确保重复查询快速跳转到下一个空闲槽。 一旦我们分配了一个位置`x`对于某个细分，我们立即通过将其链接到来“将其从可用性中删除”`x + 1`。 这确保没有两个段重复使用相同的坐标。 

关键检查`if x > r`确保如果第一个可用位置已位于段之外，则不存在有效的放置。

 ## 边缘情况

 完全嵌套的结构测试贪婪分配是否避免捕获后面的段。 对于像 [0,10]、[2,8]、[4,6] 这样的输入，算法会先分配 0，然后分配 2，然后分配 4（如果可用），但在更严格的配置中，当最里面的段中不存在新槽时，算法将会失败。 并查找机制确保坐标一旦被消耗就无法重复使用，从而防止意外重叠。 

相同的段测试算法是否错误地将它们合并到一个约束中。 对于重复的 [0,2] 段，每个段都必须接收一个不同的点。 联合步骤迫使前进到下一个自由位置，确保分离。 

紧密链接的段（例如 [0,2]、[2,4]、[4,6]）验证边界共享不会导致冲突。 由于坐标被视为离散槽，并且每次分配都会立即提高可用性，因此可以一致地处理共享端点，而不会违反重用。
