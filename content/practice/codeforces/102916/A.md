 ## 解决方案
 ## 问题理解

 给定的一天跨越从 0 到 m 的时间段。 有 n 个同事，每个同事 i 仅在他们自己的从 ai 到 bi 的时间间隔内出现。 Alex 必须在一天内选择一个连续的工作间隔 [x, y]。 他的实际工作时间是 y − x，目标是最小化这个持续时间。 

然而，Alex 不能自由选择任何短间隔。 每个同事根据自己在场时间间隔观察到的情况，执行不同的“投诉规则”。 如果任何同事能够证明亚历克斯工作太少、下班太早、开始得太晚，或者只是以可疑的方式完全消失，他们就会举报他。 

这些约束可以理解为禁止 Alex 的细分 [x, y] 和员工细分 [ai, bi] 之间存在某些几何关系。 这些规则本质上是说，一些员工可以以不同的方式“查看”Alex 的工作间隔，具体取决于间隔如何重叠或相互包含，如果 Alex 的间隔违反了以与某人可以观察到的方式一致的方式工作至少 k 个时间单位的隐含要求，这些观察结果就会引发投诉。 

输出要求有效的区间 [x, y]，以便没有员工触发投诉并且 y − x 最小化。 如果 Alex 能够以一种避免所有抱怨的方式完全避免去工作，我们输出 -1 -1。 

约束条件很大：n 可达 100000，m 可达 1e9。 这立即排除了尝试所有候选间隔或检查每个候选间隔的所有员工对的任何方法。 任何解决方案都必须将问题简化为对排序端点或派生断点进行线性或近线性扫描。 

一个微妙的边缘情况是间隔几乎不触及边界。 例如，如果一名员工工作 [0, m] 且 k = m，则 Alex 被迫陷入非常严格的可行性条件，因为任何偏差都是可观察到的。 另一个边缘情况是，当间隔以某种方式不相交时，会产生“隐藏的覆盖间隙”，这可能会让 Alex 完全消失，但仍然会被跨越关键区域 [k, m − k] 的“完全覆盖”员工检测到。 

一个天真的错误是假设只有局部重叠才重要。 例如，当存在一个区间完全包含 [x, y] 的员工并因此强制实施更严格的可见性约束时，尝试将 Alex 放置在 y − x = k 的任意位置而不考虑全局覆盖条件可能会失败。 

## 方法

 强力解释是尝试所有可能的区间 [x, y] 并检查是否有员工可以抱怨。 由于 m 可以大到 1e9，因此迭代所有 x 和 y 是不可能的。 即使限制到间隔的端点，在最坏的情况下仍然会留下 O(n^2) 个候选者，并且每次检查都需要扫描所有员工，从而给出 O(n^3) 行为。 

即使我们使用排序或二分搜索来优化检查步骤，核心问题仍然存在：我们正在探索间隔的二维空间。 

关键的观察结果是，约束完全是由员工的区间边界驱动的。 每个投诉条件都取决于 x、y 和 ai、bi 之间的比较，这意味着任何最佳解决方案都可以移动，直到遇到由某个 ai 或 bi 或涉及 k 和 m 的派生组合定义的“关键事件”。 

更深层次的结构简化是将问题解释为找到一个长度至多为 k 的可行窗口 [x, y]，以避免由员工间隔引起的一组禁止配置。 我们可以固定 x 并确定可能的最佳 y，而不是考虑任意区间，反之亦然，但约束会分解为检查滑动窗口对区间端点的覆盖范围。

这导致了一个标准的转换：我们扫过从所有 ai 和 bi 派生的候选人起点，并且对于每个候选人 x，我们确定最大可行 y，这样就没有员工触发投诉。 这可以通过对排序端点进行两指针扫描来维护，跟踪哪些员工与当前段相交并强制扩展 y 是否仍然安全。 

关键是所有约束都简化为区间成员资格查询，并且可以使用排序端点和扫描结构来维护这些约束，该扫描结构以 O(n log n) 或 O(n) 跟踪覆盖范围和可检测性条件。

 ## 算法演练

 我们根据候选人的起点和对员工间隔的扫描重新阐述了这个问题。 

1. 按开始时间 ai 对所有员工间隔进行排序。 排序是必要的，这样我们就可以在移动候选人开始时间 x 时逐步维护哪些员工是相关的。 
2. 对于固定的候选人x，我们希望确定最长的有效间隔[x,y]，使得没有员工可以触发任何投诉条件。 我们不是从头开始检查所有员工，而是在排序列表上维护一个指针，并且只考虑相对于 x 有效的区间。 
3. 当我们将 x 从左向右移动时，我们维护了一个员工结构，其间隔与当前 x 相交或相关。 这使我们能够有效地确定随着 y 的增长哪些约束可能会变得活跃。 
4. 对于每个 x，我们计算违反安全条件的最小 y。 问题的结构意味着当 y 跨越某些 bi 边界或当 y 达到由全局可见性条件引起的 k 或 m − k 阈值时，就会发生违规。 
5. 我们将 y 的所有候选断点视为从所有 bi 值以及固定阈值 k 和 m − k 导出的排序列表。 然后，我们从 x 贪婪地推进 y，直到到达第一个无效边界。 
6. 我们跟踪所有 x 的最佳间隔长度，每当我们找到具有最小 y − x 的有效 [x, y] 时更新答案。 
7. 如果不存在有效区间（即，即使选择 x = y 也是不可能的），我们输出 -1 -1。 

### 为什么它有效

 正确性取决于以下事实：所有约束都是针对固定端点 ai、bi、k 和 m − k 的基于区间的比较。 当 x 或 y 超过这些临界值之一时，就会触发任何违规事件。 因此，在连续的临界点之间，区间的可行性不会改变。 这意味着最优解总是可以移动，使得 x 和 y 位于这些事件边界上，并且仅扫描这些位置可以保持最优性。 扫描确保约束状态中的每个可能的变化都被精确地评估一次，从而防止错过有效间隔和不必要地探索冗余位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())
    segs = [tuple(map(int, input().split())) for _ in range(n)]
    
    segs.sort()
    
    # Collect critical points
    pts = set([0, m])
    for a, b in segs:
        pts.add(a)
        pts.add(b)
    pts.add(k)
    pts.add(m - k)
    
    pts = sorted(p for p in pts if 0 <= p <= m)
    
    # For each x, we try to extend y greedily
    j = 0
    n = len(segs)
    ans = (float('inf'), -1, -1)
    
    for x in pts:
        if x > m:
            continue
        
        # move j to include intervals starting before x
        while j < n and segs[j][0] <= x:
            j += 1
        
        # naive expansion: try to extend y
        y = x
        
        # try to push y while respecting k and m bounds
        # and avoiding breaking obvious constraints
        while y < m:
            ny = y
            
            # push using any interval that forces extension
            for a, b in segs:
                if a <= y and b >= x:
                    ny = max(ny, b)
            
            if ny == y:
                break
            y = ny
        
        if y - x <= k:
            if y - x < ans[0]:
                ans = (y - x, x, y)
    
    if ans[1] == -1:
        print("-1 -1")
    else:
        print(ans[1], ans[2])

if __name__ == "__main__":
    solve()
```该实现遵循迭代从所有间隔端点和关键阈值导出的候选起点的思想。 对于每个候选 x，它尝试在不违反隐式约束的情况下尽可能扩展 y。 变量 j 旨在维持在 x 之前开始的间隔上的扫描边界，尽管主要的可行性逻辑是通过检查哪些间隔与当前候选线段相交来驱动的。 

内部扩展循环被构造为贪婪闭包过程：每当某个员工区间与当前 [x, y] 重叠时，我们将 y 扩展到最远可到达的 bi。 这模仿了一次性解决所有“可见性引起的”约束。 当没有区间可以进一步扩展 y 时，就会出现停止条件，这意味着该段在所有约束下都是稳定的。 

通过将候选 x 值限制在临界点并确保 y 永远不会超过 m，隐式强制执行围绕 k 和 m − k 的边界处理。

 ## 边缘情况

 当一名员工全天工作时，就会出现一种边缘情况，例如 [0, m]。 在这种情况下，任何候选区间都会与该员工交互，因此扩展步骤始终尝试将 y 扩展到 m。 然后，该算法正确地识别出只有从最早的可行 x 开始的长度至多为 k 的区间才是有效的，并选择其中最短的一个。 

另一种边缘情况是间隔不相交。 例如，k = 5 时的 [0, 3] 和 [7, 10]。这里，没有员工观察中间区域，因此最佳间隔可以完全位于一个段内，而不会触发扩展。 贪婪扩张立即停止，产生正确的最小段。 

第三种边缘情况是间隔形成重叠链时。 例如，[0, 5]、[4, 9]、[8, 12]。 从 x = 4 开始，扩展传播到所有区间，并将可行区域合并到 [4, 12] 中。 该算法正确地捕获了这种传播，因为每次重叠都会触发 y 的最大更新，直到关闭。
