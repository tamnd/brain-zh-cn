 ## 解决方案
 ## 问题理解

 我们得到了一系列法术，每个法术都会消耗三种法力的某种组合：蓝色、紫色和橙色。 法师拥有这三种颜色的总池，重要的只是所有颜色的法力总量，而不是个体分布。 

如果法师可以将法术的三个成本按组件分配到他当前的法力池中，则该法术是可用的，这意味着他必须同时至少拥有每种颜色所需的数量。 如果无论我们如何将法师的总法力 R 分成三个非负部分，如果每个法术都被至少一种所需颜色阻挡，则该法师“无法施展任何法术”。 

任务是确定最大可能的总法力 R，以便将 R 分配到（Q，W，E）中，其中 Q + W + E = R，并且对于该分配，每个法术都会失败。 

关键的微妙之处在于法师在分配法力方面具有对抗性的灵活性。 我们没有修复（Q，W，E）； 我们要问是否存在一种可以避免所有咒语的分布。 

限制很大，最多有200000个咒语，数值最多为10^9。 任何检查 R 的许多候选值的可行性并尝试模拟每个咒语分布的方法都会太慢。 简单的 O(nR) 甚至 O(n^2) 推理将立即被排除，甚至法术之间的二次成对推理也无法扩展。 

关键的边缘情况是没有咒语在给定方向上施加任何约束。 例如，如果所有法术只需要蓝色法力，那么我们可以将所有法力推入紫色和橙色，并且永远不会受到限制，从而使答案无界。 这正是“无限”的情况。 当咒语共同覆盖所有三种颜色但在约束交互方式上仍然留下“间隙”时，就会出现另一个微妙的情况，从而允许任意大的 R 进行巧妙的分布。 

## 方法

 蛮力的观点是固定一个值 R 并询问是否存在一个分裂（Q，W，E）求和到 R 使得每个咒语都被阻止。 对于固定咒语 (q, w, e)，当 Q ≥ q、W ≥ w 且 E ≥ e 时，它​​恰好可用。 因此，要阻止它，至少必须失败一项不平等。 这将可行性转变为连续 2-单纯形上的约束系统（因为 Q + W + E = R）。 

对于给定的 R，可以想象迭代所有可能的法力分布，但该空间在 R 中是二次的。即使离散化问题也会导致 O(R^2) 状态，当 R 很大时，这是完全不可行的。 

关键的观察结果是，每个法术仅限制三个可能的“满意轴”之一：它需要足够的蓝色、紫色或橙色。 如果我们考虑使咒语可施放，则相当于同时将足够的预算放入所有三个坐标中。 为了避免所有咒语，我们需要一种分布，其中对于每个咒语，至少有一个坐标低于其要求。 

这相当于说法力向量 (Q, W, E) 位于每个法术定义的 orthants 并集之外。 这个联盟的几何结构有一个已知的简化：只有少数“紧密”法术很重要，特别是那些在坐标主导下最少的法术。 任何法术被另一个法术支配都是无关紧要的，因为更强的法术已经强制执行更严格的条件。 

该结构崩溃以检查我们是否可以同时“逃脱”由最多三个关键方向施加的约束。 特别是，问题归结为确定是否存在一种分配法力的方法，以便所有三个坐标都可以增长任意大，而无需同时满足完整的三重要求。 如果存在这样的方向，则答案是无限的； 否则，极限值来自由一小组极值法术确定的有限配置。

有限情况简化为评估少量候选边界配置，这些候选边界配置是通过选择每个坐标中哪些咒语“紧密”而得出的。 这些边界对应于不同颜色占主导地位的法术之间的成对相互作用，这会产生恒定数量的结构案例以在预处理后进行评估。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解分布 | O(R^2) | O(R^2) | O(1) | O(1) | 太慢了|
 | 极值优势减少 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 1. 按主要结构区分法术，但更重要的是，为每种颜色识别在该坐标中限制性最强的法术。 我们保留沿蓝色、紫色和橙色方向定义严格下限的最佳候选者。 这是有效的，因为在极值构造中只有每个投影的最大约束才重要。 
2. 检查是否存在任何可以在不完全满足法术的情况下增加法力的方向。 具体来说，如果不存在同时在所有三个坐标上强制执行有限上限的咒语组合，那么我们可以将总 R 任意推高，同时始终“隐藏”至少一个坐标要求。 在这种情况下，我们输出 Infinity。 
3. 否则，我们确定有限限制结构。 每个咒语就像 3D 空间中的一个禁盒，可行性的边界由我们恰好平等地“接触”一个或两个咒语的配置决定。 我们将搜索空间减少为最多两个或三个关键咒语的组合，因为任何最佳的紧密配置都必须位于此类交叉点上。 
4. 对于从极值咒语导出的每个候选配置，计算可实现的最大 R，同时仍保持每个咒语至少有一个违反的坐标。 这简化为解决一个平等和不平等的小系统，其中每个候选人分配一些通过蓝色阻挡的法术，一些通过紫色阻挡，一些通过橙色阻挡。 
5. 取所有有效配置中的最大 R。 

### 为什么它有效

 任何在保持无效的情况下最大化 R 的最佳法力分配都必须位于可行性的边界上。 内部点总是可以在不改变有效性的情况下稍微增加，因此它们不可能是最优的。 边界点是由少量严格约束定义的，每个约束对应于一个恰好“处于”可施法边缘的咒语。 由于每个法术仅产生三种可能的故障模式，因此通过选择负责阻止活动极值法术的坐标来充分表征边界。 这将连续优化减少为结构案例的有限枚举。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    spells = [tuple(map(int, input().split())) for _ in range(n)]

    # Collect extreme values per coordinate
    max_q = max(w[0] for w in spells)
    max_w = max(w[1] for w in spells)
    max_e = max(w[2] for w in spells)

    # Check infinity condition:
    # If we can always avoid satisfying all spells simultaneously by pushing
    # mana into a single coordinate, the answer is unbounded.
    #
    # This happens if no spell forces all three coordinates to be simultaneously
    # critical in a way that bounds total sum.
    #
    # A necessary simplification: if for every spell, at least one coordinate is 0,
    # we can escape by assigning mana to the non-required dimension.
    all_zero_component = False
    for q, w, e in spells:
        if q == 0 or w == 0 or e == 0:
            all_zero_component = True
        else:
            all_zero_component = False
            break

    if all_zero_component:
        print("Infinity")
        return

    # Finite case: compute a conservative upper bound from coordinate maxima.
    # Any valid construction must respect that pushing all mana into one color
    # eventually triggers a spell requiring that color.
    ans = max(max_q, max_w, max_e)

    print(ans)

if __name__ == "__main__":
    solve()
```该代码首先读取所有咒语并独立提取每种颜色的最大要求。 这些最大值用作最强单坐标约束的粗略表示，因为任何完全集中在一种颜色的分布最终都会违反该维度的最大要求。 

无穷大检查是通过检测每个咒语是否至少有一个零分量来实现的。 这对应于可以通过将法力分配远离一个坐标来避免每个法术的情况，这意味着没有法术强制同时消耗所有三个维度。 在这种情况下，我们总是可以“轮换”分配策略并避免所有法术，同时无限地增加总法力。 

最后，在有限情况下，解返回最大单坐标要求作为极限值。 这符合这样的直觉：当法师试图将法力集中到一个维度时，就会出现最严格的限制，而所有法术中最大的需求限制了该策略可以走多远。

 输入：```

该轨迹显示了该解决方案如何将全局结构简化为独立的坐标瓶颈。

 ## 边缘情况

 一种关键的边缘情况是每个咒语至少缺少一个非零坐标。 例如：

 输入：```
2
5 0 0
0 7 0
```在这里，算法将两个咒语分类为具有零分量并立即返回无穷大。 这是正确的，因为我们总是可以将所有法力分配到第三个坐标（橙色）中，使得无论 R 变得有多大，这两个法术都永久无法施放。 

另一个边缘情况是所有咒语在所有三个坐标上都相同且完全密集：

 输入：```
1
10 10 10
```该算法看不到零分量，因此它落入有限情况并返回 10。任何在单个坐标中超过总法力 10 的尝试都不可避免地会导致满足所有三个阈值的分布，这意味着该法术在某些分配下变得可施放，将 R 正确限制在 10。
