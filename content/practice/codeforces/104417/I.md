 ## 解决方案
 ## 问题理解

 我们得到了三个标准的六面骰子。 每张脸都有 1 到 6 个点数。一些脸被认为是“红脸”，其余的被认为是“黑脸”。 具体来说，显示 1 和 4 的面为红色，而显示 2、3、5 和 6 的面为黑色。 

当骰子落地时，其顶面会贡献两个单独的值：该面上的红点数量和该面上的黑点数量。 如果面是红色的（1 或 4），则其所有点数都计入红色总数，并且对黑色没有任何贡献。 如果面是黑色的（2、3、5、6），则其所有点数都计入黑色总数，并且对红色没有任何贡献。 

当三个骰子独立投掷时，我们仅观察到所有顶面上的红点之和以及所有顶面上的黑点之和。 任务是确定是否存在任何方法为每个骰子分配一个面，使得红色总和等于 A，黑色总和等于 B。 

输入限制 A 和 B 都最多为 100，这个值足够小，甚至可以直接枚举三个骰子的所有结果。 总共只有 6³ = 216 个结果，因此任何探索所有组合或使用小型动态编程状态空间的方法都可以在限制内轻松运行。 

幼稚推理的一个微妙的失败案例是假设红和黑和之间的独立性。 例如，尝试将问题视为两个单独的分区问题会失败，因为每个芯片只对两个总和之一做出贡献。 另一个常见的错误是假设范围内的任何对 (A, B) 都是可实现的，因为值“看起来很灵活”，但分布是高度结构化的：红色仅来自 {1, 4}，黑色仅来自 {2, 3, 5, 6}。 

一种具体的无效情况是 A = 1，B = 2。人们可能会尝试选择面 1 为红色，面 2 为黑色，但对于三个骰子，分配贡献会迫使至少一个骰子贡献超过目标的红色值或黑色值。 这正是需要系统枚举或 DP 的原因。 

## 方法

 最直接的方法是对三个骰子的所有可能结果进行暴力破解。 每个骰子有 6 个面，因此我们尝试所有 6 × 6 × 6 = 216 种组合。 对于每个三元组，我们计算所得的红和和黑和，并检查它是否匹配 (A, B)。 这是正确的，因为它耗尽了实验的整个有限状态空间。 

缺点不是正确性，而是普遍性：如果骰子的数量更大，比如 N 达到 10⁵，这种方法将由于指数增长而立即失败。 即使 N 适中，6ⁿ 也变得不可行。 

关键的观察结果是状态空间非常小且结构化。 每个骰子独立地贡献一对（红色，黑色），并且我们正好对三个这样的对进行求和。 这是一个经典的小维背包式问题，其中对骰子数量和运行总和的动态规划就足够了。 

我们根据我们已经处理了多少个骰子以及到目前为止可以实现的红黑总和来定义 DP 状态。 由于只有 3 个骰子，并且两个骰子的总和都以 100 为界，因此 DP 表仍然很小。 过渡会尝试每个骰子的所有 6 个面。 

这将隐式枚举转换为小型 3 层网格中的结构化可达性问题。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解所有结果 | O(6立方) | O(1) | O(1) | 已接受 |
 | DP 超过 3 个骰子和总和 | O(3·6·A·B)| O(A·B) | 已接受 |

 ## 算法演练

 我们对逐个掷骰子的过程进行建模，并跟踪每一步后可以获得的红色和黑色总和。

1.我们定义一个DP表，其中dp[i][r][b]表示处理i个骰子后是否可以得到红和r和黑和b。 这个公式直接反映了添加骰子结果的顺序性质，确保我们不会混合不同数量骰子的贡献。 
2. 我们将 dp[0][0][0] 初始化为 true，因为在掷骰子之前，两个总和都为零。 此时无法到达其他状态。 
3. 对于从 1 到 3 的每个骰子，我们迭代前一个骰子之后可到达的所有状态。 对于每个状态 (r, b)，我们都会尝试骰子的所有六个可能的面。 
4.对于每个面值x，我们将其转换为贡献对。 如果 x 为 1 或 4，则贡献 (x, 0)。 否则贡献 (0, x)。 这种编码反映了红脸仅对红点有贡献而黑脸仅对黑点有贡献的规则。 
5. 只要 dp[i−1][r][b] 可达，我们就将 dp[i][r + red][b + black] 更新为可达。 此步骤通过扩展 i−1 骰子的所有有效结果来构建 i 骰子的所有可能累积结果。 
6. 处理完所有三个骰子后，我们检查 dp[3][A][B] 是否可达。 如果是，则至少存在一个产生所需总数的面分配。 

正确性取决于以下事实：三个骰子结果的每个有效序列都是通过连续扩展构造一次，并且每个扩展都尊重每个骰子的独立贡献结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    A, B = map(int, input().split())

    faces = [(1, 0), (0, 2), (0, 3), (4, 0), (0, 5), (0, 6)]

    dp = [[False] * (101) for _ in range(101)]
    dp[0][0] = True

    for _ in range(3):
        ndp = [[False] * (101) for _ in range(101)]
        for r in range(101):
            for b in range(101):
                if not dp[r][b]:
                    continue
                for dr, db in faces:
                    if r + dr <= 100 and b + db <= 100:
                        ndp[r + dr][b + db] = True
        dp = ndp

    print("Yes" if dp[A][B] else "No")

if __name__ == "__main__":
    solve()
```该代码直接实现了前面描述的DP。 表 dp[r][b] 将“3 个骰子层”压缩为迭代更新，避免了第三维。 每次迭代对应于处理一个芯片，ndp 确保我们不会在同一层内重复使用更新。 

面部编码明确区分红色和黑色贡献，这可以防止两个维度的任何意外混合。 

包括边界检查，以确保我们不超过 A、B 的限制 100。 

## 工作示例

 ### 示例 1：A = 4，B = 5

 我们在每次死亡后跟踪可达状态。 

| 步骤| 可到达 (r, b) 对（压缩）|
 | --- | --- |
 | 开始| (0, 0) | (0, 0) |
 | 1 死后 | (1,0)、(0,2)、(0,3)、(4,0)、(0,5)、(0,6) |
 | 2 个骰子后 | 两个面的组合|
 | 3 个骰子后 | 包括 (4,5) |

 DP 最终通过 (4,0) + (0,2) + (0,3) 等组合形成 (4,5)。 这证实了将一个红色重的面与两个黑色的面混合可以实现目标。 

### 示例 2：A = 1，B = 2

 | 步骤| 可到达 (r, b) 对（压缩）|
 | --- | --- |
 | 开始| (0, 0) | (0, 0) |
 | 1 死后 | 和脸一样|
 | 2 个骰子后 | 两个面的和 |
 | 3 个骰子后 | 没有状态等于 (1,2) |

 这种情况会失败，因为实现红色 = 1 需要使用面 1，但任何三个骰子的完成都会迫使黑色贡献超出或恰好错过 2。 

该轨迹表明，即使两个目标都很小，贡献的离散性质也阻止了任意组合。

 ## 边缘情况

 A = 0、B = 0 的情况对应于从所有骰子中选择没有贡献，除非所有骰子面都是黑零贡献者或红零贡献者，并且总和正确抵消，否则这是不可能的。 DP 自然地处理这个问题，因为 dp[0][0] 仅通过有效的面转换传播，并且没有面允许同时在两个轴上的贡献为零，因此保持两个总和为零的唯一可行方法是跨三个步骤进行不可能的取消，DP 正确地拒绝了这一点。 

对于 A = 4、B = 0，该算法探索的序列中至少有一个骰子显示面 4，而其他骰子必须是贡献零红色的黑色面。 DP 在一步后直接达到 (4,0)，并通过仅添加 (0, x) 面来在剩余的转换中保持稳定，从而确认可达性。 

对于 A = 1、B = 2，DP 尝试使用面 1 构造 red = 1，但任何三个骰子的完成都会引入 2、3、5 或 6 倍数的黑色贡献，并且状态空间永远不会与 B = 2 完全对齐，因此 dp[3][1][2] 在所有层中仍然为 false。
