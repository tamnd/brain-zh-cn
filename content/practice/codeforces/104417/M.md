 ## 解决方案
 ## 问题理解

 我们给出一个凸多边形，其顶点按逆时针顺序列出。 我们必须从这个多边形中选择三个不同的顶点$a, b, c$，同样按逆时针顺序，带有附加的结构约束：当沿着边界行走时$b$到$c$逆时针方向，我们必须精确地遍历$k$边缘。 这修复了$c$一次$b$选择，因为在一个简单的多边形上“$k$向前的边缘”对应于沿着顶点循环的固定索引移位。

 选择后$a, b, c$，我们构建一个新的多边形$Q$。 的一侧$Q$是原始边界链$b$到$c$，这正好贡献了$k$原始多边形的边。 剩下的两条边是对角线$ab$和$ac$。 生成的图形是一个简单的多边形$k+2$边，其面积取决于我们放置“顶点”顶点的位置$a$相对于固定边界弧$b$到$c$。 

任务是最大化面积$Q$超过所有有效的选择。 

约束条件很大：最多$10^5$每个测试用例的顶点总数。 这立即排除了任何三次甚至二次解。 尝试所有三元组或根据选择重新计算多边形区域的任何操作都会太慢。 我们应该期待$O(n)$或者$O(n \log n)$每个测试用例方法。 

一个微妙的点是，虽然多边形是凸的，但三元组中的顶点允许共线。 这不会破坏基于凸性的优化，但它确实意味着我们不能依赖严格的凸性参数，例如严格增加角度。 

一个天真的但重要的误解是治疗$a$独立于弧$[b,c]$。 现实中，有一次$b$是固定的，弧是固定的，并且贡献$a$与该弧线线性相互作用，这是允许优化的关键结构。 

通常打破天真的思维的边缘情况包括：

 如果$k = n-2$，则弧从$b$到$c$几乎覆盖了除一个顶点之外的整个多边形。 唯一的自由是在哪里$a$坐，许多实现错误地假设对称或尝试以错误的方向“闭合多边形”。 

如果多边形是三角形（$n=3, k=1$），那么每个有效的选择都会崩溃为使用完整的三角形，并且任何优化的尝试$a$仍必须返回整个区域。 

当所有点几乎共线时，会出现另一种故障模式。 在这种情况下，如果实现没有仔细评估所有候选者，叉积会变小，并且数值不稳定可能会翻转所选的最大值。 

## 方法

 暴力解释很简单。 我们选择$b$，然后修复$c$作为顶点$k$向前迈出一步，然后尝试一切可能的方法$a$与圆弧端点不同。 对于每个三元组，我们计算多边形面积$Q$直接对其使用叉积求和$k+2$顶点。 

这是正确的，但太慢了。 有$O(n)$选择$b$，并且对于每个$b$有$O(n)$选择$a$。 计算每个区域的成本$O(k)$在最坏的情况下，导致$O(n^3)$每个测试用例，这远远超出了限制。 

关键的观察是一旦电弧从$b$到$c$是固定的，则面积$Q$分为两部分：仅取决于弧的常数项和线性贡献$a$。 这是因为多边形$a, b, \dots, c$贡献的术语涉及$a$仅通过边缘$(a,b)$和$(c,a)$，其他一切都已确定。 

这意味着对于每个固定对$(b,c)$，我们只是最大化以下形式的线性函数：$$\text{contribution}(a) = \frac{1}{2} \cdot (a \times (b - c))$$所以对于固定的$b,c$，我们只需要凸多边形的顶点与从下式导出的方向向量最大化点积$b-c$。 

这将问题简化为：对于每个$b$, 计算$c=b+k$，然后在所有多边形顶点上最大化线性函数。 由于多边形是凸多边形并且顶点按顺序给出，因此可以使用旋转指针（旋转卡尺的一种形式）有效地维持该最大值，因为随着方向沿船体遍历的变化，最佳顶点单调移动。 

因此，我们避免从头开始重新计算最大值，并将总复杂度降低到线性。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(1)$| 太慢了 |
 | 直线+旋转卡尺|$O(n)$|$O(1)$| 已接受 |

 ## 算法演练

 我们按照给定的逆时针顺序处理多边形，并使用基于 0 的索引。 

1.修复起始顶点$b$。 相应的$c$被确定为顶点$k$沿着多边形边界向前迈出一步。 这确保了电弧从$b$到$c$总是准确地包含$k$边缘。 
2. 计算方向向量$d = p_b - p_c$。 这个向量完全决定了如何选择$a$影响该地区。 
3. 观察面积贡献$a$内多边形$(a, b, \dots, c)$与叉积成正比$a \times d$。 这意味着最大化面积减少为最大化所有顶点上的线性表达式。 
4. 维护指针$i$代表最佳候选人$a$。 对于每一个新$b$，我们不会从头开始。 相反，我们保留$i$并向前移动，同时下一个顶点提高了$cross(p_i, d)$。 既然两者$b$和$c$随着我们的增加而前进$b$, 方向$d$变化平滑，允许分摊恒定的指针移动。 
5. 对于每个$b$，使用当前最佳值计算最佳面积贡献$a$，将其与弧的常数部分结合起来$[b,c]$，并更新全局最大值。 
6. 返回所有的最大值$b$。 

### 为什么它有效

 关键的不变量是对于每个固定的$b$，我们最大化的函数$a$在坐标中是线性的$a$。 在凸多边形上，线性函数在极值点处达到最大值，并且随着方向向量在滑动时演化$b$，最大化顶点沿着船体单调移动。 这种单调性使得单个指针可以跟踪所有状态的最佳状态，而无需重新访问早期的候选状态。 多边形的凸性保证了任何局部改进在被跳过之后都不能再次变得最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cross(ax, ay, bx, by):
    return ax * by - ay * bx

def solve():
    n, k = map(int, input().split())
    p = [tuple(map(int, input().split())) for _ in range(n)]

    # prefix cross sum for polygon area contributions of fixed arcs
    def arc_area(b, c):
        # area of polygon chain b -> ... -> c (mod n)
        area = 0
        i = b
        while i != c:
            j = (i + 1) % n
            area += cross(p[i][0], p[i][1], p[j][0], p[j][1])
            i = j
        return area

    def contrib(a, b, c):
        return cross(p[a][0], p[a][1], p[b][0] - p[c][0], p[b][1] - p[c][1])

    ans = 0
    j = 0

    for b in range(n):
        c = (b + k) % n

        # move j to best a for current direction
        while True:
            nj = (j + 1) % n
            if nj == b or nj == c:
                break
            cur = contrib(j, b, c)
            nxt = contrib(nj, b, c)
            if nxt > cur:
                j = nj
            else:
                break

        base = arc_area(b, c)
        best = contrib(j, b, c)
        ans = max(ans, base + best)

    print(ans / 2.0)

t = int(input())
for _ in range(t):
    solve()
```该代码将边界弧的固定几何贡献与顶点引入的变量贡献分开$a$。 这`arc_area`函数计算边界链的有符号面积$b$到$c$，这与选择无关$a$。 功能`contrib`编码线性依赖$a$通过与方向向量的叉积$b-c$。 

指针`j`在迭代中保持$b$，所以我们永远不会重新开始寻找最好的$a$从头开始。 这是保持解线性的关键优化。 

一个微妙的实现问题是多边形边界上的模块化移动。 由于多边形是循环的，因此$b$和$c$计算模数$n$。 指针逻辑必须避免选择端点$b$和$c$作为$a$，因为它们是无效的选择。 

## 工作示例

 考虑一个小的凸四边形：

 输入多边形：$$(0,0), (4,0), (4,4), (0,4)$$让$k=1$。 

为了$b = (0,0)$,$c$是下一个顶点$(4,0)$。 圆弧是单边，因此底面积为零。 最好的$a$是顶点最大化叉积$b-c = (-4,0)$，对应于最大化垂直坐标。 即选择$(0,4)$或者$(4,4)$，两者给出相同的贡献。 

| 乙| c | 最好的一个| 方向 (b-c) | 贡献 | 总计 |
 | --- | --- | --- | --- | --- | --- |
 | (0,0) | (0,0) | (4,0) | (0,4) | (-4,0) | 正最大值 | 最大面积 |

 这证实了该算法正确地将问题简化为方向极值搜索。 

现在考虑一个五边形，其中的点排列使得最佳$a$转变为$b$移动。 作为$b$进步，向量$b-c$轻微转动，指针$j$相应地向前推进。 这证明了卡钳运动的摊销性质。

 ## 边缘情况

 对于$n=3$，每个有效的选择力$Q$成为原来的三角形。 弧已经跨越整个结构，因此算法的常数弧项占主导地位，并且最好$a$选择变得无关紧要。 叉积公式仍然返回整个区域，因为所有候选区域$a$产生等值的贡献。 

为了$k=n-2$，弧从$b$到$c$仅排除一个顶点。 该算法仍然处理$c=b+k$正确且最好的$a$从剩余的顶点中选择。 由于弧面积最大，解决方案减少为选择最佳方向极值，指针策略无需修改即可处理。 

对于几乎共线的点，叉积变小，但比较仍然有效，因为该算法仅依赖于值的排序，而不依赖于幅度稳定性。 指针从不依赖于浮点阈值，因此即使面积接近于零也不会发生振荡。
