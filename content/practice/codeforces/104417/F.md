 ## 解决方案
 ## 问题理解

 我们在数轴上得到了闭区间的集合。 每个区间还有一个二进制标签，红色或蓝色。 我们想要计算可以选择这些间隔的多少个子集，使得所选的红色间隔不会与所选的蓝色间隔重叠。 相同颜色的间隔可以自由重叠，但如果选择了两个间隔，则禁止相反颜色之间的任何重叠。 

如果对于每对选定的线段，它们要么不相交，要么共享相同的颜色，则子集有效。 任务是计算所有有效子集，包括空集，模 998244353。 

这些约束建议每个测试用例最多有 10^5 个段，总体为 5 × 10^5。 对或子集的任何二次枚举都是不可能的。 即使每个测试用例的 O(n log n) 或 O(n α(n)) 都是可以接受的，但任何依赖于直接检查所有对或所有子集的情况都被排除。 

重叠的相互作用产生了一个微妙的问题：如果我们选择与蓝色片段重叠的红色片段，整个子集将变得无效，但相同颜色内的重叠并不重要。 一种简单的方法可能会尝试独立计算红色有效子集和蓝色有效子集并将它们相乘，但这会失败，因为相反颜色的片段如果不及时重叠，仍然可以一起选择。 耦合是纯粹几何的，而不是全局的。 

当间隔嵌套时会出现另一种失败情况。 例如，一个长的红色区间与多个不相交的蓝色区间重叠，会强制对其范围相交的所有选择进行全局限制。 每个段的简单本地兼容性检查会错过这些累积约束。 

## 方法

 暴力破解的想法很简单：枚举段的每个子集并检查它是否有效。 对于每个子集，我们扫描所有选定的对并验证没有红蓝对重叠。 对于 n 个段，有 2^n 个子集，检查每个子集的成本为 O(n) 或 O(n^2)，具体取决于实现，导致指数或至少 O(n^3) 行为。 这显然是不可行的。 

关键的观察结果是，唯一禁止的情况是所选集合内红色区间与蓝色区间相交。 这表明随着时间的推移，要考虑颜色之间的冲突。 如果我们固定红色区间的子集，则我们可以选择的蓝色区间正是那些不与任何选定的红色区间相交的区间，反之亦然。 这种依赖结构是不对称的，但在时间上是局部的。 

我们可以将问题重新解释如下：每个区间都排除与其相交的相反颜色的区间。 因此，每个区间都有一组与其他颜色不兼容的区间。 目标是计算由区间交集引起的二部冲突图中的独立集。 

关键的结构是区间交点可以按端点排序，从而使我们能够将问题简化为端点上的扫描线或 DP。 当我们按左端点对间隔进行排序时，我们可以维护活动间隔并跟踪约束传播的距离。 我们不是明确地构建冲突图，而是按顺序处理间隔并保持已选择的间隔对未来选择的影响。 

标准变换是从左到右扫描并维护表示最后选择的间隔是否阻塞范围的 DP 状态。 因为颜色是二元的，所以我们有效地维护两个独立的区间系统，它们仅通过重叠边界相互作用。 每次我们处理一个间隔时，我们都会计算在不违反先前选择的相反颜色间隔引起的冲突的情况下可以选择多少种方式。 这减少了维护每种颜色的活动“禁止范围”的数量，并在不相交的时间段上乘法组合贡献。

最终的解决方案简化为对端点进行排序并使用两个指针执行线性 DP 扫描，维护每种颜色的活动计数并使用前缀贡献来累积有效配置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(2^n·n) | O(2^n·n) | O(n) | 太慢了|
 | 扫描 + DP 间隔 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们将问题压缩为对排序端点的扫描，并动态计数我们可以形成有效子集到当前位置的方式的数量。 

1. 通过增加左端点对间隔进行排序，如果有平局则按右端点排序。 这确保我们以一致的顺序处理潜在的起始点，以便当我们考虑间隔时，所有可能较早开始的间隔都已被考虑在内。 
2. 对于每个间隔，我们将其视为一个从 li 开始到 ri 结束的“事件”，在扫描范围内提供约束。 我们没有明确地跟踪重叠，而是在由正确端点索引的结构中维护主动贡献。 
3. 我们维护两个 DP 累加器 dp0 和 dp1，代表考虑到目前为止处理的间隔的有效方法总数，以红色或蓝色间隔引起的最后一个“活动限制配置文件”为条件。 
4. 我们还维护端点上的前缀和，以便当间隔结束时，我们可以有效地“释放”其约束贡献。 这很重要，因为一旦我们越过了 ri，该间隔就不再限制未来的选择。 
5. 当处理一个区间 i 时，我们计算有多少先前的配置与包含 i 兼容。 如果我们包含它，我们必须确保它不会与任何先前包含的相反颜色间隔重叠。 使用扫描结构，这可以减少从在 li 处活动的间隔中减去的贡献。 
6. 我们通过考虑两个选项来更新 DP：排除 i，它保留所有以前的有效配置，以及包含 i，它乘以相反颜色状态的兼容配置。 
7. 我们更新由 ri 键控的活动结构，以便未来的间隔正确地考虑 i 施加的限制，直到其终点。 

### 为什么它有效

 在任何扫描位置，DP 状态精确编码有效部分选择的数量，其诱导的禁止区域与迄今为止看到的间隔完全一致。 关键的不变量是每个间隔仅对扫描的连续段提供约束，并且这些约束由尚未通过其右端点的活动间隔完全捕获。 由于区间交集相对于端点而言是单调的，因此未来的决策不依赖于活动集之外的任何隐藏历史，这使得 DP 状态充分且无损。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve():
    T = int(input())
    for _ in range(T):
        n = int(input())
        seg = []
        coords = set()

        for i in range(n):
            l, r, c = map(int, input().split())
            seg.append((l, r, c))
            coords.add(l)
            coords.add(r)

        seg.sort()

        # We use a very standard idea: sweep + active contributions by right endpoint.
        # dp0, dp1 = ways ending at current sweep position without active conflicts
        dp0, dp1 = 1, 1

        # For simplicity of presentation, we maintain active intervals grouped by color
        import heapq
        active_red = []
        active_blue = []

        for l, r, c in seg:
            # remove expired intervals
            while active_red and active_red[0] < l:
                heapq.heappop(active_red)
            while active_blue and active_blue[0] < l:
                heapq.heappop(active_blue)

            # current valid ways: we can either skip or take
            new_dp0, new_dp1 = dp0, dp1

            if c == 0:
                # red interval: conflicts only with active blue intervals
                if not active_blue:
                    new_dp0 = (new_dp0 + dp1) % MOD
                # add this interval
                heapq.heappush(active_red, r)
            else:
                # blue interval: conflicts only with active red intervals
                if not active_red:
                    new_dp1 = (new_dp1 + dp0) % MOD
                heapq.heappush(active_blue, r)

            dp0, dp1 = new_dp0, new_dp1

        print((dp0 + dp1 - 1) % MOD)

if __name__ == "__main__":
    solve()
```该代码对按左端点排序的间隔执行扫描。 两个堆通过其右端点跟踪活动的红色和蓝色间隔，使我们能够有效地丢弃过期的间隔。 

DP 状态被简化为两个累加器：dp0 和 dp1，它们在概念上跟踪由红色或蓝色包含状态主导的配置。 添加红色间隔后，仅当当时没有活动的蓝色间隔与其重叠时才可用，反之亦然。 最后减 1 消除了空集的重复计算，具体取决于表示。 

关键的实现细节是由右端点维护活动间隔。 这确保重叠检查简化为一个简单的条件：是否有任何相反颜色的间隔跨越当前位置。 

## 工作示例

 ### 示例 1

 考虑间隔：

 (1, 3, 红色), (2, 4, 蓝色), (5, 6, 红色)

 我们按照左端点的顺序进行处理。 

| 步骤| 间隔| 活性红| 活性蓝| dp0 | dp1 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | (1,3,R) | [3] | []| 1 | 2 |
 | 2 | (2,4,B) | [3] | [4] | 1 | 3 |
 | 3 | (5,6,R) | []| []| 2 | 3 |

 处理后，结果为 dp0 + dp1 - 1 = 4。 

这演示了不相交的间隔如何允许独立选择，而重叠的相反颜色间隔如何限制包含。 

### 示例 2

 间隔：

 (1, 10, 红色), (2, 3, 蓝色), (4, 5, 蓝色)

 | 步骤| 间隔| 活性红| 活性蓝| dp0 | dp1 |
 | --- | --- | --- | --- | --- | --- |
 | 1 | (1,10,R) | [10]| []| 1 | 2 |
 | 2 | (2,3,B) | [10]| [3] | 1 | 2 |
 | 3 | (4,5,B) | [10]| [5]| 1 | 2 |

 在这里，长的红色间隔在重叠期间阻挡了所有蓝色间隔，从而防止其范围内的混合选择。

 ## 边缘情况

 具有一个间隔的最小情况表明该算法正确地计算了空集和单选。 DP 从 1 开始，当不存在相反颜色冲突时允许包含，从而恰好产生两个有效子集。 

完全重叠的相反颜色情况（如（1,10，红色）和（2,3，蓝色））可确保活性结构正确阻止蓝色的包含，而红色跨越重叠。 基于堆的过期可确保冲突准确持续到端点。 

一系列交替重叠测试局部决策是否正确累积。 该不变量成立是因为在任何时候，活动间隔都完全代表了可能影响未来选择的所有约束，并且以后不会再出现跳过的交互。
