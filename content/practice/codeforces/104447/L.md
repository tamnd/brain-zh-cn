 ## 解决方案
 ## 问题理解

 我们得到了一系列独立的测试用例。 每个测试用例都包含一个整数，代表学生的原始分数（从 0 到 10）。 任务是根据一个简单的规则集来转换这个分数，该规则集模拟了一位老师，该老师将及格或非零提交的成绩升级为满分，但拒绝对完全空白的提交进行升级。 

对于每个测试用例，如果原始分数严格大于零，则输出变为 10。如果分数恰好为零，则输出保持为 0。 

限制非常小。 最多有 11 个测试用例，每个分数在 0 到 10 之间。这立即排除了对复杂数据结构或优化的任何需要。 任何在恒定时间内处理每个测试用例的解决方案就足够了。 

唯一微妙的边缘情况是零值和正值之间的边界。 一个幼稚的错误是将零视为升级有效，或者错误地假设所有输入都应映射到 10，无论值如何。 例如，输入`0`必须保留`0`，同时输入`1`必须成为`10`。 另一个可能的错误是使规则过于复杂并意外引入逐一逻辑，例如检查`n >= 0`，这也会错误地升级零。 

## 方法

 对问题的强力解释可能会尝试使用多个条件分支显式模拟评分逻辑，甚至存储从 0 到 10 的所有可能值的映射表。这可以正常工作，因为输入域很小，我们可以简单地为每个可能的分数预定义输出。 

然而，这种方法是不必要的。 问题的结构表明只有一个条件很重要：该值是否为零。 蛮力方法独立处理每个值，但我们可以将所有非零情况压缩为单个结果，因为它们的行为相同。 这一观察结果将问题简化为单个条件检查。 

关键的见解是整个转换是一个二元分类：零映射到零，其他所有映射到十。 一旦认识到这一点，解决方案就变成每个测试用例的恒定时间。

 ## 算法演练

 1.读取测试用例的数量。 这决定了必须应用多少个独立变换。 
2. 对于每个测试用例，读取整数分数。 
3. 检查分数是否为零。 这是唯一有意义的决策点，因为所有正值的行为都是相同的。 
4. 如果分数为零，则输出零。 否则，输出十。 

每个决策都是其测试用例的本地决策，因此无需在迭代中携带任何状态。 

### 为什么它有效

 正确性取决于以下事实：转换仅取决于输入域的二元素分区中的成员资格：包含零的单例集合，以及范围内所有正整数的集合。 由于所有非零输入都映射到相同的输出，因此无需区分它们。 因此，该算法通过应用精确的规则定义来保持正确性，而无需超出规则本身允许的近似或聚合。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        if n > 0:
            print(10)
        else:
            print(0)

if __name__ == "__main__":
    solve()
```实现直接遵循算法。 使用快速 I/O 处理输入，因为这是竞争性编程中的标准做法，尽管此处的约束并不要求这样做。 

有条件的`if n > 0`是解决方案的核心。 它故意避免写作`if n != 0`或者`if n >= 0`因为后者会错误地将零映射到十。 两个输出的分离是明确的，避免了任何可能引入错误的隐藏算术技巧。

 ## 边缘情况

 唯一有意义的边缘情况是零值本身。 用于输入`0`，算法评估条件`n > 0`作为错误和正确的输出`0`。 正确的轨迹是：

 | 步骤| n | 条件 (n > 0) | 输出|
 | --- | --- | --- | --- |
 | 阅读 | 0 | - | - |
 | 评价| 0 | 假 | 0 |

 这证实了零永远不会升级。 

对于最小正值，例如`1`，条件评估为真，产生`10`，并且无论大小如何，这种行为在所有正输入中都是一致的。
