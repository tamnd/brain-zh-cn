 ## 解决方案
 ## 问题理解

 我们得到了多个独立的测试用例。 在每个测试用例中，都有一个代表 Kaito 朋友的整数列表和一个目标值 x。 我们想要选择一组朋友，使得所有选定值的按位与恰好是 x，并且在所有有效组中我们想要最大可能的大小。 如果不可能形成任何 AND 等于 x 的群，我们输出 -1。 

关键操作是对所有选定的数字进行按位与。 如果在每个选定的数字中设置了该位，则此操作仅保留该位设置。 所以最终的结果是子集中所有二进制表示的交集。 

约束条件很大：测试用例总数最多为 5 × 10^5。 这立即排除了尝试所有子集甚至尝试所有对或三元组的任何方法。 任何比每个测试用例的线性更差的东西都会失败。 

当 x 包含某些选定数字中不存在的位时，就会出现微妙的失败情况。 例如，如果 x = 6 (1102)，但我们选择像 1 (0012) 这样的数字，则 AND 会立即丢失所需的位，并且永远无法恢复它们。 所以这样的数字永远不可能成为有效子集的一部分。 另一个重要的边缘情况是，所有可能使用的数字仍然共享一些不在 x 中的额外位。 在这种情况下，没有子集可以从 AND 中删除该位，因为删除元素只会减少 AND，并且如果每个元素都设置了该位，则它将始终存活。 

## 方法

 最直接的想法是尝试朋友的每个子集，计算每个子集的按位与，并跟踪结果等于 x 的最大大小。 这是正确的，因为它探索了所有可能性。 问题是子集的数量以 n 为指数，即使 n = 40 这也已经变得不可行，而这里 n 高达 10^5。 

更结构化的观察来自于理解 AND 的行为方式。 一个数字只有在没有遗漏 x 所需的任何位的情况下才能参与有效的子集。 这意味着每个选定的值必须满足 (ai & x) = x。 这会立即将输入过滤到候选池中。 

现在我们只使用这个过滤池。 如果我们把它们全部取走，它们的 AND 就固定了。 如果此 AND 已经等于 x，则采用所有候选者是最佳选择，因为添加更多元素只会使 AND 的位数更小或相等，而绝不会更大。 因此，如果我们已经恰好位于 x 处，则应该保留所有内容。 

如果所有有效候选者的 AND 不等于 x，则所有候选者中至少存在一位为 1，但 x 中为 0。 该位永远无法通过删除某些元素来消除，因为删除不会在不存在的地方引入零。 在这种情况下，根本不存在有效的子集。 

这将问题分解为通过按位过滤和最终 AND 计算对数组进行单次传递。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对所有子集进行暴力破解 | O(2^n·n) | O(2^n·n) | O(1) | O(1) | 太慢了|
 | 过滤+全局AND检查| 每个测试用例 O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们独立处理每个测试用例，并将问题简化为简单的过滤和聚合步骤。 

### 步骤

 1. 读取n和x，以及数组a。 
2. 丢弃每个不满足(ai & x) = x 的值ai。 

此步骤确保我们永远不会在最终 AND 中丢失 x 所需的位。 
3. 如果过滤后没有剩余值，则输出-1。 
4. 计算所有剩余值的按位与。 
5. 如果最后的 AND 等于 x，则输出剩余元素的数量。 
6. 否则输出−1。 

### 为什么它有效

 任何有效子集必须仅由包含 x 所有位的数字组成，因为单个违规会破坏最终 AND 中的 x。 所以过滤不是限制，而是可行性的必要条件。

一旦我们将自己限制在有效数字上，就位包含而言，任何子集的 AND 都只能大于或等于全集的 AND，因为删除元素只能将 1 位变成 0 位。 这意味着完整的过滤集给出了最强的可能的 AND。 如果即使这个最强的 AND 也不匹配 x，则没有子集可以修复丢失或额外的位，因此不存在解决方案。 如果它确实与 x 匹配，则删除元素只会削弱 AND 并远离 x，因此保留所有元素是最佳选择。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, x = map(int, input().split())
        arr = list(map(int, input().split()))

        filtered = []
        for v in arr:
            if (v & x) == x:
                filtered.append(v)

        if not filtered:
            print(-1)
            continue

        cur = filtered[0]
        for v in filtered[1:]:
            cur &= v

        if cur == x:
            print(len(filtered))
        else:
            print(-1)

if __name__ == "__main__":
    solve()
```实现直接反映了推理。 过滤步骤强制执行必要条件，即每个选定的数字必须包含 x 的所有位。 第二个循环计算所有剩余候选者的全局 AND，这代表从任何子集可实现的最强可能 AND。 

一个常见的陷阱是在过滤后试图对子集进行聪明处理，但没有必要。 按位与的单调性质确保要么整个过滤集有效，要么什么都不有效。 

## 工作示例

 ### 示例 1

 输入：

 n = 4，x = 2

 a = [2, 4, 7, 6]

 首先我们通过 (ai & x) = x 进行过滤。 由于 x = 2 (0102)，我们只保留第二位设置的数字。 得到 [2, 7, 6]。 

现在我们一步步计算它们的AND。 

| 步骤| 当前和 |
 | --- | --- |
 | 开始| 2 (010) |
 | 与 7 | 2 (010) |
 | 与 6 | 2 (010) |

 最终结果是2，与x匹配，所以我们输出3。 

这表明保留所有有效候选者可以保留足够的结构来实现目标。 

### 示例 2

 输入：

 n = 3，x = 3

 a = [1, 2, 3]

 使用 (ai & 3) = 3 进行过滤仅保留 3，因为 1 和 2 都丢失了所需的位。 

现在过滤=[3]。 AND 为 3，与 x 匹配，因此答案为 1。 

这表明该算法可以正确处理大多数元素无效的情况。

 ## 边缘情况

 关键的边缘情况是 x = 0 时。在这种情况下，每个数字都会自动满足 (ai & x) = x，因为 x 没有设置位。 过滤后的集合成为整个数组，所有数字的 AND 确定是否存在任何额外的约束。 如果所有数字共享一个公共位，则 AND 不会为零，结果将变为 -1； 否则可以采用所有元素。 

另一个重要的边缘情况是过滤删除所有内容。 例如，如果 x 需要一个没有数字包含的位，则没有元素可以满足 (ai & x) = x。 该算法立即正确输出 -1，而无需尝试任何进一步的计算。 

当过滤后的集合非空但所有元素共享 x 中不存在的额外公共位时，会出现最后一种微妙的情况。 在这种情况下，全局 AND 保留该位，并且由于没有子集可以消除它，因此最终答案正确为 -1。
