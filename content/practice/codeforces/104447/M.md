 ## 解决方案
 ## 问题理解

 我们从无限整数网格上坐标 (0, 0) 处的一枚硬币开始。 时间从 1 开始以离散步长进行。在每一步 i，我们选择任意整数 xi，然后硬币根据步长是奇数还是偶数以受约束的方式移动。 

如果 i 是奇数，则移动会将 xi 添加到两个坐标，因此 (a, b) 变为 (a + xi, b + xi)。 如果 i 是偶数，则移动会将 xi 添加到第一个坐标并从第二个坐标中减去 xi，因此 (a, b) 变为 (a + xi, b − xi)。 任务是使用最少数量的此类移动到达目标点 (n, m)，并输出所选的 xi 值序列。 如果无法完成，我们输出-1。 

约束最多可达 100,000 个测试用例，坐标最多可达 ±10^9。 这立即排除了对大量步骤的任何每次测试模拟或任何依赖于 xi 值迭代搜索的构造。 每个测试必须在恒定或接近恒定的时间内解决。 

一个微妙的点是 xi 的大小和符号不受限制。 这意味着每次移动都是具有自由选择标量的线性变换，因此该结构完全是代数的而不是组合的。 

一个幼稚的错误是将其视为 x 和 y 坐标的独立移动。 例如，尝试分别求解 x 和 y 会失败，因为每一步都会将它们耦合起来。 另一个常见的错误假设是奇偶校验交替会产生不可逆的约束。 事实上，它只是交替一种符号模式。 

朴素推理的一个具体失败案例是首先尝试贪婪地匹配 n。 假设我们尝试通过仅在 x 上累积 x 贡献来达到 (n, m)。 我们很快发现每一步行动也会以关联的方式影响 y，因此独立的贪婪积累被打破。 

## 方法

 这个问题的关键是以更加结构化的形式重写运动。 

设k次移动后的位置为(X,Y)。 每次移动总是将 xi 贡献给 X，而 Y 根据奇偶性在 +xi 和 -xi 之间交替。 所以我们可以分别考虑对 X 和 Y 的贡献：

 对于奇数 i：贡献为 (xi, xi)

 对于偶数 i：贡献为 (xi, −xi)

 所以在 k 次移动之后：

 X = sum(xi 对于所有 i)

 Y = sum(xi 对于奇数 i) − sum(xi 对于偶数 i)

 我们介绍：

 S = 所有 xi 的总和

 O = 奇数 i xi 的总和

 E = 偶数 i xi 的总和

 然后：

 X = S = O + E

 Y = O − E

 求解这些方程得出：

 O = (X + Y) / 2

 E = (X – Y) / 2

 因此任何解的必要条件是 (n + m) 和 (n − m) 都是偶数。 否则，O 和 E 不是整数，因此没有整数序列可以生成目标。 

一旦确定了可行性，我们仍然需要最少的移动次数。 由于每次移动恰好贡献一个 xi，因此我们希望将 O 和 E 表示为整数之和。 最佳策略是最多走两步：

 如果 O 和 E 都不为零，我们可以使用两种移动方式：

 一次奇数指数移动贡献 O

 一次偶数指数变动贡献 E

 如果其中一个为零，但另一个不为零，只有奇偶校验正确对齐（第一个移动必须是奇数），我们仍然可以在一次移动中完成，否则我们需要通过引入补偿类零调整来进行两次移动。 

然而，干净的观察更简单：我们总是可以使用最多 2 次移动来实现任何有效的 (O, E)，并且只有当 Y = X 或 Y = -X 时，即当 m = n 或 m = -n 时，1 次移动才可能。 

所以最优答案是：

 如果 (n + m) 或 (n − m) 为奇数，则输出 -1。 

否则如果 (n, m) 为 (0, 0)，则答案为 0。 

否则，如果 m = n 或 m = −n，则答案为 1。 

否则答案是 2。 

为了构建 2 次移动的序列，我们设置：

 x1=O

 x2 = E

 ## 算法演练

我们将每个测试用例简化为从目标点派生的两个变量的代数。 

1. 计算 S1 = n + m 和 S2 = n − m。 如果 S1 或 S2 为奇数，则返回 -1，因为 O 和 E 的方程组无法生成整数。 这直接来自求解定义贡献的线性系统。 
2. 如果(n, m) 等于(0, 0)，则输出0，因为不需要移动。 
3. 如果m = n，则输出1并选择x1 = n。 这是有效的，因为每个奇数移动都会相等地添加到两个坐标，因此单个步骤可以精确地落在对角线上。 
4. 如果 m = −n，则输出 1 并选择 x1 = n。 在这种情况下，第一步产生 (n, -n)，直接匹配目标。 
5.否则输出2步。 计算 O = (n + m) / 2 和 E = (n − m) / 2。设置 x1 = O 和 x2 = E，独立地构造奇数和偶数贡献。 

### 为什么它有效

 该变换将过程分成两个独立的线性分量，对应于沿网格轴的对称和反对称运动。 每一步都会以固定模式对两个分量产生影响，因此系统简化为求解整数上的 2×2 线性系统。 奇偶条件保证了解的完整性，并且两步构造就足够了，因为我们可以将每个独立分量分配给专用的时间奇偶类。 

没有比最优序列更短的序列可以同时表示这两个约束，除非在退化情况下，其中派生分量之一为零（这恰好对应于一步对角线）。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n, m = map(int, input().split())
        
        if n == 0 and m == 0:
            out.append("0")
            continue
        
        if (n + m) % 2 != 0 or (n - m) % 2 != 0:
            out.append("-1")
            continue
        
        if n == m or n == -m:
            out.append(f"1 {n}")
            continue
        
        o = (n + m) // 2
        e = (n - m) // 2
        out.append(f"2 {o} {e}")
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案读取所有测试用例并独立处理每个测试用例。 奇偶校验直接来自 O 和 E 必须是整数的要求。 一举的情况与目标位于两条对角线之一上的情况完全对应，从而可以通过单次对称或反对称跳跃来到达目标。 否则，分成 O 和 E 可以保证构造的序列准确地再现两个坐标。 

一个常见的实现陷阱是混淆 E 构造中的符号。正确的推导来自求解线性系统，而不是基于坐标行为的猜测。

 ## 边缘情况

 对于原始情况 (0, 0)，算法立即返回 0，而不尝试分解。 这可以避免构造不必要的 xi 值，否则会产生虚假的两步解。 

对于 (n, n) 这样的对角线目标，算法检测到 m = n 并输出单个移动。 例如，(3, 3) 导致 x1 = 3，通过奇数步骤直接生成 (3, 3)。 

对于像 (n, −n) 这样的反对角目标，适用相同的一着逻辑。 例如 (2, −2) 产生 x1 = 2，并且第一步与目标完全匹配。 

对于 (-10, 8) 等一般情况，算法会退回到两次移动，显式计算 O 和 E 并明确地重建轨迹。
