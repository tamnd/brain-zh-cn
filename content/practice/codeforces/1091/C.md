 ## 解决方案
 ## 问题理解

 我们正在放置$n$人们围成一圈，并以固定的跳跃方式反复移动球。 从人做起$1$，我们选择一个步长$k$。 每次传球时，我们都会移动$k$顺时针位置，绕一圈。 最终，由于运动在有限集合上是确定性的，因此该过程返回到人$1$，并且经过停止。 

对于固定的$k$，我们不会访问所有$n$一般人。 我们只访问通过重复添加确定的循环位置子组$k$模数$n$。 访问集恰好是$1$在这个模块化添加下。 对此的答案$k$是返回之前遇到的所有不同索引的总和$1$。 

任务是计算每个有效的$k$，所访问位置的总和，并输出所有不同此类总和的集合。 

约束条件$n \le 10^9$排除任何模拟$k$或者直接遍历所有位置。 甚至迭代所有$k$是不可能的。 因此，关键结构必须来自模运算和循环结构的数论性质。 

一个天真的想法是模拟每个人的行走$k$。 对于固定的$k$，周期长度为$n / \gcd(n, k)$，所以即使是一次模拟也会花费$O(n)$在最坏的情况下。 为所有人做这件事$k$使得复杂度至少$O(n^2)$，这是完全不可行的。 

一种更微妙的故障模式是尝试仅计算周期长度。 知道长度是不够的，因为总和取决于访问的确切残基，而不仅仅是多少。 

另一个边缘问题是对称性：不同的值$k$可以产生相同的循环，因为逐步$k$或通过$n-k$以相反的顺序遍历同一个子群。 正确的解决方案必须避免重复计算这些等效生成器。 

## 方法

 该过程“添加$k$模数$n$” 将圆划分为长度的循环$$\frac{n}{d}, \quad d = \gcd(n, k).$$而不是从以下角度思考$k$，我们将视角切换到$d$，gcd。 每个步长$k$对应于某个除数$d$的$n$，并且共享相同 gcd 的所有步长都会生成相同的访问节点集。 

对于固定除数$d$，轨道$1$是：$$1, 1 + k, 1 + 2k, \dots$$哪个模$n$与步形成算术级数$k$和长度$n/d$。 访问集仅取决于$d$，不是具体的$k$，所以问题简化为枚举除数$n$。 

现在关键观察：固定的访问位置$d$所有数字都完全等于$1 \bmod d$最多$n$，但是“包裹”在由 生成的乘法子群中$k/d$模数$n/d$。 这简化为一个经典事实：循环由每个$d$简化系统中的倍数，总和仅取决于$n$和$d$，不在$k$。 

让$m = n/d$。 周期有长度$m$，访问的值为：$$1, 1 + k, 1 + 2k, \dots, 1 + (m-1)k \pmod n.$$如果我们通过将所有内容除以来标准化$d$，我们将结构简化为置换循环$m$元素，意味着每个除数$d$恰好贡献了一笔不同的总和。 

因此我们只需要迭代所有除数$d$的$n$，计算相应的总和，并收集结果。 

除数枚举为$O(\sqrt{n})$，这甚至对于$10^9$。 不同结果的数量受到除数数量的限制，与语句保证一致。

 ## 算法演练

 1. 迭代所有整数$d$这样$d \mid n$。 我们只考虑除数，因为每个循环结构都是由下式唯一确定的：$\gcd(n, k)$，它必须是除数$n$。 
2. 对于每个除数$d$, 计算$m = n / d$。 这表示由任何步长引起的循环长度，gcd 等于$d$。 
3. 计算本周期的贡献（乐趣值）。 该循环包括$m$不同的位置均匀分布在圆圈周围。 所有访问值的总和简化为仅取决于$n$和$d$，其计算结果为：$$\text{sum}(d) = \frac{m \cdot (2n - (m-1)d)}{2}.$$这来自对整个周期的算术级数模环绕求和。 
4. 将每个计算值存储在一个集合中，以避免因对称除数而产生重复。 
5. 对结果值进行排序并输出。 

### 为什么它有效

 每个步长$k$将圆划分为不相交的循环，其结构仅取决于$d = \gcd(n, k)$。 两种不同的$k$具有相同 gcd 的值会产生相同的访问集，因此它们贡献相同的乐趣值。 相反，不同的除数会产生不同的循环分解，因此每个不同的 fun 值恰好对应于一个除数类。 gcd 类和答案之间的这种双射可确保完整性并避免计数过多。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    res = set()
    
    d = 1
    while d * d <= n:
        if n % d == 0:
            for div in (d, n // d):
                m = n // div
                # sum of arithmetic structure derived from cycle decomposition
                val = m * (2 * n - (m - 1) * div) // 2
                res.add(val)
        d += 1
    
    ans = sorted(res)
    print(*ans)

if __name__ == "__main__":
    solve()
```该代码使用标准平方根方法枚举除数。 对于每个除数，它计算诱导循环长度$m$。 使用的公式编码了该循环配置中所有访问节点的总贡献。 我们将结果插入到一个集合中，因为由于互补除数之间的对称性，多个除数可能会导致相同的值$d$和$n/d$。 

需要最后的排序步骤，因为问题要求增加订单输出。

 ## 边缘情况

 对于$n = 2$，只有两个除数。 该循环要么只访问节点 1，要么访问两个节点，具体取决于$k$。 该算法枚举$d=1,2$并产生两个不同的总和而不重复。 

对于总理$n$， 每一个$k \neq n$gcd 为 1，因此所有非平凡循环都崩溃为单个结构。 除数枚举正确地仅产生两种有意义的配置：完整循环和平凡循环。 

对于高度复合的数字，例如$n = 12$，多个除数产生重叠的循环长度。 集合结构确保重复项被自动删除，反映了不同的$k$值仍然可以产生相同的遍历集。
