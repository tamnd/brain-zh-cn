 ## 解决方案
 ## 问题理解

 我们得到一条由连续线段组成的长一维路线。 每个部分都有长度和地形类型，可以是草、水或熔岩。 Bob 在第一个段之前开始，并希望在最后一个段之后到达远端。 

鲍勃可以以三种模式移动：步行、游泳和飞行。 只允许在草地上行走，只允许在水中游泳，到处都允许飞行。 每种模式每米都有固定的成本：步行速度很慢，但会提供耐力，游泳比步行更快，并且还可以获得耐力，而飞行速度很快，但会消耗耐力。 耐力从零开始，并且永远不会低于零。 

关键难点在于飞行，这是穿越熔岩的唯一途径，也是消耗体力的唯一途径。 因此，决定的不仅仅是速度上的最短路径，还包括何时在安全地形上积累耐力，以便可以通过飞行穿越熔岩部分而不会耗尽耐力。 

输入描述了每个路段的长度及其类型，我们必须计算从开始到结束的最短总时间，同时考虑地形限制和耐力动态。 

约束 n 高达 10^5，段长度高达 10^12，立即排除每米的任何模拟。 任何解决方案都必须将每个片段压缩为聚合推理并在线性时间内处理路线。 任何单独处理仪表的方法在最坏的情况下都需要多达 10^17 次操作，这是不可行的。 

当熔岩段很长并且在建立足够的耐力之前出现时，就会出现微妙的边缘情况。 例如，如果鲍勃很早就遇到了熔岩，他可能会被迫绕道走或在前面的路段来回游泳，以在尝试长途飞行之前积累足够的耐力。 一种仅“积累直到足够的耐力然后飞行”的幼稚贪婪策略会失败，因为耐力取决于先前的选择，而不仅仅是局部积累。 

当草地和水段交替存在时，会出现另一种不明显的情况。 由于步行和游泳都会提供耐力，但速度和限制不同，因此在路段上选择较慢的模式有时是纯粹为了操纵耐力时间的最佳选择，即使它会增加即时旅行成本。 

## 方法

 蛮力观点会将问题视为位置和耐力的状态图。 从沿线的每个点，鲍勃都可以根据地形选择移动模式，从而相应地更新耐力。 飞行引入了全局耦合，因为耐力的变化取决于之前的所有决定。 直接状态空间搜索需要跟踪以米为单位的位置和当前耐力，并考虑可能根据模式增加或减少耐力的转换。 即使按分段压缩，可能的耐力值的数量也会随着总行进距离而增加，并且由于可能来回行走或游泳，因此可以在两个方向上发生转换。 这很快就会导致指数或伪多项式行为。 

关键的观察是，最佳运动从来不需要任意的回溯结构。 向后移动的唯一原因是在尝试飞越熔岩之前将廉价地形（草或水）转化为额外的耐力。 这种向后移动可以理解为“花时间购买耐力”的一种方式，并且在最便宜的可用耐力获得地形部分中执行它总是最佳的，而不是任意混合多个部分。

这使我们能够重新解释该过程，即在处理从左到右的片段后保持当前可用的耐力，同时考虑到通过飞行强制立即消耗耐力的熔岩片段。 当我们遇到熔岩时，一定要保证自己有足够的体力积累； 否则，我们必须以结构化的方式追溯“延长”之前的草/水使用量，以最小的时间成本产生缺失的耐力。 这导致了对先前看到的耐力获得部分的贪婪分配策略。 

我们没有考虑位置和回溯，而是将草和水部分视为耐力增益的储存库，并具有相关的单位时间成本，而熔岩部分则视为强制支出点，其消耗的耐力与长度成正比。 优化决定了我们将多少先前积累的廉价耐力转化为飞行可行性。

 ## 算法演练

 1. 从左到右遍历各个段，维护两个累加器：花费的总时间和当前耐力。 我们还维护了一个代表以前见过的草和水部分的结构，这些部分可以贡献耐力。 
2. 当处理长度为 L 的草段时，我们添加行走时间 5L，并将耐力增加 L。我们将此段存储为“耐力库”，每个耐力单位的有效成本为 5。 
3. 当处理长度为 L 的水段时，我们添加游泳时间 3L 并将耐力增加 L。我们以每耐力单位成本 3 存储该段。 
4.处理长度为L的熔岩段时，我们必须飞行L米，这需要花费L时间并需要L耐力。 如果当前体力足够的话，我们直接从体力中减去L，然后在时间上加上L。 
5. 如果熔岩的耐力不足，我们计算不足 D = L - 耐力。 我们必须从之前储存的草/水贡献中产生额外的耐力。 
6. 为了最大限度地减少额外时间，我们总是首先从最便宜的可用来源“退款”耐力，这意味着我们优先考虑水源耐力（每单位成本 3），然后是草源耐力（每单位成本 5）。 这相当于总是先反向消除最昂贵的体力。 
7. 对于每个单位的耐力，我们将之前的部分转换为飞行可行性，我们用飞行代替其之前的步行/游泳贡献。 这通过减少存储的差旅成本差异来改变净成本，但增加了灵活性； 实际上，我们将其建模为支付额外的升级成本（每米的模式成本差异）。 
8. 我们继续提取体力，直到满足赤字，相应地更新总时间，并确保体力永远不会低于零。 
9. 解决熔岩后，我们将进入下一个部分，并更新耐力池。 

### 为什么它有效

 该算法依赖于单调结构：仅熔岩段需要耐力，并且只有先前处理的非熔岩段才能产生耐力。 由于所有耐力生成部分都是在任何熔岩消耗决策最终确定之前添加的独立贡献，因此我们可以重新排序“分配”耐力来源的方式，而不会影响可行性。 首先总是使用更便宜的耐力来源的贪婪选择确保了每当我们被迫将过去的运动转化为额外的耐力时，我们都会以最少的时间增加来做到这一点。 这防止了混合部分后向运动策略的任何优势，因为任何此类策略都会分解为对早期片段的单位耐力调整，而这些调整是贪婪地进行最佳处理的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    l = list(map(int, input().split()))
    s = input().strip()

    # we store available stamina sources as [cost_per_stamina, remaining_amount]
    # cost per stamina: water=3, grass=5
    import heapq
    heap = []

    stamina = 0
    total_time = 0

    for i in range(n):
        length = l[i]
        t = s[i]

        if t == 'G':
            total_time += 5 * length
            stamina += length
            heapq.heappush(heap, (5, length))

        elif t == 'W':
            total_time += 3 * length
            stamina += length
            heapq.heappush(heap, (3, length))

        else:  # Lava
            need = length - stamina
            total_time += length  # flying cost
            stamina -= length

            if need > 0:
                # pull from cheapest stamina sources first
                temp = []
                while need > 0:
                    cost, amt = heapq.heappop(heap)
                    take = min(amt, need)
                    need -= take
                    amt -= take
                    total_time += (5 - cost) * take
                    if amt > 0:
                        heapq.heappush(temp, (cost, amt))

                for item in temp:
                    heapq.heappush(heap, item)

                stamina = 0

    print(total_time)

if __name__ == "__main__":
    solve()
```该代码维护了一个耐力来源的优先级队列，其关键在于基础运动的成本有多高。 草以每米 5 的成本贡献耐力，水以每米 3 的成本贡献。 遇到熔岩时，飞行直接消耗体力； 如果体力不足，我们会将之前的步行或游泳追溯为飞行，只需支付原模式与飞行的费用差额。 堆确保我们始终首先调整最便宜的耐力来源，从而最大限度地减少额外成本。 

一个微妙的实现细节是，耐力被视为全局计数器，而堆则跟踪分解的贡献。 当我们从堆中消耗时，我们部分消耗了段，因此我们必须重新插入剩余部分。 当仅部分需要某个段来弥补赤字时，这可以避免错误。

 ## 边缘情况

 当熔岩段比当前耐力长但之前的段存在时，就会出现临界边缘情况。 该算法通过部分消耗早期的耐力贡献来解决这个问题。 例如，如果我们有草3、水2和熔岩10，我们只需要升级5个额外的体力单位。 堆确保我们首先转换水源体力，然后再转换草，从而最大限度地减少成本增加。 

另一个边缘情况是连续的熔岩段。 由于每个熔岩段独立消耗体力，因此该算法通过在每次消耗后重新平衡堆来重复确保可行性。 这可以防止过时的耐力假设。 

最后的边缘情况是很长的初始熔岩段，其中不存在先前的耐力。 在这种情况下，赤字等于完整的熔岩长度，并且堆保持未使用，这意味着解决方案正确地退化为等于总距离的纯飞行成本。
