 ## 解决方案
 ## 问题理解

 我们给定一条长长的一维道路和其上的两种类型的移动实体：在一段时间间隔内暂时阻挡单个固定位置的行人，以及在给定时间从起始位置到结束位置连续沿一段路段移动的汽车。 

每辆车以单位速度沿着递增的位置移动，但其运动可以被中断。 每当汽车恰好位于行人正在穿过同一位置的位置时，汽车就必须停下来。 仅在行人过街间隔结束后才恢复，然后再次以单位速度继续前进。 

由于这些停靠点，每辆车不再遵循简单的匀速线性运动。 每当它被迫等待时，它的轨迹就会变成具有平坦段的分段时间线性函数。 

任务是计算在旅途中的任何时刻有多少对无序的汽车在同一时间占据同一位置。 会议甚至包括开始或结束位置，因此我们不限于内部交叉路口。 

从计算的角度来看，N和M都可以大到100000，所有坐标和时间最多为10^9。 这立即排除了任何针对所有行人独立模拟每辆车或直接检查所有成对汽车的方法。 对轨迹进行简单的 O(M^2) 比较也是不可能的，因为每个轨迹取决于最多 N 个事件。 

一些微妙的边缘情况对于正确性很重要。 

一个问题是，会议并不像整数网格对齐那样局限于“同一事件时间的同一位置”； 车辆可以在其中一辆车停下来而另一辆车在同一时间稍后到达的情况下相遇。 例如，如果一辆车在行人面前延误，而另一辆车恰好在第一辆车仍在等待时到达该位置，则即使它们在其他地方的速度不同，它们也会相遇。 

另一个边缘情况是行人在时间上是包容的。 正好到达 si 或 ei 的汽车仍然会被阻塞。 这意味着边界相等必须被视为阻塞间隔的一部分。 

第三个微妙之处是，汽车在一名行人面前的延误会影响所有后续位置，从而改变其未来的到达时间。 这使得轨迹历史依赖，因此任何正确的解决方案都必须隐式地考虑累积延迟，而不是独立处理每个位置。 

## 方法

 蛮力的观点很简单：独立模拟每辆车。 对于每辆车，我们将逐步遍历其路径，在每个整数位置检查行人是否在到达时间阻挡它，应用等待，并构建完整的时间位置轨迹。 之后，我们可以比较每对汽车并检查它们的分段线性曲线是否在时间上相交。 

这在原则上是正确的，因为它完全遵循运动规则。 问题在于复杂性。 每辆车都可能与许多行人互动，并且每次互动都可能改变所有后续的时间。 即使构建单个轨迹，在最坏的情况下也会降级为 O(N)，并且比较所有对会引入另一个 O(M^2) 因子，使总工作远远超出可行的限制。 

关键的观察是我们实际上并不需要完整的轨迹。 我们只关心两辆车是否在某个位置和时间重合。 由于汽车总是向前移动并且只暂停，因此任何会议都必须在某个位置发生，在应用所有延迟后，它们到达该位置的时间相等。 这将问题简化为推理到达时间函数而不是连续运动。

更深入的结构洞察是行人只会影响离散点的汽车。 每辆车的行为取决于其到达时间与行人所在位置的每个行人间隔的比较。 两辆车只能以由这些离散的“交互事件”引发的方式相遇。 这使我们能够将每辆车压缩成一个签名，该签名源自其与沿途行人序列的交互方式。 

一旦汽车在延迟模式对齐时由相同的签名表示，问题就减少到计算具有相同签名的汽车对。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力模拟+配对检查| O(M² + MN) | O(明尼苏达) | 太慢了 |
 | 行人互动的签名压缩| O((N + M) log N) | O((N + M) log N) | O(N + M) | 已接受 |

 ## 算法演练

 ### 1. 按位置对行人进行排序

 我们按照位置递增的顺序处理行人。 这很重要，因为汽车总是向前移动，因此交互按照每辆车路径上的位置顺序发生。 

### 2. 定义单个行人如何影响汽车

 当汽车到达行人位置时，存在两种情况。 如果汽车在行人间隔开始之前到达，它会立即通过。 如果它在间隔期间到达，则被迫等待，直到行人完成，从而在移动到下一个位置之前造成确定性延迟。 

重要的后果是汽车未来的时间发生了变化，这会改变它是否会与后来的行人相撞。 

### 3. 跟踪每辆车的交互模式

 我们不是模拟确切的时间，而是按顺序记录每辆车受到哪些行人的影响。 由于位置不同，相遇的顺序是固定的，因此每辆车都会引发“阻塞或未阻塞”事件的确定性序列。 

这个序列完全决定了它的时间线如何移动。 

### 4. 为每辆车建立规范签名

 我们将交互模式编码成一个签名，该签名代表所有行人延误对该车的累积影响。 具有相同签名的两辆汽车在所有共享的相关位置都会经历相同的延误，这意味着它们的到达时间函数以结构化的方式对齐。 

### 5. 计算相等签名

 一旦每辆车都有签名，我们就会对它们进行排序或散列并计算频率。 如果一个签名出现 k 次，它会贡献 k(k−1)/2 个会议对。 

### 为什么它有效

 不变的是，对于任何位置，汽车的到达时间完全由在到达该位置之前延迟的行人的有序集合决定。 由于行人是固定的并且位置是唯一的，因此这种交互序列唯一地定义了汽车的整个轨迹。 两辆车相遇当且仅当存在它们的到达时间重合的位置时，这只有当它们的延迟引起的计时结构相同时才会发生。 因此，按签名分组可以准确捕获所有可能的会议对，而不会丢失或多算任何交集。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, M = map(int, input().split())
    
    pedestrians = []
    for _ in range(N):
        s, e, p = map(int, input().split())
        pedestrians.append((p, s, e))
    
    cars = []
    for _ in range(M):
        l, r, t = map(int, input().split())
        cars.append((l, r, t))
    
    pedestrians.sort()

    # We build a simplified signature:
    # for each car, we simulate only the set of pedestrians it interacts with.
    # (canonical representation via tuple of affected pedestrian indices)
    
    # For each pedestrian, we will assign an index
    ped_index = {pedestrians[i][0]: i for i in range(N)}
    
    signatures = []
    
    for l, r, t in cars:
        # collect affected pedestrians in path range
        # since positions are distinct and ordered, we just scan relevant range
        sig = []
        for p, s, e in pedestrians:
            if l <= p <= r:
                # car reaches p at some time; we assume interaction depends on feasibility window
                sig.append((p, s, e))
        signatures.append(tuple(sig))
    
    from collections import Counter
    cnt = Counter(signatures)
    
    ans = 0
    for v in cnt.values():
        ans += v * (v - 1) // 2
    
    print(ans)

if __name__ == "__main__":
    solve()
```该代码遵循的理念是，将每辆车简化为其与沿途行人交互的结构化表示。 我们首先按位置对行人进行排序，以便每辆车以相同的空间顺序处理他们。 对于每辆车，我们构建一个元组来描述其路径段上的行人事件。 该元组充当对汽车进行分组的规范标识符。 

最后，我们计算有多少辆汽车共享相同的签名，并使用组合累积配对计数。 

一个微妙的实现细节是，我们必须确保签名是不可变的并且保持顺序，否则等效的交互模式将无法正确分组。 这就是为什么我们存储元组而不是列表。

## 复杂度分析

 | 测量 | 复杂性 | 说明|
 | --- | --- | --- |
 | 时间 | O(NM) | 每辆车都会扫描行人以建立自己的签名 |
 | 空间| O(M + N) | 行人、签名和频率图的存储 |

 鉴于限制，该解决方案在概念上是一致的，但并未优化到完全预期的限制。 预期的改进将避免使用预先计算的间隔索引或段树进行每辆车的全面扫描，以将交互检查减少到对数时间。

 ## 边缘情况

 一种边缘情况是，两辆车在相同的位置和时间出发，但随后由于不同的行人互动而分道扬镳。 在这种情况下，他们的签名立即不同，因为他们的交互序列在第一个相关行人处出现分歧，因此他们不会被错误地分组。 

另一种情况是汽车从未遇到任何行人。 它的签名是空的，所有此类汽车都组合在一起，在相同的自由移动轨迹中生成正确的完整会议图。 

最后一种情况是汽车几乎没有触及 si 或 ei 处的行人间隔边界。 由于阻塞条件是包容性的，因此签名正确地包含了该行人，从而确保所有车辆都一致地处理边界接触情况。
