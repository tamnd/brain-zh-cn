 ## 解决方案
 ## 问题理解

 我们得到一棵具有加权节点和加权边的树。 每个节点包含一定数量的苹果。 每条边都有一个长度。 我们可以选择节点的任何子集，然后我们必须设计树中的封闭游走，从某个节点开始，至少访问每个选定的节点一次，然后返回到起始节点。 计划的得分是从所选节点收集的苹果总数减去步行的总长度。 目标是最大化这个分数。 

关键细节是，一旦固定了一组节点，访问树结构中所有节点的最便宜的可能封闭游走就不是任意的。 在树中，访问一组节点的任何闭合行走必须精确遍历跨越这些节点的最小连通子树，并且该子树中的每条边都精确行走两次，每个方向一次。 这意味着行程成本完全由导出的连接子树中的边长度之和乘以 2 决定。 

输入大小高达一百万个节点，这会立即排除任何尝试枚举节点子集或显式模拟路由的解决方案。 即使是线性或近线性内存访问也必须仔细设计，因为递归深度和常数因子很重要。 这推动我们采用树结构的动态规划公式。 

当贪婪地考虑仅根据苹果数量来选择节点时，会出现一个微妙的问题。 如果连接两个有利可图的区域，贡献较小甚至为负的节点可能仍然值得保留，因为它可以减少跨越结构中对附加边的需求。 相反，选择许多相距较远的正节点可能会变得无利可图，因为连接它们会强制包含遍历两次的昂贵边。 

例如，考虑具有苹果值的三个节点的链`[100, 1, 100]`和边长`1000`连续节点之间。 选择所有节点会产生巨大的边缘成本`2 * 2000 = 4000`而苹果增益是`201`，产生负分。 正确的答案是仅选择一个端点节点。 始终包含正节点的幼稚策略在这里会失败，因为它忽略了连接成本。 

另一个极端情况是所有苹果值都为零时。 正确答案是零，通过什么都不选择来实现。 任何假设必须选择至少一个节点的方法都会错误地产生负值。 

## 方法

 暴力策略将尝试节点的每个子集，计算跨越它们的最小子树，并评估其成本。 即使计算树中的 Steiner 树成本与节点数量呈线性关系，子集的数量也是指数级的，因此即使对于小型实例，这种方法也是不可能的。 

一旦我们确定了根源并思考贡献如何传播，问题的结构就会简化。 如果我们考虑以某个节点为根的连接选择，那么每个子子树要么向上贡献一些有用的东西，要么被完全丢弃。 关键的观察结果是，包含子子树的成本恰好是该子树内的最佳增益减去连接边长度的两倍。 如果该值为负数，则最佳决策是完全排除该子树，因为包含它只会增加旅行成本，而不是增加苹果。 

这将问题转化为树动态规划任务，其中每个节点聚合所有有益子子树的最佳可能贡献。 由于所选结构必须保持连接，因此每个 dp 值代表以该节点为根的最佳连接组件。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 对子集的暴力破解 | O(2^N·N) | O(2^N·N) | O(N) | 太慢了 |
 | 树DP修剪负枝| O(N) | O(N) | 已接受 |

 ## 算法演练

 ### ## 算法演练

 1. 在任意节点处建立树的根，因为树是无向的，并且任何连接的选择在根视图中都有唯一的最顶层节点。 这让我们可以为 DP 定义一个父子结构。 
2. 执行后序遍历，以便在处理节点时，其所有子节点都已被处理并且已知它们的最佳贡献。 
3. 对于每个节点，将其当前值初始化为该节点处的苹果数。 这代表了如果我们只采用这个节点而不连接其他任何东西的增益。 
4. 对于每个子树，计算该子树如果附加的话可以提供多少好处。 子节点贡献其 dp 值减去将其连接到当前节点的边长度的两倍。 如果该值为正，则将其添加到当前节点的值。 否则，完全丢弃子子树。 

减去两倍边的原因是任何包含的子树都必须在闭合行走中在两个方向上遍历该边。 
5. 将这个累加值存储为 dp[node]，表示根树中最高节点是该节点的连通子树的最佳分数。 
6. 最终答案是所有节点上的最大 dp 值，但如果所有值为负数，则返回零也是有效的，因为不选择任何节点会产生零分。 

### 为什么它有效

 每个有效的节点选择都会在原始树中产生一个连接的子树。 如果我们将该子树植根于全局根方向的最高节点，则所有包含的边都会向下进入子子树。 总分分解为节点增益减去两倍边成本，并且一旦其连接边固定，每个子子树就独立贡献。 由于不同子子树之间不共享树边，因此包含或排除每个子树的决策是独立的。 这种独立性保证了本地丢弃负贡献永远不会阻止全局最优解决方案，因为任何包含负子树的最优解决方案都可以通过删除它来改进，而不会破坏父级之上的连接。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    g = [[] for _ in range(n)]
    
    for _ in range(n - 1):
        u, v, w = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append((v, w))
        g[v].append((u, w))

    parent = [-1] * n
    order = []
    stack = [0]
    parent[0] = -2

    while stack:
        u = stack.pop()
        order.append(u)
        for v, w in g[u]:
            if parent[v] == -1:
                parent[v] = u
                stack.append(v)

    dp = [0] * n

    for u in reversed(order):
        dp[u] = a[u]
        for v, w in g[u]:
            if parent[v] == u:
                gain = dp[v] - 2 * w
                if gain > 0:
                    dp[u] += gain

    print(max(0, max(dp)))

if __name__ == "__main__":
    solve()
```该代码首先为树构建一个邻接表。 然后，它使用迭代 DFS 构建遍历顺序以避免递归深度问题，考虑到最多一百万个节点的约束，这一点很重要。 

dp 数组存储以每个节点为根的连通子树的最佳贡献。 在反向遍历过程中，每个节点都以自己的 apple 值开始。 对于每个子树，我们在考虑连接边的往返遍历后评估包含该子子树是否会提高总数。 仅添加积极的贡献，确保不包含有害的子树。 

最终答案是所有 dp 状态中的最佳值，下限为零，以处理每种可能的选择都是有害的情况。 

## 工作示例

 考虑一棵小树，其中节点 1 有 10 个苹果，节点 2 有 5 个苹果，节点 3 有 8 个苹果，边 1-2 的长度为 3，边 2-3 的长度为 4。 

| 节点| 初始 a[u] | 儿童捐款| dp[u] |
 | --- | --- | --- | --- |
 | 3 | 8 | 无 | 8 |
 | 2 | 5 | 8 - 8 = 0（丢弃）| 5 |
 | 1 | 10 | 10 5 - 6 = -1（丢弃）| 10 | 10

 该算法仅选择节点 1，因为附加其他节点并不能补偿边遍历成本。 这演示了局部修剪如何防止不必要的扩展。 

现在考虑一棵星形树，其中中心节点有 1 个苹果，三个叶子各有 100 个苹果，边成本为 1。 

| 节点| 初始 a[u] | 儿童捐款| dp[u] |
 | --- | --- | --- | --- |
 | 叶子| 各 100 | 无 | 100 | 100
 | 中心 | 1 | 100-2 = 98 各 | 295 | 295

 这里所有叶子都被包括在内，因为在支付两次边缘成本后它们的收益仍然为正。 这证实了 DP 正确聚合了多个有益分支。

 ## 边缘情况

 单节点树可以正确处理，因为 DP 使用自己的 apple 值初始化每个节点，并且从不需要子节点。 答案变为该单一值，如果为负则为零。 

与边权重相比，所有中间节点具有较小苹果值的长链被正确修剪，因为在减去两倍边成本后，每个子贡献都变为负值，导致 DP 崩溃为孤立节点。 

许多叶子单独受益的密集星形仍然完全包含在内，因为每个叶子都是独立评估的，并且每个叶子在边缘成本调整后都做出积极贡献，确保所有有利可图的叶子都附着在中心而不受干扰。
