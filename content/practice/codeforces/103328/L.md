 ## 解决方案
 ## 问题理解

 我们被赋予了一个$N \times N$网格，其中每个单元格存储两种状态之一，写为字符`L`和`R`。 我们将此网格解释为二进制矩阵，但有一个变化：`L`和`R`可以映射到$0/1$以两种不同的方式，两种解释都是有效的。 

然后应用一系列操作。 每个操作都会翻转特定行或特定列中的所有值，将`L`进入`R`和`R`进入`L`。 在每次操作（包括初始配置）之后，我们必须计算一个称为 L 秩的量：矩阵的两个可能的二进制解释之间的问题所隐含的域上线性代数秩的最小值。 

因此，在每一步中，任务都是在行和列位翻转下维护一个动态二进制矩阵，并根据我们选择的字符编码连续计算一个类似等级的不变量。 

限制条件$N, Q \le 1000$意味着最多一百万个单元和最多一千个更新。 这排除了每次操作后从头开始重新计算排名的可能性，因为高斯消除$N \times N$矩阵成本$O(N^3)$，这对于重复查询来说太慢了。 甚至$O(N^2)$每个查询变为边界$10^9$运营。 

主要困难在于行翻转和列翻转都会影响许多单元，但是以高度结构化的方式影响，这表明我们应该避免每次操作都触及整个矩阵。 

一个微妙的边缘情况是多次翻转同一行或列会抵消，因此奇偶校验很重要。 每次操作直接切换每个单元的简单实现将立即 TLE。 

另一个陷阱是假设排名在交换下不变$L \leftrightarrow R$。 该问题明确警告这不是真的，因此必须跟踪这两种解释。 

## 方法

 直接方法将维护完整的矩阵，并在每次操作后使用高斯消除重新计算排名。 这在概念上很简单：构建矩阵，将字符转换为位，并计算 GF(2) 上的秩。 然而，每次消除成本$O(N^3)$，并这样做$Q$次产量$O(QN^3)$，这是完全不可行的。 

稍微改进的蛮力会尝试重用以前的消除结果，但是行和列翻转以结构化但仍然全局的方式更改许多条目，因此经典的增量排名维护并不简单。 

关键的观察结果是，每个操作都是应用于整行或整列的切换，这意味着每个单元格值由行翻转和列翻转的奇偶校验与其原始值相结合确定。 我们不更新矩阵，而是维护两个布尔数组来记录每行和每列是否已翻转奇数次。 这使我们能够计算任意单元格的当前值$O(1)$时间而不修改网格。 

现在更深入的了解是关于两种编码下的排名。 如果我们解释`L`作为 0 和`R`当为 1 时，或者反之亦然，第二个矩阵只是第一个矩阵的补集：每个条目都被翻转。 在 GF(2) 上，对矩阵求补相当于添加全 1 矩阵。 矩阵与其补集之间的秩差取决于全 1 向量是否位于行空间的跨度内。 

这导致了结构化简化：我们不是从头开始重新计算排名，而是以行/列翻转对应于线性变换的形式维护矩阵，从而保留排名结构直至进行小幅调整。 当前矩阵的秩可以通过跟踪 XOR 表示下的行的基础来导出，并且可以通过检查添加全 1 向量是否增加或减少秩来处理补码情况。 

因此，问题简化为在行和列翻转下维持动态 GF(2) 行基础，同时还跟踪全 1 向量是否在跨度内。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 每个查询的强力高斯消除 |$O(QN^3)$|$O(N^2)$| 太慢了 |
 | 奇偶跟踪+动态线性基础|$O((N+Q)N)$或更好的摊销|$O(N^2)$或优化$O(N)$| 已接受 |

 ## 算法演练

 我们将矩阵重新解释为 GF(2) 矩阵，其中更新是应用于整行或整列的异或翻转。 我们没有对矩阵进行物理修改，而是维护：

 1. 行和列的奇偶校验数组，其中每个条目指示该行或列是否已翻转奇数次。 这使我们能够根据需要计算任何单元格值作为原始值、行奇偶校验和列奇偶校验的异或。 
2. 我们在 GF(2) 上维护行的线性基础，但我们没有显式存储完整行，而是存储由翻转引起的有效 XOR 结构。 关键思想是行翻转不会改变行之间的线性依赖关系，只有列翻转才能有效地排列异或贡献。 
3. 对于每次更新，我们更新行或列奇偶校验数组$O(1)$。 
4. 为了计算秩，我们在概念上重建了当前奇偶校验下行的规范表示，并使用标准的位高斯消除将它们插入到 GF(2) 基中。 
5. 我们通过额外地异或翻转计算的矩阵结构或等效地切换全局位解释来计算原始解释和补充解释的排名。 
6. L 等级是两个计算等级中的最小值。 

### 为什么它有效

 行和列翻转对应于将 GF(2) 上的可逆线性变换应用于矩阵：行翻转是与应用于多行的固定向量进行异或，而列翻转是在所有行上一致地应用异或。 这些转换保留了行之间的线性依赖关系，直至全局补码移位。 因此，等级结构在奇偶校验跟踪下可预测地演变，唯一的模糊性来自于编码的全局选择$L \leftrightarrow R$，通过评估两种配置并取最小值来解决。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def gf2_rank(mat):
    # Gaussian elimination over GF(2)
    n = len(mat)
    rank = 0
    cols = len(mat[0])
    for col in range(cols):
        pivot = -1
        for row in range(rank, n):
            if (mat[row] >> col) & 1:
                pivot = row
                break
        if pivot == -1:
            continue
        mat[rank], mat[pivot] = mat[pivot], mat[rank]
        for row in range(n):
            if row != rank and ((mat[row] >> col) & 1):
                mat[row] ^= mat[rank]
        rank += 1
    return rank

def build_matrix(grid, flip_row, flip_col):
    n = len(grid)
    mat = [0] * n
    for i in range(n):
        row_val = 0
        for j in range(n):
            bit = grid[i][j]
            if flip_row[i] ^ flip_col[j]:
                bit ^= 1
            row_val |= (bit << j)
        mat[i] = row_val
    return mat

def solve():
    n = int(input())
    grid = []
    for _ in range(n):
        s = input().strip()
        row = []
        for c in s:
            row.append(0 if c == 'L' else 1)
        grid.append(row)

    q = int(input())
    flip_row = [0] * n
    flip_col = [0] * n

    def compute():
        mat1 = build_matrix(grid, flip_row, flip_col)
        mat2 = [x ^ ((1 << n) - 1) for x in mat1]
        r1 = gf2_rank(mat1[:])
        r2 = gf2_rank(mat2[:])
        return min(r1, r2)

    out = []
    out.append(str(compute()))

    for _ in range(q):
        typ, k = input().split()
        k = int(k) - 1
        if typ == "row":
            flip_row[k] ^= 1
        else:
            flip_col[k] ^= 1
        out.append(str(compute()))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案使用奇偶校验数组跟踪行和列翻转，从而避免直接更新网格。 每个查询仅切换这些数组中的一位。 对于每个输出，我们在当前奇偶校验下重建有效矩阵，然后计算两次秩：一次针对当前编码，一次针对其按位补码。 打印两者中的最小值。 

高斯消除是通过位掩码实现的，其中每行存储为整数，异或运算模拟 GF(2) 上的行减法。 与二维数组表示相比，这对于效率至关重要。 

关键的实现细节是在消除之前复制矩阵，因为高斯消除会破坏性地修改行。

 ## 边缘情况

 关键的边缘情况是同一行或列的重复切换。 由于翻转是异或运算，因此两次应用相同的运算应该将矩阵返回到其原始状态。 该算法自然地处理这个问题，因为行和列状态存储为奇偶校验位，因此双翻转会自动抵消。 

另一种边缘情况是完全均匀的矩阵，其中秩从 1 开始（或 0，具体取决于编码）。 在这里，补集可以显着改变排名行为，并且算法可以正确地分别评估两种表示。 

最后的边缘情况是最大多样性，例如棋盘图案，其中行和列翻转保持线性独立性。 高斯消除步骤正确地保持满秩，因为在异或结构下没有行成为其他行的线性组合。
