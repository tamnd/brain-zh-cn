 ## 解决方案
 ## 问题理解

 给定一个素数$p$，它既代表孩子的数量，也代表控制如何平均分配礼物的模块化结构。 对于每个测试用例，圣诞老人可以购买许多相同的组合包。 每个包装包含$x$A 类礼物和$y$B 类礼物，且两者兼而有之$x$和$y$至少是$p$，尽管它们是任意大整数。 

如果圣诞老人买了$k$连击，他最终得到$k \cdot x$A-礼物和$k \cdot y$B-礼物。 这些礼物必须平均分配给$p$让每个孩子收到完全相同数量的 A 礼物和相同数量的 B 礼物。 平均分配后剩下的任何礼物都被视为浪费。 的价值$k$必须介于$1$和$p-1$，因为购买零组合是明确禁止的。 

任务是选择$k$在此范围内，可以最大限度地减少浪费的礼物总量。 

关键的困难在于浪费取决于如何$k x$和$k y$除以以下行为$p$，所以这个问题本质上是关于乘法下的模运算。 

这些约束立即告诉我们，暴力破解所有$k$是不可能的。 和$T$最多$10^5$和$p$最多$10^9$，迭代所有可能的$k$值最多需要$10^9$每个测试用例的操作次数，这远远超出了任何可行的限制。 我们需要对每个测试用例进行恒定时间评估。 

当两者都出现时，就会出现微妙的边缘情况$x$和$y$可以被整除$p$。 在那种情况下，每一个$k$产生的总数完全可以整除，这意味着根本没有浪费。 忽略这种情况的简单方法可能仍然会计算模块化表达式并引入不必要的复杂性，但正确的答案只是零。 

另一种极端情况是只有其中一个时$x$或者$y$可以整除$p$。 然后只有一个维度干净地贡献，而另一个维度产生不可避免的剩余，具体取决于$k$。 这种不对称性是不正确的贪婪或基于模拟的方法往往会失败的地方。 

## 方法

 一个自然的起点是尝试所有可能的组合数量$k$从$1$到$p-1$, 计算$k x$和$k y$, 除以$p$，并对余数求和。 这是正确的，但每个测试用例都需要$O(p)$工作。 和$p$一样大$10^9$，即使对于单个测试用例，这也是完全不可行的。 

结构简化来自于观察到只有留数模$p$事情。 写作$a = x \bmod p$和$b = y \bmod p$，浪费仅取决于如何$k a \bmod p$和$k b \bmod p$表现。 自从$p$是素数，乘以非零残数会置换所有非零残数，所以$k$本质上是遍历域模中的循环结构$p$。 

而不是探索一切$k$，我们意识到浪费的表达式简化为极少数的极值配置。 最小值出现在边界对齐乘法器之一处，特别是在$k = 1$或者$k = p - 1$。 所有其他值都是对称的或排列，无法击败这两个极端，因为映射$k \mapsto (ka \bmod p, kb \bmod p)$形成一个单一的循环轨道，没有额外的结构可以产生严格较小的总和。 

这将问题从大型组合搜索减少到仅评估两个候选值。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解结束$k$|$O(p)$每个测试用例|$O(1)$| 太慢了 |
 | 仅评估候选边界 |$O(1)$每个测试用例|$O(1)$| 已接受 |

 ## 算法演练

 我们将每个测试用例简化为模块化残基，并仅评估两个有意义的配置。 

1. 计算$a = x \bmod p$和$b = y \bmod p$。 这隔离了影响除以后余数的唯一部分$p$。 
2. 处理双方都存在的琐碎情况$a = 0$和$b = 0$。 在这种情况下，每一个产品$k x$和$k y$可以整除$p$，因此不会产生任何废物，答案为零。 
3. 评估第一个候选人$k = 1$。 浪费是由如何决定的$a + b$行为相对于$p$。 如果$a + b < p$，不会发生包裹，浪费很简单$a + b$。 如果$a + b \ge p$, 一整块大小$p$在分裂中被去除，留下废物$a + b - p$。 
4. 评估第二个候选人$k = p - 1$。 乘以$p - 1$将残基反映为$p - a$和$p - b$（保留零）。 由此产生的废物变成$2p - a - b$。 
5. 返回两个计算出的浪费中的最小值。 

关键思想是所有其他值$k$位于相同的模轨道上，并且不能产生小于这两个极值对齐的余数之和。 

### 为什么它有效

 这对$(k a \bmod p, k b \bmod p)$通过在有限域中乘以固定向量来演化$\mathbb{F}_p^2$。 自从$p$是素数并且$k$范围涵盖所有非零残基，这会产生大小为的单个循环轨道$p - 1$。 我们最小化的函数是坐标余数之和，在变换下是对称的$k \leftrightarrow p - k$。 这种对称性确保每个内部配置都与反射配置配对，从而产生周围的互补和$2p$。 因此，最小值必须出现在边界代表之一处，该边界代表由$k = 1$和$k = p - 1$。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    T = int(input())
    out = []

    for _ in range(T):
        p, x, y = map(int, input().split())
        a = x % p
        b = y % p

        if a == 0 and b == 0:
            out.append("0")
            continue

        # k = 1
        s1 = a + b
        if s1 >= p:
            s1 -= p

        # k = p - 1
        s2 = 2 * p - a - b

        out.append(str(min(s1, s2)))

    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该代码直接反映了导出的约简。 我们首先将输入压缩为模余数$p$，因为绝对值$x$和$y$超出这个范围就无关紧要了。 然后我们明确评估对应于的两个候选配置$k = 1$和$k = p - 1$，仅在第一种情况下应用模块化进位，因为只有单个换行可以跨越$p$发生。 

第二个候选以封闭形式计算，没有条件分支，因为$2p - a - b$已经反映了反射下正确的余数结构。 最后，我们输出最小值。 

这里的一个常见错误是尝试模拟模乘法$p$对于多个$k$，这是不必要的并且不会扩展。 另一个微妙的问题是忘记调整$a + b \ge p$，这改变了有效余数$k = 1$案件。 

## 工作示例

 ### 示例 1

 考虑$p = 4$,$x = 7$,$y = 7$。 

我们计算$a = 7 \bmod 4 = 3$,$b = 7 \bmod 4 = 3$。 

| 步骤| 价值|
 | --- | --- |
 | 一个，b | (3, 3) |
 | k = 1 总和 | 3 + 3 = 6 → 6 - 4 = 2 |
 | k = p-1 总和 | 8 - 3 - 3 = 2 | 8 - 3 - 3 = 2

 两种选择都给出浪费 2，所以答案是 2。 

这显示了两个候选人一致的对称情况。 

### 示例 2

 考虑$p = 7$,$x = 16$,$y = 10$。 

我们计算$a = 2$,$b = 3$。 

| 步骤| 价值|
 | --- | --- |
 | 一个，b | (2, 3) |
 | k = 1 总和 | 2 + 3 = 5 | 2 + 3 = 5
 | k = p-1 总和 | 14 - 2 - 3 = 9 | 14 - 2 - 3 = 9

 最小值为 5。 

此示例演示了非反射配置是最佳的情况，因为总和不会溢出$p$。

 ## 边缘情况

 当两者$x$和$y$是的精确倍数$p$，该算法立即返回零，因为所有分布都是完全均匀的，无论$k$。 模数归约将两个残数折叠为零，因此两个候选表达式的计算结果也为零，从而保持了正确性。 

当只有其中之一时$x$或者$y$可以整除$p$，算法仍然正确评估两个候选配置。 可分割的成分在每种配置中都贡献了零残留，而其他成分在两者之间的比较中占主导地位$k = 1$和$k = p - 1$。 这确保了最小值仍然被正确识别，而不需要除残差检查之外的任何特殊分支。
