 ## 解决方案
 ## 问题理解

 我们得到一个简单的无向图，它保证是一棵仙人掌，这意味着每条边最多属于一个简单的循环。 任务是判断这个图是否是完美图。 

这里的完美不是我们可以直接从定义中计算出来的，因为检查所有导出的子图并将团数与色数进行比较是不可行的。 相反，我们依赖于强完美图定理，该定理将完美图描述为那些没有诱导长度至少为 5 的奇数循环并且没有其补集的图。 

因此，问题归结为仙人掌的结构问题：它是否包含任何长度至少为 5 的诱导奇数环，或任何诱导奇数反洞。 

仙人掌约束是关键的简化。 由于边缘最多属于一个循环，因此循环被完全分离并且不会以复杂的方式重叠。 这意味着任何非树行为都局限于简单的循环，我们可以通过树附件仅进行轻微的交互来独立地推理每个循环。 

输入大小高达 100000 个顶点和边，因此任何解决方案都必须接近线性。 任何带有重常数的二次甚至 n log n 的东西都可以，但是任何试图枚举子图、天真的循环或模拟着色的东西都太慢了。 

一个微妙的点是，并不是仙人掌中的每个循环都会自动成为图中的诱导循环。 如果一个循环有弦，那么它就不是仙人掌中的简单循环，所以我们在那里是安全的。 然而，主要的陷阱是通过树边对循环的附加不会影响循环本身的诱导子图结构，因此每个循环都可以被视为诱导的。 

当图是一棵树、只有一个循环或多个循环共享连接点时，就会出现边缘情况。 例如，一棵树总是完美的，因为它是二分的。 单个 5 周期并不完美，但 4 周期是完美的。 补条件更加微妙，但对于仙人掌来说，任何违规都直接来自长度至少为 5 的奇数循环的存在。 

尝试运行通用完美图形识别算法的简单方法将会失败，因为这些算法在仙人掌约束下要复杂得多且不必要。 

## 方法

 蛮力视角从定义开始：对于每个归纳子图，我们需要比较派系数和色数。 即使我们使用强完美图定理来限制自己，我们仍然需要检测诱导奇循环和诱导奇反洞。 

对于一般图，检测所有长度及其补集的诱导环是昂贵的。 在最坏的情况下，枚举所有简单的循环已经花费了指数时间。 即使在仙人掌中，列出周期也是可以管理的，但直接检查反漏洞则不然。 

关键的观察结果是仙人掌结构解决了这个问题。 除关节点外，每个循环都是孤立的，因此图中出现的任何诱导循环正是仙人掌中的简单循环之一。 此外，诱导奇反孔不能出现在仙人掌中，除非它们对应于非常小的结构情况，并且这些在简化后减少到相同的循环条件。 

这意味着问题简化为扫描仙人掌中的所有简单循环并检查长度至少为 5 的任何循环是否为奇数。 

一旦我们接受这种减少，任务就变得简单了：使用 DFS 找到仙人掌中的所有循环长度，计算它们的奇偶校验，并检查是否存在任何禁止循环。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力诱导子图检查 | 指数| O(n) | 太慢了 |
 | 仙人掌中的循环提取+奇偶校验| O(n) | O(n) | 已接受 |

 ## 算法演练

我们将仙人掌扎根到任何地方，并执行 DFS 来检测形成循环的后边缘。 由于图是一棵仙人掌，每条边最多属于一个环，因此每条后边唯一确定一个环。 

1. 运行 DFS，同时维护每个顶点的父指针和深度。 这使我们能够重建通过后沿遇到的任何循环。 
2. 当我们遇到从节点 u 到祖先 v 的后边时，我们通过从 u 返回到父指针直到 v 来重建循环。循环长度是本次回溯中访问的顶点数加上闭边的 1。 这很干净，因为图表没有重叠的循环。 
3. 对于每个检测到的周期，计算其长度。 如果长度至少为 5 并且为奇数，我们立即得出结论该图并不完美。 
4. 如果在探索完所有边之后不存在这样的循环，我们就得出结论该图是完美的。 

重建步骤是安全的，因为在仙人掌中，DFS树确保任何非树边恰好对应于一个简单循环，并且循环结构不存在歧义。 

### 为什么它有效

 在仙人掌中，每个循环都是孤立的，因为它不与其他循环共享边缘。 因此，通过 DFS 后沿检测到的任何循环都与图中的唯一简单循环完全对应。 由于保留了循环上的导出子图（不存在额外的和弦），因此违反完美性的唯一方法是包含长度至少为 5 的奇数循环，该奇数循环可通过扫描循环长度直接检测到。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n, m = map(int, input().split())
g = [[] for _ in range(n)]
for _ in range(m):
    u, v = map(int, input().split())
    u -= 1
    v -= 1
    g[u].append(v)
    g[v].append(u)

parent = [-1] * n
depth = [0] * n
vis = [0] * n

bad = False

def dfs(u, p):
    global bad
    vis[u] = 1
    for v in g[u]:
        if v == p:
            continue
        if not vis[v]:
            parent[v] = u
            depth[v] = depth[u] + 1
            dfs(v, u)
        else:
            if depth[v] < depth[u]:
                length = 1
                cur = u
                while cur != v:
                    length += 1
                    cur = parent[cur]
                if length >= 5 and length % 2 == 1:
                    bad = True

for i in range(n):
    if not vis[i]:
        dfs(i, -1)

print("No" if bad else "Yes")
```DFS维护了一个父指针树，这样当我们遇到后沿时，我们可以通过向上行走来重建循环。 深度检查确保我们只计算祖先的后边，并避免在无向遍历中重复计算交叉边。 

循环重建在循环长度上是线性的，并且由于仙人掌中的每条边最多属于一个循环，因此所有循环上的总重建工作总体上是线性的。 

一个微妙的实现细节是跳过直接父边缘，否则每个无向边缘都会被错误分类为后边缘。

 | 测量 | 复杂性 | 说明|

 ## 边缘情况

 树输入不包含循环，因此 DFS 永远不会触发后沿重建。 该算法离开`bad`为 false 并正确输出 Yes。 

纯 4 周期恰好触发长度为 4 的一个周期重建。由于条件要求长度至少为 5，因此它不会设置错误标志，这与 4 周期是二分的这一事实相匹配，因此是完美的。 

5 循环是该设置中唯一最小的禁止结构。 当后边缘关闭循环时，重建恰好走了五个节点，并立即将图标记为不完美。
