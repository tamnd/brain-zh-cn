 ## 解决方案
 ## 问题理解

 给定一个素数$P$和一个多重集$S$确切地包含$P-1$正整数。 每个数字都应该被解释为模数$P$，并且我们可以选择任何非空多重集子集（因此我们可以选择具有多重性的元素，考虑到它们出现在$S$）。 

任务是决定我们是否可以选择一些其乘积全等的非空子集$1$模数$P$。 

因为$P$是素数，所有非零元素取模$P$形成一个乘法群。 中的每个元素$S$介于$1$和$10^9$，所以取模后$P$，每个元素都在范围内$[1, P-1]$，意味着所有的模数都是可逆的$P$。 

约束条件$|S| = P-1$是第一个强烈的结构性信号。 大小与乘法群模的大小完全匹配$P$。 这通常暗示了群体完整行为，其中全局组合约束迫使某些产品或取消存在。 

简单的解读表明对子集进行指数搜索。 这是不可能的$P \le 10^5$， 自从$P-1$元素给出$2^{P-1}$子集，这远远超出了任何计算限制。 

重复和值的存在会产生微妙的边缘情况$1$。 例如，如果所有元素都是$2$模数$P$，暴力子集产品仍然可能意外击中$1$取决于顺序，但纯粹根据计数进行推理变得非常重要。 另一种边缘情况是多重集仅包含一个重复元素时$P-1$次。 诸如“如果 1 存在，则回答是”或“如果存在重复项，则回答是”之类的粗心启发式通常会在没有结构合理性的情况下失败。 

关键是问题不是要求任意子集乘积行为，而是要求存在于完整的组大小的多重集中，这迫使对组中的前缀乘积进行鸽子争论。 

## 方法

 暴力方法会尝试每个子集，计算其乘积模$P$，并检查是否有等于$1$。 这是正确的，因为它直接匹配定义。 然而，它需要评估$2^{P-1}$子集，甚至增量计算每个乘积也无济于事，因为状态的指数数量完全占主导地位。 和$P \approx 10^5$，以任何标准来看这都是不可行的。 

关键的观察来自于观察模乘法$P$作为一个有限群。 由于所有元素都是可逆的，因此我们可以根据前缀积来思考。 考虑对多重集进行任意排序并形成前缀积：$$a_1, a_1 a_2, a_1 a_2 a_3, \dots$$每个前缀乘积都是一个值$\{1, 2, \dots, P-1\}$，所以只有$P-1$可能的值。 然而，我们正在生产$P-1$前缀产品。 这立即表明了一种鸽子结构：要么某个前缀乘积等于$1$，或者两个前缀积相等。 

如果某个前缀积等于$1$，那么对应的前缀已经形成一个有效的子集。 

如果两个前缀积相等，则：$$a_1 \cdots a_i \equiv a_1 \cdots a_j \pmod P$$和$i < j$，然后除以：$$a_{i+1} \cdots a_j \equiv 1 \pmod P$$这再次产生一个有效的子集。 

因此，无论排序如何，只要我们考虑所有前缀产品$P-1$元素，必须存在有效​​的子集。 

这将问题简化为确定性结论：对于每个有效输入，答案始终为“是”。 

这种结构并非巧合。 尺寸$P-1$与组大小匹配，并且有限组上的任何该长度的序列都保证在乘法下产生类似零和的子序列。

 ## 算法演练

 该解决方案不需要构造子集； 它只需要确定存在。 

1. 阅读素数$P$和$P-1$多重集的整数。 每个值都可以按模减少$P$，尽管这对于最终决策来说甚至不是必要的，因为结构本身决定了结果。 
2. 认识到我们正在研究以素数为模的乘法群。 多重集中的每个元素都属于一个大小完全相同的集合$P-1$，这意味着我们实际上正在查看一个长度与可能的非零残基数量匹配的序列。 
3.考虑形成整个序列的前缀积。 每个前缀产品保证位于范围内$1$到$P-1$。 
4. 正好有$P-1$前缀（从长度 1 到$P-1$）并且仅$P-1$他们可以采取的可能值。 这会强制前缀等于 1 或重复的前缀值。 
5. 如果某个前缀等于1，我们可以立即选择该前缀作为子集。 如果两个前缀相等，则它们之间的段形成乘积 1 的子集。无论哪种方式，始终存在有效的子集。 
6. 得出结论，答案始终是“是”。 

### 为什么它有效

 核心不变量是前缀产品存在于一组有限的大小中$P-1$，当我们生成$P-1$其中。 在群中，前缀积的相等直接转化为连续子序列的存在，其积是单位元。 由于我们保证通过一个结构化序列超过可能状态的数量，因此必须出现重复或同一性，从而迫使找到有效的解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    P = int(input().strip())
    _ = input().split()
    print("Yes")

if __name__ == "__main__":
    main()
```该实现故意忽略各个值，因为结构参数使得计算变得不必要。 唯一需要的操作是正确使用输入并输出有保证的结果。 

一个常见的错误是尝试模拟子集搜索或计算产品。 这是不必要的，并且会在不改变正确性的情况下引入溢出或低效率的风险。

 ## 边缘情况

 对于单元素情况$P=2$，多重集恰好包含一个数字。 该算法仍然有效，因为前缀乘积是数字本身，而在模 2 算术中，唯一的非零元素是 1，它充当恒等式。 输出保持“是”。 

对于所有元素都相同的情况，例如$S = [2,2,2,2,2,2]$什么时候$P=7$，前缀乘积在乘法群的子群中循环。 由于序列长度为$P-1$，重复是不可避免的，等于1的段积必须出现在两个相等的前缀状态之间。
