 ## 解决方案
 ## 问题理解

 我们得到了最多 20 名玩家之间的有向关系图。 每个玩家都认识一些其他玩家，并且这种知识不是对称的。 游戏从玩家 1 持有令牌开始。 每当玩家持有该令牌时，他们必须将其传递给他们认识的以前从未持有过该令牌的人。 如果他们不能，他们就会立即失败，比赛随即结束。 

关键的变化是，每当玩家有多个有效的选择来传递令牌时，他们就会在所有可用的候选者中统一随机选择。 我们被要求，对于每个玩家 i，计算玩家 i 最终输掉游戏的概率，这意味着令牌到达 i，然后 i 没有有效的下一步行动。 

输入是大小为 n × n 的邻接矩阵，描述图的有向边。 从玩家 i 到玩家 j，值 1 意味着 i 可以将令牌传递给 j，但前提是 j 之前未被访问过。 这使得该过程成为一种自回避随机游走，当到达没有未访问的传出邻居的节点时终止。 

约束非常小，n ≤ 20。这立即表明访问节点子集上的任何指数状态都是可行的，因为 2^20 大约为一百万。 这完全在子集上动态编程的限制范围内，特别是如果转换不太昂贵的话。 

对过程的简单模拟是不够的，因为随机性在每个节点处都有多个未访问的传出边缘分支。 该过程不是单一路径，而是指数级多路径上的概率分布。 

当玩家根本没有传出边，或者所有传出边都通向已经访问过的节点时，就会出现微妙的边缘情况。 在这种情况下，该玩家处于直接失败者状态。 另一个边缘情况是当多个玩家具有相同的输出选择时，这可能会创建必须准确保留的对称概率分裂。 第三个重要的情况是当图形包含循环时； 访问限制会阻止重新访问，因此循环仅与可用的排序约束相关，而不是无限循环。 

## 方法

 暴力解释将模拟每一个可能的有效传递序列，跟踪访问的集合并在玩家有多个选择时进行分支。 每个状态由当前玩家和访问过的玩家集定义。 对于每个状态，我们将概率平均分布在所有有效的传出边缘上。 

这种强力方法在概念上是正确的，但如果以不带记忆的简单递归方式实现，它将以指数方式多次重新计算相同的状态。 即使使用记忆化，状态数量也是 O(n · 2^n)，每个状态最多可以扫描 O(n) 次转换，导致大约 O(n^2 · 2^n)，在 n ≤ 20 的情况下这仍然是可行的。 

关键的观察结果是，该过程完全由配对（当前玩家、访问集）决定。 一旦我们知道了那个状态，未来就与我们如何到达那里无关。 这是子集上的经典马尔可夫决策结构。 因此，我们可以定义一个动态编程函数，返回给定玩家从任何状态开始最终成为失败者的概率，或者等效地向前计算转移概率，直到最终状态并累积失败概率。 

一个更简洁的公式是将 dp[mask][u] 定义为我们当前处于玩家 u 恰好访问过 mask 的概率，然后向前传播概率。 每当 u 没有有效的传出边时，该状态就会直接影响 u 的丢失概率。 

这将问题转化为具有均匀转换的子集状态上的有限动态规划。

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 无备忘录的强力递归 | 指数（比 2^n 条路径超级差）| O(n) 堆栈 | 太慢了 |
 | 位掩码 DP over (mask, u) | O(n^2 · 2^n) | O(n^2 · 2^n) | O(n·2^n) | O(n·2^n) | 已接受 |

 ## 算法演练

 我们将每个状态视为游戏的配置：哪些玩家已经接触过土豆以及哪些玩家当前持有它。 

1.初始化一个DP表，其中dp[mask][u]表示游戏当前处于状态(mask,u)的概率。 我们从 dp[1 << 0][1] = 1 开始，因为最初只有玩家 1 拥有土豆，并且只有玩家 1 被访问。 
2.从小到大迭代所有掩码，确保当我们处理一个状态时，所有导致该状态的状态都已被考虑在内。 这种排序是自然的，因为每次转换都会增加访问的掩码。 
3. 对于每个非零概率的状态（掩码，u），计算有效的下一步动作列表。 如果存在边并且 v 不在掩码中，则移动 u → v 是有效的。 这体现了任何玩家都不能两次收到土豆的规则。 
4.如果有效步数为零，则在此状态下u被迫失败。 我们将 dp[mask][u] 添加到 ans[u] 中，累积 u 是失败玩家的概率。 
5. 如果有 k 个有效的移动，则每个下一个状态 (mask ∪ {v}, v) 接收 dp[mask][u] / k 概率。 这会均匀分布概率，符合随机选择规则。 
6. 处理完所有状态后，数组 ans[u] 包含玩家 u 由于无法移动而结束游戏的总概率。 
7. 将每个概率转换为约简分数 ai / bi 并输出 ai * bi^{-1} mod 1e9+7。 

它的工作原理基于一个简单的不变量：dp 始终表示每个探索阶段所有可到达的游戏状态的精确概率分布。 每个转变都保留总概率质量，因为每个状态将其概率平均分配给所有合法的下一步动作，并且最终状态将概率质量移入相应的失败者桶中。 由于每个有效的游戏序列恰好对应于通过该状态图的一条路径，并且每条路径均由沿其的均匀分支概率的乘积加权，因此节点的累积终端质量等于游戏以该节点无法移动而结束的概率。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

n = int(input())
g = [list(map(int, input().split())) for _ in range(n)]

N = 1 << n

# dp[mask][u] stored as probability numerator in modular form
dp = [[0] * n for _ in range(N)]
dp[1][0] = 1  # only player 0 visited, at node 0

ans = [0] * n

for mask in range(N):
    for u in range(n):
        cur = dp[mask][u]
        if cur == 0:
            continue

        # collect valid moves
        moves = []
        for v in range(n):
            if g[u][v] and not (mask >> v) & 1:
                moves.append(v)

        if not moves:
            ans[u] = (ans[u] + cur) % MOD
            continue

        invk = modinv(len(moves))
        nxt_mask_base = mask

        for v in moves:
            dp[mask | (1 << v)][v] = (dp[mask | (1 << v)][v] + cur * invk) % MOD

# convert probabilities (they are already modular sums)
print(*ans)
```实现直接遵循子集 DP 解释。 dp表存储处于每个状态的概率质量。 当扩展状态时，我们枚举所有有效的传出边并平均分配概率。 

一个微妙的点是掩码必须包含当前节点。 因此，初始掩码为 1 << 0，而不是 1，并且每次转换都会设置下一个节点位。 另一个细节是模除法，它是使用选择数量的模逆来处理的。 

ans 数组累积每个节点的终端概率，对应于丢失事件。 

## 工作示例

 考虑一个对称三角形，其中每个玩家都认识其他每个玩家。 从玩家 1 开始，每一步移动都会在剩余的未访问节点之间均匀分支。 

对于前几个步骤，DP 的演变如下。 

| 面膜| 当前| 有效动作| 转移概率 |
 | --- | --- | --- | --- |
 | {1} | 1 | 2,3 | 1/2 至 (1,2)、1/2 至 (1,3) |
 | {1,2} | 2 | 3 | 1 至 (1,2,3) |
 | {1,3} | 3 | 2 | 1 至 (1,3,2) |

 最终，两条完整路径都在最后一个剩余节点处结束，该节点没有未访问的传出边，因此丢失。 由于对称性，最后两名玩家成为最终失败者的概率相等，因此玩家 2 和 3 的概率各为 1/2。 

该迹线证实了 DP 在分支点正确地分割概率并在终端状态聚合它。 

现在考虑一个没有向后边缘的线性链 1 → 2 → 3。 

| 面膜| 当前| 有效动作| 转移概率 |
 | --- | --- | --- | --- |
 | {1} | 1 | 2 | 1 到 (1,2) |
 | {1,2} | 2 | 3 | 1 至 (1,2,3) |
 | {1,2,3} | 3 | 无 | 3 输 |

 这里玩家 3 失败的概率是 1，因为链条是被迫的。 DP 正确地将所有概率质量路由到玩家 3。

 ## 边缘情况

 关键的边缘情况是起始节点已经没有传出边缘。 在这种情况下，玩家 1 立即成为失败者。 DP 很自然地处理这个问题，因为初始状态没有移动，直接添加到 ans[1] 中。 

另一种情况是节点仅具有到已访问节点的传出边缘。 当访问的掩码很大时，这可能会发生在 DP 深处。 无论原始图结构如何，算法都会正确地将其视为最终状态。 

最后一个微妙的情况是高度对称的图，其中多条路径导致相同的状态。 DP 自然地合并这些贡献，因为概率被累积到相同的 dp[mask][u] 中，确保不会重复计算或丢失质量。
