 ## 问题理解

 给定一个正整数序列和一个固定乘数$x$。 目标是通过添加尽可能少的新正整数来扩展序列，以便最终的多重值集可以完美地划分为对。 每对必须具有严格的乘法关系：如果该对中的一个值是$v$，另一个必须是$v \cdot x$。 

配对在序列内没有排序，这意味着我们可以在配对之前任意重新排列元素。 重要的只是值的多重集以及我们是否可以将每个元素与满足乘法规则的恰好一个伙伴匹配。 

输出是使这样的完整配对成为可能所需的附加元件的最小数量。 

这些约束表明序列可以很大，最多可达$10^5$元素和值可以达到$10^9$。 这排除了任何尝试通过在阵列上重复扫描或嵌套匹配来显式模拟配对形成的方法。 我们需要一种线性或近线性的方法，可能基于频率计数。 

有几种微妙的边缘情况会影响正确性。 

什么时候$x = 1$，每个有效对必须由相同的数字组成。 如果一个值出现奇数次，我们必须添加一个副本使其成为偶数。 尝试匹配不同值的简单方法在这里会失败，因为每个元素只与自身配对。 

什么时候$x = 0$，该规则退化为将数字与零配对。 任何非零数除非与零匹配，否则无法形成有效的对，而零只能与零配对。 这会产生与所有其他情况不同的不对称约束。 

例如，当值形成长乘法链时，会发生另一种微妙的情况$1, x, x^2, x^3$。 如果不按正确的顺序处理值，贪婪的本地配对策略可能会失败，因为过早配对值可能会破坏以后形成有效链的能力。 

## 方法

 暴力方法会重复扫描多重集，选择一个不成对的元素$v$，寻找匹配的$v \cdot x$，并将两者标记为已使用。 如果不存在，我们将添加一个新元素来修复不匹配的情况。 这在概念上是正确的，因为它直接强制执行配对规则，但每次搜索合作伙伴都会花费$O(n)$，我们可以执行这个操作$O(n)$次，导致$O(n^2)$行为。 和$n = 10^5$，这远远超出了可行的限度。 

关键的观察是配对仅取决于频率和固定变换$v \rightarrow v \cdot x$。 这表明我们不应该考虑单个元素，而应该考虑计数以及它们如何在这一转变中流动。 

如果我们按大小对值进行排序并从小到大进行处理，我们可以确保当我们处理一个值时$v$，所有可以配对的潜在较小贡献者$v$通过除以$x$已经解决了。 我们可以贪婪地匹配可用的出现次数$v$与出现$v \cdot x$，相应地减少两个计数。 任何剩余的$v$无法找到合作伙伴的问题必须通过添加新元素来修复$v \cdot x$，因为只有那些可以与之配对。 

微妙的部分是剩余部分沿着乘法链向前传播。 如果一个值$v$仍然无与伦比，它迫使未来的插入$v \cdot x$，并且这些可能会进一步级联。 这种方向依赖性使得按值排序变得至关重要。 

当出现以下情况时需要特殊处理$x = 1$，因为转换不再改变值。 在这种情况下，配对减少到使所有频率均匀。

 | ---| ---| ---| ---|
 | 暴力匹配|$O(n^2)$|$O(n)$| 太慢了 |
 | 频率 + 排序贪心 |$O(n \log n)$|$O(n)$| 已接受 |

 ## 算法演练

 1. 计算序列中每个数字出现的频率。 这将问题减少到管理多重性而不是单个元素，这是必要的，因为配对决策仅取决于计数。 
2.如果$x = 1$，迭代所有频率并添加$f \bmod 2$到答案。 每个奇数计数都需要一个额外的元素来完成相同值内的配对。 
3.如果$x \neq 1$，按升序对所有不同的值进行排序。 排序确保当我们处理一个值时，它产生的任何赤字都可以朝着一致的方向推进。 
4. 按排序顺序遍历值。 对于每个值$v$，如果有剩余频率$f[v] > 0$，尝试将其与$f[v \cdot x]$。 让$m = \min(f[v], f[v \cdot x])$。 将两个频率降低$m$。 这将在此阶段执行所有可能的有效配对。 
5. 匹配后，如果$f[v]$仍然是正数，这些是必须修复的不成对的事件。 每次出现这样的情况都需要添加相应的$v \cdot x$，所以我们将答案增加$f[v]$并且还增加$f[v \cdot x]$经过$f[v]$，然后设置$f[v] = 0$。 
6. 继续，直到处理完所有值。 累加的次数就是最终的答案。 

### 为什么它有效

 该算法保持了值之间所有可配对交互作用的不变性$v$及其前身$v / x$已经解决了，当$v$已处理。 排序确保我们永远不会重新访问之前的状态，并且所有修改仅向前流动$v \rightarrow v \cdot x$。 因为每个元素都有一个可能的合作伙伴值，所以不存在可以改善以后结果的歧义或替代配对选择。 任何赤字$v$除非通过引入新元素才能解决$v \cdot x$，所以立即贪婪地修复它是最佳的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

from collections import defaultdict

def solve():
    n, x = map(int, input().split())
    a = list(map(int, input().split()))

    freq = defaultdict(int)
    for v in a:
        freq[v] += 1

    if x == 1:
        ans = 0
        for v in freq:
            ans += freq[v] % 2
        print(ans)
        return

    vals = sorted(freq.keys())
    ans = 0

    for v in vals:
        if freq[v] == 0:
            continue

        f = freq[v]
        nx = v * x

        match = min(f, freq[nx])
        freq[v] -= match
        freq[nx] -= match

        f = freq[v]
        if f > 0:
            ans += f
            freq[v] = 0
            freq[nx] += f

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案首先将输入压缩为频率图。 这是至关重要的，因为所有操作都是基于多重性的，而不是基于位置的。 特殊情况$x = 1$被单独处理，因为变换图崩溃成自循环，使得一般的贪婪传播无效。 

为了$x \neq 1$，对键进行排序可以确保当我们处理一个值时，所有较小的值都已被最终确定。 匹配步骤删除了之间所有直接可能的对$v$和$v \cdot x$。 任何剩余盈余$v$被迫通过添加新元素来解决$v \cdot x$，我们立即传播该效果。 

一个微妙的实现细节是我们总是就地更新频率图，包括插入$freq[nx]$即使它之前为零。 这可以避免丢失新创建的值必须参与后续匹配的传播链。

 ## 边缘情况

 的$x = 1$这种情况是单独处理的，因为否则算法会尝试将值推入自身并错误地创建无限的自传播。 基于奇偶校验的解决方案直接计算必须插入多少个元素才能使所有频率均匀。 

第二个边缘情况是当值创建长链时，例如$v, vx, vx^2$。 排序处理保证了链中的每个级别在继续之前都已完成，因此赤字不会被重复计算或丢失。 

另一个微妙的情况是当$v \cdot x$超出了现有键的典型范围。 频率图自然地将缺失的条目视为零，因此新的需求仍然可以正确累积，无需特殊处理。
