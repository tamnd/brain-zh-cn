 ## 问题理解

 我们有一排人，每个人要么可能被感染，要么健康，但我们从不直接观察他们的个人状态。 相反，我们收到有关线段的陈述。 

每个类型零的语句描述从 l 到 r 的索引区间，并断言该区间内是否至少有一个感染者，或者该区间是否完全干净。 这些陈述保证与感染者和健康者的至少一项隐藏分配一致。 

与这些语句交织在一起的是类型一的查询，其中我们被问及特定的人 j。 对于每个这样的查询，我们必须确定收集的间隔信息​​是否已经迫使人 j 被感染，迫使他们保持健康，或者保留这两种可能性。 

关键的困难在于信息仅以间隔内的聚合形式给出，因此每个约束都限制了长度为 n 的数组上的一组可能的二进制分配。 查询询问单个位置的值是否在所有有效分配中都是固定的。 

约束 n、q 最多为 2 乘以 10^5，立即排除任何尝试从头开始每个查询维护或重新计算全局一致性的解决方案。 任何重建完整可行分配或按时间间隔执行每个查询传播的方法都会太慢。 

当信息局部矛盾但全局一致时，就会出现微妙的边缘情况。 例如，已知某个段可能包含至少一个感染者，随后另一个重叠段迫使该感染位于该间隔的不同部分。 正确答案取决于所有一致分配是否都同意单个位置，而不取决于是否存在单个重建。 

## 方法

 蛮力的想法是维持长度为 n 的所有有效二进制数组的集合与间隔约束一致。 每次查询后，我们都会检查位置 j 是否在所有查询中都是固定的。 这在概念上是正确的，但在实践中是不可能的，因为有效分配的数量是 n 的指数，并且当约束任意重叠时，甚至隐式表示它们也变得困难。 

看待问题的一种更结构化的方法是将每个约束视为限制一定范围内的可能配置。 如果某个区间不包含感染者，则将该区间内的所有位置固定为零。 一个区间至少包含一名感染者的陈述消除了该区间中所有位置同时为零的可能性。 

关键的见解是扭转观点：我们不是跟踪所有有效数组，而是跟踪哪些位置仍然“未确定”，因为它们在某些有效分配中仍然可以是 0 或 1。 使其可行的结构是“全零”约束是局部分配，而“至少一个”约束仅在已知的强制约束尚未满足时才有意义。 

这导致标准的离线减少。 我们按顺序处理约束和查询，但我们维护一个有效支持两种操作的结构：将范围标记为强制零，并检查某个位置是否由于是某些不满足的“至少一个”间隔的唯一可能见证而被强制为一。 为了支持这一点，我们可以使用线段树来维护每个线段是否完全强制为零，并且我们还维护活动“至少一个”约束的覆盖计数。

基本思想是，如果一个位置需要满足某个间隔，而该间隔的所有其他位置都已被强制为零，则该位置被强制为 1。 这将全局一致性问题转换为间隔内的动态“唯一见证”查询。

 ## 算法演练

 我们维持两个互补的结构。 其中一个跟踪位置被“无感染”间隔强制归零。 另一个跟踪如何仍然满足“至少一次感染”间隔。 

对于每个位置，我们还维护它是否仍然可以用作某个活跃区间的见证人。 

### 步骤

 1.初始化一个长度为n的数组is_zero，最初全部为false，意味着没有位置被强制健康。 
2. 维护一个线段树，支持范围更新将位置标记为强制置零，以及点查询询问位置是否已强制置零。 这确保了我们能够快速消除无法遏制感染的岗位。 
3. 维护所有“[l,r] 中至少有一个感染者”约束的列表或结构。 每个这样的约束都是活动的，并且如果其范围内的所有位置当前被强制为零，则被认为不满足。 
4. 当处理 x = 0 的零类型查询时，我们在线段树中将整个区间 [l, r] 标记为强制零。 这会立即消除该区域的所有潜在感染。 
5. 当处理 x = 1 的零类型查询时，我们将此区间注册为实时约束，要求其中至少有一个非零位置。 这一约束保持相关性，直到至少一个未强制归零的位置满足该约束。 
6. 每次更新后，我们为每个活动的“至少一个”间隔维护是否已经有一个不强制为零的候选位置。 这可以使用线段树查询来跟踪，该查询检查线段是否完全为零。 
7. 对于询问位置 j 的查询类型一，我们检查两个条件。 如果 is_zero[j] 为 true，则 j 肯定是健康的。 否则，我们检查 j 是否是任何仍不满足的活动区间中唯一可能的非零位置。 如果存在一个区间，其中 j 是唯一剩下的候选者，则 j 必定被感染。 
8. 如果强制置零或强制置一条件均不适用，则答案不明确。 

### 为什么它有效

 该算法依赖于以下不变量：每个 x = 0 类型的约束永久消除其区间内的所有感染可能性，并且每个 x = 1 类型的约束必须由其区间内至少一个幸存候选位置支持。 任何作为某些主动约束的唯一剩余候选者的位置在逻辑上都被迫满足该约束。 由于所有约束都是单调的，即删除候选者永远不会使一致性失效，因此一旦某个位置变得唯一必要，它在所有有效的完成中都会保持如此。 这可确保强制分配在所有令人满意的配置中保持一致。
