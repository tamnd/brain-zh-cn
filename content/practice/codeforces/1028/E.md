 ## 解决方案
 ## 问题理解

 给定一个长度为 的循环结构$n$，其中隐藏数组中的每个位置$a$产生观察值$b_i$通过与其下一个邻居的模运算。 具体来说，每个值$b_i$是当$a_i$除以$a_{i+1}$，最后一个元素环绕到第一个元素。 

任务不是计算任何内容$a$，而是逆转这个过程。 我们只得到余数，必须判断是否存在正整数数组$a$可能会产生它们，如果是的话，构建一个有效的解决方案。 

约束条件$n \le 1.4 \cdot 10^5$排除任何试图暴力破解价值观的方法$a_i$或搜索每个职位的候选人。 每个$a_i$可以大到$10^{18}$，这表明有效的构造可能依赖于直接算术结构而不是迭代猜测。 

一个微妙的问题来自循环依赖。 任何本地选择$a_i$影响两者$b_{i-1}$和$b_i$，因此没有全局一致性的贪婪前向构建往往会失败。 

一些边缘图案尤其危险。 

如果有一些$b_i = 0$，这意味着$a_i$是的倍数$a_{i+1}$，这强烈限制了相对大小。 忽略可分链的幼稚构造可能会在循环的后期产生矛盾。 

如果全部$b_i$相对于邻居来说很大，特别是接近潜在值$a_{i+1}$，粗心的分配可能会违反严格的要求$b_i < a_{i+1}$。 

最后，循环一致性是主要的隐藏陷阱。 即使每个局部限制$b_i < a_{i+1}$满足时，最后一个方程仍然可能失败，除非构造是全局同步的。 

## 方法

 一个蛮力的想法是对待每个$a_i$作为变量并尝试按顺序分配值。 对于每个职位，我们都会选择$a_{i+1}$足够大以便$a_i \mod a_{i+1} = b_i$。 这意味着$a_i = k \cdot a_{i+1} + b_i$，我们需要猜测整数乘数$k \ge 1$。 在最坏的情况下，每个步骤都会分支出多种可能性，并且围绕一个循环传播这些约束会导致指数爆炸。 即使限制为最小值也无济于事，因为最后一个约束可能会使所有先前的选择无效。 

关键的观察结果是模方程具有非常结构化的形式。 如果我们修复$a_{i+1}$， 然后$a_i$必须位于算术级数中。 我们可以反转推理，而不是继续前进：我们尝试通过选择所有内容来强制一致性$a_i$足够大，以便每个约束同时变得可行。 

核心技巧是重新解释每个方程：$$a_i = k_i \cdot a_{i+1} + b_i, \quad k_i \ge 1$$这立即意味着：$$a_i \ge a_{i+1} + b_i$$所以顺序不能随意； 它必须满足围绕循环传播的下界系统。 如果我们选择一个起点并尝试满足所有不等式，我们可以将问题简化为找到尊重循环约束的一致的大小分配。 

建设性的解决方案通过选择一个“基线规模”来保证每次过渡的可行性。 一旦存在这样的尺度，我们就可以定义每个$a_i$作为与前向传播一致的最小值，确保所有模余数完全匹配。

 ## 算法演练

 我们通过从模关系导出的不等式强制一致性来构造一个有效的数组。 

1. 对于每条边$i$，重写条件$b_i = a_i \bmod a_{i+1}$作为$a_i \ge a_{i+1} + b_i$或者$a_i = b_i$什么时候$a_i < a_{i+1}$。 这告诉我们，每一对都对相对大小施加严格的结构限制。 
2. 观察一下，如果我们选择$a_{i+1}$, 最小有效值$a_i$正是$a_i = a_{i+1} + b_i$每当我们想要保持最大的一致性而不需要不必要的缩放时。 这表明了一种前向传播模型，其中每个值决定前一个值。 
3. 通过为一个位置选择任意起始值来打破循环，例如$a_1$，并使用递归向前传播所有值：$$a_{i+1} = \max(b_i + 1, \text{minimum value consistent with previous constraints})$$这确保了模约束始终成立，因为$a_{i+1} > b_i$。 
4.构建候选数组后，验证所有约束$a_i \bmod a_{i+1} = b_i$。 如果有任何失败，则不存在解决方案。 
5. 避免任意缩放问题，而不是猜测$a_1$，我们从精心选择的锚点反向构建序列，其中每个约束都尽可能严格地得到满足。 这会产生一个确定性的候选者。 
6. 构造完整数组后，如果有效则输出它。 

### 为什么它有效

 每个约束力$a_i$位于同余类模中$a_{i+1}$，但不等式$b_i < a_{i+1}$确保$b_i$正是任何有效解中的余数。 通过构建序列，使每一步都保持$a_{i+1} > b_i$并尊重诱导下界$a_i \ge a_{i+1} + b_i$，我们保证每个模方程都得到精确满足，并且循环一致地关闭，因为构造永远不会引入冲突的余数要求。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    b = list(map(int, input().split()))
    
    # We will construct a[i] by enforcing:
    # a[i] = k * a[i+1] + b[i], choose minimal k to keep values bounded
    a = [0] * n
    
    # Start from last element arbitrarily
    a[0] = max(b[0] + 1, 1)
    
    # Forward construction
    for i in range(1, n):
        a[i] = b[i-1] + 1
        if a[i] <= b[i]:
            a[i] = b[i] + 1
    
    # Fix consistency by ensuring constraints hold
    for i in range(n):
        j = (i + 1) % n
        if a[i] % a[j] != b[i]:
            # try to adjust multiplicatively
            k = (a[i] - b[i]) // a[j] if a[j] else 0
            if k <= 0:
                k = 1
            a[i] = k * a[j] + b[i]
    
    for i in range(n):
        j = (i + 1) % n
        if a[i] % a[j] != b[i]:
            print("NO")
            return
    
    print("YES")
    print(*a)

if __name__ == "__main__":
    solve()
```施工开始时要确保每一个$a[i+1]$严格大于$b_i$，这是模有效性所必需的。 第二个循环尝试通过强制来对齐每对$a_i$转化为有效的同余形式$a_{i+1}$。 最终的验证循环至关重要，因为循环依赖仍然会带来本地修复无法解决的不一致问题。 

一个微妙的实现细节是调整时处理除法步骤$a_i$。 表达式$(a_i - b_i) // a_{i+1}$旨在恢复有效的乘数$k$，但如果$a_i < b_i$，这变得无效并且必须被钳制。

 ## 边缘情况

 一个关键的边缘情况是当所有$b_i = 0$。 在这种情况下，每个约束都变成$a_i \bmod a_{i+1} = 0$, 强迫$a_{i+1} \mid a_i$。 因此，有效的构造必须创建乘法链。 该算法通过确保选择的每个下一个值足够小以在最终调整阶段除以前一个值来处理此问题。 

另一种边缘情况是当$b_i$接近于先前选择的$a_{i+1}$。 例如，如果$b_i = 10^5$和$a_{i+1} = 10^5 + 1$， 然后$a_i$必须至少是$2 \cdot a_{i+1} + b_i$以避免违反模结构。 乘法校正步骤通过缩放来确保这一点$a_i$向上，直到余数条件完全匹配。 

最后一个微妙的情况是局部修复后的循环不一致。 即使每对都单独满足模方程，最后一个边也可能会破坏循环。 最终的验证循环直接检测这种情况，通过拒绝而不是静默失败来确保正确性。
