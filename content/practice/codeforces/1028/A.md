 ## 解决方案
 ## 问题理解

 我们给出了一个尺寸的网格$n \times m$由两种类型的细胞组成：白色和黑色。 最初，整个网格是白色的，但在某些时候，具有奇数边长的单个正方形区域被漆成黑色。 该正方形是轴对齐的并且完全充满黑色单元格。 那个广场之外，一切都是白色的。 

任务是确定黑色方块的确切中心单元格。 因为边长保证是奇数，所以有一个唯一的中心单元，并且它与正方形的所有四个边的距离恰好相等。 

输入只是绘制后的网格。 输出是黑色区域中心单元的坐标。 

限制很小：$n, m \le 115$。 这立即告诉我们，即使是网格的二次扫描在复杂性方面也是微不足道的。 任何多次检查每个单元的解决方案仍然可以在限制范围内舒适地运行。 该问题的结构也非常严格，因为所有黑色单元都属于一个连续的轴对齐正方形，没有噪声或附加形状。 

主要的微妙之处在于我们没有直接给出正方形，只有其填充的像素。 如果我们不仔细提取黑色区域的确切范围，天真的解释可能会尝试错误地推断几何形状或边界形状。 

一些边缘情况很重要：

 最小正方形的尺寸$1 \times 1$表示网格恰好包含一个黑色单元格。 任何依赖于检测“区域”或“多行”的算法仍然必须正确处理这个问题。 例如：

 输入：```
1 1
B
```输出：```
1 1
```另一种情况是当正方形接触网格边界时。 黑色单元格的边界框可能从第 1 行或第 1 列开始，因此索引中的任何差一错误都会立即破坏正确性。 

最后，因为正方形保证是完美的，所以我们不能假设多个不相连的黑色分量。 任何尝试合并区域或执行洪水填充的解决方案都是不必要的，但在约束下仍然有效。 

## 方法

 直接的暴力策略是扫描网格中每个可能的子方格，检查它是否是全黑的，以及它是否形成唯一的绘制区域。 对于每个候选方格，我们将验证其中的所有单元格，这会花费$O(k^2)$每个候选人，导致总体$O(n^4)$最坏情况下的方法。 虽然$n \le 115$使得这个边界在优化的 C++ 中可行，但在 Python 中却不必要地复杂和缓慢。 

更自然的观察来自输入的结构。 每个黑色单元都属于一个轴对齐的正方形，因此所有黑色单元形成一个连续的矩形区域，其中高度等于宽度。 我们可以找到所有黑色单元格的边界框，而不是直接搜索正方形。 最上面、最下面、最左边和最右边的黑色单元格定义了一个矩形。 由于形状保证是边长为奇数的正方形，因此这个边界矩形已经是我们想要的确切正方形。 

一旦我们知道了正方形的角，中心就是垂直和水平范围的中点。 

蛮力方法浪费精力检查我们已经保证的结构的正确性。 边界框方法将问题简化为网格的单次扫描。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2 m^2)$|$O(1)$| 太慢了|
 | 最佳（边界框扫描）|$O(nm)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 初始化变量以跟踪黑色单元格的极端边界：顶部为非常大的值，底部为非常小的值，左侧和右侧类似。 这确保任何黑色单元格都会正确更新这些边界。 
2. 逐行扫描网格中的每个单元格。 每当遇到黑色单元时，使用其坐标更新四个边界值。 此步骤至关重要，因为正方形可能出现在网格中的任何位置。 
3. 完成扫描后，黑色方块完全被这些边界定义的矩形包围。 
4. 将中心行计算为顶部和底部边界的平均值，将中心列计算为左侧和右侧边界的平均值。 因为正方形边长保证为奇数，所以这些平均值是整数。 
5. 输出计算出的中心坐标。 

### 为什么它有效

 所有黑色单元格都属于一个填充的正方形。 沿每个方向的极端黑色单元必须与该正方形的边缘完全对应，因为超出它们的任何延伸都会与固体填充区域的定义相矛盾。 由于形状是轴对齐且完全填充的，因此所有黑色单元格的最小外接矩形与原始正方形完全重合。 因此，该矩形的中点是正方形的几何中心，并且奇数边长具有唯一性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    
    top, bottom = n, -1
    left, right = m, -1
    
    for i in range(n):
        row = input().strip()
        for j, ch in enumerate(row):
            if ch == 'B':
                if i < top:
                    top = i
                if i > bottom:
                    bottom = i
                if j < left:
                    left = j
                if j > right:
                    right = j
    
    # convert to 1-based indexing
    r = (top + bottom) // 2 + 1
    c = (left + right) // 2 + 1
    
    print(r, c)

if __name__ == "__main__":
    solve()
```该实现在扫描一次网格时保持四个运行极端。 重要的细节是，网格是在内部使用基于 0 的索引读取的，因此仅在最后一步才转换为基于 1 的索引。 中点计算使用整数除法，这是安全的，因为正方形大小保证是奇数。

 | 开始 | - | 5 | -1 | 6 | -1 |
 | 找到 B | (0,2) | 0 | 0 | 2 | 2 |
 | 找到 B | (0,4) | 0 | 0 | 2 | 4 |
 | 找到 B | (1,2) | 0 | 1 | 2 | 4 |
 | 找到 B | (1,4) | 0 | 1 | 2 | 4 |

 最终边界定义了一个从 0 行到 2 行、2 到 4 列的矩形。中心是：

 行 = (0 + 2) / 2 + 1 = 2，列 = (2 + 4) / 2 + 1 = 4。 

输出：```

import sys, io

def run(inp: str) -> str:
    sys.stdin = io.StringIO(inp)
    from collections import deque
    n, m = map(int, input().split())

        row = input().strip()
        for j, ch in enumerate(row):

                bottom = max(bottom, i)
                left = min(left, j)
                right = max(right, j)

 ## 边缘情况

 包含单个黑色单元格的最小网格表明该算法不需要对退化正方形进行特殊处理。 扫描将所有边界设置为该单个坐标，并且中点返回相同的单元格。 

接触网格边界的正方形表明边界的初始化必须在有效索引之外开始。 如果我们从零开始而不是从极端哨兵开始，我们可能会错误地缩小检测到的区域。 

填充大部分网格的大方块测试扫描是否正确聚合多个更新而不覆盖早期的极端值。 每个黑色单元格都会导致边界收紧或扩展，只有全局极端值才会影响最终结果。
