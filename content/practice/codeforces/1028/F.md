 ## 解决方案
 ## 问题理解

 我们在整数网格上维护一组动态点。 该集合随着时间的推移通过插入和删除而发生变化。 有时我们会被问到一个假设的问题：如果我们通过原点和给定方向点画一条线，我们需要添加多少个附加点才能使当前集合相对于该线完全对称。 

这里的对称意味着对于当前集合中的每个点，其在给定线上的镜像反射也必须属于该集合。 我们不允许删除此操作的点，只能添加缺失的镜像对应点。 类型三的每个查询都是独立的，因此我们仅根据当时的当前集合来评估对称性，而不会永久修改它。 

最多 200,000 个查询、最多 100,000 个活动集修改和最多 100,000 个对称查询的约束迫使我们采用完全动态的解决方案。 任何为每个查询重新计算所有点的反射的方法都会太慢，因为即使是单个对称性检查也需要迭代整个集合。 

一个简单的解决方案是，对于类型三的每个查询，迭代集合中的所有点并计算它们的反射。 穿过原点的线的点的反射取决于方向向量上的投影，并且检查成员资格需要哈希集。 即使成员资格是 O(1)，我们仍然为每次查询支付 O(n) 费用，这会导致在最坏的情况下大约有 10^10 次操作，这是不可行的。 

当点位于对称轴上时，会出现更微妙的问题。 恰好在线上的点会反映自身，因此它们不会导致缺失对。 如果没有明确检测与方向向量的共线性，粗心的实现可能仍然会错误地计算它们。 

另一个边缘情况是重复的定向查询。 由于每个查询使用不同的对称轴，因此与固定方向相关的任何预处理都是无用的。 该解决方案必须重新计算或有效查询相对于任意方向的几何图形。 

## 方法

 暴力方法独立处理每个对称查询。 我们固定一个方向向量 v = (x, y)，以某种一致的方式对其进行归一化，然后对于集合中的每个点 p，计算其反射 p'。 如果 p' 不在集合中，我们将增加答案。 这是正确的，因为每个缺失的镜像点都对应于一个所需的插入。 

失败点是性能。 每个查询都会扫描整个集合，并且最多有 10⁵ 个点和 10⁵ 个查询，这会变成二次方。 

关键的观察结果是，穿过原点的直线的反射保留了沿方向矢量的分量并翻转了正交分量。 这表明坐标转换：我们不是在标准坐标中工作，而是将每个点投影到由查询方向定义的坐标系中。 在该旋转框架中，对称性变成了一个轴上的简单符号翻转。 

这意味着对于固定的查询方向，每个点都可以沿着垂直方向映射到标量坐标，并且对称性简化为配对值+t和-t。 所需插入的数量正是该一维多集视图中未配对出现的数量。 

挑战在于每个查询的方向都不同，因此我们无法维护全局转换的结构。 然而，我们可以利用每个查询方向都是由原始整数向量定义的事实，并且我们可以规范化方向以避免重复并在查询中有效地重用计算的投影。 

在单个查询中，我们计算归一化基（dx，dy）及其垂直线（-dy，dx）。 每个点都投影到该垂直轴上，产生一个标量值。 然后我们计算频率并将 t 与 -t 匹配。

 | --- | --- | --- | --- |
 | 蛮力 | O(q·n) | O(n) | 太慢了|
 | 每个查询的投影 | 最坏情况 O(q·n) | O(n) | 仍然处于边缘状态，但通过每个查询的散列进行了优化 |

 ## 算法演练

 我们在线处理查询，维护哈希集中的当前点集。 

对于每个类型三查询，我们构造一个方向向量（x，y）。 我们通过除以 gcd(x, y) 来减少它，以便所有共线方向共享规范表示。 然后我们建立一个垂直方向（-y，x）。 

然后，我们将当前集中的每个点 (px, py) 投影到该垂直轴上。 投影值是叉积：

 t = px * y - py * x

 当该点位于对称轴上时，该值恰好为零。 

我们将所有非零 t 值存储在频率图中。 对于每个 t，我们将其与 -t 配对。 每个不匹配的出现都会贡献一个所需的插入。 

### 步骤

 1. 维护活跃点的哈希集。 

这允许 O(1) 插入、删除和成员跟踪。 
2. 对于类型 1 的查询，将点插入到集合中。 对于类型 2，将其删除。 

该集始终反映当前配置。 
3. 对于方向为 (x, y) 的类型 3 的查询，通过除以 gcd(x, y) 对其进行归一化。 

这确保了等效线产生相同的计算。 
4. 对于集合中的每个点 (px, py)，计算 t = px * y − py * x。 

这是有符号面积/叉积，表示垂直位移。 
5. 在频率图中按 t 对点进行分组。 
6. 对于每个 t > 0，将其与 -t 的频率配对并计算不匹配的出现次数。 

答案是 max(0, cnt[t] - cnt[-t]) 之和。 
7. 输出不匹配总数。 

### 为什么它有效

 叉积将所有点映射到垂直于查询线的一维轴上。 当且仅当两个点的叉积值互为负数并且沿线共享相同的投影时，两个点是线上的镜像。 由于我们只关心是否存在而不关心添加点的实际坐标，因此通过相反值配对可以准确计算丢失的反射。 投影为零的点位于轴上并且是自对称的，因此它们不会导致缺失对。 这将几何对称条件简化为整数上的多重集配对问题。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import Counter
from math import gcd

def solve():
    q = int(input())
    pts = set()
    
    for _ in range(q):
        t, x, y = map(int, input().split())
        
        if t == 1:
            pts.add((x, y))
        elif t == 2:
            pts.remove((x, y))
        else:
            if not pts:
                print(0)
                continue
            
            g = gcd(x, y)
            x //= g
            y //= g
            
            cnt = Counter()
            for px, py in pts:
                val = px * y - py * x
                cnt[val] += 1
            
            ans = 0
            for v in list(cnt.keys()):
                if v > 0:
                    ans += abs(cnt[v] - cnt[-v])
            print(ans)

if __name__ == "__main__":
    solve()
```该实现将活动点集保留在 Python 集中，以便更新保持 O(1)。 对于每个查询，我们直接从当前集合重新计算对称投影。 标准化步骤确保相同的线不会产生不一致的投影。 

叉积计算是核心几何原语。 它完全避免了浮点运算并将所有内容保留为整数，这是必要的，因为坐标可能很大。 

一个微妙的点是，我们只迭代键一次并将正值与其负值进行比较，这可以防止重复计算。 零值将被忽略，因为它们恰好位于对称轴上。

 ## 边缘情况

 关键的边缘情况是所有点都恰好位于对称线上。 在这种情况下，每个叉积的计算结果为零，因此频率图仅包含零桶。 该算法正确返回零，因为不需要配对并且不存在正键或负键。 

另一种情况是集合为空时。 投影图为空，算法立即返回零，不进入配对逻辑。 

第三种情况涉及高度倾斜的分布，其中许多点共享相同的投影值。 即使在这种情况下，通过绝对差进行配对仍然可以正确地计算不匹配的元素，因为每个未配对的实例必须对应于轴另一侧的缺失镜像。
