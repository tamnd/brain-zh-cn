 ## 解决方案
 ## 问题理解

 我们得到了几种管道设计。 每个设计都是同一组建筑物上的一棵加权树。 每根管道连接两座建筑物并具有耐久性值。 

对于任何设计，请考虑任意两座建筑物。 树中它们之间有一条独特的路径。 该连接的“弱点”被定义为沿该路径的最小耐久性。 因此，对于每对节点，我们将其连接路径上的最小边权值关联起来。 

如果每对节点的成对“最小路径”值相同，则认为两种设计是等效的。 对于每个设计，任务是在输入中找到与其等效的最早的设计。 

所以问题不在于直接比较边缘。 它是关于比较由加权树导出的所有节点对上导出的完整矩阵。 

输入约束很大：所有设计的边总数与总输入大小呈线性关系，最多 500,000 个。 这立即排除了任何显式计算所有成对值的解决方案，因为每棵树都需要 O(n^2) 比较，并且会爆炸。 

不明显的困难是，两棵不同的树在结构上可能看起来非常不同，但仍然会产生相同的成对最小路径边值。 

暴露幼稚思维的一个关键边缘情况是：

 权重为 5 和 1 的链 1-2-3 产生的成对结构与交换边的同一链不同，但具有精心选择权重的星形可以为所有对重现相同的成对最小值，尽管具有完全不同的结构。 因此，比较邻接表或边多重集是不够的。 

## 方法

 暴力解释将通过运行 DFS 或 LCA 查询来计算每棵树的每对 i、j 的值 v(i, j)。 每棵树的成本为 O(n^2)，因为有 n(n−1)/2 对，并且每个查询在进行预处理时至少为 O(1)，在不进行预处理时为 O(n)。 在 d 棵树上，这变成了 O(d n^2)，这远远超出了 d·n ≤ 5e5 的任何限制。 

关键的观察结果是，加权树的成对最小路径结构完全由其排序边和并查找样式重建过程决定。 

如果我们按权重降序对边进行排序并将它们一一相加，我们就可以有效地从最强的边向下构建连接。 当我们添加权重为 w 的边时，它会连接两个组件。 这意味着通过该边连接的每对节点的路径最小值至少为 w，并且未来更小的边不会改变这些对的该最小值，因为未来的边仅具有更小的权重。 

这导致了一个关键的重新表述：每棵树都会产生一个连接层次结构，其中组件按边权重的降序合并。 这与最大跨度森林构建捕获的结构完全相同，并且等效地通过按降序排序的边上的不相交集并集过程捕获。 

因此，我们可以计算每棵树的规范表示，而不是比较所有成对值：按边权重排序的 DSU 合并历史记录。 当且仅当两棵树诱导的 DSU 合并结构在组件合并上的同构上相同（可以确定地序列化）时，两棵树才是等效的。 

我们通过按降序模拟类似 Kruskal 的处理并以标准化结构记录合并，将每个设计编码为规范签名，例如通过压缩组件 ID 并以一致的可哈希序列记录联合事件。 一旦每个设计都有签名，等价性就简化为查找相同签名的第一次出现。

 ## 算法演练

 1. 对于每个设计，读取所有 n−1 条边并将它们存储为 (u, v, w)。 当我们首先考虑更强的边缘时，我们将把树转换成一个捕获连通性如何演变的结构。 
2. 按权重降序对边进行排序。 这种排序至关重要，因为成对路径上的最小值仅取决于路径上最弱的边缘，并且从强到弱的处理可确保我们在较弱的边缘影响任何事物之前定义连通性。 
3. 初始化一个不相交的并集结构，每个节点都在其自己的组件中。 此时，不存在任何连接，因此每个节点都是孤立的。 
4. 按排序顺序处理边。 对于每条边 (u, v, w)，检查 u 和 v 是否属于不同的分量。 如果存在，请将它们合并。 合并的时刻对应于为通过该合并而连接的所有对引入最小边约束 w。 
5. 在每次联合期间，为结果组件构造一个规范标识符。 这可以通过为每个 DSU 根维护一个代表性标签并以确定性方式组合子表示来完成，例如合并排序的子 ID 或使用滚动哈希。 目标是同构合并历史产生相同的表示。 
6.处理完所有边后，最终的结构由DSU森林编码表示。 每个设计将其转换为单个散列或规范字符串。 
7. 使用从签名到最早索引的字典。 对于每个设计 i，如果之前在 j 处见过其签名，则输出 j，否则存储 i。 

### 为什么它有效

 不变的是，在处理权重大于或等于 w 的所有边之后，DSU 组件仅使用可以强制执行至少为 w 的最小路径值的边来准确地表示连接性。 在此阶段，同一组件中的任何节点对都具有一条路径，其最小边权重至少为 w，并且一旦两个节点以权重 w 合并，以后涉及较小边的操作都不会减少该对的最小值。 

因此，DSU 合并序列恰好在每对节点首次在阈值下连接时进行编码，该阈值相当于它们的成对最小路径值。 由于等效性纯粹是根据这些成对最小值来定义的，因此相同的合并历史意味着等效，不同的历史意味着至少一对存在差异。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.sz = [1] * n
        self.sig = [(i,) for i in range(n)]

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return a
        if self.sz[a] < self.sz[b]:
            a, b = b, a
        self.p[b] = a
        self.sz[a] += self.sz[b]

        merged = self.sig[a] + self.sig[b]
        self.sig[a] = tuple(sorted(merged))
        return a

def build_signature(n, edges):
    edges.sort(key=lambda x: -x[2])
    dsu = DSU(n)

    for u, v, w in edges:
        dsu.union(u, v)

    roots = []
    for i in range(n):
        if dsu.find(i) == i:
            roots.append(dsu.sig[i])

    roots.sort()
    return tuple(roots)

def solve():
    d, n = map(int, input().split())
    seen = {}
    ans = []

    for i in range(d):
        edges = []
        for _ in range(n - 1):
            a, b, c = map(int, input().split())
            edges.append((a - 1, b - 1, c))

        sig = build_signature(n, edges)

        if sig in seen:
            ans.append(seen[sig] + 1)
        else:
            seen[sig] = i
            ans.append(i + 1)

    print(*ans)

if __name__ == "__main__":
    solve()
```按降序对边进行排序后，DSU 构造可确保每个合并事件都对结构签名做出贡献。 签名按组件存储，最后所有组件都聚合到全局元组中。 这保证了结构相同的合并历史产生相同的密钥。 

一个微妙的实现点是使用基于元组的规范化。 虽然这不是最节省内存的表示形式，但它保留了正确性和确定性比较，这对于等价性检测至关重要。

 输入设计：```
n = 3
edges: (1-2, 1), (2-3, 1)
```| 步骤| 边缘 | 联合后的组件 |
 | ---| ---| ---|

 最终签名是包含所有节点的单个组件。 

现在考虑第二个相同的结构但不同的边顺序； 由于排序是降序的，因此两者产生相同的合并顺序，从而产生相同的签名。 这证实了输入中的排序不会影响等效性。

 ## 边缘情况

 关键的边缘情况是所有边缘具有相同的权重。 在这种情况下，任何树结构都会产生相同的成对最小路径值，因为每个路径最小值始终是共享权重。 该算法可以自然地处理此问题，因为所有边都在同一 DSU 级别合并，无论拓扑如何，都会生成相同的签名。 

另一个边缘情况是权重严格增加或减少的路径图。 DSU 过程确保合并发生在严格的层次结构中，因此签名反映了连接形成的确切顺序，从而防止与任何非同构结构的错误等价。 

最后的边缘情况是多个组件以相同的权重级别合并。 确定性地对边进行排序和处理可确保即使多个并集以相同的权重发生，所得的组件表示也是一致的并且不依赖于输入顺序。
