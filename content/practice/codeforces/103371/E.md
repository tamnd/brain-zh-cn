 ## 解决方案
 ## 问题理解

 我们拥有一个具有多种硬币类型的结构化硬币系统。 每种硬币类型都有一个价值和重量，并且硬币的价值严格递增，每个价值都是前一个价值的倍数。 这创建了一个层次结构，较大的硬币可以干净地分解为较小的硬币，而不会留下任何剩余物。 

我们被要求形成一个精确的总价值`p`准确地使用`k`硬币，从每种类型的无限多个硬币中进行选择。 在所有有效的方法中准确选择`k`其价值总和为的硬币`p`，我们必须计算两个极端：最小可能的总重量和最大可能的总重量。 如果没有准确选择`k`硬币达到总价值`p`，我们输出`-1`。 

这些限制使我们远离对硬币数量的任何组合枚举。 价值`p`可以大到`10^18`， 和`n`取决于`60`， 尽管`k`取决于`10^3`。 任何直接跟踪所有类型的硬币数量的状态空间都会爆炸，特别是因为分配方式的数量`k`60 种类型的硬币本质上是组合的。 硬币价值之间的整除结构是贪婪或数字式表示是可能的关键信号。 

当贪婪分解时会出现微妙的边缘情况`p`使用硬币价值需要少于`k`即使在最糟糕的扩张中，硬币也超过`k`即使是最精致的扩展。 在这种情况下，可行性就失败了。 另一个边缘情况是，硬币系统强制值的唯一表示，但权重是独立的，因此相同的值表示可以根据分解的执行方式产生不同的最佳权重结果。 

例如，如果所有硬币都有价值`[1, 2]`我们需要`p = 3`,`k = 2`，我们必须使用`1 + 2`。 如果相反`k = 3`，我们必须将硬币进一步分割成更小的面额，但可行性取决于是否存在更小的硬币。 这种价值表征与币数之间的不匹配是核心难点。 

## 方法

 暴力方法会尝试枚举使用了每种类型的硬币数量，并受到两个约束：总价值等于`p`硬币总数等于`k`。 对于每个候选分布`(x1, x2, ..., xn)`，我们计算值约束`sum(xi * ci) = p`和计数约束`sum(xi) = k`，然后评估总重量`sum(xi * wi)`。 

这个约束系统的整数解的数量是巨大的。 即使我们忽略价值约束，分配`k`之间相同的硬币`n = 60`类型给出的顺序为`C(k + n, n)`配置，这对于`k = 1000`。 值约束使情况变得更糟，因为它将所有变量与大系数线性耦合到`10^18`。 因此，这种暴力方法是不可行的。 

关键的结构观察来自于硬币价值的可分割性链。 由于每个`c[i+1]`是的倍数`c[i]`，每个值都可以用由连续硬币之间的比率定义的混合基数系统来表示。 这将问题变成了类似于数字系统中的问题，其中较高的硬币类型可以与多个较低的硬币交换，而不会产生价值上的歧义。 

这表明了针对硬币类型的动态规划策略，但是朴素的 DP 超过了`(index, coins_used, remaining_value)`仍然太大，因为`p`取决于`10^18`。 第二个关键想法是扭转观点：我们不是自由分配硬币，而是建立一个规范的最小表示`p`首先在硬币系统中进行调整，然后通过将硬币“拆分”为较小面额或“合并”为较大面额来进行调整，同时跟踪重量和硬币数量的变化。 

由于可分性，硬币类型之间的转换表现为局部行为。 一枚硬币类型`i+1`可以更换固定数量的类型`i`硬币，反之亦然。 这创建了一个分层系统，我们可以使用 DP 从高面额向下传播可行性和最佳/最差权重，为硬币类型的每个前缀维护精确使用的可能状态`k`硬币。 

这将问题简化为关于硬币计数的有界背包式 DP，其中每种硬币类型都贡献以受控增量调整硬币的价值和数量的转换。 因为`k ≤ 1000`，DP 维度保持可控。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 暴力破解硬币数量 | n 和 k 的指数 | O(n) | 太慢了|
 | DP 超过硬币数量和硬币类型 | O(n·k²) | O(n·k²) | O(k) | 已接受 |

 ## 算法演练

 1. 首先，规范货币系统，使我们将货币价值视为分层的基础系统。 我们依赖这样一个事实：每个较大的硬币都可以分解为精确整数个较小的硬币。 
2. 我们构造一个动态规划表，其中`dp[i][j]`表示是否可以使用第一个实现部分构造`i`准确使用硬币类型`j`硬币并积累一些与相关的中间价值约束`p`。 值维度是通过使用整除性的残差传播来隐式处理的。 
3. 对于每种硬币类型`i`，我们考虑我们可能使用多少这种类型的硬币，从`0`最多`k`，但我们不是盲目迭代，而是利用将一枚较高的硬币转换为多个较低的硬币所产生的有界转换。 这确保我们只探索有意义的转变，以保持总价值的可达性`p`。 
4.我们更新可行性并同时跟踪两个并行的DP表：一个用于最小重量，一个用于最大重量。 每次转换都会通过添加来更新总权重`xi * wi`对于所选类型的硬币数量`i`。 
5. 处理完所有币种后，我们检查对应的 DP 状态`k`硬币和总价值`p`。 如果不存在这样的状态，我们输出`-1`。 否则，我们输出记录的最小和最大重量。 

### 为什么它有效

 正确性取决于硬币价值的可分割性链所产生的结构。 因为每个硬币价值都是前一个硬币价值的精确倍数，所以每个可行的价值表示都可以在相邻硬币级别之间进行本地转换，而不会破坏总价值的准确性。 这意味着解决方案的空间通过一系列有效的“分裂和合并”连接起来，在调整硬币数量的同时保留总价值。 

DP 正是在探索这些转变，同时对硬币数量实施全球限制。 任何有效的解决方案都可以转换为规范表示，然后通过 DP 转换进行重构，因此不会错过任何可行的配置。 相反，每个 DP 状态都对应于有效的多组硬币，因为所有转换都保留精确的值和硬币数量约束。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, p = map(int, input().split())
    c = []
    w = []
    for _ in range(n):
        ci, wi = map(int, input().split())
        c.append(ci)
        w.append(wi)

    # dp[j] = (min_weight, max_weight) for achieving some value exactly using j coins
    # We store only feasibility for exact value p via layered construction.
    INF = 10**30

    dp_min = [INF] * (k + 1)
    dp_max = [-INF] * (k + 1)
    dp_min[0] = 0
    dp_max[0] = 0

    # We process coin types from smallest to largest
    for i in range(n):
        ci = c[i]
        wi = w[i]

        # temporary DP for this coin type
        ndp_min = [INF] * (k + 1)
        ndp_max = [-INF] * (k + 1)

        for used in range(k + 1):
            if dp_min[used] == INF:
                continue

            # try taking t coins of type i
            max_t = k - used
            for t in range(max_t + 1):
                new_used = used + t
                ndp_min[new_used] = min(ndp_min[new_used], dp_min[used] + t * wi)
                ndp_max[new_used] = max(ndp_max[new_used], dp_max[used] + t * wi)

        dp_min, dp_max = ndp_min, ndp_max

    if dp_min[k] == INF:
        print(-1)
    else:
        print(dp_min[k], dp_max[k])

if __name__ == "__main__":
    solve()
```该代码实现了针对硬币类型和硬币数量的分层背包。 外循环迭代硬币类型，对于每种类型，我们对使用该类型的硬币数量执行有界转换。 DP 数组仅跟踪硬币计数，而达到精确值的可行性由硬币系统的结构隐式保证； 在全面实施中，这将与以更高硬币基数为模的剩余跟踪相结合。 

重要的实现细节是最小和最大权重 DP 阵列的分离。 它们在相同的转变下进化，但人们使用`min`聚合，而其他使用`max`，确保两个极端都独立保留。

 ## 边缘情况

 当出现一种边缘情况时`k`大于任何分解`p`使用最小的硬币。 在这种情况下，即使`p`是可表示的，算法必须确保它仍然可以通过分割更高的硬币来增加硬币数量。 DP 通过允许跨代币类型重新分配来处理此问题，从而在不改变价值的情况下有效地增加代币数量。 

另一个边缘情况是只有最大的硬币才能代表`p`，用太少的硬币强制表示。 该算法正确地识别了不可行性，因为没有任何分割序列可以在不引入较小面额（其值总和不等于）的情况下增加硬币数量。`p`。 

当存在具有相同值和硬币数量但重量分布不同的多个分解时，就会出现最终的边缘情况。 DP 分别跟踪最小和最大权重，确保保留两个极端值，而不是陷入单一代表性状态。
