 ## 解决方案
 ## 问题理解

 我们得到一个无向图，每个顶点都有一个权重，并且在步行的步长上定义了一个成本函数。 任务是选择一个起始顶点，然后沿着边执行一系列移动，以最大化收集的总利润，其中利润取决于访问的顶点以及步行的每个部分的距离。 

图表本身对于我们如何在其中移动并不是任意的。 每次遍历都会产生一个结构重要的序列，而奖励取决于当前段的长度与给定成本数组的匹配。 直观上，每次移动都会贡献顶点增益，同时受到惩罚或调整成本，该成本仅取决于我们在当前“阶段”已经走了多远。 

这些限制意味着我们无法对路径进行任何二次或三次推理。 具有最大尺寸的图和距离上的路径相关动态规划的存在立即表明枚举所有步行甚至所有简单路径是不可能的。 任何解决方案都必须将图的结构压缩成几乎非循环的结构或至少在路径长度上受到严格约束。 

当顶点具有相同度数时，会出现微妙的边缘情况。 如果两个相邻顶点具有相同的度数，则简单的方向规则会破坏对称性，并可能意外地允许振荡或不正确的过渡。 

例如，考虑一个所有度数都相等的三角形。 如果我们尝试任意定向边缘，我们可能会引入循环并允许无限的改进循环。 此构造中的正确行为是等度边实际上不可用，必须将其删除。 

另一个边缘情况是星图。 中心的度数较高，叶子的度数为 1。 任何不严格按程度强制执行方向的不正确方向规则都将创建循环或允许以违反预期单调结构的方式重新访问中心。 

## 方法

 强力解释是考虑从每个顶点开始的每种可能的步行，并计算段长度和顶点贡献的最佳可能序列。 每个状态都需要存储当前顶点、当前段长度以及可能的先前决策。 即使在一棵树上，这也会导致指数级的行走，因为每一步都会分支到除父级之外的所有邻居。 即使进行剪枝，稠密图中长度达到 k 的不同路径的数量在 k 中呈指数关系，并且在最坏的情况下 k 本身在 n 中可以是线性的。 

关键的观察是图结构可以使用顶点度转换为有向无环图。 每条边的方向都是从低度数到高度数，并且等度顶点之间的边被删除。 这创建了一个单调性条件：沿着任何有向边，次数严格增加，因此循环是不可能的。 

一旦我们有了 DAG，我们就需要了解任何路径可以有多长。 关键的组合见解是，如果一条路径的长度为 k，那么沿着该路径的度数必须严格增加，并且每一步都会迫使图中存在一定量的度数“质量”。 沿路径求和度贡献会产生 k 的下界二次方，该下限不能超过边总数 m。 这将最大路径长度限制为 O(sqrt(m))。 这种限制使得路径长度上的动态规划变得可行。

通过这个结构，我们定义了顶点和路径长度的动态编程状态。 对于每个顶点 v 和长度 d，我们计算以 v 结束的路径的最佳可能利润，其中最后一段的长度为 d。 转换来自 DAG 中的传入邻居，我们仔细维护辅助值，以避免重新计算每个状态的所有邻居。 

我们还维护一个辅助数组 opt[v]，它表示从 v 开始的最佳答案，对第一段没有限制。 这使我们能够在初始化段长度为 1 的转换时减少嵌套依赖性。 

一般 d 的转换基于将路径从邻居 u 延伸到 v，通过删除和添加顶点贡献进行调整并应用段长度之间的成本差异。

 | --- | --- | --- | --- |
 | 蛮力横扫各行各业| 指数| O(n) 递归深度 | 太慢了 |
 | 路径长度上的面向度的 DAG + DP | O(n sqrt(m)) | O(n sqrt(m)) | O(n sqrt(m)) | O(n sqrt(m)) | 已接受 |

 ## 算法演练

 ### 1. 计算顶点度数

 我们首先计算每个顶点的 deg[v]。 这是必要的，因为所有进一步的结构都依赖于比较度数来决定边缘方向。 

### 2. 按度数定向边缘

 对于每条边（u，v），我们将其从较小度数的顶点引导到较大度数的顶点。 如果度数相等，我们完全丢弃边缘。 这强制执行严格的排序约束，以防止循环并确保任何有效路径都朝着严格增加度值的方向移动。 

### 3.限制最大路径长度

 我们观察到，沿着任何有向路径，度数都会严格增加。 长度为 k 的路径意味着累积度数增长，迫使总度数质量至少为二次方。 由于总边数为 m，这意味着 k 至多与 sqrt(m) 成正比。 这是使动态规划有限的关键结构压缩。 

### 4.定义DP状态

 我们将 dp[v][d] 定义为以 v 结束的路径的最佳可实现利润，其中最后一段的长度恰好为 d。 我们还将 opt[v] 定义为 dp[v][d] 中所有 d 的最大值。 这种分离允许我们在扩展路径时重用结果，而无需重新计算所有段长度。 

### 5. 基本情况初始化

 对于任何顶点 v，长度为零的路径贡献的 dp[v][0] 等于顶点权重 a[v]。 这对应于在 v 处开始一条没有采取任何边的路径。 

### 6. 段长度为 1 的转换

 对于 d = 1，我们考虑从邻居 u 开始一个新段到 v。我们将 opt[u] 与 v 的贡献结合起来，并减去第一个段成本。 这捕获了 v 在 u 之后立即开始新阶段的情况。 

### 7. d 大于 1 时的转换

 对于较长的段，我们将前一段从 u 延伸到 v。我们采用 dp[u][d−1]，删除在前一个状态中计算的 u 的贡献，添加 v，并使用连续段长度之间的成本差异进行调整。 这可以小心地保持分段计算的正确性，而不会重复计算顶点贡献。 

### 8.最终答案

 答案是所有顶点 v 上的最大 opt[v]。 

### 为什么它有效

 定向步骤保证所有转换都沿着 DAG 移动，因此不存在循环，也不存在重复的无效状态重组。 基于度的排序确保任何路径都具有有界长度，这限制了 DP 维数。 dp 公式正确地分离了段转换，以便通过一系列状态转换准确地表示每个有效游走一次，并且不会形成无效游走，因为边缘始终遵守严格的度数增加。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())
    a = [0] + list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    g = [[] for _ in range(n + 1)]
    edges = []
    
    deg = [0] * (n + 1)
    
    for _ in range(m):
        u, v = map(int, input().split())
        deg[u] += 1
        deg[v] += 1
        edges.append((u, v))
    
    dag = [[] for _ in range(n + 1)]
    
    for u, v in edges:
        if deg[u] < deg[v]:
            dag[u].append(v)
        elif deg[u] > deg[v]:
            dag[v].append(u)
    
    import math
    lim = int(2 * math.isqrt(m)) + 5
    
    dp = [[float('-inf')] * (lim + 1) for _ in range(n + 1)]
    opt = [float('-inf')] * (n + 1)
    
    for v in range(1, n + 1):
        dp[v][0] = a[v]
        opt[v] = a[v]
    
    for d in range(1, lim + 1):
        for v in range(1, n + 1):
            best = float('-inf')
            
            for u in dag[v]:
                if d == 1:
                    best = max(best, opt[u] + a[v] - b[0])
                else:
                    if dp[u][d - 1] != float('-inf'):
                        best = max(best, dp[u][d - 1] - a[u] + b[d - 2] + a[v] - b[d - 1])
            
            dp[v][d] = best
            opt[v] = max(opt[v], best)
    
    print(max(opt[1:]))

if __name__ == "__main__":
    solve()
```实现首先构建度数数组，然后构建遵守严格度数排序的有向邻接表。 等度边立即被丢弃，这对于保持非循环性是必要的。 

DP 表仅分配到 sqrt(m) 界限，这对于内存可行性至关重要。 每个 dp[v][d] 是通过扫描 DAG 中传入的邻居来计算的。 转换内的两种情况与开始新段或扩展现有段完全对应。 

一个微妙的点是数组 b 的索引，因为段成本是从 1 开始定义的，但 Python 数组是从 0 开始的。 该代码一致地移动索引，以便 b[d-1] 对应于第 d 段成本。

 ### 示例 1

 考虑一个小链图，其中度沿着路径严格增加。 

| 步骤| v | d | 选择了你| dp[v][d] | dp[v][d] |
 | --- | --- | --- | --- | --- |
 | 初始化| 1 | 0 | - | 一个[1] |

 此跟踪显示每个步骤如何扩展先前的最佳段，而不是从头开始重新计算。 DP 累积贡献，同时在正确的边界减去段成本。

 考虑一个中心为 1、叶子为 2、3、4 的星图。 

| 步骤| v | d | 过渡| dp[v][d] | dp[v][d] |

 这表明方向可以防止等度节点之间无效的来回移动，并强制执行严格的方向结构。

 | 测量 | 复杂性 | 说明|

 ## 边缘情况

 所有顶点的度数均为二的三角形是等度规则最清晰的应力情况。 计算度数后，所有边都被删除，留下孤立的顶点。 然后，DP 会正确地减少到采用最大单个顶点权重，因为不存在任何过渡。 

星形图显示了相反的行为。 中心的度数高于叶子，因此所有边缘都从叶子到中心。 这迫使所有路径都通过中心，并且 DP 正确地累积从叶到中心的转换，而不允许无效返回。 

线图确认 DP 在段长度上工作正常。 度数向中间增加然后减少，产生定向后行为类似于 DAG 的定向结构，并且最长路径边界保持足够紧密以允许 DP 状态的完全传播。
