 ## 解决方案
 ## 问题理解

 我们得到一个无向图，表示由通道连接的位置。 任务是确定如果我们重复进入可以充分探索并返回入口点的区域，则该图的哪些部分仍然可用，但我们不能陷入无法返回图其余部分的位置。 

关键思想是，某些顶点实际上是死胡同：一旦进入它们，就无法保证通过其他未探索的边的返回路径。 问题的输出是删除所有此类“不安全”顶点后剩余的子图，其中每个剩余顶点都是允许进入和离开而不会被困住的结构的一部分。 

输入大小意味着在最坏的情况下图最多具有约 200,000 个顶点和边。 这立即排除了任何试图模拟所有可能的遍历或独立地从每个顶点运行搜索的方法。 任何解决方案都必须在图大小的线性或近线性时间内运行，通常为 O(n + m)，因为即使 O(n²) 也已经太慢了。 

当图表看起来几乎像一个循环但附加了悬挂的树时，就会出现一种微妙的情况。 考虑四个顶点的循环，其中两个顶点的链附加到一个节点。 链顶点不是任何循环的一部分，因此，如果仅检查一侧的可达性，天真的直觉可能会错误地保留它们。 然而，这些顶点是死胡同，因为一旦进入，它们就会迫使你沿着同一条路径向后走。 

另一个棘手的情况是两个顶点之间的单边。 两个顶点在技术上是相连的，但都不允许在更大的结构中有意义的“进入和退出”行为，因此根据解释，它们不应该在迭代修剪中幸存，除非定义明确允许双节点循环结构。 

## 方法

 思考这个问题的蛮力方法是直接模拟探索的想法。 对于每个顶点，我们尝试看看是否可以进入它并仍然返回到它，同时继续在其他地方探索。 这建议从每个节点运行 DFS 或 BFS，并检查是否存在允许返回而无需重新追踪最后一条边的循环或替代路线。 

这立即变得昂贵，因为每次这样的检查都会花费 O(n + m)，并且对所有顶点执行此操作会导致 O(n(n + m))，这远远超出了限制。 

关键的结构观察是，“进入和返回”的能力相当于属于一个没有顶点作为强制端点的组件。 用图术语来说，安全的顶点是那些在迭代修剪过程意义上没有离开的顶点：如果顶点的度数为 1，则它不能是任何允许重新访问而无需回溯的循环结构的一部分。 一旦这样的顶点被移除，它的邻居也可能变成叶子，从而向内传播效果。 

这将问题简化为重复删除 1 度顶点直至稳定。 剩下的正是每个顶点的度数至少为 2 的子图，它对应于图的所有包含环的部分的并集，通常称为图的 2 核。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(n(n + m)) | O(n(n + m)) | O(n + m) | 太慢了 |
 | 叶子修剪（2核）| O(n + m) | O(n + m) | 已接受 |

 ## 算法演练

1. 构建图的邻接表表示并计算每个顶点的度。 这是必要的，因为整个过程仅取决于每个顶点当前有多少条边。 
2. 初始化一个队列，其中包含度数恰好为 1 的所有顶点。这些是直接的死胡同，因为进入它们会强制返回单个退出路径。 
3. 反复从队列中删除顶点。 当删除一个顶点时，从概念上讲，将其从图中删除并减少其所有邻居的度数。 这模拟了切断死胡同并更新成为新死胡同的内容。 
4. 如果任何邻居的度数因此下降为 1，则将其添加到队列中。 这是传播步骤，捕捉修剪一片叶子如何产生另一片叶子的过程。 
5. 继续，直到队列中没有 1 度顶点。 此时，每个剩余顶点在简化图中的度数至少为 2。 
6. 输出在此过程中从未移除的所有顶点。 

这种贪婪删除起作用的关键原因是，在此过程中成为叶子的任何顶点都不能属于允许保证返回的遍历的结构。 删除它不会破坏任何有效的基于循环的结构，因为循环中的顶点始终在循环内维护至少两个关联边。 

### 为什么它有效

 不变的是，在每一步中，保证所有删除的顶点不属于任何子图的一部分，其中每个顶点在剩余结构中至少有两个邻居。 在当前简化图中，任何被删除的顶点的度数最多为 1，这意味着它不能位于循环或任何允许替代返回路径的结构上。 由于循环永远不会失去其属性，即循环内每个顶点的度数至少为 2，因此不会删除任何有效的解顶点。 当过程停止时，每个剩余顶点的内部度至少为 2，因此不可能进一步修剪，并且剩余集合相对于此属性而言是最大的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
from collections import deque

def solve():
    n, m = map(int, input().split())
    g = [[] for _ in range(n)]
    deg = [0] * n

    for _ in range(m):
        u, v = map(int, input().split())
        u -= 1
        v -= 1
        g[u].append(v)
        g[v].append(u)
        deg[u] += 1
        deg[v] += 1

    q = deque()
    removed = [False] * n

    for i in range(n):
        if deg[i] == 1:
            q.append(i)
            removed[i] = True

    while q:
        u = q.popleft()
        for v in g[u]:
            if removed[v]:
                continue
            deg[v] -= 1
            if deg[v] == 1:
                removed[v] = True
                q.append(v)

    res = [str(i + 1) for i in range(n) if not removed[i]]
    print(len(res))
    if res:
        print(" ".join(res))

if __name__ == "__main__":
    solve()
```该实现维护邻接列表和度数数组，以便仅当删除其端点之一时才处理每条边。 队列确保以自然的剥离顺序处理顶点，始终立即处理新形成的叶子。 

一个微妙的细节是在顶点进入队列时将其标记为已删除，而不是在处理它们时。 当同一顶点的度数通过不同的邻居下降时，这可以防止对同一顶点进行多次入队操作。 

## 工作示例

 考虑一个形状像四个顶点的循环的图，尾部连接到一个节点。 尾巴由两个顶点组成。 

初始度数如下所示。 

| 步骤| 已删除队列 | 度数（A、B、C、D、E、F）|
 | --- | --- | --- |
 | 开始| [E，F] | A=3，B=2，C=2，D=2，E=1，F=1 |
 | 删除 E | [F] | A=2，B=2，C=2，D=2，F=1 |
 | 删除 F | []| A=2，B=2，C=2，D=2 |

 修剪后，只剩下循环。 这表明链顶点首先被消除，并且它们的消除不会影响核心循环结构。 

现在考虑一个包含五个顶点的简单线图。 

| 步骤| 已删除队列 | 度数 (1-5) |
 | --- | --- | --- |
 | 开始| [1,5]| 1=1, 2=2, 3=2, 4=2, 5=1 |
 | 删除 1 | [5]| 2=1, 3=2, 4=2, 5=1 |
 | 删除 5 | [2] | 2=1, 3=2, 4=2 |
 | 删除 2 | []| 3=2, 4=2 |
 | 删除 3,4 隐式稳定 | []| 核心空 |

 这表明在一棵树中，所有东西最终都会消失，因为每个顶点最终都会暴露为叶子。

 ## 边缘情况

 两个顶点之间的单条边会立即触发两个端点的删除。 队列最初包含两个顶点，因为每个顶点的度数都是 1。删除第一个顶点会将邻居的度数降低到 0 或 1，具体取决于表示形式，并且它也会被删除。 最终的输出是空的，这与没有顶点可以支持可返回遍历的想法一致。 

纯粹的循环从一开始就表现出稳定性。 每个顶点的度数都是 2，因此初始队列为空，不会发生删除操作。 该算法立即返回所有顶点，证明循环是修剪过程的固定点。 

一棵树完全倒塌。 首先移除每片叶子，向内传播直到没有顶点剩余。 每个步骤都会减小树的大小，而无需创建稳定的核心，从而确认树不包含基于循环的可生存结构。
