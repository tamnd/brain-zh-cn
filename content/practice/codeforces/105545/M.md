 ## 解决方案
 ## 问题理解

 我们得到一个图，其顶点代表岛屿，其边缘代表岛屿对之间的直接旅行路线。 关键的结构约束是该图不包含循环，这意味着该图的每个连接组件都是一棵树。 

在每个连接的组件内，盗版者只能沿着该组件的边缘移动。 它们不能在不同的组件之间移动。 我们想要计算的量是位于同一连通分量中的每对顶点之间的最短路径距离的总和，并对所有分量求和。 

所以问题自然分为两个层面。 首先，我们需要了解输入图如何分解为树组件。 其次，我们需要为每棵树计算所有无序顶点对的距离总和，然后将这些结果相加。 

输入大小隐含的约束（最大 n，在这种类型的问题中通常为 10^5 或更多）迫使解决方案避免任何对的二次枚举或显式的全对最短路径。 任何尝试直接计算每个组件内部所有对之间距离的方法都会立即变得太慢，因为即使是大小为 k 的单个大组件也需要 O(k^2) 对处理。 

一个微妙但重要的一点是边的数量 m 是给定的。 因为该图是一个森林，所以我们可以使用它来推断连接组件的数量，而无需运行完整的 DFS 或 DSU（如果需要）：具有 n 个顶点和 s 个组件的森林始终具有恰好 n − s 个边，因此 s = n − m。 

当人们假设每个组件都可以独立处理但仍然尝试从每个组件内的每个节点进行 BFS 时，就会出现天真的推理的典型失败案例。 例如，在 100000 个节点的链中，这已经意味着大约 10^10 次操作。 

另一个微妙的错误是假设任何树结构都会产生相似的距离和； 事实上，每个组件内树的结构决定了距离是否最小化，而问题是要求与组件大小一致的最小可能总距离。 

## 方法

 蛮力视角首先获取每个连接的组件，从每个顶点运行 BFS 或 DFS，然后将到所有其他顶点的距离相加。 原则上这是正确的，因为树中的最短路径是唯一的，并且 BFS 给出了精确的距离。 然而，对于大小为 k 的组件，这需要 O(k) BFS 运行，每次成本为 O(k)，导致每个组件的时间为 O(k^2)。 对所有组件求和，就变成了 O(n^2)，当 n 很大时，这远远超出了可行的限制。 

关键的结构见解是问题实际上并不取决于每个组件中树的具体形状。 相反，我们隐含地要求给定大小的所有树之间的成对距离的最小可能总和。 这使我们能够用最佳树结构替换每个组件，从而最小化所有顶点对之间的距离总和。 

对于固定的组件大小 k，当树尽可能“星形”时，距离总和最小。 在这种结构中，一个中心节点与所有其他节点相连。 这将创建一个配置，其中任何一对叶子的距离恰好为 2，并且涉及中心的叶子对的距离为 1。这给出了最小距离和的闭合形式表达式。

一旦我们了解组件的最佳值仅取决于其大小，剩下的任务就是确定如何将 n 个顶点拆分为 s 个组件（其中 s 从 m 导出），以便最小化贡献平方和的总和。 凸性论证表明组件的大小最多相差 1； 否则，将顶点从较大的组件转移到较小的组件会严格降低总成本。 

因此，该解决方案将整个图问题简化为 n 的整数分区上的组合优化。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | BFS/DFS 组件中的所有对 | O(n^2) | O(n^2) | O(n) | 太慢了 |
 | 组件尺寸优化 | O(n) | O(1) | O(1) | 已接受 |

 ## 算法演练

 我们首先观察到，因为图是一个森林，所以每个连接的组件都是一棵树。 如果有 s 个分量和总共 n 个顶点以及 m 个边，则 s = n − m。 

接下来，我们考虑在大小为 k 的单个组件内对答案有什么贡献。 我们感兴趣的是该组件中所有无序顶点对的最短路径距离之和。 

我们对最小化这个总和的结构进行推理。 最小化成对距离的树将所有顶点集中在靠近中心节点的位置。 极端情况是一棵星形树，其中一个中心连接到所有其他 k − 1 个节点。 

在此配置中，距离 1 处有 k - 1 对，对应于入射到中心的每个边缘。 所有其他对都位于两个叶子之间，距离为 2。这给出了简化为 (k − 1)^2 的总贡献。 

现在我们需要将 n 个顶点分配到 s 个分量中，以最小化所有分量上这些值的总和。 每个大小为 x 的分量贡献 (x − 1)^2，因此我们希望在固定总和约束下最小化平方项之和。 

我们比较大小为 a 和 b 的两个组件，其中 b 明显大于 a。 如果 b − a ≥ 2，我们可以将一个顶点从较大的组件移动到较小的组件。 这严格降低了总成本，因为平方函数是凸函数。 重复此过程会导致所有组件大小相差最多 1 的情况。 

因此，所有组件的大小必须为 k 或 k + 1，其中 k = ⌊n / s⌋，剩余的余数决定有多少个组件的大小为 k + 1。 

最后，我们通过对所有分量求和 (size − 1)^2 来计算总答案。 

### 为什么它有效

 函数 f(x) = (x − 1)^2 在 x 上是凸函数。 对于固定的顶点总数，不均匀地分布它们会增加凸成本的总和。 可以通过将顶点从较大的组件移动到较小的组件来减少两个组件之间的任何不平衡，这会严格降低总值。 因此，最佳配置必须使所有组件尺寸尽可能相等，这完全决定了解决方案的结构。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m = map(int, input().split())

    # number of connected components in a forest
    s = n - m

    # each component size should be either k or k+1
    k = n // s
    r = n % s  # r components will have size k+1

    # sum of (size - 1)^2
    ans = 0

    # r components of size k+1
    ans += r * (k * k)

    # (s - r) components of size k
    ans += (s - r) * ((k - 1) * (k - 1))

    print(ans)

if __name__ == "__main__":
    solve()
```第一步使用森林的身份计算连接组件的数量。 这完全避免了运行任何图遍历。 

然后我们将 n 分成 s 几乎相等的部分。 商给出了基础尺寸，余数决定了有多少个分量比一个顶点大。 

最后，我们对每个组件应用导出的公式 (size − 1)^2。 对于大小 k，这变为 (k − 1)^2，对于大小 k + 1，它变为 k^2。 总结这些就得出了最终的答案。 

必须小心整数除法和余数的处理； 将它们混合在一起会导致较大组件的数量出现相差一的错误。

import sys, io

def run(inp: str) -> str:

 ## 边缘情况

 当 n 等于 m + 1 时，只有一个连通分量，即 s = 1。算法简化为 k = n，r = 0，因此答案变为 (n − 1)^2。 这对应于一棵树，并且该公式正确地捕获了其最小星形配置。 

当 m = 0 时，每个顶点都是孤立的，因此 s = n。 每个组件的大小为 1，即 k = 1 且 r = 0。每项贡献 (1 − 1)^2 = 0，因此总答案为 0。这与任何组件内都不存在顶点对的事实相匹配。 

当 n 不能被 s 整除时，余数 r 确保正好有 r 个分量接收额外的顶点。 凸性论证保证任意分布这些额外的顶点会增加总和，因此余数处理对于正确性至关重要。
