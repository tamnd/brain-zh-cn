 ## 解决方案
 ## 问题理解

 我们有一块大小为的方板$n \times n$。 我们对有序对感兴趣$(a, b)$，其中两者$a$和$b$是之间的整数$1$和$n$，我们想要计算有多少这样的对满足整除条件：至少有一个数字$a$或者$b$能被一个固定整数整除$k$。 

虽然最初的动机是谈论用平铺矩形$1 \times k$块，计算任务完全减少到计算满足简单算术属性的正方形中的格点。 

输出是一个整数：对的数量$(a, b)$在网格中$[1, n] \times [1, n]$在哪里$a$可以整除$k$， 或者$b$可以整除$k$，或两者兼而有之。 

约束没有明确说明，但这是一个典型的 Codeforces 算术问题，所以我们应该期望$n$和$k$至少达到$10^9$或更高，并且可能有多个测试用例。 这立即排除了迭代所有$n^2$成对，因为即使$n = 10^5$已经给出了$10^{10}$成双成对，这是远远不可能的。 

一个常见的陷阱是尝试模拟或枚举对或过度考虑平铺解释。 另一个错误是重复计算对，其中两个坐标都可以被整除$k$。 例如，如果$n = 5$,$k = 2$，然后像这样配对$(2, 4)$在这两种情况下都被计算在内并且必须被纠正。 

## 方法

 暴力解决方案将迭代所有对$(a, b)$，检查是否$a \bmod k = 0$或者$b \bmod k = 0$，并统计满足条件的数量。 这是正确的，因为它直接实现了需求的定义。 然而，它执行$n^2$检查，每次检查都是常数时间，所以总复杂度是$O(n^2)$。 即使对于中等值，这也是不可能的$n$， 自从$n = 10^5$需要$10^{10}$运营。 

关键的观察是，该条件仅取决于各个坐标的整除性，而不取决于之间的任何相互作用$a$和$b$除非通过工会。 这使我们能够计算互补的结构化集而不是枚举对。 

我们定义两个集合：第一个坐标可被整除的对$k$，以及第二个坐标可被整除的对$k$。 每个集合很容易独立计数，因为整除性是周期性的。 在范围内$1$到$n$， 确切地$\lfloor n / k \rfloor$数字可以整除$k$。 一旦我们知道了这一点，计数对就变成了一个简单的乘积$n$，因为不受限制的坐标可以取任何值。 

唯一的复杂之处是重叠：两个坐标都可以被整除的对$k$计算了两次，所以我们减去那个交集。 这直接导致了干净的包含-排除结构。

 | --- | --- | --- | --- |
 | 蛮力 |$O(n^2)$|$O(1)$| 太慢了 |
 | 最佳 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 我们想要计算所有对$(a, b)$使得至少一个坐标可以被$k$。 

1. 计算有多少个整数$[1, n]$可以被整除$k$。 这是$cnt = \lfloor n / k \rfloor$。 该值表示对于必须满足整除条件的坐标，我们有多少种选择。 
2. 计算对，其中$a$可以整除$k$。 对于每个有效的$a$,$b$可以是以下任意一个$n$价值观。 这给出了$cnt \cdot n$对。 这样做的原因是条件只限制$a$， 和$b$是完全免费的。 
3. 同样，计算对，其中$b$可以整除$k$。 这也给出了$cnt \cdot n$。 对称性确保两个贡献是相同的。 
4. 减去对，其中两者$a$和$b$可以被整除$k$。 两个坐标必须来自同一组可整除的大小$cnt$，所以这个交集贡献$cnt^2$对。 
5. 使用包含-排除进行组合：$ans = 2 \cdot cnt \cdot n - cnt^2$。 

为什么有效：每个有效对都根据每个坐标是否可被整除来分类$k$。 两个坐标都不可整除的对将被自动排除。 恰好有一个可整除坐标的对在正确的项中计算一次。 具有两个可整除坐标的对被计数两次并通过减去交集来校正。 这保证了每个有效对都被精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k = map(int, input().split())
    cnt = n // k
    ans = 2 * cnt * n - cnt * cnt
    print(ans)

if __name__ == "__main__":
    solve()
```直接按照推导公式实现。 唯一的计算是整数除法以确定有多少倍$k$位于范围内。 最终表达式在 Python 中使用 64 位安全算术，因此不必担心溢出。 

一个微妙的点是确保使用整数除法，而不是浮点除法，因为结果必须精确。 另一个细节是，两项都是对称的，因此不需要单独跟踪$a$和$b$超出共享计数。

 ## 边缘情况

 一种边缘情况是当$k > n$。 在这种情况下，没有倍数$k$范围内，所以$cnt = 0$。 公式自然变为零，无需任何特殊分支。 用于输入$n = 10, k = 11$，我们计算$cnt = 0$，所以两项都消失，结果是$0$，这符合预期的解释。 

另一个边缘情况是当$k = 1$。 每个整数都可以被$1$， 所以$cnt = n$。 公式变为$2n^2 - n^2 = n^2$，意味着每对都是有效的。 为了$n = 4, k = 1$，网格有$16$对，并且公式正确返回$16$。 

最终的结构边缘情况是小网格，其中包含-排除重叠主导直觉。 为了$n = 2, k = 2$，只有数字$2$是可整除的，所以$cnt = 1$。 公式给出$2 \cdot 1 \cdot 2 - 1 = 3$。 列出对证实了这一点：$(2,1), (2,2), (1,2)$，正好三个有效对，表明减法正确地消除了重复计算$(2,2)$。
