 ## 解决方案
 ## 问题理解

 我们得到一个以十进制形式书写的数字，我们需要构造另一个严格大于它的数字，同时满足数字限制。 限制是位置上的：如果我们将两个数字按十进制表示形式对齐，那么在每个数字位置，新数字的数字必须与原始数字在同一位置的数字不同。 

任务是构建仍然严格大于此规则下给定数字的最小可能数字。 

约束的结构是造成困难的原因。 按十进制形式按字典顺序比较数字意味着它们不同的第一个位置决定哪个数字更大。 同时，数字限制对每个位置都是独立的，所以我们不能随意复制数字，然后只调整一个位置。 

输入由单个整数组成，被视为一串数字。 输出是满足这两个约束的另一个整数。 

即使没有正式的限制，该结构也意味着我们必须处理每个数字一次或恒定的次数。 当数字达到 10^5 位时，任何尝试枚举候选值或增量模拟所有有效数字的解决方案都会太慢，因为即使对每个候选值进行线性扫描也是不可接受的。 

一些边缘情况很重要。 

如果输入以 9 开​​头，“将第一个数字增加 1”的天真的逻辑必须小心处理十进制表示形式的进位。 例如，如果 x = 9，则下一个有效的前导数字不能是 9，并且还必须产生更长的数字，因此正确的构造可能会完全改变长度。 

如果所有数字都是 9，则必须重新考虑任何试图按数字递增而不扩展长度的结构，因为同一位置的每个数字都被禁止匹配。 

另一个微妙的情况是，贪婪尝试修改后面的数字，同时保持前缀与 x 相同。 这种方法失败了，因为数字限制迫使每个位置都存在差异，因此复制任何前缀从根本上来说是不安全的。 

## 方法

 暴力策略会尝试从 x + 1 开始并按顺序测试数字，检查每个候选者是否满足数字约束。 每次检查都需要扫描所有数字并逐个位置进行比较。 在最坏的情况下，我们可能需要在找到有效的整数之前检查许多连续的整数，并且每次检查都会花费位数线性的时间。 这导致在对抗性情况下，数字长度的复杂性实际上呈指数增长，因为在限制下有效数字是稀疏的。 

关键的观察结果是数字约束以非常强的方式消除了位置之间的耦合。 每个位置都可以独立选择，只要它与该位置的原始数字不同即可。 一旦我们停止尝试保持与 x 的前缀相等，通过使第一个数字大于 x 的相应数字即可满足“y > x”的要求。 

这导致了一个直接的建设性解决方案：选择第一个数字严格大于 x 的第一个数字，然后用与 x 的相应数字不同的任何数字填充其他位置。 每个位置的最小可能选择会产生全局最小的有效数字。 

该问题从全局排序约束减少到独立的每位数选择。

 ## 算法演练

 我们将输入数字视为字符串`s`。 

1. 读取第一个数字`s`并计算严格大于它的答案的最小可能前导数字。 如果第一个数字不是“9”，则只是该数字加一。 如果是“9”，我们必须使用两位数前缀的想法，逻辑上相当于生成一个以“10”开头的更长的数字，因为没有任何一位数字可以超过 9。这保证了结果数字严格大于紧邻最高有效位置的输入。 
2. 确定答案的第一位数字后，我们独立处理剩余的每个位置。 
3. 对于每个位置`i`，我们看数字`s[i]`。 我们选择其中最小的数字`0..9`那不等于`s[i]`。 由于我们按字典顺序最小化，因此我们总是更喜欢 0，除非它违反了约束，在这种情况下我们使用 1。 
4. 按顺序附加这些选定的数字以形成最终数字。 
5. 输出构造好的数字。 

每个位置都是独立处理的，因为一旦第一个数字确保严格不平等，后面的位置就不再影响“大于 x”条件。 

### 为什么它有效

 正确性在于等长数字之间的字典顺序比较完全由第一个位置决定。 通过强制结果的第一个数字超过输入的第一个数字，我们保证构造的数字严格更大，无论后缀如何。 

在那之后，唯一剩下的约束是数字的位置不平等。 由于每个位置都是独立的，并且可以完整选择 9 个有效数字，因此为每个位置选择最小的有效数字会产生最小可能的后缀。 未来不需要调整，因为没有约束连接不同的位置。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    s = input().strip()
    
    # if input is empty or single digit handled naturally
    n = len(s)
    if n == 0:
        return
    
    # construct result
    res = []
    
    # first digit handling
    if s[0] != '9':
        res.append(str(int(s[0]) + 1))
    else:
        # when first digit is 9, we cannot pick a single digit > 9
        # so we conceptually switch to "10..."
        res.append("10")
    
    # remaining digits
    for i in range(1, n):
        d = s[i]
        if d != '0':
            res.append('0')
        else:
            res.append('1')
    
    sys.stdout.write("".join(res))

if __name__ == "__main__":
    solve()
```实现直接遵循构造逻辑。 第一个数字单独处理，因为它决定了严格的顺序。 其余数字将贪婪地填充与该位置的输入数字不同的最小有效数字，该数字始终为 0 或 1。 

第一个数字中“9”的特殊情况是必要的，因为否则数字递增将超出有效的个位数范围。 产生前导“10”可以有效地增加长度并保证严格的排序。

 ## 边缘情况

 对于单个数字输入，例如`0`，算法选择下一个数字`1`，并且不存在后缀，因此结果立即正确。 

为了`9`，特殊规则触发并产生`10`，这是严格大于 9 的最小数字，也避免了相应位置的数字匹配。 

对于重复的数字，例如`1111`，第一个数字变为`2`，剩下的每个数字都变成`2`由于每个位置都必须不同于`1`，产生`2222`。 这表明即使所有位置都相同，贪婪后缀规则仍然保持一致。 

对于混合图案，例如`808`，第一个数字变为`1`，第二个变成`0`因为它不同于`0`，最后变成`1`，生产`1010`。 这证实了每个位置都是独立处理的并且约束是严格局部的。
