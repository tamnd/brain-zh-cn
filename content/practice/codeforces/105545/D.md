 ## 解决方案
 ## 问题理解

 我们有一个反复产生分数然后将它们组合起来的过程。 在某些时候，我们最终会得到以下形式的分数的集合$\frac{a_1}{b_1}, \frac{a_2}{b_2}, \dots, \frac{a_k}{b_k}$。 任务是在应用所有可能的运算后计算最终值，其中关键运算是组合分数。 

声明中隐藏的关键观察是，该过程允许重新排列或组合分数，并且我们可以自由“翻转”任何分数$\frac{a}{b}$进入$\frac{b}{a}$如果有好处的话。 目标是在所有转换后最大化结果总和，这最终减少到计算组合分数的最佳方式。 

尽管该语句是以某种压缩的代数形式编写的，但核心结构是这样的：我们对分数求和，但我们可以在求和之前反转任何分数，并且最终的组合规则的行为类似于加权合并，当分子超过分母时，它保留了单调改进属性。 

从语句中显示的不等式中，我们看到将两个分数组合成一个“合并”分数总是产生一个至少与将它们分开一样大的值。 这意味着重复合并会将所有内容折叠成一个有效分数，而最终答案仅取决于所有翻转后调整后的分子和分母的总和。 

如果我们考虑此类问题（大量分数）的典型约束，则解决方案必须在输入上以线性时间运行。 任何模拟成对合并或重复重新计算值的方法都会导致二次行为并失败。 

当某些分数大于 1 时，就会出现微妙的边缘情况。如果我们不翻转它们，它们会贡献比必要的更大的分母权重，从而降低最终结果。 例如，像这样的分数$\frac{5}{2}$应该翻转到$\frac{2}{5}$，否则最终总和中的综合比率会恶化。 

另一种边缘情况是当所有分数都已经小于或等于 1 时。在这种情况下，不会发生翻转，解决方案会简化为分子和分母的简单求和。 

## 方法

 天真的想法是按字面意思来对待这个过程。 我们采用所有分数，尝试每种可能的合并序列，并计算结果值。 每次合并都使用给定的规则将两个分数合并为一个新分数，然后继续，直到只剩下一个分数。 

这在原则上是正确的，因为它直接遵循允许的操作。 然而，每次合并都会将分数的数量减少 1，并且对于 k 个分数，有 k−1 次合并。 如果我们通过扫描或重建结构来重新计算每次合并的效果，那么在最坏的情况下，成本会成二次方。 对于较大的 k，这是不可行的。 

关键的结构见解是，合并以保留单调改进属性的方式关联：如果我们首先对每个分数进行最佳标准化，则组合分数永远不会减少最佳可实现值。 声明中提供的不等式表明，在最佳翻转下，用合并后的等值替换两个分数总是会产生一个严格优于或等于将它们分开的值。 

这意味着我们根本不需要模拟合并顺序。 相反，每个分数可以独立标准化：如果$a > b$，翻转减少了其“坏”贡献并提高了最终的聚合值。 标准化后，通过对变换后的分子和分母求和，可以安全地一次性组合所有分数。 

因此，整个过程简化为对输入的单个线性传递，对每个分数应用局部规则并聚合。

 ## 算法演练

 我们独立处理每个部分并维护代表最终合并结构的运行总计。 

1. 读每个分数$a_i, b_i$。 我们将每个分数视为对最终合并结果的独立贡献，因为一旦应用归一化，合并就不依赖于顺序。 
2.如果$a_i > b_i$，用其翻转版本替换分数$\frac{b_i}{a_i}$。 此步骤可确保每个分数的贡献不会不必要地夸大分母。 声明中的不等式保证了翻转对于最终的综合价值总是有利的。 
3. 维护两个累加器：一个用于分子，一个用于分母。 对于每个归一化分数，将其分子添加到分子累加器，并将其分母添加到分母累加器。 这代表最终的合并分数结构。 
4. 处理完所有分数后，计算最终结果，即累加分子和与累加分母和的比值。 
5. 以所需的格式输出该值。 

### 为什么它有效

 核心不变量是每一步之后，当前累积的对$(N, D)$表示相当于在最佳翻转规则下合并所有处理过的分数。 语句中给出的不等式确保将任意两个归一化分数合并为一个分数所产生的值不会比将它们分开更差，这使得聚合实际上具有关联性。 因为每个分数都是独立标准化的，以确保$a \le b$，以后的合并都不能从重新翻转中受益，因此局部最优选择组成全局最优。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    data = input().strip().split()
    k = int(data[0])
    idx = 1

    num = 0
    den = 0

    for _ in range(k):
        a = int(data[idx]); b = int(data[idx + 1])
        idx += 2

        if a > b:
            a, b = b, a

        num += a
        den += b

    print(num / den)

if __name__ == "__main__":
    main()
```该解决方案一次性读取所有分数，从而避免了重复解析或中间结构带来的任何开销。 条件交换确保每个分数都标准化，以便分子始终小于或等于分母。 这是唯一重要的本地决策，因为不平等保证了保留一小部分不翻转永远不会有好处。 

累加器直接代表合并状态。 不需要 gcd 约简或中间简化，因为最终输出是实数比率，而不是约简分数。

 ## 边缘情况

 一种重要的边缘情况是所有分数完全相等，例如$\frac{5}{5}$。 在这种情况下，翻转不会改变值，并且算法保留对称性。 用于输入`1 5 5`，累加器变为分子 5 和分母 5，按预期产生 1。 

另一种情况是当一个分数占主导地位时，例如$\frac{10^9}{1}$。 该算法将其翻转为$\frac{1}{10^9}$，防止它压倒分母之和。 如果没有这条规则，最终的比率将在预期的优化下严重倾斜和不正确。 

最后一种情况是具有单个分数的最小输入大小。 该算法直接返回$a/b$或者$b/a$，取决于比较，即使在没有合并的退化情况下，它也符合最佳翻转的定义。
