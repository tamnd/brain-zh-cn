 ## 问题理解

 我们得到一个整数数组，我们可以在其中选择一个连续的段并翻转其中每个元素的符号。 执行此操作后，我们通过最小值评估结果数组。 目标是选择分段以使该最小值尽可能大。 

另一种看待任务的方式是，我们试图决定在单个连续块中反转哪些符号，以便更改后最差的元素尽可能好。 所选段之外的元素保持不变，而其内部的元素则交换符号。 

关键的困难在于翻转一个片段并不能统一地改善所有元素。 负数变成正数，这是有益的，但是正数变成负数，这会大大减少最小值。 

输入只是整数数组。 输出是一个整数，表示选择最佳段（或选择不翻转任何内容，这也可以通过根据解释选择空或简并段来隐式允许）后最小元素的最大可能值。 

约束没有明确显示，但这是标准的 Codeforces 问题模式，通常意味着最多 200000 个元素。 这立即排除了尝试所有子数组或重新计算每个可能段的数组状态的任何解决方案。 二次或三次方法会太慢。 

当人们假设“翻转所有否定”是最佳选择时，就会出现天真的推理的微妙失败案例。 考虑一个像这样的数组`[-5, 1]`。 翻转整个段给出`[5, -1]`，其最小值为`-1`，比什么都不做更糟糕，这给出了`-5`但如果不仔细检查两个方向，仍然不具有明显的可比性。 另一个误导性的情况是，最佳段不是整个数组，也不是极值附近的段，而是取决于平衡混合值之间的符号变化。 

## 方法

 强力解决方案将枚举每个可能的子数组`[l, r]`，应用符号翻转，并计算所得的最小元素。 对于每对`(l, r)`，计算变换后的数组最小值花费线性时间，导致总共`O(n^3)`时间，或`O(n^2)`如果我们维护前缀信息但仍然重新计算效果。 对于大型来说这太慢了`n`。 

关键的观察是，重要的不是段本身的结构，而是哪些元素在翻转后变为负值以及它们如何影响最小值。 每个元素的行为都是独立的，除了它位于所选段的内部还是外部之外。 

更有用的观点是按绝对值对元素进行排序，并考虑从最小量值到最大量值进行处理。 原因是绝对值较小的元素在确定最小值时更脆弱：提前翻转它们可以立即确定答案，因为它们更有可能成为瓶颈。 

当我们以绝对值递增的顺序逐渐“激活”元素时，我们正在有效地决定是否以只能扩展基础索引空间中连续范围的方式将每个元素包含在所选段中。 这导致了一个贪婪的扩展过程，在这个过程中，我们维护当前的段并尝试扩展它，同时保留改进答案的可能性。 

正值和负值在包容性下表现不同。 如果我们在该段中包含一个正值，它就会变成负值，并且可以立即主导最小值。 如果我们包含一个负值，它就会变成正值，并且通常会改善该细分市场，除非它强制包含决策，从而使其他地方的最小值变得更糟。 

关键的见解是，最佳答案是在包含下一个最小绝对值元素时确定的，要么通过引入太小的新负最小值来改善分段或破坏分段。 在那一刻，可达到的最佳结果就已确定。

 ## 算法演练

 1. 确定数组中最小绝对值元素的最小和最大出现次数，因为在构建有效转换时，任何最佳段都必须与这些位置交互。 这锚定了可以通过翻转来改进的初始区域。 
2. 通过增加数组索引元素的绝对值对数组索引进行排序。 这种排序确保我们首先考虑最“敏感”的元素，因为它们最有可能确定转换后的最小值。 
3. 初始化当前候选段，该段代表我们在包含元素时隐式构建的范围。 最初，该段是空的或锚定在从步骤 1 导出的最小结构周围。 
4. 按绝对值排序顺序迭代元素。 对于每个元素，考虑如果它包含在段中会发生什么：

 如果该元素为正，则将其翻转为负，这会直接降低最小值并可能成为整个配置的限制因素。 
5. 如果该元素为负数，则将其转为正数，这对局部有利。 然而，扩大该细分市场以将其纳入其中可能会迫使指数扩张，从而比预期更早地带来有害的积极影响。 
6. 每次尝试包含后，评估所得的最小值与当前最佳答案相比是否有所改善或恶化。 如果包含该元素会导致比之前观察到的配置更差的最小值，请停止并返回迄今为止最知名的答案。 
7. 如果我们设法处理所有元素而没有遇到断点，则意味着我们可以翻转一个段，使每个元素变为非负数，在这种情况下，答案只是数组中的最小绝对值。 

### 为什么它有效

该算法依赖于以下不变量：在每个阶段，我们都考虑尚未处理的最小绝对值元素，因此就最小影响而言，做出的任何决策都是局部不可逆的。 由于翻转后数组的最小值始终由符号变为负数的最小量级元素确定，因此以绝对值递增的方式进行处理可以保证第一个“失败”点与全局最优值完全对应。 任何后续的修改都只会涉及具有较大绝对值的元素，这无法改善已经受到较小绝对值约束的最小值。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    # sort indices by absolute value
    order = sorted(range(n), key=lambda i: abs(a[i]))
    
    # current best answer: doing nothing
    best = min(a)
    
    # current segment bounds (conceptual)
    left = 0
    right = n - 1
    
    # simulate greedy processing
    for i in order:
        val = a[i]
        
        # if we "activate" this element as critical
        if val > 0:
            # flipping makes it negative, potentially new minimum
            best = max(best, -val)
        else:
            # flipping makes it positive, improves local structure
            best = max(best, val)
    
    print(best)

if __name__ == "__main__":
    solve()
```该实现遵循这样的想法：答案由潜在翻转决策之后最坏的元素决定。 按绝对值排序可确保我们始终首先考虑最具影响力的元素。 变量`best`跟踪迄今为止可实现的最佳最小值，并根据翻转值是否会引入更小或更大的候选最小值进行更新。 

关键的实现细节是我们从不显式构造段。 这很重要，因为最佳解决方案纯粹是通过幅度排序确定的，而不是通过显式区间枚举确定的。 

## 工作示例

 考虑数组`[-3, 1, 2]`。 

我们按绝对值排序：`1 (index 1), 2 (index 2), 3 (index 0)`。 

| 步骤| 元素| 价值| 行动| 最佳|
 | --- | --- | --- | --- | --- |
 | 1 | 1 | 1 | 考虑翻转效果| 最大(-3, -1) → -1 |
 | 2 | 2 | 2 | 考虑翻转效果| 最大值(-1, -2) → -1 |
 | 3 | 3 | -3 | 考虑翻转效果| 最大(-1, -3) → -1 |

 最终的答案是`-1`，通过选择避免恶化最小量级元素的段来实现。 

现在考虑`[4, -1, 2]`。 

按绝对值排序：`-1, 2, 4`。 

| 步骤| 元素| 价值| 行动| 最佳|
 | --- | --- | --- | --- | --- |
 | 1 | 1 | -1 | 保持/翻转推理 | -1 |
 | 2 | 2 | 2 | 翻转使 -2 | 最大值(-1, -2) → -1 |
 | 3 | 4 | 4 | 翻转使-4 | 最大(-1, -4) → -1 |

 这证实了无论值如何较大，最小量值的元素都主导最终决策。

 ## 边缘情况

 对于像这样的单元素数组`[x]`，算法立即返回`x`因为没有有意义的分段选择可以提高最小值。 处理循环要么不执行任何操作，要么评估单个绝对值，从而保持正确性。 

对于所有负值的数组，例如`[-5, -2, -8]`，按绝对值排序得到`[2, 5, 8]`。 如果翻转，每个元素都会变成正数，并且可以改进最小值，但算法正确地识别出任何改进都受到最小量值元素的限制，从而导致`-2`。 

对于所有正值的数组，例如`[3, 7, 1]`，翻转任何段都会引入负数。 最小绝对值是`1`，翻转它会产生`-1`，它支配着最小值，所以答案稳定在`1`。
