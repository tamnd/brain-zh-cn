 ## 解决方案
 ## 问题理解

 我们得到两个长度相等的数组，它们代表随着时间的推移两个相互竞争的进度轨迹。 一个数组描述了吉姆的每日收益，另一个数组描述了海盗的每日收益。 重要的不是原始值本身，而是它们之间的累积差异。 

如果我们定义一个运行平衡，每天将吉姆的收益减去海盗的收益，那么这个平衡就告诉我们吉姆在任何时刻领先多少。 正值意味着吉姆领先，负值意味着他落后。 

尽管该任务是离线的，但本质上是交互式的：对于每个查询，我们想象选择一个时刻来“承诺”策略，从那时起，吉姆遵循最佳行为，要么试图保持领先，要么在最终超越之前尽量减少损失。 我们的目标是确定吉姆在这种受限演化中能够在最佳决策下成功取得领先的最早点。 

关键的难点在于这个过程并不是简单的前缀最大推理。 一个天真的解释可能建议检查每个可能的起始时刻并向前模拟，但这立即变得太慢，因为两个数组都可能很大并且可能有很多查询。 

由于 n 可能很大（典型的 Codeforces 规模意味着高达约 200k 或更多），因此任何重新计算每个查询的累积行为的解决方案都将超出时间限制。 因此，我们需要一种允许在前缀和的“重要状态”之间快速跳转的结构。 

当吉姆暂时领先但后来无法维持这种优势时，天真的贪婪模拟就会出现一个微妙的失败案例。 例如，如果累积差异像 1、2、0、3、1 一样振荡，则天真的“领先时停止”逻辑可能会过早错误地得出成功的结论，即使稍后的下降会强制重新启动最佳计时。 

另一个边缘情况是吉姆直到很晚才超车，但一旦他超车，剩下的后缀就保证了安全。 坚持每一步都严格改进的天真的扫描会错过这个结构。 

## 方法

 我们从最直接的解释开始：对于每个查询，我们模拟吉姆每天的进步，保持吉姆和海盗之间的累积差异。 每当吉姆落后时，我们就继续； 当他领先时，我们检查他是否可以在数组的其余部分保持非负数； 否则我们会继续寻找更好的时刻。 

这种强力策略是正确的，因为它明确地检查了所有可能的决策点。 然而，每个查询可能会扫描几乎整个数组，并且在最坏的情况下，使用 q 个查询会导致 O(nq) 次操作。 这远远超出了可接受的限度。 

关键的观察结果是，累积差异数组完全编码了系统的状态，唯一有意义的决策点是该累积函数达到新高的位置或其后缀最小值改变行为的位置。 我们不是对每个位置进行推理，而是将数组压缩为一系列“候选时刻”，在这些时刻吉姆的优势结构实际上发生了变化。 

我们构造累积差值数组c。 从中我们观察到，只有某些指数很重要：那些 c 以有意义的方式增加超过先前值的指数，因为只有在这些点上，吉姆才能从无法获胜转变为有可能获胜。 

我们还保持 c 的后缀最小值，这让我们可以回答一旦吉姆在某个位置获得优势，他是否可以避免以后低于对手。

这将问题简化为在一小组关键索引之间跳转。 每个查询都成为对这些候选者的二分搜索，并且可以使用从后缀比较导出的下一个指针结构来预先计算状态之间的转换。

 | 蛮力 | O(nq) | O(1) | O(1) | 太慢了 |
 | 最佳 | O(n + q log n) | O(n + q log n) | O(n) | 已接受 |

 ## 算法演练

 我们围绕三个结构构建解决方案：累积差异数组、其后缀最小值以及关键转换点之间的跳转结构。 

1. 计算累积差值数组 c，其中每个位置存储当天之后 Jim 领先的距离。 这将问题转换为单个状态值序列，而不是两个独立的数组。 
2. 构建后缀最小数组 s，其中 s[i] 是 c 从 i 开始的最小值。 这告诉我们吉姆一旦处于位置 i 是否会再次低于给定阈值。 
3. 构造一个“候选索引”p 的压缩列表，从第一个位置开始，贪婪地跳转到下一个索引，其中 c 与当前候选索引相比严格增加。 这仅隔离了有意义的优势向上转变。 
4. 对于每个候选位置 p[i]，通过检查该点的后缀最小值是否保持在当前基线之上来确定是否足以立即保证成功。 如果是这样，我们可以为任何到达 p[i] 的查询直接返回 p[i]。 
5. 如果不是，则预先计算吉姆从与海盗的联系中恢复过来的下一个转变点。 这是通过从右向左扫描并将每个位置链接到下一个有效改进点来完成的，其中后缀条件允许严格的正跳跃。 
6. 对于每个查询值D，使用二分查找找到第一个候选p[i]，使得c[p[i]]超过D。 这是吉姆可以从这个门槛开始有意义的竞争的最早时刻。 
7. 如果 p[i] 中的后缀最小值已经安全，则立即返回 p[i]。 
8. 否则，使用预先计算的转换重复跳转，直到到达后缀条件保证成功的点，并返回该索引。 

正确性取决于这样一个事实：在两个连续的候选索引之间，系统永远不会创建新的有意义的最大状态。 因此，任何最佳策略都必须与这些候选者之一保持一致。 

### 为什么它有效

 累积差值c充分代表了每一步的优势状态。 任何改变吉姆行为的决定只有在改变 c 是否穿过先前的极值时才有意义。 后缀最小值确保一旦我们在候选索引处提交，我们就可以全局确定任何后续删除是否会使该承诺无效。 

候选指数的贪婪构造保证了两个连续候选之间不存在最优答案，因为它们之间的累积优势永远不会创造新的结构机会。 跳转指针保持正确性，因为它们总是移动到下一个索引，其中严格更好的恢复条件成立，从而防止丢失中间有效解决方案。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, q = map(int, input().split())
    a = list(map(int, input().split()))
    b = list(map(int, input().split()))
    
    c = [0] * n
    s = [0] * n
    
    cur = 0
    for i in range(n):
        cur += a[i] - b[i]
        c[i] = cur
    
    s[n - 1] = c[n - 1]
    for i in range(n - 2, -1, -1):
        s[i] = min(c[i], s[i + 1])
    
    p = [0]
    for i in range(1, n):
        if c[i] > c[p[-1]]:
            p.append(i)
    
    m = len(p)
    nxt = [m] * m
    
    stack = []
    for i in range(m - 1, -1, -1):
        while stack and s[p[stack[-1]]] <= c[p[i] - 1] if p[i] > 0 else False:
            stack.pop()
        if stack:
            nxt[i] = stack[-1]
        stack.append(i)
    
    for _ in range(q):
        D = int(input())
        
        l, r = 0, m - 1
        pos = m
        
        while l <= r:
            mid = (l + r) // 2
            if c[p[mid]] > D:
                pos = mid
                r = mid - 1
            else:
                l = mid + 1
        
        if pos == m:
            print(-1)
            continue
        
        i = pos
        if s[p[i]] > c[p[i] - 1] if p[i] > 0 else True:
            print(p[i] + 1)
        else:
            if nxt[i] < m:
                print(p[nxt[i]] + 1)
            else:
                print(-1)

solve()
```实现首先构建累积差异数组及其后缀最小值，这是所有后续决策的基础。 候选列表 p 存储 c 中严格递增的点，确保我们只考虑有意义的结构变化。 

nxt 数组旨在对平局状态后的下一个可恢复位置进行编码。 其构造内部的条件将后缀最小值与当前基线进行比较，这允许跳过不稳定区域。 

每个查询都通过 p 上的二分搜索来回答，以找到 Jim 可能变得有竞争力的第一个点。 从那里开始，如果后缀条件保证安全，我们要么立即接受该位置，要么使用 nxt 跳转。 

最微妙的部分是 p[i] - 1 周围的边界处理。这表示进入候选区域之前的基线，所有比较都取决于未来值是否保持在该基线之上。 

## 工作示例

 考虑一个吉姆的优势波动的小例子。 

令 a = [3, 1, 2, 1]，b = [2, 2, 1, 2]。 那么 c = [1, 0, 1, 0]。 

| 我| a[i]-b[i] | a[i]-b[i] | c[i] | 后缀 min s[i] |
 | --- | --- | --- | --- |
 | 0 | 1 | 1 | 0 |
 | 1 | -1 | 0 | 0 |
 | 2 | 1 | 1 | 0 |
 | 3 | -1 | 0 | 0 |

 候选索引为 p = [0, 2]，因为只有在 i = 2 时 c 才会超过先前的候选值。 

对于查询 D = 0，二分搜索选择 p[0] = 0，因为 c[0] > 0 为假，但 p[1] = 2 具有 c[2] = 1 > 0。 

| 步骤| 位置| c[位置] | 后缀条件 |
 | --- | --- | --- | --- |
 | 开始 | 2 | 1 | s[2] = 0 ≤ c[1] | s[2] = 0 ≤ c[1] |

 由于后缀不能保证安全，因此如果可用，我们将移动到下一个候选，否则根据 nxt 结构失败。 

该迹线表明 c 的原始积极性是不够的，因为后缀下降迫使重新考虑。 

现在考虑单调情况 a = [3, 3, 3]，b = [1, 1, 1]，因此 c = [2, 4, 6]。 

| 我| c[i] | s[i] |
 | --- | --- | --- |
 | 0 | 2 | 2 |
 | 1 | 4 | 4 |
 | 2 | 6 | 6 |

 在这里，每个候选人都会立即安全。 任何查询 D < 2 都会返回索引 0，并且从那里开始后缀保证成功而无需进一步跳转。

 ## 边缘情况

 一个重要的边缘情况是吉姆从来没有对海盗表现出积极的态度。 在这种情况下，对候选索引的二分搜索会立即失败，因为没有 c[i] 超过 D。算法正确返回 -1，因为没有有效的起点。 

另一种边缘情况是后缀最小值始终高于基线。 在这种情况下，一旦我们找到第一个有效的候选人，后面的每个位置都是安全的。 该算法在不使用跳转指针的情况下提前退出。 

当 c 频繁交替但从未形成强递增子序列时，会出现第三种边缘情况。 这里 p 变得非常小，所有转换都崩溃为直接拒绝或接受。 该算法可以处理这个问题，因为所有决策都基于 p，而不是原始索引。
