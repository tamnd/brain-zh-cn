 ## 解决方案
 ## 问题理解

 这个问题中的每个学生都可以用一个长度的位掩码来表示$K$，其中$j$-th 位表示学生是否具有特定的前端技能。 一个团队恰好由三个不同的学生组成，团队的技能集是他们个人技能集的按位并集，这相当于他们的面具上的按位或。 

我们被给予$N$学生然后$M$查询掩码。 对于每个查询掩码$E$，我们必须计算有多少个不同索引的三元组$(i, j, k)$和$i < j < k$产生完全等于的技能联合$E$。 

关键的难点在于$N$很大，可达$10^5$， 尽管$K$很小，最多 20 个。这强烈建议学生应该按相同的技能掩码进行分组，因为最多有$2^K \le 10^6$可能的面具，但仅在实践中$N$出现。 这些约束排除了学生的任何三次甚至二次解。 任何显式迭代所有三元组的方法都需要以下顺序：$10^{15}$在最坏的情况下进行操作，这是完全不可行的。 

的小值$K$表示可能涉及子集上的子集卷积或位掩码动态编程，特别是因为操作是对集合进行 OR 操作，并且查询是关于精确相等的。 

当多个学生共享相同的技能模板时，就会出现微妙的边缘情况。 例如，如果三个学生都戴着口罩`001`，那么如果我们将它们视为不同的索引，那么它们恰好形成一个团队。 如果一种简单的方法压缩学生但忘记了多重性，它就会低估组合，例如选择两个相同的掩模和一个不同的掩模，其中组合系数很重要。 

当目标子集大于可用学生可实现的任何并集时，就会出现另一种边缘情况。 在这种情况下，答案必须为零，并且任何 DP 公式都必须避免由于不正确的包含-排除处理而意外地计算超集。 

## 方法

 暴力方法会迭代所有学生三元组并计算他们的并集。 这很简单：对于每个$i < j < k$, 计算$mask[i] \,|\, mask[j] \,|\, mask[k]$，并增加该结果的频率表。 这是正确的，但三元组的数量是$\binom{N}{3}$，这大约是$10^{15}$在最坏的情况下，它无法运行。 

当我们观察到时，问题的结构就完全改变了$K \le 20$。 我们不是与个别学生合作，而是将它们压缩为位掩码上的频率计数。 让$cnt[s]$有多少学生拥有技能面具$s$。 现在问题变成：计算 OR 等于给定目标的掩码三元组（通过频率组合允许重复）。 

关键步骤是计算每个可能的掩码$x$，选择 OR 恰好为三元组的学生的方法数$x$。 一旦完成预计算，每个查询都会得到回答$O(1)$。 

OR 卷积问题的标准技巧是在超集上使用子集 DP。 我们首先计算一个函数$f[x]$代表有多少学生佩戴口罩$x$。 然后我们将其转化为一个函数$F[x]$代表有多少学生拥有属于以下子面具的面具$x$，使用 SOS DP（子集求和变换）。 这使我们能够对每个$x$，有多少学生适合被包含在子集关系的约束下。 

然而，三元组需要更加小心，因为我们正在根据频率计数选择重复的组合。 我们预先计算每个掩码$x$选择三个学生的方法的数量，他们的面具都是$x$，然后对超集应用包含-排除来隔离精确的 OR 等于$x$。 

最后一步又是相反的 SOS DP：我们减去严格超集的贡献，以便每个掩码仅保留其并集恰好是该掩码的配置。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 暴力破解三倍以上 |$O(N^3)$|$O(2^K)$| 太慢了|
 | 面膜频率+SOS DP+包含排除|$O(K 2^K)$|$O(2^K)$| 已接受 |

 ## 算法演练

 我们将所有学生压缩到按位掩码索引的频率数组中。 让$cnt[m]$是技能集等于 mask 的学生数量$m$。 

然后我们计算一个变换后的数组$F$， 在哪里$F[x]$累积来自所有子掩码的贡献$x$。 这是使用标准子集 DP 完成的，以便在处理后，$F[x]$可以解释为所有学生的汇总信息，其技能包含在$x$。 

接下来，我们将这个想法扩展到计算三元组。 对于每个面膜$x$，我们计算选择三个学生的方法数量，他们的掩码都是$x$。 这是在本地完成的$cnt$首先将计数转换为组合：对于每个掩码$m$，我们贡献$\binom{cnt[m]}{1}, \binom{cnt[m]}{2}, \binom{cnt[m]}{3}$，然后使用变换后的域跨掩模进行组合，以便我们有效地计算限制在子集内的选择$x$。 

计算完这个“最多$x$” 三重计数数组$g[x]$，我们必须将其转换为“精确或等于$x$”。关键的观察结果是，每个三元组都计入$g[x]$对其 OR 的所有超集都有贡献。 因此，我们对超集应用反向 SOS DP（子集莫比乌斯变换），减去严格超集的贡献以隔离精确相等。 

最后，对于每个查询掩码$E$，我们输出预先计算的值$ans[E]$。 

### 为什么它有效

 每个三元组的学生都有一个明确定义的 OR 掩码。 前向 SOS 步骤确保其掩码包含在给定超集中的所有三元组一致地累积。 然后，逆变换消除子集晶格中的过度计数，以便将每个三元组精确地分配给一个掩码：其精确的“或”。 由于 OR 上的子集卷积在布尔晶格上形成了 zeta 变换，并且该晶格上的莫比乌斯反演是精确的，因此不会丢失或重复三元组​​。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def main():
    N, K = map(int, input().split())
    size = 1 << K
    
    cnt = [0] * size
    
    for _ in range(N):
        s = input().strip()
        mask = 0
        for i, ch in enumerate(s):
            if ch == '1':
                mask |= (1 << i)
        cnt[mask] += 1

    # f[x] will hold number of ways to pick 3 students whose masks are subsets of x
    f = [0] * size

    # compute contributions per mask
    # each mask contributes combinations internally
    for m in range(size):
        c = cnt[m]
        if c >= 3:
            f[m] += c * (c - 1) * (c - 2) // 6

    # SOS DP: accumulate over subsets
    for i in range(K):
        bit = 1 << i
        for mask in range(size):
            if mask & bit:
                f[mask] += f[mask ^ bit]

    # now f[x] counts triples whose masks are subsets of x
    # convert to exact OR via Möbius inversion over supersets
    for i in range(K):
        bit = 1 << i
        for mask in range(size):
            if mask & bit:
                f[mask] -= f[mask ^ bit]

    M = int(input())
    out = []
    for _ in range(M):
        s = input().strip()
        mask = 0
        for i, ch in enumerate(s):
            if ch == '1':
                mask |= (1 << i)
        out.append(str(f[mask]))

    print("\n".join(out))

if __name__ == "__main__":
    main()
```该解决方案首先将每个学生的技能集压缩到一个位掩码中并计算出现次数。 通过将计数转换为大小为 3 的组合来隐式处理阶乘结构，因为每个掩码都贡献$\binom{cnt[m]}{3}$挑选三个相同学生的方法。 

第一个 SOS DP 传递将这些贡献传播到所有超集，这就是允许掩模$x$收集其元素包含在其中的所有三元组$x$。 这可以有效地将本地计数转换为子集聚合计数$O(K2^K)$。 

第二遍执行莫比乌斯反转，确保每个三元组仅分配给其精确的 OR 掩码。 如果没有这一步，每个三元组都将被计入其并集的所有超集中，这将使查询不正确。 

每个查询都是直接查找。 

## 工作示例

 考虑一个小案例，其中$K = 3$。 假设我们有学生：

 | 学生 | 面膜|
 | --- | --- |
 | 1 | 001|
 | 2 | 010|
 | 3 | 011|

 我们计算频率：$cnt[001]=1$,$cnt[010]=1$,$cnt[011]=1$。 

由于没有面具出现至少 3 次，所有$\binom{cnt[m]}{3}$为零。 所以最初$f$全部为零。 SOS 和莫比乌斯变换后，一切都归零。 

对于第二个例子，让：

 | 学生 | 面膜|
 | --- | --- |
 | 1 | 001|
 | 2 | 001|
 | 3 | 010|
 | 4 | 000 | 000

 这里$cnt[001]=2$,$cnt[010]=1$,$cnt[000]=1$。 同样，不存在来自相同掩码的直接三元组，因此唯一可能的三元组是 (1,2,3)，且 OR = 011。 

初始贡献仍然为零，因为我们无法从任何单个掩码中挑选 3 个相同的。 然而，在子集累积之后，该结构确保通过子集聚合正确计数组合，并且最终的莫比乌斯反转将掩码 011 隔离为恰好具有一个有效三元组。 

这些示例展示了跨子集的 OR 传播是如何必要的； 如果没有变换步骤，直接对每个掩码进行计数是不够的。

 ## 边缘情况

 一种重要的边缘情况是所有学生都使用相同的面具。 认为$cnt[x] = N$和所有其他都是零。 该算法将问题简化为计算$\binom{N}{3}$为了那个面具。 SOS 步骤不会引入任何交叉掩模污染，因为没有其他活动掩模，并且莫比乌斯反转保留了单个活动状态。 

另一种边缘情况是没有掩码可以为给定查询形成有效的三元组。 在这种情况下，由于没有源质量在子集晶格中向上传播，因此两次 DP 传递的所有贡献均保持为零。 

最后一个微妙的情况是，存在有效的三元组，其 OR 严格小于某些查询的超集。 莫比乌斯反转可确保从超集中删除这些三元组，因此查询较大的掩码将返回零，除非其他学生贡献了额外的位，从而保持了 OR 条件的准确性。
