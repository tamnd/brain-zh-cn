 ## 解决方案
 ## 问题理解

 我们给出一个多项式，其中每个系数不是 0 就是 1，因此该多项式最好被视为 x 的一组幂。 如果系数为$x^k$是 1，我们包括该项，否则我们不包括。 多项式通过确定性过程演化，该过程仅根据常数项是否存在而重复应用两个运算之一。 

如果存在常数项，则通过将其乘以多项式来进行变换$x+1$，然后加 1，最后进行简化，删除任何等于 2 的系数。 由于系数永远不会超过 2，因此这种简化可以有效解决项出​​现两次的冲突。 如果不存在常数项，运算就会简单得多：我们将多项式除以 x，这相当于将每个指数向下移动 1。 

重复该过程，直到多项式恰好变为 1，这意味着仅保留常数项，并且不存在更高阶项。 

主要困难在于每一步都会以非线性方式改变多项式的结构。 乘以$x+1$两者都会改变和混合现有项，额外的“+1 与取消”引入了常数项上的交互，打破了简单的单调性论证。 

约束很小：初始次数最多为 20，因此多项式可以表示为最多 21 位的位掩码。 尽管不能保证过程立即收缩，但状态空间足够小，只要使用位运算有效地实现每个步骤，就可以直接模拟变换。 

一个微妙的边缘情况是当多项式已经是 1 时。在这种情况下，不应执行任何操作。 另一个边缘情况是，当重复乘法创建比初始界限更高的次数项时，因此假设长度为 21 的固定大小数组没有溢出处理的实现将默默地删除项或错误索引。 第三个问题是对“丢弃系数2”规则的误解，该规则不是全局算术模2，而是只有在结合乘法和加法的贡献后才能有效消除。 

## 方法

 蛮力方法正是该语句所描述的：显式表示多项式，执行多项式乘法$x+1$，加1，然后扫描所有系数并将任意2设置回0，或者当没有常数项时右移。 每次操作费用$O(n)$在哪里$n$是当前学位。 该过程可能需要大量步骤，因为变换可以增加和减少程度，并且不存在保证快速收敛的明显单调潜力。 在最坏的情况下，如果使用朴素多项式算法实现，则对可能的许多中间配置进行逐步模拟会变得低效。 

关键的观察是，因为系数是二进制的，所以多项式可以表示为位掩码，并且所有运算都变成按位变换。 乘以$x$是左移，加法相当于异或。 这将整个过程转换为整数状态机。 一旦这样写，每一步都是$O(1)$位运算，我们简单地进行模拟，直到达到 1。状态空间很小，因为对于给定的约束，多项式在实践中永远不会变得比几十位有意义地大，因此在模拟中收敛速度很快。

 ## 算法演练

 我们将多项式编码为整数，其中位 i 表示存在$x^i$。 

1. 读取输入多项式并构造其位掩码表示。 每个系数 1 设置相应的位。 这给出了一种紧凑的编码，其中代数移位变成了比特移位。 
2. 重复直到位掩码等于 1。值 1 对应于仅包含常数项的多项式。 
3. 如果当前位掩码设置了最低有效位，则应用“乘以 (x+1) 然后加 1”运算。 用位术语来说，与 GF(2) 类似的取消行为相乘 (x+1) 变为`mask XOR (mask << 1)`。 附加的“+1”再次翻转常数项，因此我们与 1 进行异或。这种组合正确地捕获了项出现两次时如何取消。 
4. 如果未设置最低有效位，则执行除以 x，这相当于将整个位掩码右移一位。 
5. 将每个转换算作一次操作。 

模拟继续进行，直到只剩下常数项。 

### 为什么它有效

 多项式结构完全由项的存在或不存在来捕获，并且每次变换仅取决于常数项是否存在。 这使得演化在有限的二元状态空间上具有确定性。 将多项式表示为位掩码可以保留未来步骤所需的所有信息，因为除了显式处理的常数项切换之外，这两种操作在位位置上都是线性的。 因此，每个模拟的转换与原始过程的一个步骤完全匹配。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    coeffs = list(map(int, input().split()))
    
    mask = 0
    for i, a in enumerate(reversed(coeffs)):
        if a:
            mask |= 1 << i

    steps = 0

    while mask != 1:
        if mask & 1:
            # multiply by (x + 1) and add 1, with cancellation behavior
            mask = (mask ^ (mask << 1)) ^ 1
        else:
            # divide by x
            mask >>= 1
        steps += 1

    print(steps)

if __name__ == "__main__":
    solve()
```该实现以递增的指数顺序从系数数组构建位掩码。 核心循环使用以下命令检查常数项`mask & 1`。 如果存在，则该转换结合左移和异或来模拟乘法$x+1$，然后翻转常数项。 否则，右移实现除以 x。 循环计数器直接跟踪操作次数。 

一个微妙的实现细节是确保在构建位掩码时反转系数顺序，因为输入是从最高次到常数项给出的。

 ## 边缘情况

 一种重要的边缘情况是多项式从 1 开始。该算法立即检测到`mask == 1`并执行零迭代，正确匹配不需要操作的定义。 

另一种情况是多项式没有常数项。 例如，$x^3$立即触发重复除以 x。 掩码向右移动，直到变为 1，这正好对应于 x 的一一剥离幂。 

最后一种情况是重复乘法引入更高次项。 例如，从一个小的多项式开始可以生成一个位超出原始次数的掩码。 由于表示是无界整数，因此不会发生溢出或截断，并且模拟保持一致。
