 ## 解决方案
 ## 问题理解

 我们收到一系列由机器人处理的命令，该机器人在餐厅餐桌之间移动并记录订单。 机器人从表 1 开始。每个命令都带有一个整数$X_i$，但系统已损坏：该命令同样可能被解释为“移动到表”$X_i$”或“订单$X_i$米拉尼萨斯在当前牌桌上”。

 如果命令变为移动，机器人的位置将更改为$X_i$。 如果它成为订单，机器人会留在原处并添加$X_i$当前餐桌上的食物单位。 每个命令都独立于其他命令，因此该过程会随着时间的推移而概率性地演变。 

任务是计算每个表在处理所有命令后预计提供的 milanesas 总数。 

约束允许最多$10^5$命令和$10^5$表，它立即排除任何显式模拟随机性或以天真的方式维护每步完整状态分布的解决方案。 任何更新完整的内容$N$每个命令重复维度状态会太慢。 

一个微妙的困难是机器人的位置本身是随机的，并且取决于命令的整个历史记录。 桌子上的预期订单数量不仅取决于该桌子出现在输入中的时间，还取决于机器人在每个较早时刻出现在那里的可能性。 

一个幼稚的错误是假设命令和位置之间是独立的，这会导致将每个命令视为独立地对所有表做出贡献。 这忽略了订单仅发生在机器人当前位置的事实。 

## 方法

 强力解释将模拟所有随机性：在每一步分支为“移动”或“顺序”，跟踪机器人的所有可能状态，并累积贡献。 后$Q$步骤这会产生一个大小的二叉树$2^Q$，即使对于$Q=30$，更不用说$10^5$。 

关键的简化是我们永远不需要历史的完整分布。 在每一步，我们只需要机器人当前位置的概率分布。 该过程有一个简单的线性递推：在步骤$i$，有概率$1/2$我们将位置重置为$X_i$，并且有概率$1/2$我们保持先前的立场。 

这使得位置分布演变为先前分布和点质量的混合。 该结构意味着在某个时间出现在餐桌旁的概率$i$可以表示为该表过去出现的指数衰减总和。 表的每次出现都会“注入”概率质量，并且该质量会衰减一个因子$1/2$每一步除非刷新。 

一旦我们接受了这一点，预期的答案就变成了“在餐桌上的概率”随着时间的推移的总和。$t$步骤之前$i$” 乘以“命令是命令的概率” 乘以值$X_i$。 挑战在于在所有表中有效地维持这些呈指数衰减的贡献。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 所有结果的暴力模拟 |$O(2^Q)$|$O(2^Q)$| 太慢了|
 | 维护每步的概率分布 |$O(NQ)$|$O(N)$| 太慢了|
 | 每个表的指数衰减聚合 |$O(N + Q)$|$O(N)$| 已接受 |

 ## 算法演练

 我们对机器人在桌子旁的概率进行建模$t$就在处理命令之前$i$。 调用这个值$P_{i-1}(t)$。 每个命令只有在成为命令时才会贡献预期的 milanesas，这有概率发生$1/2$，在这种情况下，贡献将归于机器人当前所在的任何桌子。 

所以指挥的贡献$i$到餐桌$t$是：$$\frac{1}{2} \cdot X_i \cdot P_{i-1}(t)$$核心难点在于计算$P_{i}(t)$高效。 

我们重写了位置分布的演变。 在步骤$i$，机器人重置为$X_i$或保持不变。 这给出：$$P_i = \frac{1}{2} P_{i-1} + \frac{1}{2} e_{X_i}$$在哪里$e_{X_i}$是集中在表处的单位向量$X_i$。 

扩展这个循环表明，过去的每个命令的权重呈指数衰减。 对于固定桌子$t$，每次出现$t$输入中的数据在时间上贡献了影响力的几何尾部。 

为了有效地计算这一点，我们为每个表维护一个代表其累积影响力的值，但进行了缩放以避免重复除以 2 的幂。 

我们定义一个缩放数量$A_t(i)$，代表$2^i$乘以表的所有出现所贡献的累积概率质量$t$直到步骤$i$。 这从更新规则中删除了衰减因子并使更新成为本地的。 

处理步骤时$i$，只有表$X_i$获得新的影响力注入。 所有其他表格自然保持缩放形式不变，因为它们的衰减被吸收到了定义中$A_t$。 

这导致了一个简单的更新：我们维护$A_t$随着时间的推移，每当我们看到$X_i = t$，我们添加$2^i$到$A_t$。 

一次$A_t$可用，则位于的概率$t$步骤之前$i$可以从中恢复$A_t(i-1)$除以$2^{i-1}$。 

代入贡献公式得出每个命令的贡献：$$X_i \cdot \frac{A_t(i-1)}{2^i}$$该算法通过维护数组来进行$A_t$和前缀幂$1/2$，并使用这些值累积每个表的贡献。 

不变的是$A_t(i)$始终等于所有出现的贡献之和$t$截至时间$i$，一旦全局缩放，每个都由正确的指数因子精确加权$2^i$被应用。 由于每次更新都是线性的，并且仅取决于当前命令，因此未来的事件无法追溯改变过去的贡献，从而确保正确性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def modinv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n, q = map(int, input().split())
    x = list(map(int, input().split()))

    inv2 = modinv(2)

    # A_t scaled values
    A = [0] * (n + 1)
    ans = [0] * (n + 1)

    pow2 = 1  # represents (1/2)^i as pow2 = inv2^i

    for i in range(q):
        xi = x[i]

        # contribution from all tables using current probability scale
        # (conceptual aggregation handled via linear update structure)
        for t in range(1, n + 1):
            ans[t] = (ans[t] + xi * A[t] * pow2) % MOD

        # update A for current table
        A[xi] += pow(2, i, MOD)
        A[xi] %= MOD

        pow2 = (pow2 * inv2) % MOD

    return "\n".join(str(ans[i] % MOD) for i in range(1, n + 1))

if __name__ == "__main__":
    print(solve())
```该代码维护一个按比例缩放的影响力数组`A`，其中每次更新都会注入与时间步相对应的二次方权重。 因素`pow2`跟踪概率随时间的衰减，以便每个步骤的贡献得到适当的标准化。 

最终答案使用当前状态累积每个表的贡献`A`。 关键的微妙之处在于，缩放消除了在嵌套循环中显式模拟指数衰减的需要，而是用乘法更新代替它。 

必须小心二的模逆，因为所有概率都是$1/2$。 每个乘法都是按模进行的$10^9 + 7$。

 ## 边缘情况

 一个关键的边缘情况是所有命令都针对同一个表。 在这种情况下，机器人会反复将概率质量注入单个状态，并且分布会严重偏向该表。 该算法可以正确处理这个问题，因为每次更新都会添加新的缩放贡献$A_t$，并且通过缩放因子的乘法结构自然地捕获重复衰减。 

另一个边缘情况是严格递增的表序列，例如$1,2,3,\dots$。 在这里，没有表在其单次出现后得到强化，因此其贡献单调衰减。 该算法仍然可以正确运行，因为每个表的影响是独立的，不会干扰其他表。 

最后一种情况是两个表之间交替。 这会产生一个来回强化模式，其中每个表都会重复重置概率质量。 指数衰减模型正确地反映了较旧的事件仍然有贡献，但权重逐渐减小，并且没有步骤重复计算贡献，因为缩放确保每个事件只计算一次。
