 ## 解决方案
 ## 问题理解

 我们从一个数组开始，其第 i 个位置最初包含 i。 然后我们执行一系列 n − 1 次随机更新。 在第 i 次更新时，我们查看数组的缩小前缀，从当前活动范围的右端获取特定元素，并将其值的两倍添加到早期索引中的均匀随机位置。 

具体地，在步骤i中，位置n-i+1处的元素被用作“源”。 我们从 1 到 n − i 中统一选择一个索引 j，并将源当前值的 2 倍添加到 a[j] 中。 此后，源位置将不再用作目标，但较早的位置仍可能作为源参与未来的操作。 

我们要求的数量是所有这些随机操作之后 a1 的值，对 998244353 取模。因为每一步都涉及随机性，所以这是一个随机变量。 预期的解释是它的期望值，因为只有在对 j 的所有选择进行期望之后，最终结果才是确定性的。 

约束 n ≤ 10^9 以及最多 10^4 个测试用例会立即排除任何模拟，甚至排除每个测试用例的线性 DP。 任何解决方案都必须将整个过程折叠成仅依赖于 n 的封闭式表达式。 

一个微妙的问题是值会多次传播。 早期添加的元素稍后可能被选择作为源本身，然后再次传播其累积值。 除非我们正确地考虑到整个过程中的重复扩增，否则天真的“单次累积”就会变得不正确。 

一个小的健全性检查显示了为什么随机性很重要。 对于 n = 2，唯一的操作是强制的，a1 变为 1 + 2·2 = 5。任何正确的解释都必须与这个确定性结果相匹配。 

## 方法

 直接模拟将明确执行每个操作，随机选择 j 并更新数组。 这原则上是正确的，但每个测试用例的成本为 O(n)，这对于 n 高达 10^9 来说是不可能的。 

即使是跟踪每个索引的期望值的改进的 DP 也会导致每个状态依赖于所有后续状态的重复。 每个测试用例的复杂度为 O(n^2) 或 O(n)，这两种情况都是完全不可行的。 

关键的观察结果是系统的值是线性的，因此我们可以独立分析贡献。 每次通过操作传播一个值时，它都会乘以 2。这建议将该过程解释为随机根结构，其中每个节点最终都会为 a1 贡献一个权重，该权重取决于它通过随机选择“向上推”的次数。 

这将问题转化为理解随机递归树中沿着随机父链的预期放大。 一旦以这种形式重写，期望就变得容易处理，因为随机递归树的结构具有很强的对称性，从而降低了复杂性。

 ## 算法演练

 我们将这个过程重新解释为在节点 1 到 n 上构建随机递归树。 每个节点 i 从节点 1 到 i − 1 中统一选择一个父节点。这直接来自于以下事实：当前节点 i 作为源，它从较早的索引中统一选择目标 j。 

从节点到其父节点的每条边都会将贡献值乘以 2。因此，每个节点 i 贡献其初始值 i 乘以 2，再乘以该随机树中从 i 到节点 1 的距离。 

该任务变成计算所有节点上的总和的期望值。 

我们按如下方式进行。

1. 定义di为随机递归树中节点i的深度，其中节点1为根。 
2. 将最终答案表示为a1 = Σ i · 2^{di}。 这是有效的，因为每当一个值向节点 1 移动一步时，它就会乘以 2，因此总放大正好是 2^{距离}。 
3. 对所有节点进行线性期望：E[a1] = Σ i · E[2^{di}]。 剩下的任务是计算每个 i 的 E[2^{di}]。 
4. 定义 fi = E[2^{di}]。 在随机递归树中，节点 i 均匀地附加到任何先前的节点 j < i，因此

 fi = 2 · (1/(i − 1)) · Σ fj（j < i）。 
5. 令 Si = Σ_{j ≤ i} fj。 那么递推式变为 Si = Si−1 · (1 + 2/(i − 1))。 
6. 将乘法因子重写为 (i + 1)/(i − 1)。 这个望远镜干净利落地：

 Si = ∏_{k=2..i} (k + 1)/(k − 1)。 
7、产品彻底简化：

 分子变为 3 · 4 · ... · (i + 1)，

 分母变为 1 · 2 · ... · (i − 1)，

 其计算结果为 Si = i(i + 1)/2。 
8. 从 Si 中恢复 fi：fi = Si − Si−1 = i。 
9. 代入期望值：

 E[a1] = Σ i · fi = Σ i^2 在 i = 1..n 上。 
10. 计算封闭式平方和。 

### 为什么它有效

 整个减少取决于期望的线性度以及贡献沿着树边缘独立倍增的事实。 随机结构仅影响到 1 的路径上的节点，但不会影响预期的指数深度贡献，因为 E[2^{深度}] 的递归会折叠为伸缩乘积。 一旦 fi 简化为 i，随机性从最终表达式中完全消失，只留下 n 中的确定性多项式。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def solve(n):
    n %= MOD
    return n * (n + 1) % MOD * (2 * n + 1) % MOD * pow(6, MOD - 2, MOD) % MOD

t = int(input())
for _ in range(t):
    n = int(input())
    print(solve(n))
```该代码直接应用闭式平方和对给定素数进行模运算。 唯一需要注意的是模除以 6，使用费马逆元进行处理，因为模数是素数。 

该推导说明了为什么不需要模拟或动态规划：随机过程崩溃为简单的多项式恒等式。 

## 工作示例

 考虑 n = 3。最初数组是 [1, 2, 3]。 公式得出 1^2 + 2^2 + 3^2 = 14。

 ## 边缘情况

 当n = 1时，不发生任何操作，a1保持1。公式给出1^2 = 1，因此结果是一致的，不需要任何特殊处理。 

对于 n = 2，只有一次操作，j 的选择没有随机性。 推导仍然适用，因为预期深度计算完全退化，并且结果 5 与直接计算匹配。 

对于接近 10^9 的大 n，直接模拟会立即溢出时间限制，但封闭式表达式在模运算下保持稳定，因为它仅取决于多项式求值和模逆。
