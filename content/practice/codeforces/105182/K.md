 ## 解决方案
 ## 问题理解

 我们得到一个整数序列，并且允许我们重复应用一种非常特定的操作：在数组中精确选取 k 个位置，并将所有选定的元素乘以相同的非零整数。 该操作可以重复任意次。 

目标是转换序列，使所有元素变得相等，并且我们希望找到可能的 k 最大值。 

重新解释该运算的一个有用方法是思考素数指数如何演变。 将 k 个选定元素乘以相同的值意味着我们一次将相同的因子更改添加到恰好 k 个索引。 这意味着元素之间的“差异”是重要的真实结构，而不是绝对值。 

约束 n 高达 10^5，总和高达 10^6，这意味着我们基本上需要对每个测试用例进行线性或近线性处理。 任何尝试模拟操作或对所有元素之间的转换进行成对推理的方法都将立即失败。 

当所有数字都相等时，就会出现微妙的边缘情况。 在这种情况下，任何 k 直到 n 都是有效的，因为不需要任何操作。 另一个重要的情况是当数字具有完全不相关的素因式分解时，例如所有成对的互质值。 在这种情况下，我们仍然可以使它们相等，但最大 k 受到结构可分关系的约束，而不是相等本身。 

## 方法

 思考问题的一种直接但低效的方法是想象实际执行的操作。 每个操作都会同步修改 k 个元素，因此我们会尝试逐渐“同步”所有值。 然而，可能的操作序列的数量是巨大的，甚至决定固定的 k 是否有效也需要对更新的指数组合进行推理。 

关键的转变是停止考虑价值，而是考虑比率。 如果所有数字最终变得相等，则对于任意两个索引 i 和 j，比率 a[i] / a[j] 必须可以使用允许的运算进行约简。 每个操作只能同等地影响 k 个元素，因此真正重要的是有多少元素共享每个素因子的指数不平衡。 

对于固定的 k，如果我们可以重复分配调整，以便所有指数差异可以在大小为 k 的组中“中和”，则该过程可以成功。 只有当素数之间的总不平衡可以排列成大小为 k 的组时，这才有可能，这意味着 k 必须以非常具体的方式划分总“指数偏差质量”。 

看到这一点的简洁方法是确定目标最终值。 由于所有元素最终都相等，因此该值必须根据分布在指数中的贡献来构建。 对于每个质因子，我们考虑相对于目标其指数需要增加或减少的元素数量。 每个操作统一调整 k 个索引，因此每步有效地移动 k 个调整单位。 可行性条件分解为可分性约束：k 必须将总调整预算除以每个素数的所有索引，最严格的约束来自数组的全局 gcd 结构。 

关键的观察是所有操作都保留元素之间比率的 gcd，唯一的自由度在于缩放同步组。 这导致最大 k 恰好是最大除数，使得可以通过相等的“差异签名”对索引进行分组，这通过除以全局 gcd 来简化序列归一化后的任何值的最大频率的大小。

我们通过除以整个数组的 gcd 来标准化所有值。 此后，问题就简化为查找归一化序列中任何值的最大频率。 该频率是最大的 k，因为这些相同的值已经不需要相对于彼此进行调整，并且任何更大的分组将强制不兼容的指数移位。 

因此，答案是将数组除以其 gcd 后的任何数字的最大频率。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | ---| ---| ---| ---|
 | 蛮力模拟操作| O(exp) | O(n) | 太慢了 |
 | GCD归一化+频率计数| 每次测试 O(n) | O(n) | 已接受 |

 ## 算法演练

 1. 计算整个数组的gcd。 这消除了不影响相等可行性的公共缩放因子，因为始终可以通过包含所有位置的操作来实现对所有元素进行相等相乘。 
2. 将每个元素除以该 gcd。 在此步骤之后，所有剩余的结构都是纯粹相对的，这意味着我们实现的任何平等都必须来自平衡这些减少的值。 
3. 计算每个标准化值的频率。 
4. 答案是这些计数中的最大频率。 这对应于已经相同的最大元素组，它直接决定了同步操作可以对齐所有内容的最大 k。 

### 为什么它有效

 去除全局gcd后，剩余的值代表不可约的相对差异。 任何有效的操作序列都会保留共享相同标准化值的组内的相等关系。 每个操作均匀地影响 k 个索引，因此只有具有相同归一化结构的组才能被合并，而不会在因子调整中产生冲突。 始终可以选择最大的此类组作为操作块大小，任何较大的 k 都需要组合不同的归一化值，这将导致不一致的乘法历史，无法解析为单个公共端点。 

## Python 解决方案```python
import sys
import math
from collections import Counter

input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        
        g = 0
        for x in a:
            g = math.gcd(g, x)
        
        for i in range(n):
            a[i] //= g
        
        freq = Counter(a)
        ans = max(freq.values())
        out.append(str(ans))
    
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该实现首先增量计算 gcd，确保我们不会溢出或存储大型中间产品。 标准化后，整数除法是安全的，因为 gcd 保证能除掉每个元素。 

频率计数步骤使用哈希图，这对于高达 10^9 的大值范围是最佳的。 最后，我们直接提取最大频率。 

一个微妙的点是，我们必须独立地重新计算每个测试用例的 gcd，并且我们不能跨用例重复使用计数器，因为跨测试的约束求和但值是不相关的。

 ## 边缘情况

 当所有元素都相同时，gcd 等于值本身，并且标准化会生成一个全为 1 的数组。 频率图返回 n，这正确地反映了 k 可以是 n，因为任何操作都会立即保持相等。 

当所有元素不同但共享共同的 gcd 时，规范化仍可能产生重复项，但只有这些重复项对答案有贡献。 例如，[6,10,14] 的 gcd 为 2，变为 [3,5,7]，给出答案 1，这与不可能进行非平凡分组的事实相匹配。 

当数组包含主要重复值和一些异常值时，算法自然会隔离最大的相同簇，因为 gcd 归一化不会干扰相等类，并且频率计数直接捕获最大可行分组大小。
