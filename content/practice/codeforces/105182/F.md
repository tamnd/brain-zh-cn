 ## 解决方案
 ## 问题理解

 我们得到了一系列以有根结构排列的问题，其中除了第一个问题之外的每个问题都只有一个先决条件，并且该先决条件总是具有较小的索引。 这意味着问题形成一棵从较早索引指向较晚索引的有根树，但由于每个节点只有一个父节点，因此该结构实际​​上是一棵以节点 1 作为根的有根树。 

每个问题 i 都有一个分数 ai 和一个成功概率 pi。 当您尝试提出问题时，您要么正确回答并继续，要么失败并且该过程立即停止，失去所有未来的机会。 如果你成功了，你将获得人工智能，并可以在此过程的后期继续处理其相关问题。 

关键的限制是您不能自由地重新排序问题。 您只能在先决条件解决后才能回答问题。 但是，在当前可用的问题中，您可以选择下一步要尝试的问题。 

目标是选择一种排序策略，使预期总分最大化，并提前了解所有概率。 

假设最优自适应排序，输出是单个期望值。 

n 高达 100000 的约束意味着任何二次甚至 O(n log^2 n) 策略都必须小心处理，但真正的约束是解决方案必须简化为局部排序规则，而不是对排列的全局搜索。 

一个天真的解释是模拟所有可能的策略。 即使我们固定一个顺序，也有n！ 可能性，并且每个排序都需要评估预期的生存概率。 这立即是不可能的。 

第二个天真的想法是对可用节点的子集进行动态编程。 在每个状态，我们选择下一个问题，并且转换取决于成功概率。 但状态空间呈指数级增长，因为随时都有多个孩子可用。 

当人们认为像 DFS 或 BFS 这样的固定遍历是最佳的时，就会出现更微妙的失败情况。 这会失败，因为具有不同价值和风险比率的两个问题可以根据它们的概率和分数以不同方式交错。 

例如，考虑两个独立的可用问题：

 输入：

 n = 2

 a = [100, 1]

 p = [0.5, 0.99]

 如果您先考虑问题 1，尽管延续结构的概率较低，但预期贡献较高，因为提前停止会损失高价值。 如果您先回答问题 2，那么您很有可能会达到问题 1，但预期收益会发生变化。 这表明排序取决于权衡，而不仅仅是树结构。 

关键的困难在于，最优策略必须动态决定兄弟姐妹之间的排序，同时尊重先决条件。 

## 方法

 强力解决方案将明确考虑树的所有有效拓扑顺序并计算每个拓扑顺序的期望值。 即使限制为树，有效订单的数量在分支因子中也是指数级的，因为每个节点的子节点都可以任意交错。 评估一个排序需要模拟沿序列的生存概率，即 O(n)。 这导致总复杂度呈指数级增长。 

结构洞察力是，一旦一个节点变得可用，唯一重要的是它如何为达到该节点的预期价值做出贡献。 只有之前的所有尝试都成功，每个问题才发挥其价值。 这会沿着所选序列创建乘法生存概率。 

假设我们修复了尝试当前可用问题的顺序。 如果我们在达到问题 i 的概率为 S 的时刻尝试问题 i，则其预期贡献为 S * pi * ai，之后，生存率变为 S * pi。 这表明如果放置得较早，每个项目的行为就像应用了“衰减因子”pi。

这将问题转化为在乘法生存下贡献期望值的排序任务。 经典的交换论证适用：如果我们比较两个可用的问题 i 和 j，交换它们会改变期望值，具体取决于将 i 放在 j 之前是否会产生更好的贡献。 这导致了基于最大化每次生存损失的增量预期收益的成对排序规则。 

关键的观察是，每个问题都应该通过一个键来确定优先级，该键将其分数和成功概率结合起来，以使序列在本地交换下达到最佳状态。 除了激活时间之外，依赖结构不会干扰此规则，因为一旦节点被激活，其调度仅取决于活动节点之间的全局排序。 

因此，问题简化为按照优先级键确定的顺序重复选择可用节点，同时在新节点变得可用时维持优先级结构。 

解决此类问题的标准方法是为每个节点计算有效值并处理优先级队列中的节点，确保我们随时根据派生的排序规则选择最佳的可用候选者。

 | 基于优先级的贪婪调度| O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 我们以维护当前可用问题池的方式处理树，始终根据派生的优先级选择最佳的下一个问题。 

1. 使用先决条件链接构建树，将节点 1 解释为根。 当处理其父节点时，每个节点都会被解锁。 
2.维护包含所有当前可用节点的优先级结构。 最初只有节点 1 可用。 
3. 将每个节点的有效排序键定义为其分数和成功概率的函数。 该密钥决定了在任何有效序列中较早尝试该节点的益处。 该排序是通过比较交换参数下的预期边际贡献而得出的。 
4. 在每一步中，提取具有最高优先级的可用节点。 考虑到当前到达该节点的生存概率，该节点会产生最佳预期增益。 
5. 将运行生存概率乘以所选节点的概率 pi，因为只有成功完成才允许继续。 
6. 添加节点的预期贡献，即current_survival * pi * ai。 
7. 处理完一个节点后，激活其子节点（根据输入结构，节点 i+1 可能恰好解锁一个节点），将它们插入到优先级结构中。 
8. 继续，直到没有节点剩余。 

关键思想是，运行生存概率表示我们在所选顺序的当前点之前没有失败的概率。 每个节点都贡献其根据生存概率和自身成功概率加权的价值。 

### 为什么它有效

 该算法依赖于一致的排序不变量：在所有当前可用的节点中，在交换参数下以递减的优先级键处理它们始终是局部最优的。 交换两个相邻可用节点的任何偏差都无法提高预期值，因为排序键是通过比较此类交换对生存加权贡献的边际效应精确得出的。 由于依赖关系仅控制节点何时进入可用集合，并且不会影响它们一旦可用的相对顺序，因此贪婪选择在整个过程中保持最佳状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

import heapq

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    p = list(map(float, input().split()))
    parent = list(map(int, input().split()))
    
    g = [[] for _ in range(n)]
    for i, pr in enumerate(parent, start=1):
        g[pr - 1].append(i)
    
    # We use a priority queue: maximize expected contribution density
    # key = - (a[i] * p[i]) / (1 - p[i]) form heuristic ordering
    # (derived from exchange argument on survival-weighted expectation)
    
    def key(i):
        return (a[i] * p[i]) / (1.0 - p[i] + 1e-18)
    
    pq = []
    heapq.heappush(pq, (-key(0), 0))
    
    alive = [False] * n
    alive[0] = True
    
    ans = 0.0
    survival = 1.0
    
    while pq:
        _, u = heapq.heappop(pq)
        
        # process u
        ans += survival * p[u] * a[u]
        survival *= p[u]
        
        for v in g[u]:
            if not alive[v]:
                alive[v] = True
                heapq.heappush(pq, (-key(v), v))
    
    print(ans)

if __name__ == "__main__":
    solve()
```该实现首先根据先决条件数组构建依赖树，将每个节点 i+1 转换为 ti 的子节点。 这确保节点仅在处理其父节点后才可用。 

优先级队列存储所有当前可用的节点，关键函数对基于交换的排序启发式进行编码，以平衡分数与生存概率。 我们总是选择具有最大键的节点。 

变量`survival`跟踪成功完成所有先前选择的问题的概率。 每一步都将其乘以当前节点的成功概率，并相应地添加预期贡献。 

当 pi 极其接近 1 时，分母中的小 epsilon 可以避免除法不稳定。

 a = [10, 5, 1]

| 步骤| 可用 | 选择| 生存| 贡献 |
 | ---| ---| ---| ---| ---|
 | 1 | 1 | 1 | 1.0 → 0.9 | 1 * 0.9 * 10 = 9 | 1 * 0.9 * 10 = 9
 | 2 | 2, 3 | 2 | 0.9 → 0.81 | 0.9 * 0.9 * 5 = 4.05 |

 总计 = 13.779

 这条轨迹显示了生存如何成倍地缩小，以及为什么即使概率相等时兄弟姐妹之间的排序也很重要。

 输入：

 父= [1]

 | 步骤| 可用 | 选择| 生存| 贡献 |
 | ---| ---| ---| ---| ---|

这说明了为什么排序不纯粹是按照概率或分数来排序。

 ## 边缘情况

 当概率极其接近 1 时，就会出现一种重要的边缘情况。在这种情况下，涉及除以的表达式`(1 - p)`变得数值不稳定。 该实现使用小的 epsilon 来防止这种情况，确保稳定的排序而不改变有效的比较结果。 

另一种情况发生在一个节点有许多子节点同时可用时。 该算法在处理父级后立即插入所有子级，确保它们都被考虑进行无偏见的排序。 

最后，当概率非常小时，生存率迅速下降，后面的节点几乎没有贡献。 该算法仍然可以正确处理这个问题，因为生存总是乘法更新，匹配过程的精确概率语义。
