 ## 解决方案
 ## 问题理解

 我们得到了一组正整数，表示放置在序列中的权重。 两名玩家交替轮流，在每一轮中，他们要么通过将两个选定元素合并到其总和中来压缩序列，要么通过选择一个元素立即结束游戏。 当玩家选择一个数字时，该玩家将获得该数字的值，而对手将获得所有剩余元素的总和。 然后游戏停止。 

除非游戏结束，否则所有数字的总和在整个游戏过程中不会改变。 影响最终结果的唯一方法是合并数字，这会重塑总和在未来选择中的分配方式。 

事后看来，任何结束行动的最终决定都非常简单。 如果玩家从当前总和 S 中获取值 x，则对手收到 S − x，因此当 2x > S 时，获取者获胜。 

因此，整个问题在于，尽管对手也能够合并并可能提前结束游戏，但第一个玩家是否可以迫使他们选择相对于剩余总数足够大的值。 

这些限制意味着直接模拟游戏状态是不可能的。 由于测试用例多达 10^5 个，总数组大小高达 10^6，对所有合并历史或博弈树的任何状态空间搜索都将远远超出可行的限制。 即使维持动态游戏状态也会引入至少二次行为，因为每次合并都会将序列大小减少一，但选择的分支因子很大。 

一个天真的错误是将其视为一个简单的“获取最大元素”游戏。 但这会失败，因为合并可能会创建最初不存在的新值，而这些新值在与剩余总和的最终比较中可能会发挥决定性作用。 

另一种失败模式是假设贪婪游戏，例如总是合并最小的两个值。 这忽略了对手在有利的不平衡出现时提前结束游戏的反应能力。 

## 方法

 游戏的强力解释是将每个状态视为多重集，并模拟两个玩家探索所有有效的动作。 每次合并都会将元素数量减少一个，并且每次终止移动都会评估最终的分数分割。 这会导致游戏状态呈指数级增长，因为玩家在每一步都会选择任何对进行合并或选择任何元素来终止游戏。 即使忽略终止，可能的合并历史的数量也对应于 n 个叶子上的二叉树的数量，这是加泰罗尼亚语大小的，并且在实践中以超指数方式增长。 

关键的观察是合并不会改变总和，它只会改变总和的分组方式。 任何合并序列都相当于在初始元素上构建完整二叉树，其中每个内部节点代表子集的总和。 因此，当只剩下两个聚合组时，游戏简化为控制如何形成数组的最终分区。 

我们可以推断最终不可避免的结构，而不是跟踪游戏状态。 最终，最佳游戏会将序列减少到两个数字，因为早于这个数字停止主要是由持续合并决定的：过早结束游戏会给对手带来潜在的大余数和。 在两个元素中，不存在进一步的安全合并，因为合并将立即允许对手获取单个剩余值并以全部总和获胜。 

所以整个游戏就归结为一个问题：第一个玩家是否可以强制在序列减少为两组的那一刻，他们可以选择一个值超过总和一半的组。

这将游戏变成了如何形成最终二分的控制问题。 每个合并操作都有效地决定元素如何分组，并且两个玩家都通过对抗性选择来塑造二元分区树。 

决定性的见解是，尽管存在对抗性游戏，问题的结构却分解为一个简单的不变量：第一个玩家保证获胜的唯一方法是是否存在一个元素，其值严格大于所有其他元素的总和。 如果存在这样的元素，第一个玩家可以通过合并迫使它保持足够的“孤立”，并最终在对手压制它之前将其作为最终选择。 

如果不存在这样的主导因素，对手总是可以通过强制进行最终分割来做出回应，以避免给第一个玩家一个获胜的选择。

 ## 算法演练

 该解决方案简化为检查最大元素是否可以支配数组的其余部分。 

1. 计算序列中所有元素的总和S。 该值在合并下保持不变，因此它是评估每个最终决策的参考。 
2. 找出数组中的最大元素mx。 任何获胜策略最终都必须依赖于强制在正确的时刻选择这个值，因为如果它已经在全球占据主导地位，那么任何较小元素的组合都无法超过它。 
3. 将 mx 与 S − mx 进行比较。 如果 mx 严格大于 S − mx，则 mx 足够强大，如果它被选为最终选择，则选择的玩家立即获胜。 
4. 如果此条件成立，则得出结论，第一个玩家可以强制执行一系列合并，从而保留在正确时间选择 mx 的可能性，从而导致强制获胜。 
5. 否则，得出结论，对手总是可以以阻止第一个玩家获得获胜终端选择的方式进行响应。 

关键思想是合并仅重新分配权重，并且任何重新分配序列都不能创建比全局最大值更强的新元素。 因此，整个游戏从根本上取决于单个元素是否已经在总和中占主导地位。 

### 为什么它有效

 每次移动都会保留总和 S，并且每次最终移动都是通过将单个选定值 x 与 S − x 进行比较来决定。 由于合并仅用两个值的总和替换两个值，因此它们永远不会将最大可能的个体“最终可选择优势”增加到超出某些子集中已存在的值。 保证最终获胜的唯一方法是确保存在一个可以被隔离为满足 2x > S 的选择的值。如果最初不存在这样的值，则任何合并序列都无法创建它，因为任何合并值都对应于一个子集，其补码和至少一样大，从而允许对手在最终步骤中抵消优势。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))
        s = sum(a)
        mx = max(a)
        if mx > s - mx:
            out.append("YES")
        else:
            out.append("NO")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```代码直接实现了上面导出的不变量。 对于每个测试用例，它计算总和和最大元素。 该决定仅取决于最大元素是否严格大于所有剩余元素的总和。 不需要合并模拟，因为合并不会影响总和或最大元素的标识。 

该实现小心地避免了每个操作的开销，并使每个测试用例的所有内容保持线性，考虑到聚合输入大小高达一百万个元素，这是必要的。

 ## 边缘情况

 当所有元素相等时，仅在简并 n=2 的情况下，每个候选值恰好等于总和的一半，否则严格小于。 该算法正确输出“否”，因为没有元素满足严格的支配性。 

当n等于1时，最大值等于总和，因此条件成立，答案为YES。 玩家立即获得唯一的元素并获胜。 

当存在非常大的离群元素时，合并无法减少其值或将其优势从最终比较中重新分配。 该算法正确识别了这一点并返回 YES。
