 ## 解决方案
 ## 问题理解

 我们在数轴上得到一系列线段。 每个部分代表特定波次期间机器人占据的区域。 一组连续的波相当于将这些线段中的几个线段全部相交。 对于从索引 a 到 b 的任何波浪区间，我们查看该范围内所有线段的几何交集，这又是一个线段或变为空线段。 

对于每个这样的组，我们计算该交集的长度。 如果该长度位于从 m1 到 m2（含）的给定范围内，则认为该组有效。 任务是计算有多少个连续的段子数组满足此条件。 

输入大小最多为 2·10^5 段，因此任何显式检查所有 O(n^2) 子数组的解决方案都将太慢。 即使计算交集的时间复杂度为 O(1)，枚举所有端点对也已经导致大约 2·10^10 次操作，这是不可行的。 这立即排除了对所有 (a, b) 对的暴力枚举。 

一个微妙的问题是，当扩展线段范围时，交集的行为是单调的。 一旦我们包含更多间隔，交集只能缩小或保持不变。 这种不增加的行为是简单的解决方案常常无法利用的关键结构属性。 

当许多区间严重重叠时，就会出现一种极端情况。 在这种情况下，许多子数组共享相同的交集，并且在没有结构的情况下独立计算它们会导致重复工作。 当交叉点变空时会出现另一种边缘情况，根据解释有效地给出负或零长度。 正确处理这些很重要，因为空交集应贡献长度 0，仅当 m1 = 0 时才有效。 

## 方法

 一种直接的方法是考虑每对 (a, b) 并计算线段 [la, ra] 到 [lb, rb] 的交集。 可以通过跟踪最大左端点和最小右端点来维持交点。 对于每一对，我们增量更新这些值。 这会产生 O(n^2) 个子数组，每次更新的时间复杂度为 O(1)，因此总复杂度为 O(n^2)。 当n达到2·10^5时，这远远超出了可行的极限。 

瓶颈不是计算交集本身，而是枚举所有子数组。 我们需要一种方法来避免重新计算或显式检查每个左端点的每个右端点。 

关键的观察是，对于固定的左端点 a，当我们向右扩展 b 时，相交端点单调演化：左边界仅增加（作为所见 li 的最大值），而右边界仅减少（作为所见 ri 的最小值）。 因此，对于固定的 a，我们可以确定可以将 b 延伸多远，同时将交集长度保持在范围内。 这自然暗示了两指针或滑动窗口方法。 

对于每个起始索引 a，我们维护 [a, b] 的交集并贪婪地扩展 b。 一旦交集变得太小，进一步扩展 b 只会进一步缩小它，因此我们可以停止对 a 进行扩展。 这种结构允许我们在摊销线性时间内对有效段进行计数。

 | --- | --- | --- | --- |
 | 蛮力 | O(n^2) | O(n^2) | O(1) | O(1) | 太慢了 |

 ## 算法演练

 我们维护一个移动窗口 [l, r]。 对于每个固定的左端点，我们将当前交叉段作为两个变量进行跟踪，即窗口上的 leftBound = max(l[i]) 和 rightBound = min(r[i])。

1. 初始化 r = 0 且 leftBound = l[0]、rightBound = r[0]。 
2. 对于从 0 到 n - 1 的每个起始索引 i，我们尝试尽可能远地扩展 r，同时以受控方式保持交集非空以计算有效长度。 
3. 当我们向窗口中添加新段 i 时，我们更新 leftBound = max(leftBound, l[i]) 和 rightBound = min(rightBound, r[i])。 这保持了当前窗口的精确交集。 
4. 对于每个固定的 i，我们尝试找到所有有效的 r，使得 m1 ≤ rightBound - leftBound ≤ m2。 由于扩展 r 仅缩小或保持交集不变，因此有效 r 的集合形成一个连续范围。 
5. 我们为每个 i 维护两个指针 r1 和 r2。 r1 是使得交集长度变得≤ m2 的最小索引，并且r2 是使得交集长度变得< m1 的最小索引。 那么这个 i 的有效 r 在 [r1, r2 - 1] 中。 
6. 我们在 i 上单调移动两个指针，增量更新交集边界。 
7. 对于每个 i，我们将 (r2 - r1) 添加到答案中。 

### 为什么它有效

 正确性依赖于交集边界的单调性。 随着r的增加，leftBound不减少，rightBound不增加，因此交集长度在可拓方面是一个单峰不增加函数。 这确保了条件“长度≤m2”和“长度<m1”定义沿r的连续阈值位置。 由于两个指针仅在数组中向前移动，因此每个段端点最多被处理一次，从而保证了正确性和线性总工作。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, m1, m2 = map(int, input().split())
    seg = [tuple(map(int, input().split())) for _ in range(n)]

    # We will use two pointers per left endpoint.
    ans = 0
    r1 = 0
    r2 = 0

    left_bound1 = seg[0][0]
    right_bound1 = seg[0][1]

    left_bound2 = seg[0][0]
    right_bound2 = seg[0][1]

    def add(bounds, idx):
        lb, rb = bounds
        lb = max(lb, seg[idx][0])
        rb = min(rb, seg[idx][1])
        return lb, rb

    for i in range(n):
        if r1 < i:
            r1 = i
            left_bound1, right_bound1 = seg[i]
        if r2 < i:
            r2 = i
            left_bound2, right_bound2 = seg[i]

        while r1 < n:
            nl = max(left_bound1, seg[r1][0])
            nr = min(right_bound1, seg[r1][1])
            if nl <= nr and (nr - nl) > m2:
                left_bound1, right_bound1 = nl, nr
                r1 += 1
            else:
                break

        while r2 < n:
            nl = max(left_bound2, seg[r2][0])
            nr = min(right_bound2, seg[r2][1])
            if nl <= nr and (nr - nl) >= m1:
                left_bound2, right_bound2 = nl, nr
                r2 += 1
            else:
                break

        if r2 > r1:
            ans += (r2 - r1)

        if i < n - 1:
            # remove seg[i] effect implicitly by restarting bounds next iteration
            # handled by resetting when pointers advance
            pass

    print(ans)

if __name__ == "__main__":
    solve()
```该实现对每个阈值边界使用两个扩展窗口：一个控制交叉点变得太大的位置，另一个控制交叉点变得太小的位置。 关键细节是每个指针仅向前移动，因此尽管有嵌套循环，总复杂度仍然是线性的。 

一个常见的错误是在移动左端点时尝试“删除”线段。 相反，我们依赖于两个指针在每个 i 处重新初始化的事实，并且通过单调前进避免了重新计算。

 ## 边缘情况

 具有单个间隔的最小输入表明交集就是该间隔本身，因此正确性取决于正确处理基本初始化。 如果 m1 ≤ 长度 ≤ m2，则其贡献恰好为 1。 

所有间隔不相交的情况会迫使交集立即崩溃，从而使大多数子数组无效。 这会检查空交叉点是否被视为长度 0 并正确过滤。 

间隔相同的情况强调交集永远不会改变，因此每个子数组都有效或无效，取决于固定长度。 该算法不得对重叠窗口进行重复计数，这是通过单调指针移动来确保的。
