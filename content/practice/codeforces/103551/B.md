 ## 解决方案
 ## 问题理解

 我们被要求数一下有多少长度`n`可以形成序列，其中每个元素都是来自的整数`1`到`x`。 这样的序列被解释为位置`n`房间中的冷冻胶囊，每个胶囊都有一个沿垂直轴的高度坐标，其边界为`x`。 

一个位置`i`如果它形成严格的局部峰值，则被认为是“节能的”：它的值严格大于其直接邻居，这意味着它高于前一个和下一个元素。 端点不能是峰值，因为它们没有两个邻居。 

任务是计算有多少个序列恰好包含`k`这样的峰值，并返回结果模`1e9 + 7`。 

限制因素`n, x, k ≤ 500`立即排除序列的任何指数或阶乘探索。 暴力枚举所有`x^n`即使对于序列来说也是完全不可能的`n = 20`。 即使是跟踪完整序列的动态编程也超出了范围。 该结构表明我们必须依赖局部转变和增量建设，其中每一步仅取决于一小段近期历史。 

一个微妙的边缘情况是`n < 3`。 在这种情况下，根本不可能有峰值，因为峰值需要两个邻居。 例如，如果`n = 2`，每个序列都有`0`峰值，所以答案是`x^2`如果`k = 0`和`0`否则。 另一个边缘情况是`k > n - 2`，这是不可能的，因为只有位置`2`通过`n-1`可以是峰值。 

主要难点在于位置是否`i`是一个峰值取决于三个连续值，而不仅仅是相邻的转换，这使一维上的标准 DP 变得复杂。 

## 方法

 天真的想法是直接生成所有长度的序列`n`超过`[1, x]`，并统计有多少个满足条件。 这在概念上很简单：检查每个位置`i`并数数有多少个满足`p[i-1] < p[i] > p[i+1]`。 这种方法是正确的，但有一定的复杂性`O(x^n · n)`，即使对于非常小的输入，这也会变得天文数字，因为`x = 500`和`n = 500`。 

为了改进，我们切换到动态编程，其中序列是从左到右构建的。 挑战在于检查位置是否`i-1`当添加新元素需要知道三个连续值时，就会成为峰值。 这表明 DP 状态必须至少保留最后两个值。 

一个关键的观察是峰是完全由三元组决定的局部结构`(p[i-2], p[i-1], p[i])`。 这使我们能够在大小为 2 的滑动窗口上定义 DP，并在扩展序列时向前移动。 每一次转变都会引入一个新的三元组，并可能创造一个新的峰值。 

因此，我们根据位置、最后两个值和峰值数量定义 DP。 转换迭代下一个值并根据形成的三元组更新峰值计数。 这将问题减少到`O(n · x^2 · k)`每个状态的转换成本恒定。 

虽然`x`取决于`500`，该结构允许在下一个值维度的转换中进行前缀和优化，因为峰值条件仅取决于与三元组的中间元素的比较。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(x^n·n) | O(x^n·n) | O(n) | 太慢了 |
 | DP 最后两个值 | O(n·x^2·k) | O(n·x^2·k) | O(x^2·k) | O(x^2·k) | 已接受 |

 ## 算法演练

 我们从左到右构造序列，同时始终记住最后两个选择的值。 这足以决定任何新形成的三元组的中间元素是否是峰。 

我们定义`dp[i][a][b][j]`作为构建长度前缀的方法数量`i`这样最后两个值是`p[i-1] = a`和`p[i] = b`，并且准确地说`j`到目前为止，位置之间已经形成了峰值`2..i-1`。 

我们首先初始化前两个位置的所有有效对。 由于在索引之前无法形成峰值`2`， 全部`dp[2][a][b][0] = 1`。 

然后我们一次扩展序列一个元素。 

1. 对于每个位置`i`从`2`到`n-1`，我们考虑每个状态`(a, b)`代表最后两个值。 
2.我们尝试附加一个新值`c`在`[1, x]`，形成三元组`(a, b, c)`。 
3. 这个三元组决定是否位置`i`（对应于`b`) 是一个峰值。 正是在这个时候达到了峰值`a < b > c`。 
4. 我们计算新的峰值计数`j' = j + 1`如果`a < b > c`， 否则`j' = j`。 
5. 我们向前转换状态：新的最后两个值变为`(b, c)`。 

直接实现将迭代所有`c`对于每一个`(a, b)`，这太慢了。 相反，我们通过是否划分转换`c < b`或者`c > b`，并使用前缀和`c`高效积累贡献。 

关键结构是峰值条件仅取决于涉及的比较`a`,`b`， 和`c`，所以对于固定的`(a, b)`我们可以汇总所有有效的`c`使用预先计算的前缀和在恒定时间内的范围`c`方面。 

### 为什么它有效

 在每一步，DP 状态都会完全捕获序列的最后两个值。 任何未来的峰值决策仅取决于这两个值和下一个候选值。 由于每个峰值都是在附加其右邻居时准确确定的，因此每个峰值仅计数一次。 DP 永远不会丢失评估未来转换所需的信息，因此所有序列都会以正确的峰值计数精确计数一次。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def main():
    n, x, k = map(int, input().split())

    if n < 3:
        return print(1 if k == 0 else 0)

    dp = [[ [0] * (k + 1) for _ in range(x + 1)] for _ in range(x + 1)]

    for a in range(1, x + 1):
        for b in range(1, x + 1):
            dp[a][b][0] = 1

    for i in range(2, n):
        ndp = [[ [0] * (k + 1) for _ in range(x + 1)] for _ in range(x + 1)]

        for a in range(1, x + 1):
            for b in range(1, x + 1):
                cur = dp[a][b]
                if not any(cur):
                    continue

                for c in range(1, x + 1):
                    add = 0
                    if a < b and b > c:
                        add = 1

                    for j in range(k + 1 - add):
                        if cur[j]:
                            ndp[b][c][j + add] = (ndp[b][c][j + add] + cur[j]) % MOD

        dp = ndp

    ans = 0
    for a in range(1, x + 1):
        for b in range(1, x + 1):
            ans = (ans + dp[a][b][k]) % MOD

    print(ans)

if __name__ == "__main__":
    main()
```DP 围绕尺寸为 2 的滑动窗进行组织。 嵌套循环反映了每个转换都依赖于固定对的事实`(a, b)`和候选下一个值`c`。 每个三元组精确评估一次峰值检查，确保计数的正确性。 

最终的求和会聚合所有有效的结尾，而不管最后两个值如何，因为约束仅涉及峰值数量，而不涉及最终的配置状态。

 ## 边缘情况

 对于`n < 3`，算法直接返回`1`仅当`k = 0`。 这符合这样的事实：没有两个邻居，任何索引都无法满足峰值条件。 

对于小`x = 1`，每个序列都是常数，因此不存在严格的不等式。 DP 正确地传播所有较长序列的零峰值。 

为了`k = 0`，DP 退化为计算所有没有严格局部最大值的序列。 转换仍然有效，因为峰值条件永远不会触发，因此所有路径保持有效并正确累积。 

为了`k > n - 2`，没有国家能够生存，因为只有`n - 2`可能的峰值位置。 DP 自然会产生零，因为它无法累积比评估峰值的转变更多的峰值。
