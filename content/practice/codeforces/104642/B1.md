 ## 解决方案
 ## 问题理解

 我们得到了排列成一行的数字序列，并且允许我们执行非常有限的重新排列操作。 该操作不允许我们交换任意元素，只能交换数组中相距两个位置的元素。 这意味着一个元素只能与位于相同索引奇偶性的其他元素交互，无论是所有偶数位置还是所有奇数位置。 

任务是在重复应用这些允许的交换后确定最终配置，直到无法进行更多改进，这实际上意味着每个奇偶校验类都独立排序，而偶数索引和奇数索引之间的相对顺序被保留，因为它们从不混合。 

从输入的角度来看，我们读取一个长度为n的数组，并且必须在过程稳定后输出结果数组。 

此类问题的约束水平通常足够大，以至于 O(n^2) 的交换模拟不可行。 如果 n 达到 10^5，任何重复扫描和交换相邻有效对的方法在最坏的情况下都会降级到大约 10^10 次操作，这远远超出了 2 秒时间限制的实际限制。 这立即表明操作结构必须简化为排序或分解技巧，而不是直接模拟。 

当值在两个奇偶校验组之间交错时，会出现微妙的边缘情况。 例如，如果数组已经全局排序，但奇偶校验分隔的排序不同，则简单的“全局排序检查”将错误地假设不需要任何操作。 

考虑输入：

 n = 5，数组 = [1, 5, 2, 4, 3]

 一种幼稚的方法可能会假设，由于数组“几乎已排序”，因此需要很少的交换。 然而，奇偶校验分离给出偶数索引 [1, 2, 3] 和奇数索引 [5, 4]，并且这些索引独立演化，产生未全局排序的最终排列。 

这是核心的结构性陷阱：转换尊重指数平价，而不是价值邻接。 

## 方法

 暴力解释直接模拟允许的操作。 我们重复扫描数组，并在位置 i 和 i + 2 处的元素发生乱序时交换它们。 每一遍都可以稍微改善排序，但在最坏的情况下，每个元素可能需要通过重复交换来遍历数组的一半。 这与分别应用于偶数和奇数索引的冒泡排序类似，导致每个奇偶校验组的行为为 O(n^2)，因此总体上仍然为 O(n^2)。 

关键的观察结果是互换永远不会跨越奇偶校验边界。 索引为 0、2、4... 的元素形成一个独立系统，索引为 1、3、5... 的元素形成另一个独立系统。 在每个系统中，允许的交换实际上是在该简化序列中的相邻元素之间进行的。 这正是对每个子序列进行完全排序所需的操作。 

因此，我们不模拟交换，而是直接提取两个子序列，对它们进行独立排序，然后将它们放回原来的奇偶校验位置。

 | 暴力模拟| O(n^2) | O(n^2) | O(1) | O(1) | 太慢了|

 ## 算法演练

 我们通过基于索引奇偶校验分隔数组，将问题简化为两个独立的排序任务。

1. 将数组拆分为两个列表：一个包含偶数索引中的元素，另一个包含奇数索引中的元素。 这是可行的，因为没有任何操作会在这两个组之间移动元素。 
2. 以非降序对两个列表独立排序。 每个列表代表一个在允许的操作下的全连接交换图，因此对它们进行排序相当于应用所有可能的交换。 
3. 通过将排序后的偶数索引列表放回偶数位置并将排序后的奇数索引列表放回奇数位置来重建原始数组。 
4. 输出重构后的数组。 

关键的推理步骤是，在每个奇偶校验组内，允许的操作有效地允许我们在该简化序列中执行任何相邻交换，这足以达到完全排序的顺序。 

### 为什么它有效

 该操作保留索引的奇偶校验，因此数组分解为两个独立的子系统。 在每个子系统中，交换原始数组中的 i 和 i+2 对应于交换提取的子序列中的相邻元素。 由于相邻交换生成完整的排列组，因此对每个子序列进行排序对于模拟所有可达配置来说既是必要的也是充分的。 因此重建的数组正是所有有效操作后的最终状态。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    
    even = a[0::2]
    odd = a[1::2]
    
    even.sort()
    odd.sort()
    
    ei = 0
    oi = 0
    res = []
    
    for i in range(n):
        if i % 2 == 0:
            res.append(even[ei])
            ei += 1
        else:
            res.append(odd[oi])
            oi += 1
    
    print(*res)

if __name__ == "__main__":
    solve()
```实现直接反映了概念分解。 切片操作在线性时间内隔离奇偶校验组，并且对每个组进行排序决定了复杂性。 重建步骤小心地在两个排序数组之间交替，确保准确保留原始索引奇偶校验。 这里最常见的实现错误是在重建过程中混淆了奇偶校验分配，即使两个排序列表都是正确的，这也会默默地破坏正确性。

 ## 边缘情况

 一种重要的边缘情况是数组已全局排序但奇偶校验分隔排序未排序。 例如，考虑 [1, 3, 2, 4]。 偶数位置包含 [1, 2]，奇数位置包含 [3, 4]。 该算法对两个组进行排序，但在这种情况下保持全局顺序不变，生成 [1, 3, 2, 4]，它匹配允许的变换空间而不是完全排序。 

另一个边缘情况是所有元素都相等。 由于排序不会改变相对顺序，因此两个奇偶校验列表保持相同，并且重建会产生相同的数组。 这证实了统一输入下的稳定性，并确保不会引入意外的交换。 

最后一个微妙的情况是当 n = 1 或 n = 2 时。对于 n = 1，根本不存在奇偶校验交互，并且数组保持不变。 对于 n = 2，每个元素都属于一个单独的奇偶校验类，因此同样不可能进行交换，并且输出必须与输入完全匹配。
