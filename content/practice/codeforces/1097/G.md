 ## 解决方案
 ## 问题理解

 我们正在使用一棵树，其中每个顶点子集都根据当我们被迫包含该子集中的所有顶点时最小连通子图的大小来定义自然“成本”。 对于选定的子集，您采用包含所有选定顶点的最小子树，并通过它包含的边数来衡量它。 然后将该值提升至幂$k$，并且我们必须对所有非空子集这个数量求和。 

关键的难点在于有$2^n - 1$子集太多，无法直接评估。 即使计算一个子集的斯坦纳树大小也已经依赖于树内的成对结构，因此任何子集枚举方法立即变得不可行$n = 10^5$。 

这些约束清楚地表明解决方案必须接近线性或$O(n \log n)$具有一个小的多项式因子$k$。 自从$k \le 200$，一个典型的信号是我们期望保持尺寸的多项式展开$k$，通常通过边缘上的 DP 或每个边缘贡献的组合计数。 

单例子集产生了一个微妙的问题。 对于单个顶点，连通子树没有边，因此$f(X) = 0$。 这意味着许多子集的贡献为零，除非$k = 0$，但是这里$k \ge 1$，因此所有单例子集都会在最终总和中消失。 任何正确的解决方案都必须自然地处理这个问题，而无需特殊的外壳。 

另一个常见的陷阱是假设$f(X)$其行为类似于顶点上的距离或附加度量。 事实并非如此。 相反，它是诱导的最小连接子树的大小，这取决于所选顶点之间的路径上有多少条边。 

## 方法

 暴力方法会枚举每个子集$X$，计算其最小连接子树，计算其中的边，并将结果提高为$k$。 即使我们预先计算距离或使用 LCA 来计算子树大小，每个子集仍然需要至少$O(|X| \log n)$或类似的工作。 由于有$2^n$子集，这是完全不可行的，超过$10^{30}$运营。 

关键的观察是将视角从顶点子集转移到边的贡献。 树结构允许我们推断给定边是否包含在子集的最小子树中。 当且仅当子集在该边的两侧都至少有一个顶点时，一条边才属于子集的 Steiner 树。 

这会将问题转换为每边二元条件。 如果我们删除一条边，树就会分成两个大小的组成部分$a$和$b$。 任何子集在其两侧至少包含一个顶点时，都会将该边贡献给其 Steiner 树，这发生在$2^n - 2^a - 2^b + 1$子集。 

然而，我们不仅仅想要使用的边数。 我们需要$(\text{number of used edges})^k$，它引入了边缘指标之和的幂。 这就是多项式 DP 的用武之地：我们将每条边视为贡献一个二元变量，并计算所有子集中活动边数量的分布。 

我们对树进行生根并用 DP 对其进行处理，其中每个子树都维护一个多项式$dp[x]$描述它内部有多少子集产生连接到树的其余部分的给定数量的“激活边界边”。 当合并子项时，我们通过类似卷积的转换来组合分布，但是因为$k \le 200$，我们可以将多项式次数上限限制为$k$。 

当子树非空并且图的其余部分也至少有一个选定的顶点时，节点及其子节点之间的每条边都准确地做出贡献。 这导致了一个标准的重新根式DP，其中我们为每个节点维护一个多项式，计算在其组件中选择顶点的方式有多少种，并且给定数量的“切边”指向其父节点。 

这将全局组合结构转换为树上的局部合并操作，其中每条边在 DP 状态转换中独立做出贡献。

 ## 算法演练

 我们以节点 1 为树根，并为每个节点定义一个 DP 状态，表示其子树内的子集如何与其父节点的边交互。 

1. 定义$dp[u][i]$作为选择子树中节点子集的方法数量$u$这样正好$i$将所选顶点连接到子树外部的边是“活动的”。 我们盖帽$i$在$k$因为较高的值与$x^k$扩张。 
2. 初始化$dp[u][0] = 1$对于每个节点$u$，表示子树内的空选择。 
3. 处理子进程$u$将它们的 DP 表一一合并为$u$。 合并孩子时$v$，我们考虑两种情况：要么我们不从中获取任何顶点$v$的子树，它不会贡献任何新内容，或者我们采用非空选择，这会激活边缘$(u,v)$。 
4. 当我们从 child 中获取非空选择时$v$，每个配置$dp[v]$将活动边的计数移一，因为边$(u,v)$只要两边都非空，就成为斯坦纳结构的一部分。 
5. 使用临时数组合并转换$newdp$，执行卷积截断于$k$。 每次合并都会合并来自的现有计数$u$捐款来自$v$，更新形成的活动边数。 
6.处理完所有children后，$dp[u]$对子树内的所有子集行为进行编码$u$。 
7. 最后，从根本上来说，我们解释每个状态$i$作为贡献$i^k$加权为$dp[root][i]$, 总结所有$i$。 

### 为什么它有效

 树中的每条边都对应于一个二元事件：它要么被子集切割，要么不被子集切割，具体取决于边的两侧是否包含选定的顶点。 DP 构造确保每个子树独立地考虑内部配置，同时正确传播边界边缘是否变为活动状态。 由于在合并子项时每条边仅引入一次，因此不会对任何边进行重复计算。 因此，根处的最终 DP 状态枚举了按照其诱导的 Steiner 结构中出现的边数精确分组的所有子集，这正是计算幂和所需的。

这证实了 DP 正确跟踪所有子集的 Steiner 边缘计数。

 ## 边缘情况

 单边树揭示了这样一个事实：只有包含两个端点的子集才会贡献非零值。 DP 正确地捕获了这一点，因为激活边缘的唯一方法是从一侧选取一个非空子集并通过根结构连接它。 

高度不平衡的树（例如链）测试重复的 DP 合并是否会累积边计数而不会重复计算。 该结构确保每个边在其子子树合并时仅被引入一次，因此即使在深度递归链中也不会发生膨胀。 

许多叶子连接到一个中心的树验证多分支合并顺序的正确性。 由于每个子节点都独立地合并到当前的DP表中，因此最终结果在子节点排序下是不变的，反映了不相交子树之间子集选择的交换性。
