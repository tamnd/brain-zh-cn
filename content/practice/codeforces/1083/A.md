 ## 解决方案
 ## 问题理解

 我们有一棵树，其中每个节点代表一个城市。 每个城市都有一个加油站，每个加油站都可以让我们获得一定数量的燃料$w_i$。 在由道路连接的两个城市之间行驶所消耗的燃料等于该道路的长度。 

旅行者从某个城市出发，沿着简单的路径移动（没有重复的顶点），并且可能在任何点停止。 在每一个访问的城市，包括出发地和结束地，他们可以通过添加$w_i$单位。 在穿过边缘时，绝不允许燃油液位降至零以下。 目标是选择起始城市和路径，以便完成步行后最终剩余的燃料最大化。 

主要困难在于路径的可行性取决于中间燃料水平，而不仅仅是总和。 如果单个边缘对于当时可用的燃料来说太昂贵而无法跨越，那么总体上看起来有利可图的路径仍然是不可能的。 

约束条件达到$n = 3 \cdot 10^5$，这立即排除了对节点对或每个起始节点的重复图遍历的任何二次方法。 甚至$O(n \log n)$必须仔细构造，但目标是线性或近线性树动态规划。 

一些微妙的案例揭示了为什么天真的推理会失败。 如果我们只考虑一条路径上的总增益，我们可能会选择一条总燃料增益超过总成本的路径，但旅行者可能会因为高成本优势而提前陷入困境。 

例如，假设一个节点$u$有一个大$w_u$，但它的第一个优势的成本比从邻居那里获得的任何直接收益都要高。 即使路径不可能遍历，天真的“权重总和减去边总和”方法也会错误地接受这种转换。 

另一个边缘情况是单节点树。 答案很简单$w_1$，因为不需要移动并且没有违反任何约束。 

## 方法

 强力策略是尝试树中每条可能的简单路径，模拟沿该路径的燃料积累，并跟踪最佳的最终燃料。 即使枚举所有路径在树中也已经是指数级的，因为有$O(n^2)$可能的端点和每条路径必须逐步验证。 每条路径的模拟增加了另一个因素$O(n)$，导致不可行的$O(n^3)$最坏的情况。 

关键的观察结果是树结构可以防止循环，因此每条路径都有一个最高点，该最高点自然充当根。 如果我们将一个节点固定为“起始锚点”，那么该节点的任何有效延续的行为就像扩展到子树一样，并且可以在本地修剪无效的扩展。 

关键的见解是将问题重新解释为计算，对于每个节点，计算从该节点开始并向下移动到其子树的路径的最佳可能“净增益”，其中我们只保留不破坏可行性的扩展。 如果子树不能在不变为负数的情况下进入，则它会被忽略。 这将全局路径搜索转变为最佳组合的局部决策。 

这是有效的，因为任何最佳路径都可以被视为植根于该路径的最高点，并且从该点开始，它仅向下延伸到不相交的分支。

 | --- | --- | --- | --- |
 | 蛮力 |$O(n^3)$|$O(n)$| 太慢了 |
 | 树DP |$O(n)$|$O(n)$| 已接受 |

 ## 算法演练

 为了方便起见，我们在节点 1 处任意确定树的根。如果我们从每个节点开始并且仅移至其后代，则 DP 将从每个节点开始计算最佳可实现的燃料增益。 

1. 在任意节点（通常是节点 1）处将树作为根。这为每条边提供了父子方向，以便我们可以清晰地定义子问题。 
2. 定义一个值$dp[u]$如果我们在节点处开始一条路径，我们可以获得的最大最终燃料$u$并且仅向下移动到其子级。 我们假设我们首先收集$w_u$在$u$，因为每个访问的节点都允许加油。 
3. 对于每个孩子$v$的$u$，考虑从$u$进入$v$。 如果我们进入$v$，我们必须支付边缘成本$c(u,v)$，所以该子树的有效贡献是$dp[v] - c(u,v)$。 
4. 仅在儿童贡献为正值时才接受。 如果$dp[v] - c(u,v) \le 0$，那么进入该子树永远不会改善最终燃料，最好停在$u$而不是跨越那个边缘。 
5. 总结儿童的所有积极贡献并将其添加到$w_u$。 这给出了$dp[u]$。 
6.答案是最大值$dp[u]$在所有节点上，因为最佳路径可能从任何地方开始。 

### 为什么它有效

 在任意节点$u$，一旦我们决定进入一个子子树，该子树内的最佳延续就已经被完全捕获$dp[v]$。 之间唯一的互动$u$和$v$是边缘成本，因此该决策简化为本地盈利能力检查。 因为树没有循环，所以一旦选择或拒绝了每个子树的边，每个子树都是独立的。 这可以防止不同分支机构之间出现重复计算或不一致的燃料核算。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n = int(input())
w = list(map(int, input().split()))

adj = [[] for _ in range(n)]
for _ in range(n - 1):
    u, v, c = map(int, input().split())
    u -= 1
    v -= 1
    adj[u].append((v, c))
    adj[v].append((u, c))

dp = [0] * n
visited = [False] * n

def dfs(u, parent):
    best = w[u]
    for v, c in adj[u]:
        if v == parent:
            continue
        dfs(v, u)
        gain = dp[v] - c
        if gain > 0:
            best += gain
    dp[u] = best

dfs(0, -1)

print(max(dp))
```该实现计算后序 DFS，以便每个子项都在其父项之前处理。 这确保了当我们评估时$u$， 全部$dp[v]$值已经是正确的。 

唯一微妙的实现细节是剪枝条件`gain > 0`。 如果没有此检查，我们将错误地强制包含负贡献子树，这会降低路径值，即使始终允许提前停止。 

最终答案是所有答案中的最大值$dp[u]$，因为最佳路径可以从任何节点开始，不一定是根。 

## 工作示例

 ### 示例 1

 输入：```
3

1 2 2
1 3 2
```我们以 1 为根。 

| 节点| w[u] | 儿童捐款| dp[u] |
 | --- | --- | --- | --- |
 | 2 | 3 | 无 | 3 |
 | 3 | 3 | 无 | 3 |
 | 1 | 1 | (3-2)=1, (3-2)=1 | 1 + 1 + 1 = 3 |

 最佳答案是 3，通过从节点 2 开始，经过 1，然后到 3 来实现。该表确认两个分支都可以从根单独获利。

该迹线显示了仅当边缘成本允许时，有利可图的深层子树如何向上传播。

        v -= 1

        adj[v].append((u, c))

    dfs(0, -1)

 ## 边缘情况

 单节点输入被正确处理，因为 DFS 分配$dp[1] = w_1$，且不处理任何边。 最大超过$dp$就是那个值。 

在每个边成本都大于相邻收益的链中，每个子贡献都变为非正，因此算法正确地在每个节点处停止而不是强制遍历。 这可以防止无效路径，否则会进入负中边缘。 

在只有一个分支有益的星形图中，所有其他子贡献都被过滤掉`gain > 0`健康）状况。 这确保了不相关的子树不会污染结果，即使它们直接附加到根。
