 ## 问题理解

 该问题描述了一个过程，在该过程中，我们实际上对特定算术序列是否产生可被给定整数整除的数字感兴趣$N$。 该序列是固定的，并从一个小的偏移量开始以恒定的步长增长：$2, 5, 8, 11, \dots$。 每个下一个值都是通过添加 3 获得的，因此序列可以视为以下形式的所有数字$a_i = 3i + 2$。 

对于每个测试用例，输入提供一个整数$N$。 任务是判断是否存在某个索引$i$这样$3i + 2$可以整除$N$。 如果存在这样的元素，则答案为正，否则为负。 

尽管该陈述是用组合或基于过程的解释来表达的，但计算核心是纯粹的数论：我们正在检查线性算术级数是否包含多个$N$。 

约束没有明确给出，但声明中的讨论暗示着潜在的巨大$T$和大$N$，朴素迭代达到$N$总体而言步骤太慢。 这立即排除了进程中任何每次测试的线性扫描。 

当人们假设“最终”每个$N$由于无限增长，将出现在序列中。 例如，与$N = 3$，序列是$2, 5, 8, 11, \dots$，并且这些都不能被 3 整除。错误来自于忽略序列模 3 的固定残基类。 

如果试图在不了解周期性的情况下仅检查一个小前缀，则会出现另一种误解。 例如，仅检查前几个术语可能会错误地表明解决方案存在或不存在，具体取决于$N$，但真正的行为取决于模块化结构，而不是有限采样。 

## 方法

 一种直接的方法是模拟序列并测试每一项的整除性。 我们生成$a_i = 3i + 2$并检查是否$a_i \bmod N = 0$。 这是正确的，因为它完全按照要求测试条件。 然而，在最坏的情况下，如果$N$很大，我们可能需要检查$N$候选人在得出不存在解的结论之前，因为余数模$N$重复句号$N$。 在许多测试用例中，这会导致总复杂度约为$O(T \cdot N)$，当两个值都很大时，速度太慢。 

关键的见解是停止思考生成大数，而是转向模块化算术。 顺序$3i + 2$具有模 3 的固定留数，始终等于 2。这意味着每个项都位于模 3 的单个同余类中。这些数字中的任何一个是否可以被整除$N$完全取决于残基类别$N$与该结构相互作用。 

如果$N$能被 3 整除，那么每个倍数$N$也可被 3 整除，因此它位于模 3 的残数类别 0 中。由于该序列卡在模 3 的残数类别 2 中，因此不可能有交集。 

如果$N \equiv 1 \pmod{3}$，然后乘以$N$除以 2 得到一个数字$2N$其模 3 的余数为$2 \cdot 1 = 2$，匹配序列。 所以序列的一个元素命中了的倍数$N$。 

如果$N \equiv 2 \pmod{3}$， 然后$N$它本身已经有余数 2 模 3，因此它与序列属于同一类，这意味着立即存在合适的索引。 

因此，整个问题就归结为检查是否$N$不能被3整除。

 ## 算法演练

 我们将每个测试用例简化为单个模块化检查。 

1.读取整数$T$，测试用例的数量。 每个测试用例都是独立的，因此它们之间不传递任何状态。 
2. 对于每个测试用例，读取整数$N$。 这是我们根据算术序列检查的值。 
3. 计算$N \bmod 3$。 这决定了哪个残留类别$N$属于相对于序列的固定结构。 
4.如果$N \bmod 3 = 0$，输出“否”。 这是根据以下事实得出的：所有倍数$N$位于残基类别 0 模 3 中，而所有序列元素位于残基类别 2 中，因此不可能有重叠。 
5. 否则输出“YES”。 在剩余的两种剩余情况下，存在建设性交集，这意味着序列的某些项等于$N$。 

### 为什么它有效

 顺序$a_i = 3i + 2$是模 3 的常数，始终等于 2。任何能被$N$必须有由以下测定的残留物$N$模 3。如果$N \equiv 0 \pmod{3}$，它的倍数无法匹配残数 2，因此不存在解。 如果$N \not\equiv 0 \pmod{3}$，乘以适当的因子对齐残基，保证某些倍数$N$属于同一算术级数类别。 由于序列和倍数集$N$是无穷大且周期模 3，留数相容性是必要且充分的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    out = []
    for _ in range(t):
        n = int(input())
        if n % 3 == 0:
            out.append("NO")
        else:
            out.append("YES")
    print("\n".join(out))

if __name__ == "__main__":
    solve()
```该解决方案读取所有测试用例并在恒定时间内独立处理每个测试用例。 每个案例的唯一计算是模运算，它直接捕获从算术级数导出的结构条件。 

决策规则的实现与派生条件完全相同：被 3 整除意味着失败，否则成功。 

一个常见的实现错误是尝试显式模拟序列或搜索倍数。 这是不必要的，因为序列的残基类完全消除了枚举的需要。

 ## 边缘情况

 对于$N = 3$，算法计算$3 \bmod 3 = 0$并立即返回“NO”。 序列仍然存在$2, 5, 8, \dots$，都与 2 模 3 全等，因此不能被 3 整除。 

对于$N = 6$，计算结构相同：$6 \bmod 3 = 0$，所以答案又是“NO”。 尽管 6 更大，但它的倍数仍然保留在残基类 0 模 3 中，它永远不会与序列的固定残基类相交。 

为了$N = 4$，我们得到$4 \bmod 3 = 1$，因此算法输出“YES”。 这对应于残基类别 2 模 3 中存在 4 的倍数，与级数一致。 该结构保证了这样的倍数存在，而无需显式搜索。
