 ## 解决方案
 ## 问题理解

 我们给定一个固定的整数$N$和一个整数范围$[L, R)$，表示所有整数$X$这样$L \le X < R$。 对于每一个这样的$X$，我们需要确定它是否满足涉及最大公约数的条件$N$。 

该条件简化为检查是否$\gcd(N, X)$等于$\min(N, X)$。 既然问题保证了$N \le L < R$, 每个候选人$X$范围内至少是$N$， 所以$\min(N, X)$总是$N$。 这减少了检查是否$\gcd(N, X) = N$，这相当于说$N$划分$X$。 

所以任务就变成了：计算中有多少个整数$[L, R)$是的倍数$N$，并输出该计数。 

约束条件意味着$R$可以非常大，可达约$10^{18}$，这会立即排除迭代范围内的每个整数。 线性扫描最多需要$10^{18}$步骤，即使忽略 gcd 计算的成本也是不可行的。 

一种微妙的边缘情况是由相差一的边界产生的。 由于区间是半开区间，$[L, R)$，值$R$其本身不得包括在内。 一个天真的包容循环$L$到$R$会错误地计数$R$当它能被整除时$N$。 

另一个问题出现在基于 gcd 的幼稚思维中：即使 gcd 计算效率很高，检查每个数字仍然在复杂性中占主导地位，使得 gcd 优化变得无关紧要。 

## 方法

 直接方法尝试每一种方法$X$从$L$到$R - 1$，计算$\gcd(N, X)$，并对匹配项进行计数。 这在概念上是正确的，因为它直接应用了定义。 然而，其成本随着区间的大小而线性增长。 什么时候$R - L$达到$10^{18}$，该算法根本无法使用。 

关键的观察是 gcd 条件崩溃为整除条件：我们正在计算$N$。 我们不是单独检查每个数字，而是转换视角并计算有多少个数字的倍数$N$位于给定界限以下。 这将问题从对值的迭代转换为对整数范围的算术。 

如果我们定义一个计算倍数的函数$N$严格低于某个值$S$，那么答案为$[L, R)$变成下面计数之间的差$R$和下面的计数$L$。 这些计数中的每一个都可以使用整数除法在恒定时间内计算出来。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(R - L)$|$O(1)$| 太慢了|
 | 最佳 |$O(1)$|$O(1)$| 已接受 |

 ## 算法演练

 1. 观察自$X \ge L \ge N$，表达式$\min(N, X)$简化为$N$。 这消除了对 gcd 条件结构的依赖，并将问题简化为整除性测试。 
2.更换条件$\gcd(N, X) = N$和$N \mid X$。 此步骤将问题从数论转换为计算算术级数。 
3. 将间隔计数重新表述为前缀计数的差异：计数的倍数$N$在$[L, R)$通过计算有多少倍数严格小于$R$，然后减去严格小于的数量$L$。 这避免了直接处理范围端点。 
4. 计算的倍数$N$小于某个值$S$使用整数除法。 每个倍数的形式为$kN$，以及最大的有效值$k$满足$kN < S$，这意味着$k \le \frac{S-1}{N}$。 
5. 应用公式$\left\lfloor \frac{R-1}{N} \right\rfloor - \left\lfloor \frac{L-1}{N} \right\rfloor$并输出结果。 

### 为什么它有效

 正确性取决于以下事实：$N$形成一个完全规则的算术级数。 前缀计数的转换确保了每个有效倍数$[L, R)$在前缀中只计算一次$R$并从前缀中排除，最多$L$。 减法消除了以下所有贡献$L$，将那些恰好留在目标区间内。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    N, L, R = map(int, input().split())
    
    def count_less(x):
        if x <= 0:
            return 0
        return (x - 1) // N
    
    ans = count_less(R) - count_less(L)
    print(ans)

if __name__ == "__main__":
    solve()
```辅助函数`count_less(x)`实现倍数的封闭形式计数$N$严格低于$x$。 表达式`(x - 1) // N`是整数形式$\lfloor (x-1)/N \rfloor$，直接计算大小的完整步长$N$之前适合$x$。 

减法结构确保我们永远不会显式迭代该范围。 边界处理通过严格的不等式进行编码`x - 1`，这避免了意外包含$x$本身。

 ## 边缘情况

 一种边缘情况是当$L$本身是一个倍数$N$。 例如，$N=4, L=4, R=9$。 正确答案包括 4。该公式可以处理这个问题，因为$count\_less(4)=0$和$count\_less(9)=2$，给出结果 2，对应于 4 和 8。 

另一种边缘情况是区间内根本不存在倍数，例如$N=10, L=11, R=19$。 两个前缀计数相等，产生零。 算术差异自然抵消了所有贡献。 

最终的边界情况非常大$R$。 由于计算仅使用除法，因此不会出现溢出或性能下降，并且 Python 的任意精度整数无需修改即可安全地处理达到最大约束的值。
