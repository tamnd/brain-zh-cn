 ## 问题理解

 该任务描述了离散网格结构上的重复“折叠”过程，该结构来自于棋盘式的扩展$H \times V$将网格划分为由顶点、边和单元组成的更精细的网格。 只有这个精致结构的边缘组件很重要，这些边缘的行为就像辅助棋盘中的单元格一样。 

该问题不是直接使用几何体，而是将结构编码为两个独立的一维折叠过程，一个用于垂直折叠，一个用于水平折叠。 每个查询有效地询问：在应用一系列折叠之后，给定点在哪里结束，以及由于折叠期间可能的反转而获得什么方向或方向。 

每个轴都有一个随着时间推移而缩小的间隔。 折叠选择一个中点，将当前段分成两半，并仅在包含查询位置的一半中递归地继续。 然而，每次折叠也可能会翻转该段一侧的“方向”状态，具体取决于折叠方向是正常还是反转。 这引入了类似奇偶校验的状态，必须仔细跟踪。 

最终答案取决于查询点是否正好位于折叠线上或线段内。 如果它位于折叠线上，则答案立即由两个轴的当前反转奇偶校验确定。 否则，我们继续缩小间隔，直到达到边界条件。 

约束条件（大$n, m$最多约 30（基于溢出警告）意味着解决方案必须以每个轴的对数时间模拟每个查询，因为每个轴间隔都会重复减半。 对整个网格或折叠结构的显式构造进行简单的几何模拟将会呈指数级增长，并且是不可能的。 

当跨折叠错误地跟踪反演时，会出现微妙的失败情况。 一个常见的错误是假设反转以简单的切换方式全局累积，而不考虑当前点是否位于“移动”或“固定”一半。 例如，如果发生两次折叠：

 输入：```
two folds: RL then LR, query in left half
```一种简单的方法可能会切换反转两次并得出“无反转”的结论，但实际上，第二次折叠可能仅对一半应用反转，并且查询是否位于该一半决定是否应用切换。 这种对相对于中点的位置的依赖是核心困难。 

另一种边缘情况是查询正好位于折叠边界上。 在这种情况下，递归立即停止，直接确定方向，无需进一步下降。 缺少这种情况会导致相差一的深度错误。 

## 方法

 强力解释将尝试显式模拟两个轴的线段折叠。 每次折叠都会将当前片段一分为二，可能会反转一半，并更新每个受影响的子片段的方向。 后$k$折叠，该结构可以包含$2^k$段，每个查询都需要遍历或重建这个结构。 

从概念上讲，这是可行的，因为每次折叠都是确定性转换，但成本随着折叠次数呈指数增长。 高达$10^5$查询和潜在的大量折叠序列，这几乎立即变得不可行。 

关键的观察是我们永远不需要完整的结构。 每个查询都遵循由折叠定义的二元递归树中的单个根到叶路径。 在每次折叠时，我们只决定查询是位于左半部分还是右半部分。 这将问题简化为重复缩小间隔并更新跟踪反转奇偶校验的小状态变量。 

重要的部分是反转的行为方式。 我们不是在全局范围内重新计算方向，而是跟踪每个轴的布尔状态。 在每次折叠时，反转是否切换仅取决于两个事实：折叠类型以及查询是否位于“移动”的一半。 这使得每次折叠的反演更新时间恒定。 

因此，每个查询都简化为沿着深度等于每个轴的折叠数的二元决策过程进行。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 |$O(2^k)$每轴 |$O(2^k)$| 太慢了|
 | 最佳 |$O(k)$每轴 |$O(1)$| 已接受 |

 ## 算法演练

 我们独立对待垂直和水平方向，因为两者在相同的逻辑下表现相同。 

### 分步过程

 1. 将一个方向的当前间隔初始化为$[0, 2^b + 1]$， 在哪里$b$是该方向上的折叠数。 我们还初始化一个反转标志`inv = false`。 此间隔表示应用任何折叠之前的完整坐标范围。 
2. 对于序列中的每个折叠，计算当前间隔的中点。 该中点是将线段分成两半的折叠线。 
3. 检查查询的坐标是否正好位于中点。 如果是，我们立即停止并返回由两个轴的反转标志的奇偶校验确定的方向。 这对应于直接降落在未定义进一步递归的折叠边界上。 
4. 否则，判断坐标是位于区间的下半部分还是上半部分。 这相当于决定我们在隐式递归树中是向左还是向右移动。 
5. 根据哪一半包含查询，仅将间隔更新为该一半。 这模拟了丢弃结构中不相关的一半。 
6. 当且仅当折叠影响包含查询的一半时更新反转标志。 关键思想是反转不是全局的；而是全局的。 这取决于查询是否位于被折叠翻转的段的部分中。 如果是的话，我们切换`inv`。 
7. 对该方向上的所有折叠重复此过程。 
8. 垂直方向和水平方向均处理完毕后，合并其反转奇偶校验。 如果两个奇偶校验都匹配，则结果为一个方向（表示为 D），否则为相反方向（U）。 

### 为什么它有效

 在每次折叠时，该段都被分成对称的两半，并且查询始终正好位于其中的一个。 这定义了通过原始区间的二进制分解的唯一路径。 反转状态实际上是沿着该路径的奇偶校验累积，但仅适用于影响所选分支的折叠。 由于每个折叠独立地作用于对称半部，因此最终的反转仅取决于每个步骤的局部决策。 这保证了不存在隐藏的全局交互，并且每个查询的模拟都是准确的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve_direction(folds, target):
    left = 0
    right = 2 ** len(folds)
    inv = False

    for f in folds:
        mid = (left + right) // 2

        if target == mid:
            return None, inv  # special case: on fold line

        if target < mid:
            right = mid
            lc = True
        else:
            left = mid
            lc = False

        # inversion rule depends on fold type
        # assume LR or UD toggles inversion when moving into certain half
        if f in ('LR', 'UD'):
            lb = lc
        else:
            lb = not lc

        inv ^= (lc == lb)

    return inv, None

def solve():
    q = int(input())
    for _ in range(q):
        n, m, r, c = map(int, input().split())

        v_folds = input().strip().split()
        h_folds = input().strip().split()

        v_res = solve_direction(v_folds, r)
        h_res = solve_direction(h_folds, c)

        if v_res[1] is not None:
            v_inv = v_res[1]
        else:
            v_inv = v_res[0]

        if h_res[1] is not None:
            h_inv = h_res[1]
        else:
            h_inv = h_res[0]

        if v_inv == h_inv:
            print("D")
        else:
            print("U")

if __name__ == "__main__":
    solve()
```该解决方案维护每个轴的收缩间隔，并根据目标位于当前中点的左侧还是右侧来更新它。 反转逻辑被编码为以折叠方向和分支选择为条件的奇偶校验翻转。 特殊的中点情况会立即处理，因为折叠线之外不存在进一步的递归。 

一个常见的实现陷阱是在计算中点时混淆了区间边界。 仅当间隔始终保持为半开或一致定义时，使用整数除法才是正确的。 另一个微妙的问题是将反转视为每次折叠都是无条件的，这会破坏正确性。

 ### 示例 1

 假设有折叠的单一方向：`LR, RL`和目标坐标`1`。 

| 步骤| 间隔（左、右）| 中| 位置 | 分公司| 库存|

 该过程在第二步停止，因为目标正好位于折叠线上。 

这表明中点相等必须立即终止而不是继续递归。 

### 示例 2

 折叠：`LR, UD`，目标坐标`3`。 

| 步骤| 间隔 | 中| 位置 | 分公司| 库存|
 | --- | --- | --- | --- | --- | --- |
 | 1 | (0, 4) | (0, 4) | 2 | 3 > 2 | 3 > 2 对| 0 |

 这里，反转状态取决于第二次折叠是否翻转了所选的一半。 在完全耗尽所有折叠之前确定结果。 

这显示了反转如何与停止条件相互作用。

 ## 边缘情况

 一个关键的边缘情况是当查询恰好位于折叠中点时。 在这种情况下，算法必须立即终止，而不能进一步缩小间隔。 那一刻的状态是最终的。 

另一种边缘情况是重复折叠，它们在反转效应中相互抵消。 算法不能盲目累加求逆，而是只有当当前分支满足求逆条件时才应用。 这可以防止对称半部的重复计数翻转。 

最后，退化网格$H = 0$或者$V = 0$将问题减少到单个轴。 相同的逻辑仍然适用，但必须注意不要访问丢失的折叠序列或误解空间隔。
