 ## 解决方案
 ## 问题理解

 我们正在处理一个简单的指数增长模型，其中病毒的初始数量每秒以固定的乘法因子扩展。 形式上，当时的病毒数量$t$遵循以下形式的幂函数$v(t) = b \cdot q^t$，其中初始金额$b$和生长因子$q$是未知整数。 

而不是被给予$b$和$q$，我们给出了该过程的两个观察到的测量值：$v(t)$在两个不同的时间。 从指数曲线上的这两点，我们被要求重建基础参数并输出完整的公式$v(t)$。 

因此，输入表示未知指数函数的两个约束，输出是该函数的显式封闭形式，简化为具体的整数值。 

尽管该结构乍一看似乎不确定，但两次测量就足够了，因为指数增长将除法转变为初始值的彻底消除。 

不存在导致算法复杂性问题的复杂数据结构或大输入约束。 整个问题简化为整数算术以及对幂和根的仔细处理。 唯一微妙的失败情况来自浮点精度和格式，特别是当中间计算不以整数形式保存时。 

将表达式重写为替代的等价形式时，会出现一个常见的陷阱，例如$v(t) = 21 \cdot 7^{t-1}$。 虽然有效数字在代数上与实数相同$t$，该表示法在评估时引入了分数值$t = 0$，这可能会导致浮点输出，例如`3.0`而不是`3`，违反严格的输出格式要求。 

## 方法

 思考该问题的一种直接方法是将其视为具有两个未知数的两个方程。 从模型来看$v(t) = b \cdot q^t$，我们立即得到：$v_3 = b \cdot q^3$和$v_6 = b \cdot q^6$。 

一个蛮力的想法是尝试所有合理的整数值$b$和$q$，模拟两个方程，并检查一致性。 如果值很小，这会起作用：迭代可能的值$q$, 计算$b$从一个方程出发，并根据第二个方程进行验证。 

这种强力方法是正确的，因为每个候选对都针对这两个约束进行了显式测试。 然而，即使对于适度的界限，它也是不必要的，并且在概念上也变得浪费，因为可能性的空间随着输入的大小而增长，并且我们没有使用求幂的结构。 

关键的观察结果是，将两个方程相除消除了$b$完全地。 这将问题从双变量搜索转换为单变量提取：$$\frac{v_6}{v_3} = \frac{b q^6}{b q^3} = q^3$$这将整个结构折叠成一个纯功率方程。 一次$q^3$已知，提取$q$变成立方根问题。 在那之后，$b$直接代入任一方程即可得出。 

这是关键的简化：指数增长将观测值的比率转化为基数的干净幂。

 ## 算法演练

 1. 阅读给出的该过程的两个观察结果：$(t_1, v_1)$和$(t_2, v_2)$。 在这个问题中，这些对应于$(3, v_3)$和$(6, v_6)$。 
2. 计算比率$r = \frac{v_2}{v_1}$。 此步骤是有效的，因为两个值都来自同一指数函数，因此未知乘数完全抵消。 这隔离了生长因子的影响。 
3. 将比率解释为底数的幂：$r = q^{t_2 - t_1}$。 由于时间差为3，我们得到$r = q^3$。 
4. 计算$q$通过提取整数立方根$r$。 这必须作为整数运算小心完成，因为$q$保证是预期结构中的整数。 
5. 恢复$b$使用前面的方程$v_1 = b \cdot q^3$。 重新排列给出$b = \frac{v_1}{q^3}$。 这种划分是通过构造精确进行的。 
6.输出最终函数$v(t) = b \cdot q^t$简化的整数形式。 

### 为什么它有效

 正确性来自于两个测量值位于具有相同基数的相同指数曲线上的不变量$q$。 任何两点之比都会消除系数$b$，只留下纯粹的力量$q$。 由于指数差直接转化为指数减法，因此系统简化为单个未知指数方程。 一次$q$唯一确定为比率的整数立方根，代回恢复$b$完全没有歧义。 没有其他整数对可以同时满足这两个方程，因此重构是唯一的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def cube_root(x):
    lo, hi = 0, int(1e6)
    while lo <= hi:
        mid = (lo + hi) // 2
        val = mid * mid * mid
        if val == x:
            return mid
        if val < x:
            lo = mid + 1
        else:
            hi = mid - 1
    return hi

v3 = 1029
v6 = 352947

ratio = v6 // v3
q = cube_root(ratio)
b = v3 // (q ** 3)

print(f"{b} * {q}^t")
```实现直接反映了推导过程。 第一步修复给定的观察结果。 比率计算使用整数除法，因为该问题保证精确整除。 立方根是通过二分搜索计算的，以避免浮点错误，因为在验证整数相等时，即使很小的精度问题也会破坏正确性。 

确定后$q$, 基数$b$通过直接替换来恢复。 最终输出以符号形式打印重构的公式，与预期格式匹配。

 ## 边缘情况

 当比率是完美立方但使用浮点运算来提取根时，会出现最敏感的边缘情况。 例如，如果$v_3 = 3$和$v_6 = 1029$，比率为$343$。 使用`round(ratio ** (1/3))`产生风险`6.999999999`根据精度，它会错误地舍入为 7 或有时为 6，从而导致错误的重建。 

正确的基于整数的方法可以完全避免这种情况。 二分查找计算立方不超过比例的最大整数，保证准确性。 

另一个微妙的情况是使用代数变换时的输出格式。 写作$v(t) = 21 \cdot 7^{t-1}$产生正确的值$t \ge 1$，但是如果系统曾经评估或打印过$t = 0$，表达式变为$21 / 7$，可以表示为`3.0`以浮点形式。 所需的输出格式需要没有小数点的整数表示，因此保留规范形式$b \cdot q^t$避免此类错误。 

最终的结构边缘情况是$q = 1$。 在这种情况下，两个测量值相等，比率变为 1。该算法仍然有效：1 的立方根为 1，并且$b$直接恢复为$v_3$，无需特殊外壳即可保持正确性。
