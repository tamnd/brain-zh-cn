 ## 解决方案
 ## 问题理解

 我们得到一长串值，代表一天中不同时刻的客流。 “班次”由三个参数定义： 开始时间索引`s`, 固定行程次数`k`，以及恒定的时间间隔`d`连续行程之间。 

如果我们在某个时间开始`s`, 地铁有时运行`s, s + d, s + 2d, ..., s + (k−1)d`。 此班次的总乘客负载是这些位置处的数组值的总和。 任务是选择起始位置`s`因此，在所有选定索引必须保留在数组内的约束下，总和尽可能大。 

形式上，我们正在最大化数组内固定算术级数的总和。 对有效性的约束是最后一个索引`s + (k−1)d`不得超出数组边界。 

输入大小足够大，任何迭代所有有效开始并直接重新计算每个总和的解决方案都会太慢。 一种简单的方法可以有效地计算`O(N · k)`操作，折叠到大约`10^10`在最坏的情况下配置是不可行的。 

一个关键的微妙之处是逐步`d`创建一个不是单个连续段的结构，除非`d = 1`。 什么时候`d > 1`，索引分为独立的残基类模`d`，每个类的行为就像它自己的较小的线性数组。 

朴素滑动窗口方法的一个典型失败案例是当人们试图将其视为正常子数组问题时`d > 1`。 例如，如果`d = 2`，和中的相邻元素在原始数组中不相邻，因此连续段上的标准前缀或滑动窗口不适用。 

## 方法

 暴力解决方案确定了一个起点`s`并明确求和`k`元素间隔为`d`。 这是正确的，因为它直接遵循定义。 然而，每次这样的计算成本`O(k)`，并且最多有`O(N)`可能的起点，因此总复杂度变为`O(Nk)`。 当两者`N`和`k`很大，这很快就会变得太慢。 

关键的观察是由步骤定义的算术级数`d`将数组划分为独立的链。 任意索引`i`仅与以下形式的索引交互`i + td`。 这意味着如果我们按索引模对元素进行分组`d`，每个组形成一个单独的序列，其中问题简化为选择一个连续的长度段`k`在该序列内。 

一旦进行了分解，就可以使用前缀和或滑动窗口来解决每个组，就像经典的固定长度最大子数组和问题一样。 由于每个元素都属于一个组，因此所有组的总工作量呈线性关系`N`。 

| 方法| 时间复杂度| 空间复杂度| 判决 |
 | --- | --- | --- | --- |
 | 蛮力 | O(Nk) | O(1) | O(1) | 太慢了|
 | 残基分解/前缀或窗口| O(N) | O(N) | 已接受 |

 ## 算法演练

 我们通过基于索引模将数组分割成独立的链来解决这个问题`d`。 

1. 我们按余数类模对元素进行分组`d`。 当我们确定起始残基类别时，每个组都包含将在算术级数中连续出现的元素。 这一步是必要的，因为逐步`d`切勿混合不同的残留类别。 
2. 对于每个残留类别`r`，我们提取序列`b = a[r], a[r+d], a[r+2d], ...`。 这将原始的“跳过索引”问题转换为标准线性数组问题。 
3. 在每个序列上`b`，我们精确计算任何连续长度段的最大总和`k`。 这与计算大小的滑动窗口相同`k`。 我们初始化第一个窗口的总和，然后通过删除传出元素并添加传入元素来移动它。 
4. 我们跟踪所有残留类别中所有窗口的最大值。 该全局最大值对应于原始数组中的最佳有效起始位置。 

### 为什么它有效

 从索引开始的每个有效算术级数`s`完全属于单一残基类别`s mod d`。 在该类中，进程成为一个连续的块。 因此，每个候选班次恰好对应于一个类中的一个窗口，并且每个这样的窗口对应于一个有效班次。 由于我们评估所有类别中的所有窗口，因此我们不会错过任何候选者，也不会混合不相关的索引。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

def solve():
    n, k, d = map(int, input().split())
    a = list(map(int, input().split()))

    if k == 1:
        print(max(a))
        return

    ans = -10**30

    for r in range(d):
        # build window on residue class r
        window = []
        s = 0

        for i in range(r, n, d):
            window.append(a[i])
            s += a[i]

        if len(window) < k:
            continue

        cur = sum(window[:k])
        ans = max(ans, cur)

        for i in range(k, len(window)):
            cur += window[i] - window[i - k]
            ans = max(ans, cur)

    print(ans)

if __name__ == "__main__":
    solve()
```该解决方案读取数组并迭代所有残基类模`d`。 对于每个类，它构造一个简化数组，表示通过步进可到达的所有位置`d`。 然后它使用长度为固定大小的滑动窗口`k`有效地计算总和。 

重要的实现细节是我们从不尝试直接在原始索引系统中计算总和。 所有计算都发生在压缩的残差序列中，这保证了线性总工作。

 ## 边缘情况

 对于`k = 1`，算法应立即返回最大元素。 滑动窗口逻辑仍然有效，但如果没有防护，它可能会错误地假设至少存在一个转换。 

对于以下情况`len(b) < k`在剩余课程中，必须完全跳过该课程。 否则，将错误地考虑部分窗口。 

对于大型`d`附近`n`，大多数残基类最多包含一个元素，这意味着不存在有效窗口，除非`k = 1`。 该算法自然会处理这个问题，因为在这种情况下滑动循环永远不会运行。
