 ## 解决方案
 ## 问题理解

 给定一个二分图\(K_{n,m}\)第一个在哪里\(n\)顶点形成一侧和下一侧\(m\)顶点形成另一边。 左侧的每个顶点都连接到右侧的所有顶点，并且两侧内部都没有边。 

在此图上执行随机游走。 首先，从所有顶点中均匀随机选择一个起始顶点\(n+m\)顶点。 然后，在每一步中，我们从当前顶点移动到均匀随机的邻居。 我们第一时间停止进程\(T\)当图中的每个顶点都至少被访问过一次时。 任务是计算期望值\(T\), 取模\(998244353\)，即作为模分数。 

约束条件允许\(n, m \le 1000\)，所以图的大小最多为 2000 个顶点。 访问节点子集上的朴素马尔可夫链是不可能的，因为状态空间将是\(2^{2000}\)。 即使存储每个状态的概率也是不可行的。 

一个微妙的点是，行走并不是以一种简单的方式对称的：同一侧的顶点表现相同，但过渡结构在很大程度上取决于我们是在左侧还是右侧。 停止条件取决于完全覆盖两个分区。 

一些边缘案例暴露了幼稚的推理问题。 如果\(n=1, m=1\)，无论从哪一开始，两个顶点都会在一步中立即访问。 期望值正好是 1。如果\(n=1, m>1\)，如果我们从右侧开始，则总是立即访问左侧顶点，但到达所有右侧顶点取决于星形结构，其中中心是唯一的桥。 任何假设在所有节点上独立收集优惠券的方法都会失败，因为访问通过交替侧具有很强的相关性。 

## 方法

 暴力尝试会将过程建模为由 \((u, S)\) 定义的状态上的马尔可夫链，其中\(u\)是当前节点并且\(S\)是访问过的节点的集合。 从每个状态，我们过渡到邻居并更新访问集，累积到达吸收状态的预期命中时间，其中\(S\)包含所有顶点。 这在原则上是正确的，因为一旦知道完整状态，该过程就是无记忆的。 

然而，状态的数量是\((n+m)\cdot 2^{n+m}\)，这是一个天文数字。 即使是为了\(n=m=20\)，这已经变得不可行了。 

关键的观察结果是该图只有两个对称类的顶点。 所有左顶点都相同，所有右顶点也相同。 我们不需要跟踪单独的访问集，而是只需要跟踪到目前为止每侧有多少个顶点被访问过，以及我们当前位于哪一侧。 该过程变成了一个基于计数而不是子集的小马尔可夫链。 

从我们位于左侧的状态开始，下一步总是到达统一选择的某个右侧顶点。 从右侧的状态，我们总是到达左侧的某个顶点。 唯一重要的是下一个顶点是新的还是已经见过的。 这减少了跟踪访问次数 \((i, j)\) 的过程，其中\(i\)左顶点和\(j\)已发现正确的顶点以及当前的边。 

这将问题转化为大小为 \(O(nm)\) 的网格上的两层动态规划，其中的转换由优惠券收集器类型的概率驱动，该概率仅取决于对面有多少个看不见的顶点。 

| 方法| 时间复杂度| 空间复杂度| 判决|
 |---|---|---|---|
 | 完整子集 DP | \(O((n+m)2^{n+m})\) | \(O(2^{n+m})\) | 太慢了 |
 | 对称 DP 计数 | \(O(nm)\) | \(O(nm)\) | 已接受 |

 ## 算法演练

 我们定义了两个DP表。 让\(E_L[i][j]\)是当我们当前位于左顶点时预计要完成的剩余步骤，已经看到\(i\)左顶点和\(j\)右顶点。 相似地，\(E_R[i][j]\)当我们位于右顶点时定义。 

吸收条件为\(i=n\)和\(j=m\)，无论哪一边，期望值都为零。 

从左侧状态开始，下一步总是转到统一选择的右侧邻居。 有\(m\)右顶点，其中\(m-j\)是新的并且\(j\)已经被看到了。 因此，我们根据是否发现新的右顶点来转换到新的状态。 

从右侧状态开始，对称逻辑适用，但角色相反。 

我们按递减顺序向后计算期望\(i+j\)，因为每次转换要么增加访问顶点的数量，要么保持相同但改变一侧。 为了避免循环，我们将方程重新组织成线性形式。 

核心递归来自直接编写期望方程。 从左侧状态：

 1. 我们总是立即花费一步，所以加 1。 
2. 我们匀速移动到右顶点。 
3.有概率\(\frac{m-j}{m}\)，我们增加\(j\)1.
 4.有概率\(\frac{j}{m}\),\(j\)保持不变。 
5. 在这两种情况下，我们都切换到右侧状态。 

所以我们得到一个线性方程：\[
E_L[i][j] = 1 + \frac{m-j}{m} E_R[i][j+1] + \frac{j}{m} E_R[i][j]
\]相似地：\[
E_R[i][j] = 1 + \frac{n-i}{n} E_L[i+1][j] + \frac{i}{n} E_L[i][j]
\]这些方程不是直接 DP 转换，因为\(E_L[i][j]\)通过间接出现在两侧\(E_R[i][j]\)学期。 我们通过以\((n-i)+(m-j)\)降序迭代状态来解决它们，确保右侧的任何未知数都已被计算。 

### 为什么它有效

 在每个状态\((i,j)\)，该过程仅取决于下一步移动是否在相反分区上发现新顶点。 分区内的所有顶点都是对称的并且在过渡动力学下无法区分。 这种对称性将子集的指数状态空间折叠成计数的多项式状态空间。 

这种排序确保每当我们评估一个状态时，所有具有严格访问次数较多的顶点的状态都已被解析。 任何自相关性都是线性的，并通过重新排列期望方程来解决，因此评估顺序中不存在循环相关性。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

def inv(x):
    return pow(x, MOD - 2, MOD)

def solve():
    n, m = map(int, input().split())

    inv_n = inv(n)
    inv_m = inv(m)

    E_L = [[0] * (m + 1) for _ in range(n + 1)]
    E_R = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(n, -1, -1):
        for j in range(m, -1, -1):
            if i == n and j == m:
                continue

            # compute E_L[i][j]
            if j < m:
                p_new = (m - j) * inv_m % MOD
                p_old = j * inv_m % MOD
                val_L = (1 + p_new * E_R[i][j + 1] + p_old * E_R[i][j]) % MOD
            else:
                val_L = (1 + E_R[i][j]) % MOD

            E_L[i][j] = val_L

            # compute E_R[i][j]
            if i < n:
                p_new = (n - i) * inv_n % MOD
                p_old = i * inv_n % MOD
                val_R = (1 + p_new * E_L[i + 1][j] + p_old * E_L[i][j]) % MOD
            else:
                val_R = (1 + E_L[i][j]) % MOD

            E_R[i][j] = val_R

    # initial expectation averages over starting point
    total = (E_L[1][0] * n + E_R[0][1] * m) % MOD
    total = total * inv(n + m) % MOD
    print(total)

if __name__ == "__main__":
    solve()
```DP 表存储以双方为条件的期望值和访问次数。 这些转换编码了发现新顶点与重新访问已知顶点的概率。 

自下而上的迭代之所以有效，是因为在计算 \((i,j)\) 时，所有具有较高值的​​状态\(i\)或者\(j\)已经满了。 边界\(i=n, j=m\)为零，因为不需要进一步的步骤。 

最终答案对起始点进行平均。 从左边开始有贡献\(E_L[1][0]\)从右边开始有助于\(E_R[0][1]\)，按其选择概率加权。 

## 工作示例

 ### 示例 1：\(n=1, m=1\)我们从任一顶点开始，一步一步立即到达另一个顶点。 

| 状态| L 的值 | R 的值 |
 |---|---|---|
 | (1,1) | 0 | 0 |
 | 初始| E_L[1][0]=1 | E_R[0][1]=1 |

 最终的期望是1，符合一步总是完成覆盖的直接推理。 

这证实了边界条件和平均步骤与平凡图一致。 

### 示例 2：\(n=2, m=1\)这里的图是以单个右侧节点为中心的星形。 

| 状态| E_L[i][j] | E_R[i][j] |
 |---|---|---|
 | (2,1) | 0 | 0 |
 | (1,1) | 1 | 1 |
 | (0,1)| 2 | - |

 从左节点开始，我们到达中心，然后可能到达另一个左节点。 重新访问中心的 DP 捕获不会贡献新的发现，迫使重复尝试，直到找到第二个左节点。 

这表明递归可以正确处理重复访问，而不会重复计算进度。

 ## 边缘情况

 对于\(n=1, m=1\)，DP 立即崩溃，因为 \((1,1)\) 已经是终结符。 算法集\(E_L[1][1]=E_R[1][1]=0\)，并且两个初始状态一步直接转变为终止状态，在求平均后产生 1 的期望。 

为了\(n=1, m=1000\)，该过程的行为类似于重复采样右侧顶点，直到看到所有顶点，但始终通过单个左侧顶点返回。 DP 正确地编码左侧充当确定性桥梁，并且仅当发现新的右侧顶点时才会发生进展。 

为了\(n=1000, m=1000\)，每个状态都是完全对称的，DP 减少到网格上的平滑梯度。 递归确保不会出现循环依赖性，因为每次转换要么增加覆盖范围，要么使用更高覆盖级别的已计算状态。
