 ## 解决方案
 ## 问题理解

 我们得到一个无向简单图\(n\)顶点和\(m\)边缘。 该图可能已经包含多个连接的组件，这意味着某些顶点组可以在内部相互到达，但不同组之间可能没有路径。 

任务是添加尽可能少的新边，以便添加它们后整个图变得连通。 我们还需要输出一组有效的此类边，不一定是唯一的。 

输入描述任意图结构。 输出描述了我们选择插入的额外边。 每条添加的边必须连接两个不同的顶点，并且对于该边是否已存在于输入中没有任何限制，除了我们可以自由地假设我们避免了重复，因为我们自己选择了边。 

约束条件\(n, m \le 10^5\)意味着任何解决方案都必须在线性或近线性时间内运行。 尝试所有节点对或重复检查任意顶点之间的连接性的二次方法会太慢，因为\(n^2\)操作已经\(10^{10}\)，远远超出了可行的限度。 

关键的结构边缘情况是当图已经连接时。 在这种情况下，正确的答案是零添加边缘。 另一种边缘情况是当有许多孤立的顶点时，例如\(n = 5, m = 0\)，其中每个顶点都是其自己的组件。 那么解决方案必须将它们全部精确地连接起来\(n-1\)边形成一棵树。 

如果我们在没有首先了解组件结构的情况下尝试贪婪地添加边缘，则会出现微妙的故障模式。 例如，如果我们随机连接顶点而不确保桥接不同的组件，我们可能会在同一组件内添加冗余边，浪费允许的操作，并且可能无法最小化\(k\)。 

## 方法

 一个幼稚的策略是重复选择任何两个未知的连接顶点并在它们之间添加一条边。 每次添加之后，我们都会重新计算连通性，例如从头开始使用 DFS 或 BFS，直到整个图变得连通。 

这在逻辑上是有效的，因为每个添加的边只能合并组件或不执行任何操作，但在每个边之后重新计算连接性的成本很高。 每个 DFS 的成本为 \(O(n + m)\)，我们可能需要最多\(n\)添加，导致 \(O(n(n+m))\)，这对于\(10^5\)节点。 

关键的观察是，最终的连通图必须仅由一个组件组成，因此我们只需要合并现有的连通组件。 如果我们首先计算所有连接的组件，则每个组件都可以由单个代表节点表示。 一旦我们有了这些代表，连接组件就相当于连接链中的这些代表节点。 一种简单的方法是将组件 1 连接到组件 2，将组件 2 连接到组件 3，依此类推。 这正是使用\(c-1\)边缘在哪里\(c\)是连通分量的数量，它是最小的，因为每添加一条边最多可以减少一个分量的数量。 

| 方法| 时间复杂度| 空间复杂度| 判决|
 |---|---|---|---|
 | 每条边后重复 DFS | \(O(n(n+m))\) | \(O(n+m)\) | 太慢了 |
 | 连接组件+链接| \(O(n+m)\) | \(O(n)\) | 已接受 |

 ## 算法演练

 我们首先识别图中所有连接的组件。 这可以使用 DFS 或 DSU 来完成。 在此过程中，我们为每个顶点分配一个组件标识符，并为每个组件记录一个代表顶点。 

一旦确定了组件，我们就会将代表收集到列表中。 假设有\(c\)组成部分和代表\(r_1, r_2, \dots, r_c\)。 

然后我们通过连接来构造答案\(r_1\)到\(r_2\),\(r_2\)到\(r_3\)，依此类推，直到\(r_{c-1}\)到\(r_c\)。 每条这样的边都会合并两个不同的组件，因此在所有添加之后，图就变成了连通的。 

最后，我们输出添加的边的数量\(c-1\)，然后是我们构建的边。 

这是正确的原因是连接的组件形成了顶点集的一个分区，并且两个不同组件之间的任何边严格地将组件数量减一。 开始于\(c\)组件，至少\(c-1\)需要边缘才能将它们合并为单个组件。 构建的链恰好达到了这个下限，因此它是最优的。 

## Python 解决方案```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**7)

n, m = map(int, input().split())
g = [[] for _ in range(n + 1)]

for _ in range(m):
    u, v = map(int, input().split())
    g[u].append(v)
    g[v].append(u)

vis = [False] * (n + 1)
components = []

def dfs(start):
    stack = [start]
    vis[start] = True
    rep = start
    while stack:
        u = stack.pop()
        rep = u
        for v in g[u]:
            if not vis[v]:
                vis[v] = True
                stack.append(v)
    return rep

for i in range(1, n + 1):
    if not vis[i]:
        components.append(dfs(i))

k = len(components) - 1
print(k)
for i in range(k):
    print(components[i], components[i + 1])
```该图是使用邻接列表构建的，因此遍历是线性的\(n + m\)。 DFS 为每个组件收集一名代表。 每个未访问的节点都会触发一个新的 DFS，确保所有组件都被恰好发现一次。 

最后的循环连接连续的代表。 不需要额外的验证，因为保证代表来自不同的组件。 

一个常见的错误是尝试联合操作而不跟踪代表，这导致不知道要输出哪些实际边。 在这里，我们明确地为每个组件存储一个顶点，使输出构造变得简单。

## 复杂度分析

 | 测量| 复杂性 | 说明|
 |---|---|---|
 | 时间 | \(O(n + m)\) | 每个顶点和边在 DFS 期间都会被访问一次 |
 | 空间| \(O(n + m)\) | 邻接表加上访问数组和递归/堆栈 |

 约束允许最多\(10^5\)节点和边，因此线性时间遍历非常适合时间限制。 内存使用也是线性的，并且完全在竞争性编程环境的典型限制内。

 ## 边缘情况

 不需要额外边的全连接图可以自然地处理，因为来自节点 1 的 DFS 已经访问了所有节点，恰好生成了一个组件。 代表列表的大小为 1，因此添加的边数为零，并且不打印输出边。 

一个完全空的图，其中\(m = 0\)，产生\(n\)隔离的组件。 DFS 循环为每个节点启动一个新组件，收集所有顶点作为代表。 然后，该算法将它们连接成一条链，确保精确连接\(n-1\)边缘。 

已经分成两个大组件的图的行为类似。 例如，如果节点\(1..k\)形成一个组件并且\(k+1..n\)对于另一个，该算法从每个中选择一个代表并用一条边将它们连接起来。 这是最优的，因为没有一条边可以将组件数量减少一个以上。
