 ## 解决方案
 ## 问题理解

 我们得到了几个测试用例。 在每个测试用例中，有 n 个实数，每个实数代表数轴上的一个点。 从这些点中，我们必须精确选择 k 个不相交的点对，这意味着每个点最多可以用于一个选定的点对中。 每对贡献一个等于两个端点的绝对差的值，目标是确定两个极端：所有 k 对的最小可能总和以及最大可能总和。 

该结构本质上是要求我们在一条线上的一组点上建立一个大小 k 的匹配，并在该约束下优化边长度的总和。 

这些限制以非常直接的方式产生影响。 所有测试用例的总点数最多为 10^6，因此任何总体上比大约 O(n log n) 更差的解决方案都将无法生存。 这立即排除了任何尝试考虑所有对组合或使用 n 中的二次动态规划的情况。 这个问题也是围绕排序清晰地构建的，因为只有在对点进行排序后，线上的距离才变得有意义。 

当点聚集或重复时，会出现微妙的边缘状况。 如果多个点共享相同的坐标，则零长度边成为可能，并且必须正确处理。 当 k 接近 n/2 时，会出现另一种重要的情况，此时几乎所有点都被使用，而 k 较小时，仅选择少数对。 该策略必须在两个极端中表现一致。 

一种简单的方法是枚举 k 对的所有可能匹配，计算它们的总和，并跟踪最小值和最大值。 即使我们只是尝试贪婪地生成匹配，选择 k 个不相交对的方法数量也会以组合方式增长，因此即使 n 约为 30，这也很快变得不可行。 

## 方法

 蛮力的观点是想尽一切可能的方法从 n 个点中挑选 k 个不相交的对。 人们可以想象生成 2k 点的所有子集，然后以所有可能的方式将它们配对。 这原则上是正确的，因为每个有效的解决方案都对应于 2k 点的某个子集以及其中的完美匹配。 问题是即使选择子集也已经是\(\binom{n}{2k}\)，并且其中的匹配数量呈阶乘增长。 这远远超出了任何可行的计算。 

关键的结构观察是点位于一条线上。 一旦我们对它们进行排序，距离就索引分离而言表现得单调。 这实现了极大的简化：最佳解决方案不需要以复杂的方式在交叉索引之间进行任意配对。 相反，我们可以根据订购和本地交换进行推理。 

对于最大总和，目标是最大化距离，这促使我们将相距较远的点配对。 排序后，最好的策略是集中精力在极端情况上。 如果我们选择任何 2k 个点的集合，该集合中的最佳配对是将最小的点与最大的点匹配，将第二小的点与第二大的点匹配，依此类推。 这种结构意味着全局最优是通过从数组中取出 k 个最小点和 k 个最大点并将它们配对来实现的。 

对于最小总和，我们想要相反的行为：将点配对尽可能靠近。 排序后，自然候选者是相邻元素，因为任何非相邻配对都可以通过向内交换端点来改进。 这将问题简化为从路径中选择 k 个不相交的边，其中边仅存在于连续点之间，并且每条边的权重等于相邻差值。 

这成为路径图上大小为 k 的最小权重匹配。 贪婪策略有效：重复选择最小的可用相邻间隙，采用该对，并从进一步考虑中删除两个端点。 该过程持续进行，直到形成 k 对。

 |---|---|---|---|
 | 蛮力 | 指数| 大| 太慢了 |
 | 最佳 | O(n log n) | O(n log n) | O(n) | 已接受 |

 ## 算法演练

 首先，我们按非降序对所有点进行排序。 排序至关重要，因为有关最佳配对的所有推理都取决于表示线路上邻近程度的排序顺序中的邻接关系。 

其次，我们计算所有相邻差异。 如果我们决定使用本地配对，每个差异代表配对两个相邻点的成本。 

第三，为了获得最大答案，我们只关注排序数组的外部结构。 我们取 k 个最小元素和 k 个最大元素。 我们将最小的与最大的配对，将第二小的与第二大的配对，依此类推。 总贡献被累积为这 k 个差异的总和。 

第四，对于最小答案，我们将每个相邻对视为路径中的候选边。 我们维护一个允许我们重复选择最小可用边的结构。 一旦选择了 i 和 i+1 之间的边，两个索引都被标记为不可用，因此不会形成重叠对。 

第五，我们重复这个选择过程，直到我们选择了 k 个边。 每个选定的边将其相应的相邻差值贡献给总差值。 

最终的输出是累积的最小和最大总和。 

这样做的原因是基于两个方向的交换论证。 为了最大化，任何使用内部结构的配对都可以通过向外交换端点来改进，增加总距离。 这将所有群体推向极端配对。 至少，任何不相邻的配对都可以通过向内收缩端点直到它们变得相邻来局部改进，并且在相邻选择中，首先选择较小的间隙永远不会损害可行性，因为冲突是通过删除已使用的点来处理的。 

## Python 解决方案```python
import sys
import heapq

input = sys.stdin.readline

def solve():
    T = int(input())
    for tc in range(1, T + 1):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))
        a.sort()

        # maximum: pair k smallest with k largest
        max_ans = 0
        for i in range(k):
            max_ans += a[n - 1 - i] - a[i]

        # minimum: greedy on adjacent gaps
        if k == 0:
            min_ans = 0
        else:
            used = [False] * (n - 1)
            heap = []
            for i in range(n - 1):
                heapq.heappush(heap, (a[i + 1] - a[i], i))

            taken = 0
            min_ans = 0

            while heap and taken < k:
                w, i = heapq.heappop(heap)
                if used[i]:
                    continue
                if used[i + 1]:
                    continue

                used[i] = used[i + 1] = True
                min_ans += w
                taken += 1

        print(f"Case #{tc}: {min_ans} {max_ans}")

if __name__ == "__main__":
    solve()
```代码中的最大部分直接实现了排序后的极值配对参数。 索引对称性确保每个选定的对都是独立的，并且准确地贡献相对端之间的距离。 

最小部分构造所有相邻差异并使用堆来始终选择最小的剩余候选者。 这`used`数组确保一旦一个点被消耗，所有涉及它的边都会隐式失效。 这就是为什么跳过的堆条目在弹出时被简单地忽略。 

这里的一个常见错误是试图在不强制不相交的情况下选择 k 个最小间隙。 当两个小间隙共享一个点时，这种方法就会失败，因为不能同时选择两者。 

## 工作示例

 考虑输入`a = [1, 3, 4]`,`k = 1`。 

对于最大值，排序给出相同的数组。 我们将 1 和 4 配对，给出距离 3。对于最小值，最小相邻间隙在 3 和 4 (1) 之间，因此我们选择该对。 

| 步骤| 堆/对选择 | 二手 | 当前总和|
 |---|---|---|---|
 | 1 | (3,4) | {3,4} | 1 |

 这表明贪心最小策略优先考虑局部紧密度。 

现在考虑`a = [2, 1, 4, 3]`,`k = 2`，这排序为`[1,2,3,4]`。 

为了获得最大值，我们将 (1,4) 和 (2,3) 配对，得到 3 + 1 = 4。 

对于最小值，相邻间隙为 1,1,1。 堆可以先选择任意 1，例如 (1,2)，然后选择 (3,4)。 

| 步骤| 选定的边缘 | 二手 | 总和 |
 |---|---|---|---|
 | 1 | (1,2) | {1,2} | 1 |
 | 2 | (3,4) | {3,4} | 2 |

 这证实了局部贪婪选择产生了有效的不相交对。

 ## 边缘情况

 对于相等的点，例如`a = [5, 5, 5, 5]`和`k = 2`，每个相邻间隙为零。 基于堆的最小策略重复选择零权重边，并且由于每条边具有相同的成本，因此任何有效的不相交选择都会产生总零。 最大值也与极值配对，但仍然产生零，因此两个输出自然匹配。 

对于紧密聚集的点，例如`[1,2,3,4,100]`和`k = 2`，最大解迫使极端配对，产生像 (1,100) 这样的大贡献，加上剩余点之间的另一个极端配对。 相反，最小解决方案消耗 (1,2) 和 (3,4)，而未使用异常值 100，这正确地避免了总数的膨胀。 

对于大n小k的情况，例如均匀分布的序列，算法仍然只对最小部分进行k次选择，对最大部分进行k次直接计算，因此运行时保持稳定并且独立于不必要的结构。
