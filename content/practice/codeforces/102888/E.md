 ## 问题理解

 我们给出了一组\(n\)被贴上标签的人，以及一组\(m\)游戏。 每场比赛\(i\)有固定的所需团体规模\(a_i\)，以及所有\(a_i\)价值观是不同的。 每个人都必须被准确地分入一组，并且每个组必须由一种游戏类型组成，这意味着所有组都被分配到游戏\(i\)尺寸必须准确\(a_i\)。 一个游戏可以多次使用，因此可能存在许多相同大小的不相交组\(a_i\)。 

如果某人改变了他们玩的游戏，或者一对人在一个组中在一起但在另一个组中分开，则两个分区被认为是不同的。 换句话说，这是一个带标签的分区问题，其中块受到允许的大小的约束，并且每个块都带有由其大小确定的固定标签。 

任务是计算有多少个有效分区以 998244353 为模。 

限制条件\(n, m \le 500\)表明具有 \(O(n^2)\) 或 \(O(nm)\) 动态规划的解决方案是可行的。 任何指数超过\(n\)是不可能的，因为即使是中等的分区数量\(n\)生长速度极快。 立即排除阶乘或子集枚举方法。 

一个关键的微妙之处是组没有排序，并且交换两个相同大小的组不会创建新的解决方案。 这很重要，因为天真的“逐个选择组”方法很容易通过将组顺序视为重要而过度计算。 

一种常见的失败案例是独立处理每一组而不固定对称性。 例如，如果\(n = 4\)允许的尺寸是\(\{2\}\)，那么有效分区只是配对：\(\{\{1,2\}, \{3,4\}\}\),\(\{\{1,3\}, \{2,4\}\}\),\(\{\{1,4\}, \{2,3\}\}\)。 首先选择一对，然后选择另一对而不进行对称处理的简单结构可能会以不同的顺序对同一分区进行多次计数。 

## 方法

 直接蛮力方法尝试构建的所有分区\(n\)标记人员，将他们分成几组，并检查每个组的大小是否符合允许的条件之一\(a_i\)。 这对应于生成所有集合分区，其由贝尔数控制。 即使是为了\(n = 20\)，这变得巨大，并且在\(n = 500\)这是完全不可行的。 

一位人士表示，一旦我们专注于一个突出的元素，结构就变得易于管理\(1\)。 我们不是对称地构建所有组，而是强制每个有效分区由包含 person 的组来描述\(1\)。 该组必须有一定的允许大小\(k\)，一旦我们决定了它的大小，我们就选择剩余的\(k-1\)来自对方的成员\(n-1\)人们。 修复该组后，问题的其余部分变得独立并且具有规模\(n-k\)。 

这一观察结果将问题变成了标准重现\(n\)，其中转换对应于选择包含固定元素的块的大小。 每个选择都会贡献一个二项式系数，用于选择该块的剩余成员，乘以划分剩余元素的方式数量。 

由于每个尺寸\(a_i\)恰好对应于一个游戏并且大小不同，一旦选择了大小，将块分配给游戏就不会产生歧义。

 ## 算法演练

 我们定义\(dp[x]\)作为有效分区方式的数量\(x\)规则下的人。 

1. 初始化\(dp[0] = 1\)。 这表示空集恰好具有一个有效分解。 

2. 适合各种尺寸\(k\)在允许的组大小集中，预先计算二项式系数直至\(n\)。 这些表示一旦群体规模固定后选择群体成员的方式数量。 

3. 对于每个总尺寸\(x\)从\(1\)到\(n\), 计算\(dp[x]\)通过考虑包含人的组\(1\)。 该组必须具有一定的允许大小\(k \le x\)。 

4. 对于每一个这样的\(k\)， 选择\(k-1\)从剩下的人中选出额外的人\(x-1\)，这有助于\(\binom{x-1}{k-1}\)。 其余\(x-k\)人们可以独立划分\(dp[x-k]\)方式，所以我们添加\(\binom{x-1}{k-1} \cdot dp[x-k]\)到答案。 

5. 所有有效的总和\(k\)获得\(dp[x]\)，并计算直到\(dp[n]\)。 

固定人背后的关键思想\(1\)是它消除了群体之间的对称性。 每个分区都有一个包含区分元素的组的唯一表示，因此不会对任何配置进行多次计数。 

### 为什么它有效

 每个有效分区\(x\)elements 恰好包含一组包含 element 的组\(1\)。 该组唯一确定大小\(k\)以及一个选择\(k-1\)同伴。 删除该组后，剩余元素形成大小独立的有效分区\(x-k\)。 相反，任何包含以下内容的组的有效选择\(1\)其余元素的任何有效分区都会生成唯一的完整分区。 这在递归计数的结构和实际有效分区之间建立了一对一的对应关系。 

## Python 解决方案```python
import sys
input = sys.stdin.readline

MOD = 998244353

n, m = map(int, input().split())
a = list(map(int, input().split()))

allowed = set(a)

# precompute binomial coefficients
C = [[0] * (n + 1) for _ in range(n + 1)]
for i in range(n + 1):
    C[i][0] = 1
    for j in range(1, i + 1):
        C[i][j] = (C[i - 1][j - 1] + C[i - 1][j]) % MOD

dp = [0] * (n + 1)
dp[0] = 1

for x in range(1, n + 1):
    total = 0
    for k in allowed:
        if k <= x:
            total += C[x - 1][k - 1] * dp[x - k]
            total %= MOD
    dp[x] = total

print(dp[n])
```代码直接遵循递归。 二项表在\(O(n^2)\)中构建一次，并且DP转换迭代每个状态的所有允许的大小。 减法\(x-k\)对应于删除包含元素的组\(1\)， 尽管\(\binom{x-1}{k-1}\)用于选择该组的其余成员的帐户。 

一个微妙的点是确保每次累加后应用模运算以避免溢出。 另一个是递归总是使用\(x-1\)选择\(k-1\)， 不是\(x\)选择\(k\)，因为一个元素是固定在可区分组内的。 

## 工作示例

 ### 示例 1
 输入：```
3 3
1 2 3
```我们计算\(dp\)步步。 

| x| 考虑 k | 贡献 | dp[x] | dp[x] |
 |---|---|---|---|
 | 0 | - | - | 1 |
 | 1 | 1 | C(0,0)*dp0 = 1 | 1 |
 | 2 | 1,2 | C(1,0)*dp1 + C(1,1)*dp0 = 1 + 1 | 2 |
 | 3 | 1,2,3 | C(2,0)*dp2 + C(2,1)*dp1 + C(2,2)*dp0 = 2 + 2 + 1 | 5 |

 结果是 5，与任何块大小 1 到 3 的分区数量相匹配。 

此跟踪显示每个分区如何由包含元素 1 的块唯一分解，从而防止重复计数。

 ## 边缘情况

 一个关键的边缘情况是只有尺寸\(1\)是允许的。 该算法将每个状态减少为选择单例，并且递归正确地为任何状态生成一个分区\(n\)。 DP总是选择\(k=1\)， 和\(\binom{x-1}{0}=1\)， 所以\(dp[x]=dp[x-1]\)。 

另一种边缘情况是当允许的大小组合不能总和时\(n\)。 在这种情况下，每个\(dp[x]\)最终变为零，除了\(dp[0]\)，因为每个转变都无法完全分解\(x\)。 最终正确答案为零。 

什么时候\(n\)很小但存在多种大小，递归自然会计算所有分区而不重复，因为每个构造都由包含元素的唯一组锚定\(1\)，防止跨组排序的对称过量计数。
