## 简介

 创建辛达语地名是该语言最有价值的应用之一，如果不小心，也是最危险的应用之一。 中土世界同人小说、角色扮演游戏战役和世界建设项目都受益于真实的精灵名字。 本课教您**托尔金自己的构建地名的方法**：元素词汇、两种复合模式、接缝突变以及所有常见的描述性限定词。 

本课程以**20 个完整的名称结构**结束，逐步完成每个步骤，以便您可以自己复制该过程。

 ## 1. 为什么创建辛达语名字？ 

托尔金本人用辛达语（和昆雅语）命名了中土世界的数千个地方。 他并不是随意发明这些名字的——每个名字都遵循一致的语言模式并具有真正的含义。 当托尔金命名*米那斯提力斯*（“守望之塔”）、*卡拉斯加拉东*（“树之堡垒”）或*Henneth Annûn*（“日落之窗”）时，每个元素都贡献了其含义。 

你也可以这样做。 这是托尔金自己意义上的“次创造”——使用真正的语言工具来创造感觉真实的东西，因为它在语言上是真实的。

 ## 2.托尔金的两种复合模式

 辛达语地名的二元复合词使用两种不同的词序：

 ### 模式 A：修饰语 + 头部

 **描述元素排在第一位**，**所描述的内容排在第二位**：

 > [形容词/性质] + [名词/地名] → [修改的地名]

 示例：
 - *Dol Guldur* = *dol*（头，山）+ *guldur*（巫术）=“巫术之山”（描述元素 *guldur* 如下）

 等等 — *Dol Guldur* 实际上是模式 B。让我用更清晰的例子来澄清：

 **模式 A（调整/限定词优先）**：
 - *Minas* Tirith = *minas* (tower) + *tirith* (watch) = "Tower of Watch" — 头在前，然后是修饰语 — 这也是模式 B
 - *Emyn* Muil = 山丘 + grey/confusion = 灰色的令人困惑的山丘

 这些模式在实践中是重叠的。 托尔金使用了**两种命令**，区别有时是：
 - **复合专有词**（无属格）：两个元素直接融合 (*Gondor* = *gond* + *dor*)
 - **属格结构**（带有 *en/i* “of”）：中心名词在前，然后是属格短语（*Minas Tirith* = Tower of the Watch；*Bar-en-Danwedh* = Home of the Ransom）

 ### 模式 B：中心词 + 修饰语（属格）

 > [中心名词] + *en* (of-the) + [属格名词/短语]

 示例：
 - *Bar-en-Danwedh* = 家 + of-the + 赎金 = “赎金之家”
 - *Amon Hen* = hill + of-sight =“Hill of Sight”（没有 *en* 因为 *hen* 直接修改）
 - *Tol-in-Gaurhoth* = 岛 + *in* (of-the) + 狼人宿主 = "狼人宿主之岛"

 介词/属格元素有所不同：
 - *en* =（以下单词的朗读）：*Bar-en-Danwedh*
 - *-in-* =（旧/诗意形式）：*Tol-in-Gaurhoth*
 - 不带粒子的直接复合：*刚铎*、*魔多*、*卡拉德拉斯*

 ### 实用规则

 建立自己的名字时：
 1. **直接复合**（两种元素融合）：无粒子，无接缝突变 — *Gondor*、*Fangorn*
 2. **复合属格**（*en/i* + 软突变）：中心名词 + *en* + 软突变第二个元素

 ## 3. 复合接缝处的突变

 当两个元素加入复合词时，**第二个元素**的声母可能会发生变化。 

### 在直接化合物中

 在托尔金证实的地名中，**接缝处的突变是不一致的**——有些名字显示了这一点，有些则没有。 原因很可能是，在系统地应用突变规则之前，长期建立的名称就已经具体化了，或者复合名称被视为单个词汇单元（并且单个单词不会在内部发生突变）。 

没有突变的例子：*Gondor*（不是*'ondor*），*Fangorn*（不是*Fanghorn* with lenited *o*...）

突变示例：*Barad-dûr* — 连字符表明它们被视为两个词； *dûr* 在属格软突变后将是 *dhûr*，但独立形容词 *dûr*（黑暗）在这里没有突变（它是修饰 *barad* 的形容词）。 

### 在属格结构中

 当使用 *en* + 名词时，以下名词 **确实** 进行软突变：

 > *en* + *tol* → *en dol* （山上；注 *t* → *d*）
 > *i* + *Periannath* → *i Pheriannath* (P → Ph)
 > *en* + *galadh* → *en 'aladh* (g → ' 在软突变下)

 **实用规则**：在 *en* 和 *i* （属格/冠词标记）之后应用软突变。 对于没有粒子的直接化合物，您可以遵循托尔金的不突变做法，或应用软突变以获得更“原生”的感觉 - 但要始终如一且诚实地表明它是 ᴺS。

 ## 6. 逐步名称创建过程

 每次构建新的辛达语地名时，请遵循以下五个步骤：

 ### 第 1 步：确定该地点是什么

 决定**地理类型**——是山、河、森林、堡垒、湖泊、山谷还是山口？ 

选择适当的地理元素：
 - 希尔 → *阿蒙*
 - 河流 → *sîr* / *duin*（大河）
 - 森林 → *eryn* / *taur*（伟大的古老森林）
 - 塔楼/城市 → *米纳斯* / *ost*
 - 通行证 → *cirith*
 - 湖 → *nen* / *lín*
 - 山谷 → *nan* / *imlad*
 - 岛屿 → *托尔*

 ### 第 2 步：确定场所的关键品质

 是什么让这个地方与众不同？ 它的颜色？ 它的方向（北/南）？ 自然特征（风、水、石头）？ 历史事件？ 一个生物？ 

### 步骤 3：选择元素和词序

 - **直接复合**（融合）：*[质量元素] + [地点类型]* — 例如，*Mornnan*（黑暗山谷）
 - **属格结构**（*en* + 软突变）：*[地点类型] + en + [描述]* — 例如，*Nan en Morchond*（黑心谷）

 ### 步骤 4：检查接缝处的突变

 如果使用 *en* （的）：
 - 对以下单词的第一个辅音应用软突变
 - *en* + *galadh* → *en 'aladh*
 - *en* + *barad* → *en varad*
 - *en* + *caras* → *en garas*

 ### 步骤 5：检查语言时期和标签

 - 更喜欢托尔金自己的著作中证实的元素（S.=辛达林语本身）
 - 如果仅使用来自诺尔多林的元素：标记 N. 但注意相似性
 - 所有新建形式：标签ᴺS。 
- 避免在一个名称中混合昆雅语和辛达林语元素（尽管托尔金在一些过渡名称中做到了这一点）

 ## 7. 二十个工作名称结构

 全部ᴺS。 除非表格与经证明的姓名完全匹配。

### 1.“绿色森林”
 - 类型：森林 → *eryn*
 - 品质：绿色 → *calen*（*en* 之后 *c* → *g* 的软突变）
 - 建筑：*Eryn Galen* ᴺS。 
- 注意：托尔金实际上将 Mirkwood *Eryn Lasgalen* =“绿叶之木”命名为“Wood of Green Leaves”——这证实了这一模式； *艾琳·卡伦* ᴺS。 将是“Green Wood”（略有不同）

 ### 2.“黑暗之塔”
 - 类型：塔 → *巴拉德*
 - 品质：深色 → *dûr*
 - 建造：*Barad-dûr* — **已证实**（索伦的堡垒！）

 ### 3.《西塔》
 - 类型：塔 → *米纳斯*
 - 质量：西边 → *annûn*（日落/西边）
 - 建筑：*Minas Annûn* ᴺS。 （西塔/日落）

 ### 4.“红山”
 - 类型：山脉（复数）→ *ered*（*orod* 的复数）
 - 品质：红色 → *caran* （软突变：*c* → *g* 如果与 *en* 一起使用）
 - 建筑：*Ered Garan* ᴺS。 （*c* → *g* 的软突变：*en* + *caran* → *en garan*）
 - 无颗粒的直接化合物：*Ered Caran* ᴺS。 

### 5.《寂静的山谷》
 - 类型：山谷 → *nan*
 - 质量：安静→*dínen*（来自*dîn*）
 - 施工：*Nan Dínen* ᴺS。 （寂静谷）
 - 注意：*Rath Dínen* 已得到证明（“Silent Street”）——使用 *dínen* 得到了很好的支持

 ### 6.“灰河”
 - 类型：河流→ *sîr*（小）或*duin*（大）
 - 品质：灰色 → *史密斯*
 - 建造：*Sîr Mith* ᴺS。 或 *Mithsîr* ᴺS。 
- 比较：*Mithlond*（灰色天堂）使用 *mith-* 作为前缀 — 所以 *Mithduin* ᴺS。 =“灰河”

 ### 7.“白岛”
 - 类型：岛屿 → *tol*
 - 品质：白色 → *nim*（或 *glân*）
 - 建筑：*Tol Nim* ᴺS。 或*Tolnim* ᴺS。 （白岛）

 ### 8.“长传”
 - 类型：pass → *cirith*
 - 质量：长 → *和*
 - 结构：*Cirith 和*ᴺS。 或*Cirith Annen* ᴺS。 

### 9.《星湖》
 - 类型：湖泊 → *nen*
 - 质量：星级 → *elen* / *el*
 - 建筑：*Nen Elenath* ᴺS.？ — 但 *elenath* 是类复数； 使用 *elen* （单星）
 - 更好：*Nen Elent* ᴺS.？ 或者*Elennen* ᴺS.？ 
- 最简单的：*Nenelenath* ᴺS。 = 众星之湖（诗意）
 - 实用：*Nen Elen* ᴺS。 =星湖（使用裸茎）

 ### 10.“北方森林”
 - 类型：森林 → *eryn*
 - 质量：北 → *forod*
 - 施工：*Eryn Forod* ᴺS。 （北方森林）

 ### 11.“铁门”
 - 类型：门 → *annon*（大门）
 - 品质：铁 → *angren*
 - 施工：*Annon Angren* ᴺS。 （铁门）
 - 替补：*Angren Annon* ᴺS。 

### 12.《金色的田野》
 - 类型：字段 → *部分*
 - 品质：金色 → *glaur* / *glor*
 - 结构：*Part Glaur* ᴺS。 或*帕斯·劳尔*ᴺS。 

### 13.“古老的森林”
 - 类型：森林 → *taur*（古老的大森林）
 - 质量：旧 → *iaur*
 - 结构：*Taur Iaur* ᴺS。 =“古老的大森林”
 - 注意：*Iarwain* 的意思是“最古老的”（Bombadil）； *Iaur*单独=老； *Taur Iaur* 是合理的ᴺS。 

### 14.“蜘蛛通道”
 - 类型：pass → *cirith*
 - 品质：蜘蛛 → *ungol*
 - 建造：*Cirith Ungol* — **已证实**（尸罗巢穴通道！）

 ### 15.“黑湖”
 - 类型：湖泊 → *nen* / *lín*
 - 品质：黑色 → *morn* 或 *dûr*
 - 建筑：*Nen Dûr* ᴺS。 或*Môrlín* ᴺS。 (*早上* + *晚上*)

 ### 16.《风之丘》
 - 类型：山 → *阿蒙*
 - 质量：风 → *sûl*
 - 建筑：*Amon Sûl* — **已证实**（Weathertop，佛罗多被刺伤的地方！）

 ### 17.《银河》
 - 类型：河流 → *rant*（路线、河床）或 *duin*
 - 品质：银 → *名人*
 - 建造：*名人* — **已证实**（银脉，洛斯罗瑞恩河！*名人* + *咆哮*）

 ### 18.《泪谷》
 - 类型：山谷 → *nan*
 - 质量：眼泪 → *nîn*（湿/泪）— *nîn* 也是“湿”
 - 属格：*Nan* + *en* + *nîn* → *Nan-en-nîn* ᴺS。 
- 或者直接使用：*Ninnanan* 很尴尬； *Nan Níniel* ᴺS。 =“泪女谷”

 ### 19.“北方要塞”
 - 类型：堡垒 → *ost*
 - 质量：北 → *forod*
 - 属格：*Ost en Forod* ᴺS。 （使用软突变：*forod* 以 *f* 开头 — *f* 没有软突变变化；f 在软突变中保持 f 不变）
 - 或复合词：*Forost* ᴺS。 — 实际上，*Fornost* 已被证明 = *for-*（北）+ *nost*（堡垒）=“诺伯里”

### 20.《星星的天堂》
 - 类型：避风港/港口 → *伦敦*（避风港、内陆避风港）
 - 质量：星星→ *elen*（复数*elin*）； 或所有格*elenath*（所有星星）
 - 属格：*Lond en Elenath* ᴺS。 = 众星之港
 - 或复合词：*Elenlond* ᴺS。

 ## 8. 常见的命名错误

 ### 错误 1：混淆语言句号

 在一个名称中使用昆雅语元素和辛达林语元素（例如，*Calas* Sindarin + *-rin* Quenya）。 解决方案：检查每个元素的来源并保持在一种方言内。 

### 错误 2：忘记 *en* 之后的突变

 写 *Bar-en-Daer* 而不是 *Bar-en-Daer* — 等等，*daer* 以 *d* 开头：*en* + *daer* → *en Naer*（鼻音突变下的 d → n？）。 不——在*en*（属格“of the”）之后，它是**软突变**：*d* → *dh*。 所以 *en* + *daer* → *en Dhaer* ᴺS。 

实际上：属格助词 *en* 触发软突变：b→v，d→dh，g→'（g-drops），m→v，p→b，t→d，c→g。 

所以 *Bar-en-Danwedh* 是正确的： *en* + *danwedh* （赎金）→ *en Danwedh* — 等等，*d* 应该变成 *dh*: *en Dhanwedh*。 但经过验证的形式是 *Bar-en-Danwedh*！ 这说明托尔金在地名中应用突变时并不总是 100% 一致，或者 *d* 被视为 *-an-* 前缀的一部分（不会突变）。 

**课程**：查阅经证明的姓名表格； 如果托尔金没有变异，你可以遵循他的惯例来命名既定的名字，即使它看起来没有变异。 

### 错误 3：使用常规复数 + 后缀

 书写*Gelaidh-orn*（复数+树后缀）而不是单数词干：使用*galadh*（单数）+ *-orn*，而不是复数形式。 

### 错误 4：将 *dûr*（黑暗）与 *dor*（陆地）混淆

 *Barad-dûr* = 黑暗之塔。 “黑暗之地”将是*Mордor*...即*mor-*（黑暗/黑色）+*dor*（土地）——托尔金使用的是*Mordor*，而不是*Durdor*。 合并前检查已证明的用途。 

### 错误 5：过于复杂

 刚开始的命名者试图包含太多元素。 托尔金的名字通常是**两个元素**。 对于真正需要特殊性的名称，应保留三个或更多元素（例如 *Taur-e-Ndaedelos* =“可怕的森林”——三个元素，但每个元素都赢得了自己的位置）。

 ## 需要记住的要点

 1. **两种模式**：直接复合（融合）或属格结构（*en* + 软突变）
 2. **使用地理词汇表**——其中的每个元素都在托尔金的文本中得到证实
 3. ***en***后的突变：对后面单词的声母进行软突变
 4. **两个元素通常就足够了**：不要过于复杂
 5. **将所有内容标记为 ᴺS.** 除非您的名字与经过验证的托尔金形式相匹配
 6. **最好的测试**：你的名字*听起来*辛达林语吗？ 大声朗读。 辛达林语具有特有的流畅品质，具有液体辅音（*l*、*r*、*dh*）和特定的元音模式。
